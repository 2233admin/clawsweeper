## Summary

Keep open. Current main still evaluates the retained `toolCallHistory` without any stale-gap pruning, records new calls by append-and-size-trim only, and the before-tool hook runs detection before recording the new call. The provided GitHub context also shows open unmerged PRs with closing syntax for this issue, so the correct maintainer path is to review/land or close those implementation candidates rather than close the issue now.

## Best Possible Solution

Keep this issue open. Review and either land one focused fix from #40217/#40798 or implement a maintainer-owned fix around `src/agents/tool-loop-detection.ts` and `src/agents/pi-tools.before-tool-call.ts` that prunes or ignores stale history before detection, keeps outcome recording consistent, and adds regression coverage for separated heartbeat cycles plus a continuous-loop control case.
