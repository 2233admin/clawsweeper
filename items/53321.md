## Summary

Keep open. Current main has substantial long-session mitigations, including silent memory flush, threshold-driven compaction, optional successor transcript rotation, queued-followup refresh, and context-overflow reset recovery. It still does not provide the requested first-class, configurable auto session split policy with `session.autoSplit`-style token/window thresholds, explicit rollover semantics, active-tool-call boundaries, and continuity payload handling for active tasks or pending confirmations.

## Best Possible Solution

Keep this issue open as the canonical core product/design tracker. Build on the existing memory flush, preflight compaction, queued-followup refresh, and `truncateAfterCompaction` machinery, then add a supported auto-split policy with threshold config, active tool-call boundaries, session-key routing updates, continuity payload handling for summaries/tasks/pending confirmations, docs, and focused regression tests.
