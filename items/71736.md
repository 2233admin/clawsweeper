## Summary

Keep #71736 open. Current main still has adjacent plugin slash commands and generic plugin approval UI, but no Control UI contribution registry or gateway/UI descriptor projection for plugin-provided chat modes, input guards, event classifiers, or status surfaces. The linked #72287 implementation PR is open and can become the canonical fix if merged, but the requested core SDK/UI contract is not shipped on main yet.

## Best Possible Solution

Keep #71736 open as the focused RFC/API-gap tracker until a main-branch implementation lands. The best path is to review and land #72287 or a narrowed successor that adds a safe data-only Control UI descriptor contract, gateway projection such as plugins.uiDescriptors, host-owned renderer catalogs, enable/disable semantics, sanitization, and fixture coverage for plugin-projected modes, guards, statuses, classifiers, and actions. After that is on main, re-review #71736 for implemented_on_main or close it against the merged PR.
