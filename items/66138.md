## Summary

Keep open. Current main has meaningful partial UX improvements for configuration and ClawHub, including Quick Settings, schema-backed config editing, a Control UI ClawHub skills panel, and CLI/chat-native ClawHub plugin install support. It still does not implement the broader WordPress-like guided dashboard experience or dashboard-native ClawHub plugin discovery/install/configuration requested here.

## Best Possible Solution

Keep this open as a product/UI umbrella or split it into narrower maintainer-owned issues: guided Control UI reconfiguration flows, contextual help for complex config sections, secure dashboard ClawHub plugin discovery/install/configuration, and any additive gateway/plugin manifest/config-schema APIs needed for third-party plugins to expose safe dashboard configuration.
