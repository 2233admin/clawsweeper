## Summary

Keep open. Current main has added controller/status plumbing around WhatsApp listener lifecycle, but the evidence is not strong enough to close this regression: one concrete reported bug still exists (`--channel web` is still emitted in the monitor error path), and the current tests codify 440 conflict as a terminal stop that halts monitoring rather than proving the reported delivery failure is fixed.
