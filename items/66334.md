## Summary

Current `main` still intentionally persists user-selected session model overrides. `/model` and `session_status(model=...)` write a per-session override, docs say that override persists until `model=default`, and tests explicitly assert that user-selected overrides survive across turns. Only auto-failover overrides self-clear back to the configured primary. This issue therefore still describes current behavior and is not safely closable as already fixed or obsolete.
