## Summary

Keep this issue open. Current main already supports Slack channel history and thread reads through the existing `read`/`readMessages` path with user-token preference for reads, but the requested Slack message search action is still absent from discovery, dispatch, runtime implementation, CLI shape, tests, and docs. An open PR, #65691, is the current implementation candidate for the remaining Slack search work and should be reviewed rather than closing this issue now.

## Best Possible Solution

Keep this issue open and narrow the remaining work to Slack message search. The maintainer path is to review and either land or close #65691; if accepted, the implementation should stay Slack-owned, back the generic `search` message action with Slack `search.messages` and `search:read`, require or prefer a user token without enabling writes, advertise the action only when usable credentials exist, update CLI output/options and Slack docs, and add focused tests around discovery, dispatch, runtime token gating, and result formatting. The existing `read`/`readMessages` path can remain the solution for channel history and thread history.
