## Summary

Keep open. Current `main` still has the `nodes list`/`nodes status` inconsistency that PR #50970 is trying to fix, but the PR's unconditional `node.list` fetch would also broaden permissions and drop older-gateway compatibility, so this is not a safe conservative close.

## Evidence

- **Current main still reproduces the mismatch in code:** `nodes status` always reads live nodes from `node.list`, while plain `nodes list` still reads only `node.pair.list` unless `--connected` or `--last-connected` is set. That means a stale pairing list can still make `nodes list` undercount paired live nodes on current `main`.
