## Summary

Keep open. Current `main` already has a dedicated Control UI Sessions view with keyword filtering, bulk selection, and bulk delete that archives transcripts, so the issue’s clutter-management use case is partially addressed. But the requested non-destructive visibility management is still missing: I did not find a `hidden`/archive metadata field or a session-hide/archive action that preserves the session entry while removing it from the list.

## Risks / Open Questions

- Closing as implemented would be premature because delete+archive is not equivalent to hiding/archiving a session entry while preserving it in metadata.
