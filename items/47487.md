## Summary

Keep open. Current `main` still auto-re-enables `exec`/`process` whenever any `tools.exec` section exists, which matches the reported bug and contradicts the documented profile/`alsoAllow` semantics.

## Evidence

  - file: [src/agents/tool-catalog.ts:83](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/tool-catalog.ts#L83)

## Risks / Open Questions

- This is a security-boundary regression: configuring exec safety or host settings currently grants exec visibility to agents that should stay on a restricted profile.
- The same implicit exposure pattern also exists for `tools.fs`, so any eventual fix should check whether filesystem tool visibility has the same unintended privilege expansion.
