## Summary

Keep open. Current main still does not appear to implement the reported LiteLLM onboarding behavior: the bundled LiteLLM auth choice prompts for an API key, preserves an already-configured baseUrl or falls back to localhost, and only honors an explicit baseUrl if it was already present in config. I did not find source evidence that `openclaw onboard --auth-choice litellm-api-key` now prompts for a LiteLLM server URL or live-discovers the user's LiteLLM model list. The related #54872 match is only a LiteLLM title overlap and does not supersede this onboarding regression.
