## Summary

Keep open. Current main still does not expose `sessionModelOverride` or `sessionProviderOverride` on `PluginHookAgentContext`, and `before_model_resolve` still receives only prompt/attachment metadata while hook overrides are applied after the hook context is built. The requested public SDK signal remains missing and is a coherent core/plugin-SDK API gap, not a ClawHub/plugin-only item or a duplicate of the related SDK exposure reports.

## Best Possible Solution

Keep the issue open and address it as a narrow additive Plugin SDK contract change: add optional session override metadata to `PluginHookAgentContext` for `before_model_resolve`, populate it from the resolved session entry before hooks run, document the field in the public hooks/SDK docs, and add a regression test showing a routing hook can yield when a user-pinned `/model` override is active. Consider exposing override source as well if maintainers want plugins to distinguish user pins from auto fallback state.
