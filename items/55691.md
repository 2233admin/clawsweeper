## Summary

Keep open. Current `main` supports explicit abort commands and RPC aborts, but there is no implemented path that turns channel message recall/delete/edit events into task cancellation or context replacement. The only concrete edit/delete channel handling I found on `main` is Slack system-event enqueueing, not run abort.

## Evidence

- **User-facing stop flow is explicit text commands:** The FAQ documents cancellation as sending standalone stop phrases like `stop`, `stop run`, `abort`, or `interrupt`; it does not mention recall/delete/edit events as a cancellation path.
  - file: [docs/help/faq.md:1881](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/help/faq.md#L1881)
  - command: `nl -ba docs/help/faq.md | sed -n '1881,1905p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Abort trigger matcher only recognizes `/stop` and stop-like text:** `isAbortRequestText()` reduces cancellation to `/stop` or the hard-coded stop-phrase set. There is no hook here for message-recall or message-edit event types.
  - file: [src/auto-reply/reply/abort-primitives.ts:5](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/reply/abort-primitives.ts#L5)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Session abort handlers are wired to commands/triggers, not message lifecycle events:** `handleStopCommand()` only fires for `/stop`, and `handleAbortTrigger()` only fires when the raw body matches an abort trigger. No delete/edit event path is handled here.
  - file: [src/auto-reply/reply/commands-session-abort.ts:129](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/reply/commands-session-abort.ts#L129)

- **Core run cancellation exists, but only as explicit RPC/session abort:** `chat.abort` aborts active work when called with a `sessionKey` or `runId`. This proves the cancellation primitive exists, but I found no code on `main` that automatically invokes it from channel recall/edit/delete events.
  - file: [src/gateway/server-methods/chat.ts:1956](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/server-methods/chat.ts#L1956)
