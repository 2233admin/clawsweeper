## Summary

Keep open. Current `main` has explicit user-triggered abort support via `/stop` and stop-like text, but there is no general path that turns channel message edit/delete/recall events into active-run cancellation. Slack edit/delete events are only queued as system events for a later prompt, and Telegram/Discord/Feishu do not wire equivalent inbound edit/delete cancellation handlers on the inspected surfaces.

## Evidence
