## Summary

Current `main` does not implement direct script execution for cron jobs. The public CLI, cron payload types, gateway schema, and isolated-run execution path still only support `systemEvent` and agent-backed `agentTurn` jobs, so this feature request remains open rather than closable.

## Evidence

- **CLI surface has no `--exec` flag:** `openclaw cron add` exposes `--system-event` and `--message` payloads plus model/thinking/tools options, but no direct script/command execution flag.
  - file: [src/cli/cron-cli/register.cron-add.ts:72](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/cli/cron-cli/register.cron-add.ts#L72)

- **CLI enforces exactly one payload kind:** The add command throws unless exactly one of `--system-event` or `--message` is provided, then maps only to `kind: "systemEvent"` or `kind: "agentTurn"`.
  - file: [src/cli/cron-cli/register.cron-add.ts:137](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/cli/cron-cli/register.cron-add.ts#L137)
