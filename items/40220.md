## Summary

Current `main` still appears to have the reported raw JSON log inconsistency. The file transport computes a local-offset top-level `time`, but serializes the original tslog payload unchanged, so `_meta.date` can remain the UTC `Date` serialization. The related PR #48717 correctly described the likely fix, but the provided GitHub context says it was closed unmerged, and the latest release tag still has the same behavior.

## Best Possible Solution

Keep the issue open. Patch `src/logging/logger.ts` so the file transport computes one timestamp string and writes both top-level `time` and serialized `_meta.date` to that same value while preserving other `_meta` fields. Add focused regression coverage in `src/logging/logger-timestamp.test.ts` asserting `_meta.date === time` and that the value uses an explicit offset rather than trailing `Z`. Keep docs aligned with the chosen semantics: current docs make `userTimezone` prompt context, so the narrowest product-consistent fix is host-local file-log consistency, not redefining `userTimezone` as a logging timezone.
