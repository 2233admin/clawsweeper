## Summary

Current main still leaves the reported raw JSON log timestamp mismatch unresolved. The file logger writes a local-offset top-level `time` field, but serializes tslog's `_meta.date` metadata unchanged, so raw JSON readers can still see `_meta.date` as UTC `Z` while `time` uses OpenClaw's local-offset formatter. The related implementation PR was closed unmerged and does not supersede this issue.

## Best Possible Solution

Keep this issue open. Patch `src/logging/logger.ts` so the file-log JSON payload copies `_meta` and serializes `_meta.date` with the same derived local-offset timestamp string used for top-level `time`, without mutating the original tslog object passed to other transports. Add focused coverage in `src/logging/logger-timestamp.test.ts` asserting `_meta.date === time` and the explicit offset format. Treat changing file logs from host-local timezone to `agents.defaults.userTimezone` as a separate product/docs decision.
