## Summary

Keep open. Current `main` does not implement this Feishu gateway-health wiring, and the PR is now stale against the current health-policy contract because readiness/stale-socket checks key off `connected` plus `lastTransportActivityAt`, not the `lastEventAt`-only flow described in the PR body.
