## Summary

Keep #59903 open. Current main does not implement the requested behavior: `/status` now delegates to the shared status renderer, and that renderer still has a `recently finished` path that can render `terminalSummary` for a recent successful completed task. The PR discussion is useful and the branch addressed the review comments, but the change needs to be rebased/adapted to the current shared status surface rather than closed as obsolete.

## Best Possible Solution

Keep the PR open and ask for a rebase/update against current main: move the completed-success suppression into `src/status/status-text.ts` by returning no task line when `activeCount === 0 && recentFailureCount === 0`, preserve active and recent-failure behavior including agent-local recent failures, and add regression tests in `src/auto-reply/reply/commands-status.test.ts`. Maintainers should also decide whether to make `src/agents/tools/session-status-tool.ts` match the documented shared status behavior.
