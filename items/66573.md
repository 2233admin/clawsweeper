## Summary

Keep open. Current `main` still allows `sessions_history` callers to pass `limit > 1000` through to `chat.history`, while Gateway itself validates and caps `chat.history` at 1000. The issue matches current code, and the direct fix appears to exist only in still-open PR #66650, not on `main`.

## Evidence

- **`sessions_history` schema still lacks a 1000 max:** The tool schema only requires `limit >= 1`; it does not advertise the Gateway ceiling, so agent/tool callers can still supply 2000.
  - file: [src/agents/tools/sessions-history-tool.ts:28](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/tools/sessions-history-tool.ts#L28)

  - file: [src/agents/tools/sessions-history-tool.ts:246](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/tools/sessions-history-tool.ts#L246)
