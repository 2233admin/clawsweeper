## Summary

Keep this issue open. Current main still permits `sessions_history` to forward an oversized caller-supplied limit to `chat.history`, while the Gateway protocol schema rejects `limit > 1000` before the handler's internal cap can run. The provided GitHub context also shows open PR #66650 as an unmerged implementation candidate that specifically references this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Review and land PR #66650 or an equivalent focused fix that adds the Gateway maximum to the `sessions_history` tool schema, clamps outgoing `chat.history` requests to 1000 before the Gateway call, and adds regression coverage for both schema metadata and oversized caller-supplied limits. If maintainers reject that PR, close or supersede the PR first and keep this issue tracking the remaining bug.
