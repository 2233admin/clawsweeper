## Summary

Keep open. Current `main` does not show a landed fix for the duplicate plugin-load path, and the latest release still carries the same risk patterns called out in the report and follow-up discussion.

## Evidence

- **Plugin loader cache is still module-scoped:** `registryCache` and `inFlightPluginRegistryLoads` are still plain module-level `Map`/`Set` state, not a process-global singleton. That means cross-chunk module duplication would still bypass the shared cache.
  - file: [src/plugins/loader.ts:220](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/plugins/loader.ts#L220)
