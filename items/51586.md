## Summary

Current `main` does not implement a configurable override for the bare `/new` or `/reset` startup prompt. The reset flow still injects a hardcoded prompt string, and the exposed config/docs only cover `startupContext` prelude sizing/enablement, not replacing that prompt.

## Evidence

- **Hardcoded reset prompt remains:** `BARE_SESSION_RESET_PROMPT_BASE` is still defined as a fixed string and `buildBareSessionResetPrompt()` selects among hardcoded variants before appending time metadata.
  - file: [src/auto-reply/reply/session-reset-prompt.ts:11](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/auto-reply/reply/session-reset-prompt.ts#L11)
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Chat reply path still injects resolved hardcoded prompt:** Bare `/new` and `/reset` handling calls `resolveBareSessionResetPromptState()` and uses `bareResetPromptState.prompt` as the user body for the first turn.
  - file: [src/auto-reply/reply/get-reply-run.ts:358](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/auto-reply/reply/get-reply-run.ts#L358)
