## Summary

Current Mattermost threading on `main` does not have a separate plain-text `replyToMode: "all"` path. `replyToMode` only selects the reply root, outbound delivery stays in markdown mode, and the send path posts the same `message` while only adding `root_id`. The shipped `v2026.4.22` tag contains the same implementation and docs.

## Close Comment

Closing from a Codex review because current OpenClaw already implements this correctly. On `main`, `replyToMode: "all"` only selects the Mattermost reply root in `extensions/mattermost/src/mattermost/monitor.ts:342`; the outbound path still uses markdown chunking in `extensions/mattermost/src/mattermost/reply-delivery.ts:24` and `extensions/mattermost/src/channel.ts:434`, and the actual post send in `extensions/mattermost/src/mattermost/send.ts:395` / `extensions/mattermost/src/mattermost/client.ts:485` keeps the same `message` body and only adds `root_id`. The shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) contains the same behavior, and `docs/channels/mattermost.md:140` now documents `all` as equivalent to `first` for Mattermost. I could not find a current code path where enabling `replyToMode: "all"` strips Markdown formatting, so this report appears already resolved on current OpenClaw. If you still hit it on a newer build, please open a fresh issue with the exact outbound payload, Mattermost server/version, and a minimal repro.
