## Summary

Keep #69386 open. Current main has generic `/stop`/abort plumbing, but it does not fully solve the reported denial-cascade problem: denied async exec approvals still intentionally start an agent follow-up, the tests assert that behavior, and the exact reported Chinese stop phrases are not covered by the stop-command matcher. The related #69570 tracks a major overlapping async-completion cascade, but it does not fully supersede this denial-specific report.

## Best Possible Solution

Keep #69386 open. The best maintainer path is a core runtime fix that treats repeated exec denials as a fail-fast condition, suppresses or safely collapses denied async completion follow-ups, makes stop/abort recognition cover the reported localized phrases, and ensures recognized stop commands clear pending follow-up/session-lane work before queue overflow can hide them. Coordinate the async completion queue work with open #69570, but do not close this issue until the denial-specific behavior has source and regression-test coverage.
