## Summary

Keep this PR open. Current main still accepts the first embedded-agent payload text directly as the slug source, strips edge dashes before truncation, and then lets any non-null slug become the session-memory filename. The PR remains a focused implementation candidate; the merged timeout fix and the broader open #38161 hardening work do not supersede these safeguards unless they merge first.

## Best Possible Solution

Keep this PR open and review or land a focused fix in `src/hooks/llm-slug-generator.ts`: extract/test pure slugification, reject error-marked or non-slug-like LLM payloads before filename construction, and strip leading/trailing dashes after truncation. Coordinate with #38161, but close this PR only after an equivalent merged change ships these safeguards.

## Risks / Open Questions

- The PR's raw-response word/character thresholds are product-sensitive, though the existing prompt asks for a 1-2 word slug and the existing timestamp fallback is appropriate for rejected responses.
- The proposed fix should also consider rejecting payloads where `isError === true`; shape checks catch many provider errors, but a short error string could still look slug-like.
- The broader open #38161 touches session-memory slug hardening, so maintainers should coordinate if both move forward; it should not be treated as superseding this PR until equivalent safeguards merge.
