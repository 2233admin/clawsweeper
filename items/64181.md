## Summary

Keep open. Current main still contains the vulnerable slug pipeline this PR is meant to replace: it slugifies the first embedded-agent payload text without checking `isError`, response shape, word/character limits, or post-truncation dashes. The related merged timeout PR #66455 is present, but it does not address this PR's error-payload or trailing-dash behavior, and the broader #38161 remains open and does not clearly supersede this focused fix.

## Best Possible Solution

Keep the PR open and review it as a focused hooks/session-memory bug fix. The best path is to land a guarded `slugifyLLMResponse`-style helper that rejects error-like or rambling payload text, strips leading/trailing dashes after truncation, and keeps colocated regression tests including the corrected dash-boundary case from the Greptile review. If maintainers prefer broader slug hardening from #38161, explicitly fold these two behaviors and tests into that canonical PR before closing #64181 as superseded.
