## Summary

Keep open. Current main still does not implement the PR's version-aware stale bundled runtime dependency reinstall behavior in the postinstall eager-install path. Adjacent runtime/doctor staging code does validate installed versions, but that is a separate repair path and does not make this focused postinstall PR obsolete. The linked bug #72058 and adjacent #72114 remain open in the supplied context, and the PR discussion contains actionable review feedback rather than evidence for closure.

## Best Possible Solution

Keep PR #72084 or a focused successor open. The best path is to implement version-aware stale dependency detection in `scripts/postinstall-bundled-plugins.mjs`, add regression coverage for stale exact/range versions, preserve the optional native child reinstall test path, and resolve the semver/prerelease review feedback before merge. If maintainers decide postinstall should remain sentinel-only, open or link a canonical runtime/doctor repair tracker before closing this PR as superseded.
