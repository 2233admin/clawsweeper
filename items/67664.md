## Summary

Keep open. Current `main` at `7e52223d327555c611e8c91ee2500897e9eab0bf` still persists fallback context tokens into session-store entries when model-window lookup is unresolved, and `/status` treats persisted `contextTokens` as an explicit override. The PR addresses a real current bug, and the later PR commits appear to answer the review-thread concerns about stale model/provider drift rather than making the change obsolete.
