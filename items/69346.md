## Summary

Keep PR #69346 open. Current main has newer empty-response retry/replay handling and OpenRouter `/v1` self-healing, but it still does not implement the PR’s requested operator-facing diagnostic for zero-payload, zero-token empty-stream configuration failures. The terminal path still falls back to the generic “couldn't generate a response” copy after retries.

## Best Possible Solution

Keep #69346 open. The best path is to rebase it onto current main and integrate the diagnostic after the existing empty-response retry/fallback behavior, preserving current replay and side-effect safeguards while adding provider/model/baseUrl guidance only for explicit zero-usage empty streams. Regression coverage should include nonzero usage, missing usage, side effects, error-stop, OpenRouter, and non-OpenRouter providers.
