## Summary

Keep open. Current main has related zero-token empty-stop retry/replay handling from #71880, but it still ends in the generic incomplete-turn message and does not implement this PR's actionable provider/model configuration diagnostic for empty streams. The PR discussion also shows substantive review feedback that the author addressed, so the branch remains plausibly useful rather than obsolete.

## Best Possible Solution

Keep the PR open and adapt it to current main: preserve the shipped #71880 retry/replay/fallback behavior, then add a narrow actionable diagnostic for proven zero-payload, explicit-zero-usage provider configuration failures, with tests covering usage aliases, missing usage, provider-specific hints, and side-effect/generic fallback paths.
