## Summary

Keep open. Current `main` has recovery tooling for compaction mishaps, but I did not find a landed fix that prevents or reconciles duplicate compaction children under one `parentId` during timeout/overflow retry.

## Evidence

- **Compaction write still comes from the underlying session runtime:** `compactEmbeddedPiSessionDirect()` still delegates the transcript mutation to `activeSession.compact(...)` inside `compactWithSafetyTimeout()`. The surrounding OpenClaw code records checkpoints afterward, but this path does not add any explicit de-duplication, overwrite, or supersede handling for a prior compaction entry on the same parent.
