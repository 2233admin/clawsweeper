## Summary

Keep open. Current `main` still renders TUI chat content as text-only Markdown, drops non-text content blocks during message extraction/stream assembly, and has no terminal-image capability detection or `tui.inlineImages` config/docs. I found no history or docs evidence that inline image rendering for iTerm2/Kitty/Sixel has already landed.

## Evidence

- **TUI message extraction is text-only:** `extractTextBlocks()` and `extractTextFromMessage()` only collect `type: "text"` (plus optional `thinking`) and return strings for rendering; no image block path exists.
