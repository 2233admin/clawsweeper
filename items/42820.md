## Summary

Keep open. Current main still has the reported failure mode for non-empty or true poll-looking fields on `action="send"`; only the narrower zero-valued default mitigation is present. The shared message tool still exposes shared poll creation fields, Feishu still does not advertise poll support, and open PR #46455 remains an unmerged implementation candidate that references this issue with closing syntax.

## Best Possible Solution

Keep the issue open and review/land or replace PR #46455. The durable fix should either sanitize accidental poll creation params before the `send` guard, or narrow the LLM-facing schema so poll creation fields are only visible for poll-capable actions/channels, while preserving `action="poll"` behavior and adding regression coverage for Feishu/Telegram send-with-attachment calls that include auto-populated poll fields.
