## Summary

Keep open. Current `main` does not include the Slack Block Kit table send path described in PR #57717: Slack sends still render markdown tables through the text-only mrkdwn path, and `block` table mode is still coerced back to `code`.

## Evidence

- **Slack send path is still text-only:** `sendMessageSlack` resolves a table mode, renders with `markdownToSlackMrkdwnChunks(...)`, and sends only `text`/`blocks` through `postSlackMessageBestEffort`. There is no table-metadata extraction, no Slack table attachment builder, and no attachments parameter on the post helper.
  - file: [extensions/slack/src/send.ts:100](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/slack/src/send.ts#L100)
  - command: `nl -ba extensions/slack/src/send.ts | sed -n '300,470p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Slack formatter has no table-extraction API:** `extensions/slack/src/format.ts` only exposes `markdownToSlackMrkdwn` and `markdownToSlackMrkdwnChunks`, both built on `markdownToIR(...)`. The PR's proposed `markdownToSlackMrkdwnWithTables()`/`markdownToIRWithMeta(...)` path is not present on `main`.
  - file: [extensions/slack/src/format.ts:121](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/slack/src/format.ts#L121)
  - command: `nl -ba extensions/slack/src/format.ts | sed -n '1,220p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Config still disables block-mode delivery:** `resolveMarkdownTableMode(...)` still returns `code` whenever the resolved mode is `block`, with an inline comment stating Slack send support has not landed yet.
  - file: [src/config/markdown-tables.ts:88](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/config/markdown-tables.ts#L88)
  - command: `nl -ba src/config/markdown-tables.ts | sed -n '1,180p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Tests still assert Slack does not default to block:** Current tests expect `DEFAULT_TABLE_MODES.get("slack")` to be undefined and expect explicit Slack `markdown.tables: "block"` to coerce back to `code`, which is the opposite of the PR’s intended behavior.
