## Summary

Keep open. Current main has the shared markdown table metadata seam, but it still deliberately coerces `markdown.tables: "block"` back to `"code"` at runtime and the Slack send path only posts text or explicit authored blocks. The PR's central send-path work is not implemented on main and is not clearly superseded by the broader Block Kit tracking issue.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a rebased, scoped version that wires `markdownToIRWithMeta()` table extraction into `extensions/slack/src/send.ts`, adds the Slack table attachment helper and tests, preserves safe fallback behavior for multiple tables and media sends, and only then changes Slack's default table mode if that product behavior is still desired.
