## Summary

Keep open. The report is still credible against current main: native Windows gateway service install is still implemented as a Scheduled Task, and the create path still uses ONLOGON/LIMITED plus /RU ... /NP /IT when a task user is available. The fallback is also a per-user Startup-folder login item, not a service-like Windows background service. Current docs describe that same user-login-oriented model and route headless/pre-login Windows guidance through WSL2/systemd, so I do not see strong evidence that the reported native Windows disconnect/session-lifetime problem is fixed or superseded.
