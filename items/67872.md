## Summary

Keep open. Current `main` still does not emit a Slack `message:received` hook from the mention-skip path, so the requested behavior is not implemented. The proposed PR direction also appears contract-mismatched: Slack channel policy is configured under `channels.slack.channels`, while `resolveChannelGroupPolicy()` only reads `groups`, so the new `ingest` lookup described in the PR would not activate for valid Slack configs.

There is no internal hook emission in this path on current `main`.

## Risks / Open Questions

- If this PR were merged as written, the new hook path would likely remain inert for valid Slack configs because the lookup targets the wrong config surface.
- A correct fix likely needs an explicit Slack config contract decision first: either add `ingest` to `SlackChannelConfig` and wire Slack-specific resolution, or choose a different supported surface and document it.
- Current Slack tests cover `channels.slack.channels` mention behavior, but there is no current-main regression test for mention-skip silent ingest comparable to Telegram's coverage.
