## Summary

Keep open. Current `main` does not provide strong evidence that this issue is fully fixed in Control UI/WebChat. The browser Control UI still only accepts `image/*` uploads and serializes every outgoing attachment as `type: "image"`, while gateway-side attachment parsing still drops non-image payloads. That means the broader reported "file/image attachments" path is not fully implemented, and I did not find a Control UI-specific proof that image delivery is fixed end to end.
