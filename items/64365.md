## Summary

Keep PR #64365 open. Current `main` does not implement Slack outbound `replyBroadcast`/`reply_broadcast`, so the PR is not obsolete or already shipped. The feature is still plausibly useful for the bundled Slack plugin, but the PR needs to be rebased and corrected against the current message-tool/schema/docs paths before it can be merged.

## Best Possible Solution

Keep the PR open and ask for an update against current `main`: add a Slack-specific message-tool schema contribution for `replyBroadcast`, thread the boolean through `handleSlackMessageAction`, `handleSlackAction`, `sendSlackMessage`, and `sendMessageSlack`, emit Slack `reply_broadcast` only when a real thread id is present and only once for chunked text, explicitly handle or reject the media-upload case, update the Slack docs example to use the current `message` tool contract, and add focused tests for schema discovery, dispatch forwarding, action runtime forwarding, and Slack payload construction.
