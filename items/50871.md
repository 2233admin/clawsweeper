## Summary

Keep open. Current `main` at `42514156e03c53dec6be6c581069d8c5b2d4cd42` still implements `openclaw nodes list` as a `node.pair.list` view with optional `node.list` filtering, so the reported inconsistency is not implemented away on `main` and the PR is still addressing a live CLI gap.
