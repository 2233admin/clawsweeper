## Summary

Keep open. Current main still lacks this PR's remaining session-key targeting work for #52305: there is no event-session-key remapping helper, cron-run event queues are not aligned with wake/session main-key resolution, and the affected bash exec, node exec, ACP stream, and CLI watchdog call sites still enqueue under the original session key. The PR was recently rebased and slimmed to this narrower scope after #70258 handled the separate reply-route fallback, so it remains useful rather than obsolete.

## Best Possible Solution

Keep #50818 open. Review or land a narrow equivalent for #52305 that remaps ephemeral cron-run event enqueue keys and heartbeat wake keys consistently with `session.mainKey` across bash exec, node exec, ACP stream, and CLI watchdog paths, while preserving the already-shipped #70258 reply-route fallback and current explicit `heartbeat.target: "none"` behavior.
