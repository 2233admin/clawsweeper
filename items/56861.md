## Summary

Keep open. Current `main` does not implement the PR's claimed Signal features end-to-end: outbound sends still pass local file paths directly, mention gating is still regex-only, and quoted-message media is not modeled or injected into the shared media pipeline.

## Evidence

- **Outbound attachments are still sent as local paths:** `sendMessageSignal()` resolves an attachment, then sends `attachments = [resolved.path]` with no data-URI or remote-daemon branch.
  - file: [extensions/signal/src/send.ts:153](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/signal/src/send.ts#L153)
  - command: `nl -ba extensions/signal/src/send.ts | sed -n '153,160p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Mention gating is still regex-based only:** `wasMentioned` still comes only from `matchesMentionPatterns(...)`, and `canDetectMention` is only `mentionRegexes.length > 0`; there is no structured bot-mention detection from `dataMessage.mentions` or reply-to-bot logic on `main`.
  - file: [extensions/signal/src/monitor/event-handler.ts:665](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/signal/src/monitor/event-handler.ts#L665)
