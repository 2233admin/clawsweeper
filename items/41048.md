## Summary

Current `main` does not implement this PR’s requested `getFile() -> null` behavior. The null-return path is still treated as a non-error in Telegram media resolution, the inbound-body code still preserves media placeholders when no media was downloaded, and current tests explicitly lock in that behavior. This PR remains relevant rather than being conservatively closable.
