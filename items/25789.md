## Summary

Keep open. Current `main` does not implement `tools.exec.approvalTimeoutMs` or equivalent config wiring, so this PR is neither already shipped nor obsolete.

## Evidence

- **Config type still lacks the setting:** `ExecToolConfig` has `approvalRunningNoticeMs` and then goes straight to `cleanupMs`; there is no `approvalTimeoutMs` field on current `main`.
  - file: [src/config/types.tools.ts:268](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/config/types.tools.ts#L268)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Runtime still hardcodes the approval timeout:** `buildExecApprovalRequestToolParams()` always sends `timeoutMs: DEFAULT_APPROVAL_TIMEOUT_MS`, and `registerExecApprovalRequest()` falls back to `Date.now() + DEFAULT_APPROVAL_TIMEOUT_MS` instead of any config-derived value.
  - file: [src/agents/bash-tools.exec-approval-request.ts:55](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/agents/bash-tools.exec-approval-request.ts#L55)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Agent config resolution does not read any approval-timeout override:** `resolveExecConfig()` forwards `approvalRunningNoticeMs` from agent/global config, but there is no parallel `approvalTimeoutMs` resolution on current `main`.
  - file: [src/agents/pi-tools.ts:213](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/agents/pi-tools.ts#L213)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Schema and docs expose no approval-timeout field:** The agent runtime schema only declares `approvalRunningNoticeMs`; labels/help likewise include only that exec approval notice field, so users cannot configure an approval timeout through the current config surface.
  - file: [src/config/zod-schema.agent-runtime.ts:474](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/config/zod-schema.agent-runtime.ts#L474)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Main uses a fixed default timeout constant:** The shared exec approval default is currently a fixed `1_800_000` ms constant, confirming the timeout exists but is still global/static rather than configurable per the PR.
  - file: [src/infra/exec-approvals.ts:167](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/infra/exec-approvals.ts#L167)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
