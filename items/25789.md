## Summary

Keep open. Current main no longer has the old 2-minute default, but it still does not expose or wire a configurable exec approval timeout. The PR branch also contains a follow-up wiring commit, so this is stale/rebase work rather than a safe cleanup close.

## Best Possible Solution

The implementation should add the public config type, schema, labels/help/docs, wire the value through global/agent exec config into gateway and node approval registration/wait paths, preserve the current 30-minute default unless intentionally changed, and add focused regression tests for both configured and default timeout behavior.
