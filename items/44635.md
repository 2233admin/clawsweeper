## Summary

Keep #44635 open. Current main has browser profile support, including per-profile CDP URLs and `userDataDir` for `existing-session` profiles, but it does not implement the requested per-agent browser configuration, per-agent default browser profile binding, or per-agent access isolation for profile/CDP settings. The original `agents.list[].browser.userDataDir` shape would still be rejected by the strict agent schema, and managed local browser profiles still derive storage from the profile name rather than a configurable `userDataDir`.
