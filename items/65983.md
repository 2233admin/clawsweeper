## Summary

Keep open. Current main still has the reported process-lifecycle gap: the process supervisor keeps active runs and run records in memory, `reconcileOrphans()` remains a deliberate no-op, process listing is backed by in-memory bash session maps, and the generic background PTY exec path can run without an effective timeout or watchdog. The same author also has #66001 open as an implementation candidate that targets this issue, so this should remain open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Review #66001 as the active implementation candidate. Maintainers should decide whether OpenClaw wants durable startup orphan reaping for PTY exec runs, smaller in-process reconciliation against bash session ownership, or both, then land the chosen fix with focused regression coverage for background PTY exec ownership, process-list visibility, and lost-session cleanup.
