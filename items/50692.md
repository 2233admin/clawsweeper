## Summary

Keep open. Current `main` does not ship Anthropic as a web search provider, and the PR head still has unresolved functional gaps already identified in review: the native-search wrapper is attached to Anthropic requests without checking web-search enable/provider selection or passing Anthropic search config, and onboarding-entered Anthropic keys would still not persist.

## Risks / Open Questions

- Merging the PR as-is would enable Anthropic native web-search injection outside the configured web-search policy, including cases where web search is disabled or another provider is selected.
- Anthropic-specific search settings such as tool version, domain allow/block lists, max uses, and user location would be ignored because the wrapper never receives search config.
- Users who paste an Anthropic key during managed web-search setup could complete onboarding without any credential actually being persisted, leaving the provider effectively unconfigured on the next run.
