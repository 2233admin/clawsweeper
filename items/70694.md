## Summary

Keep open. The report is recent, concrete, and about native `runtime: "subagent"` runs. Current main does pass `task` through the obvious tool and gateway paths, and the stats line splits uncached input from prompt/cache, so the 8-token observation is not enough by itself to prove the task field was dropped. But I did not find a shipped fix, diagnostic, or regression test that covers long `sessions_spawn` task delivery or surfaces a warning when observed input tokens are implausibly low. The ACP-related duplicate context is adjacent but does not cover this native subagent report.

## Best Possible Solution

Keep the issue open and add a focused native subagent regression/diagnostic path: assert a 1.5k-3.5k character `sessions_spawn.task` reaches the gateway `agent` message and embedded prompt submission, then add runtime telemetry or a warning comparing task length with observed first-call input/cache counters. If investigation shows this is only token-accounting confusion, clarify the stats wording/docs and add a test for that display contract.
