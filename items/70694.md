## Summary

Keep open. Current main forwards the `sessions_spawn.task` into both the child system prompt and the initial child agent message, but that only disproves a simple dropped-argument bug. The issue is fresh, reports repeated live subagent failures, and no shipped fix, regression test, diagnostic guard, or canonical duplicate covers the reported subagent-runtime prompt-loss/usage-accounting failure.

## Best Possible Solution

Keep #70694 open. The best maintainer path is a narrow diagnostic/regression pass around native `sessions_spawn`: preserve a task length/hash through tool args, gateway `agent` params, child prompt submission, and completion stats; add a long-task regression test for the subagent runtime; and surface a warning/error when child usage accounting is implausibly low or the initial task is missing from the submitted prompt.
