## Summary

Keep open. Current main’s intended native subagent path does carry the `sessions_spawn` task into the child Gateway `agent` message, and I did not find a protocol schema cap that would obviously truncate a 2k-character task. That is not enough to close this report: the issue is recent, reports concrete live failures for `runtime: "subagent"`, no current test proves long task delivery into the spawned run, and the related ACP initial-task item covers a different runtime. The `in 8 / prompt/cache 40.6k` stats line is not standalone proof because OpenClaw separates uncached input from cache accounting, but the wrong-output behavior remains unaddressed.

## Best Possible Solution

Keep #70694 open. The best next step is a focused diagnostic/regression for `sessions_spawn(runtime: "subagent", mode: "run")` that captures the exact Gateway `agent` payload for a long task and verifies the spawned run receives the full task text. If the bytes-on-wire are correct, investigate provider runtime/cache accounting and add the requested completion warning when observed input tokens are implausibly low relative to the task length.
