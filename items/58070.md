## Summary

Keep open. Current main still does not implement this PR's macOS LaunchAgent defaults: generated plists lack `ProcessType=Interactive`, and `WorkingDirectory` is emitted only when a caller passes one. The provided PR discussion and linked #58061 show a concrete launchd startup regression with focused code and tests rather than obsolete or duplicate cleanup.

## Best Possible Solution

Keep this PR open or adapt its patch into a maintainer branch. The best code path is to update `src/daemon/launchd-plist.ts` so generated macOS LaunchAgent plists always include `ProcessType=Interactive` and a deliberate `WorkingDirectory=/` fallback, then add focused launchd tests covering fresh install, explicit working-directory preservation, and ideally restart-fallback rewrite behavior. If maintainers want one canonical tracker for the overlapping ProcessType reports, keep #58061/#58070 or another explicit canonical item open until the behavior ships.
