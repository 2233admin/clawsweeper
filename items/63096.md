## Summary

Keep this PR open. Current main still forwards gateway `agent` `extraSystemPrompt` unchanged for inter-session completion wakes, while subagent announce paths call gateway `agent` with `inter_session` `task_completion` events and no explicit prompt context. Normal chat still builds the inbound/group prompt sections separately, so the reported completion-wake prompt-context gap remains. The PR is a focused implementation candidate, though it is currently dirty against main and should be refreshed for current inbound metadata/cache-hash behavior before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for the inter-session `task_completion` completion-wake prompt-context gap. Refresh it onto current main, update assertions/implementation for current inbound metadata and cache-hash behavior, preserve explicit `extraSystemPrompt` pass-through, synthesize persisted inbound/group context only when existing session delivery metadata supports it, and run the targeted gateway tests plus `pnpm check:changed` before merge. The linked broader issue should either stay open for heartbeat drift or have the PR closing syntax narrowed to the completion-wake portion.
