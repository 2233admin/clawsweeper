## Summary

Keep open. Current `main` still exposes `deliverOutboundPayloads()` as `Promise<OutboundDeliveryResult[]>`, so an empty array remains ambiguous between hook cancellation and delivery failure, and non-best-effort throws still propagate without any partial-delivery metadata.

## Evidence

- **Return type unchanged on main:** `deliverOutboundPayloads` still returns only `Promise<OutboundDeliveryResult[]>`; there is no outcome wrapper or cancellation/error metadata in the public result shape.
