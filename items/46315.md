## Summary

Keep open. Current `main` does not implement cross-channel reply mirroring for shared sessions; it intentionally routes replies back to the originating channel and only supports cross-channel session sharing, not cross-channel delivery.

## Evidence

- **Cross-channel replies are explicitly prevented:** `resolveAgentDeliveryPlan` documents that `turnSourceChannel` overrides session `lastChannel` specifically to prevent cross-channel reply routing in shared `dmScope="main"` sessions.

## Risks / Open Questions

- Changing this would affect a deliberate safety boundary that avoids accidental cross-channel misdelivery in shared sessions.
- The current docs and schema support cross-channel session continuity (`identityLinks`) but not cross-channel reply fan-out, so the request is still a real feature gap rather than already-shipped behavior.
