## Summary

Current `main` already uses Anthropic's provider-reported `usage.output_tokens` for streamed Claude responses, and downstream transcript/cost code consumes that normalized usage shape. The same implementation is present in the latest shipped release `v2026.4.23`.

## Close Comment

Closing this as already implemented on current `main`.

- Codex review checked `src/agents/anthropic-transport-stream.ts:786` and `src/agents/anthropic-transport-stream.ts:973`: Anthropic streaming usage now copies `usage.output_tokens` directly from the provider on both `message_start` and `message_delta`, then recomputes `totalTokens` and cost from those counters.
- `src/agents/usage.ts:139` still normalizes provider-native `output_tokens` / `total_tokens` into OpenClaw's usage shape, and `src/infra/session-cost-usage.ts:852` consumes persisted `usage.output` / `usage.total` for transcript-backed cost summaries.
- I also verified the same implementation is present in the latest shipped release `v2026.4.23` (`12e84dc571ec2c4274f71b8102d0dc1235cc8579`), so this is not only fixed on unreleased `main`.
- I did not find a remaining code path on this Anthropic transport route that derives billed output from visible reasoning text instead of the provider-reported token counters.
