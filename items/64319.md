## Summary

Keep #64319 open. Current main still routes `openclaw models auth login-github-copilot` through the GitHub Copilot device-flow poller that displays `Waiting for GitHub authorization...`, and there is no current source, docs, tests, or release-history evidence that the WSL2 stuck-after-browser-authorization report has been fixed. The linked #39687 is useful corroborating context, but it was closed without discussion or implementation evidence and is not a canonical superseding tracker.

## Best Possible Solution

Keep the issue open and fix it in the GitHub Copilot provider login path, primarily `extensions/github-copilot/login.ts`. A good maintainer path would add provider-local tests for device-code polling outcomes, improve diagnostics/timeout behavior when the token endpoint keeps returning pending after browser authorization, and document a recovery or manual-token fallback if WSL2 or browser/device-flow state can cause this stall. Do not close as duplicate of #39687 unless a new canonical issue or PR is opened to track the same WSL2 GitHub Copilot auth failure.
