## Summary

Keep #72038 open. Current main still finalizes normal text-only Telegram preview-streamed replies by editing the existing preview message, with no visible-since metadata or long-lived-preview threshold. The PR is a focused bundled Telegram plugin bug fix, and the provided discussion shows reviewer fallback concerns were addressed before the latest head.

## Best Possible Solution

Keep #72038 open for maintainer review. The best path is to land a focused bundled Telegram plugin fix that records when message previews become visible, carries that timestamp through active/superseded/archived preview state, sends a fresh final for previews older than the accepted threshold, cleans up stale previews, preserves short-preview edit-in-place behavior, and keeps the active/archived/send-failure regression tests plus docs/changelog updates.
