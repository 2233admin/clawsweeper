## Summary

Keep open. Current main still finalizes Telegram text-only preview-streamed replies by editing the existing preview message, with no visible-since tracking or long-lived threshold to send a fresh final. The PR addresses a real current Telegram behavior gap and has active review/follow-up evidence, though it should be cleaned up before merge because it includes unrelated non-Telegram changes.

## Best Possible Solution

Keep #72038 open for maintainer review. The product fix belongs in the bundled Telegram plugin: track message-preview visibility time, send a fresh final for long-lived previews, keep short previews editing in place, preserve archived-preview fallback behavior, and land focused regression tests. Before merge, split or justify the unrelated lockfile/plugin-test changes so the PR stays scoped to the Telegram behavior and docs/changelog update.
