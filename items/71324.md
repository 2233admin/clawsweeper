## Summary

Keep #71324 open. Current main still only strips canonical internal-runtime blocks that start at a true line boundary and does not model quote-wrapped legacy rows, legacy background-task status prefixes, gateway restart status blocks, or startup prompt rows in the WebChat/history projection. Related #71229 fixed the newer write path and some legacy projection behavior, while #71312 was an earlier closed/unmerged attempt; neither makes this PR obsolete.

## Best Possible Solution

Keep the PR open and adapt the sanitizer to current main's `projectChatDisplayMessages`/`chat-sanitize` flow. The best implementation should strip only user-role legacy runtime transcript artifacts, preserve following real user text, keep raw transcript sequence accounting on raw rows, and retain the PR's focused regression tests after resolving the bot-review edge cases.
