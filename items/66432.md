## Summary

Keep this PR open. Current main still has the Feishu thread-history media gap: current inbound media is resolved, but fetched thread-history messages are still reduced to parsed text placeholders and formatted without raw Feishu body content or attachment hints. The PR is a focused Feishu channel implementation candidate, not obsolete, duplicate, or superseded.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best maintainer path is to finish the small review items, retain raw Feishu body content for thread-history records, resolve media for media-typed history messages before building `ThreadHistoryBody`, inline runner-compatible attachment hints, add a download-failure regression test, and run the Feishu plugin test lane before merge.
