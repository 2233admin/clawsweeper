## Summary

Keep open. Current `main` still drops `skillsSnapshot.skillFilter` in `resolveSkillsPromptForRun` when it rebuilds the skills prompt from entries, so the reported fix is not implemented on `main` and the PR remains relevant.

## Evidence

- **Rebuild path still ignores snapshot filter:** `resolveSkillsPromptForRun` returns the snapshot prompt if present, but when it falls back to rebuilding from `entries` it calls `buildWorkspaceSkillsPrompt` with `entries`, `config`, and `agentId` only. It does not pass `params.skillsSnapshot?.skillFilter`.
  - file: [src/agents/skills/workspace.ts:821](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/skills/workspace.ts#L821)
  - command: `nl -ba src/agents/skills/workspace.ts | sed -n '800,840p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Prompt builder already supports skillFilter:** `buildWorkspaceSkillsPrompt` accepts `skillFilter`, and `resolveWorkspaceSkillPromptState` applies it through `filterSkillEntries`. This means the omission in `resolveSkillsPromptForRun` is the missing link, not an unsupported feature downstream.
  - file: [src/agents/skills/workspace.ts:745](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/skills/workspace.ts#L745)
