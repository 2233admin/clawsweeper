## Summary

Keep open. Current `main` does not implement the reported Feishu WebSocket reconnect fix: `extensions/feishu/src/client.ts` still constructs the SDK `WSClient` without any reconnect backoff or `PingInterval` guard, `extensions/feishu/src/monitor.transport.ts` has no compensating reconnect logic, and the referenced fix is still an open PR rather than code on `main`.

## Evidence

- **WS client creation has no backoff or PingInterval handling:** `createFeishuWSClient()` passes only credentials, domain, logger level, and optional proxy agent into `new feishuClientSdk.WSClient(...)`. There is no reconnect policy, backoff wrapper, or patch around SDK pong/control handling.
