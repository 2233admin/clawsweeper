## Summary

Keep open. Current `main` still delegates Feishu WebSocket reconnect behavior to `@larksuiteoapi/node-sdk@1.61.1`, and that dependency still uses a fixed 120s reconnect interval, unguarded pong-field destructuring, and the token-cache path noted in the issue comment. The issue discussion is still relevant: the cross-referenced fix PR is not reflected in the checked-out `main` code.

## Evidence

There is no OpenClaw-side exponential backoff, pong guard, or reconnect health check layer here.
