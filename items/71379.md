## Summary

Keep open. Current `main` still lacks `--size`, `--aspect-ratio`, and `--resolution` on `capability image edit`, and it still omits those fields from `image.edit` capability metadata even though the shared image runtime already supports forwarding them. This PR appears to address a real gap rather than duplicating behavior already shipped on `main`.

## Evidence

- **`image.edit` metadata still omits geometry flags on current main:** `CAPABILITY_METADATA` advertises geometry flags for `image.generate`, but `image.edit` still lists only `--file`, `--prompt`, `--model`, `--output`, and `--json`. That means `capability inspect --name image.edit` is still out of sync with the runtime surface the PR describes.
  - file: [src/cli/capability-cli.ts:167](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/cli/capability-cli.ts#L167)
  - command: `nl -ba src/cli/capability-cli.ts | sed -n '152,175p'`
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Shared runtime already supports edit-time geometry overrides:** `runImageGenerate()` accepts `size`, `aspectRatio`, and `resolution` for both `image.generate` and `image.edit`, then forwards them into `generateImage()`. The gap is the CLI wiring, not the underlying runtime contract.
  - file: [src/cli/capability-cli.ts:682](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/cli/capability-cli.ts#L682)
  - command: `nl -ba src/cli/capability-cli.ts | sed -n '682,718p'`
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **`capability image edit` still does not expose or forward the overrides:** The `image.edit` command currently registers only `--file`, `--prompt`, `--model`, `--output`, and `--json`, and its call into `runImageGenerate()` passes only `capability`, `prompt`, `model`, `file`, and `output`. No geometry options are parsed or forwarded on current `main`.
  - file: [src/cli/capability-cli.ts:1395](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/cli/capability-cli.ts#L1395)

- **Current tests do not cover the missing `image.edit` geometry path:** A targeted search on current `main` found no existing `capability-cli` test covering image-edit geometry forwarding or expanded `image.edit` capability-inspect flags, which matches the PR's claim that it is adding new coverage rather than duplicating existing tests.
