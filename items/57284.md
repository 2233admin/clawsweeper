## Summary

Keep open. Current main still routes `openclaw agent --deliver` through an agent turn and only delivers the agent result payloads afterward. If the agent produces no payloads, the delivery path returns success with an empty payload list instead of sending the original `--message`. The linked PR #57300 appears to target this exact bug, but it is open/unmerged and its bypass is not present on main.

## Best Possible Solution

Keep #57284 open and land or adapt PR #57300: for `agent --deliver` without `internalEvents`, bypass the LLM/agent run and send the original `--message` through the existing outbound delivery path, with regression coverage for gateway and local fallback. Preserve the LLM path for internal-event/subagent announcement delivery, and track direct session injection as either part of the same fix or a narrower follow-up.
