## Summary

Keep open. The report is still plausible on current `main`, and I did not find a mainline fix or other strong evidence to close it. Current code still expects `openclaw channels add --channel telegram --token ...` to create an enabled Telegram account, the gateway still auto-starts enabled/configured channel accounts, and Telegram still enters the polling path when no webhook is configured.
