## Summary

Keep open. Current main does not implement this PR’s proactive startup/hourly session-archive sweep or orphaned `.tmp` detection, so there is not strong evidence to close it as already implemented or obsolete. One premise in the PR body is partly stale because the default session maintenance mode on main is now `enforce`, not `warn`, but the cleanup gap for idle gateways and orphaned atomic-write temp files still exists in the inspected code.
