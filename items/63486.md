## Summary

Keep open. This is a recent, concrete Docker/local Synapse regression report tied to v2026.4.9, and current main only proves related Matrix E2EE/decrypted-message and verify-status paths improved, not that the reporter's broader no-response/no-inbound-log symptom is fixed.

## Best Possible Solution

Keep this issue open until a maintainer or Matrix owner reproduces against current main or the next release in a Docker deployment with a local Synapse setup like the reporter described. Capture Matrix gateway startup logs, sync readiness, room encryption status, openclaw matrix verify status output, and whether inbound events reach room.message or room.decrypted_event. If current main fixes it, close as implemented with the specific Matrix decrypted-message/verify-status commits and release evidence; if it still reproduces, fix likely belongs in extensions/matrix around shared-client startup, sync readiness, or encrypted/decrypted event delivery.
