## Summary

Keep PR #69634 open. Current main still has the same layout contract gap the PR targets: the UI renders floating message-level copy/expand actions and has CSS reserved for `.chat-bubble.has-copy`, but `renderGroupedMessage` does not add `has-copy` to the bubble when those actions are present. The PR is a plausible focused fix, though it should be rebased and ideally add the regression test described by related PR #69689.

## Best Possible Solution

Keep the PR or an equivalent maintainer patch alive: add `has-copy` to assistant chat bubbles whenever `canCopyMarkdown || canExpand` is true, rebase onto current main, and add a focused `ui/src/ui/chat/grouped-render.test.ts` regression asserting assistant bubbles with actions receive `has-copy` while bubbles without actions do not.
