## Summary

Keep open. Current main still does not implement the requested generic fallback from `resolveGatewayModelSupportsImages()` to `models.providers[provider].models[].input` when the gateway model catalog misses. The issue discussion's distinction from #61716 remains valid: #61716 is onboarding/UI metadata collection, while this item is the runtime gateway capability check.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the gateway/session model-capability path: make `resolveGatewayModelSupportsImages()` able to consult the normalized OpenClaw config, first keeping the catalog lookup, then falling back to `models.providers[provider].models[]` or a uniquely inferred configured provider for the selected model. Add focused tests for explicit provider, inferred provider, catalog miss plus config hit, and catalog miss plus config text-only. #61716 can remain the complementary onboarding/UI follow-up unless the same change also propagates modality metadata from `agents.defaults.models[]`.
