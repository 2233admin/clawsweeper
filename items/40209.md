## Summary

Keep this PR open. Current main still stores system-event queues under the raw trimmed session key, while heartbeat session resolution canonicalizes agent-scoped session keys to lowercase before peeking/draining events. The PR remains a focused implementation candidate for the linked channel-session heartbeat bug.

## Best Possible Solution

Review and either land this focused PR with the mixed-case lookup and cross-case dedupe regression tests, or adjust it to canonicalize only recognized agent-scoped session keys before merge. The linked user problem is still present on current main, so closing this PR now would drop a plausible fix.
