## Summary

Keep PR #40209 open. Current main still has the raw, case-sensitive system-event queue key path while heartbeat session resolution canonicalizes agent-scoped session keys to lowercase before peeking queued events. The linked issue #34338 was closed stale, but the underlying source-level mismatch remains, and the PR’s regression tests are not present on main.

## Best Possible Solution

Keep PR #40209 open for maintainer review or landing. The best code direction remains the scoped fix proposed by the PR: normalize `requireSessionKey()` to the lowercase canonical form used by session routing, and add focused regression tests for mixed-case channel enqueue/peek and cross-case deduplication. If maintainers prefer a broader session-key normalization cleanup, fold this PR into that work rather than closing it as stale.
