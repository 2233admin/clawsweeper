## Summary

Keep this PR open for maintainer review or rework. Current main still requires `params.context1m: true` for Anthropic Opus/Sonnet 4.6 to resolve to 1M context, while the no-flag path is limited to Claude Opus 4.7. The PR addresses the linked user problem, but as written it remains only a partial fix because the Anthropic request beta-header path still requires the explicit flag and strips the beta for OAuth/legacy-token auth.

## Best Possible Solution

Revise this PR, or replace it with a coordinated fix for #66766, so the context resolver/status/memory-flush limit agrees with the Anthropic request beta-header capability path. Preserve OAuth and legacy-token fallback behavior, update `src/agents/context.test.ts` plus Anthropic stream-wrapper/header tests, and update the Anthropic provider docs if the product decision becomes no-flag 1M for Opus/Sonnet 4.6 API-key auth.
