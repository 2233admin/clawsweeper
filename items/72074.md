## Summary

Keep open. Current main has optional explicit `sessions_spawn.label`, target-agent session keys, `sessions_list` agentId filtering, and Control UI grouping by agent, but it does not implement the requested configurable fallback label template or `{agentId}`/`{taskPreview}` auto-prefix behavior for omitted labels.

## Best Possible Solution

Keep the issue open for a core agent/session config change. The likely implementation is to add a documented `agents.defaults.subagents.labelTemplate` plus optional per-agent override, validate it in types/zod/generated schema/help, apply it only when `sessions_spawn.label` is omitted, support well-defined variables such as target `agentId` and sanitized task preview, and cover native subagent plus any intended ACP behavior with focused tests and docs.
