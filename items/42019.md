## Summary

Keep open. Current main still classifies a successful WebSocket connect followed by a detail-RPC timeout as unreachable, so the reported cold-start false negative remains plausible. PR #69215 improved the probe/status wording and the scope-limited case, but it did not make post-connect timeouts count as reachable recovery.

## Evidence

- **Post-connect timeout still resolves as failure:** `probeGateway()` records connect success at `hello-ok`, then gives follow-up detail RPCs their own timer. If that timer fires, it still settles the probe as `{ ok: false, error: "timeout" }` even though connect already succeeded.
  - file: [src/gateway/probe.ts:284](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/gateway/probe.ts#L284)
  - command: `nl -ba src/gateway/probe.ts | sed -n '272,330p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Gateway probe output uses that failure as 'unreachable':** `isProbeReachable()` only returns true for `probe.ok` or missing-scope failures. The JSON/text writers use that helper for top-level `ok`, per-target `connect.ok`, and exit code 1, so a post-connect timeout still produces a false-negative reachability result.
  - file: [src/commands/gateway-status/output.ts:100](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/commands/gateway-status/output.ts#L100)

  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Current tests intentionally preserve the problematic behavior:** `helpers.test.ts` explicitly says non-scope RPC failures stay unreachable and asserts `isProbeReachable(probe) === false` even when `connectLatencyMs` is set, which matches the reported connect-ok / RPC-timeout case.
  - file: [src/commands/gateway-status/helpers.test.ts:251](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/commands/gateway-status/helpers.test.ts#L251)

  - file: [docs/cli/gateway.md:224](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/cli/gateway.md#L224)
