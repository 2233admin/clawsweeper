## Summary

Keep open. Current main still does not implement the PR's central TUI reconnect UX: generic gateway disconnects remain idle, the offline send path still only says the message was not sent, there is no ctrl+r editor binding, and the TUI backend/GatewayChatClient do not expose a manual reconnect method. The PR discussion shows the contributor addressed several bot-reviewed branch issues, and some adjacent macOS remote-probe work exists on main, but the specific reconnect guidance/retry shortcut remains unshipped.

## Best Possible Solution

Keep the PR open for maintainer review, but ask the contributor to rebase and split/narrow it. The best product path is a focused TUI/gateway change that adds a safe manual reconnect API, wires a visible retry shortcut or command, shows reconnect/restart guidance only when auto-reconnect is actually active, preserves pairing/auth-specific guidance, and includes regression tests for disconnect state and reconnect timer behavior. The macOS remote-probe work should be split into a separate reviewed PR or explicitly owned as a separate surface.
