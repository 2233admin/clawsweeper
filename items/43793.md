## Summary

Keep open. Current main still lacks the central TUI reconnect UX from this PR: generic gateway disconnects still resolve to idle, there is no Ctrl+R editor hook, and the TUI backend/GatewayChatClient still have no manual reconnect contract. The discussion shows active contributor follow-up and relevant bot-review fixes, so this is an implementation candidate, not an obsolete PR.

## Best Possible Solution

Keep this PR open for maintainer review, but ask the contributor to rebase and narrow it. The best path is a focused TUI/gateway change that exposes a safe manual reconnect API, wires a visible retry shortcut or command, shows retry/restart guidance only for reconnectable disconnects, preserves auth/pairing-specific guidance through the current onReconnectPaused/detailCode seam, resets activity status on reconnect, and includes regression tests. Split the macOS remote-probe and diffs-store cleanup work unless a maintainer explicitly wants those surfaces reviewed together.
