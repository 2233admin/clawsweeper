## Summary

Keep open. Current main still does not implement the PR's central TUI reconnect UX: generic gateway disconnects still show idle, there is no Ctrl+R editor binding or public TUI reconnect method, and the disconnect handler only surfaces pairing-specific guidance. The PR is broad and may need maintainer narrowing, but it is not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review, but ask the contributor to rebase and narrow it. The best path is a focused TUI/gateway change that exposes a safe manual reconnect API, wires a visible retry shortcut or command, shows reconnect/restart guidance only when auto-reconnect is actually active, preserves auth/pairing-specific guidance, and includes regression tests. The macOS remote-probe side-effect fix should be split or reviewed as a separate surface.
