## Summary

Keep this PR open. Current `main` does not already carry the requested doc change, and the proposed wording in the PR is not fully correct because `schedule.tz` accepts fixed-offset IANA IDs like `UTC` and `Etc/GMT+5`, not only region/city zones.

## Evidence

- **Current main only documents `tz` as an IANA timezone in the schema field, but the inline cron schedule example is still vague:** `CronScheduleSchema` says `IANA timezone (kind=cron)`, while the user-facing schedule example still shows `"tz": "<optional-timezone>"` with no clarification.
