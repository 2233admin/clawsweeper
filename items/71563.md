## Summary

Keep #71563 open. Current main still routes Telegram `/new` and `/reset` through the normal per-chat/topic sequential lane, while the PR is a focused Telegram-channel fix with tests for the missing control-lane behavior. The related ClawSweeper items concern adjacent reset/session UI behavior, not a canonical replacement for this queue-bypass fix.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent fix on main: add a small shared command-resolution helper in `extensions/telegram/src/sequential-key.ts`, route registered session lifecycle commands `new` and `reset` to `telegram:<chatId>:control`, and keep regression coverage in `extensions/telegram/src/sequential-key.test.ts` plus the existing busy-lane behavior coverage. Do not include `/clear` unless it becomes a registered command.
