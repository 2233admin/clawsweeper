## Summary

Keep open. Current `main` does not implement the proposed Telegram routing change: `getTelegramSequentialKey()` only sends abort text and read-only status commands to the `:control` lane, while `/reset` and `/new` are still registered session commands and therefore remain on the normal per-topic lane. The PR is concrete, recent, and aligned with existing control-lane behavior, so there is not strong evidence for any conservative auto-close reason.
