## Summary

Keep #69513 open. Current main still validates announce cron delivery channels at create/update time, but it does not validate `delivery.accountId` or `delivery.failureDestination.accountId` against configured channel accounts before persisting the job. The linked #69518 appears to be a focused candidate fix, but the provided GitHub context says it was closed unmerged, so it does not supersede or ship this issue.

## Best Possible Solution

Keep the issue open and implement the #69518-style fix on current main: in the gateway cron add/update validation path, resolve the normalized announce channel, enumerate configured account ids for that channel through the existing channel plugin config/account-listing surface, reject unknown `delivery.accountId` and announce-mode `delivery.failureDestination.accountId`, and add focused gateway tests for invalid and valid account ids plus a compatibility case for single-account/env-only setups.
