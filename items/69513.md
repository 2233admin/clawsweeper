## Summary

Current `main` still accepts and persists arbitrary cron `delivery.accountId` values without validating them against configured channel accounts, so the reported bug remains plausible and is not implemented on `main`. The linked cross-referenced PRs do not appear to have landed in the checked-out `main` state.

## Evidence

- **CLI forwards raw account id:** `cron add/create` trims `--account` and writes it directly into `delivery.accountId`; there is no configured-account lookup before the gateway call.
  - file: [src/cli/cron-cli/register.cron-add.ts:195](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/cli/cron-cli/register.cron-add.ts#L195)
  - command: `nl -ba src/cli/cron-cli/register.cron-add.ts | sed -n '185,240p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Gateway validation checks channel, not account:** `cron.add` calls `assertValidCronCreateDelivery`, but that helper only validates announce channel selection via `assertConfiguredAnnounceChannel`; it never checks `delivery.accountId` against channel accounts.
  - file: [src/gateway/server-methods/cron.ts:71](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/gateway/server-methods/cron.ts#L71)
  - command: `nl -ba src/gateway/server-methods/cron.ts | sed -n '1,170p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Normalization preserves arbitrary account ids:** Cron delivery normalization keeps any non-empty `accountId` value and only strips blanks, so an unknown id survives request normalization unchanged.
  - file: [src/cron/normalize.ts:260](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/cron/normalize.ts#L260)
  - command: `nl -ba src/cron/normalize.ts | sed -n '220,340p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Delivery plan and outbound path pass account id through:** The cron delivery plan copies `delivery.accountId` into the resolved plan, and outbound delivery passes that account id straight into the channel handler.
  - file: [src/cron/delivery-plan.ts:54](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/cron/delivery-plan.ts#L54)

  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)

  - file: [src/cli/cron-cli.test.ts:378](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/cli/cron-cli.test.ts#L378)
