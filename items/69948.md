## Summary

Keep PR #69948 open. Current main still does not expose `api.runtime.interSession.send` or its SDK types/docs/tests, while core already has `sessions.send` behavior that this PR is trying to wrap for plugins. This is a real core/plugin SDK seam proposal, not ClawHub-only work, but it still needs maintainer review and the remaining cache-mode concern resolved before merge.

## Best Possible Solution

Keep the PR open and have maintainers review it as a core plugin SDK/gateway seam. The right path is to add the minimal `api.runtime.interSession.send` facade in core with stable SDK types, gateway dispatch/error mapping, plugin-id scoping, cache-key compatibility for explicit and gateway-bound runtimes, generated API baseline updates, docs, and focused runtime/gateway tests.
