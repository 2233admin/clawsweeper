## Summary

Keep PR #69948 open. Current main has the underlying gateway `sessions.send` method, but it still does not expose the proposed plugin-facing `api.runtime.interSession.send` contract, SDK types, runtime binding, docs, or tests. The PR is not obsolete; it remains a plausible narrow core/plugin SDK seam, with one provided review concern still needing maintainer/author follow-up before merge.

## Best Possible Solution

Keep the PR open and continue it as a narrow core/plugin SDK addition: expose `api.runtime.interSession.send` over existing `sessions.send`, preserve plugin-id scoping, map stable runtime errors, document the opaque `messageRef`, update SDK baselines, and fix the cache-mode handling for explicit `runtimeOptions.interSession` before running the relevant changed gates.
