## Summary

Keep open. Current `main` still collapses missing-binary, missing-`oauth2.js`, parse-failure, and other extraction failures into a single generic `Gemini CLI not found` error, so the PR's behavior is not implemented on `main` or in the latest release. The contribution looks stale against the current `extensions/google` layout, but the underlying diagnostic gap remains real.
