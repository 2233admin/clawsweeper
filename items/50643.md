## Summary

Current `main` still contains the reported macOS first-use TLS TOFU bypass, so this PR should stay open. The code still auto-saves and accepts the first certificate it sees for new `wss://` gateways before any `SecTrustEvaluateWithError` check, and that connection path still reaches authenticated node invocation including `system.run`.

## Evidence

- **Current main still accepts and pins first-use certs before trust evaluation:** In `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayTLSPinning.swift:120-144` on `42514156e03c53dec6be6c581069d8c5b2d4cd42`, the `params.allowTOFU` branch saves the presented fingerprint and returns `.useCredential` immediately. `SecTrustEvaluateWithError` only runs afterward, so it does not protect first-use connections.

- The security bug is still present on current `main`; closing this PR would leave the macOS direct-remote `wss://` first-use path unaddressed.
