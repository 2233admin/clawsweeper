## Summary

Keep this issue open. Current main now includes the generic image-tool timeout fix from ca67762b, but I found no source, test, docs, changelog, or history evidence proving the reported NVIDIA NIM Kimi K2.5 image abort is fixed. The exact reported setup did not include a longer image timeout, and current main still defaults image understanding to a 60s abort path, while bundled NVIDIA Kimi remains text-only and untested for image analysis.

## Best Possible Solution

Keep this issue open and add a focused NVIDIA NIM Kimi K2.5 image regression through the configured-provider image path used by Feishu/Telegram. First verify whether current main succeeds with a longer tools.media.image.timeoutSeconds; if it does, document the required config and consider whether the default should change. If it still fails, fix the NVIDIA/Kimi OpenAI-compatible image payload or streaming compatibility, or surface a clear unsupported-provider error.
