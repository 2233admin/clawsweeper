## Summary

Keep #64212 open. Current main has fixed the broad custom-provider image-model registration path, but I did not find source, docs, tests, or release evidence that the NVIDIA NIM Kimi K2.5 image-abort behavior is fixed. The remaining report is recent, provider-specific, and plausible against the current generic image-understanding timeout/AbortController path.

## Best Possible Solution

Keep the issue open and investigate the NVIDIA NIM Kimi image-understanding path directly. A good fix would add a focused regression around a custom `openai-completions` provider at `https://integrate.api.nvidia.com/v1` with `input: ["text", "image"]`, then adjust the provider compatibility, payload shape, or timeout behavior needed for Kimi image requests. If the root cause is only slow inference, document/configure the timeout path for `tools.media.image.timeoutSeconds` and the explicit `image` tool.
