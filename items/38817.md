## Summary

Keep #38817 open. Current main still strips unsupported tool-schema keywords only when model compat declares `unsupportedToolSchemaKeywords` or a provider hook rewrites the schemas, and the bundled Volcengine/Ark/Kimi provider does neither. The related fix PRs #38883 and #46887 were closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Implement provider-owned compatibility for bundled Volcengine/Ark/Kimi models. Prefer adding `unsupportedToolSchemaKeywords` through the Volcengine provider's resolved-model compat path or model definitions, or a provider `normalizeToolSchemas` hook if that better matches the transport family. Add focused regression tests for `volcengine`, `volcengine-plan`/Ark, and Kimi model IDs, plus a negative case proving unrelated providers keep validation keywords.
