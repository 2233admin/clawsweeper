## Summary

Current `main` still discovers the full bundled/global plugin set and builds a manifest registry for every discovered candidate before `tools.allow`/startup scoping is applied. The reported boot-time overhead remains plausible on `af56926e2fc410809fcb0399435787c7fa8daef8`, so this should stay open.

## Evidence

- **Bundled discovery still scans the full stock plugin tree:** `discoverOpenClawPlugins()` unconditionally calls `discoverInDirectory()` on `roots.stock`, and `discoverInDirectory()` walks every directory entry under that tree, reading package metadata and manifest hints for each plugin directory.
  - file: [src/plugins/discovery.ts:1187](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/plugins/discovery.ts#L1187)
  - command: `nl -ba src/plugins/discovery.ts | sed -n '838,960p'; nl -ba src/plugins/discovery.ts | sed -n '1115,1215p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Manifest registry is explicitly not keyed by allow/deny state:** `buildCacheKey()` documents that the manifest registry 'does not depend on allow/deny/entries enable-state', which is direct evidence that `tools.allow` is not intended to narrow manifest-registry construction.
  - file: [src/plugins/manifest-registry.ts:291](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/plugins/manifest-registry.ts#L291)

  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Registry load still parses every discovered manifest:** `loadPluginManifestRegistry()` takes the full discovery result, then loops every candidate and calls `loadPluginManifest()` / `loadBundleManifest()` before building records. There is no early allowlist gate in this loop.
  - file: [src/plugins/manifest-registry.ts:626](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/plugins/manifest-registry.ts#L626)

  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Gateway startup filters after full registry load:** `resolveGatewayStartupPluginIds()` first calls `loadPluginManifestRegistry()` and only then filters `registry.plugins` for configured channels, activation state, and restrictive allowlists.
  - file: [src/plugins/gateway-startup-plugin-ids.ts:243](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/plugins/gateway-startup-plugin-ids.ts#L243)

  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)

  - file: [src/plugins/loader.ts:2133](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/plugins/loader.ts#L2133)
