## Summary

Keep open. Current `main` already has the chat model picker/provider-ref handling this PR wanted, but the main feature in the PR — a top-level `llm` shorthand config that expands into `models.providers` and `agents.defaults.model.primary` — is not implemented on `main` today. That makes `implemented_on_main` too weak, and there is no basis for another close reason.
