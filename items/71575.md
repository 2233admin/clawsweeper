## Summary

Keep #71575 open. It is the canonical Telegram bug for visible intermediate tool/progress output, and current main still intentionally renders `Working…`, tool names, command/progress text, and related status lines into Telegram preview edits by default. The related #71663 was already closed as a duplicate of this issue, not as a shipped fix.

## Best Possible Solution

Keep #71575 open as the canonical tracker. The likely product/code fix belongs in `extensions/telegram/src/bot-message-dispatch.ts` and the shared streaming config contract: make Telegram partial preview hide or strictly sanitize tool metadata by default, possibly making `streaming.preview.toolProgress` opt-in or generic-only, then update `extensions/telegram/src/bot-message-dispatch.test.ts`, `docs/channels/telegram.md`, and `docs/concepts/streaming.md` to match the chosen behavior.
