## Summary

Keep #64384 open. The requested bug fix is not implemented on current main: `runPreflightCompactionIfNeeded` still returns early whenever persisted totals are fresh, before it can compare those totals against the preflight compaction threshold. The PR is a plausible focused bug fix with regression-test intent, and the related open PR #66716 is not enough by itself to auto-close this one without a merged canonical fix or maintainer choice.

## Best Possible Solution

Keep this PR open until a maintainer either lands a focused fix or explicitly chooses another canonical PR such as #66716. The implementation should adjust `runPreflightCompactionIfNeeded` so fresh persisted totals below threshold can still fast-path, but fresh totals at or above threshold fall through to `shouldRunPreflightCompaction`, with regression tests for both fresh-total above-threshold and below-threshold cases.
