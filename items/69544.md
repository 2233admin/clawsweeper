## Summary

Keep open. Current `main` still derives mention regexes from `agents.list[].identity.emoji` and uses them for Matrix group mention gating when `mentionPatterns` are not explicitly overridden. There is no existing Matrix or per-agent config switch to disable emoji-derived mentions, so the requested feature is not implemented on `main`.

## Evidence

- **Emoji still becomes a mention regex by default:** `deriveMentionPatterns` appends `identity.emoji` as a plain escaped regex, and `resolveMentionPatterns` falls back to those derived patterns whenever agent/global `groupChat.mentionPatterns` are not set.
  - file: [src/auto-reply/reply/mentions.ts:24](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/auto-reply/reply/mentions.ts#L24)

  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Matrix consumes the generic mention regex set:** Matrix mention resolution calls the shared `matchesMentionPatterns` helper against the configured mention regexes and treats a regex hit as a mention signal, so an emoji-derived regex affects Matrix routing.
  - file: [extensions/matrix/src/matrix/monitor/mentions.ts:218](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/matrix/src/matrix/monitor/mentions.ts#L218)
