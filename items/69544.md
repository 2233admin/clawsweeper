## Summary

Keep #69544 open. Current main still derives default mention patterns from both agent identity name and emoji, Matrix uses those regexes for room mention gating, and the strict config schemas/types do not expose a channel-level or per-agent emoji-mention toggle. The existing workaround is still to override `mentionPatterns`, which is exactly the gap the issue asks to avoid.

## Best Possible Solution

Add a backwards-compatible opt-out for derived emoji mention patterns, likely in the shared mention derivation path with Matrix channel/account defaults plus a per-agent `groupChat` override if maintainers want that granularity. Update strict schemas, exported config types, generated config metadata/help, Matrix and agent docs, and regression tests showing Matrix room messages containing only the agent emoji no longer trigger when the switch is disabled, while explicit `mentionPatterns` still work.
