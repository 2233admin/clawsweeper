## Summary

Keep #69544 open. Current main still derives mention regexes from `identity.emoji` with a bare escaped emoji pattern, Matrix still feeds those shared regexes into room mention detection, and the Matrix/global/per-agent config schemas do not expose a switch to exclude identity emoji mentions. The provided related items concern Discord mention policy or UI emoji rendering, not a canonical Matrix emoji-mention toggle.

## Best Possible Solution

Keep the issue open and implement a backward-compatible mention-derivation control. The likely code path is `src/auto-reply/reply/mentions.ts` plus Matrix inbound wiring in `extensions/matrix/src/matrix/monitor/handler.ts`; add schema/type support for a Matrix channel-level default and/or per-agent `groupChat` override, update generated config metadata/docs, and add tests proving Matrix can keep name mentions while suppressing identity emoji false positives.
