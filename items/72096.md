## Summary

Keep open. Current main still builds each light-phase diary prompt from the capped set of recent short-term recall entries in the lookback window and appends a new DREAMS.md diary entry whenever the subagent returns text. There is no per-phase last-emitted cutoff or automatic diary-level dedupe that would suppress the repeated light/work-summary block described in #72096. The related #68882 and #70737 context is orthogonal: promotion contamination and cron decoupling do not solve this diary write-path repetition.

## Best Possible Solution

Implement this in memory-core's dreaming diary path: persist a per-workspace/per-phase emitted-input fingerprint or last-emitted cutoff under memory/.dreams, make light narrative generation skip or summarize only entries not already emitted for that phase/window, and add regression tests showing repeated identical light/work-summary input across hourly cycles does not append duplicate DREAMS.md blocks while genuinely new session/daily signals still append.
