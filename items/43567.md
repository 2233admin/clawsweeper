## Summary

Current main still does not implement the requested distinct system/runtime chat presentation. The pre-compaction memory flush path still submits the flush directive as the embedded run prompt without a hidden transcript prompt, chat.history does not reclassify runtime-originated user rows, the Control UI maps system/unknown roles to normal “other” bubbles, and the TUI history loader renders user-role history as user messages. The related #7532 cleanup only covers exec lifecycle suppression, not this broader distinct-rendering request.

## Best Possible Solution

Keep this issue open. The best maintainer path is a generic runtime/system-message presentation contract: tag runtime-originated transcript entries with stable provenance/kind or make internal-only turns use transcriptPrompt: "", project those entries through chat.history with explicit system/runtime metadata, render them as a muted/badged block in the Control UI, and load them through TUI addSystem rather than addUser. Add focused coverage for memory-flush/runtime projection, Control UI rendering, and TUI history loading; related filtering PR work should use provenance rather than prefix-only text matching.
