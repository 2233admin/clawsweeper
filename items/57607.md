## Summary

Keep open. Current main still has the reported Mattermost behavior: `GROUP_POLICY_NOT_ALLOWLISTED` returns before the pending-history helper is defined or called, so non-allowlisted group messages cannot be included in later Mattermost group context. The linked PR #57609 directly tracks the proposed fix but is still open/unmerged in the provided GitHub context, so the original bug should remain open until an equivalent fix lands.

## Best Possible Solution

Keep #57607 open and land or rework the linked Mattermost fix. The implementation should let `GROUP_POLICY_NOT_ALLOWLISTED` group messages reach the route/history setup, record them through `recordPendingHistoryEntryIfEnabled`, then return before response generation; add a focused Mattermost monitor regression test proving history is recorded while no response/control action is triggered.
