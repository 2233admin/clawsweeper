## Summary

Keep #72124 open. Current main still rejects `sk-ant-api03-` Anthropic API keys in the exact `models auth paste-token --provider anthropic` path reported by #72121: the shared validator only accepts `sk-ant-oat01-`, and the paste-token command calls that validator for Anthropic input. The PR is a focused bug fix with relevant tests in its diff, and no current-main implementation or duplicate shipped fix was found in local source/history search.

## Best Possible Solution

Keep #72124 open and either merge it after normal maintainer/CI review or land an equivalent narrow fix in `src/plugins/provider-auth-token.ts` that accepts both `sk-ant-oat01-` and `sk-ant-api03-` for Anthropic paste-token input, with colocated regression coverage for both prefixes and the error message. The linked bug #72121 should close through that shipped fix.
