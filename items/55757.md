## Summary

Keep open. Current `main` does not implement this change, and the underlying stdin/watchdog mismatch still exists for generic child-spawn CLI paths. At the same time, the PR’s Claude-specific rationale is stale against current `main`: bundled Claude now uses the live-session path, which keeps stdin open, so disabling the watchdog purely because `stdin` exists would also weaken the live-session stall protection.

## Evidence

- **Current main still arms the watchdog for stdin-backed child spawns:** `executePreparedCliRun()` always computes `noOutputTimeoutMs` and passes it into both `runClaudeLiveSessionTurn()` and `supervisor.spawn()`, while separately passing `input: stdinPayload`. There is no guard that disables the watchdog when stdin is prewritten/closed.
  - file: [src/agents/cli-runner/execute.ts:329](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/cli-runner/execute.ts#L329)
  - command: `nl -ba src/agents/cli-runner/execute.ts | sed -n '226,490p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Child-spawn adapter closes stdin immediately when `input` is provided:** The supervisor child adapter defaults stdin to `pipe-closed` when `input` is set, writes the payload, and immediately calls `child.stdin.end()`. That matches the bug report’s core premise for non-live child spawns.
