## Summary

Close PR #41710 as already implemented on current main and shipped in v2026.4.23. iMessage outbound delivery now strips `[[reply_to_current]]` / `[[reply_to:<id>]]` via the shared delivery-time directive helper, with committed tests covering both forms, and main also already contains the separate LanceDB retry behavior from this branch.
