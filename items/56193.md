## Summary

Keep open. Current `main` still implements the exact failure this issue reports: `sessions_spawn` exposes ACP-only fields in the shared schema and the executor still rejects `streamTo`/`resumeSessionId` when `runtime!="acp"`. The silent-drop fix discussed in the thread exists only in open PR #68397 and is not on `main`.
