## Summary

Keep open. Current `main` still lacks both requested behaviors: `openclaw models auth paste-token` remains interactive-only, and the built-in `github-copilot` provider still unconditionally exchanges the supplied GitHub token through `/copilot_internal/v2/token` instead of supporting a PAT direct path. The prior PR discussion also matters: the submitted patch was flagged for missing PAT header wiring and incomplete stdin-path coverage, so there is still maintainer work to do rather than a conservative cleanup close.
