## Summary

Keep open. Current `main` still lacks the PR's requested non-interactive Copilot auth path: `openclaw models auth paste-token` has no `--token` option and still always prompts interactively, the bundled `github-copilot` plugin only exposes device login auth, the docs still state that Copilot login requires an interactive TTY, and the Copilot token resolver still always calls the `/copilot_internal/v2/token` exchange instead of handling PATs directly.
