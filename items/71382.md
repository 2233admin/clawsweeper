## Summary

Keep #71382 open. Current main does not implement a WhatsApp channel/account readOnly flag or hard WhatsApp transport-level outbound gate, and the PR remains the active implementation path for #71166. The PR is not obsolete; it has concrete review follow-ups that should be addressed before landing.

## Best Possible Solution

Keep #71382 open and finish it as the implementation for #71166: add typed/schema/generated WhatsApp `readOnly` support at channel and account scope, resolve it per account, pass it into the monitor, enforce it in send helpers and transport closures, suppress WhatsApp socket side effects including presence/read receipts/reactions/polls/composing/pairing replies, preserve pairing request creation without sending a reply, and add focused regression tests before landing.
