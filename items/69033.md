## Summary

Keep PR #69033 open. Current main has already grown a broader persisted `toolResult.details` byte cap, so the PR's persistence half likely needs rebasing/adaptation, but the central false-overflow fix is still not implemented: the preemptive context estimator still adds `toolResult.details` even though docs, provider replay, and compaction paths treat details as non-model-visible metadata. The PR discussion and bot review describe the same narrow reliability issue and do not provide a close/supersede reason.

## Best Possible Solution

Keep the PR open and ask for a rebase/adaptation: preserve the estimator change that excludes `toolResult.details` and the false-overflow regression test, while reconciling or dropping the older persistence hunk in favor of current main's 8 KiB persisted-details sanitizer and its tests.
