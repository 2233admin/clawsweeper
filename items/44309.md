## Summary

Keep #44309 open. Current main still lacks a first-class one-way dispatch mode for `sessions_send`; the existing implementation either runs the normal A2A reply/announce lifecycle or skips it only for a narrow parent-owned ACP child case. Related #39476, #33478, and #8299 are adjacent lifecycle issues, but they do not supersede this request for an explicit dispatch/handoff semantic.
