## Summary

Keep open. Current `main` still does not propagate `NODE_USE_ENV_PROXY` into supervised service environments, so the PR's stated gap is not already implemented and remains plausible on Node 24 setups that depend on env-proxy mode.

## Evidence

- **Current main still omits the requested env key:** `SERVICE_PROXY_ENV_KEYS` on current `main` includes only the standard proxy URL variables and lowercase variants; it does not include `NODE_USE_ENV_PROXY`. `buildCommonServiceEnvironment` then forwards only `sharedEnv.proxyEnv` into the managed service environment, so this flag cannot be preserved today.
