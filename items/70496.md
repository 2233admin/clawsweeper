## Summary

Keep open. Current main still lacks the password-store (`pass`) SecretRef exec example requested by #62730 and proposed by PR #70496. The existing docs only include 1Password, Vault, and sops examples, while the shipped exec-provider contract supports the proposed wrapper shape. The earlier PR #69720 is closed because it was replaced by this reopened PR, so #70496 is the canonical open docs change rather than a duplicate to close.

## Best Possible Solution

Keep #70496 open as the canonical docs PR for the password-store SecretRef exec example. The best path is to address or accept the minor JSON parse robustness review, then merge the docs-only addition into `docs/gateway/secrets.md`; no new OpenClaw core API or runtime change appears necessary.
