## Summary

Keep PR #69212 open. Current main still lacks the PR's OpenAI-completions `encrypted_content` replay behavior and lacks the HTTP `openai-responses`/`openai-codex-responses` `previous_response_id` tool-continuation path; the similar WebSocket implementation does not cover the HTTP transport changed by this PR.

## Best Possible Solution

Keep the PR open and port the still-missing behavior deliberately: add provider/model-gated replay of OpenAI-completions reasoning `encrypted_content` with regression tests for encrypted-only and multi-chunk streams, then add HTTP Responses `previous_response_id` continuation only behind a route capability check or other safe guard. If the behavior is Volcengine-specific, prefer a provider/plugin-local hook if the current provider SDK supports it; otherwise add the narrowest core transport seam with tests.
