## Summary

Keep PR #69212 open. Current main does not implement the PR's two central behaviors: HTTP Responses/Codex `previous_response_id` tool-result continuations, and OpenAI-compatible completions replay of `encrypted_content` for Volcengine/Doubao reasoning summaries. The provided PR discussion also shows substantive review feedback and follow-up commits rather than an obsolete or superseded change.

## Best Possible Solution

Keep the PR open and have a maintainer or contributor port the useful pieces onto current main: gate HTTP `previous_response_id` continuation to native-capable Responses/Codex routes, capture and replay completions `encrypted_content` with robust metadata merging and buffer accounting, and keep the added payload-shape tests. The existing bot review comments should be addressed before merge, especially metadata coalescing and route-capability gating.
