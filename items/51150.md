## Summary

Keep open. Current `main` still appears to have the bug this PR fixes: image-model runtime resolution only does a bare registry lookup, so config-defined/custom providers can still fail with `Unknown model`, and the `nvidia-api` alias handling from the PR is not present on `main`.

## Evidence

- **Image runtime on `main` still does registry-only lookup and throws `Unknown model` on miss:** `resolveImageRuntime()` normalizes the ref, calls `modelRegistry.find(resolvedRef.provider, resolvedRef.model)`, and immediately throws `Unknown model` if that single lookup misses. There is no config/plugin/ad-hoc fallback in current `main`.
