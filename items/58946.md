## Summary

Keep PR #58946 open. Current main still lacks node-unavailable user-facing handling for the linked #58903 scenario, especially the mid-turn path where a mixed fallback error string containing both `rate_limit` and `node invoke timed out` is still formatted as API rate-limit copy. The PR is focused and backed by useful discussion/tests, though the remaining review comments about structured mid-turn classification still need maintainer resolution before merge.

## Best Possible Solution

Keep the PR open and finish review around `src/auto-reply/reply/agent-runner-execution.ts`: classify node invoke `NOT_CONNECTED`/`UNAVAILABLE`/`TIMEOUT` ahead of generic rate-limit copy for both pre-turn and mid-turn failures, prefer structured `FallbackSummaryError.attempts` when available, and keep focused regression tests for the #58903 mixed rate-limit plus node-timeout scenario.
