## Summary

Keep PR #70609 open. Current main does not implement the central Hermes arbiter metadata routing, and it also lacks the generic outbound metadata seam this PR tries to add across gateway send, direct delivery, queue persistence/recovery, and channel plugin outbound context. This is not a clean ClawHub close because the current plugin/channel APIs do not expose caller-provided outbound metadata end-to-end, but the PR branch still needs maintainer rework because the supplied review context shows unresolved review concerns and an unrelated auth-profile commit in the branch.

## Best Possible Solution

Keep the PR open, but ask for a narrower rework: separate the generic outbound metadata contract from Hermes-specific policy, update the gateway schema/server/client artifacts, direct delivery, queue persistence/recovery, and ChannelOutboundContext with focused tests, then move Hermes builder/default topic/bot behavior to plugin-owned or configurable code. Drop the unrelated auth commit before maintainer review.
