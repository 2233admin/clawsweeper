## Summary

Keep open. Current main does not implement the PR's requested outbound metadata path: gateway send rejects opaque metadata, sendMessage/deliverOutboundPayloads do not accept or forward it, channel outbound contexts have no metadata field, and delivery queue recovery cannot preserve it. The PR is not closeable as implemented or duplicate, but it needs maintainer cleanup before merge because the branch has review-identified wiring/test issues, hardcoded Hermes-specific values, and an unrelated auth commit.

## Best Possible Solution

Keep the PR open, but ask for a narrower follow-up: split the unrelated auth commit, make the core change a generic opaque outbound metadata contract through gateway send, sendMessage, deliverOutboundPayloads, queue storage/recovery, and ChannelOutboundContext, and keep Hermes-specific topic/bot mapping in plugin/config code unless maintainers explicitly approve bundling it in core. Add focused tests for gateway, direct delivery, channel adapter visibility, and recovery replay.
