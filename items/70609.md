## Summary

Keep open. Current main does not implement the requested outbound metadata propagation through gateway send, direct delivery, channel outbound context, or delivery queue recovery. The PR is not superseded by the related metadata items in the provided context, but it should not land as-is without maintainer cleanup because it changes public gateway/plugin contracts and includes unrelated branch content plus unresolved review concerns.

## Best Possible Solution

Keep the PR open, but narrow it before review: split unrelated auth changes, design a generic opaque outbound metadata contract, wire it through gateway send schema/generated clients, `sendMessage`, `deliverOutboundPayloads`, `ChannelOutboundContext`, queue storage, and recovery replay, then add focused tests. Hermes-specific topic/bot defaults should live in config or a plugin-owned layer rather than hardcoded core paths.
