## Summary

Current `main` has partial Telegram reaction support, but it does not implement the first-class reaction-trigger workflow requested in #47677. Reactions are received and converted into queued system events, but the handler does not dispatch an agent turn, custom emoji reactions are ignored, and the exposed Telegram config surface has no per-emoji trigger/semantic mapping.

## Evidence

- **Reaction updates are only enqueued as system events:** The Telegram `message_reaction` handler filters added reactions and calls `enqueueSystemEvent(...)` for each one. It does not call the inbound message processing path or any reply dispatch entrypoint.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:864](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/telegram/src/bot-handlers.runtime.ts#L864)
