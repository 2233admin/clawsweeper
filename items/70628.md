## Summary

Keep open. Current main still contains the Telegram DM no-visible-response fallback path that synthesizes `NO_REPLY`, routes it through outbound delivery, and lets direct silent-reply rewrite turn it into visible filler text. The regression test still asserts that unwanted behavior, and open PR #70630 from the same author remains an unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review or land PR #70630, or apply an equivalent narrow Telegram adapter fix on main. The fix should treat successful no-visible-response Telegram direct turns as handled silent no-ops, avoid synthesizing a transport-visible `NO_REPLY`, preserve the real dispatch-error and non-silent-delivery-failure fallback paths, and flip the Telegram dispatch regression test to assert no DM delivery for this case.
