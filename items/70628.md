## Summary

Keep open. Current main still has the Telegram no-visible-response branch that fabricates a `NO_REPLY` fallback and sends it through normal delivery, while direct silent replies still default to visible rewrite. The Telegram regression test still asserts the unwanted DM rewrite behavior, and the provided related PR #70630 remains the narrow proposed fix rather than a merged implementation.

## Best Possible Solution

Keep #70628 open and land or otherwise implement the narrow Telegram adapter fix from #70630: for successful no-visible-response Telegram direct turns, do not synthesize a transport-visible `NO_REPLY`; treat the turn as a handled silent no-op for status reaction cleanup, preserve the real dispatch-error and non-silent delivery-failure fallback paths, and flip the current Telegram dispatch regression test to assert no DM chat message is delivered for this case.
