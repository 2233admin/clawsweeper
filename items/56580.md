## Summary

Current `main` does not implement Discord slash-command description localization. Native command specs and plugin command definitions still expose only a single `description` string, Discord command objects are built from that single field, and the provider still runs Carbon reconcile/deploy on startup. Based on the current code, #56580 should stay open.

## Evidence

- **Native command spec has no localization field:** `NativeCommandSpec` only carries `name`, `description`, `acceptsArgs`, and optional `args`; there is no `descriptionLocalizations` or equivalent field in the core native-command contract.
