## Summary

Current main still matches the reported bug. Manual `/compact` aborts any prior run, then calls `compactEmbeddedPiSession` without an `abortSignal` and without registering the compaction in the active-run registry. The issue is concrete, includes CoClaw downstream impact/workaround context, and has an open unmerged implementation candidate (#66894) with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or replace #66894. The fix should wire manual `/compact` cancellation in `src/auto-reply/reply/commands-compact.ts` by creating an `AbortController`, registering a compacting active-run handle only after any previous run drains, passing `abortSignal` to `compactEmbeddedPiSession`, clearing the handle in `finally`, and adding regression coverage proving `/stop` or `abortEmbeddedPiRun(sessionId)` cancels an in-progress compaction. The gateway `sessions.compact` path should be addressed explicitly, either in the same patch or a linked follow-up.
