## Summary

Current `main` still appears to have the reported `/compact` cancellation gap. The command handler calls `compactEmbeddedPiSession(...)` without wiring an `abortSignal` or registering an active embedded run, while the compaction runtime only honors caller-provided abort signals and the `/stop` path still aborts through the active-run registry.

## Evidence

- **Current /compact callsite still lacks cancel wiring:** `handleCompactCommand` aborts any pre-existing embedded run, then directly calls `runtime.compactEmbeddedPiSession({...})`. The argument object includes session/runtime metadata but no `abortSignal`, and this file does not register a compaction handle with `setActiveEmbeddedRun`.
  - file: [src/auto-reply/reply/commands-compact.ts:119](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/auto-reply/reply/commands-compact.ts#L119)
  - command: `nl -ba src/auto-reply/reply/commands-compact.ts | sed -n '1,240p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Compaction runtime only exposes external cancellation through optional params.abortSignal:** `CompactEmbeddedPiSessionParams` still declares optional `abortSignal`, and `compactEmbeddedPiSessionDirect` passes `params.abortSignal` into `compactWithSafetyTimeout(...)`. If the caller omits that field, there is no external abort signal for the manual compaction path to honor.
