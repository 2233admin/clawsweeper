## Summary

Keep open. Current `main` does not implement this smoke-only UI-build skip: the root Dockerfile still always runs `pnpm ui:build`, and the `install-smoke` workflow does not pass an `OPENCLAW_SKIP_UI_BUILD` build arg. The PR remains a live, repo-scoped optimization rather than an item that can be conservatively closed.
