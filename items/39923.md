## Summary

Keep #39923 open. Current main still implements the numbered config backup ring, not datetime-suffixed backup files. The live write path still calls that numeric maintenance module, tests still assert `.bak.1` through `.bak.4`, and the related implementation PRs in the provided context are not merged.

## Best Possible Solution

Keep the issue open until a datetime-suffix implementation lands or maintainers explicitly decide to preserve the numeric backup ring. A correct fix should update `src/config/backup-rotation.ts` and `src/config/config.backup-rotation.test.ts`, covering bounded pruning, chmod discovery, collision behavior, and legacy `.bak.N` migration deliberately.
