## Summary

Keep open. Current main still recreates `HEARTBEAT.md` for any workspace where it is missing, while the heartbeat runtime and docs intentionally treat a missing file differently from an empty/comment-only file. This PR remains a focused implementation candidate, but it needs a rebase onto the current test layout and a tighter negative fixture before landing.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, move the regression coverage into `src/agents/workspace.test.ts`, tighten the negative fixture to model a fully initialized workspace with only `HEARTBEAT.md` absent, then land this guard or apply the same guarded write directly on main.
