## Summary

Keep open. Current `main` does not include this PR's session-status persistence change: isolated cron pre-run persistence never marks the session `running`, and the completion/abort/error paths still return without writing `done`/`failed`/`timeout` back to the session entry.

## Evidence

- **Pre-run session mutation still omits status:** `markCronSessionPreRun` only writes model/provider/systemSent; it does not set `entry.status = "running"`. The pre-run persist in `run.ts` therefore cannot store a running status.
  - file: [src/cron/isolated-agent/run-session-state.ts:67](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/cron/isolated-agent/run-session-state.ts#L67)
  - command: `nl -ba src/cron/isolated-agent/run-session-state.ts | sed -n '60,90p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Successful finalize path persists before any terminal status is assigned:** `finalizeCronRun` calls `await prepared.persistSessionEntry()` and then branches on abort / payload outcome, but no `sessionEntry.status` assignment appears before the persist or before returning.
  - file: [src/cron/isolated-agent/run.ts:801](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/cron/isolated-agent/run.ts#L801)
