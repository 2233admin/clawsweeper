## Summary

Keep #59441 open. Current main supports `--tz` for full offset-less ISO datetimes such as `2026-03-23T23:00:00`, but it still does not support the issue’s central `HH:MM` / `HH:MM:SS` time-only `--at` input. The active fix PR #70046 remains open and unmerged in the provided GitHub context, so the issue is not implemented or superseded on main.

## Best Possible Solution

Keep #59441 open and review/land PR #70046 or an equivalent narrow fix that adds validated `HH:MM` / `HH:MM:SS` handling in the CLI `parseAt` layer, resolves the date in the supplied IANA timezone, rejects out-of-range times, and adds focused regression coverage for time-only `--at` values.
