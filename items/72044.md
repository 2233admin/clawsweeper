## Summary

Keep open. The issue is recent, concrete, and describes a current DeepSeek V4 thinking/tool replay failure that is not covered by the existing 2026.4.24 placeholder fix. Current main still fills missing assistant tool-call `reasoning_content` with an empty string, and the DeepSeek tests explicitly assert that placeholder behavior rather than replaying actual prior reasoning across long multi-turn tool-call chains.

## Best Possible Solution

Keep the issue open and fix the DeepSeek V4 thinking replay path. The likely product path is to preserve and replay actual prior non-empty DeepSeek `reasoning_content` for assistant tool-call history in multi-turn flows, add a focused regression test that simulates a long `sessions_spawn`/tool-call chain rather than a one-turn placeholder case, and coordinate an upstream pi-ai fix or dependency bump if the transform must live there.
