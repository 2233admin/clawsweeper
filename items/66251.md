## Summary

Keep open. This tracker still has unresolved Codex harness stability scope: #66263 and #68209 are still open, and current main still documents and implements native Codex compaction as start/completion-only without reliable post-compaction token metadata. The cumulative-token projection fix from #64669 is present, but it does not satisfy the remaining native compaction/context-accounting acceptance criteria.

## Best Possible Solution

Keep this tracker open until maintainers resolve or explicitly supersede the remaining Codex compaction/context-accounting follow-ups. The likely implementation path is in extensions/codex/src/app-server/compact.ts plus session/status accounting: capture tokensAfter if Codex app-server exposes it, otherwise persist an explicit unknown/stale post-compaction state, prevent transcript fallback from using pre-compaction usage as fresh, and update focused tests/docs.
