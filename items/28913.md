## Summary

Current `main` does not implement source-based auto-capture filtering for `memory-lancedb`. The requested `captureFilter` config is absent from the schema, and auto-capture still evaluates raw user text with regex triggers without checking `EXTERNAL_UNTRUSTED_CONTENT` wrappers.

## Evidence

- **Config schema has no `captureFilter`:** `memoryConfigSchema` only allows `embedding`, `dreaming`, `dbPath`, `autoCapture`, `autoRecall`, `captureMaxChars`, and `storageOptions`, so the proposed `memory.captureFilter.excludeWrapped` config is not supported on current `main`.
  - file: [extensions/memory-lancedb/config.ts:90](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/extensions/memory-lancedb/config.ts#L90)
  - command: `sed -n '1,260p' extensions/memory-lancedb/config.ts`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Auto-capture logic does not inspect external-content wrappers:** `shouldCapture()` only checks length, recalled-memory tags, generic XML-like content, markdown summary shape, emoji count, and prompt-injection patterns before applying `MEMORY_TRIGGERS`; there is no check for `EXTERNAL_UNTRUSTED_CONTENT` or any configurable wrapper exclusion.
  - file: [extensions/memory-lancedb/index.ts:247](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/extensions/memory-lancedb/index.ts#L247)

  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Agent-end hook feeds user text directly into `shouldCapture()`:** The `agent_end` handler extracts all user text blocks and filters them with `shouldCapture(text, { maxChars: currentCfg.captureMaxChars })`; it does not strip or reject wrapped external content before storage.
