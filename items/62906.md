## Summary

Keep open. Current main has exact duplicate-frame guards and WebChat optimistic-tail cleanup, plus prompt guidance to reduce narration, but the attached report describes semantic repetition across separate tool/approval steps in WebChat with an Omlx/Qwen setup. I did not find source, tests, docs, or history proving that current main fixes that pattern.

## Best Possible Solution

Keep this issue open and request a current-main WebChat reproduction with the raw transcript, gateway `chat` events, agent events, provider/model config, and whether tools or approvals were involved. If the duplicate is UI/history rendering, add a regression around `ui/src/ui/controllers/chat.ts` or `ui/src/ui/app-gateway.ts`; if it is duplicate event delivery, fix the gateway/subscriber idempotency path; if it is model-generated repetition for Omlx/Qwen, tune the WebChat/system prompt or provider guidance and cover it with a focused regression.
