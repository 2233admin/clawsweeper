## Summary

Keep open. Current main appears to avoid the exact fresh-default `./agents/main` config described in the report, but it has not implemented either proposed fix: packaged gateway installs still do not provide a default launchd `WorkingDirectory`, and explicitly configured relative `agentDir` values still resolve through `process.cwd()`. The issue comments' root-cause analysis still matches the current code path.

## Best Possible Solution

Keep the issue open and fix the service/path contract in core. The smallest fix is to set a deterministic `workingDirectory` for packaged gateway installs before writing the LaunchAgent plist, likely in `buildGatewayInstallPlan()` or `resolveGatewayProgramArguments()`. The more robust follow-up is to anchor configured relative `agentDir` values to the config/state directory instead of `process.cwd()`. Add regression coverage for packaged `openclaw gateway install` emitting `WorkingDirectory` and for relative `agentDir` not resolving under `/`.
