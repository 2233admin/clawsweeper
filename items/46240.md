## Summary

Current main already handles the reported Control UI/webchat MEDIA rendering path, and the latest release snapshot contains the same behavior. The remaining `channel=webchat` rejection is intentional: WebChat is an internal UI surface that uses `chat.send`, not the outbound message channel API.

## Close Comment

Codex review: closing as implemented on main. Current OpenClaw already normalizes assistant `MEDIA:` lines into attachment blocks in `ui/src/ui/chat/message-normalizer.ts:172`, and Control UI renders allowed local assistant attachments via the assistant-media path in `ui/src/ui/chat/grouped-render.ts:1117`. There is direct regression coverage for both parsing and rendering in `ui/src/ui/chat/message-normalizer.test.ts:208` and `ui/src/ui/chat/grouped-render.test.ts:850`. The separate `channel=webchat` rejection is intentional in current code: `src/gateway/server-methods/send.ts:118` now returns an explicit internal-only error telling callers to use `chat.send`, and `docs/channels/channel-routing.md:16` documents that `webchat` is not an outbound message channel. The same implementation is present in release `v2026.4.22` (`00bd2cf7`).
