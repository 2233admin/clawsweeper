## Summary

Keep #58737 open. Current main still preserves custom Slack identity on initial sends but does not forward agent username/avatar through the Slack edit paths that the issue reports. The linked PR #58800 is relevant and useful, but it is open/unmerged, so the bug is not implemented on main yet.

## Best Possible Solution

Keep #58737 open as the user-facing regression until #58800 or an equivalent fix lands. The best fix is to extend the Slack edit helper to accept the same custom identity shape as sends, forward it through draft streaming, preview finalization, and status-final update paths, and add regression tests covering send-to-edit identity preservation.
