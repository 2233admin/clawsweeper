## Summary

Keep PR #70046 open. Current main still does not support bare `HH:MM` / `HH:MM:SS` for `cron add --at`, and `--tz` is only applied to full offset-less ISO datetimes. The PR is a focused, active respin of the same bug fix, linked to open issue #59441 and superseding closed unmerged PR #59444.

## Best Possible Solution

Keep #70046 open as the active implementation candidate for #59441. The best path is maintainer review of the focused parser change and regression tests, then either merge it or request targeted changes if the implementation has edge-case issues.
