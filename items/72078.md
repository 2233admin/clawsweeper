## Summary

Keep open. Current main at 6d60b035b4e7c1739f038808e041567448f58060 still has both reported sessions_spawn wrapper bugs: subagent calls with streamTo:"parent" are rejected before dispatch, and model:"default" is passed through as a literal model override. The linked PR #72101 is directly relevant and appears to be the right fix, but it is open/unmerged and the fix is not on main.

## Best Possible Solution

Keep #72078 open until #72101 is merged or an equivalent fix lands on main. The likely implementation should normalize sessions_spawn `model:"default"` to an unset override, ignore ACP-only `streamTo` for `runtime:"subagent"`, and update the sessions_spawn regression tests to assert the new behavior.
