## Summary

Keep open. Current `main` does not implement this PR's requested behavior: non-Azure custom onboarding still hardcodes `input: ["text"]`, and the existing tests explicitly assert that text-only default. The item is still relevant, though it now needs rebasing onto `src/commands/onboard-custom-config.ts` rather than the older `src/commands/onboard-custom.ts` implementation path.

## Evidence
