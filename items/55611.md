## Summary

Current `main` does not fully implement this request. The browser plugin has no config timeout surface, and the browser tool's `open`/`navigate` paths still ignore `timeoutMs` and hit fixed client-side request ceilings.

## Evidence

- **No browser timeout config surface:** The bundled browser plugin declares an empty `configSchema.properties`, so there is no plugin config key for a browser-tool timeout.
  - file: [extensions/browser/openclaw.plugin.json:4](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/browser/openclaw.plugin.json#L4)
  - command: `nl -ba extensions/browser/openclaw.plugin.json | sed -n '1,220p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Browser tool does not forward timeout for page open/navigate:** `open` posts only `{ url }` and `navigate` posts only `{ url, targetId }`; neither branch reads or forwards the top-level `timeoutMs` parameter.
  - file: [extensions/browser/src/browser-tool.ts:524](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/browser/src/browser-tool.ts#L524)
  - command: `nl -ba extensions/browser/src/browser-tool.ts | sed -n '520,652p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Open-tab client still uses a fixed 15s timeout:** Host-path `browserOpenTab()` hard-codes `timeoutMs: 15000`, so a slow page open can still fail without any configurable override.
  - file: [extensions/browser/src/browser/client.ts:200](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/browser/src/browser/client.ts#L200)
  - command: `nl -ba extensions/browser/src/browser/client.ts | sed -n '190,220p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Navigate client still uses a fixed 20s timeout:** Host-path `browserNavigate()` hard-codes `timeoutMs: 20000`, again without taking a tool/config override.
  - file: [extensions/browser/src/browser/client-actions-core.ts:43](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/browser/src/browser/client-actions-core.ts#L43)
  - command: `nl -ba extensions/browser/src/browser/client-actions-core.ts | sed -n '1,120p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Generic gateway timeout support is only partial:** `browser.request` now accepts an optional `timeoutMs`, but that only helps callers that explicitly send it; the browser tool's `open`/`navigate` paths above do not.
  - file: [extensions/browser/src/gateway/browser-request.ts:141](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/browser/src/gateway/browser-request.ts#L141)
