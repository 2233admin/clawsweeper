## Summary

#40786 remains an actionable core CLI feature request. Current main does not expose `--exclude`, `--exclude-file`, `.backupignore`/`.openclawignore`, `--smart-exclude`, or any tar filter/pattern-matching path for `openclaw backup create`. Related open PRs (#40804, #40810, #44288) propose implementations, and Keep My Claw is a useful ClawHub workaround, but none of that has shipped in current main or replaces the native backup CLI gap.

## Best Possible Solution

Keep #40786 open as the canonical native backup exclude-pattern request. The best core path is to evaluate or consolidate the open PR work, then land a tested implementation that adds repeatable `--exclude` and `--exclude-file` support, applies gitignore-style filtering inside planned state/workspace assets, preserves required restore-critical paths by default, records safe manifest-level exclusion stats, and updates `docs.openclaw.ai/cli/backup` plus CLI forwarding/archive behavior tests.
