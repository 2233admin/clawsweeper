## Summary

Current main does not give strong enough evidence to close this. The intended contract is still that `typingMode: "never"` means no typing indicator at all, but there are still direct typing entry points that bypass the normal `resolveTypingMode`/`createTypingSignaler` gating, and there is no Telegram-specific regression proof that `sendChatAction` is skipped in never mode.

## Evidence

- **Docs still promise never means no typing ever:** `docs/concepts/typing-indicators.md` defines `never` as "no typing indicator, ever."
  - file: [docs/concepts/typing-indicators.md:24](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/docs/concepts/typing-indicators.md#L24)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Normal run path resolves configured typingMode and disables typing when mode is never:** `get-reply-run` reads `sessionCfg?.typingMode ?? agentCfg?.typingMode`, and `createTypingSignaler` treats `mode === "never"` as disabled.
  - file: [src/auto-reply/reply/get-reply-run.ts:274](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/auto-reply/reply/get-reply-run.ts#L274)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Typing disable logic exists but is not universal:** `createTypingSignaler` short-circuits all signal methods when `mode === "never"`.
  - file: [src/auto-reply/reply/typing-mode.ts:67](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/auto-reply/reply/typing-mode.ts#L67)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **There is still a direct typing call that bypasses mode gating:** When the inbound body normalizes empty, `get-reply-run` still calls `typing.onReplyStart()` directly if `suppressTyping` is false; that path does not check `typingMode`.
  - file: [src/auto-reply/reply/get-reply-run.ts:417](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/auto-reply/reply/get-reply-run.ts#L417)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Another direct typing start bypass remains in BTW command flow:** `commands-btw` calls `params.typing?.startTypingLoop()` directly, outside `createTypingSignaler` gating.
  - file: [src/auto-reply/reply/commands-btw.ts:54](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/auto-reply/reply/commands-btw.ts#L54)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Telegram still maps typing callbacks to sendChatAction:** Telegram reply dispatch wires typing start to `sendTyping`, and `sendTyping` invokes `sendChatActionHandler.sendChatAction(..., "typing", ...)`. That means any bypassed typing start still reaches Telegram `sendChatAction`.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:729](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/telegram/src/bot-message-dispatch.ts#L729)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **No Telegram-specific never-mode regression proof found:** Searches found generic never-mode coverage in `src/auto-reply/reply/agent-runner.runreplyagent.e2e.test.ts:458` and heartbeat coverage in `src/infra/heartbeat-runner.typing.test.ts:132`, but no Telegram test asserting `sendChatAction` is skipped when `typingMode: "never"`.
  - command: `rg -n "typingMode: \"never\"|sendChatActionSpy\).*not\.toHaveBeenCalled|typingMode.*telegram" extensions/telegram/src/*.test.ts src/**/*.test.ts`
- **401 backoff mitigation exists, but it is not the same as fixing never-mode semantics:** Telegram has a global `sendChatAction` 401 backoff/circuit breaker and tests for suspension after repeated 401s, which mitigates spam but does not prove the config request itself is implemented.
  - file: [extensions/telegram/src/sendchataction-401-backoff.ts:69](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/telegram/src/sendchataction-401-backoff.ts#L69)
