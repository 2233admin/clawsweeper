## Summary

Keep open. Current main has superseded the webhooks half through synchronous registration and request-time SecretRef handling, but the memory-search config-resolution fix is still not implemented: `resolveMemorySearchConfig` continues to use the registered-only memory provider registry and never forwards the active `OpenClawConfig` for primary, fallback, or multimodal provider lookup.

## Best Possible Solution

Keep this draft PR open, but require a rebase and narrowing. Drop the obsolete webhooks changes covered by merged #67941, keep the memory-search work, switch `resolveMemorySearchConfig` to the cfg-aware memory embedding provider runtime resolver for primary, fallback, and multimodal validation lookup, add the focused runtime-config forwarding regression test, and run the changed agent/plugin gates before review.
