## Summary

Keep PR #59695 open. Current main still has the two exact SHA-1 call sites the PR changes, and the runtime fingerprint is actively used to decide whether MCP session runtimes can be reused. This is still a plausible narrow cleanup, not implemented, duplicate, external, or stale-insufficient-info.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrow SHA-256 replacement in `src/agents/pi-bundle-mcp-runtime.ts` and `src/media-understanding/apply.test.ts`, with wording adjusted if needed to avoid claiming all repo-wide SHA-1 use is gone. If maintainers accept the cleanup, run the changed lanes before merge.
