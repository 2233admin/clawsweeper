## Summary

Current main still has the reported gap: Gateway startup loads internal hooks only from `defaultWorkspaceDir`, the hook loader remains single-workspace, and docs still describe workspace hooks as per-agent. This PR is stale against current main and needs rework, but it is not obsolete or superseded.

## Best Possible Solution

Keep this PR open, but require a rebase and scoped rewrite against current main. The best path is a startup-specific multi-workspace hook loading flow wired from `src/gateway/server-startup-post-attach.ts`, using `listAgentWorkspaceDirs`, while preserving current hook policy: workspace hooks remain explicit opt-in, boundary-checked trusted local code, and unable to override bundled, managed, or plugin hooks. Add focused regression tests for an enabled hook in a non-default agent workspace, workspace isolation, failure isolation, and current precedence behavior; then review and land the refreshed PR or replace it with a maintainer-owned patch.
