## Summary

Keep open. Current `main` still intentionally forces TLS for non-loopback manual iOS gateway connections, the existing tests/docs still codify that policy, and the PR leaves a plausible regression around missing `gateway.manual.tls` defaults unresolved. One small build-fix from the PR (`bootstrapToken: nil` in the Share Extension) is already present on `main`, but the actual behavior change is not implemented and is not yet clearly safe.

## Evidence

I found no counter-evidence on `main` that eliminates that concern.

## Risks / Open Questions

- Closing would discard a still-unresolved behavior change: current `main` does not honor explicit non-loopback `tls=false` for manual iOS connections.
- The PR as written appears to weaken an intentional mobile cleartext restriction documented in `docs/gateway/security/index.md` without adding the explicit opt-in path the docs describe.
- The open review concern about missing `gateway.manual.tls` persistence remains plausible and is not disproven by current code.
