## Summary

Keep #71060 open. Current main has hardened the `gateway restart` path, but the reported `gateway stop` drain-over-1s behavior still exists: stop waits only 10 x 100ms before using a bootout fallback that explicitly leaves the LaunchAgent unloaded. Because this is a fresh, concrete macOS LaunchAgent report and the current tests codify the bootout fallback rather than disproving it, none of the conservative close reasons apply.

## Best Possible Solution

Keep the issue open and narrow the maintainer decision: either change macOS `gateway stop` to avoid unloading after a slow graceful drain, increase the stop confirmation budget and/or re-bootstrap after fallback, or explicitly decide that persistent stop may unload and update the issue/docs to scope remaining work to `gateway restart` only. The current restart path already addresses much of the older bootout-without-bootstrap failure mode, so any fix should be targeted at the remaining stop fallback semantics and verified with `src/daemon/launchd.test.ts`.
