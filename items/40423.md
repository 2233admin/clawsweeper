## Summary

Keep #40423 open. Current main still exposes only `OPENCLAW_SHELL` for the exec, ACP client, and TUI local-shell spawned-process paths; it does not implement the PR's requested `OPENCLAW_AGENT_ID` or `OPENCLAW_SESSION_KEY` contract. The PR is still a plausible core runtime change, not obsolete or superseded by the separate `OPENCLAW_MCP_*` CLI harness context.

## Best Possible Solution

Keep the PR open or ask the contributor for a rebase. The best implementation path is to add a shared runtime env resolver for deterministic `OPENCLAW_AGENT_ID` and optional `OPENCLAW_SESSION_KEY`, thread explicit `agentId` through exec direct, PTY, sandbox, gateway approval/safeBin, ACP client, and TUI local-shell paths, and update docs plus focused tests for explicit-agent precedence and session-key fallback.
