## Summary

Keep open. Current `main` still sends the admin success response for `device.pair.remove`, `device.token.rotate`, and `device.token.revoke` before the queued device disconnect runs, and the websocket request path still only revalidates shared-gateway auth sessions. I did not find the PR’s proposed `invalidated` client state, per-RPC device-token recheck, or the new regression tests on `main`, so this is not implemented-on-main and does not look obsolete.
