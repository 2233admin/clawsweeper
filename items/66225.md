## Summary

Keep this PR open. Current main still has the narrow final-tag regexes in both affected paths, so the self-closing `<final/>` leak is not implemented on main. The linked bug #65867 is still the canonical user-facing report, and #69685 was a separate persistence-side PR that closed unmerged rather than superseding this streaming/sanitizer fix.

## Best Possible Solution

Keep this PR open and have a maintainer review the branch. The best path is to land this fix or an equivalent shared final-tag parser with regression coverage for `sanitizeUserFacingText`, streaming with enforcement off, streaming with enforcement on, slash-containing attributes, and malformed-attribute fail-closed behavior; after that lands, #65867 can be closed as implemented.
