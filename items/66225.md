## Summary

Keep open. Current `main` still has the narrow `<final>` regexes in both the streaming subscriber and user-facing sanitizer, while the shared reasoning-tag helper already uses the broader pattern. A local read-only reproduction against `main` shows `<final/>` is still left in output, so this PR is not redundant and the underlying bug remains plausible on current code.

## Evidence
