## Summary

Keep open. Current `main` and the latest shipped release still register Discord reaction listeners unconditionally, so the reported overhead path remains present when guild `reactionNotifications` are all `"off"`.

## Evidence

- **Unconditional reaction listener registration on current main:** `registerDiscordMonitorListeners` always calls `registerDiscordListener(... new DiscordReactionListener(...))` and `registerDiscordListener(... new DiscordReactionRemoveListener(...))` with no guard around all-off guild configs.
