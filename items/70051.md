## Summary

Current main does not contain the PR's proposed suppression. The gateway still turns node `exec.started` / `exec.finished` / `exec.denied` events into system events and immediate `exec-event` heartbeat wakes after only canonicalizing the session key. Existing heartbeat fixes cover adjacent behavior, especially isolated key stability, but they do not remove this self-wake path for session entries marked `heartbeatIsolatedBaseSessionKey`.

## Best Possible Solution

Keep the PR open for maintainer review or implementation. The likely best path is the PR's narrow guard in `src/gateway/server-node-events.ts` immediately after `loadSessionEntry`, plus a focused regression test proving a marked synthetic heartbeat session does not enqueue another system event or request an `exec-event` heartbeat wake, while existing normal exec notification tests keep passing.
