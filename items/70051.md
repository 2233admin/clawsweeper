## Summary

Keep PR #70051 open. Current main still enqueues node `exec.*` lifecycle events and requests an `exec-event` heartbeat wake after canonicalizing the session key, without inspecting `heartbeatIsolatedBaseSessionKey`. The merged #59606 work stabilizes repeated `:heartbeat` suffixes, but it does not remove this redundant node-event feedback path; the related #67059 thread also explicitly scoped away from `server-node-events.ts`.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrow server-node-events guard, with the regression test from the PR, while preserving existing normal-session exec notification behavior and the already-merged isolated heartbeat key-stability behavior from #59606.
