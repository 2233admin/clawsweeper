## Summary

Keep PR #70051 open. Current main still relays node `exec.started` / `exec.finished` / `exec.denied` events from any resolved session through `enqueueSystemEvent(...)` and `requestHeartbeatNow(...)`; it does not inspect `heartbeatIsolatedBaseSessionKey` in `server-node-events`. The merged heartbeat key-stability work on main prevents repeated `:heartbeat` suffix growth after a wake reaches the runner, but it does not suppress the redundant exec-event wake at the gateway node-event source. The PR’s focused regression test is also not present on main.

## Best Possible Solution

Keep the PR open or land an equivalent maintainer patch in `src/gateway/server-node-events.ts`: load the session entry alongside the canonical key, return early for `exec.started` / `exec.finished` / `exec.denied` when the resolved entry has `heartbeatIsolatedBaseSessionKey`, and keep the focused regression test proving synthetic heartbeat-isolated exec events neither enqueue a system event nor request another `exec-event` heartbeat wake.
