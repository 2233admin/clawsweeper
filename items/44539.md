## Summary

Keep open. The current main code still has a plausible Telegram mismatch matching the report: doctor/health formats the top-level channel line from the preferred/default account summary, while status can report OK from any enabled account with a token. With the issue's config shape, a numeric named Telegram account plus a default account sharing the same token can leave the default summary configured=false and still let status show token-ready accounts.
