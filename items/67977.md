## Summary

Keep open. Current `main` still only supports `active-memory` gating by `allowedChatTypes`, so this PR is not implemented on main. The PR discussion also leaves credible unresolved correctness risks: OpenClaw supports multiple direct-message session-key shapes and `:thread:` suffixes, while the proposed chat-id parsing was reviewed as assuming a narrower canonical shape. This is a recent contributor PR with useful, scoped work, so none of the auto-close reasons apply.
