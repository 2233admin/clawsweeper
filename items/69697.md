## Summary

Keep PR #69697 open. Current main still has the reported `undefined` tool-params crash path: `hashToolCall()` forwards `params` into `digestStable()`, `stableStringifyFallback()` returns `stableStringify(value)` directly, and `stableStringify(undefined)` reaches `JSON.stringify(undefined)`, which returns the JavaScript value `undefined`. Passing that into `createHash().update()` throws `ERR_INVALID_ARG_TYPE`. The provided PR is a narrow fix in the implicated file and should be reviewed/merged with regression coverage rather than closed as obsolete.

## Best Possible Solution

Keep the PR open and land an equivalent narrow fix in `src/agents/tool-loop-detection.ts`: make the serializer contract reflect `JSON.stringify(undefined)`, guard fallback/error hashing with a stable string representation, and add regression tests for undefined tool params. This is a real current-main runtime bug, not a ClawHub/core-scope cleanup item or an already-shipped fix.
