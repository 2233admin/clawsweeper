## Summary

Keep PR #69697 open. Current main still has the exact crash path: `hashToolCall()` hashes raw `params`, `stableStringify(undefined)` returns the JS value `undefined`, `stableStringifyFallback()` returns it directly, and `digestStable()` passes it into `createHash().update()`, which throws `ERR_INVALID_ARG_TYPE`. The PR is a focused source fix for a real current-main bug, and the provided review thread supports the same root cause; related title-search items are unrelated null/undefined fixes on other surfaces, not superseding work.

## Best Possible Solution

Keep PR #69697 open and land either this focused fix or an equivalent main-branch patch that makes the stable serialization helpers truthfully handle `undefined` before hashing. Add targeted regression coverage in `src/agents/tool-loop-detection.test.ts` for undefined params, and ideally one wrapper-path assertion showing an undefined tool params value does not crash loop detection.
