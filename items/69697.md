## Summary

Keep PR #69697 open. Current main still has the claimed crash path: `hashToolCall` hashes `params` through `stableStringifyFallback`, but `stableStringify(undefined)` returns the JavaScript value `undefined`, and `crypto.Hash.update(undefined)` throws `ERR_INVALID_ARG_TYPE`. The PR targets the right owner file and addresses the Greptile-reviewed root cause; it should not be closed as obsolete.

## Best Possible Solution

Keep this PR open and land a targeted tool-loop detection fix: make the local `stableStringify` type match `JSON.stringify` reality, ensure `stableStringifyFallback` and `formatErrorForHash` always return strings, and add regression coverage for undefined params. The fix belongs in `src/agents/tool-loop-detection.ts` because this is an agent loop-detection runtime bug, not a ClawHub/plugin boundary issue.

## Evidence

- **Current hashing path still accepts undefined output:** On current main, `hashToolCall` calls `digestStable(params)`, `digestStable` passes `stableStringifyFallback(value)` directly into `createHash("sha256").update(...)`, and `stableStringifyFallback` simply returns `stableStringify(value)` from the try block. For primitive `undefined`, `stableStringify` returns `JSON.stringify(undefined)`, which is not a string.
