## Summary

Keep open. Current main still only deletes Feishu Bitable placeholder rows when `fields` is missing or has zero keys, while this PR addresses the still-missing case where Feishu returns default-but-empty field values. The PR is a focused bundled Feishu plugin bug fix and is not implemented, duplicated, or superseded by the related Feishu Bitable permission/delete-record work.

## Best Possible Solution

Keep this PR open as a plausible focused Feishu plugin bug fix. The best maintainer path is to add narrow regression coverage around the create-app cleanup path or an extracted empty-value predicate, verify meaningful rows are not deleted, then merge the predicate if the tests confirm the intended behavior.
