## Summary

Keep open. Current main only includes the narrower Feishu Schema 2.0 `context.open_chat_id` fallback; it still does not accept the nested `operator.user_id.{open_id,user_id,union_id}` identity shape that this PR targets, and the provided maintainer discussion explicitly called this PR the better base for #71670.

## Best Possible Solution

Keep #71787 open and adapt it on top of current main: add nested Feishu Schema 2.0 identity extraction for `operator.user_id.{open_id,user_id,union_id}`, preserve the existing `context.open_chat_id` fallback and malformed-payload guards, keep the lifecycle regression for the nested identity shape, and update the changelog without duplicating the existing #71670 entry.
