## Summary

Keep PR #40481 open. Current main has native Telegram approval buttons and shared interactive approval actions, but the generic forwarded fallback still emits the old placeholder text instead of ready-to-use `/approve <actual-id> <decision>` commands. The prior ClawSweeper comment is still accurate against current main SHA 8ba82534e6c98410e19b226f3534b285fd75d278.

## Best Possible Solution

Keep this PR open and rework or rebase it so the generic fallback text in `src/infra/exec-approval-forwarder.ts` emits concrete per-decision `/approve <actual-id> ...` commands while preserving existing Telegram native approval buttons and shared interactive approval actions. Use `buildExecApprovalActionDescriptors` or the existing approval command helper so allowed-decision handling stays consistent, and keep coverage for markdown-capable and plain/non-markdown channel text.
