## Summary

Keep this issue open. Current main still lacks a general web_fetch opt-in for localhost, LAN, or private/internal address ranges; it only exposes tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange for the RFC 2544 fake-IP proxy range. The same-author implementation PR #39630 is open and unmerged with closing syntax for this issue, so the best maintainer path is to review, land, revise, or close that PR rather than closing the issue now.

## Best Possible Solution

Keep this issue open while maintainers review #39630 or a replacement implementation. The likely core path is a default-off web_fetch SSRF policy option, preferably with explicit dangerous naming, wired through the strict runtime schema, exported config types, generated config labels/help, runWebFetch guard policy, cache-key isolation, public docs, and SSRF regression tests. If maintainers reject broad private-network access, document that product/security decision and point users to narrower alternatives such as the existing RFC2544 fake-IP policy or plugin-owned service integrations.
