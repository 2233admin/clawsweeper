## Summary

Current `main` already keeps `--json` CLI startup/plugin logs off stdout and writes `openclaw agent --json` payloads through the runtime JSON writer, so this PR’s targeted change is obsolete.

## Close Comment

Closing from a Codex review: current `main` already covers this behavior, so this PR is no longer needed. The CLI now handles JSON-mode stdout hygiene generically in `src/cli/program/preaction.ts:71`, `src/cli/command-execution-startup.ts:36`, `src/cli/command-bootstrap.ts:34`, and `src/cli/plugin-registry-loader.ts:25`, which keeps plugin-loading logs on stderr during JSON-mode startup while restoring normal behavior afterward. `openclaw agent --json` itself writes through `writeRuntimeJson` in `src/commands/agent-via-gateway.ts:157`, and that writer goes to stdout for normal runtimes in `src/runtime.ts:106`. There is also direct regression coverage in `src/cli/program/preaction.test.ts:394`. This behavior is already present in the tagged release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so I’m closing this PR as already implemented on main.
