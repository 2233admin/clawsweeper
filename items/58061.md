## Summary

Keep open. Current main does not implement the requested launchd plist defaults: `ProcessType` is absent from the LaunchAgent template, and `WorkingDirectory` is still conditional rather than defaulting to `/`. The linked PR #58070 directly tracks the proposed fix but is still open/unmerged in the provided GitHub context, so the issue should not be closed as implemented.

## Best Possible Solution

Keep #58061 open until #58070 or an equivalent maintainer-reviewed change lands. The fix should add `ProcessType=Interactive` to generated macOS gateway LaunchAgents, default `WorkingDirectory` to `/` when no explicit directory is provided, and add regression coverage in `src/daemon/launchd.test.ts`; consider extending service audit/doctor repair so existing LaunchAgents can be detected and rewritten to the new defaults.
