## Summary

Keep this PR open. One hunk is already shipped on current main, but the PR is not obsolete: the gateway health snapshot still reads the pinned runtime config, while the provider-normalization and skill-symlink hunks conflict with current provider/auth and path-containment contracts. It should be rebased and split rather than closed or landed as-is.

## Best Possible Solution

Keep this PR open, but request a rebase and split. Drop the already-shipped Feishu hunk; implement the health fix against current readBestEffortConfig with a gateway health regression test; handle any plan-provider context-window issue through the current distinct provider/catalog plus manifest-auth-alias contract; and treat the skill symlink fallback as a separate security-sensitive path-canonicalization change with targeted tests.
