## Summary

Current main still lacks the future `lastRunMs` guard, and heartbeat task mode still uses `isTaskDue` to skip runs as `no-tasks-due`. This PR remains a narrow, relevant implementation candidate with no observed security-sensitive surface in the provided diff.

## Best Possible Solution

Keep this PR open and have a maintainer review or land the narrow equivalent change: treat future heartbeat task `lastRunMs` values as due in `src/auto-reply/heartbeat.ts`, keep focused `isTaskDue` regression coverage for future timestamps, then run the targeted heartbeat tests and `pnpm check:changed` before merging.

## Evidence
