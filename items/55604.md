## Summary

Keep open. Current `main` still has a matching ACP image-media-understanding gap for Feishu: Feishu downloads and stages the inbound image, but the ACP media-understanding path calls `applyMediaUnderstanding` without `agentDir`, while image understanding explicitly requires `agentDir`. The ACP fallback then forwards the raw image attachment instead, which fits the reported symptoms of no MiniMax VLM call plus later image-resize / `[image data removed]` behavior.
