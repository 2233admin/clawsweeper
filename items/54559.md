## Summary

Current main still has the reported behavior: `generic_repeat` is warning-only, warning-level detections do not block tool execution, and `process(action="list")` is not treated as a known poll tool. The related implementation PR is still open/unmerged in the provided GitHub context, so this issue should stay open for maintainer review or a replacement fix.

## Best Possible Solution

Keep this issue open until `main` changes. The best maintainer path is a narrow runtime fix in `src/agents/tool-loop-detection.ts` that evaluates `genericRepeat` against `criticalThreshold` before the warning branch, with updated unit/e2e coverage and docs/config-help alignment. Maintainers should review or replace PR #60248 and separately decide whether `process(action="list")` belongs in known poll/no-progress detection.
