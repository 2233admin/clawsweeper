## Summary

Keep #70466 open. Current main still exits 0 for supervised SIGUSR1/update restart handoff, while Linux systemd markers are classified as supervised. The generated OpenClaw systemd unit now uses Restart=always, which mitigates fresh standard installs, but the PR's requested Restart=on-failure behavior is not implemented and remains a plausible compatibility fix for older/custom systemd units. The linked #70354 report has concrete reproduction and logs, and the PR is focused; it should get maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best code path is to split spawned and supervised restart handling so spawned restarts still exit 0, launchd keeps its delay, and non-launchd supervised restarts exit 1, with an added regression test for the Linux/non-launchd supervised branch. If maintainers consider `Restart=always` the complete product boundary, close manually with that explicit rationale instead of automated cleanup.
