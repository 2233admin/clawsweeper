## Summary

Keep open. Current main still takes the supervised gateway restart path and exits with code 0, so the PR's requested behavior is not implemented on main. The standard generated systemd unit now uses Restart=always, which mitigates the reported default-install failure, but it does not make this PR obsolete for older/custom systemd units or other Restart=on-failure supervised deployments.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to update the patch on current main, preserve spawned restarts as exit 0, make supervised restarts exit non-zero after the existing launchd delay, and add a Linux/systemd supervised regression test that asserts exitProcess(1) without the launchd delay. Maintainers should also decide whether the existing Restart=always systemd unit policy is sufficient for standard installs or whether the runtime exit-code hardening should still land for custom and older Restart=on-failure units.
