## Summary

Do not auto-close PR #59670. Current main still has the documented behavior gap on the tools overview page: owner-only tool filtering is real in source/tests, but `docs/tools/index.md` does not explain why `cron`, `gateway`, and `nodes` disappear for non-owner senders or how `commands.ownerAllowFrom` fixes it. The PR remains useful, but its patch needs maintainer/author cleanup before merge.

## Best Possible Solution

Keep the PR open or replace it with an updated docs patch. The best fix is a concise `docs/tools/index.md` owner-only tools section that lists current core owner-only tools from source, explains that plugin tools can also be owner-only via metadata, documents `commands.ownerAllowFrom` with clearly fake channel-native ID examples, and cross-links the public tools/config docs.
