## Summary

Keep open. Current main still injects skipped group-chat messages into Telegram context without per-message target classification, so the reported ambiguity between messages for this agent vs another agent is not implemented away.

## Evidence

- **Skipped Telegram group messages are preserved as undifferentiated history:** When a mention-gated Telegram group message is skipped, OpenClaw records it into pending history as only `{ sender, body, timestamp, messageId }`. There is no field marking whether that skipped message was aimed at this agent, another agent, or nobody.
  - file: [extensions/telegram/src/bot-message-context.body.ts:292](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/telegram/src/bot-message-context.body.ts#L292)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Telegram inbound payload exposes history without routing metadata:** The Telegram session payload maps stored history entries to `InboundHistory` with only `sender`, `body`, and `timestamp`, then passes that into the context payload. The current-turn payload adds `ReplyTo*` and `WasMentioned`, but not an equivalent target marker for prior messages.
  - file: [extensions/telegram/src/bot-message-context.session.ts:312](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/telegram/src/bot-message-context.session.ts#L312)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Prompt assembly renders history as sender/timestamp/body only:** Prompt assembly serializes chat history as JSON objects containing only `sender`, `timestamp_ms`, and `body`. That means prior group-chat lines still reach the model without an explicit 'addressed to this agent/other agent/nobody' signal.
  - file: [src/auto-reply/reply/inbound-meta.ts:317](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/auto-reply/reply/inbound-meta.ts#L317)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Shared context type has no per-history targeting field:** `InboundHistory` is typed as `{ sender, body, timestamp }[]`, while `WasMentioned` exists only as a single boolean for the active inbound turn. The public context shape itself does not currently carry per-message agent-target metadata for group history.
  - file: [src/auto-reply/templating.ts:39](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/auto-reply/templating.ts#L39)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Docs still describe generic pending-message history injection:** The group-message docs say pending group messages are prefixed under `[Chat messages since your last reply - for context]` before the current message. The documented behavior matches the code path above and does not mention any multi-agent targeting annotation.
  - file: [docs/channels/group-messages.md:19](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/docs/channels/group-messages.md#L19)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
