## Summary

Keep #69978 open. Current main still has the TUI streaming watchdog fixed at a default 30s handler option, no `/watchdog` command or persistence path, and no per-run warning suppression. The related #68596 and #69081 reports overlap the timeout/status area, but they do not fully supersede this issue's combined `/watchdog` runtime-control request plus duplicate-warning cleanup.

## Best Possible Solution

Keep the issue open or explicitly consolidate it into a canonical TUI watchdog follow-up. The best implementation path is in `src/tui`: add per-run watchdog warning suppression cleared on finalize/terminate/session switch, expose a supported user-facing timeout/disable control through either `/watchdog` or the broader config surface from #68596, persist the preference, and add focused tests plus TUI slash-command docs.
