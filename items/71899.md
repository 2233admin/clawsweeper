## Summary

Keep #71899 open. Current main still uses `agents.defaults.workspace` as a base for non-default agents, so a setup-created main workspace such as `workspace` still resolves new agents to nested `workspace/<agentId>`. The PR is a focused, active fix for #71889 with regression coverage and follow-up responses to reviewer/bot findings; it is not obsolete, implemented on main, or superseded by another shipped change.

## Best Possible Solution

Keep the PR open for maintainer review and likely landing after final code review. The right product path is to fix `resolveAgentWorkspaceDir()` so setup-managed main workspaces do not become shared bases for new agents, preserve explicit per-agent workspaces and intentional custom shared bases, add resolver plus wizard/default coverage, and keep docs/changelog aligned with the peer-level multi-agent layout.
