## Close Comment

Thanks for the contribution. After a Codex review against current `main`, I’m closing this because it fits the external/community plugin path rather than a new bundled OpenClaw plugin.

- `VISION.md:54-59` says optional capability should usually ship as plugins, with the preferred path being npm/local extension loading and plugin authors maintaining their own repository.
- `docs/plugins/building-plugins.md:16-19` explicitly says you do not need to add your plugin to the OpenClaw repository and should publish it to ClawHub or npm instead.
- `docs/plugins/community.md:9-22` makes ClawHub the canonical discovery surface for community plugins, and `docs/plugins/community.md:64-75` already shows context-management functionality living in that external plugin ecosystem.
- Current `main` still keeps the bundled plugin slash-command set small in `docs/tools/slash-commands.md:148-163`; `/save` and `/load` are not part of that curated bundled surface.
- This was a Codex review of current `main`, and there is still no `extensions/save-load` package present there.
