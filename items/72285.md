## Summary

Keep #72285 open. Current main still has the stale-token behavior the PR is trying to fix: status context totals are computed from `entry.totalTokens` without consulting `totalTokensFresh`, and the status formatter does not fall back to the latest compaction checkpoint `tokensAfter`. The PR needs rebase/adaptation because `buildStatusMessage` now lives in `src/status/status-message.ts`, not the older implementation location in `src/auto-reply/status.ts`.

## Best Possible Solution

Keep the PR open and update it against current main: apply the stale-total handling in `src/status/status-message.ts`, use `resolveFreshSessionTotalTokens(entry)` before trusting persisted totals, fall back to the latest valid compaction checkpoint `tokensAfter` when `totalTokensFresh === false`, and keep the regression test in `src/auto-reply/status.test.ts` adjusted for the current implementation path.
