## Summary

Keep open. Current `main` still routes Telegram `callback_query` clicks into agent sessions as plain callback text and sets `MessageSid` from the callback id, not the source Telegram `message.message_id`, so the agent still lacks the original message id needed to call `editMessage`. There is relevant prior art in the Telegram plugin interactive handler path, which already exposes `callback.messageId`/`chatId` for plugins, but that does not satisfy this issue’s requested agent-session behavior.
