## Summary

Keep open. Current main still forwards `options.temperature` into the Anthropic Messages request for non-thinking `claude-opus-4-7` calls, and the PR targets a real remaining path from #72123 rather than obsolete work. The submitted patch needs cleanup and regression coverage before it is safe to land.

## Best Possible Solution

Keep the PR open or land an equivalent small fix on main: strip `temperature` for both `claude-opus-4-7` and `claude-opus-4.7` in the Anthropic Messages transport paths, preserve temperature forwarding for unaffected Anthropic models, and add focused regression tests proving the Opus 4.7 variants omit temperature while a supported model still forwards it.
