## Summary

Keep #39899 open. Current main has broader prevention for the common unpaired-surrogate replay path via transport text sanitization and failed-assistant-turn filtering, but it still does not implement this PR's surrogate-specific error detector, friendly error text, or session reset branch. The PR may need rebasing/narrowing, but it is not safely obsolete from source evidence alone.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it against current main and add a focused regression test that demonstrates whether a surrogate provider-error path still reaches runAgentTurnWithFallback after transport sanitization. If it does, implement the smallest detector, friendly message, and reset path consistent with the existing role-ordering and Gemini corruption recovery branches; if the test proves all supported paths are already prevented, a maintainer can close it as obsolete with that proof.
