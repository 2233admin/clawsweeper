## Summary

Keep #71936 open. Current main still returns an empty GitHub Copilot provider catalog and relies on the synthetic dynamic-model fallback for newly available Copilot chat models, so the reported GPT-5.5 capability-mapping gap is not implemented yet. The linked PR #71924 appears to be the appropriate fix path, but it is open/unmerged in the supplied GitHub context.

## Best Possible Solution

Review and land PR #71924, or an equivalent GitHub Copilot plugin-local change, so the provider catalog fetches Copilot `/models`, maps advertised limits/vision/reasoning/endpoints into OpenClaw model definitions, and adds focused GPT-5.5 coverage. After that lands on main, #71936 can close as implemented.
