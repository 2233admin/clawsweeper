## Summary

Keep open. Current `main` still implements the permissive fallback described in the issue: bearer auth is retried on 401/403 unless callers explicitly opt into `requireHttps` or provide a rejecting `shouldAttachAuth` predicate. The current tests added with the helper lock that behavior in. The helper is not currently exported as a public SDK entrypoint, which narrows present blast radius, but that is not a fix.
