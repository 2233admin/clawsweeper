## Summary

Keep open. Current `main` still only applies `sendChatAction` backoff/circuit-breaking to 401 errors, and the typing keepalive path still resets its failure guard on every `onReplyStart()`, so the reported 3-second retry loop remains plausible and is not fixed on `main` or in the latest release.

## Evidence
