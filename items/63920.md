## Summary

Keep this issue open. Current main still does not expose Slack global shortcuts or message-action shortcuts through the bundled Slack channel. The Slack interaction monitor registers block actions and modal view lifecycle handlers only; docs, tests, and the plugin-facing Slack interactive handler surface are still button/select oriented, not shortcut oriented.

## Best Possible Solution

Implement this in the bundled Slack plugin unless maintainers first add a typed external shortcut API. The likely path is a new Slack interaction shortcut module that registers `ctx.app.shortcut` for global and message-action shortcuts, acknowledges and mismatch-checks payloads, parses sanitized shortcut summaries, applies the existing Slack authorization/session-key rules, enqueues stable system events with context keys, and updates Slack interaction tests plus `docs/channels/slack.md`. If maintainers want this outside bundled Slack, the prerequisite is a plugin-facing Slack shortcut receiver or raw Slack event hook rather than closing this as ClawHub-ready today.
