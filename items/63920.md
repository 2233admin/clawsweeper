## Summary

Keep open. Current main still lacks a Slack `app.shortcut` / `message_action` / global shortcut handler, while the existing Slack interaction path covers block actions plus modal submit/close events only. The current plugin interactive API also exposes Slack button/select dispatch, not shortcut payload routing, so this is not cleanly closeable as ClawHub work without first adding a Slack plugin/API seam.

## Best Possible Solution

Implement this in the Slack plugin or first add an explicit Slack shortcut/plugin-facing API: register Bolt shortcut handlers for global and message shortcuts, parse and sanitize shortcut payloads, authorize and resolve session routing, enqueue structured system events with tests in `extensions/slack/src/monitor/events/interactions.test.ts`, and update the Slack docs/manifest guidance. If maintainers want this fully plugin-owned, the missing API seam should be designed before closing the issue.
