## Summary

Keep open. Current main resolves the SecretRef crash and the plain `agents list --json` plugin-preload path, but the broader CLI exit-hang report is still actionable because human `agents list` and `agents add` still load plugins, and short-lived CLI teardown remains memory-runtime-specific rather than a generic plugin shutdown path.

## Best Possible Solution

Keep this issue open but narrow maintainer action to the remaining CLI exit-hang surface. The best fix is to keep config-only `agents` paths, especially non-interactive `agents add` and human `agents list` where feasible, off broad runtime plugin loading; if a path truly needs plugin runtime code, add a short-lived CLI plugin-runtime teardown contract that external plugins can implement. Treat the SecretRef fallback, no-plugin bind/delete overrides, and plain JSON list fast path as resolved subparts.
