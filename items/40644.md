## Summary

Keep #40644 open. Current main still renders the Control UI cron surface as a summary strip, filterable flat jobs list, run-history list, and edit form. It does not implement the requested 24-hour timeline, daily/week calendar modes, heatmap, click-detail popup, drag-to-reschedule UI, or cron-expression visualization helper. The related PR #41892 is useful context and explicitly targets this issue, but the provided GitHub context shows it is still open and unmerged, so the issue remains the canonical tracking item.

## Best Possible Solution

Keep #40644 open as the canonical Control UI feature request. Review, shrink, or replace PR #41892 as the implementation path, and close the issue only after a merged mainline change ships a maintainer-approved calendar/timeline view or after maintainers create a narrower canonical tracking issue for the remaining work.
