## Summary

Keep this issue open. Current main still renders the Control UI cron surface as a summary strip, filterable jobs list, run-history list, and edit form; it does not implement the requested timeline/calendar/day/week views, heatmap, drag-to-reschedule flow, or parser-backed cron-expression validation. The provided GitHub context also shows #41892 as an open, unmerged PR with closing syntax for the same request, so the issue should remain open until that implementation lands or maintainers choose a replacement scope.
