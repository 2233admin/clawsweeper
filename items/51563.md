## Summary

Keep open. Current `main` still has the reported `openclaw update status` package-manager misdetection for package installs, so the PR is not obsolete, but the proposed fix is weaker than the install-manager detection already used elsewhere in core. In particular, the PR discussion correctly identified that its Bun path heuristic would misclassify supported custom `BUN_INSTALL` layouts that current updater code already handles.

## Evidence

- **Current status path still reads the package root manifest/lockfiles before deciding git vs package install:** `checkUpdateStatus()` resolves `pm` via `detectPackageManager(root)` and then returns that value as `packageManager` for package installs. There is no package-install-specific global-manager detection in this path on `main`.
  - file: [src/infra/update-check.ts:395](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/infra/update-check.ts#L395)
  - command: `nl -ba src/infra/update-check.ts | sed -n '392,420p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **The repo manifest advertises pnpm, which explains the npm-global misreport:** Root `package.json` still declares `"packageManager": "pnpm@10.33.0"`. `detectPackageManager()` prefers `package.json.packageManager`, so a packaged install rooted at this manifest will report `pnpm` regardless of how the CLI was globally installed.
  - file: [package.json:1682](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/package.json#L1682)
  - command: `rg -n '"packageManager"' package.json`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **OpenClaw already has a stronger global-install detector that honors custom Bun roots:** `resolveBunGlobalRoot()` uses `BUN_INSTALL` and `detectGlobalInstallManagerForRoot()` compares the actual package root against npm/pnpm global roots plus the Bun global root. Tests cover `BUN_INSTALL` and manager detection from resolved roots.
  - file: [src/infra/update-global.ts:366](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/infra/update-global.ts#L366)
  - command: `nl -ba src/infra/update-global.ts | sed -n '366,533p' ; nl -ba src/infra/update-global.test.ts | sed -n '176,207p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **That stronger detector is already the established implementation for update flows:** Both the updater and update CLI shared logic call `detectGlobalInstallManagerForRoot(...)` for package installs, which makes the PR's separate path-heuristic approach look like the wrong seam to extend.
  - file: [src/infra/update-runner.ts:1285](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/infra/update-runner.ts#L1285)
  - command: `nl -ba src/infra/update-runner.ts | sed -n '1282,1290p' ; nl -ba src/cli/update-cli/shared.ts | sed -n '239,258p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **PR discussion surfaced a real correctness gap in the proposed patch:** The existing PR review comment points out that the submitted `detectGlobalInstallManager(resolvedRoot)` only checks for `/.bun/`, so Bun installs under a custom `BUN_INSTALL` such as `/opt/bun/install/global/node_modules/openclaw` would still be reported as `npm`. That matches the stronger main-branch helper's explicit `BUN_INSTALL` support and is a substantive reason not to close this PR as implemented or obsolete.
  - sha: [d62f06c3230b](https://github.com/openclaw/openclaw/commit/d62f06c3230bc75e324f5472f493db81d981e754)

## Risks / Open Questions

- Closing this PR would leave the reported `openclaw update status` bug unfixed on current `main` for npm global installs.
- Merging the PR as written would introduce a second, weaker install-manager detection path and still misclassify supported Bun installs that use a custom `BUN_INSTALL` root.
