## Summary

Current main already routes Slack room messages through the bound agent before any partial/native streaming logic, and the Slack dispatcher reuses that prepared route rather than resolving a new default-agent path. The added routing/tests on main make the reported bound-channel fallback to the default agent no longer match current behavior.

## Evidence
