## Summary

Keep this PR open. Current main has improved exact, JSON, and mixed NO_REPLY handling plus other user-facing sanitizers, but it still does not implement this PR's channel-agnostic guard for plain internal route-marker/tool-JSON leakage in the shared outbound reply normalizer.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, then either land the shared normalizeReplyPayload guard with its regression tests or adapt the same narrow route-marker/tool-JSON protection into the current sanitizer stack. Preserve tests for media/channelData preservation, custom silent tokens, pretty-printed JSON, standalone route markers, and explanatory-prose false positives.
