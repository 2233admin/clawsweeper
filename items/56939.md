## Summary

Keep open. Current `main` still performs ambient Ollama discovery against the default local host even without explicit Ollama config, so the report is not implemented on `main` and not obsolete.

## Evidence

- **Ambient discovery still runs by default:** `resolveOllamaDiscoveryResult()` only skips implicit discovery when `discovery.enabled === false` or in test envs; otherwise it calls `buildProvider(...)`, which is the path that probes Ollama.
  - file: [extensions/ollama/src/discovery-shared.ts:102](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/ollama/src/discovery-shared.ts#L102)
  - command: `nl -ba extensions/ollama/src/discovery-shared.ts | sed -n '90,145p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Test coverage asserts probing without API key or explicit config:** Current tests explicitly expect Ollama to auto-register from `http://127.0.0.1:11434` with no ambient `OLLAMA_API_KEY`, confirming this behavior is intentional on `main` rather than fixed away.
  - file: [extensions/ollama/provider-discovery.test.ts:194](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/ollama/provider-discovery.test.ts#L194)
  - command: `nl -ba extensions/ollama/provider-discovery.test.ts | sed -n '185,245p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Config normalization still steers empty Ollama config into discovery:** The public Ollama provider policy normalizes an empty provider config to `baseUrl: http://127.0.0.1:11434` plus `models: []`, and its test says this is done so implicit discovery can run before validation.
  - file: [extensions/ollama/provider-policy-api.ts:27](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/ollama/provider-policy-api.ts#L27)
  - command: `nl -ba extensions/ollama/provider-policy-api.ts | sed -n '1,60p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Docs still do not match the no-config discovery path:** The Ollama provider docs say implicit discovery happens when `OLLAMA_API_KEY` or an auth profile is set and no explicit provider entry exists, but current test coverage shows discovery also happens with neither present.
  - file: [docs/providers/ollama.md:151](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/providers/ollama.md#L151)
