## Summary

Keep #39766 open. Current main still rejects `models.providers.<id>` entries without `models`, and the related runtime path would use `models.providers.openai.baseUrl` only if validation allowed the config through. The related issue #39589 is still open, while the cross-referenced PR attempts #46980 and #71636 are closed unmerged, so this PR is not implemented or superseded on main.

## Best Possible Solution

Keep this PR or canonical issue #39589 open until main accepts generic `models.providers.<id>` entries that omit `models`, preserves a safe downstream runtime contract for model iteration, updates the generated schema/type surface intentionally, and adds focused regression coverage for config parsing plus OpenAI embedding `baseUrl` fallback behavior.
