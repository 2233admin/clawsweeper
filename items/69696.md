## Summary

Keep PR #69696 open. Current main still performs OpenRouter base URL matching case-sensitively in the OpenRouter plugin, Arcee plugin, and embedded-runner test support, while existing tests cover only lowercase stale `/v1` URLs. The PR is a focused follow-up to the already-merged stale-base-url fix and still carries useful behavior/test coverage rather than being implemented, duplicate, or superseded.

## Best Possible Solution

Keep the PR open and have it rebased or adjusted as needed. The best product/code path is still to make the OpenRouter and Arcee provider-owned base URL canonicalization handle case-insensitive OpenRouter host input, with regression tests for uppercase OpenRouter URLs, canonical no-op URLs, and non-OpenRouter custom proxy passthrough. This belongs in the bundled provider normalization seams touched by the PR, not in ClawHub or an external plugin.
