## Summary

Keep PR #69696 open. Current main still normalizes OpenRouter and Arcee OpenRouter base URLs case-sensitively: the three helpers touched by the PR trim and strip trailing slashes but do not lowercase before comparing against the canonical and legacy OpenRouter constants. Existing regression coverage only covers lowercase stale `https://openrouter.ai/v1` inputs; the PR's uppercase-domain regression remains missing and still targets the live provider normalization paths.

## Best Possible Solution

Keep the PR open and either merge or port its narrow fix: lowercase only the normalized comparison value in the OpenRouter, Arcee OpenRouter, and embedded-runner normalization helpers, then keep the uppercase-domain and no-op regression tests alongside the existing stale `/v1` coverage.
