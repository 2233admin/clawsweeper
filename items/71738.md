## Summary

Keep #71738 open. Current main still has the Matrix thread-history and reply-placement gaps this PR targets: thread roots are detected and used for routing/session isolation, but room-history storage, history watermarks, and readMessages remain room-flat; outbound thread sends still include fallback reply metadata unconditionally; threaded edits still place reply context on the outer replace relation. The PR is a plausible bundled Matrix plugin bug fix, not conservative cleanup material, although the posted review comments mean it should not land as-is without addressing the noted pagination/watermark risks.

## Best Possible Solution

Keep #71738 open and ask for a focused revision or maintainer port. The best path is to land thread-scoped Matrix history/read behavior and outbound relation fixes in the Matrix plugin, with regression tests for per-thread room-history queues and watermarks, thread reads via Matrix relations, main-room read pagination after filtering, thread sends without fallback metadata unless explicitly requested, and threaded edits storing thread context under m.new_content. Address the existing review comments before merge.
