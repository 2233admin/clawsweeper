## Summary

Keep PR #69394 open. Current main still strips a trailing `NO_REPLY` token and delivers the preceding text, and the current unit tests explicitly assert that behavior for substantive text. The related #66755 PR is a narrower alternative for reasoning-marked payloads, not a shipped replacement for this broader trailing-token proposal. This PR should not merge unchanged because its `stripped.length < text.length` detector can misclassify non-trailing-token messages with trailing whitespace, and its full-payload suppression may drop non-text content that current tests/docs preserve, but the underlying bug remains live and useful to resolve.

## Best Possible Solution

Keep this open or consolidate it into the canonical #66701/#66755 discussion after maintainer direction. The best code path is a targeted helper that can report whether a trailing silent token was actually removed, then suppress text-only trailing-token replies while preserving existing exact-token/media behavior and adding regression tests for raw reasoning, substantive text, non-trailing literals, whitespace, and media/channelData payloads.
