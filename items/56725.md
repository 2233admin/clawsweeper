## Summary

Keep open. Current `main` still has the accumulation path, and this PR is not strong enough to justify closing the item: the proposed dedup is not present on `main`, it keeps the first duplicate rather than the newest hook-updated entry, and it does not add the missing regression test coverage.

## Evidence

- **Current main still lacks dedup in bootstrap sanitization:** `sanitizeBootstrapFiles` only normalizes/validates `path` and pushes every valid file through; there is no path-based dedup on current `main`. That means the reported duplicate bootstrap entries are still possible.
  - file: [src/agents/bootstrap-files.ts:147](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/bootstrap-files.ts#L147)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Cached bootstrap array is reused per session:** `getOrLoadBootstrapFiles` returns the cached `WorkspaceBootstrapFile[]` by session key. For normal sessions, that cached array is handed back directly on later runs.
  - file: [src/agents/bootstrap-cache.ts:9](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/bootstrap-cache.ts#L9)

- **Hooks can mutate the reused array:** For non-subagent, non-cron sessions, `filterBootstrapFilesForSession` returns the original array reference, and `applyBootstrapHookOverrides` passes that array into hook context. A hook using `event.context.bootstrapFiles.push()` therefore mutates the cached session snapshot and can accumulate duplicates across restarts/reloads.
  - file: [src/agents/workspace.ts:681](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/workspace.ts#L681)

- **Mainline tests do not cover duplicate-path regression:** The bootstrap-file tests cover hook overrides and malformed paths, but there is no test asserting duplicate paths are deduplicated. That matches the review discussion on this PR and means the bug is not currently locked down on `main`.
  - file: [src/agents/bootstrap-files.test.ts:79](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/bootstrap-files.test.ts#L79)
