## Summary

Keep #70055 open. Current main partially mitigates the reported noise: the old `synced ... credentials from external cli` info-level path is gone, and external CLI OAuth use is now runtime-only with debug/warn logging and tests. But the requested first-class config/env opt-out is still missing: external CLI profiles are still resolved from a hardcoded provider list, `overlayExternalAuthProfiles` calls that path unconditionally for runtime auth overlays, and the strict `auth` config schema has no `externalCliSync` field. The provided related ClawSweeper items are title-overlap only and do not supersede this auth-profile sync request.

## Best Possible Solution

Keep the issue open and implement a narrow additive opt-out, likely `auth.externalCliSync` plus an env override, validated in `src/config/types.auth.ts` and `src/config/zod-schema.ts`, then plumb that policy into `src/agents/auth-profiles/external-auth.ts`/`external-cli-sync.ts` and remove or wire the currently unused `syncExternalCli` store option. Add focused tests for disabled, include/exclude/default-on behavior and update the model auth docs if a public config surface is added.
