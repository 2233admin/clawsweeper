## Summary

Keep open. Current `main` still only strips top-level `type: "image"` `data` fields from chat history blocks; it does not implement the PR's added redaction for dashboard-style `content[].source.data`, bare inline `url` data URLs, or `image_url.url` data URLs. Existing tests cover audio redaction and assistant-generated image URLs, but not the reported inbound inline-image history case. The PR discussion also shows the one review note was addressed in a follow-up commit, so this remains useful work rather than obsolete.
