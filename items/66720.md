## Summary

Keep this PR open. Current main still does not implement the stricter gateway/provider schema simplification requested here: `sessions_spawn` still advertises `lightContext`, `attachments`, and `attachAs`, and current provider schema compatibility only strips schema keywords rather than named tool properties. The PR is not obsolete or implemented on main, and the linked report #66719 is closed with this PR left as the canonical implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a provider/deployment-aware `sessions_spawn` schema filter, wired from the actual OpenAI-compatible gateway/model compatibility path that serves affected providers, while preserving current ACP availability hiding and adding focused schema-shape tests.

## Risks / Open Questions

- Merging the PR as-is may still leave affected OpenAI-compatible gateway deployments unfixed unless a real provider/deployment compatibility path activates the stripped schema.
- Schema simplification is behavior-visible to models; if applied too broadly, it could hide useful native subagent fields from providers that can handle them.
- The PR needs regression coverage for default full schema behavior, ACP-unavailable schema behavior, and the activated strict-gateway schema shape.
