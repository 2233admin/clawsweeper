## Summary

Current `main` no longer depends on `systemPromptReport` to restore the active model. Model switches persist through `providerOverride`/`modelOverride`, stale runtime `model`/`modelProvider` are cleared on switch, and regression tests cover the stale-runtime-vs-override case.

## Evidence

- **Model switches clear stale runtime model fields:** `applyModelOverrideToSessionEntry` persists the selected override and deletes stale `model`, `modelProvider`, and `contextTokens` when the selected model changes or runtime fields are out of sync.
  - file: [src/sessions/model-overrides.ts:10](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/sessions/model-overrides.ts#L10)
- **`/model` uses shared override persistence:** Directive handling routes model changes through `applyModelOverrideToSessionEntry(..., markLiveSwitchPending: true)`, so slash-model switches update the persisted override path instead of relying on the old runtime fields.
  - file: [src/auto-reply/reply/directive-handling.impl.ts:415](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/auto-reply/reply/directive-handling.impl.ts#L415)
- **`session_status` uses the same override path:** The session status tool applies requested model changes with the same helper before recomputing session status, covering the other switch path named in the report.
  - file: [src/agents/tools/session-status-tool.ts:484](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/tools/session-status-tool.ts#L484)
- **Session restore reads persisted overrides, not `systemPromptReport`:** Model selection first resolves a stored override from `providerOverride`/`modelOverride`, then applies that stored override to the restored session. The helper that finds stored overrides only reads session/parent override fields.
  - file: [src/auto-reply/reply/model-selection.ts:131](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/auto-reply/reply/model-selection.ts#L131)
- **Stored-override helper ignores `systemPromptReport`:** `resolveStoredModelOverride` only reads `providerOverride` and `modelOverride` from the session entry or parent entry.
  - file: [src/auto-reply/reply/stored-model-override.ts:27](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/auto-reply/reply/stored-model-override.ts#L27)
- **Gateway model identity path does not consult `systemPromptReport`:** `resolveSessionModelIdentityRef` resolves identity from runtime `model`/`modelProvider`, fallback refs, and then `resolveSessionModelRef`; there is no `systemPromptReport` read in this path.
  - file: [src/gateway/session-utils.ts:1186](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/gateway/session-utils.ts#L1186)
- **Regression test for stale runtime model vs stored override:** Test coverage explicitly asserts that a session with stale runtime `model`/`modelProvider` still restores the persisted `providerOverride`/`modelOverride` selection.
  - file: [src/auto-reply/reply/model-selection.test.ts:451](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/auto-reply/reply/model-selection.test.ts#L451)
- **Regression test for clearing stale runtime fields on switch:** Test coverage asserts that switching overrides clears stale runtime `model`/`modelProvider` and cached `contextTokens`.
  - file: [src/sessions/model-overrides.test.ts:23](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/sessions/model-overrides.test.ts#L23)
- **No code path reads `systemPromptReport.model` or `.provider` for restore:** A repository search for `systemPromptReport` model/provider field reads in `src`, `ui`, and `test` found no restore/status consumers beyond context-report/debug persistence surfaces.
  - command: `rg -n "systemPromptReport\.(model|provider)|systemPromptReport\?.(model|provider)|systemPromptReport[^\n]{0,80}(model|provider)|modelProvider.*systemPromptReport|systemPromptReport.*modelProvider" src test ui`

## Close Comment

Closing from a Codex review because current `main` already handles this class of desync. Model switches now persist through `providerOverride`/`modelOverride` and clear stale runtime `model`/`modelProvider`/`contextTokens` on switch (`src/sessions/model-overrides.ts:10`). Session restore resolves stored overrides from those persisted override fields (`src/auto-reply/reply/model-selection.ts:131`, `src/auto-reply/reply/stored-model-override.ts:27`), and the gateway/session identity path does not read `systemPromptReport` as the source of truth (`src/gateway/session-utils.ts:1186`). There is also regression coverage for stale runtime fields not overriding a stored model selection (`src/auto-reply/reply/model-selection.test.ts:451`, `src/sessions/model-overrides.test.ts:23`). I could not confirm a released tag for this yet, so this looks fixed on `main` but not necessarily shipped in the latest release.
