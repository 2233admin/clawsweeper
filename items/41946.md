## Summary

Keep open. Current `main` does not implement the Feishu underscore-escaping change, and the PR as proposed is not merge-ready: it imports a non-exported helper and uses a markdown-link regex that would regress existing parenthesized-URL behavior already covered by tests.

## Evidence

- **Current main still sends raw markdown URLs:** `sendMessageFeishu` and `editMessageFeishu` pass `convertMarkdownTables(...)` directly into the Feishu post payload; there is no URL-underscore escaping step on current `main`.
  - file: [extensions/feishu/src/send.ts:469](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/extensions/feishu/src/send.ts#L469)
  - command: `nl -ba extensions/feishu/src/send.ts | sed -n '1,560p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Current main post parsing preserves raw hrefs:** `renderLinkElement` returns markdown links as `[text](href)` with the raw `href`; there is no escaping helper in `post.ts` on current `main`.
  - file: [extensions/feishu/src/post.ts:83](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/extensions/feishu/src/post.ts#L83)
  - command: `nl -ba extensions/feishu/src/post.ts | sed -n '1,360p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Existing test locks in parenthesized URL behavior:** Current Feishu tests expect `parsePostContent` to preserve a link whose URL contains `(` and `)`: `https://example.com/guide(a)`. The PR's proposed `[^)]+` URL capture would stop at the first `)` and break that already-tested behavior.
  - file: [extensions/feishu/src/post.test.ts:37](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/extensions/feishu/src/post.test.ts#L37)

  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **PR patch has a direct export/import mismatch:** The PR diff imports `escapeUnderscoresInMarkdownLinks` from `./post.js` in `send.ts`, but the new helper shown in the patch is declared without `export`. The GitHub review comments correctly flag this as a blocker.
  - file: [extensions/feishu/src/post.ts:36](https://github.com/openclaw/openclaw/blob/be98173193234a86d397ed756d72555eaf90e8c6/extensions/feishu/src/post.ts#L36)
