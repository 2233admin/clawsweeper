## Summary

Keep this PR open. Current main still loses Discord upload MIME types in the standard, component, and native interaction upload paths, and WebP images still go through the shared optimizer rather than being preserved. The PR addresses a real linked bug and has relevant tests, but it should stay under maintainer review because the shared WebP bypass affects WhatsApp behavior, has unresolved validation/privacy concerns, omits at least one Discord upload path, and overlaps with open PRs #41265 and #41714.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it with a focused canonical fix for #41256. The best path is to preserve verified `media.contentType` in all Discord Blob upload paths, including standard sends, component sends, and native interaction replies; decide whether WebP/animated-image preservation should be Discord-specific or shared; add regression tests for MIME preservation and frame preservation; address WebP metadata and unknown-dimension validation before bypassing optimization; keep WhatsApp docs/tests aligned with the chosen shared behavior; and consolidate the overlapping work in #41265 and #41714.
