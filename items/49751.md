## Summary

Keep open. Current main has the shared 10s gateway/client handshake timeout from the earlier #49262 work, but it has not implemented this PR's remaining semantic fix: clearing the pre-auth WebSocket handshake timer immediately after a syntactically valid connect request and replacing it with a bounded connect-auth timer. A maintainer also recently pushed a narrow repair to this PR branch and explicitly kept the contributor path canonical, so this is an active implementation candidate rather than cleanup noise.

## Best Possible Solution

Keep this PR open for maintainer review and landing if CI remains green. Main already shipped the shared 10s timeout budget, but the best remaining fix is the PR's narrower timer split: clear the pre-auth timer once a valid connect frame arrives, bound stalled async auth before registration, keep the regression coverage and protocol docs, and preserve the original contributor credit.
