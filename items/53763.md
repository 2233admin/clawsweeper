## Summary

Keep this issue open. Current main has a bundled browser plugin, local headless mode, custom launch flags, remote CDP support, and optional Docker/sandbox browser paths, but it still does not satisfy the central request: a first-class bundled Chromium runtime that works without external browser setup across ordinary host installs. The current managed launcher still resolves a configured or system-installed Chromium-family executable and fails when none is found; Docker/browser binaries remain opt-in through build args, manual Playwright install, or an optional sandbox browser image.

## Best Possible Solution

Keep this issue open, but narrow it to the remaining product decision: whether OpenClaw should ship a pinned Chromium runtime by default, publish an official browser-enabled Docker image, or add installer/doctor repair flows that install and persist Playwright Chromium and wire that path into executable discovery. Reuse the current browser plugin, headless mode, remote CDP support, extraArgs, and sandbox browser container rather than rebuilding browser control.
