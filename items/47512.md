## Summary

Current `main` no longer uses the legacy node shell wrapper described in the report. Node shell execution is now routed through `exec host=node`, the `nodes` CLI explicitly blocks `system.run`/`system.run.prepare`, and headless-node approvals are documented and implemented as local `~/.openclaw/exec-approvals.json` policy rather than requiring a Linux approval socket daemon.

## Close Comment

Closing from a Codex review because current `main` has already retired the reported path. Node shell execution is now explicitly routed through `exec host=node` (`docs/tools/exec.md:72`), the `nodes` CLI docs say to use that path instead of `openclaw nodes run` (`docs/cli/nodes.md:62-64`), and the CLI rejects `system.run` / `system.run.prepare` direct node invocation with an explicit redirect to `exec host=node` (`src/cli/nodes-cli/register.invoke.ts:12-37`). The supported node-exec path now prepares `system.run.prepare` before execution (`src/agents/bash-tools.exec-host-node.ts:99-122`), and the current docs describe headless Linux node approvals as local `~/.openclaw/exec-approvals.json` policy with `askFallback` behavior rather than requiring a Linux approval socket daemon (`docs/tools/exec-approvals.md:34-49`, `docs/nodes/index.md:347-349`). I could not confirm a released tag containing this change yet, so `fixedRelease` is left unset. If `exec host=node` still reproduces on current `main`, please open a fresh issue with the exact command, current commit/release, and the node-side logs from the supported exec path.
