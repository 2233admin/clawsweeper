## Summary

Keep open. Current `main` still intentionally omits the `message` tool from the default `coding` profile, and the repo tests assert that behavior. The repo does document alternate audio-delivery paths (`MEDIA:` + `[[audio_as_voice]]`, and the `tts` tool), but those are workarounds rather than proof that the reported missing agent delivery tool surface was restored.

## Evidence

- **`message` is not part of the coding profile:** `src/agents/tool-catalog.ts` defines `message` with `profiles: ["messaging"]`, while `coding` uses `listCoreToolIdsForProfile("coding")`; `tts` also has `profiles: []`, so neither is included by the coding-profile allowlist.
