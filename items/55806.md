## Summary

Keep open. Current `main` has partial tool observability improvements (tool start/end events, `after_tool_call` duration, and `apply_patch` summaries), but it still does not implement the core asks in #55806: sanitized persisted input logging, structured tool error categorization, and generic context writeback tracking.

## Evidence

- **Tool start/end logging still lacks sanitized arg audit output:** `handleToolExecutionStart` logs only `embedded run tool start...` and emits the raw `args` object on the `tool` event; `handleToolExecutionEnd` logs only the matching end line. There is no sanitized INFO-level parameter summary here.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:626](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/pi-embedded-subscribe.handlers.tools.ts#L626)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Current after_tool_call shape has duration but only string errors:** The hook event documented and typed on `main` includes `result`, `error?: string`, and `durationMs?: number`; it does not define structured fields such as `errorType` or `errorRecoverable`. `handleToolExecutionEnd` populates only the string `error` plus `durationMs`.
  - file: [src/plugins/hook-types.ts:357](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugins/hook-types.ts#L357)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Docs describe current tool observability surface as results/errors/duration only:** Public plugin-hook docs say `after_tool_call` observes tool results, errors, and duration. That matches the partial implementation, but not the broader request for sanitized input logging and writeback tracking.
  - file: [docs/plugins/hooks.md:76](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/plugins/hooks.md#L76)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Writeback tracking is narrow and patch-only:** `main` emits a dedicated patch summary event for `apply_patch` with `added`, `modified`, and `deleted` file lists, but there is no generic tool writeback event/schema for other file or memory writes.
  - file: [src/infra/agent-events.ts:88](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/agent-events.ts#L88)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Gateway docs explicitly limit tool redaction to console summaries, not file logs:** The logging docs state tool summary redaction is 'tools-only' and 'does not alter file logs', which means the requested sanitized audit logging for persisted logs is still not documented as implemented.
  - file: [docs/gateway/logging.md:53](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/gateway/logging.md#L53)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
