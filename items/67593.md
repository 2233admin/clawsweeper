## Summary

Keep open. Current `main` still does not implement Kimi/Moonshot provider usage snapshots: the supported usage-provider types/lists exclude `kimi`/`moonshot`, the bundled Kimi provider does not register usage hooks, and the docs still enumerate other providers only. This belongs in OpenClaw scope because Kimi is already a bundled provider plugin and provider-owned usage hooks are an established core/plugin seam.

## Evidence

There is no `kimi` or `moonshot` path for `openclaw status --usage` by default.
