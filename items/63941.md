## Summary

Keep open. Current main still rejects self-hosted Firecrawl baseUrl values by requiring HTTPS and the official api.firecrawl.dev host, and it still sends Firecrawl API calls through the strict web-tools SSRF guard. The PR addresses a real linked bug (#63877), and the provided discussion shows the author responded to review feedback by restoring SecretRef coverage and adding trusted-guard routing for private/local Firecrawl endpoints.

## Best Possible Solution

Keep the PR open and continue it as the focused Firecrawl fix: accept admin-configured self-hosted Firecrawl baseUrls, allow HTTP only for private/local endpoints, keep public endpoints on HTTPS, route private/local Firecrawl API calls through the trusted web-tools guard, preserve the SecretRef security tests, and update Firecrawl/web-fetch docs before landing.
