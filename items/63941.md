## Summary

Keep this PR open. Current main still blocks admin-configured self-hosted Firecrawl baseUrl values, the linked bug remains open, and this PR is a focused implementation candidate with recent fixes for the SSRF-routing and SecretRef-test review concerns.

## Best Possible Solution

Keep this PR open for maintainer review. Land this PR or a maintainer-owned equivalent that permits admin-configured self-hosted Firecrawl endpoints, keeps public hosts on HTTPS and the strict guard, allows cleartext HTTP only for private/loopback targets, routes trusted private endpoints through the existing trusted web-tools guard, preserves SecretRef coverage, adds endpoint/SSRF regression tests, updates docs/tools/firecrawl.md and docs/tools/web-fetch.md, then close linked bug #63877 as implemented and resolve or supersede overlapping #59666.
