## Summary

Keep open. Current main still maps OpenRouter/OpenAI-compatible `reasoning`, `reasoning_content`, and `reasoning_text` deltas into `type: "thinking"` blocks, while final reply delivery parses `[[reply_to_current]]` only from user-visible text blocks. Hunter Alpha is still cataloged as a reasoning-capable OpenRouter model, and the newer reasoning-only retry path does not cover OpenRouter, so the issue's central request is not implemented on main.

## Evidence

- **Issue evidence:** The reporter's follow-up in the provided GitHub context states that hunter-alpha returned the entire generated reply, including `[[reply_to_current]]`, exclusively in the OpenAI-compatible `reasoning` field, which OpenClaw stored as a `type: "thinking"` block and filtered from user-visible reply extraction.
