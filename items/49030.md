## Summary

Current `main` does not already implement this fix. A custom-named provider configured with `api: "ollama"` and `apiKey: "ollama-local"` still loses auth resolution on the subagent path, so closing this PR as implemented/obsolete would be incorrect.

## Evidence

- **`ollama-local` is treated as a non-secret marker, not a usable literal key:** `extensions/ollama/openclaw.plugin.json:7` declares `ollama-local` as a non-secret auth marker, `src/agents/model-auth-markers.ts:94` treats known non-secret markers specially, and `src/agents/model-auth.ts:153` returns `null` for non-env markers in `resolveUsableCustomProviderApiKey()`.
  - file: [src/agents/model-auth.ts:153](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/agents/model-auth.ts#L153)
  - command: `nl -ba src/agents/model-auth.ts | sed -n '114,176p'; sed -n '1,20p' extensions/ollama/openclaw.plugin.json; nl -ba src/agents/model-auth-markers.ts | sed -n '94,115p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Generic synthetic local auth is disabled when any explicit `apiKey` is present:** `resolveSyntheticLocalProviderAuth()` bails out on `hasExplicitProviderApiKeyConfig(providerConfig)`, so a custom provider that explicitly sets `apiKey: "ollama-local"` will not get the existing generic local-auth fallback.
  - file: [src/agents/model-auth.ts:329](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/agents/model-auth.ts#L329)
  - command: `nl -ba src/agents/model-auth.ts | sed -n '322,344p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Plugin-owned Ollama synthetic auth only matches the canonical provider id:** The Ollama plugin registers provider id `ollama` in `extensions/ollama/index.ts:74`, and provider hook resolution matches only the provider id or declared aliases in `src/plugins/provider-hook-runtime.ts:17` and `src/plugins/provider-hook-runtime.ts:155`.   - file: [src/plugins/provider-hook-runtime.ts:155](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/plugins/provider-hook-runtime.ts#L155)
