## Summary

Keep open. Current main still has the stricter skill-selection prompt that PR #63940 proposes to relax: it requires exactly one clearly applicable skill, tells the model to choose one when multiple apply, and forbids reading more than one skill up front. The matching prompt tests still assert that old wording. The PR is recent, focused, and not superseded by the related ClawSweeper items provided; #8813 covers a different read-tool-vs-Skill-tool concern.

## Best Possible Solution

Keep PR #63940 open for maintainer review. The best path is to rebase or reapply the focused prompt-text change on current main, keep the targeted-read guardrail, update `src/agents/system-prompt.test.ts`, and consider whether it should be coordinated with the separate #8813 follow-up about preferring a Skill tool over generic file reads.
