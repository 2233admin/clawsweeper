## Summary

Keep this issue open. Current main still has the reported Discord voice join abort path: `/vc join` waits for `manager.join`, the initial voice Ready transition uses a fixed 15s timeout, failures destroy the connection and return the reported error prefix, and the Discord voice config/schema/docs do not expose a connect timeout or retry policy. The supplied discussion is also active and concrete, with a second reporter and related open reports rather than a merged fix or maintainer-designated duplicate.

## Best Possible Solution

Keep this issue open and fix the Discord plugin voice join path. Add a focused regression test for initial `entersState` abort/timeout before session activation, then choose a longer default, bounded retry/backoff, and/or documented `channels.discord.voice.connectTimeoutMs` tuning. If a new knob lands, update `DiscordVoiceConfig`, the strict zod schema, config metadata/UI hints, and Discord docs, then reconcile related open reports #39825, #63125, and #65039 after the behavior is fixed or a maintainer chooses a canonical tracker.
