## Summary

Current `main` still falls back to a one-entry Kilocode static catalog (`kilo/auto`) on discovery failure, and the generic model resolver still surfaces catalog misses as `Unknown model`/`model_not_found`. Docs and tests both encode that behavior, so this does not meet any conservative auto-close reason.
