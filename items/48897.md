## Summary

Keep open. Current `main` still routes TUI input through `@mariozechner/pi-tui`, and the shipped `0.70.0` package still enables Kitty keyboard protocol flag 7 and still rejects printable CSI-u input when `alt|ctrl` modifiers are present. That matches the report's suspected AltGr failure mode, and I found no OpenClaw-side workaround or test coverage showing this was fixed on `main`.
