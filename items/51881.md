## Summary

Keep open. Current `main` still prefers the sniffed attachment MIME over the client-provided MIME in Gateway parsing, with no `image/apng -> image/png` normalization, and the Anthropic transport forwards that MIME unchanged as `source.media_type`. The latest release tag `v2026.4.22` has the same behavior in `src/gateway/chat-attachments.ts`, so there is no strong evidence this issue was fixed on `main` or shipped.
