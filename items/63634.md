## Summary

Keep this PR open. Current main still has the strict bundle-MCP `client.listTools()` path, so the linked MCP compatibility bug is not implemented on main. The latest PR revision adds the right high-level pieces for typeless schemas and non-object filtering, but it still replaces `Client.listTools()` with raw `client.request()`. In the pinned MCP SDK 1.29.0, `listTools()` also caches tool metadata after the request, so this needs rework before merge rather than cleanup closure.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rework `listAllTools()` so bundle MCP discovery accepts tools whose `inputSchema.type` is missing while preserving the SDK 1.29.0 `listTools()` metadata behavior, keep the v3 catalog-time rejection of explicit non-object root schemas, and add a harness regression for a configured MCP server that returns typeless `inputSchema` if feasible.
