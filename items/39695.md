## Summary

Keep #39695 open. Current main has Discord session routing and general transcript plumbing, and fresh or explicitly coding-profile configs can expose exec, but the reported Discord-bound exec/no-session-log path is still plausible for legacy or explicit messaging-profile configs. The related migration PR was closed unmerged, and the later silent-tool work was described as partial/open rather than a shipped fix for this scenario.

## Best Possible Solution

Keep #39695 open until maintainers either ship a narrow migration/doctor diagnostic for legacy or explicit messaging-profile configs that expect runtime tools, or add a targeted Discord-bound regression showing an inbound Discord turn can execute exec and persist the JSONL session transcript. If messaging profiles are intentionally chat-only, add a clear user-facing diagnostic pointing users to tools.profile="coding", tools.alsoAllow, or tools.exec.
