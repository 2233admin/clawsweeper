## Summary

Keep open. Current main still treats `configure`/`models auth` writes as single-agent operations and preserves existing per-agent overrides, so stale non-main `auth-profiles.json` entries can still remain after updating credentials on the default agent. Main now has partial OAuth inheritance/mirroring, but that is not a full cross-agent sync for `configure` / `auth add`.

## Evidence

- **`models auth` persists to one resolved agentDir:** `resolveModelsAuthContext()` resolves the configured default agent directory, and `runProviderAuthMethod()` passes that single `agentDir` into the provider auth flow and persistence path.
  - file: [src/commands/models/auth.ts:109](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/commands/models/auth.ts#L109)
  - command: `nl -ba src/commands/models/auth.ts | sed -n '104,120p'; nl -ba src/commands/models/auth.ts | sed -n '284,320p'`
- **`configure` docs still describe one-agent auth-profiles scope:** The docs say `--agent <id>` scopes `auth-profiles.json` target discovery and writes to one agent store, and `configure` targets `auth-profiles.json` for the selected agent scope.
  - file: [docs/cli/secrets.md:129](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/docs/cli/secrets.md#L129)
  - command: `nl -ba docs/cli/secrets.md | sed -n '129,140p'`
- **Agent overrides still win over main-store credentials:** The auth-store merge test explicitly verifies that main auth profiles are merged into an agent store while keeping the agent's own override; the stale agent `openai:default` key remains `agent-key` instead of adopting `main-key`.
  - file: [src/agents/auth-profiles.ensureauthprofilestore.test.ts:167](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/auth-profiles.ensureauthprofilestore.test.ts#L167)

  - file: [src/agents/auth-profiles/persisted.ts:365](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/auth-profiles/persisted.ts#L365)
