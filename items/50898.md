## Summary

Current `main` already has a bundled first-class NVIDIA provider, documented setup/configuration, provider/model routing coverage, and a post-`v2026.4.22` fix that adds native auth-choice metadata so the provider shows up in setup flows instead of only being loadable internally.

## Evidence

- **Bundled NVIDIA provider entry:** `extensions/nvidia/index.ts` registers provider id `nvidia`, links docs at `/providers/nvidia`, declares `NVIDIA_API_KEY`, and now exposes native API-key auth/wizard metadata for setup flows.
  - file: [extensions/nvidia/index.ts:6](https://github.com/openclaw/openclaw/blob/bcdacfa1b3df000ce04e3ce26bbe3051e56da3e8/extensions/nvidia/index.ts#L6)

  - file: [extensions/nvidia/provider-catalog.ts:3](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/extensions/nvidia/provider-catalog.ts#L3)

## Close Comment

Closing from a Codex review because current `main` already implements this as a first-class NVIDIA provider. The repo now has native NVIDIA provider registration and API-key setup metadata in `extensions/nvidia/index.ts`, bundled NVIDIA endpoint/model configuration in `extensions/nvidia/provider-catalog.ts`, user docs in `docs/providers/nvidia.md`, and coverage for auth resolution plus provider/model routing in `extensions/nvidia/index.test.ts`, `src/agents/models-config.providers.nvidia.test.ts`, and `src/gateway/server.sessions.gateway-server-sessions-a.test.ts`. This has not shipped in a tagged release yet, so `fixedRelease` is left null.
