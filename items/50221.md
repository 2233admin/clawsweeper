## Summary

Keep open. Current main has already moved part of this hardening into narrower command/directive gates, but #50221 is not safely closeable as implemented or obsolete: its `chat.send` authorization change is absent and was flagged as a P1 regression, while the plugin runtime config-write sink remains ungated on main.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require it to be rebased and redesigned around the current main approach: preserve targeted command capability gates for read-only vs mutating behavior, decide whether plugin commands need explicit capability metadata or a runtime config-write sink guard, and add regression tests for any remaining plugin config mutation path.
