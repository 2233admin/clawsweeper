## Summary

Keep open. Current main still lacks the requested memory-wiki bridge and lint durability fixes: link extraction remains markdown-blind, broken-link validation still compares raw targets without resolving relative Markdown links, stale-pages native links are still emitted as vault-root-relative paths from inside reports/, bridge source rendering has no renderer contract/version token and preserves raw reply tags inside the fenced source content, and cached source-page skip logic still trusts file existence without validating source-page frontmatter. The related PR #65197 was closed unmerged, and open #70986 covers only part of the parser/resolver problem, not the bridge render invalidation or malformed cached-page repair criteria.

## Best Possible Solution

Keep this issue open and implement the bounded memory-wiki fixes in maintained source with regression coverage: make link extraction markdown-aware or otherwise prevent bridged reply directives from becoming lint targets, resolve native Markdown links relative to the containing page, emit report-local native links for reports/stale-pages.md, include a bridge renderer-version token in render fingerprinting, and validate existing imported source-page frontmatter before skip. Coordinate parser/resolver overlap with #70986, but do not close this issue until the bridge cache/render invalidation and malformed cached-page repair acceptance criteria are covered too.
