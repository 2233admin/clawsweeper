## Risks / Open Questions

- The PR's parseNamedDmSessionKey/isNamedDmSessionKey design relies on parseAgentSessionKey, whose current implementation filters empty segments; the review comment correctly notes malformed keys such as agent:main:dm-named:123::work can collapse into a valid-looking parsed shape unless the PR uses stricter parsing.
- getActiveNamedSessionKey can throw when persisted activeNamedSession contains a colon. Current normalizeSessionStore only normalizes delivery/model fields, so a future implementation should validate or ignore malformed persisted named-session state instead of crashing callers.
- The PR's chat-type change adds dm-named after group/channel token checks in the old helper shape, while current main now has session-chat-type-shared.ts and send-policy.ts as separate paths. A rebased implementation should classify dm-named as direct before arbitrary user-provided name tokens can match group/channel policy.
- The PR includes broad CHANGELOG synchronization relative to a focused infrastructure feature; this is not a supply-chain risk, but it increases review noise and likely needs a narrow rebase before maintainer review.
