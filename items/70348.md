## Summary

Keep open. Current main still does not implement the PR’s requested chat.history filtering for compaction markers or pre/post-compaction prompts, and the PR itself needs revision because its prefix-based filter can drop legitimate visible messages. The existing UI also intentionally consumes synthetic compaction markers as dividers, so that part needs maintainer/product judgment rather than cleanup closure.

## Best Possible Solution

Keep the PR open but require a revision against current main’s src/gateway/chat-display-projection.ts path. The safer fix is provenance/metadata-backed filtering for internal memory-flush and post-compaction artifacts, with regression tests for chat.history/session-history snapshots, while preserving or separately deciding the intended compaction-divider UI behavior.
