## Summary

Keep #70348 open. Current main has not implemented the PR’s specific `chat.history` filtering for pre-compaction memory-flush prompts or post-compaction context-refresh prompts, and the related safer provenance-based PR #32068 was closed unmerged. The PR still needs maintainer/reviewer changes because its prefix-based filtering can drop legitimate user-authored content and it would also remove synthetic compaction markers that current main intentionally exposes for the UI divider path.

## Best Possible Solution

Keep the PR open for revision. The best path is a metadata/provenance-backed fix in the shared chat display projection, likely by tagging internal memory-flush and post-compaction refresh turns at write time, preserving or explicitly redesigning compaction divider behavior, and adding gateway/history tests that prove internal artifacts are hidden while user-authored text with the same prefixes remains visible.

## Risks / Open Questions

- Filtering `__openclaw.kind === "compaction"` from `chat.history` may remove the current UI’s compaction divider signal.
- No current-main regression test covers hiding memory-flush or post-compaction refresh artifacts from `chat.history`.
