## Summary

Keep open. Current main does not implement the PR’s requested chat.history filtering, and the remaining behavior is nuanced enough that this should not be auto-closed: compaction markers are still emitted and currently consumed by the UI as dividers, while memory-flush/post-compaction artifacts lack a shipped, safer provenance-based filter. The PR likely needs revision rather than cleanup closure.

## Best Possible Solution

Keep the PR open but request a safer revision: move/port the fix to the current chat-display-projection path, preserve or explicitly redesign the UI compaction divider behavior, and filter memory-flush/post-compaction artifacts by internal provenance or structured metadata instead of broad text prefixes. Add regression coverage for chat.history plus a false-positive case where user-authored text begins with the same phrases.
