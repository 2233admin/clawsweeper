## Summary

Keep open. Current `main` still treats Ollama query and document embeddings the same way, so the requested behavior is not implemented. The PR is also stale against the current Ollama plugin layout and has an unresolved correctness issue from review (`String.replace` with literal `$` sequences), so this needs rework rather than conservative closure.

## Evidence

- **Current main still uses identical query and document embedding flow for Ollama:** `createOllamaEmbeddingProvider()` defines one `embedOne()` helper, assigns `embedQuery: embedOne`, and implements `embedBatch` as `Promise.all(texts.map(embedOne))`. There is no Ollama query-specific prefixing or asymmetric query/document handling on `main`.
