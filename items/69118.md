## Summary

Keep #69118 open. Current main has a partial static-prompt hashing fix for volatile inbound metadata, but the specific group-channel failure remains: `extraSystemPromptStatic` still includes `groupIntro`, while `groupIntro` is intentionally present on the first group turn and absent on later turns. Because `resolveCliSessionReuse` still invalidates on `extraSystemPromptHash` mismatch, this can still produce `reason=system-prompt` on turn 2.

## Best Possible Solution

Keep the issue open and patch the group-channel path so CLI session reuse hashes only prompt material that is stable across normal turn transitions. Concretely, exclude turn-variable `groupIntro` from `extraSystemPromptStatic` or make the reuse comparator tolerant of expected extra-system-prompt drift. Add a focused regression test that simulates a group-channel first turn followed by a second turn and asserts reuse instead of `invalidatedReason: "system-prompt"`.
