## Summary

Keep #69998 open. Current main still accepts secret-bearing `apiKey` and `env` values in `skills.update`, writes them to config, and returns `config: current` directly without `redactConfigObject()`. The related implementation attempts #66928 and #66986 are closed unmerged, and this PR remains a focused active fix for open bug #66769.

## Best Possible Solution

Keep #69998 open. The best path is to merge a focused fix for #66769 that either redacts or omits the secret-bearing response config and also handles redacted-sentinel round trips safely, with the PR's regression test covering response redaction and persisted full values.
