## Summary

Keep #69998 open. Current main still echoes the updated `skills.update` config object directly, including `apiKey` and `env` values. The PR is a focused fix for open bug #66769, and the related PRs #66928/#66986 are closed unmerged rather than canonical replacements. The remaining maintainer question is how to handle redacted sentinel round-trips before merge.

## Best Possible Solution

Keep #69998 open. The best fix is to either restore redacted sentinel values from the existing skill config before writes, matching the config write-endpoint contract, or change the `skills.update` success payload to avoid returning writable secret-bearing config. Keep the regression test that proves response redaction and full-value persistence, then merge and close #66769.
