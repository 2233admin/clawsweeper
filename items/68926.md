## Summary

Keep #68926 open. Current main already has chat-header session/model/thinking controls, but it still lacks the PR's central dedicated agent selector in chat controls, and mobile controls still do not expose the model selector. The PR is focused web UI work, not implemented or superseded, though it needs rebase/rework for the bot-reported correctness concerns.

## Best Possible Solution

Keep the PR open. The best path is a narrow rework against current main that adds a chat-header agent selector beside the existing session/model/thinking controls, keeps the model/thinking logic in `ui/src/ui/chat/session-controls.ts` or a small shared helper, and adds focused tests for multi-agent switching, legacy/non-agent session fallback, mobile controls, and session-specific thinking options.
