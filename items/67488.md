## Summary

Keep open. Current `main` has a malformed tool-call argument repair path for certain provider streams, but the implementation on `main` is still ASCII-prefix constrained and the local tests only cover ASCII prefix/suffix corruption. That aligns with the issue timeline’s still-open PR `#67551` ('Handle Unicode smart quotes in tool call argument JSON'), so there is not strong evidence that `main` already fixes this report.

## Evidence

- **Repair is active on the affected embedded stream path:** Cron isolated agent turns flow through the embedded runner’s stream wrapper, and `attempt.ts` still wires malformed tool-call argument repair only as a live wrapper around the provider stream; there is no separate cron-side serialization fix here.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:1733](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/agents/pi-embedded-runner/run/attempt.ts#L1733)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)

Unicode smart quotes are not accepted by this matcher, so the current repair path does not clearly cover the linked PR’s stated smart-quote case.
