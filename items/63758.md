## Summary

Current main still leaves this issue unresolved. Managed cron task records are created and finalized, but the cron producer paths do not set progressSummary at creation time or call the task progress API during execution, while /tasks, /status, and session_status only show live task detail for active tasks when progressSummary is present.

## Best Possible Solution

Keep this issue open. Add coarse cron-owned progressSummary values when scheduled, startup catch-up, and manual cron task runs are created, update them at meaningful execution phases via the existing task progress API, and add regression coverage proving active cron work appears with live detail in /tasks, /status, or session_status before terminalSummary is written.
