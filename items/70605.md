## Summary

Keep PR #70605 open. Current main still forwards context-file content through `sanitizeContextFileContentForPrompt()` without removing Markdown or HTML strikethrough spans, and the existing tests cover context-file injection but not this sanitizer behavior. The PR is a recent, focused bug-fix attempt with author follow-up and bot-review fixes, but it should not be closed as implemented or obsolete. It also should not be treated as fully resolving #47668, which is a broader context-growth issue already covered by separate truncation/compaction mechanisms.

## Best Possible Solution

Keep the PR open for maintainer review. If the product decision is to treat strikethrough in workspace context files as removed/deprecated content, implement it in `sanitizeContextFileContentForPrompt()` with focused regression tests for inline, multiline, attributed HTML tags, unpaired markers, and tilde-fenced code blocks. Track broader context growth under #47668 or narrower compaction/truncation follow-ups rather than closing those from this PR alone.
