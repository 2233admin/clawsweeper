## Summary

Keep open. Current `main` does not implement this behavior: struck-through text from injected context files still flows into the agent system prompt, and the PR remains unmerged with review discussion on regex edge cases.

## Evidence

- **Sanitizer on main does not strip strikethrough:** `sanitizeContextFileContentForPrompt()` only removes the default heartbeat prompt block and collapses extra blank lines. There is no handling for markdown `~~...~~` or HTML `<s>/<del>/<strike>` content on current `main`.

  - file: [src/agents/system-prompt.ts:121](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/agents/system-prompt.ts#L121)
