## Summary

Keep open. Current `main` exposes auth health checks and some passive UI surfacing, but it does not implement the requested built-in global monitoring flow: there is no `authMonitoring` config, no core scheduled checker, and no built-in proactive broken/recovered notification path across providers.

## Evidence

- **CLI health check exists, but it is only an exit-code check:** `models status --check` returns `1` for expired/missing auth and `2` for expiring auth; this is a manual/automation-facing check, not a built-in monitor.
  - file: [src/commands/models/list.status-command.ts:365](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/commands/models/list.status-command.ts#L365)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Docs still describe auth monitoring as external automation:** Authentication docs present `openclaw models status --check` as an automation-friendly command, not a built-in scheduler/notification feature.
  - file: [docs/gateway/authentication.md:104](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/docs/gateway/authentication.md#L104)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Repo docs explicitly classify auth-monitor scripts as optional extras:** The scripts guide says auth-monitoring scripts are optional systemd/Termux helpers, which contradicts the issue's requested built-in global feature.
  - file: [docs/help/scripts.md:14](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/docs/help/scripts.md#L14)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Existing monitor script is not core and is Claude-specific:** `scripts/auth-monitor.sh` is an external shell script wired to `~/.claude/.credentials.json`, with ad hoc dedupe and notification env vars; it is not a provider-agnostic built-in subsystem.
  - file: [scripts/auth-monitor.sh:2](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/scripts/auth-monitor.sh#L2)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **No proposed global config surface exists on main:** Search for `authMonitoring`, `warnBeforeHours`, `notifyOnBroken`, `notifyOnRecovered`, `notifyOnExpiring`, and `dedupeWindowHours` returned no matches under `src/config`, `docs`, `ui`, `packages`, or `extensions`.
  - command: `rg -n "authMonitoring|warnBeforeHours|notifyOnBroken|notifyOnRecovered|notifyOnExpiring|dedupeWindowHours" src/config docs ui packages extensions -g '!**/node_modules/**'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Control UI has passive attention state, not proactive notification:** The UI can show expired/expiring model-auth attention items once status is loaded, but that is not an independent background notifier or recovery-alert system.
  - file: [ui/src/ui/app-settings.ts:731](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/ui/src/ui/app-settings.ts#L731)

## Risks / Open Questions

- `gh` read access was unavailable in this environment because `GH_TOKEN` was not set, so this review relies on the checked-out `main` tree rather than additional GitHub discussion/history.
