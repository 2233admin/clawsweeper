## Summary

Keep open. Current main still fingerprints `auth-profiles.json` by file mtime, so no-op rewrites can still invalidate the models ready cache. The PR addresses a real, current cache behavior and is not implemented or superseded on main; it still needs normal maintainer review for the remaining OAuth-token fingerprint tradeoff noted in review.

## Best Possible Solution

Keep #70134 open for maintainer review. The best path is a semantic auth-profile fingerprint in `src/agents/models-config.ts` that preserves profile order, invalidates on all credential inputs that can affect provider catalog output, keeps `models.json` mtime invalidation, and includes focused tests for no-op rewrites, real auth changes, SecretRefs, OAuth, normalized legacy aliases, parse/read errors, and models file changes.
