## Summary

Keep PR #70134 open. Current main still fingerprints `auth-profiles.json` by file mtime, so a no-op rewrite still changes the `ensureOpenClawModelsJson` ready-cache fingerprint and can force replanning. The PR is a plausible targeted fix with focused tests, but it is not merged and still has a branch-specific OAuth token-content trade-off called out in review, so there is no conservative auto-close reason.

## Best Possible Solution

Keep the PR open for maintainer review. The likely path is to land a semantic auth-profile fingerprint in `src/agents/models-config.ts` or a helper module, with regression tests for no-op rewrites, meaningful API-key/SecretRef/OAuth changes, profile reordering, parse/read errors, and `models.json` mtime invalidation. Before merge, maintainers should resolve the OAuth token-content fingerprint trade-off raised in review so the ready cache cannot skip replanning when token changes affect discovery entitlements.
