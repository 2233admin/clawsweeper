## Summary

Keep PR #59322 open. Current main still renders the shared iOS/macOS chat composer session picker with a 160pt max-width but no picker-level line limit or truncation modifier, and the PR targets that exact code path. The PR is still draft and the tail-truncation UX should be revisited because channel-backed session keys often differ at the suffix, but this remains plausible core app UI work rather than an implemented, duplicate, ClawHub, or unactionable item.

## Best Possible Solution

Keep the PR open, ask for a small revision that preserves distinguishing suffixes for channel-backed session keys, and verify the composer toolbar on iOS and macOS with long session names. A maintainer could also implement the same fix directly in `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatComposer.swift`, preferably using middle truncation or a compact display label rather than tail-only truncation.
