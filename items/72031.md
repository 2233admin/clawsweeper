## Summary

Keep #72031 open. Current main still has the reported core bug: the image path and media runner path require a concrete API key before dispatch, while Bedrock `aws-sdk` auth intentionally resolves without `apiKey`. The related fixes #72066 and #72092 are open/unmerged in the provided GitHub context, so the issue is not implemented on main yet.

## Best Possible Solution

Keep #72031 open until a fix lands on main. The best core fix is to mirror the existing chat-path missing-key allowance for Bedrock `aws-sdk` auth in `src/media-understanding/image.ts` and `src/media-understanding/runner.entries.ts`, skip storing empty runtime API keys, and add regression coverage for the `amazon-bedrock` image path. PR #72092 appears to be the better-scoped candidate because it covers image plus audio/video media paths; #72066 covers the narrower image-only path.
