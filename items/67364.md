## Summary

Keep open. I did not find a current-main fix for this Control UI transcript bug: `chat.history` still reads raw persisted transcript messages, its sanitizer only drops assistant-side control replies, the UI still labels any `user` role as the local user, and heartbeat runs still use the main session unless `heartbeat.isolatedSession` is explicitly enabled.

## Evidence

- **chat.history reads raw transcript messages:** `readSessionMessages()` parses each JSONL line and pushes every `parsed.message` into the returned history with only sequence metadata added; there is no heartbeat/internal-role filtering at read time.
  - file: [src/gateway/session-utils.fs.ts:93](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/gateway/session-utils.fs.ts#L93)
  - command: `nl -ba src/gateway/session-utils.fs.ts | sed -n '93,121p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **chat.history sanitizer is assistant-only:** `shouldDropAssistantHistoryMessage()` exits early unless `entry.role === "assistant"`, and `sanitizeChatHistoryMessages()` only applies that assistant-side drop path before returning messages to the client. There is no corresponding user-role suppression for heartbeat/internal prompts here.
  - file: [src/gateway/server-methods/chat.ts:1213](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/gateway/server-methods/chat.ts#L1213)

  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Control UI still renders user-role entries as the local user:** The grouped chat renderer maps normalized `user` messages to the resolved local user label. If an internal heartbeat prompt is still persisted as `role: "user"`, Control UI will continue to attribute it to the user.
  - file: [ui/src/ui/chat/grouped-render.ts:303](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/ui/src/ui/chat/grouped-render.ts#L303)

- **Heartbeat isolation is opt-in, not the default behavior:** The heartbeat runner only switches to a separate `:heartbeat` session when `heartbeat?.isolatedSession === true`. The adjacent test confirms that when `isolatedSession` is not set, the run uses the main session key instead.
  - file: [src/infra/heartbeat-runner.ts:798](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/infra/heartbeat-runner.ts#L798)
