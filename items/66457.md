## Summary

Keep open. Current `main` still uses the old hard cap behavior for gateway logs, so the underlying request in #66394 is still valid, but this PR is not a conservative cleanup candidate: the logging change has unresolved review findings, and the PR also bundles unrelated ACPX runtime behavior changes with contract drift.
