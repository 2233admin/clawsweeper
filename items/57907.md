## Close Comment

Closing this as a Codex review because current `main` no longer matches the reported failure mode.

- `openclaw configure --section model` still routes through the provider auth flow, and that flow now intentionally persists the provider-returned model ref (`src/commands/configure.gateway-auth.ts:149`, `src/plugins/provider-auth-choice.ts:285`).
- Current tests explicitly expect Anthropic Claude CLI setup to save `claude-cli/claude-sonnet-4-6` and matching `claude-cli/...` fallback refs, with notes saying OpenClaw switched Anthropic model selection to the local Claude CLI backend (`src/commands/models/auth.test.ts:474`, `src/commands/models/auth.test.ts:563`).
- Current docs define CLI backend provider ids as the left side of model refs, and document the bundled Anthropic `claude-cli` backend as supported (`docs/gateway/cli-backends.md:96`, `docs/gateway/cli-backends.md:152`).
- Current tests also treat `claude-cli/claude-opus-4-6` as an Anthropic-family model ref rather than an unknown model (`extensions/qa-lab/src/providers/mock-openai/server.test.ts:2415`).
