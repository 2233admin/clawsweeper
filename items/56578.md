## Summary

Keep open. Current `main` at `6b618f063516aa9ba85903aa4565c5a204c055b2` still contains the read-back/mismatch warning path this PR is trying to change, so it is not implemented already, but the PR as described is not safe to close-cleanup either: `dispatchCommandInteraction` has multiple success-without-apply early-return paths, so unconditionally showing `✅ Model set...` would create false confirmations, and the updated test assertion called out in review is valid because the picker notice payload has no `content` field.
