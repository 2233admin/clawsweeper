## Summary

Keep this PR open. Current main still cannot clear cron payload model, fallbacks, or thinking overrides with null, and this PR is a focused implementation candidate. It still needs a small revision before landing because the current PR diff preserves null in the shared create/patch normalization path, which can regress cron.add callers that serialize optional fields as null.

## Best Possible Solution

Keep this PR open and request a focused revision: make null-clear a cron.update patch-only contract for model, fallbacks, and thinking; preserve null through normalizeCronJobPatch without changing normalizeCronJobCreate optional-null compatibility; delete those fields in mergeCronPayload when patch values are null; coerce null to undefined only when building a replacement payload; add regression coverage for normalizeCronJobPatch plus applyJobPatch and a cron.add optional-null compatibility test. If the web UI is in scope, also emit null when editing an existing job and clearing a previously set model/thinking override.
