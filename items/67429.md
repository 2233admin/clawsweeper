## Summary

Keep open. Current `main` still treats `payload.model`, `payload.fallbacks`, and `payload.thinking` nulls as no-ops on `cron.update`, while the PR discussion also surfaced an unresolved create-path regression in the proposed fix, so there is not strong evidence to conservatively close this PR.

Those review comments make this useful work, but not safely obsolete.

## Risks / Open Questions

- Current `main` still lacks a supported way to clear per-job `model`, `fallbacks`, and `thinking` overrides through `cron.update`.
- The proposed fix path needs a patch-only null-preservation strategy or equivalent split so `cron.add` keeps its existing normalization behavior.
