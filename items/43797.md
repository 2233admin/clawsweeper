## Summary

Keep open. Current main still does not remove Docker sandbox workspace directories during prune: the prune path removes the runtime and registry entry, and the Docker backend only runs `docker rm -f`. The related fix PRs #43835 and #43941 are still open, while #43831 was closed unmerged, so the original bug remains unimplemented on main.

## Evidence

- **Prune path removes runtime and registry only:** `pruneSandboxRegistryEntries` calls `removeRuntime(entry)`, then `remove(entry.containerName)`, and optionally `onRemoved`; `pruneSandboxContainers` does not provide any `onRemoved` cleanup for workspace directories.
  - file: [src/agents/sandbox/prune.ts:54](https://github.com/openclaw/openclaw/blob/b1b29a8fc210a0772eb3728e03783890586e07f8/src/agents/sandbox/prune.ts#L54)
  - command: `nl -ba src/agents/sandbox/prune.ts | sed -n '38,79p'`
  - sha: [b1b29a8fc210](https://github.com/openclaw/openclaw/commit/b1b29a8fc210a0772eb3728e03783890586e07f8)
- **Docker runtime removal only removes the container:** The Docker sandbox backend `removeRuntime` implementation only invokes `docker rm -f <containerName>` and does not remove the corresponding workspace directory under the sandbox workspace root.
  - file: [src/agents/sandbox/docker-backend.ts:143](https://github.com/openclaw/openclaw/blob/b1b29a8fc210a0772eb3728e03783890586e07f8/src/agents/sandbox/docker-backend.ts#L143)
  - command: `nl -ba src/agents/sandbox/docker-backend.ts | sed -n '115,150p'`
  - sha: [b1b29a8fc210](https://github.com/openclaw/openclaw/commit/b1b29a8fc210a0772eb3728e03783890586e07f8)
- **Workspace directory resolver exists but is not used by prune:** `resolveSandboxWorkspaceDir(root, sessionKey)` computes the per-scope workspace directory, but `src/agents/sandbox/prune.ts` does not import or call it.
  - file: [src/agents/sandbox/shared.ts:18](https://github.com/openclaw/openclaw/blob/b1b29a8fc210a0772eb3728e03783890586e07f8/src/agents/sandbox/shared.ts#L18)
  - command: `nl -ba src/agents/sandbox/shared.ts | sed -n '1,46p'; rg -n "resolveSandboxWorkspaceDir|fs\.rm" src/agents/sandbox/prune.ts`
  - sha: [b1b29a8fc210](https://github.com/openclaw/openclaw/commit/b1b29a8fc210a0772eb3728e03783890586e07f8)
- **Related fixes have not landed:** Provided GitHub context shows PR #43831 was closed unmerged, and PRs #43835 and #43941 remain open as attempted fixes for this exact issue.
