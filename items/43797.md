## Summary

Keep this issue open. Current main still removes sandbox runtimes and registry entries without deleting the local sandbox workspace directory, and #43835 is still an open, unmerged implementation candidate that uses closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review #43835 as the implementation candidate. The fix should live in `src/agents/sandbox/prune.ts`: after successful runtime removal for non-shared entries, derive the workspace path from `cfg.workspaceRoot` and the registry scope key, remove it recursively with safe error handling, preserve shared and still-in-use agent workspaces, and add focused tests proving the cleanup behavior.
