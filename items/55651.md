## Summary

Keep open: current `main` still handles Discord slash commands with only the local instance's allowlist checks, and the linked ownership-claim fix for multi-instance bot tokens is still an open PR rather than shipped code.

## Evidence

- **Slash-command deny path still uses only local config:** `dispatchDiscordCommandInteraction` resolves `guildInfo`/`channelConfig` from the current config and replies `This channel is not allowed.` when the local channel config or group-policy authorizer rejects the guild channel. There is no sibling-instance ownership skip before those replies.
