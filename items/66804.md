## Summary

Keep open. Current main still bounds Active Memory with a 15s default timeout and fails open with no injected memory when the selected recall model times out. The implemented `modelFallback` path is only a pre-run model selection fallback when no explicit/current/agent model resolves, not a retry after MiniMax-M2.7 or another selected model times out. The issue also has multiple recent user confirmations, including OpenClaw 2026.4.22.

## Best Possible Solution

Keep this issue open unless maintainers intentionally consolidate it into #65309. The likely fix belongs in `extensions/active-memory/index.ts`: add bounded fallback-on-timeout behavior for `config.modelFallback`, or produce a local `memory_search`-based summary when the selected recall model times out, without extending the foreground wait beyond the configured budget. Add regression coverage for selected-model timeout fallback and live MiniMax validation when credentials are available.
