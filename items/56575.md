## Summary

Keep open. Current `main` has adjacent cache/pruning and idle-reset features, but it does not implement this PR’s explicit per-model time-based session maintenance modes (`sessionCacheTtl` + `timeBasedContextCompact`) or the gateway maintenance sweep that would act on recorded session activity timestamps.
