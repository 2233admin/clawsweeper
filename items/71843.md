## Summary

Keep PR #71843 open. Current main still propagates Slack `already_reacted` from the agent-facing `react` action, while the PR is a focused fix for the linked issue #69005 with regression coverage. The existing status-reaction path already treats `already_reacted` as a no-op, so this is a real behavior asymmetry rather than a cleanup-only change.

## Best Possible Solution

Keep #71843 open for maintainer review. Land this PR or an equivalent Slack plugin patch that treats Slack `already_reacted` as idempotent success for the agent `react` tool, returns the normal `{ ok: true, added: emoji }` payload, preserves propagation for unrelated Slack errors, and keeps the focused regression tests. If maintainers want broader idempotency, consolidate it intentionally with the older helper-level Slack reaction PRs rather than closing this useful fix as stale.
