## Summary

Keep PR #71843 open. Current main still propagates Slack `already_reacted` from the agent-facing `react` action, while the PR is a focused fix for linked issue #69005 with regression coverage. The existing Slack status-reaction path already treats duplicate reaction adds as a no-op, so this is a real behavior gap rather than stale cleanup.

## Best Possible Solution

Keep #71843 open for maintainer review. Land this PR or an equivalent Slack plugin patch that treats Slack `already_reacted` as idempotent success for the agent `react` tool, returns the normal `{ ok: true, added: emoji }` payload, preserves propagation for unrelated Slack errors, and keeps focused regression tests. If maintainers want broader idempotency, consolidate it intentionally with the older helper-level Slack reaction PRs rather than closing this useful fix as stale.
