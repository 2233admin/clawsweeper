## Summary

Keep this PR open. Current main still contains the Windows schtasks restart stdout writes that can be emitted again during the stale-PID retry path, so the reported duplicate restart output is still plausible. The PR is not ready as-is because it is stale/unfocused, lacks a targeted regression test, and bundles unrelated browser SSRF/CDP policy changes into a Windows gateway output fix.

## Best Possible Solution

Keep the PR open, but request a focused gateway-only update: rebase onto current main, preserve exactly one human success message for completed Windows restarts, de-duplicate stdout across stale-PID retries, add a targeted regression test for the retry/output behavior, and split the unrelated browser SSRF/CDP changes into their own PR or drop them from this branch.
