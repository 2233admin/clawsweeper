## Summary

Keep open. Current main still exposes session maintenance as global retention/cap/disk controls only. There is no accepted `session.maintenance.pinnedKeys` field, no `/pin` session command or equivalent protected-session state, and the enforcement paths only preserve the active session key during one save path rather than arbitrary pinned sessions/topics. The request is a core session-retention capability, not a ClawHub/plugin cleanup item.
