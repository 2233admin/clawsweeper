## Summary

Keep open. Current main still does not provide the proposed gateway startup auth warning, and skill discovery only has narrow one-level plus `dir/skills/*` handling rather than the bounded recursive discovery this PR proposes. The PR also has unresolved review blockers around auth-store I/O, false auth warnings, and scan limits.

## Best Possible Solution

Keep the PR open or ask for a focused rework. The recursive skill discovery work should use a bounded candidate budget, preserve root-containment and symlink protections, update watcher coverage, and add nested bundle tests. The startup auth warning should be separate, best-effort, read-only, and based on the existing `hasAvailableAuthForProvider()` runtime contract so it cannot block gateway startup or warn for valid auth sources.
