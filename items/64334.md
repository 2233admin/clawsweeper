## Summary

Keep this issue open. Current main still does not add a cumulative transcript-backed `Session cost` line to `/status` or `session_status`; it only shows an API-key-gated estimated `Cost:` based on displayed token counts. The issue is also linked by open, unmerged PR #72517 with closing syntax, so repo policy says the issue should stay open until that PR is reviewed, landed, or closed.

## Best Possible Solution

Keep this issue open. Review PR #72517 as the active closing candidate; if it remains README-only, request code changes or close it. The product fix should add a cumulative transcript-backed `Session cost` line in the shared `/status` and `session_status` renderer, reuse existing session cost aggregation, preserve current cost-visibility rules, and add focused tests for multi-turn totals plus OAuth/token hiding.
