## Summary

Keep open. Current main does show a `/status` cost line, but it is the estimated last response/run cost derived from the current session entry tokens, not the requested cumulative session cost summed from transcript `usage.cost.total`. The cumulative transcript-backed cost surface exists today under `/usage cost`, so the requested `/status` behavior remains missing.

## Best Possible Solution

Keep the issue open and add a dedicated cumulative session-cost line to `/status` and the `session_status`-equivalent surface, reusing the existing transcript-backed `loadSessionCostSummary` behavior or a cached/persisted session total. Keep the existing last-reply estimated-cost display distinct so users can see both current-run and cumulative spend.
