## Summary

Current `main` has partial mitigation paths for macOS TCC scoping, but not a clear repo-wide implementation of the requested fix. OpenClaw documents a local-prefix installer that puts Node under `~/.openclaw`, and the macOS app’s CLI installer calls that flow, but the primary install docs still recommend the standard installer/global package path and the iMessage docs still require Full Disk Access for the process context running OpenClaw/`imsg`. That is not strong enough to say the dedicated/bundled-Node request is implemented on `main`.

## Risks / Open Questions

- The repo shows partial mitigation, but the requested behavior is not clearly the default or fully documented as fixed for macOS TCC-sensitive integrations like iMessage.
- Docs are internally mixed: `docs/platforms/macos.md` still says the app installs the global CLI via npm/pnpm/bun, while the current Swift app code uses `install-cli.sh`. That ambiguity weakens any high-confidence close decision.
