## Summary

Current main still preserves the reported problematic behavior: the Moonshot thinking wrapper changes pinned tool-choice requests with enabled thinking to `thinking: { type: "disabled" }` instead of falling back to `tool_choice: "auto"` or clearing reasoning fields. Existing tests also assert that disabled-thinking behavior, and the related fix PRs in the provided context were closed unmerged rather than landing on main.
