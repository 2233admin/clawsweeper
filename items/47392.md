## Close Comment

Closing this after a Codex review because the report no longer matches current `main`. This issue was filed against `@larksuite/openclaw-lark v2026.3.15`, but current OpenClaw ships Feishu as the bundled `feishu` plugin (`docs/channels/index.md:16`, `extensions/feishu/openclaw.plugin.json:1`), and the current docs target OpenClaw 2026.4.24+ (`docs/channels/feishu.md:19`). On current `main`, the Feishu inbound path preserves routable reply metadata (`extensions/feishu/src/bot.ts:1008`), reset-hook replies route back through the originating channel/target (`src/auto-reply/reply/commands-reset-hooks.ts:119`), and bare `/new` turns are covered so they use the normal agent reply path instead of a separate reset notice (`src/auto-reply/reply/get-reply-run.ts:771`, `src/auto-reply/reply/get-reply-run.media-only.test.ts:503`). Given that the old external-plugin failure mode is obsolete against current `main`, I’m closing this as no longer reproducible. If you can still reproduce it on a current 2026.4.24+ build, please open a new issue with the exact Feishu mode (DM/group/topic), config, and logs.
