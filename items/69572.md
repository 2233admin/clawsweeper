## Summary

Keep #69572 open. Current main still implements Feishu message typing as a `Typing` message reaction, and the Feishu docs/tests still describe and assert typing reactions. The related #8722/#43084 cleanup reports show the reaction-based feature and keepalive behavior were previously considered implemented, but they do not resolve this newer report that native Feishu typing/status should replace the reaction UX.

## Best Possible Solution

Keep this issue open and treat it as a Feishu plugin behavior bug. The likely fix belongs in `extensions/feishu/src/typing.ts` and `extensions/feishu/src/reply-dispatcher.ts`: replace message-reaction typing with the native Feishu/Lark typing or follow-up-bubble API after confirming the correct endpoint semantics, preserve `typingIndicator` gating and backoff handling, update cleanup/keepalive behavior as needed, and change Feishu tests/docs so they no longer describe message reactions as the typing indicator implementation.
