## Summary

Current main already provides authenticated real-time agent observability over the gateway WebSocket protocol (`agent`, `session.tool`, `sessions.changed`), so this PR's separate `/api/events/stream` SSE route is redundant rather than missing functionality.

## Risks / Open Questions

- Current main exposes this capability over authenticated WebSocket events, not a browser `EventSource`/SSE endpoint, so clients hard-coded to `/api/events/stream` would still need to use the existing WS protocol instead.
- The existing gateway event model is run/session scoped rather than a new global HTTP event feed, so this closure assumes that satisfies the PR's underlying observability goal.

## Close Comment

Closing after Codex review. Current `main` already exposes authenticated real-time agent observability through the gateway WebSocket protocol, so this separate SSE endpoint is redundant. The runtime subscribes to `onAgentEvent()` in `src/gateway/server-runtime-subscriptions.ts:35`, streams those payloads to clients in `src/gateway/server-chat.ts:880`, and already surfaces the relevant event families as `agent`, `session.tool`, and `sessions.changed` in the documented gateway protocol (`docs/gateway/index.md:304`, `docs/gateway/protocol.md:385`). The release commit for `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) already contains the same implementation, so there is no remaining gap on `main` for this PR to land.
