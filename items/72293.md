## Summary

Keep #72293 open. Current main still does not propagate a terminal assistant `stopReason: "aborted"` from the embedded runner lifecycle path into subagent run status/announce outcome. There is a helper that can map an aborted stop reason to `killed`/`abortedLastRun: true`, but the relevant lifecycle event currently omits that stop reason, and the subagent registry treats a plain lifecycle `end` as `ok`.

## Best Possible Solution

Patch the embedded lifecycle/subagent path so a final assistant `stopReason: "aborted"` with empty content/error message is surfaced as an aborted or failed terminal state, with `abortedLastRun`/run outcome preserved and a parent-visible failure announcement. Add focused regression tests for `handleAgentEnd` and subagent registry/announce behavior covering an empty aborted assistant turn.
