## Summary

Current `main` still has the reported product gap. Media-understanding outputs carry provider/model telemetry, and provider/CLI execution populates it, but the formatter that feeds `BodyForAgent` renders only `[Image]`/`[Audio]`/`[Video]` text blocks without an `Analysis model:` line. The issue discussion’s local validation remains relevant rather than obsolete.

## Best Possible Solution

Keep this issue open. Implement a narrow media-understanding formatter/apply-path change so successful, non-synthetic current-turn outputs render an `Analysis model:` line from `output.provider` and `output.model` when available. Add focused coverage in `src/media-understanding/format.test.ts` and an apply-path regression in `src/media-understanding/apply.test.ts` proving `BodyForAgent` receives actual telemetry while missing or synthetic telemetry is omitted or clearly labeled.

## Risks / Open Questions

- A fix should expose only current-turn `output.provider`/`output.model`; it should not infer the conversation model or reuse stale fallback state.
- Synthetic placeholder outputs such as `openclaw/synthetic-empty-audio` are present in the same output type, so the formatter should omit or clearly label them instead of presenting them as real analysis-model telemetry.
