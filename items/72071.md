## Summary

Keep #72071 open. Current main has partial safeguards: structured `toolCall` blocks are not extracted as assistant-visible text, and tagged/phased reasoning is suppressed. But the Ollama/Kimi path still treats unphased `message.content` text in a tool-use turn as visible reply text, closes/emits that text before structured `tool_calls`, and Telegram sends non-empty reply payload text. That leaves the reported Kimi pre-tool reasoning/content leak unresolved enough that conservative cleanup should not close it.

## Best Possible Solution

Keep the issue open and fix it in the Ollama/assistant-delivery boundary. The likely maintainer path is to classify Ollama/Kimi `content` emitted in a `toolUse` turn as commentary/thinking when it precedes structured `tool_calls`, or suppress unphased assistant text for tool-use messages unless a final-answer phase/text block exists. Add regression coverage in `extensions/ollama/src/stream-runtime.test.ts` and the Telegram/block-reply subscriber path so structured tool calls stay internal and pre-tool reasoning is not sent to Telegram.
