## Summary

Keep this issue open. Current main still has the reported behavior: the enqueue-followup path re-checks stale liveness but then unconditionally calls `typing.cleanup()` even when `isRunActive()` remains true, so a real queued followup behind an active run still loses its typing/liveness signal.

## Best Possible Solution

Fix `src/auto-reply/reply/agent-runner.ts` by computing the post-enqueue liveness state once: drain immediately when the active snapshot is stale, but keep or refresh typing for a true queued-behind-active-run case instead of calling `typing.cleanup()` immediately. Add focused regression tests for stale drain and still-active queued liveness, including an assertion that cleanup is not called while `isRunActive()` remains true.
