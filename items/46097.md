## Summary

Current `main` already implements the reported handshake-timeout fix: the pre-auth gateway handshake default is now 10 seconds, production supports `OPENCLAW_HANDSHAKE_TIMEOUT_MS`, the websocket server uses that shared timeout, and the changelog records this as the 2026.3.22 fix for dropped healthy CLI connections at 3 seconds.

## Evidence

- **Default timeout raised and prod env override added:** `DEFAULT_PREAUTH_HANDSHAKE_TIMEOUT_MS` is `10_000`, and `getPreauthHandshakeTimeoutMsFromEnv()` reads `OPENCLAW_HANDSHAKE_TIMEOUT_MS` in normal runtime.
  - file: [src/gateway/handshake-timeouts.ts:1](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/gateway/handshake-timeouts.ts#L1)
- **Gateway websocket server uses shared pre-auth timeout:** The pre-auth handshake timer now calls `getPreauthHandshakeTimeoutMsFromEnv()` before closing unauthenticated sockets.
  - file: [src/gateway/server/ws-connection.ts:366](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/gateway/server/ws-connection.ts#L366)
- **Regression coverage for override behavior:** Tests explicitly verify `OPENCLAW_HANDSHAKE_TIMEOUT_MS` is preferred and falls back only when empty.
  - file: [src/gateway/handshake-timeouts.test.ts:21](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/gateway/handshake-timeouts.test.ts#L21)

  - file: [CHANGELOG.md:2442](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/CHANGELOG.md#L2442)
