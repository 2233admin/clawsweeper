## Summary

Keep PR #69567 open. Current main still has 12 custom checkout blocks in `.github/workflows/ci.yml` that derive `auth_header` from `${{ github.token }}` and pass it to `git` as `http.extraheader` without any `::add-mask::` registration. The PR discussion is focused and includes author verification plus bot review support; the work may need a rebase against current main, but it is not obsolete or implemented.

## Best Possible Solution

Keep the PR open and rebase it onto current main. The best fix remains a workflow-only hardening in `.github/workflows/ci.yml`: add `echo "::add-mask::${auth_header}"` immediately after each derived `auth_header` assignment and before the first `git` command that uses it, then let the existing workflow audit/CI validate the updated file.
