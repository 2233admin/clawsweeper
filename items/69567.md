## Summary

Keep open. Current main still constructs base64 Basic-auth checkout headers from `github.token` and passes them to `git` without registering the derived value with `::add-mask::`. The PR addresses a real CI log-hardening gap, but the provided PR scope only changes `.github/workflows/ci.yml`; current main also has the same custom checkout pattern in `.github/workflows/ci-check-testbox.yml`, so maintainer review should keep this open and likely ask for that remaining workflow to be covered too.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to add `echo "::add-mask::${auth_header}"` immediately after every workflow `auth_header` assignment, including `.github/workflows/ci-check-testbox.yml`, then validate through the existing workflow-audit path (`security-scm-fast` / `zizmor`) and normal CI before merging.
