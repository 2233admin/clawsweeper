## Summary

Keep PR #69567 open. Current main still constructs base64 `x-access-token` checkout auth headers in custom checkout blocks and passes them to `git` via `http.extraheader` without registering the derived value with `::add-mask::`. The PR is a plausible focused hardening change, but it should be rebased/expanded because current main also has the same unmasked custom checkout pattern in `.github/workflows/ci-check-testbox.yml`, while the PR file list only covers `.github/workflows/ci.yml`.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, add `::add-mask::${auth_header}` immediately after every current `auth_header` construction before any `git` use, including `.github/workflows/ci-check-testbox.yml`, then run the relevant workflow/security validation for changed workflow files.
