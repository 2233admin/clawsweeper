## Summary

Keep open. Current main does not contain this Windows exec/supervisor decoding fix: `src/process/exec.ts` still forces UTF-8 decoding for `runExec()` and still uses `chunk.toString()` in `runCommandWithTimeout()`, while supervisor adapters also stream raw `toString()` output. The only GBK/code-page-aware decoder on main lives in `src/node-host/invoke.ts` and is not wired into these paths. Docs still describe garbled Chinese exec output on Windows as an active workaround/reportable issue.

## Evidence

  - file: [src/process/exec.ts:136](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/process/exec.ts#L136)
