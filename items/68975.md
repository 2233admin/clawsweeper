## Summary

Keep PR #68975 open. Current main has not implemented the requested bge-m3 default: the active local memory embedding default is still embeddinggemma, exported through the memory host SDK and used by the memory-core local adapter. The PR is now stale against moved code paths and would need rebasing to the current memory-host SDK surface plus docs/test updates, but the requested behavior is not obsolete or superseded.

## Best Possible Solution

Keep the PR open but require a rebase/rework onto the current memory-host SDK architecture: change `src/memory-host-sdk/host/embedding-defaults.ts`, update the source and package/barrel tests to assert the exact bge-m3 model id, verify memory-core adapter behavior through the exported SDK path, and update `docs/reference/memory-config.md` plus `docs/concepts/memory-builtin.md` so the documented default and size match runtime behavior. Maintainer review should explicitly accept the larger default model trade-off before landing.
