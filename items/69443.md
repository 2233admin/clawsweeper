## Summary

Keep #69443 open. Current main has improved the supported subagent auto-announce path, but the exact reported RPC shape still appears validly problematic: `agent` RPC keeps `to` separate from `sessionKey`, and when `sessionKey` is omitted a direct-looking `to=agent:main:openclaw-weixin:direct:...` can still collapse through default DM session resolution to `agent:main:main`.

## Best Possible Solution

Keep the shipped auto-announce/sessionKey delivery improvements, then make the raw Gateway `agent` RPC defensive: when `sessionKey` is absent and `to` parses or classifies as an `agent:*` session key, either map it to `sessionKey` for compatibility or reject it with an actionable error telling callers to use `sessionKey`. Add a regression using a WeChat-like key such as `agent:main:openclaw-weixin:direct:<id>@im.wechat` under default DM scope to prove it no longer lands in `agent:main:main`.
