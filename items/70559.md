## Summary

Keep open: current `main` still contains the reported `runUnsafeReindex()` path that drops `chunks_vec` and then prepares SQL against `chunks_vec`, and the latest release `v2026.4.23` still carries the same behavior.

## Evidence

- **Unsafe full reindex still drops the vector table first:** `runUnsafeReindex()` immediately calls `resetIndex()`, and `resetIndex()` still calls `dropVectorTable()` and clears `vector.dims`, removing `chunks_vec` before the reindex sync begins.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:1243](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/memory-core/src/memory/manager-sync-ops.ts#L1243)
  - command: `nl -ba extensions/memory-core/src/memory/manager-sync-ops.ts | sed -n '1228,1310p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Current sync path still prepares DELETE statements against `chunks_vec`:** `syncMemoryFiles()` still eagerly prepares `DELETE FROM chunks_vec ...` whenever `vector.enabled && vector.available`, so after `resetIndex()` drops the table this prepare step can still fail before any lazy recreation occurs.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:674](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/memory-core/src/memory/manager-sync-ops.ts#L674)
  - command: `nl -ba extensions/memory-core/src/memory/manager-sync-ops.ts | sed -n '650,810p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Vector availability is not cleared when the table is dropped:** The only place in this area that sets `this.vector.available = false` is the sqlite-vec load-failure path. `ensureVectorTable()` recreates the table later once dimensions are known, but `dropVectorTable()` itself does not mark vectors unavailable.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:248](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/memory-core/src/memory/manager-sync-ops.ts#L248)
