## Summary

Current main still does not cover the reported WhatsApp leak of `[TOOL_CALL]{tool => ..., args => ...}[/TOOL_CALL]` as assistant-visible text. The shared visible-text sanitizer strips XML-style tool tags and `[Tool Call: ...]` downgraded markers, but not the all-caps bracket form, and WhatsApp final/block delivery still forwards normalized payload text to `msg.reply`.

## Best Possible Solution

Keep this issue open. Add a shared assistant-visible delivery/runtime fix for `[TOOL_CALL]{tool => ..., args => ...}[/TOOL_CALL]` before channel delivery; if it can be parsed safely, convert it into a structured tool call, otherwise suppress it from user-visible replies. Add regression coverage in `src/shared/text/assistant-visible-text.test.ts` plus a WhatsApp auto-reply dispatch or delivery test.

## Evidence
