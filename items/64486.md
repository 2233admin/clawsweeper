## Summary

Current main still implements a post-generation transparency note, not the requested pre-send forced choice. The existing source, tests, and hook surface support keeping this open rather than closing as implemented or ClawHub-only.

## Best Possible Solution

Keep this issue open. The best implementation is a focused gate in the agent finalization or reply pipeline that reuses the existing reminder-commitment detector, `successfulCronAdds`, and session-cron checks, then requests one revision before dispatch: either remove unsupported commitment language or create a concrete scheduled action. If maintainers want this as plugin-first policy, first expose documented per-turn scheduling/task/heartbeat evidence through a hook that runs on the affected reply path.
