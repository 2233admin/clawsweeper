## Summary

Keep this issue open. Current main still intentionally skips `MEDIA:` extraction inside fenced code blocks, tests assert the skipped-token behavior, and the related implementation PR #41980 is closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open. The best implementation path is a focused change in `src/media/parse.ts`: either extract valid own-line `MEDIA:` directives from fenced blocks or deliberately keep skipping them while surfacing a warning and updating assistant/docs guidance. Add targeted tests in `src/media/parse.test.ts` for backtick fences, tilde fences, invalid/prose examples, and valid own-line directives, while preserving existing media validation.
