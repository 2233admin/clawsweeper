## Summary

Keep open. Current `main` still treats fenced `MEDIA:` lines as non-delivery text: `splitMediaFromOutput()` explicitly skips extraction inside fenced code blocks, the parser test codifies that behavior, and reply parsing still flows through that function. I did not find a compensating warning or docs note covering fenced/backticked `MEDIA:` tokens.

## Evidence

- **Parser skips fenced MEDIA lines:** `splitMediaFromOutput()` parses fence spans, then for any line inside a fenced block it pushes the original line back into visible text and continues without MEDIA extraction.
  - file: [src/media/parse.ts:428](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/media/parse.ts#L428)
  - command: `nl -ba src/media/parse.ts | sed -n '392,690p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Test expects the reported behavior:** The parser test named `returns ordered text and media segments while ignoring fenced MEDIA lines` asserts that a fenced `MEDIA:https://example.com/ignored.png` stays in the text segment instead of becoming media.
  - file: [src/media/parse.test.ts:95](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/media/parse.test.ts#L95)
  - command: `nl -ba src/media/parse.test.ts | sed -n '1,260p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Outbound reply parsing still depends on splitMediaFromOutput:** `parseReplyDirectives()` calls `splitMediaFromOutput(raw)` and returns its `mediaUrls`, so the fenced-code behavior directly affects reply/media delivery on current main.
  - file: [src/auto-reply/reply/reply-directives.ts:16](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/auto-reply/reply/reply-directives.ts#L16)
  - command: `nl -ba src/auto-reply/reply/reply-directives.ts | sed -n '1,80p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Current docs do not warn about fenced/backticked MEDIA tokens:** User-facing docs say `MEDIA:<path-or-url>` must be on its own line, but I did not find a note that code fences/backticks disable extraction or trigger a warning.
  - file: [docs/start/openclaw.md:187](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/start/openclaw.md#L187)
