## Summary

Current `main` does not show a restart-safe fix for #55489. The in-memory skills snapshot version still resets to `0` on gateway restart, persisted snapshots can also remain at `version: 0`, and the main session refresh path only rebuilds when the stored version is behind the current in-memory version or the skill filter changes. That leaves the reported stale-snapshot-after-restart case still plausible on current `main`.
