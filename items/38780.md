## Summary

Keep PR #38780 open. Current main still lacks the PR's self-elected continuation feature: no `continue_work`, `continue_delegate`, or `request_compaction` tools; no `CONTINUE_WORK`/`CONTINUE_DELEGATE` token parser; no `agents.defaults.continuation` config contract; and no gateway continuation trigger/drain fields. Main has narrower restart-continuation, context-injection, and provider/incomplete-turn continuation plumbing, but those do not implement the proposed agent-controlled continuation/delegation runtime. The PR discussion is active and includes unresolved behavior-review surfaces, so this needs maintainer product/code review rather than cleanup closure.

## Best Possible Solution

Keep PR #38780 open for explicit maintainer review. If maintainers want this in core, the best path is a smaller, well-tested continuation runtime that reuses the merged restart/session-delivery, heartbeat, subagent, and TaskFlow substrate, remains disabled by default, updates config/protocol/docs/client surfaces coherently, and resolves the existing review comments before merge. If maintainers decide the product direction is wrong, close it deliberately as a maintainer decision rather than via conservative cleanup.
