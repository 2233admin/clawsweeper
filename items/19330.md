## Summary

Keep this issue open. Current main still uses a global `media/inbound` write contract for inbound media, bundled Discord/Telegram/Signal saves still pass the literal `"inbound"` subdir, and the managed-media read side still rejects nested inbound references such as `media://inbound/<account>/<file>`. The related implementation PR #46162 is closed and unmerged, while later duplicate reports point back here as the canonical tracker.

## Best Possible Solution

Keep this issue open as core media-pipeline work. A maintainer-quality fix should define a backwards-compatible scoped inbound media contract, thread channel account or resolved agent context through bundled and plugin media saves, support scoped `media://inbound` and absolute-path resolution, update sandbox staging and cleanup, preserve access to existing flat `media/inbound` files, and add regression coverage plus docs. PR #46162 is useful prior art but should be reviewed or replaced because it is closed and unmerged.
