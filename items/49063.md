## Summary

Keep open. Current main does not implement this PR’s channel_post native-command path: Telegram native/plugin slash-command handlers still require `ctx.message`, while channel posts are only normalized through the regular inbound message pipeline.

## Evidence

- **Native command handlers still drop non-`message` contexts:** Both native and plugin slash-command registrations read `const msg = ctx.message;` and immediately return when it is absent, so a `channel_post` context is not handled here on current main.
  - file: [extensions/telegram/src/bot-native-commands.ts:726](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/telegram/src/bot-native-commands.ts#L726)

- **Auth path is still typed and routed around regular messages/groups:** `resolveTelegramCommandAuth` still accepts `NonNullable<TelegramNativeCommandContext["message"]>` and computes `isGroup` only from `group`/`supergroup`, with no channel-post-specific authorization branch on main.
