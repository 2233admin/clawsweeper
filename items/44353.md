## Summary

Keep open. Current main has improved generic model fallback traversal for unrecognized provider errors, so a Bedrock provider-level error can advance when fallback candidates are actually present. However, the issue's explicit config uses `agents.defaults.subagents.model.fallbacks`, and current subagent model selection/run wiring still appears to persist only the selected subagent primary model while sourcing fallback candidates from per-agent/default main `model` settings, not from `defaults.subagents.model.fallbacks`. The existing fix evidence therefore does not fully cover this issue's reported subagent configuration path.
