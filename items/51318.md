## Summary

Keep open. Current `main` does not implement a per-invocation `openclaw agents add --skip-bootstrap`; it only honors the global `agents.defaults.skipBootstrap` setting. The PR also appears to introduce a real CLI regression called out in review: adding `skipBootstrap` to the explicit-option watchlist would make `openclaw agents add <name> --skip-bootstrap` count as non-interactive, which then requires `--workspace` and blocks the interactive wizard path.

## Evidence

- **Current main `agents add` docs do not expose `--skip-bootstrap`:** `docs/cli/agents.md` lists `--workspace`, `--model`, `--agent-dir`, `--bind`, `--non-interactive`, and `--json` for `agents add`, and notes that explicit add flags switch the command into the non-interactive path. There is no per-invocation `--skip-bootstrap` option on current `main`.
