## Summary

Keep open. Current `main` only persists fragments of working-location state (`cwd` in transcript headers, ACP `cwd`, subagent `spawnedWorkspaceDir`) and `/status` still reports runtime/context toggles rather than a per-session working-context capsule. The linked follow-up PR #71135 also indicates maintainers are treating this as unfinished work, not already-resolved cleanup.

## Evidence

- **Compaction docs still describe semantic summary, not working-state restoration:** `docs/concepts/compaction.md` explains compaction as summarizing older turns, saving the summary in the transcript, and keeping recent messages intact; the only `/status` signal called out is `🧹 Compactions: <count>`, and `/new` is described as starting a fresh session without compacting. There is no documented persisted working-context capsule for repo/cwd/branch recovery.
