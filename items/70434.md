## Summary

Keep PR #70434 open. Current main still does not implement the PR's central bounded retry/readiness behavior for subagent spawn/finalize/sweep gateway lifecycle gaps, and the PR discussion identifies concrete issues that should be fixed rather than used as a close reason. The work is plausible but needs rework before maintainer cleanup can close or merge it.

## Best Possible Solution

Keep the PR open and ask for a focused rework: move local validation before any gateway readiness handling, replace the mutating global sessions.patch probe with a non-mutating health/readiness path or retry around the actual lifecycle call, centralize the lifecycle retry classifier, and add targeted tests for the five transient strings across spawn, announce finalize, and sweep cleanup. Related timeout/config work such as #65415 is adjacent, not a canonical replacement for this PR.
