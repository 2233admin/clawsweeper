## Summary

Current `main` does not include this PR's behavior change. The Telegram menu sync path still unconditionally calls `deleteMyCommands()` before any non-empty `setMyCommands()` update, and the regression test on `main` still asserts the old `delete -> set` ordering. This PR remains an unresolved behavior change, not an already-implemented cleanup.
