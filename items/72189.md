## Summary

Keep #72189 open. Current main still lacks the Google Live voice-call tool-response changes the PR targets: Google tool responses omit the function name, the consult tool is not declared NON_BLOCKING, the realtime voice bridge cannot pass continuation options, and the voice-call handler only submits one final result after awaiting the long consult. The PR is recent and focused, and the author reported live success in discussion, but the submitted diff still needs the Greptile P1/P2 scoping fix before it is ready to land.

## Best Possible Solution

Keep the PR open and request a narrow revision: include the Google Live function name in tool responses, preserve call-id-to-name mapping, mark only openclaw_agent_consult NON_BLOCKING, add and forward continuation options through the realtime voice bridge/session contract, send the provisional working response only for that consult tool, schedule the final consult result appropriately, preserve single-result behavior for OpenAI and other providers/tools, and keep the regression tests/docs. After the Greptile P1/P2 is fixed, maintainers should require targeted tests plus changed-gate or equivalent scoped validation, and preferably attach redacted live Gemini evidence if available.
