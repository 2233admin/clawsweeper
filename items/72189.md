## Summary

Keep #72189 open. Current `main` still lacks the Google Live tool-response behavior this PR targets: Google function responses still omit the required function name, consult declarations are not marked `NON_BLOCKING`, the realtime voice bridge contract does not carry continuation options, and the voice-call handler submits only one final result after awaiting the handler. The PR is still potentially useful, especially with the author’s live-testing follow-up, but it should be revised to address the Greptile review that scopes continuation responses to `openclaw_agent_consult` only.

## Best Possible Solution

Keep the PR open and request a narrow revision: include Google Live function names in tool responses, preserve call-id-to-name mapping, mark only `openclaw_agent_consult` as `NON_BLOCKING`, add and forward continuation options through the realtime voice bridge/session SDK, send the provisional working response only for that consult tool, schedule its final result appropriately, preserve the existing single-result behavior for OpenAI and other providers/tools, and keep the focused regression tests/docs. After the Greptile P1/P2 is fixed, maintainers should ask for targeted tests plus changed-gate or scoped validation, with redacted live Gemini evidence if available.
