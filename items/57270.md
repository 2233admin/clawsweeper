## Summary

Keep open. Current main still appears to reject `tools.exec.approvalRunningNoticeMs` at the global config path: the field is present only on `AgentToolExecSchema`, while global `tools.exec` uses strict `ToolExecSchema` built from `ToolExecBaseShape`, which does not include the field. The docs, generated schema metadata, labels/help, and TypeScript type advertise the key, so this remains a real docs/schema/runtime mismatch. The linked PR #57293 describes the right root cause and fix, but it is still open/unmerged rather than shipped on main.

## Best Possible Solution

Keep #57270 open until the schema fix lands. The concrete fix should move `approvalRunningNoticeMs: z.number().int().nonnegative().optional()` into `ToolExecBaseShape` so both `ToolExecSchema` and `AgentToolExecSchema` accept it, add a regression test for `ToolsSchema` or `OpenClawSchema` accepting `{ tools: { exec: { approvalRunningNoticeMs: 3000 } } }`, and keep the generated schema/help/docs in sync. PR #57293 appears to be the relevant implementation candidate to revive, review, or supersede.
