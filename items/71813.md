## Summary

Current main already keeps Telegram reply typing alive through the shared typing lifecycle. Telegram wires sendTyping to sendChatAction("typing"), the shared callback sends an immediate cue and 3-second keepalive pings until idle/cleanup, and regression tests cover repeated keepalive behavior. The same implementation is present in the latest tagged release v2026.4.24.

## Best Possible Solution

Close this issue as implemented, keep the shipped shared typing lifecycle, and leave broader active-run presence lease or transient sendChatAction cooldown work to the existing narrower follow-ups.
