## Summary

Current `main` does not implement WhatsApp sender notification after pairing approval. Core CLI already has a generic `openclaw pairing approve ... --notify` path, but the WhatsApp channel plugin only declares `pairing.idLabel` and does not provide `pairing.notifyApproval`, so the notify step returns without sending anything.
