## Summary

Keep open. Current `main` has partial ciao-assertion suppression logic, but the bundled Bonjour plugin does not wire that handler into production startup, so the reported `@homebridge/ciao` unhandled assertion path still appears able to reach OpenClaw's fatal global `unhandledRejection` exit path.

## Evidence

- **Bundled Bonjour plugin does not register the ciao rejection handler:** The production plugin entry calls `startGatewayBonjourAdvertiser(...)` with only `{ logger: api.logger }`; it does not pass or import `registerUnhandledRejectionHandler`.
  - file: [extensions/bonjour/index.ts:23](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/bonjour/index.ts#L23)
  - command: `nl -ba extensions/bonjour/index.ts | sed -n '1,80p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Advertiser suppression is conditional on an injected handler:** `handleCiaoUnhandledRejection` classifies the reported `Reached illegal state! IPV4 address change from defined to undefined!` assertion and would suppress it, but only if `deps.registerUnhandledRejectionHandler` is provided; otherwise no handler is registered.
  - file: [extensions/bonjour/src/advertiser.ts:178](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/bonjour/src/advertiser.ts#L178)
  - command: `nl -ba extensions/bonjour/src/advertiser.ts | sed -n '170,270p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Tests confirm the default path does not install rejection handling:** One test explicitly asserts the default advertiser path does not install a process-level `unhandledRejection` handler, while separate tests only exercise the ciao-classification behavior when a mock `registerUnhandledRejectionHandler` is injected.
  - file: [extensions/bonjour/src/advertiser.test.ts:231](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/bonjour/src/advertiser.test.ts#L231)
  - command: `nl -ba extensions/bonjour/src/advertiser.test.ts | sed -n '228,304p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Unhandled rejections still terminate the process when unclaimed:** OpenClaw's global handler returns early only when `isUnhandledRejectionHandled(reason)` is true; otherwise ordinary unhandled rejections hit `process.exit(1)` via `exitWithTerminalRestore(...)`.
  - file: [src/infra/unhandled-rejections.ts:317](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/infra/unhandled-rejections.ts#L317)
  - command: `nl -ba src/infra/unhandled-rejections.ts | sed -n '317,360p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Public plugin runtime already exposes the needed API:** `registerUnhandledRejectionHandler` is already exported from the plugin runtime boundary, and other bundled plugins such as Telegram use it directly for production suppression/recovery wiring.
  - file: [src/plugin-sdk/runtime-env.ts:31](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/plugin-sdk/runtime-env.ts#L31)
