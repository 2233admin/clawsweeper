## Summary

Current `main` does not implement the PR's usage-summary cache or the new `usage.cacheTtlMs` config, and the Anthropic usage path still performs a fresh `/api/oauth/usage` fetch each time. The PR discussion also shows real review work on cache correctness and transient-error handling, so this remains a plausible, useful bugfix rather than something safe to auto-close.
