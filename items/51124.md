## Summary

Keep open. Current `main` does not implement the Telegram-specific session-label cleanup described in PR #51124, and the proposed patch has unresolved correctness issues in the recorded review discussion, so this is still actionable maintainer work rather than a conservative auto-close.

## Evidence

There is no Telegram-specific cleanup for `telegram:` prefixes, `id:<digits>` suffixes, DM/slash naming, or `main` alias handling in the current implementation.
