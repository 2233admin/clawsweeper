## Summary

Keep open. Current `main` does not already implement this Telegram session-label cleanup: the equivalent UI code still passes `label`/`displayName` through unchanged, does not consult `origin.label`, and the UI session-row type still omits `origin`. The PR's requested behavior is therefore not implemented on `main`, even though it likely needs rebasing to the newer `ui/src/ui/chat/session-controls.ts` split.
