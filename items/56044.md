## Summary

Keep open. Current `main` has added a fast `/stop` abort path, but the underlying steer/interrupt behavior reported in #56044 is still not fully fixed: the embedded-run queue path still rejects steering unless the run is actively streaming, and the docs still describe `steer` as falling back to followup when not streaming while `collect` remains the default followup/coalescing mode.

## Evidence
