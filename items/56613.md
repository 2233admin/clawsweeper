## Summary

Keep open. Current main implements the per-agent TTS half through `agents.list[].tts`, but the Android Talk/Voice tab still lacks agent/session selection and still routes Talk Mode sends through the resolved main session key. The recent maintainer comment also explicitly keeps this open for the remaining mobile/macOS Talk/Voice session-selection work.

## Best Possible Solution

Keep this issue open for the remaining Talk/Voice session-selection work. The likely fix is to add an Android Voice/Talk selected-agent or session picker, wire that selected key into `TalkModeManager` and manual voice sends instead of only the resolved main/default key, add focused Android routing/UI tests, and preserve the shipped `agents.list[].tts` per-agent voice configuration. Apple surfaces should be checked for parity, but Android alone still leaves the core request actionable.
