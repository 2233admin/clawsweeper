## Summary

Keep open. Current main still lacks the requested Android Voice/Talk session switching, and Talk voice configuration is still defined as a global `talk` section rather than a per-agent setting.

## Evidence

- **Android Voice tab has no session/agent picker:** `VoiceTabScreen` only binds mic/speaker/conversation state and renders the voice UI; there is no session selector analogous to chat.
