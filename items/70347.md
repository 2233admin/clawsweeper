## Summary

Current `main` does not implement the requested fix. The cron outer-timeout path still aborts and rejects without emitting any terminal lifecycle event, while gateway session finalization still only happens when a lifecycle event is observed. Recent post-release changes in this area do not touch the timeout or gateway lifecycle files.

## Evidence

- **Outer cron timeout still only aborts and rejects:** `executeJobCoreWithTimeout(...)` races the cron run against a timer that calls `runAbortController.abort(timeoutErrorMessage())` and immediately `reject(new Error(timeoutErrorMessage()))`. There is no lifecycle emission or session-store update in this timeout callback.
  - file: [src/cron/service/timer.ts:79](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/cron/service/timer.ts#L79)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Gateway session finalization still depends on lifecycle events:** `server-chat` persists session state only when it handles lifecycle `start`/`end`/`error` agent events, by calling `persistGatewaySessionLifecycleEvent(...)`. Without a lifecycle event, this path does not run.
  - file: [src/gateway/server-chat.ts:650](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/gateway/server-chat.ts#L650)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Lifecycle persistence logic exists, but only behind the lifecycle-event entrypoint:** `persistGatewaySessionLifecycleEvent(...)` updates the stored session row only after resolving a valid lifecycle phase from the incoming event. It does not provide an alternate cron-timeout write path.
  - file: [src/gateway/session-lifecycle-state.ts:146](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/gateway/session-lifecycle-state.ts#L146)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Isolated cron session persistence does not synthesize terminal lifecycle state:** Cron pre-run/session persistence writes the agent session entry and run-session snapshot to the session store, but this helper only clones and writes the current entry. It does not set terminal status/timestamps or emit a lifecycle event on timeout.
  - file: [src/cron/isolated-agent/run-session-state.ts:26](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/cron/isolated-agent/run-session-state.ts#L26)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Post-release diffs do not show the requested fix landing on main:** Comparing `v2026.4.23` to current `main` shows changes only in `src/cron/isolated-agent/run-executor.ts`, `src/cron/isolated-agent/run-session-state.ts`, and `src/cron/isolated-agent/run.ts`; the timeout path in `src/cron/service/timer.ts` and gateway lifecycle persistence files were not changed.
