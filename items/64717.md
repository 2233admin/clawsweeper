## Summary

Keep this PR open. Current main still hardcodes Telegram send-time HTML/plain-text chunk planning to 4000 characters, while the PR is a narrow Telegram-owned fix for an already documented `channels.telegram.textChunkLimit` configuration surface.

## Best Possible Solution

Review and land this PR, or replace it with an equivalent Telegram-owned patch. The implementation should resolve `channels.telegram.textChunkLimit` through the existing plugin SDK resolver, clamp the effective Telegram send limit to 4096, use that value consistently throughout `buildChunkedTextPlan`, and keep focused regression coverage for both the 4096 cap and a lower configured limit.
