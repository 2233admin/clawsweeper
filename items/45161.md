## Summary

Keep open. Current `main` still does not retry transient Discord outbound network failures on the active send path, and the repository’s current tests explicitly expect a network error to fail without retry. The PR’s original file paths are stale after the Discord plugin move into `extensions/discord`, but the underlying bug report remains credible on current `main`.

## Evidence

  - file: [extensions/discord/src/send.creates-thread.test.ts:550](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/extensions/discord/src/send.creates-thread.test.ts#L550)
