## Summary

Keep open. Current main still drops manifest-declared plugin skills when materializing PluginRecord instances, so #58805 remains a valid focused bug-fix PR rather than an implemented or superseded item.

## Best Possible Solution

Keep #58805 open. The best path is to rebase and land the narrow contract fix: add `skills: string[]` to `PluginRecord`, thread `manifestRecord.skills` through every manifest-backed `createPluginRecord` path in `loadOpenClawPlugins()` and `loadOpenClawPluginCliRegistry()`, initialize `[]` for bundled capability and test-helper records, and keep the focused loader regression test.
