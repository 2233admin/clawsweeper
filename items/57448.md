## Summary

Keep #57448 open. Current main partially addresses the DM binding prerequisite: Telegram DMs can be routed through a current-conversation ACP binding, and there is test coverage for DM bind creation/routing. The requested reversible handoff behavior is still missing: existing bindings are overwritten/metadata-merged rather than saved as a previous binding, unbind/reset paths delete the ACP binding rather than restoring an earlier parent binding, and the docs explicitly say `/new` and `/reset` reset the bound ACP session in place instead of reverting to the parent agent.

## Best Possible Solution

Keep the issue open and narrow it to the missing reversible handoff semantics. The likely implementation path is to add a validated `previousBinding`/handoff metadata contract in the session binding layer, preserve the previous DM route when an ACP `--bind here` takeover replaces it, restore it on ACP close/reset/session end/expiry/startup stale cleanup, and add focused tests in `src/agents/acp-spawn.test.ts`, `extensions/telegram/src/thread-bindings.test.ts`, and `src/infra/outbound/current-conversation-bindings.test.ts`. Docs should then state whether `/new` reverts or resets in place.
