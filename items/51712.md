## Summary

Keep open. Current `main` still hardcodes `memory-lancedb` to a single `memories` table, the plugin schema does not accept any `tableName`/collection option, and the documented/runtime plugin config surface is global `plugins.entries.<id>.config` rather than per-agent plugin config. The requested per-agent collection isolation is not implemented on `main`.

## Evidence

- **LanceDB table is still hardcoded:** `extensions/memory-lancedb/index.ts` defines `const TABLE_NAME = "memories"` and uses that same constant when opening or creating the table, so one configured DB path maps to one shared LanceDB table.
  - file: [extensions/memory-lancedb/index.ts:54](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/extensions/memory-lancedb/index.ts#L54)
