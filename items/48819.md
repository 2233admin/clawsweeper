## Summary

Keep open. Current `main` does not contain the PR’s install-record-based config/global duplicate suppression, and the proposed PR still has unresolved correctness/test gaps, so there is not strong evidence to close this item as already implemented or obsolete.

## Evidence

- **Current main only de-dupes identical or realpath-equal plugin roots:** In `loadPluginManifestRegistry`, the `samePlugin` check returns true only for identical `rootDir` strings or equal `safeRealpathSync(rootDir)` results. There is no install-record fallback for config/global pairs on current `main`.
  - file: [src/plugins/manifest-registry.ts:625](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/plugins/manifest-registry.ts#L625)
  - command: `nl -ba src/plugins/manifest-registry.ts | sed -n '625,647p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Installed-plugin matching only affects global precedence, not warning suppression:** `matchesInstalledPluginRecord` is only used by `resolveDuplicatePrecedenceRank` for `origin === "global"`. Current `main` can rank an installed global ahead of bundled, but it does not suppress a config/global duplicate warning based on `plugins.installs`.
  - file: [src/plugins/manifest-registry.ts:453](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/plugins/manifest-registry.ts#L453)

- **Existing tests do not cover the PR’s claimed config/global suppression path:** The registry tests cover explicit installed globals overriding bundled duplicates and same-directory dedupe, but there is no test for a config/global pair tied together by `plugins.installs`.
  - file: [src/plugins/manifest-registry.test.ts:371](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/plugins/manifest-registry.test.ts#L371)
