## Summary

Current `main` still contains the reported raw-mode failure path: safe-bin normalization leaves own properties with `undefined`, and raw round-trip validation still compares the restored raw parse against the materialized runtime config rather than the source config. The linked fix PR #59336 is still open/unmerged in the provided GitHub context, so #59330 remains a useful canonical bug report rather than cleanup-close material.

## Best Possible Solution

Keep #59330 open as the canonical bug until a fix lands. The best path is to update or replace #59336 with a source-backed patch that deletes empty safe-bin keys, validates raw redaction against the authored/source config rather than materialized runtime config, and adds focused regression tests for the safe-bin undefined-key case, runtime materialization differences, and the later comment's raw text replacement/corruption scenario involving `source: "file"`.
