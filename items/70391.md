## Summary

Keep PR #70391 open. Current main still performs the redundant `chat.history` reload after a final chat event even when the final event already supplied and locally appended an assistant message. The PR is a focused, still-relevant UI fix, though its review comments identify correctness issues that should be addressed before merge.

## Best Possible Solution

Keep the PR open, revise the implementation so final assistant-payload reload suppression is narrowly gated and does not drop legitimate deferred session updates, then land it with the existing targeted UI gateway and reload tests updated to cover those edge cases.
