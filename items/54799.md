## Summary

Keep this issue open. Current main still does not implement declarative SKILL.md schedule/event trigger metadata, automatic reconciliation into cron jobs, skill trigger history, or an `openclaw skills audit --never-triggered` surface. Existing heartbeat `tasks:` blocks and plugin cron hooks are adjacent, but they do not satisfy the requested SKILL.md metadata/status contract.

## Best Possible Solution

Keep this open for maintainer product/API design. A strong implementation would add a versioned `metadata.openclaw.triggers` contract for SKILL.md, validate schedule/event trigger forms, reconcile schedule triggers into managed cron jobs, define approved lifecycle events for event-triggered skills, persist last-triggered history, and expose the data through `openclaw skills list`, `openclaw skills info`, and a new audit surface. If maintainers prefer ClawHub/plugin ownership, first add the missing public skill metadata/status and cron reconciliation APIs so an external plugin can implement it without private imports.
