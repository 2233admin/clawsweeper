## Summary

Keep open. Current `main` still has the Telegram `/model` and `/models` path calling `loadModelCatalog({ config: cfg })` without `useCache: false`, while the catalog loader keeps and reuses a long-lived in-process promise by default. Other interactive model-picker flows already bypass that cache, so this PR still addresses a real inconsistency on `main` rather than something already implemented.
