## Summary

Keep open. Current main still matches the reported root cause: the bundled OpenProse grammar and compiler docs constrain `model:` to `sonnet`, `opus`, or `haiku`, so fully qualified values like `openai/gpt-5.2` are not supported by the OpenProse surface. Core subagent spawning can already carry arbitrary model strings, so this looks fixable in the bundled OpenProse skill pack, but it is not implemented on main and no close reason applies.
