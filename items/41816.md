## Summary

Current `main` already implements the requested behavior through a broader config-mutation path: plugin install/update writes now go through `replaceConfigFile`, which writes isolated single-section mutations directly into single-file `$include` targets instead of flattening `openclaw.json`. The checked-in regression test, docs, and changelog all confirm the fix, so this PR is obsolete on `main`.
