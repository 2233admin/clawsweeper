## Summary

Current `main` already implements the requested behavior for `openclaw plugins install` include-backed config writes through the newer `replaceConfigFile()` mutation path, with docs and regression tests, so PR #41816 is redundant.

## Evidence

- **Plugin install persistence uses include-aware replace path:** `persistPluginInstall()` and `persistHookPackInstall()` now persist through `replaceConfigFile(...)` instead of a direct `writeConfigFile(...)` call, so install writes flow through the centralized config mutation logic.
  - file: [src/cli/plugins-install-persist.ts:29](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/cli/plugins-install-persist.ts#L29)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Current writer preserves single-file top-level includes:** `replaceConfigFile()` detects when exactly one top-level section changed, resolves a single-file `$include` target for that section, writes the included file directly, and only falls back to the root writer when the write is not isolated.
  - file: [src/config/mutate.ts:56](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/config/mutate.ts#L56)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Regression tests cover write-through and fail-closed fallback:** Tests verify that a plugins include write updates the included file while leaving the root `$include` intact, and that non-isolated writes fall back to the root writer instead of flattening includes.
  - file: [src/config/mutate.test.ts:140](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/config/mutate.test.ts#L140)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **User-facing CLI docs describe the shipped behavior:** The plugins CLI docs say that when `plugins` is backed by a single-file `$include`, `plugins install/update/enable/disable/uninstall` write through to the included file and leave `openclaw.json` untouched; unsupported include shapes fail closed.
  - file: [docs/cli/plugins.md:71](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/docs/cli/plugins.md#L71)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Reference docs define the same config contract:** The configuration reference documents the same OpenClaw-owned write behavior for single-file top-level includes and the fail-closed behavior for root includes, include arrays, and sibling overrides.
  - file: [docs/gateway/configuration-reference.md:1193](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/docs/gateway/configuration-reference.md#L1193)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Shipped release already contains the implementation:** Inspected release tag `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`) and confirmed the same `replaceConfigFile()` include-write logic and plugins docs are present there, so the behavior is already shipped.
  - file: [src/config/mutate.ts:111](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/config/mutate.ts#L111)
