## Summary

Keep open. Current `main` still routes hook fallback system events off `!result.delivered` in `src/gateway/server/hooks.ts`, while the isolated cron delivery path explicitly treats silent `NO_REPLY` outcomes as `deliveryAttempted: true` and `delivered: false`. That mismatch matches the reported leak, and the same fallback logic is still present in the latest shipped release `v2026.4.23`.

## Evidence

- **Hook fallback on main still keys only on `delivered`:** `dispatchAgentHook()` enqueues `Hook ...` system events whenever `runCronIsolatedAgentTurn()` returns `!result.delivered`, with no guard for silent/handled outcomes via `deliveryAttempted`.
  - file: [src/gateway/server/hooks.ts:91](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/server/hooks.ts#L91)
