## Summary

Keep open. Current `main` still has the reported hook fallback behavior: `src/gateway/server/hooks.ts` enqueues a main-session `Hook ...` system event whenever `runCronIsolatedAgentTurn()` returns `delivered: false`, and the silent/no-change cron path still returns `delivered: false` while marking `deliveryAttempted: true`. That suppression signal is honored by cron heartbeat fallback logic, but it is ignored by the shared hook path, so the issue is not implemented on `main` and cannot be conservatively closed.
