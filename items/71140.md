## Summary

Keep #71140 open. Current main still has the Telegram reaction-directive gap: media/GIF delivery is already covered, but `[[react...]]` / `[[react_to_current:...]]` is not parsed into structured outbound data, Telegram reply delivery still requires text or media before inspecting Telegram channel data, and the Telegram outbound adapter only handles quote/buttons payload metadata. The provided maintainer comment explicitly says this remains valid, and the linked fix PR #71141 is open/draft rather than merged.

## Best Possible Solution

Keep #71140 open as the canonical bug until a fix lands. The best path is the maintainer-described one: extend the shared reply directive parser to emit structured reaction metadata, preserve it through outbound normalization, allow reaction-only Telegram replies when a reply target exists, execute the reaction in Telegram delivery/sendPayload via the existing native reaction primitive, and add parser plus Telegram delivery/adapter regression tests. PR #71141 may be the right vehicle if it is updated, reviewed, and merged.
