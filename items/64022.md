## Summary

Keep this PR open. Current main still routes Slack media through the dispatcher-backed runtime fetch path and still has a regression test asserting the dispatcher is forwarded, while the linked Slack attachment bug remains open. The PR is a focused implementation candidate, but it changes the SSRF pinned-dispatcher boundary and conflicts with the shipped/current contract, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, decide whether Slack media should strip the upstream SSRF dispatcher or preserve the pinned dispatcher path, update the regression test to lock that chosen contract, and then either land a source-backed fix for #63905 or close the PR with a documented maintainer decision.
