Keep this PR open, but require a focused revision: implement ACP backend failover in the core ACP session manager after resolving session metadata, preserve binding/agent/global backend precedence, close or retire old handles safely before switching, avoid retrying once streamed output or tool side effects may have occurred, use precise unavailable/rate/quota classification, record failures consistently, add `acp.fallbacks` schema/help/generated docs if that config is accepted, and add targeted ACP manager tests.
