## Summary

Keep #70202 open. Current main has most of the underlying model infrastructure: primary/fallback model config, `/model` switching, Control UI model override state, and a multi-select allowlist in `openclaw configure --section model`. The specific requested onboarding flow is still not fully implemented: `openclaw onboard` still prompts for one default model and applies only that selected model, while the multi-select allowlist/fallback path is wired into configure, not the onboarding wizard.

## Best Possible Solution

Keep this open and wire the existing `promptModelAllowlist` plus `applyModelFallbacksFromSelection`/`applyModelAllowlist` flow into `src/wizard/setup.ts` after provider auth/default-model selection. Reuse the provider `wizard.modelAllowlist` metadata already used by configure, add focused onboarding tests that prove multiple selected models are preserved in `agents.defaults.models` with one `agents.defaults.model.primary`, and update `docs/cli/onboard.md` if the intended product behavior differs.
