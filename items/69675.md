## Summary

Keep PR #69675 open. Current main does not contain the proposed ACP follow-up wording or the new guide, and the remaining question is a real product/runtime semantics decision: current docs and tests describe bound ACP conversations as routing follow-up text directly to the ACP session, while this PR proposes parent-assistant interpretation/rewrite semantics without implementing the runtime behavior change.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to decide the intended ACP bound-session semantics first: either keep current direct-to-ACP routing and reject/rework the wording, or implement parent-orchestrated follow-up rewriting in the ACP/session continuation path and then update docs, note text, and regression tests together.
