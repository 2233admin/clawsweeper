## Summary

Keep open. Current `main` appears to have fixed the issue’s secondary `payload.model` precedence complaint, but the primary custom-Ollama failure is still plausible in code: cron now reaches embedded provider registration, yet that registration resolves plugin ownership by `model.provider`, and the bundled Ollama plugin still registers only the canonical `ollama` id with no aliases/hook aliases for custom `models.providers.<id>` entries such as `ollama-mini` or `ollama-spark`.

## Evidence

- **Cron passes the resolved provider/model into the embedded runner:** The isolated cron executor forwards `providerOverride` and `modelOverride` directly into `runEmbeddedPiAgent(...)`, so custom provider ids still matter on the cron path.
  - file: [src/cron/isolated-agent/run-executor.ts:160](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/cron/isolated-agent/run-executor.ts#L160)

  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Embedded runner now does provider-stream registration before streaming:** `runEmbeddedPiAgent` attempts to register a provider stream for the resolved model on each attempt, so the original report’s 'cron skipped lazy registration entirely' root cause looks partially outdated.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:1362](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/pi-embedded-runner/run/attempt.ts#L1362)

- **Provider-stream registration resolves ownership by the model's provider id:** `registerProviderStreamForModel(...)` calls `resolveProviderStreamFn(...)` with `provider: params.model.provider`; if that provider id does not match a plugin id/alias, no stream is registered for the custom API.
  - file: [src/agents/provider-stream.ts:16](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/provider-stream.ts#L16)

  - file: [src/plugins/provider-hook-runtime.ts:17](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/plugins/provider-hook-runtime.ts#L17)
