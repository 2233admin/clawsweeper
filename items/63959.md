## Summary

Keep open. The PR is a focused Docker bug fix for linked issue #61279, and current main still does not include the image-level `/home/node/.openclaw` pre-create/ownership change. Existing Docker setup code repairs bind-mounted directories during the scripted setup flow, but that does not fully supersede a direct Docker Compose named volume mounted at `/home/node/.openclaw`.

## Best Possible Solution

Keep the PR open and either merge or adapt the Dockerfile image-level fix so fresh named volumes mounted at `/home/node/.openclaw` inherit `node:node` ownership. Before landing, verify with the full OpenClaw image and a direct Docker Compose named-volume smoke, and preserve or add docs for repairing already-created root-owned volumes.
