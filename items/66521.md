## Summary

Keep open. The PR's pre-dispatch `sendTypingSignal()` would bypass OpenClaw's current `sendPolicy: "deny"` outbound-suppression contract, and the proposed guard would fire for direct messages as well as groups. Current `main` intentionally suppresses typing indicators under deny, so this patch is not safe to auto-close as already-implemented or obsolete.

## Risks / Open Questions

- Merging the PR as written would emit a visible Signal typing indicator before centralized delivery-policy checks, violating the existing `sendPolicy: "deny"` contract.
- The implementation scope in the PR body/comments does not match the code path: `ctxPayload.To` covers DMs and groups, so the behavior change is wider than described.
- There is no corresponding Signal monitor coverage in the repo for an early-typing deny-policy path, so the regression would be easy to miss.
