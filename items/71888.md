## Summary

Keep open. Current main already has `openai-codex/gpt-5.5-pro` forward-compat runtime and catalog support, but the issue asks for more than the raw route: explicit ChatGPT Pro / Deep Research naming, docs that prevent route guessing, and a recommended read-only research-agent pattern. Those docs/UX gaps are still actionable in this repo.

## Best Possible Solution

Keep the issue open but narrow it: document the exact supported `openai-codex/gpt-5.5-pro` route if it is intended, clarify whether ChatGPT Pro / Deep Research is or is not exposed through Codex OAuth, fix the `models list` guidance so users know which command should show provider catalog rows, and add or link a recommended read-only research-agent config using existing sandbox/tool-policy and web-search/citation settings. Any standalone deep-research skill should remain a ClawHub/plugin item, but the provider/docs clarity belongs in OpenClaw core docs.
