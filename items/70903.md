## Summary

Keep open. Current main has partial recovery work, including immutable active cooldown windows and direct `models auth login` cleanup, but it still leaves the reported recovery path actionable: same-provider fallback chains still suppress early billing probes, and `openclaw configure --section model` still re-authenticates through the plugin auth-choice path without clearing stale usage state.

## Best Possible Solution

Keep this issue open and finish the shared provider-auth recovery work. Treat same-provider-only/shared-profile fallback chains as having no effective billing fallback for recovery probing, clear stale usage state after successful provider re-auth through `openclaw configure --section model` and other auth-choice flows, and add regression tests for Anthropic same-profile Opus fallback billing recovery plus GitHub Copilot configure re-auth clearing a `format` cooldown.
