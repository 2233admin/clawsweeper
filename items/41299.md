## Summary

Keep open. Current `main` still uses the old double-newline join between inbound metadata and user text, and current sanitizers do not implement the proposed `openclaw:user-msg` separator flow. The PR therefore is not implemented on `main`, and the discussion shows active iteration on review feedback rather than an obsolete or incoherent change.
