## Summary

Keep PR #71791 open. Current main still loses the resolved WhatsApp E.164 sender identity when buffering pending group history, then allowlist-filters that supplemental history using only `senderJid`; the PR is a focused plugin bug fix with a targeted regression test for the LID sender plus E.164 allowlist case.

## Best Possible Solution

Keep PR #71791 open for maintainer review and normal validation. If CI and the WhatsApp changed gates pass, the best path is to land the focused plugin fix: preserve optional `senderE164` in pending group history, include it in supplemental-context allowlist checks, and keep the new unit test as the regression guard.
