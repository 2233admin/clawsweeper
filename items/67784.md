## Summary

Keep open. Current `main` still has the Mattermost `message` tool `send` path dropping the reported fields: `actions.handleAction()` only reads `params.media`, never reads `threadId`, and does not forward `mediaLocalRoots`, even though the shared message tool exposes `path`/`filePath`/`threadId`, the dispatch context supplies `mediaLocalRoots`, and `sendMessageMattermost()` already supports local-media roots and threaded posting. The linked PR #67796 exists but is not merged into `main`.

## Evidence

- **Mattermost send action still ignores the reported params:** `mattermostPlugin.actions.handleAction()` takes only `{ action, params, cfg, accountId }`. In the `send` branch it derives `mediaUrl` solely from `params.media` and calls `sendMessageMattermost()` without `threadId`, `filePath`, `path`, or `mediaLocalRoots`.
