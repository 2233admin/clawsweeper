## Summary

Keep open. Current main has partial restart-loss mitigations, but it still does not persist or replay pending inbound/followup work across a gateway restart. The core report remains valid for queued followups, and drain-window arrivals are still rejected rather than durably recovered.

## Evidence

## Risks / Open Questions

- Part of the original report has been mitigated on main: some interrupted turns now surface an explicit restart notice instead of failing silently, so the issue title overstates the current user-visible behavior for every path.
- The remaining gap is still substantive: queued followups and drain-window inbound work are not durably replayed after restart, so closing as fixed would be premature.
