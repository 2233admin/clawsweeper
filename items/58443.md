## Summary

Keep #58443 open. Current main has partial duplicate-suppression work for Telegram polling, Slack event races, and generic inbound dispatch, but it does not yet cover the full reported bug class: the generic auto-reply error path still releases an inbound dedupe claim without committing it, and subagent announce idempotency is still scoped to childRunId, matching the reporter's later run-remap duplicate-delivery analysis.

## Best Possible Solution

Keep #58443 open as the umbrella for gateway duplicate inbound/announce replay. The best path is to preserve the narrower shipped Telegram polling fixes, keep #69303 for the specific error-path replay gap if maintainers want a focused tracker, and implement remaining core fixes: durable or terminal-state inbound idempotency for selected provider/runtime failure paths, plus stable logical-completion announce idempotency that does not change when orphan recovery/remap creates a new childRunId. After that, add regression coverage for Telegram same message_id replay, Slack duplicate ts injection, and subagent remap announce replay before closing the umbrella.
