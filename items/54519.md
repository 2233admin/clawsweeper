## Summary

Current main still does not implement the requested per-account read-only channel mode or a hard no-outbound channel/account gate. Shared channel config and WhatsApp account config/schema/docs lack `readOnly`, existing `sendPolicy` is session-scoped, and direct gateway sends, polls, WhatsApp sends, reactions, typing/presence, and read receipts still have outbound paths without an account-level lockout. The related #71382 PR remains useful but, per the provided GitHub context, is open/unmerged and WhatsApp-only, so it should not close this broader generic request.

## Best Possible Solution

Keep this issue open. The best product path is a generic channel/account outbound delivery policy exposed in shared config/schema/docs/UI metadata and enforced before every outbound surface: auto-replies, direct gateway sends, message actions, polls, reactions, typing/presence, read receipts, and plugin-owned transport side effects where feasible. Coordinate with #71382 for the WhatsApp transport-layer subset, but close this only after the generic contract or a maintainer-approved scoped replacement ships with tests and docs.
