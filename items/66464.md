## Summary

Current `main` still matches the report: the Dashboard agent model dropdown is populated from the full `models.list` catalog, and gateway-side filtering only narrows that catalog when `agents.defaults.models` is populated. A config that merely has a provider configured in `models.providers` and a primary model set can still surface unrelated providers in the dropdown, so this should stay open.

## Evidence

- **Dashboard dropdown still merges the entire catalog:** The agent overview panel renders its primary-model `<select>` with `buildModelOptions(configForm, ..., params.modelCatalog)`, so whatever arrives in `modelCatalog` is exposed in the Dashboard dropdown.

  - file: [ui/src/ui/views/agents-utils.ts:624](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/views/agents-utils.ts#L624)

  - file: [ui/src/ui/views/agents-utils.ts:578](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/views/agents-utils.ts#L578)

## Risks / Open Questions

- I did not run an interactive Dashboard reproduction because this review is read-only; the conclusion is from current main code and tests.
- There is a possible product-design question between 'filter to configured providers' and 'show all catalog entries but mark unavailable', but current main does neither in the Dashboard dropdown.
