## Summary

Keep #70643 open. Current main has temp-file-plus-rename writes, validation/rejection, and last-known-good recovery, but it still does not serialize the full cross-process read-modify-write cycle for openclaw.json with the existing file-lock helper. The linked fix PR #70655 specifically proposed adding that advisory lock and was closed unmerged, so the core requested prevention remains unshipped.

## Best Possible Solution

Keep the issue open and implement a small core fix around the central config mutation path: acquire the existing advisory file lock for the target openclaw.json before reading the write snapshot, preparing the merge, validating, backing up, and renaming. Add focused tests that simulate two concurrent config mutations and assert the final JSON remains valid and contains both independent updates, plus a re-entrant write case if owner-display-secret or plugin writeback can nest writes.
