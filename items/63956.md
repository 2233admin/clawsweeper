## Summary

Keep open. The current Control UI streaming pipeline still emits each committed stream segment and the live tail as separate `kind: "stream"` chat items, and each one renders through `renderStreamingGroup`, which creates a full `.chat-group.assistant`. The linked fix PR #63986 directly addressed this with a single in-flight assistant group, but that PR is closed and unmerged, so the bug is not implemented on current `main`.

## Best Possible Solution

Keep #63956 open. The best fix is the direction already described in PR #63986: introduce a single in-flight assistant stream-run/group that can render committed segment text, optional tool cards, the live tail, and the reading indicator within one outer `.chat-group.assistant`, with focused UI tests for tool calls visible and hidden.
