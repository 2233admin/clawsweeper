## Summary

Keep open. Current `main` does not implement this fix, and the PR still needs maintainer judgment because it targets only `cmd.exe`-wrapped batch execution while the reported Windows exec surface spans multiple shell paths and the proposed `chcp 65001` change could alter behavior for arbitrary plugin/user batch files.

## Risks / Open Questions

- Closing now would discard a still-plausible Windows non-ASCII output bug on `cmd.exe`-based paths without proof that current `main` fixed it.
- The PR may not solve the default Windows gateway exec path if that path goes through PowerShell rather than the batch-wrapper helpers changed here.
- Prepending `chcp 65001` before every wrapped batch launch could break locale-encoded `.cmd`/`.bat` scripts owned by plugins or users, so the change needs explicit maintainer review.
