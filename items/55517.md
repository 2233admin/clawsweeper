## Summary

Keep open. Current `main` still appears to have the gap the issue describes: plugin runtime `api.runtime.subagent.run()` forwards a Gateway `agent` call and returns `runId`, but I could not find any path from that runtime helper into `registerSubagentRun(...)`. The later `subagent_delivery_target` hook and final external completion delivery are still wired through the subagent announce flow that runs from the registry lifecycle for registered runs, so there is no strong evidence that #55517 is already implemented or obsolete on current `main`.
