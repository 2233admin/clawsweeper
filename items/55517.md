## Summary

Keep open. Current `main` includes a new fallback for external requester-route completion delivery, but the plugin runtime path in #55517 still forwards `api.runtime.subagent.run()` as a plain gateway `agent` call without registering the run in the subagent registry. Since `subagent_delivery_target` still only runs from the tracked subagent announce flow, there is not high-confidence evidence that a `before_dispatch`-started plugin subagent now triggers that hook or auto-delivers its final result.
