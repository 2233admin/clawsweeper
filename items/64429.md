## Summary

Keep this PR open. Current main still does not implement structured Slack file ID propagation: file IDs are visible in Slack placeholders and docs, but `SlackMediaResult`, `prepareSlackMessage()`, and the shared message context still lack `fileId` / `MediaFileId` / `MediaFileIds`. The prior ClawSweeper keep-open comment remains accurate against current main, and the related #45626 schema gap is adjacent rather than a shipped replacement.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, choose the intended agent-facing contract for inbound attachment file IDs, add targeted Slack media/prepare tests, and coordinate with #45626 or an equivalent message-tool schema fix so agents can both see and use Slack file IDs reliably.
