## Summary

Current `main` is still workspace-scoped, not project-scoped: OpenClaw injects a fixed workspace bootstrap set as `Project Context`, recommends one active workspace at a time, and exposes memory/context customization through plugin slots and context-engine plugins. I did not find any built-in support for explicit per-project declarations, project-switch confirmation, or multiple project memories within one session/workspace, so this remains a plausible open feature request rather than a cleanup close.
