## Summary

Keep open. Current main still lacks the requested schema-driven per-tool security classification contract. The prior Codex review comment remains accurate against edbab0e2: OpenClaw has adjacent safety mechanisms, including owner-only filtering, allow/deny policy, plugin before_tool_call approvals, ACP read-only heuristics, provider parallel_tool_calls transport settings, and mutation heuristics for replay/error handling, but none is the requested public metadata surface for read-only, destructive, concurrency-safe, interrupt, open-world, or classifier input behavior.

## Best Possible Solution

Keep this open. The best implementation is a backwards-compatible AgentToolSecurityProps-style metadata contract on AnyAgentTool and the public plugin SDK, preserved through registration, wrappers, adapters, inventories, and runtime definitions. Annotate core and bundled plugin tools incrementally, add conformance checks, and wire policy/UI/scheduling consumers behind conservative fallbacks while keeping the current heuristic safety paths as compatibility fallback.

## Risks / Open Questions

- This is a core/public SDK contract change, so defaults and optionality must preserve existing third-party plugin behavior.
- Wrappers, inventories, runtime adapters, hooks, trajectory/export surfaces, and provider bridges need explicit preserve-or-drop rules for any new metadata.
- Concurrency metadata should remain conservative and must not weaken sandbox, owner-only, allow/deny, approval, or provider safety controls.
- toAutoClassifierInput would need careful redaction and SecretRef/media handling so classifier prompts do not leak sensitive arguments.
