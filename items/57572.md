## Summary

Keep PR #57572 open. Current main still does not implement a configurable channel health restart mode or any graceful/reconnect branch: unhealthy channels still go through stop/reset/start, the gateway startup path only threads the existing interval/stale-threshold/restart-cap knobs, and config/docs/tests do not expose `channelHealthRestartMode` or per-channel `healthMonitor.restartMode`. Later main work (#69833) reduces false stale-socket restarts by requiring provider transport liveness, but it does not supersede this PR's remaining restart-mode proposal.

## Best Possible Solution

Keep the PR open as the canonical review thread for #57569, but ask for maintainer/product direction before merge. The best path is to revise it into a generic, documented channel health restart contract with schema/docs/tests and, if prompt-cache preservation is still the goal, a real provider/channel runtime reconnect seam rather than only preserving restart counters.
