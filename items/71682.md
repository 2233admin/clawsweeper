## Summary

Keep #71682 open. Current main partially addresses the timeout knob: `agents.defaults.llm.idleTimeoutSeconds` exists, is schema-backed, and has CLI write coverage. But the reported user problem is broader than that: the agent-facing gateway config tool still cannot edit this key, timeout profile cooldown/faster fallback remains a separate open issue (#51057), and the local Ollama slowdown/model-selection behavior has not been reproduced or ruled out on current main.

## Best Possible Solution

Keep open for maintainer triage. Document the immediate user workaround (`openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>` or `0` to disable), decide whether the owner-only gateway tool should allow bounded edits to `agents.defaults.llm.idleTimeoutSeconds`, and keep timeout-profile cooldown/faster fallback scoped to open #51057 unless #71682 produces a distinct reproduction. The local Ollama model-selection/performance symptoms should be reproduced or narrowed before closing this as implemented.
