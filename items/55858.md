## Summary

Current `main` partially addresses the request: it now supports per-channel queue debounce overrides, but per-channel queue `cap` and `drop` are still not implemented in the config schema or resolver. The issue remains a real core gap and should stay open.

## Evidence

- **Reporter narrowed the ask:** The issue comment already clarified that per-channel queue mode was supported, and the remaining gap was specifically per-channel `debounceMs`, `cap`, and `drop` overrides.
- **Queue resolver only applies per-channel debounce:** `resolveQueueSettings()` reads `queueCfg?.debounceMsByChannel` for debounce, but `capRaw` and `dropRaw` fall back only through inline/session/global values (`queueCfg?.cap`, `queueCfg?.drop`) with no per-channel lookup.
