## Summary

Keep open: current `main` still uses a Node-shaped `isDirectModuleNotFoundError` bootstrap check with no Bun `specifier` fallback, the same logic is present in the latest release snapshot, and Bun is still documented as an optional local runtime from source.

## Evidence

- **Current bootstrap logic still lacks Bun-specific handling:** At `main` SHA `5fe333ada85afd0def9e85985ce57e36aa00108a`, `isDirectModuleNotFoundError` only accepts `ERR_MODULE_NOT_FOUND` when `err.url` matches the resolved file URL or when the error message contains the absolute resolved path. `installProcessWarningFilter()` relies on that helper to swallow missing `./dist/warning-filter.*` imports, so a Bun-shaped error that reports the original specifier instead would still be rethrown.
