## Summary

Keep open. Current main still does not contain the PR's `enabledByDefault` manifest addition, and the relevant activation/public-surface code still depends on that manifest field for bundled default activation outside explicit slot/config activation. The PR is small and may still be useful, but it should get maintainer review because `memory-core` also has a separate default-memory-slot activation path, so the right product behavior is whether memory-core should be manifest-default-enabled or slot-only.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw wants `memory-core` to be default-enabled as a bundled plugin, land the manifest field with a focused regression test for doctor/update or activated runtime facade access. If the intended design is slot-only activation, close or revise the PR with an explanation and, ideally, add a regression test proving the reported doctor/update failure is handled through the default memory slot without enabling memory-core when the memory slot is disabled.
