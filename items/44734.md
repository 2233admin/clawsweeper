## Summary

Keep open. Current main still advertises `.ogg` and `.opus` as multimodal memory audio formats, builds raw inline-data embedding inputs with the detected MIME type, and the Gemini embedding adapter forwards those parts unchanged. Google’s current Gemini Embedding 2 material confirms audio is a supported modality, but the official examples show MPEG/MP3-style audio rather than proving raw OGG/Opus is accepted, so the reported provider-format gap is still actionable rather than safe to close. ([blog.google](https://blog.google/innovation-and-ai/models-and-research/gemini-models/gemini-embedding-2/?utm_source=openai))
