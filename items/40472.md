## Summary

Keep PR #40472 open. Current main still does not refresh the TUI after an external session reset: the gateway now emits a subscriber-gated `sessions.changed` reset signal, but the TUI gateway path neither subscribes to session-change events nor dispatches/handles `sessions.changed` or `session` lifecycle events. The linked bug #38966 remains applicable, and this PR is still a focused fix attempt, though it likely needs adaptation to the current `sessions.changed` subscription model.

## Best Possible Solution

Keep the PR open or supersede it with a maintainer-owned patch that makes `openclaw tui` subscribe to session change events, dispatch `sessions.changed` reset/new events for the current `state.currentSessionKey`, clear active/local/finalized run and stream state, reload history and session metadata, ignore unrelated sessions, and cover the gateway/TUI contract with focused tests.
