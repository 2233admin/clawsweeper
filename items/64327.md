## Summary

Keep open. Current main has the resolved effective context budget internally, but the plugin hook event/context types and emitters still do not expose that budget to `llm_output` or model-call hooks. The related #64260 is open and broader, but it does not clearly supersede this narrower request for the effective `contextTokens` cap.

## Best Possible Solution

Keep the issue open and implement an additive plugin hook contract change: expose the resolved effective context budget, likely from `contextTokenBudget`/`ctxInfo.tokens`, on `llm_output` and the model-call hook context or event with a clearly named field such as `contextTokens` or `effectiveContextTokens`. Update hook types, embedded and CLI emitters, hook tests, and plugin docs so plugins can calculate utilization without guessing from native model metadata.
