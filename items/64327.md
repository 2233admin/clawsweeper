## Summary

Keep this issue open. Current main resolves and uses an effective context-token budget internally, but the public plugin hook contract and the embedded, CLI, and Codex hook emitters still do not expose that effective limit on `llm_output` or sanitized `model_call_*` hook payloads. The related #64260 is still open and adjacent, not a shipped replacement.

## Best Possible Solution

Keep this issue open. The best fix is an additive plugin hook contract update that exposes the resolved effective context-token limit, and ideally the native model context metadata, on `llm_output` and sanitized `model_call_started`/`model_call_ended` telemetry. Source the value from existing `resolveContextWindowInfo()`/`ctxInfo.tokens`, update `src/plugins/hook-types.ts`, embedded/CLI/Codex emitters, `docs/plugins/hooks.md`, and focused hook tests, coordinating names with #64260.
