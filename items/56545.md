## Summary

Current `main` does not implement this PR's `agent-turn` cron failure-alert mode end to end. The CLI still rejects it, the gateway protocol and config schemas still only accept `announce|webhook`, and the runtime failure-alert / failure-destination paths on `main` still only model `announce|webhook`, so this is not a conservative close candidate.

## Evidence
