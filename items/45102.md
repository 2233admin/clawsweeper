## Summary

Keep open. The issue is fresh and reports a concrete gateway liveness failure under Windows/FunASR voice-recognition load. Current main has diagnostics for stuck sessions and an opt-in session JSONL truncation path, but I did not find a shipped, default-on fix that would make the gateway recover from or avoid the reported slowdown/timeouts. Related #64767 is useful evidence for one gateway-hang class, but it is narrower and still open, so it does not confidently supersede this FunASR/Windows report.

## Evidence

- **Issue report is concrete and not stale:** Issue #45102 reports OpenClaw 2026.3.7 on Windows with local FunASR processing, repeated voice recognition requests, gateway slowdown, /v1/responses timeouts, and restart as the workaround. It was created 2026-03-13 and updated 2026-03-30, so the missing-data stale close reason does not apply.
- **Related but not fully superseding issue:** The timeline cross-reference to #64767 tracks a concrete bloated-session-JSONL gateway hang, but #64767 is macOS/session-file specific and remains open. It helps explain a possible class of hangs, but does not fully cover the Windows/FunASR long-task path reported here.
