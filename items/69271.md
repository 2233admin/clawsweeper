## Summary

Keep #69271 open. Current main still supports Control UI session deletion only through checkbox selection plus the bulk delete bar; the Sessions row renderer has no per-row delete/archive/actions cell or callback. The gateway delete path does archive transcript files after removing the active session entry, but there is no first-class soft archive API, archived-session filter, or recovery view matching the request.

## Best Possible Solution

Keep this issue open. A narrow first step should add a row-level Actions column in `ui/src/ui/views/sessions.ts` and wire a single-session Delete button through the existing confirmed `deleteSessionsAndRefresh(state, [key])` path, with rendering/controller coverage. Treat soft archive as a separate gateway/product change unless maintainers explicitly want it here: define archive storage semantics, list/filter/recovery APIs, Control UI filters/actions, docs, and tests.
