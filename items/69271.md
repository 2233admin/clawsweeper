## Summary

Keep open. Current main still only supports Control UI session deletion through checkbox selection plus the bulk delete bar. The row renderer has no per-session action callback, no actions column, and no archive/filter/recovery UI; the gateway has `sessions.delete`, which removes the session entry and optionally archives transcript files, but that does not implement the requested per-row delete/archive action.

## Best Possible Solution

Keep the issue open. The narrow first step should add a per-row action affordance in `ui/src/ui/views/sessions.ts` wired through `ui/src/ui/app-render.ts` to the existing `deleteSessionsAndRefresh(state, [key])` path, preserving confirmation/loading/error handling. Treat the safer archive option as separate follow-up work unless maintainers want a new gateway/UI contract for archived session visibility and recovery.
