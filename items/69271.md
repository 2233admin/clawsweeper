## Summary

Keep #69271 open. Current main still only exposes session deletion in the Control UI through checkbox selection plus the bulk delete bar, and the renderer has no per-row delete/archive/dropdown action column. The gateway has `sessions.delete`, and deletion archives transcripts by renaming them to `.deleted.<timestamp>`, but that is still a delete operation that removes the session store entry; there is no `sessions.archive` API, archive-only state, or archived-session filter/view for the requested safer archive workflow.

## Best Possible Solution

Keep the issue open and implement the request in two scoped pieces: add a row-level Sessions table action that reuses the existing `sessions.delete` path for quick single-session deletion, then decide separately whether archive should become a first-class session state/API (`sessions.archive`, archived filter/view, restore semantics) instead of the current delete-plus-transcript-rename behavior.
