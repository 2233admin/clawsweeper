## Summary

Keep #71728 open. Current main has adjacent truthfulness mechanisms, including prompt guidance, input provenance, memory citations, and memory-wiki claim metadata, but it does not implement the RFC's requested claim-level confidence tiers, source attribution for every factual assistant claim, known-vs-inferred output contract, or autonomous completion reporting for unverified claims.

## Best Possible Solution

Keep the RFC open for maintainer product/design judgment. A credible implementation would define a core or SDK-visible trust signal for agent output, including claim-level evidence/source metadata where available, explicit known/believed/uncertain/unknown semantics, wording guidance for known-vs-inferred claims, and autonomous completion-event fields that summarize unverified claims. Memory citations and memory-wiki claims should be treated as useful evidence sources, not as the whole solution.
