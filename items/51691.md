## Summary

Keep open. Current main already shares DM context through the agent main session and supports optional cross-channel identity linking, but it does not implement the requested real-time Telegram <-> TUI session sync end to end. The key gap is that transcript/session update events are emitted by the gateway, while the TUI only reacts to chat run events, so Telegram-originated updates are not fully live-synced into the TUI UI.
