## Summary

Keep open. Current main does not implement either half of PR #69328: active-run `session.message` events are still deferred for a later history reload instead of appending the included transcript payload, and ACP background task terminal handling still has no worktree-diff verifier or zero-diff downgrade path. The PR is also not clearly merge-ready as-is because review comments identify a real edge case around `git diff --stat HEAD` missing untracked or unborn-HEAD worktrees.

## Best Possible Solution

Keep the PR open but request a revision: preserve the active-run `session.message` append/dedupe behavior, add focused tests for event-level `messageId`/`messageSeq`, and fix the ACP verifier to account for untracked files and unborn-HEAD repositories before using it to downgrade or fail zero-diff oneshot runs. If the ACP and UI fixes remain independent, splitting them would make review safer.
