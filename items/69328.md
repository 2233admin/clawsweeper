## Summary

Keep PR #69328 open. Current main has not implemented the PR's two central changes: there is no ACP worktree diff verifier/zero-diff terminal handling, and the control UI still defers active-run `session.message` updates instead of appending the payload directly. The PR itself needs revision before merge because the review comments identify a real gap in the proposed `git diff --stat HEAD` verifier for untracked files and unborn HEAD repositories.

## Best Possible Solution

Keep the PR open, but require a revision. The best path is to make the ACP diff verification robust for tracked, staged, untracked, and unborn-HEAD repositories or drop that gate if it is no longer product direction, then keep the UI change as a focused active-run `session.message` append path using the existing server-provided `message`, `messageId`, and `messageSeq` metadata with dedupe and optimistic echo replacement coverage.
