## Summary

Keep open. Current main still uses the older chat scroll behavior: auto-follow re-sticks from a broad 450px near-bottom threshold, there is no manual-scroll release lock, and the chat view does not wire wheel-intent handling. The related #70105 work is adjacent Debug event log behavior, not a superseding chat fix.

## Best Possible Solution

Keep #69520 open for maintainer review or a focused follow-up patch. The best path is to add an explicit chat manual-scroll release lock in ui/src/ui/app-scroll.ts, wire wheel intent through ui/src/ui/views/chat.ts, ui/src/ui/app-render.ts, ui/src/ui/app.ts, and AppViewState, and keep regression tests for slight upward manual scroll, return-to-bottom reacquire, nested scrollables, and smooth auto-scroll interruption. Treat #70105 as separate Debug event log work.
