## Summary

Keep open. Current main still has the old chat auto-follow behavior: it treats the broad 450px near-bottom threshold as enough to keep sticking, has no explicit manual-scroll release lock, and does not wire a wheel-intent handler on the chat thread. PR #69520 is still a plausible focused fix for the chat surface; the related #70105 work is for the debug event log and does not supersede this chat PR.

## Best Possible Solution

Keep PR #69520 open. The best path is to review/rebase it or apply an equivalent narrow fix in `ui/src/ui/app-scroll.ts`, `ui/src/ui/views/chat.ts`, `ui/src/ui/app.ts`, `ui/src/ui/app-render.ts`, and `ui/src/ui/app-view-state.ts`: add explicit manual-scroll release state, wheel-intent handling that ignores nested scrollables, true-bottom reacquisition, smooth-scroll interruption handling, and focused tests in `ui/src/ui/app-scroll.test.ts`. Treat #70105 as a separate debug event-log auto-follow item, not the canonical chat fix.
