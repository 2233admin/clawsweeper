## Summary

Keep open. Current `main` still uses the bare fallback filename `"file"` for WhatsApp document sends in the inbound send API, and the corresponding test still expects that behavior. The PR is not implemented on `main` as reviewed. Part of the PR's auto-reply intent appears obsolete because current auto-reply media normalization already derives document filenames from the URL, but the send-API bug remains.
