## Summary

Current `main` does not implement this request. Anthropic 1M context is still treated as an opt-in beta for Opus/Sonnet 4.x, and the Anthropic plugin explicitly drops that 1M beta path for OAuth/token auth, which is the affected case described in the issue.

## Evidence

- **1M only when `context1m` is set; automatic 1M is limited to Opus 4.7:** `resolveContextTokensForModel()` returns `ANTHROPIC_CONTEXT_1M_TOKENS` for Anthropic only when `modelParams?.context1m === true`, and the only unconditional 1M special-case is `shouldUseAnthropicOpus47ContextWindow()` for Opus 4.7.
  - file: [src/agents/context.ts:377](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/context.ts#L377)
  - command: `nl -ba src/agents/context.ts | sed -n '370,463p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **OAuth/token auth still falls back to the standard context window:** The Anthropic stream wrapper adds the 1M beta header only for `context1m`, then removes it for OAuth/legacy token auth and logs that it is 'falling back to the standard context window'.
  - file: [extensions/anthropic/stream-wrappers.ts:98](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/anthropic/stream-wrappers.ts#L98)

  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Current docs still describe 1M as beta-gated for Anthropic 4.6 models:** The Anthropic provider docs instruct users to enable `params.context1m: true` for 1M context and separately say only `anthropic/claude-opus-4.7` has 1M by default.
  - file: [docs/providers/anthropic.md:246](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/docs/providers/anthropic.md#L246)
