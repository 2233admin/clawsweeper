## Summary

Keep open. Current `main` only deletes the agent config entry, bindings/allow-list references, session-store rows, and local workspace/agent/transcript directories. It does not remove cron jobs or offer an `agents reset` flow, while cron creation/execution still persists and reuses session-specific targets. The reported `lcm.db` part is plugin-owned state, but the core cron/session persistence concern is still real on `main`.
