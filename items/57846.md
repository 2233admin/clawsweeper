## Summary

Current `main` does not implement separate Mattermost mention policy for channel posts vs thread replies. The code, config schema, docs, and tests all still model a single `requireMention` decision for Mattermost group/channel traffic.

## Evidence

- **Mention gate does not use thread-specific policy:** `evaluateMattermostMentionGate` accepts `threadRootId`, but computes `shouldRequireMention` only by resolving policy for `channelId`; `threadRootId` is otherwise unused in the decision.
  - file: [extensions/mattermost/src/mattermost/monitor-gating.ts:48](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/mattermost/src/mattermost/monitor-gating.ts#L48)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Mattermost group policy resolver is channel-scoped only:** `resolveMattermostGroupRequireMention` forwards a single `groupId` into `resolveChannelGroupRequireMention`; there is no thread-level override input or fallback.
  - file: [extensions/mattermost/src/group-mentions.ts:5](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/mattermost/src/group-mentions.ts#L5)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Config schema exposes only one mention flag:** The Mattermost schema defines `groups.<id>.requireMention` and has no `requireMentionInThread` or equivalent thread-specific field.
  - file: [extensions/mattermost/src/config-schema-core.ts:11](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/mattermost/src/config-schema-core.ts#L11)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Docs still document one per-channel mention override:** The Mattermost docs describe `groups.<channelId>.requireMention` as the per-channel override and do not document any separate thread mention behavior.
  - file: [docs/channels/mattermost.md:179](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/docs/channels/mattermost.md#L179)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Tests assert thread replies reuse the channel policy key:** The current Mattermost tests explicitly expect an unmentioned thread reply to resolve mention policy with `groupId = "chan-1"` and not the thread root id.
  - file: [extensions/mattermost/src/mattermost/monitor.test.ts:123](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/mattermost/src/mattermost/monitor.test.ts#L123)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
