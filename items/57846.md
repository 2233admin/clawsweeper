## Summary

Keep open. Current main does not implement a Mattermost-specific split between top-level channel mention gating and thread-reply mention gating. Mattermost still resolves one effective `requireMention` value per account/channel and applies it to all non-DM posts, including posts with `root_id`/thread context. The existing `onmessage` path permits unmentioned thread replies, but it also permits unmentioned top-level channel posts, so it does not solve the requested `requireMention=true` for channels plus relaxed thread replies behavior.

## Best Possible Solution

Keep the issue open and implement this in the bundled Mattermost plugin. The likely path is to add a Mattermost account/group thread mention policy, validate it in `extensions/mattermost/src/config-schema-core.ts`, teach `evaluateMattermostMentionGate` to distinguish top-level posts from `post.root_id` thread replies, and add regression tests for `requireMention=true` on root channel messages with unmentioned thread replies allowed only under the intended thread-participation rules. Update `docs/channels/mattermost.md` with the public config contract.
