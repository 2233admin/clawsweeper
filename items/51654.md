## Summary

Current `main` does not implement session-level environment variables for ACP spawns. The public `sessions_spawn` schema and ACP docs expose no `env` parameter, the ACP spawn path only forwards `model`/`cwd`, and the control-plane initializer does not pass any env through even though the lower ACP runtime interface already has an `env` field.

## Risks / Open Questions

- There is partial lower-level support (`AcpRuntimeEnsureInput.env`) that is not wired through the public spawn path, so closing as implemented would be premature.
- `AcpSessionRuntimeOptions` only allows `runtimeMode`, `model`, `cwd`, `permissionProfile`, `timeoutSeconds`, and `backendExtras`; there is still no persisted/session-scoped env surface for ACP sessions.
