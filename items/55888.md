## Summary

Keep open. Current `main` already supports the issue's proposed `agents.defaults.compaction` tuning, but it does not implement hardware-aware 64GB/M4-specific defaults or an automatic higher compaction threshold for this class of machine.

## Evidence

- **Current default remains a fixed 20k reserve floor:** `DEFAULT_PI_COMPACTION_RESERVE_TOKENS_FLOOR` is still `20_000`, and `resolveCompactionReserveTokensFloor()` only reads `agents.defaults.compaction.reserveTokensFloor`; there is no RAM- or machine-class-based defaulting here.
