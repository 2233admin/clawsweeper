## Summary

Current `main` already avoids the reported manual-cron deadlock by redirecting nested embedded-agent work away from the `cron` lane, so this PR's behavior change is already present through a different implementation.

## Close Comment

Closing as implemented on current `main` based on this Codex review. The reported deadlock path is already avoided in a different place now: manual `cron run` still enters the outer cron lane at `src/cron/service/ops.ts:704`, but the inner isolated embedded-agent call in `src/cron/isolated-agent/run-executor.ts:179` now goes through `resolveNestedAgentLane(params.lane)`, and `src/agents/lanes.ts:10` explicitly remaps `"cron"` to `CommandLane.Nested` so the nested run no longer contends for the same cron slot. There is also direct coverage for that remapping in `src/agents/lanes.test.ts:14`. I could not tie this to a shipped release tag yet, but the behavior is present on `main` in commit `29d0e930bb3de4886b2e799781eb5620d973d1be`, so this PR is obsolete.
