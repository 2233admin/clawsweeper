## Summary

Keep open. Current `main` still advertises Gemini `web_search` freshness as unsupported, rejects it before any Gemini API call, and the docs still tell users Gemini does not support `freshness`; there is no landed OpenClaw fix to close this issue. Official Gemini grounding docs also do not show a documented freshness option for the current `google_search` tool, so the request still needs maintainer/product judgment rather than conservative cleanup.
