## Summary

Keep open. Current main still does not pass Gemini `web_search` freshness through: the Google plugin schema marks `freshness` and date filters unsupported, the Gemini runtime returns the shared unsupported-filter response before building the request, and the request body still sends only an empty `google_search` grounding tool. The related open #17925 is broader native-search passthrough work, not a canonical replacement for this narrower Gemini freshness gap.

## Best Possible Solution

Keep this issue open. The best path is a Google-plugin-owned fix or explicit maintainer decision: verify the Gemini grounding API contract, then map OpenClaw `freshness` and date filters into a supported native request field if one exists, or implement a documented query/date normalization strategy if native freshness is unavailable. Update `extensions/google/src/gemini-web-search-provider.ts`, `extensions/google/src/gemini-web-search-provider.runtime.ts`, Gemini web-search tests, cache-key construction, and `docs/tools/gemini-search.md`.
