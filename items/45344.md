## Summary

Keep open. Current `main` still routes compat HTTP requests through `resolveAgentIdForRequest` without validating explicit agent ids, so the requested behavior is not already implemented. The PR also has unresolved review concerns around allowlist semantics and explicit `main` compatibility, so this needs maintainer judgment rather than conservative auto-close.
