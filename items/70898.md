## Summary

Current `main` does not implement the DM-continuity replay described in PR #70898. The reset path still mints a fresh session/transcript and immediately repoints follow-up runs to it without seeding prior user/assistant turns, so this item should remain open.

## Evidence

- **Replay helper absent on current main:** Current `main` only contains the existing reset/session-reset symbols; there is no `session-transcript-replay` helper or `replayRecentUserAssistantMessages` implementation in the tree.
  - command: `rg -n "replayRecentUserAssistantMessages|session-transcript-replay|resetReplyRunSession|performGatewaySessionReset" src/auto-reply src/gateway`
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Reset path still switches to a brand-new transcript without copying prior turns:** `resetReplyRunSession` generates a new session id, resolves a new transcript path, stores it on the session entry, and repoints the queued follow-up run to that new file. There is no step between those operations that copies or replays prior DM messages into the new transcript.
