## Summary

Keep PR #70898 open. Current main still lacks transcript replay/hydration during silent auto-reply session rotations, so the linked DM continuity bug is not implemented on main. The PR is relevant and focused, but its latest head still has unresolved correctness and scalability review blockers, so it should not be closed as implemented or obsolete.

## Best Possible Solution

Keep #70898 open and ask for another revision: seed recent DM context during silent `resetReplyRunSession` rotations, but make the replay tail provider-safe by normalizing/collapsing same-role runs or reusing existing transcript hygiene semantics, use a bounded tail reader instead of full-file loading, add regression coverage for same-role and assistant-only tails, and fix the changelog attribution before maintainer review.
