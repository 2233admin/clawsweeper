## Summary

Keep PR #70898 open. Current main still rotates silent in-reply sessions to a new session id/file without seeding recent direct-chat context, and Telegram DMs do not have the group pending-history fallback. The PR is therefore not obsolete, implemented, duplicate, or ClawHub-scoped; it is a plausible core auto-reply/session fix for the open #70853 user-visible bug, though the PR implementation still needs maintainer review/fixups before merge.

## Best Possible Solution

Keep #70898 open and continue it as a core auto-reply/session fix. The best implementation should preserve a small, bounded, provider-safe recent user/assistant context across silent compaction/role-ordering rotations, keep explicit `/new` and `/reset` fresh, avoid full transcript reads on the recovery path, normalize replay so it cannot recreate strict role-ordering failures, and add regression tests covering the silent reset path plus Telegram direct-chat continuity.
