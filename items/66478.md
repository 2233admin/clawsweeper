## Summary

Keep open. Current `main` still passes raw `emoji` strings straight into the Feishu reactions API, while the Feishu API contract expects `emoji_type` values like `SMILE`/`THUMBSUP`. The reported bug is still plausible on current `main`, the latest release does not contain a fix on this surface, and the PR discussion points to a real, bounded patch rather than an obsolete change.

## Evidence

- **Current main still forwards raw emoji input:** In the `react` action handler, `ctx.params.emoji` is only trimmed and then passed directly as `emojiType` into both `listReactionsFeishu` and `addReactionFeishu`; there is no normalization step on current `main`. That matches the PR's stated root cause.
