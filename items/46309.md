## Summary

Keep open. Current `main` contains explicit billing/rate-limit fallback code, docs, and tests, but that does not conclusively prove this reported end-to-end bug is fixed. The control UI still uses the same freeform fallback chip input the reporter called out, and the small post-release fallback diff is not strong enough evidence to close this issue conservatively.
