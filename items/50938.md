## Summary

Current `main` contains a narrower follow-up, not this PR's full behavior. Telegram preview-finalized replies now emit only the internal `message:sent` hook, but preview-retained paths still do not emit sent hooks and the preview path does not use the shared plugin+internal hook emitter used by `deliverReplies`. That is not strong enough evidence to close this PR as already implemented.
