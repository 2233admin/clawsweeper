## Summary

Current `main` does not contain the PR’s reviewed package.json-based manifestless-runtime fix. The facade activation path still rejects manifestless bundles unless they hit a hardcoded three-item allowlist, and the runtime package `package.json` files on `main` do not carry the explicit `openclaw.alwaysAllowedRuntimeApi` metadata the PR discussion converged on. That means this PR is not cleanly implemented on `main`, so conservative cleanup should leave it open.

## Evidence

- **Manifestless bundles still return `null` without `openclaw.plugin.json`:** `readBundledPluginManifestRecordFromDir` immediately returns `null` when `openclaw.plugin.json` is absent; there is no fallback to derive identity from `package.json` on current `main`. That is the core behavior this PR proposes to change.
