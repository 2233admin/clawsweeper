## Summary

Keep open. Current main has adjacent Telegram polling watchdogs, channel health-monitor recovery, and per-session nested `sessions_send` lanes, but I did not find code, tests, docs, changelog, or issue discussion proving that nested subagent activity can no longer freeze all Telegram pollers plus gateway health timers. The recent “Affects me” comment is current field signal, so conservative cleanup should not close this issue.

## Best Possible Solution

Keep this issue open until maintainers reproduce it on current main with multiple Telegram bot accounts and nested `sessions_send`, or land/link a targeted fix with a regression proving nested subagent completion cannot stall Telegram polling and health-monitor timers. The most useful implementation path is an integration/liveness regression around coordinator-to-subagent `sessions_send`, Telegram polling watchdog behavior, and gateway health-monitor activity.
