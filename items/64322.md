## Summary

Keep open. Current main still has the Discord ACP parent-channel fallback returning the parent channel ID as the materialized conversation, and the existing tests still encode that fallback behavior. The PR is a focused bug fix with useful review history; it is not implemented on main and is not clearly superseded by the broader open #50955 work.

## Best Possible Solution

Keep the PR open and review it as a focused Discord ACP bug fix. The best path is a surgical Discord parent-match change that materializes thread IDs while preserving provider-canonical remaps such as Feishu topic folding, plus regression coverage for distinct Discord thread session keys and explicit handling or documentation for session-key-only reverse lookup.
