## Summary

Keep this PR open. Current main still has the Discord configured-ACP thread collision that this PR targets: parent-channel matches materialize the parent channel ID, and configured ACP session keys hash that materialized conversation ID, so sibling Discord threads under the same configured parent can share one ACP session. The paired open bug report and PR discussion support the focused fix, and the PR is not superseded by current main.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to rebase and review a surgical Discord fix where parent-channel matches materialize the runtime thread ID while preserving `parentConversationId`, retain registry support for channel-canonical matches, and land regression coverage proving two Discord threads under one configured parent get distinct ACP session keys plus the intended reset/reverse-lookup behavior.
