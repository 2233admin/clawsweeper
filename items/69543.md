## Summary

Keep open. Current `main` still routes dreaming narrative sessions into the normal session listing, and the Control UI session picker still only hides cron sessions, not dreaming/internal ones.

## Evidence

- **Gateway session listing still includes non-cron store entries:** `listSessionsFromStore` filters out cron, global, and unknown keys, then turns the remaining session-store entries into `GatewaySessionRow` items for `sessions.list`. There is no dreaming/internal-session exclusion here.
  - file: [src/gateway/session-utils.ts:1511](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/gateway/session-utils.ts#L1511)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1490,1548p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Control UI picker only hides cron sessions:** `resolveSessionOptionGroups` builds the picker options from every returned session row and only skips rows when they are `global`/`unknown` or match `isCronSessionKey(...)`. Dreaming sessions are not filtered or separated.
  - file: [ui/src/ui/chat/session-controls.ts:473](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/ui/src/ui/chat/session-controls.ts#L473)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '378,560p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Dreaming still creates normal session-keyed narrative runs:** Memory-core still launches narrative runs with `sessionKey: params.sessionKey`, and those keys are built as `dreaming-narrative-<phase>-<workspaceHash>-<timestamp>`. That keeps dreaming runs in the ordinary session-key space.
  - file: [extensions/memory-core/src/dreaming-narrative.ts:153](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/memory-core/src/dreaming-narrative.ts#L153)
  - command: `nl -ba extensions/memory-core/src/dreaming-narrative.ts | sed -n '140,210p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Current tests assume dreaming sessions persist in sessions.json:** The current test suite writes `agent:main:dreaming-narrative-light-...` entries into `sessions.json`, and the memory host code explicitly scans the normal session store for dreaming-narrative keys. That is evidence the persisted-store behavior is still intentional on `main`, even though it is only filtered for dreaming ingestion, not for Control UI session selection.
