## Summary

Keep open. Current `main` appears to have addressed 2 of the 3 reported items: the Control UI now reads saved model selections back from config/defaults and preserves unknown current models in the selector, and the BOOT prompt/template both use `NO_REPLY` rather than a conflicting `NO_REPLY_NEEDED` token. But the restart-sentinel path still has no config surface to suppress channel-facing restart notices, and startup still actively delivers the sentinel message back to the routed channel when a destination resolves.
