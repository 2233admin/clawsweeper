## Summary

Keep open. Current `main` and the latest shipped release still hardcode the MiniMax CN usage endpoint for coding-plan polling, while current config/docs support global `models.providers.minimax.baseUrl` values. A plausible fix exists in open PR #65059, but it has not landed; the other linked PRs were auto-closed for author queue limits, not because the report was disproven.
