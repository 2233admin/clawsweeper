## Summary

Keep open. Current main still treats fish completion installation as a shell-profile edit: it sources the cached fish script from `~/.config/fish/config.fish`, checks installed state through that profile, and docs/doctor/setup messaging still describe profile sourcing. The directly related PR #27135 addressed this request but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep tracking this issue until a fish-specific completion install path lands. The likely fix belongs in the CLI completion runtime and related doctor/setup surfaces: copy or write the generated fish cache to `~/.config/fish/completions/<binName>.fish`, detect that file in installed-state checks, clean up legacy `config.fish` source blocks during upgrade, update docs and user messages, and add regression tests for fish install/upgrade behavior.
