## Summary

Keep #69727 open. Current main still has assistant-message copy/expand actions and whole-chat markdown export, but no individual assistant-message Print button, isolated print view, print-friendly message CSS, or `window.print()` flow for one message.

## Best Possible Solution

Keep the issue open and implement a focused Control UI chat feature: add an assistant-only print action next to the existing copy action in `ui/src/ui/chat/grouped-render.ts`, backed by a helper that renders the same sanitized visible message into an isolated print document/window, applies print-friendly CSS, and triggers `window.print()`. Add targeted tests for action rendering and print helper behavior; update user-facing Control UI docs only if chat message actions are documented there.
