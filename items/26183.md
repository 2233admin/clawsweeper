## Summary

Current `main` does not implement automatic WhatsApp ack-reaction removal after reply. The WhatsApp config/schema only supports `emoji`, `direct`, and `group` under `channels.whatsapp.ackReaction`, the inbound path only sends the pre-reply ack, and the shared docs still state `messages.removeAckAfterReply` applies to Slack, Discord, and Telegram only.

## Evidence

- **WhatsApp ackReaction config has no removal fields:** `WhatsAppAckReactionConfig` only defines `emoji`, `direct`, and `group`; there is no `remove`, `removeAckAfterReply`, or equivalent WhatsApp-specific lifecycle option.
  - file: [src/config/types.whatsapp.ts:38](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/config/types.whatsapp.ts#L38)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **WhatsApp schema would reject the reported keys:** `WhatsAppAckReactionSchema` is a strict object with only `emoji`, `direct`, and `group`, so `channels.whatsapp.ackReaction.remove` / `removeAckAfterReply` would be unrecognized exactly as reported.
  - file: [src/config/zod-schema.providers-whatsapp.ts:42](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/config/zod-schema.providers-whatsapp.ts#L42)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **WhatsApp inbound flow only sends the ack reaction:** `processMessage` calls `maybeSendAckReaction()` immediately on inbound receipt. A repo-wide search of `extensions/whatsapp/src` finds no `ackReactionPromise`, `ackReactionValue`, `removeAfterReply`, or `removeAckReactionAfterReply` usage, so there is no follow-up removal lifecycle wired for WhatsApp.
  - file: [extensions/whatsapp/src/auto-reply/monitor/process-message.ts:166](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/extensions/whatsapp/src/auto-reply/monitor/process-message.ts#L166)
  - command: `rg -n "ackReactionPromise|ackReactionValue|removeAfterReply|removeAckAfterReply" extensions/whatsapp/src`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Core docs still scope removeAckAfterReply away from WhatsApp:** The configuration reference explicitly says `removeAckAfterReply` removes ack reactions on Slack, Discord, and Telegram.
  - file: [docs/gateway/configuration-reference.md:2108](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/docs/gateway/configuration-reference.md#L2108)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **WhatsApp removal exists only as a manual reaction tool action:** The WhatsApp action runtime maps `remove: true` to an empty emoji for the `react` tool, and the reactions doc describes that manual tool behavior. That is separate from automatic ack cleanup after reply.
  - file: [extensions/whatsapp/src/action-runtime.ts:46](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/extensions/whatsapp/src/action-runtime.ts#L46)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
