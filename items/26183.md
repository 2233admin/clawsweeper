## Summary

Keep open. Current main still sends WhatsApp ack reactions but does not track the ack send, does not call the shared post-reply removal helper, and does not accept a WhatsApp ackReaction remove/removeAckAfterReply config key. The issue is also reinforced by the latest maintainer/Codex review comment on 2026-04-26, which explicitly kept it open for the same missing lifecycle behavior.

## Best Possible Solution

Implement WhatsApp automatic ack cleanup in the WhatsApp plugin: have maybeSendAckReaction return or record whether the ack was sent, wire processMessage/dispatchWhatsAppBufferedReply to call removeAckReactionAfterReply after a successful reply when messages.removeAckAfterReply is true, and add schema/types/docs/tests for the chosen config contract if maintainers want a WhatsApp-specific override.
