## Summary

Keep PR #70681 open. Current main still has the Unix `lsof -Fpc` command-field filter that the PR is trying to replace, and the existing tests still pin that command-name behavior. The PR appears to address a real, focused gateway restart bug linked to #70664, though the submitted diff should address the review comments about poll-path timeouts before merge.

## Best Possible Solution

Keep the PR open and ask the contributor or maintainer to finish the Unix ps/argv verification fix on top of current main, including timeout-safe `pollPortOnce` verifier calls and updated regression tests. If maintainers prefer not to use this branch, the same behavior should still be implemented in `src/infra/restart-stale-pids.ts` because the current main bug remains observable from the source and tests.
