## Summary

Current main still does not implement the requested `agents.setDefault` Gateway RPC. The Gateway handler table, advertised method list, operator scope map, protocol schemas, and protocol docs only cover `agents.list/create/update/delete` plus file helpers. `agents.update` still rejects a `default` field, and the Control UI's current workaround still stages invalid `agents.defaultId`, matching the open related bug #59070.

## Best Possible Solution

Keep this issue open and implement an additive core Gateway method: add `agents.setDefault` params/result schemas, validators, method registry entry, ADMIN scope entry, and a handler that rejects unknown configured agent ids, sets only the selected `agents.list[]` entry to `default: true`, clears/removes the flag on all others, writes config, and returns `{ ok: true, defaultId }`. Update protocol docs and switch the Control UI Set Default action away from `agents.defaultId`; add focused protocol/handler tests plus UI/controller coverage. This should also resolve the related #59070 invalid-config path.
