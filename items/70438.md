## Summary

Keep open. Current `main` still hard-codes WebChat uploads to images and drops non-image attachments in the gateway, and there is no existing `webchat` attachment config knob matching the request.

## Evidence

- **WebChat picker only accepts images:** `CHAT_ATTACHMENT_ACCEPT` is hard-coded to `image/*`, so the file picker is intentionally image-only.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/ui/src/ui/chat/attachment-support.ts#L1)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **UI filters non-image files on select/drop/paste:** The chat view only keeps files whose MIME type starts with `image/`, and the file input uses the image-only accept string.
  - file: [ui/src/ui/views/chat.ts:213](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/ui/src/ui/views/chat.ts#L213)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **RPC payload serializes every attachment as an image:** WebChat converts attachments into `{ type: "image", mimeType, content }` before calling `chat.send`, with no document/text path.
  - file: [ui/src/ui/controllers/chat.ts:349](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/ui/src/ui/controllers/chat.ts#L349)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Gateway drops non-image attachments:** `parseMessageWithAttachments` warns and skips attachments when MIME sniffing finds a non-image or cannot confirm an image MIME type.
  - file: [src/gateway/chat-attachments.ts:347](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/gateway/chat-attachments.ts#L347)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **No requested config surface exists:** `GatewayWebchatConfig` currently exposes only `chatHistoryMaxChars`; there is no `webchat.attachments` or equivalent WebChat attachment policy config.
  - file: [src/config/types.gateway.ts:398](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/config/types.gateway.ts#L398)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
