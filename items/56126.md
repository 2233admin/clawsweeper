## Summary

Keep open. Current `main` does not implement a cron `payload.batch` flag or Anthropic Batch API path, and while provider-specific transport work is intentionally plugin-owned in OpenClaw, the existing cron/provider seams do not make this a high-confidence `clawhub` cleanup.

## Evidence

- **Vision keeps optional capability out of core, but asks for plugin/API extension when needed:** `VISION.md` says optional capability should usually ship as plugins, and if a useful feature cannot be built as a plugin yet, maintainers prefer extending the plugin API instead of adding one-off core behavior.

  - file: [docs/concepts/model-providers.md:37](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/concepts/model-providers.md#L37)
