## Summary

Keep open. Current `main` does not implement shared session context between group chats and DMs for the same identity. The supported config only scopes DMs (`main`/`per-peer`/`per-channel-peer`/`per-account-channel-peer`) and `identityLinks` only participates in the direct-message path; group chats still resolve to separate `group`/`channel` session keys.

## Evidence

- **Session docs still describe groups as isolated:** The current session docs say direct messages are shared by default, but group chats are isolated per group. They also document `identityLinks` only as a way to link the same person across channels for DM sharing.
