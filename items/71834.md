## Summary

Keep #71834 open. The 2026.4.22 generated thinking-bundle syntax regression appears addressed by the 2026.4.24/current source shape, but the remaining Telegram webhook startup failure is still present on current main: `setWebhook` failure closes the already-bound local listener and rethrows into the generic channel restart backoff, leaving the reverse proxy with no local webhook listener during the backoff window.

## Best Possible Solution

Keep the issue open and fix the Telegram webhook startup path in the bundled Telegram plugin. The likely maintainer direction is to decouple local listener lifetime from Telegram `setWebhook` registration: once the listener binds, keep it alive through recoverable `setWebhook` network failures, retry registration with bounded backoff, and add a regression test that makes `setWebhook` fail transiently while proving the health/webhook listener is still reachable or is restored before retry backoff leaves the reverse proxy with no backend.
