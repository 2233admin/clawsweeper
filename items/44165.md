## Summary

Keep open. Current `main` still has the omission this PR targets: the Channels view only renders ids from `snapshot.channelMeta` or `snapshot.channelOrder`, so built-in channel cards disappear when the snapshot only reports custom channels. The proposed fix is not present on `main`, and there is no conservative auto-close reason.

## Evidence

- **Channels view still drops built-ins when snapshot metadata is custom-only:** `resolveChannelOrder()` on current `main` returns `snapshot.channelMeta.map(...)` or `snapshot.channelOrder` directly, and only falls back to the built-in list when neither exists. That means a custom-only snapshot suppresses built-in cards instead of appending them.
  - file: [ui/src/ui/views/channels.ts:97](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/ui/src/ui/views/channels.ts#L97)
  - command: `nl -ba ui/src/ui/views/channels.ts | sed -n '1,220p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Gateway payload order comes straight from the UI catalog:** The gateway status payload sets `channelOrder` and `channelMeta` from `buildChannelUiCatalog(plugins)`, so the UI depends on whatever plugin list the gateway resolved for that run.
  - file: [src/gateway/server-methods/channels.ts:289](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/gateway/server-methods/channels.ts#L289)
  - command: `nl -ba src/gateway/server-methods/channels.ts | sed -n '250,320p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **UI catalog does not append built-in channels:** `buildChannelUiCatalog()` simply maps the supplied `plugins` array into `entries` and `order`; it does not merge in a built-in fallback list. If the resolved plugin set is custom-only, the snapshot order is custom-only too.
  - file: [src/channels/plugins/catalog.ts:287](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/channels/plugins/catalog.ts#L287)
  - command: `nl -ba src/channels/plugins/catalog.ts | sed -n '260,360p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Read-only channel resolution can produce non-built-in-driven lists:** Channel resolution starts from currently loaded channel plugins and then adds configured channel ids as needed. That makes a custom-only/plugin-specific channel set plausible, which matches the PR's reported scenario.
  - file: [src/channels/plugins/read-only.ts:543](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/channels/plugins/read-only.ts#L543)
