## Summary

The reported dashboard breakage matches a dashboard-v2 chat/history reload bug that was fixed after 2026.3.12. Current main only reloads chat history after terminal events, includes a regression test for live tool-result events, and the changelog shows the fix shipped in 2026.3.13.

## Evidence

- **Release note for the dashboard fix:** The 2026.3.13 changelog explicitly says: 'Dashboard/chat UI: stop reloading full chat history on every live tool result in dashboard v2 so tool-heavy runs no longer trigger UI freeze/re-render storms while the final event still refreshes persisted history.' This directly matches a 2026.3.12-era dashboard session breaking mid-response and requiring refresh/reconnect.
  - file: [CHANGELOG.md:2532](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/CHANGELOG.md#L2532)
  - command: `nl -ba CHANGELOG.md | sed -n '2518,2540p'`
- **Current main only reloads history after terminal tool runs:** The dashboard now reloads chat history only when tools were used and the run reached a final state, instead of on each live tool result event.
  - file: [ui/src/ui/app-gateway.ts:422](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/ui/src/ui/app-gateway.ts#L422)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '418,430p'`

  - file: [ui/src/ui/app-gateway.node.test.ts:622](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/ui/src/ui/app-gateway.node.test.ts#L622)

## Close Comment

Closing from a Codex review because this looks implemented on current `main`. The report was filed against OpenClaw `2026.3.12`, and the `2026.3.13` changelog already includes a dashboard fix for reloading full chat history on every live tool result, which caused dashboard-v2 freeze/re-render storms (`CHANGELOG.md:2532`). Current main keeps that fix in code by only reloading history after terminal tool runs (`ui/src/ui/app-gateway.ts:422`) and by covering the regression with a test that asserts live tool-result events do not trigger a history reload (`ui/src/ui/app-gateway.node.test.ts:622`). The large fallback screen the reporter described also matches the intentional disconnected login gate (`ui/src/ui/app-render.ts:663`). If this is still reproducible on a current build such as `v2026.4.22` or newer, please open a fresh issue with exact repro steps and current logs/screenshots.
