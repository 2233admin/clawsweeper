## Summary

Keep PR #69454 open. Current main still has the main Control UI churn and stale-index behaviors the PR targets: unbounded chat/session refresh in key paths, eager node/device work on connect, unconditional node polling, immediate sessions.changed reloads, and Cache-Control: no-cache for Control UI index.html. The PR is still potentially useful, but it also has an unresolved review note about deferred session reloads after tab switches, so it should be fixed/reviewed rather than closed as obsolete.

## Best Possible Solution

Keep the PR open and ask for a small follow-up: rebase against current main, fix the deferred sessions reload by re-checking the active tab or clearing the timer on tab change, and keep the focused tests for bounded session loading, tab-scoped node polling/preloads, sessions.changed debounce behavior, and no-store index.html caching. Close only after equivalent code and tests land on main or a newer canonical PR supersedes it.
