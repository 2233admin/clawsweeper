## Summary

Keep #58887 open. Current main still downloads Telegram voice media and performs audio preflight transcription while building the Telegram message context, before the reply pipeline is created and before the `typingMode: "instant"` run-start signal can call Telegram `sendChatAction("typing")`. The related #39052/#39075 Telegram performance work does not supersede this UX request, and the same-title ClawSweeper review for #58889 also found the capability missing.

## Best Possible Solution

Implement this in the Telegram inbound voice/audio path by sending a best-effort `sendChatAction(chatId, "typing", threadParams)` immediately after the message is accepted and before media download/STT preflight begins, while preserving authorization/mention-gating semantics and the existing `sendChatAction` backoff behavior. Add a focused Telegram test that proves voice/audio receipt triggers typing before `transcribeFirstAudio` resolves, and ensure related media-download failure paths do not leak typing for blocked or unauthorized messages.
