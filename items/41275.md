## Summary

Current `main` still rejects `timeoutSeconds: 0` in the Control UI and drops it from the built cron payload, while the cron runtime/protocol already treat `0` as valid no-timeout behavior. This PR remains materially useful and should stay open.

## Evidence

- **UI validation still rejects `0`:** `validateCronForm` trims the field, parses with `toNumber(..., 0)`, and flags any `timeout <= 0` as invalid, so a literal `0` still fails on current `main`.
