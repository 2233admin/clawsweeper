## Summary

Keep open. Current `main` fixes the silent-swallow path and snapshot sync, and it adds Gemini project-discovery fallbacks, but the Gemini OAuth refresh path still depends on `@mariozechner/pi-ai`'s generic `getOAuthApiKey` wrapper that strips the underlying refresh cause, and `google-gemini-cli` is still not exempt from auth cooldowns after transient refresh failures.

## Evidence

- **Refresh failures are no longer silently swallowed on main:** `resolveApiKeyForProfile` now wraps OAuth refresh failures and throws a user-facing `OAuth token refresh failed for ...` error instead of returning `null` from a bare `catch {}` path.
  - file: [src/agents/auth-profiles/oauth.ts:330](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/agents/auth-profiles/oauth.ts#L330)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Runtime auth snapshots are now updated after saves:** `saveAuthProfileStore` writes the persisted store, refreshes the in-memory cache, and updates an existing runtime snapshot via `setRuntimeAuthProfileStoreSnapshot(...)`, addressing the stale-snapshot part of the report.
  - file: [src/agents/auth-profiles/store.ts:375](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/agents/auth-profiles/store.ts#L375)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Gemini project discovery now has explicit env fallback coverage:** `discoverProject` now returns `GOOGLE_CLOUD_PROJECT` / `GOOGLE_CLOUD_PROJECT_ID` when `loadCodeAssist` fails or when Google returns a tier without a project, and the regression test covers that fallback.
  - file: [extensions/google/oauth.project.ts:159](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/extensions/google/oauth.project.ts#L159)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Upstream OAuth helper still erases the underlying refresh error:** OpenClaw still pins `@mariozechner/pi-ai` `0.70.2`. Inspecting the published package shows `getOAuthApiKey` catches provider refresh failures and rethrows `Failed to refresh OAuth token for ${providerId}` without preserving the original cause, so the HTTP/status detail described in this issue is still lost on the shared fallback path.
  - file: [package.json:1604](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/package.json#L1604)
  - command: `curl -Lsf https://registry.npmjs.org/@mariozechner/pi-ai/-/pi-ai-0.70.2.tgz | tar -xzOf - package/dist/utils/oauth/index.js | nl -ba | sed -n '100,150p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Gemini is still not bypassed for auth cooldowns:** `isAuthCooldownBypassedForProvider` still returns `true` only for `openrouter` and `kilocode`; `google-gemini-cli` is not included, so the transient-refresh-failure cooldown behavior called out in the issue remains on current `main`.
  - file: [src/agents/auth-profiles/usage-state.ts:4](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/agents/auth-profiles/usage-state.ts#L4)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
