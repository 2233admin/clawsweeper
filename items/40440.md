## Summary

Current main still matches the reported Telegram bug: skipped mention-gated group messages can have downloaded media in scope, but the pending history entry records only sender/body/timestamp/messageId, and later @mention context construction only forwards current-turn and reply media. The related fix PRs #40470 and #40490 are closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open and implement the focused Telegram/shared-history fix: extend the reply-history and `MsgContext.InboundHistory` contracts with optional media references, persist or refresh media for skipped mention-gated group messages, merge replayed history media into later Telegram `MediaPaths`/`MediaTypes`, update prompt metadata if needed, and add a regression test for image-without-mention followed by an @mention asking about that image.

## Risks / Open Questions

- No live Telegram group reproduction was performed because this was a read-only review, but the current source path directly matches the reported failure mode.
- The eventual fix needs an explicit media lifetime policy so history entries do not point at expired inbound files or retain group media longer than intended.
- Extending `HistoryEntry` affects a plugin-SDK-exported contract, so the implementation should be backwards-compatible and covered by focused Telegram plus shared history tests.
