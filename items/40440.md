## Summary

Current main still has the reported Telegram group-history media loss: skipped no-mention group messages record only sender/body/timestamp/messageId, the shared HistoryEntry and MsgContext.InboundHistory shapes have no media fields, and later Telegram context construction only merges current-turn and reply media into MediaPaths. Related fix PRs #40470 and #40490 are closed unmerged, so the requested behavior remains missing.

## Best Possible Solution

Keep #40440 open and implement the Telegram plugin fix: extend the reply-history/InboundHistory contract with media references, persist or refresh skipped group-message media beyond the current temp-file lifetime, merge history media into later MediaPaths/MediaTypes when the bot is mentioned, and add a focused regression test for image-without-mention followed by @mention.
