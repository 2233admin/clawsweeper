## Summary

Keep #40332 open. Current main still has only global ACPX `permissionMode` and `nonInteractivePermissions` config, while the per-agent runtime and per-binding ACP config contracts remain strict and do not accept or propagate those fields. Related #63859 closed the global workaround/documentation path but explicitly left per-agent permission scoping to this issue.

## Best Possible Solution

Implement the requested declarative permission scoping as an ACP/runtime contract change: add `permissionMode` and `nonInteractivePermissions` to `agents.list[].runtime.acp` and `bindings[].acp`, validate and document those fields, persist them in configured binding metadata, and thread them into ACP session initialization/runtime ensure so ACPX resolves precedence as binding over agent runtime over global plugin config.
