## Summary

Keep PR #69529 open. Current main does not contain the proposed cross-provider account suppression, and the gateway approvals schema still omits the `source` metadata that existing exec-approval code writes and tests preserve. The PR likely needs maintainer cleanup because it bundles unrelated fixes and its test placement is stale, but the core bugs it targets still map to current code.

## Best Possible Solution

Keep the PR open, but ask for a focused rebase: implement cross-provider `defaultAccountId` suppression in `src/infra/outbound/message-action-runner.ts` before `params.accountId` is set, add a regression in the current `message-action-runner.plugin-dispatch.test.ts` or another focused outbound runner test, add `source` to the gateway exec approvals schema with protocol validation coverage, and move the skill-description cleanup to #69475 or a separate PR.
