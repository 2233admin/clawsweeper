## Summary

Keep #57621 open. Current main still lacks Mattermost WebSocket protocol ping/pong keepalive, and the open PR addresses a real remaining reliability gap in the bundled Mattermost plugin rather than obsolete or external work.

## Best Possible Solution

Keep the PR open and finish it as a focused Mattermost plugin reliability fix in `extensions/mattermost/src/mattermost/monitor-websocket.ts`: add WebSocket ping/pong termination, cover the missing-pong timeout with a unit test, resolve the timer cleanup nit, and align any config/docs claims before marking the linked Mattermost issues fixed.
