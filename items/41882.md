## Summary

Keep open. Current `main` still serializes Discord inbound work per session/channel in the inbound worker, so the PR's steer-mode bypass behavior is not implemented on `main` after the Discord code moved into `extensions/discord`. The current queue semantics still recognize `steer`/`steer-backlog` as immediate-steer modes, and the Discord regression tests only cover serialized same-session handling, not a steer-mode bypass.
