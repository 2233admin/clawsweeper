## Summary

Current `main` already replaced the PR's targeted CLI plugin-loading path with centralized bootstrap/loading code, and the shared registry state now lives in `src/plugins/runtime/runtime-registry-loader.ts`. The specific static-vs-dynamic `preaction.ts`/`route.ts` double-registration path described in the PR no longer exists on `main`.
