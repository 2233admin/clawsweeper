## Summary

Keep open. Current `main` defines the `model.usage` payload in TypeScript, but the public docs still only summarize it and do not publish the exact JSON keys the issue asks for. The code also shows these diagnostics are in-process events consumed by the stability recorder and optional OTel exporter, not a documented stable schema for `/tmp/openclaw/openclaw-YYYY-MM-DD.log` entries.

## Evidence

- **Docs are still too vague:** The logging docs only say `model.usage` includes “tokens, cost, duration, context, provider/model/channel, session ids” and do not list the actual JSON field names.
  - file: [docs/logging.md:200](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/docs/logging.md#L200)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Exact payload exists only in source types:** `DiagnosticUsageEvent` defines the concrete fields as `sessionKey`, `sessionId`, `channel`, `provider`, `model`, nested `usage.{input,output,cacheRead,cacheWrite,promptTokens,total}`, optional `lastCallUsage`, `context.{limit,used}`, `costUsd`, and `durationMs`.
  - file: [src/infra/diagnostic-events.ts:12](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/infra/diagnostic-events.ts#L12)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Emitter matches the source type, not the docs:** The main emitter constructs `model.usage` with those exact keys, including nested `usage`, `lastCallUsage`, `context`, `costUsd`, and `durationMs`.
  - file: [src/auto-reply/reply/agent-runner.ts:1425](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/auto-reply/reply/agent-runner.ts#L1425)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Diagnostics are separate from normal file logs:** The docs say diagnostics ‘do not replace logs’ and are emitted in-process; `emitDiagnosticEvent` only dispatches listeners, and the only repo listeners on `main` are the stability recorder and the optional `diagnostics-otel` exporter.
  - file: [docs/logging.md:176](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/docs/logging.md#L176)
