## Summary

Keep open. Current `main` still treats `maxMissedJobsPerRestart` and `missedJobStaggerMs` as internal `CronServiceDeps` knobs only; they are used by startup catch-up logic, but they are not part of the public `CronConfig` type, the strict config schema, or the documented cron config surface, so they still cannot be set from `openclaw.json`. The issue's secondary `runningAtMs` restart-loop concern is partly addressed on `main`, but the requested config exposure itself is not implemented.
