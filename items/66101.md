## Summary

Keep this PR open. Current main still does not implement the proposed main-specific host-level OpenClaw self-inspection boundary, and the PR remains a plausible but security-sensitive implementation candidate with unresolved bypass concerns.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to decide whether main should receive this narrow self-inspection exception, then revise the implementation to use parser/segment-based command detection, handle OpenClaw binary paths cross-platform, reject all mutating doctor aliases and flag forms, block mixed shell and pipeline forms, and add focused regression tests for those bypass cases before merge.
