## Summary

Keep #70944 open. Current main still lacks Discord identity-link resolution into a canonical trusted sender principal and does not render `sender_principal` in inbound metadata. The PR is recent, focused, and the provided review thread shows its bot-raised PluralKit/import concerns were addressed in follow-up commits, so there is no strong conservative close reason.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best path is to land or revise the focused Discord implementation: resolve canonical principals from `session.identityLinks`, pass `TrustedSenderPrincipal` through normal messages, components, and native commands, render `sender_principal` in trusted inbound metadata, and include the accompanying regression tests plus any required SDK contract documentation.
