## Summary

Keep PR #70944 open. Current main does not implement the PR's central behavior: Discord sender identity is not resolved through `session.identityLinks`, Discord inbound/native contexts do not carry a trusted canonical principal, and the inbound meta prompt does not render `sender_principal`. The related #57709 is closed/unmerged per the provided context and does not supersede this PR because current main still lacks the shared inbound metadata field. The PR is recent, focused, has tests in its proposed diff, and the review comments in the provided context were addressed by follow-up commits, so there is no conservative close reason.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or adapt the focused implementation: expose a suitable identity-link resolver through the plugin-safe SDK surface, resolve Discord senders from `session.identityLinks` including PluralKit sender handling, thread the canonical principal through preflight/process/native contexts, render `sender_principal` in inbound metadata, and keep the proposed regression tests for Discord and inbound-meta behavior.
