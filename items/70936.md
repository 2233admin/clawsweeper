## Summary

Keep open. The PR addresses a real remaining PDF fallback bug, but its patch targets the old core `src/media/pdf-extract.ts` implementation. Current main delegates PDF extraction to the bundled `document-extract` plugin, and the active PDF.js call there still omits `standardFontDataUrl`. The useful fix idea should be retargeted to the plugin-owned implementation rather than closed as implemented.

## Best Possible Solution

Retarget the PR's package-root `standardFontDataUrl` resolution and focused regression coverage to the bundled `document-extract` plugin, where current main now owns PDF.js fallback rendering. Keep #51455 open as the canonical bug until that plugin-owned fix lands.
