## Summary

Keep open. Current main still has the underlying PDF.js standard-font gap: PDF fallback extraction now delegates through the bundled document-extract plugin, and the active plugin getDocument call still passes only data and disableWorker. PR #70936 has useful fix and test direction, and its body correctly links #51455 and replaces dirty PR #51465, but the patch needs to be retargeted from the old src/media implementation shape to the current plugin-owned extractor path before merge.

## Best Possible Solution

Keep the PR open and request a retargeted patch: resolve pdfjs-dist/package.json from the document-extract plugin runtime, pass the resulting trailing-separator standard_fonts path as standardFontDataUrl in extensions/document-extract/document-extractor.ts, widen the local PdfJsModule/test mock shape, and add a focused regression assertion in extensions/document-extract/document-extractor.test.ts. The adjusted PR can remain the implementation path for open issue #51455.
