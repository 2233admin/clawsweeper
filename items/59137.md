## Summary

Keep open. Current main has adjacent session full-reindex gating fixes, but it still does not implement this PR's core rollback-recovery work: no retry-state snapshot/restore around reindex rollback, no explicit pending full-session retry state, no embedding-cache mirror back to the committed DB during safe reindex, and no focused regression coverage for those contracts.

## Best Possible Solution

Keep the PR open or ask for an updated version. The best implementation path is still a narrow memory-core rollback fix: snapshot and restore retry-relevant sync state across safe and unsafe reindex failures, keep a full-session retry pending signal until a full session pass succeeds, mirror successful embedding-cache writes to the committed DB during safe temp-DB reindex, guard older DBs that lack `embedding_cache`, and land focused regression tests for rollback retry and cache preservation.

If temp-DB work mutates those fields before a later failure, the catch path restores the original DB handle and rethrows without restoring retry-relevant in-memory state.
