## Summary

The request is a large optional orchestration architecture that current OpenClaw scope explicitly pushes out of core and into plugins/skills distributed through ClawHub.

## Close Comment

Closing after a Codex review. This proposal is for a large optional orchestration layer, and current project scope points that kind of capability away from OpenClaw core and toward external plugins/skills instead. The relevant guardrails are in `VISION.md:52-60` and `VISION.md:99-107`: core stays lean, optional capability should usually ship as plugins, new core skills that can live on ClawHub are out of scope, and heavy orchestration layers / default agent-hierarchy frameworks are explicitly listed under what OpenClaw will not merge for now. The existing extension path already covers this kind of experimentation in `docs/tools/plugin.md:11-15`, and `docs/plugins/community.md:9-16` makes ClawHub the canonical place for community plugin distribution. I also checked current `main` at `0e7250f37bf347c72790abf2a6692d5ec84e5ea1`; there is no built-in swarm/pheromone implementation here already. If someone wants to explore this idea, the maintainable path is an external plugin or skill published via ClawHub rather than a core OpenClaw feature.
