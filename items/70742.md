## Summary

Keep open. Current `main` does not already implement this PR, and the shipped tests still encode behavior that differs from the proposed unconditional timeout return. At the same time, current production call sites already pass `clearPendingOnTimeout: true` with guarded session managers, so there is not strong enough evidence to auto-close it as obsolete or non-reproducible. The latest PR discussion also says the earlier review defect was fixed and the remaining failing check was unrelated OOM noise.
