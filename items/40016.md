## Summary

Keep #40016 open. Current main still has only the lower-level Gateway `sessions.patch` label plumbing and generic `openclaw gateway call` escape hatch; it does not provide the requested first-class `openclaw sessions rename` or `openclaw sessions set-subject` command, and the `openclaw sessions` list output remains key-centric rather than showing a user-set label/name.

## Best Possible Solution

Keep the issue open. Implement a focused `openclaw sessions rename <session-key> <label>` command, with aliases such as `set-label` or `set-subject` left to maintainer product judgment. Reuse the existing Gateway `sessions.patch` label validation/RPC path instead of direct store edits, preserve channel-owned `subject` and `displayName` semantics, expose an effective label/name in `openclaw sessions` text and JSON output, and update `docs/cli/sessions.md` plus targeted CLI registration/behavior tests.
