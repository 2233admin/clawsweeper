## Summary

Keep #68955 open. Current main still has separate Telegram native slash-command handling for `/new`/`/reset` that does not invoke the regular Telegram inbound feedback machinery for ack reactions or typing indicators. The issue is recent, concrete, non-maintainer-authored, and has no protected label, so it remains actionable rather than stale or closeable.

## Best Possible Solution

Implement the feedback behavior in the Telegram native slash-command path, ideally by sharing the existing message feedback helper or passing the ack/typing dependencies into `registerTelegramNativeCommands`. Add regression coverage for `/new` or `/reset` in a Telegram DM with `reactionLevel: "ack"`, `ackReactionScope: "direct"`, and `typingMode: "instant"`, while preserving current authorization behavior so unauthorized or skipped commands do not emit feedback.
