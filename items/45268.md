## Summary

Keep open. The linked discussion and PR #45270 identify a real `/status` bug, but current `main` still contains the reported fallback in the shared status renderer: when `entry.totalTokens` is absent it synthesizes context usage from `inputTokens + outputTokens` without checking `totalTokensFresh`. Related UI/session-list paths have partial stale-token handling, but the central `/status` rendering issue is not implemented on current main.

## Evidence

- **Reported root cause still present in status renderer:** Current main still initializes `totalTokens` as `entry?.totalTokens ?? (entry?.inputTokens ?? 0) + (entry?.outputTokens ?? 0)` with no `totalTokensFresh` guard before rendering the Context line.
