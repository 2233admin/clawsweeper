## Summary

Keep this PR open. Current main still lacks the proposed OpenShell platform deployment guide and navigation entry, while the existing OpenShell docs cover using OpenShell as a sandbox backend from a host OpenClaw gateway, not running OpenClaw inside an OpenShell sandbox. The PR is docs-only and plausibly useful, but it still needs maintainer docs/security review and reconciliation with current proxy support before merge.

## Best Possible Solution

Keep this PR open for maintainer docs/security review. The best path is to reconcile the guide with docs/gateway/openshell.md and current Slack/proxy/fetch-guard behavior, remove the duplicate glossary entries, then either merge it as a dedicated Platforms deployment guide or ask the author to fold the durable operator guidance into the canonical OpenShell docs.
