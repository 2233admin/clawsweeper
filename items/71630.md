## Summary

Keep #71630 open. Current main has adjacent Web UI chat lifecycle plumbing, abort/queue controls, Talk-mode status text, and compaction/fallback indicators, but it does not implement the requested compact composer runtime status chip with In progress / Interrupted / Done semantics. The related #39127 and #52640 remain broader, open trackers rather than shipped canonical replacements for this UI-specific composer request.

## Best Possible Solution

Keep this as an open Web UI enhancement. The best path is to add a composer-owned runtime status view model and chip in the chat toolbar, backed initially by existing chatRunId/chatSending/chat event/reconnect lifecycle signals if necessary, while leaving room to consume the broader sessions.activity API tracked by #39127 when it exists. Implementation should include CSS/layout coverage, accessibility labels/tooltips, no transcript mutation, tests for active/final/aborted/reconnect states, and docs/i18n updates as needed.
