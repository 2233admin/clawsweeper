## Summary

Keep PR #69322 open. Current main still exposes `doctor.memory.status` only for the default agent, while the linked issue #69295 asks for additive per-agent `{agentId, dirty, files, chunks}` status. The PR is focused and relevant, but should address the review finding that per-agent manager acquisition errors need to be contained per agent before merge.

## Best Possible Solution

Keep the PR open and ask for a small revision: implement the additive `perAgent` response on `doctor.memory.status` using `listAgentIds`, default-first ordering, and cheap `manager.status()` reads, but wrap each per-agent manager acquisition/status read so a broken secondary agent returns an `{ agentId, dirty: false, files: 0, chunks: 0, error }` entry instead of failing the whole RPC. Keep the focused tests and protocol docs with that behavior.
