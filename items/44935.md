## Summary

Keep open. Current main has partial fallback recovery behavior, including cooldown-time primary probes and clearing transient auto-fallback overrides before later turns, but it does not implement the requested configurable background health-probe recovery model or the requested per-session sticky fallback semantics. The issue comment also splits notifications to #44941, which remains separate and open.

## Evidence

- **Current main clears auto-fallback overrides before each turn:** The reply model-selection path explicitly clears auto-failover overrides and retries the configured primary on the next turn. That is useful partial recovery behavior, but it is the opposite of the issue's requested 'existing sessions stay on fallback' stickiness.
