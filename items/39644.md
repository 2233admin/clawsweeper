## Summary

Keep PR #39644 open. Current main still advertises and generates PowerShell completions, but the install path still rejects PowerShell after cache validation, writes only POSIX `source` lines, and prints a POSIX reload hint. The Windows time-format path also still hard-codes `powershell` instead of using the existing resolver that prefers PowerShell 7. The PR is mechanically stale because install logic now lives in `src/cli/completion-runtime.ts`, but the reported bugs remain present on current main.

## Best Possible Solution

Keep PR #39644 open, or supersede it only with a landed port. The fix should update `src/cli/completion-runtime.ts` to install PowerShell completion via `. "..."`, emit a PowerShell-safe reload hint, update `src/agents/date-time.ts` to call `resolvePowerShellPath()` with non-interactive flags, and add focused regression tests for PowerShell profile installation, install detection, reload hints, and Windows date-time executable selection.
