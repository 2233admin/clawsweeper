## Summary

Keep open. Current main still has useful partial prompt-injection hardening for web/external content, inbound metadata, special-token sanitization, and owner-based tool policy, but it does not implement the central request: a shared trust-boundary contract for non-owner current message bodies and generic/open-world tool results, plus stable system-prompt guidance for those boundaries. The linked implementation PR #62973 is closed unmerged.

## Best Possible Solution

Keep this issue open and implement a core-owned, tested trust-boundary strategy: add stable pre-cache system-prompt guidance, wrap or otherwise label non-owner current message bodies during prompt assembly, classify/wrap open-world tool results at a shared transport or tool-result boundary, preserve the existing external-content sanitizer behavior, and add regression coverage across owner/non-owner ingress plus OpenAI, Anthropic, and Google payload shapes.
