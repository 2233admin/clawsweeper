## Summary

The report is valid ClawHub CLI/plugin-catalog feedback, but the requested `clawhub search/inspect/explore --type plugin` behavior is not implemented in this repository because `openclaw/openclaw` only ships the `openclaw` CLI. Current main already separates native OpenClaw flows: `openclaw skills search/install/update` for ClawHub skills and `openclaw plugins install clawhub:<package>` for ClawHub plugin packages. The missing plugin discovery/inspection commands belong in the separate ClawHub CLI/API/site surface rather than in the OpenClaw source repository.

## Best Possible Solution

Keep OpenClaw's shipped native flows as-is, and track the real fix in the ClawHub CLI/API/site codebase: add plugin package search/detail/explore commands or a type filter using the existing package catalog endpoints, then update the public ClawHub docs after that ships.
