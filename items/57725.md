## Summary

Keep open. Current main still contains the websocket-connect-failed -> HTTP fallback path for embedded OpenAI Responses runs, and the only clearly shipped mitigation in the repo is for non-public/custom OpenAI base URLs. The issue body does not establish that configuration, so there is not strong evidence to close this as already fixed or obsolete.

## Evidence

- **Current main still falls back on websocket connect failure:** `createOpenAIWebSocketStreamFn` still attempts a websocket connect and, on connection error, logs `WebSocket connect failed ... falling back to HTTP` before switching transports. That means the reported behavior is still a live code path on current `main`, not something clearly removed.
