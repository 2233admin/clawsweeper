## Summary

Keep open. Current main still schedules the gateway model-pricing refresh for normal startup, still fetches OpenRouter and LiteLLM pricing catalogs with a 60s timeout, and still has no `models.pricing.enabled` or equivalent disable surface. The linked open PR #68651 is a direct implementation candidate with closing syntax, so this issue should remain open until that PR is reviewed and merged, replaced, or closed.

## Best Possible Solution

Keep this issue open and review #68651 as the canonical fix candidate. A complete solution should add a validated `models.pricing.enabled` config key with default behavior preserved, skip all pricing bootstrap/catalog network work when disabled, update schema/help/labels/docs/generated baselines and changelog as appropriate, and add focused tests proving disabled pricing does not fetch OpenRouter or LiteLLM while default/enabled behavior still works.
