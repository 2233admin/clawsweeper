## Summary

Keep #69132 open. Current main only changed the bundled Ollama provider from `/api/experimental/web_search` to `/api/web_search` on the configured/default Ollama host; it does not implement the local-path compatibility order or cloud `OLLAMA_API_KEY` fallback described in the issue comments and #69222. The reported Ollama 0.16.0 path can still fail because that release exposes `/api/me` but no local `/api/web_search` or `/api/experimental/web_search` route.

## Best Possible Solution

Keep #69132 open until a focused Ollama web_search fix lands. The best implementation is a tested provider contract that either follows Ollama's documented hosted `https://ollama.com/api/web_search` API with `OLLAMA_API_KEY`, or implements a deliberate compatibility order for local proxy endpoints with clear 404/auth guidance and aligned docs/tests. #69222 is the closest current fix path to review or supersede.
