## Summary

Keep #69132 open. Current main removed the hard-coded `/api/experimental/web_search` path, but it still sends exactly one request to the resolved Ollama host at `/api/web_search`; with the reporter's default/local Ollama 0.16.0 setup, that endpoint was also observed returning 404. The linked full compatibility fix path remains unmerged, so the canonical bug should not be closed yet.

## Best Possible Solution

Keep #69132 open as the canonical bug until a compatibility fix lands. The best path is to rebase or replace #69222 with a focused change in `extensions/ollama/src/web-search-provider.ts` that handles local 404s deliberately, scopes local versus cloud bearer tokens, retries `https://ollama.com/api/web_search` only under the chosen credential contract, and updates `docs/tools/ollama-search.md`, `docs/providers/ollama.md`, `CHANGELOG.md`, and `extensions/ollama/src/web-search-provider.test.ts` to cover the documented behavior.
