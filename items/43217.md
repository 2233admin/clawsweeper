## Summary

Keep open. Current main still does not implement the PR's central sanitizer change: the shared Google thinking sanitizer and wrapper only receive `modelId`, not resolved `model.reasoning`, and the generic negative `thinkingBudget` fallback still deletes the invalid budget without applying a fixed Gemini 2.5 reasoning level. The PR is stale against the current shared wrapper location and should be rebased/adapted, but it remains plausible useful bug-fix work rather than a cleanup close.

## Best Possible Solution

Keep this PR open and ask for a rebase onto the current shared Google thinking wrapper. The maintainer path is to adapt `src/plugin-sdk/provider-stream-shared.ts` so the negative-budget sanitizer can use resolved model capability/profile data, repair fixed-level Gemini 2.5 payloads with the model-family-correct `thinkingLevel` or budget mapping, add a `gemini-2.5-flash` fixed-level regression test, and update the stale Gemini 3.1-only comment.
