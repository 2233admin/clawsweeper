## Summary

Current main still contains the reported configured ACP session-key collision path for Discord threads that inherit a parent-channel binding. A same-author implementation PR, #64322, is still open and paired with this issue, so the conservative maintainer action is to keep this issue open until that PR is reviewed, merged, or replaced.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #64322. The implementation should make inherited Discord parent-channel ACP bindings produce distinct persistent configured ACP session keys per thread, add regression coverage for two thread ids under one parent resolving to different target session keys, and preserve direct Discord thread bindings plus non-Discord binding behavior.
