## Summary

Keep #72111 open. Current main still has an actionable mixed MCP config problem: a server entry with `command`, `url`, and `transport: "streamable-http"` is resolved as stdio before the HTTP transport is considered, so the reported 30000ms timeout remains plausible and is not superseded by the closed Accept-header issue #66940.

## Best Possible Solution

Keep #72111 open. Maintainers should either implement an explicit managed streamable-http child-process mode with readiness probing, HTTP connection, lifecycle cleanup, and regression coverage, or reject mixed `command` plus `url`/HTTP transport configs early with a clear diagnostic and docs/test alignment. The current silent stdio precedence should not remain the user-visible behavior.
