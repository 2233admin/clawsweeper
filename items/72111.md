## Summary

Keep #72111 open. Current main still has an actionable mixed-config MCP behavior: a server config containing both `command` and `url` with `transport: "streamable-http"` is resolved as stdio first, so OpenClaw can spawn the process and then wait on stdio instead of connecting to the HTTP endpoint. The linked Accept-header issue #66940 does not supersede this report because this one is about managed child-process HTTP transport or clearer config rejection.

## Best Possible Solution

Keep #72111 open. Maintainers should either implement an explicit managed streamable-http child-process mode with readiness probing, HTTP connection, lifecycle cleanup, and regression coverage, or reject `command` plus `url`/HTTP transport configs early with a clear diagnostic and docs/test alignment. The current silent stdio precedence should not remain the user-facing timeout path.
