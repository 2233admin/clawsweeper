## Summary

Keep #72004 open. Current main still has the Control UI chat finalization ordering the PR targets: `handleChatEvent` appends final/aborted assistant messages to `state.chatMessages` before clearing `state.chatStream`, while the chat view builds items from both committed messages and a non-null live stream. The related bug #71992 remains open in the provided context, prior PR #34164 was closed unmerged, and I found no current-main implementation or regression test for this ordering invariant.

## Best Possible Solution

Keep #72004 open as the focused candidate fix for #71992. The best maintainer path is to review and land this stream-clearing reorder, or an equivalent patch, with its regression test and normal Control UI validation; do not close it as implemented, stale, duplicate, ClawHub work, or not actionable.
