## Summary

Keep #72004 open. Current main still has the Control UI race the PR targets: the `final` and `aborted` branches append to `chatMessages` while `chatStream` is still populated, and the chat renderer includes both history messages and the live stream bubble. The earlier related PR #34164 was closed unmerged, while #71992 remains the active bug report, so this PR is not obsolete or superseded.

## Best Possible Solution

Keep the PR open and review it as the active narrow fix for #71992. The best path is to rebase if needed, land the ordering change in both `final` and `aborted`, keep or strengthen the regression test that observes `chatStream` during `chatMessages` mutation, and validate with the changed UI test lane plus a quick Control UI webchat repro.
