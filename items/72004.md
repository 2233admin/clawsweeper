## Summary

Keep PR #72004 open. Current main still has the ordering the PR is trying to fix: `handleChatEvent` appends the final or aborted assistant message to `chatMessages` before clearing `chatStream`, while the renderer builds chat items from both committed messages and the live stream. The related issue #71992 remains open, and the earlier fix attempt #34164 was closed unmerged, so this PR is not obsolete or superseded.

## Best Possible Solution

Keep #72004 open and review it as the likely focused fix for #71992. The best path is to land the stream-clearing reorder with the regression test, after normal CI or equivalent targeted validation, rather than closing it as implemented, duplicate, stale, or ClawHub work.
