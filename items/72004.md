## Summary

Keep PR #72004 open. Current main still has the Control UI final/aborted event ordering that the PR targets: `handleChatEvent` writes assistant messages into `state.chatMessages` before clearing `state.chatStream`, while the renderer can render committed messages and a non-null live stream together. The related bug #71992 is still open in the provided context, the prior equivalent PR #34164 was closed unmerged, and #72290 tracks a different duplicate user-message delivery surface.

## Best Possible Solution

Keep #72004 open and review it as the focused candidate fix for #71992. The best maintainer path is to land this stream-clearing reorder, or an equivalent patch, with the regression test after CI/manual Control UI validation. Do not close it as implemented, duplicate, stale, ClawHub work, or not actionable.
