## Summary

Keep #72363 open. Current main has meaningful reasoning-stripping paths, including tagged/native thinking suppression and a MiniMax reasoning_content mitigation, but the Telegram non-preview path still preserves default tool progress delivery instead of buffering it until the final answer. That directly conflicts with the issue's tool-intermediate complaint, so this is not safe to close as implemented.

## Best Possible Solution

Keep the issue open and fix the Telegram delivery contract: when Telegram preview streaming is truly off, buffer or suppress text-only tool progress/intermediate payloads until final delivery, while preserving required media-only payloads and exec-approval prompts. Add a Telegram regression test that simulates streamMode "off" with exec/tool progress plus a final answer and asserts only the final visible message is delivered. Separately verify GLM/Z.AI reasoning_content behavior against the current thinking/reasoning settings and add provider-specific coverage if native reasoning can still enter visible text.
