## Summary

Keep open. Current main has Telegram-side reasoning stripping/suppression for normal visible assistant text, but it still intentionally delivers default Telegram tool progress/tool-result messages when answer preview streaming is off. That directly conflicts with the reported expectation that non-streaming Telegram should only deliver the final assembled response.

## Best Possible Solution

Keep the issue open and decide the intended Telegram non-streaming contract. If the desired behavior is final-only delivery, change the Telegram dispatch path so stream-off/non-preview sessions suppress text-only tool progress/tool-result summaries before delivery, while preserving required approval/media/error payloads, and add regression tests covering GLM/MiniMax reasoning tags plus tool-call intermediates. If maintainers prefer visible tool progress in non-preview mode, update the issue with that product decision and clarify the Telegram streaming docs/config.
