## Summary

Keep open. Current main still leaves Telegram non-streaming turns with default tool-progress delivery enabled, and an open unmerged PR (#72482) explicitly proposes the fix with closing syntax. The reasoning/thinking side has partial guards, but the reported GLM/MiniMax behavior is not fully proven fixed on main.

## Best Possible Solution

Keep this issue open while maintainers review, land, or reject PR #72482. The best fix is to suppress default tool-progress delivery for Telegram non-streaming turns, update the current regression test that expects the opposite behavior, and add focused GLM/MiniMax reasoning_content coverage so only the final assembled answer is delivered when preview streaming is off.
