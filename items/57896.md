## Summary

Current `main` no longer matches the reported stale-exec-config-after-restart path: SIGUSR1 restart clears the active runtime snapshot, startup reactivates secrets/config, and chat runs rebuild tools from the current config rather than reusing a session-baked exec tool.

## Close Comment

Thanks for the report. I reviewed current `main` with Codex, and I can't match the restart-stale-exec-config path anymore.

- `src/gateway/server.impl.ts:564` wires `clearSecretsRuntimeSnapshot` into restart shutdown, and `src/gateway/server-close.ts:45` clears the active runtime snapshot before the next boot.
- `src/cli/gateway-cli/run-loop.ts:140` handles authorized `SIGUSR1` by closing the server and starting a fresh gateway iteration, so restart goes back through normal startup.
- `src/gateway/server-startup-config.ts:262` reactivates runtime secrets with `activate: true` during startup, rebuilding the runtime config from the latest config snapshot.
- `src/gateway/session-utils.ts:417` reloads config when resolving a session, and `src/agents/pi-embedded-runner/run/attempt.ts:568` recreates the tool set from the current config for each run, so exec settings are not baked once forever into a persistent Discord session.

If you still hit this on a newer build, please open a fresh issue with the exact version/commit and current restart trace.
