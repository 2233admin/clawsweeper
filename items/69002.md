## Summary

Keep #69002 open. Current main still uses the old `process.platform !== "win32"` Homebrew prompt gate, so FreeBSD and other non-Windows unsupported platforms can still get the onboarding Homebrew recommendation when a brew-backed skill is selected. The PR remains a focused fix for the still-open #68893 path, though it should also address the bot-reviewed test guard suggestion before merge.

## Best Possible Solution

Keep the PR open or land an equivalent maintainer patch: change `needsBrewPrompt` to an explicit Homebrew-supported platform allowlist (`darwin` or `linux`), add/keep the FreeBSD regression test, and update the existing positive-path test guard to the same allowlist before merge.
