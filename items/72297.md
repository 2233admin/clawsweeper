## Summary

Keep #72297 open. Current main does not implement the PR's requested name-from-add-error fallback; it deliberately avoids destructive rebinding when `qmd collection list --json` only returns names. The PR is a focused memory-core bug-fix with a regression test, but its current approach needs maintainer/security review because it would use unstructured `qmd` stderr to select a collection for removal.

## Best Possible Solution

Keep the PR open, but ask for a safer implementation: only auto-remove a conflicting collection when OpenClaw can verify the candidate name against authoritative path/pattern metadata, for example via structured `collection list --json` details or a structured per-collection get/show command. If QMD only provides name-only list output plus stderr, OpenClaw should continue warning and requiring manual intervention, with tests covering both verified and unverified cases.
