## Summary

Issue #72382 should stay open. It is a fresh, concrete report against the documented curl-bash installer path, and current main still has the relevant behavior: the installer can warn that the npm global bin directory is missing from PATH while still printing a successful install message, with post-install command verification disabled by default.

## Best Possible Solution

Keep the issue open and fix the installer behavior. The default npm install should either leave `openclaw` immediately runnable, or avoid declaring success when the command will not be available after the script exits. Add focused coverage for npm global prefix/PATH behavior and update the public install troubleshooting docs if the intended fix requires opening a new shell or reloading a profile.
