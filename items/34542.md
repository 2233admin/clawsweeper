## Summary

Keep open. Current main already has stronger runtime diagnostics for the failing sessions_spawn thread-binding cases, but this PR's remaining prompt, tool-description/schema, and public subagent docs guidance is still not fully present. The useful path is a refresh/rebase, not closure.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main. Preserve the shipped runtime diagnostics, drop obsolete docs/tools/index.md and older runtime-copy hunks, then add focused fallback guidance to buildAgentSystemPrompt, buildSubagentSystemPrompt, describeSessionsSpawnTool/createSessionsSpawnToolSchema, and the public subagent/ACP docs. Add prompt/tool-copy tests that lock in unsupported-channel fallback wording.
