## Summary

This PR is a useful device-specific integration, but current OpenClaw policy routes new optional tool/plugin capabilities to external plugins distributed through ClawHub or npm rather than bundling them into the main repo.

## Close Comment

Thanks for the contribution. Codex review against current `main` says this belongs on the external plugin path rather than as a new bundled repo plugin. `VISION.md:54-61` sets the direction that optional capability should usually ship as plugins hosted in their own repo, with a high bar for adding optional plugins to core, and `docs/plugins/building-plugins.md:11-19` plus `docs/plugins/community.md:9-16` explicitly route new tool/command/service capabilities to ClawHub or npm via `openclaw plugins install`. Current `main` also does not have a bundled `extensions/insta360` surface or `./plugin-sdk/insta360` export. Closing on scope, not usefulness: if you want to ship this, the supported path is an external OpenClaw plugin published on ClawHub or npm.
