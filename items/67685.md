## Summary

Keep open. Current `main` still renders exec/system events inline in the chat transcript, and the existing Control UI toggle only hides tool-result messages, not `system` messages. Docs/tests also still describe exec lifecycle output as system messages and only document session-setting workarounds, not a separate/collapsible system-message surface.
