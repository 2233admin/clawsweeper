## Summary

Keep open. Current main still plans channel reloads at whole-channel granularity for Telegram config changes, including `channels.telegram.accounts.<accountId>.*`, and the hot-reload handler still calls `stopChannel(name)` / `startChannel(name)` without an account id. The lower-level lifecycle already supports account ids, and the linked fixing PR #43938 is still only contextual evidence of remaining work, not an implementation on main.
