## Summary

Current main still plans account-scoped channel config changes as channel-wide hot reloads. The lower channel manager already supports account-scoped start/stop, and the same-author implementation PR #43938 remains open with closing syntax for this issue, so this should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while #43938 remains open. The best maintainer path is to review, rebase if needed, and land that PR or an equivalent gateway change that detects account-only paths under `channels.<channel>.accounts.<accountId>.*`, plans account-scoped restarts, routes them through `stopChannel(channelId, accountId)` / `startChannel(channelId, accountId)`, and preserves full-channel restarts for channel-global or mixed changes with focused planner and handler tests.
