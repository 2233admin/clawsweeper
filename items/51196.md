## Summary

Current `main` does not include this fix. Both Feishu upload helpers still create `fs.ReadStream` objects for string-path inputs and await the SDK upload without any `try/finally` or explicit stream cleanup, so the PR still appears to address a real unfixed resource-leak path.

## Evidence

- **Current main still lacks cleanup in image upload:** `uploadImageFeishu()` creates a read stream from string input and immediately awaits `client.im.image.create(...)`; there is no `finally` block or `destroy()` call on the owned stream.
