## Summary

Keep #43549 open. Current main does not appear to implement the requested Telegram persisted-state diagnosis or recovery UX: Telegram channel status only reports configuration/audit issues, while Telegram-owned JSON artifacts fail closed or log verbosely without surfacing a status warning, reset command, backup/quarantine flow, or operator remediation. The Keep My Claw ClawHub skill linked in the discussion is a useful backup safety net, but the commenter explicitly described it as not fixing the status/recovery UX requested here.

## Evidence

- **Issue asks for status detection and safe recovery of bad Telegram session JSON:** The issue body requests `openclaw status telegram` detection for oversized/malformed/unusable Telegram persisted session JSON, plus a safe reset/backup/restart recovery path. Comment 4045462621 links Keep My Claw as a general backup safety net but says it is not a fix for the detection/recovery UX.
- **Telegram status issue collector does not inspect persisted JSON state:** `collectTelegramStatusIssues` only emits warnings for unmentioned-group privacy mode, wildcard/unresolved group audit data, and unreachable groups.
