## Summary

Keep open. Current main has improved several consistency surfaces: gateway session rows now expose model-specific thinking options, WebUI and `/think` consume those options, and explicit unsupported session patches are rejected. However the issue’s requested GitHub Copilot mini-model xhigh support is still not implemented, and `/status` still reports configured/persisted thinking intent without validating or labeling the effective supported level. The related #69440/#50703 context covers adjacent xhigh persistence/startup symptoms, but it does not supersede the remaining Copilot mini support and status semantics work here.

## Best Possible Solution

Keep #59416 open. Validate current Copilot xhigh capability for `gpt-5.4-mini` and `gpt-5-mini`; if supported, add them to the Copilot provider thinking profile and cover the provider/gateway/WebUI `/think` paths with focused tests. Separately, make `/status`, TUI, and WebUI distinguish configured intent from effective supported thinking when a stale or unsupported xhigh value is present.
