## Summary

Keep this issue open. Current main still lacks a nested `agents.defaults.compaction.memoryFlush.model` config field, the memory flush plan cannot carry a model override, and the runtime still resolves the flush turn from the session run provider/model. PR #66846 is an open implementation candidate for the same request and should be reviewed or replaced before closing this issue.

## Best Possible Solution

Keep this issue open until PR #66846 is reviewed and merged, or an equivalent implementation lands on main. The implementation should add `agents.defaults.compaction.memoryFlush.model` to config types, zod/generated schema, help/labels, and docs; carry the override through the memory flush plan or runtime boundary; resolve provider/model/auth behavior consistently with `agents.defaults.compaction.model`; and add regression coverage for override and fallback behavior.
