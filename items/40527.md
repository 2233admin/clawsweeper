## Summary

Current main still leaves the reported macOS app-backed remote skill bin probe race open. The original “app does not implement system.which/system.run” diagnosis is outdated, but the gateway still starts the automatic probe during node connect before the Swift listener is active, and the remote probe still has no loopback/co-located-node skip.

## Best Possible Solution

Keep this issue open. The best fix is to defer the initial refreshRemoteNodeBins trigger until after the node connect response has been sent and the Swift listener is active, or make the Swift connect path buffer/dispatch early event frames while waiting for the connect response. Maintainers should also decide whether loopback/co-located macOS app nodes should skip remote bin probing when the gateway already has local shell access, then add regression coverage for connect-time remote-bin probing.
