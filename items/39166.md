## Summary

Keep this PR open. Current main already forwards explicit `runTimeoutSeconds` for `sessions_spawn(runtime="acp")`, but it still has no ACP config-level default for omitted ACP spawns. The linked feature request #38419 was closed stale rather than shipped, and the PR discussion shows review feedback was addressed, so the remaining work is still a plausible focused rebase candidate.

## Best Possible Solution

Keep this PR open and ask for a focused rebase onto current main. Preserve the existing explicit ACP `runTimeoutSeconds` forwarding, add `acp.defaultRunTimeoutSeconds` to the public config type, strict schema, generated schema metadata, help/labels, and docs, then resolve ACP timeout as explicit per-call value -> ACP config default -> `0`. Tests should cover omitted config default, explicit nonzero override, and explicit `0` overriding a nonzero config default across ACP runtime initialization and Gateway dispatch.
