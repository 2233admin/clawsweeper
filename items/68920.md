## Summary

Current main still lacks the requested `/v1/chat/completions` light-context/voice-mode contract. The endpoint still builds normal agent command input and calls `agentCommandFromIngress`; stable `x-openclaw-session` is also not on main, with only the full `x-openclaw-session-key` override and OpenAI `user`-based session reuse implemented. The related PR #69060 remains a partial implementation candidate for the session-warm/QMD part, not a reason to close this issue.

## Best Possible Solution

Keep this issue open. Review and either land or close #69060 for the narrow stable-session/QMD warm-cache subproblem, then add an explicit, documented HTTP prompt policy for `/v1/chat/completions` in `src/gateway/openai-http.ts`, `src/gateway/http-utils.ts`, config schema/types/help, docs, and focused tests. The default should remain the current full-agent behavior, with a reviewed opt-in path for low-latency voice/realtime callers that reduces bootstrap context, prompt sections, tools, skills, and memory behavior safely.
