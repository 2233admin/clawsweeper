## Summary

Keep open. Current `main` still short-circuits auto-reply memory flush/preflight when `reserveTokensFloor + softThresholdTokens >= contextWindowTokens`, and the shipped small-context compaction-floor cap is applied in `src/agents/pi-settings.ts`, not in memory-flush plan resolution. The draft PR discussion is also unresolved: the latest revision discussed in the PR switched from the stated clamp behavior to preserving `return false`, so this needs maintainer judgment instead of auto-close.

## Evidence
