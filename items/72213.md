## Summary

Keep PR #72213 open. Current main still has the reported Discord block-streaming display bug: tool progress deduplication only compares the newest line against the immediately previous entry, while the live callbacks can enqueue both `tool: session_status` and bare `session_status`. The PR is a focused fix with a targeted regression test and is not superseded by current main or the provided related items.

## Best Possible Solution

Keep the PR open for normal maintainer review and CI. The best product/code path is to land this focused Discord plugin fix or an equivalent change that normalizes bare tool names across all existing preview progress entries, with the proposed regression test covering the `onToolStart` plus `onItemEvent` path.
