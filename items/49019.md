## Summary

Keep open. Current `main` still implements `POST /hooks/agent` as an async trigger that returns only `{ ok, runId }`, with no request fields or response shape for waiting on and returning a deterministic result. The documented workaround gap described in the issue still exists: `sessions_send` can wait inline, but it remains denied over HTTP `/tools/invoke` by default, so `/hooks/agent` has not gained the requested machine-result mode.

## Evidence

- **`/hooks/agent` response is still async-only:** The request handler for `subPath === "agent"` normalizes the existing payload, dispatches the hook run, and always responds `200` with `{ ok: true, runId }`. There is no branch for `waitForResult`, `resultMode`, or any inline result body.
  - file: [src/gateway/server-http.ts:663](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/server-http.ts#L663)

No `waitForResult`, `resultMode`, or `announceToMain` support exists.
