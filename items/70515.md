## Summary

Keep PR #70515 open. Current main still has the reported backup-restore false-success behavior in both async and sync suspicious-read recovery paths, and the PR remains a focused, useful bug fix rather than obsolete cleanup.

## Best Possible Solution

Keep #70515 open and continue normal review/landing. The best implementation path is the PR's focused fix: capture async and sync backup copy failures, emit a failure warning instead of the success warning, set the observe audit record's valid value from actual restore success, persist nullable restore error details, and keep async/sync regression coverage for injected copy failures.
