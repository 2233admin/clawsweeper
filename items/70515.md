## Summary

Keep #70515 open. Current main still has the reported async and sync false-success paths: failed backup restore copies are swallowed, success is logged, and the observe audit record is written as valid. The PR is a focused bug fix with regression coverage, and its bot-review follow-ups were addressed in the PR discussion.

## Best Possible Solution

Keep the PR open and have maintainers review or land a rebased version of the same focused fix: capture async and sync backup restore copy errors, log a distinct failure warning, set the observe audit record validity from actual restore success, persist nullable restoreErrorCode/restoreErrorMessage fields, and keep the async plus sync regression tests.
