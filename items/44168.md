## Summary

Current `main` still lacks the proposed WebSocket first-response watchdog and still relies on the overall embedded run timeout if OpenAI WS accepts a request but never emits an early event. The PR remains relevant cleanup rather than something already implemented or obsolete.

## Evidence

- **WS stream options still have no first-response timeout knob:** `OpenAIWebSocketStreamOptions` on current `main` only exposes `managerOptions` and `signal`; there is no watchdog timeout field in the production interface.
