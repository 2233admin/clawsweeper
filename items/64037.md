## Summary

Keep open. Current main still does not expose inbound media on PluginHookAgentContext, and the before_agent_reply path can still return before normal inbound media staging/prompt media handling. The PR addresses a real plugin API gap and its latest head appears to have incorporated review feedback for mediaUrls, defensive copies, and mediaTypes wording, but it needs rebase onto the current hook-types split plus focused tests, docs, and maintainer API/security review.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, add mediaPaths, mediaUrls, and mediaTypes to PluginHookAgentContext in src/plugins/hook-types.ts, pass defensive copies from getReplyFromConfig, document the fields on the public plugin hooks page, and add focused tests for text-only turns, local media paths, URL-only attachments, mediaTypes behavior, and mutation safety before maintainer API/security review.
