## Summary

Current main still appears vulnerable to duplicate hook-injected bootstrap files accumulating for a cached session. The bootstrap snapshot cache returns the same `WorkspaceBootstrapFile[]` instance by reference, hooks receive that same array, and neither bootstrap resolution nor context assembly deduplicates by path before prompt injection.

## Risks / Open Questions

- The static evidence directly proves accumulation across repeated runs within the same cached session. The issue body also mentions gateway restarts; that exact restart path was not dynamically reproduced in this read-only review.
- The reported reproducer depends on an external skill hook, but the core framework path still lacks defensive deduplication, so similar hooks can trigger the same problem.
