## Summary

Keep open. Current `main` still routes OpenRouter Anthropic models through the OpenAI-compatible streaming transport, while the default embedded-runner LLM idle watchdog remains a 120s no-chunk timeout. Native Anthropic now requests fine-grained tool streaming, but that does not cover the reported `openrouter/anthropic/claude-opus-4.6` path, and I found no regression test or docs/source evidence proving the large silent tool-parameter reproducer succeeds by default.

## Best Possible Solution

Keep this issue open as the canonical tracker for OpenRouter/Anthropic large tool-parameter generation failures. A maintainer fix should add a synthetic regression where an OpenAI-compatible stream withholds tool-call argument chunks longer than the default idle window, then either enable supported OpenRouter/Anthropic tool-argument streaming, adjust the LLM idle policy for silent tool-argument generation, or add a tool-loop strategy that avoids requiring one large silent `Write`-style payload.
