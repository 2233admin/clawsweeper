## Summary

Keep open. Current main appears to mitigate the common built-in GitHub Copilot Claude path by marking Copilot Claude models as non-reasoning, so default/dynamic `github-copilot/claude-opus-4.7` should not enter the Anthropic thinking branch. But the reported provider-agnostic payload bug is not fixed in the transport: if a Copilot Opus 4.7 model is configured or discovered with `reasoning: true`, `output_config.effort` is still emitted with `high`/`xhigh` and there is no provider/model effort capability guard. The exact fix PRs linked from the issue were closed unmerged, and the related low/minimal clamp work remains a separate open/salvageable path in the provided ClawSweeper context.

## Best Possible Solution

Keep the issue open or fold it into a maintainer-owned provider-effort capability fix. The best code path is a small provider/model guard or capability table in `src/agents/anthropic-transport-stream.ts` that omits or clamps `output_config.effort` for `github-copilot/claude-opus-4.7`, with regression tests covering high/xhigh and the related low/minimal cases while preserving direct Anthropic Opus 4.7 behavior.
