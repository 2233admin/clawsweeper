## Summary

Keep open. Current main still renders the Control UI Agents page selector as a native change-driven `<select>`, latest release v2026.4.24 has the same implementation, and I found no source, docs, tests, changelog, or canonical duplicate evidence that the reported browser-automation click behavior has been fixed or intentionally documented as unsupported.

## Best Possible Solution

Keep this issue open until current main is verified with the supported browser automation path. If pointer-based automation still cannot operate the Agents selector, replace the native select with an accessible automation-friendly combobox/listbox or visible agent list and add focused browser/UI coverage. If maintainers decide `element.click()`/untrusted native-select popup opening is not supportable, document the supported automation path explicitly before closing with maintainer judgment.
