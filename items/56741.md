## Summary

Keep open: current `main` still routes Telegram native-command auth through `evaluateTelegramGroupPolicyAccess` with `useTopicAndGroupOverrides: false`, and the resolver explicitly uses that flag to skip `topicConfig.groupPolicy` and `groupConfig.groupPolicy`. The same gap is still present in the latest release tag, so this PR addresses a live, unshipped bug rather than an already-implemented fix.
