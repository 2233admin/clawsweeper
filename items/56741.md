## Summary

Keep open. Current `main` still runs Telegram native command auth with `useTopicAndGroupOverrides: false`, while the regular message path uses `true`. The policy resolver only consults `topicConfig.groupPolicy` and `groupConfig.groupPolicy` when that flag is true, so the PR’s reported forum-topic override bug is not implemented on `main` and remains plausible.

## Risks / Open Questions

- Forum-topic or group-level `groupPolicy` overrides can still be ignored for native Telegram commands such as `/reset`, `/new`, and `/status` on current `main`.
- The existing test suite has a coverage gap for topic-scoped native-command auth, so this behavior could persist unnoticed without a targeted test.
