## Summary

Keep open. Current `main` does not implement `memorySearch.remote.encodingFormat` in the active memory host SDK config/request/parse path, while OpenClaw already supports `encoding_format: "base64"` on the gateway embeddings surface. The PR discussion also surfaced real integration gaps and follow-up fixes, so this is still substantive work rather than conservative cleanup.

## Evidence

- **Runtime config schema still lacks `encodingFormat`:** `MemorySearchSchema.remote` on current `main` only accepts `baseUrl`, `apiKey`, `headers`, and `batch`; there is no `encodingFormat` field to configure.
