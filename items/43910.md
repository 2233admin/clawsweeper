## Summary

Keep this PR open. Current main still does not implement configurable `memorySearch.remote.encodingFormat`, and the PR remains a relevant implementation candidate, but it is stale against the current memory-host-sdk/provider-plugin architecture and needs more work before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase the configurable `encodingFormat` design onto the current memory-host-sdk and provider-plugin architecture, update both source and package mirror paths, propagate the setting through resolved memory config, inline requests, OpenAI/Voyage batch requests, and cache keys as needed, add robust base64 decoding with validation, update docs/schema metadata, and coordinate with #41670 so maintainers can choose a minimal float compatibility fix or one broader configurable feature.
