## Summary

Keep #38687 open. Current main still creates one env-proxy fetch before the media runner resolves the configured audio/video backend baseUrl, so it cannot skip the proxy wrapper specifically for a NO_PROXY-covered self-hosted STT backend. The PR discussion shows the author addressed review feedback around private-IP bypasses, IPv6 loopback, host:port handling, wildcard ports, and lowercase no_proxy precedence, but the requested target-aware runner behavior is not implemented on main yet.

## Best Possible Solution

Keep #38687 open and adapt it to current main: resolve the media provider execution context/baseUrl before choosing fetchFn, use the shared shouldUseEnvHttpProxyForUrl/matchesNoProxy path where possible, make any CIDR NO_PROXY extension deliberate and tested, and add focused runner/proxy tests for both NO_PROXY-covered self-hosted audio backends and non-excluded backends that should still use the proxy fetch.
