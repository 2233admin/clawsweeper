## Summary

Keep open. Current main still does not wire Signal inbound messages into the shared status reaction lifecycle, while the linked fix PR #72025 is open and unmerged. The issue is concrete, recent, reproducible by code inspection, and represents an unshipped Signal behavior gap rather than cleanup-ready stale or duplicate noise.

## Best Possible Solution

Keep #72020 open until #72025 or an equivalent Signal-owned fix lands. The fix should create a Signal status reaction controller for eligible direct and group inbound messages, route dispatch lifecycle callbacks through dispatchInboundMessage, use sendReactionSignal/removeReactionSignal for the adapter, and add focused Signal monitor regression coverage for direct and group routing.
