## Summary

Current main still does not implement the requested `X-Actual-Model` transport header. The OpenAI-compatible HTTP handlers return and stream the request-side agent target model, while actual runtime model selection exists only in internal reply/session paths and is not wired into the HTTP response headers. The related #18820 fallback/session metadata work shipped, but it does not cover this header; #25043 was a broader sessions/CLI observability issue and is closed stale, so it does not supersede this specific Gateway HTTP request.

## Best Possible Solution

Keep this issue open. The core Gateway HTTP compat layer should explicitly expose the resolved runtime provider/model for `/v1/chat/completions` and `/v1/responses`, likely by threading actual model selection or `result.meta.agentMeta` through the handlers, setting `X-Actual-Model` before JSON/SSE headers are sent, and adding non-streaming, streaming, fallback, and docs coverage at `https://docs.openclaw.ai/gateway/openai-http-api`.
