## Summary

Keep this PR open. Current main still does not implement the central behavior: tool-enabled and shell-like embedded agent runs only receive a temperature when one is explicitly configured, and the extra-params/runtime-plan path has no tool-name input that could derive a default. The PR remains a plausible implementation candidate, but it should be narrowed and rebased because its stale diff also carries security-sensitive secrets/config and lockfile/package-surface changes unrelated to the agent temperature change.

## Best Possible Solution

Keep the PR open, but require a narrow rework before merge: rebase on current main, drop already-shipped or unrelated secrets/config/lockfile/test-loosening hunks, implement the dynamic temperature heuristic through the agent extra-params/runtime-plan path for normal attempts and compaction, preserve explicit user temperatures, respect provider thinking constraints, avoid accidental cache-retention changes, and add focused tests for plain chat, generic tools, shell-like tools, explicit temperature, thinking-enabled cases, and compaction.
