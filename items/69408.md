## Summary

Keep #69408 open. Current main still emits warning-level loop-detection logs for warning-only repeated tool calls, and the tool-loop path does not receive cron session target, run lifetime, or job interval metadata needed to implement the requested isolated-cron downgrade. The related ClawSweeper context around #40217 concerns stale tool history false positives, not this cron-isolated warn-vs-info logging policy.

## Best Possible Solution

Keep the issue open and implement it in the core agent/cron boundary: pass enough cron run context into loop-detection logging, downgrade only warning-level loop events for bounded isolated cron sessions to info, keep critical blockers unchanged, and add regression coverage for isolated cron plus persistent/main session behavior.
