## Summary

Keep open. Current `main` still does not wire Telegram into `sessions_spawn` thread-bound subagent spawning. The generic spawn path errors when no `subagent_spawning` hook is registered, Telegram's bundled entry does not register such hooks, and the docs still say Discord is the only supported channel for persistent thread-bound subagent sessions. Telegram has partial child-topic binding code, but it is not hooked up to the subagent spawn path.

## Risks / Open Questions

- Docs and config types mention Telegram thread-binding knobs, which may create false confidence; the issue should stay open until the subagent path is actually wired and documented.
