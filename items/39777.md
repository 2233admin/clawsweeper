## Summary

Keep open. Current main still has the reported Control UI Debug Snapshots behavior: disconnected refresh returns without user-visible feedback, the Snapshots card only reflects loading in the button label, and the debug view is not given connection state or a disabled reason to render inline.

## Best Possible Solution

Keep the issue open. The maintainable fix belongs in the Control UI Debug surface: pass connection state or a refresh-disabled reason into `renderDebug()`, show an inline loading/progress state inside the Snapshots card while `debugLoading` is true, show a disconnected callout or disabled reason when refresh cannot run, and add focused tests for `loadDebug()` and `renderDebug()` behavior. PR #39806 can be reviewed or refreshed as the existing implementation attempt.
