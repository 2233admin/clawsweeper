## Summary

Keep #69835 open. Current main still rejects explicit gateway `workspaceDir` at protocol validation and still injects missing bootstrap files as `[MISSING]` prompt entries. The PR is also not merge-ready as submitted because the provided review comments identify a concrete unguarded workspace override and a stale missing-file stability test.

## Best Possible Solution

Keep the PR open for revision. The best path is to preserve existing spawned-session workspace inheritance, add an explicit trust/authorization gate before accepting request-level `workspaceDir`, deliberately update the gateway protocol type/export surface, and update the full bootstrap missing-file test set if maintainers want missing defaults omitted instead of injected as `[MISSING]` markers.
