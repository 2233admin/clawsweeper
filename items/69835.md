## Summary

Keep PR #69835 open. Current main has not shipped the PR's two central changes: explicit gateway `workspaceDir` is still not accepted by the validated agent protocol or forwarded from the request, and missing workspace bootstrap files are still represented and injected as `[MISSING]` markers. The PR is plausible work, but the existing review comments identify real blockers: the explicit workspace override needs a trusted/internal authorization gate, and the omit-missing behavior needs the remaining stability/cache/docs expectations updated intentionally.

## Best Possible Solution

Keep the PR open for rework rather than cleanup-closing it. The best maintainer path is to split or harden the changes: add protocol support for explicit `workspaceDir` only behind a trusted internal/admin authorization path, with gateway tests for unauthorized callers; separately decide the bootstrap product behavior, then update loader/rendering, stability/cache/workspace tests, and docs consistently, or leave missing markers intact and handle the #40210 symlink case through a narrower fix.
