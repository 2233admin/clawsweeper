## Summary

Keep PR #69835 open. Current main still lacks both central PR changes: explicit gateway `workspaceDir` is not part of the validated agent protocol and is not forwarded from the request, and missing bootstrap files are still returned/injected as `[MISSING]` markers. The PR is plausible, but the existing review comments identify real blockers around authorization and stale bootstrap expectations.

## Best Possible Solution

Keep the PR open for rework. The best maintainer path is to harden explicit `workspaceDir` support behind a trusted internal/admin authorization path with negative tests for unauthorized callers, and separately decide the bootstrap product behavior before updating loader/rendering, stability/cache/workspace tests, and docs consistently.
