## Summary

#64046 should stay open. Current main has useful partial mitigations for config/API/UI redaction, but the core request is still not fully implemented: config files and backups are still plaintext JSON, the main file-log transport still writes raw log records, and current docs explicitly say log redaction does not alter file logs. The issue discussion also treats this as a valid enhancement tracker with active follow-up work, not as completed cleanup.

## Best Possible Solution

Keep #64046 open as the umbrella tracker until maintainers either split and link the remaining work or land it: a config-at-rest/backup strategy, sink-level redaction for file and diagnostic logs, and a final UI sensitive-field sweep with regression tests. Follow the active canonical log-redaction work (#67953 / #69651) for the file-log phase rather than closing this issue as complete.
