## Summary

Keep open. Current main has real progress on log redaction and Control UI masking, but the central request is broader sensitive-data handling. Plaintext config remains supported and persisted as JSON, generic config backups still copy the current config, and the docs still treat plaintext-at-rest findings as audit coverage rather than a solved encryption-at-rest design.

## Best Possible Solution

Keep this issue open as the umbrella until maintainers either split and link canonical sub-issues or land the remaining work: an at-rest secret strategy for primary config/auth artifacts/backups/service outputs, a completed redaction audit for remaining diagnostic/raw sinks, a full UI sensitive-field inventory, and regression coverage across config persistence, logs, diagnostics, API snapshots, and Control UI.
