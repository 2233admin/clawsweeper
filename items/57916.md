## Summary

Current `main` already avoids the busy-parent direct gateway path for completion announces by detecting an active requester session and steering the completion into that active embedded run instead of calling the gateway agent lane first.

## Evidence

## Close Comment

This is already implemented on current `main` based on a Codex review.

- `src/agents/subagent-announce-delivery.ts:491` now checks whether the parent/requester session has an active embedded run and, for completion announces, queues the message into that active session with `queueEmbeddedPiMessage(...)` instead of going through the direct gateway `agent` call first.
- `src/agents/subagent-announce-delivery.ts:60` derives that busy-state from the active embedded-run/session mapping, which matches the issue's reported contention case.
