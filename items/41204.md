## Summary

Keep open. Current `main` does not implement the PR's session-store queue cap or its proposed runtime orphan-sweep cleanup, and the PR discussion already identified concrete correctness/regression risks in the submitted diff. One part of the PR (installer input hardening) has been addressed separately on `main`, but not in a way that makes this whole PR obsolete.
