## Summary

Keep open. Current main still classifies Google Chat spaces using only the legacy `space.type` field, while the local type does not model `space.spaceType` or `space.singleUserBotDm`. The related fix PR #58993 is useful context, but it is open/unmerged in the provided GitHub context, so the bug remains actionable on main.

## Best Possible Solution

Keep the issue open and land a Google Chat plugin fix: add `spaceType` and `singleUserBotDm` to the inbound space type, centralize DM/space detection across legacy and modern Google Chat payloads, and add regression tests for legacy `type: "DM"`, modern `spaceType: "DIRECT_MESSAGE"`, `singleUserBotDm: true`, and `spaceType: "SPACE"` group routing with `groupPolicy: "open"` and mention gating.
