## Summary

Keep PR #64182 open. Current main still suppresses text-only tool summaries and tool-start statuses for all non-forum group sessions, while the PR targets a real remaining gap: authorized, explicitly invoked group command sessions cannot receive the same tool updates as DMs/forum topics. The PR is recent, not maintainer-authored, has no protected labels, and the provided final PR context shows it moved away from the earlier Feishu/Lark hardcoding review concern toward a provider-agnostic `CommandAuthorized` plus real control-command check with regression coverage.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a provider-agnostic core dispatch change that allows tool summaries and tool-start statuses only for non-forum group turns where `CommandAuthorized === true` and the current inbound body is an actual control command, while preserving existing suppression for normal group chatter, Slack non-direct surfaces, media-only tool-result delivery, and exec approval payloads. The PR should retain regression coverage for authorized group commands, unauthorized groups, mention-prefixed commands, and path-like slash text such as `/tmp/...`.
