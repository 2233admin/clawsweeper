## Summary

Keep this PR open. Current main still suppresses tool summaries and tool-start statuses for normal non-forum group sessions without a `CommandAuthorized` plus real-command exception, while the provided latest PR context is a focused implementation candidate for that behavior.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to rebase/adapt it to current main, preserve a provider-agnostic authorized group command predicate, apply it consistently to tool summaries and tool-start statuses, keep Slack/channel suppression semantics intact, and retain regression coverage for authorized, unauthorized, mention-prefixed, and path-like slash inputs.

## Risks / Open Questions

- The PR appears based on an older dispatch shape; current main now has a shared `shouldSendVerboseProgressMessages` gate and Slack non-direct suppression, so the patch likely needs rebase/adaptation before merge.
- The behavior intentionally expands group-visible tool updates. Final review should verify the strict `CommandAuthorized === true` plus real control-command gate remains in place, and that ordinary group chatter, unauthorized messages, and path-like slash text such as `/tmp/...` stay suppressed.
