## Summary

Keep open. The PR addresses a real Mattermost channel bug that still exists on current main: `GROUP_POLICY_NOT_ALLOWLISTED` returns before the pending-history helper is defined or called, while mention-gated drops later in the same handler do record pending history. The linked issue #57607 includes concrete repro details, and the PR discussion includes a positive Greptile review of the proposed flag-based fix. This is not implemented, duplicated, or superseded on main.

## Best Possible Solution

Keep the PR open. The best path is to add a narrow Mattermost regression test around the monitor's group allowlist path, then review and merge a fix that lets `GROUP_POLICY_NOT_ALLOWLISTED` messages reach `recordPendingHistory()` before returning without a bot response.
