## Summary

Keep PR #72069 open. Current main still does not implement claude-cli prior-session seeding for cross-provider fallback retries; the existing code only adds a generic retry marker when OpenClaw session history exists, and Claude JSONL support is limited to chat-history import/display paths. The PR remains a focused active fix for #69973, with author follow-up addressing review findings in head commit 9987e7797fc6a9247e7cbe7829c5863ef040f2b7.

## Best Possible Solution

Keep #72069 open for maintainer review. The likely correct path is core fallback-runner support: when a fallback chain started on claude-cli and retries on a different provider/runtime, read the bound Claude CLI session, extract a bounded summary plus recent-turn prelude, prepend it to the retry prompt, gate it by the original provider, and cover the behavior in src/agents/command and src/gateway tests.
