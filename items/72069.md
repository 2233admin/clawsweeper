## Summary

Keep PR #72069 open. Current main still does not implement the PR's central behavior: cross-provider fallback from claude-cli has no seeded context from Claude's JSONL history, and the existing Claude history import is only used for chat history display. The PR remains a focused active fix, with author follow-up addressing review findings.

## Best Possible Solution

Keep #72069 open for maintainer review. The likely correct path is core fallback-runner support: when a fallback chain starts on claude-cli and retries on a different provider/runtime, read the bound Claude CLI session, extract a bounded compact-summary plus recent-turn prelude from Claude JSONL, prepend it to the retry prompt, and keep regression coverage in src/agents/command and src/gateway tests.
