## Summary

Keep #72069 open. Current main already mitigates the original #69973 path by persisting successful CLI-backed turns into OpenClaw's transcript and routing fallback candidates through that same session file, but this PR's narrower behavior is not implemented on main: harvesting Claude Code JSONL summary/post-boundary context into a fallback retry prelude when OpenClaw's transcript is missing or truncated. The PR also has active correctness review comments that look fixable rather than a reason to auto-close.

## Best Possible Solution

Keep the PR open, but narrow it to a legacy/missing-transcript fallback-seed path rather than the already-solved general #69973 path. The best next revision should gate seeding by the failed source provider, preserve current OpenClaw-transcript-first behavior, handle Claude JSONL summary/compact-boundary edge cases, and add an integration regression for claude-cli billing/402 fallback to a non-CLI model when the OpenClaw transcript is empty or truncated but Claude JSONL contains prior context.
