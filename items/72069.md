## Summary

Keep PR #72069 open. Current main still does not seed cross-provider fallback prompts from Claude CLI session history; the existing Claude JSONL reader is only wired into chat-history display, while fallback retries still depend on OpenClaw session history being present.

## Best Possible Solution

Keep #72069 open for maintainer review. The likely correct path is core fallback-runner support: when a fallback chain started on `claude-cli` and retries on a different provider/runtime, read the existing Claude CLI session binding, extract a bounded summary/recent-turn prelude from Claude's JSONL, prepend it to the retry prompt, and cover that path in `src/agents/command` and `src/gateway` tests.
