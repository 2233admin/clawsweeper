## Summary

Keep open. Current main still has the reported behavior: the phantom `plugins.allow` audit only trusts installed extension directory names and channel plugin IDs, while bundled non-channel manifest plugins such as `openai`, `google`, and `acpx` exist and other validation/doctor paths already use the manifest registry. The PR is addressing a real gap, but the supplied review context also identifies merge-blocking refinements around audited `stateDir` scoping and isolated test setup.

## Best Possible Solution

Keep the PR open and revise it rather than auto-closing. The right fix is to make the phantom `plugins.allow` audit trust the full manifest registry for bundled and installed plugin IDs while preserving the audit's target `stateDir` scope, then add focused tests for bundled non-channel IDs and for the stateDir false-negative case before running the changed security test lane.
