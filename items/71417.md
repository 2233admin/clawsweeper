## Summary

Keep #71417 open. Current main still routes `openclaw agent --agent <id>` without `--session-id` to that agent's main session key, silently reuses the existing fresh session id, and has no `--new-session`/`--fresh` option or visible resume banner. The issue's `--channel` wording is partly stale because current docs/code treat `--channel` as delivery routing, but the operator-facing fresh-session problem remains actionable.

## Best Possible Solution

Keep the issue open. Add an explicit `openclaw agent` fresh-session control, such as `--new-session` or `--fresh`, wire it through the Gateway and embedded paths, and print a non-JSON stderr resume banner when an invocation without explicit `--session-id` reuses an existing session. Update `docs/cli/agent.md` and add focused tests for same-session reuse, fresh-session opt-in, JSON stdout safety, and the delivery-channel versus session-routing distinction.
