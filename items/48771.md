## Summary

Keep open. Current `main` already fixes the `schtasks` field-name half of the report, but the restart-health probe still only treats a narrow subset of WebSocket `1008` policy closes as healthy. Current code and tests still show `pairing required` / `device identity required` closes as normal live-gateway outcomes, so the false-timeout path described in #48771 is not fully resolved on `main`.

## Risks / Open Questions

- Windows `openclaw gateway restart` can still falsely report unhealthy/time out when a running local gateway rejects the probe with `1008` `pairing required` or `device identity required`.
- Current restart-health tests only cover `1008` `auth required` as a healthy signal; there is no corresponding coverage for `pairing required` or `device identity required` in `src/cli/daemon-cli/restart-health.test.ts`.
- `gh` was unauthenticated in this checkout, so this review is based on current `main` code/tests/history rather than live PR metadata.
