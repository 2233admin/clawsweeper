## Summary

Keep #70953 open. Current main still models foreground exec approvals as an async pending tool result: the exec tool registers the approval, starts a detached follow-up path that waits for the decision and runs or denies later, then returns `details.status: "approval-pending"` to the agent turn. Docs, tests, and prompt guidance all describe this current behavior rather than a pause-until-resolved flow. The commenter’s heartbeat report is also relevant and not covered by older popup-disabling or pending-list issues.

## Best Possible Solution

Keep the issue open and treat it as current exec approval flow work. The likely fix belongs in the exec approval/tool execution path: foreground interactive exec approvals should either await `exec.approval.waitDecision` through approval, denial, or timeout before the agent turn proceeds, or the agent harness should treat `approval-pending` as a hard blocked-turn state rather than ordinary tool output. Preserve explicit background/headless behavior, and separately verify the heartbeat comment’s policy-downgrade path where `security=full`/`ask=off` appears to become `allowlist`/`on-miss`.
