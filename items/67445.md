## Summary

Keep open. Current `main` does not implement mention-overrides-default-auto-responder behavior, and the linked implementation is still an open PR rather than shipped code.

## Evidence

- **Core mention-gating logic has no always-on suppression path:** `InboundMentionPolicy` on current `main` has no `suppressIfOtherAgentMentioned` field, and `resolveMentionDecisionCore` only sets `shouldSkip` when `requireMention && canDetectMention && !effectiveWasMentioned`. That means an always-on agent (`requireMention=false`) is never suppressed just because some other agent was explicitly mentioned.
  - file: [src/channels/mention-gating.ts:47](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/channels/mention-gating.ts#L47)
  - command: `nl -ba src/channels/mention-gating.ts | sed -n '40,240p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Slack path still only skips when mention is required:** In the Slack inbound path, the policy passed into `resolveInboundMentionDecision` has no override flag, and the message is dropped only under `if (isRoom && shouldRequireMention && mentionDecision.shouldSkip)`. For the issue’s example case of a channel-default always-on responder, `shouldRequireMention` is false, so explicit mention of another agent would not suppress the default responder on current `main`.
  - file: [extensions/slack/src/monitor/message-handler/prepare.ts:439](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/slack/src/monitor/message-handler/prepare.ts#L439)
  - command: `nl -ba extensions/slack/src/monitor/message-handler/prepare.ts | sed -n '430,540p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Current tests do not cover the requested override behavior:** `src/channels/mention-gating.test.ts` covers implicit mentions, command bypass, and detection-unavailable cases, but there is no test asserting that an always-on agent suppresses itself when another agent is explicitly mentioned. That matches the missing implementation in current `main`.
  - file: [src/channels/mention-gating.test.ts:74](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/channels/mention-gating.test.ts#L74)

Its patch adds `suppressIfOtherAgentMentioned` and `suppressedByOtherAgentMention`, which are absent from current `main`, so the requested behavior has not shipped yet.
