## Summary

Keep PR #69310 open. Current main has a narrow generic warning only when every reply-media item fails in `createReplyMediaPathNormalizer`, but it still silently drops partial failures and the broader normalizer catch still clears media without user-visible dropped-media metadata. Main also lacks the PR’s `droppedMedia` type/pipeline, outbound JSON/delivery preservation, and routing/followup guards needed for dropped-media-only payloads.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch that extends the current generic warning into a complete dropped-media contract: per-item sanitized diagnostics, partial-drop visibility, sanitized warning logs, payload/json/delivery preservation, route/followup/block-streaming guards, and regression tests for all current swallow paths.
