## Summary

Keep open. Current main still shows the reported timeout-propagation gaps, so this is not implemented on main, and the draft PR should not be conservatively cleaned up as resolved.

## Evidence

- **sessions_send still does not forward a target run timeout:** Current main computes `timeoutSeconds`, `timeoutMs`, and `announceTimeoutMs`, but `sendParams` does not include a `timeout` field before dispatching the nested `agent` run. That means `timeoutSeconds` only governs the caller-side wait path here, not the child run budget.
  - file: [src/agents/tools/sessions-send-tool.ts:233](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/tools/sessions-send-tool.ts#L233)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '233,289p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **A2A helper still omits target timeout and caps wait at 60s:** `runAgentStep` starts the nested `agent` call without a `timeout` param, then waits with `Math.min(params.timeoutMs, 60_000)`. The helper path described in the PR is therefore still unimplemented on main.
  - file: [src/agents/tools/agent-step.ts:31](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/tools/agent-step.ts#L31)

- **Embedded session lock acquisition still uses the lock helper default timeout:** `runEmbeddedAttempt` calls `acquireSessionWriteLock` without `timeoutMs`. The lock helper resolves omitted `timeoutMs` to `10_000`, matching the PR's cited `session file locked (timeout 10000ms)` failure mode.

- **Lock helper default is 10 seconds when timeoutMs is omitted:** `acquireSessionWriteLock` explicitly falls back to `10_000` ms when no timeout is supplied.
  - file: [src/agents/session-write-lock.ts:486](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/session-write-lock.ts#L486)

## Risks / Open Questions

- Closing this PR would drop a still-unfixed timeout propagation bug on current main.
- The draft change is not obviously merge-ready as written, because lock wait time currently sits outside the later embedded-run abort timer window.
- Main lacks regression coverage for forwarding nested agent timeouts in `sessions_send` and `runAgentStep`, so this area remains easy to regress.
