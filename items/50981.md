## Summary

Keep open. Current `main` still does not propagate `sessions_send`/A2A timeouts into nested agent runs or the session write-lock wait, so the reported gap is not implemented on `main`. The draft PR also has unresolved correctness blockers already called out in review: it forwards `timeout` as a string even though the gateway schema requires an integer, it would shorten fire-and-forget child runs from the normal agent budget to 30 seconds, and its lock-wait change would still let lock contention sit outside the actual embedded-run timeout window.
