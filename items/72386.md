## Summary

Keep #72386 open. Current main still builds the reported `openclaw.runtime-context` hidden custom message with the exact human-readable preface, queues it before the visible prompt is submitted, and only strips legacy/delimited runtime-context headers from user-facing text. This remains an active core runtime regression path, not implemented, stale, duplicate, or ClawHub-scoped.

## Best Possible Solution

Keep the issue open for a core runtime fix. The best path is to stop placing human-readable runtime-context framing in ordinary model-visible conversational content for channel envelope deltas. Use a structured/internal runtime-context representation, provider-aware/system-level wrapping, or another non-user-authored channel that weak models cannot quote as reply text. Add regression coverage for `runtime-context-prompt`, channel prompt/transcript deltas, and outbound sanitization of the exact new header.
