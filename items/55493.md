## Summary

Current `main` already has a unified background-task ledger plus Task Flow inspection surfaces that separate user-visible task/flow state from raw sessions and processes, including CLI, chat, and status views.

## Close Comment

Closing as implemented on main after a Codex review. OpenClaw already has a unified background-task ledger across ACP, subagents, cron, and CLI runs (`docs/automation/tasks.md:12`), plus Task Flow as the task-centric layer above raw sessions/tasks (`docs/automation/taskflow.md:10`). Current main exposes the requested status surface through stored flow metadata like `status`, `goal`, `currentStep`, `blockedSummary`, `waitJson`, and timestamps (`src/tasks/task-flow-registry.types.ts:14`), the `openclaw tasks flow show` inspection view with owner/current step/state/updatedAt/linked tasks (`src/commands/flows.ts:185`), and chat/status dashboards via `/tasks` and `/status` (`src/auto-reply/reply/commands-tasks.ts:81`, `src/status/status-text.ts:85`). The changelog shows this shipped in the Task Flow rollout in `v2026.4.2` (`CHANGELOG.md:1542`) with the chat-native `/tasks` board added in `v2026.4.1` (`CHANGELOG.md:1637`).
