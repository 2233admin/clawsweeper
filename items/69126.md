## Summary

Keep #69126 open. Current main has global compaction tuning under `agents.defaults.compaction`, but it does not implement the requested `agents.defaults.sessionCompactionLimit` key or any `session_status` `compactionLimit` parameter. The issue is recent, non-maintainer-authored, unprotected, and no canonical duplicate was present in the provided related-item context.

## Best Possible Solution

Keep the issue open and have maintainers define the intended semantics. If this is a real new global default, implement it through the typed config, zod/generated schema/help, docs, and the compaction/runtime path that creates or refreshes session state; add focused config-validation and runner/status tests. If existing `agents.defaults.compaction.reserveTokens` or `keepRecentTokens` is considered the supported answer, update the issue/docs with that mapping rather than auto-closing this request.
