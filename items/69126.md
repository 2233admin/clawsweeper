## Summary

Keep #69126 open. Current main has global token-based compaction knobs under `agents.defaults.contextTokens` and `agents.defaults.compaction`, but it does not implement the requested `agents.defaults.sessionCompactionLimit`, nor does the current `session_status` tool expose a `compactionLimit` argument to inherit from a global default. The related items are analogous global-default requests, not a canonical replacement for this one.

## Best Possible Solution

Keep the issue open as a concrete core config/runtime feature request. The best path is to clarify the intended semantics and likely place the new control under the existing `agents.defaults.compaction` namespace if it is a compaction policy knob; then update the config type, zod/generated schema, docs, session creation or run setup path, and tests so new channel sessions inherit the default while explicit per-session overrides continue to win.
