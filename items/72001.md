## Summary

Keep PR #72001 open. Current main still has the Gmail setup wizard writing the `gmail` preset without `hooks.allowedSessionKeyPrefixes`, while the bundled Gmail preset still uses a templated `sessionKey` that current gateway validation requires to be bounded. The PR is not implemented or superseded on main, but it likely needs revision before landing because runtime/docs also require `hooks.allowRequestSessionKey=true` for templated mapping session keys and the existing review flagged a default-session-key edge case.

## Best Possible Solution

Keep the PR open and ask for a small revision rather than closing it. The best fix is for the Gmail wizard to emit a complete runtime-valid session policy for the templated Gmail preset, likely `allowRequestSessionKey: true` plus narrow preserved `allowedSessionKeyPrefixes`, while handling existing `defaultSessionKey` and static-shadow mappings without making previously valid configs fail. Add regression coverage for `resolveHooksConfig` and actual `/hooks/gmail` dispatch, including the default-session-key edge case raised in review.
