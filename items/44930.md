## Summary

Keep open. Current main has Telegram polling offset work, but it implements at-most-once accepted-update persistence rather than a clear fix for the reported silent loss of in-flight Telegram messages during long processing/restart windows. I did not find a canonical duplicate or a source-backed proof that #44930's central user problem is solved.

## Evidence
