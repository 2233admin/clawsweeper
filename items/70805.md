## Summary

Keep #70805 open. Current main still skips clean runtime-postbuild sync under `OPENCLAW_WATCH_MODE=1`, and the current tests explicitly preserve that behavior. Main has an adjacent packaged-install/plugin-SDK alias repair (#70852), but it does not implement this PR's watch-mode freshness checks for missing runtime-postbuild outputs or unbuilt bundled-plugin entry changes.

## Best Possible Solution

Keep the PR open or supersede it with a smaller maintainer patch that implements the same core behavior: in `scripts/run-node.mjs`, allow watch mode to verify required runtime-postbuild outputs before launching, rebuild when bundled plugin package entries point at unbuilt outputs, preserve frozen dist package metadata only when safe, and keep focused regression coverage in `src/infra/run-node.test.ts`, `test/scripts/stage-bundled-plugin-runtime-deps.test.ts`, and `src/plugins/copy-bundled-plugin-metadata.test.ts`.
