## Summary

Keep #70805 open. Current main still has the older clean-watch behavior: when the build is clean and `OPENCLAW_WATCH_MODE=1`, `run-node` skips the runtime-postbuild requirement check before launching, and the requirement resolver still only checks stamps, git state, and mtimes rather than required staged outputs. The PR has a concrete repro/log in its body and follow-up commits in the provided discussion; I did not find an implementation or canonical superseding item on current main.

## Best Possible Solution

Keep #70805 open. The best path is to rebase or recreate the equivalent focused fix on current main: validate required runtime-postbuild outputs before `gateway:watch` launches, preserve last-built package metadata when dirty package entries point at unbuilt outputs, detect staged runtime dependency drift, and keep targeted regression coverage in the run-node/runtime-postbuild seam.
