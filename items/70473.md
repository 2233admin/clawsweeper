## Summary

Keep open. Current main still has the narrow overflow-token extractor and only forwards `currentTokenCount` when that extractor returns a value. Main recognizes Kimi/additive overflow text as overflow, but it does not extract the observed budget from those formats or apply the PR's synthetic over-budget fallback for unparseable confirmed overflow. The related #70479/#70472 work is explicitly separate session-preservation follow-up, not a superseding implementation of this PR.

## Best Possible Solution

Keep #70473 open for maintainer review or landing after validating the fallback semantics. The narrow extractor/forwarding gap remains on current main, while the later session-rotation/orphaning problem should continue to be handled separately by #70479/#70472.
