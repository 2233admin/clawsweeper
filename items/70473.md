## Summary

Keep open. Current main only extracts three older overflow token-count shapes and still passes `currentTokenCount` into overflow compaction only when that extraction succeeds. The PR's central changes remain absent: MiniMax-style `prompt is too long: N, model maximum...`, `requested: N`, additive `A + B > limit` parsing, and the synthetic `ctxInfo.tokens + 1` fallback for confirmed but unparseable overflow messages. Related PR #70479 is explicitly separate and addresses later session preservation after failed recovery, not this overflow-budget extraction path.

## Best Possible Solution

Keep #70473 open or land an equivalent targeted fix in the embedded runner/overflow extractor. The best implementation path is to broaden `extractObservedOverflowTokenCount`, pass a synthetic minimally over-budget count for confirmed unparseable overflow, keep the docs update, and retain the regression tests. Treat #70479 as the separate session-preservation follow-up, not as a replacement for this PR.
