## Summary

Keep open. Current main still emits `reason: "target-none"` when heartbeat delivery target resolves to `none`, and the tests still assert that exact reason for both explicit `target: "none"` and the unset/default heartbeat target case. The related PR #46870 was closed unmerged, so it does not supersede or implement the issue on main.
