## Summary

Keep open. Current `main` still boots Telegram native approval handling per account, and the Telegram request-account filter does not consult `approvals.exec.targets[].accountId`, so the reported multi-account fan-out remains plausible and is also explicitly confirmed in the issue discussion.

## Evidence

- **Telegram native approvals are started per account:** The Telegram monitor registers a native approval runtime context for each configured Telegram account whose exec approvals are enabled, so multiple account-specific handlers can be active at once.
  - file: [extensions/telegram/src/monitor.ts:133](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/telegram/src/monitor.ts#L133)
  - command: `nl -ba extensions/telegram/src/monitor.ts | sed -n '131,172p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Each runtime handler is bound to one accountId:** Approval bootstrap constructs a separate native approval handler with the supplied `accountId`, which matches the issue's description that parallel account handlers can each evaluate the same request.
  - file: [src/infra/approval-handler-bootstrap.ts:64](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/infra/approval-handler-bootstrap.ts#L64)
  - command: `nl -ba src/infra/approval-handler-bootstrap.ts | sed -n '64,83p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Telegram request matching does not read explicit target accountIds:** `matchesTelegramRequestAccount` only checks turn-source/session binding via `resolveApprovalRequestChannelAccountId(...)` and an eligible-account-count fallback. In this code path it does not inspect `approvals.exec.targets[].accountId`, so explicit target scoping is not enforced here.
  - file: [extensions/telegram/src/exec-approvals.ts:109](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/telegram/src/exec-approvals.ts#L109)

- **Target accountId filtering exists in generic target-recipient logic, but not in the Telegram native handling path above:** The shared approval client helper rejects mismatched target accountIds when checking target recipients, which supports the report that the remaining gap is specifically in Telegram native approval delivery/account selection rather than target config parsing.
  - file: [src/plugin-sdk/approval-client-helpers.ts:72](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/plugin-sdk/approval-client-helpers.ts#L72)
