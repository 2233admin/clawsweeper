## Summary

Keep #72076 open. The PR should not be auto-closed as ClawHub or duplicate because a maintainer already said a simple plugin is not sufficient and that OpenClaw is looking at a computer-use SDK plus security/permissions wiring. Current main has generic plugin tool and approval surfaces, but no dedicated computer-use extension contract; the submitted PR also has concrete review blockers, so it needs maintainer/API design rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open or convert it into a focused design discussion for a computer-use extension SDK: define permission prompts, approval policy, OS/TCC expectations, trusted driver boundaries, tool result/screenshot handling, and how cross-harness tools should register. After that API is agreed, either update this PR to target the new contract or publish the cua-driver integration as an external ClawHub/plugin package using the approved surface.
