## Summary

Keep PR #72076 open. Current main only ships the Codex-specific Computer Use setup from #71842, while this PR is proposing a cross-harness macOS desktop automation plugin. The PR discussion also has an explicit maintainer signal that a simple plugin is not sufficient because OpenClaw still needs a computer-use SDK plus security/permission wiring, so this is not a conservative auto-close as implemented, duplicate, or ClawHub-only cleanup.

## Best Possible Solution

Keep the PR open for maintainer product judgment. The best path is to use this PR and #1946 as design input for a computer-use extension SDK and permission/security model, then decide whether cua-driver belongs as an external ClawHub/plugin implementation using that API or as a vetted bundled plugin. The already-merged #71842 should remain the Codex-specific setup path, not the canonical answer for all harnesses.
