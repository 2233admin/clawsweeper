## Summary

Keep open. This is an active contributor PR for cross-harness macOS computer automation, and it has explicit maintainer discussion saying a simple plugin is not sufficient because OpenClaw needs a computer-use SDK plus security and permission wiring. Current main has a Codex-only Computer Use setup path, but it intentionally delegates desktop control to Codex app-server and does not implement the proposed `extensions/computer` cua-driver plugin or the broader cross-harness contract.

## Best Possible Solution

Keep the PR open or convert it into a focused design/API discussion. Maintainers should first define the computer-use extension SDK, permission prompts, capability boundaries, auditability expectations, and harness integration contract; then decide whether cua-driver belongs as an external ClawHub plugin, a sample plugin, or a bundled plugin once that security model exists.
