## Summary

Keep this issue open. Current main still scopes the `message` action enum and plugin-provided schema contributions, but the shared core schema groups are still added to every generated message tool schema regardless of the resolved action set.

## Best Possible Solution

Implement action-aware shared schema trimming in `src/agents/tools/message-tool.ts`: pass the resolved action set into `buildMessageToolSchemaProps()`, include shared core groups only when an active action can use them, keep always-needed routing/send/reaction/fetch fields stable, preserve cross-channel behavior, and add focused tests proving minimal send/react configurations omit unsupported poll/sticker/thread/event/moderation/gateway/channel-management/presence fields while supported fields remain.
