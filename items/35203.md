## Summary

Keep open. Current main has isolated/nested subagents, session visibility controls, usage/cost reporting, and plugin runtime seams, but it does not implement the RFC's central capabilities: learned capability profiling, a shared blackboard, layered memory visibility/promotion, or token budget enforcement. The issue is also actively discussed and only partially overlaps scoped related work, so closing as implemented, duplicate, or ClawHub would be premature.

## Best Possible Solution

Keep this open as an RFC until a maintainer explicitly scopes it. The practical path is to split it into narrower items: capability routing/metrics, blackboard or meeting-artifact search, memory visibility/promotion, and budget enforcement. Plugin-first experiments can use `api.runtime.subagent` and `api.runtime.taskFlow`; any missing extension hooks should become focused plugin API issues.
