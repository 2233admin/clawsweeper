## Summary

Keep open. Current `main` still overwrites `chatMessages` in `loadChatHistory()` and still calls `loadChatHistory()` after terminal chat events, so the optimistic-message-loss surface is not clearly gone. Separately, the PR discussion already identified unresolved correctness bugs in the proposed pending-message merge.
