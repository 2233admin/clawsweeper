## Summary

Current `main` does not carry this fix. The status memory probe still threads only the resolved config through the memory-manager path, while the codebase separately models `sourceConfig`/`activationSourceConfig` for read-only plugin decisions. That means the reported allowlist-context loss is still plausible on `main`, so this PR should stay open for maintainer review rather than be auto-closed.
