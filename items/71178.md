## Summary

Keep #71178 open. Current main still lacks the `update.run` restart-continuation producer path: the protocol schema does not accept `continuationMessage`, `update.run` writes a restart sentinel without `continuation`, the agent gateway tool does not forward `continuationMessage` for `update.run`, and tests still assert that update sentinels have no continuation. The linked PR #71573 is the direct fix but is still separate from current main.

## Best Possible Solution

Keep #71178 open. The best next product/code path is to land or otherwise implement the #71573-style fix: add `continuationMessage` to `UpdateRunParamsSchema`, parse and forward it through restart request handling and the agent gateway tool, build a `buildRestartSuccessContinuation(...)` payload for successful session-scoped `update.run`, and update the current tests that assert `continuation` is absent. After that lands, maintainers should decide whether graceful drain/fallback interruption notices and post-reset bridge behavior stay here or become narrower follow-up issues.
