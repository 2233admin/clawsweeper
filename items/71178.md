## Summary

Keep open. Current main still lacks the `update.run` restart-continuation producer path called out in the maintainer discussion, and the linked fix PR #71573 is provided as open/unmerged rather than shipped on main.

## Best Possible Solution

Keep #71178 open until #71573 or an equivalent fix lands on main. The concrete path is to add `continuationMessage` to the `update.run` protocol params, parse it through restart request handling, build a restart success continuation in `src/gateway/server-methods/update.ts` when an agent session update succeeds, forward it from `src/agents/tools/gateway-tool.ts`, and update the tests that currently assert `continuation` is undefined. After that lands, maintainers should decide whether the graceful-drain and post-system-prompt-reset bridge pieces remain in this issue or move to narrower follow-ups.
