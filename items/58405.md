## Summary

Keep PR #58405 open. Current main still does not implement a per-request `skills` override for `POST /v1/responses`: the strict request schema omits `skills`, the OpenResponses HTTP runner does not parse or pass a skills override, `AgentCommandOpts` has no override field, and agent skill snapshots are still derived from agent config/session state only. The PR is a focused additive feature with useful review context, not conservative cleanup material.

## Best Possible Solution

Keep the PR open. The best path is to rebase/review it as an additive feature: add `skills` to the strict OpenResponses schema, thread it through `openresponses-http.ts` into `AgentCommandOpts`, build an ephemeral `buildWorkspaceSkillSnapshot` using `normalizeSkillFilter`, avoid persisting override snapshots to session state, and add focused schema/HTTP/agent tests plus docs.
