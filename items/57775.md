## Summary

Keep #57775 open. Current main still has the reported mismatch: the headless node host declares and handles `system.execApprovals.get/set`, but the gateway filters advertised node commands through platform defaults that do not include those command IDs for Windows, so `nodes describe` and the Control UI can still omit them while node-targeted approvals RPCs continue to work.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the gateway node command advertisement contract: include `NODE_EXEC_APPROVALS_COMMANDS` in the default allowlist for platforms whose node hosts implement them, or otherwise special-case these node approvals commands so they survive reconciliation. Add regression coverage for a Windows/`win32` headless node connect/list/describe path and for the Control UI target filtering so a node that declares `system.execApprovals.get/set` appears as an exec-approvals-capable target.
