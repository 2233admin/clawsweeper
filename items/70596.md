## Summary

Keep PR #70596 open. Current main still lacks builtin explicit-local memory prewarming, and the related #70460 discussion points to this PR as the focused remaining work rather than evidence that the behavior shipped. The PR is plausible, but not safely obsolete or ready for cleanup closure because the current gateway startup scheduler is still QMD-only and the unresolved review risk about poisoning the cached builtin manager is source-backed.

## Best Possible Solution

Keep the PR open and request changes. The best path is a narrow builtin explicit-local prewarm that is actually scheduled from post-attach startup, avoids `auto` and remote provider side effects, does not poison the cached default manager on transient local initialization failure, and adds regression coverage for scheduler gating, explicit-local success, non-local skip, and failure/retry behavior. Split or justify the unrelated provider-runtime test-support change.
