## Summary

Keep PR #70596 open. Current main does not implement the requested builtin explicit-local memory prewarm, and the PR is not safely obsolete or superseded. The remaining work belongs in core gateway/memory startup, but the current PR needs changes before it is a complete fix because the production post-attach gate still only starts the memory startup path for QMD, and the unresolved review comment is supported by the current manager caching/provider-init code.

## Best Possible Solution

Keep the PR open and request changes. The best fix is a core gateway/memory startup change that invokes startup work for builtin memory only when an agent has explicit `memorySearch.provider = "local"`, keeps `auto` and remote providers skipped, and avoids poisoning the cached default manager on transient local provider initialization failure. Add regression coverage for the production post-attach gate, explicit-local builtin prewarm, non-local skip behavior, and failure/retry semantics.
