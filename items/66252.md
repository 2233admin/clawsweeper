## Summary

Keep open. Current `main` still exposes TTS and audio transcription as global config, and the per-agent schema/runtime do not show agent-level TTS/STT override support despite the linked PR discussion.

## Evidence

- **Per-agent config shape lacks TTS/STT fields:** `AgentConfig` lists per-agent model, identity, groupChat, subagents, sandbox, params, tools, and runtime overrides, but no `tts`, `stt`, `talk`, `messages`, or media-audio override fields.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/config/types.agents.ts#L70)
  - command: `nl -ba src/config/types.agents.ts | sed -n '55,150p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Validated agent entry schema also omits speech overrides:** `AgentEntrySchema` allows per-agent `model`, `heartbeat`, `identity`, `groupChat`, `sandbox`, `params`, `tools`, and `runtime`, but there is no agent-level speech/TTS/STT schema. `AgentToolsSchema` is policy-only and does not include `media` config.
  - file: [src/config/zod-schema.agent-runtime.ts:594](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/config/zod-schema.agent-runtime.ts#L594)

- **TTS runtime still resolves only global config:** The speech runtime resolves TTS from `cfg.messages?.tts` and returns that as the effective TTS config; there is no agent-id input or merge with per-agent settings here.
