## Summary

Keep this PR open. Current main supports global `session.dmScope` and newer route-binding-level `bindings[].session.dmScope`, but it still does not implement the PR's requested `agents.list[].dmScope` per-agent default. The PR remains a plausible core routing/config feature, but it needs rebasing against current binding-level overrides and an explicit precedence rule.

## Best Possible Solution

Keep this PR open and request a rebase/rework against current main. The best implementation would add `AgentConfig.dmScope`, strict zod validation, help/label/generated schema/docs, and route-resolution lookup for the selected agent while preserving binding-level overrides with explicit precedence and regression tests for per-agent override, global fallback, binding interaction, and route-cache behavior.

## Risks / Open Questions

- The PR predates current main's `bindings[].session.dmScope`; rebasing must preserve binding-level overrides and define precedence, likely binding override > agent override > global default.
