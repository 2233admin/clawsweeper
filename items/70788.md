## Summary

Keep open. Current main still generates the Windows Startup-folder fallback as a `.cmd` launcher containing `start "" /min cmd.exe /d /c ...`; the requested silent `wscript.exe`/`.vbs` companion launcher is not implemented, and the existing tests explicitly assert the old command. The related closed #57682 covers the separate `openclaw update` restart helper path, not this login-item fallback.

## Best Possible Solution

Keep #70788 open. The likely fix belongs in `src/daemon/schtasks.ts`: replace the Startup-folder fallback's visible/minimized `cmd.exe` launcher with a tested no-window Windows launcher, keep the existing `.cmd` path/name for detection, write and remove any companion file during install/uninstall, and update `src/daemon/schtasks.startup-fallback.test.ts` plus user-facing Windows gateway docs if behavior changes.
