## Summary

Keep open. Current `main` still returns from `startDocsPiClient()` without closing previously acquired caller-owned pipes on `StdoutPipe()` and `StderrPipe()` setup failures, so the reported leak is not implemented on `main`, and the PR discussion shows the initial review findings were already addressed on the PR head rather than disproving the fix.

## Evidence

- **Current main still has the leak path:** `startDocsPiClient()` acquires `stdin`, then `stdout`, then `stderr`, but on `StdoutPipe()` failure it returns immediately, and on `StderrPipe()` failure it also returns immediately. There is no cleanup before those returns on current `main`.
  - file: [scripts/docs-i18n/pi_rpc_client.go:93](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/scripts/docs-i18n/pi_rpc_client.go#L93)
  - command: `nl -ba scripts/docs-i18n/pi_rpc_client.go | sed -n '70,120p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **The code path is still live:** `NewPiTranslator()` still constructs the docs translation client via `startDocsPiClient()`, so this is not dead or orphaned code.
  - file: [scripts/docs-i18n/translator.go:47](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/scripts/docs-i18n/translator.go#L47)

  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Go stdlib behavior supports the bug report:** Go 1.24.13 `os/exec` records the returned pipe ends in `parentIOPipes`, and `Cmd.Start()` only auto-closes `parentIOPipes` from its deferred cleanup when `Start()` is actually entered. That means early returns before `Start()` do not get the automatic cleanup the PR discussion describes for `Start()` failures.
  - command: `rg -n "closeDescriptors|StdinPipe|StdoutPipe|StderrPipe" $(go env GOROOT)/src/os/exec/exec.go && nl -ba $(go env GOROOT)/src/os/exec/exec.go | sed -n '632,652p;1043,1113p'`
- **PR review feedback was already addressed on the PR head:** The provided PR context shows review comments flagging a missing `stdout.Close()` and a misleading GC cleanup comment on an earlier revision, and the follow-up head commit `3045a19bd7573b090805d3444d4373cc77909b06` is explicitly titled `fix: add missing stdout.Close() and correct GC comment`. That makes this PR still actionable rather than obsolete.
