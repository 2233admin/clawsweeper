## Summary

Keep open. Current `main` still appears vulnerable when heartbeat isolation is enabled: async exec completions are queued on the base session, but isolated heartbeat turns run on `<session>:heartbeat` and drain prompt-visible system events from that isolated session instead. The cron path was hardened by embedding event text directly into the prompt, but the exec path still tells the model the result is "shown in the system messages above," which is not true for that isolated-session flow.
