## Summary

Current `main` still restricts the default 5-minute cron stagger to exact top-of-hour schedules (`minuteField === "0"`, plus recurring hour field). Docs and tests consistently describe and assert that narrower behavior, while non-top-of-hour cron schedules are expected to use explicit `staggerMs`. The issue is therefore not implemented on `main`, but there is not strong enough evidence to auto-close it as invalid or obsolete.
