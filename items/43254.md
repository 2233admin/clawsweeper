## Summary

Keep this PR open. Current main still scopes Control UI session tokens by the exact parsed gateway URL, so localhost, 127.0.0.1, and [::1] aliases remain fragmented. The PR targets the right owner surface and addresses a plausible current Control UI auth footgun, but its loopback detection still needs a security fix before merge because host.startsWith("127.") also matches DNS hostnames such as 127.example.com and 127.0.0.1.nip.io.

## Best Possible Solution

Keep the PR open and request a small revision: normalize only `localhost` and actual numeric loopback IP literals, preferably via the shared canonical IP helper or an equivalent UI-local parser; preserve port and pathname isolation; add negative tests for `127.example.com` and `127.0.0.1.nip.io`; keep the IPv4/IPv6 alias reuse tests; then run the appropriate UI changed gate before landing.

## Risks / Open Questions

- The current PR diff can broaden token-scope sharing beyond loopback aliases because `host.startsWith("127.")` matches DNS hostnames, creating possible wrong-token reuse if a configured gateway host like `127.example.com` or `127.0.0.1.nip.io` collides with local loopback scope.
- The added tests in the PR cover positive alias reuse and port isolation, but the security-sensitive negative cases for DNS hostnames beginning with `127.` are still missing.
