## Summary

Keep PR #70422 open. Current main still hardcodes MiniMax VLM detection to `MiniMax-VL-01`, while the PR proposes the still-unshipped explicit `MiniMax-M2.7` / `MiniMax-M2.7-highspeed` VLM detection surface. The related #69691 is closed unmerged and was treated as superseded by this broader PR, not as a merged canonical fix. This needs maintainer/API validation rather than conservative cleanup closure.

## Best Possible Solution

Keep #70422 open for maintainer review. The best path is to verify MiniMax `/v1/coding_plan/vlm` endpoint semantics for explicit `MiniMax-M2.7` and `MiniMax-M2.7-highspeed`, then either merge a narrowed allowlist with aligned tests/docs and model-forwarding if supported, or intentionally document that OpenClaw routes MiniMax image understanding only through the plugin-owned `MiniMax-VL-01` path.
