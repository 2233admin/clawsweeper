## Summary

Keep PR #70422 open. Current main still hardcodes MiniMax VLM detection to `MiniMax-VL-01`, and current tests/docs continue to treat `MiniMax-M2.7` and `MiniMax-M2.7-highspeed` as text chat models whose image-understanding path routes through `MiniMax-VL-01`. The PR is recent, author is not protected, labels are not protected, and the related #69691 is closed unmerged rather than a canonical merged superseder. This needs maintainer/API validation, not conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to verify MiniMax's `/v1/coding_plan/vlm` support for explicit `MiniMax-M2.7` and `MiniMax-M2.7-highspeed`, then either merge a narrowed allowlist with tests/docs aligned to current runtime behavior, or intentionally document that OpenClaw only routes MiniMax image understanding through `MiniMax-VL-01`.
