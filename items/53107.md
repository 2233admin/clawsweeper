## Summary

Keep this issue open. Current main still lacks persisted cron job tags/categories and tag filtering across the core cron model, Gateway protocol, CLI, Control UI, macOS cron UI, docs, and agent cron tool schema. The prior ClawSweeper keep-open comment remains accurate against current main.

## Best Possible Solution

Keep this issue open. The best implementation is an additive core cron contract change: add normalized optional tags?: string[] to the cron job model and persistence path, Gateway add/update/list schemas and tag filtering, agent cron tool create/update/list semantics, CLI add/edit/list/show tag flags and tag filters, Control UI/macOS display and filtering controls, docs, and focused regression tests. Tag-based bulk enable/disable should either ship after list/update tag filtering is in place or be split into a narrower follow-up.
