## Summary

Keep this PR open. Current main still uses a direct inequality for the diffs artifact viewer token, and the PR is a narrow security hardening change that reuses an existing SDK-exported helper without adding risky infrastructure, dependency, workflow, or publishing changes.

## Best Possible Solution

Keep this PR open and review it as the focused implementation candidate. The best path is to land the narrow DiffArtifactStore.getArtifact change using safeEqualSecret from openclaw/plugin-sdk/browser-security-runtime, then run the changed diffs/plugin validation before merge.
