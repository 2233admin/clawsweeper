## Summary

Keep #64126 open. Current main still has the exact ad hoc `meta.token !== token` comparison in the diffs artifact store, while `safeEqualSecret` is an exported SDK helper with tests and matching usage across other token/secret checks. The related merged PR #58432 established the pattern but did not cover `extensions/diffs`.

## Best Possible Solution

Keep the PR open and either merge it or apply the same scoped change on main: import `safeEqualSecret` from `openclaw/plugin-sdk/browser-security-runtime` in `extensions/diffs/src/store.ts` and use it for `DiffArtifactStore.getArtifact` token comparison. A small regression/audit test or changed-lane validation would be useful before landing, but no core API work or ClawHub redirection is needed.
