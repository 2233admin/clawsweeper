## Summary

Keep open. The issue comments correctly identify a real code-path mismatch, and current main still does not make `openclaw health --json` use the same runtime-backed account snapshot as `channels status --probe --json`. For Discord, the health path can still default missing `running` to `false` and missing `tokenSource` to `none`, matching the reported stale JSON shape.

## Evidence

  - file: [src/commands/health.ts:396](https://github.com/openclaw/openclaw/blob/496d90c3b5ee849dba2f5aae3c8275236fa2ebb6/src/commands/health.ts#L396)

  - file: [src/gateway/server-methods/health.ts:13](https://github.com/openclaw/openclaw/blob/496d90c3b5ee849dba2f5aae3c8275236fa2ebb6/src/gateway/server-methods/health.ts#L13)
