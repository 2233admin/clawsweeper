## Summary

Keep #71482 open. Current main still has the reported false-orphan path: `loadSessionStore()` can return `{}` after read/parse failure, and `scrubDreamingNarrativeArtifacts()` then builds an empty reference set and can archive old dreaming transcripts. The PR is a plausible hotfix, but its exact empty-store guard is not strong enough for conservative cleanup because empty session stores are also a legitimate state and existing review comments already identified that cleanup regression. This should remain open for a narrower fix plus regression tests.

## Best Possible Solution

Keep the PR open, but ask for a tighter implementation: make the scrubber distinguish load failure from a valid empty store, either by adding a strict/session-store read result that reports parse/read errors or by checking file existence/readability before orphan archival. Add regression coverage for malformed or unreadable `sessions.json` not archiving active transcripts, and for a legitimately empty store still allowing safe orphan cleanup when appropriate.
