## Summary

Keep #71482 open. Current main still has the underlying memory-core cleanup shape: `loadSessionStore()` can silently return `{}` on read/parse failure, and `scrubDreamingNarrativeArtifacts()` then treats every old dreaming transcript absent from that empty reference set as an orphan. The PR is not implemented or superseded, but its exact empty-store guard has credible review feedback because it would also skip cleanup for legitimately empty stores.

## Best Possible Solution

Keep the PR or a successor open and tighten the implementation: make the dreaming scrubber distinguish failed/invalid/transient session-store loads from a valid empty store, skip archival only on the failed-load case, preserve orphan cleanup for genuinely empty stores, and add focused memory-core tests for both paths.

## Evidence

Current main has no guard for an empty loaded store before orphan scanning.

- A high-confidence fix needs regression coverage for both failed/corrupt store loads and legitimate empty stores.
