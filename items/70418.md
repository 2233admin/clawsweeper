## Summary

Keep open. Current `main` documents heartbeat as the existing cache keep-warm mechanism, but there is no dedicated cache-warmer subsystem or config surface, and heartbeat runs are still implemented as full heartbeat turns with prompt/delivery semantics. This proposal is therefore not implemented on `main`, not stale, and not clearly ClawHub/plugin scope.

## Evidence
