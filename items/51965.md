## Summary

Keep open. Current `main` does not implement the PR’s non-Azure multimodal onboarding change: `applyCustomApiConfig` still hard-codes non-Azure custom models to `input: ["text"]` and `reasoning: false`, and existing tests still assert that behavior. That means the underlying request is not implemented on `main`, so there is no basis to conservatively close this PR as already fixed or obsolete.
