## Summary

Keep open. Current main still has the reported inbound Ollama gap: `buildAssistantMessage` copies `toolCall.function.arguments` directly, while the existing normalization helper is only used on replay/outbound paths. PR #72102 is a focused fix with regression coverage for direct and streaming inbound string arguments. Related PR #69910 tracks the same bug, but #72102 is not clearly superseded; maintainers should choose one minimal implementation and close the duplicate/linked issue after landing.

## Best Possible Solution

Keep #72102 open for maintainer review. Land one minimal fix that normalizes `toolCall.function.arguments` in `extensions/ollama/src/stream.ts` `buildAssistantMessage` with `ensureArgsObject()`, keep focused inbound direct and streaming regression tests, and then close #69735 plus whichever duplicate PR (#69910 or #72102) is not used.
