## Summary

Keep #72101 open. Current main still has both behaviors the PR fixes: non-ACP `streamTo` is rejected before subagent spawning, and `model: "default"` is forwarded as a literal model override. The related #68397 is closed unmerged and does not cover the model-default normalization, so it does not supersede this PR.

## Best Possible Solution

Keep #72101 open, or land an equivalent consolidated agents-tool patch that ignores ACP-only `streamTo` for `runtime="subagent"`, normalizes empty/`default` model input to no override via shared helper logic, keeps ACP behavior unchanged, and updates the sessions_spawn/session_status regression tests. Consider whether to include the related `resumeSessionId` silent-drop behavior from #68397 before merging.
