## Summary

Keep #72101 open. Current main still has the reported sessions_spawn behavior: subagent calls with streamTo:"parent" are rejected before reaching the subagent path, and model:"default" is read as a literal model override instead of being normalized away. The related prior ClawSweeper context points to older streamTo-focused duplicate/superseding work, but this PR also addresses the model:"default" mismatch from #72078 and I do not have strong evidence that a canonical open item already covers both remaining behaviors.

## Best Possible Solution

Keep #72101 open for maintainer review. The best path is to land this PR or an equivalent small patch that normalizes sessions_spawn model overrides with the same default/reset semantics as session_status, ignores ACP-only streamTo for subagent runtime, and updates the focused regression tests. Only close as duplicate if maintainers confirm a canonical PR already covers both #72078 behaviors.
