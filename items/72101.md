## Summary

Keep #72101 open. Current main still has the two behaviors this PR fixes: `sessions_spawn` rejects `runtime="subagent"` calls that include ACP-only `streamTo`, and it forwards `model:"default"` as a literal model override instead of treating it as unset. The related #68397 is closed unmerged and does not cover the model-default normalization portion.

## Best Possible Solution

Keep #72101 open, or land an equivalent agents-tool patch that ignores ACP-only `streamTo` for `runtime="subagent"`, normalizes blank or `default` model input to no explicit override through shared helper logic, preserves ACP behavior, and updates the `sessions_spawn` and `session_status` regression tests. Maintainers should explicitly decide whether to include the adjacent `resumeSessionId` silent-drop behavior from #68397 before merging.
