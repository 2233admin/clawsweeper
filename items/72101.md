## Summary

Keep #72101 open. Current main still has both reported sessions_spawn problems, and the nearest overlapping PR (#68397) covers ACP-only streamTo/resumeSessionId handling but does not cover the model="default" normalization that #72101 adds.

## Best Possible Solution

Keep the PR open or fold its unique model="default" normalization into the canonical sessions_spawn fix. The best maintainer path is to land one consolidated patch that drops ACP-only fields for runtime=subagent, preserves ACP behavior, normalizes model="default" to no override, and updates sessions_spawn regression tests for both behaviors.
