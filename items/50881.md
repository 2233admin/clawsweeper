## Summary

Keep open. Current `main` only ignores Slack `already_reacted` / `no_reaction` errors in the Slack status-reaction dispatch path, while the shared Slack reaction helpers and action runtime still call the raw Slack reactions API without idempotent guards. That means I cannot say PR #50881 is already implemented on `main` well enough to close. The PR discussion also surfaced a real opts-forwarding regression and the branch contains a follow-up fix, so the PR still has review value rather than looking obsolete.
