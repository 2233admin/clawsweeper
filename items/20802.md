## Summary

Keep open. Current main still exposes llm_input, llm_output, and after_tool_call as observation-only void hooks, and the embedded runner fires llm_input/llm_output without awaiting or applying returned mutations. The PR is not implemented on main, the related focused PR #37128 was closed for workflow reasons rather than because the feature shipped, and the existing maintainer Codex review also explicitly kept this item open.

## Best Possible Solution

Keep this PR open for maintainer/product review. A fresh implementation should first decide the mutable llm_input/llm_output contract, wire the embedded runner to await and apply results, update public plugin docs and contract tests, and handle after_tool_call separately because current main intentionally dispatches it once from the subscription handler.
