## Summary

Keep open. Current main still has the old stuck-session diagnostic behavior: it warns solely from session `lastActivity`, has no explicit session-progress marker, no repeated-warning backoff state, and the dispatch/ACP paths do not mark reply/tool/status/block or ACP runtime events as diagnostic progress. The PR is recent, non-maintainer-authored, has no protected labels, replaces the auto-closed #71969, and the recorded review issue about respecting `diagnostics.enabled` was addressed in the PR context rather than superseding the remaining work.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. The best path is to retain the substantive diagnostic progress/backoff changes, drop or reconcile already-landed auxiliary churn, and ensure the final diff keeps the `diagnostics.enabled` gate plus focused regression coverage for progress events, no-progress warnings, and repeated-warning backoff.
