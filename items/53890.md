## Summary

Current main still does not implement config-level default outbound Telegram topic/thread binding per agent. Telegram topic config remains inbound/topic-routing focused, the requested `outbound` topic field would still be rejected by the strict schema, and outbound thread resolution only uses explicit params, topic-qualified targets, or current tool context. The prior ClawSweeper keep-open review remains accurate against current main, and the related items do not supersede this Telegram-specific feature request.

## Best Possible Solution

Keep this issue open. If maintainers want the feature, implement it in the Telegram plugin/config surface with typed topic/default-outbound config, zod schema and generated metadata/help/docs updates, and focused tests. A conservative design should keep the logic Telegram-owned and define precedence as explicit `threadId` or `:topic:` target first, current conversation/tool context next, then configured per-agent outbound default.
