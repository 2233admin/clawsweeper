## Summary

Keep open. Current main added `agents.defaults.systemPromptOverride` / `agents.list[].systemPromptOverride`, but that is a full prompt replacement, not the requested additive agent-level `systemPrompt` that appends to the normal OpenClaw system prompt across sessions. I did not find a config surface for `agents.defaults.systemPrompt` or per-agent additive `systemPrompt` on main.

## Evidence

- **Docs describe replacement, not append:** The new config surface is documented as `agents.defaults.systemPromptOverride`, and the docs say it will "Replace the entire OpenClaw-assembled system prompt with a fixed string" with per-agent precedence.
  - file: [docs/gateway/config-agents.md:457](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/docs/gateway/config-agents.md#L457)
  - command: `nl -ba docs/gateway/config-agents.md | sed -n '457,459p'`
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Config types only expose override keys:** Agent defaults expose `systemPromptOverride?: string`, and per-agent config exposes `systemPromptOverride?: AgentDefaultsConfig["systemPromptOverride"]`. I did not find additive `agents.defaults.systemPrompt` / `agents.list[].systemPrompt` keys in the agent config types.
  - file: [src/config/types.agent-defaults.ts:216](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/config/types.agent-defaults.ts#L216)

  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Runner resolves override or built prompt:** Embedded runs first resolve `systemPromptOverride` and otherwise build the normal prompt. That means the config path is a replacement branch, not an additive per-agent system prompt layer.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:949](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/agents/pi-embedded-runner/run/attempt.ts#L949)

That is a runtime parameter, not a persisted agent config key, and it is explicitly labeled as contextual extra text rather than agent-level config.
