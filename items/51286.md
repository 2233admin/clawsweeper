## Summary

Keep open. Current `main` still contains the buggy normalization path in `sandbox explain`, so the PR addresses a real unfixed regression rather than stale or already-landed work.

## Evidence

- **Current main still returns raw colon keys unchanged:** `normalizeExplainSessionKey` immediately returns any session containing `:`. That means `telegram:slash:<id>` is not wrapped into the agent-scoped store key the rest of the command expects.
  - file: [src/commands/sandbox-explain.ts:49](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/commands/sandbox-explain.ts#L49)
  - command: `nl -ba src/commands/sandbox-explain.ts | sed -n '37,59p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Channel inference only works for agent-scoped session keys:** `inferProviderFromSessionKey` calls `parseAgentSessionKey`, and `parseAgentSessionKey` rejects keys unless they start with `agent:` and have at least three parts. A raw `telegram:slash:200` key therefore cannot infer `telegram` on current main.
  - file: [src/sessions/session-key-utils.ts:28](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/sessions/session-key-utils.ts#L28)
  - command: `nl -ba src/sessions/session-key-utils.ts | sed -n '24,48p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Current tests do not cover the reported regression:** `src/commands/sandbox-explain.test.ts` currently has only the baseline JSON-shape and alsoAllow tests. There is no test asserting `telegram:slash:<id>` normalization or downstream `elevated.channel` / `allowedByConfig` behavior on main.
  - file: [src/commands/sandbox-explain.test.ts:16](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/commands/sandbox-explain.test.ts#L16)
  - command: `nl -ba src/commands/sandbox-explain.test.ts | sed -n '1,120p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **PR discussion improved the test to match the real bug:** The review comment correctly pointed out that a normalization-only assertion was insufficient, and the PR author followed up with commit `6bb837aeb73526ac028b29d863eaffcead8abb2c` to assert the end-to-end outcome (`elevated.channel === "telegram"` and `allowedByConfig === true`). That strengthens the PR rather than weakening it.
