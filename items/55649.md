## Summary

Current main already uses the unified presentation-based message tool for Mattermost and does not expose a Mattermost-native schema where top-level `buttons` could be required.

## Close Comment

Closing from a Codex review: current `main` already no longer exposes a Mattermost-native message-tool schema where top-level `buttons` could be required. `extensions/mattermost/src/channel.ts:102` now advertises only `actions` plus `capabilities: ["presentation"]`, and `extensions/mattermost/src/channel.test.ts:270` explicitly asserts `discovery.schema` is `undefined`. The shared message-tool schema uses `presentation.blocks[].buttons`, where `buttons` is optional (`src/agents/tools/message-tool.ts:70`), and the scoped schema test verifies there is no top-level `buttons` property (`src/agents/tools/message-tool.test.ts:446`). This same implementation is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
