## Summary

Keep open. Current main only supports reasoning visibility modes `on|off|stream`. Telegram's `stream` mode reduces clutter by putting reasoning into the live preview, but there is no evidence of routing reasoning to a different Telegram bot or alternate channel target.

## Evidence

- **Reasoning modes are limited to on/off/stream:** Agent config exposes `reasoningDefault?: "on" | "off" | "stream"`, with no alternate-destination or bot-routing option.
  - file: [src/config/types.agents.ts:86](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/config/types.agents.ts#L86)
  - command: `nl -ba src/config/types.agents.ts | sed -n '80,90p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Schema matches the same three-mode model:** Runtime schema validates `reasoningDefault` as only `on`, `off`, or `stream`. That rules out a built-in config surface for a separate reasoning bot.
  - file: [src/config/zod-schema.agent-runtime.ts:825](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/config/zod-schema.agent-runtime.ts#L825)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '818,832p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Docs describe Telegram stream as same-chat preview only:** Telegram docs say `/reasoning stream` sends reasoning to the live preview while generating, then sends the final answer without reasoning text. This addresses clutter, but not delivery to another bot.
  - file: [docs/channels/telegram.md:290](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/docs/channels/telegram.md#L290)

  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Telegram implementation binds reasoning to the current chat:** The Telegram dispatcher creates the reasoning draft stream with the current `chatId`, and the shared delivery base options also hard-code that same `chatId` for replies. There is no alternate bot/chat target in this path.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:343](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/telegram/src/bot-message-dispatch.ts#L343)
