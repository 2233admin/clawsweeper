## Summary

Keep #59294 open for maintainer judgment. Current main already covers the basic Teams thread session-key isolation path, but this PR still contains potentially useful unlanded work around thread-scoped pending history, thread-aware debouncing, mention preservation, and mixed attachment fallback. It should be rebased and split rather than closed as implemented.

## Best Possible Solution

Keep the item open, but ask for a rebase or maintainer split. Preserve the already-landed main behavior for route session keys, feedback threading, and proactive thread fallback, then extract the remaining useful fixes into narrow changes: thread-scoped pending history, thread-aware debounce keys, bot-only mention stripping with command-safe text, and a carefully targeted mixed attachment fallback. Cross-link or coordinate the proactive fallback and malformed-key cases with #59314 and #66771.
