## Summary

Keep open. Current `main` partially covers the idea with guild/channel `users` and `roles` allowlists plus per-channel `requireMention`, but it does not implement the requested `bypass`/`mention`/`deny` access model, permission shortcuts, or additive per-channel inheritance/override semantics.

## Evidence

- **Current Discord config surface is still `requireMention` + allowlists:** The resolved Discord guild/channel config types only expose `requireMention`, `users`, `roles`, and `channels`; there are no `bypass`, `mention`, `deny`, or `inherit` fields in the current main implementation.
  - file: [extensions/discord/src/monitor/allow-list.ts:26](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/discord/src/monitor/allow-list.ts#L26)
  - command: `nl -ba extensions/discord/src/monitor/allow-list.ts | sed -n '1,120p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Channel access rules override rather than merge:** `resolveDiscordMemberAccessState` picks `channelConfig.users ?? guildInfo.users` and `channelConfig.roles ?? guildInfo.roles`, so a channel-level list replaces the guild-level list instead of unioning with it. That does not match the issue's proposed additive inheritance model.
  - file: [extensions/discord/src/monitor/allow-list.ts:221](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/discord/src/monitor/allow-list.ts#L221)

  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Mention policy is still a single boolean:** Group mention handling still resolves to one boolean by checking channel `requireMention`, then guild `requireMention`, then defaulting to `true`; there is no separate bypass-vs-mention access tier.
  - file: [extensions/discord/src/group-policy.ts:93](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/discord/src/group-policy.ts#L93)

The example config shows only `requireMention`, `users`, `roles`, and `channels`.
