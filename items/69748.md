## Summary

Keep #69748 open. Current main still exposes Discord guild/channel access as `requireMention` plus `users`/`roles`, not the requested `deny`/`bypass`/`mention` lists with permission shortcuts and additive/inherit channel semantics. The related ClawSweeper context is overlapping but not canonical: #39661, #62066, and #40895 were all kept open for adjacent Discord mention-policy gaps rather than superseding this broader access-list migration.

## Best Possible Solution

Keep the issue open as a bundled Discord plugin feature request. The likely implementation belongs in `extensions/discord` plus shared config schema/types: add explicit `deny`/`bypass`/`mention` access entries with user, role, all, and selected permission references; define guild/channel merge versus `inherit: false`; migrate or repair legacy `requireMention`/`users` configs through doctor/fix with docs and focused preflight tests.
