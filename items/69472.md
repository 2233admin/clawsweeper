## Summary

Keep #69472 open. Current main still does not implement the PR's channel-agnostic suppression for a bare reasoning preamble followed by `NO_REPLY`, especially doubled forms such as `NO_REPLY.NO_REPLY`. The PR is a focused attempt to fix a real remaining leak, though its existing review comment should be addressed before merge to preserve the #19537 substantive-reply semantics.

## Best Possible Solution

Keep #69472 open as a plausible focused bug fix. The best path is to revise the preamble detector so it suppresses the reported bare reasoning-prefaced silent replies and doubled sentinels while preserving #19537 substantive replies, then add/keep targeted token tests plus a delivery-path regression test through `isSilentReplyPayloadText` or reply normalization.
