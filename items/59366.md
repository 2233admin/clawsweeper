## Summary

Keep #59366 open. Current main still only accepts `streamTo: "parent"` for ACP spawns, documents only that target, and the relay implementation still writes ACP progress into parent-session system events rather than sending directly to a bound thread. The PR is unmerged work for still-open #55574, but its current patch needs the existing P1 Slack/Mattermost thread-target review comments addressed before landing.

## Best Possible Solution

Keep the PR open. The best path is to revise #59366 or a successor PR so core `sessions_spawn` accepts `streamTo: "thread"` only with `thread: true`, resolves a concrete bound-thread delivery target for Discord/Slack/Mattermost-style channels, streams progress through `callGateway({ method: "send" })`, preserves existing `streamTo: "parent"` behavior, and updates the ACP docs plus regression tests. Once that lands, it can close #55574.
