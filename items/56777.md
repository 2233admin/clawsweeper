## Summary

Keep open. Current `main` still appears to have the reported bug: Telegram native slash-command auth resolves the topic config, but the native-command policy check explicitly ignores topic/group `groupPolicy` overrides. Regular inbound message handling uses the override-aware path, so commands and normal messages are inconsistent in forum topics.

## Evidence

- **Native command auth ignores topic/group policy overrides:** `resolveTelegramCommandAuth()` passes `topicConfig` and `groupConfig` into `evaluateTelegramGroupPolicyAccess()` but sets `useTopicAndGroupOverrides: false`, so command gating falls back to account/default policy instead of the effective topic/group policy.
  - file: [extensions/telegram/src/bot-native-commands.ts:406](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/telegram/src/bot-native-commands.ts#L406)
  - command: `nl -ba extensions/telegram/src/bot-native-commands.ts | sed -n '394,410p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Policy helper only consults topic/group overrides when enabled:** `evaluateTelegramGroupPolicyAccess()` uses `topicConfig.groupPolicy` and `groupConfig.groupPolicy` only when `useTopicAndGroupOverrides` is true; otherwise it uses the fallback account/default policy.
  - file: [extensions/telegram/src/group-access.ts:146](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/telegram/src/group-access.ts#L146)
  - command: `nl -ba extensions/telegram/src/group-access.ts | sed -n '138,153p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Normal inbound Telegram messages do use effective topic/group policy:** `shouldSkipGroupMessage()` calls the same policy helper with `useTopicAndGroupOverrides: true`, so regular forum-topic messages respect per-topic `groupPolicy` while native commands do not.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:606](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/telegram/src/bot-handlers.runtime.ts#L606)

- **Topic config is resolved for forum commands:** The native-command path resolves `topicConfig` from the forum `message_thread_id` before doing auth, so the missing behavior is not due to absent topic lookup.
  - file: [extensions/telegram/src/bot/helpers.ts:177](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/telegram/src/bot/helpers.ts#L177)
