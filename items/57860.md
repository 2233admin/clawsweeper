## Summary

Current `main` already preserves the embedded session stream function and injects the resolved runtime API key into provider-owned/custom-provider stream functions, which covers the reported `codex-lb` auth-loss path. The same implementation is present in the shipped `v2026.4.23` release.

## Close Comment

This looks fixed on current `main`, so I'm closing it as implemented.

- Codex review of `src/agents/pi-embedded-runner/run/attempt.ts:1357` shows the embedded runner now preserves the session's original base stream and rebuilds the final stream through `resolveEmbeddedAgentStreamFn(...)`; it no longer hard-overwrites the generic path with bare `streamSimple`.
- `src/agents/pi-embedded-runner/stream-resolution.ts:75` now wraps provider-owned/custom-provider stream functions and injects the resolved runtime API key (`resolvedApiKey` or `authStorage.getApiKey(...)`) into `options.apiKey`, which is the missing auth propagation described here.
- Regression coverage exists in `src/agents/pi-embedded-runner/stream-resolution.test.ts:114` and `src/agents/pi-embedded-runner/run/attempt.test.ts:490`, both asserting that provider-owned stream functions receive the runtime API key.
- The shipped `v2026.4.23` release already contains the same implementation, so this is not only fixed on unreleased `main`.
