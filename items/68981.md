## Summary

Keep #68981 open. Current main has useful partial work: session rows have labels, Control UI can edit labels, the chat dropdown prefers labels/display names, and `sessions_spawn` supports `mode`, `cleanup`, and `label`. But the central request is broader and still unresolved: there is no `/session rename` or auto-title flow, `sessions.list` has no channel/status/kind filter contract, Control UI still presents the Sessions table with only active/limit/global/unknown/search controls, and the chat session dropdown groups by agent rather than by channel/kind/status. The comment's `cleanup: "delete"` workaround is now reflected in code/tests, but default keep-mode subagent runs still do not auto-delete.

## Best Possible Solution

Keep this open and implement it as core session-management work. A good path is: add a stable rename/label command or extend existing session patch surfaces for user-facing renames; decide whether auto-title should be derived from transcript text or model-generated and persisted as `label`/`displayName`; add additive `sessions.list` filters for channel/kind/status; update the Sessions page and chat selector to expose channel/kind/status filtering, grouping, and status indicators; and separately decide whether default subagent cleanup should remain opt-in or become configurable by kind/TTL.
