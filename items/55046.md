## Summary

Current main still preserves the reported ACP/media-understanding gap. ACP dispatch invokes media understanding for inbound media, but passes only ctx and cfg; image provider execution still requires agentDir, and active-model context is still needed for native-vision skip and MiniMax text-model fallback behavior. The prior related comment is consistent with current main.

## Best Possible Solution

Keep this issue open. Fix ACP dispatch by resolving the effective ACP agent directory from the canonical session or resolved ACP agent, pass that agentDir into applyMediaUnderstanding, and pass active model context when ACP runtime/session configuration can provide it so native-vision skip and text-only fallback behavior match normal replies. Add a focused dispatch-acp.test.ts regression for an inbound image ACP turn with configured tools.media.image, asserting media understanding receives the agentDir and enriches the prompt before runTurn. Treat the Windows OPENCLAW_AGENT_DIR doctor expectation as a separate follow-up if it still reproduces after the ACP fix.
