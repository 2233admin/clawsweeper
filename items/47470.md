## Summary

Current `main` does not implement a dedicated cross-session broadcast primitive. The session tool surface still exposes only per-session send/list/history/status/spawn helpers, and `sessions_send` only targets one session at a time. `sessions_list` can find active sessions, so a caller could script a manual fan-out loop, but that is not the requested built-in `sessions_broadcast`/system-event capability.
