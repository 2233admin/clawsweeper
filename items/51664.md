## Summary

Current `main` does not show this issue as fixed. WebChat code-block copy still uses a direct `navigator.clipboard.writeText(...)` call with no fallback path, and the two linked fix PRs were closed without merge, so there is not strong evidence to close #51664.

## Evidence

- **Current WebChat copy handler still uses clipboard API only:** `handleCodeBlockCopy` reads the code from `data-code` and calls `navigator.clipboard.writeText(code).then(...)` directly. There is no fallback path if Edge rejects the Clipboard API call.
