## Summary

Keep open. Current `main` preserves env-backed references and SecretRef round-trips, but the requested first-class Control UI UX is still not implemented: SecretRef-backed fields are rendered as blank read-only inputs with a generic placeholder rather than a clear `From env`/source-aware state.

## Evidence

- **Control UI still blanks structured SecretRef fields:** `renderTextInput` treats structured SecretRefs as redacted, sets the displayed value to an empty string, and uses only a generic placeholder (`Structured value (SecretRef) - use Raw mode to edit` / `...edit the config file directly`). There is no source badge or env-var-name display.
