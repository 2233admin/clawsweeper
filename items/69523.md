## Summary

Keep open. Current `main` has some internal auto-expand plumbing for tool cards, but the Control UI still hardcodes the feature off, persists no user setting for it, and still tests collapsed-by-default behavior. This issue remains a valid, coherent enhancement request.

## Evidence

- **Control UI render path forces auto-expand off:** `renderChat(...)` is called with `autoExpandToolCalls: false`, so current main does not enable default-expanded tool cards for users.
  - file: [ui/src/ui/app-render.ts:2264](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/ui/src/ui/app-render.ts#L2264)

  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **No persisted UI setting exists for this preference:** `UiSettings` only persists `chatFocusMode`, `chatShowThinking`, and `chatShowToolCalls`; load logic restores those fields and nothing like `expandToolsByDefault`/`chatAutoExpandToolCalls`.
  - file: [ui/src/ui/storage.ts:52](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/ui/src/ui/storage.ts#L52)
