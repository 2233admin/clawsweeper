## Summary

Keep open. Current main still uses the sandbox browser socat relay plus CDP port-offset design this issue asks maintainers to evaluate. The related #62873 work hardened readiness/fail-fast behavior but explicitly left the proxy architecture in place, and current source, docs, and tests still treat `cdpSourceRange` and the relay as supported sandbox-browser behavior.

## Best Possible Solution

Keep this issue open as the sandbox browser CDP architecture tracking item unless a maintainer explicitly decides to retain the relay. A complete implementation should remove `socat` and CDP port-offset logic from `Dockerfile.sandbox-browser` and `scripts/sandbox-browser-entrypoint.sh`, update `src/agents/sandbox/browser.ts` orchestration/config hashing and `cdpSourceRange` handling, preserve equivalent readiness and access-control guarantees, and update sandbox docs plus tests.
