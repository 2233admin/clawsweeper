## Summary

Keep #63877 open. Current main still has the exact Firecrawl endpoint validation that the reporter hit: configured Firecrawl base URLs must use HTTPS and must be hosted on api.firecrawl.dev. The related discussion is useful, but the referenced fix PR #63899 was closed unmerged and PR #63941 remains open, so the issue is not implemented or safely superseded on main.

## Best Possible Solution

Keep the issue open and use the existing PR discussion as the implementation path. The likely fix belongs in the Firecrawl plugin: replace the hard `api.firecrawl.dev` allowlist with a reviewed private-network/custom-host policy, keep HTTPS required for public endpoints, preserve credential and URL sanitization, and update Firecrawl/web_fetch docs plus endpoint validation tests. PR #63941 or the earlier self-hosting PR work can be the starting point, but it still needs maintainer review around SSRF/private-network trust before the issue can close.
