## Summary

Current `main` does not implement this behavior, and the proposed PR scope misses the code path that actually drops malformed skills during discovery. This should stay open.

## Evidence

- **Malformed `SKILL.md` files are still dropped during loading:** `loadSingleSkillDirectory` returns `null` when `parseFrontmatter(raw)` throws, so malformed skills never make it into discovery. `loadSkillsFromDirSafe` then filters those `null` entries out.
  - file: [src/agents/skills/local-loader.ts:61](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/skills/local-loader.ts#L61)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Current skill/status types have no place to carry a parse error:** `SkillEntry` and `SkillStatusEntry` on `main` do not include any `parseError` field, so there is no current-main plumbing for surfacing loader/frontmatter failures.
  - file: [src/agents/skills/types.ts:76](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/skills/types.ts#L76)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **CLI output paths still only distinguish ready/disabled/blocked/needs-setup:** `formatSkillStatus`, `formatSkillInfo`, and `formatSkillsCheck` have no parse-error branch or reporting bucket on `main`, so the user-facing CLI still cannot show YAML/frontmatter parse failures.
  - file: [src/cli/skills-cli.format.ts:29](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/cli/skills-cli.format.ts#L29)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Workspace loading still falls back to empty frontmatter:** When a record lacks cached frontmatter, `loadSkillEntries` uses `readSkillFrontmatterSafe(...) ?? {}`; that path still does not surface an error message.
  - file: [src/agents/skills/workspace.ts:604](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/skills/workspace.ts#L604)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **PR scope does not touch the loader that drops malformed skills:** The provided PR metadata changes only `src/agents/skills-status.ts`, `src/agents/skills/types.ts`, `src/agents/skills/workspace.ts`, and `src/cli/skills-cli.format.ts`; it does not modify `src/agents/skills/local-loader.ts`, which is where malformed skills are currently discarded.
