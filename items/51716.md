## Summary

Keep open. Current `main` still subscribes to Signal events via `fetch`/`globalThis.fetch`, with no `node:http` fallback or Node-24-specific workaround, so there is no strong evidence that #51716 is implemented or obsolete. The Signal reconnect path only logs and retries on SSE fetch errors, docs still describe Signal as JSON-RPC + SSE on a Node 24-recommended runtime, and the changelog does not show a shipped fix for this regression.
