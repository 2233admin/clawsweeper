## Summary

Keep PR #69491 open. Current main does not implement the requested Bedrock Converse `additionalModelRequestFields.anthropic_beta` forwarding for the standard `amazon-bedrock` provider. The analogous default beta behavior exists for direct Anthropic and a separate Bedrock Mantle route, but not for the native Bedrock Converse path targeted by this PR. The PR is still useful, though it needs revision for the existing review concerns before it should land.

## Best Possible Solution

Keep the PR open and ask for a narrow revision in `extensions/amazon-bedrock/register.sync.runtime.ts`: add a Bedrock Converse payload patch for definite Anthropic Claude model IDs that merges/deduplicates existing `additionalModelRequestFields.anthropic_beta` values with the default Anthropic betas, composes cleanly with guardrail and cache-point wrappers, and adds tests for no-guardrail, guardrail, existing beta merge, and non-Anthropic/profile false-positive cases.
