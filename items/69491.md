## Summary

Keep PR #69491 open. Current main still does not forward default Anthropic beta flags through the bundled amazon-bedrock Bedrock Converse stream route, while the direct Anthropic path does preserve and merge default beta headers. The PR is focused and addresses a plausible missing provider behavior, but it should not land as-is because review comments identify real follow-up work around preserving existing `anthropic_beta` values and avoiding broader heuristic false positives for application inference profiles.

## Best Possible Solution

Keep the PR open and ask for a small revision: merge and dedupe existing `additionalModelRequestFields.anthropic_beta` values with the default Anthropic betas, restrict injection to definitive Anthropic Bedrock model IDs or profiles resolved as Claude, and add regression tests for existing beta preservation plus the application-profile false-positive case.
