## Summary

Keep PR #69491 open. Current main still does not forward default Anthropic beta flags through the Amazon Bedrock Converse stream path, while the direct Anthropic plugin already maintains those defaults. The PR addresses a real Bedrock parity gap, but it should not land as-is because review comments identify actionable correctness issues: preserving existing `anthropic_beta` values and avoiding the current broad `isAnthropicBedrockModel` ARN-name heuristic for beta injection.

## Best Possible Solution

Keep the PR open and ask for a revised Bedrock implementation on current main: inject the default Anthropic betas into `additionalModelRequestFields.anthropic_beta` only for definitive Claude Bedrock targets, merge with any existing beta array instead of replacing it, compose cleanly with guardrail and cache-point payload patches, and add focused tests for Anthropic Bedrock with and without guardrails plus pre-existing beta values.
