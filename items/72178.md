## Summary

Keep PR #72178 open. Current main still does not implement cursor-based `chat.history` pagination or the web UI load-older flow, and the related open PR #56383 is not merged or clearly canonical enough to make this PR obsolete. The PR also has unresolved review feedback about duplicate pages, so it should not merge as-is.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to coordinate with #56383, implement cursor-based `chat.history` pagination across the protocol schema, handler, generated clients, docs, and web UI load-older flow, and require a regression test for sanitized mid-page `NO_REPLY` or commentary rows so older-page cursors cannot overlap already returned messages.
