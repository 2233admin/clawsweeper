## Summary

Keep open. Current `main` does not implement this PR: Telegram DM answer lanes are still forced onto message preview transport, tests still assert that behavior, and the extra draft-cleanup/materialize-ambiguity guards from the PR are not present. A changelog entry claims a similar fix, but the checked-out code and latest release tag still contradict it, so closing as implemented would be unsafe.
