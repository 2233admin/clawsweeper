## Summary

Keep open. Current main handles Discord thread routing, thread starter context, channel topics, and optional parent transcript inheritance, but it does not automatically read or inject parent channel pinned messages into Discord thread session context, and there is no pin-inheritance config surface.

## Best Possible Solution

Keep the issue open and implement it in the bundled Discord plugin: when an inbound message is in a Discord thread, use the resolved `threadParentId` to optionally fetch the parent channel pins, bound and sanitize them as untrusted supplemental context, expose an explicit config flag such as `channels.discord.thread.inheritPins`, and add focused tests/docs for regular threads, auto-created threads, permissions/errors, and rate-limit-safe behavior.
