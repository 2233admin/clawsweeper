## Summary

Current main still has the reported approval-restart UX gap. Pending approvals are process-memory state, the gateway still emits the raw `unknown or expired approval id` sentinel, `/approve` still formats that raw message for users, and Telegram callback handling still rethrows stale approval failures. The related narrow fix PR #67982 was closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open. The near-term maintainer path is to translate isApprovalNotFoundError in both the generic /approve reply path and Telegram inline callback path into clear guidance that the approval is no longer available, may have expired or been invalidated by a gateway restart, and should be re-requested. Add focused regression coverage in src/auto-reply/reply/commands-approve.test.ts and extensions/telegram callback tests. Treat disk-backed approval persistence as a separate product/storage follow-up.
