## Summary

Keep #38781 open. Current main still has the macOS node contract gap: the agent node exec path invokes `system.run.prepare`, the gateway rejects commands not declared by the node, and the Swift macOS node still does not define, advertise, or dispatch `system.run.prepare`. The PR discussion is also useful evidence: one user reported that locally applying the PR restored Mac node use from an Ubuntu gateway, and the author recently rebased the PR and extracted the handler to address lint. Related PR #69725 is an alternate open fallback path, but it is not merged and does not supersede this direct macOS-node fix on current main.

## Best Possible Solution

Keep #38781 open until maintainers either merge a direct macOS `system.run.prepare` implementation or merge and designate a canonical fallback such as #69725. The direct fix should align the Swift macOS node with the TypeScript node-host contract: add shared prepare params/plan types, advertise `system.run.prepare`, dispatch it in `MacNodeRuntime`, return the canonical `systemRunPlan` shape consumed by `executeNodeHostCommand`, and add focused macOS tests for declared commands, prepare payload shape, wrapper/rawCommand consistency, and path hardening.
