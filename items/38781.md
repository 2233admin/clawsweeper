## Summary

Keep #38781 open. Current main still has the macOS Swift node contract gap: the TypeScript/node exec path still calls `system.run.prepare`, and the gateway rejects undeclared node commands before forwarding, but the macOS Swift node does not define, advertise, or dispatch `system.run.prepare`. The related #69725 is an open alternate fallback PR, not a merged superseding fix, and #38781 was actively rebased with follow-up fixes on 2026-04-26.

## Best Possible Solution

Keep #38781 open until maintainers either merge a direct macOS Swift `system.run.prepare` implementation or explicitly choose and merge an alternate fallback such as #69725. The direct fix should define the Swift command/types, advertise the command from the macOS node, dispatch it in `MacNodeRuntime`, return the prepared plan shape expected by `executeNodeHostCommand`, and add focused tests for advertisement, dispatch, shell-wrapper/rawCommand, env, and path-canonicalization behavior.
