## Summary

Keep PR #69121 open. Current main has pre-restart deferral for queued work and pending replies, but it does not implement this PR's second shutdown-phase drain: the run loop does not pass a remaining drain budget into `server.close`, the `GatewayServer.close` type has no `drainTimeoutMs`, and `createGatewayCloseHandler` does not wait for pending reply dispatchers or active chat runs before closing channels/sockets. The PR is recent, scoped to gateway restart correctness, and has active author follow-up rather than evidence that it is obsolete or superseded.

## Best Possible Solution

Keep #69121 open for gateway maintainer review. The right path is to finish or revise the PR so `runGatewayLoop` passes the remaining restart drain budget into `server.close`, `createGatewayCloseHandler` performs a bounded pending-reply/active-chat drain only on restart shutdown, active chat runs are aborted through the existing abort path when the budget is exhausted, and focused run-loop/server-close regression tests cover the handoff and zero-budget behavior.
