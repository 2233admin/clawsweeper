## Summary

Keep #69121 open. Current main still has pre-restart deferral for queued work and pending replies, but it does not implement the PR's second-stage bounded drain inside `server.close()` or the active chat-run abort behavior described by the PR. The PR discussion shows recent, targeted contributor work on a real gateway restart correctness gap, not an obsolete or superseded change.

## Best Possible Solution

Keep #69121 open for maintainer review. The best path is a narrow gateway fix that passes the remaining restart drain budget into `server.close()`, drains pending reply dispatchers and active chat runs before closing sockets/channels, aborts still-active chat runs via the existing chat abort path when the bounded budget is exhausted, includes zero-budget regression coverage, and splits any unrelated qa-lab changes unless they are required for the reproduction.
