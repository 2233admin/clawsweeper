## Summary

Keep this issue open. The reported behavior is a recent Web UI/WebChat whitespace bug where live-rendered assistant code differs from refreshed history, and current main still has separate live delta/final, history reload, markdown rendering, and Codex terminal-event paths without a targeted parity regression proving it is fixed.

## Best Possible Solution

Keep this issue open. The maintainer path is to reproduce on current main with local Web UI/WebChat using the openclaw-cpa-codex route, capture the live chat delta/final payload and the refreshed chat.history payload for the same assistant turn, then add a regression at the first differing layer: Gateway live projection in src/gateway/server-chat.ts or src/gateway/live-chat-projector.ts, UI live rendering in ui/src/ui/controllers/chat.ts or ui/src/ui/chat/grouped-render.ts, markdown rendering in ui/src/ui/markdown.ts, or Codex projection in extensions/codex/src/app-server/event-projector.ts/run-attempt.ts.
