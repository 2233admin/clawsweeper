## Summary

Keep this issue open. Current main still provides whole-prompt replacement and provider/runtime-owned section overrides, but not the requested user-level targeted overrides or trust policy for Safety, Tool Call Style, and OpenClaw Self-Update/update-permission guidance.

## Best Possible Solution

Keep this open for core agent configuration design. A conservative implementation would add a documented, schema-backed user-level and per-agent prompt policy surface for selected sections, or an explicit trust policy, wire it through `src/agents/system-prompt.ts`, config schema/help/generated docs, and gateway config mutation guards, and add tests proving Safety, Tool Call Style, and OpenClaw Self-Update guidance can be selectively replaced or relaxed without requiring full `systemPromptOverride`. Coordinate with #25493 while preserving this issue's narrower safety-policy requirement.

## Risks / Open Questions

- Relaxing Safety and self-update guidance changes OpenClaw's operator trust boundary and needs explicit maintainer product/security judgment, not just prompt text plumbing.
- Whole-prompt replacement exists today, but using it as the answer would force users to maintain the entire OpenClaw prompt and miss future prompt updates instead of getting the targeted, schema-backed control requested here.
- Plugin hooks can mutate prompt text, but the documented hook surface is plugin-owned and not a user-level, schema-backed targeted override for the cited core sections.
