## Summary

Keep this issue open. Current main still exposes only partial Dreams status: enabled/idle state, counts, phase next-run times, and diary content. It still does not expose a last dreaming run timestamp, run result or partial-failure outcome, per-run processed trace count, or diary updated/workspace status in the Dreams UI.

## Best Possible Solution

Keep this issue open. Add a memory-core-owned Dreaming run summary exposed through doctor.memory.status or a sibling Dreams status method, with last run time, outcome such as ok/partial/error/skipped, duration, processed trace/candidate/applied/failed counts, diary path/updatedAt/workspace status, and next run. Then render it in the Dreams tab and cover the gateway payload, UI controller normalization, and view rendering with focused tests.
