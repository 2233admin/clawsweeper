## Summary

Keep open. Current main still leaves key Control UI chat and form/config text controls below the 16px iOS Safari focus threshold, and open PR #64673 remains an unmerged implementation candidate that explicitly targets this issue.

## Best Possible Solution

Keep this issue open. Review and land PR #64673 if it is still the right patch after rebasing and broadening coverage, or replace it with a maintainer patch that sets focusable Control UI text inputs to at least 16px on iOS/touch-primary devices. The fix should cover the main chat composer, agent chat textarea, generic .field controls, config inputs/textareas/selects, and add a focused CSS regression test for those selectors.
