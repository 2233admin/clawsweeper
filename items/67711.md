## Summary

Keep open. Current `main` still has the bridge-prune data-loss path this PR is trying to fix: an unavailable memory capability becomes an empty public-artifact list, `activeKeys` stays empty, and bridge sync still unconditionally prunes prior bridge entries. The follow-up PR discussion also shows the author addressed the review concern by switching the guard to `publicArtifacts.length`, so this is still live, relevant work rather than a stale or superseded PR.

## Evidence
