## Summary

Keep open. Current main still lacks the main resource-cleanup changes from #54436: `downloadToFile` does not drain redirect/error responses, does not use a double-settle guard, does not destroy the write stream before size-limit abort, and has no early response-level error handler. `saveMediaSource` also lacks the outer temp-file cleanup invariant proposed by the PR. The PR discussion is positive and the provided CI note says relevant lanes passed, so this remains plausible useful work rather than conservative cleanup.

## Best Possible Solution

Keep #54436 open and ask for a rebase/narrow refresh against current main. The best path is to preserve current `maxBytes` and redirect-header safety behavior, add the resource cleanup changes in `downloadToFile`, keep or add focused regression tests for overflow, response errors, HTTP error/temp cleanup, and redirect/error body draining, then run the changed media test/check lanes.
