## Summary

Keep open. Current `main` does not expose `tools.web.fetch.useEnvProxy`, does not thread that flag through `web_fetch`, and still lacks the PR's shared gating helper for `HTTP(S)_PROXY`/`NO_PROXY`. The PR remains an active, non-stale code change rather than something already implemented or safely auto-closable.
