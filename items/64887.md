## Summary

Keep open. Current main still records `livenessState` in embedded-run metadata and lifecycle events, but blocked/abandoned state is not projected into the user-facing reply payload contract or delivery path.

## Best Possible Solution

Keep this open and implement a generic surfacing layer. When final embedded-run `meta.livenessState` is `blocked` or `abandoned`, project that state into the final user-facing payload path, either through a documented `ReplyPayload`/`AgentRuntimeReplyPayload` field or through a stable footer such as `Run ended early: blocked.` Add focused regressions for strict-agentic blocked exits, abandoned incomplete-turn exits, and at least one delivery renderer/adapter receiving the surfaced state.

## Risks / Open Questions

- Closing now would conflate lifecycle/meta observability with the separate request to surface blocked/abandoned state in outbound user-facing payloads.
- Adding a structured payload field touches public `ReplyPayload` and `AgentRuntimeReplyPayload` compatibility; a footer-only fix is narrower but should be documented and regression-tested so channels can rely on it.
