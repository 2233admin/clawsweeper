## Summary

Keep open. This issue tracks a real deferred liveness gap in the open orchestrator PR before shadow/live mode is enabled. Current main does not contain the orchestrator extension, PR #72086 remains open/unmerged, and the PR author explicitly deferred the `in_progress` expiry concern to #72095. The PR branch still excludes `in_progress` from expiry and the spawn watcher has no max duration, so no close reason applies.

## Best Possible Solution

Keep #72095 open and resolve it in the orchestrator PR stack before any shadow/live traffic. The concrete fix should add a watcher-side max duration that emits a failed outcome, and preferably also add a store-level second-tier reclaim path such as `force_expire` for watcher crashes, with store and spawn-watch tests plus runbook notes updated.
