## Summary

Keep #43737 open. Current main still can omit the main session from the Control UI dropdown when the current/remembered session is heartbeat and `sessions.list` does not return a main row. The linked #43984 directly tracks a plausible fix but is still open, while #43746 was closed unmerged, so the bug is not implemented on main.

## Evidence

- **Dropdown options still come only from listed rows plus current session:** `resolveSessionOptionGroups` builds options from `sessions?.sessions`, filters some rows, then calls `addOption(sessionKey)`. It does not add `hello.snapshot.sessionDefaults.mainSessionKey` or otherwise synthesize a main-session option.
