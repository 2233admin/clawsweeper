## Summary

Keep this issue open. Current main still lacks a guaranteed main-session option in the chat dropdown when `sessions.list` only returns heartbeat and the selected session is heartbeat, and the provided GitHub context shows open PR #43984 still uses closing syntax for this issue.

## Best Possible Solution

Review and land PR #43984 or explicitly supersede it with an equivalent narrow fix. The shipped behavior should guarantee the resolved main session key is selectable in the chat dropdown when `sessions.list` returns only heartbeat, with regression coverage for stale main plus active heartbeat.
