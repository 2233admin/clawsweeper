## Summary

Current `main` already preserves device identity for authenticated loopback probes, and it has broader probe coverage than this PR. The remaining behavior in this PR that disables identity for authenticated remote probes does not match current `main` semantics.

## Evidence

## Close Comment

Closing from a Codex review because current `main` already implements the loopback fix this PR was aiming for. `src/gateway/probe.ts:148` now keeps authenticated local probes device-bound and only disables device identity for unauthenticated loopback probes, and that behavior is covered in `src/gateway/probe.test.ts:129` and `src/gateway/probe.auth.integration.test.ts:29`. Current `main` also intentionally keeps device identity enabled for authenticated remote probes (`src/gateway/probe.test.ts:152`), so this PR no longer matches the behavior on `main` and is superseded rather than mergeable as-is.
