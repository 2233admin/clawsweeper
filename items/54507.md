## Summary

Current main still does not implement the requested Feishu/Lark runtime-footer preference. The footer is still constructed unconditionally from Agent/Model/Provider and passed into both streaming and non-streaming card replies, while the Feishu config schema, plugin manifest, docs, and tests do not expose a footer-specific hide/customize setting. Existing `renderMode: "raw"` is too broad to count as this feature because it also disables card/streaming presentation.

## Best Possible Solution

Keep this issue open. Implement a Feishu-owned typed setting such as `channels.feishu.showRuntimeFooter` or a small `replyFooter` mode/template, support top-level and per-account config, thread it through `resolveCardNote`, streaming start/close, and non-streaming card delivery, then update the Feishu zod schema, plugin manifest metadata, public Feishu docs, and reply dispatcher tests while preserving the current default footer.
