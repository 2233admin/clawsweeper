## Summary

Keep #70915 open. Current main still routes the bundled Z.AI media-understanding provider through the generic image helper without any ZAI vision endpoint override, while the PR is a focused fix for the still-open #70886. The PR also has an unresolved review concern about request-scoped config rewrites and the shared models.json path, so it should not be closed as implemented or obsolete.

## Best Possible Solution

Keep #70915 open and ask for a maintainer-quality fix that routes known Z.AI vision/media-understanding requests from coding endpoints to the standard endpoint at a stable model/request-resolution boundary, with regression coverage for coding-global, coding-cn, v4/v5 path conversion, and the generic media-understanding models.json flow. Once that lands, close #70886 as fixed by the shipped implementation.
