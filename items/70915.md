## Summary

Keep #70915 open. Current main still appears to route ZAI image-understanding calls through the same configured provider baseUrl used for normal ZAI chat models, so a Coding Plan base URL can still reach the coding endpoint for `glm-4.6v`. The PR is a focused recent bug-fix attempt for open issue #70886, but the best fix should address the review concern about request-scoped config rewrites, likely by adding provider-owned image endpoint routing at model/request resolution time.

## Best Possible Solution

Keep the PR open, but steer it toward a provider-owned endpoint selection fix for ZAI image models, preferably at the model/request routing boundary so `glm-4.6v`, `glm-4.5v`, and `glm-5v-turbo` use the standard ZAI chat-completions endpoint without mutating shared config or depending on models.json timing. Add focused regression coverage for coding-global and coding-cn base URLs on the image-understanding path.
