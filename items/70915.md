## Summary

Keep PR #70915 open. Current main still has the Z.AI media-understanding provider bound directly to the generic image helpers, while Z.AI onboarding can configure Coding Plan base URLs and the linked bug reports 404s for Z.AI vision models on that path. The PR is therefore not obsolete, though its current request-scoped config rewrite should be revised before merge because the image runtime persists/reads shared models.json during resolution.

## Best Possible Solution

Keep the PR open and revise it into a provider-owned fix in the Z.AI plugin. The safest direction is to route Z.AI image-capable vision models away from /api/coding/paas/... without mutating request cfg or shared models.json, using an existing resolved-model/media-understanding seam or adding a narrow tested seam if needed. Keep regression coverage for glm-4.6v, glm-4.5v, glm-5v-turbo, both global and CN coding endpoints, and the vN regex case already added by the PR.
