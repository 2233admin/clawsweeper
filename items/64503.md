## Summary

Current main still does not forward Microsoft Teams messageBack Action.Submit activities that arrive as type="message" with empty text and activity.value. This PR is open, unmerged, and directly targets the remaining gap with focused regression tests, so it should stay open for maintainer review rather than be closed as obsolete.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the remaining Microsoft Teams messageBack Action.Submit gap. Maintainers should review the [CARD_ACTION] serialization contract, fix the optional log.warn call if needed, run the targeted msteams regression test/typecheck lane, and then land the PR or request focused changes.
