## Summary

Keep this issue open. Current main still preserves the reported 200k fallback for Anthropic Opus/Sonnet 4.6 unless `params.context1m: true` is configured, and open PR #66862 is an unmerged implementation candidate that references this issue with closing syntax.

## Best Possible Solution

Keep this issue open while maintainers review PR #66862 or an equivalent patch. The fix should align `resolveContextTokensForModel()`, session/status persistence, Control UI behavior, Anthropic beta-header and auth semantics, docs, and regression coverage for Opus/Sonnet 4.6 without `context1m`, including an explicit opt-out or legacy-auth behavior if needed.
