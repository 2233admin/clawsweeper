## Summary

Keep this PR open. Current main still lacks the central behavior: typing TTL remains a fixed two-minute default at the controller call site, and embedded-agent failures still do not have timeout-specific final user copy. The PR is focused and not security-sensitive, but it has grounded functional review blockers around fallback attempt budgeting and over-broad timeout classification. Same-author #41957 and related #43204 are open consolidation/review candidates, not safe automatic close reasons.

## Best Possible Solution

Keep this PR open until maintainers either review it directly or consolidate it with #41957 and the broader TTL-feedback work in #43204. The implementation should compute typing TTL from the actual fallback candidate budget, including configured-primary append and transient retry replay, use a narrow timeout predicate such as `isTimeoutError` for final timeout copy, and add regression coverage for true timeout failures versus bare network/DNS errors.
