## Summary

Keep open. Current main has delivery inference for isolated `agentTurn` cron jobs, but it does not implement this PR’s central behavior: promoting implicit chat reminder `systemEvent` jobs into deliverable isolated `agentTurn` jobs. A reminder-like `systemEvent` without explicit `sessionTarget` still defaults to `main`, runs as an internal system event, and records delivery as not requested. The SonicBotMan comment is useful operational cron feedback, but it describes scheduler drift under load rather than superseding this PR’s delivery fix.

## Best Possible Solution

Keep the PR open for maintainer review or revision. The best code path is a narrow fix in `src/agents/tools/cron-tool.ts` or the cron normalization/tooling layer that makes implicit chat-created reminders deliverable while preserving explicit `sessionTarget: "main"` internal semantics, with regression tests covering promotion, explicit-main preservation, current chat delivery inference, and context history. Scheduler drift from the PR comment should be tracked separately if still reproducible.
