## Summary

Keep open. Current main appears to have the Discord path/filePath upload route, but the direct buffer attachment path advertised by ordinary message.send is still not delivered or rejected; buffer/contentType/mimeType are accepted in the schema and then ignored before the Discord send runtime.

## Best Possible Solution

Keep this issue open and scope it to direct buffer attachments on ordinary Discord message.send. The best fix is to make buffer, filename, and contentType/mimeType flow end to end into Discord multipart uploads with regression coverage, or remove/fail-fast those fields for ordinary send if buffers are intentionally unsupported. Review #57335 and #63578 as implementation candidates, but do not close until the fix is merged and covered for Discord.
