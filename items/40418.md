## Summary

Keep #40418 open. Current main only partially addresses the request: the bundled session-memory hook saves recent raw user/assistant text on explicit `/new` or `/reset`, and startup context reloads existing memory files as bounded untrusted notes. It does not implement automatic synthesis, `.session_memory_preserve.json`, full transcript exports with anchored links, or a generated daily continuation package. Related PR #50584 is still open/unmerged in the provided GitHub context, and related bug #69943 remains a live safety concern for raw memory persistence and re-injection.

## Best Possible Solution

Keep #40418 open until maintainers either land a scoped core implementation or explicitly redirect the broader design to a plugin/context-engine path. A strong implementation should coordinate with #50584 and #69943: opt-in synthesis, sanitizer/filtering before persistence, compact continuation packages, canonical daily summary append, receipt links to full transcripts, timezone-aware filenames, and regression tests for `/new`/`/reset` preservation plus startup hydration.
