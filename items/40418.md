## Summary

Keep this issue open. Current main has partial `/new` and `/reset` session-memory preservation plus startup hydration from existing memory files, but it does not implement automated synthesis, `.session_memory_preserve.json`, full transcript exports with anchored receipts, or generated daily continuation packages. Related PR #50584 remains open/unmerged, and #69943 is still an active safety concern for expanding automatic persistence.

## Best Possible Solution

Keep this open until maintainers either land a scoped core implementation or explicitly redirect the feature to a ClawHub/context-engine plugin design. A strong implementation should coordinate with #50584 and #69943: opt-in synthesis, sanitization/filtering before persistence, compact daily continuation packages, canonical daily summary handling, receipt links to full transcripts, timezone-aware filenames, and regression coverage for `/new`/`/reset` preservation plus startup hydration.
