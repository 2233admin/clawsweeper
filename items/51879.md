## Close Comment

Closing from a Codex review: current `main` already contains the handshake changes that address this report, and those changes are also present in released `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The current gateway code uses a `10_000` ms preauth timeout in `src/gateway/handshake-timeouts.ts:1` and applies it in `src/gateway/server/ws-connection.ts:366`, so the reported 3-second handshake cutoff is no longer the shipped behavior. The current handshake policy also allows operator shared-auth connects without requiring device identity (`src/gateway/server/ws-connection/connect-policy.ts:139`), and the loopback/shared-auth CLI path is covered in `src/gateway/server/ws-connection/handshake-auth-helpers.test.ts:268`. The changelog records both the 10-second handshake fix and the loopback shared-auth/device-identity adjustment (`CHANGELOG.md:2446`, `CHANGELOG.md:2554`). If this still reproduces on `v2026.4.22` or newer, open a fresh issue with exact version plus current `gateway health --json` / gateway logs so maintainers can check for a new regression.
