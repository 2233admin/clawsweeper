## Summary

Keep open. Current main routes `message(action=send)` through the Feishu plugin and supports semantic `presentation` cards, but raw Feishu interactive-card JSON supplied as the `message` string is still treated as text and sent via the post/plain-text path rather than `sendCardFeishu`. The related Feishu reports #53295 and #53310 cover adjacent validation/card-delivery bugs, not this remaining raw-message-card compatibility regression.

## Best Possible Solution

Keep this issue open for Feishu-owned maintainer handling. The narrow code path is to add a conservative Feishu native card JSON detector in `extensions/feishu/src/channel.ts` for `send` and `thread-reply` text/message params, route valid v1/v2/wrapped interactive card payloads to `sendCardFeishu`, and add a regression test. The alternative is an explicit maintainer decision and docs update that raw provider-native Feishu card JSON in `message` is unsupported and users should send semantic `presentation` payloads instead.
