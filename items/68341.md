## Summary

Keep open. Current main does not make this PR obsolete: the poll/send inflight dedupe problem is still a real gateway surface, and the PR is an active implementation candidate. It should not be merged as-is, though, because parts of the current diff conflict with current main behavior and the inflight registration still appears too late to fully close the race called out in review.

## Best Possible Solution

Keep this PR open for maintainer review, but require it to be narrowed or split. The safest path is to preserve current MCP bearer-token owner propagation, add focused failing tests for poll/message.action/send concurrent idempotency before channel resolution, register inflight work synchronously before the first await, and leave redundant cron/failover tweaks out unless they have a concrete repro and regression tests.
