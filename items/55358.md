## Summary

Keep open. Current `main` still derives Slack inbound DM content from `message.text` plus attachment text, and the assistant-DM `message_changed` rehydration path also copies only `message.text`. I did not find a current-main fix or a regression test covering long DM payloads whose full content lives outside `text`, so there is not strong evidence this issue is implemented or obsolete.
