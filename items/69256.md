## Summary

Keep #69256 open. Current main still has the reported cron direct-delivery bug: structured/threaded direct delivery calls `sessions.delete` from the cleanup `finally` path whenever `deleteAfterRun` is true, without checking active descendant subagent runs. The PR is not obsolete, but it should be revised before merge because review comments correctly flag missing regression coverage and a second-stage cleanup problem, and current main now distinguishes `runSessionKey` from `agentSessionKey` for isolated cron descendant tracking.

## Best Possible Solution

Keep the PR open and ask for a revised patch against current main. The fix should make direct structured/threaded delivery share the text path’s run-scoped descendant awareness, defer cleanup while descendants are active, re-attempt `deleteAfterRun` cleanup after descendants drain, and add regression coverage for active-descendant structured/threaded direct delivery plus the later cleanup completion path.
