## Summary

Keep open. Current main does not expose `diagnostics.pricing`: the diagnostics handler, advertised method list, and READ_SCOPE table only include `diagnostics.stability`. The underlying pricing cache metadata helper already exists, and the PR is a focused implementation of a plausible gateway observability surface rather than obsolete or duplicate cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a small core gateway change that adds `diagnostics.pricing` through the diagnostics handler, READ_SCOPE, advertised method list, focused tests for empty/populated cache state, and public protocol or CLI documentation if maintainers want this to be a supported operator diagnostic RPC.
