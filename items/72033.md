## Summary

Keep PR #72033 open. Current main still does not implement or advertise `diagnostics.pricing`; only `diagnostics.stability` is registered, scoped, documented, and tested. The existing pricing cache metadata helper is present in core, and startup still runs pricing refresh with timeout-warning behavior, so the PR remains a focused gateway observability change rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers want this operator diagnostic surface, the best path is the PR's small core gateway change: add `diagnostics.pricing` to the diagnostics handler, READ_SCOPE table, advertised method list, focused cache-state tests, and public gateway protocol/operator docs if the RPC is intended to be supported.
