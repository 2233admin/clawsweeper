## Summary

Keep PR #72033 open. Current main still lacks the proposed `diagnostics.pricing` Gateway RPC, even though the core pricing-cache metadata seam and timeout warning path already exist. The PR remains a focused core Gateway observability change, and the provided PR discussion shows reviewers found the approach useful while the author addressed the concrete review comments in follow-up commits.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers want this operator diagnostic surface, the best path is the PR's small core Gateway change: add `diagnostics.pricing` to the diagnostics handler, READ_SCOPE table, advertised method list, focused cache-state tests, and public Gateway protocol/operator docs if the RPC is intended to be supported.
