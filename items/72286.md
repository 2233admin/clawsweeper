## Summary

Keep #72286 open. Current main does not implement Google Live browser Talk sessions: Control UI Talk is still documented and coded as OpenAI/WebRTC-only, the Gateway result schema has no transport/websocket/setup fields, and the bundled Google realtime voice provider only exposes backend bridge creation, not browser session creation. The draft PR is recent, unmerged, and addresses missing core/UI/SDK behavior rather than an already-shipped or clearly ClawHub-only capability.

## Best Possible Solution

Keep the PR or an equivalent maintainer branch open. The right product/code path is to extend the realtime browser session contract with a transport discriminator and provider setup metadata, implement Google createBrowserSession in the bundled Google provider using constrained Live auth tokens, update Gateway and Control UI to branch between OpenAI WebRTC and Google Live WebSocket, and land focused provider/UI/protocol tests plus docs. This should not be closed as implemented or ClawHub-only because current main lacks the core browser transport and UI hooks a plugin would need.
