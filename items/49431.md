## Summary

Keep open. Current main still lacks this PR’s durable pending-inbound store, startup replay, and file-backed active-turn recovery, and this PR remains a live implementation candidate with unresolved correctness and security-sensitive review findings rather than something already implemented or superseded.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should either review it and require fixes for the remaining correctness/security concerns, or explicitly supersede it with a canonical durable restart-resilience design such as #65650 after that design is accepted.
