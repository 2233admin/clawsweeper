## Summary

Current `main` does not implement the requested fallback-chain behavior for either compaction or the in-tree summary-model path. Compaction still accepts only a single `agents.defaults.compaction.model` override and runs one resolved provider/model, with only a think-level retry. The in-tree `summaryModel` path also resolves a single model string and does not walk fallbacks. The `lossless-claw` config mentioned in the issue is plugin-owned and not defined in this checkout, so there is not enough evidence to close this as already implemented or obsolete.
