## Summary

Keep #69206 open. Current main still generates and registers `/dock-discord` / `/dock_discord` as Telegram native commands, and Telegram native dispatch now reaches the shared reply pipeline with `CommandSource: "native"`, which addresses the broad fallback class discussed in related #45596. However, I found no current command handler or plugin command that performs the requested channel handoff/delivery-context switch for dock commands; unmatched slash commands still continue to the agent path, which is the reported failure mode.

## Best Possible Solution

Implement dock commands as a real built-in command path, likely in the shared reply command handler layer: resolve the requested channel from `dock:<channel>`, verify that channel/account has a usable route for the same conversation or linked identity, update the target session delivery context/last route accordingly, return a deterministic confirmation or actionable error, and add Telegram-native plus text-command regression tests covering `/dock_discord` from Telegram to Discord.
