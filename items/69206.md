## Summary

Keep open. Current main still advertises generated Telegram `/dock_discord` / `/dock-discord` commands, and Telegram native command handling now preserves `CommandSource: "native"` and targets the chat session, but there is still no dock command handler. The command text falls through the shared command pipeline and can continue to the normal model path, which is the core behavior reported in #69206. The related #45596 generic Telegram command fallback discussion is relevant, but it does not close this dock-specific gap.

## Best Possible Solution

Implement dock command execution in the shared command path, keyed from `dock:<channel>` / generated aliases, so Telegram native `/dock_discord` stops before model dispatch and performs the intended channel handoff using the current session/identity-link delivery metadata. Add a focused Telegram native-command regression test proving `/dock_discord` is consumed as a native command and does not reach normal chat fallback; keep #45596 as related background rather than a canonical close for this dock-specific bug.
