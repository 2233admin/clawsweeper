## Summary

Keep #69206 open. Current main still advertises generated dock commands and Telegram registers native handlers for them, but the command-dispatch path has no dock-specific handler; after no handler matches, the turn can continue into the normal agent/model path, which matches the reported fall-through behavior. Related #45596 was a broader Telegram native skill/custom command routing issue and does not cover the missing dock command implementation.

## Best Possible Solution

Implement a deterministic dock command handler in the shared command path, likely as a new `src/auto-reply/reply/commands-dock.ts` loaded by `commands-handlers.runtime.ts`. It should match generated `docks` commands such as `/dock-discord` and `/dock_discord`, resolve the target channel plugin, update or select the target session delivery route using existing session delivery/identity-link machinery, and return a clear success or configuration error without invoking the model. Add a Telegram native regression covering `/dock_discord` with both Telegram and Discord enabled, plus docs updates if the exact handoff semantics differ from the current “Switch to discord for replies” wording.
