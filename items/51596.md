## Summary

Keep open. Current `main` still contains the exact docs inconsistency this PR addresses: `docs/tools/creating-skills.md` uses a hyphenated folder example but an underscored `name` example and labels the field as `snake_case`. Other current docs and bundled skills use hyphenated names instead, so this is not implemented on `main`. The only caveat is that current code still accepts underscores, so the final wording may need a small maintainer pass rather than an auto-close.
