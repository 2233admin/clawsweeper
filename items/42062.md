## Summary

Keep open. Current `main` still has no generic or Ollama-specific guard for plain assistant narration leaked in `message.content`; it only separates/suppresses structured reasoning fields or payloads explicitly marked/prefixed as reasoning.

## Evidence

- **Ollama stream makes `message.content` visible text:** `extensions/ollama/src/stream.ts:524` maps `message.thinking`/`message.reasoning` into a thinking block, but `extensions/ollama/src/stream.ts:528` still appends any `message.content` directly as visible assistant text. If GLM-5 emits internal narration in `content`, current main will surface it.
  - file: [extensions/ollama/src/stream.ts:524](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/extensions/ollama/src/stream.ts#L524)
  - command: `nl -ba extensions/ollama/src/stream.ts | sed -n '519,531p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Current Ollama behavior is only to forward `think` upstream:** `extensions/ollama/src/stream-runtime.test.ts:100` verifies native Ollama requests send `think: false` when thinking is off, and `docs/providers/ollama.md:464` documents the same. That is an upstream toggle, not a downstream filter for leaked plain text if the model ignores it.
  - file: [extensions/ollama/src/stream-runtime.test.ts:100](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/extensions/ollama/src/stream-runtime.test.ts#L100)
  - command: `nl -ba extensions/ollama/src/stream-runtime.test.ts | sed -n '100,149p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Core outbound suppression only handles flagged reasoning payloads:** `src/auto-reply/reply/reply-payloads-base.ts:82` returns true only for `payload.isReasoning === true`, and `src/infra/outbound/payloads.ts:133` drops payloads only through that check. Plain-text narration in an ordinary text payload is not suppressed at the core outbound layer.
  - file: [src/auto-reply/reply/reply-payloads-base.ts:82](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/auto-reply/reply/reply-payloads-base.ts#L82)

That helper, in `src/plugin-sdk/reply-payload.ts:49`, recognizes `isReasoning` or text starting with `Reasoning:` (including quoted prefix), not generic 'thinking out loud' prose.
  - file: [extensions/whatsapp/src/auto-reply/deliver-reply.ts:42](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/extensions/whatsapp/src/auto-reply/deliver-reply.ts#L42)

That confirms there is no broader content scrub for untagged/plain-text reasoning-like prose.
