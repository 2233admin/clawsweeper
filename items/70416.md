## Summary

Current main has Anthropic Opus 4.7 model support and a pricing-cache path that canonicalizes OpenRouter’s dotted Anthropic ids, but I could not verify that the exact reported `usage.cost.cacheWrite` bug is fixed end-to-end for direct Anthropic Opus 4.7 sessions. The remaining gaps are material: LiteLLM tiered pricing is only matched on exact `provider/model` keys, while the current LiteLLM catalog exposes Opus 4.7 as `anthropic.claude-opus-4-7`; and transcript cost recomputation only happens when tiered pricing is present or the original cost is missing. That is not strong enough evidence for an auto-close.

## Evidence

- **OpenRouter pricing lookup handles dotted Anthropic ids:** `canonicalizeOpenRouterLookupId()` rewrites Anthropic dotted ids like `claude-opus-4.7` to dashed form, and `buildOpenRouterExactCandidates()` also tries the dotted variant when resolving cached pricing.
  - file: [src/gateway/model-pricing-cache.ts:288](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/gateway/model-pricing-cache.ts#L288)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **LiteLLM lookup still requires exact slash-qualified keys:** `resolveLiteLLMPricingForRef()` only checks `catalog.get(`${provider}/${model}`)`. The refresh path only attaches tiered pricing when that exact lookup succeeds.
  - file: [src/gateway/model-pricing-cache.ts:275](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/gateway/model-pricing-cache.ts#L275)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Transcript cost totals are not always recomputed:** `session-cost-usage` only recomputes transcript costs when `tieredPricing` exists or when `entry.costTotal` is missing. If the transport already wrote a wrong flat `usage.cost`, aggregation can still trust that stored value.
  - file: [src/infra/session-cost-usage.ts:252](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/infra/session-cost-usage.ts#L252)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Anthropic 4.7 support is present, but pricing is not directly covered by tests here:** The Anthropic plugin has explicit Opus 4.7 forward-compat and context handling, and tests cover dynamic resolution/context normalization for 4.7. The nearby test coverage is about model resolution, not `usage.cost` pricing correctness.
  - file: [extensions/anthropic/register.runtime.ts:45](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/extensions/anthropic/register.runtime.ts#L45)
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Current live catalogs do not line up cleanly for direct Anthropic tiered pricing:** Read-only network inspection showed OpenRouter currently exposes `anthropic/claude-opus-4.7` with flat pricing, while LiteLLM currently exposes `anthropic.claude-opus-4-7` under `bedrock_converse`. That mismatch means the current slash-key LiteLLM lookup does not obviously supply tiered pricing for direct `anthropic/claude-opus-4-7` refs.
  - command: `curl -fsSL https://openrouter.ai/api/v1/models | python -c '...'; curl -fsSL https://raw.githubusercontent.com/BerriAI/litellm/main/model_prices_and_context_window.json | sed -n '1128,1145p'`
