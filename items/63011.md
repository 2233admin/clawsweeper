## Summary

Keep this issue open. Current main still has the reported split: gateway sends reach the delivery path but collapse hook cancellation into an empty-result error, while direct message sends can still be handled by plugin action runtimes before the core hook-bearing delivery path. The same author also has open draft PR #63007 targeting the gateway identity and guarded-denial gaps, so this issue should not be closed before that implementation candidate is reviewed, landed, or superseded.

## Best Possible Solution

Keep this issue open. Review and either land #63007 or replace it with a maintainer-approved canonical outbound send contract that threads agent/session/request identity into message_sending, returns real provider delivery results on success, returns explicit guarded-denial results on hook cancellation, and aligns direct CLI, gateway send, and plugin action behavior or documents the intentional canonical path.
