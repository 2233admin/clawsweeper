## Summary

Keep open. Current main has meaningful adjacent gateway status, health, restart-health, LaunchAgent recovery, channel-health restarts, stability bundles, diagnostics export, and multi-gateway rescue-bot documentation, but it still does not implement the requested gateway-down watchdog, external notification/escalation path, human-readable startup-error artifact, or first-class multi-instance monitoring. The provided GitHub context also shows PR #53716 remains open and unmerged with explicit closing syntax for this issue, so cleanup should not close the issue ahead of reviewing that implementation candidate.

## Best Possible Solution

Keep this issue open while PR #53716 remains open and unmerged. Maintainers should review that PR or a successor against the current gateway status/probe/stability, service lifecycle, LaunchAgent/systemd behavior, and multi-gateway docs, then land a focused watchdog/startup-diagnostics/notification implementation with tests and docs or close the PR and keep this as the canonical feature request.
