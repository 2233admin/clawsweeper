## Summary

Keep open. Current `main` still does not carry the PR's Signal quote-reply completion work, so this is not conservative cleanup material. The existing code forwards quote text/sender/quote-flag in some cases, but it still omits `ReplyToId`, does not model `quote.id` or `quote.authorNumber`, and still keeps last-entry reply metadata when debounced messages are merged.
