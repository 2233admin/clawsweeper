## Summary

Keep open. Current main still propagates `os.networkInterfaces()` failures through the gateway LAN and tailnet discovery helpers, and the regression tests still assert thrown `uv_interface_addresses` errors instead of graceful fallback. The linked PR #71950 is directly relevant and appears to target the right unsafe callers, but the provided GitHub context shows it is open and unmerged, so it does not implement or supersede this issue on main.

## Best Possible Solution

Keep #71721 open and land PR #71950 or an equivalent maintainer-reviewed patch that changes the gateway LAN and tailnet callers to use `safeNetworkInterfaces()` and updates `src/gateway/net.test.ts` and `src/infra/tailnet.test.ts` to assert graceful fallback instead of thrown errors.
