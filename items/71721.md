## Summary

Keep open. Current main still has gateway-critical paths that call `os.networkInterfaces()` through the unsafe `readNetworkInterfaces()` helper, and current tests explicitly assert the reported failure is thrown. The linked PR #71950 is relevant and positively corroborates the root cause, but it is open/unmerged in the provided context and the issue stack may also involve the bundled Bonjour/ciao path, so the issue should not be closed yet.

## Best Possible Solution

Keep #71721 open until a fix lands on main. The best path is to make gateway startup and discovery tolerate `os.networkInterfaces()` failures by using `safeNetworkInterfaces()` where LAN/tailnet addresses are optional, and to validate or harden the Bonjour/ciao advertisement path if the reported `NetworkManager` stack is from `@homebridge/ciao`. Add regression coverage that expects graceful fallback for `pickPrimaryLanIPv4()`, `listTailnetAddresses()`, and any confirmed Bonjour/ciao unhandled-rejection path, then close via the landed PR.
