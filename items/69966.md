## Summary

Keep #69966 open. Current main still treats `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` as global `agents.defaults` settings: agent entries do not type or validate those keys, runtime/bootstrap resolvers read defaults-only config, Control UI presets patch only `agents.defaults`, and docs only document per-agent overrides for other budget families such as `contextLimits` and `skillsLimits`. The issue is recent, has no protected labels, no maintainer author association, and no linked canonical item in the provided related context that supersedes this exact bootstrap-profile request.

## Best Possible Solution

Keep this issue open and implement the feature in core config/runtime rather than treating it as external plugin work. The likely path is to add typed `agents.list[]` overrides for the three bootstrap profile fields, resolve them by session agent id with fallback to `agents.defaults`, update Control UI to edit either global or selected-agent profile settings, and add schema/runtime/UI/docs tests covering inheritance and per-agent override behavior. Named reusable profiles are a product/design layer that can be considered after the direct per-agent override contract is in place.
