## Summary

Keep #69966 open. Current main still treats bootstrap context profile settings as global `agents.defaults` settings: `agents.list[]` does not type or validate `contextInjection`, `bootstrapMaxChars`, or `bootstrapTotalMaxChars`; runtime budget/context-injection resolvers read only `agents.defaults`; and the Control UI profile presets patch only `agents.defaults`. This is a core config/runtime/UI request, not a ClawHub plugin item, and the related reports in the provided context cover adjacent per-agent settings rather than a shipped canonical duplicate for this bootstrap-profile gap.

## Best Possible Solution

Implement this as a core config/runtime/UI enhancement: add typed and schema-validated per-agent bootstrap profile fields under `agents.list[]` or a documented named-profile mechanism; update generated config help/docs; thread `agentId` through `resolveContextInjectionMode`, bootstrap budget resolution, CLI/embedded runner/context-report/doctor call sites; and update the Control UI profile selector so teams can apply or display profile settings per agent without rewriting global `agents.defaults`. Add schema, resolver, runtime, docs, and focused UI/controller tests.
