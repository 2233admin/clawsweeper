## Summary

Keep #63992 open. Current main still has the exact shared resolver behavior the issue reports: `resolveSessionKey()` hardcodes `DEFAULT_AGENT_ID` into direct and non-group derived keys. Some current Discord inbound paths now route through `resolveAgentRoute()` and should honor the configured default agent, but the underlying resolver and remaining direct call sites are not fixed, and the linked fixing PR #64108 is still open rather than merged.

## Best Possible Solution

Keep the issue open until a fix lands that either makes `resolveSessionKey()` config-aware or removes/canonicalizes all direct write/read call sites. The likely maintainer path is to finish #64108 or an equivalent change with regression coverage for custom default agents across the shared resolver, Slack fallback, command-session resolution, and any channel paths that still bypass `resolveAgentRoute()`. The separate `agents.list[].sessionsDir` idea should be evaluated against the existing `session.store` template/config surface rather than bundled into the regression fix unless maintainers want a new config key.
