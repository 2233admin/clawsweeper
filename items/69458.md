## Summary

Keep open. Current main has shipped adjacent fixes for cached plugin-registry hits and activate:false snapshot loads, but it has not implemented this PR's central OC-40 behavior: keeping an active memory capability visible when an activating plugin scope excludes the memory plugin. The PR discussion and PRtags context show this is related to #68041/#67063, but those related PRs are still open/unmerged and current main still has the capability stored inside scope-swapped memory state.

## Best Possible Solution

Keep the PR open for maintainer review or consolidation. The best path is to land a focused fix that preserves the selected memory capability across active plugin-scope swaps without making stale capabilities survive full loader resets or memory-slot disablement, and to keep regression coverage for cache hits, activate:false snapshots, failed registration rollback, `slots.memory: "none"`, and the memory-wiki bridge artifact path.

## Evidence

This is the structure #69458 proposes to change by hoisting `activeMemoryCapability` outside the scope-swapped state.
