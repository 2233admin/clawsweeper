## Summary

Keep open. Current main has adjacent fixes for cached loads, activate:false snapshot loads, and failed registration rollback, but it does not implement PR #69458's central OC-40 behavior: preserving the active memory capability across an activating plugin-scope swap that excludes the memory plugin. The PRtags context links related open PR #68041, but there is no clearly merged or canonical superseding item, and #69458 still contains useful remaining regression coverage/work.

## Best Possible Solution

Keep #69458 open for maintainer review, but narrow it to the still-missing behavior on current main: memory-wiki/public-artifact access should remain correct when an activating plugin scope excludes the selected memory plugin. The best path is to rebase on current main, preserve the existing cache/snapshot/error rollback fixes, add or keep the exact scope-swap regression test, and coordinate with #67063/#68041 so only one canonical PR lands.
