## Summary

Keep open. Current main does not contain the PR's central fix: memory capability is still stored inside `memoryPluginState`, and active plugin reloads still call `clearMemoryPluginState()`, which clears that capability before loading a new scoped registry. Main does have narrower snapshot/cache fixes for memory capability restoration, but it does not preserve an already-active memory capability across an activating scope that excludes the memory plugin, and the CLI scoped loader still returns only the manifest-planned command plugins without appending the configured memory slot.

## Best Possible Solution

Keep a focused maintainer-reviewed fix open for OC-40. The best path is to consolidate #69458 with the related memory-wiki scope PRs, preserve the active memory capability across plugin-scope swaps without leaving stale capability on full resets, normalize `plugins.slots.memory = "none"`, and keep regression coverage for active scope exclusion, snapshot loads, cache hits, failed registration rollback, and wiki CLI/status artifact counts.
