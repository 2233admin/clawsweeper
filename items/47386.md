## Summary

Current `main` does not implement a webchat-level collapsible summary mode that groups multiple tool executions into one expandable summary, and it does not expose a detailed-vs-summary setting. The existing UI only supports per-tool/per-message collapse or hiding tool calls entirely.

## Evidence

- **Tool messages still render as separate chat messages:** When `showToolCalls` is enabled, each entry in `toolMessages` is appended as its own chat item/message rather than being merged into a single summary row.
  - file: [ui/src/ui/chat/build-chat-items.ts:282](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/ui/src/ui/chat/build-chat-items.ts#L282)
  - command: `nl -ba ui/src/ui/chat/build-chat-items.ts | sed -n '225,290p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Inline tool cards are rendered one-by-one:** `renderInlineToolCards` maps every tool card to `renderToolCard(...)`. There is no aggregate `"N tool calls"` disclosure here.
  - file: [ui/src/ui/chat/grouped-render.ts:1220](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/ui/src/ui/chat/grouped-render.ts#L1220)
