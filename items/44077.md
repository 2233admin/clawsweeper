## Summary

Keep open. Current main still matches the reported sandbox write/edit permission regression: sandbox write uses a pinned Python helper that creates replacement temp files with mode 0o600 and then atomically replaces the target without preserving the original file mode. The sandboxed edit tool writes through the same bridge path. The linked fix PR #44114 is still open/unmerged in the provided GitHub context, so it should not be treated as shipped or superseding the issue.
