## Summary

Keep open. The report is paired with same-author open PR #53315, which uses closing syntax, and current main still treats an undiscoverable configured channel id as a fatal validation issue that can block gateway startup.

## Best Possible Solution

Keep this issue open while maintainers review PR #53315 or a replacement patch. The likely fix belongs around `src/config/validation.ts` and gateway startup handling: a configured but undiscoverable plugin-owned channel should warn and be skipped while other channels still start, with regression coverage and docs updated if the validation contract changes.

## Risks / Open Questions

- Changing every unknown `channels.*` key to a warning could hide ordinary config typos; the fix should make stale plugin-owned channel config warnings prominent and actionable, or distinguish that case where feasible.
- Matrix is currently documented and packaged as bundled on main, so regression coverage should use a genuinely undiscoverable plugin-owned channel or fixture rather than relying only on Matrix's historical packaging state.
