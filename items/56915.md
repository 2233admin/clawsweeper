## Summary

Current `main` does not implement recursive/grouped discovery for paths like `~/.openclaw/skills/coze/koze-retrieval/SKILL.md`. The loader still only accepts the root itself, immediate child directories with `SKILL.md`, or the special repo-style `<root>/skills/*` layout, so the reported managed-skill path remains unsupported.

## Evidence

- **Nested-root heuristic is limited to `<root>/skills/*`:** `resolveNestedSkillsRoot()` only special-cases a sibling `skills/` directory under the configured root and returns the original root otherwise. It does not recurse arbitrary subdirectories such as `~/.openclaw/skills/coze/koze-retrieval`.

  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Discovery still requires immediate child `SKILL.md` files:** After choosing `baseDir`, `loadSkillEntries()` enumerates only direct children of that directory and skips any child that does not contain `SKILL.md` directly under it. A grouping folder like `coze/` is therefore ignored.
