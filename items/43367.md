## Summary

Keep open. The supplied discussion and current main show this is still an actionable multi-agent reliability report: #44997 is an open same-author implementation candidate for the config-write part, and current code only partially mitigates the other symptoms rather than solving the end-to-end orchestration problem.

## Best Possible Solution

Keep this issue open until #44997 is reviewed and either merged or closed. After that, maintainers should split or continue the remaining work: serialized config mutation for `agents add`, session-run queue/lock behavior for isolated agents, CLI/gateway cancellation and visibility for detached work, and subagent completion delivery/recovery. OAuth refresh serialization appears largely covered on main and should be treated as one resolved symptom, not the full fix.
