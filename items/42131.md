## Summary

Keep open. Current `main` still has the reported normalization mismatch: configured safe-bin names are lowercased, but trusted directory entries and resolved executable directories are compared with case-preserving `path.resolve(...)` values. I did not find an equivalent fix on `main`, and current tests do not cover the macOS/Windows case-insensitive-path scenario described in the PR.
