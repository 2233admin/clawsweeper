## Summary

Keep #70971 open. Current main has a partial mitigation that skips bundled MCP creation when a runtime toolsAllow list contains only non-bundled tool names, which helps the exec-only cron leak described in #70939. But the PR's broader fix is still not present: final effectiveTools still appends bundled tools without applying params.toolsAllow, LSP tools are still materialized whenever tools are enabled, and namespaced bundled MCP allowlists can still materialize more bundled tools than requested.

## Best Possible Solution

Keep #70971 open and have a maintainer review/rebase it against current main. The best product/code path is to make runtime toolsAllow authoritative at the final effectiveTools boundary, or equivalently filter filteredBundledTools before append, and add focused regression coverage for at least one namespaced bundled MCP allowlist and one bundled LSP tool being excluded when not explicitly allowed.
