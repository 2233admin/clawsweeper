## Summary

Keep this PR open. Current main still builds final embedded-run tools by appending bundled MCP/LSP tools after the runtime toolsAllow pass, so the reported class of bug is not fully solved. The submitted patch is a plausible starting point, but it needs plugin-aware allow semantics and focused regression coverage before merge.

## Best Possible Solution

Keep this PR open or replace it with a maintainer patch in src/agents/pi-embedded-runner/run/attempt.ts that applies runtime toolsAllow after bundled MCP/LSP materialization using plugin-aware policy semantics, not only raw tool-name equality. Add regression coverage for toolsAllow: ["exec"], an exact bundled MCP tool name, toolsAllow: ["bundle-mcp"], and configured bundled LSP tools before landing.
