## Summary

Keep this issue open. The report is recent, concrete, hardware-specific, and still not disproven by current main: the macOS chat path remains a native SwiftUI/WebChat window with transparent non-opaque AppKit windows, active visual-effect material, material/gradient chat backgrounds, shadows, and visible update animations, with only smoke-test coverage rather than compositor or WindowServer CPU proof.

## Best Possible Solution

Keep this open as a macOS chat rendering performance investigation. Profile current native WebChat on affected Intel Retina hardware, compare opaque/no-material/no-shadow/no-animation variants, verify Reduce Motion behavior, and add a measured low-compositor fallback for older Intel or high-resolution displays if the regression is confirmed.
