## Summary

Keep #71988 open. Current main still lacks the fallback `inbound_claim` broadcast this PR is trying to add, and the related issue/PR context shows this is the active focused implementation path after #48434 was superseded and #71644 was closed dirty. The PR still needs a follow-up patch before merge because the proposed terminal broadcast path would need to preserve synthetic replies and inbound dedupe cleanup.

## Best Possible Solution

Keep the PR open and request a narrow follow-up patch: add the fallback broadcast in `src/auto-reply/reply/dispatch-from-config.ts`, deliver any synthetic reply the global `inbound_claim` handler returns, commit/release inbound dedupe before any new terminal return, and add regression tests for missing-plugin/no-handler broadcast, handled-with-reply, and dedupe cleanup.
