## Summary

Keep this issue open. Current main has substantial partial session management: Gateway session RPCs, Control UI listing/search/label edit/delete/checkpoint controls, Web Chat session switching, TUI session picker, and CLI listing/cleanup. It still does not implement the full requested Topic/Session Management UI, especially first-class metadata such as description/tags/message count/created_at, import/export, archive/restore of deleted topics, and CLI/TUI create/rename/delete management workflows.

## Best Possible Solution

Keep this issue open, but narrow it into concrete maintainer-owned follow-ups: add a first-class Control UI create/manage workflow around sessions.create, sessions.patch, and sessions.delete; decide whether description, tags, createdAt, and messageCount belong in the Gateway session-row contract; add TUI/CLI rename/delete/archive actions only if maintainers want those surfaces; and separately define deleted-archive restore plus import/export semantics with docs and tests.

- Adding description/tags/messageCount/createdAt to session rows would be a public protocol/UI contract change and should be handled additively with docs and focused tests.
