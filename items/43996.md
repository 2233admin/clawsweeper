## Summary

Keep open. Current main still hard-codes Docker `--security-opt no-new-privileges`, has tests expecting that flag, and does not expose a Docker sandbox config knob/fallback/diagnostic for this failure mode. The linked PR #70229 is relevant and appears to target the requested fix, but it is open/unmerged in the provided GitHub context, so the original bug is not yet implemented on main or safely superseded by a landed canonical fix.
