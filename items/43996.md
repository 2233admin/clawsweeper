## Summary

Keep open. Current main still unconditionally adds Docker `--security-opt no-new-privileges` to sandbox containers, exposes no `noNewPrivileges`/equivalent config opt-out, and an open PR explicitly targets this issue with closing syntax. The issue should remain open until that implementation is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open while PR #70229 remains open. Review that PR as the implementation candidate: it should add a default-on `sandbox.docker.noNewPrivileges` or equivalent explicit override, update schema/types/docs/security audit output, and add regression coverage for both default hardening and the disabled override. After a vetted fix lands on main and ships, this issue can close as implemented.

## Risks / Open Questions

- An opt-out for `no-new-privileges` weakens sandbox hardening when disabled, so the fix should keep the secure default and make the downgrade explicit in config and security reporting.
- A silent automatic fallback would be riskier than an explicit operator opt-out because it could downgrade sandbox isolation without clear consent.
