## Summary

Keep open. The current main cleaner still does not strip JSON Schema `if`/`then`/`else`, even though the issue discussion identified those keywords as the likely Gemini 400 trigger and PR #69964 proposed exactly that fix but was closed unmerged.

## Best Possible Solution

Keep the issue open and land a small core agents fix: add `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS`, add regression coverage proving top-level and nested conditionals are stripped while ordinary properties are preserved, then separately decide whether cron/model docs should add operational guidance for `cron.failureDestination` and Gemini Flash tool-heavy workloads.
