## Summary

Current `main` does not already contain this PR's startup-only queue-owner retry behavior, and part of the PR now targets legacy ACPX config that `main` explicitly ignores. There is not strong evidence to close this as implemented or obsolete.

## Evidence

- **`queueOwnerTtlSeconds` is legacy/ignored on main:** `createAcpxRuntimeService` warns that `queueOwnerTtlSeconds` is ignored by the embedded runtime, so the PR's Windows-default TTL change is not present as an active runtime behavior on current `main`. The plugin schema/help also marks the field as a reserved compatibility field.
