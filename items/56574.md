## Summary

Keep open. Current `main` still does not implement the PR's two headline plugin fixes: per-plugin tool registration is still appended without deduplication, and plugin tool factories are still invoked without `pluginConfig` in their runtime context. One related piece from the PR is already present on `main`: default runtime requests can reuse a gateway-bindable active registry via the fallback in `src/plugins/loader.ts`.

## Evidence

- **`registerTool` still appends duplicate registrations:** `registerTool` normalizes names, pushes them into `record.toolNames`, and always appends a new `registry.tools` entry. There is no exact-match dedupe guard in current `main`.
  - file: [src/plugins/registry.ts:346](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/plugins/registry.ts#L346)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Registry tool entries still do not carry `pluginConfig`:** `PluginToolRegistration` contains `pluginId`, `pluginName`, `factory`, `names`, `optional`, `source`, and `rootDir` only; there is no stored `pluginConfig` field to pass through later.
  - file: [src/plugins/registry-types.ts:52](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/plugins/registry-types.ts#L52)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Tool factories are still called with the raw context only:** `resolvePluginTools` still invokes `entry.factory(params.context)`, so plugin tool factories on `main` do not receive `pluginConfig` through the tool context.
  - file: [src/plugins/tools.ts:136](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/plugins/tools.ts#L136)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Tool context type still lacks `pluginConfig`:** `OpenClawPluginToolContext` includes config/runtime/session fields, but no `pluginConfig` property, matching the current call site behavior.
  - file: [src/plugins/tool-types.ts:8](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/plugins/tool-types.ts#L8)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Gateway-bindable registry reuse is already on `main`:** `getCompatibleActivePluginRegistry` already has the fallback that retries the cache-key computation with `allowGatewaySubagentBinding: true` when the request is `default` and the active registry is `gateway-bindable`. This covers one part of the PR, but not the dedupe/config-passthrough changes.
  - file: [src/plugins/loader.ts:1033](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/plugins/loader.ts#L1033)
