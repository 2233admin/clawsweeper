## Summary

Keep open. Current main has not implemented this PR's central behavior: dreaming narrative session keys still include `nowMs`, the run `idempotencyKey` is still the same timestamped session key, cleanup still goes through `subagent.deleteSession`, and `sessions.delete` remains `operator.admin` scoped. The PR is also not cleanly mergeable as-is because the review discussion identifies that its synthetic-scope approach does not affect the common request-scoped client path.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up, or replace it with a narrower PR that fixes memory-core dreaming cleanup end to end: stable per-workspace/per-phase narrative keys, non-deduping idempotency keys, a request-scope-safe least-privilege cleanup authorization path for `memory-core` only, regression tests for both fallback and request-scoped clients, and removal of unrelated dependency-manifest changes.
