## Summary

Keep open. Current main still does not implement the PR's central cleanup behavior: dreaming narrative session keys remain timestamped, the narrative idempotency key still equals that timestamped session key, cleanup still relies on `subagent.deleteSession`, and gateway `sessions.delete` remains `operator.admin`-scoped. Current main did move the managed dreaming cron to isolated sessions, which mitigates some main-session/UI pollution, but it does not solve the bounded cleanup/admin-scope problem this PR is trying to fix. The PR also should not be merged as-is because the provided review discussion identifies an ineffective synthetic-scope path for request-scoped clients and the PR carries unrelated runtime-dependency file changes.

## Best Possible Solution

Keep the PR open but require changes before merge: preserve the useful stable-key/idempotency/pre-cleanup direction, fix the gateway request-scope authorization path so memory-core can clean only its own `dreaming-narrative-*` transient sessions, remove unrelated runtime-dependency edits, and add regression coverage for both fallback and request-scoped clients. If maintainers prefer a fresh patch, track the remaining bug under #68252 and close this PR only after a corrected implementation lands.
