## Summary

Keep open. Current `main` still does not implement this schema-validator improvement: `src/plugins/schema-validator.ts` formats AJV errors by reusing `error.message` plus allowed-values hints, with no `additionalProperties` branch, so extra-property failures still fall through to AJV's generic message. The targeted regression test is also still missing.

## Evidence

- **Schema-validator still lacks additionalProperties special-case:** `formatAjvErrors` on `main` resolves the path, then builds `message` from `error.message` and optional allowed-values hints only. There is no helper or branch to extract `params.additionalProperty` and rewrite the message to `unexpected property '<name>'`.
  - file: [src/plugins/schema-validator.ts:134](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/plugins/schema-validator.ts#L134)
  - command: `nl -ba src/plugins/schema-validator.ts | sed -n '120,240p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **No regression test for schema-validator additionalProperties message:** The schema-validator test file contains one `additionalProperties: false` schema in the defaults test, but the asserted coverage on `main` is for defaults, enum/const hints, required/dependencies paths, and terminal sanitization. There is no test asserting an `unexpected property '<key>'` message from `validateJsonSchemaValue`.
  - file: [src/plugins/schema-validator.test.ts:79](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/plugins/schema-validator.test.ts#L79)
