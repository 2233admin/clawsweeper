## Summary

Keep this PR open. Current main still does not implement a complete generic `message_received` / internal `message:received` emission at the active-session queued inbound boundary, and the author’s own follow-up says the branch remained incomplete after live WhatsApp testing. The PR is a plausible implementation candidate, but it needs revision rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review, but require an update or replacement PR that emits plugin `message_received` and internal `message:received` at the true inbound/enqueue boundary for active-session queued messages across channels. Add regression coverage for the follow-up drain path and explicitly reconcile the generic hook path with WhatsApp’s opt-in privacy behavior.
