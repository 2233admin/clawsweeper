## Summary

Keep open. Current `main` still couples Slack lifecycle status reactions to the ack-reaction path, so the PR addresses a real gap, and the review discussion also shows the proposed tests likely need one more assertion before merge.

## Evidence

- **Current Slack prepare logic still requires ack gating:** On `main`, `statusReactionsWillHandle` is only true when `shouldAckReaction()` is true. That means `messages.statusReactions.enabled === true` does not independently activate lifecycle reactions if ack reactions are disabled or unset for the message.

That is consistent with the PR's intent and inconsistent with the current Slack implementation.
