## Summary

Keep open. Current `main` already emits a generic doctor warning when a channel uses `groupPolicy: "allowlist"` with no `groupAllowFrom`, but it does not fully subsume this PR’s Feishu-specific validation. Feishu runtime semantics are account-merged and distinguish group IDs from sender allowlists, while the generic warning text is channel-agnostic. Given that gap, and because this PR is heavily polluted with unrelated CI/secrets/android/skill changes, there is not strong enough evidence to auto-close it under the allowed close reasons.
