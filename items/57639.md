## Summary

Keep open. Current main adds `maxSpawnDepth` and `maxChildrenPerAgent`, but there is still no lifetime/session-total subagent cap matching the requested `maxPerSession` behavior, so the feature request is not implemented on `main`.

## Evidence

- **Config surface only exposes active-child limit:** `agents.defaults.subagents` defines `maxConcurrent`, `maxSpawnDepth`, and `maxChildrenPerAgent`; there is no session-total limit field.
  - file: [src/config/types.agent-defaults.ts:392](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/config/types.agent-defaults.ts#L392)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Spawn gate checks active children only:** `sessions_spawn` rejects only when `countActiveRunsForSession(...) >= maxChildrenPerAgent`, which does not limit sequential spawns over a session lifetime.
  - file: [src/agents/subagent-spawn.ts:629](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/subagent-spawn.ts#L629)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Registry helper counts active runs, not total historical runs:** The exported counter used by the spawn gate is `countActiveRunsForSession`, confirming enforcement is scoped to currently active children.
  - file: [src/agents/subagent-registry.ts:858](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/subagent-registry.ts#L858)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Docs describe the limit as active-only:** Subagent docs explicitly say `maxChildrenPerAgent` caps active children per session and `maxConcurrent` is a concurrency safety valve.
  - file: [docs/tools/subagents.md:338](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/docs/tools/subagents.md#L338)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Tests cover only active-child enforcement:** The depth-limit test asserts rejection at `maxChildrenPerAgent` when active children already equal the cap; there is no test for a lifetime/session-total spawn ceiling.
  - file: [src/agents/subagent-spawn.depth-limits.test.ts:121](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/subagent-spawn.depth-limits.test.ts#L121)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **No `maxPerSession` or `maxTotal` implementation found:** Repository search for `maxPerSession|maxTotal` under `src`, `docs`, `packages`, and `extensions` found no subagent session-total limit setting or enforcement.
  - command: `rg -n "maxConcurrent|maxPerSession|maxTotal|subagents" src packages extensions docs -g '!**/dist/**'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)

## Risks / Open Questions

- Sequential subagent spawning can still grow session context indefinitely because current guards only limit concurrency and currently active children, not total children spawned over the life of a session.
