## Summary

Keep open. Current `main` still shows raw session keys in the TUI status surfaces this issue calls out. The gateway/session layer already exposes `label` and `displayName`, and Control UI session controls prefer them, but the TUI footer, `/status`, and TUI session picker still keep the raw cron/subagent key visible instead of surfacing a friendly cron/job/session name.

## Evidence

- **TUI footer still keeps the raw key and only appends `displayName` in parentheses:** `updateFooter()` builds `sessionLabel` from `formatSessionKey(currentSessionKey)` and then renders `raw-key (displayName)` when a display name exists. That matches the issue's reported current behavior rather than the requested replacement with a friendly cron/session label.
  - file: [src/tui/tui.ts:816](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/tui/tui.ts#L816)

- **TUI key formatter does not translate cron UUID sessions to labels:** `formatSessionKey()` only strips the `agent:<id>:` prefix via `parseAgentSessionKey(key)?.rest ?? key`; it does not consult session `label`, `displayName`, or cron job metadata. For keys like `agent:main:cron:<uuid>:run:<uuid>`, the UUID-heavy tail remains.
