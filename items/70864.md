## Summary

Keep #70864 open. Current main still does not implement scoped mention-pattern policy: config only supports the existing `mentionPatterns` array, the SDK/runtime surfaces expose only existing regex helpers, channel handlers still build mention regexes directly, and docs do not describe global/agent/provider allow/deny scopes. The PR is the active clean successor to closed dirty predecessor #70846, with review follow-ups already addressed in the provided discussion context.

## Best Possible Solution

Keep #70864 open as the active implementation path. The best maintainer path is to review the latest PR head, verify the scoped config schema, SDK/runtime helper, docs, generated metadata, and every bundled channel use the same routed-agent and conversation-id policy semantics, then merge if CI and changed-surface checks are green. Do not close as implemented, duplicate, ClawHub, or stale: current main lacks the requested capability and #70846 is only the closed dirty predecessor.
