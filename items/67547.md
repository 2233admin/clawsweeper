## Summary

Keep open. Current `main` still has install-layout-sensitive A2UI bundling logic, so the underlying reproducibility bug is not convincingly fixed; what changed since `v2026.4.23` is the committed A2UI artifact/hash, not the generator/config code that the report identified.

## Evidence

- **Rolldown config still prefers `ui/node_modules` over repo root:** `resolveUiDependency()` checks `ui/node_modules` first and only falls back to repo-root `node_modules`, so the generated bundle can still differ by install layout whenever A2UI is rebuilt.
