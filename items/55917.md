## Summary

Keep open: current `main` still has the reported Telegram placeholder-only behavior. The inbound media resolver intentionally returns `null` for Telegram Bot API `file is too big` responses (>20MB) and for exhausted `getFile` retries, the handler then proceeds with `allMedia = []`, and the inbound body builder falls back to `<media:document>`/`[file_id:...]` placeholder text instead of a saved attachment path.

## Evidence

- **Current resolver still drops attachments on Telegram Bot API >20MB or repeated `getFile` failure:** `resolveTelegramFileWithRetry` explicitly documents Telegram Bot API's 20MB download limit, returns `null` for `file is too big`, and also returns `null` after retry exhaustion so the message can continue with a type-based placeholder instead of an attachment.

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
