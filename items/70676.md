## Summary

Keep open. Current `main` still has the three silent revocation catches the PR targets, the device-removal/admin paths still depend on that code, and current tests cover normal 4001 disconnects but not `socket.close()` throwing. The PR discussion also shows the only concrete review finding was addressed on the PR branch, so there is not strong evidence for an auto-close.
