## Summary

Keep #70676 open. Current main still has the silent WebSocket revocation close-failure behavior the PR targets: shared-auth and device-token revocation paths catch `socket.close()` errors and ignore them, with no warning and no `terminate()` fallback. The PR is a focused gateway bug/security-hardening candidate, not obsolete or superseded by current main.

## Best Possible Solution

Keep the PR open for maintainer review, or land an equivalent focused fix: add warning logs and `socket.terminate()` fallback for shared-auth and device-token revocation close failures, thread binding-safe loggers through the shared-auth call sites, and add unit tests for throwing `close()`, throwing `terminate()`, and loop continuation across remaining clients.
