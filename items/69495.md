## Summary

Keep PR #69495 open. Current main still does not implement heartbeat `{ primary, fallbacks }`: heartbeat schema/types/generated schema/docs remain string-only, the heartbeat runner only resolves a single string override, and the reply/fallback pipeline has no heartbeat fallback override field. The PR direction is still relevant, but it should address the existing review concern about preserving explicit `fallbacks: []` before merge.

## Best Possible Solution

Keep PR #69495 open or replace it with an equivalent maintainer patch. The right implementation is to make `agents.*.heartbeat.model` use `AgentModelConfig` in types, Zod, generated schema, help/labels/docs; resolve primary and ordered fallbacks in `runHeartbeatOnce`; carry a defined fallback override, including `[]`, through `GetReplyOptions` and `FollowupRun.run.modelFallbacksOverride`; prefer that override in both fallback resolution paths; update model pricing collection; and add regression coverage for object form, no-fallbacks field, explicit empty fallbacks, per-agent/default merging, and string-form compatibility.
