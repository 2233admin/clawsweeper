## Summary

Keep PR #69495 open. Current main still has the heartbeat fallback gap: `agents.*.heartbeat.model` remains string-only in runtime schema/types/generated schema/docs, and heartbeat execution only passes a single `heartbeatModelOverride` string into the reply pipeline. The existing model fallback engine already supports explicit fallback overrides, including empty arrays, so this PR is still aimed at a real missing plumbing layer, though its review comment about preserving `fallbacks: []` semantics should be handled before merge.

## Best Possible Solution

Keep #69495 open or replace it with an equivalent maintainer patch that makes `agents.defaults.heartbeat.model` and `agents.list[].heartbeat.model` use `AgentModelConfig`, updates generated schema/help/docs/pricing collection, carries heartbeat fallbacks through `GetReplyOptions` into `FollowupRun.run.modelFallbacksOverride`, preserves explicit empty-array override semantics, and adds focused heartbeat runner/reply fallback tests.
