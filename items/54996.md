## Summary

Keep this issue open. Current main has partial local transcript estimation for compaction preflight and session forking, but the requested shared estimated context snapshot is still missing from `/status`, `/usage`, and session row usage surfaces when provider usage is missing or all-zero.

## Best Possible Solution

Implement a shared estimated context snapshot path for sessions with missing or all-zero provider usage. Reuse `readSessionMessages` with `estimateMessagesTokens` or the existing character/token estimator, carry an explicit estimated source through session rows/events, render it clearly in `/status`, `/usage`, and context notices, and let compaction thresholds consume the estimate without overwriting fresh provider snapshots. Add regression coverage for MiniMax-style all-zero usage and transcripts that contain messages but no assistant usage metadata.

## Risks / Open Questions

- A fix should keep provider-reported fresh snapshots authoritative and carry a distinct estimated source/flag instead of marking heuristic totals as provider-fresh usage.
- Transcript-message estimates can miss provider tokenization, hidden prompts, system/tool schema overhead, and provider-specific formatting, so compaction thresholds should retain conservative headroom.
