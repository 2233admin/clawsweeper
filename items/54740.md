## Summary

Keep open. Current main still has the central doctor/status false-positive problem: the Darwin LaunchAgent scanner can report non-gateway OpenClaw ecosystem plists as gateway-like, and doctor/status still print static cleanup hints for the profile gateway service whenever any extra service is found. The gateway.mode source subclaim looks mostly obsolete, but the noisy LaunchAgent and unsafe cleanup-hint behavior remains. Related #39118 and #52515 are implementation candidates, not merged fixes.

## Best Possible Solution

Keep this issue open. Fix src/daemon/inspect.ts so Darwin LaunchAgent detection requires an explicit gateway service marker or parsed ProgramArguments that actually invoke the gateway, and make doctor/status cleanup hints target only detected stale/inactive extra services while suppressing hints for the loaded/running active gateway. Review and land or close #39118 and #52515, then add fixture tests for non-gateway OpenClaw ecosystem plists and active-gateway cleanup suppression.
