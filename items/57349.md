## Summary

Keep open. Current main has meaningful memory mitigations, including session-store maintenance and agent-run context cleanup, but I did not find a source-backed fix for this issue’s broader gateway heap exhaustion report. The latest comments make the report broader than the closed Discord stale-socket issue and broader than the sessions.json-only tracker: users report OOM under webchat polling, later under reduced plugins, and later on Telegram-only heavy tool/compaction sessions.

## Best Possible Solution

Keep #57349 open as the broader gateway heap-exhaustion tracker. Maintainers should link it to #55334 for the sessions.json/session-store portion, but not close it as a duplicate unless the remaining Telegram-only compaction/session-runtime leak is split into a narrower canonical issue. The best fix path is to add heap snapshot/diagnostics guidance, capture retained-object evidence from a current build, then address the proven hot path: session-store eager materialization, transcript/history full reads, compaction checkpoint/session state retention, or browser/CDP cleanup as indicated by profiling.
