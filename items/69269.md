## Summary

Keep #69269 open. Current main still does not preserve the full compaction/reset invariant cluster described in the issue: reset hook contexts omit messageProvider, engine-owned compaction hook context still uses only raw compaction params without a shared resolver, and the auto-compaction accounting path ignores tokensAfter === 0. The related PR #69270 in the provided GitHub context is still open and directly targets these remaining gaps.

## Best Possible Solution

Keep the issue open and finish the coordinated fix already scoped by #69270: add one shared messageProvider resolver for compaction/reset hooks, use it in command reset, gateway reset, and engine-owned compaction hook contexts, treat tokensAfter === 0 as a valid fresh accounting result, clear stale cost/breakdown fields consistently, and add regression tests for mixed-case providers, non-channel keys, gateway/manual reset hooks, and zero-token compaction outcomes.
