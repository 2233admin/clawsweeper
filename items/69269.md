## Summary

Keep #69269 open. Current main still shows the reported invariant gaps: compaction hook contexts pass raw `messageChannel ?? messageProvider` rather than a normalized/inferred provider, reset hook contexts still omit `messageProvider`, and the shared compaction-count/token update path ignores `tokensAfter: 0`, leaving stale totals possible. The linked #69270 PR appears to be the canonical coordinated fix, but it is still open in the provided GitHub context, so the issue should remain open until that work or an equivalent lands.

## Best Possible Solution

Keep #69269 open and land a coordinated fix like #69270: add a shared hook-message-provider resolver that normalizes explicit providers, infers only real deliverable channel providers from channel-scoped session keys, leaves non-channel keys unresolved, uses it in auto/direct compaction, engine-owned queued compaction, command reset hooks, and gateway `sessions.reset`, and updates compaction bookkeeping so `tokensAfter: 0` resets `totalTokens`, marks `totalTokensFresh`, and clears stale cost/breakdown fields. Add regression tests for mixed-case providers, channel-scoped session-key inference, non-channel keys, command/gateway reset contexts, and zero-token compaction accounting.
