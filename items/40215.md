## Summary

Keep this issue open. Current main still only appends per-response token/cache/cost data plus the session key in `/usage full`; it does not include cumulative context usage, context window percentage, model metadata, or an inherited `agents.defaults.responseUsage` default. The provided GitHub context also shows open PR #69618 with closing intent for this exact request, so the issue should remain open while that implementation is reviewed or replaced.

## Best Possible Solution

Keep this issue open until #69618, or a smaller replacement, is reviewed and either landed or closed. The eventual implementation should enrich `/usage full` with model and cumulative context/window/percentage data, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover formatter, runner, command, config schema, docs, and reply-composition behavior.
