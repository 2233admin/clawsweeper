## Summary

Keep open. Current `main` still does not treat Telegram HTTP 421 as retryable for outbound sends, and the docs/tests on `main` still describe and verify the narrower retry set.

## Evidence

- **Non-idempotent Telegram sends use a strict retry predicate:** `createTelegramNonIdempotentRequestWithDiag` wires outbound send retries to `isSafeToRetrySendError(err) || isTelegramRateLimitError(err)` with `strictShouldRetry: true`, so generic channel retry regex fallback is disabled for sendMessage/sendPhoto-style calls.
  - file: [extensions/telegram/src/send.ts:599](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/telegram/src/send.ts#L599)
  - command: `nl -ba extensions/telegram/src/send.ts | sed -n '500,620p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Safe-to-retry send errors exclude HTTP status handling like 421:** `isSafeToRetrySendError` only accepts the pre-connect codes in `PRE_CONNECT_ERROR_CODES`: `ECONNREFUSED`, `ENOTFOUND`, `EAI_AGAIN`, `ENETUNREACH`, and `EHOSTUNREACH`. A Telegram `421 Misdirected Request` is not included in that set.
  - file: [extensions/telegram/src/network-errors.ts:30](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/telegram/src/network-errors.ts#L30)
  - command: `nl -ba extensions/telegram/src/network-errors.ts | sed -n '24,220p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Existing Telegram status helpers still exclude 421:** `isTelegramRateLimitError` only matches `429`, and `isTelegramServerError` only matches `>=500`. There is no separate `421` handling on current `main`.
  - file: [extensions/telegram/src/network-errors.ts:218](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/telegram/src/network-errors.ts#L218)
  - command: `nl -ba extensions/telegram/src/network-errors.ts | sed -n '218,235p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Tests still reflect the narrower retry policy:** Current send tests cover retrying pre-connect failures and 429-style `retry_after`, and explicitly assert no retry for non-transient send errors / generic grammY failed-after envelopes. There is no 421-specific test on `main`.
  - file: [extensions/telegram/src/send.test.ts:1038](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/telegram/src/send.test.ts#L1038)
