## Summary

Keep open. Current `main` still applies a raw-file `$[A-Z_][A-Z0-9_]{1,}` scan during exec preflight with no Python string-literal/docstring/help-text awareness, so the reported `$SIM` false positive class remains present. The report appears to overstate lowercase f-string cases like `$cost`, but the uppercase literal cases are still a real bug and no merged fix is visible on `main`.
