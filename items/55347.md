## Summary

Keep open. Current `main` has adjacent recovery work for invalid config and restart notifications, but it does not implement the requested native crash watchdog with configurable retry policy, restart interval, or `gateway.selfHeal.*` config. The issue timeline also points to open PR #64684, so this area is active rather than obsolete.

## Evidence

- **Startup invalid-config recovery exists, but only as partial coverage:** Gateway startup restores a last-known-good config snapshot when the config is invalid and enqueues a recovery notice. That addresses one failure mode from the issue, but it is not a general self-healing crash monitor.
  - file: [src/gateway/server-startup-config.ts:73](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/gateway/server-startup-config.ts#L73)
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Manual restart path validates config before restart:** `openclaw gateway restart` aborts on invalid config and points users to `openclaw doctor`, which is restart-time validation for explicit restarts, not automatic crash-loop recovery.
  - file: [src/cli/daemon-cli/lifecycle-core.ts:390](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/cli/daemon-cli/lifecycle-core.ts#L390)
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Restart/report delivery plumbing already exists in limited form:** Restart sentinel payloads already support restart/update/config-apply notices plus captured delivery context and thread IDs, so part of the proposed 'recovery brief' surface exists, but only for known restart flows rather than native crash supervision.
  - file: [src/infra/restart-sentinel.ts:41](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/infra/restart-sentinel.ts#L41)
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Docs still anchor crash recovery on the service manager:** The Gateway runbook still documents systemd `Restart=always` as the crash-restart mechanism, which matches the issue's claim that OpenClaw currently relies on OS/service-manager supervision rather than a native watchdog.
  - file: [docs/gateway/index.md:274](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/docs/gateway/index.md#L274)
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Requested config surface is absent on main:** A repo search found no `gateway.selfHeal`, `configValidateOnRestart`, `maxRestartAttempts`, or `restartIntervalSeconds` fields in current code or docs.
  - command: `rg -n "selfHeal|configValidateOnRestart|maxRestartAttempts|restartIntervalSeconds" src docs`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Related work is still open:** The issue timeline cross-references open PR #64684 (`Run doctor automatically on restart and update failure paths`), which suggests the topic is under active development rather than an obsolete or out-of-scope request.
