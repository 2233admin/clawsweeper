## Summary

Current main still matches the reported false-positive path. `gateway probe` can probe both an SSH tunnel target and the configured remote URL, removes only exact URL duplicates, then emits `multiple_gateways` solely from the count of reachable targets. The existing `self` presence identity is extracted but not used for warning de-duplication, and the related ClawSweeper items cover different gateway-probe timeout/scope false-negative behavior rather than this same-gateway multi-transport warning.

## Best Possible Solution

Keep the issue open. The likely fix belongs in `src/commands/gateway-status/output.ts`: group reachable probes by a stable gateway identity derived from `self` presence when available, suppress or downgrade the warning when all reachable transports identify the same gateway, and keep `multiple_gateways` for distinct identities or ambiguous cases. Add focused tests for `sshTunnel` plus `configRemote` sharing one identity and for two distinct identities still warning, then update CLI/gateway docs if warning copy changes.
