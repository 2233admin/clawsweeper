## Summary

Current `main` is only a partial match for PR #51930. Discovery already prefers `openclaw.plugin.json` ids, so the PR's discovery half is largely redundant, but the install-path warning suppression for the `@openclawbrain/openclaw` -> `openclawbrain` alias is not implemented on `main`. The existing install match logic only treats exact package-name matches or simple unscoped-name matches as equivalent, so this PR is not implemented on `main` and should stay open.
