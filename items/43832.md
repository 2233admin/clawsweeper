## Summary

Keep open. Current `main` still reloads cron with a synchronous `stop()` that only clears the timer, while cron timer/manual-run paths still do work outside the lock and can persist later; the PR addresses a real bug, and the latest discussion still contains a plausible unresolved regression (unbounded graceful-drain blocking reload), so there is no strong basis to auto-close it.

## Evidence
