## Summary

Current `main` does not implement the requested live browser preview panel in the dashboard. OpenClaw has generic live chat tool cards and browser primitives for screenshots/snapshots/console inspection, but the browser tool still returns one-off images or wrapped text output rather than a live embedded browser panel with URL/title/console streaming.

## Evidence

  - file: [docs/web/control-ui.md:105](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/docs/web/control-ui.md#L105)

  - file: [extensions/browser/src/browser-tool.ts:596](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/extensions/browser/src/browser-tool.ts#L596)

## Risks / Open Questions

- There is partial overlap with existing functionality: operators can inspect browser state through chat tool output plus explicit `browser.screenshot` / `browser.console` calls, but that is not the requested always-on live dashboard panel.
- The repo has generic canvas-preview infrastructure, so future work could implement this without a brand-new UI primitive; current main just does not wire the browser tool into that path.
