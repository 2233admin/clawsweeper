## Summary

Keep #69629 open. Current main protects execution and routing for Discord native slash commands, but the reported UI visibility problem is still present: docs explicitly say commands may remain visible in Discord UI for unauthorized users, config only exposes native/nativeSkills toggles, and the Discord startup/deploy path does not reconcile per-guild/per-channel command visibility or permissions.

## Best Possible Solution

Keep the issue open as a Discord plugin feature request. A maintainer-grade fix should live in the bundled Discord plugin, first verifying the current Discord application-command permission model, then adding a documented config shape that can reconcile native command visibility against channels.discord.guilds.*.channels for each account where Discord supports it. If Discord cannot reliably hide commands per channel, document that platform limitation and improve the existing workaround guidance for per-account commands.native=false plus text commands/server isolation.
