## Summary

Keep #69629 open. Current main protects Discord native slash command execution and routing, but it does not implement the requested per-channel Discord command-palette visibility for multi-bot setups. The docs explicitly acknowledge commands may remain visible in Discord UI for unauthorized users, and the Discord deploy/config paths do not expose or reconcile per-guild/per-channel command visibility or command permissions.

## Best Possible Solution

Keep this open as a bundled Discord plugin feature request. A good fix should first verify the current Discord application-command permission model, then add a documented config shape that can reconcile native command visibility against channels.discord.guilds.*.channels per account where Discord supports it. If Discord cannot reliably hide commands per channel, document that platform limitation and improve workaround guidance for per-account channels.discord.commands.native=false, text commands, or server isolation.
