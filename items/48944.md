## Summary

Keep open. Current `main` still skips sub-1024-byte audio before transcription and only injects agent-facing audio context when a real transcription output exists, so the reported silent/empty voice-note gap is still present.

## Evidence

- **Tiny audio is still skipped with no transcript output:** The shared media-understanding runner throws a skip error for audio smaller than `MIN_AUDIO_FILE_BYTES`, with reason `tooSmall`, instead of producing a placeholder transcript.
  - file: [src/media-understanding/runner.entries.ts:494](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/media-understanding/runner.entries.ts#L494)

- **Regression tests still assert empty outputs for tiny audio:** Current tests expect `result.outputs` to be empty and the outcome to be `skipped` for tiny/empty audio, which matches the issue rather than a fix.
  - file: [src/media-understanding/runner.skip-tiny-audio.test.ts:81](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/media-understanding/runner.skip-tiny-audio.test.ts#L81)
