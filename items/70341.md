## Summary

Keep #70341 open. Current main still lacks the PR's narrower PowerShell-cmdlet allowlist behavior: unresolved bare cmdlets cannot match normal path-backed allowlist entries, and allow-always derivation skips commands without a candidate executable path. The PR targets a real remaining gap, but it also needs the review-identified execution-path fix before merge because allowing a bare cmdlet can make Windows execution try to spawn `Get-ChildItem` directly instead of preserving the PowerShell host.

## Best Possible Solution

Keep the PR open and request changes. The best fix should add the PowerShell cmdlet allowlist fallback and allow-always pattern generation, but also preserve or reconstruct the PowerShell host for Windows shell-payload execution so OpenClaw does not try to spawn `Get-ChildItem` directly. Add focused regression coverage in both `src/infra` allowlist tests and `src/node-host` system-run execution tests.
