## Summary

Keep #70341 open. Current main still has no PowerShell-cmdlet allowlist fallback: unresolved bare cmdlets cannot match normal allowlist entries or produce PowerShell-path allow-always entries. The PR targets a real remaining gap from #52952, but it also needs the execution-path issue from review addressed before merge because marking a bare cmdlet allowlisted can make Windows execution try to spawn the cmdlet token directly.

## Best Possible Solution

Keep the PR open and request changes. The right fix should add the PowerShell cmdlet allowlist fallback and allow-always pattern generation, but also preserve or reconstruct the PowerShell host for Windows shell-payload execution so OpenClaw does not try to spawn `Get-ChildItem` directly. Add focused regression coverage in both `src/infra` allowlist tests and `src/node-host` system-run execution tests.
