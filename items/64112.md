## Summary

Keep open. Current main has related manifest command-alias diagnostics, but it has not implemented this PR's central bundled-plugin-root guard. With a restrictive plugins.allow, resolveMissingPluginCommandMessage still returns a plugin allowlist message for any missing primary command string, so the linked openclaw run false-fallback case is not clearly obsolete or superseded.

## Best Possible Solution

Keep #64112 open. The best fix is to adapt its bundled-plugin-root probing to current main so plugin allowlist/disablement diagnostics only appear for real bundled plugin CLI roots or manifest-owned command aliases, while arbitrary unknown commands such as openclaw run continue to the normal unknown-command path. Preserve the alias and denylist handling from the PR follow-up commits and add a regression test for the linked #63907 scenario.
