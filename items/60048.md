## Summary

Keep open. Current main still lacks the proposed `docs/concepts/memory-deep-dive.md` page, and the related issue #43032 was closed specifically in favor of this PR as the canonical follow-up. The PR is useful docs work, but it still needs factual reconciliation around daily memory injection before it is safe to land.

## Best Possible Solution

Keep PR #60048 open as the canonical docs follow-up for #43032. Update the proposed guide to say daily memory is on-demand on ordinary turns, while recent daily files can be runtime-preloaded for bare `/new` and `/reset` via `agents.defaults.startupContext`; then re-check the memory-search/backend defaults and land through normal docs review.
