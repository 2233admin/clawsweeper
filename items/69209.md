## Summary

Keep open. Current main still does not expose `plugins.entries.memory-core.config.dreaming.model` in the memory-core manifest, does not read top-level `dreaming.model` into the resolved dreaming defaults, and does not pass a model override into Dream Diary subagent runs. The PR is a focused implementation for a real missing behavior linked from #67409; the PRtags mention of #66583 is not enough to close this as a duplicate without a readable canonical PR/state.

## Best Possible Solution

Keep this open unless maintainers identify a readable canonical duplicate. The best product path is to land one canonical implementation that adds `dreaming.model` to the memory-core manifest/docs, resolves it through `MemoryDreamingExecutionConfig`, forwards it to light/REM/deep Dream Diary subagent runs, and retries only for configured-model startup/wait/authorization failures with tests covering unset, configured, and fallback behavior.
