## Summary

Keep open. Current main still does not implement the requested user-facing `plugins.entries.memory-core.config.dreaming.model` override, and PR #69209 remains a plausible implementation path rather than an obsolete or safely closeable item. The related PRtags note identifies another open duplicate PR (#66583), but the provided context does not establish a canonical merged/superseding item, and the shipped code/docs still lack the behavior.

## Best Possible Solution

Keep #69209 open unless maintainers explicitly choose a canonical duplicate. The product path is to land one focused implementation that accepts `plugins.entries.memory-core.config.dreaming.model`, resolves it in the memory host config, passes it into light/REM/deep Dream Diary subagent runs, retries with the session default only for intended model-override/startup failures, and updates the memory-core manifest, tests, and public Dreaming/memory config docs.
