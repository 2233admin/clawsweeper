## Summary

Keep open. Current main at 6d60b035b4e7c1739f038808e041567448f58060 still does not implement the PR's central feature: a user-facing `plugins.entries.memory-core.config.dreaming.model` override for Dream Diary narrative runs. The manifest rejects that top-level key, host config resolution only reads `model` from nested `dreaming.execution.*` records, and the narrative subagent call sites do not pass a model override for light, REM, or deep dreaming. The PR discussion and related issue #67409 describe a real product gap, and the provided duplicate context is not strong enough to close this PR as superseded by a shipped or verified canonical PR.

## Best Possible Solution

Keep the PR open or explicitly consolidate it only after maintainers verify a canonical duplicate PR. The product/code path should add `plugins.entries.memory-core.config.dreaming.model` to the memory-core manifest, resolve it through `src/memory-host-sdk/dreaming.ts`, pass it into all Dream Diary narrative subagent runs, and cover configured-model fallback behavior with host-SDK and memory-core tests.
