## Summary

Current `main` only partially matches the request. The webchat UI now shows grouped, human-readable session labels in the selector, and the gateway supports creating labeled dashboard sessions, but the Control UI still does not appear to create additional webchat/dashboard sessions from the requested `+ New` flow. The visible `New session` button sends `/new` on the current session key, which resets/rolls that session instead of creating a separate parallel labeled session.

## Evidence

- **Selector shows labeled sessions:** The chat header renders a grouped session `<select>` and resolves option labels from `row.label` / `row.displayName`, including duplicate-label disambiguation.
  - file: [ui/src/ui/chat/session-controls.ts:23](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/ui/src/ui/chat/session-controls.ts#L23)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Label preference is explicitly tested:** Tests verify the selector prefers grouped session labels over raw display names and disambiguates duplicates.
  - file: [ui/src/ui/app-render.helpers.node.test.ts:408](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/ui/src/ui/app-render.helpers.node.test.ts#L408)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Visible New session button does not create a second session:** The chat toolbar exposes a `New session` button, but its handler is wired to `onNewSession` only.
  - file: [ui/src/ui/chat/run-controls.ts:24](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/ui/src/ui/chat/run-controls.ts#L24)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)

  - file: [ui/src/ui/app-chat.ts:408](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/ui/src/ui/app-chat.ts#L408)

- **Creation path is not wired from Control UI:** A repo search found `sessions.create` references in gateway code/tests, but no caller under `ui/src/ui`, so current webchat UI evidence does not show an end-user path to create extra labeled webchat sessions.
  - command: `rg -n "sessions.create" ui/src/ui src --glob '!**/node_modules/**'`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
