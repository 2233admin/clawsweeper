## Summary

Keep open. Current main still does not implement the PR's two requested behaviors: `optimizeImageWithFallback` sends non-alpha PNG images through JPEG optimization even when the original PNG is already under the cap, and `loadWebMediaInternal` only renames JPEG outputs to `.jpg` for HEIC sources. The PR is a focused bug fix with tests, but it likely needs rebasing from the old `src/web/media.*` path to the current `src/media/web-media.*` files and should address the existing review concern about validating the PNG fast path.

## Best Possible Solution

Keep the PR open, rebase it onto current `src/media/web-media.*`, preserve validated under-cap PNGs without JPEG transcoding, normalize filenames whenever optimization actually emits JPEG, and add focused regression tests for both behaviors while addressing the PNG validation review comment.
