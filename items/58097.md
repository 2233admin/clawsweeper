## Summary

Keep open. Current main still has the exact Nextcloud Talk signature-verification behavior this PR addresses: an early return on signature-length mismatch followed by a hand-rolled XOR comparison. The PR is focused on a bundled webhook-auth path, has supportive review discussion, and is not implemented or safely superseded on main.

## Best Possible Solution

Keep this PR open or explicitly choose between it and the older related Nextcloud Talk hardening PR. The desired code path should replace the current early-return/manual-XOR verifier with a Node `crypto.timingSafeEqual`-based comparison that runs on length mismatches and add a focused regression test proving the compare path still executes for truncated signatures.
