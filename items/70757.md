## Summary

Keep open. Current `main` still authenticates BlueBubbles webhooks with `channels.bluebubbles.password`, still documents that password in the public webhook URL, and does not expose or validate a separate `webhookSecret`. The PR remains materially useful and its review thread shows the author addressed the reported follow-up gaps in commit `77a007f065f52589fd091fc07d417aa20a36f596`, but that work is not present on `main` in this checkout.
