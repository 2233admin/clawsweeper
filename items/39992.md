## Summary

Current main still lacks the requested `openclaw doctor` warning for provider-owned/synthetic runtime catalog models that are available but absent from a non-empty `agents.defaults.models` allowlist. The direct PR #46235 is closed unmerged in the provided context, and the only current doctor model allowlist check is still the narrow `hooks.gmail.model` validation.

## Best Possible Solution

Keep #39992 open. Implement a read-only doctor health contribution in core that loads the same runtime model catalog used by model listing/gateway surfaces, identifies provider-owned supplemental or forward-compatible catalog rows through a stable marker/contract, compares them with a non-empty `agents.defaults.models` allowlist, and warns with concrete model IDs plus `openclaw models allowlist add <model-id>` guidance. It should skip when no allowlist is configured, never auto-modify config, and include focused tests for no allowlist, empty allowlist, no gaps, detected gaps, provider marker behavior, and warning wording.
