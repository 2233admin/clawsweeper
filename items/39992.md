## Summary

Keep open. Current main still lacks the requested `openclaw doctor` diagnostic that compares available runtime/catalog synthetic or provider-owned model rows against an active `agents.defaults.models` allowlist. Main has adjacent behavior for hooks model validation, model-list visibility for already-allowlisted missing rows, and Codex route warnings, but none of those warn operators about available models hidden by the allowlist. Related PR #46235 attempted this exact feature and was closed unmerged per the provided GitHub context.

## Best Possible Solution

Implement a read-only doctor health contribution in core that loads the same runtime model catalog used by gateway/model-list surfaces, identifies provider-owned synthetic or forward-compatible rows through a stable marker or documented equivalent, compares them against a non-empty `agents.defaults.models` allowlist, and warns with concrete model ids plus current allowlist-edit guidance. It should skip when no allowlist is configured, never modify config automatically, and include focused tests for no allowlist, empty allowlist, all synthetic rows already allowed, gap detection, non-synthetic/provider filtering, and message wording. The follow-up cost-profile concern should be either a separate doctor warning or an explicit extension of this check if maintainers want that product behavior.
