## Summary

Keep open. Current main has useful bundled runtime-dependency hardening, external staging, and Docker update coverage, but the broad #72058 npm update/runtime-deps loop is not fully closed: the postinstall path still treats an existing package.json sentinel as satisfied without checking the installed version, the linked stale-version repair PR #72084 is still open in the provided context, and #72114 still tracks the cold-cache/shutdown-race variant.

## Best Possible Solution

Keep #72058 open until maintainers land #72084 or an equivalent regression-tested stale bundled runtime-deps reinstall fix, then decide whether #72114 remains the canonical tracker for the cold-cache/update shutdown race. If both paths are covered on main, close #72058 as implemented with the shipped commit and release; otherwise close only if a clearly canonical open issue supersedes all remaining behavior.
