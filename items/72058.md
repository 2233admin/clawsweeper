## Summary

Keep open. Current main has post-v2026.4.24 bundled runtime-dependency fixes, but the issue is not safe to close: the linked fix PR #72084 is still open, #72114 still tracks related update/runtime-deps failures, and current postinstall logic still lacks stale installed-version repair coverage.

## Best Possible Solution

Keep #72058 open until an equivalent of #72084 lands or maintainers consolidate it into a canonical open tracker. The fix should validate stale or mismatched installed bundled runtime dependency versions during npm updates/postinstall, preserve the current external stage-dir behavior, and include regression coverage plus a packaged npm/global update smoke for doctor/gateway startup.

## Evidence

- **Multiple current-release repros in issue context:** Issue #72058 reports OpenClaw 2026.4.24 npm/global update failures with Telegram/TUI unusable and repeated bundled runtime dependency installs. Follow-up comments from voytas75, stemkat100, and ACC-Tony report similar 2026.4.24 packaged/global-install runtime-deps symptoms, including repeated staging and package-root ESM resolution gaps.
- **Linked PR is not merged:** Provided GitHub context shows PR #72084, 'postinstall: reinstall stale bundled runtime deps after npm updates', is open and explicitly says 'Closes #72058'. Its proposed fix and regression test are not evidence that current main has shipped the repair.
- **Current postinstall still only treats missing sentinel or missing native optional child as install triggers:** `runtimeDepNeedsInstall()` returns true when the sentinel package.json is missing, when a platform-targeted optional native child is missing, or when reading the sentinel fails. It does not compare the installed package version against the required bundled runtime dep spec, which is the repair direction described by open PR #72084.
