## Summary

Keep #70821 open. Current main still lacks the PR's Amazon Bedrock Opus 4.7 `xhigh` thinking profile advertisement, while the shared thinking and Anthropic transport paths already support `xhigh` for Opus 4.7. The PR is a narrow, still-relevant bug fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and land a narrow Bedrock-owned fix: add an Opus 4.7-specific matcher in `extensions/amazon-bedrock/register.sync.runtime.ts`, expose `xhigh` only for matching Bedrock Opus 4.7 model IDs, and keep the regression test in `extensions/amazon-bedrock/index.test.ts` asserting positive Opus 4.7 and negative Opus 4.6/Sonnet cases. Validate with the Amazon Bedrock extension tests and the changed gate before merge.
