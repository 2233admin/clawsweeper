## Summary

Keep open. Current `main` has the earlier crash fix and new WhatsApp stale-state diagnostics, but it still does not show a clear fix for the reported connected-but-silent wedge: gateway stale-socket recovery only works when a channel reports transport liveness, and the WhatsApp runtime status on `main` still does not publish `lastTransportActivityAt`.

## Evidence

- **Earlier crash was fixed, matching the reporter's note, but this is a different failure mode:** `CHANGELOG.md` records the shipped fix for `"Agent listener invoked outside active run"`, which matches the issue body's statement that the old hard crash was fixed before this silent-freeze report.
  - file: [CHANGELOG.md:1180](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/CHANGELOG.md#L1180)

  - file: [src/gateway/channel-health-policy.ts:115](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/gateway/channel-health-policy.ts#L115)
