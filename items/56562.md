## Summary

Keep open. Current `main` has partial hard-coded required-param checks for a few coding tools, but the requested generic `tools.validate_before_call` / `tools.retry_on_missing_params` config is not present, and the tool adapter still just returns an error result rather than automatically retrying the model.

## Evidence

- **Requested config keys are absent:** A repo search for `validate_before_call` and `retry_on_missing_params` returned no matches in `src/` or `docs/`.
  - command: `rg -n "validate_before_call|retry_on_missing_params" src docs`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Runtime tools schema does not define the feature:** `AgentToolsSchema` defines `elevated`, `exec`, `fs`, `loopDetection`, and `sandbox`; there is no `validate_before_call` or `retry_on_missing_params` field.
  - file: [src/config/zod-schema.agent-runtime.ts:594](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/config/zod-schema.agent-runtime.ts#L594)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Public tool config types/docs only expose loop detection:** `AgentToolsConfig` includes `loopDetection` but no validation/retry knobs, and the user docs section documents `tools.loopDetection` without the requested options.
  - file: [src/config/types.tools.ts:304](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/config/types.tools.ts#L304)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Existing validation is narrow and hard-coded:** Required-param validation is implemented via `REQUIRED_PARAM_GROUPS` for only `read`, `write`, and `edit`, and `wrapToolParamValidation()` simply checks those groups before calling the tool.
  - file: [src/agents/pi-tools.params.ts:75](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/pi-tools.params.ts#L75)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Current error path does not auto-retry the model:** `toToolDefinitions()` catches tool execution errors, logs them, and returns a structured error result; there is no automatic retry loop or retry-count handling in this path.
  - file: [src/agents/pi-tool-definition-adapter.ts:226](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/pi-tool-definition-adapter.ts#L226)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Tests confirm only partial behavior:** Tests assert rejection of malformed params for `read`/`write`/`edit` and include the guidance text `Supply correct parameters before retrying`, which shows the current behavior is tool-specific rather than generic schema-driven retry/config support.
  - file: [src/agents/pi-tools.create-openclaw-coding-tools.test.ts:578](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/pi-tools.create-openclaw-coding-tools.test.ts#L578)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
