## Summary

Keep open. Current main still hard-codes the TUI footer, the public plugin API/docs expose no `registerStatusProvider` or equivalent footer contribution seam, and the related ContextClaw/PR discussion shows a concrete plugin API gap rather than an item that can be closed as implemented or ClawHub-only.

## Best Possible Solution

Keep the issue open unless maintainers deliberately consolidate it with a canonical TUI footer extensibility tracker. The best product path is a small documented plugin API for footer/status contributions, wired through the plugin registry/SDK and `src/tui/tui.ts`, with clear polling/caching/error behavior and tests covering plugin-provided footer entries.
