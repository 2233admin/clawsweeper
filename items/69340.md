## Summary

Keep open. Current main still has the underscore-only copy-sensitive token guard in the TUI formatter, so hyphen-separated long package/component names are still split with inserted spaces. The PR is a focused one-line fix for linked bug #48432, but should add a regression test before landing.

## Best Possible Solution

Keep PR #69340 open. The best path is to update it with a focused `sanitizeRenderableText` regression test for a long hyphen-separated token, keep the existing path/URL/credential protections intact, and then run the targeted TUI formatter tests. Maintainers can separately decide whether the same hyphen preservation should be applied to `wrapNoteMessage`.
