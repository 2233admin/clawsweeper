## Summary

Keep PR #69340 open. Current main still has the exact TUI sanitizer gap described by the PR: `FILE_LIKE_RE` accepts hyphens, but `isCopySensitiveToken` only preserves file-like tokens when they contain underscores, so long hyphen-separated identifiers can still be chunked with inserted spaces. The PR is a narrow fix for still-open bug #48432 and is not implemented or clearly superseded on main.

## Best Possible Solution

Keep #69340 open and finish it as the narrow TUI sanitizer fix for #48432, preferably by adding a regression test that `sanitizeRenderableText` preserves a long hyphen-separated identifier verbatim. If maintainers later consolidate it with the broader related sanitizer work in #39565, that PR should explicitly cover this hyphen-only case before #69340 is closed as superseded.
