## Summary

Keep open. Current `main` still supports the QMD backend, and the active `QmdMemoryManager` path does not subscribe to transcript-update events or watch session collections, so the PR’s stated behavior is not already implemented there. The PR discussion also includes an unresolved P1 review note about dropped session-delta updates during an in-flight update, which makes a conservative close unsafe.
