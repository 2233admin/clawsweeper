## Summary

Keep #70443 open. Current main still lacks the requested `--offline` path for `scripts/docker/setup.sh`: the script does not parse a top-level `--offline` flag, still builds or pulls the main image before the later setup steps, and still builds the sandbox image when sandbox setup is enabled. The related #69143/#69314 context is about `OPENCLAW_DOCKER_PLATFORM`, not this offline rerun behavior, and does not supersede the request.

## Best Possible Solution

Keep the issue open and implement the flag in core Docker setup: parse `--offline` near the top of `scripts/docker/setup.sh`, skip the main build/pull branch, require `docker image inspect "$OPENCLAW_IMAGE"` with a clear `docker load` message when missing, warn rather than fail for a missing optional sandbox image, and leave the later env sync, permissions, onboarding, gateway config, compose startup, and sandbox overlay steps unchanged. Add e2e coverage in `src/docker-setup.e2e.test.ts` and document the workflow on the public Docker install page.
