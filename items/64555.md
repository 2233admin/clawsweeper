## Summary

Keep open. This issue has an open implementation candidate in #64563 with closing syntax, and current main still shows the reported WhatsApp auth-dir risk: `WA_WEB_AUTH_DIR` is computed at module scope and account auth discovery still resolves the OAuth root through ambient defaults. The documented multiple-gateway contract says each instance should have a unique `OPENCLAW_STATE_DIR` for credentials, so the reported profile-isolation bug remains actionable.

## Best Possible Solution

Keep this issue open and resolve the linked implementation path in #64563: review and land it if the lazy export/runtime compatibility is acceptable, or request a focused rework that makes the WhatsApp default auth directory lazy or explicitly env/state-dir backed, aligns `listWhatsAppAuthDirs()` with the same OAuth root, and adds an `OPENCLAW_STATE_DIR` profile-isolation regression test. Doctor/docs guidance for `OPENCLAW_OAUTH_DIR` can be a follow-up, but the cleanup action now is PR review rather than issue closure.
