## Summary

Keep #58133 open. The reported sessions_send timeout ambiguity is still present on current main, and this PR is a focused attempt to fix a real agent-to-agent delivery problem. It should not be closed as implemented, stale, or ClawHub work. It also should not be closed as superseded yet because the related #68196 is open/unmerged and has its own unresolved correctness concern.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. The best fix should stay in the sessions_send/run-wait surface, distinguish soft wait-window expiry from terminal aborted/error timeout, preserve the pre-run assistant baseline when doing delayed delivery, and add regression tests for delayed reply delivery, second-wait timeout/no-delivery, stale-history prevention, and terminal aborted timeout behavior. If #68196 becomes the accepted canonical fix after addressing its own P1 concern, then #58133 can be closed as superseded with a link to that PR.
