## Summary

Keep #57996 open. Current main still creates an isolated QMD XDG cache and `index.sqlite` for each agent, still runs QMD update/embed maintenance on boot and intervals, and exposes cadence/timeout knobs but not the requested shared cache, WAL checkpoint/vacuum, auto-vacuum, cache size/LRU, or reduced-write mode. The related #66339 lock-contention report remains open but does not supersede this disk-I/O/cache-growth issue.

## Best Possible Solution

Keep the issue open and handle it in the memory-core QMD runtime. The likely path is to decide whether per-agent isolation should remain the default, then add explicit multi-agent disk controls: a supported shared QMD cache mode or shared XDG cache root with agent-scoped collections if QMD supports it, cross-process-safe update/maintenance locking, periodic checkpoint/compaction where supported, status/audit warnings for large QMD indexes, and documented tuning for high-agent deployments.
