## Summary

Keep this issue open. Current main still lacks native Google Vertex transport/auth for `google-vertex`, and the linked implementation candidate, PR #60860, is still open with closing syntax for this issue. ([github.com](https://github.com/openclaw/openclaw/issues/58775))

## Best Possible Solution

Keep this issue open while PR #60860, or a maintainer-owned equivalent, is reviewed. The best path is to land a native Google Vertex provider that keeps ADC auth, project/location resolution, and Vertex URL construction separate from Google AI Studio while sharing only safe model-id normalization where appropriate.
