## Summary

Keep open. Current `main` at `6b618f063516aa9ba85903aa4565c5a204c055b2` still accepts a trailing-`@` ClawHub spec, routes it into ClawHub install resolution, and can surface `package_not_found` instead of `invalid_spec`.

## Evidence

- **Parser still accepts trailing @ as part of name:** `parseClawHubPluginSpec` returns `{ name: spec }` when `atIndex >= spec.length - 1`, so `clawhub:demo@` is treated as name `demo@` instead of rejected.
  - file: [src/infra/clawhub-spec.ts:16](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/infra/clawhub-spec.ts#L16)
  - command: `nl -ba src/infra/clawhub-spec.ts | sed -n '1,220p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **CLI still dispatches any truthy ClawHub parse result into install flow:** `plugins install` calls `parseClawHubPluginSpec(raw)` and immediately invokes `installPluginFromClawHub` when the parse result is truthy.
  - file: [src/cli/plugins-install-command.ts:498](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/cli/plugins-install-command.ts#L498)
  - command: `nl -ba src/cli/plugins-install-command.ts | sed -n '470,560p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Install flow only returns invalid_spec when parser returns null or empty name:** `installPluginFromClawHub` checks `if (!parsed?.name)` for `INVALID_SPEC`; otherwise it fetches package detail. A 404 during package fetch maps to `PACKAGE_NOT_FOUND`.
  - file: [src/plugins/clawhub.ts:736](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/plugins/clawhub.ts#L736)
  - command: `nl -ba src/plugins/clawhub.ts | sed -n '720,810p' && nl -ba src/plugins/clawhub.ts | sed -n '120,180p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Coverage does not lock the malformed trailing-@ case:** Current parser tests cover `clawhub:demo`, `clawhub:demo@1.2.3`, and a non-ClawHub scoped package, but no `clawhub:demo@` rejection case.
  - file: [src/infra/clawhub.test.ts:37](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/infra/clawhub.test.ts#L37)
  - command: `nl -ba src/infra/clawhub.test.ts | sed -n '1,80p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)

## Risks / Open Questions

- Users can get a misleading `package_not_found` error for malformed input, which obscures the actual problem and complicates support/triage.
- Without a regression test for trailing-`@`, a future fix is not locked in even though `INVALID_SPEC` support already exists in the install layer.
