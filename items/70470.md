## Summary

Keep open. Current `main` does not show an implemented fix for this report, and the existing comment’s proposed `shouldMarkRead` root cause does not match the actual control flow. The issue is still plausible, but there is not enough evidence to conservatively close it as fixed or obsolete.

## Evidence

- **Blocked messages never reach read-receipt send path:** `normalizeInboundMessage()` calls `checkInboundAccessControl()` and immediately returns `null` when `access.allowed` is false. Only admitted messages continue to `maybeMarkInboundAsRead()`.
