## Summary

Keep #69906 open. Current main does not implement the WhatsApp multi-account `groups` inheritance guard; the open PR still targets a real remaining behavior gap against Telegram parity and linked maintainer-tracked issue #69874.

## Best Possible Solution

Keep the PR open for maintainer review or merge an equivalent fix. The best product direction is to implement the narrow WhatsApp `groups` guard matching Telegram, preserve single-account behavior and `accounts.default.groups` shared defaults, leave `direct` unchanged unless maintainers choose a broader cross-channel change, and ship the PR's regression tests plus migration docs/changelog for #69874.

## Risks / Open Questions

- The PR is intentionally breaking for multi-account WhatsApp users who relied on root `groups` inheritance, so release-note and maintainer review remain important.
- The security review identified a narrow fail-open migration side effect when operators used root `groups` as an implicit allowlist with `groupPolicy="open"`; the PR documents/doctor-warns it, but maintainers should still decide whether that is acceptable.
