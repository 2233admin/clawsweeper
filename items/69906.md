## Summary

Keep #69906 open. Current main still flat-merges WhatsApp account config so root `channels.whatsapp.groups` can inherit into multi-account accounts, while Telegram already has the proposed `isMultiAccount` guard. The current WhatsApp docs explicitly document that this guard is still missing, and the linked maintainer issue #69874 remains open with a protected `maintainer` label. The PR is a focused implementation attempt rather than obsolete cleanup.

## Best Possible Solution

Keep #69906 open for maintainer review. The best path is to land or revise a focused WhatsApp account-resolution change that mirrors Telegram's multi-account `groups` guard, with regression tests for single-account inheritance, multi-account suppression, account-level precedence, `accounts.default.groups`, and docs/changelog migration guidance that covers the group allowlist side-effect.
