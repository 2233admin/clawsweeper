## Summary

Keep this PR open. Current main only partially covers the heartbeat replay slice; it still lacks the Feishu websocket health snapshot, the proposed exec non-zero foreground failure semantics, and the first-visible diagnostics/design surface. The PR is also not merge-ready as-is because the supplied review discussion contains blocking module-load/event-loss findings and the branch mixes unrelated runtime, ACPX, diagnostics, and docs work.

## Best Possible Solution

Keep this PR open only as a source of candidate fixes, not as something to land wholesale. Ask for a current-main rebase or replacement split into focused PRs: compare the heartbeat slice against main's consume-on-success implementation, carry Feishu connection health through a focused Feishu PR, handle exec/process terminal status semantics with explicit product approval and tests, and keep first-visible diagnostics/design plus ACPX dependency changes in separate reviews.
