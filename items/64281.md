## Summary

Keep open. Current main still does not implement the requested `sessions_delegate` / `sessions_delegate_batch` tools, and the existing `sessions_spawn` plus `sessions_send` behavior is not equivalent to a fresh spawn-and-wait delegation primitive. The PR remains a plausible implementation candidate, but it needs maintainer product judgment and cleanup around sanitizer coverage and output-capture/cleanup ordering before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for synchronous subagent delegation. Maintainers should decide the product/API direction, rebase it against current main's existing `prepareSubagentSpawn` hook, require sanitizer coverage for delegate attachments, fix the output-capture gate ordering and finally-release behavior, and land focused tests plus docs/display/policy updates if the core tool surface is accepted.
