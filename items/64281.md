## Summary

Keep open. Current main does not implement the requested `sessions_delegate` or `sessions_delegate_batch` tools: the core session tool surface still exposes `sessions_spawn` as non-blocking/push-announced work plus `sessions_send` for existing sessions. The PR is not obsolete, but it needs rework because one part (`prepareSubagentSpawn`) already exists on main and the PR review context identifies unresolved security/lifecycle concerns around attachment redaction and output-capture cleanup races.

## Best Possible Solution

Keep the PR open if maintainers still want synchronous delegation. The best path is to rebase onto current main, drop or align the already-shipped `prepareSubagentSpawn` hook work, add the delegate tools through the normal tool catalog/openclaw-tools/display/docs surfaces, extend transcript redaction to delegate attachments, and cover output-capture/cleanup ordering with focused tests.
