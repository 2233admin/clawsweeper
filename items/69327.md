## Summary

Keep #69327 open. Current main at 998e37fcb3d5b937fcf70d7509b677efba939f10 still has no source path that carries a parent resolved Docker sandbox environment into `sessions_spawn(..., sandbox="inherit")`; subagent spawn metadata carries workspace/group context only, and the child sandbox resolves Docker env fresh from global/target-agent config. Existing Docker env, setupCommand, config-hash, and recreate mechanisms exist, but they do not conclusively cover the reported parent-to-subagent env and stale runtime/bootstrap behavior.

## Best Possible Solution

Keep the issue open. The best maintainer path is to add a focused regression around a sandboxed parent spawning a sandboxed subagent with configured `sandbox.docker.env` and `setupCommand`, then either thread the parent resolved Docker env/context through the spawned-run context or explicitly redefine `sandbox="inherit"` to share the parent sandbox runtime. The fix should also prove that `openclaw sandbox recreate --agent <id>` invalidates the exact runtime used by spawned subagents.
