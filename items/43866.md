## Summary

Keep open. Current origin/main still does not preserve `accountId` on the direct `POST /hooks/agent` path: the hook payload type and normalizer omit the field, and `dispatchAgentHook()` builds `delivery` with only `mode`, `channel`, and `to`. The cron delivery path can consume `deliveryPlan.accountId`, but this hook path never supplies it. The linked open PR #43951 appears to be the appropriate follow-up vehicle, but the issue is not implemented on main.
