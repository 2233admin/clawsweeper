## Summary

Keep this PR open. Current main still lacks an Android Settings-level talk-mode language override, and this PR remains the only matching implementation candidate found, but it is dirty against main and still needs Android maintainer review, targeted tests, and a safer language-refresh path before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to rebase onto current main, replace the unconditional language-change reconnect with an endpoint-aware option refresh that does not create a false offline failure, add focused Android/Robolectric coverage for persistence and locale propagation, and get Android maintainer sign-off on the language list and Settings placement.
