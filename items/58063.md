## Summary

Current main does not implement a clear fix for #58063. The Grok web_search path still uses xAI Responses through the bundled xAI plugin, still inherits the shared 30 second web_search timeout, and the related #58733 remains the narrower open timeout-tracking issue rather than a full supersession of the original immediate-abort report.

## Best Possible Solution

Keep #58063 open unless a maintainer intentionally chooses to consolidate it into #58733. The likely code path to fix is the bundled xAI Grok web_search provider: add a provider-appropriate timeout/default or documented config path, cover it with focused tests in `extensions/xai/web-search.test.ts` or the provider runtime tests, and then close #58063 only after confirming the original immediate-abort symptom is fixed or clearly the same as #58733.
