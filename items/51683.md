## Summary

Keep open. Current `main` still does not add a time line to `session_status`, so the PR addresses a real gap rather than an already-shipped change. The code path builds status text without any `timeLine`, `buildStatusMessage` would render one if supplied, and current tests do not cover the missing time output even though the system prompt still tells agents to use `session_status` for time questions.
