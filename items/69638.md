## Summary

Keep PR #69638 open. Current main still rejects password-mode WebSocket handshakes that provide only `connectAuth.token`: the password branch reads only `connectAuth.password`, while the WS auth context passes token and password separately. The PR is a focused gateway bug fix tied to #51953 and includes regression coverage for the Rabbit/Funnel token-only path plus the edge cases raised by the bot review.

## Best Possible Solution

Keep the PR open and review it as the likely narrow fix: add `connectAuth.token` fallback only inside password mode, preserve `connectAuth.password` precedence, keep mismatch rejection, and land the included regression tests after normal gateway/security review.
