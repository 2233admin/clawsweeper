## Summary

Keep PR #69638 open. Current main still does not implement the PR’s requested Rabbit/Funnel compatibility path: password-mode gateway auth reads only connectAuth.password, while token-only shared credentials are still passed through separately and rejected as missing password. The PR remains a focused gateway bug fix with targeted regression coverage in its branch context, and it is not protected/maintainer-authored or superseded by current main.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. The best product/code path is to land a narrow gateway auth change that lets password mode treat connectAuth.token as a fallback only when connectAuth.password is absent, preserving password priority, mismatch rejection, and rate-limit behavior, with the branch’s targeted regression tests or equivalent coverage on current main.
