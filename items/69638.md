## Summary

Keep PR #69638 open. Current main still accepts `connectAuth.password` only in password-mode gateway auth, while the PR’s stated Rabbit R1/Funnel case sends the shared secret in `connectAuth.token`. The branch also addresses the bot’s requested edge-case tests in later commits, so the PR remains a plausible focused fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent focused gateway fix: in password mode, accept `connectAuth.token` only as a fallback when `connectAuth.password` is absent, preserve password precedence, and keep the regression coverage for token-only success, token mismatch, and dual-field precedence.
