## Summary

Keep this PR open. Current main still rejects password-mode clients that provide only `connectAuth.token`, while the PR is a narrow implementation candidate with regression coverage for the Rabbit/Funnel path and follow-up edge cases from review.

## Best Possible Solution

Keep this PR open, rebase it onto current main, preserve the narrow `connectAuth.password ?? connectAuth.token` password-mode fallback and regression tests, then run the changed gateway lane and land it if review stays clean. Longer-term Rabbit onboarding can move to device-specific bootstrap tokens as a separate follow-up.
