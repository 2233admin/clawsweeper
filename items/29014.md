## Summary

Keep open. Current `main` already handles much of the original bug report by classifying overloaded/transient provider failures and entering model fallback, but the issue asks for a broader feature set that is not fully evidenced on `main` yet: the documented behavior is bounded same-provider rotation plus fallback, not a three-attempt exponential auto-retry flow with chat UI escape-hatch controls.

## Evidence
