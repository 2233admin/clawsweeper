## Summary

Keep open. Current `main` still defers maintenance only for `reason: "turn"`, while deferred compaction execution is enabled only for background runs. Bootstrap maintenance is still invoked in foreground, and I found no local fix or test coverage for the reported bootstrap/hot-cache debt cases.

## Evidence

- **Background deferral is still turn-only:** `runContextEngineMaintenance()` only schedules deferred background work when `params.reason === "turn"` and `turnMaintenanceMode === "background"`. `bootstrap` and `compaction` reasons fall through to immediate execution on the current call path.
  - file: [src/agents/pi-embedded-runner/context-engine-maintenance.ts:615](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/agents/pi-embedded-runner/context-engine-maintenance.ts#L615)

  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Deferred compaction execution is still background-only:** The runtime context passed into `maintain()` sets `allowDeferredCompactionExecution: params.executionMode === "background"`. Foreground bootstrap/compaction maintenance therefore cannot consume deferred compaction debt on the current implementation.

That preserves the foreground default for bootstrap maintenance on `main`.
