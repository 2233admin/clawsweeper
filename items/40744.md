## Summary

Keep this PR open. Current main still does not strip own non-array `required` values from the shared tool-schema normalization path, so strict non-Gemini/OpenAI-compatible providers can still receive `required: null`. The Gemini cleaner also still only removes empty `required` arrays, and current tests do not cover the reported malformed-`required` regression. The PR remains a focused implementation candidate, but it needs rebasing onto the current `src/agents/pi-tools-parameter-schema.ts` layout.

## Best Possible Solution

Keep this PR open and request a rebase onto current main. The best implementation is to strip own non-array `required` fields in the shared `normalizeToolParameterSchema` path before provider-specific cleaning, preserve valid `required` arrays, add focused regression tests for strict non-Gemini/OpenAI-compatible providers plus nested Gemini schemas, and explicitly decide whether the existing Gemini prototype-pollution hardening note is in scope or a follow-up.
