## Summary

Keep #69698 open. Current `main` still has the reported get-then-set mismatch: internal exec allowlist entries may include `source`, the Control UI preserves and resubmits the loaded file, and the gateway schema rejects `source` as an unexpected allowlist property. The linked PR #69732 appears to be the right fix path, but it is open/unmerged in the supplied GitHub context and its head commit is not present on current `main`.

## Best Possible Solution

Keep #69698 open until #69732 or an equivalent maintainer-approved patch lands. The likely fix belongs in the exec approvals transport boundary: redact internal allowlist metadata such as `source` before gateway and node-host snapshots reach Control UI clients, and add regression coverage for an `exec.approvals.get` snapshot containing `source` that can be saved back through `exec.approvals.set` without schema rejection.
