## Summary

Current `main` does not include the PR's `encoding_format: "float"` change in `extensions/memory-lancedb`, and the existing regression test still asserts an embeddings request without that field. The plugin also still documents an OpenAI-compatible `embedding.baseUrl` with an Ollama `/v1` example, so the reported compatibility gap is still on a supported config surface rather than already resolved on `main`.
