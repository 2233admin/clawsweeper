## Summary

Keep #71792 open. Current main still rolls daily/idle-stale sessions without emitting the internal `command:reset` hook that bundled `session-memory` listens for, so the PR addresses a real remaining behavior gap rather than obsolete work.

## Best Possible Solution

Keep #71792 open and review it as a focused bug fix for bundled session-memory persistence on automatic daily/idle rollover. The likely best path is to emit the internal `command:reset` event for non-command stale rollovers with `commandSource` values such as `auto:daily` and `auto:idle`, add regression coverage for daily, idle, and no-double-fire cases, and reconcile separately with any broader before-reset/plugin lifecycle proposal.
