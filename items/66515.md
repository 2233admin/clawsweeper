## Summary

Keep this PR open. Current main still does not ship a Yandex/YandexGPT provider, and the PR is an active unmerged provider implementation rather than obsolete work. The existing provider-plugin API can support this as either a bundled provider or a ClawHub/npm plugin, so the remaining question is maintainer product review plus PR fixes, not automated cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. If Yandex is accepted as a bundled provider, require the remaining auth/config preservation and configured-folder fallback fixes, keep Yandex-specific behavior inside the provider plugin boundary, avoid a provider-specific public SDK facade unless maintainers intentionally want that stable surface, and rebase away or justify any unrelated release-script delta. If maintainers decide not to bundle it, close with explicit direction to publish it as a ClawHub/npm provider plugin using the documented provider-plugin API.
