## Summary

Keep open. Current `main` contains substantial partial fixes for the reported stale-context warning path, but not a full single-source-of-truth rollout: the chat context notice now uses fresh `totalTokens` and hides stale totals, and compaction paths clear old token breakdowns, yet `/usage` still computes context percent from `inputTokens` rather than the canonical total-context metric. The requested end-to-end compaction/UI regression test is also not present.

## Evidence

- **Chat warning now ignores stale `inputTokens` and keys off fresh `totalTokens`:** `getContextNoticeViewModel` returns null when `totalTokensFresh === false` and computes the displayed percentage from `session.totalTokens / contextTokens`, not `inputTokens`.
  - file: [ui/src/ui/chat/context-notice.ts:64](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/chat/context-notice.ts#L64)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **UI unit test proves stale `inputTokens` no longer drive the warning:** The test explicitly asserts that a session with `inputTokens: 757_300` and `totalTokens: 46_000` produces no notice, and that the rendered notice shows `190k / 200k` rather than the stale `757.3k / 200k`. It also asserts stale totals are hidden when `totalTokensFresh` is false.
  - file: [ui/src/ui/chat/context-notice.test.ts:32](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/chat/context-notice.test.ts#L32)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Compaction code clears stale token breakdowns in backend session state:** Manual compaction deletes `inputTokens` and `outputTokens`; when `tokensAfter` is known it stores that as `totalTokens` and marks `totalTokensFresh = true`. The generic compaction session update path likewise clears `inputTokens`, `outputTokens`, `cacheRead`, and `cacheWrite` after compaction.
  - file: [src/gateway/server-methods/sessions.ts:1591](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/server-methods/sessions.ts#L1591)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **UI session-state merge also drops old totals when the gateway marks them stale:** `applySessionsChangedEvent` deletes the cached `totalTokens` field when an incoming session event says `totalTokensFresh === false`, and the adjacent test verifies that stale totals are removed from the row.
  - file: [ui/src/ui/controllers/sessions.ts:258](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/controllers/sessions.ts#L258)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **A remaining UI surface still uses `inputTokens` for context percentage:** `/usage` still computes `pct` as `Math.round((input / ctx) * 100)`, and its test expects `30%` from `inputTokens: 1200` with `totalTokens: 1500`, showing this surface is not yet aligned to the canonical post-compaction metric.
  - file: [ui/src/ui/chat/slash-command-executor.ts:380](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/chat/slash-command-executor.ts#L380)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
