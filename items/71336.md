## Summary

Keep open. Current `main` still keeps the WebChat queue only in UI memory and clears it on session changes, so the underlying #51549 behavior is not implemented on `main`. This PR is also brand new, and the discussion shows the author already narrowed scope to the WebChat-only fix after the initial blocking session-store findings, so there is no conservative auto-close basis here.
