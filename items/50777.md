## Summary

Keep open. Current `main` still renders both `Copy` and `Copied!` spans without an HTML-level hide, so the reported bug is not implemented there, and the PR itself has an unresolved review concern because it would globally allow the `hidden` attribute through the Markdown sanitization pipeline.

## Evidence

- **Current main still relies on CSS-only hiding for the copy button text:** Both fenced and indented code block renderers emit `<span class="code-block-copy__done">Copied!</span>` with no `hidden` attribute, so initial invisibility still depends on CSS loading.
