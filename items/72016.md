## Summary

Keep open. Current main has useful doctor/plugin-related pieces, but not the requested general, upgrade-safe doctor extensibility surface for custom environment checks and `--fix` repairs. The health pass is a hard-coded core contribution list, and the existing plugin doctor seams are narrow channel/config compatibility hooks rather than a documented API for arbitrary doctor checks. The related #68348 report was about native loading for doctor contract modules, not this feature request.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A good implementation would add a documented plugin-facing doctor contribution contract for read-only checks and safe `--fix` repairs, define ordering and non-interactive behavior, preserve SecretRef-safe config access, handle disabled/untrusted plugins explicitly, and add tests/docs. The existing multi-gateway guidance should remain as mitigation, but it does not replace the requested extensibility API.
