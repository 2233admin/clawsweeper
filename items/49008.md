## Summary

Current `main` already changed Telegram media downloads from a single-shot fetch into a classified multi-attempt transport fallback path, so the reported intermittent `TypeError: fetch failed` download failure is implemented on `main` even though it does not appear to be in the latest tagged release yet.

## Evidence

- **Telegram media download path now opts into retryable transport fallback:** `downloadAndSaveTelegramFile` passes `dispatcherAttempts`, `trustExplicitProxyDns`, and `shouldRetryFetchError` into `fetchRemoteMedia`, so the file download leg no longer uses a bare one-shot fetch.
  - file: [extensions/telegram/src/bot/delivery.resolve-media.ts:235](https://github.com/openclaw/openclaw/blob/8fade9df27a9a25cd017db1154409b3ad9bd3741/extensions/telegram/src/bot/delivery.resolve-media.ts#L235)
  - command: `nl -ba extensions/telegram/src/bot/delivery.resolve-media.ts | sed -n '188,245p'`
  - sha: [8fade9df27a9](https://github.com/openclaw/openclaw/commit/8fade9df27a9a25cd017db1154409b3ad9bd3741)
- **Generic media fetcher retries across attempts and preserves causes:** `fetchRemoteMedia` loops over dispatcher attempts, retries only when `shouldRetryFetchError` says to, and wraps the final failure in `MediaFetchError(..., { cause })` with combined attempt errors when multiple attempts fail.
  - file: [src/media/fetch.ts:127](https://github.com/openclaw/openclaw/blob/8fade9df27a9a25cd017db1154409b3ad9bd3741/src/media/fetch.ts#L127)
  - command: `nl -ba src/media/fetch.ts | sed -n '124,190p'`
  - sha: [8fade9df27a9](https://github.com/openclaw/openclaw/commit/8fade9df27a9a25cd017db1154409b3ad9bd3741)
- **Telegram transport classifies the reported failure shape and provides IPv4/pinned fallbacks:** Fallback rules match the `fetch failed` envelope plus network codes such as `ETIMEDOUT`, `ENETUNREACH`, `EHOSTUNREACH`, `UND_ERR_CONNECT_TIMEOUT`, and `UND_ERR_SOCKET`, and the transport exports default, IPv4-only, and pinned-IPv4 attempts for Telegram traffic.
  - file: [extensions/telegram/src/fetch.ts:96](https://github.com/openclaw/openclaw/blob/8fade9df27a9a25cd017db1154409b3ad9bd3741/extensions/telegram/src/fetch.ts#L96)

  - sha: [8fade9df27a9](https://github.com/openclaw/openclaw/commit/8fade9df27a9a25cd017db1154409b3ad9bd3741)
- **Runtime actually threads the Telegram transport into media resolution:** `bot-core` creates `telegramTransport` with `resolveTelegramTransport(...)`, and `resolveTelegramMediaRuntimeOptions` passes that transport through to media resolution, so real Telegram inbound media downloads use the fallback-enabled transport path.
