## Summary

Current `main` still omits the active `modelId` when incomplete-turn exhaustion paths persist auth-profile failures, while the cooldown state machine relies on `modelId` to preserve model-scoped rate-limit behavior. Other failover paths already pass `modelId`, so this issue remains unresolved on `main` rather than obsolete or already fixed.

## Evidence

- **Incomplete-turn terminal branches still persist failure without modelId:** Both incomplete-turn exhaustion exits call `maybeMarkAuthProfileFailure` with `profileId` and `reason` only. The helper can forward `modelId`, but these branches do not supply it.

- **Helper and normal failover paths support modelId:** `maybeMarkAuthProfileFailure` accepts `modelId` and forwards it to `markAuthProfileFailure`. Prompt-error and rotate-profile failover paths already pass `modelId`, so the omission is specific to incomplete-turn paths.
