## Summary

Keep PR #69268 open. Current main already has Anthropic adaptive-thinking payload support inside the OpenClaw boundary-aware transport, but the runner still selects that transport only for undefined or exact `streamSimple` session functions. A wrapped pi-coding-agent `streamSimple` path is still treated as `session-custom`, and the PR's missing-user-tail/trailing-assistant cleanup helpers are not present on main. The related adaptive-thinking reports do not supersede this PR because they cover the payload shape once the Anthropic transport is used, not this wrapper-bypass path.

## Best Possible Solution

Keep the PR open and continue toward a scoped fix: route supported Anthropic/default pi-coding-agent wrapper paths through OpenClaw's boundary-aware transport without breaking truly provider-owned custom streams, then add the trailing non-user/assistant cleanup after effective prompt and images are finalized, with regression coverage in stream-resolution and validate-turns or attempt tests.
