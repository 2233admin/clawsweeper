## Summary

Keep #57967 open. Current main still writes and chmods the macOS gateway LaunchAgent plist with `0o644`, and the current launchd test suite explicitly asserts `0o644`. The linked PR #57971 is the right narrow follow-up, but it is still open/unmerged and would need to account for the current test expectation before this issue can be closed as fixed.

## Best Possible Solution

Keep the issue open until PR #57971 or an equivalent maintainer patch lands. The fix should set `LAUNCH_AGENT_PLIST_MODE` to `0o600`, update the launchd permission test to expect `0o600`, and verify the install/rewrite path still works with launchd on macOS.
