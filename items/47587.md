## Summary

Current `main` fixes the specific header-drop symptom for image descriptions, but it does not clearly implement the issue's full expected behavior of transcribing every image in a Telegram album by default. The Telegram media-group path now preserves all inbound media paths, and prompt assembly explicitly keeps image attachments even after image understanding runs. However, media-understanding still selects image attachments with policy default `mode: "first"`, with `mode: "all"` only when configured. That makes this issue not safe to close as fully implemented.
