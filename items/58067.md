## Summary

Keep open. Current main has improved the old stuck-parent path: `sessions_yield` now cleanly ends the parent turn and subagent completion is pushed back through announce delivery. But the requested parent-surface progress/streaming behavior is still not implemented for native `runtime="subagent"`: `streamTo:"parent"` remains ACP-only, and Control UI only renders existing tool/compaction/fallback agent events, not a native subagent-wait/progress lifecycle.

## Best Possible Solution

Keep the issue open or split it into a narrower native-subagent progress item. The best fix is to preserve the current clean `sessions_yield` and final announce behavior, then add a requester-scoped progress lifecycle for native subagents that Control UI and TUI can render as a waiting/background-task indicator. If streaming is desired, extend the native subagent runtime with an explicit, tested `streamTo:"parent"` or equivalent relay contract rather than relying on the ACP-only path.
