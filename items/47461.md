## Summary

Keep open. Current `main` supports Control UI/avatar persistence and rendering, but there is no Telegram profile-photo sync implementation. The Telegram plugin exposes only message/topic/reaction actions, and repo-wide inspection found no `setChatPhoto`/profile-photo Bot API call. This is a coherent bundled-Telegram enhancement request, not a ClawHub-scope item and not already implemented on `main`.

## Evidence

- **Control UI avatar changes stay local to config/IDENTITY:** `agents.create`/`agents.update` only sanitize and persist `avatar` into agent identity and `IDENTITY.md`; there is no Telegram handoff in this path.
  - file: [src/gateway/server-methods/agents.ts:474](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/gateway/server-methods/agents.ts#L474)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Gateway returns avatar for UI rendering only:** Agent identity lookup resolves an avatar URL/path and returns it to the Control UI response; this is local rendering behavior, not Telegram account mutation.
  - file: [src/gateway/server-methods/agent.ts:1015](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/gateway/server-methods/agent.ts#L1015)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Avatar resolver is local/http/data only:** `resolveAgentAvatar` reads `ui.assistant.avatar`, agent config, or `IDENTITY.md` and resolves a local file / remote URL / data URI. No Telegram synchronization hook exists here.
  - file: [src/agents/identity-avatar.ts:23](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/identity-avatar.ts#L23)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Telegram plugin does not expose a profile-update action:** Telegram message-tool discovery only offers `send`, `poll`, `react`, `delete`, `edit`, `sticker`, `sticker-search`, `topic-create`, and `topic-edit`. There is no `set-profile`-style action comparable to Matrix.
  - file: [extensions/telegram/src/channel-actions.ts:40](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/telegram/src/channel-actions.ts#L40)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Repo-wide search found no Telegram photo-update Bot API call:** A repo-wide search for `setChatPhoto|setMy.*Photo|deleteChatPhoto|getUserProfilePhotos` under `extensions/telegram` and `src` returned no matches on current `main`, which strongly indicates the requested sync path is not implemented.
  - command: `rg -n "setChatPhoto|setMy.*Photo|deleteChatPhoto|getUserProfilePhotos" extensions/telegram src`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Contrast: Matrix already has explicit profile sync:** Matrix exposes `set-profile` with avatar media parameters and docs explicitly describe avatar/profile sync, which underscores that Telegram does not currently have equivalent support.
  - file: [extensions/matrix/src/actions.ts:20](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/matrix/src/actions.ts#L20)

- Telegram bot photo updates may have Bot API permission/format constraints, so eventual work will need channel-specific validation and tests.
