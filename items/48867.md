## Summary

Keep open. Current main adds an alternate browser-node route for `profile="user"`, but the host existing-session path still launches Chrome MCP over stdio and I did not find a repo-side fix for the reported Android Termux stdin-parent-death failure.

## Evidence

- **Host existing-session path still uses stdio Chrome MCP:** `extensions/browser/src/browser/chrome-mcp.ts` still creates the local existing-session attach via `new StdioClientTransport({ command: "npx", args: buildChromeMcpArgs(...), stderr: "pipe" })`. I did not find any Android/Termux-specific handling or alternate transport on this host path.
  - file: [extensions/browser/src/browser/chrome-mcp.ts:225](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/browser/src/browser/chrome-mcp.ts#L225)
  - command: `nl -ba extensions/browser/src/browser/chrome-mcp.ts | sed -n '220,245p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **CLI reproduction path still reaches the gateway browser request flow:** `openclaw browser ...` goes through `browser.request` from the CLI, and `browser.request` only proxies to a browser node when one is connected/configured; otherwise it falls through to the local host control service. That means the reported Termux host attach path still exists on main.
  - file: [extensions/browser/src/cli/browser-cli-shared.ts:46](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/browser/src/cli/browser-cli-shared.ts#L46)

- **Shipped change is browser-node routing, not a host-side Termux fix:** The relevant shipped change in `CHANGELOG.md` is `profile="user"` auto-routing to a connected browser node. The matching code/tests in `extensions/browser/src/browser-tool.ts` and `extensions/browser/src/browser-tool.test.ts` cover node proxy routing and explicit host/node targeting, but they do not change the local host stdio attach implementation.
  - file: [CHANGELOG.md:430](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/CHANGELOG.md#L430)
