## Summary

Keep open. Current main has overlapping Telegram polling watchdog hardening, but it does not implement this PR's central active update-handler bypass or a pre-handler middleware hook/order that would let polling liveness distinguish a long-running local handler from a dead poller. The branch is stale against the pluginized Telegram paths, but the focused fix still looks plausibly useful for maintainer review or transplant.

## Best Possible Solution

Keep this PR open for maintainer or author follow-up. The best path is to transplant the active update-handler watchdog bypass into `extensions/telegram/src`, either by integrating active-handler state into `TelegramPollingLivenessTracker` or by adding a narrow pre-handler middleware seam in `bot-core` after sequentialization and before handler registration. Add a regression test for a long-running update handler that performs no recent Telegram API call. If maintainers decide the shipped configurable `pollingStallThresholdMs` and API-liveness tracking are the intended product answer, close manually with that explicit decision.
