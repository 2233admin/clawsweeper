## Summary

Current main still only partially addresses the reported JSON CLI cold-path problem. The agents-specific related work for #71739 is present, but `tasks list --json` and `tasks audit --json` still lack route-first lightweight paths and still fall through the shared status/health/sessions/tasks registrar. The lean `status --json` path avoids plugin preload, but it does so by explicitly disabling `channelSummary`, which leaves the reporter's completeness requirement unresolved.

## Best Possible Solution

Keep this issue open. The best fix is to add route-first lightweight JSON handling for `openclaw tasks list --json` and `openclaw tasks audit --json`, or split the tasks registrar/modules so these JSON paths do not import status, health, sessions, Task Flow audit, and maintenance work unnecessarily. Separately, implement a lightweight `status --json` channelSummary strategy that avoids full plugin/runtime preload while preserving the documented JSON surface, with regression coverage for the tasks JSON load path and configured channel-summary behavior.
