## Summary

Keep open. `status --json` has been partly optimized on current `main`, but the issue is not fully resolved: `tasks` still shares a grouped registration module that eagerly imports unrelated `status`/`health`/`sessions` command code, and the lean `status --json` path explicitly disables populated `channelSummary` instead of preserving it.

## Evidence

- **`tasks` still registers through a shared status/health/sessions module:** The core CLI registry still maps `status`, `health`, `sessions`, and `tasks` to the same loader module, so entering `tasks` loads that combined registration surface rather than an isolated tasks-only registrar.
  - file: [src/cli/program/command-registry-core.ts:114](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/cli/program/command-registry-core.ts#L114)
  - command: `nl -ba src/cli/program/command-registry-core.ts | sed -n '1,220p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Shared registrar still statically imports unrelated command families:** `register.status-health-sessions.ts` imports `healthCommand`, `sessionsCommand`, `statusCommand`, and the task commands at top level. That means a `tasks list` or `tasks audit` entry still pays import cost for unrelated status/health/sessions command code.
  - file: [src/cli/program/register.status-health-sessions.ts:2](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/cli/program/register.status-health-sessions.ts#L2)

- **`status --json` is partially optimized on `main`:** Current `main` routes `status --json` through a dedicated lean command and marks the `status` command path with `loadPlugins: "never"`, which addresses part of the reporter's preload complaint.
  - file: [src/cli/program/routed-command-definitions.ts:62](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/cli/program/routed-command-definitions.ts#L62)

So current `main` intentionally skips populated `channelSummary` on the fast path instead of preserving it.
