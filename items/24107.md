## Summary

Keep open. Current `main` at `d3f6783b16ecc10c67b101bf019ca4294453206a` does not implement general RPM/RPD-aware fallback cooldowns. The code still uses generic failure-count cooldowns, the fallback loop only consults existing cooldown state, and provider config/schema still have no `rpm`/`rpd` fields. There is a partial provider-specific override for `openai-codex` via WHAM usage probing, but it does not satisfy this issue’s general per-provider quota-aware design.
