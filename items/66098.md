## Summary

Keep this PR open. Current main has partially superseded the thinking-default display path through session rows/defaults, but it still does not expose or consume an effective reasoning default for Web UI chat history, so the PR remains a plausible implementation candidate for the reasoning-visibility mismatch tracked in related #64830. The PR needs rework against current main because review comments correctly flag stale history-cache precedence after model/session updates.

## Best Possible Solution

Keep this PR open for maintainer review/rework. The best path is a backward-compatible gateway/UI contract for effective reasoning state that matches the current session-row/session-controls architecture, preserves the already-shipped thinkingDefault row behavior, addresses stale-cache precedence after session/model patches, and leaves the broader config.patch gap tracked in related #64830 unless this PR is deliberately expanded.
