## Summary

Keep PR #69664 open. Current main still has the Node-shaped `ERR_MODULE_NOT_FOUND` check only, so the Bun bootstrap bug described in linked issue #69589 is not implemented on main. The PR is a focused fix for the affected launcher path, but should ideally gain a regression test for Bun-shaped errors before landing.

## Best Possible Solution

Keep the PR open and land this fix or an equivalent change in `openclaw.mjs`, with a focused regression test for a Bun-shaped direct `ERR_MODULE_NOT_FOUND` on the optional `warning-filter` import while preserving the existing transitive-miss behavior.
