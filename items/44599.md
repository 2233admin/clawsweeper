## Summary

Current main still contains the reported failure path. `./docker-setup.sh` delegates to `scripts/docker/setup.sh`, which validates `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` with a whitespace rejection before setup proceeds. The linked discussion and related PRs correctly identify the same root cause, but the fix is not present on current main, so this issue should stay open.
