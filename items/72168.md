## Summary

Keep #72168 open. Current main still matches the reported failure shape: Playwright browser actions soft-load `pw-ai`, `pw-session` still imports bare `playwright-core`, packaged plugin deps are staged outside the package tree, and the provided related PR #72238 is open/unmerged rather than shipped.

## Best Possible Solution

Keep #72168 open and use PR #72238, or an equivalent maintainer patch, as the fix path. The fix should make `playwright-core` resolvable for packaged browser Playwright actions after `openclaw doctor --fix`, ideally through a browser-local runtime shim or another resolver-visible mechanism, and add regression coverage for packaged global/Homebrew-style installs with external plugin runtime dependency staging.
