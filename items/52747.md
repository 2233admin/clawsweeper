## Summary

Keep this PR open. Current main still lacks a timeout/deadline for started ACP session-actor lane tasks outside active turn execution, including `initializeSession()` and `ensureRuntimeHandle()` paths that await `runtime.ensureSession()`. The PR addresses a real ACP control-plane gap, but its current timeout strategy needs maintainer security/concurrency review because the proposed queue timeout can release the per-session lane while the original operation continues running.

## Best Possible Solution

Keep this PR open as an active implementation candidate for a real ACP control-plane bug. The best path is a session-lane deadline design that covers initialize/status/cancel/close/idle-eviction actor tasks, preserves per-session serialization or fences stale post-timeout side effects, propagates cooperative cancellation/abort support to runtimes where available, keeps runtime cache/concurrency accounting until close is confirmed, clamps and unrefs timers safely, and wires `acp.sessionLane.taskTimeoutMs` through types, schema/help/docs, and regression tests for hung `ensureSession` followed by queued work.

## Risks / Open Questions

- The proposed `runWithTaskTimeout()` rejects the queue-visible promise while the underlying `op()` continues, which can break the per-session serialization guarantee and allow stale side effects from a timed-out task to race with newer same-session work.
- Applying the timeout to idle eviction is risky with the current clear-before-close pattern: `evictIdleRuntimeHandles()` clears the runtime cache before awaiting `runtime.close()`, so a timed-out or hung close can orphan a live runtime and undercount active sessions.
- The proposed configurable timeout needs bounded/clamped timer semantics and cooperative cancellation where possible; an unbounded millisecond value can produce surprising `setTimeout()` behavior and should not become a DoS footgun.
- The PR is size XL and touches generated config surfaces plus ACP lifecycle behavior, so it needs normal maintainer review and targeted regression coverage before merge.
