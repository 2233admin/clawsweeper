## Summary

Keep #72265 open. The issue is a focused docs request, and current main still has the confusing README sequence plus lacks the requested Node service-runtime verification checklist in the Node and gateway troubleshooting docs. The QEEK-AI comment describes the right intended patch, but those changes are not present in the checked-out main tree.

## Best Possible Solution

Make a docs-only patch: split README Quick start into an explicit daemon path using `openclaw onboard --install-daemon` plus `openclaw gateway status`, and a foreground/debug path that stops the managed service before running `openclaw gateway --port 18789 --verbose`. Add a Node service-runtime verification section to `docs/install/node.md` and fold the same checklist into `docs/gateway/troubleshooting.md`, including shell Node vs service Node drift and restart guidance after Node upgrades. Keep public docs links canonical where the docs team wants users to land.
