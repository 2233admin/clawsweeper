## Summary

Keep #40652 open. Current main still only partially implements the requested recovery guidance: the gateway startup error suggests `openclaw doctor --fix` but does not mention `openclaw.json.bak`, and daemon start/restart invalid-config preflight still suggests generic `openclaw doctor` without `--fix` or backup rollback.

## Best Possible Solution

Keep the issue open and make a narrow UX fix across the invalid-config startup surfaces: centralize or standardize the recovery hint so gateway startup and daemon start/restart say `openclaw doctor --fix`, then add an explicit `openclaw.json.bak` rollback fallback where a backup may exist. Extend the existing gateway startup and daemon lifecycle config-guard tests to assert both recovery steps.
