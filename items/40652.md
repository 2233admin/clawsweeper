## Summary

Current main still only partially satisfies the requested recovery guidance. Direct gateway startup invalid-config errors now suggest `openclaw doctor --fix`, and startup can try last-known-good recovery, but the thrown error still omits an explicit `openclaw.json.bak` rollback hint. The daemon start/restart preflight path still emits the older generic `openclaw doctor` wording and also omits backup recovery guidance.

## Best Possible Solution

Keep this issue open and make a narrow UX fix across the gateway invalid-config startup surfaces. The best implementation is to centralize the invalid-config recovery hint, use it from direct gateway startup validation and daemon start/restart preflight, mention `openclaw doctor --fix` plus an explicit `openclaw.json.bak` fallback where appropriate, and extend the gateway recovery and daemon config-guard tests to assert both recovery steps. Because this is user-facing recovery text, add a concise changelog entry when the fix lands.
