## Summary

Keep open. Current `main` still does not implement a channel-account `displayLabel`/friendly-label field that propagates into Control UI session names. Session display resolution only uses stored session `label`, `displayName`, derived group titles, or `origin.label`; account config names are separate and not wired into session metadata.

## Evidence

- **No `displayLabel` config/support in repo:** Exact search for `displayLabel` only finds unrelated local variable names in UI/render helpers; no channel config field, schema entry, or session propagation path exists.
  - command: `rg -n "displayLabel" docs src extensions ui -S`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Session selector uses only session label/displayName:** Control UI session-name resolution reads `row.label` and `row.displayName`, then falls back to parsed key text. It does not consult channel account config or account metadata.
  - file: [ui/src/ui/chat/session-controls.ts:390](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/ui/src/ui/chat/session-controls.ts#L390)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Gateway session rows do not derive names from account config:** Gateway session rows compute `displayName` from `entry.displayName`, derived group display name, `entry.label`, or `origin.label`. There is no lookup from `origin.accountId` into configured account names.
  - file: [src/gateway/session-utils.ts:1256](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/gateway/session-utils.ts#L1256)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Session origin stores conversation label and accountId only:** Session metadata captures `origin.label` from `resolveConversationLabel(ctx)` and stores `origin.accountId`, but does not attach any account display/name field that could drive the UI label later.
  - file: [src/config/sessions/metadata.ts:54](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/config/sessions/metadata.ts#L54)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Configured account `name` exists, but only for account snapshots/status:** Safe channel account projection includes `record.name`, showing current config supports per-account naming for channel status surfaces; that is separate from session labeling. Docs also show `name` on Feishu accounts.
  - file: [src/channels/account-snapshot-fields.ts:157](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/channels/account-snapshot-fields.ts#L157)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Current workaround is still manual session label editing:** Control UI Sessions view exposes a manual editable `label` input per session, matching the issue report’s workaround rather than a config-driven automatic label feature.
  - file: [ui/src/ui/views/sessions.ts:518](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/ui/src/ui/views/sessions.ts#L518)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
