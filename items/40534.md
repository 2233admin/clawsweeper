## Summary

Keep #40534 open. Current main has partial mitigation for text-only Telegram/Discord cron announce delivery via `finalAssistantVisibleText`, but it still does not implement the PR's core helper-level `isReasoning` delivery filtering. The PR branch addresses real remaining gaps: reasoning payloads are created, the payload map still drops the flag, and cron payload selection/error-recovery helpers still treat later non-error text as deliverable/successful even when it is reasoning.

## Best Possible Solution

Keep the PR open for maintainer review or ask the author to rebase it. The best fix is to propagate `isReasoning` out of `buildEmbeddedRunPayloads`, keep it in the run result and cron delivery payload typing, filter it in summary/text/deliverable selection and post-error recovery detection, and add focused tests for mixed final text, reasoning-only payloads, and error-plus-reasoning-only recovery while preserving the current `finalAssistantVisibleText` behavior for Telegram/Discord.
