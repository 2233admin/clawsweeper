## Summary

Keep this PR open. Current main still has the cron announce reasoning-filter gap: reasoning payloads are created by the embedded runner, but the returned payload map drops `isReasoning`, and the centralized cron outcome resolver still selects/falls back around `isError` only. The PR is a focused implementation candidate, but it needs rebasing onto the current `resolveCronPayloadOutcome` path before it can be merged.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, propagate `isReasoning` from `buildEmbeddedRunPayloads`, include `isReasoning` in the cron `DeliveryPayload` surface, skip reasoning in summary/output/deliverable selection and `firstText` fallback inside `resolveCronPayloadOutcome`, exclude reasoning from post-error recovery detection, and add regression tests for mixed reasoning plus final text, reasoning-only, and error-then-reasoning-only cron announce cases.
