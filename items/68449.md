## Summary

Keep this issue open. Current main has partial mitigations for the original Dreaming noise report, but the remaining behavior described in the discussion is still present: REM and light Dreaming still consume persisted conceptTags on read paths with narrow or no filtering, unclassified direct cron prompts still preserve paired assistant narration, decimal/template tokens are not comprehensively normalized, and there is no operator-facing Dreaming session/cron exclusion config. Related work is still open in #68876 and #72611.

## Best Possible Solution

Keep this issue open until maintainers either land a complete memory-core Dreaming fix or explicitly split the remaining work into canonical follow-ups such as #72611 plus a narrower concept-tag read-path cleanup. The concrete path is to preserve the current stopword and transcript-classification improvements, add reliable configurable Dreaming session/cron exclusions, handle unclassified cron narration without weakening spoofing protections, normalize or filter persisted conceptTags for both REM and light narrative inputs, and add focused regression tests for decimal/template/path/date tags and cron/NO_REPLY ingestion.
