## Summary

Current main still appears to have the reported sandbox skill prompt mismatch. Skills are synced into the sandbox workspace, but the embedded attempt and compaction paths still pass the original skills snapshot through unchanged; when that snapshot has resolved skills, live sandbox skill loading is skipped and the snapshot prompt wins. The focused fix remains open as PR #38789, while #43526 and #51471 were closed unmerged.

## Best Possible Solution

Keep this issue open and review or land PR #38789, or implement the same fix on main: for non-rw sandbox attempt and compaction runs, rebuild or remap the skills prompt from sandbox-readable skill entries instead of returning the host-built snapshot prompt. Preserve skill filters, agent allowlists, environment override behavior, remote eligibility notes, and empty-sandbox behavior, then add regression coverage for snapshot-present non-rw attempt and compaction runs.
