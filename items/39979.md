## Summary

Current main still lacks the requested path-keyed RWX permission model shared by exec and filesystem tools. Main has exec approvals/allowlists, newer arg-pattern metadata, and workspace-only filesystem guards, but not configurable per-path read/write/execute rules with deny/default semantics across read/write/edit/apply_patch and exec.

## Best Possible Solution

Keep #39979 open until OpenClaw ships a maintainer-approved filesystem permission model covering file tools and exec target paths, with backward-compatible exec approvals, schema/help/docs updates, and focused tests. Alternatively, make #48532 explicitly cover this exact path-scoped RWX requirement and then close #39979 as superseded.
