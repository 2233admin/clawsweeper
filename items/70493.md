## Summary

Keep this issue open. Current main still has the reported Playwright-unavailable failure path for advanced browser routes, sandboxed/sub-agent tools still default to the sandbox bridge, and the issue thread has explicit maintainer direction not to close until isolated-session browser coverage exists.

## Best Possible Solution

Keep this issue open. The best fix is to make isolated sub-agent browser actions consistently use a Playwright-capable runtime, either by ensuring the sandbox/isolated bridge loads the browser plugin runtime dependencies or by explicitly delegating browser actions to the host/node browser owner when that is the intended architecture. Add a regression test that exercises navigate or snapshot through the real isolated sub-agent path and asserts it does not return the Playwright-unavailable error when the main gateway/browser node supports Playwright.
