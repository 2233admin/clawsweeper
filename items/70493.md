## Summary

Keep open. Current `main` has partial browser-tool improvements for `profile="user"`, but sandboxed/isolated sessions still default to the sandbox browser, host control is still opt-in, and I found no isolated-session regression coverage proving `navigate`/`snapshot`/`act` parity. The Apr 25, 2026 maintainer comment also explicitly says not to close without isolated-session coverage.

## Evidence

- **Sandboxed sessions are still documented as a different browser target path:** The browser docs say sandboxed sessions default to `target: "sandbox"`; `target: "host"` requires `agents.defaults.sandbox.browser.allowHostControl=true`; and auto-routing to a browser node is only opportunistic. That is not evidence of full parity with the main-session host browser path.
  - file: [docs/tools/browser.md:640](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/docs/tools/browser.md#L640)
