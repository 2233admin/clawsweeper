## Summary

Keep PR #71582 open. Current main at a35d259 still routes exec/system-event heartbeat wakes through the old scoped wake path without clearing pinned heartbeat delivery fields, so the PR is not implemented or obsolete. The provided discussion also supports keeping it open: Greptile found only a cosmetic duplication concern, and a maintainer/Codex review already concluded the bug still exists on main.

## Best Possible Solution

Keep PR #71582 open. The best path is to preserve the focused bug fix, consolidate the duplicated exec-event wake helper into a shared routing/session helper, then run the changed gates for the touched agent, gateway, infra heartbeat, cron, and outbound routing surfaces before merging.
