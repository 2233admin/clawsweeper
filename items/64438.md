## Summary

Keep open. Current main still supports remote embedding endpoints and local/MMR/QMD reranking paths, but not a configurable remote reranker endpoint for builtin or QMD memory search.

## Best Possible Solution

Keep this open for a focused memory-search design. The best path is an additive remote reranker contract covering config schema/help/docs, resolver support, a Cohere-compatible HTTP reranker for builtin search with explicit vector-only fallback, batching/timeout behavior, tests, and a QMD integration story. If maintainers prefer plugin ownership, first add a documented reranker provider API so Qwen, Cohere, Jina, and similar adapters can live in plugins without replacing the whole memory backend.
