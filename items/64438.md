## Summary

Keep open. Current main supports remote embedding endpoints, local/MMR-style reranking in the builtin memory path, and QMD's own reranking via the QMD sidecar, but it does not implement a Cohere-compatible remote reranker endpoint, `memorySearch.remote.rerank`, `/v1/rerank` client, timeout/header/api-key config for reranking, or tests/docs for that behavior. The issue is recent, concrete, unprotected, and non-maintainer-authored; the single comment is unrelated spam and does not resolve the request.

## Best Possible Solution

Keep the issue open. The best path is a focused memory-core design/implementation: add a typed reranker contract and config surface, decide whether it belongs under `memorySearch.query.hybrid.rerank` or an expanded provider-adapter API, implement a Cohere-compatible `/v1/rerank` client with SSRF/timeout/header/api-key handling, integrate it into builtin candidate ranking with vector-only fallback on failure, and either add QMD support only where QMD exposes a compatible hook or document QMD as still QMD-owned. Add config schema/help/docs plus unit tests for request/response mapping, failure fallback, batching/top_n, and secret redaction.
