## Summary

Keep open. Current `main` does not implement a provider-level `models.providers.*.enabled` flag. The config type and schema for `models.providers.*` still only allow transport/auth/model fields, and the model-catalog/fallback code still iterates every configured provider without any enabled-state filter.

## Evidence

- **Provider config type has no enabled field:** `ModelProviderConfig` defines `baseUrl`, auth/request fields, and `models`, but no `enabled` property.
  - file: [src/config/types.models.ts:114](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/config/types.models.ts#L114)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Schema still rejects extra keys under models.providers.*:** `ModelProviderSchema` is strict and lists no `enabled` key, so `models.providers.<id>.enabled` is not part of the current supported config surface.
  - file: [src/config/zod-schema.core.ts:346](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/config/zod-schema.core.ts#L346)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Configured model catalog does not filter disabled providers:** `buildConfiguredModelCatalog()` walks every entry in `cfg.models.providers` and pushes all provider models into the catalog; there is no enabled-state check.
  - file: [src/agents/model-selection-shared.ts:633](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/agents/model-selection-shared.ts#L633)
