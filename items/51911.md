## Summary

Keep open. Current main has partial mitigation for the reported Anthropic token-path breakage, but it does not fully implement the issue's requested fixes: token onboarding still writes only to the current agent dir, there is still no explicit post-write validation on the token path, and the Anthropic setup-token UX/docs still do not explain the full token-obtaining flow. That is not strong enough evidence for a conservative close.

## Risks / Open Questions

- The exact March 2026 gateway failure may now be harder to reproduce when sibling agent stores start empty, because main-agent auth-profile inheritance can mask that specific symptom.
- Closing as implemented would overstate the current state: the token path still lacks explicit sibling-sync and write-verification, and the docs/prompt gap is still present.
