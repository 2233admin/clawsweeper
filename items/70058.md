## Summary

Keep PR #70058 open. Current main still lacks the requested opt-in skills watcher polling fallback: `ensureSkillsWatcher()` constructs the chokidar watcher with `ignoreInitial`, `awaitWriteFinish`, and `ignored` only, and the skills docs/config docs only expose `skills.load.watch` and `watchDebounceMs`. Chokidar 5 does support `usePolling` and `interval`, so the PR's core direction is plausible, but it needs rebasing against the newer watcher repair on main and cleanup of an unrelated QA scenario change before merge.

## Best Possible Solution

Keep the PR open and ask for a current-main revision: add opt-in polling to `src/agents/skills/refresh.ts`, include `usePolling` and a valid numeric `interval` only when requested, track those values in watcher reuse state, add focused tests in `src/agents/skills/refresh.test.ts`, document the env vars in the skills docs, and drop the unrelated QA scenario change.
