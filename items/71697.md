## Summary

Keep #71697 open. Current main still documents and implements Ollama Cloud through native `api: "ollama"` / `/api/chat`, but the native Ollama provider path converts and forwards replay history as-is and the plugin tests still assert that native Ollama has no replay policy. The related #66505/direct-cloud setup is implemented, but this agent/Gateway role-alternation bug remains actionable.

## Best Possible Solution

Keep the issue open and fix the native Ollama provider path. The likely maintainer path is an Ollama-owned replay policy or native converter normalization for `api: "ollama"` that merges or reshapes non-alternating user/assistant turns while preserving native Ollama tool-call semantics, plus regression coverage that exercises the agent/Gateway replay history before it reaches `/api/chat`.
