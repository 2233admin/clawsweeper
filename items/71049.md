## Summary

Keep #71049 open. Current main still hardcodes the Microsoft Teams informative stream/status path for personal DMs and does not expose a server-side config option to disable it. The existing `typingIndicator` flag only controls native typing keepalive behavior; tests explicitly preserve that `typingIndicator: false` still sends the informative update.

## Best Possible Solution

Keep this open as a bundled Microsoft Teams plugin feature request. A good fix would add a typed `channels.msteams` option for DM stream status or Teams streaming mode, validate it in the strict schema and generated metadata, document it on `docs/channels/msteams.md`, pass it into `createTeamsReplyStreamController`, and add tests proving personal DMs can use native typing plus final-message delivery without the streaminfo informative status.
