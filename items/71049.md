## Summary

Current main still hardcodes the Teams DM streaminfo/informative-status path and does not expose a config option to disable it. The request is recent, unprotected, not maintainer-authored, and the provided related matches are generic title overlaps rather than a canonical duplicate for MS Teams DM stream status.

## Best Possible Solution

Keep #71049 open. The best fix belongs in the bundled `msteams` plugin: add an explicit DM stream-status/native-typing config field to `MSTeamsConfig`, the zod schema, generated metadata, and docs; pass it into the reply stream controller so personal chats can skip `TeamsHttpStream` and use native typing plus final-message delivery; add tests proving the default still streams while the opt-out sends no `streaminfo` informative/streaming activities.
