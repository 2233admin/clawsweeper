## Summary

Current main still lacks a Microsoft Teams setting to disable the Copilot-style informative stream status in 1:1 DMs. The issue is recent, unprotected, and actionable in the bundled `@openclaw/msteams` plugin rather than stale cleanup.

## Best Possible Solution

Keep the issue open and implement a Teams plugin config option that can disable personal-chat `TeamsHttpStream`/`streaminfo` delivery, use native typing when requested, and send a single final message. The likely change belongs in `extensions/msteams/src/reply-stream-controller.ts`, `extensions/msteams/src/reply-dispatcher.ts`, `src/config/types.msteams.ts`, `src/config/zod-schema.providers-core.ts`, generated config metadata/docs, and focused msteams tests covering DM stream disabled vs existing default behavior.
