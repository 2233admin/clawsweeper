## Summary

Current `main` already implements unique-prefix exec approval lookup in a safer form than this PR: prefix matching is limited to approval lookup paths, resolved events use the canonical full id, duplicate explicit-id checks stay exact, and `system.run` replay checks still require the exact run id. The same behavior and tests are present in shipped release `v2026.4.23`.
