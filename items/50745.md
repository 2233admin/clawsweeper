## Summary

Keep open. Current `main` does not ship `lookupPnLidEntry`, and the PR still needs maintainer review against the newer WhatsApp LID-resolution path already on `main` rather than an auto-close. The existing code now centralizes LID resolution through `resolveJidToE164` with authDir reverse-mapping fallback plus Baileys lookup fallback, while the proposed listener API is absent from the current `ActiveWebListener` surface.
