## Summary

Keep open. Current main still has only the merged watchdog mitigation from #67401, not the requested prompt transition when visible assistant content finishes. The TUI still marks chat deltas as `streaming`, clears normal busy state on `final` or lifecycle terminal events, and the lifecycle terminal path can still wait for block/channel flush work before the gateway emits the terminal chat event.

## Best Possible Solution

Keep this issue open until OpenClaw decouples the TUI's visible `streaming` state from full chat/lifecycle completion. The cleanest maintainer path is an additive content-complete or finalizing signal when model-visible output is done, or carefully scoped TUI logic that moves from `streaming` to a distinct non-streaming finalizing state after visible deltas stop while still tracking the active run. Add regression coverage for delayed lifecycle finals, concurrent runs, tool-only or hidden deltas, and the existing watchdog fallback.
