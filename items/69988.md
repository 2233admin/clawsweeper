## Summary

Keep #69988 open. Current main at 43a003b8 still exposes compaction only under agents.defaults.compaction: the per-agent AgentConfig type and strict AgentEntrySchema do not accept agents.list[].compaction, resolved agent config does not carry compaction, and Pi compaction runtime paths still read defaults-only settings. The PR discussion shows active review follow-up and a related reporter explicitly identifying this PR as the cleaner focused successor to #60807, so it is not a conservative cleanup-close candidate.

## Best Possible Solution

Keep #69988 open as the current focused implementation path. Maintainers should review and land an agent-aware config/schema/resolution change that accepts agents.list[].compaction, merges it over agents.defaults.compaction, threads the effective config into Pi settings, direct/manual compaction, safeguard runtime setup, timeout/hooks/truncation paths as needed, and adds schema/runtime tests plus docs. After a reviewed implementation lands, close the older duplicate/superseded per-agent compaction reports against the merged PR.
