## Summary

Keep #69988 open. Current main still only supports compaction configuration under `agents.defaults.compaction`; it does not accept `agents.list[].compaction`, does not merge compaction into resolved per-agent config, and the embedded Pi compaction paths still read defaults-only settings. The PR is recent, active, and the discussion shows it supersedes older duplicate attempts while addressing review feedback.

## Best Possible Solution

Keep #69988 open as the current focused implementation path. Maintainers should review and, if acceptable, land an agent-aware config/schema/resolution change that accepts `agents.list[].compaction`, merges it over `agents.defaults.compaction`, threads the effective config into Pi settings, direct/manual compaction, safeguard runtime setup, and any other compaction consumers intended to be overrideable, with schema/runtime tests and docs. After it lands, close the older duplicate feature reports against the merged PR.
