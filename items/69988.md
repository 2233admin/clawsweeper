## Summary

Keep #69988 open. Current main still does not implement the requested per-agent compaction override surface: `agents.list[]` has no `compaction` type/schema field, resolved agent config does not carry compaction, and key Pi compaction runtime paths still read `agents.defaults.compaction` directly. The related PRs in the provided context are closed/unmerged duplicates or prior art, so this open PR remains a plausible implementation candidate rather than cleanup-closeable.

## Best Possible Solution

Keep the PR open and have maintainers review or rework it into a shared effective compaction resolver for `agents.list[].compaction` over `agents.defaults.compaction`, then wire that resolver through schema/types, `resolveAgentConfig`, Pi settings, safeguard extension setup, compaction target/model selection, safety timeout, post-compaction hooks, memory flush, and manual/preflight compaction paths with focused tests for defaults fallback, per-agent `customInstructions`, nested `qualityGuard`/`memoryFlush` merges, and explicit/session-derived agent ids.
