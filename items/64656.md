## Summary

Keep this PR open. Current main has shipped partial mitigations for sticky auto-failover and bounded overflow recovery, but it still does not implement the PR's dedicated failoverState model or the /new-vs-/reset LCM conversation identity split. The PR also has unresolved review findings on context-engine key consistency and legacy /reset semantics, so the best maintainer path is rebase and narrowing rather than closing as obsolete.

## Best Possible Solution

Keep this PR open, require a rebase onto current main, and narrow it into focused changes. The failover portion should either adopt a dedicated failoverState model that coexists cleanly with the shipped auto-override cleanup or be dropped if maintainers prefer the current override-source approach. The LCM portion should implement memoryConversationKey consistently across gateway reset, auto-reply session reset, embedded run params, compaction, bootstrap, assemble, afterTurn, and ingest paths, with legacy /reset continuity preserved. The overflow portion should keep only diagnostics or circuit-breaker behavior that improves on current bounded overflow recovery, and the P1/P2 review comments should be addressed before maintainer review.
