## Summary

Keep open. Current main still matches the reported memory-core behavior: light dreaming ingests daily/session signals, then sorts by `lastRecalledAt` before `recallCount` and slices to the light limit; deep promotion still counts `dailyCount` and `groundedCount` together with real `recallCount`; and rehydration still uses the narrow `preferredSpan + 3`/`8` window. The reporter’s reproduction and follow-up analysis are concrete and align with current source rather than an obsolete compiled artifact.

## Best Possible Solution

Keep the issue open for a targeted memory-core fix. The likely maintainer path is to add regression coverage for light-phase ordering with mixed daily-only and real recall entries, decide whether deep promotion gates should use recall-only versus mixed signal counts, recalibrate scoring if daily/grounded signals remain eligible, and widen or redesign `relocateCandidateRange` for shifted multi-line snippets.
