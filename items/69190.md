## Summary

Keep open. The report is recent and plausibly covers two still-actionable areas: a site/runtime-specific browser timeout on WSL/Docker and a tool-governance/curl fallback complaint that may overlap with the still-open #47487 exec/profile bypass, but is not fully duplicated by it. Current main has hard tool-policy controls and browser timeout knobs, yet the issue lacks the target URL, logs, and exact config needed to prove fixed, unreproducible, or purely misconfigured.

## Best Possible Solution

Keep the issue open. Ask for the concrete target URL, current version retest, effective tools config, session transcript/tool list showing the curl call, and browser/gateway logs from WSL/Docker. Maintainers should first reproduce the browser timeout on current main or latest release; if the curl fallback is caused by exec/process remaining callable, either route the reporter to the #47487 fix path or add a focused regression showing that a browser-only policy really exposes only browser, with docs clarifying that prompt text and disabling web_fetch do not hard-disable shell curl unless exec/process are also removed by tool policy.
