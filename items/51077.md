## Summary

Keep open. Current `main` does not already implement this PR’s core macOS behavior changes, so `implemented_on_main` does not apply, and none of the other allowed close reasons fit. The menu-bar app still opens the anchored panel on left-click, not the proposed persistent window; the Talk Mode initialization path still uses the immediate `Task { await TalkModeController.shared.setEnabled(...) }` call that this PR was trying to change; and while drag-and-drop support now exists on `main`, that only covers one slice of the PR. This remains useful but unmerged work, not a safe conservative close.
