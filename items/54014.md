## Summary

Keep open. Current main partially honors OPENCLAW_HOME for shared state/config resolution, but the reported git checkout and installer/onboarding defaults still derive from os.homedir() or $HOME. The related open #65536 PR is narrower sandbox tilde-expansion work and does not supersede this broader path-contract issue.

## Best Possible Solution

Keep this issue open and narrow the remaining work to the OPENCLAW_HOME path contract. If git checkout and installer/onboarding defaults are considered internal OpenClaw paths, update src/cli/update-cli/shared.ts, scripts/install.sh, docs/help/environment.md, docs/cli/update.md, docs/install/installer.md, and focused tests so OPENCLAW_HOME seeds those defaults unless OPENCLAW_GIT_DIR, OPENCLAW_STATE_DIR, or OPENCLAW_CONFIG_PATH is explicitly set. Review #65536 separately as sandbox tilde-expansion work.

## Risks / Open Questions

- Some os.homedir() and $HOME uses are likely intentionally tied to the real OS account, such as shell startup files, platform trash, browser/profile discovery, and personal skill locations, so the fix should not be a mechanical global replacement.
- Maintainers still need to decide whether git checkout, installer workspace, and personal ~/.agents skills paths are part of the OPENCLAW_HOME internal-path contract or should be documented as explicit exceptions.
