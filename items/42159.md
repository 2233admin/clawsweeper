## Summary

Keep open. Current `main` still waits for the full model response before starting call playback, and the telephony TTS path still buffers complete synthesized audio before Twilio chunk sending begins. That means the issue's two latency bottlenecks are not implemented away on `main`.

## Evidence

- **Inbound voice response still serializes LLM completion before TTS:** `handleInboundResponse` awaits `generateVoiceResponse(...)` and only after that calls `this.manager.speak(callId, result.text)`, so playback cannot start until the full response text exists.
  - file: [extensions/voice-call/src/webhook.ts:813](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/extensions/voice-call/src/webhook.ts#L813)
  - command: `nl -ba extensions/voice-call/src/webhook.ts | sed -n '780,845p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Response generator still collapses completed payloads into one final string:** `generateVoiceResponse` awaits `runEmbeddedPiAgent(...)`, then post-processes `result.payloads` with `extractSpokenTextFromPayloads(...)` into a single string. There is no sentence-level or incremental handoff to TTS here.
  - file: [extensions/voice-call/src/response-generator.ts:249](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/extensions/voice-call/src/response-generator.ts#L249)
  - command: `nl -ba extensions/voice-call/src/response-generator.ts | sed -n '1,280p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Twilio stream playback still waits for full synthesized audio buffer:** `playTtsViaStream` starts keepalive silence, but it does not send spoken audio until `await ttsProvider.synthesizeForTelephony(text)` resolves; only then does it iterate `chunkAudio(muLawAudio, CHUNK_SIZE)` and send chunks to Twilio.
  - file: [extensions/voice-call/src/providers/twilio.ts:676](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/extensions/voice-call/src/providers/twilio.ts#L676)
  - command: `nl -ba extensions/voice-call/src/providers/twilio.ts | sed -n '600,760p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Telephony TTS contract still returns a complete audioBuffer:** `createTelephonyTtsProvider` awaits `runtime.textToSpeechTelephony(...)` and then converts the returned `audioBuffer` to mu-law. The interface is whole-buffer based, not streaming.
  - file: [extensions/voice-call/src/telephony-tts.ts:39](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/extensions/voice-call/src/telephony-tts.ts#L39)

  - file: [extensions/speech-core/src/tts.ts:951](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/extensions/speech-core/src/tts.ts#L951)
