## Summary

Keep open: current `main` does not contain this hardening. Both uncaught-exception handlers still exit unconditionally, the gateway upgrade path still lacks the raw-socket `error` listener described in the PR, and there is no `isStrictTransientNetworkError` implementation on `main`, so the PR is not implemented or obsolete.
