## Summary

Keep this issue open. Current main still has a /status path where a channels.modelByChannel-selected model can be the selected model, but stale sessionEntry.contextTokens is passed as the authoritative contextTokensOverride before the active model's configured window is consulted. The prior ClawSweeper discussion identified this same gap, and I did not find a later source or regression-test change that closes the channel-override case.

## Best Possible Solution

Keep this issue open and fix either the /status context resolution or the channels.modelByChannel application path so a channel-selected model can ignore, clear, or recompute stale sessionEntry.contextTokens when there is no explicit session model override. Add a regression test with a stale 1M session context value, a channel override to a smaller MiniMax-style model, and /status showing the active model's configured context window.
