## Summary

Keep open. Current main still throws for unsupported gateway service platforms, including OpenBSD, and the status paths still resolve the service before the fallback catch can handle read failures. The related PR #25690 directly addressed this with an unsupported-platform stub, but GitHub context shows it was closed unmerged, and current main does not contain that behavior.

## Best Possible Solution

Keep #25621 open until OpenClaw either implements a safe unsupported-platform `GatewayService` stub for read-only status paths, or a maintainer explicitly decides that unsupported platforms should receive a clearer, non-crashing diagnostic instead. PR #25690 is a useful prior implementation reference, but it was closed unmerged and should not be treated as shipped.
