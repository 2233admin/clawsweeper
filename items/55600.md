## Summary

Current checked-out `main` still carries the Control UI logo/favicon behavior reported in #55600, and the latest tagged release in this checkout (`v2026.4.22`, `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) still ships the same code. The linked fix from PR #56357 is not present here, so there is not strong evidence to close this issue.
