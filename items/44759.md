## Summary

Keep open. Current main still has the reported browser screenshot media-delivery path: screenshot results are converted to image results carrying a structured media path, `browser` remains trusted for local tool-result media, and pending tool media is attached to the next assistant/block reply or flushed as a media-only reply. The browser tool schema does not expose the requested `private`/visibility opt-out. The Pinchy comment is relevant supporting evidence of real shared-channel impact, and this is core tool-result delivery/security behavior rather than ClawHub scope.
