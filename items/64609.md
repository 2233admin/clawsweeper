## Summary

Current main still matches the reported Telegram multi-account bug. Account merging deliberately drops channel-level `groups` when more than one Telegram account is configured, prompt/skill/topic resolution reads only that merged account config, while `requireMention` still falls back from account groups to channel-level groups. The linked fix PR #64650 is closed and unmerged in the provided context, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open. The best implementation is a Telegram-owned prompt/skills/topic lookup change, likely in `extensions/telegram/src/bot-core.ts`, that reads `channels.telegram.accounts.<accountId>.groups` first and falls back to `channels.telegram.groups` only for group-context prompt/skill/topic resolution. Keep account routing inheritance unchanged, and add a regression test in `extensions/telegram/src/bot.create-telegram-bot.test.ts` for two configured accounts with channel-level group/topic `systemPrompt` and `skills`. If maintainers reject fallback semantics, add a Telegram doctor warning and public docs clarification for multi-account channel-level group prompt config.
