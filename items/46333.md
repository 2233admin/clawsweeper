## Summary

Keep open. Current `main` only implements the OpenAI half of this request. `extensions/openai/tts.ts` now uses `fetchWithSsrFGuard(...)` with `ssrfPolicyFromHttpBaseUrlAllowedHostname(baseUrl)`, but `extensions/elevenlabs/tts.ts` still sends TTS requests with raw `fetch()` against configurable `baseUrl`, so the SSRF/API-key-exfiltration concern is not fully fixed on `main`. The PR also targets obsolete code in `src/tts/tts-core.ts`, which no longer contains these providers, so the patch itself is stale but the underlying cleanup remains useful.
