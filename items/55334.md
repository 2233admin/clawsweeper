## Summary

Keep open. Current `main` has partial mitigations for session-store growth, especially cron-run pruning and transcript archiving, but it does not implement the core parts needed to conservatively close this report: `skillsSnapshot` is still stored per session entry, `sessions.json` is still eagerly deserialized as one JSON blob, and default subagent spawns still use `cleanup: "keep"`, so their session entries are not automatically archived/deleted.
