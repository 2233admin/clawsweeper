## Summary

Keep this issue open. Current main at edbab0e2db7f450f4d63f9700289ae1ba838a419 still lacks an explicit TUI observe or mirror mode for opt-in cross-channel activity. The issue has concrete use cases in the discussion, and the broader same-author #41548 remains open but does not clearly supersede this narrower read-only TUI monitoring workflow.

## Best Possible Solution

Keep this open as a core TUI/Gateway UX feature. Implement an explicit `openclaw tui --observe` or `tui.mirrorChannels` mode that preserves default `/new` isolation, aggregates recent same-agent activity through a deliberate Gateway/session event or history surface, relaxes exact-session filtering only in observe mode, labels channel/session origin, and enforces read-only or clearly scoped interactive behavior. Add focused tests for same-agent different-session events, channel-originated final and side-result events, history aggregation, and read-only behavior; coordinate with #41548 only if maintainers choose that broader issue as the canonical umbrella.
