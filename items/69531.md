## Summary

Keep open. Current `main` does not expose a dreaming-level `staggerMs` setting: the memory-core plugin schema/docs only advertise `enabled` and `frequency`, and the managed dreaming cron builder only passes `expr`/`tz`. Core cron already adds a default 5-minute stagger for recurring top-of-hour cron expressions, so the issue's `0 3 * * *` example is partly mitigated today, but explicit configuration for dreaming remains unimplemented. The issue is also cross-referenced to open PR #69548, so there is active follow-up work rather than a stale/incoherent report.
