## Summary

Keep open. Current main still applies the realpath containment check to all skill sources, including `openclaw-managed`, so a managed skill directory whose realpath escapes `~/.openclaw/skills` can still be skipped. The linked open PR #59219 appears to track the intended fix, but that fix is not present on current main.

## Evidence
