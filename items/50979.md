## Summary

Keep #50979 open. Current main has Slack Markdown-to-mrkdwn conversion for plain text sends, but it does not yet add the PR's message-tool formatting guidance. The PR is also not clearly merge-ready as-is: its current Markdown-only hint matches plain text sends but overstates behavior for Slack Block Kit payloads, which are passed through as mrkdwn blocks, and current inbound context still tells Slack agents to use Slack mrkdwn directly.

## Best Possible Solution

Do not close this PR as obsolete. The best path is to update the Slack prompt contract in one focused change: plain outbound reply/message text should be described as standard Markdown that OpenClaw converts to Slack mrkdwn, while Slack Block Kit or presentation block text should be described as native Slack mrkdwn. The fix should also reconcile or replace the current inbound response_format rules that still instruct agents to use Slack mrkdwn for Slack replies, and add tests that cover disabled/enabled interactive hints plus block vs non-block sends.
