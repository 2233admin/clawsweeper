## Summary

Keep open. Current `main` still cancels safeguard compaction when summarization fails, leaving the oversized session unchanged, and the automatic recovery path only resets sessions for overflow-shaped compaction failures, not generic API error/timeout/rate-limit summarization failures described in #42158.

## Evidence

- **Safeguard still cancels on summarization failure:** The safeguard LLM path catches summarization errors, records a human-readable cancel reason, and returns `{ cancel: true }` instead of writing a fallback compaction boundary or rotating the session.
  - file: [src/agents/pi-hooks/compaction-safeguard.ts:1118](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/agents/pi-hooks/compaction-safeguard.ts#L1118)
  - command: `nl -ba src/agents/pi-hooks/compaction-safeguard.ts | sed -n '724,1145p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Failed compaction propagates as non-compacted result:** The embedded compaction runner converts the thrown/cancelled safeguard result into `ok: false, compacted: false` and returns the failure reason; nothing in this path reduces session size.
  - file: [src/agents/pi-embedded-runner/compact.ts:1221](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/agents/pi-embedded-runner/compact.ts#L1221)

  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Overflow recovery keeps failed sessions blocked:** When auto-compaction fails during overflow recovery, the run returns an error payload telling the user to `/reset` or `/new`; it does not compact or auto-recover the existing session unless a later higher-level reset path fires.
  - file: [src/agents/pi-embedded-runner/run.ts:1226](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/agents/pi-embedded-runner/run.ts#L1226)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '1128,1328p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Automatic session reset only covers overflow-shaped compaction failures:** `isCompactionFailureError()` requires both a compaction-related message and an overflow/context-overflow shape. Rate-limit/API-auth/timeout summarization failures are excluded by the overflow heuristics, so the `resetSessionAfterCompactionFailure()` recovery in the reply runner does not cover the issue report's failure modes.
  - file: [src/agents/pi-embedded-helpers/errors.ts:189](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/agents/pi-embedded-helpers/errors.ts#L189)

- **Reply-runner reset path is narrower than the reported bug:** The reply runner wires a session-reset helper for compaction failures, but only invokes it when `isCompactionFailureError(message)` matches. That leaves non-overflow summarization failures without the reset escape hatch.
