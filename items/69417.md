## Summary

Keep open. Current main still does not implement the PR's central behavior for core `mcp.servers`: the schema and exported MCP config type only allow primitive literal `env`/`headers` values, the SecretRef target registry and agent-runtime command target set omit `mcp.servers.*.env.*` and `mcp.servers.*.headers.*`, and core runtime secret collection has no MCP collector. Existing ACPX plugin `mcpServers.*.env.*` SecretInput support is real, but it is a plugin surface and does not satisfy this PR's core `mcp.servers` request. The PR discussion also shows reviewers found edge cases and the author posted follow-up fixes, so the branch still appears to contain potentially useful work rather than being obsolete.

## Best Possible Solution

Keep the PR open, or replace it with a narrower maintainer-owned patch that implements core `mcp.servers.*.env.*` and `mcp.servers.*.headers.*` SecretInput support end to end: schema/types, generated config artifacts, target registry, command/runtime resolution for active MCP transports, `secrets configure`/`audit` heuristics, docs, and regression tests. Existing ACPX plugin SecretInput support should remain documented as related but not treated as completing the core `mcp.servers` surface.
