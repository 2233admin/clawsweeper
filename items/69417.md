## Summary

Keep open. Current main still treats core `mcp.servers.*.env.*` and `mcp.servers.*.headers.*` as scalar string/number/boolean config, does not list them as SecretRef targets, and does not resolve them in the agent runtime or core secrets runtime. Related ACPX/plugin MCP SecretRef support and static MCP header forwarding are distinct and do not supersede this PR.

## Best Possible Solution

Keep the PR open or replace it with a focused maintainer-owned patch that adds `SecretInput` to core MCP env/header schema and types, registers `mcp.servers.*.env.*` and `mcp.servers.*.headers.*` as supported SecretRef targets, resolves only the active MCP transport surface at runtime, updates `secrets configure`/`secrets audit` heuristics, and documents/tests the new credential surface.
