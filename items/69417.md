## Summary

Do not close PR #69417. Current main still lacks the central SecretRef support for core `mcp.servers.*.env.*` and `mcp.servers.*.headers.*`: config types/schema remain primitive-only, the secret target registry and agent command target list do not include MCP env/header targets, and runtime secret collection does not materialize MCP SecretRefs before launch. The PR discussion also shows review feedback was actively addressed, so this remains plausible useful work for maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review, or land a focused equivalent implementation that adds `SecretInput` support to MCP env/header config schema and types, registers both MCP targets, resolves only the active MCP transport surface at runtime, updates `secrets configure` and `secrets audit` heuristics, and documents/tests the new supported SecretRef surfaces.
