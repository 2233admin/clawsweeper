## Summary

Keep open. Current `main` still has the malformed-provider crash surface in `buildAssistantMessageFromResponse`, so this is neither implemented on `main` nor safely obsolete. The original PR is also stale against current code because `response.output` is now used through `.filter()` before the proposed loop-only guard would help.

## Evidence

- **Current main still assumes `response.output` is an array:** `buildAssistantMessageFromResponse` calls `(response.output ?? []).filter(...)`, which still throws for non-array truthy payloads from third-party relays.
  - file: [src/agents/openai-ws-message-conversion.ts:503](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/openai-ws-message-conversion.ts#L503)

  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)

  - file: [src/agents/openai-ws-message-conversion.ts:510](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/openai-ws-message-conversion.ts#L510)
