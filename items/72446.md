## Summary

Keep open. Current main still lets Microsoft Foundry onboarding select any succeeded deployment and still routes the built-in provider through OpenAI-compatible Foundry URLs/tests only. The PR is a focused implementation candidate for #60546, but it needs normal maintainer review because the provided diff appears to filter the prompt choices without filtering the persisted discovered deployment catalog.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to finish the provider-owned guardrail in `extensions/microsoft-foundry`: filter Anthropic deployments consistently from both selection and persisted discovered model catalog, keep the custom-provider workaround note, and add regression tests for mixed and Anthropic-only deployment lists before landing it as the implementation candidate for #60546.
