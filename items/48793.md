## Summary

Current `main` does not implement the proposed centralized channel-plugin resource manager. Cleanup remains plugin-specific, the gateway only offers optional per-plugin `stopAccount` hooks, and several channel plugins still use fire-and-forget server shutdown paths.

## Evidence

- **Gateway contract is still optional per-plugin lifecycle, not a centralized resource manager:** `ChannelGatewayAdapter` exposes optional `startAccount` and `stopAccount` hooks only; there is no shared `PluginResourceManager`-style interface or shared `cleanup()` contract.
  - file: [src/channels/plugins/types.adapters.ts:337](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/channels/plugins/types.adapters.ts#L337)
  - command: `sed -n '237,345p' src/channels/plugins/types.adapters.ts`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Gateway stop path depends on plugin-specific hooks:** `stopChannel()` aborts the running task and calls `plugin.gateway.stopAccount` only when that plugin implements it, so cleanup is not centrally standardized across channel plugins.
  - file: [src/gateway/server-channels.ts:528](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/server-channels.ts#L528)
  - command: `sed -n '528,590p' src/gateway/server-channels.ts`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Most channel plugins still do not implement `stopAccount`:** Repository search found `stopAccount` implementations only in `extensions/synology-chat/src/channel.ts` and `extensions/twitch/src/plugin.ts`; the named channels from the issue do not provide a shared stop hook.
  - file: [extensions/synology-chat/src/channel.ts:280](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/synology-chat/src/channel.ts#L280)

  - file: [extensions/feishu/src/monitor.state.ts:146](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/feishu/src/monitor.state.ts#L146)

- **Telegram and Matrix still use plugin-specific cleanup strategies:** Telegram webhook shutdown still calls `server.close()` directly, while Matrix has its own bespoke async `cleanup()` routine. Current `main` therefore still has inconsistent cleanup patterns across channel plugins.
