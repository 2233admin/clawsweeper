## Summary

Keep this PR open. Current main still lacks the embedded-runner payload sanitizer for `tools: []` and orphaned `tool_choice`, while the related `toolsAllow` forwarding work is already present, so this remains a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review or a successor patch. The best fix is a narrow embedded-runner egress sanitizer, preferably using the existing payload-patch pattern, with focused tests for empty `tools`, orphaned `tool_choice`, non-empty tool preservation, `onPayload` composition, session HTTP fallback, boundary-aware/custom streams, and explicit proof for provider-owned/websocket paths.
