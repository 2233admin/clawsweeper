## Summary

Keep open. Current `main` still documents and implements the default `memory-core` path as a prompt-driven pre-compaction memory flush, not as a deterministic transcript-extraction pipeline that writes daily logs automatically. The repo also ships a separate memory plugin with auto-capture behavior, which reinforces that this is an active product/architecture choice rather than a clearly obsolete or already-resolved report.
