## Summary

Keep this issue open. Current main still has the reported runtime gap: streamed `choice.delta.content` is accepted on truthiness rather than a runtime string guard, fallback reasoning fields are string-only, and there is no Mistral-specific stream wrapper or regression test for array/object reasoning blocks. The public PR page also still shows #67203 open with closing syntax for this issue, so cleanup rules require leaving the issue open while that implementation candidate remains unresolved. ([github.com](https://github.com/openclaw/openclaw/pull/67203))

## Best Possible Solution

Keep this issue open. Maintainers should review #67203 or land an equivalent focused fix in `src/agents/openai-transport-stream.ts`: string-guard streamed text content, normalize Mistral `reasoning_content` and content block arrays/objects into standard thinking/text deltas, and add regression coverage for the non-string Mistral stream shapes. Once that PR is merged, rejected, or explicitly superseded by a landed fix, this issue can be closed as implemented.
