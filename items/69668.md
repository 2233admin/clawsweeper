## Summary

Keep open. Current `main` supports multiple isolated agents and agent-scoped session grouping, but there is no built-in team/project entity, no team/group metadata on agent config or `agents.list`, and the Control UI still renders agents as a flat picker. This is a coherent product request rather than something already implemented or safely auto-closeable.

## Evidence

- **Agent config has no team/project field:** `AgentConfig` includes `id`, `name`, `workspace`, `agentDir`, `identity`, `groupChat`, `subagents`, and runtime/tool settings, but no team/group/project metadata that could drive hierarchical UI grouping.
