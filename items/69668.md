## Summary

Keep issue #69668 open. Current main has multi-agent isolation and session routing, but it does not implement the requested team/project parent grouping or hierarchical session-list UI. The existing Control UI sessions view remains a flat table, the agent/session protocol shapes do not carry team/project metadata, and the plugin system does not expose a Control UI session-list contribution surface that would make this a clean ClawHub-only item today.

## Best Possible Solution

Keep the issue open for maintainer product judgment. A likely implementation would add an explicit core grouping model, such as `agents.list[].team`/`project` metadata or a separate `agents.groups` config shape, expose it through `agents.list`/session list data, and update the Control UI Sessions and Agents views to group by that metadata while preserving current per-agent workspace/session isolation. If maintainers want this to be plugin-owned instead, the prerequisite would be a generic Control UI contribution/API for session-list grouping; current plugin surfaces are not enough for that.
