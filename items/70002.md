## Summary

Keep #70002 open. Current main still lacks the fork-only job guards in both docs workflows, while both jobs depend on an upstream-only docs sync token and operate against the external openclaw/docs publish repo. The PR is a focused 2-line CI fix with a clear reproduction and positive bot review; it is recent and has maintainer attention via triage:risky-infra, so it should receive explicit workflow review rather than be auto-closed.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land the two job-level repository guards, or an equivalent upstream-only guard, after confirming maintainers are comfortable skipping these publish/translate jobs outside `openclaw/openclaw`. No core product or docs implementation is needed beyond the workflow change.
