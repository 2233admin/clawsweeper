## Summary

Keep open. Current main still lacks the requested Feishu `ignoreAtAll` / `ignoreBroadcast` behavior: `@_all` is treated as a bot mention unconditionally, and the Feishu config schema, generated metadata, docs, and tests do not expose the new option. The PR is a focused bundled Feishu plugin improvement rather than obsolete or ClawHub-only, but it should not land as submitted because the review comments identify a real direct-mention suppression bug.

## Best Possible Solution

Keep the PR open or replace it with a corrected implementation for issue #59401. The right fix belongs in the bundled Feishu plugin: add top-level and per-group `ignoreAtAll` config, resolve it before mention gating, treat `@_all` as a mention only when the option is disabled, still fall through to structured direct-bot mention checks when enabled, and add Feishu tests/docs/generated config metadata for both broadcast-only and broadcast-plus-direct-mention cases.
