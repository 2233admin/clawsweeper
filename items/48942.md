## Summary

Current `main` does not already implement this PR, so there is no basis to close it as shipped or obsolete. `deriveSessionTitle` still lives in `src/gateway/session-utils.ts`, still ignores `entry.label` and `entry.origin?.label`, and the duplicated tests still live in `src/gateway/session-utils.test.ts`. At the same time, the PR as described carries a plausible regression: `origin.label` is auto-derived for direct chats, while multiple consumers prefer `derivedTitle` ahead of `displayName`, so moving `origin.label` ahead of transcript-derived titles would change visible session titles broadly. That means this item is neither safely closeable nor clearly ready to discard.

## Evidence

- **Current main still uses the old title precedence:** `deriveSessionTitle` on `main` returns `displayName`, then `subject`, then transcript `firstUserMessage`, then `sessionId`; it does not consider `entry.label` or `entry.origin?.label`.
  - file: [src/gateway/session-utils.ts:188](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/session-utils.ts#L188)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Test consolidation is not implemented on main:** The `deriveSessionTitle` tests are still embedded in `session-utils.test.ts`, including the stale `updatedAt missing` description. There is no dedicated `src/gateway/session-title.ts` or `src/gateway/session-title.test.ts` in the checked-out tree.
  - file: [src/gateway/session-utils.test.ts:990](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/session-utils.test.ts#L990)

- **`origin.label` is auto-populated for direct chats:** Gateway session metadata derives `origin.label` from `resolveConversationLabel(ctx)`, and for `ChatType === "direct"` that resolver returns `SenderName` or `From`. This means unlabeled inbound direct sessions commonly have an `origin.label` even without an explicit user label.
  - file: [src/config/sessions/metadata.ts:54](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/config/sessions/metadata.ts#L54)

- **Derived titles outrank display names in user-facing consumers:** The TUI session picker uses `session.derivedTitle ?? session.displayName`, and ACP metadata similarly prefers `derivedTitle` before `displayName`/`label`. A precedence change that makes `origin.label` win inside `deriveSessionTitle` would therefore be user-visible, not just a test cleanup.
  - file: [src/tui/tui-command-handlers.ts:174](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/tui/tui-command-handlers.ts#L174)

## Risks / Open Questions

- Closing this PR would discard work that is not present on `main` today.
- Treating `origin.label` like an explicit user label appears risky: for direct chats it is populated automatically, so the PR's precedence change could replace transcript summaries across unlabeled sessions.
