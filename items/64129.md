## Summary

Current main still has the reported boundary: Google API-key setup can set Gemini as the primary default model, while heartbeat runs continue through the normal reply path and only avoid that default when an explicit heartbeat model override is configured. Empty/comment-only HEARTBEAT.md skipping reduces some background calls, but actionable HEARTBEAT.md content still runs and can inherit the paid provider default.

## Best Possible Solution

Keep this issue open and fix the boundary explicitly. The best maintainer path is to preserve an existing primary model during Google/provider auth unless the user clearly confirms the switch, or to require confirmation/disablement before heartbeat can inherit a newly configured paid external provider. Add regression coverage for an existing actionable HEARTBEAT.md plus paid-provider default-model setup.
