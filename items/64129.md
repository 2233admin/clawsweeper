## Summary

Keep open. Current main still has the paid-background-usage path: Google API-key setup can apply the Gemini default model, while heartbeat remains a scheduled main-session turn by default and only avoids default-model inheritance when users explicitly configure a heartbeat model or isolated/light context.

## Best Possible Solution

Keep #64129 open for a maintainer fix. The likely fix should make paid provider auth/setup avoid silently changing the primary model, or preserve the pre-auth primary before provider config patches run, and add a Google-specific regression test. A stronger product fix would also guard heartbeat background turns from newly paid defaults unless the user explicitly configures heartbeat `model`, `isolatedSession`, `lightContext`, or confirms paid background usage.
