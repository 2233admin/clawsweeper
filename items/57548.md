## Summary

Current `main` still has the Telegram session-metadata mismatch described by this PR. The production path persists inbound metadata to `ctxPayload.SessionKey` (the slash envelope key) instead of the resolved target session, and the regression tests on `main` still lock in that old slash-key behavior.
