## Summary

Keep open. Current `main` still has the warn-only compaction-checkpoint miss path, still rotates session ids on compaction failure in the reply runner, and the WebChat gateway path re-reads the latest session mapping for transcript writes, so the reported silent remap/orphaning remains plausible and is not clearly fixed on `main`.

## Evidence

- **Checkpoint persist still bails out with only a warning:** `persistSessionCompactionCheckpoint` checks `store[target.canonicalKey]`, returns immediately when `sessionId` is missing, then logs `skipping compaction checkpoint persist: session not found` and returns `null`. There is no recovery or invariant enforcement in this path.
  - file: [src/gateway/session-compaction-checkpoints.ts:166](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/gateway/session-compaction-checkpoints.ts#L166)
  - command: `nl -ba src/gateway/session-compaction-checkpoints.ts | sed -n '120,188p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Compaction failure path still rotates the active session mapping:** The reply runner defines `resetSessionAfterCompactionFailure`, and `resetReplyRunSession` generates a new session id, rewrites `store[sessionKey]`, updates the session file, and refreshes queued followups. That is the same class of session-id rotation the report describes.
  - file: [src/auto-reply/reply/agent-runner.ts:1185](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/auto-reply/reply/agent-runner.ts#L1185)
  - command: `nl -ba src/auto-reply/reply/agent-runner.ts | sed -n '1185,1190p'; nl -ba src/auto-reply/reply/agent-runner-session-reset.ts | sed -n '37,106p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **WebChat chat.send follows the latest remapped session id/file:** The gateway `chat.send` flow repeatedly calls `loadSessionEntry(sessionKey)` and then uses `latestEntry?.sessionId` / `latestEntry?.sessionFile` for transcript updates and assistant appends. If the store mapping changes mid-run, WebChat silently starts writing against the new session entry.
  - file: [src/gateway/server-methods/chat.ts:2341](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/gateway/server-methods/chat.ts#L2341)
