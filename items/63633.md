## Summary

Keep this PR open. Current main still has the Telegram media SOCKS proxy failure path, and #60472 was closed specifically because this open PR was the canonical follow-up. The PR is useful but not ready as-is: the remaining fix belongs in the guarded explicit-proxy/media path, and the current diff over-claims SOCKS variants that undici 8.1.0 rejects.

## Best Possible Solution

Keep this PR open and request focused rework against current main: allow only undici-supported socks5:// and socks:// explicit proxies through the guarded media path, preserve proxy-host validation and Telegram dispatcher policy/pool behavior, remove unsupported socks4/socks4a/socks5h claims unless separately implemented, and add regression tests for channels.telegram.proxy: socks5:// covering Telegram media downloads and transport behavior.
