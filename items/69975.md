## Summary

Keep open. Current main still has the misleading cron-only `--tz` help text in both `cron add` and `cron edit`, while the runtime and tests already show `--tz` is applied to offset-less ISO `--at` values. The PR is useful but should be revised before merge because its proposed `(default: UTC)` wording is wrong for cron expressions, which default to the gateway host timezone.

## Best Possible Solution

Keep the PR open, but ask the author to revise the help text instead of closing it. The likely maintainer path is to update both `cron add` and `cron edit` so `--tz` says it applies to `--cron` and offset-less ISO `--at` values, while avoiding a blanket `default: UTC` claim; if defaults are mentioned, split them explicitly: offset-less `--at` without `--tz` is UTC, cron without `--tz` uses the gateway host timezone. Coordinate separately with #59441/#70046 for bare `HH:MM` behavior.
