## Summary

Keep #69975 open. Current main still has the old `--tz` help text in both `cron add` and `cron edit`, so the PR is not implemented on main. The PR is useful but should be amended before merge because its proposed combined help text says `default: UTC`, while cron schedules without `--tz` use the gateway host timezone; only offset-less `--at` parsing falls back to UTC when no timezone is supplied.

## Best Possible Solution

Keep #69975 open and ask for a small amendment: update `cron add` and `cron edit` help so `--tz` clearly applies to `--cron` and offset-less `--at` values, but avoid a combined `default: UTC` claim. The best wording should distinguish defaults: cron without `--tz` uses the gateway host timezone, while offset-less `--at` values are treated as UTC unless `--tz` is supplied.
