## Summary

Keep open. Current main has not implemented the PR's proposed doctor auto-enable skip: the auto-enable materialization path still writes `plugins.entries` and allowlist entries for non-built-in channel plugin candidates. The PR also has unresolved review concerns around workspace plugin activation and allowlist behavior, so it needs maintainer revision rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review or request changes. The best fix is to first verify the duplicate-warning reproduction on current main, then make any skip narrowly target the proven global-installed duplicate case while preserving workspace plugin trust/enablement and handling restrictive `plugins.allow`; add regression tests for global, workspace, bundled, and allowlist cases.
