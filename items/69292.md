## Summary

Keep this PR open. Current main still has the pre-PR doctor auto-enable behavior, so it is not implemented on main, and the work is a core doctor/plugin trust fix rather than ClawHub work. The PR also has unresolved functional review risk around workspace and allowlist semantics, so the best cleanup is maintainer review/revision, not auto-close.

## Best Possible Solution

Keep this PR open for maintainer review. The fix likely belongs in core doctor/plugin auto-enable logic, but it should be revised to align with existing activation/trust policy: only skip materializing config when the plugin is already trusted and will actually participate, preserve workspace plugin explicit enablement, cover restrictive `plugins.allow`, and add regression tests across global, workspace, bundled, and allowlist cases before landing.
