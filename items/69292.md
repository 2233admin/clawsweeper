## Summary

Keep open. Current main does not implement the PR's proposed doctor auto-enable skip, and the PR still targets a plausible core doctor/plugin-loading bug rather than ClawHub cleanup. However, the patch as described needs revision before merge because current policy explicitly keeps workspace-origin plugins disabled unless they are allowlisted or enabled, and the existing restrictive `plugins.allow` path can still fall through to `registerPluginEntry`.

## Best Possible Solution

Keep the PR open and ask for a narrowed core fix: skip redundant `plugins.entries` only for origins that the loader actually activates without an explicit entry, preserve workspace plugin enablement semantics, handle restrictive `plugins.allow` intentionally, and add regression tests for those cases. Do not close as implemented, duplicate, or ClawHub because the current-main doctor path still lacks the requested behavior and this is core config repair logic.
