## Summary

Keep open. Current `main` still contains the same failure path the issue describes: exec-event heartbeats use a generic prompt with no injected payload, isolated heartbeat runs can classify events from the base session but drain system messages from the isolated `:heartbeat` session instead, and `process poll` still does not cancel the pending `notifyOnExit` wake. The relevant files are unchanged between `v2026.4.23` and current `HEAD`.
