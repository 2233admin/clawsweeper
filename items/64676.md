## Summary

Keep open. Current main still regenerates per-agent `models.json` during gateway startup/model warmup and the merge path still replaces matching providers from regenerated config/catalog data while preserving only selected secret/base URL fields. That means the reported user-pruned provider model list can still be restored after restart, and the current docs/tests do not settle whether pruning `models.json` is supported or unsupported.

## Best Possible Solution

Do not close yet. Maintainers should decide the intended contract for per-agent `models.json`: if user-pruned provider model arrays are supported, add a gateway-restart regression around `ensureOpenClawModelsJson`/`planOpenClawModelsJsonWithDeps` and preserve deletions or introduce an explicit tombstone/replace mechanism. If `models.json` is generated cache, update docs, doctor, and CLI guidance so persistent edits go through `models.providers` or `agents.defaults.models` with the intended `--merge`/`--replace` flow.
