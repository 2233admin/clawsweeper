## Summary

Keep this issue open. Current main still has the reported bundled `gh-issues` skill frontmatter and size problems, and the related PR that would have removed the redundant `user-invocable` field was closed unmerged. The evidence points to an unresolved contract mismatch between the shipped skill, the skill-creator validator, and runtime/docs support for `user-invocable`, not to a cleanup item that is already fixed or safely closeable.

## Best Possible Solution

Keep this issue open and fix it with a focused bundled-skills cleanup: decide whether `user-invocable` is a supported public frontmatter key, then align `skills/gh-issues/SKILL.md`, `skills/skill-creator/scripts/quick_validate.py`, skill-creator guidance, public skills docs, and tests. Because runtime defaults `user-invocable` to true, removing the redundant field from this bundled skill is likely the narrowest immediate fix; separately split the large orchestration material into `references/` files while keeping the top-level `SKILL.md` concise and trigger-oriented.
