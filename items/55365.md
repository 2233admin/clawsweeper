## Summary

Keep open. Current `main` does not reject embedded credentials on browser navigation URLs, still forwards the original URL into the Playwright navigation path, and only applies generic secret redaction rather than explicit `user:pass@` stripping on navigation errors. The request is also aligned with an existing supported auth mechanism (`setHTTPCredentials`), so this remains a valid browser-tool guardrail request rather than cleanup noise.

## Evidence

- **Navigation guard has no username/password rejection:** `assertBrowserNavigationAllowed()` trims, parses, checks protocol, proxy mode, and SSRF policy, but never rejects `parsed.username` or `parsed.password` before allowing navigation.
  - file: [extensions/browser/src/browser/navigation-guard.ts:93](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/browser/src/browser/navigation-guard.ts#L93)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Original navigation URL is still passed to Playwright:** `navigateViaPlaywright()` validates the input URL, then passes the same `url` through to `gotoPageWithNavigationGuard()`, whose route handler ultimately calls `page.goto(opts.url, ...)` with the original string.
  - file: [extensions/browser/src/browser/pw-session.ts:916](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/browser/src/browser/pw-session.ts#L916)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Navigation error formatting uses generic redaction only:** `formatErrorMessage()` returns `redactSensitiveText(formatted)`, and the default redaction patterns cover tokens, passwords, headers, and PEM blocks but do not include URL userinfo patterns like `https://user:pass@host`.
  - file: [src/infra/errors.ts:68](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/errors.ts#L68)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Supported alternative already exists:** The browser control surface already exposes explicit HTTP auth via `/set/credentials`, which calls Playwright `page.context().setHTTPCredentials({ username, password })`. That matches the issue's suggested alternative auth path.
  - file: [extensions/browser/src/browser/routes/agent.storage.ts:323](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/browser/src/browser/routes/agent.storage.ts#L323)
