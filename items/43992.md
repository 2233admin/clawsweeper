## Summary

Keep open. Current `main` still does not propagate `agent.wait` result payloads through the gateway or subagent wait paths, so the PR's stated bugfix is not implemented on `5fe333ada85afd0def9e85985ce57e36aa00108a`. The review discussion matters here: the author said the overwrite concern was addressed on the PR branch, but that work is not present on `main`, and the newer subagent wait refactor still relies on chat-history capture rather than preserved `agent.wait` results.

## Risks / Open Questions

- The race described in the PR remains plausible on current `main`: a subagent can finish before chat history exposes the final assistant reply, leaving announce/freeze logic without the terminal text.
- `src/agents/subagent-registry.ts` was refactored after the PR was opened, so the branch likely needs a rebase/adaptation, but that is not evidence that the underlying fix is obsolete.
