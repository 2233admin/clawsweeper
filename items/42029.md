## Summary

Current `main` does not implement Slack attachment text ingestion into inbound agent context. The active Slack path in `extensions/slack` still resolves direct files only as media/placeholders, and the tests still assert placeholder-only behavior, so this PR is not implemented on `main` and should stay open.
