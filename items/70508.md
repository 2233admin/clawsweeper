## Summary

Keep #70508 open. Current main still has the Mattermost draft-stream behavior described by the PR: assistant/reasoning boundaries only reset local partial-text state, and tool starts update the current draft post body without rotating to a new Mattermost post. The existing Mattermost draft stream already has a `forceNewMessage()` seam, and Slack uses the analogous boundary pattern, so this remains a plausible focused channel bug fix rather than obsolete work.

## Best Possible Solution

Keep the PR open as a real Mattermost channel bug fix candidate. The best path is a narrow Mattermost-owned patch that rotates/flushed draft preview posts at assistant-message, reasoning, and tool boundaries using the existing `forceNewMessage()` seam, accounts for the fire-and-forget callback ordering in the agent stream, and adds targeted regression tests in the Mattermost test lane before merge.
