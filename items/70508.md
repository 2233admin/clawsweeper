## Summary

Keep PR #70508 open. Current main still has the Mattermost draft-stream overwrite path the PR is trying to fix: assistant/reasoning boundaries only reset `lastPartialText`, and tool starts update the active draft post body without rotating to a new post. The PR discussion and review comments show a real, scoped Mattermost channel bug, but also unresolved sequencing/test concerns that need maintainer review or a revised patch rather than auto-closure.

## Best Possible Solution

Keep the PR open as a valid Mattermost channel bug-fix candidate. The best path is a Mattermost-owned patch that rotates/seals draft preview posts at assistant-message, reasoning, and tool boundaries, handles fire-and-forget item/tool callback ordering deterministically, and adds focused regression tests for streamed assistant text followed by tool status and later assistant text.
