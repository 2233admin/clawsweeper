## Summary

Keep #71416 open. Current main still catches any Gateway CLI failure, logs a fallback diagnostic through stderr, and then runs the embedded agent. The tests and docs describe that behavior, and I found no opt-in fallback flag, nonzero-exit behavior, stdout banner, or stable JSON transport/fallback marker that would satisfy the report.

## Best Possible Solution

Keep the issue open for a focused CLI behavior change. The best fix is to make Gateway fallback explicit and machine-detectable: fail closed unless an `--allow-embedded-fallback` option is passed, or add a stable JSON marker such as `meta.transport`/`meta.fallback` plus a clear warning path, then update `src/commands/agent-via-gateway.ts`, CLI registration, tests, and `docs/cli/agent.md` together.
