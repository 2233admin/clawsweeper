## Summary

Current main still implements the reported silent Gateway-to-embedded fallback: `openclaw agent` catches Gateway failures, writes only a stderr diagnostic, then runs the embedded agent. The CLI has no opt-in fallback flag, and the JSON embedded result path still returns the normal payload/meta envelope without a stable transport/fallback marker.

## Best Possible Solution

Keep #71416 open for a focused CLI contract change. The best fix is to make fallback explicit and script-detectable: fail closed unless an `--allow-embedded-fallback` flag is passed, or add a stable JSON marker such as `meta.transport`/`meta.fallback` plus a clear non-JSON warning path, then update `src/commands/agent-via-gateway.ts`, CLI option registration, tests, and `https://docs.openclaw.ai/cli/agent` together.
