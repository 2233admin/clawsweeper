## Summary

Keep #40210 open. Current main still routes workspace bootstrap files through the boundary file reader, rejects symlink targets that resolve outside the workspace root, and reports those failed guarded reads as missing Project Context entries. The linked fix PR #40230 is still separate from main, so the reported behavior is not implemented on main.

## Best Possible Solution

Keep the issue open and use PR #40230 or a maintainer-owned replacement as the implementation starting point. The likely fix belongs in `src/agents/workspace.ts` as a narrow bootstrap-file symlink fallback, with regression coverage for out-of-workspace symlinked bootstrap files while preserving hardlink rejection, file-size limits, and the broader boundary-path security behavior. If maintainers prefer #38622 as the canonical tracker, close only one issue explicitly as a duplicate after choosing that canonical item.
