## Summary

Keep open. Current `main` at `6944d7025d9c0ba4891cbe646ab5dab039d504ca` still has the underlying `/model list`/`/model status` Control UI bug, and the PR’s proposed fixes are not implemented on `main`. The PR also still appears to have substantive correctness risks in both the gateway and CLI halves, so there is not strong evidence for a conservative close.

## Evidence

- **Control UI still forwards `/model <args>` directly into `sessions.patch`:** The Control UI slash-command executor trims the `/model` arguments and sends them unchanged as `model: requestedModel` in `sessions.patch`, so `/model list` and `/model status` still hit the session-patch model resolver path on current `main`.
  - file: [ui/src/ui/chat/slash-command-executor.ts:211](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/ui/src/ui/chat/slash-command-executor.ts#L211)
  - command: `nl -ba ui/src/ui/chat/slash-command-executor.ts | sed -n '210,220p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **`sessions.patch` on `main` has no special handling for `list`/`status`:** Current `src/gateway/sessions-patch.ts` only rejects empty model strings, then calls `resolveAllowedModelRef(...)`. There is no guard for command aliases like `list` or `status`, so the behavior described in the PR is not already implemented on `main`.
  - file: [src/gateway/sessions-patch.ts:401](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/gateway/sessions-patch.ts#L401)

- **Current model resolution intentionally accepts bare aliases and bare model ids:** `resolveModelRefFromString` first checks alias matches for bare strings, and `resolveAllowedModelRefFromAliasIndex` then resolves other bare strings against a unique provider or the default provider. Hard-rejecting bare `list`/`status` in `sessions.patch` would therefore bypass the existing alias/model-id resolution rules rather than match current model-selection behavior.
  - file: [src/agents/model-selection-shared.ts:359](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/model-selection-shared.ts#L359)

- **Docs still describe `/model list` and `/model status` as valid chat commands:** The current docs list `/model`, `/model list`, and `/model status` as supported commands, which matches the reported UI misuse scenario and confirms this is still a live command surface rather than obsolete behavior.
