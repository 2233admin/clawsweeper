## Summary

Keep open. Current `main` still appears to allow an unrelated provider plugin's `normalizeConfig` hook to rewrite another provider's `baseUrl`, which matches the reported `/v1`-append failure mode for Z.AI and the corroborating Ollama comment. The downstream 404 classification part of the report is partly obsolete on `main` because HTTP 404 now maps to `model_not_found`, but the primary `models.json` normalization bug is not clearly fixed.

## Evidence

- **Cross-provider normalizeConfig fallback still exists:** `normalizeProviderConfigWithPlugin()` first checks the bundled/matched provider, then falls through to every provider plugin hook and accepts the first config mutation it sees. There is no guard that the fallback plugin must match the provider being normalized. See lines 416-437.
