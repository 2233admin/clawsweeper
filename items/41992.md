## Summary

Keep open. Current `main` has partial WebChat attachment support for images, but it does not implement general file upload/paste/drag-and-drop for documents/PDFs as requested in issue #41992.

## Evidence

- **WebChat UI is explicitly image-only:** The chat input uses `accept="image/*"`, and the active-attachment placeholder says `Add a message or paste more images...`, which does not match a general file-upload feature.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/ui/chat/attachment-support.ts#L1)
  - command: `nl -ba ui/src/ui/chat/attachment-support.ts | sed -n '1,120p'; nl -ba ui/src/ui/views/chat.ts | sed -n '768,776p;1204,1254p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Paste, picker, and drag/drop handlers only accept images:** `handlePaste` collects only clipboard items whose MIME starts with `image/`, and both `handleFileSelect` and `handleDrop` skip anything that fails `isSupportedChatAttachmentMimeType`, which is also image-only.
  - file: [ui/src/ui/views/chat.ts:206](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/ui/views/chat.ts#L206)
  - command: `nl -ba ui/src/ui/views/chat.ts | sed -n '180,340p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **WebChat send path serializes attachments only as images:** Before `chat.send`, the UI controller maps every attachment to `{ type: "image", mimeType, content }`; there is no document/file path from WebChat to the gateway.
  - file: [ui/src/ui/controllers/chat.ts:213](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/ui/controllers/chat.ts#L213)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '200,280p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Gateway attachment handling still rejects non-image payloads for chat attachments:** Gateway tests assert non-image attachments are rejected or dropped: `buildMessageWithAttachments` throws on `application/pdf`, and `parseMessageWithAttachments` logs/drops non-image payloads instead of exposing them to the agent.
  - file: [src/gateway/chat-attachments.test.ts:39](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/gateway/chat-attachments.test.ts#L39)
  - command: `nl -ba src/gateway/chat-attachments.test.ts | sed -n '1,170p'; nl -ba src/gateway/chat-attachments.ts | sed -n '260,420p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)

## Risks / Open Questions

- The issue title says 'file upload support', and current `main` does have image-only attachment support, so a superficial read could incorrectly treat this as already implemented.
- If maintainers want to narrow the request to 'image upload support in WebChat', that narrower feature appears present; the open gap is non-image files such as PDFs/documents.
