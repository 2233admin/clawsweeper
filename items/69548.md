## Summary

Keep #69548 open. Current main has generic cron `staggerMs` support, but the memory-core dreaming surface still does not expose `dreaming.staggerMs`, normalize it from plugin config, or pass it into the managed dreaming cron job. The PR remains a focused implementation for the same user problem as #69531, and the provided PR discussion shows the author addressed the main zero/blank-string review defects while leaving one removal-semantics question for maintainer judgment.

## Best Possible Solution

Keep the PR open or land an equivalent maintained patch. The best path is to add `dreaming.staggerMs` to `extensions/memory-core/openclaw.plugin.json`, normalize and carry it in `src/memory-host-sdk/dreaming.ts`, thread it through `ShortTermPromotionDreamingConfig` and `buildManagedDreamingCronJob` in `extensions/memory-core/src/dreaming.ts`, add focused tests for positive values, `0`, blank/unset config, and managed-job drift behavior, and update the dreaming/memory config docs with the final removal/default semantics.
