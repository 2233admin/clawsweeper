## Summary

Keep #69548 open. Current main has the generic cron `staggerMs` infrastructure, but memory-core dreaming still does not expose, normalize, document, or pass a `dreaming.staggerMs` value into the managed dreaming cron job. The PR is a focused implementation of a real missing wiring layer, though the review discussion shows it needs maintainer judgment on the unset/clear semantics before merge.

## Best Possible Solution

Keep the PR open and ask for maintainer review or revision rather than closing it. The desired product path is to expose `plugins.entries.memory-core.config.dreaming.staggerMs` through memory-core's manifest schema, host resolver, short-term dreaming config, and managed cron job builder, with explicit semantics for unset versus `0` and tests/docs covering those semantics.
