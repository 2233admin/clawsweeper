## Summary

Keep #69548 open. Current main still does not expose `dreaming.staggerMs` in the memory-core plugin config, host SDK resolver, managed dreaming cron schedule, tests, or memory config docs. The PR is a focused implementation for linked #69531, and its review discussion contains useful follow-up fixes plus one acknowledged behavior tradeoff that needs maintainer judgment rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish the pass-through feature on the PR or an equivalent branch: add `dreaming.staggerMs` to `extensions/memory-core/openclaw.plugin.json`, parse it in `src/memory-host-sdk/dreaming.ts`, propagate it through `resolveShortTermPromotionDreamingConfig` and `buildManagedDreamingCronJob`, update schedule drift behavior deliberately, add focused memory-core/host SDK tests for set/unset/0/blank values, and update the memory config docs.
