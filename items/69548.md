## Summary

Keep open. Current main has cron-level `staggerMs` support, but memory-core dreaming still does not expose or pass `dreaming.staggerMs`; PR #69548 and linked issue #69531 target that exact gap and are currently open, focused items rather than obsolete cleanup. ([github.com](https://github.com/openclaw/openclaw/pull/69548))

## Best Possible Solution

Keep the PR open and have maintainers either finish/review this branch or use it as the implementation basis for #69531. The likely correct path is to add `dreaming.staggerMs` to the memory-core manifest/schema and docs, parse non-negative numeric/string values in `src/memory-host-sdk/dreaming.ts`, propagate it through `ShortTermPromotionDreamingConfig` into the managed cron schedule, and add focused tests for positive values, explicit `0`, blank/unset values, reconciliation drift, and removal/clear semantics.
