## Summary

Keep this issue open. Current main still does not make OPENCLAW_WORKSPACE_DIR the source of truth for the default agent workspace fallback, while the system prompt displays the workspace value produced by that fallback path. The related fix PR #66831 was closed unmerged, and current tests still cover OPENCLAW_HOME/profile fallback rather than the reported env override.

## Best Possible Solution

Implement the closed PR #66831 behavior on current main: make resolveDefaultAgentWorkspaceDir prefer a non-empty OPENCLAW_WORKSPACE_DIR before OPENCLAW_PROFILE/OPENCLAW_HOME fallback logic, keep explicit per-agent workspace and agents.defaults.workspace precedence unchanged, and add focused regression tests for resolveDefaultAgentWorkspaceDir, resolveAgentWorkspaceDir, and resolveRunWorkspaceDir so prompt and runtime workspace resolution stay aligned.
