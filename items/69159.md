## Summary

Keep open. Current main partially supersedes the error-handling half of the PR with `SessionWriteLockTimeoutError` and failover suppression, but it still does not pass the agent run `timeoutMs` into the embedded session write-lock acquisition. The lock wait therefore still falls back to the 10s default in the path this PR targets.

## Best Possible Solution

Keep the PR open and update it against current main: pass `timeoutMs: params.timeoutMs` to `acquireSessionWriteLock` in `src/agents/pi-embedded-runner/run/attempt.ts`, reuse the existing `SessionWriteLockTimeoutError`/code helper instead of adding a second error class, and decide whether the current error/helper should be exported through the plugin SDK harness runtime for typed plugin consumers.
