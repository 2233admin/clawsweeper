## Summary

Keep PR #70471 open. Current main at 998e37fcb3d5b937fcf70d7509b677efba939f10 still lacks the core Feishu hardening this PR proposes: recipient gating, duplicate final-reply suppression after tool delivery, angle-bracket escaping, account-scoped typing reaction state, and force-fresh/credential-aware bot identity probing. The related #66427 is closed/unmerged in the provided context and appears superseded by #70471, not a canonical replacement.

## Best Possible Solution

Keep #70471 open for maintainer review or land an equivalent Feishu-extension fix. The best path is to implement the hardening in the Feishu plugin with focused regression tests for wrong-recipient comment notices, tool-delivery duplicate suppression, Drive comment text escaping, account-scoped typing cleanup, and credential-aware force-fresh bot identity probing with cached fallback, then resolve the remaining P2 review comments before merge.
