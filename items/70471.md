## Summary

Keep PR #70471 open. Current main at 43a003b8a0629ad2c73947b2a863d99dd506bcb9 does not implement the PR's central Feishu hardening: wrong-recipient comment notices are not gated, automatic final comment replies are not suppressed after tool-visible delivery, Drive comment text still passes raw angle brackets, typing-reaction state is not account-scoped, and bot identity probing lacks the proposed force-fresh credential-aware fallback behavior. The related #66427 is closed unmerged in the provided context, so it is not a landed canonical replacement.

## Best Possible Solution

Keep #70471 open or replace it with a canonical Feishu comment-hardening PR that ports the still-missing fixes with focused regression tests: recipient gating on `to_user_id.open_id`, successful-tool-delivery suppression of automatic finals, Drive comment text escaping, account-scoped typing-reaction state, and credential-aware force-fresh bot identity probing with cached-success fallback. Do not close it as superseded by #66427 unless a maintainer opens or lands a real canonical replacement, because #66427 was closed unmerged and current main lacks the behavior.
