## Summary

Keep open. Current `main` already contains the Slack `ignoreSelf: false` fix and still accepts inline TTS directives, so those PR hunks are obsolete, but the config-validation change is not on `main`: validation still builds known plugin IDs only from discovered manifest registry entries, even though bundled alias/auto-enable metadata exists separately. That means the PR still contains useful work, but it likely needs to be split/rebased rather than auto-closed.

## Evidence
