## Summary

Keep #40716 open. Current main has partially addressed the LLM replay side by filtering transcript-only OpenClaw assistant messages from model replay and by skipping some redundant delivery-mirror writes, but it does not implement the PR's full consumer-path scope: `chat.history` and `sessions.get` still read and return raw transcript messages without a delivery-mirror predicate.

## Best Possible Solution

Keep the PR open, but rebase or replace it with a current-main patch that uses the existing transcript-only OpenClaw assistant classification. Rely on the shipped replay-history filter for LLM context, then add focused filtering and regression tests for the remaining visible/API surfaces (`chat.history`, session-history HTTP/SSE if applicable, and `sessions.get` if maintainers confirm it should be a visible consumer API rather than raw transcript access).
