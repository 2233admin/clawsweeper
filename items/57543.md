## Close Comment

This looks better scoped as a ClawHub plugin or skill than an OpenClaw core issue.

- Codex review against current `main` found the project direction already calls for optional capability to ship as plugins, keeps memory behind a plugin slot, and says new skills should go to ClawHub first (`VISION.md:54`, `VISION.md:63`, `VISION.md:68`, `VISION.md:101`).
- Current `main` already has plugin-owned building blocks for this space: shared daily notes in `memory/YYYY-MM-DD.md` (`docs/concepts/memory.md:17`), automatic pre-compaction memory flush into that daily note (`docs/gateway/config-agents.md:548`), and managed dreaming cron promotion into `MEMORY.md` plus review output in `DREAMS.md` (`docs/concepts/dreaming.md:117`).
- There is also already a documented cross-session memory plugin path via Honcho, which persists conversations across resets, compaction, and channel switches (`docs/concepts/memory-honcho.md:16`).
- Because this proposal is additive and optional, it fits the existing ClawHub/community plugin path better than OpenClaw core cleanup. If you want to pursue it, the best next step is to prototype it as a ClawHub plugin/skill and link that implementation back in a fresh issue or PR.
