## Summary

Keep #64217 open. Current main has partial prompt-cache cleanup and observability, but the specific source-dependent drift remains: direct auto-reply turns still add inbound context to the system prompt, sessions_send still adds requester/target A2A context as extraSystemPrompt, OpenAI Codex Responses still sends the full system prompt as instructions, and owner-only tool filtering can still remove cron for non-admin/inter-session gateway runs.

## Best Possible Solution

Keep the issue open and fix it in core prompt/tool assembly: move direct inbound metadata and A2A requester/target context out of provider-facing `instructions`/system prompt for OpenAI Responses/Codex, or otherwise make them stable for a given target session; keep the provider tool schema stable for the same session and enforce cron restrictions at execution/authorization time; retain the existing prompt-cache observability as a regression guard.
