## Summary

Keep #72045 open. Current main at 998e37fc still lacks most of the PR’s qa-lab determinism fixes: blank suite/gateway model refs are preserved as explicit empty strings, the QA gateway enables memory-core but does not pin `plugins.slots.memory`, parity workflows still pass a possibly blank candidate label, and session creation has no transient session-store lock retry. The mock OpenAI all-input-text routing is already present on main, but that only makes one small hunk redundant; it does not supersede the PR.

## Best Possible Solution

Keep #72045 open for maintainer review and likely merge after checking current CI. The best path is to preserve the blank model-ref normalization in suite and gateway code, pin `plugins.slots.memory` for qa-lab gateway runs, align parity workflow candidate labels with the resolved mock default, keep the targeted session-lock retry with its regression test, and optionally simplify the redundant mock-provider prompt OR branch before landing.
