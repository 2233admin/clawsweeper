## Summary

Keep #72045 open. Current main still has the qa-lab behavior the PR targets: blank model refs are passed through as empty strings, the QA gateway does not pin the memory slot, the mock threaded-memory dispatcher keys off the final prompt instead of the full request text, workflow parity labels can stay blank, and the session helper has no transient session-lock retry. The PR is recent, focused, and its discussion reports passing targeted tests/checks, so no conservative close reason applies.

## Best Possible Solution

Keep #72045 open for maintainer review and likely merge after current CI/changed-gate validation. The best path is to preserve the PR's targeted qa-lab fixes, consider simplifying the redundant mock-provider prompt check to `allInputText`, and avoid closing it as obsolete because current main still lacks the implemented behavior.
