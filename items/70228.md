## Summary

Keep open. The current main implementation still matches the report: macOS gateway status audits the LaunchAgent PATH against the full Darwin generated user-bin superset and emits `Gateway service PATH missing required dirs` for every absent expected entry. The issue's proposed narrowing to existing/configured dirs or wording downgrade is not implemented on current `main`.

## Best Possible Solution

Keep #70228 open and fix the Darwin gateway service PATH audit. The best path is a targeted macOS audit change that treats generated user-bin guesses as preferred/recommended rather than required: warn only for configured or existing user dirs, keep true system/runtime PATH drift signal, and add regression tests for a healthy LaunchAgent PATH like `/opt/homebrew/opt/node/bin:/Users/.../.local/bin:/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin` where nonexistent guessed dirs do not produce `gateway-path-missing-dirs`. If the broad generated PATH remains intentional, at minimum change the user-facing wording away from `required dirs`.

## Risks / Open Questions

- The audit and generated service PATH share helpers across macOS and Linux, so the fix should be scoped carefully to Darwin audit semantics without regressing Linux systemd PATH repair behavior.
- Filtering by filesystem existence may require making the PATH audit asynchronous or injecting a filesystem probe, because the current missing-dir check is synchronous inside `auditGatewayServiceConfig`.
