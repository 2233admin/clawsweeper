## Summary

Keep open. Current main has meaningful partial inline media support, including MEDIA directives, managed/generated images, and assistant attachment rendering for image/audio/video, but the full request is not implemented: web/link previews are absent, remote markdown images are still intentionally flattened, direct tool-result image shapes remain incomplete, and persisted raw audio blocks are redacted rather than normalized into durable playable attachments.

## Best Possible Solution

Keep this open as the umbrella unless maintainers split it. Preserve the shipped MEDIA directive, managed-image, and attachment player support, then implement or track server-side link previews with a CSP/privacy/fetch policy, settle the remote markdown image policy alongside #41025/#40945, finish direct tool-result media rendering for #50779/#52025-style payloads, and normalize generated audio/video into durable renderable attachment or managed-media references.
