## Summary

Keep #72314 open. Current main still returns no inbound dedupe key when `MessageSid` is missing, so idless inbound retries remain outside the dedupe path. The PR is a focused, recent bug-fix candidate with tests for the missing case, and its follow-up commit addresses the bot review concern about preserving existing provider-id cache keys.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrow main-branch fix that preserves existing provider-message-id dedupe keys, adds a content/timestamp fallback only for idless inbound messages, and keeps regression coverage for idless retries, same-body/different-timestamp non-collapse, and provider-id key compatibility.
