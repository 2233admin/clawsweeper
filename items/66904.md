## Summary

Keep open. Current main still does not implement WhatsApp reconnect history sync: the WhatsApp runtime does not subscribe to Baileys `messaging-history.set`, the socket does not configure history-sync filtering beyond `syncFullHistory: false`, and stale append/history backlog is explicitly skipped instead of being added to agent-visible history. Related #56088 is a separate closed on-demand `fetchMessageHistory` report, not an implementation of reconnect sync.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin runtime: add opt-in typed reconnect-history config, attach a `messaging-history.set` handler with `syncType` filtering, normalize and dedupe synced messages through existing inbound/history surfaces, populate DM/group history or session context without auto-replying to stale backlog by default, and update config types, strict zod schema, generated metadata/docs, and regression tests for sync filtering, stale backlog handling, dedupe, and `historyLimit`/`dmHistoryLimit` behavior.
