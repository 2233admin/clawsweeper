## Summary

Keep PR #58421 open. Current main still has gateway/TUI session reset support, but it does not expose the proposed plugin-facing `api.resetSession()` contract or `PluginResetSessionResult` SDK type, and the PR discussion shows useful maintainer-followed work with an unresolved broadcast concern rather than an obsolete change.

## Best Possible Solution

Keep the PR open. The best path is a focused plugin SDK/core implementation that adds `api.resetSession()` to `OpenClawPluginApi`, wires it through the API builder and full-mode registry behind the existing gateway `sessions.reset` capability, exports the result type from the SDK, updates generated SDK baselines/docs, and resolves the open session-change broadcast behavior with tests.
