## Summary

Keep open. I did not find strong evidence that current `main` fixes this Moonshot/Kimi duplicate tool-call-id report. Current Moonshot replay policy explicitly opts out of OpenAI-compatible tool-call ID sanitization, so replayed history is not deduped through the generic strict-ID path. The only clear dedupe I found on `main` is a live-stream normalization that rewrites duplicate IDs within a single assistant message, which does not prove the WhatsApp group-chat replay failure is resolved.
