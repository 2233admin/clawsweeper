## Summary

Current main still appears to support explicit WhatsApp outbound sends and same-provider cross-context sends, but it still lacks a WhatsApp-owned way to resolve a trusted direct contact's natural-language name, such as "Jack", to an E.164/JID target from another direct WhatsApp chat.

## Best Possible Solution

Keep this issue open. Add a focused WhatsApp regression where an authorized direct sender asks to relay to another approved direct contact by name, then implement a WhatsApp-owned target-resolution path. Maintainers should decide whether names come from explicit config aliases, WhatsApp live contact metadata, prior session metadata, or only configured E.164/JID IDs; add a generic core/plugin API seam only if WhatsApp cannot expose the needed metadata through the existing directory and target-resolver contracts.
