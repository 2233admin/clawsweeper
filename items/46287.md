## Summary

Keep open. Current `main` still implements startup catch-up for recurring cron jobs after restart, with no job-level `skipMissedOnRestart`/`skipMissedRuns` flag, no restart-payload detection, and tests that explicitly assert the replay behavior. That means the reported restart-loop footgun is not implemented away on `main`.

## Evidence

- **Startup path replays recurring jobs after restart:** `start()` clears stale `runningAtMs`, skips only interrupted one-shot (`schedule.kind === "at"`) jobs, then calls `runMissedJobs(...)` for the remaining recurring jobs.
  - file: [src/cron/service/ops.ts:93](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cron/service/ops.ts#L93)
  - command: `nl -ba src/cron/service/ops.ts | sed -n '87,121p'`
- **Catch-up planner runs missed jobs without payload-specific filtering:** `planStartupCatchup()` collects runnable missed jobs with only `skipJobIds`, `skipAtIfAlreadyRan`, and `allowCronMissedRunByLastRun`; it then logs `cron: running missed jobs after restart` and schedules those jobs for execution.
  - file: [src/cron/service/timer.ts:962](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cron/service/timer.ts#L962)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '962,1028p'`
- **Runnable-job logic has no restart-job exemption:** `isRunnableJob()` treats enabled overdue jobs as runnable and has special handling for one-shot jobs and cron missed-slot replay, but no branch for restart payloads or any per-job skip-on-restart flag.
  - file: [src/cron/service/timer.ts:857](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cron/service/timer.ts#L857)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '857,921p'`
- **Cron job schema has no requested opt-out field:** `CronJob`, `CronJobCreate`, and `CronJobPatch` include schedule/payload/delivery/failureAlert/state fields, but there is no `skipMissedOnRestart`, `skipMissedRuns`, or similar startup-catch-up opt-out.
  - file: [src/cron/types.ts:168](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cron/types.ts#L168)
  - command: `nl -ba src/cron/types.ts | sed -n '168,191p'`
- **Tests lock in replay-after-restart behavior:** Current tests explicitly assert that startup replays interrupted recurring jobs and missed cron slots after restart, which is the opposite of the requested behavior for restart-type jobs.
  - file: [src/cron/service/ops.test.ts:131](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cron/service/ops.test.ts#L131)
  - command: `nl -ba src/cron/service/ops.test.ts | sed -n '130,171p'`
- **Additional restart catch-up tests cover missed-slot replay:** `service.restart-catchup.test.ts` contains cases such as `replays interrupted recurring job on first restart` and `replays the most recent missed cron slot after restart`, confirming current intended behavior.
  - file: [src/cron/service.restart-catchup.test.ts:118](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cron/service.restart-catchup.test.ts#L118)
