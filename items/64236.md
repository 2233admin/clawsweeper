## Summary

Keep this issue open. Current main still has per-agent `reasoningDefault` plus a catalog-derived fallback, but it does not define, validate, document, or consume `agents.defaults.models[modelRef].reasoningDefault` or a global `agents.defaults.reasoningDefault` config field. The related #71817 remains a partial channel fix for per-agent Telegram/Feishu behavior, not the requested per-model configuration surface.

## Best Possible Solution

Add a first-class `reasoningDefault?: "on" | "off" | "stream"` to `AgentModelEntryConfig` / `agents.defaults.models[modelRef]`, update Zod and generated config schema/help/docs, and teach reply/model-selection resolution to honor precedence such as directive, session override, per-agent default, selected-model config, then catalog fallback. Maintainers should also decide whether `agents.defaults.reasoningDefault` should be implemented globally or removed from the gateway-tool allowlist.
