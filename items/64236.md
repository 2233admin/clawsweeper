## Summary

Current main does not implement the requested per-model `reasoningDefault` configuration. It has per-agent `agents.list[].reasoningDefault`, implicit model-capability reasoning defaults, and per-model provider `params`, but there is no typed/schema-supported `agents.defaults.models[model].reasoningDefault` field and no runtime resolver for a per-model `on|off|stream` override.

## Best Possible Solution

Keep the issue open. A proper fix would add `reasoningDefault?: "on" | "off" | "stream"` to `AgentModelEntryConfig` and the strict `agents.defaults.models` schema, resolve it in the reply/model-selection path after directive/session/per-agent overrides and before implicit catalog capability defaults, then update config docs/help/generated schema and add regression tests for reasoning-capable and non-reasoning model entries.
