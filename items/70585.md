## Summary

Keep open. Current `main` at `cc0992564bc565791810e6a560d12b430a8544ee` still has the queued subagent announce bug: the queued path can enqueue an announce with no resolved external target, then `sendAnnounce` still sends `deliver: true` without a matching `bestEffortDeliver` fallback.

## Evidence

- **Queued send path still forces external delivery:** In current `main`, `sendAnnounce` forwards `origin?.channel`, `origin?.to`, and `origin?.threadId`, but still sets `deliver: !requesterIsSubagent` unconditionally for non-subagent requesters. There is no `hasExplicitDeliveryTarget` guard and no queued-path `bestEffortDeliver` fallback here.
  - file: [src/agents/subagent-announce-delivery.ts:351](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/agents/subagent-announce-delivery.ts#L351)
  - command: `nl -ba src/agents/subagent-announce-delivery.ts | sed -n '340,390p'`
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Queued path can enqueue an item with no origin target:** When an active requester is in follow-up/collect/interrupt modes, the code enqueues `origin = resolveAnnounceOrigin(entry, params.requesterOrigin)`. `resolveAnnounceOrigin` returns `mergeDeliveryContext(...)`, which can be `undefined` when both the session entry and requester origin lack routing data. That matches the PR's reported reproduction shape.
  - file: [src/agents/subagent-announce-delivery.ts:459](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/agents/subagent-announce-delivery.ts#L459)
  - command: `nl -ba src/agents/subagent-announce-delivery.ts | sed -n '455,478p'`
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Origin resolver does not guarantee channel/to:** `resolveAnnounceOrigin` normalizes requester and session delivery context, then returns `mergeDeliveryContext(normalizedRequester, entryForMerge)`. If neither side has channel/to/account/thread data, the helper returns `undefined` rather than synthesizing a deliverable external route.
  - file: [src/agents/subagent-announce-origin.ts:143](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/agents/subagent-announce-origin.ts#L143)
  - command: `nl -ba src/agents/subagent-announce-origin.ts | sed -n '143,166p'`
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Current gateway behavior expects best-effort fallback to avoid channel errors:** Gateway tests on `main` explicitly cover that `deliver: true` only downgrades to session-only when `bestEffortDeliver: true` is also set and no external channel is configured. The queued announce path above does not set that flag today, so the reported failure mode remains plausible on current `main`. This also matches the PR discussion and Greptile review describing the change as a narrow queued-path fix.
