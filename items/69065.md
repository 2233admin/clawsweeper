## Summary

Keep open. Current main still does not provide Android/Termux gateway service support or an Android-specific fallback for `openclaw gateway start/install`: service resolution only registers macOS, Linux, and Windows adapters and throws for unsupported platforms. The Android docs now state that Android is a companion node and does not host the Gateway, but they do not document the reported Termux foreground workaround or make the service-management error actionable for npm-on-Termux users. The related OpenBSD unsupported-service report is similar evidence of the same service boundary, but it is not a canonical Android/Termux tracker.

## Best Possible Solution

Keep the issue open for maintainer product judgment. If Android/Termux gateway hosting is intended, add a supported Termux/Android service strategy such as a `termux-services` adapter or an explicit foreground/daemon path, with service registry coverage, CLI lifecycle behavior, docs, and tests. If Android hosting is intentionally unsupported, add an Android/Termux-specific error and docs that explicitly direct npm-on-Termux users to a supported Gateway host or a documented foreground-only workaround.
