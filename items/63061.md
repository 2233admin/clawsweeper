## Summary

Keep open. Current main still has the profile/OPENCLAW_HOME state-dir behavior this PR targets, and the PR remains a focused implementation candidate rather than obsolete or superseded. The related plugin-install profile-dir reports are adjacent but do not replace this CLI path-precedence decision.

## Best Possible Solution

Keep this PR open for maintainer review. Decide the intended precedence between OPENCLAW_HOME physical isolation and named profile suffixing, then either land a normalized version of this PR with matching docs/help updates and daemon/service-path alignment, or explicitly document that profile suffixing should win under OPENCLAW_HOME and close the PR after that product decision.
