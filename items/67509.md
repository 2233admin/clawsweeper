## Summary

Keep PR #67509 open. Current main does not implement an equivalent root guard in either CLI entrypoint, and the linked #67478 failure mode remains supported by the current state-dir and user-systemd code paths. The PR is still a focused, relevant core bug fix and has addressed review feedback around static imports, container forwarding, legacy entrypoint coverage, and effective UID handling.

## Best Possible Solution

Keep #67509 open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both `src/entry.ts` and `src/index.ts`, checks effective UID, preserves safe container-forwarded behavior, includes regression tests, and reconciles the behavior with DigitalOcean/install documentation before release.
