## Summary

Keep open. Current `main` appears to have fixed the specific `seqByRun` leak, but the broader issue is not implemented well enough to close because the other cited memory-retention paths still exist in source on `main`.

## Evidence

- **`seqByRun` cleanup is present on current `main`:** `clearAgentRunContext()` now deletes both `runContextById` and `seqByRun`, and Gateway terminal lifecycle handling calls it on end/error. There is also a regression test explicitly covering the memory-leak case.

  - file: [src/infra/agent-events.test.ts:236](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/infra/agent-events.test.ts#L236)
