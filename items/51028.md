## Summary

Keep open. Current `main` still uses a single `updatedAt` timestamp to order sessions end-to-end, and the shipped macOS sessions UI does not expose a separate meaningful-activity timestamp or a visible reason label for why a session is on top.

## Evidence

- **Gateway session list still sorts by `updatedAt`:** `listSessionsFromStore` builds session rows and sorts them with `.toSorted((a, b) => (b.updatedAt ?? 0) - (a.updatedAt ?? 0))`. There is no `lastMeaningfulActivityAt` or equivalent secondary sort key in this path.
  - file: [src/gateway/session-utils.ts:1540](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/gateway/session-utils.ts#L1540)

- **Gateway/macOS session payload still exposes only `updatedAt`:** The decoded session entry model includes `updatedAt` plus display/token fields, but no `lastMeaningfulActivityAt`, `lastVisibleConversationAt`, or activity-reason field.
  - file: [apps/macos/Sources/OpenClaw/SessionData.swift:16](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/apps/macos/Sources/OpenClaw/SessionData.swift#L16)
