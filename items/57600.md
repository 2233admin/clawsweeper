## Summary

Current `main` does not contain this PR's binary-document read guard, and the underlying read path still decodes non-image files as UTF-8 text. A broader follow-up PR exists, but it is still open and unmerged, so there is not enough evidence to close this item.

## Evidence

- **OpenClaw read wrapper has no binary-document normalization on main:** `createOpenClawReadTool` on `main` runs adaptive paging, strips truncation details, normalizes images, and returns. There is no MIME-based fallback for `.pdf/.doc/.xls/.ppt/.docx/.xlsx/.pptx`.
  - file: [src/agents/pi-tools.read.ts:674](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/pi-tools.read.ts#L674)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Host tool wiring does not pass any file-buffer hook for document detection:** The host `read` tool setup only passes `modelContextWindowTokens` and `imageSanitization`, so the draft PR's `readBufferForToolPath` path is not present on `main`.
  - file: [src/agents/pi-tools.ts:473](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/pi-tools.ts#L473)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Underlying read implementation still treats all non-image files as UTF-8 text:** The bundled `@mariozechner/pi-coding-agent@0.70.2` read tool checks only for image MIME, then falls through to `const textContent = buffer.toString("utf-8")` for every other file. That still matches the reported garbled Office/PDF behavior.
  - command: `curl -L --max-time 20 https://unpkg.com/@mariozechner/pi-coding-agent@0.70.2/dist/core/tools/read.js | nl -ba | sed -n '95,180p'`
- **Current main still models native document support as PDF-only:** The media-understanding provider type is `nativeDocumentInputs?: Array<"pdf">`, and manifest normalization filters allowed values down to `pdf` only. There is no landed broader document-handling path that would make this PR obsolete.
