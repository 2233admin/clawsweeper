## Summary

Keep open. Current `main` has overlapping generic hook seams (`inbound_claim` and `before_dispatch`) that can already intercept inbound flow before model work, but it does not implement the PR's proposed `channels.telegram.ingressMiddlewares` config or Telegram-specific awaited ingress runtime. The PR also still has unresolved architectural-fit concerns around extension/core boundaries, so there is not strong evidence to auto-close it as implemented or obsolete.
