## Summary

Keep open. Current main at a35d259 still has the reported behavior: loop-emitted chunks are `trimEnd()`ed, but the final `remaining` chunk is pushed unchanged, and the issue's counterexamples still return trailing whitespace. The related fix PRs #64054 and #64079 were closed unmerged, so there is no canonical shipped fix to close against.

## Best Possible Solution

Keep the issue open and land a narrow fix in `src/shared/text-chunking.ts`: compute `const finalChunk = remaining.trimEnd()`, push it only when non-empty, and add regression tests in `src/shared/text-chunking.test.ts` for `"  ! "` and `"a b "`. The closed unmerged PRs #64054 and #64079 can be mined for the exact patch/test shape, but the fix still needs to be applied to current main.
