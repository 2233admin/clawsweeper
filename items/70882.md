## Summary

Keep #70882 open. Current main still exposes bundle MCP tools but passes tool-call input through to the MCP runtime without schema-aware coercion, so the linked #70872 failure mode is not implemented on main. The PR is a focused bug-fix attempt with regression tests in its diff; bot comments identify hardening/completeness follow-ups rather than a reason to close it as obsolete.

## Best Possible Solution

Keep the PR open and either revise it or land an equivalent core fix in the bundle MCP materialization/runtime boundary. The fix should coerce only schema-declared object/array properties before MCP SDK `callTool`, add regression tests for #70872, and address hardening before merge: avoid prototype-sensitive assignment, bound or otherwise constrain JSON parsing, and consider nullable JSON Schema type arrays.
