## Summary

Current `main` has scheduling (`cron`), detached-work tracking (`tasks`), and durable flow orchestration (`Task Flow`), but it still lacks a native commitment/deferred-task primitive that turns a conversational promise into a tracked obligation with due-date verification and escalation. The only related recent change is a narrow reminder guard that warns when the agent says it will remind/follow up without creating a cron job; that does not implement the broader feature requested here.

## Evidence
