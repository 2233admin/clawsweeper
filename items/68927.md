## Summary

Keep open. Current main does not implement PR #68927's requested label behavior: the chat model picker still stores qualified values but intentionally renders friendly catalog names, aliases, and disambiguated provider labels. The PR is a recent draft that explicitly asks for product review, and the related cleanup reports cover qualified-value preservation rather than this stricter label-display policy.

## Best Possible Solution

Keep the PR open for explicit maintainer product review. If the desired product direction is exact provider/model labels in the web chat picker, update `ui/src/ui/chat-model-ref.ts` and `ui/src/ui/chat-model-select-state.ts` while preserving qualified-value normalization and stale-provider coverage. If maintainers prefer friendly names, close manually as a product decision rather than as already implemented or duplicate.
