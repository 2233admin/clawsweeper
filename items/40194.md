## Summary

Keep #40194 open. Current main still passes Feishu accountId through routing, but the non-DM group session-key shape remains account-agnostic, so two Feishu bot accounts in the same group routed to the same agent can still share one session.

## Best Possible Solution

Implement account-aware Feishu group session isolation, preferably as a Feishu-specific or generic opt-in account-scoped group mode. Use normalized accountId rather than raw app credentials in the key, preserve legacy session continuity where possible, document the behavior on the Feishu channel docs page, and add a regression test with two Feishu accounts in the same group routed to the same agent asserting distinct session keys and the correct reply account.
