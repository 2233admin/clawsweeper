## Summary

Keep PR #72361 open. Current main still has the `/allowlist --store` path mutating the pairing allowlist store without applying the existing `configWrites` authorization guard, while the PR is a focused fix with targeted regression coverage and a positive bot review.

## Best Possible Solution

Keep the PR open and land a focused fix: apply `resolveConfigWriteDeniedText` to the store-only `/allowlist` path before `updatePairingStoreAllowlist`, using `resolveExplicitConfigWriteTarget({ channelId, accountId })`, and keep regression tests for channel-level denial, account-level denial, and the allowed `configWrites=true` case.
