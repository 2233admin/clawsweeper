## Summary

Keep PR #40238 open. Current main still routes inline skill `command-dispatch: tool` through `createOpenClawTools()` plus only `applyOwnerOnlyToolPolicy()`, while the normal command/coding path uses `createOpenClawCodingTools()` and its broader config/profile/provider/group/subagent policy pipeline. The PR remains a focused, plausible fix and is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best implementation is the PR's current approach or an equivalent main-branch patch: inline `command-dispatch: tool` should build tools through `createOpenClawCodingTools()` with the same config, session, route, model, sender, group, and spawned-session context as normal command/coding paths; preserve message target/thread routing; remove the standalone owner-only fallback filter; and keep regression tests proving non-owner/profile-disallowed tools are unavailable while policy-critical arguments are passed correctly.
