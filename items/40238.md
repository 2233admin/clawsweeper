## Summary

Keep this PR open. Current main still has the policy divergence the PR is trying to fix: inline skill `command-dispatch: tool` builds raw OpenClaw tools and applies only owner-only filtering, while normal command sessions use `createOpenClawCodingTools` and its broader policy pipeline. The PR is a focused security/policy fix, but it needs rebase/adaptation because current main now preserves the inline path behind a lazy runtime boundary.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase or adapt it on current main so inline skill `command-dispatch: tool` resolves through `createOpenClawCodingTools`, removes the standalone owner-only fallback, preserves the lazy runtime boundary, forwards the same policy-critical route/session/group/sender/model/config context as normal command runs plus inline message target/thread context, and keeps focused regression tests for profile-disallowed tools and argument forwarding.
