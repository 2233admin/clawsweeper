## Summary

Keep #40846 open. Current main still does not prevent macOS idle sleep while the gateway is running, and the PR contains a focused core gateway lifecycle change that is not superseded by the LaunchAgent KeepAlive work or the related closed unmerged PR #28317. The PR should not be closed as obsolete, but it needs the acknowledged async spawn error handling and focused tests before it is merge-ready.

## Best Possible Solution

Keep the PR open. The best path is a refreshed macOS-only gateway lifecycle implementation that starts `caffeinate -i -w <gateway pid>` when the gateway loop starts, cleans it up in the existing shutdown/finally path, attaches a child `error` listener, and adds focused tests by mocking `node:child_process` and platform detection. The related closed PR #28317 was not merged, and the current LaunchAgent KeepAlive behavior is complementary rather than a substitute.

## Risks / Open Questions

- Closing this PR would drop a still-missing core macOS gateway behavior: preventing idle sleep for always-on local workflows such as Telegram long polling.
- Merging the current head as-is would carry the acknowledged async spawn error-handling gap; a missing or inaccessible caffeinate binary can emit an unhandled child process error unless the child has an `error` listener.
- The power-management behavior is user-visible on macOS, so the final implementation should stay tightly scoped to idle sleep, avoid display/user-initiated sleep changes, and include macOS-specific lifecycle tests with mocked dependencies.
