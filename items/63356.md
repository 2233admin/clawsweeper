## Summary

Keep this PR open. Current main still normalizes Signal shared allowlist entries and DM-policy entries as E.164 phone numbers, so documented `uuid:` Signal sender IDs are lost on the shared config/security path. The PR remains a focused Signal fix, and its diff only touches Signal source and tests, with no security-sensitive CI, dependency, install, publishing, or secrets changes.

## Best Possible Solution

Rebase and review this PR, then land this targeted Signal fix or an equivalent shared Signal allow-entry normalizer used by `signalConfigAdapter.formatAllowFrom` and `signalSecurityAdapter.normalizeDmEntry`. Ideally share or deliberately align it with approval-auth normalization and add regression coverage for `uuid:`, `signal:uuid:`, wildcard preservation, E.164 phone normalization, and the compact UUID edge case.
