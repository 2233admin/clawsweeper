## Summary

Keep #70479 open. Current main still contains the compaction-failure session-rotation path this PR is trying to remove, and the latest release still has the old reset behavior. The PR remains a focused fix candidate for linked issue #70472, with useful reviewer discussion calling out validation checks rather than a reason to close it.

## Best Possible Solution

Keep the PR open for maintainer review or an equivalent patch. The best path is to preserve the active session mapping on embedded overflow payloads and thrown compaction-failure errors, remove the dead compaction reset hook, add regression tests for both paths, update the compaction docs, and add a release-note mention for the user-facing string change if this lands.
