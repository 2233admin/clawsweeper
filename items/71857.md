## Summary

Keep PR #71857 open. Current main still rejects owner-prefixed ClawHub skill slugs before registry lookup, while the PR is a focused fix for the concrete #71767 copy-paste install failure.

## Best Possible Solution

Keep PR #71857 open for maintainer review. The best path is to land a narrow parser change in `src/agents/skills-clawhub.ts` that accepts optional `<owner>/<slug>` only for requested ClawHub skill install/update, validates both segments under the existing ASCII slug guard, strips the owner before registry/install-dir/lockfile use, preserves legacy tracked-slug behavior, and keeps the focused regression tests plus changelog entry.
