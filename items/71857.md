## Summary

Keep PR #71857 open. Current main still rejects owner-prefixed ClawHub skill slugs before any registry call, while the PR is a focused, recent bug fix with tests for the reported copy-paste failure from #71767.

## Best Possible Solution

Keep #71857 open for maintainer review or merge/adapt its approach: add a small ClawHub skill slug parser in `src/agents/skills-clawhub.ts` that accepts `<owner>/<slug>` for requested installs/updates, canonicalizes to the bare slug for registry/install/lockfile use, and preserves the existing ASCII, traversal, and malformed-segment rejection tests.
