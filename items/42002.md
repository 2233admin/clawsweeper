## Summary

Current `main` does not implement this iMessage cleanup. The outbound iMessage path still sanitizes only assistant/scaffolding markers, then sends the resulting text directly, so `[[reply_to:...]]` / `[[reply_to_current]]` tags can still leak into visible outbound text. The PR remains useful and should stay open.

## Evidence

- **iMessage outbound sanitize path still omits reply-tag stripping:** `sanitizeOutboundText` removes assistant scaffolding markers and trims blank lines, but it does not call the shared inline-directive stripping helper or otherwise remove `[[reply_to:...]]` / `[[reply_to_current]]` tags.
  - file: [extensions/imessage/src/monitor/sanitize-outbound.ts:16](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/extensions/imessage/src/monitor/sanitize-outbound.ts#L16)
