## Summary

Keep open. Current `main` still marks cleanup-pending subagent runs as finished in the list/status helpers, so the PR's reported bug is not already implemented on `main` and there is not enough evidence to close it as obsolete.

## Evidence

- **Active-run predicate still ignores cleanup-pending window:** `isActiveSubagentRun` only treats a run as active when `endedAt` is missing or descendants are still pending. It does not consider `cleanupCompletedAt` or `suppressAnnounceReason`, so a run that has ended but whose cleanup is still pending is treated as inactive.
