## Summary

Keep open. Current main still ships `skills/canvas/SKILL.md` without YAML frontmatter, and the active skill loader requires a frontmatter-derived description before a skill is loaded. The provided GitHub context also shows open, unmerged PR #54361 as the implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open. Review and land PR #54361, or apply the equivalent fix directly on main: add valid `name` and trigger-rich `description` YAML frontmatter to `skills/canvas/SKILL.md`, and preferably expand bundled-skill frontmatter coverage to include canvas or all bundled skills. After that reaches main, close this issue as implemented.
