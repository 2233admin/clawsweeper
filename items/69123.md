## Summary

Keep open. Current main still requires `agents.defaults.cliBackends.<id>.command` in the source Zod schema, TypeScript config type, and generated JSON schema, so the partial `modelAliases` override bug is not implemented on main. The PR targets a real config/runtime mismatch because backend merging already inherits base fields and resolution already rejects unresolved commandless backends, but the patch still needs generated-schema sync and resolved-command typing through runner paths before it is safely mergeable.

## Best Possible Solution

Keep the PR open and finish the focused fix: make `CliBackendSchema.command` and the user-facing `CliBackendConfig` input optional, introduce or propagate a resolved backend type with non-empty `command`, update `src/config/schema.base.generated.ts` and related schema artifacts, and add regression coverage for a `modelAliases`-only override inheriting the plugin-owned base command.
