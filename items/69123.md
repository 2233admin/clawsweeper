## Summary

Keep open. Current main still requires `agents.defaults.cliBackends.<id>.command` in the Zod schema, TypeScript config type, and generated base config schema, so a `modelAliases`-only override remains unsupported. The PR addresses a real config gap, but its review discussion correctly identifies that generated schema artifacts also need updating before the fix is complete.

## Best Possible Solution

Keep PR #69123 open and ask for a focused revision: make `CliBackendSchema.command` and `CliBackendConfig.command` optional for input partials, preserve a resolved backend type or guard with `command: string` after `resolveCliBackendConfig`, add the modelAliases-only regression test, and regenerate/check the base config schema so schema-driven consumers accept the same partial override contract.
