## Summary

Keep #40694 open. Current main has useful partial browser tab cleanup for subagent/cron completion, session reset/rollover, and idle/excess primary-session sweeps, but it still does not implement the requested default of closing temporary primary-agent browser tabs/windows when an ordinary browser task finishes.

## Best Possible Solution

Keep #40694 open for a focused browser-plugin/core lifecycle design: distinguish ephemeral automation-opened tabs from reusable or user-owned tabs, add a primary-agent task/turn completion cleanup path, and keep the current idle/excess sweep plus session/subagent/cron lifecycle cleanup as fallback behavior.

## Risks / Open Questions

- Changing this default can break workflows that intentionally reuse active primary-session tabs across turns; the implementation likely needs explicit product policy for reusable, user-owned, labeled, pinned, or preserved tabs.
- The existing partial cleanup can make the issue look fixed from a quick scan, but the documented primary-session behavior remains materially different from the requested temporary-tab cleanup default.
