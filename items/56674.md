## Summary

Keep open. Current `main` does not implement OpenResponses reasoning output/events for `/v1/responses`: non-stream responses only serialize assistant payload text, streaming only listens for `assistant` and `lifecycle` events, and the streaming schema/tests do not include `response.reasoning.delta` or `response.reasoning.done`.

## Evidence

- **Non-stream `/v1/responses` still discards reasoning:** `src/gateway/openresponses-http.ts:703-777` builds non-stream output only from `result.payloads[].text`, returning either assistant text or function_call items. It never subscribes to reasoning events and never creates an output item with `type: "reasoning"`.
