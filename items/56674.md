## Summary

Keep open. Current `main` still does not return reasoning/thinking content from `/v1/responses` in either non-streaming or streaming mode, so this PR's requested behavior is not already implemented on `main` and is still materially relevant.

## Evidence

- **Non-streaming `/v1/responses` still returns only assistant text and optional function calls:** The non-streaming handler builds `output` from assistant text plus optional `function_call` items only. There is no reasoning collector, no `type: "reasoning"` output item, and the fallback remains `"No response from OpenClaw."` when no assistant text exists.
  - file: [src/gateway/openresponses-http.ts:681](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/openresponses-http.ts#L681)
  - command: `nl -ba src/gateway/openresponses-http.ts | sed -n '680,860p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Streaming `/v1/responses` listens only for assistant and lifecycle events:** The SSE path subscribes to `assistant` and `lifecycle` agent events and emits only `response.output_text.*` completion flow from those inputs. There is no handling for reasoning/thinking streams, so `response.reasoning.delta` / `response.reasoning.done` cannot be produced on current `main`.
  - file: [src/gateway/openresponses-http.ts:923](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/openresponses-http.ts#L923)
