## Summary

Keep open. Current main still matches the reported bug: bundled LSP stdio servers are spawned without a disposable process group, disposal only kills the immediate child process, and Gateway shutdown only sweeps bundled MCP runtimes, not active bundled LSP runtimes. The provided related PR #72367 is the right canonical fix path but is still open in the supplied context and the fix is not present at current main SHA 998e37fcb3d5b937fcf70d7509b677efba939f10.

## Best Possible Solution

Keep #72357 open and land a focused lifecycle fix, likely via #72367 or equivalent: spawn bundled POSIX LSP stdio servers into disposable process groups, perform graceful LSP shutdown/exit with a bounded wait, call killProcessTree for still-running servers so descendants such as tsserver exit, add a Gateway shutdown sweep for active bundle-LSP runtimes, and cover it with bundled LSP runtime plus server-close tests.
