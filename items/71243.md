## Summary

Keep open. Current `main` does not already contain this onboarding UX, and the PR still has concrete regressions: it hard-codes new onboarding chrome in English, collapses distinct auth failure modes into one token-specific hint, and adds a user-facing doc example with an author-specific local path. The latest head also appears to have resolved some earlier bot comments (`python3`, full target cleanup, translated locale keys), so this looks like useful work that still needs follow-up rather than a cleanup close.

## Risks / Open Questions

- Merging as-is would ship partially untranslated onboarding chrome to non-English Control UI users.
- Operators hitting password, Tailscale, or other non-token auth failures could be told the problem is a missing/rejected token, which sends them to the wrong remediation path and removes existing guidance.
- The new runbook would publish a contributor-specific local path into user-facing docs.
