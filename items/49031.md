## Summary

Keep open. Current main still has the reported compatibility gap on the macOS local node path: node exec unconditionally requires `system.run.prepare`, but the macOS companion app's local node still neither advertises nor implements `system.run.prepare`. Main does contain a partial follow-up for the separate CLI command drift (`openclaw node start --json`), but that landed after `v2026.4.22` and does not resolve the local-node exec failure on its own.

## Evidence
