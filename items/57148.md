## Summary

Keep open. Current `main` still injects full subagent completion text into parent-session internal event blocks with no dedicated truncation/config limit, and the existing context guard only truncates tool-result messages.

## Evidence

- **Subagent findings stay unbounded:** `findings` is assigned directly from `childCompletionFindings || reply || "(no output)"` with no truncation step before it is attached to the internal event payload.
  - file: [src/agents/subagent-announce.ts:461](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/subagent-announce.ts#L461)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Internal event formatter emits full result text:** `formatTaskCompletionEvent` sanitizes multiline text but then writes the full `event.result` between `<<<BEGIN_UNTRUSTED_CHILD_RESULT>>>` markers; there is no max-length enforcement in this formatter.
  - file: [src/agents/internal-events.ts:43](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/internal-events.ts#L43)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Announcement path injects the formatted block into the parent session:** The announce flow builds `triggerMessage` from `formatAgentInternalEventsForPrompt(...)`, and agent command execution prepends that rendered internal context block to the outgoing message body.
  - file: [src/agents/subagent-announce.ts:517](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/subagent-announce.ts#L517)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Prepended internal events are sent as normal prompt text:** `prependInternalEventContext` joins rendered internal events directly ahead of the message body when internal events are present.
  - file: [src/agents/command/attempt-execution.shared.ts:29](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/command/attempt-execution.shared.ts#L29)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Existing truncation guard is tool-result-specific only:** `installToolResultContextGuard` computes `maxSingleToolResultChars` and applies truncation only through tool-result guard helpers; there is no parallel `maxSubagentResultChars` path on `main`. Repo search found `maxSingleToolResultChars` only in tool-result guard code and no `maxSubagentResultChars` symbol.
  - file: [src/agents/pi-embedded-runner/tool-result-context-guard.ts:304](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/pi-embedded-runner/tool-result-context-guard.ts#L304)
  - command: `rg -n "session\.maxSubagentResultChars|maxSubagent|subagentResultChars|maxSingleToolResultChars" src docs packages ui --glob '!**/dist/**'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
