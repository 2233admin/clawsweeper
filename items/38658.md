## Summary

Keep PR #38658 open. Current main still implements Slack room history as pending-only: skipped mention-gated messages are recorded for later context, but processed `requireMention=false` channel messages are not retained after dispatch, and the current regression test still asserts the second always-on channel message does not see the first as history. The PR is a plausible focused behavior change, but it needs maintainer product judgment rather than conservative cleanup closure.

## Best Possible Solution

Keep PR #38658 open for maintainer review. If always-on Slack channel context is the desired behavior, update `extensions/slack/src/monitor/message-handler/*` so prepared messages carry the resolved mention policy into dispatch, `requireMention=false` processed room messages are retained in both reply and no-reply paths, retained history ordering is deterministic, regression tests assert the new prompt context, and Slack/group history docs describe the non-pending history behavior. If maintainers prefer the current pending-only model, close the PR with that explicit product rationale.
