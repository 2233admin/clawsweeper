## Summary

Current main already registers `message_sent` as a typed plugin hook, routes normal external-channel replies through the outbound delivery path, and emits `runMessageSent(...)` there with regression coverage. The same outbound hook files are unchanged between current main and release `v2026.4.22`, so this behavior is already shipped.

## Close Comment

Closing from a Codex review because current `main` already implements this path for external-channel replies. `src/auto-reply/reply/route-reply.ts:221` routes normal replies through `deliverOutboundPayloads(...)`, `src/infra/outbound/deliver.ts:516` checks `hasHooks("message_sent")` and calls `runMessageSent(...)`, and extension `api.on(...)` registrations are stored in `registry.typedHooks` at `src/plugins/registry.ts:1271`. The behavior is covered by regression tests in `src/infra/outbound/deliver.test.ts:1078`, `src/infra/outbound/deliver.test.ts:1138`, and `src/infra/outbound/deliver.test.ts:1469`. I also checked the shipped release snapshot: `v2026.4.22` has no diff versus current `main` for `src/infra/outbound/deliver.ts`, `src/auto-reply/reply/route-reply.ts`, or `src/infra/outbound/deliver.test.ts`, so this implementation is already in a released build. Remaining outbound-hook gaps appear to be tracked separately in #70928 for webchat/`openclaw-tui`, which is a different path from this Discord report.
