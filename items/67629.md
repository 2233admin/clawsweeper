## Summary

Keep open. Current `main` has the underlying cron runtime state and the CLI already exposes both job IDs and a `running` status, but the web cron jobs view still shows only `job.name` and derives status solely from `lastStatus`, so the requested per-cron identifier and running/not-running indicator are not implemented on the main cron management surface.

## Evidence

- **Web cron list shows name, not identifier:** `renderJob` renders the title from `job.name` and does not display `job.id` anywhere in the job card.
  - file: [ui/src/ui/views/cron.ts:1483](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/ui/src/ui/views/cron.ts#L1483)
  - command: `nl -ba ui/src/ui/views/cron.ts | sed -n '1470,1665p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Web cron status ignores running state:** `renderJobState` reads `job.state?.lastStatus` for the status pill and never checks `job.state?.runningAtMs`, so an actively running cron is not labeled as running in the web jobs list.
  - file: [ui/src/ui/views/cron.ts:1622](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/ui/src/ui/views/cron.ts#L1622)

- **Running state exists in data model:** The UI type for cron jobs includes `runningAtMs`, so the runtime already provides a field that can support a running/not-running indicator.
  - file: [ui/src/ui/types.ts:566](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/ui/src/ui/types.ts#L566)
