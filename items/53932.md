## Summary

Current main still has no Slack upload setting or call-path option to preserve original image bytes. Slack outbound uploads still route through the optimized outbound media helper, the public helper used by Slack does not expose `optimizeImages`, and the related implementation attempt #53988 is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open and implement the opt-out in the bundled Slack plugin. A conservative fix would add a documented Slack config key that preserves optimized uploads by default, then pass `optimizeImages: false` through an additive outbound-media SDK option or route Slack through an existing raw media loader when disabled. Update Slack schema/types/generated config metadata/docs and add regression coverage proving PNG uploads can preserve original bytes and MIME while existing size and security guards still apply.
