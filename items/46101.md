## Summary

Keep open. Current `main` still lacks the requested `openclaw doctor` warning for mixed `agents.defaults.heartbeat` plus per-agent heartbeat blocks. The underlying shadowing behavior is still intentional and present, and recent `health`/`status`/docs work only makes it more visible after the fact rather than adding the proactive doctor finding this issue asks for.

## Evidence

- **Shadowing behavior still exists:** `isHeartbeatEnabledForAgent` disables default-heartbeat agents whenever any `agents.list[]` entry has an explicit `heartbeat` block, and `resolveHeartbeatAgents` then schedules only those explicit entries.
  - file: [src/infra/heartbeat-summary.ts:32](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/infra/heartbeat-summary.ts#L32)
  - command: `nl -ba src/infra/heartbeat-summary.ts | sed -n '1,119p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Runtime scheduler still filters to explicit heartbeat agents:** The runner builds its active agent map from `resolveHeartbeatAgents(cfg)` and skips agents whose resolved interval is null/disabled; no compensating warning is emitted there.
  - file: [src/infra/heartbeat-runner.ts:202](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/infra/heartbeat-runner.ts#L202)
