## Summary

Keep open. Current `main` has help-path mitigations that skip preaction bootstrap and narrow builtin command registration, but I did not find strong evidence that the reported `openclaw <subcommand>` 60-180s latency is resolved on current `main`, and I did not find a direct post-release fix in the entry/run-main startup path that would justify a conservative auto-close.

## Evidence

- **Help preaction returns before bootstrap:** `registerPreActionHooks` exits immediately on help/version, before `ensureCliExecutionBootstrap(...)` would run. That means current help handling intentionally avoids config guard and plugin bootstrap work.
  - file: [src/cli/program/preaction.ts:64](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/cli/program/preaction.ts#L64)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Builtin command help registers only the primary command:** `runCli` resolves the primary command and, when present, loads only that builtin/sub-CLI before parsing; plugin CLI registration is skipped for builtin primaries. This is a targeted mitigation for command-help startup work, but it does not by itself prove the reported latency is gone.
  - file: [src/cli/run-main.ts:239](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/cli/run-main.ts#L239)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Tests cover the narrowed help path:** `run-main.exit.test.ts` asserts the real primary command is loaded before rendering command help, and `preaction.test.ts` asserts help/version skips preaction config work.
  - file: [src/cli/run-main.exit.test.ts:193](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/cli/run-main.exit.test.ts#L193)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **No obvious post-release fix in the main startup entry path:** Diffing `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`) to current `main` shows some CLI changes, but not `openclaw.mjs`, `src/entry.ts`, or `src/cli/run-main.ts`, so I did not find a clear landed fix for this specific startup-dispatch report after the latest release.
  - command: `git diff --name-only a9797214338ba31c52c796adbb75afb16e0684a9..5fe333ada85afd0def9e85985ce57e36aa00108a -- openclaw.mjs src/entry.ts src/cli CHANGELOG.md`
