## Summary

Keep PR #40873 open. Current main still has the TUI inline reasoning-tag leak for plain string assistant content on the stream/final extraction paths, while the PR discussion identifies a real implementation concern that should be fixed before merge: tag stripping must be scoped to assistant/provider output, not user or command strings that may contain literal XML examples.

## Best Possible Solution

Keep the PR open and revise it into an assistant-scoped TUI fix: strip inline reasoning tags from assistant string content in both final/history formatting and stream assembly, optionally surface tagged reasoning only when thinking display is enabled, and add focused regression tests for assistant string payloads plus a non-assistant literal `<think>` preservation case.
