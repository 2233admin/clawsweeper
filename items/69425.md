## Summary

Keep PR #69425 open. Current main does not implement OPENCLAW_WRAPPER or an equivalent wrapper-aware daemon ProgramArguments path, and the linked issue #69400 describes a still-plausible reinstall/update problem in the current install flow. The PR is focused and useful, but the existing review comment about using installer/merged service env instead of only process.env looks material before it is ready to land.

## Best Possible Solution

Keep the PR open and request changes rather than closing it. The best path is to implement wrapper-aware daemon ProgramArguments in src/daemon/program-args.ts for both gateway and node service args, pass the effective install environment through buildGatewayInstallPlan/buildNodeInstallPlan into that resolver, preserve or explicitly manage OPENCLAW_WRAPPER during reinstall/update, and add focused regression tests for valid wrapper, missing/non-file/non-executable wrapper, and existing LaunchAgent env preservation. CLI --wrapper and doctor reporting can either be added here if maintainers want the full #69400 shape or tracked as narrower follow-ups.
