## Summary

Keep open. Current main still hardcodes daemon service ProgramArguments through the Node/Bun CLI resolver and has no OPENCLAW_WRAPPER support. The linked issue #69400 is a concrete core daemon install/reinstall problem, and this PR is a plausible implementation direction, but it should be revised before merge because the supplied PR context shows it reads only process.env and would miss the merged install environment used during reinstall/update flows.

## Best Possible Solution

Keep the PR open and ask for a revision rather than auto-closing it. The core daemon resolver should support an explicit wrapper contract, but the implementation should pass the install-plan environment into program-argument resolution, intentionally preserve or persist OPENCLAW_WRAPPER when appropriate, validate executable access, and add focused regression tests for gateway/node arguments and reinstall/update preservation. Docs/changelog should be updated once the final CLI/env contract is settled.
