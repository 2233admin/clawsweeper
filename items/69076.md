## Summary

Keep PR #69076 open. Current main only has a partial manual-cancel notify suppression for no-output background execs; it does not implement the PR’s core behavior: explicit `killed` exec outcomes, completed control-action status for successful process `kill`/`remove`, suppressing notify-on-exit for killed sessions with output, or Telegram sent-message metadata.

## Best Possible Solution

Keep the PR open for maintainer review. The best code path is to adapt or merge the killed-status semantics in `src/agents/bash-tools.exec-runtime.ts` and `src/agents/bash-tools.process.ts`, keep regression coverage for manual-cancel notify suppression and process poll/log status, and separately decide whether the additive Telegram sent-message metadata belongs in the same PR or a follow-up.
