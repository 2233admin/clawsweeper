## Summary

Keep PR #72149 open. Current main still forwards `temperature` in the native Anthropic Messages transport whenever `options.temperature` is set and thinking is disabled, with no Opus 4.7 guard. The PR is a focused fix for a real linked live repro (#72123), and it is not implemented or superseded on main.

## Best Possible Solution

Keep PR #72149 open and finish it as a focused Anthropic Messages request-shaping fix: omit unsupported sampling parameters for Claude Opus 4.7 on the first attempt, preserve existing behavior for other models, and land regression coverage for the native Anthropic transport. Consider a small shared helper or follow-up for Anthropic-compatible provider plugins that have the same model-specific constraint.

## Risks / Open Questions

- QC's `top_p`/`top_k` concern is worth checking before merge. I did not find OpenClaw's embedded extra-param adapter forwarding those through this native Anthropic path, but Anthropic's docs group all sampling parameters together.
- The PR test covers the exact lower-case model id. Since the implementation also changes normalization, adding an uppercase/mixed-case regression would make the intended behavior harder to regress.
- Anthropic Vertex still passes `temperature: options?.temperature` in `extensions/anthropic-vertex/stream-runtime.ts:142`; that may need a separate provider-owned follow-up if the same Opus 4.7 request shape applies there.
