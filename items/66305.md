## Summary

Keep this PR open. Current main still clears the shared model catalog cache after a successful empty catalog load, so the PR's core behavior is not implemented. The linked user issue was closed by narrower paths that avoid catalog loading for ordinary allowlist-backed turns, but shared catalog surfaces still call `loadModelCatalog()` and can still repeat discovery after empty results.

## Best Possible Solution

Keep this PR open and review/rebase it, or port the equivalent fix to current main: briefly cache successful empty catalog loads, retry after a bounded window, keep import/filesystem failures retryable, make `useCache: false` and cache resets clear negative-cache state, preserve current `readOnly` semantics, guard stale concurrent loads, and keep the focused regression tests in `src/agents/model-catalog.test.ts`.
