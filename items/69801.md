## Summary

Keep open. Current `main` still does not implement the requested Control UI flow for creating a new direct session and navigating to it. The visible chat toolbar button only sends `/new`, which resets the current chat's `sessionId` for the same `sessionKey`; the UI does not call `sessions.create`, and the Sessions view only links existing sessions.

## Evidence

- **Visible button routes to `/new`:** The chat toolbar renders a visible plus button titled `New session`, but it only invokes the provided `onNewSession` callback.
  - file: [ui/src/ui/chat/run-controls.ts:24](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/ui/src/ui/chat/run-controls.ts#L24)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Control UI wires the button to reset semantics:** `onNewSession` is bound to `state.handleSendChat("/new", { restoreDraft: true })`, so the UI sends the reset command instead of creating a separate dashboard session.
  - file: [ui/src/ui/app-render.ts:2295](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/ui/src/ui/app-render.ts#L2295)

  - file: [docs/reference/session-management-compaction.md:136](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/docs/reference/session-management-compaction.md#L136)
