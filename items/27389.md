## Summary

Keep open. Current `main` does not implement Android `screen.record`: the Android app manifest only declares a `dataSync` foreground service, the Android node command registry/tests do not advertise `screen.record`, Android docs omit it from the supported command surface, and the changelog says Android/Play removed `screen.record` from the app.

## Evidence

- **Android manifest lacks MediaProjection service support:** The Android app declares `FOREGROUND_SERVICE` and `FOREGROUND_SERVICE_DATA_SYNC`, and `NodeForegroundService` is restricted to `android:foregroundServiceType="dataSync"`. There is no MediaProjection foreground-service permission/type here.
