## Summary

Keep open. Current `main` still turns heartbeat/cron/exec runtime prompts into transcripted `role:"user"` messages for the heartbeat run itself. The code now keeps drained queued system-event blocks out of `transcriptPrompt`, but it still persists the heartbeat/exec prompt text as the user turn, so the core misattribution reported in #66496 is not fixed.
