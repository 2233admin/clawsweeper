## Summary

Keep open. Current `main` still intentionally clears self-declared scopes for device-less operator WebSocket connections authenticated with a shared token/password, which matches the reported `missing scope` failure rather than disproving it or showing a fix on `main`.

## Evidence

- **Current gateway code still clears scopes on missing device identity:** `handleMissingDeviceIdentity()` still calls `clearUnboundScopes()` for device-less connections when `shouldClearUnboundScopesForMissingDeviceIdentity(...)` returns true. The helper currently returns true for allowed shared-token/password operator sessions outside narrow Control UI exceptions, so declared scopes are stripped to `[]`.
