## Summary

Keep this issue open. Current main still implements config-write authorization only at channel/account scope, and the provided related-item context does not identify a canonical duplicate or merged implementation. The earlier ClawSweeper comment reached the same conclusion, and rechecking edbab0e2 shows the same gap remains for DM/group/topic/conversation-specific permissions.

## Best Possible Solution

Keep this issue open. The best implementation is a generic core/plugin contract change: add backward-compatible schema/types for context-scoped configWrites, carry normalized conversation identity through /config and /allowlist authorization, expose SDK helpers for plugins, add contract tests for inheritance and denial behavior, then update bundled multi-context channel schemas/docs as they adopt the contract.
