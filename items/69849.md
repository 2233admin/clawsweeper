## Summary

Keep #69849 open. Current main still pins `@agentclientprotocol/sdk` 0.20.0, and the published SDK schema for that exact version still has optional array fields without `default([])` except for `authMethods`. OpenClaw’s ACP bridge actively uses that SDK for `openclaw acp`, and local ACP tests only cover requests with explicit `mcpServers: []`, so the reported Claude/API Pro schema-validation problem is not implemented away on main. The cross-referenced anomalyco/opencode ACP SDK issue is relevant external corroboration, but it is not a canonical OpenClaw duplicate or shipped fix.

## Best Possible Solution

Keep the issue open and fix the ACP SDK schema compatibility path. The best maintainer path is to update or patch `@agentclientprotocol/sdk` so optional array Zod fields that Claude turns into tool parameters have explicit `default([])` where protocol semantics allow omission, then add a focused regression test around the generated schema/request handling for omitted ACP array fields and Claude-compatible tool schema validation. Because OpenClaw pins the SDK and exposes `openclaw acp` as a supported Claude/acpx surface, this is actionable in the repo rather than a stale or external-only report.
