## Summary

Keep #70475 open. Current main still lacks a user-facing, schema-accepted Dreaming model override that is actually used for the Dreaming subagent/narrative LLM calls. There is a partial internal `execution.model` parser shape, and magejosh's comment correctly points to local embedding/QMD options as a related workaround, but that does not satisfy the requested dedicated Dreaming chat/subagent model selection.

## Best Possible Solution

Implement a documented Dreaming model override in memory-core, preferably one clear user-facing key such as `plugins.entries.memory-core.config.dreaming.model` or `dreaming.execution.defaults.model`, with optional per-phase overrides only if maintainers want that complexity. Add it to the plugin manifest/config docs, pass the resolved model into `api.runtime.subagent.run({ model })` for Dream Diary/narrative Dreaming calls, preserve current behavior when unset, and add regression tests proving configured model reaches the subagent runtime while unset inherits the agent default. Keep the local embedding/QMD workaround documented separately because it addresses memory search embeddings, not this LLM/subagent cost path.

## Risks / Open Questions
