## Summary

Keep open. Current `main` does not provide a supported Dreaming-specific model override. The only documented model-backed Dreaming work is the Dream Diary subagent, and it explicitly uses the default runtime model with no per-phase/per-dreaming model override wired at runtime. The issue body overstates current LLM use in light/REM/deep, but the underlying request is still a plausible feature request rather than something already implemented or incoherent.

## Risks / Open Questions

- The issue body assumes all Dreaming phases are LLM-backed; current main shows only the Dream Diary subagent uses the runtime chat model, so any follow-up should narrow the request to the actual model-backed path or separately justify new LLM-backed phase work.
- There is partial config-shape groundwork for Dreaming execution settings in `src/memory-host-sdk/dreaming.ts`, but it is not wired into runtime or docs. Closing as implemented would be premature.
