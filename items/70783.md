## Summary

Keep open. Current main already has multiple stored sessions, a chat session dropdown, and links from the Sessions table into the Chat view, but it does not implement the requested tab strip with create/switch/close/reorder/persisted open chat tabs. The Control UI still tracks one active session key for the chat surface and resets transient chat state when switching sessions.

## Best Possible Solution

Keep this open as Control UI product work. The best path is to build a chat-session tab strip on top of the existing sessions APIs and `?session=` routing, persist the open tab list/order locally, preserve per-tab draft/attachment/visible run state, support close and reorder affordances, and update Control UI docs plus responsive/UI tests. This does not belong in ClawHub because it is core Control UI navigation behavior, not an optional plugin capability.
