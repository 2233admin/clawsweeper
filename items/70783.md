## Summary

Keep #70783 open. Current main supports multiple sessions through a grouped session dropdown, URL session selection, and a separate Sessions management page, but it does not implement the requested top-of-chat tab system with open/close/reorder/persisted session tabs. This is a Control UI product feature, not a ClawHub/plugin close, and the provided related items do not supersede this UX request.

## Best Possible Solution

Keep the issue open as a Control UI enhancement. The likely implementation belongs in `ui/src/ui` using existing Gateway session APIs: add a chat-session tab strip backed by per-browser open-tab state, keep one active `sessionKey` for Gateway calls, preserve per-tab drafts/local UI state, support close and optional reorder/persistence, and update Control UI docs/tests around session switching, `/new`, active runs, and the Sessions page link flow.
