## Summary

Keep #47994 open. Current main has a partial auto-failover override self-heal for reply sessions, but it still does not implement the PR's fallback provenance flag or the related #68991 persisted-selection/runtime-accounting split. Command, reply, followup, cron, and gateway resolution paths still persist or prefer raw runtime model fields, so the sticky fallback problem is not clearly fixed or superseded on main.

## Best Possible Solution

Keep #47994 open for maintainer reconciliation. The best fix is to explicitly separate per-turn fallback runtime accounting from next-turn selected model persistence, or add robust fallback provenance on session runtime fields, while preserving the current `modelOverrideSource: "auto"` self-heal behavior. Close only after equivalent behavior lands on main with coverage for command, reply, followup, cron, status, sessions JSON/table, and gateway session-row consumers.
