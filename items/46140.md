## Summary

Keep open. Current `main` has per-agent tool-loop detection for repeated/no-progress calls, but it does not implement a hard per-turn or per-session tool-call quota like `maxToolCalls` / `maxToolCallsPerSession`. The report is recent, coherent, and still describes missing behavior rather than something already shipped.

## Evidence

- **No matching config names in repo:** A repo-wide search for `maxToolCalls` and `maxToolCallsPerSession` returned no matches on current `main`. That is strong evidence the requested config surface does not exist yet.
  - command: `rg -n "maxToolCalls|maxToolCallsPerSession" src docs packages extensions .github`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Current tool safety config is loop-detection only:** `ToolLoopDetectionConfig` defines repetition/no-progress thresholds (`historySize`, `warningThreshold`, `unknownToolThreshold`, `criticalThreshold`, `globalCircuitBreakerThreshold`), and `AgentToolsConfig` exposes `loopDetection` as the per-agent tool-call safety knob. There is no field for absolute per-turn or per-session call limits.
  - file: [src/config/types.tools.ts:165](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/config/types.tools.ts#L165)

- **Schema does not accept per-agent maxToolCalls fields:** The runtime Zod schema validates only the loop-detection fields under `tools.loopDetection`, and the agent schema accepts `tools: AgentToolsSchema` with no `maxToolCalls`-style fields. So current config validation would reject the proposed setting rather than enforce it.
  - file: [src/config/zod-schema.agent-runtime.ts:500](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/config/zod-schema.agent-runtime.ts#L500)

- **Docs describe per-agent loop detection, not hard quotas:** The user-facing config docs document `tools.loopDetection` as the per-agent override for tool-call safety and describe warning/critical thresholds for loop analysis. They do not document any absolute call-count cap per turn or per session.
  - file: [docs/gateway/config-tools.md:114](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/docs/gateway/config-tools.md#L114)
