## Summary

Keep open. Current `main` still blocks or drops non-image Control UI webchat attachments, so the PR addresses a real bug. It is not a conservative cleanup candidate because the bug is still present and the PR discussion already surfaced unresolved correctness risks that need maintainer review before merge.

## Evidence

- **Control UI picker is still image-only on main:** `CHAT_ATTACHMENT_ACCEPT` is `"image/*"`, and the MIME gate only allows values starting with `image/`, so generic files are rejected before send.
