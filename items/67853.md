## Summary

Keep open. Current `main` still does not implement the PR’s primary fixes: `memory_search`/`memory_get` still rely only on registered corpus supplements with no direct `memory-wiki` fallback, and plugin tool resolution still prefers the active gateway-bindable registry instead of the compatibility-checked snapshot path described in the PR. The PR discussion also contains valid follow-up findings that the author said were fixed locally but not yet pushed.
