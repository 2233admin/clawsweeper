## Close Comment

Closing from a Codex review because current `main` already contains the fix for this regression. The `2026.3.22` changelog explicitly notes the gateway WebSocket handshake change that raised the default pre-auth timeout to 10 seconds so busy local gateways stop dropping healthy CLI connections (`CHANGELOG.md:2442`). Current code still reflects that shared timeout in `src/gateway/handshake-timeouts.ts:1`, applies it on the server handshake path in `src/gateway/server/ws-connection.ts:366`, and keeps the client/connect-challenge path pinned to the same contract in `src/gateway/handshake-timeouts.test.ts:13`. Current `main` also includes additional handshake hardening in `src/gateway/call.ts:479` for command-path event-loop starvation. Based on that evidence, this specific 2026.3.12/2026.3.13 regression is implemented on `main` and was shipped in 2026.3.22. If you still see similar timeouts on a newer build, please open a fresh issue with current-version logs and a reproduction so it can be tracked separately.
