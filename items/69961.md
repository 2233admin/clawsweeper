## Summary

Keep PR #69961 open. Current main at 43a003b8a0629ad2c73947b2a863d99dd506bcb9 still persists raw session transcript text into session-memory files, so the linked #69943 bug is not implemented on main. The PR is a real, focused fix candidate with tests and active affected-user discussion; newer overlapping PR #71257 is also open and does not clearly supersede this one.

## Best Possible Solution

Keep #69961 open and treat #69943 as the canonical bug. The best maintainer path is to land one focused session-memory fix that sanitizes transcript content before persistence/reuse, preferably reusing the existing shared assistant-visible text sanitizer where it fits, adds targeted tests for chat-template tokens, closed and dangling tool-call XML, role-marker leakage, `NO_REPLY`, and short valid replies, then explicitly resolves or supersedes the overlapping #71257 work.
