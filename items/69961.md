## Summary

Keep open. Current main still has the raw session-memory persistence path that #69961 is trying to fix: transcript text is copied into `user:`/`assistant:` lines and written under `## Conversation Summary` without sanitizing chat-template tokens, raw `<tool_call>` XML, role-marker leakage, or `NO_REPLY`. The PR is open and unmerged, the canonical bug #69943 is still open, and the later overlapping #71257 is also open/unmerged rather than a shipped superseding fix.

## Best Possible Solution

Keep #69961 open and use it, #71257, and #69943 to converge on one scoped session-memory fix. The implementation should sanitize or elide model artifacts before memory persistence, preferably reuse or align with `src/shared/text/assistant-visible-text.ts`, cover `NO_REPLY`, chat-template tokens, dangling and closed tool-call XML, role-marker leakage, and false-positive cases in tests, then close #69943 after merge.
