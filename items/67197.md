## Summary

Keep open. Current main still only prepends model fallback and fallback-cleared reply notices when verbose mode is enabled, the existing e2e tests assert verbose-off suppression, and there is no independent config/schema/docs surface for the requested opt-in. The same author also has open PR #69399 with closing syntax for this issue, so the issue should remain open until that PR lands or maintainers choose another implementation.

## Best Possible Solution

Keep this issue open and review #69399. The likely implementation path is a default-false agent-defaults option that lets runReplyAgent prepend fallback and fallback-cleared notices when either verbose mode is enabled or the opt-in is true, with config type/schema/help/docs updates and e2e coverage for fallback, once-per-active-state suppression, and fallback-cleared behavior.
