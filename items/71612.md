## Summary

Keep PR #71612 open. Current main still records tool-loop history against the stable cron agent session key, and `recordToolCall` has no idle-gap or run-boundary reset, so the PR addresses a plausible current bug with focused code and regression tests rather than obsolete or duplicate work.

## Best Possible Solution

Keep the PR open for maintainer review. The best product fix is still to scope loop-detection history to an actual isolated cron run boundary, either with this PR's idle-gap reset plus regression tests or a more explicit diagnostic reset/keying hook that uses the existing per-run cron session key.
