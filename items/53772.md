## Summary

Keep open. Current main has the narrower `agents.defaults.compaction.model` override for compaction summarization, including tests and docs, but the requested local-model routing for pre-compaction memory flush and startup/housekeeping work is still missing. The pre-compaction memory flush still calls the normal active-run model fallback path, and startup context is read from disk and prepended to the active user turn rather than processed by a separate housekeeping model.

## Best Possible Solution

Keep this issue open and narrow the implementation target. The most concrete path is to add a dedicated maintenance or memory-flush model override, route `runMemoryFlushIfNeeded` through it, and cover schema/docs/tests for fallback, auth-profile, local-model context, and tool/workspace behavior. Startup reads should remain a separate product decision unless maintainers decide to add summarized startup-context preprocessing.
