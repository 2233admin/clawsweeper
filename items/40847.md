## Summary

Keep open. Current main has a generic media-understanding warning for failed decisions, but the remaining requested image zero-output diagnostics are not fully implemented: no-provider and all-skipped image paths still return as skipped without a warning, and the existing failed warning does not include failed attempt provider/model details. The PR itself is also not ready as-is because review comments identified the same unreachable/no-warning gaps, but the underlying observability request remains valid.

## Best Possible Solution

Keep the item open for a revised, focused patch in src/media-understanding/runner.ts. The best fix is a single non-duplicative diagnostic path for selected image attachments that produce zero outputs: move the no-provider warning before the early return, cover all-skipped attempts such as empty output, include provider/model/type/outcome/reason where available, and add focused tests for provider failure, empty-output skip, and no resolved providers.
