## Summary

Keep PR #39888 open. Current main still does not emit a `model.usage` diagnostic event from the `openclaw agent` CLI path, while the reply-runner path does and diagnostics exporters consume that event for token, cost, context, and duration telemetry. The PR remains useful, but it is stale against current main because `src/commands/agent.ts` is now only a re-export and the implementation moved to `src/agents/agent-command.ts`.

## Best Possible Solution

Refresh this PR or replace it with a small current-main change in `src/agents/agent-command.ts` or a shared post-run helper. After `updateSessionStoreAfterAgentRun`, emit a trusted `model.usage` event for CLI/local agent runs with `channel: "cli"`, session/agent/provider/model metadata, normalized usage and last-call usage, context limit/used, cost, duration, and a focused regression test while preserving the reply-runner behavior.
