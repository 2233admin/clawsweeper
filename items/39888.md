## Summary

Keep PR #39888 open. Current main still has the observability gap the PR targets: the CLI/agent command path updates session usage and cost state, then persists/delivers the result, but it does not emit a `model.usage` diagnostic event. The submitted patch is stale because `src/commands/agent.ts` is now only a re-export and the real implementation lives in `src/agents/agent-command.ts`, but the underlying fix remains useful and not superseded.

## Best Possible Solution

Refresh this PR or replace it with a small main-targeted change in `src/agents/agent-command.ts` or a shared command helper that emits a trusted `model.usage` diagnostic after the CLI/agent run updates session state. Reuse current context-token, usage, and cost helpers, include `agentId` plus an appropriate CLI/ingress channel label, preserve the existing reply-runner event behavior, and add a focused CLI command regression test for diagnostic emission.
