## Summary

Keep open. Current main has Matrix bot-to-bot `allowBots` handling, including mention-only gating for messages sent by other configured Matrix accounts, but it does not implement the requested opt-in suppression for a human room message that explicitly mentions a different configured bot while this account has `requireMention: false`. The Matrix mention resolver only reports whether the current account or `@room` was mentioned, and the handler only skips unmentioned room messages when `shouldRequireMention` is true, so the reported Bot B dispatch path still remains.

## Evidence

- **Issue request and follow-up are concrete:** The issue asks for pre-dispatch skipping when a message mentions a different bot account in the same OpenClaw Matrix instance, specifically to avoid typing indicators and LLM calls for non-mentioned accounts with `requireMention: false`. The reporter reiterated on 2026-03-20 that a 10-bot Matrix setup still processes every message regardless of who was mentioned.
- **Existing Matrix bot policy is sender-based, not other-mentioned-target-based:** The handler resolves `allowBots` from room/account config, checks whether the sender is one of the configured bot user IDs, and drops only those configured-bot senders when `allowBots` is off. That does not cover a human sender mentioning a different configured bot.
