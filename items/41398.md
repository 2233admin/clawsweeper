## Summary

Current `main` does not already implement this Google model-prefix fix. The relevant normalization paths still leave provider-prefixed Google model IDs intact, and the native Gemini PDF path still builds the REST URL from the raw `modelId`, so there is not enough evidence to close this PR as already implemented or obsolete.
