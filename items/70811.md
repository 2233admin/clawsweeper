## Summary

Keep PR #70811 open. Current main still unconditionally validates `entry?.config` in the normal plugin loader and CLI metadata loader, so the false-positive undefined-config path described in #70658 is not fixed on main. The PR is a focused, active contributor bug-fix with regression coverage in its diff and follow-up commits addressing the bot feedback about `validatedConfig` scoping, normalized `pluginConfig`, and schema defaults.

## Best Possible Solution

Keep the PR open for maintainer review and land this fix, or an equivalent core-loader patch, with the same boundaries: defer validation only for non-activating loads when config is undefined, keep activation and explicit-config validation strict, preserve normalized/defaulted `api.pluginConfig`, and ship focused loader plus CLI metadata regression tests. #70658 should close through that fix; #64946 remains a related but distinct validator-noise issue.
