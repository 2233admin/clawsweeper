## Summary

Keep open. Current `main` still validates `entry?.config` in both non-activating plugin snapshot loads and CLI metadata loads, so the reported bug is not implemented on `main`. The PR appears directionally correct and adds targeted coverage, but the latest review context still raises a concrete unresolved behavior gap around schema defaults in the deferred-validation path.
