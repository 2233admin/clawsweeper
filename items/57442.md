## Summary

Keep open. Current main still does not add elapsed/response duration to the shared `/status` / `session_status` status card. The renderer only shows the session key plus last-updated age, while the `session_status` tool and `/status` command both delegate through `buildStatusText()` without passing any duration/time-line field. Related #68226 and PR #51683 overlap, but they are open prior art rather than a landed or clearly canonical superseding item.

## Best Possible Solution

Keep the issue open and implement the elapsed-time display in the shared core status path. A conservative implementation would derive duration from `SessionEntry.sessionStartedAt` with existing transcript-header fallback for legacy rows, pass it through `buildStatusText()` into `buildStatusMessage()`, render it on the `🧵 Session:` line for both `/status` and `session_status`, and add focused renderer/tool tests plus a docs update. If maintainers prefer consolidation, choose one canonical thread among #57442, #68226, and #51683 before closing duplicates.
