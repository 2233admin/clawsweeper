## Summary

Keep open. Current `main` still loads internal hooks exactly once at gateway startup, and that load is scoped to `defaultWorkspaceDir`, so non-default agent workspace hooks are still not discovered. The docs still describe workspace hooks as per-agent.

## Evidence

- **Startup hook load still uses only the default workspace:** `startGatewaySidecars()` calls `loadInternalHooks(params.cfg, params.defaultWorkspaceDir)` and does not iterate agent workspaces or reload per session.
  - file: [src/gateway/server-startup-post-attach.ts:235](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/gateway/server-startup-post-attach.ts#L235)
  - command: `nl -ba src/gateway/server-startup-post-attach.ts | sed -n '220,320p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **`defaultWorkspaceDir` is derived from the default agent only:** Gateway bootstrap resolves `defaultAgentId`, then resolves a single `defaultWorkspaceDir` from that agent and passes it through startup.
  - file: [src/gateway/server-startup-plugins.ts:61](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/gateway/server-startup-plugins.ts#L61)
  - command: `nl -ba src/gateway/server-startup-plugins.ts | sed -n '1,140p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Hook loader only accepts one workspace directory:** `loadInternalHooks()` takes one `workspaceDir` argument and discovers workspace hooks from that directory only.
  - file: [src/hooks/loader.ts:80](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/hooks/loader.ts#L80)

- **Current docs still describe workspace hooks as per-agent:** Docs say workspace hooks live at `<workspace>/hooks/` and are `per-agent`, which still conflicts with the single-default-workspace startup load path.
  - file: [docs/automation/hooks.md:136](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/automation/hooks.md#L136)
