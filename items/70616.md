## Summary

Keep #70616 open. Current main still has the reported memory-core dreaming config gap: the public manifest schema rejects `dreaming.execution` and per-phase `execution`, docs only document `enabled`/`frequency`, and the existing execution resolver is not wired into the managed dreaming cron or narrative subagent fallback path. Recent cron code supports `payload.fallbacks` generally, but memory-core does not pass model/fallback overrides into that surface.

## Best Possible Solution

Keep the issue open and fix it in memory-core/core config. Maintainers should decide the public shape for `dreaming.execution` and phase overrides, update `extensions/memory-core/openclaw.plugin.json`, generated config/docs/tests, and wire resolved execution settings into the managed dreaming cron payload using the existing cron `model`/`fallbacks` support. If dream narrative runs also need fallback chains, add a typed plugin subagent fallback override or route them through a surface that already supports model fallback rather than accepting schema-only config.
