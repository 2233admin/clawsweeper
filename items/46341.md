## Summary

Keep open. Current `main` still applies transient-error retry logic only to one-shot (`at`) jobs. Recurring `cron`/`every` jobs always schedule the later of the natural next slot and the backoff floor, so long-interval jobs still wait until the next scheduled window instead of retrying soon after transient failures.

## Evidence

- **One-shot path has transient retry gating:** `applyJobResult()` checks `isTransientCronError(...)` and `resolveRetryConfig(...)` only inside the `job.schedule.kind === "at"` branch, then schedules `result.endedAt + backoff` for transient failures.
  - file: [src/cron/service/timer.ts:467](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cron/service/timer.ts#L467)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '466,550p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Recurring path still ignores transient/permanent distinction:** The recurring error branch does not call `isTransientCronError()` or `resolveRetryConfig()`. It always computes `backoffNext`, then uses `resolveCronNextRunWithLowerBound(...)` for `cron` jobs or `Math.max(normalNext, backoffNext)` for `every` jobs.
  - file: [src/cron/service/timer.ts:510](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cron/service/timer.ts#L510)

  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Lower-bound helper proves long-period jobs skip immediate retry:** `resolveCronNextRunWithLowerBound()` returns `Math.max(params.naturalNext, params.lowerBoundMs)`, so when `naturalNext` is hours or days out, the backoff timestamp is discarded rather than used as an earlier retry.
  - file: [src/cron/service/timer.ts:223](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cron/service/timer.ts#L223)

That separate handling is the non-transient lower-bound logic above, so the reported asymmetry is still present on `main`.
