## Summary

Keep #39811 open. Current main still does not perform catalog-aware validation for configured primary/fallback model refs in `openclaw config validate`; schema validation remains shape-only, configured unknown models are preserved as valid selection entries, and the related PR that would add advisory checks is still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #39811 open. The best maintainer path is to add catalog-aware warnings to `openclaw config validate` and doctor/startup surfaces for configured model refs across defaults, per-agent overrides, fallbacks, allowlists/aliases, and relevant hook/channel/tool model settings. PR #40355 can be reused if rebased and broadened, but the policy should preserve dynamic/custom provider IDs as warnings where OpenClaw cannot prove invalidity.
