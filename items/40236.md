## Summary

Keep #40236 open. Current main has overlapping cron behavior, but it still does not implement the requested job-level lifecycle contract: there is `payload.timeoutSeconds`, one-shot `deleteAfterRun`, failure notifications, run/session pruning, and plugin hooks with `ctx.jobId`, but no persisted per-job `lifecycle` block, no run-count/TTL/completed-state cleanup policy, and no cron lifecycle events routed through internal hooks. Related #37855 and PR #45465 are narrower/open hook efforts; #33167 was closed stale rather than shipped as this broader cleanup/self-termination policy.

## Best Possible Solution

Keep the issue open but narrow the implementation path. If core wants this first-class, define an additive cron lifecycle schema in `src/gateway/protocol/schema/cron.ts` and `src/cron/types.ts`, implement scheduler cleanup and timeout/retry semantics in `src/cron/service/*`, and route cron lifecycle events through the internal hook system with docs/tests. Otherwise split it into focused follow-ups: cron internal hook events, recurring-job auto-cleanup/self-deletion, and timeout/retry semantics.
