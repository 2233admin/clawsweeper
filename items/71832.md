## Summary

Issue #71832 asks for a directory-scoped `subclaw` child-agent/TUI command. Current main does not ship a `subclaw` entrypoint, and the related #72050 implementation PR is open and unmerged, but the requested capability is an optional orchestration wrapper that current plugin/ClawHub surfaces can provide: plugins can register root CLI commands, use subagent runtime helpers, and rely on existing TUI workspace/session behavior. The issue discussion also already links a ClawHub skill for multi-agent coordination patterns.

## Best Possible Solution

Close the core issue as ClawHub/plugin scope. A `subclaw` wrapper should live as an external ClawHub plugin or skill using `api.registerCli`, existing TUI workspace inference, and subagent/session runtime helpers. If that external implementation uncovers a concrete missing plugin API, open a narrow API issue for that seam.
