## Summary

Keep #71584 open. Current main still rejects explicit `/think max` for `ollama/deepseek-v4-flash:cloud` before provider dispatch because generic/Ollama thinking profiles only expose `off`, `minimal`, `low`, `medium`, and `high` unless a provider hook declares `max`, and the Ollama plugin does not. The DeepSeek V4 `max` payload mapping exists only for the bundled `deepseek` providerâ€™s exact model IDs, not for Ollama Cloud `:cloud` refs.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the Ollama provider/plugin path: expose a provider-owned `resolveThinkingProfile` for Ollama models that support `max` (possibly using `/api/show` thinking capability plus model-family rules), and add tested payload mapping so `/think max` either sends Ollama's supported `reasoning_effort: "max"` where applicable or consistently downgrades to the highest supported level with an explicit acknowledgement. If config-level `compat.supportedReasoningEfforts` is intended to drive `/think` options, wire that into profile resolution with regression coverage.
