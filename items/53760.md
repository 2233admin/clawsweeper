## Summary

Keep this issue open. Current main still has no enforced DM-level sender tool policy. Telegram has typed/schema support for `direct.*.toolsBySender`, but active tool filtering only resolves group/channel tool policy; Signal still exposes `toolsBySender` only under group config.

## Best Possible Solution

Implement a generic direct-message sender tool-policy contract, then wire it through the same enforced tool filtering paths as group policy. Concretely: add shared typed config/schema/docs for DM-level `tools` and `toolsBySender`, resolve verified direct sender identity for Signal and Telegram, feed the resolved DM policy into `createOpenClawCodingTools`, `applyFinalEffectiveToolPolicy`, gateway scoped tool resolution, and media/tool helpers, and add regression tests proving one allowlisted DM sender can retain tools while another allowlisted sender is denied them.
