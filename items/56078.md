## Summary

Current `main` does not show a strong implemented fix for #56078. New isolated cron jobs still default to announce delivery, Control UI still submits announce delivery unless the user switches to `none`, delivery previews explicitly say unresolved `channel:"last"` routes will fail closed, and the cron runtime only downgrades unresolved delivery to a successful session-only run when `bestEffort` is explicitly enabled. That leaves the reported local/webchat-only failure mode plausible rather than clearly resolved.
