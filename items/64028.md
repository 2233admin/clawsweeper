## Summary

Keep this PR open. Current main still does not support `session.reset.mode = "off"`, and the PR remains a plausible implementation candidate for a core session lifecycle config feature, but it still needs focused cleanup before maintainer merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is a focused patch that adds `session.reset.mode: "off"` consistently across exported types, Zod validation, generated schema/help/docs, and `evaluateSessionFreshness`; adds a unit test proving stale daily/idle timestamps remain fresh when mode is off; regenerates `docs/.generated/config-baseline.sha256`; and removes or splits the unrelated SDK runtime-logger cleanup.
