## Summary

Keep PR #64028 open. Current main still does not implement the requested explicit `session.reset.mode = "off"` contract: runtime types, Zod validation, generated schema, and help text all expose only `daily` and `idle`. There is a lower-level behavior where `idleMinutes: 0` evaluates as never expiring, but public config validation disallows explicit `0` and docs/schema help do not present this as a supported automatic-reset opt-out. The PR is focused useful core config work, though it should get a targeted reset-policy test and should drop or justify the unrelated `runtime-logger` cleanup before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land the explicit `session.reset.mode = "off"` core config contract with aligned types, Zod schema, generated config schema/help/hash, and focused tests, while removing unrelated cleanup from the PR. If maintainers prefer the existing implicit idle-no-timeout behavior instead, document that as the supported opt-out and then close this PR with that product decision.
