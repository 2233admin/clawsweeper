## Summary

Keep open. Current `main` still stores cron definitions in a single `cron/jobs.json` file with runtime state in `cron/jobs-state.json`; the proposed per-job `cron/jobs/*.json` hierarchy is not implemented on `main`, and the PR discussion already documents unresolved correctness risks in the submitted approach.
