## Summary

Keep this issue open. Current main mitigates the related corruption class with atomic writes to `exec-approvals.json`, but it still stores and mutates `lastUsedAt`, `lastUsedCommand`, and `lastResolvedPath` inside the allowlist file. I found no `exec-runtime.json` store or merge/display path for separated runtime tracking.

## Best Possible Solution

Keep this open as a focused core hardening feature. A maintainer-quality fix should add a backward-compatible `exec-runtime.json` usage store, stop normal gateway/node/macOS exec usage tracking from mutating `exec-approvals.json`, preserve existing allowlist reads and legacy `lastUsed*` display, update protocol/UI/CLI/docs, and add tests proving a read-only approvals file can still enforce allowlist policy while runtime tracking remains writable.
