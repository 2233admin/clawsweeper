## Summary

Current main still leaves the requested behavior missing: direct replies resolve and apply `messages.responsePrefix`, but message-tool `send` normalizes the raw message text, optionally adds only cross-context decoration, and passes that text through plugin/core/gateway delivery without resolving or applying `responsePrefix`. The prior keep-open comment remains accurate against current main `8ba82534e6c98410e19b226f3534b285fd75d278`.

## Best Possible Solution

Keep #39913 open. Implement response-prefix application once at the shared message-tool outbound send boundary before plugin/core/gateway dispatch, resolving the effective prefix with agent, channel, and account context and preserving the same idempotent/template semantics as direct replies. Add focused regression coverage for `runMessageAction`/`handleSendAction`, `executeSendAction`, and direct/gateway delivery, including media-only sends, cross-context decoration, mirrors/transcripts, and plugin-dispatch behavior.
