## Summary

Current main still has the reported gap. Direct replies resolve and apply `messages.responsePrefix`, including the per-channel/account cascade, but `message(action="send", ...)` keeps the tool text raw through plugin dispatch, core send, and gateway delivery. The related per-channel prefix work is implemented, but that cascade is not wired into the message-tool outbound path.

## Best Possible Solution

Keep #39913 open and implement prefixing at the shared message-tool outbound boundary. Resolve `responsePrefix` with agent, channel, and account context, normalize the outgoing text through `normalizeReplyPayload()` or an equivalent shared helper before plugin/core/gateway dispatch, preserve media-only and empty-message semantics, and add regression coverage for `runMessageAction` plus the gateway send path. If maintainers want per-send suppression, add an explicit documented option rather than leaving the two outbound paths inconsistent.
