## Summary

Current `main` has some overflow mitigation for exec approvals, but I did not find strong evidence that PR #50980's requested layout contract and regression coverage are already implemented well enough to justify closing it as obsolete or implemented.

## Evidence

- Closing as implemented_on_main would rely on inference rather than a matching mainline change or equivalent regression test, which is too weak for conservative cleanup.
