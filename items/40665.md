## Summary

Keep open. Current main still has the Discord voice autoJoin path wired only to READY, not RESUMED, and the voice connection teardown still calls destroy() directly without an idempotent stale-connection cleanup path. The provided GitHub context also shows same-author PR #40676 is open, unmerged, and explicitly intended to fix this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open as the canonical Discord voice reliability bug. Review and either land PR #40676 or replace it with an equivalent Discord plugin fix that registers autoJoin for RESUMED as well as READY, makes stale VoiceConnection cleanup idempotent when the dependency reports Destroyed, scopes cleanup safely for same-guild multi-account setups, and adds regression coverage for RESUMED autoJoin and already-destroyed stale cleanup.
