## Summary

Keep open. The issue and linked PR #40676 identify READY-only autoJoin plus unsafe stale/destroyed voice connection cleanup, and current main at 8ba82534 still has those gaps: Discord voice autoJoin is registered only through DiscordVoiceReadyListener, and the join/stop paths still call connection.destroy() without a destroyed-state guard or account-scoped stale connection handling.

## Best Possible Solution

Keep #40665 open as the canonical Discord voice reliability bug until a maintainer-reviewed fix lands. The likely fix belongs in the Discord plugin: add and register a RESUMED listener that triggers autoJoin, make VoiceConnection teardown idempotent when the upstream state is Destroyed, and scope any stale connection cleanup to the owning account/manager, likely through an explicit group or ownership check. Add regression coverage for RESUMED autoJoin, already-destroyed cleanup, and multi-account same-guild safety, then close this issue from the landed PR.
