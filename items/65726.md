## Summary

Current main still does not implement the requested ACP binding notifyPolicy config or a core bound-thread duplicate guard. Task notify policies exist, and there is a Slack thread-ownership plugin that can help some multi-agent Slack setups, but ACP binding/runtime config still cannot set notifyPolicy, ACP spawn task creation still falls back to visible done_only notifications, and the existing delivery suppression only applies to parent-owned oneshot ACP sessions rather than persistent ACP-bound threads.

## Best Possible Solution

Keep this issue open. Add typed notifyPolicy support to AgentRuntimeAcpConfig and/or AgentAcpBinding.acp, update strict Zod schemas, schema help/docs, binding materialization, and ACP spawn task creation so a resolved silent policy suppresses ACP-bound completion notices. Separately design a channel/account/conversation/thread ownership guard for active ACP-bound sessions, with Slack tests covering persistent bound threads and ordinary intentional manual replies.
