## Summary

Current main has related delivery-route hardening, but there is not strong evidence that issue #49055 is fixed. The agent delivery path still treats delivery as successful without surfacing a delivery result when no outbound target is resolved, and the added tests only cover stale session-route precedence, not overloaded_error/model-fallback recovery with real channel delivery.

## Evidence

- **Agent delivery still has a silent no-send branch:** `deliverAgentCommandResult()` only calls `deliverOutboundPayloads()` when `deliveryTarget` is truthy; otherwise it falls through and returns `{ payloads, meta }` with no error or delivery-status signal.
  - file: [src/agents/command/delivery.ts:365](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/agents/command/delivery.ts#L365)

## Risks / Open Questions

- The recent turn-source routing hardening may fix one race behind some reports, but the repo does not provide strong proof that it fixes the overloaded_error recovery path described in #49055.
- Because `deliverAgentCommandResult()` can return success without a delivery acknowledgment, silent drops remain plausible when routing state is missing or stale.
