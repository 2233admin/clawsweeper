## Summary

Keep open. The issue is coherent, recent, and current main does not give enough evidence to close it as implemented or obsolete. The code still refreshes persisted skill snapshots from process-local version counters initialized to 0, while the linked discussion specifically identifies restart-stale `skillsSnapshot` state as a likely cause. Current main also still has prompt count/character limits that can intentionally omit visible skills from `<available_skills>`, so the central user complaint is not conclusively solved.
