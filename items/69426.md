## Summary

Keep #69426 open. Current main still does not implement the PR’s central fix: `context1m: true` is still gated to `anthropic` or `claude-cli`, so proxied Claude Opus/Sonnet models such as `github-copilot/claude-opus-4-6` remain excluded from the 1M context path. The PR is a focused bug fix for the closed linked report #69353, not obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The likely product path is to accept the provider-agnostic model-prefix check for explicit `context1m` opt-ins, keep the non-Opus/Sonnet guard, add the proxied-provider regression tests, and fix the changelog entry to include the required `Thanks @sk7n4k3d` attribution before merge.
