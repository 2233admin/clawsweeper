## Summary

Keep #69426 open. Current main does not implement the PR's central proxied-provider behavior: `context1m: true` is still honored only when the resolved provider is `anthropic` or `claude-cli`, so the reported `github-copilot/claude-opus-4-6` case still falls through to configured/cache/fallback context limits. Later main-line work appears to have extended this path to Claude CLI only, not to GitHub Copilot, LiteLLM, Bedrock, or other Anthropic-proxy providers.

## Best Possible Solution

Keep the PR open or ask for a refreshed patch against current main. The best implementation should preserve explicit opt-in plus model-family gating, add the GitHub Copilot positive/negative regression tests proposed in #69426, and clarify whether additional proxy providers such as LiteLLM, custom OpenAI-compatible Anthropic routes, and Bedrock should use the same resolver rule or provider-owned context metadata.
