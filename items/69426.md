## Summary

Keep #69426 open. Current main still has the provider guard that the PR removes, so `context1m: true` remains ineffective for `github-copilot` and other proxied Claude Opus/Sonnet 4 models unless another configured context window happens to override it. The PR is a focused fix for the reported #69353 behavior and is not obsolete on main.

## Best Possible Solution

Land a scoped variant of #69426: make the explicit `context1m` planner override depend on the Claude Opus/Sonnet model-id capability prefix rather than the provider id, keep the non-matching-model guard, add the PR's proxied-provider regression tests, fix changelog attribution, and verify whether proxy transports need any separate request-header/API handling.
