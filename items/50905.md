## Summary

Current `main` already suppresses TUI/non-webchat agent events from the Control UI chat path. Runs are marked `isControlUiVisible` only for the internal `webchat` channel, hidden runs drop `sessionKey` during event emission, and the gateway chat broadcaster skips chat/session delivery when `isControlUiVisible` is false. Regression tests cover both the hidden-session-key case and the no-broadcast case.

## Close Comment

Closing from a Codex review because current `main` already implements this fix. The Control UI visibility gate now only treats internal `webchat` runs as browser-visible (`src/utils/message-channel.ts:61`, `src/utils/message-channel-constants.ts:1`), agent runs store that as `isControlUiVisible` (`src/auto-reply/reply/agent-runner-execution.ts:668`), hidden runs drop their `sessionKey` before agent events are emitted (`src/infra/agent-events.ts:209`), and the gateway chat broadcaster skips chat/session delivery when a run is not Control-UI-visible (`src/gateway/server-chat.ts:590`). There are also regression tests covering both the hidden-session-key case and the no-broadcast case (`src/infra/agent-events.test.ts:78`, `src/gateway/server-chat.agent-events.test.ts:1333`). I could not confirm a released tag containing the fix from this checkout, so I’m marking it implemented on `main` rather than shipped in a known release.
