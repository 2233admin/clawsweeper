## Summary

Keep PR #69932 open. Current main has related lower-level BlueBubbles Private API guards, but it does not contain this PR's monitor-level typing guard or disabled-path monitor test. The PR is a focused, recent bug-fix candidate with active discussion about the correct unknown-cache behavior, so it needs maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for BlueBubbles maintainer review. The likely path is to land a monitor-level guard plus the disabled-path test, but first decide the desired behavior for unknown Private API cache state: fail closed as the PR currently does, preserve the lower-level fallback, or add a lightweight lazy probe before typing.
