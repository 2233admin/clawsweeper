## Summary

Do not close PR #69932. Current main still has the BlueBubbles monitor typing start, restart, and stop calls unguarded by `privateApiEnabled`. The lower-level chat helper only skips calls when cached status is explicitly `false`, so expired/unknown cache state can still attempt the Private API typing endpoint. Related PR #65447 fixed send/attachment lazy refresh, not this monitor typing path, and current monitor tests still lack the disabled-path coverage added by this PR.

## Best Possible Solution

Keep #69932 open for maintainer review. The best path is to either merge the focused BlueBubbles monitor fix with its disabled-path test, or adjust it to a maintainer-preferred policy for unknown Private API status, such as a tri-state helper or lazy server-info refresh before typing indicators.
