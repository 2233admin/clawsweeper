## Summary

Keep open. Current `main` still does not implement the reported fallback or make chunk classification stable: the OpenAI-compatible stream parser sends any non-empty `delta.content` straight to visible text and skips reasoning parsing for that chunk, while `reasoning_content`/`reasoning`/`reasoning_text` are still mapped only to `thinking` with no fallback to visible text when normal content is absent. Downstream runner code recognizes reasoning-only turns as an incomplete result and retries or surfaces an error, which is mitigation rather than a fix for the underlying assembly/classification problem.
