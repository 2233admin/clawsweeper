## Summary

Current `main` does not expose main-vs-subagent lineage on `PluginHookAgentContext`, so the reported SDK gap still appears real and in scope for OpenClaw core rather than ClawHub.

## Evidence

- **Agent hook context type lacks subagent fields:** `PluginHookAgentContext` currently exposes only run/session/workspace/model/message/channel fields; there is no `spawnDepth`, `spawnedBy`, `isSubagent`, or similar lineage field.
  - file: [src/plugins/hook-types.ts:155](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/plugins/hook-types.ts#L155)

  - file: [src/agents/harness/hook-context.ts:3](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/harness/hook-context.ts#L3)
