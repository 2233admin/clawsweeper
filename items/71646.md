## Summary

Current main still matches #71646. `OpenClawChannelBridge` keeps `pendingClaudePermissions` and `pendingApprovals` as plain Maps without TTL/sweeper, close cleanup, or cap, while sibling queue/waiter state has bounded cleanup behavior. The linked fix PR #71648 is still the right path, but it is not merged on current main.

## Best Possible Solution

Keep #71646 open until #71648 or an equivalent maintainer-approved fix lands. The fix should clear both pending Maps on bridge close, expire stale Claude permission and approval entries, preserve the MCP protocol surface, and add focused regression tests. Maintainers can decide whether the cap/FIFO guard belongs in the same change or a follow-up.
