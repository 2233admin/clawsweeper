## Summary

Keep open. Current main still contains the unbounded suffix-overlap scan in the gateway live-chat merge path, and `server-chat.ts` still uses that helper for assistant delta buffering/final chat output. PR #59057 is not implemented on main, but it also needs rebasing/adaptation because the helper now lives in `src/gateway/live-chat-projector.ts`, and the existing review comments raised a real correctness concern about a hard 256-character cap duplicating text for legitimate long replay windows.

## Best Possible Solution

Keep the PR open, but do not merge it as-is. Rebase the work onto the current `src/gateway/live-chat-projector.ts` helper and replace the quadratic scan with a correctness-preserving approach, such as a linear-time overlap search or a bounded candidate strategy that still handles long replay windows safely. Add focused tests for repeated-character worst cases, segmented/tool-call replay with overlaps longer than 256 characters, and final buffered chat output.
