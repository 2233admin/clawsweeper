## Summary

Current main still matches the reported failure shape. Isolated `agentTurn` cron jobs still run through the embedded agent stream path, OpenAI-compatible tool-call argument chunks are still blindly appended into `partialArgs`, completed tool-call blocks still retain that scratch field, and the repair wrapper still rejects duplicated complete JSON objects as unsafe trailing content. The prior ClawSweeper comment correctly identified the remaining gap, and the relevant code is still present at the current main SHA.

## Best Possible Solution

Keep this open and fix the OpenAI-compatible streaming/tool-call boundary. The best fix should either detect cumulative or repeated complete argument payloads before appending, or deterministically repair duplicated balanced JSON objects without allowing unsafe trailing content. It should also strip scratch `partialArgs` from completed tool-call blocks and add a focused regression test for isolated `agentTurn` cron using an OpenRouter/Kimi-shaped stream that emits the same complete arguments twice.
