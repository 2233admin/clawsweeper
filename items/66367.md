## Summary

Keep this PR open. Current main still blocks `qwen3.6-plus` on Qwen Coding Plan endpoints through the catalog builder, config normalization, runtime suppression, tests, and docs. The PR remains a relevant broader implementation candidate; related maintainer PR #72664 is narrower and explicitly keeps built-in catalog suppression unless maintainers choose the broader provider-truth policy.

## Best Possible Solution

Keep this PR open for maintainer product review of the broader provider-truth approach. If maintainers want `qwen3.6-plus` advertised on Coding Plan endpoints, rebase this PR or land an equivalent maintained patch covering `extensions/qwen/models.ts`, `extensions/qwen/index.ts`, `extensions/qwen/provider-catalog.test.ts`, and `docs/providers/qwen.md`. If maintainers choose the narrower opt-in policy in #72664, close this PR only after that policy lands or is explicitly accepted as canonical.
