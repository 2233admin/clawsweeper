## Summary

Keep open. Current `main` still uses a single-attempt `withCdpSocket` path for remote CDP operations, tests still expect mid-call socket closes to fail with `CDP socket closed`, and the only exposed remote-CDP knobs are timeouts rather than reconnect/backoff controls. The issue is also recent and already has a linked open PR (#67748), so there is not strong evidence for an auto-close.
