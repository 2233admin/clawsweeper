## Summary

Keep #40205 open. Current main still uses Ollama's deprecated `/api/embeddings` contract in the active bundled Ollama embedding provider, while the PR targets the correct `/api/embed` API shape but is stale because the implementation moved from `src/memory` to `extensions/ollama`.

## Best Possible Solution

Keep this PR open and ask for an update against `extensions/ollama/src/embedding-provider.ts` and `extensions/ollama/src/embedding-provider.test.ts`: use `POST /api/embed`, send `{ model, input }`, parse the first vector from `embeddings`, update assertions away from `prompt`/`embedding`, and preserve the current serialized batch behavior for local Ollama stability. Then run the relevant Ollama plugin changed checks before merge.
