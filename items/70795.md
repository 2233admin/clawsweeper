## Summary

Keep #70795 open. The current source still has a plausible policy mismatch: lower-level host read behavior is covered, but the effective direct-session read path and approved non-workspace roots are not proven fixed on main.

## Best Possible Solution

Keep the issue open. The best path is to trace the direct-session read tool host, add a regression that reads an approved non-workspace root with `tools.fs.workspaceOnly=false`, then thread one canonical effective FS policy/resolver through read, sandbox bridge, and extension bridge. If sandboxed direct sessions intentionally cannot read arbitrary host roots, document that explicitly and align the error with the effective policy.
