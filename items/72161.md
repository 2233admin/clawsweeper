## Summary

Current main still contains the Brave Search legacy docs URL in both Brave provider metadata entry points, and it still has both the legacy `docs/brave-search.md` page and the canonical `docs/tools/brave-search.md` page. PR #72161 remains a focused fix for the linked #65870 report rather than something to close as implemented or superseded.

## Best Possible Solution

Keep #72161 open for normal maintainer review. The best path is to land a small docs/provider metadata fix that updates both Brave `docsUrl` fields to the canonical `https://docs.openclaw.ai/tools/brave-search` URL and removes or stubs the duplicated legacy docs page while preserving the existing docs redirect for old links.
