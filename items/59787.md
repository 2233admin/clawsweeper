## Summary

Keep PR #59787 open. Current main still has the target-only same-channel suppression behavior that #59743 reports, so the PR is not obsolete or implemented elsewhere. The PR head is also not safe to close as implemented because review discussion identifies an unresolved bug: `hasText` is set unconditionally for extracted targets, so media-only sends would still suppress the following text reply.

## Best Possible Solution

Keep the PR open and ask for the minimal correction already suggested in review: derive `hasText` from the same `content`/`message` string check, keep `pendingMessagingTexts` tracking outside the `sendTarget` guard so `sessions_send` text dedupe still works, and add regression coverage for media-only same-target sends preserving the final text reply while text-bearing same-target sends still suppress duplicates.
