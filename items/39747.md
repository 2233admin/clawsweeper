## Summary

Keep #39747 open. Current main still has the heartbeat plugin-target load-order gap: explicit heartbeat targets are normalized before any cfg-backed plugin channel bootstrap can make plugin-only channel ids deliverable. The PR is not obsolete, though its exact implementation still needs maintainer review and regression coverage.

## Best Possible Solution

Keep the PR open or replace it with a focused successor. The fix should bootstrap or read plugin channel metadata before deliverable-channel recognition for explicit heartbeat targets, retry normalization after registry initialization, and add regression coverage for agents.defaults.heartbeat.target pointing at a plugin-only channel such as BlueBubbles or a custom fixture with an initially empty active registry.
