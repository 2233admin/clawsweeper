## Summary

Current main still does not implement the requested public session lifecycle feature set. OpenClaw has useful primitives such as `sessions.abort`, `sessions.delete`, `sessions.reset`, `sessions.compact`, usage display, cleanup, and a chat Stop button, but the requested `openclaw sessions stop/kill/pause/resume` commands, Sessions-page Stop/Pause/Resume controls, and real pause/resume/resource-limit enforcement remain missing.

## Best Possible Solution

Keep this issue open as the broad session lifecycle tracker. Add a narrow CLI stop command first, likely backed by existing `sessions.abort`, coordinate delete/rm/clear cleanup with #35735, and only add pause/resume/resource limits when Gateway protocol, runtime state, and harness/provider behavior can enforce them. Sessions UI should expose only controls backed by real Gateway behavior.

## Risks / Open Questions

- Closing now would hide a real public CLI and Sessions UI lifecycle gap that current lower-level Gateway primitives do not fully cover.
- Implementing pause/resume as only session metadata would be misleading unless active harnesses/providers can actually enforce paused and resumed execution states.
- The narrower deletion/clear work in #35735 should stay coordinated there and should not be treated as completion of this broader lifecycle request.
