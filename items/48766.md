## Summary

Keep open. Current `main` still appears to have the false-negative restart health path described in #48766: the restart waiter treats only a narrow set of 1008 auth-close reasons as proof of life, while the gateway still emits `device identity required` for the rejected probe path. The probe also still identifies itself as `mode: probe` with `clientVersion: "dev"`, matching the report.

## Evidence

- **Restart health probe still rejects `device-required` as unhealthy:** `looksLikeAuthClose()` only accepts 1008 close reasons containing `auth`, `token`, `password`, `scope`, or `role`. `confirmGatewayReachable()` returns healthy only for `probe.ok` or those accepted auth closes, so a `device identity required` close still fails the restart health check.
