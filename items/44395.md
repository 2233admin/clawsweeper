## Summary

Keep #44395 open. Current main does not implement heading-aware memory-search chunking: the active memory indexer still passes only `{tokens, overlap}` into `chunkMarkdown`, and `chunkMarkdown` still accumulates/splits by estimated character budget rather than Markdown headings. Config/schema/docs likewise expose only token and overlap chunking knobs. Current main has adjacent dreaming/daily-ingestion work that keeps heading context for short-term promotion, but that is a separate path and does not solve the reported memory-search embedding chunk blending. The provided GitHub context shows prior heading-aware PRs (#45545 and #51913) were closed unmerged, while #49108 is related broader ingestion/entity-extraction work but not a clear canonical replacement for this focused issue.
