## Summary

Keep this issue open. Current main still indexes memory through fixed-size `chunkMarkdown(content, { tokens, overlap })` chunks, with no heading/section strategy in the runtime config, index metadata, or user docs. The related open PR #68019 is an implementation candidate, not shipped behavior, and entity/graph extraction remains a separate larger memory-wiki or memory-core design track.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review #68019 or a smaller focused PR that adds a documented `markdown-heading`/section-aware strategy in the memory-host SDK path used by memory-core, exposes it in config/schema/help/docs, persists the effective strategy in reindex metadata, and adds tests proving adjacent `##`/`###` sections do not blend into one embedding/FTS chunk. Track entity extraction and graph memory as a separate memory-core or memory-wiki design item unless maintainers intentionally keep this as the umbrella.
