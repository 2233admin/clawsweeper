## Summary

Keep #69364 open. Current main has useful partial pieces for multi-agent work, including per-agent file editing, tools/skills configuration, and session parent/child metadata, but the Control UI still exposes them as separate flat surfaces. It does not implement the requested cross-agent relationship view that joins delegation, dataflow/shared workspace artifacts, task sequencing, and live session status in one UI. The related #52803 overlaps in the broad Control UI multi-agent area, but it does not clearly supersede this focused graph/config/debugging proposal.

## Best Possible Solution

Keep #69364 open as a focused Control UI product/design issue. The best path is a staged core Control UI implementation: start with a read-only team relationship graph built from existing config/session/task metadata, reuse the current per-agent file/tools/skills editors as detail panels, and add only the minimal safe Gateway API needed to inspect shared-workspace/TASK relationships. If maintainers want a single canonical tracker, explicitly merge this scope into #52803 first rather than auto-closing it as a duplicate.
