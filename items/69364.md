## Summary

Keep #69364 open. Current main has useful partial pieces for multi-agent work, including per-agent file editing, tools/skills configuration, and session metadata, but the Control UI still exposes them as separate flat surfaces. It does not implement the requested cross-agent relationship view that joins delegation, shared-workspace/dataflow artifacts, task sequencing, and live session status in one UI. Related #52803 overlaps, but it is broader and does not clearly supersede this focused graph/config/debugging proposal.

## Best Possible Solution

Keep #69364 open as a focused Control UI product/design issue. The best path is a staged core Control UI implementation: start with a read-only team relationship graph built from existing agents/subagent/session/task metadata, reuse the current per-agent file/tools/skills editors as detail panels, and add only the minimal safe Gateway API needed to inspect shared-workspace/TASK relationships. If maintainers want a single canonical tracker, explicitly fold this scope into #52803 first.
