## Summary

Keep this issue open. Current main still routes `feishu_doc` write/append through `document.convert`, while the public Feishu setup docs and manual credential prompt still do not name the reporter-verified `docx:document.block:convert` scope. The related descendant-children normalization issue is also not fully fixed on the small descendant insert path.

## Best Possible Solution

Keep the issue open and make a focused Feishu plugin cleanup: add `docx:document.block:convert` and related document scopes to public Feishu setup guidance and manual credential help, explicitly mention republishing the Feishu app after scope changes, consider a clearer runtime diagnostic for `document.convert` permission failures, and fix `toDescendantBlock()` so normalized `children` cannot be overwritten by raw converted block data, with a narrow regression test.
