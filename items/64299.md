## Summary

Keep open. The reporter found and verified the immediate cause as a missing Feishu `docx:document.block:convert` permission, and current main mentions that scope in the Feishu document skill. But the user-facing Feishu setup/docs path still does not enumerate it, the manual setup help only lists chat/contact scopes, and the related docs PR was closed unmerged. Separately, the related descendant-children code PR was also closed unmerged and current main still has that small-document serialization risk, so there is not enough implemented-on-main proof for conservative auto-close.

## Best Possible Solution

Keep #64299 open until main either updates the user-facing Feishu setup/docs/manual permission guidance to explicitly include `docx:document.block:convert` for `feishu_doc write`/`append`, or a maintainer confirms the current automatic app-registration flow fully supersedes manual scope guidance. While touching this area, resolve or deliberately dismiss the unmerged descendant-children normalization concern from #64331 with a focused regression test.
