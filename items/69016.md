## Summary

Keep open. Current main has a partial CLI-backend mitigation: CLI turns are now persisted to the OpenClaw transcript, CLI usage can refresh session token counts, and an over-budget CLI transcript can call the active context engine's compact() path. But the reported lifecycle gap is not fully implemented: the CLI runner still does not run the context-engine assemble() path before invoking the external CLI, and it does not run afterTurn()/ingest after every CLI turn. The linked lossless-claw companion issues remain relevant because per-turn indexing/proactive compaction is still not equivalent to the budget-triggered compact-only path.

## Best Possible Solution

Keep the issue open and narrow it around the remaining lifecycle contract. Preserve the current CLI transcript persistence and budget-triggered compact() path, then add one of two explicit product paths: either wire CLI turns into context-engine afterTurn()/ingest with runtimeContext.currentTokenCount from parsed CLI usage or transcript estimates, and document assemble limitations for resumed external sessions; or add a visible startup/status/docs warning that context-engine assemble/afterTurn parity is not supported for CLI-owned history. If full assemble support is desired, it likely belongs in src/agents/cli-runner/prepare.ts by projecting assembled transcript context into fresh/reseed prompts and in the post-run path by finalizing against the persisted transcript snapshot.
