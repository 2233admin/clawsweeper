## Summary

Current main still does not implement PR #64027's central behavior: Feishu card-action envelopes have no `uc` update-card payload, the card-action handler never calls `updateCardFeishu`, and the card-action event parser does not preserve the callback `message_id` needed to patch the original card. The related #64828 issue tracks the same feature request, but it is open and not a shipped or merged replacement for this PR.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a narrower Feishu-card-update patch: add `uc`/`updateCard` support, preserve the card callback `message_id`, invoke `updateCardFeishu` with isolated error handling in the relevant cancel/confirm/quick paths, add focused tests for successful update, malformed `uc`, missing `message_id`, and update failure, and split or remove unrelated reply-dispatcher/test churn. Track the feature against #64828 and correct the unrelated #54023 close reference.
