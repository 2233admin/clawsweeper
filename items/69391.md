## Summary

Keep PR #69391 open. Current main still tells the bundled coding-agent skill to use `bypassPermissions` and `--yolo`, but it does not include the proposed destructive-spawn recovery guidance. The PR is a focused docs change for a bundled safety-sensitive workflow, though it needs maintainer/editorial review because the proposed example promotes the author's third-party tool and has an unresolved reviewer concern about guaranteeing a baseline snapshot before the agent runs.

## Best Possible Solution

Keep the PR open for maintainer docs review.
