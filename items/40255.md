## Summary

Current main still mutates user-configured heartbeat prompts that mention HEARTBEAT.md, while the public docs promise custom heartbeat prompts are sent verbatim. PR #40756 remains open and unmerged with closing syntax for this regression, so this issue should stay open until that PR or an equivalent maintainer patch lands.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing PR #40756 or an equivalent maintainer patch. The fix should detect whether the plain heartbeat prompt came from user config, skip `appendHeartbeatWorkspacePathHint` for user-authored configured prompts, preserve the hint for built-in/default and internally generated exec or cron prompts, and add a focused regression test for a custom prompt that mentions `HEARTBEAT.md`.
