## Summary

Current main still reproduces the reported regression: configured heartbeat prompt text is resolved first, then `resolveHeartbeatRunPrompt` unconditionally passes the resulting prompt through `appendHeartbeatWorkspacePathHint`, which mutates any custom prompt that mentions `heartbeat.md` before it is sent as the heartbeat body. The public docs still promise custom heartbeat bodies are sent verbatim, and the nearby test coverage only asserts default prompt hinting, not the custom-prompt guard. Related PR #40756 remains the active open fix attempt; earlier duplicate PRs were closed unmerged.

## Best Possible Solution

Keep #40255 open and use the active fix path from PR #40756 or an equivalent patch: teach `resolveHeartbeatRunPrompt` whether the base prompt came from user config versus built-in/internal prompt generation, skip `appendHeartbeatWorkspacePathHint` for configured `heartbeat.prompt` text, and add a regression test where a custom prompt containing `HEARTBEAT.md` reaches `getReplyFromConfig` unchanged while the default prompt still receives the workspace path hint.
