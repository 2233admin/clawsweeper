## Summary

Keep open. Current `main` still builds the webchat session dropdown from `state.sessionsResult` only, while `sessions.list` still enumerates entries from the session store. I did not find code on `main` that merges in configured agents from `agents.list`, so the reported gap still exists.

## Evidence

- **Chat dropdown source:** `renderChatSessionSelect()` derives options from `resolveSessionOptionGroups(state, state.sessionKey, state.sessionsResult)`, so the dropdown is driven by the loaded session list rather than configured agents.
  - file: [ui/src/ui/chat/session-controls.ts:23](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/ui/chat/session-controls.ts#L23)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '20,45p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Option builder only adds session rows/current key:** `resolveSessionOptionGroups()` iterates `sessions?.sessions ?? []` and adds those keys plus the current session key. It does not append missing configured agents. `state.agentsList` is only consulted later to label an already-parsed agent group.
  - file: [ui/src/ui/chat/session-controls.ts:447](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/ui/chat/session-controls.ts#L447)

- **agents.list fetched separately but not merged into dropdown:** `loadAgents()` requests `agents.list` and stores it on `state.agentsList`, but the dropdown code above does not use that list to create selectable session entries.
  - file: [ui/src/ui/controllers/agents.ts:60](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/ui/controllers/agents.ts#L60)
