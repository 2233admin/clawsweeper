## Summary

Keep this PR open. Current main still contains the targeted Anthropic `options.thinkingBudgetTokens || 1024` fallback, and the zero-budget path is still reachable for non-adaptive reasoning models. The PR is a narrow implementation candidate, but it should add focused regression tests before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. Add focused tests in `src/agents/anthropic-transport-stream.test.ts` proving that an explicit `thinkingBudgetTokens: 0` omits the legacy thinking block and that an undefined budget still uses the existing `1024` default, then let maintainers decide whether to merge this narrow guard or fold it into a broader Anthropic reasoning cleanup.
