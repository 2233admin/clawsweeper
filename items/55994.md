## Summary

Keep open. Current `main` and the latest shipped release still enforce a generic `message.send` poll-intent guard in core outbound routing, and repo tests explicitly assert that non-Telegram channels are rejected when poll-shaped params are present. The WeChat docs also place `openclaw-weixin` on the external plugin contract, which means this issue still points at a real core/plugin seam rather than a ClawHub-only feature request. The issue timeline already has two open fix PR cross-references, so there is not strong evidence for conservative auto-close.
