## Summary

Current main still does not resolve the reported Telegram reminder-note leak. The core reminder guard can still append the exact contradictory note when `successfulCronAdds` is zero, reminder success telemetry is still limited to direct `cron` add actions, and the Telegram outbound adapter still forwards direct text, media text, and payload fallback text without a specific suppressor for this note.

## Best Possible Solution

Keep this issue open. The best fix is a narrow Telegram plugin change that suppresses only the exact false unscheduled-reminder note variants across direct text, media, and payload fallback sends, or a broader telemetry fix that reliably marks every successful reminder creation path before the guard runs. Add focused regression tests for Telegram direct text, media, interactive/payload fallback, direct `cron.add`, and any supported helper-created reminder path.
