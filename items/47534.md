## Summary

Keep open. Current `main` still persists `skillsSnapshot` in session entries, refreshes it only on snapshot-version/filter changes, and `doctor` state-integrity checks do not inspect cached `skillsSnapshot` or stale bundled-skill runtime roots. A related fix landed for `skills.*` config changes, but it does not cover runtime-root moves or the requested `doctor` diagnostic.
