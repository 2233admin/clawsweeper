## Summary

Keep open. Current main confirms the CLI is intentionally preview-only for implicit `devices approve` selection, and the main CLI docs already match that behavior, but bundled guidance is still stale on `main` (`skills/node-connect/SKILL.md` still tells operators to run `openclaw devices approve --latest` as if that completes approval). The issue is recent and already cross-linked to an open docs PR (#70667), so there is not strong evidence for an auto-close.
