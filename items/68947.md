## Summary

Keep open. Current main and the latest release still contain the broken `/export-session` HTML template, so the user-facing bug is not implemented on main. The PR is directionally useful, but it still needs revision because restoring the placeholders without changing the string replacement path can mutate inlined script bytes containing `$` replacement sequences.

## Best Possible Solution

Keep the PR open and ask for a small revision: retain the restored `{{MARKED_JS}}`, `{{HIGHLIGHT_JS}}`, and `{{JS}}` placeholders with Prettier guards, update `generateHtml()` to insert bundled script content with function replacers so `$&` and `$$` bytes are preserved literally, and keep the portable regression test reading the real template/assets. Once green, this PR can close the still-open #49957 regression.
