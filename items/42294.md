## Summary

Keep this PR open. The current main tree has moved Telegram outbound code into the Telegram plugin, and the proposed patch is stale against that layout, but the underlying bug is not fully solved on main: delivery now routes `audioAsVoice` payloads toward Telegram `sendPayload`, yet the Telegram payload sender still does not forward that flag as `asVoice`, which is the option the Telegram transport uses to choose `sendVoice` instead of `sendAudio`.

## Best Possible Solution

Keep the PR open, ask for or perform a rebase onto the current Telegram plugin layout, and carry the useful part forward by adding `asVoice: payload.audioAsVoice === true` to the current Telegram payload/send-media option path plus a focused regression in `extensions/telegram/src/outbound-adapter.test.ts` or equivalent delivery coverage. After that, run the Telegram extension/outbound targeted tests before merge.
