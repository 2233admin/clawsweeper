## Summary

Current main still has the exact transport-stream behavior this PR targets: `output.errorMessage` is populated for provider finish reasons, but the three finalization paths still throw `"An unknown error occurred"`, and the catch paths overwrite `output.errorMessage` with that generic string. The PR is a focused, plausible bug fix and should not be auto-closed.

## Best Possible Solution

Keep this PR open and either land or revise it against current main: propagate `output.errorMessage || "An unknown error occurred"` in the shared helper and both OpenAI Responses variants, add the proposed regression coverage, and explicitly verify that preserving provider messages does not unintentionally change retry/fallback behavior.
