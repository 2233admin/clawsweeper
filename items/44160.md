## Summary

Keep open. Current main still does not implement Mattermost WebSocket protocol ping/pong keepalive: the Mattermost WebSocket wrapper only exposes open/message/close/error plus send/close/terminate, and the monitor only runs the existing bot update_at health check. Related PR #57621 appears to track the exact fix but is still open, while prior PR #46345 was closed unmerged, so the reported bug is not implemented on main yet.
