## Summary

Keep #58310 open. Current main and latest release still write `identity/device.json` in place with `fs.writeFileSync`, so the requested atomic device-identity write behavior is not implemented or shipped. The PR is a plausible focused fix for core key-material durability, but it still needs maintainer review and likely should reuse or match the stronger existing atomic JSON writer pattern.

## Best Possible Solution

Keep the PR open or replace it with a maintainer-polished patch that makes device identity persistence atomic using the existing `saveJsonFile()`/atomic JSON writer contract or an equivalent sync helper, preserves 0o600 file mode, removes the misleading `EEXIST` handling, and adds focused tests for temp cleanup, destination preservation on write failure, and permissions.
