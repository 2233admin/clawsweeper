## Summary

Keep #58830 open. Current main already has partial transparency and recovery around compaction: opt-in start/end notices, lifecycle hooks, checkpoint metadata, and UI/API restore or branch from compaction checkpoints. The core requested capability is still missing: there is no preview-before-apply flow, no approve/reject path, and no `auto` / `notify-only` / `require-confirmation` compaction mode for automatic context management. Related ClawSweeper items cover narrower surfaces and do not supersede this broader user-visible confirmation/control request.

## Best Possible Solution

Keep the issue open as a core product/UX tracker. The best path is to build on the shipped `notifyUser`, compaction hooks, and checkpoint restore APIs by adding an explicit compaction policy with at least notify-only and require-confirmation behavior, a preview payload containing summary/token/message-impact metadata, and CLI/UI/API flows that can approve, reject, or branch/restore without breaking existing automatic recovery. Any custom summarization strategy should continue to use the context-engine or compaction-provider plugin seams.
