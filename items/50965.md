## Summary

Keep open. Current `main` still reports `ok: true` from `cron.remove` when no job was deleted, and the CLI `cron rm/remove/delete` path still prints the payload without converting a missing-job result into a non-zero exit. The proposed PR is not implemented on `main`, and the current code evidence matches the review comment that the user-visible silent-success path is still unresolved.

## Evidence
