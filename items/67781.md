## Summary

Keep open. Current `main` does not already contain this Vertex/empty-payload fix, and the submitted patch has unresolved P1 regression risk: it would replace the existing Google thinking `wrapStreamFn`, and it would turn several already-supported empty-final-payload success paths into user-facing errors.

## Evidence

- **Current Google provider already depends on a shared thinking stream wrapper:** `buildGoogleProvider()` spreads `GOOGLE_GEMINI_PROVIDER_HOOKS`, and that hooks object defines `wrapStreamFn: createGoogleThinkingStreamWrapper`. Current tests assert Google and Gemini CLI share that wrapper and preserve thinking config behavior, so the PR's later `wrapStreamFn` addition would override existing behavior rather than just add Vertex handling.

That means the PR's new hard-error branches would be a behavior change, not an implementation already present on `main`.
