## Summary

Keep open. Current main still does not implement the proposed gateway `doctor.memory.remHarness` RPC. The existing REM harness is CLI-only, while this PR remains a focused implementation candidate for remote read-only gateway diagnostics and still needs normal maintainer review, especially around the memory data exposed under `operator.read`.

## Best Possible Solution

Keep this PR open for normal maintainer review. If remote read-only REM diagnostics are desired, land a gateway implementation that registers `doctor.memory.remHarness`, scopes it under `operator.read`, routes through memory-core public/facade exports, preserves CLI/REM lookback parity, bounds response size and work, and includes targeted gateway and facade tests plus the changed gate/build. If maintainers want REM previewing to remain CLI-only, close this PR only with that explicit product decision.
