## Summary

Keep open. Current `main` still has the reported gap: Google Chat `app-url` auth compares `channels.googlechat.appPrincipal` against the JWT `sub`, but the user docs/config examples do not explain that requirement, webhook auth failures still collapse to a generic 401 without surfacing the rejection reason, and channel status/probe does not validate this binding.

## Evidence

- **Auth still binds `appPrincipal` to JWT `sub`:** `verifyGoogleChatRequest()` lowercases `expectedAddOnPrincipal`, reads `payload.sub`, and rejects with `missing add-on principal binding` or `unexpected add-on principal: ...` when they do not match.
  - file: [extensions/googlechat/src/auth.ts:169](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/googlechat/src/auth.ts#L169)
  - command: `nl -ba extensions/googlechat/src/auth.ts | sed -n '135,185p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Webhook path still drops the reason and returns only unauthorized:** `isAuthorizedGoogleChatTarget()` calls `verifyGoogleChatRequest()` but returns only `verification.ok`; the handler then uses `resolveWebhookTargetWithAuthOrReject(...)`, so failed auth becomes a bare 401 with no surfaced rejection reason.
  - file: [extensions/googlechat/src/monitor-webhook.ts:104](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/googlechat/src/monitor-webhook.ts#L104)

- **User docs still omit the `appPrincipal = JWT sub` requirement:** The Google Chat docs explain `audienceType`/`audience` and show a config example, but do not mention `appPrincipal` at all in the setup or config notes.
  - file: [docs/channels/googlechat.md:139](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/docs/channels/googlechat.md#L139)
