## Summary

Current `main` does not include the Telegram `ackReactionTiming` feature from PR #46258. The shared and Telegram config types/schema/docs still expose only `ackReaction`/`ackReactionScope`, and the Telegram dispatcher still starts status reactions immediately instead of supporting a deferred `run-start` lifecycle. This PR is therefore not implemented on `main`, and there is not enough basis to close it conservatively.
