## Summary

Keep open. Current main still routes the affected Qwen/legacy ModelStudio models through the generic OpenAI-compatible stream path, and that path still only promotes structured `delta.tool_calls` into tool calls. Kimi-style marker sections inside reasoning/thinking text are only handled by the Kimi provider wrapper, not by Qwen/ModelStudio, so the reported premature completion path remains plausible and unimplemented.

## Best Possible Solution

Keep this issue open and fix it in provider-owned stream normalization. The best path is a Qwen/ModelStudio wrapper or shared OpenAI-compatible compat seam that detects validated Kimi-style tagged tool-call sections in reasoning/thinking content, converts them to structured `toolCall` blocks, and normalizes `stopReason` after extraction succeeds. Add focused regression coverage for `qwen/kimi-k2.5`, `qwen/MiniMax-M2.5`, and the existing Kimi wrapper so text-block and thinking-block behavior are explicit.
