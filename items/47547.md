## Summary

Current main already implements WhatsApp env-proxy support for the Baileys socket path and its upload fetch path, and the same behavior is present in the latest shipped release `v2026.4.22`. The issue's requested behavior is therefore already implemented.

## Evidence

That addresses the reported direct-DNS WebSocket path.

## Close Comment

Closing as implemented on main after Codex review. WhatsApp socket creation now resolves an ambient proxy agent from standard proxy env vars and passes it into Baileys, along with a proxy-aware upload dispatcher (`extensions/whatsapp/src/session.ts:147`, `extensions/whatsapp/src/session.ts:210`). If this still reproduces on `v2026.4.22` or newer, please open a fresh issue with the exact proxy env configuration and current logs so maintainers can check for a remaining proxy-specific regression.
