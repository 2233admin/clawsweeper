## Summary

Current main still does not implement the requested Feishu card-to-plain-text fallback for table-limit card failures. The card delivery path sends structured cards without a catch/retry, and markdown table conversion remains applied only to text/post paths, not card content. The linked fix PR #50429 is relevant evidence, but it is an open proposed fix rather than shipped behavior, so the original bug should stay open.
