## Summary

Current main already provides a live running indicator in the chat UI: sending sets an active run immediately, empty active streams render a reading indicator, and non-empty active streams render a visibly animated streaming bubble. The same behavior is present in the shipped v2026.4.23 release.

## Evidence

- **Run state starts immediately on send:** Chat send sets `chatRunId`, initializes `chatStream` to an empty string, and records `chatStreamStartedAt`, which gives the UI an active-running state before any text arrives.
  - file: [ui/src/ui/controllers/chat.ts:334](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/ui/src/ui/controllers/chat.ts#L334)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Empty active stream renders a reading indicator:** When `props.stream !== null` but has no text yet, the chat item builder emits a `reading-indicator` item instead of showing nothing.
  - file: [ui/src/ui/chat/build-chat-items.ts:291](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/ui/src/ui/chat/build-chat-items.ts#L291)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Reading indicator is visible UI:** The renderer turns that item into an assistant bubble with animated dots, providing a clear in-progress indicator before streamed text appears.
  - file: [ui/src/ui/chat/grouped-render.ts:214](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/ui/src/ui/chat/grouped-render.ts#L214)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Streaming response is visually marked as active:** Streaming assistant bubbles use the `streaming` class, which applies the `chatStreamPulse` animation so active execution remains visible while text is arriving.
  - file: [ui/src/styles/components.css:2627](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/ui/src/styles/components.css#L2627)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)

## Close Comment

Closing from a Codex review: current `main` already implements a running status indicator in the chat UI. Sending a message immediately creates an active run and initializes streaming state in `ui/src/ui/controllers/chat.ts:334`; while that active stream is still empty, the UI renders a reading indicator in `ui/src/ui/chat/build-chat-items.ts:291`, and that indicator is displayed as an assistant bubble with animated dots in `ui/src/ui/chat/grouped-render.ts:214`. Once text starts arriving, the active assistant bubble uses the `streaming` animation defined in `ui/src/styles/components.css:2627`. This behavior is also present in the shipped `v2026.4.23` release (`9b18ae877018a34933e27bb7b2986cb97e9602aa`), so there is no remaining maintainer action on this issue.
