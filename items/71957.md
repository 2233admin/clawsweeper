## Summary

Keep open. Current main appears to cover the explicit `agentRuntime.id` path for CLI backend owners, but the reported reproduction is a legacy `claude-cli/...` primary model path and docs still promise those legacy refs work. Startup plugin selection still derives required harness runtimes from `agentRuntime`/agent overrides/env, not from legacy model refs, so there is not enough evidence that `anthropic` or `google` will be loaded before channel startup for the reported config.

## Best Possible Solution

Keep the issue open and implement a startup-planning fix that loads CLI backend owner plugins implied by configured legacy CLI model refs, or runs the existing legacy-ref migration before resolving startup plugin ids. Add regression coverage for `claude-cli/*` and `google-gemini-cli/*` model-ref-only configs in `src/plugins/channel-plugin-ids.test.ts`, plus a gateway startup ordering test proving owner plugin registration happens before `startChannels`.
