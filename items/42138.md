## Summary

Keep open. Current `main` already has separate auth-profile ordering, cooldown-based profile rotation, and ordered model fallbacks, but it does not implement the requested single CLI-first pool of ordered provider/profile/model entries with explicit enable/disable management.

## Evidence

- **CLI docs expose separate auth and fallback primitives, not a pooled route manager:** `openclaw models` documents `models fallbacks` and `models auth ...`, but there is no `auth-pool` surface or combined provider/profile/model entry management.
  - file: [docs/cli/models.md:101](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/cli/models.md#L101)

- **Credential ordering is provider-scoped only:** The auth docs describe only per-agent `models auth order get|set|clear --provider ...`, which orders profile ids for one provider. That is narrower than the issue's ordered cross-provider route pool.
  - file: [docs/gateway/authentication.md:175](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/gateway/authentication.md#L175)
