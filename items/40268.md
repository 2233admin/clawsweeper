## Summary

Current main does not implement the requested tolerant read path. The root config schema is still strict, normal load and snapshot reads still validate through `validateConfigObjectWithPlugins`, startup/hot reload recover via last-known-good rather than stripping arbitrary unknown keys, and the provided context shows #42201 is still an open implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open. The best path is to review and either land or close #42201; if landed, this can close as implemented with regression coverage for startup and hot reload accepting unknown keys while strict write/config-validate behavior and invalid known-field failures remain intact. If #42201 is abandoned, implement the same behavior directly in the config read/snapshot path with focused tests for the root and nested misplaced-key examples from this issue. #40317 overlaps as a broader tracker, but this issue has concrete reproduction detail and is the better canonical bug until the focused fix is resolved.
