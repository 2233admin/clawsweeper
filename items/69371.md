## Summary

Keep open. Current main has adjacent fixes and mitigations, including production handshake timeout support, cron retry/backoff, Pi startup guidance, and some delivery queue recovery, but the central ARM64 edge-device request is not fully implemented. Sessions/status still do not expose the requested lightweight/compact ARM path, compaction config lacks the requested context high-water and interval controls, and `openclaw message send` has no `--background`/`--retry` CLI surface. The related closed issues #45440 and #46097 cover narrower prior problems, not this broader recent stability report.

## Best Possible Solution

Keep #69371 open, or split it into targeted maintainer-owned issues: fast/lightweight sessions and status paths for large stores on ARM64, configurable context high-water compaction policy if desired, durable/background retry for CLI message sends, and an edge-device doctor preflight. Treat current cron retry and handshake timeout work as partial prior progress, not a full resolution.
