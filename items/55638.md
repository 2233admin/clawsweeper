## Summary

Keep open. Current main still treats non-404 ClawHub install failures like the reported `423` as hard-stop raw errors, with no special retry/progress handling, and there are no post-issue commits touching that path. The impact is narrower now because Microsoft Teams is documented as bundled in current packaged releases, but manual/custom installs still go through the same unchanged ClawHub path.

## Evidence

- **Non-404 ClawHub errors are still passed through unchanged:** `mapClawHubRequestError()` only special-cases 404s; every other ClawHub error becomes a generic install failure via `formatErrorMessage(error)`. The archive download path also returns `formatErrorMessage(error)` directly, so a `423 pending VirusTotal scan` response would still surface as a raw hard failure.
