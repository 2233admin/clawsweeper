## Summary

Keep open. Current `main` still unconditionally prefixes ACP client server args with `"acp"`, so the duplicated-subcommand behavior described by the PR is not implemented on `main`. The PR's proposed fix looks directionally correct, but the existing review comment is substantive: the added test targets `resolveAcpClientSpawnInvocation`, while the behavior change lives in `buildServerArgs`, so the regression is still unguarded.
