## Summary

Current `main` still contains the run-id mismatch that can leave the Control UI busy indicator stuck after a subagent-triggered follow-up turn finishes. I did not find a source-level fix strong enough to close this issue.

## Evidence

- **Initial chat tracking only records the original client run id:** `context.addChatRun(idem, { sessionKey: canonicalSessionKey, clientRunId: idem })` records the top-level chat run under the original idempotency key; I did not find corresponding logic here to remap later follow-up turns spawned after `sessions_spawn`/`sessions_yield`.
  - file: [src/gateway/server-methods/agent.ts:732](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/gateway/server-methods/agent.ts#L732)
  - command: `nl -ba src/gateway/server-methods/agent.ts | sed -n '732,741p'`
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Terminal lifecycle handler still drops or miskeys final chat events:** In `finalizeLifecycleEvent`, the gateway looks up chat state by `evt.runId`. If `chatLink` exists but `shift(evt.runId)` returns nothing, it clears context and returns without `emitChatFinal`. If `chatLink` does not exist, it emits the final event with `eventRunId = evt.runId` instead of the original tracked client run id.
  - file: [src/gateway/server-chat.ts:587](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/gateway/server-chat.ts#L587)
  - command: `nl -ba src/gateway/server-chat.ts | sed -n '587,633p'`
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **UI only clears the busy state for matching run ids:** `handleChatEvent` treats a `final` from a different `payload.runId` as history-only and returns without clearing `state.chatRunId`. The busy indicator is only reset in the matching-run branch that sets `state.chatRunId = null`.
  - file: [ui/src/ui/controllers/chat.ts:439](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/ui/src/ui/controllers/chat.ts#L439)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '437,472p'`
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
