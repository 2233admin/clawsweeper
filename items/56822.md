## Summary

Current `main` does not implement `sessions_await` or the proposed await-only `sessions_spawn` flags, and the code/docs now point to a different shipped flow: push-based subagent completion plus `sessions_yield`. Given that gap and the still-relevant PR review concerns, this PR is not a safe auto-close candidate.

## Evidence

- **Core tool set on main has no `sessions_await`:** `createOpenClawTools` registers `sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, and `sessions_yield` in the session-tool block, but no `sessions_await` tool is added on current `main`. The shipped tool catalog likewise lists `sessions_spawn` and `sessions_yield`, with no await tool entry.
  - file: [src/agents/openclaw-tools.ts:273](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/openclaw-tools.ts#L273)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **`sessions_spawn` schema on main lacks the PR surface:** The current `sessions_spawn` schema exposes `task`, `label`, `runtime`, `agentId`, `resumeSessionId`, `model`, `thinking`, `cwd`, timeout fields, `thread`, `mode`, `cleanup`, `sandbox`, `context`, `streamTo`, `lightContext`, and attachments. It does not expose `waitForCompletion` or `suppressAnnounce`.
  - file: [src/agents/tools/sessions-spawn-tool.ts:100](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/tools/sessions-spawn-tool.ts#L100)

  - file: [docs/concepts/session-tool.md:15](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/concepts/session-tool.md#L15)
