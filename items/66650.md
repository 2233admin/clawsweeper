## Summary

Keep this PR open. Current main still has the `sessions_history`/`chat.history` limit mismatch described by #66573: the tool accepts and forwards oversized limits, while the gateway validates `chat.history.limit` with a hard maximum of 1000 before the handler clamp runs. The PR is a narrow implementation candidate, not obsolete or superseded.

## Best Possible Solution

Keep this PR open as the canonical narrow fix candidate for #66573. The maintainer path is to clamp `sessions_history` outgoing limits to the gateway's 1000-message contract, expose `maximum: 1000` in the tool schema, and add an execute-level runtime-clamp regression test before merge. Any decision to raise the `chat.history` protocol limit should be handled separately as a gateway/API contract change.
