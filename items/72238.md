## Summary

Keep #72238 open. Current main still has the browser plugin resolving `playwright-core` through static runtime imports, no browser-local `playwright-core.runtime.ts` shim or equivalent landed, and the linked bug #72168 remains the same observable packaged-build failure. The PR is an active, focused fix candidate with maintainer Docker/gateway-first-start proof, but maintainer discussion asks for packaged-topology regression coverage before merge.

## Best Possible Solution

Keep #72238 open. The best path is to finish the maintainer-requested packaged-topology regression coverage, then merge this PR or an equivalent patch that routes browser Playwright runtime values through a local `createRequire(import.meta.url)` shim while preserving type-only `playwright-core` imports.
