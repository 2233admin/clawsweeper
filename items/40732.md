## Summary

Keep open. Current main still uses the native textarea for raw JSON/JSON5 config editing, with no CodeMirror component, status bar, breadcrumb path resolver, or matching browser tests. Related #40009 and #56401 overlap, but both are open and neither has merged or clearly superseded this work.

## Best Possible Solution

Keep this PR open for maintainer review or explicitly consolidate it with the canonical raw config editor effort tracked by #40009 and overlapping #56401. The best path is one selected UI implementation in ui/src/ui/views/config.ts with focused browser coverage for raw save/change behavior, sensitive redaction/reveal behavior, CodeMirror dependency impact, status text, breadcrumbs, JSON5 edge cases, formatting/validation expectations, and regenerated locale output.

## Risks / Open Questions

- The PR edits generated non-English locale bundles; it should be rebased and regenerated through the Control UI i18n pipeline, especially because current main has more locales than the PR touched.
- There is overlapping open raw-editor work in #40009 and #56401, so maintainers should choose one canonical implementation path before landing conflicting UI/editor dependencies.
