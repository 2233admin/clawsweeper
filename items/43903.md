## Summary

Keep open. Current main still has a single shared `gateway.auth.token` path; token auth does not map tokens to labels/scopes or propagate a per-token user identity for Control UI sessions. The issue discussion is concrete and reinforced by the comment describing a real multi-user deployment workaround, so this remains actionable core gateway/auth work rather than cleanup.

## Evidence

- **Issue discussion confirms the remaining use case:** #43903 asks for a `gateway.auth.tokens` map with per-token labels/scopes so multiple Control UI users can have isolated sessions. The follow-up comment reports the same problem in a 17-agent, 9-member deployment and says shared Control UI auth cannot identify who is at the keyboard.
- **Gateway auth config type only has one shared token:** `GatewayAuthConfig` exposes `mode`, one `token`, one `password`, Tailscale/rate-limit settings, and `trustedProxy`; there is no `tokens` map or token identity shape.
