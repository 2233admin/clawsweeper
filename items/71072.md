## Summary

Keep open. Current `main` preserves explicit empty plugin scopes functionally, but it does not implement the PR's O(1) empty-scope fast path, so there is no strong auto-close basis such as `implemented_on_main` or `cannot_reproduce`.

## Evidence

- **Explicit empty scope is distinct from unscoped:** `createPluginIdScopeSet` returns `null` only for `undefined`; an explicit `[]` becomes an empty `Set`, so the loader can distinguish scoped-empty from full-load requests.
  - file: [src/plugins/plugin-scope.ts:25](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/plugins/plugin-scope.ts#L25)
  - command: `nl -ba src/plugins/plugin-scope.ts | sed -n '1,80p'`
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **`main` still does discovery and manifest parsing for scoped-empty loads:** In `loadOpenClawPlugins`, `onlyPluginIdSet` is computed, but there is no early return for `size === 0`; the code proceeds to cache/reentry handling and then unconditionally calls `discoverOpenClawPlugins` and `loadPluginManifestRegistry`.
  - file: [src/plugins/loader.ts:1984](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/plugins/loader.ts#L1984)

- **Current tests lock behavior, not the performance optimization:** `loader.test.ts` asserts that `onlyPluginIds: []` yields an empty registry, but it does not assert that discovery, manifest parsing, or cache lookups are skipped.
  - file: [src/plugins/loader.test.ts:2621](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/plugins/loader.test.ts#L2621)
