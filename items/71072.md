## Summary

Keep open. Current main has the explicit empty-scope semantics from the related activation/scope work, but it does not implement the PR's O(1) empty plugin-load fast path: `loadOpenClawPlugins` still enters cache/load state, constructs registry/runtime scaffolding, then calls discovery and manifest registry loading before filtering to zero plugins.

## Best Possible Solution

Keep the PR open. The best path is to rebase it onto current main, implement the empty-scope fast path inside `loadOpenClawPlugins` with `PluginLoaderCacheState` reentry semantics preserved, and add a focused unit test proving `onlyPluginIds: []` returns an empty registry without calling discovery or manifest loading while `undefined` still performs a normal full load.
