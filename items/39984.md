## Summary

Keep open. Current main still lacks the requested `web_fetch` / browser fallback guidance in runtime `web_search` missing-key messages, and the PR's original core edit target is now obsolete because web search execution delegates to provider-owned runtimes.

## Best Possible Solution

Keep this PR open and ask for a refresh against the provider-owned web search runtimes. Update the relevant `extensions/*` missing-key messages, preserve real env var names, avoid baseline churn, and add focused message coverage where practical. The docs page at https://docs.openclaw.ai/tools/web already supports the product direction, but current runtime behavior is still missing the requested guidance.
