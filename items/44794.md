## Summary

Keep open. Current main still has no `gateway.preStartCommand` or `gateway.wrapperScript` config surface, and the gateway service installers still render fixed launchd/systemd entrypoints from generated program arguments. This is core gateway service lifecycle/setup behavior, not a ClawHub plugin candidate, because a plugin cannot run before the Gateway process starts.
