## Summary

Keep #70043 open. Current main still makes unset Telegram `reactionLevel` resolve to agent reaction guidance `minimal`, and the normal embedded-runner prompt path injects `Reactions are enabled for Telegram in MINIMAL mode.` into the system prompt. That matches the reported user-visible behavior rather than proving it fixed or obsolete.

## Best Possible Solution

Keep the issue open for a maintainer/product fix. The likely path is to make Telegram reaction prompt guidance stable for existing sessions by either changing the unset default/prompt injection behavior, gating prompt-affecting reaction guidance behind explicit `reactionLevel`, or adding a migration/doctor/config explanation that makes the effective default explicit. Align docs, config type comments, schema metadata, and tests once the intended behavior is chosen.
