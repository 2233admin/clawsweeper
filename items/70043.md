## Summary

Keep #70043 open. Current main still defaults an unset Telegram `reactionLevel` to `minimal`, feeds that effective default into Telegram agent prompt guidance, and injects the exact `Reactions are enabled for Telegram in MINIMAL mode.` text into the system prompt. The provided related items cover other Telegram/session behaviors and do not supersede this reaction-default prompt issue.

## Best Possible Solution

Keep the issue open. Maintainers should decide whether unset Telegram `reactionLevel` means legacy/no prompt guidance or explicit default-minimal. If existing sessions should not receive new reaction guidance without opt-in, distinguish missing from explicit `minimal` in the Telegram resolver or `agentPrompt.reactionGuidance`, avoid injecting `## Reactions` for legacy unset configs, and align docs, type comments, generated metadata, tests, changelog, and setup/doctor behavior around that contract.
