## Summary

Keep #70043 open. Current main still defaults unset Telegram `reactionLevel` to `minimal`, exposes that as agent reaction guidance, and injects the exact `Reactions are enabled for Telegram in MINIMAL mode.` prompt text. The issue is recent, author is not protected, labels are not protected, and the provided related items are only broad title-search overlaps rather than a canonical duplicate or shipped fix.

## Best Possible Solution

Keep the issue open and decide the product behavior explicitly. A focused fix would likely live in the Telegram plugin prompt/reaction-level path: either suppress prompt-affecting reaction guidance for implicit defaults on existing sessions, persist/announce the effective default during setup, or add migration/session-state logic that makes default introduction stable and visible. Add regression coverage around an unset Telegram `reactionLevel` in a GPT-backed Telegram session so the prompt either omits the MINIMAL guidance or includes it only after the chosen explicit opt-in/defaulting rule.
