## Summary

Current main still matches the reported problem: the session-history helper returns true on the first assistant message anywhere in the transcript, and the fallback retry prompt remains hard-coded with no supported override hook. The related PR #69109 carried a plausible guard fix and tests, but it is closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #69086 open. The best maintainer path is to land the assistant-after-latest-user guard with regression coverage similar to #69109, then either add a supported retry-prompt override/hook at the attempt-execution boundary or split that second request into a focused product/API issue.
