## Summary

Keep open. Current `main` still limits `sessions.usage` discovery to configured gateway agents, while `agents.delete` can preserve transcript directories with `deleteFiles=false` and now purges deleted-agent store entries. That means preserved deleted-agent transcript history can still disappear from usage/history on current `main`, so this PR is not implemented and the underlying bug remains relevant.

## Evidence

There is no current-main path here that scans preserved on-disk session directories for deleted agents.
