## Summary

Keep this PR open. Current main still requires `operator.admin` for cron mutation methods and has no persisted `creatorScopes` or cron execution scope ceiling, so the authorization change is not implemented or obsolete. The PR remains a plausible core security/authorization fix, but the branch needs focused maintainer security review and cleanup of unrelated browser/QA-lab changes before it can land.

## Best Possible Solution

Keep this PR open for maintainer review. Ask the author to split unrelated browser and QA-lab changes, then review a focused cron-only branch that lowers only the intended methods, persists creator scopes at creation time, enforces the ceiling in embedded and CLI/MCP execution paths, and includes regression coverage proving write-scoped callers cannot update or run admin-created jobs to gain admin tool access.
