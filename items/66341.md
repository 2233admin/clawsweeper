## Summary

Keep open. Current `main` does not implement this PR's core behavior: cron mutators still require admin scope, `cron.add` still does not persist caller scopes, and the cron execution paths on `main` still only rely on `senderIsOwner` rather than a stored scope ceiling. The PR discussion also shows the original security blockers were addressed in-branch, but the latest review still flags unrelated browser and QA-lab changes bundled into the PR, so this is not a conservative close candidate.
