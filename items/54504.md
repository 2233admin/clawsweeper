## Summary

Keep this issue open. Current main still does not implement lightweight label-scoped subagent instruction files, workspace/user precedence, YAML frontmatter role metadata, or a documented pre-spawn plugin/API contract that can mutate the child system prompt, model, and tool policy for ordinary sessions_spawn calls.

## Best Possible Solution

Keep this open for maintainer product/API design. The strongest path is either a narrow native resolver for workspace/user-scoped subagents/<label>.md files applied before spawn, or a documented pre-spawn plugin hook that can mutate the pending child prompt, model/thinking, and tool policy. The implementation should define precedence, security limits for tool frontmatter, docs, and focused tests around sessions_spawn behavior.

## Risks / Open Questions

- A plugin can approximate pieces of this by rewriting sessions_spawn tool parameters or running its own subagent through api.runtime.subagent, but that does not provide the requested automatic role-file lookup, frontmatter schema, precedence rules, or tool-policy override for ordinary sessions_spawn calls.
- The design still needs maintainer decisions on precedence between explicit spawn params, per-agent config, global defaults, role-file frontmatter, and tools.subagents policy.
