## Summary

Keep open. Current main has adjacent usage surfaces through session-store persistence, usage RPCs, and `session.message`/`sessions.changed` WebSocket transcript events, but it does not expose the requested per-run `stream: "lifecycle"` / `phase: "usage"` event with accumulated usage, provider/model, duration, and last-call usage. The linked PR #51317 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open for maintainer product/API judgment. The likely implementation belongs in the core agent event contract: add a documented, backward-compatible lifecycle usage event or an equivalent run-scoped agent event emitted consistently from embedded, CLI, followup, memory, auto-reply, and cron run paths, with tests covering WebSocket delivery and existing transcript/RPC usage surfaces.
