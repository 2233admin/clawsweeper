## Summary

Keep open. Current `main` still does not implement Mattermost WebSocket ping/pong keepalive, and the health-monitor fallback now depends on `lastTransportActivityAt`, which the Mattermost plugin does not publish. That means a half-dead but still-open Mattermost socket can still look healthy on current `main`.

## Evidence

- **No ping/pong support in Mattermost socket wrapper:** `MattermostWebSocketLike` only defines `open`/`message`/`close`/`error`, plus `send`/`close`/`terminate`; there is no `ping`, `pong`, or listener wiring for WebSocket keepalive frames.
  - file: [extensions/mattermost/src/mattermost/monitor-websocket.ts:33](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/extensions/mattermost/src/mattermost/monitor-websocket.ts#L33)

  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Current connect loop uses auth + bot `update_at` polling, not socket keepalive:** `createMattermostConnectOnce` opens the socket, sends `authentication_challenge`, and optionally polls `getBotUpdateAt()` on a timer. It registers `open`, `message`, `close`, and `error` handlers only; there is no periodic `ws.ping()` / `pong` timeout path.
