## Summary

Keep open. Current main does not implement the requested `doctor.memory.status.perAgent` payload: the gateway type and handler still report only the active default agent, and the public protocol docs describe default-agent readiness only. The issue is well-scoped and has an exact linked open PR (#69322), but that PR is not shipped on current main, so closing now would be premature.

## Best Possible Solution

Keep #69295 open and review or land #69322, or an equivalent additive gateway change. The implementation should add a typed `perAgent` array to `src/gateway/server-methods/doctor.ts`, populate `{agentId, dirty, files, chunks}` from configured agents using `manager.status()` without deep probes, cover success/unavailable/error branches in `doctor.test.ts`, and update the public gateway protocol docs and any UI/client payload types that consume `doctor.memory.status`.
