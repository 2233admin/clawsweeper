## Summary

Current main does not expose per-agent memory status from `doctor.memory.status`. The handler still builds only `agentId`, `provider`, `embedding`, and `dreaming` for the default agent, while the CLI path can enumerate agents and return `manager.status()` data containing `dirty`, `files`, and `chunks`. PR #69322 directly targets this request, but it is open and unmerged, so the issue should remain open as the canonical tracker until that work lands or maintainers choose a different shape.

## Best Possible Solution

Keep #69295 open and review or rebase PR #69322 against current main. The likely implementation should add an additive `perAgent` field to `DoctorMemoryStatusPayload`, collect `{agentId, dirty, files, chunks}` from `manager.status()` for the default and configured agents, handle per-agent errors without failing the whole status call, update `docs/gateway/protocol.md`, and add tests for default, multi-agent, and manager-unavailable branches.
