## Summary

Keep open. Current `main` does not implement native cross-gateway `sessions_send`: the tool schema still only accepts local session targeting fields, the execution path resolves and sends through the currently connected gateway, and config has `gateway.remote` but no `gateway.peers`/remote agent registry. The linked PR #43656 is relevant proposed implementation work, but it is still open/unmerged, and the related xihe-jianmu IPC MCP hub is a useful external workaround rather than a native `sessions_send` fix with session-history integration.
