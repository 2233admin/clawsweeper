## Summary

Keep this issue open. It is paired with open same-author implementation PR #43656, which uses closing syntax for this issue, and current main still does not expose native cross-gateway routing for `sessions_send` or `sessions_spawn`.

## Best Possible Solution

Keep this issue open as the canonical native cross-gateway `sessions_send` tracker. Maintainers should review, revise, land, or close PR #43656; if it is not the right shape, a replacement should add a documented secure peer/remote-gateway contract across config schema, gateway protocol, session tool schemas, auth/token resolution, tests, and docs.
