## Summary

Keep PR #58823 open. Current main still implements and tests the opposite subagent model precedence from this PR, while the public subagents docs support the reporter's global subagent default expectation. The PR also reverses merged PR #58003's named-agent behavior and has a directly relevant unresolved allowlist/security discussion, so this is not conservative cleanup material.

## Best Possible Solution

Keep PR #58823 open for maintainer product/security review. Maintainers should decide the intended precedence contract, then either land a narrow fix in `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression coverage plus any needed direct-spawn allowlist validation, or keep PR #58003's precedence and update `docs/tools/subagents.md` plus close #58822/#58823 with that rationale.
