## Summary

Keep open. Current main still implements the opposite precedence from PR #58823 in both subagent model-selection paths: per-agent subagents.model, then the agent's own model, then agents.defaults.subagents.model. Checked-in tests also assert that agent primary model wins over the global subagent default, while the docs describe agents.defaults.subagents.model as the configured sub-agent default. The PR remains a plausible fix for the linked #58822 regression and needs maintainer product judgment because it reverses behavior intentionally merged in #58003.

## Best Possible Solution

Keep PR #58823 open for maintainer review. Decide the intended product contract for main-agent versus named-agent subagent model precedence, then either land a narrow fix in src/agents/model-selection.ts and src/cron/isolated-agent/model-selection.ts with the PR's regression coverage, or keep #58003's agent-primary precedence and update docs/tests to state that contract explicitly.
