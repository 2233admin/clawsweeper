## Summary

Keep this PR open. Current main still implements and tests the opposite precedence from the PR, the docs support the global subagent default expectation, and the PR is paired with open same-author issue #58822. It is an active implementation candidate that needs maintainer product/security review, not cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. If `agents.defaults.subagents.model` is intended to outrank an agent's own `model.primary`, land a narrow fix in `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts`, update the regression tests, and address or explicitly document model allowlist validation for normal `sessions_spawn` persistence. If #58003's named-agent precedence is the intended contract, update `docs/tools/subagents.md` and `docs/gateway/config-tools.md`, then close #58822 and this PR with that product decision.
