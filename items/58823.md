## Summary

Current main still gives the target agent's own model priority over `agents.defaults.subagents.model` in both the shared subagent resolver and isolated cron path, and checked-in tests assert that behavior. The PR is therefore not implemented on main or superseded; it remains a plausible fix for #58822, but it conflicts with the merged #58003 behavior and needs maintainer product judgment.

## Best Possible Solution

Keep the PR open for maintainer review. Decide the intended subagent model precedence contract, then either land a narrow fix across `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression coverage, or explicitly keep #58003's behavior and update #58822/docs/tests to reflect that contract.
