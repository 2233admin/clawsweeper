## Summary

Keep #69131 open. Current main has several related ACPX/Codex and Discord thread-binding fixes after the reported baseline, but I did not find source/test/release evidence that the full reported clean Discord Codex ACP failure is fixed. The remaining central symptom is the Codex ACP/acpx prompt/account-state turn failure (`ACP_TURN_FAILED` / account inactive / prompt contract divergence), and current OpenClaw still delegates the actual turn to the bundled `acpx` runtime without a targeted regression for this issue.

## Best Possible Solution

Keep the issue open until a maintainer can either add/pass a focused regression or record a live clean-baseline smoke that covers both layers: Discord `sessions_spawn(runtime="acp", agentId="codex", mode="session", thread=true)` binding and a non-thread `runtime="acp"` Codex run completing a simple prompt through the bundled ACPX/Codex ACP path. If current `acpx@0.6.1` plus the Codex wrapper is intended to fix it, link that fix explicitly and add coverage around the prompt/account-state failure mode before closing.
