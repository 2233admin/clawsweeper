## Summary

Keep open. The issue is fresh, concrete, and reports a real core exec/runtime failure mode on v2026.4.24. Current main has related process-tree kill hardening after that release, but that only addresses one known self-SIGTERM class and does not prove the underlying simple-command hangs are fixed. The provided related items are title-overlap noise, not duplicates.

## Best Possible Solution

Keep #72296 open and ask for a focused follow-up: exact `tools.exec.host`/sandbox/service mode, gateway stability bundle or diagnostics export, logs around the first hung exec session id, and a retest on current main or the next release containing the #71662 process-tree hardening. If the report reproduces on current main, the likely maintainer path is a targeted exec/supervisor regression test around simple shell commands under the reporter's Linux/npm/runtime mode, then a fix in `src/process/supervisor/*` or `src/agents/bash-tools.exec-runtime.ts`.
