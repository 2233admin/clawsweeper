## Summary

Keep open. Current `main` has generic plugin hook error handling and session-lock cleanup, but it does not implement the requested plugin-scoped circuit breaker: there is no per-plugin breaker state, no plugin/hook lock attribution, no persisted breaker registry, no gateway-status visibility for breaker state, and no manual reset command.

## Evidence

- **Hook failures are handled per call, not via a rolling breaker:** `createHookRunner` only supports hook-level `fail-open` vs `fail-closed`; `handleHookError` logs or throws for the current invocation, and the hook runners continue without any persisted per-plugin failure budget or open/half-open state.
  - file: [src/plugins/hooks.ts:214](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/plugins/hooks.ts#L214)
  - command: `nl -ba src/plugins/hooks.ts | sed -n '206,340p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Session-lock telemetry is not attributed to plugin/hook ownership:** The session lock payload stores only process metadata (`pid`, `createdAt`, `starttime`), and held locks track timing/path only. The watchdog releases overdue locks generically; there is no `pluginId` or `hookName` attribution path here.
  - file: [src/agents/session-write-lock.ts:7](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/agents/session-write-lock.ts#L7)
  - command: `nl -ba src/agents/session-write-lock.ts | sed -n '1,70p;195,210p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Plugin status model has no breaker lifecycle states:** `PluginRecord.status` is limited to `"loaded" | "disabled" | "error"`. There is no built-in plugin runtime state for `open`, `half-open`, or `forced-open`.
  - file: [src/plugins/registry-types.ts:250](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/plugins/registry-types.ts#L250)
  - command: `nl -ba src/plugins/registry-types.ts | sed -n '231,262p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Operator CLI has no circuit-breaker reset/admin command:** The plugin CLI defines `list`, `inspect`, `enable`, `disable`, `uninstall`, `install`, `update`, `doctor`, and `marketplace`; there is no `plugins circuit-breaker reset <plugin-id>` or similar operator surface.
  - file: [src/cli/plugins-cli.ts:141](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/cli/plugins-cli.ts#L141)

I also found no repo matches for `plugin-circuit-breakers.json`, `plugins circuit-breaker`, or similar breaker surfaces.
