## Summary

Current `main` does not contain this PR's nested repo-style skill-root watcher/loader changes. The gap described in PR #51733 still appears real for managed/workspace skill roots, so there is not strong evidence to close it as already implemented or obsolete.

## Evidence

- **Watcher still only covers shallow skill layouts:** `resolveWatchTargets()` adds only `<root>/SKILL.md` and `<root>/*/SKILL.md`. There is no watcher glob for repo-style nested roots like `<root>/*/skills/*/SKILL.md`.
  - file: [src/agents/skills/refresh.ts:81](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/agents/skills/refresh.ts#L81)
  - command: `nl -ba src/agents/skills/refresh.ts | sed -n '68,92p'`
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Loader still resolves only one nested `skills/` hop:** `resolveNestedSkillsRoot()` only upgrades `dir` to `dir/skills` when it finds `dir/skills/*/SKILL.md`, and `loadSkillEntries()` then loads only `baseDir/SKILL.md` or immediate `baseDir/*/SKILL.md` children. That does not discover managed/workspace layouts like `workspace/skills/<repo>/skills/<skill>/SKILL.md`.
  - file: [src/agents/skills/workspace.ts:317](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/agents/skills/workspace.ts#L317)

- **Current watcher test still asserts only shallow targets:** `refresh.test.ts` expects the watcher to include only the shallow `skills/SKILL.md` and `skills/*/SKILL.md` patterns for workspace and `.agents` roots; it has no coverage for nested repo-style watch targets.
