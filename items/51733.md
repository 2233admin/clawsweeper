## Summary

Keep open. Current `main` still misses the nested repo-style watcher and loader behavior this PR targets, and the PR discussion shows the author iterated on real gaps identified in review rather than an obsolete report.

## Evidence

- **Watcher still only covers top-level and one-level skill roots:** `resolveWatchTargets()` on current `main` only adds `<root>/SKILL.md` and `<root>/*/SKILL.md`. It does not watch nested repo-style paths like `<root>/<repo>/skills/<skill>/SKILL.md`, so edits there cannot trigger a skills snapshot refresh.
  - file: [src/agents/skills/refresh.ts:81](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/agents/skills/refresh.ts#L81)
  - command: `nl -ba src/agents/skills/refresh.ts | sed -n '80,130p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Loader still only resolves one nested `skills/` hop from the configured root:** `resolveNestedSkillsRoot()` only checks `dir/skills/*/SKILL.md`, and `loadSkillEntries()` calls it once on each configured root before scanning only that resolved base dir. That supports `extraDirs` pointing at a repo bundle root, but not managed/workspace roots shaped as `skills/<repo>/skills/<skill>/SKILL.md`.
  - file: [src/agents/skills/workspace.ts:317](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/agents/skills/workspace.ts#L317)
  - command: `nl -ba src/agents/skills/workspace.ts | sed -n '317,520p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Current tests do not cover the nested managed/workspace cases from the PR:** `refresh.test.ts` asserts only the simple watch targets, and `skills.loadworkspaceskillentries.test.ts` on current `main` lacks the repo-style nested managed/workspace regression cases described in the PR discussion.
  - file: [src/agents/skills/refresh.test.ts:45](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/agents/skills/refresh.test.ts#L45)

That aligns with the PR discussion: current `main` handles that layout in one path, but not when the same bundle sits under workspace-managed skill roots.
