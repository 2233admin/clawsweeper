## Summary

Keep open. Current main reduces the common default path by marking bundled GitHub Copilot Claude models as non-reasoning, but it still has no provider/model clamp for `github-copilot/claude-opus-4.7`: the Anthropic adaptive path maps `minimal`/`low` to `output_config.effort: "low"` when reasoning is enabled. The auth-profile cooldown concern also remains for shared `format` failures. Related PR #70584 is still open/unmerged, and #70376 was closed unmerged.

## Best Possible Solution

Keep #70322 open. Review or supersede #70584 with a scoped Copilot Anthropic Messages fix that clamps unsupported `minimal`/`low` effort for `github-copilot/claude-opus-4.7` to a supported value, adds transport regression tests with direct-Anthropic scope guards, and separately prevents `invalid_reasoning_effort`/schema-format rejections from poisoning shared GitHub Copilot auth profiles.
