## Summary

Keep open. Current `main` still applies the full 4,000-character error-detail cap to all non-OK `web_fetch` responses, so the soft-block reduction proposed in PR #41897 is not implemented on `main`. The PR remains a small, plausible improvement, and the discussion shows the author addressed the main review concern about preserving the untrusted-content wrapper.
