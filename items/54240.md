## Summary

Keep this issue open. Current main still lacks a Control UI ordering guard between the chat model dropdown's async `sessions.patch` and the next `chat.send`; the UI does not store or await a pending model-switch promise, `chat.send` carries no model override, and the Gateway can process the two requests concurrently.

## Best Possible Solution

Implement the UI-side ordering guard proposed in the issue: store a pending model-switch operation in Control UI state, disable the model picker/send controls while it is pending, have `handleSendChat` await that operation before `chat.send`, roll back and do not send if `sessions.patch` fails, and add a regression test that holds the patch promise unresolved while an immediate send is attempted.
