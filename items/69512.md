## Summary

Current `main` does not forward `exec-approvals.json` allowlists into `claude-cli` backend sessions. The bundled Anthropic CLI integration only maps OpenClaw's YOLO exec policy (`tools.exec.security: "full"` plus `tools.exec.ask: "off"`) to Claude's `--permission-mode bypassPermissions`; otherwise Claude permission handling is left to its own CLI flags, so #69512 is still a valid open gap.

## Evidence

- **Bundled claude-cli backend only whitelists OpenClaw MCP tools:** The Anthropic CLI backend hard-codes `--allowedTools mcp__openclaw__*` in both fresh and resume args. It does not derive any binary allowlist from `exec-approvals.json` or per-agent exec policy.
  - file: [extensions/anthropic/cli-backend.ts:31](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/anthropic/cli-backend.ts#L31)
  - command: `nl -ba extensions/anthropic/cli-backend.ts | sed -n '29,54p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Claude permission mapping only handles YOLO:** `resolveClaudePermissionMode()` returns `bypassPermissions` only when OpenClaw requested YOLO (`security === "full"` and `ask === "off"`). For allowlist/safer configs it returns no mode, so there is no translation of OpenClaw allowlists into Claude CLI policy.
