## Summary

Keep #69512 open. Current main does not forward OpenClaw exec-approvals allowlist entries into Claude CLI command execution. The bundled claude-cli backend only maps OpenClaw's YOLO exec posture to Claude's bypassPermissions mode, deliberately leaves safer/allowlist policies without a Claude permission mode, and preserves explicit raw Claude permission-mode overrides. Native OpenClaw gateway exec still enforces exec-approvals.json allowlist misses, but Claude CLI internal Bash/tool execution remains governed by Claude Code's own permission layer rather than the OpenClaw allowlist.

## Best Possible Solution

Keep the issue open and implement this in the bundled Anthropic claude-cli integration plus CLI runner plumbing. A conservative fix should derive the effective per-agent OpenClaw exec policy and exec-approvals allowlist, then either translate it into Claude Code Bash allow/deny rules or route Claude Code permission prompts through an OpenClaw-controlled permission-prompt-tool/MCP bridge that reuses the existing allowlist evaluator and denies misses. Add regression coverage for allowed and denied commands under claude-cli, explicit per-agent overrides, resume/live stdio sessions, and raw Claude arg precedence.
