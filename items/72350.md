## Summary

Keep PR #72350 open. Current main still eagerly warms the context-window cache for `openclaw tui`, and the PR is a focused two-line code/test change against the right helper. The only maintainer follow-up is scope: `terminal` and `chat` are also TUI entrypoints, while current tests intentionally keep `chat` eager-warming, so review should confirm whether this PR should cover only the canonical Gateway-backed `tui` command or all TUI aliases.

## Best Possible Solution

Keep the PR open for normal maintainer review. The likely product/code path is to merge the narrow `tui` skip plus regression test if only Gateway-backed `openclaw tui` should avoid eager context warmup, or ask the author to extend the skip set/tests to `terminal` and `chat` if all TUI entrypoints should avoid model/provider discovery during startup.
