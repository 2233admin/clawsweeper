## Summary

Do not close this PR. Current main still lacks the requested TUI skip: `tui` is a real CLI command, but it is absent from the eager-warmup skip set, so `shouldEagerWarmContextWindowCache(["node", "openclaw", "tui"])` would still return true. The PR is a focused bug-fix candidate with supporting bot review comments, not an obsolete or already-shipped change.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best code path is to add `tui` to the eager-warmup skip set with the proposed regression test, then make an explicit decision on whether `chat` and `terminal` aliases should keep or skip eager warmup.
