## Summary

Keep open. Current `main` mitigates fresh Linux installs by generating systemd units with `Restart=always`, but the gateway restart path still exits `0` for supervised `SIGUSR1` restarts. That means the specific `Restart=on-failure` failure mode reported in #70354 is still present for existing/manual systemd units, so this is not cleanly implemented on `main` yet.

## Evidence

- **Supervised restart still exits cleanly:** `runGatewayLoop()` treats both `spawned` and `supervised` restarts the same and calls `exitProcess(0)`, so a systemd unit using `Restart=on-failure` would still see a clean exit after a supervised `SIGUSR1` restart.

  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
