## Summary

Keep open. Current `main` still matches the PR's core claim: heartbeat dedupe writes use `loadSessionStore` followed by `saveSessionStore`, while the session-store helper explicitly says concurrent-safe mutations must re-read inside the lock via `updateSessionStore`. I did not find evidence that this fix already landed on `main` or shipped in a release.
