## Summary

Keep open. Current main still has the original waiting/promoted badge mapping, and the PR addresses a real-looking mismatch between daily and grounded signals. However, reviewer feedback is still unresolved: the staged section and summary still use `groundedCount` while displaying daily-log/replayed wording, so this PR is not cleanup-close material and likely needs a fuller UI/test update.

## Best Possible Solution

Keep the PR open and request changes: make the Dreaming Advanced provenance model consistent across the summary, staged section, waiting list, and promoted list, then add a focused regression test in `ui/src/ui/views/dreaming.test.ts` for daily-only, grounded-only, and mixed entries. If maintainers decide the grounded lane should remain the primary concept, update the UI wording/i18n instead of only switching the badge conditional.
