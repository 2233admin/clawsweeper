## Summary

Keep PR #71645 open. Current main still records Mattermost partial reply text into the draft stream, then sends tool-start status with another plain draft-stream update on the active preview, so the pending streamed text can be replaced before it is flushed. The PR is a focused Mattermost bug fix using existing draft-stream primitives, but it still needs maintainer review and ideally a regression test/docs decision before landing.

## Best Possible Solution

Keep PR #71645 open for maintainer review. The best path is a Mattermost-owned fix that preserves streamed assistant text across tool-start events using the existing draft stream lifecycle, adds a regression test for partial text followed by tool status, and updates the Mattermost preview-streaming docs if separate status posts are the intended UX. If maintainers prefer the broader overlapping Mattermost draft-boundary work in PR #70508, fold this narrower behavior and test into that canonical PR rather than treating this as implemented on main.
