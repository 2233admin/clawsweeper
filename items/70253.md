## Summary

Keep #70253 open. The reporter's follow-up analysis still matches current main: Mattermost's main inbound reply path forces draft preview streaming/block-stream suppression instead of honoring the account setting, and the channel-level agent-to-agent behavior is not implemented by `tools.agentToAgent`, which is documented as session-tool access. The later open #71569 appears to cover the Mattermost streaming-config subset, but it does not cover the broader channel A2A/bot-to-bot routing gap.

## Best Possible Solution

Keep this open or split it into two maintainer-trackable fixes: first, make the Mattermost main inbound path honor `channels.mattermost.streaming`/`blockStreaming` consistently with the other Mattermost paths and add regression coverage; second, decide and implement the channel-level A2A contract, likely including Mattermost `allowBots` parity and/or a documented broadcast/routing API for multiple configured agents in the same shared channel.
