## Summary

Keep #70253 open. Current main still matches the reported behavior: the Mattermost inbound path forces a draft-preview/block-stream-suppression path instead of honoring the documented streaming controls, and the broader channel-level bot-to-bot/A2A routing gap is not superseded by the narrower open #71569 streaming issue.

## Best Possible Solution

Keep this open or split it into maintainer-trackable fixes: implement Mattermost's documented streaming controls in schema/types/generated metadata/runtime with regression tests, and separately define the channel-level A2A/bot-to-bot contract, likely including Mattermost `allowBots` parity and/or an explicit multi-agent shared-channel routing/fanout API.
