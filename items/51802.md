## Summary

Keep open. Current `main` still routes the web UI `New session` action through `/new` on the active session, and the gateway test suite explicitly codifies `/new` as resetting that same session. The independent create-and-switch flow and recency sorting described in PR #51802 are not implemented on `main`.

## Evidence

- **Web UI still sends `/new`:** The chat run controls wire `onNewSession` to `state.handleSendChat("/new", { restoreDraft: true })`, so the UI still resets via a slash command instead of creating a separate session.
  - file: [ui/src/ui/app-render.ts:2364](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/ui/src/ui/app-render.ts#L2364)
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Current behavior is tested as same-session reset:** Gateway coverage explicitly says bare `/new` resets the same session: `handles bare /new by resetting the same session and sending reset greeting prompt`. That matches the PR's problem statement rather than its proposed fix.
  - file: [src/gateway/server-methods/agent.test.ts:1216](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/gateway/server-methods/agent.test.ts#L1216)
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Session selector code has no recency sort:** `resolveSessionOptionGroups` builds groups/options in input order and returns `Array.from(groups.values())` without sorting by `updatedAt`, so the recent-first ordering claimed by the PR is not present on `main`.
  - file: [ui/src/ui/chat/session-controls.ts:478](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/ui/src/ui/chat/session-controls.ts#L478)
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **No create-and-switch helper on main:** A symbol search on `main` found no `createAndSwitchChatSession` or `buildUniqueChatSessionIdentity` implementation, which are the core helpers described in the PR body.
  - command: `git grep -n "createAndSwitchChatSession\|buildUniqueChatSessionIdentity" -- ui/src/ui src`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
