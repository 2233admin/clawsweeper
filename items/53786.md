## Summary

Keep this issue open. Current main has generic live transcript events and Feishu routing, but it still does not implement exact bidirectional Feishu Bot and Local Dashboard mirroring: Control UI connects as WebChat, dashboard sends are session-local, WebChat clients are guarded from inheriting external routes, and inbound Feishu updates only refresh the dashboard when the resolved session is the active session.

## Best Possible Solution

Keep this open for maintainer product triage. Reproduce current main with Feishu DM and group/thread sessions, recording the active Control UI session key, resolved Feishu route/session key, gateway logs, and whether session.message arrives. If bidirectional mirroring is desired, add an explicit opt-in Control UI delivery/mirror flow for Feishu-backed sessions with tests for Feishu inbound-to-active-session refresh and dashboard-to-Feishu delivery. If the intended boundary is that Control UI/WebChat is a session viewer/editor rather than an external chat mirror, document that clearly in Control UI, WebChat, Feishu, and channel-routing docs before closing.
