## Summary

Keep open. Current `main` does not implement this PR's requested behavior: `knownLongFlags` is still absent from the strict exec safe-bin config schema, absent from generated schema output, excluded from config normalization/types, and not merged into compiled safe-bin profiles.

## Evidence

- **Strict runtime schema still rejects `knownLongFlags`:** `ToolExecSafeBinProfileSchema` only allows `minPositional`, `maxPositional`, `allowedValueFlags`, and `deniedFlags`, then calls `.strict()`. A config object with `knownLongFlags` would still be an unrecognized key on current `main`.

  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Config normalization path still drops the field:** `normalizeExecSafeBinProfilesInConfig` casts safe-bin profiles to a shape that excludes `knownLongFlags`, so the normalization path is not wired for this property on current `main`.
  - file: [src/config/normalize-exec-safe-bin.ts:14](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/config/normalize-exec-safe-bin.ts#L14)
