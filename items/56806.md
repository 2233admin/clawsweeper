## Summary

Keep open. Current `main` still does not expose `knownLongFlags` through the safe-bin config schema or normalization path, so the PR is not implemented on `main` and still addresses a real gap in the shipped config surface.

## Evidence

- **Zod config schema still rejects `knownLongFlags`:** `ToolExecSafeBinProfileSchema` on current `main` only allows `minPositional`, `maxPositional`, `allowedValueFlags`, and `deniedFlags`, then calls `.strict()`. A `knownLongFlags` key would still be treated as unrecognized.
  - file: [src/config/zod-schema.agent-runtime.ts:447](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/zod-schema.agent-runtime.ts#L447)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '447,458p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Normalization path still drops the field:** `normalizeExecSafeBinProfilesInConfig` casts `safeBinProfiles` fixtures without any `knownLongFlags` property, so the config normalization path on `main` does not preserve that field.
  - file: [src/config/normalize-exec-safe-bin.ts:14](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/normalize-exec-safe-bin.ts#L14)
  - command: `nl -ba src/config/normalize-exec-safe-bin.ts | sed -n '10,24p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Safe-bin fixture/compiler on `main` does not accept custom known flags:** `SafeBinProfileFixture` lacks `knownLongFlags`, `compileSafeBinProfile` derives known long flags only from `allowedValueFlags` and `deniedFlags`, and normalization only preserves those two arrays. That means the requested config surface is still absent on `main` even though the runtime validator supports long-flag metadata.
  - file: [src/infra/exec-safe-bin-policy-profiles.ts:14](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/exec-safe-bin-policy-profiles.ts#L14)
  - command: `nl -ba src/infra/exec-safe-bin-policy-profiles.ts | sed -n '14,90p'; nl -ba src/infra/exec-safe-bin-policy-profiles.ts | sed -n '248,292p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **User-facing docs still describe only four profile fields:** The current exec docs list `safeBinProfiles` as supporting `minPositional`, `maxPositional`, `allowedValueFlags`, and `deniedFlags`; there is no shipped docs surface for `knownLongFlags` yet.
  - file: [docs/tools/exec.md:108](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/tools/exec.md#L108)
