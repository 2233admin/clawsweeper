## Summary

Keep #44417 open. Current main still has the reported sudo/SUDO_USER behavior: when SUDO_USER is a non-root value, execSystemctlUser returns the --machine path before attempting direct systemctl --user. Existing tests also assert that sudo mode does not fall back to direct --user. The related fix PR #44816 is open/unmerged and is not present in this checkout, so it is not enough to close the issue.

## Evidence

That matches the issue's described root cause rather than the requested direct-first behavior.
