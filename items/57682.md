## Summary

Keep open. Current `main` still generates the Windows update restart helper as a `.bat` file with the same `netstat | findstr` polling loop and launches it through `cmd.exe`, so there is no strong evidence that the reported visible/hanging console-window path has been removed.

## Evidence

- **Windows update helper still embeds the reported `findstr` polling loop:** The Windows branch of `prepareRestartScript()` writes a batch script that polls `netstat -ano | findstr /R /C:":${port} .*LISTENING"` in a retry loop and then runs `taskkill` on the listener PID. That matches the issue's visible console title and hanging path.
