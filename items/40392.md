## Summary

Keep #40392 open. Current main still uses numeric `.bak.1`-`.bak.4` config backup rotation, and the linked datetime-suffix request remains open. The PR is coherent and targets live config-write behavior, but it is not safely closeable as implemented or duplicate cleanup because no maintainer-selected canonical replacement is evident and the PR has an unresolved migration data-loss concern.

## Best Possible Solution

Keep the PR open unless a maintainer chooses another canonical item. The best implementation should replace numeric config backup names with bounded datetime-suffixed backups, preserve or migrate existing `.bak.N` files until equivalent retention exists, avoid same-timestamp clobbering, and update the focused config backup tests around migration, retention, permission hardening, and collision behavior.
