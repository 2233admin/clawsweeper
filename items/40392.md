## Risks / Open Questions

- The PR's cleanup path intentionally treats legacy numeric `.bak.N` files as orphans, which can delete existing rollback snapshots on first upgrade before enough datetime backups exist.
- The PR documents same-millisecond timestamp collisions but does not implement a collision fallback, so burst writes can still clobber or drop a timestamped backup depending on platform behavior.
- Maintainers still need to choose a canonical implementation between this PR and #39982, or request a combined revision, before closing #39923.
