## Summary

Keep PR #69344 open. Current main at 6d60b035b4e7c1739f038808e041567448f58060 still has the original TUI long-token behavior: generic non-whitespace tokens are chunked at 32 characters, and the formatter has no code-like dot/paren/bracket preservation. The PR is not implemented or safely superseded, but its review discussion raises real product tradeoffs around weakening narrow-terminal protection, so it needs maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep #69344 open for maintainer review and coordinate it with #39565/#39505. The best implementation path is a focused `src/tui/tui-formatters.ts` policy that preserves genuinely copy-sensitive or code-like fragments without broadly disabling the narrow-terminal safeguard, backed by regression tests in `src/tui/tui-formatters.test.ts` for both code snippets and generic long-token wrapping.
