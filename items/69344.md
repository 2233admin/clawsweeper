## Summary

Current main does not implement PR #69344. The TUI formatter still chunks generic long non-whitespace tokens at 32 characters and the tests still assert the 32-character narrow-terminal safeguard. The PR is a plausible focused behavior change, but its own review discussion flags unresolved regression risk from raising the threshold to 128 and broadly exempting dot/paren/bracket tokens, so this should remain open for maintainer review rather than be conservatively closed.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to preserve genuinely copy-sensitive/code-like fragments in `src/tui/tui-formatters.ts` with focused regression coverage in `src/tui/tui-formatters.test.ts`, while retaining a clear narrow-terminal safeguard for generic long tokens. Coordinate with the still-open #39565/#39505 formatter work so OpenClaw lands one coherent TUI copy-safety policy rather than overlapping heuristic changes.
