## Summary

Keep PR #69344 open. Current main still has the pre-PR TUI sanitizer behavior: generic non-whitespace tokens are chunked at 33+ characters into 32-character segments, and `isCopySensitiveToken` does not preserve long code-like fragments containing dots, parentheses, or brackets unless they already match existing URL/path/underscore/credential heuristics. The provided review discussion shows the author fixed the original test-coverage concern, while later P2 comments raise product/implementation tradeoffs about preserving narrow-terminal protection. Related PR #39565 overlaps on dotted/copy-sensitive tokens, but it is narrower and does not clearly supersede #69344's threshold/code-fragment scope.

## Best Possible Solution

Keep #69344 open for maintainer review. The best path is a focused TUI formatter change in `src/tui/tui-formatters.ts` with regression tests in `src/tui/tui-formatters.test.ts` that preserves genuinely copy-sensitive code-like fragments, including common method-chain/call tokens, while retaining a narrow-terminal guard for generic long prose or punctuation-only tokens. Coordinate with #39565's narrower dotted-identifier fix rather than auto-closing this PR as implemented or superseded.
