## Summary

Current main does not implement the requested hot-path fix. The subagent registry read path still reaches a synchronous readFileSync + JSON.parse, getSubagentRunsSnapshotForRead still calls that disk loader outside tests, and sessions.list/session row hydration still invokes subagent lookups from the listing path. PR #57863 attempted the exact mtime-cache fix, but the provided GitHub context says it was closed unmerged and its head commit is not present in this checkout.

## Best Possible Solution

Keep #57557 open. The best fix is to avoid repeated full parses in hot reads: add a small mtime/invalidation cache around subagent registry disk loading or pass a single registry snapshot through sessions.list row hydration, invalidate on save, refresh when the file changes externally, and add regression coverage proving repeated session/RPC reads do not re-parse runs.json when unchanged.
