## Summary

Keep open. Current `main` does not implement the composing-aware inbound debounce described in PR #47542: WhatsApp config/runtime still expose only `debounceMs`, account/runtime wiring does not carry a `composingDebounceMs` field, and the shared inbound debouncer still returns only `enqueue`/`flushKey` with no timer-extension helpers. I also found no shipped docs/changelog/config-surface evidence that this feature landed on `main`.
