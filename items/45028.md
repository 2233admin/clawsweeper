## Summary

Keep open. Current main still does not implement the reported Control UI operator-reply path: the browser Control UI connects as `webchat`, sends normal chat messages with `deliver: false`, and gateway routing explicitly prevents webchat clients from inheriting external routes on channel-scoped sessions. Main does include a related metadata-preservation fix for externally backed sessions, but that protects stored route state/subagent completion routing rather than making normal Control UI replies deliver to Telegram.
