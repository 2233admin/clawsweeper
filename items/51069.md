## Summary

Keep open. Current `main` only routes Feishu `send` attachments through plugin actions when the tool passes a string media source (`media`/`path`/`filePath`). The shared message-tool schema still advertises `buffer` for send attachments, but the send-action normalization does not hydrate `send` params into a media payload, and Feishu's action handler does not read `buffer`/`filename`/`contentType`. That means the PR's core buffer-based image-send gap is not implemented on `main`.
