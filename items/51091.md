## Summary

Keep open. Current `main` intentionally preserves mutating-tool failure warnings even when `messages.suppressToolErrors` is set, and recent tests/changelog encode that safety invariant. This PR changes behavior rather than matching an already-implemented fix, so it needs maintainer product/safety judgment rather than conservative auto-close.
