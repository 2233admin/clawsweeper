## Summary

Keep #69608 open. Current main has adjacent startup-context support for canonical daily files and some slugged same-day files, but it does not contain the PR's shared daily-memory/session-summary model or several concrete fixes described in the PR: session-memory still writes UTC-dated summary notes without a sentinel/provenance index, missing bootstrap placeholders are still injected into context, the slug generator still runs with full embedded context/tools, and doctor/dreaming/short-term flows still mostly assume canonical `memory/YYYY-MM-DD.md` paths.

## Best Possible Solution

Keep the PR open for maintainer review. Ask the author to rebase/trim if needed, address the existing review threads, and preserve the useful core direction: a shared daily-memory/session-summary helper model with focused tests across startup context, session-memory, doctor, dreaming, REM backfill, short-term promotion, bootstrap placeholder handling, and utility embedded slug generation.
