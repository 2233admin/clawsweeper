## Summary

Keep open. Current `main` still has the reported docs/schema mismatch: Google Chat auth rejects add-on tokens when `appPrincipal` is absent, but the shipped TypeScript config type, zod schema, and generated config metadata still mark `appPrincipal` as optional and do not require it under `channels.googlechat.accounts.*`.

## Evidence

- **Runtime still requires principal binding for add-on tokens:** `verifyGoogleChatRequest` rejects app-url add-on issuers when `expectedAddOnPrincipal` is empty, returning `missing add-on principal binding`. The webhook path passes `target.account.config.appPrincipal` into that check.
  - file: [extensions/googlechat/src/auth.ts:169](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/extensions/googlechat/src/auth.ts#L169)
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Webhook auth uses config.appPrincipal directly:** Google Chat webhook target authorization forwards `target.account.config.appPrincipal` as `expectedAddOnPrincipal`, so an omitted config value still fails this path on current `main`.
  - file: [extensions/googlechat/src/monitor-webhook.ts:108](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/extensions/googlechat/src/monitor-webhook.ts#L108)
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Regression is still encoded in tests:** Current tests explicitly expect add-on tokens without a configured principal binding to fail with `missing add-on principal binding`.
  - file: [extensions/googlechat/src/targets.test.ts:333](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/extensions/googlechat/src/targets.test.ts#L333)
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Public config schema still marks appPrincipal optional:** `GoogleChatAccountSchema` still declares `appPrincipal: z.string().optional()`, and `accounts` remains optional on `GoogleChatConfigSchema`.
  - file: [src/config/zod-schema.providers-core.ts:825](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/config/zod-schema.providers-core.ts#L825)
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Exported TS config type and generated metadata still present appPrincipal as optional:** The exported `GoogleChatAccountConfig` type uses `appPrincipal?: string`, and generated bundled channel config metadata lists `appPrincipal` without any required marker in both top-level and account-scoped shapes.
  - file: [src/config/types.googlechat.ts:77](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/config/types.googlechat.ts#L77)
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Generated config metadata still lacks a required entry:** Bundled config metadata includes `appPrincipal` as a plain string property at both the top-level Google Chat shape and the `accounts.*` shape, matching the issue's docs mismatch rather than a fix.
  - file: [src/config/bundled-channel-config-metadata.generated.ts:4582](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/config/bundled-channel-config-metadata.generated.ts#L4582)
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)

## Risks / Open Questions

- The issue title overstates the scope: `channels.googlechat.accounts` itself is still legitimately optional because top-level default-account config is supported. The unresolved bug is the `appPrincipal` requirement mismatch for add-on/app-url auth paths.
- I could not inspect PR #51817 with `gh` because GitHub CLI auth is unavailable in this checkout, so this review is based on current `main` only.
