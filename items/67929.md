## Summary

Keep open. Current `main` still hardcodes a 60s timeout for MiniMax VLM image requests and still omits `timeoutMs` on the MiniMax fallback/direct paths, so the bug described by this PR remains live. The PR appears directionally correct from the supplied discussion, with the only noted review issue being an unrelated iMessage test bundled into the branch.
