## Summary

Keep PR #39905 open. Current main still lacks the PR's central Telegram per-conversation `replyToMode` and preview `streaming` override surface: scoped direct/group/topic config types and strict schemas do not accept those keys, and both normal Telegram replies and native command replies still receive account-level startup values. The PR branch is stale against the current `extensions/telegram` layout, but the requested capability remains missing rather than implemented or superseded.

## Best Possible Solution

Keep this PR open until a maintainer ports or explicitly rejects the feature. The best implementation is to add typed and schema-validated Telegram scoped overrides for `direct`, `groups`, and `topics`; resolve effective `replyToMode` and preview streaming with topic > direct/group > account precedence before normal dispatch; apply the same reply-threading resolution to native command replies; update Telegram/config docs; and adapt the branch to the current `extensions/telegram` plugin layout and nested streaming config contract.
