## Summary

Keep PR #69707 open. Current main still forwards configured `embedding.dimensions` to the OpenAI embeddings client and has a regression test asserting that behavior, so the PR is not implemented on main. The PR is a focused memory-lancedb compatibility fix with relevant tests and an addressed review concern; no provided related item clearly supersedes this exact outbound-`dimensions` behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best path is to land a scoped memory-lancedb fix that omits `dimensions` from OpenAI-compatible outbound embedding requests, validates configured dimensions as a finite positive integer, truncates locally only after receiving a long-enough vector, and keeps the focused regression tests. Reconcile this with adjacent memory-lancedb provider-compatibility PRs rather than closing it as obsolete.
