## Summary

Keep PR #69707 open. Current main still forwards configured `embedding.dimensions` to the OpenAI embeddings request and the current regression test explicitly expects that forwarding, so the PR is not implemented or obsolete. The PR is a focused memory-lancedb compatibility fix, and its final commit addresses the bot review by validating non-positive dimensions before local truncation.

## Best Possible Solution

Keep PR #69707 open for maintainer review. The likely best path is to land this scoped memory-lancedb behavior, or an equivalent patch, that omits `dimensions` from OpenAI-compatible outbound embedding requests, validates configured dimensions as a finite positive integer, truncates locally after receiving a long-enough vector, and keeps the focused regression tests.
