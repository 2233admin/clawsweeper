## Summary

Keep PR #69707 open. Current main still forwards configured `embedding.dimensions` to the OpenAI embeddings request, and the existing test suite explicitly asserts that forwarding behavior. The PR is a focused, still-relevant bug fix proposal for OpenAI-compatible providers that reject `dimensions`, with regression coverage in its diff; it is not implemented on main and is not clearly superseded by the related memory-lancedb reports.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers agree with the compatibility direction, the best path is a scoped memory-lancedb fix that omits `dimensions` from upstream OpenAI-compatible embedding requests, truncates locally only after validating a finite positive configured dimension, fails clearly for too-short embeddings, and updates the existing regression test that currently asserts upstream forwarding. Review should explicitly reconcile this with the prior #32036 behavior and the adjacent baseUrl/encoding compatibility work rather than auto-closing it as stale or duplicate.
