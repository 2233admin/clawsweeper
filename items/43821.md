## Summary

Keep this issue open. Current main has meaningful partial env-proxy support, but an open implementation PR still targets this issue, and main does not yet satisfy the broad global HTTP proxy request across all network surfaces.

## Best Possible Solution

Keep this issue open while PR #43919 is reviewed and either landed or closed. If it lands, verify the shipped behavior on current main before closing as implemented. If it is closed without landing, keep this umbrella open or split it into narrower tracked gaps for global fetch, ALL_PROXY/SOCKS handling, browser proxy behavior, and channel/provider-specific transport paths.
