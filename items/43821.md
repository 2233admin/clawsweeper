## Summary

Keep #43821 open. Current main has meaningful partial environment-proxy support, but it does not satisfy the broad request for comprehensive/global HTTP proxy behavior across web_fetch, model/provider guarded fetches, and all network paths. The issue discussion also identifies remaining gaps and explicitly says #58034 only covers one slice, while #43919 is still only a proposed PR in the provided context.
