## Summary

Keep open. Current main has partial ACP runtime-option support, but it does not implement the requested per-agent ACPX config defaults for model or reasoning/config options. The ACPX plugin config and manifest still expose only per-agent command overrides, `agents.list[].runtime.acp` only covers agent/backend/mode/cwd, and `/acp spawn` still lacks model/thinking options. This is a recent, concrete feature request from a non-maintainer with no protected labels, so it should not be conservatively closed.

## Best Possible Solution

Keep the issue open and implement a typed per-agent ACP default surface. The cleanest product path is to extend either `agents.list[].runtime.acp` or `plugins.entries.acpx.config.agents.<id>` with `model`, `thinking`/`reasoning_effort`, and a generic backend config-options bag, then merge those defaults before ACP session initialization for both `sessions_spawn(runtime="acp")` and `/acp spawn`, with explicit call options taking precedence. Update the strict Zod schema, plugin manifest/UI hints, ACP docs, and tests covering Codex normalization plus generic `session/set_config_option` behavior.
