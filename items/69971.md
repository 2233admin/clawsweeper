## Summary

Keep #69971 open. Current main still lacks both requested behaviors: direct plugin installs do not pass a profile-aware extensions directory, and plugin discovery still skips top-level symlinked plugin directories. The PR remains the canonical useful fix candidate, but it needs revision for the unresolved install/discovery root-alignment concern before merge.

## Best Possible Solution

Keep #69971 open and ask for a revision that makes plugin install and discovery share one managed extensions-root contract after profile/env resolution. The fix should pass that root through npm/path/link install paths, account for the channel-setup install path called out in the PR body, and retain regression coverage for --profile installs, OPENCLAW_CONFIG_PATH without OPENCLAW_STATE_DIR, legacy-state-dir fallback, symlinked plugin directories, and broken symlinks.
