## Summary

Keep open. Current main still lacks both requested behavior changes: plugin install does not pass a profile/config-aware extensions root into direct path/npm installs, and plugin discovery still skips directory symlinks. The PR is not obsolete or fully superseded, but it also has an unresolved review concern about aligning the install root with the discovery root contract.

## Best Possible Solution

Keep this PR open, but require a revision that uses one shared, late-bound plugin install root aligned with resolvePluginSourceRoots/resolveConfigDir after profile env is applied. Then pass that root through the direct path/npm install paths, add regression coverage for --profile / OPENCLAW_STATE_DIR plus OPENCLAW_CONFIG_PATH-only root selection, and keep the symlink + broken-symlink discovery tests. After that, run the changed CLI/plugin lanes before maintainer review.
