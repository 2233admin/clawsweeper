## Summary

Keep PR #39610 open. Current main at 8ba82534e6c98410e19b226f3534b285fd75d278 still supports only global `agents.defaults.userTimezone`; it does not accept, validate, document, or consistently resolve `agents.list[].userTimezone`. The provided PR remains the canonical unmerged implementation for the per-agent timezone request from #39579.

## Best Possible Solution

Keep #39610 open as the canonical implementation path. The maintainer-ready path is to rebase it onto current main, add `agents.list[].userTimezone` to config types, runtime schema, generated schema/help, and docs; centralize resolution as per-agent override -> `agents.defaults.userTimezone` -> host timezone; thread the active agent id through system prompt params, `session_status`, current-time helpers, heartbeat active-hours and prompt injection, envelope/system-event timestamps, gateway/chat timestamp injection, cron, memory flush, post-compaction context, and plugin inbound-envelope surfaces; validate IANA timezone strings consistently; then run the appropriate changed gates.
