## Summary

Keep open. Current `main` still has the unguarded `channel ? buildGroupDisplayName(...)` fallback in `buildGatewaySessionRow()`, so the reported cosmetic regression is not implemented on `main` and remains plausible for mixed TUI/Telegram sessions.

## Evidence

- **Session row display name still prefers group formatting whenever `channel` is set:** `buildGatewaySessionRow()` still computes `displayName` as `entry?.displayName ?? (channel ? buildGroupDisplayName(...) : undefined) ?? entry?.label ?? originLabel`, with no `chatType`/group guard before calling the group formatter.
  - file: [src/gateway/session-utils.ts:1239](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/session-utils.ts#L1239)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1232,1252p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Group formatter always produces group-style provider tokens:** `buildGroupDisplayName()` normalizes the provider and generates `provider:g-...` style labels from the supplied id/subject/channel. If a direct or main session reaches this path with `provider: "telegram"`, the label will be group-like rather than `openclaw-tui`.
  - file: [src/config/sessions/group.ts:45](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/sessions/group.ts#L45)
  - command: `nl -ba src/config/sessions/group.ts | sed -n '41,70p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **TUI still identifies itself as `openclaw-tui`:** The TUI gateway client is still instantiated with `clientDisplayName: "openclaw-tui"`, which matches the issue's expected status-line label and confirms the source-side expectation remains current.
  - file: [src/tui/gateway-chat.ts:121](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/tui/gateway-chat.ts#L121)
  - command: `nl -ba src/tui/gateway-chat.ts | sed -n '116,126p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Mixed webchat/TUI plus external-channel sessions are still an active supported path:** Current tests explicitly preserve an established external route when webchat views an existing external session, including direct-session cases. That makes the issue's mixed-surface scenario current rather than obsolete.
  - file: [src/auto-reply/reply/session.test.ts:3375](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/reply/session.test.ts#L3375)

Inspection of current source shows the corresponding guard has not been added in `src/gateway/session-utils.ts`.
