## Summary

Keep open. Current main still has the misleading normal status wording: zero pending BOOTSTRAP.md files renders as `no bootstrap files`, even though the measured signal is only BOOTSTRAP.md presence. The open related PR #43614 appears to target the exact fix, but it is not merged, so the original issue remains valid.

## Evidence

- **Misleading wording still present:** `buildStatusAgentsValue` still formats `bootstrapPendingCount === 0` as `no bootstrap files` and nonzero counts as `N bootstrap file(s) present`.
