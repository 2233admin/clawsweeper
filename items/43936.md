## Summary

Keep open. Current `main` still drops OpenAI TTS `instructions` unless the model name contains `gpt-4o-mini-tts`, even when `providers.openai.baseUrl` points at a non-default OpenAI-compatible endpoint. The code was refactored from `src/tts/*` into `extensions/openai/*`, but the behavior this PR targets is still not implemented on `main`.

## Evidence

- **Current main still gates instructions only on model name:** `resolveOpenAITtsInstructions()` trims `instructions` and returns them only when `model.includes("gpt-4o-mini-tts")`; it does not consider `baseUrl`. `openaiTTS()` calls that helper before sending the request body.

- **Custom endpoints already relax model and voice validation on main:** `isValidOpenAIModel()` and `isValidOpenAIVoice()` both allow arbitrary values when `isCustomOpenAIEndpoint(baseUrl)` is true, so `main` already treats non-default `baseUrl` values as OpenAI-compatible endpoints for model/voice validation.
  - file: [extensions/openai/tts.ts:42](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/openai/tts.ts#L42)
