Keep open. Current main still lacks the requested `openclaw doctor` warning for invalid `agents.defaults.model.primary` and `agents.list[].model` primaries, and this PR remains a plausible implementation candidate for open #39811. It needs fresh maintainer review on current main plus cleanup around terminal sanitization, alias-consistent resolution, and catalog-loading side effects before merge.

## Best Possible Solution

Keep this PR open and ask for a current-main-ready implementation in the doctor health contribution layer or a shared validation helper used by that layer. The final patch should check default and per-agent primary model refs against the catalog, resolve aliases the same way runtime/model-picker code does, sanitize terminal-facing text, use an explicit read-only/offline catalog-loading policy for doctor diagnostics, and add targeted tests for defaults, per-agent models, aliases, empty catalogs, and terminal sanitization. Keep #39811 as the canonical issue until the warning ships.
