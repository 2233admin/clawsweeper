## Summary

Keep #40355 open. Current main still does not implement the PR's requested `openclaw doctor` warning for invalid `agents.defaults.model.primary` or `agents.list[].model`. Main has a hooks-only doctor catalog warning and a gateway startup warmup warning for the explicit default primary, but neither covers the general doctor diagnostic this PR is trying to add.

## Best Possible Solution

Keep the PR open and ask for a rebase/port. The best maintainer path is a dedicated doctor health contribution or shared validation helper that checks `agents.defaults.model.primary` and each `agents.list[].model` against the current model catalog, resolves aliases the same way runtime model selection does, emits sanitized warnings, and adds focused regression tests. #39811 should remain the canonical issue for the broader model-misconfiguration visibility problem.
