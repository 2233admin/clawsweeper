## Summary

Keep #71849 open. Current main still routes Voice Call realtime memory/context questions through the shared `openclaw_agent_consult` embedded-agent path and does not expose a `realtime.fastContext` config, fast memory/session lookup hot path, partial-transcript accumulation for context detection, or explicit fast-context miss/timeout result. The maintainer comment in the issue already says to keep it open, and local source/docs/tests agree with that assessment.

## Best Possible Solution

Keep #71849 open and implement it in the bundled Voice Call plugin: add a default-off `realtime.fastContext` config block, resolve memory/session context with a short hard deadline before or alongside `openclaw_agent_consult`, accumulate/normalize partial user transcript chunks for context-trigger detection, submit an explicit miss/timeout result instead of leaving the realtime model in an open-ended checking state, keep full consult as fallback, and update `docs.openclaw.ai/plugins/voice-call` plus focused config/runtime tests.
