## Summary

Keep PR #71449 open. Current main does not already implement the PR's central changes: the gateway startup log still uses the old plain `ready (...)` wording, there is no sidecar/channel startup summary helper, Telegram polling still blocks/retries webhook cleanup before polling without a startup budget timeout, and Feishu startup still only treats timeout/abort as non-fatal probe outcomes. The PR is fresh, focused, and its review thread shows the author addressed several concrete bot-review findings, so this should get maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review and iteration. The best path is to validate the PR against the still-relevant gateway/channel readiness problem, preserve the legacy `ready (` marker for existing tooling, add or keep focused regression tests for the startup log, sidecar summary ordering, Telegram webhook cleanup timeout/background retry, and Feishu transient startup probe behavior, then decide whether the sidecar summary should reuse the existing `channel-health-policy` readiness semantics.
