## Summary

Keep open. Current `main` does not implement the PR's security fix: exec child-process env sanitization still inherits provider auth env vars from `process.env`, and the sanitizer does not use the existing provider-auth env-var registry.

## Evidence

- **Current sanitizer still forwards inherited env except policy-blocked keys:** `sanitizeHostExecEnvWithDiagnostics` copies `baseEnv` entries unless `isDangerousHostInheritedEnvVarName(key)` returns true, then merges them into the child env. There is no provider-auth blocklist in this path on current `main`. That means keys like `OPENAI_API_KEY` remain inherited if present in `process.env`.
  - file: [src/infra/host-env-security.ts:217](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/infra/host-env-security.ts#L217)
  - command: `nl -ba src/infra/host-env-security.ts | sed -n '156,239p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Override sanitization also lacks provider-auth blocking:** `sanitizeHostEnvOverridesWithDiagnostics` rejects only PATH, `isDangerousHostEnvVarName`, and `isDangerousHostEnvOverrideVarName`. It does not consult provider-auth env names, so the gap noted on the PR is also still present on current `main`.
  - file: [src/infra/host-env-security.ts:188](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/infra/host-env-security.ts#L188)
  - command: `nl -ba src/infra/host-env-security.ts | sed -n '156,207p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Exec tool path still sanitizes from `process.env`:** The host exec path builds `inheritedBaseEnv` from `process.env` and passes it to `sanitizeHostExecEnvWithDiagnostics`, so any provider auth vars left in the inherited env reach child processes.
  - file: [src/agents/bash-tools.exec.ts:1536](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/bash-tools.exec.ts#L1536)
  - command: `nl -ba src/agents/bash-tools.exec.ts | sed -n '1536,1544p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Provider auth env registry exists but is unused by the sanitizer:** `listKnownProviderAuthEnvVarNames()` already enumerates provider auth env vars such as `OPENAI_API_KEY` and `ANTHROPIC_API_KEY`, but current `src/infra/host-env-security.ts` does not import or use it.
