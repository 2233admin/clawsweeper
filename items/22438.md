## Summary

Keep open. Current main partially addresses the original motivation by keeping persona files in subagent/cron bootstrap and by adding lightContext/lightweight paths, but it still does not implement the requested configurable agents.defaults.bootstrapTier surface or minimal/standard/full tier selection. The linked implementation PR #22439 is still open in the provided GitHub context, and the latest maintainer comment also kept #22438 open.

## Best Possible Solution

Keep #22438 open until maintainers either merge a schema-backed bootstrap tier feature or explicitly choose another product direction. A good implementation would add typed/config-documented per-role bootstrap selection, tests for default and override behavior, and docs that reconcile bootstrapTier with current lightContext and bootstrap budget controls.
