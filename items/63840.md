## Summary

Keep open. Current main still leaves Slack Agents & Assistants DM root events with `thread_ts === ts` without an explicit `MessageThreadId` for non-`all` reply modes, so this PR is not obsolete. The branch is directionally useful, but it still needs revision because its DM gate uses raw `channel_type` and it includes an unrelated agents test-harness change.

## Best Possible Solution

Keep this PR open and revise it: pass normalized Slack direct-message state into `resolveSlackThreadContext` or derive the DM-root condition from the same normalization path used by prepare; set `MessageThreadId` only for Slack direct-message assistant roots where `thread_ts === ts`; preserve channel/group top-level behavior for `off`, `first`, and `batched`; add focused tests for raw `channel_type: "im"`, D-prefix DMs with missing/wrong `channel_type`, non-DM self-thread roots, and the concurrent assistant-thread path from #63659; remove or split the unrelated agents harness edit.

- The linked bug includes concurrent DM assistant-thread contamination; the PR should prove whether `MessageThreadId` alone is sufficient or whether DM assistant roots also need thread-scoped session routing.
