## Summary

Keep open. Current main does not implement `agent:turn:end`, and #71399 is the focused implementation attempt for open issue #37833 rather than obsolete cleanup. The PR still needs maintainer/code-review follow-up because review comments identify real contract and dispatch concerns.

## Best Possible Solution

Keep #71399 open for maintainer review. The best path is to finish this PR or a successor by emitting `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()` on success and failure, including `sessionKey`, `success`, `durationMs`, and optional `errorCode`, using bounded non-blocking hook dispatch or documenting a deliberate exception, then covering success/failure manager emission plus the type guard in tests. Update public hooks docs if this becomes an operator-facing internal hook event.
