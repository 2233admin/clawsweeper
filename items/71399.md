## Summary

Keep PR #71399 open. Current main still does not implement an `agent:turn:end` internal hook: ACP turn completion only updates latency/error stats, and the internal hook contract/docs do not define this event. The PR is the current focused implementation attempt and supersedes the older closed PR #60734, but it still needs maintainer/code review follow-up for the advertised `errorCode` payload and bounded hook dispatch concerns.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish this PR or a successor by emitting `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()` on success and failure, including `sessionKey`, `success`, `durationMs`, and optional `errorCode`, using bounded non-blocking hook dispatch or explicitly documenting why this hook is different, then covering the type guard and ACP manager success/failure emission in tests and updating internal hook docs if this becomes operator-facing.
