## Summary

Keep open. Current main still does not implement `agent:turn:end`, and #71399 is the active PR tracking the work after issue #37833 was closed as superseded by this PR. The PR is useful but still needs maintainer review for the missing advertised `errorCode` payload and unbounded hook dispatch concerns already raised in review.

## Best Possible Solution

Keep #71399 open for maintainer review. Finish this PR or a successor by emitting `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()` on success and failure, including `sessionKey`, `success`, `durationMs`, and optional `errorCode`, using bounded non-blocking hook dispatch or documenting a deliberate exception, then cover success/failure manager emission plus the type guard in tests. Update public hooks docs if the event becomes operator-facing.
