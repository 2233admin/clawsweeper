## Summary

Keep open. Current `main` still matches the reported Feishu silent-failure path: oversized inbound media causes `saveMediaBuffer` to throw, `resolveFeishuMediaList` catches that error and only logs it, then `handleFeishuMessage` continues into normal agent dispatch with an empty media payload and no user-facing Feishu warning. The related items are adjacent media bugs, not a canonical replacement for this issue.

## Best Possible Solution

Keep the issue open and fix it in the bundled Feishu plugin. The best implementation is to make Feishu media resolution return structured success/failure state or throw a typed recoverable media error, send a chat-visible Feishu warning that includes the configured `mediaMaxMb` limit, and avoid a normal agent run when the triggering input was only dropped media. Add regression tests for an oversized single file and embedded post media failure asserting both user notification and no unqualified agent dispatch.
