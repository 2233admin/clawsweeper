## Summary

Keep open. Current main still drops Feishu inbound media failures into logs only, then continues normal agent dispatch with an empty media payload. The issue is a bundled Feishu plugin behavior gap, not stale, not protected by another canonical item, and not solved on main.

## Best Possible Solution

Keep the issue open and fix it in the bundled Feishu plugin. The most robust path is to make media resolution return structured success/failure state or throw a typed recoverable media error, send a Feishu-visible warning that includes the configured `mediaMaxMb` limit, and avoid an unqualified normal agent run when the triggering input was only dropped media. Add regression tests for oversized single media and embedded post media asserting user notification and no silent empty-media dispatch.
