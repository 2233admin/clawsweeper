## Summary

Keep open. Current `main` does not implement automatic Feishu DM topic/thread-based session isolation: DM routing still keys on `senderOpenId`, Feishu config has no DM session-scope option, and Feishu conversation IDs only model group topic scopes. There is an explicit ACP thread-binding workaround for DMs, but it is not the requested default session-routing behavior.

## Evidence

- **DM routing is still per-user only:** Feishu message handling parses `root_id` and `thread_id`, but later resolves DM `peerId` as `ctx.senderOpenId` whenever `isGroup` is false, so DM session routing ignores DM topic/thread identifiers on current main.
  - file: [extensions/feishu/src/bot.ts:132](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/extensions/feishu/src/bot.ts#L132)

  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **No Feishu DM session-scope config exists:** The Feishu DM config schema only accepts `enabled` and `systemPrompt`. There is no `dm.sessionScope` or equivalent DM topic/thread isolation setting in the current schema.
