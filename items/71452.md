## Summary

Keep #71452 open. Current main at 6d60b035b4e7c1739f038808e041567448f58060 still has the reported hardcoded `messages.slice(0, 25)` in the shared message formatter, and the human-readable `read`, `list-pins`, and `search` paths still route through that formatter. Teams Graph-side pagination is partially implemented and tested, but it does not resolve the remaining generic display cap or full pagination/limit contract requested by the issue.

## Best Possible Solution

Keep the issue open and implement a narrow message formatting/action contract follow-up: pass the requested limit or pagination metadata into `renderMessageList`, remove or configure the hardcoded 25-row cap, add regression tests for more than 25 rendered `read`, `list-pins`, and `search` rows, and keep the existing provider-side Teams pagination coverage green.
