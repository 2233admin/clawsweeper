## Summary

Keep open. Current main at a35d259 still contains the reported hardcoded 25-row cap in the generic message formatter, and the read, list-pins, and search CLI text paths still route through that capped formatter. Teams-side pagination for pinned messages and channel lists is partially implemented and tested, but it does not solve the generic human-readable output limit described in the issue.

## Best Possible Solution

Keep the issue open and implement a narrow message command follow-up: pass the requested limit or pagination metadata into the formatting layer, remove or make configurable the hardcoded messages.slice(0, 25) cap, and add regression tests showing more than 25 read, list-pins, and search rows can be rendered when requested. Preserve the existing Teams Graph pagination coverage for list-pins and channel-list.
