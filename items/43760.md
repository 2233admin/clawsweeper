## Summary

Keep open. Current main still exposes `memorySearch.fallback` as a string provider id and only one shared `memorySearch.remote` config, and the fallback activation path passes that same resolved remote config to the fallback provider. The exact requested separate fallback remote/model config is therefore not implemented on main. A matching PR (#44878) exists in the provided context, but it is open/unmerged and a prior ClawSweeper review kept it open because main still lacks the behavior, so the issue should not be closed yet.
