## Summary

Keep #38520 open. Current main still has adjacent compaction safeguards, including silent memory flush, post-compaction reinjection, optional start/end user notices, and observe-only hooks, but it does not implement the requested configurable pre-compaction warning thresholds, visible structured handoff turn/window, or bounded audited deferral mechanism. The prior ClawSweeper review reached the same conclusion, and the touched compaction config/runtime/docs surfaces are unchanged since that review.

## Best Possible Solution

Keep this open for maintainer product/API design. A complete solution likely belongs in core compaction configuration and runtime event handling: validated warning/critical threshold config, defined channel/UI delivery for pre-compaction warnings, a structured pre-compaction handoff turn with timeout and prompt policy, and a bounded/audited deferral contract if maintainers want agents or plugins to pause imminent compaction. It should update config schema/types/help/docs, compaction runner/event paths, any plugin or tool contracts needed, and focused tests for threshold warnings, handoff timeout behavior, and deferral limits.
