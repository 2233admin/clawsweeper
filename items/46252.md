## Summary

Keep open. Current `main` partially fixed the report by counting `.jsonl.reset.*`/`.jsonl.deleted.*` in the global cost summary, but the per-session `sessions.usage` path still collapses each session ID to a single transcript and prefers the active `.jsonl` when it exists. That matches the reporter's April 4 reproduction: dashboard/session detail totals can still undercount after `/new` or `/reset`.
