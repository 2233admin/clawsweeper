## Summary

Keep #70554 open. Current main still has the reported memory-core dreaming behavior: request-scoped subagent runs fall back to a local snippet write, cleanup still uses the same request-scoped subagent, and the gateway subagent path does not forward `bootstrapContextRunKind`. The PR is recent, addresses a real related bug report (#67152), and its core work is still useful even though the branch needs maintainer review for unrelated/CI cleanup churn and review-comment follow-through.

## Best Possible Solution

Keep the item open and either finish this PR or replace it with a narrower maintainer patch that fixes #67152: memory-core should retry request-scoped narrative generation through a narrow host-owned gateway subagent accessor, forward `bootstrapContextMode` and `bootstrapContextRunKind` through plugin subagent dispatch, preserve cron/lightweight behavior across embedded and CLI execution, keep gateway runtime mutators out of public SDK barrels, and retain focused regression tests for non-transient rethrow plus transient cleanup fallback.
