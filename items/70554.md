## Summary

Keep #70554 open. Current main still has the reported memory-core behavior: request-scoped subagent failures in dreaming narrative generation fall back to a local diary snippet, and main still lacks the plugin-subagent run-kind propagation needed to run the retry as a lightweight cron-style background session. The PR is an active, focused attempt to fix open bug #67152 rather than obsolete work.

## Best Possible Solution

Keep #70554 open as the active fix attempt for #67152. The best path is a focused, green patch that retries memory-core dreaming narratives through a narrowly exposed gateway background subagent runtime, forwards `bootstrapContextMode` and `bootstrapContextRunKind: "cron"` through plugin subagent dispatch and both reply execution paths, preserves best-effort cleanup, and keeps focused regression coverage for retry, cleanup, and non-transient error behavior.
