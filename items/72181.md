## Summary

Keep #72181 open. Current main still lacks the requested BlueBubbles `trainerMode` / `trainerNotifyNumber` configuration and still routes allowed inbound DMs through the normal reply pipeline, which sends directly to the original BlueBubbles target. The issue is recent, concrete, authored by a non-maintainer, has no protected labels, and explicitly targets bundled BlueBubbles plugin behavior that needs owner/product review rather than cleanup closure.

## Best Possible Solution

Keep #72181 open for BlueBubbles owner/product review. A good implementation should stay under `extensions/bluebubbles`, add schema/type/docs/UI-hint coverage for `trainerMode` and `trainerNotifyNumber`, intercept eligible inbound DMs before normal reply dispatch, route draft and approval commands through existing BlueBubbles send paths, enforce pending-message TTL behavior, and add focused tests for both `training` and `supervised` modes.
