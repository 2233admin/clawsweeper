## Summary

Keep open. Current `main` still disables Feishu block streaming at the reply-dispatcher level, so the reported pre-tool text behavior is not implemented/fixed on `main`. The code also contradicts current docs: docs advertise `channels.feishu.blockStreaming`, but the Feishu config schema does not define that option and the dispatcher forces block streaming off.

## Evidence

- **Feishu dispatcher hard-disables block streaming:** `replyOptions.disableBlockStreaming` is set to `true`, which forces block streaming off for Feishu replies.
