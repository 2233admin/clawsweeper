## Summary

Keep open. The PR is a focused docs-only fix and current main still has the exact broken `docs/plugins/memory-wiki.md` QMD + bridge example: `memory-wiki` is nested under `memory`, even though the surrounding docs, config types, and plugin runtime contract all place plugin config under `plugins.entries.<id>.config`. The related Memory Wiki bridge runtime issues in the provided context are separate from this docs nesting correction.

## Best Possible Solution

Keep PR #70278 open and merge or reapply its narrow documentation fix so the QMD + bridge example nests `memory-wiki` under `plugins.entries`. No core/runtime change is needed for this item; the separate Memory Wiki bridge runtime/import issues should continue to be tracked by their canonical runtime follow-up rather than blocking this docs correction.
