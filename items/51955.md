## Summary

Current main already ships the plain-HTTP localhost Control UI compatibility this PR is trying to add, but via scope preservation during the insecure localhost auth path rather than by widening `hasExecApprovalClients()`. The behavior is documented and covered by tests, so this PR is obsolete on `main`.

## Close Comment

Closing after Codex review. Current `main` already covers the plain-HTTP localhost Control UI case this PR targets, but it does so by preserving operator scopes during the documented insecure-localhost auth path instead of widening `hasExecApprovalClients()`. The relevant behavior is in `src/gateway/server/ws-connection/connect-policy.ts:129` and `src/gateway/server/ws-connection/message-handler.ts:616`, the regression is covered in `src/gateway/server.auth.control-ui.suite.ts:317`, and the shipped localhost-only compatibility mode is documented in `docs/web/control-ui.md:264` and `docs/gateway/protocol.md:575`. This is present in the released `v2026.4.22` tree (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so the fallback change in this PR is no longer needed on `main`.
