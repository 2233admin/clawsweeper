## Summary

Keep this issue open. Current main has partial improvements for model fallback visibility and cron delivery previews, but it still does not implement the core production-operations requests: startup primary-model reachability before accepting turns, a compact agent-readable runtime model/fallback/last-error diagnostic surface, or cron add dry-run/preflight with fail-closed announce target validation before persistence.

## Best Possible Solution

Keep this open but narrow the maintainer path: add a configurable startup/provider preflight that can live-probe the configured primary model and either block startup or clearly mark the gateway degraded; add a safe agent-visible diagnostics RPC/tool exposing selected model, active responding model, fallback state, and last provider/fallback error category/code; and add cron add dry-run/preflight plus fail-closed announce target validation before job persistence. Treat the closed related fallback and cron issues as completed subwork, not proof that this production execution-discipline tracker is done.
