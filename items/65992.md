## Summary

Keep this PR open. Current main still derives local-file memory-wiki source page identity only from the normalized title slug and writes to `sources/<slug>.md`, so different inputs that share a slug can still overwrite each other. The PR remains a focused implementation candidate for the linked memory-wiki ingest bug; its provided diff is limited to memory-wiki ingest code and tests, with no CI, dependency, package, permissions, or secret-handling changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or refine a memory-wiki-owned fix that preserves readable source page names while adding a collision-resistant source identity suffix, reuses legacy source pages where possible, and keeps the regression tests focused on source page collisions.
