## Summary

Keep open. Current main still documents Notion usage with shell expansion (`NOTION_KEY=$(...)`, `$NOTION_KEY`) and backslash-continued `curl` snippets, while the current exec-approval docs and source still identify those forms as allowlist-miss territory under strict macOS allowlist policies. The two linked fix PRs, #59254 and #59870, were closed unmerged in the provided GitHub context, so there is no shipped canonical fix to close against.

## Best Possible Solution

Keep #59235 open and update `skills/notion/SKILL.md` to remove or clearly caveat shell-expanded, backslash-continued curl examples. The best fix is an allowlist-friendly Notion example that invokes one already-allowlisted executable, such as a small `python3`/stdlib helper or bundled script, plus a short note pointing strict macOS exec-approval users away from `$()`/`$VAR` shell wrappers.
