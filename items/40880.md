## Summary

Current main still has the reported sandbox/media staging cap as a hardcoded 5 MB default, and the requested user-configurable `media.maxBytes` field is not present in source types, zod schema, or docs/help metadata. The linked PR #40950 appears to be the focused fix path, but it is still open/unmerged in the provided GitHub context, so this should remain open until a fix lands or maintainers choose #40950 as the canonical tracker.

## Best Possible Solution

Keep #40880 open as the focused tracker for sandbox/media store staging configurability. The best code path is to land or rework #40950: add a bounded top-level `media.maxBytes` contract to config types, zod schema, generated/help/docs surfaces, thread it into media store/server and sandbox staging at gateway startup or runtime resolution, preserve the 5 MB default when unset, and add regression tests proving files over 5 MB stage when configured while oversized files still fail at the configured cap. Treat #67031 as a broader companion for other image/media limits, not a replacement for this sandbox-specific bug.
