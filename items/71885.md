## Summary

Keep open. Current main has not implemented the proposed SQLite hardening: the task registry store still binds `record.task` directly, and there is no structured-task regression in the current store tests. The PR is small and plausibly useful, but current public/core task contracts still type `task` as a string, so maintainers should decide whether this is defensive storage hardening or whether the author should prove and document a runtime path that can pass structured values.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either to accept a narrow defensive serialization fix with the regression test if core wants SQLite to tolerate non-string runtime values, or ask the author to identify the current runtime path that bypasses the string contract and update the relevant task/plugin API contract if structured task payloads are intended.
