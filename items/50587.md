## Summary

Keep open. Current `main` still does not pass inbound channel `MediaPath`/`MediaPaths` images into `GetReplyOptions.images` on the normal auto-reply path, so the reported bug remains real outside surfaces like webchat that already populate `replyOptions.images`. The PR addresses a valid gap, and the review discussion surfaced at least one remaining safeguard gap (matching ACP's size-limited attachment loading), so conservative cleanup should not close it.

## Evidence

- **Dispatch path still forwards only pre-supplied images:** `dispatchReplyFromConfig` sends `images: params.replyOptions?.images` into the reply-dispatch hook and does not derive any images from `ctx.MediaPath`/`ctx.MediaPaths`, so ordinary inbound channel attachments are still dropped before the agent run.
