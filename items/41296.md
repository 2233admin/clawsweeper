## Summary

Keep open. Current `main` has independent websocket/HTTP shutdown timeout handling, but it does not implement this PR's broader shutdown-observability work. The gateway close path still silently ignores many teardown failures, returns no structured shutdown result, and has no final shutdown summary/warning aggregation.
