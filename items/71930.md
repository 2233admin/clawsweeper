## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before the existing post parsing, mention gate, and agent wake path. Mattermost is a bundled supported plugin, and the report is a concrete current bug rather than ClawHub scope or a stale/duplicate cleanup candidate.

## Best Possible Solution

Keep the issue open and fix it in the bundled Mattermost plugin: parse `post_edited` WebSocket payloads as updated posts, feed them through the existing `handlePost` mention gate and wake flow, adjust the replay guard so edits are not suppressed by the original post id, and add focused tests for edited-to-mention wake plus non-actionable edit dedupe.

## Evidence

## Risks / Open Questions

- The eventual fix must avoid duplicate wakes for edits that do not newly introduce an actionable mention.
- Current replay dedupe keys are based on account id plus post id, so edit handling may need event- or revision-aware dedupe to prevent edited posts from being treated as already processed.
