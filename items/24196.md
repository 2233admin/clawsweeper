## Summary

Current `main` still sends Discord outbound media as one attachment per message, and the docs/tests explicitly reflect that behavior. I did not find evidence that issue #24196 has been implemented on `main`.

## Evidence

- **Discord media send path builds a single-file payload:** `sendDiscordMedia` always constructs `files` with exactly one entry before posting to `Routes.channelMessages(channelId)`, so a single outbound Discord message still only carries one uploaded attachment.
  - file: [extensions/discord/src/send.shared.ts:443](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/extensions/discord/src/send.shared.ts#L443)
  - command: `nl -ba extensions/discord/src/send.shared.ts | sed -n '420,470p'`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Generic outbound reply helper still fans out media URLs one-by-one:** `sendPayloadWithChunkedTextAndMedia` sends `urls[0]` first, then loops remaining URLs and calls `sendMedia` again with `text: ""`, which preserves multi-media support only as multiple sends, not multiple attachments in one Discord message.
  - file: [src/plugin-sdk/reply-payload.ts:197](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/plugin-sdk/reply-payload.ts#L197)
