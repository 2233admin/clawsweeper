## Summary

Keep open. Current `main` still appears susceptible to this Slack cascade: Slack inbound handling accepts bot-authored messages from other bot accounts when `allowBots: true`, and pre-reply OAuth/auth failures still produce outbound channel-visible error text. I did not find any Slack-side tagging, dedupe, or suppression path for gateway-generated error posts.
