## Summary

Keep open. Current `main` still has `exit 1` inside `scripts/install.ps1` `Main`, so the reported PowerShell-session bug is not implemented on `main`. The PR patch shown in the provided GitHub context also appears incomplete because replacing those exits with `return $false` and calling `[void](Main)` would suppress failure signaling for `powershell -File`/`pwsh -File` runs.
