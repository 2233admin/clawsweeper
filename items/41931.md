## Summary

Keep open. Current `main` still matches the original bug report: `scripts/install.ps1` calls `exit 1` from inside `Main` while the public docs still tell Windows users to run the installer through `iwr ... | iex`, so this is not already implemented on `main`. At the same time, the PR's proposed `[void](Main)` entrypoint would drop failure signaling for direct `powershell -File` / `pwsh -File` runs, and existing Windows smoke coverage in the repo expects installer failures to propagate through `$LASTEXITCODE`. The PR identifies a real bug, but the patch is not conservative enough to close or merge as-is.
