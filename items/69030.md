## Summary

Keep #69030 open. Current main partially mitigates the reported context leak by using `modelRun: true`, `promptMode: "none"`, and disabled tools, but `openclaw infer model run` still has no `--raw` flag or `model.raw` capability and still routes through `agentCommand`/gateway `agent` using the default agent/session path rather than a direct simple-completion call.

## Best Possible Solution

Keep the issue open and implement an explicit raw path in core, either as `openclaw infer model run --raw` or a new `model.raw` capability. The implementation should bypass `agentCommand`/gateway `agent`, avoid default-agent session writes, use the existing simple-completion model preparation/completion helpers, return provider/model attribution, and add CLI metadata, protocol/docs, and tests proving raw mode does not load agent prompt policy, tools, skills, MCP runtimes, or session transcript state while preserving the current agent-wrapped behavior for backward compatibility.
