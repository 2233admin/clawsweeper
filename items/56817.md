## Summary

Current `main` has session switching/listing primitives and an internal child-session API, but I did not find a complete, user-facing conversation branch-management feature that clearly satisfies the issue as filed.

## Evidence

- **TUI supports switching among sessions, not branch management:** The TUI docs describe `/session <key>`, a session picker, and switching between existing sessions for the current agent.
  - file: [docs/web/tui.md:59](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/web/tui.md#L59)

  - file: [docs/cli/sessions.md:8](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/cli/sessions.md#L8)

## Risks / Open Questions

- There may be another UI surface that calls `sessions.create` which was not obvious from the code/docs inspected.
- Some users could treat arbitrary session keys as ad hoc branches, but that is not clearly documented or surfaced as the requested feature.
