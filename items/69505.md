## Summary

Keep open. Current `main` does not implement an `assistantPrefix`/prefill field on cron `agentTurn` payloads, agent defaults, or the `openclaw agent` / `openclaw cron add|edit` CLI surfaces. The only nearby feature is delivery-layer `messages.responsePrefix`, which prepends outbound payload text after generation and does not provide the protocol-level start-of-response guarantee requested here, especially when block streaming can emit multiple channel messages.
