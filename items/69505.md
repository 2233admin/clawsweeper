## Summary

Keep #69505 open. Current main does not implement an `assistantPrefix`/provider prefill field for cron `agentTurn`, agent defaults, Gateway `agent` calls, hook dispatch, or the CLI. The existing `messages.responsePrefix` surface is only outbound delivery decoration after model output exists, so it does not solve the reported protocol-level need to force generation to begin with fixed text or prevent streamed narration from becoming separate downstream messages.

## Best Possible Solution

Keep the issue open for maintainer product judgment and a focused additive implementation. If accepted, add an `assistantPrefix` or equivalent field across cron payload types, Gateway protocol schemas, hook dispatch, agent defaults/config schema/help, CLI flags, and runtime execution options; then pass it into provider wrappers with at least Anthropic plus one OpenAI-compatible implementation and regression tests that distinguish provider prefill from the existing post-generation `messages.responsePrefix`.

## Risks / Open Questions

- Do not treat `messages.responsePrefix` as a full workaround: it prefixes delivered output after generation and can still allow unwanted narration payloads through.
- Provider-native prefill behavior differs by transport, so an implementation should verify Anthropic, OpenAI-compatible, and Ollama request contracts before wiring a shared field.
