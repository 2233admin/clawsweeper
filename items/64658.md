## Summary

Keep open. Current main still does not expose a documented `sessions_spawn` policy to suppress or redirect native run-mode sub-agent completion announcements for channel-bound sessions. ACP-only `streamTo: "parent"`, nested-subagent internal delivery, and the hidden internal `expectsCompletionMessage` boolean do not satisfy the requested `none | parent | channel` caller control.

## Best Possible Solution

Add a documented `sessions_spawn` completion delivery policy for native sub-agent runs, preserving current channel delivery as the default. A small enum such as `completionDelivery: "channel" | "parent" | "none"` should thread through the tool schema, native spawn params, registry/task delivery status, announce dispatch, docs, and focused regression tests for channel-bound sessions such as BlueBubbles/iMessage, WhatsApp, and Telegram. Reuse the existing nested-subagent internal injection path for parent-only delivery where practical.
