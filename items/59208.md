## Summary

Keep open. Current main still does not implement the PR's central fix: the status auth label receives `sessionEntry.authProfileOverride`, but provider usage resolution has no preferred-profile plumbing, so usage can still be resolved from provider-level profile order. The PR remains relevant, but it needs a current-main rebase/update because status rendering moved to `src/status/status-text.ts` and the provider-override path raised in review still needs coverage.

## Best Possible Solution

Keep this PR or the linked #58498 work open, then update the patch on current main: pass the active session OAuth profile from `src/status/status-text.ts` into `loadProviderUsageSummary`, thread a provider-validated preferred profile through `resolveProviderAuths` and OAuth resolution, preserve fallback order for stale/cross-provider overrides, and add regression tests for multiple `openai-codex` profiles plus plugin provider-override usage auth such as MiniMax.
