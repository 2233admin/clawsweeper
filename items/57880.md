## Summary

Current `main` now rejects duplicate inbound channel messages before dispatch, rejects redelivered follow-up queue items by `messageId`, and has regression coverage for both the duplicate-message path and billing/rate-limit fallback behavior. That is strong evidence that issue #57880's reported retry-loop duplication is already fixed on `main`.
