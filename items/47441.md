## Summary

Keep open. Current `main` does not implement the reported Android/Termux sharp recovery: `openclaw update` still performs a generic global package-manager install, non-macOS image processing still depends on `sharp`, there is no `fix-native-modules` command, and the image tool path still goes through the sharp-backed optimizer instead of falling back to raw image bytes.

## Evidence

- **`openclaw update` still uses the generic package install path:** `runPackageInstallUpdate()` resolves the global manager, builds a generic install env, and executes `globalInstallArgs(...)` for the update. There is no Android/Termux-specific rebuild or post-install repair hook in this flow.
  - file: [src/cli/update-cli/update-command.ts:339](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/cli/update-cli/update-command.ts#L339)
  - command: `nl -ba src/cli/update-cli/update-command.ts | sed -n '339,385p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Global install env/argv contain no sharp or Termux handling:** `createGlobalInstallEnv()` only merges PATH/Corepack/Windows env, and `globalInstallArgs()` emits plain `npm i -g`, `pnpm add -g`, or `bun add -g` commands. No `SHARP_IGNORE_GLOBAL_LIBVIPS`, Termux toolchain env, or native-module rebuild step is added here.
  - file: [src/infra/update-global.ts:334](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/infra/update-global.ts#L334)
