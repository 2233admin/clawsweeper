## Summary

Keep open. Current main still has the reported secrets-audit false-positive paths for bare `$VAR` references and auth-profile inline refs, and #54141 is still the open implementation candidate with closing intent. Some one-way apply/scrub behavior exists, but the central audit issue and the configure type-mismatch path are not resolved on main.

## Best Possible Solution

Keep this issue open while #54141 is reviewed and either landed or closed. The best maintainer path is a focused core fix with regression tests for bare `$VAR` and `${VAR}` env references in config and auth-profile audit paths, plus an explicit follow-up or product decision for OAuth/configure type-mismatch handling, `.env` audit behavior, and post-migration rotation/scrubbing guidance.
