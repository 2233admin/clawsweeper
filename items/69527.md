## Summary

Keep this issue open. Current main has some OpenRouter model-ref compatibility work, but the reported `infer model run --json` diagnostic gap is still present: both local and gateway model-run envelopes hard-code `attempts: []`, while the fallback runtime below them records failed candidates. The related PR #69580 was closed unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Keep this issue open. The best maintainer path is a focused OpenRouter/model-run regression fix: live-verify `openrouter/auto` and `openrouter/@preset/...`, add regression coverage for configured default model selection through `openclaw infer model run`, and propagate `runWithModelFallback` attempt details through `agentCommand`, gateway responses, and the CLI JSON envelope for both local and gateway transports.
