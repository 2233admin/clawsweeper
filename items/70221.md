## Summary

Keep #70221 open. Current main still has the reported Control UI feature gap: `[embed]` supports and applies height, but there is no first-class parsed, typed, normalized, documented, or rendered width parameter for hosted/canvas embeds. This is a core Control UI rich-output contract, not a ClawHub/plugin cleanup item, and the provided GitHub context shows no protected labels or maintainer author signal that would block normal triage.

## Best Possible Solution

Keep the issue open and implement a first-class Control UI embed width contract: add a normalized/clamped `preferredWidth` or equivalent field to the canvas preview/UI/sidebar types, parse it from `[embed width="..."]` and JSON preview payloads if desired, apply it in inline and sidebar iframe rendering with responsive overflow/max-width behavior, and update Hosted embeds/rich-output docs plus tests alongside the existing height coverage.
