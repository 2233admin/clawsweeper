## Summary

Keep open. Current main still scans only JS/TS-family skill/plugin source files, so markdown skill definitions remain outside the built-in scanner. This PR is a focused security-hardening candidate, but its critical markdown download-and-exec rule still has practical false negatives that should be fixed before merge.

## Best Possible Solution

Keep this PR open as the active core security-hardening candidate. Ask for a revision that broadens the markdown download-and-exec detection, adds regression tests for the bypass variants already identified in review, and then runs the relevant changed gate before merge.
