## Summary

Keep open. Current main still excludes markdown from the built-in skill/plugin scanner, while this PR is a focused security-hardening candidate for that gap. It should not be merged as-is because the proposed critical markdown download-and-exec rule still has practical false negatives, and the PR is currently dirty against main.

## Best Possible Solution

Keep this PR open as the active core security-hardening candidate. Ask for a revision that broadens `markdown-download-exec` detection, adds regression tests for sudo-wrapped shells, absolute-path shells, versioned interpreters, and line-continuation pipelines, rebases onto current main, and then runs the relevant changed gate before merge.
