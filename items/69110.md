## Summary

Keep #69110 open. Current main still does not pass resolved provider/model provenance into the `message_sending` delivery hook or otherwise enforce truthful model tags at the gateway delivery boundary. The related open PRs are relevant coordination points, but they are not merged on main and do not make this issue obsolete yet.

## Best Possible Solution

Keep the issue open and implement a core provenance path: carry the resolved provider/model/run identity from the agent result through final reply assembly into the outbound delivery hook surface, then add tests for fallback-model mismatch. PR #65149 is the closest related direction because it proposes a final reply hook with `providerUsed` and `modelUsed`, but it should be reviewed as part of the broader delivery provenance design rather than treating this issue as already resolved.
