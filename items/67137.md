## Summary

Current main still does not implement this PR's central hardening: `createClackPrompter().text` returns the Clack result directly despite the OpenClaw `WizardPrompter.text` contract being `Promise<string>`, and the channel setup paths still trim raw prompt results. The PR is a focused bug-fix candidate with regression tests and no visible security-sensitive file changes, so it should remain open for maintainer review rather than be auto-closed.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt a small boundary fix that makes `createClackPrompter().text` always satisfy the existing `Promise<string>` contract, with regression coverage for undefined prompt results; downstream setup guards should be retained only where maintainers want defense-in-depth without changing required-field behavior.
