## Summary

Current `main` does not provide high-confidence evidence that this issue is fixed or obsolete. The reported "image data removed" theory does not match current pruning logic for active/recent turns, but the code still documents an unresolved attachment-order limitation in the image pipeline, and the reporter's Xiaomi model (`mimo-v2.5-pro`) does not match the current built-in Xiaomi vision catalog. Conservative cleanup should leave this open for maintainer triage or a real reproduction against current `main`.
