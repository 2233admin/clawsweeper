## Summary

Keep open. Current main still has unsafe non-array `content` handling in the OpenAI Responses/WebSocket conversion path implicated by the report, and the related fix PRs in the provided context were closed unmerged. Main has some replay-history normalization for legacy string assistant content, but that does not cover malformed provider response output before it is converted into an assistant message.

## Evidence

- **Related fixes did not land:** Provided GitHub context shows PR #43836 and PR #43864 both targeted #43795, but both are closed and unmerged.
