## Summary

Keep open. The report has concrete Feishu/GptGod GLM logs and prior root-cause discussion, while current main still has unguarded non-array assistant content handling in the OpenAI-compatible response/gateway paths. The proposed fix PRs are closed unmerged, and the shipped code only covers some replay-string normalization cases.

## Best Possible Solution

Keep this issue open and land a focused core fix at the OpenAI-compatible ingress points: normalize or safely reject non-array assistant content in response conversion and gateway handling, preserve existing string replay repair, and add regression tests for string, object, and null content shapes. #43864 is useful prior analysis but should be reviewed or reimplemented against current main rather than treated as shipped.
