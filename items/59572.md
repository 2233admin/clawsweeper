## Summary

Keep open. Current main still does not implement the proposed interactive TUI context visualization: the TUI only offers a mode selector for text `/context list|detail|json` output or forwards explicit `/context ...` commands to the Gateway. The PR is a plausible core TUI UX feature, not a ClawHub/plugin-only item and not a duplicate of the provided related reports, but its implementation still needs maintainer/contributor follow-up because the PR discussion includes unresolved correctness and UX review comments.

## Best Possible Solution

Keep the PR open for maintainer/contributor handling. The best path is to revise the TUI implementation against current main: add a local `/context viz|visualize` overlay or equivalent TUI visualization, preserve existing `/context list|detail|json` Gateway behavior, use a lightweight or cached data path if needed, add focused TUI tests, and resolve the existing review comments before merge.
