## Summary

Keep #72097 open. Current main still refreshes the persisted/cold plugin registry after install and tells users to restart the gateway, while agent runs build the plugin tool list once during session creation and pass that fixed customTools/allowlist into the Pi session. I found active-registry reuse for gateway-bindable tool construction, but no registry-change broadcast, toolset delta, or per-turn active-session tool refresh that would inject newly installed plugin tools into an already-created agent session.

## Best Possible Solution

Implement this as a core plugin-runtime and agent-session lifecycle improvement. The likely fix is to make plugin install/registry refresh notify active agent sessions, or to re-resolve plugin tools at the next turn boundary, then update customTools, the active tool allowlist, and any prompt/tool inventory state before the next model call. Add regression coverage for a tool-providing plugin install followed by immediate use in an existing session, while preserving the existing gateway-bindable active-registry reuse behavior.
