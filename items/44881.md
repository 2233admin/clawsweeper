## Summary

Keep open. Current main has adjacent hardening for OpenClaw-managed services and clearer EADDRINUSE guidance, but it does not implement the reported PM2/manual `npm install -g` handoff: package lifecycle scripts do not stop/restart the gateway, stale-listener cleanup is gated to OpenClaw service mode, and supervisor detection covers launchd/systemd/schtasks but not PM2. The related #45943 tracks direct package-manager auto-restart more broadly, but this issue also has PM2-specific stop/respawn behavior from the discussion, so it is still actionable rather than safely duplicate-closable.
