## Summary

Current main already implements restart-safe WebChat session recovery: the gateway now publishes canonical session defaults in the hello snapshot, the UI normalizes/preserves the active session across reconnects and refreshes chat on hello, chat history is reloaded from persisted transcript storage, and the client retries transient chat.history unavailability during gateway startup after restart.
