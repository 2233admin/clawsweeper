## Summary

Keep open. Current `main` does not implement non-thread persistent ACP affinity for cron/orchestrator sessions. ACP `sessions_spawn` still exposes only `mode: "run" | "session"`, rejects persistent ACP sessions unless `thread=true`, generates a fresh random ACP session key on each spawn, and the docs/tests still describe persistence as thread-bound only.

## Evidence

- **ACP spawn tool schema has no affinity key input:** `sessions_spawn` for ACP accepts `runtime`, `agentId`, `resumeSessionId`, `cwd`, `thread`, `mode`, `sandbox`, `streamTo`, etc., but no `affinityKey` or similar persistent non-thread reuse field.
