## Summary

Keep open. Current `main` still treats Slack interactive replies/buttons as opt-in, not default-on, and the latest shipped release says the same. The issue's proposed implementation is also partially outdated: Slack now uses `interactiveReplies` and shared `presentation` support rather than a Slack `inlineButtons` config flag, so the report likely needs a maintainer to decide whether to change the default, fix the prompt wiring, or both.

## Risks / Open Questions

- The issue text and proposed fix are stale relative to current architecture: Slack now uses `interactiveReplies` plus shared `presentation` rendering, not a Slack `capabilities.inlineButtons` config key.
- Slack already exposes generic `presentation` support for `message action=send`, so the remaining gap may be specifically Slack-only interactive reply directives and/or incorrect system-prompt guidance rather than basic button rendering.
