## Summary

Keep PR #72251 open. Current main still uses a single `tar.c()` write over the planned backup sources with no volatile-path filter, EOF retry/backoff wrapper, enriched final EOF error, runtime log notices, or `skippedVolatileCount` output. The linked issue #72249 is closed as superseded by this PR, while #67417 and #67990 remain adjacent but do not replace this focused fix.

## Best Possible Solution

Keep PR #72251 open for maintainer review. The best path is a rebased, narrowly scoped backup-archiver fix on main that adds a built-in volatile-path filter, tar EOF retry with partial temp archive cleanup, enriched final errors, `skippedVolatileCount` stdout/JSON reporting, runtime log notices, focused regression tests, and docs clarification for skipped live artifacts. Track #67417 and #67990 separately unless maintainers choose to consolidate broader backup consistency design.
