## Summary

Keep open. Current `main` still builds Feishu text messages as `post`/`md` payloads with the raw message text, so bare URLs containing underscores are still handed to Feishu markdown unchanged. The same implementation is present in the latest shipped tag `v2026.4.22`, and the existing Feishu send tests only assert the unescaped `md` payload shape rather than covering underscore-containing URLs.
