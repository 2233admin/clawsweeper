## Summary

Keep this issue open. Current main still has the live deadlock shape described: streamed tool-result delivery tasks are collected in `pendingToolTasks`, and `runReplyAgent` waits on them with an unbounded `Promise.allSettled()` before the reply operation reaches cleanup. Existing transcript/session repair covers some replay recovery, but it does not bound the active live drain. The related PR #64733 remains open and is not a shipped fix.

## Best Possible Solution

Keep this issue open and resolve it by reviewing, landing, or superseding PR #64733 with an equivalent main-branch fix: add a bounded timeout or cancellation path around the `pendingToolTasks` drain, log/surface the timeout, ensure reply-operation cleanup runs so the session lane can recover, and add focused regression coverage for a never-settling tool-result delivery task.
