## Summary

Keep open. Current `main` does not implement Control UI quote-reply/threaded-reply composition. The UI can only display reply metadata already present on messages, while the Control UI send path and chat RPC schema do not expose reply-target fields or a reply action.

## Evidence

- **Control UI send path has no reply-target parameters:** `requestChatSend` sends only `sessionKey`, `message`, `deliver`, `idempotencyKey`, and optional attachments. There is no `replyToId`/`replyToCurrent` in the Control UI request path.
  - file: [ui/src/ui/controllers/chat.ts:460](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/controllers/chat.ts#L460)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '460,470p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Gateway chat schema does not accept reply-target fields for Control UI chat.send:** `ChatSendParamsSchema` defines the WebChat/Control UI RPC payload and includes message/session metadata, but no `replyToId` or `replyToCurrent` fields.
  - file: [src/gateway/protocol/schema/logs-chat.ts:35](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/protocol/schema/logs-chat.ts#L35)
  - command: `nl -ba src/gateway/protocol/schema/logs-chat.ts | sed -n '35,52p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Message bubble actions are limited to existing expand/copy controls:** In grouped chat rendering, bubble actions are gated by `canCopyMarkdown || canExpand`, and the rendered controls are only the expand button and copy-as-markdown button. No reply affordance is present.
  - file: [ui/src/ui/chat/grouped-render.ts:1354](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/chat/grouped-render.ts#L1354)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Current UI only renders reply metadata already embedded in messages:** `renderReplyPill` shows a passive "Replying to ..." pill when a normalized message already has a `replyTarget`; this is display-only and does not provide quote-reply composition.
  - file: [ui/src/ui/chat/grouped-render.ts:728](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/chat/grouped-render.ts#L728)
