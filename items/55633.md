## Summary

Current main already turns the missing/mis-stamped gateway config case into an early startup guard and, for systemd-managed installs, prevents restart loops on that config-exit path. That addresses the repeated missing-config log spam described in #55633.

## Evidence

- **Startup guard for missing config:** `getGatewayStartGuardErrors()` returns the exact missing-config / missing-`gateway.mode` messages, and `runGatewayCommand()` emits them once then exits with `EXIT_CONFIG_ERROR` before starting the gateway server.
  - file: [src/cli/gateway-cli/run.ts:223](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/cli/gateway-cli/run.ts#L223)

## Risks / Open Questions

- This appears to be fixed on current `main` but not in the latest tagged release `v2026.4.22`, so users on the release build may still see the older behavior until a newer release ships.
- The strongest restart-loop proof here is for systemd-managed installs via `RestartPreventExitStatus=78`; this review did not verify equivalent suppression semantics for every supervisor platform.

## Close Comment

Closing from a Codex review because current `main` already handles this case. The gateway now treats missing or clobbered local-mode config as a startup guard in `src/cli/gateway-cli/run.ts:223` and exits with config status 78 before the server starts (`src/cli/gateway-cli/run.ts:542`). For systemd-managed installs, the generated unit also sets `RestartPreventExitStatus=78` in `src/daemon/systemd-unit.ts:79`, which stops the supervisor from relaunching the same failing startup over and over. There is regression coverage for the missing-`gateway.mode` path in `src/cli/gateway-cli/run.option-collisions.test.ts:297`. This is present on `main` at commit `4630ce3d9e2f089062ccd386b0ae241325961d1a`; I could not confirm a released tag containing it yet, so `fixedRelease` is left null. If this is still reproducible on a newer build from `main`, especially on a non-systemd supervisor, please open a fresh issue with the supervisor/platform details and the current restart log output.
