## Summary

Keep open. Current main has strong evidence that Discord path/file URL media sends now route through a real upload path, matching the related #53163 cleanup context, but #44555 explicitly asks for buffer-based Discord attachment sends too. The current Discord message action path still only reads media/path/filePath references and does not consume buffer/mimeType as an attachment source, so the issue is only partially implemented.
