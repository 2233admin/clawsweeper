## Summary

Keep #72355 open. Current main appears to address the fatal CIAO crash-loop path, but the concrete hostname-selection request remains unresolved: Bonjour still defaults the advertised hostname and TXT `lanHost` to `openclaw`/`openclaw.local` unless `OPENCLAW_MDNS_HOSTNAME` is set, and there is no `discovery.mdns.hostname` config key. Related closed reports cover the crash-loop mitigation, while PR #72055 concerns the service instance name rather than the advertised host/lanHost.

## Best Possible Solution

Keep the issue open but narrow it to Bonjour hostname selection/configuration. A good fix would add a plugin-owned or gateway-config-backed hostname override, or deliberately switch the default to a sanitized system hostname, then align `MdnsDiscoveryConfig`, the strict zod schema, schema help/generated docs, public gateway docs, and Bonjour tests while preserving the existing CIAO cancellation suppression and bounded advertiser recovery.
