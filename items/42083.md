## Summary

Keep open. Current `main` at `7ac35b4f69009490180a35bf404638d4280a4865` still reprocesses the full `agent_end` message history in `extensions/memory-lancedb/index.ts`, so the PR’s stated bug is not already implemented on main. The current test coverage exercises one-shot auto-capture, but I did not find a regression test for multi-turn "only new messages" behavior.

## Evidence
