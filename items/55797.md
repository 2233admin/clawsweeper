## Close Comment

Closing from a Codex review because current `main` already includes the post-`2026.3.24` fixes that match this report, and they shipped in `v2026.4.22`. The release notes call out the relevant areas directly: `CHANGELOG.md:166` fixes `models auth login` model-map clobbering, `CHANGELOG.md:205-206` adds last-known-good config recovery for clobbered `openclaw.json`, `CHANGELOG.md:517-520` moves Codex CLI OAuth imports to a runtime-only overlay instead of mutating persisted auth state, and `CHANGELOG.md:543` fixes the container/sidecar restart loop where stale-gateway cleanup could kill the active gateway. The current code matches that: `src/agents/auth-profiles/store.ts:151-233` overlays external auth instead of writing it back during normal runtime loads, `src/agents/auth-profiles.readonly-sync.test.ts:43-76` locks that behavior in, and `src/infra/restart-stale-pids.ts:111-127` / `:203-208` now exclude ancestor PIDs in container topologies. If this still reproduces on `v2026.4.22` or newer, open a fresh issue with current logs plus the newest `openclaw.json.clobbered.*` and `config-audit.jsonl` entries.
