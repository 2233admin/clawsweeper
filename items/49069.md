## Summary

Keep open. Current `main` does not implement configurable timestamps for TUI chat messages, and the PR’s documented config shape is still not part of the strict config schema.

## Evidence

- **TUI chat log still renders plain system/user/assistant text:** `ChatLog` currently renders system text via `theme.system(text)`, user text via `new UserMessageComponent(text)`, and assistant text via `new AssistantMessageComponent(text)`/`setText(text)`. There is no timestamp formatter, toggle, or prefix logic on `main`.
  - file: [src/tui/components/chat-log.ts:94](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/tui/components/chat-log.ts#L94)

## Risks / Open Questions

- The open PR appears stale against current TUI internals: `ChatLog` now has pending-user/history-reconciliation behavior that any timestamp implementation must preserve.
- A future implementation should use persisted message timestamps during history replay instead of stamping replayed messages with the current local clock.
