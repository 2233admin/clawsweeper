## Summary

Keep open. Current main still does not parse `[[react:...]]` / `[[react_to_current:...]]` during outbound normalization, does not carry Telegram reaction intent through `channelData.telegram.reaction`, and the Telegram outbound paths still reject or send only text/media payloads. The PR is a focused draft for the still-open linked bug #71140 rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and review it as the active implementation candidate for #71140. The best path is to land a maintained version of this fix or an equivalent one that parses reaction directives in shared outbound normalization, preserves structured Telegram reaction data, allows reaction-only replies with a valid target, sends native Telegram reactions before optional text/media, and includes regression coverage in outbound payload and Telegram adapter tests.
