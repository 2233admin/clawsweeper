## Summary

Keep open. Current main still lacks the requested Telegram reaction-directive path: reply directive parsing does not recognize `[[react...]]`, shared outbound normalization does not convert it into `channelData.telegram.reaction`, Telegram reply delivery still rejects no-text/no-media replies, and the outbound adapter does not call `reactMessageTelegram` for payload channel data. The PR is a focused, recent draft against an open linked bug (#71140), so it remains useful for maintainer review rather than conservative cleanup.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent fix: parse `[[react:...]]` / `[[react_to_current:...]]` into structured Telegram channel data, preserve that data through outbound normalization, allow reaction-only Telegram replies when a reply target exists, and have the Telegram outbound adapter call `reactMessageTelegram` before optional text/media sends with focused regression tests.
