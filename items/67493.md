## Summary

Keep open. This PR is not implemented on `main`, and the submitted change is intentionally incomplete: the author says the chat-lifecycle wiring is deferred to a follow-up PR, while current `main` still only has APNs support for `push.test`, `node.wake`, and exec-approval notifications. The review discussion also surfaces substantive correctness issues in the proposed helper.

## Risks / Open Questions

- The PR as submitted would add dead code unless the deferred chat-lifecycle wiring lands with it.
- The proposed implementation appears to reuse `sendApnsAlert(...)`, which currently emits `openclaw.kind = "push.test"`; that would misclassify chat-reply notifications on iOS.
- The proposed main-session gate described in review discussion is likely too strict for canonical session keys such as `agent:<defaultAgent>:<mainKey>` or `global`.
