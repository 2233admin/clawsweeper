## Summary

Keep this issue open. Current main still exposes only `systemPromptOverride`, which is a full replacement path, not the requested additive `agents.defaults.systemPrompt` / `rules` and per-agent prompt/rules config surface. The related PR is closed and unmerged, and the related duplicate issue is closed rather than an active canonical tracker.

## Best Possible Solution

Implement a distinct core additive prompt contribution surface: typed and schema-backed `agents.defaults.systemPrompt` and/or `agents.defaults.rules`, plus per-agent fields with documented merge semantics. Thread the resolved contribution through the shared prompt-building path, update generated schema/docs/config metadata and any Control UI or config mutation allowlists, and keep `systemPromptOverride` as the separate full replacement mechanism.
