## Summary

Keep open. The issue reports garbled CJK text in the WebUI thinking/working output card, and current main still renders that content with `.chat-thinking` while the `.chat-thinking` rule has no explicit body/CJK-capable font-family. The linked PR #43843 directly targeted this issue but was closed unmerged, so it does not supersede or implement the fix on main.
