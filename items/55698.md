## Summary

Keep open. Current `main` has partial browser-tab cleanup, but it does not fully cover the issue as reported: cleanup is wired for `sessions.reset`/`sessions.delete`, isolated cron teardown, and subagent completion, while `sessions_yield` has no browser-cleanup hook, there is no browser tab idle-timeout config, and node/proxy browser opens bypass session tab tracking entirely.

## Risks / Open Questions
