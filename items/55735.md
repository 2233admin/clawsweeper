## Summary

Keep open. Current main gives the agent stronger docs/source/status guidance for OpenClaw diagnosis, but it does not actually provide OpenClaw-version-aware self-knowledge in the agent prompt, and the suggested generic `openclaw explain <topic>` surface is not present.

## Evidence

- **Docs/source-first diagnostic guidance exists:** The agent system prompt now tells the model to consult OpenClaw docs first, inspect source if docs are stale or incomplete, and run `openclaw status` itself when diagnosing issues. That is a mitigation, but it is instruction-based rather than embedded version/change awareness.
