## Summary

Keep this PR open, but do not land it unchanged. Current main still lacks a fail-fast startup guard for missing default model provider auth before channel startup, while the PR is stale against the current post-attach startup split and uses a narrower auth check than runtime model execution accepts.

## Best Possible Solution

Keep this PR open for revision. Rebase or rewrite it onto src/gateway/server-startup-post-attach.ts, run the guard before channel startup while honoring OPENCLAW_SKIP_CHANNELS and OPENCLAW_SKIP_PROVIDERS, and base the decision on the same non-mutating auth contract used by runtime execution, such as hasAvailableAuthForProvider or a safe resolveApiKeyForProvider wrapper supplied with cfg and agentDir. Add focused tests for missing auth, config-backed apiKey auth, Bedrock/aws-sdk auth, synthetic local/plugin auth, skip-channel bypass, and non-circular remediation text.
