## Summary

Keep open. Current main still does not implement the PR's central behavior: web chat run errors are not appended as inline assistant error bubbles, persisted assistant error-only history entries still have no renderable text path, and the UI has no error bubble styling or regression coverage. The PR remains a plausible focused bug fix, though it needs rebase/adaptation for current main.

## Best Possible Solution

Keep the PR open or fold the narrow behavior into a maintainer-owned follow-up. The implementation should append structured assistant error messages for live chat error events, preserve/render persisted assistant `stopReason: "error"` entries including empty-detail fallbacks, avoid filtering those entries as heartbeat acknowledgements, style them distinctly in web chat, and add controller/extraction/rendering tests against current main's `normalizeMessage` and grouped-render path.
