## Summary

Keep this PR open. The original runtime-model display bug was shipped on main by #55425, but the follow-up problem raised in this PR is still real: `session_status` has only one model path, so using the persisted runtime model as the selected model can hide active fallback state.

## Best Possible Solution

Keep this PR open or replace it with a focused follow-up. Preserve the shipped #55425 runtime-model behavior for ordinary sessions, add a display-only runtime model path through the current `src/status` status-text/status-message interfaces, keep selected-model semantics intact during active fallback, derive fallback selected refs from `fallbackNoticeSelectedModel` or the actual routed selection rather than the global default, and add tests for runtime-only display, unknown-provider preservation, active fallback visibility, and channel/group-selected fallback sessions. Rebase the implementation onto the current `src/status` module split before merge consideration.
