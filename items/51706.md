## Summary

Keep open. Current `main` already includes the base runtime-model display change for `session_status`, but the shipped implementation still feeds that runtime model into the selected-model slot. In `src/status/status-message.ts`, fallback visibility depends on selected-vs-active mismatch plus matching `fallbackNotice*` fields, so I do not have high-confidence evidence that the PR's fallback-preservation behavior is already implemented well enough on `main`.
