## Summary

Keep open. Current `main` does not implement the PR’s core fix: tool-result media extraction still ignores original `read` call args when inline image content lacks `details.path`, and the existing tests still assert the old empty-result behavior.

## Evidence

- **Media extraction still stops at `details.path`:** `extractToolResultMediaArtifact()` checks `details.media`, then `MEDIA:` text blocks, then `details.path` when image content exists. `extractToolResultMediaPaths()` still only accepts `result` and has no `startArgs` fallback.

  - command: `nl -ba src/agents/pi-embedded-subscribe.tools.ts | sed -n '250,350p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
