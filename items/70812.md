## Summary

Keep open. The reported false-positive looks real, but this PR’s suffix-based fix is not safe to auto-close or merge as-is: current OpenClaw explicitly allows dotted session IDs, and transcript paths are still derived as `<sessionId>.jsonl`, so a valid session ID like `debug.trajectory` would produce a real primary transcript named `debug.trajectory.jsonl`. The proposed `endsWith('.trajectory.jsonl')` exclusion would misclassify that real transcript and change doctor/usage/disk-budget behavior.

## Risks / Open Questions

- Merging the PR as written would skip orphan scanning, usage discovery, and disk-budget handling for any legitimate transcript whose session ID ends with `.trajectory`.
- The fix surface is wider than the PR body suggests; it likely needs an explicit trajectory-artifact predicate or a stronger contract change, not only a blanket `endsWith('.trajectory.jsonl')` exclusion.
