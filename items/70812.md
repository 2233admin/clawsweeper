## Summary

Keep PR #70812 open. Current main still has the reported doctor false-positive path: trajectory sidecars are documented and generated as `<session>.trajectory.jsonl`, while the current primary transcript matcher still accepts any non-archived `.jsonl` file and doctor feeds that matcher directly into orphan cleanup. The related PR #72047 is closed and unmerged, and the canonical bug #70680 remains open, so this PR is not obsolete or superseded.

## Best Possible Solution

Keep #70812 open as the active fix path for #70680. The best implementation should exclude real trajectory sidecars from doctor orphan transcript detection, preserve valid primary transcripts whose session ID ends in `.trajectory`, and explicitly settle how trajectory sidecars participate in usage and disk-budget cleanup before merge.
