## Summary

Keep open. Current main still has the self-PID/no-starttime reclamation behavior that this PR targets, and transcript writes depend on this lock path for session consistency. The PR is a focused bug-fix candidate, not obsolete or implemented on main, though it should keep or replace regression coverage instead of leaving a skipped test as the only proof for the changed legacy-lock behavior.

## Best Possible Solution

Keep the PR open as an active, focused bug-fix candidate. The best path is to review and refine the lock logic in `src/agents/session-write-lock.ts`, preserve PID-recycling coverage, replace the skipped legacy-lock test with an explicit regression for live self-PID locks without `starttime`, and land only after the session-write-lock tests or changed gate validate the behavior.
