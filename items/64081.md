## Summary

Keep open. Current main still hardcodes session-backed task notification defaults to `done_only`, ACP/subagent creation paths do not pass a configurable policy, docs still describe `done_only` as the default for ACP/subagent tasks, and the agent-default config schema has no `tasks.defaultNotifyPolicy` or per-runtime equivalent. Related cron-silent work does not cover this request because cron already uses explicit `silent` task records.

## Best Possible Solution

Keep the issue open and implement this as a core config/API change, not a ClawHub plugin. The most conservative path is to add an exported task notify policy config shape under `agents.defaults.tasks` with per-runtime defaults for `acp`, `subagent`, `cron`, and `cli` or a single default plus overrides; validate it in the zod config schema, thread it into ACP/subagent task creation before `createRunningTaskRun`, preserve existing defaults when unset, and update `docs/automation/tasks.md`, `docs/cli/tasks.md`, config help/generated schema artifacts, and focused task-registry/ACP/subagent tests.
