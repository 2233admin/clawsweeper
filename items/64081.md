## Summary

Keep this issue open. Current main still lacks any config-level task notify default, and the current ACP/subagent/task-flow paths still create deliverable background task records without a config-derived notify policy, so they fall back to `done_only`. The existing ClawSweeper comment is still accurate against current `main`.

## Best Possible Solution

Keep this issue open and implement a typed, schema-backed `agents.defaults.tasks` notify default. Resolve the effective policy before ACP, subagent, and Task Flow records are created; preserve current behavior when unset; keep cron, CLI, and media silent defaults intact; and update Background Tasks plus config/CLI docs with focused tests for ACP, subagent, cron, media, and per-task overrides.
