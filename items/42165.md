## Summary

Keep open. Current `main` still matches the reported behavior: an explicit `tools.fs` block implicitly adds `read`/`write`/`edit` back on top of restrictive profiles, which conflicts with the docs that say `minimal` exposes only `session_status` and `messaging` omits filesystem tools.

## Evidence

- **Current policy code re-exposes fs tools:** `resolveImplicitProfileAlsoAllow()` adds `read`, `write`, and `edit` whenever either `globalTools.fs` or `agentTools.fs` is present, and `resolveEffectiveToolPolicy()` merges that into the profile-stage allowlist.
  - file: [src/agents/pi-tools.policy.ts:287](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/agents/pi-tools.policy.ts#L287)
  - command: `nl -ba src/agents/pi-tools.policy.ts | sed -n '279,360p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Current test suite asserts the reported behavior:** There is an explicit test named `implicitly re-exposes read, write, and edit when tools.fs is configured`, expecting `profileAlsoAllow` to become `["read", "write", "edit"]` under `profile: "messaging"` plus `fs: { workspaceOnly: false }`.
  - file: [src/agents/pi-tools.policy.test.ts:254](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/agents/pi-tools.policy.test.ts#L254)
  - command: `nl -ba src/agents/pi-tools.policy.test.ts | sed -n '238,285p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Docs still describe narrower profiles:** The docs say `minimal` includes `session_status` only, and `messaging` includes messaging/session tools but not `group:fs`.
  - file: [docs/gateway/config-tools.md:22](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/gateway/config-tools.md#L22)
  - command: `nl -ba docs/gateway/config-tools.md | sed -n '18,40p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)

## Risks / Open Questions

- The mismatch affects more than `minimal`; current code and tests show the same implicit re-exposure for `messaging` and any other restrictive profile when `tools.fs` is configured.
- Closing as implemented/cannot_reproduce would be incorrect because the behavior is still present on `main` and is currently codified by tests.
