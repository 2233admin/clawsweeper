## Summary

Current main still appears to match the reported least-privilege bug. `tools.fs` and `tools.exec` config section presence still implicitly adds filesystem/runtime tools into the profile-stage allowlist without considering whether the effective profile is `minimal` or `messaging`, while docs and catalog define those profiles as excluding filesystem tools. There is also an open implementation candidate, #43061, so this issue should remain open until a focused fix lands or that PR is closed.

## Best Possible Solution

Keep this issue open. The best fix is to change `src/agents/pi-tools.policy.ts` so `tools.fs` and `tools.exec` configure already-granted tools but do not implicitly re-grant filesystem/runtime tools under restrictive profiles unless explicitly requested through `alsoAllow`, preserving deny precedence. Add regression coverage for both `minimal` and `messaging` with configured `tools.fs`/`tools.exec`, and review or split #43061 as the active implementation candidate.
