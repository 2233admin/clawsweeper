## Summary

Keep this issue open. Current main still exposes `running | done | failed | killed | timeout` as the persisted and agent-facing session status vocabulary, and the session list/tool docs do not add a clear `resumable`, `lastTurnStatus`, `lifecycleStatus`, or `taskStatus` contract that would prevent agents from treating `failed`, `timeout`, or `done` as unusable sessions.

## Best Possible Solution

Keep this issue open and treat it as an agent-facing session API contract problem. Add a backward-compatible field such as `resumable` plus a distinct `lastTurnStatus`, `lifecycleStatus`, or `taskStatus` field to `sessions.list` rows, document the semantics on https://docs.openclaw.ai/concepts/session-tool, and update tool descriptions or prompt guidance so orchestrators prefer `sessions_send` for known visible sessions unless the session is deleted or inaccessible. Add focused tests for returned row shape and guidance; defer any status-string rename to a versioned migration.
