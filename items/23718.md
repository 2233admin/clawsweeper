## Summary

Keep open. Current main still does not implement `{ primary, fallbacks }` for `heartbeat.model`: the schema, public config type, generated config schema, heartbeat runtime, and reply option contract all treat the heartbeat model as a single string. Existing fallback-chain support applies to regular agent/subagent model config and per-agent model fallback overrides, not to a heartbeat-specific fallback chain.

## Best Possible Solution

Keep #23718 open until heartbeat-specific fallback chains are implemented or maintainers explicitly decide against the feature. A complete fix would broaden `HeartbeatSchema` and public config types to accept `AgentModelConfig`, update generated schema/help/docs, pass heartbeat primary and fallbacks through the reply options/runtime, and add tests proving string compatibility plus `{ primary, fallbacks }` behavior for defaults and per-agent heartbeat config.
