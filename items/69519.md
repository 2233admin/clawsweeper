## Summary

Keep #69519 open. Current main still renders the Sessions Limit field as a plain text input, stores the filter as a string, and only coerces it in the controller before sending `sessions.list`. The PR targets a real remaining UI bug, but the submitted patch still has unresolved review concerns around full type migration, blank/invalid handling, and unrelated formatter churn, so it should not be auto-closed or blindly merged.

Keep the PR open and request a narrow revision: migrate the Sessions Limit state/view/controller contract consistently to `number | undefined`, render the control as `type="number"` with `min="1"` and `step="1"`, preserve a blank or invalid edit as `undefined` rather than a `0` sentinel, keep `loadSessions` sending only a positive numeric `limit`, add focused view/controller regression tests, and drop unrelated CSS formatting churn.
