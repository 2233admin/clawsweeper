## Summary

Keep #69519 open. Current main still renders the Sessions page Limit control as a plain text input wired through string state, while the gateway sessions.list schema expects an integer limit. The PR targets a real UI/API mismatch, but the current PR discussion and patch shape show it is not merge-ready yet, so it should get a focused fix rather than cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a narrow, type-consistent implementation: make the Sessions Limit control numeric, keep state and props consistently typed as `number | undefined` or another explicitly chosen numeric representation, preserve blank/invalid input behavior intentionally, continue sending only positive integer `limit` values to `sessions.list`, and add focused view/controller tests for invalid text, blank input, and valid positive limits.
