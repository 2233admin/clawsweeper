## Summary

Keep #69519 open. Current main still uses a string-backed plain text Limit field on the Sessions page, so the PR is not implemented or superseded. The PR addresses a real UI/input contract issue, but the review discussion correctly flags that the branch still needs a coherent state-type migration and empty/invalid limit handling before it is mergeable.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: either keep UI filter state as a string and parse only at the controller/API boundary, or fully migrate `sessionsFilterLimit` to `number | undefined` across app state, view props, controller, and tests. The final behavior should render a numeric Limit input with `min=1`/`step=1`, omit invalid or empty limits from `sessions.list`, and add focused view/controller coverage without unrelated CSS formatting changes.
