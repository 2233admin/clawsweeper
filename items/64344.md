## Summary

Keep this PR open. Current main still lacks the requested audit distinction for exact `${ENV_VAR}` templates: there is no `MIGRATION_RECOMMENDED` code, no `migrationRecommendedCount`, and `collectConfigSecrets()` classifies resolved non-empty SecretRef-capable config values as `PLAINTEXT_FOUND` without comparing the authored pre-substitution value. The PR remains a focused implementation candidate, and the discussion shows the author intentionally removed broader config.env/dotenv attribution work to keep the scope narrow.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or revise the focused fix that compares authored config after `$include` resolution with the resolved audit target, emits `MIGRATION_RECOMMENDED` only for exact `${ENV_VAR}` templates on SecretRef-capable fields, preserves mixed-string plaintext findings, and updates CLI JSON/text summaries, docs, and regression tests. Broader env/dotenv attribution work should stay in separate issues or PRs.
