## Summary

Keep open. Current main still lacks the PR's `MIGRATION_RECOMMENDED` finding and still audits the post-substitution config, so an authored exact `${ENV_VAR}` on a SecretRef-capable field can be reduced to a resolved non-empty string and reported as `PLAINTEXT_FOUND`. The provided PR context shows a focused implementation and regression-test plan for the live bug, not obsolete or duplicate cleanup.

## Best Possible Solution

Keep the PR open and rebase/finalize a focused fix that compares authored pre-substitution config against the resolved runtime config, emits `MIGRATION_RECOMMENDED` for exact `${ENV_VAR}` templates on SecretRef-capable fields, preserves plaintext findings for mixed strings and literal secrets, and lands the corresponding audit/CLI/docs tests.
