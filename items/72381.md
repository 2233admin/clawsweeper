## Summary

Keep #72381 open. Current main still lacks the Google Meet `realtime.agentId` config/schema surface and the Google Meet consult path still calls the shared realtime consult runtime without an agent id, so consults fall back to `main`. The PR is a recent, narrow bundled-plugin fix with matching tests proposed in the PR diff; related #72372 covers bridge lifecycle cleanup, not consult agent routing.

## Best Possible Solution

Keep the PR open and review it as a plausible focused fix in the Google Meet plugin: add `realtime.agentId` to the config type, resolver, manifest schema/help, pass it through `consultOpenClawAgentForGoogleMeet` to `consultRealtimeVoiceAgent`, and keep the resolver plus embedded-agent invocation tests. #72372 should remain separate because it addresses chrome-node bridge lifecycle cleanup, not consult agent routing.
