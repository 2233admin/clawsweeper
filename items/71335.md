## Summary

Current main has not implemented #71335. `memorySearch.sync.watch` still defaults to `true` when omitted, the gateway startup path does not override that default, and both builtin and QMD memory backends still arm chokidar watchers whenever the resolved sync config has `watch: true`. The related #64549 is the same subsystem but tracks the separate behavior bug where enabled watching does not auto-index new files.

## Best Possible Solution

Keep #71335 open. Implement a gateway-runtime-aware default in the shared memory sync resolution path used by builtin and QMD managers: omitted `sync.watch` should resolve to false for the headless gateway startup path, explicit default or per-agent `sync.watch` values should still win, and normal CLI/desktop memory behavior should keep the current true default. Add focused tests for omitted watch in gateway mode, explicit true override in gateway mode, and non-gateway omitted watch behavior; update config help/docs if the default becomes context-sensitive.

## Risks / Open Questions

- Changing the default based only on `cfg.gateway.mode` may unintentionally affect CLI or desktop configs that also contain `gateway.mode=local`; the implementation should key off the actual gateway runtime path or pass an explicit resolver context.
- Explicit `sync.watch: true` must continue to win for operators who want gateway-side live indexing.
