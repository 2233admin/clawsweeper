## Summary

Current `main` does not appear to implement this PR's fresh-session cleanup for the `agentCommand` path. A rollover/new session still carries the previous `sessionEntry` into `runAgentAttempt()`, and that code still reads stored CLI bindings and bootstrap warning state from the inherited entry, so the stale-resume bug remains plausible on `main`.
