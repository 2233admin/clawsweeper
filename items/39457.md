## Summary

Keep #39457 open. Current main still refreshes an already-loaded gateway service with `gateway install --force` and then immediately runs the prepared restart script or daemon restart. The shipped hook/plugin surfaces cover Gateway/session/plugin lifecycle, but not an updater-owned post-refresh/pre-restart hook, and the related wrapper proposal #69400 is still only an open, unshipped narrower path in the provided context.

## Best Possible Solution

Keep the issue open. The best maintainer path is either to add a core updater-owned post-update hook after `refreshGatewayServiceEnv` succeeds and before `runRestartScript`/`runDaemonRestart`, or explicitly supersede this with a shipped durable wrapper/config mechanism such as #69400/#69425. Either path should include focused coverage for the loaded macOS LaunchAgent update flow plus user-facing docs for the supported contract.

## Risks / Open Questions

- A future hook needs explicit executable, timeout, logging, environment, and failure semantics so updates cannot hang or hide restart failures.
- Maintainers may prefer to supersede the generic hook with a durable wrapper/config mechanism such as #69400, but that needs to ship first and should be linked as the canonical resolution.
