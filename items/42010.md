## Summary

Keep open. Current `main` does not implement this fix: LINE downloads still go through temp-file writes under the OpenClaw tmp dir instead of the media store, tests still assert that tmp-path behavior, and the LINE handler still does not pass a file name through for extension-based MIME fallback.

## Evidence

- **LINE download path still uses tmp files on main:** `downloadLineMedia()` still calls `client.getMessageContent(...)`, derives an extension locally, builds a temp path with `buildRandomTempFilePath(...)`, and writes the buffer with `fs.promises.writeFile(...)` instead of `saveMediaBuffer(...)`. That means the PR's core behavior is not present on current `main`.
