## Summary

Keep open. Current main still actively manages Tailscale Serve/Funnel on gateway startup, still runs `tailscale serve --bg --yes <port>` for `gateway.tailscale.mode="serve"`, and still rejects `gateway.tailscale.mode="funnel"` unless resolved gateway auth mode is `password`. The requested token-auth Funnel path or preserve-existing-Funnel behavior is not implemented.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the gateway Tailscale exposure path: either make `serve` non-clobbering by detecting and preserving existing public Funnel paths, or add an explicit supported Funnel/token policy with regression tests, setup/docs updates, and maintainer approval for the public-exposure auth change.
