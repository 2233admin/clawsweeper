## Summary

Keep open. Current `main` does not ship `gateway.bind: "netbird"` or an equivalent NetBird-specific auto-TLS path, and the PR is recent active work rather than stale cleanup material.

## Evidence

- **Config schema still lacks NetBird bind mode:** `GatewayBindMode` only includes `auto|lan|loopback|custom|tailnet`, and the Zod config schema accepts the same set only. That means current `main` rejects the PR's proposed `gateway.bind: "netbird"` surface.
  - file: [src/config/types.gateway.ts:3](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/config/types.gateway.ts#L3)

- **Config validator also omits NetBird:** The config schema bind union stops at `tailnet`; there is no shipped `netbird` literal on current `main`.
  - file: [src/config/zod-schema.ts:707](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/config/zod-schema.ts#L707)
