## Summary

Current `main` does not implement execute-time plugin context injection. Plugin tools are still resolved and returned unchanged in `src/plugins/tools.ts`, the runtime tool contract is still four-argument `execute(toolCallId, params, signal?, onUpdate?)`, and the adapter still calls plugin tools with only those four arguments. The unrelated `getApiKeyAndHeaders` compaction update is already on `main` under `src/agents/pi-hooks/*`, which makes this PR stale to rebase, not safe to close as implemented.
