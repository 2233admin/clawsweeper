## Summary

Keep open. Current `main` has meaningful WebChat/Control UI session management, but it does not fully implement this issue’s requested scope. The shipped surface is a single-chat WebChat plus a flat sessions admin table with search/sort/pagination and per-session overrides; I found no current implementation of the requested tree/folder/tag/pin/archive conversation organizer or any multi-AI collaboration/committee/comparison workflow.

## Evidence

- **Current Control UI scope:** The docs describe today's web UI as chat plus session listing and per-session overrides, not a collaborative multi-AI workspace.
  - file: [docs/web/control-ui.md:105](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/docs/web/control-ui.md#L105)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)

  - file: [docs/platforms/mac/webchat.md:8](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/docs/platforms/mac/webchat.md#L8)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
