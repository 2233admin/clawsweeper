## Summary

Keep open. Current `main` still matches the reported failure mode: `ensureSkillSnapshot()` reads the workspace snapshot version before attaching the skills watcher, the watcher setup does not seed a non-zero version, and the version comparison still treats cached `0` vs current `0` as fresh. The linked fix discussion exists, but the corresponding behavior is not present on `main`.

## Evidence

- **Version is read before watcher attachment in session refresh path:** `ensureSkillSnapshot()` calls `getSkillsSnapshotVersion(workspaceDir)` and captures `snapshotVersion` before `ensureSkillsWatcher({ workspaceDir, config: cfg })`, so a restarted process can still compare an old persisted snapshot against `0` before any watcher-side state exists.

Version bumps only happen later inside the debounced file-event callback.
