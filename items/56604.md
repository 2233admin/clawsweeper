## Summary

Keep open. Current main does not implement per-number voice/persona routing for the voice-call plugin, and the request is still coherent and in scope for the existing bundled plugin.

## Evidence

- **Voice-call config schema has only global response/TTS/inbound settings:** The current schema defines global `fromNumber`, `toNumber`, `inboundGreeting`, `tts`, `responseModel`, `responseSystemPrompt`, and `responseTimeoutMs`, but no `numbers` map or other per-dialed-number override field.
