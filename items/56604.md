## Summary

Keep open. Current main still lacks per-dialed-number voice/persona routing in the bundled voice-call plugin, and recent maintainer discussion explicitly kept the feature open. The dialed number is stored on call records, but config, manifest validation, inbound greeting, response generation, realtime consult, and TTS selection still use one global voice-call config.

## Best Possible Solution

Keep this issue open and implement it in the bundled `voice-call` plugin. Add a typed and documented per-number route table to the config schema and manifest; normalize and match inbound `event.to` at call creation; persist a stable per-call effective config or route id; and make inbound greeting, agent/model/system prompt, realtime consult behavior, and TTS voice selection use that effective config with global fallback. Add focused tests for Twilio, Telnyx, and Plivo `to` routing, fallback behavior, and per-number overrides.
