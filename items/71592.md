## Summary

Keep #71592 open. Current main still advertises `/status` and `/compact` in the TUI slash surface, but `createCommandHandlers()` has no `status` or `compact` cases; its default branch sends the raw slash text through `sendMessage()`. In `openclaw tui --local`, that path uses `EmbeddedTuiBackend`, which calls `agentCommandFromIngress()` directly rather than the gateway `chat.send` command pipeline. The related items are adjacent but do not supersede this local embedded TUI execution bug.

## Best Possible Solution

Keep the issue open and fix the TUI local embedded command boundary. Recognized shared commands such as `/status` and `/compact` should either execute through a shared local command adapter or return an explicit local unsupported/system response, but they should not fall through to `sendMessage(raw)` when `opts.local` is true. Add focused regression tests for local `/status` and `/compact` while preserving gateway-connected forwarding for commands intentionally handled by `chat.send`.
