## Summary

Current main still does not expose `stopReason`, `hasToolCalls`, or an equivalent agent completion context to `message_sending` hook handlers. The requested signal exists in agent run metadata, but outbound delivery and the public hook event only carry delivery/routing metadata. Related PR #39761 remains open and unmerged, so it is an implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open and implement a backward-compatible core hook contract change. Thread agent completion metadata from `EmbeddedPiRunMeta` through agent reply delivery and `DeliverOutboundPayloadsCoreParams`, then expose optional `stopReason` and `hasToolCalls` or a broader typed `agentContext` on `PluginHookMessageSendingEvent`. Add focused tests for agent command delivery and outbound `message_sending` hook payloads, plus plugin hook documentation. If #39761 is the chosen implementation, review and land it first; close this issue only after the change is merged and current main exposes the requested signal.
