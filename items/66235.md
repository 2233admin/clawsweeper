## Summary

Keep open. Current `main` does not expose `stopReason` or `hasToolCalls` on `message_sending` hook events, and the existing delivery/docs surface still only provides routing fields plus generic metadata. There is some core reasoning-payload suppression elsewhere, but that is not the requested hook contract and is not enough evidence to treat the issue as already fixed or obsolete.

## Evidence

- **`message_sending` event type still has only generic metadata:** `PluginHookMessageSendingEvent` exposes `to`, `content`, optional `replyToId` / `threadId`, and `metadata?: Record<string, unknown>`; there is no typed `stopReason` or `hasToolCalls` field.
  - file: [src/plugins/hook-message.types.ts:69](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/plugins/hook-message.types.ts#L69)
  - command: `nl -ba src/plugins/hook-message.types.ts | sed -n '1,170p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Outbound delivery still forwards only channel/account/media metadata:** `applyMessageSendingHook()` constructs metadata with `channel`, `accountId`, and `mediaUrls` only before calling `runMessageSending`; it does not forward assistant stop reason or tool-call state.
  - file: [src/infra/outbound/deliver.ts:620](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/infra/outbound/deliver.ts#L620)

- **Docs still describe `message_sending` as content rewrite/cancel only:** Current plugin-hook docs describe `message_sending` as rewriting `content` or returning `{ cancel: true }`, with no documented stop-reason/tool-call metadata on the event.
  - file: [docs/plugins/hooks.md:188](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/docs/plugins/hooks.md#L188)

  - file: [src/infra/outbound/payloads.ts:133](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/infra/outbound/payloads.ts#L133)
