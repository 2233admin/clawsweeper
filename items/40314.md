## Summary

Keep PR #40314 open. Current main still emits `message:received` and `message:sent` internal hooks without draining `event.messages`, while the PR is a focused unmerged fix for open bug #39688. The newer PR #41375 may be a maintainer-preferred successor for `message:received`, but it is also unmerged and explicitly leaves `message:sent` out of scope, so #40314 is not safe to close as implemented or superseded.

## Best Possible Solution

Keep #40314, #39688, or an explicit successor open until the hook reply contract is resolved. The maintainer path should consolidate #40314 with #41375, decide whether `message:sent` may reply, drain `event.messages` only where a safe reply target exists, preserve session context, suppress or persist hook-loop prevention for hook-generated sends, and update regression tests plus `https://docs.openclaw.ai/automation/hooks` to match the final behavior.
