## Summary

Keep open. Current main has useful adjacent pieces: docs:list summaries/read_when, docs hubs, live docs search, and system-prompt guidance for local docs plus config.schema.lookup. It does not implement the requested generated docs_map.md/docs-map.md with nested H2/H3/H4 heading hierarchy, nor a workflow/publish path for that file. The issue is recent, contributor-authored, unprotected, and still actionable as docs tooling work.

## Best Possible Solution

Keep the issue open and implement a deterministic docs-map generator in the docs tooling. The likely path is to extend or add a script that walks the published English docs set, parses front matter plus H2/H3/H4 headings, emits a stable docs_map.md/docs-map.md alongside the docs site LLM entrypoints, and wires it into docs checks or the docs publish workflow without replacing the existing docs:list, docs hubs, or live docs search.
