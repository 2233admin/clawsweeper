## Summary

Current `main` still hardcodes macOS voice wake forwarding to the default `sessionKey`/channel instead of plumbing any selected session target from the app UI, so the issue is not implemented on `main` and should remain open.

## Evidence

- **Voice wake forwarding defaults are still hardcoded:** `VoiceWakeForwarder.ForwardOptions` defaults to `sessionKey: "main"` and `channel: .webchat`, and `forward()` sends those values unless callers override them.
  - file: [apps/macos/Sources/OpenClaw/VoiceWakeForwarder.swift:35](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/apps/macos/Sources/OpenClaw/VoiceWakeForwarder.swift#L35)
  - command: `nl -ba apps/macos/Sources/OpenClaw/VoiceWakeForwarder.swift | sed -n '1,120p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Voice wake runtime never supplies a selected target:** When voice wake capture finishes, it calls `VoiceWakeForwarder.forward(transcript: finalTranscript)` with no options, so the hardcoded defaults are used.
  - file: [apps/macos/Sources/OpenClaw/VoiceWakeRuntime.swift:662](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/apps/macos/Sources/OpenClaw/VoiceWakeRuntime.swift#L662)

  - file: [apps/macos/Sources/OpenClaw/VoiceSessionCoordinator.swift:96](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/apps/macos/Sources/OpenClaw/VoiceSessionCoordinator.swift#L96)
