## Summary

Keep open. Current `main` still packages `openshell@0.1.0`, still resolves and prefers that bundled package over the `openshell` binary on `PATH`, and still tests that preference explicitly. The latest release tag `v2026.4.23` shows the same behavior, so there is no evidence the reported bug has been fixed on `main` or in the latest release.
