## Summary

Keep this PR open. Current main still lacks the interactive custom-provider metadata prompt this PR proposes: the custom provider configure path collects endpoint/auth/model identity, then writes fixed model metadata defaults. The related custom-provider context-window UX issue #48338 is still open, and this PR remains a plausible implementation candidate rather than stale cleanup.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate. The best path is to rebase it onto current main, align the prompt with the current metadata surface or explicitly document its narrower scope, rerun targeted command/configure tests, and then either land it as the guided custom-provider metadata setup for #48338 or supersede it with an equivalent maintainer-owned implementation.
