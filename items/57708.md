## Summary

Keep open. Current main supports Slack mrkdwn rendering, explicit Block Kit passthrough, interactive replies, and semantic presentation blocks, but it does not implement automatic Markdown-to-Slack rich_text block conversion. This is a recent RFC with unresolved product/design questions and related open Slack rich-output work, so no conservative close reason applies.

## Best Possible Solution

Keep #57708 open as a Slack plugin design follow-up under the broader #12602 umbrella. The likely implementation belongs in `extensions/slack`: convert public `MarkdownIR` from `openclaw/plugin-sdk/text-runtime` into Slack rich_text blocks, decide opt-in vs default with maintainer input, skip conversion when explicit Slack blocks or semantic presentation are present, handle streaming/chunking deliberately, and document the chosen behavior.
