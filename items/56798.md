## Summary

Keep open. Current `main` does not include this PR's net-level `node:net` bootstrap, and I did not find strong evidence that the remaining built-in fetch timeout gap is already fixed or obsolete. The PR description is partly stale because OpenClaw now documents Node 22+ as already defaulting `autoSelectFamily` to `true`, but the latest PR diff still proposes a distinct built-in-fetch timeout adjustment that is not present on `main`.

## Risks / Open Questions

- I could not reproduce on an IPv6-broken network in this environment, so I cannot confidently downgrade the reported built-in-fetch timeout problem to `cannot_reproduce`.
- The PR description is out of date relative to current Node/OpenClaw defaults, but the latest code still proposes a real behavior change for built-in fetch paths not present on `main`.
- Some OpenClaw/networking paths already avoid ambient `globalThis.fetch`, so the PR's claimed blast radius may be overstated; however, that is not enough evidence to close a still-plausible networking fix PR.
