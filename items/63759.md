## Summary

Keep this issue open. Current main still exposes Telegram `sendTyping` only as a downstream reply typing callback, after accepted inbound context construction and substantial Telegram dispatch setup. The issue’s requested behavior is narrower than #6946: start native Telegram `sendChatAction("typing")` near message acceptance, not add a visible processing-placeholder message.

## Best Possible Solution

Implement a Telegram plugin-owned fix that starts native `sendChatAction("typing")` at the earliest safe point after a message is accepted and normalized, but before queue handling, compaction, model dispatch, tool execution, and other slow downstream work. Add focused regression coverage proving accepted Telegram messages emit typing before a delayed dispatch path, while ignored or unauthorized messages do not. Keep #6946 as the separate processing-placeholder feature.
