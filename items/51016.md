## Summary

Keep open. Current `main` still contains the reported status-probe failure mode: when local gateway auth resolves to no token/password, the `status` path probes with empty auth and treats reachability as `probe.ok === true`, while `probeGateway()` explicitly disables device identity for unauthenticated loopback probes. That means a paired local operator device is not used on this path, which matches the issue’s `device identity required` false negative.

## Evidence

- **Status uses probe result as reachability:** `resolveGatewayProbeSnapshot()` resolves probe auth, calls `probeGateway()` with that auth, and sets `gatewayReachable` strictly from `gatewayProbe?.ok === true`. There is no fallback to paired device identity in this status path.
