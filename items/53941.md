## Summary

Current main still does not implement the requested LiteLLM advisory docs change. The LiteLLM provider docs continue to show an unpinned `pip install 'litellm[proxy]'` command and focused searches found no affected-version notice, downgrade guidance, key-rotation guidance, or server-audit remediation text. The code review also supports the issue body's framing: OpenClaw's bundled LiteLLM plugin is a proxy/network integration, not a PyPI LiteLLM runtime dependency, so the remaining action is documentation for users running their own LiteLLM proxy.

## Best Possible Solution

Keep this issue open and land or adapt the contributor's LiteLLM documentation advisory in `docs/providers/litellm.md`: replace the unpinned install example with current safe guidance, add a dated notice for the v1.82.7/v1.82.8 PyPI compromise, and include downgrade, key-rotation, and server-audit steps for users who ran an affected proxy. The public docs target is `https://docs.openclaw.ai/providers/litellm`.
