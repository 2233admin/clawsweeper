## Summary

Keep open. Current `main` still falls back to embedded execution for all gateway errors, while the gateway code still emits ambiguous timeout/close errors that can plausibly leave the remote request alive. The PR is addressing a real current gap; it is not implemented on `main`, not obsolete, and not strong enough to auto-close as-is because the remaining classifier approach is still string-fragile.
