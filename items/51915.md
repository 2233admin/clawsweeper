## Summary

Keep open. Current main still has the strict-mode unclosed-tag drop this PR targets, but the proposed two-layer fallback in PR #51915 would also change preserve-mode behavior and can leak closed reasoning text on user-facing internal-scaffolding paths. The fallback also bypasses the code-region-aware parser and would strip literal `<think>` text inside code examples when it fires.

## Evidence

- **Current strict-mode bug is still present on main:** The parser only appends trailing text when `!inThinking || mode === "preserve"`, so a strict-mode string like `<think>Here is my answer` ends with `inThinking=true` and returns an empty string. Current tests also encode this behavior: `"Before <think>unclosed content after"` is expected to become `"Before"` in strict mode.

The PR body/patch says its empty-result safety net would deliver content from fully wrapped closed `<think>...</think>` responses, which changes preserve-mode semantics for empty-result cases.
