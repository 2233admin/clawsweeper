## Summary

Keep open. Current `main` does not implement this PR’s proposed `models.providers.*.connectTimeoutMs` config surface, and the transport timeout work that did land uses a different mechanism: run-level timeouts are propagated into guarded per-request dispatchers, while embedded-run bootstrap still applies only `params.timeoutMs` to global undici tuning.
