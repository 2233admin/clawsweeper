## Summary

Keep open. Current main still sends the concrete subagent task through both the subagent system prompt path and the initial child user message, so PR #72028 is not obsolete or already implemented. The PR is a focused, recent bug-fix candidate for linked issue #72019, but the Aisle comment raises a real task-binding/security tradeoff that should be resolved by maintainer review rather than auto-closing the PR.

## Best Possible Solution

Keep #72028 open for maintainer review. The current main bug remains real, and the narrow PR is a plausible fix, but the best path is to resolve the task-binding concern explicitly, either by preserving non-duplicated immutable task authority or documenting why the user-message-only task is acceptable for persistent subagent sessions.
