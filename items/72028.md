## Summary

Keep open. Current main still duplicates the native `sessions_spawn` task between the subagent system prompt and the first child message. PR #72028 is an active, focused fix attempt, and the latest provided patch addresses the earlier forked-transcript concern by pointing to the `[Subagent Task]` kickoff section as authoritative. Related PR #72053 overlaps but is open and unmerged, so it is not a shipped superseding fix.

## Best Possible Solution

Keep #72028 open for maintainer review and consolidate it with the overlapping #72019 fixes. The best final implementation should put the long delegated task in exactly one authoritative model-visible location, avoid repeating it across system and user roles, preserve multiline/structured task text, and include regression coverage for the actual child `agent` payload. If #72053 or an equivalent fix lands first, close #72028 as superseded by that landed commit.
