## Summary

Keep #72028 open. Current main still duplicates the native subagent task in both the subagent system prompt and first child user message, so the linked #72019 bug is not fixed on main. #72028 is an active, focused fix attempt and the latest patch addresses the earlier forked-transcript concern by pointing to the `[Subagent Task]` section of the current subagent kickoff message. Related #72053 overlaps, but it is also open and unmerged in the provided context, so there is no shipped or clearly canonical superseding fix to justify automated cleanup closure.

## Best Possible Solution

Keep #72028 open for maintainer review and consolidate the overlapping #72019 fixes. The final accepted fix should keep the long delegated task in exactly one authoritative model-visible place, preserve multiline or structured task text, avoid ambiguity for `context: "fork"`, and include regression coverage for the final `message` plus `extraSystemPrompt` gateway payload. If #72053 or another equivalent fix lands first, close #72028 then as superseded by that shipped commit.
