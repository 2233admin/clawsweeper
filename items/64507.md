## Summary

Keep this issue open. Current main still lacks a typed, validated per-agent delivery default contract, and the message tool still infers missing targets only from explicit args or trusted run/session context. The related cron-default and announce-model items are adjacent but do not supersede this cross-session agent-level default request.

## Best Possible Solution

Keep this issue open for a core product/design fix. Add a typed, documented per-agent delivery default contract, likely under agents.list[].delivery, validate it in the config schema, carry it through resolved agent config, and apply it only as a low-precedence fallback when constructing trusted message tool context for programmatic sessions. Cover explicit override precedence, stored session delivery context, account/thread handling, multi-channel ambiguity, sessions_send, cron-created sessions, webchat-routed sessions, and backward compatibility.

## Risks / Open Questions

- A generic per-agent delivery default could silently route messages to the wrong chat unless explicit args, per-run overrides, stored session delivery context, accountId/threadId, and multi-channel ambiguity precedence are specified carefully.
- Existing session deliveryContext support can make some established sessions work, but it does not provide the requested typed per-agent default for fresh sessions_send, cron-created, or webchat-routed runs with no trusted external route.
