## Summary

Keep open. Current main still has the hooks CLI one-shot gap: hooks list/info/check/enable/disable and bare hooks use the generic hooks action wrapper with no post-output exit path, while the report path can load plugin modules and execute plugin register handlers. The PR is focused, recent, and the provided discussion shows prior review comments were addressed rather than leaving unresolved concerns.

## Best Possible Solution

Keep #70255 open for maintainer review and CI. The best path is to merge this PR or an equivalent current-main patch that adds a one-shot hooks CLI exit path after output/config mutation, preserves test-mode behavior through isVitestRuntimeEnv, and keeps the focused regression coverage for the guard.
