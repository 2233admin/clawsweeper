## Summary

Keep PR #70255 open. Current main still uses the plain hooks CLI action wrapper for `hooks list`, `info`, `check`, `enable`, `disable`, and bare `hooks`, so the proposed one-shot exit behavior is not implemented on main. The reported hang path is plausible because those commands build hook reports through plugin diagnostics, and diagnostics still load plugin modules and invoke plugin `register()` during snapshot-style loads. The PR is recent, focused, contributor-authored, and its review comments were addressed in follow-up commits according to the provided PR discussion.

## Best Possible Solution

Keep the PR open for maintainer review or landing. The best product/code path is to add a one-shot hooks CLI wrapper for report/config-mutation commands that write output and then exit after stdout drains, preserve deprecated `hooks install`/`hooks update` and relay behavior, and keep the shared `isVitestRuntimeEnv` guard plus focused regression coverage before running the normal changed gate.
