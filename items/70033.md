## Summary

Keep #70033 open. Current main has malformed tool-call argument repair, but it is only enabled for Kimi Anthropic-compatible streams and OpenAI completions, not `amazon-bedrock`/`bedrock-converse-stream`. The pinned upstream Bedrock stream parser still initializes tool calls with `arguments: {}` and finalizes by parsing the accumulated `toolUse.input`; if that input is empty or unparsable, it emits `{}` into the tool call path, matching the reported failure class.

## Best Possible Solution

Keep the issue open and add a Bedrock-specific fix or upstream pi-ai fix for `bedrock-converse-stream`: detect tool calls whose `toolUse.input` buffer is empty/malformed while the target tool requires parameters, then surface a clear provider/tool-argument truncation error or perform a bounded retry instead of dispatching `{}`. Add focused regression coverage with a fake Bedrock Converse stream for large `write` and `exec` arguments, plus a loop-prevention assertion so repeated empty-argument failures cannot continue indefinitely.
