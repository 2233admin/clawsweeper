## Summary

Keep #70033 open. Current main has malformed streamed tool-call argument repair, but it is gated to Kimi on `anthropic-messages` and all `openai-completions` transports, not `amazon-bedrock` / `bedrock-converse-stream`. The pinned Bedrock Converse parser can still produce `{}` when `toolUse.input` is empty or unparsable, which matches the reported large-argument `write`/`exec` failure class.

## Best Possible Solution

Keep #70033 open and fix the Bedrock Converse path specifically, either in OpenClaw's provider wrapper or upstream `pi-ai`: detect empty/malformed `toolUse.input` for parameterized tools before dispatch, surface a clear truncation/provider-stream error or bounded retry instead of executing `{}`, and add focused fake-Bedrock stream tests for large `write` and `exec` arguments plus loop-prevention behavior.
