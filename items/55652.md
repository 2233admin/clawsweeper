## Summary

Keep this PR open. Current main still lacks Discord cross-instance channel ownership, and this is an open, recent, targeted implementation candidate for the same-token multi-instance Discord routing bug described in #55651.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best product path is a Discord-plugin-owned fix under `extensions/discord/src/monitor` that performs robust per-instance and per-bot ownership checks before local allowlist denial in message preflight, autocomplete, and native slash-command paths, with tests for same-token multi-instance routing, guild-wide and channel-scoped allowlists, bot scoping, denied/disabled channels, wildcard/name entries, resolved channel IDs, shared-file concurrency or per-instance claim files, and lifecycle cleanup.
