## Summary

Keep open. Current `main` still does not carry channel/account-configured `mediaLocalRoots` through outbound normalization or outbound media access, and the submitted PR is also incomplete against current schema/config surfaces.

## Evidence

- **Current root resolver ignores channel/account config:** `getAgentScopedMediaLocalRoots` only returns default roots plus the agent workspace. It accepts no channel/account context, so configured `channels.<channel>.mediaLocalRoots` / `accounts.<id>.mediaLocalRoots` cannot be included from this path.
  - file: [src/media/local-roots.ts:54](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/media/local-roots.ts#L54)
  - command: `nl -ba src/media/local-roots.ts | sed -n '1,220p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Message-action normalization still uses the old root set:** `runMessageAction` builds `normalizationPolicy` from `getAgentScopedMediaLocalRoots(cfg, resolvedAgentId)` before attachment hydration. That means configured channel/account roots are still invisible during local media normalization.
  - file: [src/infra/outbound/message-action-runner.ts:861](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/infra/outbound/message-action-runner.ts#L861)
  - command: `nl -ba src/infra/outbound/message-action-runner.ts | sed -n '856,900p'`

- **Outbound delivery/plugin dispatch path still lacks configured roots:** Both `deliverOutboundPayloadsCore` and plugin send dispatch now go through `resolveAgentScopedOutboundMediaAccess`, but that helper still derives local roots from `getAgentScopedMediaLocalRootsForSources` / `getAgentScopedMediaLocalRoots`, neither of which accepts channel/account config.
  - file: [src/media/read-capability.ts:83](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/media/read-capability.ts#L83)
