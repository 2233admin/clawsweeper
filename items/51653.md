## Summary

Keep PR #51653 open. Current `main` does not implement the requested outbound metadata stripping, so this is not `implemented_on_main`, but the proposed patch is also not safe to close on because `stripInboundMetadata()` currently removes leading timestamp prefixes from arbitrary text and would alter legitimate assistant replies that start with timestamp-like brackets.
