Keep PR #39891 open. Current main still creates a Google Chat typing message before dispatch, clears it only from the delivery callback, and the shared dispatcher can skip exact `NO_REPLY` finals before delivery, so the reported orphaned typing-message path remains real.

## Best Possible Solution

Keep PR #39891 or linked issue #39843 active. The best maintainer path is to refresh the Google Chat fix against current main, delete the orphaned `typingMessageName` when dispatch returns without delivery, and add a focused monitor-level regression test for the reaction-plus-`NO_REPLY` path.
