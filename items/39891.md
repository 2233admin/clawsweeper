## Summary

Keep PR #39891 open. Current main still sends a real Google Chat typing message before dispatch and only clears that message from the delivery path, while group/channel exact NO_REPLY can skip delivery entirely. The PR targets a real remaining bug, though it likely needs a refresh against current main and a focused regression test.

## Best Possible Solution

Keep PR #39891 and linked issue #39843 active. The best maintainer path is to refresh the Google Chat fix against current main, delete any leftover `typingMessageName` after dispatch when delivery never ran, catch and log cleanup failures, avoid double-deleting after normal delivery, and add a focused regression test for Google Chat group/channel NO_REPLY cleanup.
