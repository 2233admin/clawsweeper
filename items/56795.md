## Summary

Keep open. Current `main` still lacks inbound Feishu interactive-card parsing in the event path, and the PR discussion shows useful progress but also an unresolved backward-compatibility concern around `RawBody` semantics before this can be safely closed or treated as already implemented.

## Evidence

- **Inbound event path still does not parse interactive cards on main:** `parseMessageContent()` handles `post`, `share_chat`, and `merge_forward`, but has no `interactive` branch. For interactive inbound events it falls through to returning the raw JSON envelope, and `parseFeishuMessageEvent()` uses that value directly as `ctx.content` for agent dispatch.
  - file: [extensions/feishu/src/bot-content.ts:132](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/bot-content.ts#L132)
  - command: `nl -ba extensions/feishu/src/bot-content.ts | sed -n '132,163p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Agent context builder on main only forwards parsed event content, not raw card JSON:** `parseFeishuMessageEvent()` populates `content`/`contentType` only, and `buildFeishuAgentBody()` has no Feishu interactive-card JSON attachment path. That means the PR's advertised raw-card context propagation is not present on `main`.
  - file: [extensions/feishu/src/bot.ts:104](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/bot.ts#L104)
