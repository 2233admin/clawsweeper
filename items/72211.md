## Summary

Keep PR #72211 open. Current main has general dashboard connectivity, service-not-running, and post-upgrade troubleshooting, but it does not already include this PR's local dashboard after-update recovery flow with `lsof`/`curl`, root dashboard URL, and stale browser-state guidance. The LaunchAgent re-bootstrap behavior is source-backed, and the only review feedback is an actionable docs clarity issue, so the right path is revision/review rather than maintainer cleanup close.

## Best Possible Solution

Keep the PR open and ask for a small docs revision: make the flow check `lsof`/`curl` first, restart/start only when the gateway is not listening or the LaunchAgent recovery signature appears, then keep the useful note that returned OpenClaw HTML means the gateway is healthy and the browser should open the root dashboard URL or clear stale tab/session state.
