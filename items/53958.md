## Summary

Keep open. Current main still does not visibly show the active config file path in Control UI Overview or prominently in Settings/Config. The path is already available through admin-scoped gateway/config data, but the Overview UI does not consume it and the Config UI only uses it as an Open-button tooltip. Related PR #54025 was closed unmerged, so it does not resolve the request.

## Best Possible Solution

Keep this issue open and add a visible, copyable active config path in Control UI, preferably in the Overview snapshot card or Settings/Config header, sourced from the existing admin-scoped hello snapshot or config.get snapshot path. Add focused UI coverage proving the path renders for authorized Control UI sessions and that low-privilege snapshots still do not expose sensitive paths.
