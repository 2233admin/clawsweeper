## Summary

Keep #58570 open. Current main still drops configured channel denials through verbose/debug-only paths rather than a warn-level structured observability event, and the related Slack-only PR attempts are unmerged/incomplete relative to the cross-channel request.

## Best Possible Solution

Keep the issue open and implement a shared channel-denial warning path used by Slack, Discord, and any other configured route policy drop sites. The fix should emit warn-level logs with channel/provider, account/guild when available, route key/channel id, match metadata, and a stable reason such as `channel_not_allowed`, plus focused tests that assert warning behavior and dedupe/rate limiting.
