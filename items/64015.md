## Summary

Keep open. Current main has partial improvements around plugin allowlists/disabled bundled runtime deps, but the central diagnostics-consistency issue is still actionable: `status --deep --json`/health still derive Discord channel state from a read-only health snapshot that does not merge the gateway channel runtime, while `channels.status` uses the live runtime snapshot; doctor’s gateway-auth SecretRef check also still reports an unavailable SecretRef before using the resolver path that can prove the token is available.

## Best Possible Solution

Keep the issue open and narrow the remaining fix to diagnostic convergence: make `status --deep`/health either consume the live channel runtime snapshot or clearly stop reporting runtime `running` state from the read-only health path; update doctor gateway-auth to use the same read-only SecretRef resolution path as `resolveGatewayAuthToken`; add regression tests covering Discord `channels.status` versus `status --deep --json` and a resolvable gateway token SecretRef.
