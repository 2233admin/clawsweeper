## Summary

Keep open. Current main still has the central diagnostics consistency bug: `status --deep --json` includes the health snapshot, but health builds channel summaries from a minimal snapshot that omits runtime and credential-source fields, while `channels.status` uses the richer runtime/account snapshot path. Doctor also still warns that a `gateway.auth.token` SecretRef is unavailable from a synchronous raw-config path instead of sharing the SecretRef resolvability path used by secrets audit. Some subpoints appear improved, but enough of the reported false positives remain.

## Best Possible Solution

Keep this issue open. Fix `health` and `status --deep` to reuse the same channel runtime/account projection contract as `channels.status`, preserving `running`, `connected`, `tokenSource`, `probe`, and audit fields. Then update doctor gateway auth to resolve applicable `gateway.auth.token` SecretRefs through the async SecretRef credential path or consume the same resolvability result as `secrets audit` before warning. Add focused regressions for a running configured Discord account and a resolvable gateway auth SecretRef.
