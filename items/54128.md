## Summary

Keep this issue open. Current main still has no `memorySearch.local.maxThreads` config key or equivalent wiring for the built-in local node-llama-cpp embedding provider, and the issue is concrete, recent, unprotected, and not superseded by the provided related items.

## Best Possible Solution

Keep this issue open. The best fix is to add a validated `agents.defaults.memorySearch.local.maxThreads` setting with default/per-agent merge behavior, carry it through `MemorySearchConfig`, `ResolvedMemorySearchConfig`, `EmbeddingProviderOptions`, the node-llama shim, and `createLocalEmbeddingProvider`, then pass it to `getLlama` only when configured. Add tests for omission, configured forwarding, invalid values, merge behavior, and docs/schema coverage.
