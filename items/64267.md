## Summary

Keep #64267 open. The reported leak is still actionable on current `main`: the block-streaming path still uses the stale thinking-tag scanner that misses `antml:` namespaced tags and tags with attributes, while the shared final-message sanitizer has the broader regex. The tests cover plain `<think>`, `<thinking>`, `<thought>`, and `<antthinking>` streaming cases, but not the namespaced/attributed cases called out in the issue comments.

## Best Possible Solution

Keep the issue open and fix the core stream-time stripping path. The best fix is to make `src/agents/pi-embedded-subscribe.ts` share the canonical reasoning-tag semantics from `src/shared/text/reasoning-tags.ts` or otherwise update `THINKING_TAG_SCAN_RE` to match `antml:` namespaced and attributed thinking tags, then add `onBlockReply`/block-streaming regression coverage for `<antml:thinking>...</antml:thinking>` and `<thinking hidden='1'>...</thinking>` across chunk boundaries. After that lands, verify the fix against the affected channel path and close this issue as implemented.
