## Summary

Keep open. Current main has fixed several adjacent thinking-leak paths, including the canonical sanitizer and MiniMax reasoning_content defaults, but the streamed/block reply scanner still uses a narrower thinking-tag grammar than the canonical sanitizer. Attributed reasoning tags can still pass through the user-visible block delivery path before final-message sanitization runs.

## Best Possible Solution

Keep this issue open and make a narrow core fix in the embedded streaming utilities. The best implementation is to share the canonical reasoning-tag grammar with `stripBlockTags`, `splitThinkingTaggedText`, and reasoning extraction, including `antml:` prefixes and attributes, while preserving current code-span awareness and chunk-boundary state. Add focused streaming/block regression tests for attributed `<think>`, `<thinking>`, and `<antml:thinking>` across chunk boundaries, plus final-message promotion coverage if the split path is updated.
