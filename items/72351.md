## Summary

Keep open. Current main still accepts `openclaw tui --history-limit 0` at the CLI, preserves that zero through the TUI session history path, and sends it to `chat.history`, while the Gateway protocol requires `limit >= 1`. PR #72351 remains a narrow fix for the implicated TUI path with targeted regression coverage; the related history-limit items in the provided context cover channel/group configuration, not this TUI Gateway request path.

## Best Possible Solution

Keep PR #72351 open and land this narrow TUI guard plus regression test, or recreate the same behavior on main. The Gateway protocol should continue rejecting raw `chat.history.limit: 0`; the TUI should interpret `--history-limit 0` locally as skip history loading. Related item #65425 concerns group/channel history configuration and does not supersede this TUI request path.
