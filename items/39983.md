## Summary

Keep #39983 open. Current main still sends Ollama memory embeddings to `/api/embeddings` with a `prompt` payload and expects `embedding[]`; tests also lock that deprecated contract. Upstream Ollama docs now document `/api/embed` with `input` and `embeddings[][]`, and the related PRs in the issue timeline have not landed on main.

## Best Possible Solution

Keep the issue open and land one focused fix in `extensions/ollama/src/embedding-provider.ts`: call `/api/embed`, send `input`, parse `embeddings[0]` or batch `embeddings`, preserve existing base URL/auth/header/SSRF behavior, and update `extensions/ollama/src/embedding-provider.test.ts` to assert the new Ollama contract. After that PR lands, close this issue and close or supersede the duplicate open PRs against the merged fix.
