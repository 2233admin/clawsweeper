## Summary

Issue #51851 appears already fixed. Current `main` and the released `v2026.4.22` code reload chat history on `session.message` for the active session, defer that reload while a run is still active, and replay it once the run terminates, which matches the reported `/new`/`/reset` startup-greeting-not-visible-until-refresh regression.

Closing from a Codex review because this looks already implemented in OpenClaw. That matches the reported `/new` / `/reset` symptom where the greeting was persisted server-side but only became visible after a manual refresh.
