## Summary

Keep open: current `main` still drops Feishu `mediaLocalRoots` between bot dispatch and `sendMediaFeishu`, so the PR addresses a real bug on current code rather than stale or already-landed work.

## Evidence

- **Feishu media send path still requires `mediaLocalRoots` for local files:** `sendMediaFeishu` documents that local paths require `mediaLocalRoots` after the CVE hardening, and it only passes `localRoots` into `loadWebMedia` when that field is provided.
  - file: [extensions/feishu/src/media.ts:678](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/extensions/feishu/src/media.ts#L678)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '678,730p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Current Feishu bot dispatch does not pass agent-scoped roots:** Both active broadcast dispatch and single-agent dispatch construct `createFeishuReplyDispatcher(...)` without any `mediaLocalRoots` field on current `main`, so generated local media from those reply paths cannot reach `sendMediaFeishu` with the allowed roots attached.
  - file: [extensions/feishu/src/bot.ts:1093](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/extensions/feishu/src/bot.ts#L1093)

  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Current Feishu reply dispatcher has no plumbing for the field:** `CreateFeishuReplyDispatcherParams` on current `main` does not include `mediaLocalRoots`, and the `sendMediaFeishu(...)` call in `sendMediaReplies` likewise omits it, matching the bug described in the PR.
  - file: [extensions/feishu/src/reply-dispatcher.ts:110](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/extensions/feishu/src/reply-dispatcher.ts#L110)
