## Summary

Current main still routes bare `openclaw` directly to Crestodian without checking whether the active config is missing or an empty authored object. The PR remains a focused, active change for behavior that is not implemented on main, and the provided PR discussion shows the author addressed the Greptile e2e coverage concern in head commit `6ba1243b4cf3afae50abc4f76edff16e57166d49`.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, the best path is the PR's scoped CLI route before the bare-root Crestodian branch: read the config snapshot and send missing or valid empty authored config to `setupWizardCommand({})`, while preserving Crestodian for configured installs and invalid configs that need repair. Keep the paired CLI docs, TUI docs, and first-run smoke updates so the startup contract is explicit.
