## Summary

Keep this issue open. Current main at c7543701009ccdae83d38ee341e072c8a7c4af6e still exposes only the existing browser act text paths (`type` and `fill`), and those paths dispatch to Playwright `fill`/`type` or Chrome MCP `fill`, not CDP `Input.insertText` or an IME-style insertion action.

## Best Possible Solution

Keep this issue open and implement the capability in the bundled browser plugin as an additive browser act kind. The likely path is to extend `BrowserActRequest`, `BROWSER_ACT_KINDS`, `ACT_KINDS`, normalization, CLI/tool docs, timeout handling, and `/act` dispatch; focus the target element/tab; call CDP `Input.insertText` for managed/raw CDP profiles; define supported behavior or a clear unsupported error for Chrome MCP existing-session profiles; and add focused schema, normalization, dispatch, and docs tests.
