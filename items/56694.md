## Summary

Keep open. Current `main` still has the warning-path mismatch: the embedded runner suppresses the `read tool called without path` warning only for `path` and `file_path`, not for `file` or `filePath`, and there is no evidence on `main` that #56694 was implemented.

## Evidence

- **Warning logic still checks only two parameter names:** `handleToolExecutionStart` resolves the read path from `record.path` or `record.file_path` only, then warns if that trimmed value is empty. There is no `record.file` or `record.filePath` branch in current `main`.
