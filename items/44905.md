## Summary

Keep open. Current main has partial fixes for commentary-phase output and NO_REPLY-with-media text stripping, but the broader Discord/internal-trace sanitizer request is not fully implemented. The shared outbound path still only applies channel-provided sanitization, Discord send still forwards the transformed text to sendDiscordText/sendDiscordMedia, and the visible-text sanitizer does not strip the raw `to=functions.*` / envelope-field patterns described here. The provided discussion also explicitly left this issue open after #61282 for the non-commentary sanitizer remainder.
