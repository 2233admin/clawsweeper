## Summary

Keep open. Current `main` still has `updateLastRoute` reading the session store without `skipCache: true` inside `withSessionStoreLock`, while the other lock-holding writers already force a fresh read. `loadSessionStore` still serves cached data unless `skipCache` is set, so the PR’s targeted fix is not implemented on `main` and should not be auto-closed.
