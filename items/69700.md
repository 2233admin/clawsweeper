## Summary

Current `main` still lints `sources/` pages as first-class wiki pages that need `id`, `pageType`, and freshness metadata, while the docs still describe `sources/` as holding imported raw material. Supported import paths do add that frontmatter automatically, but the docs/behavior mismatch in the issue remains plausible, so this is not safe to auto-close.
