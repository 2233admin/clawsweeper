## Summary

Keep #69700 open. Current main still treats every `sources/*.md` file as a memory-wiki source page and applies required `id`, `pageType`, and freshness checks, while the public memory-wiki docs still describe `sources/` as imported raw material. The importer and bridge flows wrap raw content in generated source pages with provenance frontmatter, but that does not resolve the reported ambiguity for raw markdown placed directly under `sources/`.

## Best Possible Solution

Keep the issue open for a small memory-wiki product/docs fix. The conservative path is to clarify `docs/plugins/memory-wiki.md`, `docs/cli/wiki.md`, and lint/CLI messaging so users know `sources/` contains OpenClaw source pages with provenance frontmatter, and raw files should be brought in through `openclaw wiki ingest` or bridge import. If raw unmodified files are intended to be supported directly, add an explicit raw-source mode or ignored raw-source location with focused lint/compile tests.
