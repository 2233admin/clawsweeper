## Summary

Keep #69700 open. Current main still has the reported policy mismatch: lint treats any `sources/*.md` page as a structured wiki page and requires `id`/`pageType` plus freshness metadata, while the user-facing docs still describe `sources/` as imported raw material. The supported import paths wrap raw source content in frontmatter-bearing source pages, so maintainers should either clarify that contract in docs or intentionally add a raw-source lint mode/skip rule.

## Best Possible Solution

Keep the issue open for maintainer product/docs cleanup. The conservative fix is to document that `sources/` contains structured source pages that preserve raw material inside fenced content/provenance metadata, not verbatim raw files; if true verbatim storage is desired, add an explicit raw-source mode or narrowly scoped lint exemption with tests so malformed bridge/unsafe-local pages are still detected.
