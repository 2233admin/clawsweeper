## Summary

Keep open. Current `main` supports Gemini Google Search grounding only as a dedicated `web_search` provider, not as a native per-model toggle that injects `tools: [{ google_search: {} }]` into normal Gemini chat/completion requests.

## Evidence

- **Docs describe Gemini grounding as the `web_search` provider path:** The public docs wire Gemini grounding through `plugins.entries.google.config.webSearch.*` plus `tools.web.search.provider = "gemini"`, which matches a separate search-tool provider rather than a model-level toggle.
  - file: [docs/tools/gemini-search.md:32](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/docs/tools/gemini-search.md#L32)
  - command: `nl -ba docs/tools/gemini-search.md | sed -n '1,120p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Google plugin config exposes only web-search settings:** The bundled Google plugin declares UI/config for `webSearch.apiKey` and `webSearch.model`, and its contract advertises `webSearchProviders: ["gemini"]`. There is no provider/chat setting for built-in Gemini search toggling.
  - file: [extensions/google/openclaw.plugin.json:35](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/google/openclaw.plugin.json#L35)
  - command: `nl -ba extensions/google/openclaw.plugin.json | sed -n '30,90p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **`google_search` is injected only in the dedicated Gemini web-search runtime:** The only inspected code path that sends `tools: [{ google_search: {} }]` is the Gemini web-search provider runtime, which posts a single search query to `generateContent` and returns synthesized citations.
  - file: [extensions/google/src/gemini-web-search-provider.runtime.ts:64](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/google/src/gemini-web-search-provider.runtime.ts#L64)
  - command: `nl -ba extensions/google/src/gemini-web-search-provider.runtime.ts | sed -n '1,120p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Normal Gemini transport does not add Google grounding tools:** `buildGoogleGenerativeAiParams()` builds normal Gemini requests from messages, optional cached content, thinking config, and function tools. When tools are present, it maps them to `functionDeclarations`; it does not inject `google_search` or any Gemini-native search toggle.
  - file: [extensions/google/transport-stream.ts:441](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/google/transport-stream.ts#L441)
  - command: `nl -ba extensions/google/transport-stream.ts | sed -n '420,500p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Native search support is currently Codex-only:** Config and runtime support a native-search toggle only under `tools.web.search.openaiCodex`, and managed `web_search` suppression is gated by Codex-native activation. There is no analogous Gemini-native activation path.
  - file: [src/config/types.tools.ts:494](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/config/types.tools.ts#L494)

- **Generic extra params do not provide a raw tool-payload escape hatch here:** The extra-params wrapper only forwards known stream options like `temperature`, `maxTokens`, `transport`, `openaiWsWarmup`, and `cachedContent`; it does not pass arbitrary raw `tools` through to the Google transport.
