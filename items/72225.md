## Summary

PR #72225 is not obsolete on current main. The existing `CHAT_SESSIONS_ACTIVE_MINUTES` constant is still defined and used by one chat-completion session refresh path, but the three Control UI refresh paths targeted by the PR still pass `activeMinutes: 0`; `loadSessions` strips non-positive values before calling `sessions.list`, so those paths still bypass the active-session window. The PR discussion and prior bot review both describe the same narrow, test-backed fix, and no protected label or maintainer-author rule requires auto-closure.

## Best Possible Solution

Keep PR #72225 open for maintainer review or merge an equivalent scoped fix: replace the three remaining Control UI chat-session refresh callsites in `app-chat.ts`, `app-render.helpers.ts`, and `chat/session-controls.ts` with `CHAT_SESSIONS_ACTIVE_MINUTES`, and keep/update regression coverage so initial chat refresh and session-switch refresh no longer lock in `activeMinutes: 0`.
