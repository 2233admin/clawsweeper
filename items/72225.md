## Summary

PR #72225 is still relevant. Current main still has the three Control UI chat-session refresh paths hardcoding `activeMinutes: 0`, and `loadSessions` omits non-positive active windows from the `sessions.list` request, so those paths still bypass the existing `CHAT_SESSIONS_ACTIVE_MINUTES` window.

## Best Possible Solution

Keep PR #72225 open for maintainer review or merge an equivalent scoped fix: replace the three remaining Control UI `activeMinutes: 0` chat-session refresh callsites with `CHAT_SESSIONS_ACTIVE_MINUTES`, and keep regression coverage for initial chat refresh and session-switch refresh so the shared active-session window cannot drift again.
