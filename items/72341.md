## Summary

Keep #72341 open. The report has a concrete reproduction plus an independent confirmation comment, and current main still has the code shape that can produce cumulative stream cards: Gateway live chat deltas are merged into a run-wide buffer, while Control UI snapshots the current live stream as a new segment when a tool card starts and renders each segment as its own stream item. I did not find a source/test change that resets or deltas the assistant text accumulator at tool boundaries.

## Best Possible Solution

Fix the live WebChat stream boundary contract: either have Gateway emit per-text-block deltas/reset its chat buffer when a tool/item starts, or have Control UI track the already-committed prefix and store only the fresh post-tool text in `chatStreamSegments`. Add focused tests covering `text -> tool -> text -> tool -> text` so rendered segments are A, B, C, not A, A+B, A+B+C.
