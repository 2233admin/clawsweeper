## Summary

Keep open. The issue is recent, has a confirming user comment, and current main still has code/test evidence for cumulative live chat deltas while the WebChat item builder renders each stored/live stream text verbatim. I did not find a current-main fix, changelog entry, or regression test for fresh assistant cards between tool calls.

## Best Possible Solution

Keep the issue open and fix the live WebChat boundary contract. The likely product/code path is either to make gateway chat deltas block-scoped/reset-aware around tool-start boundaries, or to make Control UI track the last committed stream segment and derive only the fresh suffix for the next visible card. Add focused tests in `ui/src/ui/app-tool-stream.node.test.ts` or `ui/src/ui/chat/build-chat-items.test.ts`, plus a server-chat contract test if the gateway payload semantics change.
