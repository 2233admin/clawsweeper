## Summary

Keep PR #69954 open. Current `main` at `998e37fcb3d5b937fcf70d7509b677efba939f10` still prunes a session entry when the resolved persisted `sessionFile` path is missing, and it does not check the canonical `<sessionId>.jsonl` transcript path first. The linked bug #50248 remains a plausible real product bug, and the related PR #72167 was closed unmerged rather than shipped. PR #69954 is not obsolete, but it should address the review finding that repair-only runs can leave `wouldMutate` false / mutation accounting inaccurate.

## Best Possible Solution

Keep the PR open and have the contributor update `pruneMissingTranscriptEntries` to return both prune and repair counts or another explicit mutation report, then wire that into `missing`, `wouldMutate`, dry-run action reporting, and applied summaries. The core fix belongs in `src/commands/sessions-cleanup.ts` with focused regression coverage in `src/commands/sessions-cleanup.test.ts`; optional follow-up can align doctor warnings if maintainers want the same fallback there.
