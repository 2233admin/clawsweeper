## Summary

Keep open. Current main still does not implement this PR's central behavior: reply media normalization failures are still converted into dropped media plus a generic warning, and final media dedupe still strips media when sent-media target metadata is absent. #69310 is related and broader, but the provided context does not show it clearly superseding the same-target-proof/dedupe part of #70109, and #70109 also has unresolved review findings that should be fixed before landing.

## Best Possible Solution

Keep #70109 open for maintainer/product judgment and either refresh it or consolidate it with #69310. The best path is one implementation that makes reply media failures visible without unsafe text-only fallback, preserves final `MEDIA:` when same-target proof is absent, keeps same-target dedupe for proven duplicate sends, and includes regression tests for final payloads, block delivery, sandbox/media staging failures, and ambiguous `messagingToolSentMediaUrls` metadata.
