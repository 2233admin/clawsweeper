## Summary

Keep open. Current main does not implement the requested Feishu send-time cross-bot synthetic broadcast. The existing Feishu broadcast path is an inbound multi-agent dispatch for eligible group messages, with observer agents using a no-op dispatcher, and the outbound Feishu send path does not inject synthetic events into other bot/account handlers. Feishu config/docs also do not expose `channels.feishu.crossBotBroadcast` or Feishu `allowBots`. The closed related #59677 appears to be another report of the same remaining gap, not a superseding canonical fix.

## Evidence

- **Requested behavior remains distinct:** Issue #43563 asks for send-time cross-bot broadcast: after Bot A sends to a Feishu group, OpenClaw should inject a synthetic message event into other registered bots in that same group, with opt-in `channels.feishu.crossBotBroadcast.enabled`. Related #59677 was cross-referenced as the same Feishu bot-authored-message visibility problem, not as an implementation.
- **Existing Feishu broadcast is inbound multi-agent dispatch, not send-time bot-to-bot delivery:** `handleFeishuMessage` resolves top-level `broadcast` for group inbound events and dispatches configured agents for the same incoming message. Non-active broadcast agents use a no-op dispatcher, so this path does not make bot replies visible as new inbound messages to other Feishu bots.
