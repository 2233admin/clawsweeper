## Summary

Keep open. Current `main` does not contain the PR’s APNG normalization: `sniffMime` still returns `file-type`’s MIME verbatim, so a sniffed `image/apng` can still propagate through media storage and vision payload paths.

## Evidence

- **No APNG normalization in MIME sniffer:** `sniffMime` still returns `type?.mime ?? undefined` directly, with no `image/apng` to `image/png` mapping.
  - file: [src/media/mime.ts:74](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/media/mime.ts#L74)
  - command: `sed -n '1,220p' src/media/mime.ts`
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Detected MIME is stored unchanged:** `saveMediaSource` and `saveMediaBuffer` use `detectMime(...)` as the saved `contentType`, so any sniffed `image/apng` would be persisted unchanged.
  - file: [src/media/store.ts:381](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/media/store.ts#L381)
