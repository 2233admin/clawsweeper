## Summary

Keep #71495 open. The reported `/subagents list` empty result is recent, reporter-confirmed on v2026.4.24, and current `main` has not materially changed the slash-list registry/read path since that release. A separate unreleased completion-delivery fallback may address the later parent-result observation, but it does not prove the list-empty symptom is fixed.

## Best Possible Solution

Keep this issue open. Add a regression scenario that spawns a native subagent from a real chat session, immediately invokes `/subagents list`, and expects the child to appear in active or recent output. Instrument `registerSubagentRun`, `getSubagentRunsSnapshotForRead`, and `listControlledSubagentRuns` with requester/controller/session-key and map-size diagnostics; if packaged runtime module splitting is confirmed, move `subagentRuns` to a process-global singleton or a durable shared registry read path.
