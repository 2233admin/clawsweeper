## Summary

Keep open. Current `main` still has the same likely failure mode: under `sudo`, systemd commands target the invoking user's user manager, but the gateway unit path is still derived only from `HOME`. In a typical `sudo` session that means OpenClaw can write `openclaw-gateway.service` under `/root/.config/systemd/user/` and then run `systemctl --machine <sudo-user>@ --user enable ...`, which matches the reported `Unit file openclaw-gateway.service does not exist` error rather than proving the issue is fixed.
