## Summary

Keep open. Current main still does not register `skills.uninstall`: the gateway method list, admin scope map, skills handler, protocol schema/types, docs, and ClawHub backend only cover `skills.install` and `skills.update`. The related PR #69319 explicitly fixes this issue, but it is open and not merged, so the issue remains the canonical tracker until that PR or an equivalent implementation lands.

## Best Possible Solution

Keep #69299 open as the canonical feature request and review/land PR #69319 or an equivalent change that adds the typed `skills.uninstall` protocol schema/validator, Gateway handler, method registration, `operator.admin` scope classification, ClawHub backend removal semantics, docs, and regression tests. Once that ships on main, close the issue via the PR's `Fixes #69299` linkage or with shipped implementation evidence.
