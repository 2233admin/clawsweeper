## Summary

Keep open. Current main intentionally surfaces mutating-tool failures even when reply text exists, and this PR changes that to key only on `hasUserFacingReply`. Code inspection shows reply text can be accumulated from streamed partial output before a turn later ends with `stopReason === "error"`, so the proposed suppression can hide a real failed mutation rather than only removing stale warnings after a successful recovery.
