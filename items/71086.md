## Summary

Keep #71086 open. Current main has not implemented the requested suppression, and the PR is a plausible focused fix, but the posted P1 review identifies a real correctness gap in the proposed gate: it suppresses mutating-tool failures on any user-facing text, not only on a successfully completed/recovered turn.

## Best Possible Solution

Keep the PR open and request a revision: suppress stale mutating-tool warnings only when the turn has a successful terminal/recovered state, not merely any user-facing reply. Update the stale block comment in `payloads.ts` and add regression coverage for both cases: successful recovered reply suppresses the stale warning, while error/interrupted turns with partial assistant text still surface the mutating failure.
