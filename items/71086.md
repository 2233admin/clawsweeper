## Summary

Keep PR #71086 open. Current main still has the old mutating-tool warning behavior, so the PR is not obsolete or implemented on main. The PR also has a substantive unresolved review finding: suppressing on `hasUserFacingReply` alone can hide unresolved mutating-tool failures in error/interrupted turns, so it needs revision rather than cleanup closure.

## Best Possible Solution

Keep the PR open for revision. The best fix is to add an explicit successful-terminal-state signal to the payload warning policy, suppress stale mutating-tool warnings only when a user-facing reply was produced by a successful/recovered turn, preserve warning surfacing for error/interrupted turns, update the stale block comment, and add regression tests for both successful-recovery suppression and partial/error-turn warning preservation. Coordinate with the broader open #39406 policy discussion rather than treating this narrow PR as a full config-surface solution.
