## Summary

Current `main` does not implement per-agent MCP env overrides in `openclaw.json`. MCP server definitions are still modeled as global `mcp.servers`, agent config has no `mcp` block, and the embedded Pi MCP loader merges one global config layer into each runtime. There is a partial workspace/project-settings path for agent-specific MCP settings, but that is not the requested per-agent OpenClaw config/env injection.
