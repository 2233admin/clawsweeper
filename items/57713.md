## Summary

Keep open. Current main still contains the reported failure mode: when the default sandbox image is missing, `ensureDockerImage()` pulls and tags plain `debian:bookworm-slim` as `openclaw-sandbox:bookworm-slim`, while the sandbox mutation helper still requires Python inside the container. The repo now has a Dockerfile/setup path that installs `python3`, but that does not remove the incompatible runtime fallback. PR #57789 is the active unmerged fix for #57713, and #51185 remains an open root-cause issue.

## Best Possible Solution

Keep #57713 open until PR #57789 or an equivalent fix lands. The fix should remove the silent Debian alias path, fail fast or provision a compatible default image, surface clear repair guidance for existing incompatible Docker sandboxes, and keep regression coverage around `ensureDockerImage()` plus Docker mutation failures. #51185 can remain the broader root-cause tracker if maintainers prefer that canonical issue.
