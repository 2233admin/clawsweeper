## Summary

Current main still does not generate zsh positional argument completions for `openclaw wiki ingest`, so the reported path-completion bug remains actionable. Related PRs exist, but none is merged or shipped on main.

## Best Possible Solution

Keep the issue open until one canonical fix lands on main. The best implementation is to update `src/cli/completion-cli.ts` so zsh leaf-command generation reads Commander positional arguments, emits path-like args as `_files`, directory-like args as `_files -/`, handles variadic positions, and adds focused regression coverage in `src/cli/completion-cli.test.ts` for `wiki ingest <path>`.
