## Summary

Keep open. Current main still omits positional Commander arguments from generated zsh leaf-command completions, while `openclaw wiki ingest` still declares a `<path>` positional. PR #69305 appears to be the active fix path, but it is open and the implementation is not on current main.

## Best Possible Solution

Keep #69293 open until #69305 or an equivalent maintainer-reviewed patch lands. The fix should add a zsh positional-argument generator for Commander registered arguments, map path-like arguments such as `wiki ingest <path>` to `_files`, preserve subcommand dispatch behavior, and add focused tests in `src/cli/completion-cli.test.ts`.
