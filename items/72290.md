## Summary

Keep PR #72290 open. Current main still lacks both behaviors proposed by the PR: inbound dedupe still returns null when MessageSid is missing, and chat.send user transcript updates still omit the request idempotencyKey. The PR is not merge-ready as provided because it includes an accidental chat.ts.backup file and has no regression test for the new fallback, but the underlying cross-channel duplicate-delivery bug remains plausibly actionable in this repo and the linked canonical issue #72176 is still the right product surface to fix.

## Best Possible Solution

Keep the item open but require a cleaned, narrow patch: remove the backup file, add targeted inbound-dedupe coverage for missing MessageSid retries, add chat.send transcript-update coverage that asserts the user message carries the run idempotencyKey, and then resolve whether content-based fallback is the right conservative key for all channels or should be scoped by provider/account/session/thread/peer.
