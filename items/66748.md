## Summary

Keep this issue open. Current main still has the reported exec-event heartbeat wake path for subagent session keys, and the provided GitHub context shows #66749 as an open, same-author implementation PR with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review #66749 as the narrow fix path, or explicitly retarget it to #69492 if maintainers want the broader event-audience design first. The implementation should guard exec completion heartbeat wake requests for subagent session keys while preserving system-event enqueueing, with regression coverage for both `maybeNotifyOnExit` and `emitExecSystemEvent` behavior.
