## Summary

Keep open. Current `main` has some hardening in `extensions/feishu/src/monitor.startup.ts` and `extensions/telegram/src/polling-session.ts`, and it changed startup semantics after `v2026.4.23` so the exact old split `ready` timeline is less likely. But slow channel recovery is still on the startup path, Feishu still spends up to 30s on startup probe timeouts before continuing, Telegram still gates polling on webhook cleanup, and there is still no per-channel startup timing/recovery summary. The discussion's proposed async/fault-isolated channel boot is not implemented on current `main`.
