## Summary

Keep open. Current main still uses raw trimmed provider/modelApi strings when deciding OpenAI websocket transport, so the normalization bug described by PR #57140 is not fully implemented on main. One smaller piece of the PR is already present: warm-up payloads now omit empty `tools` arrays.

## Risks / Open Questions

- The PR was authored against an older layout: websocket stream resolution now lives in `src/agents/pi-embedded-runner/stream-resolution.ts`, so the proposed fix does not map 1:1 onto current main.
- Current main explicitly documents that `openai-codex` should stay on HTTP because it targets ChatGPT backend paths, so the PR's codex-websocket portion likely needs maintainer review rather than blind merge.
