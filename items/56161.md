## Summary

Current `main` still routes `openclaw skills install` and `openclaw skills update` through the default agent workspace, not the workspace implied by the current directory, so #56161 remains an actionable bug rather than a cleanup close.

## Evidence

- **Skills CLI still hard-codes default agent workspace:** `resolveActiveWorkspaceDir()` loads config and immediately returns `resolveAgentWorkspaceDir(config, resolveDefaultAgentId(config))`. The `install` and `update` subcommands both call that helper, so they do not inspect `process.cwd()` or infer the active agent from the current workspace path.
  - file: [src/cli/skills-cli.ts:44](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cli/skills-cli.ts#L44)
  - command: `nl -ba src/cli/skills-cli.ts | sed -n '1,180p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **No cwd-based inference is present in the CLI entrypoint:** A direct search of `src/cli/skills-cli.ts` finds `resolveDefaultAgentId` uses at lines 29 and 46, and no use of `process.cwd()` or `resolveAgentIdByWorkspacePath`, matching the reporter's claim that the command ignores the current agent workspace.
  - file: [src/cli/skills-cli.ts:2](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cli/skills-cli.ts#L2)
  - command: `rg -n "process\.cwd\(|resolveAgentIdByWorkspacePath|resolveDefaultAgentId" src/cli/skills-cli.ts`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Workspace-path resolution helper exists elsewhere in core:** `resolveAgentIdByWorkspacePath()` is already implemented in `agent-scope` and returns the most specific matching agent for an arbitrary workspace path. That makes the issue technically coherent and confirms the current CLI is simply not using the available agent-path inference seam.
  - file: [src/agents/agent-scope.ts:200](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/agent-scope.ts#L200)
  - command: `nl -ba src/agents/agent-scope.ts | sed -n '180,245p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Current tests only cover default-workspace behavior:** The CLI tests mock `resolveDefaultAgentId()` to `main`, mock `resolveAgentWorkspaceDir()` to `/tmp/workspace`, and assert installs go to `/tmp/workspace/skills/calendar`. There is no coverage for running from a non-default agent workspace, so the test suite does not contradict the reported bug.
  - file: [src/cli/skills-cli.commands.test.ts:152](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cli/skills-cli.commands.test.ts#L152)
