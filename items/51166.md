## Summary

Keep open. Current `main` still appears to have the reported sandbox regression, and this PR is not safe to close as implemented-on-main or cannot-reproduce. The PR identifies the right failure point, but the proposed fix in discussion is incomplete: it hardcodes an amd64-only GHCR tag and uses a fallback build path that would not work for normal npm-installed users.

## Risks / Open Questions

- Closing this PR would discard a still-relevant report while current `main` retains the same default-image behavior that can leave sandbox write/edit tools without `python3`.
- Merging this PR as written would likely regress ARM64 users and still fail for many npm-installed users when the fallback path tries to build from a non-packaged `Dockerfile.sandbox`.
- The discussion already points toward a better follow-up path, including the later cross-reference to PR #57789, so conservative cleanup should leave this open rather than force-close it.
