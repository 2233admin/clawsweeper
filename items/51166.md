## Summary

Keep open. Current `main` still has the reported behavior, so this is not implemented on `main`, but the PR as proposed is not strong enough to close out conservatively because it would hardcode the amd64-only slim image and use a fallback build path that does not exist for normal npm-installed users.

## Risks / Open Questions

- Closing this PR would leave an unfixed live bug on current `main`: the default sandbox image path still creates a container image without Python even though sandbox mutation helpers require it.
- Merging the PR unchanged would likely regress ARM64 users because it pins `ghcr.io/openclaw/openclaw:main-slim-amd64` instead of the published multi-arch `main-slim` manifest.
- The PR fallback path is effectively dead for packaged installs because `Dockerfile.sandbox` is not included in the published package and `process.cwd()` is user-controlled.
