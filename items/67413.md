## Summary

Keep this issue open. Current main still exposes only global memory-core dreaming configuration, the strict per-agent agent schema/type do not accept an `agents.list[].dreaming` block, and the managed dreaming path still resolves and processes all configured agent workspaces. The issue is also paired with open same-author PR #67420, which is an implementation candidate and should be reviewed or closed before this issue is resolved.

## Best Possible Solution

Keep this issue open while #67420 is reviewed. The likely product path is to add a documented `agents.list[].dreaming` contract, at minimum `enabled`, wire it through strict config schema/types and `resolveMemoryDreamingWorkspaces`, and add resolver/runtime/docs tests. If per-agent schedules remain in scope, explicitly choose whether memory-core should own multiple scoped cron jobs or track schedule flexibility separately.
