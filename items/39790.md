## Summary

Current main still lacks both behaviors targeted by PR #39790: gateway session-title extraction does not strip inbound metadata before deriving `firstUserMessage`, and the computed row `displayName` is not passed into `deriveSessionTitle`. The PR remains actionable, and the linked issue #39722 plus Greptile review support the same root cause while only flagging missing regression coverage.

## Best Possible Solution

Keep PR #39790 or linked issue #39722 open until main implements the narrow gateway fix: strip inbound metadata before extracting first-user-message titles, pass the computed row displayName into `deriveSessionTitle` or equivalent title logic, and add regression coverage for metadata stripping plus external displayName precedence/fallback.
