## Summary

Keep open. Current main still resolves runtime assistant identity with global `ui.assistant` name/avatar before `agents.list[].identity`, and the Agents panel still lets that runtime identity override the per-agent list payload. Emoji-only identities also still render as `—` in the Agent Context avatar row. The closed fix PR #54730 was not merged, and open PR #72579 is an implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open. The best fix is a focused change that makes non-default agents prefer `agents.list[].identity` name/avatar/emoji over global `ui.assistant` in `resolveAssistantIdentity()` and `agent.identity.get`, preserves default-agent global override behavior, updates `buildAgentContext()` to show emoji/avatar state correctly, and adds regression tests for backend resolver precedence, RPC output, and Agent Context rendering.
