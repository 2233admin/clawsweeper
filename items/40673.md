## Summary

Keep open. Current main partially addresses the runtime toggle by treating `OPENCLAW_RAW_STREAM=0` as disabled, but the issue’s maintainer-actionable surface is still unresolved: the public docs still only show enabling raw stream logging, the LaunchAgent/durable environment path can still source persisted values from state-dir `.env` or config env, and there is no focused regression coverage for the reported disable/restart path.

## Best Possible Solution

Keep the issue open and resolve it with a narrow docs-and-regression pass: document that raw stream logging is enabled only by truthy values such as `1`, `true`, `yes`, and `on`, and disabled by unset or non-truthy values such as `0`, `false`, and `off`; explain LaunchAgent/service precedence with `launchctl`, `~/.openclaw/.env` / `$OPENCLAW_STATE_DIR/.env`, and config `env`; add focused tests for `appendRawStream` disabled values and managed gateway service env planning/restart behavior.
