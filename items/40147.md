## Summary

Keep this PR open. Current main still forces Feishu thread replies from either `thread_id` or `root_id`, including when `replyInThread` is disabled, and the open related #39507 is an overlapping but conflicting semantics proposal rather than a clear superseding duplicate. This PR needs a current-main refresh and maintainer product decision on root/thread semantics before it can land or be consolidated.

## Best Possible Solution

Keep this PR open for maintainer review or explicitly consolidate it with #39507 after deciding Feishu root/thread semantics. A current-main fix should update `extensions/feishu/src/bot-content.ts` and `extensions/feishu/src/reply-dispatcher.ts` together, scope any `root_id` hard-forcing to the intended group surfaces, preserve or intentionally change DM quote behavior, replace the current thread-id-only disabled expectation, and add regression coverage for normal groups, topic groups/topic-sender routing, and p2p quote replies.

## Risks / Open Questions

- The PR is based on an older resolver location: current main imports `resolveFeishuGroupSession` from `extensions/feishu/src/bot-content.ts`, while the PR patch targets `extensions/feishu/src/bot.ts`, so it likely needs a refresh before review can be completed.
