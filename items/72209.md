## Summary

Keep PR #72209 open. Current main does not implement passive session recovery checkpoints, recovery bundle/ledger storage, or session recovery prompt discipline. The related items are adjacent session reset/snapshot/compaction/background-task work, not a canonical replacement, and the existing Greptile comments are actionable P2 review feedback rather than evidence that the PR is obsolete.

## Best Possible Solution

Keep PR #72209 open. The best path is to preserve the passive, informational recovery design while asking the contributor to address the two existing P2 comments in `src/agents/session-recovery-state.ts`: narrow free-text redaction so task context is not corrupted, and make best-effort checkpoint writes non-blocking or async. After that, maintainer review should focus on the recovery prompt contract, state-file schema, and whether this belongs as core agent-session infrastructure.
