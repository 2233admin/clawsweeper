## Summary

Keep #68996 open. Current main already has baseline Gemma 4 support from the shipped Google provider work, but it does not implement this PR's remaining explicit-config transport normalization for Gemma models pinned to `api: "openai-completions"`. The PR is recent, includes concrete author verification, and the review comments identify changes needed before it could be merged safely rather than evidence that the work is obsolete.

## Best Possible Solution

Keep the PR open, but revise it into a narrower Google-owned fix: preserve shipped dynamic Gemma support, only switch explicit Gemma configs when the model API is actually moved to `google-generative-ai`, normalize or validate the compatible native baseUrl at the same time, and add regression coverage for direct Google explicit configs plus `google-gemini-cli` template-source behavior.
