## Summary

Keep open. Current `main` has not implemented this cleanup: `skills/gh-issues/SKILL.md` still contains multiple `/data/.clawdbot` references even though repo docs and path resolution treat `~/.openclaw` / `$OPENCLAW_STATE_DIR` as the canonical state location. The submitted PR is also incomplete as shown in the GitHub patch: it leaves several legacy-path references untouched and changes one fallback section into a duplicate no-op lookup.
