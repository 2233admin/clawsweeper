## Summary

Keep #72419 open. It is a fresh, focused gateway/security PR linked to #72418, with active author follow-up and bot review context. Current main still has the direct-local backend self-pairing bypass for shared token/password auth, and the PR diff in the provided context appears to add bootstrap-token to that bypass path rather than clearly replacing the shared-secret bypass called out by the issue.

## Best Possible Solution

Keep the PR open for maintainer/security review. The likely product direction is to decide whether backend self-pairing bypass should move from shared gateway token/password to a dedicated server-controlled credential such as bootstrap-token, then update handshake-auth-helpers.ts, message-handler integration tests, the compat baseline, callGateway device-identity behavior, and gateway protocol docs together. At minimum, require a regression test showing a spoofed direct-local backend client using only shared token/password can no longer preserve operator.admin without a registered device or dedicated backend credential.
