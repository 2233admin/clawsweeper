## Summary

Keep open for maintainer/security review. This PR is paired with open issue #72418 using closing syntax, and the final diff appears to add bootstrap-token to the backend self-pairing bypass while leaving the shared token/password bypass that the linked report challenges. Current main still documents and tests that shared-token direct-local backend skip as intentional behavior, so this needs explicit maintainer judgment rather than cleanup closure.

## Best Possible Solution

Keep this PR open and review it together with #72418. Maintainers should either revise the fix to remove or tightly narrow the shared-token/password backend self-pairing skip, preferably binding it to a verifiable backend credential such as a dedicated bootstrap path with regression tests for spoofed gateway-client/backend loopback connections without a registered device, or explicitly reject/document the shared-token exception as accepted product behavior.
