## Summary

Keep this PR open. Current main still has no reset-archive fallback in `readSessionMessages`: it resolves primary transcript candidates, reads the first existing primary file, and returns `[]` when none exists. Reset still archives transcripts with `.reset.<timestamp>`, and the affected `chat.history` and sessions-history surfaces still call this helper. The PR head remains a focused implementation candidate, with related work complementary rather than superseding.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for reading the latest reset archive when `readSessionMessages` is asked for a session whose primary transcript is missing. Maintainers should review/rebase it, run the gateway changed checks, and either land this patch or fold equivalent fallback behavior into a broader reset-archive recovery change while leaving UI/archive listing and logical-chain metadata to related follow-ups.
