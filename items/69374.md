## Summary

Keep #69374 open. Current main still does not carry WhatsApp `contextInfo.forwardingScore` or `isForwarded` from the Baileys inbound envelope into `WebInboundMessage`, the canonical inbound hook context, `inbound_claim`, or `message_received`. The issue is recent, unlabelled, non-maintainer-authored, and the related local reports are adjacent WhatsApp/plugin metadata work rather than a canonical duplicate for forwarding metadata.

## Best Possible Solution

Add an explicit WhatsApp forwarding carrier through the existing hook pipeline: extract `contextInfo.forwardingScore` and deprecated `isForwarded` in the WhatsApp inbound normalization path, store them on `WebInboundMessage` and `CanonicalInboundMessageHookContext`, expose them in `inbound_claim` and `message_received` metadata, and add focused WhatsApp extraction plus hook-mapper tests. Keep this in OpenClaw rather than ClawHub because the missing piece is core/bundled channel-to-plugin contract plumbing, not an optional external plugin.
