## Summary

Keep #69374 open. Current main still does not expose WhatsApp forwardingScore/isForwarded to plugin inbound_claim or message_received hooks: WhatsApp contextInfo is used for mentions and quoted replies, but no forwarding fields are extracted, carried on the inbound message/context types, or emitted in hook metadata.

## Best Possible Solution

Implement the request as an additive WhatsApp/plugin hook metadata path: extract forwardingScore/isForwarded from Baileys contextInfo in the WhatsApp inbound normalization path, carry the values through WebInboundMessage, FinalizedMsgContext, and CanonicalInboundMessageHookContext, then include them in inbound_claim and message_received metadata. Add focused tests for WhatsApp extraction and hook mapper output, and update plugin hook/WhatsApp docs if the new metadata is documented publicly.
