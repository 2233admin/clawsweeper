## Close Comment

Closing from a Codex review. The upgrade-driven WhatsApp session-loss regression described here is already addressed on current `main`, and the changelog shows the fix shipped in `v2026.4.15`: `CHANGELOG.md:509` records the WhatsApp auth-persistence/backup-recovery work, and `CHANGELOG.md:582` records the queued-`creds.json` bootstrap race fix. The current implementation now keeps default WhatsApp auth under the stable credentials path (`extensions/whatsapp/src/accounts.ts:81`, `src/config/paths.ts:236`), preserves legacy root creds as a fallback during migration (`extensions/whatsapp/src/accounts.ts:109`), and has an explicit legacy move plan into `credentials/whatsapp/default/` (`extensions/whatsapp/src/state-migrations.ts:24`). If this still reproduces on a current release, please open a new issue with fresh version details and current logs.
