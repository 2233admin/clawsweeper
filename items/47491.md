## Summary

Keep open. Current `main` only partially covers the PR: `before_reset` and the main compaction hook helpers already resolve/pass agent context correctly, but the embedded subscribe compaction hooks still drop `agentId` and `sessionId`, so the request is not implemented on `main` well enough to close.

## Evidence

- **Embedded subscribe compaction hooks still omit agent context:** `handleCompactionStart` and `handleCompactionEnd` call plugin compaction hooks with `{ sessionKey }` only; they do not forward `agentId` or `sessionId` even though both are available on subscribe params.
  - file: [src/agents/pi-embedded-subscribe.handlers.compaction.ts:26](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/pi-embedded-subscribe.handlers.compaction.ts#L26)

  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)

## Risks / Open Questions

- Closing this PR would leave embedded auto-compaction hooks inconsistent with the other compaction paths, and plugins triggered from that path would still miss `agentId`/`sessionId` context.
- Because current tests only assert `sessionKey` for the embedded hook wiring, the remaining gap is not covered by regression tests on `main`.
