## Summary

Keep open. Current `main` at `6944d7025d9c0ba4891cbe646ab5dab039d504ca` still explicitly rejects Brave `llm-context` `freshness` and `date_after`/`date_before`, the llm-context request path still lacks date-range plumbing, and the docs still document those filters as rejected. This PR is not implemented on main and is not a strong close candidate under the allowed reasons.
