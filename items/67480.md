## Summary

Keep this PR open. Current main still applies the nested `dir/skills` redirect before checking whether the configured root already contains loadable root-level skills, so the reported root-plus-nested layout can still skip valid skills. The PR is a focused implementation candidate with a targeted regression test, and there is no evidence that the fix has already shipped or been superseded.

## Best Possible Solution

Keep this PR open and review or land a version that verifies the configured directory is already a loadable skills root before redirecting to its nested `skills` directory. Preserve the new root-plus-nested regression test, repo-style nested root support, root-containment checks, byte caps, and existing scan/load limit behavior.
