## Summary

Keep #57553 open. Current main still exposes brew-only skill installers during Docker/Linux-container onboarding and still steers selected brew-backed installs toward a Homebrew prompt. The linked bug #14593 has a concrete official-Docker reproduction, and this PR is a focused attempt to address that remaining behavior rather than obsolete or superseded work.

## Best Possible Solution

Keep the PR open and use it as the narrow fix path for #14593: hide brew-only skill install options during Linux container onboarding when brew is unavailable, show a container-specific note, and keep the normal Homebrew prompt outside container contexts. Before merge, refresh it on current main, rerun the targeted onboarding/container tests, and decide whether to reuse the existing gateway container detection helper or keep the new daemon helper.
