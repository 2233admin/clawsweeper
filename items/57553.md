## Summary

Keep open. Current `main` still lacks the Linux-container-specific onboarding behavior this PR adds, so the reported broken Homebrew path in Docker remains plausible and the PR is not stale or already implemented.

## Evidence

- **Onboarding still exposes brew-backed installs without a container guard:** `setupSkills` still marks all missing skills with install metadata as installable, and only later shows the generic Homebrew prompt when a selected install uses `brew` and `brew` is missing. There is no Linux-container branch and no filtering of brew-only installs.
  - file: [src/commands/onboard-skills.ts:84](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/commands/onboard-skills.ts#L84)

  - file: [src/daemon/container-context.ts:3](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/daemon/container-context.ts#L3)
