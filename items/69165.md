## Summary

Keep open. Current main has adjacent Telegram retry and durability pieces, but it does not implement the requested Telegram outbound scheduler with per-chat/topic serialization, configurable rate/depth limits, drop-oldest best-effort behavior, or queueing shared across cron/message-tool/direct outbound, final replies, previews, and media sends.

## Best Possible Solution

Keep #69165 open. Implement a Telegram-owned outbound scheduler shared by direct outbound, message tools, cron/heartbeat delivery, bot final/block replies, previews, and media sends. Key it by Telegram account plus chat/topic, serialize same-key sends, enforce configurable per-chat/group/global pacing, honor Telegram retry_after before retrying, integrate with the existing write-ahead delivery queue for crash recovery, and add config/docs/tests for bestEffort, maxQueueDepth, and drop-oldest behavior.
