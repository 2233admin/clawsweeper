## Summary

Keep open. Current `main` already appears to keep Mattermost DMs on the base session, so the session-splitting part of the PR looks at least partly obsolete, but the rest of the PR is not on `main`: current Mattermost inbound handling still does not preserve replied-post body/sender context, the typing payload still omits root `parent_id: ""`, and the websocket monitor still lacks the liveness heartbeat added by the PR. That is useful, unshipped work, so this is not a strong close candidate.
