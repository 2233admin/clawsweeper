## Summary

Keep open. Current main still builds gateway install environments with literal managed env values and then serializes non-empty environment entries into the LaunchAgent plist. The PR addresses a real remaining bug from #70612, but its current purge approach has an unresolved review concern around deleting service-owned vars such as PATH, so it needs maintainer review or a small adjustment rather than closure.

## Best Possible Solution

Keep the PR open and finish it in the gateway install environment builder: remove literal values for SecretRef/.env-managed keys before plist serialization, but protect generated service-owned variables from the purge. Add regression coverage for managed secrets not appearing in plan.environment plus coverage that PATH/HOME/TMPDIR and required service env fields survive.
