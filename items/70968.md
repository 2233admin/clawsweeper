## Summary

Keep PR #70968 open. Current main still builds the gateway install environment by copying managed dotenv/config/auth variables into the LaunchAgent environment, then only adds OPENCLAW_SERVICE_MANAGED_ENV_KEYS; it does not remove the literal managed values. The linked bug #70612 is closed, but current source and tests still match the reported stale/embedded-secret behavior, and the PR contains a focused fix direction that should either be merged after review cleanup or superseded by an equivalent main change.

## Best Possible Solution

Keep the PR open or replace it with an equivalent main-branch fix. The maintainer path should remove literal values for managed dotenv/config/auth keys from the gateway install plan before LaunchAgent plist serialization, preserve generated service-owned variables such as PATH/HOME/TMPDIR/OPENCLAW_PORT as intended, and update daemon-install-helpers regression tests to assert that managed secrets are represented only through OPENCLAW_SERVICE_MANAGED_ENV_KEYS.
