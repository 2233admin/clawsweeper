## Summary

Keep open. Current `main` still does not expose or consume a Signal `configPath` setting, so the requested behavior is not implemented. The PR also appears stale against the current plugin layout because Signal runtime code now lives under `extensions/signal/` rather than the old `src/signal/` paths in the PR.
