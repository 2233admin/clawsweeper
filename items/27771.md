## Summary

Keep open. Current main still does not implement `channels.signal.configPath` or pass `--config` to signal-cli daemon startup, while the PR discussion describes a valid user need and an earlier maintainer/Codex review already confirmed the feature remains unimplemented after Signal moved into the bundled plugin layout.

## Best Possible Solution

Keep the PR open or ask the contributor to rebase/adapt it. The best product path is to implement `channels.signal.configPath` in the current Signal bundled plugin and config contract, trim/omit empty values, pass it before daemon startup as `signal-cli --config <path>`, update docs/generated config metadata, and add focused Signal autostart tests covering present and blank/unset values.
