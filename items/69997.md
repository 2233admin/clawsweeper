## Summary

Keep open. Current main still has the unsafe stored-model override normalization path, so the central bug-fix idea is not implemented or superseded. The PR should not be merged as-is because it bundles unrelated heartbeat behavior changes that appear to alter current empty-object heartbeat semantics; related PR #68300 is only hook-error logging/context work, not a replacement fix.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up, but require it to be split and narrowed: one focused patch should harden stored model override normalization and any direct fallback callers, with regression tests for malformed session-store values. The heartbeat changes should be removed from this PR and handled separately with explicit tests/product review if #64293 still needs a behavior change.
