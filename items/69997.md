## Summary

Keep #69997 open. The underlying model-override crash path still appears plausible on current main because session-store JSON is accepted as a top-level object and stored override fields are later trimmed as strings. However, the PR should not merge as-is: it bundles the focused model-selection guard with unrelated heartbeat behavior changes that conflict with current documented/current semantics and were already flagged in review.

## Best Possible Solution

Keep the PR open but require a focused rework: drop or split the heartbeat commit, add regression tests for malformed providerOverride/modelOverride values loaded from the session store, and preferably sanitize these fields at the session-store/load or shared normalization boundary so all display, gateway, and live-switch paths share the same defensive behavior.
