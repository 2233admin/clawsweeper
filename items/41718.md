## Summary

Keep open. Current `main` still intentionally clears the gateway token when the Gateway URL changes, and that behavior is backed by current tests and token-scoping logic. This PR's UI change would regress the per-gateway token isolation contract, while its separate LanceDB retry fix appears to already be implemented on `main`, so the PR needs maintainer judgment or a split rather than auto-close.
