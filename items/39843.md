## Summary

Keep open. Current main still starts a Google Chat typing message before dispatch, exact `NO_REPLY` final payloads can skip delivery, and the Google Chat monitor does not clean up the stored `typingMessageName` outside the delivery callback. Open PR #39891 is still the concrete unmerged fix path.

## Best Possible Solution

Keep #39843 open until PR #39891 or an equivalent Google Chat monitor fix lands. The fix should delete any remaining `typingMessageName` after dispatch completes without delivery, or wire the existing `onTypingCleanup` hook, and add a focused regression test for the emoji reaction plus exact `NO_REPLY` path.
