## Summary

Keep open. The issue is a fresh regression report with concrete Slack thread-routing evidence, and current main still has the specific gap described in the latest comment: exec-event delivery context is available, but explicit heartbeat targets do not preserve the event thread route into Slack delivery.

## Best Possible Solution

Keep the issue open and fix the event-driven heartbeat path. For reason exec-event or pending system events with deliveryContext, the delivery target should prefer that event route, or at least preserve its threadId when the configured heartbeat target/to resolves to the same Slack channel. Add a regression test for Slack explicit heartbeat target plus exec-event deliveryContext with threadId, and keep separate coverage proving scheduled explicit heartbeats do not inherit stale threads.
