## Summary

Keep #72277 open. The report has a concrete v2026.4.24 Slack-thread repro and current main still appears to drop the pending exec-event delivery context when heartbeat delivery is configured with an explicit Slack target/to instead of target:last.

## Best Possible Solution

Keep the issue open and fix the heartbeat delivery resolution so exec-event pending system events with deliveryContext preserve their route, including Slack threadId/thread_ts, even when heartbeat.target is an explicit Slack target with explicit to. Add a regression test for Slack target:"slack" + explicit to + queued exec-event deliveryContext.threadId, and keep a separate assertion that normal scheduled heartbeats do not inherit stale Slack threads.
