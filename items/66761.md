## Summary

Current main still has the unguarded `provider?.models?.some(...)` call, so the crash described by the linked HTTP chat-completions bug remains reachable. This PR is a focused implementation candidate with matching regression tests and no evident security-sensitive changes in the provided diff.

## Best Possible Solution

Keep this PR open for maintainer review and land the `Array.isArray(provider?.models)` guard with the included regression coverage, or apply an equivalent main-branch fix that prevents non-array runtime provider model values from crashing provider policy lookup. Treat upstream provider normalization as a follow-up.
