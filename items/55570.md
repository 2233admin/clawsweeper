## Summary

Current `main` does not implement the Slack `ignoreOtherMentions` option. The Slack runtime, config types/schema, and docs still lack this behavior, so there is not strong evidence to close PR #55570 as already handled or obsolete.

## Evidence

- **Slack runtime still has no `ignoreOtherMentions` gate:** `prepareSlackMessage` handles mention gating via `requireMention` and returns early for missing mentions, but then proceeds directly into thread/content handling. There is no Slack-side drop path for messages that mention other users/bots without mentioning this bot.
