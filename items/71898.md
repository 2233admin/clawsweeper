## Summary

Keep PR #71898 open. Current main still accepts `corpus=sessions` and filters memory-manager hits to session sources, but the final `memory_search` result shaping still hard-codes every memory-manager hit as `corpus: "memory"`. The PR is a narrow, active bug fix with a focused regression test, and the bot readability comment was addressed in the PR context by moving the test into a dedicated corpus-label describe block. Related memory/session reports are broader or orthogonal and do not supersede this PR.

## Best Possible Solution

Keep #71898 open and review or land this PR, or an equivalent patch, after normal CI. The best code direction is to preserve `MemorySearchResult.source` when shaping `memory_search` results and keep the focused regression proving `corpus=sessions` returns `corpus: "sessions"`; the existing Greptile readability concern appears addressed in the provided PR context.
