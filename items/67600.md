## Summary

Keep open. Current `main` still bypasses the configured default exec timeout when `background`/`yieldMs` is used without an explicit per-call `timeout`, and the repository includes a regression test that asserts that exact behavior. The docs still describe `tools.exec.timeoutSec` as the exec default, so the report remains valid rather than obsolete.

## Evidence

- **Current exec implementation nulls the timeout on auto-backgrounded runs:** `createExecTool` computes `backgroundTimeoutBypass` when backgrounding is allowed and no explicit per-call timeout is set, then sets `effectiveTimeout` to `null` instead of `defaultTimeoutSec` for `background` or `yieldMs` requests.
