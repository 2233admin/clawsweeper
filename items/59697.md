## Summary

Keep PR #59697 open. Current main still does not implement the PR's explicit user-message `<relevant-memories>` sanitization in either `chat.history` or the WebUI extraction helper, and the overlapping related PRs have not established a single canonical replacement. The PR is useful but not merge-ready as-is because review feedback flags unconditional leading-whitespace trimming.

## Best Possible Solution

Keep the item open and consolidate the overlapping PRs around one maintainer-approved fix: introduce or export a neutral shared helper for user-visible memory-scaffolding cleanup, apply it in both `src/gateway/chat-sanitize.ts` and `ui/src/ui/chat/message-extract.ts`, preserve user-authored leading whitespace and literal/code-fence tag text, and add focused regression tests for string and array user content. After one canonical implementation is chosen, close the duplicate PRs with links to that chosen item.
