## Summary

Keep this issue open. Current main has partial progress, including fallback lifecycle events, plugin tool hooks, direct-edit config recovery, and tool loop detection, but multiple central platform requests remain unresolved or only partially covered. The discussion also says the failover follow-up is intentionally narrow and does not cover non-verbose fallback notices or workspace-context re-injection.

## Best Possible Solution

Keep this issue open for maintainer triage, or replace it with focused canonical issues before closing the umbrella. Split remaining work by surface: fallback visibility and fallback-specific bootstrap/context re-injection, config-write health rollback, graduated crash recovery, doctor dry-run/diff preview, per-path filesystem policy across native and MCP tools, repeated sub-agent failure circuit breaking, prompt-injection scanning policy, spend/budget gates, and scheduled model-health routing. Treat #70976/#70990 as covering only the failover hook slice unless and until merged code/docs explicitly cover the broader fallback and reliability requests.
