## Summary

Keep #40289 open. Current main still drops Discord component/card text before button/select interactions reach the ordinary agent fallback path. The PR is stale against the current `extensions/discord` layout, but the reported bug remains real and is not superseded by the related broader Discord metadata or referenced-message issues.

## Best Possible Solution

Keep the PR open, but require a rebase/rework against `extensions/discord`: add structured card/context text to `DiscordComponentEntry`, collect `spec.text`, fallback text, and relevant text/section block content in `buildDiscordComponentMessage`, preserve it through the registry, append it in fallback event text for button/select interactions, and update `extensions/discord/src/components.test.ts` plus `extensions/discord/src/monitor/monitor.test.ts`. The prompt-guidance part of the PR should be reconciled with the current Discord `messageToolHints` path rather than copied directly from the old `src/agents/system-prompt.ts` patch.
