## Summary

Keep open. Current main still drops Discord component card text before ordinary button/select fallback events reach the agent. The PR addresses a real product gap, but it targets the removed `src/discord` layout and needs to be ported into the current `extensions/discord` plugin implementation with regression tests. Security review found no supply-chain-sensitive changes in the provided diff, but the stale path makes the PR unsafe to merge as-is.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into the current Discord plugin. The best patch would add structured card-context storage to `DiscordComponentEntry`, collect `spec.text`, fallback text, and visible text/section block content in `extensions/discord/src/components.ts`, preserve it through the registry, include it in ordinary fallback event text from `extensions/discord/src/monitor/agent-components.ts`, and add regression coverage in `extensions/discord/src/components.test.ts` and `extensions/discord/src/monitor/monitor.test.ts`. Any prompt guidance should use the current Discord plugin `messageToolHints` path rather than the stale prompt hunk alone.
