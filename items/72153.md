## Summary

Keep #72153 open. Current main has not absorbed the fix: the Windows config opener still emits `Start-Process -LiteralPath`, the regression test still locks that form, and the dashboard Open Config flow reaches this handler. PowerShell metadata/docs support the PR's `-FilePath` change, and linked #69933 contains the matching Windows failure.

## Best Possible Solution

Keep the PR open for maintainer review and merge once validation is acceptable, or land the same minimal change on main: replace `Start-Process -LiteralPath` with `Start-Process -FilePath` in `resolveConfigOpenCommand()` while preserving `escapePowerShellSingleQuotedString()`, and update `src/gateway/server-methods/config.test.ts` to assert `-FilePath`.
