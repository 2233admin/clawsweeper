## Summary

Keep #72153 open. Current main still contains the Windows `config.openFile` bug: the gateway builds `Start-Process -LiteralPath ...`, the regression test still asserts that invalid parameter, and the dashboard action reaches this gateway method. PowerShell accepts `-FilePath` and rejects `-LiteralPath` for `Start-Process`, so the PR remains a useful focused fix rather than cleanup/obsolete work.

## Best Possible Solution

Keep the PR open for maintainer review and merge, or land the same minimal fix on main: change the Windows branch of `resolveConfigOpenCommand()` to use `Start-Process -FilePath ...` while preserving the existing single-quote escaping, and update `src/gateway/server-methods/config.test.ts` to assert `-FilePath`. Since linked #69933 is closed but current main still has the failing command, #72153 should not be closed as obsolete.
