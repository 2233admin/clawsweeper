## Summary

Keep open. Current main has manual/daily/idle session reset, abort tracking, and timeout/overflow compaction recovery, but I did not find the requested gateway circuit breaker that counts consecutive aborts per session and automatically rotates to a fresh session after a configurable threshold.

## Best Possible Solution

Keep #59161 open or consolidate it into a canonical session-level breaker issue. The best implementation would add a documented config surface for the abort threshold, persist per-session consecutive-abort state, reset the counter on successful runs or explicit reset, call the existing session reset path when the threshold trips, and emit a visible/logged breaker event so operators know why a fresh session was created.
