## Summary

Current `main` still has the reported empty-modal failure path, so this is not implemented already, but the PR as submitted appears to introduce at least two behavior regressions called out in review: it would silently drop malformed `modal.fields` values and it risks downgrading container-only component specs that rely on top-level text fallback. That leaves the item useful but not ready for conservative cleanup closure.

## Evidence

## Risks / Open Questions

- The underlying empty-default Discord bug still appears real on current `main`, so closing this PR would discard useful work before the behavior is covered another way.
- The PR needs follow-up before merge: keep empty-array-as-absent handling if desired, but preserve strict errors for malformed `modal.fields` and add coverage for container-only specs that rely on top-level content fallback.
