## Summary

Keep open. Current main and the latest release still require `channels.feishu.accounts.<id>.name` under a strict per-account schema, and there is no doctor migration for legacy `botName`, so the upgrade breakage reported in #56942 is not implemented away on `main`.

## Evidence

- **Current Feishu account schema is strict and only allows `name`:** `FeishuAccountConfigSchema` defines `name` and ends with `.strict()`. A legacy per-account `botName` key would still fail as an additional property, matching the report.
  - file: [extensions/feishu/src/config-schema.ts:191](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/feishu/src/config-schema.ts#L191)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '191,206p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Runtime account resolution consumes `name`, not `botName`:** Feishu account resolution reads `(merged as FeishuAccountConfig).name` and returns it as the account display name. There is no config alias for `botName` here.
  - file: [extensions/feishu/src/accounts.ts:269](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/feishu/src/accounts.ts#L269)
