## Summary

Keep open. Current main still appears to route `openclaw --profile <name> gateway restart` through the inherited launchd label from the running agent/gateway process, so the requested profile can still be ignored inside an agent session. The CLI does set `OPENCLAW_PROFILE`, but the restart path prefers `OPENCLAW_LAUNCHD_LABEL` over that profile, and agent exec inherits the service env that contains that label.

## Evidence

- **CLI does apply `--profile` early:** `runCli()` parses `--profile` and calls `applyCliProfileEnv()` before dispatch, so the report is not obsolete because of missing CLI parsing.
  - file: [src/cli/run-main.ts:156](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/cli/run-main.ts#L156)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Gateway service env exports the current launchd label:** `buildServiceEnvironment()` always includes `OPENCLAW_LAUNCHD_LABEL` alongside `OPENCLAW_PROFILE`, so commands executed from the managed gateway process inherit the current profile label.
  - file: [src/daemon/service-env.ts:269](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/daemon/service-env.ts#L269)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Agent exec inherits base process env:** The exec tool starts from `process.env` and sanitizes it, but keeps non-blocked inherited vars; there is no removal of `OPENCLAW_LAUNCHD_LABEL` before launching host commands.
  - file: [src/agents/bash-tools.exec.ts:1536](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/bash-tools.exec.ts#L1536)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Launchd restart still prefers label over profile:** `resolveLaunchAgentLabel()` checks `OPENCLAW_LAUNCHD_LABEL` first, then `OPENCLAW_PROFILE`; `restartLaunchAgent()` uses that resolved label as the restart target.
  - file: [src/daemon/launchd.ts:48](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/daemon/launchd.ts#L48)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Behavior is explicitly covered by current test:** Current test fixture asserts that `OPENCLAW_LAUNCHD_LABEL` wins over `OPENCLAW_PROFILE`, which matches the issue’s failure mode rather than contradicting it.
  - file: [src/daemon/launchd.test.ts:832](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/daemon/launchd.test.ts#L832)
