## Summary

Keep open. Current main still contains the reported stop-timeout failure mode: the timeout branch marks the account running and returns before clearing the task/abort maps, while the start path still silently returns when a task entry exists. The existing regression test also asserts the buggy behavior. PR #71456 is the plausible fix, but it is open/unmerged in the provided GitHub context and the equivalent change is not present on current main.

## Best Possible Solution

Keep #71412 open and review or land PR #71456, or an equivalent patch, that clears the stale store.aborts and store.tasks entries on stop timeout, records the account as stopped instead of running, and updates the regression test to require a fresh task to start after the timeout. Treat the doctor/liveness probe improvement as a separate follow-up unless maintainers intentionally bundle it.
