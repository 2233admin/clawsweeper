## Summary

Current `main` does not implement this PR’s behavior. The web UI still accepts only `image/*`, sends attachments only as `type: "image"`, the chat renderer still extracts/renders only images, and gateway `chat.send` still persists/transcripts only parsed images. This is not strong evidence for closure under the allowed reasons.
