## Summary

Keep open. Current `main` still has the reported alias/canonical session-key mismatch in WebChat, and the proposed PR as described is not strong enough to close conservatively because session aliasing is runtime-configurable, not just `main` ↔ `agent:main:main`.

## Evidence

- **Current main still uses strict session-key equality for chat events:** `handleChatEvent()` immediately drops events unless `payload.sessionKey === state.sessionKey`, so alias/canonical mismatches are still rejected on current `main`.
  - file: [ui/src/ui/controllers/chat.ts:433](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/ui/src/ui/controllers/chat.ts#L433)

- **UI sends whatever session key it is holding:** `requestChatSend()` sends `state.sessionKey` directly in `chat.send`, so a UI session set to `main` will send the alias as-is.
  - file: [ui/src/ui/controllers/chat.ts:236](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/ui/src/ui/controllers/chat.ts#L236)
