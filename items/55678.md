## Summary

Keep open. Current `main` still does not implement DashScope/Qwen provider-usage integration for the UI/status usage surfaces described in the issue. The issue title may be conflating provider quota display with per-session token accounting, but the requested DashScope usage API support is not present on `main`.

## Evidence

- **Core usage-provider allowlist excludes DashScope/Qwen:** The supported `UsageProviderId` union and `usageProviders` list only include `anthropic`, `github-copilot`, `google-gemini-cli`, `minimax`, `openai-codex`, `xiaomi`, and `zai`; there is no `dashscope`, `qwen`, or equivalent alias.
  - file: [src/infra/provider-usage.types.ts:20](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/infra/provider-usage.types.ts#L20)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Loader only fetches usage for that allowlist:** `loadProviderUsageSummary()` defaults to `usageProviders`, so unsupported providers are never queried unless explicitly added to the core list.
  - file: [src/infra/provider-usage.load.ts:95](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/infra/provider-usage.load.ts#L95)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Docs still list supported provider-usage surfaces without Qwen/DashScope:** The usage-tracking doc enumerates supported provider usage/quota integrations and omits Qwen/DashScope entirely, which matches the current code.
  - file: [docs/concepts/usage-tracking.md:30](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/docs/concepts/usage-tracking.md#L30)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Qwen bundled plugin does not register usage hooks:** The Qwen plugin entry defines auth/catalog/media/video behavior, but there is no `resolveUsageAuth` or `fetchUsageSnapshot` implementation in the plugin entry.
  - file: [extensions/qwen/index.ts:49](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/qwen/index.ts#L49)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Usage surfaces depend on provider-owned hooks for unsupported core providers:** Provider plugins can participate in usage reporting only via `resolveUsageAuth` and `fetchUsageSnapshot`; since Qwen does not implement those hooks, there is no plugin-owned DashScope usage snapshot path on `main`.
  - file: [src/plugins/types.ts:1388](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/plugins/types.ts#L1388)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Adjacent token-accounting support exists, but it is distinct from provider quota UI:** Tests show native DashScope/Qwen endpoints enable streaming usage compatibility for per-turn token accounting, which suggests the title may be broader than the actual missing feature. That does not add DashScope provider usage/quota snapshots.
  - file: [src/agents/openai-transport-stream.test.ts:1422](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/openai-transport-stream.test.ts#L1422)
