## Summary

Keep open. Current main has partial mitigations, but not a strong default fix for #55694: tool-loop detection is still opt-in, repeated generic tool failures are only warned on before a much later breaker, tool validation failures are still returned as ordinary tool errors, and the embedded runner still allows 32+ retry-loop iterations.

## Evidence

- **Tool-loop detection is off by default:** `docs/gateway/config-tools.md:114` says tool-loop safety checks are disabled by default and require `tools.loopDetection.enabled: true`. `src/agents/tool-loop-detection.ts:33` also sets `enabled: false` in the default config.
  - file: [docs/gateway/config-tools.md:114](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/docs/gateway/config-tools.md#L114)
