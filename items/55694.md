## Summary

Keep open. Current `main` has partial loop-mitigation work, but the generic tool-loop detector is still opt-in, and the always-on safety net only covers unavailable-tool loops, not repeated validation failures on an existing tool like `exec` missing `command`.

## Evidence

- **Loop detection is still disabled by default:** The user-facing docs and config type both say `tools.loopDetection.enabled` defaults to `false`, so the generic repeat / no-progress protections do not apply unless the operator explicitly enables them.
