## Summary

Keep open. Current `main` still contains the exact SSH-mode rewrite path the issue describes: Bonjour selection and SSH-mode config sync both write `gateway.remote.url` to the discovered host instead of preserving the loopback tunnel URL, while the runtime endpoint/docs still treat SSH mode as `ws://127.0.0.1:<port>`. The issue discussion’s linked fix PR does not change the fact that current `main` remains inconsistent with the documented SSH-tunnel contract.
