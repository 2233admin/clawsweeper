## Summary

Keep open. Current `main` still hardcodes `minimal: 128` for Gemini 2.5 Pro and Flash thinking budgets, and Gemini 2.5 requests still flow through the integer `thinkingBudget` path rather than the Gemini 3 `thinkingLevel` path. This PR is a small, plausible bug fix with matching regression coverage, not an obsolete or already-shipped change.

## Evidence

- **Current main still uses 128 for Gemini 2.5 minimal budgets:** `getGoogleThinkingBudget()` on current `main` returns `minimal: 128` for both `modelId.includes("2.5-pro")` and `modelId.includes("2.5-flash")`; the proposed PR changes exactly those two constants to 512.
