## Summary

Keep PR #70629 open. Current main still emits `thinkingBudget: 128` for `reasoning: "minimal"` on Gemini 2.5 Pro/Flash-family models, and no current main test covers the proposed 512 floor. The PR is not implemented, duplicated, or superseded by the related Gemini 2.5 Pro zero-budget fix, which addressed a different `thinkingBudget=0` failure mode.

## Best Possible Solution

Keep the PR open, but review it as a real provider bug fix rather than auto-closing it. The best path is a narrow Google plugin change in `extensions/google/transport-stream.ts` plus regression tests for `gemini-2.5-flash-lite`, `gemini-2.5-flash`, and `gemini-2.5-pro`; maintainers should confirm whether raising all Gemini 2.5 minimal budgets to 512 is intended or whether the fix should be model-family-specific.
