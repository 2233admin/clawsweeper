## Summary

Keep this PR open. Current main still has no direct config-warning dedupe in config IO: valid config warnings are logged unconditionally in both the load and write paths, and current tests only cover newline formatting. The PR remains a relevant focused implementation candidate, but it needs revision before merge to clear dedupe state on failed/early/no-warning transitions and to decide whether the write path should share the guard.

## Best Possible Solution

Keep this PR open for maintainer review, but ask for a revised patch that centralizes config-warning fingerprinting per config path, clears the fingerprint on every no-warning and failed/early-load path, re-logs when raw config content or formatted warning details change, and adds focused tests for repeated load plus warning -> no-warning/invalid/missing -> same-warning transitions. Maintainers should also decide whether `writeConfigFile()` warnings should share the same dedupe guard. If #38295 becomes the canonical broader fix, close this PR only after that fix lands or maintainers explicitly supersede it.
