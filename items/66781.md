## Summary

Keep open. Current main has a targeted Active Memory mitigation and new local onboarding defaults, but the broader least-privilege default requested here is still not implemented: unset/`full` profiles remain unrestricted, no-policy matching allows tools, empty runtime `toolsAllow` preserves the full tool list, and docs still describe broad sub-agent behavior when no restrictive profile is configured.

## Best Possible Solution

Keep this issue open for an explicit maintainer product decision on least-privilege tool defaults. A complete fix should define the desired default per-agent/per-run policy, preserve a deliberate escape hatch such as `tools.profile: "full"`, migrate or warn existing broad configs, update schema/help/public docs, and add regression coverage for core runs, embedded runs, plugin/MCP tools, and sub-agent policy resolution.
