## Summary

Keep PR #59115 open. Current main still drops Slack forwarded/shared attachment content from thread history: the thread-history implementation lives in `extensions/slack/src/monitor/thread.ts`, does not model `attachments`, filters out attachment-only replies, and maps only `text` or file placeholders. The PR addresses a real remaining Slack context gap, but it needs to be rebased/adapted to the current `thread.ts` split and should address the latest rendered-range cap review feedback.

## Best Possible Solution

Keep the PR open and ask for a current-main update: move the attachment-aware thread-history logic into `extensions/slack/src/monitor/thread.ts`, reuse or mirror the existing forwarded attachment formatting/capping contract from `resolveSlackAttachmentContent`, add focused tests for forwarded text/fallback, attachment-only forwarded messages, ignored non-share attachments, image-only/no-readable-content drops, and the capped rendered-range edge case, then rerun the Slack extension changed lane.
