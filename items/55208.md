## Summary

Current main still does not implement the requested macOS LaunchDaemon/system-scope Gateway mode. The managed macOS gateway service remains per-user LaunchAgent-only, the CLI has no `--daemon` or service-scope option, the service registry maps Darwin only to the LaunchAgent adapter, the macOS app invokes the same LaunchAgent install path, and docs still describe LaunchDaemons only as extra-service discovery or custom/not-shipped context.

## Best Possible Solution

Keep this issue open for maintainer product/security review. The likely implementation is an explicit macOS system-scope Gateway service mode on the managed lifecycle surface, probably on `openclaw gateway install` with matching start/stop/restart/status/doctor/docs/tests, while preserving the per-user LaunchAgent as the default.
