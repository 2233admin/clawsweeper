## Summary

Keep open. Current main still has the embedded-run abort/wait/active/queue helpers internally, and it also has adjacent Gateway chat.abort wiring, but the requested in-process plugin runtime contract is still missing from api.runtime.agent.

## Best Possible Solution

Keep this issue open and implement the additive plugin runtime contract: re-export the embedded-run abort/wait/isActive/queue helpers from src/plugins/runtime/runtime-embedded-pi.runtime.ts, lazy-bind them in createRuntimeAgent(), declare them on PluginRuntimeCore["agent"], and update the runtime shape test plus docs/API inventory. Treat the shipped Gateway chat.abort path and harness-only aliases as related surfaces, not substitutes for the requested in-process plugin API.
