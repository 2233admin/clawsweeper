## Summary

Keep open. Current `main` does not already implement this PR’s core Feishu affinity changes: sender-scoped `:topic:...:sender:...` session keys are still treated as having no recoverable thread id, run-context still restores thread/channel only from explicit opts, and Feishu outbound still does not carry an explicit `replyToId` through plain-text/media sends with `replyInThread: false`. There is adjacent sender-scoped binding work on `main`, but not enough to treat this PR as already implemented.
