## Summary

Keep #40216 open. Current main has a shared CJK-aware estimator and uses it for most context-pruning message text, but this PR still points at real remaining gaps: soft-trim eligibility is still measured with raw UTF-16 length, and assistant tool-call arguments are still estimated with raw JSON string length. The already-merged #39985 fix is narrower than this PR's stated coverage.

## Best Possible Solution

Keep the PR open or replace it with a rebased maintainer patch scoped to the remaining context-pruner estimation gaps: apply the shared CJK estimator to assistant tool-call arguments, and either make soft-trim eligibility/extraction consistently weighted or document and test that soft-trim thresholds intentionally remain raw character limits.
