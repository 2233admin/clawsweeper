## Summary

Keep this PR open. Current main has the shared CJK estimator and uses it for much of the active context-pruning path, but the PR's stated behavior is still not fully implemented on main: soft-trim eligibility and assistant tool-call argument estimation still use raw UTF-16 length. The PR also edits the removed `pi-extensions` path, so it needs a rebase/replacement against `pi-hooks` rather than direct merge or closure.

## Best Possible Solution

Keep this PR open, or replace it with a rebased maintainer patch against `src/agents/pi-hooks/context-pruning/pruner.ts`. The best fix should keep the shared `src/utils/cjk-chars.ts` estimator, use weighted estimation for assistant tool-call argument JSON, make soft-trim eligibility and head/tail extraction units consistent, and add focused regression tests for CJK-heavy tool results and tool-call arguments. No supply-chain or security-sensitive diff was observed; this is a functional/staleness review issue.
