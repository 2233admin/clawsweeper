## Summary

Current main still has the reported split: the status auth label can honor `sessionEntry.authProfileOverride`, but provider usage loading is only given the usage provider and agent dir, so OpenAI Codex usage auth falls back to provider-level profile order. The related clean fix PR #59208 tracks exactly this bug but is still open/unmerged, so #58498 should remain open as the canonical bug report until that work lands or is replaced.

## Best Possible Solution

Keep #58498 open as the canonical bug report. The best code path is the one described by PR #59208: thread the session-selected OAuth profile into provider usage resolution, validate that the preferred profile belongs to the requested provider, preserve existing provider-level fallback when no valid preferred profile exists, and add regression coverage for multi-profile Codex OAuth status usage.
