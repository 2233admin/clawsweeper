## Summary

Keep open. Current main has mitigations for the older node_modules recursion scanner bug and some short-lived/persisted registry caches, but it does not resolve this issue's central complaint: default plugin discovery and manifest caches still use 1-second TTLs, gateway activation still performs filesystem discovery/manifest registry work before scoped plugin imports, and the repo still has a large bundled plugin set. The issue is recent, non-maintainer-authored, and actionable rather than closeable.

## Best Possible Solution

Keep the issue open and narrow it into a measurable plugin-discovery performance task: profile gateway startup, config reload, and plugin read-only commands on low-core ARM-class hardware; then either lengthen/adapt discovery and manifest cache windows, avoid full filesystem discovery during scoped activation by using the persisted plugin registry where safe, add reload/state-change hysteresis, or introduce an explicit constrained-host performance preset with regression coverage.
