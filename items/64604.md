## Summary

Keep this PR open. Current main still collapses unclassified compaction failures to `reason=unknown`, so the reported diagnostic gap is real. The PR is a useful fix candidate, but it should not be closed or landed as-is because the current head mixes the focused runtime fix with unrelated README localization and still puts arbitrary, unbounded failure text into the `reason=` classification token instead of a separate bounded detail field.

## Best Possible Solution

Keep this PR open and request changes. Split the README localization into a separate docs PR, then land a focused compaction diagnostic change that preserves `reason=unknown` as a stable classifier and adds a separate capped, log-safe diagnostic detail field, with regression tests for unrecognized errors, empty input, whitespace/control characters, and overly long details.
