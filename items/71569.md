## Summary

Keep open. Current main still has the Mattermost streaming docs/config/runtime mismatch reported in #71569: the public docs advertise `channels.mattermost.streaming` modes, but Mattermost's schema, TypeScript config type, generated metadata, and inbound runtime still do not implement the documented preview streaming mode contract. Related #70253 overlaps on ignored Mattermost streaming config, but it is broader and does not supersede this focused issue; #71565 was explicitly superseded by #71569.

## Best Possible Solution

Keep #71569 open as the focused Mattermost streaming tracker. The best fix belongs in the bundled Mattermost plugin: add canonical `streaming` support to Mattermost schema/types/generated metadata and compatibility/doctor handling, use the shared preview-mode resolver in the runtime, skip draft preview creation for `off`, implement `progress` so the final answer is delivered as a normal notifying post or equivalent final notification, preserve/fix in-place `partial` behavior, and add focused schema/runtime tests for top-level and per-account Mattermost streaming settings. Reference #70253 only for the overlapping ignored-config symptom.
