## Summary

Keep #71569 open. Current main partially improves the visual-spam part with a single draft preview and in-place finalization tests, but the core report is still valid: Mattermost docs advertise `channels.mattermost.streaming`, while the plugin schema/type/metadata surface still lacks that key, and the normal inbound path still creates draft streaming unconditionally while forcing block streaming off.

## Best Possible Solution

Keep the issue open. The best fix is to add Mattermost support for the documented unified streaming config in schema, TypeScript types, generated metadata, account resolution, and runtime dispatch. The normal inbound path should resolve `channels.mattermost.streaming`/per-account overrides, skip draft previews for `off`, avoid hard-disabling block streaming when `block` is intended, and define the `progress`/final-notification behavior explicitly. Add focused tests for schema acceptance, config preservation, `off` creating no draft, `partial` using one editable draft, and the chosen final-notification path.
