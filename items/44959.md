## Summary

Keep open. Current `main` does not implement the PR’s core behavior: the Ollama stream path still only promotes native `message.tool_calls` into executable `toolCall` blocks and otherwise returns `stopReason: "stop"`. Main does have newer sanitization that strips leaked Qwen-style tool-call XML from visible assistant text, but that is only a display cleanup and does not execute the embedded call. The PR discussion also shows meaningful follow-up on early review findings, so this is not a good conservative auto-close candidate.
