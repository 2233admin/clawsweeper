## Summary

Keep #70153 open. Current main still has the gap described by the PR: the auto-reply CLI path calls `runCliAgent` directly and returns the result without invoking `persistCliTurnTranscript`, while the direct CLI command path does persist successful CLI turns. Related closed items #58176/#58194 only prove the direct/generic CLI command persistence shipped; they do not cover this auto-reply bypass. The PR also addressed the review comment about `threadId` propagation in head commit `9147e95`.

## Best Possible Solution

Keep #70153 open and review it as a real remaining auto-reply bug fix. The best code path is to add a best-effort `persistCliTurnTranscript` call after successful `runCliAgent` completion in `src/auto-reply/reply/agent-runner-execution.ts`, pass the same transcript fields as `agent-command.ts` including `transcriptCommandBody` and `originatingThreadId`, update the active session entry/store from the returned entry, and keep targeted `runReplyAgent claude-cli routing` regression tests for success, persistence failure suppression, and thread-id propagation.
