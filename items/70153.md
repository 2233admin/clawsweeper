## Summary

Keep #70153 open. Current main still has the auto-reply CLI bypass the PR describes: `runAgentTurnWithFallback` calls `runCliAgent` directly and returns the successful result without calling `persistCliTurnTranscript`. The existing generic CLI persistence in `agent-command.ts` does not cover this auto-reply path, and the related closed generic-CLI items do not supersede this narrower channel/auto-reply fix.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent fix on main. The best implementation is to mirror the direct CLI persistence contract in `src/auto-reply/reply/agent-runner-execution.ts` after successful auto-reply CLI runs, keep transcript writes best-effort, include the transcript body/session metadata, and preserve thread scoping via `followupRun.originatingThreadId` with focused auto-reply regression tests.
