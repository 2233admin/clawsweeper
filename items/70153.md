## Summary

Keep open. Current main still lacks the auto-reply CLI transcript persistence that this PR adds. The direct CLI path persists successful CLI turns, but `runAgentTurnWithFallback` still calls `runCliAgent` directly and the current auto-reply tests do not cover `persistCliTurnTranscript`. The PR also addressed the review comment by forwarding `originatingThreadId`, which is already present on `FollowupRun`.

## Best Possible Solution

Keep the PR open for maintainer review or merge. The best path is to land the auto-reply-side `persistCliTurnTranscript` call after successful CLI results, keep it best-effort so transcript write failures do not fail delivered replies, pass `transcriptCommandBody` and `originatingThreadId`, and retain the targeted regression tests for successful persistence and suppressed persistence failures.
