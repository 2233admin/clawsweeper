## Summary

Current main still leaves the reported Control UI crash path plausible. The UI accepts `cron.list` rows into state without runtime shape validation, `renderApp` still dereferences `job.payload.kind` while computing cron model suggestions before the tab-specific Cron render branch, and Cron edit/render helpers also assume every job has a valid payload. The focused fixes linked in the issue discussion, #54550 and #54552, were closed unmerged.

## Best Possible Solution

Keep this issue open. The best maintainer path is to revive or reimplement the focused #54552 approach on current main: validate or filter malformed `cron.list` rows at the UI and/or server list boundary, guard shared Control UI Cron helpers before reading `payload.kind`, and add a jsdom/browser regression test proving malformed Cron entries do not prevent switching back to Chat.
