## Summary

Keep PR #69084 open. Current main has not implemented the PR's requested webchat heartbeat visibility change, but the submitted one-line `showOk` to `showAlerts` swap is not clearly safe because the documented heartbeat contract treats `showOk` and `showAlerts` as separate controls. The PR still needs a revised fix and tests that suppress alert content when `showAlerts=false` without breaking configured OK-only acknowledgments when `showOk=true`.

## Best Possible Solution

Keep the PR open and ask for a revised implementation in `src/gateway/server-chat.ts` that classifies heartbeat final text using both visibility flags: suppress non-OK alert content when `showAlerts=false`, preserve configured OK acknowledgments when `showOk=true`, and keep default OK suppression when `showOk=false`. Add focused tests for `showOk=false/showAlerts=true`, `showOk=true/showAlerts=false`, and both-false webchat heartbeat outputs before merge.
