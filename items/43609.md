## Summary

Keep this PR open. Current main still lacks the central Grok web_search fixes: the xAI Responses parser is not null-safe, Grok still inherits the shared 30 second timeout, and AbortError from the xAI web-search request path can still propagate as a tool/session abort. The PR should not be landed as-is because the supplied PR file set also includes unrelated status-command changes with review-noted regressions.

## Best Possible Solution

Keep this PR open as useful implementation work, but ask for a rebase and a narrower patch. The best path is to drop or split the unrelated status-command changes, port the Grok work to current main by adding null/object guards in `extensions/xai/src/responses-tool-shared.ts`, wrapping xAI web-search timeout AbortErrors into plain `Error`s with `{ cause }` using the repo’s abort detection helper, making Grok default to 60 seconds while preserving explicit user timeout overrides, and adding focused xAI regression tests.
