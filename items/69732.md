## Summary

Keep PR #69732 open for maintainer review rather than auto-closing. The linked bug #69698 is still plausible on current main: transport snapshots can carry durable allowlist `source` metadata, the Control UI round-trips the snapshot file, and the gateway `exec.approvals.set` schema still rejects unknown allowlist fields. The PR’s later commit preserved `source` to avoid durable-approval data loss, so it no longer clearly fixes the reported INVALID_REQUEST by itself, but the underlying product issue remains actionable.

## Best Possible Solution

Keep the item open and ask for a focused revision or replacement that makes exec approval snapshots safely round-trip: either extend the set schema/contract to accept durable `source` metadata, or strip UI/transport-only fields while rehydrating `source` from the current snapshot by stable id or `(pattern,argPattern)` before save. Add a regression test for GET -> edit security mode -> SET with an allow-always entry and a socket token, covering both gateway and node-host paths if practical.
