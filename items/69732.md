## Summary

Keep #69732 open. Current main still has a real GET-to-SET contract mismatch for exec approvals: gateway/node snapshot transport only strips socket tokens, durable allowlist `source` metadata can still be present in snapshots, and the public `exec.approvals.set` schema rejects unknown allowlist properties. The PR is focused on the right area, but its latest discussion says it preserves `source`; without a schema change or set-side metadata rehydration, that does not fully resolve linked issue #69698.

## Best Possible Solution

Keep the PR open for maintainer/code review, but require one explicit contract direction before merge: either allow `source?: "allow-always"` in the exec approvals snapshot/set schema, or redact `source` from transport and merge it back on SET by stable allowlist identity such as `id` or `(pattern,argPattern)`. In either case, centralize socket-token redaction and add a gateway/node/UI or protocol regression proving GET -> edit -> SET succeeds while durable allow-always metadata survives.
