## Summary

Keep open. Current `main` still documents direct `claude --permission-mode bypassPermissions --print ...` dispatch in the bundled `coding-agent` skill, and the host exec path still inherits `process.env` through the generic sanitizer without any code or policy entry for `CLAUDECODE` or `CLAUDE_CODE_SESSION`. The Claude-specific clear-env list clears `CLAUDE_CODE_ENTRYPOINT` but not the other nesting-guard vars from the report.
