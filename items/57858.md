## Summary

Keep #57858 open. Current main still has the bundled `coding-agent` skill teaching direct Claude Code background dispatch without unsetting the reported Claude nesting variables, and the nearby managed `claude-cli` backend env-clearing path does not cover this skill path. The related ClawHub cleanup around a separate delegation bundle does not supersede this narrower bug against an already-bundled core skill.

## Best Possible Solution

Keep the issue open and update the bundled `skills/coding-agent/SKILL.md` Claude Code examples to clear `CLAUDECODE`, `CLAUDE_CODE_SESSION`, and `CLAUDE_CODE_ENTRYPOINT` before invoking `claude`; consider adding a small skill-text regression check or a targeted exec env policy change only if maintainers want this handled automatically for all direct Claude launches.
