## Summary

Keep open. Current `main` does not implement this PR's gateway-startup/global fetch proxy bootstrap or its `ALL_PROXY` bridging. The existing code only enables env-proxy dispatch in narrower call paths, and the current proxy-env contract/tests still explicitly treat `ALL_PROXY` as ignored for `EnvHttpProxyAgent`-style resolution.

## Evidence

- **Current main still ignores ALL_PROXY for env-proxy bootstrap:** `resolveEnvHttpProxyUrl()` documents and implements `ALL_PROXY` as ignored for `EnvHttpProxyAgent` semantics, and `hasEnvHttpProxyConfigured()` depends on that helper.
  - file: [src/infra/net/proxy-env.ts:28](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/infra/net/proxy-env.ts#L28)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Tests on main still assert ALL_PROXY is not used:** `proxy-env.test.ts` includes a case named `does not use ALL_PROXY for EnvHttpProxyAgent-style resolution`, expecting `expectedConfigured: false` when only `ALL_PROXY`/`all_proxy` are set.
  - file: [src/infra/net/proxy-env.test.ts:73](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/infra/net/proxy-env.test.ts#L73)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)

  - file: [src/agents/pi-embedded-runner/run/attempt-http-runtime.ts:7](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/pi-embedded-runner/run/attempt-http-runtime.ts#L7)

A file existence check also shows `src/infra/net/global-proxy.ts` is absent on `main`.
