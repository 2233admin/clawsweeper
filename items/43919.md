## Summary

Keep this PR open. Current main implements the HTTP_PROXY/HTTPS_PROXY gateway and embedded-runner EnvHttpProxyAgent bootstrap, but it still intentionally excludes ALL_PROXY/all_proxy and SOCKS fallback behavior, which is the distinct remaining capability this PR attempts to add. The PR needs rebase and security cleanup before merge, especially around global dispatcher scope and proxy credential redaction, but it is not obsolete on current main.

## Best Possible Solution

Keep this PR open for maintainer review, but require a rebase and narrowing around the remaining ALL_PROXY/SOCKS gap. The best implementation would extend the centralized proxy-env and undici-global-dispatcher helpers with explicit tests for lowercase precedence, malformed or authenticated proxy URL redaction, NO_PROXY and loopback behavior, dispatcher replacement, and SSRF-preserving strict paths. If maintainers decide OpenClaw should intentionally keep Undici's HTTP_PROXY/HTTPS_PROXY-only semantics, close this only with that explicit product/security rationale and leave #43821 or a narrower follow-up to track the rejected ALL_PROXY/SOCKS scope.
