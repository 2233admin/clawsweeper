## Summary

Keep #40464 open. Current main has shipped important guardrails for the catastrophic #40410-style config truncation symptom, but it still does not implement the PR's core concurrency fix: serializing config writes or rereading the latest on-disk config inside an ownerDisplaySecret write critical section.

## Best Possible Solution

Keep the PR open for maintainer review or rewrite. The best path is a narrow current-main fix that makes ownerDisplaySecret auto-persist read the latest source config inside a documented serialized write transaction, while preserving destructive-write rejection, last-known-good recovery, env-ref restoration, runtime snapshot refresh, and plugin index commit behavior. Add regression coverage that starts ownerDisplaySecret persistence while another config write is pending and proves unrelated gateway/auth fields are not lost.
