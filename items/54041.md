## Summary

Keep open. Current main still does not implement the requested Anthropic context-management beta support. The bundled Anthropic provider has raw `anthropicBeta`/`context1m` header handling and service-tier/cache behavior, but it still lacks first-class Anthropic `context_management` payload shaping, compaction/context-editing stream replay, `pause_after_compaction` behavior, and a `memory_20250818` provider-defined tool path. The related open issues #10213 and #48558 cover narrower compaction and memory-tool slices, but they do not supersede this broader passthrough/design request.

## Best Possible Solution

Keep this issue open for maintainer/product design in the bundled Anthropic provider. A complete fix should add documented Anthropic model params for beta/header handling, validated `context_management` payload shaping for compaction and context editing, stream and replay support for compaction blocks and `pause_after_compaction`, coverage for both direct Anthropic transport and provider-wrapper paths, and a clear provider-owned design for `memory_20250818` or a generic provider-defined-tool SDK seam if that is required.
