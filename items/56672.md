## Summary

Keep open. Current `main` does not implement this PR’s behavior: failed subagent spawn still suppresses generic lifecycle emission after `subagent_ended`, and `src/cron/isolated-agent/run.ts` still has no `session_end` hook emission path.

## Evidence

- **Failed subagent spawn still skips generic lifecycle hooks after `subagent_ended`:** On spawn failure, `endedHookEmitted` flips `emitLifecycleHooks` to `false`, then `sessions.delete` is called with that value. That means the existing delete-time `session_end` path is suppressed when `subagent_ended` already ran; there is no direct `runSessionEnd(...)` call in this block.
  - file: [src/agents/subagent-spawn.ts:1010](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/subagent-spawn.ts#L1010)
  - command: `nl -ba src/agents/subagent-spawn.ts | sed -n '1008,1053p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Delete path would emit `session_end` only when lifecycle hooks remain enabled:** `sessions.delete` emits `emitGatewaySessionEndPluginHook(...)` and then unbound lifecycle hooks, but the caller can disable the latter with `emitLifecycleHooks: false`. The failed-spawn path above uses that suppression when `subagent_ended` succeeds.
