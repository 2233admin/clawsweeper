## Summary

Keep open. Current main does not implement the requested configurable output-level regex/pattern filter, and the existing cleanup only covers explicit reasoning payloads, reasoning/thinking tags, special tokens, and a few hard-coded scaffolding formats. The available message_sending plugin hook can modify or cancel outbound text, but that does not make this per-agent/core-config feature request obsolete, especially with related leak reports still being tracked separately.
