## Summary

Keep open. Current `main` has partial mitigations for large skill catalogs (per-agent skill allowlists, prompt compaction, prompt-size caps), but it does not implement the requested global whitelist-style `skills.mode`/`skills.entries` loading behavior, and it still scans and merges all configured skill roots before applying the agent filter.

## Evidence

- **No global skills whitelist mode in config schema:** `SkillsConfig` exposes `allowBundled`, `load`, `install`, `limits`, and `entries`, but there is no `mode` field or equivalent top-level whitelist toggle for all skills.
  - file: [src/config/types.skills.ts:40](https://github.com/openclaw/openclaw/blob/e8191e5b8f65f0e0544182e5ce193e096aa3354c/src/config/types.skills.ts#L40)
  - command: `nl -ba src/config/types.skills.ts | sed -n '1,160p'`

- **Loader still scans all skill roots before filtering:** `loadSkillEntries` loads bundled, extra, managed, personal `.agents`, project `.agents`, and workspace skill roots, then merges them. This is the opposite of the issue's proposed 'only load declared entries' behavior.
  - file: [src/agents/skills/workspace.ts:533](https://github.com/openclaw/openclaw/blob/e8191e5b8f65f0e0544182e5ce193e096aa3354c/src/agents/skills/workspace.ts#L533)
