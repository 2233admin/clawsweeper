## Summary

Keep open. Current `main` still strips `agent:<id>:` subagent session keys down to `subagent:<uuid>`, then returns `undefined` for QMD scope normalization, so default-deny QMD scope rules do not match subagent sessions and the memory engine falls back to builtin search.

## Evidence

- **Subagent session keys are canonical `agent:<agentId>:subagent:<uuid>` keys:** The public subagents docs define spawned subagent sessions with the exact `agent:<agentId>:subagent:<uuid>` shape described in the issue.
  - file: [docs/tools/subagents.md:243](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/docs/tools/subagents.md#L243)
  - command: `nl -ba docs/tools/subagents.md | sed -n '236,246p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Agent session parsing drops the `agent:<id>:` prefix:** `parseAgentSessionKey()` returns `rest = parts.slice(2).join(":")`, so `agent:marketing-coo:subagent:<uuid>` is reduced to `subagent:<uuid>` before QMD scope parsing.
  - file: [src/sessions/session-key-utils.ts:43](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/sessions/session-key-utils.ts#L43)
  - command: `nl -ba src/sessions/session-key-utils.ts | sed -n '28,48p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **QMD scope normalization still discards subagent keys:** `normalizeQmdSessionKey()` normalizes `parsed?.rest ?? trimmed` and returns `undefined` when the normalized key starts with `subagent:`. `parseQmdSessionScope()` then returns `{}`, leaving `channel` and `chatType` unset for subagent sessions.
  - file: [src/memory-host-sdk/host/qmd-scope.ts:104](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/memory-host-sdk/host/qmd-scope.ts#L104)
  - command: `nl -ba src/memory-host-sdk/host/qmd-scope.ts | sed -n '65,110p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Default QMD scope is still default-deny with chat-type allow rules:** The runtime default QMD scope is `default: "deny"` with allow rules only for `chatType: "direct"` and `chatType: "channel"`. A subagent key that parses to no `chatType` still falls through to deny.
  - file: [src/memory-host-sdk/host/backend-config.ts:104](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/memory-host-sdk/host/backend-config.ts#L104)

- **Denied QMD searches log `unknown` scope data and the memory layer falls back:** When scope denies a search, `qmd-manager` logs `channel`/`chatType` as `unknown` if derivation failed, and `search-manager` logs `qmd memory failed; switching to builtin index` on primary failure.
