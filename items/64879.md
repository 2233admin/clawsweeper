## Summary

Keep this issue open. Current main still records provider/model metadata in the qa-lab mock, but the scenario dispatcher remains shared and prompt-driven; it does not branch on provider/model to produce divergent OpenAI vs Anthropic plans. The CI parity gates are still mock-only, while live-frontier workflows cover transport/live lanes rather than a real OpenAI-vs-Anthropic parity-report comparison.

## Best Possible Solution

Keep this issue open. Implement Option A in `extensions/qa-lab/src/providers/mock-openai/server.ts` by deriving the provider variant before scenario dispatch and adding provider-specific branches for at least three agentic parity scenarios, with tests asserting divergent OpenAI vs Anthropic plans. If maintainers prefer Option B, add a separate secret-gated live-frontier workflow that runs the same parity pack against real OpenAI and Anthropic models and feeds both summaries into `openclaw qa parity-report` with uploaded artifacts.
