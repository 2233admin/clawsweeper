## Summary

Keep PR #72023 open. Current main still reconciles every persisted ACP session with a pending stable identity during startup and then logs a warning whenever any are checked, with no `mode === "oneshot"` skip. The newer close-time one-shot identity reconciliation helps future successful runs, but it does not cover historical or failed one-shot records like #72013 describes. The PR’s bot-reviewed `closed`-state dead-code issue was addressed in the later PR head by keeping only the one-shot guard, so the remaining change is still plausibly useful rather than obsolete.

## Best Possible Solution

Keep #72023 open and land either this PR or an equivalent maintainer patch that skips `acp.mode === "oneshot"` before pending-identity startup reconciliation, with the focused regression test. Do not close it as implemented on main unless current main gains that explicit startup skip or another change demonstrably suppresses historical/error one-shot pending records from the warning path.
