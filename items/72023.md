## Summary

Keep PR #72023 open. Current main still runs ACP startup identity reconciliation for any persisted pending identity with a stable runtime id, without skipping `mode: "oneshot"`, so the linked #72013 warning path is still present. The PR's useful one-shot guard is not obsolete on main.

## Best Possible Solution

Keep #72023 open and land it or an equivalent maintainer patch that skips `session.acp.mode === "oneshot"` before startup pending-identity reconciliation, with the focused regression test. Do not close it as implemented unless main gains that explicit startup skip or another source-level change demonstrably suppresses historical/error one-shot pending records from the warning path.
