## Summary

Keep #72023 open. Current main still reconciles every ACP session with a pending identity and stable runtime id, regardless of `acp.mode`, and the gateway logs the startup warning whenever that scan checks at least one session. The PR remains a focused, useful fix for #72013: its latest diff skips `mode === "oneshot"` before the pending-identity path and adds a regression test. The earlier bot concern about an unreachable `state === "closed"` guard was addressed in the PR discussion and is not present in the latest provided patch.

## Best Possible Solution

Keep the PR open and review it as the active fix for #72013. The best path is to merge the narrowed one-shot-only guard plus regression test, with any PR-body cleanup needed to remove stale references to the dropped `closed` guard. If maintainers want broader terminal-session semantics later, that should be a separate typed lifecycle change rather than blocking this focused noise fix.
