## Summary

Keep open. Current `main` still handles context-overflow/compaction-failure by rotating to a fresh session and returning a generic hard-reset message. The reset path clears runtime session fields and reuses the generic `/new`/`/reset` startup prompt rather than injecting the interrupted task goal or progress, so the reported no-recovery-path behavior is still materially present. Existing mitigations are pre-compaction memory flush and successful-compaction context refresh, not automatic hard-reset recovery.
