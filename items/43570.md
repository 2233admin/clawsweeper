## Summary

Keep #43570 open. Current main still matches the reported Discord bug: `thread-reply` only reads `media`, while Discord `send` accepts `media`, `path`, and `filePath`. The shared message action runner normalizes/validates media parameters but does not remap `path` or `filePath` to `media` for plugin actions, and the downstream Discord `threadReply` runtime only consumes `mediaUrl`. The linked PR #43574 appears directly relevant, but it is still open/unmerged, so the issue should remain until a fix lands.
