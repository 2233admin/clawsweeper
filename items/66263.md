## Summary

Current main still does not satisfy the native Codex compaction accounting request. The Codex native compaction path reports success without a post-compaction token snapshot, and session/status transcript fallbacks can still promote pre-compaction transcript usage into a fresh-looking current context total after the store total is cleared. The related merged #64669 work remains a narrower cumulative-usage projection fix, not a native compaction-boundary fix.

## Best Possible Solution

Keep this issue open and implement the native Codex compaction accounting boundary end to end. The fix should propagate a reliable post-compaction token snapshot when the app-server exposes one; otherwise it should persist an explicit unknown/stale state, add or reuse a compaction boundary that transcript and status fallback honor, and add regression coverage for known tokensAfter, missing tokensAfter, and post-compaction session/status fallback. Treat #64669 as the already-merged cumulative-usage guardrail, not the canonical fix for this issue.
