## Summary

Keep open. Current `main` still leaves the Codex native-compaction path able to preserve or rehydrate stale context usage when `tokensAfter` is absent, so the issue is not implemented well enough to close conservatively.

## Evidence

- **Native Codex compaction still returns no post-compaction token snapshot:** `compactCodexNativeThread` completes successfully but returns only `tokensBefore` and compaction metadata; it does not set `tokensAfter` on the native Codex path.
  - file: [extensions/codex/src/app-server/compact.ts:156](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/extensions/codex/src/app-server/compact.ts#L156)
  - command: `nl -ba extensions/codex/src/app-server/compact.ts | sed -n '156,174p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Auto-reply compaction summary still falls back to pre-compaction cached totals:** After compaction, `/compact` uses `result.result?.tokensAfter` if present, otherwise falls back to `resolveFreshSessionTotalTokens(targetSessionEntry)`, which can surface the old pre-compaction count in the user-visible context summary.
  - file: [src/auto-reply/reply/commands-compact.ts:181](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/auto-reply/reply/commands-compact.ts#L181)

- **Session compaction state is not marked unknown when `tokensAfter` is missing:** `incrementCompactionCount` only updates token state when `tokensAfter` is provided. Otherwise it leaves prior `totalTokens` / freshness state intact. The existing test explicitly expects `totalTokens` to remain unchanged when `tokensAfter` is absent.
  - file: [src/auto-reply/reply/session-updates.ts:257](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/auto-reply/reply/session-updates.ts#L257)

- **Regression is codified by test coverage:** `reply-state.test.ts` asserts that compaction without `tokensAfter` keeps `totalTokens` at the old value (`180_000`) instead of clearing or marking it unknown.
  - file: [src/auto-reply/reply/reply-state.test.ts:559](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/auto-reply/reply/reply-state.test.ts#L559)
