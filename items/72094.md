## Summary

Keep #72094 open. Current main has a native Codex app-server harness, `/codex` chat binding, ACP fallback, and PeekabooBridge, but it does not expose the requested Codex Desktop Computer Use route: no `codex-desktop` desktop-control provider, no `computerUse`/`desktopControl` config, no feature probe wired into `/codex status`, and no channel-safe desktop-control routing contract. The related context also says #67776 was closed for PeekabooBridge while explicitly leaving this narrower provider-native Codex request as separate work.

## Best Possible Solution

Keep #72094 open for maintainer/product design. The best path is to define an explicit desktop-control capability or provider contract with config/schema/docs, availability probing for Codex app-server/Codex Desktop features, channel allowlist and approval policy, and a routing surface from chat-bound Codex or `sessions_spawn`; then implement `codex-desktop` as one provider and keep PeekabooBridge as a fallback where appropriate.
