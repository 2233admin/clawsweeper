## Summary

Current `main` and the latest shipped release still intentionally use round-robin/LRU auth-profile ordering (`lastUsed` oldest first) and explicitly avoid prioritizing the last successful profile, so the MRU-sticky part of #55897 is not implemented. The 402/failover side appears partially addressed on `main` via broader 402 classification and retry-on-rotation behavior, but that is not enough to conservatively close the issue.
