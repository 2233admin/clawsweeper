## Summary

Keep this issue open. Current main still uses `abortedLastRun` as both an explicit user-abort marker and a restart-recovery marker, and both main-session and subagent recovery can still synthesize continuation messages through `callGateway` from that marker.

## Best Possible Solution

Split explicit user-abort state from restart-interruption recovery state. `/stop` and recognized no-action phrases should persist a terminal user-abort or recovery-suppression marker, recovery scans should skip those sessions and clear pending retry state, and focused regression tests should prove `/stop` followed by gateway startup does not call `callGateway` while legitimate gateway-reload recovery still resumes interrupted work.
