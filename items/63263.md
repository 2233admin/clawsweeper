## Summary

Keep this PR open. Current main still lacks several central infer CLI reliability fixes proposed here: local `model run` calls do not pass a per-run infer `sessionKey`, `image describe`/`describe-many` still do not register or thread `--prompt`, structured web-search `{ error: string }` payloads are returned as successful provider results instead of falling through, and `openai-codex` still has no image `autoPriority`. The PR is a plausible implementation candidate, but the existing review concerns remain material before landing.

## Best Possible Solution

Keep this PR open as an implementation candidate. The maintainer path is to revise it so local infer model runs get a per-run session key, image `--prompt` is registered and threaded through an image request override that beats per-entry prompts, web-search fallback only continues for provider availability/auth failures, `agentDir` behavior stays aligned with current main, and `openai-codex` image `autoPriority` is added only if that provider should participate in auto image fallback. Then rerun the targeted CLI, web-search, and media-understanding checks before review/landing.
