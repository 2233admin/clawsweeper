## Summary

Keep #40540 open. Current main still runs the npm global self-update from the active OpenClaw process for package installs, and the gateway/dashboard update path only schedules restart after a successful update. There is no merged detached Windows updater, no pre-update gateway/process handoff, and no EBUSY rename recovery coverage on main. The related #40724 was closed unmerged and scoped only to hints; #44614 is an open, unmerged PR for the likely detached-helper fix.

## Best Possible Solution

Keep #40540 open until OpenClaw ships a Windows-safe package self-update path. The likely maintainer path is to land or rework #44614, or an equivalent design, so both `openclaw update` and dashboard `update.run` stop or exit the locking process before npm renames the global package root, or delegate to a detached/temp helper. Add regression coverage for `EBUSY`/`rename` output and document the recovery path if a full automatic fix is not shipped immediately.
