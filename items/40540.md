## Summary

Current main still attempts Windows npm global self-updates from the active OpenClaw process. The known EBUSY/rename path is not fixed; #44614 remains the relevant open implementation candidate rather than a shipped fix.

## Best Possible Solution

Keep this issue open as the canonical Windows npm global self-update bug. Review and revise #44614 or land an equivalent Windows-safe design that releases the active CLI/gateway before npm renames the global package root, with regression coverage for EBUSY/rename output, gateway update.run behavior, and a Windows smoke/manual verification path that does not pre-stop the gateway.
