## Summary

## Close Comment

Closing as implemented on main after a Codex review. Current main suppresses duplicate-plugin warnings when the "duplicate" candidates resolve to the same physical plugin directory and simply keeps the higher-precedence origin instead (`src/plugins/manifest-registry.ts:627`). There is also regression coverage for the exact bundled-vs-config same-directory case, which expects zero duplicate warnings and keeps the config-selected plugin (`src/plugins/manifest-registry.test.ts:1112`). The documented supported override path is `plugins.entries.<id>` with config-selected precedence over bundled plugins (`docs/plugins/manifest.md:654`). I also checked the shipped `v2026.4.22` tree and it already contains the same suppression logic, so this report no longer looks actionable on supported main.
