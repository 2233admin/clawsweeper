## Summary

Keep open. Current `main` still does not thread post-overflow-compaction `tokensAfter` into the agent session store, so the PR’s core fix is not implemented on `main`. There is partial mitigation for `#67667` and `/status` transcript fallback, but neither consumes compaction-checkpoint `tokensAfter`, and the PR also bundles unrelated fixes plus an unresolved review comment on an overflow error path.
