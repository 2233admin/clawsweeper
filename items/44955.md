## Summary

Keep open. Current `main` does not implement the PR, and the proposed change is not ready for conservative cleanup: provider-level `contextWindow` is not part of the current public provider config contract, the docs only describe per-model overrides, and the discussion already identified an uncovered `inlineMatch?.api` path that later got cross-referenced by open issue #64914.

## Evidence

That means the PR is changing runtime semantics for a field that is not in the current typed config surface.
