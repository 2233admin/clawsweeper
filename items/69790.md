## Summary

Keep open. Current `main` still uses intentionally non-reentrant session transcript locks, but same-process Telegram inbound is also serialized and default-queued per session. That makes the supported-path behavior in this report ambiguous, while the concrete synthetic reproduction relies on an internal Telegram runtime path that bypasses normal gateway ingestion. I did not find a shipped fix or enough evidence for a conservative auto-close.

## Evidence
