## Summary

Keep this PR open. Current main still omits the model-derived max-token fallback in both OpenAI-compatible request builders, so the transport bug the PR targets remains real. The PR should not land unchanged because it bundles unrelated TUI changes with review-reported regressions and lacks focused fallback regression tests.

## Best Possible Solution

Keep this PR open, but ask the contributor to rebase and narrow it to the transport fix. The best patch should resolve an effective positive max-token value from options.maxTokens first and model.maxTokens second, preserve the existing max_output_tokens / max_tokens / max_completion_tokens compatibility logic, and add focused regression tests for omitted options.maxTokens in both request builders. The unrelated TUI refactor should be dropped or handled separately after its review comments are resolved.
