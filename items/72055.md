## Summary

Keep PR #72055 open. Current main still does not implement a configurable Bonjour/mDNS service instance name: the bundled Bonjour plugin always derives `instanceName` from `ctx.machineDisplayName`, the plugin manifest exposes no config properties, and the docs only describe disabling/hostname-related controls. The PR is a focused plugin-owned implementation path for the concrete #54467 multi-gateway same-host problem, and current core/plugin plumbing already supports validated `api.pluginConfig` without a new core gateway key.

## Best Possible Solution

Keep PR #72055 open as the active implementation path. The best product/code direction is to land a Bonjour-plugin-owned `plugins.entries.bonjour.config.instanceName` schema/runtime/docs/test change, using the existing `api.pluginConfig` and advertiser `instanceName` paths, rather than adding a core `gateway.bonjour.*` key or closing it as already implemented.
