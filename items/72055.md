## Summary

Keep PR #72055 open. Current main still does not expose a Bonjour plugin config key for per-instance mDNS service names: the bundled plugin always derives `instanceName` from `ctx.machineDisplayName`, the manifest rejects unknown Bonjour config properties, and the docs do not document the requested key. The PR is a focused implementation for the still-missing behavior, though it likely needs a rebase over the newer Bonjour crash-suppression changes on main.

## Best Possible Solution

Keep #72055 open as the active implementation path for #54467's remaining Bonjour per-instance service-name config. Ask the author to rebase onto current main, preserve the newer ciao crash-suppression handler wiring from `bd95baa4`, keep the config under `plugins.entries.bonjour.config.instanceName`, and rerun the Bonjour extension tests plus the changed gate before merge.
