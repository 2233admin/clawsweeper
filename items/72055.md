## Summary

Keep PR #72055 open. Current main does not implement the requested Bonjour per-instance service-name config: the bundled Bonjour entry still derives `instanceName` only from `ctx.machineDisplayName`, the plugin manifest exposes no config properties, and the public Bonjour docs list only the existing disable/env knobs. The advertiser and plugin API already have the needed seam, so the PR is a plausible focused implementation rather than obsolete, ClawHub-only, or outside the repo.

## Best Possible Solution

Keep the PR open for maintainer review or merge a scoped equivalent: add `instanceName` to the Bonjour plugin manifest schema, resolve it from `api.pluginConfig` inside `extensions/bonjour/index.ts`, preserve current fallback/suffix behavior, add plugin-entry tests for unset/empty/configured/suffixed/non-string cases, and document the public config key. If maintainers prefer a different public key name, adjust the PR before landing rather than closing it as already implemented.
