## Summary

Keep this issue open. Current main still treats Mattermost `oncall` channel/group traffic as wakeable only via explicit mention, authorized command bypass, or `onchar`; it records `root_id` for thread/session routing but does not verify that the referenced post was authored by the bot or convert replies to bot posts into an implicit mention. The closed related #45284 covers outbound Mattermost auto-threading, not this inbound reply-wake behavior.

## Best Possible Solution

Implement this in the bundled Mattermost plugin. The likely path is to identify the relevant reply/root post, fetch or cache it when needed, verify its `user_id` equals the Mattermost bot user id, feed `reply_to_bot` into the shared mention-gating model or equivalent Mattermost gate, and add regression tests for unmentioned threaded and inline replies in default `oncall` mode.
