## Summary

Current main still leaves the hooks CLI reporting and enable/disable paths on the full plugin diagnostics loader, so the Windows hang reported for hooks commands remains plausible. The broader #64306 startup mitigation does not cover this command-body path.

## Best Possible Solution

Keep this issue open and fix the hooks CLI path directly. openclaw hooks list/info/check/enable/disable should use manifest, config, workspace snapshot data, or a bounded hook-specific metadata API instead of buildPluginDiagnosticsReport/loadOpenClawPlugins. Add regression coverage proving these commands avoid the loadModules=true diagnostics path while still showing plugin-managed hooks as documented.
