## Summary

Current `main` shows a partial CLI-startup mitigation, not a high-confidence fix for this report. `openclaw --help/--version` now bypass preaction bootstrap and `status` is explicitly marked `loadPlugins: "never"`, but the `hooks` commands still build a full plugin diagnostics report, which calls into `loadOpenClawPlugins()` during command execution. I did not find Windows-specific evidence that the reported 60-second timeout/SIGKILL path is resolved for `openclaw hooks list` / `openclaw hooks enable ...`, so this should stay open.
