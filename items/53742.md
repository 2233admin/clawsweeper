## Summary

Keep this issue open. Current main has addressed the `gateway.auth.token` SecretRef drift/persistence symptom, but the broader macOS LaunchAgent install-path privacy problem remains: durable `.env`, config service env, preserved service env, and env-backed auth-profile values can still be assembled into the install environment and rendered inline under LaunchAgent `EnvironmentVariables`.

## Best Possible Solution

Keep this issue open and finish the macOS install-path fix: avoid serializing secret-bearing durable `.env`, config service env, auth-profile, provider, and channel values into LaunchAgent plist `EnvironmentVariables`; load those values through a launchd-safe runtime env/SecretRef entrypoint or equivalent; scrub stale inline plist entries; and add regression coverage for forced install, update/reinstall, ENV/FILE/structured SecretRefs, auth profiles, provider/channel secrets, and gateway auth token behavior.
