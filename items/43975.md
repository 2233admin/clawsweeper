## Summary

Keep this PR open. Current main still lacks the Windows `gateway.cmd` UTF-8 bootstrap and still lacks parser support to skip that bootstrap line, so the PR remains a focused implementation candidate for the concrete Chinese-username Windows gateway report it references.

## Best Possible Solution

Review and either land this PR after rebasing/validation or replace it with an equivalent main-branch fix. The core fix should remain in `src/daemon/schtasks.ts`: emit a fixed UTF-8 code-page bootstrap in generated Windows `gateway.cmd`, teach `readScheduledTaskCommand` to ignore that exact bootstrap line during read-back, and keep the Chinese-path regression coverage. Do not auto-close this PR unless an equivalent implementation lands on main or a maintainer chooses a different Windows encoding strategy.
