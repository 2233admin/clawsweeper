## Summary

Keep open. Current `main` still writes Windows gateway task scripts without a `chcp 65001>nul` bootstrap and still does not skip that bootstrap when re-parsing task scripts, so the fix described in PR #43975 is not implemented on `main` and the PR remains materially relevant.

## Evidence

- **Task script still starts without UTF-8 bootstrap:** `buildTaskScript()` initializes the generated Windows task script with only `"@echo off"`, and `writeScheduledTaskScript()` writes that script as UTF-8. There is no `chcp 65001>nul` prelude on current `main`.
  - file: [src/daemon/schtasks.ts:242](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/daemon/schtasks.ts#L242)
  - command: `nl -ba src/daemon/schtasks.ts | sed -n '240,320p'; nl -ba src/daemon/schtasks.ts | sed -n '590,620p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Parser still would treat a bootstrap line as the command:** `readScheduledTaskCommand()` skips `@echo`, `rem`, `set`, and `cd /d`, then takes the first remaining line as the command. There is no special-case for `chcp 65001>nul`, so the parser half of the PR is also absent on current `main`.
  - file: [src/daemon/schtasks.ts:111](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/daemon/schtasks.ts#L111)
  - command: `nl -ba src/daemon/schtasks.ts | sed -n '100,220p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Current tests do not cover the proposed UTF-8 regression:** The existing scheduled-task tests cover quoted args, backslash paths, UNC paths, env parsing, and PATH filtering, but there is no test asserting a UTF-8 bootstrap line or a Chinese/non-ASCII round-trip in the current tree.
  - file: [src/daemon/schtasks.test.ts:186](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/daemon/schtasks.test.ts#L186)
  - command: `nl -ba src/daemon/schtasks.test.ts | sed -n '170,340p'; nl -ba src/daemon/schtasks.install.test.ts | sed -n '65,245p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **PR discussion matches an unimplemented gap:** The PR body and linked closure of issue #43943 describe adding `chcp 65001>nul`, teaching scheduled-task parsing to ignore that line, and adding Chinese-path regression coverage. Those exact changes are not present in the checked-out `main`, so this is not an `implemented_on_main` cleanup candidate.
