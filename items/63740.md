## Summary

Keep open. Current main has adjacent stale-dist inventory and pruning work, but it still does not prove, detect, fail fast on, or repair byte-level mutation of an expected installed `dist/run-main-*.js` chunk like the reporter's corrupted file.

## Best Possible Solution

Keep this issue open as a packaging/runtime integrity investigation. The maintainer path is to reproduce or rule out the 2026.4.5 WSL/global-install corruption, audit update/postinstall/restart/logging flows for writes under the installed package root, and either prove OpenClaw cannot mutate installed core `dist` chunks at runtime or add content-hash inventory/startup or doctor verification with clear reinstall/repair guidance for corrupted expected chunks.
