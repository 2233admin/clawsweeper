## Summary

Keep open. Current `main` only partially covers this PR: Signal inbound handling preserves quoted text/sender in `MsgContext`, but it still does not populate `ReplyToId`, does not accept the newer Signal quote schema (`quote.id` / structured `quote.author`), and does not forward reply metadata through hook mappers. That means the PR's main hook-propagation behavior is still missing on `main`.
