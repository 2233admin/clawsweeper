## Summary

Current `main` already contains generic delivery-mirror dedupe in both the session-transcript append path and the embedded outbound reply path, and the shipped changelog says this landed in `v2026.4.23`.

## Close Comment

This looks fixed on current `main`, so I'm closing it.

- Codex review found transcript dedupe for `openclaw`/`delivery-mirror` in `src/config/sessions/transcript.ts:117` and `src/config/sessions/transcript.ts:286`, which reuses the latest identical assistant message instead of appending a second mirror entry.
- Regression coverage in `src/config/sessions/transcript.test.ts:149` verifies that a matching latest assistant reply is not duplicated, while `src/config/sessions/transcript.test.ts:179` keeps repeated answers on later turns working.
- The embedded outbound reply path now ignores transcript-only `delivery-mirror` / `gateway-injected` assistant messages in `src/agents/pi-embedded-subscribe.handlers.messages.ts:44` and `src/agents/pi-embedded-subscribe.handlers.messages.ts:613`, preventing a second visible channel send.
- Regression coverage in `src/agents/pi-embedded-subscribe.subscribe-embedded-pi-session.suppresses-message-end-block-replies-message-tool.test.ts:96` confirms those transcript-only assistant messages no longer emit block replies.
- This was shipped in `v2026.4.23`; see `CHANGELOG.md:606`.
