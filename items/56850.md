## Summary

Keep open. Current `main` does not implement deadline-aware or condition-driven cron intervals: the cron contract still only supports `at`, `every`, and `cron`, and the scheduler computes next runs strictly from those fixed schedule shapes.

## Evidence

- **Cron schedule type is still fixed-shape:** `CronSchedule` only defines three variants: one-shot `at`, fixed-interval `every`, and cron-expression `cron`. There is no conditional schedule field or state-aware interval shape in the persisted job contract.
  - file: [src/cron/types.ts:6](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/types.ts#L6)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Next-run computation has no condition/state branch:** `computeNextRunAtMs` switches only on `schedule.kind` and computes from `at`, `everyMs`/`anchorMs`, or cron expression timing. No state predicate or alternate interval path exists in scheduler logic.
  - file: [src/cron/schedule.ts:65](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/schedule.ts#L65)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **User-facing docs still expose only three schedule kinds:** Current cron docs list only `--at`, `--every`, and `--cron` as supported schedule types, with no documented conditional or deadline-aware interval feature.
  - file: [docs/automation/cron-jobs.md:56](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/automation/cron-jobs.md#L56)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Agent-facing cron tool schema also limits schedules to three kinds:** The built-in cron tool advertises `CRON_SCHEDULE_KINDS = ["at", "every", "cron"]`, so current agent/plugin integrations do not expose a conditional schedule contract either.
  - file: [src/agents/tools/cron-tool.ts:24](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/tools/cron-tool.ts#L24)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)

## Risks / Open Questions

- There may be a userland workaround by having an agent reschedule jobs via `cron.update`, but that is not the declarative conditional scheduling requested in the issue.
