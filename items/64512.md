## Summary

Keep this PR open. Current main still lacks Amazon Bedrock service_tier forwarding, and this PR remains the focused implementation candidate for the linked Bedrock feature request. The latest diff is scoped to Bedrock docs, Bedrock plugin runtime/tests, and a shared provider-stream helper, but maintainer review should still resolve the public Plugin SDK export/baseline question and tighten the non-overwrite regression test before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to keep the behavior owned by the Amazon Bedrock plugin, inject top-level { serviceTier: { type } } only for bedrock-converse-stream requests through the existing extraParams/wrapStreamFn seam, decide whether the helper belongs in the public Plugin SDK barrel or should stay plugin-local/internal, update API baseline/docs if the public export is intentional, strengthen the preserve-existing-payload test, and run the relevant changed checks before merge.
