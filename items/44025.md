## Summary

Keep open. Current main supports explicit ordered model fallbacks, and the OpenAI Codex default has already moved to gpt-5.5, but the requested wildcard/auto provider-pool fallback syntax is not implemented. The public model-resolve hook can choose one provider/model before resolution, but it does not expose a way to expand or mutate the fallback candidate chain dynamically.
