## Summary

Keep open. Current `main` still lacks the nested `usage` normalization described in the PR, so this is not implemented-on-main, and the PR's follow-up commit in the provided discussion appears to have resolved the main review concern by limiting the fix to `normalizeUsage()`.

## Evidence

- **Current main does not read nested usage payloads:** `UsageLike` has no `usage` property on `main`, and `normalizeUsage()` only reads top-level token fields (`prompt_tokens`, `completion_tokens`, `total_tokens`, cache aliases, timings). A Bailian-style payload shaped as `{ usage: { ... } }` would therefore normalize to `undefined` on current `main`.

That matches the review feedback that nested Bailian handling belongs in `normalizeUsage()`, not in widening `hasNonzeroUsage()`.
