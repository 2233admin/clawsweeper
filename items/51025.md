## Summary

Keep open. Current `main` does not implement the PR's nested `usage` normalization, so this is not an `implemented_on_main` close and there is no strong basis for another auto-close reason.

## Evidence

- **`UsageLike` still lacks nested `usage` field:** Current `main` defines many top-level token aliases but no nested `usage?: {...}` property, so Bailian-style `{ usage: { prompt_tokens, completion_tokens, total_tokens } }` payloads are not represented here.
  - file: [src/agents/usage.ts:3](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/usage.ts#L3)
  - command: `nl -ba src/agents/usage.ts | sed -n '1,40p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **`normalizeUsage()` only reads top-level fields:** Normalization checks `raw.cacheRead`, `raw.prompt_tokens`, `raw.completion_tokens`, `raw.total_tokens`, `input_tokens_details`, `prompt_tokens_details`, and llama timing aliases, but there is no `raw.usage ?? raw` fallback or any nested `usage.*` reads.
  - file: [src/agents/usage.ts:100](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/usage.ts#L100)
