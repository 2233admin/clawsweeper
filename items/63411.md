## Summary

Current main still does not implement the requested semantic streaming repetition breaker or loop-specific context sanitization. The idle timeout remains chunk-activity based, assistant text/thinking streams are processed without n-gram or structural-loop checks, and the shipped loop detection surface is scoped to tool-call patterns. The related open implementation candidate #39961 remains the best path to review or adapt.

## Best Possible Solution

Keep this issue open. The maintainer path is to review or adapt #39961, or implement an equivalent core stream guard near the assistant stream subscription/wrapper layer: detect repeated text/thinking patterns while the stream is active, abort with explicit loop metadata, and quarantine loop-aborted partial assistant output from future model replay while preserving user-visible transcript history.
