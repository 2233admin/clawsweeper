## Summary

Keep open. Current `main` fixed the warning/classification side of the report, but the routed-provider failure mode discussed in the issue is still consistent with current code: `apply_patch` is only created when the resolved provider is exactly `openai` or `openai-codex`. The remaining issue should not be auto-closed as implemented, cannot-reproduce, or duplicate based on the available evidence.
