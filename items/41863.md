## Close Comment

Closing as implemented on `main` based on this Codex review. The current pruning code in `src/agents/pi-hooks/context-pruning/pruner.ts:225` and `src/agents/pi-hooks/context-pruning/pruner.ts:309` already allows image-bearing `toolResult` messages to be pruned by converting image blocks to `[image removed during context pruning]` during soft trim and hard-clearing them when required, instead of exempting them from pruning. Current tests in `src/agents/pi-hooks/context-pruning/pruner.test.ts:224`, `src/agents/pi-hooks/context-pruning/pruner.test.ts:266`, and `src/agents/pi-hooks/context-pruning/pruner.test.ts:301` cover the image soft-trim, image-only replacement, and hard-clear paths. This means the overflow case described here is already addressed on `main`, so this PR is no longer needed.
