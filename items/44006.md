## Summary

Keep open. Current main still routes inbound Telegram document media into media understanding/file extraction, allows application/pdf by default, and the bundled document-extract plugin still loads @napi-rs/canvas for low/no-text PDFs. I did not find a CPU-feature guard, musl-specific fallback, or user-facing kill switch that would make the reported SIGILL path safe to auto-close.
