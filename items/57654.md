## Summary

Keep open. The checked-out main still has the out-of-order `message_start` behavior unfixed: `handleMessageStart` resets assistant state immediately instead of finalizing an in-flight message, Kimi Coding still hardcodes the `User-Agent: claude-code/0.1.0` header reporters identified, and current main only sanitizes this exact error into a friendlier message. The related fix PR #57853 is still open and unmerged.
