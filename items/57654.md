## Summary

Current main already fixes the observable #57654 failure path: `anthropic-messages` now uses OpenClaw's boundary-aware raw SSE transport instead of the Anthropic SDK stream state machine that emitted the `message_start`/`message_stop` ordering error. The related PR #57853 proposed a narrower workaround, but current main and the shipped transport path supersede it.

## Best Possible Solution

Close #57654 as implemented on main. Keep the shipped boundary-aware `anthropic-messages` transport as the canonical fix, and treat PR #57853 as superseded by the broader transport-path change rather than merging its narrower state-reset workaround.
