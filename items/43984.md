## Summary

Keep open. Current `main` still filters stale sessions purely by `updatedAt` when `sessions.list` receives `activeMinutes`, with no carve-out for top-level agent main sessions, and the chat UI still requests that filter on refresh. The PR's targeted fix is not present on `main`, and the provided discussion shows the author already addressed the custom-`mainKey` review concern.

## Evidence

- **Current main still drops stale sessions under activeMinutes:** `listSessionsFromStore` computes `activeMinutes` and then filters with `sessions = sessions.filter((s) => (s.updatedAt ?? 0) >= cutoff)`, with no exception for `agent:<id>:<mainKey>` sessions. That means a stale `agent:main:main` row still disappears today when the filter is used.
