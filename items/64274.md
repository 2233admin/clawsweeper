## Summary

Keep open. Current main still keys runtime auth-profile snapshots by resolved auth-store path, and `resolveRuntimeAuthProfileStore()` still treats an explicit agent dir as the main store when `OPENCLAW_AGENT_DIR` resolves to that same path. The linked PR #64320 identified the likely root cause and proposed keying snapshots by the original `agentDir`, but that PR is closed and not merged, and current main does not contain its described regression test or implementation.

## Best Possible Solution

Keep #64274 open. The likely maintainer path is to land a scoped auth-profile runtime fix equivalent to PR #64320: key runtime snapshots by the original `agentDir` identity, preserve main-agent fallback semantics, and add the `OPENCLAW_AGENT_DIR` sub-agent collision regression test covering MiniMax or a provider-generic OAuth profile.
