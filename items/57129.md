## Summary

Keep open. Current `main` still shares Slack top-level room messages on a per-channel session instead of creating a fresh isolated run for each new top-level question, so the reported cross-question context bleed remains plausible and is explicitly encoded in code, tests, and docs.

## Evidence

- **Routing code keeps top-level room turns on channel session:** `resolveSlackRoutingContext` explicitly says top-level channel messages must stay on the per-channel session, uses `message.channel` as the base conversation id for room messages, and only adds a thread-specific session key for true thread replies.
  - file: [extensions/slack/src/monitor/message-handler/prepare-routing.ts:68](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/slack/src/monitor/message-handler/prepare-routing.ts#L68)
  - command: `nl -ba extensions/slack/src/monitor/message-handler/prepare-routing.ts | sed -n '45,125p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Tests lock in shared session behavior for separate top-level channel messages:** The Slack routing tests assert that two distinct top-level channel messages get the same `sessionKey` and no `:thread:` suffix, including a loop that enforces this for `replyToMode` `all`, `first`, and `off`.
  - file: [extensions/slack/src/monitor/message-handler/prepare.thread-session-key.test.ts:45](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/slack/src/monitor/message-handler/prepare.thread-session-key.test.ts#L45)
  - command: `nl -ba extensions/slack/src/monitor/message-handler/prepare.thread-session-key.test.ts | sed -n '40,140p'`

- **Inbound room history is still keyed by channel for non-thread turns:** `prepareSlackMessage` builds `combinedBody` and `InboundHistory` from `ctx.channelHistories` using `historyKey`; for non-thread room messages that key resolves to `message.channel`, so top-level room turns still inherit prior room context.
  - file: [extensions/slack/src/monitor/message-handler/prepare.ts:595](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/slack/src/monitor/message-handler/prepare.ts#L595)
