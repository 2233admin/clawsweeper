## Summary

Keep open. Current main has partial realtime plumbing for Control UI reconnects and `session.message` reloads, but the linked fix for #44707 (#56291) was closed unmerged and the current gateway still drops transcript updates when `emitSessionTranscriptUpdate()` is called without an inline `message`. That is the exact remaining path the linked PR identified for external-channel messages only appearing after refresh, so there is not enough evidence to conservatively close this regression.
