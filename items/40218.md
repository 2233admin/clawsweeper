## Summary

Keep open. Current main still does not implement caller-provided cron job IDs or duplicate-ID validation, and this PR addresses a real core cron/API gap rather than a ClawHub/plugin-boundary item. The branch likely needs maintainer-quality cleanup and hardening before landing, but there is no strong close reason.

## Best Possible Solution

Keep the PR open and converge it into a narrow core cron contract change: add optional stable id support to CronJobCreate/createJob, preserve id through normalization, gateway validation, CLI --id, cron-tool flat-param recovery, and Swift protocol models; reject duplicate ids case-insensitively under the cron add lock with guards for malformed persisted jobs; document the behavior and keep targeted regression tests. Drop unrelated secret-scanning and fixture changes from this PR.
