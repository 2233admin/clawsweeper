## Summary

Keep PR #58034 open. Current main has the #59007 baseline for trusted env-proxy DNS ordering, but it does not implement this PR's remaining central behavior: a user-facing `tools.web.fetch.useTrustedEnvProxy` opt-in, propagation from `web_fetch` into `fetchWithWebToolsNetworkGuard`, matching docs/schema coverage, or the trusted-proxy pre-dispatch hostname-policy check. The PR discussion also contains useful branch verification from cosmicnet and independent proxy-only sandbox testing by mjamiv, while later comments correctly distinguish this work from #59007 and #60785.

## Best Possible Solution

Do not close #58034 as implemented or superseded. The best path is maintainer review of the narrow opt-in design: add `tools.web.fetch.useTrustedEnvProxy`, pass it through `web_fetch` only when explicitly enabled, keep protocol-aware proxy and `NO_PROXY` fallback behavior aligned with Undici, add the trusted-proxy hostname-policy check before dispatch, and ship the schema/docs/tests together. #59007 should remain treated as the landed baseline, not a full replacement for this PR.
