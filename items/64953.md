## Summary

Keep #64953 open. Current main still does not implement the requested replace-mode behavior for the CLI model picker or gateway catalog: both picker paths and `loadGatewayModelCatalog` still call the shared full catalog loader, and `loadModelCatalog` currently returns the merged registry/plugin catalog without a `models.mode === "replace"` filter. The related PRs in the provided context are closed unmerged, while the linked bug #64950 remains open, so there is no implemented or superseding mainline fix to close this PR against.

## Best Possible Solution

Keep #64953 and/or canonical issue #64950 open. Maintainers should decide the replace-mode catalog contract, then implement a shared mode-aware catalog path for `promptDefaultModel`, `promptModelAllowlist`, and gateway `models.list`, preserving any intended implicit/provider-plugin discovery behavior. Add focused regression tests covering replace mode for the CLI picker and gateway catalog before landing.
