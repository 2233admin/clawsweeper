## Summary

Keep open. Current main has meaningful prompt-cost mitigations: compact/on-demand skill catalogs, static tool profiles and allow/deny policy, bootstrap size caps plus continuation-skip/lightweight modes, and compaction. It still does not implement the central request for automatic per-turn conditional tool/schema relevance routing or default summary/index/on-demand workspace bootstrap injection, and the narrower tool-schema tracker #14785 remains open.

## Best Possible Solution

Keep this issue open as the broad per-turn prompt overhead tracker. Use #14785 for narrower tool-schema footprint work, but solve this issue with a compatible automatic tool relevance/schema-routing design plus a default or opt-in workspace bootstrap summary/index/on-demand mode. Preserve the shipped mitigations: compact skill catalogs, explicit tool profiles and allowlists, bootstrap caps/contextInjection controls, lightweight cron/heartbeat modes, and compaction.

## Risks / Open Questions

- Automatic conditional tool/schema routing still needs a policy-preserving design for explicit allow/deny lists, owner-only tools, recovery tools, plugin and bundled MCP/LSP tools, provider schema constraints, and channel-specific capabilities.
- A default workspace bootstrap summary/index mode could hide standing instructions unless it has deterministic ordering, clear cache behavior, and a reliable path for the agent to read full files on demand.
- Closing this now would lose the broader tracker while #14785 only covers the narrower tool-schema footprint and does not cover workspace bootstrap summary/index behavior.
