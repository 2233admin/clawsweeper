## Summary

Keep open. Current main still lacks the PR’s central behavior: provider model config does not accept `params`, resolved runtime models do not carry merged `params`, and both Ollama request paths derive `num_ctx` from `contextWindow` rather than a configured `params.num_ctx`. The related #44678 context does not supersede this PR because it was closed unmerged and narrower than #69464’s schema/runtime/OpenAI-compat coverage.

## Best Possible Solution

Keep #69464 open and review or adapt its schema, runtime model, params-merging, Ollama native/OpenAI-compatible request, docs, and regression-test changes. The best product outcome is to make explicit `params.num_ctx` from both `agents.defaults.models["ollama/<id>"]` and `models.providers.ollama.models[]` win over discovered GGUF context metadata when constructing Ollama requests.
