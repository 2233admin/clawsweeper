## Summary

Keep open. Current main does not implement the PRâ€™s central behavior: provider model entries still cannot declare `params`, resolved runtime models do not carry configured params, and both native Ollama and Ollama OpenAI-compatible `num_ctx` paths still derive from `contextWindow` rather than `params.num_ctx`. The related #44678 is closed unmerged in the provided context, so it does not supersede this PR.

## Best Possible Solution

Keep #69464 open for maintainer review. The best implementation path is to land or adapt its approach: allow optional model-level `params` in provider model config schema/types/generated schema, thread merged params from `agents.defaults.models` and `models.providers.*.models[]` onto the resolved runtime model with explicit user config precedence, and make both native Ollama and OpenAI-compatible Ollama request builders honor finite positive `params.num_ctx` with regression tests and docs.
