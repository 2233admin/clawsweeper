## Summary

Keep open. Current main still lacks the config schema, runtime model plumbing, and Ollama request behavior that PR #69464 proposes. The PR discussion also shows review feedback was addressed on the PR branch, so this is an active plausible fix rather than an obsolete cleanup item.

## Best Possible Solution

Keep #69464 open as the active candidate fix. The best path is to review or land a scoped implementation that accepts provider model params, threads agents.defaults.models and models.providers params onto the runtime model with documented precedence, uses a validated params.num_ctx in both Ollama native and OpenAI-compatible request paths, and keeps the proposed regression tests/docs. Broader Ollama parameter forwarding from #48010 can remain separate if not fully solved here.
