## Summary

Keep this PR open. Current main still skips bundled pre-install resolution for scoped npm specs, so the narrow helper/test change is not implemented. The PR is still a plausible focused fix, though its body should be reconciled with current main's duplicate precedence tests before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt the narrow helper change if maintainers agree with the product behavior: trim the raw spec, keep plugin-id lookup for bare specs, use exact `npmSpec` lookup for non-bare scoped specs, preserve non-matching scoped specs falling through to ClawHub/npm, and update the CLI docs or PR wording to match current duplicate precedence.
