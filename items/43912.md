## Summary

Keep open. Current `main` does not implement this PR's proposed behavior: scoped npm specs still skip the bundled pre-install path. At the same time, current `main` has newer install-precedence behavior that weakens the PR's original rationale, because registry specs are first routed through preferred ClawHub/npm install flow and explicit installed globals now beat bundled duplicates. That mixed evidence is not strong enough for a conservative close.

## Evidence

- **Scoped specs still bypass bundled pre-plan on main:** `resolveBundledInstallPlanBeforeNpm` immediately returns `null` for any non-bare spec, so `@openclaw/<plugin>` still does not take the bundled pre-install path on current `main`.
