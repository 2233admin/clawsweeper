## Summary

Current `main` does not implement the requested MCP connectivity panel. The Control UI only exposes MCP as config editing and a configured-server count, while the MCP docs explicitly say saved server definitions are not connected or validated at config time.

## Evidence

- **Control UI tracks only configured MCP server count:** `extractMcpServerCount` reads `config.mcp.servers` and returns `Object.keys(servers).length`; there is no per-server connectivity, tool inventory, reconnect action, or surfaced error state in this path.
  - file: [ui/src/ui/app-render.ts:500](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/ui/src/ui/app-render.ts#L500)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Quick Settings exposes MCP as configure-only:** The quick settings card shows only the number of configured MCP servers and a `Configure` button.
  - file: [ui/src/ui/views/config-quick.ts:318](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/ui/src/ui/views/config-quick.ts#L318)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Control UI MCP section is definitions-oriented:** The config form metadata labels `mcp` as `Model Context Protocol server definitions`, which matches editing config rather than a live connectivity/status surface.
  - file: [ui/src/ui/views/config-form.render.ts:324](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/ui/src/ui/views/config-form.render.ts#L324)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Current Control UI docs do not describe an MCP connectivity panel:** The `What it can do (today)` section lists current browser surfaces such as chat, channels, skills, nodes, config, debug, logs, and updates, but not MCP connectivity status, per-server tools, reconnect, or error details.
  - file: [docs/web/control-ui.md:105](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/docs/web/control-ui.md#L105)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **OpenClaw MCP registry commands explicitly do not validate connectivity:** The MCP CLI docs state that `openclaw mcp list/show/set/unset` only manage config, `do not connect to the target MCP server`, and `do not validate whether the command, URL, or remote transport is reachable right now`.
  - file: [docs/cli/mcp.md:362](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/docs/cli/mcp.md#L362)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
