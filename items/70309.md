## Summary

Keep this issue open. Current main still exposes MCP config editing and session-scoped MCP tool materialization, but it does not provide the requested Control UI connectivity panel with per-server connected/error state, tool names, manual refresh/reconnect, or surfaced redacted error details.

## Best Possible Solution

Keep this issue open. The best path is an additive Gateway operator diagnostics API, likely under the Gateway protocol/server-methods surface and backed by MCP config/runtime loading, returning each configured server name, transport summary, connected/error state, tool names/count, and redacted last error, plus a scoped refresh/reconnect action. Then render it in Control UI under Infrastructure > MCP and document the operator behavior.
