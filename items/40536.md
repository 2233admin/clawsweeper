## Summary

Keep #40536 open. Current main still lacks a first-class per-skill model override: the skill config/type/schema surface does not accept `skills.entries.<skill>.model`, docs only describe enable/env/apiKey/custom config fields, and runtime model selection still happens at agent/session/run scope before the active model chooses a skill from the prompt catalog. Related model-override work for sessions, channels, cron, compaction, media tools, subagents, and plugin hooks does not supersede this per-skill routing request.

## Best Possible Solution

Keep the issue open for maintainer/product design. A real implementation should add a first-class `skills.entries.<skillKey>.model` or documented skill metadata contract, update config types/zod/schema help/generated schema/docs/UI/tests, enforce model allowlist/catalog behavior, and define a runtime decision point for both model-selected skills and explicit slash skill invocations without permanently changing the session model.
