## Summary

Keep open. Current `main` does not implement the PR's cross-model tool-ID normalization: `repairToolUseResultPairing` still matches `toolResult` IDs only by exact raw ID, while strict tool-ID sanitization still removes underscores and hyphens. That leaves the reported mismatch path plausible on `main`, and I did not find a regression test covering sanitized-result/raw-call pairing.

## Evidence

- **Exact-ID matching still on main:** `repairToolUseResultPairing` builds `toolCallIds` from raw assistant tool-call IDs and only accepts a tool result when `toolCallIds.has(id)` succeeds on the extracted result ID. There is no fallback through `sanitizeToolCallId` or any normalized-ID lookup.
