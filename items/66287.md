## Summary

Current main still has the reported memory-core behavior: default deep dreaming only promotes candidates that pass score, aggregate signal-count, and diversity gates, and current tests explicitly preserve the behavior where a one-off high-scoring recall does not reach MEMORY.md. Manual CLI threshold overrides exist, but they are not a durable priority lane for important single-event feedback.

## Best Possible Solution

Keep this open for memory-core maintainer/product design. The best path is a tested durable lane for high-priority single-event candidates, such as explicit remember-directive metadata, a manual priority flag, or a critical-feedback classifier, with narrowly scoped bypass semantics for signal/diversity gates. Update memory-core tests and public dreaming/memory docs once that behavior is defined.
