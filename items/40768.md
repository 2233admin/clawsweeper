## Summary

Keep open. Current main still recognizes Feishu bot mentions by exact `open_id` equality and still drops group messages when `requireMention` is enabled and that equality fails. The related #42410 review treated #40768 as the canonical tracker, and #40782 remains an open, incomplete fix attempt rather than shipped behavior.

## Best Possible Solution

Keep #40768 open as the canonical Feishu multi-bot mention identity bug. The fix should live in the Feishu plugin and thread a verified stable identity strategy through `checkBotMentioned`, mention stripping/forwarding, debounce merging, startup identity discovery, and regression tests. #40782 may be updated, but it should not merge until the Feishu API identity source is verified and the non-open_id paths are covered.
