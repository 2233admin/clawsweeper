## Summary

Keep open. Current main still hard-codes the TUI footer and exposes no footer template, footer fields, balance/cooldown display, or plugin status-provider seam in the TUI render path. Existing `/status`, `/usage full`, `/usage cost`, and provider usage windows cover adjacent usage/cost visibility, but not the persistent customizable TUI footer requested here. The supplied context also shows an open README-only PR with closing syntax, so this issue should not be closed before that PR is reviewed/closed/reworked or a real implementation merges.

## Best Possible Solution

Keep this issue open. The best maintainer path is to close or rewrite the README-only closing PR, then either implement a real TUI footer configuration surface across `src/tui/tui.ts`, `TuiOptions`, CLI/config/docs, and focused tests, or split the work into a canonical footer-template issue plus a reviewed plugin status-provider API. Existing usage and provider quota plumbing can supply data, but persistent customizable TUI footer rendering still needs a core product/API decision.
