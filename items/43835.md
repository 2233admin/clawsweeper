## Summary

Keep open. Current `main` still removes sandbox runtimes and registry entries during prune but does not delete the per-scope workspace directories under `cfg.workspaceRoot`, so the requested behavior is not implemented on `main`. The PR also has substantive review feedback and follow-up commits addressing safety concerns, so it is not a conservative cleanup close candidate.
