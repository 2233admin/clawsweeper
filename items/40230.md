## Summary

Keep PR #40230 open. Current main still loads workspace bootstrap files only through the boundary reader, which returns boundary validation failures directly; symlinked AGENTS.md/SOUL.md/USER.md/MEMORY.md files that resolve outside the workspace are still treated as missing. The linked issues and PR discussion show this is a concrete user-facing bug, not stale cleanup.

## Best Possible Solution

Keep the PR open for maintainer review or replacement. The best fix belongs in src/agents/workspace.ts: support symlinked fixed workspace bootstrap files only if intentionally allowed, read through a verified descriptor or equivalent race-safe path, preserve hardlink and size validation, avoid widening extra-bootstrap traversal behavior, and add regression coverage for external symlinks, dangling symlinks, oversized targets, hardlinked targets, and extra-bootstrap boundaries.
