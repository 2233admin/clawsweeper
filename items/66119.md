## Summary

Keep this issue open. The report is recent, detailed, destructive, and current main still auto-starts the bundled Signal plugin's signal-cli daemon without a preflight for a local signal-cli account file with `registered:false`, without automatic backup, and without a documented recovery guard for this failure mode.

## Best Possible Solution

Keep the issue open and investigate in the bundled Signal plugin. The best path is a Signal-owned, non-mutating startup preflight before daemon auto-start where feasible, fail-closed warning behavior when a local account file is detectably `registered:false`, focused mocked regression coverage around startup, and user-facing backup/recovery docs. If the destructive transition is ultimately upstream signal-cli behavior, OpenClaw should still avoid auto-starting the daemon when it can safely detect an obviously invalid local account state.
