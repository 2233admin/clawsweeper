## Summary

Current main still does not satisfy the documented Telegram inline-button callback text contract. The callback_query path exists and routes fallback callbacks into the agent, but generic callback data is still forwarded as the raw payload, while the docs promise `callback_data: <value>`. PR #54962 remains the open implementation candidate and should be reviewed or landed before this issue is closed.

## Best Possible Solution

Keep this issue open. Review and land PR #54962, or apply an equivalent patch in `extensions/telegram/src/bot-handlers.runtime.ts` that forwards generic Telegram callback fallback text as `callback_data: ${data}` while preserving native command callbacks. Add or update a regression test that asserts the documented prefix for generic callbacks, then close this issue after the fix merges.
