## Summary

Keep this PR open. Current main still has the unguarded assistant `surface_error` throw path, while the PR is a focused source/test fix for a plausible regression where stale classified `errorMessage` text can convert a successful assistant turn into a hard client error. Affected surface: `pi-embedded-runner` failover handling for webchat/channel error delivery; PR URL: https://github.com/openclaw/openclaw/pull/70900.

## Best Possible Solution

Review and land this PR, or land an equivalent minimal fix in `src/agents/pi-embedded-runner/run/assistant-failover.ts` that requires `params.failoverFailure` before throwing from the assistant `surface_error` branch. Preserve #70848/#70124 behavior for genuine provider failures and include the proposed regression test for `failoverFailure=false` with a classified `failoverReason`.
