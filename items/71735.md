## Summary

Keep #71735 open. Current main has plugin command registration, duplicate/reserved-name checks, and passes `gatewayClientScopes` into command handlers, but it does not implement declarative command `requiredScopes`, host-side pre-handler scope enforcement, trusted command-root ownership, or `continueAgent` semantics. The linked RFC PR #71731 is open and described in the provided GitHub context as docs-only/proposed, so it does not supersede the requested SDK/API work.

## Evidence

- **SDK command type lacks proposed fields:** `PluginCommandContext` exposes `gatewayClientScopes`, but `PluginCommandResult` is just `ReplyPayload` and `OpenClawPluginCommandDefinition` only declares `name`, native metadata, `description`, `acceptsArgs`, `requireAuth`, and `handler`; there is no `requiredScopes`, `ownership`, or `continueAgent` surface.
