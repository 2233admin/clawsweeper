## Summary

Keep PR #72253 open. Current main still treats structured `Exec completed (..., code 0)` and `Exec failed (..., signal SIGTERM)` events as relayable exec completions, with no trusted local-exec/internal-only classifier. The PR is a recent, focused fix for linked bug #72252; related #69492/#67273/#71860 overlap, but current main has not absorbed this narrower SIGTERM/code-0 behavior.

## Best Possible Solution

Keep #72253 open for maintainer review. The best path is a narrow core heartbeat fix that marks trusted/local structured `code 0` and cleanup `signal SIGTERM` exec completions internal, preserves relay for actionable failures such as `code 1`, and coordinates with #69492 without waiting for the broader audience-classification design to resolve the concrete #72252 chat-noise bug.
