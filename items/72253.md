## Summary

Keep PR #72253 open. Current main still treats structured async exec completions as relayable heartbeat events, still revives stripped HEARTBEAT_OK replies for any exec completion, and does not carry the PR's proposed local-exec origin/context metadata from the exec producer into system events. The PR remains a focused recent fix for the open #72252 restart-cleanup/SIGTERM chat-noise bug, with overlap to broader #69492 but no current-main implementation or canonical landed replacement.

## Best Possible Solution

Keep #72253 open for maintainer review. The best path is a narrow core heartbeat fix that marks trusted/local structured `code 0` and cleanup `signal SIGTERM` exec completions internal, preserves user relay for actionable failures like `code 1`, carries enough producer metadata to avoid spoofing, and coordinates with #69492 without waiting for the broader audience-classification redesign to solve the immediate #72252 chat-noise bug.
