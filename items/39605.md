## Summary

Keep #39605 open. Current main at 43a003b8a0629ad2c73947b2a863d99dd506bcb9 still has the reported routing risk on Discord native commands and Slack slash commands: both paths resolve command routes using the config object captured at provider startup, while route/session-key construction reads `session.dmScope` from the supplied config. Telegram now has a fresh-runtime-config pattern and a regression test, but the Discord and Slack surfaces named by the issue remain unresolved. The related PR #39617 is still the canonical open fix attempt in the supplied GitHub context, not shipped behavior.

## Best Possible Solution

Keep #39605 open and land PR #39617 or an equivalent maintainer-reviewed fix that reloads current runtime config before Discord native command route-state resolution and Slack slash command route resolution. Add focused Discord and Slack regression tests proving DM native/slash commands honor updated `session.dmScope` without recreating the provider; Telegram can remain on its existing fresh-runtime-config pattern unless maintainers want the issue narrowed after Discord/Slack are fixed.
