## Summary

Keep this PR open. Current main still preserves leading whitespace after stripping inline directive tags before the Gateway/TUI live-chat merge, and that merge still falls back to overlap dedupe when prefix comparison fails. The reported xumum/GOOGLE corruption path remains plausible, but the PR needs to be rebased/adapted to the current shared live-chat projector and narrowed so intentional whitespace is preserved.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, move the production change into src/gateway/live-chat-projector.ts, normalize only whitespace introduced by stripped leading inline directive tags before merging, preserve intentional leading/trailing delta separators, and add a regression test through the shared projector helper or the real Gateway event path that proves the xumum/GOOGLE corruption no longer occurs.
