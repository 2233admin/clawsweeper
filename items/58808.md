## Summary

Keep PR #58808 open. Current main still lacks the requested trusted sender context on the public ChannelAgentToolFactory path, while adjacent message-tool/plugin discovery paths already receive requesterSenderId and senderIsOwner. This is a real core plugin SDK seam gap, not a cleanup-close candidate.

## Best Possible Solution

Keep the PR open or land an equivalent core SDK change: extend ChannelAgentToolFactory params with optional requesterSenderId and senderIsOwner, thread options.senderId/options.senderIsOwner through listChannelAgentTools, and add a narrow regression test for a fake channel plugin agentTools factory. This should remain in OpenClaw core because the missing data is trusted host-provided context on a public plugin contract.
