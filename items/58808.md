## Summary

Current main still has the gap described by PR #58808: channel agent tool factories only receive `{ cfg }`, while the adjacent core tool path receives trusted sender context. The PR is a focused public channel/plugin contract improvement, related issue #58806 is still open, and there is no shipped implementation or superseding item in the provided context.

## Best Possible Solution

Keep #58808 open. The best path is to land or revise the PR as an additive SDK/channel contract change: widen `ChannelAgentToolFactory` and `listChannelAgentTools`, pass `options.senderId`/`options.senderIsOwner` from `createOpenClawCodingTools`, and add a focused unit test for the forwarding behavior before closing #58806.
