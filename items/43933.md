## Summary

Keep open. Current main has useful generic crash-loop and health-monitor guardrails, plus a narrow Telegram sendChatAction 401 breaker, but it still does not implement the requested channel-level auth-failure circuit breaker that tracks 401/403 per channel/account and pauses the broken channel after consecutive credential failures.

It is not maintainer-authored and not protected, but it asks for a core/bundled-channel reliability behavior rather than an optional plugin/ClawHub feature.
