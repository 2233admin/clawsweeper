## Summary

Keep this issue open. Current main still passes the configured compaction reserve budget through to Pi summarization without clamping it to the selected model output limit, and same-author PR #54392 remains open as an implementation candidate with closing syntax for this issue.

## Best Possible Solution

Review and either land PR #54392 or replace it with an equivalent maintainer-approved fix that clamps compaction summarization reserve/output budgeting against the selected model's maxTokens before calling Pi generateSummary. Add regression coverage for Anthropic Claude 4.6 1M-context compaction proving reserveTokensFloor=300000 cannot produce max_tokens above the model output ceiling.
