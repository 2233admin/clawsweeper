## Summary

Keep open. Current main still has the stale `status --deep` guidance in the `channels status` CLI description and both human status footers, while `channels status` only registers `--probe`, `--timeout`, and `--json`. The PR is a focused fix for open issue #69341 and is not obsolete or superseded by current main.

## Best Possible Solution

Keep PR #71887 open for maintainer review or merge after normal validation. The best product outcome is to land this narrow docs/help/status-tip cleanup, preserving the existing command contract where only top-level `openclaw status --deep` accepts `--deep` and `openclaw channels status` remains limited to channel status options.
