## Summary

Current main still generates macOS LaunchAgent StandardOutPath and StandardErrorPath from the OpenClaw state directory, which defaults to $HOME/.openclaw/logs, and writes those paths verbatim into the plist during install and restart regeneration. The reported symlinked/external-volume LaunchAgent failure remains actionable in core.

## Best Possible Solution

Keep the issue open. The fix should change the macOS LaunchAgent stdout/stderr path generation used by install and restart regeneration: either default those launchd service logs to a stable non-symlinked macOS user log directory, or resolve symlinked state/log paths before writing StandardOutPath and StandardErrorPath. Add regression coverage for symlinked OPENCLAW_STATE_DIR or ~/.openclaw and plist regeneration, then align the macOS/gateway logging docs.
