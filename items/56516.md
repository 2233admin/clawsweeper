## Summary

Keep open. Current main still returns completed subagent sessions in `sessions.list`, the web UI loads that list directly, and newer gateway/UI behavior explicitly relies on ended subagent rows remaining visible for follow-up and `/redirect` flows. This issue is not implemented on main and is not stale enough for insufficient-info closure.
