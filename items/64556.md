## Summary

Current main still reproduces the reported mismatch: wake hook mappings accept agentId/sessionKey in config but drop those fields before dispatch, and dispatchWakeHook still enqueues to the resolved main session. The provided GitHub context also shows open PR #70268 with closing syntax for this issue, so this should stay open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #70268 as the active implementation candidate. The fix should thread agentId/sessionKey through the wake HookAction and mapped dispatch path, reuse the existing hook agent/session policy helpers, preserve legacy main-session fallback when routing is omitted, and add regression tests for explicit sessionKey, agentId-derived main session routing, templated session-key guardrails, and direct /hooks/wake behavior.
