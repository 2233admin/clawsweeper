## Summary

Keep PR #39772 open. Current main still does not implement the PR's first-preview-only reply gating inside Telegram draft/message streaming, and the PR remains a plausible fix direction for #39718. It should not be closed as implemented or superseded, but it also should not land unchanged because the unresolved review concerns about first-send failure recovery and `replyToMode: "all"` semantics are still material.

## Best Possible Solution

Keep the PR open for maintainer or author revision. The best fix should suppress the transient Telegram `Deleted message` preview artifact while preserving reply-threading semantics: distinguish `replyToMode: "first"` from `"all"`, preserve `message_thread_id` after stream rotation, avoid consuming reply context before the first successful send, handle concurrent `forceNewMessage()` safely, and add regression coverage for rotation, thread params, first-send failure/materialize recovery, and all-mode behavior.
