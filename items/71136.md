## Summary

Keep #71136 open. Current main has a partial embedded-runner guard that caps an oversized `agents.defaults.contextTokens` against runtime model metadata, but the requested conservative minimum across agent config, provider model config, discovery/runtime metadata, and defaults is still not implemented consistently, and `maxTokens` still preserves configured overrides in key model-resolution paths.

## Best Possible Solution

Keep the issue open and implement a shared conservative token-limit resolver. It should collect positive limits from agent defaults/per-agent config, provider model config, runtime/discovered model metadata, persisted session state where appropriate, and the hard default, then use the minimum for `contextTokens`. For `maxTokens`, clamp configured model metadata and stream/request params to the selected model's known output limit at model-resolution or transport boundaries, with focused tests for oversized agent defaults, oversized provider config over smaller discovery metadata, per-agent context caps, and oversized `maxTokens`.
