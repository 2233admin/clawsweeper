## Summary

Keep open. Current `main` does not implement clickable Feishu command-option buttons: the Feishu docs still describe command use as plain text only, and the Feishu presentation renderer turns button/select blocks into markdown fallback text rather than interactive controls. There is existing Feishu card-action plumbing for bespoke quick-action cards, which makes the request coherent and still actionable rather than a close candidate.

## Evidence

- **Feishu docs still describe command use as plain text:** The Feishu channel docs say: "Feishu/Lark does not support native slash-command menus, so send these as plain text messages." That matches the reported current UX rather than the requested clickable option buttons.
  - file: [docs/channels/feishu.md:172](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/channels/feishu.md#L172)
  - command: `nl -ba docs/channels/feishu.md | sed -n '164,176p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Feishu presentation renderer is markdown fallback, not interactive controls:** `buildFeishuPresentationCard` wraps the presentation in a single markdown element using `renderMessagePresentationFallbackText(...)`; it does not emit Feishu `action`/`button` card elements for presentation buttons or selects.
  - file: [extensions/feishu/src/channel.ts:134](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/channel.ts#L134)

- **Tests assert buttons and selects degrade to text labels:** Current tests explicitly expect presentation buttons to render as `- Run help` and selects as `Pick one:\n- Option A` inside markdown card content, confirming current `main` intentionally falls back to text instead of clickable controls.
  - file: [extensions/feishu/src/channel.test.ts:357](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/channel.test.ts#L357)
