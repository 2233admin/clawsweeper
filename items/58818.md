## Summary

Keep #58818 open. Current main has pieces of adjacent behavior, including configurable compaction tail budgets, startup memory prelude, session lifecycle hooks, and reset transcript archiving, but it does not implement the requested user-facing `session.minTailMessages` contract. Resets still mint a new session id and archive the old transcript without seeding the last N raw messages into the new context, and context-engine `assemble()` has no min-tail parameter or host-enforced floor. The linked #45608 is complementary pre-reset memory-flush work, not a canonical duplicate for raw-message carry-forward.

## Best Possible Solution

Keep the issue open and treat it as a concrete product/API request. A proper implementation would add an explicit user-facing raw-tail policy, wire reset paths to carry or inject bounded raw transcript tail context, make compaction honor the same floor, and extend the context-engine contract or host enforcement so plugins cannot silently drop the configured tail.
