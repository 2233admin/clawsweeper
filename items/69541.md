## Summary

Keep #69541 open. Current main has several point sanitizers for known injected blocks, but it does not implement the RFC’s central manifest-backed registry or an SDK/core getter for plugin-declared injected context tags. This is not a ClawHub-only item because the missing piece is an additive core plugin contract that downstream core consumers would need to read.

## Best Possible Solution

Keep the RFC open for maintainer product/API judgment. The likely best implementation is an additive `contracts.injectedContextTags` manifest contract with validation/normalization, a deterministic aggregated getter in the plugin manifest/registry surface, SDK/docs/tests for plugin authors, and migration of display/history/session-corpus/dreaming/compaction/retain consumers from hardcoded tag lists to the registry plus conservative built-in defaults.

## Risks / Open Questions

- Closing as clawhub would lose a valid plugin API gap; third-party plugins cannot make downstream core consumers discover new injected tags without a core-readable contract.
