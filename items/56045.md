## Summary

Keep open. Current `main` still has a plausible path where a `sessionTarget: "main"` cron job enqueues in-memory system events, `cron remove` only deletes the persisted job record, and queued events can continue to drive later heartbeat deliveries even when `cron list` shows zero jobs. I did not find a main-branch fix or regression coverage for clearing pending system-event queue state on job removal.

## Evidence

- **Main-session cron jobs enqueue a separate system-event queue before waking heartbeat delivery:** `executeMainSessionCronJob` pushes the cron payload into the session-scoped system-event queue with `contextKey: cron:<jobId>` and then triggers heartbeat processing. That means delivery state is decoupled from the persisted cron job list.
  - file: [src/cron/service/timer.ts:1206](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cron/service/timer.ts#L1206)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '1188,1278p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Job removal only updates the cron store:** `remove()` filters the job out of `state.store.jobs`, persists, re-arms the timer, and emits `removed`. It does not clear any pending system events or heartbeat work for that job.
  - file: [src/cron/service/ops.ts:396](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cron/service/ops.ts#L396)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **System events are held in a global in-memory queue outside the cron store:** `system-events.ts` defines a global singleton queue for session-scoped events. Entries remain until consumed/drained; they are not tied to cron job existence in the store.
  - file: [src/infra/system-events.ts:1](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/system-events.ts#L1)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Queued system events are only cleared on explicit consume/drain paths:** `enqueueSystemEvent()` appends to the queue, while `drainSystemEventEntries()` clears it. There is no removal hook from cron job deletion into this queue management.
  - file: [src/infra/system-events.ts:90](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/system-events.ts#L90)

That leaves pending cron events available for later retries even with no jobs left.
