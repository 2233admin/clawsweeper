## Summary

Keep #72368 open. Current main still only strips the literal `ollama/` prefix in the Ollama chat and embedding paths, while the linked issue/PR describe custom Ollama-compatible provider refs such as `ollama-spark/<model>`. The PR is a plausible focused fix for a still-present gap, though it needs maintainer review before landing.

## Best Possible Solution

Keep the PR open and either merge it after review or adjust it into the maintainer-preferred fix: normalize Ollama wire model IDs for custom Ollama-compatible provider prefixes in chat and embeddings, add regression coverage for `ollama-<key>/<model>` plus a registry namespace counterexample such as `library/llama3`, and fix the changelog attribution before landing.
