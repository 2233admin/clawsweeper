## Summary

Keep open. Current main has several narrower display/history scrubs for runtime-only context, heartbeat prompts, and silent assistant entries, but it does not implement the event-level audience classification or backend delivery-routing primitive requested by #69492. The provided issue discussion also shows PR #72201 is only an open, deliberately tight awareness-event slice, while other tactical heartbeat/exec slices remain separate.

## Best Possible Solution

Keep #69492 open as the canonical maintainer discussion for system-event-shape audience/routing. The best path is to review the scoped PRs (#72201, #72218, #72253) on their merits, then either implement a small SystemEvent/exec/heartbeat audience contract with tests across chat.history, sessions history, and channel delivery, or explicitly replace this umbrella with a maintainer-approved narrower canonical issue.
