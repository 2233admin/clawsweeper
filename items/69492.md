## Summary

Keep open. Current `main` still lacks the per-event audience/classification mechanism this issue proposes, still enqueues exec completion summaries as generic system events, still relays them based on channel visibility alone, and only hides heartbeat-specific transcript noise in visible history. A post-issue fix narrowed one cron/manual-cancel symptom, but the broader system-event leakage class is not implemented away.

## Evidence

- **SystemEvent shape still has no audience field:** `SystemEvent` only carries `text`, `ts`, `contextKey`, `deliveryContext`, and `trusted`. `SystemEventOptions` also has no audience/classification input, so current main has no event-level primitive for internal-vs-user-facing routing.
