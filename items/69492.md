## Summary

Keep #69492 open. Current main does not implement the proposed system-event audience primitive, does not carry completion audience through exec sessions, and still decides user relay from routing/visibility rather than per-event intent. The linked #72201 discussion is useful, but it is explicitly one slice and remains open in the provided GitHub context, so it does not supersede the canonical issue yet.

## Best Possible Solution

Keep #69492 open as the canonical maintainer discussion for system-event audience/visibility until a shipped change either adds a generic `SystemEvent` audience/internal-context primitive with producer and consumer tests, or maintainers explicitly decide to close the broad abstraction in favor of narrower tracked slice fixes. Any implementation should cover the exec, cron, heartbeat, transcript/history, and channel-delivery boundaries together, or clearly document which reports remain covered by separate issues.
