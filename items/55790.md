## Summary

Current `main` still has the subagent-specific thinking resolver wired to `subagents.thinking` only, while docs say spawned subagents inherit caller thinking unless overridden. The reported bug is still present in code and has not been fixed on `main`.

## Evidence

- **Docs still promise caller inheritance:** The subagents docs say default thinking for `sessions_spawn` inherits the caller unless `agents.defaults.subagents.thinking` or `agents.list[].subagents.thinking` is set, with explicit `sessions_spawn.thinking` winning.
  - file: [docs/tools/subagents.md:85](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/docs/tools/subagents.md#L85)
  - command: `nl -ba docs/tools/subagents.md | sed -n '80,90p'`

- **Current resolver ignores `thinkingDefault` and caller inheritance:** `resolveSubagentThinkingOverride` only reads `targetAgentConfig.subagents.thinking` and `cfg.agents.defaults.subagents.thinking`. It does not consult target/global `thinkingDefault`, and if neither subagent override exists it returns no thinking override at all.
