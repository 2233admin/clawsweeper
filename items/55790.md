## Summary

Keep open. Current `main` still appears to violate the documented `sessions_spawn` thinking behavior: docs say child thinking inherits the caller unless `agents.defaults.subagents.thinking` or `agents.list[].subagents.thinking` is set, but the subagent resolver only checks those two config fields plus an explicit tool override. When none are present, the spawn path passes no thinking override and the child run falls back to the generic model default resolver instead of caller inheritance. I found no evidence on `main` that this has been fixed.
