## Summary

Current `main` does not implement per-finding security-audit acknowledgement or suppression. The CLI still only exposes `--deep`, `--token`, `--password`, `--fix`, and `--json`; the audit report schema has no field for muted/acknowledged findings; and the docs still describe review/remediation-only handling for checks such as `plugins.code_safety` and `gateway.trusted_proxies_missing`. This issue should remain open.
