## Summary

Keep open. Current main has partial mitigations for the CLI timeout path, but the central 2026.4.8 gateway/RPC-stall umbrella is not proven fixed, a current-release gateway CPU/RPC-stall report remains open, and the same-author Discord voice report remains open with current code still preserving the fixed 15s join-ready timeout and abort/destroy behavior.

## Best Possible Solution

Keep this issue open as the umbrella for the 2026.4.8 gateway CPU/RPC-stall and downstream CLI/Messenger symptoms. Route the Discord voice portion through #63098 with a Discord plugin fix such as a configurable/longer Ready timeout and bounded retry behavior. Re-test the gateway CPU/RPC-stall on current main/latest release using diagnostics export and either fix the gateway starvation path here or explicitly split the remaining current-release work to #72338 before closing this issue.
