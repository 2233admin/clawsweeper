## Summary

Keep open. Current `main` still keeps the Discord component/modal registry only in process memory, so the restart-survival behavior in PR #66241 is not implemented. The PR discussion also raises real correctness questions around persisted auth/routing fields and state-dir handling, so this is not a safe conservative close.
