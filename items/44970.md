## Summary

Keep open. Current `main` still computes a single `fallbackConfigured` snapshot from `params.config` and still gates prompt-side and auth-profile failover escalation on that boolean, so the PR's stale-config/followup bug is not clearly implemented on `main`. The assistant-side path has since been refactored separately, which means the PR likely needs a rebase/adaptation rather than closure.

## Evidence

If a followup run holds an older config snapshot, later failover decisions still use that stale boolean.
