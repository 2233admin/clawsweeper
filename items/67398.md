## Summary

Keep this PR open. Current main still has the focused heartbeat outbound-session routing gap the PR targets: isolated heartbeat agent execution uses the `:heartbeat` `runSessionKey`, but outbound delivery metadata is still built from the original base `sessionKey`. The PR is not obsolete, and the discussion contains a concrete root-cause analysis plus a targeted test plan. Maintainer review should keep the useful one-line direction while checking the transcript-mirror claim and policy-key behavior.

## Best Possible Solution

Keep this PR open for maintainer review. The likely implementation direction is to route heartbeat outbound delivery metadata to the isolated `runSessionKey`, preserve the base session as `policySessionKey` if delivery policy should still be resolved from the main session, and land a regression test that asserts `deliverOutboundPayloads(...).session.key` is the stable `:heartbeat` key for both fresh-base and wake re-entry paths. If transcript mirroring is part of the intended user-visible fix, add or adjust the explicit mirror path instead of relying only on `session.key`.
