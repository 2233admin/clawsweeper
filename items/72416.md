## Summary

Keep PR #72416 open. Current main still implements `openclaw.install.minHostVersion` as a stable-only `>=x.y.z` floor, and that helper is used by both plugin install and manifest-registry loading. The PR is a focused core/plugin compatibility fix with matching docs and tests in the supplied diff, and it is not already implemented or superseded on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or adapt the focused core fix in `src/plugins/min-host-version.ts` to validate `>=<semver>` floors with the existing `semver.runtime.ts` wrapper, compare with `includePrerelease: true`, and keep the supplied regression tests plus docs updates for plugin authors.
