## Summary

Keep open. Current `main` fixes part of the report, but not the full issue: `writeRestartSentinel` now uses the async atomic helper and `saveJsonFile` now does temp-write + fsync + rename + dir sync, while shell profile installation still writes directly with `fs.writeFile`, and exec approvals still use a narrower temp+rename path without the same durability steps as the repo's atomic helpers.

## Evidence

- **Restart sentinel is now atomic:** `writeRestartSentinel` calls `writeJsonAtomic(..., { trailingNewline: true, ensureDirMode: 0o700 })`, covering the async restart-sentinel path called out in the issue.
  - file: [src/infra/restart-sentinel.ts:80](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/infra/restart-sentinel.ts#L80)
  - command: `nl -ba src/infra/restart-sentinel.ts | sed -n '1,220p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **saveJsonFile is now crash-safer:** `saveJsonFile` resolves symlinks, writes to a temp file, fsyncs the temp file, renames it into place, chmods, and fsyncs the parent directory. That addresses the old direct `writeFileSync` concern for this helper.
  - file: [src/infra/json-file.ts:109](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/infra/json-file.ts#L109)
  - command: `nl -ba src/infra/json-file.ts | sed -n '1,220p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Shell profile writes are still non-atomic:** `installCompletion` still creates a missing shell profile with `fs.writeFile(profilePath, "", "utf-8")` and updates it with `fs.writeFile(profilePath, update.next, "utf-8")` instead of `writeTextAtomic`. This is one of the exact remaining paths described in the issue.
  - file: [src/cli/completion-runtime.ts:241](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cli/completion-runtime.ts#L241)
  - command: `nl -ba src/cli/completion-runtime.ts | sed -n '190,270p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Exec approvals still do not use the repo atomic helper path:** `writeExecApprovalsRaw` now does temp file + rename + chmod, but it does not fsync the temp file or parent directory like `src/infra/json-files.ts` / `src/infra/json-file.ts`. The issue asks for crash-safe atomic writes on this path too, so the full request is not yet implemented on `main`.
  - file: [src/infra/exec-approvals.ts:487](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/infra/exec-approvals.ts#L487)
  - command: `nl -ba src/infra/exec-approvals.ts | sed -n '460,520p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)

## Risks / Open Questions

- Closing now would lose the remaining shell-profile corruption case in `src/cli/completion-runtime.ts`.
- The exec-approvals path still uses a bespoke writer with weaker durability semantics than the repo's established atomic helpers, so the original crash-safety concern is only partially resolved.
