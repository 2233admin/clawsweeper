## Summary

Keep open. Current main still generates zsh leaf-command completions from option specs only, so positional arguments such as `openclaw wiki ingest <path>` are not implemented on main. PR #69305 is not obsolete; it contains useful targeted work and tests, though the path-name heuristic still needs maintainer review because some `<path>` arguments are non-filesystem config paths.

## Best Possible Solution

Keep #69305 open as the likely useful implementation path for #69293. Maintainers should either refine its positional-action mapping to avoid non-filesystem `<path>` false positives, or adopt the same core approach with explicit command/argument metadata for filesystem positionals, then add regression coverage in `src/cli/completion-cli.test.ts` for `wiki ingest`, optional/variadic arguments, and at least one non-file `<path>` command.
