## Summary

Current main still omits Commander positional arguments from generated zsh leaf-command completions, while the memory-wiki `wiki ingest` command still declares a real `<path>` positional. This PR is not implemented or superseded on main; it should stay open, though the filesystem-action heuristic needs maintainer review before merge.

## Best Possible Solution

Keep the PR open and land a refined version: teach the zsh generator to emit positional specs for leaf Commander arguments, add regression tests for file, directory, optional, variadic, and neutral/non-filesystem arguments, and avoid treating every `<path>` argument as a filesystem path without stronger metadata or an explicit allowlist.
