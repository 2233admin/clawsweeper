## Summary

Keep open. Current main still has the reported zsh completion gap: leaf command completion emits option specs only, while positional Commander arguments are ignored. The PR remains relevant to #69293, but it should address the remaining review risk that a name-only `<path>` heuristic can mis-complete non-filesystem arguments such as config dot paths.

## Best Possible Solution

Keep the PR open and ask for a targeted refinement: emit zsh positional specs for leaf commands, preserve required/optional semantics, add regression tests for file, directory, variadic, and optional positionals, and avoid forcing `_files` for non-filesystem `<path>` arguments such as config dot paths. A command-level completion metadata helper or conservative allowlist would be preferable to broad name-only inference.
