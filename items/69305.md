## Summary

Keep open. Current main still generates zsh leaf-command completions from option specs only, so Commander positional arguments like `wiki ingest <path>` are not exported. The PR addresses a real remaining bug, and the related duplicate PRs in the provided PRtags context are closed/unmerged rather than canonical replacements.

## Best Possible Solution

Keep the PR open and continue review. The right product direction is to add positional specs to the core zsh completion generator, preserve required/optional/variadic Commander semantics, escape zsh labels correctly, and choose `_files`/directory completion only when the command argument domain is actually filesystem-backed. Add focused regression tests in `src/cli/completion-cli.test.ts` for `wiki ingest <path>`, optional/variadic positionals, and at least one non-filesystem `<path>` command.
