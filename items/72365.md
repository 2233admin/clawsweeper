## Summary

Keep this issue open. Current main still has the Control UI-visible `tools.effective` request path running a synchronous effective-tool inventory resolver inside the gateway request handler, and there is an open, unmerged implementation candidate (#72558) that references this issue with closing syntax. That PR should be reviewed as a candidate fix rather than using it as a reason to close the issue before merge.

## Best Possible Solution

Keep this issue open. Review #72558 as the current implementation candidate, but require an event-loop-safe fix before closure: make effective-tool inventory resolution incremental with internal yield points, cache/debounce the expensive computation, or move CPU-heavy work off the gateway event loop, then add focused regression coverage proving concurrent gateway/control UI requests remain responsive while `tools.effective` is resolving.
