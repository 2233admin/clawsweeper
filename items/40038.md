## Summary

Keep #40038 open. Current main still uses Ollama's old `/api/embeddings` + `prompt` + `embedding` contract in the active bundled Ollama memory embedding provider, while upstream Ollama docs mark `/api/embeddings` as superseded by `/api/embed`. The PR needs to be rebased/ported from the old `src/memory` layout to the current `extensions/ollama` plugin path, but the underlying bug remains real and useful to fix.

## Best Possible Solution

Keep #40038 open as a focused bug-fix candidate for #39983, but require a rebase/port onto the current bundled Ollama plugin implementation. The final change should update `extensions/ollama/src/embedding-provider.ts` and `extensions/ollama/src/embedding-provider.test.ts` from `/api/embeddings` + `prompt` + `embedding` to the current `/api/embed` + `input` + `embeddings[0]` contract, with explicit maintainer judgment on fallback behavior for older Ollama installations.
