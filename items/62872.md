## Summary

Keep this issue open. Current main still runs the normal `sessions_send` A2A ping-pong plus announce path for peer and persistent sessions, and the tool still has no per-call ping-pong disable/max-turn option. The merged #69817 work only shipped the parent-owned one-shot ACP child skip; #62814 was closed as a duplicate, so this issue remains the canonical tracker for the persistent-session behavior.

## Best Possible Solution

Keep this issue open as the canonical tracker. The likely fix belongs in `src/agents/tools/sessions-send-tool.ts` and `src/agents/tools/sessions-send-tool.a2a.ts`: make `sessions_send` context-aware for persistent peer-session sends or add an explicit per-call ping-pong max/disable option, preserve the #69817 parent-owned one-shot ACP child skip, and add regression coverage for persistent peer-session duplicate behavior.

## Risks / Open Questions

- The global `session.agentToAgent.maxPingPongTurns=0` workaround can reduce the symptom, but it disables intended A2A chaining everywhere and does not solve persistent peer sends per call or per context.
- Changing this too broadly could break legitimate peer A2A follow-up and announce workflows, so the fix needs regression coverage that separates persistent peer-session sends from the already-shipped parent-owned one-shot ACP child skip.
