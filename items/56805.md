## Summary

Keep open. Current `main` still does not expose `knownLongFlags` in the config schema or fixture normalization path, while the runtime validator still supports and consults that metadata internally.

## Evidence

- **Strict config schema still rejects the field:** `ToolExecSafeBinProfileSchema` is `.strict()` and only defines `minPositional`, `maxPositional`, `allowedValueFlags`, and `deniedFlags`; `knownLongFlags` is absent, so config input with that key would still be rejected.
  - file: [src/config/zod-schema.agent-runtime.ts:447](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/config/zod-schema.agent-runtime.ts#L447)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '445,460p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Fixture type still cannot carry `knownLongFlags`:** `SafeBinProfileFixture` currently exposes only the same four fields, so the config/runtime normalization layer has no typed path for user-supplied `knownLongFlags`.
  - file: [src/infra/exec-safe-bin-policy-profiles.ts:14](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/infra/exec-safe-bin-policy-profiles.ts#L14)

  - file: [src/infra/exec-safe-bin-policy-profiles.ts:251](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/infra/exec-safe-bin-policy-profiles.ts#L251)
