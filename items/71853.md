## Summary

Keep open. Current main appears to have fixed the google-vertex ADC marker leak in the bundled pi-ai provider, but the Node 25/gaxios side of the report is still not clearly implemented for google-vertex ADC. The pinned @google/genai ADC path still creates GoogleAuth without an OpenClaw-supplied transporter, while pinned gaxios 7.1.4 still falls back to dynamic node-fetch unless fetchImplementation is provided. The existing scoped gaxios native-fetch transport is Google Chat-specific, not google-vertex.

## Best Possible Solution

Keep the issue open and fix the remaining google-vertex ADC transport path, ideally by adding a google-vertex-specific runtime auth/transport seam that supplies native fetch (or otherwise avoids gaxios's node-fetch fallback) without regressing the already-correct pi-ai 0.70.2 behavior that ignores the ADC marker and sets vertexai/project/location. Add a targeted regression test around google-vertex ADC ensuring the marker is not sent as x-goog-api-key and that the Google auth transport does not use gaxios's default node-fetch path.
