## Summary

Keep open. Current main does not add context window size, effective utilization, or model maxTokens to the llm_output hook payload. The hook type, embedded runner emission path, CLI runner emission path, docs, and hook tests all still expose only provider/model identifiers plus assistant output and usage token counts. Related #64268 and #64327 are useful context but do not supersede this issue: #64268 tracks per-call emission timing, while #64327 tracks effective contextTokens once model limit metadata is available to hooks.

## Best Possible Solution

Keep this issue open and implement a public plugin hook contract change: add optional model limit metadata to llm_output, probably including native contextWindow/maxTokens and a resolved effective context limit when available, then populate it from the resolved runtime model in embedded and CLI paths. Document the new fields in plugin hook docs and add contract tests that assert the fields are present when model metadata is known. Treat #64268 as separate per-call timing work and #64327 as the follow-up/effective-limit semantics to align with, not as a replacement for this issue.
