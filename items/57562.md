## Summary

Keep open. Current `main` clearly broadened Feishu account-schema support, but I did not find strong evidence that issue #57562's specific upgrade/doctor validation failure is fixed, and the report still lacks the concrete config needed to verify the exact bad property.

## Evidence

- **Feishu account schema on current main is broad:** `FeishuAccountConfigSchema` now accepts `name`, credentials, `connectionMode`, `webhookPath`, and the shared Feishu channel config shape at account scope, so some older `accounts.main` additional-property failures would no longer happen.
  - file: [extensions/feishu/src/config-schema.ts:191](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/feishu/src/config-schema.ts#L191)
- **Current tests confirm account-scoped fields parse:** Schema tests explicitly accept account-scoped `replyInThread`, `typingIndicator`, `resolveSenderNames`, and `actions`, showing the account object is intentionally much less restrictive on current `main`.
  - file: [extensions/feishu/src/config-schema.test.ts:192](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/feishu/src/config-schema.test.ts#L192)
- **Backward-compat account resolution exists, but not as proof of this fix:** `resolveFeishuAccount` still supports top-level Feishu credentials together with `defaultAccount`, which helps compatibility for some upgraded configs, but this does not prove the reporter's exact `accounts.main` validation error is resolved.
  - file: [extensions/feishu/src/accounts.test.ts:315](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/feishu/src/accounts.test.ts#L315)
- **Doctor migration remains generic, not Feishu-specific:** The generic single-account promotion only moves a fixed common key list and then seeds `accounts.default`; that list does not include Feishu credential keys like `appId`, `appSecret`, `encryptKey`, or `verificationToken`, so I could not verify a targeted doctor fix for this upgrade path.
  - file: [src/channels/plugins/setup-promotion-helpers.ts:11](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/channels/plugins/setup-promotion-helpers.ts#L11)
- **No explicit Feishu doctor migration surfaced:** Searching `extensions/feishu/src` for doctor/config-migration hooks did not reveal a Feishu-specific compatibility normalizer that would directly explain this issue as fixed.
  - command: `rg -n "doctor|normalizeCompatibilityConfig|compatibility|migration|migrate" extensions/feishu/src -g '*.ts'`
