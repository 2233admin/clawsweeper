## Summary

Keep open. Current `main` still creates a fresh subagent session key for each native `sessions_spawn` call, still gates `thread: true` behind a channel `subagent_spawning` hook, and the MSTeams plugin still does not register that hook. The newer ACP-only `resumeSessionId` path is related but does not implement the requested native subagent/session reuse behavior for Teams.
