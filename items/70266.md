## Summary

Keep #70266 open. Current main still renders the native macOS Talk Mode overlay as the generic SwiftUI orb, and the existing assistant avatar work is limited to Control UI/web chat identity surfaces. The reporter's follow-up correctly distinguishes this issue from the closed #22199 web/control UI identity item.

## Best Possible Solution

Keep the issue open. The best implementation path is to expose the resolved assistant avatar for the active Talk Mode agent through an appropriate gateway/native contract, parse and cache it in the macOS app, render it as a circular image inside TalkOverlayView while preserving the existing rings, pulse, pause, and thinking-state visuals, fall back to the current orb when unresolved, and add protocol/parser plus macOS overlay coverage for the new field.
