## Summary

Current main has useful low-level SecretRef batching, but the product-level request is not implemented. The resolver dedupes exact refs and batches by `source/provider`, while the public external-backend examples and configure flow still model direct CLI commands with backend paths baked into provider args and `id: "value"`. The related #17311 is open but not a clear canonical replacement for generic external-vault batching/memoization across backends.

## Best Possible Solution

Keep #72278 open. The maintainer path should either add a first-class backend-aware external SecretRef provider contract, or revise the supported exec-provider pattern/docs/configure flow so one configured provider represents one backend/account/context and SecretRef ids carry backend paths. Add pass-scoped memoization expectations and regression coverage proving shared backend container/item/entity lookups are reused without disk caching or plaintext logging. Coordinate with #17311 and #56217, but do not close this until one of those clearly owns or ships the generic external-vault batching fix.
