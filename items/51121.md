## Summary

Current `main` still contains the reported un-cleared 500ms shutdown timer in `IMessageRpcClient.stop()`, so this PR is not already implemented. But the proposed patch is not strong close evidence either: on current `main`, `closed` is created once in the constructor and never reset on `start()`, so clearing the fallback timer would change behavior on repeated `start()`/`stop()` cycles exactly as the PR review warned. This needs a fresh maintainer fix, not closure.

## Evidence

## Risks / Open Questions

- Closing this PR would leave a plausible shutdown timer leak still present on `main` in `extensions/imessage/src/client.ts:132`.
