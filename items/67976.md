## Summary

Current `main` still does not recognize the reported Coding Plan model as an image-capable Volcengine model, and repo docs/catalog still describe `volcengine-plan` as text-only. Upstream Volcengine material now points to `doubao-seed-2.0-pro` on the Coding Plan endpoint, so this remains a plausible provider/catalog gap rather than a safe close.

## Evidence

- **Bundled Volcengine coding catalog is text-only and omits Doubao 2.0 models:** `DOUBAO_CODING_MODEL_CATALOG` only contains `ark-code-latest`, `doubao-seed-code`, `glm-4.7`, `kimi-k2-thinking`, `kimi-k2.5`, and `doubao-seed-code-preview-251028`, each with `input: ["text"]`. There is no `doubao-seed-2.0-pro` or `doubao-seed-2.0-code` entry in the bundled catalog.
  - file: [extensions/volcengine/models.ts:22](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/extensions/volcengine/models.ts#L22)
  - command: `nl -ba extensions/volcengine/models.ts | sed -n '1,220p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **User-facing Volcengine docs still advertise `volcengine-plan` as text-only:** The provider docs list all `volcengine-plan/*` models with `Input` = `text` only and do not mention the newer Coding Plan names from the report.
  - file: [docs/providers/volcengine.md:74](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/docs/providers/volcengine.md#L74)

That conflicts with the current bundled OpenClaw catalog/docs and makes the report plausible.
