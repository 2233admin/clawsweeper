## Summary

Keep this PR open. Current main still routes iMessage inbound text through raw `BodyForAgent`/`RawBody`/`CommandBody`, so the hardening topic is not implemented. The submitted patch is security-relevant but not merge-ready because it wraps only the legacy `Body` envelope produced by `formatInboundEnvelope(...)`; that does not change the primary prompt-bound body path or add `UntrustedContext` coverage/tests.

## Best Possible Solution

Revise this PR or replace it with a maintainer-reviewed iMessage hardening patch. The implementation should make an explicit product/security decision about primary DM text, protect the actual model-bound path through `UntrustedContext` or an equivalent prompt-bound field, keep command parsing raw and deterministic via `CommandBody`/`RawBody`, and add focused iMessage tests for prompt assembly, commands, reply context, history, and echo behavior.

## Risks / Open Questions

- Adding randomized external-content markers to legacy envelope/history text may affect deterministic history, echo, or rendering behavior without protecting the actual prompt-bound path.
- No iMessage regression coverage is included for prompt content, command parsing, reply context, history, or echo behavior.
