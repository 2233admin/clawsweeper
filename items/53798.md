## Summary

Keep open. Current main still routes Feishu `.opus`/`.ogg` message-tool media to the native audio path, but `sendMediaFeishu` uploads the audio without computing or passing `duration`, so the missing Feishu UI duration remains a real bundled Feishu plugin bug.

## Best Possible Solution

Keep this issue open and fix it in the bundled Feishu plugin media path. Compute a finite audio duration for native `.opus` / `.ogg` sends and for `audioAsVoice` transcode outputs, convert it to Feishu's expected upload units, pass it to `uploadFileFeishu`, and add focused Feishu media regression tests proving `im.file.create` receives `duration` for direct Opus/Ogg audio and voice-intent transcodes.
