## Summary

Keep open. Current `main` does not implement the requested tightening: `maskApiKey()` still exposes both prefix and suffix characters, and that helper is still used in `/models list` and auth-label output. The PR addresses a real issue, but the proposed `> 4 => first 4 chars` rule would regress masking for 5-6 character secrets by revealing more of the secret than `main` does today.

## Evidence

- **Current main still exposes suffix characters:** `maskApiKey()` on current `main` returns `first1...last1` for values of length <= 6, `first2...last2` for values <= 16, and `first8...last8` otherwise. That means the requested behavior is not already implemented on `main`.

- Merging the PR unchanged would make 5-6 character secrets more exposed than they are on `main`, so it should not be closed as already-correct cleanup.
