## Summary

Keep open. Current main still does not expose the requested `subagent:ended`, `agent:turnComplete`, or tool-call-threshold events through the workspace/internal `HOOK.md` system. The Plugin SDK has adjacent lifecycle hooks and even a bundled skill-workshop plugin using `agent_end`, but that is not the same as the requested workspace hook API, and the existing issue discussion already called out this product/API boundary rather than a stale reproduction problem.

## Best Possible Solution

Keep this issue open for maintainer API judgment. The concrete path is either to add a narrow internal-hook bridge for `subagent:ended` plus well-defined turn/tool-count lifecycle events, or to make an explicit product decision that these workflows should use Plugin SDK hooks such as `subagent_ended`, `after_tool_call`, and `agent_end`, with docs updated so workspace hook authors are not left guessing.
