## Summary

Current `main` still writes session transcript headers with only the basic fields (`type`, `version`, `id`, `timestamp`, `cwd`) across the active creation/reset paths. The requested `sessionKey`/`agentId`/`channel`/`kind` metadata is not persisted in those headers, and current gateway code still falls back to store/path-based lookup when it needs a session key from a transcript file.
