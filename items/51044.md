## Summary

Current `main` does not show this PR as already implemented or obsolete. Ollama synthetic auth is now plugin-owned, but the bundled Ollama plugin only synthesizes auth for meaningful explicit `models.providers.ollama` config, while the public docs still describe implicit/configless Ollama usage as needing `OLLAMA_API_KEY` or an auth profile. The PR discussion also already identified missing tests and broader auth-surface inconsistencies, so conservative cleanup should leave this PR open rather than auto-close it.
