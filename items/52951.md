## Summary

Keep PR #52951 open. Current main still has only the binary `tools.fs.workspaceOnly` filesystem policy and does not implement `tools.fs.roots`, per-root `ro`/`rw` modes, root-scoped host writes/apply_patch/media handling, or the related config schema/docs. The PR is broad and still has active review concerns, but the requested capability is a real core security/config surface, not ClawHub-only and not shipped or superseded.

## Best Possible Solution

Keep the item open, but require maintainer review before merge. The best path is to narrow or split the PR around the core config/policy contract, keep `tools.fs.roots` in core if accepted, preserve sandbox/workspaceOnly semantics, and land only after the root-scoped FS/media/apply_patch behavior has focused tests and the unrelated generated/import-order churn is removed or clearly justified.
