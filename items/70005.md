## Summary

Keep #70005 open. Current main has useful mitigations for the reported failure mode, including filtering heartbeat/system/cron transcript noise and blacklisting chat-scaffolding tags such as `assistant` from REM theme output, but the core request is broader: REM/dreaming material selection should prefer semantically important decisions, preferences, conclusions, or unusual events over frequency-driven signals. Current main still ranks REM themes by concept-tag counts and deep promotion still includes a frequency component, and there is no documented `dreaming.promptStyle` or semantic-density material-selection control.

## Best Possible Solution

Keep this issue open and treat it as memory-core product work. The likely fix belongs in `extensions/memory-core/src/dreaming-phases.ts` and possibly `extensions/memory-core/src/short-term-promotion.ts`: add a semantic-density/material-importance filter before REM narrative input, preserve the existing heartbeat/metadata/scaffolding filters, and add regression coverage using examples like one-off game decisions/preferences versus high-frequency role/scaffolding terms. If a public strategy knob such as `dreaming.promptStyle` is added, update `docs/reference/memory-config.md` and `docs/concepts/dreaming.md`.
