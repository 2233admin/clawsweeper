## Summary

Keep #70005 open. Current main has useful mitigations for the noisy `assistant`/`HEARTBEAT_OK` example, but the scheduled REM and Dream Diary path still selects material from lexical concept-tag counts and recent short-term traces rather than a semantic importance filter for decisions, preferences, conclusions, or unusual events. The author’s follow-up explicitly says prompt changes were insufficient and that the remaining bottleneck is pre-narrative material selection, which still matches the code shape on current main.

## Best Possible Solution

Keep the issue open and fix it in `memory-core` by adding a semantic material-selection layer before scheduled REM and Dream Diary generation. The implementation should rank session/daily chunks for durable user decisions, preferences, conclusions, explicit outcomes, and unusual events; demote role/scaffolding/heartbeat/current-state chatter; feed the ranked grounded snippets and themes into `previewRemDreaming` and the diary prompt; and add regression tests where high-frequency role/scaffolding tokens lose to low-frequency meaningful decisions. If maintainers want a user knob, validate and document it under `plugins.entries.memory-core.config.dreaming`.
