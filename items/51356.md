## Summary

Keep open. Current `main` does not implement this PR's core changes: model config still does not accept `models.providers.*.models[].options`, the Ollama compat `num_ctx` path still derives from resolved `contextWindow` rather than model `options`, and current assistant-text handling strips raw `<tool_call>` XML instead of recovering and dispatching tool calls. That is not strong enough for an `implemented_on_main` or `cannot_reproduce` close.

## Evidence

- **Model config type still has no `options` field:** `ModelDefinitionConfig` includes `contextWindow`, `contextTokens`, `maxTokens`, `headers`, and `compat`, but no `options` property.
  - file: [src/config/types.models.ts:77](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/config/types.models.ts#L77)

  - file: [src/config/zod-schema.core.ts:338](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/config/zod-schema.core.ts#L338)
