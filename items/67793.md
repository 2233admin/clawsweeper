## Summary

Not safe to close. Current main clearly documents that `messages.queue.collect` only coalesces followups while a run is already active, and Feishu still routes idle-arrival batching through `messages.inbound`. But the reported Feishu idle-DM repro remains plausible: the current Feishu debounce key includes `root_id`, and the issue discussion never ruled out per-message `root_id`/thread variance or another Feishu-specific bypass. There is no direct regression test for the exact `idle DM + three plain text messages within debounce window` scenario on current main.
