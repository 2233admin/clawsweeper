## Summary

Keep open. Current main still does not implement either acceptable outcome from #71710: the `openclaw agent` CLI command has no top-level SIGTERM/SIGINT cancellation and bounded drain path for gateway or embedded runs, and the public agent/cron docs do not warn script/cron users to use `timeout -k` or equivalent escalation. The related #66399 remains adjacent process-supervisor work, not a complete tracker for the parent `openclaw-agent` cron process leak.

## Best Possible Solution

Keep #71710 open and implement a top-level `openclaw agent` signal lifecycle: install SIGTERM/SIGINT handlers at the CLI command boundary, abort the in-flight gateway request or embedded run through shared AbortSignal plumbing, stop progress and GatewayClient resources, flush feasible logs/transcript state, and force exit after a bounded grace window. Until that ships, document `timeout -k <delay>` or equivalent supervisor escalation for external cron use. Keep #66399 separate unless maintainers intentionally broaden it to cover the parent agent CLI process as well.
