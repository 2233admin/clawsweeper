## Summary

Keep open. Current main still has the exact call-shape mismatch reported: top-level `openclaw health` calls the generic gateway helper without CLI identity, while `openclaw gateway health` passes `clientName: cli` and `mode: cli`. Gateway auth/locality logic can distinguish those identities before the `health` RPC runs, so the reporter's workaround is consistent with current source rather than obsolete behavior.

## Best Possible Solution

Keep #72079 open. Patch the top-level health path to use the same CLI caller identity as `openclaw gateway health` (for example by passing `GATEWAY_CLIENT_NAMES.CLI` and `GATEWAY_CLIENT_MODES.CLI`, or by routing through a shared CLI gateway-call helper), then add a focused regression test asserting `healthCommand({ json: true })` sends the health RPC with CLI identity. Consider auditing the shared status health probe separately because it currently uses the same generic call shape.
