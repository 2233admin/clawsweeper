## Summary

Current `main` does not contain this proxy fix for memory remote embedding fetches, and the PR still has a credible unresolved regression for local Ollama / `NO_PROXY` cases. The item is still actionable, but it is not safe to conservatively close.

## Evidence

- **Memory remote helper still uses strict guarded fetch on main:** The current exported memory-host helper calls `fetchWithSsrFGuard` without `mode: "trusted_env_proxy"` or any auto-upgrade logic, so env proxy vars are not honored by this path today.
  - file: [src/memory-host-sdk/host/remote-http.ts:13](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/memory-host-sdk/host/remote-http.ts#L13)
  - command: `nl -ba src/memory-host-sdk/host/remote-http.ts | sed -n '1,24p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Env proxy routing only happens when trusted mode is explicitly enabled:** `fetchWithSsrFGuard` only creates `EnvHttpProxyAgent` when `mode === TRUSTED_ENV_PROXY` and proxy env is configured; otherwise it stays in strict pinned-DNS mode. That means the claimed behavior change is not already present on `main`.
  - file: [src/infra/net/fetch-guard.ts:357](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/infra/net/fetch-guard.ts#L357)
  - command: `nl -ba src/infra/net/fetch-guard.ts | sed -n '346,366p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Current main still has the loopback Ollama case the review flagged:** Ollama embeddings default to `http://127.0.0.1:11434`, and the embedding provider uses the same remote HTTP pattern. An unconditional switch to trusted env proxy mode would plausibly route that loopback traffic through `HTTP_PROXY` when `NO_PROXY` is incomplete, matching the PR review concern.
  - file: [extensions/ollama/src/defaults.ts:1](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/extensions/ollama/src/defaults.ts#L1)
  - command: `nl -ba extensions/ollama/src/defaults.ts | sed -n '1,20p' && nl -ba extensions/ollama/src/embedding-provider.ts | sed -n '118,156p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Main has a newer safe auto-upgrade pattern elsewhere, but memory does not use it:** `src/media-understanding/shared.ts` now auto-upgrades to trusted env proxy mode only when `HTTP_PROXY`/`HTTPS_PROXY` applies and `NO_PROXY` does not match. That code explicitly documents the security/regression edge cases. The memory path has not adopted that safer pattern yet, so this PR still needs design/work, not closure.
  - file: [src/media-understanding/shared.ts:253](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/media-understanding/shared.ts#L253)
