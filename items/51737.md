## Summary

Keep open. Current main still has the overview-picker bug this PR fixes: `resolveConfiguredModels()` only surfaces `agents.defaults.models` entries, so provider-qualified primary/fallback refs outside that alias map still fall through to the synthetic `Current (...)` option instead of remaining selectable. The chat-header side is largely handled by newer qualified-model helpers, but the PR is not fully implemented on `main`.
