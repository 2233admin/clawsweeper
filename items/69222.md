## Summary

Keep #69222 open. Current main only calls the stable local `/api/web_search` endpoint and honors an explicit Ollama Cloud base URL; it does not implement the PR's local experimental fallback, conditional Ollama Cloud fallback after local 404s, cloud/local auth scoping, or the associated docs/test updates. The linked bug #69132 and duplicate PR #69178 are still open, and an unresolved review note on #69222 points to a real mixed-auth edge case the PR should address before it is considered merge-ready.

## Best Possible Solution

Keep the PR or one clearly chosen canonical PR open until maintainers decide the desired product behavior for #69132. The best fix likely belongs in `extensions/ollama/src/web-search-provider.ts` with a small ordered target list, explicit credential-source handling for local versus Ollama Cloud, regression tests for stable/local-experimental/cloud fallback and mixed-auth cases, and aligned docs/changelog updates. #69222 should also address the existing review comment about allowing cloud fallback when both a provider-config local key and env-sourced `OLLAMA_API_KEY` are configured.
