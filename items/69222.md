## Summary

Keep #69222 open. Current main has a narrower Ollama web-search change: it uses `/api/web_search` and honors the configured base URL, but it does not implement the PR’s 404 compatibility chain, legacy local fallback, env-scoped Ollama Cloud fallback, or matching docs/tests. The linked bug #69132 remains open, while related PR #69178 was closed unmerged.

## Best Possible Solution

Keep the PR or its linked bug open. The best path is to update current main with a maintainer-approved Ollama web-search compatibility flow: local `/api/web_search`, legacy local `/api/experimental/web_search` on 404, scoped `OLLAMA_API_KEY` cloud retry when appropriate, regression tests for those paths, and docs/contract metadata that clearly distinguish local host auth from Ollama Cloud auth. Address the mixed provider-config plus env-key review comment before landing.
