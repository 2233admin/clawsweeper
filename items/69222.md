## Summary

Keep #69222 open. Current main only has a partial Ollama web-search fix: it uses `/api/web_search` and honors configured cloud base URLs, but it does not implement the PR's local 404 fallback chain, Ollama Cloud retry, or source-scoped auth behavior. The linked bug #69132 is still open, #69178 was closed unmerged, and #69222 also has an unresolved review finding that should be addressed before it could land.

## Best Possible Solution

Keep #69222 open as the active implementation candidate for the still-open #69132 regression, but require a revision against current main: preserve the already-landed `/api/web_search` and explicit cloud-base behavior, add the missing 404 fallback chain only when appropriate, resolve provider-config and env keys independently so cloud fallback still works in mixed-auth setups, and update focused tests/docs for those exact cases.
