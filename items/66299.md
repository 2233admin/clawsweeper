## Summary

Keep open. Current `main` still has the early return in `resolveAcpTargetSessionKey`, so the reported fallback bug is not implemented on `main`, and the only linked fix PR was closed without merging.

## Evidence

- **Current main still short-circuits on unresolved token:** `resolveAcpTargetSessionKey` on current `main` returns `Unable to resolve session target` when `resolveSessionKeyByToken(token)` is falsy, and only checks `resolveBoundAcpThreadSessionKey` afterward. That matches the issue report rather than a fix.
