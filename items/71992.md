## Summary

Issue #71992 should stay open. Current main still has the Control UI `final` and `aborted` event paths writing `chatMessages` before clearing `chatStream`, which matches the client-side duplicate-render race described in the issue and in the prior closed PR #34164. The provided GitHub context also shows an open fix PR, #72004, that has not landed on main, plus later comments connecting some server-side duplicate transcript cases to the broader open maintainer umbrella #69208.

## Best Possible Solution

Keep #71992 open until PR #72004, or an equivalent minimal patch, lands on main with a regression test proving the `final` and `aborted` paths clear `chatStream` before mutating `chatMessages`. After that lands, any remaining persisted transcript duplication should be tracked or split under the existing maintainer umbrella #69208 rather than closing this concrete Control UI regression prematurely.
