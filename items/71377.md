## Summary

Keep open. The PR adds useful Telegram logging, but current main already logs general Telegram API failures, and the bundled routing change appears unnecessary at best and risky at worst. All current `resolveTelegramConversationRoute` call sites already pass `replyThreadId`, so the extra `resolvedThreadId` fallback does not have a clear demonstrated need; meanwhile current DM code explicitly distinguishes DM thread IDs from forum `resolvedThreadId`, matching the existing review concern about accidental DM binding-key changes.
