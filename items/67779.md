## Summary

Keep open. Current `main` still has a plausible, code-backed path for this exact symptom: the Control UI preserves the `session` URL param as typed, but chat events are ignored unless `payload.sessionKey === state.sessionKey` exactly, while gateway session handling canonicalizes agent session keys to lowercase/canonical form before routing and broadcasting. That means a mixed-case deep link like `agent:DB_Dude:main` can miss live events, yet still show the completed reply after refresh because history lookup resolves the canonical key.
