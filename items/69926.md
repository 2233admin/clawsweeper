## Summary

Keep open. Current main still does not implement WhatsApp per-group `allowFrom`: the WhatsApp group config schema/type/generated metadata only admit `requireMention`, `tools`, `toolsBySender`, and `systemPrompt`, while runtime group sender authorization is still resolved from account/channel-level `groupAllowFrom` with fallback to `allowFrom`. The related #59553 work added per-group prompts, and #69874 tracks multi-account group inheritance separately; neither supersedes this sender-authorization feature.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin/config surface: add optional `allowFrom?: string[]` to `WhatsAppGroupConfig` and the WhatsApp Zod/generated metadata for both root and account `groups`; resolve specific group then wildcard group `allowFrom` against the already-resolved account groups map; treat a defined per-group list as an override, not a merge, falling back to `groupAllowFrom` only when undefined; apply the same effective list in inbound access control and command authorization; add config-schema, account-scope, group-gating, and docs coverage. Coordinate the multi-account inheritance semantics with #69874 rather than closing this as duplicate.
