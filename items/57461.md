## Summary

Keep open. Current main still matches the reported failure mode: `openclaw mcp serve` exposes generic MCP channel tools, but the actual payloads for those tools are returned in `structuredContent` while `content` remains summary/status text. The related fix PR #57576 was closed unmerged, so the issue remains the canonical tracker rather than a duplicate or shipped fix.

## Best Possible Solution

Keep #57461 open and fix the core MCP channel tool result shape. A conservative implementation should preserve `structuredContent` while adding self-sufficient JSON text content for successful channel tool results, ideally through one shared helper in `src/mcp/channel-shared.ts` plus focused tests in `src/mcp/channel-server.test.ts` that call representative tools and assert both `content[0].text` parses to the payload and `structuredContent` remains present.
