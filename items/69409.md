## Summary

Keep #69409 open. Current main has reload modes and restart deferral, but it still does not implement the requested approval-gated restart queue, admin notification, or explicit approve-restart command/API for restart-required direct config edits.

## Best Possible Solution

Keep the issue open. The best product path is a core gateway reload policy under `gateway.reload` that can queue restart-required reload plans, notify configured operator targets, expose a CLI/RPC approval resolver, and then reuse the existing deferral/audit/restart machinery once approved. Existing `hot` or `off` modes can be documented as workarounds, but they do not satisfy the requested approval workflow.
