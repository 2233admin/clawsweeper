## Summary

Keep #69220 open. Current main still combines tagged Gemini reasoning-output selection with native Gemini thinking payloads/parts, so the reported conflict is not fixed. The issue is recent, has concrete reproduction details plus two affected-user confirmations, and the current tests/docs still lock in the disputed tagged-mode behavior.

## Best Possible Solution

Keep the issue open and fix the Google/Gemini reasoning-output contract in source: remove the direct `google-generative-ai` tagged fallback, change the bundled Google `google-gemini` provider hook or registration so native Gemini thinking uses `native` output mode or a safe conditional mode, update docs/tests that currently describe or assert tagged mode, and add regression coverage for native Gemini thought parts plus a tool call before final visible text so the run does not enforce `<final>` and collapse into `payloads=0`.
