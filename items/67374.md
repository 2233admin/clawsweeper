## Summary

Keep open. Current `main` does not already implement this fix, and the exact one-line change proposed in PR #67374 is not a conservative cleanup candidate because it can silently drop a required credential update while also papering over a `WizardPrompter.text()` contract mismatch.

## Evidence

- **Current main still trims `rawValue` directly and would silently skip a required credential if it became empty/undefined after fallback:** `src/channels/plugins/setup-wizard.ts` still does `const trimmedValue = rawValue.trim();`. The downstream path deletes `credentialValues[textInput.inputKey]` and `continue`s when `normalizedValue` is falsy, with no required-field error at that stage.
  - file: [src/channels/plugins/setup-wizard.ts:458](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/channels/plugins/setup-wizard.ts#L458)
  - command: `nl -ba src/channels/plugins/setup-wizard.ts | sed -n '430,490p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Wizard prompter contract says text prompts return strings:** The shared interface is `text: (params: WizardTextParams) => Promise<string>;`, so the PR's optional chaining is compensating for a runtime case that is outside the current declared contract.
  - file: [src/wizard/prompts.ts:44](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/wizard/prompts.ts#L44)
  - command: `nl -ba src/wizard/prompts.ts | sed -n '1,120p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Shipped prompter implementations already normalize cancellation/empty answers instead of returning undefined:** `createClackPrompter().text()` wraps the clack result in `guardCancel(...)`, and `WizardSessionPrompter.text()` coerces `null`/`undefined` to `""` before validation and return. That makes the reported `undefined` path plausible as a real edge case somewhere, but not something this PR proves or fixes holistically.
  - file: [src/wizard/clack-prompter.ts:116](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/wizard/clack-prompter.ts#L116)

  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **PR review already identified the exact regression risk in this patch:** The review on commit `f205beb92eca05aaf84a25ebc7b01971ef027a69` flags two unresolved problems: the `""` fallback can silently drop required fields, and the actual type contract mismatch should be addressed at the `WizardPrompter.text()` boundary if `undefined` is truly possible.
  - sha: [f205beb92eca](https://github.com/openclaw/openclaw/commit/f205beb92eca05aaf84a25ebc7b01971ef027a69)

## Risks / Open Questions

- Closing this PR would discard a plausible onboarding bug report even though current `main` still has no merged, verified fix for the reported trim crash.
- Landing this PR as-is could replace a visible crash with a quieter failure mode where a required credential update is skipped without surfacing an error to the user.
