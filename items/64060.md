## Summary

Keep open. Current main still has the reported cron normalizer bug for nested `payload: { text, model }`, and the PR is a focused useful fix, but it is incomplete because the top-level legacy `text` + `model` path still infers `systemEvent` and drops the model override.

## Best Possible Solution

Keep PR #64060 open and ask for a small follow-up: apply the same `text` plus `hasAgentTurnPayloadHint()` promotion in `inferTopLevelPayload()`, preserve explicit `kind: "systemEvent"` pruning, and add regression tests for both nested `payload: { text, model }` and legacy top-level `{ text, model }` create/patch inputs.
