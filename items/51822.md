## Summary

Keep open. Current `main` does not implement this hardening, and the PR as written still has unresolved regressions around legacy cron webhook handling and logging of invalid stored targets.

## Risks / Open Questions

- Merging this PR as-is would reject new credentialed webhook URLs but still expose existing stored `delivery.to` secrets in the `cron: skipped webhook delivery` warning path.
- Legacy `notify: true` jobs using credentialed `cron.webhook` would silently stop posting after upgrade because the invalid legacy target path has no warning.
- `openclaw doctor --fix` currently migrates simple legacy notify jobs to `delivery.to = cron.webhook`; with this change alone, that documented repair path can convert a previously working legacy fallback into a skipped invalid webhook.
