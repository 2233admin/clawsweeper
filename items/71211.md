## Summary

Keep open. Current main still routes exec stdout/stderr into agent-visible tool results and process logs without applying the existing secret redaction helper, while redaction is only wired into adjacent surfaces such as logs, tool argument display, and session-history recall.

## Best Possible Solution

Keep this open and fix in core. Add a centralized exec-output redaction boundary before any stdout/stderr reaches agent-visible `content`, `details.aggregated`, process `poll/log`, running update `tail`, node-host exec results, and chat-bash replies. Reuse or extend `src/logging/redact.ts`, make the policy explicit for tool outputs, and add focused regression tests that print env-style secrets, bearer/API-key-like tokens, and PEM-like content through foreground and background exec paths.
