## Summary

Keep this PR open. Current main still treats malformed matched session-kill paths as unhandled, while the PR is a narrow gateway bug fix with focused regression coverage and no unrelated security-sensitive file changes.

## Best Possible Solution

Keep this PR open for normal maintainer review and CI. The best path is to land the narrow handler change plus regression test so matched malformed session-kill paths return `400 invalid_request_error` before auth/session lookup, while valid session auth, ownership, kill, and RPC behavior stay unchanged.
