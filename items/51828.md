## Summary

Keep open. Current `main` does contain real follow-up work for parts of this umbrella report, but the issue is only partially obsolete: subagent cleanup/restart recovery now exist, while cron startup catch-up still intentionally replays missed jobs (with throttling), `cron.sessionRetention` still only targets cron run sessions, and I did not find strong evidence that the specific `gateway closed 1000` cron failure path is fixed on `main`. That is not strong enough for a conservative close.
