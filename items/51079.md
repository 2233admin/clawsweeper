## Summary

Keep open. Current `main` still reuses `sessionId` as the `runId` for `agent.request` node ingress, while adjacent gateway code treats `runId` as per-run state. The PR's one-line fix matches the existing `voice.transcript` path and the current test suite still encodes the buggy behavior for `agent.request`, so this is not implemented on `main` and should not be conservatively closed.
