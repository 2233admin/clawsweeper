## Summary

Current `main` does not implement this PR. `src/gateway/server-node-events.ts` still passes `runId: sessionId` for `agent.request`, and the test suite still asserts that behavior. Because gateway abort/stream state is keyed by `runId`, repeated node requests on the same persisted session can collide on shared per-run state instead of getting distinct run ids.

## Evidence
