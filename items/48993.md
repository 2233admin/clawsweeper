## Summary

Current `main` does not implement this PR. The renderer still degrades remote markdown images to alt text, the Control UI CSP still blocks remote image fetches, and the docs explicitly describe that privacy-preserving behavior as intentional and non-configurable. This is not safe to close as implemented or obsolete.

## Risks / Open Questions

- Closing this PR would be incorrect: `main` still enforces the opposite behavior at the renderer, CSP, test, and docs levels.
- The change still needs a product/security decision on whether operator browsers may fetch attacker-controlled remote images from shared chat content, or whether any allowance must be restricted to trusted sources / opt-in configuration.
