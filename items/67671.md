## Summary

Keep open. Current main still does not provide the requested direct single-job cron read path: the CLI single-job display remains `cron.list`-backed, the gateway method inventory, protocol schemas/validators, method scopes, server handlers, agent cron tool, and docs still lack `cron.get`, and the related implementation PR #67677 is closed unmerged.

## Best Possible Solution

Keep this issue open. The best product path is to implement an additive core read API: add `cron.get` to the gateway method list, read scopes, protocol schemas/types/validators, and `src/gateway/server-methods/cron.ts`, backed by `CronService.getJob(id)` with a clean not-found error. Then add `openclaw cron get <id>` or make `cron show` use the direct RPC, add a cron agent-tool `get` action if agent workflows remain in scope, update the public cron/protocol docs, and cover CLI/protocol/server/tool behavior with focused tests. Treat #67677 as prior art, not shipped behavior.
