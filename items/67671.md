## Summary

Keep open. Current `main` does not provide the requested single-job read surface: the CLI still resolves `openclaw cron show` by paging through `cron.list`, the Gateway method set still stops at `cron.list`/`cron.status`/`cron.add`/`cron.update`/`cron.remove`/`cron.run`/`cron.runs`, and the protocol validators/docs likewise expose no `cron.get`. There is an internal `CronService.getJob(...)`, but it is not surfaced as the requested CLI or RPC API.

## Evidence
