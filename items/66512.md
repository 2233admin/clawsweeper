## Summary

Keep open. Current main still lacks the requested systemd readiness notification contract: the generated Linux gateway unit does not use `Type=notify`/`NotifyAccess`, the gateway readiness boundary only logs and flips OpenClaw-internal readiness state, and systemd restart/install paths still return after `systemctl` succeeds rather than after an `sd_notify(READY=1)` handoff.

## Best Possible Solution

Keep this issue open for a focused core daemon/gateway change: add a small systemd notification helper that detects `NOTIFY_SOCKET` and sends `READY=1` only after the selected gateway readiness boundary is reached, update generated Linux units and docs to use `Type=notify` with `NotifyAccess=main`, add unit/startup-hook tests, and coordinate optional `WATCHDOG=1` support with #11973.
