## Summary

Keep open. Current `main` still hardcodes the Dream Diary narrative prompt and does not expose any `dreaming.prompt`-style config/schema/UI surface. The request is coherent and targets bundled `memory-core` behavior, so no conservative auto-close reason applies.

## Evidence

- **Dream diary system prompt is hardcoded:** `memory-core` defines a fixed `NARRATIVE_SYSTEM_PROMPT` with tone/format rules in code, including an explicit ban on markdown headers and a fixed English prompt.
