## Summary

Keep #69533 open. Current main still hardcodes the Dream Diary narrative system prompt in the bundled `memory-core` plugin, the Dreaming config schema/resolver/docs do not expose a `dreaming.prompt` or equivalent template field, and the Dreams UI only patches the enabled flag rather than any prompt text. The provided related items concern broader system-prompt behavior and do not supersede this specific Dreaming customization request.

## Best Possible Solution

Implement this in the bundled `memory-core` plugin rather than closing it: add a typed Dreaming narrative prompt/template config contract, validate it in the plugin manifest/schema, resolve it through `src/memory-host-sdk/dreaming.ts`, pass it into `generateAndAppendDreamNarrative`/`buildNarrativePrompt` with documented placeholders, add focused unit tests, and expose a constrained editor or settings field in the Dreams/Settings UI with docs updates.
