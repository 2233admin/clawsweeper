## Summary

Keep #70680 open. Current main still classifies any non-archived `.jsonl` file as a primary session transcript unless it is a compaction checkpoint, so `<session>.trajectory.jsonl` still passes the helper. Doctor state-integrity still builds orphan candidates from that helper and only cross-checks resolved primary session transcript paths from `sessions.json`; trajectory sidecars documented as valid runtime artifacts are not excluded or referenced there. The provided GitHub context also shows PR #70812 is the focused unmerged fix for this exact bug, so the canonical issue should remain open until that fix or an equivalent lands.

## Best Possible Solution

Land PR #70812 or an equivalent core fix that excludes `*.trajectory.jsonl` from primary transcript and usage-counted transcript classification, adds a doctor regression test proving trajectory sidecars are ignored while real orphan transcripts are still reported, and checks disk-budget/session-usage call sites for the same helper behavior.
