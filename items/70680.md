## Summary

Keep open. Current `main` still misclassifies `*.trajectory.jsonl` sidecars as primary session transcripts, and doctor's orphan scan still uses that broad matcher. The referenced fix exists only in open PR #70812, so the issue is not implemented on `main` yet.

## Evidence

- **Primary transcript matcher is still too broad on current main:** `isPrimarySessionTranscriptFileName()` only excludes `sessions.json`, non-`.jsonl` files, and archived suffixes. A live sidecar like `<session>.trajectory.jsonl` still returns true on current `main`.
  - file: [src/config/sessions/artifacts.ts:27](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/config/sessions/artifacts.ts#L27)
  - command: `sed -n '1,140p' src/config/sessions/artifacts.ts`
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Doctor orphan scan still consumes that matcher:** `noteStateIntegrity()` builds orphan candidates by filtering session-dir files through `isPrimarySessionTranscriptFileName()` and then says doctor can archive them safely as `*.deleted.<timestamp>`. That is the exact path the report describes.
  - file: [src/commands/doctor-state-integrity.ts:898](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/commands/doctor-state-integrity.ts#L898)
  - command: `sed -n '880,940p' src/commands/doctor-state-integrity.ts`
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Trajectory sidecars are an intentional current feature:** Runtime code writes trajectory output beside the session transcript as `<session>.trajectory.jsonl`, and the public docs describe the same default capture location. That makes the false-positive path plausible on shipped/current behavior, not just a hypothetical.
