## Summary

Keep open. Current main still contains a code path consistent with the report: sandboxed non-subagent sessions are documented as having `tree` visibility (current session + spawned children), but `sessions_list` switches to a `spawnedBy=<requester>` gateway query that excludes the requester session itself and can legitimately return `count: 0` when no children exist. I did not find a fix for that path on current `main` or in the diff from the latest release `v2026.4.22` to `6944d7025d9c0ba4891cbe646ab5dab039d504ca`.
