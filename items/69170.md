## Summary

Keep open. Current main still exposes `sessions_spawn.streamTo` in the ACP-capable schema without an ACP-only description, and the runtime still rejects `runtime="subagent"` calls that include `streamTo`. PR #69170 addresses a real remaining issue, though it likely needs a current-main rebase because ACP fields are now schema-gated by backend availability.

## Best Possible Solution

Keep #69170 open for maintainer review, or fold its schema-description assertion into the chosen canonical fix. A current-main version should add the ACP-only `streamTo` description inside the `acpAvailable` schema branch, update the regression test to cover the ACP-backend-present schema, and coordinate with #68397 only if maintainers choose the silent-drop behavior change.
