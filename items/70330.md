## Summary

Keep open. Current `main` still implements WebChat session resets through the generic `chat.send` path, still strips `/new` or `/reset` before the post-reset message is sent, and still resumes queued chat sends after reconnect. That matches the reported silent-rotation failure mode closely enough that this is not a conservative auto-close.
