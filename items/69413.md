## Summary

Keep #69413 open. Current main still has the Signal mention-gating gap the PR targets: Signal mention metadata is hydrated into `@uuid`/`@phone`, but `requireMention` gating only checks text regexes built from configured or derived mention patterns, not the bot's own `account`/`accountUuid`. The PR is not superseded by the related items provided, and its review thread shows the malformed-span concern was addressed in the second commit.

## Best Possible Solution

Keep the PR open and review or adapt it into main: add a span-validating `doesSignalMentionTargetBot`, OR it into Signal `wasMentioned`, add uuid/phone/non-bot/malformed regression tests, decide whether native identity should affect `canDetectMention`, and update the Signal docs that currently say native mentions are unsupported.
