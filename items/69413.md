## Summary

Keep PR #69413 open. Current main still hydrates Signal native mention placeholders into @uuid/@phone text and then applies only text-regex mention gating; it does not compare dataMessage.mentions[].uuid or .number with the bot's own Signal account identity before requireMention gating. The PR is a focused Signal plugin bug fix with relevant tests and active review follow-up, not obsolete or superseded.

## Best Possible Solution

Keep PR #69413 open and merge or adapt its Signal-owned native mention detector plus regression tests. The implementation should compare validated Signal mention metadata spans against deps.accountUuid and normalized deps.account before requireMention gating, preserve the existing text-regex path, and either handle or explicitly track the canDetectMention behavior for configs relying solely on native Signal mention metadata.
