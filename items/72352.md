## Summary

Keep PR #72352 open. Current main still leaves ordinary non-live implicit provider discovery unscoped: `resolveProviderDiscoveryFilter` returns `undefined` before considering configured models/providers, even though the provider discovery/runtime layers already support plugin-id scoping and model/provider ownership helpers. The PR is a focused core agents startup optimization, not obsolete, duplicated, or out of scope.

## Best Possible Solution

Keep the PR open. Ask the contributor to address the existing review comment by changing the broad fallback regression test to include an unresolved configured model ref, then validate the agents/plugin discovery surface with the repo's changed gate or targeted build/test lane. The implementation belongs in core agents provider discovery because it optimizes an existing non-live startup path using existing ownership helpers and the `onlyPluginIds` discovery seam.
