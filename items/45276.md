## Summary

Keep open. Current `main` still has the IPv6 canonicalization bug in `src/gateway/net.ts`: `isLocalGatewayAddress` normalizes the candidate IP, but the tailnet IPv6 branch still compares raw lowercased strings, so equivalent expanded/compressed IPv6 forms can fail to match. The PR’s production fix is not on `main`, and current `main` also lacks dedicated regression coverage for this path.

## Evidence

- **Current main still does raw IPv6 string comparison:** `isLocalGatewayAddress` computes `const normalized = normalizeIp(ip)`, but the IPv6 tailnet branch still checks `normalizeLowercaseStringOrEmpty(ip) === normalizeLowercaseStringOrEmpty(tailnetIPv6)`. That means equivalent spellings like `fd7a:115c:a1e0:0:0:0:0:1` and `fd7a:115c:a1e0::1` still compare unequal on current `main`.
  - file: [src/gateway/net.ts:213](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/gateway/net.ts#L213)
  - command: `nl -ba src/gateway/net.ts | sed -n '200,240p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Repo already has the canonical IP normalizer this PR uses:** `normalizeIpAddress` parses the IP, normalizes IPv4-mapped IPv6, and returns `normalized.toString()` lowercased. That helper canonically rewrites equivalent IP literals, but `isLocalGatewayAddress` only applies it to the input, not the stored tailnet IPv6 value.
  - file: [src/shared/net/ip.ts:173](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/shared/net/ip.ts#L173)
