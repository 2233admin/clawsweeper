## Summary

Keep open. Current `main` still forces a new cron session for every `sessionTarget: "isolated"` run, while `/hooks/agent` already supports stable webhook `sessionKey` values. This PR is not implemented on `main`, and it changes behavior that current docs/tests still describe as intentional, so it needs maintainer judgment rather than cleanup closure.
