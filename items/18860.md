## Summary

Current `main` does not implement the requested `after_tools_resolved` plugin hook or any equivalent plugin API for enumerating resolved tools and their schemas. The runtime clearly resolves and normalizes tool definitions internally, but the public hook catalog and plugin API still expose only per-call tool hooks (`before_tool_call` / `after_tool_call`) plus `registerTool`, so this PR is not obsolete or already shipped.
