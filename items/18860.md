## Summary

Keep this PR open. Current main still lacks the requested `after_tools_resolved` hook and lacks an equivalent plugin-facing resolved-tool/schema discovery API. The related feature issue #21330 was closed as superseded by this still-open PR, so this remains the canonical implementation/design path rather than obsolete work. The PR diff is focused on plugin hook/API/docs/tests and does not introduce CI, dependency, lockfile, release, or package-script changes; the remaining security question is the intended API boundary for exposing complete resolved tool metadata to registered plugins.

## Best Possible Solution

Keep this PR open as the active implementation/design path for plugin-facing resolved tool/schema discovery. A maintainer-quality outcome is to either finish and review the additive hook across `src/plugins/hook-types.ts`, `src/plugins/hooks.ts`, `src/plugins/types.ts`, `src/agents/pi-embedded-runner/run/attempt.ts`, docs, generated SDK baseline, and focused contract tests, or explicitly supersede it with a chosen pull-based resolved-tools API.
