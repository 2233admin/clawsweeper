## Summary

Keep #70529 open. The provided maintainer comment explicitly says to keep this as the Desktop-specific variant, and current main still makes the built-in `user` existing-session profile depend on spawning ambient `npx chrome-devtools-mcp@latest`. The older Browser Relay/EasyClaw path appears obsolete on main, but the packaged Desktop `spawn npx ENOENT` failure remains a concrete, distinct bug from the broader open #62288 attach-hardening issue and #63926 browser-login UX request.

## Best Possible Solution

Keep this issue open and fix the Desktop-specific existing-session attach path in the browser/Desktop runtime: either bundle or resolve the Chrome DevTools MCP driver without relying on ambient `npx`, or fail with a structured diagnostic that recommends the managed `openclaw` profile and points users toward the login handoff path. Leave #62288 to track generic existing-session brittleness and #63926 to track the visible browser/login handoff UI.
