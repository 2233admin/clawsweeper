## Summary

Keep PR #69562 open. Current main still validates cron announce channels only; it accepts `delivery.accountId` and `delivery.failureDestination.accountId` in the gateway protocol and later uses explicit delivery account IDs at runtime, but the `cron.add`/`cron.update` validation path does not check that those account IDs are configured. The PR is a focused gateway fix with targeted tests and review discussion, not obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best implementation path is a gateway create/update validation that checks explicit primary and failure-destination announce account IDs against the resolved channel plugin, includes plugin default accounts, normalizes account IDs consistently, and skips validation when the runtime channel is intentionally inferred from `last`. Keep focused regression tests for both `cron.add` and `cron.update`.
