## Summary

Keep open. Current `main` still exposes `ui.seamColor` from `talk.config`, but the Control UI bootstrap contract/payload and frontend state handling do not carry or apply that value, and the UI codebase has no `seamColor` consumer. The report matches current code, and the issue is also cross-referenced by open PR #56103, so conservative cleanup should not close it.
