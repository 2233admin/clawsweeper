## Summary

Do not close. Current `main` has some adjacent ACP/thread-binding fixes, but the core request in #50955 is still not fully on `main`: native `sessions_spawn(... runtime:"acp", thread:true, mode:"session")` still uses a different placement path than `/acp spawn`, it still skips the sender ownership gate for current rebinds, and Discord bound-delivery fallback still resolves malformed generic targets for DM bindings.

## Evidence

- **Native ACP spawn still uses a separate placement path:** `prepareAcpThreadBinding(...)` in `src/agents/acp-spawn.ts` picks placement from the channel default (`resolveChannelDefaultBindingPlacement`) and never uses the current-context placement logic that `/acp spawn` uses. On Discord, the plugin default remains `"child"`, so native ACP spawns are still not unified with the command path.
  - file: [src/agents/acp-spawn.ts:552](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/acp-spawn.ts#L552)
  - command: `nl -ba src/agents/acp-spawn.ts | sed -n '540,649p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Command path still has distinct current-vs-child logic:** `/acp spawn` still computes placement from live context via `resolveThreadBindingPlacementForCurrentContext(...)`, including sender-bound rebind checks for `placement === "current"`. That is a different codepath from native ACP spawn, so the PR's 'unify thread-bound session binding' goal is not satisfied on `main`.
  - file: [src/auto-reply/reply/commands-acp/lifecycle.ts:362](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/auto-reply/reply/commands-acp/lifecycle.ts#L362)

- **Discord still defaults top-level ACP bindings to child placement:** The Discord channel contract still declares `defaultTopLevelPlacement: "child"`, which feeds the native ACP spawn path. Combined with the separate placement logic above, this keeps Discord DM/current-surface ACP binding behavior divergent from the command path the PR was trying to unify.
  - file: [extensions/discord/src/channel.ts:521](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/extensions/discord/src/channel.ts#L521)
