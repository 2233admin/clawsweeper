## Summary

Keep open. Current `main` supports QR plus manually shareable setup codes for mobile pairing, so the issue's "no alternative to QR" claim is partly outdated, but the requested short cloud-resolved `openclaw invite` flow is not implemented on `main`. There is no invite command, invite-code config, invite endpoint integration, or mobile invite-code UI in the checked-out code, and the issue is recent enough that a stale close would be incorrect.
