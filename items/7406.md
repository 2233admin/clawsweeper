## Summary

Current main still only partially supports Telegram forum topic names. It learns and exposes `TopicName` for Telegram inbound context/prompt metadata, but does not persist it into session `label`/`displayName`, does not expose a `topicName` field on session rows, and the Control UI selector only consumes `label`/`displayName` before falling back to parsed session-key labels. #7406 should remain open as the canonical tracker; the related implementation PRs in the provided context (#7407 and #43105) are closed and unmerged, and duplicate reports #53701/#68652 were closed back toward this issue.

## Best Possible Solution

Keep #7406 open as the canonical implementation tracker. The best fix is to map cached Telegram forum `TopicName` into persisted session display metadata, likely by setting `ThreadLabel` or a dedicated `topicName`/display field during inbound session metadata recording, then ensure `sessions.list` and the Control UI selector render the human-readable group/topic label with a stable fallback to topic id/key when no name is known. Add regression coverage for Telegram context extraction, session-store metadata persistence, gateway session rows, and Control UI label resolution.
