## Summary

Keep this PR open. Current main still has the attributed-prefix iMessage echo-cache gap this PR targets: echo text keys only normalize CRLF and trim whitespace, while the self-chat and normal inbound echo paths still depend on that text key. The PR is a focused implementation candidate, but it should address the existing review comments before maintainer merge.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to update the PR to resolve the review comments, keep the normalization scoped to the iMessage echo-cache seam, preserve the echo-cache and self-chat regression tests, then run the relevant iMessage monitor tests plus pnpm check:changed before maintainer review and merge.
