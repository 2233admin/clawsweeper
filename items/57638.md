## Summary

Keep open. Current `main` does not implement cron-level defaults for `agentTurn` model, delivery, or `contextTokens`; cron still relies on per-job fields plus agent/session/model defaults, and isolated jobs still hard-default delivery to `announce`.

## Evidence

- **Cron config has no `defaults` block:** `CronConfig` exposes `enabled`, `store`, `maxConcurrentRuns`, `retry`, `webhook`, `webhookToken`, `sessionRetention`, `runLog`, `failureAlert`, and `failureDestination`, but nothing like `cron.defaults` for model/delivery/context tokens.
  - file: [src/config/types.cron.ts:30](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/config/types.cron.ts#L30)
  - command: `nl -ba src/config/types.cron.ts | sed -n '1,120p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Validated config schema still rejects unknown cron keys:** The runtime schema for `cron` only admits the same top-level keys and is `.strict()`, so `cron.defaults` is not part of the supported config surface on current `main`.
  - file: [src/config/zod-schema.ts:544](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/config/zod-schema.ts#L544)
  - command: `nl -ba src/config/zod-schema.ts | sed -n '540,590p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Isolated jobs still default delivery to `announce`:** Cron job creation hardcodes `{ mode: "announce" }` when an isolated `agentTurn` job omits `delivery`, and the test suite asserts that behavior.
  - file: [src/cron/service/initial-delivery.ts:3](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/cron/service/initial-delivery.ts#L3)
  - command: `nl -ba src/cron/service/initial-delivery.ts | sed -n '1,80p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Cron payload does not support `contextTokens`:** The public cron payload schema includes `model`, `fallbacks`, `thinking`, `timeoutSeconds`, `allowUnsafeExternalContent`, `lightContext`, and `toolsAllow`; there is no per-job `contextTokens` field to pair with a cron-level default.
  - file: [src/gateway/protocol/schema/cron.ts:4](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/gateway/protocol/schema/cron.ts#L4)
  - command: `nl -ba src/gateway/protocol/schema/cron.ts | sed -n '1,24p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Model precedence still falls back to agent/session defaults, not cron defaults:** Current model resolution order is configured default -> agent/subagent override -> Gmail override -> per-job payload model -> stored session override, with docs explicitly stating the final fallback is agent/default model selection.
  - file: [src/cron/isolated-agent/model-selection.ts:45](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/cron/isolated-agent/model-selection.ts#L45)

- **Context-token resolution uses agent config or session/runtime state only:** `contextTokens` for cron runs are derived from `prepared.agentCfg?.contextTokens`, model-catalog lookup, prior cron session state, or the built-in default constant; there is no read from `cfg.cron`.
  - file: [src/cron/isolated-agent/run.ts:732](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/cron/isolated-agent/run.ts#L732)
