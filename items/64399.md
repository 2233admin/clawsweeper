## Summary

Current main still only partially addresses the report. The Overview Gateway Logs tail strips ANSI locally, but the dedicated Control UI Logs tab still requests `logs.tail`, parses JSON/raw lines without stripping ANSI, and renders `entry.message ?? entry.raw` directly. The related fix PR #64431 was closed unmerged, so this issue should remain open.

## Best Possible Solution

Keep this issue open and land a focused Control UI logging fix. The best path is one shared ANSI strategy for all `logs.tail` consumers: strip ANSI or render it safely before display in both the Overview tail and the dedicated Logs tab, decide whether exported logs should preserve raw text or use sanitized display text, and add regression tests for ANSI-bearing JSON log messages, subsystem/context parsing, and raw fallback lines. PR #64431 is useful prior art but cannot close this issue because it was not merged.
