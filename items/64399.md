## Summary

Keep open. Current main only strips ANSI in the Overview gateway log tail, while the full Control UI Logs path still parses and renders log messages without ANSI stripping or rendering. The related PR #64431 was closed unmerged, and current tests do not cover the reported ANSI regression.

## Best Possible Solution

Keep the issue open and land a focused Control UI fix that uses one shared ANSI strategy for all displayed gateway logs: either strip ANSI consistently in `parseLogLine()`/rendering or render ANSI styles safely. Cover message, subsystem/context parsing, raw fallback lines, and the Overview tail with regression tests; PR #64431 is useful prior art but should be reworked against current main rather than treated as shipped.
