## Summary

Keep open. Current main does not implement per-agent skill pinning via `agents.list[].skills.always` or an equivalent additive pinning mechanism. Per-agent config only supports a final string allowlist replacement, `metadata.openclaw.always` only bypasses skill eligibility gates, and skill selection in the prompt still depends on applicability scanning rather than auto-loading pinned skills.

## Evidence
