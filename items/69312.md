## Summary

Keep this PR open. Current main still has the reported MEDIA false-positive path for 4-space/tab indented code blocks, while the PR is a focused implementation candidate. It still needs maintainer review before landing, especially because the current PR also changes MEDIA case semantics without updating existing lowercase-media tests or the streaming directive accumulator.

## Best Possible Solution

Keep this PR open and have a maintainer review or land a corrected version that protects indented code blocks in src/media/parse.ts, preserves code indentation during cleanup, adds the missing media and fence regression tests, and deliberately aligns MEDIA case semantics across splitMediaFromOutput(), reply delivery, and streaming directive buffering.
