## Summary

Keep #69312 open. Current main still has the MEDIA extraction behavior the PR targets: `splitMediaFromOutput()` only protects fenced code spans, then strips leading indentation before checking for `MEDIA:`, so 4-space/tab indented code lines can still be treated as directives. The PR is recent, focused, linked to #69311, and the author actively addressed review findings rather than leaving a low-signal stale change.

## Best Possible Solution

Keep the PR open or land an equivalent focused fix. The best path is to add an indented-code guard for `splitMediaFromOutput()`, preserve code-line whitespace during MEDIA cleanup, add regression tests for 4-space/tab code blocks and mixed real MEDIA plus retained code, and review whether the fence trailing-text and streaming case-sensitivity changes should land in the same PR or as narrower follow-ups.
