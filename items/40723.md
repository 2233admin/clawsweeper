## Summary

Keep PR #40723 open. Current main still does not implement the PR's central user-facing change: the Agents overview has a selected-agent model selector only, with no explicit global `Active Primary Model` control and no save path that writes `agents.defaults.model` from that overview. The PR remains a plausible focused UI improvement, and the prior discussion shows useful fallback-handling review feedback that the PR author addressed in later commits.

## Best Possible Solution

Keep the PR open for maintainer review or a fresh port. The best path is to add an explicit global default primary-model selector in the Agents overview, wire it to `agents.defaults.model`, preserve fallback lists in canonical `fallbacks` form, keep per-agent overrides unchanged, and include focused unit coverage for default-vs-override model state resolution.
