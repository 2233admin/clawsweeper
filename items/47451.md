## Close Comment

Closing this as a Codex review because the requested behavior is out of OpenClaw core scope and fits the existing skill/plugin path better.

Current `main` is explicit that core should stay lean and optional/domain behavior should ship as plugins/skills, with new skills going to ClawHub first (`VISION.md:54-70`). OpenClaw already supports per-workspace prompt/skill customization (`README.md:253-257`, `docs/tools/skills.md:13-25`), and the Skill Workshop plugin is specifically meant to turn reusable corrections and recurring pitfalls into workspace `SKILL.md` instructions (`docs/plugins/skill-workshop.md:16-29`). An Austrian legal-term glossary and “ask before guessing on ambiguous legal abbreviations” policy fits that path much better as a workspace or ClawHub skill for Lexa than as a core product change.

I also checked current `main` (`0e7250f37bf347c72790abf2a6692d5ec84e5ea1`) and did not find any existing Lexa- or Austrian-legal-specific core surface to extend here.
