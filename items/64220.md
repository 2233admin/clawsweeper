## Summary

Current main still matches the reported Control UI stale-state bug. Active compaction status is set without a stale timeout, the renderer shows active/retrying compaction indefinitely, and several terminal/reconnect/clear-history paths still clear run/stream state without clearing compactionStatus or fallbackStatus. The closed related #39908 is broader symptom evidence, not a canonical fix or superseding tracker for this narrower Control UI state-machine issue.

## Best Possible Solution

Keep this issue open and fix it in the Control UI state layer. Add a run/session-scoped stale timeout or watchdog for active/retrying compaction, centralize cleanup of compactionStatus/fallbackStatus and their timers, call that cleanup on terminal chat events, reconnect/disconnect, session/agent switch, overview session changes, clear/reset, and send-failure paths, then add regression tests for missed compaction end after final reply plus reconnect and clear-history cleanup.
