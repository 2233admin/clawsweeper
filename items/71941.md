## Summary

Keep PR #71941 open. Current main still has the reported wizard behavior: `agents add` seeds the workspace prompt from `resolveAgentWorkspaceDir`, and that resolver nests non-default agents under `agents.defaults.workspace` when the default workspace is configured. The docs continue to promise peer-level `~/.openclaw/workspace-<agentId>` defaults, so the PR addresses a real unresolved core bug tracked by #71889. The PR should not be closed as cleanup; it needs the review-comment fixes before merge.

## Best Possible Solution

Keep #71941 open as the active fix for #71889. The best path is to preserve the PR's narrow product direction: add a wizard-only peer-level workspace suggestion for brand-new agents when `agents.defaults.workspace` exactly matches the documented setup default, keep `resolveAgentWorkspaceDir` runtime behavior unchanged for explicit shared-base configurations, preserve existing-agent update prompts, fix the HOME/OPENCLAW_HOME portability issue in the new test, correct changelog credit, and then run the changed gate before maintainer merge.
