## Summary

Keep this issue open. Current main still has the reported macOS launchd crash-loop shape: the LaunchAgent plist emits `KeepAlive=true` with `ThrottleInterval=1`, while missing or damaged gateway config exits with code 78 before the gateway loop starts. The related #60885 is adjacent launchd supervision work, but it is still open and does not supersede this missing-config respawn/log storm report.

## Best Possible Solution

Keep this issue open for a macOS LaunchAgent/gateway startup guard that stops repeated missing-config or `EXIT_CONFIG_ERROR` respawns while preserving intentional restarts. A good fix would add launchd-specific suppression, bounded backoff, a max retry/sentinel path, or a degraded wait-for-config mode, plus LaunchAgent regression tests and gateway troubleshooting docs. Coordinate with #60885, but do not close this as a duplicate unless that work explicitly covers the missing-config failure mode.
