## Summary

Keep open. Current `main` does contain a later fix for one reported symptom: raw `404 status code (no body)` assistant errors are now classified as `model_not_found`, which lets model fallback continue instead of leaving `failoverReason` null. But I did not find strong evidence that the broader regression in #56594 itself was fully fixed: the issue reports that every OpenAI and Google model call started 404ing after the post-update `openclaw doctor --fix` run, and the read-only inspection here did not establish the underlying cause or show a repro-proof / fix proof for that doctor-triggered cross-provider failure.
