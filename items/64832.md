## Summary

Current main has the related isolated-session `sessionFile` rollover fix, but it still does not archive the prior raw transcript after isolated heartbeat or cron rotation. This PR remains unique cleanup work, though its cron archive reference-set concern should be addressed before merge.

## Best Possible Solution

Keep this PR open for maintainer review and revision. The best path is to preserve the shipped `sessionFile` rollover behavior, add a narrow archive-on-rotation flow for heartbeat and isolated cron, capture the prior entry before overwriting it, archive after the new entry is persisted with `reason: "reset"` and `restrictToStoreDir: true`, and adjust cron reference checks so historical run rows do not block archiving genuinely rotated transcripts. Add targeted tests for heartbeat, cron run-key retention, once-only persistence, and archive failure handling.
