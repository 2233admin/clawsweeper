## Summary

Current `main` at `42514156e03c53dec6be6c581069d8c5b2d4cd42` still lacks both requested surfaces from PR #51163: there is no `/interrupt` command wired into the reply command pipeline or command registry, and there is no cross-session thread-context persistence/load path for cron-triggered threads. This is not `implemented_on_main`, and there is not strong evidence to auto-close it for any other conservative cleanup reason.
