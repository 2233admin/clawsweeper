## Summary

Keep open. Current main still enforces boundary containment to the configured root, so the linked workspace-bootstrap symlink behavior is not implemented. This PR is paired with open issue #64472 by the same author and remains an implementation candidate, but the submitted patch needs a narrower security design before it should merge.

## Best Possible Solution

Keep this PR and #64472 open, but require a revised implementation before merge. The safer path is a narrow, tested exception for the intended workspace-bootstrap symlink traversal only: use the effective and canonical OpenClaw state root, avoid allowing direct absolute paths to sibling state directories, preserve rejection for credentials/session paths and non-workspace boundaries, and add positive symlink plus negative direct-path/security regression tests.

## Risks / Open Questions

- Merging the submitted patch as-is could allow direct absolute access to other OpenClaw state paths, such as credentials or sessions, rather than only allowing intended workspace-bootstrap symlink traversal.
- The proposed fallback is in shared boundary enforcement used outside workspace bootstrap, so it could weaken sandbox, upload, or other root-confinement callers.
- The submitted change has no automated positive or negative regression tests for this security-sensitive behavior.
- The submitted fallback uses the OS home path shape rather than the effective OpenClaw home/state root contract, so it could fail or behave inconsistently when `OPENCLAW_HOME` or state relocation is used.
