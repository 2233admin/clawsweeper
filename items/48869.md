## Summary

Keep open. Current `main` documents the underlying mechanisms for waking work (`docs/cli/system.md:32`, `docs/automation/cron-jobs.md:80`, `docs/gateway/heartbeat.md:13`), but it does not yet document the specific failure mode from the issue: scripts that drop inbox files without also waking the session. The proposed doc surface is still absent from `docs/`, so there is not strong evidence to close as implemented.

## Evidence

- **System event docs exist, but only as command reference:** `openclaw system event` is documented as enqueueing a system event on the main session and `--mode now` triggers heartbeat immediately. This covers the trigger primitive, not the wakeup-script pitfall or a best-practices guide.
  - file: [docs/cli/system.md:32](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/docs/cli/system.md#L32)
