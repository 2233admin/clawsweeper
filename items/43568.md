## Summary

Keep open. Current main has provider/model-level request proxy support, but it does not implement the issue's requested per-auth-profile `proxyUrl` behavior. Auth profile credential types do not define proxy fields, profile auth resolution returns only credential material plus provider/email, and the model fetch proxy path is driven by model/provider request overrides rather than the selected auth profile. The related #67228 is broader per-provider/per-channel proxy work and does not fully supersede this auth-profile-specific request.

## Evidence
