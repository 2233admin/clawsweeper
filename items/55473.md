## Summary

Keep open. Current `main` does not implement the PR’s core behavior: gateway startup still treats secret-resolution failures as fatal, and the runtime snapshot/reload path has no startup-degrade recovery field like the PR proposed. The underlying problem described in the PR therefore still appears real on `main`, even though parts of the PR’s supporting refactor (such as merge-patch helpers) now exist elsewhere.
