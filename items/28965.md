## Summary

Keep open. Current `main` has partial mitigation for model-config drift and stale session overrides, but it does not implement the issue's proposed maintainer/admin surfaces: there is no `openclaw models reconcile`, no `openclaw sessions model-reset`, and the current model-status output does not show the full requested source chain.

## Evidence

- **No `models reconcile` user-facing command is documented:** The models CLI docs list `status`, `list`, `set`, and `scan` as the common commands, with aliases/fallback/auth subcommands later, but no reconcile command.
  - file: [docs/cli/models.md:19](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/docs/cli/models.md#L19)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **No `models reconcile` command is exported on main:** The command barrel exports aliases, auth, fallbacks, image-fallbacks, list/status, scan, set, and set-image only.
  - file: [src/commands/models.ts:1](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/commands/models.ts#L1)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **No `sessions model-reset` user-facing command is documented:** The sessions CLI docs cover listing sessions plus `sessions cleanup`, but no model-reset or override-reset command.
  - file: [docs/cli/sessions.md:8](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/docs/cli/sessions.md#L8)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Current sessions CLI is list/display oriented, not override-reset oriented:** `sessionsCommand` reads session stores, renders rows/JSON, and exposes filters like `--agent`, `--all-agents`, `--store`, and `--active`; there is no reset path here.
  - file: [src/commands/sessions.ts:110](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/commands/sessions.ts#L110)

This addresses part of the stated pain, but only during runtime selection, not via an explicit admin reset command.

  - file: [docs/providers/openai.md:634](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/docs/providers/openai.md#L634)

It does not expose the full source chain requested in the issue (`config default`, `session override`, `parent-session inherited override`, `fallback-applied result`).

## Risks / Open Questions

- Some of the issue's underlying pain is already partially reduced on `main`, so any future implementation should target the remaining missing maintainer/admin surfaces instead of duplicating existing runtime self-healing.
- There may be alternate UX paths (`/model ...`, `model=default`, Sessions UI clearing) that satisfy some operators, but there is still no batch/CLI equivalent for the proposed explicit reset and reconcile workflows.
