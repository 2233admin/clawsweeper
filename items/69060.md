## Summary

Keep PR #69060 open. Current main still does not implement `x-openclaw-session`; `/v1/chat/completions` only honors the full `x-openclaw-session-key` override, the OpenAI `user` field, or a fresh random UUID fallback. The linked #68920 discussion explicitly identifies this PR as the right narrow fix for the QMD/session-warm portion while keeping the larger light-context issue open.

## Best Possible Solution

Keep the PR open and either land a tightened version with validation, focused tests, and docs for `x-openclaw-session`, or fold the same narrow change into a maintainer-owned fix for #68920 while leaving the larger `lightContext`/voice-mode work tracked separately.
