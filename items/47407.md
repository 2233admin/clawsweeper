## Summary

Keep open. Current `main` does not implement the PR’s requested edit-time allowlist warning. `cron edit --model` still forwards the raw patch without validating against the target gateway’s allowlist, while the only existing safeguard is a runtime warning/fallback when the cron executes.

## Risks / Open Questions

- The original failure mode is mitigated on current `main` by runtime fallback, but the UX requested by this PR still is not present, so closing as `implemented_on_main` would overstate what shipped.
- Any CLI-side warning must resolve against the target gateway’s config and model-resolution rules; a local-only check would be misleading in remote-gateway setups.
