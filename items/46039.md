## Close Comment

Closing as already fixed. Codex review of current `main` found that Anthropic model normalization now lives in the static helper at `src/agents/model-ref-shared.ts:29`, while provider-specific runtime normalization is loaded lazily through `src/agents/provider-model-normalization.runtime.ts:16` and wired from `src/agents/model-selection-normalize.ts:43`, which removes the eager import/initialization path behind this CLI startup failure. The issue thread also already confirms the regression reproduces on `v2026.3.12` but not on `v2026.3.13`, so this is implemented on main and appears to have shipped in `v2026.3.13`.
