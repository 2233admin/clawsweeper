## Summary

Keep open. Current `main` has partial Gemini multimodal memory support, but it only covers image/audio files from `memorySearch.extraPaths`; it does not implement the full request for video/PDF indexing or direct remembering of media sent to the agent.

## Evidence

- **Docs scope is narrower than the issue:** The memory-search docs say Gemini supports only image/audio indexing, not video or PDFs.
  - file: [docs/concepts/memory-search.md:116](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/docs/concepts/memory-search.md#L116)

- **Config docs limit multimodal support to images/audio in extraPaths:** The config reference documents `multimodal.enabled` for Gemini, but says it applies only to `extraPaths` and lists supported formats only for images and audio.
  - file: [docs/reference/memory-config.md:308](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/docs/reference/memory-config.md#L308)

  - file: [packages/memory-host-sdk/src/host/multimodal.ts:3](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/packages/memory-host-sdk/src/host/multimodal.ts#L3)
