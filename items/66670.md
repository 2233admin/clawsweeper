## Summary

Current main still has the reported attribution gap. Native and ACP `sessions_spawn` paths resolve the target agent and encode it in the child session key, and the task model/SQLite store can persist `agentId`, but the task-run creation calls still omit `agentId`, so `task_runs.agent_id` remains null for those spawned rows.

## Best Possible Solution

Keep this open and fix the core attribution plumbing. Add `agentId` to `RegisterSubagentRunParams`, pass `targetAgentId` from native `spawnSubagentDirect`, pass `targetAgentId` into both ACP `createRunningTaskRun` calls, and add regression coverage proving cross-agent `sessions_spawn` writes `TaskRecord.agentId` and SQLite `task_runs.agent_id`. Treat any historical backfill/doctor behavior as a separate product decision.
