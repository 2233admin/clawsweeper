## Summary

Keep #69186 open. Current main has Control UI web push infrastructure and a final chat-event boundary, and core/plugin hooks expose server-side agent completion (`agent_end`), but there is no Control UI setting or browser-side completion/success sound played when an agent turn finishes. The related #59199 cleanup only covers the hook signal needed for automation; it does not implement the requested WebChat audible cue or custom sound option.

## Best Possible Solution

Keep the issue open as focused Control UI work. The likely implementation belongs in `ui/src/ui/controllers/chat.ts` or `ui/src/ui/app-gateway.ts` on successful `final` events, backed by a persisted UI setting in `ui/src/ui/storage.ts` and a Communication settings control in `ui/src/ui/views/config.ts`. Existing `agent_end` hooks can remain the plugin/server-side automation path but should not be treated as satisfying the browser sound request.
