## Summary

Keep open. Current main has related notification infrastructure and a server-side `agent_end` plugin hook, but it does not implement the requested optional Control UI completion/success sound, settings toggle, or custom sound URL. The earlier related #59199 cleanup is still relevant because `agent_end` covers post-turn plugin automation, but that does not satisfy this issue’s browser WebChat audible cue request.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. A narrow implementation would add a browser-local Control UI setting for completion sound, persist it in `UiSettings`, expose controls in the Notifications or Appearance/Chat settings surface, and trigger only from successful terminal chat events in the active WebChat session. If maintainers prefer not to add core UI behavior, the next best path is to define a Control UI/plugin extension seam that lets a plugin react to `agent_end` or chat terminal events inside the operator browser without hard-coding this feature into core.
