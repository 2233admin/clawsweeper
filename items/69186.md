## Summary

Keep #69186 open. Current main has the backend/plugin completion hook (`agent_end`) that was enough to close related #59199, but it does not implement the requested Control UI/WebChat audible completion cue, persisted on/off setting, custom sound URL, or automatic browser notification on chat final events.

## Best Possible Solution

Keep the issue open and implement it in the Control UI chat terminal-event path: add a persisted browser-local setting for completion sound/notification behavior, provide an enable/test control in settings, play a short built-in Web Audio cue on successful `chat` final events for the active session, and document the behavior. The existing `agent_end` hook can remain the plugin-side automation seam, but it should not be treated as satisfying this UI feature.
