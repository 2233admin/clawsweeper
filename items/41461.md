## Summary

Current `main` does not contain the PR’s fix. The Anthropic provider wrapper still only installs `createAnthropicBetaHeadersWrapper()` when `resolveAnthropicBetas()` returns a non-empty list, while model-level `headers` are resolved separately from `extraParams`; that leaves the reported OAuth-token path unhandled when betas are implied by headers rather than `params.context1m`/`params.anthropicBeta`.

## Evidence

- **`extraParams` resolution ignores model headers:** `resolveExtraParams()` merges only default/model/agent `params`; it never reads configured model `headers`, so header-only Anthropic beta settings do not make `extraParams` non-empty.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:73](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/agents/pi-embedded-runner/extra-params.ts#L73)
  - command: `nl -ba src/agents/pi-embedded-runner/extra-params.ts | sed -n '67,83p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Model headers are carried separately on the runtime model:** Configured provider/model headers are sanitized from `configuredModel?.headers` and end up on `requestConfig.headers` / `runtimeModel.headers`, separate from `extraParams`. That matches the PR’s described reproduction path: model-level headers can exist without any Anthropic beta extra params.
  - file: [src/agents/pi-embedded-runner/model.ts:554](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/agents/pi-embedded-runner/model.ts#L554)

- **Current Anthropic plugin still gates the wrapper on non-empty betas:** `wrapAnthropicProviderStream()` computes `anthropicBetas` from `ctx.extraParams` and only applies `createAnthropicBetaHeadersWrapper()` when `anthropicBetas?.length` is truthy. The wrapper itself would add OAuth/default betas once called, but the guard prevents that in the empty-betas case this PR targets.
