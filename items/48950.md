## Summary

Keep open. Current `main` does not contain this PR's change, and the underlying empty-auth/device-identity behavior is still live in current code. `src/gateway/probe.ts` still strips device identity for any loopback probe whose auth object lacks a concrete token/password, while multiple current callers still construct and pass empty auth objects. The PR also still needs maintainer judgment because current callers intentionally use empty auth objects for degraded/diagnostic probes, so the proposed `opts.auth === undefined` check is not obviously safe to close as already-handled or obsolete.
