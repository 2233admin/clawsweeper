## Summary

Keep #70238 open. Current main still has at least one OpenClaw-owned Windows child-process spawn path without `windowsHide: true`, and the linked fix work is not shipped: #70308 was closed unmerged, while #72332 is an open follow-up PR that explicitly proposes the remaining Windows flicker/Bonjour fixes. The issue is recent, has multiple independent user confirmations, and should stay open until a focused Windows fix lands or maintainers intentionally close it through the canonical PR.

## Best Possible Solution

Keep #70238 open until a Windows-focused fix lands. The best path is to verify the exact process creating the flashing window on Windows, add `windowsHide: true` to OpenClaw-owned background spawn surfaces such as the TUI relaunch where appropriate, address the bundled Bonjour/@homebridge/ciao ARP/discovery path if it is the source, and add mockable regression coverage for the affected spawn options. PR #72332 is the current linked implementation candidate, but the issue should not be closed as shipped until that or an equivalent focused fix is merged.
