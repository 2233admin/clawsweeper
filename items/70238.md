## Summary

Keep #70238 open. The report is recent, has multiple independent confirmations, and the related fix PR #70308 is closed unmerged. Current main does not contain the proposed `windowsHide: true` change in the cited TUI relaunch spawn path, while the bundled Bonjour gateway discovery path remains enabled by default through the `bonjour` plugin and `@homebridge/ciao`, which overlaps with the related Windows flashing-console reports. This needs a real Windows root-cause pass rather than conservative cleanup closure.

## Best Possible Solution

Keep the issue open and do a Windows-focused fix pass: identify the exact process creating the flashing window, add `windowsHide: true` to any OpenClaw-owned background spawn surface that can run on Windows, and if the source is Bonjour/@homebridge/ciao ARP probing, either patch/wrap that path or make the bundled discovery behavior avoid visible `cmd.exe` windows. Add mockable regression coverage for spawn options and document any `discovery.mdns.mode="off"` workaround only as a temporary mitigation.
