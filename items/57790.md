## Summary

Current `main` still appears to miss `session_end` emission for active gateway sessions during graceful shutdown/restart. The shutdown path runs `gateway_stop` and transport cleanup, but I found no session-store iteration or call site that emits per-session `session_end` hooks outside reset/delete flows.

## Evidence

- **Gateway close path only runs gateway_stop then generic shutdown:** `createGatewayServer` wraps `close()` by first calling `runGlobalGatewayStopSafely(...)`, then `runClosePrelude()`, then the close handler. There is no session lifecycle handling in that wrapper.
  - file: [src/gateway/server.impl.ts:910](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/gateway/server.impl.ts#L910)
  - command: `nl -ba src/gateway/server.impl.ts | sed -n '904,920p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Shutdown handler does not touch the session store or emit session_end:** `createGatewayCloseHandler` stops channels/services, broadcasts shutdown, closes clients/websockets/HTTP servers, and unsubscribes listeners. In this function there is no call to `emitGatewaySessionEndPluginHook`, no session-store load, and no iteration over active sessions.
  - file: [src/gateway/server-close.ts:105](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/gateway/server-close.ts#L105)
  - command: `nl -ba src/gateway/server-close.ts | sed -n '100,235p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Per-session session_end emission exists, but in reset/delete flow:** `emitGatewaySessionEndPluginHook(...)` is the gateway helper that actually fires `session_end`; it lives in `session-reset-service.ts` and is used for explicit session lifecycle operations, not shutdown.
  - file: [src/gateway/session-reset-service.ts:96](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/gateway/session-reset-service.ts#L96)
  - command: `nl -ba src/gateway/session-reset-service.ts | sed -n '95,160p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Tests cover reset/delete lifecycle hooks, not shutdown lifecycle hooks:** Current tests assert `session_end` for `sessions.delete` and `sessions.reset`, while shutdown tests only verify listener unsubscription and socket/server teardown. I found no shutdown test asserting per-session `session_end` emission.
