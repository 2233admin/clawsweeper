## Summary

Keep #57790 open. Current main has a generic `gateway_stop` shutdown hook and explicit `session_end` emissions for reset/delete/compaction/session-rollover paths, but the SIGTERM/SIGINT/SIGUSR1 gateway shutdown/restart flow does not enumerate active sessions or emit `session_end` for them. The related #71756 is still open and broader, so it is not a shipped superseding fix.

## Best Possible Solution

Keep this issue open. The best fix is to add a gateway shutdown/restart lifecycle step before runtime state is cleared that identifies active sessions and emits per-session `session_end` with an explicit terminal reason such as shutdown/restart, with regression coverage for SIGTERM, SIGINT, and SIGUSR1 restart paths. If maintainers prefer `gateway_stop` as the supported shutdown API instead, document that decision and expose enough active-session context for plugins that need to finalize per-session state.

## Risks / Open Questions

- A shutdown implementation must avoid double-firing `session_end` for sessions already reset, deleted, or compacted in the same lifecycle.
- Session-end hook execution during SIGTERM/SIGINT should be bounded or best-effort so plugin latency cannot defeat the gateway shutdown watchdog.
