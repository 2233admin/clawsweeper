## Summary

Current main still matches the reported broken Browserbase path: OpenClaw documents the static Browserbase root WebSocket URL and the browser plugin still resolves/connects a configured CDP URL directly, while current Browserbase docs describe creating a session via POST /v1/sessions and using the returned connectUrl. citeturn0search0

## Best Possible Solution

Keep this issue open. The narrow fix should update docs/tools/browser.md to stop recommending the obsolete static Browserbase root URL and either document the manual create-session/connectUrl workflow or add Browserbase session creation/release support in the bundled browser plugin. If maintainers want this to stay vendor-neutral, add a typed remote-CDP session-provider seam in the browser plugin/SDK and implement Browserbase through that seam rather than hard-coding provider behavior in core.
