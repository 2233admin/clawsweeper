## Summary

Keep #71396 open. Current main still contains the Feishu mention-cascade behavior this PR removes: inbound `ctx.mentionTargets` are still passed into `createFeishuReplyDispatcher`, and the dispatcher still injects them into the first outbound plain-text/card reply and streaming-card close. The linked issue #70065 is closed, but the fix is not present on current main, and the latest PR discussion indicates the author pushed follow-up changes that still need normal maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open and have the Feishu plugin maintainer review the latest branch. The desired fix belongs in the bundled Feishu plugin: stop unconditional automatic propagation of inbound `mentionTargets` into fallback/error/ack/ordinary replies, keep mention information as bounded and escaped agent context, and preserve any intentional explicit-mention behavior with regression coverage for plain text, card, streaming-card, and fallback paths.
