## Summary

Keep open. Current `main` still auto-forwards Feishu `mentionTargets` into outbound plain-text, card, and streaming-card replies, so the underlying bug remains real. This PR is directionally useful, but the existing review comment identifies a concrete unresolved regression: removing programmatic mention injection also removes the only card-mode conversion path from mention targets to Feishu card-compatible `<at id=...></at>` markup.

## Evidence

- **Current main still auto-attaches mention targets to outbound replies:** `createFeishuReplyDispatcher` passes `mentionTargets` into both plain-text and structured-card sends on the first chunk, and prepends converted card mentions when closing a streaming card.
  - file: [extensions/feishu/src/reply-dispatcher.ts:307](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/extensions/feishu/src/reply-dispatcher.ts#L307)

  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Current main prompt contract says mentions are automatic, not model-authored:** `buildFeishuAgentBody` tells the agent that replies will automatically @mention the listed users and explicitly says not to write mentions itself; the matching unit test asserts that exact prompt text.
  - file: [extensions/feishu/src/bot.ts:175](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/extensions/feishu/src/bot.ts#L175)

- **Open review already identified the card-regression risk on this PR:** The PR review comment notes that dropping `mentions` from `sendStructuredCardFeishu` removes the only conversion path that turns mention targets into card-compatible `<at id=...>` tokens, so explicit mention replies in card and streaming-card mode can degrade to plain text and stop notifying users.
  - sha: [a2c862392f8d](https://github.com/openclaw/openclaw/commit/a2c862392f8dc52ea67662b31985265b79d31046)

## Risks / Open Questions

- Until a fix lands, current `main` can still notify every inbound-mentioned user on fallback, error, ack, and ordinary replies because `mentionTargets` are forwarded automatically.
- Merging the PR as-is risks breaking legitimate explicit mention behavior in Feishu card and streaming-card flows, because card delivery depends on code-side conversion to `<at id=...></at>` markup.
