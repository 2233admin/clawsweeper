## Summary

Keep PR #71396 open. Current main still implements the Feishu mention-cascade behavior this PR is trying to remove: inbound `mentionTargets` are still passed into the reply dispatcher, and the dispatcher still attaches them to the first outbound text/card reply and streaming-card close. The PR is not obsolete or implemented on main, and its unresolved review/security discussion needs maintainer handling before merge.

## Best Possible Solution

Keep #71396 open and have the Feishu plugin owner/maintainer finish the bundled-plugin fix: stop unconditional automatic propagation of inbound `mentionTargets` for fallback/error/ack/ordinary replies, keep mention context safe for the agent, preserve explicit mention behavior through allowlisted structured conversion for text/card/streaming-card modes, and keep regression coverage for those paths.
