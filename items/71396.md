## Summary

Keep #71396 open. Current main still has the Feishu mention-cascade behavior this PR removes, and the PR is open/unmerged with a focused Feishu plugin fix plus useful regression coverage. The latest author push improves prompt-name hardening, but review/security concerns remain for maintainer handling rather than cleanup closure.

## Best Possible Solution

Keep #71396 open. Finish the Feishu plugin fix in this PR or a maintainer follow-up: stop unconditional inbound mention propagation for ordinary, ack, fallback, error, card, and streaming-card replies; keep mention context bounded and injection-safe; and either preserve a deliberate explicit-mention conversion path for card/streaming replies or document that explicit notification is intentionally no longer supported from mention-forward context. Regression coverage should include plain text, card, streaming-card close, and fallback/error paths.
