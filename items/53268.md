## Summary

Current `main` still does not provide a configurable URL or hostname deny list for `web_fetch`. The typed config, strict runtime schema, generated schema, runtime web-fetch wiring, and shared SSRF policy only expose existing fetch settings plus RFC2544/allowlist-style SSRF behavior, so the requested selective public-host deny policy remains missing.

## Best Possible Solution

Keep this issue open and implement a typed, documented web-fetch host deny-list if maintainers want the behavior. A conservative implementation would add `tools.web.fetch.denyHosts`, validate it in strict runtime and generated schemas, support exact and `*.domain` matching case-insensitively, enforce it before DNS and on redirects/provider fallback, return concise blocked-fetch errors, and add focused config/runtime tests.
