## Summary

Keep open. Current `main` still contains the reported lock-write resource leaks in both TypeScript paths, so this is not implemented on main, and the PR as provided also has an unresolved pipe-cleanup hole in `scripts/docs-i18n/pi_rpc_client.go`.

## Evidence

- **Gateway lock leak still present on main:** `src/infra/gateway-lock.ts:274-285` still does `const handle = await fs.open(lockPath, "wx")` followed by `await handle.writeFile(...)` and only exposes `handle.close()` inside the returned `release()` closure. If `writeFile()` throws, control goes to the outer `catch` and the handle is never closed.
  - file: [src/infra/gateway-lock.ts:274](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/infra/gateway-lock.ts#L274)
  - command: `nl -ba src/infra/gateway-lock.ts | sed -n '246,340p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Plugin SDK file lock leak still present on main:** `src/plugin-sdk/file-lock.ts:185-190` has the same pattern: `fs.open(lockPath, "wx")`, then `handle.writeFile(...)`, then the handle is stored in `HELD_LOCKS` only on success. A `writeFile()` failure leaves the file descriptor untracked and unclosed.
  - file: [src/plugin-sdk/file-lock.ts:185](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/plugin-sdk/file-lock.ts#L185)
  - command: `nl -ba src/plugin-sdk/file-lock.ts | sed -n '166,255p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Current main also lacks pipe cleanup in docs PI client init:** `scripts/docs-i18n/pi_rpc_client.go:93-114` acquires `stdin`, `stdout`, and `stderr` sequentially and returns immediately on later failures without closing previously acquired pipes. That matches the extra scope added in the PR, but it is not fixed on main either.
  - file: [scripts/docs-i18n/pi_rpc_client.go:93](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/scripts/docs-i18n/pi_rpc_client.go#L93)
  - command: `nl -ba scripts/docs-i18n/pi_rpc_client.go | sed -n '88,145p'`
