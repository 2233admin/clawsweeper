## Summary

Keep this issue open. Current main still lacks `--thread-id` on `openclaw cron add/create` and `openclaw cron edit`; Telegram topic delivery is only available through manual `:topic:` target encoding or lower-level/thread-aware paths. PR #60890 remains the open implementation candidate and explicitly links itself to this issue, so maintainers should review, land, or close that PR rather than close the issue now.

## Best Possible Solution

Keep this issue open while PR #60890 is reviewed. The best path is a focused cron CLI change that adds `--thread-id` to `cron add/create` and `cron edit`, composes Telegram topic targets as `<chatId>:topic:<threadId>`, preserves existing manual topic targets, rejects invalid combinations early, and adds regression coverage in `src/cli/cron-cli.test.ts`.
