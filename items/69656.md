## Summary

Keep open. Current `main` still does not expose QQ Bot proxy settings, and the QQ Bot gateway opens a bare `ws` client without a proxy agent. The upstream `ws` docs say proxy connections require a custom `http.Agent`, so the reported need is still plausible on current `main`. ([github.com](https://github.com/websockets/ws))
