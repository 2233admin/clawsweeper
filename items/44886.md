## Summary

Current main still accepts Telegram DM `direct.*.tools` in config, but the mounted tool-policy enforcement path remains group/channel-only. The related #53530 is useful context but has not been incorporated into this checkout and was explicitly described as not adapting Telegram's legacy `direct.*` surface by itself.

## Best Possible Solution

Keep this issue open. The best maintainer path is to adapt Telegram's legacy `channels.telegram.accounts.<account>.direct[peer] ?? direct["*"]` surface onto the shared direct-session tool-policy resolver, enforce both `tools` and `toolsBySender` for normal DM runs and native-command target sessions, preserve explicit-over-wildcard and account-scoped precedence, and add focused Telegram regression tests for mounted tool inventory, direct topics/thread fallback, and sender-scoped overrides.
