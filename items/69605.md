## Summary

Current `main` does not provide high-confidence evidence that #69605 is fixed. The message-level copy control is still rendered as an absolute top-right bubble action, the code-block copy control is still a separate top-right control inside markdown-rendered blocks, and the only apparent spacing guard (`.chat-bubble.has-copy`) is not wired up anywhere in the UI code. There is also no interaction test covering which control wins when these regions overlap, so this is not safe to close as implemented or cannot-reproduce.
