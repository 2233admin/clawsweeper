## Summary

Current main still matches the reported bug path. Assistant bubbles render the message-level copy/expand action bar, but the bubble class list does not include the existing `has-copy` spacing class, so the absolute-positioned message-level action can still overlap the code-block Copy control. The related open PR #69634 directly tracks the likely fix; it has not landed.

## Best Possible Solution

Keep #69605 open and land a tested version of PR #69634 or an equivalent Control UI patch: add `has-copy` to assistant chat bubbles whenever copy/expand actions render, and add a `ui/src/ui/chat/grouped-render.test.ts` regression assertion that assistant bubbles with markdown actions receive the class while bubbles without actions do not.
