## Summary

Keep #69605 open. Current main still renders assistant bubbles with floating message-level copy actions but does not apply the existing `has-copy` spacing class, so the reported overlap/mis-click path remains plausible. The related PR #69634 directly tracks the likely fix but is still open and unmerged; closed PRs #69635 and #69689 did not land on main.

## Best Possible Solution

Keep #69605 open and use it as the canonical bug tracker until a fix lands. The best current path is to merge or rework the #69634-style change: add `has-copy` to assistant bubbles when copy/expand actions render, and add a focused `grouped-render` regression test that locks the spacing contract for assistant messages while leaving user bubbles unchanged.
