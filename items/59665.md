## Summary

Keep open. Current main still writes a curated/minimal service PATH for the macOS LaunchAgent rather than capturing the current login-shell PATH at install time. The issue discussion already correctly re-scoped the likely root cause to `buildMinimalServicePath()` / LaunchAgent environment generation, and the reporter's workaround still describes missing shell PATH entries such as `~/.bin` and `~/.cargo/bin` that current main does not add by default.

## Best Possible Solution

Keep the issue open and re-scope it around macOS LaunchAgent service PATH policy. Maintainers should decide whether `openclaw gateway install` should capture a sanitized login-shell PATH at install time, or keep the minimal heuristic and extend/document it. A concrete fix likely belongs in `src/daemon/service-env.ts` and `src/commands/daemon-install-helpers.ts`, with launchd/service-env tests for `~/.bin`, `~/.cargo/bin`, and reinstall behavior after uninstall.
