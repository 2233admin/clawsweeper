## Summary

Keep open. Current `main` at `aa3a5f9ff7907c2a41d6259d24b529bda41f293a` still does not implement the PR’s behavior: `isRecurringTopOfHourCronExpr` only treats a minute field equal to `"0"` as top-of-hour, so step/list/range minute expressions that fire at minute 0 are still missed.

## Evidence

- **Current implementation still uses strict equality:** `isRecurringTopOfHourCronExpr` returns true only when the 5-field minute is exactly `"0"`, and in 6-field form when both seconds and minutes are exactly `"0"`. That does not cover `*/15`, `0,30`, `0-5`, or similar minute fields.
