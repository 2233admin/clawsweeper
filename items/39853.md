## Summary

Keep open. Current main still does not expose structured non-self Feishu @mentions in inbound conversation metadata, and the PR addresses a real multi-bot routing gap. The patch should not be closed as stale, but it needs maintainer work before merge because Feishu mention IDs can be `user_id`/`union_id` rather than only `open_id`, and there is no regression coverage yet.

## Best Possible Solution

Keep #39853 open. The best maintainer path is to implement a robust Feishu mention extraction helper that excludes the receiving bot, falls back across `open_id`, `user_id`, and `union_id` where appropriate, adds a typed structured-mentions metadata contract, and includes focused tests for inbound metadata serialization and Feishu ID fallback. Reconcile the metadata approach with the broader sibling mention arbitration work in #38939 before merging either path.
