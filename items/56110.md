## Summary

Keep open. Current `main` does not support `STATE.md` as a workspace bootstrap file, and the existing hook/plugin extension point only accepts the current fixed bootstrap basename set. This is still a coherent core feature request, not a conservative cleanup close.

## Evidence

- **Bootstrap filename contract is fixed:** `WorkspaceBootstrapFileName` and `VALID_BOOTSTRAP_NAMES` only include `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, and `MEMORY.md`. `STATE.md` is not part of the current runtime contract.
  - file: [src/agents/workspace.ts:130](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/workspace.ts#L130)
  - command: `nl -ba src/agents/workspace.ts | sed -n '130,190p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Auto-load enumeration excludes STATE.md:** `loadWorkspaceBootstrapFiles()` hard-codes the same eight workspace files for injection/missing-file handling. A new `STATE.md` would require a core change here.
  - file: [src/agents/workspace.ts:602](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/workspace.ts#L602)
  - command: `nl -ba src/agents/workspace.ts | sed -n '602,760p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **User-facing docs match the fixed set:** The system prompt docs list the injected bootstrap files and do not include `STATE.md`, so current documented behavior matches the code rather than the issue request.
  - file: [docs/concepts/system-prompt.md:99](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/concepts/system-prompt.md#L99)
  - command: `nl -ba docs/concepts/system-prompt.md | sed -n '96,140p'`

- **Existing hook path does not solve this request:** The bundled `bootstrap-extra-files` hook is documented for extra paths of recognized bootstrap files only, and the loader rejects unsupported basenames. That means this is not already addressable as a plugin/hook-only workaround for `STATE.md`.

## Risks / Open Questions

- `docs/concepts/system-prompt.md:134` says sub-agent sessions only inject `AGENTS.md` and `TOOLS.md`, but `src/agents/workspace.ts:669` currently allows `SOUL.md`, `IDENTITY.md`, and `USER.md` for subagent/cron filtering too. Bootstrap-scope docs and code are slightly out of sync.
- The issue is recent (2026-03-28) and coherent, so closing it now would be premature even if maintainers later decide to solve it through a broader bootstrap-config design such as the cross-referenced open issue #59050.
