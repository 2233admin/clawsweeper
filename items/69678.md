## Summary

Keep #69678 open. Current main still has the reported 2000-character hard cap in the isolated cron summary path, and the cron run-log config/schema/docs still expose only `maxBytes` and `keepLines`. The provided related items cover other cron UI/output/status issues, not a canonical fix for configurable run-log summary length.

## Best Possible Solution

Keep the issue open. A good fix would add a first-class `cron.runLog.maxSummaryChars` config field, validate and document it, plumb it into the isolated cron summary selection or run-log append path, preserve existing pruning by `maxBytes`/`keepLines`, and add regression tests for the default and configured limits.
