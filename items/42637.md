## Summary

Keep this PR open. Current main still drops the omitted suffix of prompt-visible skills and emits only the generic truncation count, while this contributor PR is a focused implementation candidate for exposing omitted skill names and recovery guidance. It is not safe maintainer cleanup to close because the central bug remains on main, the PR is active, and the changed surface is model-facing prompt content that still needs maintainer review rather than stale closure.

## Best Possible Solution

Keep this PR open. The best path is to review the current contributor patch, require/keep coverage for both count-based and character-budget truncation, verify the final emitted prompt respects `maxSkillsPromptChars`, and settle the safe rendering policy for omitted names so they remain usable with `openclaw skills info` without introducing prompt-injection or privacy regressions.
