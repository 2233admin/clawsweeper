## Summary

Keep #69423 open. Current main still has the reported failure shape: the Anthropic transport converts every tool by reading `tool.parameters.properties` without a missing-parameters guard or `inputSchema` fallback, while plugin tool registration/resolution still accepts and forwards returned tools without schema validation. The related fix PR #69432 was closed unmerged, and no equivalent regression coverage appears on main.

## Best Possible Solution

Keep the issue open and land a focused core fix in `src/agents/anthropic-transport-stream.ts` or an earlier shared tool-normalization seam: normalize `parameters ?? inputSchema`, fall back to an empty object schema or skip/log the bad tool without aborting the run, preserve plugin/tool identity in diagnostics where available, and add regression tests covering malformed third-party plugin tools on Anthropic.
