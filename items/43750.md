## Summary

Current main partially improves native-command routing, but the requested explicit `/new @AgentName` or `/reset @AgentName` targeting is still missing. Discord native commands resolve the target agent from configured route/binding state, not from a command argument or agent mention, and the public slash-command docs still describe `/new` arguments as model/message input rather than agent selection.
