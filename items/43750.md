## Summary

Keep this issue open. Current main has deterministic Discord native-command routing through bindings, bound sessions, and CommandTargetSessionKey, but it still does not implement explicit `/new @AgentName` or `/reset @AgentName` targeting for shared multi-agent Discord channels.

## Best Possible Solution

Keep the issue open and implement a focused Discord native-command enhancement: add a supported target-agent option or mention/name parser for `/new` and `/reset`, resolve it through existing agent routing and access rules, reset only the selected target session, and add regression tests for shared guild channels, configured bindings, bound ACP sessions, unauthorized target attempts, and the no-target default behavior. Treat smart defaults such as highest-context or active-participant selection as a separate product decision if maintainers want that behavior.
