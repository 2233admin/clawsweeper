## Summary

Current `main` already covers parts of the request: docs now explain that tool outputs count toward context, Control UI shows context pressure with a compact action, compaction can notify users, and pruning has several config knobs. But there is still no distinct user-facing pruning preview/log or pruning-specific notification surface, so the feature request is only partially addressed and should stay open.

## Evidence

- **Tool outputs are explicitly documented as part of context:** The context docs now call out tool calls/results, attachments, and pruning artifacts as part of the context window, and explain that pruning only affects the in-memory prompt.
  - file: [docs/concepts/context.md:81](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/docs/concepts/context.md#L81)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Session pruning is documented and configurable:** The gateway config docs expose `agents.defaults.contextPruning` with knobs for TTL, preserved assistant count, soft/hard trim ratios, placeholder text, and per-tool allow/deny matching.
  - file: [docs/gateway/config-agents.md:576](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/docs/gateway/config-agents.md#L576)
