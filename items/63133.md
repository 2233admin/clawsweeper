## Summary

Keep open. Current main still lacks checked-vs-skipped memory collection state, so the lean `openclaw status --json` fast path can only report skipped memory inspection as `memory: null`, and the renderer still labels enabled-null memory as `unavailable`. The latest provided PR patch remains a focused implementation candidate and addresses the earlier review concern by preserving `unavailable` for attempted probe failures.

## Best Possible Solution

Keep this PR open for maintainer review and CI. The best fix is to land this focused checked/skipped memory-collection state, or reimplement the same behavior on main: lean `status --json` without `--all` should report skipped memory collection as `not checked`, while attempted probe failures and unavailable memory runtimes should continue to render as `unavailable`.
