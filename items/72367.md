## Summary

Keep #72367 open. Current main still has the bug the PR targets: bundled LSP servers are spawned without a detached process group, dispose only calls `session.process.kill()` on the immediate child, and Gateway shutdown only sweeps bundled MCP runtimes. The PR is a focused fix for linked bug #72357, with useful regression-test coverage in the provided diff, though the Greptile P2 note about sequential MCP/LSP shutdown grace should be considered before merge.

## Best Possible Solution

Keep the PR open and either merge a polished version of its approach or ask the author to update it: spawn bundled LSP servers as disposable process groups, dispose them through `killProcessTree`, maintain an idempotent active-session sweep for Gateway shutdown, and add the LSP runtime plus Gateway shutdown regression tests. Addressing the sequential disposal latency comment would make the shutdown path cleaner before landing.
