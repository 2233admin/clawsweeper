## Summary

Keep open. Current `main` still persists and reuses `sessionTarget:"current"`/`session:<id>` without any direct-message guard, and there is no evidence this PR's fix landed elsewhere on `main`. The reported bug remains plausible and the PR still addresses a real gap.
