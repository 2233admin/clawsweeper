## Summary

Keep open: current `main` already lets Control UI inspect non-webchat sessions by session key, but there is no implemented channel-based separated/merged view toggle or unified cross-channel conversation timeline matching the request.

## Evidence

- **Control UI docs list chat and sessions, not a channel-view toggle:** The Control UI docs describe Chat (`chat.history`, `chat.send`, etc.) and a Sessions panel (`sessions.list`, `sessions.patch`) as current capabilities, but do not describe any separated/merged channel conversation mode.
  - file: [docs/web/control-ui.md:105](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/web/control-ui.md#L105)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Sessions rows already deep-link any stored session into Chat:** The Sessions view builds a chat URL for each non-global session and routes clicks to `onNavigateToChat(row.key)`, so individual channel/session transcripts are already inspectable from Control UI.
  - file: [ui/src/ui/views/sessions.ts:464](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/ui/src/ui/views/sessions.ts#L464)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Chat history loads the currently selected session key:** The chat controller calls `chat.history` with the active `sessionKey`, confirming the UI can display whichever specific session is selected rather than only a hard-coded main webchat transcript.
  - file: [ui/src/ui/controllers/chat.ts:152](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/ui/src/ui/controllers/chat.ts#L152)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Session UI already recognizes channel-scoped direct/group keys:** `parseSessionKey` explicitly recognizes `agent:<agent>:<channel>:direct:*` and `agent:<agent>:<channel>:group:*` keys, and `resolveSessionOptionGroups` turns session rows into selectable chat options.
  - file: [ui/src/ui/chat/session-controls.ts:362](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/ui/src/ui/chat/session-controls.ts#L362)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Gateway session listing includes all stored sessions plus channel metadata:** `sessions.list` reads the combined gateway session store, and `buildGatewaySessionRow` attaches `channel`, `subject`, `groupChannel`, `space`, and a derived `displayName` for each row. That supports per-channel session inspection, but it is still session-by-session rather than a merged timeline toggle.
  - file: [src/gateway/server-methods/sessions.ts:584](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/gateway/server-methods/sessions.ts#L584)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **No implementation for merged/separated channel timeline mode found:** A repo search for `merged view`, `separated view`, `unified timeline`, `channel-based`, `session view mode`, and similar terms found no matching Control UI implementation; only unrelated hits remained.
  - command: `rg -n "merged view|separated view|unified timeline|channel-based|session view mode|all channel conversations|each channel conversation independently" ui/src ui docs src`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
