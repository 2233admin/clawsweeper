## Summary

Keep open. Current main still strips credential-shaped keys from admin-configured `sandbox.docker.env`, so the PR’s central sandbox env bypass is not implemented or obsolete. The PR also remains unsafe to merge as-is because its skill-filter change can bypass `skills.entries.<key>.enabled === false` and bundled-skill policy safeguards.

## Best Possible Solution

Keep the PR open for focused follow-up. The best path is to narrow it to the sandbox env bug: add a `forceAllowKeys`-style bypass only for explicitly configured `sandbox.docker.env` at Docker container creation, add tests for configured API-key-shaped env names passing through while process/unapproved credentials remain blocked, and remove or redesign the skill eligibility bypass so `enabled: false` and bundled-skill policy still win.
