## Summary

Keep open. Current main still has no admin/configured-key escape hatch in sandbox env sanitization, so credential-shaped keys in configured Docker sandbox env are still blocked at container creation. The PR addresses a real gap, but it should be revised before merge because the submitted diff still mixes in an unrelated skill eligibility bypass and has no regression tests for the sandbox env behavior.

## Best Possible Solution

Keep this PR open for a focused revision. Narrow it to the Docker sandbox env problem: add a configured-key escape hatch to `sanitizeEnvVars`, apply it only to keys explicitly present in resolved `sandbox.docker.env` when building Docker create args, preserve value validation and host/process-env blocking, add sanitizer and Docker create-args regression tests, and drop the unrelated skill eligibility bypass unless a separate maintainer-approved skill policy change is designed.
