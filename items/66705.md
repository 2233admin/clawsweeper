## Summary

Current main still does not expose `OPENCLAW_SESSION_KEY` or `OPENCLAW_AGENT_ID` to exec child-process environments. The trusted session context reaches the exec tool, but local, PTY, sandbox, and node-host env construction still only documents and injects existing runtime markers. Related PR #68859 remains an open partial implementation candidate for `OPENCLAW_SESSION_KEY`, not a shipped fix for both requested variables.

## Best Possible Solution

Keep this issue open for a focused core exec-runtime change. Add a shared trusted runtime-env helper that injects both `OPENCLAW_SESSION_KEY` and `OPENCLAW_AGENT_ID` after host/sandbox env construction and sanitization, pass trusted `agentId` through the local runtime path, forward equivalent env for node-host execution, update docs for the runtime env contract, and add regression tests covering stale inherited/requested values plus gateway, PTY, sandbox/backend, Docker, and node paths. Review PR #68859 as a starting point, but broaden it before considering the issue resolved.
