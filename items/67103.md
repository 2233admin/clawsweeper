## Summary

Keep this PR open. Current main still does not implement Control UI LaTeX/KaTeX/MathJax rendering, #42840 remains the canonical open feature request, and this PR is still a plausible implementation candidate. It should not land as-is because the provided PR-head review context shows merge-blocking functional regressions and security-sensitive markdown sanitizer regressions.

## Best Possible Solution

Keep this PR open as a rework candidate for #42840. The best path is to update it against current main, preserve the markdown cache and DOMPurify link-hardening hooks, add robust delimiter parsing that avoids currency and code-span/code-fence false positives, narrow any KaTeX-required style allowance, include the lockfile/dependency review for KaTeX, and add focused tests for inline math, display math, invalid LaTeX, mixed markdown, currency text, and backtick-delimited code.
