## Summary

Keep open. Current `main` still contains the sidebar-group visibility logic that forces any group containing the active tab to stay expanded, which matches the reported behavior where the chevron rotates but child items remain visible. The latest release `v2026.4.22` contains the same logic, so there is no strong evidence this was fixed on `main` or shipped.

## Evidence

- **Current main keeps active groups visible even when collapsed:** `showItems` is computed as `navCollapsed || hasActiveTab || !isGroupCollapsed`. When the clicked group contains the active tab, `hasActiveTab` stays true, so the section never receives the collapsed class.
  - file: [ui/src/ui/app-render.ts:1310](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/ui/app-render.ts#L1310)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '1304,1338p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Collapsed state only hides items via CSS class:** The child items are hidden only when `.nav-section--collapsed` is present. Because `showItems` stays true for the active group, this CSS never applies and the children remain visible.
  - file: [ui/src/styles/layout.css:496](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/styles/layout.css#L496)
