## Summary

Keep open. Current main has partial mitigation for leaked tool-call scaffolding on the agent-delivery path, but the shared visible-text sanitizer still does not recognize `<function_response>` and the `chat.history`/UI path shown to users does not apply that sanitizer. I do not have strong evidence that issue #47444 is fully fixed on `main`.

## Evidence

- **Shared sanitizer strips `function_calls` but not `function_response`:** The XML-stripper allowlist covers `tool_call`, `tool_result`, `function_call`, `function_calls`, `function`, and `tool_calls`; there is no `function_response` case, so the exact wrapper shown in the report is not handled by this sanitizer.

## Risks / Open Questions

- Some user-facing delivery surfaces may already be protected, so the leak may now be surface-specific rather than universal.
- I did not run a live reproduction because this review was read-only; the conclusion is based on current code and test inspection.
- Docs currently claim `chat.history` strips plain-text tool-call XML, but the inspected implementation does not obviously match that claim for all tag forms, especially `function_response`.
