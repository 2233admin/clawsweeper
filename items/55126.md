## Summary

Current main still accepts and resolves `memorySearch.outputDimensionality`, but the bundled OpenAI memory embedding path does not pass it as the OpenAI `dimensions` field for inline or batch embeddings. The linked implementation candidate #55507 remains open and unmerged with closing syntax for this issue, so this issue should stay open until that PR or a replacement patch lands.

## Best Possible Solution

Keep this issue open. Review and update #55507, or replace it with a current-main patch that threads `memorySearch.outputDimensionality` into the OpenAI embedding client, sends OpenAI `dimensions` for inline and batch `/v1/embeddings` requests when supported, includes the dimension in runtime cache-key/reindex state, and adds regression tests for request bodies and cache behavior.
