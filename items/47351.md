## Summary

Keep open. Current `main` partially addresses the report by rendering inline `data:image/...;base64,...` markdown images, but remote `http(s)` markdown images are still intentionally blocked in the Control UI. The issue is therefore not fully implemented on `main`, and it is too recent for stale closure.

## Evidence

- **Remote markdown images are still flattened instead of rendered:** `md.renderer.rules.image` only emits `<img>` when the source matches `^data:image/...;base64,`; otherwise it returns escaped alt text. The adjacent tests still assert that `https://example.com/img.png` does not render as `<img>`.
  - file: [ui/src/ui/markdown.ts:409](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/ui/src/ui/markdown.ts#L409)
