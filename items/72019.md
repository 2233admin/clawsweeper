## Summary

Keep #72019 open. Current main still duplicates the native subagent task text: `sessions_spawn` passes `task` into `spawnSubagentDirect`, the subagent system prompt embeds it under `You were created to handle`, and the first child user message repeats it under `[Subagent Task]` while both are sent in the same gateway `agent` call. The linked ClawHub skill is a useful workaround/pattern reference, but this is still a current OpenClaw core behavior bug. Related PRs #72028 and #72053 are plausible fixes but are open/unmerged in the provided context; #72026 is closed unmerged.

## Best Possible Solution

Keep #72019 open until one canonical PR lands. The best fix is to single-source the native subagent task text across `buildSubagentSystemPrompt` and the initial child user message, preserve multiline task formatting, and add a focused regression test that inspects both `extraSystemPrompt` and the first gateway `message`.
