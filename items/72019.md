## Summary

Current main still matches the reported structural duplication: `sessions_spawn` passes the same task into the subagent system prompt and into the first child user message. The linked ClawHub skill/comment is a useful workaround for orchestration style, but it does not make this a ClawHub-only request because the duplication is in OpenClaw core prompt assembly. The related fix PRs #72026, #72028, and #72053 are open attempts, not shipped on main, so the issue should remain open as the canonical bug tracker.

## Best Possible Solution

Keep #72019 open until one focused fix lands on main. The best fix is to make the long task appear in exactly one place in the child run payload, either by removing it from `buildSubagentSystemPrompt` or by replacing the first user turn with a short bootstrap reference, then add tests that inspect the actual `callSubagentGateway({ method: "agent" })` payload and prove the task appears once across `message` plus `extraSystemPrompt`.
