## Summary

Keep open. Current main still appears to have the reported Telegram group activation gap: captionless photos become `<media:image>`, but mention detection for `requireMention` groups checks only text/caption or an audio preflight transcript. That means a group configured with `mentionPatterns: [".*"]` still has no text to match for a photo-only message, while adding a caption makes the same media pass. The existing maintainer comment’s root-cause suspicion is consistent with the current code, and related #44908 is still open and tracks a different WhatsApp/audio surface rather than a canonical Telegram-photo fix.

## Evidence
