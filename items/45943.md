## Summary

Keep open. Current `main` has the adjacent fix for `openclaw update` and installer-driven upgrades, but direct manual package-manager upgrades (`npm i -g openclaw@latest`) still require a separate gateway restart and there is no package postinstall hook that refreshes/restarts the gateway service automatically.

## Evidence

- **Manual npm update path still documents a separate restart:** `docs/install/updating.md` recommends `openclaw update` because it restarts the gateway automatically, but the manual npm/pnpm/bun section still shows raw package-manager commands and the later 'After updating' section still requires `openclaw gateway restart`. That means the direct `npm i -g openclaw@latest` path is not yet automatic.
  - file: [docs/install/updating.md:11](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/docs/install/updating.md#L11)

- **`openclaw update` already implements the reporter's suggested Option 1:** The CLI docs say `--no-restart` exists and that restart is the default, and they describe package-manager installs going through `openclaw update`'s package flow. This covers the 'enhance `openclaw update`' part of the issue, but not plain external npm installs.
  - file: [docs/cli/update.md:33](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/docs/cli/update.md#L33)

A search of that script finds no `gateway install` or `gateway restart` logic, so raw `npm i -g openclaw@latest` still has no automatic gateway refresh/restart path.

## Risks / Open Questions

- The issue is partially obsolete because `openclaw update` now does the recommended auto-refresh/restart work, so maintainers may want to narrow the scope to direct external npm/pnpm/bun upgrades only.
- Installer-driven upgrades also refresh and restart the gateway, which may reduce urgency but does not satisfy the reported `npm i -g` workflow.
