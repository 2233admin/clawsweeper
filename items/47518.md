## Summary

Keep open. Current `main` does not implement the two core fixes described by PR #47518, so it should not be closed as `implemented_on_main` or `cannot_reproduce`. The Gemini CLI OAuth path still calls guarded fetch without `trusted_env_proxy`, and dispatcher teardown still prefers `close()` over immediate `destroy()`. The PR also remains too broad for conservative cleanup because the submitted diff spills into unrelated Discord/LINE/Nostr/plugin-sdk surfaces instead of a narrow Gemini/proxy fix.
