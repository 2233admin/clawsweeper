## Summary

Keep open. Current main still treats billing failures as profile-wide disabled windows, not model-scoped cooldowns, and the documented model-scoped behavior applies only to rate-limit cooldowns. The related #66314/#66315 discussion fixed billing error messaging for fallback summaries, not the LiteLLM/proxy behavior reported here; #70903 tracks persistent billing recovery but does not cover the proxy-specific model-scoping request.

## Best Possible Solution

Add a proxy-aware billing cooldown scope. For LiteLLM and similar OpenAI-compatible aggregator providers, a billing-classified failure should either record a model-scoped billing disable/cooldown or support an explicit config opt-out so sibling models behind the same auth profile remain usable. Keep direct-provider billing failures profile-wide by default, and update `auth-profiles` usage state, model fallback skip/probe tests, config schema/help, and LiteLLM/model-failover docs.
