## Summary

Keep open. Current `main` still has the exact fallback this PR is trying to remove, and it still lacks the Docker-specific repair path for missing Python in sandbox mutation helpers.

## Evidence

- **Default image still silently aliases plain Debian:** `ensureDockerImage()` on current `main` still pulls `debian:bookworm-slim` and tags it as `openclaw-sandbox:bookworm-slim` when the default image is missing, instead of failing fast.
  - file: [src/agents/sandbox/docker.ts:289](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/sandbox/docker.ts#L289)
  - command: `nl -ba src/agents/sandbox/docker.ts | sed -n '275,300p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Default sandbox image contract includes Python:** The actual sandbox image built by `scripts/sandbox-setup.sh` installs `python3`; plain Debian does not satisfy that contract.
  - file: [Dockerfile.sandbox:3](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/Dockerfile.sandbox#L3)
  - command: `nl -ba Dockerfile.sandbox | sed -n '1,24p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Pinned mutation helper requires Python:** The mutation helper probes for `python3`/`python` and emits `sandbox pinned mutation helper requires python3 or python` before exiting 127 if none is present.
  - file: [src/agents/sandbox/fs-bridge-mutation-helper.ts:9](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/sandbox/fs-bridge-mutation-helper.ts#L9)
  - command: `nl -ba src/agents/sandbox/fs-bridge-mutation-helper.ts | sed -n '9,18p;301,315p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **No Docker-specific repair rewrite on current main:** `fs-bridge.ts` still forwards Docker backend command failures directly; there is no catch/rewrite for missing-Python mutation-helper failures.
  - file: [src/agents/sandbox/fs-bridge.ts:219](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/sandbox/fs-bridge.ts#L219)

## Risks / Open Questions

- Closing this PR would drop a still-relevant fix: current `main` can still create an incompatible default-tagged sandbox image when the default image is missing.
