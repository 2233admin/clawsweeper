## Summary

Keep open. Current main still has TTS backend/config/CLI support and a per-message browser Read aloud button, but not the requested first-class Control UI Settings panel for enablement, provider selection, default voice selection, or sample playback. The existing maintainer-thread comment also explicitly kept this item open, and recent changes after that review did not add the requested UI surface.

## Best Possible Solution

Keep the issue open and implement a first-class Control UI Settings Text-to-Speech section. It should use existing `tts.status`, `tts.providers`, `tts.enable`, `tts.disable`, `tts.setProvider`, and `tts.convert` where possible; show configured versus unavailable providers; filter voices by provider; add sample playback; and define/test a safe default voice persistence path that coordinates with `messages.tts` and local TTS prefs.
