## Summary

Keep PR #70229 open. The linked bug #43996 still maps to a real current-main gap: Docker sandbox creation unconditionally adds `--security-opt no-new-privileges`, and current config/types/schema/docs do not expose `sandbox.docker.noNewPrivileges` or dangerous-config reporting for the weaker opt-out. The PR is a focused, plausible fix with tests/docs, and the bot review gap was addressed in the later PR commit per the provided PR context.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. The right implementation belongs in the Docker sandbox config surface: add `sandbox.docker.noNewPrivileges` with default `true`, honor agent/global/shared-scope precedence, omit the Docker flag only when explicitly false, update strict config schema/generated labels/docs, and report `noNewPrivileges=false` as a dangerous configuration. The linked bug #43996 provides concrete reproduction evidence for the compatibility need.
