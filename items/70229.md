## Summary

Keep #70229 open. Current main still unconditionally adds Docker `--security-opt no-new-privileges` for sandbox containers and does not expose `sandbox.docker.noNewPrivileges` in the config type, resolver, Zod schema, docs, or dangerous-config reporting. The PR is a focused fix for the still-open linked bug #43996, and the provided review context shows the only bot coverage concern was addressed by a follow-up commit.

## Best Possible Solution

Keep the PR open and have maintainers review or land the focused implementation after normal validation. The best product path is a secure default of `noNewPrivileges: true`, an explicit documented opt-out for known-incompatible Docker hosts, config schema/type/docs alignment, dangerous-config reporting for `false`, and regression coverage for global, per-agent, and shared-scope precedence.
