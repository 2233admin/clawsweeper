## Summary

Keep this PR open. Current main still does not implement the PR's central cron timeout-cause behavior: failure alerts omit the classified cause, run-log entries and the gateway run-log schema omit `errorReason`, CLI JSON output is passed through unchanged, and generated Swift protocol models have no `errorReason` field. The PR remains a focused implementation candidate, and the latest author comment says the earlier enum-clamping and Swift initializer review concerns were addressed on the PR head.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to validate and, if checks remain clean, land the additive cron run-log `errorReason` protocol field, cause-first failure alert and CLI display, persisted run-log reason normalization/backfill, Swift initializer compatibility, and regression coverage together.
