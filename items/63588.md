## Summary

Keep open. Current main still strips Slack bearer auth on cross-origin Slack CDN redirects, which is the behavior this PR is trying to change. The PR is a focused Slack plugin bug-fix candidate for #63523, but it needs maintainer security review and rebase cleanup before merge.

## Best Possible Solution

Keep this PR open and update it as the implementation path for the Slack `download-file` redirect bug. The best fix belongs in `extensions/slack/src/monitor/media.ts`: preserve Authorization for Slack-owned HTTPS redirect targets such as `slack.com`, `slack-edge.com`, and `slack-files.com`, while keeping SSRF/pinned-DNS checks, redirect limits, HTML-login-page detection, and non-Slack token stripping or blocking. Add/keep regression tests for Slack CDN redirects retaining auth and external redirects not receiving auth.
