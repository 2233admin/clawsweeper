## Summary

Keep this PR open. Current main still lacks the proposed `api.runtime.llm.complete()` Plugin SDK surface and context-engine LLM helper, while the linked feature request remains open. The PR is a plausible implementation candidate, but it needs maintainer API/security review before landing because it grants plugins/context engines a new LLM-call capability and the final diff still leaves active-agent binding ambiguous for injected context-engine calls.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is an additive `api.runtime.llm.complete()` seam with exported SDK types, active-session agent binding or a clear explicit-agent contract, docs/API baseline updates, and focused tests for model/auth resolution. Keep #45865 as the canonical product/API discussion if the API shape changes.

## Risks / Open Questions

- The PR exposes a new way for plugin and context-engine code to initiate LLM calls with OpenClaw-managed model credentials, so maintainers need an explicit permission, cost, observability, and trust-boundary stance before merge.
- The final PR diff still defaults unbound completions to the config default agent unless `agentId` is supplied; context-engine injection does not appear to bind the active session agent automatically.
- The proposed public SDK surface needs docs, API/baseline coverage, and focused tests for default/non-default agent model and auth-profile resolution, including AWS SDK auth mode.
