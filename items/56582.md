## Summary

Keep open. Current `main` still contains the bug this PR fixes: `parseClawHubPluginSpec` treats trailing-`@` specs as valid names, so malformed ClawHub specs are not rejected early.

## Evidence

- **Current parser still accepts trailing-@ specs:** `src/infra/clawhub-spec.ts:16-23` returns `{ name: spec }` when `atIndex <= 0 || atIndex >= spec.length - 1`. For inputs like `clawhub:demo@` or `clawhub:@scope/pkg@`, `atIndex >= spec.length - 1` is true, so the malformed spec is accepted instead of returning `null`.
  - file: [src/infra/clawhub-spec.ts:16](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/infra/clawhub-spec.ts#L16)
  - command: `nl -ba src/infra/clawhub-spec.ts | sed -n '1,220p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Current tests do not cover the regression cases from the PR:** `src/infra/clawhub.test.ts:37-45` only asserts `clawhub:demo`, `clawhub:demo@1.2.3`, and non-prefixed `@scope/pkg`. It does not include `clawhub:@scope/pkg`, `clawhub:demo@`, `clawhub:@scope/pkg@`, or `clawhub:@scope/pkg@1.2.3`.
  - file: [src/infra/clawhub.test.ts:37](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/infra/clawhub.test.ts#L37)
  - command: `nl -ba src/infra/clawhub.test.ts | sed -n '1,140p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Malformed specs still flow past the parser and only fail later:** `src/plugins/clawhub.ts:736-742` only reports `invalid_spec` when `parseClawHubPluginSpec` returns null. Because trailing-`@` inputs currently return a name object, they proceed into package resolution and can still surface as downstream lookup failures instead of early invalid-spec rejection.
  - file: [src/plugins/clawhub.ts:736](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/plugins/clawhub.ts#L736)
