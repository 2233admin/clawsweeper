## Summary

Keep open. Current main mitigates the "silent empty string / no warning" part, but `gateway install --force` still intentionally drops manual `OPENCLAW_*` entries from an existing LaunchAgent environment, so the reported plist-loss behavior remains plausible for `OPENCLAW_TELEGRAM_BOT_TOKEN` even though non-`OPENCLAW_` custom vars are now preserved.

## Evidence

- **Install refresh reads old service env but strips `OPENCLAW_*` before reuse:** `runDaemonInstall` loads the existing service environment when the gateway is already loaded, then `mergeInstallInvocationEnv` explicitly skips `HOME`, `PATH`, `TMPDIR`, and any key starting with `OPENCLAW_`. That means a manually added `OPENCLAW_TELEGRAM_BOT_TOKEN` will not be carried into the refresh/update path.
  - file: [src/cli/daemon-cli/install.ts:27](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cli/daemon-cli/install.ts#L27)
  - command: `nl -ba src/cli/daemon-cli/install.ts | sed -n '1,190p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Plist rewrite preserves only non-`OPENCLAW_` custom vars plus durable managed env:** `buildGatewayInstallEnvironment` preserves existing service env vars only after `collectPreservedExistingServiceEnvVars`, which also drops any `OPENCLAW_*` key. Tests cover preserving non-OpenClaw vars like `GOBIN`/`BLOGWATCHER_HOME`, but not OpenClaw-prefixed custom vars.
  - file: [src/commands/daemon-install-helpers.ts:178](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/commands/daemon-install-helpers.ts#L178)
  - command: `nl -ba src/commands/daemon-install-helpers.ts | sed -n '166,320p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Current tests confirm partial preservation only:** The daemon-install tests assert that safe custom vars survive and PATH is merged, while managed/dangerous keys are dropped. This matches `TELEGRAM_BOT_TOKEN_FAMILY` surviving but does not cover the report's `OPENCLAW_TELEGRAM_BOT_TOKEN` example.
  - file: [src/commands/daemon-install-helpers.test.ts:381](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/commands/daemon-install-helpers.test.ts#L381)

- **Missing `${VAR}` handling is improved but separate from plist preservation:** Config read now collects missing env-var references as warnings and preserves the placeholder instead of silently substituting an empty string. The env-substitution tests cover that behavior, so the warning aspect is mitigated even though the LaunchAgent env-loss path is not fully solved.
