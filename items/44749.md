## Summary

Keep open. Current main still writes allow-always allowlist updates from the approval handler's captured approvals snapshot, so concurrent handlers can still overwrite each other's entries. The related PR #52403 described the right class of fix, but the provided GitHub context shows it was closed unmerged, and the current code does not include its reload-under-lock behavior.

## Evidence

- **addAllowlistEntry still writes caller snapshot:** `addAllowlistEntry` takes an `ExecApprovalsFile` object supplied by the caller, mutates `approvals.agents`, and calls `saveExecApprovals(approvals)` without reloading the latest file or taking a lock around the read-modify-write merge.
