## Summary

Keep open. Current `main` still wires the Control UI `New session` button directly to `/new` with draft restoration and no confirmation step, so the reported accidental-reset risk remains.

## Evidence

- **New session button still exists in chat run controls:** `renderChatRunControls` renders a `New session` button and binds its click directly to `props.onNewSession`; there is no dialog, confirm call, or danger-state handling in this component.
  - file: [ui/src/ui/chat/run-controls.ts:24](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/ui/src/ui/chat/run-controls.ts#L24)
