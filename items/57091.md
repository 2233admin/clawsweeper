## Summary

Current `main` still silently drops malformed workspace `SKILL.md` files, and there is no built-in `skills validate` command or surfaced parse-error reporting in the CLI or gateway-backed skills status path.

## Evidence

- **Loader drops invalid skills silently:** `loadSingleSkillDirectory()` returns `null` when `parseFrontmatter()` throws and also returns `null` when `description` is missing, without recording or surfacing any error. That means malformed or incomplete workspace skills are omitted from later status/list output.
  - file: [src/agents/skills/local-loader.ts:44](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/skills/local-loader.ts#L44)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Frontmatter parser does not surface fence/YAML failures:** `parseFrontmatterBlock()` returns `{}` when the file does not start with `---` or lacks a closing fence, and falls back instead of emitting a parse error object. Current parser behavior does not preserve a diagnostic the CLI/UI could show.
  - file: [src/markdown/frontmatter.ts:183](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/markdown/frontmatter.ts#L183)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **No `skills validate` CLI command exists:** The registered skills subcommands on `main` are `search`, `install`, `update`, `list`, `info`, and `check`. There is no `validate` command in the skills CLI.
  - file: [src/cli/skills-cli.ts:172](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cli/skills-cli.ts#L172)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **CLI and gateway only report successfully loaded skills:** `buildWorkspaceSkillStatus()` builds its report from `loadWorkspaceSkillEntries(...)`, and the CLI formatter renders only `report.skills`. The gateway `skills.status` method returns that same report, so malformed workspace skills are omitted rather than reported as load failures in either CLI or web-backed status surfaces.
  - file: [src/agents/skills-status.ts:228](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/skills-status.ts#L228)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Docs describe required frontmatter but not a validator or failure visibility:** The skills docs still state that `SKILL.md` must include `name` and `description` and note parser constraints, but they do not document a validation command or built-in parse-failure reporting flow.
  - file: [docs/tools/skills.md:144](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/tools/skills.md#L144)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
