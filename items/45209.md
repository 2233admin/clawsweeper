## Summary

Keep open. Current `main` still does not implement a `channels.feishu.tools.bitable` / `tools.bitable` toggle, so the PR is not obsolete, and the requested behavior remains a coherent extension-scoped feature consistent with existing Feishu per-tool toggles.

## Evidence

- **Current Feishu tools config type omits `bitable`:** `FeishuToolsConfig` on current `main` exposes `doc`, `chat`, `wiki`, `drive`, `perm`, and `scopes`, but no `bitable` flag, so users cannot configure Bitable enablement through the typed config surface.
  - file: [extensions/feishu/src/types.ts:95](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/feishu/src/types.ts#L95)

- **Schema and defaults also omit `bitable`:** The Zod schema accepts no `bitable` key, and `DEFAULT_TOOLS_CONFIG` has no default for it. That means current `main` has no supported config path for disabling Bitable tools.
  - file: [extensions/feishu/src/config-schema.ts:94](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/feishu/src/config-schema.ts#L94)
