## Summary

Keep open. Current `main` does not implement the PR’s catalog-warning behavior, and the PR is not obsolete: `models set` still writes the model directly without loading a catalog, while runtime still explicitly tolerates configured models that are absent from the catalog. The review-thread concerns were addressed on the PR branch, but no equivalent change is present on `main` or in shipped docs/changelog.
