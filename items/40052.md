## Summary

Keep PR #40052 open. Current main still routes default-agent model and fallback edits through the per-agent `agents.list[index].model` path, while the documented/configured default model surface is `agents.defaults.model`. The PR is still relevant, but its review discussion identifies a real stale-override cleanup gap that should be handled before landing as-is.

## Best Possible Solution

Keep this item open or replace it with a maintainer-owned patch that routes default-agent primary and fallback edits to `agents.defaults.model`, preserves non-default writes to `agents.list[].model`, removes or migrates stale default-agent `agents.list[].model` overrides when saving defaults, and adds regression coverage for default, non-default, and `defaultId === null` path resolution.
