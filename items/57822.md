## Summary

Current `main` already passes `identity.name` through the gateway agent list and renders the Web UI dropdown label from that value, so the reported bug is already implemented/fixed on `main` and shipped in `v2026.4.23`.

## Close Comment

Closing this as already implemented on current `main`.

- Codex review checked the current Gateway and Web UI code paths without modifying the checkout.
- `ui/src/ui/views/agents.ts:162` renders the agent switcher label with `normalizeAgentLabel(agent)` rather than raw `agent.id`.
- `ui/src/ui/views/agents-utils.ts:197` resolves labels as `agent.name`, then `agent.identity?.name`, then `agent.id`, so configured `identity.name` is used when no explicit `name` is set.
- `src/gateway/session-utils.ts:698` and `src/gateway/session-utils.ts:733` include `identity.name` in the `agents.list` payload that the Web UI consumes.
- The same implementation is present in release `v2026.4.23` (`9b18ae877018a34933e27bb7b2986cb97e9602aa`), so this behavior is already shipped.
