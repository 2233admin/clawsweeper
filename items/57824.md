## Summary

Keep PR #57824 open. Current main still has both defects the PR targets: URL-only ACP image attachments are skipped before `MediaAttachmentCache` can fetch them, and Synology invalid-token throttling still locks only after the configured budget is exceeded rather than when it is exhausted. The provided PR discussion also shows the zero-budget regression review was addressed in the branch head, so the PR remains actionable rather than obsolete.

## Best Possible Solution

Keep the PR open and review/land a rebased version that removes the ACP path-only gate while preserving the local-path no-fallback invariant, changes Synology invalid-token locking to threshold-based behavior, clamps the pre-auth invalid-token budget for zero/low limits, and keeps the targeted regression tests for URL-only ACP images, blocked local path plus URL, exact invalid-token threshold, and zero-budget valid-token behavior.
