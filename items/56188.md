## Summary

Keep open. Current `main` still contains the alias-gated skill-command loading in `src/auto-reply/reply/get-reply-directives.ts` and the empty-array truthiness reuse in `src/auto-reply/reply/get-reply-inline-actions.ts`, so the reported `/skill ...` rewrite gap is still present in code.

## Evidence

- **Directive path still skips skill command loading without aliases:** `resolveReplyDirectives` only loads workspace skill commands when `allowTextCommands && commandTextHasSlash && rawAliases.length > 0`. If no model aliases are configured, `rawAliases` is `[]`, so `skillCommands` is returned as `[]`.
  - file: [src/auto-reply/reply/get-reply-directives.ts:249](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/reply/get-reply-directives.ts#L249)
  - command: `nl -ba src/auto-reply/reply/get-reply-directives.ts | sed -n '230,275p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Directive result passes empty skill command list forward:** `get-reply.ts` takes `skillCommands` from `resolveReplyDirectives` and passes it into `handleInlineActions` unchanged, so an upstream `[]` becomes the downstream input for inline command handling.
  - file: [src/auto-reply/reply/get-reply.ts:435](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/reply/get-reply.ts#L435)
  - command: `nl -ba src/auto-reply/reply/get-reply.ts | sed -n '430,545p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Inline actions still treat empty array as a loaded result:** `handleInlineActions` still uses `shouldLoadSkillCommands && params.skillCommands ? params.skillCommands : ...`; because `[]` is truthy, it reuses the empty array instead of falling back to `listSkillCommandsForWorkspace(...)`. That prevents the later skill invocation and body rewrite path from running.
  - file: [src/auto-reply/reply/get-reply-inline-actions.ts:233](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/reply/get-reply-inline-actions.ts#L233)
  - command: `nl -ba src/auto-reply/reply/get-reply-inline-actions.ts | sed -n '225,255p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Rewrite path still depends on non-empty skill command resolution:** The `/skill` transformation to `Use the "..." skill for this request.` only happens inside the `if (skillInvocation)` branch, and `skillInvocation` is only computed when `skillCommands.length > 0`. With the empty-array reuse above, the rewrite cannot happen.
  - file: [src/auto-reply/reply/get-reply-inline-actions.ts:245](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/reply/get-reply-inline-actions.ts#L245)
