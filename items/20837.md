## Summary

Keep open. Current main implements channel awareness for the normal embedded chat flow and trusted inbound metadata, but the slash-command prompt reconstruction path still drops the same runtime channel/capability/shell fields that #21271 attempted to add. The related PR was closed unmerged, so the issue remains partially addressed rather than closeable.

## Best Possible Solution

Keep this issue open and narrow the remaining work to the command prompt bundle. `resolveCommandsSystemPromptBundle()` should mirror the embedded runner by resolving real host/os/arch/shell plus normalized channel, account-aware capabilities, message-tool capabilities, and channel actions, then add focused tests that assert `buildSystemPromptParams()` receives those runtime fields for Telegram/WebChat and multi-account contexts. The optional selective-reply part should be treated as already partly covered by deterministic source-channel routing, `session.sendPolicy`, and the message tool; any prompt-level channel-selection behavior would need explicit maintainer product direction.
