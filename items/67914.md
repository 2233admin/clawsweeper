## Summary

Keep open. Current `main` does not show a shipped fix for this report, and code inspection does not disprove it. The shared media store writes whatever bytes each channel hands it, while bundled Feishu still downloads message-resource bytes and persists them through that store. Two of the three named channels (`WeChat`, `WeCom`) are external plugins, so the report may span multiple repos rather than being safely closeable from OpenClaw core alone.

## Evidence

- **Core media store is not content-addressed or MD5-based:** `saveMediaBuffer` creates a fresh UUID-based filename, detects MIME, and writes the provided buffer directly. There is no dedupe or hash-cache path here that would explain identical files by itself.
  - file: [src/media/store.ts:407](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/media/store.ts#L407)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Bundled Feishu still persists downloaded bytes exactly as received:** For inbound Feishu media, `resolveFeishuMediaList` calls `downloadMessageResourceFeishu(...)` and then passes `result.buffer` into `saveMediaBuffer(...)` for `media/inbound`. If distinct images are landing with identical MD5s, the unresolved suspect remains the download path or upstream API response, not a fixed core-store path on `main`.
  - file: [extensions/feishu/src/bot-content.ts:427](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/extensions/feishu/src/bot-content.ts#L427)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Feishu download path unchanged since latest release:** Diffing latest release `v2026.4.23` against current `main` for `extensions/feishu/src/media.ts`, `extensions/feishu/src/bot-content.ts`, and `src/media/store.ts` shows no functional change in the Feishu download/save flow tied to this report; the only `media.ts` change is an import-path swap.
  - command: `git diff a9797214338ba31c52c796adbb75afb16e0684a9 fa976e5b93b3819f9e137fe08635764edee38290 -- src/media/store.ts extensions/feishu/src/media.ts extensions/feishu/src/bot-content.ts`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **WeChat channel is external to OpenClaw core:** The user report names WeChat, but OpenClaw docs state the WeChat runtime lives in the external `@tencent-weixin/openclaw-weixin` plugin and that WeChat media upload/download is owned there, not in this repo.
