## Summary

Keep open. Current `main` still flags the first `$VARNAME` match anywhere in a Python/Node script and throws immediately, so the false positive described by the PR is not implemented on `main`. The proposed PR also has unresolved review concerns about JS comment/regex handling that could weaken preflight detection, so this is not a conservative cleanup close.
