## Summary

Keep open. Current `main` at `95b7a85f0607784d8e33b4ce7e42617174196490` still unconditionally enqueues ACP parent relay `:progress`, `:stall`, and `:resumed` events into the parent session, and `acp-spawn.ts` still routes those system events back through the parent delivery context. The reported Discord thread chatter is therefore still plausible and not clearly fixed on `main`.
