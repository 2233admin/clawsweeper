## Summary

Current main still matches the reported failure mode for literal per-agent session stores. `sessions.list` depends on `loadCombinedSessionStoreForGateway`, and that helper still short-circuits non-templated `session.store` values to one resolved store attributed to the default agent, bypassing the all-agent discovery path. The Control UI sessions dashboard reads from `sessions.list`, so the API bug also explains the dashboard symptom. The provided issue context also shows an open, unmerged implementation candidate, #54716, with closing syntax and before/after proof, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while #54716 is reviewed, rebased if needed, and either landed or replaced with an equivalent fix. The fix should update `src/config/sessions/combined-store-gateway.ts` so literal per-agent paths under `agents/<id>/sessions/sessions.json` use all-agent discovery while preserving true shared literal-store behavior, and it should add a regression test for that literal path.
