## Summary

Keep open. Current `main` at `3a64aa49a9014518fa1977fbc023f3e0b978ae5d` still passes the existing `sessionFile` straight through during gateway reset, so the reported `sessionId`/`sessionFile` mismatch fix from PR #57126 is not implemented on `main`.

## Evidence

- **Gateway reset still reuses the old sessionFile unchanged:** `performGatewaySessionReset` generates `nextSessionId` and then calls `resolveSessionFilePath(nextSessionId, currentEntry?.sessionFile ? { sessionFile: currentEntry.sessionFile } : undefined, ...)`, which preserves an old UUID-based filename instead of rewriting it to the new session ID.
  - file: [src/gateway/session-reset-service.ts:544](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/gateway/session-reset-service.ts#L544)
  - command: `nl -ba src/gateway/session-reset-service.ts | sed -n '540,552p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **The rewrite helper exists, but only in compaction code:** `resolveCompactionSessionFile` already calls `rewriteSessionFileForNewSessionId(...)` before `resolveSessionFilePath(...)`, but those helpers remain private and are not used by gateway reset on current `main`. That is the exact gap the PR proposes to close.
  - file: [src/auto-reply/reply/session-updates.ts:291](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/auto-reply/reply/session-updates.ts#L291)
  - command: `nl -ba src/auto-reply/reply/session-updates.ts | sed -n '291,366p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Current tests cover custom-name preservation, not UUID rewrite on reset:** The reset test on `main` asserts that a custom transcript filename stays unchanged after reset (`custom-owned-child-transcript.jsonl`). There is no corresponding test on `main` asserting that a UUID-based `sessionFile` basename is rewritten to the new `sessionId` during reset.
