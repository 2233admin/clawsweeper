## Summary

Keep this PR open. Current main still lacks NUL-byte normalization in the iMessage sent-message echo cache, and the open PR is a narrow implementation candidate for the linked iMessage echo bug rather than obsolete or superseded work.

## Best Possible Solution

Review and land this focused iMessage echo-cache normalization fix, or land an equivalent patch that strips leading NUL-prefixed echoes and adds regression coverage for single and repeated leading NUL bytes. Keep broader U+FFFD/control-prefix normalization as a separate maintainer decision unless it is intentionally combined.
