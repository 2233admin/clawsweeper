## Summary

Keep open. Current main still exposes the agent file browser as a fixed core-workspace-file editor: `ALLOWED_FILE_NAMES` remains in the gateway, list/get/set do not accept a scope or arbitrary workspace path, the protocol schema has no scope selector fields, and the Control UI still presents only “Core Files” tabs. The requested all-workspace/system file browsing capability is therefore still missing and should not be auto-closed as implemented, duplicate, or ClawHub-only.
