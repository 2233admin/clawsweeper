## Summary

Keep #70607 open. Current main still contains the cross-agent behavior the PR is trying to change, so this is not implemented or obsolete. The submitted patch also needs maintainer review/adjustment because flipping the guard to `===` appears to conflict with the existing same-agent invariant and with the P1 review comment on the PR.

## Best Possible Solution

Keep the PR open, but do not merge the current patch as-is. The best fix is a focused implementation that preserves the requester/parent `accountId` for cross-agent completion delivery while also preserving the existing same-agent active-account invariant, with regression tests for both paths in `src/agents/spawn-requester-origin.test.ts` and, ideally, one spawn-level test showing the registered `requesterOrigin` for cross-agent `sessions_spawn`.
