## Summary

Keep PR #70789 open. Current main still has the reverse newest-first tool-card pairing logic, and the PR targets the implicated Control UI extraction path for the linked same-name sequential tool result bug. It is not implemented on main or superseded by the closed linked issue.

## Best Possible Solution

Keep #70789 open for maintainer review. The best path is to merge or adapt the exact-ID-first, earliest-unmatched pairing fix in ui/src/ui/chat/tool-cards.ts, with a focused regression test in ui/src/ui/chat/tool-cards.node.test.ts for sequential same-name calls before their results.
