## Summary

Keep open. Current `main` mitigates the crash-loop by reading existing systemd env values before `gateway install --force`, but it still regenerates the unit file and does not preserve user-authored `EnvironmentFile=` directives or move managed env into a drop-in. Custom env-file values are flattened into env entries during reinstall, so the reported unit-regeneration behavior and secret/source-of-truth concerns are not fully resolved.

## Risks / Open Questions

- Custom secrets from arbitrary user `EnvironmentFile` sources can still be rewritten into inline `Environment=` entries on reinstall/update.
- Users who rely on an external env file remaining the source of truth will still lose that directive after reinstall, even if some values survive functionally.
- The code on `main` does not implement the issue's proposed drop-in approach, so related migration and secret-handling edge cases can still regress.
