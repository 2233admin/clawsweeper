## Summary

Keep open. Current main still appears to miss the reported custom-named Ollama marker-auth path: `ollama-local` is still classified as a non-secret marker, explicit marker apiKeys are not resolved as usable config credentials, and the generic synthetic local-auth fallback is skipped when an explicit apiKey exists. The linked #49030 is the narrow open PR for this issue, while #69857/#59954 address a separate/partial Ollama Cloud-vs-local synthetic-auth path rather than this subagent custom-provider lookup bug.
