## Summary

Keep open. Current `main` still treats `runtime="acp"` as a host-side/gateway-side runtime, with no node-routing field in ACP session/runtime options. Node routing exists for `exec host=node`, not for ACP backend/session initialization.

## Evidence

  - file: [src/agents/acp-spawn.ts:170](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/acp-spawn.ts#L170)

  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Host-side behavior is documented and regression-tested:** User docs say `Sandboxed sessions cannot spawn ACP sessions ...` because `ACP runtime is host-side`, and the changelog records the same host-side guardrail. The test suite also asserts sandboxed requester sessions are forbidden from ACP spawn.
  - file: [docs/tools/acp-agents.md:543](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/docs/tools/acp-agents.md#L543)

  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)

## Risks / Open Questions

- I did not run a live ACP reproduction because this review is read-only and the environment lacks `pnpm`, so the conclusion is based on code/docs/tests rather than an end-to-end session spawn.
- The bundled `acpx` dependency could gain remote-node capabilities upstream, but current OpenClaw `main` does not surface any node-routing input into ACP session initialization.
