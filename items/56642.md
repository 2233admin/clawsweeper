## Summary

Current `main` has several WhatsApp reconnect hardening changes after the reporter's `2026.3.7` version, including centralized connection ownership and reconnect-gap cleanup, but I did not find strong evidence that the specific multi-hour heap-growth / `SIGABRT` report is fixed or no longer reproducible. Keep open.

## Evidence

- **Post-2026.3.7 reconnect hardening shipped:** The changelog after `2026.3.7` includes WhatsApp reconnect-focused fixes such as resetting watchdog state after reconnect loops and centralizing live connection ownership.
  - file: [CHANGELOG.md:1255](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/CHANGELOG.md#L1255)
- **Current reconnect lifecycle explicitly tears down connection resources:** `closeCurrentConnection()` clears heartbeat/watchdog timers, waits for background tasks, closes the listener, and closes the socket before retry/shutdown.
  - file: [extensions/whatsapp/src/connection-controller.ts:505](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/whatsapp/src/connection-controller.ts#L505)
- **Current inbox listener cleanup detaches event handlers on close:** The live inbox monitor detaches `messages.upsert` and `connection.update` listeners and closes the socket in its `close()` path, which is relevant to leak prevention during reconnects.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:699](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/whatsapp/src/inbound/monitor.ts#L699)
- **Reconnect behavior has regression coverage, but not heap-growth coverage:** There are reconnect/watchdog tests for repeated listener replacement, but I did not find a WhatsApp-specific RSS/heap regression test or any reference to the reported `BaseNameDictionary::Add` crash signature.
  - file: [extensions/whatsapp/src/auto-reply.web-auto-reply.connection-and-logging.e2e.test.ts:233](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/whatsapp/src/auto-reply.web-auto-reply.connection-and-logging.e2e.test.ts#L233)
  - command: `rg -n "heap|memory leak|rss|SIGABRT|BaseNameDictionary|reconnect loop|watchdog|closeCurrentConnection|backgroundTasks" extensions/whatsapp/src test src -g '*.test.ts' -g '*.ts'`
