## Summary

Do not auto-close #41165. Current main still has code and tests showing default Telegram private DMs without a topic/thread route to `agent:main:main`, and the heartbeat resolver still lets a forced wake/session key win before `heartbeat.session`, matching the later system-event heartbeat pollution reports. The provided related PR context also shows the direct Telegram isolation PR was closed unmerged and the system-event heartbeat routing PR remains open, so the issue is not implemented on main or safely superseded by shipped work.

## Best Possible Solution

Keep #41165 open for maintainer judgment. The clean path is to either implement and test strict Telegram DM isolation from `agent:main:main` when heartbeat/main traffic is active, or explicitly document/configure `session.dmScope: "per-channel-peer"` as the supported isolation mode and close only with that product decision. The system-event heartbeat subcase should be resolved by making `heartbeat.session` precedence explicit over originating forced sessions, or by landing a narrowed successor to PR #56092 with regression tests.
