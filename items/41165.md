## Summary

Keep open. Current main still has the two remaining behaviors implicated by the report: default Telegram DMs can resolve to the agent main session, and wake-triggered heartbeats still let an originating forced session win before `heartbeat.session`. The provided GitHub context also shows PR #56092 is open, unmerged, and explicitly positioned as the implementation candidate for the heartbeat-routing part, so this issue should not be closed before that PR is resolved.

## Best Possible Solution

Keep this issue open while maintainers review PR #56092 and either land it or close it with explicit rationale. After that, decide the separate Telegram product behavior: either keep and document default DM-to-main routing as intentional, or implement Telegram DM isolation in the Telegram plugin with matching tests and channel-routing docs.
