## Summary

Keep open. Current `main` still omits `--exclude-labels` from Gmail watcher startup args, and the config schema still has no `hooks.gmail.excludeLabels` field, so the reported regression is not implemented on `main` or in the latest release evidence reviewed.

## Evidence

- **Current main drops the flag:** `buildGogWatchServeArgs()` builds the `gog gmail watch serve` argv from account/bind/port/path/token/hook URL/hook token, with optional `--include-body` and `--max-bytes`, but no `--exclude-labels` entry.
  - file: [src/hooks/gmail.ts:225](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/hooks/gmail.ts#L225)

  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Missing args are used at runtime:** The gateway watcher spawns `gog` with `const args = buildGogWatchServeArgs(cfg)`, so the missing flag directly affects the running Gmail watcher.
  - file: [src/hooks/gmail-watcher.ts:62](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/hooks/gmail-watcher.ts#L62)
