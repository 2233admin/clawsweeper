## Summary

Keep this issue open. Current main still receives Telegram reaction updates, but it exposes them only as human-readable system events; there is still no stable structured managed/internal hook or plugin hook contract for downstream reaction-driven feedback automation.

## Best Possible Solution

Add an additive typed reaction event seam. Telegram should emit structured reaction data after its existing authorization and filtering path, including channel/account context, session or conversation context when available, Telegram chat id, target message id, emoji, add/remove operation, sender metadata, authorization outcome, and bounded raw metadata. Expose it through managed/internal hooks and preferably the plugin hook contract, then add focused Telegram handler tests, hook contract tests, and public docs. Feedback recording should remain a hook or plugin built on that seam, not hardcoded into the Telegram channel.
