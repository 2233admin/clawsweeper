## Summary

Keep open. Current `main` still short-circuits `openclaw gateway install` to `already-installed` for loaded services unless the special `NODE_EXTRA_CA_CERTS` refresh case applies; it does not check for an embedded stale `OPENCLAW_GATEWAY_TOKEN`. Separate audit code still treats embedded gateway tokens as a problem that requires `openclaw gateway install --force`, and the provided issue discussion points to an open PR rather than a landed fix.

## Evidence

There is no embedded-token check in this branch.
