## Summary

Keep #69303 open. Current main still releases the inbound dedupe claim on every dispatch error without stamping the dedupe cache, and existing tests explicitly preserve retry-after-error behavior. The linked PR #71819 appears to be the active fix path for the agreed poisoned-dedupe direction, but it is open/unmerged in the provided GitHub context, so the bug is not implemented on main.

## Best Possible Solution

Keep the issue open as the canonical bug until PR #71819 or an equivalent patch lands on main. The likely fix belongs in the auto-reply inbound dedupe/dispatch path: preserve release-style retry for failures before any externally visible progress, but stamp a terminal/poisoned dedupe state once an outbound reply or other side effect has occurred, with regression tests for both paths.
