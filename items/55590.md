## Summary

Keep open. Current `main` already supports per-agent `memorySearch` overrides and a single fallback embedding provider, but it does not implement the issue's requested multi-model / multi-fallback embedding configuration with separately configurable primary and fallback model blocks.

## Evidence

- **Schema only exposes one provider, one model, and one fallback string:** `MemorySearchConfig` defines singular `provider?: string`, `fallback?: string`, `model?: string`, and one shared `remote` object. There is no `{ primary, fallbacks[] }` shape or per-fallback model config.
  - file: [src/config/types.tools.ts:362](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/config/types.tools.ts#L362)
  - command: `nl -ba src/config/types.tools.ts | sed -n '336,390p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Docs describe only a single fallback adapter:** The memory config reference lists `provider`, `model`, and `fallback` as singular keys, with `fallback` documented as a single fallback adapter ID when the primary fails.
  - file: [docs/reference/memory-config.md:38](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/docs/reference/memory-config.md#L38)
  - command: `nl -ba docs/reference/memory-config.md | sed -n '34,48p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Resolved runtime config carries one shared fallback and one shared remote config:** `resolveMemorySearchConfig` merges `provider`, `fallback`, and `remote` as single values. The same merged `remote` settings are used regardless of whether the primary or fallback provider is active.
  - file: [src/agents/memory-search.ts:153](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/memory-search.ts#L153)
  - command: `nl -ba src/agents/memory-search.ts | sed -n '150,180p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Execution path tries at most one fallback provider:** `createEmbeddingProvider(...)` attempts the primary adapter, then optionally one fallback adapter. There is no loop over multiple fallbacks or support for same-provider alternate model entries.
  - file: [extensions/memory-core/src/memory/embeddings.ts:141](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/memory-core/src/memory/embeddings.ts#L141)

  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)

That still does not allow separately configured fallback credentials, base URL, or an ordered fallback list.
