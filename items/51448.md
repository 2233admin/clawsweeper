## Summary

Current `main` still does not forward configured `tool_choice`/`toolChoice` into embedded-runner stream options, so the reported bug remains real. The PR should stay open because the requested behavior is not implemented on `main`, and the review discussion also identified a precedence bug in the proposed patch that still needs maintainer resolution.

## Evidence

- **Current main still omits toolChoice from stream option forwarding:** `createStreamFnWithExtraParams()` forwards `temperature`, `maxTokens`, `transport`, `openaiWsWarmup`, and cached-content fields, but there is no branch that copies `tool_choice`/`toolChoice` into `streamParams.toolChoice`.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:290](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/agents/pi-embedded-runner/extra-params.ts#L290)
  - command: `nl -ba src/agents/pi-embedded-runner/extra-params.ts | sed -n '280,360p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Downstream OpenAI request builders only honor tool choice from stream options:** OpenAI HTTP transport sets `params.tool_choice` only from `options.toolChoice`, and the WS request builder likewise maps only `streamOpts.toolChoice` into `extraParams.tool_choice`. Without the embedded-runner forwarding step, configured agent params never reach these requests.
  - file: [src/agents/openai-transport-stream.ts:1767](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/agents/openai-transport-stream.ts#L1767)

  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Current tests prove analogous forwarding for other params, not toolChoice:** `main` has regression coverage that configured model params like `maxTokens` reach stream options, but there is no matching `tool_choice` forwarding test in the checked-out tree. That matches the missing implementation above.
  - file: [src/agents/pi-embedded-runner-extraparams.test.ts:1645](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/agents/pi-embedded-runner-extraparams.test.ts#L1645)

- **PR discussion identified an unresolved precedence bug in the proposed patch:** The PR review pointed out that resolving `tool_choice` from an already-merged params object would break alias precedence. Current `resolveExtraParams()` can preserve both snake_case and camelCase keys across layers, and `resolveAliasedParamValue()` prefers the snake_case key when both exist on one object, so a lower-precedence `tool_choice` could beat a higher-precedence `toolChoice`.
