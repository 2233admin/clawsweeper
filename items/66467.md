## Summary

Keep this issue open. Current main still pins the ACPX path to acpx 0.6.1 and @agentclientprotocol/sdk 0.20.0; that SDK schema requires usage_update.used and size to be numbers and validates session/update before acpx's lenient runtime projection can turn missing or non-finite usage into a best-effort status event.

## Best Possible Solution

Fix the ACPX/OpenClaw ACP runtime boundary. Prefer updating or pinning ACPX and @agentclientprotocol/sdk to a protocol/schema version that accepts nullable or absent usage fields, or normalize/drop nullable usage_update fields before SDK validation can reject the notification. Add regression coverage for a session/update payload with sessionUpdate: "usage_update", used: null, and numeric size so Discord-bound ACP turns remain stable and project a best-effort status event.
