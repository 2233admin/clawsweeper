## Summary

Current main still does not implement the requested dummy shell/no-op tool-call hardening. The planning-only retry path only allows one safe read/search-style tool through, treats `exec`/`bash` as mutating by name, and still returns before prose classification when the assistant stop reason is `toolUse`.

## Best Possible Solution

Keep this issue open. Implement a structural progress check in the embedded runner planning-only path, likely by threading enough shell command and result metadata into `src/agents/pi-embedded-runner/run/incomplete-turn.ts` to classify no-op/no-output shell calls as non-progress while preserving conservative side-effect handling for real shell mutations. Add the requested regression where a single dummy shell call plus plan-only prose triggers the strict-agentic retry and eventual blocked exit.
