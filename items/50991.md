## Summary

Keep open. Current `main` still limits the bundled `session-memory` hook to `/new` and `/reset`; `/compact` emits separate compaction lifecycle hooks, but the requested bundled-session-memory behavior is not implemented.

## Evidence

- **Bundled hook metadata still excludes compact:** The bundled `session-memory` hook declares only `command:new` and `command:reset` events.
  - file: [src/hooks/bundled/session-memory/HOOK.md:10](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/hooks/bundled/session-memory/HOOK.md#L10)
  - command: `nl -ba src/hooks/bundled/session-memory/HOOK.md | sed -n '1,40p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Handler only accepts new/reset command actions:** `saveSessionToMemory` gates on `event.action === "new" || event.action === "reset"` and returns for anything else, so a `compact` command would be ignored even if emitted.
  - file: [src/hooks/bundled/session-memory/handler.ts:55](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/hooks/bundled/session-memory/handler.ts#L55)

## Risks / Open Questions

- There is adjacent compaction-hook infrastructure on main, so a future maintainer may decide the feature should reuse `session:compact:before` instead of adding `command:compact` directly.
- The automatic memory-flush feature may partially address the user problem, but it is not a byte-for-byte replacement for the bundled `session-memory` archive behavior requested in the issue.
