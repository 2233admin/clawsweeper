## Summary

Keep #70407 open. Current main still validates `plugins.entries.memory-core.config.dreaming.storage` as an object, and the config read path fails closed before a legacy string value can be used. I did not find a current memory-core migration, doctor contract, setup migration, or regression test that converts old string values like `"both"` into the new object shape.

## Best Possible Solution

Keep the issue open and add a backward-compatible migration owned by memory-core/config repair: convert legacy `plugins.entries.memory-core.config.dreaming.storage` string values `inline`, `separate`, and `both` into `{ mode: <value>, separateReports: false }` before plugin schema validation or through a doctor/setup migration that can run from invalid config. Add regression coverage for config read/validation and `openclaw doctor --fix`, plus a friendly diagnostic if migration cannot be applied.
