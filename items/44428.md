## Summary

Current main still appears to have the reported bug. `deliveryBestEffort` is passed into `dispatchCronDelivery`, but the text-delivery subagent follow-up path still waits on active descendants with the full cron timeout and still suppresses delivery when active descendants remain, without checking `params.deliveryBestEffort`. The related fix PR #46160 was closed unmerged in the provided context, and its intended guard is not present on current main.
