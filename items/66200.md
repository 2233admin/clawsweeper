## Summary

Keep this PR open. Current main still routes BlueBubbles inbound text to the model through raw `BodyForAgent`/`BodyStripped`, while the submitted patch only wraps the legacy envelope `Body`. The security hardening goal remains relevant, but the PR is not merge-ready as-is because it misses the model-facing path, lacks targeted tests, and still needs the Discord-style empty-body guard already requested in review.

## Best Possible Solution

Keep the PR open and request a targeted revision or replacement. The fix should harden the actual BlueBubbles model-facing text path, likely by populating `UntrustedContext` or wrapping the value that becomes `BodyForAgent`/`BodyStripped`, while preserving raw `RawBody`, `CommandBody`, and `BodyForCommands` for command parsing, reset handling, dedupe, and metadata. It should add the Discord-style non-empty guard and focused BlueBubbles tests proving inbound text reaches the agent with the intended untrusted-content boundary.

## Risks / Open Questions

- Merging the PR as-is would create a false sense of hardening because raw BlueBubbles text can still reach the model through `BodyForAgent`.
- The unconditional wrap can turn attachment-only or empty-body messages into marker-only prompt text, diverging from the Discord guard pattern already requested in review.
- The patch touches a security-sensitive prompt boundary without tests proving the actual BlueBubbles model-facing prompt is protected.
