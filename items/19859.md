## Summary

Keep #19859 open. Current main still hardcodes Enter-to-send for WebChat/Control UI, lacks a persisted UI or gateway `sendOnEnter` setting, and the WebChat docs/config schema only expose `gateway.webchat.chatHistoryMaxChars`. The existing issue discussion and the prior maintainer-side Codex review both support keeping this as an open core WebChat enhancement.

## Best Possible Solution

Keep this issue open as a core WebChat/Control UI enhancement. The best implementation path is to add a maintainer-approved composer shortcut preference, wire it through the browser WebChat render/state path and native macOS composer if applicable, support the chosen modifier-send behavior when Enter inserts a newline, update config/docs if a gateway-level option is added, and add focused tests for Enter, Shift+Enter, IME composition, and Ctrl/Cmd+Enter behavior.
