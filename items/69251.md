## Summary

Keep #69251 open. Current main still POSIX-wraps `exec host=node` commands with `/bin/sh -lc`, still performs `system.run.prepare` before the effective `security=full` / `ask=off` policy can bypass anything, and the node-side planner still fail-closes for shell payloads that point at mutable or script-like files. The related #69077/#69308 path is orthogonal: it fixed placeholder `SHELL` fallback, not this approval-binding denial.

## Best Possible Solution

Keep the issue open and fix the core node exec preparation/binding path. A conservative fix should either make `system.run.prepare` compute and revalidate node-local mutable file operands for shell payloads such as `/tmp/test.sh`, `/bin/zsh /tmp/test.sh`, and `python3 /tmp/script.py`, or pass enough effective approval-policy context to avoid mandatory stable binding when the node is explicitly `security=full` with `ask=off`. Coordinate with the still-open #48457 stable-binding class, but do not close this issue as a duplicate because #69251 has distinct file-reference reproductions and useful discussion about the `/bin/sh -lc` transport wrapper.
