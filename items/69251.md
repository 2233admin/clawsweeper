## Summary

Keep open. Current main still wraps non-Windows `exec host=node` commands in `/bin/sh -lc`, always sends that wrapped argv through `system.run.prepare`, and the node-host approval planner still rejects file-backed shell payloads with the exact `SYSTEM_RUN_DENIED: approval cannot safely bind this interpreter/runtime command` message. The related #69308 LaunchDaemon fix is orthogonal; it only handles placeholder `$SHELL` values.

## Best Possible Solution

Keep #69251 open as a real current node-exec bug unless maintainers intentionally consolidate it with a broader canonical tracker. The fix should preserve the `systemRunPlan` security boundary while making `security=full` + `ask=off` node exec usable for direct script and interpreter file-operand commands, likely by passing effective node approval policy into prepare or by computing a bindable snapshot for the inner shell payload on the node. Add focused regression coverage for `exec host=node` direct scripts and interpreter file args under full/off, plus allowlist/ask cases that must still fail closed.
