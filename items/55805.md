## Summary

Current `main` does not implement npm fallback for generic ClawHub request failures. The install path only falls back when ClawHub returns typed `package_not_found` or `version_not_found` results, while generic errors like `fetch failed` still stop the command. Docs/changelog on `main` also describe that narrower behavior, so there is no strong evidence this issue is already fixed on `main`.

## Evidence

- **CLI stops on non-fallback ClawHub errors:** The bare-spec install flow tries ClawHub first, then exits on any ClawHub failure unless `decidePreferredClawHubFallback(...)` returns `fallback_to_npm`.
