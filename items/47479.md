## Summary

Keep open. Current `main` still does not implement this PR’s requested behavior: the Docker setup path does not load project `.env` values as defaults, it still rewrites multi-word `.env` values without preserving quotes, and it still does not pass `OPENCLAW_INSTALL_BROWSER` through even though docs and the `Dockerfile` advertise that knob.

## Evidence

- **Setup script only reads `.env` for gateway-token reuse, not general defaults:** `scripts/docker/setup.sh` initializes Docker settings from process env/defaults and never loads project `.env` up front. The only `.env` reader is `read_env_gateway_token()`, called solely when `OPENCLAW_GATEWAY_TOKEN` is unset, so values like `OPENCLAW_DOCKER_APT_PACKAGES` or `OPENCLAW_IMAGE` in `.env` are ignored by setup.
  - file: [scripts/docker/setup.sh:90](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/scripts/docker/setup.sh#L90)
  - command: `nl -ba scripts/docker/setup.sh | sed -n '1,220p'; nl -ba scripts/docker/setup.sh | sed -n '220,560p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **`.env` rewrite path still strips quoting from multi-word values:** `upsert_env()` writes replacements and appended keys with bare `printf '%s=%s\n'`, with no quoting/escaping helper. A value such as `OPENCLAW_DOCKER_APT_PACKAGES="jq ripgrep"` would be rewritten as `OPENCLAW_DOCKER_APT_PACKAGES=jq ripgrep` on the next setup run.
  - file: [scripts/docker/setup.sh:434](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/scripts/docker/setup.sh#L434)
  - command: `nl -ba scripts/docker/setup.sh | sed -n '417,453p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Current tests codify the unquoted multi-word `.env` output:** The e2e test explicitly expects `.env` to contain `OPENCLAW_DOCKER_APT_PACKAGES=ffmpeg build-essential`, which matches the buggy rewrite behavior instead of preserving quotes for a whitespace-containing value.
  - file: [src/docker-setup.e2e.test.ts:222](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/docker-setup.e2e.test.ts#L222)
  - command: `nl -ba src/docker-setup.e2e.test.ts | sed -n '212,240p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **`OPENCLAW_INSTALL_BROWSER` is documented and supported by the Dockerfile, but setup still never forwards it:** `scripts/clawdock/README.md` documents `OPENCLAW_INSTALL_BROWSER=1`, and `Dockerfile` defines `ARG OPENCLAW_INSTALL_BROWSER`. But `scripts/docker/setup.sh` never exports, persists, or passes that variable in its build args; it only forwards `OPENCLAW_DOCKER_APT_PACKAGES`, `OPENCLAW_EXTENSIONS`, and `OPENCLAW_INSTALL_DOCKER_CLI`.
  - file: [scripts/docker/setup.sh:476](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/scripts/docker/setup.sh#L476)
