## Summary

Keep open. Current `main` still routes `openclaw agent --local --json` through `runtime.log(...)` in the local delivery path, while the runtime JSON contract and CLI docs expect JSON mode to use the structured stdout writer. The PR addresses a real current-main bug and adds the missing regression coverage, so there is not strong evidence to close it conservatively.

## Evidence

- **Current local JSON path still logs instead of writing JSON:** `deliverAgentCommandResult()` on current `main` checks `opts.json` and calls `runtime.log(JSON.stringify(...))` for the outbound result envelope, which is the behavior the PR is trying to change.
