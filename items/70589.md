## Summary

Keep PR #70589 open. Current main still routes the local `openclaw agent --local --json` envelope through `runtime.log()`, while JSON mode intentionally routes console logs to stderr. The PR targets the implicated local delivery path and adds the missing regression coverage; the related issue #62440 remains real on current main, and the related PR #62453 was closed unmerged rather than shipping a canonical fix.

## Best Possible Solution

Land or adapt the focused fix from PR #70589: import and use `writeRuntimeJson(runtime, buildOutboundResultEnvelope(...))` in `src/agents/command/delivery.ts`, keep the envelope shape unchanged, and retain a local delivery unit test asserting `runtime.writeJson` is used when available. That directly fixes #62440 without changing gateway-mode or non-JSON behavior.
