## Summary

Keep open. Current `main` still has the overwrite behavior this PR targets, so it is not implemented on `main`. At the same time, the PR's one-time copy approach conflicts with the current Kubernetes redeploy contract documented in `docs/install/kubernetes.md` and exercised by `scripts/k8s/deploy.sh`, so there is not strong evidence to close it as obsolete or already fixed.

## Risks / Open Questions

- Closing this PR would leave an unresolved, still-present overwrite/data-loss behavior in current `main`.
- Merging the PR as-is would likely break the currently documented Kubernetes redeploy flow, because ConfigMap updates would stop propagating once the PVC already contains the files.
- A proper fix likely needs an explicit ownership/sync policy for PVC-backed config versus declarative ConfigMap updates, not just a bare existence check.
