## Summary

Keep open. Current main still has the reported mismatch: `openclaw doctor` warns about missing recent session transcripts but suggests a dry-run cleanup command without `--fix-missing`, while `sessions cleanup --dry-run` only counts missing transcript pruning when `fixMissing` is true. The prior related fix PRs were closed unmerged in the provided GitHub context, so the issue remains actionable.

## Best Possible Solution

Land a narrow commands fix in `src/commands/doctor-state-integrity.ts` so the missing-transcript `Preview cleanup impact` command includes `--dry-run --fix-missing`, then update `src/commands/doctor-state-integrity.test.ts` to assert that exact hint. The explicit `sessions cleanup --fix-missing` behavior can stay flag-gated unless maintainers choose a broader CLI behavior change.
