## Summary

Keep open. Current main supports Telegram forum-topic `threadId` routing for `message(action=send)`, but the issue reports an intermittent network-level `sendMessage` failure after cross-topic sends have been working. I did not find high-confidence evidence that that intermittent send failure is fixed; current tests still keep generic grammY `sendMessage` network envelopes non-retriable for non-idempotent sends.
