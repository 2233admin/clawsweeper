## Summary

Keep this issue open. Current main still has a plausible Control UI blank-screen path matching the report and prior discussion: `cron.list` can return malformed persisted cron jobs, and the global Control UI render plus Cron view/edit code still dereference `job.payload` without null-safe guards. The related fix PR #54552 was closed unmerged, and the newer cron hardening in the changelog covers scheduler/CLI paths, not the Control UI render path shown here.

## Best Possible Solution

Keep this issue open. The maintainer path is to reproduce against current main with a synthetic or reporter-provided stale cron row missing a valid payload, then add a defensive normalization/filter layer before `cron.list` returns jobs plus null-safe guards in the Control UI global render, Cron list, and edit/clone paths. Add a focused jsdom/controller regression test proving malformed cron rows do not blank unrelated Control UI sections, and keep doctor repair as cleanup rather than relying on it before the UI can render.
