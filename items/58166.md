## Summary

Keep PR #58166 open. Current main still lacks the PR's defensive filter in `repairToolUseResultPairing()`: whitespace-only tool-call IDs pass through `extractToolCallsFromAssistant()` and can still be used to synthesize a `toolResult`. Existing producer/session sanitizers reduce the common live path, but direct repair callers remain, so this PR is not safely obsolete.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. The best implementation is a narrow guard in the shared repair path: filter out tool calls whose `id.trim()` is empty before building the pairing set and before synthetic result creation, with regression coverage for empty and whitespace-only IDs while preserving valid tool-call repair behavior.

## Evidence

A whitespace-only ID would therefore still produce a whitespace-only synthetic `toolResult`.
