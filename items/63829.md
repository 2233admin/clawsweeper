## Summary

Current main still lacks per-agent memory-wiki vault isolation. The memory-wiki plugin captures one resolved vault config at plugin registration, the agent config schema does not accept per-agent plugin entries, and the related implementation candidate #67584 is still open rather than merged.

## Best Possible Solution

Keep this issue open as the narrow memory-wiki isolation request. Review and land #67584 or an equivalent memory-wiki-owned implementation that resolves the vault path per active tool/gateway context, covers wiki_search, wiki_get, wiki_apply, status/lint/CLI or clearly documents their scope, and adds tests proving two agents on one gateway cannot read or write each other's wiki vaults while preserving the current single-vault default.
