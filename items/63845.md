## Summary

Keep this issue open. Current main has several relevant mechanisms for reducing tool and skill context cost, but it still does not answer the central maintainer guidance requested: tested high-count tool/skill ceilings, collision-avoidance practices for many same-domain SKILL.md descriptions, recommended compaction/pruning values for large multi-tool turns, known-good low-end local-model profiles, or a clear product recommendation on hybrid smart aggregation tools.

## Best Possible Solution

Keep open for maintainer guidance or convert it into a focused docs task. The best outcome is a practical guide under Context, Skills, Tools/plugins, or Local models that states whether tested tool/skill-count ceilings exist; gives SKILL.md trigger collision practices; recommends `/context detail`, `skills.limits`, `agents.defaults.skills`, `tools.profile`, `tools.allow`/`deny`, pruning, compaction, and `localModelLean` patterns for high-count tool sessions; and explicitly describes when a hybrid domain aggregation plugin/workflow tool is acceptable, including traceability, output caps, and composability tradeoffs.
