## Summary

Keep #70575 open. Current main still constructs the agent session-store patch from the cached `entry` and then merges that patch into the fresher store entry, so the reported stale writeback class remains present for `lastChannel`/`lastTo`/`lastAccountId`/`lastThreadId` and `channel`. The related PR #19328 is open and unmerged, and its current diff still leaves these residual delivery/channel writebacks in place, so it does not close this follow-up.

## Best Possible Solution

Keep the issue open and address it either by expanding PR #19328 before merge or by a focused follow-up patch. The fix should keep stale cached `entry` values out of `nextEntryPatch`, derive fallback delivery/channel values inside the `updateSessionStore` mutator from the fresh store entry when needed, and add regression coverage matching the #5369 stale cached-entry/fresh store pattern for the four delivery fields and `channel`.
