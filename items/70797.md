## Summary

Keep open. Current main still lacks the raw OpenAI-style `{"tool_calls":[...]}` JSON sanitizer described by #70797, and the Telegram partial-preview path can still forward the cleaned assistant text to `onPartialReply` without a streaming-aware guard. The two related fix attempts, #71954 and #72011, are closed unmerged, so they do not implement or supersede the issue.

## Best Possible Solution

Keep #70797 open. Implement a code-span-aware raw OpenAI-style `tool_calls` JSON sanitizer in shared assistant-visible text handling and apply a streaming-aware accumulator/guard before partial preview delivery in `src/agents/pi-embedded-subscribe.handlers.messages.ts`. Add regressions in `src/shared/text/assistant-visible-text.test.ts` and `src/agents/pi-embedded-subscribe.handlers.messages.test.ts` for complete and truncated `tool_calls` JSON, while preserving ordinary JSON and fenced/inline examples.
