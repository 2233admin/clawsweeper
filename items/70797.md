## Summary

Keep open. The issue discussion reports a concrete recurring user-visible leak and a triage comment points to the exact sanitizer gap. Current main still returns bare `{"tool_calls":[...]}` text unchanged in the downgraded-tool-call sanitizer, and the partial reply path can still forward cleaned stream text to `onPartialReply` without a streaming-aware bare JSON guard. The two related fix PRs, #71954 and #72011, were closed unmerged in the provided context, so they do not supersede the issue.

## Best Possible Solution

Keep #70797 open. The best fix is to add a shared bare `tool_calls` JSON sanitizer that is safe around inline/fenced code and ordinary JSON, then use it both in final assistant-visible text cleanup and in the Telegram/direct partial preview path before `onPartialReply`. Add focused regressions in `src/shared/text/assistant-visible-text.test.ts` and `src/agents/pi-embedded-subscribe.handlers.messages.test.ts` for incomplete and complete `tool_calls` payloads with visible lead-in text.
