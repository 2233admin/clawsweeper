## Summary

Keep #69857 open. Current main still mints Ollama synthetic auth for any meaningful explicit Ollama config without classifying whether the configured baseUrl is local/LAN or remote, so the PR is not implemented or obsolete. The PR also has useful focused work and review feedback to address rather than a clean close path.

## Best Possible Solution

Keep the PR open and have it realign both Ollama synthetic-auth hooks around a shared, well-tested local/LAN baseUrl classifier. Prefer reusing existing net parsing helpers where possible, update or replace the current remote-provider synthetic-auth tests, cover Ollama Cloud/public-IP negative cases plus local/LAN/mDNS/bare-host positive cases, and leave #43945 open for the broader subagent credential and fallback behavior.
