## Summary

Keep open. Current main and latest release still do not gate Ollama synthetic auth by local/LAN baseUrl: the Ollama provider returns `ollama-local` for any meaningful explicit config, and the current test suite even asserts that a non-default remote Ollama baseUrl mints synthetic auth. The PR targets a real remaining gap, though the submitted patch should also cover the lightweight `provider-discovery.ts` hook and address the existing review comments.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch that adds a shared `isLocalOllamaBaseUrl` classifier, uses it in both Ollama synthetic-auth hooks, updates the existing remote-baseUrl tests to expect real credential fallback, and incorporates the loopback/IPv6 review feedback before merge.
