## Summary

Current `main` does not require `--agent` on `openclaw cron add`, does not warn when it is omitted, and still falls back to the default agent at run time. The issue remains valid.

## Evidence

- **`cron add` keeps `--agent` optional:** The CLI registers `--agent <id>` with `.option(...)`, not `.requiredOption(...)`, so omission is accepted.
  - file: [src/cli/cron-cli/register.cron-add.ts:77](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/cli/cron-cli/register.cron-add.ts#L77)

  - file: [src/cli/cron-cli/register.cron-add.ts:127](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/cli/cron-cli/register.cron-add.ts#L127)
