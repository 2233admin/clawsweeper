## Summary

Keep open. Current `main` still leaves Ollama without any provider-supplied catalog rows for the setup/model-picker path, and the picker only scopes to the selected provider when that provider already has catalog matches. That matches the PR’s reported failure mode, so there is not strong evidence to close this as already implemented or obsolete. The main follow-up I see is test coverage: the proposed Ollama hook follows established provider patterns, but the review thread correctly notes the PR should add a dedicated `augmentModelCatalog` test before landing.

## Evidence

- **Ollama provider still exposes discovery + model picker, but no catalog or `augmentModelCatalog` hook on current main:** In current `main`, the Ollama plugin registers `discovery` and `wizard.modelPicker` but there is no `catalog` or `augmentModelCatalog` entry in the provider definition. That means the picker has no bundled Ollama rows before discovery/auth runs.

If Ollama contributes zero rows, the picker falls back to showing other providers' models instead of an Ollama-scoped list.
