## Summary

Keep open. Current main still has the reported behavior for the `openclaw config set agents.list '[...]'` path: if `agents.list` contains one ACP agent and no explicit `default: true`, that first entry becomes the default route target, and there is no source-visible config-set warning or automatic insertion of `main`. The dedicated `openclaw agents add` helper has a safer path that preserves the existing default as an explicit first entry, but that does not cover the direct `config set agents.list` reproduction in this issue.
