## Summary

Keep this issue open. Current main still suppresses estimated dollar cost in both reported display paths when the provider auth is `aws-sdk` or other non-API-key modes, even though configured model pricing can be resolved independently of auth. The related same-author PR #53303 documented the right root cause but was closed unmerged, so there is no shipped fix on main.

## Best Possible Solution

Keep this issue open. Implement the display-layer fix by resolving explicit model pricing independently of auth mode, then show estimated cost in both `/status` and `/usage full` for `aws-sdk`, OAuth, token, mixed, and API-key providers when usable pricing is explicitly configured. Add regression coverage for a Bedrock-style `aws-sdk` provider with nonzero model cost and update the public token/cost docs to describe the new behavior.
