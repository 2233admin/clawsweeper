## Summary

Keep open. Current main has the related Linux child OOM victim-selection change, but that does not resolve this issue's central SIGKILL observability/reporting problem for broad exec discovery commands. The exec path still maps external signal exits to a generic `signal` failure with `Command aborted by signal ...`, and the Codex app-server command item type exposes exit code/duration/output but no signal or cause metadata.

## Best Possible Solution

Keep #69242 open as exec/process observability work. The best fix is to add structured kill metadata for exec results and logs: distinguish supervisor timeout/no-output/manual cancel from external OS signal, include runtime and configured timeout, run/session id or log pointer, stdout/stderr tail, and any available process/resource snapshot. The existing Linux `oom_score_adj` wrapper should remain as the separate victim-selection improvement from #70419.
