## Summary

Current main has related Linux child OOM victim-selection work, but it does not implement the core request in #69242: clear exec-tool reporting for intermittent Linux SIGKILLs on broad discovery commands when no host OOM evidence is visible. The related merged PR #70419 explicitly treated #69242 as separate SIGKILL observability work rather than the issue it fixed.

## Best Possible Solution

Keep #69242 open as the canonical Linux exec-tool SIGKILL observability/resource-reporting issue. A good fix should preserve the Linux child OOM-score behavior from #70419, then improve the exec/supervisor result path to distinguish known OpenClaw timeouts, manual/session cancellation, external signal kills, and probable kernel/cgroup OOM where evidence is available, with focused tests for child and PTY-backed SIGKILL result rendering.
