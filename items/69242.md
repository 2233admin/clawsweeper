## Summary

Keep open. Current main has the related Linux child OOM-score mitigation from #70419 and generic signal reporting, but that evidence does not prove the reported broad `find`/`grep -R` exec SIGKILLs are fixed or now explained with a clear resource/timeout cause. The related PR explicitly scoped #69242 as separate SIGKILL observability work, and current node-host exec still drops process exit signals in one path.

## Best Possible Solution

Keep the issue open. The best maintainer path is to either land/finish the structured signal-observability work tracked by #68680 and then reassess #69242, or add a focused exec regression path that distinguishes timeout, supervisor cancellation, external signal, and detectable Linux OOM/cgroup kills for broad discovery commands. The existing OOM-score wrapper should stay, but it is not sufficient closure evidence for this report.
