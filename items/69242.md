## Summary

Keep #69242 open. Current main has a Linux child OOM victim-selection mitigation from the related #70419 work, but that PR explicitly scoped #69242 as separate SIGKILL observability work. The foreground exec path still surfaces signal exits as a generic signal failure and does not include structured exitSignal/failureKind/terminationReason in ExecToolDetails, so the reported user-facing diagnostic gap remains actionable. The later #72240 report is related macOS evidence, not a Linux replacement for this issue.

## Best Possible Solution

Keep the issue open and implement exec SIGKILL observability in core: propagate RunExit.reason, exitSignal, failureKind, timeout/manual-cancel status, duration, output tail, and a run/log correlation id into ExecToolDetails and user-facing failure text; add regression tests for external SIGKILL versus timeout/manual-cancel; cross-link #72240 as related cross-platform evidence rather than replacing this Linux report.
