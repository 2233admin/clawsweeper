## Summary

Keep open. Current main still routes browser control through Playwright CDP in the gateway process, the global unhandled-rejection handler still exits on generic non-transient errors, and there is no browser-scoped Playwright assertion handler or worker/CDP isolation on main. Related #45791 and #52051 are still open and useful context, but they do not clearly supersede this specific CRSession._onMessage report.

## Evidence

- **Fatal default for generic unhandled rejections:** `installUnhandledRejectionHandler()` only suppresses registered plugin handlers, AbortError, config/fatal buckets, and transient network/SQLite cases; otherwise it logs `Unhandled promise rejection` and calls `process.exit(1)`. A plain Playwright `Error: Assertion error` fits the generic path.
