## Summary

Keep open. Current main already fixes the `--agent` + `--session-id` collapse by prioritizing a requested session id over the agent main-key fallback, but the PR also adds `--new-session`, and that user-facing flag is still absent from current main code and docs. The remaining work is still useful and is directly tracked by open issue #54864, so this PR is not fully obsolete.

## Best Possible Solution

Keep the PR open, but ask for it to be rebased and narrowed against current main: preserve the already-shipped `--session-id` precedence/scoped-store behavior, then add `--new-session` with mutual exclusion from `--session-id`, UUID generation before both gateway and local execution paths, JSON/stdout surfacing of the generated id, CLI docs, changelog, and focused tests. If another PR lands #54864 first, then this PR can be closed as superseded.
