## Summary

Keep #59130 open. Current `main` still matches the reported failure mode: `memory-lancedb` auto-recall retrieves top semantic vector matches and injects category/text-only memory context without freshness-intent handling, recency reranking, source timestamps, or assistant-visible provenance. The related PR #59141 is the right active fix path, but it is open and unmerged in the provided GitHub context, so the issue should remain as the canonical tracker until that or an equivalent fix lands.

## Best Possible Solution

Keep #59130 open as the canonical bug. Land or revise PR #59141, or an equivalent change, to detect freshness-sensitive prompts, widen/rerank candidates by recency, surface timestamp/provenance caveats in injected recall context, and add regression tests for a latest/newest prompt where semantic similarity would otherwise choose the older memory. Consider a narrower follow-up for structured source metadata if the minimal `createdAt` fix ships first.
