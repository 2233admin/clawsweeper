## Summary

Keep open. Current `main` does not contain the requested markdown-table integration coverage, and the relevant streaming chunking code on this path is still explicitly fence-aware rather than table-aware, so there is no strong basis for conservative cleanup.

## Evidence

- **Paragraph-preference integration coverage stops at fenced-code parity:** The current `subscribeEmbeddedPiSession` paragraph-preference integration test file contains only two cases: basic paragraph splitting and fenced code block preservation. There is no markdown-table case in this harness.
  - file: [src/agents/pi-embedded-subscribe.subscribe-embedded-pi-session.streams-soft-chunks-paragraph-preference.test.ts:7](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/pi-embedded-subscribe.subscribe-embedded-pi-session.streams-soft-chunks-paragraph-preference.test.ts#L7)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Current chunker safe-break logic is fence-only:** `EmbeddedBlockChunker` imports `parseFenceSpans` / `isSafeFenceBreak`, and its paragraph-break search filters only against fence spans. I did not find any table-aware break handling in this file.
  - file: [src/agents/pi-embedded-block-chunker.ts:1](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/pi-embedded-block-chunker.ts#L1)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Fence helper only models code fences:** `parseFenceSpans` recognizes backtick/tilde fences, and `isSafeFenceBreak` is just a fence-span containment check. There is no table-state tracking in the helper used by the chunker.
  - file: [src/markdown/fences.ts:1](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/markdown/fences.ts#L1)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Current unit coverage is also fence-focused:** `pi-embedded-block-chunker.test.ts` exercises paragraph boundaries, flush behavior, and fenced block behavior, but does not add a markdown-table scenario that would make this issue obsolete.
  - file: [src/agents/pi-embedded-block-chunker.test.ts:20](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/pi-embedded-block-chunker.test.ts#L20)
