## Summary

Keep open. Current main does not implement the PR's requested `model_failover` or `model_failure_terminal` plugin hooks, and the remaining work is a plausible core plugin API extension rather than ClawHub-only plugin work. Existing `model_call_started` / `model_call_ended` hooks provide adjacent call telemetry, but they do not expose failover decisions or terminal all-models-failed summaries to plugins.

## Best Possible Solution

Keep the PR open as the active implementation candidate for #70976. The best path is to tighten the requested hook payloads, document the new plugin hooks, add or preserve focused hook-runner and failover-emission tests, and then close #70976 only when the core plugin API ships.
