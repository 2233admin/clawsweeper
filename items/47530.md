## Summary

Current main already provides a generic pre-write interception path through plugin hooks. `before_tool_call` is a documented, public hook that receives the tool name and params before execution and can block, require approval, or rewrite params, and file mutations already route through the `write`, `edit`, and `apply_patch` tools.

## Evidence

- **Public pre-tool hook exists:** `PluginHookName` includes `before_tool_call`, and the typed event/result expose `toolName`, `params`, `block`, `blockReason`, and `requireApproval` for pre-execution enforcement.
  - file: [src/plugins/hook-types.ts:57](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/plugins/hook-types.ts#L57)
- **Pre-tool hook sees tool params before execution:** `runBeforeToolCall` is explicitly described as allowing plugins to modify or block tool calls before they run.
  - file: [src/plugins/hooks.ts:777](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/plugins/hooks.ts#L777)
- **Prompt injection hook already exists:** The agent-loop docs document `before_prompt_build` for injecting per-turn guidance into prompt/system context, which covers the 'reminder' part of the request when a plugin chooses to use it.
  - file: [docs/concepts/agent-loop.md:94](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/docs/concepts/agent-loop.md#L94)
- **File writes already use standard tools:** Core file mutation surfaces are the `write`, `edit`, and `apply_patch` tools, so a `before_tool_call` plugin can target exactly the write/edit path patterns described in the issue.
  - file: [src/agents/tool-catalog.ts:62](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/tool-catalog.ts#L62)
- **Released docs/code already include the hook:** The release tag `v2026.4.22` contains `before_tool_call` in `src/plugins/hook-types.ts` and documents it in `docs/concepts/agent-loop.md`, so this is not just unreleased main-only behavior.
  - command: `git show v2026.4.22:src/plugins/hook-types.ts | rg -n 'before_tool_call|before_message_write'; git show v2026.4.22:docs/concepts/agent-loop.md | rg -n 'before_tool_call|before_prompt_build'`

## Risks / Open Questions

- The exact proposed hook name (`file:preWrite` / `agent:beforeEdit`) does not exist verbatim; the existing supported path is the generic `before_tool_call` hook plus optional prompt injection via `before_prompt_build`.
- A plugin implementing this policy must match the relevant file-writing tools and path params itself.

## Close Comment

Closing from a Codex review because current OpenClaw already has the core mechanism this request asks for. Plugins can intercept file writes before execution with `before_tool_call` (`src/plugins/hook-types.ts:318`, `src/plugins/hooks.ts:777`), and the file-mutation surfaces are the standard `write`, `edit`, and `apply_patch` tools (`src/agents/tool-catalog.ts:62`). OpenClaw also already exposes `before_prompt_build` for per-turn prompt/system guidance (`docs/concepts/agent-loop.md:94`). This capability is present in released code/docs as well, so there is not a missing core pre-write enforcement seam to add here. If you still see a concrete gap beyond the existing hook path, please open a focused follow-up describing the exact behavior that cannot be implemented with `before_tool_call` + prompt injection.
