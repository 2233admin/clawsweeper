## Summary

Keep open. Current main fixed the original #42982 transcript-leak symptom for configured email patterns, but it still does not implement this PR's additive `logging.redactPatterns` contract, and the PR needs revision for the newer transcript redaction path and docs contract before it should land or be closed.

## Best Possible Solution

Keep the PR open for maintainer review or revision. Decide the `logging.redactPatterns` contract, implement one shared config-derived pattern resolver used by `resolveRedactOptions`, `redactToolDetail`, and persisted transcript redaction, preserve explicit caller-supplied pattern semantics if intended, then update docs/schema help and add tests proving custom config patterns do not drop default token/PEM redaction in each affected sink.
