## Summary

Keep open. Current `main` still returns and applies `agentModelOverride` during `openclaw agents add`, so the PR’s requested behavior is not already implemented on `main`.

## Evidence

- **`agents add` still bakes agent override into config:** In the interactive add flow, any `authResult.agentModelOverride` is immediately written into the new agent entry via `applyAgentConfig(..., { model: authResult.agentModelOverride })`. That preserves the provider default on the agent instead of letting it inherit `agents.defaults.model`.
  - file: [src/commands/agents.commands.add.ts:298](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/commands/agents.commands.add.ts#L298)
  - command: `nl -ba src/commands/agents.commands.add.ts | sed -n '260,330p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Provider auth path still returns override when `setDefaultModel=false`:** `applyAuthChoiceLoadedPluginProvider` restores the global default model, but still sets `agentModelOverride = applied.defaultModel` and returns it in the deferred path. `applyAuthChoicePluginProvider` does the same and also emits the agent-scoped "Default model set" note.
  - file: [src/plugins/provider-auth-choice.ts:282](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/plugins/provider-auth-choice.ts#L282)
  - command: `nl -ba src/plugins/provider-auth-choice.ts | sed -n '250,410p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Default-model helper still returns override in deferred path:** `applyDefaultModelChoice` still calls `noteAgentModel` and returns `{ config: nextWithModel, agentModelOverride: params.defaultModel }` when `setDefaultModel` is false.
  - file: [src/commands/auth-choice.default-model.ts:23](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/commands/auth-choice.default-model.ts#L23)
  - command: `nl -ba src/commands/auth-choice.default-model.ts | sed -n '1,120p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)

That is the opposite of this PR’s intended behavior.
