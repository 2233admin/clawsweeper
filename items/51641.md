## Summary

Keep open. Current `main` only routes diagnostic/plugin-load logs to stderr for `--json` or other `suppressDoctorStdout` flows, not for generic piped/captured stdout, so the broader bug this PR targets is not implemented on `main`. The PR also still has an unresolved review concern: its `!process.stdout.isTTY` rule would reroute normal subsystem info logs to stderr for all non-TTY runs, including daemon/container contexts.

## Evidence

There is no non-TTY branch on current `main`, so the PR's proposed change is not already implemented.
