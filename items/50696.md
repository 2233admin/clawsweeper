## Summary

Keep open. Current `main` still renders recent-session labels in the old `displayName || label || key` order, so the underlying UI bug is not implemented on `main`. But this PR's replacement is also incomplete: it still falls back to the raw key when `label === key`, while the existing shared session-display helper and tests on `main` already treat that case as a reason to prefer `displayName` instead.

## Evidence

- **Overview card still uses old fallback on main:** The recent sessions card in the overview still renders `blurDigits(s.displayName || s.label || s.key)`, so `displayName` wins over a human-readable `label` on current `main`. There is no equivalent fix in the checked-out branch.

That matches the PR review concern about avoiding raw keys when a saved label duplicates the key.

- Merging this PR as-is would partially improve the common case but diverge from the shared session display contract already tested elsewhere, leaving edge-case regressions around duplicate `label`/`key` values.
