## Summary

Keep this PR open. Current checked-out main still requires `onboardingScopes` for web-search setup visibility, while DuckDuckGo's current shared provider metadata still omits it. The PR is also paired with the same author's still-open bug report #65862, and the related alternate PR #70170 is closed unmerged.

## Best Possible Solution

Review or rebase this PR together with #65862. The narrow current-main fix should add `onboardingScopes: ["text-inference"]` to DuckDuckGo's shared web-search provider metadata, preserving both runtime and public contract paths, and add a focused DuckDuckGo provider/contract/setup assertion so wizard visibility does not regress.
