## Summary

Keep this PR open. Current main still suppresses text-only tool summaries for ordinary group chats without consulting the session verbose level, while docs and directive tests show `/verbose` is a persisted session setting and group-scoped. The PR remains a plausible focused fix, but it is stale against current main and needs rework around native-command behavior plus regression tests and a maintainer security/product review for group verbose exposure.

## Best Possible Solution

Keep this PR open, but require a rebase and update the implementation against current `dispatch-from-config.ts`. The best fix is to let explicit session verbose `on` or `full` opt ordinary non-Slack group chats into text tool summaries while preserving current native-command, sendPolicy, forum, Slack non-DM, media-only, exec-approval, and preview-suppression behavior unless maintainers intentionally change those contracts. Add focused regression tests and coordinate the broader config-option discussion in #54663.

- The PR needs focused regression coverage for ordinary group chats with verbose off/on/full, while preserving forum, Slack non-DM, sendPolicy deny, media-only, exec-approval, preview suppression, and native-command behavior.
