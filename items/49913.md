## Summary

Keep open. Current main still routes `pnpm check:docs` through the Unix-only `git ls-files ... | xargs oxfmt --check` docs formatting pipeline, and no replacement `scripts/check-docs-format.mjs` helper has landed. PR #49913 targets a real unresolved PowerShell docs-check issue (#44293), but the submitted branch still has unresolved review feedback about Windows `.cmd` shim spawning and the sibling write command still using `xargs`, so it needs changes rather than maintainer cleanup closure.

## Best Possible Solution

Keep the PR open or consolidate it with a canonical fix for #44293. The best implementation is a shared docs-format script or wrapper that lists tracked docs files without `xargs`, invokes the repo-local `oxfmt` in a Windows-safe way, chunks long file lists, and covers both `--check` and `--write` variants if the maintainer goal is full native PowerShell docs formatting support.
