## Summary

Keep open. Current main partially fixes future successful one-shot ACP runs by reconciling identity before close, but the startup reconciler still does not skip persisted one-shot or terminal ACP records. A pending identity with an acpxSessionId still enters the startup reconcile path and can produce the reported warning. The linked PR #72023 is still the relevant fix path, not a reason to close this issue yet.

## Best Possible Solution

Keep #72013 open until #72023 or an equivalent main-branch fix lands. The best fix is to make `AcpSessionManager.reconcilePendingSessionIdentities` skip one-shot and any legacy terminal ACP records before runtime ensure/reconcile, add focused regression tests in `src/acp/control-plane/manager.test.ts`, and keep the existing successful-one-shot post-turn identity reconciliation.
