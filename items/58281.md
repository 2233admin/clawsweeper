## Summary

Keep open. Current main still treats `$VAR` / `${VAR}` path text as literal input in shared user-path resolution, while PR #58281 is a focused fix for the still-open #53628 skill-install/workspace path bug. The PR likely needs maintainer review of the `$HOME` fallback concern before merge, but it is not obsolete or closeable.

## Best Possible Solution

Keep PR #58281 open and move it toward a mergeable fix: add env-reference expansion in `src/infra/home-dir.ts`, preserve existing home fallback semantics for `$HOME`/`${HOME}` or document the chosen behavior, and keep regression tests in `src/infra/home-dir.test.ts` for braced/unbraced vars plus the linked `${XDG_CONFIG_HOME}` workspace/skill-install path.
