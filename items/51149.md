## Summary

Current `main` already has the requested guardrails: commentary-phase assistant text is suppressed before delivery, assistant-visible text is sanitized before channel emission, internal runtime context blocks are stripped, and orphaned tool results are dropped by replay sanitization. The relevant guardrail files are unchanged between `v2026.4.22` and current `main`, so this appears to have shipped in that release.

## Close Comment

Closing as implemented on `main` after Codex review. The current delivery path suppresses commentary-phase assistant text before it reaches the chat surface (`src/agents/pi-embedded-subscribe.handlers.messages.ts:457`, `src/agents/pi-embedded-subscribe.handlers.messages.ts:624`), and assistant-visible text is routed through `sanitizeUserFacingText()` (`src/agents/pi-embedded-utils.ts:37`), which strips internal runtime context and rewrites raw runtime/API errors (`src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts:362`, `src/agents/internal-runtime-context.ts:1`). Regression coverage now includes commentary suppression, internal-context stripping, and orphaned tool-result sanitization (`src/agents/pi-embedded-subscribe.handlers.messages.test.ts:414`, `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts:226`, `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.test.ts:6`, `src/agents/pi-embedded-runner/run/attempt.test.ts:2068`). These guardrail files are unchanged from `v2026.4.22` to current `main`, so this behavior shipped by `v2026.4.22`. If you still hit a new leak path on a newer build, please open a fresh issue with the exact leaked text, affected channel/plugin, and a minimal repro.
