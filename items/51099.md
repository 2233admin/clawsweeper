## Summary

Keep open. Current `main` partially mitigates the report by defining a default sandbox image that installs `python3` and by documenting/build-prompting that image, but the runtime fallback still auto-tags plain `debian:bookworm-slim` as `openclaw-sandbox:bookworm-slim` and the sandbox fs bridge still invokes `python3` directly. That leaves the reported failure mode plausible on current `main`, so there is not strong evidence for an auto-close reason.
