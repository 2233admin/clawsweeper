## Summary

Keep this PR open. Current main still has the Mistral reasoning stream gap: non-string `delta.content` can be treated as text, and array/object `reasoning_content` fallback fields are ignored. The PR is still a plausible implementation candidate for the open Mistral crash bug, but it needs a focused rework before merge because it also carries unrelated logging-cache changes and has review-noted stream-order/cache risks.

## Best Possible Solution

Keep this PR open as an implementation candidate, but request a focused rework: rebase onto current main, split the unrelated logging-cache changes into a separate PR, keep the Mistral fix scoped to `src/agents/openai-transport-stream.ts`, normalize array/object `reasoning_content` and non-string content-block deltas without coercing them to text, preserve post-tool-call reasoning/text order, and add regression tests for string, array, object, and empty-block Mistral reasoning stream shapes. After a clean fix lands, close #67192 as implemented and close any duplicate PRs as superseded.
