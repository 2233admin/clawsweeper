## Summary

Keep open. The report’s root diagnosis does not match current `main` exactly: the macOS app has its own app LaunchAgent (`ai.openclaw.mac`) and intentionally manages the gateway through the shared gateway LaunchAgent (`ai.openclaw.gateway`), and the app now attaches to an already-running gateway instead of spawning a second one. But `doctor --fix` still has no explicit macOS-app guard/skip path, and I could not verify the launchd interaction on macOS from this checkout, so the specific conflict scenario is not disproved strongly enough for an auto-close.
