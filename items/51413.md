## Summary

Keep open. Current `main` still appears to have the reported gap: device-less non-Control-UI operator sessions only bypass device-identity enforcement when `sharedAuthOk` is true, while `evaluateMissingDeviceIdentity()` does not consider successful primary `token`/`password` auth. The PR's intended fix is not present on `main`, and the PR also still has an unresolved review note about missing `password`-branch coverage.

## Evidence

- **Current main still gates operator bypass only on `sharedAuthOk`:** `evaluateMissingDeviceIdentity()` allows operator sessions without device identity only via `roleCanSkipDeviceIdentity(params.role, params.sharedAuthOk)`. There is no `authMethod` parameter here on current `main`, so successful primary token/password auth cannot affect this decision.
