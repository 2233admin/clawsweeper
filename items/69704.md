## Summary

Current main already sends Brave API keys with the required X-Subscription-Token header in both web and llm-context modes, but PR #69704 is specifically a test-only regression PR and equivalent header/no-query-param coverage is not present on main. The PR remains useful for maintainer review rather than conservative auto-close.

## Best Possible Solution

Keep PR #69704 open for maintainer review or merge an equivalent test-only change in extensions/brave/src/brave-web-search-provider.test.ts. The best coverage is to assert that both Brave web and llm-context requests send X-Subscription-Token with the configured key and never place the key in apikey/key URL query parameters, preserving the fixed behavior from #69673.
