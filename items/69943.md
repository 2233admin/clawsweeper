## Summary

Keep #69943 open. Current main still persists session-memory transcript text verbatim, and the open fixing PR #69961 has not merged. The issue remains the canonical bug tracker rather than a cleanup candidate.

## Best Possible Solution

Keep #69943 open as the canonical tracker. Land or revise #69961, or implement an equivalent fix that sanitizes both transcript extraction and final memory writes using the existing assistant-visible scaffolding/token stripping where appropriate, adds regression tests for chat-template tokens, Qwen-style `<tool_call>` XML, orphan role markers, and `NO_REPLY`, and documents any required cleanup for already-written memory files.
