## Summary

Keep this issue open. Current main satisfies the model-attribution half via `messages.responsePrefix` templates, but the requested opt-in approval gate before model fallback execution is still missing from schema, runtime fallback control flow, docs, and tests. The latest discussion also confirms attribution is covered and fallback approval remains the actual gap.

## Best Possible Solution

Keep this open, preferably as the canonical fallback-approval tracker or retitle/split it so attribution remains documented as solved by `messages.responsePrefix`. Implement an opt-in per-agent fallback approval policy in the core model fallback path, with schema/types, docs, and tests for approval, denial, timeout, proceed/fail behavior, cancellation, and channel delivery through the existing approval/notification surfaces where appropriate.
