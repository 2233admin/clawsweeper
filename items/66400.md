## Summary

Keep this PR open. Current main still drops same-session TUI agent tool events when they arrive before a chat event registers the run, and the PR remains a focused implementation candidate with no obvious security-sensitive file surface.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to land this approach or an equivalent TUI fix that uses evt.sessionKey to adopt same-session early agent events, preserves activeChatRunId/isLocalBtwRunId/pendingOptimisticUserMessage/noteLocalRunId bookkeeping, updates the AgentEvent type, and adds focused TUI handler regression coverage for gateway tool-first ordering.
