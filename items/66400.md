## Summary

Keep open. Current `main` still has the reported gateway/TUI race: `handleAgentEvent` drops tool/lifecycle events for runs that have not yet been registered by a chat event, and the PR’s follow-up commit appears to address the review feedback rather than making the report obsolete.

## Evidence

- **Current main still drops unknown agent-event runIds:** `handleAgentEvent` computes `isKnownRun` from `activeChatRunId`, `sessionRuns`, and `finalizedRuns`, then returns early for any other run. There is no same-session preregistration path before this guard, so tool/lifecycle events that arrive before the first chat delta are still discarded on `main`.
  - file: [src/tui/tui-event-handlers.ts:400](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/tui/tui-event-handlers.ts#L400)
  - command: `nl -ba src/tui/tui-event-handlers.ts | sed -n '290,450p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Run registration still happens only from chat events:** On current `main`, `noteSessionRun(evt.runId)` and the first `activeChatRunId` assignment happen inside `handleChatEvent`, not `handleAgentEvent`. That matches the PR’s described race: if the first event is a tool/lifecycle event, the run is not known yet.
  - file: [src/tui/tui-event-handlers.ts:305](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/tui/tui-event-handlers.ts#L305)
