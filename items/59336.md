## Summary

Keep #59336 open. Current main still lacks the PR's fix: exec safe-bin normalization still materializes undefined-valued keys, raw round-trip validation still compares restored raw against the materialized runtime config, and the Control UI still disables Raw mode when the redacted snapshot raw is null. The related user reports and review comments show this is a plausible active product bug, not obsolete or superseded.

## Best Possible Solution

Keep the PR open, but require a maintainer-quality revision before merge: preserve the intended safe-bin clearing semantics without re-exposing inherited values, compare raw round-trip restoration against the authored/source snapshot where appropriate, add focused regression tests for undefined safe-bin keys and materialization-only diffs, and consider a separate fix/test for global raw string replacement corrupting unrelated keys.
