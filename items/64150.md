## Summary

Keep open. Current main still lacks the requested `Glob` -> `find` alias and does not register `find`/`grep`/`ls` in the OpenClaw coding tool set. The related #64582 is closed unmerged, so it does not supersede this PR. However, #64150 should not land as-is because the existing review comments identify real workspace/sandbox and policy-catalog gaps that a maintained fix needs to address.

## Best Possible Solution

Keep the PR open or ask for a maintainer-owned revision. The best fix is a guarded OpenClaw implementation that wires `find`/`grep`/`ls` through the same workspace/sandbox path protections as read/write/edit, adds them to `CORE_TOOL_DEFINITIONS` under the `fs` section and coding profile, and adds regression tests for OAuth `Glob`/`Grep`/`LS` mapping plus workspaceOnly, sandbox, and tool-policy behavior.
