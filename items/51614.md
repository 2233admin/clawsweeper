## Summary

Current `main` does not implement a WhatsApp channel/account config key for stale-event threshold. The only shipped stale threshold config is the global `gateway.channelStaleEventThresholdMinutes`, while `channels.<provider>.healthMonitor` still exposes only `enabled`. The WhatsApp runtime path also still defaults `messageTimeoutMs` to 30 minutes and is not wired from channel config.
