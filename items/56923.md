## Summary

Current `main` does not include the proposed fix, and the reported false-positive still follows from the present audit logic. `listKnownNodeCommands()` only gathers commands from `resolveNodeCommandAllowlist()`, which excludes `DEFAULT_DANGEROUS_NODE_COMMANDS` unless the user also adds them to `allowCommands`, so entries like `camera.snap` or `sms.send` in `gateway.nodes.denyCommands` are still treated as unknown.

## Evidence

- **Known-command source omits dangerous commands:** `listKnownNodeCommands()` builds its set only from `resolveNodeCommandAllowlist(baseCfg, { platform })` and returns immediately after that loop; it does not add `DEFAULT_DANGEROUS_NODE_COMMANDS`.
  - file: [src/security/audit-extra.sync.ts:175](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/security/audit-extra.sync.ts#L175)
  - command: `nl -ba src/security/audit-extra.sync.ts | sed -n '160,220p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Dangerous commands are separate from the allowlist defaults:** `DEFAULT_DANGEROUS_NODE_COMMANDS` contains `camera.snap`, `screen.record`, `sms.send`, and related commands, while `resolveNodeCommandAllowlist()` starts from platform defaults plus `allowCommands`, then removes `denyCommands`. Those dangerous commands are not part of the default platform lists unless explicitly allowed.
