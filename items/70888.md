## Summary

Keep PR #70888 open. Current main still has the exact no-cfg early return in subagent depth resolution, and the announce path still calls that resolver without cfg when deciding internal vs external delivery. The PR discussion and bot review describe a concrete bug, root cause, and regression coverage; no allowed conservative close reason applies.

## Best Possible Solution

Keep the PR open and ask for a narrowly scoped, current-main branch that removes the no-cfg early return in `src/agents/subagent-depth.ts`, adds the explicit requester/default-store regression tests, and either drops or separately justifies the unrelated Bedrock Mantle, WhatsApp, and QA helper changes.
