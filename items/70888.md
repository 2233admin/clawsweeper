## Summary

Keep #70888 open. Current main still contains the no-cfg early return that prevents default on-disk session-store lookup, and the announce path still calls the depth resolver without cfg before deciding whether to treat a requester as internal. The PR appears to address a real remaining bug, though it should likely be narrowed before merge because it carries unrelated extension/SDK/WhatsApp drift.

## Best Possible Solution

Keep the PR open as a plausible real bug fix, but ask for a narrowed merge shape: land the src/agents/subagent-depth.ts no-cfg default-store fix with focused regression coverage in src/agents/subagent-depth.test.ts and src/agents/subagent-announce-delivery.test.ts, address the temp-dir cleanup review comment, and drop or split the unrelated extension/SDK/WhatsApp drift unless maintainers explicitly want it bundled.
