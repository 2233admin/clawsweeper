## Summary

Keep open. Current `main` still uses a single per-agent `sessions.json` metadata store and serializes all store mutations through a store-path-scoped queue plus on-disk lock, so the architectural contention described in #42160 is still present on `7ac35b4f69009490180a35bf404638d4280a4865`.

## Evidence

- **Default session store is still one `sessions.json` file per agent:** `resolveStorePath()` still defaults every agent to `.../sessions/sessions.json`; there is no directory-per-session metadata layout on current `main`.
  - file: [src/config/sessions/paths.ts:287](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/config/sessions/paths.ts#L287)
  - command: `nl -ba src/config/sessions/paths.ts | sed -n '280,340p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Docs still describe a monolithic session store:** User-facing docs say discovered stores must be regular `sessions.json` files and document the session store as a key/value map `sessionKey -> SessionEntry` stored at `~/.openclaw/agents/<agentId>/sessions/sessions.json`.
  - file: [docs/cli/sessions.md:29](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/cli/sessions.md#L29)
  - command: `nl -ba docs/cli/sessions.md | sed -n '20,90p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Every update re-reads and rewrites the whole store under a single lock:** `updateSessionStore()` acquires `withSessionStoreLock(storePath, ...)`, re-loads the full store with `loadSessionStore(storePath, { skipCache: true })`, mutates it, and `saveSessionStoreUnlocked()` later serializes the full object with `JSON.stringify(store, null, 2)`.
  - file: [src/config/sessions/store.ts:421](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/config/sessions/store.ts#L421)
  - command: `nl -ba src/config/sessions/store.ts | sed -n '232,439p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Locking is scoped to the store path, not the individual session key:** The in-process queue is keyed by `storePath` (`LOCK_QUEUES: Map<string, SessionStoreLockQueue>`), `withSessionStoreLock()` defaults to a 10s timeout, and the underlying file lock is `${sessionFile}.lock`, which throws `session file locked (timeout 10000ms)` on contention.
  - file: [src/config/sessions/store.ts:620](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/config/sessions/store.ts#L620)
