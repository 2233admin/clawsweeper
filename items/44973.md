## Summary

Current `main` still contains the duplicate-delivery path this PR targets. Main-session cron jobs enqueue the reminder as a system event, heartbeat prompt resolution still wraps cron-tagged system events into the scheduled-reminder prompt, and reply assembly later drains the same queued system events into `System:` lines. The proposed `:main-system-event` carveout is not present on `main`, so this PR still addresses a real gap rather than something already fixed.

## Evidence

There is no special marker to distinguish already-injected main-session system events.
