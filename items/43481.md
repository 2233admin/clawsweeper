## Summary

Keep this issue open. Current main only strips explicit image inputs on every fallback retry; it does not implement provider-level/configurable trust boundaries, and prompt-detected images can still be loaded during fallback. There is also an open same-author PR (#57959) with closing syntax for this issue, so the issue should stay open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Keep this issue open while maintainers review, update, land, or close PR #57959. The best implementation path is a focused fallback-boundary change that threads primary/target provider context through runWithModelFallback and runAgentAttempt, centralizes a tested image trust decision, suppresses prompt-detected image loading when the fallback target is not trusted, and documents any user-facing trust configuration if maintainers choose configurable behavior over conservative cross-provider stripping.
