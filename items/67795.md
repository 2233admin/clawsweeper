## Summary

Keep open. Current `main` still hard-codes Linux detection to `linux-gnu*`, while the installer docs promise generic Linux support and the reported openSUSE `OSTYPE=linux` case would still fall through to `Unsupported operating system`. The proposed one-line change is not implemented on `main`, and the PR remains a plausible, low-risk bug fix rather than cleanup material.

## Evidence

- **Current main still rejects non-gnu Linux OSTYPE values:** `detect_os_or_die()` on current `main` only accepts `darwin*`, `linux-gnu*`, or WSL. There is no broader `linux*` match in the checked-out code.
  - file: [scripts/install.sh:258](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/scripts/install.sh#L258)
  - command: `nl -ba scripts/install.sh | sed -n '252,266p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Installer docs promise Linux support broadly:** The public installer docs say `install.sh` supports macOS and Linux, including WSL. That contract is broader than the current shell match in `scripts/install.sh`.
  - file: [docs/install/installer.md:71](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/docs/install/installer.md#L71)
  - command: `nl -ba docs/install/installer.md | sed -n '61,80p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Direct evaluation of the current condition reproduces the reported gap:** Evaluating the same conditional logic shows `linux -> unknown`, `linux-gnu -> linux`, and `linux-musl -> unknown`, which is consistent with the reporter's openSUSE screenshot and indicates the current matcher is too narrow.
  - file: [scripts/install.sh:258](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/scripts/install.sh#L258)
  - command: `bash -lc 'for ost in linux linux-gnu linux-musl; do if [[ "$ost" == "linux-gnu"* ]] || [[ -n "${WSL_DISTRO_NAME:-}" ]]; then printf "%s -> linux\n" "$ost"; else printf "%s -> unknown\n" "$ost"; fi; done'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **PR discussion already identified the intended fix as backwards-compatible:** The PR changes only one line, from `linux-gnu*` to `linux*`, and the existing review comment notes that this preserves previous matches while covering additional Linux distros such as openSUSE and Alpine.
