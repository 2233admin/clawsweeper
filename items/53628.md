## Summary

Keep this issue open. Current main still does not expand raw `$VAR` or `${VAR}` placeholders in the shared user-path resolver, the skill install path still depends on the resolved active workspace, and the provided GitHub context shows open unmerged PRs that directly target this bug, including one with closing syntax.

## Best Possible Solution

Keep this issue open. Review and either land or close the linked implementation PRs, preferably consolidating on one fix in the shared user-path resolver with regression coverage for `${XDG_CONFIG_HOME}` and `$XDG_CONFIG_HOME` so ClawHub skill installs cannot create literal placeholder directories. After the selected PR merges, close this issue as implemented.
