## Summary

Keep open. Current `main` still does not include the cron guard this PR is about: `onTimer` and `runDueJobs` both call `collectRunnableJobs(...)` without `skipAtIfAlreadyRan`, while `isRunnableJob` only suppresses already-completed one-shot `kind: "at"` jobs when that flag is set. The startup catch-up path does pass the flag, so the gap described in the PR still exists in current code. The PR is also mixed-scope and includes unrelated changes, with an existing review comment flagging a regression in the session-transcript repair part, so there is no strong basis to close it as already handled.
