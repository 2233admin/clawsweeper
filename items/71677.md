## Summary

Keep open. Current main still accepts Feishu card.action.trigger events, deduplicates callback tokens, and dispatches the synthetic command, but it does not return an instant card update/toast response or expose an ocf1 disable-state field for button UX.

## Best Possible Solution

Keep the issue open and implement this in the bundled Feishu plugin: extend the ocf1 interaction contract with typed opt-in disable-state metadata, have card.action.trigger return an immediate Feishu callback response with a disabled/transitional card and optional processing toast while dispatching the synthetic command asynchronously, preserve duplicate-token protection, and add focused tests for the returned callback payload plus existing quick-action and approval flows.
