## Summary

Keep open. Current `main` still treats plugin configuration as global `plugins.entries.<id>.config`; agent entries have no `plugins` field in the strict config schema, and agent-resolution code does not merge any per-agent plugin overrides. This is an unresolved core config/plugin-SDK gap, not something already implemented on `main` or cleanly closable as a ClawHub-only request.

## Evidence

- **Strict agent schema has no `plugins` key:** `AgentEntrySchema` enumerates supported per-agent keys (`skills`, `sandbox`, `params`, `tools`, `runtime`, etc.) and then ends with `.strict()`. There is no `plugins` property, so `agents.list[].plugins` is rejected by current schema validation.
  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/zod-schema.agent-runtime.ts#L812)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Type-level agent config also lacks plugin overrides:** `AgentConfig` defines the supported per-agent fields and does not include any `plugins` member. That matches the validation behavior reported in the issue.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/types.agents.ts#L70)
