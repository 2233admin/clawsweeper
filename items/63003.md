## Summary

Current main still does not implement the requested `sandbox.mounts` JSON field or `Effective mounts:` text output. The paired issue #62988 was closed as superseded by this PR, so this PR remains the active implementation candidate rather than something to close. The current PR direction is useful, but its diff still needs maintainer cleanup because mount reporting is security-boundary introspection and must match canonical session status and the selected sandbox backend.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Revise it to compute sandboxed status through `resolveSandboxRuntimeStatus`, derive effective mounts from the selected sandbox backend rather than only Docker semantics, keep the requested JSON and text output, add focused coverage for JSON, text, Docker binds, direct runtime, session aliases, agent workspace overrides, and remote roots, and remove unrelated test-file churn before maintainer review.
