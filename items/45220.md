## Summary

Keep open. Current `main` at `af56926e2fc410809fcb0399435787c7fa8daef8` still treats `logs` differently from the PR: `logs` is allowed as an invalid-config diagnostic command, but it is not exempted from doctor state-migration preflight, and the existing tests show that unsuppressed preflight can emit stdout noise. The PR remains a plausible, targeted fix rather than something already implemented or obsolete.
