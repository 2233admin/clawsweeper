## Summary

Keep #58110 open. Current main has meaningful multi-agent foundations, including isolated agents, per-agent session stores, sub-agents, and a chat UI session selector, but it does not implement the requested chat-application paradigm: agents as contact-style sidebar entries, separate agent chat windows surfaced in the chat UI, or user-created group chats that invite multiple agents into one collaborative conversation. Related ClawSweeper context supports keeping this open: #22774, #43235, and #43750 remain open for important multi-agent UI/routing gaps, while #18869 only closed the narrower turn-taking protocol idea as ClawHub/plugin scope.

## Best Possible Solution

Keep the issue open as a core product/UI umbrella unless maintainers choose to split it. The best path is a scoped Control UI design that builds on existing agent/session/sub-agent primitives: a contact-style agent/session navigator with status, per-agent conversation history, durable sender/agent attribution in transcripts and Chat UI rendering, and an explicit group-collaboration model backed by routing/thread/session APIs. Link or depend on narrower open items such as #22774, #43235, and #43750 rather than closing this as already implemented or ClawHub-only.
