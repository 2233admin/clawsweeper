## Summary

Keep open. Current `main` still routes drained system events into the user prompt path, and transcript/history filtering only strips heartbeat prompt/ACK pairs, not cron or exec-event injections. That means the reported webchat pollution path is still supported by code on `main`, so there is not strong evidence for `implemented_on_main` or `cannot_reproduce`.
