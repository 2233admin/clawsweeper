## Summary

Current `main` still matches the report. `isHeartbeatEnabledForAgent()` ignores `agents.defaults.heartbeat` when deciding which agents are enabled, and the fallback still enables heartbeat for the default agent even with no heartbeat config. The existing test suite also still asserts the defaults-only bug as expected behavior.

## Evidence

- **Defaults-only heartbeat still ignored in enablement check:** `hasExplicitHeartbeatAgents()` only scans `agents.list[].heartbeat`. If no per-agent heartbeat block exists, `isHeartbeatEnabledForAgent()` falls back to `resolvedAgentId === resolveDefaultAgentId(cfg)`, so a defaults-only config enables only the default agent instead of all agents inheriting defaults.

Combined with the default-agent fallback above, the default agent remains enabled on current `main` even with no heartbeat configured.
