## Summary

Keep PR #69982 open. Current `main` still has the reported listener cleanup gap: Cancel and Delete remove the delete-confirmation popover without unregistering the document click listener, and the latest release tag still has the same behavior. The PR is a plausible focused fix, though it should add regression coverage and should also consider the existing-popover toggle path.

## Best Possible Solution

Keep the PR open and either update it or cherry-pick the fix into `ui/src/ui/chat/grouped-render.ts`. The maintainer-quality version should route every dismissal path through one cleanup helper, including Cancel, Delete, outside-click, and the existing-popover toggle, then add a focused `grouped-render.test.ts` regression test that observes document listener removal.
