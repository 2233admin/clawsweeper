## Summary

Keep PR #69982 open. Current main at 43a003b8 still has the listener cleanup gap described by the PR: Cancel and Delete remove the popover DOM node without unregistering the document click listener, while only the outside-click path removes that listener. The provided PR discussion also supports the patch as a minimal fix, with Greptile finding only a style nit.

## Best Possible Solution

Keep the PR open. The best maintainer path is to land or cherry-pick the shared dismissPopover cleanup approach, optionally simplify the Cancel listener per the bot nit, and add a focused regression test proving Cancel, Delete, and outside-click all unregister the document click listener.
