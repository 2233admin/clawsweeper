## Summary

Keep PR #40522 open. Current main still lacks the requested root-to-Control-UI-basePath redirect, so GET/HEAD `/` with `gateway.controlUi.basePath` configured still falls through toward the gateway's plain `Not Found` response. The PR remains a narrow fix for the linked #34298 behavior, and the provided PR discussion shows the review-requested query-string test was addressed.

## Best Possible Solution

Keep the PR open. A maintainer should merge it or reapply the same narrow change on main: when `gateway.controlUi.basePath` is non-empty, redirect read-method requests for `/` to `${basePath}/` while preserving the query string, keep non-read root requests falling through for plugin/webhook safety, and retain regression tests for GET root redirect, query-string preservation, and POST fall-through.
