## Summary

Keep this PR open. Current main still does not redirect gateway root `/` to the configured Control UI `basePath`; the PR remains a narrow implementation candidate for the linked Control UI `Not Found` bug, with tests added for the read-method redirect, query-string preservation, and write-method fallthrough.

## Best Possible Solution

Keep this PR open. A maintainer should review/rebase and land or reapply the narrow gateway routing change: in `classifyControlUiRequest`, when `basePath` is non-empty and `pathname === "/"`, redirect `GET` and `HEAD` to `${basePath}/${search}`, preserve non-read fallthrough for plugin/webhook paths, and keep regression tests for root redirect, query preservation, and write-method fallthrough. Consider a small basePath normalization hardening follow-up if protocol-relative base paths are not intended.
