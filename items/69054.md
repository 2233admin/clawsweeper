## Summary

Keep #69054 open. Current main has first-class optional `delivery.accountId` plumbing, but it does not implement `openclaw cron lint`, does not warn at gateway startup, and does not require `delivery.accountId` for announce-mode cron jobs targeting multi-account channel configs. The related #69055 is a companion logging-attribution request, not a duplicate, and the nearby implemented multi-channel validation only covers missing `delivery.channel`, not missing account identity.

## Best Possible Solution

Keep the issue open and implement it in core cron surfaces: add a reusable cron lint module that can count configured accounts per channel, expose `openclaw cron lint --warn-only`, call it from gateway startup during the migration window, enforce it in `cron.add`/`cron.update` after the deprecation plan, and document `--account`/`accountId: "default"` migration guidance with focused unit and gateway method tests.
