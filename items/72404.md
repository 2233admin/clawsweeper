## Summary

Keep open. Current main does not implement PR #72404's requested behavior: explicit-only model entries without `input` are still preserved unchanged, and downstream image handling still treats missing `input` as not image-capable. The PR is a focused attempt to fix the still-open #71921 Bedrock Claude vision bug, but the review comments identify concrete false-positive risks in the proposed regex defaults, so this should remain open for maintainer rework rather than cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a narrower fix: preserve explicit `input` opt-outs, default only well-bounded known vision IDs or provider-owned Bedrock Claude IDs, route all explicit-only merge paths through the same helper, and add regression tests for Bedrock Claude stale static catalogs plus false-positive cases such as Llama 3.2 1B/3B instruct and arbitrary IDs containing `o1`, `o3`, or `o4`. A provider-owned Bedrock discovery repair may also be needed for the root first-boot/stale-catalog path.
