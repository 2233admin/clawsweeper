## Summary

Keep open. Current `main` still limits SecretRef to explicit credential surfaces, and the phone-number/identifier fields called out by the issue remain plain-string schemas rather than `SecretInput`/`SecretRef`-aware fields.

## Evidence

- **SecretRef support is opt-in per schema field:** Core config defines `SecretInputSchema` as `string | SecretRef`, so a field only accepts SecretRef when its schema uses that type.
  - file: [src/config/zod-schema.core.ts:84](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/config/zod-schema.core.ts#L84)
