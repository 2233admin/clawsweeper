## Summary

Keep open. Current main has partial static request-body override support through `agents.defaults.models.*.params.extraBody`, but it does not implement the issue's core request: provider API calls cannot automatically include per-session/per-graph/per-task metadata such as OpenRouter `trace` or `user`, and the model-provider config schema has no `extraBody`/`setUserToSessionKey` field. The related PR #64813 is an open/unmerged attempt rather than shipped behavior.

## Best Possible Solution

Keep the issue open and resolve it with a narrow provider-runtime design: add an explicit, validated way to pass safe per-request metadata for OpenAI-compatible providers, likely gated by provider/API capability, with session/agent/task context plumbing, schema/docs updates, and regression tests. The related #64813 direction can inform the implementation, but it should be completed with provider-specific compatibility checks before closing this issue.
