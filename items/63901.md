## Summary

Keep open. Current main still lacks the requested session-aware provider request metadata: `extra_body`/`extraBody` is static request-body passthrough, provider extra-param hooks do not receive `sessionKey`/`sessionId`, and OpenRouter does not inject `user` or `trace`. The same-author paired PR #64813 is still an implementation candidate per the provided GitHub context, so this issue should not be closed ahead of resolving that PR.

## Best Possible Solution

Keep this issue open. The best path is to review and either land or explicitly close #64813 and #54725, preferably converging on a narrowly scoped session-aware provider attribution seam or request-body option with coverage for `sessionKey`, `sessionId`, agent id, static `extra_body` precedence, and OpenRouter/LiteLLM `user`/`trace` compatibility.
