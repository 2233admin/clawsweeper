## Summary

Current main already fixes the two runtime-dependency behaviors this PR targets, and does so with a safer disabled-channel implementation than the PR diff. Bundled runtime installs now use an OpenClaw-owned cache under the install execution root instead of the user's npm cache, packaged/global installs stage deps in an external runtime root, and doctor/health scans preserve explicit plugin/channel disablement even with includeConfiguredChannels enabled.

## Best Possible Solution

Close this PR without merging it. Keep the current main implementation: external/runtime-root staging plus an OpenClaw-owned npm cache for bundled runtime-dep repairs, and the existing hasExplicitChannelDisable behavior that prevents disabled default-enabled channel plugins from falling through to enabledByDefault. Any remaining operator guidance should continue in the runtime-deps docs or the still-open #71070 discussion rather than through this stale PR diff.
