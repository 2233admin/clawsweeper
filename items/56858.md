## Summary

Keep open. Current `main` does not implement the PR’s `cacheMedia` behavior: context pruning still replaces images with `[image removed during context pruning]`, there is no `cacheMedia` config/schema/docs surface, and the runtime has no disk-cache write or cached-marker reinjection path.
