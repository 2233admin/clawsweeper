## Summary

Keep PR #69859 open. Current main still does not accept or propagate `threadId` from the direct `POST /hooks/agent` payload, while downstream cron delivery already supports explicit delivery thread IDs. The PR is relevant, but it needs a focused rework before merge because the reviewed head included unrelated changes, a backup artifact, and incomplete propagation concerns.

## Best Possible Solution

Keep the PR open, but request a focused revision or create a maintainer replacement PR. The correct path is to add `threadId` to direct `/hooks/agent` payload normalization, include it in replay/idempotency scope, propagate it into the hook-created cron `delivery.threadId` path, update public hook docs, and add gateway/cron tests proving a posted Slack thread timestamp reaches outbound delivery.
