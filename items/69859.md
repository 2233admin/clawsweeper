## Summary

Keep PR #69859 open. Current main still does not accept or forward `threadId` from `POST /hooks/agent`, so the central user problem remains. However, the PR is not merge-ready as submitted: main now has generic cron `delivery.threadId` support, while this PR was written against an older path, includes an unrelated model-selection fix, and adds an accidental backup file per the review context.

## Best Possible Solution

Keep the PR open as a real product gap, but ask for a focused rework: remove the backup file and unrelated model-selection commit, parse and validate `threadId` in `normalizeAgentPayload`, include it in the hook dispatch/replay surface, place it on cron `delivery.threadId` for announce delivery, and add gateway plus cron delivery tests and docs.
