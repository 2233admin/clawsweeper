## Summary

Keep PR #70598 open. Current main already implements the original exec-detail prompt path, but it still lacks the PR's later deferred-event preservation fix for mixed exec/non-exec heartbeat queues. The remaining behavior is user-visible: main filters exec events for prompt construction, then consumes the whole inspected pending-event prefix, so cron/reminder events queued alongside an exec completion can be removed without being rendered or relayed.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase and validate the branch, or extract the remaining `restoreSystemEventEntries` and deferred-event preservation behavior plus regression tests onto current main. Do not close as implemented until mixed exec plus cron queues preserve non-exec events, including duplicate deferred reminders.
