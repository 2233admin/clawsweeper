## Summary

Keep open. Current `main` still stores sandbox backend registrations in a module-local `Map`, while plugin backends register through the plugin SDK and runtime lookup happens later from core. The issue-linked fix to move this registry onto `globalThis` is not present in `7e52223d327555c611e8c91ee2500897e9eab0bf`, and current tests do not cover cross-module-instance registration, so there is not strong evidence the reported bug is fixed or obsolete.

## Evidence

- **Sandbox backend registry is still module-local:** `src/agents/sandbox/backend.ts` defines `const SANDBOX_BACKEND_FACTORIES = new Map...` at module scope and `requireSandboxBackendFactory()` reads from that same map only; there is no process-global singleton/shared registry in current main.
  - file: [src/agents/sandbox/backend.ts:26](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/agents/sandbox/backend.ts#L26)
