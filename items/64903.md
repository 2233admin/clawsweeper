## Summary

Keep open. Current main still contains the reported Android foreground-service path: app launch starts NodeForegroundService, service creation immediately calls startForeground through ServiceCompat, and the base foreground service type remains dataSync on targetSdk 36. I found no current source, test, docs, or related-item evidence that the dataSync quota/ForegroundServiceStartNotAllowedException path has been fixed or superseded.

## Best Possible Solution

Keep this issue open and fix it in the Android app. The likely maintainer path is to stop relying on a long-running dataSync foreground service for the always-on gateway connection on targetSdk 36, add explicit handling for startForeground failures and dataSync quota exhaustion, implement required timeout/demotion/stop behavior, and cover the lifecycle with focused Android tests plus real-device or emulator proof.
