## Summary

Keep PR #39761 open. Current main still does not expose agent reasoning metadata to the `message_sending` plugin hook, and the PR remains a plausible backward-compatible plugin API addition rather than obsolete cleanup. The related #69110 model-identity issue is adjacent, not a complete replacement for tool/token/session context in `message_sending`.

## Best Possible Solution

Keep PR #39761 open for maintainer review or adaptation. The best path is a backward-compatible optional `agentContext` or equivalent typed field on `PluginHookMessageSendingEvent`, threaded through `DeliverOutboundPayloadsParams` and `applyMessageSendingHook()`, populated from embedded-runner metadata for agent-originated sends, documented in `docs/plugins/hooks.md`, and covered by focused hook contract tests. Coordinate provider/model identity with #69110 instead of closing this broader tool/token/session context request as obsolete.
