## Summary

Keep open. Current `main` has adjacent pieces for provider usage visibility and friendly billing-error messaging, but it does not implement the requested per-bot/public monitoring URL or low-credit alert delivery. The report is coherent enough to remain actionable.

## Evidence

- **Provider usage surfaces exist, but they are generic admin surfaces:** Docs describe provider usage/quota in `openclaw status --usage` and Control UI, normalized as `% left`, not a per-bot public page.
  - file: [docs/cli/index.md:366](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/docs/cli/index.md#L366)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Gateway protocol exposes usage APIs for dashboard/session views:** `usage.status`, `usage.cost`, and `sessions.usage*` RPCs exist, which supports current usage reporting but does not match the requested per-bot monitoring URL or alerting flow.
  - file: [docs/gateway/protocol.md:281](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/docs/gateway/protocol.md#L281)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Control UI usage page is dashboard/session-oriented:** The UI loads `sessions.usage` and `usage.cost`, then optional per-session timeseries/logs; this is an operator dashboard, not a per-bot owner URL.
  - file: [ui/src/ui/controllers/usage.ts:180](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/ui/src/ui/controllers/usage.ts#L180)

  - file: [ui/src/ui/views/overview-cards.ts:177](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/ui/src/ui/views/overview-cards.ts#L177)
