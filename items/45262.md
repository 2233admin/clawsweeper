## Summary

Keep open. Current `main` still requires non-empty Slack slash text and does not expose a `channels.slack.slashCommand.contextAware` setting in config types, schema, or docs, so the requested behavior is not implemented. The PR also still has substantive design/security concerns on today’s code: slash directive parsing falls back to prompt text, so embedding raw recent channel history into the slash prompt would let prior channel messages influence `/think`-style inline directives, and the repo’s current guidance is to attach channel context as structured `UntrustedContext` rather than flattening inbound history into prompt text.
