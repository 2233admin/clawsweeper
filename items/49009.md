## Summary

The request is for a LanceDB memory plugin variant, and current main explicitly scopes optional capability like this to plugins distributed outside core. OpenClaw already has the memory-plugin slot, external plugin install flow, community-plugin policy, and a bundled `memory-lancedb` plugin, so this should live as a ClawHub/npm plugin rather than an OpenClaw core issue.

## Evidence

## Close Comment

Closing as `clawhub` after Codex review. Current `main` already scopes this kind of work to plugins rather than OpenClaw core: `VISION.md:54` says optional capability should usually ship as plugins, plugin authors should host and maintain them in their own repos, and the bar for adding optional plugins to core is intentionally high. The repo also already has the memory-plugin slot and external plugin install path (`docs/tools/plugin.md:11`, `docs/plugins/community.md:9`), plus a bundled LanceDB memory plugin with its own tools and CLI (`extensions/memory-lancedb/index.ts:297`). An enhanced LanceDB variant like `memory-lancedb-pro` is better maintained and distributed as a ClawHub/npm plugin, not tracked as a core OpenClaw issue. If there is a specific missing SDK seam or host limitation blocking that external plugin, please open a focused issue for that gap.
