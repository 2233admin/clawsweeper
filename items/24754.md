## Summary

Keep open. Current main already has browser chat via the Gateway Control UI/WebChat, but the issue’s proposed native E2EE browser chat layer with QR-based device onboarding and no shared third-party transport is not implemented on `main`. The request is coherent and appears to fit OpenClaw core/web frontend scope rather than ClawHub.
