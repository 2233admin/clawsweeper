## Summary

Current `main` does not contain this Mattermost threading fix. The code still routes `replyToMode: "off"` thread replies, typing indicators, and `MessageThreadId` through `effectiveReplyToId`, which becomes `undefined` for threaded messages in that mode, so the PR remains relevant.
