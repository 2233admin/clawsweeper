## Summary

Keep open. Current `main` still appears to block provider/image-generation requests when public provider hostnames resolve into RFC 2544 fake-IP space. The SSRF layer has an `allowRfc2544BenchmarkRange` escape hatch, but provider transport/config only wires `allowPrivateNetwork` for trusted self-hosted endpoints; the fake-IP compatibility knob is exposed for `web_fetch` and Telegram, not provider API traffic.

## Evidence

- **Provider transport only forwards `allowPrivateNetwork`:** `buildGuardedModelFetch()` resolves provider request policy, then calls `fetchWithSsrFGuard()` with only `policy: { allowPrivateNetwork: true }` when enabled. There is no provider-side path here for `allowRfc2544BenchmarkRange` or any fake-IP-specific SSRF override.
