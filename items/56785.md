## Summary

Keep open. Current `main` partially supersedes the local Docker write/edit path by selecting `python3` or `python` and emitting an explicit missing-runtime error, but it does not implement this PR's full behavior: the remote-shell sandbox bridge still hardcodes `python3 /dev/fd/3` with no friendly translation layer, and current remote tests do not cover the missing-`python3` failure mode.

## Evidence

That addresses part of the original UX problem for the local fs bridge without this PR's `toFriendlySandboxMutationError` wrapper.
