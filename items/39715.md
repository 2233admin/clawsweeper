## Summary

Keep PR #39715 open. Current main still resolves the normal child close path and snapshots background exec output without an event-loop drain, so the PR's stdout/stderr-at-exit data-loss path is not implemented or superseded. The related merged sandbox stdin/write fix addresses a different pipe direction.

## Best Possible Solution

Keep PR #39715 open. The best path is to land or adapt a minimal stdout/stderr drain before RunExit and finished-session aggregation are snapshotted, while preserving the settled/cancel/runId safeguards from the later PR commits and adding focused regression coverage for backgrounded stdout/stderr flushed at process exit.
