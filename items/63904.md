## Summary

Keep open. Current main supports Slack thread_ts routing and accepts thread_broadcast events, but it does not implement the requested reply_broadcast outbound flag or surface inbound broadcast visibility to agents. The related items cover Slack thread replies generally, not this specific “Also send to channel” mirroring behavior.

## Best Possible Solution

Keep the issue open. The best path is a scoped Slack plugin change: add a typed replyBroadcast option through sendMessageSlack/sendSlackMessage and Slack message-action params, pass reply_broadcast to chat.postMessage only with a thread_ts, expose inbound thread_broadcast/reply_broadcast state in agent-visible Slack context metadata, and add regression tests plus Slack docs updates.
