## Summary

Keep open. Current main still does not implement Slack `reply_broadcast`: outbound Slack send/action/generic message-tool paths only plumb `threadTs`, and inbound `thread_broadcast` events are accepted as normal messages without an agent-visible broadcast/mirroring flag. The related PR #64365 is an open implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this issue open and review PR #64365 or a successor Slack-plugin patch. The best fix belongs in the Slack plugin: add typed `replyBroadcast` plumbing through Slack send/action/message-tool/outbound paths, pass Slack `reply_broadcast` only for threaded replies, expose inbound `thread_broadcast` visibility to agent/tool context so mirroring is possible, and add focused regression tests plus Slack docs.
