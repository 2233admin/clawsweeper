## Summary

Keep open. Current main has safety primitives and an experimental planning tracker, but it does not implement the requested first-class Ask, Plan, and Modify workflow modes with enforced read-only, preview/validation, and execution states.

## Best Possible Solution

Keep open for maintainer product/design work in core runtime, config, and UI. A strong implementation would add an enforced read-only Ask mode at the tool-policy boundary, a Plan mode that captures intended changes and blocks mutating tools until explicit confirmation, and a Modify mode that enables existing write/exec-capable paths after validation. Tests should prove enforcement at runtime/tool-policy boundaries, not just prompt wording.
