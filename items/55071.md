## Summary

Keep this issue open. Current main still does not implement skill-driven plugin context visibility: skill metadata has no `activates.plugins` contract, parser/snapshot paths do not preserve activation dependencies, and request-time tools are still assembled from the runtime plugin registry plus static tool policy before the model reads a selected `SKILL.md`. The linked PR #72533 remains open/unmerged and, per the provided PR metadata, is README-only despite closing syntax, so it is an implementation candidate to review or close separately rather than a reason to close this issue.

## Best Possible Solution

Keep this issue open for maintainer design and implementation. The likely path is to add a typed `metadata.openclaw.activates.plugins` skill metadata contract, parse and preserve it through skill entries/snapshots, define a concrete pre-model activation trigger, feed activated plugin ids into request-time plugin tool planning, document the contract, and add tests proving deny/allow policy and optional-tool precedence remain intact. Maintainers should separately review or close PR #72533 because it is open, unmerged, and README-only.
