## Summary

Keep this issue open. Current main still supports the bundled Anthropic Claude CLI backend, but the official Docker runtime image and Docker release workflow do not pre-install `@anthropic-ai/claude-code`, and the user-facing Docker docs still do not provide a Claude-specific persistent install plus `~/.claude` credential persistence recipe. The existing generic Docker persistence and custom-image docs help adjacent use cases, but they do not satisfy the requested official-image or Docker-specific Claude CLI path.

## Best Possible Solution

Keep this open until maintainers choose the Docker product path. The conservative implementation is a Docker-specific Claude CLI section in `docs/install/docker.md`, cross-linked from `docs.openclaw.ai/providers/anthropic` and `docs.openclaw.ai/gateway/cli-backends`, showing a persistent npm prefix or `/home/node` volume, `~/.claude` persistence, an explicit `agents.defaults.cliBackends.claude-cli.command` plus `agentRuntime.id: "claude-cli"` example, and Anthropic API-key fallback guidance. If maintainers want the lowest-friction path, add a pinned `@anthropic-ai/claude-code` install to the official Docker image and cover it in Docker release/test/docs.

## Risks / Open Questions

- Pre-installing `@anthropic-ai/claude-code` in the official image would add third-party package pinning, image-size, update-policy, SBOM, and supply-chain maintenance concerns; that should be an explicit maintainer product decision.
- A docs-only fix needs to be precise about both binary persistence and Claude OAuth persistence. Persisting `/home/node/.openclaw` alone does not preserve `~/.claude/.credentials.json`, and a full `/home/node` volume changes mount layering with the existing `.openclaw` bind mounts.
- Because this was a read-only cleanup review, no Docker image build or live Claude subscription run was executed; the keep-open decision is based on current source, docs, workflow, and provided issue context.
