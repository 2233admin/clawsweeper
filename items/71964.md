## Summary

Keep PR #71964 open. Current main still builds gateway/node service PATHs with hard-coded Volta, asdf, Bun, fnm, nvm, and pnpm fallback directories regardless of whether those directories exist. The linked issue #71944 is closed and duplicate PR #72109 is closed unmerged, but neither provides a shipped main implementation for this PR's central behavior.

## Best Possible Solution

Keep this PR open or land a maintainer-equivalent fix. The right product path is to filter only hard-coded version-manager/package-manager fallback directories through an injectable existence predicate, keep explicit env-configured roots and stable user-bin conventions unconditional, add deterministic tests for missing and existing fallbacks, and fix the changelog attribution before landing.
