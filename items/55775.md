## Summary

Keep open. Current `main` does not implement `--agent-id` for `openclaw system event`, and the wake RPC path still only accepts and forwards `{ mode, text }`, so this PR is not obsolete and should not be closed as already implemented.

## Evidence

## Risks / Open Questions

- Closing this PR would discard still-missing functionality for multi-agent system-event targeting.
- The existing wake path still defaults to the main/default agent, so users cannot explicitly target a non-default agent from `openclaw system event` on current `main`.
- If this PR is revisited for merge, the review comments about unknown-agent fallback and the unrelated `SendParamsSchema` tightening should be resolved before landing.
