## Summary

Keep #70634 open. Current main has useful agent-to-agent loop bounds, but it does not implement the requested human-priority scheduling: command lanes and busy-session follow-up queues still append and drain FIFO, while provenance is recorded for inter-session messages but not used to order queue entries.

## Best Possible Solution

Implement this in core scheduling rather than closing it: mark external_user at channel/gateway ingress and inter_session at sessions_send/A2A boundaries, add priority tiers to command-lane and follow-up queue entries, preserve FIFO within each tier, define how cap/drop policies interact with priority, and add regression tests where a human message arrives behind an active agent-to-agent loop and is handled before further agent-originated backlog.
