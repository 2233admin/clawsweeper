## Summary

Current main still lacks origin-aware priority for agent lane queues. The command queue and follow-up queue both enqueue with push and drain with shift/FIFO semantics, and the embedded runner submits human and inter-session turns through the same session/global lane path without a priority option. sessions_send does mark inter-session provenance and has a bounded ping-pong loop setting, but that does not preempt queued agent-originated work for a later human message. The issue is recent, concrete, and corroborated by the KinthAI comment describing the same starvation pattern and a two-tier queue workaround.

## Best Possible Solution

Keep the issue open and implement a focused core scheduling change: carry trusted ingress provenance/priority into session-lane and follow-up queue entries, drain external human turns ahead of inter_session/background turns while preserving FIFO within each tier, and add regression tests for command-queue and follow-up queue behavior under a replenishing sessions_send/A2A loop. Consider pairing this with bounded A2A back-pressure, but do not treat maxPingPongTurns alone as the fix.
