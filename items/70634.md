## Summary

Keep open. Current `main` still runs agent work through strict FIFO session lanes with no origin-aware priority or human-preemption path, so the reported starvation is plausible and not implemented away on `main`. There is partial handling for a user message that arrives during an active turn, but that only merges the next prompt after the active turn and does not address a continuously replenished backlog.

## Evidence

- **Session lanes are strict FIFO:** `drainLane()` dequeues with `state.queue.shift()` and `enqueueCommandInLane()` appends with `state.queue.push(...)`, so lane execution is first-in-first-out with no priority insertion path.
