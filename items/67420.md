## Summary

Keep this PR open. Current main still does not implement per-agent dreaming control, and the PR is an active same-author implementation candidate paired with open feature issue #67413 via closing syntax. It should not be auto-closed before maintainers decide whether to fix and merge it or replace it with a successor patch.

## Best Possible Solution

Keep this PR open for maintainer review or a successor patch. The implementation should add agents.list[].dreaming to the exported config type, strict Zod schema, generated schema/help/docs surfaces, and memory dreaming workspace resolution; preserve default-agent fallback only when no agents are configured; skip explicitly disabled agents without falling back when all configured agents opt out; and add tests for mixed enabled/disabled agents, all-disabled agents, and config validation. Maintainers should also decide whether per-agent schedule support belongs in this PR or a separate follow-up for #67413.
