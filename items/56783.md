## Summary

Keep open. Current `main` still uses a count-only 200-message history cap in Control UI, does not implement the PR's char-budget logic, and does not include the PR's targeted regression coverage. The PR also appears stale against a refactor because history-windowing now lives in `ui/src/ui/chat/build-chat-items.ts` instead of `ui/src/ui/views/chat.ts`, but the underlying bugfix remains unshipped on `main`.
