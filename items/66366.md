## Summary

Current main still supports the reported Discord-visible raw exec failure follow-up. Backgrounded exec failures enqueue untrusted raw-tail system events, exec-event heartbeats build user-relay prompts from those details when a delivery target exists, and Discord outbound delivery sends the resulting final reply through its normal text path without an exec-failure sanitizer or card-only policy. The prior keep-open comment remains accurate; the local related items cover other raw-text leaks or fixed feedback loops, not this remaining exec notification path.

## Best Possible Solution

Keep this issue open. Fix it at the exec notification, heartbeat prompt, or Discord delivery-policy boundary: suppress card-duplicated exec failure relays, sanitize exec failure output into a short user-facing summary, or add a scoped Discord exec-notification policy. Add regression coverage proving Discord no longer receives raw `System (untrusted)` or command-tail follow-ups after exec-card failures while legitimate async completion notifications still work.
