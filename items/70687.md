## Summary

Keep open. Current main still expects broader device scopes to go through a pending upgrade approval flow, and the bootstrap pairing profile still does not include `operator.admin` or `operator.pairing`. The tree shows a supported recovery path (`openclaw devices list` / `openclaw devices approve <requestId>`), but it does not show a fix for the reporter's claim that upgrades are resetting approved scopes on macOS/Homebrew.

## Evidence

A client that now asks for `operator.admin` or `operator.pairing` would still need a scope-upgrade approval.
