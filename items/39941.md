## Summary

Keep open. Current main still recurses into the first object-valued anyOf/oneOf variant during config form coercion, so the PR’s targeted bug remains useful unmerged work rather than conservative cleanup.

## Best Possible Solution

Keep PR #39941 open. The best code path is to land or rework a small discriminator-aware fix in ui/src/ui/controllers/config/form-coerce.ts that prefers object variants with matching top-level const constraints, falls back to the first object variant only when no discriminator match exists, preserves primitive union behavior, and adds anyOf/oneOf regression tests in ui/src/ui/controllers/config/form-utils.node.test.ts, including the unconstrained-variant edge case from the review discussion.
