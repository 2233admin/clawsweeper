## Summary

Keep this PR open. Current main still lacks the proposed allowlist-backed reverse lookup for hex-suffixed, underscore-stripped, dot-prefixed, and dropped-leading-character malformed tool names, and the PR remains a focused agent-runtime implementation candidate. It needs maintainer review and regression tests before landing.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should review or rework the fallback in src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts, keep all recovery paths allowlist-backed and fail-closed on ambiguity, and add targeted tests for hex-suffixed names, functionsread5f-style names, underscore-stripped and dot-prefixed variants, dropped-leading-character collisions, blacklist1a2b3c, readfeed, and foo_bar/foobar collisions before landing.
