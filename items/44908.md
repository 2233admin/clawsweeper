## Summary

Keep #44908 open. Current main contains WhatsApp audio transcription support, but it still runs after `applyGroupGating`; a group voice note with `requireMention` and raw body `<media:audio>` can still be skipped before transcription is attempted, which is the reported bug.

## Evidence

- **Group gating runs before audio preflight:** `createWebOnMessageHandler` calls `applyGroupGating` for group messages and returns immediately when `gating.shouldProcess` is false; the audio preflight block starts only later.
  - file: [extensions/whatsapp/src/auto-reply/monitor/on-message.ts:147](https://github.com/openclaw/openclaw/blob/b1b29a8fc210a0772eb3728e03783890586e07f8/extensions/whatsapp/src/auto-reply/monitor/on-message.ts#L147)
  - command: `nl -ba extensions/whatsapp/src/auto-reply/monitor/on-message.ts | sed -n '1,280p'`
  - sha: [b1b29a8fc210](https://github.com/openclaw/openclaw/commit/b1b29a8fc210a0772eb3728e03783890586e07f8)
- **Mention gate still skips placeholder body:** `applyGroupGating` computes mention status from `params.msg.body` and skips with `Group message stored for context (no mention detected)` when mention is required and not detected.
  - file: [extensions/whatsapp/src/auto-reply/monitor/group-gating.ts:145](https://github.com/openclaw/openclaw/blob/b1b29a8fc210a0772eb3728e03783890586e07f8/extensions/whatsapp/src/auto-reply/monitor/group-gating.ts#L145)
