## Summary

Keep open. Current `main` at `0e7250f37bf347c72790abf2a6692d5ec84e5ea1` does not implement quoted-image support for Feishu replies: the handler only downloads media for the inbound message, and quoted parent messages are still reduced to text/placeholder content with no media keys to fetch.

## Evidence

- **Quoted replies are text-only on current main:** `dispatchMessage` resolves media for the inbound message via `resolveFeishuMediaList(...)`, then separately fetches the parent with `getMessageFeishu(...)` and stores only `quotedContent = quotedMessageInfo.content`. There is no quoted-media download or merge into `mediaPayload`.
  - file: [extensions/feishu/src/bot.ts:712](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/feishu/src/bot.ts#L712)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '708,760p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Fetched quoted file/image messages still collapse to placeholders:** `parseFeishuMessageContent` only has special handling for `text`, `post`, and `interactive`; all other message types fall through to ``[`msgType message`]`` placeholders. A quoted image/file therefore does not expose downloadable metadata through this path.
  - file: [extensions/feishu/src/send.ts:219](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/feishu/src/send.ts#L219)
  - command: `nl -ba extensions/feishu/src/send.ts | sed -n '182,265p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Message model lacks quoted-media keys on main:** `FeishuMessageInfo` currently contains `content`, `contentType`, `createTime`, and `threadId`, but no `imageKey` or `fileKey`. `parseFeishuMessageItem` returns that shape unchanged from `getMessageFeishu`, so the quoted-message fetch path has no media key to download.
  - file: [extensions/feishu/src/types.ts:65](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/feishu/src/types.ts#L65)

There is no current-main test proving quoted image media is attached to the agent context.
