## Summary

Keep #57987 open. Current main still has the OpenRouter-specific Anthropic cache wrapper marking system/developer messages only; it does not add a cache_control marker to the trailing user turn, and the current OpenRouter tests explicitly assert that user-only or system+user payloads leave the user content unchanged. The PR remains a focused, plausible fix, with prior review comments already incorporated in the branch design described in the GitHub context.

## Best Possible Solution

Keep the PR or an equivalent maintainer patch open and finish it against current main: update the OpenRouter Anthropic payload helper to mark only the last system/developer breakpoint plus the trailing user turn, reuse a safe cacheable-block helper, preserve assistant thinking cleanup, and update the OpenRouter cache-control tests and changelog/docs so the behavior matches the direct Anthropic path without exceeding provider breakpoint limits.
