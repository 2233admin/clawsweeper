## Summary

Keep PR #69201 open. Current main still builds the child requester origin from the tool ctx only; when `agentTo` / `agentThreadId` are missing, the child `agent` request only carries the channel, and the gateway seeding path cannot recover the parent session's stored `deliveryContext`. The PR discussion shows active review and author follow-up addressing concrete edge cases, and the related Slack/threading fixes do not supersede this core `sessions_spawn` parent-context propagation path.

## Best Possible Solution

Keep #69201 open or land an equivalent core fix: backfill missing delivery routing fields from the parent session's stored `deliveryContext` before resolving child requester origin in both native `spawnSubagentDirect` and ACP spawn, preserve ctx-over-parent precedence, guard stale parent `threadId` leakage for root-level/current-target spawns, and keep gateway `agent` seeding as the single write path with regression coverage for persisted child `deliveryContext`.
