## Summary

Keep PR #69201 open. Current main still does not implement the PR's central fix: `sessions_spawn` and ACP spawn resolve child delivery only from the current tool context, while the gateway can seed a child session only from the `agent` request fields it receives. If `ctx.agentTo` / `ctx.agentThreadId` are empty, current main still has no parent-session `deliveryContext` backfill to supply those fields. The landed Slack threading fixes are related downstream routing work, but they do not supersede this core spawn-path gap.

## Best Possible Solution

Keep #69201 open for maintainer handling. The best path is to merge or adapt the parent `deliveryContext` backfill into `src/agents/subagent-spawn.ts` and `src/agents/acp-spawn.ts`, preserve gateway `agent` seeding as the single write path, and keep the PR's regression coverage for persisted child delivery context plus stale-thread guard cases. Related Slack fixes (#69429/#69752) should remain separate downstream routing fixes, not substitutes for this core spawn-path fix.
