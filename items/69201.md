## Summary

Keep #69201 open. Current main still lacks the parent-session deliveryContext backfill proposed for native and ACP `sessions_spawn` children. The gateway can seed child delivery only from the child `agent` request, and current coverage explicitly shows requests without `to`/`threadId` persist channel-only context. The PR discussion is substantive: review findings were addressed and re-verified, so this remains a plausible focused core routing fix rather than cleanup-close material.

## Best Possible Solution

Keep #69201 open for maintainer review or land an equivalent core fix. The best direction is to backfill missing delivery fields from the parent session entry in both `spawnSubagentDirect` and `resolveAcpSpawnRequesterState`, keep current ctx values authoritative, guard channel/to/root-level stale-thread cases, and retain regression coverage proving child session entries get `deliveryContext.to` and `deliveryContext.threadId` after spawn.
