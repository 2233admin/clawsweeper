## Summary

Keep open. Current `main` has auth-profile cooldowns plus ordered model fallback, but it does not implement a configurable per-model circuit breaker that auto-disables a failing provider/model after repeated failures across runs. The provided issue context also shows an open related PR (#55736), so this is still active work rather than conservative cleanup.

## Evidence

- **Fallback still starts with the requested model:** The failover docs say the requested model is always first in the candidate chain, so current behavior still re-attempts that model on new runs unless auth-profile cooldown logic blocks it.
  - file: [docs/concepts/model-failover.md:237](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/concepts/model-failover.md#L237)

- **Skip logic is auth-profile based, not model-breaker based:** `runWithModelFallback` only skips a candidate when all auth profiles for that provider are already unavailable/in cooldown. There is no per-model failure ledger or breaker state checked before trying the candidate.
  - file: [src/agents/model-fallback.ts:754](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/model-fallback.ts#L754)
