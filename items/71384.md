## Summary

Keep open. Current `main` still uses whitespace-only Discord line splitting, lacks CJK/surrogate regression coverage, and a read-only reproduction shows the reported gap is still present. The PR discussion also shows the branch addressed the initial Ext-B review finding, so this is still actionable work rather than obsolete cleanup.

## Evidence

- **Current main still hard-splits non-whitespace Discord text:** `splitLongLine` in `extensions/discord/src/chunk.ts` only searches backward for `/\s/`; when none is found it sets `breakIdx = limit` and slices there. There is no CJK-aware fallback and no surrogate-pair guard in current `main`.

  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Current tests do not cover the reported CJK behavior:** `extensions/discord/src/chunk.test.ts` covers fences, whitespace, and italics, but there are no CJK punctuation or surrogate-pair boundary cases in current `main`.
  - file: [extensions/discord/src/chunk.test.ts:1](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/extensions/discord/src/chunk.test.ts#L1)
