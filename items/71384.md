## Summary

Current main still uses the old Discord chunking behavior: `splitLongLine` searches only for whitespace and falls back to a hard split at the character limit. The PR is an active, relevant fix for the linked CJK Discord bug, and the review discussion shows the initial Ext-B surrogate-pair concern was addressed in the PR head rather than making the PR obsolete.

## Best Possible Solution

Keep the PR open for maintainer review and CI. The right product path is to fix this in the Discord plugin chunker, preserving existing whitespace behavior while adding CJK punctuation/character fallback and code-point-safe surrogate handling with the proposed regression tests. If merged, it should close #38597 and leave unrelated CJK token-estimation or table-alignment issues to their own tracks.
