## Summary

Keep open. Current `main` still passes the `emoji` parameter straight through to Feishu as `reaction_type.emoji_type`, and the implementation/comments expect Feishu reaction type strings like `THUMBSUP` rather than Unicode emoji like `👍`. I did not find any normalization layer or tests proving Unicode emoji input now works, so the reported failure remains plausible on current code.
