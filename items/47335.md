## Summary

Keep open. Current `main` still contains the ordering that the issue reports: compaction retry resolution happens before async reply/channel flush completion, and final payload assembly still drops all non-error finals whenever any block streaming succeeded. I did not find evidence in the checked-out code that this was fixed on `main`.

## Evidence

- **Compaction retry still unblocks before async flush completes:** `handleAgentEnd()` calls `finalizeAgentEnd()` immediately after `ctx.flushBlockReplyBuffer()`, before awaiting either the block buffer flush or `onBlockReplyFlush()`. `finalizeAgentEnd()` resolves the compaction retry when `pendingCompactionRetry > 0`.
