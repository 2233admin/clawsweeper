## Summary

Keep open. Current main still returns many hook 4xx/408/413 responses from `createHooksRequestHandler` without a hook warning log, including invalid JSON/body errors, direct wake/agent validation failures, mapping validation failures, auth failures, method errors, and not-found cases. The related PR #69267 attempted a fix but was closed unmerged, so it does not supersede or implement the issue.

## Best Possible Solution

Keep the issue open and implement a shared rejection path in `src/gateway/server-http.ts` for hook ingress failures. It should emit one bounded `logHooks.warn` line for every non-2xx hook response with stable reason codes, path/status/content metadata, and active trace context, then add focused tests around invalid JSON, mapping `requires message`, auth/method/path, agent policy, channel, and session-key rejection branches. Also split or code the ambiguous `hook mapping requires message` branches in `src/gateway/hooks-mapping.ts` so production logs identify the actual validator.
