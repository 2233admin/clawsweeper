## Summary

Keep #69192 open. Current main still returns many hook gateway 4xx responses without a hook journal/log entry, including the reporter's invalid JSON path and mapped-hook validation failures. The linked PR #69267 directly targeted this issue but was closed unmerged, and its proposed behavior is not present in current main.

## Best Possible Solution

Keep the issue open and implement a small hook rejection logging helper in `createHooksRequestHandler` so every hook non-2xx response records one sanitized warn line with path, status, stable reason code, content-type/body metadata where available, and request id/correlation data. Add focused tests for invalid JSON, mapped-hook validation, auth/method/path failures, and distinct branch-specific reason codes for the `hook mapping requires message` cases.
