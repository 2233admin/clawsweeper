## Summary

Keep open. Current main still returns hook 4xx responses without a hook journal warning on the exact paths reported: invalid JSON, direct wake/agent validation failures, mapped-hook validation failures, and most auth/path/method rejects. The prior direct fix PR #69267 was closed unmerged, so it is useful prior art but not a shipped implementation.

## Best Possible Solution

Keep the issue open and implement a small hook-specific rejection helper in `src/gateway/server-http.ts` that logs one sanitized warn line before every non-2xx hook response, including stable reason codes, path, status, content type, body type/bytes when safely known, and the active trace/request id. Add focused Vitest coverage for invalid JSON, mapped `hook mapping requires message`, auth/path/method rejects, and agent/session/channel policy rejects. Consider splitting mapping validation failures into distinct internal reason codes while keeping user-facing messages compatible.
