## Summary

Keep open. Current main still returns most hook gateway 4xx responses without a `logHooks.warn` journal line, including the reported invalid JSON and mapped-hook validation paths. The related PR #69267 was closed unmerged in the provided context, and its patch is not present on current main.

## Best Possible Solution

Keep the issue open and implement a small rejection-logging helper inside `createHooksRequestHandler` so every non-2xx hook response logs one warning with path, status, stable reason code, content type, body type/size where available, and a request id. Add focused tests for invalid JSON, mapping validation, auth/policy, method/path, and body-size/timeout branches, and split `hook mapping requires message` into distinct internal reason codes or distinct user-facing errors where the branches differ.
