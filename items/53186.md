## Summary

Keep open. Current main still has the Mattermost DM threading bypass described in the report: docs and upstream reply-mode resolution say direct messages stay non-threaded, but the outbound root-id resolver is still chat-kind unaware and can forward a surviving payload replyToId as Mattermost root_id.

## Best Possible Solution

Keep this issue open and review/land PR #72305 or an equivalent Mattermost plugin patch. The fix should make the outbound Mattermost root-id resolution direct-chat aware, prevent root_id from being sent for direct-message auto-replies under the current docs contract, and add focused regression coverage for direct chats where payload.replyToId is still populated. Treat configurable DM threading as a separate product/API decision.
