## Summary

Do not close this PR. The reported Discord bug is still present on current main: the `allowBots="mentions"` bot-message drop path returns `null` without calling `recordPendingHistoryEntryIfEnabled()`, while adjacent mention-required and ignore-other-mentions drop paths do record history. The PR targets the right code path, but the current regression test in the PR needs the reviewer-noted adjustment because default guild behavior requires a mention first, so it can pass through the earlier mention-required drop path instead of the new allowBots-only branch.

## Best Possible Solution

Keep the PR open and ask for the test adjustment from the existing review: configure the regression test so guild mention requirement is false, keep `allowBots: "mentions"`, pass a positive `historyLimit` and shared `guildHistories`, then assert the recorded entry matches the dropped bot message. Once that guard actually reaches the allowBots-only branch, the production change should be a small, appropriate fix in `extensions/discord/src/monitor/message-handler.preflight.ts`.
