## Summary

Keep #68959 open. Current main has a grouped chat session selector and the separate Agents management tab, and related #70610 fixed narrower switcher label behavior, but the requested workflow is still missing: the chat UI does not expose a configured-agent picker, and its selector is populated from existing session rows rather than all agents in agents.list.

## Best Possible Solution

Implement a chat-visible configured-agent switcher populated from agents.list. It should show configured display or identity names, switch to an existing main session when present, and otherwise call sessions.create with agentId and key main before loading chat history. Add focused Control UI tests for multiple configured agents where a non-default agent has no existing session row.
