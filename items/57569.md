## Summary

Keep open. Current `main` still handles stale-socket health events with an unconditional `stopChannel` -> `resetRestartAttempts` -> `startChannel` flow, and the shipped config/docs expose only health-monitor enablement and timing knobs. There is no implemented reconnect/graceful restart mode, cache prewarm path, or `channelHealthRestartMode`/`channelStaleRestartMode` setting on `main`.
