## Summary

Current `main` has targeted mitigations for stale gateway service tokens and Control UI auth/reconnect loops, but the exact reported failure path remains plausible and I do not have strong enough proof to close this issue conservatively.

## Evidence

- **Stale service-token mismatch is now explicitly detected:** `auditGatewayToken()` flags embedded `OPENCLAW_GATEWAY_TOKEN` values and reports a mismatch when the service token differs from `gateway.auth.token`, matching the issue’s `gateway token mismatch` symptom.
  - file: [src/daemon/service-audit.ts:212](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/daemon/service-audit.ts#L212)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Doctor repair treats config token as source of truth:** Gateway service repair resolves the expected token from config/runtime and audits the installed service against it before reinstalling, which is relevant to stale LaunchAgent metadata.
  - file: [src/commands/doctor-gateway-services.ts:225](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/commands/doctor-gateway-services.ts#L225)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Browser client no longer loops indefinitely on non-recoverable auth failures:** Control UI reconnect logic treats auth rate-limit and other non-recoverable auth errors as terminal, and token mismatch gets only a bounded trusted retry path instead of open-ended reconnects.
  - file: [ui/src/ui/gateway.ts:70](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/ui/src/ui/gateway.ts#L70)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Gateway client also pauses mismatch reconnects after one-shot retry:** The shared gateway client stops reconnecting after token mismatch once the one-shot device-token retry budget is consumed, which addresses the reported lockout/retry storm pattern.
  - file: [src/gateway/client.ts:612](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/gateway/client.ts#L612)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Regression coverage exists for the new auth handling:** Tests cover stale service-token detection/repair and bounded Control UI retry behavior after `AUTH_TOKEN_MISMATCH`, but they do not prove the reporter’s exact macOS LaunchAgent scenario is fully gone.
  - file: [src/daemon/service-audit.test.ts:115](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/daemon/service-audit.test.ts#L115)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Docs now describe this family of failures and recovery steps:** Troubleshooting documents `AUTH_TOKEN_MISMATCH`, `retry later`, and service-config disagreement, including reinstalling service metadata when runtime and config disagree.
  - file: [docs/gateway/troubleshooting.md:149](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/docs/gateway/troubleshooting.md#L149)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
