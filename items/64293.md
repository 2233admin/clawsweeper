## Summary

Keep open. Current main still appears to schedule heartbeats for an empty `agents.defaults.heartbeat: {}` path, and the provided GitHub context shows open PR #64540 explicitly uses closing syntax for this issue and remains unmerged. Per cleanup rules, this issue should stay open while that implementation candidate is reviewed, landed, or rejected.

## Best Possible Solution

Keep this issue open and review PR #64540 or an equivalent maintainer patch. The fix should explicitly define whether `{}` disables heartbeat, apply consistently to default and per-agent heartbeat resolution, preserve the documented `every: "0m"` disable behavior, and add focused regression coverage for empty defaults and empty per-agent overrides.
