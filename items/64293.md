## Summary

Keep open. Current main still treats `agents.defaults.heartbeat: {}` as an enabled heartbeat config that falls through to the default cadence, so the reported 30-minute background runs are not fixed on `main`. The related fix attempts are useful context, but #64329 is closed/unmerged and #64540 is still open rather than shipped.

## Best Possible Solution

Keep #64293 open. The best maintainer path is to either land a scoped fix that defines empty defaults heartbeat as disabled across scheduler, run-once, summary, and system-prompt/bootstrap gating with regression tests, or explicitly document that only `every: "0m"` disables heartbeats and reject/doctor-warn on `agents.defaults.heartbeat: {}` to avoid silent token burn.
