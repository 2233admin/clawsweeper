## Summary

Current `main` does not implement the PR’s requested per-connection control-plane write isolation. The new gateway rate-limit module keys by `deviceId|clientIp` whenever either identity field is present, and the current tests explicitly assert that `connId` is ignored in that case. Main has separately landed stale-bucket pruning and a hard cap to address the bot’s memory-growth concern, so the PR still targets a real unresolved behavior but now needs a rebase/update rather than closure.
