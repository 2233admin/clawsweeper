## Summary

Keep open. Current `main` does not implement per-connection control-plane write rate-limit isolation. The active code still uses `deviceId|clientIp` as the key whenever either identity field is present, and the current test suite explicitly asserts that behavior. Main only contains a separate follow-up for stale bucket pruning and map bounding, which addresses the memory-growth concern but not this PR's requested behavior change.
