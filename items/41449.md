## Summary

Current `main` appears to already allow Ollama memory embeddings against the configured localhost/LAN host, so the PR's stated bug is likely obsolete. But the PR also adds redirect-pinning hardening that is not on `main`, and the mixed evidence is not strong enough for a conservative auto-close.

## Evidence

  - file: [src/infra/net/ssrf.ts:122](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/infra/net/ssrf.ts#L122)
