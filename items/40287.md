## Summary

Keep PR #40287 open. Current main still has no Slack `file_shared` event handler or Slack setup/docs subscription for `file_shared`, so the large-file fallback remains missing. The PR is stale because it targets the old `src/slack` layout, but the central behavior is still useful and should be ported into `extensions/slack`.

## Best Possible Solution

Keep the PR open and ask for a rebase/port into `extensions/slack`. The best implementation is a Slack plugin `file_shared` fallback that subscribes to the event, fetches/hydrates the matching Slack message with a robust bounded lookup, feeds it through the existing message pipeline, uses race-safe deduplication with the existing seen-message semantics where possible, updates setup/docs for `file_shared`, and adds focused tests for normal message/file_shared dedup and large-file fallback behavior.
