## Summary

Current main still does not implement silent reasoning persistence. OpenClaw separates thinking production from reasoning visibility, and it has provider replay sanitizers for some historical thinking blocks, but there is no schema-backed mode that lets reasoning run while discarding thinking/redacted_thinking blocks from the durable session transcript and transcript APIs.

## Best Possible Solution

Keep this issue open. The best product direction is a core transcript-persistence policy separate from thinking effort and reasoning visibility: keep thinkingLevel for model reasoning production, keep reasoningLevel for user/channel visibility, add a documented persist/discard setting for thinking blocks, and define where provider-required replay signatures or opaque reasoning state live so user-facing transcript/history/memory APIs can omit them safely. The implementation should cover transcript writes/reads, sessions.get, ACP loadSession, compaction and replay sanitization, memory/session-history indexing, UI/config docs, and plugin hook expectations.

## Risks / Open Questions

- A naive strip-before-write fix could break providers that require signed or opaque thinking blocks for replay, so the implementation needs an explicit policy for durable user/history/memory transcript APIs versus provider replay state.
- The existing before_message_write hook can support narrow plugin workarounds, but it is not a documented core setting and does not settle sessions.get, ACP loadSession, compaction/replay, UI/config, or memory indexing semantics.
