## Summary

Keep open. Current main has partial producer-side support and overlap heuristics, but it does not consume the assistant `replace` flag in the Gateway path that this PR is now centered on, and the PR has active, relevant review feedback rather than being obsolete.

## Best Possible Solution

Keep the PR open and ask for a focused update against current main: thread `replace: true` through the Gateway assistant-event path into `resolveMergedAssistantText`, preserve prior text on empty replacements, bypass throttling only for intentional non-empty replacements or final flushes, and add Gateway regression coverage for shorter replacement text inside the throttle window.
