## Summary

Keep open. Current `main` still drives Control UI chat startup through the same `chat.history` and `models.list` paths, and I did not find a shipped or current-main fix that clearly removes the shared slow path behind the report.

## Evidence

- **Chat tab still loads history and models on connect:** `refreshActiveTab()` still sends the chat tab through `refreshChat()`, which concurrently loads chat history, sessions, avatar, models, and slash commands during startup.
  - file: [ui/src/ui/app-settings.ts:335](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/ui/src/ui/app-settings.ts#L335)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Startup refresh still includes `chat.history` and `models.list`:** `refreshChat()` still calls `loadChatHistory(...)` and `refreshChatModels(...)` in the same `Promise.all(...)`, so the reported slow RPCs remain part of initial chat startup on current `main`.
  - file: [ui/src/ui/app-chat.ts:510](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/ui/src/ui/app-chat.ts#L510)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **`models.list` still uses heavyweight model-catalog loading:** The gateway handler still answers `models.list` by calling `context.loadGatewayModelCatalog()`. That loader still imports the Pi model discovery runtime, discovers auth storage, instantiates the registry, reads all entries, and merges provider-plugin catalog entries before returning.
  - file: [src/gateway/server-methods/models.ts:13](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/gateway/server-methods/models.ts#L13)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **`chat.history` can still block on the same catalog path:** `chat.history` still calls `context.loadGatewayModelCatalog()` when the session lacks a stored `thinkingLevel`, so the two slow RPCs in the issue can still share the same bottleneck on current `main`.
  - file: [src/gateway/server-methods/chat.ts:1936](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/gateway/server-methods/chat.ts#L1936)

- **No clear fix between latest release and current main:** Comparing `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`) to current `main` showed changes in catalog internals and chat code, but no targeted change in `ui/src/ui/controllers/models.ts` or `src/gateway/server-methods/models.ts` that would clearly close this performance report.
