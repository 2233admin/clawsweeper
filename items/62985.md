## Summary

Current main still matches the reported regression: Telegram account entries are validated by a strict account schema that does not include `agentId`, so `channels.telegram.accounts.<id>.agentId` remains an invalid additional property. The documented current routing path is top-level `bindings[]`, and the issue has an open unmerged implementation candidate, #63380, with closing syntax for this issue. Per the review rules, this should stay open while that PR is still the candidate fix.

## Best Possible Solution

Keep this issue open and review #63380 as the implementation candidate. If backward compatibility is desired, land a focused fix that accepts and correctly routes or migrates `channels.telegram.accounts.<id>.agentId`, with regression coverage for the reported multi-account shape. If maintainers want top-level `bindings[]` to remain the only supported routing path, close the PR with that rationale and improve Telegram upgrade docs or doctor diagnostics so upgraded users can find the migration path.
