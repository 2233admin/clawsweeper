## Summary

Keep open. Current `main` still treats invalid plugin entry config as a fatal config-validation failure during config load/startup, so the reported tenant-wide boot failure is still consistent with shipped behavior. The linked follow-up PR in the issue timeline is still open, which also argues against cleanup closure.

## Evidence

- **Plugin config validation still produces hard config issues:** `validateConfigObjectWithPluginsBase()` validates each enabled plugin or plugin entry with config, and on schema failure pushes `invalid config: ...` issues at `plugins.entries.<id>.config...` instead of degrading the plugin entry in-memory.
