## Summary

Keep open. Current main already supports thread-bound ACP sessions for interactive output, but it does not implement the requested `streamTo:"thread"` relay path. `sessions_spawn` still only accepts `streamTo:"parent"`, and that path relays progress by enqueueing system events on the requester session instead of sending visible thread messages directly.

## Evidence

- **`sessions_spawn` only allows `streamTo:"parent"`:** The tool schema hardcodes `SESSIONS_SPAWN_ACP_STREAM_TARGETS = ["parent"]`, and runtime parsing only preserves `params.streamTo === "parent"`.
  - file: [src/agents/tools/sessions-spawn-tool.ts:29](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/tools/sessions-spawn-tool.ts#L29)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '1,340p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Parent stream relay is session-event based, not direct thread messaging:** The ACP parent relay emits updates via `enqueueSystemEvent(...)` to the parent session and wakes heartbeat processing; it does not call any outbound channel/thread send path.
  - file: [src/agents/acp-spawn-parent-stream.ts:199](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/acp-spawn-parent-stream.ts#L199)
  - command: `nl -ba src/agents/acp-spawn-parent-stream.ts | sed -n '1,360p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **`streamTo:"parent"` disables inline channel delivery:** Bootstrap delivery only goes inline when `spawnMode === "session"` and `!effectiveStreamToParent`. The dedicated test for `streamTo:"parent"` also asserts `agentCall.params.deliver === false`.
  - file: [src/agents/acp-spawn.ts:1012](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/acp-spawn.ts#L1012)
  - command: `nl -ba src/agents/acp-spawn.ts | sed -n '960,1355p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Current main already has thread-bound ACP sessions, but as a different path:** Docs say thread-bound ACP sessions deliver ACP output back to the same thread, and the spawn test asserts `deliver === true` with a bound thread target. The docs also explicitly say stream-relay paths are a separate richer integration pass.
  - file: [docs/tools/acp-agents.md:134](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/docs/tools/acp-agents.md#L134)
