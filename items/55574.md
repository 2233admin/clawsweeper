## Summary

Keep open. Current `main` does not implement `streamTo:"thread"` for ACP spawns: the accepted enum is still only `"parent"`, the `sessions_spawn` tool only forwards that one value, and the ACP docs explicitly describe parent-session system-event relay as the only supported `streamTo` mode. The request is coherent, in scope for core ACP/channel delivery, and the issue already has a cross-linked open PR (#59366).

## Evidence

- **ACP spawn enum only allows parent relay:** `ACP_SPAWN_STREAM_TARGETS` is still defined as `["parent"]`, and `SpawnAcpParams.streamTo` is typed from that single-value enum on current `main`.
  - file: [src/agents/acp-spawn.ts:92](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/acp-spawn.ts#L92)

- **sessions_spawn tool only accepts and forwards parent:** The tool schema uses the ACP stream-target enum, and runtime parsing reduces `params.streamTo` to `"parent"` or `undefined`, so `"thread"` is not currently accepted or passed into ACP spawn.
  - file: [src/agents/tools/sessions-spawn-tool.ts:125](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/tools/sessions-spawn-tool.ts#L125)
