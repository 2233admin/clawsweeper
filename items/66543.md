## Summary

Keep open. The underlying cron whitespace-corruption report is still plausible on current `main`, but this PR is not a safe cleanup candidate because its fix strips all internal whitespace from `model` and `fallbacks`, which conflicts with the current alias contract and would break valid spaced aliases during create, patch, and persisted-job hydration.

## Evidence

- **Current main still preserves internal whitespace:** On current `main`, `coercePayload` and `copyTopLevelAgentTurnFields` only trim outer whitespace for `model` and `fallbacks`; they do not strip internal spaces, so there is no shipped implementation of the requested fix in the checked-out code.
