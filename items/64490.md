## Summary

Keep open. Current main still emits zsh option descriptions inside double-quoted `_arguments` specs without escaping `$` or backticks, while current CLI option descriptions include both. The PR remains a focused implementation candidate and is not superseded by the related completion reports.

## Best Possible Solution

Keep this PR open as the implementation candidate. A maintainer should review the escaping helper in `src/cli/completion-cli.ts`, keep regression coverage for `$` and backtick descriptions in `src/cli/completion-cli.test.ts`, confirm single quotes remain literal in the double-quoted zsh context, then run the targeted completion test plus a live zsh `openclaw models <TAB>` check before merging or requesting changes.
