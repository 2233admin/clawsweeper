## Summary

Keep #71889 open. Current main still seeds the interactive `agents add` workspace prompt from `resolveAgentWorkspaceDir()`, and that resolver still appends a non-default agent id under `agents.defaults.workspace`. Setup/onboard still store the main default workspace in `agents.defaults.workspace`, while current docs say added agents should use peer-level `workspace-<agentId>` paths. Related PRs #71899 and #71941 are useful candidate fixes, but they are open/unmerged in the provided context, so the canonical bug issue should stay open until a fix lands.

## Best Possible Solution

Keep the issue open and land one reviewed scoped fix, likely following the #71941 shape: introduce a wizard/default suggestion path that returns peer-level `~/.openclaw/workspace-<id>` when `agents.defaults.workspace` is the normal main workspace, while leaving runtime `resolveAgentWorkspaceDir()` behavior and custom shared workspace bases intact. Add focused tests and, optionally, docs/warning text for existing nested `workspace/<id>` layouts.
