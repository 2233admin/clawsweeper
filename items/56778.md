## Summary

Keep open. Current `main` still lacks a native `cron list` agent filter and does not auto-scope cron listings to the requesting agent, so the issue's requested isolation is not implemented.

## Evidence

- **CLI list has no agent filter:** `openclaw cron list` only defines `--all` and `--json`, then calls `cron.list` with `{ includeDisabled }`. There is no `--agent` flag or equivalent exact filter.
  - file: [src/cli/cron-cli/register.cron-add.ts:40](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/cli/cron-cli/register.cron-add.ts#L40)
  - command: `nl -ba src/cli/cron-cli/register.cron-add.ts | sed -n '20,60p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Protocol schema omits agentId for cron.list:** `CronListParamsSchema` allows `includeDisabled`, pagination, `query`, `enabled`, and sorting only. It does not accept `agentId`, so the RPC surface has no exact agent filter.
  - file: [src/gateway/protocol/schema/cron.ts:272](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/gateway/protocol/schema/cron.ts#L272)
  - command: `nl -ba src/gateway/protocol/schema/cron.ts | sed -n '250,290p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Gateway handler does not scope by requester:** The `cron.list` handler validates the params above and forwards only `includeDisabled`, `limit`, `offset`, `query`, `enabled`, `sortBy`, and `sortDir` into `context.cron.listPage(...)`. No requester session or agent scoping is applied here.
  - file: [src/gateway/server-methods/cron.ts:137](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/gateway/server-methods/cron.ts#L137)
  - command: `nl -ba src/gateway/server-methods/cron.ts | sed -n '137,173p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Service list is global, not isolated:** `CronListPageOptions` has no agent field, and `listPage()` filters the whole store by enabled/query only. `job.agentId` is merely appended into the free-text haystack for `query`, not used as an isolation boundary.
  - file: [src/cron/service/ops.ts:208](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/cron/service/ops.ts#L208)
  - command: `nl -ba src/cron/service/list-page-types.ts | sed -n '1,80p' && nl -ba src/cron/service/ops.ts | sed -n '208,240p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Agent cron tool also does not auto-scope list:** The agent-facing `cron` tool's `list` action always calls `cron.list` with only `{ includeDisabled }`. Unlike `add`, it does not derive or pass the caller's `agentSessionKey` as an agent filter.
  - file: [src/agents/tools/cron-tool.ts:505](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/tools/cron-tool.ts#L505)
  - command: `nl -ba src/agents/tools/cron-tool.ts | sed -n '500,520p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)

## Risks / Open Questions

- This review is based on current code/docs inspection rather than a live seeded multi-agent repro, but the missing filter/scoping is explicit in the shipped interfaces and service path.
- A fuzzy `query` search can match `agentId`, but that is not an exact filter and does not provide default-agent-only visibility for unset `agentId` jobs.
