## Summary

Keep open. Current `main` still routes custom `anthropic-messages` providers through a generic replay fallback that only drops thinking blocks for Claude model ids, not Kimi ids like `moonshotai/kimi-k2.5`. The linked fix PR (#47459) was closed unmerged, and I did not find an equivalent `qiniu`/`moonshotai`-specific replacement on `main`.

## Evidence

- **Generic fallback only drops thinking for Claude ids:** `src/agents/transcript-policy.ts:74-116` builds the unowned-provider replay policy. Its only `dropThinkingBlocks` branch is `isAnthropic && modelId.includes("claude")`, so a custom `anthropic-messages` Kimi route would not get this protection.
  - file: [src/agents/transcript-policy.ts:74](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/transcript-policy.ts#L74)

  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Custom providers fall back to that generic path:** `src/agents/transcript-policy.ts:197-209` first checks for a provider runtime plugin hook, then falls back to `buildUnownedProviderTransportReplayFallback(...)`. That is the path a custom provider like `qiniu` would use.
  - file: [src/agents/transcript-policy.ts:197](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/transcript-policy.ts#L197)
