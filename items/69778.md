## Summary

Keep open. Current main still has the core behavior reported in #69778: restart-aborted subagent sessions with `abortedLastRun: true` are treated as recoverable without an age bound, and current docs/tests explicitly preserve recovery for a restored run that is already 3 hours old. The related #66909 documentation clarification does not supersede this concrete bug because the remaining issue is autonomous stale-task re-execution, not just unclear task-state documentation.

## Best Possible Solution

Add an explicit age-bound policy for restart recovery of detached/subagent tasks, probably under `agents.defaults.taskRecovery.maxAgeMinutes` or a closely scoped subagent task-recovery config. Apply it before `recoverOrphanedSubagentSessions` dispatches a synthetic resume and before task-registry maintenance preserves backed active tasks, terminalizing stale interrupted work as `lost` with clear audit/startup reporting. Update config schema/help/docs and replace the current 3-hour stale-aborted recovery expectation with bounded recovery/lost regression coverage.
