## Summary

Current `main` still auto-recovers interrupted subagent runs without any age bound, and the repo has an explicit regression test that preserves and resumes a 3-hour-old restored run after restart. This matches the reported behavior closely enough that the issue should stay open.

## Evidence

- **Recovery path has no staleness cutoff:** `recoverOrphanedSubagentSessions()` resumes any active run whose child session entry exists and has `abortedLastRun=true`. The gating checks cover missing session rows, non-aborted sessions, and ended runs that were not restart-aborted timeouts, but there is no comparison against `createdAt`, `startedAt`, `updatedAt`, or any max-age config before `resumeOrphanedSession()` is called.
  - file: [src/agents/subagent-orphan-recovery.ts:264](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/subagent-orphan-recovery.ts#L264)
  - command: `nl -ba src/agents/subagent-orphan-recovery.ts | sed -n '264,348p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Recovered runs re-send the old prompt automatically:** The recovery message is a synthetic prompt beginning with `[System] Your previous turn was interrupted by a gateway reload.` and ending with `Please continue where you left off.`, which is consistent with the issue's report that old prompts are auto-executed after restart.
  - file: [src/agents/subagent-orphan-recovery.ts:58](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/subagent-orphan-recovery.ts#L58)

- **Current tests intentionally preserve stale restored runs for restart recovery:** `subagent-registry.persistence.test.ts` creates a run with `createdAt` and `startedAt` set to 3 hours earlier, marks the child session `abortedLastRun: true`, restarts the registry, and then expects `callGateway` to resume that old run. That is direct proof that current `main` still treats multi-hour-old interrupted runs as restart-recoverable.
  - file: [src/agents/subagent-registry.persistence.test.ts:590](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/subagent-registry.persistence.test.ts#L590)

- **Restart paths actively schedule orphan recovery:** When a recoverable `agent.wait` interruption happens, the run manager schedules orphan recovery; separately, the registry loop schedules the same recovery whenever it sees `sessionEntry.abortedLastRun === true`. That means restart-aborted runs are intentionally re-queued for resume rather than aged out.
  - file: [src/agents/subagent-registry-run-manager.ts:119](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/subagent-registry-run-manager.ts#L119)

That documented behavior leaves old-but-still-backed interrupted runs eligible for recovery, which aligns with the report rather than disproving it.

That discussion is consistent with the current code paths above.
