## Summary

Keep open. Current `main` still permits root-scoped state resolution and root-scoped systemd user-service installation, so the reported failure mode remains plausible for a packaged image that runs the real service as a different user. The report is concrete, recent, and actionable, and there is no shipped fix on `main` that supports `implemented_on_main` or `cannot_reproduce`. Current docs do warn against third-party DigitalOcean Marketplace 1-click images, but OpenClaw's supported clean-Droplet docs still use `root`, so a blanket CLI root guard would be a product change with compatibility implications rather than evidence that the issue is obsolete.
