## Summary

Keep open. Current main still lacks a CLI-wide uid-0/root guard or a complete DigitalOcean 1-Click direct-root workaround, and the provided GitHub context shows open, unmerged PR #67509 as the direct implementation candidate whose body says it would close this issue.

## Best Possible Solution

Keep this issue open and review/land PR #67509 or an equivalent maintainer-approved fix. The likely repo-side solution is an early CLI guard before state/config resolution, with an OPENCLAW_ALLOW_ROOT-style escape hatch, root-level help/version exceptions, regression tests, and optional DigitalOcean 1-Click recovery or wrapper documentation if maintainers want image-specific mitigation.

## Risks / Open Questions

- A CLI-wide root guard needs maintainer judgment for legitimate root, container, CI, and package-verification workflows; an explicit override and help/version behavior should be tested.
- The DigitalOcean Marketplace image may also need an image-level wrapper, MOTD, or recovery documentation outside this repository, but current main still lacks the core/docs mitigation requested here.
- Do not treat the recent sudo-to-root systemd hardening as full resolution; it helps sudo-scoped installs but does not prevent direct root SSH invocation from entering OpenClaw state/config paths.
