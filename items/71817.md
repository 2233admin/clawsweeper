## Summary

Keep open. Current main documents and core-resolves per-agent `agents.list[].reasoningDefault`, but the Telegram and Feishu plugin preview/streaming gates still independently read only session-store `reasoningLevel` and fall back to `off`/`false`. PR #71817 is a focused fix for a real remaining channel-specific behavior gap, with related discussion clarifying that Feishu only previews the `stream` variant. It is not already implemented on main and is not fully superseded by the broader global/per-model defaults issues.

## Best Possible Solution

Keep PR #71817 open for maintainer review. The best path is to land or revise the focused Telegram/Feishu fallback fix with regression tests for config fallback and explicit session override precedence, while leaving broader global `agents.defaults.reasoningDefault` and per-model reasoning defaults to the existing related issues such as #13607, #54919, and #64236.
