## Summary

Keep open. Current main at 998e37fcb3d5b937fcf70d7509b677efba939f10 still ignores per-agent agents.list[].reasoningDefault in the Telegram and Feishu reasoning-preview paths, while the config contract and docs already expose that setting. The PR is a focused, recent bug fix with tests and active review discussion, not an obsolete cleanup candidate.

## Best Possible Solution

Keep the PR open for maintainer and channel-owner review. The best path is to land or revise a focused fix that makes Telegram and Feishu reasoning preview resolution fall back to agents.list[].reasoningDefault while preserving explicit session overrides and Feishu's stream-only preview behavior. Broader global or per-model reasoning defaults should remain separate follow-up work.
