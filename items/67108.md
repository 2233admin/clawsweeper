## Summary

Keep this PR open. Current main still returns failure when temp-dir repair sees an already-secure directory or when chmod fails, the related race report remains open with a concrete production trace, and the PR is a narrow implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or refine a narrow fix in `src/infra/tmp-openclaw-dir.ts` with regression coverage in `src/infra/tmp-openclaw-dir.test.ts`, confirming whether both the chmod-throw race and the already-secure re-stat race from #66867 are covered.
