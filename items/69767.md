## Summary

Keep open. Current main contains a narrower fix for parent announce turns that complete with no visible payload, but it still allows the subagent announce flow to proceed with `(no output)` when no usable child result/fallback payload is available. That is the specific fail-closed behavior requested by #69767, and current tests still assert the open behavior.

## Best Possible Solution

Keep #69767 open until `runSubagentAnnounceFlow` fails closed for completion messages with no usable child result: defer/retry without marking cleanup delivered, preserve the child session/state for another attempt, and add a regression test that currently would fail by asserting no requester announce is sent with `(no output)`. The relevant durable-payload pieces from #68464 may be the right implementation path if maintainers want the broader hardening.
