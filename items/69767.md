## Summary

Keep open. Current `main` still allows subagent completion delivery to proceed after a bounded retry window even when no usable result text exists, and it explicitly falls back to announcing `"(no output)"` instead of deferring. Mainline tests lock that behavior in for `expectsCompletionMessage` flows, so there is not strong evidence that issue #69767 is already fixed on `main` or in `v2026.4.23`.
