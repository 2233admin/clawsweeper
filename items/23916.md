## Summary

Keep open. Current `main` still does not surface the requested extra device-identifying details in the human-facing `openclaw devices list` output or Control UI, and there is no `--full`/notes-style feature for this request.

## Evidence

- **CLI list output is still limited to generic columns:** `openclaw devices list` renders pending rows with `Request/Device/Requested/Approved/Age/Status` and paired rows with only `Device/Roles/Scopes/Tokens/IP`. There is no rendering for `clientMode`, `clientId`, `platform`, `deviceFamily`, custom notes, or a special explanation for the default CLI device.
  - file: [src/cli/devices-cli.ts:352](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/cli/devices-cli.ts#L352)
  - command: `nl -ba src/cli/devices-cli.ts | sed -n '340,430p'`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Control UI also omits the requested metadata:** The paired-device and pending-device UI shows display name, device id, IP, roles, scopes, and token rows, but not client mode/platform/browser/OS labeling or editable notes.
  - file: [ui/src/ui/views/nodes.ts:129](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/ui/src/ui/views/nodes.ts#L129)
  - command: `nl -ba ui/src/ui/views/nodes.ts | sed -n '118,220p'`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Pairing state already stores the missing metadata:** The pairing model includes `platform`, `deviceFamily`, `clientId`, and `clientMode` on both pending and paired device records, so the data exists in core state but is not surfaced by the current list/UI output.
  - file: [src/infra/device-pairing.ts:23](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/infra/device-pairing.ts#L23)
  - command: `nl -ba src/infra/device-pairing.ts | sed -n '20,95p'`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Docs do not show an implemented `--full` or notes workflow:** The CLI docs for `openclaw devices list` only describe listing pending and paired devices and show `openclaw devices list` / `openclaw devices list --json`; there is no documented `--full`, rename/note support, or explanation of default CLI-device labeling.
  - file: [docs/cli/devices.md:15](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/docs/cli/devices.md#L15)
  - command: `nl -ba docs/cli/devices.md | sed -n '1,70p'`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)

## Risks / Open Questions

- `--json` may already pass through some extra metadata fields at runtime, but the issue explicitly asks for clearer human-facing `devices list` output and labeling, which current main still lacks.
