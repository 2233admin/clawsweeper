## Summary

Keep open. I did not find strong evidence that current `main` already fixes this report. The current host-browser `/act` path still uses a fixed 20s client timeout, the dispatcher path still turns local timeouts into the same generic `timed out ... Restart the OpenClaw gateway` surface, and those timeout files are unchanged between `v2026.4.22` and current `main`.
