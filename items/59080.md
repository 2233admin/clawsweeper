## Summary

Keep open. Current main has a new generic runner-failure privacy fix, but it does not specifically handle the reported JSON.parse SyntaxError shape. The formatter and sanitizer can still pass a short raw parse error through the assistant-error payload path, and the related recurring parse-error issue/PR remain open rather than shipped.

## Best Possible Solution

Keep #59080 open and fix it in the agent error boundary: add a narrow JSON.parse/SyntaxError classifier for malformed/truncated tool-call or stream fragments, route it through formatAssistantErrorText and sanitizeUserFacingText(errorContext), and decide whether it should classify as retryable/failover rather than only cosmetic suppression. Add regression coverage for the exact `Expected ':' after property name in JSON at position...` message through formatAssistantErrorText, sanitizeUserFacingText, buildEmbeddedRunPayloads, and the chat delivery path. Coordinate with the broader open #60593 and PR #61349 rather than closing this Feishu/compaction report prematurely.
