## Summary

Keep open. I did not find a main-branch or released fix for #67803. Current Codex harness code still keeps a run active until a scoped `turn/completed` notification arrives, so if the final response is visible before that terminal notification, follow-up messages would still queue exactly as reported.

## Evidence

- **Completion still gated on `turn/completed`:** `completed` flips and `resolveCompletion()` runs only for a matching `turn/completed` notification.
  - file: [extensions/codex/src/app-server/run-attempt.ts:397](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/extensions/codex/src/app-server/run-attempt.ts#L397)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Active-run state persists until completion resolves:** The active embedded run reports `isStreaming: () => !completed`, then awaits `completion`, and only clears the active run in the final cleanup path. That means late terminal notifications still hold the session open.
  - file: [extensions/codex/src/app-server/run-attempt.ts:611](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/extensions/codex/src/app-server/run-attempt.ts#L611)
