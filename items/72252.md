## Summary

Keep #72252 open. Current main still treats every structured exec completion, including `code 0` and `signal SIGTERM`, as an exec-completion heartbeat that can be user-relayed, and the heartbeat runner still bypasses normal `HEARTBEAT_OK` ack skipping for that class. The directly linked fix PR #72253 is open and appears to target exactly this issue, but it is not merged into current main.

## Best Possible Solution

Keep #72252 open until #72253 or an equivalent change lands on main. The product fix should classify structured successful exec completions and routine `signal SIGTERM` cleanup completions as internal-only for heartbeat delivery, while preserving visible relay for actionable exec failures and adding focused tests in `src/infra/heartbeat-events-filter.test.ts` and `src/infra/heartbeat-runner.returns-default-unset.test.ts`.
