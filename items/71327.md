## Summary

Keep open. Current `main` still has the LINE schema gap this PR describes: `dmPolicy: "open"` is accepted without requiring `allowFrom: ["*"]`, while runtime handling treats `open` as allowing any DM sender. The PR is fresh, non-maintainer-authored, and the provided review context shows it already addressed the main bot comments with added tests and a clearer account-level error message, so there is no conservative cleanup basis to close it.

## Evidence

- **Current LINE schema has no explicit-open wildcard guard:** `LineCommonConfigSchema`, `LineAccountConfigSchema`, and `LineConfigSchema` define `allowFrom` and `dmPolicy`, but there is no `superRefine` or `requireOpenAllowFrom` call on current `main`. That means configs with `dmPolicy: "open"` and missing/specific `allowFrom` still parse.
