## Summary

This PR is a substantial new optional memory plugin, and current OpenClaw policy routes that kind of work to community plugin distribution rather than bundling it into core.

## Close Comment

Thanks for the work here. After a Codex review against current `main` (`c1383680405879a3c3f43dceb20a694f54d3c13c`), this looks like a better fit as a community plugin rather than a new bundled OpenClaw core plugin.

The current project direction is explicit that optional capability should usually ship as plugins, that the preferred path is npm/local extension loading, that plugin authors should host and maintain plugins in their own repository, and that the bar for adding optional plugins to core is intentionally high (`VISION.md:54-61`). The plugin docs also point community plugins to ClawHub/npm as the normal distribution path (`docs/plugins/community.md:9-22`), and current core memory plugins remain narrowly scoped (`docs/tools/plugin.md:98-112`).

Because this PR is a large additive optional memory plugin rather than a core bug fix or required platform surface, we’re closing it on scope grounds. If you want to keep pursuing it, the recommended path is to publish and maintain it as a community plugin installable via `openclaw plugins install` / ClawHub.
