## Summary

Keep open. Current main still has the core failure mode for legacy Feishu configs: `channels.feishu.accounts.*.botName` is not accepted or migrated to `name`, and the Feishu plugin exposes no doctor compatibility normalizer for that legacy field. Main does have last-known-good recovery around gateway startup/reload, but `openclaw doctor --fix` preflight only repairs a narrow non-JSON-prefix case and does not restore arbitrary corrupted JSON from `.bak`/last-known-good backups. The linked #65177 is useful related evidence for the Feishu `botName` gap, but it is still open and narrower than this issue's broader doctor recovery request.

## Best Possible Solution

Keep the issue open unless maintainers intentionally split it. The concrete path is to add a Feishu doctor/plugin compatibility migration for legacy `accounts.*.botName -> name` and legacy `accounts.default` policy promotion, then add a doctor recovery path that can safely restore a syntactically invalid `openclaw.json` from the newest valid last-known-good or backup before applying schema migrations. Link #65177 as the narrower Feishu migration tracker if the broader backup/doctor recovery work is moved elsewhere.
