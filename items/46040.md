## Summary

Close as `cannot_reproduce`: the reporter’s own follow-up attributes the crash to plugin-side `embedding.ts` parsing, and current `main` shows the `before_prompt_build` hook path passing `messages` explicitly, tolerating `void` returns, and covering empty-message cases in tests without any core-side `reduce` path.

## Evidence

- **Reporter follow-up says root cause was plugin code, not OpenClaw:** The issue comment states the `reduce` crash came from the plugin’s `embedding.ts` handling of nested llama.cpp embeddings, and that after fixing the plugin plus upgrading OpenClaw, `before_prompt_build` worked normally.

## Close Comment

Closing as `cannot_reproduce` after a Codex review of current `main`.

The current OpenClaw hook path already passes `messages` into `before_prompt_build` (`src/plugins/hook-before-agent-start.types.ts:21`, `src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts:43`), accepts `void` hook returns (`src/plugins/hook-types.ts:661`), and merges prompt-build results with `concatOptionalTextSegments` rather than a `reduce` path (`src/plugins/hooks.ts:231`, `src/plugins/hooks.ts:347`). Current tests also exercise `before_prompt_build` with `messages: []` without an internal crash (`extensions/memory-lancedb/index.test.ts:240`).

Most importantly, the follow-up comment on this issue identifies the actual fault as plugin-side `embedding.ts` parsing and says the hook worked after fixing that plugin code. Given that evidence, there is no remaining actionable OpenClaw bug to track in this issue.
