## Summary

Keep PR #39957 open. Current main still lacks the PR's unphased `message_end && stopReason === "toolUse" && tool-call block` suppression, while the newer commentary-phase suppression only covers messages explicitly marked as commentary. The PR remains a focused, concrete fix for a still-uncovered Telegram block-reply leak.

## Best Possible Solution

Keep this PR open and either merge/rebase the narrow `message_end && stopReason === "toolUse" && has tool-call block` suppression with regression coverage, or implement equivalent behavior through the current display-phase model so unphased intermediate tool-use turns cannot leak into Telegram-visible block replies.
