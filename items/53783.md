## Summary

Keep open. Current main still routes Telegram group conversations into agent-scoped session keys and then applies the generic session visibility plus agent-to-agent policy for both sessions_list and sessions_send. I found no current same-Telegram-group cross-agent symmetry rule, no regression test for two agents in the same group seeing/sending both directions, and no merged/superseding PR. Related #59933 remains open and covers same-agent group visibility, not this cross-agent one-way failure.

## Best Possible Solution

Keep this issue open. Reproduce with two configured agents in the same Telegram group using explicit tools.sessions.visibility="all" and enabled, allowlisted tools.agentToAgent. If it reproduces, add regression coverage that each agent sees both agent:<agent>:telegram:group:<id> keys in sessions_list and can sessions_send both directions, then fix the gateway/session-store/visibility path. If the intended behavior is opt-in only, improve diagnostics so users see the missing config reason instead of one-way missing-session behavior. Track #59933 separately because it is same-agent group listing, not this cross-agent mismatch.
