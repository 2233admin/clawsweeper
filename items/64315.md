## Summary

Current main still does not implement optional marquee/overflow-scrolling for long status information. The relevant Control UI and TUI surfaces remain static, ellipsized, wrapped, hidden on compact layouts, or plain text updates. The prior Codex review comment keeping this issue open remains valid after rechecking current main at c7543701009ccdae83d38ee341e072c8a7c4af6e.

## Best Possible Solution

Keep this issue open as a product/UI enhancement. The best path is a reusable accessible overflow-status component/helper, then explicit opt-in use in selected Control UI and TUI status/footer surfaces. It should render statically when content fits, scroll only on real overflow, respect reduced-motion preferences, pause or allow user control on hover/focus, and treat #13670 as related but narrower model/token visibility work rather than a replacement.
