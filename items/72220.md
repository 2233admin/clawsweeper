## Summary

Keep PR #72220 open. Current main still has the ownerless, unconditional `.openclaw-runtime-deps-*` cleanup path targeted by the PR, and the overlapping runtime-postbuild race is still plausible on the active build surface. The PR discussion is useful: Greptile and QC both identify the same owner-metadata approach as addressing the main race, while Greptile also notes a small backup-directory protection gap that maintainers should review before merge.

## Best Possible Solution

Keep PR #72220 open for maintainer review. The best path is to land this PR or an equivalent focused fix in `scripts/stage-bundled-plugin-runtime-deps.mjs` that marks active runtime-deps staging temp directories with owner metadata, skips cleanup for live owners, includes the concurrent-cleanup regression test, and either fixes or explicitly accepts the backup-directory owner-protection gap.
