## Summary

Keep PR #72220 open. Current main still has the build-time runtime-deps temp cleanup behavior this PR targets: staging temp directories are ownerless, and `removeStaleRuntimeDepsTempDirs` deletes every `.openclaw-runtime-deps-*` entry before staging. The existing runtime install lock is adjacent runtime behavior and does not protect this postbuild staging cleanup path.

## Best Possible Solution

Keep PR #72220 open for maintainer review. The best path is to land this PR or an equivalent fix in `scripts/stage-bundled-plugin-runtime-deps.mjs` that marks active build/postbuild staging temp directories with owner metadata, skips cleanup for live owners, adds concurrent-cleanup regression coverage, and either fixes or explicitly accepts the backup-directory owner gap.
