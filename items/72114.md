## Summary

Keep #72114 open. Current main has useful bundled-runtime-deps hardening, including filesystem lock cleanup and restart deferral for active agent work, but I did not find a source-backed fix for the reported cold-cache browser runtime-deps staging failure plus update/service shutdown race. The supplied related #72058 tracker and #72084 PR are adjacent and still open in the context, but they do not clearly supersede this narrower SIGTERM/SIGKILL staging interruption report.

## Best Possible Solution

Keep #72114 open as a focused runtime-deps/update restart-race bug. The likely core fix is to track bundled runtime-deps staging as pending restart/shutdown work, prevent update/service-manager restarts while an npm staging subprocess is active, harden interrupted staging cleanup/retry for cold cache installs, and add regression coverage for browser runtime-deps staging during an update-driven gateway restart. If maintainers want #72058 to own this, mark #72114 duplicate only after that tracker explicitly names this shutdown/SIGKILL failure mode.
