## Summary

Keep open. Current main still supports provider/model hook overrides only; it does not let `before_model_resolve` or legacy `before_agent_start` prefer an auth profile. PR #71862 is an active, focused core/plugin API change that adds the missing `authProfileOverride` propagation and runner selection behavior. Related PR #68621 tracks the same idea but is older and lacks the latest legacy auth-source fixes, so #71862 is not the duplicate to close.

## Best Possible Solution

Keep #71862 open as the current candidate implementation for hook-selected auth profile preference. Maintainers should compare it with older #68621, likely close or supersede the older PR if #71862 remains the better branch, then land the API/docs/tests once validation is green.
