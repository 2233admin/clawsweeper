## Summary

Keep PR #71862 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still exposes `before_model_resolve` as provider/model-only, and the embedded runner still derives auth profile selection only from request/session state rather than hook output. The PR is a focused core/plugin API change with active author follow-up, review history, docs/tests in the PR context, and it appears to supersede the older overlapping PR #68621 rather than being superseded by it.

## Best Possible Solution

Keep #71862 open as the current candidate implementation. The best path is to add `authProfileOverride` to the typed `before_model_resolve` and legacy `before_agent_start` result contract, merge it with the same priority semantics as provider/model overrides, apply it in runner auth-profile selection only when there is no explicit user pin, keep the SDK/docs/tests aligned, and then close older PR #68621 as superseded if #71862 lands.
