## Summary

Keep open. Current `main` still does not implement the proposed `agents.defaults.systemPromptFile` behavior, and the shipped alternatives are not equivalent to this PR's operator-managed file-prepend design.

## Evidence

- **`agents.defaults` schema on `main` has no `systemPromptFile` field:** The current agent-defaults schema accepts `systemPromptOverride` and `promptOverlays`, but there is no `agents.defaults.systemPromptFile` key to parse or validate.
  - file: [src/config/zod-schema.agent-defaults.ts:66](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/config/zod-schema.agent-defaults.ts#L66)

  - file: [src/config/types.agent-defaults.ts:128](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/config/types.agent-defaults.ts#L128)

## Risks / Open Questions

- Closing this PR would drop a feature that is still absent on current `main`.
- The presence of backend `systemPromptFileConfigKey` plumbing could be misread as the feature already existing, but it is not an `agents.defaults` user-facing config surface.
- `systemPromptOverride` is a stronger and different mechanism; treating it as equivalent would ignore the PR's preserve-and-prepend semantics and operator-managed file workflow.
