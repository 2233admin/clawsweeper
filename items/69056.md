## Summary

Keep PR #69056 open. Current main still does not implement the PR’s central Windows no-listener restart fix: `emitGatewayRestart()` falls back to `process.kill(process.pid, "SIGUSR1")`, and `scheduleGatewaySigusr1Restart()` still reports mode solely from listener count. The PR is a focused bug-fix attempt for a real Windows restart path, with related issue #69057 already closed as superseded by this PR, but it still needs maintainer review because the PR discussion raised state/token handling and regression-test concerns.

## Best Possible Solution

Keep #69056 open for maintainer review or a maintainer replacement patch. The best fix should preserve the listener-driven `process.emit("SIGUSR1")` path when a gateway run-loop listener exists, avoid `process.kill(..., "SIGUSR1")` on Windows when no listener exists, consume or roll back restart tokens on every Windows fallback outcome, report scheduler mode accurately, and add focused tests for Windows success/failure and coalescing behavior. Coordinate with overlapping PR #68853 if maintainers choose one canonical restart-path fix, but #69056 is not safely closeable from current main.
