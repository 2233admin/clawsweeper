## Summary

Keep open. Current `main` still routes Windows `.cmd`/`.bat` execution through `cmd.exe /c` without the proposed UTF-8 code-page prefix or `runExec` encoding override, the regression tests on `main` do not cover that behavior, and the docs still describe `chcp 65001` as a user workaround for garbled Chinese output. The PR discussion also shows the contributor addressed the review-noted regressions in follow-up commits, so this remains plausible useful work rather than something that should be conservatively auto-closed.
