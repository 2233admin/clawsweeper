## Summary

Issue #57796 should stay open. Current `main` only fixes the `before_reset` path by reloading transcript messages from disk; `agent_end` and `before_compaction` still dispatch in-memory snapshots or omit `messages`, so provenance is not preserved consistently across the requested hook events.

## Evidence

- **`agent_end` still uses in-memory messages:** The embedded runner calls `runAgentEnd` with `messagesSnapshot` directly. There is no transcript reload before dispatch, so this path still depends on the in-memory message array the report identified as losing `provenance`.
