## Summary

Keep #72066 open. Current main still has the Bedrock aws-sdk media-auth bug this PR is trying to fix, and the PR discussion identifies concrete follow-up needed rather than showing the work is obsolete or superseded.

## Best Possible Solution

Keep the PR open for revision. The best fix should align media understanding with the existing aws-sdk no-static-key contract: allow aws-sdk mode without `requireApiKey` throwing, avoid registering an empty runtime credential, use the existing runtime-auth/sentinel pattern where auth storage must be marked configured, update runner key-rotation handling for aws-sdk, and add real regression coverage for image plus the shared auth helper or runner path.
