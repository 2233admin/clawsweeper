## Summary

Keep #56132 open. The attachment describes a concrete Feishu/Lark feature/regression request: `sessions_spawn({ thread: true, mode: "session" })` should bind a spawned subagent to a normal group chat. Current `main` still rejects non-topic Feishu group conversations, and the existing Feishu docs only cover static per-group routing as a workaround, not the requested dynamic subagent binding behavior.

## Best Possible Solution

Keep the issue open for maintainer/product review. A real fix would likely extend Feishu current-conversation subagent binding in `extensions/feishu/src/subagent-hooks.ts` to safely resolve normal group conversations, add tests replacing the current fail-closed expectation, and document any security/routing constraints. If maintainers decide static `bindings[]` is the supported path, close manually with that product decision and docs link rather than auto-closing.
