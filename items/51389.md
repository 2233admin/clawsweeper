## Summary

Keep open. Current `main` does not implement `fallbackOnErrors`: the shared model config type and strict Zod schema still only allow `primary`/`fallbacks`, and the fallback runtime/call sites still route through the legacy `coerceToFailoverError` path with no configurable error-code allowlist. The PR discussion reflects concrete, review-driven iteration on a real core fallback feature, not an incoherent, out-of-scope, or already-shipped change.
