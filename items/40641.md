## Summary

#40641 should stay open. Current main still contains the unquoted bracket-notation config examples that trigger zsh `no matches found`, including the exact `agents.list[0].tools.exec.node` command from the issue. The CLI path parser itself accepts bracket notation once the shell passes the argument through, so this remains a docs regression rather than an implemented fix.

## Best Possible Solution

Keep the issue open as a concrete docs regression. Reapply the intent of closed-unmerged PR #40642: quote every config CLI example that uses bracket notation, or change those examples to dot-index paths like `agents.list.0.tools.exec.node`, and add a short note in the config docs explaining that paths containing `[]` must be quoted in shells such as zsh. Update the English source docs (`docs/cli/config.md`, `docs/tools/exec.md`, `docs/nodes/index.md`) and let the docs publish/i18n pipeline refresh zh-CN output.
