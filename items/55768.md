## Summary

Keep open. Current `main` still creates a fresh random session key for OpenAI-compatible chat requests that omit both `x-openclaw-session-key` and `user`, and the compat handler still feeds that session key into the full agent runtime. The lightweight probe endpoints exist and deployment docs already point some environments at `/health` or `/healthz`, but I did not find an explicit current-main fix for the reported session-bloat path or a gateway warning/rate-limit mitigation.
