## Summary

Keep open. Current `main` still creates a fresh session key for OpenAI-compatible requests without `x-openclaw-session-key` or `user`, still routes `/v1/chat/completions` through the full agent ingress path, and still exposes a separate shallow `/health` probe. The issue's core request and operational guidance are not implemented on `main`.
