## Summary

Keep PR #63870 open. Current main has superseded the original Claude CLI session-reset symptom with the stable `mcpResumeHash` path, but it still does not implement the PR's `gateway.mcpLoopback.port` config surface or any other operator-facing way to pin the loopback MCP port. That remaining capability is plausible core gateway/config work, so this is not a conservative cleanup close.

## Best Possible Solution

Keep the PR open, but rebase/rework it against current main: add `gateway.mcpLoopback.port` to gateway config types, Zod/schema/help/generated metadata and docs, then thread it into the current lazy `ensureMcpLoopbackServer(configuredPort)` path with targeted tests for configured-port binding and bind-failure diagnostics. If maintainers decide the `mcpResumeHash` fix fully eliminates the need for a stable operator port, close only after that explicit product judgment or a canonical follow-up is recorded.
