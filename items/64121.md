## Summary

Keep #64121 open. Current main still exposes one combined chat session picker on desktop and mobile; agent grouping exists only as optgroups inside that picker, and the requested separate Agent plus Agent Session controls have not been implemented. The related #69552 work is adjacent subagent-noise cleanup, not a canonical replacement for this broader split.

## Best Possible Solution

Keep this issue open and implement it in the Control UI chat surfaces: derive the selected agent from the active session key and `agentsList`, render a dedicated Agent control, render an Agent Session control filtered to that agent's sessions, and preserve the existing Model/Thinking `sessions.patch` behavior against the active session key. Treat #69552 as a narrower follow-up for transient/subagent-session noise rather than as a replacement for this feature.
