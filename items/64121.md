## Summary

Current main still has a single chat session picker grouped by agent, not separate Agent and Agent Session controls. Model and Thinking remain correctly bound to the active session, but the requested UI split for desktop header and mobile controls is still missing. The related open agent-switcher issue #68959 is complementary, not a complete superseding tracker for this split-picker behavior.

## Best Possible Solution

Keep this issue open. Implement the split in the Control UI chat surfaces: add an Agent selector from agentsList, add an Agent Session selector filtered from sessionsResult by parseAgentSessionKey(...).agentId, switch agents to the most recent session for that agent or buildAgentMainSessionKey({ agentId }) as fallback, and preserve existing Model/Thinking sessions.patch behavior against state.sessionKey. Update both desktop header and mobile controls, with focused tests for filtering, agent-switch fallback, and session-bound Model/Thinking behavior. Coordinate with #68959, but do not close this issue unless a canonical item explicitly covers the full split-picker behavior.
