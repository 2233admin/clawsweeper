## Summary

Current `main` still contains the reported WebChat state bug. The cross-run `final` branch in `ui/src/ui/controllers/chat.ts` appends the assistant message but does not clear `chatRunId`, `chatStream`, or `chatStreamStartedAt`, and the existing tests explicitly lock in that behavior. Separate UI code still treats a non-null `chatRunId` as busy/typing state, so the report remains actionable rather than closable.

## Evidence
