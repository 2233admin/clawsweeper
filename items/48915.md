## Summary

Current `main` does not implement the reported snake_case handling in session transcript repair, so this is not safe to close. `src/agents/session-transcript-repair.ts` still recognizes only `toolCall`/`toolUse`/`functionCall`, and the matching tests still cover only those camelCase forms. Some other current-main code already accepts snake_case tool-call markers, which makes the mismatch real, even though the exact Anthropic/Feishu reproduction path was not live-verified here.
