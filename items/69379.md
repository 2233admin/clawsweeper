## Summary

Keep #69379 open. Current main still uses `reserveTokensFloor` alone for both preflight compaction and memory-flush gate threshold math, while the config help and Pi settings path define `reserveTokensFloor` as a minimum floor for `reserveTokens`. The related #67297 is a duplicate open PR, but #69379 is active and more complete, so the conservative maintainer cleanup action is not to close this PR.

## Best Possible Solution

Land or adapt the #69379 approach: add a single effective-reserve helper equivalent to `max(agents.defaults.compaction.reserveTokens, reserveTokensFloor)`, use it for both preflight-compaction and memory-flush gate thresholds, keep regression tests for `reserveTokens > reserveTokensFloor` and the threshold clamp, then close #66830 and supersede the duplicate #67297.
