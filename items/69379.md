## Summary

Keep #69379 open. Current main still has the floor-only threshold behavior the PR fixes: both preflight compaction and memoryFlush gates compute from `reserveTokensFloor`, while the config help and Pi settings path define the effective reserve as `max(reserveTokens, reserveTokensFloor)`. The related duplicate PR #67297 was closed unmerged, and #66830 remains the open bug this PR addresses.

## Best Possible Solution

Keep #69379 open for maintainer review, or land an equivalent fix that threads an effective reserve (`max(reserveTokens, reserveTokensFloor)`) through both preflight and memoryFlush gate paths with regression tests, then close #66830. #67297 should not be treated as the canonical replacement because it was closed unmerged.
