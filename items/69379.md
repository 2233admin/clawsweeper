## Summary

Keep this PR open. Current main still computes both preflight compaction and memory-flush thresholds from `reserveTokensFloor` only, while current config help, docs, and the Pi settings path treat `reserveTokensFloor` as a minimum under `reserveTokens`. This PR remains the active implementation candidate for #66830; the related earlier PR #67297 is closed unmerged.

## Best Possible Solution

Keep this PR open as the canonical implementation path for #66830. Land this helper or equivalent `max(reserveTokens, reserveTokensFloor)` logic through both preflight compaction and memory-flush gates, keep the focused regression coverage and changelog entry, then let #66830 close after merge.
