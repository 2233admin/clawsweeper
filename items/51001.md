## Summary

Keep open. Current `main` still does not implement the requested per-job cron retry feature: recurring-job error handling still schedules the later of the natural next run and the exponential backoff, so low-frequency jobs can still sit until the next scheduled slot; the CLI, cron types, gateway schema, and docs also do not expose `--retry-count` / `--retry-delay`. PR #51001 is still open and unmerged, so there is no strong basis to close it as already implemented or obsolete.
