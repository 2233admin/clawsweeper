## Summary

Keep open: current `main` still does not implement the PR’s per-job recurring retry feature or its CLI/protocol surface, and the PR discussion shows unresolved maintainer-judgment questions rather than an obsolete report.

## Evidence

- **Current recurring-job behavior is still generic backoff, not per-job cadence-preserving retry:** `applyJobResult` on `main` still handles recurring job errors by computing a normal next run, then scheduling `nextRunAtMs` as the later of the natural next run and exponential backoff. That means an hourly/daily recurring job still waits until its natural next slot instead of using an earlier fixed-delay retry window.
