## Summary

Keep open. Current `main` gained a first-class `openclaw backup` flow on 2026-04-25, and it already covers `.tar.gz` archives, manifest verification, canonical-path dedupe for symlinked workspaces, and coarse size controls via `--no-include-workspace` / `--only-config`. But the proposal’s key asks for configurable exclusion rules and service-aware gateway stop/start are not present in the current codepath, and this new backup command is not in the latest shipped release `v2026.4.23`. The issue body’s linked `JIRBOY/openclaw-backup` repo is useful evidence that the design is practical, but OpenClaw core only has a narrower partial implementation so far.
