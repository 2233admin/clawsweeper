## Summary

Keep open. Current `main` and the latest shipped release still use an unguarded `withCheckedThrowingContinuation` around `URLSessionWebSocketTask.sendPing`, so this PR’s crash fix is not already implemented. The report is technically coherent, scoped to a real code path used by the keepalive loop, and the existing discussion supports the underlying bug even though the PR may still need follow-up review on the explicit `import os` detail.
