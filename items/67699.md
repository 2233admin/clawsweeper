## Summary

Keep open. Current `main` still has the core overwrite path described by this PR: tool-assisted final chat events reload `chat.history`, and `loadChatHistory()` still unconditionally replaces `state.chatMessages` with the server snapshot. Some review follow-ups from the thread are already present on `main`, but the main race this PR targets is not clearly implemented there.
