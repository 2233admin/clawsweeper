## Summary

Keep this PR open. Current main still has no default-source warning in the context-window guard or embedded runtime setup, and the PR remains a focused logging change with no observed security-sensitive diff, but it still does not cover the Bedrock fallback path that motivated the discussion because Bedrock discovery materializes unknown fallback values as model metadata.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. If this warning is still wanted, make the signal source-aware enough to distinguish true provider/model metadata from hardcoded fallback values, including Bedrock unknown-model/defaultContextWindow fallbacks, and add tests for both generic default resolution and Bedrock fallback discovery. If maintainers consider the shipped Bedrock known-context-window metadata sufficient, close this PR explicitly as superseded by that product direction.
