## Summary

Keep open. Current `main` still stores cron job definitions in a single `jobs.json` file with a separate `jobs-state.json` sidecar for runtime fields; it does not implement per-job config files or directory discovery, so there is no strong basis to close this feature request.

## Evidence

- **Cron store path is still a single `jobs.json` file:** `resolveDefaultCronStorePath()` returns `cron/jobs.json`, and `loadCronStore()` reads one store file whose `jobs` field is an array.
  - file: [src/cron/store.ts:30](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/cron/store.ts#L30)

  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Current save path still rewrites one config file plus one state sidecar:** `saveCronStore()` serializes the full job list back to `storePath` and writes runtime state to `jobs-state.json`; there is no per-job file write path or directory scan implementation here.
  - file: [src/cron/store.ts:259](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/cron/store.ts#L259)
