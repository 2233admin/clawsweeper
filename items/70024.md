## Summary

Keep open. Current main still matches the reported failure chain: stop timeout leaves `running: true`, does not clean `store.aborts`/`store.tasks`, and `startChannelInternal` still skips future starts while a stale task entry exists. The linked fix PR #70056 is open and unmerged, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep #70024 open as the canonical bug until PR #70056 or an equivalent maintainer-approved change lands. The fix should update the stop-timeout branch in `src/gateway/server-channels.ts` to clear `store.aborts` and `store.tasks`, mark the account `running: false`, record `lastStopAt`, and update the timeout regression test to prove a later `startChannel` can start a new task.
