## Summary

Keep open. Current `main` does not expose Slack Socket Mode ping/pong timeout tuning in config, schema, runtime wiring, or docs, and the upstream Slack SDK contract still supports the workaround this PR implements.

## Evidence

- **Slack config types still lack `socketMode`:** `SlackAccountConfig` and `SlackConfig` on current `main` have no `socketMode`, `clientPingTimeout`, `serverPingTimeout`, or `pingPongLoggingEnabled` fields, so the requested surface is not implemented.
  - file: [src/config/types.slack.ts:109](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/config/types.slack.ts#L109)
  - command: `nl -ba src/config/types.slack.ts | sed -n '109,225p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Runtime schema still rejects any such setting:** `SlackAccountSchema` and `SlackConfigSchema` do not define `socketMode`; both remain strict schemas, so the requested config cannot be accepted on current `main`.
  - file: [src/config/zod-schema.providers-core.ts:897](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/config/zod-schema.providers-core.ts#L897)
  - command: `nl -ba src/config/zod-schema.providers-core.ts | sed -n '897,990p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Slack runtime does not wire timeout options through:** Slack monitor startup creates the Bolt app with only `clientOptions`, and `createSlackBoltApp()` constructs `SocketModeReceiver` with `appToken`, `autoReconnectEnabled`, and installer `clientOptions` only. There is no current path to set ping/pong timeouts or ping/pong logging.
  - file: [extensions/slack/src/monitor/provider-support.ts:162](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/extensions/slack/src/monitor/provider-support.ts#L162)

- **User-facing docs do not document the feature:** Current Slack troubleshooting docs stop at generic Socket Mode connectivity checks, with no pong-timeout guidance or `channels.slack.socketMode` example on `main`.
  - file: [docs/channels/slack.md:840](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/docs/channels/slack.md#L840)
