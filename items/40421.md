## Summary

Keep this PR open. Current main still does not implement Telegram waiting-tip messages or a messages.waitingTips config surface, so the work is not obsolete or implemented. The submitted patch is also not merge-ready: it targets the pre-plugin src/telegram layout, exposes an unimplemented customTipsPath config, and review comments identify a send/cancel race that can leave Telegram tip messages behind.

## Best Possible Solution

Keep this PR open for maintainer or author follow-up rather than closing it. The useful path is to rebase onto extensions/telegram/src/bot-message-dispatch.ts, decide whether this should be a narrow Telegram processing indicator aligned with #6946 or a product-approved bilingual tips feature, fix the send/cancel cleanup race, remove or fully implement customTipsPath, and add focused Telegram dispatch coverage for fast-stream cancellation, successful cleanup, cleanup failure, forum-topic thread params, and superseded turns. If maintainers decide bilingual tips are not a core Telegram UX, close after pointing contributors toward an external plugin/API discussion rather than treating current main as implemented.
