## Summary

Keep open. Current main does not show a fix for the reported node-host restart/reconnect stall, the relevant reconnect code is unchanged from the latest release path, and the existing surface still exposes only successful-connect timestamps rather than a dedicated liveness signal. The report is recent, concrete, and still plausible against current main.

## Evidence
