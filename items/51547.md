## Summary

Keep open. Current `main` still uses UTF-8-only decoding in `runExec` and UTF-8 `toString()` in `runCommandWithTimeout`, so the Windows mojibake report remains plausible, but this PR should not be auto-closed because the proposed `gbk` change is not safe: unsupported encodings flow back as `Buffer`s and would break the string contract used by current callers.

## Evidence
