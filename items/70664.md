## Summary

Keep open. Current main still has the reported Unix/macOS behavior: lsof output is parsed through the `c` command-name field and only PIDs whose command contains `openclaw` survive, so a real Node-backed gateway reported as `cnode` would be dropped. The linked fix PR #70681 directly targets this issue but is still open in the provided GitHub context, so the bug is not implemented on main yet.

## Best Possible Solution

Keep #70664 open and review/land PR #70681 or an equivalent patch. The fix should remove gateway classification from the lsof `c` token, verify Unix listener PIDs via full argv such as `ps -ww -p <pid> -o command=`, apply that verifier consistently in `findGatewayPidsOnPortSync` and `pollPortOnce`, and update tests so `cnode` plus `openclaw-gateway` argv is recognized as a gateway while unrelated listeners remain ignored.
