## Summary

Keep open. Current `main` still identifies Unix gateway listeners solely by the lsof `c` field containing `openclaw`, with no `ps`/argv verification step. That means the code on `main` does not implement the fix described in #70664, and the existing tests still encode the same command-name assumption the report says is wrong. The issue is also fresh and already cross-linked to open PR #70681, so there is active follow-up rather than stale or obsolete state.
