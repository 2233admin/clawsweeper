## Summary

Keep open. Current `main` does not implement Discord edited-message mention activation: the Discord monitor registers only a create-message listener, there is no `MessageUpdate`/`MESSAGE_UPDATE` listener in `extensions/discord`, and mention gating runs only after a message event reaches the existing inbound preflight path.

## Evidence

- **Discord startup registers no edit-message listener:** `registerDiscordMonitorListeners()` adds `DiscordMessageListener`, reaction listeners, `DiscordThreadUpdateListener`, and optional presence listener only. There is no listener registration for edited messages.
  - file: [extensions/discord/src/monitor/provider.startup.ts:253](https://github.com/openclaw/openclaw/blob/a8edf29bd0f1bd3c869e5e3fab91043d01e6ac33/extensions/discord/src/monitor/provider.startup.ts#L253)
  - command: `nl -ba extensions/discord/src/monitor/provider.startup.ts | sed -n '245,305p'`
  - sha: [a8edf29bd0f1](https://github.com/openclaw/openclaw/commit/a8edf29bd0f1bd3c869e5e3fab91043d01e6ac33)
- **Inbound Discord message type is create-only:** `DiscordMessageEvent` is typed from `MessageCreateListener["handle"]`, and `DiscordMessageListener` extends `MessageCreateListener`. The primary inbound path is therefore wired to create events, not update events.
  - file: [extensions/discord/src/monitor/listeners.ts:46](https://github.com/openclaw/openclaw/blob/a8edf29bd0f1bd3c869e5e3fab91043d01e6ac33/extensions/discord/src/monitor/listeners.ts#L46)
  - command: `nl -ba extensions/discord/src/monitor/listeners.ts | sed -n '1,90p;200,250p'`
  - sha: [a8edf29bd0f1](https://github.com/openclaw/openclaw/commit/a8edf29bd0f1bd3c869e5e3fab91043d01e6ac33)
- **Mention gating exists but only after an inbound event is delivered:** Preflight computes `explicitlyMentioned`, `hasAnyMention`, and `wasMentioned`, then drops guild messages when `requireMention` is enabled and `mentionDecision.shouldSkip` is true. That logic would handle a message containing a new mention, but only if the edited message re-entered preflight.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:708](https://github.com/openclaw/openclaw/blob/a8edf29bd0f1bd3c869e5e3fab91043d01e6ac33/extensions/discord/src/monitor/message-handler.preflight.ts#L708)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '708,735p;916,1005p'`
  - sha: [a8edf29bd0f1](https://github.com/openclaw/openclaw/commit/a8edf29bd0f1bd3c869e5e3fab91043d01e6ac33)
- **No MESSAGE_UPDATE implementation found in Discord extension:** Repository search on current `main` found no `MessageUpdateListener`, `MESSAGE_UPDATE`, or `messageUpdate` references under `extensions/discord/src`.
  - command: `rg -n "MessageUpdateListener|MESSAGE_UPDATE|messageUpdate" extensions/discord/src`
  - sha: [a8edf29bd0f1](https://github.com/openclaw/openclaw/commit/a8edf29bd0f1bd3c869e5e3fab91043d01e6ac33)
