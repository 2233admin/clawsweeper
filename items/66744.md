## Summary

Keep open. Current main still contains the unguarded `provider?.models?.some(...)` call that can throw the reporter's exact TypeError for non-array runtime provider data, and the provided GitHub context shows open PR #66761 is the unmerged implementation candidate with closing intent.

## Best Possible Solution

Keep this issue open and review PR #66761 as the canonical implementation candidate. The likely fix remains a narrow `Array.isArray(provider?.models)` guard in `src/agents/models-config.providers.policy.lookup.ts` with focused regression tests in `src/agents/models-config.providers.policy.lookup.test.ts`; after that PR merges, close as implemented. A separate follow-up can address upstream provider-shape normalization if maintainers want stricter validation.
