## Summary

Keep #70851 open. Current main still has no picker-side internal-session filter and no gateway-level `visibility=internal` contract for `sessions.list`; the related #69552 is closed unmerged and narrower, so it does not supersede this PR. The PR should likely be revised rather than closed, because its own discussion correctly frames the UI filter as only an interim mitigation and review comments identify heuristic false-positive risk.

## Best Possible Solution

Keep the PR open for maintainer/product handling. The best path is to add an explicit gateway session visibility/internal classification contract for subagent, ACP, and node-backed runtime sessions, expose an admin/debug opt-in path for internals, then make the chat picker hide internal rows by default while preserving the active internal-session exception. If this PR remains the vehicle, revise it away from broad string heuristics and add focused tests for the explicit classification and spawnDepth fallback.
