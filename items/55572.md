## Summary

Keep open. Current `main` still rejects `api: "google-vertex"` in config validation, and the requested change is not already implemented. The codebase has partial `google-vertex` provider support (aliasing, model normalization, env auth handling), but transport normalization/runtime tables still only recognize `google-generative-ai`, so the issue remains real and the proposed 3-line fix looks incomplete rather than already shipped.
