## Summary

Keep open. Current main has partial mitigation for stale media refs, including the #71868 old-history marker scrub and transcript-prompt separation on the auto-reply path, but it still does not implement the reported current-turn-only media behavior. Recent history image blocks and media markers are intentionally preserved, and prompt image hydration still scans the submitted prompt text it receives.

## Best Possible Solution

Implement a focused embedded-runner media policy: when the current turn has attachments, default to sending only current-turn images unless the user explicitly asks to compare earlier images; mask or ignore image refs inside injected memory and untrusted-context blocks before prompt-ref hydration; and add regression coverage for Telegram topic sessions, memory/untrusted-context stale refs, and recent image-heavy history.
