## Summary

Keep open. The proposed regex-only change does not fix the user-visible Telegram send path on current `main`: outbound sends use a strict retry predicate that bypasses the fallback regex entirely, and current 421 handling is still absent from that strict path.

## Evidence

- **Telegram outbound sends use strict retry mode:** `createTelegramNonIdempotentRequestWithDiag` configures `shouldRetry: (err) => isSafeToRetrySendError(err) || isTelegramRateLimitError(err)` together with `strictShouldRetry: true`. `sendMessageTelegram` constructs this helper for normal outbound sends.
  - file: [extensions/telegram/src/send.ts:606](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/telegram/src/send.ts#L606)
