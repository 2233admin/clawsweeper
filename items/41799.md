## Summary

Keep open. Current `main` does not implement the compatibility shim from the PR: object-shaped `{ ok, to }` returns from `messaging.normalizeTarget` are still dropped by core normalization, and the send path still requires a string `to`. The PR likely needs a refresh because current tests are split differently, but the underlying core bug/compatibility gap is still present and not superseded on `main`.

## Evidence
