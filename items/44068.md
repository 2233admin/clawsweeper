## Summary

Keep open. Current `main` does not include the PR's `application/msword` binary guard or its regression test, and the existing text-coercion path still makes the reported misclassification plausible.

## Evidence

- **Current binary-MIME guard still omits legacy Word MIME:** `isBinaryMediaMime` on current `main` treats images/audio/video, `application/octet-stream`, archive types, `+zip`, and `application/vnd.*` as binary, but there is no `application/msword` case.
  - file: [src/media-understanding/apply.ts:309](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/media-understanding/apply.ts#L309)
  - command: `nl -ba src/media-understanding/apply.ts | sed -n '309,340p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Main code still allows heuristic text coercion for non-binary attachments:** After the binary-MIME early exit, `extractFileBlocks` still runs UTF-16 detection and text/delimited heuristics, with only a PDF-specific heuristic guard. That leaves `application/msword` eligible to fall through to `text/plain`/TSV coercion if its bytes look text-like.
  - file: [src/media-understanding/apply.ts:390](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/media-understanding/apply.ts#L390)
  - command: `nl -ba src/media-understanding/apply.ts | sed -n '386,406p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **No equivalent regression test exists on main:** Current tests cover UTF-16 text attachments and PDF non-reclassification, but there is no test for legacy `.doc` attachments with `application/msword`, which is the regression this PR adds.
  - file: [src/media-understanding/apply.test.ts:1201](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/media-understanding/apply.test.ts#L1201)

  - file: [src/media/mime.ts:30](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/media/mime.ts#L30)
