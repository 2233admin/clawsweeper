## Summary

Keep open. Current `main` does not already implement this PR's CLI change, and the proposed approach still has unresolved contract/scope problems. `openclaw nodes list` on `main` still uses `node.pair.list` as its primary source, while `node.list` now already merges device-pairing, node-pairing, and live registry state. The PR's extra Ollama auth commit is also stale against the current plugin-owned Ollama auth path.

## Evidence

- **`nodes list` still keys off `node.pair.list` on main:** The `list` command fetches `node.pair.list` first, builds `paired` rows from that payload, and only consults `node.list` for `--connected` / `--last-connected` filtering. JSON output still serializes `{ pending, paired: filteredPaired }` from the pairing-list shape.
