## Summary

Keep open. Current main still has the reported behavior gap: cron marks a trailing error payload fatal unless a later non-error payload text exists, and there is no `===DONE_OK===` or equivalent success sentinel handling.

## Best Possible Solution

Keep PR #72135 open for maintainer review. The remaining work belongs around `src/cron/isolated-agent/helpers.ts` with focused regression coverage, while maintainers decide whether the right contract is `===DONE_OK===`, a cron-specific token, a config gate, or an explicit outcome signal.
