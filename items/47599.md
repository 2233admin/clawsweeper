## Summary

Current main already provides supported ingress for this use case: Gateway WS `sessions.send` / `sessions.steer` can inject text into an existing or active session, and built-in hook/webhook endpoints let local scripts post text into OpenClaw with explicit session routing and optional channel delivery. A watched inbox directory would be a convenience wrapper over existing ingress, not a missing core capability.

## Close Comment

Closing as implemented on current main after Codex review. OpenClaw already ships supported ingress paths for this use case: `sessions.send` / `sessions.steer` send text into an existing or active session (`docs/gateway/protocol.md:345`-`346`, covered by `src/gateway/server.chat.gateway-server-chat.test.ts:198` and `:225`), and local scripts can post text into OpenClaw through built-in hook/webhook ingress instead of spoofing a chat API (`docs/automation/cron-jobs.md:195`, `docs/gateway/configuration-reference.md:467`, `src/gateway/server/hooks.ts:35`). That shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). A watched inbox directory would be a convenience wrapper on top of those existing ingress surfaces rather than a missing core capability.
