## Summary

Keep open. Current `main` still has the broken `/export-session` HTML wiring: `template.html` no longer contains contiguous `{{MARKED_JS}}`/`{{HIGHLIGHT_JS}}`/`{{JS}}` tokens, while `generateHtml()` still relies on exact-string `.replace()` calls for those placeholders. The latest release tag shown in this review contains the same code, so this is still a shipped user-facing bug rather than a stale PR. Existing tests also mock a simplified template path and do not cover the real template injection failure.

## Evidence

- **Current main template still breaks placeholder matching:** `template.html` on `main` renders the JS placeholders as multi-line block statements (`MARKED_JS;`, `HIGHLIGHT_JS;`, `JS;`) instead of contiguous `{{TOKEN}}` strings, so the export HTML cannot receive the vendored/app scripts through exact token replacement.
