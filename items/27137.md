## Summary

Keep open. Current `main` has partial shared DM/group-policy helpers, but it does not have the unified ingress authorization gate requested here, and multiple channels still implement their own per-channel/per-event authorization paths. This is core security architecture work, not a ClawHub-style skill/plugin request.

## Evidence

- **No unified ingress gate on main:** `src/security/` contains `dm-policy-shared.ts` but no `ingress-auth.ts` or equivalent unified ingress authorization module, and the public security runtime only re-exports `dm-policy-shared` helpers.
  - file: [src/plugin-sdk/security-runtime.ts:9](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/plugin-sdk/security-runtime.ts#L9)
