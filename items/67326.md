## Summary

Keep open. Current `main` still resolves only an API key for media image requests and does not forward request headers into `complete(...)`, so the underlying bug described by the PR is not implemented on `main`. The PR discussion also shows an unresolved regression risk around dropping caller-supplied `authStore`, so this is not a conservative close candidate.

## Evidence

- **Current image runtime still returns only apiKey + model:** `resolveImageRuntime` on `main` calls `getApiKeyForModel(...)`, stores the runtime API key, and returns only `{ apiKey, model }`. There is no `modelRegistry.getApiKeyAndHeaders(...)` lookup and no header value carried forward from model-registry auth.

  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
