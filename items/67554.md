## Summary

Keep open. Current `main` still hardcodes file-based memory behavior in workspace templates, while `memory-core` only contributes recall/citation prompt text. The decoupling proposed in PR #67554 is not implemented on `main`, and it matches the repo's existing active-memory-plugin architecture rather than an out-of-scope ClawHub idea.

## Evidence

- **Current main still hardcodes memory behavior in AGENTS template:** `docs/reference/templates/AGENTS.md` on `main` still tells agents to use `memory/YYYY-MM-DD.md` and `MEMORY.md` directly, including main-session-only loading and 'remember this' file edits. That is the exact template-level coupling the PR is trying to remove.
