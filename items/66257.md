## Summary

Keep open. Current `main` does not include the PR’s core behavior changes: `openclaw devices approve` still attempts `device.pair.approve` over the gateway before any local approval path, and the local-fallback guard still only excludes explicit `--url`, not explicit auth flags. The current tests also cover only the gateway-first fallback path, not the direct-local approve and unknown-local-request fallback cases described in the PR.
