## Summary

Keep open. Current `main` still routes local CLI gateway calls to loopback even when `gateway.bind` is `tailnet` or `custom`, and silent close formatting still lacks the proposed auth-required hint. The PR also has unresolved review concerns from the discussion, so there is not strong evidence to auto-close it conservatively.

## Evidence

- **Current main still hardcodes loopback for local gateway targets:** `buildGatewayConnectionDetailsWithResolvers()` always builds the local URL as `scheme://127.0.0.1:<port>` and labels it `local loopback`, independent of `gateway.bind`. That means the bind-mode host-resolution part of this PR is not already implemented on `main`.
