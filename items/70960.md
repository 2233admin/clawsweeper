## Summary

Keep this open. Current main still does not fully resolve the bundled report: the QClaw Electron Windows installer path is not present in this repository, Control UI agent-name precedence is still inconsistent, and the topbar still does not render current session token/cost/cache usage.

## Best Possible Solution

Keep this open or split it into focused follow-ups. Route the QClaw Windows Electron upgrade deletion report to the QClaw installer owner with the supplied install log and registry details. In this repo, make Control UI agent display surfaces consistently honor agents.list[].name before identity-derived names where that is the intended contract, and either restore the topbar current-session token/cost/cache display or document and implement an intentional replacement surface.
