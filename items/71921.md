## Summary

Current main has not implemented the requested Bedrock Claude vision fix. The Bedrock plugin now has known context-window metadata for Claude Opus 4.7, Opus 4.6, and Sonnet 4.6, but input capabilities are still derived from AWS `inputModalities` or an already discovered backing foundation model. If Bedrock reports these models as text-only, or an inference profile is resolved without a discovered base model, current main still emits `input: ["text"]`; the image paths then treat the model as non-vision and block image use.

## Best Possible Solution

Keep #71921 open. The best fix is provider-owned Bedrock capability metadata in `extensions/amazon-bedrock/discovery.ts`: recognize the affected Claude Opus 4.7, Opus 4.6, and Sonnet 4.6 base IDs plus regional/global inference-profile forms as text+image capable, apply that override when mapping foundation models and inference profiles, and add focused discovery tests for the exact affected IDs when AWS returns stale text-only/missing modality data. Update Bedrock docs or changelog if the shipped behavior changes.
