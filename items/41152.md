## Summary

Keep open. Current main still contains the reported gateway async approval result formatter: it takes only the tail of the aggregated output and passes it through whitespace-collapsing normalization before sending the follow-up. The linked fix PR #41170 was closed unmerged, so the issue is not implemented on main or cleanly superseded.

## Best Possible Solution

Keep the issue open and land a focused fix in the async approval completion path. The best implementation is to send the full captured exec output with newlines preserved for approved gateway exec follow-ups, add an explicit notice only when the existing exec capture cap is hit, and add regression tests that exercise the real 400-character/default normalization behavior. The node async approval path should be audited at the same time because it uses the same tail-and-normalize pattern.
