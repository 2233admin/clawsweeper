## Summary

Keep PR #58101 open. Current main still implements the old edge-only `HEARTBEAT_OK` stripping behavior in heartbeat mode, the heartbeat delivery path still depends on that helper, and the docs/tests still reflect that mid-text tokens are not treated specially. The PR’s ancillary MiniMax/transcript-policy work appears partly superseded by current provider replay hooks, but the central heartbeat bug is not implemented on main.

## Best Possible Solution

Keep the item open, but ask for a focused update: rebase onto current main, keep or split out the stale MiniMax/transcript-policy changes, change `stripHeartbeatToken` so heartbeat mode removes all `HEARTBEAT_OK` occurrences after markup normalization while preserving message-mode behavior, add regression tests for mid-text, long-remainder, edge-plus-middle, repeated, and HTML/Markdown-wrapped tokens, then update the heartbeat response-contract docs to match the new behavior.
