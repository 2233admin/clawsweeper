## Summary

Current main still implements only the config-level `plugins.entries.<id>.hooks.allowConversationAccess` path. The requested manifest-owned `permissions.conversationAccess` contract is not modeled, normalized, persisted into manifest records, documented, or used by hook enforcement, so this remains valid core plugin-manifest/security-contract work.

## Best Possible Solution

Add an explicit additive manifest-level permission contract for conversation-access hooks, likely `permissions.conversationAccess`, normalize it in `src/plugins/manifest.ts`, carry it through `src/plugins/manifest-registry.ts`, merge it with `plugins.entries.<id>.hooks.allowConversationAccess` in the loader/registry policy path, and document/test precedence. Keep the existing config flag as the operator override and backward-compatible path.
