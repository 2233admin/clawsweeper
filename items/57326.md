## Summary

Keep open. Current main has the broader CLI backend ownership and `isCliProvider()` fixes, but the narrowed issue remains valid: queued followups, `models status --probe`, and LLM slug generation still call the embedded runner directly instead of branching through `runCliAgent()` for CLI-backed providers. The related PR #57327 targeted exactly this remaining cleanup but was closed unmerged per the provided GitHub context.

## Best Possible Solution

Keep #57326 open and implement the narrowed helper-path cleanup: add a shared helper or equivalent branch so queued followups, auth probes, and LLM slug generation use the same `isCliProvider`/`runCliAgent` dispatch contract as the main agent paths, preserving active runtime config, auth profile/session binding where applicable, and adding focused regression tests for those three surfaces.
