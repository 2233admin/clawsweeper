## Summary

Keep this open. Current main still routes several reported helper paths through embedded/API execution instead of the CLI dispatch path, and cron still checks CLI-ness before resolving canonical providers such as anthropic to configured CLI runtimes. Open same-author PR #72499 remains an unmerged implementation candidate with closing syntax for this issue, so the maintainer path is to review/land or close that PR rather than auto-close this issue now.

## Best Possible Solution

Keep this issue open and review #72499, or an equivalent scoped patch, that applies the existing resolveCliRuntimeExecutionProvider(...) ?? provider plus isCliProvider(...) dispatch pattern to queued followups, models status --probe, LLM slug generation, and cron. The fix should preserve runtime config, auth profile, session binding, routing/cancellation context, and add focused regression coverage for those paths before the issue is closed.
