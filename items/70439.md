## Summary

Keep open. Current main partially addresses the old Signal audio metadata failure by threading MediaPath/MediaType/MediaPaths/MediaTypes, so the shared tools.media.audio pass can classify Signal audio. The requested Signal audio preflight/config hook is still not implemented: Signal skips mention-gated group messages before attachment download/transcription, has no Signal audio preflight config surface, and does not call the shared transcribeFirstAudio helper that Telegram, WhatsApp, and Feishu use. The existing maintainer comment on #70439 explicitly reached the same conclusion and kept the issue open.

## Best Possible Solution

Keep #70439 open and implement a shared channel audio-preflight integration for Signal using the existing transcribeFirstAudio/tools.media.audio path. The fix should run after access checks but before mention gating and agent dispatch, set Transcript/MediaTranscribedIndexes when it succeeds, preserve the shared media-understanding reuse behavior, and document Signal-specific template variables and conversion requirements if a configurable hook is added.
