## Summary

Keep open. Current main has the related Signal metadata/MIME fallback fix, so Signal voice attachments can enter the shared media-understanding path, but it still does not implement the requested Signal/shared audio preflight hook that runs before Signal mention gating or substitutes a transcript at the Signal channel layer. The existing maintainer comment on the issue also explicitly says this was only partially addressed and should remain open.

## Best Possible Solution

Keep the issue open and implement a shared channel audio-preflight helper plus a Signal integration that runs before Signal mention gating and before agent dispatch, reusing the existing plugin-sdk/media-runtime transcribeFirstAudio path and marking transcribed media to avoid duplicate STT. Prefer that over a Signal-only shell-command path unless maintainers explicitly want a configurable external converter; update Signal docs/config tests and add Signal DM plus mention-gated group coverage.
