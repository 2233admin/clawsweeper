## Summary

Current `main` has explicit Telegram forum-topic routing/session support in code, docs, and tests, including the exact `groupPolicy: "open"` + `groups: {"*": {requireMention:false}}` shape. But I did not get high-confidence proof that issue #51119's live long-poll silent-drop repro is fully gone, and one reported symptom (`channels status --probe` warninging about privacy mode) still appears to be present in current code. Conservative cleanup should keep this open.
