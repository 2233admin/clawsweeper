## Summary

Keep open. Current `main` and the latest shipped release still rely solely on persisted `lastObservedSuspiciousSignature` while `writeConfigHealthState*` silently swallows failures, so the reported failure mode is not clearly implemented away. At the same time, this PR's in-process `Set` approach has unresolved correctness concerns already raised in review: it can suppress later distinct incidents after a healthy interval and can suppress retries after snapshot persistence itself fails. That leaves a plausible real bug on `main`, but not a strong basis to close this PR as fixed or obsolete.
