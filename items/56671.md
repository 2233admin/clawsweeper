## Summary

Keep open. Current `main` still has the reported failure mode available in code: config-snapshot dedup relies on `lastObservedSuspiciousSignature` persisted to `config-health.json`, while both health-state writers still swallow write failures. In addition, current `main` now has parallel recovery logic in `src/config/io.observe-recovery.ts`, so this PR's single-file change would not cover all active read paths.
