## Summary

Keep open. Current main still emits the relevant `cron: timer armed` debug payload with numeric `nextAt` only, and the cron event/status/store types still expose millisecond fields without a readable companion. Related PRs in the issue timeline proposed the right shape, but the provided context shows them closed unmerged or still open, so they do not make the issue obsolete or shipped.

## Best Possible Solution

Implement an additive readable ISO companion on the cron timer-arm log, preserving numeric `nextAt` for compatibility. If maintainers want broader coverage, extend the same helper to retry/backoff logs and cron event/status payloads with tests around the logger payload and contracts. The existing open PR #58908 can be used as prior art, but it should be updated or replaced against current main before closing #58574.
