## Summary

Keep #39722 open. Current main still has the reported TUI `/sessions` title bug: the TUI prefers `derivedTitle` over `displayName`, gateway session rows compute an outer `displayName` but do not pass it into title derivation, and transcript title extraction still does not strip `Sender (untrusted metadata)` blocks. Related PR #39790 is directly relevant, but the provided GitHub context shows it is still open and unmerged.

## Best Possible Solution

Keep #39722 open until PR #39790 or an equivalent maintainer-reviewed fix lands. The fix should pass the computed outer session `displayName` into derived-title construction, strip inbound metadata before first-user-message and last-message-preview extraction, preserve subject/message/session-id fallbacks, and add regression tests for external display-name precedence plus metadata stripping in session-title extraction.
