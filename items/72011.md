## Summary

Keep PR #72011 open. Current main still lacks the PR's central behavior: the Telegram partial-preview path forwards `parsedFull.text` to `onPartialReply` without a streaming-aware bare `{"tool_calls":[...]}` sanitizer, and the shared downgraded-tool sanitizer on main only handles marker-style text such as `[Tool Call ...]`, not the bare OpenAI-style JSON shape from #70797. #71954 was closed unmerged, while #72011 is a focused active successor with relevant regression coverage, though it still needs maintainer review for the bot-raised sanitizer edge cases.

## Best Possible Solution

Keep #72011 open for maintainer review. The best path is to land this PR or a focused successor that adds a streaming-aware bare `tool_calls` JSON sanitizer before `onPartialReply`, keeps shared final-delivery sanitizer coverage for the #70797 leak shape, and resolves the outstanding sanitizer edge cases without losing ordinary user JSON.
