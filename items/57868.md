## Close Comment

Closing this as implemented on `main` after a Codex review.

- Current `main` has a shared assistant-text sanitizer that strips leaked `<tool_call>`, `<tool_result>`, `<function_call>`, `<function_calls>`, and truncated tool-call blocks before user-visible delivery (`src/shared/text/assistant-visible-text.ts:199`, `src/shared/text/assistant-visible-text.ts:537`).
- Final assistant reply extraction routes through that sanitizer, and fallback final reply text is built from `extractAssistantVisibleText(...)` before channel delivery, so Discord no longer sends the raw XML scaffolding (`src/agents/pi-embedded-utils.ts:33`, `src/agents/pi-embedded-runner/run/payloads.ts:233`).
- Regression tests cover the exact leak pattern and related edge cases (`src/shared/text/assistant-visible-text.test.ts:109`, `src/agents/pi-embedded-utils.test.ts:441`).
- This was shipped in `v2026.4.5`; the changelog explicitly notes that outbound user-visible assistant text now strips leaked tool-call XML across surfaces (`CHANGELOG.md:1311`).
