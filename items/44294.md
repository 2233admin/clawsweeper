## Summary

Keep open. Current main has a structured gateway `errorKind` field and the ACP translator preserves only `refusal` as ACP `stopReason: "refusal"`; timeout/rate-limit/context/unknown errors still resolve to `end_turn`, and tests explicitly lock that behavior. The related #55919 discussion addressed a narrower Anthropic rate-limit copy bug, not this broader ACP client-visible error classification gap.

## Evidence

- **Translator still collapses non-refusal errors:** For `state === "error"`, the ACP translator reads `payload.errorKind` but maps only `refusal` to `refusal`; every other error kind maps to `end_turn`.
