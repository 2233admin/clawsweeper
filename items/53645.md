## Summary

Current main still appears to drop Feishu message-tool `replyTo` for ordinary `send` actions handled by the Feishu plugin. Core resolves `replyTo`, and the lower Feishu outbound adapter can forward it, but the plugin action handler only maps a reply target for `thread-reply`, so text, card, and media sends can still reach the Feishu send layer without the quoted-reply target.

## Best Possible Solution

Keep this issue open and fix it in the Feishu plugin owner module. `extensions/feishu/src/channel.ts` should resolve `replyTo`/`reply_to` for ordinary `action: "send"`, pass it as `replyToMessageId` or media `replyToId` for text, presentation/card, and media sends, and preserve `thread-reply` as the explicit `reply_in_thread` path requiring `messageId`. Add focused Feishu channel action tests for `send` with explicit `replyTo` across text, presentation/card, and media sends, while keeping the existing media-only-without-card behavior covered.
