## Summary

Keep open. Current main still drops Feishu message-tool `replyTo` for ordinary `send` actions: core resolves and applies the reply target, but the Feishu plugin action handler only forwards a reply target for `thread-reply`, so text, card, and media sends handled through the plugin can still reach the Feishu send layer without the quoted-reply target.

## Best Possible Solution

Fix this in the Feishu plugin owner module. `extensions/feishu/src/channel.ts` should resolve `replyTo`/`reply_to` for ordinary `action: "send"`, pass it as `replyToMessageId` or media `replyToId` for text, presentation/card, and media sends, and keep `thread-reply` as the explicit `reply_in_thread` path requiring `messageId`. Add focused Feishu channel action tests for `send` with explicit `replyTo` across text, card/presentation, and media sends, plus a guard that media-only sends do not require a card payload.
