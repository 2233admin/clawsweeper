## Summary

Keep open. Current main still lacks a working CLI path for `wiki bridge import` to see memory public artifacts, and the PR addresses a real user-visible bug from #70181. The exact patch likely needs maintainer review/rebase because current CLI command loading is scoped and `activate:false`, but this item should not be auto-closed.

## Best Possible Solution

Keep the PR open and ask for a current-main update with a regression test that exercises the real `openclaw wiki bridge import` CLI loading path. The fix should give bridge import access to the active memory plugin's `publicArtifacts` provider without relying on restored global state or an undocumented cli-metadata side effect.
