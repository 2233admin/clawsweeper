## Summary

Current `main` already handles stale skills snapshots after restart, and the fix is also present in release `v2026.4.22`.

## Evidence

## Close Comment

Closing as implemented on `main` after Codex review. The restart-stale snapshot path described here is already handled in current code, and the same fix is present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). In particular, `src/agents/skills/refresh-state.ts:71` now treats a cold-start `nextVersion === 0` as stale when an existing session snapshot has a prior version, and both runtime paths use that check in `src/auto-reply/reply/session-updates.ts:148` and `src/agents/agent-command.ts:578`. The release notes also document the related `#67401` skills-snapshot invalidation fix in `CHANGELOG.md:593`. The secondary ideas in this issue about deduplicating per-session snapshots or adding manual refresh controls are separate follow-up work, but the reported bug itself is no longer current on supported builds.
