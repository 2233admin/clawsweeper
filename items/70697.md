## Summary

Keep open. Current `main` still handles `openclaw gateway restart` for an unmanaged Linux gateway by signaling the live gateway PID with `SIGUSR1`, and that path is still the tested behavior. Docker/container support exists as separate docs/helpers (`docker compose ...`, `clawdock-restart`), but the core restart command is not container-aware and does not switch to a Docker-native restart path.

## Evidence

- **Unmanaged restart still sends SIGUSR1 directly:** `runDaemonRestart` falls back to `restartGatewayWithoutServiceManager(...)` when the service is not loaded; that helper resolves the gateway listener PID and calls `signalVerifiedGatewayPidSync(pid, "SIGUSR1")` before waiting for health.

- **Current tests lock in the unmanaged SIGUSR1 path:** `lifecycle.test.ts` explicitly asserts that restarting a single unmanaged gateway process calls `signalVerifiedGatewayPidSync(4200, "SIGUSR1")` and does not use `service.restart()`.
  - file: [src/cli/daemon-cli/lifecycle.test.ts:358](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/cli/daemon-cli/lifecycle.test.ts#L358)
