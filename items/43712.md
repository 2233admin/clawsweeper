## Summary

Keep open. Current main has improved update flows and the Telegram helper mismatch shown in the report appears fixed, but the central reported problem remains a real, repo-actionable source-update edge case: a mutable git checkout under a node_modules-style path can still pass the updater clean check while stale ignored A2UI build output remains in place, and the build step still runs in-place without cleaning that vendor output first.

## Evidence

- **Git updater still builds the source checkout in place:** For git installs, the updater only checks `git status --porcelain -- :!dist/control-ui/`, then runs dependency install and `pnpm build` in the same checkout. There is no cleanup step for ignored vendor build output before the build.
  - file: [src/infra/update-runner.ts:643](https://github.com/openclaw/openclaw/blob/1531123d3547f9ff8ee8094590fc773305f018f2/src/infra/update-runner.ts#L643)

That tsconfig emits to `dist`, matching the TS5055 collision area from the issue, but the bundler does not remove or validate that directory first.
  - file: [scripts/bundle-a2ui.mjs:195](https://github.com/openclaw/openclaw/blob/1531123d3547f9ff8ee8094590fc773305f018f2/scripts/bundle-a2ui.mjs#L195)
