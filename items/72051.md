## Summary

Keep PR #72051 open. Current main at 43a003b8 still lacks the two substantive changes: mirrored terminal TaskFlow timestamps can still produce `endedAt < updatedAt`, and packaged builds still have no stable root `task-registry-control.runtime` entry even though task cancellation tries to load that file. The related #72320 report supports the packaging failure as a real shipped issue, but the PR should address the existing review comment about preserving historical `endedAt` when no newer event timestamp exists.

## Best Possible Solution

Keep the PR open and request a narrow revision: preserve historical terminal `endedAt` when no newer event timestamp exists, clamp only the mirrored TaskFlow case where `lastEventAt` or another event timestamp is newer than `endedAt`, keep the stable `task-registry-control.runtime` dist entry, and retain regression coverage for both the timestamp invariant and packaged build entry.
