## Summary

Keep #72051 open. Current main still lacks the PR's two core fixes: mirrored terminal TaskFlows can produce `endedAt < updatedAt`, and the stable root `task-registry-control.runtime` dist entry is not present. The PR is focused and useful, but it has an unresolved review concern about rewriting historical `endedAt` values, so it needs revision rather than maintainer cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a small revision: preserve historical `endedAt` when `lastEventAt` is absent, still ensure mirrored terminal TaskFlows satisfy `endedAt >= updatedAt` when a later event timestamp exists, keep the stable `task-registry-control.runtime` dist entry, and retain/add regression coverage for both timestamp cases plus the build entry.
