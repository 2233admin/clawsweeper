## Summary

Keep PR #72051 open. Current main still lacks the PR's two substantive fixes: mirrored TaskFlow terminal records can write `endedAt` earlier than `updatedAt`, and `task-registry-control.runtime` is still not emitted as a stable root dist entry. The PR is a focused, recent bug-fix/packaging hardening change with an actionable review comment, so cleanup-closing it would drop useful remaining work.

## Best Possible Solution

Keep the PR open and request a small revision: preserve historical terminal `endedAt` when no newer event timestamp exists, clamp only cases that would otherwise violate the TaskFlow audit invariant, keep the stable `task-registry-control.runtime` dist entry, and retain/add regression tests for both the `lastEventAt > endedAt` mirror case and the legacy/no-`lastEventAt` retention case.
