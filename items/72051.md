## Summary

Keep #72051 open. Current main does not already contain the two substantive changes: mirrored TaskFlow terminal timestamps can still produce `endedAt < updatedAt`, and the task-registry control runtime is not emitted as a stable root dist entry. The PR also has a substantive review concern about preserving historical `endedAt` when `lastEventAt` is absent, so it should be revised or reviewed rather than closed as maintainer cleanup.

## Best Possible Solution

Keep the PR open and ask for a small revision: preserve historical terminal `endedAt` when no newer event timestamp exists, clamp only the cases that would otherwise violate the TaskFlow audit invariant, keep the stable `task-registry-control.runtime` dist entry, and add regression tests for both the `lastEventAt > endedAt` mirror case and the legacy/no-`lastEventAt` retention case.
