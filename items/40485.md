## Summary

Keep this PR open. Current main still routes Telegram native command replies through the account-level replyToMode and still injects the triggering slash-command message as replyToId, so direct-chat /new and /reset can still inherit threaded reply behavior. The PR is a plausible narrow fix, but it targets the removed legacy src/telegram layout and needs to be ported to the current extensions/telegram plugin with /reset test coverage added. Security review found no supply-chain-sensitive changes in the provided diff.

## Best Possible Solution

Keep this PR open and port the narrow behavior to extensions/telegram/src/bot-native-commands.ts. Add an override or equivalent delivery option that sets replyToMode to "off" only for direct-chat native new/reset commands, preserve group/topic and other-command threading behavior, and add regression coverage for both /new and /reset when the configured Telegram replyToMode is "first".
