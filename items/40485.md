## Summary

Keep PR #40485 open. The linked Telegram bug is closed, but current main still does not contain the PR’s narrow direct-chat `/new`/`/reset` reply-threading override. Native Telegram command delivery still inherits the account-level `replyToMode`, injects the triggering command message id as `replyToId`, and delivery only strips that id when `replyToMode` is already `off`. The PR remains a plausible focused fix, but it needs porting from the old `src/telegram/*` paths to the current `extensions/telegram/*` plugin layout and should add `/reset` regression coverage.

## Best Possible Solution

Keep the PR open and port its narrow behavior to `extensions/telegram/src/bot-native-commands.ts`: allow native command delivery to override `replyToMode`, set it to `"off"` only for direct-chat native `/new` and `/reset`, preserve existing group/topic and other-command threading behavior, and add parameterized regression tests for both `/new` and `/reset` with global Telegram `replyToMode: "first"`.
