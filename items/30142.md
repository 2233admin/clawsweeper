## Summary

Keep this PR open. Current main only has current sendPayload implementations for Discord and Slack; Google Chat, Mattermost, MS Teams, and Synology Chat still lack that adapter entry point, while delivery only takes the unified payload path when sendPayload exists. The PR is stale and should not merge as-is, but its remaining adapter work is still real.

## Best Possible Solution

Keep this PR open for maintainer action, but require a rebase and narrowing before merge. Drop the already-shipped Discord and Slack pieces, implement sendPayload for Google Chat, Mattermost, MS Teams, and Synology Chat using the current shared payload helpers and contract-test patterns, and cover text-only, media-only, mixed media, channelData-only, chunking, reply-to fanout, and delivery fallback behavior.
