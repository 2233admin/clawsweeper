## Summary

Keep this PR open. Current main still has the cache-stability problem the PR targets: channel/session-sensitive Messaging and Reactions text are assembled before stable Project Context and before the internal cache boundary. The PR is still relevant, but it needs a refresh for current main's newer SYSTEM_PROMPT_CACHE_BOUNDARY layout and current test file structure.

## Best Possible Solution

Keep this PR or #40256 open, then refresh the implementation against current main. Preserve the existing cache boundary, keep stable Project Context in the reusable prefix, move or split runtime/channel/session-sensitive text such as inline-button Messaging guidance, Reactions, and any relevant webchat/runtime guidance so volatile parts land below the boundary, and add a current-layout regression test proving the stable context and boundary precede those volatile sections.
