## Summary

Keep this PR open. Current main has broader prompt-cache boundary work, but it still places channel/capability-sensitive Messaging and channel-specific Reactions before stable Project Context and before the internal cache boundary. The PR’s central invariant is not implemented on main, and the linked #40256 provides concrete local-model cache/latency evidence.

## Best Possible Solution

Refresh the PR or implement the same fix directly on main: keep large stable Project Context before volatile channel/session text, preserve the existing SYSTEM_PROMPT_CACHE_BOUNDARY, move or split runtime-channel Messaging/inline-button guidance and Reactions into the volatile suffix, and add a regression test in src/agents/system-prompt.test.ts covering Project Context/cache-boundary ordering for Messaging, Reactions, Group Chat/Subagent context, and runtime-specific channel guidance.
