## Summary

Keep PR #50357 open. Current main still has the Feishu skill docs gap the PR targets, but the submitted docs change is not safe to close or merge as-is: it says `full_access` instead of the implemented `edit` grant, does not qualify requester sharing as Feishu-channel-context only, and its workflow example uses `result.document_id` even though tool payloads are under `details`.

## Best Possible Solution

Keep the PR open and ask for a focused docs revision: document that `create` currently creates the document shell only, remove `owner_open_id`, use `grant_to_requester`, qualify requester sharing as available only from trusted Feishu-channel sender context, describe the granted permission as `edit`, and read the created token from `result.details.document_id` in the create-to-write workflow. Coordinate wording with #48373/#48377 because maintainers still need to choose whether create-with-content should be implemented instead of only documented as unsupported.
