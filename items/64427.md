## Summary

Current main still does not implement Slack 1:1 DM history hydration for `channels.slack.dmHistoryLimit`. Slack accepts and documents the config, but the Slack monitor only carries `historyLimit`, and the message preparation path only builds pending history / `InboundHistory` for `isRoomish` conversations, which excludes `channel_type: "im"` DMs. The generic runner-side `dmHistoryLimit` code only limits already-loaded session transcripts; it does not fetch Slack `conversations.history` into a new or reset DM session.

## Best Possible Solution

Keep #64427 open. The best fix belongs in the Slack plugin: resolve an effective DM history limit for `channel_type: "im"`, fetch recent DM messages with Slack `conversations.history` when a new/reset DM session is initialized, seed the inbound context/`InboundHistory` with both user and bot messages, and add focused Slack monitor tests proving `dmHistoryLimit` and per-DM overrides work for 1:1 DMs without changing room/group history behavior.
