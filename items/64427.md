## Summary

Current main still does not implement Slack top-level DM history hydration from `channels.slack.dmHistoryLimit`. Slack config, schema, and docs expose the setting, but the Slack monitor only derives/carries `historyLimit`, and inbound history assembly remains gated on `isRoomish`, which excludes Slack 1:1 DMs (`channel_type: "im"`). Existing Slack `conversations.history` calls serve other surfaces and are not a DM session bootstrap path.

## Best Possible Solution

Keep this issue open and implement the missing behavior in `extensions/slack`: resolve `channels.slack.dms.<user>.historyLimit` before `channels.slack.dmHistoryLimit` for direct messages, pass the effective DM limit through `SlackMonitorContext`, hydrate a fresh top-level Slack DM session with recent `conversations.history` messages, format/inject them consistently with existing inbound history context, and add focused tests for `channel_type: "im"`, per-DM override precedence, current-message exclusion, and reset/fresh-session behavior. Keep #7359 as the broader cross-session visibility discussion; it should not supersede this targeted Slack DM history gap.
