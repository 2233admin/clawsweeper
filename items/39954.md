## Summary

Keep PR #39954 open. Current main still documents `tools.profile` mechanically, but it does not include the PR’s practical warning that `messaging` is intentionally narrow or direct guidance to use `full` for broader command/control. The underlying behavior is real in current code and tests, and the related open PRs cover adjacent help/status/troubleshooting surfaces rather than superseding this docs/tools page change.

## Best Possible Solution

Keep the PR open and port the useful wording into the current `docs/tools/index.md` profile section. The best maintainer path is a compact note explaining that `tools.profile: "messaging"` is intentionally narrow, that `tools.profile: "full"` is the unrestricted baseline for broader command/control, and that the wording should be reconciled with the current profile table plus adjacent open help/status/troubleshooting PRs.
