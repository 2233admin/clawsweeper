## Summary

Keep #72359 open. Current main fixed the hard-timeout sibling from #72347, but it has not implemented the requested opt-in single-shot Active Memory path. The active-memory config/schema/docs expose no `mode` or `single-shot` value, the manifest rejects unknown config keys, and the runtime still routes recall through `runEmbeddedPiAgent` with `memory_search`/`memory_get`. The #72347 close comment explicitly left #72359 as the canonical follow-up for the no-agent-loop latency architecture.

## Best Possible Solution

Add an opt-in `plugins.entries.active-memory.config.mode`, defaulting to the current embedded-subagent behavior, with a new single-shot path in `extensions/active-memory`. It should reuse existing query derivation, run one bounded memory/QMD search, perform one model completion, parse `NONE` or a bounded summary, preserve timeout/cache/status/prompt-prefix behavior, and update the manifest, docs, and focused tests. If direct memory search or one-shot completion is not safely plugin-facing, add a narrow runtime/SDK seam first instead of importing core internals from the plugin.
