## Risks / Open Questions

- The PR touches gateway auth credential selection and status probing. The latest diff preserves probing overridden RPC URLs, but auto-resolved daemon credentials may still be sent to a user-supplied target unless maintainers explicitly accept that policy or require explicit credentials for overrides/remote targets.
- `failureReason` may include SecretRef identifiers or config paths from auth resolution. The PR sanitizes selected terminal renderers, but structured JSON and other human sinks still need review so sensitive ref labels and control characters do not leak or render unsafely.
- The intended behavior needs a narrow boundary: fail fast only for local loopback probes where auth is explicitly required but unresolved, while preserving unset/none/trusted-proxy auth and explicit override behavior.
