## Summary

Keep open. Current `main` still has the mismatch the issue describes: `memory status` counts every `.jsonl` file in the sessions directory, while the session indexing paths already treat checkpoint snapshots as non-indexed artifacts. The report matches current code, and the existing tests/helpers reinforce that checkpoint files are intentionally excluded from normal session indexing behavior.

## Evidence

## Risks / Open Questions

- This looks like a status-reporting bug, not an indexing bug; closing it would hide a real user-facing inconsistency.
- There does not appear to be a targeted regression test for checkpoint exclusion in `memory status`, so the mismatch could persist unnoticed until someone adds coverage around `scanSessionFiles()`.
- The issue is only a few days old and includes a concrete root-cause analysis plus a commenter agreeing with the design intent, so `stale_insufficient_info` is not applicable.
