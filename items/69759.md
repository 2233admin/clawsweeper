## Summary

Keep open. Current main still has the reported mismatch: `openclaw memory status` counts every top-level `.jsonl` session file for scan totals, while the memory indexing path uses the session filename contract that excludes `.checkpoint.<uuid>.jsonl` snapshots. The status renderer then compares indexed DB rows against the broader scan count, so checkpoint files can still make the display show fewer indexed files than scanned files.

## Best Possible Solution

Leave #69759 open. The fix should make `extensions/memory-core/src/cli.runtime.ts` `scanSessionFiles` use the same transcript filename contract as indexing, preferably by reusing `listSessionFilesForAgent` or the shared usage-counted predicate through an allowed plugin/SDK surface. Add a CLI status regression test with one normal session transcript plus one `.checkpoint.<uuid>.jsonl` snapshot proving total and per-source file counts exclude checkpoint snapshots.
