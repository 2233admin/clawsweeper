## Summary

Keep open. Current main does not contain #72294's closed-streaming-card guard: it suppresses only exact duplicate Feishu final text, clears the streaming session on idle, and will still start a new streaming card for a distinct late final in card mode. The linked #71977 has a concrete Feishu reproduction, and this PR adds targeted regression coverage, but the Aisle security comment raises a real product question about silently suppressing distinct final text that needs maintainer review rather than automated closure.

## Best Possible Solution

Keep #72294 open for maintainer review. If the product decision is to suppress Feishu card-mode late finals after a successful streaming-card idle close, implement that as Feishu-local per-reply state reset on `onReplyStart`, mark closure only on successful idle close, keep media delivery, and retain regression tests for distinct late finals plus error close recovery. If distinct finals can represent corrected output, prefer updating the existing card or sending a non-streaming fallback instead of silently dropping text.

- The current branch appears scoped to Feishu dispatcher code and tests, but it should remain open until the final-text suppression semantics, error-close behavior, and media passthrough coverage are reviewed together.
