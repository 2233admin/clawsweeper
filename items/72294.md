## Summary

Keep open. Current main still lacks #72294's closed-streaming-card guard: Feishu only suppresses exact duplicate final text, clears the streaming session on idle, and can start a new card for a distinct late final in card mode. The linked #71977 reproduction is concrete and this PR is a focused Feishu fix with regression coverage, but the Aisle/security discussion raises a real product question about silently suppressing distinct final text, so this needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep #72294 open for maintainer review. If the desired Feishu behavior is one closed streaming card per reply in card mode, land a Feishu-local guard that resets on `onReplyStart`, marks only successful idle close, keeps media delivery, and preserves the new error-recovery test. If distinct finals can be authoritative corrections, revise the approach to update the existing card or send a non-streaming fallback instead of dropping text.
