## Summary

Current main already provides a plugin-facing, channel-agnostic rich-message contract (`presentation`) plus native Discord/Slack/Telegram/Teams renderers, and the repo vision/docs explicitly push optional richer card translation work into plugins rather than adding a new core adaptive-card marker protocol.
