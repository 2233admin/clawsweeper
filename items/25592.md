## Summary

Keep open. Current `main` hardens one subset of the bug by suppressing `commentary`-phase assistant text, but it still delivers non-commentary assistant text around tool calls, and there is no explicit-send / tool-only reply mode on `main`. The broader request in #25592 is therefore not implemented.

## Evidence

- **Commentary suppression is narrow:** Visible-output suppression only checks whether the assistant message phase resolves to `commentary`.
  - file: [src/agents/pi-embedded-subscribe.handlers.messages.ts:40](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/agents/pi-embedded-subscribe.handlers.messages.ts#L40)

  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Non-commentary text still becomes user-visible output:** On `message_end`, once the message is not suppressed as `commentary`, the handler builds assistant stream data, finalizes `assistantTexts`, and emits block replies for any visible text. There is no guard here for "this turn had tool calls".
  - file: [src/agents/pi-embedded-subscribe.handlers.messages.ts:618](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/agents/pi-embedded-subscribe.handlers.messages.ts#L618)

- **Assistant text buffering is generic, not tool-call-aware:** `pushAssistantText()` appends any non-empty, non-silent assistant text, and `finalizeAssistantTexts()` pushes final text for non-streaming models. No tool-call suppression state is consulted.
