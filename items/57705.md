## Summary

Keep open. Current `main` documents `agents.defaults.subagents.maxConcurrent` as a queue-lane limit, but there is no `sessions_capacity` tool, no `remaining_subagents` field in `session_status`, and `sessions_spawn` still returns `accepted` without exposing remaining capacity.

## Evidence

- **`session_status` does not expose capacity metadata:** The tool returns only `ok`, `sessionKey`, `changedModel`, and `statusText` in `details`; there is no concurrency/capacity field such as `remaining_subagents`.
  - file: [src/agents/tools/session-status-tool.ts:603](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/tools/session-status-tool.ts#L603)
  - command: `nl -ba src/agents/tools/session-status-tool.ts | sed -n '420,620p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **`sessions_spawn` does not report queue budget:** The subagent spawn tool delegates to `spawnSubagentDirect(...)` and returns that result directly. There is no structured `capacity_exceeded` / `remaining` / `retry_after` contract in this path.
  - file: [src/agents/tools/sessions-spawn-tool.ts:343](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/tools/sessions-spawn-tool.ts#L343)

- **Current docs describe `maxConcurrent` only as queue concurrency:** The subagents docs say the subagent lane concurrency is `agents.defaults.subagents.maxConcurrent` and that `sessions_spawn` is always non-blocking and returns `accepted` immediately; they do not document any tool for checking remaining capacity.
  - file: [docs/tools/subagents.md:326](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/docs/tools/subagents.md#L326)
