## Summary

Keep open. Current `main` shows OpenClaw now resolves explicit channel login through a scoped plugin load rather than a broad registry preload, but the reported QR-fetch `AbortError` is still documented as external-plugin-owned behavior and there is no strong repo-local evidence that the underlying Weixin QR fetch failure is fixed on `main`. The issue is also recent and has corroborating follow-up reports, so conservative cleanup should not close it.
