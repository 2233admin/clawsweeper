## Summary

Current `main` still has the unguarded `WebSocketTaskBox.sendPing()` continuation, and that path is still used by the gateway keepalive loop, so this is not implemented on `main` and should not be auto-closed. The PR is directionally useful, but it likely needs maintainer cleanup because it bundles a redundant Swift workflow addition and does not add a regression test for duplicate pong callbacks.
