## Summary

Keep open. Current main still does not apply an `oc_` OpenChat-to-group correction in Feishu outbound session routing, and the inbound classifier on main is still `chat_type`-only. The related #52342 context is useful, but it is open/unmerged and does not cover the outbound session-route path this issue calls out.

## Best Possible Solution

Keep the issue open and fix it in the Feishu plugin boundary: add a Feishu-owned helper that classifies `oc_` OpenChat IDs as group when Feishu reports `p2p`, use it in inbound handling, mention forwarding, and `resolveFeishuOutboundSessionRoute`, then add focused tests for raw `oc_` outbound targets, `chat:oc_` targets, and inbound `p2p`+`oc_` events. Consider a doctor/fix cleanup path for already-split direct/group session entries rather than silent startup migration.
