## Summary

Keep #43929 open. Current main still exposes `chat.history` as a current-session-only API: the protocol has no `scope` or segment metadata parameter/result, the handler reads messages from the current session entry only, and the transcript reader selects a single current/custom `.jsonl` candidate rather than stitching `.reset.*` archives. The linked #45003 context is related archive-recovery work, but it is broader restore tooling and does not supersede this specific Gateway logical-history API request.
