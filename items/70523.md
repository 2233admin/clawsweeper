## Summary

Keep open. Current `main` still writes inline light/REM dreaming blocks before request-scoped narrative fallback is known, so the raw-dump suppression described in PR #70523 is not implemented on `main`. The existing tests cover fallback diary writes to `DREAMS.md`, but they do not verify that fallback runs keep raw candidate/reflection metadata out of `memory/YYYY-MM-DD.md`. The PR discussion also contains substantive review findings and follow-up commits, so this still needs normal maintainer review rather than auto-close cleanup.

## Evidence

That means current `main` cannot suppress the inline daily block based on request-scoped fallback outcome.
