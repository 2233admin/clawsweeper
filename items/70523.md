## Summary

Keep open. Current main does not implement this PR's fallback-aware inline suppression: light/REM still write raw phase body lines into the managed daily note block before narrative generation, and the narrative helper returns no fallback status that callers could use to remove or suppress those inline blocks. The related PRs are overlapping dream-diary work, but the provided context does not establish a merged canonical replacement for this specific inline daily-memory fallback fix.

## Best Possible Solution

Keep the PR open for maintainer review or consolidation. The best path is a focused memory-core patch that suppresses or removes raw light/REM inline daily blocks only after a successful request-scoped fallback, preserves normal inline output and note formatting, includes the added regression coverage, and drops unrelated docs churn. If maintainers choose a broader canonical dreaming pipeline PR, explicitly supersede #70523 from that item rather than auto-closing it now.
