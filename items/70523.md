## Summary

Keep #70523 open. Current main reduced the default/user-facing pollution by defaulting dreaming storage to separate reports, but it has not implemented this PR's fallback suppression for the still-supported explicit inline storage mode. The current inline writer always writes the raw body into memory/YYYY-MM-DD.md, light/REM phases write that body before narrative generation, and generateAndAppendDreamNarrative returns no fallback signal that callers could use to remove or skip raw inline blocks.

## Best Possible Solution

Keep the PR open as potentially useful, but ask for a clean, memory-core-only branch: remove unrelated docs churn, preserve the current phase-output-before-narrative ordering unless intentionally changed, keep fallback suppression limited to explicit inline mode, and verify the focused memory-core dreaming tests. If maintainers decide inline storage is legacy/internal and should not receive more behavior fixes, close it with that product decision rather than as already implemented.
