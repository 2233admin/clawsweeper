## Summary

Keep open. Current main has the earlier gateway-bindable/default registry reuse path, but it does not implement this PR's narrower fix for gateway-startup registries whose cache key includes core gateway method names. Helper/runtime loads that omit `coreGatewayHandlers` can still miss the active gateway-bindable registry because main neither records the active method names nor reconstructs a compatible synthetic handler set.

## Best Possible Solution

Keep #69690 open and either merge it or adapt the same approach on main: record the gateway method-name set for an active gateway-bindable registry, use it only to reconstruct compatible cache-key checks for callers that opted into gateway binding, reject mismatched method sets, and keep default-mode helper loads unbound. The regression tests should cover omitted handlers, mismatched handlers, and the default-mode safety boundary.
