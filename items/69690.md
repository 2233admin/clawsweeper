## Summary

Keep open. Current main has adjacent gateway-bindable registry reuse work, including the default-mode guard related to #61785, but it does not implement this PR's specific fix: a gateway startup registry loaded with coreGatewayHandlers still has a cache key containing the core gateway method names, while helper/runtime loads can request gateway binding without supplying those handlers, so the active registry remains cache-incompatible and a fresh runtime load can still occur.

## Best Possible Solution

Keep the PR open for maintainer review or adaptation. The best path is to merge or reimplement the PR's active core-gateway-method recording plus synthetic-handler compatibility check, with tests proving that a gateway-bindable startup registry loaded with coreGatewayHandlers is reused when a helper load omits those handlers, while mismatched method sets and default-mode capability widening remain rejected.
