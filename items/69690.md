## Summary

Keep open. Current main has partial gateway-bindable registry reuse, but it does not implement this PR's central fix: recording the gateway method set on the active registry and using it to match later gateway-bindable helper/runtime loads that omit `coreGatewayHandlers`. The PR still carries unique, relevant work, and the provided PR discussion shows the default-mode inheritance concern was addressed by later PR commits/tests rather than left unresolved.

## Best Possible Solution

Keep PR #69690 open for maintainer review or rebase. The product/code direction is to implement the PR's narrow registry-compatibility fix on main: persist the active gateway method names, use them only for gateway-bindable compatibility reconstruction, keep default-mode loads unbound, and land the regression tests covering both reuse and rejection cases.
