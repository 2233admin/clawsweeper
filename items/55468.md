## Summary

Keep open. Current `main` still drops `messageId` from the public `message_sent` plugin hook even though outbound delivery carries it in the canonical sent-message context, so the feature request is not implemented on `d3d9b5738f83d92787f57db9186ddcebda60af9a`.

## Evidence

- **Public hook event type lacks `messageId`:** `PluginHookMessageSentEvent` only defines `to`, `content`, `success`, and optional `error`.
  - file: [src/plugins/hook-message.types.ts:58](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/plugins/hook-message.types.ts#L58)

- **Mapper still omits `messageId`:** `toPluginMessageSentEvent()` returns `to`, `content`, `success`, and optional `error`, with no `messageId` field.
  - file: [src/hooks/message-hook-mappers.ts:316](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/hooks/message-hook-mappers.ts#L316)
