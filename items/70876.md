## Summary

Current `main` still omits MCP runtime tools from `/tools`, while the actual agent run path can expose those tools. This PR appears to address a real inventory mismatch, so there is no conservative close reason here.

## Evidence

- **`/tools` on `main` only renders the core inventory helper result:** `handleToolsCommand` resolves `resolveEffectiveToolInventory(...)` and immediately passes that result to `buildToolsMessage(...)`; there is no MCP runtime lookup or MCP tool materialization in the current handler.
