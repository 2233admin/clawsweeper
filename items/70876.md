## Summary

Keep this PR open. Current main still does not include configured MCP server tools in `/tools`, so the PR targets a real product bug, but the PR is not a conservative cleanup close candidate because the bug remains unimplemented on main and the current PR diff mixes the MCP inventory fix with unrelated plugin installer dependency-resolution changes.

## Best Possible Solution

Keep this PR open, but ask for a focused revision or supersede it with a narrow PR that only fixes `/tools` MCP inventory. The implementation should deliberately choose the MCP runtime lifecycle behavior, materialize names the same way as the run path, apply final bundled-tool policy, render a clear MCP servers group, and add compact plus verbose regression tests. The unrelated plugin installer peer-dependency change should move to a separate PR with its own security and dependency-resolution review.
