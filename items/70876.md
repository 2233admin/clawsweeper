## Summary

Keep #70876 open. Current main still has the `/tools` inventory gap: the command renders only the core effective-tool inventory, while the embedded run path separately materializes configured MCP server tools and merges them into runtime tools. The PR is not obsolete, but it should be narrowed and cleaned up before merge because the provided PR context still includes unrelated `install-package-dir` changes and review history around MCP runtime side effects, policy filtering, and collision naming.

## Best Possible Solution

Keep #70876 open or replace it with a focused maintainer patch. The best fix should add MCP server tools to `/tools` using the correct session ID, preserve `reservedToolNames`, filter MCP tools through `applyFinalEffectiveToolPolicy()`, add focused `/tools` inventory tests for an MCP group, and split out the unrelated `install-package-dir` changes.
