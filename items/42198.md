## Summary

Keep open. Current `main` has a safer `openclaw update` flow and doctor-based service repair, but I did not find strong evidence that the reported path (`sudo npm install -g openclaw@latest` followed by `openclaw gateway restart`) is fixed on Linux. The plain systemd restart path still just calls `systemctl restart`, while the updater adds an explicit `gateway install --force` refresh step that manual upgrades do not get.
