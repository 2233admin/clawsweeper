## Summary

Current `main` has rate-limit fallback/cooldown handling, but it does not expose the requested user-configurable delay between consecutive model/API requests. The closest knobs are overloaded retry backoff and rate-limit profile rotation count, so this issue is not implemented on `main` and should stay open.

## Evidence

- **Auth cooldown config has no generic request-delay field:** `auth.cooldowns` exposes billing/auth backoff controls plus `overloadedProfileRotations`, `overloadedBackoffMs`, and `rateLimitedProfileRotations`; there is no configurable delay for rate-limited requests or a generic `api_request_delay`-style setting.
  - file: [src/config/types.auth.ts:17](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/config/types.auth.ts#L17)

- **Schema validation matches the missing knob:** The Zod schema accepts the same cooldown fields and likewise has no rate-limit delay/backoff option beyond `overloadedBackoffMs`.
  - file: [src/config/zod-schema.ts:446](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/config/zod-schema.ts#L446)
