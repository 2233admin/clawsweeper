## Summary

Keep #40247 open. Current main at 43a003b8 still does not implement the PR's server-side dual-bind behavior: non-loopback bind hosts resolve to a single listener, while local gateway calls still target 127.0.0.1. The related #50751 client-side path remains open and does not clearly supersede this focused gateway-listener fix.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land a current-main version of the server-side dual-bind fix with the PR's tailnet/custom regression tests and docs/security wording updates, or explicitly choose the #50751 client-side host-resolution approach and close #40247 only after that shipped path demonstrably fixes CLI probes, internal callGateway users, and sessions_spawn for gateway.bind=tailnet and gateway.bind=custom.
