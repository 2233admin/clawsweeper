## Summary

Current main already has the production fallback that turns empty assistant turns into a visible error payload, but it does not contain this PR's specific regression coverage for non-strict-agentic NVIDIA/Nemotron unsigned-thinking-only and fully empty `content: []` stop turns. The PR is recent, focused, test-only, and tied to concrete NemoClaw hang reports; the only review feedback is a minor issue-reference clarity suggestion.
