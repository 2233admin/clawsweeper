## Summary

Keep #70197 open. Current main still enables the bundled Bonjour gateway discovery plugin by default, that plugin still imports @homebridge/ciao, and the lockfile still resolves @homebridge/ciao to 1.3.6. The published 1.3.6 dependency contains the exact Windows ARP exec path without windowsHide on a 15s poll; OpenClaw only has disable workarounds, not a default-path fix.

## Best Possible Solution

Keep #70197 open as the canonical OpenClaw bug. The likely fix is to update and stage the bundled Bonjour runtime dependency to @homebridge/ciao 1.3.7, or apply an equivalent local pnpm patch if maintainers do not want the dependency bump yet. Verification should include a Windows gateway run with Bonjour enabled and process-birth proof that the ciao ARP polling loop no longer creates visible cmd.exe/conhost.exe windows.
