## Summary

Keep this PR open. Current main has the generic uncaught-exception handler registry, but it still does not suppress transient uncaught network failures in the CLI/global handlers, and Telegram polling still only wires recovery through unhandled rejections. The PR remains relevant to linked issue #60515 and the reported Telegram polling stall symptoms.

## Best Possible Solution

Keep this PR open for maintainer review or rebase. The best implementation is to port the PR's transient uncaught suppression and Telegram uncaught polling restart onto current main, preserve fatal hooks for non-transient uncaught exceptions, keep regression coverage for CLI `ENETUNREACH` suppression and Telegram uncaught polling HttpError/getUpdates restart, and resolve or consciously accept the duplicate legacy-handler logging note.
