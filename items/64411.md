## Summary

Current main still leaves external MCP/plugin work on hot reply paths without the requested fast-degrade budget, circuit breaker, optional degraded state, or diagnostics. The earlier ClawSweeper keep-open comment remains accurate against current main.

## Best Possible Solution

Keep this issue open and implement a core resilience layer around external MCP/plugin work: configurable short catalog and tool-call budgets, failure counters with cooldown/circuit state, degraded empty-tool materialization for optional/unhealthy MCP capability, status/log diagnostics that distinguish channel health from external capability health, and regression tests for slow catalog listing, slow callTool, repeated failure cooldown, and blocking plugin-hook phases.
