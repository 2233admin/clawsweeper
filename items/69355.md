## Summary

Keep #69355 open. Current main still lacks the proposed `openclaw doctor` rebuild-vs-pristine npm diagnostic: the doctor install path only runs source-checkout hygiene, there is no rebuild-guard module/test/doc entry on main, and package-manager updates still refresh same-version global installs. The PR is a concrete repo-actionable doctor/update safety improvement, not a duplicate, ClawHub item, or external-admin task.

## Best Possible Solution

Keep the PR open and ask for a maintainer-quality revision: integrate a non-blocking doctor rebuild guard in the install section, keep it silent when registry data is unavailable, add a bounded fetch timeout, validate `package.json` and `dist/build-info.json` version parity, avoid stale-cache false positives, and keep docs/changelog aligned with repo attribution rules.
