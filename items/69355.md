## Summary

Keep #69355 open. Current main still does not implement a doctor install-freshness/rebuild guard: the doctor flow only runs source-install hygiene, current update checks compare against npm latest rather than npm publish time, and there is no rebuild-guard source/test/doc surface. The PR is focused and recent, with useful discussion and review feedback, so it should be refreshed rather than closed.

## Best Possible Solution

Keep the PR open and ask for a current-main refresh. The best path is a focused doctor helper wired after `noteSourceInstallIssues`, using existing build-info metadata plus npm publish-time data with a strict timeout/cache policy, explicit corrupt/version-mismatch verdicts, tests for the emission matrix and network/cache cases, and docs/changelog updates that match current repository conventions.
