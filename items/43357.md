## Summary

Keep this PR open. Current main still has ASCII-only Discord outbound mention candidate parsing, so the Unicode/CJK handle rewrite requested here is not implemented. The PR remains relevant, but its latest review thread still raises a correctness risk around wrapped URL tokens, and related open PR #59818 overlaps the same Discord CJK mention area without superseding or shipping this narrower fix.

## Best Possible Solution

Keep this PR open and ask for a revision that recognizes or strips leading URL wrappers before the CJK URL-context check. Then compare it with #59818 so maintainers can choose or consolidate one Discord plugin fix. The final patch should preserve reserved mentions, avoid emails and URLs, support Unicode/CJK handles where intended, and add regression tests for Unicode handles, Unicode local-part emails, plain URL paths, wrapped URL tokens, and existing ASCII behavior.
