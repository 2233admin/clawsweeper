## Summary

Keep open. Current `main` appears to address two parts of the report: `/hooks/agent` now rewrites agent-scoped session keys to the explicit target agent before dispatch, and the BlueBubbles webhook route is pinned into the startup plugin route registry so `/bluebubbles-webhook` is still reachable after later registry swaps. But the phone-format mismatch is still present: BlueBubbles handle normalization does not canonicalize `+1...` vs `1...`, and inbound/configured conversation matching still relies on exact string equality after that normalization. Because the issue is bundled and one reported bug remains, there is not strong evidence to close it as implemented on `main`.
