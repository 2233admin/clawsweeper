## Summary

Keep open. Current `main` does not implement configurable fallback agent IDs: bootstrap agent-dir resolution, default-agent selection, config schema/types, and main-session/session-key helpers still hardcode `"main"` when `agents.list` is absent.

## Evidence

- **Bootstrap agent dir still hardcodes `main`:** `resolveOpenClawAgentDir()` only honors `OPENCLAW_AGENT_DIR`/`PI_CODING_AGENT_DIR`, then unconditionally joins `agents/<DEFAULT_AGENT_ID>/agent`; there is no `OPENCLAW_DEFAULT_AGENT_ID` or config-based fallback path here on current `main`.
  - file: [src/agents/agent-paths.ts:6](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/agents/agent-paths.ts#L6)
  - command: `nl -ba src/agents/agent-paths.ts | sed -n '1,80p'`

- **Default-agent resolver still falls back only to `DEFAULT_AGENT_ID`:** `listAgentIds()` and `resolveDefaultAgentId()` return `DEFAULT_AGENT_ID` whenever `agents.list` is empty; they do not consult `agents.defaultAgentId` or `OPENCLAW_DEFAULT_AGENT_ID`.
  - file: [src/agents/agent-scope-config.ts:60](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/agents/agent-scope-config.ts#L60)
