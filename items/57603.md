## Summary

Current `main` does not contain this change, and the proposed unconditional handling looks too broad to close the PR conservatively. The lifecycle still throws `reconnect-exhausted` on `main`, while the PR would convert even the existing `maxAttempts: 50` startup-failure test into a clean stop.
