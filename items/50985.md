## Summary

Keep open. Current `main` still intentionally streams partial assistant text and tool-progress previews on messaging surfaces, and there is no matching per-agent or per-channel config to suppress assistant text that precedes a later tool call.

## Evidence

- **Repo remained clean:** `git status --short` returned no output, so the checkout stayed byte-for-byte clean during inspection.
  - command: `git status --short`
- **Streaming model still emits intermediate channel text:** Docs describe two live output layers: block streaming emits channel messages as the assistant writes, and preview streaming updates a preview message while generating. `blockStreamingBreak` only switches between `text_end` and `message_end`; it does not define a result-only or pre-tool-suppression mode.
  - file: [docs/concepts/streaming.md:12](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/docs/concepts/streaming.md#L12)
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Preview streaming is still a first-class feature on Slack/Discord/Telegram:** Current docs still advertise `channels.<channel>.streaming` modes `off | partial | block | progress`, with live preview behavior for Slack, Discord, and Telegram, plus tool-progress preview updates ahead of the final reply.
  - file: [docs/concepts/streaming.md:108](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/docs/concepts/streaming.md#L108)
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **No matching suppression config found:** Searches across `src`, `docs`, and `extensions` found no agent/channel config or runtime hook named `suppressIntermediateText` and no channel-level `outputMode: result-only`; the only `outputMode` hits were unrelated CLI output formatting code.
  - command: `rg -n "suppressIntermediateText|result-only|outputMode|intermediate text|pre-tool narration|suppress.*partial reply|partial reply" src docs extensions -g '!**/*.test.ts'`
- **Core reply pipeline forwards partial assistant text before tool events:** The agent runner still forwards `onPartialReply` payloads immediately, then separately forwards tool-start events. There is no branch here that buffers/drops assistant text once a tool call follows.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:1109](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/auto-reply/reply/agent-runner-execution.ts#L1109)

- **Slack still turns partial replies into visible preview updates:** Slack dispatch wires `onPartialReply` into `updateDraftFromPartial(payload.text)` and also pushes tool-progress preview lines on `onToolStart`, so intermediate narration remains part of the live preview path.
  - file: [extensions/slack/src/monitor/message-handler/dispatch.ts:858](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/extensions/slack/src/monitor/message-handler/dispatch.ts#L858)
