## Summary

Keep this issue open. Current main still has the pair-aware enforcement gap described in the issue and reinforced by the discussion: the global circuit breaker is current-tool/current-signature scoped, ping-pong critical blocking still requires stable per-side result hashes, and the before-tool-call hook blocks only the current invocation rather than persisting a block for both tools in the alternating pair.

## Best Possible Solution

Keep this issue open and fix the agent tool-loop guard in src/agents/tool-loop-detection.ts plus the before-tool-call enforcement path. The best implementation should make ping-pong protection pair-aware, either by allowing sustained A/B alternation to escalate without identical per-side result hashes, adding a pair-scoped threshold before the current per-tool global breaker, or persisting short-lived session-level blocks for both tool signatures when a critical pair breaker trips. Add regression coverage for variable-result memory_search <-> memory_get alternation and for blocking both sides of the offending pair; update docs/config if a new threshold is introduced.
