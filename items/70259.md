## Summary

Keep PR #70259 open. Current main still has the exact chat.history fallback gap: the handler falls from a missing session entry thinkingLevel directly to resolveThinkingDefault, while that resolver does not consult agents.list[].thinkingDefault. The related #70302 is an adjacent SessionRow/sessions.list fix, not a replacement for this chat.history path.

## Best Possible Solution

Keep the PR open or port its focused fix: in src/gateway/server-methods/chat.ts, preserve entry.thinkingLevel as highest priority, then resolve resolveAgentConfig(cfg, sessionAgentId)?.thinkingDefault, and only then call resolveThinkingDefault with the catalog-aware model fallback. Add the focused chat.history regression tests described in the PR so future changes do not drop per-agent thinkingDefault again.
