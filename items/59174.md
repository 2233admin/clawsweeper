## Summary

Keep open. Current main still appears to expose a replied-to WhatsApp image only as quoted reply context with a `<media:image>` placeholder, while media download and `MediaPath` propagation are tied to the current inbound message, not the referenced `contextInfo.quotedMessage`. The related ClawSweeper matches cover outbound WhatsApp media or WebChat rendering, not this inbound quoted-image path.

## Best Possible Solution

Keep #59174 open and fix the WhatsApp inbound path. The likely implementation should resolve downloadable media from `contextInfo.quotedMessage` or the referenced stanza when a message replies to an image, save it through the same bounded media store used for direct inbound images, and pass it to the reply pipeline as agent-visible media while preserving existing reply metadata, group mention gating, access-control visibility rules, and size/type limits. Add a regression test for a group message that mentions OpenClaw while replying to an image.
