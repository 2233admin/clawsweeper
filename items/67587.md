## Summary

Keep open. Current `main` only rewrites outbound Discord `@name` text into `<@userId>` when an account-scoped directory-cache entry already exists. The cache is seeded by live directory lookups and explicit target resolution, not by deterministic config-backed aliases or ordinary inbound activity, so the reported reliability gap still exists.

## Evidence

- **Outbound send path is cache-only:** `sendMessageDiscord` rewrites message text through `rewriteDiscordKnownMentions`, and that helper only calls `resolveDiscordDirectoryUserId`; on a miss it returns the original `@name` unchanged.
  - file: [extensions/discord/src/send.outbound.ts:175](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/discord/src/send.outbound.ts#L175)

  - file: [extensions/discord/src/mentions.ts:59](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/discord/src/mentions.ts#L59)
