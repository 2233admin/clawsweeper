## Summary

Current main still does not provide the requested per-channel or broader tool-output visibility policy. OpenClaw has a global `messages.suppressToolErrors` setting and heartbeat/run-scoped `suppressToolErrorWarnings`, but the embedded-run payload policy still surfaces mutating tool failures before honoring the global suppression flag, and Telegram dispatch still delivers `isError` reply payloads rather than suppressing them from chat surfaces.

## Best Possible Solution

Keep #38458 open for a core runtime/config design: add a documented tool-output visibility policy with global defaults and per-channel overrides, apply it before Telegram/Discord/Slack chat delivery while preserving tool errors in the agent context, update schema/docs, and add focused regression tests for mutating `message` and `exec` failures on streaming chat surfaces. PR #51091 can be treated as a narrower related fix, but it is not a complete replacement for this issue unless expanded to cover the per-channel visibility policy.
