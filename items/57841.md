## Summary

Current `main` already contains the Anthropic OAuth/setup-token fix: `sk-ant-oat-*` credentials no longer get `service_tier` injected on the Anthropic stream path, regression tests cover that behavior, and the changelog records it as shipped in `2026.4.10`.

## Evidence

- **Current fix in Anthropic stream wrapper:** `createAnthropicServiceTierWrapper` returns early for `sk-ant-oat-*` credentials, so OAuth/setup-token requests bypass `service_tier` injection.
  - file: [extensions/anthropic/stream-wrappers.ts:151](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/anthropic/stream-wrappers.ts#L151)
- **Regression tests for setup-token/OAuth path:** Tests assert that both `/fast` and explicit `serviceTier` do not inject `service_tier` for `sk-ant-oat01-*` tokens.
  - file: [extensions/anthropic/stream-wrappers.test.ts:131](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/anthropic/stream-wrappers.test.ts#L131)
- **Fix documented in shipped release notes:** The `2026.4.10` changelog explicitly says Anthropic now skips `service_tier` injection on OAuth-authenticated stream wrapper requests to stop Claude OAuth HTTP 401 failures.
  - file: [CHANGELOG.md:1118](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/CHANGELOG.md#L1118)
- **Setup-token auth still supported and exercised:** Anthropic still exposes `setup-token` auth, and there is a live test that completes a request using a resolved setup-token profile.
  - file: [extensions/anthropic/provider-contract-api.ts:29](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/anthropic/provider-contract-api.ts#L29)
- **Live setup-token smoke coverage:** The live test `completes using a setup-token profile` validates request execution with Anthropic setup-token auth on current `main`.
  - file: [src/agents/anthropic.setup-token.live.test.ts:181](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/anthropic.setup-token.live.test.ts#L181)

## Close Comment

- Codex review found that Anthropic OAuth/setup-token credentials (`sk-ant-oat-*`) now bypass `service_tier` injection in the stream wrapper, which matches the reported Claude Opus/setup-token breakage path; see `extensions/anthropic/stream-wrappers.ts:151`.
- Regression coverage now asserts that both `/fast` and explicit `serviceTier` do **not** add `service_tier` for `sk-ant-oat01-*` tokens; see `extensions/anthropic/stream-wrappers.test.ts:131` and `extensions/anthropic/stream-wrappers.test.ts:168`.
- Current `main` still supports Anthropic `setup-token` auth and includes a live smoke test that completes a request using a resolved setup-token profile; see `extensions/anthropic/provider-contract-api.ts:29` and `src/agents/anthropic.setup-token.live.test.ts:181`.
- The shipped fix is documented in the `2026.4.10` changelog: `Providers/Anthropic ... skip service_tier injection on OAuth-authenticated stream wrapper requests so Claude OAuth streaming stops failing with HTTP 401`; see `CHANGELOG.md:1118`.
- This checkout's latest release is newer than that fix (`v2026.4.23`), so the reported `2026.3.28` regression is already addressed in supported releases.
