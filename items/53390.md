## Summary

Keep open. Current main still does not implement a viewport-aware text snapshot contract for the browser tool. Snapshot inputs and route planning expose format/mode/refs/selector/frame/labels/urls, but no visible-only or current-viewport option; the implementation still reads page/root accessibility snapshots or full CDP/Chrome MCP snapshot trees. The prior ClawSweeper comment also kept this open, and current main's HEAD is an unrelated docs-only commit, so there is no newer browser fix to close against.

## Best Possible Solution

Keep this issue open. The best fix is to define an explicit viewport-aware text snapshot mode or separate visible-viewport inspection action, then implement it consistently across Playwright, raw CDP, and Chrome MCP snapshot paths with a long-page regression test that scrolls and verifies the next snapshot represents post-scroll visible content.
