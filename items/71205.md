## Summary

Keep PR #71205 open. Current main still hardens the default state/config directory to `0o700` on config writes and has no `OPENCLAW_SKIP_STATE_DIR_HARDEN` or equivalent multi-uid sidecar opt-out. The PR is a recent, focused core config/IO bug fix with a concrete reproduction and an added regression test; it is not obsolete, implemented, or clearly superseded.

## Best Possible Solution

Keep PR #71205 open for maintainer review. The best path is to decide whether an opt-in `OPENCLAW_SKIP_STATE_DIR_HARDEN` escape hatch is the desired core contract for multi-uid sidecar deployments, then land a version that imports the existing env helpers, logs accepted use, preserves the default `0o700` hardening behavior, and includes the proposed regression test plus any needed env/deployment docs.
