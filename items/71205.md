## Summary

Keep #71205 open. Current main still hardens the state/config directory to `0o700` whenever the config path lives in the state dir, with no `OPENCLAW_SKIP_STATE_DIR_HARDEN` guard or companion regression test. The PR is a focused core config/runtime fix, not ClawHub/plugin work, and the provided review thread shows the only bot feedback was addressed by adding `logAcceptedEnvOption`.

## Best Possible Solution

Keep the PR open for maintainer review and either land this patch or an equivalent core config fix: add a truthy `OPENCLAW_SKIP_STATE_DIR_HARDEN` guard before the chmod, log accepted use via `logAcceptedEnvOption`, preserve the default `0o700` hardening behavior, and keep the regression test that proves a `0o2770` setgid shared state dir is left unchanged when the opt-out is set.
