## Summary

Keep open. Current main still has the unscoped BlueBubbles short-ID resolver and outbound call sites resolve numeric IDs without any caller chat context. PR #70244 remains a focused, relevant bundled BlueBubbles plugin fix with tests and active author follow-up; it is not implemented on main, not superseded by a local/current-main change, and not a ClawHub/out-of-repo cleanup item.

## Best Possible Solution

Keep PR #70244 open for maintainer review. The best product/code path is to land a BlueBubbles-owned fix that scopes short numeric message-ID resolution to the actual caller/destination chat across react/edit/unsend/reply, channel sendText, media send, and deliver paths, with regression tests for same-chat success, cross-chat rejection, ambiguous fail-open cases, and normalized handle targets.
