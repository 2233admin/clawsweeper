## Summary

Keep open. Current main has not implemented PR #69421's no-op package-update skip; it currently documents and tests the opposite contract: package installs are refreshed even when the installed version already matches the resolved target. The linked issue/duplicate PR context was reviewed, but because the remaining choice is a real CLI behavior/product decision rather than an already-shipped fix or clean duplicate, no conservative auto-close reason applies.

## Best Possible Solution

Keep the PR open for explicit maintainer product judgment. If OpenClaw wants idempotent package updates to skip the global package manager, rebase the idea onto current main and add a guarded skip that handles requested channel changes, plugin/sidecar refresh policy, doctor/restart behavior, --json and --dry-run output, and regression tests. If the current documented refresh behavior is intentional, a maintainer should close the PR manually as not planned with that rationale.
