## Summary

Keep open. Current main does not implement PR #69421’s requested no-op package-update short-circuit: the CLI still reaches the package manager when currentVersion matches targetVersion, and the current regression test expects that install. Related PR #69645 is an open duplicate and appears to address the channel-persistence review concern, but it is not merged or clearly designated as canonical. The docs and changelog also conflict, so this needs maintainer judgment rather than auto-close.

## Best Possible Solution

Keep #69421 open until a maintainer decides the intended policy. If OpenClaw should skip same-version package updates, use the safer shape from #69645 or update #69421 to preserve channel changes, define the JSON contract, align docs/changelog, and add regression coverage. If refresh/verify is intentional, update docs and close both PRs with that maintainer rationale.
