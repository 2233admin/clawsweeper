## Summary

Keep PR #70497 open. Current main still does not strip JSON Schema `if`, `then`, or `else` for Gemini tool schemas: the unsupported-keyword set omits those keywords, and the cleaning loop only drops keys present in that set. The PR is a focused fix with a targeted regression test, and the prior PR #69964 was closed unmerged rather than shipped.

## Best Possible Solution

Keep #70497 open, rebase if needed, and validate the targeted Gemini schema-cleaning test before merging. The best code path is to add `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS` and keep the regression test that proves normal properties survive while conditionals are stripped.
