## Summary

Keep #70578 open. Current main has added important config-write guardrails and now preserves source keys that are missing from the runtime shape, but the central requested behavior is not implemented: the root config schema is still strict, validation still goes through `OpenClawSchema.safeParse`, and bundled plugin CLI surfaces can still be blocked by `plugins.allow`. Related PR #67557 only covered the gateway startup hash guard/restart loop and does not supersede the remaining config-loss/product-boundary questions.

## Best Possible Solution

Keep the issue open for maintainer product judgment and a focused fix. The likely path is to split the remaining work into: decide whether unknown root keys should be supported as inert preserved data; if yes, change the root config schema/persistence path with regression tests for plugin install and gateway writes; keep the current source-shape projection and destructive-write rejection guardrails; and separately decide whether bundled plugin CLI command surfaces should bypass or be seeded into restrictive `plugins.allow`.
