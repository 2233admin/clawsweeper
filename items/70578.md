## Summary

Keep #70578 open. Current main has meaningful safeguards against silent config clobbering, including source-shape preservation, destructive-write rejection, and last-known-good restore, so the old data-loss path appears substantially mitigated. But the item is not fully implemented: the root OpenClaw schema is still strict rather than passthrough, unknown root fields still fail validation, and the reported bundled CLI command/`plugins.allow` behavior is still present and covered by tests.

## Best Possible Solution

Keep the issue open or split it into narrower maintainer-owned work: decide whether root `openclaw.json` should preserve unknown top-level fields or explicitly reject them, add a focused regression for plugin install/gateway writes proving no silent data loss, and separately decide whether bundled CLI command surfaces should bypass or auto-seed `plugins.allow`.
