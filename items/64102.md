## Summary

Keep open. Current main only suppresses hook-pack fallback for security scan block/failure when `--dangerously-force-unsafe-install` is set, so PR #64102's broader unforced blocked-install behavior is still not implemented.

## Best Possible Solution

Keep the PR open or land an equivalent main change that treats `security_scan_blocked` and `security_scan_failed` as terminal plugin install failures before hook-pack fallback, with a focused unforced npm blocked-install regression test.

## Evidence
