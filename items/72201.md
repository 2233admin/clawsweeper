## Summary

Keep #72201 open. Current main still lacks a `SystemEvent` audience primitive and still formats all queued system events as visible `System:` / `System (untrusted):` prompt lines. The PR is recent, focused, tied to the open canonical issue #69492, includes tests in its diff, and the one bot review concern was addressed by the author in commit c2ae546b8f.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase/update it against current main, preserve the addressed Greptile fix, run the normal changed gates, and decide whether this narrow hidden-runtime-context lane is the accepted first step for #69492 while leaving broader delivery-routing and remaining leakage work tracked separately.
