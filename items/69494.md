## Summary

Keep #69494 open. Current main still resolves the macOS browser-proxy local gateway token/password path as plain strings only, while docs and the SecretRef registry mark `gateway.auth.token` and `gateway.auth.password` as supported SecretRef credential paths. The PR is active, targeted at that gap, and maintainer discussion confirms the root cause while asking for merge-shape and regression-test tightening.

## Best Possible Solution

Keep #69494 open and review or land it, or an equivalent patch, after final macOS validation. The best implementation path is to add an app-side SecretInput resolver seam for local `gateway.auth.token` and `gateway.auth.password`, support the intended keychain-backed exec SecretRef shape without changing remote auth behavior unless product-approved, avoid blocking actor/main endpoint refresh on slow keychain lookup, and keep regression coverage for the actual `localConfig()` path used by the desktop browser proxy.
