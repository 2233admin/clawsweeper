## Summary

Keep PR #69494 open. Current `main` still has the macOS browser proxy/local gateway auth gap: the proxy gets credentials from `GatewayEndpointStore.localConfig()`, but that path only accepts plain string `gateway.auth.token` / `gateway.auth.password` values, while the public config surface supports SecretRef objects. The PR is recent, focused, and maintainer discussion says the root cause is real; the remaining work is to add regression coverage for the actual `localConfig()` / browser-proxy path before merge.

## Best Possible Solution

Keep the PR open and finish it in its current direction: add an internal/defaulted SecretRef resolver seam to `GatewayEndpointStore.localConfig(...)`, resolve local `gateway.auth.token` and `gateway.auth.password` SecretRefs consistently for the macOS browser proxy path, keep remote auth behavior unchanged unless explicitly requested, and add the maintainer-requested regression test covering `_testLocalConfig(...)` with injected SecretRef token/password resolution before merge.
