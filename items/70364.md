## Summary

Keep open. Current `main` still does not clean up bundled MCP runtime for the primary `sessions_send` nested agent run: `sessions_send` starts a gateway `agent` run and waits for completion, but never sets `cleanupBundleMcpOnRunEnd` and never calls `retireSessionMcpRuntimeForSessionKey` afterward. Cleanup exists only in the embedded-runner opt-in flag path and in the separate `runAgentStep()` helper used for A2A follow-up steps. The issue discussion also includes a fresh confirmation that this still reproduces on `v2026.4.23`, plus a user-side workaround.
