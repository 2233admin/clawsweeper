## Summary

Keep open. Current `main` does not implement the main-queue drain, so the reported behavior is still plausible for non-main DM sessions. The proposed PR also appears incomplete for current multi-agent code because it drains `resolveMainSessionKey(cfg)`, which is the default agent’s main key, not necessarily the active session’s agent.

## Risks / Open Questions

- Merging the PR as described could misroute or consume the default agent’s main-session events during a non-default agent turn.
- There is still no proof on current `main` that main-queued system events are consumed by per-channel-peer DM sessions; the targeted regression test from the PR is not present.
