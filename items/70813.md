## Summary

Keep this PR open. Current main still does not implement WhatsApp `edit` or `unsend` message actions, and the PR is a focused WhatsApp bundled-plugin implementation candidate rather than cleanup closure material. The security-sensitive part is the new message-mutation capability itself, not an unrelated supply-chain or CI change.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or revise a WhatsApp-plugin-owned implementation that advertises `edit` and `unsend` only when message sends are enabled, dispatches them through the existing gateway action surface, reuses outbound target authorization, constructs own-message Baileys keys with `fromMe: true` and no caller-controlled participant, adds focused tests, resolves the remaining security/input review items, and updates the public WhatsApp/message-action docs if the feature ships.
