## Summary

Keep open. Current `main` still lacks WhatsApp `edit`/`unsend` support in discovery, dispatch, capabilities, and send APIs, while those action names already exist in OpenClaw’s channel contract and are implemented by other bundled channels. This PR is coherent, in-scope OpenClaw work rather than cleanup-close material.

## Evidence

- **WhatsApp plugin only accepts `react` actions on current main:** The WhatsApp channel action adapter currently advertises message actions via `describeWhatsAppMessageActions`, but `supportsAction` only returns true for `react`, gateway execution is only enabled for `react`, and action handling still routes through `handleWhatsAppReactAction`. That means `edit`/`unsend` are not implemented on `main`.
