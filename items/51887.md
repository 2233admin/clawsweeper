## Close Comment

Closing from a Codex review because current `main` already implements this fix. The gateway client now reuses cached device-token scopes on cached-token reconnects (`src/gateway/client.ts:575`), and the regression test covers reconnects sending stored scopes like `operator.read`/`operator.write` instead of dropping to an admin-only request (`src/gateway/client.test.ts:700`). Current scope compatibility also treats `operator.admin` as satisfying `operator.read` (`src/shared/operator-scope-compat.ts:18`). This was shipped in `v2026.4.5` per `CHANGELOG.md:1294`. If someone can still reproduce this on `v2026.4.5` or newer, please open a fresh issue with the exact version, auth mode, and current gateway logs.
