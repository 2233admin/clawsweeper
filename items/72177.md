## Summary

Keep PR #72177 open. Current main still lacks the PR's central LiteLLM image-generation hardening: multipart LiteLLM edits, a LiteLLM provider default timeout, removal of model-supplied image timeout overrides, chat-completions routing for chat-native LiteLLM image models, and Gemini duplicate-image parsing coverage. The PR is recent, contributor-authored, unprotected by maintainer labels, and the provided discussion shows active follow-up rather than obsolete or superseded work.

## Best Possible Solution

Keep #72177 open for maintainer review. The best path is to land this PR or an equivalent focused patch that uses multipart uploads for LiteLLM edits, applies a provider-owned default timeout, prevents model-authored image timeout overrides when that is the intended contract, adds a robust LiteLLM chat-completions image route with dedupe/parsing budgets, and keeps regression tests on the LiteLLM provider and shared image_generate tool.
