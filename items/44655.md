## Summary

Keep open. Current main has improved the bundled Z.AI catalog, including the example `glm-4.7-flash`, and has a limited GLM-5 forward-compat resolver, but it still does not implement dynamic Z.AI `/models` discovery or populate the catalog from the provider endpoint. The remaining request is concrete and not obsolete.

## Evidence

- **Static Z.AI catalog:** The bundled Z.AI onboarding path seeds `ZAI_DEFAULT_MODELS` from a fixed in-repo list and passes it as `catalogModels`; it is not populated from an API response.
  - file: [extensions/zai/onboard.ts:14](https://github.com/openclaw/openclaw/blob/1531123d3547f9ff8ee8094590fc773305f018f2/extensions/zai/onboard.ts#L14)
