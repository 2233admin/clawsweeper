## Summary

Current `main` still routes uncategorized embedded-run failures, including mixed-cause fallback exhaustion after compaction, to the generic external-run `/new` message. Compaction-specific user messaging is still opt-in and not preserved in the final fallback text, so I did not find strong evidence that #67750 is fixed on `main`.

## Evidence

- **Generic external-run fallback still ends in `/new` message:** `buildExternalRunFailureText()` still returns the generic "Something went wrong... use /new" text unless the error matches a small set of special cases (tool-result mismatch, missing API key, OAuth refresh). There is no timeout-aware or post-compaction-aware branch here.
