## Summary

Keep #70164 open. The issue is recent and has concrete WebSocket logs; the reporter's follow-up shows the OpenClaw gateway itself emitted tool and lifecycle events with no assistant stream before the terminal chat event. Current main still has a source-level path where lifecycle end emits `state:"final"` with no `message` whenever no assistant text event populated the live chat buffer, and the existing tests cover buffered assistant text, not this missing-assistant case.

## Best Possible Solution

Keep the issue open. Reproduce a MiniMax embedded run that performs a tool call and then emits lifecycle end without assistant text, then fix the responsible layer: make the embedded runner retry or surface empty terminal assistant output as an error, emit terminal assistant text from the final transcript when available, or adjust gateway finalization to recover from a valid transcript message. Add a regression test for tool event -> lifecycle end -> chat final so WebSocket clients do not silently receive only `state:"final"` for non-silent assistant turns.

## Evidence
