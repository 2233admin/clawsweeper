## Summary

Keep open. The issue reports intermittent Telegram forum cross-topic send failures with the generic grammY `Network request for 'sendMessage' failed!` symptom. Current main does forward forum `threadId`/`messageThreadId` into `message_thread_id`, but it does not provide enough evidence that the reported intermittent network failure is fixed, and tests intentionally avoid retrying broad generic send-context network envelopes unless a safe pre-connect cause is present.
