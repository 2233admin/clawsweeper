## Summary

Keep open. Current `main` still returns the request target from the OpenAI-compatible `model` field rather than the resolved backend provider/model, and docs explicitly describe that field as an agent target (`openclaw`, `openclaw/default`, `openclaw/<agentId>`). There is no current response header or documented surface exposing the underlying model for `/v1/chat/completions`.

## Risks / Open Questions

- The request conflicts with the current documented agent-first contract for the OpenAI-compatible API, so implementing it may require an API design decision rather than a narrow bug fix.
- The handler may not currently retain a single authoritative 'resolved backend model' for response emission, especially when agent defaults, aliases, or runtime fallback logic are involved.
