## Summary

Current `main` does not expose the resolved backend provider/model in `/v1/chat/completions` responses. The Gateway contract is explicitly agent-first: request/response `model` is an OpenClaw agent target (`openclaw`, `openclaw/default`, `openclaw/<agentId>`), while the underlying provider/model is selected separately via `x-openclaw-model`. That means the issue is still a real, coherent, in-scope feature request and should remain open.

## Evidence
