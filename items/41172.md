## Summary

Keep open. Current `main` does not implement the Groq-specific `tool call validation failed: attempted to call tool '...' which was not in request.tools` matcher in `formatAssistantErrorText`, and the latest shipped release also lacks it. The PR still addresses a real gap, and the only review concern shown in the timeline (missing test coverage) was already addressed by the author in a follow-up commit.
