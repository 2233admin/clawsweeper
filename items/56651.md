## Summary

Current main does not implement configurable gateway announce/connection/plugin-init timeouts. The gateway config/schema/docs still expose only channel health-monitor knobs, and startup/sidecar channel boot still runs without any timeout wrapper around `startChannels()` or plugin `startAccount()` startup handoff.

## Evidence

- **Gateway config has no requested timeout fields:** `gateway` schema includes `channelHealthCheckMinutes`, `channelStaleEventThresholdMinutes`, and `channelMaxRestartsPerHour`, but no `announceTimeoutMs`, `connectionTimeoutMs`, or `pluginInitTimeoutMs`.
  - file: [src/config/zod-schema.ts:746](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/config/zod-schema.ts#L746)
  - command: `nl -ba src/config/zod-schema.ts | sed -n '736,908p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Public gateway types only expose health-monitor settings:** `GatewayConfig` documents only the health-monitor interval, stale threshold, and restart cap in this area; there is no configurable startup/announce timeout surface.
  - file: [src/config/types.gateway.ts:441](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/config/types.gateway.ts#L441)
