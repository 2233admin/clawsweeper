## Summary

Keep open. Current `main` still intentionally clears the in-memory gateway token when the WebSocket URL changes, and the surrounding storage/tests show that tokens are scoped per gateway. This PR's proposed behavior conflicts with that contract and matches the unresolved security concern raised in review rather than an already-implemented fix.

## Evidence

- **Current UI behavior clears token on gateway URL edits:** The Overview WebSocket URL input still resets `token` to `""` whenever the edited URL differs from the current `gatewayUrl`, so `main` does not implement this PR's behavior.
  - file: [ui/src/ui/views/overview.ts:268](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/ui/src/ui/views/overview.ts#L268)
  - command: `nl -ba ui/src/ui/views/overview.ts | sed -n '245,330p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Browser regression test locks in the current behavior:** Current browser coverage explicitly expects the token to be cleared after changing the gateway URL: `hydrates hash tokens, restores same-tab refreshes, and clears after gateway changes`.
  - file: [ui/src/ui/navigation.browser.test.ts:311](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/ui/src/ui/navigation.browser.test.ts#L311)

  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Storage layer scopes tokens by gateway URL:** Session tokens are loaded and persisted under per-gateway keys derived from `normalizeGatewayTokenScope(gatewayUrl)`, which establishes a deliberate per-endpoint auth boundary.
  - file: [ui/src/ui/storage.ts:105](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/ui/src/ui/storage.ts#L105)

  - file: [ui/src/ui/app-settings.ts:215](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/ui/src/ui/app-settings.ts#L215)
