## Summary

Keep open. Current `main` still intentionally clears Telegram webhooks before polling, and the issue’s reported `probe.status=401` would come from `getMe`, not `deleteWebhook`, so there is not strong evidence that the report is already fixed or safe to close as invalid/support-only.

## Evidence

- **Polling startup still clears webhook first:** `TelegramPollingSession.runUntilAbort()` calls `#ensureWebhookCleanup()` before entering the polling cycle, and `#ensureWebhookCleanup()` invokes `bot.api.deleteWebhook({ drop_pending_updates: false })`. Non-recoverable errors are not swallowed.
  - file: [extensions/telegram/src/polling-session.ts:130](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/telegram/src/polling-session.ts#L130)
  - command: `nl -ba extensions/telegram/src/polling-session.ts | sed -n '120,225p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Tests still enforce deleteWebhook-before-polling behavior:** Current tests explicitly expect polling startup order `deleteWebhook` then `run`, and only retry recoverable cleanup failures.
  - file: [extensions/telegram/src/monitor.test.ts:454](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/telegram/src/monitor.test.ts#L454)

- **Probe 401 is sourced from getMe, not webhook cleanup:** `probeTelegram()` calls `getMe` first and returns a 401/`Unauthorized` result immediately when that response is not OK. `getWebhookInfo` is only attempted after a successful `getMe`.
  - file: [extensions/telegram/src/probe.ts:96](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/telegram/src/probe.ts#L96)

- **Webhook clearing before polling is documented shipped behavior:** The changelog documents Telegram polling startup clearing webhooks before long-poll `getUpdates`, and the Telegram docs describe long polling and webhook as distinct modes.
  - file: [CHANGELOG.md:4175](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/CHANGELOG.md#L4175)

## Risks / Open Questions
