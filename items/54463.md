## Summary

Keep this issue open. Current main still shows the reported QMD traversal gap: docs document only a workaround for workspace-visible temp repos, QMD config has no exclusion/no-follow/cycle-safe traversal knob, configured paths are passed through as plain QMD collections, and runtime update still delegates traversal to `qmd update` without OpenClaw-side symlink pruning or realpath-cycle handling. The issue discussion also includes explicit maintainer direction to keep the runtime problem open after the docs correction and a current-behavior repro from the reporter.

## Best Possible Solution

Keep this issue open as a focused QMD traversal robustness item. Add the controlled symlink-loop repro as regression coverage, then make traversal cycle-safe/no-follow upstream or add OpenClaw-side pre-update pruning/exclusion before `qmd update` can recurse through workspace-visible temp monorepos. Keep the current docs workaround until a shipped runtime fix exists.
