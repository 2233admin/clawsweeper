## Summary

Keep #71553 open. Current main has not implemented the PR: WhatsApp ack reactions still build reaction options from `sender.jid` only, while the current identity helper can normalize LID-only group participants into `sender.lid` with `sender.jid` unset. The PR is a focused bug fix with a matching regression test for the missing group LID ack-reaction path.

## Best Possible Solution

Keep the PR open and either merge this one-line LID fallback plus regression test after normal maintainer review and CI, or apply the same fix on main. The change belongs in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.ts` because the lower-level WhatsApp reaction send path already preserves LID participants.
