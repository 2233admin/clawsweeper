## Summary

Keep #71553 open. Current main still drops LID-only WhatsApp group ack reaction participants because the ack path passes only `sender.jid`; inbound normalization can place an `@lid` sender in `sender.lid`, and the outbound reaction path already supports LID participants. The PR is recent, focused, and adds the missing regression coverage, so it is neither obsolete nor implemented on main.

## Best Possible Solution

Keep the PR open and review or land this narrow fix, or apply an equivalent patch on main: use `sender.jid ?? sender.lid ?? undefined` in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.ts`, keep direct-message behavior unchanged, and add the LID-only group ack regression test in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.test.ts`.
