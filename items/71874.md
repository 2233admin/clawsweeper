## Summary

Keep open. Current main does not contain this refactor: the inbound dispatcher wrappers still finalize the same context for silent-reply setup and then pass the original context into `dispatchInboundMessage()`, which finalizes again. The PR is narrow, non-maintainer-authored, has no protected labels, and the provided review/context does not show a canonical duplicate or shipped replacement.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, the best path is the PR's scoped approach: finalize once in the buffered and non-buffered wrapper paths, pass a `FinalizedMsgContext` through silent-reply and hook setup, route the shared dispatch body through a private helper, and keep focused unit coverage asserting one finalization pass while preserving existing hook and silent-reply behavior.
