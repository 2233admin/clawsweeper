## Summary

Keep PR #72158 open. Current main still has the bundled Microsoft Foundry provider, but the provider index, docs navigation, and provider docs page are still missing. The PR targets a real docs gap, though it should be coordinated with the overlapping Foundry provider-page PRs because adding only the index link would point at a page that does not exist on main.

## Best Possible Solution

Keep the PR open for now, but fold or rebase its provider-index entry into the canonical Microsoft Foundry docs change that adds `docs/providers/microsoft-foundry.md`, updates `docs/docs.json`, and aligns the content with the current `extensions/microsoft-foundry` auth and API-selection behavior.
