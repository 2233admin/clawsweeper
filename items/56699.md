## Summary

Current `main` still appears to have the reported bug. Native Telegram commands load `topicConfig`, but their policy check explicitly disables topic/group policy overrides, so topic-level `groupPolicy: "disabled"` is ignored for `/commands` while regular inbound group messages still honor those overrides.

## Evidence

- **Native command path ignores topic/group policy overrides:** `resolveTelegramCommandAuth` passes `topicConfig` into `evaluateTelegramGroupPolicyAccess`, but sets `useTopicAndGroupOverrides: false`, so command auth falls back to account/default policy instead of topic/group overrides.
  - file: [extensions/telegram/src/bot-native-commands.ts:394](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/telegram/src/bot-native-commands.ts#L394)
  - command: `nl -ba extensions/telegram/src/bot-native-commands.ts | sed -n '380,410p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Policy helper only consults topic/group `groupPolicy` when overrides are enabled:** `evaluateTelegramGroupPolicyAccess` reads `topicConfig?.groupPolicy` and `groupConfig?.groupPolicy` only when `useTopicAndGroupOverrides` is true; otherwise it uses fallback account/default policy.
  - file: [extensions/telegram/src/group-access.ts:146](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/telegram/src/group-access.ts#L146)
  - command: `nl -ba extensions/telegram/src/group-access.ts | sed -n '136,155p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Regular Telegram message path does honor topic/group policy overrides:** The normal inbound group-message handler calls the same helper with `useTopicAndGroupOverrides: true`, which matches the issue report that regular messages respect topic-level policy while `/commands` do not.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:594](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/telegram/src/bot-handlers.runtime.ts#L594)
  - command: `nl -ba extensions/telegram/src/bot-handlers.runtime.ts | sed -n '590,608p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Topic config is available at runtime for Telegram forum topics:** `resolveTelegramGroupConfig` returns `topicConfig` from `groups[chatId].topics[messageThreadId]`, so the missing enforcement is not due to lack of topic data.
  - file: [extensions/telegram/src/bot-core.ts:544](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/telegram/src/bot-core.ts#L544)
