## Summary

Keep #40200 open. Current main at 8ba82534e6c98410e19b226f3534b285fd75d278 still runs POSIX exec shells with plain ["-c"] and does not implement the PR's zsh/bash/fish startup-file suppression. The linked #40179 is closed and main has related PATH/login-shell handling, but that work only adjusts PATH/env probing; it does not change the actual command shell args used by host exec or PTY exec.

## Best Possible Solution

Keep the PR open. Rebase or refresh it onto current main, preserve the current non-interactive SHELL and PATH fallback behavior, add the shell-args helper for zsh/bash/fish, keep focused regression tests for those args, and run the changed agents/exec lanes. If maintainers decide startup files should remain allowed, close or redirect only after that product decision is explicit.
