## Summary

Keep open. Current `main` already covers part of the request (`--agent`, recent-activity filtering via `--active`, and `--json` output), but it does not implement the requested `--older-than` / `--newer-than` / size-based query surface for `openclaw sessions`. The only size-related session behavior on `main` is maintenance and disk-budget cleanup, not metadata querying.

## Evidence

- **Current CLI surface:** The `sessions` command on `main` registers `--json`, `--store`, `--agent`, `--all-agents`, and `--active <minutes>` only. There are no built-in flags for older-than, newer-than, or larger-than filtering.
  - file: [src/cli/program/register.status-health-sessions.ts:124](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/cli/program/register.status-health-sessions.ts#L124)

  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Implemented filtering behavior:** `sessionsCommand` only parses `opts.active` and filters rows by `Date.now() - row.updatedAt <= activeMinutes * 60_000`. It does not evaluate any older-than predicate or any size-based predicate.
  - file: [src/commands/sessions.ts:135](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/commands/sessions.ts#L135)
