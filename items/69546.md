## Summary

Keep open. Current `main` still gives the QQBot gateway a startup-time config snapshot for inbound routing, while gateway reload rules treat top-level `bindings` as dynamic/no-restart. That makes the reported divergence plausible: CLI reads current config, but a running QQBot gateway can keep routing against stale bindings until restart.

## Evidence

- **QQBot gateway keeps using `ctx.cfg` during message handling:** `handleMessage()` passes `ctx.cfg` into `buildInboundContext()` and outbound dispatch, and the gateway connection is also created with `cfg: ctx.cfg`, so the long-lived QQBot runtime keeps the config object it received at startup.
  - file: [extensions/qqbot/src/engine/gateway/gateway.ts:133](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/qqbot/src/engine/gateway/gateway.ts#L133)
  - command: `nl -ba extensions/qqbot/src/engine/gateway/gateway.ts | sed -n '108,180p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Inbound route resolution uses the provided config object:** QQBot inbound routing calls `runtime.channel.routing.resolveAgentRoute({ cfg, channel: "qqbot", ... })`, so any stale `cfg` object directly affects binding selection for incoming messages.
  - file: [extensions/qqbot/src/engine/gateway/inbound-pipeline.ts:52](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/qqbot/src/engine/gateway/inbound-pipeline.ts#L52)

- **Gateway reload rules treat top-level `bindings` as dynamic/no-restart:** `buildGatewayReloadPlan()` classifies the `bindings` prefix as `kind: "none"`, meaning config changes there are expected to apply via dynamic reads rather than a channel restart.
  - file: [src/gateway/config-reload-plan.ts:102](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/gateway/config-reload-plan.ts#L102)

- **QQBot only requests channel restart for `channels.qqbot` config changes:** The QQBot channel plugin declares `reload: { configPrefixes: ["channels.qqbot"] }`, so a top-level binding edit does not ask the gateway to restart the QQBot channel.
  - file: [extensions/qqbot/src/channel.ts:70](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/extensions/qqbot/src/channel.ts#L70)
