## Summary

Keep open. Current main still lacks auth-alias-aware lookup for config-backed provider auth in `src/agents/model-auth.ts`, so the underlying bug is not clearly implemented on main; the PR’s specific patch is also stale because `normalizeProviderIdForAuth` no longer performs `-plan` alias mapping on current main.

## Evidence

- **Config-backed auth lookup is not alias-aware on main:** `resolveProviderConfig()` only tries the raw provider key plus `normalizeProviderId()` key/scan matches. It does not call `resolveProviderIdForAuth()` or consult manifest auth aliases, so `models.providers.volcengine` will not be found when the runtime provider is `volcengine-plan`.
  - file: [src/agents/model-auth.ts:51](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/model-auth.ts#L51)

  - file: [src/agents/provider-auth-aliases.ts:124](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/provider-auth-aliases.ts#L124)
