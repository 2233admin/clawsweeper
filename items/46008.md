## Summary

Keep open. Current `main` has some gateway-connect hardening after `v2026.4.22`, but I did not find strong evidence that issue #46008 is fully fixed. The repo still allows the exact bundled-plugin shadowing path reported in comments (`openclaw plugins install @openclaw/feishu`), and loader precedence still favors an explicitly installed global plugin over the bundled duplicate.

## Evidence

- **Scoped bundled package specs still go through npm install:** The install planner only short-circuits bare plugin ids before npm and only falls back to bundled sources on `NPM_PACKAGE_NOT_FOUND`. A scoped spec like `@openclaw/feishu` is not intercepted.

## Risks / Open Questions

- The issue comments point to at least two plausible failure modes: gateway handshake/close timing and bundled-plugin shadowing after installing `@openclaw/feishu`.
- Without a safe reproduction on current `main`, closing would rely on inference rather than direct proof.
- Because the duplicate-plugin path still exists in current code, a user could plausibly still hit the same regression class even if some gateway timing bugs were fixed.
