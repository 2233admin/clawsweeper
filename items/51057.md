## Summary

Keep open. Current `main` still intentionally excludes `timeout` from auth-profile failure cooldown, and the tests lock that behavior in. The original report's exact root cause appears stale because profile rotation now walks forward through candidates instead of cycling them, but current code still allows extra timeout retries before failover via timeout-triggered compaction, so the slow-fallback concern is not strong enough to close as obsolete.
