## Summary

Keep open. Current main still has the internal pricing helpers, but it does not expose a `runtime.usage` namespace on the plugin runtime type, runtime object, tests, or plugin runtime docs. This is also not a ClawHub-only idea: the linked ContextClaw use case needs a core plugin-runtime seam to access OpenClaw's existing pricing resolution instead of copying internal pricing logic.

## Best Possible Solution

Keep PR #64436 open for maintainer review. The best path is to land a small additive `api.runtime.usage` surface, update `docs/.generated/plugin-sdk-api-baseline.sha256`, add/adjust a focused runtime shape test and mock typing, and document the new runtime namespace. If maintainers prefer issue tracking first, #64445 can remain the canonical issue, but the PR itself still contains useful implementation work and should not be auto-closed.
