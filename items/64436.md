## Summary

Keep open. Current main still does not expose `api.runtime.usage` or an equivalent public plugin SDK path for model pricing, while the underlying pricing helpers already exist and are tested internally. This PR remains a narrow, active plugin runtime API proposal, not cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer API review. If maintainers accept the shape, the best path is to rebase onto current main, keep the narrow additive `api.runtime.usage` implementation, add `docs/plugins/sdk-runtime.md` coverage and a small runtime-shape or assembly test, retain the regenerated SDK API baseline, and provide the relevant changed-gate/API-check proof before merge.
