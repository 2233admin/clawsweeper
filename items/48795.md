## Summary

Keep open. Current `main` does not already contain this PR's modular memory chunking work or its async parallelization changes. Main still uses a single inline `chunkMarkdown` implementation in the memory host SDK, there is no `src/memory/chunking/` module on `main`, and `src/agents/pi-embedded-runner/run/attempt.ts` still awaits bootstrap-context resolution and machine-name lookup sequentially. Some chunking improvements have landed separately on `main` (notably CJK-aware sizing and surrogate-pair handling), but that is not strong enough to treat this PR as implemented or obsolete.
