## Summary

Keep open. #64004 is a recent, concrete Control UI performance bug with follow-up timing evidence on v2026.4.15. Current main narrows the Sessions tab itself to `sessions.list`, but it still runs background `node.list` refreshes and has expensive `usage.cost` / `sessions.usage` / `cron.runs` paths matching the reporter's slow-call evidence. The broader #71631 responsiveness issue is related, but it does not supersede this specific bug report.

## Best Possible Solution

Keep #64004 open as the concrete bug. The best fix path is to add end-to-end Control UI timing instrumentation for click -> request -> response -> render, correlate concurrent RPCs, isolate or cancel background `node.list` / usage / cron refreshes so they cannot make unrelated views feel stuck, and optimize `sessions.usage`, `usage.cost`, and `cron.runs` for large local histories. Cross-link #71631 as the broader responsiveness umbrella, but do not auto-close this report yet.
