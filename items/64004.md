## Summary

Keep open. Current main still does not show a shipped fix for the reported Control UI latency mismatch: the Sessions tab still refreshes through `sessions.list`, the Overview path still overlaps Sessions and Usage refreshes, and the Usage controller still calls the slow endpoints named by the reporter. The related #71631 item is open and related, but it is not a merged fix or explicit maintainer supersession for this concrete Windows timing report.

## Best Possible Solution

Keep this issue open as a concrete performance bug. The best next step is to instrument the full Control UI path from tab click to visible render and correlate it with concurrent `usage.cost`, `sessions.usage`, `node.list`, `cron.runs`, `config.schema`, and `channels.status` calls on a Windows/global-npm setup. If maintainers want #71631 to become the canonical umbrella, they should say so explicitly or close this only after a merged implementation covers this reproduction.
