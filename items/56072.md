## Summary

Keep open. Current `main` still rolls daily-stale sessions by archiving the old transcript and starting a new session without invoking the compaction memory-flush path; the bundled `session-memory` hook remains limited to explicit `/new` and `/reset`, and reset config still lacks an `off`/`disabled` mode or any reset-time flush option.

## Evidence

- **Daily stale rollover still archives and replaces the session:** `initSessionState()` captures `previousSessionEntry` for stale sessions, archives the old transcript, and starts a new session on daily expiry. The regression test explicitly asserts this for a daily/scheduled reset.
