## Summary

Keep #71040 open. Current main still lacks `activeJobIds` mark/clear coverage for startup catch-up and manual cron runs, while task maintenance can still treat authoritative cron tasks as backed only by `isCronJobActive(jobId)` after the 5-minute grace. The PR remains a focused, plausible fix with relevant regression coverage rather than an already-implemented or superseded change.

## Best Possible Solution

Keep the PR open for maintainer review and likely land it, or land an equivalent patch that wraps startup catch-up and manual cron execution with `markCronJobActive`/`clearCronJobActive` in `try/finally` and keeps the proposed producer-side regression tests. Treat #68191 as broader durable reconciliation follow-up, not as a replacement for this focused live-run fix.
