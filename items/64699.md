## Summary

Keep this issue open. Current main still rejects any `sessions_send` call that contains both `sessionKey` and `label`, which is the reported failure mode. The linked PR #64846 is open, unmerged, and uses closing syntax for this issue, so it is an implementation candidate rather than a reason to close the issue before merge.

## Best Possible Solution

Keep the issue open until maintainers review and either land PR #64846 or choose another canonical fix. The fix should normalize duplicate `sessionKey`/`label` payloads in `src/agents/tools/sessions-send-tool.ts` while preserving rejection for conflicting targets, and it should add a regression test for equal duplicate values plus conflicting values.
