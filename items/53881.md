## Summary

Current main still matches the reported Control UI bug: the Logs view renders entries inside `.log-stream`, and the active CSS still caps that scroll area at fixed pixel heights. A linked open PR, #53916, uses closing syntax and remains the implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, refresh, and land #53916 if its CSS approach is acceptable. An equivalent fix should make the Control UI Logs `.log-stream` height responsive to available viewport space while preserving usable behavior on shorter screens; after that lands, this issue can close as implemented.
