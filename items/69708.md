## Summary

Current main has Bedrock `aws-sdk` runtime fallback coverage, but the specific reported auth-profile store behavior remains: persisted `type: "aws-sdk"` entries are still rejected as `invalid_type`, and the linked fix PR was closed unmerged.

## Best Possible Solution

Keep #69708 open and land a narrow auth-profile store fix: treat SDK-managed `aws-sdk` entries as non-secret markers to skip or repair without warning, add regression coverage for a mixed `aws-sdk` marker plus valid credentials, and keep the existing Bedrock `aws-sdk` runtime fallback intact. If maintainers prefer not to support the marker, add a doctor/docs repair path and close only after a canonical follow-up exists.
