## Summary

Keep #39703 open. Current main still has FIFO behavior in the cron schedule cache: `resolveCachedCron()` returns cache hits without promoting the key, while capacity eviction deletes the first Map key. The PR remains a focused fix, and its second commit addresses the prior Greptile review by adding key-level cache assertions that would fail under FIFO.

## Best Possible Solution

Keep #39703 open as the canonical cron cache LRU fix, or land an equivalent maintainer patch that promotes cache hits with delete/re-set and includes key-level regression coverage proving the promoted expression survives eviction while the next-oldest entry is evicted.
