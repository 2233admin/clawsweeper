## Summary

Current main still leaves `/bot-clear-storage` on the QQBot pre-dispatch slash-command path because its registration lacks `requireAuth: true`. The PR addresses a live authorization gap with the same framework-auth pattern already shipped for `/bot-approve`, and current regression coverage only asserts the `/bot-approve` routing.

## Best Possible Solution

Keep the PR open and land the focused fix, or cherry-pick an equivalent change: add `requireAuth: true` to `bot-clear-storage`, add the framework-registry regression assertion, and keep a changelog entry with contributor attribution. Maintainers may also consider a follow-up defense-in-depth explicit allowlist check because the command deletes local host files.
