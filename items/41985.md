## Summary

Keep open. Current `main` has Control UI/WebChat image attachment support, but the compose path is still explicitly image-only and does not implement the broader document/file upload request in #41985. A maintainer comment on 2026-04-25 also explicitly kept this issue open for the generic file workflow.

## Evidence

- **Control UI attachment contract is image-only:** `ui/src/ui/chat/attachment-support.ts:1` sets `CHAT_ATTACHMENT_ACCEPT = "image/*"`, and `ui/src/ui/chat/attachment-support.ts:3` only accepts MIME types starting with `image/`. That excludes PDFs and generic documents from the WebChat upload path.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/ui/src/ui/chat/attachment-support.ts#L1)
  - command: `sed -n '1,220p' ui/src/ui/chat/attachment-support.ts`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Paste, file picker, and drag/drop all filter to images:** `ui/src/ui/views/chat.ts:208` only collects pasted `ClipboardItem`s whose type starts with `image/`; `ui/src/ui/views/chat.ts:252` and `ui/src/ui/views/chat.ts:283` skip any selected or dropped file whose MIME type is not supported; `ui/src/ui/views/chat.ts:1224` wires the file input with `accept=${CHAT_ATTACHMENT_ACCEPT}`. Current main therefore does not support arbitrary file upload in WebChat.
  - file: [ui/src/ui/views/chat.ts:208](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/ui/src/ui/views/chat.ts#L208)
  - command: `nl -ba ui/src/ui/views/chat.ts | sed -n '180,340p' && nl -ba ui/src/ui/views/chat.ts | sed -n '1060,1255p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Outbound API payload is serialized as images only:** `ui/src/ui/controllers/chat.ts:349` converts UI attachments into API attachments, and `ui/src/ui/controllers/chat.ts:358` hard-codes each one as `{ type: "image", mimeType, content }`. Even if the UI accepted other files, the current send path is still image-scoped.
  - file: [ui/src/ui/controllers/chat.ts:349](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/ui/src/ui/controllers/chat.ts#L349)

That does not satisfy this issue's request for user-side file upload/paste/drag support in WebChat.
  - file: [ui/src/ui/chat/grouped-render.ts:1223](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/ui/src/ui/chat/grouped-render.ts#L1223)
