## Summary

Current `main` only implements image attachments in WebChat, not the broader file upload support requested for documents/PDFs and other files. The issue should stay open because the composer, MIME allowlist, and gateway attachment parser still explicitly restrict uploads to `image/*`.

## Evidence

- **Composer allowlist is image-only:** `CHAT_ATTACHMENT_ACCEPT` is hard-coded to `image/*`, and supported upload MIME types are only those starting with `image/`.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/ui/chat/attachment-support.ts#L1)
  - command: `nl -ba ui/src/ui/chat/attachment-support.ts | sed -n '1,20p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Paste and drag/drop only collect images:** Clipboard handling filters `item.type.startsWith("image/")`, and drag/drop skips any file whose MIME type is not supported by the image-only allowlist.
  - file: [ui/src/ui/views/chat.ts:206](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/ui/views/chat.ts#L206)

  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Attach button only opens an image picker:** The chat input renders `<input type="file" accept=${CHAT_ATTACHMENT_ACCEPT} ...>`, so the UI picker itself is limited to images.
  - file: [ui/src/ui/views/chat.ts:1205](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/ui/views/chat.ts#L1205)
