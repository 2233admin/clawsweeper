## Summary

Keep PR #39894 open. Current main still does not implement this PR's central shared outbound-normalization fix: `createOutboundPayloadPlanEntry()` parses raw `payload.text` before delivery/JSON projection, and `src/infra/outbound` still has no `stripInboundMetadata()` call. The PR is draft and mixed with unrelated CI/macOS/test-runner edits, but its core bug fix and regression test remain useful for #39847.

## Best Possible Solution

Keep PR #39894 or canonical bug #39847 open until current main lands a narrow core fix: strip copied inbound metadata before `parseReplyDirectives()` in `src/infra/outbound/payloads.ts`, and add a regression in `src/infra/outbound/payloads.test.ts` covering `Conversation info (untrusted metadata)` followed by visible text plus `MEDIA:`. Split the unrelated CI/macOS/test-runner changes into separate work if still needed.
