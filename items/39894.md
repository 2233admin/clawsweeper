## Summary

Keep PR #39894 open. Current main still parses outbound reply directives from raw payload text, so echoed inbound metadata can remain in user-visible outbound text while MEDIA directives are extracted. The linked bug #39847 remains open, and this PR contains a relevant narrow fix plus regression coverage, although its unrelated CI/macOS/test-runner changes should be split before landing.

## Best Possible Solution

Keep PR #39894 or canonical issue #39847 open until main lands a narrow fix in src/infra/outbound/payloads.ts: call stripInboundMetadata(payload.text ?? "") before parseReplyDirectives() inside createOutboundPayloadPlanEntry(), add a focused outbound payload regression for Conversation info/Sender metadata followed by visible text plus MEDIA, and split unrelated CI/macOS/test-runner edits into separate work if still needed.
