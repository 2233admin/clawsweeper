## Close Comment

Closing from a Codex review because current `main` already has the shared progress/final ownership model this issue was asking for. The repo now defines a canonical cross-channel streaming surface with `mode: "progress"` and `preview.toolProgress` in `src/config/types.base.ts:12` and resolves that canonical object ahead of legacy per-channel knobs in `src/plugin-sdk/channel-streaming.ts:76`. Final handoff and cleanup are centralized in `src/channels/draft-preview-finalizer.ts:14`, which decides whether a final reply edits the live preview in place or falls back to normal delivery with cleanup. That shared helper is already wired into multiple channel adapters instead of each adapter re-implementing ownership rules locally, including Discord (`extensions/discord/src/monitor/message-handler.process.ts:801`), Slack (`extensions/slack/src/monitor/message-handler/dispatch.ts:678`), and Mattermost (`extensions/mattermost/src/mattermost/monitor.ts:297`). The behavior is also documented in `docs/concepts/streaming.md:108` and covered by tests such as `src/channels/draft-preview-finalizer.test.ts:14` and `src/auto-reply/reply/dispatch-from-config.test.ts:1544`. Best proving commit on `main` is `8fade9df27a9a25cd017db1154409b3ad9bd3741`. I could not tie that change to a released tag yet, so I am leaving `fixedRelease` unset. If there is still a specific current-main channel path that violates this shared ownership model, please open a fresh bug with that exact repro.
