## Summary

Keep open. Current main does re-check mention policy for ordinary user-owned Discord threads, but it still deliberately bypasses `requireMention` in bot-owned `autoThread` threads and in bound thread sessions. That matches the follow-up comment that `autoThread: true` creates a bot-started thread where every later message is handled, and the requested Discord thread-specific mention policy is not present in config, docs, or tests.

## Best Possible Solution

Keep #40895 open and fix the Discord plugin behavior explicitly. A maintainable path is to add a Discord thread-reply mention policy, likely under Discord guild/channel or `channels.discord.thread`, wire it through config types, Zod schema, generated docs/help, and preflight, then add regression tests for bot-owned autoThreads and bound thread follow-ups with `requireMention: true`. If maintainers want the narrower product shape from #52568 to be canonical, link #52568 from this issue before closing this one as a duplicate.
