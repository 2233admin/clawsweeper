## Summary

Keep open. Current `main` still sends Gemini CLI OAuth HTTP requests through `fetchWithSsrFGuard` in its default strict mode, which explicitly ignores env proxy routing unless callers opt into `mode: "trusted_env_proxy"`. The Google Gemini CLI OAuth path does not opt in, and the latest local release tag `v2026.4.22` shows no landed change in `extensions/google/oauth.http.ts` or `extensions/google/oauth.shared.ts` that would implement the reported proxy fix.
