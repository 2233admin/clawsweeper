## Summary

Keep open. Current `main` still does not provide a first-class `browser.proxy` config or `openclaw browser set proxy` CLI surface. The only current path is the existing `browser.extraArgs` passthrough, which can carry raw Chrome flags like `--proxy-server=...` for locally launched managed profiles.

## Evidence

  - file: [src/config/types.browser.ts:33](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/config/types.browser.ts#L33)
