## Summary

Keep open. Current `main` still exposes `MessageReceivedHookContext.metadata` as a generic escape hatch, but the Telegram `message:received` path only maps the standard canonical metadata fields and does not surface Telegram-specific `forward_*`, `via_bot`, or `reply_to_message` data. This is a concrete additive bundled-plugin feature request, not an obsolete or already-shipped fix.
