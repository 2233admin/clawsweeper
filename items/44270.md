## Summary

Keep open. Current main still strips the `:thread:` suffix before cron delivery inference, so a session key like `agent:main:telegram:direct:12345:thread:12345:99` still infers plain `delivery.to=12345` instead of `12345:topic:99`. Existing cron tests cover Slack thread stripping and Telegram group/forum `:topic:` preservation, but not the reported Telegram direct-chat thread suffix. Related PRs discuss the right fix, but the fix is not present on current main.
