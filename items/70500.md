## Summary

Keep open. Current main contains general macOS TCC/signing and PeekabooBridge guidance, but it does not contain this PR's specific warning that granting Accessibility to a shared node runtime can grant GUI automation access to packages run through that node executable. The related closed PR #69654 was the same unmerged contribution, and the provided #69561 close report is not enough to supersede this PR because the current source still lacks the node-specific docs text.

## Best Possible Solution

Keep #70500 open and let maintainers either merge the focused docs warning or fold equivalent wording into docs/platforms/mac/permissions.md and docs/platforms/mac/peekaboo.md. If #69561 was closed as implemented, reconcile that closure because current main does not contain the specific node Accessibility warning requested here.
