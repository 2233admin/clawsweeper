## Summary

Keep open. Current main still computes the memoryFlush and preflight compaction token thresholds from `reserveTokensFloor`, not the user-facing `agents.defaults.compaction.reserveTokens`, while docs and Pi settings treat `reserveTokensFloor` as a floor under `reserveTokens`. The provided GitHub context also shows open, unmerged PR #69379 with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review or land PR #69379, or an equivalent core fix that resolves an effective reserve for both preflight compaction and memoryFlush gates from configured `reserveTokens` plus `reserveTokensFloor` using the same semantics as the Pi settings path. Add regression coverage for `reserveTokens > reserveTokensFloor` and preserve the small-window `threshold <= 0` behavior deliberately.
