## Summary

Keep open. Current main still has the duplicated task text path described by #72019: `buildSubagentSystemPrompt` embeds `params.task` in the system prompt, while `spawnSubagent` passes the same `task` into that builder and also sends it as `[Subagent Task]` in the first user message. PR #72026 is a focused, unmerged fix for a real current behavior, with a regression-test patch in the provided PR context and only a minor follow-up concern about the now-dead `task` parameter.

## Best Possible Solution

Keep #72026 open for maintainer review. The best path is to land a revised version of the PR that keeps the subagent task in the first user message only, adds the focused regression test, and either removes the now-unused `task` parameter from the prompt builder/call sites or marks the compatibility choice clearly.
