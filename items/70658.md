## Summary

Keep open. Current `main` still validates missing plugin config during non-activating snapshot and CLI-metadata loads, which matches the reported false-positive error path, and the referenced fix exists only as an open PR rather than shipped code on `main`.

## Evidence

- **Undefined config is coerced into schema validation input:** `validatePluginConfig` replaces an undefined value with `{}` via `params.value ?? {}` before AJV validation, so a schema with top-level `required` will fail even when no plugin config entry was materialized for that load path.
  - file: [src/plugins/loader.ts:1215](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/plugins/loader.ts#L1215)
  - command: `nl -ba src/plugins/loader.ts | sed -n '1215,1234p'`
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Manifest-only/snapshot loader still validates `entry?.config` before early-returning:** In `loadOpenClawPlugins`, the loader calls `validatePluginConfig({ value: entry?.config })` and only afterwards checks `if (!shouldLoadModules) { ... continue; }`. That means non-activating or manifest-only snapshot loads still run config validation on an undefined config entry.
  - file: [src/plugins/loader.ts:2525](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/plugins/loader.ts#L2525)
  - command: `nl -ba src/plugins/loader.ts | sed -n '2495,2542p'`
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **CLI metadata loader has the same undefined-config validation path:** `loadOpenClawPluginCliRegistry` also validates `entry?.config` directly and treats failures as plugin load errors, which matches the report about CLI subcommands surfacing false-positive invalid-config errors.
  - file: [src/plugins/loader.ts:3175](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/plugins/loader.ts#L3175)
  - command: `nl -ba src/plugins/loader.ts | sed -n '3170,3184p'`
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Referenced fix is not on main yet:** The open patch for PR #70811 adds targeted tests named `skips undefined config validation during CLI metadata capture` and `skips undefined config validation during non-activating manifest-only snapshots`, plus loader changes to defer validation when config is undefined and the load is non-activating. That confirms the bug is still being fixed, not already fixed on `main`.
  - command: `curl -L --silent https://patch-diff.githubusercontent.com/raw/openclaw/openclaw/pull/70811.diff | sed -n '1,420p'`
- **Issue discussion already points to related external repro and an in-repo fix PR:** The provided timeline cross-references CortexReach/memory-lancedb-pro#685 as the external reproduction and openclaw/openclaw#70811 as the proposed fix, which strengthens that this is a concrete current bug rather than an incoherent or stale report.
