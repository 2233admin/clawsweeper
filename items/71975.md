## Summary

Keep #71975 open. Current main has no LTX/Lightricks model entries in the video-generation docs or provider implementations, and the bundled fal and ComfyUI paths do not advertise an LTX preset/template through `video_generate`. The dedicated Lightricks-provider portion likely belongs in a third-party plugin/ClawHub path, but the issue's lowest-risk request for bundled fal/Comfy discoverability remains actionable in this repo.

## Best Possible Solution

Keep the issue open but scope it narrowly: add LTX discoverability for existing bundled paths only if upstream API/docs verification confirms the fal/Comfy payload shape, likely by adding fal model entries/docs and, if maintainers want it, a Comfy workflow example. Treat a dedicated Lightricks hosted provider as a separate ClawHub/third-party plugin unless a maintainer explicitly wants to own it in core.
