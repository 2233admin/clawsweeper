## Summary

Keep open. Current main still requires explicit `deliver: true` for gateway agent delivery, so Telegram group-bound agent sessions with omitted `deliver` remain session-only despite carrying Telegram group/topic routing metadata. The linked glfruit/openclaw PR/source-port is useful prior validation, but the behavior is not implemented in this checkout.

## Best Possible Solution

Implement a small gateway-side helper for Telegram group-bound agent session keys, use it when computing `wantsDelivery` and the final `deliver` flag, and add focused tests for group, topic, explicit `deliver:false`, and non-upgraded session shapes. The existing external glfruit/openclaw PR can be used as prior validation, but the change still needs to land in OpenClaw main.
