## Summary

Keep open. The docs/client mismatch is still real: current OpenClaw docs and release scripts describe a `clawhub package publish <source>` surface with `--dry-run`/`--json`, while published `clawhub@0.9.0` still registers folder-only package publish without those options. The reported 120s message also maps to the published ClawHub CLI's client-side upload timeout, but the reporter's exact publish failure still needs ClawHub-side reproduction or registry logs.

## Best Possible Solution

Keep the issue open and resolve it at the product boundary: either publish a new `clawhub` CLI that matches the OpenClaw docs/scripts (`<source>` input, GitHub source forms, `--dry-run`, `--json`, and clearer timeout behavior), or revise the OpenClaw docs/release script to the actually shipped `clawhub@0.9.0` folder-only surface. The publish timeout should be diagnosed with the reporter's package/auth path or ClawHub registry logs after the client/docs mismatch is made explicit.
