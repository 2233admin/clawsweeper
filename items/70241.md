## Summary

Keep open. The issue is recent, reporter provided a concrete publish command and failure, and a maintainer comment already live-verified the docs/client mismatch on 2026-04-26. Current main still documents `clawhub package publish <source>` with `--dry-run`/`--json`, while OpenClaw release automation appears to depend on a pinned ClawHub source checkout rather than the public npm CLI surface described to users.

## Best Possible Solution

Keep this issue open. Reconcile the public `clawhub` npm client with the documented `package publish` source/`--dry-run`/`--json` surface, or update the public ClawHub/plugin docs to match the currently shipped CLI. Then investigate the 120-second publish timeout with the reporter's frozen package/provenance and ClawHub client/server logs, especially because OpenClaw release automation appears to use a pinned ClawHub source CLI rather than the public npm package.
