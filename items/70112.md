## Summary

Keep open. Current main still uses the upstream CJK-blind token estimator in the compaction and pre-prompt compaction paths, so this PR is not implemented or obsolete. The PR is also a plausible focused fix, though it still has review feedback to resolve before merge.

## Best Possible Solution

Keep the PR open and finish it or a successor. The maintainer path is to add a CJK-aware wrapper around the upstream estimator, use it in `src/agents/compaction.ts`, `src/agents/pi-embedded-runner/compact.ts`, and `src/agents/pi-embedded-runner/run/preemptive-compaction.ts`, preserve ASCII parity, include toolCall/tool-result CJK payloads, and add focused regression tests. The remaining preemptive routing math should stay in token units instead of converting CJK-aware overflow back through a fixed `* 4` character heuristic.
