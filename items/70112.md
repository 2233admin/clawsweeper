## Summary

Keep open. Current main still uses the upstream CJK-blind `estimateTokens()` in the compaction and preemptive-compaction paths, and the PR's `estimateTokensCjkAware` wrapper/regression tests are not present on main. The linked bug #70052 remains a real core behavior issue rather than obsolete cleanup.

## Best Possible Solution

Keep #70112 open as the active implementation vehicle, or land an equivalent main-branch fix that adds a CJK-aware wrapper around the dependency estimator, uses it consistently in compaction and preemptive-compaction paths, includes toolCall argument handling, and adds focused regression tests for ASCII parity, CJK uplift, and tool-call CJK arguments. Also address the preemptive truncation route unit mismatch before merging.
