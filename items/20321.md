## Summary

Keep #20321 open. Current main still does not implement Telegram `groupPolicy: "members"`: the shared type/schema reject the value, Telegram runtime only evaluates `open`, `disabled`, and `allowlist`, and docs/tests still describe the existing allowlist model. The linked chat-only-trigger plugin is useful related work, but it is a workaround rather than the requested all-group-members trusted allowlist policy.

## Best Possible Solution

Keep #20321 open until maintainers either implement `groupPolicy: "members"` across shared config schema/types, Telegram runtime membership verification, tests, and docs, or explicitly decide this policy belongs in a plugin/API design rather than bundled Telegram behavior.
