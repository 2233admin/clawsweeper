## Summary

Keep open. Current main still has the reported mismatch for Dreaming/backfill: the bundled session-memory hook writes `memory/YYYY-MM-DD-slug.md`, while memory-core daily ingestion, historical seeding, CLI `rem-backfill`, and gateway doctor backfill all select only `YYYY-MM-DD.md`. The related #46687 cleanup covered AGENTS/startup context loading, not this Dreaming promotion/backfill path.

## Best Possible Solution

Keep the issue open. Implement a shared daily-memory source matcher for memory-core that accepts the canonical `YYYY-MM-DD.md` files and session-memory `YYYY-MM-DD-slug.md` files where appropriate: live daily ingestion, historical seeding, CLI `rem-harness`/`rem-backfill`, and gateway doctor backfill. Preserve source paths in grounded diary output, keep generated Dreaming directories excluded, and add regression tests using a hook-style file such as `memory/2026-04-21-ai-assistant-config.md` for both daily ingestion and `rem-backfill --path ./memory --stage-short-term`.

## Risks / Open Questions

- A fix should avoid broad arbitrary Markdown ingestion: keep excluding `memory/dreaming/**`, `.dreams`, generated phase reports, symlinks, and oversized files where those guards already exist.
