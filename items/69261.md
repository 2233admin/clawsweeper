## Summary

Keep PR #69261 open. Current main still resolves Ollama embedding auth without host scoping: remote key, provider key, then env `OLLAMA_API_KEY`, and attaches the result as `Authorization` to whichever embedding `baseUrl` is selected. Existing tests still assert the problematic behavior the PR fixes, including sending a provider placeholder as a bearer token and sending env auth to a local Ollama URL. The related #69222 is a separate open web-search PR, not a landed or canonical replacement for this embedding-path fix.

## Best Possible Solution

Keep the PR open and have the author or maintainer rebase it, resolve the provider opt-out review finding, and land host-scoped Ollama embedding auth with focused regression tests. If #69222 also proceeds, consider sharing a small Ollama auth-boundary helper between embedding and web-search paths, but do not close this embedding PR as superseded unless a landed main commit covers the same embedding behavior.
