## Summary

Keep #69261 open. Current main still selects Ollama embedding auth from remote config, provider config, then env without checking the resolved request host, and then unconditionally attaches `Authorization: Bearer` to the resolved embedding base URL. Tests still assert that the `ollama-local` placeholder is sent as a literal bearer token. The PR is not implemented or superseded on main; it remains useful but should be rebased and should address the provider opt-out review comment before merge.

## Best Possible Solution

Keep the PR open. The best path is to rebase it onto current main, keep the host-boundary auth selector in the bundled Ollama plugin, and add or preserve regression tests proving env `OLLAMA_API_KEY` is only sent to Ollama Cloud, provider config keys do not cross remote host boundaries, remote keys remain scoped to their remote URL, and non-secret markers such as `ollama-local` are not emitted as bearer tokens.
