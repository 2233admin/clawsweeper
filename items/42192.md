## Summary

Keep open. Current `main` does not implement this PR’s behavior: the manifest registry still warns on cross-origin duplicate IDs, the tests still assert those warnings for bundled/global and bundled/workspace collisions, and the docs still describe duplicate drops as logged by precedence. The only shipped suppression on `main` is the older same-physical-plugin/multi-path case, not the broader cross-origin distinct-root shadowing proposed here.

## Evidence

There is no per-origin bucketing like the PR proposes.
