## Summary

Keep open. Current main still hard-codes cron job self-deletion to `schedule.kind === "at"`, so the `every`/`cron` bug described by PR #64375 and issue #63770 is not implemented or obsolete. The later changelog entries cover cron session/MCP cleanup, not removal of recurring job definitions after a successful run.

## Best Possible Solution

Keep PR #64375 open, rebase if needed, and land a focused fix that removes the `schedule.kind === "at"` restriction while adding regression tests for `every` and `cron` success deletion plus the error-retention invariant. If maintainers prefer a different open PR as canonical, choose that explicitly, but do not close this item until #63770 is fixed on main.
