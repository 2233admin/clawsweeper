## Summary

Keep open. Current main still rejects the requested trusted-network `gateway.auth.mode: "none"` path for non-loopback binds. The related PR #43820 was closed unmerged per the provided GitHub context, and the broader open RFC #69066 discusses adjacent gateway auth design but does not supersede this narrow feature request.

## Evidence

- **Runtime guard still blocks non-loopback none auth:** `resolveGatewayRuntimeConfig` refuses non-loopback binds unless a shared token/password is present or `authMode` is `trusted-proxy`; `auth.mode: "none"` has no shared secret and is not exempted.
