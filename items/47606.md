## Summary

Keep open. Current `main` has partial anti-stall protections, but it does not implement the issue's proposed runtime guard system. OpenClaw now retries GPT-5 plan-only turns, can surface a blocked state, and has a CLI no-output watchdog, yet there is no authoritative persisted per-run guard state, no artifact-gated status update enforcement, no `ENFORCEMENT_GUARDS_V1` flag, no explicit escalation-timer framework for `noArtifactTimeout`/`noPhaseChangeTimeout`, and no evident 'no-idle-after-green' enforcement.

## Risks / Open Questions

- Current protections are lane-specific: mainly strict-agentic OpenAI/Codex GPT-5 runs and CLI no-output watchdogs, not a unified anti-drift controller for all multi-batch execution.
- Prompt-level heartbeat guidance can reduce repetitive status chatter, but without runtime enforcement it does not satisfy the issue's acceptance criteria.
