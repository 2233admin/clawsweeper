## Summary

Keep open. Current main still unconditionally requires a resolved API key in the media-understanding image and audio/video auth paths, while the Bedrock docs and chat path already support aws-sdk credential-chain auth without a static key. PR #72092 is a focused, recent fix for linked bug #72031, and its latest provided head also addresses the bot-reviewed audio/video empty-key rotation gap with an allowEmptyKey path and new tests.

## Best Possible Solution

Keep PR #72092 open for maintainer review and CI. The best path is to finish validating the focused implementation that carries aws-sdk missing-key allowance through image, MiniMax fallback, and audio/video execution, including the new empty-key rotation contract, then merge or iterate on that PR rather than closing it as cleanup.
