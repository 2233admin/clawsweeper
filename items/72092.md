## Summary

Keep PR #72092 open. Current main still unconditionally requires a static API key in the media image, MiniMax fallback, and audio/video execution paths, so the linked Bedrock `aws-sdk` credential-chain bug in #72031 is not implemented on main. The PR is focused on the real affected paths, and its latest head addresses the bot-reviewed empty-key rotation gap with an `allowEmptyKey`/`allowEmptyExecution` path plus regression coverage.

## Best Possible Solution

Keep PR #72092 open for maintainer review and CI. The best path is to validate or iterate on the focused implementation that carries the Bedrock `aws-sdk` missing-key allowance through `src/media-understanding/image.ts`, `src/media-understanding/runner.entries.ts`, and `src/agents/api-key-rotation.ts`, then merge it to close #72031 if the scoped tests and maintainer review pass. PR #72066 is closed unmerged and should not be treated as superseding this work.
