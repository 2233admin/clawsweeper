## Summary

Keep #38430 open. Current main still does not implement a final-reply policy or delivery-time formatter that expands `~/...` to full absolute paths. Main only expands tilde paths for host file-tool inputs, while prompt/test code still intentionally compacts some skill paths to `~/...`, and a host-edit recovery test still expects a user-visible tool success string containing `~/demo.txt`. The related PR #38495 proposed `OPENCLAW_EXPAND_PATHS`, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep the issue open. Maintainers should decide the product boundary between internal prompt/tool path handling and user-visible final replies, then implement either explicit final-reply prompt guidance, a delivery-time text formatter, or a documented config switch with regression coverage for final replies while preserving intentional skill-prompt compaction and host file-tool tilde input expansion.
