## Summary

Keep PR #64335 open. Current main has generic auth-profile rotation and rate-limit cooldown behavior, but it does not implement this PR's narrower Z.AI env-backed runtime auth-profile synthesis, API-key external-auth overlay support, or Z.AI rotation secret metadata. The provided PR discussion and bot reviews also show a real implementation thread with edge-case fixes, not a low-signal or already-obsolete PR.

## Best Possible Solution

Keep PR #64335 open and review it as the active focused fix for Z.AI env-backed API-key rotation. The likely best product/code path is to add provider-owned Z.AI runtime-only auth profiles, extend the generic external-auth overlay contract safely for API-key credentials, keep runtime-only env secrets out of persisted auth-profile writes, register Z.AI rotation env vars for scrub/setup metadata, and retain the targeted tests around Z.AI profile synthesis, auth order, and persistence.
