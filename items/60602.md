## Summary

Keep this issue open. Current main still does not inject or configure Bedrock `requestMetadata` for per-agent attribution, and #60698 is an open, unmerged implementation candidate that explicitly links itself as the fix. The active agent id is available on the runner path, but the current stream option wrapper and Amazon Bedrock plugin only handle other request mutations such as temperature, cache retention, region, guardrails, and cache points.

## Best Possible Solution

Keep the issue open and have maintainers review #60698 or a successor patch. The best fix should live in the Bedrock-owned/agent transport path, derive agent id and display name from verified runtime context, merge cleanly with any future configurable metadata from #57360, enforce AWS metadata constraints, preserve non-Bedrock behavior, and add focused tests proving the final Bedrock request includes metadata only when appropriate.
