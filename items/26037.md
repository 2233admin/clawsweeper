## Summary

Keep open. Current `main` supports Qwen/Bailian Coding Plan endpoints, but it does not appear to officially enable reasoning/thinking for the requested Coding Plan models (`glm-5`, `kimi-k2.5`). The bundled catalog and docs still advertise reasoning only for `MiniMax-M2.5`, so the feature request is not implemented on `main`.

## Evidence

- **Coding Plan endpoint support exists, but only as general Qwen provider setup:** The bundled `qwen` provider exposes Coding Plan auth choices for `coding.dashscope.aliyuncs.com` and `coding-intl.dashscope.aliyuncs.com`, listing `glm-5`, `kimi-k2.5`, and `MiniMax-M2.5` as available models.
  - file: [extensions/qwen/index.ts:105](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/extensions/qwen/index.ts#L105)
  - command: `nl -ba extensions/qwen/index.ts | sed -n '1,230p'`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Catalog does not mark GLM/Kimi Coding Plan models as reasoning-capable:** In the built-in Qwen model catalog, `MiniMax-M2.5` is `reasoning: true`, but `glm-5`, `glm-4.7`, and `kimi-k2.5` are all `reasoning: false`. That means current main does not model the requested Bailian reasoning support as implemented.
  - file: [extensions/qwen/models.ts:74](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/extensions/qwen/models.ts#L74)

- **Docs still describe reasoning as enabled only for MiniMax on Qwen:** Provider docs list the built-in Qwen catalog and explicitly call out `qwen/MiniMax-M2.5` as 'Reasoning enabled', while `qwen/glm-5` and `qwen/kimi-k2.5` have no such note.
  - file: [docs/providers/qwen.md:155](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/docs/providers/qwen.md#L155)
