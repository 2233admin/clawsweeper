## Summary

Current main still leaves the heartbeat-specific uppercase bracket-block leak unresolved. The delivery path still passes heartbeat payloads through without stripping `[TOOL_CALL]...[/TOOL_CALL]` or `[TOOL_RESULT]...[/TOOL_RESULT]`, and the provided GitHub context shows open same-author PR #54165 with closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #54165, or land an equivalent current-main fix in the heartbeat reply delivery path. The implementation should strip uppercase `[TOOL_CALL]...[/TOOL_CALL]` and `[TOOL_RESULT]...[/TOOL_RESULT]` before channel delivery, preferably with focused coverage in `src/auto-reply/reply/agent-runner-payloads.test.ts`; if maintainers choose a shared sanitizer, ensure the heartbeat branch actually uses it.
