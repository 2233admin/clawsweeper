## Summary

Keep open. Current `main` still leaves stale Control UI session data in place when `sessions.list` resolves `undefined`, which matches the linked fix PR for this issue and means the reported phantom subagent count is not clearly resolved on `main` or in the latest shipped release.

## Evidence

- **Current main still preserves stale sessions on undefined responses:** `loadSessionsOnce()` only assigns `state.sessionsResult` inside `if (res)`. If the client request resolves `undefined`, prior rows remain cached and the UI can keep showing old session/subagent counts.
