## Summary

Keep open. Current `main` still contains the stale-session-state guard that can preserve an old subagent/session count when `sessions.list` returns no payload, and the linked fix has not landed on `main`. There is also no regression test covering the undefined-response path.

## Evidence

- **Current UI logic still keeps stale result on falsy response:** `loadSessions()` only writes `state.sessionsResult` inside `if (res)`, so an `undefined` response leaves the previous cached result in place instead of clearing it.
  - file: [ui/src/ui/controllers/sessions.ts:170](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/ui/src/ui/controllers/sessions.ts#L170)
  - command: `nl -ba ui/src/ui/controllers/sessions.ts | sed -n '150,205p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **No regression test for the reported path:** The sessions controller tests cover checkpoint refreshes, but there is no assertion that `loadSessions()` clears stale state when `sessions.list` resolves to `undefined`. Some delete-path tests stub `sessions.list` as `undefined` without checking `sessionsResult` reset.
  - file: [ui/src/ui/controllers/sessions.test.ts:55](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/ui/src/ui/controllers/sessions.test.ts#L55)
  - command: `nl -ba ui/src/ui/controllers/sessions.test.ts | sed -n '1,240p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Proposed fix is not on main:** The linked PR `#55831` is still open and describes replacing the `if (res)` guard with unconditional assignment (`state.sessionsResult = res ?? null`), which matches the still-unfixed code on current `main`.
  - command: `web open https://github.com/openclaw/openclaw/pull/55831`

## Risks / Open Questions

- If the gateway now guarantees `sessions.list` never returns `undefined`, the exact trigger may be narrower than reported, but current `main` still has the stale-state behavior if that path occurs.
- I did not run the UI interactively; this decision is based on code inspection plus the linked unmerged fix.
