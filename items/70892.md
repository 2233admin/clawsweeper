## Summary

Keep open. Current main has the existing heartbeat prompt-section gate and a usable skills allowlist for filtering `<available_skills>`, but it does not implement the requested user-facing per-section gates for `## OpenClaw CLI Quick Reference`, `## Reply Tags`, or the messaging components sub-block, and it does not add the requested systemPromptOverride cookbook table. The related ClawSweeper item #66612 is unrelated to this prompt-section/config request.

## Best Possible Solution

Keep the issue open as an actionable core prompt/config enhancement. The likely maintainer path is to add typed, default-true prompt section gates under a stable config surface, wire them through the runtime prompt assembly, update generated schema/help/docs, add system-prompt tests for each omitted block, and document which system prompt building blocks are structural versus optional. For `<available_skills>`, prefer documenting or extending the existing `agents.defaults.skills` / `agents.list[].skills` allowlist rather than adding a parallel include surface unless maintainers specifically want an exclude list.
