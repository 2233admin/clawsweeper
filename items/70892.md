## Summary

Keep open. Current main partially addresses the `<available_skills>` concern through existing agent skill allowlists, but it does not implement the requested per-section gates for the CLI quick reference, reply/output directives, or messaging guidance, and the requested `systemPromptOverride` cookbook is not present.

## Best Possible Solution

Keep the issue open. The best path is to add typed, documented, default-true prompt section controls in the agent config surface, wire them into `buildAgentSystemPrompt` without moving the cache boundary, and add focused tests for omitting the CLI quick reference, reply/output directives, and messaging/component guidance. For skills, document the existing `agents.defaults.skills` / `agents.list[].skills` allowlist as the conservative include filter, and only add an exclude form if maintainers want that extra config shape. Add a systemPromptOverride cookbook/table if maintainers still want operators to own full prompt replacement safely.
