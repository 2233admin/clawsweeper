## Summary

Keep open. Current main partially reduces the reported prompt waste: the old Reply Tags section is not rendered in `src/agents/system-prompt.ts`, the messaging component text is now limited to inline-button guidance, and include-style `<available_skills>` filtering already exists through `agents.defaults.skills` / `agents.list[].skills`. But the requested core prompt-section gates are not implemented: the CLI quick reference is still emitted in the full prompt, Messaging is only omitted by runtime `promptMode=minimal`, there is no `agents.defaults.prompt.*` config surface, and the docs still only describe full `systemPromptOverride` rather than a safe cookbook of required vs optional blocks.

## Best Possible Solution

Keep the issue open, but narrow the implementation path: use existing `agents.defaults.skills` / `agents.list[].skills` for include-style skill filtering now; if maintainers want the rest, add a core config surface for prompt section gates with defaults preserving current behavior, wire it through types/zod/generated schema and prompt composition tests, and add a public docs cookbook for safe `systemPromptOverride` building blocks. Decide separately whether a subtractive `available_skills.exclude` is worth adding beyond the existing allowlist.
