## Summary

Keep open. Current main still only partially addresses the report: cron agent-turn jobs accept and honor `model` and `timeoutSeconds`, but `contextTokens` and `maxSpendUsd` are not part of the cron tool, gateway protocol, stored cron payload type, normalization/merge path, docs, or per-run enforcement contract.

## Best Possible Solution

Keep this issue open. Preserve the already-shipped `model` and `timeoutSeconds` support, then add a deliberate `contextTokens` cron agentTurn contract through the cron tool schema/flat recovery, gateway protocol schema, exported cron types, normalizer, service patch merge behavior, isolated runtime selection, docs, and regression tests. For `maxSpendUsd`, either implement tested per-run spend-cap enforcement or split that part into a focused product/API issue before accepting the field.
