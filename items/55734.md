## Summary

Keep open. Current `main` still feeds all CDP `/json/list` targets into tab selection and the default selection path still picks the first `type === "page"` candidate without filtering browser-internal URLs like `chrome://...`. The related fix PR #55740 was closed unmerged by automation, and its proposed helper/tests are not present on `main`.

## Evidence

- **Default tab selection still prefers the first page target with no browser-internal URL filter:** `ensureTabAvailable()` builds `candidates` from `listTabs()`, then `pickDefault()` returns the first tab whose `type` is `page`. There is no check for `chrome://`, `devtools://`, or other browser-internal surfaces before choosing that tab.
