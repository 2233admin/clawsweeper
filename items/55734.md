## Summary

Keep open. Current `main` still has no Chrome-internal-target filter in the remote CDP/Playwright tab-selection path, so the reported `chrome://...top-chrome/` mis-selection is not implemented away and remains plausible.

## Evidence

- **Remote CDP listing returns every Playwright page as a normal page:** `listPagesViaPlaywright` iterates all connected Playwright pages, returns `url: page.url()`, and hard-codes `type: "page"` without filtering `chrome://` or `chrome-untrusted://` URLs. That means Chrome internal UI targets remain eligible tab candidates.
  - file: [extensions/browser/src/browser/pw-session.ts:1090](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/browser/src/browser/pw-session.ts#L1090)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Default tab selection still picks the first page candidate with no URL guard:** `ensureTabAvailable()` builds `candidates` directly from `listTabs()`, then `pickDefault()` prefers the first tab whose type is `page`. There is no exclusion for Chrome internal UI URLs, so an omnibox popup/tab-search target can still win default selection.
  - file: [extensions/browser/src/browser/server-context.selection.ts:48](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/browser/src/browser/server-context.selection.ts#L48)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Agent/browser actions use the default selection path when no targetId is provided:** `withRouteTabContext()` calls `profileCtx.ensureTabAvailable(params.targetId)`. For ordinary agent actions without an explicit target id, navigation operates on whatever default tab selection chose.
  - file: [extensions/browser/src/browser/routes/agent.shared.ts:115](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/browser/src/browser/routes/agent.shared.ts#L115)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **No main-branch code path mentions filtering these Chrome internal targets:** A source search for `chrome://`, `chrome-untrusted`, `top-chrome`, `omnibox-popup`, and `Tab Search` under `extensions/browser/src/browser` found no filtering logic; the only hit was unrelated docs (`docs/tools/browser.md`).
  - command: `rg -n "chrome://|chrome-untrusted|top-chrome|omnibox-popup|Tab Search" extensions/browser/src/browser extensions/browser/src/browser/**/*.test.ts docs/tools/browser*.md -g '!node_modules'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Cross-referenced PR is not evidenced on current main history:** The issue references PR #55740, but local history on this checkout shows no commit message or merge reference for `55740`, and no fix-specific test coverage for internal Chrome UI target filtering was found on `main`.
