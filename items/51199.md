## Summary

Keep open. Current `main` has changed the npm package surface since `openclaw@2026.3.13`, but I did not find strong evidence that the tar-entry-order problem is fixed. The current release checker only validates packed path presence/forbidden content, not archive ordering, and the latest published tarball I inspected (`openclaw@2026.4.22`) still contains `skills/*` payload entries without matching directory-header entries.

## Evidence

- **Current package still publishes skills payloads:** `package.json` on `main` still includes `skills/` in the published `files` list.
  - file: [package.json:23](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/package.json#L23)
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Pack validation discards archive order:** `collectPackedTarballErrors()` reads `npm pack --json --dry-run --ignore-scripts`, converts the returned file list into a `Set`, and then checks only required/forbidden paths and content. There is no validation of tar header ordering or parent-directory entries.
  - file: [scripts/openclaw-npm-release-check.ts:488](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/scripts/openclaw-npm-release-check.ts#L488)
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Latest published tarball still lacks directory-header matches:** Inspecting `openclaw@2026.4.22` from the registry showed payload entries under `package/skills/` (for example `package/skills/sherpa-onnx-tts/bin/sherpa-onnx-tts`), while `tar -tvzf` produced no matches for `package/(extensions|skills|dist/plugin-sdk/config)/?$` directory entries.
  - command: `curl -L --silent https://registry.npmjs.org/openclaw/-/openclaw-2026.4.22.tgz | tar -tvzf - | rg "package/(extensions|skills|dist/plugin-sdk/config)/?$"`
- **Original broken package exposed the reported roots directly:** Extracting `package/package.json` from `openclaw@2026.3.13` shows the published `files` list included `docs/`, `extensions/`, and `skills/`, matching the paths named in the report.
  - command: `curl -L --silent https://registry.npmjs.org/openclaw/-/openclaw-2026.3.13.tgz | tar -xOzf - package/package.json | sed -n '1,40p'`
- **Current release docs add install verification but not tar-order proof:** The release docs now require a post-publish verifier against a fresh temp prefix and pack-content checks, but they only mention required payloads and bundled runtime contents, not tar entry ordering.
  - file: [docs/reference/RELEASING.md:87](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/docs/reference/RELEASING.md#L87)
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
