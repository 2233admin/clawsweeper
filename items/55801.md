## Summary

Current `main` still lacks per-tool-call audit logging that identifies which tool-policy rule/path made the allow or deny decision. The embedded runner logs only tool start/end, and the policy pipeline/matcher only filters tools to booleans without preserving or logging matched rule attribution.

## Evidence

- **Embedded runner still logs only start/end:** `handleToolExecutionStart` and `handleToolExecutionEnd` emit DEBUG logs for `embedded run tool start` / `embedded run tool end` with runId, tool, and toolCallId only; there is no adjacent policy-decision log or matched-rule field.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:604](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/pi-embedded-subscribe.handlers.tools.ts#L604)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.tools.ts | sed -n '560,760p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **End log also has no policy attribution:** The end-of-call log remains the same minimal shape and does not include allow/deny provenance or config-path attribution.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:1108](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/pi-embedded-subscribe.handlers.tools.ts#L1108)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.tools.ts | sed -n '1088,1124p'`

- **Policy pipeline applies steps without recording which one matched:** `applyToolPolicyPipeline` iterates configured steps such as `tools.allow`, `tools.byProvider.allow`, agent policy, and group policy, but only mutates the filtered tool list. It does not return match metadata, emit logs, or attach the winning policy layer.
  - file: [src/agents/tool-policy-pipeline.ts:110](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/tool-policy-pipeline.ts#L110)
  - command: `nl -ba src/agents/tool-policy-pipeline.ts | sed -n '108,160p'`

- **Underlying matcher returns boolean only:** `isToolAllowedByPolicyName` / `isToolAllowedByPolicies` compute allow/deny as booleans only. There is no structure for matched rule, config path, or decision provenance that a logger could emit.
  - file: [src/agents/tool-policy-match.ts:5](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/tool-policy-match.ts#L5)
