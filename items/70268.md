## Summary

Current main still does not implement the PR’s core behavior: wake hooks are normalized and dispatched with only text/mode, mapped wake actions drop agentId/sessionKey metadata, and the gateway wake dispatcher still hard-codes the main session. The PR is a focused, recent fix for open bug #64556, while related PR #35231 was closed unmerged, so none of the conservative close reasons apply.

## Best Possible Solution

Keep #70268 open. The best path is to finish reviewing and land this PR or an equivalent gateway fix that threads explicit `sessionKey`/`agentId` through direct and mapped wake hooks, canonicalizes the target consistently before enqueueing and heartbeat wake, enforces prefix policy for computed agent wake targets, and keeps the added regression coverage.
