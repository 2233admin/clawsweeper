## Summary

Keep open. Current `main` does not implement a configurable Brave Search API `baseUrl`: the Brave runtime still uses hardcoded Brave endpoints, the Brave plugin config schema only exposes `apiKey` and `mode`, and the Brave docs/tests do not describe or cover any `baseUrl` option.

## Evidence

- **Brave runtime hardcodes Brave API endpoints:** `BRAVE_SEARCH_ENDPOINT` and `BRAVE_LLM_CONTEXT_ENDPOINT` are fixed to `https://api.search.brave.com/...`, and request builders instantiate `new URL(...)` from those constants rather than any config-derived base URL.
  - file: [extensions/brave/src/brave-web-search-provider.runtime.ts:30](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/extensions/brave/src/brave-web-search-provider.runtime.ts#L30)
  - command: `nl -ba extensions/brave/src/brave-web-search-provider.runtime.ts | sed -n '1,220p'`
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Brave config type has no baseUrl field:** `BraveConfig` only defines `mode?: string`, and `resolveBraveConfig` returns that shape from `searchConfig.brave`; there is no parsed or normalized `baseUrl` field.
  - file: [extensions/brave/src/brave-web-search-provider.shared.ts:7](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/extensions/brave/src/brave-web-search-provider.shared.ts#L7)

- **Plugin manifest/schema exposes no baseUrl setting:** `extensions/brave/openclaw.plugin.json` exposes UI hints and schema properties for `webSearch.apiKey` and `webSearch.mode` only. `webSearch` has `additionalProperties: false`, so `baseUrl` is not part of the declared config contract.
  - file: [extensions/brave/openclaw.plugin.json:7](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/extensions/brave/openclaw.plugin.json#L7)

  - file: [docs/tools/brave-search.md:19](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/docs/tools/brave-search.md#L19)
