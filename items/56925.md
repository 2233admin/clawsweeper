## Summary

Current main already handles the reported wrong-transcript case: `sessions_list` reads `sessionFile` and derives a correct `transcriptPath` for cross-agent sessions, and Control UI loads transcript content by `sessionKey` through `chat.history` instead of reconstructing a path from `sessionId` alone.

## Evidence

- **sessions_list now consumes sessionFile when deriving transcriptPath:** The tool reads `entry.sessionFile`, resolves agent-aware store-path options, and passes that metadata into `resolveSessionFilePath(...)` before returning the row.
  - file: [src/agents/tools/sessions-list-tool.ts:201](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/tools/sessions-list-tool.ts#L201)
- **Regression coverage matches the reported cross-agent mismatch:** Tests cover worker/non-default agent sessions and verify `transcriptPath` resolves correctly when `sessions.list.path` is relative, missing, non-string, `(multiple)`, or uses an `{agentId}` template.
  - file: [src/agents/tools/sessions.test.ts:457](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/tools/sessions.test.ts#L457)
- **Control UI reads transcript history by sessionKey:** The chat controller requests `chat.history` with the selected `sessionKey`; it is not deriving a transcript filename from `sessionId` on this path.
  - file: [ui/src/ui/controllers/chat.ts:152](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/ui/src/ui/controllers/chat.ts#L152)
- **Latest release contains the same relevant code paths:** `git diff --stat v2026.4.23^{}..HEAD -- src/agents/tools/sessions-list-tool.ts src/agents/tools/sessions.test.ts ui/src/ui/controllers/chat.ts` returned no diff, so the shipped `v2026.4.23` release already has the same implementation for this behavior.
  - command: `git diff --stat v2026.4.23^{}..HEAD -- src/agents/tools/sessions-list-tool.ts src/agents/tools/sessions.test.ts ui/src/ui/controllers/chat.ts`
  - sha: [9b18ae877018](https://github.com/openclaw/openclaw/commit/9b18ae877018a34933e27bb7b2986cb97e9602aa)

## Risks / Open Questions

- The tool still does not expose raw `sessionFile` on the returned row; the fix is via derived `transcriptPath` and `chat.history` behavior. If a future consumer specifically needs the raw field, that would be a separate enhancement request.
