## Summary

Keep open. Current main still intentionally flattens remote Control UI markdown images to alt text, tests and docs preserve that behavior, the Control UI CSP still blocks remote image loads, and open PR #41025 remains the implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review PR #41025 as the implementation candidate. If maintainers want remote images in Control UI chat, the fix should handle renderer behavior, CSP, docs, and the privacy/security policy together, likely through an explicit opt-in, allowlist, managed-media proxy, or similar deliberate design. If maintainers prefer the current no-remote-fetch policy, close this issue and related PRs with that product/security rationale.

## Risks / Open Questions

- A renderer-only change would not solve the visible behavior while the current Control UI CSP continues to exclude remote http(s) image sources.
- Allowing model-supplied remote image URLs inline would make the operator browser fetch third-party URLs, which has privacy and tracking implications and conflicts with the current documented hardening unless maintainers intentionally change that policy.
