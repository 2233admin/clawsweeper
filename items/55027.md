## Summary

Current main still does not implement true progressive Feishu block delivery. Feishu continues to disable core block streaming, drops non-card block payloads, and routes the remaining card-mode block path through Feishu Card Kit streaming rather than sending each emitted block as its own progressive reply. Related Feishu fixes address silent drops and duplicate streaming cards, but they do not supersede this narrower request.

## Best Possible Solution

Keep this open for a Feishu plugin-owned fix. The likely path is to add an explicit true block delivery mode for Feishu, map the channel/global block-streaming configuration to `disableBlockStreaming` like other channels, deliver runtime block payloads through normal Feishu messages or cards without relying on Card Kit playback, and update Feishu tests plus docs/config metadata so streaming-card preview and progressive block replies are separate behaviors.
