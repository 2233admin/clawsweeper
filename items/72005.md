## Summary

Keep PR #72005 open. Current main does not implement server-rendered outbound status footers, fabricated-footer stripping, or context-threshold warning state, and the linked feature request #71984 remains unresolved. The PR is not ready to close as implemented or obsolete; it is a plausible core change, but the supplied review comments identify issues that should be fixed before merge.

## Best Possible Solution

Keep the PR open and ask for a focused revision: apply the processed text to both direct and gateway send paths, avoid state mutation on dry-run, fix threshold persistence so skipped lower thresholds cannot re-fire, and add regression tests for direct, gateway, dry-run, and threshold-jump behavior. If maintainers decide this should be plugin-owned instead, the prerequisite would be a documented plugin hook/context contract that exposes the relevant session telemetry or session key consistently across all outbound send paths.
