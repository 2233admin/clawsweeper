## Summary

Keep PR #40578 open. Current main still implements memory-lancedb as one shared LanceDB memory pool with no agentId field, no agentScoping config, and no scoped store/search/delete path. The PR is a focused bundled-plugin implementation path, not implemented on main and not cleanly superseded by the closed related reports/PRs, though its known review blockers should be fixed before merge.

## Best Possible Solution

Keep the draft PR open as the active implementation path for bundled memory-lancedb per-agent isolation. The best maintainer path is to update the PR to use the existing hook/tool agentId context consistently for store, recall, duplicate detection, and forget; add an opt-in backward-compatible config and migration/schema story; address the bot review findings; and add regression coverage proving two-agent isolation plus legacy shared-memory visibility. If maintainers prefer a different isolation model, use this PR or a linked successor as the canonical implementation discussion rather than closing it as stale.
