## Summary

Current main still lacks per-agent scoping for the bundled memory-lancedb plugin, and this draft PR is a plausible implementation candidate rather than obsolete cleanup. It should stay open, but it is not merge-ready because the provided review context identifies unresolved correctness and data-scope risks in the PR implementation.

## Best Possible Solution

Keep this PR open as the remaining implementation candidate for bundled memory-lancedb per-agent isolation. The best path is to rebase onto current main, keep the fix inside extensions/memory-lancedb, use trusted ctx.agentId consistently from tool factories and hook contexts, add a backward-compatible config/manifest surface, fix LanceDB filtering and migration observability, and add regression coverage for agent A/B isolation, legacy shared memories, duplicate detection, forget/delete scoping, and default unscoped behavior.
