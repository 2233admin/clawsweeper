## Summary

Keep open. Current main still duplicates the subagent task in both the system prompt and first user message, so the linked bug #72019 is not implemented on main. PR #72053 is a focused attempt to fix the right code path, but it has an unresolved correctness concern: the proposed branch removes the exact task copy from the first user turn while current `buildSubagentSystemPrompt` collapses whitespace in the task, which can damage multiline/code-formatted tasks.

## Best Possible Solution

Keep PR #72053 open and ask for a small revision: dedupe the task from one model role while ensuring the child model still receives the assigned task verbatim, either by preserving exact task formatting in the system prompt or by choosing a single verbatim user-message source of truth. Add regression coverage for both no duplicate `[Subagent Task]` transmission and preservation of multiline/code task content.
