## Summary

Keep open. Current main no longer appears to sync every 1.5 seconds, but the core requested fix, append-only/incremental session indexing, is still not implemented. Changed session JSONL files are still rebuilt as whole-file entries and then passed through an index path that clears existing rows and rewrites all chunks for that session file. Related PR #50908 was closed unmerged, and the provided related-item context for #59577/#53660 also says the remaining incremental-session-sync work is still unresolved.

## Best Possible Solution

Keep #40919 open until a reviewed incremental session-sync implementation lands. The best path is a memory-core change that persists safe per-session sync state, indexes only newly appended transcript content when chunking/embedding contracts allow it, falls back to full reindex on compaction/shrink/model/chunking changes, and includes regression tests for append-only updates, unchanged transcripts, overlap boundaries, targeted post-compaction sync, and stale row cleanup. The existing thresholded reindex controls can remain as mitigation, but they do not close this bug.

## Risks / Open Questions

- Closing as implemented would hide a still-valid performance issue: main currently reduces how often session reindexing happens, but when a changed transcript is indexed it still rebuilds and rewrites the whole session file.
- A naive offset-only implementation could be incorrect around chunk overlap, transcript compaction/replacement, line-map changes, and FTS/vector cleanup; those are exactly the concerns raised in the issue discussion and related PR reviews.
