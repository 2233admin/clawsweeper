## Summary

Keep open. Current `main` no longer matches the issue body's claimed forwarding bug: the active-memory sub-agent still passes `toolsAllow`, `runEmbeddedPiAgent` forwards it, and `runEmbeddedAttempt` filters tools by that allowlist. `main` also has a post-release mitigation to skip bundled MCP runtime when the allowlist only contains local memory tools. But there is not enough evidence that the underlying active-memory failure is fixed end-to-end, especially for the non-PI provider path discussed in the thread, and the latest report says v2026.4.23 still times out. Closing as implemented or cannot-reproduce would be premature.
