## Summary

Keep open. Current main has some provider-auth env stripping for the default `openclaw acp client` bridge, but the ACPX harness path that launches Gemini still has no generic auth-contract metadata, no per-agent ambient auth scrubbing, and no fail-closed auth preflight hook. The docs still frame provider auth as host-owned harness state rather than documenting mixed-credential drift risks.

## Best Possible Solution

Keep the issue open and implement a generic ACP auth contract in the OpenClaw ACP control plane/plugin API, then wire the acpx Gemini route to scrub Gemini/GenAI auth env when OAuth is required, run a fail-closed preflight against Gemini CLI auth selection, and document mixed-credential host risks with focused tests around manager runtime input and acpx Gemini behavior.
