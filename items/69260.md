## Summary

Keep open. Current main has adjacent ACP hardening, especially default OpenClaw ACP bridge env scrubbing, but it does not implement the issue's requested ACP auth-contract surface for Gemini/acpx harness launches. The config/runtime contracts still lack expected auth mode, fallback policy, provider-auth env scrub policy, and fail-closed preflight hooks, and the docs do not cover the mixed OAuth/API-key host risk called out in the issue.

## Best Possible Solution

Keep the issue open and design an additive ACP auth-contract surface: typed config/runtime metadata for expected auth mode, fallback allowance, provider-auth env scrub lists, and fail-closed preflight hooks; wire it through the acpx backend; add Gemini-focused regression tests for ambient GEMINI/GOOGLE_GENAI env stripping and fail-closed behavior; then document mixed OAuth/API-key host guidance on the public ACP agents docs.
