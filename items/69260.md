## Summary

Keep open. Current main has adjacent hardening for the OpenClaw ACP bridge client and Codex ACP isolation, but it does not implement the issue's requested ACP auth-contract surface. The ACP runtime/config contracts still lack expected auth mode, fallback policy, provider-auth env scrub policy, and fail-closed preflight hooks; the bundled acpx plugin only supports command overrides plus MCP env injection, and there is no Gemini-specific scrub or oauth-personal guard in the OpenClaw acpx path.

## Best Possible Solution

Keep the issue open and design an additive ACP auth-contract surface: typed config/runtime metadata for expected auth mode, fallback allowance, provider-auth env scrub lists, and preflight hooks; wire it through the acpx backend; add Gemini-focused regression tests for ambient GEMINI/GOOGLE_GENAI env stripping and fail-closed behavior; then document mixed OAuth/API-key host guidance on the public ACP agents docs.
