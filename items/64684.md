## Summary

Keep this PR open. Current main still only writes a doctor hint on update.run failures and does not run a bounded non-interactive doctor summary, add doctorSummary to restart-sentinel payloads/formatting, or add doctor-backed diagnostics for config-reload restart-check failures. Related #19992 is closed for broader config safety work and #55347 remains a broader self-healing request, so neither supersedes this focused PR. Security pass found no CI, dependency, lockfile, package metadata, or third-party supply-chain changes in the provided PR file list; the execution-sensitive part is a local doctor subprocess that should remain fixed-argv, no-shell, and timeout-bounded.

## Best Possible Solution

Keep the PR open for maintainer review or author refresh. Rebase it onto current main, preserve the useful update.run failure doctor-summary and restart-sentinel formatter pieces if maintainers want this UX, align or remove the config-reload claim unless a safe immediate diagnostic path is added, and keep the subprocess hardening. If maintainers prefer the broader native self-healing design, fold this focused update-failure summary into #55347 rather than closing it as already implemented.
