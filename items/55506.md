## Summary

Keep open. Current `main` still has a session-keyed bootstrap-file cache and normal Telegram DM traffic reuses the same direct-message session, so updated `AGENTS.md`/`IDENTITY.md`/`BOOTSTRAP.md` can plausibly stay stale for an active DM session until that session rolls over or is reset. The stale `systemPromptReport.injectedWorkspaceFiles` observation is also consistent with current code because run reports are persisted and `/context` reuses the stored report.

## Evidence

  - file: [src/agents/bootstrap-cache.ts:3](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/bootstrap-cache.ts#L3)

- **Run path uses the session-key cache:** `resolveBootstrapFilesForRun` calls `getOrLoadBootstrapFiles(...)` whenever a run has `params.sessionKey`, so normal chat turns do not bypass the cache.
  - file: [src/agents/bootstrap-files.ts:229](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/bootstrap-files.ts#L229)
