## Summary

Keep PR #69486 open. Current main still has the branch-and-reappend transcript rewrite behavior without the PR's ancestry-aware on-disk cleanup, and raw transcript readers still scan every JSONL line, so abandoned rewrite entries can remain user-visible. A shipped doctor repair covers historical duplicated prompt-rewrite branches, but it is a manual repair path and does not supersede the runtime cleanup proposed by the PR. The PR also has unresolved review feedback about non-parent references such as label targetId, so the best path is maintainer review/fix rather than cleanup closure.

## Best Possible Solution

Keep PR #69486 open and require a maintainer-quality revision: implement runtime cleanup in `src/agents/pi-embedded-runner/transcript-rewrite.ts` that removes only entries provably abandoned by the current rewrite, preserves all surviving references including `parentId`, `label.targetId`, and other DAG metadata, and adds regression coverage for repeated rewrites, sibling branches, and label/reference integrity. The existing doctor repair should remain as historical cleanup, not as the runtime fix.
