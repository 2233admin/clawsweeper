## Summary

Keep this PR open. Current main has the shared outbound plumbing and iMessage text chunking, but the bundled iMessage plugin still does not implement the PRâ€™s central behavior: preserving `audioAsVoice` through iMessage-specific delivery and suppressing generic `<media:audio>` placeholder behavior for voice-intent audio sends. The PR is stale against the current `extensions/imessage` layout, but it still represents useful bug-fix work that should be rebased or rewritten rather than closed.

## Best Possible Solution

Keep the PR open and narrow/rebase it onto `extensions/imessage`. The best fix is an iMessage-owned outbound payload path, or equivalent adapter/runtime propagation, that sends `audioAsVoice` media without the generic `<media:audio>` placeholder, sends visible caption text separately when needed, preserves existing text chunking for text-only payloads, and adds iMessage-specific regression tests around `sendIMessageOutbound`, `sendMessageIMessage`, and shared delivery.
