## Summary

Keep #40352 open. Current main at 8ba82534 still has the reported Control UI stale model-selection paths: explicit Agents reload calls `loadConfig(state)` without clearing dirty config form state, `applyConfigSnapshot` still refuses to replace `configForm` while dirty, and the Agents model `<select>` still renders option entries without option-level `?selected` binding. The linked fix PRs remain open in the provided context rather than shipped on main.

## Best Possible Solution

Keep the issue open and land one canonical fix on main. The likely path is to add explicit reload semantics that clear or discard stale config form state for the Agents, Config, and Channels reload buttons, add option-level `?selected` bindings for the affected `<select>` renderers, and add regression coverage showing the configured primary model after initial render and after Reload Config.
