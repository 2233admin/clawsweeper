## Summary

Keep open. Current main still routes the Agents Reload Config button through the ordinary `loadConfig(state)` path, which does not clear dirty form state before applying the snapshot. The related closing PR #52948 is still open and unmerged, and the maintainer-labeled repair PR #72624 is also open, so this issue should not be closed before an implementation lands.

## Best Possible Solution

Keep this issue open and let maintainers review, land, or close the active implementation candidates. The best code path is to make explicit Reload Config discard stale local config state while passive refresh preserves unsaved edits, then either land the selected-option binding fix from #52948 or add browser regression coverage proving it is unnecessary. Regression coverage should exercise the Agents primary model dropdown after reload and page render.
