## Summary

Current `main` still has the mismatch this PR fixes: the streamed tool-call path builds an `incomplete` response resource but emits the terminal SSE event as `response.completed`, and the schema/tests on `main` still encode that old behavior. This PR remains relevant and should not be conservatively auto-closed.

## Evidence

That is the exact bug described by the PR and shows it is not implemented on current `main`.
