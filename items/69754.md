## Summary

Keep #69754 open. Current main improved generic approval prompt text, but Feishu still does not render the built-in exec approval interactive payload into Feishu buttons/cards, and the Feishu docs do not explain the approval fallback distinction. The related #58298 remains separate because it is about agent-authored message cards, while this issue is about the built-in approval prompt and fallback UX.

## Best Possible Solution

Keep the issue open and fix it in the Feishu approval surface: either render the existing approval interactive payload as Feishu interactive cards with working approval callbacks, or explicitly degrade the Feishu prompt to clear manual-approval copy that says buttons are unavailable on this surface and gives the exact /approve path. Add focused tests for Feishu reply rendering/fallback and update docs/channels/feishu.md plus the public docs page for Feishu approval behavior.
