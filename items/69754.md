## Summary

Keep #69754 open. Current main has generic same-chat `/approve` fallback behavior, but Feishu still lacks a native/shared approval rendering surface, Feishu outbound delivery does not render `ReplyPayload.interactive` approval buttons, and Feishu docs do not explain the button-unavailable/degraded-to-text path. The linked #58298 is related Feishu card work for agent-authored messages, not this built-in approval fallback UX.

## Best Possible Solution

Keep the issue open. The likely fix is to add Feishu-specific approval fallback handling around the shared approval payload path: either implement Feishu `sendPayload`/approval render hooks for buttons where supported, or explicitly rewrite degraded approval text for Feishu to say buttons are unavailable on this surface, show the exact `/approve ...` commands, and point to Web UI/terminal when in-channel approval is not valid. Update Feishu/exec-approval docs and add focused tests for the text fallback and any future Feishu native approval-card path.
