## Summary

Current `main` already normalizes Control UI model selections against the model catalog, so a stale provider does not produce `anthropic/gpt-5.4`; the same implementation is present in release `v2026.4.23`.

## Close Comment

Closing from a Codex review: this is already implemented on current `main`, and the same Control UI model-resolution code is present in release `v2026.4.23`. The picker path now resolves session/default models through the catalog before trusting a recorded provider (`ui/src/ui/chat-model-ref.ts:82`, `ui/src/ui/chat-model-select-state.ts:34`), and the UI tests cover the stale-provider case (`ui/src/ui/chat-model-select-state.test.ts:76`). That means a bare model like `gpt-5.4` is normalized to the unique catalog-qualified value instead of being rebuilt as a stale provider pair such as `anthropic/gpt-5.4`. If this still reproduces on a newer build, please open a fresh issue with the exact session payload and model catalog shown by the server.
