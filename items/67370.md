## Summary

Keep open. Current `main` added generic `sessions.delete`, but the requested event-driven session-lifecycle wiring is still missing from the webhook/plugin surfaces. Hook mappings only produce `wake` or `agent` actions, session plugin hooks are observation-only, and the documented plugin runtime delete path is limited to subagent sessions. That means the issue's core ask is not implemented well enough yet, and it is not a clean `clawhub` close because the plugin seam for arbitrary session cleanup is still incomplete.

## Evidence

- **Webhook mappings cannot express session-close actions:** The hook mapping schema and runtime only allow `action: "wake" | "agent"`; there is no `close`, `delete`, or lifecycle-maintenance action for external webhook payloads.
  - file: [src/config/zod-schema.hooks.ts:34](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/config/zod-schema.hooks.ts#L34)
