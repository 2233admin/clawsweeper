## Summary

Current main still appears vulnerable on the macOS doctor service-repair path. The install-plan merge can preserve custom service env only when existingEnvironment is supplied, but doctor rebuilds and stages/installs the LaunchAgent plan without passing the current plist EnvironmentVariables through that merge. The related systemd PR #66295 explicitly scoped out macOS launchd, so it does not resolve #45163.
