## Summary

Keep open. Current `main` still gates `xhigh` through provider-owned thinking profiles and does not implement the alias/proxy-ref behavior described in PR #41561, while the PR discussion also shows later stale merge damage unrelated to the original fix.

## Evidence

- **Current main still routes `xhigh` only through provider thinking hooks:** `resolveThinkingProfile()` adds `xhigh` only when `resolveProviderXHighThinking(...)` returns true for the selected provider/model, and `supportsXHighThinking()` is just a wrapper over that profile lookup. There is no fallback for alias providers or embedded upstream refs in current `main`.
