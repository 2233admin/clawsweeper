## Summary

Keep this issue open. Current main still has both reported memory-wiki lint defects: wikilink extraction scans raw rendered markdown with regexes, so fenced or inline literal [[...]] tokens can be treated as links, and broken-link validation only accepts extensionless relative paths and basenames, not page frontmatter titles or normalized aliases.

## Best Possible Solution

Keep this issue open and fix it in extensions/memory-wiki. Make extractWikiLinks markdown-aware by excluding frontmatter, fenced code, inline code, and comments before regex extraction, then replace lint's path/basename-only resolver with a normalized resolver that accepts relative path, basename, title, slugified title, id, and future aliases. Add focused regression tests for fenced and inline [[...]] literals, bridged/imported source fences containing reply tags, natural Unicode or special-character frontmatter-title links, and any relative Markdown-link behavior coordinated with #64696.
