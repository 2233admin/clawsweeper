## Summary

Keep open. Current main does not implement an `op`-specific mitigation for this report: exec providers only forward configured `args` and explicitly allowed env vars, with no automatic `OP_CACHE=false`/`--cache=false` handling. The 1Password docs example still shows `op read` without the Tahoe workaround. One part of the issue is already obsolete because exec providers already support prepending global CLI flags through the existing `args` array.

## Evidence

- **Exec resolver has no 1Password-specific cache mitigation:** `resolveExecRefs()` builds the child environment only from `passEnv` and provider `env`, then passes `providerConfig.args` directly into `runExecResolver()`. There is no branch that detects `op`, injects `OP_CACHE=false`, or prepends `--cache=false`.
