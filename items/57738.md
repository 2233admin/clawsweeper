## Summary

Keep open. Current `main` has Telegram polling hardening and unit coverage for the intended open-group path, but there is no issue-specific fix on `main`, and the reported symptom still matches current pre-session drop paths for Telegram group messages. That is not strong enough evidence to close conservatively.

## Evidence

- **Group messages can still be dropped before processing/session creation:** `shouldSkipGroupMessage(...)` returns early on group-policy/chat-allowlist checks and logs `reason: "not-allowed"`; later mention gating in `resolveTelegramInboundBody(...)` can also skip a group message before any session is recorded. That matches the report's "no group session / no directory entry" symptom.
