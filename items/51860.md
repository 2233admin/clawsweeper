## Summary

Keep open. Current `main` still implements macOS gateway service management as a per-user LaunchAgent only, with install/status/help text centered on `~/Library/LaunchAgents`, and there is no `--system`/LaunchDaemon install path. The issue’s core request is not implemented on `main`.

## Evidence

  - file: [src/daemon/launchd.ts:56](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/daemon/launchd.ts#L56)

  - file: [src/daemon/service.ts:172](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/daemon/service.ts#L172)
