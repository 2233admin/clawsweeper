## Summary

Keep open. Current main only partially overlaps with PR #33914: message hooks now carry sessionKey, but PluginHookAgentContext still does not expose sourceProvider, sender identity, groupId, or spawnedBy, and the main hook construction paths still pass only run/session/model/channel basics. This is a real plugin API gap, not a ClawHub-only plugin request, because external plugins cannot add missing core hook context fields themselves.

## Best Possible Solution

Keep PR #33914 open for maintainer review. The best path is an additive plugin SDK/context update that exposes the requested trust fields on PluginHookAgentContext, threads them consistently through agent and compaction hook emission sites, preserves the already-landed message-hook sessionKey work, and adds contract/docs/tests before landing.

## Risks / Open Questions

- The PR branch is partially stale because main independently added some sessionKey message-hook plumbing and refactored related hook surfaces; it likely needs a careful rebase or cherry-pick rather than a blind merge.
- These are security-sensitive trust inputs. If maintainers land this, propagation should be consistent across normal runs, queued runs, memory/compaction paths, reset hooks, and outbound hook paths to avoid phase-dependent policy decisions.
- This changes the public plugin SDK contract, so the final implementation should include docs, API baseline updates, and focused tests for null/undefined semantics and propagation paths.
