## Summary

Keep open. Current main no longer has a purely unconditional direct `config.patch` SIGUSR1 call, but the requested browser-profile behavior is still not implemented: the browser plugin registers `browser` config as restart-required, the reload planner classifies browser paths as gateway-restart paths, and `config.patch` still directly schedules SIGUSR1 in hot mode when the planner says restart is required. The linked PR #45624 addressed this exact issue but was closed unmerged, so it is not a canonical shipped fix.

## Evidence

- **config.patch restart gate still schedules direct SIGUSR1 for restart-required paths in hot mode:** `config.patch` now calls `shouldScheduleDirectConfigRestart(...)` instead of always scheduling, but that helper returns true when `gateway.reload.mode` is `hot` and the reload plan has `restartGateway`. That preserves the reported SIGUSR1 path for any path still classified as restart-required.
