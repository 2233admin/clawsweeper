## Summary

Keep open. Current `main` at `aa3a5f9ff7907c2a41d6259d24b529bda41f293a` still implements heartbeat as one fixed `every` interval plus one optional `activeHours` window; there is no `heartbeat.schedules[]` shape or per-time-block interval resolution on main.

## Evidence

- **Heartbeat config shape is still single-interval:** `AgentDefaultsConfig.heartbeat` exposes `every` and one `activeHours` object, with no `schedules` array or per-block interval fields.
  - file: [src/config/types.agent-defaults.ts:332](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/config/types.agent-defaults.ts#L332)

  - file: [src/infra/heartbeat-active-hours.ts:70](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/infra/heartbeat-active-hours.ts#L70)
