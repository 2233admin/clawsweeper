## Summary

Keep #68997 open. The report still matches current main and the latest published package: `task-registry.ts` still tries to `require()` `./task-registry-control.runtime.js`/`.ts` for ACP and subagent cancellation, current build entries do not emit that runtime as a stable root artifact, and `openclaw@2026.4.24` still ships the loader chunk without `dist/task-registry-control.runtime.js`.

## Best Possible Solution

Keep the issue open and fix the packaging boundary: emit `task-registry-control.runtime` as a stable root dist entry or replace the loader with a bundler-traceable runtime import, then add a postpublish/package verification check and a packaged CLI smoke that exercises `openclaw tasks cancel` through the real built artifact.
