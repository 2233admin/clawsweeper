## Summary

Keep open. Current `main` still has broken export-session template placeholders, the generator still expects contiguous `{{...}}` markers, and the build copies that broken template into `dist`, so the blank HTML export report remains valid on `main` and in `v2026.4.22`.

## Evidence

- **Current template still contains non-matching placeholder blocks:** `src/auto-reply/reply/export-html/template.html` has `MARKED_JS;`, `HIGHLIGHT_JS;`, and `JS;` inside nested block braces instead of contiguous `{{MARKED_JS}}`, `{{HIGHLIGHT_JS}}`, and `{{JS}}`, so the template no longer contains the strings the generator replaces.
  - file: [src/auto-reply/reply/export-html/template.html:61](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/auto-reply/reply/export-html/template.html#L61)
  - command: `nl -ba src/auto-reply/reply/export-html/template.html | sed -n '1,220p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Generator still relies on exact contiguous placeholder replacement:** `generateHtml()` still does `.replace("{{JS}}", templateJs)`, `.replace("{{MARKED_JS}}", markedJs)`, and `.replace("{{HIGHLIGHT_JS}}", hljsJs)`, which cannot match the current `template.html` contents.
  - file: [src/auto-reply/reply/commands-export-session.ts:91](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/auto-reply/reply/commands-export-session.ts#L91)
