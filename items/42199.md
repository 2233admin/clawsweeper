## Summary

Keep open. Current `main` has a newer, more conservative proxy design, but it does not implement this PR's behavior on the default `fetchWithSsrFGuard` path. Env-proxy routing is opt-in via `TRUSTED_ENV_PROXY`, and the tests explicitly assert that strict/default mode must ignore proxy env vars. There is partial helper-level auto-upgrade for some provider HTTP flows, but direct memory remote calls still invoke `fetchWithSsrFGuard` without that mode.
