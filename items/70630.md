## Summary

Keep open. Current `main` still implements the behavior this PR changes: when a Telegram turn ends with no visible final response, `bot-message-dispatch.ts` synthesizes a `NO_REPLY` fallback, and direct-session silent-reply policy rewrites that sentinel into visible filler text. Current tests on `main` explicitly expect that visible DM chatter, so this is not already implemented and not obsolete.
