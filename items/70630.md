## Summary

Keep PR #70630 open. Current main still contains the Telegram no-visible-response path that synthesizes a `NO_REPLY` fallback and still has a regression test asserting that a Telegram DM sends rewritten silent-reply chatter. The PR’s central fix is therefore not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open and adapt its small Telegram-owned fix onto current main: suppress the synthetic `NO_REPLY` fallback for no-visible Telegram direct turns, mark that path as handled so status reactions finish as done, preserve group/failure fallback behavior, and keep the updated regression test covering the DM silence path.
