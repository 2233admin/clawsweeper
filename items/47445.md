## Close Comment

Closing from a Codex review because current `main` already implements the Windows restart fix that matches this report. The restart path now detects stale gateway listeners and retries after terminating them in `src/cli/daemon-cli/lifecycle.ts:249`, and the Windows stale-listener cleanup explicitly waits for port release in `src/infra/restart-stale-pids.ts:518` so restart does not fall back into an `EADDRINUSE` loop. The regression is also covered by Windows-specific tests in `src/infra/restart-stale-pids.test.ts:923` and `src/cli/daemon-cli/lifecycle.test.ts:292`. This was shipped in `v2026.4.5`, where `CHANGELOG.md:1273` documents the Windows restart fix for stale listeners and inconclusive listener/argv probes. If you still hit this on a current release, please open a new issue with the exact OpenClaw version, install method, and Windows logs from `openclaw gateway status --deep` and the failed restart attempt.
