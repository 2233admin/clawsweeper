## Summary

Keep #69139 open. Current main still rejects root-level `paperclip` on `agent` params: `AgentParamsSchema` is strict, has no `paperclip` or generic metadata field, and the `agent` handler validates against it before dispatch. The PR remains the focused fix path, with live triage in the discussion corroborating the production failure and the linked closed issue #69137 matching the same schema mismatch.

## Best Possible Solution

Keep the PR open for maintainer review. The likely fix belongs in the gateway protocol contract: add an explicitly typed optional opaque metadata field accepted by `AgentParamsSchema`, keep `additionalProperties: false` for the rest of the request, add/keep the focused regression test in `src/gateway/server-methods/agent.test.ts`, and update generated protocol/docs artifacts if the gateway schema publication path requires it.
