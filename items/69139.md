## Summary

Keep PR #69139 open. Current main still does not accept root-level `paperclip` on `agent` params: the published `AgentParamsSchema` remains closed and has no `paperclip` or generic opaque metadata field, and the gateway validates that schema before dispatch. The related shipped Paperclip change only relaxed `wake` payloads, not `agent` requests. The proposed literal `paperclip` field may not be the best protocol shape, but the underlying compatibility bug is still real and not implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review, but do not merge it mechanically as-is without protocol judgment. The best path is to fix the compatibility bug through an additive, documented Gateway `agent` contract: either accept a generic opaque metadata field that external clients can use, or coordinate Paperclip to stop sending unsupported root metadata. If OpenClaw accepts metadata, update `AgentParamsSchema`, runtime types, docs/generated protocol surfaces, and add a regression test for a Paperclip-shaped `agent` request.
