## Summary

Keep open. Current `main` already matches the PR thread's correction on the memory side, but the skills-side change is not clearly redundant: `src/agents/skills/refresh.ts` now relies on `SKILL.md` target patterns instead of a `depth` cap, and OpenClaw pins `chokidar` `^5.0.0` while chokidar's current README says v4+ removed glob support and documents `depth` as the traversal limiter, so I cannot high-confidence say the original FD-exhaustion report is already resolved on `main`. ([github.com](https://github.com/paulmillr/chokidar))
