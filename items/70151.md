## Summary

Keep PR #70151 open. Current main still has the empty-turn terminal path that builds payloads from the attempt snapshot and immediately returns the incomplete-turn error; I did not find the proposed late assistant/session-file recovery poll or the regression test on main. The related local reports are adjacent empty-response/delivery issues but do not supersede this specific embedded-runner race.

## Best Possible Solution

Keep the PR open and review it as a focused bug fix. The likely best path is to land a bounded, abort-aware recovery read from the persisted session before the incomplete-turn error is surfaced, then build payloads from the refreshed assistant message and keep the regression test in `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts`.
