## Summary

Keep open. Current main still has the stale-snapshot failure mode described by PR #70151: the embedded runner builds reply payloads from the attempt's captured assistant message and, when that produces an incomplete-turn error, immediately surfaces the error without polling the persisted session file for a late assistant reply. The PR is a focused draft with a concrete regression test proposal, and no protected/maintainer-author rule blocks normal maintainer review.

## Best Possible Solution

Keep the PR open and adapt the narrow fix to current main: before terminal empty-turn failure, refresh from the persisted session or otherwise return a guaranteed-current assistant from the attempt path, build payloads from that refreshed assistant, and keep the stale-snapshot/late-assistant regression test in the embedded runner suite.
