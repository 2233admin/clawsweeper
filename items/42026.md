## Summary

Keep open. Current `main` still documents and implements a single Gateway process with embedded agent execution, lacks any per-agent runtime/control-plane split, and does not expose the requested per-agent lifecycle commands.

## Evidence

- **Architecture docs still describe one Gateway owning all messaging and agent traffic:** The current architecture doc says a single long-lived Gateway owns all messaging surfaces, and nodes connect to that same WS server. That contradicts the RFC's proposed separate per-agent runtime plane.
  - file: [docs/concepts/architecture.md:10](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/concepts/architecture.md#L10)
  - command: `nl -ba docs/concepts/architecture.md | sed -n '1,80p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Network docs still define the Gateway as one single long-running control-plane process:** `docs/network.md` says most operations flow through `openclaw gateway`, a single long-running process that owns channel connections and the WebSocket control plane, with one Gateway per host recommended.
  - file: [docs/network.md:17](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/network.md#L17)
  - command: `nl -ba docs/network.md | sed -n '1,80p'`
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Gateway protocol only supports operator and node roles:** The protocol doc describes the WS protocol as the single control plane + node transport and enumerates only `operator` and `node` roles. There is no runtime/agent-compute role or separate runtime registration protocol.
  - file: [docs/gateway/protocol.md:12](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/gateway/protocol.md#L12)
  - command: `nl -ba docs/gateway/protocol.md | sed -n '1,220p'`

  - file: [src/cli/program/register.agent.ts:90](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/cli/program/register.agent.ts#L90)

  - file: [docs/concepts/multi-agent.md:124](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/concepts/multi-agent.md#L124)
