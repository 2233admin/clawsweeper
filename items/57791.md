## Summary

Keep open. Current `main` does not implement hook-level `systemPrompt` support for the Gmail preset or generic hook agent mappings, and the config/docs/runtime still only support message text plus model/thinking overrides.

## Evidence

- **Hook mapping config has no systemPrompt field:** `HookMappingConfig` defines `messageTemplate`, `textTemplate`, `model`, `thinking`, and related routing fields, but no `systemPrompt` or equivalent hook-level prompt field.
  - file: [src/config/types.hooks.ts:11](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/config/types.hooks.ts#L11)
  - command: `sed -n '1,220p' src/config/types.hooks.ts`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Config schema would reject systemPrompt on hooks.mappings[]:** `HookMappingSchema` is strict and lists allowed keys for mappings; it includes `messageTemplate` and excludes `systemPrompt`, so adding that key in config is not supported on current `main`.
  - file: [src/config/zod-schema.hooks.ts:34](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/config/zod-schema.hooks.ts#L34)

- **Built-in Gmail preset still only sends a rendered message:** The built-in `gmail` preset in `hookPresetMappings` sets `messageTemplate` and `sessionKey`, but no system-prompt field.
  - file: [src/gateway/hooks-mapping.ts:72](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/gateway/hooks-mapping.ts#L72)

- **Hook runtime dispatch has no system-prompt channel:** `HookAgentPayload`/`HookAgentDispatchPayload` and the cron `agentTurn` payload only carry `message`, `model`, `thinking`, `timeoutSeconds`, and related metadata. There is no field forwarded for a dedicated system prompt.
  - file: [src/gateway/hooks.ts:213](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/gateway/hooks.ts#L213)
