## Summary

Keep open. Current `main` still implements regex-based inline TTS directive parsing, and both tests and docs on `main` explicitly preserve inline `[[tts:...]]` and inline `[[tts:text]]...[[/tts:text]]` behavior. This PR's standalone-line parsing change is not implemented on `main`, and the report is coherent enough that it should not be conservatively closed.
