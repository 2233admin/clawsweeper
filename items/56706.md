## Summary

Keep open. Current `main` does not implement the standalone-line-only TTS contract from this PR: `src/tts/directives.ts` still uses broad case-insensitive regex parsing for inline `[[tts:...]]` directives and inline `[[tts:text]]...[[/tts:text]]` blocks, and the shipped tests/docs still encode that inline behavior. The PR also mixes unrelated Slack/config changes; the Slack `ignoreSelf: false` piece already landed independently on `main`, while the PR discussion still contains unresolved review feedback about TTS parsing regressions.

## Evidence
