## Summary

Keep open. Current main still reuses a single unowned GitHub Copilot API-token cache entry across profiles, and this PR remains a plausible fix candidate tied to the still-open paired issue #43658.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Port the cache-ownership check to `src/agents/github-copilot-token.ts`, preserve the `openclaw/plugin-sdk/github-copilot-token` export, add regression tests for different GitHub tokens and legacy cache entries in the current test locations, and review it together with the paired open issue #43658.
