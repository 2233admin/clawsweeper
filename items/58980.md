## Summary

Keep open. PR #58980 is not obsolete on current main: the proxy wrapper still deletes only `reasoning_effort` and returns when thinking is `off` or unset, leaving a preexisting nested `reasoning: { effort: "none" }` payload intact. Existing main has reactive retry handling for reasoning-required errors and some OpenRouter tests, but it does not cover or implement the PR’s pre-request cleanup for nested upstream reasoning objects.

## Best Possible Solution

Keep the PR open or reimplement its minimal fix on main: delete `payloadObj.reasoning` in `normalizeProxyReasoningPayload` when thinking is `off` or undefined, and add focused regression coverage for preexisting nested `reasoning: { effort: "none" }` on OpenRouter and Kilocode. Keep the existing retry fallback as defense-in-depth, but do not treat it as a replacement for pre-request payload normalization.
