## Summary

Keep open. Current `main` still matches both reported behaviors: the managed gateway install path can carry durable `OPENCLAW_GATEWAY_TOKEN` values into the service environment, and macOS LaunchAgent generation still writes environment variables inline into the plist; separately, `openclaw channels list` still unconditionally fetches provider usage after printing auth profiles and logs fetch errors instead of treating a no-chat-channels setup as a clean empty list.

## Evidence

- **Gateway install still merges durable service env into the managed service plan:** `collectDurableServiceEnvVars` reads state-dir `.env` and config env vars, and `buildGatewayInstallEnvironment` injects that managed environment into the gateway install plan. A durable `OPENCLAW_GATEWAY_TOKEN` therefore still flows into service metadata on current `main`.
