## Summary

Keep open. Current main still does not implement the PR's central behavior: `parseCliJsonl()` only handles Claude `type:"result"` specially and otherwise collects `parsed.item.text`, with no handling for `task_complete.last_agent_message`, `phase:"final_answer"`, or suppression of explicit `commentary` items. The PR remains a plausible focused fix for #59918, though its current patch should address the unresolved bot review about preserving readable fallback text for commentary-only successful runs.

## Best Possible Solution

Keep the PR open and ask for a small revision: make `parseCliJsonl()` prefer non-empty `task_complete.last_agent_message`, then explicit `final_answer` text, while preserving a human-readable fallback for legacy/commentary-only successful JSONL outputs and adding focused unit coverage in `src/agents/cli-output.test.ts`. The adjacent phase-aware fixes in #61463/#61481 should remain separate; this parser-specific gap is still real on current main.
