## Summary

Do not close #69197. Current main still lacks any active-poll suppression around `maybeNotifyOnExit()`, while the PR is a focused, plausible bug fix with concrete reproduction, live validation, and regression tests. It needs review/revision rather than conservative cleanup closure, especially because the latest review thread raises a real overlapping-poll concern with the boolean approach.

## Best Possible Solution

Keep #69197 open. The best path is to revise the active-poll suppression to a small refcount/token mechanism, keep the existing regression tests, add coverage for overlapping polls if the refcount is adopted, and then run the changed agent test/typecheck lane before maintainers consider landing it. Broader async completion routing problems should remain tracked by the related canonical issues rather than blocking this focused runtime race fix.
