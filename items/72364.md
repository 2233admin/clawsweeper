## Summary

Keep #72364 open. Current main still has a pre-run budget compaction path that calls the compaction runtime directly rather than routing through `/compact`, and the direct compaction implementation still performs JSONL truncation after post-compaction side effects/hooks rather than proving it is inside the same visible lifecycle envelope. The reporter's follow-up narrows the issue to that truncation/lifecycle ordering, and the current code does not contain a clear shipped fix for it.

## Best Possible Solution

Keep the issue open and treat it as a core WebChat/compaction lifecycle bug. The likely fix should either route automatic budget compaction through the same user-visible command/run lifecycle as `/compact`, or more narrowly move/gate `truncateSessionAfterCompaction` so the JSONL rewrite completes before the compaction lifecycle becomes idle and before any history refresh can observe a torn transition. Add focused regression coverage around `truncateAfterCompaction: true`, active WebChat history refresh/deferred reload behavior, and the preflight budget compaction path.
