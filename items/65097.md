## Summary

Keep this issue open. Current main still does not implement a Dreaming diary language or locale setting: the memory-core narrative prompt remains English-only, the narrative generation API and phase call sites only carry timezone, the runtime config contract/resolver omit language and locale, the strict memory-core manifest schema does not accept those keys, and the public Dreaming docs expose no language control. Related #69354 was closed as a duplicate of this same remaining feature gap, not as an implementation.

## Best Possible Solution

Keep this issue open and implement the feature in `memory-core`: add a documented `dreaming.language` or `dreaming.locale` config contract, decide whether `auto` is supported, resolve the hint in the memory-host config path, thread it into the Dream Diary prompt/subagent run and request-scoped fallback behavior, update `extensions/memory-core/openclaw.plugin.json`, update public Dreaming and memory config docs, refresh generated config metadata if required, and add focused resolver/narrative tests.
