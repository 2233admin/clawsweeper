## Summary

Keep open. Current `main` still implements async completion delivery as a multi-path/fallback flow rather than the proposed strict single-path rule. The code and docs explicitly say subagent/task/media completions may try direct delivery and then fall back to queue/session wake, so #51621 is not clearly implemented on `aa3a5f9ff7907c2a41d6259d24b529bda41f293a`.
