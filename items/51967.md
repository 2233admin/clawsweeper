## Summary

Current `main` already fixes the zero-duration false positive that motivated this PR, but it does not implement the PR's broader `> 0` behavior. The repo now intentionally treats `0` as unset while preserving negative non-zero durations as poll intent so they reach poll validation, so I do not have high-confidence evidence to close this PR as fully implemented on `main`.

## Evidence

- **Current intent in production code:** `hasPollCreationParams` treats zero-valued numeric poll params as absent (`value !== 0`) but keeps any other finite non-zero number as poll intent, with an explicit comment that invalid durations should still hit the poll-only validation path.
