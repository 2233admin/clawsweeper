## Summary

Keep open: current `main` still contains the session-scoped L1 `bootstrap-cache` layer, still routes bootstrap reads through it whenever `sessionKey` is present, and only clears that cache on specific reset/rollover paths. The staleness class described by PR #43901 is therefore still plausible on current `main`, and this is not already implemented there.
