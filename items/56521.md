## Summary

Keep open. Current `main` does not implement cron failure alerts as an agent-triggering event for the main session. The failure-alert contract only supports outbound `announce`/`webhook` delivery, and the Gateway wires failure alerts to outbound delivery instead of `enqueueSystemEvent`/heartbeat wake. Main-session agent-triggering exists for regular cron `systemEvent` jobs, but not for `failureAlert`.

## Evidence

- **Failure-alert type has no agent-turn/session routing:** `CronFailureAlert` only exposes `after`, `channel`, `to`, `cooldownMs`, `mode`, and `accountId`; `mode` is limited to `"announce" | "webhook"` with no `agent-turn` or session field.
