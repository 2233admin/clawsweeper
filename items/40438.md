## Summary

Keep this PR open. Current main still lacks a configurable directory exclusion contract for memory indexing, so the central feature remains useful. The PR should not merge as-is because it is stale against the relocated memory implementation and includes an unrelated network preloader file.

## Best Possible Solution

Keep this PR open and salvage the focused feature: rebase onto current main, remove `ipv4-fix.cjs`, add an additive `memorySearch.excludeDirs` or broader exclude-pattern contract across schema/types/help/generated docs, thread it through `resolveMemorySearchConfig`, `packages/memory-host-sdk/src/host/internal.ts`, and `extensions/memory-core` sync/status/watch/reindex paths, skip excluded roots as well as descendants, and either implement equivalent QMD semantics or document QMD scope explicitly with tests.
