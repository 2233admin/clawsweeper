## Summary

Keep #40438 open. Current main still lacks any memorySearch excludeDirs/excludePatterns contract, and the builtin memory indexer still recursively walks configured extraPaths without directory-basename filtering. The PR targets a real problem from #6109, but it needs a rebase and cleanup because current main moved the relevant code into the memory-host SDK and memory-core plugin, and the PR contains unrelated ipv4-fix.cjs plus an unresolved extraPaths-root bypass concern.

## Best Possible Solution

Keep the PR open and salvage the feature: rebase onto current main, remove ipv4-fix.cjs, add an agent-scoped memorySearch exclusion contract to schema/types/help/docs/generated config, thread the resolved exclusions through src/agents/memory-search.ts into the memory-core settings/status/reindex hash/watcher paths, implement filtering in packages/memory-host-sdk/src/host/internal.ts before walking both default memory roots and extraPaths roots, and add focused tests for defaults, user-added exclusions, default memory dir exclusions, and excluded extraPaths roots.
