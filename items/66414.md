## Risks / Open Questions

- Merging the current patch as-is can still re-enable send too early or leave the UI counter wrong because pending-count deltas are computed from stale render props.
- Merging the current patch as-is can still drop attachments because async FileReader callbacks append to captured attachment arrays rather than the latest app state.
- The early return in handleSendChat can block messageOverride and local command flows that do not depend on in-flight attachments, such as new-session, compact, BTW, and local slash-command actions.
- The PR is behind current main’s chat control refactor and needs a rebase/rework before it can be evaluated cleanly.
