## Summary

Keep open. Current `main` still shows the reported gateway lookup and Control UI attachment-send gaps, and the PR discussion already surfaces unresolved race/state problems in the proposed fix.

## Evidence

- **Current gateway lookup still misses provider-qualified catalog ids:** `resolveGatewayModelSupportsImages()` still looks up the catalog with `entry.id === params.model` plus optional provider equality, so a runtime model like `gpt-5.4` will not match a catalog row keyed as `clipro-codex/gpt-5.4`.

- **No regression test on main for provider-qualified lookup:** The current `resolveGatewayModelSupportsImages` tests cover stale Foundry and claude-cli metadata only; there is no test for provider-qualified catalog ids matching a bare runtime model name.
  - file: [src/gateway/session-utils.test.ts:1136](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/session-utils.test.ts#L1136)
