## Summary

Keep open. Current `main` already bootstraps the global undici env-proxy dispatcher for normal CLI startup, but the Discord gateway path still does not fall back to ambient proxy env vars for its WebSocket agent and still uses default `ProxyAgent` / `EnvHttpProxyAgent` construction without the proxy-tunnel hardening described in PR #47446. That means the request is not implemented on `main`, so there is not strong evidence to close this PR as obsolete.
