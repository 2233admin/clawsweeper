## Summary

Keep open. Current main has privacy-safe OpenTelemetry traceparent propagation and static provider request headers, but it does not implement the PR’s configurable per-run/session x-session-* header feature. The existing provider wrapper hook also does not receive runId or sessionKey, so this is still a live core/API design decision rather than something clearly obsolete or ClawHub-only.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either a privacy-reviewed core implementation for opt-in dynamic provider headers, or a narrower plugin SDK extension that exposes safe run/session correlation data so Helicone/DodoRouter-style integrations can live outside core.
