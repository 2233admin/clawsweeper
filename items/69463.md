## Summary

Keep open. Current main does not implement this PR's `diagnostics.sessionTracing` config or dynamic `x-session-id` / `x-session-name` injection for outgoing LLM requests. Main has static provider headers, OpenTelemetry `traceparent` propagation, and provider-owned hook seams, but those do not fully replace the PR's all-provider, opt-in run/session correlation header behavior. The supplied PR discussion is also positive, with Greptile calling the implementation safe aside from P2 cleanup suggestions, so there is no strong conservative close reason.

## Best Possible Solution

Keep the PR open for maintainer product/API review. The best path is either to land a revised, documented, opt-in core or SDK-backed mechanism for dynamic LLM request correlation headers, or to extend the plugin/provider hook context so an external observability plugin can attach run/session headers across providers without one-off core behavior.
