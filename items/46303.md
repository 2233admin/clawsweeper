## Summary

Keep #46303 open. Current main still does not implement the PR's central restart-time behavior: inbound debounce buffers remain closure-local, followup queues have no all-queue drain waiter, and the SIGUSR1 run loop only drains active command/embedded work before closing the server. The provided discussion also shows the PR is a real focused bug fix, but the branch still needs maintainer-quality changes before landing.

## Best Possible Solution

Keep #46303 open, but ask for a smaller additive fix: add a controlled lifecycle-safe debouncer registration/flush seam that preserves the existing public `flushKey` contract, flush active inbound debouncers before teardown with bounded timing, wait for followup queues using the same pending predicate as the real drain loop after active turns can complete, and cover SIGUSR1 ordering plus channel teardown races with targeted tests.
