## Summary

Current `main` does not implement an `agents.defaults.workspace.extraFiles` config surface, and the bootstrap cache still returns the cached `WorkspaceBootstrapFile[]` by reference. Main does have a documented bundled `bootstrap-extra-files` hook that provides a declarative workaround, but that is a different config path and does not eliminate the underlying shared-array mutation hazard for custom `agent:bootstrap` hooks.

## Evidence

- **No `workspace.extraFiles` in agent defaults type:** `AgentDefaultsConfig` defines `workspace`, `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars`, but no nested `workspace.extraFiles` or equivalent agent-default field.

## Risks / Open Questions

- A maintainer could decide the bundled `bootstrap-extra-files` hook is sufficient for the user-facing need, but current code and schema do not show the requested first-class config key, so closing as `implemented_on_main` would be hard to defend.
- The cache-reference bug remains for custom `agent:bootstrap` hooks unless they replace the array instead of mutating it.
