## Summary

Keep open. Current `main` still lacks the PR's two substantive Windows fixes: `scripts/pnpm-runner.mjs` does not preserve a Windows `npm_execpath` that points at `pnpm.exe`, and `extensions/lobster/src/lobster-runner.ts` still hard-imports `@clawdbot/lobster/core` with no installed-package fallback. The PR discussion also ties this work to follow-up Windows regression issue #69456, so this does not have strong evidence for cleanup closure.

## Evidence

- **Current pnpm runner still falls back to `pnpm.cmd` on Windows:** `resolvePnpmRunner` only honors `npm_execpath` when it is node-runnable; otherwise every Windows path falls through to `cmd.exe /c pnpm.cmd ...`. There is no `.exe` direct-exec branch on current `main`.
