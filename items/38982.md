## Summary

Keep #38982 open. Current main has startup light/dark TUI palette support through `OPENCLAW_THEME` and `COLORFGBG`, but it still does not implement this PR's central runtime behavior: no `/theme` command, no `/theme list`, no runtime theme application API, no searchable theme picker with live preview, and no cancel/revert or commit flow. The overlapping #44382 remains only an open related PR in the provided context, not a maintainer-designated canonical replacement.

## Best Possible Solution

Keep #38982 open unless a maintainer explicitly designates #44382 or another PR as canonical. The best product path is to coordinate the overlapping TUI theming PRs and land one focused implementation that adds a runtime theme API, registers `/theme`, supports `/theme list`, opens a searchable picker with live preview, commits on select, reverts on cancel, repaints all TUI surfaces consistently, updates the public TUI docs at `https://docs.openclaw.ai/web/tui`, and adds targeted command/theme tests.
