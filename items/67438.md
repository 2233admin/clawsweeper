## Summary

Keep open. Current main still sends Feishu setup through the QR scan-to-create path and prints a compact terminal QR code. The linked fix candidate, #67450, was closed unmerged, and the current tests/docs do not show a shipped scan-friendly fallback for this Feishu onboarding path.

## Best Possible Solution

Keep this issue open and fix it in the Feishu plugin-owned setup flow. A conservative patch should make printQrCode scan-friendlier, likely by using small: false and/or adding a PNG/browser fallback using the existing QR image helpers, then add a focused regression test for the render option/fallback and manually verify the Windows Feishu/Lark scan path.
