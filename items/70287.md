## Summary

Keep #70287 open. Current main still has the exact MS Teams search behavior the PR is trying to fix: `searchMessagesMSTeams` builds `/messages?$search=...`, sends `ConsistencyLevel: eventual`, and the existing unit test still asserts `$search` is present. The PR is a focused, still-useful bug fix for the app-only Graph permissions failure reported in #70127, with reviewer feedback already addressed in the PR discussion.

## Best Possible Solution

Keep the PR open for maintainer review or landing. The best path is to rebase or otherwise validate #70287 against current main, keep the list-and-local-filter approach with `$top=50` pagination and sender `$filter` pushdown, and retain the expanded regression tests that assert `$search` is no longer sent for `msteams:search`.
