## Summary

Keep open. Current `main` still reproduces the catalog-miss behavior from #67759, so this is not `implemented_on_main` or `cannot_reproduce`. This PR also is not a safe cleanup close: it carries unrelated `extensions/memory-core` changes with a reviewed P1 regression risk, and its Qwen name-pattern heuristic is narrower than the issue discussion's explicit-provider-config fallback.
