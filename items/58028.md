## Summary

Keep #58028 open. Current main still has `/new`, `/reset`, and `/compact`, but no core `/resume` slash command, no resume handler, no session-history metadata field matching the proposed design, and no docs entry. Reset currently rotates to a new session and archives the old transcript; existing session list/history tools and UI compaction restore do not reload a previous JSONL session into the active chat context. The directly linked PR #36481 was closed unmerged, and the related #51889 restore PR appears to be an open implementation path rather than shipped behavior.

## Best Possible Solution

Keep the feature request open and handle it as first-party session infrastructure work. A good implementation should register `/resume`, add a core handler that lists eligible active/archived sessions with safe labels and timestamps, validates sender/session permissions, restores or forks the selected transcript without corrupting `sessions.json`, and updates slash-command docs plus focused session-store/command tests. Maintainers should coordinate with the related restore/resume PR work (#51889 or a successor) rather than closing this issue now.
