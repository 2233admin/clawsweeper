## Summary

Keep PR #69270 open. Current main still lacks several central pieces from the PR: the shared hook message-provider resolver is absent, subscribe-time compaction hooks and reset hooks still omit messageProvider, runEmbeddedAttempt still does not pass a provider into subscribeEmbeddedPiSession, and incrementCompactionCount still ignores tokensAfter: 0. The PR also has useful linked reports and review follow-ups, but the latest PR discussion includes a remaining concern about preserving explicit non-gateway/synthetic providers, so the best maintainer path is review/request changes rather than cleanup-close.

## Best Possible Solution

Keep the PR open and have maintainers review or request changes. The best implementation is a focused shared provider resolver that normalizes known channel aliases, preserves explicit intentional internal/synthetic providers where existing behavior depends on them, infers providers only from real channel-scoped session-key shapes, threads that provider through subscribe compaction, engine-owned compaction, command reset, and gateway reset contexts, and fixes compaction accounting for tokensAfter: 0 while clearing stale cache/cost buckets. Regression tests should cover zero-token compaction, maxLines/manual compaction cache clearing, mixed-case reset channels, channel-scoped inference, non-channel direct/custom keys, and explicit node/heartbeat/webchat providers.
