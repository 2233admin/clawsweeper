## Summary

Keep #70344 open. Current main still does not implement the PR's missing-metadata tool-use suppression or the proposed sessions_send session-local behavior, but the PR is not merge-ready as-is because the reviewed diff has credible route-persistence and sanitizer-regression concerns.

## Best Possible Solution

Keep the item open but ask for a narrower, corrected patch: add phase-independent tool-use suppression with regression coverage; decide the sessions_send product boundary explicitly; if sessions_send should be session-local, suppress current-turn announce/external delivery without mutating persisted lastChannel/lastTo; and either remove the self-instruction sanitizer or replace it with a much stricter, reviewed sanitizer after addressing the hardcoded-name and over-stripping review comments.
