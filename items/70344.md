## Summary

Keep #70344 open. Current main does not include the PR’s broader tool-use-without-phase suppression or its sessions_send behavior changes, and the related PRs only cover overlapping slices. The PR is still plausibly useful, but it should not be closed as already implemented or merged as-is because the review discussion identifies real regression risks in the proposed route mutation and sanitizer heuristic.

## Best Possible Solution

Keep the PR open, but ask for a safer, narrower update before maintainer review: add targeted unphased tool-use suppression with regression coverage; handle sessions_send as session-local for the current turn without overwriting established external route state or removing intentional A2A behavior unless maintainers explicitly want that product change; remove the hardcoded sanitizer artifact and tighten any self-instruction stripping heuristic. Coordinate overlap with #70372 for provider-specific thinking narration rather than closing this as a duplicate.
