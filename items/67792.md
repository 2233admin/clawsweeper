## Summary

Keep open. Current `main` still renders Telegram model-menu button text from the bare model id, so configured OpenRouter refs like `openrouter/openai/gpt-5.4-mini` would display as `openai/gpt-5.4-mini` instead of the full ref. I did not find evidence of a landed fix for #67792 on `main`.

## Evidence

- **Telegram menu still drops provider in button text:** `buildModelsKeyboard()` derives the visible label from `modelNames?.get(`${provider}/${model}`) ?? model`, then renders that text directly. Without a catalog display-name override, the button shows only `model`, not `provider/model`.
  - file: [extensions/telegram/src/model-buttons.ts:220](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/extensions/telegram/src/model-buttons.ts#L220)
  - command: `nl -ba extensions/telegram/src/model-buttons.ts | sed -n '197,223p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Configured refs are split into provider + nested model id before reaching Telegram UI:** `buildModelsProviderData()` resolves raw configured refs with `resolveModelRefFromString()` and stores them as separate `provider` and `model` parts in `byProvider`. For `openrouter/openai/gpt-5.4-mini`, that yields provider `openrouter` and model `openai/gpt-5.4-mini`.
  - file: [src/auto-reply/reply/commands-models.ts:97](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/auto-reply/reply/commands-models.ts#L97)
  - command: `nl -ba src/auto-reply/reply/commands-models.ts | sed -n '97,119p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Model ref parser preserves only the first segment as provider:** `parseModelRef()` splits on the first `/`: text before the slash becomes `provider`, everything after becomes `model`. That matches the issue report's root-cause analysis for OpenRouter-style ids.
  - file: [src/agents/model-selection-normalize.ts:87](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/agents/model-selection-normalize.ts#L87)
  - command: `nl -ba src/agents/model-selection-normalize.ts | sed -n '75,96p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **OpenClaw docs require full provider-prefixed refs for OpenRouter-style ids:** The docs state model refs are parsed on the first `/`, and if the model id itself contains `/` you must include the provider prefix, with `openrouter/...` as the example shape. That makes the Telegram menu's provider-less label inconsistent with documented model refs.
