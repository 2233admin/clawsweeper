## Summary

Keep open. Current `main` only binds `senderIsOwner` to the loopback bearer token; it still takes `sessionKey`, `accountId`, and `messageChannel` from mutable request headers/env, so the PR’s per-backend token-bound scope design is not implemented on `main`. The PR also has unresolved review findings, including a P1 close/restart race in the new runtime-clearing approach and a P2 token-revocation leak on cleanup failure, so this is not a conservative maintainer-close candidate.
