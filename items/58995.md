## Summary

Keep PR #58995 open. Current main still uses the ASCII-only normalizeHyphenSlug and normalizeAtHashSlug filters that this PR targets, the linked #58932 group display-name bug still follows the normalizeHyphenSlug path, and the Unicode regression tests are not present on main. The PR is overlapping with #58942/#59068 and has actionable review feedback about combining marks, but there is no maintainer-selected canonical replacement that makes this PR safe to close as cleanup.

## Best Possible Solution

Keep the PR open for maintainer/contributor action. The best fix is a narrow shared string-normalization update that preserves Unicode letters, decimal numbers, and combining marks, normalizes consistently where needed, and adds regression tests for CJK, Cyrillic, Arabic/diacritic, Devanagari or other combining-mark scripts, plus existing ASCII behavior. Maintainers should consolidate #58942/#58995/#59068 against #58932 before landing.
