## Summary

Keep open. Current main at c1383680405879a3c3f43dceb20a694f54d3c13c still has the reported skills discovery truncation: it slices candidate directories by `maxSkillsLoadedPerSource` before checking whether those directories actually contain `SKILL.md`, so valid later skills can still be skipped. The PR is also mixed-scope, so it should be split/refined rather than closed.
