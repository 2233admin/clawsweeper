## Summary

Keep open. Current `main` does not implement `tools.sessions.visibility=allowlist` or any companion `tools.sessions.allowlist` setting, and the existing visibility logic still treats same-agent access as a coarse `self|tree|agent|all` switch with no per-session pattern filtering.

## Evidence

- **Runtime visibility type and parser only allow four modes:** `SessionToolsVisibility` is defined as `"self" | "tree" | "agent" | "all"`, and `resolveSessionToolsVisibility()` accepts only those values before defaulting to `tree`. There is no `allowlist` branch.
  - file: [src/plugin-sdk/session-visibility.ts:20](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugin-sdk/session-visibility.ts#L20)
  - command: `nl -ba src/plugin-sdk/session-visibility.ts | sed -n '1,80p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Access checker has no same-agent filtering beyond self/tree:** `createSessionVisibilityChecker()` blocks cross-agent access unless visibility is `all`, blocks same-agent access only for `self` and `tree`, and otherwise returns `{ allowed: true }`. That means `agent` remains all-or-nothing for the whole agent, with no session-key allowlist path.
  - file: [src/plugin-sdk/session-visibility.ts:187](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugin-sdk/session-visibility.ts#L187)

- **Public config types expose only the existing four values:** `tools.sessions.visibility` is documented in the TypeScript config surface with only `self`, `tree`, `agent`, and `all`; there is no `tools.sessions.allowlist` field.
