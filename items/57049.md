## Summary

Current `main` treats WeChat support as external plugin-owned behavior, not OpenClaw core. A `mp.weixin.qq.com` captcha/article-access workaround would belong in the external WeChat plugin or another external web/plugin integration rather than this repo.

## Evidence

- **WeChat support is explicitly external:** `docs/channels/wechat.md` states OpenClaw connects to WeChat through Tencent’s external `@tencent-weixin/openclaw-weixin` plugin and marks the channel status as `external plugin`.
  - file: [docs/channels/wechat.md:10](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/channels/wechat.md#L10)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **WeChat-specific runtime is not owned by core:** The same doc says the WeChat code does not live in the core repo and that WeChat login, Tencent iLink API calls, media upload/download, context tokens, and account monitoring are owned by the external plugin.
  - file: [docs/channels/wechat.md:27](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/channels/wechat.md#L27)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Vision keeps optional capability in plugins:** `VISION.md` says core stays lean, optional capability should usually ship as plugins, and new skills should go to ClawHub first rather than core by default.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/VISION.md#L54)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Community capabilities belong in external plugins:** `docs/plugins/community.md` defines community plugins as third-party packages for new channels/tools/providers, published on ClawHub or npm and installed separately.
  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/plugins/community.md#L9)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **No bundled WeChat implementation exists in `extensions/`:** A repo search found no `extensions/*wechat*` or `extensions/*weixin*` package; current main only contains docs/tests referencing `openclaw-weixin`, not an in-repo WeChat runtime to patch here.
  - command: `find extensions -maxdepth 2 \( -iname '*weixin*' -o -iname '*wechat*' \)`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)

## Risks / Open Questions

- If OpenClaw later decides to bring WeChat article retrieval into a first-party in-repo plugin, this scope decision would need to be revisited.

## Close Comment

This looks out of scope for the OpenClaw core repo after a Codex review.

- `docs/channels/wechat.md:10` and `docs/channels/wechat.md:27` say WeChat support is provided by the external `@tencent-weixin/openclaw-weixin` plugin, and that the WeChat-specific runtime is owned outside this repo.
- `VISION.md:54` says optional capability should usually ship as plugins, with new skills/plugins kept out of core by default unless there is a strong reason.
- `docs/plugins/community.md:9` points new third-party capabilities to external plugins published on ClawHub or npm.
- Current `main` (`3a64aa49a9014518fa1977fbc023f3e0b978ae5d`) has no bundled WeChat implementation under `extensions/`, so there is no core-owned `mp.weixin.qq.com` article-access path here to fix.

If this is still worth pursuing, the right home is the external WeChat plugin or another external web-access plugin, not `openclaw/openclaw` core.
