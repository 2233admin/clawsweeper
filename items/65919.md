## Summary

Current main still has the active-run typing TTL gap. The run-side keepalive loop continues to send channel typing callbacks, but those periodic ticks do not refresh the run-side 2 minute TTL, so configured typing can still be cleaned up before run completion and dispatch idle during a healthy long quiet run. The issue also has a recent Discord reproduction and an open targeted PR candidate, so it should remain open for maintainer review.

## Best Possible Solution

Keep this issue open and review/land PR #71828 or an equivalent lifecycle-owned active-run presence lease. The fix should keep configured typing alive through healthy long quiet phases by refreshing the internal run-side lease while the run is active, preserve stale-presence cleanup, avoid new default disclosure/progress surfaces, avoid increasing channel/provider keepalive cadence, and stop after both run completion and dispatch idle.
