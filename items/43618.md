## Summary

Keep open. Current main still does not implement the PR's central capability: a live queue diagnostics snapshot exposed through `diagnostics.queue` and `openclaw gateway diagnostics queue`. The existing stability diagnostics recorder is adjacent but event-based, not a live per-lane backlog/age snapshot. Related #9797 remains separate agent-facing `queue_status` work, not a supersession of this operator diagnostics PR.

## Best Possible Solution

Keep this PR open for maintainer review or an updated successor. The best path is a minimal core-owned, read-only queue snapshot seam in `src/process/command-queue.ts`, exposed through `diagnostics.queue` and `openclaw gateway diagnostics queue`, scoped to `operator.read`, documented, and covered with focused Gateway/CLI/queue tests. Keep #9797 separate for the in-agent `queue_status` tool request.
