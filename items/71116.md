## Summary

Keep #71116 open. Current main has SecretRef runtime documentation and low-level secret/file-lock helpers, but it does not yet add the requested contributor-facing credential-persistence contract or the hotspot-specific secrets guidance. The related PR #71156 is still open and only partially covers the issue, so the work is not implemented or safely superseded on main.

## Best Possible Solution

Keep the issue open and either narrow it to the documentation work or split it into follow-ups. The best path is to review or rework PR #71156 for the scoped AGENTS guidance, add a public plugin credential-persistence contract in the plugin docs that anchors the existing file-lock/json-store/secret-file primitives, and handle CI token-scope review as a separate maintainer-owned infra/security task if desired. An optional `openclaw/plugin-sdk/credentials` helper should stay a follow-up only if docs reveal repeated unsafe local patterns.
