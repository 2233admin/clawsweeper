## Summary

Keep #39857 open. Current main still has the core override-before-known-window behavior in `resolveContextTokensForModel()`, and the reply/status/session persistence paths still pass agent defaults or persisted session values through that override path. The linked fix PR #39859 remains the canonical related work in the provided GitHub context and has not landed in the checked-out main SHA.

## Best Possible Solution

Keep #39857 open until PR #39859 or an equivalent maintainer-approved fix lands. The implementation should make `resolveContextTokensForModel()`, reply setup, status rendering, and session persistence treat overrides or stored values as ceilings when a smaller known model window is available, while preserving existing behavior when the model window is unknown. Add focused regression coverage for the shared resolver, `/status`, and persisted session metadata.
