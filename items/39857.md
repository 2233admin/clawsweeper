## Summary

Keep #39857 open. Current main at 8ba82534e6c98410e19b226f3534b285fd75d278 still has the core override-first behavior: a positive contextTokens override can be returned or persisted before the selected model's known smaller window is applied. The linked fix PR #39859 is useful and directly addresses the issue, but the provided GitHub context still shows it open/unmerged.

## Best Possible Solution

Keep #39857 open until PR #39859 or an equivalent narrow maintainer-approved fix lands. The fix should make positive configured or persisted contextTokens values act as ceilings when a smaller selected-model window is known, preserve existing override behavior when no model window is known, align reply setup/status/session persistence with the shared resolver, and add regression tests for resolver, /status, and session metadata paths.
