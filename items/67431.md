## Summary

Keep this PR open. Current main already implements the chat-control and markdown-sidebar accessible-name fixes with test coverage, but the PR still contains unresolved accessibility behavior: the grouped tool disclosure change would replace contextual button text with identical expand/collapse labels, and the streaming `aria-busy` change is absent from main and needs explicit assistive-technology validation or coordination with the active streaming-announcement work. The same author also has paired open accessibility work in #67432, so closing only this PR would be premature.

## Best Possible Solution

Keep this PR open for scoped contributor follow-up: rebase on current main, drop the already-shipped chat-control and markdown-sidebar label changes, preserve contextual accessible names for grouped tool disclosures or append state without overriding context, and either split/validate the streaming `aria-busy` behavior with a screen-reader test plan or coordinate it with #65538/#65575. Coordinate the remaining Control UI accessible-name work with #67432 instead of closing only one half of the same-author sweep.
