## Summary

Keep open. Current `main` still has the icon-only button accessibility gaps this PR is trying to address, so it is not implemented on main, and the PR as written also introduces at least one accessibility regression plus an additional live-region behavior change that needs maintainer review.

## Evidence

- **Current main still lacks `aria-label` on icon-only chat controls:** The refresh, thinking, tool-calls, focus, and cron filter buttons still expose only `title` on `main`; there is no `aria-label` on those icon-only controls.

- **Current main still lacks `aria-label` on the markdown sidebar close button:** The close button in the markdown sidebar is still icon-only with `title="Close sidebar"` and no `aria-label` on `main`.
  - file: [ui/src/ui/views/markdown-sidebar.ts:34](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/ui/src/ui/views/markdown-sidebar.ts#L34)

- The underlying accessibility gaps on several icon-only controls are still present on current `main` and remain worth fixing.
- Closing this PR would discard useful work, but merging it as-is risks regressing the tool-disclosure accessible name and changing screen-reader live-region behavior without validation.
- Related work exists in this area (`#67432`, later cross-reference to `#70593`), so maintainers may want to consolidate rather than auto-close.
