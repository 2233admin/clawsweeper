## Summary

Current `main` no longer appears to use a hardcoded 15s LLM idle timeout for embedded runs, but the specific behavior requested in #46006 is still not implemented: `sessions_spawn` does not inherit `agents.defaults.timeoutSeconds` (or per-agent `timeoutSeconds`) when its own timeout is omitted. It resolves from `agents.defaults.subagents.runTimeoutSeconds` or `0`, and the subagent lane then treats omitted timeout as an explicit no-timeout override.

## Evidence

- **`sessions_spawn` default timeout source ignores `agents.defaults.timeoutSeconds`:** `resolveConfiguredSubagentRunTimeoutSeconds` reads only `agents.defaults.subagents.runTimeoutSeconds`; if unset, it falls back to `0` rather than `agents.defaults.timeoutSeconds`.
  - file: [src/agents/subagent-spawn-plan.ts:21](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/subagent-spawn-plan.ts#L21)

  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Spawn path forwards that subagent-specific timeout:** `spawnSubagentDirect` resolves `runTimeoutSeconds` through that helper, then sends it as `timeout` on the gateway `agent` call for the child run.
  - file: [src/agents/subagent-spawn.ts:597](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/subagent-spawn.ts#L597)

  - file: [src/agents/agent-command.ts:305](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/agent-command.ts#L305)

## Risks / Open Questions

- The original 15s symptom may already be gone, so the report may need reframing around config inheritance rather than a still-present 15s hardcode.
- Per-agent `agents.list[].timeoutSeconds` also does not appear to be wired into the `sessions_spawn` path on current `main`, so closing as implemented would be premature.
