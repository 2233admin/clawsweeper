## Summary

Keep open. Current `main` added a generic plugin `before_install` hook for gateway-backed skill installer actions, but it does not implement the requested ClawHub skill lifecycle hooks for ClawHub download/install/update flows, and there is no post-install hook on the public hook surface.

## Evidence

- **Generic install hook exists, but only as `before_install`:** The public plugin hook list includes `before_install` and no `after_install`/post-install skill hook.
  - file: [src/plugins/hook-types.ts:85](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/plugins/hook-types.ts#L85)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '80,125p'`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Gateway-backed skill installer runs scan + `before_install`:** `installSkill()` calls `scanSkillInstallSource()`, and that runtime invokes `runBeforeInstallHook()` with `targetType: "skill"` and `requestKind: "skill-install"`. This covers declared `metadata.openclaw.install` actions on already-present skills.
  - file: [src/agents/skills-install.ts:443](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/agents/skills-install.ts#L443)

- **Hook payload is for install-policy scanning, not separate skill lifecycle events:** The install hook payload builder only shapes the generic `before_install` contract with built-in scan data plus optional `skill`/`plugin` metadata.

## Risks / Open Questions

- Closing as implemented would overstate current coverage: the requested pre-install protection for ClawHub skill downloads/sync is not present on `main`.
- Closing as clawhub would also be weak because the issue explicitly asks for OpenClaw-side integration points, and OpenClaw still owns the `skills.install { source: "clawhub" }` gateway path.
