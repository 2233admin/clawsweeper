## Summary

Keep open. Current `main` still routes the Control UI chat speaker button through browser `speechSynthesis` instead of OpenClaw's configured TTS pipeline, so the reported Microsoft Edge/OpenClaw voice mismatch is not implemented away on `main`. The backend TTS stack still honors `messages.tts.providers.microsoft` and legacy `edge.*` config for other surfaces.
