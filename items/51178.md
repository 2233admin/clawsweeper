## Summary

Keep open. Current main has partial mitigations for lane blocking, including per-session/global lane queueing, immediate accepted acks for agent runs, and non-blocking/background subagent flows, but it does not implement the requested durable event-driven decoupling for agent-to-agent work. The current design is still an in-process promise queue, and docs explicitly note that some announce/completion work is best-effort and can be lost on gateway restart.
