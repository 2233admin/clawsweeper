## Summary

Keep open. Current main has adjacent compaction-wait/liveness safeguards, but the issue discussion also includes a successful timeout-compaction retry that replays already-completed message-tool sends. Current main still retries the prompt after successful timeout recovery compaction, and I did not find a side-effect/idempotency guard on that path.
