## Summary

Keep PR #69725 open. Current main still has the reported product mismatch: agent `exec host=node` checks only for `system.run` and then unconditionally sends `node.invoke` with `system.run.prepare`, while the current macOS companion command enum/advertisement still lists `system.run`, `system.which`, and `system.notify` without `system.run.prepare`. The gateway policy still rejects commands not declared by the connected node, producing the exact error described in the PR. The open related PR #38781 is adjacent but not a complete superseder because it adds prepare support to the macOS app, while #69725 provides backward-compatible agent-side behavior for nodes that advertise only `system.run`.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent fix on main: route `exec host=node` based on the selected node's declared commands, preserve the existing remote `system.run.prepare` path when available, and add a regression test for macOS-style nodes that advertise `system.run` without `system.run.prepare`. Maintainers should also decide separately whether #38781 or another app-side change should add native `system.run.prepare` for new macOS app builds.
