## Summary

Keep open. Current main has useful context-overflow mitigations, including pre-prompt compaction/tool-result truncation, but the reported startup/restart failure mode is not fully solved: the embedded runner still opens the session transcript before any size/quarantine guard, and bootstrap completion entries are still append-only without a count-based circuit breaker under the default context-injection mode.

## Best Possible Solution

Keep the issue open and implement a narrow recovery hardening path: add a pre-`SessionManager.open` transcript size/message-count guard, mark or quarantine sessions that repeatedly fail overflow/maintenance during restart, suppress excessive `openclaw:bootstrap-context:full` appends, and expose repair through `openclaw sessions cleanup` or `openclaw doctor` so users can recover without Time Machine restores.
