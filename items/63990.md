## Summary

Keep open. The issue and comments describe a credible production reliability gap, and current main still only supports a single active memory embedding provider/model/store with a single fallback provider. Fallback activation switches provider state and forces a full reindex; it does not maintain per-model indexes, dual-write ingestion state, activeIndex/fallbackIndexes routing, or per-index observability.

## Best Possible Solution

Keep #63990 open as a real memory-core feature request. The best path is an explicit design/implementation in `memory-core` and the public memory config surface: define `indexes[]`, `activeIndex`, fallback index policy, per-index ingestion/backfill status, single-index query routing, migration from existing `provider/model/store.path`, CLI/status observability, and regression tests proving no mixed vector spaces and runtime-safe active-index switches.
