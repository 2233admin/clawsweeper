## Summary

Keep open. Current main still has the exact gap described by PR #70803: the shared WhatsApp target normalizer accepts `@lid` but not `@hosted.lid`, while the runtime JID-to-E.164 path already recognizes `@hosted.lid`. The PR is a narrow bug fix with matching regression-test additions and is not obsolete on the inspected checkout or latest release tag.

## Best Possible Solution

Keep #70803 open for maintainer review and CI. The best path is to land the focused regex update in `extensions/whatsapp/src/normalize-target.ts`, keep the lower/upper-case `@hosted.lid` regression assertions in `extensions/whatsapp/src/resolve-target.test.ts`, and verify the WhatsApp plugin test/lint lane so shared target normalization aligns with the existing runtime `jidToE164` behavior.
