## Summary

Keep open. Current `main` still lacks OpenAI Chat Completions client-tool parsing, `tool_calls` response mapping, and the tool-result follow-up loop on `/v1/chat/completions`, while the sibling `/v1/responses` path already implements those contracts. This PR addresses a real gateway/API gap rather than an obsolete report.

## Evidence
