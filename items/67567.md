## Summary

Keep open. Current `main` has independently adopted the session-reset ordering fix, but it still does not wire bundled MCP runtime disposal into gateway shutdown, and the remaining cleanup paths are still unbounded if MCP transport teardown hangs. That means the PR's core shutdown leak concern is not fully implemented on `main` yet.
