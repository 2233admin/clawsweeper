## Summary

Current `main` does not implement a `plugins.entries.memory-core.config.dreaming.model` override. The `memory-core` plugin schema rejects unknown dreaming keys, the public docs only expose `enabled`/`frequency`, and Dream Diary generation calls the subagent without any per-run model override. The request is still actionable and has an active linked PR (#69209), so conservative cleanup should leave this issue open.

## Evidence

- **`memory-core` config schema has no `dreaming.model` field:** `dreaming` is `additionalProperties: false` and only lists fields like `enabled`, `frequency`, `timezone`, `verboseLogging`, `storage`, and `phases`; a `model` key would currently be invalid.
