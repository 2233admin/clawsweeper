## Summary

Keep open. Current main still contains the Control UI/backend mismatch: the Agents page Set Default handler writes `agents.defaultId`, but the config schema only accepts `agents.defaults` and `agents.list`, and runtime default resolution uses `agents.list[].default` or the first agent. Related PRs #59200 and #59202 were closed unmerged in the provided GitHub context, so they do not supersede or ship a fix.

## Best Possible Solution

Keep the issue open and fix the Control UI default-agent path on main. The narrow fix is to replace the `agents.defaultId` write with an `agents.list` update that sets `default: true` on the selected agent and removes/clears it on the others, then add regression coverage proving the saved config contains `agents.list[].default` and no `agents.defaultId`. A later `agents.setDefault` RPC from #52928 could simplify clients, but it is not required to fix this UI bug.
