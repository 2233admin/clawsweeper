## Summary

Keep open. Current `main` still enforces a single desktop chat window/panel at a time, so the requested multi-window parallel-session behavior is not implemented on `6b618f063516aa9ba85903aa4565c5a204c055b2`.

## Evidence

- **Singleton desktop chat manager:** `WebChatManager` stores only one `windowController`/`windowSessionKey` and one `panelController`/`panelSessionKey`, which is a singleton design rather than multi-window support.
  - file: [apps/macos/Sources/OpenClaw/WebChatManager.swift:29](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/apps/macos/Sources/OpenClaw/WebChatManager.swift#L29)
  - command: `nl -ba apps/macos/Sources/OpenClaw/WebChatManager.swift | sed -n '1,140p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Opening a different chat window closes the existing one:** `show(sessionKey:)` closes the existing window controller when the requested session differs, then replaces it with a new controller. That directly contradicts concurrent chat windows for separate sessions.
  - file: [apps/macos/Sources/OpenClaw/WebChatManager.swift:41](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/apps/macos/Sources/OpenClaw/WebChatManager.swift#L41)
  - command: `nl -ba apps/macos/Sources/OpenClaw/WebChatManager.swift | sed -n '1,140p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **UI entry points all route through the singleton manager:** Desktop menu actions open chat via `WebChatManager.shared.show(...)` / `togglePanel(...)`, so the app surface is wired around the singleton manager rather than spawning independent chat windows.
  - file: [apps/macos/Sources/OpenClaw/MenuContentView.swift:117](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/apps/macos/Sources/OpenClaw/MenuContentView.swift#L117)
  - command: `nl -ba apps/macos/Sources/OpenClaw/MenuContentView.swift | sed -n '112,136p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Each controller is bound to one immutable session:** `WebChatSwiftUIWindowController` captures a single `sessionKey` at init time, reinforcing the current one-controller-per-session-window model without any manager support for multiple simultaneous controllers.
  - file: [apps/macos/Sources/OpenClaw/WebChatSwiftUI.swift:217](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/apps/macos/Sources/OpenClaw/WebChatSwiftUI.swift#L217)
