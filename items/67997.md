## Summary

Keep open. The PR addresses a real GitHub-rendering annoyance, but the proposed change conflicts with current docs policy and would degrade docs tooling on `main`; it needs a different implementation rather than conservative auto-close.

## Evidence

- **Docs policy requires root-relative links inside docs pages:** `docs/AGENTS.md` explicitly says internal doc links in `docs/**/*.md` must stay root-relative and reserves absolute `https://docs.openclaw.ai/...` links for README and other GitHub-rendered docs.
  - file: [docs/AGENTS.md:7](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/docs/AGENTS.md#L7)
  - command: `nl -ba docs/AGENTS.md | sed -n '1,220p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Touched page on main still follows that convention:** The exact section this PR edits in `docs/tools/plugin.md` currently uses root-relative links for Plugin Bundles, Building Plugins, SDK Overview, and plugin/channel docs entries, so the PR is not already implemented on `main` and would be changing an intentional convention.
  - file: [docs/tools/plugin.md:83](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/docs/tools/plugin.md#L83)

  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Absolute URLs bypass docs i18n localization:** `localizeURL` returns inputs unchanged when they already have a URL scheme, so replacing internal root-relative links with `https://docs.openclaw.ai/...` would prevent locale rewriting for translated docs.
  - file: [scripts/docs-i18n/localized_links.go:293](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/scripts/docs-i18n/localized_links.go#L293)

  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Absolute URLs bypass internal route auditing:** The docs link audit skips `http(s)` targets entirely before route checking, so converting docs-internal links to absolute URLs would remove these links from broken-route validation.
  - file: [scripts/docs-link-audit.mjs:342](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/scripts/docs-link-audit.mjs#L342)
