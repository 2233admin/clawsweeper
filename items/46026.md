## Summary

Keep open. Current `main` still mounts the sandbox workspace read-only when `workspaceAccess` is `"none"`, and both tests and docs explicitly encode that behavior, so the requested change is not implemented on `main`.

## Evidence

- **Current mount logic still makes `none` read-only:** `appendWorkspaceMountArgs` sets the main workspace bind to read-only for anything except `"rw"`, so `"none"` still becomes `:ro`.
  - file: [src/agents/sandbox/workspace-mounts.ts:28](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/sandbox/workspace-mounts.ts#L28)

  - file: [src/agents/sandbox/workspace-mounts.test.ts:8](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/sandbox/workspace-mounts.test.ts#L8)
