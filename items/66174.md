## Summary

Keep this PR open. Current main still lacks the requested save-time canonicalization for persisted session transcript paths, and the PR is a focused implementation candidate, but it needs revision before merge because it uses the compatibility resolver that can preserve cross-root agent session paths instead of forcing the active sessions directory.

## Best Possible Solution

Keep this PR open and request a small revision: add save-time normalization in `src/config/sessions/store.ts`, derive the canonical target from `path.dirname(storePath)` plus a validated `sessionId` rather than `resolveSessionFilePath`, persist both `sessionFile` and the compatibility `transcriptPath` alias if that alias remains supported, add coverage for structurally valid cross-root paths, and keep the external-transcript deletion safeguards covered by the existing pruning tests.
