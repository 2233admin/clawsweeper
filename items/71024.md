## Summary

Keep open. Current `main` still lacks a Mattermost read-only account inspector, and the status/config snapshot path still resolves Mattermost accounts through strict SecretRef resolution. The PR is addressing a live gap rather than an already-fixed or obsolete item.

## Evidence

- **Mattermost still has no read-only inspector on main:** `mattermostConfigAdapter` wires `resolveAccount` only. Unlike the other chat plugins, current main does not expose `inspectAccount` for Mattermost, so read-only status surfaces have no non-dereferencing path for unresolved credentials.

  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Mattermost account resolution is still strict for SecretRef bot tokens:** `resolveMattermostAccount` calls `normalizeResolvedSecretInputString` for `merged.botToken` unless `allowUnresolvedSecretRef` is explicitly set. The shared secret-input implementation throws in strict mode for unresolved SecretRefs, matching the PR's reported failure mode.
  - file: [extensions/mattermost/src/mattermost/accounts.ts:95](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/extensions/mattermost/src/mattermost/accounts.ts#L95)
