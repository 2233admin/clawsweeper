## Summary

Do not close PR #71024. Current main still has the Mattermost status bug surface: Mattermost has only the strict runtime resolver, no read-only account inspector, and the status table path falls back to that strict resolver when inspection returns nothing. The PR is a focused fix for a real remaining gap, with related issue #70942 describing the matching crash and PR review context noting the intended read-only SecretRef behavior.

## Best Possible Solution

Keep the PR open and review/land the focused fix or an equivalent main-branch patch: add a Mattermost read-only account inspector that reports unresolved SecretRef bot tokens as configured_unavailable without dereferencing them, wire it into Mattermost config/status inspection, preserve botTokenStatus in snapshots, and keep runtime resolveMattermostAccount strict for actual token use. Add the PR's targeted Mattermost account/status regression tests before merge.
