## Summary

Keep open. Current main still does not accept, persist, document, expose in UI, or propagate a `silent` option for cron `failureAlert`, even though the generic outbound/Telegram delivery stack already supports silent Telegram messages. This remains a narrow cron failure-alert integration gap, not a duplicate, ClawHub-only request, or already-shipped feature.

## Best Possible Solution

Implement a narrow additive cron contract change on main: add `silent?: boolean` to per-job `CronFailureAlert` and global `cron.failureAlert` config; update TypeBox/Zod/generated config docs, CLI docs, and Control UI state/forms; propagate it through `resolveFailureAlert`, `emitFailureAlert`, `sendCronFailureAlert`, `sendGatewayCronFailureAlert`, `sendCronAnnouncePayloadStrict`, and finally `deliverOutboundPayloads({ silent })`; add focused regression tests proving Telegram cron failure alerts set `disable_notification` when `silent: true` and preserve current default behavior when omitted.
