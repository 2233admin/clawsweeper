## Summary

Current main already downloads inbound Feishu video/media attachments, stores their local paths in `MediaPath`/`MediaPaths`, and prepends those paths into the agent prompt as `[media attached: ...]`. The requested `"[Video: path]"` text change is unnecessary because the agent already receives the local file path through the generic inbound media pipeline, and this behavior is present in release `v2026.4.23`.

## Close Comment

This is already implemented on current `main`, so I would close it.

- Codex review of current `main` found that Feishu inbound handling already downloads video/media attachments, saves them locally, and builds agent media payload from those saved paths in `extensions/feishu/src/bot.ts:714` and `extensions/feishu/src/bot.ts:1015`.
- The actual local path comes from `extensions/feishu/src/bot-content.ts:329`, where `resolveFeishuMediaList(...)` downloads the resource and records `saved.path`.
- OpenClaw then prepends attachment paths into the agent prompt through the generic media-note pipeline in `src/auto-reply/reply/prompt-prelude.ts:33` and `src/auto-reply/media-note.ts:103`.
- Video coverage is explicit in tests: `src/auto-reply/media-note.test.ts:231` expects a rendered video path, and `src/auto-reply/reply.media-note.test.ts:6` verifies media paths are included in the agent prompt.
- This behavior is already present in the shipped release `v2026.4.23` (`9b18ae877018a34933e27bb7b2986cb97e9602aa`).
