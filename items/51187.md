## Summary

Current `main` already exposes ACP runtime backend registration to plugins through the supported `openclaw/plugin-sdk/acp-runtime` seam, and the bundled `acpx` plugin uses that seam with lifecycle-safe register/unregister behavior. The PR's requested capability exists on `main`; it was implemented as a dedicated SDK subpath rather than by widening `OpenClawPluginApi`.
