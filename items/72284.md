## Summary

Keep #72284 open. Current main has DREAMS.md/Control UI support, but the reported failure is still plausible: REM phase report writing is independent of Dream Diary narrative writing, and current tests explicitly allow no DREAMS.md when narrative generation has no material or times out. Related #70523 and #68505 do not supersede this report; they cover raw inline fallback pollution and heading normalization, not missing DREAMS.md creation after REM output.

## Best Possible Solution

Keep the issue open for maintainer triage. The best fix is to decide whether a successful REM phase with material must always create/update `DREAMS.md`; if yes, make the scheduled REM path write a deterministic fallback diary entry when narrative generation is unavailable/empty/timeout and add a regression test covering separate-storage REM report plus Dream Diary creation. If the diary is intentionally best-effort only, update the docs and Control UI empty-state wording so a `memory/dreaming/rem/YYYY-MM-DD.md` report without `DREAMS.md` is not presented as a broken Dreams tab.
