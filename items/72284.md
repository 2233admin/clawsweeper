## Summary

Keep #72284 open. Current main still has a real contract gap: REM report creation under `memory/dreaming/rem/` is independent from `DREAMS.md`, the Dream Diary write is best-effort and conditional on the narrative subagent path, and the Control UI reads only root `DREAMS.md`/`dreams.md`. Related PR #70523 is useful context for fallback behavior but does not supersede this missing diary/UI artifact issue.

## Best Possible Solution

Fix the memory-core and Control UI contract directly: after a successful REM run, either reliably create/update `DREAMS.md` with a human-readable diary entry or return an explicit no-diary status that the UI can show. Add regression coverage for a managed REM run where `memory/dreaming/rem/YYYY-MM-DD.md` exists and `doctor.memory.dreamDiary`/the Dreams tab show the expected content or a clear status.
