## Summary

Keep #72369 open. Current main still builds the internal bare `/new`/`/reset` startup instruction, passes it as the transcript prompt for the agent run, and the WebChat/history display paths do not recognize or hide a startup-kind marker. The related ClawSweeper reports cover adjacent WebChat/history issues, but they do not supersede this user-role startup reset transcript leak.

## Best Possible Solution

Keep the issue open and implement a narrow WebChat/history fix: carry bare-reset startup metadata alongside the transcript prompt, tag persisted reset-trigger turns with an internal `__openclaw` startup kind or equivalent, and project them as hidden or as a lightweight `New session` divider in both `chat.history` display projection and `ui/src/ui/chat/build-chat-items.ts`. Add regression coverage for bare `/new` and `/reset` showing the agent still receives the startup instruction while WebChat/history no longer renders it as user-authored text.
