## Summary

Current `main` does not already contain this fix, and the PR has an unresolved correctness concern from review. The Telegram matcher on `main` still only treats unbound approvals as ambiguous for non-Telegram sources, so the reported multi-account Telegram fan-out remains plausible. Separately, the PR discussion includes a P1 review note that the proposed target-account gate breaks `approvals.exec.mode: "both"`, so there is not strong evidence for closing this PR as implemented, obsolete, or incoherent.

## Evidence

- **Current main still routes unbound Telegram-source requests to any account:** `matchesTelegramRequestAccount` only applies the ambiguity guard when `turnSourceChannel` is present and not `telegram`; otherwise it returns true whenever no bound account is resolved. That means a Telegram-sourced approval without `turnSourceAccountId` can still match multiple eligible Telegram accounts on current `main`.
