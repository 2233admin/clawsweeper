## Summary

Keep open. Current main still does not provide per-LLM-call `llm_output` payloads or a raw `llm_call` hook with per-iteration usage/output. Main now has sanitized `model_call_started`/`model_call_ended` telemetry, but that surface intentionally carries metadata, duration, and outcome only; it does not include the per-call usage, assistant output, or tool-call details requested here.

## Best Possible Solution

Keep this issue open. A conservative fix should add a documented per-call observation contract without silently breaking existing `llm_output` consumers: either introduce a new conversation-access-gated per-call hook carrying per-iteration usage/provider/model/run/session/agent data, or extend the existing model-call telemetry with a compatible terminal usage/output companion. Preserve current aggregate `llm_output` semantics unless a versioned migration is explicitly designed.
