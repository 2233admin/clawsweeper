## Summary

Keep this issue open. Current main still buffers QQBot `kind: "tool"` text behind the tool-only fallback path, and `streaming.mode: "partial"` only controls block streaming for QQBot rather than immediate tool/progress delivery. The existing Telegram/Discord/Slack/Mattermost preview-progress support does not cover QQBot.

## Best Possible Solution

Keep this issue open and implement a QQBot-owned fix in `extensions/qqbot/src/engine/gateway/outbound-dispatch.ts`. When `streaming.mode` is partial, QQBot should deliver visible tool/progress text immediately or through a QQBot preview/edit path, with a `streaming.preview.toolProgress`-style opt-out if the channel adopts the shared preview config. Add a focused regression test in `extensions/qqbot/src/engine/gateway/outbound-dispatch.test.ts` for a `kind: "tool"` text delivery followed by a block/final reply, and update QQBot docs/schema if the public config surface changes.
