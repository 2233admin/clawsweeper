## Summary

Keep #70541 open. Current main still has the reported auto-compaction post-hook reading post-compaction context from process.cwd() before enqueueing the refresh system event, even though the run already carries the configured workspaceDir and nearby preflight compaction code uses that workspaceDir correctly. The latest v2026.4.24 release also still contains the same process.cwd() call, so this is not implemented on main or shipped.

## Best Possible Solution

Patch the auto-compaction post-hook in src/auto-reply/reply/agent-runner.ts to call readPostCompactionContext with followupRun.run.workspaceDir, not process.cwd(). Add a focused regression test where process.cwd() differs from followupRun.run.workspaceDir and a run result reports compactionCount > 0, then assert the enqueued post-compaction refresh comes from the configured workspace AGENTS.md. Keep the existing preflight compaction workspaceDir path unchanged.
