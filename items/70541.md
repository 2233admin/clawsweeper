## Summary

Current `main` still appears to have the reported bug. The auto-compaction follow-up path in `src/auto-reply/reply/agent-runner.ts` injects post-compaction context from `process.cwd()` instead of the run workspace, while the related follow-up/preflight path and the context reader itself are workspace-rooted. The latest release `v2026.4.23` still contains the same `process.cwd()` call, so there is no strong basis to close this item as already fixed or obsolete.

## Evidence

- **Current main still uses process cwd for auto-compaction injection:** The auto-compaction branch sets `const workspaceDir = process.cwd();` and passes that to `readPostCompactionContext(...)`, matching the report rather than using `followupRun.run.workspaceDir`.
