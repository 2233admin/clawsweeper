## Summary

Keep #58079 open. Current main still exposes `before_tool_call` with only `toolName`, `params`, `runId`, and `toolCallId` on the event, and the embedded runner/tool adapters do not thread the PR's requested model-facing `systemPrompt`, `messages`, or `tools` snapshot into the hook. The PR is a focused additive plugin contract change with useful discussion and review feedback; it is not implemented on main, not clearly superseded by the broader related issues, and not a ClawHub-only item because it changes the core hook payload API needed by external policy plugins.

## Best Possible Solution

Keep the PR open for maintainer review or replacement by an equivalent core hook-contract PR. The best implementation path is an additive, documented `before_tool_call` payload extension that exposes stable snapshots of the effective system prompt, conversation messages, and model-visible tool definitions across wrapped OpenClaw tools, MCP/LSP-style tools, and client tool paths, with tests covering mutation safety and failure behavior.
