## Summary

Keep open. Current `main` still persists session rows as `running` on lifecycle start, but I did not find any generic startup path that resumes, fails, or times out orphaned non-subagent sessions after a crash/restart. The only restart-time recovery paths on `main` are restart-sentinel wake handling and a subagent-specific orphan recovery flow.

## Evidence

- **Lifecycle state remains `running` until a later terminal event arrives:** `deriveGatewaySessionLifecycleSnapshot()` writes `status: "running"` on `start`, and only flips to a terminal status on later `end`/`error` events. A crash between those phases would leave the persisted row in `running` state.
  - file: [src/gateway/session-lifecycle-state.ts:102](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/gateway/session-lifecycle-state.ts#L102)
  - command: `nl -ba src/gateway/session-lifecycle-state.ts | sed -n '95,170p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Gateway startup does not schedule generic main-session orphan recovery:** Current startup sidecars only schedule `restart-sentinel` handling and `subagent-recovery`; there is no corresponding startup scan/recovery step for ordinary running sessions.
  - file: [src/gateway/server-startup-post-attach.ts:333](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/gateway/server-startup-post-attach.ts#L333)
  - command: `nl -ba src/gateway/server-startup-post-attach.ts | sed -n '330,360p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Existing orphan recovery is explicitly limited to subagent sessions:** `subagent-orphan-recovery.ts` is documented and implemented as post-restart recovery for subagent sessions only, scanning active subagent runs with `abortedLastRun: true` and sending a synthetic resume message. That does not cover ordinary gateway sessions stuck mid-turn.
  - file: [src/agents/subagent-orphan-recovery.ts:1](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/subagent-orphan-recovery.ts#L1)
