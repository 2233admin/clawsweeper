## Summary

Current `main` still acknowledges Telegram webhook deliveries before processing has finished, and webhook mode has no durable inbound queue or replay path. The issue remains valid and should stay open.

## Evidence

- **Webhook handler returns success before work completes:** `extensions/telegram/src/webhook.ts` configures `grammy.webhookCallback(..., { onTimeout: "return", timeoutMilliseconds: 5000 })` and then sends `200` once the handler replies or returns, without any persistence step between receipt and completion.

  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
