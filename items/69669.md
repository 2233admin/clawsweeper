## Summary

Current `main` does not show a fix for #69669. The ACP/thread-bound guidance still says follow-ups continue directly in the bound thread, while the only explicit rewrite guidance in current docs/system prompt is for completion events, not shorthand parent-orchestrated follow-ups. I did not find source or docs evidence that thread-bound ACP follow-ups are now rewritten into explicit child tasks by default.
