## Summary

Keep open. Current main has useful adjacent tooling (`memory promote-explain`, `memory promote --json`, Dreams/doctor status counts), and related #68882-style promotion reachability appears improved, but #70072’s central request remains unimplemented: scheduled deep reports still contain only aggregate lines, verbose candidate details still go to the daemon logger rather than a workspace artifact, and there is no aggregate `openclaw memory dreaming doctor`-style gate diagnostic.

## Best Possible Solution

Keep #70072 open as an additive memory-core introspection feature. The best path is to add a shared candidate evaluation/explanation helper that can compute rank, score components, thresholds, and rejection reasons for all considered entries, then use it in the scheduled deep report, optional verbose JSONL output, and a CLI/status diagnostic surface with focused tests and docs.
