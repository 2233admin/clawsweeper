## Summary

Keep open. Current `main` still intentionally emits `gateway.trusted_proxies_missing` for loopback Control UI without `gateway.trustedProxies`, the test suite still asserts that behavior, and the docs explicitly describe same-host loopback reverse proxies as a valid `trustedProxies` use case. The PR’s `bind !== "loopback"` inversion would remove that documented warning and still keys off bind mode instead of the effective bind host, which is inconsistent with current runtime/config handling for `custom` and `auto` binds.
