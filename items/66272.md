## Summary

Keep this PR open. Current main still emits `gateway.trusted_proxies_missing` for `gateway.bind: "loopback"`, so the reported noisy security-audit finding is not implemented. The PR is a plausible fix direction, but its submitted patch is stale against the current split audit implementation and its raw `bind !== "loopback"` check would still misclassify loopback-resolving `custom`/`auto` configurations.

## Best Possible Solution

Keep this PR open and request a revision against `src/security/audit-gateway-config.ts`. The fix should classify `gateway.trusted_proxies_missing` from effective gateway exposure, stop warning for definitely local-only loopback-resolving binds, still warn for genuinely exposed reverse-proxy cases, and add regression coverage for loopback, LAN, custom loopback, custom exposed, and auto behavior.
