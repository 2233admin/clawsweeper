## Summary

Keep this PR open. Current main still lacks the requested host-read allow path for HTML/XML/CSS/application XML fallback documents, while the latest PR head is a focused implementation candidate that routes those formats through the validated text guard and addresses the earlier security review blocker.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to review the latest validated-text approach, rebase if needed, run the targeted media/outbound tests in CI, and land it if the checks remain green and maintainers accept the scoped host-read policy expansion.
