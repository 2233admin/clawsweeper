## Summary

Current `main` and the latest shipped release still reject host-read fallback HTML/XML/CSS documents. The PR addresses a real remaining gap after #51562/#67047, but the PR discussion also surfaced an unresolved security regression in the submitted implementation, so this item should stay open rather than be auto-closed.

## Evidence

- **Current host-read guard still rejects fallback document MIME:** `assertHostReadMediaAllowed()` only permits buffer-verified document MIME or validated CSV/Markdown text. If `kind === "document"` and the MIME came from the file-path fallback, it throws `Host-local media sends require buffer-verified media/document types`.
