## Summary

Keep this PR open. Current main still has the reported empty-heartbeat fallback behavior, and this PR is a focused implementation candidate, but its one-file change would only update the summary helper while scheduler, system prompt, ACP spawn policy, docs, and regression tests remain on the documented `every: "0m"` disable contract.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rework. If OpenClaw wants `heartbeat: {}` to mean disabled, add one shared empty-heartbeat predicate and apply it consistently across `src/infra/heartbeat-summary.ts`, `src/infra/heartbeat-runner.ts`, `src/agents/heartbeat-system-prompt.ts`, and `src/agents/acp-spawn.ts`; then add regression tests for defaults and per-agent empty heartbeat configs, including inherited defaults, and update the public heartbeat/config docs. If maintainers want `every: "0m"` to remain the only disable contract, document that on the linked bug report and close or revise this PR instead of landing a partial behavior change.
