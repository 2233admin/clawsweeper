## Summary

Keep PR #57849 open. Current main still sends `agent_end` messages from the in-memory `messagesSnapshot`, and the main compaction hook paths still either omit `messages` or pass in-memory session messages. `before_reset` has since grown transcript-reading behavior on main, so that part of the PR needs rebasing/adaptation, but the central hook provenance gap is not fully implemented on main.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and narrow it to the still-missing plugin hook surfaces: source `agent_end` and `before_compaction` messages from transcript data when available, preserve current timing/order diagnostics, keep the already-improved reset-hook archive fallback, and add targeted provenance regression tests.
