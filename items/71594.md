Keep PR #71594 open. Current main still lacks the PR's IPv4-only BYOH documentation and onboarding hint, while the current gateway bind/link implementation supports the PR premise by using IPv4-only helpers or IPv4 validation for the relevant `lan`, `tailnet`, and `custom` paths.

## Best Possible Solution
