## Summary

Keep open. I did not find strong evidence that current `main` already implements the Feishu `NO_PROXY` WebSocket behavior, and the PR as proposed still carries unresolved scope/contract issues rather than a clearly shippable fix.

## Risks / Open Questions

- Closing this PR as implemented would be premature because the current repo state does not provide strong proof that Feishu WebSocket proxy bypass already works on `main`.
- The PR is not a clean one-topic fix: it also changes agent self-update guidance, and that guidance is not obviously aligned with the current gateway tool action schema.
- Feishu WebSocket proxy behavior remains under-tested for `NO_PROXY` cases, so either the reported bug or a future regression could persist unnoticed.
