## Summary

Keep open. Current main still serializes direct multi-collection QMD CLI searches, and the PR remains a focused performance change with regression tests. It is not implemented, duplicate, or obsolete, though the current patch still deserves maintainer review for the resource/backpressure and logging concerns raised in the PR discussion.

## Best Possible Solution

Keep PR #69583 open for maintainer review/rework. The likely best path is to land a bounded parallel per-collection QMD search implementation with robust abort cleanup, sanitized command summaries, and the proposed regression coverage once the resource and logging tradeoffs are resolved.
