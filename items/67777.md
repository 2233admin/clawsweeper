## Summary

Keep open. Current `main` still uses direct-first completion delivery with only conditional queue fallback, stores only a transient `completionAnnouncedAt` marker instead of a durable completion spool, and can prune orphaned ended runs during resume/restore before any durable redelivery path exists. The issue's planning comment matches the current code seams rather than an already-landed fix.

## Evidence

- **Run record has no durable completion spool state:** `SubagentRunRecord` still tracks only retry counters, frozen result text, and a transient `completionAnnouncedAt`; there is no persisted inbox/spool payload or delivery state beyond announced/not announced.

If direct delivery fails, it tries one queue fallback, and if that fallback returns `none`, the function returns the original direct failure instead of persisting a durable completion for later replay.
