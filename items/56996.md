## Summary

Current `main` already covers this report well enough: the CLI `openclaw cron add` builds a normalized cron job payload and sends it straight to `cron.add`, and the cron tool on `main` now also recovers flat add params into a `job` object before validation. The flat-param recovery was added in commit `68315792677f9628d1370d6869366442f1d3ecad`, which is newer than the latest tagged release `v2026.4.23`, so this appears fixed on `main` but not yet in a release tag I can cite.

## Evidence

- **CLI add path sends normalized params directly:** `registerCronAddCommand` builds `schedule`, `payload`, `sessionTarget`, and `delivery` into a single params object, then calls `callGatewayFromCli("cron.add", opts, params)`. That current path does not send the flat `{ action, name, every, message }` shape described in the issue.
