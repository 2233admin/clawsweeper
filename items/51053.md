## Summary

Keep open. Current `main` still has the inconsistency this PR targets: `openclaw nodes list` builds its paired inventory from `node.pair.list`, while `openclaw nodes status` and the gateway’s canonical known-node view come from `node.list`, which merges device pairing, node pairing, and live connections. Public docs also state that WS node/device pairing is separate from `node.pair.*`, so the reported mismatch is still credible on current `main`. The PR discussion also shows the author responded to the initial legacy-fallback review concerns, so this is not a conservative auto-close candidate.
