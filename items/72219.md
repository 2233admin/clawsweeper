## Summary

Keep #72219 open. The checked-out main at 998e37f still serializes a failure-shaped `delivery.resolved` object for bare `delivery.mode: "none"` cron runs, while the PR is a focused fix for linked issue #72210 with a regression test for the missing assertion.

## Best Possible Solution

Keep the PR open for maintainer review and land #72219, or an equivalent targeted patch, that omits `delivery.resolved` only for intentionally disabled fallback delivery while preserving announce/webhook and explicit-target message-tool behavior. Keep the regression test so future cron run records do not reintroduce a false failure signal.
