## Summary

Keep open. Current main has a built-in dangerous-code scanner for gateway-backed skill dependency installers, but the requested on-by-default scan for ClawHub skill install/update and first-load/manual skill paths is still missing. The issue discussion also points to useful external scanners, AgentShield and ClawGuard, but those do not make the OpenClaw ClawHub install/update path fail closed today.

## Evidence

- **Docs state ClawHub skill installs are not covered by the installer scanner:** The skills docs tell users to treat third-party skills as untrusted and explicitly distinguish gateway-backed skill dependency installs from `openclaw skills install <slug>`, which downloads a ClawHub skill folder and does not use the installer-metadata scanner path.
