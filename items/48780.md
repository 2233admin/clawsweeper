## Summary

Keep open. Current `main` has related OpenAI-compatible tool-call repair logic for malformed streamed arguments, but I could not prove that it fixes this specific Windows/Qwen report where a long literal `</arg_value>>` suffix is said to reach `exec()`/`read()` arguments. The existing repair path targets malformed JSON preambles and very short trailing junk, not an exact `arg_value` suffix embedded in a command/path string.
