## Summary

Keep #71733 open. Current main does not implement the requested durable host-owned next-turn injection queue or a typed agent_turn_prepare hook. The existing surfaces are current-turn prompt hooks plus an explicitly in-memory system-event queue, and the linked #71731 is an open docs/RFC package rather than an implementation. This is also not a ClawHub cleanup close because the request is for a missing generic plugin SDK/host seam, which VISION.md treats as valid core API design discussion when useful plugin behavior cannot be built with existing plugin surfaces.
