## Summary

Keep open. Current main still lets the model-supplied exec `security` argument lower the effective configured policy through `minSecurity`, so the reported configured `full` plus requested `allowlist` failure remains real. The PR is a focused implementation candidate, but the supplied head still needs cleanup before merge.

## Best Possible Solution

Keep this PR open and review it as a real exec policy bug fix. The best landing shape is a minimal change in `src/agents/bash-tools.exec.ts` that makes configured/session exec security authoritative for model tool calls, preserves the explicit operator-granted elevated-full override, removes unused imports and the unused `maxSecurity` helper/tests, adds focused regression tests for configured `full` with model `allowlist` and configured `allowlist`/`deny` with model attempts to change policy, and updates or intentionally preserves the model-facing `security` schema as a compatibility decision.
