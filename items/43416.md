## Summary

Keep this PR open. Current main still does not provide a native shared Swift assistant-message copy button, and the PR is a focused UI improvement, but it should not be closed as implemented or landed as-is because the proposed copy payload is raw `primaryText` rather than the same visible assistant text rendered after thinking/final tag parsing.

## Best Possible Solution

Keep this PR open and request revisions: derive the clipboard payload from the same visible assistant-text segments used by `ChatAssistantTextBody`, preserve the assistant-role guard and cancellable copied-state task, and verify the shared native chat UI on both macOS and iOS. A focused parser/helper test for visible copied text would make the remaining privacy and behavior risk reviewable.
