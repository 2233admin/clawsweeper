## Summary

Keep PR #69337 open. Current main does not include the proposed commentary fallback, but the PR should not be auto-closed as implemented or superseded: the user-visible TUI problem is plausible, while the current patch applies the fallback in a shared helper that current tests and product contracts intentionally use to hide commentary-only assistant text outside TUI.

## Best Possible Solution

Keep the PR open for author or maintainer follow-up, but do not merge the shared-helper fallback as written. The best path is a TUI-scoped rendering/streaming change, or an explicit caller option that preserves existing shared defaults and updates targeted TUI tests while keeping gateway preview, chat.history, Web UI, transcript, and direct-delivery commentary-hiding tests green.
