## Summary

Keep #69337 open. The PR is a focused TUI visibility fix, but current main still intentionally keeps commentary-only assistant text hidden in the shared extractor, WebChat extraction, session previews, and chat history. The PR’s proposed shared fallback conflicts with existing contracts and review feedback, but the underlying TUI/product question is not cleanly implemented or superseded on main.

## Best Possible Solution

Keep the PR open for maintainer judgment or request a revised patch: preserve the shared `extractAssistantVisibleText` contract, keep commentary-only text hidden from history/previews/WebChat, and if TUI should show in-flight commentary, add a TUI-specific option or event-path fallback with targeted TUI tests plus unchanged shared/gateway preview tests.
