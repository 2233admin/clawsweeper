## Summary

Keep open. Current `main` removed the in-repo ACPX runtime code path this PR patches, but the local tree does not provide strong enough evidence that the specific orphaned-grandchild/process-group cleanup was implemented elsewhere and covered after the refactor.

## Evidence

## Risks / Open Questions

- Closing as `implemented_on_main` would overstate the evidence: the old code path is gone, but the equivalent orphan-cleanup behavior is not demonstrated in-tree.
- Closing as `cannot_reproduce` would also be weak because this review was limited to read-only code/history inspection and could not validate runtime behavior end-to-end.
- The PR itself is likely stale against the refactored ACPX architecture, but it may still point at a real lifecycle gap in the external `acpx/runtime` dependency or in how OpenClaw uses `cancel()` versus `close()`.
