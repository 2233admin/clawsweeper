## Summary

Current main still appears to have the reported Telegram regression. The Telegram block reply path spreads `replyPipeline` into `dispatcherOptions`, but neither that pipeline nor the local dispatcher options include `humanDelay`; the core reply dispatcher only delays block replies when `options.humanDelay` is supplied. The provided context shows an open PR (#69022) with the exact fix, but it is not merged, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #68945 open and land #69022 or an equivalent focused fix that passes `resolveHumanDelayConfig(cfg, route.agentId)` into Telegram's block reply `dispatcherOptions`, preferably through the existing Telegram runtime seam and with a regression test in `extensions/telegram/src/bot-message-dispatch.test.ts` proving configured humanDelay reaches the dispatcher.
