## Summary

Keep this PR open. Current main still returns early for fresh persisted token totals before evaluating the token threshold, so the reported Anthropic prompt-cache auto-compaction bug remains plausible on main. The PR is not clearly superseded because it adds the explicit `totalTokensFresh === true` handling requested by review, while overlapping work still needs consolidation and rebase against the newer transcript-byte compaction path.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best maintainer path is to consolidate it with #64384 and #65600, then land one rebased fix in `src/auto-reply/reply/agent-runner-memory.ts` that evaluates explicitly fresh persisted totals against the preflight token threshold, falls back to transcript estimation for `totalTokensFresh: false` or `undefined`, preserves the active-transcript byte trigger, and keeps regression coverage for 100% cached Anthropic totals, below-threshold fresh totals, stale fallback, legacy unknown freshness, and heartbeat/CLI skips.
