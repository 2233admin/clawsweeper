## Summary

Keep open. Current `main` does not implement the requested gateway-install refresh for loaded services that still embed `OPENCLAW_GATEWAY_TOKEN`, and this PR is the explicitly retained duplicate over #70850 because it covers the env-file-backed token distinction the earlier attempt missed.

## Evidence

- **Current main still early-returns without an embedded-token refresh check:** In `runDaemonInstall`, the non-`--force` loaded-service path only checks `gatewayServiceNeedsAutoNodeExtraCaCertsRefresh(...)`; if that is false it emits `already-installed` and returns. There is no `OPENCLAW_GATEWAY_TOKEN` auto-refresh branch on `main`.
