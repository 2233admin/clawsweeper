## Summary

Keep this PR open. Current main still routes WhatsApp inbound text and audio transcripts to the agent through raw `BodyForAgent`, so the hardening goal is not implemented. The submitted patch is also not merge-ready because it wraps legacy envelope text, leaves the active model-facing path raw, and would make echo keys nondeterministic while placing trusted `messagePrefix` text inside an untrusted block.

## Best Possible Solution

Keep this PR open as an incomplete security-hardening candidate. The maintainer path is to revise or replace it so WhatsApp isolates or wraps the actual model-facing `BodyForAgent`, transcript, reply-context body, and group-history text while keeping deterministic echo keys and trusted operator `messagePrefix` outside untrusted markers. Add focused WhatsApp tests for direct messages, audio transcripts, group history, reply context, prefix handling, and duplicate-message echo suppression before landing.

## Risks / Open Questions

- Current main still exposes WhatsApp inbound message text and audio transcripts to the model through raw `BodyForAgent`.
- Merging this PR unchanged would likely create a false sense of hardening because it wraps legacy `Body` while the active prompt path still prefers `BodyForAgent`.
- Merging this PR unchanged would make combined-message echo detection nondeterministic because wrapped content includes a fresh random marker ID.
- Merging this PR unchanged would wrap trusted operator `messagePrefix` text inside the untrusted-content block.
- The PR has no focused regression tests for WhatsApp direct messages, audio transcripts, group history, reply context, prefix handling, or duplicate-message echo suppression.
