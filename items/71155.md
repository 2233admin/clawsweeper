## Summary

Keep open. Current main still drops `message`, `stack`, and `cause` from diagnostic stability bundle errors: the bundle error type and serializer only preserve `name` and `code`, and the startup/restart failure paths pass thrown errors through that serializer. PR #71155 remains a relevant fix path, with maintainer feedback identifying it as the better base than closed duplicate #71352 but still requiring the requested review fixes before merge.

## Best Possible Solution

Keep PR #71155 open as the active implementation path for #71071. The best maintainer path is to update this PR with an escaped NUL-stripping regex, focused tests for NUL stripping and stack truncation, and then merge after the changed logging tests/checks pass. Closed PR #71352 should remain superseded by this narrower base.
