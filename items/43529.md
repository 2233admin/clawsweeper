## Summary

Keep this PR open. Current main has timeout-triggered compaction and explicit timeout error payloads, but it still does not implement the PR’s requested runReplyAgent session reset after all fallback models time out. The PR is a focused implementation candidate for the user problem from #20910, but it needs a rebase and tighter timeout classification before maintainer review or landing.

## Best Possible Solution

Keep this PR open as the current implementation candidate. Rebase it onto current main, move the tests into the current runReplyAgent/execution test split, implement a timeout reset hook or equivalent session-rotation path in runReplyAgent/runAgentTurnWithFallback, classify only structured all-model timeout exhaustion from fallback attempts, handle timeout isError payloads without meta.error, preserve heartbeat and transient HTTP guards, and then run the changed auto-reply/agent test lane before maintainer review.
