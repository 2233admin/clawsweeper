## Summary

Current main fixes the immediate microphone regression by allowing same-origin microphone in the default gateway Permissions-Policy header, but it still does not implement this PR's main requested capability: runtime configuration of `gateway.http.securityHeaders.permissionsPolicy`, including override and explicit-disable behavior. The PR remains a plausible implementation candidate after rebase and scope cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or contributor follow-up. The best path is to rebase it onto current main, preserve or explicitly re-decide the shipped `camera=(), microphone=(self), geolocation=()` default, add `gateway.http.securityHeaders.permissionsPolicy?: string | false` through config types, zod/generated schema metadata, public gateway docs, runtime resolution, and `createGatewayHttpServer`, and keep focused tests for default, override, and explicit-disable behavior. Split or remove unrelated auth-resolution or schema metadata changes before merge.
