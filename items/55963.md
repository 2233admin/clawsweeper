## Summary

Keep open. Current `main` still has both reported gaps: the bundled `session-memory` handler derives the filename date from UTC, and the bundled hook is only wired for `command:new` / `command:reset`, so `subagent_ended` cannot trigger it in production.

## Evidence

- **UTC filename date still uses ISO UTC day:** Current `main` still computes `dateStr` with `now.toISOString().split("T")[0]`, which is the UTC calendar date rather than local date for memory filenames.
  - file: [src/hooks/bundled/session-memory/handler.ts:85](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/hooks/bundled/session-memory/handler.ts#L85)
  - command: `nl -ba src/hooks/bundled/session-memory/handler.ts | sed -n '45,210p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Handler still ignores non-command events:** The handler returns unless `event.type === "command"` and the action is `new` or `reset`, so there is no current `subagent_ended` handling path in the shipped code.
  - file: [src/hooks/bundled/session-memory/handler.ts:53](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/hooks/bundled/session-memory/handler.ts#L53)
