## Summary

Keep this PR open. Current main still lacks the requested configurable sandbox initialization timeout, public config surface, cooperative startup cancellation, and timeout regression tests. The PR is the active successor to closed, unmerged PR #33556, but it still needs maintainer review because the provided diff has correctness and security-sensitive risks before it should merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best maintainer path is to request a revised patch that adds and documents `agents.defaults.sandbox.initTimeoutMs`, wires it through config types, strict schema, generated schema/help/docs as needed, wraps `resolveSandboxContext` in a bounded timeout, makes startup cancellation effective through backend factories, Docker/SSH/OpenShell setup, prune, browser image/network/container setup, CDP polling, and port reads, preserves the current sandbox browser security hash epoch, keeps focused timeout/abort tests, and splits unrelated OpenShell e2e guard changes unless they are necessary for this fix.
