## Summary

Keep #59280 open. Current main still has first-party `openclaw backup create` and `openclaw backup verify`, but no `openclaw backup restore <archive>` command, implementation, docs, or tests. The PR targets a real remaining core CLI gap, has community interest, and the author's OpenClaw Snapshot skill is a useful workaround rather than a shipped/superseding core fix. The PR still needs maintainer/author resolution of restore safety and integrity review feedback before it is merge-ready.

## Best Possible Solution

Keep the PR open or supersede it only with a maintainer-owned native restore implementation. The best product path is a safe core `openclaw backup restore <archive>` that shares archive verification checks, derives or validates restore targets from encoded archive paths, handles file assets and empty directories correctly, rejects malformed/zero-asset/duplicate-entry archives, re-checks conflicts at execution time, and lands with focused CLI/infra tests plus docs. The linked OpenClaw Snapshot skill can remain a useful community workaround, but it should not close the core CLI gap by itself.
