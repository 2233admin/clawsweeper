## Summary

Current `main` has some adjacent pieces from this PR’s review thread (legacy fail-closed coverage and warning-log formatting), but it does not implement the core startup behavior change. Unknown config keys still fail schema validation and are only removable through doctor/repair flows, so this PR is not implemented on `main` and should remain open.
