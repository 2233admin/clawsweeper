## Summary

Keep open. Current main has a `memory-lancedb` `agent_end` auto-capture hook and unit coverage for obvious trigger phrases, but that is not enough to close this concrete report: capture is intentionally heuristic-only, the main Pi runner fires `agent_end` without awaiting it, there is no end-to-end Telegram/cron/gateway persistence proof for the reported path, and the adjacent doctor warning still appears to match current source.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Reproduce on current main with `plugins.slots.memory = "memory-lancedb"`, an Ollama-compatible embedding config, and at least one gateway/channel turn containing a capturable trigger phrase. Then either fix the runtime path so `agent_end` capture is reliable end to end, or document/diagnose that `autoCapture` is heuristic-only and add skip-reason logging. The adjacent doctor UX should be fixed by making memory health resolve the selected memory slot/plugin surface instead of only the legacy backend config path.
