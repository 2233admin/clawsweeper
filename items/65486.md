## Summary

Current main still has the reported restart/resume gap. Exec approvals are process-local, stale IDs still resolve as `INVALID_REQUEST`, and the replay/restart recovery paths I inspected do not expire or re-register persisted `approval-pending` exec tool results after a gateway restart. The existing ClawSweeper keep-open comment remains consistent with current main; the related approval item #65437 is a separate Slack authorization issue, not a canonical fix for this stale approval-ID loop.

## Best Possible Solution

Keep this issue open. The fix should live in session replay/restart recovery or in a deliberate approval-manager persistence contract: when resumed history contains an exec result with `status: "approval-pending"` whose ID is absent from the live manager, rewrite it to an expired/error result so the agent re-issues exec, or securely persist and re-register pending approvals with original command/cwd/session binding metadata. Add a regression test that simulates gateway restart with an approval-pending transcript tail and proves the resumed turn does not keep resolving the stale ID.
