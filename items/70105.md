## Summary

Keep open. Current main still lacks the debug Event Log sticky-at-bottom behavior from PR #70105: the debug log has no scroll state/handler, no lifecycle scheduling, no constrained scroll container, and the event buffer is still newest-first. The PR is a focused Control UI bug fix, its bot review concern was addressed in follow-up commit 7b4232285f7fa150f87d6ce0702a6cc94113210b, and the related #69520 is about chat scroll interrupts rather than the debug event log surface.

## Best Possible Solution

Keep #70105 open for maintainer review and likely merge or cherry-pick. The right product/code path is a Control UI-local fix: add debug event log bottom-follow state and scheduler in `ui/src/ui/app-scroll.ts`/`app-lifecycle.ts`, wire the debug view scroll handler, constrain the debug log container, preserve overview recent-first display if buffer ordering changes, and keep the focused `ui/src/ui/app-scroll.test.ts` coverage.
