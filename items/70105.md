## Summary

Keep open. Current main still renders the Debug Event Log as a plain list without sticky-at-bottom state or an event-log scroll handler, and gateway events are still prepended into the shared event log buffer. The related #69520 work is chat-scroll-specific and does not supersede this debug Event Log fix.

## Best Possible Solution

Keep PR #70105 open for maintainer review or landing. The best code path is to add the debug Event Log sticky-at-bottom state/scroll handler, constrain the event-log panel to its own scroll area, keep overview ordering intentional, and land focused scroll behavior tests like the PR proposes.
