## Summary

Keep open. Current `main` does not implement this PR’s behavior: the shared status snapshot still marks reachability strictly from `gatewayProbe?.ok === true`, so a probe that connects successfully and then fails with `missing scope: ...` is still treated as unreachable in the `status` family of commands.
