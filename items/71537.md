## Summary

Keep #71537 open. Current main only has a partial session-memory reset fallback: it can read a reset archive when an empty live transcript path is already known, but `findPreviousSessionFile` still does not select `.jsonl.reset.*Z` archives when the active transcript is gone or the session file pointer is itself archived. The bundled `session-logs` skill also still documents/searches active `*.jsonl` files and does not teach agents to include reset/deleted archives. The PR is recent, focused, has relevant tests/docs, and the discussion says it is complementary to #60409 rather than superseded by it.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or cherry-pick the focused session-memory archive fallback and archive-aware `session-logs` skill docs after rebase and normal changed-lane validation, while leaving #60409 as the separate gateway history/readSessionMessages work.
