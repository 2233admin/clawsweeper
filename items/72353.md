## Summary

Keep #72353 open. Current main still only strips the literal `ollama/` prefix before native Ollama `/api/chat` requests, while the reported custom-provider case is not covered by source or tests. The related #72368 appears to target this exact gap, but the provided GitHub context says it is still open and the fix is not present in the checked-out main at 998e37fcb3d5b937fcf70d7509b677efba939f10.

## Best Possible Solution

Keep #72353 open and land #72368 or an equivalent bundled Ollama-provider fix. The implementation should normalize native Ollama wire model ids by stripping the active provider key for `api: "ollama"`, not just literal `ollama/`, and add regression coverage for a custom provider such as `ollama-remote/qwen3:32b` while preserving legitimate namespaced model ids.
