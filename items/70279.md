## Summary

Keep open. Current main partially addresses this area, but it still does not implement the requested CLI-specific pre-dispatch skip diagnostic or document the Claude CLI root/systemd constraint and `IS_SANDBOX=1` workaround from the reporter's follow-up.

## Best Possible Solution

Keep #70279 open as a focused CLI backend operability issue. The best maintainer path is to add a CLI-backend-specific pre-dispatch skip log for CLI candidates skipped by auth/cooldown/root-style guards, document Claude CLI under root/systemd with a supported non-root recommendation or explicit `IS_SANDBOX=1` caveat, and add regression coverage around fallback skips and session override persistence without reopening already-fixed session-runtime behavior.
