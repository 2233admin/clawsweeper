## Summary

Keep open. The reporter’s follow-up shows a working local workaround, and current main has addressed some Claude CLI configuration pitfalls, but the remaining maintainer requests are still real: pre-dispatch skips are logged only as model-fallback decisions, root/systemd Claude CLI constraints are not documented, and fallback model selection can still be persisted to sessions when the primary candidate was skipped before dispatch.

## Best Possible Solution

Keep #70279 open and narrow it into actionable maintainer work: add a CLI-backend-specific pre-dispatch skip diagnostic with auth/cooldown/root-style reasons, document the Claude CLI root/systemd constraint and supported workaround in the public CLI backend docs, and revisit fallback session persistence so a skipped primary does not silently pin future channel sessions to the fallback unless that is an explicit product choice.
