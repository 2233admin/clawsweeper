## Summary

Keep this PR open. Current main still has only a binary `tools.fs.workspaceOnly` filesystem guard and has not shipped the requested PathGuard ACL layer with canonicalized `allowedPaths` / `denyPaths`, structured policy-denial results, full apply_patch/media parity, or persisted spawned-subagent filesystem ceilings. The PR is a plausible core security implementation candidate, but it is large and security-sensitive and still needs maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer handling. The best path is to request a current-main rebase and focused security/design review, then either land a narrowed core implementation under `tools.fs` or replace it with maintainer-owned work that adds canonicalized deny-wins `allowedPaths` / `denyPaths`, structured policy denials, file/media/apply_patch/prompt-image coverage, schema/docs updates, and monotonic spawned-subagent filesystem ceilings. Coordinate the remaining filesystem scope with #7722 and the broader security profile in #8719.
