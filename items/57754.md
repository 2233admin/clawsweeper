## Summary

Keep #57754 open. Current main still renders Android chat bubbles without a long-press action surface, clipboard/share handling, or reply/quote wiring, and the only selectable chat text in the Android markdown renderer is still fenced/indented code blocks. The related PR #59603 is an open implementation attempt, but it has not landed on main, so the issue remains the canonical tracker for the missing Android chat copy/selection UX.

## Best Possible Solution

Keep #57754 open until Android main implements and verifies a chat-bubble action surface. The most concrete path is to review, update, or supersede PR #59603 with current-main code: add long-press actions to Android chat bubbles, copy whole-message text to the clipboard, expose selective copy without breaking `LazyColumn` scrolling, route share through the Android share sheet, and wire Reply into the existing chat draft/composer flow with focused Android UI tests or real-device verification.

## Risks / Open Questions

- The reporter already noted that wrapping all message text in `SelectionContainer` can conflict with `LazyColumn` scrolling; an implementation should prefer a scoped long-press action menu and a separate selectable dialog or similar isolated selection surface.
- Reply/quote needs composer-state wiring in `ChatSheetContent`/`ChatComposer`, not just a visual menu action, so the UX should be validated on real Android gesture behavior before landing.
