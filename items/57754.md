## Summary

Keep open. Current `main` does not implement general message copy/text selection for Android chat messages, and it also lacks the proposed long-press copy/share/reply affordances.

## Evidence

- **Selection exists only for code blocks:** `ChatMarkdown` wraps only fenced and indented code blocks in `SelectionContainer`; normal paragraphs and headings render as plain `Text`, so non-code message text is still not selectable/copyable.
