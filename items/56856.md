## Summary

Keep open. Current `main` already distinguishes several TUI activity states (`idle`, `streaming`, `running`, `waiting`) in the status bar, but it does not implement the issue's key remaining ask: a parent session that ends via `sessions_yield` is still treated as `idle`, not a distinct yielded/waiting-on-subagent state, and there is no dedicated `reconnecting` label path.

## Evidence

- **Status bar can already show distinct activity labels:** The TUI renders `connectionStatus | activityStatus` when not busy, and `activityStatus • elapsed | connectionStatus` when busy, so `idle`, `streaming`, `running`, and `waiting` are already separate labels in normal flows.
  - file: [src/tui/tui.ts:724](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/tui/tui.ts#L724)
  - command: `nl -ba src/tui/tui.ts | sed -n '585,726p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **`waiting` is only set immediately after sending a chat message:** After `client.sendChat(...)` resolves, the command handler sets `activityStatus` to `waiting`; this is a client-side post-send state, not a persisted yielded-subagent state.
  - file: [src/tui/tui-command-handlers.ts:604](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/tui/tui-command-handlers.ts#L604)
  - command: `nl -ba src/tui/tui-command-handlers.ts | sed -n '560,660p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Final chat events still collapse back to `idle`:** When a run finalizes normally, the event handler calls `finalizeRun(..., status: "idle")`, so there is no distinct status retained for a parent that yielded and is now waiting on child completion.
  - file: [src/tui/tui-event-handlers.ts:341](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/tui/tui-event-handlers.ts#L341)
  - command: `nl -ba src/tui/tui-event-handlers.ts | sed -n '288,398p'`

- **Lifecycle end also resets to `idle`:** Lifecycle `phase === "end"` sets `activityStatus` to `idle`; there is no separate yielded/waiting lifecycle state in the TUI handler.
  - file: [src/tui/tui-event-handlers.ts:458](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/tui/tui-event-handlers.ts#L458)

  - file: [src/agents/pi-embedded-runner/sessions-yield.orchestration.test.ts:28](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/pi-embedded-runner/sessions-yield.orchestration.test.ts#L28)
