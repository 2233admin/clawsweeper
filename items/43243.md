## Summary

Keep this PR open. Current main still lacks abort-aware queued-entry removal in `enqueueCommandInLane`, while the cron path already passes an abort signal down to `runEmbeddedPiAgent`; the remaining gap is exactly the queue boundary that this PR targets for open bug #43141. The PR is a focused implementation candidate, not conservative cleanup material.

## Best Possible Solution

Keep this PR open and review or land it after addressing the minor review comments and running the targeted command-queue and cron embedded-runner checks. An equivalent fix should add queued-entry abort removal to `enqueueCommandInLane`, preserve active-task behavior, clean up listeners, thread the existing cron abort signal through the default embedded-runner enqueue wrappers, and include regression tests for pre-aborted, queued-aborted, active-after-dequeue, and never-run aborted entries.
