## Summary

Keep open. Current `main` still lacks both behaviors the PR proposes: restart-health probes only with shell env gateway auth, and rate-limited embedded runs still rotate auth profiles before falling back to the configured model.

## Evidence

- **Restart health still uses env-only probe auth:** `confirmGatewayReachable` reads only `process.env.OPENCLAW_GATEWAY_TOKEN` / `OPENCLAW_GATEWAY_PASSWORD` and passes that directly to `probeGateway`; it does not load daemon config or call `resolveGatewayProbeAuthSafe`.
  - file: [src/cli/daemon-cli/restart-health.ts:72](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/cli/daemon-cli/restart-health.ts#L72)

That contrast matches the PR's claimed root cause that restart-health is the odd path out.
