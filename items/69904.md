## Summary

Keep PR #69904 open. Current main still lacks the stale transient system-event filtering this PR is trying to add, so the work is not implemented or superseded. The PR is also not ready to close as obsolete because it targets a real current gap, but the existing P1 review comments identify merge-blocking edge cases around mixed cron queues and non-completion exec events that should be addressed before merge.

## Best Possible Solution

Keep the PR open and require a revision that implements a generic stale transient event policy without broad heartbeat short-circuits: add timestamp/session-advance filtering for explicitly marked transient success/completion events, keep blocked/error/timeout/start/denied/cron notices durable, and add regression coverage for stale completion filtering, mixed stale-exec plus cron queues, and non-completion exec-event wakes.
