## Summary

Current `main` still treats `agents.defaults.heartbeat.model` as a plain string in both config schema and runtime, while other agent model surfaces use the shared `{primary,fallbacks}` shape. I did not find evidence that #66232 is already implemented or obsolete, so it should stay open.

## Evidence

- **Shared agent model schema already supports object form elsewhere:** `AgentModelSchema` accepts either a string or a strict object with `primary` and `fallbacks`, which matches the issue's requested shape for other model settings.
