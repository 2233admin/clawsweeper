## Summary

Current main still returns the full redacted config payload from both `config.get` and `config.patch`, and the protocol does not provide a scoped `paths`-style read option. This issue is not implemented on `main` and should remain open.

## Evidence

- **`config.get` still returns full snapshot:** The gateway handler for `config.get` reads the config snapshot and responds with `redactConfigSnapshot(...)`, which is the full redacted snapshot payload rather than a minimal acknowledgement.
  - file: [src/gateway/server-methods/config.ts:397](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/gateway/server-methods/config.ts#L397)
  - command: `nl -ba src/gateway/server-methods/config.ts | sed -n '395,625p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **`config.patch` success responses still embed `config`:** Both the noop branch and normal success branch of `config.patch` include `config: redactConfigObject(...)` in the response payload, so patch calls still return the full redacted config object.
  - file: [src/gateway/server-methods/config.ts:553](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/gateway/server-methods/config.ts#L553)
