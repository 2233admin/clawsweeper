## Summary

Keep PR #70651 open. Current main still lacks a Feishu card table-limit fallback on the outbound card paths, and the PR is a focused fix attempt for the real 230099/11310 delivery-loss bug described in #69789 and related Feishu table-limit reports. It should not be closed as implemented or superseded. The PR still needs revision before merge: the latest head still checks JSON.stringify on thrown objects before inspecting Axios response data, so circular Axios-style errors can bypass the new fallback, and there is no regression test coverage in the two-file patch.

## Best Possible Solution

Keep PR #70651 open and request a small revision: inspect Axios response.data before any JSON.stringify attempt in parseCardKitError, add focused tests for the supported error shapes and outbound fallback paths, and coordinate scope with the broader Feishu table-limit tracking so direct outbound and reply-dispatcher behavior do not diverge.
