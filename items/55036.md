## Summary

Current main still suppresses commentary-phase assistant output before it reaches Discord-visible delivery. The related closed report #55236 covered late main-result delivery, but this issue’s separate early-commentary durability/loss window remains unimplemented.

## Best Possible Solution

Keep this issue open for Discord/channel delivery work. The likely fix belongs in the assistant subscription and channel delivery boundary: materialize approved user-visible commentary as a block or preview/finalizable payload before tool or restart boundaries, route it through the existing reply dispatcher/restart-deferral machinery, and add regression tests for early commentary plus restart-sensitive turns. If maintainers decide commentary suppression is intentional product policy, document that explicitly and close only after that decision.
