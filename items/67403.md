## Summary

Keep open. Current `main` and the latest release still use contiguous `includes(queryLower)` checks in the memory-wiki lexical path, so the reported long multi-term wiki-query drop is not implemented on `main`. This PR is also not merge-ready cleanup material: it is unmerged, and its review thread already flags plausible ranking/candidate regressions that were not resolved in `#67403` itself.

## Evidence

- **Current main still gates digest candidates on full-string inclusion:** `buildDigestCandidatePaths` lowercases the digest metadata and immediately returns score 0 unless `metadataLower.includes(queryLower)`. That matches the issue report's root cause for long multi-term queries that do not appear as one contiguous phrase.
