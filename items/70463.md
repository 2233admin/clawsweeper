## Summary

Keep open. Current `main` still has two independent recovery paths for WhatsApp: the channel’s own watchdog/reconnect loop and the gateway health monitor’s disconnected-channel restart path. The code does not show an explicit coordination guard between them, and the user-facing config/docs still expose `web.reconnect.*` and health-monitor toggles but not the requested WhatsApp watchdog/message-timeout tuning. That leaves the reported race plausible on current `main`, so there is not strong enough evidence to auto-close conservatively.
