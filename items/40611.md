## Summary

Keep open. Current main still retries `requests-in-flight` heartbeat skips after 1 second, the runner comments and tests assert that behavior, and there is no `heartbeat.retryDelayMs` or separate in-flight retry delay in the config schema. The exact proposed fix in PR #40629 is closed unmerged, while #13744 and #14051 are broader related items rather than a canonical replacement for this regression.

## Best Possible Solution

Implement a core heartbeat scheduling fix for `requests-in-flight` skips: add a separate in-flight retry delay in `src/infra/heartbeat-wake.ts` with a conservative default such as 60 seconds, optionally expose it under `agents.defaults.heartbeat` and per-agent heartbeat config, wire it through runner config updates, and update wake/runner tests plus config docs/schema. Keep #13744 for broader session-lock scope/configuration and #14051 for activity-based heartbeat semantics.
