## Summary

Current `main` has a Control UI exec-approval modal for actual `exec`/plugin approval events, but it does not implement the issue’s requested chat-side "Approve and Run" action for assistant-suggested shell commands. Suggested code blocks in chat are still copy-only, so the reported copy/paste friction is not fully resolved.

## Evidence

- **Chat code blocks are still copy-only:** The markdown renderer adds only a `Copy` button to fenced and indented code blocks; there is no run/approve action for assistant-suggested shell snippets.
  - file: [ui/src/ui/markdown.ts:434](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/ui/src/ui/markdown.ts#L434)

- **Control UI approval UI exists, but only for exec approval events:** The browser modal renders pending exec approvals with command metadata and `Allow once` / `Always allow` / `Deny` actions. This is an approval surface for already-requested exec runs, not a parser/action card for ordinary chat code blocks.
  - file: [ui/src/ui/views/exec-approval.ts:29](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/ui/src/ui/views/exec-approval.ts#L29)
