## Summary

Keep open. Current `main` has Control UI support for copying assistant code blocks and for approving actual `exec`/`system.run` requests, but I did not find an implementation that turns assistant-suggested terminal commands in chat into runnable command cards with an `Approve and Run` action.

## Evidence

- **Assistant code blocks only render a Copy button:** The markdown renderer wraps fenced and indented code blocks with `code-block-copy` buttons only; there is no run/approve action added to assistant code blocks.
  - file: [ui/src/ui/markdown.ts:434](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/markdown.ts#L434)
  - command: `nl -ba ui/src/ui/markdown.ts | sed -n '430,470p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Control UI chat handler only copies code blocks to clipboard:** The chat view listens for `.code-block-copy` clicks and calls `navigator.clipboard.writeText(code)`. No execution or approval flow is attached to assistant-rendered code blocks.
  - file: [ui/src/ui/views/chat.ts:728](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/views/chat.ts#L728)

  - file: [ui/src/ui/views/exec-approval.ts:63](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/views/exec-approval.ts#L63)
