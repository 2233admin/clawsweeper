## Summary

Keep open. Current `main` still queues undelivered hook results and hook errors back into the main session as untrusted system events, mapped hooks still default their display name to `Hook` (so `Hook Hook ...` remains possible), and the hooks config schema still has no suppression knob for this relay path.

## Evidence

- **Hook failures and undelivered runs are still re-injected as system events:** `dispatchAgentHook` enqueues `Hook ${safeName}: ...` when `!result.delivered`, and enqueues `Hook ${safeName} (error): ...` on exceptions, both to `mainSessionKey` with `trusted: false`.
