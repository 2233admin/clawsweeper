## Summary

Current `main` has a newer parameter-validation helper that adds `(received: ...)` hints for malformed tool calls, but it does not implement this PR's tool-specific example strings, and the tests still codify the generic `{}` failure path. That means the requested behavior is not already on `main`, so this PR is not a safe auto-close candidate.
