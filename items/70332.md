## Summary

Keep open. Current main still has the older dreaming diary behavior that #70332 is trying to harden: managed inline dreaming blocks are stripped only from marker starts and stop at any heading, request-scoped fallback writes the first raw snippet/promoted fragment, and diary dedupe fingerprints include the timestamp. The related items do not fully supersede this PR, and the PR itself has a review finding that should be fixed before merge.

## Best Possible Solution

Keep #70332 open, but require an update before merge: fix the italic-line fingerprint review finding, reconcile the fallback cleanup with #70523, and keep the regression tests for nested managed block stripping, human-readable request-scoped fallback text, and duplicate diary body suppression. Current main does not yet implement those behaviors well enough to close this PR.
