## Summary

Keep open. Current `main` does not contain this Feishu fix: bare text posts are still always emitted as a single markdown post element, so the reported underscore-in-URL rendering problem remains plausible, and the PR’s targeted regression coverage is not present.

- **Current main still forces markdown for all Feishu text posts:** `buildFeishuPostMessagePayload` returns `zh_cn.content` with a single `{ tag: "md", text: messageText }` element and has no bare-URL detection or `{ tag: "a" }` branch.
  - file: [extensions/feishu/src/send.ts:432](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/extensions/feishu/src/send.ts#L432)
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
