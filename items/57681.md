## Summary

Current `main` already contains the model-switch self-heal and retry-bounding logic that addresses the reported permanent "noodling" state after fallback/provider switches, and the same code is present in release `v2026.4.23`.

## Evidence

## Close Comment

Closing this as implemented on current `main` after a Codex review.

- `src/auto-reply/reply/model-selection.ts:173` now clears transient auto-failover overrides and resets the in-memory provider/model back to the configured primary, so a fallback switch does not leave the session stuck on the wrong model.
- `src/auto-reply/reply/agent-runner-execution.ts:82` and `src/auto-reply/reply/agent-runner-execution.ts:1401` now cap repeated `LiveSessionModelSwitchError` retries and surface a final error instead of looping in the TUI forever.
- `src/agents/model-fallback.ts:921` treats live-session switch conflicts as normal failover and continues the fallback chain instead of rethrowing into another outer retry cycle.
- Regression tests were added for the same failure shape in `src/auto-reply/reply/model-selection.test.ts:603`, `src/auto-reply/reply/agent-runner-execution.test.ts:2220`, and `src/cron/isolated-agent/run.live-session-model-switch.test.ts:70`.
- The same code is already present in release `v2026.4.23` (`9b18ae877018a34933e27bb7b2986cb97e9602aa`).
