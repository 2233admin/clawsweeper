## Summary

Keep #40311 open. Current main at 8ba82534e6c98410e19b226f3534b285fd75d278 still does not implement Brave Goggles support: the repo has no `goggles`/`unsupported_goggles` symbols, the bundled Brave tool schema exposes only the existing search fields, the Brave runtime URL builders and cache key omit the parameter, and docs/tests do not cover it. The PR discussion remains useful, but the patch targets an older core web-search shape and should be ported to the current provider-owned Brave plugin.

## Best Possible Solution

Keep #40311 open and ask for a rebase/port into the current bundled Brave provider: add `goggles` to the active Brave tool schema, thread it through `extensions/brave/src/brave-web-search-provider.runtime.ts` for both web and llm-context modes, include it in the Brave cache key without lossy normalization collisions, and update Brave/web docs plus focused extension tests. Keep the behavior provider-owned rather than reintroducing Brave-specific logic into core.
