## Summary

Keep open. Current main still does not expose Brave Goggles in the active provider-owned Brave web_search implementation. This PR remains a useful feature candidate, but its current diff targets the older core web-search path instead of the current Brave plugin/provider boundary, so it should be rebased and ported rather than closed.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but ask for a rebase/port into the active Brave provider. Add optional goggles support to extensions/brave shared schema and provider schema, pass it through both Brave web and llm-context URL builders, include it in the cache key without lossy normalization or missing-value collisions, update docs/tools/brave-search.md and docs/tools/web.md, and add focused Brave provider tests. No new core/plugin API appears necessary.
