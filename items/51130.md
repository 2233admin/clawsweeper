## Summary

Keep open. Current main has partial restart-acknowledgement support for explicit restart/config/update flows, but it does not fully implement durable user notification for completed work across gateway restarts. Some completion delivery still falls back to ephemeral in-memory system events, and current docs explicitly say pending sub-agent announce work is lost on restart.

## Risks / Open Questions

- Closing as implemented would overstate what the current restart-sentinel work covers; it handles explicit restart acknowledgements, not all completed-work notification gaps.
- The issue scope is broader than restart confirmation alone, and at least one documented path (`subagents`) still loses pending completion announce work on restart.
- Task records are durable, but completion delivery fallback is not fully durable because queued system events are in-memory only.
