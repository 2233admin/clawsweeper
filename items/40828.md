## Summary

Keep #40828 open. Current main still does not provide the requested `/total_usage` chat command, Telegram quick-select UI, or `CostUsageSummary.models` aggregate, so this is not implemented or duplicated by a canonical open item. Main does have overlapping usage-dashboard model/provider aggregates now, so the PR should be reworked against the current usage surfaces rather than auto-closed.

## Best Possible Solution

Keep the PR open but require a current-main rework: either implement a focused `/total_usage` command on top of the existing `sessions.usage`/`usage.cost` aggregates, or intentionally extend `CostUsageSummary` with provider+model aggregates. Preserve date prefiltering, key model totals by provider plus model, and add tests for all-time, empty/no-usage, narrow ranges, and provider/model collisions before merge.
