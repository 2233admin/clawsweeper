## Summary

Keep this PR open. Current main still has the reported Discord `thread-reply` attachment gap: `send` accepts `media`, `path`, and `filePath`, but `thread-reply` still reads only `media` and calls the Discord runtime without the media access options needed for scoped local files. The linked same-author bug #43570 remains open, so this PR is still an implementation candidate rather than cleanup.

## Best Possible Solution

Keep this PR open and update it against current main. The best patch is still narrow: in `extensions/discord/src/actions/handle-action.guild-admin.ts`, make `thread-reply` read `media ?? path ?? filePath`, forward the same scoped media options used by `send` into `handleDiscordAction`, and add current-location tests for `filePath`, `path`, precedence, and media option passthrough before landing it with #43570.
