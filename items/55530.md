## Summary

Current main already contains the plugin-runtime/channel-registry fix for tools/channels re-registering from mismatched runtime loads, and the changelog says it shipped in v2026.3.28.

## Close Comment

Closing from a Codex review because this appears to already be fixed on current `main`. The shipped changelog for `v2026.3.28` explicitly calls out that plugin tools/channels no longer "re-register" from mismatched runtime loads (`CHANGELOG.md:2001`). The implementation is present in `src/plugins/loader.ts:1015` (reuse compatible active registries instead of reloading), `src/plugins/runtime.ts:151` (pin the channel registry across later registry swaps), with regression coverage in `src/plugins/loader.runtime-registry.test.ts:27` and `src/plugins/runtime.channel-pin.test.ts:99`. Feishu tool wiring itself still comes through a single `registerFull` path in `extensions/feishu/index.ts:73`, so the repeated same-startup registration logs reported against `2026.3.24` are already addressed by the runtime fix. If this is still reproducible on a current release, please open a new issue with fresh logs and the exact current version/config/plugin set.
