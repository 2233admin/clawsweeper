## Summary

Keep this issue open. Current main has adjacent ACP and ACPX improvements, but I did not find source, tests, docs, release notes, dependency-source evidence, or live-smoke proof that the specific rapid two-spawn Claude Code ACP launch failure is fixed.

## Best Possible Solution

Keep this open until current main is verified with the bundled acpx 0.6.1 or newer using two rapid `sessions_spawn({ runtime: "acp", agentId: "claude", mode: "run" })` calls. If it still reproduces, fix the ACPX runtime/control-plane launch path with a backend-level init guard, health check, or retry and add focused regression coverage around distinct ACP session spawn/init dispatch. If it no longer reproduces, close later with live proof plus the exact fixing commit or release.
