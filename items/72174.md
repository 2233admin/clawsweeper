## Summary

Keep PR #72174 open. Current main still has the reported attach-only bug: both the `--attach-only`/`--no-launchd` path and the Debug Settings toggle write the disable marker, then call the launchd disable/uninstall path. The related merged launchd lifecycle work improves daemon stop/restart behavior, but it does not supersede this app-side attach-only fix.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best product direction is the PR's marker-only attach-only path for both the launch flag and Debug Settings toggle, plus the focused regression test proving no daemon commands are issued.
