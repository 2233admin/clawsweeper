## Summary

PR #72174 should remain open. Current main still has the reported macOS attach-only bug: both the `--attach-only` / `--no-launchd` startup path and the Debug Settings toggle write the disable marker, then still call the launchd disable/uninstall path. The PR discussion is consistent with the code: it links the matching prior user report #42530, and the Greptile review correctly describes the PR as a narrow marker-only fix with a regression test.

## Best Possible Solution

Keep PR #72174 open for maintainer review or landing. The best fix is the PR’s marker-only attach-only path for both the launch flags and Debug Settings toggle, plus the focused regression test proving attach-only writes the marker without issuing daemon commands.
