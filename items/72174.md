## Summary

PR #72174 should remain open. Current main still has the macOS attach-only bug the PR targets: the launch flag path and Debug Settings toggle persist the disable marker, then still call the LaunchAgent disable/uninstall path. The related launchd lifecycle fixes on main address stop/restart semantics, but they do not make attach-only marker-only.

## Best Possible Solution

Keep PR #72174 open for maintainer review or landing. The best implementation is the PR's marker-only attach-only helper used by both `MenuBar.swift` launch flags and the `DebugSettings.swift` toggle, plus the regression test proving attach-only writes the marker without issuing daemon commands.
