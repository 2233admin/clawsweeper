## Summary

Keep open. Current `main` still only refreshes `models.json` during startup for the default agent dir, and only when `agents.defaults.model.primary` is explicitly set, so the stale-cache report for configured non-default agent dirs remains plausible. The PR discussion also shows the author reproduced the gap and revised the patch to cover the no-default-primary case, which is not implemented on `main` yet.

## Evidence

- **Startup warmup only targets the active default agent dir:** `prewarmConfiguredPrimaryModel()` resolves a single `agentDir` via `resolveOpenClawAgentDir()` and calls `ensureOpenClawModelsJson()` only for that directory before model resolution.
