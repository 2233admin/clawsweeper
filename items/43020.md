## Summary

Keep this issue open. Current main still routes Mattermost preview streaming through one draft post, disables core block streaming for that path, and only clears local partial text at assistant-message boundaries rather than finalizing or rotating the Mattermost draft post. The related fix PRs in the provided context are closed unmerged.

## Best Possible Solution

Keep this open and fix it in the Mattermost plugin path. At assistant-message boundaries, the preview streaming path should preserve the completed streamed segment by flushing/finalizing or rotating the draft post before the next segment updates the preview, while suppressing duplicate final payloads. Add a focused Mattermost regression test that simulates text, a tool boundary/new assistant message, more text, and final delivery.
