## Summary

Current main still implements active-memory as a blocking `before_prompt_build` pre-step. The later timeout work appears to bound the wait and fail open after the configured timeout, but it does not solve #69182's central request that active-memory should not delay the first visible model output by the full timeout.

## Best Possible Solution

Keep #69182 open as the focused performance/product follow-up. The likely path is in `extensions/active-memory/index.ts`: either make recall asynchronous relative to first visible output, add a lower/auto-tuned latency budget for the prompt-build hook, or introduce a separate non-blocking active-memory mode while preserving the current bounded blocking behavior for users who prefer recall quality over latency.
