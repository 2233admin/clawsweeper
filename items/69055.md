## Summary

Keep #69055 open. Current main has generic outbound delivery diagnostics and cron run-log delivery status, but it does not emit the requested single structured `cron.delivery.sent` attribution record per cron outbound attempt with cron job/run IDs, agent/account IDs, target, status, error, and provider message ID. The companion #69054 remains a separate config-lint request, not a canonical duplicate for this logging gap.

## Best Possible Solution

Keep the issue open and implement a first-class, privacy-bounded core telemetry path: thread a typed cron delivery attribution object from `dispatchCronDelivery` into `deliverOutboundPayloads`, preserve provider message IDs from channel adapters, and emit exactly one structured `cron.delivery.sent` record after each success or error attempt with job/run/agent/account/target/status fields. Add focused tests around cron direct delivery and outbound success/error paths.
