## Summary

Current `main` does not implement suppression of assistant text that appears before tool calls. The gateway explicitly flushes buffered pre-tool text so clients can render it, tests assert that both intermediate and final payloads preserve that pre-tool text, and the config/schema/docs expose block-streaming and preview tool-progress controls but no `suppressIntermediateText`-style option.

## Evidence

- **Gateway intentionally flushes pre-tool text:** `server-chat` flushes pending assistant text on tool-start so clients can render complete pre-tool text above tool cards.
  - file: [src/gateway/server-chat.ts:932](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/gateway/server-chat.ts#L932)

  - file: [src/gateway/server-chat.agent-events.test.ts:617](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/gateway/server-chat.agent-events.test.ts#L617)
