## Summary

Keep open. Current main still has an actionable bundled Signal group-ingress bug: Signal group messages are gated by sender-based groupAllowFrom before the Signal group-id config path is considered, and requireMention still defaults to true when no explicit group/default value is configured. The current docs indicate groupAllowFrom is a sender allowlist and groups is keyed by Signal group id, so the fix should preserve that split rather than treating groupAllowFrom as the group-id allowlist.

## Best Possible Solution

Keep this issue open and fix the bundled Signal plugin. The likely path is to make Signal group admission consult channels.signal.groups/resolveChannelGroupPolicy for the group id under groupPolicy="allowlist" before sender authorization drops the event, while preserving groupAllowFrom as sender authorization. Add regression tests for an allowed Signal group id with no sender groupAllowFrom, for sender allowlist enforcement inside configured groups, and for the explicit requireMention:false workaround. Decide separately whether an explicit group entry without requireMention should default to false or whether docs/config should continue requiring an explicit override.
