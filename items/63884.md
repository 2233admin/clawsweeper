## Summary

Keep open. Current main enforces Feishu `requireMention` for whether the bot replies, but it still records unmentioned group messages into the pending group-history buffer and injects that buffer into the next mentioned turn. I found no `contextIsolation`, `contextPolicy`, or other per-bot mention-only context policy in the Feishu schema, docs, or generated config metadata. `historyLimit: 0` is a broad workaround that disables pending group history, not the requested per-bot isolation.

## Best Possible Solution

Keep the issue open and implement this in the bundled Feishu plugin as an explicit, backwards-compatible context policy. The likely path is to add typed config at top-level and per-group scope, such as `contextPolicy: "all" | "mention-only"`, then change `extensions/feishu/src/bot.ts` so unmentioned group messages are not recorded into pending history when mention-only isolation is enabled, and so messages mentioning other bots can be excluded once bot identity matching is reliable. Update `extensions/feishu/src/config-schema.ts`, generated config metadata/docs, and Feishu tests covering multi-bot group pending-history behavior.

## Risks / Open Questions

- A backwards-compatible implementation needs a default that preserves today's pending-history behavior unless the new policy is enabled.
- Reliable filtering of messages aimed at other Feishu bots may need the mention-identity work tracked around #40782, especially where Feishu delivers `union_id` instead of `open_id`.
