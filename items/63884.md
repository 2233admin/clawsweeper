## Summary

Keep open. Current main still treats Feishu `requireMention` as a reply gate, not a context-recording boundary: unmentioned group messages can be buffered and injected into the next mentioned turn, and there is no Feishu `contextIsolation`, `contextPolicy`, `excludeOtherBots`, or addressed-bot-scoped history policy on main.

## Best Possible Solution

Keep this issue open. Implement a backwards-compatible Feishu plugin context policy, likely global and per-group, defaulting to today’s behavior. When enabled, `extensions/feishu/src/bot.ts` should avoid recording unmentioned group messages into pending history and should filter messages addressed to other bots once reliable multi-bot mention identity is available. Update `extensions/feishu/src/config-schema.ts`, generated config metadata if required, `docs/channels/feishu.md`, and Feishu tests covering unmentioned pending-history replay plus multi-bot mention handling.
