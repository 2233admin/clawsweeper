## Summary

Keep open. Current main still has the Signal daemon shutdown race described in #22676: the daemon handle stop method is synchronous, only sends SIGTERM, and the monitor teardown path does not await signal-cli exit before gateway restart can continue. The latest release tag v2026.4.24 contains the same implementation. The related fix PR #46357 was closed unmerged, and the reporter confirmed the issue again on April 24, 2026.

## Best Possible Solution

Keep #22676 open as the canonical tracker. The best fix is to make the Signal daemon handle stop async, wait for signal-cli exit with a bounded timeout and SIGKILL escalation, and update monitor/channel teardown so gateway restart does not proceed until the auto-started daemon has actually exited. Add a focused regression test around abort/restart teardown so the channel task cannot report clean stop before daemon shutdown completes.
