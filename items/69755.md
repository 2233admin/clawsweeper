## Summary

Current `main` still emits `Run: openclaw doctor --non-interactive` as an unconditional restart/config/update follow-up and routes that exact message back into post-restart session surfaces. I found no context-aware gating, fallback text, or approval-capable rerouting for heartbeat/approval-constrained contexts, so the reported UX gap is still present.
