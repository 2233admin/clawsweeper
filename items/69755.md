## Summary

Current main still unconditionally formats restart/config/update follow-up as `Run: openclaw doctor --non-interactive` and appends it to restart sentinel messages. I did not find context-aware wording, deferral, or routing for heartbeat/approval-constrained delivery surfaces, so the reported UX problem remains actionable.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the restart-sentinel follow-up path: make `formatDoctorNonInteractiveHint` or the sentinel payload builder aware of delivery/execution context, then render wording such as “Recommended follow-up: run `openclaw doctor --non-interactive` in a terminal or approvals-capable Control UI” when the current surface cannot reliably complete approval-gated command execution. Add focused tests for restart, config-patch, and update sentinel messages so the wording distinguishes “recommended” from “actionable here.”
