## Summary

Keep open. Current main still emits the `openclaw doctor --non-interactive` restart follow-up as an unconditional `Run:` hint, including after `config.patch` / `config.apply`, and the restart-sentinel delivery path does not distinguish terminal/control-UI actionable contexts from heartbeat or approval-constrained channel contexts.

## Best Possible Solution

Keep the issue open and implement context-aware restart-sentinel doctor guidance. The likely fix is to carry or derive an originating surface/actionability signal for config.patch/config.apply/update/restart sentinels, format constrained channel/heartbeat contexts as a recommended terminal or approvals-capable Control UI follow-up instead of an immediate `Run:` command, and add regression tests for config patch/apply plus restart-sentinel delivery formatting.
