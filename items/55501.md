## Summary

Current `main` only partially covers this umbrella request. Some lower-priority asks already exist (`openclaw config schema`, cron retry/failure alerts, per-task heartbeat intervals, subagent tool allow/deny), but the top token-efficiency and subagent-UX requests are not clearly implemented away: bootstrap files are still injected on every turn by default, and subagent completion delivery still carries internal metadata/stats that must be rewritten out before user delivery.

## Evidence

- **Bootstrap injection still defaults to every turn:** The docs still state that `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, and `MEMORY.md` are injected into the context window on every turn unless a file-specific gate applies.
