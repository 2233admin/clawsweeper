## Summary

Keep open. Current `main` does not implement `%DATE%` expansion for `logging.file`, and the PR still appears to have unresolved correctness risks in its review history, so there is not strong evidence for a conservative close.

## Evidence

- **Current logger treats `logging.file` as a literal path:** `resolveSettings()` assigns `const file = cfg?.file ?? defaultRollingPathForToday();` with no `%DATE%` substitution step, so custom `logging.file` values are used verbatim on `main`. `buildLogger()` only prunes when the resolved filename already matches the built-in rolling pattern.
  - file: [src/logging/logger.ts:127](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/logging/logger.ts#L127)
  - command: `nl -ba src/logging/logger.ts | sed -n '89,175p'`
