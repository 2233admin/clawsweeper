## Summary

Keep open. Current `main` does not implement explicit active-memory chat types, and the proposed PR still has an unresolved correctness issue called out in review.

## Evidence

- **Current manifest still rejects `explicit`:** `extensions/active-memory/openclaw.plugin.json:20` still limits `allowedChatTypes` to `"direct"`, `"group"`, and `"channel"`; `"explicit"` is not in the schema on current `main`. That means the PR's requested behavior is not already implemented.
  - file: [extensions/active-memory/openclaw.plugin.json:20](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/extensions/active-memory/openclaw.plugin.json#L20)
  - command: `nl -ba extensions/active-memory/openclaw.plugin.json | sed -n '15,100p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Runtime normalization/gating still lacks `explicit`:** `extensions/active-memory/index.ts:67`, `:102`, `:612`, and `:874` still type/filter/resolve chat types as only `direct|group|channel`. Current `main` therefore does not accept or classify explicit sessions for active-memory gating.
  - file: [extensions/active-memory/index.ts:67](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/extensions/active-memory/index.ts#L67)
  - command: `nl -ba extensions/active-memory/index.ts | sed -n '60,130p'; nl -ba extensions/active-memory/index.ts | sed -n '600,640p'; nl -ba extensions/active-memory/index.ts | sed -n '874,910p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Explicit session keys preserve raw session ids:** `src/agents/command/session.ts:51` builds explicit session keys as `agent:<agent>:explicit:<sessionId.trim()>` without escaping or normalizing embedded `:channel:`, `:group:`, or `:direct:` segments. That makes the PR review's ordering concern concrete.
  - file: [src/agents/command/session.ts:51](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/command/session.ts#L51)
  - command: `nl -ba src/agents/command/session.ts | sed -n '45,58p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **PR review found an unresolved blocker in the proposed fix:** The existing review on commit `88aff48335d6d3ae25a9e6189848ae1f877b7ee1` notes that the PR checks `:channel:` before `:explicit:` in `resolveChatType`, so explicit session ids containing those substrings would be misclassified. The PR timeline shows only that single commit and no follow-up fix.
  - sha: [88aff48335d6](https://github.com/openclaw/openclaw/commit/88aff48335d6d3ae25a9e6189848ae1f877b7ee1)

## Risks / Open Questions

- Current `main` still rejects `allowedChatTypes: ["explicit"]` for the active-memory plugin.
- Closing this PR would discard a still-relevant bugfix request that has not landed yet.
- Merging the PR as-is would risk misclassifying explicit session keys that contain `:channel:`, `:group:`, or `:direct:` in the raw session id.
