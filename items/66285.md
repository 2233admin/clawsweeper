## Summary

Keep this PR open. Current main still does not support `explicit` as an Active Memory chat type: the manifest schema, TypeScript config types, normalization filter, runtime chat-type resolver, docs, and tests remain limited to `direct`, `group`, and `channel`. The PR is a focused implementation candidate, but its current resolver change should be adjusted so structured `agent:<agent>:explicit:<sessionId>` keys are parsed before broad marker checks.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Update the resolver to parse structured `agent:<agent>:explicit:<sessionId>` keys before broad marker checks, then land the equivalent Active Memory schema/types/normalization/docs/test patch with regression coverage for explicit sessions, including marker-containing explicit session IDs.
