## Summary

Keep #71062 open. Current main still lacks the PR's central fixes: the Pi client-tool adapter returns a normal pending stub without `terminate: true`, `/v1/responses` streamed function calls do not emit `response.function_call_arguments.{delta,done}`, and the upstream Responses stream parser still ignores `.done` argument events. The PR is a focused, recent bug-fix with concrete reproduction details, targeted tests, and follow-up to review comments, so no conservative close reason applies.

## Best Possible Solution

Keep #71062 open as the current canonical fix path. The best implementation is to land an equivalent of the PR after review: add `terminate: true` to the client-tool handoff stub, emit `response.function_call_arguments.delta` and `.done` between function-call output item bookends, handle done-only upstream args in the Responses stream parser, and keep the targeted regression tests. Do not close as implemented until that behavior is present on main.
