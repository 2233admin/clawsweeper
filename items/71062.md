## Summary

Keep open. Current main still lacks the PR's core fixes: client-tool handoff returns a plain pending stub without `terminate: true`, `/v1/responses` streaming does not emit dedicated `response.function_call_arguments.{delta,done}` events, and the upstream Responses stream parser handles argument deltas but not `.done`-only argument payloads. This is a focused Gateway/Agents bug fix, not a ClawHub item or duplicate cleanup candidate.

## Best Possible Solution

Keep #71062 open and continue review toward a corrected core fix. The right product/code path is in `src/agents` and `src/gateway`: return `terminate: true` on the Pi client-tool handoff stub, emit the Responses-compatible `function_call_arguments.delta` and `.done` SSE events while preserving a populated final/completed response snapshot, add the `.done`-only parser branch, and keep the focused regression tests in the agent adapter, gateway SSE, and Responses stream parser tests.
