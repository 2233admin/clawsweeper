## Summary

Keep open. Current `main` does not implement Android-hosted Gateway support: the service layer still only supports `darwin`, `linux`, and `win32`, and the Android docs/README still describe Android as a companion node that connects to a Gateway running on another machine.

## Evidence

- **Gateway service platform guard still excludes Android:** `SupportedGatewayServicePlatform` is limited to `darwin | linux | win32`, and `resolveGatewayService()` throws `Gateway service install not supported on ${process.platform}` for anything else, which matches the issue report on Android/Termux.
  - file: [src/daemon/service.ts:170](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/daemon/service.ts#L170)
  - command: `nl -ba src/daemon/service.ts | sed -n '170,225p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Published Android docs say Android does not host the Gateway:** The current Android platform doc explicitly says `Role: companion node app (Android does not host the Gateway)` and `Gateway required: yes (run it on macOS, Linux, or Windows via WSL2)`.
  - file: [docs/platforms/android.md:14](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/docs/platforms/android.md#L14)

  - file: [apps/android/README.md:136](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/apps/android/README.md#L136)
