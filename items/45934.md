## Close Comment

Closing from a Codex review. The status/probe part of this PR is already on current `main`: `src/commands/gateway-status/helpers.ts:13` and `src/commands/gateway-status/helpers.ts:272` classify `missing scope: ...` probe failures as degraded but reachable, with coverage in `src/commands/gateway-status/helpers.test.ts:226` and matching docs in `docs/cli/gateway.md:224`. For the remaining OpenSandbox backend work, current `main` has moved sandbox backends onto the plugin path: `src/plugin-sdk/sandbox.ts:24` exports `registerSandboxBackend`, `src/agents/sandbox/backend.ts:61` explicitly tells callers to load a plugin for unknown backends, and `extensions/openshell/index.ts:1` is the current precedent for a vendor-specific sandbox backend. Given that direction, OpenSandbox belongs as a plugin/ClawHub backend against the current sandbox API rather than as a core PR, so I’m closing this combined PR.
