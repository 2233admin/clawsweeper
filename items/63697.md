## Summary

Keep this PR open. Current main still has the native Chat UI behavior the PR is trying to change: exact reset/compact slash commands are intercepted in `OpenClawChatViewModel` and routed through local `sessions.reset` / `sessions.compact` before `chat.send` can run. The PR remains a plausible implementation candidate, but it has grown into broader iOS/macOS chat UI work and has unresolved functional and security-sensitive review concerns, plus a same-author companion PR that is still open.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either re-scope it to a focused native Chat UI change that forwards slash commands through `chat.send` while preserving reset/compact transcript semantics with regression tests, or split the broader UI/model-picker/exec-approval work into smaller PRs coordinated with #67493. Before merge, resolve the outstanding review comments and explicitly verify the operator approval scope behavior does not regress.
