## Summary

Keep PR #40747 open. Current main still requires `after_block_id` for `feishu_doc(action: "insert")` and has no insert `index` parameter, so the PR's central user problem remains unresolved. The PR is focused and still useful, but should be updated to validate `index` as a non-negative integer and add targeted regression coverage before merge.

## Best Possible Solution

Keep the PR open or land an equivalent maintainer patch in `extensions/feishu`: make `after_block_id` optional for `action: "insert"`, add `index: Type.Optional(Type.Integer({ minimum: 0, ... }))`, pass `p.index` into `insertDoc`, use `index ?? 0` for document-root insertion when `after_block_id` is omitted, preserve the existing after-block branch, and add targeted Feishu doc tests plus skill/docs updates.
