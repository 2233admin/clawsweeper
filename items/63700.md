## Summary

Keep this issue open. Current main still exposes `openclaw infer model run` as a text-only model-run wrapper: the command metadata and registered options have no repeatable `--file`, and execution forwards only the prompt string to local and gateway agent paths. Separate image/audio/video infer commands overlap with parts of the workflow, but they do not provide the requested custom prompt plus multiple file attachments on `model run`.

## Best Possible Solution

Keep this issue open and implement the missing core CLI capability. Add repeatable `--file` handling to `src/cli/capability-cli.ts`, MIME-detect and validate supported media, forward image files to local `agentCommand` through `images`/`imageOrder`, forward gateway runs through `attachments`, decide whether and how audio should be represented, then update `docs/cli/infer.md` and focused CLI tests for single image, multiple images, unsupported media, and local/gateway forwarding.
