## Summary

Keep open. Current `main` still snapshots workspace bootstrap files per `sessionKey`, so live sessions can keep stale `AGENTS.md`/`USER.md`/`MEMORY.md` content until an explicit reset or session rollover clears that snapshot.

## Evidence

- **Per-session bootstrap snapshot is still on main:** `getOrLoadBootstrapFiles` stores bootstrap files in a `Map<string, WorkspaceBootstrapFile[]>` keyed by `sessionKey` and returns the cached entry on later calls instead of re-reading disk.
  - file: [src/agents/bootstrap-cache.ts:3](https://github.com/openclaw/openclaw/blob/a8edf29bd0f1bd3c869e5e3fab91043d01e6ac33/src/agents/bootstrap-cache.ts#L3)
  - command: `nl -ba src/agents/bootstrap-cache.ts | sed -n '1,36p'`
  - sha: [a8edf29bd0f1](https://github.com/openclaw/openclaw/commit/a8edf29bd0f1bd3c869e5e3fab91043d01e6ac33)
- **Live runs go through that cache:** `resolveBootstrapFilesForRun` calls `getOrLoadBootstrapFiles(...)` whenever `sessionKey` is present, which is the normal live-session path.
  - file: [src/agents/bootstrap-files.ts:229](https://github.com/openclaw/openclaw/blob/a8edf29bd0f1bd3c869e5e3fab91043d01e6ac33/src/agents/bootstrap-files.ts#L229)
  - command: `nl -ba src/agents/bootstrap-files.ts | sed -n '220,253p'`
