## Summary

Keep open. Current main has partial error-surfacing for provider failures, but the reported Gemini-over-HTTP-proxy path is not implemented well enough to close: `buildGuardedModelFetch` still calls the SSRF-guarded fetch without selecting the trusted env-proxy mode, so the guarded model request path still performs local DNS/SSRF checks before the proxy can resolve the target. That leaves the Fake-IP/proxy scenario described in #70453 plausibly unresolved.

## Best Possible Solution

Keep #70453 open. The maintainer path is to decide the intended contract, then either make model-provider requests automatically use `TRUSTED_ENV_PROXY` when process proxy env vars apply, or document/configure an explicit `models.providers.google.request.proxy` path that actually bypasses local target DNS for trusted proxy use. Add regression coverage for Google Gemini with env proxy, Fake-IP/private DNS behavior, and CLI/WebUI-visible provider failure events.
