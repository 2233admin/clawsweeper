## Summary

Keep #38775 open. Current main still does not implement usage-aware auth-profile routing: Anthropic usage snapshots exist for status/usage reporting, but Gateway profile selection remains explicit-order, credential-type/lastUsed, and cooldown based, with no auth.balancing config or async per-profile usage-routing API.

## Best Possible Solution

Keep #38775 open for maintainer/product design. A solid implementation would add first-class usage-aware auth-profile routing across Gateway request paths: provider-owned Anthropic usage parsing mapped to per-profile headroom, cached or polled utilization state, proactive 5h/7d circuit thresholds, shadow accounting for concurrent starts, fallback to current reactive cooldown rotation for profiles without usage data, auth.balancing schema/help/docs, and regression tests for utilization-based profile choice plus cooldown interaction. If maintainers prefer plugin ownership, first add a documented async pre-request profile-selection API with usage-cache integration.
