## Summary

Keep open. Current main shows the Control UI and gateway preserve chat input essentially verbatim, and the Control UI tool-card renderer also preserves tool arguments verbatim for display. That means the report still points at a plausible unresolved runtime/model-side bug rather than an already-fixed UI transport issue. I did not find a main-branch fix tied directly to issue #56534, and I could not do a high-confidence live repro in this read-only review environment because `pnpm` is unavailable here.
