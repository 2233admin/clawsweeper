## Summary

Current main already has reset-time lifecycle hooks for `/new` and `/reset`, including a typed `before_reset` plugin hook plus the bundled `session-memory` hook that silently persists prior-session context before reset. The request is implemented through the generic hooks system rather than a new `session.onReset` config key.

## Evidence

- **Reset commands emit pre-reset hooks:** `emitResetCommandHooks()` creates a `command:new`/`command:reset` internal hook event, awaits `triggerInternalHook()`, then also dispatches the typed `before_reset` plugin hook with the prior session transcript/session file.
  - file: [src/auto-reply/reply/commands-reset-hooks.ts:95](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/auto-reply/reply/commands-reset-hooks.ts#L95)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Typed before_reset hook is public in the plugin hook contract:** `before_reset` is a first-class plugin hook name, with `PluginHookBeforeResetEvent` carrying `sessionFile`, `messages`, and `reason`, and a typed hook signature in the registry contract.
  - file: [src/plugins/hook-types.ts:67](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/plugins/hook-types.ts#L67)

## Risks / Open Questions

- The implementation is hook/plugin-based, not literally a hidden extra model turn with tool calls. If maintainers consider that distinction product-significant, reopening as a narrower UX/API request could still be reasonable.

## Close Comment

Closing as implemented on main after Codex review. OpenClaw already runs reset-time hooks for `/new` and `/reset` in `src/auto-reply/reply/commands-reset-hooks.ts:95`, exposes a typed `before_reset` plugin hook in `src/plugins/hook-types.ts:229`, and ships the bundled `session-memory` hook documented in `docs/automation/hooks.md:34` and implemented in `src/hooks/bundled/session-memory/handler.ts:53` to persist prior-session context before reset. `CHANGELOG.md:931` also records the shipped gateway `/new`/`/reset` `before_reset` hook, and the current release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` contains these paths. This landed through the generic hooks system rather than a new `session.onReset` config key, but it covers the requested pre-reset housekeeping/persistence behavior. If there is still a missing gap specifically around "run one hidden model turn with tool access" beyond the existing hook/plugin path, please open a fresh issue describing that narrower difference.
