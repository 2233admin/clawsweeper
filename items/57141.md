## Summary

Keep open. Current `main` still has a plausible unresolved path for this bug: Telegram DM-topic bindings are resolved against the bound ACP session, but the inbound Telegram path still materializes a `:thread:<chat>:<topic>` session row, and ACP dispatch still prefers `ctx.SessionKey` for store lookup on non-native messages. That means the metadata-only thread row can still be selected and hit the same `ACP metadata is missing` failure instead of the bound base ACP session.
