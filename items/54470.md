## Summary

Current main still has the reported Windows command-resolution mismatch for Gmail webhook setup. Dependency checks can succeed for PATH/PATHEXT shims, but the Gmail setup and watcher paths still execute bare `gcloud`, `gog`, and `tailscale` without resolving Windows `.cmd` shims or using the repo's newer Windows spawn resolver.

## Best Possible Solution

Keep this issue open. Fix Gmail webhook setup, run, and gateway watcher execution by resolving `gcloud`, `gog`, and `tailscale` through a Windows-aware executable/shim resolver before spawning, while preserving the existing no-shell hardening. Add win32 regression tests proving PATH-only `.cmd` shims work for the Gmail setup and watcher paths.
