## Summary

Keep this PR open. Current main still builds the status session-default summary from one global configured model, while the supported per-agent `agents.list[].model` override path can make effective defaults differ by agent. The PR remains a plausible narrow fix, but it needs rebasing/adaptation to current status runtime and overview rendering seams.

## Best Possible Solution

Keep this PR open or replace it with a narrow maintainer patch. Compute `sessions.defaults` from each listed agent's effective status-safe default model, keep a concrete model only when all agent defaults match, expose an optional variation marker through `StatusSummary` and `GatewayStatusSummary`, render `varies by agent` in CLI and TUI status surfaces, and add divergent/uniform per-agent regression tests.
