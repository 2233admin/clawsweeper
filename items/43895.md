## Summary

Keep open. Current `main` still computes `status` session defaults from one global configured model and renders that single value in both the CLI overview and TUI summary, so the PR's reported multi-agent/default-model mismatch does not appear implemented elsewhere on `main`. The discussion also points to a scoped display-only fix with unrelated CI noise, not an obsolete report.
