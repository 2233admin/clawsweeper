## Summary

Keep open. Current main still resolves `~/...` in `normalizeExtraMemoryPaths()` as workspace-relative, so the reported memory extraPaths bug remains reproducible from source inspection and a direct Node path check. Related PR #58050 was closed unmerged, and the current tests cover trimming, relative paths, absolutes, and dedupe but not tilde expansion.

## Best Possible Solution

Keep the issue open and land a focused fix that expands `~` before `path.isAbsolute()` in `normalizeExtraMemoryPaths()` for the package and mirrored host implementation, reusing `expandHomePrefix()` or the existing user-path resolver. Add regression tests for `~/...` in the memory extraPaths normalizer/listing path, and keep existing workspace-relative and absolute-path behavior unchanged.
