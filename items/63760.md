## Summary

Keep open. Current main still does not provide announce-specific model selection or light/no-history context for subagent and ACP completion handoff; existing controls only cover spawned child runs and announce timeout.

## Best Possible Solution

Keep this open and implement a schema-backed announce policy, likely under agents.defaults.subagents.announce with per-agent mirroring where subagent defaults already exist. Wire it into direct, queued, and descendant-wake announce agent calls so completion handoff can use an allowed cheaper model and light/no-history context without changing spawned child model behavior. Add focused tests for native subagents, ACP completions, nested requester sessions, queue fallback, and unauthorized model overrides, then document the knobs on the public Sub-agents and ACP agents pages.
