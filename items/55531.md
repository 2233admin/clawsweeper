## Summary

Keep open. Current `main` still hands Feishu WebSocket reconnect behavior to the pinned upstream Lark SDK, and neither the local plugin code nor its tests/changelog show an OpenClaw-side fix for exponential backoff or the reported `PingInterval` crash path.

## Evidence

- **OpenClaw Feishu plugin creates the WS client without any reconnect/backoff policy overrides:** `createFeishuWSClient()` constructs `new feishuClientSdk.WSClient(...)` with only `appId`, `appSecret`, `domain`, `loggerLevel`, and optional `agent`. There is no OpenClaw-level reconnect interval/backoff configuration or wrapper here.
  - file: [extensions/feishu/src/client.ts:230](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/client.ts#L230)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **OpenClaw transport layer only starts the SDK client and cleans it up on abort:** `monitorWebSocket()` creates the SDK client, stores it, calls `wsClient.start({ eventDispatcher })`, and only handles abort-time cleanup. It does not implement retry classification, exponential backoff, or a health-check before reconnect.
  - file: [extensions/feishu/src/monitor.transport.ts:85](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/monitor.transport.ts#L85)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Pinned dependency is still `@larksuiteoapi/node-sdk` `^1.61.1`:** The bundled Feishu plugin on current `main` still depends on the same upstream SDK family, so reconnect behavior is still delegated there unless OpenClaw wraps it locally.
  - file: [extensions/feishu/package.json:7](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/package.json#L7)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Upstream SDK uses a fixed reconnect interval and unguarded `PingInterval` parsing:** Inspection of `@larksuiteoapi/node-sdk@1.61.1` shows default WS settings `reconnectInterval: 120 * 1000` and `reconnectNonce: 30 * 1000`, then a reconnect loop that reuses the same `reconnectInterval` for every retry rather than exponential backoff. The same file also parses `const { PingInterval, ReconnectCount, ReconnectInterval, ReconnectNonce } = JSON.parse(dataString)` and immediately multiplies those fields, which matches the general shape of the reported `PingInterval` undefined crash risk.
  - file: [npm:@larksuiteoapi/node-sdk@1.61.1/package/lib/index.js:85161](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/npm%3A%40larksuiteoapi/node-sdk%401.61.1/package/lib/index.js#L85161)
  - command: `curl -L https://registry.npmjs.org/@larksuiteoapi/node-sdk/-/node-sdk-1.61.1.tgz | tar -xzOf - package/lib/index.js | nl -ba | sed -n '85156,85545p'`
- **Current OpenClaw tests/changelog cover cleanup/proxy wiring, not reconnect backoff:** The current Feishu WS tests shown in `client.test.ts` only verify proxy-agent wiring for `createFeishuWSClient()`. The shipped changelog entry for Feishu WebSocket on this branch is about closing connections on stop/abort, not reconnect throttling or `PingInterval` handling.
  - file: [extensions/feishu/src/client.test.ts:352](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/client.test.ts#L352)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Latest shipped Feishu WebSocket note is cleanup-only:** `CHANGELOG.md` on current `main`/latest release includes `#52844` for closing Feishu WebSocket connections on monitor stop/abort. I found no shipped changelog entry for exponential reconnect backoff or a `PingInterval` guard.
  - file: [CHANGELOG.md:2355](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/CHANGELOG.md#L2355)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
