## Summary

Keep #71581 open. Current main still lacks dmScope-aware session resolution for forced exec-event and cron-event heartbeat wakes: exec and cron queue/wake on the resolved literal session key, heartbeat drains that resolved queue, and the only single-owner dmScope helper remains in inbound channel routing. The unrelated #72160 cross-reference does not supersede this issue.

## Best Possible Solution

Implement the requested routing in core, either by re-keying exec/cron system events to the effective main session when session.dmScope=main plus the existing single-owner allowlist precondition holds, or by teaching heartbeat forced-session resolution to redirect same-agent per-channel DM keys to agent:<id>:main while preserving deliveryContext for outbound replies. Add focused regression coverage for exec notifyOnExit and cron requested-session wakes, including negative cases for multi-owner allowlists and per-channel/per-peer dmScope.

## Evidence

- **Forced heartbeat sessions ignore dmScope:** resolveHeartbeatSession canonicalizes a forced session key and returns it when it belongs to the same agent. It does not inspect cfg.session.dmScope or the single-owner allowlist, so a per-channel DM key remains the heartbeat run session instead of being redirected to agent:<id>:main.
  - file: [src/infra/heartbeat-runner.ts:246](https://github.com/openclaw/openclaw/blob/631552c554095b0c520376db9131dac0f5427f94/src/infra/heartbeat-runner.ts#L246)
  - sha: [631552c55409](https://github.com/openclaw/openclaw/commit/631552c554095b0c520376db9131dac0f5427f94)
- **Heartbeat preflight reads the resolved queue:** resolveHeartbeatPreflight calls resolveHeartbeatSession and then peeks system events for session.sessionKey. If the forced key resolves to a per-channel session, the main session queue is not inspected.
  - file: [src/infra/heartbeat-runner.ts:551](https://github.com/openclaw/openclaw/blob/631552c554095b0c520376db9131dac0f5427f94/src/infra/heartbeat-runner.ts#L551)
  - sha: [631552c55409](https://github.com/openclaw/openclaw/commit/631552c554095b0c520376db9131dac0f5427f94)
- **Exec notify-on-exit uses the original session key:** maybeNotifyOnExit trims session.sessionKey, enqueues the exec completion with that sessionKey, and requests an exec-event heartbeat through scopedHeartbeatWakeOptions(sessionKey). There is no dmScope rewrite before enqueue or wake.
  - file: [src/agents/bash-tools.exec-runtime.ts:314](https://github.com/openclaw/openclaw/blob/631552c554095b0c520376db9131dac0f5427f94/src/agents/bash-tools.exec-runtime.ts#L314)

- **System-event queues are exact-session keyed:** enqueueSystemEvent stores events under options.sessionKey and drain/peek APIs use the same key, so queue semantics currently require any dmScope redirect to happen explicitly before enqueue or before heartbeat resolves the queue.
  - file: [src/infra/system-events.ts:90](https://github.com/openclaw/openclaw/blob/631552c554095b0c520376db9131dac0f5427f94/src/infra/system-events.ts#L90)
