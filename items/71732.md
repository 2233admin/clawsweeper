## Summary

Keep open. Current main still lacks the requested plugin session-extension host seam: `sessions.patch` is a closed, core-field patch path; plugins can register separate gateway RPCs and observe lifecycle hooks, but there is no typed namespaced session state registration, host-routed plugin patch action, public projection into session rows, or host-owned reset/delete/compaction cleanup. The issue is an in-scope plugin API gap rather than ClawHub-only work, and the linked RFC PR #71731 is explicitly design/docs-only, not an implementation.
