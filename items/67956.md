## Summary

Keep open. Current `main` has partial groundwork for CLI streaming, but not the end-to-end channel preview behavior requested here. The CLI runner can parse JSONL/live-session deltas and emit `assistant` events, yet the CLI reply path still collapses output to a final payload and the auto-reply layer only emits a final assistant event for server-chat/TUI/WebSocket clients rather than wiring incremental CLI output into channel preview updates.
