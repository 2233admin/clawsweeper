## Summary

Keep open. Current `main` does not implement Azure Speech TTS, and the PR is not a clean close candidate because it targets an outdated core TTS layout instead of the current speech-provider plugin architecture.

## Evidence

- **Azure is not a shipped TTS provider on main:** The user-facing TTS docs list only ElevenLabs, Google Gemini, Microsoft, MiniMax, OpenAI, and xAI as supported services; Azure Speech is not listed.
  - file: [docs/tools/tts.md:10](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/docs/tools/tts.md#L10)

- **Speech providers are resolved from plugins, not `src/tts/providers/*` builtins:** `src/tts/provider-registry.ts` only loads speech providers through `resolvePluginCapabilityProviders(...)`. There is no builtin registry hookup for an Azure provider living under `src/tts/providers/*` on current main.
  - file: [src/tts/provider-registry.ts:16](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/tts/provider-registry.ts#L16)

  - file: [docs/plugins/architecture-internals.md:379](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/docs/plugins/architecture-internals.md#L379)
