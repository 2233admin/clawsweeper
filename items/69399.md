## Summary

Keep PR #69399 open. Current main still only shows model fallback and fallback-cleared notices in verbose mode, does not expose `agents.defaults.notifyOnModelFallback` in config types/schema/docs, and the existing tests explicitly assert that verbose-off replies suppress those notices. The PR remains a focused implementation for the still-open linked feature request #67197 rather than an implemented, duplicate, ClawHub, or out-of-repo cleanup candidate.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a core-level opt-in such as `agents.defaults.notifyOnModelFallback` if maintainers want this UX, including config type/schema/help/docs updates and regression tests for transition, once-per-state, and fallback-cleared behavior outside verbose mode.
