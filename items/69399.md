## Summary

Keep PR #69399 open. Current main still only shows reply-level model fallback and fallback-cleared notices when verbose mode is enabled, and the config/docs/schema surface has no `agents.defaults.notifyOnModelFallback` or equivalent opt-in. The PR directly addresses the still-open feature request #67197 and remains useful implementation work, not conservative cleanup.

## Best Possible Solution

Keep the PR open for maintainer review as the likely canonical implementation for #67197. The best path is to review the config name and UX, verify the PR branch’s generated config artifacts and e2e coverage, then merge this focused opt-in or request small revisions rather than closing it.
