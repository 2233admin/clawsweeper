## Summary

Keep #71828 open. Current main at 6d60b035 still lacks the PR's narrow TTL refresh fix: the keepalive interval calls `triggerTyping`, but `triggerTyping` only sends the channel typing signal and does not refresh the controller TTL. The PR's one-line change plus regression test directly cover that remaining gap, and the Greptile review comment is positive supporting evidence rather than a close reason.

## Best Possible Solution

Keep #71828 open for maintainer review. The best path is to land this narrow controller fix, or an equivalent current-main change, so each active keepalive tick refreshes the typing TTL, with the proposed beyond-TTL regression test preserved or adapted. If maintainers prefer the broader long-quiet lease design from the related work, this PR should be explicitly superseded by that canonical implementation only once it is open or merged.
