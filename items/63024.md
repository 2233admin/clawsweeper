## Summary

Keep this open. Current main still has the same fragile startup shape: classic CLI startup imports the logging barrel before command dispatch, the logger still imports bare `tslog`, and the latest inspected npm package still ships that bare logger import. The documented Linux quick path still recommends Node 24 plus `openclaw onboard --install-daemon`. Related PR #72493 is open and unmerged, so it is only a candidate fix, not shipped proof for this issue.

## Best Possible Solution

Keep this issue open. The best maintainer path is to run a packaged pnpm-global install smoke on Node 24 for `openclaw onboard --install-daemon`, then fix the CLI startup/logging packaging path so the classic onboarding command cannot fail on `tslog` resolution before command dispatch. If #72493 or another PR lands and proves this exact path, close this afterward as implemented.
