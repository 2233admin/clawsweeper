## Summary

Keep open. Current `main` does not implement the macOS `launchctl getenv` proxy fallback, so the reported gap is still plausible. The PR moves in the right direction, but the posted review feedback identifies a real precedence bug: lowercase shell proxy vars can still be overridden by uppercase `launchctl` fallbacks.

## Risks / Open Questions

- The underlying macOS LaunchAgent proxy gap appears to remain on `main`, so closing this PR would leave a plausible user-facing daemon/networking bug unresolved.
- Merging the PR as-is risks conflicting proxy values in service mode when users set lowercase shell proxy vars and `launchctl` exposes different uppercase values.
