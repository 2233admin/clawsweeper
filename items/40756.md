## Summary

Keep this PR open. Current `main` still appends the workspace-path hint to any resolved heartbeat prompt that mentions `HEARTBEAT.md`, including supported user-configured prompts, while the linked regression remains open and this PR is still a focused implementation candidate.

## Best Possible Solution

Keep this PR or the linked regression open. The best path is a small heartbeat-runner fix that distinguishes non-empty user-configured heartbeat prompts from built-in/generated heartbeat prompts, preserves the workspace-path hint and idempotency for default/internal prompts, adds a regression test for custom prompts mentioning `HEARTBEAT.md`, and explicitly resolves the prompt-safety tradeoff before merge.

## Risks / Open Questions

- The PR intentionally stops adding the workspace-file disambiguation hint to user-configured prompts. That matches the documented verbatim-prompt contract, but maintainers should decide whether heartbeat file selection needs a separate non-overridable or tool-layer safety guard.
- The PR exports `appendHeartbeatWorkspacePathHint` for direct testing. That is likely acceptable for an internal runner module, but an integration test through `runHeartbeatOnce()` would avoid widening the module surface.
- The branch is stale relative to current `main` and should be rebased or refreshed before landing.
