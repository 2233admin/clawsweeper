## Summary

Keep open. The request is coherent, recent, and not implemented on current `main`: OpenClaw has some surface-specific freshness signals, but there is no standardized tool-response freshness envelope across shared tool result helpers, `session_status`, and `memory_search`.

## Evidence

- **Shared tool result helper has no standard freshness envelope:** The common JSON tool-result helper only serializes the payload into text plus `details`; it does not attach a shared `freshness` object or metadata contract for agent tools.
  - file: [src/agents/tools/common.ts:285](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/agents/tools/common.ts#L285)
  - command: `nl -ba src/agents/tools/common.ts | sed -n '270,300p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **`session_status` still returns ad hoc status details:** Current `session_status` returns text content plus `{ ok, sessionKey, changedModel, statusText }`. The docs say it can backfill sparse token/cache counters, but the tool response does not expose standardized freshness metadata.
  - file: [src/agents/tools/session-status-tool.ts:603](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/agents/tools/session-status-tool.ts#L603)
  - command: `nl -ba src/agents/tools/session-status-tool.ts | sed -n '555,620p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **`memory_search` returns tool-specific fields, not a shared freshness block:** `memory_search` currently returns `results`, `provider`, `model`, `fallback`, `citations`, `mode`, and `debug`. There is no `freshness` envelope like the issue proposes.
  - file: [extensions/memory-core/src/tools.ts:306](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/memory-core/src/tools.ts#L306)
  - command: `nl -ba extensions/memory-core/src/tools.ts | sed -n '290,335p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Freshness exists today only on specific non-tool surfaces:** The sessions command exports `totalTokensFresh` in its JSON output, which shows OpenClaw already has ad hoc freshness indicators on some surfaces, but not a standardized cross-tool response contract.
  - file: [src/commands/sessions.ts:186](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/commands/sessions.ts#L186)
