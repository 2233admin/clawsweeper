## Summary

Keep open. Current main supports `promptMode` internally and on the low-level `agent` gateway method, but the public `sessions_spawn` tool does not expose, read, type, document, or forward a `promptMode` parameter. The request is actionable core/API work, not a ClawHub-only item or a duplicate of the related sessions_spawn parameter reports.

## Best Possible Solution

Implement a narrow core/API change: add optional `promptMode: "full" | "minimal" | "none"` to the `sessions_spawn` tool schema and `SpawnSubagentParams`, forward it into the native child `agent` gateway call, keep the default omitted behavior as today, and add docs plus regression tests for `promptMode: "none"`. Consider a follow-up for plugin `api.runtime.subagent.run` if plugin-owned deterministic blank workers are also intended.
