## Summary

Keep open. Current main does not implement the PR's requested Control UI behavior: protected same-origin canvas/a2ui iframe URLs still fall back to raw `/__openclaw__/canvas/...` paths when no scoped `canvasHostUrl` is present, and the browser gateway client has no Control UI capability-refresh path. The PR is also not safe to close as implemented because its timer-based websocket reconnect approach matches the bot review concern: reconnect/close rejects pending RPCs and the app reconnect path clears in-flight chat state.

## Best Possible Solution

Keep the PR open but request changes: preserve the fail-closed iframe URL behavior, add positive a2ui scoped-host coverage, and replace timer-forced websocket reconnect with a non-disruptive Control UI-safe capability refresh path, likely by exposing/using a scoped canvas capability refresh RPC for the browser operator session or refreshing only when idle without tearing down the transport.
