## Summary

Keep #69372 open. Current main does not implement the PR's Control UI fail-closed iframe behavior or a Control UI-side canvas capability refresh, and the PR discussion includes a substantive P1 concern that the proposed timer-based websocket reconnect can interrupt active runs. This is a plausible focused bug fix, but it needs a safer refresh design before maintainer cleanup should close or merge it.

## Best Possible Solution

Keep the PR open and ask for a safer revision: preserve the fail-closed canvas/a2ui URL handling with regression tests, but replace timer-forced websocket teardown with a refresh path that does not reject active RPCs or clear chat run state, such as an operator-safe capability refresh API or an idle-gated refresh. Also address the review's duplicated scoped-host validation by sharing the capability-host parser/predicate.
