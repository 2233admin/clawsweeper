## Risks / Open Questions

- The proposed polling loop masks every gh api failure as 0 assets, so auth, rate-limit, or GitHub outage errors would be misreported as missing assets after a long timeout.
- The loop sleeps after the last check and then fails without a final asset recheck, so assets that appear during the final minute can still produce a false failure.
- The proposed stable tag validation is prefix-only rather than fully anchored to the repo's stricter release tag format.
- The PR makes appcast absence a warning even though release docs describe appcast advancement as part of stable macOS readiness; that is a release-owner policy decision.
- The CODEOWNERS change names individual owners for appcast.xml rather than the existing openclaw-release-managers team, so ownership should be confirmed by maintainers.
