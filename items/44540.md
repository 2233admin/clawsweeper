## Summary

Keep open. Current main still uses a weighted hybrid merge that gives vector-only results a zero textScore, so multimodal image/audio chunks with no BM25 hit are still multiplied by the default 0.7 vector weight. The current schema/search path also does not carry modality into the merge layer, so the suggested non-text bypass is not present.

## Evidence

- **Weighted hybrid merge still penalizes vector-only chunks:** Vector results enter the merge with textScore: 0, and final score is computed as vectorWeight * vectorScore + textWeight * textScore. A multimodal result that misses BM25 therefore remains capped by the vector weight rather than using vector score only.

- **Default hybrid weights match the reported penalty:** The default hybrid configuration remains enabled with vectorWeight 0.7 and textWeight 0.3, matching the issue's described default behavior.
