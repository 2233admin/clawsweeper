## Summary

Keep open. Current main has clearer ACP/acpx and Discord binding docs, and the prior ClawSweeper comment already reached the same conclusion, but the reported topology still is not implemented: `/acp spawn` and `sessions_spawn({ runtime: "acp" })` can select an ACP backend/agent/cwd, not a remote node host such as a Mac mini separate from the Gateway host.

## Best Possible Solution

Keep this open for maintainer product/docs judgment. The short-term fix should explicitly document that bundled ACP/acpx harnesses execute on the Gateway host today and recommend running the Gateway/acpx on the Mac or using an operator-managed wrapper for multi-machine setups. If OpenClaw should support the reported topology, add an explicit ACP runtime node selector and node-host process execution contract rather than overloading `cwd`, backend ids, or Discord thread bindings.
