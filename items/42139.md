## Summary

Keep open. Current `main` documents ACP much more clearly for Discord bindings, but the code/docs still point to ACP harnesses running on the gateway host via the embedded `acpx` plugin, not on a paired remote Mac node. I found no node-target setting in ACP runtime/binding config, and the issue's reported `acpx exited with code 1` failure is not disproven by current `main` without a live repro.

## Evidence

- **ACP is host-side, not node-routed:** The ACP docs explicitly say ACP sessions run on the host runtime, outside the OpenClaw sandbox.
  - file: [docs/tools/acp-agents.md:404](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/tools/acp-agents.md#L404)

  - file: [extensions/acpx/src/service.ts:95](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/extensions/acpx/src/service.ts#L95)
