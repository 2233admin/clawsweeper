## Summary

Keep this PR open. Current main still uses the Mattermost bot token to derive the interaction HMAC secret with the public `openclaw-mattermost-interactions` key, so the linked forgery bug remains present. This PR is the active implementation candidate for open bug #64545 by the same contributor, but its random in-memory fallback still needs revision so button signing and gateway verification remain stable across processes and restarts.

## Best Possible Solution

Keep this PR and #64545 open. Revise the Mattermost fix so interaction signing uses a persistent per-account secret shared by send and gateway monitor paths, supports SecretRef/config or credential-store persistence, handles existing deployments intentionally, updates schema/types/docs/secret metadata, and adds regression coverage for both bot-token forgery resistance and CLI/gateway cross-process verification before merge.

## Risks / Open Questions

- The proposed `interactions.secret` surface needs matching docs, credential/SecretRef metadata, and migration or operational guidance before it is a complete product fix.
