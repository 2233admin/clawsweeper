## Summary

Keep #69102 open. Current main has generic model `params`, but the bundled Google/Gemini provider does not map `serviceTier` or `service_tier` into Gemini request bodies. Official Google Flex docs require `service_tier: "flex"` in REST requests or `config.serviceTier: "flex"` in the JS SDK, while OpenClaw's Google docs and tests only cover direct Gemini `cachedContent` passthrough for this provider. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/flex-inference))

## Best Possible Solution

Keep the issue open and implement this in the bundled Google provider: accept `agents.defaults.models["google/<model>"].params.serviceTier` and `service_tier`, map valid Flex values into the direct Gemini request body as top-level `service_tier: "flex"`, add transport tests for camelCase and snake_case params, and document it on the Google provider page with any timeout guidance.
