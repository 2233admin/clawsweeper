## Summary

Keep #71737 open. Current main only has adjacent raw/runtime surfaces: raw `api.runtime.events.onAgentEvent`, an internal host `AgentRunContext`, generic prompt hooks, and low-level `gateway_start` cron access. It does not expose the requested typed plugin-owned event subscriptions, namespaced run-context storage, session-owned scheduler cleanup, or heartbeat-specific contribution hook. The related #72082 PR appears to be the right implementation vehicle, but it is still open and not present on current main SHA `631552c554095b0c520376db9131dac0f5427f94`.

## Best Possible Solution

Keep #71737 open as the live RFC/acceptance-criteria issue until the generic host-hook implementation lands. The likely product/code path is to review and, if accepted, merge #72082 or an equivalent focused PR that adds sanitized agent event subscriptions, plugin-owned run context, session-scoped scheduler ownership/cleanup, and heartbeat-only prompt contributions with contract fixture coverage. After that lands on main, close #71737 as implemented or as superseded by the merged implementation PR.
