## Summary

Current main still has the reported peer `sessions_send` A2A path: fire-and-forget sends still start announce delivery, peer replies are forwarded unless blank or exactly `REPLY_SKIP`, `NO_REPLY` is not treated as a silent/control reply in the A2A loop, and the announce step still submits the literal placeholder as an inter-session user input. The Hollychou924 analysis and the prior ClawSweeper comment remain consistent with current source. Related #44213 is cron-specific and already closed; #61433 is broader raw metadata hardening and does not supersede this specific `sessions_send` issue.

## Best Possible Solution

Keep this issue open. The fix should land in `src/agents/tools/sessions-send-tool.a2a.ts` and adjacent helpers: apply the shared silent/control-reply policy before ping-pong or announce delivery, treat exact `NO_REPLY` as silent alongside A2A skip tokens, avoid submitting or persisting the announce placeholder as a normal inter-session user turn, prevent acknowledgement/follow-up text from being reinjected as peer input, and add focused regression coverage for peer `sessions_send({ timeoutSeconds: 0 })` covering placeholder suppression, `NO_REPLY` suppression, and no echo loop.

## Risks / Open Questions

- The A2A path is documented as a peer-session fallback and is only skipped for parent-owned ACP children, so broadly disabling it would be too blunt.
