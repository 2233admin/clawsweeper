## Summary

Keep open. Current main does have a supported Z.AI image-understanding path via `glm-4.6v`, but the exact report path is not clearly resolved: bundled ZAI metadata still marks `glm-5` as text-only, and the model-merge path preserves bundled `input` over explicit config overrides. That leaves the reported `zai/glm-5` + `input: ["text","image"]` configuration plausibly still able to fail with `Model does not support images`.
