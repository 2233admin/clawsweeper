## Summary

Keep open. Current main has partial mitigation for session-store growth, but it still does not have a general cleanup path for subagent session entries left in `sessions.json` as `status: "running"` after an abnormal exit. Terminal status only gets written once a subagent run reaches a terminal outcome, default subagent waits are effectively no-timeout, the sweeper only runs after cleanup has already completed, and restart recovery intentionally excludes ordinary subagent sessions unless `abortedLastRun` was set through a narrower path.

## Evidence

- **Terminal session status is only persisted from terminal subagent run state:** `persistSubagentSessionTiming()` updates `startedAt`, `endedAt`, `runtimeMs`, and `status` in `sessions.json`, but only from the in-memory/persisted subagent run record after terminal timing/outcome exists.
  - file: [src/agents/subagent-registry-helpers.ts:90](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/agents/subagent-registry-helpers.ts#L90)
  - command: `nl -ba src/agents/subagent-registry-helpers.ts | sed -n '90,138p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Restored subagent runs still wait with the default no-timeout sentinel:** `restoreSubagentRunsOnce()` resumes persisted runs, and `resolveSubagentWaitTimeoutMs()` passes `runTimeoutSeconds ?? 0` into `resolveAgentTimeoutMs()`. In `timeout.ts`, `0` maps to `2_147_000_000ms`, so a crashed run can sit for about 24.8 days before the wait path times out and writes a terminal status.
  - file: [src/agents/subagent-registry.ts:504](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/agents/subagent-registry.ts#L504)

  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **The gateway wait helper only resolves on lifecycle events or timeout:** `waitForAgentJob()` waits for lifecycle `end`/`error` or for its timer to expire; if a crashed subagent never emits a terminal lifecycle event, there is no separate crash detector here.
  - file: [src/gateway/server-methods/agent-job.ts:144](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/gateway/server-methods/agent-job.ts#L144)

  - file: [src/agents/subagent-registry.ts:581](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/agents/subagent-registry.ts#L581)

  - file: [src/agents/subagent-orphan-recovery.ts:136](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/agents/subagent-orphan-recovery.ts#L136)
