## Summary

Keep this issue open. Current main still loads and watches the personal `~/.agents/skills` source without a source-level disable config; existing allowlists, per-skill disables, and prompt-budget limits are post-discovery mitigations and do not satisfy the requested ability to skip scanning `agents-skills-personal`.

## Best Possible Solution

Keep this issue open. Implement a narrow core skills-loader config surface that can disable `agents-skills-personal` before both loading and watcher registration, likely as a typed source list or explicit `personal` scan boolean. Update config types, strict Zod schema, generated schema/help metadata, docs at `docs/tools/skills-config.md` and `docs/tools/skills.md`, and regression tests proving the personal source is skipped while existing precedence, per-agent allowlists, prompt limits, bundled allowlist behavior, and per-skill `enabled: false` behavior continue to work.
