## Summary

Keep PR #72218 open. Current main still has the exact heartbeat bypass the PR removes: `buildReplyPayloads` returns raw heartbeat payloads when `params.isHeartbeat` is true, so the shared `HEARTBEAT_OK` stripping and OK-only suppression branch is not applied at this final-payload seam. The PR is a focused bug fix with targeted tests, and the related broader exec/system-event items do not supersede this narrow gap.

## Best Possible Solution

Keep #72218 open for maintainer review or land an equivalent narrow change in `src/auto-reply/reply/agent-runner-payloads.ts`: apply `HEARTBEAT_OK` stripping to heartbeat final payloads, preserve media-bearing payloads after stripping, keep Bun socket error formatting scoped to non-heartbeat runs, and add focused `isHeartbeat: true` regression coverage in `src/auto-reply/reply/agent-runner-payloads.test.ts`. Treat #72253 and #69492 as adjacent broader exec/system-event leakage work, not superseding fixes for this PR.
