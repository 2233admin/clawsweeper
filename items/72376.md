## Summary

Keep #72376 open. Current main still opens the memory-core SQLite index with only busy_timeout, so the WAL/NORMAL behavior requested by the PR is not implemented. The older related WAL PR #51986 is not a strong superseding vehicle because its patch targets the pre-plugin `src/memory/manager-sync-ops.ts` path, while current main uses `extensions/memory-core/src/memory/manager-db.ts` as the shared opener. #72376 is a plausible current-path fix and should get maintainer review rather than cleanup closure.

## Best Possible Solution

Keep #72376 open for maintainer review. The best path is to land a current-layout memory-core change in `extensions/memory-core/src/memory/manager-db.ts` that retains `busy_timeout`, sets WAL/NORMAL deliberately, adds focused regression coverage for `PRAGMA journal_mode` and `busy_timeout` through the current helper/reindex path, and then close/supersede the older stale WAL PRs once the chosen implementation lands.
