## Summary

Keep open. Current `main` already supports OS-filtered skill installers, but the affected skill manifests in `main` still do not carry the needed `os` metadata, so the reported Windows onboarding noise is not implemented away. The PR also includes a useful `xurl` metadata correction (`npm` -> `node`) that matches the parser’s accepted install kinds.
