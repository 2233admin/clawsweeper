## Summary

Keep #39961 open. Current main still has tool-call loop detection only; it does not implement a streaming text-repetition guard, a textRepetitionGuard config/schema surface, or user-facing docs for pure text generation loops. The PR is a plausible focused fix with substantial contributor follow-up and remaining integration concerns that need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer engineering review. The best path is to either land a robust streaming text-repetition guard with config/schema/docs/tests that cover abort, queued-event, and flush edge cases, or have a maintainer explicitly decide that OpenClaw core should not add this guard.
