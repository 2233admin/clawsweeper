## Summary

Keep open. Current `main` does not implement the PR’s Telegram inline-query feature set. The repo has only partial inline-related scaffolding (`inline_query`/`chosen_inline_result` in fallback update-type constants and a probe/status flag), but no inline query handlers, no `ChatType` support for `"inline"`, and no sequentialization bypass or pre-handler timestamping described in PR #51963.
