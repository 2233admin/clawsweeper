## Summary

Keep PR #70040 open. Current main and the latest release still lack Chinese/CJK trigger support in `memory-lancedb`, so the requested behavior is not implemented. The PRtags duplicate group is useful context, but the related PRs are closed and unmerged, and current main confirms their changes did not land. The PR itself needs changes before merge because the existing `< 10` character gate still blocks common short Chinese memory statements and category detection remains English/Czech-only.

## Best Possible Solution

Keep the PR open and request a focused update: add CJK-aware `memory-lancedb` capture triggers, adjust the minimum-length logic so short high-signal Chinese memory statements can pass safely, add Chinese `detectCategory` branches for preference/decision/entity/fact where appropriate, and add regression tests for both short and longer Chinese examples. If maintainers prefer not to continue this branch, open or link a narrower canonical PR rather than closing without a shipped replacement.
