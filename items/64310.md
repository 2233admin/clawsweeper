## Summary

Keep this PR open. Current main still has the mutable module-global `workspacePackagePaths` and still feeds accumulated state into `missingPackages`; the PR is a narrow, plausible fix that has not landed or been superseded. It needs normal maintainer review, small cleanup, and regression coverage rather than conservative closure.

## Best Possible Solution

Review and land a focused version of this PR rather than close it. The maintainer path is to make `collectWorkspacePackagePaths()` side-effect-free, pass the fresh deterministic package-path list into `buildMissingPackages()`, address the redundant-sort review note, and add a small regression test in `test/scripts/audit-seams.test.ts` that calls `main()` twice in-process and verifies `missingPackages` does not duplicate entries.
