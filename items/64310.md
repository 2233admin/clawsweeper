## Summary

Current main does not contain the PR’s idempotency fix. `scripts/audit-seams.mjs` still keeps `workspacePackagePaths` as a mutable module-global array, `collectWorkspacePackagePaths()` still appends extension package paths into that shared array, and `main()` still calls it before building `missingPackages`. The PR is a narrow, plausible fix rather than obsolete cleanup, though it should get a small regression test or explicit maintainer acceptance before landing.

## Best Possible Solution

Keep the PR open. The best path is to land the local-state version of `collectWorkspacePackagePaths()` after adding a small idempotency regression test for repeated in-process `main()` calls, or have a maintainer apply the same small fix with test coverage.
