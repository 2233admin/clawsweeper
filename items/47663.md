## Summary

Keep open. Current `main` still appends the unscheduled-reminder guard note when reminder language is detected and no new/existing cron covers it, and there is no `messages.unscheduledReminderNote` config on `main`. The requested opt-out is not implemented on `main`, so this PR should not be closed as already fixed or obsolete.
