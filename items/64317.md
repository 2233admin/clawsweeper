## Summary

Current main does not implement the reported Hyprland/Wayland headed viewport screenshot fallback. The normal managed-browser viewport screenshot path still routes non-label, non-ref, non-element screenshots to the CDP `Page.captureScreenshot` implementation, and there is no Hyprland/grim native capture path in the current browser plugin. The issue is recent, has concrete reproduction details, and the linked fix PR #67149 is closed/unmerged in the provided context, so this should remain open.

## Best Possible Solution

Keep the issue open and either revive a narrow version of PR #67149 or implement the same behavior in the browser plugin: detect headed Hyprland/Wayland only for normal viewport screenshots, use a native grim/fake-output fallback there, and preserve the existing CDP/Playwright paths for full-page, element, ref, labels, and non-Hyprland cases. Add focused tests for routing/fallback behavior and a short troubleshooting or browser-control docs note if user setup requirements such as `grim`/`hyprctl` are exposed.
