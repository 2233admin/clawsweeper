## Summary

Keep open. Current main at 6d60b035b4e7c1739f038808e041567448f58060 still does not implement this PR's central behavior: health snapshots have no runtime snapshot input, the gateway health RPC still serves through the cached refresh path, and Discord still only publishes the connected status patch when the gateway plugin is already connected. The PR discussion includes a concrete live repro and an addressed review blocker, so this is plausible maintainer backlog rather than cleanup-close material.

## Best Possible Solution

Keep this PR open or consolidate it with a maintainer-chosen canonical health-runtime PR. The product fix should thread runtime channel/account state into health snapshots, preserve explicit probe intent, update gateway health cache/broadcast behavior without overwriting runtime fields, remove the Discord gateway.isConnected startup gate, and add regression coverage for the health snapshot and Discord race cases.
