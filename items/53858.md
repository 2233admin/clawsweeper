## Summary

Current main still appears to contain the reported Nostr restart-loop bug. The Nostr bus passes an array-wrapped DM filter to `nostr-tools`, the gateway start path returns immediately after creating the bus, and the channel supervisor restarts settled account tasks. Related PR #42832 remains open and is the right implementation candidate or reference point, but this issue should not be closed before a fix lands.

## Best Possible Solution

Keep this issue open as the canonical current Nostr restart-loop/subscription tracker. Review and rebase PR #42832 or prepare a fresh patch on current main that passes a plain `Filter` object to `SimplePool.subscribeMany`, removes the unsafe cast, keeps `startNostrGatewayAccount` pending until `ctx.abortSignal` aborts using the existing channel lifecycle helper, closes both the subscription and SimplePool relay connections during cleanup, and adds regression coverage for the serialized REQ filter shape plus gateway lifecycle so a successful Nostr start is not treated as an exit.
