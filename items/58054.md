## Summary

Keep #58054 open. Current main still has the reported pinned-write error masking behavior: non-SafeOpenError failures from the pinned write helper are converted to the generic "path is not a regular file under root" message, and current main has no python3/Xcode install-hint helper. The PR remains a focused fix for a real current path, though it should get regression coverage before landing or be reimplemented equivalently.

## Best Possible Solution

Keep the PR open. The maintainer path is to add focused regression coverage around normalizePinnedWriteError and/or runPinnedWriteHelper error propagation, then merge this patch or implement the same behavior on main so plugin/archive install failures surface the real python3/Xcode cause instead of the generic safe-open message.
