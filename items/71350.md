## Summary

Keep #71350 open. Current main already implements the narrow per-agent `thinkingDefault` schema/runtime path and flat per-agent inference params via `agents.list[].params`, including `temperature` and `maxTokens`. The broader requested surface is still incomplete: there is no `agents.list[].models` per-agent per-model merge layer, and no `thinkingStrategy` rules/escalation/AGENTS.md-driven adaptive thinking implementation.

## Best Possible Solution

Keep #71350 open as the product/design tracker. The best path is to acknowledge that `agents.list[].thinkingDefault` and flat `agents.list[].params` already ship, decide whether OpenClaw wants a true `agents.list[].models` merge layer or just clearer docs for flat params, and design adaptive thinking as an explicit core agent-loop policy or approved agent/tool API with schema, generated config help, docs, and tests for rule matching, provider-supported levels, and failure escalation.
