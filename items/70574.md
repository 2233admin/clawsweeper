## Summary

Keep open. Current main still matches the reported failure mode: cross-agent `sessions_spawn` resolves the requester origin through the child/target agent's binding and then persists that origin for completion delivery. PR #70607 is an open proposed fix for the same bug, but it has not landed on `main`, and the issue is a confirmed recent core routing bug rather than cleanup-closable stale work.

## Best Possible Solution

Keep #70574 open until a fix lands. The best path is to preserve the parent's requester origin/accountId for subagent completion registration and delivery, while retaining target-agent account resolution only where it is actually needed for child thread-binding hooks or bound session setup. PR #70607 is the canonical starting point, but maintainers should verify it does not break the Matrix/Discord thread-binding behavior covered by existing tests.

## Evidence
