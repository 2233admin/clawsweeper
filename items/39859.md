## Summary

Keep open. Current main still does not implement PR #39859's central fix: a positive contextTokens override returns before any known model window is resolved, so status and session metadata can still over-report the effective window.

## Best Possible Solution

Keep the PR or linked issue open until main implements the narrow resolver behavior: resolve the known model context window first, return Math.min(contextTokensOverride, modelTokens) when both are known, preserve existing override behavior when the model window is unknown, and add focused regression coverage for larger-override capping, lower-override preservation, and normalized provider-alias lookup.
