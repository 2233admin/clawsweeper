## Summary

Keep open. Current main still returns a positive `contextTokensOverride` before resolving known model/config/discovery context windows, and normal status/session paths still pass or persist that override, so PR #39859 is not obsolete or implemented on main.

## Best Possible Solution

Keep #39859 and linked #39857 open. Land a narrow current-main fix that resolves the known model/config/discovery window before honoring positive `contextTokensOverride`, returns `Math.min(override, modelTokens)` when both are known, preserves override behavior when the model window is unknown, updates the duplicate status-summary/session persistence paths, and adds focused resolver/status/session regression tests. Do not carry over the stale PR branch's unrelated CI, secrets-baseline, or timeout-only commits.
