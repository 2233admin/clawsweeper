## Summary

Keep #72146 open. This is a fresh regression report with concrete logs and reproduction steps, and current main does not contain a clear shipped fix for the reported LaunchAgent gateway failure. The timeline-referenced commit proposes a macOS app-side LaunchAgent change, but that change is not reflected in current main: the app still ignores `bundlePath` and still installs with `--runtime node`. The CLI install surface also does not expose the proposed `--path` option, so there is not enough evidence to close this as implemented or superseded.

## Best Possible Solution

Keep the issue open for maintainer investigation. The likely path is to reproduce on macOS 26 with `v2026.4.24`, inspect the generated `ai.openclaw.gateway.plist`, verify whether the failure comes from CLI LaunchAgent lifecycle, the macOS app's `GatewayLaunchAgentManager`, runtime path selection, token/config drift, or a stale listener loop, and then land a focused fix with macOS-side or daemon-cli regression coverage.
