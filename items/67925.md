## Summary

Keep open. Current `main` still appears to have the reported false-positive path: the phantom-allowlist audit only excludes installed plugin directories and `listChannelPlugins()` IDs, not bundled non-channel plugin IDs such as `browser`, `acpx`, `memory-core`, or `qqbot`.

## Evidence

- **Current audit logic only excludes channel plugin IDs:** `collectPluginsTrustFindings()` builds `bundledPluginIds` from `listChannelPlugins()` and then flags any `plugins.allow` entry not found in installed extension dirs or that channel-plugin set. There is no lookup against bundled plugin metadata for non-channel bundled plugins.
  - file: [src/security/audit-plugins-trust.ts:283](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/security/audit-plugins-trust.ts#L283)
  - command: `nl -ba src/security/audit-plugins-trust.ts | sed -n '270,295p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Channel registry is channel-only:** `listChannelPlugins()` returns loaded channel plugins from the channel registry; it is not a general bundled-plugin inventory, so stock bundled plugins are outside this exclusion set.
  - file: [src/channels/plugins/registry.ts:8](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/channels/plugins/registry.ts#L8)
