## Summary

Keep this PR open. Current main still lacks the two central behaviors: agent command delivery does not synthesize a fallback when raw payloads normalize/project to zero deliverable outbound payloads, and Slack preview/progress turns that end without a final reply still clear state and return without the PR's in-thread no-final fallback. The PR discussion contains useful review guardrails, but the branch should be rebased and narrowed before merge review.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rebase. The best path is to port a scoped normalized-empty fallback into `src/agents/command/delivery.ts` after outbound projection and a scoped Slack preview-only no-final fallback into `extensions/slack/src/monitor/message-handler/dispatch.ts`, preserving the review-driven guardrails and dropping unrelated UI formatting churn. Related #45290 is broader delivery recovery work, but it is not established as the canonical replacement for this PR's normalized-empty command fallback and Slack preview-only no-final behavior.
