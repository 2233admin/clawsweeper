Keep PR #40366 open. Current main still does not implement the PR's two core behaviors: agent command delivery has no normalized-empty fallback, and Slack preview/progress turns that finish without a final reply still clear state and return silently.

## Best Possible Solution

Keep #40366 open for maintainer review or contributor rebase. The best path is to adapt the branch's normalized-empty agent delivery fallback and Slack preview-only no-final fallback to current src/agents/command/delivery.ts and extensions/slack/src/monitor/message-handler/dispatch.ts, preserving the reviewed guardrails for intentional NO_REPLY, tool-sent empty success, block/tool success, same-thread status delivery, draft clearing, and JSON output.
