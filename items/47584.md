## Summary

Keep open. Current `main` does not implement Siri/App Intent integration for the iOS app, and this is an in-scope OpenClaw companion-app feature rather than a ClawHub-only request.

## Evidence

- **No App Intent implementation on main:** A repo search over `apps/ios` source, extensions, and tests found no `AppIntent` conformances, `AppShortcutsProvider`, Siri, or Shortcuts implementation code. The only App Intents references are framework links in project configuration.
  - command: `rg -n "struct .*: AppIntent|AppShortcutsProvider|AppIntent|AppIntents|Siri|Shortcuts|IntentDescription|ProvidesDialog|ReturnsValue" apps/ios/Sources apps/ios/ShareExtension apps/ios/WatchExtension apps/ios/ActivityWidget apps/ios/Tests`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **App Intents metadata is disabled:** The iOS app target explicitly sets `ENABLE_APPINTENTS_METADATA` and `ENABLE_APP_INTENTS_METADATA_GENERATION` to `NO`, which is inconsistent with shipping discoverable Siri/App Shortcuts intents.
  - file: [apps/ios/project.yml:99](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/apps/ios/project.yml#L99)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Current documented iOS feature surface omits Siri/Shortcuts:** The platform docs list pairing, canvas, camera, location, talk mode, and voice wake as iOS capabilities, with no Siri/App Intent/Shortcuts surface described.
  - file: [docs/platforms/ios.md:12](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/docs/platforms/ios.md#L12)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Current shipped iOS features are chat/voice/share/device actions:** The iOS changelog and App Store promotional text mention chat, voice, sharing, automations, camera, and device actions, but not Siri or App Intents. Dynamic Island support exists only through the Live Activity widget.
  - file: [apps/ios/CHANGELOG.md:47](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/apps/ios/CHANGELOG.md#L47)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **This belongs in OpenClaw app scope, not ClawHub:** `VISION.md` lists companion apps, including iOS, as a project priority area. This is a native iOS app integration request, not a new core skill/plugin that should be redirected to ClawHub.
  - file: [VISION.md:25](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/VISION.md#L25)
