## Summary

Keep open. Current `main` still renders `last exit` and `reason` for any runtime object, including `status === "running"`, so the stale-systemd-status bug described by the PR is not implemented on `main`. The PR’s formatter-only approach also matches the review concern that shared runtime fields should remain available for restart/onboarding diagnostics, but there is still a real test gap around the status-rendering path.

## Evidence

- **Current formatter still prints exit details unconditionally:** `formatRuntimeStatus()` always appends `last exit` and `reason` whenever `lastExitStatus` / `lastExitReason` are present; there is no `status === "running"` suppression on current `main`. That means the user-facing `openclaw status` path described in the PR is still present in code.
  - file: [src/daemon/runtime-format.ts:23](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/daemon/runtime-format.ts#L23)

  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
