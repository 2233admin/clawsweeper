## Summary

Keep open. Current `main` does not implement Brazilian WhatsApp ninth-digit equivalence in outbound target resolution, inbound/self-chat/command authorization, pinned main `lastRoute` refresh, or heartbeat recipient authorization. The PR is addressing a real gap, and the review thread also raised follow-up reaction/session-routing regressions that still need maintainer judgment before merge.

## Evidence

- **Outbound resolution is still exact-match only on main:** `resolveWhatsAppOutboundTarget()` still accepts direct targets only when `allowList.includes(normalizedTo)`. There is no BR ninth-digit equivalence or canonical allowlist reuse in current `main`, so `+55 AA 9XXXX XXXX` vs `+55 AA XXXX XXXX` still diverge.
