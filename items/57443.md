## Summary

Keep #57443 open. Current main has a shared schema normalizer and a compat flag for unsupported tool-schema keywords, but Ark/BytePlus/Volcengine models do not currently receive `patternProperties` stripping by default. The linked #57480 PR appears to be the active fix path, while #57539 was closed unmerged, so the user-visible bug is not safely implemented on main yet.

## Best Possible Solution

Keep the issue open and use #57480 or an equivalent core/provider compat patch as the canonical implementation path. The fix should either add Ark/BytePlus/Volcengine `modelCompat.unsupportedToolSchemaKeywords: ["patternProperties"]` coverage with regression tests for `exec.env`, or deliberately move to a broader non-Anthropic/OpenAI-compatible schema-cleaning policy with tests proving Anthropic keeps full JSON Schema support.

## Evidence
