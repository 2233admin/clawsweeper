## Summary

Keep PR #71249 open. Current main still has the targeted stale `pluginDebugEntries` behavior: verbose/trace replies refresh persisted session metadata and append plugin debug lines, but main has no turn-start clear, no preserve-activity store-update option for metadata-only clears, and no suppression path when a persisted clear fails. The PR context also includes focused regression coverage and a bot review finding that the author addressed, so this is not obsolete cleanup.

## Best Possible Solution

Keep PR #71249 open for maintainer review and eventual merge or an equivalent maintainer-authored patch. The best code path is the one described by the PR: clear turn-scoped plugin debug entries at reply turn start, preserve `updatedAt` for that metadata-only clear, make cleanup best-effort, suppress stale rehydration after clear failure, and keep the regression coverage in `src/config/sessions.test.ts` plus `src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts`.
