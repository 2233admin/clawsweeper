## Summary

Keep open. Current `main` already converts Discord markdown tables into code-style output, so the issue body's raw-Discord-table claim is partly outdated, but the shared table renderer still measures columns with `String.length` semantics. That leaves the reported CJK misalignment unresolved for Telegram and for Discord's converted code-block tables.

## Evidence

- **Shared table renderer still uses character count, not display width:** `renderTableAsCode` computes widths from `cell?.text.length` and pads rows with `widths[i] - (cell?.text.length ?? 0)`, so double-width CJK glyphs will still misalign later columns in monospace output.
  - file: [src/markdown/ir.ts:521](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/markdown/ir.ts#L521)
  - command: `nl -ba src/markdown/ir.ts | sed -n '521,549p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Telegram still renders table code blocks through the shared markdown IR path:** Telegram maps `code_block` to `<pre><code>` and calls `markdownToIR(..., { tableMode })`; the send path resolves `tableMode` for Telegram before rendering HTML, so the same width bug flows into Telegram output.
  - file: [extensions/telegram/src/format.ts:59](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/telegram/src/format.ts#L59)
  - command: `nl -ba extensions/telegram/src/format.ts | sed -n '45,95p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Discord no longer sends raw markdown tables as-is:** Discord now resolves `tableMode` and runs `convertMarkdownTables(text ?? '', effectiveTableMode)` before sending. That means the report's Discord 'raw pipe syntax' symptom is partially obsolete on current `main`, but it still depends on the same shared table conversion path.
  - file: [extensions/discord/src/send.outbound.ts:158](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/discord/src/send.outbound.ts#L158)
  - command: `nl -ba extensions/discord/src/send.outbound.ts | sed -n '153,180p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Default table mode remains code-style rendering:** `resolveMarkdownTableMode` falls back to `"code"` when a channel has no explicit default override, which keeps Telegram/Discord on the code-block table path unless config changes it.
  - file: [src/config/markdown-tables.ts:88](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/markdown-tables.ts#L88)
