## Summary

Keep open. Current `main` does not implement this change, and the current command-auth / DM-policy code explicitly treats pairing-store identities as DM-only rather than a source of group command authorization. The proposed fallback-only patch also does not match the current provider resolution path for Telegram/Discord.
