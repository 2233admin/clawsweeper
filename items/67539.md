## Summary

Keep open: current `main` still only injects a generic TTS system hint, and the speech provider plugin contract does not expose any provider-owned prompt-hint hook. Google Gemini expressive tags are documented and covered by provider tests, but they are not surfaced to agents at runtime.

## Evidence

- **Speech provider contract has no prompt-hint hook:** `SpeechProviderPlugin` currently ends at config/directive/voice-listing methods; there is no provider-specific system-prompt or TTS-hint callback on the public speech provider contract.
  - file: [src/plugins/types.ts:1709](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/plugins/types.ts#L1709)
  - command: `nl -ba src/plugins/types.ts | sed -n '1700,1745p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Current TTS runtime hint is generic only:** `buildTtsSystemPromptHint` returns only generic lines about TTS being enabled, max length, and `[[tts:...]]` / `[[tts:text]]` usage. It does not inspect the active provider or append provider/model-specific expressive guidance.
  - file: [extensions/speech-core/src/tts.ts:387](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/speech-core/src/tts.ts#L387)

  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Agent prompt assembly uses that single generic hint:** The system prompt path still pulls exactly `buildTtsSystemPromptHint(params.cfg)` and passes it through as `ttsHint`, so runtime agent guidance remains whatever the generic helper returns.
  - file: [src/auto-reply/reply/commands-system-prompt.ts:147](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/auto-reply/reply/commands-system-prompt.ts#L147)

  - file: [docs/providers/google.md:277](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/docs/providers/google.md#L277)
