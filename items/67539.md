## Summary

Keep this issue open. Current main still has only the generic TTS system prompt hint, the speech-provider contract has no provider-owned prompt-hint hook, and Gemini audio-tag support is documented/tested only at synthesis time rather than injected into runtime agent guidance.

## Best Possible Solution

Add an additive optional provider-owned prompt-hint hook to SpeechProviderPlugin and the public SDK; have speech-core resolve only the active provider, resolved provider config/model/voice, target, and modelOverrides before appending that provider's short hint; then let the Google provider own the Gemini square-bracket audio-tag guidance and cover active-provider-only and policy-gated cases in tests.

## Risks / Open Questions

- Adding this should change the public speech-provider/plugin SDK contract, so SDK exports, provider docs, and contract tests need to move together.
- Provider hints must be gated to the selected provider/model/target and modelOverrides policy so inactive provider syntax or disabled controls are not advertised.
