## Summary

Keep this PR open. Current main still returns memory corpus supplements in registration/re-registration order, while adjacent prompt supplements are sorted by plugin id for prompt-cache-stable output. The PR remains a focused implementation candidate with matching regression-test coverage in the provided diff, and the diff does not expand security-sensitive or supply-chain surfaces.

## Best Possible Solution

Review or land the focused fix after any needed rebase: sort `listMemoryCorpusSupplements()` by `pluginId` in `src/plugins/memory-state.ts`, and keep regression tests covering reverse registration and same-plugin re-registration stability.
