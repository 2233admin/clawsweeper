## Summary

Current main still hardcodes session-memory snapshots under `<workspace>/memory/`, so the requested configurable `outputDir` is not implemented. The PR addresses a real workflow problem raised in discussion, but it should stay open for revision because the proposed filesystem write path has a symlink side-effect risk and the docs changes include localized files that current repo policy says not to edit.

## Best Possible Solution

Keep this PR open and request a focused revision: preserve the default `memory` output, add optional workspace-relative `hooks.internal.entries.session-memory.outputDir`, reject empty/absolute/traversal values, write through `writeFileWithinRoot` rooted at the workspace with `relativePath` set to `outputDir/filename`, remove the separate pre-write mkdir or replace it with an alias-safe helper, add regression coverage for custom output dirs, invalid fallback, and symlink alias behavior, and drop localized docs edits. Coordinate with #54524 for broader duplicate-indexing guidance or dedupe policy.
