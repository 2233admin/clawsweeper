## Summary

Keep PR #40618 open. Current main still hardcodes bundled session-memory output to <workspace>/memory/, while the PR addresses a concrete memory-search/durable-memory pollution problem reported in discussion. The work is not implemented or superseded, but the PR should be revised before merge to avoid unsafe pre-creation of configured output directories and to follow docs i18n policy.

## Best Possible Solution

Keep the PR open and request a small focused revision: add hooks.internal.entries.session-memory.outputDir as an optional workspace-relative handler config with default memory; validate empty, absolute, traversal, and alias/symlink cases before any filesystem side effects; write through writeFileWithinRoot rooted at the workspace with a relative outputDir/filename path or an equivalent alias-safe helper; add regression tests for custom output directories, invalid fallback, and symlink alias behavior; update English hook docs and bundled metadata only.
