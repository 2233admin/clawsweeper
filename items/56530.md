## Summary

Keep open. Current `main` still does not match Slack channel allowlist entries stored as `channel:<CHANNEL_ID>` inside `resolveSlackChannelConfig(...)`, so this PR appears to fix a real unfixed bug rather than duplicate behavior already shipped on `main`.

## Evidence

- **Resolver still omits `channel:`-prefixed candidates:** On current `main` (`6b618f063516aa9ba85903aa4565c5a204c055b2`), `resolveSlackChannelConfig(...)` only builds candidates from the bare channel ID, lower/upper case variants, and optional name-based matches. It does not add `channel:${channelId}` variants before calling `resolveChannelEntryMatchWithFallback(...)`.
  - file: [extensions/slack/src/monitor/channel-config.ts:114](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/slack/src/monitor/channel-config.ts#L114)
  - command: `nl -ba extensions/slack/src/monitor/channel-config.ts | sed -n '100,145p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Current tests cover bare ID case-folding but not `channel:` keys:** The existing `resolveSlackChannelConfig` tests verify lowercase/uppercase bare channel IDs, then jump to name-matching behavior. There is no regression test covering a config entry like `channel:C0...` matched against a runtime bare `C0...` channel ID.
  - file: [extensions/slack/src/monitor/monitor.test.ts:61](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/slack/src/monitor/monitor.test.ts#L61)
  - command: `nl -ba extensions/slack/src/monitor/monitor.test.ts | sed -n '1,110p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Related Slack code already treats `channel:` keys as legitimate config shape:** `resolveSlackChannelType(...)` explicitly recognizes configured channel keys normalized as either the bare channel ID or `channel:${channelIdLower}`, which makes the reported mismatch in `resolveSlackChannelConfig(...)` plausible and internally consistent with the rest of the Slack integration.
  - file: [extensions/slack/src/channel-type.ts:40](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/slack/src/channel-type.ts#L40)
  - command: `nl -ba extensions/slack/src/channel-type.ts | sed -n '20,55p'`
