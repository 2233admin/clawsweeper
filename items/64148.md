## Summary

Keep PR #64148 open. Current `main` still archives the state asset with `tar.c(...)` without any `filter` option, so `~/.openclaw/extensions/*/node_modules` would still be included as part of the state directory payload. The PR remains a focused implementation for open issue #64144, and its review thread already addressed the two important edge cases raised during review: Windows path separators and canonical state asset paths.

## Best Possible Solution

Keep the PR open or land an equivalent narrow fix on `main`: add a `tar.c` filter in `src/infra/backup-create.ts` that excludes `node_modules` only under the canonical state `extensions/` subtree, while keeping manifests, plugin source, and non-extension `node_modules` untouched. Retain focused tests for POSIX paths, Windows-style paths, nested extension dependencies, and non-extension paths.
