## Summary

Keep this PR open. Current main still backs up the resolved state directory as a whole through `tar.c()` without an exclusion filter for `extensions/*/node_modules`, and neither the CLI/docs nor the current tests expose or verify that behavior. The PR remains a focused implementation candidate; the broader backup exclude-pattern PR is open but does not clearly supersede this default extension-dependency exclusion. Security review found no dependency, workflow, lockfile, publishing, secret, or external code-execution changes in the provided PR diff.

## Best Possible Solution

Keep this PR open for maintainer review. Either land or recreate the narrow default exclusion with canonical path handling, Windows path normalization, restore-facing docs/tests as needed, and regression coverage, or explicitly fold the same `extensions/*/node_modules` behavior into the broader backup exclude-pattern work before closing this PR.
