## Summary

Keep this issue open. Current `main` still does not parse Nextcloud Talk rich-object `object.content`, does not retain rich-object `parameters`, and computes the group mention gate only by running configured regexes against the raw message text. The related open PR #49869 is useful context for rich-object parsing, but it is not merged, is scoped to file shares/#29152, and does not close this mention-gating issue.

## Best Possible Solution

Keep this issue open. The fix should live in `extensions/nextcloud-talk`: parse JSON rich-object `object.content`, preserve `parameters`, resolve `{mention-userN}` placeholders against the configured bot identity such as normalized `apiUser` or a new explicit bot-user setting, feed that as explicit mention evidence into the shared mention-decision helper, and add regression tests where `requireMention: true` accepts a bot placeholder mention but drops non-bot placeholders. #49869 can be reviewed as a rich-object parsing starting point, but this issue should close only after the mention-gating behavior ships on `main`.
