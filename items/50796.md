## Summary

Keep open. Current `main` still does not sanitize assistant internal scaffolding on the Control UI display or streamed chat paths, so this request is neither implemented on `main` nor safely obsolete. The PR discussion also identified that the proposed patch only covered part of the surface.

## Evidence

- **Display-message sanitizer on main still only strips inline reply/audio directives:** `stripInlineDirectiveTagsFromMessageForDisplay` still maps text parts through `stripInlineDirectiveTagsForDisplay(record.text).text` only; it does not call `stripAssistantInternalScaffolding` or `sanitizeAssistantVisibleText`. That means `<relevant-memories>...</relevant-memories>` would remain in final chat payloads that use this helper.

No assistant-visible scaffolding sanitizer is applied there, matching the review comment that the normal streamed path was still uncovered.
