## Summary

Keep this PR open. It is the active implementation candidate for the still-open same-author bug #63069, and current main still hardcodes shell completion profile paths instead of respecting ZDOTDIR/XDG_CONFIG_HOME.

## Best Possible Solution

Keep this PR open as the implementation candidate for #63069. The maintainer path is to adapt it to current main by putting a shared env-aware profile resolver in src/cli/completion-runtime.ts, using it consistently for install/check/doctor/setup/update-facing hints, preserving the intended bash fallback behavior, and adding focused tests for zsh ZDOTDIR, fish XDG_CONFIG_HOME, bash .bashrc/.bash_profile selection, and non-Windows PowerShell XDG_CONFIG_HOME if kept in scope.
