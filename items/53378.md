## Summary

Current main still does not show a channel icon/name in the Control UI Sessions table. The table continues to render the raw session key as the primary identity with optional displayName and custom Label fields, while backend session rows already carry channel/origin metadata that the UI type and live-update allowlist do not expose for this view.

## Best Possible Solution

Keep this issue open. Implement the feature in the Control UI Sessions view by adding a compact Channel column or key-cell badge derived from GatewaySessionRow.channel/origin and session-key parsing, preserve the existing editable Label field, align ui/src/ui/types.ts and ui/src/ui/controllers/sessions.ts with backend channel/origin metadata, and add focused Sessions view coverage for Telegram and Feishu direct/group rows.
