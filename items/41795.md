## Summary

Keep open. Current `main` already routes Telegram through standard `HTTP_PROXY`/`HTTPS_PROXY` handling, but it does not implement the PR's new `OPENCLAW_TELEGRAM_PROXY` env var. The review comments also identified a real gap for malformed explicit proxy values, and current `main` still treats explicit Telegram proxy initialization as fail-closed rather than graceful fallback.
