## Summary

Current main already fixes the v2026.4.24 Bonjour/CIAO crash class. The issue discussion's #72089 cross-reference matches the same missing/scoped rejection-handler path, and current code now wires Bonjour to shared process error handlers, classifies CIAO cancellation/assertion failures as handled, preserves handler registration across advertiser restarts, and has targeted regression coverage for the watchdog restart/self-disable behavior.

## Best Possible Solution

Close as implemented on main. Keep the current Bonjour process-handler wiring and watchdog recovery behavior; users pinned to v2026.4.24 can keep the documented Bonjour-disable workaround until they update to a build containing these changes.
