## Summary

Current main implements the bundled runtime-deps hardening needed for #72200: npm installs are anchored to an OpenClaw-owned execution root, inherited npm global/prefix settings are stripped, runtime-deps installs are serialized by filesystem locks, and bundled channel loading now goes through the same prepared runtime root path. The issue discussion also narrowed the original report to home-directory npm project leakage, and a later commenter confirmed the same ENOTEMPTY family was fixed in beta.

## Best Possible Solution

Close #72200 as implemented on current main and keep the existing bundled runtime-deps hardening. If a post-main or post-beta build still shows ENOTEMPTY or home-level node_modules writes, it should be reopened as a new narrow report with fresh logs, version, install method, stage-dir setting, and affected bundled channel/plugin details.
