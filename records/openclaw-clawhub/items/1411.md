## Summary

Keep this PR open. Current main does not implement owner-editable skill summaries: the detail header still renders the summary read-only, there is no `api.skills.updateSummary` wiring, and the publish/docs path still treats summary as publish/frontmatter-derived metadata. The PR is a useful implementation attempt tied to open #1345, but it needs revision around authorization, validation, valid JSX, tests, and moderation/security semantics before merge.

## Best Possible Solution

Keep this PR open for maintainer/contributor revision. A mergeable version should add a validated owner-facing summary update mutation, align authorization with the owner/publisher-member/moderator model, trim and cap the summary, preserve trigger-driven `skillSearchDigest` sync, define how post-publish summary edits affect moderation/security state, render valid JSX, and add focused Convex/frontend tests while coordinating with #1345.
