## Summary

Keep this PR open. Current main still has a read-only skill summary, and the proposed branch adds the feature but leaves blocking correctness and security-sensitive gaps around publisher ownership, moderation state, denormalized search/catalog data, invalid HTML, and length validation.

## What This Changes

The PR adds a public Convex mutation, detail-page state wiring, inline header controls, and CSS so authorized users can edit a skill summary from the skill detail page.

## Best Possible Solution

Land a revised implementation that treats editable summaries as first-class metadata: define override semantics versus publish-derived descriptions, authorize the same direct-owner, publisher-member, and moderator actors exposed by the UI, update all denormalized list/search/API surfaces, and recompute or explicitly reset moderation state with audit coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep search digests in sync when summaries change:** `convex/skills.ts:4814-4818`
  - body: The mutation patches only the `skills` document. Current catalog/search/list paths read `summary` from `skillSearchDigest`, so after saving an edited summary the detail page may show the new text while browse, catalog search, and API list results continue returning the old summary until another backfill or publish updates the digest.
  - confidence: 0.91
- **[P1] Keep moderation state in sync with edited summaries:** `convex/skills.ts:4814-4818`
  - body: The publish path derives moderation flags from the persisted summary, but this mutation changes public searchable text without recomputing `moderationFlags`, `isSuspicious`, or moderation summary fields. Owners could change text after the latest version was scanned while public safety state still describes the old summary.
  - confidence: 0.9
- **[P1] Authorize publisher-owned skill managers:** `convex/skills.ts:4808-4812`
  - body: The UI shows this editor through `canManage`, which includes members of `skill.ownerPublisherId`, but the mutation only allows the direct `ownerUserId` or moderators. Publisher members can open the editor and will always fail on save for publisher-owned skills.
  - confidence: 0.93
- **[P2] Move the summary edit form out of the paragraph:** `src/components/SkillHeader.tsx:211-214`
  - body: When edit mode is active, the new code renders a `<form>` and block children inside `<p className="section-subtitle">`. Browsers implicitly close paragraphs around flow content, which can cause hydration and layout issues; use a non-paragraph wrapper for both static and editable states.
  - confidence: 0.95
- **[P2] Cap editable summary length on the server:** `convex/skills.ts:4800-4803`
  - body: The public mutation accepts an unrestricted Convex string and writes it to skill metadata that is returned on public list, detail, search, and API surfaces. Add a server-side max length, with a matching textarea limit, to prevent response bloat and document-size pressure.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The PR adds a public metadata write path that can change public searchable text without moderation recomputation or size limits.

Concerns:

- **[medium] Edited summaries bypass moderation recomputation:** `convex/skills.ts:4814`
  - body: `updateSummary` changes public summary text but does not recompute moderation flags or `isSuspicious`, even though current publish logic derives safety flags from the persisted summary. Public safety labels can become stale after an owner edit.
  - confidence: 0.9
- **[low] Summary writes are unbounded:** `convex/skills.ts:4800`
  - body: The new public mutation accepts any string and stores it on public skill metadata surfaces. A server-side cap is needed to limit document-size and response-bloat pressure from malformed or hostile clients.
  - confidence: 0.78
