## Summary

Keep this PR open. Current main still does not implement owner-editable skill summaries: the detail header renders the summary read-only, no `api.skills.updateSummary` wiring exists, and the owner settings route only exposes rename/merge tools. The PR remains a useful implementation path for related #1345, but it needs maintainer/contributor revision for authorization, validation, valid JSX, tests, and moderation/security semantics before merge.

## What This Changes

The PR adds a public Convex `updateSummary` mutation plus detail-page state, an inline header edit form, Save/Cancel controls, and CSS so skill owners or moderators can edit a skill summary from the skill detail page.

## Best Possible Solution

Keep this PR open for contributor or maintainer revision. A mergeable version should add a validated owner-facing summary update path, align authorization with the owner/publisher-member/moderator model, trim and cap the summary, preserve `skillSearchDigest` synchronization, decide how summary edits affect moderation/rescan state, render valid JSX, and add focused Convex/frontend tests while coordinating with related #1345.
