## Summary

Keep open. This is an open ClawHub issue, and the repo cleanup policy only permits auto-closing ClawHub PRs that current main certainly implements. Current main now exposes suspicious/moderation state through the JSON API and docs, but the requested static machine-readable skill-page metadata is still missing.

## Best Possible Solution

Keep this issue open for maintainer product/security follow-up. The likely path is to pass public moderationInfo from the skill page loader into the route head data, emit stable static metadata such as clawhub:suspicious and possibly a conservative public verdict/reason summary, add route/head and og helper tests for flagged and clean skill pages, and document when agents should use static metadata versus the existing v1 detail/moderation APIs. Coordinate with #731 for deeper scanner findings, but do not treat it as a replacement for this static metadata request.
