## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy for this review only permits auto-closing pull requests that current main definitely implements. Substantively, current main exposes suspicious/moderation state through JSON APIs and docs/tests, but the requested static HTML/meta-tag path for agents fetching skill pages is still absent.

## Best Possible Solution

Keep this issue open for maintainer product handling. A good implementation would carry public moderationInfo from the skill page loader into the route head path, add stable clawhub:suspicious metadata and a public verdict/reason summary where appropriate, extend route-head tests for flagged and clean skills, and keep the already-shipped JSON API moderation fields documented as the richer machine-readable path.
