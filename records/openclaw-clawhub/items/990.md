Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy for this review run says ClawHub issues should remain open; only pull requests may be auto-closed when current main definitely implements them. Substantively, current main exposes suspicious/moderation status through the v1 JSON API and docs, but the requested static skill-page metadata path is still missing.

## Best Possible Solution

Keep this issue open for maintainer product handling. A concrete path is to pass public moderationInfo from the skill page loader into the route head metadata path, extend buildSkillMeta or the route head output with stable clawhub:suspicious metadata plus a public verdict/reason summary where appropriate, add tests for flagged and clean skill pages, and document when agents should use static metadata versus the existing detail, moderation, and scan APIs.
