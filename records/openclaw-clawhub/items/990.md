## Summary

Keep open. Current main covers the JSON API and UI parts of the request, but the skill page head still does not expose machine-readable static moderation metadata for HTML-only agent fetches; ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

Request to expose suspicious/moderation status in static skill-page metadata and API responses so agents can avoid flagged skills.

## Best Possible Solution

Add stable public HTML head metadata for skill moderation using already-loaded moderationInfo, keep the v1 JSON moderation response as the API source of truth, add focused tests for flagged and clean skill pages, and document how agents should consume static metadata versus API endpoints.
