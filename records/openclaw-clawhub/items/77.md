## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits automated closure for pull requests that are certainly implemented on main. Current main still does not implement a concrete skill title/display-name limit: schema/shared API types accept plain strings, publish/import paths only require non-empty names, and the card UI only clamps long titles visually.

## Best Possible Solution

Keep this issue open for maintainer product follow-up. Maintainers should define the public skill title/display-name policy first, including a maximum length and handling for existing long names, then implement it consistently across Convex validation, web publish, GitHub import, shared CLI/API schemas, docs, and listing/detail display behavior.
