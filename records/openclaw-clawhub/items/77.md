## Summary

Keep open. This is an open ClawHub contributor issue, and the repository-specific cleanup policy says to keep ClawHub issues open while only auto-closing pull requests that are certainly implemented on main. A source sweep also confirms the requested skill title/display-name limit is still not implemented consistently across the publish UI, GitHub import, backend publish path, API schema, CLI, docs, or focused tests.

## Best Possible Solution

Keep this issue open for maintainer product judgment and implementation. A conservative path is to add a shared skill display-name limit constant and validator, apply it in frontend publish/import flows plus backend/API/CLI publish validation, document the chosen limit, add focused tests, and decide how to handle existing long display names.
