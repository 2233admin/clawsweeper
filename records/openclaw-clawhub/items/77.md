## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open, and current main still does not implement a skill title/display-name length limit in the publish UI, GitHub import path, backend publish path, schema/API payload validation, tests, or docs.

## Best Possible Solution

Keep this issue open for maintainer product judgment and implementation. A conservative implementation would add a shared skill display-name limit constant and validator, apply it in frontend publish/import flows and backend/API/CLI publish validation, add focused tests, document the chosen limit, and decide how existing long display names should be handled.
