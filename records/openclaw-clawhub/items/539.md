## Summary

Keep this issue open. Current main has meaningful mitigations for authenticated CLI/API requests and retry guidance, but the reported anonymous/shared-IP install and search failure mode still exists in current source, related reports #390 and #1410 are still open, and the ClawHub review policy keeps issues open rather than auto-closing them.

## What This Changes

Reduce ClawHub registry API rate-limit false positives for VPS/shared-IP users so search, install, and download flows work reliably or provide a clear auth/mirror path.

## Best Possible Solution

Keep this issue open for maintainer triage and consolidate with #390 and #1410 if desired. The best path is to choose one concrete direction: make CLI authentication the default onboarding path, adjust anonymous shared-IP limits, verify production client-IP forwarding, improve 429 diagnostics across install paths, or design official mirror/offline registry support.
