## Summary

Keep this issue open. The repository policy for openclaw/clawhub only allows automated closes for pull requests that are certainly implemented on main, and current main only partially mitigates the reported shared-IP/VPS rate limiting: authenticated requests use per-user buckets and the CLI handles retry headers, but anonymous search, package metadata, and download/install paths still use per-IP buckets.

## Best Possible Solution

Keep this issue open for maintainer triage. The concrete path is to decide whether this should be consolidated with #390 and #1410, then choose the product/backend fix: default CLI login, higher anonymous shared-IP headroom, verified production IP forwarding, clearer anonymous 429 diagnostics, or official mirror/offline support.
