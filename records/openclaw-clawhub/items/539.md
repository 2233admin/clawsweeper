## Summary

Keep this issue open. Current main has real mitigations: authenticated API-token traffic uses per-user buckets, the CLI passes stored tokens through search/install/download, and 429 responses now include retry guidance. But anonymous VPS/shared-egress traffic is still explicitly IP-enforced for search, package detail, and downloads, and related open issues still track the same remaining failure mode. The ClawHub review policy also says issues should stay open; only pull requests may be auto-closed when certainly implemented on main.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to consolidate it with #390 and #1410 if maintainers want one canonical thread, then choose a backend/product fix: make CLI authentication the default onboarding path, reshape anonymous shared-IP limits, verify production client-IP forwarding, improve 429 diagnostics across all install paths, or design official mirror/offline registry support.
