## Summary

Keep this issue open. Current main has meaningful ClawHub registry mitigations for malicious skill uploads, including static detection of obfuscated terminal payloads, malicious verdict mapping, hidden/quarantine behavior, public filtering, and CLI install/update gates. However, this item is an issue in openclaw/clawhub, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. Substantively, the report also asks for OpenClaw runtime installer sandboxing/approval and keychain-backed credential storage, which are not proven resolved in this registry repository.

## Best Possible Solution

Keep this issue open for maintainer security triage. Preserve the shipped ClawHub registry mitigations, then split or link the remaining work explicitly: ClawHub static scanning and abuse-response hardening here, and OpenClaw core installer sandboxing, script approval, and keychain-backed credential storage in the core repository if still applicable.
