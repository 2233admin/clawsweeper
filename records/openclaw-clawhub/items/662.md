## Summary

Keep this issue open. Current main contains substantial ClawHub registry mitigations for the reported malicious-skill upload pattern, and the related #123 incident is closed with those mitigations present. However, the repository-specific cleanup policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, not issues. The report also includes broader OpenClaw runtime installer sandboxing, script approval, outbound network policy, and keychain-backed credential storage asks that are not proven resolved in this registry repository.

## Best Possible Solution

Keep this issue open for maintainer security triage. Preserve the current ClawHub registry mitigations and explicitly split or link the remaining work: ClawHub static scanning and abuse-response hardening in this repository, and OpenClaw runtime installer sandboxing, script approval, outbound network policy, and keychain-backed credential storage in the core repository if still applicable.
