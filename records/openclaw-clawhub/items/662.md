## Summary

Keep open. Repository policy for openclaw/clawhub does not auto-close open issues in this cleanup pass, and current main only partially mitigates the reported attack with ClawHub registry-side scanning, hiding, download/install blocking, and uploader moderation. The broader report still spans security-sensitive OpenClaw runtime sandboxing, install approval UX, credential storage, outbound network controls, and production moderation policy.

## What This Changes

The issue reports that malicious ClawHub skills can use SKILL.md prerequisite or install instructions to execute shell commands and exfiltrate local OpenClaw credentials, and asks for sandboxing, approval prompts, keychain storage, static analysis, and network controls.

## Best Possible Solution

Keep this open for maintainer security triage. Preserve the registry hardening already on main, then explicitly split remaining work between ClawHub scanning/moderation/distribution policy and OpenClaw core security work for sandboxed prerequisite execution, explicit install approval, keychain-backed credential storage, and outbound network controls.
