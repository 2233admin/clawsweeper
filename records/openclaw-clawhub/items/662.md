## Summary

Keep open. This is an open ClawHub security issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. Current main has useful registry-side mitigations for obfuscated install-prompt malware, but the broader report still spans OpenClaw runtime installer sandboxing/approval, outbound network controls, keychain-backed local credential storage, and registry distribution policy decisions that are not fully resolved in this source tree.

## What This Changes

The issue requests mitigation for malicious skills that can execute install or prerequisite shell commands and exfiltrate plaintext OpenClaw credentials, including registry static analysis and runtime hardening.

## Best Possible Solution

Keep this open for maintainer security triage. Preserve the registry static-scan and moderation mitigations already on main, then split or link the remaining work explicitly: ClawHub static-analysis/distribution hardening in this repository, and OpenClaw core issues for sandboxed prerequisite execution, script approval, outbound network controls, and keychain-backed local credential storage.
