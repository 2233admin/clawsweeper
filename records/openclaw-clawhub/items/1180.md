## Summary

Keep this issue open. The repository policy for openclaw/clawhub says to keep issues open, and current main still intentionally renders the dashboard deploy-drift warning when frontend VITE_APP_BUILD_SHA and Convex APP_BUILD_SHA differ.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify the production VITE_APP_BUILD_SHA and Convex APP_BUILD_SHA values; if they differ unintentionally, run the manual full deploy path from main so frontend and backend are stamped from the same commit. If the warning is too noisy during healthy partial deploys, adjust DeploymentDriftBanner policy/severity and update the drift/banner tests.
