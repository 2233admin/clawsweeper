## Summary

Keep this issue open. The repository policy for openclaw/clawhub allows auto-closing only PRs that are certainly implemented on main, and this is an issue. Current main also still contains the reported deployment-drift warning behavior: the frontend SHA is compared with Convex APP_BUILD_SHA, and the banner renders when both are present and different.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify the current production VITE_APP_BUILD_SHA and Convex APP_BUILD_SHA; if they differ unintentionally, run the manual full deploy path so frontend and backend are stamped from the same commit. If the warning is considered too noisy during otherwise healthy operation, adjust DeploymentDriftBanner policy or severity and update the drift/banner tests.
