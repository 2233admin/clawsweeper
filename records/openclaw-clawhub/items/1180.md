## Summary

Keep this ClawHub issue open. The report describes a concrete deployment-version mismatch warning, and current main still renders that warning whenever `VITE_APP_BUILD_SHA` and Convex `APP_BUILD_SHA` differ. The deploy workflow/docs show the intended fix path is to stamp and deploy matching frontend/backend SHAs, but there is no evidence in current main that this user-visible drift warning has been made obsolete. The repository policy for openclaw/clawhub also says this cleanup workflow should keep issues open and only auto-close PRs that are certainly implemented on main.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely next step is to verify production deploy state for the reported SHAs, then either run the manual full deploy path so Convex `APP_BUILD_SHA` and Vercel `VITE_APP_BUILD_SHA` point at the same commit, or adjust the banner/deploy checks if the warning is too noisy during otherwise healthy dashboard operation.
