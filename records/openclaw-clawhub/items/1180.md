## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing PRs that are certainly implemented on current main, and this item is an issue. Current main also still implements the reported drift-warning path: the frontend build SHA is compared with Convex's APP_BUILD_SHA and the banner renders when both are present and different. The related #618 title match concerns CLI sync version display and is not a canonical duplicate for this dashboard deployment-SHA warning.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify production VITE_APP_BUILD_SHA and Convex APP_BUILD_SHA; if they differ unintentionally, run the manual full deploy path so frontend and backend are stamped from the same commit. If the warning is considered too noisy during healthy operation, adjust DeploymentDriftBanner policy or severity and update the drift/banner tests.
