## Summary

Keep open. The repository-specific ClawHub cleanup policy only permits automated closure for pull requests that are certainly implemented on current main, and this item is an open issue. Source review also does not prove the deleted-account OAuth/session problem is fixed: current main intentionally blocks reauthentication for deleted/deactivated accounts, returns no active user for those accounts, and the header still renders as logged out when auth exists but the user document is null. The provided timeline also links the closely related open account-deletion login loop report #886, so maintainer triage should handle the product outcome rather than automated closure.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify the production OAuth callback/session behavior for deleted accounts, decide whether permanent deletion should surface a clear unrecoverable error or support purge/re-registration, and add focused auth/UI coverage around the chosen path. Coordinate with the related open deletion-login-loop report #886 rather than auto-closing this issue.
