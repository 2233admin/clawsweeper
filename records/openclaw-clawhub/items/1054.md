## Summary

Keep this issue open. ClawHub cleanup policy only permits automated closure for PRs that are certainly implemented on current main, and this is an issue. Current main still deliberately blocks deleted/deactivated account reauth instead of recreating a session/profile, although it now has a clearer deleted-account auth error path. The remaining question is account-lifecycle policy, not conservative cleanup.

## What This Changes

Allow a user who deleted their ClawHub account to sign back in via GitHub OAuth and receive an active session/profile instead of remaining logged out.

## Best Possible Solution

Keep this issue open for maintainer account-lifecycle triage. Maintainers should decide whether deleted GitHub identities remain permanently blocked with clearer support UX, can be administratively purged for re-registration, or should be allowed through a deliberate tombstone/re-registration migration coordinated with #647, #886, #906, #1244, and #1552.
