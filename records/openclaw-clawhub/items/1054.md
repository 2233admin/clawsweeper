## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only allows auto-closing PRs that are certainly implemented on current main. The report also maps to an intentional but user-facing account lifecycle behavior: deleted accounts are marked inactive, OAuth reauth is rejected, and active-session helpers return no user, so the remaining decision needs maintainer policy and support handling rather than conservative cleanup closure.

## What This Changes

The issue asks ClawHub to create a usable user session and load the profile after a previously deleted account completes GitHub OAuth again.

## Best Possible Solution

Maintainers should choose and document the intended post-deletion lifecycle, then implement that path consistently: permanent blocking with clear support UX, a verified administrative purge process, or a deliberate tombstone/re-registration flow with tests preserving deletion and security invariants.
