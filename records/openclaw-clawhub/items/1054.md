## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub only allows automated closure for PRs that are certainly implemented on current main, and this item is an open issue. Source review also does not prove the deleted-account OAuth/session problem is fixed: current main intentionally blocks reauthentication for deleted/deactivated accounts, returns no active user for those accounts, and can render the signed-out UI when auth exists but the user document is null. The related deleted-account login-loop report #886 is still open, so maintainers should decide the intended account recovery/error path rather than auto-close this report.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify production OAuth behavior with a deleted account, decide whether deleted-account reauth should surface a clear permanent-deletion error or support purge/re-registration, coordinate with the related open deletion-login-loop report #886, and add focused auth/UI tests for the chosen path.
