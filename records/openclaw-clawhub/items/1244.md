## Summary

Keep this issue open. Current main still treats user-initiated account deletion as irreversible deactivation: OAuth completion rejects deleted/deactivated users, active-user helpers exclude those records, admin unban excludes permanently deleted accounts, and tests/docs assert that behavior. The ClawHub cleanup policy also keeps issues open unless maintainers decide otherwise.

## What This Changes

The issue requests recovery from a GitHub OAuth login loop after the reporter deleted a ClawHub account, likely requiring a purge/reset or supported re-registration path for the same GitHub identity.

## Best Possible Solution

Keep this open for maintainer support plus product/security handling. Maintainers should verify the reporter controls the GitHub identity, inspect the production Convex Auth authAccounts binding and linked user record, then decide whether deleted accounts remain irreversible or whether ClawHub should support an audited restore, GitHub identity unlink, or clean re-registration flow. If recovery is approved, implement it in the Convex Auth/users/admin path with tests and docs; if not, surface the terminal state and support route clearly in web and CLI login flows.
