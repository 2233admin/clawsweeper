## Summary

Keep open. This is an open ClawHub issue, and current main still intentionally blocks OAuth sign-in for deleted or deactivated users; the remaining work is production account support plus account-lifecycle/security policy, not a conservative cleanup close.

## What This Changes

The issue requests restoring, purging, or unbinding the reporter's deleted ClawHub account state so GitHub OAuth can create an active session again.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy, then either provide an audited support or self-service path to restore or unlink the GitHub identity binding with tests, or keep deletion irreversible while showing a clear terminal error and support route during web and CLI login.
