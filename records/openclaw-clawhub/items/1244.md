## Summary

Keep this issue open. The ClawHub cleanup policy says issues should remain open, and current main still treats user-initiated account deletion as irreversible: OAuth reauth for the same deleted/deactivated Convex Auth user is blocked rather than clearing the stale binding, restoring the account, or creating a clean replacement.

## Best Possible Solution

Keep this issue open for maintainer triage of the GitHub OAuth/account deletion lifecycle. Maintainers should decide whether deleted accounts are permanently blocked or whether same-GitHub re-registration should be allowed. If recovery is intended, add a safe backend path to unlink or supersede the old Convex Auth binding, cover it with auth tests, and provide an admin/support repair path for already-stuck accounts. If deletion is intentionally irreversible, make the hosted login flow and support policy clearly surface that state instead of presenting as a silent OAuth loop.
