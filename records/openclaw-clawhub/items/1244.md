## Summary

Keep this issue open. Current main still treats user-initiated account deletion as terminal: deletion writes `deactivatedAt`/`purgedAt`, OAuth completion rejects deleted or deactivated users, active-user helpers exclude those records, and admin unban is explicitly not a permanent-deletion restore path. Because this is a ClawHub issue, repository policy also says not to auto-close issues; the remaining work needs maintainer support plus account-lifecycle/security judgment.

## What This Changes

The issue requests account recovery or record purge so the reporter can re-register and sign in with the same GitHub account after deleting a previous ClawHub account.

## Best Possible Solution

Maintainers should decide the account lifecycle policy, then either add an audited recovery/re-registration path that safely restores or unlinks the GitHub identity binding with tests and docs, or keep deletion irreversible while surfacing the terminal state and support route clearly in web and CLI login flows.

## Risks / Open Questions

- Some related OAuth-loop reports may have browser/session causes separate from deletion-specific account state, so maintainers should cluster carefully before declaring one universal fix.
