## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub says issues should not be auto-closed, and current main still does not implement the requested purge or same-GitHub re-registration path after account deletion. The code intentionally treats user-initiated deletion as irreversible, blocks OAuth completion for deactivated users, and only surfaces a clearer deleted-account error.

## Best Possible Solution

Keep this issue open for maintainer triage of the account-deletion and GitHub OAuth lifecycle. Maintainers should decide whether self-deleted accounts are permanently blocked or whether same-GitHub re-registration should be supported. If recovery is intended, add a safe backend/admin support path to unlink or supersede the old Convex Auth binding, cover it with auth/account lifecycle tests, and provide a repair process for already-stuck users. If irreversible deletion is intended, make the hosted login/support flow clearly surface that state and document the support policy.
