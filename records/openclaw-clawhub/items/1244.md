## Summary

Keep this issue open. It is an open ClawHub issue, and the cleanup policy only permits auto-close for pull requests that are certainly implemented on current main. Current main also still treats user-initiated account deletion as irreversible: deletion writes deactivated/purged state, OAuth completion rejects deleted or deactivated users, active-user helpers exclude them, tests assert blocked reauthentication, and docs/UI warn that deletion cannot be undone. The reporter's requested purge/re-registration support remains a maintainer account-lifecycle/support decision.

## Best Possible Solution

Keep this open for maintainer triage of the deleted-account and GitHub OAuth lifecycle. Maintainers should decide whether self-deleted accounts are permanently blocked or whether same-GitHub re-registration/recovery is supported. If recovery is intended, add a safe backend/admin repair path to unlink or supersede the old Convex Auth binding, cover it with auth/account lifecycle tests, and document a support process for already-stuck users. If irreversible deletion is intended, make the hosted login/support flow clearly surface that state and document where affected users should request manual help.
