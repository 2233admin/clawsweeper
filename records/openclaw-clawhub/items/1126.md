## Summary

Keep open. The ClawHub cleanup policy for this review keeps issues open, and the remaining user-visible problem is still real: current main and production now cover the exact slug query for `openclaw-checkpoint`, but the keyword query `checkpoint` can still miss that skill. The broader direct name/slug prefix recall gap is already tracked by open #1254.

## Best Possible Solution

Keep this issue open under the ClawHub issue policy and treat it as a concrete reproduction for open #1254. The best code path is to add an indexed direct lexical candidate layer for normalized slug/display-name prefix or token matches, merge those candidates ahead of vector/recent fallback ranking, preserve exact slug behavior and all filters, and add regression coverage for both `openclaw-checkpoint` and `checkpoint`.
