## Repair Work Prompt

Fix the remaining ClawHub skill search recall gap tracked by #1254 and reproduced here with `openclaw-checkpoint`: exact slug search already works, but keyword/direct slug-token queries such as `checkpoint` can miss older skills when vector recall and bounded recent fallback do not include them. Add an indexed direct lexical candidate layer for skills using normalized slug/display-name exact, token, and prefix fields on `skillSearchDigest`; update write paths and add a cursor-based backfill for existing rows before relying on the fields in reads. Merge direct lexical candidates into `searchSkills` ahead of vector/recent fallback ranking while preserving exact slug injection, score ordering, and `nonSuspiciousOnly`, `highlightedOnly`, and `capabilityTag` filters. Do not widen fallback scan limits or introduce table scans. Add regression tests for `openclaw-checkpoint`/`checkpoint`-style slug token recall and a Midscene-style direct name/slug prefix miss. Update docs/changelog only if the repo convention requires it for user-visible search behavior.
