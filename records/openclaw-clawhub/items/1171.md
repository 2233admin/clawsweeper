## Summary

Keep this issue open. ClawHub cleanup policy only permits auto-closing PRs that are certainly implemented on current main, and this is an open issue. Substantively, current main still leaves a real Convex pagination failure path: owner-publisher package and skill digest sync are now scheduled separately, but each scheduled internal mutation can still call `.paginate()` repeatedly in one invocation for multi-page owners.

## Best Possible Solution

Keep the issue open and land one focused backend fix in `convex/functions.ts`: refactor owner-publisher package and skill digest rebuilds into cursor-batched internal mutations where each Convex function invocation performs at most one `.paginate()` call, processes that page, and schedules itself with the continuation cursor until complete. Add regression tests for multi-page package and skill owner-publisher sync, then verify publish/import paths and consolidate sibling reports after the fix ships.
