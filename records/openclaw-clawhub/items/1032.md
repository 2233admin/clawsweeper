## Summary

This issue requests promoting the already-published EcoCompute skill into a recommended or built-in OpenClaw capability. Current main supports community skill publishing, install, curation badges, and package/plugin catalog surfaces, while the vision explicitly says new skills should usually live on ClawHub rather than core. Because this is a ClawHub issue, repository policy says to keep issues open; the remaining decision is maintainer product/curation judgment, not an auto-closeable implementation on main.

## Best Possible Solution

Keep this issue open for maintainer triage. The practical path is to leave EcoCompute available as the published ClawHub skill, and if maintainers want to promote it, use or extend existing ClawHub curation surfaces such as Highlighted/Official badges or package/channel filtering. Any true built-in OpenClaw runtime monitor should be handled as a separate OpenClaw core product/API decision with a clear security and maintenance rationale.
