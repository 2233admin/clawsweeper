## Summary

Keep this issue open. The ClawHub cleanup policy only allows automated closure for pull requests that are certainly implemented on main, and this item is an open issue. Current main also does not prove the reported Unauthorized-after-login/token path is fixed: the CLI still defaults and discovers `https://clawhub.ai`, login still verifies `/api/v1/whoami` before writing config, and related auth reports remain open while the `www.clawhub.ai` workaround PR was closed unmerged.

## Best Possible Solution

Keep this issue open for maintainer follow-up alongside #72, #100, #115, and #702. The next useful step is a current smoke test of `clawhub login`, `clawhub login --token`, and `clawhub whoami` against `https://clawhub.ai` using a real token. If it still fails, add regression coverage around discovery, callback registry propagation, Authorization behavior across redirects, token lookup/user state, and clearer 401 error reporting; then close duplicates only after a confirmed fix lands.
