## Summary

Keep this issue open. The ClawHub cleanup policy only allows automated closure for pull requests that are certainly implemented on main, and this item is an open issue. The current source also does not prove the reported Unauthorized-after-login path is fixed: defaults and discovery still point at https://clawhub.ai, login still verifies through /api/v1/whoami before writing config, and related Unauthorized reports remain open while the www workaround PR was closed unmerged.

## Best Possible Solution

Keep this issue open for maintainer follow-up alongside #72, #100, #115, and #702. The most useful next step is a current smoke test of `clawhub login`, `clawhub login --token`, and `clawhub whoami` against https://clawhub.ai and, if it still fails, regression coverage around discovery, callback registry propagation, Authorization behavior across redirects, token verification, and clearer error reporting. The existing www workaround comment should remain visible as useful operational context.
