## Summary

Keep open. The reported CLI discovery ambiguity is still present on current main: public skill search and `clawhub search` omit owner identity, while install still accepts only bare slugs. The web skill-detail prompt path partially helps by showing owner-qualified targets, but it does not resolve the reporter's search/install confusion.

## What This Changes

The issue asks for clearer owner-aware skill identifiers in ClawHub search and install flows, such as owner-qualified names, a username column, or clearer install guidance.

## Best Possible Solution

Expose owner identity additively in the public v1 skill search response, update the shared and CLI-local schemas, render owner information in `clawhub search`, and document the response/output change. Treat full owner-qualified install syntax as a separate maintainer product/security decision unless maintainers explicitly expand the scope.
