## Summary

Keep this issue open. The web skill page now partially addresses owner/slug clarity for guided OpenClaw prompts, but current main still leaves the reported CLI/API ambiguity unresolved: search results are ownerless, ClawHub install remains slug-only, public v1 search responses omit owner identity, and downloads are still named slug-version.

## Best Possible Solution

Keep this issue open and implement a scoped identity clarification: expose owner handle/id in /api/v1/search and the CLI search schema, display owner or owner/slug in clawhub search, decide whether clawhub install should accept @owner/slug or owner/slug while preserving slug-only compatibility, then align download filenames, docs, and skill-page commands with that decision.
