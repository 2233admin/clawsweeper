## Summary

Keep open. Current main has partial web-detail mitigation for owner-qualified skill pages and prompts, but the reported ambiguity remains in the ClawHub CLI/API path: CLI search still prints only slug/name/version/score, the public search response omits owner data, install still accepts only bare slugs, and download filenames are still slug-version only. Repository policy for this review also says ClawHub issues should stay open unless a maintainer decides otherwise.

## Best Possible Solution

Keep this issue open and implement a scoped identity clarification: expose ownerHandle/owner in `/api/v1/search`, update the CLI search schema and output to show `owner/slug`, decide whether `clawhub install owner/slug` should resolve to a slug while preserving slug-only installs, and consider adding owner to download filenames or making the web install surface emphasize the exact canonical skill page.

## Risks / Open Questions

- Supporting owner-qualified install identifiers affects filesystem paths, lockfile keys, update/uninstall behavior, and backwards compatibility because the current CLI explicitly treats slashes as invalid slug input.
- Adding owner fields to search output requires a coordinated API/schema/CLI change so older clients continue to parse responses correctly.
