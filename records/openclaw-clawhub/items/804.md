## Summary

Keep open. Current main still leaves the reported owner/slug ambiguity unresolved for CLI search and install: the CLI search output is ownerless, the v1 search response and both CLI/schema packages omit owner identity, and install still only accepts bare slugs. The web skill page has a partial owner-qualified prompt surface, but the actual CLI command path remains bare-slug only.

## Best Possible Solution

Keep this issue open and land a narrow owner-disambiguation PR: expose owner identity in the v1 search response, update both search response schemas, print owner or owner/slug in clawhub search, and refresh CLI/API docs and tests. Treat owner-qualified install locators as a separate product/security design unless maintainers explicitly scope it into the same work.
