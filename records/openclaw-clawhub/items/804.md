## Summary

Keep this issue open. ClawHub cleanup policy says open issues should not be auto-closed, and current main still only partially addresses the reported owner/slug confusion: skill pages know about owner-qualified identity, but CLI search output, the public search response/schema, docs, and `clawhub install` remain slug-oriented.

## Best Possible Solution

Keep this issue open and implement a scoped identity clarification: expose owner handle/id in `/api/v1/search`, update CLI schemas and search output to show owner/slug or an owner column, decide whether `clawhub install` should accept owner-qualified targets while preserving slug-only compatibility, then align download naming and docs with that decision.
