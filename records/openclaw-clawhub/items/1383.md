## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy only permits auto-closing pull requests that are certainly implemented on current main. Current main also still lacks the requested reliable registry discovery path: CLI search is q/limit ranked search, CLI list is local-lockfile only, and the public skill list/search APIs do not expose exact-slug or slug-prefix enumeration filters.

## Best Possible Solution

Keep this issue open. The best implementation would add an indexed skill-registry discovery path for exact slug and slug-prefix queries, expose it through CLI flags such as `search --exact`, `search --prefix`, or a registry-backed `skills list --source hub --cursor --json` mode, and update API/CLI tests plus public docs.

## Risks / Open Questions

- Closing this would drop a still-missing CLI workflow for reliable skill namespace or family discovery.
- Exact-slug injection and token prefix boosts can improve ranked search recall, but they can be mistaken for full enumeration even though results remain bounded and score-sorted.
- A future fix should avoid client-side full scans and use an indexed backend exact/prefix path with cursor pagination to fit the repository's Convex bandwidth rules.
