## Summary

Keep open. This is an open ClawHub issue, which the cleanup policy does not auto-close, and current main still lacks a deterministic CLI/API path for exact-slug or slug-prefix registry enumeration. Existing search has exact/prefix boosts, but it remains relevance-ranked and limit-sliced rather than a complete listing mechanism.

## What This Changes

Request to add registry-oriented ClawHub CLI discovery so users can reliably enumerate public hub skills by exact slug, slug prefix, source/list mode, or pagination instead of relying only on ranked search.

## Best Possible Solution

Keep this issue open and implement a narrow indexed discovery path for public hub skills: exact slug lookup can reuse or wrap the existing detail endpoint, while slug-prefix enumeration should use a Convex indexed/cursor-paginated query exposed through conservative CLI flags or a hub-list mode with JSON support. Preserve ranked `search <query...>` behavior, and update CLI/API docs plus focused tests.
