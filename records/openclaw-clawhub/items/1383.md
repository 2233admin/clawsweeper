## Summary

Keep open. This is an open ClawHub issue, which the repository cleanup policy does not auto-close, and current main still lacks a deterministic CLI/API path for exact-slug or slug-prefix registry enumeration. The existing paths are ranked search, bounded recent/list browsing, and exact detail lookup, not prefix/list-style discovery.

## Best Possible Solution

Keep this issue open and implement a narrow registry-discovery path: exact slug lookup and indexed, cursor-paginated slug-prefix enumeration for public hub skills, exposed through conservative CLI flags or a hub-list mode with JSON support. Preserve ranked `search <query...>` behavior and update API/CLI docs plus focused tests.

## Clownfish Work Prompt

Implement deterministic skill registry discovery for this issue without changing ranked `clawhub search <query...>` semantics. Add a narrow public hub-skill exact/prefix path: exact slug lookup can reuse existing detail semantics where appropriate, but slug-prefix discovery must be indexed and cursor-paginated, not a client-side full scan. Prefer adding an active slug index on `skillSearchDigest` or an equivalent lightweight public projection, then expose it through conservative CLI flags such as `clawhub search --exact <slug>` and `clawhub search --prefix <prefix>` or a clearly named hub-list mode with `--json`, `--limit`, and cursor support. Update API/CLI docs and schemas as needed. Coordinate with related PR #1256 conceptually, but do not replace ranked search with prefix enumeration and do not broaden this into package/plugin discovery beyond public skills unless the implementation naturally shares existing public package catalog helpers. Preserve safety filters and existing search/explore output behavior.
