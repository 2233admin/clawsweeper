## Summary

Keep open. The ClawHub cleanup policy for this pass keeps issues open, and the current evidence still points to live marketplace/security moderation rather than an implemented repository fix: the public skill is still flagged suspicious, its latest bundle still references a parent-path file and leaves wallet signing underspecified, and current main intentionally preserves corroborated scanner-derived suspicious state while offering rescan and staff override paths.

## What This Changes

The issue asks maintainers to review why the live `buy-handshake-domain` skill is flagged suspicious when it registers Handshake/Impervious domains through Ethereum.

## Best Possible Solution

The publisher should republish a corrected latest version with the complete registration workflow, ABIs, contract addresses, and safety constraints inside the bundle, remove the parent-path `SKILL.md` reference, and document a user-side signing flow that never asks the agent for private keys or seed phrases. After that, the owner should request a rescan and staff can decide whether a clean manual override is appropriate from refreshed scanner evidence.
