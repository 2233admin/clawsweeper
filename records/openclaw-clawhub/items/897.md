## Summary

Current main still does not intentionally support `.ps1` skill files: the shared and CLI-mirrored text-file allowlists exclude PowerShell, and the browser upload, Convex publish validation, GitHub import/archive handling, and CLI discovery paths all depend on those allowlists. Because this is an open ClawHub issue and accepting a new executable script format needs maintainer product/security review, it should stay open and be triaged with open related issue #290.

## What This Changes

Request to allow PowerShell `.ps1` files as uploaded and published skill files rather than requiring users to embed the script body in `SKILL.md`.

## Best Possible Solution

Keep this issue open and triage it with #290. If maintainers approve PowerShell as an accepted text helper format, add `ps1` consistently to the shared and CLI-mirrored text-file allowlists/content-type normalization, update docs and tests, and verify browser upload, Convex publish validation, GitHub import, static scan/read paths, archive/drop handling, and CLI discovery/publish all agree.
