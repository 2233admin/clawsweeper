## Summary

Keep open. Current main still does not intentionally support `.ps1` skill files, the request has concrete user impact and confirmations, and related #290 remains open. Because this is a ClawHub issue and the requested change expands the published skill-file surface to another executable script format, it needs maintainer product/security triage before an allowlist change.

## What This Changes

The issue requests support for uploading `.ps1` PowerShell files as skill files instead of requiring authors to embed PowerShell code in `SKILL.md`.

## Best Possible Solution

If maintainers approve PowerShell support, add `ps1` consistently to the shared schema allowlist and CLI-mirrored allowlist, normalize it to an appropriate text content type, update focused tests and docs, and verify browser upload, archive/drop expansion, Convex publish, GitHub import, raw file serving, and CLI publish/discovery behavior together. If maintainers reject it, document the intentional exclusion and recommended workaround.
