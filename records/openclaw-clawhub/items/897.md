## Summary

Keep this issue open. Repository policy for openclaw/clawhub says issues should not be auto-closed, and current main still does not clearly support .ps1 uploads across the web, server, archive, and CLI publish paths.

## Best Possible Solution

Keep this issue open and implement explicit PowerShell script support if maintainers accept the policy tradeoff: add ps1 to the shared schema allowlist and CLI schema copy, normalize/guess an appropriate text content type, update web/archive/server/CLI tests, and update the skill-format docs so all publish paths consistently accept .ps1 files.
