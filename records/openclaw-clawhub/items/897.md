## Summary

Current main still does not intentionally accept `.ps1` as a text-based skill file extension. The shared schema allowlist, browser upload validation, server publish validation, CLI file discovery, and GitHub import path all remain extension-allowlist based and exclude PowerShell. Because this is an issue in `openclaw/clawhub`, repository policy says to keep it open; it also overlaps with the older open sibling #290 and needs maintainer product/security judgment before accepting another executable script format.

## Best Possible Solution

Keep this issue open and handle it together with #290. If maintainers approve PowerShell as an accepted text-based skill helper format, add `ps1` to the shared and CLI-mirrored text-file allowlists, normalize/guess it as text, update tests and docs, and verify web upload, Convex publish validation, CLI discovery/publish, and GitHub import behave consistently.
