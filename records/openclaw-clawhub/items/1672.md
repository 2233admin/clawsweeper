## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy only permits auto-closing pull requests that are certainly implemented on current main. Current main also still contains the reported scanner surfaces: the curl-pipe hits are detector/tests/docs examples rather than runtime execution sites, while the package-publish workflow is real security-sensitive publishing infrastructure that should get explicit maintainer triage.

## Best Possible Solution

Keep this issue open for maintainer security triage. Mark the Convex detector tests, comment-scam fixture, moderation rule, and acceptable-usage examples as intentional scanner fixtures/examples where appropriate, then separately review and harden .github/workflows/package-publish.yml and the CLI docs around OIDC claim handling, GITHUB_OUTPUT-safe writes, temporary credential config handling, generated shell execution, and reusable-workflow pinning guidance.
