## Summary

Keep this issue open. ClawHub cleanup policy does not auto-close issues, and current main only partially mitigates the reported failure: package and skill digest syncs are now scheduled separately, but each owner-publisher digest helper can still run multiple `.paginate()` calls inside one internal mutation for larger owner catalogs. The related publish-failure reports and the closed, unmerged fix PR show this is still a concrete backend cleanup cluster.

## What This Changes

The issue reports first-time publisher CLI publish and settings/profile failures caused by `ensurePersonalPublisherForUser` triggering Convex owner-publisher digest syncs that violate Convex's single-pagination-per-function limit.

## Best Possible Solution

Update the owner-publisher digest rebuilds so each internal mutation performs at most one indexed paginated read, carries the continuation cursor through validated args, self-schedules follow-up work until complete, adds focused regression coverage, deploys the Convex backend, then consolidates the duplicate publish/profile reports.

Preserve the existing package/skill split from `scheduleOwnerPublisherDigestSync`. Do not replace the pagination with unbounded `.collect()`, do not change CLI publish request/response semantics, and avoid unrelated publish, scanner, or UI refactors. Add regression coverage in `convex/functions.test.ts` proving multi-page package and skill digest rebuilds schedule continuations and do not issue a second paginated query in the same mutation. Regenerate Convex generated API files if internal function args change.
