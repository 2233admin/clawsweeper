Keep open. The ClawHub review policy keeps issues open, and the discussion escalates this from slug-collision UX into security-sensitive registry abuse and production cleanup. Current main has meaningful mitigations for exact slug collisions, duplicate/fork metadata, suspicious-skill filtering, and malicious install-payload detection, but it does not prove the named production clones/accounts were remediated or that canonical-only discovery policy is complete.

## What This Changes

The issue asks ClawHub to stop duplicate skill slugs with random-looking suffixes from polluting registry discovery and to improve collision, canonical, fork, or cleanup behavior.

## Best Possible Solution

Keep this issue open for maintainer-led registry/security cleanup: verify the named slug families and uploader accounts in production, quarantine or delete malicious clones, reconcile GitHub-import suffix guidance with the actual publish collision policy, and decide whether default browse/search should suppress duplicate/fork entries or only suspicious ones.
