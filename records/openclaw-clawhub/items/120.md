## Summary

Keep open. This is an issue in openclaw/clawhub, and the supplied repository policy only allows auto-closing pull requests that are certainly implemented on main. Current main has meaningful mitigations for exact slug collisions and the malicious-install-payload concern raised in discussion, but GitHub import still intentionally suggests suffixed duplicate-style slugs, docs still describe that behavior, and source review cannot prove production registry cleanup or moderation is complete.

## Best Possible Solution

Keep this issue open for maintainer/security triage. Preserve the shipped exact-slug collision rejection and moderation safeguards, then explicitly resolve or split the remaining work: production duplicate/malware cleanup, whether GitHub import should stop suggesting suffixed slugs or mark forks differently, and whether owner-scoped slug uniqueness needs a schema/API migration.
