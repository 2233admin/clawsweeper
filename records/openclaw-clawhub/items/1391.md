## Summary

Keep open. ClawHub cleanup policy keeps issues open in this lane, and current main still only partially handles the reported artifact false positives: package and CLI paths ignore some repository artifacts, but the skill web upload, multipart API, and backend publish scan paths still rely on macOS-junk filtering before storage and scanning.

## What This Changes

The issue requests that ClawHub stop flagging or requiring manual cleanup for standard Git and Python generated artifacts such as `.git/`, `__pycache__/`, `.pyc`, and `.pyo` when publishing skills.

## Best Possible Solution

Add a small shared publish-artifact cleanup helper used before persistence and static/LLM moderation across skill publish entry points, covering only well-known generated/VCS artifacts such as `.git/`, `__pycache__/`, `.pyc`, and `.pyo` while preserving normal source-file scanning and existing macOS-junk handling.
