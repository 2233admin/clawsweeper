## Summary

Keep open. The repository-specific cleanup policy keeps ClawHub issues open, and current main still lacks a shared artifact exclusion policy for `.git/`, `__pycache__/`, `*.pyc`, and `*.pyo` across skill/package publish and scan paths. There is partial client-side `.git/` filtering, but Python cache/bytecode artifacts are not covered and server-side normalization does not enforce the same exclusions.

## Best Possible Solution

Keep this issue open. Implement a shared artifact exclusion helper used consistently by CLI skill/package collection, web folder/archive expansion, HTTP/API publish normalization, server-side release persistence, static scan input construction, and VirusTotal ZIP construction. Exclude `.git/`, `__pycache__/`, `*.pyc`, and `*.pyo`; preserve the existing `.clawhubignore` behavior and documented skill `.gitignore` support; add focused tests for CLI, web, API, static scan, and VT payload paths; document the interim ignore-file workaround where applicable.
