## Summary

Keep open. The repository policy for this cleanup pass keeps ClawHub issues open, and current main still does not provide a shared artifact exclusion policy for `.git/`, `__pycache__/`, `*.pyc`, and `*.pyo` across skill/package upload, API publish normalization, persistence, static scanning, and VirusTotal payload construction.

## Best Possible Solution

Keep this issue open. Implement a shared publish artifact exclusion helper used by CLI skill/package collection, web folder/archive expansion, HTTP multipart and JSON publish normalization, server-side release persistence, static scan input construction, and VirusTotal ZIP construction. Exclude `.git/`, `__pycache__/`, `*.pyc`, and `*.pyo`; preserve `.clawhubignore` behavior; add focused tests for CLI, web, API, multipart, JSON, static-scan, and VirusTotal payload paths; document the interim `.clawhubignore` or `.gitignore` workaround where applicable.
