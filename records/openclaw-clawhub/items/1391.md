Implement a shared publish-artifact cleanup helper for ClawHub skill publishing. Apply it consistently to web skill upload expansion, API JSON/multipart skill publish, GitHub skill import, CLI skill publish, and package publish paths where the same artifact cleanup is applicable. Keep the filter narrow; do not suppress scan findings after scanning, do not broaden to arbitrary dotfiles beyond existing behavior, and do not change scanner heuristics for real source files. Add focused tests for each affected path and update user-facing docs only if the cleanup contract is exposed.
