## Summary

Keep this issue open. ClawHub cleanup policy for this review only permits auto-closing PRs that are certainly implemented on current main, and this item is an issue. Source review also shows the named owner-publisher skill digest helper still loops over `.paginate()` inside one internal mutation, so the reported multi-page Convex failure is not conclusively fixed. Related #1182 remains the best consolidation target if maintainers want one.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it under #1182. The robust fix should make owner-publisher package and skill digest sync cursor-batched: each internal mutation should accept a cursor, call `.paginate()` at most once, process that page, and schedule the next invocation with `continueCursor`. Add regression coverage for an `isDone: false` skill page.
