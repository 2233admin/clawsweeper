## Summary

Keep this issue open. Current main still contains the reported Convex failure shape: the owner-publisher skill digest sync path can call `.paginate()` repeatedly inside a single internal mutation, and the sibling package digest helper has the same pattern. The scheduler split reduces inline publish impact but does not pass continuation cursors or guarantee one paginated query per function invocation. ClawHub cleanup policy also keeps open issues open in this pass.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it under #1182. The code fix should make owner-publisher package and skill digest sync cursor-batched: each internal mutation should accept an optional cursor, perform at most one `.paginate()` call, process that page, and schedule itself with `continueCursor` when `isDone` is false. Add focused tests for both skill and package multi-page paths.
