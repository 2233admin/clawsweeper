## Summary

Keep this issue open. Current main appears to have reduced one likely trigger by scheduling package and skill digest syncs as separate background mutations, but the specific `syncSkillSearchDigestsForOwnerPublisherId` path still performs cursor pagination in a loop inside one mutation, so the reported Convex limitation is not conclusively fixed. Also, the ClawHub cleanup policy says issues should stay open unless maintainers explicitly handle them.

## Best Possible Solution

Keep the issue open or consolidate it under the open canonical report #1182. The robust code path should make owner digest sync cursor-batched: each internal mutation should perform at most one `.paginate()` call, accept a cursor argument, process that page, and reschedule itself with `continueCursor` until done. Add regression tests that simulate `isDone: false` and verify the next page is scheduled rather than fetched in the same mutation.
