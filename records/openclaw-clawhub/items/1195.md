## Summary

Keep this issue open. The ClawHub cleanup policy for this review does not auto-close issues, and current main still has the named skill digest sync helper structured as a same-invocation pagination loop. The existing background scheduling split is a useful mitigation for the original publisher-triggered publish failure, and related #1182 remains the likely canonical consolidation target, but the cursor-batched fix is not present on main.

## Best Possible Solution

Keep this issue open or have maintainers consolidate it under #1182. The robust code path should make owner-publisher package and skill digest sync cursor-batched: each internal mutation should accept a cursor, call .paginate() at most once, process that page, and schedule the next invocation with continueCursor. Add regression coverage for an isDone: false skill page.
