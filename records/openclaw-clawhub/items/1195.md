## Summary

Keep open. This is a ClawHub issue, and the cleanup policy keeps issues open unless maintainers consolidate or a fix lands; current main still has the reported Convex limit shape because both owner-publisher digest helpers can call `.paginate()` repeatedly inside one internal mutation, while the scheduled internal mutations do not accept continuation cursors. The related reports around #1182, #1171, and #1197 are still open, so this should be fixed or explicitly consolidated rather than auto-closed.

## What This Changes

The issue reports that `clawhub publish` fails with Convex's multiple-paginated-queries error in `syncSkillSearchDigestsForOwnerPublisherId` during personal publisher setup.

## Best Possible Solution

Land one canonical backend fix that makes owner-publisher package and skill digest rebuilds cursor-rescheduled: each invocation should process one indexed page, update digests for that page, and schedule the next page with `continueCursor` until complete, with tests proving both skill and package multi-page paths.

## Repair Work Prompt

Fix the owner-publisher digest rebuild pagination bug behind the publish/import Convex `multiple paginated queries` failure. In `convex/functions.ts`, change the package and skill owner-publisher digest sync path so each internal mutation invocation performs at most one `.paginate()` call, processes that page, and when `isDone` is false schedules itself with `continueCursor`. Preserve the indexed `by_owner_publisher` reads, existing digest upsert behavior, missing-table handling, and separate package/skill jobs. Do not replace pagination with `.collect()` or other unbounded reads, and do not change CLI publish behavior, auth, ownership, scanner, or moderation logic. Add focused tests in `convex/functions.test.ts` for both package and skill multi-page paths: first page returns `isDone: false` and schedules a continuation with the cursor, a continuation invocation uses the passed cursor, and each invocation calls `.paginate()` only once. Keep the existing scheduler-missing behavior covered. If Convex generated API types change because internal mutation args change, regenerate only the committed Convex generated files needed by the repo.
