## Summary

Keep this issue open. Current main still does not provide a GitHub-style device-code or CLI/web pairing flow for first-time headless authentication, and the ClawHub cleanup policy says issues should not be auto-closed under this review path.

## Best Possible Solution

Keep this issue open for maintainer prioritization. The likely implementation belongs in first-party ClawHub CLI plus web/Convex auth code: add a device-code or CLI/web pairing flow that prints a verification URL and user code, lets an authenticated browser approve it, has the CLI poll with backoff and expiry/denial handling, then stores a scoped ClawHub API token. Cover CLI behavior, web routes, Convex state, abuse/rate-limit handling, docs, and tests for success, expiry, denial, and polling edge cases.
