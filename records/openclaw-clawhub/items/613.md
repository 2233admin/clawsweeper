## Summary

Keep this issue open. Current main still only supports loopback browser login and manual token paste for headless use, and the related implementation PR is open and unmerged with closing syntax for this issue.

## Best Possible Solution

Keep the issue open and review PR #1867 as the implementation candidate. A complete solution should add both CLI support for clawhub login --device/auth login --device and the server-side device-code endpoints plus verification UI, with tests covering polling, expiry, denial, slow_down/backoff, and token storage.
