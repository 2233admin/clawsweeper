## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Current main also still preserves the reported behavior: self-deleted accounts are blocked from GitHub re-sign-in/recreation, and publishing can still surface the exact GitHub API rate-limit error from the account-age gate.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to decide the intended product policy for self-deleted GitHub-linked accounts, then either document the irreversible/manual-support path more clearly or implement controlled account recreation/relinking. Separately, improve the publish flow around GitHub 403/429 account-age lookups with clearer retry guidance or backend configuration checks for GITHUB_TOKEN.
