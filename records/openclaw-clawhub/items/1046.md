## Summary

Keep open. ClawHub cleanup policy keeps open issues open, and current main still matches both reported surfaces: deleted/deactivated GitHub-linked accounts are rejected on reauth, and publish still raises the reported GitHub 403/429 rate-limit error through the shared account-age gate.

## What This Changes

The issue requests clearer or recoverable handling for GitHub API rate-limit failures during skill publishing and a clean sign-in/recreation path after deleting a GitHub-linked ClawHub account.

## Best Possible Solution

Keep open for maintainer triage. Maintainers should decide whether self-deleted GitHub-linked accounts remain intentionally irreversible or need a controlled recreation, relink, or admin recovery path with audit and abuse protections, then align UI/docs with that policy. Separately, improve publish recovery around GitHub 403/429 account-age lookups with clearer retry guidance, Retry-After/backoff handling, deployment checks for GITHUB_TOKEN, or a narrower canonical follow-up for publish-rate-limit recovery.
