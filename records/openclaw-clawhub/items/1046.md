## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this cleanup pass keeps ClawHub issues open. Substantively, current main still has the reported surfaces: deleted/deactivated GitHub-linked accounts are blocked from reauth, and skill publishing still uses the GitHub account-age lookup that throws the reported rate-limit error on GitHub 403/429 responses. Related account/publish-access reports #504, #886, and #1674 are also still open.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether self-deleted GitHub-linked accounts are intentionally irreversible or should have a controlled recreation/relink/admin recovery path with audit and abuse protections, then align UI and docs with that policy. Separately, improve the publish flow around GitHub 403/429 account-age lookups with clearer retry guidance, backoff, deployment checks for GITHUB_TOKEN, or a narrower canonical follow-up for publish-rate-limit recovery.
