## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup rule only allows auto-closing pull requests that are certainly implemented on current main. The current source also still preserves both reported behaviors: deleted accounts are blocked from re-authentication/recreation, and publish can still fail with the exact GitHub API rate-limit message during the GitHub account-age gate.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to decide the intended product policy for self-deleted GitHub-linked accounts, then either document the irreversible/manual-support path more clearly or implement controlled account recreation/relinking. Separately, improve the publish flow around GitHub 403/429 account-age lookups with clearer retry guidance, backoff, or backend configuration checks for GITHUB_TOKEN.
