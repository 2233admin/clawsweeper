## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main still matches both reported surfaces: self-deleted/deactivated GitHub-linked accounts are rejected on reauth, and skill publish still surfaces the GitHub 403/429 account-age lookup as the reported rate-limit error.

## What This Changes

The issue reports a failed skill publish after a GitHub API rate-limit error and inability to sign back in after deleting a ClawHub account.

## Best Possible Solution

Maintainers should make an explicit product/security decision for self-deleted GitHub-linked accounts, then either keep the irreversible policy and improve user-facing/admin recovery messaging or add a controlled recreation/relink path with audit and abuse protections. Separately, publish should give clearer 403/429 recovery guidance around the GitHub account-age lookup, including Retry-After/backoff handling or deployment checks for GITHUB_TOKEN.
