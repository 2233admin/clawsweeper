## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy says to keep ClawHub issues open rather than auto-close them. Substantively, current main still does not provide the requested recovery path: user-initiated deletion is treated as irreversible, reauth is blocked for deactivated/deleted users, and the publish GitHub account-age gate still surfaces the reported rate-limit error on GitHub 403/429. Related account-lock/sign-in reports #504 and #1674 remain open in the provided context, so maintainer/product triage is still needed.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely path is to split or explicitly resolve the two concerns: add a clearer/retryable recovery path for GitHub API rate-limit failures in the publish flow, and decide whether deleted GitHub-linked accounts should remain permanently blocked or support a controlled recreate/relink/manual restore process. If the latter remains intentionally irreversible, the UI and support path should make that explicit and point affected users to account review where appropriate.
