## Summary

Keep open. The ClawHub cleanup policy keeps open issues open, and current main still matches both reported surfaces: self-deleted/deactivated GitHub-linked accounts are rejected on reauth, and publish still raises the GitHub 403/429 rate-limit error through the shared GitHub account-age gate. The related open reports #504 and #886 cover the deleted-account login loop, while #1674 overlaps broader account-blocking recovery.

## What This Changes

The issue requests clearer recovery for skill publishing when GitHub API rate limits occur and a way for users who deleted their ClawHub account to sign in, recreate, or recover it through GitHub auth.

## Best Possible Solution

Maintainers should make an explicit product/security decision for self-deleted GitHub-linked accounts, then either keep the irreversible policy and improve user-facing messaging or add a controlled recreation/relink/admin recovery path with audit and abuse protections. Separately, publish should give clearer 403/429 recovery guidance around the GitHub account-age lookup, including Retry-After/backoff behavior or deployment checks for GITHUB_TOKEN.
