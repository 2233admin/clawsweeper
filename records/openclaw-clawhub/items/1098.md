## Summary

Keep this issue open. Current main fixes the normal default `clawhub explore` path by using the canonical `https://clawhub.ai` registry and API, and the live canonical endpoint is returning skill data. However, this is a ClawHub issue, and the repository policy says issues should remain open for maintainer triage. There is also a real residual edge: explicit or cached `registry.clawhub.ai` is still not classified as a legacy/dead registry host, and that live host still returns Vercel `DEPLOYMENT_NOT_FOUND`.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely code path is to add `registry.clawhub.ai` to the CLI legacy registry migration logic, add a regression test for that host, and document how users with explicit or cached stale registry config should recover. Alternatively, maintainers can explicitly decide that the dead host is unsupported when configured manually and close this after confirming production canonical API behavior.
