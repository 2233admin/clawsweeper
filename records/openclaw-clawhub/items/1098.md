## Summary

Keep open. Current main defaults discovery and CLI requests to https://clawhub.ai and the canonical live API is returning skills, but the reported legacy host still returns Vercel DEPLOYMENT_NOT_FOUND and can still be used through cached or explicit registry configuration. ClawHub review policy also keeps issues open for maintainer follow-up.

## What This Changes

The issue reports that `clawhub explore` stopped working because it used `registry.clawhub.ai`, which returns Vercel DEPLOYMENT_NOT_FOUND, while `clawhub search` reportedly still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, and consider a targeted CLI compatibility change that treats cached `registry.clawhub.ai` values as legacy while preserving intentional custom registries.
