## Summary

Keep open under the ClawHub review policy. Current main and live production no longer show the primary default-path failure: `clawhub explore` is documented and implemented against `https://clawhub.ai/api/v1/skills`, and the live endpoint returned a skill item. However this is an issue, not a PR, and the policy for openclaw/clawhub says to keep issues open. There is also a plausible remaining stale-cache/explicit-registry edge because `registry.clawhub.ai` still fails and is not listed in the current legacy registry host set.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely product/code path is to either document that users should upgrade/use the default `https://clawhub.ai` registry, or add `registry.clawhub.ai` to the stale legacy registry migration/refresh path so cached configs are rewritten even when discovery later fails. Once maintainers confirm the production outage and stale-registry behavior are resolved, they can close the issue manually under the ClawHub policy.
