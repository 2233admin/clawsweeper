## Summary

Keep this issue open. Current main no longer uses the broken legacy host for the default CLI path and the canonical API is returning skill results, but the reported `registry.clawhub.ai` host still returns Vercel `DEPLOYMENT_NOT_FOUND`, so older or explicitly configured clients can still hit the failure.

## Best Possible Solution

Keep the issue open for maintainer follow-up. The best outcome is to restore or redirect `registry.clawhub.ai` for backward compatibility, or explicitly confirm the supported migration path to current `clawhub` defaults and provide upgrade/config cleanup guidance. If maintainers want repo-side hardening too, add `registry.clawhub.ai` to legacy registry handling with focused tests.
