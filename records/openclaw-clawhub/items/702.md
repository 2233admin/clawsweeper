## What This Changes

The issue reports that `clawhub login` succeeds in CLI v0.7.0, but `whoami`, `sync`, and `publish` subsequently return `Unauthorized`, possibly due to token storage, bearer propagation, or `clawhub.ai`/`www.clawhub.ai` redirects.

## Best Possible Solution

Verify the current released CLI against production with a fresh OAuth login and API token. If the Unauthorized path still reproduces, fix the narrow auth/publish chain across registry discovery/canonicalization, loopback token delivery, config writes, bearer preservation, backend token lookup, and user/publisher resolution with regression coverage for `whoami`, `sync`, and `publish`.
