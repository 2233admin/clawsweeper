## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-close for pull requests that are certainly implemented on main. Substantively, current main still intentionally blocks GitHub OAuth sign-in for deleted/deactivated accounts instead of restoring, unlinking, or allowing re-registration. The code now documents and surfaces permanent deletion, but that does not satisfy the reported request for account recovery or backend cleanup.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated deletion is intentionally irreversible or whether GitHub identities may be safely unlinked/re-registered. If deletion remains irreversible, the web and CLI login flows should show a durable deleted-account message and public support policy. If re-registration should be supported, add a backend path that safely restores or supersedes the old Convex Auth identity, revokes old tokens, preserves ban and ownership protections, and covers web OAuth plus CLI login with tests.
