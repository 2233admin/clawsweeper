## Summary

Keep this issue open. It is a ClawHub issue, and the repository-specific cleanup policy says only pull requests that are certainly implemented on main should be proposed for auto-close. Current main still intentionally blocks GitHub OAuth sign-in for deleted/deactivated users rather than restoring, unlinking, or allowing re-registration, so the reported account lifecycle problem remains live.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated deletion is intentionally irreversible or whether GitHub identities may be safely unlinked/re-registered. If deletion remains irreversible, the web and CLI login flows should show a durable deleted-account message and public support policy. If re-registration should be supported, add a backend path that safely supersedes or unlinks the old Convex Auth identity, revokes old tokens, preserves ban and ownership protections, and covers web OAuth plus CLI login with tests.
