## Summary

Keep this issue open. It is a ClawHub issue, and the repository-specific cleanup policy says not to auto-close ClawHub issues. Substantively, current main still treats user-initiated deletion as irreversible: deletion writes `deactivatedAt`, the OAuth callback blocks deleted/deactivated users, API-token auth rejects deactivated users, and tests/docs codify that deleted accounts cannot be restored by logging in again. The issue discussion also has multiple recent reporters plus related open issues with the same post-deletion GitHub OAuth loop, so this remains an active product/support problem rather than stale cleanup.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated deletion is truly irreversible or whether GitHub identities may be safely unlinked/re-registered. If deletion is irreversible, the web and CLI login flows should show a durable deleted-account message and public support policy. If re-registration should be supported, add a backend path that safely supersedes or unlinks the old Convex Auth identity, preserves ban and ownership protections, revokes old tokens, and covers web OAuth plus CLI login with tests.
