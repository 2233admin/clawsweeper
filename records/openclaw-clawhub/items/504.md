## Summary

Keep this issue open. Current main still intentionally rejects GitHub OAuth for deactivated/self-deleted users and documents account deletion as irreversible; it improves the visible auth error path, but it does not implement the requested restore, unlink, or clean re-registration flow. ClawHub policy also says issues should stay open unless maintainers handle them explicitly.

## What This Changes

Allow users who deleted their ClawHub account to sign in again with GitHub by recreating, restoring, or unlinking the account instead of remaining logged out after OAuth.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item. Maintainers should decide whether self-deleted accounts are permanently unrecoverable or whether ClawHub should support an audited restore, GitHub identity unlink, or clean re-registration flow; the implementation should cover web OAuth, CLI login/token creation, revoked token handling, ban preservation, and audit logging.
