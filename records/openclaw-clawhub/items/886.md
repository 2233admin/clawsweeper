## Summary

Keep this issue open. Repository policy for openclaw/clawhub says issues should not be auto-closed by this cleanup pass, and current main still does not implement the requested purge, relink, restore, or same-GitHub re-registration path after user-initiated deletion. Main now gives clearer deleted-account messaging, but the backend still treats self-deletion as irreversible.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. The concrete path is either an admin-controlled purge/relink/recovery flow for self-deleted GitHub identities, with tests around Convex Auth bindings and ban invariants, or clearer public/support documentation plus manual production cleanup for affected users if deletion is intentionally permanent.
