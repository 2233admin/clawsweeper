## Summary

Keep this issue open. Current main still treats user-initiated account deletion as irreversible: deletion writes deactivated/purged state, OAuth reauth rejects deactivated users, tests and docs assert that deleted accounts cannot sign in again, and related deleted-account OAuth loop reports remain open. The ClawHub cleanup policy also says open issues should stay open unless they are PRs implemented on main.

## Best Possible Solution

Keep this open and consolidate with the related deleted-account OAuth loop reports. Maintainers should decide whether ClawHub deletion remains irreversible or gains a verified recovery/re-registration path; if recovery is allowed, implement it explicitly in Convex Auth/account lifecycle and admin support tooling, with tests and docs covering deleted, banned, and legacy soft-deleted users.
