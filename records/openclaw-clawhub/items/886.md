## Summary

Keep this issue open. Current main still treats user-initiated account deletion as irreversible deactivation: deletion writes deactivated/purged state, OAuth completion rejects deleted/deactivated users, active-session helpers exclude those users, and docs/UI say deleted accounts cannot be restored by logging in again. The issue also has multiple matching open related reports, so this needs maintainer product/support handling rather than cleanup closure.

## Best Possible Solution

Keep this issue open and route it to maintainers for an explicit account recovery policy. The clean product direction is to decide whether deleted GitHub-linked accounts should be restorable, hard-purgeable by support, or allowed to re-register, then implement that path in the Convex Auth callback/account deletion/admin tooling with tests covering deleted, banned, and legacy `deletedAt` users. If maintainers keep deletion irreversible, the UI should surface the permanent-deletion sign-in error clearly and support should handle affected existing accounts outside automated cleanup.
