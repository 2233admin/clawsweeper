## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy says ClawHub issues should remain open rather than auto-closed. Current main still treats self-deletion as irreversible: deletion writes deactivated/purged state, OAuth completion rejects deactivated users, tests/docs assert that behavior, and the related account-deletion sign-in reports remain open.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. The concrete implementation path is either an admin-controlled purge/relink/recovery flow for self-deleted GitHub identities, with tests around Convex Auth bindings and ban invariants, or clearer public support documentation explaining that deletion is permanent and where affected users should request manual help.
