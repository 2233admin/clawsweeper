## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy says ClawHub issues should not be auto-closed. The reported account-deletion login loop also still matches current main: self-deletion marks the user as deactivated/purged, the OAuth callback blocks deactivated users from signing in again, and tests/docs explicitly describe this as irreversible rather than implementing a purge/re-register recovery path.

## Best Possible Solution

Keep this issue open for maintainer policy and support work. The practical fix is either a manual production purge/relink for affected self-deleted GitHub identities, or a deliberate recovery flow that safely clears/reuses the Convex Auth GitHub binding for self-deleted accounts while preserving the ban/deactivation security invariants. If the intended product policy remains irreversible deletion, update the sign-in UX/support docs so users get a clear recovery/support message instead of a loop.
