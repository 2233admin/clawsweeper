## Summary

Keep this issue open. Repository policy for openclaw/clawhub says open issues are not auto-close candidates, and the current main branch still treats user-initiated deletion as irreversible rather than providing the requested purge, relink, restore, or same-GitHub re-registration path. The existing discussion and related open reports describe a real production support problem, not an obsolete report.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. The concrete path is either an admin-controlled purge/relink/recovery flow for self-deleted GitHub identities, with tests around Convex Auth bindings and ban invariants, or clearer public/support documentation plus manual production cleanup for affected users if deletion is intentionally permanent.
