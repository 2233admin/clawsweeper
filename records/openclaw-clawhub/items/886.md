## Summary

Keep open. Current main still treats user-initiated account deletion as terminal during GitHub OAuth, so the reported authorize-then-return-to-signed-out loop remains valid. Because this is an open ClawHub issue and the remaining decision is account-lifecycle/security policy plus possible production support, it should stay with maintainers.

## What This Changes

Allow or support recovery for a GitHub-linked ClawHub account after user-initiated deletion so sign-in can succeed again.

## Best Possible Solution

Define a canonical policy for deleted GitHub-linked accounts, then implement the approved recovery path across the Convex auth callback, deletion/admin tooling, user-facing error messaging, docs, and regression tests for self-deleted, deactivated, banned, and legacy deletedAt users.
