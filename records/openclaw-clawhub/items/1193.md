## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has the reported failure shape: token login can succeed while the API-token `whoami` response has no handle, and publish still depends on personal-publisher resolution that can throw `Personal publisher not found`. The digest pagination path has been moved to scheduled work, but that does not prove the reporter's wrong/null handle or production publisher rows are repaired.

## What This Changes

The issue reports that CLI login succeeds but resolves the user as unknown, `whoami` returns unknown or empty, and CLI/web publishing fails because the authenticated user's personal publisher is missing.

## Best Possible Solution

ClawHub should guarantee that every authenticated API-token user has a canonical GitHub handle and an active personal publisher before login, `whoami`, or publish can appear successful, then repair existing bad production rows and add regression coverage for null or wrong-handle users across CLI and web publish flows.
