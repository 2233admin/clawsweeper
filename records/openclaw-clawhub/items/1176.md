## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open, and current main still has a real Convex pagination failure shape: the owner-publisher package and skill digest helpers run a `while (true)` loop that can call `.paginate()` more than once inside one internal mutation. The newer scheduling split separates package and skill rebuilds, but it does not pass a cursor through those internal mutations, so owners with more than one page of packages or skills can still hit Convex's single-paginated-query limit.

## Best Possible Solution

Keep this issue open and land one focused backend PR for the pagination cluster: each owner-publisher package or skill digest internal mutation should process at most one indexed page, then reschedule itself with the continuation cursor until complete. Preserve the `by_owner_publisher` indexed reads, avoid unbounded `.collect()`, and add tests that simulate multi-page package and skill catalogs.
