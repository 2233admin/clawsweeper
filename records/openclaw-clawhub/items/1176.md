## Summary

Keep this issue open. The repository policy for openclaw/clawhub does not allow automated closure of issues in this pass, and current main only partially mitigates the reported Convex pagination failure: publisher digest work is scheduled into separate mutations, but the owner-publisher package and skill digest helpers still contain paginated loops that can call `.paginate()` more than once for large owners.

## Best Possible Solution

Keep this issue open for maintainer handling. Consolidate the duplicate publish pagination reports under a canonical issue if appropriate, verify current production behavior for the affected accounts, and change owner-publisher digest rebuilds to process at most one paginated page per mutation or another Convex-safe equivalent.
