## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still has the reported Convex failure shape: publish and sync reach the backend publish path, then owner-publisher digest rebuild helpers can still execute repeated `.paginate()` calls inside one internal mutation when a publisher has more than one page of packages or skills. The linked fix PR is closed and unmerged, and its `.collect()` approach is not the bounded continuation pattern this code path needs.

## What This Changes

Publishing or syncing skills should stop failing with Convex’s single-paginated-query error in owner-publisher digest synchronization.

## Best Possible Solution

Implement bounded, cursor-rescheduled owner-publisher package and skill digest rebuilds so each internal mutation performs at most one indexed page of work, preserves digest semantics, and has regression tests for publishers with more than one page of packages or skills.
