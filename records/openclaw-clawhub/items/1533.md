## Summary

Keep open: current main has not implemented the shared helper, and the proposed patch has a concrete behavioral question because it changes public skills and souls pagination limits from rounding finite floats to truncating them.

## What This Changes

The PR adds a shared Convex `clampInt` helper with tests and replaces local `clampInt` definitions across 11 Convex modules with imports from it.

## Best Possible Solution

Keep the deduplication direction, but land it only after preserving existing call-site semantics or explicitly standardizing the integer coercion contract with focused tests for public pagination and maintenance batch-size behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

- **[P2] Preserve rounded limits for skills and souls:** `convex/lib/math.ts:8`
  - body: Current main's `skills.ts` and `souls.ts` helpers round finite `v.number()` limits, but this shared helper truncates them.
