## Summary

Keep open. Current main still has duplicated local `clampInt` helpers and no shared `convex/lib/math.ts`, so the PR is not implemented on main. The diff is a useful cleanup, but the existing review discussion points to a real behavior decision: the proposed shared `Math.trunc` helper would change current `Math.round` behavior in `skills.ts` and `souls.ts` pagination limits.

## What This Changes

The PR extracts a shared `clampInt` helper into `convex/lib/math.ts`, adds unit tests, and replaces local helper definitions across 11 Convex modules with imports.

## Best Possible Solution

Land the deduplication only after preserving the existing rounding behavior for affected call sites or getting explicit maintainer approval to standardize on truncation, with tests covering the shared helper and the chosen pagination semantics.
