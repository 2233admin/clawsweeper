## Summary

Keep open. This is an open ClawHub issue, and the repo cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, current main still has a live account/moderation recovery gap: banned/deactivated users can be blocked at auth, admin unban handles deletedAt-based bans, but publisher moderation holds can hide all owned skills and future publishes without a current-main API to clear the hold. The discussion also links the still-open recovery PR #1133.

## Best Possible Solution

Keep this issue open for maintainer/security triage. Staff should inspect the production records for PHY041, determine whether the account is banned, deactivated, under publisher moderation, or auth-linked incorrectly, then restore access through the existing admin unban path if it is a deletedAt-based ban or review/land an audited moderation-hold recovery path such as #1133 for requiresModerationAt cases. A maintainer response should also document safe batch publishing rates and a clear appeal path for false positives.
