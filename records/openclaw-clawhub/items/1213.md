## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing PRs that are certainly implemented on main, and current main still has the reported `Plugin not found` path from publisher dashboard links through the package detail API.

## Best Possible Solution

Keep this issue open. A maintainer should ask for the affected package name or URL, reproduce from the signed-in publisher dashboard through `/plugins/$name` and `/api/v1/packages/<name>`, then make the detail page/API distinguish owner access, private or pending visibility, blocked packages, and truly missing packages.
