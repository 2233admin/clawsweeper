## Summary

This is a ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should remain open. Current main also still contains the reported `Plugin not found` UI path: dashboard plugin rows link to `/plugins/$name`, the route renders that empty state when package detail is null, and the API/helper stack can still collapse unreadable or missing packages into a generic not-found state.

## Best Possible Solution

Keep this issue open. A maintainer should ask for the affected plugin/package URL or name, reproduce from the signed-in publisher dashboard through `/plugins/$name` and `/api/v1/packages/<name>`, then make the detail page/API distinguish owner-access, pending/private visibility, and truly missing packages instead of collapsing all unreadable states into generic not-found messaging.
