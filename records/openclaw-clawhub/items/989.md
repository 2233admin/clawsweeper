## Summary

Keep this issue open. Current main still has a plausible path for the reported mismatch: the UI renders the VirusTotal row from cached `vtAnalysis.status`, while the scanner can derive that status from VirusTotal Code Insight rather than AV engine detections, so the linked VT file page can show no vendors flagged even when ClawHub displays “Suspicious”. Also, the review policy for openclaw/clawhub keeps issues open rather than auto-closing them.

## Best Possible Solution

Keep the issue open for maintainer triage. Verify the named skill’s stored scan and moderation records in production; if the suspicious state comes from Code Insight, static analysis, or OpenClaw LLM rather than AV vendor hits, split the UI copy/status so the VirusTotal vendor row does not imply vendor detections, and show Code Insight/static/OpenClaw reasons separately. If the record is stale, run the existing rescan/backfill or manual override path to clear it, then add a regression test for clean vendor stats plus non-vendor suspicious signals.
