## Summary

Keep open. This item is an issue, and the ClawHub review policy for this run says only pull requests that are certainly implemented on current main should be proposed for auto-close. The current code also still leaves the reported VirusTotal wording mismatch plausible: the backend now records Code Insight scanner metadata and clean AV engine stats, but the UI still renders the cached status as a VirusTotal row and only detects Code Insight from the `source` field.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix belongs in `SecurityScanResults` and its `VtAnalysis` data shape: include and consume `scanner` and `engineStats`, distinguish VirusTotal AV engine detections from VirusTotal Code Insight, render Code Insight as a separate or clearly annotated signal, handle older cached records defensively, and add a regression test for suspicious Code Insight with clean AV engine stats.
