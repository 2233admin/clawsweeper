## Summary

Keep this issue open. The ClawHub review policy does not auto-close issues, and current main still has a plausible UI mismatch: cached Code Insight data can be shown as a VirusTotal suspicious row even when AV engine stats are clean.

## Best Possible Solution

Keep this issue open for maintainer triage. A good fix would update SecurityScanResults and its VtAnalysis shape to consume scanner and engineStats, distinguish VirusTotal AV engine detections from VirusTotal Code Insight, render Code Insight as a separate or clearly annotated signal, handle older cached scan records defensively, and add a component regression test for suspicious Code Insight with clean AV engine stats.
