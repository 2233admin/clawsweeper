## Summary

Keep open. This is an issue, and the ClawHub review policy says issues should not be auto-closed. Current main also still leaves the reported VirusTotal wording mismatch plausible: the UI renders the cached VT status as a VirusTotal row while Code Insight detection only checks source, not the normalized scanner field.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix belongs in SecurityScanResults and its VtAnalysis data shape: consume scanner and engineStats, distinguish VirusTotal AV engine detections from VirusTotal Code Insight, render Code Insight as a separate or clearly annotated signal, handle older cached scan records defensively or via backfill/rescan, and add a component regression test for suspicious Code Insight with clean AV engine stats.
