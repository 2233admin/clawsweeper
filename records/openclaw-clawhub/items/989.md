## Summary

Keep this issue open. The ClawHub review policy says issues should not be auto-closed, and the current main branch still leaves a plausible VirusTotal wording/data mismatch: Code Insight verdicts can be cached as the VirusTotal status, while the UI renders that status in a VirusTotal row with a report link. Recent commits partially calibrate moderation for uncorroborated Code Insight findings, but they do not fully resolve the reported UI distinction or existing production data.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to separate VirusTotal AV engine detections from VirusTotal Code Insight in the UI/API: use vtAnalysis.scanner and engineStats, not only vtAnalysis.source/status; render Code Insight as its own explanatory signal or annotate the VirusTotal row; backfill or rescan existing vtAnalysis records so source/scanner/engineStats are normalized; and add a component test for clean vendor stats plus Code Insight suspicious status.
