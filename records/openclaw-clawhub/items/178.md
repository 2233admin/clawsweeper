## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main still matches the reported failure path: update hashes local JSON files, downloaded skill archives include generated `_meta.json`, and server-side resolve checks fingerprints built from published version files rather than generated download metadata.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely fix is to make client and server fingerprint inputs match, either by excluding generated ClawHub metadata such as `_meta.json` from managed-install fingerprints or by removing generated metadata from downloaded skill archives, then add a regression test for install or force-update followed immediately by update reporting the skill as up to date. Maintainers may also consolidate this with open related #169, but this issue should not be auto-closed by cleanup policy.
