## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only allows automated closure for pull requests that are certainly implemented on current main. Substantively, current main still contains the reported failure shape: publish/profile paths reach ensurePersonalPublisherForUser, publisher writes schedule owner digest sync, and the scheduled package/skill owner sync helpers still loop through paginated pages inside one mutation invocation.

## Best Possible Solution

Keep this issue open and land one focused Convex hardening PR. Preserve the current background scheduling split, but make each owner digest sync mutation process at most one paginated page and self-schedule a continuation with continueCursor when more rows remain. Add regression tests for multi-page package and skill owner sync, and audit the sibling owner-user/release pagination helpers in convex/functions.ts so the same Convex single-pagination limit does not remain on adjacent trigger paths.
