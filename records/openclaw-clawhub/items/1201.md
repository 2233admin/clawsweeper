## Summary

Keep this issue open. The current main branch appears to mitigate the originally reported inline Convex pagination conflict by scheduling package and skill owner-publisher digest rebuilds as separate internal mutations, with focused regression coverage for that scheduling shape. However, this is an open ClawHub issue, and the repository-specific cleanup policy only permits automated closure for pull requests that are certainly implemented on current main. There is also a remaining hardening question because each scheduled owner-scoped digest helper can still execute more than one paginated query for owners with more than 100 matching records.

## Best Possible Solution

Keep this issue open for maintainer handling. Maintainers should verify the scheduled digest-sync mitigation is deployed and unblocks CLI and web publishing for first-time and existing publishers, then manually close or consolidate this with related reports such as #1175, #1182, and #1205 if production is fixed. If large-owner publishes can still fail, replace the remaining owner-scoped paginated loops with bounded self-scheduling batch mutations so each invocation performs at most one paginated query.
