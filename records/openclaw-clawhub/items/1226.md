## Summary

Keep open. Repository policy for openclaw/clawhub keeps issues open during this cleanup pass, and current main still leaves the reported local package publish failure path plausible: relative package sources are resolved against the CLI workdir, which can be a configured Clawdbot/OpenClaw workspace instead of the caller's plugin repository.

## What This Changes

The issue reports that `clawhub package publish . --family code-plugin` rejects a valid local code-plugin repository with `openclaw.plugin.json required`.

## Best Possible Solution

Keep this issue open and land a focused CLI fix for local package publish source resolution. `clawhub package publish . --family code-plugin` should inspect the caller's plugin repository when the default/global workdir points elsewhere, while preserving explicit `--workdir`, absolute and tilde paths, local `owner/repo` folder precedence, and existing GitHub source behavior. Add a regression test for the cwd-vs-configured-workdir case and ship the fix in the next CLI release.
