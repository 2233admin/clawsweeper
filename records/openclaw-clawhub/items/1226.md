## Summary

Keep open. Current main still leaves the reported false manifest error plausible: `package publish .` resolves a local source against the CLI's resolved global workdir, which can fall back to a configured Clawdbot/OpenClaw workspace instead of the plugin repository where the command was run. ClawHub cleanup policy also keeps issues open unless a maintainer handles them directly.

## What This Changes

Publishing a local code plugin with `clawhub package publish .` should find the root `openclaw.plugin.json` instead of failing with `openclaw.plugin.json required`.

## Best Possible Solution

Make local package publish resolve the package source from the caller's intended package directory when no explicit workdir override is supplied, while preserving explicit workdir/env behavior and GitHub source handling. Regression coverage should prove the configured-workspace fallback no longer masks a plugin repo's manifest.
