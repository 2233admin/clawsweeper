## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. A shell check of current main also still shows a plausible path-resolution failure for `clawhub package publish .`: local source paths are resolved against the CLI global workdir, while that workdir can fall back to configured Clawdbot/OpenClaw state rather than the caller's plugin repository.

## Best Possible Solution

Keep this issue open for maintainer triage. Add a regression test for `clawhub package publish . --family code-plugin` from a plugin repository while the configured global workdir points elsewhere, then either resolve explicit local publish paths relative to the caller's cwd or clearly document and enforce the required `--workdir` behavior. Separately track the plugin search/install/list CLI request if maintainers want that terminal workflow.
