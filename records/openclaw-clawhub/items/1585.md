## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass only permits auto-closing ClawHub pull requests that current main definitely implements, not issues. Substantively, current main still flags child_process exec/spawn style usage as `suspicious.dangerous_exec`, and the prior proposal to demote benign dangerous-exec findings was closed unmerged as a moderation policy change, so this remains a maintainer/moderator decision for the hosted bc-calc listing.

## What This Changes

The issue asks whether the Bc Calc skill should remain flagged for suspicious patterns when its documented purpose is to shell out to the Unix `bc` calculator.

## Best Possible Solution

Keep this issue open for maintainer or moderator follow-up. If the `bc` shell-out is acceptable for a declared single-command calculator wrapper, clear the hosted listing through the audited manual override or rescan workflow. If maintainers want a broader false-positive fix, reopen the policy discussion around #1688 or create a narrow scanner refinement with regression tests that preserves default `child_process` detection.
