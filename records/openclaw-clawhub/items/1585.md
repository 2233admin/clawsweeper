## What This Changes

The issue asks whether Bc Calc should remain flagged as suspicious when its stated purpose is to shell out to the dedicated Unix `bc` calculator command.

## Best Possible Solution

Have a maintainer or moderator review the hosted bc-calc listing and apply the audited clean manual override if this single-command calculator wrapper is acceptable. If maintainers want a general product fix, implement a narrow policy change for declared single-command wrappers with regression tests that preserve default shell-exec warnings for ambiguous or broader skills.
