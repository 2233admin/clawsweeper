## Summary

Keep open. The ClawHub cleanup policy for this lane does not auto-close issues, and current main still makes publication depend on a successful embedding request: exhausted OpenAI 429/5xx retries throw, the publish flow converts that into a failed publish before inserting the version, and the version insertion path requires a concrete embedding vector. The requested flagged-version cleanup also depends on production moderation/storage state and should remain a maintainer support review.

## What This Changes

The issue reports an OpenAI embedding 500 while publishing an updated `agentcop` skill and asks maintainers to fix the embedding server and remove flagged versions.

## Best Possible Solution

Maintainers should inspect the OpenAI request ID and production Convex logs, verify the live `shaymizuno/agentcop` or `agentverif` ownership, versions, aliases, moderation flags, and stored artifacts, then resolve any hosted cleanup with staff tooling. If a code change is approved, publish should persist a durable pending-embedding state and schedule backfill for exhausted retryable provider outages while still failing configuration errors and non-retryable request failures.
