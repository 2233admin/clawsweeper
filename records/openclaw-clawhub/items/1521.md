## Summary

Keep this issue open. The ClawHub cleanup policy only permits automated closure for PRs that current main certainly implements, and this is an issue. Current main also still treats exhausted retryable embedding-provider failures as publish failures, while the flagged-version removal request requires production moderation review rather than a source-only close.

## What This Changes

The issue reports that publishing an updated `agentcop` skill version is blocked by `Embedding failed (500): OpenAI server error` and asks maintainers to fix the embedding failure and remove flagged versions.

## Best Possible Solution

Keep the issue open for production follow-up. Maintainers should inspect the OpenAI request ID and Convex logs, verify the live skill/version/moderation state, and use the existing moderator or admin flows for any removal. If a source change is approved, publish should tolerate retryable provider outages by recording a durable pending embedding state and scheduling backfill, while configuration errors and non-retryable request failures continue to fail fast.
