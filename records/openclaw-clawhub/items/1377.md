## Summary

Keep this issue open. The repository policy for openclaw/clawhub only allows auto-closing pull requests that are certainly implemented on main, and current main still uses a platform-wide MIT-0 skill license rather than preserving or displaying the reporter's upstream MIT license. The existing ClawSweeper comment reached the same result, and the issue remains a real maintainer product/legal policy decision rather than a cleanup-close candidate.

## Best Possible Solution

Keep this open for maintainer judgment. Maintainers should either explicitly reaffirm ClawHub's platform-wide MIT-0 publishing policy and handle publisher consent/conflicting upstream MIT packages like openclaw-workspace-sync, or implement real per-skill SPDX license support across parsing, storage, API responses, UI, CLI, docs, migration/backfill, and tests.
