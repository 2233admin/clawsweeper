## Summary

Keep open. Current main still stores, publishes, documents, and displays skill licenses as a platform-wide MIT-0/no-attribution contract, so the reported MIT versus MIT-0 mismatch remains valid. For this repository’s cleanup policy, issues should not be auto-closed, and the remaining fix needs maintainer product/legal judgment rather than a narrow automated patch.

## What This Changes

The issue asks ClawHub to display the `openclaw-workspace-sync` skill as standard MIT instead of MIT-0 and to check whether license handling is misclassifying other skills.

## Best Possible Solution

Maintainers should choose and implement one coherent license policy: either reaffirm and enforce the MIT-0-only ClawHub publishing contract while rejecting or documenting conflicting upstream package metadata, or add per-skill SPDX license support across parsing, shared schemas, Convex storage/API responses, listing and upload UI, CLI/docs, migrations/backfills, and tests.
