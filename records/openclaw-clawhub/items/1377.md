## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on main, and this item is an issue. The current main implementation also still uses a platform-wide MIT-0 skill license rather than preserving or displaying the reporter's upstream MIT license, so the reported product/legal mismatch remains unresolved.

## Best Possible Solution

Keep this open for maintainer judgment. Maintainers should either explicitly reaffirm and document ClawHub's platform-wide MIT-0 publishing policy with handling for conflicting upstream licenses, or implement real per-skill SPDX license support across parsing, storage, API responses, UI, CLI, docs, migration/backfill, and tests.
