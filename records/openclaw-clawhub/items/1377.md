## Summary

Keep open. Current main still treats skill licensing as a platform-wide MIT-0/no-attribution contract, so the reported MIT versus MIT-0 mismatch remains valid; the ClawHub cleanup policy also keeps issues open for maintainer judgment rather than auto-closing them.

## What This Changes

The issue reports that a published skill listing displays MIT-0/no-attribution licensing even though the upstream package sources declare the standard MIT license.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests so published skill listings cannot disagree with the actual license contract.
