## Summary

Keep this issue open. It is a live ClawHub account-support/product-policy request, and current main does not provide account restoration or GitHub identity unlinking after self-deletion. The implemented behavior intentionally marks deleted accounts as deactivated/purged and blocks GitHub reauth, while related reports #504, #1452, and #1620 were also kept open for the same account lifecycle gap. Repository policy for openclaw/clawhub says issues should remain open; only PRs that are certainly implemented on main are eligible for auto-close.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy judgment. If deleted accounts must remain irreversible, maintainers should reply with the policy and provide any private support path for identity unlink requests. If same-GitHub re-registration should be allowed, add a safe backend/admin flow around Convex Auth authAccounts that can unlink or supersede the deleted account without restoring banned users, with tests for web OAuth, CLI/API token behavior, handles, publishers, and existing deleted-account records.
