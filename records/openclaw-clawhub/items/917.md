## Summary

Keep open. ClawHub cleanup policy only permits automated closure for PRs that are certainly implemented on current main, and this is an open issue; substantively, current main still blocks deleted GitHub-linked accounts from reauthenticating and has no account restore or GitHub identity unlink path.

## What This Changes

The issue asks maintainers to restore an accidentally deleted ClawHub account or unlink the reporter's GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery process for deleted GitHub-linked accounts, or document and perform an operational unlink/restore path for affected users, while preserving ban protections and covering existing deleted-account records in tests.
