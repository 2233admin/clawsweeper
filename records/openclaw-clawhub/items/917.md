## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing PRs that are certainly implemented on main, and this is an open issue. Source review also confirms the requested restoration or GitHub identity unlink path is not implemented: account deletion is intentionally irreversible, OAuth reauth is blocked, and admin unban excludes permanently deleted accounts. The discussion has a second affected user plus related open account-lifecycle reports, so this needs maintainer support/product/security triage rather than cleanup closure.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should either confirm and document an irreversible-deletion support policy with a clear contact path, or add a safe backend/admin flow to restore, unlink, or supersede deleted GitHub-linked accounts, with tests covering OAuth reauth, authAccounts, API tokens, banned-account separation, handles, publishers, skills, and existing deleted-account records.
