## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only permits auto-closing PRs that are certainly implemented on current main. The source still treats user-initiated account deletion as irreversible: deletion marks the user deactivated/purged, OAuth sign-in rejects deleted or deactivated users, admin unban refuses permanently deleted accounts, and docs/UI warn that deletion cannot be undone. The discussion also includes a second affected user and related open account-lifecycle reports, so this needs maintainer support/product/security handling rather than cleanup closure.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should either confirm and document an irreversible-deletion support policy with a clear contact path, or add a safe backend/admin flow to restore, unlink, or supersede deleted GitHub-linked accounts, with tests covering OAuth reauth, CLI/API tokens, banned-account separation, handles, publishers, skills, and existing deleted-account records.
