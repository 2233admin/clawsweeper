## Summary

Keep this issue open. Repository policy for openclaw/clawhub keeps issues open, and current main still treats user-initiated account deletion as irreversible rather than providing account restoration, GitHub identity unlinking, or same-GitHub re-registration. The issue discussion includes a second affected user, and related reports such as #504, #1452, and #1620 remain open for the same account lifecycle problem.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should either confirm and document the irreversible-deletion policy with a clear support path, or add a safe backend/admin flow to restore, unlink, or supersede deleted GitHub-linked accounts, with tests covering web OAuth, CLI/API tokens, banned-account separation, handles, publishers, and existing deleted-account records.
