## Summary

Keep open. This is a ClawHub issue, and the review policy for this repository only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still has a split path: `metadata.openclaw.requires.env` is documented and parsed into normalized Clawdis metadata, but the static moderation env collector still reads only direct raw `frontmatter`/`metadata` fields and does not descend into `metadata.openclaw` or consume normalized `parsed.clawdis`. Related env-metadata false-positive reports #1053 and #1114 are also still open.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely code fix is to make static moderation consume normalized `parsed.clawdis`, or explicitly collect declarations from `metadata.openclaw`, `metadata.clawdis`, and `metadata.clawdbot`; if package.json or metadata.json declarations are part of the supported skill format, add those to the ingestion path too. Add a regression test using the inline JSON `metadata.openclaw.requires.env` shape from the report, then rescan or manually clear the affected published versions. Maintainers may also consolidate this with open related issue #1053 if they want one canonical metadata-mismatch thread.
