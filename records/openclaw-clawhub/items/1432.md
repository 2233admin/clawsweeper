## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still leaves the documented nested `metadata.openclaw.requires.env` shape invisible to static moderation even though docs, parsing, and the LLM prompt path understand it.

## What This Changes

The issue asks ClawHub to extract `openclaw.requires.env` and `primaryEnv` from published skill metadata so correctly declared credentials do not trigger false suspicious moderation.

## Best Possible Solution

Make static publish and static rescan moderation consume the same normalized OpenClaw/Clawdis runtime metadata used by parsing/display/LLM context, add regression coverage for the inline JSON `metadata.openclaw.requires.env` shape, then rescan or manually review the affected DeepRead versions while leaving broader `metadata.json` ingestion to #1053.
