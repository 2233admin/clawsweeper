## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing PRs that are definitely implemented on current main, not issues. Substantively, current main documents and parses `metadata.openclaw.requires.env`, and the LLM prompt path now has an OpenClaw fallback, but the static publish scan still only receives raw frontmatter/metadata and its env collector reads direct fields rather than nested `metadata.openclaw`. The live affected skill records are still flagged suspicious, so this remains unresolved moderation/metadata cleanup.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The narrow code direction is to make static moderation consume the same normalized parsed Clawdis metadata used by display/LLM context, or explicitly collect env declarations from `metadata.openclaw`, `metadata.clawdis`, and `metadata.clawdbot`; add a regression test using the inline JSON frontmatter from this report. Separately decide package-level metadata ingestion with #1053, then rescan/backfill or manually review the affected published versions.
