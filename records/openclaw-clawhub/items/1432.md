## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy for this review only permits auto-closing pull requests that current main certainly implements. Substantively, current main still has an end-to-end gap: `metadata.openclaw.requires.env` is documented and parsed into normalized `parsed.clawdis`, but the static moderation env collector still receives raw frontmatter/metadata and only reads direct env fields, so nested `metadata.openclaw.requires.env` and package-level declarations are not fully covered. Related metadata-mismatch reports #1053 and #1114 are also still open.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely fix is to make static moderation consume normalized `parsed.clawdis`, or explicitly collect declarations from `metadata.openclaw`, `metadata.clawdis`, and `metadata.clawdbot`; if package.json or metadata.json env declarations are supported skill metadata, add that extraction path too. Add a regression test for the inline JSON `metadata.openclaw.requires.env` shape from the report, then rescan or manually clear affected published versions.
