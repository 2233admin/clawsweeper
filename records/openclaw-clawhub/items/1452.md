## Summary

Keep this issue open. The ClawHub cleanup policy says open issues should remain open unless a strong non-implemented close reason applies, and current main still matches the reported OAuth-after-deletion failure mode: self-service deletion permanently deactivates the user, OAuth completion rejects deactivated users, and there is no source path that clears the Convex Auth GitHub account binding during deletion.

## Best Possible Solution

Keep this issue open for maintainer product and support follow-up. Maintainers should decide whether same-GitHub re-signup after self-deletion is allowed; if yes, add a safe unlink or retired-binding path during account deletion, provide an admin/support repair path for already-stuck accounts, and cover the OAuth/deleted-account flow with tests. If deletion must remain irreversible, keep the blocking behavior but add clear support tooling and user-facing guidance.
