## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub says issues should remain open unless a strong non-implemented close reason applies, and current main still matches the reported post-deletion GitHub login failure mode: deletion permanently deactivates the user, the OAuth callback rejects deactivated users, and there is no code path that clears the separate Convex Auth GitHub account binding or restores the user record.

## Best Possible Solution

Keep this issue open for maintainer product/support follow-up. Maintainers should decide whether same-GitHub re-signup after self-deletion should be allowed; if yes, add a safe unlink or retired-binding path during account deletion, add an admin/support repair path for already-stuck accounts, and cover the OAuth/deleted-account flow with tests. If deletion must remain irreversible, keep the blocking code but add clearer support tooling and user-facing guidance.
