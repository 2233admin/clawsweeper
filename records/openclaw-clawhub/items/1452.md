## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main certainly implements. Current main still intentionally rejects deleted/deactivated GitHub OAuth sign-in and provides neither a restore flow nor an OAuth-binding unlink flow for this support case.

## What This Changes

The reporter asks for the existing GitHub OAuth binding to be removed or the deleted account restored so the same GitHub account can sign in again.

## Best Possible Solution

Define an explicit, audited recovery or re-registration policy: if maintainers choose to support self-deleted account recovery, add a controlled unlink/rekey or restore path that keeps banned and abuse-disabled accounts blocked, updates UI/docs to match the policy, and adds Convex tests for same-GitHub re-signup, operator recovery, and banned-account rejection.
