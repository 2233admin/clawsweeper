## Summary

Keep this issue open. Current main still blocks GitHub OAuth sign-in for deleted/deactivated users, self-service deletion still does not unlink the Convex Auth GitHub account binding, and the ClawHub cleanup policy keeps issues open unless a strong non-code close reason applies.

## Best Possible Solution

Keep this open for maintainer product/support follow-up. Maintainers should decide whether same-GitHub re-signup after self-service deletion is allowed; if yes, add a guarded `authAccounts` unlink, retired-binding, or restoration path plus an admin/support repair mutation for already-stuck accounts, with tests for OAuth completion, deletion, banned users, and legacy `deletedAt` migration. If deletion must remain irreversible, keep the block but add explicit support tooling and user-facing guidance for affected accounts.
