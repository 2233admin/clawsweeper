## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub says issues should remain open unless a strong non-implemented close reason applies, and current main still matches the reported failure mode: user-initiated deletion marks the user as permanently deactivated, keeps Convex Auth account storage in play, and OAuth sign-in rejects the deleted account instead of restoring it or unlinking the GitHub binding. The provided timeline also shows multiple related open reports for the same post-deletion GitHub login loop, so this remains an active account lifecycle/support problem rather than closeable cleanup.

## Best Possible Solution

Keep this issue open for maintainer product/support follow-up. Maintainers should decide whether same-GitHub re-signup after self-deletion should be allowed. If yes, add a safe unlink or retired-binding path during account deletion, provide an admin/support mutation for already-stuck accounts, and cover the OAuth/deleted-account flow with tests. If deletion must remain irreversible, keep the blocking code but add clear support tooling and user-facing guidance for affected accounts.
