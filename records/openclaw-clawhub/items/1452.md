## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open unless there is a strong close reason, and current main still matches the reported account-recovery failure: deleted/deactivated users are rejected during GitHub OAuth, self-service deletion does not unlink the Convex Auth GitHub binding, and related login-after-deletion reports remain open.

## Best Possible Solution

Keep this open for maintainer product/support follow-up. Maintainers should decide whether same-GitHub re-signup after self-service deletion is allowed; if yes, implement a guarded authAccounts unlink, retired-binding, or restoration path plus admin/support tooling for already-stuck accounts, with explicit protection for banned and moderated users. If deletion remains irreversible, add clearer user-facing support guidance and operator tooling so affected users are not left in a confusing OAuth loop.
