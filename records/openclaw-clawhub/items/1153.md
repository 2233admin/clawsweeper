## Summary

Keep this issue open. The repository policy for ClawHub allows auto-close only for PRs that are definitely implemented on current main, and this is an issue requiring production maintainer/operator action. Current main has an admin-only hard-delete path that deletes skill version rows, but the owner-facing CLI and public API delete paths remain soft-delete only, so the reporter’s requested self-service reset is not currently satisfied.

## Best Possible Solution

Keep this issue open for maintainer/operator handling. A maintainer should verify the requester controls the production owner account and target skill, decide whether storage blobs must be purged in addition to database rows, then either run the existing admin hard-delete path or define an audited owner reset/version-purge workflow if these requests are expected to recur. Triage #1167 at the same time for a consistent policy.
