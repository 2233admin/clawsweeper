## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only permits auto-close for pull requests that are certainly implemented on current main. Current main still matches the reporter's substantive problem: the owner-facing CLI/API delete path is a soft delete, existing skillVersions rows still block reusing the same semver, and the hard-delete path is admin/operator-controlled production cleanup rather than a self-service reset.

## Best Possible Solution

Keep this issue open for maintainer/operator handling. A maintainer should verify requester ownership in production, inspect the target skill, versions, fingerprints, embeddings, file-storage, and audit implications, then run the existing admin hard-delete path or design a narrower audited owner reset/version-purge workflow if these clean-republish requests are expected to recur. Triage related open request #1167 alongside this one for consistent policy.
