## Summary

This is an open ClawHub issue requesting destructive production data cleanup, so the supplied repository policy does not allow proposing auto-close. Current main matches the reporter's premise: public/CLI deletion is soft delete, while full hard delete is admin-only and deletes version records as part of a broader skill cleanup.

## Best Possible Solution

Keep this issue open for maintainer handling. A maintainer/operator should verify that the requester controls JasonStarlight/phosor-ai, then either perform an audited admin hard-delete/production cleanup for the listed versions or decide whether a narrower admin-only hard-delete workflow is needed for repeat support cases.
