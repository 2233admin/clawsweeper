## Summary

Keep this PR open. Current main still has the global stats full-table fallback, the deprecated manual stats mutation, the stale seeding command, and the single-key rate-limit counter that this PR is meant to remove. The PR is also changing an abuse-control path, and review discussion still flags the need for maintainer judgment on hard versus soft aggregate rate caps.

## Best Possible Solution

Keep this PR open for maintainer review or update. The useful path is to land an accepted patch that removes the global stats full-table fallback and deprecated manual mutation, updates the seeding docs/tests, and implements the rate-limit contention fix with accurate headers plus explicit hard-or-soft aggregate cap semantics validated against Convex insights/logs.
