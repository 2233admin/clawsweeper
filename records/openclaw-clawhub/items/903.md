## Summary

Keep this PR open. Current main still has the global-stats live recount/full-table-scan paths, the deprecated manual stats mutation, stale seeding docs, and the single-key rate-limit counter; the PR addresses real production performance concerns but changes rate-limit enforcement semantics and needs maintainer/security review before merge.

## What This Changes

The PR removes the global public-skill count full-table fallback, switches rate-limit writes to sharded counters, uses aggregate rate-limit status for response headers, deletes a deprecated stats maintenance mutation, and updates seeding docs.

## Best Possible Solution

Keep this PR open for maintainer/security review or revision. The best path is to land a patch that removes the global-stats live recount fallback and deprecated manual mutation, updates docs/tests for the action-based reconciliation path, and reduces rate-limit OCC contention while making aggregate cap and response-header semantics explicit and validated against Convex insights/logs.
