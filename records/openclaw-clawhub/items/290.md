## Summary

Keep this issue open. Current main still does not explicitly support PowerShell skill files, and the ClawHub cleanup policy says issues should remain open unless maintainers decide otherwise. Related #897 is open with the same limitation and includes a published workaround/context, so maintainers should coordinate the implementation path.

## Best Possible Solution

Keep this issue open and implement explicit, security-aware PowerShell publish support. Add intended extensions such as .ps1 and likely .psm1/.psd1 to the shared and CLI allowlists, normalize content types, update browser/server/archive/GitHub import behavior, add tests and static-scan coverage, update docs, then coordinate resolution with #897.

## Risks / Open Questions

- PowerShell files are executable scripts, so support should include explicit security review and static publish-scan coverage for encoded commands, download-and-execute behavior, and exfiltration-like patterns.
- The allowlist is mirrored across shared schema, browser upload helpers, server validation, CLI discovery, archive/GitHub import paths, generated package output, tests, and docs; a partial fix could leave inconsistent behavior.
- Maintainers should choose whether this issue or #897 is canonical before closing either report.
