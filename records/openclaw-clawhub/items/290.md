Keep this issue open. Current main still does not explicitly support PowerShell skill files, and the ClawHub cleanup policy for this review says issues should remain open rather than auto-closed.

## Best Possible Solution

Keep this issue open and implement explicit, security-aware PowerShell publish support. Add intended extensions such as .ps1 and likely .psm1/.psd1 to the shared and CLI allowlists, update content-type normalization, web and server validation tests, package discovery/archive/GitHub import behavior, static scan coverage, and docs, then coordinate resolution with related #897.

## Risks / Open Questions

- PowerShell files are executable scripts, so support should be added with explicit security review, static publish-scan coverage, and tests for risky patterns such as encoded commands, download-and-execute behavior, and exfiltration-like flows.
- The allowlist is mirrored across shared schema, generated/dist package files, browser upload helpers, server publish validation, CLI discovery, archive/GitHub import behavior, tests, and docs. A partial fix could leave inconsistent behavior.
- Related #897 remains open for the same limitation, so maintainers should coordinate which issue is canonical before closing either report.
