## Summary

Keep this issue open. Repository policy for ClawHub keeps issues open, and current main still does not explicitly support PowerShell uploads: the shared text-file extension allowlists omit PowerShell extensions, web and server publish validation depend on that allowlist unless MIME type is already text, and CLI/package discovery skips unknown extensions. Related #897 remains open with the same user problem and a SKILL.md workaround.

## Best Possible Solution

Keep this issue open and implement explicit, security-aware PowerShell publish support. Add intended PowerShell extensions such as .ps1 and likely .psm1/.psd1 to the shared schema allowlists and content-type normalization, update web upload and server publish validation tests, update CLI/package discovery tests, document supported extensions, and coordinate with related #897 before closing either issue.
