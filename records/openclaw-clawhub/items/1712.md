## Summary

Keep this issue open. Current main still exposes package stats but does not record package download/install increments, and skill install counters are still driven by authenticated `clawhub sync` telemetry rather than immediate `skill install/add` activity. The ClawHub repository policy also says issues should stay open rather than be auto-closed.

## Best Possible Solution

Keep the issue open and implement a focused stats pipeline: add package download/install stat recording with tests around `/api/v1/packages/{name}/download` and the package/plugin install path, then decide whether skill installs should report at install/add time or whether the CLI/docs should clearly preserve the current `clawhub sync` telemetry model.

## Risks / Open Questions

- Package `downloads` and `installs` can remain misleading for package authors and catalog users because the download endpoint returns archives without touching package counters.
- Changing skill install counts from sync-time telemetry to install-time reporting needs an explicit privacy/product decision so it does not bypass the documented telemetry model.
