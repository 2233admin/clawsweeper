## Summary

Keep open. Current main still exposes package stats but lacks a writer for package downloads or installs, and skill install counters are still driven by authenticated sync telemetry rather than immediate `skill add` activity. The prior ClawSweeper comment reached the same result, and the ClawHub review policy keeps issues open unless they are PRs definitely implemented on main.

## What This Changes

The issue requests real download/install counter updates for packages and community skill installs instead of leaving those stats at zero.

## Best Possible Solution

Define the intended stats semantics first, then add a ClawHub package stats pipeline for archive downloads and any ClawHub-controlled install signal, plus matching tests and API/CLI documentation. For skills, either document sync-time telemetry as the install counter source or add a deliberate immediate-install reporting path with opt-out and privacy behavior preserved.
