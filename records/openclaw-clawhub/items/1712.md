## Summary

Keep this issue open. Current main still exposes stored package stats, but package download/install/star counters do not have a recording path, and skill install counters are still driven by authenticated `clawhub sync` telemetry rather than immediate `openclaw skills install` activity. ClawHub review policy also keeps issues open rather than auto-closing them.

## Best Possible Solution

Keep the issue open and implement a focused stats pipeline: add package download/install/star recording with tests around `/api/v1/packages/{name}/download` and the eventual plugin/package install path, then make a maintainer product decision on whether skill installs should count at install time or continue to rely on documented `clawhub sync` telemetry with clearer CLI/docs guidance.
