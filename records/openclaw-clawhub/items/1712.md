## Summary

Keep open. Current main still exposes package stats but does not appear to increment package downloads or installs, and skill install counters are still fed by authenticated `clawhub sync` telemetry rather than immediate `clawhub install` activity. The ClawHub review policy for this sweep also keeps issues open; only definitely implemented PRs are auto-close candidates.

## Best Possible Solution

Keep this issue open and split the maintainer decision into two pieces: add a package stats pipeline for package archive downloads and any ClawHub-controlled package install signal, then explicitly decide whether skill install counters should remain sync-time telemetry or also count immediate CLI installs, with matching privacy/docs/test updates.
