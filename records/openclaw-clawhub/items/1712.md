## Summary

Keep open. Current main still exposes package stats but does not increment package downloads or installs, and skill install counters still depend on authenticated `clawhub sync` telemetry rather than immediate `clawhub skill add` activity. The ClawHub review policy also says issues should stay open; only definitely implemented PRs are auto-close candidates.

## Best Possible Solution

Keep this issue open. The best maintainer path is to split the work: add a focused package stats pipeline for package archive downloads and any ClawHub-controlled package install signal, then explicitly decide whether skill install counters should remain sync-time telemetry or also count immediate CLI installs with clear privacy/docs behavior.
