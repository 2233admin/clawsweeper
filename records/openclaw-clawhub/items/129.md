## Summary

Keep open. The repository policy for this ClawHub cleanup only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Current main has meaningful mirror-prune code for non-public skills, but source inspection does not prove the production openclaw/skills mirror and downstream indexes have already removed historical malicious entries such as sakaen736jih/bird-js.

## Best Possible Solution

Keep this issue open for maintainer/security follow-up. Maintainers should verify the live openclaw/skills mirror for historical malicious entries such as sakaen736jih/bird-js, run or repair the GitHub backup prune job if needed, coordinate downstream cache/tombstone behavior, and manually close only after the public mirror no longer serves removed or suspicious skills.
