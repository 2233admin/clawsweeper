Keep open. This is a ClawHub issue, and the repository policy for this review only allows auto-closing pull requests that are certainly implemented on current main. ## Best Possible Solution

Keep this issue open for maintainer/security handling. Maintainers should verify the production GitHub backup configuration for the public skills mirror, confirm or run a one-time prune for historical malicious entries such as bird-js and the sakaen736jih-owned skills, add focused regression coverage for mirror pruning if practical, and close manually only with deployment plus mirror-state evidence.
