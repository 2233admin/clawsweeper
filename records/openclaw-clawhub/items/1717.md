## Summary

Keep this issue open. The ClawHub cleanup policy for this pass keeps issues open, and the reported production failure still reproduces on both the canonical site route and the direct Convex auth route. Current main still configures GitHub OAuth with empty-string fallbacks for missing credentials, the auth routes are mounted from that config, and the CLI browser login flow still depends on successful GitHub sign-in before `tokens.create` can mint a token. The same-author PR #1720 and related production callback issue #1581 remain open.

## Best Possible Solution

Keep this issue open while maintainers verify the production Convex OAuth environment variables and GitHub OAuth app callback settings, then review and land PR #1720 or an equivalent source-side fail-fast validation with coverage. After deployment, smoke check both `/api/auth/signin/github` routes and the browser-based `clawhub login` token flow.
