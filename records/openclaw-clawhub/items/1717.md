## Summary

Keep this issue open. The ClawHub cleanup policy for this review pass only allows auto-closing PRs that are certainly implemented on current main, and the reported production OAuth failure still reproduces on both the canonical site route and direct Convex route. Current main still uses empty-string GitHub OAuth env fallbacks, the auth routes are mounted from that config, and browser-based CLI token generation still depends on successful GitHub sign-in. The same-author PR #1720 is open as an implementation candidate, and related production callback configuration issue #1581 is also open.

## Best Possible Solution

Keep this issue open while maintainers verify the production Convex OAuth env vars and GitHub OAuth app callback settings, then review and land PR #1720 or an equivalent source-side missing-env validation with focused test/CI coverage. After deployment, smoke check /api/auth/signin/github on both clawhub.ai and the Convex site URL, then verify the browser-based clawhub login token flow end to end.
