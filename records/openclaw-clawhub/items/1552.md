## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only allows auto-closing pull requests that current main certainly implements. Current main still leaves the reported GitHub OAuth no-session path unresolved: the callback is delegated to Convex Auth, the frontend still has a generic no-session branch, deleted/deactivated account reauth is intentionally rejected, and related OAuth/session reports remain open.

## What This Changes

The issue reports that GitHub OAuth completes and redirects back to ClawHub, but no authenticated session cookie/session is created, especially after the reporter previously deleted a GitHub-backed account.

## Best Possible Solution

Pick a canonical OAuth/session cluster, reproduce the production GitHub OAuth flow with request and response cookie attributes, distinguish active-account session creation from intentional deleted-account rejection, then land the narrow dependency, deployment config, or app-code fix with regression coverage and a production smoke check.
