## Summary

Current main still lacks `clawhub update --dry-run`. The update command only registers `--all`, `--version`, and `--force`, and `cmdUpdate` still follows the normal filesystem/download/origin/lockfile mutation path. Because this is an issue in `openclaw/clawhub`, repository policy keeps it open rather than auto-closing; it is a narrow, valid fix-PR candidate.

## Best Possible Solution

Keep this issue open and implement a focused CLI enhancement: add `--dry-run` to `clawhub update`, share the existing update resolution checks, print a useful preview for single-skill and `--all` flows, and guarantee no filesystem, origin metadata, download/extract, or lockfile mutation in dry-run mode. Use the existing sync and package publish dry-run patterns as precedent without changing those commands.
