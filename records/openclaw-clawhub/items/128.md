## Summary

Keep this issue open. Current main still does not implement `clawhub update --dry-run`: the CLI update command only registers `--all`, `--version`, and `--force`; `cmdUpdate` has no dry-run option or no-write planning path; and the implementation still performs update writes. Because this is an issue in `openclaw/clawhub`, the repository cleanup policy also says to keep issues open rather than auto-close them.

## Best Possible Solution

Implement the feature in the ClawHub CLI core: add `--dry-run` to `clawhub update`, share a planning path for single-skill and `--all` updates, report would-update/up-to-date/skipped/local-change states, guarantee no directory/origin/lockfile writes during dry-run, add focused no-write tests, and document the flag with the existing update command options.
