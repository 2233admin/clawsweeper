## Summary

Keep this issue open. Current main still does not implement `clawhub update --dry-run`; the update command has no dry-run option, `cmdUpdate` has no dry-run parameter or planning path, and the update implementation still performs filesystem and lockfile writes on normal update paths. Under the ClawHub cleanup policy, issues should remain open for maintainer follow-up.

## Best Possible Solution

Implement the feature in the ClawHub CLI core: add `--dry-run` to `clawhub update`, share a planning path between single-skill and `--all` flows, print would-update/up-to-date/skipped/local-change states, guarantee no directory, origin, or lockfile writes during dry-run, add focused no-write tests, and document the flag with the existing update options.
