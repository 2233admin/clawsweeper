## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main certainly implements. Current main still matches the reported account lockout: self-deletion creates a deactivated user record, OAuth completion rejects that record on reauth, and the admin unban path does not cover permanently deleted accounts.

## What This Changes

The reporter needs a way to regain access, restore, or re-register with a GitHub identity after accidentally deleting their ClawHub account.

## Best Possible Solution

Define a maintainer-owned account recovery outcome: either keep irreversible deletion as policy and document the supported support response clearly, or add an audited admin/support flow that can reactivate or unlink self-deleted GitHub identities while preserving ban and moderation boundaries.
