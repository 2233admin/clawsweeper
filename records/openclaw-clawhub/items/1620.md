## Summary

Keep this issue open. The ClawHub cleanup policy for this repository keeps issues open, and current main still matches the reported support problem: self-deletion writes permanent deactivation state, GitHub OAuth rejects deleted/deactivated users, and the admin unban flow explicitly does not restore permanently deleted accounts.

## Best Possible Solution

Keep this issue open for maintainer support and product/security policy handling. If same-GitHub recovery should be supported, add an audited admin/support path to reactivate or unlink deleted GitHub identities, clearly distinguish self-deletion from bans/moderation, update deletion warnings/docs, and add tests for re-registration plus banned/deactivated boundary cases. If irreversible deletion remains intentional, maintainers should reply with the supported manual account-resolution process.
