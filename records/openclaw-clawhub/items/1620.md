## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy keeps issues open. Current main still matches the reported account recovery problem: user deletion is treated as permanent, GitHub OAuth blocks deactivated users, admin unban cannot restore them, and docs/UI present deletion as irreversible.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. If same-GitHub recovery should be supported, add an audited admin/support path to reactivate or unlink deleted GitHub identities, update deletion warnings/docs, and add tests for re-registration plus banned/deactivated boundary cases. If irreversible deletion remains intentional, maintainers should reply with the supported manual account-resolution process.
