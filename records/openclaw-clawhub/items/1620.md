## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main also still matches the reported failure mode: self-service account deletion permanently deactivates the user, GitHub OAuth rejects deleted/deactivated identities, tests assert that behavior, and docs/UI describe deletion as irreversible rather than offering self-service re-registration or restore.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. If same-GitHub recovery should be supported, add an explicit admin/support path to reactivate or unlink deleted GitHub identities, update deletion warnings/docs, and add tests for re-registration plus banned/deactivated boundary cases. If irreversible deletion remains intentional, maintainers should respond with the supported manual account-resolution process rather than closing through automated cleanup.
