## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main also still matches the reported behavior: self-service account deletion permanently deactivates the user, GitHub OAuth completion rejects that identity, and the tests/docs preserve the irreversible-deletion path rather than allowing same-GitHub re-registration or restore.

## Best Possible Solution

A maintainer should keep this open long enough to respond to the reporter and decide the product/support policy. If same-GitHub re-registration should be supported, add an explicit admin/support path to unlink or reactivate deleted GitHub identities, update the deletion warning/docs, and add tests covering re-registration plus banned/deactivated boundary cases. If irreversible deletion remains intentional, handle this manually as account support rather than through automated cleanup.
