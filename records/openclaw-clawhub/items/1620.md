## Summary

Keep this issue open. Repository policy for openclaw/clawhub keeps issues open, and current main still matches the reported account-support failure: self-deletion stores a deactivated/purged user, OAuth rejects that account on reauth, and the admin unban path explicitly refuses permanently deleted accounts.

## Best Possible Solution

Keep this issue open for maintainer support and product/security policy handling. If same-GitHub recovery should be supported, add an audited admin/support path to reactivate or unlink deleted GitHub identities, distinguish self-deletion from bans/moderation, update deletion warnings/docs, and add tests for re-registration plus banned/deactivated boundary cases. If irreversible deletion remains intentional, maintainers should reply with the supported manual account-resolution process.
