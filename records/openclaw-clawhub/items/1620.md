## Summary

Keep open. The ClawHub cleanup policy for this review does not allow auto-closing issues, and current main still matches the reported failure: self-deletion leaves a deactivated/purged user record, OAuth completion rejects that record on reauth, and the existing admin unban path is only for banned legacy-deleted users, not permanently deleted accounts.

## What This Changes

The issue asks maintainers to restore access or allow re-registration with the same GitHub account after accidental ClawHub account deletion.

## Best Possible Solution

Define a maintainer-owned account recovery outcome: either keep irreversible deletion as policy and document the supported support response clearly, or add an audited admin/support flow that can reactivate or unlink self-deleted GitHub identities while preserving bans, moderation holds, published content ownership, token revocation, and deletion warnings.
