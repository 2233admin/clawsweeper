## Summary

Keep open. ClawHub cleanup policy keeps open issues out of auto-close scope, and current main only partially addresses the report: `metadata.openclaw` is documented, parsed, stored, and partly included in the LLM prompt path, but the static scanner still reads env declarations shallowly and the public v1 metadata response still omits env/install/dependency declarations.

## What This Changes

The issue asks ClawHub to stop flagging legitimate skills as suspicious when required environment variables or install specs declared in SKILL.md frontmatter are missing from registry/scanner metadata, either by ingesting `metadata.openclaw`, adding publish flags, or changing scanner logic.

## Best Possible Solution

Define the supported registry metadata contract for `metadata.openclaw` and related aliases, then route normalized runtime metadata consistently through static scanning, LLM prompt inputs, UI/setup prompts, public API responses, docs, and tests for `requires.env`, `envVars`, install specs, and dependency declarations including pip/uv.
