## Summary

Keep open. Repository policy keeps open ClawHub issues out of auto-close scope, and the report remains substantively valid: current main parses and stores normalized runtime metadata, but the static env scanner still reads declarations shallowly and the public v1 metadata response still omits env/install/dependency declarations.

## What This Changes

The issue requests a way for ClawHub registry metadata and security scanning to honor env var and install declarations from SKILL.md metadata so legitimate skills are not flagged as suspicious for metadata mismatches.

## Best Possible Solution

Define one supported normalized runtime metadata contract for metadata.openclaw and aliases, then route those fields consistently through static scanning, LLM prompt inputs, public API responses, UI/setup prompts, docs, and tests for requires.env, envVars, install specs, and dependency declarations.
