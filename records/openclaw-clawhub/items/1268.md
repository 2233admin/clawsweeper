## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still appears only partially aligned with the report: publish parses and stores normalized `metadata.openclaw`, and the LLM prompt can read it, but the static scanner's declared-env collection still only reads shallow `frontmatter`/`metadata` fields and the public v1 metadata response only exposes OS/system metadata, not env/install/dependency declarations. Related #1053 remains open for the same registry-summary/env mismatch family.

## Best Possible Solution

Keep this open and consolidate it with #1053. A maintainer should define the supported registry metadata contract, then route normalized SKILL.md runtime metadata through static scanning, LLM prompt inputs, UI/setup prompts, and public API responses consistently, with tests for `metadata.openclaw.requires.env`, detailed `envVars`, dependency declarations including pip, and install specs.
