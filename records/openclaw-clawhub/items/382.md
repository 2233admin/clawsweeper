## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. The current source also still supports the reported failure mode: publishing can set the skill-level moderation state to hidden while scans/review are pending, and public search hydration filters hidden skills instead of preserving the last approved public result.

## Best Possible Solution

Keep this issue open and implement a version-aware moderation/search path: when an already-approved skill publishes a new version, preserve the last approved public skill projection, digest, and searchable embedding until the new scan clears, or skip re-review when the relevant content fingerprint and scanner inputs are unchanged. Add regression coverage for a non-trusted publisher updating an already-approved skill and verify public search still returns the skill during pending review.
