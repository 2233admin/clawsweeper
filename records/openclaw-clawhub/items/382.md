## Summary

Keep this issue open. The repository policy keeps ClawHub issues open, and current main still matches the reported behavior: publishing schedules fresh VT and LLM review, ordinary publishers can have the skill-level moderation state reset to hidden, and both vector hydration and lexical fallback convert the current skill/digest through public visibility checks that filter hidden skills out.

## What This Changes

Issue requests that already-approved skills remain searchable while a newly published version is under review or scanning, and that unchanged content not retrigger review flags unnecessarily.

## Best Possible Solution

Keep this issue open and design a version-aware moderation/search path. Preserve the last approved public skill projection, digest, and searchable embedding while a newly published version is pending VT/LLM review, then promote the new latest public projection only after review clears it. Add regression coverage for ordinary publishers republishing an already-approved skill and for unchanged-content review reuse if that optimization is implemented.
