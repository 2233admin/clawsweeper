## Summary

Keep this issue open. Current main still matches the reported behavior: every skill publish schedules VT and LLM review work, ordinary publishers can have the skill-level moderation status reset to hidden while that work is pending, the search digest mirrors that current hidden state, and public search/list hydration filters hidden skills through toPublicSkill. The repository-specific ClawHub cleanup policy also says issues should remain open in this pass.

## Best Possible Solution

Keep this issue open and design a version-aware moderation/search path. The likely direction is to retain the last approved public skill projection, digest, and searchable embedding while a newly published version is pending scan/review, then promote the new latest version only after scan/review clears it. If unchanged-content reuse is added, compare the exact scanner inputs/fingerprint and add regression coverage for already-approved skills republished by ordinary publishers.
