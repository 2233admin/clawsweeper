## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. The code review also still supports the reported failure mode: publishing a new version can set the skill-level moderation state to hidden while review/scan is pending, and public search hydration rejects hidden skills instead of preserving the last approved public result.

## Best Possible Solution

Keep this issue open and implement a version-aware review/search path: when an already-approved skill publishes a new version, preserve the last approved public skill projection, digest, and embedding in search until the new scan clears, or skip re-review when scanner inputs/content fingerprints are unchanged. Add regression coverage for a non-trusted publisher updating an already-approved skill and verify public search still returns the skill during pending review.
