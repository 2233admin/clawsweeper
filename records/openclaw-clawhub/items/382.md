## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only allows automated close recommendations for pull requests that are certainly implemented on main. A source pass on current main also shows the reported behavior is still plausible: publishing can set skill-level moderation to hidden, the search digest mirrors that state, and public search hydration filters hidden skills instead of preserving a last-approved searchable projection.

## Best Possible Solution

Implement a version-aware review/search path for skills: when an existing approved skill publishes a new version, keep the last approved public version/digest/embedding searchable until the new scan clears, or skip re-review when the scanner inputs/content fingerprint are unchanged. Add regression coverage for a non-trusted publisher updating an already-approved skill and verify public search still returns the skill during pending review.
