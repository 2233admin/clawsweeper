## Summary

Keep open. This is a ClawHub issue, and the provided repository policy only allows auto-closing ClawHub PRs that are certainly implemented on main. Substantively, current main has trust/scanning primitives, including admin-set trusted publishers, official/community package channels, per-package GitHub Actions trusted publishing, and pending scan visibility, but I did not find a documented community verified-publisher path or proof that the reporter’s scan-status question is resolved.

## Best Possible Solution
