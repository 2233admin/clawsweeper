## Summary

Keep open. This is a ClawHub issue, and the provided repository policy only allows auto-closing ClawHub PRs that are certainly implemented on main. Substantively, current main has official/trusted-publisher primitives, package-level GitHub Actions trusted publishing, and async scan handling, but I did not find a documented community verified-publisher path or a source-backed operational answer for code-plugin scans stuck pending after 24+ hours.

## Best Possible Solution

Keep this issue open for maintainer/product follow-up. The best resolution is to document ClawHub's publisher trust model and code-plugin scan expectations, then either publish a community verified-publisher path or explicitly state that trusted/official status remains staff-managed while package scan status is asynchronous and may require operational review when it stays pending.
