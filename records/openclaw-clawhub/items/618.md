## Summary

Keep open. Current main still has the reported CLI presentation bug: when local skill content does not match a registry version but a registry latest exists, `clawhub sync` classifies it as an update and renders the next semver bump as the update target. Repo policy for `openclaw/clawhub` also says issues should stay open unless a maintainer handles them.

## What This Changes

The issue reports that `clawhub sync` shows semver-bumped versions as available registry updates while `clawhub inspect` shows the actual latest registry versions.

## Best Possible Solution

Update the sync planning output and interactive upload-selection label so unmatched local content is described as local changes relative to the actual registry latest, while any bumped value is clearly labeled as the proposed upload version. Add regression coverage for the `latestVersion` present plus `match: null` case and keep the dashboard warning tracked separately in #1180.
