## Summary

Keep this issue open. The ClawHub cleanup policy says issues should stay open, and current main still has the same plausible CLI wording problem: sync reports the next publish target as `UPDATE latest -> next`, while inspect reports the registry latest version.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to change sync output so it distinguishes registry latest from the proposed publish target, or to adjust classification if unchanged installed skills are being detected as local changes. Add a focused CLI test for the reported `latest -> bumped target` display case.
