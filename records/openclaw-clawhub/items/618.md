## Summary

Keep this issue open. The ClawHub review policy only permits auto-closing PRs that are certainly implemented on main, not issues, and the current CLI code still appears to have the reported misleading sync version display: `sync` prints registry latest version -> proposed bumped publish version, while `inspect` prints the actual registry latest version.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to change `clawhub sync` output so it clearly separates the actual registry latest version from the proposed publish version, or to adjust update classification if unchanged installed skills are being marked as local changes. Add a focused CLI test for the `latest -> bumped target` display case.
