## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing pull requests that are certainly implemented on main, and this is an issue. Independently, current main still appears to contain the reported misleading CLI behavior: `sync` classifies a local fingerprint miss as an update, then displays the actual registry latest version followed by a proposed bumped publish version, while `inspect` prints the actual latest registry version. The related #1180 reference is a separate dashboard deployment-SHA drift warning, not a canonical duplicate for this CLI sync report.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to change `clawhub sync` output so it clearly separates the actual registry latest version from the proposed publish version, or to classify unchanged installed skills/local fingerprint misses with wording like local changes or publish target instead of an available update. Add a focused CLI test where latest is 1.0.2, bump is patch, and the output must not imply that 1.0.3 already exists in the registry.
