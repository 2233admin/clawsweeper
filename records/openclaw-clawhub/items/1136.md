## Summary

Keep open. Current main still lacks a dedicated false-positive or suspicious-skill appeal intake, GitHub issue-template routing, label automation for generic appeals, or ClawHub-native appeal flow; the existing rescan guidance is narrower than the requested issue separation, and ClawHub policy keeps issues open for maintainer judgment.

## What This Changes

The issue asks for a separate intake or filtering path for false-positive and suspicious-skill appeals so product issues are easier to find.

## Best Possible Solution

Maintainers should choose an official appeal intake path, then implement that separation consistently by updating the suspicious-skill owner banner and adding either GitHub issue templates/label automation or a ClawHub-native appeal/support workflow.
