## Summary

Keep open. This is a ClawHub issue, and repository cleanup policy only auto-closes PRs when current main definitely implements the change. The current main code still appears to omit skill-level moderation/security from basic `clawhub inspect` text output and from inspect JSON, while open draft PR #1855 is already the focused implementation candidate for the visibility part of the request.

## What This Changes

The issue asks for the CLI to expose a skill's security or moderation status, especially so users can identify or choose benign skills during installation.

## Best Possible Solution

Land or revise a focused CLI change that includes skill-level moderation in `clawhub inspect --json` and prints a concise security summary in normal inspect output, then decide separately whether benign-only install or search filtering belongs in the CLI workflow.
