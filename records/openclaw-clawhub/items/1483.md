## Summary

Keep open. This is an open ClawHub issue, and the review policy for openclaw/clawhub says issues should stay open for maintainer triage. Current main also does not fully implement the request: the v1 skill detail API/schema carries moderation data, install/update have partial malware/suspicious safeguards, but plain `clawhub inspect` and `--json` inspect do not expose the skill-level `moderation` block. An open draft PR, #1855, explicitly references this issue with closing syntax and should be reviewed or closed before the issue is resolved.

## Best Possible Solution

Keep this issue open for maintainer triage. Review and either land or close PR #1855 as the implementation candidate for `clawhub inspect` text/JSON moderation visibility; if it lands, close this only after deciding whether benign-only install/filter behavior is included or split into a narrower follow-up that likely wires existing `nonSuspiciousOnly` API support into CLI search/explore or install flows.
