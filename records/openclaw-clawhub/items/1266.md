## Summary

Keep open. The repository-specific cleanup policy says ClawHub issues should remain open, and the current implementation still has VirusTotal/status-mapping paths that can produce the reported user-facing “Suspicious” state without showing an engine detection ratio. The named public skill page also still shows the suspicious banner and VirusTotal “Suspicious” status, so this is not obsolete. ([clawhub.ai](https://clawhub.ai/ai-warden/ai-warden-setup))

## Best Possible Solution

Keep this issue open for maintainer review. The likely code path is to revisit `convex/vt.ts` skill VirusTotal verdict mapping, decide whether skill VT status should be based on AV engine detection counts instead of Code Insight when engines are 0 detections, and/or update `src/components/SkillSecurityScanResults.tsx` to show the engine detection ratio next to the status. For this specific skill, staff can also use the existing manual override flow if they review it and decide the suspicious flag is a false positive.
