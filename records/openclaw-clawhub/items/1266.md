## Summary

Keep open. Repository policy for this ClawHub cleanup pass keeps issues open, and the reported false-positive class is still not fully resolved on current main: VirusTotal Code Insight can still provide the visible suspicious status, while AV engine counts are stored only as supporting data and are not exposed as a first-class UI/API detection ratio.

## What This Changes

The issue asks ClawHub to base VirusTotal suspicious/benign status on AV detection counts, or at least display the 0/N detection ratio next to Code Insight-derived findings.

## Best Possible Solution

Define a canonical VirusTotal policy that separates AV engine detections from Code Insight heuristics, exposes engineStats or a computed malicious+suspicious/total ratio in the public API and UI, keeps Code Insight visible as contextual analysis, and provides a migration/rescan path for already scanned affected skills.
