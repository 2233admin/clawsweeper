## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review run only permits automated close recommendations for PRs that current main definitely implements. Substantively, current main has a targeted backend mitigation for clean-engine VirusTotal Code Insight false positives, but the public scan/API/UI path still does not expose the AV engine detection ratio requested here and can still present the stored VirusTotal status as Suspicious.

## Best Possible Solution

Keep this issue open for maintainer security-product review. The best path is to define a canonical VirusTotal policy that separates AV engine detections from Code Insight/heuristic findings, expose the engine detection ratio in the public API and UI, and add a rescan/backfill or reconciliation path for existing affected skill versions once the policy is chosen.
