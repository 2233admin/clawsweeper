## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a real security-product/API gap: moderation has a targeted suppression for clean-engine VirusTotal Code Insight false positives, but VirusTotal ingestion, the public scan API, and the UI still expose a stored Code Insight-derived suspicious status without a first-class AV engine detection ratio. The reporter’s current public scan endpoint also still returns a suspicious VirusTotal block for `ai-warden-setup` v1.4.1.

## What This Changes

The issue asks ClawHub to stop showing a red VirusTotal suspicious warning when the linked VirusTotal report has zero AV engine detections, or at least show the 0/N detection count beside the status.

## Best Possible Solution

Define a canonical VirusTotal policy that separates AV engine detections from Code Insight heuristic findings, exposes engineStats or a computed 0/N detection ratio in the public API and UI, keeps useful heuristic context visible, and provides a rescan/backfill or moderation reconciliation path for affected existing skill versions.
