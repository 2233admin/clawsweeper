## Summary

This issue should remain open. The ClawHub review policy only permits auto-closing pull requests that are certainly implemented on main, not issues. Independently, current main still has exact-slug preservation and bounded recent lexical fallback, but not the requested indexed normalized slug/display-name prefix recall path. PR #1256 remains the open implementation candidate.

## Best Possible Solution

Keep this issue open and review PR #1256 as the implementation candidate. The likely code path is to add indexed normalized lexical fields for skill slug and display name on skillSearchDigest, backfill them safely, merge direct prefix candidates into searchSkills before the bounded recent fallback, and add regression coverage for Midscene-style display-name prefix, partial slug, and space-separated direct-name queries.
