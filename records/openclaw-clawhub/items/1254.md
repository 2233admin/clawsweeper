## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still lacks the requested indexed normalized slug/display-name prefix recall path; PR #1256 is the focused open implementation candidate for this issue.

## What This Changes

Add direct normalized display-name and slug prefix recall for skill search so Midscene-style queries find matching skills even when vector recall misses.

## Best Possible Solution

Keep this issue open and review/land PR #1256, or land an equivalent focused implementation that adds normalized slug/display-name digest fields, active/non-suspicious prefix indexes, digest writes, a cursor backfill, direct prefix candidate merging in searchSkills, and regression tests for Midscene-style name prefixes, partial slug queries, space-separated names, and case-insensitive queries.
