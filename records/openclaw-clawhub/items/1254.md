## Summary

This issue should remain open. It is an issue in openclaw/clawhub, and repository policy only permits auto-closing PRs that are definitely implemented on main. Independently, there is an open implementation PR (#1256) with explicit closing syntax for this issue, so the issue should stay open until that PR is reviewed and merged or closed. Current main still shows searchSkills using vector search, exact slug injection, and bounded recent digest fallback, without the normalized slug/display-name prefix candidate path requested here.

## Best Possible Solution

Keep this issue open and review PR #1256 as the implementation candidate. The likely fix belongs in `convex/search.ts`, `convex/schema.ts`, and associated search-digest/backfill tests: add indexed normalized lexical fields for skill slug/display name, merge direct prefix candidates into `searchSkills`, preserve current vector ranking and exact slug behavior, and validate with regression tests for `Midscene`, partial slugs, and space-separated display-name queries.
