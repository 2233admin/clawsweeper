## Summary

Keep open. Current main still lacks an indexed normalized slug/display-name prefix recall path for skill search, and PR #1256 is already open as the focused implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review/land PR #1256, or land an equivalent focused implementation. The fix should add normalized slug and display-name lexical fields to `skillSearchDigest`, maintain them on digest writes and through a cursor backfill, add indexed active/non-suspicious prefix lookup, merge direct prefix candidates into `searchSkills` before the bounded recent fallback, and add regression tests for Midscene-style display-name prefix, partial slug, space-separated name, and case-insensitive queries.
