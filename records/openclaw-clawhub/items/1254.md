## Summary

Keep open. ClawHub cleanup policy keeps issues open unless a PR is already implemented on main, and current main still lacks the requested indexed normalized slug/display-name prefix recall path. The issue is also paired with open PR #1256 using closing syntax, so the right maintainer action is to review/land that PR or an equivalent focused fix.

## What This Changes

Search should add a direct lexical candidate path for skill display-name and slug prefixes so matching skills are found even when vector recall misses them.

## Best Possible Solution

Implement or merge a focused Convex search change that adds normalized slug/display-name fields to skillSearchDigest, index-backed active/non-suspicious prefix lookup, digest writes and cursor backfill, and regression tests for Midscene, Gaggiuino, and ScienceClaw-style case-insensitive prefix and space-separated queries.
