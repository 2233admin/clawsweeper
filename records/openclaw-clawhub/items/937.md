## Summary

Keep this issue open. ClawHub cleanup policy does not auto-close issues, and current main still shows a credible unresolved path for the reported false `local changes (no match)` result after install or force-update: the CLI fingerprints extracted local text files, downloaded skill archives include generated `_meta.json`, and registry resolution compares against fingerprints built from published version files.

## Best Possible Solution

Keep this issue open and fix the CLI/backend integrity comparison so fresh installs and immediate `update --all --force` followed by `update --all` compare the same canonical file set as registry fingerprints. The likely fix is to exclude generated install/download metadata such as `_meta.json` from local and zip fingerprinting, keep `.clawhub/origin.json` ignored, and add regression coverage for fresh install, force update, and immediate clean update. Maintainers may consolidate this with open related #169 if they want one canonical tracker.
