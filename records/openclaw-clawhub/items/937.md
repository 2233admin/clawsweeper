## Summary

Keep this issue open. The repository-specific cleanup policy says ClawHub issues should remain open, and current main does not provide enough evidence that the reported update false-positive is fixed. Current main appears to exclude `.clawhub/origin.json` from local fingerprints, but a plausible mismatch remains because downloaded skill archives include generated `_meta.json`, while published version fingerprints are computed from published files.

## Best Possible Solution

Keep this issue open and fix the CLI integrity comparison so fresh installs and immediate force-updates compare the same canonical file set as registry fingerprints. The likely implementation is to exclude generated archive/install metadata such as `_meta.json` and `.clawhub/origin.json` from local fingerprinting, then add regression coverage for install, `update --all --force`, and immediate `update --all` reporting clean/up-to-date.
