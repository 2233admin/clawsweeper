## Repair Work Prompt

Fix the ClawHub CLI update/sync false `local changes (no match)` behavior reported here and in #169/#178. Align the fingerprint inputs used for installed local directories, downloaded skill zips, and registry resolve: generated install/download metadata such as root `_meta.json` must not cause a mismatch after install or force-update, but real published text files, including legitimate root JSON files, must remain covered by integrity checks. Update both the `update` path and related sync fingerprint path if needed. Add regression tests for install/force-update followed immediately by normal update with no warning, and for a real local edit still producing the warning. Do not make a broad change that ignores all root JSON files, do not weaken malware/suspicious moderation checks, and do not touch unrelated CLI release or publishing behavior.
