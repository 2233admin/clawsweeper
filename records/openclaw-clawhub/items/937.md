## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing pull requests that are certainly implemented on current main, and this item is an issue. The underlying bug path also still appears credible on current main: update compares a local text-file fingerprint to registry fingerprints, while downloaded skill archives can include generated `_meta.json` that is not part of the published version file fingerprint.

## Best Possible Solution

Keep this issue open and fix the CLI/backend integrity comparison so a fresh install or immediate `update --all --force` followed by `update --all` compares the same canonical file set as the registry fingerprint. The likely fix is to exclude generated install/download metadata such as `_meta.json` from local and zip fingerprinting, keep `.clawhub/origin.json` ignored, and add regression coverage for fresh install, force update, and immediate clean update. Maintainers may also consolidate this with open related #169 if they want one canonical tracker.
