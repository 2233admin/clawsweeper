## Summary

Keep open. The repository policy keeps ClawHub issues open, and current main still has a credible source-level false-positive path: skill downloads inject root `_meta.json`, install/update extracts it, local fingerprinting includes root JSON files, but backend resolve compares against publish-time `version.files` fingerprints that do not include generated download metadata.

## Best Possible Solution

Keep this issue open and fix the CLI/backend integrity comparison so a fresh install or `clawhub update --all --force` immediately followed by `clawhub update --all` compares the same canonical published file set as the registry. The narrow fix is to exclude generated install/download metadata such as root `_meta.json` from local and zip fingerprinting, or stop injecting/extracting that metadata for skill installs, while preserving integrity coverage for real published text files. Add regression tests and cluster the fix with #169 and #178.
