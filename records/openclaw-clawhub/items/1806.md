## Summary

Current main does not implement this PR’s intended per-skill pinning workflow. The CLI has install/update/uninstall/list/explore, but no pin/unpin commands; lockfile schemas and types still contain only version and installedAt; update rewrites entries without pin checks; and list prints only slug/version. The PR remains the active implementation candidate for the linked open feature request #1768.

## Best Possible Solution

Keep this PR open for maintainer review. The concrete path is to review and revalidate the current two-commit branch against current main, including lockfile metadata semantics, direct-update rejection, update --all skip reporting, list output, docs, tests, and the Greptile UX follow-up, then merge it if acceptable so #1768 can be resolved by the shipped implementation.
