## Summary

Keep open. This is an open ClawHub issue, and the repository policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still hard-blocks non-interactive installs for suspicious skills unless `--force` is used, `trustedPublisher` only bypasses pending-scan auto-hide for new publishes, and the existing manual override path requires moderator action rather than proving this publisher or the listed skills have been cleared.

## Best Possible Solution

Keep this open for maintainer moderation review. Staff can use the existing audited per-skill manual override path for verified false positives now; if durable publisher-level clearance is desired, define explicit reviewed-publisher semantics with audit logging, security limits, and tests rather than treating the current pending-scan `trustedPublisher` flag as sufficient.
