## Summary

Keep this issue open. Current main supports choosing an org owner for new skill publishes, but existing skill transfers are still modeled and routed as user-to-user transfers, so the reported no-loss personal-to-org re-home path remains missing; the ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

Support moving existing skills from individual user ownership to organization ownership while preserving stats, aliases, canonical owner URLs, and install compatibility.

## Best Possible Solution

Implement a core publisher-target ownership transfer flow: resolve org handles as valid destinations, authorize request and acceptance through publisher membership roles, update skills and slug aliases without losing stats or redirects, keep compatibility for existing user-to-user transfer clients, and add regression coverage for user-to-org, org-to-user, and org-to-org transfers.
