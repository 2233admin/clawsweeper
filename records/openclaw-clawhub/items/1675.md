## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open, and current main still lacks org-targeted skill ownership transfers: transfer records, API payloads, CLI requests, and acceptance logic are still user-recipient based. Publish-as-org exists for new publishes, but it does not re-home an existing personal-owned skill to an org while preserving stats, aliases, and canonical owner URLs.

## Best Possible Solution

Keep this open for core implementation. The best path is to migrate skill ownership transfers from user-recipient records to publisher-target records, resolve both user and org publisher handles as valid destinations, authorize acceptance according to a maintainer-approved org role rule, update skills and slug aliases without losing stats or redirects, update HTTP/CLI/web contracts, and add coverage for user-to-org, org-to-user, and org-to-org transfers.
