## Summary

Keep this issue open. Current main supports publish-as-org for new skill publishes, but the reported no-loss transfer/re-home path from an existing personal-owned skill to an org publisher is still missing; the transfer model remains user-recipient based and ClawHub review policy keeps issues open.

## Best Possible Solution

Keep this open for core implementation. The best path is to migrate skill ownership transfers to publisher-target records, resolve org handles as valid destinations, authorize acceptance by owner/admin org roles, update skills and slug aliases without losing stats or redirects, preserve compatibility for existing user-to-user transfer clients where needed, and add regression coverage for user-to-org, org-to-user, and org-to-org transfers.
