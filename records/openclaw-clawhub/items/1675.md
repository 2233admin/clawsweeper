## Summary

Keep this issue open. Repository policy for openclaw/clawhub only allows automated close proposals for pull requests that current main certainly implements, and this item is an issue. The substantive gap also remains: transfer is still modeled as user-to-user, org publisher handles are not valid transfer recipients, and existing personal-owned skills do not have a no-loss re-home path to an org.

## Best Possible Solution

Keep this issue open for core implementation. The best path is to migrate skill ownership transfers from user-recipient records to publisher-target records, resolve org handles as valid destinations, authorize acceptance by the chosen org roles, update skills and slug aliases without losing stats or redirects, and add coverage for user-to-org, org-to-user, and org-to-org transfers. Existing publish-as-org support can remain, but it does not replace a safe re-home flow for already published skills.
