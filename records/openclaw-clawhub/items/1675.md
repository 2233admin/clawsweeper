## Summary

Keep this issue open. Current main still models skill ownership transfers as user-to-user, while org publishers are only supported in publish-as-org paths. The reported no-loss re-home flow from a personal owner to an org owner is still missing, and the ClawHub review policy keeps issues open rather than auto-closing them.

## Best Possible Solution

Keep this open for core implementation. The strongest path is to migrate skill transfers to publisher-target records, resolve both user and org handles as valid destinations, authorize acceptance by the appropriate org roles, update skills and slug aliases without losing stats or redirects, preserve compatibility for existing user-to-user transfers where needed, and add regression coverage for user-to-org, org-to-user, and org-to-org transfers. The existing publish-as-org flow should remain, but it does not replace safe re-homing for already published personal-owned skills.
