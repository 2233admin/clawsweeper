## Summary

Keep this issue open. Current main has owner-authorized soft-delete/restore through the HTTP API and `clawhub` CLI, plus staff management controls, but the ordinary signed-in web app still does not expose a discoverable owner delete/restore workflow. The repository policy for this review also keeps ClawHub issues open; only pull requests that are certainly implemented on main are auto-close candidates.

## Best Possible Solution

Keep the issue open and add a signed-in owner delete/restore workflow in the web app, likely in the skill owner panel and dashboard row actions. Reuse the existing owner-authorized soft-delete logic through a public owner mutation/action that derives identity server-side, and add an include-soft-deleted owner list or restore view so authors can recover hidden skills. Keep the CLI/API workaround documented until the web UI exists.
