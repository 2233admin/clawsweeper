## Summary

Keep this issue open. Current main supports owner/moderator/admin skill soft-delete and restore through backend, HTTP API, and CLI paths, but the signed-in web app still does not expose a discoverable owner delete/restore workflow. ClawHub policy also says issues are not auto-close candidates in this pass.

## Best Possible Solution

Keep this issue open and add a signed-in owner delete/restore workflow in the web app. The fix should reuse existing soft-delete semantics, derive identity server-side, authorize both direct owner and publisher-member ownership consistently with the current settings page, expose a safe confirmation UI from skill settings and/or dashboard row actions, and provide an owner-visible restore path without introducing hard-delete behavior.
