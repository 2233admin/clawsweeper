## Summary

Keep this issue open. Current main has owner-authorized API and CLI soft-delete/restore paths, but the ordinary owner-facing web UI still does not expose a direct delete or restore action, and the repository policy for ClawHub issues is to keep them open rather than auto-close them.

## Best Possible Solution

Keep this issue open and add a discoverable owner-facing soft-delete/restore workflow, likely in the skill owner panel and dashboard row actions. Reuse the existing owner-authorized soft-delete path or expose an equivalent public owner mutation, and keep the CLI/API workaround documented until the web UI exists.
