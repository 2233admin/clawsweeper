## Summary

Keep open. Under the ClawHub cleanup policy, issues stay open, and the underlying bug still appears present on current main: README links are rendered without a skill-file resolver while the skill page URL remains a non-directory owner/slug route. PR #1858 is the open draft implementation candidate and specifically references this issue, so the best next step is maintainer review or landing of that PR rather than closing this issue early.

## What This Changes

The issue requests that relative links in rendered skill documentation resolve within the skill’s own file namespace instead of escaping to unrelated owner-level URLs.

## Best Possible Solution

Keep this issue open and review or land PR #1858, or an equivalent fix on current main. The fix should rewrite safe relative README links to /api/v1/skills/{slug}/file?path=..., preserve external URLs, mailto/tel links, and hash anchors, reject traversal or malformed relative paths, and add regression coverage for both the resolver and rendered SkillDetailTabs output. Treat #1001 as part of the same fix cluster.
