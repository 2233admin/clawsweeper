## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. The reported relative-link behavior still appears present on current main, and draft PR #1858 is the open implementation candidate with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for relative links in rendered skill README documentation to resolve inside the current skill’s file namespace rather than against the owner/slug page URL.

## Best Possible Solution

Land PR #1858 or an equivalent focused fix that rewrites safe relative README links to the skill file API, rejects traversal or malformed relative paths, preserves non-relative links, and covers both the resolver and SkillDetailTabs rendering with regression tests.
