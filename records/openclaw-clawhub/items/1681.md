## Summary

Keep open. This is a ClawHub issue, and the repo cleanup policy only permits automated close proposals for ClawHub PRs that current main definitely implements. Current main still renders skill README links without a skill-file resolver, keeps skill pages at non-directory owner/slug URLs, and PR #1858 is open, draft, and unmerged as the implementation candidate.

## What This Changes

Fix relative links in rendered skill documentation so internal file references resolve within the skill namespace instead of navigating to unrelated owner-level or same-origin paths.

## Best Possible Solution

Keep this issue open and review or land PR #1858, or an equivalent fix on current main. The fix should rewrite safe relative README links to /api/v1/skills/{slug}/file?path=..., preserve external URLs, mailto/tel links, and hash anchors, reject traversal or malformed relative paths, and add regression coverage for both the resolver and rendered SkillDetailTabs output. Treat #1001 as the same fix cluster.
