## Summary

Keep open. This is an issue in openclaw/clawhub, and the repository policy for this review only permits automated close proposals for pull requests that are certainly implemented on current main. Substantively, current main still renders skill README links without resolving relative paths into the skill file API, and PR #1858 is an open, unmerged implementation candidate.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #1858 with a fix that rewrites skill README relative links to /api/v1/skills/{slug}/file?path=... while preserving external URLs, mailto/tel links, and hash anchors, rejecting traversal, and adding regression coverage for rendered README links.
