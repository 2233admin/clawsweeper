## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. The reported relative-link behavior still appears present on current main, and draft PR #1858 is already the open implementation candidate that specifically references this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks ClawHub to keep relative links in rendered skill documentation within the current skill’s file namespace instead of letting the browser resolve them against the non-directory owner/slug page URL.

## Best Possible Solution

Land PR #1858 or an equivalent focused fix that rewrites safe relative README links to the existing skill file API, rejects traversal or malformed relative paths, preserves non-relative links, and covers both the resolver and SkillDetailTabs rendering with regression tests. Then close this issue, and evaluate whether related #1001 is covered by the same shipped fix.
