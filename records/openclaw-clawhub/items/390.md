Keep open. Current main contains substantial fixes for the likely authenticated install rate-limit failure: installs forward the saved auth token, the download endpoint uses the shared rate limiter, valid Bearer-token requests are bucketed by user instead of shared IP, and 429 responses now include retry/budget guidance. However, this is an open ClawHub issue, and the repository cleanup policy for this pass only permits automated closure for pull requests that are certainly implemented on main. The reporter has not confirmed the latest CLI behavior, and broader rate-limit reports remain open.
