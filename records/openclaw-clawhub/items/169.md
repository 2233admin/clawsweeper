## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open unless they are PRs certainly implemented on main, and current main still has a credible false `local changes (no match)` path: install writes no clean fingerprint reference, update depends on registry hash resolution, downloaded zips include generated `_meta.json`, and registry fingerprints are computed from stored published files rather than generated download metadata. Related open reports #178 and #937 describe the same remaining behavior.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Fix the CLI/backend fingerprint contract so the exact bytes hashed after install or force-update are the same bytes the registry can resolve, either by excluding generated `_meta.json` from local/download fingerprints or by including generated metadata consistently in publish-time stored fingerprints, resolve, install, and update. Add a regression test for update immediately after fresh install or force update, then choose one canonical thread among #178, #937, and this issue after the fix lands.
