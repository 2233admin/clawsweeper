## Summary

Keep this issue open. The ClawHub cleanup policy says issues should not be auto-closed, and current main still contains the reported clean-install/update failure path: downloaded skills can include generated `_meta.json`, local fingerprinting includes that JSON file, while publish/resolve fingerprints are based on stored published files.

## Best Possible Solution

Fix the CLI/backend fingerprint contract, then add a regression test for install or update against a downloaded skill zip containing `_meta.json`. A minimal fix would either exclude generated `_meta.json` from local skill fingerprinting or include generated download metadata consistently in publish-time stored fingerprints, resolve, install, and update. After the fix lands, maintainers can choose one canonical thread among this issue, #178, and #937 and close the rest as implemented or superseded.
