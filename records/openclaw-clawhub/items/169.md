## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open, and current main still has a credible clean-install/update fingerprint mismatch path: installs extract a zip containing generated `_meta.json`, local hashing includes root JSON files, while publish/resolve fingerprints are based on stored published files without generated download metadata.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Fix the CLI/backend fingerprint contract by either excluding generated `_meta.json` from local and zip fingerprints, or including generated download metadata consistently in publish-time stored fingerprints, resolve, install, and update. Add a regression test for update immediately after fresh install or force update, then choose a canonical thread among this issue, #178, and #937 after the fix lands.
