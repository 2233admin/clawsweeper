## Clownfish Work Prompt

Fix the ClawHub skill update fingerprint mismatch reported here and in #178/#937. Define one canonical installed-skill fingerprint so a freshly installed skill, or a skill immediately after `clawhub update --force`, resolves as clean instead of `local changes (no match)`. Prefer a minimal fix that excludes only the generated root `_meta.json` from local/download fingerprint calculations while continuing to hash user-authored JSON and all other eligible text files; use an equivalent backend-side contract only if it is demonstrably safer. Update regression coverage for `packages/clawhub/src/skills.ts`, `cmdUpdate`, and sync/update behavior as needed. Do not weaken detection for real local edits, do not broadly ignore JSON files, and avoid unrelated CLI or registry changes.
