## Summary

Keep this draft PR open. Current main still lacks the PR's intended root-manifest recovery in the CLI publish path, and the PR remains a focused implementation candidate for the linked open bug about `SKILL.md required` failures.

## Best Possible Solution

Keep this draft PR open for maintainer review. The likely path is to land or adapt the localized `cmdPublish` root-manifest fallback, preserve shared `listTextFiles` ignore semantics, add the broad-markdown-ignore regression test, and close #1509 only after the fix merges and ships.

## Evidence

- **Current main publish path has no recovery step:** `cmdPublish` gets `filesOnDisk` directly from `listTextFiles(folder)` and then fails with `SKILL.md required` if that discovered list lacks a root `SKILL.md` or `SKILLS.md`.

A broad `*.md` rule can therefore remove `SKILL.md` from the publish upload list.
