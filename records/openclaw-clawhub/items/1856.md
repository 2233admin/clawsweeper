## Summary

Keep open. Current main still does not preserve a root SKILL.md/SKILLS.md manifest when broad ignore rules remove markdown files from `listTextFiles`, so this PR is still a live implementation candidate for the linked publish bug rather than obsolete work already shipped on main.

## Best Possible Solution

Keep this PR open for maintainer review. If the root-manifest ignore policy is accepted, land the focused CLI change and regression test, then close the linked publish bug after the fix is merged and released.

## Evidence

- **Current main publish path still lacks manifest recovery:** `cmdPublish` calls `listTextFiles(folder)` directly, then fails with `SKILL.md required` if the returned file list does not contain a root `skill.md` or `skills.md`. There is no fallback that re-adds a root manifest from disk.
