## Summary

Keep this draft PR open. Current main still does not implement the intended root-manifest recovery for `clawhub publish`: publish validation still depends on the ignore-filtered `listTextFiles(folder)` result, so a broad markdown ignore can still hide the required root `SKILL.md`/`SKILLS.md`. The PR remains a focused implementation candidate for the linked #1509 bug.

## Best Possible Solution

Keep this draft PR open for maintainer review. The likely path is to land or adapt the localized `cmdPublish` root-manifest fallback, preserve generic `listTextFiles` ignore semantics, add the broad markdown ignore regression test, decide the explicit ignore policy for the required manifest, and close #1509 only after the fix merges and ships.
