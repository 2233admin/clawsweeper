## Summary

Keep open. Current main still publishes exactly the files returned by `listTextFiles`, and that shared discovery path applies ignore rules before `cmdPublish` checks for `SKILL.md`/`SKILLS.md`, so the linked root-manifest failure mode remains possible. The PR is a focused draft implementation candidate and the visible diff does not introduce unrelated security-sensitive surfaces.

## What This Changes

The PR updates the ClawHub CLI publish command to recover a root `SKILL.md`/`SKILLS.md` manifest when broad ignore rules exclude it, and adds a regression test for `.gitignore` containing `*.md`.

## Best Possible Solution

Land or adapt the localized publish fallback so only root `SKILL.md`/`SKILLS.md` is recovered, keep shared `listTextFiles` ignore semantics and plugin-folder rejection intact, and close linked #1509 after the fix merges and ships.
