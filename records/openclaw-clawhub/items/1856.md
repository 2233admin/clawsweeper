## Summary

Keep open. This PR is not implemented on current main: the publish command still depends directly on listTextFiles output, and listTextFiles honors root .gitignore/.clawhubignore patterns before the SKILL.md required check. The PR appears to be a focused implementation candidate for open issue #1509, not obsolete main-branch work.

## Best Possible Solution

Keep this PR open for maintainer review. The likely path is to land this focused CLI change, possibly after confirming whether root manifest recovery should bypass .clawhubignore as well as .gitignore, then close the linked bug once the fix is merged and released.
