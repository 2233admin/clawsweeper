## Summary

Keep this issue open. The ClawHub cleanup policy says issues should not be auto-closed, and the current main branch still has a real moderation/search visibility UX path behind the report: quality-quarantined skills remain hidden from public search, and `clawhub inspect` does not surface the moderation reason even though owner-facing API support exists.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to wire owner-facing moderation diagnostics into `clawhub inspect` and/or the authenticated skill detail experience, including `quality.low` explanations and recovery steps, while coordinating the stale tags/SKILL.md portion with #1407. Search should continue excluding genuinely hidden skills, but publishers need a clear reason when their own skill disappears.
