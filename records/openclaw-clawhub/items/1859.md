## Summary

Keep open. Current main does not yet implement the docs, CLI schema, and prompt alignment this draft PR targets, and the branch is still the active implementation path for the related optional-env issue. The patch also needs a change before merge because its new scanner regression covers a top-level envVars shape while the new docs tell authors to use the nested OpenClaw metadata shape.

## What This Changes

The PR documents optional skill environment variables, adds envVars support to the CLI schema copy, updates security prompt wording/output, and adds schema plus moderation regression tests.

## Best Possible Solution

Land this PR or a replacement after the docs, CLI schema copy, security prompt, static scanner, and tests all agree that optional skill environment variables are declared with envVars entries using required: false, including coverage for the raw metadata.openclaw.envVars publish-scan path.
