## Summary

Keep this PR open. Current main still only partially covers optional skill environment variables: the shared/backend schema and some moderation paths understand `envVars`, but the public skill-format docs, the `packages/clawhub` CLI schema copy, the assembled security-evaluator requirements text, and focused regression coverage do not yet implement this PR's intended cleanup for #1617.

## Best Possible Solution

Keep this draft PR open and review it as the implementation candidate for #1617. The best path is to land one consistent optional-env policy, likely `metadata.openclaw.envVars` entries with `required: false`, across public docs, the CLI schema copy, security prompt text, static moderation behavior, and focused regression tests.
