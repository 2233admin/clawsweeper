## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and this item is an issue rather than a PR. Current main does address the narrow CLI bug by forwarding the optional saved token from both commands into the HTTP client, but a maintainer still needs to decide whether the requested scope ends there or should become a separate backend visibility/API issue.

## What This Changes

The issue asks for `clawhub search` and `clawhub explore` to include the saved authentication token when calling the backend.

## Best Possible Solution

A maintainer should close this manually if CLI token forwarding fully resolves the report, or split a focused backend/API issue if authenticated skill search and explore should expose owner-visible non-public content.
