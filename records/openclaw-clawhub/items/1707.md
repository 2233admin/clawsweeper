## Summary

Keep open under the ClawHub cleanup policy for issues. Current main fixes the narrow CLI token-forwarding bug for both `search` and `explore`, with regression coverage and release-note provenance, but maintainer disposition is still required and the broader authenticated catalog semantics are not clearly settled.

## What This Changes

The issue reports that `clawhub search` and `clawhub explore` did not send the saved authentication token while other CLI commands did.

## Best Possible Solution

Keep the shipped CLI token forwarding. A maintainer should manually close the CLI-token part as fixed, or open a separate focused backend/API issue that defines authenticated owner-visible search and explore semantics.
