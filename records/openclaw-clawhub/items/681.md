## Summary

Keep this issue open. It is a concrete production registry ownership/deletion support request, and the ClawHub review policy for this run only permits auto-closing pull requests that are certainly implemented on main. Current main has owner/staff soft-delete, transfer, admin owner-change, admin reclaim, and admin hard-delete paths, and the live public API shows Gate-owned successor/current listings for the requested names, but the discussion still lacks requester or maintainer confirmation that the original sub-account cleanup is fully resolved.

## Best Possible Solution

Keep this issue open for maintainer/support follow-up. A maintainer should ask Gate to confirm whether the current `gate-exchange` listings are the intended replacements and whether any old sub-account records remain. If anything is still blocked by old ownership, staff can use the existing admin owner-change, reclaim, owner/staff soft-delete, or admin hard-delete paths, then close manually once the production state is confirmed.
