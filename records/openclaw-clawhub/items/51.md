## Summary

Keep this ClawHub issue open for maintainer release triage. Current main and the current `clawhub@0.11.0` package declare `undici`, but the originally reported legacy npm package still has `clawdhub@latest` pointing at `0.3.0`, whose registry metadata lacks `undici`.

## Best Possible Solution

Keep this issue open for maintainer release triage. The best path is to decide what to do with the legacy `clawdhub` npm package: promote or publish a fixed legacy version, move the `latest` dist-tag off `0.3.0`, deprecate the broken version with migration guidance to `clawhub`, and then consolidate related reports #23 and #33.
