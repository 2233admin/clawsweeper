## Summary

Keep this PR open. Current main still does not enforce `metadata.os` during `clawhub install`, and the PR is not yet merge-ready because it checks raw Node platform values, validates latest-version metadata before requested-version resolution, makes CLI response metadata required, and changes `--force` semantics without a clear policy decision.

## Best Possible Solution

Keep this PR open and revise it rather than closing it as obsolete. A mergeable version should parse skill metadata as optional and nullable, normalize host/platform aliases to the documented metadata vocabulary, validate OS requirements for the resolved version when `--version` is used, make a maintainer-approved decision about whether `--force` bypasses OS mismatch, and add focused `cmdInstall` tests for allowed, rejected, forced, missing-metadata, alias, and requested-version cases.
