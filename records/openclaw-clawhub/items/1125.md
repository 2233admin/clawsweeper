## Summary

Keep this PR open. Current main still does not implement install-time OS compatibility checks for `clawhub install`, and the PR remains relevant, but the current diff is not ready to close or merge because it compares raw Node platform names to documented metadata values, checks latest-version metadata before resolving requested versions, makes metadata required in the CLI schema, and lacks focused install tests. Security review did not find unexplained supply-chain or CI changes; the diff is limited to CLI/schema files.

## Best Possible Solution

Keep the PR open and revise it. The maintainable path is to parse platform metadata as optional and nullable, normalize host platform aliases to the documented `metadata.os` vocabulary, expose and validate metadata for the resolved version when `--version` is used, decide explicitly whether `--force` may bypass OS incompatibility, and add focused `cmdInstall` tests for the compatibility paths.

## Risks / Open Questions

- Merging the current diff as-is can incorrectly reject supported macOS and Windows users because Node reports `darwin` and `win32` while skill metadata is documented as `macos` and `windows` style values.
- Checking latest-version metadata before resolving `--version` can allow or reject the wrong release when OS requirements differ across versions.
- Making `metadata` required in the CLI response schema could break installs for API responses that omit the field; it should be optional and nullable.
- Changing `--force` to bypass OS mismatch is a product/safety policy decision that should be explicit and tested.
- There are no focused install tests proving the allowed, rejected, forced, missing-metadata, or requested-version paths.
