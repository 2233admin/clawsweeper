## Summary

Keep this PR open. Current main still does not enforce `metadata.os` during CLI install, and the PR remains useful but needs revision before it is merge-ready: optional metadata parsing, OS alias normalization, requested-version metadata handling, tests, and maintainer policy on `--force`.

## What This Changes

The PR adds an OS compatibility check to the CLI install flow by parsing `metadata.os`, comparing it with `node:os` platform values, and updating the `--force` help text.

## Best Possible Solution

Keep this PR open and revise it. A mergeable version should parse metadata as optional/nullable, normalize host platform aliases to the documented metadata vocabulary, validate OS requirements for the resolved version when `--version` is used, add focused `cmdInstall` tests, and settle whether `--force` should bypass OS incompatibility.

## Risks / Open Questions

- Merging the PR as-is could falsely block documented macOS and Windows skills because Node reports `darwin` and `win32`, while skill metadata examples use `macos` and `windows`.
- The PR schema makes `metadata` required instead of optional/nullable, which risks breaking installs against registries or responses that omit the key.
- The proposed check runs before requested-version resolution and would validate latest-version metadata rather than the version being installed with `--version`.
- The `--force` bypass for OS incompatibility changes install-safety semantics and should get explicit maintainer agreement.
