## Summary

Keep this PR open. Current main still does not implement install-time OS compatibility checks for `clawhub install`, and the PR remains relevant, but the current diff is not ready to close or merge because it needs platform normalization, version-specific metadata handling, optional metadata parsing, and tests.

## Best Possible Solution

Keep the PR open and revise it. The maintainable path is to parse skill response metadata as optional and nullable, normalize host platform aliases to the documented `metadata.os` vocabulary, validate compatibility against the resolved version when `--version` is used, decide explicitly whether `--force` should bypass OS mismatches, and add focused `cmdInstall` tests for the main compatibility paths.

## Risks / Open Questions

- Merging the current PR as-is could reject supported macOS or Windows users because Node reports `darwin` and `win32`, while skill metadata is documented as `macos` and `windows` style values.
- Checking latest-version metadata before resolving `--version` can allow or reject the wrong release when OS requirements differ across versions.
- Making `metadata` required in the CLI response schema could break installs for API responses that omit the field; it should be optional and nullable.
- Treating `--force` as an OS mismatch bypass changes install safety policy and should be an intentional maintainer decision.
- No focused install tests currently prove the allowed, rejected, forced, missing-metadata, or requested-version cases.
