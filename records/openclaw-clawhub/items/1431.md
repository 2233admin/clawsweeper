## Summary

Keep open. The ClawHub cleanup policy for this pass does not auto-close issues, and current main still matches the reported behavior: the publish UI can submit an organization publisher for an existing skill, but `insertVersion` rejects the existing slug when the selected publisher differs from the stored `ownerPublisherId`. The dedicated transfer path is also still user-to-user/personal-publisher based, while related org-transfer PRs were closed unmerged and #1569 remains open.

## What This Changes

The issue requests support for moving an existing skill from a personal publisher to an organization publisher without hitting the slug-collision guard during republish.

## Best Possible Solution

Land a maintainer-approved publisher-target ownership-transfer design that consistently updates the skill row, slug aliases, public URL/API behavior, CLI behavior, docs, and regression tests. If republish should not move ownership, the existing-skill publish UI should stop offering owner changes and route users to the supported transfer flow.
