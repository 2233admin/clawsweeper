## Summary

Keep this issue open. The ClawHub cleanup policy for this repository only allows auto-closing pull requests that are certainly implemented on current main, and this item is an issue. The current code also still matches the report: the UI can submit a different owner publisher while republishing, but the backend rejects an existing skill when its stored owner publisher differs from the selected publisher, and the transfer flow remains user-to-user rather than organization/publisher-targeted.

## Best Possible Solution

Keep this issue open for maintainer/product handling. The likely fix is either an explicit authorized publisher-change path in the publish/version flow, or an org-aware transfer flow that accepts publisher targets, updates skills.ownerPublisherId and skillSlugAliases consistently, and exposes the behavior through the web UI, HTTP API, CLI, tests, and docs. Review #1562, #1388, #1603, and #1604 as implementation references, and consider consolidating with #1569 if maintainers prefer one canonical tracking issue.
