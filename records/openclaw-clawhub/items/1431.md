## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Current main also still rejects the reported flow: the publish UI can submit a different owner, but `insertVersion` treats an existing skill with a different `ownerPublisherId` as a slug collision, and the transfer model remains user-to-user rather than publisher/org-targeted.

## Best Possible Solution

Keep this issue open for maintainer/product handling. A robust fix would add an authorized publisher-change path in the publish/version flow or an org-aware transfer flow that accepts publisher targets, updates `skills.ownerPublisherId` and `skillSlugAliases` consistently, and covers the web UI, HTTP API, CLI, tests, and docs. Review #1562, #1388, #1603, and #1604 as implementation references, and consider consolidating with #1569 if maintainers want one canonical tracking issue.
