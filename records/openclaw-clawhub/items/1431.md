## Summary

Keep open. The repository cleanup policy for ClawHub keeps issues open, and the current main implementation still matches the reported failure path: the publish UI can submit a different publisher for an existing skill, while the backend rejects an existing skill whose stored publisher differs from the selected publisher. The existing transfer path also remains user-to-user/personal-publisher based, so the org ownership transfer problem is still real and should be handled with the related ownership-transfer work.

## What This Changes

Allow an existing skill owner to move a skill from a personal publisher to an organization publisher, either through republish or a dedicated ownership-transfer flow.

## Best Possible Solution

Keep the issue open and resolve it through a maintainer-approved publisher-target ownership-transfer design that updates skills, slug aliases, public URLs, HTTP API, CLI behavior, docs, and tests consistently. If republish is not the supported transfer mechanism, the publish UI should not offer an owner change for existing skills without routing users to the dedicated transfer flow.
