## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy for this pass only permits auto-closing pull requests that current main definitely implements. Current main still has the reported product gap: the publish UI can submit an organization owner for an existing skill, but `insertVersion` rejects an existing skill when `ownerPublisherId` differs, and the dedicated transfer flow remains user-to-user rather than publisher/org-targeted.

## Best Possible Solution

Keep this issue open and route it with #1569 and open PR #1635. The best fix is an authorized publisher-change or ownership-transfer design that handles org targets consistently across `skills.ownerPublisherId`, `skillSlugAliases`, web publish/edit behavior, HTTP API, CLI, docs, and regression tests; if maintainers choose not to allow ownership moves through republish, the UI should stop presenting that path and direct users to the transfer flow.
