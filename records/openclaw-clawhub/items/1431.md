## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only permits auto-closing pull requests that current main definitely implements. Substantively, current main still exposes the org-capable publish owner selector but rejects an existing skill when the selected publisher differs from the stored owner publisher, and the transfer path remains user-to-user/personal-publisher based. The best route is to review, land, or supersede the related open ownership-transfer PR rather than close this issue.

## What This Changes

Support changing an existing published skill from a personal publisher to an organization publisher without hitting the slug-collision path.

## Best Possible Solution

Keep this issue open and route it with #1569 and open PR #1635. The best fix is a maintainer-approved publisher-target ownership-transfer flow that supports org targets consistently across `skills.ownerPublisherId`, `skillSlugAliases`, publish/edit UI, HTTP API, CLI, docs, and tests. If ownership moves should not happen through republish, the publish UI should stop offering an owner change for existing skills and point users to the dedicated transfer flow instead.
