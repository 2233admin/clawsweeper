## Summary

Keep open. Current main still contains the broad standalone `swap` crypto capability matcher and lacks the proposed `display=swap` and token-swap regression tests, so this draft PR is still the active implementation path for the linked false-positive report rather than work already implemented on main. A security pass found the PR scope limited to deterministic capability matching and tests.

## What This Changes

This PR narrows crypto capability detection by replacing the standalone `swap` regex with contextual crypto-adjacent swap patterns and adds regressions for Google Fonts `display=swap` and token-swap wording.

## Best Possible Solution

Land this PR or an equivalent focused change that removes the generic standalone `swap` trigger, preserves real crypto swap detection, adds both negative and positive regression coverage, and then lets the linked false-positive issue close through the normal merge path.
