## Summary

Keep this PR open. Current main only partially overlaps with the proposed stats cleanup: SkillHeader uses a Download icon for downloads, but it still renders stars first, keeps current/all-time installs as separate History rows, and has no installs tooltip utility or focused tooltip tests. Shared SkillStats, SoulStats, and SkillListItem surfaces still use Package for download counts and older ordering/style, so the PR is not implemented on main under the ClawHub PR cleanup policy.

## What This Changes

The PR branch updates skill and soul stat UI semantics across SkillHeader, SkillStats, and SoulStats, adds an installsTooltip utility with tests, and adds skill-detail regression coverage for current/all-time install edge cases.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a clean scoped update against current main. The best implementation would consistently apply semantic download/user icons and downloads-first ordering across SkillHeader, SkillStats, SoulStats, SkillListItem, and related list surfaces; keep both current and all-time install information visible or accessibly exposed; add focused regression tests for install edge cases; coordinate the narrower download-icon work with #1854; and avoid unrelated backend, moderation, security, or formatting churn.
