## Summary

Keep this PR open. Current main only partially overlaps with the proposed stats semantics cleanup: SkillHeader uses a Download icon for downloads, but it still renders stars before downloads, keeps current/all-time installs as separate History stats, and has no installs tooltip utility. The shared skill, soul, and public list stat surfaces still use Package for downloads and keep the older ordering/style. The related draft PR #1854 is narrower download-icon work and does not supersede the broader installs/order/test changes here.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a clean scoped update against current main. The best path is to consistently apply semantic download/user icons and downloads-first ordering across SkillHeader, SkillStats, SoulStats, SkillListItem, and related public list surfaces; keep both current and all-time install information visible or accessibly exposed; add focused regression tests for install edge cases; coordinate the narrower download-icon work with #1854; and avoid unrelated backend, moderation, security, or formatting churn.
