## Summary

Keep open. This is an issue in openclaw/clawhub, where the provided review policy only permits auto-closing PRs that are certainly implemented on main. The current main code also still has the reported behavior: static publishing scans every uploaded text/code file, child_process exec remains a suspicious static finding, and capability tags are still broad keyword-derived signals that can over-match cryptographic/security wording.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The practical path is to use the existing moderator override and capability-tag tools to clean up the jimcollinson/x0x listing after review, then make a focused code change that scopes static scanning to the declared install/runtime surface or safely classifies non-skill language bindings. Capability extraction should be refined toward declared/registered authority and covered with regression tests for the x0x-style cryptography/keypair case plus true-positive crypto, wallet, payment, OAuth, and sensitive-credential skills.
