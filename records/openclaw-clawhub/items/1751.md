## Summary

Keep open. ClawHub policy keeps issues open for maintainer judgment, and the current main code has not implemented the requested scanner scoping or bindings exclusion. The live scan endpoint still shows v0.17.4 with the reported static `suspicious.dangerous_exec` finding and all six requested capability tags, while the current latest release has moved to v0.19.5 with different scan signals.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best path is to review x0x as a specific listing, use the moderator capability-tag override if the chips are confirmed false positives, and decide whether to add a code change that scopes static analysis to declared install/runtime surfaces or introduces a narrower safe suppression for `bindings/**`. If the code changes, add regression tests for a post-quantum/cryptographic networking skill that should not receive cryptocurrency, wallet, purchase, transaction, OAuth, or sensitive-credential tags, then rescan/backfill the affected version data.

## Risks / Open Questions
