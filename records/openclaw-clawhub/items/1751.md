## Summary

Keep open. This is an issue in openclaw/clawhub, and the repository policy for this review only permits auto-closing pull requests that are certainly implemented on main. Independent of that policy, current main still appears to have the reported behavior: publish scanning reads all uploaded text files, child_process exec remains a suspicious static finding, and capability tags are still derived from broad keyword patterns across metadata, README text, and file contents.

## Best Possible Solution

Keep this issue open for maintainer follow-up. A practical path is to use the existing moderator override tools to clean up jimcollinson/x0x after review, then land a focused code change that scopes static scanning to the declared install/runtime surface or explicitly classifies non-skill language bindings. The capability extractor should also distinguish cryptography/keypair/security terminology from cryptocurrency wallets, payments, OAuth grants, and external credentials, with regression tests for an x0x-style case plus true-positive crypto/payment/OAuth skills.
