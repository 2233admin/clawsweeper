## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review only permits auto-closing PRs that are already implemented on main. Independently of that policy, current main still does not implement the requested scanner/tagging fix: static publish scanning still scans every uploaded text/code file, child_process exec remains a suspicious static finding, and capability tags are still broad keyword-derived signals that can confuse cryptographic/security wording with crypto/payment/wallet/OAuth authority.

## Best Possible Solution

Keep this issue open for maintainer follow-up. A pragmatic path is to use the existing moderator override and capability-tag tools to clean up jimcollinson/x0x immediately after review, then make a focused code change that scopes static scanning to the declared install/runtime surface or explicitly classifies non-skill language bindings, and refine capability extraction so cryptography/keypair/security terminology is not conflated with cryptocurrency wallets, payments, OAuth grants, or sensitive external credentials. Add regression tests for the x0x-style case and for true-positive crypto/payment/OAuth skills.
