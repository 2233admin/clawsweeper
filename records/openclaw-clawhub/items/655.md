Keep open. This is a ClawHub issue, and the repo cleanup policy only permits auto-closing ClawHub pull requests that are definitely implemented on main. Substantively, the live Task Specialist listing is still suspicious despite clean VT/LLM scans, and current main still has a static scanner path that can flag comment-only `eval()` text as `suspicious.dynamic_code_execution`.

## Best Possible Solution

Keep this issue open for maintainer/security triage. The best code path is a narrow static-scanner change that ignores comment-only `eval()` mentions while retaining real dynamic-code findings, plus regression coverage for shell comments and actual executable eval usage. After deployment, rescan or backfill Task Specialist; if maintainers judge the artifact acceptable sooner, use the existing staff clean override path with an audit note.
