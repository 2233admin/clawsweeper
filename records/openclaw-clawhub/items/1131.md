## Summary

Keep this issue open. ClawHub repository policy keeps issues open, an open implementation PR is still paired with this issue, and current main still lacks the requested moderation category/context mechanism for security tools.

## Best Possible Solution

Keep this issue open and have maintainers review PR #1132 and PR #1133, or replace them with a narrower maintained design. The likely fix belongs in the moderation reason-code pipeline, static scanner snapshot logic, LLM evaluation context/prompt, skill-format/security docs, and an admin recovery path for existing moderation holds, with tests proving malicious codes are never suppressed.
