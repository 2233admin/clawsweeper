## Summary

Keep open. ClawHub cleanup policy keeps issues open, this issue is paired with open implementation PRs, and current main still lacks the requested declared security-tool category/context mechanism. Current main has a narrower external-scanner demotion path for one suspicious code, but shell execution, file-read plus network-send, obfuscated payload, and nonstandard network findings are still emitted as direct suspicious codes without category context.

## What This Changes

The issue requests a declared security-tool category for skills so moderation can contextualize expected scanner/audit behavior while preserving malicious-code safety floors and LLM misdirection checks.

## Best Possible Solution

Keep this issue open and resolve it through maintainer review of PR #1132 and PR #1133, or a replacement design that parses declared security-tool category metadata, contextualizes only allowlisted static reason codes, preserves malicious and non-contextual safety floors, passes category context to the LLM with false-category guidance, and documents/tests moderation-hold recovery.
