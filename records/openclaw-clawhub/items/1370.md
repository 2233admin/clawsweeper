## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still only has partial plugin package lifecycle support: package/release soft-delete state and a package-wide delete API exist, but owner-facing hide, restore, version-level delisting, CLI, UI, and docs coverage remain incomplete.

## What This Changes

Add owner-facing lifecycle controls for ClawHub plugin packages so owners can hide, delete, restore, or delist broken package releases without maintainer intervention.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement an owner/admin workflow across Convex mutations, HTTP routes, clawhub CLI commands, dashboard or plugin management UI, tests, and public docs, coordinating version-specific behavior with #905 and stale plugin cleanup needs from #1744.
