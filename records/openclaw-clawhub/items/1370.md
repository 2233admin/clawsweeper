## Summary

Keep open. The supplied ClawHub policy keeps issues open, and current main still only has partial package lifecycle plumbing: package/release soft-delete fields and a package DELETE API exist, but owner-facing package hide/undelete controls are missing from the CLI, site UI, restore API, version-level workflow, and docs.

## What This Changes

Request owner/admin self-service hide, delete, undelete, and possibly version de-listing controls for ClawHub plugin packages across API, CLI, website UI, and docs.

## Best Possible Solution

Keep this issue open and have maintainers define package-level versus release-level lifecycle semantics first. Then implement the chosen owner/admin workflow end to end across Convex package mutations/API, clawhub CLI commands, owner dashboard or management controls, tests, and public docs, while coordinating version-specific behavior with #905 and stale plugin cleanup needs from #1744.
