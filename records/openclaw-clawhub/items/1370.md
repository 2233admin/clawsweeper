## Summary

Keep this issue open. Current main has package and package-release soft-delete storage plus a package DELETE API, but it still lacks the requested owner-facing plugin package lifecycle workflow: package restore/undelete, package hide/unhide commands, version-level delisting, dashboard controls, and docs. The ClawHub cleanup policy for this review also says issues stay open; only PRs may be auto-closed as implemented on main.

## Best Possible Solution

Keep this issue open and have maintainers define package-level versus release-level lifecycle semantics first. Then implement the chosen workflow end to end across Convex package mutations/API, clawhub CLI commands, owner dashboard controls including a recovery path, tests, and public docs, while coordinating version-specific behavior with #905 and stale plugin cleanup needs from #1744.
