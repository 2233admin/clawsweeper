## Summary

Keep this issue open. The ClawHub cleanup policy says issues should not be auto-closed, and current main still only partially covers the requested owner-facing plugin package lifecycle: backend package soft-delete and a DELETE API exist, but restore/undelete, hide/unhide CLI commands, dashboard controls, docs, and per-version de-list semantics are still missing.

## Best Possible Solution

Keep this issue open and finish plugin package lifecycle support end to end: define package-level versus release-level semantics, add package restore/undelete and optional version de-list backend/API paths, expose owner-facing CLI commands and dashboard controls, include a deleted/hidden package recovery view, and document the rollback workflow for plugin publishers. Coordinate version-specific behavior with #905 and validate it against cleanup cases like #1744.
