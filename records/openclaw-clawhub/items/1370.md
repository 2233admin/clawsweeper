## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still has only partial package lifecycle plumbing: package/release soft-delete fields and a package DELETE API exist, but owner-facing package restore/hide controls are missing from the CLI, public owner UI, package restore API, and docs.

## What This Changes

Add owner-facing website, CLI, API, and docs support to hide/delete/undelete ClawHub plugin packages, with possible per-version de-listing for bad package releases.

## Best Possible Solution

Define the package-level and release-level lifecycle model, then implement an owner/admin workflow end to end across Convex mutations and HTTP routes, clawhub CLI commands, owner dashboard or management UI, tests, and public docs while coordinating the version-specific behavior with #905 and stale plugin cleanup needs from #1744.
