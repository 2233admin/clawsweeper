## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. Current main also does not prove the requested unflagging happened: the static scanner still emits the appealed shell-exec and file-read-plus-network findings, metadata support is narrower than the reporter’s expanded v1.29.5 declaration model, and related security report #95 remains open for maintainer review.

## Best Possible Solution

Keep this issue open for maintainer/security follow-up. Maintainers should review the current evolver and capability-evolver artifacts, coordinate with #95, and if the reviewed versions are acceptable, apply the existing manual clean override to the exact skill versions. Separately, decide whether ClawHub should support a documented developer/system-tool metadata model that the moderation pipeline actually consumes, or document the narrower supported declaration schema and keep case-by-case overrides for high-power tools.
