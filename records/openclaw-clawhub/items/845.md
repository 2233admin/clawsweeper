## Summary

Keep open. This is a ClawHub issue, and the repository-specific policy for this cleanup run says ClawHub issues should remain open. Substantively, the report is an account-specific moderation/ban appeal that requires staff review of production account state. Current main has code paths that plausibly match the reported symptoms: static malicious findings can place a publisher under a `requiresModerationAt` moderation hold and hide all owned skills, while full bans block sign-in/API access as `Unauthorized`. Main has a ban/unban path, but I found no merged `liftModerationHold`/moderation-hold recovery path; the related recovery PR #1133 remains open in the provided GitHub context.

## Best Possible Solution

Keep this issue open for maintainer triage. Staff should inspect the production user, audit logs, hidden skill records, and scanner findings for the reported account, then either manually restore access/skills or land a reviewed admin recovery path like the open moderation-hold lift work in #1133. If maintainers want one canonical tracker, consolidate this report into the broader false-positive ban cluster without closing until the account-specific recovery outcome is resolved.
