## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. It is also still substantively valid: live package-version data for 0.1.2 and 0.1.3 still reports a suspicious LLM verdict caused by the same metadata mismatch, while the published npm metadata and plugin manifest declare the MongoDB credential and npm install metadata.

## Best Possible Solution

Keep this issue open. The best implementation path is to normalize packageJson.openclaw.install.npmSpec and openclaw.plugin.json credential hints, including providerAuthEnvVars, setup.envVars, and configSchema.required, into the metadata supplied to package LLM evaluation: requires.env, primaryEnv, envVars, and install. Add regression coverage for a MongoDB code plugin that declares MONGODB_URI through package/plugin metadata, then rerun or backfill scans for @kansodata/kansodata-mongodb-plugin 0.1.2 and 0.1.3 while preserving real suspicious/malicious detections.
