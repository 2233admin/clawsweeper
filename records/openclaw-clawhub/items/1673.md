## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this cleanup only permits auto-closing ClawHub PRs that current main definitely implements. The underlying report also remains substantiated: production still has suspicious OpenClaw LLM analysis for 0.1.2 and 0.1.3 based on the exact metadata mismatch the reporter describes, while current main does not normalize package.json/openclaw.plugin.json credential and npm install metadata into the package LLM review context.

## Best Possible Solution

Keep this issue open. A maintainer should normalize extractedPackageJson and extractedPluginManifest into the package review metadata used by static scan and SkillEvalContext: map providerAuthEnvVars/setup.envVars/configSchema required credential hints into requires.env, primaryEnv, and envVars, and map packageJson.openclaw.install.npmSpec into an install spec. Then rerun or backfill LLM/static reviews for @kansodata/kansodata-mongodb-plugin 0.1.2 and 0.1.3 and add a regression test for a MongoDB code plugin that declares MONGODB_URI through openclaw.plugin.json/package metadata.
