## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, current main still matches the reported lockout and hidden-skill symptoms: banned/deleted/deactivated users get Unauthorized on token auth, OAuth reauth blocks banned users, automated malware moderation can revoke tokens and hide skills, and moderation holds still hide future publishes. The related open recovery PR #1133 is an implementation candidate, not proof that this account was reviewed or restored.

## Best Possible Solution

Keep this issue open for ClawHub staff triage. Staff should inspect the production account and audit records, apply the existing unban flow if this is a deletedAt/deactivated account ban, and review or land a dedicated moderation-hold recovery path such as #1133 if the user is under requiresModerationAt. Skills hidden only by the user-level moderation hold should be restored after review, while individually malicious or separately moderated skills should remain hidden.
