## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on main. Current main also still supports the reported behavior: deleting a skill soft-deletes it without freeing the global slug, publishing checks that global slug against the target publisher, and publisher-target transfers are still documented as future work rather than a shipped self-service path.

## Best Possible Solution

Keep this issue open for maintainer triage. A maintainer should handle the specific pixverse production ownership request, then choose a product path: document that deleting a skill does not free its slug and route users to support, or add a safe self-service undelete/transfer/reclaim flow that can move a soft-deleted personal skill slug to an authorized company publisher.
