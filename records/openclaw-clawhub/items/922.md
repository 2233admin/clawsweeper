## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only permits automated closure for PRs that are certainly implemented on current main. Source review also confirms the reported product gap remains plausible: soft deletion keeps the global slug row, publishing blocks reuse when the slug belongs to a different publisher, and the shipped transfer/reclaim paths are not a self-service personal-to-company slug reclaim flow.

## Best Possible Solution

Keep this issue open for maintainer triage. A maintainer should handle the specific `pixverse` production ownership request, then choose and document the product path: either state that deleting a skill does not free its slug and route reclaim requests to support, or implement a safe publisher-aware undelete/transfer/reclaim flow that can move a soft-deleted personal skill slug to an authorized company publisher.
