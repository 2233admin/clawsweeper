## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy says ClawHub issues should not be auto-closed. Current main also shows the reported problem is plausible: skill deletion is a soft-delete that leaves the slug row in place, hard-delete reserves the slug for the previous owner, and publishing under a different publisher still treats an existing slug as taken. The related local reports are account/moderation recovery issues, not a canonical duplicate for this personal-to-company slug ownership case.

## Best Possible Solution

A maintainer should handle the specific `pixverse` ownership request in production, then decide the product path: document that delete does not free a slug, guide users to undelete/transfer before moving ownership, or add a safe self-service flow to transfer a soft-deleted personal skill slug to an authorized company/org publisher. If the intended path is admin support only, expose a clear user-facing message that points to that support channel.
