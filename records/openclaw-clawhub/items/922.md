## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this cleanup pass only permits auto-closing pull requests that current main certainly implements. Re-checking current main confirms the reported personal-to-company slug reuse is still blocked: skill slugs are globally indexed, soft deletion preserves the slug and publisher ownership, publishing rejects an existing slug owned by a different publisher, and the available reclaim/transfer paths do not provide a company-publisher self-service path.

## What This Changes

The issue asks how to reuse the `pixverse` skill slug under a company account after deleting it from a personal account.

## Best Possible Solution

A maintainer should resolve the live slug collision through an audited support action or a deliberately scoped publisher-aware reclaim/transfer path, then document deleted-slug behavior and align it with the planned scoped org slug model.
