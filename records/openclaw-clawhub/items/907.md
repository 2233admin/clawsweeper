## Summary

Keep open. Current main still treats user-initiated account deletion as irreversible and blocks the same deleted/deactivated user during GitHub OAuth completion; the only current improvement is surfacing a clearer auth error. This is also a ClawHub issue, and the repository cleanup policy says issues should remain open unless maintainers explicitly choose otherwise.

## What This Changes

The issue requests a way to sign up or sign in again with the same GitHub account after deleting a previous ClawHub account, instead of returning to a refresh/sign-in loop.

## Best Possible Solution

Maintainers should decide and document the account deletion policy. If same-GitHub re-signup is allowed, implement a secure recreate or relink path that preserves abuse and ownership invariants; if deletion remains irreversible, make the OAuth result and support path explicit enough that affected users are not left in a silent loop.
