## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only permits auto-closing ClawHub PRs that are certainly implemented on main. The current code also still intentionally blocks same-GitHub sign-in after account deletion, so the requested re-signup behavior remains unresolved product/security work rather than cleanup.

## Best Possible Solution

Keep this issue open and triage it with #504 and #900. Maintainers should choose the product policy: either keep account deletion permanently irreversible with clearer sign-in/support messaging, or implement a secure same-GitHub re-registration/relink flow that retires the old auth binding while preserving moderation bans. Add regression coverage for delete-account followed by same-GitHub sign-in once that policy is chosen.
