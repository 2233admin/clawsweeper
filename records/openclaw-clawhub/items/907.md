## Summary

Keep open. Current main still intentionally rejects same-GitHub sign-in after user-initiated account deletion, and the ClawHub cleanup policy keeps issues open unless maintainers explicitly decide otherwise. The remaining decision is product- and security-sensitive because it changes deletion guarantees, OAuth account binding, and abuse-prevention behavior.

## What This Changes

The issue requests that a user who deleted a ClawHub account be able to sign up again with the same GitHub account instead of landing in a sign-in loop.

## Best Possible Solution

Maintainers should decide and document the account deletion policy. If re-signup is allowed, implement a secure same-GitHub recreate or relink flow that preserves abuse protections and ownership invariants; if deletion stays irreversible, make the OAuth result and support path explicit enough that users are not left in a loop.
