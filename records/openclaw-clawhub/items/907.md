## Summary

Keep this issue open. It is an open ClawHub issue, and the repository cleanup policy for this review says to keep ClawHub issues open unless a separate close reason is strongly proven. Source review also shows the reported deleted-account GitHub re-signup problem is not implemented on current main: self-service deletion deactivates/purges the existing user, OAuth reauth throws a permanent-deletion error, and related reports such as #504/#1452 remain open for the same account lifecycle gap.

## Best Possible Solution

Keep this issue open and treat it as another concrete report of the deleted-account GitHub OAuth loop. Maintainers should decide the product policy: either keep deletion irreversible but make the login/support path explicit, or implement a safe re-registration/relink flow that retires or supersedes the old Convex Auth binding without restoring banned accounts. Coordinate the fix with related #504/#1452 and add regression coverage for delete-account followed by same-GitHub sign-in.
