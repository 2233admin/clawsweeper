## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-close for PRs that are definitely implemented on current main, and this is an issue. Current main still makes user-initiated deletion irreversible: the OAuth callback rejects deleted/deactivated users instead of re-signing up or relinking the same GitHub account, while the frontend now only surfaces a clearer auth error.

## Best Possible Solution

Keep this issue open and triage it with #504, #900, #1116, and #1452. Maintainers should explicitly choose the account-deletion policy: either keep deletion irreversible and consistently show the deleted-account support message in OAuth/sign-in UI, or implement a secure same-GitHub re-registration/relinking path that retires the old auth binding while preserving ban and auto-ban protections. Add regression coverage for delete-account followed by same-GitHub sign-in once the policy is chosen.
