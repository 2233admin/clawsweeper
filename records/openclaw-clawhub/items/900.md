## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still treats user-initiated deletion as irreversible: OAuth reauth is rejected, active-user gates return no user, settings/CLI token creation require an active user, and the UI can fall back to sign-in when `me` is null. Related open reports #504, #907, and #917 should be consolidated under an explicit account-lifecycle policy decision.

## Best Possible Solution

Keep this open and consolidate it with #504, #907, and #917, or designate one canonical deleted-account lifecycle tracker. Maintainers should explicitly choose the policy: preserve irreversible deletion and improve OAuth/CLI/settings error handling, or add a deliberate recreate, relink, admin recovery, or GitHub identity unlink flow with tests across auth callback handling, active-user gates, token creation, settings UI, CLI auth, and security docs.
