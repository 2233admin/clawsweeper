## Summary

Keep open. Current main still blocks self-deleted/deactivated GitHub identities from becoming active users again, which matches the reported OAuth loop, live sign-in button, and CLI-token lockout. The requested recovery/relink behavior is not implemented and needs maintainer product/security policy before a code fix.

## What This Changes

Users who deleted their ClawHub account want GitHub sign-in to restore, relink, or recreate access instead of leaving them logged out with the sign-in button active and CLI tokens inaccessible.

## Best Possible Solution

Make one explicit deleted-account policy and enforce it consistently: either preserve irreversible deletion with clearer OAuth/settings/CLI recovery messaging and support routing, or add a deliberate restore/relink/recreate flow with auditability and regression coverage across OAuth callbacks, active-user gates, token creation, UI, CLI, and docs.
