## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on main. Current main still treats user-initiated deletion as irreversible and blocks the same account from becoming an active user again, so the reported OAuth/account lifecycle problem remains unresolved. Related reports #504 and #907 are also open and should be triaged together.

## Best Possible Solution

Keep this issue open and consolidate it with #504/#907 or a dedicated deleted-account lifecycle tracking issue. Maintainers should decide whether deletion remains permanently blocking as documented, or implement a safe recreate/relink/admin recovery flow with tests across `convex/auth.ts`, `convex/users.ts`, `convex/lib/access.ts`, CLI token creation, settings/header UI, and security docs.
