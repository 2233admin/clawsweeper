## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy says to keep issues open unless they are PRs implemented on main. Current main also does not implement the reporter’s expected deleted-account re-login/relink behavior: account deletion deactivates and purges the user, OAuth completion blocks deleted/deactivated users, and protected settings/token flows reject deactivated users. The related #504 report is the same core problem and was previously kept open, so maintainers should consolidate or triage the product decision rather than auto-close this issue.

## Best Possible Solution

Keep this issue open for maintainer triage, preferably consolidated with #504. The maintainer decision is whether self-deleted GitHub accounts should remain permanently blocked as documented, or whether ClawHub should support a safe recreate/relink flow. If the product choice is to allow re-login, the work likely belongs in Convex Auth account linking/deleted-user handling around convex/auth.ts, deleteAccount semantics in convex/users.ts, user bootstrap/settings/token behavior, and matching docs/tests.
