## Summary

Keep open. This is an open ClawHub issue, and the supplied repository cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main uses GitHub's immutable numeric OAuth id and has a post-login profile sync path for renamed accounts, but the reported production verifier-token/session mismatch and requested account relink cannot be proven resolved from source alone.

## What This Changes

The issue reports that a user cannot sign in with GitHub after changing their GitHub username and asks maintainers to relink the account to the new username.

## Best Possible Solution

ClawHub should continue using immutable GitHub provider ids for identity, and maintainers should triage the reporter's production auth records/logs to either repair a one-off account linkage or add a focused recovery/admin path if the same verifier or duplicate-account failure is recurring.
