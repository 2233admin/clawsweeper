## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on main. Current main also still has a plausible `publish .` failure path: the publish command resolves the source path against the resolved ClawHub workdir, which can point at a configured Clawdbot/OpenClaw workspace instead of the caller's current skill directory.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to resolve explicit publish source paths relative to the caller's actual cwd in the user-facing publish commands, or otherwise make the workdir fallback behavior explicit, then add a regression test where a configured default workspace differs from cwd and `clawhub skill publish .` is run from a directory containing `SKILL.md`. Maintainers should also decide whether #1509 should become the canonical tracking issue.
