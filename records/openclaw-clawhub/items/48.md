## Summary

Keep open. This is an issue, and the repository-specific ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on main. Substantively, current main still has a plausible path-resolution failure for the reported command: both legacy `publish` and current `skill publish` call `cmdPublish`, which resolves the source path against the resolved ClawHub workdir, while that workdir can fall back to a configured Clawdbot/OpenClaw workspace instead of the caller's current skill directory.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to resolve an explicit publish source path relative to the caller's actual cwd, or make the publish workdir fallback behavior explicit for publish commands, then add a regression test where a configured default workspace differs from the current directory and `clawhub skill publish .` is run from a directory containing `SKILL.md`. Maintainers may also want to consolidate this with #1509 if they decide one should be the canonical publish-path tracking issue.
