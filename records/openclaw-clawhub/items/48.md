## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy only permits auto-closing pull requests that are certainly implemented on main. A source pass also still supports a plausible `publish .` failure path: the publish command resolves the source path against the resolved ClawHub workdir, which can differ from the caller’s current skill directory when a Clawdbot/OpenClaw default workspace is configured.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to resolve explicit publish source paths relative to the caller’s actual `process.cwd()` unless they are absolute, while preserving `workdir` for CLI state/registry behavior, then add a regression test where the configured default workspace differs from cwd and `clawhub skill publish .` is run from a directory containing `SKILL.md`. Maintainers should also decide whether #1509 should be marked as the canonical tracker or linked as a duplicate.
