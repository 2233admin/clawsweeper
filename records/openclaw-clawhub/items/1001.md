## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Source review also confirms the reported web-viewer gap remains: skill README links render as ordinary relative anchors, while bundled files are available only through the Files tab or raw file API.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Implement a safe relative-link resolver in the skill README rendering path, likely in SkillDetailTabs or a shared MarkdownPreview variant, that checks latestFiles and routes matching relative links to an in-app file viewer or /api/v1/skills/{slug}/file?path=.... Add regression coverage for references/system-design.md, URL encoding, query/hash preservation, missing files, unsafe schemes, case-insensitive file matches, and path traversal attempts.
