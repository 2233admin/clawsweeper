## Summary

Keep open. Current main still appears to leave the reported web-viewer behavior unresolved: the skill README renderer passes Markdown links through without resolving relative packaged-file links, and the owner/slug route has no child route for referenced files. The raw file API already exists as the likely target, and related #1681 plus draft PR #1858 are the active tracker and implementation candidate, so this should stay open for maintainer follow-up rather than be auto-closed.

## What This Changes

The issue asks for relative file links in rendered skill documentation, such as `references/system-design.md`, to open the packaged file in the ClawHub web viewer instead of navigating to a 404 owner/path URL.

## Best Possible Solution

Land or revise the existing focused fix so safe relative links in rendered SKILL.md content resolve through `/api/v1/skills/{slug}/file?path=...`, with regression coverage for packaged files, anchors, external schemes, and traversal rejection.
