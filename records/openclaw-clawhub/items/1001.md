## Summary

Keep open. This is a ClawHub issue, and the repository-specific policy for this review keeps all issues open. Source review also shows the reported click-through behavior is not implemented on current main: SKILL.md is rendered with plain ReactMarkdown and no relative-file link resolver, while the existing packaged-file support is only available through the Files tab or the raw `/api/v1/skills/{slug}/file?path=` endpoint.

## Best Possible Solution

Keep the issue open and implement a first-class skill-file link path: the SKILL.md renderer should resolve safe relative links against the latest version's packaged file list and navigate either to an in-app rendered file viewer or to the existing raw `/api/v1/skills/{slug}/file?path=` endpoint. Add regression coverage for links such as `references/system-design.md`, including path encoding and disallowing unsafe external/javascript URLs.

## Evidence

- **Issue context:** The report gives a concrete reproduction: a relative SKILL.md link such as `references/system-design.md` on `https://clawhub.ai/guoqunabc/learning-cards` navigates to `/guoqunabc/references/system-design.md` and 404s, while the file is present in the published `.skill` archive and installs locally.
