## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. The underlying bug also still appears present: README markdown links are rendered without a bundled-file resolver, while bundled files are only reachable through the Files tab or raw file API. Related #1681 and draft PR #1858 remain the active fix path.

## Best Possible Solution

Keep this issue open and route it through #1681 and draft PR #1858. The best implementation is a focused README-link resolver that rewrites safe relative links to the skill file API or an in-app bundled-file viewer, preserves allowed external/hash/mail/tel links, rejects traversal and unsafe schemes, handles nested paths plus URL encoding/query/hash behavior, and adds regression tests around SkillDetailTabs and the link helper.
