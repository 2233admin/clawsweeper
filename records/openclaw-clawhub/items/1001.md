## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only permits auto-closing pull requests that current main definitely implements. The bug also still appears present on current main: the skill README renderer emits Markdown links without resolving relative targets into the skill file API, while the only route for the page is /$owner/$slug, so a relative references/... link can still navigate outside the skill namespace. Related #1681 tracks the broader security form of the same problem, and draft PR #1858 is already open as an implementation candidate.

## What This Changes

The issue requests that relative links from SKILL.md to packaged files such as references/system-design.md open a web-viewable skill file instead of navigating to a 404 route.

## Best Possible Solution

Keep this issue open and route it with #1681 to review or land draft PR #1858, or an equivalent focused fix that rewrites safe relative SKILL.md links to the existing /api/v1/skills/{slug}/file?path=... endpoint and adds regression tests for normal relative files, anchors/external schemes, and traversal rejection.
