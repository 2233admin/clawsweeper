## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on main. A source review also shows the reported behavior is still plausible: README links are rendered as ordinary Markdown links, while packaged files are only exposed through the separate Files tab or raw file API.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Implement a safe relative-link resolver in the skill README rendering path, likely in SkillDetailTabs or a shared MarkdownPreview component, that recognizes packaged latestFiles and routes them to an in-app file viewer or /api/v1/skills/{slug}/file?path=...; add regression coverage for references/system-design.md, URL encoding, query/hash preservation, missing files, case behavior, and blocked unsafe schemes.
