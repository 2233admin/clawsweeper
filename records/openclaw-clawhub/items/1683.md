## Summary

Keep open. This is a ClawHub issue, and the repository policy for this cleanup pass says ClawHub issues should not be auto-closed. The current main branch also still has a plausible product bug: public skill detail responses can render duplicate/fork metadata for a referenced skill without checking whether that referenced skill is still public.

## Best Possible Solution

Keep this issue open for maintainer handling. The likely fix is to make convex/skills.ts suppress forkOf and canonical response metadata unless the referenced skill passes the same public visibility rules as normal skill detail pages, add a regression test for a duplicate target with softDeletedAt set, and use the existing duplicate-clear moderation path to clean any stale production row if needed.
