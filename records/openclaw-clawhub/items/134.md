## Summary

Keep this issue open. The ClawHub cleanup policy only permits automated closure for pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main still does not prove the reported update-disappearance path is fixed: non-trusted clean updates can still patch the skill to hidden pending-scan state, and public search/inspect surfaces filter hidden skills out.

## Best Possible Solution

Keep this issue open. Maintainers should decide whether already-active skills should remain publicly visible while update scans run. If yes, land a corrected preserve-active implementation for existing clean updates and add regression coverage for a non-trusted publisher updating an active skill. If hidden-pending behavior is intentional, document it and make web, CLI inspect/search, API responses, and owner/non-owner visibility consistent, then handle affected production skills operationally.
