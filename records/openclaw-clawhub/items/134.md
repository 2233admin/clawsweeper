## Summary

Keep this issue open. The ClawHub cleanup policy for this review only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main still does not prove the reported update-disappearance path is fixed: existing non-trusted skill updates can still be patched into hidden pending-scan state, and public detail/search/API surfaces filter hidden skills out.

## Best Possible Solution

Keep this issue open. Maintainers should decide whether already-active skills should remain public while update scans run. If yes, implement a preserve-active update path in insertVersion and add regression coverage for a non-trusted publisher updating an active skill. If hidden-pending behavior is intentional, document it and make web, CLI inspect/search, API responses, and owner/non-owner visibility consistent, then handle affected production skills operationally.
