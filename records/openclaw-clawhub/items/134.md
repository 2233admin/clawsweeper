## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits automated closure for PRs that are certainly implemented on current main, and this item is an issue. Substantively, current main still does not prove the reported non-trusted update disappearance is fixed: the publish path detects an existing skill, but insertVersion still applies a hidden pending-scan moderation status to updates for non-trusted/non-staff publishers, and public read/search surfaces filter that state out.

## Best Possible Solution

Keep this issue open. Maintainers should decide the intended update moderation policy, then either land a corrected #274-style fix that preserves public visibility for legitimate already-active skills during update scans, or document and implement hidden-pending behavior consistently across web, CLI inspect/search, and API responses. Add regression coverage for a non-trusted publisher updating an already-active skill.
