## Summary

Keep this PR open. Current main still does not implement the PR's intended dashboard skills pagination: `/dashboard` still uses `useQuery(api.skills.list, { limit: 100 })`, there is no `listDashboardPaginated` Convex query, no dashboard Load More state, and no dashboard pagination test. The author also scoped server-side search out in discussion, so the remaining useful work is pagination-only and still absent from main.

## Best Possible Solution

Keep this PR open or replace it with a refreshed pagination-only PR against current main. The best path is an indexed Convex dashboard pagination query, `/dashboard` `usePaginatedQuery` Load More UI, focused pagination tests, preservation of existing `skills.list` owner/legacy visibility semantics, and a separate search follow-up.
