## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only allows auto-closing pull requests that are certainly implemented on current main. Substantively, current main still lacks the requested local plugin scanner CLI: it has `clawhub package publish --dry-run --json`, narrow local package validation, and server-side static publish scanning, but no `package lint`, `plugins lint`, standalone `plugin-lint`, pre-publish scanner mode, severity-coded local findings output, or non-zero lint behavior matching the request. The prior ClawSweeper comment reached the same conclusion, and the current source still supports it.

## Best Possible Solution

Keep this issue open. The best product path is to add a local ClawHub CLI command such as `clawhub package lint <source>` or a clear `package publish --dry-run --pre-publish` mode that reuses the shared package schema and static moderation checks, supports `--json`, emits severity-coded findings, and exits non-zero for blocking publish errors.
