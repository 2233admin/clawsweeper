## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should remain open even after review. Substantively, current main still has package publish dry-run and server-side static publish scanning, but no local plugin/package lint scanner CLI with the requested pre-publish findings, JSON severity output, and non-zero validation behavior.

## Best Possible Solution

Keep this issue open. The best path is to add a local ClawHub CLI command such as `clawhub package lint <source>` or an explicit `package publish --dry-run --pre-publish` mode that reuses shared package schema and static-scan logic, supports `--json`, emits severity-coded findings, and exits non-zero for blocking publish errors.
