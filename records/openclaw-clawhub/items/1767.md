## Summary

Keep open. Repository policy for ClawHub keeps issues open, and the substantive request is still not implemented on current main: the CLI has publish dry-run and hosted scan persistence, but no local plugin scanner/lint command that validates manifests, SKILL.md metadata, source heuristics, JSON output, severity levels, and nonzero scanner exits before publish.

## What This Changes

Add a local ClawHub/OpenClaw plugin scanner or lint CLI for pre-publish validation of plugin manifests, SKILL.md metadata, and source-code security findings.

## Best Possible Solution

Define a maintainer-approved local ClawHub CLI scanner that reuses shared package manifest validation and deterministic static moderation logic, adds any missing shared schema checks, supports human and JSON output with severity-coded findings, and documents its relationship to hosted publish-time scanning.
