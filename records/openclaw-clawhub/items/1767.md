## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only allows auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has package publish dry-run and server-side static scan persistence, but no local plugin scanner/lint CLI that validates manifests, SKILL.md metadata, and code before publish with severity-coded findings and scanner-style exit behavior.

## Best Possible Solution

Keep this issue open for maintainer-scoped design. The best implementation path is likely a ClawHub CLI local scanner that reuses shared package schema validation and static moderation logic, validates local package manifests and SKILL.md metadata, supports --json, emits severity-coded findings, and exits nonzero for blocking publish errors without changing hosted moderation semantics.
