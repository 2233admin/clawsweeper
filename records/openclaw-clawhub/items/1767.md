## Summary

Keep open. This is a ClawHub issue, and the repository-specific review policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. Substantively, current main still lacks the requested local plugin scanner/linter CLI; the existing package publish dry-run produces a publish plan, while richer static scanning remains in the server-side publish path.

## Best Possible Solution

Keep this issue open. The best path is to add a local ClawHub CLI command such as `clawhub package lint <source>` or a stronger explicit `package publish --dry-run --pre-publish` mode that reuses shared package schema and static-scan logic, supports `--json`, emits severity-coded findings, and exits non-zero for blocking publish errors.
