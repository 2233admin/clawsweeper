## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should remain open; only pull requests that are certainly implemented on current main are eligible for auto-close. Substantively, current main still exposes package publish dry-run and hosted static scanning, but not a local plugin/package scanner CLI with the requested manifest/SKILL/code checks, JSON findings, severity levels, and non-zero lint behavior.

## Best Possible Solution

Keep this issue open. The best implementation path is to add a local ClawHub CLI command such as `clawhub package lint <source>` or a stronger `package publish --dry-run --pre-publish` mode that reuses shared package schema and static scan logic, supports `--json`, emits severity-coded findings, and exits non-zero for blocking publish errors.
