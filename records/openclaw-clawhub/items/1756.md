## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only allows auto-closing ClawHub PRs that are certainly implemented on current main. The reported bug also remains plausible: the skills page replaces the full search result array after scroll increases the requested search limit, and the backend relevance search changes candidate/fallback windows based on that same limit.

## Best Possible Solution

Keep this issue open and fix the search pagination/ranking contract. The best implementation is to make the top results stable when requesting more results, either with a server-side cursor over a fixed ranked result set or by decoupling candidate expansion and fallback recall from the display limit. Add regression coverage comparing the first page from a 25-result request with the top 25 from a larger request for the same query.
