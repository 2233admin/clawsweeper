## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review only permits auto-closing pull requests that are certainly implemented on main. Current main also still has a plausible implementation path for the reported behavior: the skills page replaces the whole search result set when scroll/load-more increases the requested limit, while backend relevance search can expand and re-rank a different candidate pool for the larger limit.

## Best Possible Solution

Keep this issue open. The best fix is to make relevance search pagination stable, either with a server-side cursor over a fixed ranked result set or by decoupling candidate expansion from the display limit so load-more cannot discover higher-scoring results above already-rendered entries. Add a regression test comparing the top relevance results from the initial 25-result request against the larger scroll/load-more request.
