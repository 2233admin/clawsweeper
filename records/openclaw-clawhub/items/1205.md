## Summary

Keep this issue open. The issue is not eligible for ClawHub auto-close because it is an open issue, and current main still leaves the reported Convex failure shape in the owner-publisher digest rebuild helpers: each internal mutation can call `.paginate()` repeatedly in one invocation. The earlier backend scheduling split mitigates the original inline trigger path for small result sets, and the reporter confirmed their immediate publish succeeded, but the code still needs a focused cursor/self-scheduling fix for multi-page owner catalogs.

## Best Possible Solution

Keep this issue open and land one focused Convex backend fix: convert both owner-publisher digest rebuild helpers into cursor-based self-scheduling internal mutations so each invocation performs at most one `.paginate()` call, while preserving the existing publisher-trigger scheduling split. Add tests that simulate multi-page package and skill digest rebuilds and assert continuation scheduling, then deploy and consolidate the duplicate publish/import pagination reports.

## Clownfish Work Prompt

Fix the ClawHub Convex owner-publisher digest pagination bug tracked by this issue and related reports #1171, #1182, and #1201. In `convex/functions.ts`, refactor both `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` so no Convex function invocation calls `.paginate()` more than once. Preserve the existing publisher-trigger scheduling behavior, but make the package and skill owner-publisher digest rebuild internals cursor-based/self-scheduling: process one page, update digests for that page, and schedule the same internal mutation with the continuation cursor when `isDone` is false. Keep table access indexed via `by_owner_publisher`; do not replace pagination with unbounded `.collect()` or full table scans. Update validators/internal function args to include the cursor where needed, and keep missing-table fallback behavior. Add focused Vitest coverage in `convex/functions.test.ts` for multi-page package and skill rebuilds, including that continuation scheduling occurs and each invocation performs only one paginate call. Also keep the existing scheduling split tests passing. Do not change CLI packaging, auth, publish file validation, unrelated package release logic, or generated Convex files unless codegen is required by the repo workflow.
