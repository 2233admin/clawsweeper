## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still contains the backend failure shape behind this report: the owner-publisher package and skill digest rebuild helpers can call `.paginate()` repeatedly inside one internal mutation, while the scheduled internal mutations do not accept cursor continuation state. The reporter confirmed the immediate production publish recovered, but the source still needs a focused Convex repair shared with the related open publish/import pagination reports.

## What This Changes

The issue reports that `clawhub publish` fails when owner-publisher digest synchronization calls Convex pagination more than once in a single backend function invocation.

## Best Possible Solution

Implement cursor-based, one-page-per-invocation owner-publisher digest rebuilds for both package and skill search digests, with continuation scheduling when `isDone` is false and focused tests proving multi-page rebuilds avoid Convex's pagination limit.

## Repair Work Prompt

Fix the Convex pagination failure in owner-publisher digest synchronization. Current main has `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` looping over `.paginate()` inside one internal mutation; Convex allows only one paginated query per function invocation. Refactor the owner-publisher package and skill digest rebuild paths to process one page per invocation, pass cursor and optional batch-size state through the internal mutation args, and self-schedule continuation with `ctx.scheduler.runAfter(0, ...)` when `isDone` is false. Preserve the existing publisher-trigger scheduling split and existing digest semantics. Add focused tests in `convex/functions.test.ts` that simulate multi-page package and skill rebuilds and assert continuation scheduling. Do not replace pagination with unbounded `.collect()`, do not broaden the publish API surface, and do not change unrelated CLI behavior. Related reports: #1170, #1171, #1175, #1176, #1182, #1184, #1195, #1197, #1201. Include a short changelog or PR note if the project convention expects one for backend publish fixes.
