## Summary

Keep this issue open. The reporter confirmed the immediate production publish recovered after an apparent backend hotfix, but current main still contains the same Convex failure shape: the owner-publisher package and skill digest rebuild helpers can call `.paginate()` repeatedly inside one internal mutation invocation, and the ClawHub cleanup policy keeps issues open unless a maintainer lands or consolidates a fix.

## What This Changes

The issue reports that `clawhub publish` fails because the owner-publisher skill digest sync can run multiple Convex paginated queries inside one mutation.

## Best Possible Solution

Keep this issue open and land one focused Convex backend fix: convert the owner-publisher package and skill digest internal mutations to process one page per invocation, accept cursor and batch-size arguments, and self-schedule continuation when `isDone` is false. Preserve the existing publisher-trigger scheduling split, add multi-page tests for both helpers, and consolidate the related publish/import pagination reports after the fix lands.
