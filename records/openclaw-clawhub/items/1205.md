## Summary

Keep this issue open. The ClawHub cleanup policy for this review says issues should not be auto-closed, and current main only partially mitigates the reported first-publish failure by scheduling owner-publisher digest sync work outside the publisher trigger. The named helper still loops over `.paginate()` calls inside one internal mutation, so the multi-page owner case is not conclusively fixed. The reporter’s confirmation that their publish later succeeded is useful maintainer context, but it is not enough for this workflow to close an issue.

## Best Possible Solution

Keep this issue open for maintainer judgment. The best code path is to convert the owner-publisher package and skill digest rebuild helpers into cursor-based self-scheduling batches so each Convex mutation performs at most one paginated query, then either close this issue manually as resolved or consolidate it into the active related pagination report.
