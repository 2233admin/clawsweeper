## Summary

Keep this issue open. Current main still has the reported double JSON serialization in all three CLI commands, and the related fix PRs called out in the discussion were closed without merging.

## What This Changes

Fix the CLI so transfer request, skill rename, and skill merge send JSON object bodies once instead of pre-stringifying bodies before calling `apiRequest`.

## Best Possible Solution

Keep this issue open and land a focused CLI fix: pass plain object bodies from transfer request, skill rename, and skill merge; update the command tests to assert object payloads; and, if `apiRequest` is hardened defensively, cover both Node fetch and Bun/curl POST paths. Closed PR #1251 is the best reference, but none of the related PRs has merged.

## Repair Work Prompt

Fix the double JSON serialization bug in the ClawHub CLI. In `packages/clawhub/src/cli/commands/transfer.ts`, `cmdTransferRequest` should pass a plain object body with `toUserHandle` and `message` to `apiRequest`. In `packages/clawhub/src/cli/commands/ownership.ts`, `cmdRenameSkill` and `cmdMergeSkill` should pass plain object bodies with `newSlug` and `targetSlug`. Update the command tests so they assert object payload shape, not string substrings. Consider also hardening `packages/clawhub/src/http.ts` so already-serialized string bodies are not double-encoded, but if you do, cover both the Node fetch path and the Bun/curl path with focused tests. Use closed PRs #973, #1251, and #1365 as references, with #1251 as the broadest prior attempt. Do not change unrelated CLI behavior, release metadata, generated files, or broad HTTP semantics beyond JSON POST body serialization. No changelog entry is expected unless the repository's current release workflow requires one for CLI bug fixes.
