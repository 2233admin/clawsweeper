## Clownfish Work Prompt

Fix the ClawHub CLI double JSON serialization bug for transfer request, skill rename, and skill merge. In `packages/clawhub/src/cli/commands/transfer.ts` and `packages/clawhub/src/cli/commands/ownership.ts`, pass plain object bodies to `apiRequest` instead of pre-stringifying them. Update `packages/clawhub/src/cli/commands/transfer.test.ts` and `packages/clawhub/src/cli/commands/ownership.test.ts` so they structurally assert object payloads rather than string substrings. Consider a narrow defensive hardening in `packages/clawhub/src/http.ts` so pre-serialized string bodies are not double-encoded in both the Node fetch path and Bun/curl path, with tests in `packages/clawhub/src/http.test.ts` and `packages/clawhub/src/http.bun.test.ts` if that helper change is made. Use related closed PRs #973, #1251, and #1365 as implementation context, especially #1251, but do not copy unrelated changes. Do not change server API semantics, auth behavior, ownership mutations, docs, release workflows, or unrelated commands. No changelog/docs update is expected unless the repo has a current release-note mechanism for CLI bug fixes.
