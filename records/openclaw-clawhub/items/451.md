## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still does not implement the requested GitHub profile links beside usernames or in the user profile header. The project/repository link side is only partially covered: skill detail pages can show `clawdis.links.repository`, but browse/list/profile owner surfaces still do not expose a separate GitHub user or repository link.

## Best Possible Solution

Keep this issue open and implement a focused core UI/data change: expose a verified GitHub profile URL or login in public user/publisher data where appropriate, add external GitHub links next to owner badges and profile headers, and surface repository/source links consistently from skill metadata without breaking card or list navigation.
