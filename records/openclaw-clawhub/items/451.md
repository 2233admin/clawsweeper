## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub says issues are not eligible for auto-close, and current main still does not implement the requested GitHub profile links beside usernames or in the user profile header. Repository/source links are partially available from package and skill metadata, but that does not cover the broader user/profile link request.

## Best Possible Solution

Keep this open and implement a focused ClawHub UI/data change: expose a safe verified GitHub login or profile URL in public user/publisher data, add external GitHub profile links beside owner badges and profile headers, and surface repository/source links consistently from skill/package metadata without breaking existing navigation.
