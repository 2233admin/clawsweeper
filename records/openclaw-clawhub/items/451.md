## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open, and current main still does not expose verified external GitHub profile links beside usernames or in the user profile header. Repository links are only rendered when explicit skill metadata provides `clawdis.links.repository`, not as a consistent user/repo linking treatment across the requested surfaces.

## Best Possible Solution

Keep this issue open and implement a focused UI/data change: store or derive a verified GitHub profile URL from GitHub provider identity/sync data, expose it through public user and publisher projections, denormalize it into hot-path browse/search owner data where needed, render an adjacent external GitHub link/icon on user/profile/owner surfaces without breaking internal ClawHub navigation, and surface trusted repository/source links consistently where skill metadata already provides them.
