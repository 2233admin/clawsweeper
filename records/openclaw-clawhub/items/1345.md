## Summary

Keep this issue open. The repository policy for openclaw/clawhub says automated cleanup should keep issues open, and the request is still a concrete product/data problem: current main lets summaries be derived from SKILL.md frontmatter during publish/backfill, but it does not provide an owner-facing edit endpoint or UI for correcting an already-published registry summary without publishing a new latest version or using internal/admin maintenance. The provided related context also shows a same-author implementation candidate in #1411, so the issue should stay open until maintainers review/land that path or handle the production metadata correction directly.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is either an owner-verified admin correction of the production registry summary for auto-create-ai-team, or review/land a safe owner-facing summary edit flow such as the related #1411 work, with validation, authorization, audit logging, and moderation re-evaluation as needed.
