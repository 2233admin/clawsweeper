## Summary

Current main still does not implement this PR's intended moderation-hold recovery path. The repository can set and enforce `requiresModerationAt`, hide all owned skills, and keep future publishes hidden while the hold remains set, but there is no `liftModerationHold` API, no matching restore batch, no `user.moderation.lift` audit action, and no docs for lifting holds. The PR should stay open for maintainer/security review rather than being auto-closed.

## Best Possible Solution

Keep this PR open for maintainer/security review, or land a corrected equivalent implementation. The eventual fix should add an admin-only moderation-hold lift API, clear `requiresModerationAt`/`requiresModerationReason`, restore only skills that should become visible after the relevant user-level hold is lifted, preserve malicious/manual/ban and pending-scan states, guard hide/restore races, write accurate audit data, test the edge cases, and document the admin Convex CLI operation.
