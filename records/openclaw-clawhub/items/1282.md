## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should remain open; only PRs that are certainly implemented on main are eligible for automated implemented-on-main closure. Current main also still intentionally blocks re-authentication for deleted/deactivated users instead of implementing the requested GitHub OAuth unbind, hard delete, restore, or re-registration path.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. Maintainers should either manually restore or unbind the affected GitHub identity when permitted, or implement a safe documented account recovery/re-registration path that handles Convex Auth GitHub provider bindings for deactivated users.
