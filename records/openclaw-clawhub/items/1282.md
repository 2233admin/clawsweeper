## Summary

Keep this issue open. Current main still intentionally treats user-initiated deletion as irreversible deactivation: deletion writes deactivated/purged state, OAuth completion rejects deleted/deactivated users, active-user helpers exclude them, admin unban explicitly refuses permanently deleted accounts, and tests/docs assert the block. The related deleted-account login reports remain open, so this needs maintainer support/product-security handling rather than cleanup closure.

## Best Possible Solution

Keep this open for maintainer support and product/security handling. The best path is to verify control of the GitHub identity, inspect the production Convex Auth binding and user record, then either perform a deliberate admin recovery/unbind/hard-delete action or document that deleted accounts cannot be restored. Any code change should be coordinated with the related deleted-account login cluster and preserve ban protections.
