## Summary

Keep this issue open. Repository-specific cleanup policy for openclaw/clawhub says issues are not eligible for automated closure, and current main still enforces user-initiated deletion as an irreversible deactivation that blocks GitHub re-login/re-registration rather than unbinding or restoring the provider identity.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. The best fix is either a verified manual production recovery/unbind/hard-delete path for the GitHub identity, or a safe documented re-registration/restoration flow that updates the user record, Convex Auth GitHub authAccounts binding, retained skills/publishers, and ban protections consistently.
