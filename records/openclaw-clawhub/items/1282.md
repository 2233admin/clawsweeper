## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should remain open unless the item is a PR implemented on main. Substantively, current main still treats user-initiated deletion as irreversible: deletion writes `deactivatedAt`/`purgedAt`, OAuth sign-in rejects deactivated users, normal active-user helpers reject them, and the admin unban path explicitly cannot restore permanently deleted accounts. The requested GitHub identity unbind, hard delete, restore, or re-registration flow is not implemented.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. Maintainers should either manually restore, unbind, or hard-delete the verified GitHub identity when allowed, or implement a safe documented recovery/re-registration flow that handles Convex Auth GitHub `authAccounts` bindings, deactivated user records, and retained public skills without bypassing ban protections.
