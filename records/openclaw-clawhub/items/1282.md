## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub says issues are not eligible for automated closure, and current main still intentionally treats user-initiated deletion as irreversible deactivation that blocks GitHub re-login rather than unbinding/restoring the provider account.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. The best path is either a verified manual production recovery/unbind/hard-delete process for the GitHub identity, or a safe documented re-registration/restoration flow that handles Convex Auth bindings, deactivated users, retained skills/publishers, and ban protections consistently. If maintainers want to consolidate, they should choose a canonical open account-recovery issue and close related reports manually after that policy is decided.
