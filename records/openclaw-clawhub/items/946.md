## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only allows auto-closing pull requests that current main definitely implements. The substantive request is also still unresolved in source: current main supports the documented `metadata.openclaw.requires.env`, `primaryEnv`, and `envVars` metadata shapes, but it does not parse the reporter's top-level `required_credentials` or top-level `permissions` aliases into registry or scanner metadata.

## What This Changes

The issue asks ClawHub to recognize buff-roundup's declared SKILL.md credentials and permissions as scanner metadata and clear the suspicious false-positive flag.

## Best Possible Solution

Maintainers should decide whether top-level `required_credentials` and `permissions` are supported ClawHub aliases. If yes, implement them consistently across the shared schema, SKILL.md parser, static declaration collector, LLM prompt context, UI/API/docs/tests, and rescan/backfill path; if not, ask the publisher to republish with the documented `metadata.openclaw.requires.env`, `primaryEnv`, or `envVars` shape and make an explicit moderation decision for buff-roundup.
