## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy only allows auto-closing ClawHub pull requests that are already implemented on main. Substantively, current main still does not show a complete metadata.json-to-normalized-registry path: publish parsing derives parsed.clawdis from SKILL.md frontmatter, the static and LLM scanners consume normalized/frontmatter metadata, and standalone metadata.json declarations remain a plausible source of the reported mismatch. Related open reports #1432 and #1667 describe the same registry/env mismatch pattern.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether standalone metadata.json/package.json env declarations are supported sources for skills. If yes, parse and merge them during publish into the same normalized parsed.clawdis requires.env/envVars shape used by latestVersionSummary, UI/API install prompts, static scan, and the LLM prompt, then add regression tests for metadata.json plus SKILL.md frontmatter and a backfill/rescan path. If no, update docs and server/CLI validation so publishers are told to declare env requirements in SKILL.md frontmatter only. Treat the uv point as already supported unless scanner calibration still mislabels it.

- The uv concern appears supported in docs/schema, but scanner wording may still need calibration if it continues to treat uv plus pip-style installation as unusual.
