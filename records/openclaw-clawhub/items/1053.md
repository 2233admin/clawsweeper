## Summary

Keep open. Repository cleanup policy for openclaw/clawhub keeps issues open, and current main still has the reported scanner-visible metadata gap: publish/CLI paths do not normalize root metadata.json env declarations into registry metadata, and the LLM prompt still prints required env vars from requires.env only while uploaded metadata.json contents may be visible later in the prompt.

## Best Possible Solution

Keep this issue open and have maintainers choose the canonical metadata contract for scanner-visible runtime requirements. The likely fix should normalize required env vars consistently from requires.env, primaryEnv, and required envVars; explicitly decide whether root metadata.json env is supported or documented as unsupported; update the LLM scanner summary/prompt and publish/parser tests; and plan rescans for affected versions if historical suspicious verdicts should be corrected.
