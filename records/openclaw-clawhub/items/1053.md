## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on main, not issues. Substantively, current main documents and parses `requires.env` from `SKILL.md` frontmatter/`metadata.openclaw`, but I did not find evidence that standalone `metadata.json` env declarations are ingested into the registry/LLM summary path; related false-positive reports remain open as well.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely product/code path is to decide whether standalone `metadata.json` env declarations are supported; if yes, parse them during publish into the same normalized `parsed.clawdis.requires.env` / `envVars` shape consumed by the registry summary, static scan, detail UI, and LLM prompt, with tests covering both `metadata.json` and SKILL.md frontmatter. If standalone `metadata.json` is not supported, the docs and CLI should make that explicit and guide publishers to `metadata.openclaw.requires.env` in SKILL.md frontmatter.
