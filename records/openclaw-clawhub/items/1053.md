## Summary

Keep open. This is a ClawHub issue, and repository policy only permits auto-closing ClawHub PRs that current main certainly implements. Substantively, current main still has a scanner-visible env metadata gap: publish parsing is centered on SKILL.md frontmatter, root metadata.json is only uploaded as a file, and the LLM registry summary renders required env vars from requires.env rather than normalizing primaryEnv, required envVars, or the reported metadata.json env shape.

## What This Changes

The issue requests that ClawHub normalize environment variable declarations from skill metadata so scanner registry summaries do not incorrectly report required env vars as none.

## Best Possible Solution

Keep this issue open for maintainer product/security review. The best path is to choose one canonical scanner-visible metadata contract, then centralize required-env normalization across requires.env, primaryEnv, required envVars, and any explicitly supported root metadata.json shape; update the LLM summary, publish/parser tests, CLI/schema/docs expectations, and the rescan/manual-override plan for affected versions and related reports.
