## Summary

Keep this PR open. Current main does not implement device-flow auth, and the PR is the open implementation path for #613 rather than obsolete work; however, the branch is client-side only and still needs maintainer auth/security review plus the missing server endpoints and verification UI before it could land.

## What This Changes

Adds a CLI `--device` login path, a `deviceAuth` request/poll helper with tests, and login command wiring for device-flow authentication in headless environments.
