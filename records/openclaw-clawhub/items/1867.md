## Summary

Keep this PR open. Current main does not implement device-flow auth, and the PR still exposes a new `--device` login path without the server endpoints or verification UI needed for that path to work safely. The earlier double body-read review finding appears addressed in the latest PR commit, but the remaining blocker is end-to-end auth scope rather than a stale or already-implemented change.

## What This Changes

The PR adds a ClawHub CLI `--device` login option, a device-code request/polling helper with tests, and a device login branch that stores the returned API token.

## Best Possible Solution

Land or supersede this with an end-to-end device-flow auth implementation covering the CLI flag, Convex/server endpoints, verification UI, secure one-time device-code storage, expiry and rate limiting, token creation, docs, tests, and maintainer auth-security review; keep the flag hidden until those pieces are ready.
