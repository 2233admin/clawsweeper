## Summary

Keep open: current main does not implement device-flow auth, and the PR still exposes a client-side `--device` login path before the required server endpoints and verification UI exist. The earlier double body-read review comment was addressed in the latest PR commit, but the remaining issue is broader auth/product/security scope rather than an obsolete PR.

## What This Changes

The PR adds a ClawHub CLI `--device` login option, a device-code request/polling helper with tests, and wires both `login` and `auth login` to call `/cli/device/code` and `/cli/device/token`.

## Best Possible Solution

Land or supersede this with an end-to-end device-flow implementation covering the CLI flag, Convex/server endpoints, verification UI, secure one-time device-code storage, expiry and polling behavior, token minting, docs, tests, and maintainer auth-security review; keep the user-facing flag hidden until those pieces are ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate device login until server support lands:** `packages/clawhub/src/cli.ts:141`
