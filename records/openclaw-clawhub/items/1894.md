## Summary

Keep this issue open. Current main makes the reported ordering plausible rather than resolved: standalone slug-token matches receive stronger lexical boosts than display-name-only matches, while downloads are only a small log-scaled popularity prior. The existing docs mention ranking ingredients and rename aliases at a high level, but they do not answer the publisher-facing discoverability and slug-rename side-effect questions.

## What This Changes

The issue asks for guidance on why a high-download skill ranks low for the query "map", whether standalone slug tokens materially affect search ranking, and what side effects slug renames have.

## Best Possible Solution

A maintainer should answer the publisher directly, decide whether the current map-versus-amap ordering is intended, then either document discoverability guidance covering slug-token effects, downloads, moderation filters, and rename aliases, or adjust ranking weights and tests if that behavior is not desired.
