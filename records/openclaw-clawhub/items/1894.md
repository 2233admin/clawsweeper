## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open in this pass, and current main makes the reported behavior plausible: standalone slug-token matches receive stronger lexical boosts than display-name-only matches, while downloads are only a small log-scaled popularity prior. The existing docs answer the broad ranking ingredients but not the reporter's practical discoverability and rename side-effect questions.

## What This Changes

The issue asks how ClawHub search ranking treats slug tokens, downloads, moderation, and slug renames after a high-download mapping skill ranked below lower-download skills for the query `map`.

## Best Possible Solution

Keep this open for maintainer/product follow-up. A maintainer should answer the reporter directly, decide whether the `map` versus `amap` ordering is intended, then either document discoverability guidance covering slug-token effects, downloads, moderation filters, and rename aliases, or adjust ranking weights and tests if the current ordering is not desired.
