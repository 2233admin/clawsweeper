## Summary

Keep this PR open. Current main moved lexical fallback onto `skillSearchDigest`, but it still uses active digest indexes and filters `nonSuspiciousOnly` after reading rows, so the PR’s central nonsuspicious-index optimization is not implemented on main. The branch is also stale against newer digest fallback behavior and includes a broader fixed-200 client-side search pagination change that main has not adopted.

## Best Possible Solution

Keep this PR open and rebase or rewrite it against current main. The focused path is to make `lexicalFallbackSkills` choose `skillSearchDigest` `by_nonsuspicious_updated` and `by_nonsuspicious_created` when `nonSuspiciousOnly` is set, preserve the current exact-slug handling, digest hydration, updated/created scan windows, highlighted and capability filtering, and keep existing search pagination semantics unless maintainers explicitly approve the broader fixed-200 pagination change. Add tests that assert nonsuspicious digest indexes are selected and that suspicious rows still pass through when the flag is unset.
