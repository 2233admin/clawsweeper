## Summary

Keep open. Current main still has only static analysis, VirusTotal, and ClawScan/LLM security signals; there is no Oathe or behavioral sandbox scan storage, submit/poll lifecycle, public API field, detail-page row, report route, or browse/search sort/filter support. The Oathe team comment and closed, unmerged PR #562 are useful implementation context, but they do not make the feature present in main, and ClawHub issues are not auto-close candidates under the repository policy.

## What This Changes

Add a behavioral sandbox security signal, proposed as Oathe, alongside VirusTotal and ClawScan/LLM with score/verdict/report display and browse/search sort/filter support.

## Best Possible Solution

Keep this open for maintainer product and security review. The best path is to revive or replace PR #562 with a scoped, opt-in behavioral scanner integration: define the provider contract, persist behavioral scan results on skill versions, trigger submit/rescan safely, poll results with bounded crons, expose score/verdict/report data through the public scan API and skill detail UI, then add indexed browse/search sort or filter support once the data model is settled.
