## Summary

Keep this open. The ClawHub cleanup policy says issues are not eligible for auto-close, and current main still does not implement the requested behavioral/Oathe security signal. The Oathe team’s comment and the closed-unmerged #562 PR are useful integration context, but they do not make the feature present in main.

## Best Possible Solution

Keep this issue open for maintainer product and security judgment. The concrete path is to revive or replace #562 with a reviewed implementation that persists behavioral scan results on skill versions, gates provider calls behind explicit configuration, safely triggers and polls scans, exposes verdict/score/report details in the skill detail UI and public API, and adds browse/search sort and filter support once the data model is stable.
