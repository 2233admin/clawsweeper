## Summary

Keep this issue open. Current main already supports deleting published skills through CLI/API soft-delete and restore, with admin-only hard delete, but the ClawHub cleanup policy for this repository only allows auto-closing PRs that are implemented on main. This issue is a user-facing how-to question, so the best next step is maintainer follow-up or clearer dashboard/docs discoverability.

## Best Possible Solution

A maintainer should answer with the supported workflow: `clawhub delete <slug> --yes` soft-deletes a published skill, `clawhub undelete <slug> --yes` restores it, and the v1 API endpoints provide the same behavior. If web self-service is intended, add owner delete/restore controls to the publisher dashboard or skill owner tools and document the distinction between owner soft-delete and admin-only hard delete.
