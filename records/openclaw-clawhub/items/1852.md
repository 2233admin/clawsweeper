## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy says ClawHub issues should remain open for maintainer triage. Current main has registry/package trust controls, but not the requested PAP-style principal, transport authorization, and multi-agent context boundary model.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to decide whether PAP-style trust belongs in OpenClaw runtime architecture, ClawHub package metadata/signing, or a split design, then define the smallest principal/context/audit boundary needed before implementation proceeds.
