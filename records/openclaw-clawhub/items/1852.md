## Summary

Keep open. This is a ClawHub issue, and the repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on main. Current main has package registry trust controls, scan/moderation gates, provenance metadata, trusted publishing, capability metadata, and audit logs, but it does not implement the requested PAP-style runtime principal hierarchy, transport authorization, context minimization, signed action receipts, or multi-agent trust boundary.

## Best Possible Solution

Keep this open for maintainer security/product triage. The next useful step is to decide whether PAP belongs in OpenClaw runtime enforcement, ClawHub package metadata, CLI install policy, or a new extension API, then write the smallest design that defines principals, context requests, authorization receipts, audit storage, and how ClawHub-published skills/plugins declare or consume those boundaries.
