## Summary

Keep open. This is a ClawHub issue, and the repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. Substantively, current main has package catalog trust, verification, scan, and audit-log primitives, but no PAP-style runtime principal hierarchy, transport authorization, signed agent actions, or context-boundary enforcement.

## Best Possible Solution

Keep this issue open for maintainer triage. The concrete path is to decide whether PAP-style trust belongs in OpenClaw runtime/security architecture, ClawHub package metadata/signing, or both; if proceeding, define the smallest core authorization, context-declaration, and audit/receipt boundary that ClawHub plugins can advertise or depend on.
