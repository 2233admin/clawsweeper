## Summary

Keep open. The ClawHub cleanup policy only allows issue auto-closes for PRs that are certainly implemented on main, and current main does not implement the requested PAP-style runtime principal hierarchy, context minimization, transport authorization, signed action receipts, or multi-agent trust boundary.

## What This Changes

The issue requests an optional PAP trust layer for Claw skill execution and inter-instance communication, with declared context needs, principal enforcement, signed action receipts, and auditability.

## Best Possible Solution

Maintainers should first decide whether ClawHub has any PAP-adjacent responsibility, then either close the request explicitly as runtime/out-of-scope or write a small design for the ClawHub-owned parts: package metadata for context requirements, install-policy signals, audit/receipt storage, and the runtime extension API needed to enforce them.
