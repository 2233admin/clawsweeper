## Summary

Keep open. This is a ClawHub issue, and the repository policy for openclaw/clawhub only permits auto-closing pull requests that are definitely implemented on current main; all issues stay open. Substantively, the request is also a security/product architecture proposal for PAP-style runtime trust boundaries, authorization, context scoping, signed actions, and inter-instance auditability. Current main has registry-side package metadata, capability display, trusted publishing, scan status, and audit logs, but not a PAP/principal/transport-layer authorization model for skill execution or collaborating agents.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to decide whether PAP-style trust belongs in OpenClaw runtime/security architecture, ClawHub package metadata/signing, or both, then define the smallest core authorization and audit boundary needed before any ClawHub registry or plugin integration work.
