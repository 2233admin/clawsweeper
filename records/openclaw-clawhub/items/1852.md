## Summary

Keep open. This is a ClawHub issue, and the repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. Substantively, current main has registry/package trust metadata, scans, verification, install warnings, and generic audit logs, but no PAP/principal/transport-layer authorization model for multi-agent execution or context boundaries.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to decide whether PAP-style trust belongs in OpenClaw runtime/security architecture, ClawHub package metadata/signing, or both, then define the smallest core authorization and audit boundary needed before registry or plugin integration work.
