## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the report is substantively valid: current main preserves and exposes the clean staff override, but the public security sidebar and scanner detail route still derive their visible status from raw latest-version scan fields.

## What This Changes

The issue reports that a manually staff-cleared skill/version shows suspicious ClawScan/static-analysis status again after a ClawHub update, even though its public moderation data still has an active clean override.

## Best Possible Solution

Define one consistent public representation for staff-cleared scanner findings, then apply it across the skill detail sidebar, scanner detail pages, and versions tab while preserving raw scanner evidence wherever maintainers want it visible; add focused UI coverage alongside the existing backend moderation tests.
