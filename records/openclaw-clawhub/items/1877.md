ClawHub policy for this cleanup pass keeps issues open, and the report still matches current behavior: the backend preserves and exposes a clean staff override, but the public skill security sidebar and scanner detail route render raw latest-version scan statuses without applying that override.

## What This Changes

The issue requests fixing a regression where a staff-cleared skill/version still shows suspicious static/LLM scan results in public security UI after an update or rescan.

## Best Possible Solution

Define one consistent public representation for staff-cleared scan findings, then apply it across the skill detail sidebar, scanner detail pages, and version tab while preserving raw scanner evidence where maintainers want it available; add focused UI tests plus existing backend moderation coverage.
