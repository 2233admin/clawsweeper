## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only allows auto-closing pull requests that are definitely implemented on current main. The report is also substantively valid: current main preserves and exposes a clean staff override, but the public sidebar and scanner-detail pages still render raw latest-version scan status, and the live page currently shows ClawScan/Static analysis as suspicious for the staff-cleared skill.

## Best Possible Solution

Keep this issue open. Maintainers should decide the intended public semantics for staff-cleared scan findings, then thread moderationInfo or a derived staffCleared state into DetailSecuritySummary and the skill scanner route so non-staff public UI is consistent with the clean manual override while preserving raw scan evidence for staff/owner review as intended.
