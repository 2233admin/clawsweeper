## Summary

Keep open. This is a ClawHub issue, and the cleanup policy for this repo only permits auto-closing pull requests that current main definitely implements. Substantively, the report still looks valid: current main preserves and exposes the clean staff override, but the skill detail sidebar and scanner detail routes still render raw latest-version scan statuses without applying that override.

## What This Changes

The issue reports that a staff-cleared skill can still show raw suspicious scan status in public security UI after a ClawHub update or rescan.

## Best Possible Solution

Keep this issue open and have maintainers decide the public semantics for staff-cleared scan findings. The likely implementation is to pass moderationInfo or a derived staffCleared state into DetailSecuritySummary and the skill security scanner route, show a consistent staff-cleared verdict/message to non-staff viewers, preserve raw scanner evidence for staff or transparent detail views as intended, and add focused React tests plus existing backend moderation coverage.
