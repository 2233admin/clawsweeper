## Summary

Keep this open. Current main still treats user-initiated ClawHub deletion as terminal deactivation: OAuth completion rejects deleted/deactivated users, deletion writes `deactivatedAt`/`purgedAt` while clearing profile fields, active-user helpers exclude those users, admin unban excludes permanently deleted accounts, and docs/UI describe deletion as permanent. The provided timeline also shows multiple related open login-loop/account-lifecycle reports, so this needs maintainer product/support triage rather than conservative cleanup closure.

## What This Changes

The issue requests server-side recovery, purge, restore, unlink, or relink support for a ClawHub GitHub OAuth login loop after account deletion and subsequent GitHub account change.

## Best Possible Solution

Keep this open for ClawHub account lifecycle triage. Maintainers should define the post-deletion state machine, then implement one explicit path: safe admin/self-service purge, restore, unlink, or relink for GitHub identities after deletion, or a clear terminal error plus support route if deletion must remain irreversible. The related login-loop reports should be handled as one cluster with docs and tests updated together.
