## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open, and current main still treats user-initiated account deletion as irreversible rather than providing the requested recovery, purge, unlink, or relink path. The discussion also frames this as a broader account lifecycle gap with related open reports, not a resolved one-off login bug.

## Best Possible Solution

Keep this open for maintainer triage. The durable fix likely belongs in ClawHub account lifecycle work: either add a documented admin/self-service recovery path that can safely purge or relink GitHub OAuth identities after deletion, or make the irreversible-deletion policy and user-facing OAuth error path explicit enough that users do not experience a silent refresh loop.
