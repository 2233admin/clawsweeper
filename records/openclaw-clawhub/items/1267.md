## Summary

Keep this issue open. The repository-specific cleanup policy says ClawHub issues should remain open, and current main still does not provide the requested server-side purge, self-service recovery, GitHub identity unlink, or relink path after account deletion. The code now gives clearer deleted-account auth messaging, but it intentionally blocks reauth for deleted/deactivated users rather than letting the reporter sign in with a new or restored account.

## Best Possible Solution

Keep this open for maintainer triage as ClawHub account lifecycle work. A durable fix should add a safe documented admin or self-service path to purge, restore, unlink, or relink GitHub OAuth identities after account deletion, or make the irreversible-deletion policy and OAuth failure path explicit enough that users get a clear terminal error rather than a silent refresh loop.
