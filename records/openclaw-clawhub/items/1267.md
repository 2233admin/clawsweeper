## Summary

Keep this issue open. Current main still treats user-initiated account deletion as irreversible deactivation: OAuth completion rejects deleted/deactivated users, account deletion writes deactivated/purged state, active-user helpers hide those accounts from sessions, tests assert blocked reauth, and docs describe deleted accounts as not restorable by login. The ClawHub cleanup policy also says issues should remain open unless a strong close reason applies; implemented-on-main auto-close is reserved for PRs in this repository.

## Best Possible Solution

Keep this open for ClawHub account lifecycle triage. Maintainers should define the intended post-deletion state machine, then add a safe admin or self-service path to purge, restore, unlink, or relink GitHub OAuth identities after deletion, or else make the irreversible policy surface a clear terminal error with a support route. The related open reports should be handled as one account lifecycle cluster.
