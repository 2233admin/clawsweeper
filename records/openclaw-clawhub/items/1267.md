## Summary

Keep this open. Current main still treats user-initiated ClawHub deletion as terminal deactivation: GitHub OAuth completion rejects deleted/deactivated users, active-user helpers exclude them, admin unban is not a restore path for deactivated accounts, and docs/UI describe deletion as permanent. The related login-loop reports remain open, so this needs maintainer account-lifecycle and support triage rather than conservative cleanup closure.

## What This Changes

The issue requests server-side cleanup or recovery for a GitHub OAuth login loop after deleting a ClawHub account and trying to sign in with a different established GitHub account.

## Best Possible Solution

Define the post-deletion account lifecycle for GitHub-linked users, then implement one supported outcome across backend, admin/support tooling, UI errors, docs, and tests: restore, purge/recreate, unlink/relink, or an explicitly terminal state with a support route.
