## Summary

Keep open. Current main still has no self-service recovery, purge/recreate, unlink/relink, or support-admin restore path for user-initiated deleted ClawHub accounts; the OAuth and active-user paths intentionally reject deactivated users. The provided timeline also shows several related account-deletion login-loop reports are still open, and the ClawHub cleanup policy keeps issues open for maintainer handling rather than auto-closing them.

## What This Changes

The issue requests a fix or server-side cleanup for a ClawHub login refresh loop after account deletion and a later GitHub account change.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user/auth account records, admin or support tooling, user-facing errors, documentation, and tests: restore, purge/recreate, unlink/relink, or a deliberately terminal state with a clear support route.
