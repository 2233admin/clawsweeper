## Summary

Keep this issue open. It is an issue in openclaw/clawhub, where the review policy only permits auto-closing PRs that are certainly implemented on main. The current main code also still leaves the reported package publish behavior actionable: local folder publishes only send an owner when `--owner` is explicit, while GitHub shorthand sources fetch the archive before `--source-path` can select a monorepo subdirectory.

## Best Possible Solution

Keep this issue open and make a focused CLI/package-publish fix: apply `--source-path` to GitHub shorthand sources before archive extraction, add regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`, and clarify the org publishing contract by either implementing safe owner inference from scoped package/source repo data or documenting `--owner <org>` as the supported flow with better help and errors.
