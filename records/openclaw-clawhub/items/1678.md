## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on main, and this item is an issue. The current main code also still leaves the reported behavior actionable: local folder publishes do not infer an org owner from a scoped package name or source repo, and GitHub shorthand sources are fetched before `--source-path` can select a monorepo subdirectory.

## Best Possible Solution

Keep this issue open and make a focused CLI/package-publish fix: apply `--source-path` to GitHub shorthand sources before archive extraction, add regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`, and clarify org publishing by either implementing safe owner inference from scoped package/source repo data with membership checks or documenting `--owner <org>` as the supported flow with better help and errors.
