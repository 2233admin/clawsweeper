## Summary

Keep open. Repository policy for openclaw/clawhub keeps issues out of auto-close, and current main still leaves the reported publish behavior unresolved: GitHub shorthand sources are fetched before `--source-path` affects the selected folder, while local folder publishes only send an org owner when `--owner` is explicitly provided.

## What This Changes

Make `clawhub package publish` publish org-scoped monorepo packages under the org namespace and honor `--source-path` for `owner/repo@ref` sources.

## Best Possible Solution

Keep this open. A narrow code fix should apply a validated `--source-path` to GitHub shorthand/URL sources before `fetchGitHubSource` extracts files and should add a regression for `clawhub package publish owner/repo@ref --source-path plugins/foo`. Maintainers should also decide the owner contract: either keep org publishing explicit via `--owner <org>` and correct docs/help, or add carefully authorized inference from scoped package names/source repos.
