## Summary

Keep open. Repository policy for openclaw/clawhub keeps issues open in this cleanup pass, and current main still leaves the core reported workflow unresolved: `owner/repo@ref --source-path plugins/foo` does not apply the subpath before fetching/inspecting the GitHub archive, and org ownership is still explicit via `--owner` rather than inferred from an npm scope or source repo.

## Best Possible Solution

Keep this open for maintainer follow-up. A focused fix should apply `--source-path` to GitHub shorthand sources before `fetchGitHubSource` and add regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`. Separately, maintainers should decide the org-namespace contract: either infer a publisher from scoped package/source metadata with the existing membership checks, or keep ownership explicit via `--owner <org>` and update CLI help/docs to make that required workflow clear.
