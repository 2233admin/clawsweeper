## Summary

Keep open. Current main still leaves the reported monorepo GitHub source-path behavior unresolved, and org publishing is explicit through `--owner` / `ownerHandle` rather than inferred from package scope or `--source-repo`. Repository policy also keeps ClawHub issues open in this cleanup pass.

## What This Changes

The issue requests that `clawhub package publish` publish org-scoped packages under the org namespace and support `owner/repo@ref --source-path <subdir>` for monorepo package publishing.

## Best Possible Solution

Apply `--source-path` to GitHub shorthand/URL sources before archive fetch and add a regression for monorepo subdirectory publishing. Separately, choose and document the owner contract: either keep org publishing explicit with `--owner <org>` or implement carefully authorized owner inference from package/source metadata.
