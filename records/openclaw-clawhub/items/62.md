## Summary

Keep open. This is a ClawHub issue, and the repository-specific policy for this review run only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has substantial malicious-skill mitigations, but the original malicious distribution concern is still a live maintainer security-policy question because malware-blocked skill pages and raw file reads remain intentionally/publicly reachable for transparency.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked skill content should remain publicly readable for transparency or be blocked/redacted across raw file, API, install, and page surfaces. If distribution prevention is the goal, add explicit malware checks to /api/v1/skills/{slug}/file and any remaining raw/install-adjacent surfaces while preserving public scan metadata, and separately evaluate AgentGuard or another scanner as a reviewed integration.
