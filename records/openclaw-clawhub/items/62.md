## Summary

Keep open. This is an open ClawHub security issue, and the repository cleanup policy only allows auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, current main has meaningful malicious-skill mitigations, but the strict malicious-distribution question is still unresolved: malware-blocked skills can remain publicly inspectable for transparency, raw/readme file access is intentionally allowed on some paths, and the linked AgentGuard work in PR #75 was closed unmerged rather than shipped in core.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked skill content should remain publicly inspectable for transparency or be blocked/redacted consistently across detail pages, readme/file APIs, downloads, and CLI/install surfaces. If strict distribution prevention is desired, add malware-blocked checks or redaction to raw/readme paths and replace install prompts on blocked pages with a clear blocked state, while preserving safe scan metadata for review. Treat AgentGuard from PR #75 as prior art to evaluate, not as shipped protection.
