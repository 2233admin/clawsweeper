## Summary

Keep open. This is an issue in openclaw/clawhub, and the provided cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, current main has real malware mitigations, but the original distribution-policy question is still unresolved because malware-blocked skills remain intentionally readable through transparency/file surfaces while install and download paths are blocked. AgentGuard from the discussion/PR #75 is useful prior art, but it is not integrated on main.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked content should remain transparent or be blocked/redacted consistently across raw file, readme/page viewer, API, download, CLI install/update, and package-skill alias surfaces. If distribution prevention is the goal, add explicit malware-blocked checks to canReadSkillVersionFiles, /api/v1/skills/{slug}/file, and skill-backed /api/v1/packages/{name}/file while keeping redacted moderation metadata visible; evaluate AgentGuard from PR #75 as separate prior art, not shipped functionality.
