## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main has meaningful malware mitigations, but the original distribution-policy question is still unresolved because malware-blocked skill files remain readable for transparency through file/readme surfaces and the raw file HTTP endpoint. AgentGuard from PR #75 is useful prior art from the discussion, but that PR was closed unmerged and is not integrated on main.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked content should remain publicly transparent or be blocked/redacted consistently across raw file, readme/page viewer, API, download, and CLI install/update surfaces. If distribution prevention is the goal, add explicit malware-blocked checks to canReadSkillVersionFiles, /api/v1/skills/{slug}/file, and related viewer/API paths while keeping redacted scan metadata visible; evaluate AgentGuard from PR #75 as separate prior art rather than treating it as already shipped.
