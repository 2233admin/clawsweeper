## Summary

Keep this issue open. The ClawHub cleanup policy only auto-closes PRs that are certainly implemented on current main, and current main still lacks the requested tailored VirusTotal false-positive appeal action: the suspicious-skill owner banner links to the generic GitHub issues list, while the security summary, scanner detail page, dashboard rescan path, and report dialog do not provide a prefilled appeal mailto or templated false-positive flow. Related open appeals (#329, #392, #579, #1669) show this remains an active user/support problem rather than stale cleanup.

## What This Changes

Add a small owner-facing false-positive appeal link or button on flagged skill pages, preferably emailing security@openclaw.ai or opening a templated report.

## Best Possible Solution

Keep this issue open and implement a focused UI fix: add a clear false-positive appeal action on flagged skill pages, preferably a private prefilled mailto to security@openclaw.ai with safe context such as owner, slug, version, canonical URL, scanner statuses, and scanner detail links. Preserve existing rescan, report, scanner detail, and staff manual override behavior, and add targeted component/page tests for the new appeal action.
