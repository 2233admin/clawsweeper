## Summary

Keep this issue open. The ClawHub cleanup policy does not permit automated closure of issues, and current main still preserves the reported anonymous-vs-authenticated split: authenticated downloads use a per-user bucket, while anonymous downloads use an IP bucket that can fall back to `ip:unknown` when no trusted client IP header is present.

## Best Possible Solution

Keep this open for maintainer or ops follow-up. Verify which client-IP headers reach Convex for anonymous package and skill download requests, use the existing `rate_limit_denied` fields (`ipSource`, `hasClientIp`) to distinguish real shared-IP throttling from `ip:unknown` collapse, then adjust trusted proxy/header configuration or anonymous bucket behavior as needed. Track any remaining OpenClaw install-client retry-header display gap separately if that client still hides `Retry-After` and rate-limit metadata.
