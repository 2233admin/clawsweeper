## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. Substantively, this remains a live hosted-skill moderation appeal: the production API still reports Overlay Market v0.1.6 as suspicious for potential exfiltration, while current main has staff manual override tooling but no publisher-side annotation or cache-read exception that would clear this automatically.

## Best Possible Solution

Keep this issue open for maintainer/moderator review of the Overlay Market listing. If the appeal is accepted, use the existing manual override flow with an audit note. If maintainers want publisher-side false-positive handling, add a narrow scanner enhancement for benign cache-directory reads or documented annotations, with tests that preserve detection of real file-content exfiltration. The publisher should also use metadata.openclaw.requires.env so RPC and private-key env usage is machine-readable.
