## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review only permits auto-close for PRs that are certainly implemented on main. The current main search path also still has an actionable core search gap: tags are stored in the skill digest, but lexical search matches only display name, slug, and summary, so a skill can still miss tag-term searches when vector recall does not retrieve it.

## Best Possible Solution

Keep the issue open or manually consolidate it with the broader open search-recall work in #1254. The code path likely needs an indexed/direct lexical candidate source over normalized searchable fields, including tags and maybe title/slug/summary tokens, while preserving the existing nonSuspiciousOnly semantics and adding focused tests for tag terms such as phone and voice.
