## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits automated closes for PRs that are certainly implemented on current main, and this item is an issue. Current main also still has a real search-recall gap: skill tags are stored in the lightweight search digest, but lexical matching for skill search still only checks display name, slug, and summary, so explicit tag terms like phone or voice are not guaranteed to surface a skill unless vector recall happens to retrieve it.

## Best Possible Solution

Keep this issue open, or have maintainers manually consolidate it with #1254. The likely fix belongs in convex/search.ts and the search digest schema: add a bounded lexical candidate source over normalized searchable fields that includes tags and capability tags, preserve existing nonSuspiciousOnly behavior, and add focused tests for tag queries such as phone and voice.
