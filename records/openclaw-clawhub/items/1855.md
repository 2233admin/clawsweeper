## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: `clawhub inspect --json` omits top-level skill moderation, plain inspect only prints version-level security after a version fetch, and the version response schema still does not type `security`. The draft PR remains a relevant implementation candidate for the open CLI security visibility request in #1483.

## Best Possible Solution

Keep this PR open and review or land it, or an equivalent change, as the implementation path for #1483. The main fix should pass `skillResult.moderation` through inspect JSON, print a concise plain-text skill moderation summary from the skill detail response without requiring a version fetch, keep version `security` typed in the CLI schema, and preserve focused regression coverage.
