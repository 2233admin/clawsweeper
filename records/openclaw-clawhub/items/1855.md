## Summary

Keep open. Current main has backend/API support for skill-level moderation, and the CLI schema can parse it, but `clawhub inspect` still drops that data from JSON output and never prints a plain-text moderation summary. This PR is therefore still useful work toward open #1483 rather than obsolete or already implemented.

## Best Possible Solution

Keep this PR open as the current implementation candidate for #1483. The best path is to review and land this PR or an equivalent change that passes `skillResult.moderation` through inspect JSON, prints a concise plain-text moderation summary, and keeps the version-security schema aligned with the v1 version response.
