## Summary

Keep this draft PR open. Current main has backend/API support for skill-level moderation and the CLI schema already accepts the skill-detail moderation block, but `clawhub inspect` still omits it from JSON and never prints it in plain output. The PR remains useful implementation work for open #1483, and its diff is limited to CLI inspect output, schema typing, and tests rather than being obsolete on main.

## Best Possible Solution

Keep this PR open as the implementation candidate for #1483. The best path is to review and land this PR or an equivalent change that passes `skillResult.moderation` through inspect JSON, prints a concise plain-text moderation summary, and keeps the CLI schema aligned with the v1 version security response.
