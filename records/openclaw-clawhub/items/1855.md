## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: `clawhub inspect` accepts the skill detail API's moderation field in the schema, but it drops that field from JSON output and never prints a plain-text moderation summary. The PR remains a useful implementation candidate for the related open security/CLI request.

## Best Possible Solution

Keep this PR open as the implementation candidate for the related CLI security/moderation request. The best path is to review and land this PR or an equivalent change that passes `skillResult.moderation` through inspect JSON, prints a concise plain-text moderation summary, and keeps the version security schema aligned with the v1 version response.
