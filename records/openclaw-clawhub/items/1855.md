## Summary

Keep open. Current main does not implement the PR’s intended inspect-output changes: the backend already returns moderation metadata, but the CLI still omits it from `inspect --json` and does not print skill-level moderation in ordinary `inspect` output. The patch is a draft implementation with no blocking correctness or security finding from this read-only review, so it needs maintainer review and validation rather than cleanup closure.

## What This Changes

The PR updates the ClawHub CLI inspect command and schemas to show skill moderation in plain and JSON output, type version security metadata, and add inspect regression tests.

## Best Possible Solution

After maintainer review, the CLI should pass through the top-level moderation object already returned by the v1 skill detail API, print a concise moderation summary in ordinary inspect output, preserve version security typing, and keep focused regression coverage; #1483 should close only after the implementation lands.
