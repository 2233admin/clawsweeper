## Summary

Keep open. Current main still contains the reported transfer-request payload bug: the CLI builds a JSON string body before handing it to `apiRequest`, and both HTTP transport paths serialize `args.body` again. The repository cleanup policy also says ClawHub issues should not be auto-closed, so this live issue needs a normal fix rather than cleanup closure.

## Best Possible Solution

Keep this issue open. Fix `packages/clawhub/src/cli/commands/transfer.ts` by passing a plain object body to `apiRequest`, update the transfer command or HTTP client regression test to assert the final POST payload is a JSON object containing `toUserHandle`, and audit the adjacent ownership commands that still pre-serialize POST bodies.
