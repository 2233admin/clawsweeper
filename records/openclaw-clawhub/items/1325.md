Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are definitely implemented on current main. Substantively, current main still has the broad `systemPrompt:` false-positive path in both the static Markdown scanner and the LLM pre-scan signal, while clearing this specific hosted skill is a staff moderation action.

## What This Changes

The issue asks maintainers to review and clear the `agentphone-ai` skill because legitimate `systemPrompt` and `system_prompt` API parameter examples are being flagged as prompt-injection patterns.

## Best Possible Solution

Keep this issue open for maintainer/security disposition. If `agentphone-ai` has been reviewed and is legitimate, staff should clear the hosted moderation state with a manual override; separately, calibrate the static Markdown scanner and LLM pre-scan so code examples using API parameter names like `systemPrompt` or `system_prompt` do not trigger `suspicious.prompt_injection_instructions`, while preserving detection for real override instructions such as `ignore previous instructions` and prose `system prompt:` injection attempts.
