## Summary

Keep this PR open. Current main still does not implement the PR’s central digest-pagination and lazy-file-fetch performance change: the GitHub backup page query still paginates full `skills` documents, performs per-row `skillVersions` and `users` reads, and returns version `files` eagerly for the node sync action. Main only contains partial overlap in the GitHub fast-forward retry behavior.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to revise the digest-based backup scan so it pages over `skillSearchDigest`, preserves handle-less owner fallback, revalidates or refreshes owner state where needed, fetches version files lazily only after a GitHub version mismatch, and keeps the fast-forward retry behavior already present on main.
