## Summary

Keep this PR open. Current main still does not implement the PR’s central digest-pagination change for GitHub backup sync: the backup page query still scans full `skills` documents, still joins version and owner records per row, and still returns version `files` eagerly for the node action to pass into `backupSkillToGitHub`. Main has partial overlap with the PR’s later fast-forward retry work, but not the digest scan or lazy file fetch performance work.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to update the digest-based backup scan so `getGitHubBackupPageInternal` pages over `skillSearchDigest`, preserves handle-less owner fallback behavior, revalidates or refreshes owner state where needed, fetches version files lazily only after a GitHub version mismatch, and keeps the fast-forward retry behavior already present on main.
