## Summary

Keep this PR open. Current main still does not implement the central digest-based GitHub backup pagination or lazy version-file loading: it scans `skills`, joins `skillVersions` and `users` per eligible row, and passes `files` eagerly into the Node backup action. Main does already contain the adjacent fast-forward retry behavior, but that does not supersede the PR’s performance change, and the PR discussion still has owner fallback/stale-owner correctness concerns to resolve before merge.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to revise the digest-based backup scan so `getGitHubBackupPageInternal` pages `skillSearchDigest`, fetches version files lazily only after a GitHub version mismatch, preserves the current owner-id fallback for owners without handles, and either revalidates live owner state or makes skill digest owner metadata reliably refresh when users rename, deactivate, or delete accounts. Keep the already-present fast-forward retry behavior.
