## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only allows auto-closing pull requests when current main definitely implements the intended change. The report still maps to a live, source-backed visibility gap: current web search intentionally asks the backend for non-suspicious skills only, while exact slug lookup also drops suspicious matches under that flag, so a public skill can be reachable by detail/API routes while being hidden from the search UI if moderation marks it suspicious. The existing bot comment and cross-references to #131, #137, #382, and #423 correctly point this toward maintainer moderation/product triage rather than automated closure.

## What This Changes

The reporter uploaded a skill with slug cloudq and asks why searching ClawHub for cloudq does not find it.

## Best Possible Solution

A maintainer should inspect the production moderation state for cloudq, rescan or clear it if the suspicious verdict is wrong, and decide whether ClawHub should make owner-facing search/status behavior clearer when a skill has a public detail page but is suppressed from public non-suspicious search. Any code change should be coordinated with the existing publish/search visibility reports rather than treated as an isolated slug fix.
