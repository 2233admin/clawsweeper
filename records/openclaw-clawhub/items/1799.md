## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only allows auto-closing ClawHub pull requests that current main certainly implements. Substantively, ZenQuote v1.0.7 still has a live suspicious moderation state: static and LLM scans are clean, but the VirusTotal/Palm result cites cron JSON/prompt-injection risk in the published setup script, so clearing it needs moderator review of the live artifact rather than source-only cleanup.

## What This Changes

The issue asks maintainers to review ZenQuote v1.0.7 as a claimed VirusTotal false positive, possibly whitelist it, document curl-based skill API calls, or improve scanner handling for legitimate read-only API usage.

## Best Possible Solution

A moderator should review the published ZenQuote v1.0.7 files and scan details. If the cron setup interpolation is acceptable or has been fixed by a republish/rescan, clear the suspicious state through the existing audited override or rescan flow; if the JSON/prompt-injection concern is valid, ask the author to republish with safe JSON construction and validation before clearing. Any broader scanner/docs work should distinguish read-only API curl calls from persistence, downloads, shell execution, and user-controlled cron payload construction.
