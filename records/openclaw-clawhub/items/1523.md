## Summary

Current main still does not implement this PR’s intended production smoke hardening. The deploy workflow does not run the production HTTP smoke, does not include the signed-out publish/import Playwright file in production UI smoke, and the production HTTP smoke test still lacks the PR’s retry/memoization hardening.

## Best Possible Solution

Keep this PR open for maintainer review. The useful path is to rebase or merge the smoke-coverage changes into main, preferably removing the unreachable post-loop code in `fetchWithRetry`, so production deploys run the HTTP smoke before UI smoke and include signed-out publish/import coverage.
