## Summary

Keep open. Current main does not implement the PR’s intended deploy-smoke hardening, and the live PR still has a blocking production-smoke assertion mismatch plus minor dead retry fallback code. Because this is an unmerged implementation PR touching production deploy CI, it needs maintainer review and branch cleanup rather than auto-close.

## What This Changes

The PR adds a production HTTP smoke step to the deploy workflow, expands production UI smoke to include publish/import entry coverage, adds bounded 429 retry handling and skill-detail memoization to prod HTTP smoke, and updates the signed-out upload redirect assertion.

## Best Possible Solution

Land a corrected version that wires the existing prod HTTP smoke script into deploy CI, expands production UI smoke to publish/import workflows only after their assertions match the rendered signed-out gates, keeps bounded 429 retry behavior and skill-detail memoization, removes unreachable retry fallback code, and preserves the current deploy secret and action boundaries.
