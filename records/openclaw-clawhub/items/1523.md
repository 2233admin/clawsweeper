## Summary

Keep this PR open. Current main still does not implement the PR’s intended deploy hardening: the production deploy workflow does not run the prod HTTP smoke, does not include the signed-out publish/import Playwright file in the production UI smoke command, and the prod HTTP smoke test still lacks the PR’s rate-limit retry hardening.

## Best Possible Solution

Keep this PR open for maintainer review. The useful path is to rebase or merge the smoke-coverage changes into main, preferably removing the unreachable post-loop code in `fetchWithRetry`, so production deploys run `test:e2e:prod-http` before UI smoke and include signed-out publish/import coverage.
