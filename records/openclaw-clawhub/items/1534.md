## Summary

Keep open. Current main still renders package icons beside download totals in shared public skill stat/list components, and this issue is already paired with open draft PR #1854, so it should remain open until that PR lands or is explicitly superseded.

## What This Changes

The issue requests consistent iconography for download counts, preferably using a download/down-arrow icon everywhere instead of mixing package and download icons across public and publisher surfaces.

## Best Possible Solution

Review and land or supersede PR #1854. The end state should use a download-specific lucide icon for public skill download metrics in SkillStatsTripletLine, SkillMetricsRow, and SkillListItem, while preserving Package for version/package concepts and avoiding stat or layout changes.
