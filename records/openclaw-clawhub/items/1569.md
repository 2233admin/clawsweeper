## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits automated close proposals for pull requests that are certainly implemented on current main, and this item is an open issue. The source also still models skill transfers as user-to-user: transfer requests take `toUserHandle`, org publishers are excluded by recipient resolution, the transfer table stores user IDs only, and acceptance moves ownership to the recipient's personal publisher. The related #1431 remains open, and #1635 is an open implementation candidate rather than merged main.

## Best Possible Solution

Keep this issue open and resolve it through #1635 or a maintainer-selected successor. The durable fix should make skill transfer targets publisher-based rather than user-only, update the Convex schema and mutations, HTTP API, CLI contract, docs, authorization rules, and tests, with explicit coverage for personal-to-org, org-to-user, and org-to-org flows.
