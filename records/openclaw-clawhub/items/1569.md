## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits automated close proposals for PRs that current main certainly implements, and current main still models skill ownership transfer as user-to-user rather than publisher/org-targeted.

## Best Possible Solution

Keep this issue open and resolve it through #1635 or a maintainer-selected successor. The durable fix should make transfer targets publisher-based rather than user-only, update the Convex transfer schema/mutations/queries, HTTP API, CLI contract, authorization rules, docs, and tests, with explicit coverage for personal-to-org, org-to-user, and org-to-org flows.
