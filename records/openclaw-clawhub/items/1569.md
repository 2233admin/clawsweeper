## Summary

Keep this issue open. ClawHub cleanup policy does not auto-close issues, and current main still implements skill transfers as user-to-user only: the CLI sends `toUserHandle`, the HTTP endpoint requires `toUserHandle`, the backend resolves that handle to an active user or personal user publisher, and the transfer table stores `fromUserId`/`toUserId` rather than publisher/org targets. Related #1431 and #1675 remain open, and #1635 is an open, unmerged implementation candidate for org-aware ownership transfers.

## What This Changes

The issue requests a way to transfer skills from a personal publisher to an organization publisher so other organization users can publish and update them.

## Best Possible Solution

Keep this issue open and resolve it through review and landing of #1635 or a maintainer-selected successor. The implementation should make transfer targets publisher-aware, support personal-to-org plus the existing user-to-user flow, define acceptance/approval semantics for org members, update Convex schema/backfill/API/CLI/docs/tests, and preserve aliases, stats, audit logs, and existing install URLs as much as the product policy allows.
