## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing PRs that are certainly implemented on current main, and this item is an issue. Substantively, current main still models the public transfer flow as user-to-user: the CLI/API send `toUserHandle`, the backend resolves only users or personal user publishers, the transfer table stores `fromUserId`/`toUserId`, and accepting a transfer moves ownership to the recipient's personal publisher. Related #1431 and #1675 remain open, and #1635 is an open, unmerged implementation candidate for org-aware ownership transfer.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing #1635 or a maintainer-selected successor. The implementation should make skill transfers target publishers/organizations rather than only users, preserve or migrate existing user-to-user transfer state safely, update Convex mutations/queries, HTTP API, CLI UX/schema, docs, authorization, audit logs, and tests for personal-to-org, org-to-user, org-to-org, and existing user-to-user flows. After that lands, this issue plus #1431 and #1675 can be closed as implemented or superseded by the merged change.
