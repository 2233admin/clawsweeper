## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still does not support transferring a skill from a personal publisher to an organization publisher. The transfer path is still user-recipient based, the schema stores user-to-user transfer ids, and related #1431/#1675 remain open while #1635 is an open, unmerged implementation candidate.

## Best Possible Solution

Keep this issue open and resolve it through review and landing of #1635 or a maintainer-selected successor. The durable fix should make skill ownership transfers target publishers/orgs rather than only users, migrate or extend transfer state safely, update Convex mutations/queries, HTTP API, CLI contract, docs, authorization rules, and tests for personal-to-org, org-to-user, and org-to-org flows. After that lands on main, this issue plus #1431/#1675 can close as implemented or be superseded by the merged change.
