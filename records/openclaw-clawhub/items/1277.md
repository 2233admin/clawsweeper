## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, current main still treats `child_process` spawn/exec usage as a non-allowlisted static finding, does not recognize `HOOK.md`/`spawns_process`/`openclaw:allow` as a suppression path, and only offers staff/manual review plus rescan workflow around suspicious findings.

## Best Possible Solution

Keep this issue open for explicit maintainer/security triage. The best path is either to document that intentional `child_process` use remains a review finding handled through staff manual override/rescan, or to design a narrow allowlist that validates trusted hook metadata, exact executable/script paths, literal argv without shell interpretation, constrained environment inheritance, and tests/docs for the moderation UI and public security API.
