## Summary

Keep open. The repository-specific policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on main, and this item is an issue. Substantively, current main still flags `child_process` spawn/exec usage as `suspicious.dangerous_exec`, does not recognize `HOOK.md`, `spawns_process`, or `openclaw:allow` as scanner suppressions, and only externally clears a different static code.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is either to document that intentional process spawns remain review findings handled through staff manual override, or to implement a narrow scanner allowlist that validates the exact executable/script path, literal arguments without shell interpretation, constrained environment inheritance, and trusted hook metadata, with tests in the moderation/package publish coverage and docs in the skill format or security docs.

- The named published skill still needs separate production moderation judgment; source changes alone would not prove that listing has been reviewed or cleared.
