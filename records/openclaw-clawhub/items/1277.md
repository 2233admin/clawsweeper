## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review lane only permits auto-closing PRs that current main definitely implements. Substantively, current main still flags `child_process` plus spawn/exec patterns as `suspicious.dangerous_exec`, does not parse `HOOK.md` `spawns_process` metadata or `openclaw:allow` comments as suppressions, and intentionally keeps non-allowlisted static findings surfaced even when external scanners are clean.

## What This Changes

The issue requests either documentation for intentional hook process-spawn suppressions or scanner support for treating `HOOK.md` `spawns_process: true` as an allowlist signal.

## Best Possible Solution

Keep this issue open for explicit maintainer/security triage. The best path is either to document that intentional `child_process` usage remains a surfaced static-analysis review finding handled through owner rescan/manual moderation workflows, or to design a narrow allowlist that validates trusted hook metadata, exact executable/script paths, literal argv with no shell interpretation, constrained environment inheritance, and updates scanner tests, docs, API/UI behavior, and rescan guidance together.
