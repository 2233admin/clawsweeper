## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy for this repository only permits auto-closing pull requests that current main definitely implements. Substantively, current main still has no documented or implemented suppression path for declared hook process spawns, and `suspicious.dangerous_exec` remains a surfaced static finding.

## What This Changes

The issue asks ClawHub to either document the correct suppression mechanism for intentional hook process spawns or recognize `HOOK.md` `spawns_process: true` metadata as an allowlist signal in the security scanner.

## Best Possible Solution

Define an explicit scanner policy for declared process spawns: either document `suspicious.dangerous_exec` as an always-surfaced manual-review finding, or implement a narrow trusted allowlist with validation for hook metadata, exact executable/script paths, literal argv/no shell interpretation, constrained environment inheritance, owner rescan behavior, API/UI presentation, docs, and regression tests.
