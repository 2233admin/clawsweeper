## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing PRs that are certainly implemented on current main, and this item is an issue. The technical review also shows current main does not restore self-deleted users on re-authentication; it now treats user-initiated deletion as permanent, while earlier history/comment context shows #177 previously attempted a soft-delete reauth fix.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether self-deleted accounts are permanently non-restorable; then either improve the deletion/login UX and support messaging, or implement a safe restoration flow that matches the GitHub identity/provider account and adds regression tests for wrong-identity restoration.
