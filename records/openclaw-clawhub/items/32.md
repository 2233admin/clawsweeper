## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on current main, and this item is an issue. The technical review also shows current main does not implement the reported self-deleted-user reauthentication path: account deletion now writes permanent deactivation state, auth blocks deactivated or legacy deleted users from signing in, and docs/UI describe deletion as irreversible.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated deletion is permanently non-restorable; then either improve the deletion/login UX and support messaging, or implement a safe restoration flow with strict provider identity matching and regression tests.
