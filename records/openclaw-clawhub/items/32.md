Keep this issue open. The original report and follow-up comments describe a concrete login failure after account deletion, and current main now intentionally blocks re-authentication for deleted/deactivated accounts rather than restoring them. Because this is an open ClawHub issue, the repository policy also says this workflow should keep issues open and reserve auto-close for PRs that are definitely implemented on main.
