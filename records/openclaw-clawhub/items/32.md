## Summary

Keep this issue open. The repository-specific policy for openclaw/clawhub only permits auto-closing pull requests when current main definitely implements the intended change, and this item is an issue. The technical review also shows current main does not implement the requested soft-deleted-account re-authentication behavior; it now treats user-initiated deletion as irreversible and blocks sign-in.

## Best Possible Solution

Keep this issue open for maintainer product and support triage. Maintainers should decide whether self-deleted accounts are intentionally non-restorable, in which case the UX/docs/support path should make that clear, or implement a deliberate reactivation flow that safely matches the GitHub identity/provider account and includes negative tests for wrong-identity reactivation.
