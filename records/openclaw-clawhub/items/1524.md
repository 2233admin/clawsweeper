## Summary

Keep this issue open. Current main still has a broad standalone `swap` Crypto trigger, publish/backfill still derive persisted capability tags from uploaded text files, and draft PR #1857 directly targets this false-positive path but remains open and unmerged.

## What This Changes

A contributor asks what criteria triggered the Crypto capability signal for a non-crypto visa itinerary skill and whether the label can be removed.

## Best Possible Solution

Keep this issue open and review or land #1857, or an equivalent maintainer patch: tighten Crypto matching so generic CSS/web text like `display=swap` does not trigger Crypto, preserve positive detection for real DeFi/token swap language, add regression tests, and decide whether existing persisted tags need targeted correction. A small UI/API improvement exposing matched capability evidence would also help future diagnostic questions.
