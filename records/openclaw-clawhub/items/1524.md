## Summary

Keep this issue open. Current main still has the broad Crypto matcher that treats standalone `swap` as crypto, publish-time capability derivation still scans uploaded text files, and the related draft PR #1857 is an open implementation candidate rather than a merged fix. ClawHub cleanup policy also keeps issues open; only pull requests definitely implemented on main are eligible for auto-close.

## Best Possible Solution

Keep this issue open and review or land #1857, or an equivalent maintainer patch: tighten Crypto matching so generic web/CSS text like `display=swap` does not trigger Crypto, preserve positive detection for real DeFi/token swap language, add regression tests, and decide whether already-published capability tags need a backfill/manual correction. A small docs/UI improvement exposing matched capability evidence would also help future reporters diagnose labels without guessing.
