## Summary

Current main still has the broad Crypto false-positive path: `swap` is a standalone Crypto regex, that matcher is used during publish/backfill, and the linked draft PR that references this issue remains open and unmerged. This ClawHub issue should stay open for maintainer review of the paired implementation rather than being closed or routed to a separate fix lane.

## What This Changes

The issue asks what triggers the Crypto capability signal and how a non-crypto skill can remove a likely false-positive Crypto label.

## Best Possible Solution

Review and land #1857 or an equivalent maintainer patch that narrows Crypto swap detection, keeps real DeFi/token swap detection, adds regression coverage for generic `display=swap` text, and decides how to correct already-persisted tags. A follow-up evidence surface for matched capability criteria would make this class of support question easier to answer.
