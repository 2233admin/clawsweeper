## Summary

Keep open. This is an open ClawHub issue, and the repo-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. The code also still has a real residual Convex pagination risk: current main separates package and skill owner digest rebuilds into different scheduled internal mutations, but each scheduled mutation still loops through `.paginate()` pages inside one mutation invocation, so owners with more than one page of packages or skills can still hit Convex's single-paginated-query limit.

## Best Possible Solution

Keep this issue open and land a narrow Convex hardening PR that changes the owner-publisher package and skill digest rebuilds to process exactly one paginated page per scheduled internal mutation, then reschedule with the returned cursor until complete. Preserve the current separate package/skill scheduling mitigation, keep indexed `by_owner_publisher` reads, add regression coverage for multi-page continuation, and consolidate the related publish-pagination reports after deployment verification.
