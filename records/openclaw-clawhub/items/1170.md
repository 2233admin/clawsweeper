## Summary

Keep open. This is an open ClawHub issue, and the repo-specific cleanup policy only allows auto-closing pull requests that current main certainly implements. Current main mitigates the original trigger shape by scheduling package and skill digest rebuilds separately, but each scheduled internal mutation can still call `.paginate()` more than once for an owner with more than one page of packages or skills.

## What This Changes

Fix CLI publish failures caused by Convex multiple-pagination errors during personal publisher creation and owner-publisher search digest syncing.

## Best Possible Solution

Implement cursor-based continuation for owner-publisher package and skill digest rebuilds: each scheduled internal mutation should process one indexed page, update digests for that page, and reschedule itself with `continueCursor` until complete, then use the shipped fix to consolidate the duplicate publish-blocking reports.
