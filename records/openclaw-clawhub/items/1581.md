Keep open. This is an open ClawHub issue, and the cleanup policy only permits auto-closing pull requests that current main certainly implements. Current main does not hardcode a localhost web OAuth callback, but live production GitHub sign-in still returns HTTP 500 from both the Vercel and Convex endpoints, so the user-visible auth failure is not proven fixed.

## What This Changes

The issue requests that production GitHub OAuth return to the production Convex callback instead of redirecting to http://127.0.0.1:3000 after authorization.

## Best Possible Solution

Restore production GitHub OAuth so clawhub.ai starts authorization with a redirect to GitHub and completes on the production Convex callback without any localhost or 127.0.0.1 target, then add or extend production smoke coverage that catches both HTTP 500 sign-in starts and localhost callback URLs.
