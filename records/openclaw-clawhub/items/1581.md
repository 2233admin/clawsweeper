## Summary

Keep open. Current main does not prove the production GitHub OAuth callback issue is fixed: the OAuth provider, issuer, and frontend API routing are environment-driven, the production sign-in endpoints still return HTTP 500, and related production auth reports remain open.

## What This Changes

The issue reports production GitHub OAuth redirecting to `http://127.0.0.1:3000/api/auth/callback/github` instead of the production Convex callback URL after authorization.

## Best Possible Solution

Restore production GitHub OAuth so `clawhub.ai` sign-in redirects through GitHub and completes on the production Convex callback without any localhost target, then add a production smoke check that fails on HTTP 500 or localhost/127.0.0.1 callback URLs and consolidate the related OAuth reports if they share the same root cause.
