## Summary

Keep this issue open. Current main does not prove the production GitHub OAuth callback problem is fixed: the GitHub provider and auth issuer are environment-driven, the frontend /api rewrite points to the Convex production site, and both production sign-in endpoints still return HTTP 500. This also remains an open issue in the active production-auth cluster with #1347 and #1717, and the ClawHub cleanup policy keeps issues open unless maintainers handle them manually.

## Best Possible Solution

Keep this open for maintainer production follow-up. Verify the GitHub OAuth App callback URL, Convex AUTH_GITHUB_ID/AUTH_GITHUB_SECRET/CONVEX_SITE_URL/JWT keys, Vercel VITE_CONVEX_SITE_URL/CONVEX_SITE_URL, and the /api rewrite target for clawhub.ai. After production auth is corrected, add or extend a production smoke check that fails when /api/auth/signin/github returns HTTP 500 or redirects through localhost/127.0.0.1, then consolidate this with #1347 and #1717 if they share the same root cause.
