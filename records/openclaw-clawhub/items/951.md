## Summary

Keep open. Current main still advertises onlycrabs.ai as the SOUL.md registry and contains host-based SoulHub code, but the domain currently resolves through Namecheap forwarding to a parking server. Under the provided ClawHub policy, issues should not be auto-closed, and this one still needs a maintainer/product decision: connect the domain or stop advertising it as live.

## Best Possible Solution

Keep this issue open for maintainer triage. The concrete fix should be either to connect onlycrabs.ai to the production app and enable the intended SoulHub host/env configuration, or to update README/docs to stop presenting onlycrabs.ai as a working registry until the domain is live. If the domain is supported, add a production smoke check for the onlycrabs.ai host so this does not regress silently.
