## Summary

This issue should remain open. The ClawHub-specific review policy only permits auto-closing PRs that are already implemented on main, and the report still matches current main: onlycrabs.ai is still documented and configured as the SOUL.md/SoulHub host, while the live domain check did not show a working ClawHub/SoulHub app.

## Best Possible Solution

Keep this issue open for maintainer triage. The best resolution is either to connect onlycrabs.ai to the production ClawHub/SoulHub app and add smoke coverage for that host, or to update README/docs/source defaults/tests so onlycrabs.ai is not presented as a live SOUL.md registry until the domain is connected.
