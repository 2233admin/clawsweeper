## Summary

Keep open. This is an open issue in openclaw/clawhub, and the repository-specific cleanup policy only permits auto-closing ClawHub PRs that are certainly implemented on main. Substantively, current main still supports the reported behavior: VirusTotal Code Insight can drive a suspicious status independently of AV engine detections, and the public UI still presents that as a generic suspicious warning.

## Best Possible Solution

Keep this issue open for maintainer moderation and product review. The likely fix is either to manually clear HostGuard after review or to change the scan and UI model so `0/64 AV detections plus Code Insight suspicion` is displayed differently from actual malicious AV detections.
