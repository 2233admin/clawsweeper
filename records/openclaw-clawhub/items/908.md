## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review says ClawHub issues should remain open unless they are pull requests already implemented on main. The current publish/upload UI also still surfaces validation through an aggregate Validation card and submit reminder rather than field-local messages beneath inputs or a popup.

## Best Possible Solution

Keep this issue open for a focused publish-skill/upload UX update: derive field-specific validation state, render concise error text beneath Slug, Display name, Version, Tags, license, and file upload controls, wire accessible error relationships with aria-invalid and aria-describedby or alert behavior, and update upload route tests to assert the new placement and accessibility behavior. The aggregate Validation card can remain as a secondary checklist if maintainers still want it.
