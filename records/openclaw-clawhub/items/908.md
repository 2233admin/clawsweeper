## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review pass keeps ClawHub issues open. A source review also shows the requested UX is still not fully implemented: upload/publish validation errors are collected in an aggregate Validation card plus a generic submit reminder, not shown inline under the affected inputs or via a validation popup.

## Best Possible Solution

Keep this issue open for a focused upload-page UX pass: map validation failures to field-level helper/error text beneath Slug, Display name, Version, Tags, file upload, and license controls; add accessible aria-describedby or role=alert behavior; keep the aggregate Validation card only as a secondary checklist; and update upload route tests to assert the new placement.
