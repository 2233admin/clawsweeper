## Summary

Keep open. This is a ClawHub issue, and the review policy says ClawHub issues should remain open. Current main also still shows publish/upload validation as an aggregate Validation card plus a generic submit reminder, not inline beneath each input or as a popup.

## Best Possible Solution

Keep this issue open for a focused publish-skill/upload UX change: derive field-specific validation state, render helper/error text beneath Slug, Display name, Version, Tags, license, and file upload controls, wire aria-invalid and aria-describedby or alert behavior, and update upload route tests to assert the new placement and accessibility behavior. The aggregate Validation card can remain as a secondary checklist if maintainers still want it.
