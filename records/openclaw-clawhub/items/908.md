## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review pass says to keep ClawHub issues open. The current upload/publish skill page does surface validation in an aggregate Validation card, but it does not implement the requested inline-under-input or popup/toast validation treatment.

## Best Possible Solution

Keep this issue open for a focused UI improvement: map validation errors to field-level helper/error text beneath Slug, Display name, Version, Tags, file upload, and license controls; add accessible aria-describedby or role=alert wiring where appropriate; keep the aggregate validation summary only as a secondary checklist; and update upload route tests to assert the new visible placement.
