## Summary

Keep this issue open. It reports an active GitHub OAuth login failure in ClawHub, and this repository's cleanup policy only allows automated closure for ClawHub PRs that are certainly implemented on main. Current source shows the OAuth path exists, but it does not prove the production callback/session failure is fixed; related open reports #647 and #1347 still describe the same symptom cluster, with #629 as closed historical context.

## Best Possible Solution

Keep this issue open and consolidate maintainer triage with #647 and #1347, using #629 as historical context.
