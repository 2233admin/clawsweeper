## Summary

This issue should remain open. It reports an active GitHub OAuth login failure in ClawHub, and the repository policy only permits automated closure for ClawHub PRs that are certainly implemented on main, not issues. Current main has GitHub OAuth callback handling, but the source and tests do not prove the reported production callback/session failure is fixed; related reports #647 and #1347 remain open for the same symptom cluster.

## Best Possible Solution

Keep this issue open and consolidate maintainer triage with #647 and #1347, using #629 as historical context. Maintainers should choose a canonical tracker, collect exact callback URLs, timestamps, browser console/network logs, cookie/session observations for clawhub.ai and the Convex site, affected GitHub account details, and production Convex/Vercel logs, then land a focused OAuth callback/session/configuration fix or manually close duplicates after verification.
