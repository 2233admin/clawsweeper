## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only allows auto-closing pull requests that are certainly implemented on current main. Current main partially addresses the request with CLI publishing, ignore handling, and acceptable-usage docs, but it still lacks a skill lint command, a skill publish dry-run, and pre-upload suspicious-scan feedback.

## Best Possible Solution

Keep this issue open. The best path is to add a shared skill preflight layer used by both CLI and web: `clawhub skill lint <path>`, `clawhub skill publish --dry-run`, schema/path/text/static-scan checks before final upload, web upload warnings before storage/final publish, clearer `.git` and ignore-file guidance for browser uploads, and links from the upload flow to acceptable-usage and remediation guidance.
