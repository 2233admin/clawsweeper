## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main partially addresses the request with skill publishing, CLI ignore handling, persisted static scan results, rescan/admin surfaces, and acceptable-usage documentation, but it still lacks a dedicated `skill lint` command, `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback for both browser and CLI skill upload flows.

## Best Possible Solution

Keep this issue open for maintainer scoping. The best path is a shared skill preflight layer used by both CLI and browser upload: `clawhub skill lint <path>`, `clawhub skill publish --dry-run`, reusable schema/path/text/static-scan checks before final publish/storage where feasible, careful remediation-oriented suspicious findings, explicit `.git`/ignore-file behavior, and links to the acceptable-usage policy.
