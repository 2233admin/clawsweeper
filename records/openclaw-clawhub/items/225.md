## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review says ClawHub issues should not be auto-closed. Substantively, current main has partial progress, but the central request is still incomplete: there is no skill-specific lint command or `skill publish --dry-run`, and suspicious findings are still produced after files have been uploaded/stored.

## Best Possible Solution

Keep this issue open. The best path is a shared skill preflight layer used by both CLI and web: add `clawhub skill lint <path>` and `clawhub skill publish --dry-run`, reuse the publish schema/path/text/static-scan checks before final upload, show static suspicious findings before final web upload/publish, document `.git` handling for browser uploads, and link acceptable-usage guidance from the upload flow with clear remediation language for suspicious findings and repeated uploads.
