## Summary

Keep open. This is a ClawHub issue, and the repository-specific policy says issues should remain open. Substantively, current main only partially addresses the request: CLI skill publishing exists and honors local ignore rules, and policy docs clarify abuse categories, but there is still no skill lint command, no skill publish dry-run, and suspicious/static scan findings are still generated from uploaded/stored files rather than shown as a pre-upload warning.

## Best Possible Solution

Keep this issue open. The best path is a shared skill preflight layer used by CLI and web: add `clawhub skill lint <path>` and `clawhub skill publish --dry-run`, reuse schema/path/text/static-scan checks before final upload, show suspicious findings before final web upload/storage, document `.git` and ignore-file handling clearly for browser uploads, and link acceptable-usage/remediation guidance from the upload flow.
