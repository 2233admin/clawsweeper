Keep this PR open. Current main overlaps by restoring a visible system/light/dark header ToggleGroup, but it does not definitely implement the PR's full intended change: runtime and pre-hydration theme loading still honor valid `clawdhub-theme` values, tests still assert that fallback, and the current header tests do not cover the PR's direct click plus keyboard/value-change regression path. The linked #1228 remains open, and #1234 was a related alternative that closed unmerged.

## What This Changes

The PR re-enables system/light/dark theme selection from the header ToggleGroup, removes the legacy `clawdhub-theme` fallback, and adds regression tests for click, keyboard, and first-visit theme behavior.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, or explicit supersession.
