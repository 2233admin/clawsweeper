## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that are certainly implemented on main. Substantively, current main has partial scanner transparency, but it still lacks the requested local `clawhub scan <path> --format json` workflow and a complete per-finding contract for remediation and blocking/informational status.

## Best Possible Solution

Keep this issue open. A complete fix should define one stable scanner findings contract and wire it through API, CLI, UI, docs, and tests: rule/category, severity, file/line range, redacted snippet, remediation guidance, and whether each finding blocks publish/install or is informational. The existing `staticScan.findings`, `/moderation`, and `/scan` surfaces are good foundations, but `clawhub scan <path> --format json` and the full remediation/blocking contract still need implementation.
