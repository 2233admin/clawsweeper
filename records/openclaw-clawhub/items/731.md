## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on main. Current main has partial scanner transparency through persisted static findings, owner/staff moderation evidence, docs, and UI display, but it still lacks the requested local `clawhub scan <path> --format json` workflow and a full stable findings contract with remediation and per-finding blocking/informational status.

## Best Possible Solution

Keep this issue open. A complete fix should define one stable scanner findings contract and wire it through API, CLI, UI, docs, and tests: rule/category, severity, file/line range, redacted snippet, remediation guidance, and whether each finding blocks publish/install or is informational. The existing `staticScan.findings`, `/moderation`, `/scan`, and UI finding display are good foundations, but `clawhub scan <path> --format json` and the full remediation/blocking contract still need implementation.
