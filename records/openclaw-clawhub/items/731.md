## Summary

Keep open. Current main exposes partial structured scanner evidence, but it still does not satisfy the actionable rule/file/line-range/remediation/blocking contract requested here, and the ClawHub cleanup policy keeps issues open unless they are certainly implemented PRs.

## What This Changes

Request to expose actionable static scanner findings, including rule/category, severity, file and line range, snippets, remediation guidance, blocking semantics, and optionally a local CLI scan command.

## Best Possible Solution

Define a stable scanner-finding contract with rule/category, severity, file plus line range, redacted evidence policy, remediation guidance, and blocking semantics, then wire it consistently through persistence, API responses, skill UI, CLI publish output or a local scan command, docs, and tests.
