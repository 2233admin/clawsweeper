## Summary

Keep open. This is an issue, and the repository policy for openclaw/clawhub says not to auto-close issues. Substantively, the request is an unresolved namespace/trademark administration case: the reporter asks maintainers to verify Mapbox ownership and transfer or remove an existing ClawHub package/name. Current main has moderation and admin ownership tools, but I found no evidence that this specific Mapbox transfer/removal has happened or that the issue is obsolete.

## Best Possible Solution

A maintainer should keep this issue open, verify the reporter’s authority for the Mapbox GitHub organization or mapbox.com domain, inspect the current production owner of the `mapbox` package/name, then either transfer the package/publisher ownership to the verified Mapbox publisher, remove the squatted package so Mapbox can publish, or add a narrowly scoped admin package-transfer/reclaim path with audit logging. Longer term, the publisher/org namespace migration in `docs/orgs.md` should cover trademark/official-publisher reclaim cases explicitly.
