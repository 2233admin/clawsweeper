## Summary

Keep open. This is an issue, and the ClawHub cleanup policy only permits auto-closing PRs when current main definitely implements the intended change. The current source still leaves the requested Mapbox package-name transfer/removal as a maintainer/admin action: packages are globally keyed by normalizedName, cross-publisher publishes are rejected, and the existing reclaim API is for skill root slugs, not package ownership transfer.

## Best Possible Solution

Keep this issue open for maintainer handling. A maintainer should verify the reporter’s authority for the Mapbox GitHub organization or a mapbox.com identity, inspect the current production owner of the mapbox package, then either remove/soft-delete the squatted package so Mapbox can publish or add a narrowly scoped audited package reclaim/transfer path. Longer term, the package/publisher namespace migration should explicitly document official-publisher and trademark reclaim cases.
