## Summary

Keep open. Current main still lacks the requested SKILL.md lifecycle hook documentation, normalized metadata, hook-specific security analysis, public API serialization, and skill page/install transparency; the existing `hooks` references are for plugin/package capabilities, a different surface. This is also a ClawHub issue, and the cleanup policy only auto-closes pull requests that are certainly implemented on main.

## What This Changes

The issue requests documenting SKILL.md lifecycle hooks and adding registry parsing, security scanning, API/UI visibility, and install transparency for hook declarations.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it consistently across the skill format docs, shared schema, parser/backfill path, public API, skill detail/install UI, static moderation, LLM moderation context, and warnings for high-privilege events or hook types.
