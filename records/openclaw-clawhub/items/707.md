## Summary

Keep open. Under this repository policy, open ClawHub issues are not auto-closed, and current main still lacks first-class SKILL.md hook documentation, normalized metadata, hook-specific scanning, API serialization, or skill page surfacing. The only current `hooks` capability surface is for plugin/package metadata, not skill frontmatter lifecycle hooks.

## What This Changes

Document Claude Code SKILL.md `hooks` and add ClawHub parsing, security analysis, and pre-install UI/API transparency for hook declarations.

## Best Possible Solution

Keep this issue open for maintainer security/product review. The best path is to define an accepted skill hook metadata contract first, then update `docs/skill-format.md`, shared schema/parser code, publish/backfill/API serialization, skill detail UI, static moderation checks, LLM scan prompt context, and install/inspect warnings or restrictions for high-privilege hook events and hook types.
