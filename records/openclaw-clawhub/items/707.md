## Summary

Keep this issue open. Current main still does not document, type, parse, validate, scan, expose, or render SKILL.md lifecycle hooks, and the repository policy for openclaw/clawhub only allows this workflow to auto-close PRs that are certainly implemented on main, not issues.

## Best Possible Solution

Keep this issue open. A good implementation should add a typed SKILL.md hooks declaration to the skill metadata contract and public docs, parse it from frontmatter, validate supported lifecycle events and hook types, verify command targets against bundle files, flag privileged or blocking events in moderation, expose hook declarations through the API, surface them in the skill detail UI, and add focused parser, scanner, API, and UI tests. Treat #129 as related security context, not the canonical hook-specific tracker.
