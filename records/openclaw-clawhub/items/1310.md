## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only allows auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has security scan surfaces plus a publish-time content quality gate, but it does not implement the public, multidimensional skill quality scoring model proposed here.

## What This Changes

The issue proposes adding public, multidimensional quality signals for ClawHub skills, distinct from security and moderation scanning.

## Best Possible Solution

Keep this open for maintainer product triage. A good path would define a public quality-signal model separate from security/moderation verdicts, choose measurable dimensions such as documentation completeness, maintenance freshness, dependency hygiene, prompt/code structure, and claim-vs-reality checks, then add schema/API/search fields with a backfill plan and render the chosen signals on browse/detail pages. Treat #288 as related security-signal work, not a replacement for this broader quality proposal.
