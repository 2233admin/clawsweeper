## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only allows auto-closing pull requests that are certainly implemented on current main; independently, current main has an internal publish-time quality gate and security scan surfaces, but not the requested public, multidimensional skill quality scoring model.

## What This Changes

This issue proposes structured quality signals for ClawHub skills beyond security/moderation scans, based on an external static analysis of popular skills and trust-signal gaps.

## Best Possible Solution

Define a public quality-signal model distinct from security/moderation verdicts, then add agreed schema/API/search support, migration or backfill, and UI presentation on browse/detail surfaces while keeping #288 as related security-signal context.
