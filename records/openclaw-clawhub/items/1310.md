## Summary

Keep open. ClawHub cleanup policy only allows auto-closing ClawHub pull requests that are certainly implemented on current main, and this is an issue. Current main has security scan surfaces and a publish-time content quality gate, but it does not implement the requested public, multidimensional skill quality scoring model; related #288 covers behavioral security signals rather than this broader product-quality proposal.

## What This Changes

The issue proposes adding public quality signals for ClawHub skills, separate from security verdicts, covering dimensions such as documentation completeness, claim-vs-reality, maintenance, dependency hygiene, alternatives, and prompt/code structure.

## Best Possible Solution

Keep this open for maintainer product triage. The desired end state is a defined public quality-signal model, distinct from security/moderation verdicts, with agreed dimensions, schema/API/search support, backfill strategy, and UI presentation on browse/detail surfaces; #288 should remain related security-signal context rather than the canonical tracker for this broader quality work.
