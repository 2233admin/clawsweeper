## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this lane only allows auto-closing ClawHub PRs that current main definitely implements. Substantively, current main still has a plausible CLI failure path where ignore rules can hide a root `SKILL.md` before `cmdPublish` validates the file list; open draft PR #1856 is already the focused implementation candidate for this issue, with #48 as a related report.

## What This Changes

The issue asks to fix `clawhub publish` so new skills with a valid root `SKILL.md` can publish instead of failing with `SKILL.md required`.

## Best Possible Solution

Keep this issue open while maintainers review and land PR #1856 or an equivalent CLI fix. The fix should preserve a root `SKILL.md` or `SKILLS.md` manifest for `clawhub publish` when broad ignore rules hide markdown files, keep other ignored markdown excluded, add regression coverage, and ship a CLI release. Until then, `https://clawhub.ai/publish-skill` remains the useful manual upload workaround.
