Keep open. This is a ClawHub issue, and this cleanup lane only auto-closes ClawHub PRs that current main definitely implements; current main also still has a credible CLI path where ignore rules can hide the root manifest before publish validation. Open draft PR #1856 is the focused implementation candidate and #48 is a related open report.

## What This Changes

The issue reports that `clawhub publish` fails with `SKILL.md required` for new skills even when a valid root `SKILL.md` exists, blocking new CLI publishes.

## Best Possible Solution

Ship a CLI fix that always includes an existing root `SKILL.md` or `SKILLS.md` manifest during `clawhub publish`, even when broad ignore rules exclude other Markdown files, with regression coverage and backend validation unchanged.
