## Summary

Keep this issue open. Current main still has a plausible CLI failure path for `clawhub publish`: the publish command requires a root `SKILL.md`/`SKILLS.md` in the files returned by `listTextFiles`, while `listTextFiles` applies ignore rules before reading markdown files. The supplied GitHub context also shows draft PR #1856 is already the focused closing PR for this issue, so the right next step is maintainer review and merge of that PR or an equivalent fix, then a CLI release.

## Best Possible Solution

Keep this issue open while maintainers review and land PR #1856 or an equivalent CLI fix. The fix should preserve a root `SKILL.md` or `SKILLS.md` manifest for `clawhub publish` when broad ignore rules hide markdown files, keep other ignored markdown excluded, validate the related #48 report shape, and then ship a CLI release. Until then, `https://clawhub.ai/publish-skill` is the practical manual upload workaround.
