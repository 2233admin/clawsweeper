## Summary

Keep open. Current main does not implement this PR's phantom dependency registry scanner, and the PR is an active security-sensitive moderation/supply-chain change with substantive review findings and an author follow-up claiming the branch now addresses them. ## What This Changes

The PR adds a Convex publish-time scanner for dependency files bundled in skills, stores and caches public-registry existence checks, injects phantom-package findings into moderation, and tweaks security scan UI rendering for line-zero findings.

## Best Possible Solution

Keep this PR open for maintainer/security review. The best path is to either land a corrected scanner or replace it with a successor that covers dependency-file parsing, scoped npm packages and aliases, direct URL/workspace specs, optional dependencies, pyproject/Poetry formats, vendored dependency files, transient registry failures, cache TTL/invalidation, indexed retry/backfill behavior, UI/API reporting, and coordination with openclaw/trust#29.
