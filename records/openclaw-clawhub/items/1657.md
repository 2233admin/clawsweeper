## Summary

Keep open. Current main does not implement dependency-registry phantom-package scanning for skill bundle dependency files, and this open PR is the active security-sensitive implementation candidate. The branch touches publish-time Convex moderation, schema, outbound registry lookups, cache/retry behavior, and static-scan reporting, so it needs maintainer/security review rather than cleanup closure.

## What This Changes

The PR adds a Convex async scanner for bundled requirements.txt, package.json, Cargo.toml, and pyproject.toml files, stores dependency-registry analysis and cache records, schedules scans after skill publish, adds a moderation reason/static-scan finding for missing packages, and tweaks security-scan UI line rendering.

## Best Possible Solution

Land this PR or a successor only after maintainer/security review confirms robust dependency parsing, registry existence checks, transient-failure handling, indexed retry state, clear moderation findings, and coordination with the broader trust tracking item.
