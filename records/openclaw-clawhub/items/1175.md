## Summary

Keep this issue open. Current main still leaves the reported publish path able to hit Convex's single-pagination limit: publishing reaches `ensurePersonalPublisherForUser`, publisher writes schedule owner-publisher digest rebuilds, and each owner-publisher digest rebuild can still call `.paginate()` repeatedly inside one internal mutation when there is more than one page of skills or packages. This is a narrow, high-confidence fix-PR candidate.

## Best Possible Solution

Land one focused Convex backend PR that makes the owner-publisher package and skill digest rebuild internal mutations accept a cursor, process at most one paginated page per invocation, and self-schedule a continuation with `continueCursor` when `isDone` is false. Add regression tests for multi-page owner rebuilds and verify the CLI publish/update path no longer fails before consolidating the duplicate issue cluster.
