## Summary

Keep this issue open. The repository policy for openclaw/clawhub says this cleanup pass should only auto-close pull requests that are certainly implemented on current main, and this item is an issue. Current main mitigates the originally reported double-pagination trigger by scheduling package and skill owner-publisher digest rebuilds as separate internal mutations, but the named skill digest helper still loops over `.paginate()` inside a single mutation invocation, and production deployment is manual-only.

## Best Possible Solution

Keep this issue open for maintainer follow-up. A complete fix should convert the owner-publisher digest rebuild helpers to a cursor-based self-scheduling pattern, or another bounded single-pagination design, then verify the production Convex backend is deployed and that an updated skill publish succeeds. Maintainers should also consolidate the duplicate publish-failure reports and close them together once production behavior is confirmed.
