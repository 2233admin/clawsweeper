## Summary

Keep this PR open. Current main still does not implement the PR’s intended soul-search parity work: `searchSouls` returns empty results when embedding generation fails, rehydrates the full soul vector result set on every loop, has no soul lexical fallback/merge helper, and scores soul results with raw vector score rather than the boosted skill-search scoring path.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, update the soul lexical fallback to use the indexed active-souls path, and land the parity behavior with focused tests for embedding-failure fallback, lexical fallback, soft-delete exclusion, merge/dedup, lexical/popularity scoring, and incremental hydration.
