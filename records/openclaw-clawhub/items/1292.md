## Summary

Keep this PR open. Current main still does not implement the PR’s intended soul-search parity work: `searchSouls` has no lexical fallback, no soul merge/dedup helper, no incremental hydration set, and no lexical/popularity scoring. The PR remains a relevant implementation candidate, with a needed rebase/performance adjustment for the active-souls fallback path.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, update the soul lexical fallback to use the indexed active-souls query path, and merge with focused tests for embedding-failure fallback, lexical fallback, soft-delete exclusion, merge/dedup, lexical/popularity scoring, and incremental hydration.
