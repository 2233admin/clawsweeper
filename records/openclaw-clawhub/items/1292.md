## Summary

Keep open: current main does not implement the PR’s intended soul-search parity. The existing `searchSouls` path still returns no results on embedding-generation failure, rehydrates the full vector result set during candidate expansion, scores only by raw vector score, and has no soul lexical fallback tests; the PR diff is useful but needs to be rebased and updated for the current indexed active-soul path.

The PR branch changes `convex/search.ts` and `convex/search.test.ts` to add soul lexical fallback, merge/deduplication, shared boosted scoring, incremental soul embedding hydration, and related tests.
