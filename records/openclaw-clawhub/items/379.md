## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy only permits auto-closing pull requests that are definitely implemented on main. Current main has backend mitigation for uncorroborated VirusTotal Code Insight suspicion, but the user-visible VirusTotal badge/API status still derives from stored `vtAnalysis.status`, so the reported badge behavior is not clearly fixed.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best product fix is to derive the VirusTotal badge and warning banner primarily from AV engine malicious/suspicious counts, present Code Insight as separate context, and reconcile or rescan affected stored analyses so existing clean-engine skills no longer show a misleading suspicious badge.
