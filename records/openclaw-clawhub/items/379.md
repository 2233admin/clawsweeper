## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy only permits auto-closing pull requests that current main definitely implements. Current main has a backend mitigation for uncorroborated VirusTotal Code Insight suspicion, but the user-visible VirusTotal badge/API status still derives from stored `vtAnalysis.status`/`verdict`, so the reported “VirusTotal: Suspicious” badge for clean engine detections is not clearly fixed.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best fix is to make the VirusTotal badge/API derive the scanner label primarily from AV engine malicious/suspicious counts when those counts are available, present Code Insight as separate explanatory context, add UI/API regression coverage for 0-detection Code Insight-suspicious cases, and run a production reconciliation/backfill for affected stored analyses.
