## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. Substantively, current main has backend moderation suppression for uncorroborated VirusTotal Code Insight suspicious results when AV engine counts and local scans are clean, but the user-facing VT badge, scanner detail pages, and public scan API still derive visible status from stored VT Code Insight status/verdict and can still show Suspicious for the reported shape.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best fix is to centralize a normalized VirusTotal display/security status that treats Code Insight-only suspicious results with 0 malicious and 0 suspicious AV engine counts as clean/undetected when ClawScan and static analysis are clean, while still exposing Code Insight text separately. Apply that shared logic to the version badge, detail summary, scanner page, public scan/version API, and regression tests; consider a cache reconciliation or derived-at-read approach for existing affected analyses.
