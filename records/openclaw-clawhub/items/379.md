## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a narrow implementation gap: backend moderation can suppress uncorroborated VirusTotal Code Insight suspicious results, but user-facing skill UI and v1 API security snapshots still derive VirusTotal display status from raw stored `vtAnalysis.status` or `verdict`.

## What This Changes

The issue asks ClawHub to show a clean/undetected VirusTotal badge and avoid the suspicious warning banner when VirusTotal has zero malicious or suspicious engine detections and OpenClaw scan signals are clean.

## Best Possible Solution

Keep this issue open and implement a central VirusTotal display/status normalization path. Code Insight-only suspicious results with zero malicious and zero suspicious AV engine counts should display as clean/undetected when ClawScan and static analysis are clean, while preserving Code Insight text as context and preserving warnings for any real AV-engine, static-scan, or ClawScan suspicious/malicious counts.
