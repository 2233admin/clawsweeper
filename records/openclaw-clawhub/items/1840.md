## Summary

Keep this PR open. Current main still does not implement the intended org-owned skill publish path for REST/CLI: the skill publish schema and parser do not carry `ownerHandle`, `/api/v1/skills` publishes directly as the API-token actor, and the CLI publish commands have no `--owner` option. The PR remains a live implementation candidate for the open #1814 gap.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so `/api/v1/skills` accepts optional `ownerHandle`, normalizes it for JSON and multipart payloads, resolves the target publisher server-side with publisher-role authorization, passes `ownerPublisherId` into `publishVersionForUser`, and exposes the same capability through `clawhub publish --owner` and `clawhub skill publish --owner`. Keep #1814 open until an implementation merges.
