## Summary

Current main does not implement the PR's intended change. The REST skill publish endpoint still drops any owner target, the skill publish schema and multipart parser do not carry ownerHandle, and both CLI skill publish commands lack --owner. The PR remains a live implementation candidate for open #1814, with an ownership-sensitive authorization path that should be reviewed before merge.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so /api/v1/skills accepts optional ownerHandle, preserves it through JSON and multipart parsing, resolves the target publisher server-side with minimumRole: "publisher", passes ownerPublisherId into publishVersionForUser, and exposes the same capability through both clawhub publish --owner and clawhub skill publish --owner. Keep #1814 open until the implementation merges.
