## Summary

Keep open. Current main still does not implement the REST/CLI org-owned skill publish path this PR adds for open #1814: `/api/v1/skills` drops any owner target and calls `publishVersionForUser` directly, while the CLI publish commands have no `--owner` option and serialize no `ownerHandle`. The PR remains a live implementation candidate. Security review of the provided diff found no unrelated CI, dependency, lockfile, secret, package-script, or release-publishing changes; the ownership-sensitive part should still be reviewed to preserve server-side publisher-role authorization.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so `/api/v1/skills` accepts optional `ownerHandle`, preserves it through JSON and multipart payload parsing, resolves the target publisher server-side with `minimumRole: "publisher"`, passes `ownerPublisherId` into `publishVersionForUser`, and exposes the same capability through both `clawhub publish --owner` and `clawhub skill publish --owner`. Keep #1814 open until the implementation merges.
