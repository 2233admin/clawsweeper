## Summary

Keep this PR open. Current main still does not implement the intended REST/CLI path for org-owned skill publishes: the skill API parser does not preserve an `ownerHandle`, the `/api/v1/skills` handler publishes directly as the API-token actor without resolving a target publisher, and the skill CLI publish commands have no `--owner` option. The PR remains a live implementation candidate for the open #1814 gap.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so `/api/v1/skills` accepts optional `ownerHandle`, normalizes it for JSON and multipart payloads, resolves the target publisher server-side with publisher-role authorization, passes `ownerPublisherId` into `publishVersionForUser`, and exposes the same capability through `clawhub publish --owner` and `clawhub skill publish --owner`. Keep #1814 open until the implementation merges.
