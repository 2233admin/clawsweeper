## Summary

Keep this PR open. Current main still does not implement the PR’s intended REST/CLI support for org-owned skill publishes: the skill publish API does not preserve or resolve ownerHandle, the skill publish schema rejects that field, and the skill CLI has no --owner option. The PR remains a live implementation candidate for the open org-owned skill publishing gap described in #1814.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so POST /api/v1/skills accepts optional ownerHandle, resolves the publisher server-side with publisher-role authorization, passes ownerPublisherId into publishVersionForUser, and exposes the same capability through both clawhub publish --owner and clawhub skill publish --owner. Keep the paired issue open until an implementation merges.
