## Summary

Keep open. Current main still does not implement this PR’s intended change: skill REST publish and CLI publish do not carry an owner handle, while the authenticated web action already resolves an owner publisher. The PR appears to be a narrow implementation for the linked #1814 gap and should be reviewed rather than closed as obsolete.

## Best Possible Solution

Review and land this PR, or an equivalent fix, so skill publish payloads accept and preserve `ownerHandle` through schema parsing, JSON and multipart REST handlers, and both skill publish CLI commands; resolve the owner server-side with publisher-role authorization before calling `publishVersionForUser`; keep the regression tests for schema, CLI payload generation, JSON API publish, and multipart API publish. Do not close this PR until that implementation is merged to main.
