## Summary

Keep open. Current main still does not implement the PR’s intended REST/CLI support for org-owned skill publishes: the web publish action can resolve an owner handle, but the V1 skill API schema/parsing/handler and the skill CLI still drop the owner target. The PR remains the focused implementation candidate for #1814, and the provided diff does not show unrelated workflow, dependency, lockfile, or release-script changes.

## What This Changes

The PR adds optional ownerHandle support to skill publish schemas and payload parsing, resolves the requested publisher server-side for JSON and multipart V1 API publishes, adds --owner to skill publish CLI commands, updates generated schema artifacts, and adds regression tests.

## Best Possible Solution

Land this PR or an equivalent focused patch so /api/v1/skills and the skill publish CLI accept an owner handle, normalize it consistently, resolve the target publisher with minimumRole: "publisher", pass the resolved ownerPublisherId into publishVersionForUser, keep schema artifacts synced, and cover JSON, multipart, schema, and CLI payload behavior. #1814 should close only after that fix merges.
