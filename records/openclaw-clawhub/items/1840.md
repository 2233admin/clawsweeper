## Summary

Keep this PR open. Current main still lacks the PR's intended REST/CLI path for publishing org-owned skills: `/api/v1/skills` publishes as the API-token actor without resolving an `ownerHandle`, the skill publish schema/parser do not preserve that field, and the skill CLI commands do not expose `--owner`.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so `/api/v1/skills` accepts an optional `ownerHandle`, resolves the target publisher server-side with `minimumRole: "publisher"`, threads `ownerPublisherId` into `publishVersionForUser`, and exposes the same capability through both `clawhub publish --owner` and `clawhub skill publish --owner`. Keep the linked #1814 issue open until an implementation merges.
