## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only allows auto-closing PRs that are certainly implemented on current main. Current main also does not implement a shareable cron/automation-template marketplace: the product scope, schema, API routes, CLI, and UI surfaces are centered on skills, souls, and package/plugin catalogs, with only internal maintenance crons and a skill category keyword for automation.

## Best Possible Solution

Keep this issue open for maintainer product triage. The next useful step is to decide whether shareable cron jobs should be a new ClawHub content type, metadata/examples inside skills, or package/bundle plugins tied to an OpenClaw cron install API; then specify the schema, preview format, publish/install API, CLI flow, and browse UI before implementation.
