## Summary

Keep open. This is an open ClawHub issue, and repository policy only permits auto-closing ClawHub pull requests that current main definitely implements. Current main does not provide a first-class shareable cron-template catalog: ClawHub is still centered on skills, souls, and plugin/package catalog work, with only an Automation skill category keyworded for cron/schedule discovery.

## Best Possible Solution

Keep this issue open for maintainer product judgment. The best next step is to decide whether cron templates should be a new ClawHub package family, a capability/tag on skills or bundle plugins, or an OpenClaw runtime/plugin feature first. If accepted, define the manifest, preview/install semantics, trust labels, and security prompts before implementing ClawHub schema/API/UI support.
