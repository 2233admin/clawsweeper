## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. The requested marketplace/content type for shareable cron or automation templates is also not implemented in current main: the product scope, schema, API route constants, package families, and CLI/docs surfaces are built around skills, SOULs, code plugins, and bundle plugins, while cron references are limited to internal maintenance jobs or keyword/category metadata.

## Best Possible Solution

Keep this issue open for maintainer product triage. The concrete next step is to decide whether shareable cron jobs should be a new ClawHub content type, package/bundle plugin metadata tied to an OpenClaw cron install API, or examples embedded in skills; after that, specify the schema, preview format, publish and install APIs, CLI behavior, and browse UI.
