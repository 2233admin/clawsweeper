## Summary

Keep open. Current main still uses the apex host as the CLI site/registry and discovery default, while live routing now sends www back to the apex host; the issue needs maintainer verification with a valid production token rather than auto-close or a blind www-default PR.

## Best Possible Solution

Keep this open for maintainer handling. A maintainer should verify clawhub login --token <fresh production token> and clawhub whoami against the default apex registry. If it still fails, fix production routing so authenticated API requests never cross-origin redirect, or intentionally switch the canonical host across CLI defaults, discovery metadata, docs, tests, deployment guidance, and release notes together. Coordinate any remaining Unauthorized-login work with #99; do not resurrect the closed www-default PR without rechecking current production routing.
