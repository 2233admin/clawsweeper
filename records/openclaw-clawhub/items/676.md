## Summary

Keep this issue open. The repository-specific ClawHub cleanup policy for this run only permits auto-closing pull requests that are certainly implemented on current main, and this item is an open issue. The underlying import/publish failure path also still exists in current main: GitHub import calls the normal publish path, publish still enforces the GitHub account-age gate by default, and 403/429 responses from GitHub are still surfaced as the reported rate-limit error. Related open issues #1222 and #654 indicate maintainers may want to consolidate the symptom rather than close this report automatically.

## Best Possible Solution

Keep this issue open for maintainer triage, likely alongside #1222. Verify production Convex `GITHUB_TOKEN` and failure logs, then improve `requireGitHubAccountAge` to distinguish true rate limits from other 403 responses, surface retry/reset detail when available, and reduce repeated GitHub lookups by syncing or caching `githubCreatedAt` earlier in the auth/profile path.
