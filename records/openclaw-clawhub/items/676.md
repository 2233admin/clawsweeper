## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass only allows auto-closing pull requests that are certainly implemented on current main. Current main still contains the reported failure path: GitHub import wraps publish failures in the exact reported message shape, publish runs the GitHub account-age gate by default, and the gate still maps GitHub 403/429 responses to the rate-limit error when `githubCreatedAt` is missing. Related open reports #1222 and possibly #654 should be consolidated during maintainer triage.

## What This Changes

Fix persistent GitHub API rate-limit failures during GitHub import publish so valid skills can publish instead of receiving generic retry and format guidance.

## Best Possible Solution

Keep this issue open for maintainer triage and consolidate it with #1222, and with #654 if production logs confirm the comment report is the same account-age gate. The best path is to inspect production Convex failures and `GITHUB_TOKEN` configuration, then either fix deployment configuration or make `requireGitHubAccountAge` more robust by distinguishing true GitHub rate limits from other 403 responses, surfacing retry/reset details where available, and caching `githubCreatedAt` earlier without weakening the account-age gate.
