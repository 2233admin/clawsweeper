## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. The source review also confirms the reported import failure path still exists: GitHub import wraps publish failures with the reported message, publish still runs the GitHub account-age gate, and that gate still turns GitHub 403/429 responses into the rate-limit error.

## Best Possible Solution

Keep this issue open for maintainer triage, likely together with open #1222. Verify production Convex GITHUB_TOKEN configuration and Convex failure logs, then improve requireGitHubAccountAge to distinguish true rate limits from other 403 responses, surface retry/reset detail when available, and reduce repeated GitHub lookups by caching or syncing githubCreatedAt earlier.
