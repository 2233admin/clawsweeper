## Summary

Keep open. The repository policy for openclaw/clawhub keeps issues open, and current main still matches the reported bug: skill publish uploads only files returned by the text-file collector, while extensionless files and `.tsv` files are excluded without an author-facing warning.

## Best Possible Solution

Keep this issue open and fix the skill publish collection path. Add `tsv` to the shared text extension allowlist, handle extensionless files with a conservative text/binary sniff, emit clear publish warnings for skipped files, and add focused tests for extensionless scripts, `.tsv` files, unsupported binary files, and skipped-file warnings. Also review sibling collectors such as zip hashing, web upload filtering, and GitHub import for consistency with the chosen policy.
