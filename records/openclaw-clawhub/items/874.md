## Summary

Keep open. This is an open ClawHub issue, and the repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. The reported bug also still appears present on current main: CLI publish uploads only files returned by `listTextFiles`, that collector still filters by extension allowlist, `.tsv` is absent from the CLI text-file list, extensionless files have no text fallback, and skipped files are not surfaced to the author.

## Best Possible Solution

Keep this issue open and fix the ClawHub skill publish collection path: add `.tsv` to the relevant text-file allowlists, include extensionless text files using a conservative binary/null-byte sniff or equivalent text detection, emit clear warnings for skipped files during publish, and add focused regression tests for extensionless scripts, `.tsv` files, binary skips, and warning behavior. Then audit related hash/import/upload paths for consistent allowlist behavior.
