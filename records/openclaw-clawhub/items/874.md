## Summary

Keep open. This is an open ClawHub issue, and the repository policy says ClawHub issues should not be auto-closed. The provided current-main review context also shows the reported bug still exists: publish only uploads files returned by `listTextFiles`, which still filters extensionless paths and `.tsv` files without an author-facing warning.

## Best Possible Solution

Keep this issue open and fix the ClawHub CLI publish collection path: add `.tsv` to the shared text-file allowlist, include extensionless text files using a conservative binary sniff, emit clear warnings for skipped files, and add focused regression tests for extensionless scripts, `.tsv`, binary skips, and warning behavior.

## Evidence

- **Repository policy keeps ClawHub issues open:** The review instructions for openclaw/clawhub say only pull requests that are certainly implemented on main should be proposed for auto-close; issues should remain open for maintainer follow-up.
  - sha: [4e13e729fb07](https://github.com/openclaw/clawhub/commit/4e13e729fb07bc8cf35e58e554ef557fca0b3713)
- **Current-main behavior remains unfixed:** The existing ClawSweeper review against current main found that `cmdPublish` calls `listTextFiles(folder)` and appends only those collected files, while `listTextFiles` derives extensions with `relPath.split('.').at(-1)` and skips paths not in `TEXT_FILE_EXTENSION_SET`; `.tsv` is not in that allowlist.
  - file: [packages/clawhub/src/cli/commands/publish.ts:55](https://github.com/openclaw/clawhub/blob/4e13e729fb07bc8cf35e58e554ef557fca0b3713/packages/clawhub/src/cli/commands/publish.ts#L55)
  - sha: [4e13e729fb07](https://github.com/openclaw/clawhub/commit/4e13e729fb07bc8cf35e58e554ef557fca0b3713)
- **Impact is independently corroborated:** The issue reports `kraken-cli` losing an extensionless binary and `.tsv` registry files during publish, and a commenter independently confirmed the same failure pattern for the `gurkerl` skill where `bin/gurkerl` was omitted.

## Risks / Open Questions

- Authors can still publish successful-looking skill versions that omit required extensionless scripts or `.tsv` configuration files, leaving installed skills broken.
- Adding only `.tsv` would not cover extensionless executable wrappers; the collector needs a conservative text/binary inclusion policy plus skipped-file warnings.
- Related upload/import/hash paths may share extension allowlist assumptions and should be checked for consistency.
