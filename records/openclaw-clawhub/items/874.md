## Summary

Keep this issue open. Current main still matches the report: skill publish only uploads extension-allowlisted files, `.tsv` is absent from the allowlists, extensionless files are skipped before content inspection, and archive/web import paths still use extension-only filtering.

## What This Changes

The issue requests that `clawhub publish` preserve extensionless text executables and `.tsv` files instead of silently omitting them from skill uploads.

## Best Possible Solution

Implement a focused publish-file classification fix: add `.tsv` to the shared and bundled CLI text allowlists, add a conservative text-content fallback for extensionless files in CLI and archive/web expansion paths, emit clear CLI publish warnings for unsupported skipped files, keep binary files rejected, and add regression tests for extensionless bin scripts and `.tsv` files.
