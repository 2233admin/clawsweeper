## Summary

This issue should stay open. The repository policy for openclaw/clawhub keeps issues open, and current main still has a real publish-path gap matching the report: extensionless files and .tsv files are filtered out by the CLI file collector before upload, with no author-facing warning.

## Best Possible Solution

Keep this issue open and fix the skill publish file collection path. A conservative implementation would add .tsv to the shared text extension allowlist, include extensionless files only when they pass a text/binary sniff such as a null-byte check, emit clear warnings for skipped files, and add focused tests for extensionless scripts, .tsv files, and unsupported binary files.
