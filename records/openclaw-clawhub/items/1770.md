## Summary

Keeping open. This is a ClawHub issue, and the provided repository policy says ClawHub issues should not be auto-closed. Current main also still treats an existing soft-deleted skill row as occupying the global slug for availability and rename, so the reporter's request remains production maintainer/admin support rather than already-shipped self-service behavior.

## Best Possible Solution

Keep this issue open for maintainer support. A maintainer/admin should verify account and org control, then hard-delete or otherwise purge/reassign the soft-deleted creator-scraper-cv row so creativault/cv-creator-scraper can be renamed. If this pattern recurs, open a separate product follow-up for self-service slug transfer/reclaim when the same authenticated user controls both publishers.
