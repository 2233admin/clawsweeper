## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy only permits auto-closing ClawHub pull requests that current main definitely implements. Current main still matches the report: rename checks the requested slug against any `skills.by_slug` row without exempting `softDeletedAt`, and the available hard-delete/reclaim paths are admin/support operations rather than something the reporter can complete themselves.

## Best Possible Solution

Keep this issue open for maintainer support. A maintainer/admin should verify account and org control, inspect the production row for `creator-scraper-cv`, then hard-delete, transfer, reserve/release, or otherwise purge the stale slug so `creativault/cv-creator-scraper` can be renamed. If this comes up repeatedly, track a separate product follow-up for safe self-service slug reclaim or transfer when the authenticated actor controls both publishers.
