## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy says ClawHub issues should not be auto-closed by this cleanup pass. Source review also shows the reported confusion is still plausible: the visible banner is driven by skill-level moderation state, while benign/clean scanner labels come from a separate scan-results surface.

## Best Possible Solution

Keep this issue open for maintainer or moderator follow-up. Ask the reporter for the affected skill slug and version, inspect the production moderation record or `/api/v1/skills/{slug}/moderation` as the owner/staff path allows, then either clear the stale flag with the existing audited manual override/rescan path or improve the banner to show/link the specific moderation summary, reason codes, and remediation guidance.
