## Summary

Keep this issue open. The ClawHub cleanup policy for this review keeps issues open, and current main still leaves the reported pre-OAuth failure mode plausible: CLI login opens `/cli/auth`, the route disables `Sign in with GitHub` while Convex auth is loading, and that loading state comes from the Convex client configured by `VITE_CONVEX_URL`. Existing route coverage exercises the post-auth fallback token path, not the stuck pre-OAuth loading case.

## Best Possible Solution

Keep this issue open for maintainer runtime verification. Check the deployed `/cli/auth` flow and production Convex/Vercel configuration; if the WebSocket 307 still occurs, fix the runtime URL/routing and decouple the unauthenticated GitHub sign-in action from indefinite Convex auth loading, then add a regression test for the stuck-loading pre-OAuth state.
