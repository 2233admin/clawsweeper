Keep this issue open. The ClawHub cleanup policy for this review only allows auto-closing pull requests that are certainly implemented on current main, and this item is an issue. A source pass also still shows the reported pre-OAuth failure mode is plausible: CLI login opens `/cli/auth`, the route disables the GitHub sign-in button while Convex auth is loading, and that loading state comes directly from the Convex client configured by `VITE_CONVEX_URL`.

## Best Possible Solution

Keep this issue open for maintainer runtime verification. Check the deployed `/cli/auth` flow and production Convex/Vercel configuration; if the WebSocket 307 still occurs, fix the deployment or runtime URL path and decouple the unauthenticated GitHub sign-in action from indefinite Convex auth loading, with a regression test for the stuck-loading pre-OAuth state. Close manually only after a verified production fix ships.
