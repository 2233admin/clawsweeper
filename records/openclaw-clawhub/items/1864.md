## Summary

Current main does not implement this PR’s intended change. Both text-file whitelists still omit `r`, while the CLI, web upload, and GitHub import paths lower-case extensions and require membership in `TEXT_FILE_EXTENSION_SET`; `.r`/`.R` files therefore remain filtered out. The PR is narrow and additive, with no security-sensitive files changed in the provided diff, so it should stay open for maintainer review/merge rather than be auto-closed.

## Best Possible Solution

Keep this PR open and review/merge the additive whitelist change, ideally with targeted tests for `.r` and `.R` through `TEXT_FILE_EXTENSION_SET` and at least one affected publishing/import path. After merge, release the CLI/schema package so R skills can republish with their source files included.
