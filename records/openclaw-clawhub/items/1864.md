## Summary

Keep this PR open. Current main still does not implement the intended additive whitelist change: both text-file extension lists omit `r`, and the CLI, web upload/archive, and GitHub import paths still lowercase extensions and require `TEXT_FILE_EXTENSION_SET` membership, so `.r` and `.R` files remain excluded unless this PR or an equivalent change lands.

## Best Possible Solution

Keep this PR open for maintainer review and merge the additive whitelist change, ideally with focused regression coverage for both lowercase `.r` and uppercase `.R` through the shared schema and the relevant publish/import filtering paths. If generated package artifacts are expected to stay committed for releases, update those before the next schema/CLI release as well.
