## Summary

Keep this PR open. Current main still omits `r` from both text-file extension whitelists, and the CLI, web upload, archive upload, and GitHub import paths still lower-case extensions and require `TEXT_FILE_EXTENSION_SET` membership. The provided PR is narrow and additive, and the Greptile/prior ClawSweeper discussion aligns with the local read: this work is still useful and not yet implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review and merge the additive whitelist change, ideally with focused tests that assert `r` is in both schema extension sets and that uppercase `.R` is accepted through at least `guessTextContentType` and the CLI/web import filtering path. After merge, release the schema/CLI package so affected R skills can republish with source files included.
