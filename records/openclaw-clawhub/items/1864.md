## Summary

Keep this PR open because current main still does not implement the requested R source-file support. Both whitelist copies omit `r`, the CLI/web/GitHub import paths still gate files through that set, and the existing focused tests do not cover lowercase `.r` or uppercase `.R`.

## What This Changes

The PR adds `r` to the shared and CLI-local text-file extension whitelists so `.r` and `.R` source files can be included as text files.

## Best Possible Solution

Land the additive R whitelist support in both schema sources, cover lowercase and uppercase R extensions in focused tests, and ensure the generated schema package output used by deployment and CLI release reflects the same whitelist.
