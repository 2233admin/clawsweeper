## Summary

Current main still does not implement this PR's intended additive whitelist change. Both text-file extension lists omit `r`, and the CLI/web/GitHub import filtering paths still require `TEXT_FILE_EXTENSION_SET` membership after lowercasing extensions, so `.r` and `.R` sources remain excluded without this PR or an equivalent change.

## Best Possible Solution

Keep this PR open for maintainer review and merge the additive whitelist change, ideally with focused regression coverage that asserts `r` is present in both schema extension sets and that uppercase `.R` is accepted through the shared content-type and upload/import filtering paths. After merge, release the schema/CLI package so affected R skills can republish with their source files included.
