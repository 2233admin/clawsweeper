## Summary

Keep this issue open. Repository policy for openclaw/clawhub allows auto-close only for pull requests that are certainly implemented on current main, and this is an issue. Current main also still lacks `.github/ISSUE_TEMPLATE` issue forms or broad support-intake labeling; the only related automation is the narrow rescan-guidance path from #1878.

## Best Possible Solution

Keep this issue open for maintainer/product follow-up. The best path is to approve the support categories and label taxonomy, add `.github/ISSUE_TEMPLATE` YAML forms plus `config.yml`, use static form-level labels where possible, link forms to maintainer-approved canonical threads, and only add a small `issues: write` triage workflow if GitHub issue-form labels are insufficient.
