Keep this issue open. Current main does not contain GitHub issue forms or a broad ClawHub support-intake labeling setup, and the only nearby merged automation is the narrower rescan-guidance path for explicit rescan/re-evaluation requests. The issue requests GitHub Issue Forms, support-category labels, required metadata fields, and links to canonical threads for recurring ClawHub support reports.

Approve the support taxonomy and canonical thread destinations, then add .github/ISSUE_TEMPLATE YAML forms plus config.yml using form-level labels where possible. Add a minimal issues: write triage workflow only for gaps that GitHub issue-form labels cannot cover.
