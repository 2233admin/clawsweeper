## Summary

Keep open. This is an issue, and the openclaw/clawhub cleanup policy for this run says to keep all issues open. The request is also still actionable on current main: skill version rows have a soft-delete field and public reads already hide/block soft-deleted versions, but I did not find an owner-facing CLI, API, or UI workflow to hide/delete/restore a single published skill version.

## Best Possible Solution

Keep this issue open and implement a first-class version lifecycle workflow: an authenticated API/mutation that targets skillVersions by skill slug plus version, owner/org-member/moderator authorization, clear handling for latest-version/tag promotion constraints, audit logging, public OpenAPI/docs updates, CLI commands such as clawhub version hide/delete/unhide/undelete, and focused tests that version history, detail, file, download, resolve, and owner/staff access behave correctly.
