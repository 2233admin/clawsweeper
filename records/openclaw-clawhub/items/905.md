## Summary

Keep this issue open. Current main already has internal version soft-delete state and public readers/downloads honor it, but the requested owner-facing workflow to hide/delete/restore one skill version is still missing from the CLI, HTTP API, docs, and management UI. Related #600 confirms a real incident-handling use case, while #1370 is adjacent package lifecycle work rather than a duplicate replacement.

## What This Changes

Add an owner-facing way to hide/delete and restore a single published skill version while keeping the skill slug live.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that resolve slug plus version, patch skillVersions.softDeletedAt, write version-specific audit logs, and keep public version lists, detail/file/download endpoints, tag/latest behavior, cache invalidation, docs, and restore semantics consistent.
