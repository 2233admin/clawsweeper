## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on main, and this item is an open issue. A fresh read of current main also confirms the requested owner-facing per-version hide/delete/restore workflow is still missing: skillVersions has soft-delete state and public reads honor it, but API, CLI, docs, and UI expose only whole-skill lifecycle controls.

## Best Possible Solution

Keep this issue open and implement first-class per-version lifecycle controls for skills. Add authenticated API and/or CLI commands that resolve slug plus version, authorize owners/org members/staff, patch skillVersions.softDeletedAt with audit logging, deliberately handle latest-version and tag constraints, preserve internal audit history, and add focused tests for version list/detail/file/download/resolve behavior. Coordinate policy with #600 while keeping #1370 as the separate package lifecycle track.
