## Summary

Keep this issue open. Current main has the data model and public read behavior for soft-deleted skill versions, but it still does not expose an owner-facing API, CLI, or UI workflow to hide/delete/restore one published skill version. The ClawHub cleanup policy also only permits auto-closing PRs that are certainly implemented on main, and this item is an issue.

## Best Possible Solution

Keep this issue open. Implement a first-class per-version lifecycle path for skills: authenticated API and CLI commands that resolve slug plus version, authorize owners/org members/staff, patch skillVersions.softDeletedAt with audit logging, define latest-version and tag behavior, block public reads/downloads consistently, and add focused tests/docs. Coordinate with #600 for the incident use case and keep #1370 as the separate package lifecycle track.
