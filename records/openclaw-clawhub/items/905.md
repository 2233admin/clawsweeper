## Summary

Keep this issue open. Current main can represent and hide an individual skill version after `skillVersions.softDeletedAt` is set, and public reads/downloads already respect that state, but there is still no owner-facing API, CLI, or UI workflow to hide/delete/restore one published version while keeping the skill live. The repository policy for this ClawHub cleanup pass also says issues should remain open; only PRs are eligible for auto-close when certainly implemented on main.

## Best Possible Solution

Keep this issue open. Implement a first-class per-version skill lifecycle path after maintainer policy review: authenticated v1 API routes and CLI or UI controls that resolve slug plus version, authorize owners/org members/staff, patch `skillVersions.softDeletedAt`, write audit logs, and handle latest/tag/download/cache semantics consistently. Coordinate with #600 for incident handling while keeping #1370 as the separate package/plugin lifecycle track.
