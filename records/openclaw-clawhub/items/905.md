## Summary

Keep this issue open. Current main can represent a version-level soft delete and public reads/downloads honor `skillVersions.softDeletedAt`, but there is still no owner-facing API, CLI, or UI workflow to hide/delete/restore one published skill version while leaving the skill live. The ClawHub cleanup policy also keeps issues open; #600 and #1370 are adjacent open reports, not superseding this skill-version request.

## What This Changes

Request owner-facing controls to hide, delete, unhide, or undelete a single published skill version without hiding or deleting the whole skill.

## Best Possible Solution

Keep this open for maintainer policy and product follow-up. Define the per-version lifecycle rules, then implement authenticated owner/org-member/staff API routes and CLI or UI controls that resolve slug plus version, authorize the actor, patch `skillVersions.softDeletedAt`, write audit logs, and consistently handle latest/tag/download/file/readme/cache semantics. Coordinate operationally with #600, and keep #1370 scoped to package/plugin lifecycle controls.
