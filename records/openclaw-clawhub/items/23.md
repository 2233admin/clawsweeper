## Summary

Keep this issue open. Current main and the modern `clawhub` npm package declare `undici`, but the originally reported legacy `clawdhub@latest` install path still resolves to `0.3.0` without `undici`, so Node users can still hit the reported runtime failure. ClawHub policy also keeps issues open for maintainer handling rather than auto-closing them.

## Best Possible Solution

Keep this open for maintainer npm-distribution handling. A maintainer should decide the legacy `clawdhub` package policy, then publish or retag a fixed legacy package, deprecate/document the broken legacy package as unsupported, or otherwise make sure affected install paths no longer resolve to `clawdhub@0.3.0` without `undici`; then close this together with #33 and #51 as appropriate.
