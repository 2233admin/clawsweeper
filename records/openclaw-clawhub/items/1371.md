## Summary

Keep open. This is a specific hosted-production listing deletion/hide request, not a source-tree cleanup that current main can prove completed. Current main has generic owner/moderator/admin skill soft-delete support and a separate code-plugin package family, but the live `unraidclaw` entry is still served as a skill-family listing today, so the issue still needs maintainer/admin follow-up and requester ownership verification.

## What This Changes

The issue requests deletion or hiding of the hosted `emaspa/unraidclaw` skill listing because UnraidClaw is an npm/OpenClaw plugin and should not be installed as a SKILL.md-based skill.

## Best Possible Solution

Keep this open for maintainer/admin follow-up. Verify requester ownership, then have the owner run `clawhub delete unraidclaw --yes`, call `DELETE /api/v1/skills/unraidclaw`, or have staff hide/delete the production skill-family record. If UnraidClaw should remain discoverable on ClawHub, publish it through the package flow as `family: "code-plugin"`. Close manually only after the public page/API no longer serve the mistaken skill-family listing, or after a corrected code-plugin listing replaces it.
