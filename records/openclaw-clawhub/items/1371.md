## Summary

Keep open. This is a concrete production catalog deletion/hide request for the hosted `emaspa/unraidclaw` listing, not a source-tree cleanup. Current main has generic owner/staff skill soft-delete support and a separate code-plugin package publishing path, but the live catalog still serves `unraidclaw` as a `family: "skill"` package and the source checkout has no committed listing to remove. ClawHub cleanup policy also keeps open issues open unless they are PRs certainly implemented on main.

## Best Possible Solution

Keep this open for maintainer/admin follow-up. Verify requester ownership, then have the owner run `clawhub delete unraidclaw --yes`, call `DELETE /api/v1/skills/unraidclaw`, or have staff hide/delete the production skill-family record. Close manually only after `https://clawhub.ai/api/v1/skills/unraidclaw` and `https://clawhub.ai/emaspa/unraidclaw` no longer serve the mistaken skill listing. If UnraidClaw should remain on ClawHub, publish it separately through the package flow as `family: "code-plugin"` and verify the install snippet becomes `openclaw plugins install clawhub:unraidclaw`.
