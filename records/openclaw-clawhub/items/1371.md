## What This Changes

The requester asks maintainers to delete the `emaspa/unraidclaw` ClawHub skill listing because UnraidClaw is an OpenClaw npm plugin, not a SKILL.md-based skill, and the listing directs users to the wrong skill installer path.

## Best Possible Solution

Verify that `emaspa` controls the listing, then hide or delete the production skill-family record. If UnraidClaw should stay discoverable, republish it through the package/plugin flow so ClawHub identifies it as a code plugin and shows the plugin installer flow.
