## Security Review

Status: needs_attention

Summary: This non-PR issue is security-sensitive because the live artifact is still flagged and the requested action would clear scanner warnings on a token-bearing shell command.

Concerns:

- **[medium] Review TLS bypass before clearing the flag:**
  - body: The published skill uses `curl -k` while sending `LOGINSIGHT_API_TOKEN` as a bearer token. Disabling certificate verification can expose the token to interception on misconfigured or untrusted networks, so a moderator should explicitly accept or reject that risk before marking the artifact clean.
  - confidence: 0.94
- **[low] Review dynamic host and token handling:**
  - body: The skill sends the bearer token to a host supplied by `LOGINSIGHT_HOST`. That may be expected for user-owned VMware deployments, but the combination of dynamic shell interpolation and credential forwarding should be reviewed before clearing the production suspicious flag.
  - confidence: 0.84
