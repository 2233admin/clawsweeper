## Summary

Keep this issue open. Current main now implements substantial automatic GitHub username/profile sync for derived personal handles, including login-triggered sync by immutable GitHub numeric ID, conflict checks, personal publisher updates, transfer lookup support, and canonical owner redirects. However, the repository cleanup policy for openclaw/clawhub only permits auto-closing PRs, not issues, and the remaining choices around explicit rename, old-handle aliases or reservations, URL semantics, docs/API coverage, and org ownership need maintainer judgment.

## What This Changes

The issue requests a supported way for ClawHub user handles and owner URLs to follow a GitHub account rename, either by explicit rename, login-time sync, or a prompted migration.

## Best Possible Solution

Maintain the current automatic sync path if that is the intended product answer, and separately define the remaining handle-rename policy: explicit opt-in rename versus automatic sync, old-handle alias or reservation behavior, public URL redirects, API/CLI/docs coverage, and coordination with org ownership in #1675.
