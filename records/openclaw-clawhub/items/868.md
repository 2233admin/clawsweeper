## Summary

Keep this issue open. Repository policy for openclaw/clawhub says issues should not be auto-closed, and current main still has the reported failure mode: `cmdSync` directly awaits each publish, `cmdPublish` rethrows upload errors, and the backend still returns slug conflicts as publish failures.

## Best Possible Solution

Keep this issue open for maintainer work. The likely code fix is in `packages/clawhub/src/cli/commands/sync.ts`: isolate `cmdPublish` failures per selected skill, continue later uploads, preserve the backend slug-conflict message and existing-owner URL, summarize successes and failures at the end, and return a failing exit status when any upload failed. Add a regression test where one selected publish rejects with `Slug is already taken` while a later selected skill still publishes.
