## Summary

Keep this issue open. Current main appears to have the direct user-to-user CLI/API payload path fixed: the CLI passes the positional recipient handle as `toUserHandle`, the HTTP handler only emits `toUserHandle required` when that JSON field is missing, and tests cover the command helper plus HTTP handler. However, this is an open ClawHub issue, and the supplied cleanup policy only permits auto-close for pull requests that are definitely implemented on main. The report also has corroboration, lacks confirmation of the reporter's installed CLI/deployed backend version, and may overlap with the still-open publisher/org transfer work in #1675 if the target is not a personal user publisher.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should verify the reporter's CLI version and current deployed backend behavior for the exact command, add a parser-level or end-to-end CLI regression if the published CLI still drops the recipient handle, and manually close once the fixed user-to-user path is confirmed live. If the destination is an org or publisher account rather than a personal user publisher, route the remaining work through #1675 and migrate transfers to publisher targets there.
