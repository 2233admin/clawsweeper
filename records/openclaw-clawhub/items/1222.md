## Summary

Keep open. This is a ClawHub issue, and the repository policy for this cleanup allows auto-close only for PRs that are certainly implemented on main. Current main also still contains the exact GitHub account-age gate failure path: skill web/API/GitHub import publishes call `publishVersionForUser`, plugin/package manual publishes call the same `requireGitHubAccountAge` gate, and that gate still throws the reported `GitHub API rate limit exceeded` error on GitHub 403/429 responses. The later comment that the same problem appears with Publish plugin is consistent with the package publish path.

## Best Possible Solution

Keep this issue open and handle it with the related publish-rate-limit reports, especially #676. A maintainer fix should make the publish gate more resilient or operationally verifiable: confirm production `GITHUB_TOKEN` coverage, improve retry/backoff and user-facing retry metadata, reduce repeated uncached GitHub lookups, and ensure both skill and plugin publish paths benefit from the same behavior.
