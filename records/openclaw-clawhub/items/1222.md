## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still contains the reported failure path: normal skill publish reaches the GitHub account-age gate, which fetches GitHub `created_at` when `githubCreatedAt` is missing and maps GitHub 403/429 to the exact rate-limit error. The Publish plugin comment is consistent with manual package publish using the same gate.

## What This Changes

Publishing a new skill can fail with `GitHub API rate limit exceeded`, and a commenter reports the same failure in the Publish plugin flow.

## Best Possible Solution

Keep the account-age gate, but remove avoidable publish-time dependence on unauthenticated GitHub API calls: verify production `GITHUB_TOKEN` and 403/429 logs, then persist or backfill GitHub `created_at` for users missing `githubCreatedAt` if maintainers choose a code mitigation.
