## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass only permits auto-closing pull requests that current main definitely implements. Substantively, current main still has the reported failure mode: when `user.githubCreatedAt` is missing, the publish gate fetches GitHub `created_at`, maps GitHub 403/429 to the exact rate-limit error, and the same gate is used by skill publish, GitHub import publish, and manual plugin/package publish.

## What This Changes

Publishing a skill reports a GitHub API rate-limit error during upload or import when the publish gate cannot fetch the user's GitHub account age.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Verify production Convex `GITHUB_TOKEN` and GitHub 403/429 logs first, then decide whether to add a code mitigation such as capturing GitHub `created_at` during sign-in and/or running a controlled backfill for users with missing `githubCreatedAt`, while preserving the account-age gate. Use #676 as the closest related publish-error thread if maintainers consolidate.
