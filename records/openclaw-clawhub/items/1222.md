## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this pass only permits auto-closing pull requests that current main definitely implements. Substantively, current main still has the reported failure mode: the exact message comes from the GitHub account-age publish gate when `githubCreatedAt` is missing and GitHub returns 403/429. The same gate is used by skill publish, GitHub import publish, and manual plugin/package publish. Docs already identify `GITHUB_TOKEN` in the Convex environment as the mitigation, so the remaining work looks like maintainer production/config verification plus a possible code follow-up to reduce first-publish lookups, not something that should be closed by cleanup.

## Best Possible Solution

Keep this issue open and have maintainers verify the production Convex `GITHUB_TOKEN`/quota state, then decide whether to add a code-side mitigation such as capturing/caching GitHub `created_at` during sign-in or a controlled backfill for existing users with missing `githubCreatedAt`. Keep the account-age gate intact, and use #676 as the closest duplicate/canonical publish-error thread if maintainers want to consolidate.
