## Summary

Keep open. The WeWork duplicate still reproduces in production, and current main only de-duplicates exact package family/name matches, so historical alias-equivalent package records remain separate catalog/search results; ClawHub policy also keeps issues open for maintainer follow-up.

## What This Changes

Collapse plugin catalog/search results so historical package aliases such as `@openclaw/wework` and `openclaw-wework` render as one canonical plugin card while preserving legacy detail/install locators.

## Best Possible Solution

Define an explicit canonical package/alias model for plugin records, preserve legacy detail/install locators, collapse browse and search on that approved canonical identity, and add WeWork-style regressions for both plugin list and search paths.
