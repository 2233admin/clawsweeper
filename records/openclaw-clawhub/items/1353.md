## Summary

Keep open. The duplicate WeWork plugin results still reproduce in production, and current main still collapses only exact package family/name matches rather than alias-equivalent package identities. This is also an issue in openclaw/clawhub, where the cleanup policy says issues should remain open for maintainer handling rather than auto-closing.

## What This Changes

The issue reports duplicate WeWork plugin catalog cards caused by historical or alias package slugs being returned as separate plugin records.

## Best Possible Solution

Define an explicit canonical package/alias model for plugin records, preserve compatibility for legacy detail and install locators, collapse browse and search on canonical identity, and add WeWork-style backend and frontend regression coverage plus any needed cursor-based data cleanup.
