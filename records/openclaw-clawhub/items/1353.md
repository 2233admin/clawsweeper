## Summary

Keep open. This is a ClawHub issue, and the repository policy for this cleanup pass only permits auto-closing pull requests that are certainly implemented on main. The current main implementation also does not show a complete fix for the reported duplicate-card class: plugin catalog browse/search is backed by package digest rows and only de-duplicates exact package IDs or exact family/name collisions, not alias-equivalent package identities such as `openclaw-wework` and `@openclaw/wework`.

## Best Possible Solution

Keep this issue open and implement package-level canonical identity handling for plugin browse/search. The likely path is to introduce or use a canonical package/alias mapping, collapse catalog results by canonical package identity or runtime plugin ID, decide which locator is canonical, add regression tests covering `openclaw-wework` versus `@openclaw/wework`, and run a data migration/backfill for historical duplicates.
