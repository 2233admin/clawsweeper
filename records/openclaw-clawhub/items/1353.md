Keep open. The ClawHub cleanup policy keeps issues open, and the underlying duplicate is still observable: production plugin search returns both WeWork package records, while current main only de-duplicates exact family/name keys and has no package alias/canonicalization layer for scoped-versus-legacy package locators.

Keep this issue open for maintainer follow-up.
