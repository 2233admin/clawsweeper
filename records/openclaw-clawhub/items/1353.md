## Summary

Keep open. This is a ClawHub issue, and the repository policy does not allow automated closure of issues. Current main also still lacks catalog-level canonical package or runtime-id de-duplication for alias-equivalent plugin records such as `openclaw-wework` and `@openclaw/wework`; related PR #1842 narrowed plugin catalog queries to plugin families but did not collapse historical aliases.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best path is to decide the canonical package/alias model for historical plugin package records, preserve legacy detail/install locator compatibility, collapse browse/search by canonical package identity or runtime plugin ID, add a WeWork-style regression for `/api/v1/plugins` and `/api/v1/plugins/search`, and run any required cursor-based backfill or moderation workflow for existing duplicate records.
