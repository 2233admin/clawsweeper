## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass only permits auto-closing pull requests that are certainly implemented on current main. The current code also still does not fully collapse alias-equivalent plugin package records such as `openclaw-wework` and `@openclaw/wework`; recent #1842 work constrained plugin catalog queries to plugin families, but de-duplication is still by exact package family/name or package id, not by canonical package identity.

## Best Possible Solution

Keep this issue open and implement package-level canonical identity handling for plugin browse/search. The likely path is to add or use a canonical package/alias mapping, decide the canonical locator, collapse list/search results by canonical package identity or runtime plugin id, preserve existing detail/install locator compatibility, add regression tests for `openclaw-wework` versus `@openclaw/wework`, and run a data migration/backfill for historical duplicate records.
