## Summary

Keep open. ClawHub policy keeps issues open for maintainer follow-up, and the reported duplicate still reproduces in production: plugin search returns both `@openclaw/wework` and `openclaw-wework`. Current main constrains `/api/v1/plugins` to plugin families, but it still does not collapse historical alias-equivalent package records into one canonical catalog card.

## What This Changes

Request that plugin catalog list/search collapse alias-equivalent package records, such as scoped and historical WeWork package names, into one canonical catalog card while preserving legacy detail/install locators.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Decide the canonical package/alias model for historical plugin package records, preserve legacy package detail/install locators, collapse browse/search by approved canonical identity, add WeWork-style regressions for `/api/v1/plugins` and `/api/v1/plugins/search`, and run any needed cursor-based backfill or moderation cleanup for existing duplicates.
