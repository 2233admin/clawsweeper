## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main still rejects the reported skill/plugin slug collision and the scan recovery paths do not prove this package’s pending VT state has been cleared.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The practical path is to resync or manually update VirusTotal data for `claw-pay` v0.2.5, or soft-delete the obsolete plugin record if `claw-pay-plugin`/the `claw-pay` skill is canonical. Separately, maintainers should make an explicit product decision on whether same-owner skill/plugin slug collisions need a moderated migration or override path, coordinated with #1272 and #1744.
