## Summary

Keep open. Current main still does not implement native per-skill pinning: there are no pin/unpin commands, update does not check or skip pinned entries, list has no pin indicator, and the lockfile schema stores only version plus installedAt. PR #1806 is open, unmerged, and explicitly targets this request with a closing reference, so maintainers should review or close that PR rather than pre-close this issue.

## Best Possible Solution

Keep this issue open and review/land PR #1806, or an equivalent maintainer-approved patch, that adds native pin/unpin commands, persists pin metadata, shows pin state in list, blocks direct updates of pinned skills, skips and reports pinned entries during update --all, and honors pins even with --force. After that PR merges, the linked closing reference can close this issue normally.
