## Summary

Keep open. Current main still lacks native per-skill pinning in the ClawHub CLI, and open PR #1806 is already the active implementation candidate linked to this requested behavior. Because this is an issue in openclaw/clawhub and a paired open PR exists, the right cleanup path is maintainer review of that PR, not closure or duplicate automation.

## What This Changes

Add native ClawHub CLI per-skill pinning so selected installed skills are preserved from direct, forced, and all-skill update flows until explicitly unpinned.

## Best Possible Solution

Land PR #1806 or an equivalent maintainer-approved patch that adds pin/unpin commands, persists optional pin metadata in the lockfile, displays pin state in list output, rejects direct updates for pinned skills, skips and reports pinned entries for update --all, and honors pins even when --force is supplied.
