## Summary

Keep this issue open. Current main still treats user-initiated account deletion as irreversible deactivation: deletion keeps the user row deactivated/purged, OAuth sign-in rejects deleted/deactivated users, active-user helpers hide those rows, and tests/docs assert that logging in again does not restore the account. This is a real account-lifecycle and support policy decision, not a conservative cleanup close candidate.

## Best Possible Solution

Keep this issue open and consolidate it with the related deleted-account sign-in reports. Maintainers should decide whether deleted GitHub identities should support self-service re-registration, admin-only hard purge/reset, or permanent blocking with clearer UX/support guidance, then implement that policy in the Convex auth/account lifecycle path with tests and docs.
