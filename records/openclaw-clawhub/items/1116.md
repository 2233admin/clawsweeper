## Summary

Keep open. For openclaw/clawhub, the cleanup policy says issues should stay open; this is also not implemented on current main. The current auth and deletion paths intentionally make user-initiated deletion irreversible and block GitHub OAuth reauthorization for deleted/deactivated users, while the report and comments describe a concrete recurring account-state problem.

## Best Possible Solution

Keep this issue open for maintainer product decision and implementation. If deleted accounts must remain irreversible, improve the OAuth/UI/support path so users see a durable deleted-account message and know how account recovery or manual cleanup is handled. If re-registration should be allowed, add a backend path that safely unlinks or supersedes the old Convex Auth identity for self-deleted users only, preserves ban semantics and handle rules, revokes old tokens, and covers web OAuth plus CLI/API token flows with tests.
