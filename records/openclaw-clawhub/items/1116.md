## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing PRs that are certainly implemented on current main, and the current backend still intentionally blocks same-GitHub sign-in after user-initiated deletion rather than implementing the requested re-registration or admin reset path. The newer code improves the failure message, but the underlying account recovery problem remains unresolved.

## Best Possible Solution

Keep this open for maintainer product and support triage. Decide whether user-initiated deletion is permanently irreversible or whether same-GitHub re-registration should be supported. If recovery is intended, add a backend flow that safely unlinks or supersedes the old Convex Auth identity for self-deleted users only, revokes old tokens, preserves ownership/audit protections, and covers web OAuth plus CLI/API-token flows with tests. If deletion remains irreversible, improve login and support documentation so affected users know whether manual cleanup is possible.

- A recovery path must distinguish self-deleted accounts from banned or moderator-deactivated accounts so OAuth rebinding cannot bypass moderation.

- If permanent deletion remains intended, the hosted login/support UX still needs an actionable outcome instead of an account-specific loading loop.
