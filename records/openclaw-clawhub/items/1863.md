Keep this issue open. It is an open ClawHub issue, and this cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, current main still lacks an indexed normalized skill name/slug prefix recall path, so the ScienceClaw case-stability report remains plausible and closely overlaps the existing direct-prefix search gap tracked by #1254 and open PR #1256.

## Best Possible Solution

Keep this issue open and handle it through PR #1256 or an equivalent focused fix. The likely fix is to add normalized slug/display-name lexical fields to `skillSearchDigest`, populate them on writes, backfill existing digest rows safely, add active/non-suspicious prefix indexes, merge direct prefix candidates into `searchSkills` before vector/recent fallback ranking, and add regression tests asserting `scienceclaw` and `ScienceClaw` return the same ScienceClaw result set.
