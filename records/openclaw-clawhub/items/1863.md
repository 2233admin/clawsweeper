## Summary

Keep open. This is a ClawHub issue, and the provided repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. The current source also still supports the reported bug: /skills query mode calls the Convex search action, searchSkills is embedding-first with only an exact slug shortcut, lexical fallback is bounded to recent skillSearchDigest windows, and skillSearchDigest has no normalized indexed display-name prefix field. The open related issue #1254 appears to track the same direct name/slug prefix recall gap.

## Best Possible Solution

Keep this issue open and either fix it directly or consolidate it with #1254. The likely code path is to add denormalized normalized skill name and slug fields to skillSearchDigest, backfill them, add active/non-suspicious prefix indexes, query that direct lexical path before vector search, merge and rank with vector results, and add regression coverage proving scienceclaw and ScienceClaw return the same ScienceClaw-prefixed result set.
