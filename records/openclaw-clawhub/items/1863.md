## Summary

Keep open. Repository policy for this ClawHub cleanup pass does not allow auto-closing issues, and the current main code still supports the reported bug: /skills query mode goes through an embedding-first search action, lexical fallback is bounded to recent digest windows, and skillSearchDigest has no normalized indexed display-name prefix path for stable case-insensitive recall. The related #1254 remains open and appears to track the same underlying direct-name/slug recall gap.

## Best Possible Solution

Keep this issue open and either fix it directly or consolidate it with #1254. The likely product/code path is to add denormalized normalized skill name and slug fields to skillSearchDigest, backfill them, add indexed active/non-suspicious prefix lookup, query that direct lexical path before vector search, merge and rank with vector results, and add regression coverage proving scienceclaw and ScienceClaw return the same ScienceClaw-prefixed result set.
