## Summary

Keep this issue open. The repository-specific ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on current main, not issues. Source review also supports the report as a plausible live search recall gap: skill search still gates most name-prefix recall through vector candidates plus a bounded recent-digest fallback, and current main does not have an indexed normalized display-name prefix lookup for skills.

## Best Possible Solution

Keep open and fix or consolidate with the existing search-recall work in #1254/#1256. The likely product/code path is to add denormalized normalized skill name/search fields on skillSearchDigest, index them for case-insensitive prefix lookup, query that direct lexical path before vector search, merge and rank with vector results, and add regression coverage asserting scienceclaw and ScienceClaw return the same ScienceClaw-prefixed result set.
