## Summary

Keep open. ClawHub cleanup policy for this run only permits auto-closing pull requests that are certainly implemented on current main, not issues. Source review also supports the report as a plausible live bug: /skills query mode still calls an embedding-first search action, lexical skill fallback is bounded to recent digest windows, and skillSearchDigest has no normalized indexed display-name prefix path to guarantee case-insensitive direct name recall. The related #1254 remains open and appears to track the same underlying search-recall class.

## Best Possible Solution

Keep open and fix or consolidate with #1254. The likely implementation is to add denormalized normalized skill name and/or slug fields to skillSearchDigest, backfill them, index them for active non-suspicious prefix lookup, query that direct lexical path before vector search, merge and rank with vector results, and add regression coverage proving scienceclaw and ScienceClaw return the same ScienceClaw-prefixed result set.
