## Summary

Keep this issue open. Current main still contains the reported Convex failure shape: the named owner-publisher skill digest helper can call `.paginate()` repeatedly inside one internal mutation, and the publish path still reaches publisher resolution and personal-publisher synchronization. Related ClawSweeper reviews for the same root cause also kept the clustered issues open.

## Best Possible Solution

Keep this open and land one focused backend fix that makes owner-publisher digest rebuilds cursor-batched: each Convex internal mutation should perform at most one `.paginate()` call, process that page, and schedule the next page when `continueCursor` is present. Cover both skill and package owner-publisher digest helpers, preserve indexed queries and missing-table fallbacks, and add tests for multi-page scheduling.

Fix the Convex multiple-paginated-query failure in owner-publisher digest sync. Current `convex/functions.ts` has `syncSkillSearchDigestsForOwnerPublisherId` and `syncPackageSearchDigestsForOwnerPublisherId` looping over `.paginate()` inside a single internal mutation invocation. Refactor these owner-publisher rebuilds so each Convex function invocation performs at most one paginated query, processes that page, and schedules the next page with the cursor when needed. Preserve the existing indexed `by_owner_publisher` queries, digest field behavior, missing-table fallbacks, and package/skill scheduling semantics.
