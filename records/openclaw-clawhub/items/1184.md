## Summary

Keep this issue open. The ClawHub cleanup policy says issues are not auto-closed, and code inspection shows current main has a likely mitigation but not a fully risk-free proof: publisher-triggered package and skill digest sync are now scheduled as separate internal mutations, but each owner-publisher sync helper still loops over a paginated query inside one function.

## Best Possible Solution

Keep the issue open for maintainer triage. If production confirms the current scheduled split fixed the reporter's case, a maintainer can manually close it after noting the deployed fix. To harden the code path, convert owner-publisher digest sync to process one page per scheduled internal mutation with an explicit cursor argument and schedule the next page, then add a regression test that covers multi-page owners.
