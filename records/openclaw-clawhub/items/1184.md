## Summary

Keep open. This is an open ClawHub issue, and current main still contains the reported Convex failure shape: the owner-publisher skill digest helper loops over `.paginate()` inside one internal mutation invocation, with a package sibling using the same pattern. The previous ClawSweeper comment and related reports point to the same unresolved clustered backend fix.

## What This Changes

Publishing can fail because `syncSkillSearchDigestsForOwnerPublisherId` may run multiple paginated Convex queries inside a single function invocation.

## Best Possible Solution

Keep this open and land one focused backend fix that makes owner-publisher package and skill digest rebuilds cursor-batched: each internal mutation should perform at most one `.paginate()` call, process that page, and schedule the next page when `continueCursor` is present. Preserve indexed queries and missing-table fallbacks, add multi-page scheduling tests, then consolidate the related reports after the fix merges.
