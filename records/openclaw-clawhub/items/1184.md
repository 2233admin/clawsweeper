## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub only permits automated closure for PRs that current main definitely implements, and this item is an issue. Source review also confirms the named owner-publisher skill digest helper still loops over `.paginate()` inside one Convex mutation invocation; current main only schedules package and skill publisher digest rebuilds as separate background mutations, which mitigates the inline double-pagination path but does not prove multi-page owners are fixed.

## Best Possible Solution

Keep this issue open for maintainer triage or manual consolidation with the related open publish-pagination reports. The durable fix is to refactor both owner-publisher digest sync internals to accept `cursor` and `numItems`, process one page per internal mutation, reschedule with `continueCursor` until done, and add regression coverage proving publisher digest sync never performs multiple paginated queries in a single Convex function invocation.
