## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy for this review only permits auto-closing pull requests that are certainly implemented on current main. Current main has meaningful rate-limit mitigations, headers, retry guidance, and docs, but the reported anonymous install UX is still not fully resolved: `clawhub install` can still run without a saved token, downloads are still IP-limited for anonymous users, and the web install UI still advertises the `npx clawhub@latest install ...` path discussed in the report. Related open #1410 tracks the narrower anonymous false-positive 429 path.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is either to manually consolidate it into #1410 as the canonical anonymous-install 429 thread, or to implement a concrete install UX fix: surface a specific login/token hint on 429, prompt or require login before rate-limited downloads when appropriate, update web install guidance if `npx clawhub@latest install ...` remains prone to anonymous rate-limit confusion, and verify production proxy/client-IP handling for anonymous buckets.
