## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main includes meaningful rate-limit mitigations, but the reported install UX is still not fully resolved: anonymous installs remain IP-limited, installs only authenticate when a saved token exists, and the web UI still advertises the unauthenticated `npx clawhub@latest install ...` path discussed in the report. The issue discussion also shows multiple users relying on workarounds, and related open #1410 tracks the narrower anonymous false-positive 429 path.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is either to manually consolidate it into open #1410 as the canonical anonymous-install 429 thread, or to implement a concrete install UX fix: surface a specific login/token hint on 429, prompt or require login before rate-limited downloads when appropriate, update web install guidance if `npx clawhub@latest install ...` remains prone to anonymous rate-limit confusion, and verify production proxy/client-IP handling for anonymous buckets.
