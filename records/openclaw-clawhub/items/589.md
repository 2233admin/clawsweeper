## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy keeps ClawHub issues open unless maintainers handle them directly. Substantively, current main still has the anonymous install/download 429 path: downloads are rate-limited before ZIP delivery, anonymous requests use an IP or `unknown` bucket, and `clawhub install` only sends auth when a saved token already exists. The discussion contains useful login/token and skills.sh workarounds, but related #1410 remains open with a narrower anonymous-install reproduction.

## What This Changes

Anonymous `npx clawhub@latest install <skill>` attempts can fail with `Rate limit exceeded`, and reporters ask for a real fix or clearer login/install guidance.

## Best Possible Solution

Keep this issue open and consolidate maintainer triage with #1410 as the canonical anonymous-install 429 reproduction, while referencing #349 for broader rate-limit visibility. The best fix is to verify production client-IP forwarding and anonymous bucket attribution, decide whether installs should prompt for or require login before rate-limited downloads, and add install-specific 429 guidance that points users to `clawhub login` or `clawhub login --token ...` when authentication would move them to a user bucket.
