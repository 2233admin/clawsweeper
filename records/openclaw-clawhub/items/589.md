## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only auto-closes ClawHub PRs that are certainly implemented on current main. Current main still has the relevant install behavior: downloads are rate-limited before ZIP delivery, anonymous installs use an IP or `unknown` bucket, and `clawhub install` only uses auth when a token already exists. The skill detail install UI has moved away from the reported `npx clawhub@latest install ...` command, but the underlying anonymous-install 429 path remains and is also tracked by open related issue #1410.

## Best Possible Solution

Keep this issue open and likely consolidate the remaining work with #1410 as the canonical anonymous-install 429 thread. The best path is to verify production client-IP forwarding and bucket attribution, decide whether installs should prompt for or require login before rate-limited downloads, add an install-specific 429 hint for token/login use, and keep the updated skill-detail install surface that avoids the old `npx clawhub@latest install ...` guidance.
