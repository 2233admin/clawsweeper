## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should not be auto-closed. Substantively, the production listing is still flagged as suspicious for `suspicious.potential_exfiltration` on `scripts/convert.js`, even though the scan details show clean/benign VT and LLM findings. Current main has staff manual-override tooling and structured scanner behavior, but it does not prove this specific moderation appeal has been resolved or that Markdown remote-image fetching has a shipped whitelist.

## Best Possible Solution

Keep this issue open for maintainer moderation. If staff agrees this is a false positive, either apply the existing manual clean override for this listing and rescan, or implement a narrow scanner exception for documented Markdown/DOCX remote-image fetching that does not involve credentials, arbitrary file upload, or hidden endpoints.
