## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, the linked live examples now return no public moderation block, but current main still has no explicit test/spec/fixture policy in the ClawScan or static moderation input paths, and the CLI still blocks non-interactive install/update when any skill is marked suspicious without `--force`.

## Best Possible Solution

Keep this open for product/security maintainer judgment. The best path is to define explicit scanner semantics for test/spec/fixture/mock/example content, apply that policy consistently across ClawScan LLM evaluation and deterministic static moderation, preserve scanning for runtime or hook-installed files, and make CLI install/update behavior reflect trusted review or cleared false positives without weakening detection of genuinely malicious skills.
