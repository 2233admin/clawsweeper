## Summary

Keep open. The ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Substantively, current main still feeds test/spec/fixture-looking files into LLM and static moderation paths, while the related PR that targeted test-file exclusion was closed unmerged; the merged false-positive PR only partially addressed a different keyword class.

## Best Possible Solution

Keep this issue open as the product/security tracking item. Maintainers should define how ClawHub scanners distinguish runtime skill behavior from security-skill tests and fixtures, add shared regression coverage for .test.*, .spec.*, __tests__, fixtures, mocks, and test directories across LLM and static scan paths, preserve scanning for runtime files, and align manual override plus CLI install/update behavior so known false positives do not unnecessarily block legitimate security skills.
