## Summary

Keep open. This is a ClawHub issue, and the cleanup policy for this repository only permits auto-closing pull requests that current main definitely implements. Current main has substantial VT pending-scan recovery and owner rescan plumbing, but it still does not expose the per-item queue/status details requested in the follow-up discussion, and exposing queue position or worker heartbeat needs maintainer product/security judgment.

## What This Changes

The issue reports a skill remaining in a pending state after VirusTotal passed, with related comments asking for clearer per-item scan queue/status visibility.

## Best Possible Solution

Provide an authenticated owner-visible scan status surface that uses indexed or denormalized metadata and exposes safe fields such as scanner status, last checked time, check count, stale reason, latest/in-progress rescan request, and a conservative next retry estimate, while keeping raw worker health and global queue internals private unless maintainers explicitly approve them.
