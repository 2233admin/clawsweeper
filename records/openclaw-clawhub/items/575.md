## Summary

Keep this issue open. ClawHub cleanup policy says issues should remain open, and current main still does not enforce uppercase-only SKILL.md / PROTOCOL.md: multiple docs, CLI, UI, import, publish, and readback paths continue accepting lowercase or plural skill marker names, and I found no PROTOCOL.md validation surface.

## Best Possible Solution

Keep this issue open for maintainer follow-up. A complete fix should define the canonical filename policy, add case-insensitive duplicate detection and uppercase validation across CLI scan/sync, CLI publish, browser upload, GitHub import, server/API publish validation, readback behavior, docs, and tests, and include a migration path for already-published lowercase or plural marker files.
