## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy says ClawHub issues should remain open unless they are PRs already implemented on main. Independently, uppercase-only enforcement is still not implemented: current docs and publish/import/read paths continue to accept lowercase `skill.md` and/or plural `skills.md`, and I found no current `PROTOCOL.md` filename validation path.

## Best Possible Solution

Keep this issue open for maintainer follow-up. A complete fix should define the canonical filename policy, add case-insensitive duplicate detection and uppercase validation across CLI scan/sync, CLI publish, browser upload, GitHub import, server/API publish validation, readback behavior, docs, and tests, and include a migration or compatibility plan for existing lowercase or plural marker files.
