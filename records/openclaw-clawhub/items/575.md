## Summary

Keep this issue open. Under the ClawHub cleanup policy, issues are not auto-closed, and current main still documents and implements lowercase `skill.md` plus `skills.md` compatibility across scan, publish, upload, import, and readme-resolution paths; `PROTOCOL.md` also has no current validation surface to enforce.

## What This Changes

Request to enforce exact uppercase `SKILL.md` and `PROTOCOL.md` marker filenames and reject lowercase or case-insensitive duplicate variants.

## Best Possible Solution

Keep this open for maintainer product follow-up.
