## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review says to auto-close only pull requests that are certainly implemented on main. The requested uppercase-only enforcement is also not present on current main: docs and multiple publish/import/read paths still intentionally accept lowercase or plural skill marker variants, and I found no `PROTOCOL.md` filename validation surface.

## Best Possible Solution

Keep this issue open for maintainer decision and implementation. A complete fix would add canonical uppercase validation and case-insensitive duplicate detection across CLI scan/sync, CLI publish, browser upload, GitHub import, API/server publish validation, readback behavior, docs, tests, and any cleanup or migration plan for already-published content; it should also explicitly decide whether legacy `skill.md` and `skills.md` compatibility remains supported.
