## Summary

Keep this issue open. Current main still documents and implements lowercase and plural skill readme compatibility across docs, CLI discovery and publish, browser upload, GitHub import, server publish validation, and stored readme resolution; uppercase-only enforcement is not implemented. This repository's cleanup policy also keeps issues open unless maintainers resolve them, and the remaining work is a breaking format-policy and migration decision rather than an automatic cleanup close.

## What This Changes

Request to require canonical uppercase `SKILL.md` and `PROTOCOL.md` filenames and reject lowercase or duplicate case variants that break case-insensitive filesystems.

## Best Possible Solution

Maintainers should decide the canonical casing policy for skill package files, define whether `PROTOCOL.md` belongs in the package contract, then update docs and validation consistently across CLI scan/sync/publish, browser upload, GitHub import, server publish, stored readme resolution, and any existing content cleanup plan.
