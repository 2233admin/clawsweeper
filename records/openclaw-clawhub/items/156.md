## Summary

Keep this issue open. Current main records download and star events, but visible counters still have plausible freshness gaps: skill document counters are synced on a 6-hour cron, list/search rows are backed by skillSearchDigest stats, and the related latency PR #176 was closed unmerged. ClawHub policy also says issues are not eligible for automatic close in this repository.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify production cron health and event backlog, then either make public counters read a fresher source, sync skillSearchDigest when stat deltas are applied, adjust the skill document sync cadence with reactive-load safeguards, or document the expected delay plus hourly download dedupe behavior if delayed counters are intentional.
