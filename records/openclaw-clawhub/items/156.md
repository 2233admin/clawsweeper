## Risks / Open Questions

- Users can still perceive downloads and stars as broken because public counters are backed by skill document fields that sync every 6 hours.
- Repeated downloads by the same authenticated user or IP within the same hour intentionally do not increment because downloadDedupes suppress duplicates.
- The download metric path is best-effort and catches errors so a successful download can still miss a metric event if the metric mutation path fails.
- Star count reconciliation exists as an internal action, but it is not wired into crons in current main, so production drift still needs operational verification.
