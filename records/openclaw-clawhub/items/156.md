## Summary

This should stay open under the ClawHub policy for issues, and the current evidence does not support an automated close or a blind repair PR. Current main does record download and star events, but visible skill counters are still synced through a delayed/stat-batching path, and the related cron-only PR was closed unmerged as not addressing the root cause.

## What This Changes

The issue reports that skill star and download actions do not update the visible skill metrics from the CLI or ClawHub website.

## Best Possible Solution

First verify production `skillStatEvents` backlog, Convex logs, and cron execution. Then either fix the event processing path if events are stuck, or define a safer freshness model for visible counters, such as a dedicated lightweight counter/summary path or an explicitly chosen sync cadence that preserves dedupe and avoids reactive query churn.
