## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are definitely implemented on main. The underlying product concern also still exists: current main renders the top warning from skill-level moderation state, while the scan panel reads latest-version scanner/static fields, and production still reports web-search-pro as suspicious for `suspicious.potential_exfiltration` even though the version security endpoint shows VirusTotal and OpenClaw as clean/benign.

## Best Possible Solution

Keep this issue open for maintainer moderation and UI follow-up. Maintainers should decide whether the current `suspicious.potential_exfiltration` finding for web-search-pro is valid; if valid, the skill page should surface that unresolved moderation source near the banner, and if false-positive, clear it through the existing moderator override or rescan path. If the team wants one tracker for this broader banner-confusion class, consolidate with related open reports such as #450, #733, or #768.
