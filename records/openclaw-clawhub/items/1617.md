## Summary

Keep open. This is an open ClawHub issue, and the repository-specific review policy only allows auto-closing pull requests. The codebase has partial optional-env support through `envVars`, but the public skill-format docs still point publishers at `requires.env`, the CLI schema copy lacks `envVars`, and draft PR #1859 is an open implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open while #1859, or an equivalent maintainer-approved change, aligns the skill-format docs, CLI schema copy, scanner/security prompt wording, and regression coverage around optional environment declarations. The narrow path is to document `metadata.openclaw.envVars` entries with `required: false`; if maintainers prefer `requires.optionalEnv`, that spelling should be implemented consistently across schema, parser, scanner, docs, CLI, API, and UI.
