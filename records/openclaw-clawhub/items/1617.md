## Summary

Keep this issue open. The ClawHub cleanup policy only permits automated close proposals for PRs that are certainly implemented on current main, not issues. Substantively, the issue still has unresolved alignment work: current main has partial optional-env support through `envVars` with `required: false`, but the public skill format docs still direct authors to `requires.env`, the CLI schema copy does not expose `envVars`, and an open PR is the active implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and use PR #1859 as the implementation candidate. The best path is to document `metadata.openclaw.envVars` entries with `required: false`, align the CLI schema copy and scanner/security prompt, and keep regression coverage showing optional env vars do not need to be declared under `requires.env`. If maintainers prefer `requires.optionalEnv`, implement that spelling consistently across schema, parser, scanner, docs, CLI, API, and UI.
