## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy keeps issues open unless the item is a pull request certainly implemented on main. Substantively, current main only partially resolves optional environment declarations: shared schema/parser/UI paths understand `envVars` with `required: false`, but the public skill-format docs still direct authors to `requires.env`, the CLI schema copy lacks `envVars`, and draft PR #1859 is the open unmerged implementation candidate for this issue.

## Best Possible Solution

Keep this issue open while PR #1859, or an equivalent maintainer-approved change, aligns the public skill-format docs, CLI schema copy, scanner/security-prompt wording, and regression coverage around optional environment declarations. The narrowest path appears to be documenting `metadata.openclaw.envVars` entries with `required: false`; if maintainers prefer `requires.optionalEnv`, that spelling should be implemented consistently across schema, parser, scanner, docs, CLI, API, and UI.
