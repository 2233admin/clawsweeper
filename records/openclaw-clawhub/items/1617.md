## Summary

Keep open. The repository cleanup policy for ClawHub keeps issues open in this lane, and current main still only partially resolves the request: shared schema/parser/UI paths understand `envVars` with `required: false`, but the public skill-format docs still direct authors to `requires.env`, the CLI schema copy lacks `envVars`, and draft PR #1859 remains the open linked implementation candidate.

## What This Changes

The issue asks ClawHub to clarify or implement how skills declare optional environment variables without falsely marking them as required under `requires.env`.

## Best Possible Solution

Align on one supported optional-env declaration, likely `metadata.openclaw.envVars` entries with `required: false` unless maintainers prefer `requires.optionalEnv`; document it in `docs/skill-format.md`, add it to the CLI schema copy, keep scanner/security-prompt behavior consistent, and land regression coverage through PR #1859 or an equivalent maintainer-approved change.
