## Summary

Keep open. The repository cleanup policy for openclaw/clawhub keeps issues open, and current main only partially addresses optional skill environment variables: shared schema/parser/UI paths support `envVars` with `required: false`, but the public skill-format docs, CLI schema copy, and security-prompt summary remain inconsistent. Open draft #1859 is the active implementation candidate and references this issue with closing syntax.

## What This Changes

The issue asks ClawHub to define and document optional skill environment variable declarations so optional credentials are not misrepresented as required or flagged as undeclared.

## Best Possible Solution

Align on one supported optional-env declaration, likely `metadata.openclaw.envVars` entries with `required: false` unless maintainers choose `requires.optionalEnv`; document it, add it to the CLI schema copy, keep scanner/security-prompt behavior consistent, and land regression coverage through #1859 or an equivalent maintainer-approved change.
