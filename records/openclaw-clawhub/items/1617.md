## Summary

Keep open. Current main has partial support for optional skill environment metadata, but the public docs and CLI schema copy are still inconsistent, and an open draft PR already references this issue with closing syntax and attempts the focused fix. Repository cleanup policy also keeps ClawHub issues open unless a close reason is strongly supported.

## What This Changes

Clarify how skill metadata should declare optional environment variables so scanner warnings do not force semantically incorrect required-env declarations.

## Best Possible Solution

Align on one supported optional-env declaration, likely `metadata.openclaw.envVars` entries with `required: false` unless maintainers choose `requires.optionalEnv`; document it, add it to the CLI schema copy, keep scanner/security-prompt behavior consistent, and land regression coverage through #1859 or an equivalent maintainer-approved change.
