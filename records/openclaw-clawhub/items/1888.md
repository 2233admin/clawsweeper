## Summary

Keep this issue open. Current main still applies the shared download rate bucket to package downloads, and the requested per-package quota/trusted treatment for a code-executing plugin is a maintainer/operator policy decision that needs production rate-limit logs plus source/provenance review.

## What This Changes

The issue asks ClawHub maintainers to raise or specialize the anonymous/package download quota for `@openclaw/openviking` after first-time installs hit `429 Rate limit exceeded`.

## Best Possible Solution

A maintainer/operator should inspect production rate-limit logs for the reported endpoint, package, and client IP source; review the posted OpenViking source/provenance if trusted treatment is requested; then either apply an operational quota/trust adjustment or open a focused code follow-up for explicit package/publisher-specific download quota overrides with tests and docs.
