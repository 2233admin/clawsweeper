## Summary

Keep this issue open. The ClawHub-specific policy only allows auto-close for PRs that are certainly implemented on main, and this is an issue. Independently, the requested post-install/update active-session system event is still not implemented in current main: the CLI install/update paths remain package-management flows, while the related web install surface uses explicit copyable prompts rather than automatic session injection.

## Best Possible Solution

Keep this issue open for maintainer product/API triage. The best path is to define a safe, explicit active-session notification contract between ClawHub and OpenClaw, then update install/update or the host integration layer to emit a post-success event that points the agent at the installed SKILL.md. The implementation should include tests and docs for install, update, non-interactive CLI use, and consent boundaries.

## Risks / Open Questions

- The feature crosses from ClawHub CLI package management into OpenClaw host/session orchestration, so it likely needs an explicit session-event API rather than a CLI-only change.
- Automatically prompting an agent to read and act on third-party SKILL.md content after install/update has security and user-consent implications, especially for setup commands, config changes, plugin activation, and service restarts.
