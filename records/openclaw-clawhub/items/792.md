## Summary

Keep open. This is a ClawHub issue, and the repository-specific review policy says issues should not be auto-closed. Current main also does not appear to implement the requested active-session post-install/update system event: the CLI installs or updates skill files and lock metadata, but there is no session-event bridge or documented onboarding trigger after those operations.

## Best Possible Solution

Keep this issue open for maintainer product/API triage. The best path is to define the OpenClaw session-event contract first, then update `packages/clawhub` install/update flows to emit an event after a successful install or update so the active agent can read `SKILL.md` and perform explicit onboarding steps safely.
