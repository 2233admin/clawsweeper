## Summary

Keep open. Current main does not implement post-install/update active-session event injection: the ClawHub CLI install/update paths still download, extract, update metadata, and print success, while the skill page provides user-copyable OpenClaw prompts only. This is also an issue, and the supplied openclaw/clawhub review policy says issues should remain open unless maintainer-handled.

## Best Possible Solution

Keep this open for maintainer product/security/API triage. The best path is to decide whether ClawHub should ever notify an active OpenClaw session after CLI install/update, define an explicit user-consented contract if yes, then implement that contract with CLI/session tests, docs, and security-boundary coverage. Maintainers may also decide the current inspectable copyable-prompt UX is the intended boundary and document that decision.
