## Summary

Repository policy for openclaw/clawhub does not allow automatic closure of issues, and current main still does not implement the requested post-install/update active-session system event. The CLI install/update paths remain package-management operations that download/extract/update metadata and print status, while the web install surface provides an explicit copy-paste OpenClaw prompt rather than automatic session injection.

## Best Possible Solution

Keep this issue open for maintainer product/API triage. The best path is to define a safe, explicit active-session notification contract between ClawHub and OpenClaw, then implement post-success install/update signaling with tests and docs covering install, update, non-interactive CLI use, and user-consent/security boundaries.

## Risks / Open Questions

- The requested behavior crosses from ClawHub package management into OpenClaw host/session control, so it likely needs a clearly defined session notification or plugin API rather than a CLI-only change.
- Automatically prompting an agent to read and act on third-party SKILL.md content after install/update has user-consent and security implications around commands, config edits, plugin activation, and restarts.
