## Summary

Keep open. Current main does not implement post-install/update active-session event injection, and ClawHub cleanup policy keeps issues open for maintainer handling. The request is concrete, but it crosses a product and security boundary around letting newly installed or updated skill content influence an active OpenClaw session.

## What This Changes

The issue requests that ClawHub inject a system event into an active OpenClaw session after `clawhub install` or `clawhub update` so the agent reads the installed skill's `SKILL.md` and performs onboarding/setup steps.

## Best Possible Solution

Maintainers should explicitly define the supported install/update boundary: either add a user-consented OpenClaw session handoff contract with focused CLI/session tests and docs, or document the inspectable copy-prompt flow as the intended setup path.
