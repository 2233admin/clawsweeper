## Summary

Keep this issue open. Repository policy for openclaw/clawhub only allows automatic closure for pull requests that are certainly implemented on main, and this item is an issue. Current main also still lacks the requested automatic post-install/update active-session system event; the CLI remains a package install/update flow, while the web UI provides an explicit copy-paste OpenClaw prompt flow instead.

## Best Possible Solution

Keep this issue open for maintainer product/API triage. The best path is to define a safe, explicit active-session notification contract between ClawHub and OpenClaw, then implement post-success install/update signaling with tests and docs covering install, update, non-interactive CLI use, and user-consent boundaries.
