## Summary

Keep open. The current main branch addresses much of the original rate-limit complaint with auth-aware buckets, higher per-minute limits, standard and legacy headers, Retry-After handling, CLI error guidance, docs, and tests. However, this is an open ClawHub issue, and the repository cleanup policy says only pull requests may be auto-closed when implemented on main. There are also still useful maintainer follow-ups: the static OpenAPI description advertises older limits, and related anonymous install/download 429 reports remain open.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify production anonymous install/download IP attribution, fix or document any proxy/header failure mode, update public/api/v1/openapi.json to match the implementation, and then consolidate with #1410, #390, or #654 if maintainers choose a narrower canonical thread.
