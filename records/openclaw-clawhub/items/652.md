## Summary

Keep this issue open. The ClawHub cleanup policy for this run says issues are not eligible for auto-close, and current main still does not carry the publishing GitHub user's identity into the skill backup commit author metadata. The previous ClawSweeper review remains accurate against current main.

## Best Possible Solution

Keep this issue open and implement future-only commit attribution in the GitHub skill backup path. Carry the publishing user's GitHub identity through both publish-time and bulk sync backups, construct a privacy-safe Git author object, choose explicit committer behavior, include those fields in both GitHub create-commit payloads, and add focused tests for the emitted payload.

- Existing bot-authored backup history should not be rewritten unless maintainers explicitly choose a separate history rewrite plan.
