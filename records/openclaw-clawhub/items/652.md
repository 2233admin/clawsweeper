## Summary

Keep this issue open. The ClawHub cleanup policy for this review keeps issues open, and current main still does not carry the publishing GitHub user's identity into the skill backup commit author metadata. The existing source still passes only owner/display/version/file fields through both publish-time and bulk backup paths, and both GitHub create-commit payloads omit author and committer.

## Best Possible Solution

Keep this issue open and implement future-only commit attribution in the GitHub skill backup path. Carry the publishing user's GitHub identity from publish-time and from skillVersions.createdBy during bulk sync, construct a privacy-safe Git author object, choose explicit committer behavior, include those fields in both create-commit payloads, and add focused tests for the emitted GitHub API payloads.
