## Summary

Keep this issue open. The repository policy for ClawHub issues requires keeping issues open, and current main still does not carry the publishing user's GitHub author metadata into skill backup commits.

## Best Possible Solution

Keep this issue open. Implement future-only commit attribution in the GitHub backup path by carrying the publishing user's GitHub identity through the publish and bulk sync paths, constructing a privacy-safe Git commit author object, adding author/committer fields to the GitHub commit payloads in convex/lib/githubBackup.ts, and adding focused tests that assert the payload includes the intended author metadata.

## Risks / Open Questions

- Author attribution should be privacy-safe; using stored email directly could leak a private address. A GitHub numeric-id noreply address or another explicitly safe source would be safer.
- Maintainers need to decide committer behavior explicitly. The GitHub App/bot may still be the committer while the publishing user is the author.
- Existing bot-authored backup history should remain unchanged unless maintainers choose a separate history-rewrite plan.
