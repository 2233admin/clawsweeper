## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only permits auto-closing ClawHub PRs that current main definitely implements. Substantively, the live Rotifer successor skill is still flagged suspicious from VT and LLM signals, current main provides staff override tooling and a narrow VT Code Insight calibration rather than an already-applied clearing decision, and the license request conflicts with ClawHub’s current platform-wide MIT-0 policy.

## What This Changes

The reporter asks ClawHub to clear a suspicious moderation flag for the Rotifer npx-based MCP skill, consider scanner threshold changes for transparent pinned npx MCP skills, and check why the skill license displays as MIT-0 instead of Apache-2.0.

## Best Possible Solution

A moderator should review the current Rotifer bundle and upstream @rotifer/mcp-server artifact, then apply the existing audited manual clean override only if the package/source/integrity/runtime behavior checks out. If maintainers want systemic relief, add a narrowly tested scanner-policy rule for transparent pinned npx MCP skills; handle per-skill license support as a separate product/legal decision or keep the MIT-0 policy unchanged.
