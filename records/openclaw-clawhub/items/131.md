## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy only permits automated closure for pull requests that current main certainly implements. The current source also still supports the reported disappearing-after-update path: publishing an update can apply a hidden moderation status to an existing skill, and public search/listing conversion excludes hidden skills.

## Best Possible Solution

Keep this issue open. Fix the publish update path so a clean new version of an already-active skill preserves public search/list/detail visibility while review runs, except for malicious uploads, quality quarantine, removed/manual moderation locks, or publishers under moderation. Add regression coverage for active skill updates, owner-visible moderation state, and slug ownership preservation; coordinate with open #134 and decide whether to revive or replace the closed-unmerged #274 approach.
