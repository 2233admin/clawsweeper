## Summary

Keep open. The ClawHub cleanup policy for this review keeps issues open, and current main still lacks the requested owner selector in REST/CLI skill publishing. The web publish action can resolve `ownerHandle`, but `/api/v1/skills` and `clawhub skill publish` do not carry an owner target; open PR #1840 is already the paired implementation candidate with closing syntax.

## What This Changes

Allow authenticated REST and CLI skill publishing to target an org publisher so org members with publisher role can publish new versions of org-owned skills from CI.

## Best Possible Solution

Review and land PR #1840 or an equivalent focused fix that threads `ownerHandle`/`--owner` through the shared publish schema, CLI multipart payload, REST JSON and multipart parsing, resolves the target publisher server-side with at least publisher role, passes `ownerPublisherId` into `publishVersionForUser`, and adds regression coverage for schema parsing, CLI payload generation, and both REST publish formats.
