## Summary

Keep open. Repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on current main, and this item is an issue. Substantively, current main still lacks an owner target in the REST/CLI skill publish path, while open PR #1840 is the active implementation candidate.

## Best Possible Solution

Keep this issue open and review/land PR #1840 or an equivalent fix. The fix should add ownerHandle/--owner support for skill publishing, preserve it for JSON and multipart REST payloads, resolve the target publisher server-side with at least publisher role, pass ownerPublisherId into publishVersionForUser, and cover schema parsing, CLI payload generation, and both REST publish formats.
