## Summary

Keep open. This is a ClawHub issue, and the repository policy only permits auto-closing ClawHub PRs that are certainly implemented on main. Substantively, current main still leaves the reported REST/CLI path unresolved: the web action accepts ownerHandle and resolves a publisher target, but the v1 REST handlers, shared publish schema/parser, and skill CLI publish commands do not preserve an org owner target. Open PR #1840 is the active implementation candidate and should be reviewed or replaced before closing this issue.

## Best Possible Solution

Keep this issue open and review/land #1840 or an equivalent fix. The implementation should add ownerHandle/--owner support for skill publishing, preserve it for JSON and multipart REST payloads, resolve the target publisher server-side with at least publisher role, pass ownerPublisherId into publishVersionForUser, and add regression coverage for schema parsing, CLI payload generation, and both REST publish formats.
