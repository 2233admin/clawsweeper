## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main definitely implements. Current main still lacks an owner target in the skill REST/CLI publish path, and open PR #1840 is the active implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #1840. The fix should add ownerHandle/--owner support to skill publishing, preserve it for JSON and multipart REST payloads, resolve the requested publisher server-side with at least publisher role, pass ownerPublisherId into publishVersionForUser, and cover schema parsing, CLI payload generation, and both REST publish formats.
