## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still renders OpenClaw scanner failures as a bare `Error`, suppresses the stored LLM error summary in the scan panel, and lacks author-facing docs or UI explaining what to do next.

## Best Possible Solution

Keep this issue open for maintainer triage. Add scan-page and public docs guidance for OpenClaw `Error`, expose a safe error explanation or stored summary to authors, document whether authors should wait, change the skill, or request a rescan, and decide a conservative moderation policy that separates scanner-side LLM failures from actual suspicious evidence. Coordinate with the broader false-positive/rescan workflow tracked by #669.
