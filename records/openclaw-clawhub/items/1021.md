## Summary

Keep open. Current main has added scanner detail and owner rescan surfaces, but ClawScan/OpenClaw `Error` is still presented without clear author-facing semantics or remediation, the main skill panel hides stored error summaries, docs do not explain the status or rescan path, and the scanner-error moderation behavior needs maintainer/security judgment alongside the still-open related false-positive thread.

## What This Changes

The issue asks ClawHub to explain ClawScan/OpenClaw `Error` statuses, give skill authors remediation and rescan guidance, and avoid treating scanner-side failures as suspicious skill findings.

## Best Possible Solution

Define a safe ClawScan/OpenClaw `Error` contract, surface a sanitized explanation and next steps on the skill page and scanner detail page, document web/API/CLI rescan plus appeal paths, and make an explicit moderation rule for scanner-side failures when static findings exist, coordinated with #669.
