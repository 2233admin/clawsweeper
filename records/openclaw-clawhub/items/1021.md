## Summary

Keep open. This is a ClawHub issue, and the repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, current main still renders OpenClaw scanner failures as a bare Error, hides stored LLM failure summaries in the scan panel, preserves API behavior where a clean scanner plus LLM error yields overall security status error, and lacks author-facing docs explaining what to do next. Related false-positive/rescan work in #669 is still open.

## Best Possible Solution

Keep this issue open for maintainer triage. Add scan-page and public docs guidance explaining what OpenClaw Error means, expose a safe scanner failure explanation or stored summary to authors, document whether authors should wait, change the skill, or request a rescan, and define a moderation rule that distinguishes scanner-side failures from actual suspicious evidence. Coordinate the remediation path with #669.
