## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, current main has partial scanner detail and rescan support, but ClawScan/OpenClaw errors still render as a terse Error, the primary skill security panel hides stored error summaries for error states, docs do not explain the Error state or author remediation path, and moderation still needs explicit security/product judgment for scanner-side failures versus suspicious static findings.

## Best Possible Solution

Keep this issue open for maintainer/security triage. The best path is to add safe author-facing ClawScan Error semantics in the skill security UI and public docs, surface a sanitized stored error summary or next-step guidance when appropriate, document owner and CLI rescan paths, and make an explicit moderation decision for scanner-side LLM failures versus static suspicious findings. Coordinate that decision with #669 so false-positive and rescan handling stay consistent.
