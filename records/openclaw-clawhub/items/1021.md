## Summary

Keep open. Repository policy for ClawHub keeps issues open, and current main still does not make ClawScan/OpenClaw `Error` actionable: the UI maps it to a bare error badge, the primary panel hides stored error summaries, docs do not define the status or remediation path, and the remaining moderation/install-blocking semantics need maintainer/security judgment. Rescan support and a rescan-guidance workflow partially help, but they do not resolve this issue or the related open false-positive thread #669.

## What This Changes

Request documentation and UI/API guidance explaining ClawScan/OpenClaw `Error`, author remediation and rescan options, and the moderation policy when scanner-side failures coexist with static findings.

## Best Possible Solution

Keep this open for maintainer/security triage. Define safe author-facing ClawScan `Error` semantics, add sanitized guidance to the skill security summary and scanner detail page, document web/API/CLI rescan and review paths, and make an explicit moderation policy for scanner-side failures when static findings exist. Coordinate the outcome with #669 so false-positive and rescan handling stay consistent.
