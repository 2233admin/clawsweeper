## Summary

Keep open. The repository-specific cleanup policy says ClawHub issues should stay open, and the current code still does not fully implement the requested author-facing explanation/action path for an OpenClaw scanner error. Current main renders the error as a bare red status, suppresses LLM detail for error states, and still lets static suspicious findings drive the moderation result unless both external scanners are clean and the static code is allowlisted.

## Best Possible Solution

Keep this issue open for maintainer triage. Add explicit scan-page and docs guidance for OpenClaw `Error`, including likely causes, whether it is scanner-side, whether the author should wait/change anything, and how to request or trigger a rescan. In code, treat scanner-side LLM errors separately from suspicious verdicts while still surfacing static findings and preserving conservative blocking for malicious/non-allowlisted evidence. Then verify/backfill the affected skill versions once the policy is decided.
