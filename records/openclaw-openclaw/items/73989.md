## Summary

Current main still lacks the reconnect-grace behavior this PR adds: disconnected running channels are marked unhealthy immediately after startup connect grace, and health-monitor/readiness do not consume typed `lastDisconnect.at`. The PR remains the active implementation candidate based on #45712, with maintainer review and changed-gate validation still needed.

## What This Changes

The PR adds reconnect-grace timing to gateway channel health policy, health monitor, and readiness checks, with regression tests and a changelog entry for recently disconnected running channels.
