## Summary

Keep this PR open. Current main still has the mid-turn transcript reset behavior, and the PR fixes the central predicate, but the proposed patch adds a transcript file-mode regression in the archive-before-clear path and a Windows-hostile test fixture path before it is mergeable.

## What This Changes

The PR narrows session initialization resets to header-only transcripts, archives before clearing, adds regression coverage for user-message-only mid-turn transcripts, and adds a shell verification wrapper.

## Best Possible Solution

Amend the existing PR so the hasAnyMessage predicate remains, the archive/reset path preserves or recreates transcripts with private 0o600 permissions, the fixture uses os.tmpdir() with path.join(), and the targeted test plus changed gate are run before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve transcript file mode after archiving:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: The new archive step renames the existing private transcript before this write runs. Because the original path no longer exists, fs.writeFile(..., "utf-8") creates a replacement using the process default permissions rather than the 0o600 mode used by transcript creation paths, so later transcript content can land in a less-private file. Recreate the file with mode 0o600 or restore the archived file's mode.
  - confidence: 0.87
- **[P2] Use the OS temp directory for fixtures:** `src/agents/pi-embedded-runner/session-manager-init.test.ts:10-11`
  - body: The new test hardcodes /tmp as the mkdtemp parent and then concatenates paths. Windows CI may not have that parent, so the suite can fail before it exercises the session-preservation behavior. Use os.tmpdir() and path.join() for the prefix and session file path.
  - confidence: 0.91
