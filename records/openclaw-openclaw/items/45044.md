## Summary

Keep this PR open. Current main still uses the broad `!hasAssistant` reset guard in `prepareSessionManagerForRun`, so the reported mid-turn restart data-loss path is not implemented on main. The PR is aimed at a real bug supported by the discussion, but it still needs a small portability fix before it should merge.

## What This Changes

The PR narrows session initialization reset behavior to header-only files, archives before clearing, adds regression coverage for session file states, and adds a shell wrapper for the new test.

## Best Possible Solution

Land this fix or a maintainer-owned successor after making the regression test portable with `os.tmpdir()`/`path.join()` and using the repo test wrapper. The end state should preserve any existing message-bearing transcript, reset only true header-only pre-created files, and keep focused coverage for `[header]`, `[header,user]`, and `[header,user,assistant]` session states.
