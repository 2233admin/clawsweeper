## Summary

Keep open. Current main still has no native `openclaw tui --agent <id>` startup selector: the TUI CLI registers `--session` but not `--agent`, `TuiOptions` cannot carry an agent id, and startup selection still comes from a canonical agent-scoped session key, cwd workspace inference, or the default agent. The #40176 workspace-isolation work and the post-launch `/agent <id>` command are useful adjacent workarounds, but they do not satisfy the generic-directory startup flag requested here.

## Best Possible Solution

Add a native `openclaw tui --agent <id>` selector in the TUI CLI and typed TUI options, validate or reconcile it against configured/available agents, and make startup precedence explicit. A conservative design is to let `--agent` override cwd inference for bare sessions, allow matching `--session agent:<same>:...`, and reject conflicting `--agent` plus `--session agent:<other>:...`. Update focused TUI tests and the public TUI docs, with a changelog entry crediting the reporter/context contributors.
