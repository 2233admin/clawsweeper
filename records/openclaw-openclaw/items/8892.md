## Summary

Keep this open. Current main still has no native `openclaw tui --agent <id>` startup selector. It supports full agent-scoped session keys, workspace-directory inference, and post-launch `/agent <id>`, but those do not satisfy the requested generic-directory startup flag. The prior maintainer/Codex comment and related #19247/#40176 context still match current main.

## Best Possible Solution

Add a native `--agent <id>` selector for `openclaw tui`: register and validate it in `src/cli/tui-cli.ts`, extend `TuiOptions`/`runTui` with a typed startup agent id, define precedence against canonical `--session agent:<id>:...` and cwd workspace inference, then update focused TUI tests plus `docs/cli/tui.md` and `docs/web/tui.md`. Keep this issue open as the canonical user-facing request while that work is designed and reviewed.
