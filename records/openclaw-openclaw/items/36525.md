## Summary

Keep this issue open. Current main still implements the existing sandbox workspace contract: `workspaceAccess: "rw"` selects the agent workspace for `/workspace`, non-`rw` modes use the sandbox workspace but keep managed writes read-only, and split non-`none` layouts can still expose the agent workspace at `/agent`. The open #46300 PR is a narrower writable-`none` fix and does not supersede this broader sandbox semantics refactor.

## Best Possible Solution

Keep this issue open for explicit sandbox contract review by sandbox/security maintainers. If maintainers accept the proposed semantics, implement it as a coordinated change across `ensureSandboxWorkspaceLayout`, Docker/browser/SSH/OpenShell mount builders, FS bridge write gates, embedded sandbox info, memory/bootstrap behavior, tests, config/status surfaces, and public sandbox docs. Treat #37634 and #46300 as narrower writable-`none` work, not replacements for this broader contract decision.
