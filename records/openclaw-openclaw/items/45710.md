## Summary

Keep this PR open. Current main does not implement the proposed VSCode debugging path: there is no checked-in VSCode launch/tasks config, no `OUTPUT_SOURCE_MAPS`/source-map toggle in `tsdown.config.ts`, no `clean:dist` script, and no VS Code debugging section in `docs/help/debugging.md`. The PR is still potentially useful, but maintainer review is needed around the prelaunch clean/rebuild flow because `run-node` rebuilds a narrower runtime set than the full build pipeline.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrow DX change only after the VSCode launch flow uses a supported source-map build path and preserves generated runtime assets, with docs that clearly distinguish the one-click debug path from manual `OUTPUT_SOURCE_MAPS=1` builds.
