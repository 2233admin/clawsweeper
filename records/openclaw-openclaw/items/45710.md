## Summary

Keep this PR open. Current main still lacks the proposed VS Code launch/tasks configuration, `OUTPUT_SOURCE_MAPS` source-map toggle, `clean:dist` script, and VS Code debugging docs, but the PR's one-click clean launch has a concrete correctness regression because it deletes all of `dist/` and then rebuilds through the narrower `run-node` path that does not recreate required generated assets.

## What This Changes

The PR adds VS Code Gateway debug launch/tasks files, an opt-in tsdown source-map toggle, a `clean:dist` script, and a VS Code debugging section in the debugging docs.

## Best Possible Solution

Land the DX improvement only after the debug launch uses a supported source-map build path that preserves or recreates required generated assets, with docs that clearly separate the automatic one-click launch from manual `OUTPUT_SOURCE_MAPS=1 pnpm build` workflows.

## Reproduction Assessment

Yes for the introduced regression: use the PR's `Rebuild and Debug Gateway` path, then run `/export-session` or request A2UI during that debug session. Source inspection shows the launch deletes `dist/`, then `run-node` rebuilds without the full-build export-template and A2UI copy steps.

## Solution Assessment

No. The opt-in source-map toggle is a plausible piece of the fix, but deleting the entire `dist/` tree before a narrower `run-node` rebuild is not the narrowest maintainable launch path; an asset-preserving clean, full debug build profile, or supported watcher/debug build path is safer.
