## Summary

Keep this issue open. Current main still matches the native `/think` menu gap: Ollama only exposes non-off thinking levels when catalog `reasoning === true` reaches the provider profile, but the native command-choice pipeline still resolves `/think` choices with provider/model only. The provided GitHub context also shows #73568 is open, unmerged, and uses closing syntax for this issue, so the right maintainer path is to review/land or close that implementation PR rather than closing the issue early.

## Best Possible Solution

Keep this issue open and review #73568 or an equivalent fix. The best code path is to add catalog support to the generic command argument choice/menu context, pass the runtime model catalog from Telegram native menus and any other native menu surfaces, and add a focused regression test proving an Ollama catalog entry with `reasoning: true` exposes `off, low, medium, high, max` in the `/think` menu without hardcoding Ollama behavior into core.
