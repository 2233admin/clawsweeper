## Summary

Keep this issue open. Current main still has session/agent sandbox modes, sandbox-only tool allow/deny filtering, browser target selection, and elevated exec escape behavior, but not the requested per-tool execution placement policy that can force web_fetch or browser into a sandbox while keeping exec, message, cron, and gateway host-routed in the same session. The existing discussion also explicitly kept this open, and merged Docker work only partially addressed/unblocked it.

## What This Changes

Add configuration for per-tool sandbox execution independent of session sandbox mode, so selected tools such as web_fetch and browser can always run isolated while trusted tools stay host-routed.

## Best Possible Solution

Add a documented, schema-backed core tool placement contract that can sandbox selected tools such as web_fetch and browser while leaving selected trusted tools host-routed in the same session, with regression tests proving the behavior is separate from tools.sandbox.tools filtering and tools.elevated exec behavior.
