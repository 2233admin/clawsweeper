## Summary

Keep this issue open. Current main still has session/agent-level sandbox selection plus sandbox-only tool availability filtering, not the requested per-tool execution placement policy that can force `web_fetch` or `browser` into a sandbox while leaving `exec`, `message`, `cron`, and `gateway` on host in the same run. The prior maintainer review on April 26, 2026 also explicitly kept this open, and no relevant sandbox/tool-routing files changed between that review anchor and current HEAD.

## Best Possible Solution

Keep this open as a core sandbox/tool-routing enhancement. The best path is a documented, schema-backed per-tool execution placement policy that can route selected tools such as `web_fetch` and `browser` through a sandbox backend while selected host tools such as `exec`, `message`, `cron`, and `gateway` remain on host in the same session, with regression tests that distinguish placement from `tools.sandbox.tools` filtering and `tools.elevated` exec escape behavior.
