## Summary

Keep open. Current main has useful partial controls for isolated cron jobs, hook-triggered agent runs, lightweight context, model/fallback selection, timeout, and tool allow-lists, but it still does not implement the requested first-class scoped/script-limited mode with an admin script registry, dry-run cost preview, per-job token/spend ceilings, daily budget aborts, or planned-vs-actual run logs. The existing maintainer/Codex comment also explicitly kept this open and treated #38248 as only a narrower cost-guard component.

## What This Changes

Request to add a first-class scoped/script-limited agent mode with whitelisted scripts, cron-first execution, token and spend caps, dry-run previews, minimal context, and structured run logs.

## Best Possible Solution

Keep this open as the broader scoped automation product/API tracker. The best path is a designed native mode across cron payload/schema, hook mappings, CLI/UI/docs, isolated-run accounting, and tests: admin-defined script-only execution, optional dry-run/cost preview, per-job token/spend caps, daily ceilings, graceful budget aborts, and structured planned-vs-actual run logs. Coordinate with #38248 so the cost guard is shared rather than duplicated.
