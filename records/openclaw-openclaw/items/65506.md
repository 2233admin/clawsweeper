## Summary

Current main still only partially addresses the QA harness problem. The suite has normal-path gateway cleanup and qa-channel readiness probing, but the host suite path still starts detached gateway children without suite-level SIGINT/SIGTERM ownership, fixed restart sleeps remain, and qa-channel config is still constructed before the transport readiness check. The linked implementation PR #65511 is closed unmerged, so there is no shipped or merged fix to close against.

## What This Changes

The issue requests hardening QA harness shutdown, gateway restart readiness, qa-channel config emission, and temp runtime cleanup so interrupted QA runs do not pollute live incidents.

## Best Possible Solution

Land a focused qa-lab fix that owns child gateway lifetimes across host interruption, safely reaps stale QA suite temp roots, replaces restart sleeps with concrete gateway/reload/transport readiness checks, and only activates qa-channel for a run after the QA bus is confirmed healthy while preserving keep-temp and sanitized artifact behavior.

## Repair Work Prompt

Fix the QA harness orphan gateway leakage and restart readiness gaps for this issue. Work in qa-lab/qa-channel boundaries only. Current main starts detached gateway children from `startQaGatewayChild`, cleans temp roots only through normal `gateway.stop()`/startup-failure paths, leaves `runQaSuiteCommand` without SIGINT/SIGTERM ownership, still uses sleep-based restart settling, and builds qa-channel config before transport readiness is proven. Add explicit host-suite signal/abort cleanup for gateway children, temp roots, lab/mock/web sessions, and registered harnesses; add a conservative stale `openclaw-qa-suite-*` root cleanup strategy that skips active or `OPENCLAW_QA_KEEP_TEMP=1` preserved roots; replace `waitForConfigRestartSettle` and the post-transport fixed sleep with concrete gateway/reload/transport readiness handshakes; and gate qa-channel config emission or activation on a healthy QA bus. Preserve existing normal cleanup, keep-temp behavior, sanitized debug artifact preservation, config hash/retry-after retry logic, and live transport paths that intentionally omit qa-channel. Add focused regression coverage for interrupted cleanup, temp-root removal/preservation, restart readiness, and config generation when the QA transport is unavailable. No changelog entry is expected unless the user-facing QA command behavior is documented as changed.
