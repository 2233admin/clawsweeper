## Summary

Keep open. Current main still only fully exposes the requested per-message metadata in the Control UI; channel replies have a partial opt-in usage footer, iOS preserves usage data without rendering it, and Android drops usage before the chat bubble layer can display it.

## What This Changes

Request to show Control UI-style per-message token, cache, cost, context percentage, and model metadata in native mobile chat and optional channel reply footers.

## Best Possible Solution

Add a compact shared usage presentation that reuses existing transcript/run usage data across iOS, Android, and the generic channel /usage footer path, with context percent and model labels included only when the source data is reliable for the displayed assistant reply.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but source inspection gives a high-confidence verification path: Control UI renders the metadata, channels omit model/context percentage, iOS does not render preserved usage, and Android does not preserve usage in ChatMessage.

## Solution Assessment

Is this the best way to solve the issue? Yes: extending the existing responseUsage footer and native chat rendering paths is narrower than adding per-channel transport-specific behavior or a separate mobile telemetry channel.
