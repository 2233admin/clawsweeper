## Summary

Keep open. Current main still only fully satisfies the requested per-message token/cost/context/model metadata in the Control UI; channel replies have an opt-in usage footer but it omits context percentage and model, while native iOS and Android chat views do not render the metadata to users.

## What This Changes

Request to show Control UI-style per-message token, cache, cost, context percentage, and model metadata in native mobile chat and optional messaging-channel reply footers.

## Best Possible Solution

Add a compact shared usage presentation that reuses transcript usage data in native chat and the existing generic channel responseUsage footer, adding context percentage and model labels only where the active context window and message-level usage source are reliable.

## Reproduction Assessment

Not applicable as a feature request; source inspection gives a high-confidence verification path. The current paths show Control UI metadata rendering, partial channel footer formatting, iOS usage preservation without rendering, and Android history parsing that drops usage entirely.

## Solution Assessment

Yes, the maintainable direction is to extend the existing generic /usage footer and native chat rendering/data models rather than adding per-channel transport-specific logic. If context percentage is unavailable, the safer behavior is to show tokens/cache/cost/model without a guessed percentage.
