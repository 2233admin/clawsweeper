## Summary

Keep this issue open. Current main still has the Control UI metadata the reporter wants, and channels have an opt-in `/usage` footer, but the request is not fully implemented: iOS preserves decoded usage without rendering a footer, Android chat models/parsing drop usage entirely, and the shared channel footer still lacks Control UI-style context percentage and model labels.

## Best Possible Solution

Implement the feature in the existing shared surfaces: add compact assistant-message usage footers to the native chat UI, parse usage into Android chat models, and extend the generic channel `/usage` footer with model/context metadata only when it is reliable. Keep per-channel plugins untouched, update token-use/mobile docs, and add focused UI/parser/channel tests.
