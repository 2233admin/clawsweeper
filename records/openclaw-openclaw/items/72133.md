## Summary

Keep open. Current main still leaves the requested visibility only partially implemented: the Control UI has the rich per-message metadata, channel replies have an opt-in footer that lacks model and context percentage, iOS preserves usage data without rendering it, and Android still drops usage while parsing chat history.

## What This Changes

Request to show Control UI-style per-message input/output/cache/cost/context/model metadata in iOS/Android native chat and optional messaging-channel reply footers.

## Best Possible Solution

Add a compact shared usage presentation that reuses transcript usage data in native chat and the existing generic responseUsage channel footer, adding model and context percentage only where the active model/context window data are reliable.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: Control UI renders the metadata, channels render only partial usage, iOS keeps usage off-screen, and Android does not parse usage into its chat model.

## Solution Assessment

Yes. The narrowest maintainable path is to extend the existing shared /usage footer and native chat data/rendering paths rather than adding transport-specific logic per channel.
