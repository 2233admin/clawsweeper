## Summary

Keep open. Current main still has only partial coverage outside the Control UI: channel reply footers can show usage tokens/cost, but not model or context percentage, and the native iOS/Android chat views do not expose the per-message metadata to users.

## What This Changes

Request to show per-message token, cache, cost, model, and context percentage metadata in native mobile chat and optional messaging-channel reply footers.

## Best Possible Solution

Add a shared compact usage presentation for assistant replies, reuse it from the existing channel responseUsage footer and native chat renderers, keep channel display opt-in, and document that context percentage is omitted when the required prompt/context metadata is unavailable.

## Reproduction Assessment

Not applicable as a feature request; source inspection gives a high-confidence verification path. Current main shows full Control UI metadata, partial channel footer formatting, iOS usage preservation without rendering, and Android history parsing that drops usage entirely.

## Solution Assessment

Yes for a future fix: extending the existing /usage footer and native chat message renderers is the narrowest maintainable direction. The safer alternative is to hide model/context fields on any surface where the run or history metadata cannot support them accurately.
