## Summary

Keep open. Current main still only has the full Control UI-style token/cache/cost/context/model display in the web chat renderer; the channel footer remains token/cache/cost-only, iOS preserves usage without rendering it, and Android still drops usage while parsing chat history.

## What This Changes

Request to expose the Control UI’s per-assistant-message token/cache/cost/context-percent/model metadata in native iOS/Android chat and optional messaging-channel reply footers.

## Best Possible Solution

Add a compact shared usage presentation that reuses existing transcript/runtime usage data across native chat and the generic channel responseUsage footer, showing context percent and model only when the source data is reliable for the displayed assistant reply.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but source inspection gives a high-confidence verification path: Control UI renders the data, channel footers omit model/context percent, iOS does not render preserved usage, and Android does not preserve usage.

## Solution Assessment

Is this the best way to solve the issue? Yes: extending the existing responseUsage footer and native chat rendering paths is narrower than per-channel implementations, with context percent guarded by the existing prompt/context accounting semantics.
