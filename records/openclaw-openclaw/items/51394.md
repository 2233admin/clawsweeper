## Summary

Current main still does not implement the requested agent-facing browser tool Network domain. The browser control API and CLI have lower-level request listing and response-body capture, but the browser tool schema and dispatcher do not expose network/requests/responsebody/intercept actions, and true request interception remains undesigned.

## Best Possible Solution

Keep this issue open. Implement a narrow agent-facing browser tool surface in the browser plugin for request listing and response-body capture by reusing the existing control routes, with schema, docs, and tests. Treat true interception as a separate design step using Playwright routing or CDP Fetch/Network with explicit target scoping, safety limits, and security regression coverage.
