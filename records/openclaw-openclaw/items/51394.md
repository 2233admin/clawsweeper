## Summary

Current main still does not implement the requested agent-facing browser Network/CDP request-interception surface. The browser control API and CLI expose passive request listing and response-body capture, but the browser tool schema and dispatcher still have no network/requests/responsebody/intercept action or act kind, and the only Playwright routing found is the navigation SSRF guard, not a user-facing interception API.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-scoped browser plugin design: expose a narrow agent-facing browser tool surface for passive request listing and response-body capture by reusing the existing control routes, with schema/docs/tests and external-content handling; treat true request interception/modification as a separate security-reviewed design using Playwright routing or CDP Fetch/Network with explicit target scoping, limits, and regression coverage.

## Risks / Open Questions
