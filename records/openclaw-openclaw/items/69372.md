## Summary

Current main has not implemented PR #69372's Control UI behavior: protected canvas/a2ui iframe URLs still fall back to raw same-origin paths when no scoped canvas host is available, and GatewayBrowserClient has no Control UI-side capability refresh path. The PR is still relevant, but it should not be closed as implemented or superseded because its proposed timer-based websocket reconnect also has a credible unresolved review risk: reconnecting can clear in-flight chat run state.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. The best fix is likely a Control UI-safe capability refresh path that updates hello.canvasHostUrl without tearing down the websocket, or an idle-gated refresh if transport reconnect remains necessary, plus fail-closed iframe URL behavior and focused tests for canvas and a2ui paths.
