## Summary

Keep this PR open. Current main still has the reported protected canvas/a2ui URL fallback and no Control UI-side capability refresh, but the PR's timed WebSocket reconnect can interrupt pending RPCs and reset active chat run state; it needs maintainer-guided rework rather than cleanup closure.

## What This Changes

This PR changes Control UI canvas iframe URL resolution to fail closed for protected canvas/a2ui paths without a scoped capability host, adds a timed browser WebSocket reconnect before the canvas capability TTL expires, and updates UI/gateway unit tests.

## Best Possible Solution

Keep this PR open for maintainer-guided rework. Preserve the fail-closed protected iframe URL behavior, replace the scheduled WebSocket reconnect with an idle-safe or non-teardown capability refresh path, and add tests covering protected canvas/a2ui fail-closed behavior, scoped canvas/a2ui rewrite, refresh without pending RPC rejection, and preservation of active chat run state.
