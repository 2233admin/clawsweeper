## Summary

Keep this PR open. Current main still allows raw protected Control UI canvas iframe URLs when no scoped host is available, and the browser client still has no Control UI-side capability refresh. The PR is useful but not ready as-is because its timer-driven WebSocket reconnect can reject pending RPCs and reset active chat run state; current main already has a non-teardown `node.canvas.capability.refresh` seam that looks like the safer direction.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The likely best path is to preserve the fail-closed protected URL behavior, but replace the forced WebSocket reconnect with an idle-safe refresh or a Control UI use of the existing `node.canvas.capability.refresh` seam, with tests proving no raw protected iframe URL is emitted and active Control UI chat runs are not reset by capability refresh.

- Canvas capability URL handling is auth-sensitive; a rework needs targeted coverage for `/__openclaw__/canvas`, `/__openclaw__/a2ui`, scoped and unscoped hosts, and active-run continuity.
- The PR does not appear to introduce supply-chain or CI risk, but the transport/auth behavior itself needs maintainer review before merge.
