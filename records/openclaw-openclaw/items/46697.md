## Summary

Keep this PR open. Current main still posts the A2A announce-step text to the target channel/thread, not the full target reply this PR is trying to deliver. The PR addresses a real remaining behavior gap, but its current patch should not be merged as-is because the new direct send path can publish externally before ANNOUNCE_SKIP suppression and it is stale against current main internals.

## Best Possible Solution

Keep this PR open, but require a rework against current main: preserve sessionsSendA2ADeps.callGateway and resolveNestedAgentLaneForSession, deliver the post-ping-pong latestReply only after the intended suppression policy is honored, avoid unnecessary announce runs when their output cannot be delivered, and add targeted tests for no-ping-pong full-reply delivery, ping-pong final-reply delivery, ANNOUNCE_SKIP suppression, and threadId/accountId passthrough.
