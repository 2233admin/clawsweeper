## Summary

Keep this PR open. Current main still posts the A2A announce text rather than the full target reply, so the requested behavior is not already implemented, but the proposed patch changes external delivery semantics in a way that can bypass announce suppression and mislabels announce provenance.

## What This Changes

This PR changes `src/agents/tools/sessions-send-tool.a2a.ts` to send the post-ping-pong `latestReply` directly to the announce target, suppress duplicate announce sends, run the announce step in the requester session, and include `threadId` on the fallback send params.

## Best Possible Solution

Rework A2A delivery on current main so the visible thread receives the intended post-ping-pong final content only under an explicit, documented suppression policy; preserve `accountId`/`threadId`, injected dependencies, and per-session lanes, with targeted tests for full-reply delivery, ping-pong final delivery, `ANNOUNCE_SKIP`, fallback announce behavior, and thread/account passthrough.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor ANNOUNCE_SKIP before external delivery:** `src/agents/tools/sessions-send-tool.a2a.ts:106-117`
  - body: The new full-reply send posts `latestReply` to the announce target before the announce step can return `ANNOUNCE_SKIP`. That regresses the existing suppression contract from “do not post” to “post the full reply anyway,” which can leak content into a linked channel or thread that the announce step intentionally wanted to keep silent.
  - confidence: 0.9
- **[P2] Use the target channel for announce provenance:** `src/agents/tools/sessions-send-tool.a2a.ts:148-149`
  - body: This announce step marks `sourceSessionKey` as the target session but keeps `sourceChannel` as the requester channel. When requester and target channels differ, downstream provenance labels the message as coming from the wrong channel, which can affect channel-specific formatting or routing in the announce response.
  - confidence: 0.78
