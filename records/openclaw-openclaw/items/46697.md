## Summary

Keep this PR open. Current main still delivers the A2A announce-step text to the visible target channel/thread, not the full target reply the PR is trying to deliver. The PR targets a real remaining behavior gap, but the current patch still needs maintainer rework because it changes external delivery semantics before ANNOUNCE_SKIP suppression and does not add the regression coverage needed for this path.

## Best Possible Solution

Keep this PR open and rework it against current main: preserve the existing A2A dependency seams, decide the intended ANNOUNCE_SKIP semantics before any visible send, deliver the post-ping-pong latestReply only when visible delivery is allowed, avoid unnecessary announce runs when their output cannot be delivered, and add focused regression tests for the delivery matrix.
