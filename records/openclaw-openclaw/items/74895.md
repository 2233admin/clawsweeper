## Summary

Close this PR as implemented on current main: the linked read-only Telegram status/accessor problem is already fixed by a config-only accessor path, while this PR would regress the runtime token resolver by masking unresolved non-env SecretRefs instead of failing closed.

## What This Changes

The PR removes Telegram's strict non-env SecretRef runtime resolution check so unresolved file/vault bot token refs return configured_unavailable.

## Best Possible Solution

Keep the current-main split: read-only Telegram status and metadata paths should use config-only/inspection helpers, while runtime Telegram token resolution should continue to fail closed for unresolved non-env SecretRefs.

## Reproduction Assessment

Yes. The PR regression is covered by the existing Telegram token test for unresolved non-env botToken SecretRefs, and the linked status/accessor behavior is covered by the shared Telegram config accessor test.

## Solution Assessment

No. This PR is not the best solution because it changes the runtime credential resolver to address a read-only inspection symptom; current main already uses the narrower accessor/inspection path without weakening runtime strictness.
