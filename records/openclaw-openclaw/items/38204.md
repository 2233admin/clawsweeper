## Summary

Current main still does not implement native Grok 4.20 multi-agent support. The bundled xAI plugin supports standard Grok 4.20 reasoning/non-reasoning IDs on the xAI Responses path, but the current catalog, dynamic resolver, docs, and regression tests explicitly exclude multi-agent model IDs. The related xAI Responses PR improved standard Grok/x_search support, but did not cover this beta multi-agent surface.

## What This Changes

Request native support for the xAI Grok 4.20 multi-agent model/endpoint in OpenClaw, beyond the current single-agent Grok 4.20 reasoning workaround.

## Best Possible Solution

Keep this open for xAI-provider maintainer judgment. If bundled support is desired, implement it in `extensions/xai` as provider-specific Grok 4.20 multi-agent Responses support with current alias handling, explicit `reasoning.effort`/agent-count semantics, unsupported-parameter filtering, clear built-in-tool behavior, docs, focused regression tests, and live xAI proof when credentials are available. If maintainers decide this beta multi-agent surface belongs outside the bundled provider, close only after explicit ClawHub/plugin guidance.
