## Summary

Current main still does not implement native Grok 4.20 multi-agent support. The bundled xAI plugin supports standard Grok 4.20 reasoning/non-reasoning IDs, but the catalog, dynamic resolver, modern-model classifier, provider wiring, docs, and regression tests still explicitly exclude `multi-agent` model IDs. The discussion has moved beyond an informational workaround into a request for native multi-agent endpoint support, so this should stay open for xAI-provider maintainer judgment.

## Best Possible Solution

Keep this issue open for xAI-provider maintainer judgment. If bundled support is desired, implement it inside `extensions/xai` as a provider-specific multi-agent path for canonical `grok-4.20-multi-agent` IDs and relevant legacy aliases, with the correct xAI endpoint/request shaping, model metadata/auth behavior, docs, and regression tests. If maintainers decide this beta/specialized surface should not be bundled, resolve that explicitly with provider-plugin or ClawHub guidance rather than closing as already implemented.
