## Summary

Current main still does not support per-Telegram-group or per-topic session reset configuration. Telegram group/topic config types and strict schemas do not accept a nested session.reset block, and runtime reset policy resolution only receives global/type/channel inputs. The linked implementation PR #8134 is closed unmerged, while #20888 is broader and closed rather than a canonical tracker for this exact feature.

## Best Possible Solution

Keep this issue open and implement an additive, account-aware Telegram group/topic reset override, or a generic conversation-scoped reset resolver if maintainers prefer the capability across plugins. The fix should define precedence as topic/group override over channel override over type override over global reset, cover multi-account Telegram cases, update docs and generated config metadata, and use #8134 as useful prior art without repeating its group-id-only lookup risk.
