## Summary

Keep this PR open. Current main still does not implement the intended Feishu workspace artifact SDK exports, and the branch would need a current-main rewrite before it could be considered mergeable. The current SDK docs and guardrails point toward generic plugin-neutral surfaces rather than a new channel-branded `openclaw/plugin-sdk/feishu` subpath, and the review discussion raised real path-containment concerns that should be resolved in any replacement implementation.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but replace the branch if the product need remains. The best path is a plugin-neutral workspace artifact helper or documented recipe built from `api.runtime.agent.resolveAgentWorkspaceDir` plus `openclaw/plugin-sdk/file-access-runtime`, with portable filename normalization, root-scoped writes, SDK entrypoint classification, docs, and contract tests updated together.
