## Summary

Keep this PR open. At the inspected main checkout 2ccdbc7dd93bfcb08e100a479af79d8d8a211d97, the macOS node first-use TLS path still saves and accepts a certificate before normal trust evaluation, so the PR’s intended security fix is not implemented on main. The PR is also paired with open issue #50642 by the same contributor and is an implementation candidate, not a cleanup-close candidate.

## Best Possible Solution

Keep the PR open for maintainer security review. The best path is to land a hardened version of the `GatewayTLSPinningSession` change, with regression coverage where feasible, and explicitly handle the supported private-TLS enrollment path, likely through an explicit fingerprint or prompt flow rather than silent macOS node TOFU. After the fix lands, close the linked security issue as implemented.
