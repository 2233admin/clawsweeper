## Summary

Keep this issue open. Current main has reduced one major trigger by making gateway-start QMD refresh opt-in and using transient CLI managers, but the remaining gateway-process versus CLI-process contention is not fully coordinated: QMD update serialization is still in-process, search only waits for the same manager instance briefly, and tests still model SQLite busy during QMD search as a failure/fallback path rather than preventing it.

## What This Changes

The issue asks for `openclaw memory search` to avoid intermittent QMD SQLite lock contention while gateway/background QMD maintenance is active.

## Best Possible Solution

Add per-agent cross-process coordination in the memory-core QMD path so background `qmd update`/embed maintenance and foreground QMD search/index share a bounded lock or skip/wait policy, preserve lazy gateway startup and existing fallback behavior, and cover the gateway-process plus CLI-process race with regression tests.

## Repair Work Prompt

Fix the QMD SQLite lock contention reported in openclaw/openclaw#66339. Current main already makes gateway-start QMD refresh opt-in and CLI managers transient; do not undo that. Add a bounded per-agent cross-process coordination mechanism in the memory-core QMD path so `qmd update`/embed maintenance and foreground QMD search/index for the same agent do not race on the same QMD SQLite index. Prefer using the existing plugin-sdk file-lock pattern or an equally bounded helper, with stale-lock handling and short foreground wait/skip/fallback behavior so `openclaw memory search` cannot hang. Keep the fix inside the memory-core/QMD owner path unless a small generic SDK helper is necessary. Cover the race with unit tests that model a gateway full manager doing background QMD work while a separate CLI/transient manager searches or indexes the same agent. Preserve existing builtin fallback semantics, QMD startup laziness, and the current `memory.qmd.update.startup` contract. Add a changelog entry only if the PR changes user-visible behavior; credit the reporter, not Codex.
