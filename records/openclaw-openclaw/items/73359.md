## Summary

Keep this PR open. Current main still does not implement the PR's intended `modelContextWindowOverrides` Bedrock discovery config surface; it only has a built-in known-model context-window table plus `defaultContextWindow` fallback. The PR remains a plausible implementation candidate for open #73328, and the latest PR context shows the contributor addressed earlier review concerns by adding plugin schema/docs and sorting the override cache key, but it still needs maintainer review and validation before merge.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to finish and validate the plugin-owned Bedrock override implementation: keep the discovery behavior in `extensions/amazon-bedrock`, expose it through `openclaw.plugin.json` and docs, add/keep regression tests for unknown foundation models and inference profiles, sort override entries for deterministic cache keys, and decide whether the legacy `models.bedrockDiscovery` schema should also accept the new field. After targeted Bedrock tests and the appropriate changed gate pass, maintainers can merge it or explicitly choose the built-in table approach instead.
