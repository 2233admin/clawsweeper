## Summary

Keep this PR open. Current main has better underlying abort-signal plumbing for embedded runs, but it does not fully implement this PR's stated session queue and sub-agent cleanup behavior, and the PR's own review thread identifies unresolved authorization/scope regressions in the proposed direct cleanup calls.

## Best Possible Solution

Revise the PR or replace it with a narrower implementation that keeps the current abort-signal path for embedded runs, adds missing queue/sub-agent cleanup only when it is semantically session-wide and authorized, and avoids clearing shared session queues for a single runId unless it is the last/only active run or a new run-scoped queue mapping exists. The existing P1 review concerns should be resolved before maintainers consider merging.

## Risks / Open Questions

- The PR as written may convert `chat.abort(runId)` from a targeted run cancellation into a session-wide stop by aborting the embedded session and clearing session queues.
- The sessionKey path may break owner isolation when multiple clients share a session key, because cleanup is keyed by session rather than by the authorized run ids returned from `abortChatRunsForSessionKeyWithPartials`.
- A correct fix needs regression coverage for embedded-run abort, queue clearing, and sub-agent stopping while preserving runId targeting and multi-owner authorization boundaries.
