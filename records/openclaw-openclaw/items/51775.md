## Summary

Keep this PR open. Current main has since added abort-signal plumbing so `chat.abort` can stop active `chat.send` and `agent` RPC runs, but it still does not implement the PR’s stated queue cleanup and sub-agent stop behavior for the Stop button. The PR as written also still has material run-scoping and owner-isolation concerns that need maintainer design input before merge or closure.

## Best Possible Solution

Revise or replace this PR with a narrower design: keep the current abort-signal cancellation for active chat and agent runs, add queue and sub-agent cleanup only when the abort is truly session-wide and authorized for all affected work, and introduce run-scoped ownership for queue/sub-agent cleanup before doing it from `chat.abort(runId)`. Maintainers should answer the author's scoping questions before this PR is merged or closed.

## Risks / Open Questions

- Merging the PR as written can make `chat.abort(runId)` clear session-keyed queues and stop session-keyed embedded/sub-agent work for unrelated runs in the same session.
- The sessionKey path can undermine current paired-device owner isolation if one authorized run abort triggers cleanup that is broader than the authorized run-id set.
- Current main still lacks regression coverage for safe queue and sub-agent cleanup from `chat.abort` that preserves both runId targeting and multi-owner authorization boundaries.
