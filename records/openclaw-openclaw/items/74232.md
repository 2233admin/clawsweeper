## Summary

Keep open. Current main still silently drops malformed or non-allowlisted assistant tool calls in the session write guard, so it does not implement this PR's stated per-drop warning and model-visible synthetic error feedback. Main does have a shipped unknown-tool loop guard that partly addresses repeated waste, but the proposed branch needs maintainer review because it appears to persist toolResult entries for dropped calls that no surviving assistant tool call owns.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether unknown-tool feedback belongs in the session write guard, the stream wrapper, or another provider-safe layer, then implement it without creating orphan toolResult transcript entries, with bounded/redacted diagnostics, and with regression tests for all-invalid, mixed valid/invalid, allowSyntheticToolResults=false, beforeMessageWrite hooks, and replay sanitization.
