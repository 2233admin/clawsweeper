## Summary

Keep open. Current main still drops all-invalid malformed or non-allowlisted assistant tool-call turns in the session write guard without WARN diagnostics or model-visible unavailable-tool feedback. The PR is relevant, and its later commits address the mixed valid/invalid ordering review by limiting injection to fully dropped assistant turns, but that still persists synthetic toolResult messages without a matching persisted assistant tool call, so the transcript-safety and product semantics need maintainer review before merge.

## What This Changes

The PR adds per-call drop diagnostics for tool-call input sanitization and updates the session write guard and tests to WARN and persist synthetic error toolResult feedback for fully dropped unknown or malformed assistant tool-call turns.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide where unavailable-tool feedback belongs: keep WARN diagnostics in the session guard if desired, but only add model-visible feedback through a provider-safe layer that preserves tool_use/tool_result pairing or rewrites the invalid assistant turn without orphan toolResult entries. Regression coverage should include all-invalid turns, mixed valid/invalid turns, allowSyntheticToolResults=false, beforeMessageWrite hooks, provider-owned thinking replay, and the existing unknown-tool stream guard behavior.
