## Summary

Keep open. Current main still drops malformed or non-allowlisted assistant tool-call turns in the session write guard without WARN diagnostics or model-visible unavailable-tool feedback. The PR is a relevant active implementation, but it changes transcript persistence semantics around synthetic toolResult entries, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether immediate unavailable-tool feedback belongs in the session write guard, the stream wrapper, or another provider-safe layer, then implement diagnostics without orphan toolResult transcript entries. Regression coverage should include all-invalid turns, mixed valid/invalid turns, allowSyntheticToolResults=false, beforeMessageWrite hooks, provider-owned thinking replay, and the existing unknown-tool stream guard behavior.
