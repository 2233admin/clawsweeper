## Summary

Keep open. Current main still has no generic trailing-colon suppression in the gateway flush helper, so this is not implemented-on-main; however, the proposed patch is not ready because the normal tool path emits an adjacent item-start event that can still flush the same buffered colon-ending text. The linked Discord symptom also has a shipped preview-streaming mitigation, so maintainers should decide whether a broader gateway policy is still desired.

## What This Changes

The PR adds a `suppressIncompleteNarration` option to gateway buffered-chat flushing, enables it for tool-start flushes when buffered text ends with a colon, and adds gateway agent-event tests for colon suppression plus period-ending flush behavior.

## Best Possible Solution

Maintainers should choose one direction: close this PR as superseded by the shipped #69611 preview-streaming behavior, or revise it into an explicit generic gateway policy that defers incomplete narration across approved tool and item boundaries while preserving final/error flushing and adding focused regression coverage.

## Review Findings
