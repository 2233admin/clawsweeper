## Summary

Keep this PR open. Current main still does not implement the Feishu P2P direct-thread routing that this PR targets: direct messages still force `replyInThread=false`, do not switch the reply target to `root_id` for `thread_id` messages, set `threadReply=false`, and strip reply metadata before the Feishu send layer. The paired contributor bug #38806 remains open and this PR is the active canonical fix candidate after overlapping alternatives #38814 and #38996 were closed.

## Best Possible Solution

Keep this PR open and review/merge the current branch, or reimplement the same narrow Feishu plugin fix on main: detect direct messages with `thread_id`, reply to `root_id` with `replyInThread`/`threadReply` enabled, preserve reply metadata for that direct-thread case, keep plain P2P quote replies unchanged, and land the focused regression tests. Do not close it before merge because it is paired with the still-open #38806 bug and remains the canonical contributor fix path.
