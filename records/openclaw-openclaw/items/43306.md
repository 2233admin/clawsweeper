## Summary

Keep this PR open. Current main still implements only plain temporal decay for memory search; the PR’s central importance-weighted decay feature is not present in the active memory-core implementation, public config schema, generated schema artifacts, docs, or tests. The PR remains a plausible implementation candidate, but it needs to be ported to the current `extensions/memory-core` layout and cleaned up before merge.

## Best Possible Solution

Keep this PR open as the live implementation candidate. Port the importance-weighted decay feature to `extensions/memory-core/src/memory/temporal-decay.ts` and the current hybrid/manager wiring, thread a well-specified `importanceBoost` config through `src/agents/memory-search.ts`, runtime schemas, public types, generated config artifacts, and docs, and add focused memory-core tests for marker matching, file-pattern matching, deep merge/default semantics, blank-pattern rejection, and ranking behavior. Drop or split the already-shipped session-memory reset changes and the unrelated `periodicExtraction` schema work unless they are fully wired and tested separately.
