## Summary

Keep open. Current main still denies cross-agent `send` under non-`all` visibility before checking the a2a policy, and this PR remains useful but is not mergeable as-is because its source edit targets the pre-move helper instead of the active SDK implementation.

## What This Changes

The PR changes the session access guard so cross-agent `send` skips the visibility check and is authorized by a2a policy, removes the send-specific visibility denial branch, and adds regression tests for allowed send versus blocked history.

## Best Possible Solution

Move the send-only visibility exception into `src/plugin-sdk/session-visibility.ts`, keep `list`, `history`, and `status` visibility-gated, add focused tests for `visibility=tree` with a2a-allowed and a2a-denied sends, and leave broader per-capability visibility design to the linked granular-visibility discussion.

## Reproduction Assessment

Yes. A source-level reproduction is `createSessionVisibilityGuard({ action: "send", requesterSessionKey: "agent:main:main", visibility: "tree", a2aPolicy: allow main+ops }).check("agent:ops:main")`; current main reaches the visibility-forbidden branch before the a2a allow check.

## Solution Assessment

No. The conceptual fix is narrow and plausible, but this branch applies it to the old helper location; the maintainable solution is to rebase and apply the same policy split in the SDK helper that current `sessions_send` calls.
