## Summary

Keep PR #57483 open. Current main still has the reported behavior: cross-agent targets are rejected by the session visibility gate before `tools.agentToAgent` can allow `sessions_send`, and main lacks the PR's regression tests. The PR is not obsolete, but it needs a rebase because the guard implementation now lives in `src/plugin-sdk/session-visibility.ts` rather than directly in `src/agents/tools/sessions-access.ts`.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase/adapt the fix into `src/plugin-sdk/session-visibility.ts`, add the two regression tests on current main, and explicitly decide whether the linked sandboxed label-lookup concern belongs in this PR or a narrower follow-up.
