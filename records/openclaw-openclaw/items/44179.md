## Summary

Current `main` still does not apply the shared model-special-token sanitizer to the TUI final/streamed assistant output path named in the issue. The shared sanitizer exists and is tested, but `resolveFinalAssistantText` returns `finalText` and `streamedText` verbatim, and `TuiStreamAssembler.finalize` feeds that result into the visible chat log. The related fix PR from the provided context was closed unmerged, so this should stay open.

## Best Possible Solution

Keep this issue open and implement a narrow TUI fix: apply the shared assistant-visible sanitizer to `finalText` and `streamedText` before `resolveFinalAssistantText` returns them, or sanitize earlier in the TUI assembler extraction path if that preserves streaming semantics better. Add focused TUI regression tests for ASCII `<|...|>` and full-width `<｜...｜>` tokens in both final and streamed fallback paths, while preserving intentional token examples inside code spans/blocks.
