## Summary

Current main still leaves the TUI final/streamed assistant output path unsanitized for GLM-5/DeepSeek special tokens. The shared tokenizer/sanitizer exists and has tests, but `resolveFinalAssistantText` returns `finalText` and fallback `streamedText` verbatim, and the only linked fix PR was closed unmerged.

## Best Possible Solution

Keep this open and land a narrow TUI fix that reuses the shared model-special-token stripping behavior, or the shared assistant-visible sanitizer if it preserves TUI semantics, on final and fallback streamed assistant text before visible chat finalization. Add focused TUI regression tests for ASCII and full-width tokens, fallback-to-streamed finalization, and code-span/code-block preservation; add a user-facing changelog fix entry.
