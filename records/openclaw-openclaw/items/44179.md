## Summary

Current main still leaves the TUI final and fallback streamed assistant output path unsanitized for GLM-5/DeepSeek model-special tokens. The shared sanitizer exists and is tested, but it is not wired into `resolveFinalAssistantText` or the TUI chat-log sink, and the related fix PR #46182 was closed unmerged.

## What This Changes

Apply the existing model-special-token sanitizer to the main TUI assistant output path so GLM-5/DeepSeek `<|...|>` and full-width token delimiters do not appear in visible chat replies.

## Best Possible Solution

Keep this open and land a narrow TUI fix that applies the shared model-special-token stripping behavior, preferably through `sanitizeAssistantVisibleText` or the narrow shared helper if that better preserves TUI semantics, to final and fallback streamed assistant text before visible chat finalization. Add focused regression tests for ASCII tokens, full-width tokens, streamed fallback finalization, and inline/fenced code preservation, plus a user-facing changelog fix entry.
