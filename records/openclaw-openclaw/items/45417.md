## Summary

Keep this PR open. Current main still lacks the PR's intended live-tail inbound user-message behavior for TUI --session mode, and the PR remains a plausible implementation candidate but needs maintainer review or revision for the shared external-channel ingress path and protocol/privacy details.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The strongest product path is to cover the shared inbound/transcript path used by Telegram and other channels: either have TUI --session subscribe to sessions.messages.subscribe and render session.message user turns with dedupe/echo protection, or emit a versioned user-message chat event from the shared dispatch pipeline with schema, docs, privacy/session scoping, and regression tests. Coordinate with #45388 and split unrelated fixes into separate PRs.
