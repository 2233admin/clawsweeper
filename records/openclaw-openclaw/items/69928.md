## Summary

Keep open. Current main still emits `output_config.effort` in the Anthropic Messages transport based only on model id, without a `github-copilot/claude-opus-4.7` compatibility gate. The default bundled Copilot catalog now mitigates the common path by setting Copilot Claude models to `reasoning: false`, but that does not implement the provider/model capability fix when reasoning is enabled. The exact fix PRs referenced from the issue were closed unmerged, and the related open PR covers only part of the effort compatibility cluster.

## Best Possible Solution

Keep this open until a maintainer lands or explicitly rejects a provider/model effort compatibility rule for `github-copilot/claude-opus-4.7`. The best code path is a small, tested capability helper or Copilot Anthropic payload patch that omits or clamps unsupported `output_config.effort` values for that provider/model pair while preserving direct Anthropic Opus 4.7 and out-of-scope Claude models.
