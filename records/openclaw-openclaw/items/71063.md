## Summary

Keep open. Current main still has the unprotected initial stdin write in the child-process supervisor path, and the PR's one-line `writable` guard is relevant but likely incomplete for the Linux missing-binary shape where stdin starts writable and later emits async `EPIPE`. The diff is small and does not introduce supply-chain or CI/security-sensitive file changes, but it should be amended before merge.

## What This Changes

This PR adds a `child.stdin.writable` guard around the supervisor child adapter's initial stdin write and adds unit coverage for a non-writable stdin stub.

## Best Possible Solution

Amend the supervisor adapter so initial stdin writes and closes are protected from both already-closed streams and async stdin `error`/`EPIPE` after early child exit, while keeping the child exit/stderr result visible to callers and preserving normal writable stdin behavior.
