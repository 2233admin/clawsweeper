## Summary

Keep open. Current main still has the unguarded child stdin write/end path and a read-only Linux shell-wrapper probe reproduced the reported uncaught EPIPE shape; the latest PR head appears to address the right supervisor boundary with focused tests, but current-head CI was still queued/in progress during review.

## What This Changes

The PR changes the supervisor child-process adapter to guard writable stdin, swallow initial sync/async stdin failures, and add regression tests for non-writable and EPIPE-style stdin failures.

## Best Possible Solution

Land a narrow supervisor-adapter fix after current-head CI finishes, keeping the behavior generic to child-process stdin handling and avoiding provider-specific fallback or PATH policy changes.

## Solution Assessment
