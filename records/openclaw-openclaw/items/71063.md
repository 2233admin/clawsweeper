## Summary

Keep open. Current main still has the supervisor stdin crash path and no related item supersedes this PR, but the latest PR head is not merge-ready because its new test uses a non-existent NodeJS type and fails `check-test-types`.

## What This Changes

The PR updates the child-process supervisor adapter to guard writable stdin, install a one-shot stdin error handler, catch initial write/end failures, and add regression tests for non-writable stdin plus async and synchronous EPIPE-style failures.

## Best Possible Solution

Keep this as a narrow supervisor adapter fix, correct the test type to `NodeJS.ErrnoException`, and land only after the relevant process-supervisor tests and changed-gate validation pass.

## Reproduction Assessment

Yes. A read-only Node probe using the same Linux shell-wrapper shape reproduced `stdin.writable === true`, a missing-binary stderr line, close code 127, and uncaught `EPIPE`, matching the reported crash path.

## Solution Assessment

No as submitted. The implementation direction is the right generic boundary, but the PR needs the test type fixed before it is a mergeable solution.
