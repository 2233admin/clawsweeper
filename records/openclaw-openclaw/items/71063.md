The PR updates `src/process/supervisor/adapters/child.ts` to guard initial child stdin write/end calls and adds child-adapter regression tests for non-writable stdin, synchronous write errors, and async EPIPE.

## Best Possible Solution

Land a narrow supervisor-adapter fix once current-head validation is acceptable, keeping the behavior generic to child-process stdin handling and avoiding provider-specific PATH, fallback, or onboarding policy changes.
