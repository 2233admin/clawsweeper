## Summary

Keep open. Current main still lacks the Open WebUI UUID session preservation behavior, but this PR is not merge-ready because the branch mixes the useful session fix with unrelated security-sensitive Codex proxy/auth/routing work and still has blocking correctness issues in the proposed patch.

## What This Changes

The PR branch preserves Open WebUI UUID session keys across gateway store resolution and Control UI URLs, threads agent context into transcript lookup, adds regression tests, and also adds Codex proxy transport, model-auth, compaction-auth, and outbound routing changes.

## Best Possible Solution

Land a narrow Open WebUI UUID session fix that preserves existing stores, normalizes legacy aliases consistently, and keeps live subscriptions/history lookups on one canonical identity; move the Codex proxy/auth/routing work to separate owner-reviewed PRs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep UUID keys scoped when merging agent stores:** `src/gateway/session-utils.ts:640-642`
  - body: Returning bare UUIDs from `canonicalizeSessionKeyForAgent` also affects combined per-agent store listings. A UUID row from an `ops` store can appear as an unscoped key, so later `sessions.get`, patch, or delete resolves through the current default agent store and can miss or mutate the wrong history.
  - confidence: 0.87
- **[P2] Normalize legacy UUID aliases for any default agent:** `ui/src/ui/app-settings.ts:48-50`
  - body: `normalizeChatUrlSessionKey` strips only `agent:main:<uuid>`, but the legacy prefixed key was generated with whichever agent was default. Installs whose default was `ops` keep `agent:ops:<uuid>` in the URL/settings and continue the split identity this PR is trying to converge.
  - confidence: 0.86
- **[P2] Resolve main-key aliases before UUID preservation:** `src/gateway/session-utils.ts:665-667`
  - body: The UUID fast path runs before the existing `session.mainKey` alias handling. If an operator configured a UUID-shaped main key, `resolveSessionStoreKey` now returns the bare UUID instead of the canonical main session key, splitting clients that use the configured main alias.
  - confidence: 0.8
- **[P2] Canonicalize UUID aliases for message subscriptions:** `src/gateway/session-utils.ts:665-667`
  - body: The bare UUID and `agent:<id>:<uuid>` forms remain different canonical keys, while session-message subscriptions and fan-out use exact string keys. A legacy client subscribed on the prefixed form can miss live messages emitted under the bare UUID form.
  - confidence: 0.78
- **[P1] Use a WebSocket client that sends custom headers:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:815`
  - body: `connectWebSocket` passes `{ headers }` to `globalThis.WebSocket`, but the standard Node/Bun WebSocket constructor treats the second argument as protocols, so auth and account headers are not sent for proxied Codex requests.
  - confidence: 0.82
- **[P2] Send the system prompt only once:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:728-729`
  - body: `buildRequestBody` sets `instructions` from `context.systemPrompt`, while `convertResponsesMessages` also injects the same prompt into the input messages. That duplicates system instructions on every Codex proxy call and can change behavior or hit context limits sooner.
  - confidence: 0.88
