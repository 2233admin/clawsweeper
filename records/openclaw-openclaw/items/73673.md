## Summary

Keep this issue open. Current main still intentionally installs a macOS Gateway LaunchAgent with `KeepAlive=true` and a one-second `ThrottleInterval`, and the docs describe crash relaunch as a supported launchd behavior. The existing `gateway stop` path disables relaunch for manual stops, but it does not address repeated automatic restarts after a startup failure. The related closed launchd items in the supplied context cover lock contention and config-triggered restart handoff behavior, not this broader debugability problem for arbitrary startup crashes.

## Best Possible Solution

Keep the issue open and implement a targeted macOS launchd crash-loop/debug path. A good fix would distinguish intentional restarts from repeated early startup failures, add a configurable or automatic backoff/stop-after-threshold behavior in the daemon LaunchAgent/runtime path, preserve fast explicit `openclaw gateway restart`, and update `src/daemon/launchd.test.ts`, service audit/doctor guidance, and the macOS Gateway docs.
