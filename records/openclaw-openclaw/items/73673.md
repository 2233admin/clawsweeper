## Summary

Keep open. Current main still generates and audits a macOS LaunchAgent with unconditional `KeepAlive=true` plus a one-second throttle, the tests assert that shape, and startup/config failures can still exit nonzero under launchd supervision. The issue is also paired with an open unmerged closing PR, so it should remain open until that PR is reviewed, narrowed, replaced, or merged.

## What This Changes

The issue reports that macOS LaunchAgent supervision restarts the gateway continuously after startup crashes, making launchd-managed startup failures difficult to debug.

## Best Possible Solution

Keep the issue open and settle a targeted macOS launchd repeated-startup-failure strategy. The end state should preserve reliable explicit `openclaw gateway restart` and macOS app Local mode behavior while adding a maintainable way to avoid tight early-failure loops, with matching plist generation, service audit, tests, and public macOS Gateway docs.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: generate/install the current LaunchAgent, trigger a startup/config failure that exits nonzero, and launchd sees `KeepAlive=true` with a one-second throttle. I did not run a live macOS launchd session in this Linux checkout.

## Solution Assessment

No for current main, and unclear for the linked PR. A plain switch to `KeepAlive` with `SuccessfulExit=false` would respect clean exits, but it still relaunches nonzero startup failures and can conflict with clean-exit supervised restart behavior unless the restart path is redesigned too.
