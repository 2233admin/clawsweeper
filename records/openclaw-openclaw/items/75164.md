Adds a Provider credentials section to the media-understanding docs explaining that `tools.media` model entries do not take `apiKey`, and points users to auth profiles, provider environment variables, or `models.providers.<providerId>` with a custom-provider example.

## Best Possible Solution

Review and land the docs clarification if the normal docs checks pass. Supporting direct `apiKey` inside `tools.media.*.models[]` would be a separate product/API decision, not needed for the documentation fix requested here.

Not applicable as a PR diff reproduction. For the linked setup failure, there is a high-confidence static reproduction path: direct `apiKey` is absent from the strict media model schema, runtime credentials resolve through model auth and `models.providers`, and the current docs omit that guidance.

## Solution Assessment
