## Summary

Keep open. This is a fresh, detailed WhatsApp/OpenRouter embedded-runtime hang report against the latest released build, with another user confirming the symptom. Current main has nearby fixes for WhatsApp listener recovery, prompt-memory hook latency, Discord timeouts, and a session-key fallback, but none proves this WhatsApp inbound reply stall is fixed.

## Best Possible Solution

Keep this issue open for maintainer triage. Ask affected users to retest on current main or the next npm release containing the post-2026.4.26 fixes, then collect gateway logs around `before_agent_reply`, `before_prompt_build`, model start/end events, OpenRouter request timeout/abort handling, WhatsApp reconnect, and session idle transition. If reproduced, add a focused regression or live smoke at the WhatsApp inbound dispatch boundary and fix whichever awaited layer is proven to never settle.
