## Summary

Keep open. Current main and the latest tagged release still do not implement the PR's two central changes: explicit gateway agent workspaceDir is not accepted by the protocol validator, and missing bootstrap files are still returned and injected as [MISSING] context entries. The PR also has unresolved review feedback on a stale test and a security-sensitive workspace path override.

## What This Changes

The PR adds an explicit gateway agent workspaceDir parameter, prefers it over inherited spawned-session workspace metadata, and changes workspace bootstrap handling so missing bootstrap files are omitted from prompt injection.

## Best Possible Solution

Keep this PR open for revision. The gateway workspace change should either be restricted to trusted internal/admin callers or derived from validated spawned-session metadata, and the bootstrap noise change should be coordinated with #73401/#40210 with all affected tests updated. If workspaceDir remains a protocol field, update the generated protocol clients and docs expected for a gateway schema change.
