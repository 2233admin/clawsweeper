## Summary

Keep open. Current main still makes a single Feishu `messageResource.get` call with the supplied `type`, while inbound non-image/video/file media paths still resolve to `type: "file"`; there is no `type=media` retry on HTTP 502. The linked bug #49855 remains open with concrete reproduction details and discussion agreeing that a file-to-media fallback is the right direction. This PR is a focused implementation candidate, and its only review concern was addressed by the second PR commit preserving the original 502 error if the retry also fails.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this patch or an equivalent change in `extensions/feishu/src/media.ts`, preferably with focused `extensions/feishu/src/media.test.ts` coverage for `type=file` returning HTTP 502, retrying with `type=media`, and preserving the original error if the retry also fails. After merge, #49855 can close through the linked fix.
