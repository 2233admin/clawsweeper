Keep this PR open. Current main still downloads Feishu message resources with a single `type=image|file` call and has no 502 retry to `type=media`, while the linked Feishu bug #49855 and overlapping implementation candidate #49967 are still open in the provided live context. The PR remains a focused Feishu-owned fix candidate, and the review-thread concern about preserving the original 502 error was addressed by the PR’s second commit.
