## Summary

Keep this PR open. Current main still drops the target agent auth directory before `/models` visibility filtering, and the PR patch only repairs the direct text handler path while leaving shared `buildModelsProviderData(cfg, agentId)` callers such as Telegram callbacks, Discord model picker loading, and Mattermost picker flows without an auth scope. The open replacement #74564 appears to implement the better shared-boundary fix, so maintainers should choose the canonical branch rather than close this as implemented.

This PR threads `agentDir` and `env` through the direct `/models` reply path into provider visibility filtering, adds a worker-agent auth regression test, and adjusts the changelog entry for the model-visibility fix.

Land a single narrow fix that resolves the effective target agent auth directory at the shared provider-data boundary, preserves explicit `agentDir` overrides, covers both direct `/models` handling and existing `agentId` helper callers, and then retire the duplicate branch once the canonical PR is merged.
