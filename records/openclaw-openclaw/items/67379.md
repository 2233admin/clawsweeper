## Summary

Current main still makes spawned subagent QMD session keys resolve with no channel/chatType, so default-deny QMD scope rules can deny them. The related open PR #70297 is a separate default-scope policy change and does not fix subagent key classification.

## Best Possible Solution

Keep this issue open and fix the QMD scope contract for spawned subagent sessions. The likely narrow fix is to make `agent:<id>:subagent:<uuid>` inherit the parent session classification or introduce an explicit documented subagent classification, then update both `src/memory-host-sdk/host/qmd-scope.ts` and `packages/memory-host-sdk/src/host/qmd-scope.ts` with regression coverage for `isQmdScopeAllowed`, `deriveQmdScopeChannel`, and `deriveQmdScopeChatType`. Treat #70297 as a separate default-scope policy review, not the implementation for this issue.
