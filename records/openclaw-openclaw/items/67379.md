## Summary

Keep open. Current main still normalizes `agent:<id>:subagent:<uuid>` to `subagent:<uuid>` and then drops it, so QMD scope sees no channel or chatType and default-deny policy still blocks subagent QMD recall. The related #70297 default-scope work is closed unmerged and does not resolve subagent classification.

## What This Changes

The issue requests fixing QMD memory-scope classification for `sessions_spawn` subagent session keys so subagents can use QMD recall under the intended policy.

## Best Possible Solution

Define explicit QMD scope semantics for subagent keys, preferably preserving the requester/session classification when available or adding a documented subagent classification, then add regression coverage for default scope behavior, explicit rawKeyPrefix overrides, and QMD manager denial/allow handling.
