## Summary

Keep open. Current main still does not implement this PR's intended verbose-off suppression for assistant chat sent to session/channel subscribers, and the PR remains an implementation candidate rather than obsolete. It also still needs maintainer review because the supplied PR diff includes unrelated plugin SDK and bundled channel surface changes alongside the gateway privacy fix.

## Best Possible Solution

Keep this PR open as a candidate, but require a focused rebase before merge. The best path is a narrow gateway fix in `src/gateway/server-chat.ts` plus targeted regression tests that suppress assistant delta, flushed delta, final done, and error session delivery when verbose resolves to off, while preserving WS/Control UI broadcasts and in-flight verbose updates. Split the unrelated plugin SDK and bundled channel changes into separate owner-reviewed work. The broader product problem should continue to be tracked by #25592 until a clean fix lands.
