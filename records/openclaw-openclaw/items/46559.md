## Summary

Keep open. Current main still has the reported verbose-off chat leak, but this PR is not ready to merge because it leaves final chat delivery unguarded, caches verbose state across in-flight changes, treats inherited default verbose-off as streamable, and bundles unrelated WhatsApp/SDK/channel changes.

## What This Changes

The PR gates assistant delta and buffered-delta session sends by verbose level, adds gateway regression tests, introduces a per-run verbose cache, and also changes plugin SDK/channel exports and bundled channel setup/docs.

## Best Possible Solution

Land a focused gateway fix that evaluates the effective verbose level at each assistant text delivery point, treats inherited/default verbose as off, suppresses session/channel assistant text for delta, flush, final done, and error chat paths when verbose is off, preserves WebSocket/control UI broadcasts, and leaves unrelated plugin SDK/channel refactors out of this PR. Keep #25592 open for the broader channel-delivery policy after the narrow leak is fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate final chat payloads by verbose level:** `src/gateway/server-chat.ts:496`
  - body: The patch suppresses delta and flush session sends, but emitChatFinal still sends final done/error payloads to the session unconditionally. With verbose off, subscribers can still receive the complete buffered assistant text or error payload at run end, so the leak remains.
  - confidence: 0.96
- **[P1] Treat missing verbose config as off for session streaming:** `src/gateway/server-chat.ts:543-548`
  - body: The docs define verbose off as the default, but shouldStreamAssistantTextToSession returns true when resolveVerboseLevel is undefined. Sessions that inherit the default verbose-off mode still stream assistant text unless they have an explicit off entry.
  - confidence: 0.9
- **[P1] Re-read verbose after session changes:** `src/gateway/server-chat.ts:510-513`
  - body: The new cache returns the first resolved verbose value for the whole run. Runtime /verbose changes are documented to affect in-flight runs, so later assistant/tool events can keep leaking or suppressing text according to a stale mode.
  - confidence: 0.88
- **[P2] Keep WhatsApp group policy adapters wired:** `extensions/whatsapp/src/channel.ts:47-52`
  - body: The unrelated WhatsApp channel change drops the groups adapter from the plugin. Group require-mention, intro hints, and tool-policy resolution depend on that wiring, so this changes configured group behavior outside the chat streaming fix.
  - confidence: 0.82
