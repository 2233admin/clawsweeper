## Summary

Keep open. Current main still does not implement the PR's intended verbose-off chat suppression for session/channel subscribers: assistant deltas, flushed buffered deltas, final done payloads, and error payloads are still sent through `nodeSendToSession` without a verbose-off guard. The PR also remains an implementation candidate rather than an obsolete change, but it needs to be narrowed and corrected before merge because the supplied review context flags unresolved final/cache behavior and unrelated SDK/channel surface changes.

## Best Possible Solution

Keep this PR open only as a candidate for a focused fix. The best path is to rebase and narrow it to `src/gateway/server-chat.ts` plus targeted gateway regression tests: gate session/node delivery for assistant deltas, flushed buffered deltas, final done payloads, and error payloads when verbose resolves to off, while keeping WS/Control UI broadcasts intact. Add coverage for delta, pre-tool flush, lifecycle final, lifecycle error, session/default verbose resolution, and in-flight verbose changes. Drop or split the unrelated plugin SDK and channel changes. The broader remaining product bug should continue to be tracked by #25592 until a clean fix lands.

## Risks / Open Questions

- Merging the PR as-is could still leak assistant text through final, error, or flush-before-tool session chat payloads if every chat send path is not gated consistently.
- A per-run cached verbose decision can become stale when the session key appears later or when `/verbose` changes while the run is in flight.
- The unrelated SDK barrel and bundled channel setup/config changes broaden public plugin surface and could affect channel config validation, group policy, import topology, or startup cost independently of the gateway leak fix.
