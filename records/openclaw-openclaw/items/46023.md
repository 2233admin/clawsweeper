## Summary

Keep open. Current main still matches the reported Telegram outbound final-reply media limit gap: the Telegram plugin computes the configured cap and direct sends pass it into media loading, but final reply and native command delivery still do not thread any cap into `deliverReplies`, `deliverMediaReply`, or `loadWebMedia`. The related PR #46425 is useful implementation context, but it is closed unmerged and does not supersede the remaining bug.

## What This Changes

Telegram final outbound replies with `mediaUrl` or `mediaUrls` should honor the configured `channels.telegram.mediaMaxMb` limit.

## Best Possible Solution

Land a narrow Telegram plugin fix that carries the computed media cap from bot setup through normal message processing and native command delivery into `deliverReplies` and `deliverMediaReply`, then passes it as `maxBytes` to `buildOutboundMediaLoadOptions`, with regression tests proving configured limits reach `loadWebMedia` for final media replies.

## Repair Work Prompt

Fix the Telegram outbound final-reply media cap bug from https://github.com/openclaw/openclaw/issues/46023. Current main computes `mediaMaxBytes` in `extensions/telegram/src/bot-core.ts`, and `extensions/telegram/src/send.ts` already passes the equivalent value as `maxBytes` for direct sends, but final reply delivery and native command replies do not pass the cap into `deliverReplies`/`deliverMediaReply`/`loadWebMedia`. Port the idea from closed unmerged PR https://github.com/openclaw/openclaw/pull/46425 to the current file layout. Keep the fix narrow: thread an optional `mediaMaxBytes` through `createTelegramMessageProcessor`, `dispatchTelegramMessage`, native command delivery, `deliverReplies`, and `deliverMediaReply`; pass it to `buildOutboundMediaLoadOptions({ maxBytes: ..., mediaLocalRoots: ... })`; preserve default behavior when the value is absent; do not change media loading internals or the already-correct direct-send path. Add focused regression tests that prove configured limits reach `loadWebMedia` for final media replies and native command replies. This is a user-facing bug fix, so add a changelog entry under the active fixes section with the issue reporter/contributor attribution, following the repository changelog rules.
