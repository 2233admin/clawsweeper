Current main still has the reported Codex app-server queued-steering drop, while the PR is a narrow implementation that makes steering report success/failure and falls back to the existing followup queue when terminal steering is rejected. The PR changes Codex app-server queued steering to return a durable boolean result, adds an async embedded-message queue helper, awaits that result in runReplyAgent before skipping followup queuing, and adds focused regression/test-routing coverage.
