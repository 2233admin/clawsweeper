## Summary

Keep open. Current main at a820a307dfeb950fa633c0ae6fb2386a35a91991 still lets positive configured or persisted context-token overrides win before known model windows are applied in the shared resolver and several reporting/persistence paths. The same-author implementation candidate #39859 remains open and unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land #39859 or an equivalent narrow maintainer-approved fix. The implementation should make positive configured or persisted contextTokens act as ceilings when a smaller selected-model window is known, preserve current override behavior when no model window is known, normalize provider aliases consistently, and add focused regression coverage for the resolver, reply setup, /status and status summary, follow-up/session usage, and session metadata persistence.
