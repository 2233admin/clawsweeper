## Summary

Keep open. Current main still returns positive configured or persisted context-token overrides before applying known selected-model windows in the shared resolver and several reporting/persistence paths. The same-author implementation candidate #39859 remains open and unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land #39859 or an equivalent narrow fix. The implementation should make positive configured or persisted contextTokens act as ceilings when a smaller selected-model window is known, preserve current override behavior when no model window is known, normalize provider aliases consistently, and add focused regression coverage for the resolver, reply setup, /status, follow-up/session usage, and session metadata persistence.
