## Summary

Keep open. Current main still does not implement this PR's intended `/status` visibility: the status path has no Telegram message context, no command target context, no channel-owned status-line slot, and no Telegram topic status helper. The underlying Telegram topic ACP routing and binding primitives exist, and the PR discussion identifies a real configured-vs-live routing visibility gap, so this remains useful maintainer-review work rather than conservative cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or a focused replacement. The best implementation is to pass message context and command target into `/status`, add a narrow channel-owned status-lines slot below the session line, resolve configured and live Telegram topic ACP bindings through existing binding helpers, render configured vs live/drift plus ACP runtime state, and cover configured binding, live binding, drift, ACP runtime metadata, and non-topic no-op behavior in tests. Keep unrelated infra or dependency changes out of this PR.
