## Summary

Keep open. Current main has bootstrap-size diagnostics in doctor, `/context`, and truncation prompt metadata, but it does not implement the requested Control UI near-limit warning or save refusal, and an open draft PR already has explicit closing linkage for this issue.

## What This Changes

Request warning and enforcement before `MEMORY.md` exceeds the workspace bootstrap truncation budget, especially in Control UI save flows and session startup.

## Best Possible Solution

Keep this issue open and use #64874, or a maintainer-owned successor, as the implementation vehicle. The end state should expose generic bootstrap-file health in Control UI, warn before truncation, make save diagnostics explicit, and split auto-archive into a separate product decision unless maintainers want that behavior in core.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to save a `MEMORY.md` larger than `agents.defaults.bootstrapMaxChars` through the Control UI/RPC file path, then inspect a run or `/context list`; code inspection shows saving is accepted while bootstrap injection later truncates the file.

Unclear. Current main is only a partial mitigation, and #64874 is a focused candidate for visibility and diagnostics, but save refusal/enforcement and auto-archive need maintainer scope review before treating it as the complete fix.
