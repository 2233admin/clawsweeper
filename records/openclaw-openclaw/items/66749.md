## Summary

Keep this PR open. Current main still has the targeted unguarded exec-event heartbeat wake in both exec completion paths, the heartbeat runner still maps subagent-scoped forced wakes back to the main session, and the same-author linked #66748 remains open, so this is still a plausible implementation PR rather than cleanup-close material.

## What This Changes

The PR adds an `isSubagentSessionKey` guard around exec-event heartbeat wakes in `maybeNotifyOnExit` and `emitExecSystemEvent` while preserving system-event enqueueing, plus an `emitExecSystemEvent` regression test for subagent session keys.

## Best Possible Solution

Review and land this PR or an equivalent narrow guard so subagent exec completions still enqueue their system events but do not wake the parent heartbeat lane; after that lands, #66748 can close as fixed.
