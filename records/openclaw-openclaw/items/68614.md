## Summary

Keep open. Current main still lacks the PR’s deferred-response timer and long-running fallback path; it only falls back after a Google Chat message update fails. The branch contains useful source work, but it is currently dirty against main and still has an unresolved timer-cleanup review finding that should be addressed before any merge or replacement lands.

## What This Changes

The PR branch modifies the Google Chat monitor to add a 25-second timer after sending a typing message, send an interim fallback message when the timer expires, delete the typing message, and route the eventual final reply as a new message with extra update-failure logging.

## Best Possible Solution

Port the deferred-response timeout into the current split Google Chat implementation: start a bounded timer only after a typing message is created, send an interim same-thread fallback and retire the typing message on expiry, preserve eventual final text/media delivery as a normal new message, and add focused fake-timer tests for expiry, successful fast delivery, update-failure fallback, and dispatch completion without delivery.
