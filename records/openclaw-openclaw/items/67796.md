## Summary

Keep this PR open. Current main still has the Mattermost message-tool send path reading media only from `params.media`, so the PR targets a real parity gap, but the proposed fallback still drops valid aliases when `media` is present but blank and it lacks focused regression coverage.

## What This Changes

The PR branch updates Mattermost `handleAction` media resolution to derive `mediaUrl` from `params.media`, then `params.filePath`, then `params.path`.

## Best Possible Solution

Land a corrected Mattermost alias fix that normalizes each media candidate independently, preserves the chosen precedence, forwards the first non-empty value to the existing Mattermost media path, and adds focused handleAction tests for both aliases plus blank-media fallthrough.

## Review Findings

- **[P2] Normalize media before alias fallback:** `extensions/mattermost/src/channel.ts:215-224`

Normalize each candidate first and then coalesce the results.
