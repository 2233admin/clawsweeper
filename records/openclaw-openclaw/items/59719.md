## Summary

Keep open. Current main still lacks the PR's central linked CLI-task plumbing for background exec/process sessions: background exec yield only marks the process session as backgrounded, process removal still deletes a live session immediately, and the public task docs scope CLI tasks to CLI agent commands rather than plain exec/process runs. The PR should not be merged as-is because the review context includes a P1 bug about overbroad CLI liveness checks, but the underlying product gap remains concrete.

## Best Possible Solution

Keep the PR open for maintainer/product judgment, but request a reworked implementation rather than merging the current branch. The best path is a narrow exec-owned task linkage: create/update task records only for background `exec`/`process` sessions, heartbeat the task on output, preserve inspectability during remove/cancel until actual exit, scope session/status overrides to those linked exec tasks, and align maintenance with the later #60310 runtime-aware task reconciliation. Regression coverage should cover background exec task creation, output heartbeat/stall behavior, remove/kill visibility, orphan reconciliation, and status/session-row derivation without touching unrelated CLI agent tasks.
