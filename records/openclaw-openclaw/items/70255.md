## Summary

Keep this PR open. Current main still routes `openclaw hooks` list/info/check/enable/disable and bare `hooks` through the generic hooks action runner with no successful-path exit, while diagnostics still load plugin modules and run `register()`, making the reported post-output hang plausible. The PR directly addresses that gap and the review comments were handled, but it is now dirty against main because the hooks CLI has migrated from `loadConfig()` to `getRuntimeConfig()`.
