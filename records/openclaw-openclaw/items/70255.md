## Summary

Keep this PR open. Current main still routes `openclaw hooks` list/info/check/enable/disable and bare `hooks` through the generic hooks action runner without a successful-path exit, while hook reports still load plugin diagnostics that can run plugin `register()` handlers. The PR directly targets that gap and its review comments appear addressed, but the diff is stale against current main because the hooks CLI command handlers now use `getRuntimeConfig()` instead of the PR's `loadConfig()` calls.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Update it onto current main, preserve the one-shot exit boundary for hooks list/info/check/enable/disable/bare hooks, keep deprecated hooks install/update and hidden relay behavior unchanged, then land after focused hooks CLI coverage plus the relevant plugin diagnostics/status test pass.
