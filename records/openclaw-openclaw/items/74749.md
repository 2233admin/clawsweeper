## Summary

Keep open. Current main and the latest release still configure the Google Gemini CLI backend with `--skip-trust`, the runner forwards that configured value into the spawned `gemini` argv, and open PR #74775 is the active closing implementation path rather than a reason to close the issue before merge.

## What This Changes

The issue requests removing the unsupported `--skip-trust` flag from the Google Gemini CLI backend defaults so `gemini-cli` 0.38.x can run successfully.

## Best Possible Solution

Review and land or replace PR #74775 with a narrow Google plugin fix that removes `--skip-trust` from both fresh and resume defaults, updates the default-argument tests, and keeps trust and approval behavior conservative instead of switching to broader auto-approval flags.

## Reproduction Assessment

Yes. Current main configures `--skip-trust`, the runner forwards backend defaults directly into the `gemini` argv, and upstream `@google/gemini-cli@0.38.2` uses strict parsing without that option.

## Solution Assessment

Yes. Dropping only the unsupported flag from `extensions/google/cli-backend.ts` is the narrow maintainable fix; replacing it with `--yolo` or broader approval modes would change security semantics beyond this bug.
