## Summary

Keep open. Current main still configures `google-gemini-cli` with `--skip-trust`, the CLI runner forwards that configured argv into `gemini`, Homebrew stable Gemini CLI 0.38.2 still lacks the option, and open PR #74775 is the active unmerged implementation path rather than a reason to close this issue early.

## What This Changes

The issue requests removing the unsupported `--skip-trust` flag from the Google Gemini CLI backend's fresh and resume command defaults.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes the incompatible flag from both fresh and resume Gemini CLI defaults, updates the backend-default test and changelog, and avoids substituting broader approval behavior such as `--yolo` unless that security posture is explicitly chosen.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures and forwards `--skip-trust`, and the Homebrew-stable Gemini CLI 0.38.2 parser does not define that option.

## Solution Assessment

Yes. The narrow maintainable fix is to remove the incompatible flag from `extensions/google/cli-backend.ts` and align tests/changelog; a version-gated alternative is possible, but replacing it with a broader approval flag would solve a different security problem.
