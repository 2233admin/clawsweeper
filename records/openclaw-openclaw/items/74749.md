## Summary

Keep open. Current main still configures `--skip-trust` for the Google Gemini CLI backend, the runner forwards that backend argument to `gemini`, the latest tagged release remains affected, and open PR #74775 is the active closing implementation path rather than a reason to close this issue before merge.

## What This Changes

Remove the unsupported `--skip-trust` flag from Google Gemini CLI fresh and resume backend arguments so `gemini-cli` 0.38.x and newer calls can run.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes only the unsupported Gemini trust flag from fresh and resume defaults, updates the backend default assertions, and avoids substituting broader approval behavior such as `--yolo` unless that security posture is explicitly chosen.

## Reproduction Assessment

Yes. The source-level path is high confidence: current main configures `--skip-trust`, the CLI runner forwards backend args into `gemini`, and upstream `@google/gemini-cli@0.38.2` does not define that option.

## Solution Assessment

Yes. The narrow maintainable fix is to drop the unsupported flag from `extensions/google/cli-backend.ts` and align `src/agents/cli-backends.test.ts`; replacing it with broader auto-approval would change trust semantics beyond this bug.
