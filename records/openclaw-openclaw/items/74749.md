## Summary

Keep open. Current main and the latest stable tag still pass `--skip-trust` to the Google Gemini CLI backend, the CLI runner forwards that backend config into the spawned `gemini` argv, and open PR #74775 is the active closing implementation path rather than a reason to close before merge.

## What This Changes

Remove the unsupported `--skip-trust` flag from the Google Gemini CLI backend fresh and resume argv defaults so `gemini-cli` 0.38.x and newer can run.

## Best Possible Solution

Review and land or replace #74775 so the Google plugin removes `--skip-trust` from both fresh and resume defaults, updates the backend default assertions, and keeps Gemini trust/approval behavior conservative and plugin-owned.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures `--skip-trust`, the runner forwards configured backend argv directly to `gemini`, and upstream Gemini CLI 0.38.2 does not define that strict yargs option.

## Solution Assessment

Yes. The best fix is the narrow plugin-owned change already proposed by #74775: remove only the unsupported flag and update tests, rather than adding broader auto-approval behavior.
