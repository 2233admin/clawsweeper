## Summary

Keep open. Current main still contains the reported Gemini CLI backend failure path, and the linked open PR is an implementation candidate rather than a reason to close this issue before it merges.

## What This Changes

The issue asks OpenClaw to stop passing the unsupported `--skip-trust` argument to the Google Gemini CLI backend because gemini-cli 0.38.x rejects it.

## Best Possible Solution

Review and land PR #74775 or an equivalent narrow replacement that removes `--skip-trust` from both fresh and resume Gemini CLI backend defaults, updates the backend default tests, and keeps Gemini trust behavior conservative.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures `--skip-trust`, the runner passes backend args directly into the `gemini` subprocess, and gemini-cli 0.38.2 has no matching option.

## Solution Assessment

Yes. The narrow maintainable fix is to remove only the unsupported flag from `extensions/google/cli-backend.ts` and update `src/agents/cli-backends.test.ts`; changing to a broader approval flag would alter security semantics.
