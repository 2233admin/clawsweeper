## Summary

Keep open. Current main still registers `google-gemini-cli` with `--skip-trust`, the CLI runner forwards backend args into the spawned `gemini` argv, upstream `@google/gemini-cli` 0.38.2 does not define that option under strict yargs parsing, and the latest tagged release remains affected. Open PR #74775 is the active unmerged closing PR, so this issue should stay open until that PR or a maintainer replacement lands.

## What This Changes

Remove the unsupported `--skip-trust` argument from the Google Gemini CLI backend fresh and resume defaults so gemini-cli 0.38.x calls do not fail before model execution.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes only `--skip-trust` from `extensions/google/cli-backend.ts` fresh and resume defaults, updates `src/agents/cli-backends.test.ts`, and avoids substituting broader approval semantics such as `--yolo` unless maintainers explicitly choose that security posture.

## Reproduction Assessment

Yes. The source-level path is high confidence: current main configures `--skip-trust`, the runner forwards that argv into `gemini`, and upstream gemini-cli 0.38.2 uses strict yargs options without that flag.

## Solution Assessment

Yes. Removing the unsupported flag from the Gemini CLI backend defaults is the narrow maintainable fix; replacing it with `--yolo` or another approval-bypass flag would solve a different problem with broader security implications.
