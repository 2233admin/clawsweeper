## Summary

Keep open. Current `main` and the latest tagged release still configure the Google Gemini CLI backend with the reported unsupported `--skip-trust` argument, and open PR #74775 specifically targets this issue with closing syntax, so the issue should stay open until that PR or a maintainer replacement lands.

## What This Changes

The issue reports that `google-gemini-cli` fails against `gemini-cli` 0.38.x because OpenClaw passes an unsupported `--skip-trust` flag in fresh and resume CLI invocations.

## Best Possible Solution

Land PR #74775 or a narrow maintainer replacement that removes only the unsupported Gemini trust flag from fresh and resume defaults, updates `src/agents/cli-backends.test.ts` and changelog as required, and avoids substituting broader approval behavior such as `--yolo` unless maintainers explicitly choose that security posture.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures `--skip-trust`, the CLI runner forwards backend args into the subprocess argv, and upstream Gemini CLI 0.38.2/official headless docs do not define that option.

## Solution Assessment

Yes. The narrow maintainable fix is to drop the unsupported flag from the Google plugin backend defaults and align tests; replacing it with `--yolo` would change approval semantics and solve a different problem.
