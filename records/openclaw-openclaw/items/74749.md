## Summary

Keep open. Current main still passes the unsupported Gemini CLI flag, the latest stable tag is still affected, and open PR #74775 is the active closing implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to stop passing `--skip-trust` to the bundled `google-gemini-cli` backend because Gemini CLI 0.38.x rejects that option.

## Best Possible Solution

Land or replace PR #74775 with a scoped Google plugin fix that removes `--skip-trust` from both fresh and resume defaults, updates the backend-default tests and changelog, and preserves conservative trust behavior without substituting broader auto-approval such as `--yolo`.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures the unsupported flag, the runner forwards it into the spawned `gemini` argv, and upstream Gemini CLI 0.38.2 does not define that option.

## Solution Assessment

Yes. The narrow maintainable fix is to remove only the unsupported flag from `extensions/google/cli-backend.ts` and update the matching assertions; replacing it with a broad approval mode would change security semantics beyond this bug.
