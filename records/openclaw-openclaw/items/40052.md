Keep this PR open. Current main still writes default-agent primary model and fallback edits through `agents.list[index].model`, while the documented default model contract is `agents.defaults.model`. The PR targets a real Control UI bug, but it should be updated before landing to clear or migrate stale default-agent `agents.list[].model` overrides and add the missing `defaultId === null` regression coverage noted in review.

Keep this PR open or replace it with a maintainer-owned patch that routes default-agent primary and fallback edits to `agents.defaults.model`, keeps non-default agents on `agents.list[].model`, clears or migrates stale default-agent `agents.list[].model` overrides during the save path, adds focused tests for default agent, non-default agent, and `defaultId === null`, and includes a user-facing changelog entry with proper attribution if the fix lands.
