## Summary

Keep this issue open. Current main still leaves the lower-level Discord retry runner limited to Carbon `RateLimitError`/HTTP 429, tests still assert plain network errors are not retried on that path, and #52480 is an open unmerged implementation candidate that declares this issue as its closing target.

## Best Possible Solution

Keep this issue open and review #52480 as the implementation candidate. The likely fix belongs in the Discord plugin retry classifier with focused tests for direct send/runner behavior and outbound adapter behavior, while preserving `RateLimitError.retry_after` handling and avoiding duplicate status-coded retry layers.
