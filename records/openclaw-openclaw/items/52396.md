Keep this issue open. Current main still leaves the low-level Discord retry runner limited to Carbon RateLimitError/HTTP 429, the delivery wrapper only mitigates status-coded 429/5xx errors, and the current tests/docs still preserve the reported asymmetry. The provided GitHub context also shows #52480 is an open, unmerged PR with closing syntax for this issue, so cleanup should wait for that PR to land or be rejected.
