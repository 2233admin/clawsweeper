## Summary

Current main still only partially overlaps with this PR. The overview cards are now native clickable buttons wired to tab navigation, but the broader affordance cleanup remains unimplemented because generic hover styles still apply to non-clickable cards, stats, and account cards. The PR also carries unrelated root Spryng/routing artifacts and an unrelated hooks test change, so it should not be treated as implemented or landed as-is.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not land it as-is. The best path is a focused UI-only rebase or replacement: preserve the current native `overview-cards.ts` button navigation, remove generic hover affordances or introduce explicit interactive modifiers consistently, add targeted click plus keyboard activation coverage, address the existing review notes where still applicable, and drop the unrelated Spryng/routing artifacts plus the hooks test change.
