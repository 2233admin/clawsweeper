## Summary

Keep #69271 open. Current main still exposes session deletion through checkbox selection plus the bulk delete bar only; there is no per-row delete/archive/actions control in the Sessions table. The backend delete path archives transcript files as a delete side effect, but there is no first-class archive action, archive filter, or recovery-oriented archive view.

## Best Possible Solution

Keep the issue open. A focused implementation should add a narrow row Actions column in `ui/src/ui/views/sessions.ts` and wire a single-session delete action through the existing `deleteSessionsAndRefresh(state, [key])` path. Treat the requested archive option as a separate design/API follow-up unless maintainers decide that the existing `sessions.delete` transcript-archive behavior is enough to label the row action as archive/delete.
