## Summary

Keep open. Current main still exposes Control UI session deletion only through checkbox selection plus the bulk delete bar. The Sessions table has no per-row Delete, Archive, or Actions affordance, and the Gateway delete path only archives transcript files as a delete side effect rather than exposing first-class archived session state, filters, or restore/recovery behavior.

## Best Possible Solution

Keep this issue open. The best narrow implementation is a compact per-row Delete or Actions affordance in the Control UI Sessions table that calls the existing deleteSessionsAndRefresh(state, [key]) flow and preserves current confirmation/loading/error/refresh semantics. Treat first-class Archive as separate Gateway/session API and product work before adding archive UI.
