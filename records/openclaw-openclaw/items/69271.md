## Summary

Keep open. Current main still only offers checkbox selection plus the bulk delete bar in the Control UI Sessions table, while the existing delete helper can already delete a single key if a per-row action is wired through it. The requested first-class Archive behavior is still broader product/API work because the current Gateway delete path removes the session entry and only archives transcript files.

## What This Changes

The issue requests a per-session Delete, Archive, or compact Actions control in the Control UI Sessions table to avoid selecting rows for single-session cleanup.

## Best Possible Solution

Add a compact per-row Delete or Actions affordance in the Sessions table that delegates to the existing deletion controller for a single key, and leave first-class Archive for a separate Gateway/session API and product-design follow-up.

## Reproduction Assessment

Yes. This is a feature gap rather than a runtime bug, and current-main source inspection shows the Sessions view has only bulk deletion plus no Actions column while the controller supports deleting a one-key array.

## Solution Assessment

Yes for the narrow delete affordance: reusing deleteSessionsAndRefresh(state, [key]) is the smallest maintainable path because it preserves existing confirmation, loading, refresh, and partial-error behavior. No for adding Archive in the same small UI patch, because the required archived-session semantics do not exist yet.
