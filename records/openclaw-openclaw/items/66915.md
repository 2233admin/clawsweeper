## Summary

Keep open. Current main still collapses ordinary verbose-off `edit` mismatch failures into a generic mutating-tool warning, while the lower edit layer has richer mismatch context that needs a sanitized payload summary. PR #67276 remains open and unmerged as the explicit implementation candidate, so this issue should stay open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

Request clearer, actionable user-facing errors for `edit` exact-match whitespace/newline mismatches, plus a reliable way for deployments to avoid `edit` when they prefer `write`.

## Best Possible Solution

Review PR #67276 as the focused fix path. The desired end state is a sanitized payload-layer summary for exact-text `edit` mismatches in verbose-off output, unchanged edit semantics, existing `tools.deny` policy as the disable-edit path, and broader whitespace normalization/read-positioning ideas handled separately under #62863.
