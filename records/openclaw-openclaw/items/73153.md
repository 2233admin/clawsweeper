## Summary

Keep open. Current main still lacks the reasoning-prefaced silent-reply handling in both the auto-reply gate and the embedded runner, so the PR is not obsolete. The proposed branch is useful but not mergeable as-is because it introduces `AssistantMessage` in `incomplete-turn.ts` without importing or defining that type.

## What This Changes

The PR adds reasoning-prefaced `NO_REPLY` detection to auto-reply token classification and threads that silent-reply classification into embedded-runner incomplete-turn retry handling, with regression tests.

## Best Possible Solution

Keep this PR open, fix the missing runner type import or reuse the existing `EmbeddedRunAttemptResult["lastAssistant"]` type, then run the targeted auto-reply and runner tests plus the changed gate before normal maintainer review.

## Reproduction Assessment

Yes. No, not as-is. The implementation direction is narrow and maintainable, but the submitted diff has a compile-blocking missing type import; after that small repair, the solution should stay scoped to reasoning-prefaced silent replies and leave broader trailing-token policy to #69394.
