## Summary

Keep this issue open. Current main has an adjacent global `agents.defaults.reasoningDefault`, but the requested per-model `reasoningDefault` under `agents.defaults.models[...]` is still not typed, accepted by schema, documented, or consumed by runtime selection.

## What This Changes

Allow `reasoningDefault` to be configured per model in `agents.defaults.models[...]` so different selected models can default to `off`, `on`, or `stream` without per-session commands.

## Best Possible Solution

Add a first-class `reasoningDefault: "off" | "on" | "stream"` field to model entries, validate and generate schema/help/labels for it, resolve it for the selected model before global/catalog fallback, and document channel-specific `stream` behavior.
