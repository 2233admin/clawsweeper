## Summary

Keep this PR open. Current main still does not implement Feishu connection lifecycle reporting into the gateway health snapshot, and the posted PR remains functionally incomplete against the current contract: it does not pass a bound status sink from the Feishu channel start path through the single-account monitor, and its proposed event timestamps would not satisfy the current stale-socket policy without `lastTransportActivityAt`.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision, but do not merge it as-is. The best path is to consolidate with #39487/#39489 or choose one canonical PR, then implement Feishu status reporting end to end: pass a bound `ctx.setStatus` sink from `extensions/feishu/src/channel.ts` through `monitorFeishuProvider` and `monitorSingleAccount`, publish lifecycle fields from websocket/webhook start, abort, disconnect, and error paths, report `lastTransportActivityAt` from real transport activity, optionally update `lastEventAt` for app events, and add focused tests proving Feishu snapshots drive the gateway health policy.
