## Summary

Keep open. Current main still does not implement Feishu lifecycle or transport-activity reporting to the gateway health monitor, and the current PR head appears incomplete: it adds a status callback in monitor.ts/monitor.transport.ts, but does not thread it from channel.ts through monitor.account.ts into the real transport calls. The current health policy also requires lastTransportActivityAt for stale-socket detection, so lastEventAt-only reporting would not satisfy the present contract.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The best fix is Feishu-owned: pass ctx.setStatus from channel.ts through monitorFeishuProvider and monitorSingleAccount into both transport modes; publish connected, mode, lastConnectedAt, and lastTransportActivityAt on real startup/activity; publish connected:false plus lastDisconnect/lastError on abort/error/shutdown; and add focused Feishu tests proving the runtime snapshot drives the current gateway health policy. Coordinate with #39487 and #39489 so one complete canonical health-status fix lands.
