## Summary

Keep open. Current main fixes the heartbeat-aware daily/idle freshness half, but the requested turn-count TTL is still missing from the public session reset contract, schema, docs, and session reuse path; the related #45390 is open, broader, and not a merged canonical replacement.

## What This Changes

Requests a session reset policy that rotates after a configured number of conversation turns and prevents heartbeat traffic from masking stale daily/idle sessions.

## Best Possible Solution

Define one coherent core session TTL contract, then add a schema-validated and documented max-turn reset policy that counts real conversation turns, excludes heartbeat/cron/exec maintenance events, rolls before session reuse, and preserves the existing daily and idle reset behavior with focused regression coverage.
