## Summary

Current main still rejects arbitrary `bootstrap-extra-files` candidates by basename before the guarded read path, so the requested behavior is not implemented. The right next step is maintainer review of the already-open implementation PR #73917 rather than closing this tracker or queuing another repair PR.

## What This Changes

Support arbitrary workspace-relative files in bootstrap context injection instead of limiting `bootstrap-extra-files` to recognized bootstrap basenames.

## Best Possible Solution
