## Summary

Keep PR #39412 open. Current main already covers the daemon gateway-token persistence/audit side, but it still does not implement the PR's ambient provider `*_FILE` environment-variable convention. Closing as implemented would drop a substantive provider-secret capability, and merging the stale branch as-is would overlap newer gateway-token code plus unresolved branch-specific review findings.

## Best Possible Solution

Keep the PR open as useful but stale work. Rebase or rewrite it against current main, drop the now-obsolete daemon gateway-token persistence edits, and focus on the still-missing provider `*_FILE` env-var behavior if maintainers want that convention. The provider implementation should integrate with `src/agents/model-auth-env.ts`, `src/agents/live-auth-keys.ts`, and provider usage auth, preserve separators for multi-key files, keep direct env precedence, surface unreadable file misconfiguration clearly, and add focused tests for direct-env precedence, missing/unreadable files, single-key vars, list vars, prefixed vars, and provider usage auth.
