## Summary

Keep this PR open. Current main still lacks POSIX SIGHUP handling and uses an unbounded TUI drain on exit, so the linked orphaned-process report is valid, but the proposed helper can skip `tui.stop()` when `terminal.drainInput()` rejects and should be repaired before merge.

## What This Changes

The PR adds POSIX SIGHUP as a TUI shutdown signal, introduces a bounded exit cleanup path that stops status UI before teardown, and adds TUI tests plus a changelog entry for the orphaned TUI process report.

## Best Possible Solution

Keep the PR as the focused implementation path, but adjust the new exit helper so `tui.stop()` is always attempted after drain timeout or drain rejection, then merge once the targeted TUI tests and changed gate are green.

## Reproduction Assessment

Yes. The linked report gives a concrete macOS terminal-close/SIGTERM reproduction, and current main source confirms the missing SIGHUP handler plus unbounded exit drain. The review finding is reproducible with a focused unit test where `terminal.drainInput` rejects and `stop` must still be called.

## Solution Assessment

No. The SIGHUP and bounded-cleanup direction is the narrow maintainable fix, but the current helper catches drain failures before stop runs; the safer implementation should make drain best-effort while guaranteeing TUI stop cleanup.
