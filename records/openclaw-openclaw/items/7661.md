## Summary

Keep this issue open. Current main still lacks the requested `openclaw message channel create` CLI path and Slack `channel-create` provider support. Core and Discord already have partial `channel-create` vocabulary/runtime plumbing, but Slack discovery, dispatch/runtime implementation, setup scopes/docs, and the CLI subcommand are still missing.

## Best Possible Solution

Keep this open and implement it as a focused message-tool/channel-action enhancement after maintainer agreement on authority defaults: add a Slack-owned channel creation action using Slack `conversations.create`, expose it through Slack discovery/runtime behind an explicit action gate, document/setup the required `channels:manage` scope, add `openclaw message channel create` CLI mapping, and cover Slack plus CLI behavior with targeted tests. Keep the implementation inside the Slack/Discord plugin boundaries and use generic core action plumbing only where it already exists.
