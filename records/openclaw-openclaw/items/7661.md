## Summary

Keep open. Current main still lacks the requested end-to-end channel creation path: the shared action vocabulary contains `channel-create` and Discord has provider-side support, but the CLI still exposes only `message channel info|list`, Slack does not advertise or execute channel creation, and Slack setup/docs omit channel-management scopes or gates. Open PR #65065 remains relevant only for part of the Discord CLI gap and is not merged.

## What This Changes

Request to add `openclaw message channel create` / shared `channel-create` support for programmatic Slack, Discord, and similar workspace channel creation.

## Best Possible Solution

Add provider-owned channel creation once the authority model is agreed: Slack should have an explicit channel-management action gate, required `channels:manage` setup/docs, discovery/runtime support backed by Slack `conversations.create`, and targeted tests; Discord CLI wiring can be landed or superseded separately while keeping provider-specific behavior inside the Discord plugin boundary.
