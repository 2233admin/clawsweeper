## Summary

Keep this issue open. Current main still lacks the requested end-to-end channel creation path: the shared `channel-create` action name exists and Discord has provider-side support, but the CLI still exposes only `message channel info|list`, Slack does not advertise or execute `channel-create`, and Slack setup docs/scopes still omit channel-management setup. Open PR #65065 remains relevant for part of the Discord CLI gap, but it is open and does not cover the Slack workspace-creation request.

## What This Changes

Request to add message-tool and CLI support for creating messaging channels, especially Slack `conversations.create` and Discord guild channel creation, with options such as name, topic, and privacy.

## Best Possible Solution

Keep this issue open for maintainer review of the workspace-management authority model, then add provider-owned implementations: land or supersede #65065 for Discord CLI channel admin wiring, and add Slack-owned `channel-create` discovery/runtime using Slack channel creation with an explicit Slack action gate, required scope/docs, and message-tool/CLI/provider tests. Keep provider-specific behavior inside Slack/Discord plugin boundaries and use shared core action plumbing only where it already exists.
