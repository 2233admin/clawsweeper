## Summary

Keep open. Current main still has only partial Discord support for channel creation and no end-to-end `message channel create` path: the CLI exposes only channel info/list, Slack does not advertise or execute `channel-create`, and Slack setup/config lacks a channel-management gate or `channels:manage` scope. The remaining work is valid but needs maintainer product/security review because it grants agents workspace/server administration authority.

## What This Changes

Request to add native message tool and CLI support for creating channels, especially Slack `conversations.create` and Discord guild channel creation, instead of requiring direct provider API calls.

## Best Possible Solution

Keep this issue open until maintainers approve the authority model, then implement provider-owned channel creation: Slack should add a channel-management action gate, `channels:manage` setup/docs, discovery/runtime support backed by `conversations.create`, and focused tests; Discord CLI wiring should land via #65065 or a replacement while keeping provider-specific behavior inside the Discord plugin boundary.
