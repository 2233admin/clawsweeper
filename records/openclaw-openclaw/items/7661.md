## Summary

Keep open. Current main still lacks the requested end-to-end channel creation path: the shared action vocabulary includes `channel-create` and Discord has provider-side support behind its channel-management gate, but the CLI/docs still expose only channel info/list and Slack has no channel-create discovery, dispatch, action gate, or setup guidance for channel-management scopes.

## What This Changes

Add a `message channel create` or `channel-create` path so OpenClaw can create Slack/Discord channels from the CLI or shared message tool.

## Best Possible Solution

Approve a provider-owned channel-management model, then add Slack channel creation behind an explicit action gate with required Slack scope/setup docs and focused discovery/runtime tests; land or supersede the Discord CLI wiring separately while keeping provider-specific behavior inside each plugin boundary.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The feature gap is high-confidence from static current-main checks: `message channel create` is absent from CLI/docs, Slack does not advertise or execute `channel-create`, and Discord only covers the provider-side slice.

## Solution Assessment

Unclear until maintainer review. A provider-owned implementation behind explicit gates is the narrow maintainable direction, but automated implementation should wait for product/security approval because channel creation grants workspace/server administration authority.
