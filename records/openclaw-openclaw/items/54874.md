## Summary

Keep open. Current main still has the same per-input Control UI/WebChat render path described by the report, and the active related fix PR is open rather than merged.

## What This Changes

The issue reports 500 ms to 1 s delays for each keystroke in the Control UI/WebChat textarea on macOS browsers.

## Best Possible Solution

Resolve this through #62790 or an equivalent narrow Control UI fix that keeps ordinary draft typing out of full Lit host re-renders, avoids unnecessary slash-menu host updates for non-slash input, preserves explicit draft sync for send/abort/history/slash paths, and adds regression coverage for draft edits not rebuilding the heavy chat thread.
