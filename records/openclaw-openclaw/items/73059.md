## What This Changes

The request is to make the Control UI build wrapper execute normally when `scripts/ui.js` is launched through a Windows global npm junction during linked source installs.

## Best Possible Solution

Add a small script-owned direct-execution helper that compares the launch path and module path after realpath normalization, falls back to resolved paths when realpath fails, and add focused regression coverage without changing the UI app, package metadata, or the existing UI dependency install behavior.

## Reproduction Assessment

Yes. The report gives a concrete Windows linked-source command, and current `scripts/ui.js` still has the exact raw path comparison that fails when `process.argv[1]` is the junction path while `import.meta.url` resolves to the real checkout path.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize both sides of the direct-execution check through `fs.realpathSync.native` or an equivalent helper with a safe fallback; PR #52291's quoting and dependency-install changes are adjacent but do not solve this direct `node` launch path.
