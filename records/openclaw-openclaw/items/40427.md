## Summary

Keep open. Current main only partially mitigates repeated config reads through runtime snapshot/reload behavior; it does not implement this PR's low-level warning dedupe in `createConfigIO().loadConfig()`. The branch also has a concrete P2 correctness issue and overlaps broader open maintainer work in #38295, so it needs maintainer review rather than cleanup closure.

## What This Changes

The PR adds a module-level per-config-path warning fingerprint cache around `createConfigIO().loadConfig()` and a focused unit test for repeated warning loads and warning changes.

## Best Possible Solution

Land one canonical config logging design that defines warning/error dedupe scope across load, write, startup, and reload emitters, clears state across warning-to-non-warning and invalid/missing transitions, preserves current real-newline formatting, and includes regression coverage for repeated loads, writes, config changes, and warning disappearance/reappearance.
