## Summary

Current main still does not implement this PR’s central behavior: there is no postHookActions event field or drain, no blockSessionSave/sessionSaveContent session-memory policy handling, no random memory filename suffix, and the slug generator still calls the embedded agent without disableTools. The PR is active and paired with dependent open PR #38162 and umbrella issue #36671, so it should remain open for maintainer review rather than be swept closed.

## Best Possible Solution

Keep this PR open as the implementation candidate for the session-memory policy-control portion of #36671. Maintainers should either land it after review or request a narrower split for postHookActions, blockSessionSave/sessionSaveContent, slug-generation disableTools hardening, filename uniqueness, docs, and targeted tests; keep #38162 blocked until this base behavior is accepted or rejected.
