## Summary

Keep this PR open. Current main does not implement the PR’s intended shared human-day semantics: daily reset still uses host-local Date#setHours, session-memory filenames still use host-local event time rather than agents.defaults.userTimezone or transcript activity time, usage day buckets still use gateway/host day labels, and /usage cost still computes Today separately. The supplied PR discussion also shows active review follow-up on real P1/P2 edge cases rather than an obsolete or superseded change.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. The best path is to review and land this human-day helper approach, or a narrower successor, so session reset, session-memory artifacts, memory flush paths, and usage summaries share one documented day-boundary contract while preserving UTC/protocol timestamps and caller-requested usage date modes.
