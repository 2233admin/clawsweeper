## Summary

Keep this PR open. Current main still has the active-registry bypass when cfg.plugins.entries is non-empty, the existing non-speech active-registry test does not cover that case, and the PR is paired with open bug report #73793 as the implementation candidate. The security pass did not find supply-chain-sensitive changes in the supplied PR diff.

## Best Possible Solution

Keep this PR open for maintainer review. If reviewers agree that active registry compatibility is already protected by the runtime cache key and config invalidation path, land this PR or an equivalent patch with the added regression test, then let #73793 close through the merge.
