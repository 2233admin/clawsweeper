## Summary

Keep open. Current main still appends the full cumulative `chatStream` into WebChat stream segments before tool cards, while the renderer displays each segment and the live stream as separate bubbles. The PR remains a plausible implementation candidate, especially after the author addressed the Greptile review by switching to total committed length, but it needs maintainer coordination with overlapping open PRs #46985 and #47399 plus regression coverage.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best maintainer path is to land one focused WebChat streaming fix that stores only newly streamed text beyond the committed segment length, also dedupes the live preview against committed segments, and adds regression tests in `ui/src/ui/app-tool-stream.node.test.ts` and `ui/src/ui/chat/build-chat-items.test.ts` for multi-tool cumulative snapshots.
