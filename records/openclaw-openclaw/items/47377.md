## Summary

Keep open. The branch addresses part of the Web UI streaming duplication and incorporates Greptile's earlier cumulative-length feedback, but it still leaves the live preview path rendering cumulative snapshots unchanged after committed segments. An overlapping open PR, #46985, covers both the segment commit path and live preview path with focused tests, so this needs maintainer coordination rather than cleanup closure.

## What This Changes

The PR changes `ui/src/ui/app-tool-stream.ts` so a stream segment committed before a tool card stores only the suffix after the total length of earlier committed segments.

## Best Possible Solution

Land one focused Control UI streaming fix that treats cumulative assistant snapshots consistently for both committed pre-tool segments and the live preview, preferably through a shared helper or explicit committed-offset state plus regression tests for repeated text/tool/text cycles.

## Reproduction Assessment

Yes. The source path is enough to reproduce: the gateway sends cumulative `mergedText`, Control UI assigns it to `chatStream`, a tool start commits the current snapshot as a segment, and `buildChatItems` later renders the next cumulative live stream unchanged after that segment.

## Solution Assessment

No. Segment suffix slicing is necessary, but this PR is incomplete because the live preview needs the same committed-offset dedupe and focused regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P2] Deduplicate the live stream preview too:** `ui/src/ui/app-tool-stream.ts:443-451`
