## Summary

Keep open. Current main still does not implement this PR's intended WebChat segment dedupe: Gateway chat deltas are cumulative snapshots, Control UI still stores the full `chatStream` as each tool-interrupted segment, and the renderer displays those segments as separate stream bubbles. The PR is narrow and security-low-risk, but it needs review alongside overlapping open PRs #46985 and #47399 rather than cleanup closure.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best maintainer path is to land one focused WebChat fix that stores only newly streamed text beyond the total committed segment length, dedupes the live preview when needed, and adds regression tests in `ui/src/ui/app-tool-stream.node.test.ts` and/or `ui/src/ui/chat/build-chat-items.test.ts` for multi-tool cumulative snapshots.
