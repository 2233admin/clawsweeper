## Summary

Keep this PR open. Current main still has the reported Windows QMD shim gap and stale QMD fallback package name, while the latest PR diff appears narrow and I did not find a blocking correctness or security issue.

## What This Changes

The PR updates QMD Windows spawn resolution to unwrap quoted absolute `.cmd` shim entrypoints, switches QMD fallback package names to `@tobilu/qmd`, and adds focused resolver/QMD coverage.

## Best Possible Solution

Land the narrow fix after maintainer/security review and targeted validation: accept existing quoted absolute JS/EXE targets from Windows `.cmd` wrappers without shell execution, use `@tobilu/qmd` consistently for QMD fallback resolution, and keep source/package resolver behavior aligned.

## Solution Assessment

Yes. The latest PR diff is the narrowest maintainable direction I found: extend direct wrapper parsing for existing absolute entrypoints while preserving the no-shell default and correcting the documented QMD package fallback.
