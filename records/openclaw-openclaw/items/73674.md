## Summary

Keep this PR open. Current main still does not handle the reported quoted absolute QMD `.cmd` shim target, and the PR is a focused implementation candidate rather than obsolete cleanup. It still needs maintainer review because the QMD package fallback appears incomplete for the runtime execution path and the command-resolution change is security-sensitive enough to review carefully before merge.

## Best Possible Solution

Keep the PR open and request a focused maintainer revision or review: support quoted Windows absolute and UNC JS/EXE entrypoints without shell execution, keep the duplicated SDK/package resolvers in sync, update every QMD package fallback call site that needs `@tobilu/qmd`, and land with portable regression coverage plus CI.
