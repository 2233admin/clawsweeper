## Summary

Keep this PR open. Current main still has the Windows QMD gap: both Windows `.cmd` resolver copies only unwrap `%~dp0`-relative quoted targets, and QMD availability/runtime paths still use `packageName: "qmd"` while docs install `@tobilu/qmd`. The PR is focused, with no workflow/dependency/lockfile/publishing changes visible, but it touches command resolution and should get normal maintainer/security review plus CI before merge.

## Best Possible Solution

Review and land this narrow resolver/QMD fix if CI is green: support quoted absolute Windows JS/EXE shim entrypoints without shell execution, keep the source and packaged resolver copies aligned, update all QMD fallback package names to `@tobilu/qmd`, preserve the fail-closed unresolved-wrapper policy from #58436, and keep portable regression coverage across plugin SDK, memory-host SDK, and memory-core runtime paths.
