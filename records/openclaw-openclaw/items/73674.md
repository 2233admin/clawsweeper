## Summary

Keep this PR open. Current main still ignores quoted absolute `.cmd` shim entrypoints, so the reported QMD-on-Windows failure remains plausible and this PR is a focused implementation candidate rather than obsolete cleanup. The patch also touches a generic Windows spawn resolver, so it needs maintainer review and CI rather than closure.

## Best Possible Solution

Keep this PR open for maintainer review or revision: support quoted Windows absolute and UNC JS/EXE entrypoints without shell execution, keep the source and packaged memory SDK resolver copies in sync, update QMD fallback package names to `@tobilu/qmd`, and land with portable regression coverage plus CI across the generic resolver call sites.
