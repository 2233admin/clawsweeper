## Summary

Keep this PR open. Current main still lacks the quoted absolute Windows `.cmd` shim handling, still uses the stale `qmd` package fallback in QMD availability/runtime paths, and the PR remains a plausible focused fix rather than obsolete work. It still needs maintainer review because it changes a generic Windows command-resolution helper and appears to miss the extension runtime QMD manager call site already flagged in the prior ClawSweeper discussion.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a narrow fix that supports quoted absolute and UNC Windows JS/EXE shim entrypoints without shell execution, keeps the source and packaged resolver copies synchronized, updates every QMD fallback package name that should use `@tobilu/qmd` including the extension runtime manager, and adds portable regression coverage for the generic resolver plus QMD availability/runtime paths.
