Keep this PR open. Current main still reassigns `memoryFlushCompactionCount` to the post-increment compaction count after a flush-triggered compaction, while the dedup gate still skips when `memoryFlushCompactionCount === compactionCount`. The linked bug #12590 is still open, duplicate repair #69138 and earlier repair PRs are closed unmerged, and this PR is a narrow runtime/test/changelog fix with no apparent security-sensitive supply-chain changes.
