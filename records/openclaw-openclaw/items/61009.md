## Summary

Keep this issue open. Current main still has the reported docs/runtime mismatch in the sandbox-active case: the runtime rejects per-call `host=node` from `tools.exec.host=auto` when a sandbox is available, while the exec, exec-approvals, and nodes docs still say `host=node` is allowed from `auto` without that caveat.

## Best Possible Solution

Update `docs/tools/exec.md`, `docs/tools/exec-approvals.md`, and `docs/nodes/index.md` to say per-call `host=node` from `tools.exec.host=auto` is allowed only when no sandbox runtime is active, or make an explicit maintainer product decision to change the runtime and tests. The conservative path is a docs clarification plus public docs redeploy.
