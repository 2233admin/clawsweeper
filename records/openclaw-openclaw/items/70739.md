## Summary

Keep #70739 open. Current main still does not implement an SSE heartbeat for the streaming `/v1/chat/completions` or `/v1/responses` gateway paths; only the session-history SSE stream has a keepalive. The PR remains a focused gateway bug fix with relevant tests and author follow-up addressing backpressure/write-error review, though a few P2 cleanup comments should be handled before merge.

## Best Possible Solution

Keep the PR open and either merge it after addressing the remaining P2 review comments, or land an equivalent main-branch implementation: a shared, backpressure-aware SSE heartbeat helper wired into `/v1/chat/completions` and `/v1/responses`, with focused unit coverage for interval validation, cleanup, finish/close/error handling, and fake-timer restoration. Treat stream TTL/concurrency limiting as a separate maintainer decision if needed.
