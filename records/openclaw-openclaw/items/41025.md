## Summary

Keep this PR open. Current main still flattens remote markdown images in the Control UI markdown renderer, the regression test still asserts that behavior, and the served Control UI CSP/docs still preserve a no-remote-fetch image policy. The PR is a focused implementation candidate for open issue #40945, but it needs maintainer product/security direction before merge because a renderer-only change conflicts with CSP/docs and introduces arbitrary browser remote-image fetches.

## Best Possible Solution

Keep this PR open as an implementation candidate for #40945, but require maintainer-approved design before merge. A complete solution should align the markdown renderer, Control UI CSP, docs, and tests around an explicit remote-media policy, likely HTTPS-only at minimum and preferably an allowlist, opt-in setting, managed-media proxy, and/or `referrerpolicy="no-referrer"`. If maintainers choose to preserve the current no-remote-fetch policy, record that decision on #40945 or #64607 before closing implementation attempts.
