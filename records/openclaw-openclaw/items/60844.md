## Summary

Keep open. Current main still does not implement this PR’s intended behavior: heartbeat isolation remains opt-in, unset `isolatedSession` is tested as the shared main-session path, and the public docs still describe `isolatedSession` as default false. The PR is still a plausible implementation candidate, but it needs a current-main rebase/update that preserves newer base-session event-drain and isolated delivery-routing safeguards.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. Update it against current main with a narrow opt-out default (`isolatedSession !== false`), preserve current base-session event consumption and isolated delivery-routing safeguards, and add/keep tests for default isolation, explicit `isolatedSession: false`, main-session abort recovery, and cron/wake-triggered event consumption under the new default.
