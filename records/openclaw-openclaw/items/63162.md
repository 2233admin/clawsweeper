## Summary

Keep open. Current main still lacks the requested Control UI file-explorer toggle/sidebar and the backing arbitrary workspace tree/read API; the existing file surfaces are chat attachments and a narrow Agents Core Files editor. Related #61368 and open PR #62417 cover broader workspace file management, but that work is still unmerged and needs maintainer product/security review rather than cleanup closure.

## What This Changes

Add a Control UI top-bar folder button near Cron/Settings that toggles a workspace file-tree sidebar with search, refresh, open-in-chat, context actions, and drag/drop support.

## Best Possible Solution

Keep this issue open while maintainers reconcile it with #61368/#62417. If accepted, land a generic scoped workspace tree/list/read API first, then add the chosen Control UI explorer surface with search, refresh, open-in-chat behavior, and focused tests for filesystem safety and UI integration.

Not applicable as a feature request. The current-main absence check is high confidence: navigation, topbar rendering, agent tabs, chat attachments, method registry, protocol schema, docs, and exact source search all show no workspace explorer/sidebar or agents.workspace API.

## Solution Assessment
