## Summary

Keep this issue open. Current main still lacks the requested Control UI file explorer toggle/sidebar next to Cron and the chat-adjacent workflow for browsing workspace files, selecting files into chat, search/refresh, and context actions. Related #61368 and open PR #62417 are relevant broader workspace-file-management work, but #62417 is open and unmerged, and current main still exposes only browser chat attachments plus the narrow Agents > Files bootstrap-file editor.

## Best Possible Solution

Keep this open as the chat-adjacent Control UI explorer request. The best path is to first settle or land a safe generic workspace tree/list/read API through the #61368/#62417 design, then decide whether the product surface belongs in Agents > Workspace, a Chat sidebar toggle, or both. If maintainers choose the sidebar, implement a scoped explorer with refresh/search/open-in-chat behavior and tests for path safety, large trees, attachment/read-command integration, and permission boundaries.
