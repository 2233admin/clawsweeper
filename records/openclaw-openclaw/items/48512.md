- **[P1] Remove generated bundle artifacts from the branch:** `dist.bak/acp-cli-BpCQk37E.js:1`
  - body: The branch adds a large `dist.bak/` generated JavaScript bundle tree unrelated to the idle-flush fix. These executable artifacts are not reviewable at source level and should be removed from this PR.
  - confidence: 0.95
- **[P1] Run memory extraction without shell interpolation:** `src/hooks/bundled/session-memory/handler.ts:186-188`
  - body: `execAsync` runs a shell string containing a workspace script path and session identifier. Use `execFile` or argv-based spawning so crafted path or session values cannot alter the command line.
  - confidence: 0.82
- **[P1] Preserve originating route in hook context:** `src/auto-reply/reply/dispatch-from-config.ts:176-181`
  - body: The new `message:received` hook context derives conversation fields from `Surface` and `To`/`From` instead of the canonical inbound context that preserves originating route fields. Cross-provider routed sessions can therefore key hook state to the wrong conversation.
  - confidence: 0.87
- **[P2] Use remaining budget for each idle re-wait:** `src/agents/pi-embedded-runner/wait-for-idle-before-flush.ts:59`
  - body: The pending-tool loop calls `waitForAgentIdleBestEffort` with the original timeout on each pass. If an idle wait resolves late while tools remain pending, cleanup can block for another full timeout window instead of honoring the configured budget.
  - confidence: 0.95
- **[P2] Keep pending-tool probing best-effort:** `src/agents/pi-embedded-runner/wait-for-idle-before-flush.ts:74`
  - body: `hasPendingToolCalls` runs in teardown paths but is not guarded. If introspection throws during shutdown, cleanup can reject before disposal and lock release finish, so the probe should be wrapped like `waitForIdle`.
  - confidence: 0.9
- **[P2] Apply the pending hint to compaction cleanup:** `src/agents/pi-embedded-runner/compact.ts:1005`
  - body: The attempt cleanup paths are wrapped, but compaction still passes `session.agent` directly into `flushPendingToolResultsAfterIdle`. Tool-capable compaction teardown can still flush during the retry gap this PR is trying to close.
  - confidence: 0.93
- **[P2] Clear only the triggering conversation's TTL timer:** `src/hooks/bundled/cache-ttl-warning/handler.ts:219-223`
  - body: On `/new` or `/reset`, this hook clears every timer in the process-wide store. In multi-conversation gateways, one reset would remove TTL notices for unrelated chats.
  - confidence: 0.84
