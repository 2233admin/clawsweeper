## Summary

Keep this issue open. Current main still lacks the requested session-only Control UI diagnostics mode for WebSocket lifecycle, pre-auth handshake visibility, close/error history, and copyable connection logs. Existing Debug and Logs surfaces are useful after a successful connection, but they do not cover the pre-auth failure path described here. The related implementation candidate, #49274, is closed unmerged, so this remains the active tracker.

## Best Possible Solution

Keep this issue open and implement a Control UI-owned ephemeral diagnostics mode. It should record sanitized WebSocket open/error/close/retry events, connect.challenge/connect response metadata, structured connect errors, close codes/reasons, and post-auth logs.tail snippets when available, then expose a copyable drawer or overlay backed only by memory or session-scoped storage.
