## Summary

Keep open. Current main still lacks the persisted cron hook lifecycle requested here: cron job types and Gateway schemas do not accept hook fields, the scheduler/manual/startup execution paths call the core executor without pre/post/error phases, and existing plugin cron hooks are observe-only. The discussion and open PR #45465 show this is active product/security design work rather than cleanup-ready.

## What This Changes

Request to add cron job lifecycle hooks that can run custom pre, post, and error logic, including deterministic shell/webhook gates that can skip LLM-backed scheduled runs.

## Best Possible Solution

Land or explicitly supersede the open lifecycle-hooks implementation with an additive contract covering persisted job/config types, Gateway protocol schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and an operator-controlled security model for user hooks.
