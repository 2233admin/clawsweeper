## Summary

Keep open. Current main has adjacent updater, backup, health/status, doctor, auto-update, and package-staging safety primitives, but it still does not provide the requested opt-in safe-update workflow with update-tied backups, retention, channel or real-agent smoke checks, and automatic rollback or state restore.

## What This Changes

Requested an opt-in safe-update workflow that snapshots OpenClaw state before updating, runs post-update health and smoke checks, and rolls back or restores on failure.

## Best Possible Solution

Make a maintainer decision on whether safe update belongs as a first-class core updater mode. If accepted, implement it as an explicit orchestrator around existing update, backup, doctor, health/status, channel-status, Control UI, and package rollback primitives, with retention controls, secret-preserving backup behavior, gateway/channel/real-agent smoke checks, and controlled downgrade plus state restore semantics.

## Security Review

Status: needs_attention

Summary: No PR diff was reviewed, but the requested feature is security-sensitive because it would archive local state and may automate downgrade plus restore after migrations.
