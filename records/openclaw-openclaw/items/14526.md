## Summary

Keep open. Current main has adjacent updater, backup, doctor, health/status, and manual rollback primitives, but it still does not implement the requested opt-in safe-update workflow tying pre-update snapshots, retention, gateway/channel/real-agent smoke checks, and automatic downgrade plus state restore into one operator-facing path. The issue discussion also contains production rollout evidence and a linked external auto-update kit, while maintainer-facing comments explicitly kept the broader request alive.

## What This Changes

Request an optional safe self-update mode that creates pre-update backups, runs post-update gateway/channel/agent health checks, and offers or performs rollback/restore with retention and secret-handling policy.

## Best Possible Solution

Make an explicit maintainer decision on whether safe update belongs in core. If accepted, implement it as a core updater mode that orchestrates existing update, backup, doctor, status/health, channel-status, Control UI update, and package rollback primitives; adds update-tied backup creation with retention controls; runs gateway, channel, and real-agent smoke checks; and defines controlled downgrade plus state restore behavior with secret-preservation guarantees.
