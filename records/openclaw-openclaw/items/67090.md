## Summary

Keep this PR open. Current main still implements and documents post-compaction AGENTS.md extraction as an H2/H3 contract, while this PR proposes widening that contract to H1-H4. The change is small and not security-sensitive, but the maintainer's request for a real H1 case and H4 rationale remains unresolved.

## Best Possible Solution

Keep this PR open for maintainer/product decision. The author should answer the maintainer's concrete-case question. If H1 support is accepted, narrow the supported heading levels to the intended contract and update the parser, regression tests, docs/gateway/config-agents.md, config type comments, generated schema/help, and public docs text together. If no real H1 workspace case exists, maintainers can close manually as not planned after that clarification.
