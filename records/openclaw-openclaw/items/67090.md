## Summary

Keep this PR open for maintainer review. Current main still implements and documents the post-compaction AGENTS.md extraction contract as H2/H3, while this PR broadens that contract to H1-H4; a maintainer already asked for a concrete H1 case and H4 rationale, and that product/contract question is still unresolved.

## What This Changes

This PR widens `extractSections` from H2/H3 matching to H1-H4 matching and adds regression tests for H1/H4 extraction, H1 boundaries, and nested subheadings.

## Best Possible Solution

End with one maintainer-approved heading-level contract where parser behavior, regression tests, config comments, generated help/schema text, and public docs agree. If the real H1 case is accepted, support only the intended additional levels across the shared helper and update the documented config surfaces together; otherwise keep the H2/H3 contract and close the PR manually as not planned.
