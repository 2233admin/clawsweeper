## Summary

Keep open. Current main still deduplicates Feishu final replies by raw string equality, so the PR’s normalized final-text dedupe for CRLF/LF, trailing whitespace, outer whitespace, and zero-width-character variants is not implemented or superseded. The PR is a focused Feishu plugin change and its diff does not show a security or supply-chain regression.

## Best Possible Solution

Keep this draft PR open for maintainer review. The best path is to land or recreate the focused Feishu dispatcher change on current main: add a Feishu-owned normalized final-text fingerprint for both delivered-final recording and duplicate-final lookup, keep exact distinct payloads deliverable, and retain regression coverage for CRLF/LF, trailing whitespace, outer whitespace, and zero-width-character variants.
