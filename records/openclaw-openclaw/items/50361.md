## Summary

Keep open. Current main only suppresses byte-for-byte duplicate Feishu final text; it does not implement this PR's normalization for CRLF/LF, trailing whitespace, or zero-width-character variants. Related cleanup reports support that the narrower exact duplicate fix shipped, but not this normalization gap.

## Best Possible Solution

Review and either land this focused Feishu dispatcher change or recreate it on current main: add a normalized final-text fingerprint for duplicate detection and recording, keep the behavior scoped to delivered final text keys, and add the proposed regression test for CRLF/trailing-whitespace/zero-width variants. Do not close as implemented until that behavior exists on main.
