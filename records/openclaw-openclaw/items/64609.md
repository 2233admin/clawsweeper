## Summary

Keep open. Current main still drops channel-level Telegram group/topic prompt and skill config from the merged account snapshot in multi-account mode, while the Telegram mention policy still falls back to the same channel-level groups block; the only linked fix PR, #64650, is closed and unmerged.

## What This Changes

The issue reports that Telegram multi-account setups apply channel-level group `requireMention` but silently ignore channel-level group/topic `systemPrompt`, `skills`, and `topics` when building message context.

## Best Possible Solution

Implement a Telegram-owned context resolver fallback so group/topic prompt and skill lookup uses account-scoped groups first and channel-level groups only when the account has no groups, while preserving the existing multi-account routing guard in account resolution. Add a regression test for two Telegram accounts with channel-level group/topic `systemPrompt` and `skills`, and clarify docs or doctor output only if maintainers intentionally choose not to support that config shape.
