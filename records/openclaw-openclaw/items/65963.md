## Summary

Keep this issue open. Current main has implemented the high-value `dreaming.model` knob, deletes future Dream Diary subagent sessions after accepted runs, scrubs some stale/orphaned dreaming artifacts, and now bounds detached narrative concurrency, but it still lacks the explicit operator cleanup and picker controls requested for surviving/legacy artifacts: no `sessions purge --kind dreaming`, no dreaming-specific retention knob, and no CLI/UI include/hide/collapse control for dreaming sessions.

## What This Changes

The issue requests reducing Dreaming session sprawl by adding automatic cleanup or purge behavior, a cheaper model override, burst/concurrency control, and session-list/session-picker filtering for dreaming artifacts.

## Best Possible Solution

Keep the shipped model override, future-session deletion, scrubber, and bounded narrative queue, then narrow the remaining work into explicit cleanup UX: either add a memory-core-owned retention/purge path for generated dreaming sessions and transcripts plus a session picker/list filter, or split those leftovers into smaller canonical issues before closing this one.
