## Summary

Keep open. The reporter's latest update says their local break was fixed upstream in the OpenAI-compatible backend/proxy, so this is not currently proven as a Feishu rendering bug. However, current main still has a narrow OpenClaw compatibility gap in the same request-shaping surface: `buildOpenAICompletionsParams` always sends `stream_options.include_usage` even though model compat can explicitly say streamed usage is unsupported. No current source, docs, changelog entry, or related merged fix shows that gap has been resolved.

## Best Possible Solution

Keep this open and implement a narrow OpenAI-completions request-shaping fix: honor the resolved `supportsUsageInStreaming` capability, including explicit `false`, when deciding whether to send `stream_options.include_usage`. Preserve the field for OpenAI/native endpoints and manifest-declared local providers that support streamed usage, add regression coverage for unsupported custom OpenAI-compatible providers, and avoid changing Feishu card streaming policy.

## Clownfish Work Prompt

Fix the OpenAI-compatible streaming compatibility gap described here. Current `buildOpenAICompletionsParams` in `src/agents/openai-transport-stream.ts` always sends `stream_options: { include_usage: true }`, ignoring `compat.supportsUsageInStreaming`, including explicit `false` preserved by `normalizeModelCompat`. Make request shaping compat-aware so custom OpenAI-compatible endpoints can omit `stream_options.include_usage` when unsupported, while preserving it for OpenAI/native endpoints and manifest-declared local providers such as vLLM, SGLang, LM Studio, and known local backends that advertise streamed usage. Add or adjust tests in `src/agents/openai-transport-stream.test.ts`, `src/agents/openai-completions-compat.test.ts`, and/or `src/agents/model-compat.test.ts` to cover: explicit `compat.supportsUsageInStreaming: false` omits `stream_options`; explicit true and known local/native providers keep it; generic custom providers do not regain it accidentally. Do not change Feishu card streaming policy or broad channel behavior. Add a changelog fix entry crediting @CharyeahOwO because this is user-facing OpenAI-compatible streaming behavior.
