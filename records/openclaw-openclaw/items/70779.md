## Summary

Keep open. #70779 is a fresh, detailed regression report with Feishu and Control UI reproduction, and current `main` still unconditionally sends `stream_options.include_usage` for `openai-completions` requests. I found no current-source, test, docs, or changelog evidence that the reported custom Gemini-compatible streaming failure has been fixed or superseded.

## Best Possible Solution

Keep #70779 open. The likely maintainer path is to make `openai-completions` streamed-usage request shaping compatibility-aware: honor an explicit `compat.supportsUsageInStreaming: false` or add a targeted provider/runtime hook so custom Gemini-compatible OpenAI endpoints can omit `stream_options.include_usage` when it breaks incremental SSE delivery. Add a regression test for a custom Gemini-compatible OpenAI completions model that verifies incremental `text_delta` handling without forcing the streamed-usage final-chunk behavior, and document the supported workaround once the toggle is real.
