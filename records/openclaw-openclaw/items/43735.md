## Summary

Keep open. Current main still has a real stale skill-snapshot path: workspace skills are documented and loaded, but CLI/gateway persisted session snapshots are still refreshed from process-local version counters plus agent skill filters, with no durable inventory fingerprint to catch skills added or changed before a restart. The direct stale-restart PR is closed unmerged, while the open prompt-path PR is adjacent and does not rebuild stale inventories.

## What This Changes

The issue asks for valid workspace skills under the OpenClaw workspace skills directory to appear consistently in agent `<available_skills>` context instead of disappearing from stale or partial session skill snapshots.

## Best Possible Solution

Add a deterministic, durable skill-inventory fingerprint to session skill snapshots and compare it in CLI, gateway, and cron snapshot resolution so restarts self-heal when effective skill roots, files, plugin skill dirs, config filters, eligibility inputs, or prompt-limit-relevant inventory changes. Preserve current precedence, realpath containment, allowlists, eligibility gates, and prompt limits.
