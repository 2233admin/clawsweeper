## Summary

Keep #57259 open. Current main has image-capable GitHub Copilot dynamic model definitions and an explicit generic image-model path, but GitHub Copilot is still not registered or documented as a media-understanding image provider. Auth/configuring Copilot alone does not put it in the media-understanding provider registry or auto image fallback path.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the bundled GitHub Copilot plugin: add a media-understanding image provider registration or manifest metadata with suitable default model handling, wire it through the existing generic image runtime, add focused registry/auto-routing tests, and update the media-understanding and GitHub Copilot provider docs. If maintainers only want explicit Copilot image use, document the `tools.media.image.models` or `agents.defaults.imageModel` workaround instead of closing this as implemented.
