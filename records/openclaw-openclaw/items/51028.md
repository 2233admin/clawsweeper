## Summary

This issue should stay open. Current main still sorts and refreshes the Sessions panel from the raw `updatedAt` session row timestamp, while the gateway and UI contracts do not expose `lastMeaningfulActivityAt`, `lastVisibleConversationAt`, an activity reason, or parent rollup of child activity. The sub-agent label/readability concern was split into #51030 and does not resolve the remaining sorting/activity contract here.

## Best Possible Solution

Keep this issue open and implement a gateway-backed session activity contract: persist and expose raw event time, meaningful activity time, visible conversation time, and a concise activity reason; classify heartbeat, cron, and internal/background writes so they do not bubble sessions; roll child/sub-agent meaningful activity up to the parent row for sorting; then update `sessions.list`, `sessions.changed`, docs, and the Control UI to sort by the meaningful timestamp and display the reason line. Treat #51030 as the already-split label/readability follow-up.
