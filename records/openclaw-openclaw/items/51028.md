## Summary

Keep open. Current main still exposes and sorts the Sessions panel by raw session-row `updatedAt`; it does not expose `lastMeaningfulActivityAt`, `lastVisibleConversationAt`, an activity reason, or a parent rollup timestamp for child/sub-agent activity. The sub-agent label/readability part was split to #51030 and closed separately, but that does not resolve this sorting/activity contract.

## Best Possible Solution

Keep this issue open and implement an additive gateway-backed session activity contract: persist/expose raw event time, meaningful activity time, visible conversation time, and a concise activity reason; classify heartbeat/cron/internal/background writes so they do not bubble sessions; roll child/sub-agent meaningful activity up to the parent row for default ordering; then update `sessions.list`, `sessions.changed`, Control UI types/sort/display, tests, docs, and changelog. Treat #51030 as the already-split label/readability follow-up rather than a resolution of this sorting work.
