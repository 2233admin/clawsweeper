## Summary

Keep this issue open. Current main has the underlying local-model, model-fallback, sub-agent model override, timeout, sandbox, and tool-policy pieces, but it still lacks the requested first-class cookbook tying those pieces into a cloud-orchestrator plus local text-only worker pattern with sample config and worker prompting guidance.

## Best Possible Solution

Keep this open and add a concise cookbook section, likely linked from `docs/gateway/local-models.md` and `docs/tools/subagents.md`, for a hosted/cloud primary orchestrator with local text-only sub-agent workers. Include sample config for hosted primary model, local provider metadata with `input: ["text"]` and `compat.supportsTools: false` when needed, `agents.defaults.subagents.model`, `runTimeoutSeconds`, `sandbox: "require"`, safe per-subagent tool-policy narrowing, and example worker wording that asks the local model to return plain text instead of attempting autonomous tool use.
