## Summary

Current main still does not implement the requested path-scoped RWX permission map across exec and file tools. The implementation PRs discussed in the issue are closed unmerged, and the current code remains split between command allowlist-based exec approvals and a boolean workspace-only file-tool guard.

## Best Possible Solution

Keep this issue open until maintainers either ship a backward-compatible filesystem permission contract covering file tools and exec, or explicitly fold this exact scope into a canonical security/permissions design issue. The implementation should update config schema/help/docs, preserve existing exec approval behavior, and include focused tests for read/write/edit/apply_patch plus exec path handling.
