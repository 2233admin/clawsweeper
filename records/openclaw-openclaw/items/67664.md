## Summary

Keep this PR open. Current main still persists `DEFAULT_CONTEXT_TOKENS` into session-store entries when context-window lookup is unresolved, so the targeted bug remains real. The PR is a plausible implementation candidate, but it is stale and should be rebased/reviewed because current main has runtime `agentMeta.contextTokens` handling and #69347 cost snapshot behavior that must be preserved.

## Best Possible Solution

Keep this PR open as the implementation candidate. Maintainers should rebase it onto current main, keep the no-fallback session-store persistence behavior, preserve current runtime context-token and cost snapshot behavior, split or justify unrelated extension/type cleanup, then run the changed gates before merge.
