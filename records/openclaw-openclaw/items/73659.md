## Summary

Keep this PR open. Current main still does not implement the PR's intended SSH-transport URL mapping: the gateway connection builder ignores `gateway.remote.transport`, selects `gateway.remote.url` directly, and runs the plaintext non-loopback WebSocket security check against that direct URL. The PR is a focused implementation candidate for the related open SSH-transport bug, but it needs maintainer/security review because it changes a client-side security guardrail.

## Best Possible Solution

Keep the PR open for normal maintainer review. The likely best path is to land a narrowly tested fix that makes SSH transport use an effective loopback tunnel endpoint while preserving the existing direct-transport security rejection, and to coordinate it with the related open SSH transport bug #26250.
