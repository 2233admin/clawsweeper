## Summary

Keep open. The underlying bug is still present on current main, and this PR is a useful attempt, but the proposed resolver is added in the completion generator module while the runtime install/check path still owns and hardcodes the profile paths; the doctor message change can also point users at a different profile file than the installer actually modified.

## What This Changes

The PR adds an environment-aware shell profile resolver and tests in the completion CLI module, then uses that resolver in doctor completion messages.

## Best Possible Solution

Revise the PR so profile-path resolution lives in the runtime completion module used by installCompletion, isCompletionInstalled, usesSlowDynamicCompletion, doctor, update, and setup onboarding. Preserve the intended bash fallback, keep completion script generation unchanged, add focused runtime tests for ZDOTDIR, XDG_CONFIG_HOME, and bash profile selection, then land it as the fix for #63069 while leaving any broader PowerShell onboarding polish to #44296 if needed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Wire the resolver into the runtime path:** `src/cli/completion-cli.ts:82-85`
  - body: The new resolver is exported from completion-cli.ts, but the code that actually checks and writes profile files is still in completion-runtime.ts and still hardcodes HOME/.zshrc and HOME/.config/fish/config.fish. As a result, `openclaw completion --install`, update, and doctor still write/check the wrong file for ZDOTDIR/XDG_CONFIG_HOME users, so this PR does not fix the reported bug. Move or expose the resolver from the runtime module and use it from install/check paths.
  - confidence: 0.94
- **[P2] Keep doctor hints matched to the installed profile:** `src/commands/doctor-completion.ts:111-114`
  - body: These notes now call the new env-aware resolver after installCompletion runs, but installCompletion still writes to its own hardcoded runtime path. With ZDOTDIR or XDG_CONFIG_HOME set, doctor can tell the user to source a file that was never modified, making the repair guidance incorrect until the installer and message share the same resolver.
  - confidence: 0.9
