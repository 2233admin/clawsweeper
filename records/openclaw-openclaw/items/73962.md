## Summary

Keep open: current main still has the old assistant-media supplement path that can append a text-only `gateway-injected` assistant message when stale or unavailable media cannot be embedded, and the proposed patch targets that exact failure mode without changing broader delivery or security surfaces.

## What This Changes

The PR adds a gateway guard that only appends an agent-run assistant-media transcript row when persisted content contains non-text media, adds a missing-media regression test, and records the fix in `CHANGELOG.md`.

## Best Possible Solution

Ship this narrow guard or an equivalent one with the missing-media regression test and changelog entry, leaving existing successful audio/image transcript injection and non-agent command TTS behavior intact.

## Reproduction Assessment

Yes. The linked bug report gives a concrete stale/missing TTS media path, and the source path on current main shows that a media-bearing payload with no embeddable content can still append a text-only `:assistant-media` assistant transcript row.

## Solution Assessment

Yes. Requiring persisted non-text media before the agent-run assistant-media supplement writes a transcript row is the narrowest maintainable fix; normal model transcript text and non-agent command TTS remain on their existing paths.
