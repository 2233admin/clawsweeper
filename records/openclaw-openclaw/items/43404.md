## Summary

Keep this PR open. Current main still has both bugs the PR targets: existing-cron suppression is session-key-only, and compaction retry still clears successfulCronAdds. The PR remains a focused implementation candidate for the linked open bug #43292, but its latest caller-side agent-id fallback still deserves maintainer review before landing.

## Best Possible Solution

Keep this PR open as the active implementation candidate for linked bug #43292. The best maintainer path is to preserve successfulCronAdds across compaction retry, extend existing-cron suppression to enabled sessionTarget="isolated" jobs by agentId only when the caller has reliable agent identity, keep session-bound matching by sessionKey, and retain regression coverage for session-bound, isolated, disabled, empty-store, store-error, missing-agentId, missing-sessionKey, and main-without-sessionKey cases.
