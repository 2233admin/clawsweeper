## Summary

Keep open. Current main still has both reported reminder-guard gaps, and this PR is a useful narrow fix, but the latest head still introduces a regression by deriving `main` from an absent session key and using that to suppress the warning for unrelated isolated main-agent cron jobs.

## What This Changes

This PR changes reminder-guard matching to include enabled isolated cron jobs by agent id, preserves the cron-add counter across compaction retry, and adds helper-level reminder-guard regression tests.

## Best Possible Solution

Land this PR or an equivalent focused replacement after the call site passes an agent id only when the runner has reliable session identity, while keeping the `sessionTarget: "isolated"` matcher guard, preserving `successfulCronAdds` across compaction retry, and adding a `runReplyAgent` regression for omitted `sessionKey` plus an isolated `main` job.
