## Summary

Current main still only buffers skipped WhatsApp mention-gated group messages as pending context and routes model selection by channel/group/session, not by mention state. The requested `mentionModel`, `backgroundModel`, and `mentionContextMessages`-style per-group controls are absent from the strict WhatsApp config schema and docs, so the prior keep-open review in the issue thread still matches current main at a820a307dfeb950fa633c0ae6fb2386a35a91991.

## Best Possible Solution

Keep this open until the WhatsApp/group pipeline supports mention-state-aware model routing plus configurable per-group mention context sizing, or until OpenClaw adds a deliberate trusted pre-gating plugin/API hook that can process unmentioned group messages safely. The implementation should update WhatsApp config types/schema/help/docs and add focused tests for skipped-message buffering, mentioned-message routing, background-model behavior, and per-group context limits.
