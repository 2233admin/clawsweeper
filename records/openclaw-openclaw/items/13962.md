## Summary

Keep this issue open. Current main still buffers unmentioned WhatsApp group messages as pending context and returns before the reply/model path, while model routing remains keyed by channel/group/session rather than mention state. The requested `mentionModel`, `backgroundModel`, and per-group `mentionContextMessages` controls are absent from the strict WhatsApp config schema and docs.

## Best Possible Solution

Keep this open for maintainer product/design follow-up. The best implementation would define a WhatsApp/group mention-state routing contract, then update the WhatsApp config schema/types/help/docs plus focused tests for skipped-message buffering, mentioned-message routing, background-model behavior, per-group context limits, and precedence with existing model overrides.
