## Summary

Keep this PR open. Current main still accepts only the new ACP stream config names and has no runtime/doctor migration for `acp.stream.maxTurnChars` or `maxToolSummaryChars`; generic unknown-key cleanup would remove those legacy values instead of preserving them. The PR is not merge-ready because it targets the old monolithic migration file layout, but the intended fix remains useful and the author already addressed the bot-reviewed no-replacement cleanup edge case.

## Best Possible Solution

Keep the PR open and ask for a rebase into the current modular doctor/runtime migration system, likely as an ACP runtime migration registered from `src/commands/doctor/shared/legacy-config-migrations.runtime.ts`. Preserve tests for rename, non-overwrite, no-replacement removal, wrong-typed removed keys, and normal config validation after migration. Also clarify the `Fixes #35957` scope mismatch before merge.
