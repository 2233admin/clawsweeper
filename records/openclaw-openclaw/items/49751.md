## Summary

Keep open. Against the provided current main checkout, the timeout-budget part is already present, but the central semantic change from this PR is not: current main still waits for async connect auth before clearing the pre-auth WebSocket handshake timer, so the reported slow-auth race can still happen. The PR remains active, unmerged, and contains the missing connect-auth watchdog plus regression coverage and docs.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent fix on main: clear the pre-auth challenge timer after a syntactically/protocol-valid connect frame, bound stalled async connect auth before registration, keep the regression coverage and protocol docs, and preserve the original contributor credit.
