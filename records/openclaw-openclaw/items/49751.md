## Summary

Keep open. Current main has shipped the shared 10s pre-auth/client challenge timeout from #49262, but it still does not implement this PR's remaining semantic fix: clearing the valid-connect pre-auth timer before async auth and bounding that auth phase separately. The PR also has recent maintainer repair activity, so it remains an active implementation candidate rather than cleanup noise.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent fix on main: clear the pre-auth timer once a connect frame is syntactically and protocol-valid, add a bounded connect-auth timer for stalled async auth before registration, keep the gateway regression coverage and protocol docs, and preserve the original contributor credit.
