## Summary

Keep open. Current main has already absorbed the shared 10s gateway/client handshake timeout work, but it still has not implemented this PR's remaining semantic fix: clearing the pre-auth WebSocket timer after a valid connect request and replacing that safety net with a bounded async connect-auth timer. The PR also has fresh maintainer repair activity and merge-review labels, so it remains an active implementation candidate rather than cleanup noise.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent fix on main: clear the pre-auth timer once the connect frame is syntactically/protocol valid, add a bounded connect-auth timer for stalled async auth before registration, keep the regression test and protocol docs, and preserve the original contributor credit.
