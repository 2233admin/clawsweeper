## Summary

Keep open: current main has not implemented the installer no-op skip, and the proposed patch still has a maintainer-reproduced correctness problem in the install path. The skip decision can be satisfied by an unrelated PATH binary instead of the npm-managed package tree, so this is not ready to close or merge as-is.

## What This Changes

This PR adds `--force` / `OPENCLAW_FORCE_INSTALL` and a pre-install version comparison to `scripts/install.sh` so npm installs can be skipped when the detected local version matches the resolved target version.

## Best Possible Solution

Keep the feature direction but make the skip depend on the npm-managed installation itself, either by reading the global npm package `package.json` or proving the resolved binary belongs to npm's global bin/root. Preserve shared finalization behavior and add targeted installer tests before relying on install smoke coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Verify the npm package before skipping install:** `scripts/install.sh:1990`
  - body: The skip path trusts `resolve_openclaw_version()`, which prefers any `openclaw` found on `PATH` before checking the npm global package tree. A wrapper or stale binary that prints the target semver can set `INSTALL_SKIPPED=1` even when the npm-managed `openclaw` package is missing, leaving the official install unrepaired. Base the skip on `$(npm root -g)/openclaw/package.json` or verify the binary belongs to npm's global install before returning.
  - confidence: 0.94

## Security Review

Status: needs_attention

Summary: The installer diff has a concrete install-integrity concern because the no-op check can trust an arbitrary PATH binary instead of the npm-managed package.

Concerns:

- **[medium] Skip can trust an arbitrary PATH binary:** `scripts/install.sh:1990`
  - body: Because the new skip check accepts the version from `resolve_openclaw_version()`, a different `openclaw` executable earlier on PATH can suppress the intended npm install when it reports the target semver. For an installer path, that is a supply-chain/integrity regression: the official package may not be installed or repaired.
  - confidence: 0.9
