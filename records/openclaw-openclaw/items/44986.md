## Summary

Keep open. Current main still does not implement the requested installer no-op, and the linked feature issue was closed as superseded by this PR, but the branch is merge-conflicted and the proposed skip path has concrete correctness and installer-integrity issues that should block merge as written.

## What This Changes

The PR adds `--force` / `OPENCLAW_FORCE_INSTALL` and a same-version no-op path to the npm branch of `scripts/install.sh`, leaving git installs unchanged.

## Best Possible Solution

Keep the optimization direction, but make the no-op decision depend on the npm-managed install itself, such as reading the global npm package `package.json` or proving the resolved binary belongs to npm's global bin/root. The installer should skip only the package download/install step, continue shared finalization, preserve current explicit package-spec handling, and add focused regression tests for fake-PATH/no-package and skipped-finalization cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Verify the npm package before skipping install:** `scripts/install.sh:1989-1990`
  - body: The skip path accepts the version from `resolve_openclaw_version()`, which can use `OPENCLAW_BIN` or any `openclaw` on `PATH` before npm package metadata. A wrapper or stale binary that prints the target semver can set `INSTALL_SKIPPED=1` even when npm's global `openclaw` package is missing, so the installer never repairs the official package.
  - confidence: 0.94
- **[P2] Keep finalization on skipped installs:** `scripts/install.sh:2340-2341`
  - body: Returning when `INSTALL_SKIPPED=1` bypasses shared finalization, including duplicate install diagnostics, PATH checks, gateway refresh, doctor/onboarding checks, and the final resolver path. Same-version reruns should skip only the npm install work and then continue through finalization.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff introduces an installer integrity concern because the no-op decision can trust an arbitrary PATH binary instead of the npm-managed package.

Concerns:

- **[medium] Skip can trust an arbitrary PATH binary:** `scripts/install.sh:1989`
  - body: Because the skip check accepts the version returned by `resolve_openclaw_version()`, a different `openclaw` executable earlier on `PATH` or supplied through `OPENCLAW_BIN` can suppress the intended npm install when it reports the target semver. For the official installer path, that can leave the npm package missing or unrepaired.
  - confidence: 0.9
