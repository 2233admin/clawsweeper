## Summary

Keep this PR open. Current main still has the Telegram regression that the PR targets: any non-off `replyToMode` builds a native-quote candidate for the inbound message and then disables answer draft streaming whenever that candidate map is non-empty. The PR remains the active implementation candidate for the open #73505 report, and the author has already addressed the prior changelog attribution note.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for #73505. Review the updated changelog credit, refresh/rebase if GitHub still reports a mergeability problem, run targeted Telegram validation plus the changed gate, and land it if those checks pass.
