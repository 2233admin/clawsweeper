## Summary

Keep this PR open. Current main still has the reported Telegram regression: with `replyToMode !== "off"`, the dispatcher adds a native quote candidate for the inbound message itself and then disables answer draft streaming whenever the candidate map is non-empty. The PR is a focused implementation candidate for the still-open #73505, not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open, review it as the active implementation candidate for #73505, ask for the changelog credit fix, then land it if Telegram targeted tests and changed-lane validation pass. Current main does not yet solve the reported behavior.
