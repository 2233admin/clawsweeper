## Summary

Keep this PR open. Current main still has Azure Speech as a TTS-only bundled plugin, while Voice Call already supports plugin-owned realtime transcription providers. The related #68697 registry work is present, so this PR is a concrete implementation candidate rather than obsolete. The latest PR head still appears to contain the lazy-connect first-frame loss called out in review, and the new Microsoft Speech SDK dependency needs normal maintainer/security review before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to fix the lazy-connect audio delivery bug and strengthen the test to assert first-frame delivery/no unexpected onError, then review the Microsoft SDK dependency and run targeted Azure Speech, Voice Call, config/docs, lint, and extension typecheck gates before considering merge.

## Risks / Open Questions

- The PR head still appears to drop or error the first audio frame in the lazy-connect path, which is user-visible data loss for Voice Call streaming callers that do not call connect() explicitly.
- The new microsoft-cognitiveservices-speech-sdk runtime dependency and lockfile additions need supply-chain, license, install footprint, and lazy-load review before merge.
- The implementation handles Azure credentials, custom endpoints, live call audio, transcript callbacks, and error paths; those need focused review to avoid leaks or reliability regressions.
