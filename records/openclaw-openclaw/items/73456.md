## Summary

Keep open. Current main still lacks Azure Speech realtime transcription for Voice Call, so the PR is not obsolete, but the latest PR head has a close-during-connect race in the new Azure session implementation that should block merge until fixed.

## What This Changes

The PR adds an Azure Speech realtime transcription provider for the Voice Call plugin, updates Azure Speech manifest metadata, docs, tests, and the pnpm lockfile for the Microsoft Speech SDK dependency, with small incidental formatting changes in LINE and onboarding files.

## Best Possible Solution

Keep the provider plugin-owned implementation, but revise the Azure realtime session so a closed session cannot finish connecting or leak SDK resources, then rebase and validate the Azure Speech, Voice Call, extension type/lint, config docs/schema, and dependency packaging surfaces before merge.

## Reproduction Assessment

Yes. A code-level reproduction is to create the Azure Speech session, start connect() through Voice Call's non-blocking STT path, close the session before the SDK load/start promise resolves, and observe that connect() can still create/start the recognizer because it never rechecks closing.

## Solution Assessment

No, not as-is. The product direction and plugin boundary are the right way to add Azure Speech STT, but the session lifecycle needs a close-during-connect guard and test before this is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Stop connecting after the session has been closed:** `extensions/azure-speech/realtime-transcription-session.ts:245-246`
  - body: Voice Call starts STT with `sttSession.connect().catch(...)` and can call `close()` on a quick stream stop before that promise resolves. This `connect()` path still creates/starts the Azure recognizer and then sets `connected = true` without checking `closing`, so a closed call can leave SDK resources running. Check `closing` before and after the async start, close any created stream/recognizer, and add a regression test for close while connect is pending.
  - confidence: 0.88
