## Summary

Keep this PR open. Current main still has Azure Speech as a TTS-only bundled plugin, while the Voice Call realtime transcription provider API and selection path already exist. The PR is a concrete implementation against the current plugin-owned provider pattern, but it still needs maintainer review rather than cleanup closure, especially because the current PR diff still appears to contain the lazy-connect first-frame data-loss issue called out in review.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to fix the lazy-connect audio delivery bug, complete a supply-chain and credential-path review for the Microsoft Speech SDK integration, then run the targeted Azure Speech, Voice Call, config/docs, lint, and extension typecheck gates before considering merge.
