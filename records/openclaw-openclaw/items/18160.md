## Summary

Keep this issue open. Current main still has no direct cron exec/shell payload: the cron contract, Gateway schema, service validator, runtime dispatch, CLI, Control UI, agent-facing cron tool, and docs all remain limited to main-session system events or isolated/current/session agent turns. The discussion also points to this issue as the canonical tracker, with duplicate reports closed toward it and PR #51276 closed unmerged.

## Best Possible Solution

Keep this issue open as the canonical core cron direct-exec tracker. The best path is an additive exec or shell cron payload contract across cron types, Gateway schema/validators, create/update normalization, runtime dispatch, CLI add/edit, Control UI, agent-facing cron tool guidance, public docs, and focused tests, using a controlled process runner integrated with existing run-history, failure-alert, timeout, cancellation, security, and delivery behavior. PR #51276 can remain useful design context, but a reviewed implementation still needs to land on main.

## Risks / Open Questions

- Closing this now would drop the canonical tracker while duplicates have been closed toward this issue and the only direct implementation PR is closed unmerged.
- Broader related requests such as #29907 and #37131 overlap with conditional pipelines or direct tool invocation, but they do not replace this narrower cron exec payload request.
