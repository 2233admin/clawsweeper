## Summary

Keep open. Current main still lacks a zero-LLM cron exec payload across the cron contract, Gateway validation, scheduler runtime, CLI, Control UI, and docs; the issue discussion adds active production datapoints and a closed-unmerged implementation PR rather than evidence of a shipped fix.

## What This Changes

Request an additive cron payload mode that runs a configured command directly on schedule, captures stdout/stderr, and optionally delivers output without starting an LLM agent turn.

## Best Possible Solution

Keep this as the canonical core tracker and land an additive, owner-reviewed cron payload kind for deterministic command execution, covering cron types, Gateway schema, normalization/validation, scheduler dispatch, run logs and error accounting, delivery, CLI, Control UI, docs, tests, and explicit secops policy for scheduled host commands.
