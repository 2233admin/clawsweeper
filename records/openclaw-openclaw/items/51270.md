## Summary

Keep this PR open. Current main still does not implement the PR’s intended TUI behavior: `/model list` and `/model status` are parsed as `/model` with non-empty args and routed to the session model patch path. The linked user-facing bug #51126 is closed for the broader chat/directive path, but the TUI-specific gap remains in current main.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best direction is to make the TUI treat `/model list` and `/model status` as reserved subcommands with regression tests, while preserving the shared chat command semantics for agent-aware model listing and detailed status diagnostics where possible.
