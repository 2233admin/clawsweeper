## Summary

Keep open. Current main still has the undefined-result crash path reported in the linked Telegram regression, and this PR is a focused, likely-correct fix, but it overlaps other open fixes and needs maintainer selection plus API-contract judgment before merge.

## What This Changes

This PR normalizes nullish or non-object plugin command results in the shared executor, generic command dispatch, and Telegram native command path, with regression tests for undefined results.

## Best Possible Solution

Land one canonical narrow fix that normalizes plugin command results at the shared executor boundary, keeps generic and Telegram native dispatch resilient, and then close or supersede the overlapping PRs. If void command handlers are accepted as supported behavior, align the plugin SDK contract in the chosen PR or a small follow-up.

## Reproduction Assessment

Yes. On current main, a plugin command handler resolving undefined reaches executePluginCommand unchanged, then handlePluginCommand reads result.continueAgent; the linked v2026.4.27 Telegram report provides matching runtime logs.

## Solution Assessment

Yes, mostly. Normalizing at executePluginCommand is the narrowest maintainable fix, and the caller-side guards protect mocked or direct runtime paths; the remaining question is maintainer choice among duplicate PRs and SDK contract wording.
