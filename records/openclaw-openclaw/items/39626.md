## Summary

Keep open. Current main still does not sanitize the reported bare `exec{"command": ...}` pseudo-tool-call text. The implemented visible-text sanitizers cover XML-style tool-call tags, MiniMax XML, model control tokens, relevant-memory tags, and downgraded `[Tool Call: ...]` scaffolding, but not bare tool-name-plus-JSON output that can flow through the shell/Telegram delivery paths.

## Best Possible Solution

Keep this open as a behavior bug. Add provider-specific repair or a conservative visible-text sanitizer for bare tool-name-plus-JSON pseudo calls such as `exec{...}`, and cover `sanitizeAssistantVisibleText`, `extractAssistantVisibleText`, streaming/block reply delivery, history/session recall, and Telegram/shell-facing output with regression tests.
