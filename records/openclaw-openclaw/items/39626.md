## Summary

Keep open. Current main still lacks handling for the reported bare tool-name-plus-JSON visible text leak, such as `exec{"command": "openclaw status", ...}`. The shared assistant visible-text sanitizer covers XML-style tool blocks, bracketed local-model blocks, model special tokens, reasoning tags, and downgraded `[Tool Call: ...]` scaffolding, but not this shape.

## Best Possible Solution

Keep this open as a behavior bug. Add narrowly scoped shared handling for bare tool-name-plus-JSON pseudo-calls such as `exec{"command": ...}` in the assistant visible-text sanitizer or an immediately adjacent provider repair before visible delivery. Cover final extraction, streaming chunks, history/session recall, and Telegram delivery behavior while preserving existing XML, bracketed, downgraded-tool, and prose/code-example safeguards.
