## Summary

Keep open. The issue is low-detail, but not safely closeable: current main has multiple plausible "search" surfaces, and while `web_search` already has provider-specific content extraction/summary support, the issue does not say whether it wants web search, channel message search, memory search, UI search, or a cross-provider summary behavior.

## Best Possible Solution

Leave the issue open for maintainer clarification. The next useful action is to ask the reporter which search surface they mean and what output they expect. If they mean `web_search`, decide whether OpenClaw should document/use an existing provider such as Exa, add provider-specific summary fields where supported, or introduce a generic summarized-result contract/middleware for all web-search providers.
