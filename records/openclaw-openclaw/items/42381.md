## Summary

Keep this PR open. Current main still lacks a remote node-host guard in `openclaw models auth login`, the linked bug #42291 remains open, and this PR is still a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review or land a focused remote node-host auth-write guard in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`. Prefer reusing the shared `src/gateway/net.ts:isLoopbackHost` helper for gateway host classification, and explicitly decide the intended scope across OAuth, device-code, token, API-key, and custom provider auth methods. After merge, #42291 can close as implemented.
