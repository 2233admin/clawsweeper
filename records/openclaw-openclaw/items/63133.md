## Summary

Keep open. Current main already covers the human fast-status label, but it does not implement the PR branch's added JSON collection-state contract, and the proposed patch still has blocking test and passthrough gaps before it can merge cleanly.

## What This Changes

The PR adds a checked/skipped memory-collection state through status scan/report/JSON plumbing, changes skipped memory rendering to `not checked`, and adds focused status regression coverage.

## Best Possible Solution

Accept one explicit status contract: either keep current main's human-only `not checked` label, or intentionally add a documented `memoryCollection: "checked" | "skipped"` JSON field with exact payload tests, end-to-end command coverage, and scan/report plumbing that distinguishes skipped collection from failed probes.
