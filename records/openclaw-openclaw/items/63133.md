## Summary

Keep open. Current main now covers the plain human `openclaw status` row by rendering skipped memory inspection as `not checked`, but it does not implement the PR's additive `memoryCollection` JSON/state contract. The PR still may be useful, but the branch needs current-main reconciliation and has blocking test/propagation issues in the proposed diff.

## What This Changes

The PR adds a checked/skipped memory-collection state to status scan results and JSON payload plumbing, changes memory rendering to show skipped collection as `not checked`, and adds focused status tests.

## Best Possible Solution

Accept one clear contract: either keep main's human-only `not checked` behavior, or intentionally add a documented `memoryCollection: "checked" | "skipped"` JSON field with updated payload tests and end-to-end status coverage.
