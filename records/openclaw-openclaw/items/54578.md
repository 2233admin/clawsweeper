## Summary

Keep open. Current main still exposes flat session, agent, task, and subagent list surfaces, while the requested user-facing CLI/TUI hierarchy for active agents and child sessions is not implemented.

## What This Changes

Add an `openclaw tree`-style command that visualizes active agents, parent-child subagent/session relationships, status, sessions, resource usage, JSON output, and watch mode.

## Best Possible Solution

Add a core inspection surface that renders the existing session/subagent lineage as a stable hierarchy, with text and JSON output first. Defer live watch mode and per-process CPU/RSS accounting until maintainers define the public command namespace and resource data contract.
