## Summary

Keep this PR open. Current main still drops Mattermost `post_edited` WebSocket events before mention handling, while the PR is a focused fix for the linked Mattermost wake bug and now includes the edit-aware replay guard needed to avoid deduping edited revisions against the original post.

## What This Changes

The PR updates the Mattermost plugin schema, WebSocket dispatcher, inbound replay guard, debounce flush path, and tests so `post_edited` events with updated post bodies are re-evaluated and keyed by `edit_at`.

## Best Possible Solution

Land this PR or an equivalent Mattermost-plugin-only fix after maintainer review confirms the repeated-edit semantics. The desired end state is `post_edited` dispatch through the existing mention/access/self-filter path, with edit-aware replay keys and regression coverage preserved.
