## Summary

Keep open. Current main still does not fully implement this PR's intended stale-token display behavior: the Gateway exposes `totalTokensFresh`, but the TUI session-list/state/footer path still ignores it, so a stale row with omitted `totalTokens` can leave an older count visible. The Control UI has partial event-clearing coverage, but `formatSessionTokens` still renders any non-null total without checking freshness. The PR remains useful, but should be rebased and adjusted for the fresh-null clearing regression noted in review.

## Best Possible Solution

Keep this PR open and ask for a small rebase/update on current main: carry `totalTokensFresh` through the TUI session list/state path, clear or skip TUI `totalTokens` based on freshness while preserving fresh-null clearing semantics, make `formatSessionTokens` return `n/a` when `totalTokensFresh === false`, and add focused tests for those cases. Keep the patch scoped to display-only UI/TUI behavior.
