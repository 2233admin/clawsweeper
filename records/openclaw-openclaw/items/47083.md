## Summary

Keep open. Current main still leaves the TUI footer and Control UI sessions formatter able to display a stale `totalTokens` value, while this PR's TUI patch has a real fresh-null clearing regression that should be fixed before merge or ported into a replacement branch.

## What This Changes

The PR adds `totalTokensFresh` to TUI and Control UI session-row types and updates token display/state paths to avoid displaying totals marked stale.

## Best Possible Solution

Port the intent onto current main: carry `totalTokensFresh` through the TUI session-list/state path, skip stale totals while allowing fresh `null` totals to clear state, make the Control UI sessions-table formatter return `n/a` for stale totals, and add focused regression tests for stale numeric values and fresh-null clearing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Allow fresh null totals to clear the display:** `src/tui/tui-session-actions.ts:194-198`

  - confidence: 0.9
