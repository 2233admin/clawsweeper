## Summary

Keep this PR open. Current main still does not fully implement the intended stale-token display behavior: the Gateway and part of the Control UI carry `totalTokensFresh`, but the TUI session state/display path still ignores it and the Control UI sessions table formatter still renders any non-null `totalTokens` even when freshness is false. The PR remains useful, but it needs a rebase and a small fix for the fresh-null clearing regression noted in review.

## Best Possible Solution

Keep this PR open and ask for a small rebase/update: add `totalTokensFresh` to the TUI session list/state path, gate or clear TUI `totalTokens` based on freshness while preserving fresh-null clearing semantics, make `formatSessionTokens` return `n/a` when `totalTokensFresh === false`, and add focused tests for those cases. The security-sensitive surface should remain limited to display-only UI/TUI code.
