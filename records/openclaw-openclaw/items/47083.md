## Summary

Keep this PR open. Current main still does not fully respect `totalTokensFresh` in the TUI footer state or the Control UI session token formatter, and the PR is pointed at the right user-visible bug, but its TUI guard leaves a stale cached total in place for stale rows that omit `totalTokens` and blocks fresh `null` clears.

## What This Changes

The PR adds `totalTokensFresh` to TUI and Control UI session row handling and changes token display paths so stale session totals are not shown.

## Best Possible Solution

Land a narrow display fix on current main: TUI should carry `totalTokensFresh`, clear or ignore cached footer totals whenever rows or patches mark totals stale, still allow fresh `totalTokens: null` to clear state, and Control UI session-token formatting should return `n/a` for stale rows with focused regression tests.

## Reproduction Assessment

Yes. Seed TUI state with a numeric `sessionInfo.totalTokens`, then apply a same-session row or patch with `totalTokensFresh: false` and no fresh total; the current TUI path leaves the old footer value visible. The Control UI formatter path is directly reproducible with a `GatewaySessionRow` containing numeric `totalTokens` plus `totalTokensFresh: false`.

## Solution Assessment

No. The PR direction is correct, but the TUI implementation is not the best fix because it checks freshness only after requiring a non-null total; freshness should be handled first so stale markers clear or suppress cached totals and fresh null totals still clear state.
