## Summary

Keep this draft PR open. Current main has some adjacent session-selector and navigation improvements, plus a backend `sessions.create` primitive, but it still does not implement the PR's central Control UI behavior: the Chat New session button still sends `/new` into the active session/reset path instead of creating a distinct persisted session and switching to it.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate. The best path is to rebase it onto current main, wire the Chat New session button to the existing `sessions.create` RPC, switch through the established session-switch path, preserve current partial selector/navigation fixes, and add focused tests for creation, switching, duplicate labels, recency ordering, and navigation-return synchronization.
