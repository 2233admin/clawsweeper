## Summary

Keep open. Current main has some adjacent session-control cleanup, but it does not implement this PR's main behavior: the Chat "New session" action still sends `/new` into the active chat/reset path instead of creating a new switchable session key via `sessions.patch`. The chat session selector also still lacks the PR's recent-first ordering and `live(...)` select binding.

## Best Possible Solution

Keep this PR open as potentially useful work, but require a rebase/update against current main. The likely best path is to implement the independent Control UI new-session flow in the current chat/session controls, add focused tests for unique key generation, `sessions.patch` creation, session switching, and recent-first ordering, and explicitly verify the WebChat/admin-scope boundary remains unchanged.
