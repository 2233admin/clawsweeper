## Summary

Keep open. Current main still hardcodes the shared gateway chat-delta broadcast throttle at 150 ms, has tests/comments built around that fixed window, and does not contain the requested environment resolver; the linked implementation PR is closed unmerged, so this still needs maintainer product/API review.

## What This Changes

Request to make the gateway chat-delta broadcast throttle configurable with `OPENCLAW_CHAT_DELTA_THROTTLE_MS` while preserving the current 150 ms default.

## Best Possible Solution

A maintained solution would centralize throttle resolution, keep the released 150 ms default, reject invalid or negative values, honor `0` only if maintainers accept disabling the throttle, update fixed-window comments/tests, and document the knob if it becomes a supported operator control.

## Reproduction Assessment

Yes. Source inspection and existing unit tests give a high-confidence reproduction path: emit two assistant deltas less than 150 ms apart and the second delta is buffered instead of broadcast until a flush path runs.

## Solution Assessment

Unclear pending maintainer product/API review. The proposed resolver is narrow and default-preserving, but the best supported surface may be an environment variable, config key, or no public knob.
