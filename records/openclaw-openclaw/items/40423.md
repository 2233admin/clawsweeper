## Summary

Keep this PR open. Current main at d22a851253b0225e72847dc6b1a7b74d8035e214 still does not expose the requested `OPENCLAW_AGENT_ID` or `OPENCLAW_SESSION_KEY` markers in exec, ACP client, or TUI local shell child environments; it only has the existing `OPENCLAW_SHELL` markers. The open #72980 overlaps only the exec session-key subset and does not supersede this broader runtime PR.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is to rebase onto current main, define the approved security/override semantics for these runtime markers, thread explicit agent/session context through exec direct/sandbox/PTY and gateway approval follow-up execution, update the current ACP helper in `src/acp/client-helpers.ts`, pass TUI agent/session state into `createLocalShellRunner`, document the contract, and add focused tests for explicit-agent precedence, session-key fallback, missing context, override behavior, and approved gateway execution.
