## Summary

Keep this PR open. Current main at a820a307 still only injects the existing OPENCLAW_SHELL markers for exec, acp-client, and tui-local; it does not implement the requested OPENCLAW_AGENT_ID or OPENCLAW_SESSION_KEY contract on those spawned child-process environments. The PR is still useful, but it needs rebase onto the current helper layout, explicit security/override semantics for session-key exposure, and reconciliation with overlapping #72980 before it can land.

## Best Possible Solution

Keep this PR open for maintainer review or rework. Rebase it onto current main, decide whether OPENCLAW_SESSION_KEY is safe to inject by default and whether caller-supplied env may override runtime markers, implement the markers in the current exec runtime, gateway approval follow-up, ACP client helper, and TUI local-shell paths, update public docs, add focused tests for explicit-agent precedence, session-key fallback, missing context, caller override behavior, PTY/sandbox/gateway approval execution, ACP client env, and TUI local shell env, then reconcile the overlapping exec-only #72980 before landing.
