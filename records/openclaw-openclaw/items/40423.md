## Summary

Keep this PR open. Current main still does not implement or document `OPENCLAW_AGENT_ID` or `OPENCLAW_SESSION_KEY` for the requested `exec`, `acp-client`, and `tui-local` child-process paths; those paths currently expose only the existing `OPENCLAW_SHELL` markers. The related #72980 is open and narrower, so it does not supersede this broader PR.

## Best Possible Solution

Keep this PR open for rebase and maintainer review. The best implementation is a shared or consistently mirrored runtime env resolver that injects stable `OPENCLAW_AGENT_ID` plus an approved optional `OPENCLAW_SESSION_KEY`, threads it through exec direct/sandbox/PTY and gateway follow-up execution, current ACP client helpers, and TUI local shell commands, and adds focused tests for explicit-agent precedence, session-key fallback, missing/legacy context, env override semantics, and session-key exposure policy.
