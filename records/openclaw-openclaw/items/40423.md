## Summary

Keep this PR open. Current main still only injects the existing OPENCLAW_SHELL markers for exec, acp-client, and tui-local child process paths; OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY are not implemented or documented. The related #72980 is narrower and open, so it does not supersede this broader PR.

## Best Possible Solution

Keep this PR open and ask for an update or maintainer review. The best path is a rebased shared runtime env resolver that injects deterministic OPENCLAW_AGENT_ID plus approved optional OPENCLAW_SESSION_KEY, threads it through exec direct/sandbox/PTY, gateway approval follow-up execution, current ACP client helpers, and TUI local shell commands, and adds focused tests for explicit-agent precedence, session-key fallback, missing/legacy context, env override semantics, and security-reviewed session-key exposure.
