## Summary

Keep open. Current main still rebuilds array-backed message content and returns a new message object even when stripping changes no text, the focused identity tests are still absent, and the direct implementation PRs are closed unmerged with no direct open replacement found.

## What This Changes

The issue asks for `stripInlineDirectiveTagsFromMessageForDisplay` to preserve the original message object and unchanged text-part objects when no inline directive tags are stripped.

## Best Possible Solution

Land a narrow utility and regression-test update that uses the existing `changed` result, reuses unchanged text-part objects, returns the original array-backed message when no text changed, and keeps directive stripping output unchanged.

## Reproduction Assessment

Yes. A focused unit test can pass an array-backed message with plain text to `stripInlineDirectiveTagsFromMessageForDisplay` and assert `result === input` plus unchanged part identity; current source necessarily fails because it always maps the array and spreads text parts.

## Solution Assessment

Yes. The best fix is the narrow helper change proposed here: keep the API and stripping semantics stable, track whether any text part changed, and allocate only for changed content.
