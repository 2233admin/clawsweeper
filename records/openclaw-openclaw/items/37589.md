Keep open. ## What This Changes

The issue requests preserving message and text-part object identity when `stripInlineDirectiveTagsFromMessageForDisplay` strips no inline directive tags.

## Best Possible Solution

Land a narrow helper and test update in `src/utils/directive-tags.ts` and `src/utils/directive-tags.test.ts` that reuses unchanged text parts, tracks whether any part changed, returns the original message when no directives were stripped, and keeps all existing directive-stripping semantics intact.

## Repair Work Prompt

Fix the identity churn reported here without changing directive-stripping semantics. In `src/utils/directive-tags.ts`, update `stripInlineDirectiveTagsFromMessageForDisplay` so it calls `stripInlineDirectiveTagsForDisplay` for text parts, reuses the original text-part object when `changed` is false, tracks whether any text part changed, and returns the original message object when no text part changed. Preserve existing behavior for `undefined`, non-array content, non-text parts, and text that strips to an empty string. Do not change public SDK exports, directive regex semantics, delivery stripping, or unrelated gateway/media behavior. Add focused Vitest coverage in `src/utils/directive-tags.test.ts` for unchanged array-backed messages returning the same message reference, mixed changed/unchanged parts preserving unchanged part references while replacing changed parts, and existing stripped-output behavior. No changelog entry is expected for this internal helper/perf fix unless a maintainer asks for one. Related refs: #37588, #37642, #47205 were prior closed-unmerged attempts for the same behavior.
