## Summary

Keep this open. Current main still unconditionally maps array-backed message content, recreates every text part, and returns a new message object from stripInlineDirectiveTagsFromMessageForDisplay even when stripInlineDirectiveTagsForDisplay reports no change. The linked implementation PRs are closed unmerged, and the latest release tag still contains the same unresolved implementation.

## Best Possible Solution

Implement the focused identity-preservation fix in src/utils/directive-tags.ts: reuse unchanged text-part objects, track whether any text part changed, and return the original message object when nothing was stripped. Add colocated Vitest coverage using toBe for unchanged array-backed messages and mixed changed/unchanged text parts. Preserve SDK exports, directive regex behavior, and existing empty-string semantics.
