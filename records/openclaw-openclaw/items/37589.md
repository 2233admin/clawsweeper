## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/37589. In `src/utils/directive-tags.ts`, update `stripInlineDirectiveTagsFromMessageForDisplay` so it calls `stripInlineDirectiveTagsForDisplay(record.text)`, recreates only text parts whose result has `changed: true`, tracks whether any content part changed, and returns the original message object when array-backed content has no stripped directive changes. Preserve existing behavior for `undefined`, non-array content, non-text parts, directive-only text becoming an empty string, the function signature, and all stripped text output. Add focused tests in `src/utils/directive-tags.test.ts` asserting: unchanged array-backed messages return the same object with `toBe`; unchanged text parts preserve object identity; mixed content recreates only changed text parts and preserves unchanged/non-text parts; existing stripping cases still pass. Do not change the directive regexes, public SDK exports, docs, or package metadata. No changelog entry is expected for this narrow utility/performance fix unless a maintainer requests one.
