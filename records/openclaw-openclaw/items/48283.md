## Summary

Keep this PR open. Current main does not appear to implement the PR’s intended generic abort cleanup: it only strips aborted assistant messages on the `sessions_yield` abort path, while ordinary prompt aborts still persist `openclaw:prompt-error`, snapshot the live messages, and let assistant `message_end` handling finalize visible text without checking `stopReason: "aborted"`. The linked bug #48241 is still open, so this PR remains a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Review and revise this PR rather than closing it: implement a shared or clearly scoped helper that strips only the current non-yield aborted assistant tail from both in-memory state and session-file entries on the runner abort error path, add a regression test for #48241, and address the duplicate-helper and `_rewriteFile` review comments before landing.
