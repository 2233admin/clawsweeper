## Summary

Keep this PR open. Current main still lacks the intended generic cleanup for ordinary prompt aborts: only the sessions_yield abort path strips trailing aborted assistant messages, while non-yield RunnerAbortError paths still record promptError and continue through snapshot/prompt-error handling. The linked bug #48241 remains open, and this PR is still a plausible implementation candidate, though it needs revision before landing.

## Best Possible Solution

Keep this PR open and revise it rather than closing it. The best path is a shared or clearly scoped tail-strip helper called on the ordinary RunnerAbortError prompt path, updating both activeSession.agent.state.messages and sessionManager file entries, with a regression test for #48241 and resolution of the duplicate-helper/_rewriteFile review comments before landing.
