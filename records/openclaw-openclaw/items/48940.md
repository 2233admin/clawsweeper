## Summary

Keep this PR open. Current main does not contain the gateway-owned `acp-node` runtime, durable ACP gateway store, node-host ACP command bridge, or durable replay/startup recovery path described by the PR. The PR is also a large security-sensitive gateway/node execution-surface change with unresolved review concerns in the provided context, so the best cleanup outcome is maintainer review or a rebased/split replacement, not auto-closing.

## Best Possible Solution

Keep the PR open unless a maintainer chooses to close it manually. The best path is to rebase or replace it with smaller reviewable slices: first the durable gateway store and protocol contract, then ACP node command ingress/worker bridge, then durable projection/replay and startup recovery. Before landing, address the P1 review findings and run focused ACP/gateway/node tests plus the appropriate changed gate.
