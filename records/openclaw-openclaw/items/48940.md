## Summary

Keep this PR open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still does not implement the PR's central change: a gateway-owned, node-backed ACP runtime with durable gateway store, ACP node command/event bridge, durable replay projection, and startup recovery. The PR also expands a security-sensitive gateway-to-node execution surface and has unresolved P1 review findings, so it needs maintainer review or a rebased/split replacement rather than cleanup closure.

## Best Possible Solution

Keep the PR open unless a maintainer manually closes or replaces it. The best path is a rebased, reviewable split: durable gateway ACP store/protocol first, node command ingress and worker bridge second, durable projection/replay and startup recovery third. Resolve the outstanding P1 review findings and run focused ACP/gateway/node tests plus the appropriate changed gate before landing.
