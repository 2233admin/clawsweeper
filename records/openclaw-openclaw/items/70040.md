## Summary

Keep #70040 open. Current main still does not implement Chinese/CJK auto-capture triggers in the bundled memory-lancedb plugin, and the related PRs called out by PRtags are closed unmerged rather than shipped or serving as an open canonical tracker. The PR is relevant, but it should not be closed as implemented or duplicate; it needs revision because the current one-line regex would still miss common short Chinese memory statements behind the existing length gate and has no CJK regression coverage.

## Best Possible Solution

Keep the item open, but ask for a corrected memory-lancedb extension patch: add CJK trigger tests, adjust/evaluate the short-input length gate for high-signal CJK triggers, and consider CJK detectCategory coverage based on the prior #16392/#16672/#20241 discussion. If maintainers want a single canonical path, supersede this PR with a new focused PR that includes those tests and closes the older unmerged attempts explicitly.
