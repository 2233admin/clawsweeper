## Summary

Keep this PR open. Current main still couples Codex app-server startup to the full per-turn timeout and has no independent `appServer.startupTimeoutMs` config, while the PR is a focused additive fix with tests/docs and no observed security-sensitive drift.

## What This Changes

The PR adds a Codex app-server `startupTimeoutMs` config/runtime option, uses it for startup instead of the per-turn timeout, updates manifest/docs/changelog, and adds resolver plus regression tests.

## Best Possible Solution

Review and land this PR, or an equivalent narrow change, so Codex app-server startup has its own bounded and documented timeout separate from the per-turn budget, with manifest/docs alignment and a regression test proving a 30-minute turn timeout no longer extends startup.
