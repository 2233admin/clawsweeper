## Summary

Keep this PR open. Current main and the latest release still make Feishu WebSocket inherit ambient proxy env through `resolveAmbientNodeProxyAgent()`, and the current tests still assert ambient proxy behavior. The PR is a focused implementation candidate for the linked open bug #65799, but it still needs the reviewed `OPENCLAW_FEISHU_WS_USE_PROXY` test cleanup and maintainer agreement on changing Feishu WebSocket proxying from ambient default to explicit opt-in.

## Best Possible Solution

Keep this PR and #65799 open. The best path is to update the PR so `OPENCLAW_FEISHU_WS_USE_PROXY` is captured, deleted before each test, and restored in cleanup, then have a Feishu/channel maintainer decide whether explicit opt-in is the desired default. If accepted, land it with targeted Feishu tests and a small docs/changelog note for the new opt-in; if rejected, close the PR while leaving #65799 to track a different mitigation or diagnostic improvement.
