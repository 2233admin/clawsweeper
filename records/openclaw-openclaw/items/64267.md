## Summary

Current main still has the narrow streaming scanner identified in the issue discussion: it recognizes bare thinking tags but not attributed tags, while the canonical final sanitizer explicitly supports attributes. Because that scanner is used before block replies are delivered, the core leak report remains valid and should stay open.

## What This Changes

The issue requests that OpenClaw stop exposing agent internal thinking/planning text in user-visible channel replies after the 2026.4.9 upgrade.

## Best Possible Solution

Keep the issue open and align all embedded streaming/tag promotion paths with the canonical reasoning-tag grammar, including `antml:` prefixes, attributes, case-insensitivity, code-span awareness, and chunk-boundary state, then cover attributed tags in focused streaming and utility regression tests.

Current main still uses a streaming scanner in `src/agents/pi-embedded-utils.ts` that requires thinking tags to end immediately with `>`, while `src/shared/text/reasoning-tags.ts` accepts attributes and optional `antml:` prefixes. Add focused tests for attributed tags, including streamed/block replies across chunk boundaries and tagged-text promotion/extraction.
