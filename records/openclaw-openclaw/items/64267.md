## Summary

Keep open. Current main still leaves the streaming/block reply sanitizer narrower than the canonical final sanitizer, matching the root-cause analysis already supplied in the issue discussion; the merged #69288 fix covers bare namespaced tags but not attributed thinking tags, so the reported leak path remains actionable.

## What This Changes

The issue requests that OpenClaw stop exposing agent internal thinking/planning text to chat users across models and channels.

## Best Possible Solution

Align the streaming and thinking-block promotion parsers with the shared reasoning-tag grammar so bare, namespaced, attributed, case-insensitive, and chunk-split thinking tags are hidden consistently before any channel-visible block is emitted.
