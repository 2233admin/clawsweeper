## Summary

Keep open: current main still lacks the strict-agentic prompt and active-narration coverage this PR targets, but the PR should not merge as-is because the new bare `do` directive can classify explicit negative prompts as actionable and steer a retry toward tool use. The inspected head also has failed lint/aggregate checks, with the lint annotation outside the PR diff, so the branch needs a small fix plus validation before maintainer landing.

## What This Changes

The PR broadens strict-agentic planning-only retry regexes and retry wording in `src/agents/pi-embedded-runner/run/incomplete-turn.ts`, adds focused regression tests in `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts`, and adjusts a Codex plugin test helper type.

## Best Possible Solution

Keep the narrow helper-based fix, but add an explicit negative-prompt guard and regression tests for `do not` / `don't` style cancellations before landing, then rebase or otherwise clear the failed lint/check state on the PR head.

## Reproduction Assessment

Yes. The helper-level path is high confidence: current main's `resolvePlanningOnlyRetryInstruction` returns null for the reported imperative prompt because the actionable prompt regexes miss it, and returns null for `Great. I'm running it now.` because there is no active-narration detector.

## Solution Assessment

No, not as currently written. Updating the central helper regexes with colocated tests is the right shape, but the PR needs to constrain the new `do`/expanded verb matching so explicit negative directives do not become retryable action prompts.

- **[P2] Guard negative do-not prompts:** `src/agents/pi-embedded-runner/run/incomplete-turn.ts:174`
