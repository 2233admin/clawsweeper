## Summary

Keep this PR open. Current main still has the strict-agentic planning-only gaps described by the linked report, and the PR is a plausible narrow fix, but the latest diff introduces an over-broad actionable prompt match that can turn explicit non-action prompts into act-now retries.

## What This Changes

The PR expands strict-agentic planning-only retry prompt and narration regexes, tightens the retry instruction text, adds regression tests for imperative and active-narration stalls, and makes a small Codex plugin test-helper typing cleanup.

## Best Possible Solution

Keep the narrow strict-agentic helper-level fix, but constrain the newly added bare request verbs so imperative prompts are covered without treating negated non-action statements as commands; add regression coverage for that boundary before merge.

## Reproduction Assessment

Yes. Current main can be checked at the helper/regex level: the reported imperative prompts miss the actionable gate, and present-continuous narration misses the planning-only detector. The PR's new false-positive boundary is also reproducible by comparing current and PR regex behavior for a negated prompt such as `I do not want you to do anything`.

## Solution Assessment

No, not as written. Updating the central helper with colocated tests is the right shape, but the non-anchored request regex should not treat every bare `do` occurrence as actionable; the safer fix is to keep imperative verbs start-anchored or add explicit negation/declination exclusions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Constrain bare request verbs before retrying:** `src/agents/pi-embedded-runner/run/incomplete-turn.ts:178`
  - body: Adding bare `do` to `ACTIONABLE_PROMPT_REQUEST_RE` makes non-directive prompts such as `I do not want you to do anything` actionable. If the model replies with planning-style text like `I'll do nothing.`, the expanded action-verb gate also matches `do`, so strict-agentic injects an act-now retry even though the user explicitly declined action. Keep `do` in the start-anchored directive path or exclude negated forms before treating it as actionable.
