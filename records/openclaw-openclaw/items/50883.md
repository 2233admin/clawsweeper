## Summary

Keep open. Current main still does not implement a `sessions --kind` filter, and this PR remains a focused CLI feature. It should not be closed as obsolete, but the submitted patch is still incomplete because it misses the route-first sessions parser and docs coverage.

## Best Possible Solution

Revise this PR against current main: add the shared session-kind parser, wire repeatable and comma-separated `--kind` through both Commander and `parseSessionsRouteArgs`, apply the filter in `sessionsCommand` for text and JSON output, add route and command tests, update `docs/cli/sessions.md`, and share the session-kind type to avoid drift.
