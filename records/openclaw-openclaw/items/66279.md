## Summary

Keep open. Current main still has the reported TUI-specific TLS trust gap: the TUI resolves local gateway URL/auth but has no field or handoff for `tlsFingerprint`, while status/probe paths already forward a scoped local TLS fingerprint and the related TUI fix PR was closed unmerged.

## What This Changes

The issue requests that `openclaw tui` load and pass the configured local gateway TLS certificate fingerprint to `GatewayClient` for local self-signed `wss://` gateways.

## Best Possible Solution

Implement the TUI trust handoff in `src/tui/gateway-chat.ts`: add an optional `tlsFingerprint` to the resolved connection, load the local gateway TLS runtime only for configured local `wss://` targets with `gateway.tls.enabled === true` and no URL override or remote target, pass it into `GatewayClient`, and add focused tests in `src/tui/gateway-chat.test.ts` for propagation, unavailable-runtime fallback, URL override, and remote cases.
