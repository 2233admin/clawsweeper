## Summary

Keep open. Current main still lacks the requested ACP turn-completion internal hook, and this PR is the focused implementation attempt, but the proposed hook contract drops the advertised failure code and dispatches a new per-turn hook through the unbounded fire-and-forget path.

## What This Changes

The branch adds an `agent:turn:end` internal hook type/guard, emits it from ACP turn completion on success and failure, and adds hook guard plus ACP manager emission tests.

## Best Possible Solution

Land a narrow update that emits `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()` on both success and failure, includes `sessionKey`, `success`, `durationMs`, and optional `errorCode`, uses bounded non-blocking dispatch or documents a deliberate exception, and keeps the scope limited to this ACP turn-completion seam.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature PR. The current-main gap is reproducible by inspecting the ACP completion path and exact symbol searches, and the PR defects are reproducible from the diff plus the existing hook helper contract.

## Solution Assessment

No, not as-is. The proposed location is plausible and narrow, but the hook contract should preserve the existing ACP failure code and the dispatch should use the bounded helper before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Propagate failure codes in turn-end hook payload:** `src/hooks/internal-hooks.ts:171-178`
  - body: `recordTurnCompletion` already receives `errorCode` for failed turns, but the new context type and emitted payload only expose `success: false`. Consumers cannot distinguish ACP failure classes such as turn failure versus session init failure, despite the PR advertising optional `errorCode`; add it now before this hook contract ships.
  - confidence: 0.93
- **[P2] Bound turn-end hook dispatch:** `src/acp/control-plane/manager.core.ts:1685-1694`
  - body: This starts `triggerInternalHook(...)` immediately and passes the promise to `fireAndForgetHook`, which has no timeout, concurrency cap, or queue limit. A hanging hook handler can leave a stuck promise per turn; use `fireAndForgetBoundedHook(() => triggerInternalHook(...))` or document why this new high-frequency lifecycle hook is exempt.
  - confidence: 0.9
