## Summary

Keep PR #71399 open. Current main at 998e37fcb3d5b937fcf70d7509b677efba939f10 still does not implement an `agent:turn:end` internal hook: ACP turn completion only updates latency/error counters, internal hook types/tests have no `AgentTurnEnd` contract or guard, and the hooks docs do not list the event. The PR remains a focused implementation attempt, but review comments identified real follow-up before merge: the advertised failure `errorCode` is missing from the proposed payload, and the PR uses unbounded hook dispatch despite an existing bounded helper.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish this PR or a successor by emitting `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()` on success and failure, carrying `sessionKey`, `success`, `durationMs`, and optional `errorCode`, using bounded non-blocking dispatch or documenting why this lifecycle hook intentionally differs, and covering the type guard plus ACP manager success/failure emission in tests. If the event becomes operator-facing, update the internal hooks docs as well.
