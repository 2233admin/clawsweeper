## Summary

Keep open. Current main still does not implement the intended `agent:turn:end` internal hook, and this PR remains the focused implementation attempt. It is not ready to close or merge as-is because the proposed payload drops the advertised failure `errorCode`, and the hook dispatch path uses the unbounded fire-and-forget helper even though a bounded helper exists.

## Best Possible Solution

Keep this PR open for maintainer review or an in-place update. The best path is to emit `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()` on both success and failure, include `sessionKey`, `success`, `durationMs`, and optional `errorCode`, use bounded non-blocking dispatch or document a deliberate exception, cover success/failure manager emission and the type guard, and update hook docs if the event becomes documented operator-facing behavior.
