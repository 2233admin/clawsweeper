## Summary

Keep open. Current main and the latest release still lack the requested ACP turn-end hook, and this PR is the focused implementation attempt, but the proposed patch is not merge-ready because it drops the advertised failure code and uses unbounded hook dispatch on a per-turn path.

## What This Changes

The PR adds an `agent:turn:end` internal hook emitted from ACP turn completion, a hook context/type guard, and focused internal-hook plus ACP manager tests.

## Best Possible Solution

Land a narrow ACP-manager internal hook that fires once per completed turn, preserves failure classification with optional `errorCode`, uses bounded non-blocking dispatch, and has focused tests for success, failure, type guarding, and stalled-hook behavior; update hook docs if this event is part of the documented operator-facing hook surface.

## Reproduction Assessment

Yes. Current main can be verified by searching for `agent:turn:end` and inspecting `AcpSessionManager.recordTurnCompletion()`; the PR reproduction path is to register an internal listener and run successful and failing ACP turns.

## Solution Assessment

No, not as-is. The hook point is the right narrow seam, but the contract should include optional `errorCode` on failures and dispatch should use the existing bounded helper or have an explicit maintainer-approved exception.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Propagate the turn failure code:** `src/hooks/internal-hooks.ts:168-174`
  - body: `recordTurnCompletion` receives `errorCode` for failed turns, but the new hook context only exposes `success` and `durationMs`. Consumers of `success: false` lose the ACP failure classification even though the data is available at emission time and advertised in the PR body.
  - confidence: 0.91
- **[P2] Use bounded dispatch for turn-end hooks:** `src/acp/control-plane/manager.core.ts:1681-1689`
  - body: The new call wraps an already-started `triggerInternalHook(...)` promise with `fireAndForgetHook`, so a listener that never settles can leave a pending hook promise on every turn. Use `fireAndForgetBoundedHook(() => triggerInternalHook(...), ...)` to apply the existing timeout, concurrency, and queue limits.
  - confidence: 0.9
