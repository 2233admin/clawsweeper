## Summary

Keep open. Current main still lacks the requested `agent:turn:end` internal hook, and this PR is the focused implementation attempt, with real downstream interest noted by zeroaltitude for a security plugin. It is not merge-ready as-is because the proposed failure payload drops the advertised `errorCode`, and the per-turn hook dispatch uses the unbounded fire-and-forget helper even though the repo already has a bounded helper for this class of async hook work.

## What This Changes

The PR adds an `agent:turn:end` internal hook emitted from ACP turn completion on success and failure, adds the hook context/type guard, and adds targeted internal-hook and ACP manager tests.

## Best Possible Solution

Land a narrow ACP-manager internal hook that fires once per completed turn, preserves failure classification in the payload, uses bounded non-blocking dispatch, and has focused tests for success, failure, type guarding, and stalled-hook safety expectations.

## Reproduction Assessment

Yes. Current main can be checked by searching for `agent:turn:end` and inspecting `AcpSessionManager.recordTurnCompletion()`; the PR reproduction path is to register an internal `agent:turn:end` listener and run successful and failing ACP turns, which the proposed tests target.

## Solution Assessment

No, not as-is. The hook point is the right narrow seam, but the proposed contract should include optional `errorCode` on failures and the dispatch should use the existing bounded fire-and-forget helper or have an explicit maintainer-approved exception.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Propagate the turn failure code:** `src/hooks/internal-hooks.ts:171-178`
  - body: `recordTurnCompletion` receives `errorCode` for failed turns, but the new hook context only exposes `success` and `durationMs`. Consumers of `success: false` lose the ACP failure classification even though the PR advertises optional `errorCode` and the call site already has it.
  - confidence: 0.91
- **[P2] Use bounded dispatch for turn-end hooks:** `src/acp/control-plane/manager.core.ts:1685-1694`
  - body: The new call uses `fireAndForgetHook(triggerInternalHook(...))`, so a listener that never settles can leave the hook promise pending indefinitely on a per-turn path. `triggerInternalHook` awaits handlers sequentially, and `fireAndForgetBoundedHook(() => ...)` already provides timeout, concurrency, and queue limits for this pattern.
  - confidence: 0.9
