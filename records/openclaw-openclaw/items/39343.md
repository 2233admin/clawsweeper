## Summary

Keep open. Current main still implements Telegram album buffering, but the LINE plugin still processes each downloadable media webhook event independently and has no LINE media batching config or docs. Shared `messages.inbound` debouncing remains text-only, so it does not satisfy the LINE image batching part of the request.

## Best Possible Solution

Keep this issue open and implement the missing bundled LINE behavior in the LINE plugin: add an optional per-account/top-level media batching delay, buffer downloadable media events by account + conversation + sender, flush on timer and before same-key text/non-media events, aggregate all media into one `LineInboundContext`, preserve access control/mention/replay/reply-token behavior, document it, and add focused LINE regression tests. Leave the existing Telegram `media_group_id` behavior unchanged.
