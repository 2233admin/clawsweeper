## Summary

Keep open. Current main handles Telegram albums with a media-group buffer, but the LINE side of the request is still unresolved: LINE webhook events are still processed one message at a time with a fresh media array and no configurable same-sender media batching surface.

## What This Changes

Request an optional media batching buffer so rapid LINE images and Telegram albums can be delivered to the agent as one turn instead of separate image turns.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary: add an optional media batching delay, buffer downloadable media by account plus conversation plus sender, flush on timer and before same-key text or non-media events, and aggregate the media into one LineInboundContext while leaving existing Telegram album buffering unchanged.
