## Summary

Keep open. Current main covers the Telegram album-buffering side, but the LINE portion remains unresolved: LINE media webhook events are still downloaded and processed one message event at a time, with no same-sender media buffer or LINE config/docs surface.

## What This Changes

Add optional inbound media batching so rapid image/media messages, especially Telegram albums and LINE image bursts, can become one agent turn instead of one turn per media event.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary: add an optional media batching delay, buffer downloadable LINE media by account plus conversation plus sender, flush on timer and before same-key text or non-media events, aggregate media into one LineInboundContext, and leave existing Telegram album buffering plus shared text-only debounce semantics unchanged.
