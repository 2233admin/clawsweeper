## Summary

Keep this PR open. Current main still does not implement the intended `infer model run --model` canonicalization: the local path passes the raw override into simple-completion preparation, the Gateway path splits and forwards raw provider/model strings, and current tests still lack mixed-case catalog ref coverage. The linked bug #73715 remains open, and this PR is still a focused implementation candidate with a narrow source/test-only diff.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73715. Review and land this PR, or an equivalent focused fix, that canonicalizes only unique case-insensitive catalog matches before both local and Gateway `infer model run` dispatch while preserving custom or ambiguous mixed-case refs, with regression tests for local dispatch, Gateway dispatch, and no-match preservation.
