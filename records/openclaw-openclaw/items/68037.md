## Summary

Keep open. Current main still returns HTTP MCP tool-call content through a lossy normalizer that rebuilds every array entry as only `type` and `text`, and #68046 is an open, unmerged closing PR for the same bug.

## What This Changes

The issue asks OpenClaw to preserve valid MCP image, audio, embedded resource, and resource_link content-block payload fields in HTTP MCP `tools/call` results instead of collapsing them to text blocks.

## Best Possible Solution

Review and revise #68046 as the focused implementation candidate: preserve only valid MCP text, image, audio, embedded resource, and resource_link blocks according to the SDK contract, fall back to text for strings or malformed entries, and add HTTP `tools/call` regression coverage before merging so GitHub can close this issue after the fix lands.
