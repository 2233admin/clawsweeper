## Summary

Keep open. Current main still hardcodes the MiniMax usage polling request to the CN host and the MiniMax provider hook does not pass configured provider base URL or region into the usage fetcher. An open implementation PR, #65059, is explicitly paired with this issue, so the right maintainer action is to review/land or close that PR rather than close the issue early.

## Best Possible Solution

Keep this issue open and review #65059 or an equivalent patch. The fix should pass MiniMax provider config or a normalized region into the usage snapshot path, build the usage endpoint for `api.minimax.io` or `api.minimaxi.com` as appropriate, preserve CN fallback behavior, and add focused tests asserting the exact usage URL for global and CN configurations.
