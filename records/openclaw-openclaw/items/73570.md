## Summary

Keep this PR open. Current main still lacks an internal `message:sending` bridge, the linked bug #73542 is still the canonical open user report, and the PR is a plausible fix direction but needs amendment before merge because review comments identify stale-content and cancelled-send notification defects.

## Best Possible Solution

Amend this PR rather than closing it. The best path is to add a tested internal `message:sending` bridge in outbound delivery that fires only for non-cancelled sends, uses the effective post-plugin hook content, documents the internal event, and adds regression coverage for normal, rewritten, and cancelled `message_sending` cases before landing the fix for #73542.
