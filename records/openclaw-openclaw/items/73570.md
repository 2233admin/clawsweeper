## Summary

Keep open. Current main still lacks the outbound internal hook bridge, but this PR is not mergeable as written because the new notification can fire for cancelled sends, can emit pre-rewrite content, and leaves the managed-hook event key contract unresolved.

## What This Changes

The PR adds a fire-and-forget internal `message:sending` hook emission in outbound delivery after the plugin `message_sending` hook runs.

## Best Possible Solution

Amend this PR into a tested outbound internal-hook bridge that emits after the plugin hook has produced its effective payload, skips cancelled sends, documents the supported managed-hook event key, and adds regression coverage for normal, rewritten, and cancelled `message_sending` paths.

## Reproduction Assessment

Yes. The current-main source path shows `message_sending` only runs through the plugin hook runner, while internal hook dispatch exists for `message:sent` only; the linked issue’s managed-hook steps are consistent with that code path.

## Solution Assessment

No. The PR is the right general direction, but the current diff should not land until it uses `hookResult.payloadSummary`, avoids cancelled notifications, and nails down the managed-hook event key with docs and tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard cancelled sends before emitting the hook:** `src/infra/outbound/deliver.ts:1032`
  - body: This block runs before the `hookResult.cancelled` guard, so a plugin hook that returns `{ cancel: true }` still emits `message:sending` to managed hooks even though no delivery will happen. Move the block after the guard or include `!hookResult.cancelled` in the condition.
  - confidence: 0.95
- **[P2] Use the post-hook payload summary:** `src/infra/outbound/deliver.ts:1041`
  - body: The event content is built from the original `payloadSummary`, but `applyMessageSendingHook` returns `hookResult.payloadSummary` when a plugin rewrites text or spoken content. Managed hooks would see stale pre-rewrite content.
  - confidence: 0.95
- **[P2] Align the managed hook event key:** `src/infra/outbound/deliver.ts:1036`
  - body: Managed hook metadata is registered as exact strings, and this event only dispatches `message` plus `message:sending`. If the linked bug's `events: ["message_sending"]` contract is intended to work, this still will not reach those hooks; add an alias/normalization or document and test `message:sending`.
  - confidence: 0.78
