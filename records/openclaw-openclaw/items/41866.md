## Summary

Keep this PR open. Current main still lacks the leading Codex-style `to=cron ... {json}` user-facing sanitizer this PR targets. Main has adjacent XML/bracketed tool-text sanitizers and a proxy/local `tool_choice` default that reduces plain-text tool directives, but neither implements this PR's exact strip pass; the PR also needs rebase into the extracted sanitizer and a tighter matcher.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. Rebase it onto current main, port the implementation into `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` or the shared assistant-visible sanitizer, constrain matching to a conservative leading registered-tool transcript shape, add regressions for the Discord `to=cron` leak and leading-prose false positives, and track structured cron tool execution separately from display sanitization.
