## Summary

Keep this PR open. Current main at 7a23b2d945cc2d2e5c452e4641213e71de8dcc66 still lacks the leading `to=<tool> ... {json}` user-facing strip pass and matching regression tests. Main has adjacent sanitizers for XML/bracketed/downgraded tool scaffolding and a diagnostic warning for tool-shaped assistant text, but those do not implement this PR's intended sanitizer behavior.
