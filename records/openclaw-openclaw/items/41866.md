## Summary

Keep this PR open. Current main at 5de284c2e30f915b769fac08dbeab45f57749702 still does not implement the leading `to=<tool> ... {json}` user-facing strip pass this PR targets. Main has adjacent assistant-visible sanitizers for XML/bracketed tool scaffolding and a diagnostic warning for tool-shaped text, but neither removes the Discord `to=cron ... {"action":"create"...}` leak shape from visible output.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. Rebase it onto current main, port the logic into `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` or the shared assistant-visible sanitizer, constrain matching to a conservative leading registered-tool transcript shape, add regressions for the Discord `to=cron` leak and leading-prose false positives, and track the structured cron execution failure separately from display sanitization.
