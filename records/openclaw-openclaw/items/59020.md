## Summary

Keep open. Current main already clears inherited `sessionFile` metadata for fresh isolated cron rollovers, but it does not fully implement the PR's store-path-aware transcript rebinding. Cron preparation and executor fallback still derive missing transcript paths with `resolveSessionTranscriptPath(sessionId, agentId)` instead of the existing `resolveSessionFilePath(..., resolveSessionFilePathOptions({ storePath }))` path. The PR discussion treats that store-aware resolution as part of the intended fix, so the PR is not safely obsolete.

## Best Possible Solution

Keep the PR open or replace it with a narrow maintainer patch that preserves current main's sanitizer behavior, derives fresh isolated cron `sessionFile` values with `resolveSessionFilePathOptions({ agentId, storePath: cronSession.storePath })` plus `resolveSessionFilePath`, updates the executor fallback to the same resolver, and keeps the stale rollover regression assertion.
