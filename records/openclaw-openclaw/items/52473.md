## Summary

Keep open. Current main still recognizes explicit skill slash commands, but normal SKILL.md-backed skills are rewritten into a regular model prompt instead of loading and injecting the selected SKILL.md content. The docs and system prompt continue to describe model-mediated on-demand reading, so the requested deterministic activation path is not implemented.

## Best Possible Solution

Keep this issue open and implement a deterministic core activation path for explicit skill invocation. After `/skill <name>` or a direct user-invocable skill command resolves to a normal SKILL.md-backed skill, the gateway should load that selected skill through the existing skills subsystem and inject its SKILL.md content into the agent run context before the user args. Preserve `command-dispatch: tool`, keep authorization and skill visibility gates intact, and add focused regression coverage for direct aliases, `/skill <name>`, disabled model invocation, transcript/provider compatibility, and docs.
