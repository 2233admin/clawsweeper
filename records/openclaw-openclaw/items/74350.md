## Summary

Keep open. Current main still classifies the reported Windows broad/unauthenticated SIDs as `group`, and open PR #74383 is already the closing implementation candidate for this security issue.

## What This Changes

Classify Windows ACL principals for Anonymous Logon, Guests, Interactive, Local, and Network as world-equivalent so filesystem audits do not downgrade them to group-level findings.

## Best Possible Solution

Keep this issue open until #74383 or an equivalent maintainer-approved patch merges; the fix should extend the world-equivalent SID/principal policy and lock the behavior with classifier plus filesystem severity regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: on current main, an ACL entry such as `*S-1-5-7:(F)` misses `WORLD_SIDS`, returns `group`, and therefore sets `groupWritable` rather than `worldWritable`.

## Solution Assessment

Unclear until maintainer security policy confirms the full SID list, but the narrow patch shape in #74383 is the right kind of solution: update the classifier policy and test both classification and audit severity without changing POSIX permission logic.
