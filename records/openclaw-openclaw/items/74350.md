## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is already the paired closing implementation candidate, so the issue should remain open until that security patch or an equivalent fix lands.

## What This Changes

Expand Windows ACL world-equivalent principal classification so Anonymous Logon, Guests, Interactive, Local, and Network writable ACEs are treated as world access rather than group access.

## Best Possible Solution

Review and land a narrow security patch, likely through #74383, that expands the approved Windows world-equivalent SID/principal policy and locks it with classifier plus filesystem-audit regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass a writable ACL entry such as *S-1-5-7:(F) through summarizeWindowsAcl or inspectPathPermissions on current main; it misses WORLD_SIDS and lands in untrustedGroup, which maps to groupWritable instead of worldWritable.

## Solution Assessment

Unclear until security-owner policy review confirms the final broad-principal set. The narrow classifier allowlist plus regression-test shape is the maintainable fix path, but the exact treatment of Interactive, Local, and Network should be approved before merge.
