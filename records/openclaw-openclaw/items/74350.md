## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is the relevant implementation candidate, so this issue should not be closed until that PR is reviewed and merged or replaced. PR #74407 also references this issue with a mismatched body/diff, which reinforces the need for maintainer review rather than automated cleanup.

## What This Changes

The issue requests treating Anonymous Logon, Guests, Interactive, Local, and Network Windows SIDs/principal names as world-equivalent in Windows ACL audits so writable ACLs do not downgrade from `worldWritable` to `groupWritable`.

## Best Possible Solution

Land a focused security patch, likely via #74383, that classifies the agreed broad Windows principals into `untrustedWorld`, preserves trusted SYSTEM/current-user handling, adds classifier and filesystem-audit regressions, and keeps unrelated PRs from closing this issue prematurely.
