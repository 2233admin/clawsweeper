## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is already the paired implementation candidate, so this security-sensitive issue should remain open until that PR or an equivalent fix is merged.

## What This Changes

The issue requests classifying Anonymous Logon, Guests, Interactive, Local, and Network Windows ACL principals as world-equivalent so writable ACLs are reported as world-writable rather than group-writable.

## Best Possible Solution

Review and land a narrow security patch, likely through #74383, that expands the Windows world-equivalent SID/principal policy and locks the behavior with classifier plus filesystem-audit regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed summarizeWindowsAcl an ACL entry with principal *S-1-5-7 and write rights; because S-1-5-7 is absent from WORLD_SIDS, current main classifies it as untrustedGroup, and the filesystem audit reports groupWritable rather than worldWritable.

## Solution Assessment

Unclear until maintainer security policy review. PR #74383 is the right current vehicle, but maintainers should confirm the full SID/name set before merging the narrow classifier and audit-test fix.
