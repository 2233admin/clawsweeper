## Summary

Keep this PR open. The reported Telegram media-null problem is real and the branch is useful context, but it cannot be merged as-is because it changes the pre-plugin `src/telegram` paths while current `main` runs the Telegram plugin from `extensions/telegram/src`; current behavior has also diverged through the merged #59948 placeholder/file_id preservation work, and the new warning path needs throttling before it is safe in groups.

## What This Changes

The PR adds a Telegram warning reply when media resolution returns null and suppresses media placeholders from inbound message bodies when no attachment was resolved.

## Best Possible Solution

Rework the fix against `extensions/telegram` so `getFile()` null failures produce an explicit unavailable-media state without losing useful metadata: keep or annotate the placeholder and `file_id` where appropriate, add a throttled user-visible warning for single media and all-failed media groups, and cover captioned, media-only, and album cases with regression tests.

  - confidence: 0.78
