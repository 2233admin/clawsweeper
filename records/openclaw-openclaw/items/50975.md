## Summary

Keep this PR open. Current main still rejects prefixless Anthropic setup tokens and requires the legacy `sk-ant-oat01-` prefix with an 80-character minimum, so the intended user-facing fix is not implemented. The PR also needs rework before merge because its proposed combined condition would let too-short legacy-prefixed tokens pass, and the implementation has since moved behind the provider auth helper.

## Best Possible Solution

Rework this PR against `src/plugins/provider-auth-token.ts`: accept both legacy-prefixed and new prefixless Anthropic setup tokens, apply the minimum length check unconditionally, and add focused unit coverage for valid prefixless tokens, valid legacy-prefixed tokens, blank input, too-short prefixless input, and too-short legacy-prefixed input. Keep the security-sensitive validation narrow and avoid unrelated auth or provider changes.
