## Summary

Keep open. Current main still exposes raw task status fields, cleanup-aware snapshots, and several independent renderers, but it does not provide the requested shared compressed operational-summary contract with current state, last successful checkpoint, current blocker, and recommended next action.

## What This Changes

Request to add a deterministic shared operational-summary layer for task status and reuse it across task CLI output, task views, and auto-reply status rendering.

## Best Possible Solution

Add an additive TaskOperationalSummary contract in the task subsystem, derive it deterministically from structured task state, expose it through task-domain views, and make CLI, chat task board, /status, session_status, and notification surfaces consume it while retaining existing raw fields.

## Reproduction Assessment

Not applicable as a runtime reproduction; this is an enhancement request. There is a high-confidence verification path by source/docs inspection showing current main has raw summaries and independent formatting, not the requested shared summary contract.

## Solution Assessment

Yes, the shared additive builder is the narrowest maintainable direction if it stays deterministic and backward-compatible. Per-surface wording tweaks would preserve the inconsistency this issue is trying to remove.
