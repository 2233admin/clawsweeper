## Repair Work Prompt

Implement issue https://github.com/openclaw/openclaw/issues/74094 as an additive task operational-summary contract. Add a shared TaskOperationalSummary type/builder in the task subsystem that deterministically derives a compact summary from TaskRecord/TaskEventRecord/Task Flow state without removing existing raw fields. The summary should include current state, last good checkpoint when the data is available, current blocker when present, and recommended next action when known; do not invent a last-good checkpoint from unavailable history. Make CLI task rows/show output, /tasks, /status, session_status, task-domain views, and task notification/status rendering consume the shared builder where appropriate. Keep core generic and extension-agnostic, preserve public JSON compatibility, and update docs/changelog only for user-facing behavior. Do not refactor unrelated task storage, Task Flow scheduling, or plugin runtime APIs beyond additive summary exposure.
