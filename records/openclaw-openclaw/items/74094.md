## Summary

Keep open. Current main still has raw task status fields, cleanup-aware snapshots, and several user-facing task formatters, but it does not provide the requested shared compressed operational-summary contract with current state, last good checkpoint, current blocker, and recommended next action reused across CLI, status/chat, session_status, plugin views, and notifications.

## What This Changes

Add a shared compact operational summary for task status and reuse it across task CLI output, task summary/status views, and auto-reply rendering.

## Best Possible Solution

Add an additive task operational-summary builder in the task subsystem, with structured fields and a compact text renderer, then have CLI list/show, /tasks, /status, session_status, task-domain/plugin views, and task notifications consume that builder while preserving raw task fields.

Not applicable as a feature request. The verification path is source inspection, and current main lacks the requested shared summary contract across the named surfaces.

## Solution Assessment
