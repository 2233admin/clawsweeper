## Summary

Keep this PR open. Current main still does not implement the PR's intended iOS/shared chat attachment processing: PhotosPicker and paste/file attachment paths pass raw image bytes into `OpenClawChatViewModel`, which rejects inputs over 5 MB before any resize/transcode, stores accepted bytes unchanged, and base64-encodes them from the `@MainActor` send path. The PR remains an active implementation candidate for the still-open iOS large-image bug #68524, with a focused Swift-only diff and follow-up review discussion rather than an obsolete or already-shipped change.

## Best Possible Solution

Keep reviewing this PR as the implementation candidate for #68524: wire client-side resize/compression and source-metadata stripping into `OpenClawChatViewModel.addImageAttachment`, keep the ImageIO logic consolidated in `JPEGTranscoder`, retain the new processor/privacy tests, resolve the remaining contract/comment and device-memory questions, then merge after targeted Swift tests plus maintainer device validation.
