## Summary

Keep this PR open. Current main still rejects chat image attachments over 5 MB before any resize and then sends raw attachment bytes as base64 from the MainActor. The existing JPEGTranscoder path covers camera capture, not the chat composer attachment path, so the PR is still a plausible implementation candidate for the open iOS composer bug it references.

## Best Possible Solution

Review and refine this PR as the candidate fix for the iOS chat composer image path: wire client-side resize/compression into `OpenClawChatViewModel.addImageAttachment`, prefer or consolidate with the existing `JPEGTranscoder` where practical, add the missing processor and view-model regression tests, and keep the linked iOS bug open until the PR lands.
