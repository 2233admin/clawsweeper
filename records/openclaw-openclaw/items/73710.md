## Summary

Keep this PR open. Current main still has the iOS composer problem this PR targets, and the branch is useful in shape, but the latest diff still routes a claimed long-edge policy through a width-only transcoder path, so portrait/tall images can exceed the advertised 1600 px long-edge cap; the contributor's own device evidence showing 1600x2133 outputs supports that source-level finding.

## What This Changes

The PR adds an iOS chat image-processing adapter that re-encodes attachments as JPEG with resize, byte-budget, and metadata-stripping policy, wires the chat view model to run it off the MainActor, extends `JPEGTranscoder` with alpha flattening, and adds Swift tests.

## Best Possible Solution

Land a corrected version that clamps the actual oriented long edge, keeps the metadata-stripping and alpha-flattening behavior centralized in `JPEGTranscoder`, adds portrait and already-narrow-tall coverage, removes or rewrites the misleading 5 MB safety-net comment, and validates both the OpenClawKit Swift tests and a physical iPhone composer/send round trip.

## Reproduction Assessment

Yes. A source-level reproduction is any portrait/tall image whose oriented width is at or below 1600 while height exceeds 1600; the PR passes `maxDimension` as `maxWidthPx`, and the transcoder only scales from oriented width, so the long edge can remain over the intended cap.

## Solution Assessment

No, not as currently proposed. Reusing `JPEGTranscoder` off the MainActor is the right maintainable direction, but the implementation needs a true long-edge mode or equivalent adapter calculation before it is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Clamp the actual long edge:** `apps/shared/OpenClawKit/Sources/OpenClawKit/ChatImageProcessor.swift:54`
  - body: `maxDimension` is documented and tested as a longest-edge cap, but this call passes it as `maxWidthPx`. Because `JPEGTranscoder` only scales from oriented width, portrait images can still return above 1600 px on the long edge, and already-narrow tall images may avoid resizing entirely. Please compute the scale from `max(orientedWidth, orientedHeight)` or add a true long-edge transcoder mode with portrait coverage.
  - confidence: 0.96
- **[P3] Remove the unreachable 5 MB post-check:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1160-1163`
  - body: `processForUpload` maps `JPEGTranscodeError.sizeLimitExceeded` into an error before returning, and successful output is constrained by the 3.5 MB processor budget. That makes this post-check unreachable through the current path, and the comment describes a best-effort return behavior the processor does not implement.
  - confidence: 0.86
