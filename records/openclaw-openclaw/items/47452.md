## Summary

Current main does not appear to fix the reported qiniu + moonshotai/kimi-k2.5 Anthropic-compatible replay path. Native bundled Kimi/Moonshot provider paths have newer thinking-default fixes, but the issue uses a custom `qiniu` provider with `api: "anthropic-messages"`; the unowned-provider transcript policy still only drops Anthropic thinking blocks for Claude model ids. The related PR #47459 identified the same direction but was closed unmerged, so this remains a valid narrow bug.

## Best Possible Solution

Keep this issue open and add a narrow regression fix in the transcript replay policy for Anthropic-compatible custom providers/models that declare thinking or reasoning unsupported, with coverage for `qiniu` + `moonshotai/kimi-k2.5` over `anthropic-messages`. The fix should preserve existing Claude signed-thinking behavior and native Kimi/Moonshot plugin hooks.
