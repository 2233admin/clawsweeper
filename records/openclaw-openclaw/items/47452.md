## Summary

Keep open. Current main still does not give the reported custom Qiniu Anthropic Messages route a policy that drops replayed `thinking` blocks for `moonshotai/kimi-k2.5`; the generic fallback remains Claude-id scoped, and the earlier fix PR was closed unmerged.

## What This Changes

The issue requests that Anthropic-compatible custom providers such as Qiniu/Kimi stop replaying `thinking` content blocks when the configured model does not support them.

## Best Possible Solution

Keep this open until a maintainer chooses and implements either a provider-owned Qiniu/Kimi replay policy or a documented generic Anthropic-compatible replay policy/config surface, with coverage proving Kimi drops or converts replayed `thinking` blocks while Claude signed-thinking replay remains preserved.
