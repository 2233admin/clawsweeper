## Summary

Keep open. Current main still does not implement the reported custom Qiniu Anthropic-compatible Kimi behavior: the unowned Anthropic replay fallback only enables `dropThinkingBlocks` for Claude model IDs, and the reported `supportsReasoningEffort` flag is not an Anthropic replay-policy control. The related fix PR was closed unmerged, so there is no implemented or superseding fix to close against.

## What This Changes

The issue requests that OpenClaw stop replaying Anthropic `thinking` blocks to a custom `qiniu/moonshotai/kimi-k2.5` route when the model/provider configuration declares that reasoning output is unsupported.

## Best Possible Solution

Keep this open until a maintainer chooses and implements a provider-owned or documented generic replay policy for Anthropic-compatible routes that reject `thinking` blocks, with regression coverage proving `qiniu/moonshotai/kimi-k2.5` drops or converts historical thinking blocks before replay while preserving signed-thinking behavior for providers that need it.
