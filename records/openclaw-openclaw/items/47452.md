## Summary

Keep open. Current main still preserves `thinking` blocks for the reported custom `qiniu` Anthropic Messages route because the unowned fallback only drops historical thinking for Claude model IDs, and the related fix PR was closed unmerged rather than shipped.

## What This Changes

The issue requests that OpenClaw stop replaying Anthropic `thinking` content blocks to `moonshotai/kimi-k2.5` through a custom `qiniu` `anthropic-messages` provider when the model is configured without reasoning support.

## Best Possible Solution

Add a documented replay-policy path for Anthropic-compatible routes that reject `thinking` content, with regression coverage proving the reported Qiniu/Kimi route drops or converts historical thinking before replay while Claude signed-thinking behavior remains intact.

## Reproduction Assessment

Yes. The source-level reproduction is to resolve transcript policy for provider `qiniu`, API `anthropic-messages`, and model id `moonshotai/kimi-k2.5`: current main falls through to the unowned fallback without `dropThinkingBlocks`, and the reporter supplied the matching provider HTTP 400.

## Solution Assessment

Unclear. The old PR's provider-capability hint is not the best current fix because that bag is deprecated; a maintainer should choose the current provider-owned or generic config contract before implementation.
