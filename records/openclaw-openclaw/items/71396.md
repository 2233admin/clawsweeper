## Summary

Keep this PR open. Current main still has the Feishu mention-cascade behavior, and the live PR is a focused but not-yet-ready fix because maintainer review identified a remaining explicit-mention regression for card, streaming-card, and post/rich-text paths.

## What This Changes

The PR removes Feishu inbound mentionTargets from reply dispatcher construction and outbound text/card/streaming replies, changes the agent prompt to sanitized context-only mention wording, adds Feishu regression tests, and adds a changelog entry.

## Best Possible Solution

Keep this PR open and have the Feishu plugin owner/maintainer revise or land a focused fix: stop unconditional inherited mentionTargets propagation for fallback/error/ack/ordinary replies, keep mention context sanitized, and preserve intentional explicit mentions through stable {openId, name} context plus a safe agent-authored card/post/streaming-card rewrite path with regression tests.
