## Summary

Keep this PR open. The branch addresses the inherited Feishu mention cascade, but current main still has the cascade behavior and the latest PR head still needs Feishu maintainer review because it removes the existing stable-ID conversion path for intentional card, streaming-card, and post/rich-text mention forwarding.

## What This Changes

The PR updates the Feishu plugin to stop passing inbound `mentionTargets` into reply dispatch, changes the agent prompt to treat mentions as context only with display-name hardening, adds regression tests for plain/card/streaming-card replies, and adds a changelog entry for #70065.

## Best Possible Solution

Land a Feishu-owned fix that disables unconditional inbound `mentionTargets` propagation for fallback, error, ack, ordinary, card, and streaming replies while preserving intentional explicit mentions through stable allowlisted `{openId, name}` context plus a safe text/card conversion path, backed by focused regression tests and the changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:312-314`
  - body: After dispatcher-side mention injection is removed, explicit mentions have to be reconstructed from agent-visible context, but this hint only exposes sanitized display names. In post/rich-text inputs the model often sees only `@name` while the stable `openId` stays internal, so explicit mention-forward replies can degrade to plain text or target the wrong duplicate name.
  - confidence: 0.86
- **[P2] Keep card mention conversion for explicit replies:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Removing `mentions` from card dispatch and streaming-card close drops the only current path that converts stable Feishu open IDs into card-compatible `<at id=...></at>` markup. That means intentional mentions in card or streaming replies can render as plain text and stop notifying users unless an explicit-only rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
