## Summary

Keep this PR open. The target bug is still present on current main, and the PR is useful, but the latest branch still drops the stable-ID conversion path that explicit Feishu mentions need in card, streaming-card, and post/rich-text flows; maintainer discussion has already asked for one more Feishu-owner pass before prepare/merge.

## What This Changes

The PR removes inbound Feishu `mentionTargets` from outbound reply dispatch, changes the agent prompt to treat mentioned users as sanitized context only, adds text/card/streaming-card regression tests, and adds a changelog entry.

## Best Possible Solution

Land a Feishu-owned fix that disables unconditional inherited mention propagation for fallback, error, ack, ordinary, card, and streaming replies while preserving intentional explicit mentions through allowlisted stable `{openId, name}` context and a safe text/card conversion path, backed by focused regression tests and the changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:316`
  - body: The new context hint only gives the model sanitized display names after dispatcher-side `mentionTargets` injection is removed. In post/rich-text inputs the model can see only `@name` text while the stable open ID stays internal, so intentional mention-forward replies cannot reliably produce notifying Feishu mentions when names are duplicated or changed.
  - confidence: 0.86

Feishu card mentions require the `openId` conversion helper that emits `<at id=...></at>`, so explicit mentions in card mode can render as plain text and stop notifying users.
  - confidence: 0.84
