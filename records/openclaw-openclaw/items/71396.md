## Summary

Keep this PR open. The PR fixes the reported Feishu mention cascade on its narrow path, but the latest branch still removes the only existing stable-ID conversion path for intentional mentions in card/streaming-card/post flows, and the PR discussion now has an explicit maintainer handoff for another Feishu pass before preparation.

## What This Changes

This PR changes the Feishu plugin to stop forwarding inbound `mentionTargets` into outbound reply dispatch, rewrites mention prompt context with display-name sanitization, adds Feishu regression tests, and adds a changelog entry for the mention-cascade bug.

## Best Possible Solution

Land a Feishu-owned fix that disables unconditional inherited mention propagation for fallback, error, ack, ordinary, card, and streaming replies while preserving intentional explicit mentions through allowlisted stable `{openId, name}` context and a safe explicit-only text/card conversion path, with focused regression coverage and the changelog entry retained.

## Reproduction Assessment

Yes. The source path on current main is high-confidence: incoming Feishu mentions become `ctx.mentionTargets`, are passed into reply dispatch, and are converted into Feishu `<at>` markup on first outbound text/card chunks and streaming-card close.

## Solution Assessment

No. The PR is directionally right for the cascade, but removing dispatcher mention metadata without replacing explicit-only stable-ID conversion is not the best complete fix for the Feishu mention contract.
