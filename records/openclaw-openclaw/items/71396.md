## Summary

Keep this PR open. Current main still has the Feishu mention-cascade behavior, and the PR is a focused fix for a real linked bug, but the latest branch still needs maintainer review because it removes the existing explicit mention conversion path for card/streaming/post flows while exposing only display-name context.

## What This Changes

This PR changes the Feishu plugin to stop passing inbound `mentionTargets` into outbound reply dispatch, rewrites the agent prompt to treat mentioned users as sanitized context only, adds regression tests for plain/card/streaming replies, and adds a changelog entry.

## Best Possible Solution

Land a Feishu-owned fix that disables unconditional inbound `mentionTargets` propagation for fallback, error, ack, ordinary, card, and streaming replies while preserving intentional explicit mentions through stable allowlisted `{openId, name}` context plus a safe conversion path for text/card delivery, backed by focused regression tests and the changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:316`
  - body: The new context hint only gives the model sanitized display names after dispatcher-side `mentionTargets` injection is removed. In post/rich-text inputs the model can see only `@name` text while the stable open ID stays internal, so an intentional mention-forward reply cannot reliably produce a notifying Feishu mention when names are duplicated or changed.
  - confidence: 0.86
- **[P2] Keep card mention conversion for explicit replies:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Card sends now call `sendStructuredCardFeishu` without mention metadata, and streaming-card close sends raw combined text. Feishu card mentions require the card form produced from an open ID (`<at id=...></at>`), so explicit mention replies in card mode can render as plain text and stop notifying users unless a safe explicit rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
