## Summary

Keep this PR open. Current main still implements the Feishu mention cascade that the PR is trying to remove: inbound `mentionTargets` are still captured, passed into reply dispatch, and converted into outbound Feishu mention markup for plain-text, card, and streaming-card replies. The PR is open and unmerged, and the latest diff is a focused Feishu plugin fix candidate with tests and prompt hardening, but it still deserves maintainer review for explicit card mention semantics and the in-progress security review.

## Best Possible Solution

Keep this PR open for Feishu plugin maintainer review. The best path is to land or revise a focused Feishu plugin fix that stops unconditional inbound mention propagation for fallback, error, ack, plain-text, card, and streaming-card replies; keeps mentioned users as sanitized context-only prompt data; and explicitly resolves whether intentional card/streaming-card mentions need a separate conversion path before merge.
