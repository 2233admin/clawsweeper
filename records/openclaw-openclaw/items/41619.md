## Summary

Keep this issue open. Current main still exposes the reported Gemini CLI OAuth onboarding path, still uses the desktop localhost:8085 callback followed by token exchange, and still only falls back to manual mode for listen/port errors. The issue discussion is concrete and corroborated, including a report that no process was listening on localhost:8085 during the OAuth step, and I did not find code, docs, tests, or related-history evidence that the reported AbortError/auth-completion regression has been fixed or superseded.

## Best Possible Solution

Keep this issue open for the Google plugin owner. The likely maintainer path is to reproduce Gemini CLI OAuth on current main or the latest release with a real macOS/Homebrew setup, then harden extensions/google/oauth.ts and the shared callback helper. A good fix should make the localhost callback listener lifecycle explicit before browser handoff, add recoverable manual fallback or clearer timeout handling for AbortError/token-exchange failures, and add a focused regression test for the desktop callback flow.
