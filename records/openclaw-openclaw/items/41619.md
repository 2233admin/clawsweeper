## Summary

Keep this issue open. Current main still ships the Google Gemini CLI OAuth path and still opens the browser before the localhost callback listener is created, matching the reported no-listener/AbortError failure mode. The discussion has multiple matching reports, and the related #48879 thread is a different post-login model/API 404 problem rather than a superseding auth-flow fix.

## Best Possible Solution

Keep this issue open for the Google plugin owner. Reproduce Gemini CLI OAuth on current main or the latest release, then make the local callback listener ready before browser handoff or make manual mode immediately available. Add a focused regression test around the desktop callback sequence and no-listener/timeout behavior.
