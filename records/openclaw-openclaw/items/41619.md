## Summary

Keep this issue open. Current main still exposes the Gemini CLI OAuth onboarding path and still runs the desktop flow through a localhost:8085 callback followed by token exchange, with no source, docs, test, or history evidence that the reported AbortError/auth-completion regression has been fixed. The issue discussion is also concrete and corroborated: multiple users confirmed the failure, and one commenter specifically observed that no process was listening on localhost:8085 during OAuth.

## Best Possible Solution

Keep this issue open for the Google plugin owner. The likely maintainer path is to reproduce Gemini CLI OAuth on current main or the latest release with a real macOS/Homebrew setup, then harden extensions/google/oauth.ts and the shared callback helper as needed. A good fix should make the localhost callback listener lifecycle explicit before browser handoff, improve manual fallback or timeout handling for AbortError/token-exchange failures, and add a focused regression test for the desktop callback flow.
