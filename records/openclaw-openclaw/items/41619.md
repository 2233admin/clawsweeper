## Summary

Keep this issue open. Current main still ships the Gemini CLI OAuth path, still opens the browser before the localhost callback listener is started, still uses the guarded token-exchange fetch that can surface AbortError, and still reports OAuth failures with the same generic Gemini CLI access help. The latest release tag has the same auth-flow ordering, so this is not implemented on main or in the shipped release.

## Best Possible Solution

Keep this open for the Google plugin/auth owner. Reproduce Gemini CLI OAuth on current main or v2026.4.26, then make the localhost callback listener ready before browser handoff and improve token-exchange timeout/network/account diagnostics. Add focused regression coverage for listener readiness before openUrl and for AbortError handling in the OAuth exchange path.
