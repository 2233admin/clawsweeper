## Summary

Keep this issue open. Current main still has the reported Gemini CLI OAuth onboarding path, still uses the fixed localhost:8085 callback, and still starts the browser before waiting for the local callback. The code only falls back to manual mode for listen/port errors, while the reported failure is an AbortError during/after callback and token exchange. I did not find source, docs, tests, or current-main history showing this regression has been fixed or superseded.

## Best Possible Solution

Keep this issue open for the Google plugin owner. The best path is to reproduce Gemini CLI OAuth on current main or the latest release with a real macOS/Homebrew setup, then harden `extensions/google/oauth.ts` and the shared callback helper so the localhost listener lifecycle is explicit before browser handoff, manual fallback is available for no-listener/timeout/AbortError cases, and a focused regression test covers the desktop callback flow.
