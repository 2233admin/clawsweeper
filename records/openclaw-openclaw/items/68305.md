## Summary

Keep open. Current main still sends the misleading Telegram-only `/reasoning stream` acknowledgment, while Feishu/Lark reasoning-preview support is present in source and covered by tests. The issue is also paired with open PR #72709 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for the `/reasoning stream` acknowledgment to stop saying Telegram-only when Feishu/Lark also supports reasoning previews.

## Best Possible Solution

Update the shared directive acknowledgment so it no longer claims stream reasoning is Telegram-exclusive, and align user-facing reasoning docs with the actual Feishu/Lark preview support.
