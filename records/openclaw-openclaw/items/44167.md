## Summary

Keep this PR open. Current main still only retries service/rate-limit embedding errors and still retries the original full text or structured batch without a transport-error split fallback. The PR remains a focused, non-security-sensitive implementation candidate for the batch reindex bug tracked by #44166, while #71784 is a separate live single-query reliability path.

## Best Possible Solution

Keep this PR open for maintainer review, then land it or an equivalent focused memory-core patch that adds a narrow transport retry classifier, reuses the existing backoff loop, recursively splits exhausted multi-item text and structured embedding batches while preserving order, and keeps focused regression coverage in `extensions/memory-core/src/memory/manager-embedding-policy.test.ts`. Treat #71784 and #56815 as related but broader follow-ups rather than blockers for this batch retry fix.
