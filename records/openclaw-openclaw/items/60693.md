## Summary

Keep open. Current main covers several subitems (Discord REST proxy wiring, ACPX staged Codex/Claude ACP dependencies, and model-precedence docs), but the umbrella issue still has unresolved asks: `openclaw models set` only updates the default model config, and I did not find an ACP allowedAgents repair/warning path that filters missing or disabled agent directories before startup identity reconciliation.

## Best Possible Solution

Leave the issue open or split it. Mark the Discord REST proxy and ACPX packaging portions as already handled, then decide the remaining product path: either add a deliberate bulk model-switch/cleanup command for defaults, per-agent overrides, and cron payloads, or document that those overrides are intentionally separate. For ACP allowedAgents, add a doctor/fix or startup warning that identifies missing/unavailable ACP harness ids without repeatedly failing startup identity reconciliation, or explicitly document the required manual cleanup.
