## Summary

Current main still has the reported defaults-only heartbeat scope bug: `agents.defaults.heartbeat` is documented as global behavior, but the enablement helper and scheduler still fall back to only the default agent when no per-agent heartbeat block exists. The zero-config part conflicts with current docs that describe default-on heartbeat cadence disabled via `0m`, so this should stay open for maintainer policy alignment rather than be auto-closed or queued blindly.

## Best Possible Solution

Keep this issue open as the canonical tracker. Maintainers should first decide whether zero-config heartbeats remain default-on or become disabled by default, then update `src/infra/heartbeat-summary.ts`, `src/infra/heartbeat-runner.ts`, status/health summaries, heartbeat docs, and focused regression tests so defaults-only multi-agent configs behave consistently. The narrower doctor-warning request can remain tracked separately in #46101.
