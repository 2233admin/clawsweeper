## Summary

Current main still matches the reported defaults-only heartbeat scope bug. `agents.defaults.heartbeat` is documented as global behavior, but the implementation only treats per-agent `agents.list[].heartbeat` blocks as explicit heartbeat scope and otherwise falls back to the default agent. The zero-config expectation needs separate product clarification because docs currently describe default heartbeat cadence as enabled, but the defaults-only multi-agent bug remains real.

## Best Possible Solution

Keep this issue open as the canonical tracker. The likely fix belongs in the heartbeat owner path: align `isHeartbeatEnabledForAgent()`, `resolveHeartbeatAgents()`, and scheduler/summary coverage so `agents.defaults.heartbeat` applies to the configured agent set when no per-agent heartbeat blocks exist, while preserving current explicit-agent scoping when any `agents.list[].heartbeat` block is present. Add regression coverage for defaults-only multi-agent enablement and scheduling, then make a separate product/docs decision on zero-config heartbeat behavior.

- Fixing defaults-only scope will increase heartbeat activity for users who configured `agents.defaults.heartbeat` with multiple agents but were unknowingly getting only the default agent scheduled.
