## Summary

Keep open. Current main still models Discord `actions.messages` as one boolean gate and applies it to read/fetch/send/edit/delete behavior, so the requested backward-compatible granular `send`/`read`/`edit`/`delete` gates are not implemented. The April 26 review comment on the issue remains accurate after checking the current main tree at 28ff82dcdae7f7fb11491deefe48bd82b68eb0bb.

## Best Possible Solution

Implement a backward-compatible Discord message action gate shape, for example `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }`, treating `fetchMessage` as read, preserving the old boolean shorthand, and applying the same semantics to discovery, runtime checks, account overrides, strict schema, generated config metadata, docs, and focused Discord/config tests.
