## Summary

Keep this issue open. Current main partially moved past the original reproduction because the Sessions table no longer has the described provider column and session rows derive their display model identity through gateway session utilities, but the Usage surface still has raw-provider paths: `sessions.usage` returns raw `modelProvider`/`providerOverride` from the store, Usage overview/detail render those values directly, and usage aggregates group by raw transcript provider strings. I also did not find a current `bailian` alias in source, so any fix should use the existing provider normalization contract and avoid adding that alias without maintainer confirmation.

## Best Possible Solution

Keep this open and land a narrow fix that canonicalizes provider ids for Control UI Usage display, filters, and aggregates using the existing `normalizeProviderId` contract. Preserve stored config/session values, avoid introducing a new `bailian` alias without maintainer approval, and add regression tests for known current aliases such as `z.ai`/`z-ai` and `bedrock`/`aws-bedrock`.
