## Summary

Keep open. Current main has a partial per-agent surface through flat `agents.list[].params`, and it covers `temperature`/`maxTokens`, but it still does not accept the requested `generation` block or generically forward `topP`/`top_p`, `topK`/`top_k`, presence penalties, or frequency penalties through the shared agent stream path. Related #32496 is the closest open sampling-parameter passthrough item; #71350 and #59168 are broader adjacent design discussions, not replacements.

## Best Possible Solution

Keep this open until maintainers choose the public config contract. The narrow implementation path is to extend the existing `agents.list[].params` contract, or add a documented `agents.list[].generation` alias if that shape is preferred, then normalize camelCase and snake_case sampling keys and forward only supported fields per provider/transport. Coordinate the sampling-field forwarding with #32496, and keep #71350/#59168 linked as broader design context rather than treating them as replacements.
