## Summary

Keep this PR open for maintainer review. The user-visible draft-loss bug is still present on current main, but this branch is stale against the centralized session-switch helper, misses a visible sidebar switch path, and overlaps the open #54655 implementation.

## What This Changes

The PR adds a module-level in-memory chat draft cache and wires save/restore calls into selected Control UI session and agent switching callbacks.

## Best Possible Solution

Ship one rebased Control UI change that stores composer drafts through the centralized session-switch helper, scopes the cache by active gateway/session context, prunes empty and deleted-session drafts, preserves the existing per-session queue behavior, and adds focused A-to-B-to-A restoration coverage for session dropdown, agent switch, and sidebar session selection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope cached drafts to the active gateway:** `ui/src/ui/app-render.helpers.ts:37`
  - body: `chatDraftCache` is module-level and keyed only by `sessionKey`, but the Control UI can reconnect the same tab to another `gatewayUrl` where common keys like `main` are reused. A draft typed in one gateway can be restored into another gateway's session, so include the gateway/context in the key or clear the cache on gateway changes.
  - confidence: 0.82
- **[P2] Preserve drafts from sidebar session selections:** `ui/src/ui/app-render.ts:1438`
  - body: `onSessionSelect` still switches sessions with `state.setSessionKey(key)` and no save/restore wrapper. Selecting a session from the chat sidebar can carry the outgoing draft into the destination session and cannot restore that destination session's cached draft.
  - confidence: 0.86
- **[P3] Clear deleted sessions from the draft cache:** `ui/src/ui/app-render.helpers.ts:37`
  - body: The module-level draft map has no clear helper or deletion hook, so drafts for removed sessions remain in memory until page reload. Wire cache eviction into the session deletion flow to keep stale draft text bounded.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: No supply-chain or code-execution risk was found, but the new in-memory draft cache has a low-severity draft isolation issue because it is keyed only by session key.

Concerns:

- **[low] Drafts can bleed across gateway contexts:** `ui/src/ui/app-render.helpers.ts:37`
  - body: The PR stores unsent text in a module-level cache keyed only by `sessionKey`; after reconnecting the same Control UI tab to another gateway with a reused key like `main`, previously typed draft text can be restored in the wrong workspace/gateway view. This is local to the browser session but can expose sensitive unsent text to the wrong context.
  - confidence: 0.74
