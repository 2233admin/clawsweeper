## Summary

Keep this PR open. Current main still loses unsent Control UI chat drafts on session switch, so the user problem is real, but the PR head is stale against the now-centralized switch helper and has blocking correctness gaps before merge.

## What This Changes

The PR adds an in-memory Control UI draft cache plus save/restore helpers and wires them into selected session and agent switching paths.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts in the centralized session-switch helper, scopes draft storage to the active gateway/session context, evicts empty and deleted-session entries, preserves the existing per-session queue behavior, and adds focused tests for session dropdown, agent switch, and sidebar selection.

## Reproduction Assessment

Yes. On current main, type a draft in the Control UI composer, switch sessions through a path that calls switchChatSession(), and resetChatStateForSessionSwitch() clears state.chatMessage before the original draft can be restored.

## Solution Assessment

No, not as posted. The in-memory cache is a reasonable narrow approach, but the PR needs gateway scoping, sidebar coverage on its head, deletion cleanup, tests, and a rebase onto the current centralized switch helper.

  - confidence: 0.86
