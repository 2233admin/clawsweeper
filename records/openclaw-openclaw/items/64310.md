## Summary

Keep this PR open. Current main still has the mutable module-global package-path list and still feeds `buildMissingPackages` from accumulated module state, while the proposed one-file patch addresses the actual idempotency bug without introducing a blocking correctness problem. The related closed Tlon dependency report is adjacent context for package audits, but it does not supersede this focused script fix.

## What This Changes

The PR changes `scripts/audit-seams.mjs` so workspace package paths are collected into a fresh sorted/de-duped list and passed into missing-package auditing instead of being accumulated in mutable module-global state.

## Best Possible Solution

Land a focused version of this script change, ideally with a small regression test in `test/scripts/audit-seams.test.ts` that calls `main()` twice in-process and verifies stable `missingPackages` package paths; the remaining redundant sort can be removed or left as harmless defensive ordering.
