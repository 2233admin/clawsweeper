## Summary

Keep open. Current main still has only DM-specific session scoping; group/channel peers still resolve to isolated non-main session keys, config schemas/types/docs do not expose `groupScope`, and the related implementation PR #47234 is open but unmerged and not a reason to close this issue yet.

## Best Possible Solution

Keep this issue open as the focused tracker, and review or refresh #47234 rather than closing this issue prematurely. The best implementation is an additive routing/config contract for selected group/channel peers to use the main or a named shared session, wired through session-key construction, route resolution, strict schemas/types/generated metadata/help, relevant channel group/topic configs, docs, and regression tests for inbound/outbound delivery, DM unaffected behavior, queue serialization, and privacy/sandbox boundary notes. Coordinate naming and scope with #19929 before landing.
