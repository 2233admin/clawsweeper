## Summary

Keep open. The supplied current-main commit still builds Feishu structured-card headers from the raw `identity.emoji` string plus the agent name, and the send/streaming card builders pass that title through verbatim, so descriptive non-emoji text can still appear in card headers.

## What This Changes

The issue requests that Feishu card headers stop rendering non-emoji descriptive text from `identity.emoji` when long messages use structured card format.

## Best Possible Solution

Add a Feishu-local card-header title resolver shared by reply, streaming, presentation, and direct outbound card paths. It should preserve explicit presentation titles and accepted header-disable configuration while preventing descriptive non-emoji text from `identity.emoji` from appearing in card headers, with focused Feishu regression tests for CJK descriptive text, pure emoji values, name-only headers, and disabled headers.
