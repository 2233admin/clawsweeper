## Summary

Keep open. Current main still does not provide this command, so the PR is not obsolete; however the branch is stale against the current channel/plugin architecture and the proposed prompt preview does not reliably match runtime prompt assembly.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` subcommand, exports its handler, assembles an embedded system prompt with optional model/channel overrides, and prints prompt metadata.

## Best Possible Solution

Land a rebased implementation that renders through shared prompt-resolution helpers used by embedded runtime, keeps channel-specific behavior behind plugin prompt APIs, includes provider prompt contributions/transforms, and adds focused CLI/runtime docs plus tests for model overrides, channel guidance, workspace selection, and provider overlays.

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use plugin prompt seams for channel guidance:** `src/commands/agents.commands.view-system-prompt.ts:29-31`
  - body: The new command imports `../signal/*` and `../telegram/*`, but current main no longer has those core modules. A rebase will fail typecheck, and the command should use the same generic channel/plugin prompt seams that runtime assembly uses.
  - confidence: 0.95
- **[P2] Build prompt params from the agent workspace:** `src/commands/agents.commands.view-system-prompt.ts:99`
  - body: Passing `process.cwd()` into `buildSystemPromptParams` makes the displayed Runtime section depend on the shell launch directory. Embedded runtime uses the effective agent workspace for both `workspaceDir` and `cwd`, so this can show a different prompt than a real run.
  - confidence: 0.9
- **[P2] Preserve runtime workspace notes:** `src/commands/agents.commands.view-system-prompt.ts:119-123`
  - body: Injecting `Loaded context files: ...` through `workspaceNotes` adds prompt text production does not send and misses the BOOTSTRAP/runtime reminders production does send. The file contents already flow through `contextFiles`, so this reduces preview fidelity.
  - confidence: 0.86
- **[P2] Apply provider prompt contributions before printing:** `src/commands/agents.commands.view-system-prompt.ts:147-170`
  - body: Runtime resolves provider prompt contributions, passes them into `buildEmbeddedSystemPrompt`, and then applies provider prompt transforms. This command prints the direct builder output, so provider overlays and plugin text transforms can be absent from the displayed assembled prompt.
  - confidence: 0.88
