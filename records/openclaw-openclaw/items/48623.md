## Summary

Keep open. Current main still records `lastGood`, but auth-profile ordering intentionally ignores it in favor of round-robin `lastUsed` ordering, and both session selection and embedded-runner profile rotation consume that order. The shared-timeout portion appears partly mitigated because each embedded attempt receives the configured timeout and schedules its own attempt timer, but the central `lastGood`/failover-ordering request and the per-slug-task model override remain unresolved.

## Best Possible Solution

Keep this issue open and split the remaining work. The core fix should add failover-aware auth-profile selection that can prefer the most recently good profile after rate-limit/auth failures while preserving normal round-robin where intended. Separately decide whether slug generation needs a task-specific model override, since current main only honors the default agent model and general timeout.
