## Summary

Keep open. The PR is useful and narrow, but it does not fully fix the behavior it describes: the added billing regex would help raw and HTTP 400 budget-overrun messages, while HTTP 429 budget-overrun payloads still route to generic rate_limit before billing can win.

## What This Changes

The PR adds a budget-exceeded billing regex to the failover pattern list and targeted tests for raw-message plus HTTP 400 classification.

## Best Possible Solution

Land a narrow failover fix that recognizes budget-exceeded provider text as billing and lets explicit billing/budget messages win over generic HTTP 429 routing, with focused coverage for raw text, HTTP 400, HTTP 429, related OpenRouter 403 budget wording, and negated/unrelated budget text.

## Reproduction Assessment

Yes. A focused classifier check is enough: current main has no budget-exceeded billing pattern, status 400 would fall to format until the new pattern exists, and status 429 currently returns rate_limit unconditionally.

## Solution Assessment

No. The regex addition is a good part of the fix, but the proposed patch is incomplete until the 429 status path and raw leading-429 message classification preserve billing-classified budget payloads.
