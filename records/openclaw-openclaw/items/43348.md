## Summary

Keep open. Current main at 2ccdbc7dd93bfcb08e100a479af79d8d8a211d97 still does not implement the PR's intended agent-facing `acp_send` tool, `sessions_cancel` tool, or `sessions.cancel` Gateway RPC. Main has partial overlap through `sessions_send` fire-and-forget behavior, owned ACP-child A2A skipping, `sessions.abort`, `/acp cancel`, and `AcpSessionManager.cancelSession()`, but those do not fully replace the proposed surfaces. The PR also remains security-sensitive because the proposed cancellation path needs session visibility/ownership checks and the proposed ACP send path needs ACP-only target validation before it can be merged safely.

## Best Possible Solution

Keep this PR open as a rework candidate. Rebase it onto current main, decide whether `acp_send` is still needed beyond current `sessions_send` behavior, and if agent-facing ACP cancellation is still desired, implement it through the existing `sessions.abort`/ACP cancellation model or a clearly documented new RPC with session resolution, visibility and ownership checks, ACP-only target validation, protocol/docs/tool-catalog updates, and focused regression tests.

## Risks / Open Questions

- The proposed `sessions_cancel` path is authorization-sensitive: without reusing the current session visibility, ownership, and A2A policy checks, an agent that learns another ACP session key could cancel work outside its intended scope.
- The proposed `acp_send` path should reject non-ACP targets before fire-and-forget dispatch, otherwise normal/subagent session replies can be silently lost because no ACP callback path exists.
- Adding `sessions.cancel` alongside shipped `sessions.abort`, `/acp cancel`, task cancellation, reset/delete cleanup, and ACP runtime cancellation could fragment cancellation semantics unless naming, docs, scopes, and tests are reconciled.
- The PR head commit is not present in the local checkout, so the security review relies on the provided PR diff/comment context plus current-main source inspection rather than local checkout of the PR branch.
