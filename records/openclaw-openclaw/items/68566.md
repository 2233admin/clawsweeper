## Summary

Keep open. Current main still records non-timeout failover reasons, including `format`, as auth-profile failures and the embedded runner does not forward raw error text/status into `markAuthProfileFailure`, so this PR is not implemented on main. The proposed branch also needs maintainer review because its unconditional raw HTTP 400 cooldown bypass conflicts with current classifier/tests that deliberately distinguish 400 billing, rate-limit, model-not-found, context-overflow, and no-body cases.

## What This Changes

This PR changes auth-profile failure handling to skip cooldown writes for `format` failures and raw HTTP 400s, plumbs embedded-runner prompt error text into status extraction, adds focused usage tests, and adds a changelog entry.

## Best Possible Solution

Revise the PR so only confirmed malformed-request/`format` failures skip auth-profile cooldowns while preserving current HTTP 400/422 classifier distinctions for billing, auth, rate-limit, model-not-found, context-overflow, and no-body cases. Add focused coverage for both the usage helper and any embedded-runner error-text plumbing, then align the changelog entry with OpenClaw attribution and ordering rules.
