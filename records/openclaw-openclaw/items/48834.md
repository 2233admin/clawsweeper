## Summary

Keep open. Current main has part of the Feishu ACP/session-binding foundation and a generic conversation-binding path, but it still does not implement this PR's central remaining behavior: Feishu ACP block delivery when card streaming is inactive, Feishu threadBindings config/lifecycle support, and Feishu-facing /focus / /unfocus docs/command surfaces. The PR is not obsolete; it needs a focused rebase/revision against the current generic channel conversation-binding contract.

## Best Possible Solution

Keep this PR open and ask for a focused revision against current main: implement Feishu ACP block delivery without final-dedupe truncation, expose Feishu current-conversation binding through the generic ChannelConversationBindingSupport contract, add Feishu threadBindings schema/config metadata/docs, wire idle/max-age lifecycle hooks and manager startup through policy-aware channel lifecycle, and cover DM/topic/group_topic_sender routing plus block-delivery behavior with targeted tests. The ACP env/SDK boundary changes should receive a separate security-oriented maintainer pass before merge.

## Risks / Open Questions

- The PR touches security-sensitive runtime and public-contract surfaces: ACP child-process environment handling, session-binding registration, and plugin SDK exports. The provided changed-file list shows no workflows, lockfiles, dependency sources, or package-publishing metadata changes, but those runtime/SDK changes still need focused maintainer review before merge.
- ACP block delivery must avoid the earlier review risk where block chunks routed through final-reply dedupe could drop repeated ACP deltas. The PR's later commits appear to move away from promoteAcpBlocksToFinals, but this needs current-main review and targeted tests.
- Feishu sender-scoped topic commands must match live routing exactly, including merged per-account/per-group config, otherwise /focus, /unfocus, /session, and reset can mutate a conversation id different from the one inbound routing uses.
- The current PR branch is based on older surfaces; current main now has generic conversation-binding helpers, so the clean path is likely a rebase or maintainer replacement using those generic contracts rather than adding Feishu-specific core deep imports.
