## Summary

Keep open. Current main has the Feishu session-binding foundation from the earlier merged work and a generic command resolver that can resolve Feishu DMs/topics, but this PR is not obsolete: current main still drops Feishu non-streaming block payloads, does not expose Feishu `threadBindings` config/lifecycle controls, and still has command/docs surfaces that omit Feishu for `/focus` and `/unfocus`. The PR also has unresolved review risk around sender-scoped Feishu account config, so it needs revision or a narrower replacement rather than cleanup closure.

## Best Possible Solution

Keep this PR open, but require a focused revision or maintainer replacement that lands the remaining Feishu behavior on current main: deliver Feishu ACP block chunks when streaming/card handling is inactive, add Feishu `threadBindings` schema and lifecycle hooks through `ChannelConversationBindingSupport`, update `/focus`/`/unfocus` docs and command metadata, add targeted tests for Feishu DM/topic/sender-scoped routing, and resolve the active-account sender-scope review concern before merge.
