## Summary

Keep this PR open. Current main has absorbed part of the Feishu conversation-binding foundation and generic `/focus` plumbing, but it still does not implement the PR's full user-visible change: Feishu ACP block payloads are still suppressed when not streaming, Feishu thread binding config/lifecycle support is incomplete, command/docs text still describes `/focus` and `/unfocus` as Discord/Telegram-only, and the PR needs a current-main rebase/security pass before it is mergeable.

## Best Possible Solution

Keep this PR open for a focused rebase against current main: preserve the generic Feishu conversation resolver already on main, fix non-streaming Feishu ACP block delivery without routing block deltas through final-text dedupe, add Feishu `threadBindings` schema/config metadata/startup gating plus idle and max-age lifecycle hooks through the generic plugin APIs, update `/focus` and `/unfocus` command/docs text, and add targeted tests for DM, topic, `group_topic_sender`, disabled policy, lifecycle updates, and repeated ACP block chunks. The ACP environment change should get an explicit security review before merge.
