## Summary

Current main still does not implement the requested operator-owned paired-device rename workflow. The CLI and Control UI can display client-supplied `displayName` metadata, but there is no `openclaw devices rename` command, no gateway rename method/schema/handler, and no persisted operator label with precedence over reconnect metadata.

## Best Possible Solution

Implement paired-device label management end to end: add an additive gateway method/schema/handler for paired-device rename, persist an operator-owned display-name override with documented precedence over reconnect metadata, expose `openclaw devices rename --device <id> --name <displayName>`, add a Control UI rename action, cover storage/protocol/CLI/UI behavior with focused tests, and update public devices/protocol docs. Keep `node.rename` separate and keep #23916 focused on broader metadata display.
