## Summary

Keep open. Current main has runtime-deps mirroring for a small hard-coded core set and for root-dist imports that can be attributed to bundled plugin package.json owners, but it does not yet implement the issue's requested invariant that every built dist bare import present in package.json dependencies is retained in the external runtime-deps mirror. Seven of the nine reported packages are direct root dependencies but are not declared by any bundled plugin package.json, so the current owner-filtered mirror logic would not include them.

## Best Possible Solution

Implement a generated or runtime-scanned package-root mirror set for all built `dist` bare imports that are backed by root `dependencies` or `optionalDependencies`, then retain/install those specs in the external runtime-deps root alongside plugin-owned deps. Add regression tests that include the reported packages or representative root-only imports, and keep the release/postpublish check aligned so future dist/import drift fails before publish.
