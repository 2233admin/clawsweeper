## Summary

Keep this issue open. Current main still has only global agent-run timeout resolution plus explicit request/cron overrides; the requested channel-aware config key, schema/type surface, docs, and runtime selection for Telegram, Discord, WebChat, and cron defaults are not present.

## What This Changes

Request per-channel agent-run timeout overrides for Telegram, Discord, WebChat, and cron while preserving existing global timeout and explicit cron/request override behavior.

## Best Possible Solution

Add a shared channel-aware agent-run timeout resolver that preserves explicit request and cron `payload.timeoutSeconds` precedence, then wire Gateway/WebChat/channel-origin and cron default paths through it with strict config schema, exported types, generated config metadata, docs, and focused regression tests.

## Reproduction Assessment

Yes for the config/runtime gap: current main can be verified by searching for `channelTimeoutSeconds` and following `resolveAgentTimeoutMs` through Gateway chat/agent and cron call sites. The reported OpenAI soft-throttle hang remains environment-dependent.

## Solution Assessment

Unclear until maintainers approve the public API name and default values. The maintainable direction is a single shared resolver used by run entry points, not more Telegram/Discord transport timeout tweaks.
