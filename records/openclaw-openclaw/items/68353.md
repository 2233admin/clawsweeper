## Summary

Keep open. Current main still does not accept a first-class `author` object on `chat.send`, does not persist per-message author metadata on chat-send transcript rows, and projects user attribution from stored `senderLabel` or inbound metadata blocks. The request is concrete and recent, but it changes a public gateway protocol and identity trust semantics, so it needs maintainer/security/product direction before implementation.

## What This Changes

Add optional per-message author metadata to `chat.send` and surface it through `chat.history` and `session.message` as authoritative user attribution for shared multi-user sessions.

## Best Possible Solution

Define an additive gateway protocol contract for per-message authorship that includes validation, transcript storage, display projection through history and live events, UI/client typing, docs, tests, and explicit trust semantics for client-declared versus attested identity.
