## Summary

Current main still does not add `moonshotai/kimi-k2.5` as a curated OpenRouter onboarding/model-picker catalog row. The bundled OpenRouter catalog, docs, and focused tests list `openrouter/auto` and K2.6 only; dynamic OpenRouter model resolution can still run arbitrary refs, but that does not satisfy the selector request. OpenRouter's live models API still lists `moonshotai/kimi-k2.5`, so this is a valid narrow catalog/docs/test follow-up.

## Best Possible Solution

Keep this issue open and make a narrow OpenRouter plugin/catalog update: add `moonshotai/kimi-k2.5` as a bundled selector row with current OpenRouter metadata, keep `openrouter/auto` as the onboarding default, update focused OpenRouter catalog tests and `docs/providers/openrouter.md`, and add a user-facing changelog entry if the picker/catalog change is accepted. If maintainers decide K2.6 supersedes K2.5, document that explicitly instead.
