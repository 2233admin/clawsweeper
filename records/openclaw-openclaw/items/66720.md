## Summary

Keep this PR open for maintainer review. Current main still lacks provider/gateway-specific stripping for the complex `sessions_spawn` fields, so the contribution is not obsolete, but the proposed patch remains incomplete because the new option is not activated by the runtime tool assembly paths that send schemas to providers.

## What This Changes

The PR adds a `simplifiedSchemaForGateway` option that filters `streamTo`, `resumeSessionId`, `lightContext`, `attachments`, and `attachAs` from the `sessions_spawn` tool schema and forwards that option through `createOpenClawTools`.

## Best Possible Solution

Add a provider/deployment-aware schema simplification path at the point where OpenAI-compatible gateway requests are assembled, keep the default and ACP-unavailable schemas unchanged, and cover default, ACP-unavailable, and strict-gateway schema shapes with focused tests.
