## Summary

Keep this PR open. Current main still does not expose `sessionId` or `sessionKey` in the system prompt `Runtime:` line, and the provided PR context shows this branch was repaired by a maintainer as an active implementation candidate. The remaining maintainer decision is the stable prompt contract, especially given the overlapping open `sessionKey` proposal in #66312.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose one stable Runtime prompt contract between this PR’s `sessionId` UUID and #66312’s `sessionKey`, then land the chosen additive metadata path across `RuntimeInfoInput`, `buildRuntimeLine`, command/embedded runner call sites, focused tests, docs, and changelog while preserving contributor credit.
