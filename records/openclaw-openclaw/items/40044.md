Keep this PR open. Current main still does not expose `sessionId` or `sessionKey` in the system-prompt `Runtime:` line, and the supplied PR discussion shows this branch was recently repaired by a maintainer and marked merge-ready/human-review rather than superseded or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best implementation path is to choose `sessionId` versus `sessionKey` as the stable prompt contract, then thread the selected value through all Runtime prompt assembly paths, including the embedded runner and command/context prompt bundle, with focused `buildRuntimeLine` coverage.
