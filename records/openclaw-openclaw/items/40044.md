## Summary

Keep PR #40044 open. Current main still does not expose the current session UUID in the system prompt `Runtime:` line: the runtime prompt types have no `sessionId`, the renderer cannot emit `session=<uuid>`, and the auto-reply prompt path has the target session UUID available but does not thread it into `runtimeInfo`. The PR remains a focused additive agents prompt change rather than obsolete, duplicated, or ClawHub-scoped work.

## Best Possible Solution

Keep the PR open, or implement the same behavior directly on main: add `sessionId?: string` to prompt runtime types, pass `targetSessionEntry?.sessionId` through `buildSystemPromptParams`, render `session=<uuid>` in `buildRuntimeLine`, and add focused runtime-line test coverage for the new segment.
