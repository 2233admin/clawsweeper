## Summary

Keep this PR open. Current main still does not expose the active session UUID in the system prompt Runtime line, and this PR remains a narrow, active implementation candidate with maintainer context preserving the contributor path.

## Best Possible Solution

Keep this PR open and review or land it as the canonical contributor path if sessionId is the desired prompt-visible identifier. The implementation should add optional sessionId plumbing through RuntimeInfoInput, buildAgentSystemPrompt, buildRuntimeLine, and resolveCommandsSystemPromptBundle using targetSessionEntry?.sessionId, with the focused session=<uuid> runtime-line assertion. If maintainers prefer sessionKey, explicitly supersede this PR with #66312 or a maintainer-linked follow-up rather than auto-closing it now.
