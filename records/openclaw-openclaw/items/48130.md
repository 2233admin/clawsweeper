## Summary

Keep this PR open. Current main still contains the Windows registry command parsing bug in the moved browser plugin file, so the PR remains functionally useful but needs a rebase and targeted regression tests.

## Best Possible Solution

Rebase the PR onto current main, apply the regex fix in extensions/browser/src/browser/chrome.executables.ts, and add targeted Vitest coverage for quoted and unquoted Windows registry command strings through resolveBrowserExecutableForPlatform or a narrow exported test seam.
