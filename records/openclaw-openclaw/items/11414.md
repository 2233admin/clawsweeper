## Summary

Keep this issue open. Current main still implements only Telegram forum topic creation/editing through the message plugin; close, reopen, and delete topic lifecycle actions are missing from the shared action names, Telegram discovery/map, runtime dispatch, transport exports, config schema, docs, and focused tests. Open PR #41410 is relevant deletion-only prior art, but it is unmerged and does not cover the full requested lifecycle.

## What This Changes

The issue requests Telegram forum topic lifecycle actions through the message plugin: create, edit, close, reopen, and delete.

## Best Possible Solution

Keep this as the canonical Telegram forum-topic lifecycle tracker and land one coherent Telegram plugin implementation that completes close, reopen, and delete alongside the existing create/edit path across shared action names, Telegram discovery/map, runtime dispatch, transport helpers, config schema/metadata, docs, and focused contract/runtime tests.
