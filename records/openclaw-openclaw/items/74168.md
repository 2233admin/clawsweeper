## Summary

Keep open. Current main still lacks this PR’s intended last-chance blank visible prompt guard: the helper treats replay history as submit-worthy content even when the current prompt is whitespace-only, and the runner only skips when prompt, history, and images are all empty. The PR remains a focused implementation candidate for #74137 with no visible supply-chain or security-sensitive file changes.

## What This Changes

The PR adds an embedded-runner helper, runner guard, and tests to skip blank visible non-runtime user prompts before provider submission.

## Best Possible Solution

Keep this PR open for normal maintainer review and targeted validation. If the additive guard is accepted, land it as the focused pre-provider fix for #74137; after merge, the linked issue can close as implemented. Keep broader empty-prompt/reset reports tracked separately unless this patch is deliberately expanded.
