## Summary

Keep open for maintainer review. Current main does not implement the new profile export/import workflow, so this PR is not obsolete, but the latest PR head still has a concrete privacy/security blocker in the profile config sanitizer.

## What This Changes

The PR adds `openclaw profile export/import`, profile archive creation/import infrastructure, privacy-safe config/workspace/plugin-install export logic, docs, tests, and related test fixture updates.

## Best Possible Solution

If core profile portability is accepted, land it as a documented privacy-safe archive format that reuses the shared sensitive-path semantics, keeps plugin install metadata as a sanitized portable projection, preserves non-destructive import behavior, rejects traversal and unsafe workspace payloads, and clearly distinguishes profile portability from full backup/restore and external migration.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reuse shared sensitive-key matching for profile export:** `src/infra/profile-portability.ts:360-372`
  - body: The exporter’s `shouldOmitPortableConfigKey` drops only a branch-local hard-coded list, so config fields such as `plugins.entries.<id>.config.privateKey`, `encryptKey`, and `serviceAccountRef` survive even though current main treats those names as sensitive. Because profile archives include arbitrary plugin/skill/MCP config and are advertised as privacy-safe, this can leak credentials; reuse `isSensitiveConfigPath` or mirror its patterns with regression coverage.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: Profile export can still include credential-like keys not covered by the branch-local sanitizer.

Concerns:

- **[high] Profile archive can include private keys:** `src/infra/profile-portability.ts:360`
  - body: The sanitizer omits tokens, passwords, secrets, and API keys, but not `privateKey`, `encryptKey`, or `serviceAccountRef` patterns that OpenClaw already classifies as sensitive. Since the archive includes arbitrary portable plugin/skill/MCP config, those values can be written into a privacy-safe profile export.
  - confidence: 0.9
