## Summary

Keep open. Current main does not implement privacy-safe `openclaw profile export/import`; the existing `backup` and `migrate` surfaces are not equivalent, and this open PR still contains useful but security-sensitive work that needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If profile portability belongs in core, the best path is to fix the sanitizer by tracking dictionary depth explicitly, add collision regression tests, validate or rebuild imported plugin install records through existing plugin index helpers, separate actual writes from dry-run projections, and split unrelated test-harness trust changes out of the feature patch.
