## Summary

Keep open. Current main still lacks the proposed `openclaw profile export/import` surface: backup only creates/verifies broad local-state archives, migrate imports from other agent systems, and the existing profile code only handles the global `--profile` state-directory option. The latest PR diff is real new functionality and appears to address the earlier sanitizer collision review note, but it is an XL privacy-sensitive import/export feature that needs maintainer product/security review rather than cleanup closure.

## What This Changes

The PR adds `openclaw profile export/import`, a `.openclaw-profile.tar.gz` archive implementation for sanitized config, workspace persona/memory markdown, plugin install records, docs, focused CLI/command/infra tests, and bundled-plugin test setup adjustments.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If core profile portability is accepted, land it as a documented privacy-safe archive format with sanitizer regression coverage, strict archive/path validation, non-destructive import semantics, plugin install metadata handled through the existing install ledger, and docs that clearly distinguish profile portability from full backup/restore and third-party migration.
