## Summary

Keep open. Current main does not implement the proposed privacy-safe `openclaw profile export/import` workflow, and the PR still contains useful but security-sensitive archive import/export work that should receive maintainer product and security review rather than cleanup closure. The existing `backup` and `migrate` commands are adjacent but not equivalent: backup is a broader state archive including credentials/sessions, while migrate imports from other agent systems through plugin-owned providers.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If profile portability belongs in core, the best path is to fix sanitizer dictionary-depth handling, add collision regression tests, validate or rebuild imported plugin install records through existing plugin index helpers, separate unrelated test-harness trust changes, and align the docs/CLI index with the accepted command shape.
