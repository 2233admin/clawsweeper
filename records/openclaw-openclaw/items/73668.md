## Risks / Open Questions

- Merging as-is can leak secret-like config fields when dictionary entry ids collide with sanitizer container key names such as `entries`, `servers`, `models`, `cliBackends`, or `providerOptions`.
- Importing raw plugin install records from an archive can affect future plugin reinstall/update behavior and should validate schema and provenance before becoming trusted local state.
- Dry-run import currently reports would-write entries in `filesWritten`, which can mislead JSON consumers that interpret that field as actual writes.
- Bundled-plugin test trust plumbing is unrelated to profile portability and changes a security-sensitive trust boundary in the test environment.
