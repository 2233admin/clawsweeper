## Summary

Keep open. Current main does not implement the PR's profile archive export/import surface: there is no top-level `profile` command, no profile portability infra, and the existing backup/migrate commands solve different problems. The PR head is an open XL implementation of new privacy-sensitive archive import/export behavior, so cleanup closure would be unsafe; it needs maintainer product/security review and normal PR validation.

## What This Changes

The PR adds `openclaw profile export/import`, profile archive creation/import infrastructure, CLI/docs/tests, and several bundled-plugin test setup adjustments.

## Best Possible Solution

If core profile portability is accepted, land it as a documented, privacy-safe archive format with regression coverage for secret stripping, traversal rejection, non-destructive import semantics, plugin install metadata handling, and docs that clearly distinguish profile portability from full backup/restore and external migration.
