## Summary

Keep open. Current main does not implement the proposed `openclaw profile export/import` surface, and the existing backup and migrate commands are not equivalent. The PR is still a live XL implementation for a privacy-sensitive archive import/export path, so maintainer product and security review is the right next step rather than cleanup closure.

## What This Changes

This PR adds `openclaw profile export/import`, a `.openclaw-profile.tar.gz` archive implementation for sanitized config, workspace profile files, memory markdown, and plugin install records, plus CLI docs and focused tests.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If core profile portability is accepted, the end state should preserve the privacy-safe archive boundary, validate or rebuild imported plugin install records through existing plugin-index paths, keep the sanitizer collision regression coverage, and align the docs with the existing backup and migrate boundaries.
