## Summary

Keep open. Current main still does not implement privacy-safe `openclaw profile export/import`, and the existing `backup` and `migrate` commands cover different recovery/import use cases. The latest PR head appears to address Greptile's sanitizer collision concern with explicit dictionary-depth tracking and regression coverage, but the new profile import/export surface remains product- and security-sensitive, especially around importing machine-managed plugin install metadata.

## What This Changes

This PR adds a new `openclaw profile` CLI with export/import commands, a `.openclaw-profile.tar.gz` archive implementation, docs, CLI/command/infra tests, and related test-fixture setup changes.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If core profile portability is accepted, the end state should keep the narrower privacy-safe archive contract, validate or rebuild imported plugin install records through existing plugin-index paths, preserve the sanitizer collision regression tests, and align the docs and command surface with the existing backup/migrate boundaries.
