## Summary

Keep open. Current main does not implement profile archive export/import, and this PR is still not merge-ready because the latest head can leak key material through the branch-local profile config sanitizer. Prior review discussion already identified privacy issues; the plugin install-record copy appears fixed on the latest head, but the shared sensitive-key mismatch remains a blocking security concern.

## What This Changes

The PR adds a new `openclaw profile export/import` CLI, profile archive infrastructure, docs, tests, sanitized config/workspace/plugin-install payload handling, and adjacent gateway/plugin test fixture updates.

## Best Possible Solution

If core profile portability is accepted, land it as a documented privacy-safe archive format that reuses the shared sensitive-path semantics, exports only sanitized portable plugin install records, preserves non-destructive import behavior, rejects unsafe archive/workspace paths, and keeps the boundary clear from full backup/restore and external-system migration.

## Review Findings

Overall correctness: patch is incorrect

  - body: The branch-local sanitizer only omits token/password/api-key style names. ## Security Review

Status: needs_attention

Summary: Profile export still has a concrete secret-disclosure risk in the config sanitizer.

Concerns:

- **[high] Profile archive can include private keys:** `src/infra/profile-portability.ts:360`

Because the archive includes arbitrary portable plugin/skill/MCP config, those values can be exported into a privacy-safe profile archive.
