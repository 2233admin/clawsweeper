## Summary

Keep open. Current main still does not provide the requested default-off durable plan/session audit stream: existing plan events are in-process update-only records, `update_plan` has no stable plan identity or terminal close semantics, and lifecycle/task tracking is keyed to run/session state rather than a plan correlation contract. The report is valid and well evidenced, but implementation needs maintainer API/product/security design before it is a safe cleanup fix.

## What This Changes

The issue requests a disabled-by-default configuration option that emits stable plan and session lifecycle records to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define an additive observability contract with stable plan/run identifiers, explicit plan open/update records, subagent spawn/yield/end correlation, and exactly one terminal close/abort record. Keep it disabled by default, document schema and redaction guarantees, wire config/help/docs, and add focused tests for disabled behavior plus enabled lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: source and docs searches show the requested config keys, durable JSONL writer, stable plan identifiers, and terminal plan close correlation are absent.

## Solution Assessment

Unclear. A default-off durable JSONL stream is plausible, but maintainers should first choose the schema, redaction/access model, sink ownership, and whether this belongs in core, the task ledger, trajectory, or plugin/runtime APIs.
