## Summary

Keep open. Current main still lacks the requested default-off durable plan/session audit contract: there is no `events.emit.plan` config, no `/var/log/openclaw-events.jsonl` writer, no stable `plan_id`/`plan_ref`, and no terminal `closes_plan` correlation. Existing in-process agent events, plugin event subscriptions, and task-ledger lifecycle tracking are useful adjacent surfaces but do not satisfy the requested canonical JSONL stream.

## What This Changes

Request to add a default-off `events.emit.plan` setting that emits plan/session lifecycle records to `/var/log/openclaw-events.jsonl` with stable plan identifiers, spawn/yield correlation, and terminal close or abort records.

## Best Possible Solution

Define an additive observability contract for stable plan/run identifiers, plan update/open records, subagent spawn/yield/end correlation, and terminal close/abort records. Keep it disabled by default, document schema stability and redaction guarantees, decide sink ownership and file permissions, add config help, and cover disabled behavior plus lifecycle completeness with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and JSONL sink are absent, while `update_plan` and lifecycle events remain run-scoped or in-process without stable plan close correlation.

## Solution Assessment

Unclear. The direct `/var/log/openclaw-events.jsonl` proposal may be viable, but maintainers should decide it against plugin event subscriptions, SDK/API stability, sink ownership, schema versioning, retention, and redaction before implementation.
