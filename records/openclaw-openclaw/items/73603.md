## Summary

Keep open. Current main has related in-process `plan`, lifecycle, tool, task, trajectory, SDK, and plugin event surfaces, but it still does not provide the requested default-off durable audit contract: no `events.emit.plan` config, no `/var/log/openclaw-events.jsonl` writer, no stable `plan_id`/`plan_ref`, and no terminal `closes_plan` semantics. This is not a clean ClawHub close because the missing piece is a core schema/API and redaction decision before a plugin can reliably implement the requested audit stream.

## What This Changes

The issue requests a default-off config flag that emits plan/session lifecycle audit events to `/var/log/openclaw-events.jsonl` with stable plan IDs, spawn/yield correlation, and terminal close/abort records.

## Best Possible Solution

Define an additive observability contract that exposes stable run/plan IDs, plan update/open records, terminal close/abort records, and subagent spawn/yield/end correlation through either a default-off core JSONL sink or a documented plugin API that can safely produce that sink. The design should include config/schema docs, redaction rules, disabled-by-default behavior, and focused tests for lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and JSONL sink are absent, while related `plan`, `lifecycle`, `task`, plugin-event, SDK, and trajectory surfaces remain separate and do not provide stable plan close correlation.

## Solution Assessment

Unclear. The requested auditability problem is valid, but the exact `/var/log/openclaw-events.jsonl` core-writer shape should be decided against the existing plugin event, SDK, diagnostics, and trajectory surfaces before a narrow implementation PR starts.
