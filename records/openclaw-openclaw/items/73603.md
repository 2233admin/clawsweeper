## Summary

Keep open. Current main still lacks the requested default-off durable plan/session audit stream: `update_plan` has no stable plan identity, plan events are in-process update-only events, lifecycle/task tracking is run-scoped, and the public docs/SDK surfaces do not define the requested JSONL plan open/spawn/yield/close contract.

## What This Changes

The issue requests a default-off configuration option to emit stable plan and session lifecycle records to the canonical OpenClaw JSONL event stream.

## Best Possible Solution

Define an additive, disabled-by-default observability contract with stable plan/run identifiers, plan update/open records, subagent spawn/yield/end correlation, and exactly one terminal close/abort record. Wire config schema/help/docs and tests for disabled behavior, enabled lifecycle completeness, redaction, and retention/permission expectations.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: targeted search and source reads show the requested config keys, canonical JSONL sink, stable plan identifiers, and terminal plan close correlation are absent.

## Solution Assessment

Unclear. The proposed default-off JSONL stream is plausible, but maintainers should first decide the stable schema, redaction/access model, sink ownership, and whether the contract belongs in core, task ledger, trajectory, or plugin/runtime APIs.
