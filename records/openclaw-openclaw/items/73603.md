## Summary

Keep open. Current main has adjacent in-process plan, lifecycle, diagnostic, task, and plugin event surfaces, but it still lacks the requested default-off plan/session audit contract, canonical JSONL writer, stable plan identifiers, spawn/yield plan correlation, and terminal close/abort semantics.

## What This Changes

Request a default-off config flag that emits plan and session lifecycle events to `/var/log/openclaw-events.jsonl` for external task-completion audit tooling.

## Best Possible Solution

Define an additive experimental observability contract with stable run/plan identifiers, plan update/open records, exactly-one terminal close/abort/timeout record, and subagent spawn/yield/end correlation, then expose it through a redacted default-off JSONL exporter or a documented plugin API capable of producing that sink safely. The change should include config schema/help, docs, and focused tests for disabled-by-default behavior and lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and JSONL sink are absent, while `update_plan`, lifecycle, tool, plugin, and diagnostic events remain separate in-process or summary surfaces without plan close correlation.

## Solution Assessment

Unclear. The requested stable audit contract is the right problem to solve, but the exact `/var/log/openclaw-events.jsonl` core-writer shape should be decided against the existing diagnostics and plugin event systems before implementation.
