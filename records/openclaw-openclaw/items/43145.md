## Summary

Keep this PR open. Current main still does not implement the PR’s central `agent.timeline` retained run-trace timeline feature: there is no advertised Gateway method, no handler, no protocol schema, and no in-memory `AgentRunTrace` store. Current trajectory bundles are related observability work, but they are per-session/export support bundles rather than a short-lived `runId`-keyed Gateway timeline. The PR remains a useful implementation candidate, but it should not merge as-is because the branch is broad and has unresolved security-sensitive review findings.

## Best Possible Solution

Keep this PR open for explicit maintainer handling, but require it to be split and rebased. Align the trace work with #68204, then land a narrow implementation with a canonical schema, safe Gateway/API exposure, ownership authorization, redaction, retention/span caps, generated client updates, docs, and focused tests. Separate or drop the unrelated cron, sandbox, planner, generated-client, and drive-by test changes.
