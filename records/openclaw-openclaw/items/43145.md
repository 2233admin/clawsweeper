## Summary

Keep this PR open. Current main still does not implement the central retained run-trace timeline store or `agent.timeline` Gateway RPC, and the existing trajectory bundle support is related but not equivalent. The PR remains an implementation candidate, but it should not merge as-is because it mixes the trace feature with unrelated cron, sandbox, planner, docs, generated-client, and test changes plus unresolved security-sensitive review findings.

## Best Possible Solution

Keep this PR open for explicit maintainer handling, but require it to be split and rebased. Drop or separate already-shipped/unrelated work, then rework the retained trace feature against #68204 with a canonical cross-flow schema, safe Gateway/API exposure, ownership checks, redaction, retention/span caps, generated client updates, docs, and focused tests.
