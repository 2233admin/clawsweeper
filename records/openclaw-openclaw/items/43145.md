## Summary

Keep this PR open. Current main still does not expose a retained `agent.timeline` Gateway RPC or the proposed in-memory run-trace store, while the existing trajectory bundle support is related but not equivalent. The PR remains a useful implementation candidate, but it should not merge as-is because it mixes unrelated agent, cron, sandbox, docs, generated-client, and test work with unresolved security-sensitive review findings.

## Best Possible Solution

Keep this PR open for explicit maintainer handling, but require it to be split and rebased. Align the trace work with #68204, then land a narrow implementation with a canonical schema, safe Gateway/API exposure, ownership authorization, redaction, retention/span caps, generated client updates, docs, and focused tests. Separate or drop unrelated cron, sandbox, planner, and drive-by test changes.
