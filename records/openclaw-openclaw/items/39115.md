## Summary

Keep this PR open. Current main still does not implement the requested `agents.defaults.reasoningDefault` surface: the global defaults schema/type/generated metadata omit it, reply and status resolution only honor per-agent `agents.list[].reasoningDefault`, and the docs still describe reasoning visibility as per-agent-only. The PR remains an active implementation candidate, but it needs maintainer policy/security review before merge because reasoning visibility affects trusted-vs-untrusted command behavior.

## Best Possible Solution

Keep this PR open for maintainer review. A mergeable path should settle the reasoning-default authorization/product policy, then implement `agents.defaults.reasoningDefault` consistently across schema, exported types, generated metadata, docs, reply directive resolution, current-level status, direct `/status`, status text, and focused precedence/security tests. If maintainers reject the global default, close with that explicit product decision rather than as stale or already implemented.
