## Summary

Keep this PR open. Current main still does not implement the requested global `agents.defaults.reasoningDefault` surface: the strict defaults schema and exported defaults type omit the key, reply/status resolution only use session or per-agent reasoning defaults, and docs still describe `reasoningDefault` as per-agent only. The PR is also active, with recent contributor follow-up and prior maintainer signal that the intended behavior is still being evaluated.

## Best Possible Solution

Keep this PR open for maintainer review. A mergeable solution should first settle the product/security policy, then implement `agents.defaults.reasoningDefault` consistently across strict schema, exported types, generated schema/help metadata, docs, reply resolution, current-level status, direct status, and status text. It should preserve explicit `off` over model-derived reasoning defaults and include tests for directive, session, per-agent, global, model-default precedence, plus authorized and unauthorized sender behavior. If maintainers reject the feature, close it with that explicit product decision rather than as stale or implemented.
