## Summary

Keep this PR open. Current main at d22a851253b0225e72847dc6b1a7b74d8035e214 still does not implement the PR's intended global `agents.defaults.reasoningDefault` config surface. Main only supports per-agent `agents.list[].reasoningDefault`, and the reply/status resolution paths still omit the global default. The PR is also active, has recent contributor follow-up, and prior maintainer discussion left the behavior open for product evaluation.

## Best Possible Solution

Keep this PR open for maintainer review. A mergeable path should settle the product and authorization policy, then implement `agents.defaults.reasoningDefault` consistently across `AgentDefaultsSchema`, `AgentDefaultsConfig`, generated config metadata/help/labels, public docs, reply directive resolution, current-level status, direct `/status`, and status text. It should preserve explicit `off` over model-derived defaults and include focused tests for directive, session, per-agent, global, model-default, authorized sender, and unauthorized sender precedence. If maintainers reject the feature, close it with that explicit product decision rather than as stale or implemented.
