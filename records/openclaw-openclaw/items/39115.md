## Summary

Keep this PR open. Current main still does not implement the requested global `agents.defaults.reasoningDefault` config surface: the strict defaults schema and exported defaults type omit the key, generated config metadata only exposes per-agent `agents.list[].reasoningDefault`, and reply/status resolution paths still ignore a global reasoning default. The PR is an active implementation candidate with recent contributor follow-up, and the earlier maintainer comment says the intended behavior is still being evaluated.

## Best Possible Solution

Keep this PR open for maintainer review. A mergeable path should settle the product/security policy, then implement `agents.defaults.reasoningDefault` consistently across schema, exported types, generated metadata, docs, reply resolution, current-level status, direct status, and status text. It should preserve explicit `off` over model-derived reasoning defaults and include focused tests for directive, session, per-agent, global, model-default precedence, plus authorized and unauthorized sender behavior. If maintainers reject the feature, close it with that explicit product decision rather than as stale or implemented.
