## Summary

Keep #70680 open. Current `main` still has the reported behavior: `isPrimarySessionTranscriptFileName()` accepts non-archived `*.jsonl` files without excluding `*.trajectory.jsonl`, and the doctor orphan scan uses that helper while only building its referenced set from primary session transcript paths. Trajectory sidecars are a documented/current artifact shape written beside session files, so they can still be misclassified. The linked PR #70812 appears to be the focused fix, but it is open/unmerged in the provided GitHub context and its referenced commit is not present in this checkout.

## Best Possible Solution

Keep #70680 open until a focused core fix lands. The best path is PR #70812 or equivalent: narrow `isPrimarySessionTranscriptFileName()` so known sidecar families such as `.trajectory.jsonl` are not primary transcript candidates, keep real orphan `*.jsonl` cleanup working, and add helper plus doctor state-integrity regression tests proving a real orphan transcript is still offered for archival while a neighboring trajectory sidecar is ignored.
