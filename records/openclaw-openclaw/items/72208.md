## Summary

Keep open. Current main and v2026.4.27 contain the startup-side manifest reuse mitigation, but the pricing refresh lifecycle still is not shutdown-aware: in-flight OpenRouter/LiteLLM fetches use independent timeout signals, the stop hook only flips a local flag and clears the current timer, and completed refreshes can still schedule more work after stop.

## What This Changes

Prevent `openclaw gateway` from hanging after OpenRouter/LiteLLM pricing fetch timeouts on Windows npm-global installs, keeping the dashboard reachable and Ctrl+C responsive.

## Best Possible Solution

Keep the shipped startup optimization, then make the model-pricing refresh lifecycle cancellation-aware: abort in-flight pricing fetches on stop, suppress follow-up scheduling after stop, unref background refresh timers, and add focused lifecycle tests around stop behavior.

## Reproduction Assessment

Yes. The issue gives concrete released-package steps, and current source supports a focused unit reproduction by starting a pricing refresh with hanging fetches, calling the returned stop function, and asserting the fetches are aborted and no timer remains scheduled.

## Solution Assessment

No. The current implementation is a useful partial mitigation, but the narrow complete fix belongs in `src/gateway/model-pricing-cache.ts` lifecycle handling rather than broader Gateway startup or provider behavior.
