## Summary

Keep open. Current main and v2026.4.27 contain a partial startup mitigation for the pricing-refresh stall, but the reported shutdown/Ctrl+C hang remains source-backed: the pricing refresh stop hook does not cancel an in-flight fetch, and the refresh path can still schedule a follow-up timer after stop.

## What This Changes

The issue reports a Windows npm-global regression where `openclaw gateway` in 2026.4.24 hangs during OpenRouter/LiteLLM pricing fetch timeouts, leaves the dashboard unreachable, and does not respond to Ctrl+C.

## Best Possible Solution

Keep the shipped startup mitigation, then make the model-pricing refresh lifecycle shutdown-aware: abort active pricing fetches on stop, prevent stopped refreshes from scheduling later timers, unref background refresh timers, and lock the behavior with focused Gateway lifecycle tests.

## Reproduction Assessment

Yes. The report gives a concrete released-package reproduction path, and current source supports a focused unit reproduction: start a pricing refresh with a hanging fetch, call the stop callback while it is in flight, then assert the fetch is aborted and no follow-up timer remains.

## Solution Assessment

No. The current implementation is a useful partial startup fix, but the best complete solution is narrower lifecycle-owned cancellation and timer cleanup inside the model-pricing refresh code.
