## Summary

Keep open. Current main and v2026.4.27 mitigate the startup side of the pricing-refresh hang, but the reported Ctrl+C/shutdown behavior still has a source-backed gap: in-flight pricing fetches are not aborted, stopped refreshes can still schedule another timer, and that timer is not unref'ed.

## What This Changes

The issue reports a Windows npm-global 2026.4.24 Gateway hang when OpenRouter and LiteLLM pricing catalog requests time out, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Keep the shipped startup mitigation, then make the model-pricing refresh lifecycle shutdown-aware: abort in-flight pricing fetches on stop, prevent any stopped refresh from scheduling another follow-up timer, unref background refresh timers, and lock this with focused Gateway pricing/runtime tests.

## Reproduction Assessment

Yes. The issue gives a concrete released-package reproduction path, and current source supports a focused unit reproduction with a hanging pricing fetch: start the refresh, call the stop callback while it is in flight, and assert the fetch is aborted and no follow-up timer survives.

## Solution Assessment

No. The current startup fix is a good partial mitigation, but it is not the best complete solution because it does not cover the shutdown behavior in the report; the narrower maintainable fix is lifecycle-owned cancellation and timer cleanup in the model-pricing refresh code.
