## Summary

Keep open. This is a fresh, detailed Docker/OpenAI latency report, and current main does not contain enough evidence to close it as fixed. The reported stuck-session diagnostic, WebChat async send path, ACP dispatch/runTurn path, and OpenAI WebSocket/SSE transport path are all still current; recent post-release commits improve startup/plugin runtime mirrors, WebChat state, and a separate pending-tool drain class, but none proves the reported 30-90s steady-state WebChat turn completion latency is resolved.

## Best Possible Solution

Keep this issue as the active tracking item. Maintainers should reproduce or collect a current-main trace on the same Docker/OpenAI setup, capturing model-call first-byte/total duration, acp-dispatch latency, session queueDepth, and WebChat finalization timing. Ask the reporter to compare OpenAI transport settings (transport: "sse" vs "websocket"/"auto"), openaiWsWarmup: false, and plugins.slots.memory: "none". If an unreleased main commit already fixes the reporter's trace, close after confirmation and release evidence; otherwise patch the specific ACP/OpenAI transport/finalization boundary that owns the delay.
