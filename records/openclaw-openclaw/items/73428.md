Keep open. The issue is fresh, reproducible in the discussion, and maintainers explicitly left it open after landing only partial runtime-deps and diagnostic slices. ## What This Changes

The issue reports severe Docker/WebChat chat-turn latency on v2026.4.26 where trivial OpenAI-backed replies take 30-90 seconds through OpenClaw while direct OpenAI calls from the same container complete in under two seconds.

## Best Possible Solution

Keep this as the active Docker/WebChat latency tracker, ask affected installs to retest current main or the next Docker package with `OPENCLAW_LOG_LEVEL=trace`, then land a narrow fix for the specific embedded-run startup/prep stage that still dominates before model I/O.
