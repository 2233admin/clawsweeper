## Summary

Keep open. Current main still restarts unhealthy channels by calling `stopChannel` and `startChannel` directly, and `stopChannel` aborts the channel lifecycle before waiting for the task to end. The PR's central drain-before-abort behavior, 60s cooldown floor, fresh timing callback, and post-drain rechecks are not implemented on main. The provided PR diff is focused on the gateway health monitor and its tests, so it remains a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate. Review the latest head against current main, verify the drain-before-abort behavior and all post-drain rechecks in `src/gateway/channel-health-monitor.ts`, keep the focused regression tests, and coordinate with adjacent health-monitor restart timing work before landing or requesting changes.
