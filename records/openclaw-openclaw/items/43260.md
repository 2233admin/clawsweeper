## Summary

Current main still does not implement SKILL.md or plugin-skill per-skill model routing. Skill metadata, parsing, prompt formatting, snapshots, and slash skill invocation all ignore model/preferredModel/modelProfile-style fields, while #58142 remains the open implementation candidate and the thread has concrete InvestorClaw validation context.

## Best Possible Solution

Keep this issue open as the canonical feature request. The best maintainer path is to review, revise, land, or supersede #58142 with a documented SKILL.md/plugin skill metadata contract, explicit precedence and fallback semantics, allowlist/alias handling, and focused parser, prompt, slash-invocation, snapshot, and runtime-routing tests.
