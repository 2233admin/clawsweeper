Current main still does not implement SKILL.md or plugin-skill model routing. The issue has fresh, concrete InvestorClaw validation context and #58142 remains an open implementation candidate rather than a shipped fix, so this should stay open.

The best path is to review, revise, land, or supersede #58142 with a documented SKILL.md/plugin-skill metadata contract, explicit precedence and fallback semantics, allowlist/alias handling, and focused parser, prompt, slash-invocation, snapshot, and runtime-routing tests.
