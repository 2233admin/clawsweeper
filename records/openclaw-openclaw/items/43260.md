## Summary

Current main still does not implement per-skill model routing from SKILL.md or plugin skill metadata. The issue remains a live feature/API design request, with an open implementation candidate in #58142 and concrete validation context from InvestorClaw rather than shipped behavior.

## Best Possible Solution

Keep this issue open as the canonical feature request. The best maintainer path is to review, revise, land, or supersede #58142 with a documented SKILL.md/plugin skill metadata contract, alias/allowlist resolution, explicit precedence against CLI/session/cron/agent model choices, fallback behavior, and focused parser, prompt, slash-invocation, and runtime-routing tests.
