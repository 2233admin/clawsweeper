## What This Changes

The issue reports that OpenClaw 2026.4.25 hangs without streamed tokens or useful errors when running `openai-codex/gpt-5.5` through the legacy OAuth Responses route, while `gpt-5.4` and direct Codex/API probes work.

## Best Possible Solution

Keep the issue open until maintainers run a current-main live probe with a working OpenAI Codex OAuth profile. If it still hangs or only reaches the idle timeout, the fix should stay in the OpenAI Codex provider and Responses transport boundary, making `openai-codex/gpt-5.5` either complete normally or fail quickly with a specific provider error.

## Solution Assessment
