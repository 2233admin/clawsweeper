## Summary

Keep open. Current main still mishandles the linked Control UI `/model status` and `/model list` path, so the PR is not obsolete, but the branch needs updating before merge because current main now requests the configured model catalog while the new assertions still expect the old empty `models.list` payload.

## What This Changes

The PR changes the Control UI slash-command executor to treat `/model status` and `/model list` as informational local commands and adds regression tests that assert `sessions.patch` is not called for those aliases.

## Best Possible Solution

Land a focused Control UI fix on current main that treats `/model status` and `/model list` as local informational modes without calling `sessions.patch`, preserves configured-catalog loading and provider-qualified model resolution, and covers both aliases with current RPC payload expectations.
