## Summary

Keep this PR open. Current `main` still handles WebUI `/model status` and `/model list` in the local Control UI slash executor as model-change requests, so the PR is not obsolete. The linked same-author bug report #46894 is also still open, so the issue/PR pair should remain together for maintainer review.

## Best Possible Solution

Keep the PR and paired bug report #46894 open. The best path is a focused Control UI fix that prevents `/model status` and `/model list` from calling `sessions.patch`, adds regression tests for both modes, and either mirrors the shared directive semantics or makes a deliberate product decision for the WebUI-local output.
