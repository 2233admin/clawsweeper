## Summary

Keep this PR open. Current main still routes Control UI `/model` through the local WebUI slash executor and still sends every non-empty `/model <arg>` to `sessions.patch`, so `/model status` and `/model list` are not implemented on main. The same-author paired bug report #46894 is also still open, so the issue/PR pair should remain together for maintainer review.

## Best Possible Solution

Keep this PR and paired bug report #46894 open. The best path is a focused Control UI fix that prevents `/model status` and `/model list` from calling `sessions.patch`, adds WebUI executor regression tests for both, and deliberately aligns or documents any difference from the richer shared `/model` directive output.
