Current Chat copy now spans split helper modules, the supported locale set is larger than the six edited locale files, the composer placeholder now includes assistant-specific context, and the proposed fallback params are not supported by the i18n helper.

## What This Changes

The PR localizes selected Chat view strings in `ui/src/ui/views/chat.ts` with `t("chat.*")` calls and adds matching `chat.*` keys to six locale files.

## Best Possible Solution

Rework or replace the branch so current Chat view and split helper copy all use the existing i18n path, English source keys are added once, all generated locale bundles and `.i18n` metadata are refreshed through the Control UI pipeline, assistant-name interpolation is preserved, and focused UI/i18n tests cover the localized behavior.
