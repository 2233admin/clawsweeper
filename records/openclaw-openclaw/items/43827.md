## Summary

Keep this PR open. Current main still has the hard-coded Chat UI strings the PR is trying to localize, but the PR is not merge-ready against current main because Chat has since been split across additional modules, the locale pipeline now covers 14 locales through generated bundles, and the proposed `{ fallback: "..." }` params still do not match the current `t()` contract. Security review found no concrete supply-chain, workflow, dependency, publishing, or secrets-handling concern in the provided file list.

## Best Possible Solution

Keep this PR open for revision. The best path is to rebase onto current main, localize the current Chat surfaces across `ui/src/ui/views/chat.ts` and `ui/src/ui/chat/*`, add English source keys, regenerate all supported locale bundles through the Control UI i18n pipeline, remove the no-op fallback params or add a real typed fallback API with tests, and update Chat tests to assert localized behavior intentionally.
