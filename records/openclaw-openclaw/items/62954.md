## Summary

Keep open. Current main still lets heartbeat runs persist the just-used model/modelProvider into the session entry, and the display path still resolves those persisted fields for session rows/history. The direct fix PR #62987 is closed unmerged, so this remains a valid, narrow bug rather than cleanup work.

## What This Changes

Prevent heartbeat turns from overwriting a session's persisted model/modelProvider display fields while leaving actual reply model selection unchanged.

## Best Possible Solution

Add a heartbeat-aware preservation mode at the session usage persistence boundary so heartbeat turns keep the existing persisted session model identity while still recording normal usage, context, cost, system-prompt, and non-heartbeat updates. Cover both persistence branches with a regression test that exercises a heartbeat run using a different heartbeat model.
