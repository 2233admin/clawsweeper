## Summary

Keep this PR open. It targets a real compaction bug and the deterministic exact-path suffix is the right direction, but the patch is not merge-ready because it extracts path literals by serializing whole messages, which can re-expose `toolResult.details` that current compaction and replay paths intentionally strip before model-facing context.

## What This Changes

The PR adds exact path literal collection and formatting to the compaction safeguard, appends an `<exact-path-literals>` suffix on provider and LLM compaction paths, and adds regression tests for path preservation.

## Best Possible Solution

Keep the deterministic exact-path preservation design, but extract from a sanitized transcript shape or from explicitly allowed fields such as user/assistant text, tool-call inputs, previous summaries, and file-operation metadata. Add regression coverage proving path-like strings in `toolResult.details` do not appear in the exact-path suffix.

## Reproduction Assessment

Yes for the PR regression: a `toolResult` message whose `details` contains a path-like string would be serialized by the new helper and copied into `<exact-path-literals>`. The original provider-specific MiniMax corruption remains credible from logs but lacks a deterministic local replay fixture.

## Solution Assessment

No, not as proposed. The suffix approach is a maintainable solution, but whole-message serialization crosses an existing model-context data boundary; the safer fix is to sanitize first or traverse only approved message fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Strip tool result details before path extraction:** `src/agents/pi-hooks/compaction-safeguard.ts:522`
  - body: `collectExactPathLiterals` now extracts paths from `JSON.stringify(message)`, which includes `toolResult.details`. Current compaction strips those details before model-facing summarization, so a hidden diagnostic value like a raw path would be persisted into `<exact-path-literals>` and shown to future turns. Extract from a `stripToolResultDetails`-sanitized shape or selected content/tool-call input fields instead.
  - confidence: 0.87
