## Summary

Keep open. Current main still has the override-first prompt assembly in the embedded attempt, compaction retry, and CLI runner paths, so a configured `systemPromptOverride` still bypasses the builder branch that appends the subagent task carried in `extraSystemPrompt`; the directly related fix PR #73637 is closed unmerged, while #73656 tracks a separate visible-bootstrap fallback request.

## What This Changes

Preserve task-specific subagent instructions from `params.extraSystemPrompt` when an agent-level `systemPromptOverride` is configured.

## Best Possible Solution

Add a shared, tested prompt composition helper near `resolveSystemPromptOverride` that preserves existing no-override behavior, returns the trimmed override alone when no runtime extra prompt exists, and appends trimmed `extraSystemPrompt` with a blank-line separator when both are present. Apply it consistently to the embedded attempt, compaction retry, and CLI runner prompt assembly paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure an agent with `systemPromptOverride`, spawn a native subagent, and the child task flows through `extraSystemPrompt` while the override branch returns before the builders append that runtime prompt block.

## Solution Assessment

Yes. The issue's proposed composition helper is the narrowest maintainable fix because it preserves agent-level overrides and only changes the override branch to retain runtime prompt additions; changing the visible bootstrap message is a separate fallback design tracked by #73656.
