## Repair Work Prompt

Fix openclaw/openclaw#73624. Current main drops subagent task instructions when an agent-level `systemPromptOverride` is set because `resolveSystemPromptOverride(...) ?? buildEmbeddedSystemPrompt/buildSystemPrompt(...)` returns the override before `extraSystemPrompt` is appended. Add a shared helper next to `resolveSystemPromptOverride` that returns `undefined` when no override exists, returns the trimmed override when no nonblank `extraSystemPrompt` exists, and returns `<override>\n\n<trimmed extraSystemPrompt>` when both exist. Use it in `src/agents/pi-embedded-runner/run/attempt.ts`, `src/agents/pi-embedded-runner/compact.ts`, and `src/agents/cli-runner/prepare.ts`. Add focused regression coverage in `src/agents/system-prompt-override.test.ts` and, where practical, existing CLI/embedded prompt tests proving override+extra preserves both pieces. Keep #73656 out of scope: do not change the visible subagent bootstrap message in this fix. Update `docs/gateway/config-agents.md` if needed to clarify that the override replaces the base assembled prompt while runtime task/context additions may still be appended, and add a user-facing changelog fix entry with reporter/PR credit. Validate with the targeted formatter and tests, then run the changed gate in Testbox before handoff if runtime files changed.
