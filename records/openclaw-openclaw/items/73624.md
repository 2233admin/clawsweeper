## Summary

Keep open. Current main still has the override-first prompt assembly that drops `extraSystemPrompt` whenever `systemPromptOverride` resolves, while native subagent task text is still delivered through that dynamic prompt field. The directly related fix PR #73637 is closed unmerged, and the related visible-message issue #73656 is distinct rather than a replacement.

## What This Changes

The issue reports that subagent runs under an agent-level `systemPromptOverride` lose the delegated task text carried in `extraSystemPrompt`.

## Best Possible Solution

Add a shared prompt composition helper near `resolveSystemPromptOverride` that returns the trimmed override alone when no runtime extra prompt is present, returns undefined when no override exists, and appends trimmed `extraSystemPrompt` with a blank-line separator when both are present. Apply that helper consistently in embedded attempt, compaction retry, and CLI runner prompt assembly, with focused regression tests and a small changelog/docs clarification for the public override semantics.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure an agent with `systemPromptOverride`, spawn a native subagent, and the delegated task is passed as `extraSystemPrompt` but bypassed by the override-first `??` branches in the child runner prompt assembly.

## Solution Assessment

Yes. The narrowest maintainable fix is to compose runtime `extraSystemPrompt` with resolved overrides at the three prompt assembly call sites, preferably through one tested helper; duplicating the task into the visible bootstrap message is a separate fallback/debuggability change tracked by #73656.

## Repair Work Prompt

Fix the subagent `systemPromptOverride` / `extraSystemPrompt` composition bug from https://github.com/openclaw/openclaw/issues/73624. Current behavior: native subagent task instructions are built by `buildSubagentSystemPrompt` and passed as `extraSystemPrompt`, but embedded attempt, compaction retry, and CLI runner prompt assembly all use `resolveSystemPromptOverride(...) ?? build...({ extraSystemPrompt })`, so any configured override drops the delegated task. Implement a small shared helper next to `resolveSystemPromptOverride` that composes a resolved override with a trimmed runtime `extraSystemPrompt` using a blank-line separator, returns the override unchanged when the extra prompt is blank/missing, and returns undefined when no override exists so existing builder fallback behavior remains unchanged. Apply it to `src/agents/pi-embedded-runner/run/attempt.ts`, `src/agents/pi-embedded-runner/compact.ts`, and `src/agents/cli-runner/prepare.ts`. Add focused tests for no override fallback, override only, and override plus `## Your Role` / task extra prompt. Preserve the existing decision not to repeat the delegated task in `buildSubagentInitialUserMessage`; #73656 tracks that separate visible-message request. Update `docs/gateway/config-agents.md` only if needed to clarify that dynamic runtime prompt additions are appended to overrides, and add a single-line user-facing changelog fix entry with proper non-bot credit.
