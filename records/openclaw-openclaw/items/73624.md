## Summary

Keep open. Current `main` still returns the agent-level `systemPromptOverride` before calling the builders that append runtime `extraSystemPrompt`, so native subagents spawned under an override can still lose their delegated task block; the directly related fix PR is closed unmerged.

## What This Changes

The issue requests that agent-level `systemPromptOverride` continue to apply while still preserving task-specific subagent instructions passed through `extraSystemPrompt`.

## Best Possible Solution

Add a small shared prompt-composition helper next to `resolveSystemPromptOverride`, use it in embedded attempts, compaction retries, and CLI preparation, and add focused regression coverage so override prompts remain intact while dynamic runtime task/context blocks are appended consistently.

## Reproduction Assessment

Yes. A static current-main reproduction exists: configure `agents.defaults.systemPromptOverride` or `agents.list[].systemPromptOverride`, spawn a native subagent so the task is passed as `extraSystemPrompt`, and the override-first branches return before the builders append that task block.

## Solution Assessment

Yes. The narrowest maintainable fix is shared composition in the existing override helper area, rather than duplicating string joins at each runner call site or moving the task into the visible bootstrap message, which is tracked separately by #73656.
