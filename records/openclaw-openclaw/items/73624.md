## Summary

Keep open. Current main still drops the subagent task prompt whenever `systemPromptOverride` is present, and the directly linked fix PR is closed unmerged with no current open PR closing this issue.

## What This Changes

Preserve per-run `extraSystemPrompt` task instructions when a subagent runs under an agent-level `systemPromptOverride`.

## Best Possible Solution

Add a small shared prompt-override composition helper that appends trimmed runtime `extraSystemPrompt` to a non-empty override, then use it in embedded attempt, embedded compaction retry, and CLI runner prompt assembly. Keep no-override fallback behavior and override-only runs unchanged, and cover the helper plus at least one affected runner path with regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure an agent with `systemPromptOverride`, spawn a native subagent, and the child task reaches the gateway as `extraSystemPrompt` before the runner returns the override through the `??` branch and skips the builder that appends it.

## Solution Assessment

Yes. The narrow helper approach from the closed implementation candidate is the best maintainable fix because it preserves existing override semantics while restoring per-run task context across all current prompt assembly paths.

## Repair Work Prompt

Fix openclaw/openclaw#73624. Current main drops subagent task instructions when `systemPromptOverride` is configured because the runner returns `resolveSystemPromptOverride(...)` before the builders that append `extraSystemPrompt` run. Add a shared helper next to `resolveSystemPromptOverride` in `src/agents/system-prompt-override.ts`, for example `combineSystemPromptOverrideWithExtra({ override, extraSystemPrompt })`, with these semantics: no override returns undefined so callers keep their existing builder fallback; override plus blank/missing extra returns the trimmed override unchanged; override plus nonblank extra returns `<override>\n\n<trimmed extra>`. Use it in `src/agents/pi-embedded-runner/run/attempt.ts`, `src/agents/pi-embedded-runner/compact.ts`, and `src/agents/cli-runner/prepare.ts`. Add focused tests in `src/agents/system-prompt-override.test.ts` and runner coverage where practical. Do not change `buildSubagentInitialUserMessage` or solve #73656 in this PR unless a maintainer explicitly broadens scope. Check for adjacent conflicts with open PR #41807 before finalizing. Changelog: add a user-facing fix entry only if repo policy expects one for agent runtime bug fixes, with proper non-bot attribution.
