## Summary

Keep open. Current main still has the reported failure mode: subagent spawn sends the task-bearing subagent prompt through `extraSystemPrompt`, while the child run tells the model to find its task under `Your Role`; however, agent-level `systemPromptOverride` bypasses the prompt builder that would append `extraSystemPrompt`, so the delegated task can be dropped.

## Best Possible Solution

Patch source prompt composition so an agent-level or default `systemPromptOverride` is retained but runtime `extraSystemPrompt` is still appended for subagent/task contexts. Add a regression test that spawns a subagent for an agent with `systemPromptOverride` and asserts the compiled child system prompt includes both the override text and the generated `## Your Role` task block; audit the CLI and compaction prompt paths for the same behavior.

## Risks / Open Questions

- A fix should preserve the documented full-prompt-replacement behavior of `systemPromptOverride` for ordinary OpenClaw-generated sections while still preserving runtime-critical task/context injection for subagents.
- The same composition pattern appears in embedded run, embedded compaction, and CLI runner paths; a narrow patch in only one generated bundle or one source path could leave related runs affected.
