## Summary

Keep open. Current main still lets a resolved `systemPromptOverride` short-circuit the prompt builders that append `extraSystemPrompt`, while native subagent task delivery still depends on that extra prompt carrying the `## Your Role` task block; the directly related fix PR was closed unmerged.

## What This Changes

Subagent runs with an agent-level `systemPromptOverride` should preserve that override while still appending task-specific `extraSystemPrompt` instructions.

## Best Possible Solution

Add a small shared composition helper next to `resolveSystemPromptOverride` and use it in the embedded attempt, compaction retry, and CLI runner prompt assembly paths so overrides remain intact while dynamic runtime prompt additions are preserved.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure `systemPromptOverride`, spawn a native subagent, and the override-first `??` branches return before the builders can append the `extraSystemPrompt` task block.

## Solution Assessment

Yes. The narrowest maintainable fix is to compose the resolved override with trimmed runtime `extraSystemPrompt` at the three prompt assembly sites, backed by helper tests and focused runner/CLI regression tests.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/73624. Current main drops task-specific `extraSystemPrompt` whenever `systemPromptOverride` resolves, so native subagents can receive the bootstrap message without the `## Your Role` task block. Add a small shared helper next to `resolveSystemPromptOverride` that returns undefined when there is no override, returns the trimmed override when there is no nonblank extra prompt, and returns `<override>\n\n<extraSystemPrompt>` when both are present. Add regression coverage in `src/agents/system-prompt-override.test.ts` and focused runner/CLI prompt tests so override-plus-extra preserves the subagent `## Your Role` block. Do not solve the separate visible-bootstrap request in #73656, and do not duplicate the full task into `buildSubagentInitialUserMessage`. Add a single-line changelog Fixes entry if behavior changes are user-facing, with Thanks attribution to the reporter and any reused prior PR contribution.
