## Summary

Keep this issue open. Current main still allows the Kimi web_search provider to return final chat-completion text as a successful search payload even when no citation or search-result evidence was collected. The docs now describe Kimi as a synthesized-answer provider, and the merged Kimi repair in #59356 fixed adjacent setup/replay problems, but neither adds the requested fail-closed guard for generic no-search answers.

## Best Possible Solution

Keep this issue open. The best fix belongs in the Moonshot plugin runtime: require evidence that native $web_search actually produced grounding, such as search_results, citation URLs, or replayable tool-result payloads, before returning success. If no grounding evidence exists, return a structured Kimi provider failure instead of wrapped chat text. Add focused unit tests for final chat-only/no-citation responses, unreplayable tool calls, and normal grounded responses, then update https://docs.openclaw.ai/tools/kimi-search if the visible behavior changes.
