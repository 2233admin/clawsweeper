## Summary

Keep open. Current main documents Kimi as a synthesized-answer provider and includes the adjacent #59356 replay/setup repair, but the Moonshot plugin still treats final chat text with empty grounding as a successful web_search result. The requested fail-closed guard for generic no-search answers remains missing.

## Best Possible Solution

Keep this open and fix it in the Moonshot plugin runtime. Kimi web_search should only return success when native $web_search produced grounding evidence, such as search_results, citation URLs, or replayable tool-result payloads. If Kimi returns final chat text with no grounding evidence, return a structured Kimi provider failure instead of wrapped chat text. Add focused tests for chat-only/no-citation responses, unreplayable tool calls, and grounded success. Update the public Kimi search docs only if the visible behavior changes.
