## Summary

Keep this PR open. Current main still rejects Feishu external media keys containing embedded `..`, so the PR’s intended fix for legitimate Feishu image/file keys is not implemented. The latest PR diff is narrow and no longer has the earlier slash-separator regression, but it still needs maintainer follow-up to reject standalone dot-segment values such as `..` before merge.

## Best Possible Solution

Refresh and land the narrow Feishu plugin fix: keep empty, length, control-character, `/`, and `\` rejection; allow embedded `..` inside opaque Feishu media keys; explicitly reject standalone dot-segment tokens such as `..` (and ideally `.`); and keep regression tests for valid embedded-dot keys plus invalid separator and dot-token inputs.
