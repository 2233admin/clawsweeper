## Summary

Keep this PR open. Current main still rejects Feishu external media keys containing embedded `..`, so the PR’s intended Feishu image/media fix is not implemented on main. The latest PR diff is narrow and no longer appears to include the earlier slash-separator regression, but it should be refreshed to reject exact dot-segment tokens such as `.` and `..` before merge.

## Best Possible Solution

Refresh this PR with the narrow Feishu plugin fix: keep empty, length, control-character, `/`, and `\` rejection; allow embedded `..` in opaque Feishu keys; explicitly reject exact dot-segment values such as `.` and `..`; and keep regression tests for valid embedded-dot keys plus invalid separator and dot-token inputs.
