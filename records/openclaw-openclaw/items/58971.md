## Summary

Keep this PR open. Current main does not implement the proposed `cron.webhookAllowPrivateNetwork` opt-in: cron webhook delivery still calls `fetchWithSsrFGuard` without a policy, the cron config type/schema do not accept the new key, and the docs/tests only cover the default blocked behavior. The PR is a focused implementation candidate for a real cron webhook/private-network use case discussed in the PR body and related issue context, so the best maintainer path is review and land an acceptable version rather than close it as obsolete.

## Best Possible Solution

Review and land this PR, or a maintainer-authored equivalent, with the config schema/type/docs/test coverage kept in sync and explicit security wording for the operator opt-in. Current main has the guard infrastructure needed, but the cron-specific config surface and policy plumbing are still missing.
