## Summary

Keep open. Current main still exposes fal media generation through static, curated image/video model arrays, while this issue asks for dynamic full-catalog discovery across heterogeneous fal image/video modalities; that remains real product/API work rather than cleanup-close material.

## What This Changes

Request dynamic discovery and selectable exposure of the full FAL.ai media model catalog through the bundled fal image/video generation provider.

## Best Possible Solution

Define a scoped fal media catalog strategy first, keep vendor behavior in the fal plugin, add a generic discovered-catalog seam only if current media provider contracts are insufficient, then implement capability-mapped fal coverage with docs and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-state check: inspect the registered fal media providers and the shared list action, which show static provider.models arrays on current main.

## Solution Assessment

Unclear. Exposing every FAL endpoint as built-in selectable media models is not yet proven to be the narrowest maintainable solution; a filtered provider-owned catalog with per-model capability/schema mapping is safer.
