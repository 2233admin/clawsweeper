## Summary

Keep open. Current main still does not expose the full FAL.ai catalog, and the remaining work is broader than conservative cleanup because it likely needs maintainer direction on bundled-provider scope versus ClawHub and on whether media providers need a discovered/async catalog contract.

## What This Changes

Request to dynamically discover and expose the full FAL.ai media model catalog in the FAL image/video generation provider instead of the current curated static model lists.

## Best Possible Solution

Define a scoped FAL catalog strategy before implementation: keep core generic, decide whether media providers need an async/discovered model catalog contract, and then implement FAL coverage either as a maintained external ClawHub plugin or as a curated bundled provider expansion with model-specific capability tests and operator docs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug report, but yes for the current-state check: source inspection on current main shows the FAL image and video providers register static model arrays and the list actions surface those arrays.

## Solution Assessment

Is this the best way to solve the issue? Unclear. Exposing the entire FAL catalog inside the bundled provider may not be the narrowest maintainable approach because the catalog spans many modalities and schema shapes; maintainer API/product direction should come first.
