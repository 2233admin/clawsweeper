## Summary

Keep open. Current main still exposes FAL image and video generation through provider-owned static model lists, while the request asks for dynamic full FAL catalog discovery across many modalities; this is real remaining work and needs maintainer product/API direction before implementation.

## What This Changes

The issue requests dynamic discovery and selectable exposure of the full FAL.ai model catalog through the bundled fal media provider instead of the current curated model lists.

## Best Possible Solution

Define a scoped FAL catalog strategy first: keep core generic, decide whether media providers need an additive discovered-catalog API, and implement provider-owned, capability-mapped FAL coverage with docs and tests rather than exposing a raw 1000+ endpoint dump.

## Reproduction Assessment

Not applicable as a bug reproduction. We do have a high-confidence current-state check from source and docs: FAL image/video support is registered through curated model arrays and the list surfaces return those arrays directly.

## Solution Assessment

Unclear. Exposing every FAL endpoint as selectable built-in models is not yet proven to be the narrowest maintainable solution; a filtered, provider-owned catalog or a generic media catalog API may be safer than a raw full-catalog dump.
