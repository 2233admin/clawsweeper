## Summary

Keep open. Current main still lacks the active-job producer signal on the startup catch-up and manual cron execution paths; newer durable cron recovery only helps after terminal history exists, so it does not protect a live long-running cron task before completion.

## What This Changes

The PR adds active-job mark/clear instrumentation around startup catch-up and manual cron runs, and adds a Vitest regression file covering both execution paths.

## Best Possible Solution

Review and land this PR or an equivalent patch that marks startup catch-up and manual cron runs active during execution, clears the marker in finally blocks, and keeps the symmetry regression tests; leave #68191 to track broader durable restart/reconciliation policy.
