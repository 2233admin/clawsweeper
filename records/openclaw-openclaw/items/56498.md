## Summary

Keep open. This is a recent bundled-channel/Hostinger setup bug report with one corroborating comment, but the current evidence does not map cleanly to a shipped fix, duplicate, or external-only action. Current main has working WhatsApp QR controls, Telegram token setup docs, channel status probes, and related transport fixes, but the report’s “save button stays grey” symptom could be Control UI dirty-state gating, a Hostinger 1-Click setup problem, an invalid Telegram token path, or a WhatsApp QR/login path failure. More reproduction data is needed before conservative cleanup can close it.

Keep this issue open and ask the reporter to retest on current main/latest release, clarify whether the failing screen is Hostinger 1-Click or OpenClaw Control UI, and attach `openclaw status --deep`, `openclaw channels status --probe`, channel logs, screenshots of the disabled Save state, Telegram token/probe outcome, and WhatsApp QR/login messages. If it reproduces in Control UI, the likely maintainer path is a focused fix in `ui/src/ui/views/channels.config.ts`, `ui/src/ui/controllers/config.ts`, or the WhatsApp/Telegram plugin setup/status surfaces; if it only reproduces in Hostinger’s managed setup, hand it to the hosted deployment owner with repo docs updated if needed.
