## Summary

Keep this PR open. Current main still does not implement the proposed live queue diagnostics snapshot, `diagnostics.queue` Gateway RPC, or `openclaw gateway diagnostics queue` CLI. The related `queue_status` issue remains an adjacent agent-facing feature request, not a shipped replacement for this operator-facing Gateway diagnostics surface.

## Best Possible Solution

Keep this PR open for maintainer review or an updated successor. The best path is a core-owned, read-only queue snapshot seam in `src/process/command-queue.ts`, exposed through a `diagnostics.queue` Gateway method and `openclaw gateway diagnostics queue`, scoped deliberately, documented in the Gateway/queue docs, and covered by focused queue, Gateway handler, CLI wiring, and authorization tests. Keep #9797 separate for the agent-facing `queue_status` tool discussion.
