Keep this PR open. Current main still lacks the requested model-visible hint for a registered skill called as an unknown tool, and the PR remains a focused implementation candidate, but its current hook wiring does not match the pinned pi-agent-core public/runtime contract.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. The best path is to rebase onto current main, wire the hint through a supported missing-tool result path for the pinned pi-agent packages, preserve existing extension `afterToolCall` behavior, define visible-skill matching deliberately, and add targeted regression tests for skill-match, non-match, and tool-collision cases.
