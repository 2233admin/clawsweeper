## Summary

Keep this PR open. Current main does not implement the requested model-visible hint for a skill name called as an unknown tool, and no merged canonical OpenClaw item in the provided related context supersedes it. The PR is a focused candidate, but it needs rework against the current pi-agent-core hook contract and the review-thread edge cases before it is mergeable.

## Best Possible Solution

Keep the PR open as a useful focused candidate, but rebase and rework it against current main. The best implementation should route unknown-tool immediate results through a supported model-visible result-rewrite path, avoid private upstream fields, share the unknown-tool parser where appropriate, check visible skills without missing valid snapshots, skip real tool collisions, and add targeted tests for skill match, non-skill unknown tool, collision, `$`/punctuation names, and the pinned pi-agent-core missing-tool path.
