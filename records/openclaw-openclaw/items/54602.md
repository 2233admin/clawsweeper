## Summary

Keep open. Current main still does not implement this PR’s intended OpenRouter free-route behavior: OpenRouter onboarding always writes `openrouter/auto`, the bundled static catalog does not include `openrouter/free`, and the current stream wrapper does not treat the `free` route like `auto` for thinking injection. The PR is still useful, but it is stale against current main because OpenRouter stream wrapping has moved out of `extensions/openrouter/index.ts` into `extensions/openrouter/stream.ts`, and the change needs focused regression tests before landing.

## Best Possible Solution

Keep this PR open, refresh it onto current main, and land a plugin-local fix if maintainers agree with the behavior: preserve an existing OpenRouter primary when applying OpenRouter API-key onboarding, add `openrouter/free` to the static OpenRouter catalog if it should be selectable, update the current `extensions/openrouter/stream.ts` wrapper to avoid thinking injection for the free route, and add focused OpenRouter onboarding/catalog/stream regression tests. The linked bug report #54581 is closed, but the current main code path still has the overwrite behavior this PR was meant to fix.
