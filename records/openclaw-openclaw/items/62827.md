The issue requests that WhatsApp group activation commands accept `/activate` as a shorter alias for `/activation`.

## Best Possible Solution

Review and land or replace #62841 with a focused fix that makes `/activate` canonicalize to the existing `/activation` command, keeps `/activation` working, and adds coverage for bare, argument, and colon forms; updating command metadata/docs to surface the alias would keep the user-facing command list consistent.
