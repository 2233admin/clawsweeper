## Summary

Keep open. Current main still lacks the requested subagent lifecycle hardening, and the refreshed branch fixes the earlier undefined-helper/lint issue, but the latest diff still has blocking correctness gaps in the new readiness probe and sweep delete retry path.

## What This Changes

The PR exports the announce transient classifier, retries announce finalize calls, adds a pre-spawn gateway readiness probe, wraps registry sweep deletion in a helper, and expands focused subagent lifecycle tests.

## Best Possible Solution

Revise the PR branch so spawn readiness uses a lifecycle-safe scope strategy, registry sweep cleanup performs bounded immediate retries for known transient delete failures, and the focused agents tests plus changed gate pass before merge.

## Reproduction Assessment

Yes. The remaining PR defects are reproducible by source inspection: the new readiness probe is `sessions.list` before higher-scope lifecycle calls, and the new sweep helper has only one `sessions.delete` call.

## Solution Assessment

No. The bug-fix direction is sound, but the current patch is not the best mergeable solution until the readiness probe avoids the documented scope-upgrade failure and the sweep helper implements real bounded retry.

## Review Findings

Overall correctness: patch is incorrect
