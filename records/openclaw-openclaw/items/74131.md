## Summary

Keep open. The refreshed PR fixes the earlier undefined-helper gap, but the current PR head still has two merge-blocking correctness issues: the spawn readiness probe uses a lower-scope method on a path that already documents scope-upgrade failures, and the sweep cleanup helper does not actually retry transient delete failures.

## What This Changes

The PR branch adds subagent gateway readiness, announce finalize, and sweep cleanup lifecycle hardening plus focused agents tests.

## Best Possible Solution

Revise the PR branch so the spawn readiness hardening uses a scope-safe lifecycle strategy, the registry sweep delete helper performs bounded immediate retries for known transient delete failures, and the focused agents tests plus changed gate pass before merge.

## Reproduction Assessment

Yes. The remaining defects are reproducible by source inspection and focused unit tests: assert the readiness probe's first gateway method/scope before spawn, and make sweep cleanup throw one transient delete error then verify a bounded retry happens before the run is kept for a later sweep.

## Solution Assessment

No. The bug-fix direction is sound, but this PR is not the best mergeable solution until the readiness probe avoids the documented scope-upgrade path and the sweep helper implements real bounded retry behavior.
