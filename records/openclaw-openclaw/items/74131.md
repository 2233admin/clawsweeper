## Risks / Open Questions

- The patch as submitted likely fails TypeScript because it imports `isTransientAnnounceDeliveryError` from a module that does not export it on current main.
- The patch calls `deleteSubagentSessionWithRetry`, but that helper is not present under `src/agents` in current main and is not added by the patch.
- The proposed readiness probe calls `sessions.list` before the `agent` call; because `sessions.list` is read-scoped and `agent` is write-scoped, maintainers should verify this does not reintroduce the scope-upgrade timing problem described in the existing `callSubagentGateway` comment.
- The readiness classifier uses broad string matching such as `gateway closed`; it should not retry permanent authorization or scope failures.
- The test change removes the archive context-engine assertion in `subagent-registry.test.ts`; the cleanup/end notification path still needs coverage if the implementation changes sweep behavior.
