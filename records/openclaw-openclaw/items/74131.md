## Summary

Keep open. Current main still lacks the requested lifecycle hardening, and this refreshed PR addresses some earlier review feedback, but the latest head still has blocking issues: a failing lint annotation in the new test mock, a read-scoped readiness probe that can reintroduce the known gateway scope-upgrade failure, and a sweep delete helper that still performs only one delete attempt.

## What This Changes

The branch adds a subagent gateway readiness preflight, transient retry handling for announce finalize calls, a registry sweep delete helper, and focused subagent lifecycle tests.

## Best Possible Solution

Keep the PR open and request changes: fix the lint failure, make the spawn readiness check avoid establishing a lower-scope connection before lifecycle calls, implement real bounded retry for registry sweep deletion, then rerun the focused agents tests plus the changed gate.

## Reproduction Assessment

Yes. The CI lint annotation reproduces the test mock failure, and the readiness-scope and sweep-helper defects are directly reproducible by inspecting the PR diff against current method-scope and gateway wrapper code.

## Solution Assessment

No. The bug-fix direction is reasonable, but the current patch is not the best mergeable solution until it uses a scope-safe readiness mechanism and a real retry wrapper for sweep cleanup.
