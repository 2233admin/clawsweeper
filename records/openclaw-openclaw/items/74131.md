## Summary

Keep open. The PR targets a real missing subagent lifecycle hardening path, but the current branch still has two concrete correctness issues: the new readiness preflight uses a lower-scope `sessions.list` call before higher-scope lifecycle calls, and the new sweep helper does not actually retry transient `sessions.delete` failures.

## What This Changes

The PR adds subagent lifecycle resilience code and tests for spawn gateway readiness, announce finalize retries, and registry sweep cleanup deletion.

## Best Possible Solution

Revise the branch so the spawn readiness check uses a lifecycle-safe scope strategy, registry sweep cleanup performs real bounded retry for known transient delete failures, and the focused agents tests plus the changed gate pass before merge.

## Reproduction Assessment

Yes. The PR defects are reproducible by source inspection against the provided head diff: `sessions.list` is the new preflight before higher-scope calls, and `deleteSubagentSessionWithRetry()` contains only one `sessions.delete` attempt.

## Solution Assessment

No. The bug-fix direction is sound, but this branch is not the best mergeable solution until the readiness probe avoids the documented scope-upgrade failure mode and the sweep helper implements actual retry behavior.
