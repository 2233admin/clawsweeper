## Summary

Keep open. Current main still shallow-clones plugin hook context while the bootstrap hook consumer and docs expect mutable `agent:bootstrap` context, and PR #75249 is open/unmerged with closing syntax for this issue.

## What This Changes

The issue requests that `api.registerHook()` preserve mutable `event.context` updates, especially `agent:bootstrap` changes to `bootstrapFiles`, instead of losing them through the pluginConfig wrapper clone.

## Best Possible Solution

Review and land PR #75249, or an equivalent narrow change in the plugin hook wrapper that passes the original context while temporarily adding/restoring `pluginConfig`, with regression coverage for plugin-registered `agent:bootstrap` mutation and pluginConfig cleanup.

## Reproduction Assessment

Yes. A source-level reproduction is clear: register a plugin hook with `api.registerHook('agent:bootstrap')`, assign a sentinel array to `event.context.bootstrapFiles`, then resolve bootstrap context and observe current main still consumes the original event context.

## Solution Assessment

Yes. The narrow maintainable fix is in the plugin hook wrapper, not an `agent:bootstrap` special case or a larger immutable hook API redesign; keeping this issue open until PR #75249 lands is the right workflow.
