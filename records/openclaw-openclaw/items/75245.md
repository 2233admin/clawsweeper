## Summary

Keep open. Current main still shallow-clones plugin hook event context inside api.registerHook(), while the bootstrap hook path reads the original event context after hooks run. The linked fix PR is open and targets this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that plugin handlers registered with api.registerHook('agent:bootstrap') lose event.context.bootstrapFiles mutations because the wrapper shallow-clones event.context while injecting pluginConfig.

## Best Possible Solution

Land PR #75249 or an equivalent narrow fix that passes the original hook context to plugin handlers while temporarily adding and restoring pluginConfig, with regression coverage for agent:bootstrap mutation propagation and pluginConfig cleanup.

## Reproduction Assessment

Yes. The high-confidence reproduction path is source-level: register a plugin hook with api.registerHook('agent:bootstrap') that assigns event.context.bootstrapFiles, then observe that current main reads the unchanged original context in applyBootstrapHookOverrides().

## Solution Assessment

Yes. Preserving the original context object while adding and restoring pluginConfig is the narrowest compatible solution; replacing the hook contract with an immutable return-value API would be a larger plugin API change.
