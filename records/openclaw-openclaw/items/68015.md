## Summary

Keep this open. Current main has added partial coverage for the exec-host footgun and has useful diagnostics, task, process, MCP, and Control UI log surfaces, but the central cleanup is not complete: config validation still has no nearest-value guidance for `tools.exec.host="host"`, fanout observability remains split across separate tools, and the deterministic local maintenance-runner pattern is still undocumented.

## What This Changes

The issue asks for better invalid `tools.exec.host` guidance, broader runtime fanout and memory observability, docs for model-free local maintenance runners, and possible inline dashboard log review improvements.

## Best Possible Solution

Split the field report into focused maintainer-owned follow-ups: add nearest-value guidance for invalid config enums if desired, design one operator status surface for runtime fanout and load, document deterministic local scripts/systemd user timers for routine maintenance outside model turns, and treat inline dashboard log review as a separate Control UI product decision.
