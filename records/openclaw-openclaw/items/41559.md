  - body: The ingress echo send and nested guard history lookup call the module-level gateway helper even though `createSessionsSendTool` has selected `opts.callGateway` as `gatewayCall`.   - confidence: 0.89

  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The diff introduces new external channel relay paths, and the current branch bypasses injected gateway context while leaving strict relay delivery fail-open.

Concerns:

- **[medium] New delivery path bypasses injected gateway context:** `src/agents/tools/sessions-send-tool.ts:297`
  - body: The new ingress echo send and nested guard history lookup call the module-level gateway helper instead of the injected caller selected by `createSessionsSendTool`, which can bypass scoped gateway resolution while sending to external channels or reading history.
