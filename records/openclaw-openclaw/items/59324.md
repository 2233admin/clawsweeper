Related open PRs overlap, but the provided context does not identify a merged or maintainer-selected canonical replacement.

## Best Possible Solution

Keep #59324 open until maintainers land it or choose an equivalent canonical fix for #41199's `sessions_send` portion. The best implementation is a narrow executor-side precedence rule in `src/agents/tools/sessions-send-tool.ts`: treat a non-empty `sessionKey` as authoritative, ignore `label` and `agentId` for lookup in that case, preserve label-only resolution and downstream visibility/A2A checks, and keep the regression test proving mixed inputs skip `sessions.resolve`.
