## Summary

Current main still has fallback-only subagent timeout config behavior: a finite model/tool-supplied `runTimeoutSeconds` wins over `agents.defaults.subagents.runTimeoutSeconds`, and there is no `minRunTimeoutSeconds`, lock option, schema support, docs entry, or regression test for a configured floor. Closed PRs #37973, #38028, and #38048 remain useful prior art but did not merge, so this issue is still actionable.

## Best Possible Solution

Keep this issue open and land a focused, backward-compatible fix that adds an optional configured floor such as `agents.defaults.subagents.minRunTimeoutSeconds`, clamps finite positive model-supplied native `sessions_spawn.runTimeoutSeconds` values below that floor, preserves current behavior when unset, updates config types/zod/generated schema/docs/changelog, and adds regression tests for omitted timeout, explicit value above the floor, explicit value below the floor, zero/no-timeout behavior, and no-floor compatibility.
