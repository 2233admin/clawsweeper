## Summary

Keep this open. Current main still lacks the requested user-facing `voice.autoJoin[].sessionChannelId` config, still routes voice turns from the joined voice channel, and still suppresses Discord text delivery for voice transcript turns; an open implementation PR also references this issue with GitHub closing syntax, so the issue should remain open until that work lands or is superseded.

## What This Changes

Add an optional Discord voice `autoJoin[].sessionChannelId` text-channel binding so voice transcripts and agent replies can be routed to a companion text channel while audio playback remains in the voice channel.

## Best Possible Solution

Implement this in the Discord plugin/config surface: add a typed and schema-validated `autoJoin[].sessionChannelId`, propagate it through auto-join and voice session creation, route the agent session to the companion text channel when configured, enable Discord delivery only for that paired route, update the Discord docs/config metadata, and add focused config plus voice manager/segment tests while reconciling the behavior with #73699 and #47095.
