## Summary

Current main no longer has the dependency chain that produced the reported false `ERR_PNPM_LOCKFILE_MISSING_DEPENDENCY` for `@cloudflare/workers-types@4.20260405.1` and `@types/bun@1.3.11`. The issue reports that `pnpm install --no-optional` fails with a false missing-lockfile error for optional dependencies even though a normal install succeeds.

## Best Possible Solution
