## Summary

Keep open. Current main still disables token-based memory flush and preflight compaction when the computed threshold is non-positive, and PR #50694 is still the open implementation path that explicitly targets this issue.

## What This Changes

The issue reports that setting `agents.defaults.compaction.reserveTokensFloor` equal to the context window makes the memory-flush threshold non-positive and asks for a safer default or clamp.

## Best Possible Solution

Review and land PR #50694, or replace it with a maintainer-preferred fix that guarantees a positive token threshold for both memory flush and preflight compaction when reserve plus soft-threshold settings exceed the context window, with regression tests for equal and greater-than-context reserve values.
