## Summary

Keep open. Current main still disables the token-based memory-flush and preflight-compaction paths when the computed threshold is non-positive, and PR #50694 is still the open unmerged implementation candidate for the same bug.

## What This Changes

The issue requests that `reserveTokensFloor` values equal to or greater than the model context window no longer prevent memory flush or preflight compaction from triggering.

## Best Possible Solution

Land PR #50694 or a maintainer-preferred replacement that guarantees a positive token-based flush and preflight-compaction threshold when reserve settings are misconfigured relative to the context window, with regression coverage for equal and greater-than-context reserve values.
