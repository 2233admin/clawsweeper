## Summary

Keep open. Current main has general native Ollama cloud/local routing, but it still does not show source, tests, docs, or release history proving that `ollama/gemini-3-flash-preview:cloud` or `ollama/deepseek-v4-pro:cloud` work through the full OpenClaw agent path. The reporter validated direct `/api/generate`, while OpenClaw’s Ollama agent transport uses streamed `/api/chat` with the agent tool surface and only has an explicit cloud-model compatibility branch for Kimi.

Keep this open and reproduce both named Ollama Cloud models through the native OpenClaw `/api/chat` agent path with tools enabled and disabled. If the failure is payload-specific, add the smallest Ollama-plugin compatibility path in `extensions/ollama/src/stream.ts` or capability handling in `extensions/ollama/src/provider-models.ts`, with regression tests for the observed request/response shape and a docs workaround only if the limitation remains model-specific.
