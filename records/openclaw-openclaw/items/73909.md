## Summary

Keep open. Current main supports Ollama cloud/local routing generally, but I do not see source, tests, docs, or history proving that `ollama/gemini-3-flash-preview:cloud` or `ollama/deepseek-v4-pro:cloud` are handled through the full OpenClaw agent path. The current Ollama plugin still sends streamed native `/api/chat` requests with the agent tool surface and only has model-specific cloud compatibility for Kimi, while this report validated direct `/api/generate`, which is a narrower and different Ollama API path.

## Best Possible Solution

Keep this issue open and reproduce the two named Ollama Cloud models through the native OpenClaw `/api/chat` path with the agent tool surface enabled. If the failure is model-specific payload incompatibility, add the smallest Ollama-plugin compatibility path in `extensions/ollama/src/stream.ts` or model capability handling in `extensions/ollama/src/provider-models.ts`, with regression tests covering the failing request/response shape and a docs note or workaround only if behavior remains model-limited.
