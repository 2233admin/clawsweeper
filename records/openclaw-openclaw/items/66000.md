## Summary

Keep this PR open. Current main still applies `--profile` early but does not clear a stale inherited `OPENCLAW_LAUNCHD_LABEL`, while launchd plist resolution still gives that inherited label precedence over `OPENCLAW_PROFILE`. The linked bug #65643 remains present on current main, and this PR is a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Review and land this PR, or an equivalent narrow patch, as the canonical fix for #65643. The fix should clear stale OpenClaw-managed launchd labels during explicit profile setup, preserve custom operator `OPENCLAW_LAUNCHD_LABEL` overrides, keep regression tests for profile env cleanup and custom label precedence, and retain the changelog credit.
