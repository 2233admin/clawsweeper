## Summary

Keep open. Current main still intentionally blocks implicit WebChat/dashboard-to-external origin delivery unless an explicit delivery route is set, and the preferred persisted-origin implementation exists only in the closed, unmerged related PR.

## What This Changes

Requests a configurable or automatic reply-routing policy that guarantees agent replies are delivered back to the messaging channel that originated the turn, even when WebChat or dashboard surfaces are active.

## Best Possible Solution

Keep this open for a maintainer-reviewed reply-routing design: either an explicit opt-in force-origin setting or an immutable first-external-origin route with stale-session and privacy safeguards, plus regression tests for Telegram/Discord/WhatsApp-origin sessions while WebChat/dashboard is active.
