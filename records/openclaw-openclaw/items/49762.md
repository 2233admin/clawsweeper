## Summary

Keep this PR open. Current main still does not implement response duration in the per-reply usage footer: duration is emitted only in diagnostics, while the user-visible footer formatter and call site remain token/cache/cost only. The PR is a plausible focused feature, but the existing review comments identify real rework needed before merge.

## Best Possible Solution

Keep this PR open for author or maintainer rework: add duration to the intended usage-footer mode, preserve existing formatting when duration is absent, decide whether `/usage tokens` should remain token-only, choose or label the measured timing span accurately, and add targeted formatter/runner tests plus docs if the user-visible footer changes.

- Passing duration unconditionally would make `/usage tokens` include stopwatch metadata despite current docs describing that mode as token-only.
- The proposed timing point may report post-generation delivery drain time as response duration unless the sample point or label is adjusted.
- The PR needs focused tests for formatter output, mode gating, separator preservation, and duration display before it is safe to merge.
