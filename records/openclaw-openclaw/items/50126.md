## Summary

Keep open. Current main has good coverage for the shared `deliverOutboundPayloads` path and Telegram reply delivery, including tests, but the broad issue is not fully resolved. WebChat/TUI-style `chat.send` replies still append and broadcast assistant output without invoking `message_sent` / `message:sent`, and plugin-handled channel message actions can return from `dispatchChannelMessageAction` without the outbound hook layer. The open related #70928 tracks one concrete remaining slice; closed-unmerged #50934 only partially overlaps the Telegram portion.

## Best Possible Solution

Keep the issue open for a maintainer-scoped outbound lifecycle audit. The best implementation direction is to define one generic post-delivery observer seam, then wire missing paths such as `chat.send` final assistant replies and plugin-handled channel actions into it with focused tests. If maintainers prefer smaller work, split the remaining gaps into canonical issues such as #70928 for WebChat/TUI and a separate plugin-action coverage issue, then close this umbrella only after those paths are fixed or deliberately scoped out.
