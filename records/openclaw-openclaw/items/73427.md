## Summary

Keep this issue open. Current main still has the reported Control UI/OpenAI Realtime gaps: CSP omits the OpenAI HTTPS origin while the browser WebRTC transport posts SDP directly to OpenAI, the WebRTC data-channel handler drops error and lifecycle events, and the OpenAI browser-session payload only configures output voice.

## Best Possible Solution

Keep this issue open and fix the bundled Control UI/OpenAI realtime path: allow the exact OpenAI Realtime HTTPS origin or proxy the SDP exchange through the Gateway, add explicit browser-session input audio/VAD/transcription config where maintainers want it, surface data-channel `error` and basic speech/response lifecycle state in the UI, and add focused CSP/provider/UI regression coverage plus live smoke proof when credentials are available.
