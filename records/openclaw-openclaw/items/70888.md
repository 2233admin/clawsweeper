## Summary

Keep this PR open: current main still skips the default on-disk session-store lookup when `cfg` is omitted, and the PR targets that root cause with a narrow helper change plus focused regression tests. The patch appears correct; the only concrete issue I found is a low-priority test temp-directory cleanup in the new announce requester tests.

## What This Changes

The PR removes the no-`cfg` early return in subagent depth resolution, passes an optional session store config into `resolveStorePath`, and adds regression coverage for explicit acct-suffixed requester session depth and announce classification.

## Best Possible Solution

Land the narrow depth-helper fix after normal validation and clean up the new announce-test temp directories, keeping the behavior scoped to persisted session-depth resolution rather than changing routing or delivery policy.

## Reproduction Assessment

Yes. Persist an `agent:main:explicit:...:acct:default` entry with `spawnDepth` in the default `sessions.json` and call `getSubagentDepthFromSessionStore(key)` without `cfg`; current main exits before reading the store and falls back to depth 0.

## Solution Assessment

Yes, with a small test cleanup. Reusing `resolveStorePath()` with optional `cfg` is the narrowest maintainable fix because the default `sessions.json` fallback already belongs there.
