## Summary

Keep open. Current main still has the reported false-negative shape: the embedded-runner treats a zero-payload terminal turn with `sessions_spawn` activity as incomplete, and the accepted child-session result is not used as a terminal-completion exemption.

## What This Changes

The issue requests that gateway incomplete-turn detection stop warning on pure relay turns that successfully finish by calling `sessions_spawn` and receiving an accepted child session.

## Best Possible Solution

Add a structured current-attempt success signal for `sessions_spawn` and have the embedded-runner completion classifier treat `status: "accepted"` with a non-empty `childSessionKey` as terminal completion, while preserving warnings for failed spawns, missing child keys, and empty turns with no successful terminal action.

## Reproduction Assessment

Yes. A focused unit reproduction can build a zero-payload `stop` attempt with no assistant text, `sessions_spawn` tool activity, side-effect replay metadata, and no committed message delivery; current `resolveIncompleteTurnPayloadText` returns the warning path instead of accepting the spawn as completion.

## Solution Assessment

Yes. The narrowest maintainable fix is to recognize successful `sessions_spawn` completion in the embedded-runner completion path, not to disable empty-turn detection globally or require every pure relay agent to emit an acknowledgement token.
