## Summary

Keep open. Current main still has the reported false-negative shape: `sessions_spawn` is tracked as a mutating side effect, but the incomplete-turn classifier has no success signal for an accepted spawn with a child session, so a pure relay can still end with zero payloads and get the warning.

## What This Changes

Treat a zero-payload agent turn ending in a successful `sessions_spawn` acceptance as complete rather than surfacing the incomplete-turn warning.

## Best Possible Solution

Add structured current-attempt metadata for successful `sessions_spawn` results and make the incomplete-turn classifier accept `status: "accepted"` with a non-empty `childSessionKey` as terminal completion, while preserving warnings for failed spawns, malformed results, and empty turns without a successful terminal tool action.

## Reproduction Assessment

Yes. A focused unit test can construct `payloadCount: 0`, `assistantTexts: []`, `toolMetas: [{ toolName: "sessions_spawn" }]`, replay metadata from `buildAttemptReplayMetadata`, and a non-error stop; current main still returns the side-effect incomplete-turn warning.

## Solution Assessment

Yes. The narrowest maintainable fix is to propagate the actual successful spawn result into the runner completion classifier, not to disable empty-turn detection globally or require agent-specific acknowledgement text.
