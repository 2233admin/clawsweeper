## Summary

Keep open. Current main still has the reported false-negative shape: successful `sessions_spawn` calls are tracked as side-effecting tool activity, but the incomplete-turn classifier does not treat an accepted spawn as terminal completion, so a zero-payload stop can still surface the warning.

## What This Changes

Treat a zero-payload terminal turn with a successful `sessions_spawn` result as complete instead of surfacing the incomplete-turn warning for pure-relay agents.

## Best Possible Solution

Add structured current-attempt completion metadata for successful `sessions_spawn` results and teach the embedded-runner incomplete-turn classifier to treat an accepted spawn with a child session key as terminal completion, while preserving existing warnings for failed spawns and empty turns without successful terminal actions.

## Reproduction Assessment

Yes. A focused unit test can build a zero-payload stop attempt with `toolMetas` containing `sessions_spawn`, replay metadata from `buildAttemptReplayMetadata`, no assistant text, and no committed messaging delivery; current `resolveIncompleteTurnPayloadText` will return the side-effect incomplete-turn warning instead of null.

## Solution Assessment

Yes. The narrow maintainable solution is in attempt metadata plus the embedded-runner completion classifier, not a global disable of empty-turn detection, an agent-specific workaround, or a config flag that leaves the documented accepted-spawn contract unrecognized.

## Repair Work Prompt

Fix the pure-relay `sessions_spawn` incomplete-turn false-negative from https://github.com/openclaw/openclaw/issues/72541. On current main, zero-payload terminal turns after a successful `sessions_spawn` are still classified as incomplete because the runner only carries tool name/meta and replay side effects into `resolveIncompleteTurnPayloadText`. Add a typed, bounded current-attempt signal for successful `sessions_spawn` results, requiring accepted status plus a non-empty `childSessionKey`, and make the incomplete-turn classifier return null for that terminal completion. Preserve existing behavior for failed/rejected spawns, missing child session keys, ordinary empty turns, silent-reply policy, committed messaging delivery, cron add side effects, and replay-invalid side-effect warnings. Keep the fix in the embedded-runner/session-spawn boundary; do not add agent-specific Claudette behavior, do not globally disable empty-turn detection, and do not broaden plugin/tool APIs. Add focused coverage in `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts` and adjust typed attempt metadata in the smallest necessary files. Because this is user-facing warning behavior, add a single-line `CHANGELOG.md` fix entry with reporter credit `Thanks @alfredneuworth`.
