## Summary

Keep open. Current main still has the reported false-negative shape: the incomplete-turn classifier can only exempt silent replies or committed visible delivery, while accepted `sessions_spawn` results are treated as replay side effects rather than terminal completion.

## What This Changes

The issue requests that a zero-payload agent turn ending in a successful `sessions_spawn` call be treated as completed instead of surfaced as an incomplete gateway response.

## Best Possible Solution

Add a narrow runner-level completion signal for current-attempt successful `sessions_spawn` results and let the incomplete-turn classifier treat that signal like committed delivery, while preserving the existing warning for truly empty turns and failed spawn results.

## Reproduction Assessment

Yes. A source-level reproduction can construct a zero-payload attempt with a normal stop and a current `sessions_spawn` side effect; current `resolveIncompleteTurnPayloadText` has no accepted-spawn exemption and will return the warning path.

## Solution Assessment

Yes. The best fix is a structured success exemption in the embedded-runner completion classifier; changing the spawn contract or disabling empty-turn detection globally would be broader and less maintainable.
