## Summary

Keep open. Current main mitigates the linked zombie-session accumulation via cron run-session pruning, but it still does not implement this PR's narrower behavior of persisting isolated cron session `running` and terminal `done`/`failed`/`timeout` status in the session store. The branch also needs conflict resolution and re-review of prior edge-case comments before it is mergeable.

## Best Possible Solution

Keep this PR open, or replace it with a rebased maintainer follow-up that persists isolated cron run session status best-effort after all terminal outcomes are classified, including success, fatal payload, thrown error, timeout, delivery failure, and persistence-failure paths, with focused regression coverage.
