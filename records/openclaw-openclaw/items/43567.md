## Summary

Keep open. Current main fixed the narrow pre-compaction memory-flush transcript leak, but it still has not implemented a distinct system/runtime display treatment across Control UI, Gateway projection, and TUI history replay.

## What This Changes

The issue asks for internal system/runtime messages to appear as clearly system-originated UI blocks instead of normal user or assistant chat bubbles.

## Best Possible Solution

Define a structural display contract for intentionally visible runtime/system messages, carry that marker through `chat.history`, render it with a dedicated Control UI system treatment, and make TUI history/live display use the same system path while keeping runtime-only turns hidden.
