## Summary

Keep open. Current main fixes the narrow pre-compaction memory-flush transcript leak, but it still lacks the requested cross-surface contract for intentionally visible runtime/system messages: gateway projection does not promote internal provenance into a display role, Control UI falls through to generic styling, and TUI history replay does not render normal `role: "system"` rows as system notices.

## What This Changes

The issue requests visually distinct rendering for internal runtime/system messages instead of showing them as ordinary user or assistant chat bubbles.

## Best Possible Solution

Define a structural display contract for intentionally visible runtime/system messages, carry that marker through `chat.history` and session message projection, render it with a dedicated Control UI system treatment, and make TUI live/history display use the same system notice path while continuing to suppress runtime-only artifacts that should not be user-visible.

## Reproduction Assessment

Yes for the remaining behavior: a source-level projection/UI replay check shows no dedicated path for `internal_system` provenance or ordinary `role: "system"` history rows. The original pre-compaction memory-flush example is no longer a high-confidence reproduction because current main now leaves `transcriptPrompt` empty for that embedded run.

## Solution Assessment

Unclear as a direct implementation lane: distinct rendering is a reasonable solution for intentionally visible system/runtime messages, but related filtering work in #70348 means maintainers should first decide which artifacts should be hidden and which should be visibly labeled.
