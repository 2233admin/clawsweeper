## Summary

Keep this PR open because current main does not implement the requested `/stop` internal `operator.admin` gate, but the proposed patch is not merge-ready: it still misses the helper import and replaces the existing sender authorization gate with an internal-channel-only helper.

## What This Changes

The PR changes the auto-reply `/stop` handler in `src/auto-reply/reply/commands-session-abort.ts` to require `operator.admin` for internal gateway clients by calling `requireGatewayClientScopeForInternalChannel` instead of `rejectUnauthorizedCommand`.

## Best Possible Solution

If maintainers want this hardening, preserve the existing sender authorization gate, add a correctly imported internal `operator.admin` scope check immediately after it, and add focused tests proving internal `operator.write` is blocked, internal `operator.admin` is allowed, and external unauthorized senders remain blocked.
