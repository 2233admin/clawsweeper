## Summary

Current main still lacks the requested Control UI per-assistant-message print action. Assistant message actions are limited to expand and copy-as-markdown, the copy helper only writes to the clipboard, and targeted searches found no Control UI single-message print view, `window.print` path, or print CSS. This is a narrow, valid UI feature request and is a good Clownfish fix-PR candidate.

## Best Possible Solution

Implement this as a Control UI-only chat feature: add a print button beside the existing assistant message expand/copy actions, render only the selected assistant message through the existing sanitized markdown path into a print-friendly isolated document or print-only view, call the browser print dialog, update action styling for three controls, add focused UI tests, and add a user-facing changelog entry crediting @Hag-Fish.
