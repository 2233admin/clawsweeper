## Summary

Keep open: current main still lacks the requested Control UI per-assistant-message print action. The webchat assistant action bar only renders open-in-canvas and copy-as-markdown controls, and targeted searches found no single-message print view, `window.print`, or Control UI print CSS path.

## What This Changes

Add a Print action beside existing assistant message actions in the Control UI webchat that prints only the selected assistant message with print-friendly styling.

## Best Possible Solution

Implement this as a Control UI-only chat feature: add a print button beside the existing assistant message actions, render only the selected assistant message through the sanitized markdown path into an isolated print document or print-only view, apply simple print CSS for readable tables/code/text, and cover the button/rendering behavior with focused UI tests plus a user-facing changelog entry crediting the reporter.

## Solution Assessment
