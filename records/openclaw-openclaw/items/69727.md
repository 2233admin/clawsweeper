## Summary

Keep #69727 open. Current main does not implement an individual assistant-message Print button in the Control UI webchat. The assistant message action row only renders expand and copy-as-markdown actions, and the existing export path downloads the full chat as Markdown rather than opening a print view for a single assistant message.

## Best Possible Solution

Keep the issue open. The likely implementation belongs in the Control UI webchat message renderer: add a print action beside the existing copy/expand actions in `ui/src/ui/chat/grouped-render.ts`, factor a small print helper near `copy-as-markdown.ts`, add print-friendly isolated HTML/CSS in `ui/src/styles/chat/grouped.css` or a generated print document, and cover it with focused render tests. The existing full-chat Markdown export should remain separate because it does not solve one-click printing for a single assistant reply.
