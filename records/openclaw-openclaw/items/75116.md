## Summary

Keep open. Current main still lacks the requested per-server caller-context injection contract, and open PR #75118 is already paired to this issue as the implementation candidate, so this should stay open until that PR is reviewed, landed, or closed.

## What This Changes

Add an opt-in `mcp.servers.<name>.injectCallerContext` flag for remote bundled MCP servers so OpenClaw can inject caller identity headers without overwriting operator-provided headers.

## Best Possible Solution

Review PR #75118 as the canonical implementation path and, if accepted, land a core bundle-MCP/config-layer change that is default-off, remote-only, non-overwriting, strips the control flag from emitted CLI config, populates placeholders when any merged server opts in, and includes schema, docs, changelog, and focused tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect current main: the flag is not typed or documented, no bundle merge path injects caller headers, and placeholder env values are only passed for the loopback runtime.

## Solution Assessment

Yes, pending maintainer approval of the trust boundary. A per-server default-off core config flag is narrower than a global switch and is the right layer because external plugins cannot reliably modify OpenClaw's generated bundled CLI MCP config without a core seam.
