## Summary

Keep open. Current main does not implement the requested per-server caller-context injection, and open PR #75118 is the active same-author implementation candidate with closing syntax, so the issue should stay open until that PR is reviewed, landed, or closed.

## What This Changes

Request adding an opt-in per-server `mcp.servers.<name>.injectCallerContext` flag that injects OpenClaw caller identity headers into remote bundled MCP server configs without overwriting user-supplied headers.

## Best Possible Solution

Review PR #75118 for the trust boundary and implementation details, then land a shared bundle-MCP/config-layer change if accepted: remote-only, default-off, non-overwriting caller-header merge, stripped downstream control flag, placeholder env population when any merged server opts in, and matching schema/docs/tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a remote `mcp.servers.<name>.url` entry with the proposed flag and inspect the generated bundle MCP config; current main has no explicit schema/type support and no merge path that injects caller headers for remote servers.

## Solution Assessment

Yes, pending maintainer approval of the trust boundary. A per-server default-off flag is the narrow maintainable direction; a global flag would create broader accidental identity leakage risk.
