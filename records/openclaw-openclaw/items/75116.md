## Summary

Keep open. Current main still does not implement a per-server `injectCallerContext` option for remote bundled MCP servers, and the linked open PR #75118 is the active implementation candidate that should be reviewed or closed before this issue is resolved.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` option that forwards OpenClaw caller headers to opted-in remote MCP servers without overwriting user headers or leaking the control flag downstream.

## Best Possible Solution

Review PR #75118 for the trust boundary and implementation details, then land a shared bundle-MCP/config-layer change if accepted: remote-only, default-off, non-overwriting caller-header merge, stripped downstream control flag, placeholder env population when any merged server opts in, and matching schema/docs/tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a remote `mcp.servers.<name>.url` entry with the proposed flag and inspect the generated bundle MCP config: current main has no explicit schema/type support, does not merge caller headers in the bundle path, and only emits `OPENCLAW_MCP_*` placeholders for the loopback runtime.

## Solution Assessment

Yes, pending maintainer approval of the trust boundary. A per-server default-off flag is the narrow maintainable direction; a global flag would create broader accidental identity leakage risk.
