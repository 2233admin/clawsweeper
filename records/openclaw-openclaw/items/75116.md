## Summary

Keep open. Current main still lacks the requested per-server caller-context injection for remote bundled MCP servers, and the linked open PR #75118 is the active implementation candidate that should be reviewed before this issue is resolved.

## What This Changes

Add an opt-in `mcp.servers.<name>.injectCallerContext` setting that forwards OpenClaw caller identity headers to trusted remote bundled MCP servers without overwriting configured headers.

## Best Possible Solution

Review PR #75118 for the trust model and implementation details, then land a shared bundle-MCP/config-layer change if accepted: remote-only, default-off, non-overwriting caller-header merge, downstream flag stripping, placeholder env population when any merged server opts in, and matching schema/docs/tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a remote `mcp.servers.<name>.url` entry with the proposed flag and inspect the generated bundle MCP config; current main tolerates the unknown key but does not inject headers or populate placeholder env for that remote server.

## Solution Assessment

Yes, pending maintainer approval. A per-server default-off flag is the narrowest maintainable direction for trusted internal MCP endpoints; a global flag would create broader accidental identity leakage risk.
