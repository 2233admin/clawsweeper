## Summary

Keep open. Current main still lacks the requested `injectCallerContext` contract and bundle-MCP injection behavior, and the issue is paired with open closing PR #75118, which should be reviewed for the caller-identity trust boundary before this issue is resolved.

## What This Changes

Add an opt-in per-server `mcp.servers.<name>.injectCallerContext` flag that forwards OpenClaw caller identity and session headers to trusted remote bundle MCP servers without overwriting configured headers.

## Best Possible Solution

Review PR #75118 for the trust boundary and implementation details, then land a shared bundle-MCP/config-layer change if accepted: typed schema/help/docs support, remote-only default-off injection, non-overwriting caller-header merge, downstream flag stripping, placeholder env population when any merged server opts in, and focused bundle/config tests.

## Reproduction Assessment

Yes. A source-level reproduction is to configure a remote `mcp.servers.<name>.url` with the proposed flag and inspect the generated bundle MCP config; current main has no explicit config support, no injection/stripping path, and loopback-only placeholder env setup.

## Solution Assessment

Yes, pending maintainer security review. A per-server, default-off, remote-only opt-in is the narrow maintainable direction; a global flag would create broader accidental identity disclosure risk.
