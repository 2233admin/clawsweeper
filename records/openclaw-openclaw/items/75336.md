## Summary

Keep open. The PR targets a real current-main gap in compaction identifier preservation, but the patch is incomplete because the new full-transcript validation is computed after history-share pruning, so identifiers that only existed in dropped chunks are not checked against the returned summary.

## What This Changes

The PR adds compaction identifier extraction/lost-identifier helpers, retries strict summaries when more than two transcript identifiers are lost, expands tests, and adds an active-version changelog entry.

## Best Possible Solution

Amend the PR so identifier source collection covers the actual pre-compaction transcript before history-share pruning, or separately validates droppedSummary and the main summary, while still excluding toolResult.details and preserving the existing strict/off policy boundaries.

## Reproduction Assessment

Yes. A compaction event that triggers maxHistoryShare pruning and has more than two opaque identifiers only in pruned.droppedMessagesList would bypass the new lost-identifier retry because the PR builds transcriptIdentifiers from the kept messages after pruning.

## Solution Assessment

No. The direction is good, but the current implementation is not the best complete fix because it claims full pre-compaction transcript validation while checking only the post-prune subset; the safer path is to snapshot/validate identifiers across the dropped and kept summarization paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Include pruned history in identifier validation:** `src/agents/pi-hooks/compaction-safeguard.ts:1038-1041`
  - body: The new `transcriptIdentifiers` set is built after `pruneHistoryForContextShare` can replace `messagesToSummarize` with `pruned.messages`, while `pruned.droppedMessagesList` is summarized separately into `droppedSummary`. If identifiers only appear in dropped chunks, losing them from the returned summary will not count toward `lostIdentifiers`, so strict mode can skip the intended retry despite the full pre-compaction transcript contract. Capture identifiers before pruning or validate the dropped-summary path too.
