## Summary

Current main still does not implement this PR's intended Slack routing change. Ordinary top-level Slack channel messages under replyToMode="all" still resolve to the shared channel session, while MessageThreadId can still point at the message timestamp for threaded reply delivery. The paired bug report remains open, and this PR is still an implementation candidate rather than cleanup-ready obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept the linked dual-session bug, rebase the fix into extensions/slack/src/monitor/message-handler/prepare-routing.ts, preserve seeded-thread and runtime-binding behavior, and update the current tests that still assert shared top-level channel sessions under replyToMode="all". If maintainers intentionally want the current shared-session behavior, close this PR and #64078 with that product rationale.
