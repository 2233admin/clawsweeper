## Summary

Current main still does not implement this PR's intended Slack routing change. Ordinary top-level Slack channel messages under replyToMode="all" still route to the shared channel session unless a seeded-room-thread path applies, while reply delivery metadata can still target the message timestamp. The linked bug report remains open, so this PR is still a live implementation candidate rather than cleanup-ready obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept the linked dual-session bug, rebase the change into extensions/slack/src/monitor/message-handler/prepare-routing.ts, update the current thread-session tests that still assert shared channel sessions for replyToMode="all", preserve seeded top-level mention/runtime-binding behavior, and close #64078 through the merged fix. If maintainers intentionally want ordinary top-level channel posts to share one session under replyToMode="all", close this PR and #64078 with that product rationale.
