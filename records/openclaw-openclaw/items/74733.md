## Summary

Keep open. This is a real WebChat/Gateway ordering and reset fix, but the latest draft head still leaves the new synchronous `/new` and `/reset` path clearing visible chat state without clearing tool-stream state, so stale tool cards or stream segments can survive if the follow-up reload/event is delayed or fails.

## What This Changes

The PR updates Control UI WebChat and Gateway chat history/session projection, related tests, and QA timeouts to stabilize history ordering, reset handling, stream/tool rendering, and terminal error display.

## Best Possible Solution

Keep this PR as the implementation candidate, add the same `resetToolStream` cleanup to `clearChatHistory` before the synchronous reset path reloads history/sessions, add focused coverage for reset-after-tool-output when history is delayed or fails, then rerun the focused WebChat/Gateway checks.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a bare WebChat `/new` or `/reset` after tool/stream output while `chat.history` or `sessions.changed` delivery is delayed or fails: `clearChatHistory` clears `chatMessages`, but `buildChatItems` still renders `chatStreamSegments` and `chatToolMessages`.

## Solution Assessment

No, not as-is. The PR is scoped to the right WebChat/Gateway surfaces, but the synchronous reset path should reuse the existing `resetToolStream` helper already used by PR-head Gateway reset handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Clear tool stream state during synchronous resets:** `ui/src/ui/app-chat.ts:649-655`
  - body: The new bare `/new` and `/reset` path clears visible chat state after `sessions.reset`, but leaves `chatStreamSegments`, `chatToolMessages`, `toolStreamById`, `toolStreamOrder`, and the sync timer untouched until history succeeds or a later gateway event arrives. Those artifacts render independently of `chatMessages`, so old tool/stream cards can remain visible after a reset when history or event delivery is delayed or fails. Call `resetToolStream` in this path before reloading history.
  - confidence: 0.86
