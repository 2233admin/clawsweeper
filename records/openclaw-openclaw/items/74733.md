## Summary

Keep this PR open. The WebChat/Gateway ordering problem is real on current main and the PR mostly targets the right boundary, but review found a remaining stale-history edge where repeated same-text optimistic sends can still be dropped, so it needs author or maintainer follow-up before merge.

## What This Changes

The PR updates Control UI WebChat and Gateway chat history projection/session metadata to stabilize history reconciliation, transcript timestamps, stream/tool rendering, deferred finals, and visible error handling, with focused tests.

## Best Possible Solution

Keep the fix in the Control UI reconciliation/rendering and Gateway history projection layers, but make optimistic-message reconciliation use stable transcript/local identity or explicitly ignore locally optimistic messages as shared anchors before this PR lands.

## Reproduction Assessment

Yes. Current main can be checked by starting a `loadChatHistory` request, sending locally before the older history resolves, and observing the request-start snapshot overwrite local state; the PR finding reproduces with the same flow when the new optimistic user message has the same visible text as an earlier persisted user message.

## Solution Assessment

No, not yet. The overall UI/Gateway boundary is right, but the proposed stale-history merge still relies on role+visible-text/content signatures where stable local or transcript identity is needed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve repeated optimistic sends during stale history:** `ui/src/ui/controllers/chat.ts:615-618`
  - body: When `localStateChanged` is true, this feeds the current local messages into `preserveOptimisticTailMessages`, whose shared-anchor search keys messages only by role plus visible text/content. If the user sends the same text as an earlier persisted message while an older `chat.history` response is in flight, the optimistic send matches that old history entry, no tail is appended, and the new send disappears. Give optimistic sends a stable local identity or skip locally optimistic messages when choosing the shared anchor.
  - confidence: 0.86
