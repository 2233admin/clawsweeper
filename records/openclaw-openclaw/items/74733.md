The PR changes Control UI WebChat, Gateway chat display projection, session transcript timestamp handling, related tests, QA scenario timeouts, and one gateway CLI test mock to stabilize message ordering, resets, stream/tool rendering, and terminal error display.

## Best Possible Solution

Yes. The source-level path is reset after a tool/streamed response while chat.history is delayed or fails: clearChatHistory clears only chatMessages and related scalar state, while buildChatItems still renders stream segments and tool messages independently.
