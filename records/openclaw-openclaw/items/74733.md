## Summary

Keep open. Current main still has the WebChat history/stream ordering behavior this PR is trying to fix, but the PR head still has two blocking UI correctness regressions: double trimming of live stream text and text-only dedupe of deferred final messages.

## What This Changes

The PR changes Gateway chat-history/session metadata and Control UI chat reconciliation/rendering to stabilize history reloads, stream/tool ordering, stable message keys, deferred finals, and visible error projection.

## Best Possible Solution

Revise the PR so live stream text has one clear representation contract across controller state, final fallback, and render-time chat item building, and preserve distinct deferred finals using stable run/message metadata or by avoiding text-only dedupe for that queue; then land after targeted WebChat and Gateway validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep stream-prefix trimming at one boundary:** `ui/src/ui/controllers/chat.ts:849`
  - body: This stores `state.chatStream` as an already-trimmed suffix, but `streamedAssistantText` and `buildChatItems` trim `chatStream` against the same `chatStreamSegments` again. If the committed segment is `A` and the next real suffix also starts with `A`, the live bubble and fallback final text drop that leading text. Keep `chatStream` cumulative until render/final time, or make all downstream consumers treat it as suffix-only.
  - confidence: 0.9
- **[P2] Preserve distinct deferred final messages:** `ui/src/ui/controllers/chat.ts:312`
  - body: Deferred finals are queued through `appendUniqueDisplayMessages`, whose signature ignores run/message ids and dedupes by role plus rendered text/content. Two unrelated background runs that both finish with text like `Done` collapse before flush, so one visible result is lost. Preserve run/message identity when available, or avoid text-only dedupe for deferred finals.
  - confidence: 0.86
