## Summary

Keep this PR open. Current main already has the WebChat agent-event plumbing and some nearby coverage, but it does not contain the two focused regression cases this PR adds for implicit `agent` RPC run-context registration and internal assistant events reaching WebChat observers on both main-session and direct-session paths. The linked bug #46106 is closed, but this PR is explicitly test-only regression coverage for that path rather than a runtime fix, so it is not obsolete under the conservative PR cleanup rules.

## Best Possible Solution

Keep the PR open for maintainer review. If the regression coverage is still desired after #46106's runtime fix, rebase and land these tests or add equivalent current-main coverage that proves internal `agent` runs created by the RPC path deliver final WebChat chat events for both main and direct WebChat session keys.
