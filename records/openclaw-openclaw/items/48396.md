## Summary

Keep open. Current main still does not implement last-message previews in the Control UI Sessions list, and this PR remains an implementation candidate rather than something that can be closed as shipped. The submitted patch also needs revision because the review context shows it used `subject` instead of the backend `lastMessagePreview` field and included a scratch planning document.

## Best Possible Solution

Keep the PR open only if it is revised into an end-to-end Control UI implementation: remove the scratch planning document, add `lastMessagePreview` to the UI session row type, request preview data in a bounded way, render normalized timestamp-plus-preview text from `row.lastMessagePreview`, and add focused controller/view tests. Coordinate with #47120 and #47243 so only one corrected implementation lands.
