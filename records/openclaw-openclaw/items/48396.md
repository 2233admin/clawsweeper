## Summary

Keep open. Current main still does not implement last-message previews in the Control UI Sessions list, and this PR is not obsolete. It is paired with the same-author feature issue #47120 and overlaps open PR #47243, but neither has landed. The submitted patch still needs revision because it renders `subject` metadata instead of the backend `lastMessagePreview` field and includes an unrelated scratch planning document.

## Best Possible Solution

Keep the PR open only if it is revised into an end-to-end Control UI implementation: remove the scratch planning document, add `lastMessagePreview` to the UI row type, request preview data in a bounded way, render normalized timestamp-plus-preview text from `row.lastMessagePreview`, and add focused controller/view coverage. Coordinate with #47120 and #47243 so only one corrected implementation lands.
