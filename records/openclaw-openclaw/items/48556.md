## Summary

Keep open. Current main still does not implement this PR's intended core proxy-fetch behavior: both `makeProxyFetch()` and `resolveProxyFetchFromEnv()` still forward caller bodies directly to `undici.fetch` while only adding a dispatcher. The separate runtime FormData normalizer covers `fetchWithRuntimeDispatcher()`, but the affected explicit proxy fetch path bypasses it. The linked same-author bug #48554 remains open, and overlapping PR #66243 is also open, so this PR is still a live implementation candidate rather than cleanup-closeable.

## Best Possible Solution

Keep this PR open for maintainer review, or explicitly supersede it with a current-main patch coordinated with #48554 and #66243. The best implementation belongs in `src/infra/net/proxy-fetch.ts`: normalize global FormData for both `makeProxyFetch()` and `resolveProxyFetchFromEnv()`, preferably by sharing the runtime normalizer behavior, clear stale multipart framing headers, and add targeted proxy-wrapper regression tests for conversion, passthrough, and non-FormData bodies.
