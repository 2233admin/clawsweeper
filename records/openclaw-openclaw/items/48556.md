## Summary

Keep this PR open. Current main still does not implement the PR's intended proxy-fetch FormData normalization: both proxy fetch wrappers pass `init.body` directly into `undici.fetch`, while the existing FormData compatibility shim only runs through the separate runtime dispatcher path. The linked same-author bug #48554 is open, and related PR #66243 is also open with a broader proxy input normalization approach, so this should stay in maintainer review rather than be closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer review, or explicitly supersede it only after a merged current-main patch coordinated with #48554 and #66243. The best fix belongs in `src/infra/net/proxy-fetch.ts`: normalize global/FormData-like bodies before both `makeProxyFetch()` and `resolveProxyFetchFromEnv()` dispatch through Undici, preserve filenames deliberately, clear stale multipart framing headers when conversion happens, and add focused proxy-wrapper tests. Then close the linked bug and any superseded PRs against the merged commit.
