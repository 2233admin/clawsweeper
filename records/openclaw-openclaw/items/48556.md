## Summary

Keep open. Current main has a separate dispatcher-runtime FormData normalization path, but it does not implement this PR’s intended change in the public proxy fetch wrappers. `makeProxyFetch()` and `resolveProxyFetchFromEnv()` still pass `init.body` straight through to `undiciFetch`, and the linked reporter issue remains the paired open tracking item for the same work.

## Best Possible Solution

Keep this PR open for maintainer review or replacement by a current-main patch. The best path is to rebase the fix onto current main, reuse or share the existing runtime FormData normalization where practical, apply it to both `makeProxyFetch()` and `resolveProxyFetchFromEnv()`, add targeted proxy-wrapper tests for global FormData conversion and header handling, and coordinate with the still-open linked issue and overlapping proxy-fetch PR #66243.
