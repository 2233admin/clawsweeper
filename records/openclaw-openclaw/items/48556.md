## Summary

Keep open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still does not implement this PR's intended change in the proxy fetch wrappers: both `makeProxyFetch()` and `resolveProxyFetchFromEnv()` forward the caller's `init.body` directly to `undici.fetch` while only replacing the dispatcher. Main does have FormData normalization in `runtime-fetch`, but `proxy-fetch.ts` does not use it, and the proxy helper is still exported as plugin SDK surface. The media audio path still builds global `FormData` and passes a proxy fetch into the provider path when proxy env vars are set. The linked same-author issue #48554 remains open, and overlapping #66243 is also open, so this PR remains a live implementation candidate or should be replaced by a merged current-main patch rather than closed.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it with a current-main patch. The best implementation is to normalize FormData in both `makeProxyFetch()` and `resolveProxyFetchFromEnv()`, preferably by sharing or aligning with the existing runtime FormData normalizer, clearing stale multipart headers, adding targeted proxy-wrapper tests for global FormData conversion and Undici FormData passthrough, and coordinating the result with #48554 and #66243.
