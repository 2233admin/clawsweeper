## Summary

Keep open. Current main still relays failed or undelivered hook agent runs as untrusted `Hook ...` system events into the configured main session, and the hook config surface still has no suppression or audience-control option for this relay. Existing tests assert that behavior, so this is not already fixed.

## Best Possible Solution

Keep the issue open and fix it in core hook/session-event handling. A conservative path is to add an explicit hook delivery relay policy or typed system-event audience, stop routing hook delivery artifacts/errors into user-facing agent sessions by default, and cover `/hooks/agent` plus mapped hook non-delivery/error cases with regression tests. Coordinate with the broader #24693 and #69492 system-event leakage work rather than treating prompt instructions as the fix.
