## Risks / Open Questions

- The PR's helper handles comma lists by checking only literal `0`, so mixed valid Croner list elements such as `0-5,30` or `*/15,45` would still miss the stagger path.
- The PR includes `0/15` as a positive minute-step case, but Croner 10.0.1 rejects numeric-prefix stepping by default unless `sloppyRanges` is enabled; OpenClaw does not enable that option in `src/cron/schedule.ts`.
- The helper remains a lightweight classifier separate from Croner's parser, so the final fix should be explicit about matching Croner-supported syntax and avoiding false positives for invalid expressions.
