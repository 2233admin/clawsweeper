## Summary

Keep this PR open. It targets a valid onboarding skill dependency bug, and current main still lacks the requested cask/npm changes, but the proposed patch is not correct yet because the new boolean `cask` manifest field is never parsed into the install spec before command generation.

## What This Changes

This PR adds a `cask` install metadata flag for Homebrew-backed skills, uses it for CodexBar command/status metadata, and changes the bundled summarize skill installer from `steipete/tap/summarize` to `@steipete/summarize` via the node installer.

## Best Possible Solution

Revise this PR so `cask` is parsed and normalized end to end, include focused regression coverage for manifest parsing plus installer/status behavior, explicitly settle the summarize package-source choice, and then land the reviewed fix for the linked onboarding bug.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Parse the cask flag before using it:** `skills/model-usage/SKILL.md:17`
  - body: The new manifest value is `"cask": true`, but `parseInstallSpec()` currently treats `raw.cask` only as an alternate string formula and never copies a boolean onto `spec`. As a result the CodexBar spec reaches `buildInstallCommand()` without `spec.cask`, so it still runs `brew install steipete/tap/codexbar` and the linked install failure remains.
  - confidence: 0.97
