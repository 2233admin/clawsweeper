## Summary

Keep this PR open. The linked onboard failure is valid and current main has not fixed it, but the proposed patch is incomplete because the new cask metadata is not parsed end to end and the summarize installer change needs a supported-runtime path before merge.

## What This Changes

The PR adds brew-cask metadata to skill install specs and installer/status handling, marks the bundled model-usage CodexBar dependency as a cask, and switches the bundled summarize installer from a Homebrew tap to the @steipete/summarize npm package.

## Best Possible Solution

Revise the PR so brew casks are parsed and normalized from SKILL.md through install command generation, status display, scan/hook contracts, and focused tests; use a summarize install source that works for OpenClaw-supported runtimes, then add the required changelog entry.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: add `"cask": true` to a SKILL.md install spec, parse it through the current frontmatter path, and the resulting install spec still lacks `cask`, so the command builder cannot emit `brew install --cask`.

## Solution Assessment

No. The CodexBar cask direction is the right fix, but this patch applies it after the parser boundary and changes summarize to an npm-only path that can be incompatible with OpenClaw's supported Node 22 runtime; a parsed cask flag plus Homebrew-core or engine-aware summarize handling is safer.
