## Summary

Keep open. Current main still builds a memory-flush plan with exactly one write target, threads only that target into embedded memory runs, and rejects `SESSION_HANDOFF.md` or any other path in the memory-triggered write wrapper. The bundled default prompt now points only at the daily memory file, so the remaining bug is the configurable-prompt/additional-write-target case rather than a default-prompt mismatch; prior PR #66765 is closed and unmerged, and related #63664, #68995, and #74198 do not fully supersede this specific request.

## What This Changes

Allow pre-compaction memory flush writes to `SESSION_HANDOFF.md` or other explicitly configured safe paths instead of only `memory/YYYY-MM-DD.md`.

## Best Possible Solution

Add a deliberate memory-flush write-target contract: expose explicit validated additional targets in config or the memory flush plan, preserve append-only behavior for the daily memory file, allow only intended overwrite targets such as `SESSION_HANDOFF.md`, reject unrelated paths and escapes, update docs/schema metadata, and add regression tests for allowed extras, daily append behavior, unrelated-path rejection, and traversal attempts.
