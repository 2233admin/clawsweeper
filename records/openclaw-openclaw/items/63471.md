## Summary

Keep this issue open. Current main has adjacent Docker ergonomics improvements, but it still does not provide the requested Dockge-style paste a Compose file plus .env and start workflow: the bundled Compose service bypasses the Dockerfile CMD and starts `gateway` without `--allow-unconfigured`, while the gateway startup guard still exits on a fresh missing config unless `gateway.mode=local` already exists or the flag is passed.

## Best Possible Solution

Keep this open and design a first-class Docker Compose/Dockge bootstrap path. The best fix should let the bundled Compose template start from a fresh persisted config directory, ensure a durable gateway token is generated or required and discoverable, seed safe Control UI origins for the published port, preserve non-loopback auth/origin guardrails, and update Docker plus ClawDock docs and targeted tests.
