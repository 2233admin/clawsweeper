## Summary

Current main has made Docker Compose friendlier in adjacent ways, including default config/workspace mount paths and a scripted skip-onboarding path, but it still does not implement the reported Dockge-style paste a Compose file plus .env and start workflow. The bundled Compose service overrides the Dockerfile CMD and still starts `gateway` without `--allow-unconfigured`; the gateway startup guard still exits on missing config unless `gateway.mode=local` is already present or that flag is passed. Docker docs and ClawDock still describe setup/helper/manual flows rather than a first-class paste-and-start Compose bootstrap.

## Best Possible Solution

Keep this open and implement a first-class Dockge/Compose bootstrap path. A good fix would let the bundled Compose template initialize a minimal persisted local gateway config, generate or require a durable gateway token, seed safe Control UI origins for the published port, keep workable config/workspace persistence defaults, and preserve the existing non-loopback auth and origin guardrails. Update the Docker and ClawDock docs once that path exists.
