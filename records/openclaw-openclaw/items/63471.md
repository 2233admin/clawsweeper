## Summary

Keep this open. Current main still improves adjacent Docker setup paths but does not implement the requested Dockge-style one-paste Compose plus .env startup: the bundled Compose command bypasses the image CMD, the gateway still exits on a fresh missing config, and docs still require setup-script or manual pre-start config writes.

## What This Changes

Request a Docker/Dockge-friendly Compose path where a pasted compose YAML plus .env can start OpenClaw without extra pre-start CLI setup.

## Best Possible Solution

Add a first-class Docker Compose bootstrap path for Dockge-style installs that idempotently creates a minimal persisted local gateway config, provisions or requires durable auth, seeds safe Control UI origins for the published local port, preserves the existing non-loopback auth/origin guardrails, and documents the flow on the Docker and ClawDock pages.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: a fresh Compose run with no mounted openclaw.json uses the Compose-defined gateway command without --allow-unconfigured, reaches getGatewayStartGuardErrors, and exits with the reported missing-config message before server startup.

## Solution Assessment

No complete solution is present yet. The setup script path is useful for scripted installs, but it still requires pre-start CLI work; simply adding --allow-unconfigured to Compose would not be the narrowest maintainable fix because auth and Control UI origin policy still need a durable bootstrap story.
