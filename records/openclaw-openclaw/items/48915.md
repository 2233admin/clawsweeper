## Summary

Keep open. Current main still recognizes only camelCase tool-call block types in the session transcript repair path, so malformed Anthropic-style snake_case blocks can bypass the missing-input sanitizer. The issue is also paired with open, unmerged PR #50802, which should be reviewed or replaced before this issue is closed.

## Best Possible Solution

Review and either land PR #50802 or replace it with a focused maintainer patch. The fix should consistently recognize `tool_use`, `tool_call`, and `function_call` in session repair and adjacent tool-call replay helpers, then add targeted regression tests for malformed snake_case blocks missing `input` or `arguments`. Because this is user-visible recovery behavior for Anthropic/Feishu sessions, include a changelog fix entry when the patch lands.
