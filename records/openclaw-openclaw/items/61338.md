## Summary

Keep this issue open. Current main still treats skill OS requirements as raw platform strings, so `metadata.openclaw.os: ["macos"]` will not match macOS where Node reports `process.platform === "darwin"`. The existing tests and docs cover canonical `darwin` spelling, not the reported `macos` alias, and the provided related items do not supersede this specific bug.

## Best Possible Solution

Keep this issue open and fix the skills requirement platform contract in core. Add a small canonical platform normalizer shared by skill metadata OS requirements, remote platform lists, and install-spec OS filters, covering at least `macos` -> `darwin` and likely `windows` -> `win32`; then add regression tests for `metadata.openclaw.os: ["macos"]` on Darwin and update the public skills docs to state accepted aliases or validation behavior.
