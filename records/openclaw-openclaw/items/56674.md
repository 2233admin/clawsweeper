## Summary

Current main does not implement this PR's intended OpenResponses reasoning output. The schema accepts reasoning items, but `/v1/responses` still only returns assistant message/function-call output, the streaming event union has no `response.reasoning.*` events, and the public docs still say `reasoning` is currently ignored. This PR remains a plausible implementation candidate rather than an obsolete PR to close.

## Best Possible Solution

Keep this PR open for normal maintainer review or ask the author to refresh it against current main. A good landing path would preserve the intended OpenResponses reasoning output/events, keep raw reasoning internal to the HTTP capture path, add/keep regression coverage for non-stream, stream, tool-call, duplicate-done, and shared-prefix reasoning cases, and update the public OpenResponses docs from `reasoning` ignored to the shipped behavior if merged.
