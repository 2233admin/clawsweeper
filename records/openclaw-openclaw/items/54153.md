## Summary

Keep open. Current main has partially hardened the Signal installer by moving the archive download to `fetchWithSsrFGuard` and adding a streamed 256 MiB cap, but the Signal release-info and archive fetches still omit an explicit timeout, focused downloader regression tests are still missing, and open PR #73489 is already the linked implementation candidate.

## What This Changes

Harden the bundled Signal plugin's `signal-cli` release downloader against hangs, redirect SSRF, oversized downloads, and proxy-bypass behavior.

## Best Possible Solution

Review and land #73489, or an equivalent Signal plugin fix, that completes the owner-module hardening with explicit timeout budgets, declared and streamed size checks, conservative proxy-mode handling, partial-download cleanup, safe archive path handling, and focused Signal installer tests before closing this issue.
