## Summary

Current main still lacks the intended Slack duplicate-reaction fix. `reactSlackMessage()` directly forwards `client.reactions.add()` errors, and the agent `react` tool only returns success after that helper resolves. Maintainer discussion requested moving `already_reacted` handling into the shared helper; the provided PR branch now appears scoped to that path with regression coverage, so this PR remains useful and should not be closed.

## Best Possible Solution

Keep this PR open for maintainer review and land it, or land an equivalent Slack plugin patch that treats Slack `already_reacted` from `reactSlackMessage()` as idempotent success, keeps unrelated Slack errors propagating, preserves focused helper-level regression coverage, and then closes linked issue #69005 while reconciling the older overlapping Slack reaction PRs.
