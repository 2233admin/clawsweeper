## Summary

Keep this PR open, but it is not merge-ready. Current main still lacks an Android talk-mode language override, so the contribution is not obsolete; however, the latest diff introduces a user-visible offline status regression when changing the setting without a cached gateway endpoint.

## What This Changes

The PR adds an Android talk-mode language override setting, persists it in preferences, and threads the selected locale into recognizer intents, local system TTS, gateway connect metadata, device.info locale reporting, Settings UI, and DeviceHandler test setup.

## Best Possible Solution

Land an Android-owned language override only after rebasing onto current main, preserving auto/system-language behavior by default, avoiding any reconnect/status change when no cached endpoint exists, and adding focused Android tests plus Talk docs for the supported override semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Skip reconnect when no cached gateway exists:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:637`
  - body: `setLanguageOverride()` calls `refreshGatewayConnection()` unconditionally. When no gateway endpoint has been cached yet, or after the user disconnects, that path reports `Failed: no cached gateway endpoint`, so changing a local language setting can surface a false connection failure even though the preference update succeeded.
  - confidence: 0.87
