## Summary

Keep this PR open. Current main still lacks an Android Talk language override, and the PR is useful feature work, but the proposed patch has a user-visible offline status regression and needs rebase/adaptation plus Android owner review before merge.

## What This Changes

The PR adds a persisted Android language override setting/dropdown and threads the selected locale into voice recognition, system TTS, gateway connect metadata, device info, and related Android tests.

## Best Possible Solution

Land an Android-owned language override that keeps `auto` as the default, applies a selected BCP 47 locale through the current recognizer/TTS/gateway/device-info paths, updates desired reconnect options without false offline failures, and documents the Android Talk setting with focused locale propagation tests.

## Reproduction Assessment

Yes. The PR-branch regression is reproducible by inspection: selecting a different language calls `setLanguageOverride()`, which calls `refreshGatewayConnection()`; with no `connectedEndpoint`, current `NodeRuntime` sets status to `Failed: no cached gateway endpoint`.

## Solution Assessment

No. The feature direction is reasonable, but this patch should not force the failed refresh path while offline; it should persist the desired locale and reconnect only when there is a cached or active endpoint, then be rebased onto current Android Talk code.

## Review Findings
