## Summary

Keep this PR open. Current main still does not implement the proposed public-facing agent mode, /v1 owner-trust header, prompt/runtime redaction, or skills suppression. This is core gateway and agent trust-boundary work, not cleanup that can be closed as obsolete.

## Best Possible Solution

Keep the PR open for maintainer review and contributor fixes. The right path is a core implementation that adds a documented `publicMode` config surface, redacts runtime/workspace/skills prompt details for public-facing agents, derives mode from the effective routed session agent, handles defaults even without an explicit agent entry, and keeps trusted owner-header behavior identical for streaming and non-streaming `/v1` requests.
