## Summary

Keep this PR open. Current main does not implement the proposed publicMode config or the /v1 owner-trust header behavior for OpenAI/OpenResponses ingress, so the PR is not obsolete or implemented-on-main. The contribution is also not a ClawHub-style optional plugin: it changes core agent prompt redaction, config schema, and gateway HTTP trust semantics. The supplied review discussion shows useful author follow-up, but later review comments still identify security/behavior gaps that should be resolved before maintainers consider landing it.

## Best Possible Solution

Keep the PR open and ask for review/fixes on the remaining branch issues rather than closing it. The right product path is a core gateway/agent implementation that redacts local runtime details for configured public-facing agents, derives publicMode from the effective session agent, applies `agents.defaults.publicMode` even without a listed agent entry, and only honors owner-restoring HTTP headers from trusted upstreams with streaming and non-streaming behavior kept identical.

## Risks / Open Questions

- Security review: this PR changes a gateway trust boundary and prompt redaction behavior. The later supplied review comments on commit `da944115659c545db83e85862bce48d03203657d` identify remaining issues: defaults-only `agents.defaults.publicMode` may not apply when no agent entry exists, and streamed `/v1/responses` may not pass `trustedProxies` consistently. It should receive maintainer security/product review plus targeted tests for default-only agents, explicit false overrides, routed session keys, trusted/untrusted owner headers, and streaming/non-streaming parity.
