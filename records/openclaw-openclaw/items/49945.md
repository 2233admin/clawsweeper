## Summary

Keep this PR open. Current main still has TUI `/new` switching to a fresh isolated `tui-*` session with `setSession(...)` only, so the PR's central fix is not implemented on main. The linked regression remains open, and this PR is the active implementation candidate, though it still needs maintainer review for hook attribution, blank-session persistence, the additive Gateway protocol field, and changelog hygiene.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate for the linked TUI hook regression. The best fix should make TUI `/new` emit hook-visible `command:new` behavior while preserving per-client `tui-*` isolation, keep hook attribution correct for `event.sessionKey` and `previousSessionEntry`, avoid cluttering session history with empty sessions, and either document/approve the Gateway protocol addition or solve the context problem without a new protocol field. Add or adjust regression tests for those semantics and fix the changelog attribution before merge.
