## Summary

Keep this PR open for maintainer review. Current main does not have `bootstrapTier`, and the direction is useful for the linked local-model prompt-pressure report, but the patch changes delegated/background bootstrap behavior and does not carry the tier through the bundled extra-bootstrap hook path.

## What This Changes

The PR adds `agents.defaults.bootstrapTier`, minimal bootstrap filtering, a loopback local-model doctor warning, generated config metadata, docs, changelog text, and focused tests.

## Best Possible Solution

Land the feature only after the tier semantics are session-aware and hook-aware: keep `standard` as the default, make `minimal` reduce ordinary bootstrap predictably, preserve or explicitly document delegated/background session behavior, and update docs/tests around the bundled extra-bootstrap hook.

## Reproduction Assessment

Yes. Static inspection of PR head is enough: `filterBootstrapFilesForSession(mockFiles, "agent:default:subagent:task-1", "minimal")` returns the new minimal allowlist with `BOOTSTRAP.md` and without `SOUL.md`, and the bundled hook still re-filters extras without the tier.

## Solution Assessment

No. The proposed config and doctor warning are plausible, but the patch is not the narrowest maintainable solution until minimal filtering is threaded through all bootstrap mutation paths and does not unintentionally change subagent/cron context.
