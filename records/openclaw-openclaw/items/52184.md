## Summary

Keep this issue open. Current main still has Homebrew-only stable Node normalization, so a supported Volta tools/image executable can still pass through unchanged and be written into the macOS LaunchAgent ProgramArguments. The issue is also paired with open implementation PR #52252 using closing syntax, so the maintainer path is to review/land or close that PR rather than closing this issue early.

## Best Possible Solution

Keep this issue open and resolve it through PR #52252 or an equivalent focused change: teach `resolveStableNodePath` to detect Volta `tools/image/node/<version>/bin/node` paths, prefer an accessible stable Volta shim from `VOLTA_HOME` or the path-derived Volta home, fall back to the original executable when unavailable, and add targeted resolver plus gateway install argument coverage.
