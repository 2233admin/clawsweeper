## Summary

Keep this issue open. Current main still only normalizes Homebrew Cellar Node paths; a supported Volta tools/image Node executable would pass through unchanged and then be written as the macOS LaunchAgent runtime. There are also open implementation candidates with closing syntax, so the maintainer path is to review/land or close those PRs rather than closing this issue early.

## Best Possible Solution

Keep this issue open and resolve it through PR #52252 or an equivalent focused change: teach `resolveStableNodePath` to detect Volta tools/image Node paths, prefer an accessible stable Volta shim path from `VOLTA_HOME` or the user's Volta home, fall back to the original executable when unavailable, and add targeted resolver plus gateway install argument tests.
