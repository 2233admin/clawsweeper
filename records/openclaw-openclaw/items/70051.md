## Summary

Keep this draft PR open. Current main still replays node exec lifecycle events through the system-event queue and requests an `exec-event` heartbeat wake without checking `heartbeatIsolatedBaseSessionKey`, so the PR is not implemented or superseded. The diff is narrow and does not show security- or supply-chain-sensitive changes, but the behavior should get maintainer review before merge.

## What This Changes

The PR changes `src/gateway/server-node-events.ts` to suppress `exec.started`, `exec.finished`, and `exec.denied` system-event enqueueing and heartbeat wakes for synthetic heartbeat-isolated sessions, with a focused regression test in `src/gateway/server-node-events.test.ts`.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt the narrow `server-node-events` guard plus regression test after confirming synthetic heartbeat-isolated sessions should not replay their own exec lifecycle events, while preserving normal node exec notifications for non-heartbeat sessions.
