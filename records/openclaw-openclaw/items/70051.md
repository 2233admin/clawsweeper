## Summary

Keep this draft PR open. Current main still handles node `exec.started`, `exec.finished`, and `exec.denied` by canonicalizing the session key, enqueueing a system event, and requesting an `exec-event` heartbeat wake without checking `heartbeatIsolatedBaseSessionKey`. The PR is a narrow, plausible fix for a remaining heartbeat-isolated self-wake gap, with no security-sensitive or supply-chain changes visible in the provided diff.
