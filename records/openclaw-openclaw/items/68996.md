## Summary

Keep this PR open, but do not merge it as-is. The underlying Gemma routing problem is real and current main still lacks the native-api override, but the proposed patch introduces two provider-routing regressions around Gemini CLI template base URLs and explicit configs where Gemma omits a per-model API.

## What This Changes

The PR changes the Google provider’s Gemma forward-compat resolver and explicit model normalization so Gemma models use `google-generative-ai`, with a regression test for explicit Gemma config.

## Best Possible Solution

Revise the PR so Gemma-native routing changes `api` and a compatible Generative Language base URL together, or scopes the override to native Google templates only. The config-normalization path should switch Gemma models and base URLs as one operation and include regression coverage for Google AI Studio, Gemini CLI, omitted per-model API, and mixed sibling OpenAI-compatible models.

## Reproduction Assessment

Yes. The user-facing bug is supported by the PR body and related issue reports, and the PR regressions are reproducible from the current unit-test harness by resolving `google-gemini-cli/gemma-*` with a CLI template and by normalizing a Google provider config containing Gemma with no per-model `api`.

## Solution Assessment

No. The direction is right, but the current patch is not the narrowest maintainable fix because it changes transport API without guaranteeing a compatible base URL and rewrites provider baseUrl before all affected model APIs are actually switched.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Patch the Gemma base URL with the native API:** `extensions/google/provider-models.ts:177`
  - body: The PR forces `api: "google-generative-ai"` for every Gemma clone. For `google-gemini-cli`, template selection tries CLI templates first, and those templates keep the Cloud Code base URL; the native stream URL builder then appends `:streamGenerateContent` to that incompatible host. Scope this override to native Google templates or patch `baseUrl` together with `api`.
  - confidence: 0.91
- **[P1] Only normalize Gemma URLs after switching the model API:** `extensions/google/provider-policy.ts:111-119`
  - body: The new explicit-Gemma trigger can rewrite a Google OpenAI-compatible base URL to native `/v1beta` even when the Gemma model omits `api`. Since the model normalizer only switches models whose `api` is already `openai-completions`, config defaults can later fill the omitted model API from the provider-level OpenAI API and send OpenAI requests to the native path, also affecting sibling models. Gate the baseUrl rewrite on actually switching the Gemma model API, or set both together.
  - confidence: 0.88
