## Summary

Keep open. Current main still hard-denies local assistant attachments when `localMediaPreviewRoots` is empty, before the server metadata endpoint can decide. The PR remains a focused fix for the paired bug #67915; related PR #72612 is separate upload-state work and explicitly does not supersede this attachment-preview fix.

## What This Changes

This PR changes Control UI attachment availability so empty bootstrap roots fall through to server metadata, adds no-store bootstrap config caching on the client and server, and adds focused UI/gateway regression coverage.
