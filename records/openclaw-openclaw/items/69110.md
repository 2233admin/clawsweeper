## Summary

Current main still does not pass resolved runtime provider/model identity into the awaited outbound `message_sending` hook or its typed event/context. The agent runner knows `providerUsed` and `modelUsed` after fallback and uses them for usage, diagnostics, trace, and fallback notices, but final delivery hooks still only receive delivery-level fields. Related PRs such as #65149 and #39761 are open implementation candidates, not merged fixes.

## Best Possible Solution

Keep this issue open and coordinate one maintainer-approved provenance design. The best path is to carry resolved provider/model identity from the agent run or fallback result into final reply assembly and the awaited delivery enforcement hook, expose it through an additive typed plugin contract, cover central outbound plus Telegram/direct channel paths with tests, document the new public fields, and reconcile overlap with #65149, #39761, and #44011 before merging competing APIs.
