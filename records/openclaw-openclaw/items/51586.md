## Summary

Current main still lacks the requested configurable reset prompt. Bare `/new` and `/reset` now acknowledge without invoking the model, but reply-producing reset paths still build prompt text from hardcoded reset constants, and the config schema/docs expose `agents.defaults.startupContext` rather than `agents.defaults.sessionResetPrompt` or a per-agent equivalent.

## Best Possible Solution

Keep this open for maintainer product judgment. A direct implementation would add a documented, schema-validated `agents.defaults.sessionResetPrompt` plus per-agent override, resolve it before the current-time append, and cover default/custom/empty prompt behavior with focused tests. If maintainers prefer the newer acknowledgement-only bare reset model, they should explicitly rescope the issue to reset tails, soft reset, or a hook/template API rather than treating it as implemented.
