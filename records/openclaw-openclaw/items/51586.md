## Summary

Current main still does not implement a configurable reset prompt. Bare `/new` and `/reset` now acknowledge without invoking the model, but reply-producing reset paths still use hardcoded reset prompt constants, and the config schema/docs expose `agents.defaults.startupContext` rather than `agents.defaults.sessionResetPrompt` or a per-agent equivalent.

## Best Possible Solution

Keep this open for maintainer product judgment. If maintainers want this config surface, add a schema-validated `agents.defaults.sessionResetPrompt` plus the repo’s per-agent shape, resolve it before the current-time append for reply-producing reset paths, document default/custom/empty behavior, and cover default/custom/bootstrap/empty prompt cases with focused tests. If the newer acknowledgement-only bare reset model is the intended product direction, rescope the issue explicitly to reset tails, soft reset, or a hook/template API instead of treating it as implemented.
