## Summary

Current main partially addresses the manifest-provider startup side by backgrounding and scoping primary model prewarm, but the reported external CLI credential path remains valid. The generic auth-store overlay still loops Codex CLI, Claude CLI, and MiniMax CLI credential readers without config/plugin/auth-profile eligibility, and the Claude reader can still call macOS Keychain by default from gateway auth status paths.

## Best Possible Solution

Keep this issue open and narrow the fix to provider-scoped external CLI credential discovery. Preserve the current scoped/backgrounded primary model prewarm behavior, then thread enough config/provider/plugin/auth-profile eligibility into the external auth overlay so an opencode-only setup does not read Claude CLI Keychain. Add regression coverage for restrictive plugins.allow plus auth.profiles/model defaults that exclude Claude/Anthropic, including the gateway models.authStatus path.
