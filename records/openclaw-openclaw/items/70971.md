## Summary

Keep this PR open. Current main still appends bundled MCP/LSP tools after the runtime `toolsAllow` pass, so the reported cron path remains valid; however, the proposed exact-name final filter is not clearly safe because the bundled-tool policy path and docs support plugin-key semantics such as `bundle-mcp`, and focused regression coverage is still missing.

## What This Changes

The PR changes `src/agents/pi-embedded-runner/run/attempt.ts` to run the final combined base and bundled MCP/LSP tool list through `applyEmbeddedAttemptToolsAllow` with `params.toolsAllow`.

## Best Possible Solution

Land a plugin-aware runtime allowlist fix after bundled MCP/LSP materialization that preserves exact tool-name filtering for exec-only cron jobs while also respecting bundled plugin metadata where current policy semantics require it. Add focused regression coverage for exec-only cron/tool runs, exact bundled MCP tool names, `bundle-mcp`, and configured bundled LSP tools before merging.
