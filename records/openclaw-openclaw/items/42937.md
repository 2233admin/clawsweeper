## Summary

Keep this PR open. Current main supports global `session.dmScope` and route-binding `bindings[].session.dmScope`, but it still does not accept or apply the requested `agents.list[].dmScope` per-agent default. The provided PR discussion is useful: Greptile’s original schema-duplication nit appears addressed by the PR head, and the prior ClawSweeper review correctly identified that the PR now needs to be rebased around the newer binding-level override surface.

## Best Possible Solution

Keep this PR open and request a focused rebase/rework. The best implementation would add `AgentConfig.dmScope`, strict zod validation, generated schema/help/label/public docs updates, and selected-agent lookup in route resolution while preserving binding-level overrides with precedence `binding > agent > global`. Add regression tests for per-agent override, global fallback, binding interaction, group behavior, and route-cache correctness.

## Risks / Open Questions

- The PR predates current `bindings[].session.dmScope`; a rebase needs an explicit precedence rule, preferably binding override > agent override > global default, so binding-specific isolation is not regressed.
- Per-agent `dmScope: "main"` intentionally collapses DMs for that agent, so docs/help/tests should call out the privacy and isolation tradeoff.
- No supply-chain or CI/code-execution regression is apparent from the provided PR diff; the main security review concern is the user-visible session-isolation behavior.
