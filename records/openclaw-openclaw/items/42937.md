## Summary

Keep this PR open. Current main still supports global `session.dmScope` and route-binding `bindings[].session.dmScope`, but it does not accept or apply the requested `agents.list[].dmScope` per-agent default. The PR remains a focused implementation candidate, but it needs a rebase around the newer binding-level override surface and an explicit precedence rule.

## Best Possible Solution

Keep this PR open and request a focused rebase/rework. The best implementation would add `AgentConfig.dmScope`, strict zod validation, schema/help/label/generated docs updates, and selected-agent lookup in route resolution while preserving current binding-level overrides with precedence `binding > agent > global`. Add regression tests for per-agent override, global fallback, binding interaction, group behavior, and route-cache correctness.

## Risks / Open Questions

- A naive rebase could regress current `bindings[].session.dmScope` behavior; the intended precedence should be binding override > agent override > global default.
