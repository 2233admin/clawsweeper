## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: a custom `models.providers.*.models[*].compat` flag that lets config-defined custom models opt into `xhigh`. Main now has provider-owned thinking profiles and runtime policy helpers, but the config type/schema, catalog thinking entry, and generic thinking resolver still do not carry or read a custom per-model xhigh compat override. The PR likely needs a rebase/adaptation to the newer `resolveThinkingProfile`/`thinkingLevels` architecture rather than closure as obsolete.

## Best Possible Solution

Keep the PR open for maintainer review or contributor refresh. The best path is to adapt the intended custom-model xhigh opt-in to current main’s provider-profile/runtime-policy design, update the strict config schema and docs if the new compat key is still the desired public API, and preserve the reviewer-identified regression coverage across reply, gateway, ACP, TUI/native command, cron, agent CLI, and `llm-task` surfaces.
