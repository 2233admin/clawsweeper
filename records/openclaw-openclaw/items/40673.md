## Summary

Keep open. Current main still has correct boolean parsing for `OPENCLAW_RAW_STREAM=0` or unset, but the LaunchAgent-managed disable path remains unresolved: raw-stream writes depend on the launched Gateway process environment, while normal LaunchAgent restart only `kickstart`s the loaded job and does not refresh generated service-env wrapper metadata. The public docs still document enabling raw-stream logging only, not disabled values or LaunchAgent precedence.

## Best Possible Solution

Keep this issue open. The best fix is a narrow code, docs, and regression pass: document accepted truthy and disabled values for `OPENCLAW_RAW_STREAM`; explain precedence between launchctl inherited env, state-dir `.env`/config durable service env, and generated LaunchAgent service-env wrappers; and make `openclaw gateway restart`/doctor/install either refresh stale raw-stream service metadata or surface a clear repair instruction when reinstall is required.

- A fix needs macOS LaunchAgent coverage or an equivalent service-metadata regression test because the bug depends on effective launchd process environment, not only the raw-stream parser.
