## Summary

Keep this issue open. Current main still has only a low-level effective-process-env truthy check for `OPENCLAW_RAW_STREAM`, while the reported bug is specifically about LaunchAgent-managed disable semantics after `launchctl unsetenv` or `OPENCLAW_RAW_STREAM=0`. The user-facing docs still explain how to enable raw-stream logging but do not document disable values or LaunchAgent/service-env precedence, and there is no focused regression coverage for raw-stream disable behavior.

## Best Possible Solution

Keep this open and fix it with a narrow docs-and-regression change. Document exact truthy enable values and disable values for `OPENCLAW_RAW_STREAM`, explain precedence across CLI flags, launchctl setenv/unsetenv, generated LaunchAgent service-env files, state-dir `.env`, and config `env.vars`, and add focused tests for disabled `appendRawStream` values plus LaunchAgent/service-env planning or restart readback. If live macOS still appends when the effective gateway env is unset or `0` and no durable source enables it, fix that runtime/service path in the same area.
