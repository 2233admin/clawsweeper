## Summary

Keep this issue open. Current main still makes raw-stream writes depend on the gateway process env, while the macOS LaunchAgent restart path can keep using an already-generated service-env wrapper without rewriting it. The docs still describe enabling raw-stream logging but do not document disable values or the precedence between launchctl inherited env, durable `.env`/config env, generated service env files, and CLI flags.

## Best Possible Solution

Keep this open and fix it with a narrow docs plus regression pass. Document exact enabled and disabled values for `OPENCLAW_RAW_STREAM`, state that `0`/`false`/`off`/unset are disabled at the effective gateway process env level, and explain precedence for CLI flags, `launchctl setenv/unsetenv`, durable `$OPENCLAW_STATE_DIR/.env` / `~/.openclaw/.env`, config `env.vars`, and generated LaunchAgent service-env wrappers. Add focused tests for `appendRawStream` disabled values and LaunchAgent restart/install behavior; if live macOS confirms a wrapper or durable source overrides `launchctl setenv 0` unexpectedly, fix the service-env refresh or status/readback path so operators can see and clear the real source.
