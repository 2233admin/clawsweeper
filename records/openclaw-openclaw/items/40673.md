## Summary

Keep open. Current main parses unset and `OPENCLAW_RAW_STREAM=0` as disabled, but the reported LaunchAgent-managed disable path is still not resolved: raw-stream writes depend on the effective gateway process env, durable LaunchAgent service-env wrapper files can persist raw-stream values outside transient `launchctl` inherited env, normal restart still uses `kickstart -k` without refreshing service metadata, and docs still only describe enabling raw-stream logging.

## Best Possible Solution

Keep this issue open for a focused macOS service-env/docs fix: document accepted truthy and disabled raw-stream values; clarify precedence among `launchctl` inherited env, state-dir `.env` / config durable env, and generated LaunchAgent wrapper files; make `openclaw gateway restart`, `gateway install`, or doctor detect and repair stale raw-stream service metadata or print a precise reinstall/repair instruction; and add a regression test for the LaunchAgent disable path.
