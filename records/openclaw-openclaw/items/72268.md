## Summary

Keep this PR open. Current main still has the nested-parentheses exec approval parsing bug, but the branch is not ready as-is because the parser widening intersects with approval-event spoofing from generic tool output and the diff still carries unrelated generated Plugin SDK baseline hash churn.

## What This Changes

The PR replaces exec approval denied/finished regex parsing with a balanced-parentheses parser, adds nested-metadata parser and followup tests, and changes the generated Plugin SDK API baseline hash file.

## Best Possible Solution

Land the nested-metadata parser fix after approval-status parsing is scoped to trusted exec approval lifecycle data or structured exec status, and after the branch contains only the parser/tests changes with generated Plugin SDK hashes unchanged or correctly regenerated against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate widened approval parsing to trusted exec events:** `src/agents/exec-approval-result.ts:88`
  - body: This change makes `parseExecApprovalResultText()` accept more `Exec denied (...)` wrappers, but `pi-embedded-subscribe.handlers.tools.ts` applies that parser to generic tool output and emits approval events from it. A command can print a nested-wrapper string and be reported as a denied approval, so scope this parsing to trusted exec approval lifecycle data or structured status before broadening the accepted grammar.
  - confidence: 0.86
- **[P2] Remove the stale generated baseline hash change:** `docs/.generated/plugin-sdk-api-baseline.sha256:1-2`
  - body: The PR changes `docs/.generated/plugin-sdk-api-baseline.sha256` even though the functional diff does not touch the Plugin SDK API. Current main already has different generated hashes, so this unrelated hash-only diff will be stale after rebase and can break `pnpm plugin-sdk:api:check`; drop it from the branch.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The patch broadens exec approval wrapper parsing while an existing consumer parses generic tool output into approval events.

Concerns:

- **[medium] Spoofable approval events from generic tool output:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:1059`
  - body: `parseExecApprovalResultText()` is called on `output` derived from exec aggregation or generic tool-result text, then emits an approval event when the text looks denied. Accepting nested wrapper metadata widens the strings that arbitrary command output can use to spoof a resolved approval status.
  - confidence: 0.87
