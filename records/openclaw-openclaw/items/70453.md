## Summary

Keep this issue open. Current main still routes Google Generative AI LLM streaming through the shared guarded model fetch path without an automatic trusted env-proxy upgrade for bare ambient `HTTP_PROXY`/`HTTPS_PROXY`, and the related #72057 report is closed as a duplicate of this issue rather than superseding it. There is now an open implementation candidate, #72480, that targets the same provider-guarded-fetch gap, so the best cleanup path is maintainer review/landing of that PR or an equivalent patch, not closing this issue yet.

## Best Possible Solution

Keep this issue open as the canonical Google LLM ambient-proxy report. Maintainers should review and either land #72480 or an equivalent narrowly gated patch in `buildGuardedModelFetch` that uses trusted env-proxy mode only when no explicit provider dispatcher policy exists and `shouldUseEnvHttpProxyForUrl(url)` is true, with focused tests. If maintainers decide not to support bare ambient proxy env for this path, the supported workaround should be documented clearly and Google LLM proxy failures should surface explicit CLI/WebUI errors.
