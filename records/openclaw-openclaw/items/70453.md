## Summary

Keep #70453 open. Current main has related Google transport and error-surfacing work, but the reported direct Gemini + HTTP_PROXY/HTTPS_PROXY Fake-IP path is not fully implemented: the LLM guarded fetch path still does not auto-select trusted env-proxy mode, so it can still do local DNS/SSRF pinning before the proxy route that the reporter needs.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the LLM provider transport path, probably by applying the same conservative env-proxy gating used by src/media-understanding/shared.ts to src/agents/provider-transport-fetch.ts, and by deciding how configured explicit/env proxy overrides should enter trusted proxy mode without weakening SSRF defaults. Add focused regression tests for google-generative-ai/openclaw infer model runs with HTTP_PROXY/HTTPS_PROXY, NO_PROXY bypass cases, proxy connection failures surfacing to CLI/WebUI, and Fake-IP-style DNS answers.
