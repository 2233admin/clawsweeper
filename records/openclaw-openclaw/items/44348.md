## Summary

Keep this PR open. Current main still exposes `agentId` and `sessionKey` through the second hook context argument, not through the `before_agent_start` or `agent_end` event payloads this PR targets. The PR remains useful and appears narrow, but it is an additive public plugin API change that needs maintainer API review and a rebase onto the current split hook-contract files.

## What This Changes

The PR adds optional `sessionKey` and `agentId` fields to the legacy `before_agent_start` and `agent_end` hook event payloads, threads them through embedded-runner hook calls, and adds regression coverage for populated and omitted fields.

## Best Possible Solution

If maintainers accept the API shape, land a rebased version that adds optional identity fields to the split hook event types, conditionally copies them from existing hook context only when present, documents the event/context relationship, and keeps focused regression coverage for populated and omitted fields. If maintainers prefer context-only identity, close the PR manually with that API rationale.
