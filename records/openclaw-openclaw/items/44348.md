## Summary

Keep this PR open. Current main exposes `agentId` and `sessionKey` through the second hook context argument, but it does not implement this PR's requested event-payload contract for `before_agent_start` or `agent_end`. The PR remains a plausible additive plugin API change that needs maintainer judgment and a rebase onto the current split hook files.

## Best Possible Solution

Review this as an additive plugin API PR. If maintainers want event-level convenience for memory plugins, rebase it onto current main, add optional `sessionKey` and `agentId` to `PluginHookBeforeAgentStartEvent` and `PluginHookAgentEndEvent`, populate them only when available in `run/setup.ts`, `run/attempt.prompt-helpers.ts`, and `run/attempt.ts`, add focused present/omitted regression tests, and update `docs/plugins/hooks.md`. If maintainers decide the second `ctx` argument is the intended contract, close manually with that API rationale rather than as already implemented.
