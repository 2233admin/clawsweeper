## Summary

Keep this PR open. Current main still exposes agentId and sessionKey on the second hook context argument, but not on the before_agent_start or agent_end event payloads this PR extends. The discussion has useful positive signal from plugin users/contributors, and the diff is narrow, but this is still an additive public plugin API change that needs maintainer API review and a rebase onto the current split hook files/docs.

## What This Changes

The PR adds optional sessionKey and agentId fields to before_agent_start and agent_end hook event payloads, threads them through embedded-runner hook call sites, and adds regression coverage for populated and omitted legacy fields.

## Best Possible Solution

Keep this PR open for maintainer review as an additive plugin API change. If accepted, rebase it onto current main by adding optional sessionKey and agentId to PluginHookBeforeAgentStartEvent and PluginHookAgentEndEvent, populate them from the existing hook context only when available, add regression coverage for present and omitted fields, and update docs/plugins/hooks.md. If maintainers prefer the existing second-argument context contract, close it manually with that API rationale.
