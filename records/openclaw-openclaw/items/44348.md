## Summary

Keep this PR open. Current main exposes `agentId` and `sessionKey` on the second hook context argument, but it still does not implement this PR's requested event-payload contract for `before_agent_start` or `agent_end`. The provided discussion and Greptile summary support the same distinction: the PR mirrors context fields that already exist on `PluginHookAgentContext` into the event object for plugin convenience and memory isolation.

## Best Possible Solution

Keep this PR open for maintainer review as an additive plugin API change. If maintainers want event-level convenience for memory plugins, rebase it onto current main's split hook files, add optional `sessionKey` and `agentId` to the two event types, populate them only when available at the call sites, keep regression coverage for populated and omitted cases, and update the plugin hooks docs. If maintainers prefer the existing second-argument context contract, close it manually with that API rationale rather than as implemented on main.
