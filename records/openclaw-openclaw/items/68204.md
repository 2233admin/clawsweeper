## Summary

Keep this issue open. Current main still has separate agent event, cache trace, trajectory bundle, ACP relay-log, task-ledger, and Task Flow state shapes rather than one canonical run-trace schema adopted across the requested flows. The related run-trace PR is still open and unmerged, so it is useful prior art but not shipped current-main behavior.

## What This Changes

Request one canonical run trace schema spanning main agent runs, ACP parent-child relay, subagents, and task flows, with parent-child, step/tool, timing, status, and timeline reconstruction fields.

## Best Possible Solution

Define a small versioned run-trace contract, decide how it composes with trajectory export and task ledgers, then adapt agent events, ACP parent-child relay, subagent lifecycle, task records, and Task Flow state transitions into that contract with focused tests and any needed gateway query surface. Review #43145 as prior art or an implementation candidate before starting a fresh design.
