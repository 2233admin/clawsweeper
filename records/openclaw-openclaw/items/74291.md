## Summary

Keep open. Current main implements the dead-PID stale-lock behavior and has adjacent lock-removal coverage, but it does not include this PR's specific direct unit coverage for `isAlive: () => false`, especially the fresh `createdAtMs` short-circuit case. The PR is small, test-only, and not superseded by current main.

## What This Changes

This PR adds two unit tests in `src/plugins/bundled-runtime-deps.test.ts` asserting that `shouldRemoveRuntimeDepsLock` removes locks when the owner PID is dead, including when the lock age is fresh.

## Best Possible Solution

Keep this PR open for normal maintainer review and merge if CI stays green. The best outcome is to add the focused dead-PID helper tests next to the existing alive-PID and ownerless lock tests without changing production behavior.
