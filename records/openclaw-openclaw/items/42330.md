## Summary

Keep this PR open for maintainer judgment, but do not merge it as-is. Current main already has stronger shared inbound dedupe than the PR-local Map, while the PR's reply-to default change, Google Chat lifecycle/add-on payload handling, and modern DM-vs-space detection are still not fully implemented on main. The PR also bundles security-sensitive auth rewrites with behavior changes and has unresolved review concerns.

## Best Possible Solution

Do not merge this PR as-is. Keep the shared inbound dedupe already on main, review focused follow-ups #58993 and #65058 on their own merits, and handle any Google Chat replyToMode default change as a small maintainer-backed PR with targeted tests, docs/config metadata updates, preserved local-media behavior, and a separate security review for any auth or webhook changes.
