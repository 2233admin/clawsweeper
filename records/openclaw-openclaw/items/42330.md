## Summary

Current main only covers part of this PR's intended Google Chat change. The duplicate-webhook goal is already handled by the shared inbound dedupe path, but the requested Google Chat reply-to default change is still absent, ADDED_TO_SPACE still does not reach message processing, and narrower Google Chat follow-up work remains open. The PR also has unresolved security and regression review findings, so it should not be closed as implemented or merged as-is.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not merge it as-is. Preserve the shared inbound dedupe already on main, split any desired Google Chat reply-to default change into a focused adapter/config/docs/test PR, and reconcile lifecycle payload handling with #65058 and DM/space detection with #58993. Rewrite or drop the manual auth, debug logging, mention, media, and local dedupe changes unless they pass focused security and regression review.
