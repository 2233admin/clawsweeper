## Summary

Keep PR #70459 open. Current main still lacks the proposed Getting Started prerequisite-check section, and the PR targets a real docs gap. It should not be closed as implemented or superseded, but it needs revision before merge because the review comments identify fixable issues in the shell snippets and installer auto-install wording.

## Best Possible Solution

Keep the PR open and ask for a scoped revision: update `docs/start/getting-started.md` with a concise prerequisite Step 0, fix the Bash/PowerShell missing-command detection, and align the auto-install text with `scripts/install.sh` and `scripts/install.ps1` behavior. The final docs should point to `https://docs.openclaw.ai/install/node` for Node/PATH troubleshooting and avoid claiming unsupported dependency remediation.
