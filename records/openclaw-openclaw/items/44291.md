## Summary

Keep this issue open. Current main has fixed the related docs formatter shell assumption by moving `format:docs:check` to a Node script, but it still does not add a native PowerShell CI smoke for contributor commands: the Windows CI matrix has only the existing test task, the Windows job defaults to bash, and the only `pwsh` step is a Defender setup helper rather than a contributor-command smoke.

## Best Possible Solution

Keep this open and add a narrowly scoped Windows native PowerShell CI smoke for selected contributor commands, starting with `pnpm check:docs` now that the formatter is Node-backed. Keep the existing bash-based Windows test lane, document the new guarantee in `docs/ci.md`, and avoid broadening workflow permissions or duplicating the full Windows matrix.
