## Summary

Keep open. Current main still lacks the requested native PowerShell contributor-command CI smoke: the Windows CI matrix has only the existing test task, the job defaults to bash, the only pwsh step is the Defender helper, and docs checks still run under Ubuntu/bash-oriented CI paths. The related docs formatter work has made `pnpm check:docs` shell-agnostic, but that does not add native PowerShell CI coverage; #44293 remains related and #74163 is a broad maintainer tracker, not an implementation PR for this request.

## What This Changes

Request to add a lightweight Windows CI lane that runs selected contributor commands under native PowerShell, starting with `pnpm check:docs`, so shell-specific failures are caught before contributors hit them.

## Best Possible Solution

Add a small `checks-windows` native PowerShell smoke for `pnpm check:docs`, preserving the existing Windows test task and read-only permissions; document the guarantee in `docs/ci.md` and add/update CI-routing tests so future workflow changes do not silently drop the pwsh smoke.
