## Summary

Keep this issue open. Current main has Windows CI and `pnpm check:docs` is now backed by a Node docs formatter, but there is still no native PowerShell smoke lane for contributor commands; the Windows job defaults to bash and runs only the focused Windows test suite.

## Best Possible Solution

Keep this open and add a scoped native PowerShell smoke in CI, either by extending `checks-windows` with a `shell: pwsh` task or by adding a small Windows job for selected contributor commands such as `pnpm check:docs` and a strict-smoke-safe subset. Update `docs/ci.md` afterward to document which contributor commands are guaranteed under native PowerShell.
