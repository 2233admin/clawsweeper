## What This Changes

The PR updates the docs i18n localized link rewriter to use bounded case-insensitive scheme-prefix checks, adds `vbscript:` to the recognized scheme list, and adds localized link regression cases for `vbscript:` and mixed-case `Javascript:` hrefs.

## Best Possible Solution

Land the narrow helper hardening if maintainers want `hasURLScheme` itself to classify known URL schemes case-insensitively and include `vbscript:`. A small direct helper test for mixed-case `HTTP`, `Javascript`, or `VBScript` would make the intended contract more explicit, but I do not see a blocking need for broader parser or docs changes.

## Reproduction Assessment

No high-confidence end-user reproduction path was established: current main already leaves `vbscript:` and mixed-case `Javascript:` hrefs unchanged because they are not root-relative routes. The source-level difference is reproducible by directly testing `hasURLScheme`, and the PR's current coverage plus green CI are enough for a normal maintainer review.

## Solution Assessment

Yes. For the source-level hardening being proposed, bounded `strings.EqualFold` prefix checks are narrower than lowercasing the full URL and preserve the original returned string; a broader URL parser would be unnecessary for this localized route-rewrite guard.
