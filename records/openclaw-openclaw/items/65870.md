## Summary

Keep open. Current main and the latest release still expose the legacy Brave Search docs URL in both Brave provider surfaces, while the docs configuration already treats `/tools/brave-search` as canonical and the legacy docs page remains full content rather than a redirect stub. Open PRs with closing syntax are already paired with this issue, so maintainer review should happen there before the issue is closed.

## What This Changes

Update Brave Search provider docs URLs from the legacy `/brave-search` page to canonical `/tools/brave-search` and remove or reduce the duplicated legacy docs page.

## Best Possible Solution

Review and land one canonical Brave plugin/docs cleanup that updates both provider `docsUrl` values to `https://docs.openclaw.ai/tools/brave-search` and resolves the duplicated legacy docs page according to the docs team's preferred redirect-stub or removal policy.
