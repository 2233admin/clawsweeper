## Summary

Keep this PR open. Current main still does not ship Arabic Control UI localization: `ar` is absent from the locale type, lazy registry, browser-locale resolver, generated locale files, docs, and registry tests. The PR is coherent and the author addressed the initial bot-review problems in follow-up commits, but it needs to be refreshed against the newer generated i18n pipeline before it is mergeable.

## Best Possible Solution

Keep the PR open, but ask for a refresh against current main using the generated Control UI i18n pipeline: add ar to `scripts/control-ui-i18n.ts` `LOCALE_ENTRIES`, `ui/src/i18n/lib/types.ts`, `ui/src/i18n/lib/registry.ts`, and the English `languages` map, run `pnpm ui:i18n:sync` to generate `ui/src/i18n/locales/ar.ts` plus `.i18n/ar.*` metadata, update docs/tests, and keep the RTL root-direction behavior covered by tests.
