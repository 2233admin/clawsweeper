## Summary

Keep this issue open. Current main still backs up the full OpenClaw state directory except installed plugin node_modules, and it does not expose built-in large-state exclusions, smart excludes, progress, or a size/failure policy for `openclaw backup create`. The keepmyclaw ClawHub plugin linked in discussion is a useful workaround, but #44288 remains the open implementation candidate for the built-in command and should be reviewed or replaced before this issue closes.

## Best Possible Solution

Keep this issue open and review #44288 as the current implementation candidate, or implement a narrower core fix in the backup CLI/planner/archive path. The fix should add conservative built-in exclusions or configurable exclude patterns for large ephemeral state, clear progress/size/failure reporting, tests, and docs updates. The keepmyclaw ClawHub plugin can remain a useful workaround, but the built-in backup command still needs to handle this class of installation better.
