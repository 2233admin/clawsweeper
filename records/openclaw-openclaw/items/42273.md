## Summary

Keep this issue open. Current main still does not provide built-in large-state exclusions, progress feedback, or a size/failure policy for `openclaw backup create`; the command still plans the full state directory and only filters plugin `node_modules` under state `extensions/`. The keepmyclaw ClawHub plugin linked in discussion is a useful workaround, but it does not replace fixing the built-in backup command. #44288 remains the implementation candidate from the provided context and should be reviewed or replaced before this issue is closed.

## Best Possible Solution

Keep this issue open and review #44288 as the current implementation candidate, or implement a narrower core fix in the backup CLI/planner/archive path. The fix should add conservative built-in exclusions or user-configurable exclude patterns for large ephemeral state, clear progress/size/failure reporting, tests, and backup docs updates. The keepmyclaw ClawHub plugin can remain a useful workaround, but the built-in backup command should handle this class of installation better.
