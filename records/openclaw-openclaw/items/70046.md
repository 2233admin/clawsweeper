## Summary

Keep this PR open. Current main still rejects bare `HH:MM` / `HH:MM:SS` one-shot cron times because `parseAt` only applies `--tz` to offset-less ISO datetimes before falling through to absolute-time and duration parsing. This PR remains the focused open implementation candidate for the related open bug #59441, with a small parser/test diff and no security-sensitive paths in the provided file list.

Keep this PR open as the active implementation candidate for #59441.
