## Clownfish Work Prompt

The observable failure is `RangeError: Maximum call stack size exceeded` while loading built bundled plugin entries for `minimax`, `google`, and `talk-voice` from `dist/extensions/.../index.js` on Windows Server 2019 / Node v24 with the default stack. Current main partially avoids default startup loading for `minimax` and `google` via `activation.onStartup: false`, but `talk-voice` still starts on Gateway startup and the underlying built-entry import depth is not proven fixed.

Implement one focused fix PR. Prefer reducing the affected built plugin entrypoint import graph or making those plugin entries/lazy registrations shallow enough to import under default Windows Node stack settings. Only use a Windows launcher `NODE_OPTIONS=--stack-size=...` mitigation if code/build evidence shows a safer import-graph fix is not feasible, and keep it tightly scoped. Preserve OpenClaw plugin boundary rules: provider-specific behavior stays in the owning plugin, core gets only generic seams if needed, and do not move bundled provider policy into core.

Add regression/package proof that imports `dist/extensions/minimax/index.js`, `dist/extensions/google/index.js`, and `dist/extensions/talk-voice/index.js` after build with default Node stack. Include Windows proof when available, plus Linux/macOS package smoke if useful. Add a user-facing changelog fix entry with reporter credit; do not add unrelated dependency, release, or plugin API changes.
