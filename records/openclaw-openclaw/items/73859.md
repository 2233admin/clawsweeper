## Summary

Keep open. The report is recent, concrete, and current main still lacks a source, launcher, packaging, or Windows package-smoke change proving that the affected bundled plugin entrypoints import under Node's default Windows stack. Current main has startup deferrals that reduce some eager provider loading, but `talk-voice` is still an explicit startup plugin and `minimax`/`google` remain enabled provider plugins that load when their capabilities are activated.

## What This Changes

Fix Windows packaged OpenClaw so the bundled `minimax`, `google`, and `talk-voice` plugin entrypoints do not fail with `RangeError: Maximum call stack size exceeded` under default Node stack settings.

## Best Possible Solution

Land a focused packaging/runtime fix that makes the affected bundled plugin entrypoints shallow enough to import under default Windows Node settings, then add a Windows package smoke that imports the three built entry files directly. Preserve non-fatal loader diagnostics, but do not treat merely skipping failed plugins as the complete fix.
