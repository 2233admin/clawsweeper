## Summary

Keep this PR open. Current main still lacks normalized Codex app-server tool stream projection, so the PR targets a real gap, but the latest patch still emits new Codex tool args and result/update payloads without the existing Pi tool-stream redaction layer.

## What This Changes

The PR adds normalized Codex app-server tool stream start/update/result event projection and regression tests in the Codex event projector.

## Best Possible Solution

Keep the Codex tool-stream bridge, but emit only sanitized producer-side payloads for start args, update chunks, and final results, using the same redaction guarantees as the Pi tool stream and adding projector tests with secret-bearing command, MCP, dynamic, image, and full-output payloads.

## Reproduction Assessment

Yes. Instantiate the Codex projector with onAgentEvent, send secret-bearing command/MCP/dynamic notifications, and inspect emitted stream:"tool" events on the PR head; the raw emit calls are visible in the diff even without running the test suite.

## Solution Assessment

No. The implementation is pointed at the right narrow surface, but it is not the best safe version until the new normalized Codex tool events route args and results through the established redaction policy.
