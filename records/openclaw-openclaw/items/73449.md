## Summary

Keep this PR open. Current main still lacks normalized Codex app-server tool stream events, so the branch addresses a real gap, but the latest PR head still emits unsanitized Codex tool arguments, partial output, and full-output results onto the agent-event path that Control UI and plugin subscribers consume.

## What This Changes

The PR adds normalized Codex app-server `stream: "tool"` start/update/result projections and regression tests in the Codex event projector.

## Best Possible Solution

Keep the Codex tool-stream bridge, but route start args plus partial and final result payloads through the same redaction policy as the Pi tool stream, preferably via an SDK-safe shared sanitizer seam rather than a deep core import; then add regression coverage for redacted command, MCP, dynamic, image, and full-output payloads while preserving the existing full-output gate and result deduplication.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact tool stream args before emitting them:** `extensions/codex/src/app-server/event-projector.ts:592`
  - body: The new start event forwards `buildToolStreamArgs(item)` directly to agent-event consumers. That includes command strings, MCP/dynamic tool arguments, web queries, and image revised prompts, while the established Pi path applies `sanitizeToolArgs` before emitting. Secrets or private prompt context can now surface in Control UI/plugin event consumers even when full output is disabled, so reduce or sanitize this payload before emission.
  - confidence: 0.91
- **[P1] Redact streamed tool results before publishing:** `extensions/codex/src/app-server/event-projector.ts:611-634`
  - body: The new update/result events publish raw `partialResult` chunks and full-output command/MCP/dynamic/image payloads without the Pi `sanitizeToolResult` redaction layer. Gateway can strip result fields for non-full clients, but global agent-event subscribers and full-output UI paths still receive the raw values. Sanitize partial and final payloads before emitting them.
  - confidence: 0.89
