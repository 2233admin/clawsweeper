## Summary

Keep this PR open. Current main still lacks the Codex app-server normalized tool stream bridge, and the branch is focused with useful projector tests, but the latest PR head still emits raw tool args and result payloads on the new `stream: "tool"` path without the redaction layer used by the existing Pi tool stream.

## What This Changes

The PR adds normalized Codex app-server tool start/update/result events in the Codex event projector and adds unit coverage for command events, result deduplication, and output-gated bulky payloads.

## Best Possible Solution

Keep the Codex tool-stream bridge, but apply the same producer-side redaction guarantee as the Pi path to start args, update chunks, and final result payloads before they enter the global/session agent-event bus; add focused projector tests for secret-bearing command args, MCP/dynamic args, output deltas, and full-output results while preserving the full-output gate and result deduplication.

## Reproduction Assessment

Yes. Instantiate `CodexAppServerEventProjector` with `onAgentEvent`, send an `item/started` command or MCP/dynamic item containing a token-like string and an output delta/result, then inspect the emitted `stream: "tool"` event payloads on the PR head.

## Solution Assessment

No. The bridge itself is the right narrow direction, but this implementation is not the safest mergeable form until it redacts tool args and results consistently with the existing Pi tool-stream contract.
