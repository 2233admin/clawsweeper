## Summary

Keep this PR open. Current main still lacks the Codex app-server normalized `stream: "tool"` projection that the PR is trying to add, and the live PR remains a focused implementation candidate rather than obsolete or already implemented. The security pass did not find unrelated supply-chain/CI/dependency changes, but the new live event payloads still need maintainer review against the existing Pi tool argument/result sanitization policy.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is maintainer review of the Codex projector bridge, align the new Codex `stream: "tool"` args/results with the established Pi sanitization/output-gating policy, then merge after the targeted Codex projector tests plus extension format/type checks are green. If the existing sanitizer is the intended policy, expose or reuse it through an approved plugin SDK/runtime seam rather than importing core internals from the Codex plugin.
