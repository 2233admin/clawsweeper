## Summary

Current main has not implemented this PR’s intended Telegram behavior. The inbound Telegram message path still resolves `requireMention` with `activationOverride` before `topicConfig?.requireMention`, so a persisted `/activate` or `/deactivate` state can still override explicit per-topic config. The linked bug report #49864 remains open, and the PR is a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate for the linked Telegram topic `requireMention` bug. The best path is to update or land a focused fix in `extensions/telegram/src/bot-message-context.ts` that makes explicit topic `requireMention` win over activation state, with regression tests covering both `false` and `true` topic overrides against activation state.
