## Summary

Keep this PR open. Current main still computes Telegram group mention gating with persisted activation state before explicit topic-level `requireMention`, so the PR's intended behavior is not implemented on main. The PR remains a focused implementation candidate for #49864, with the existing review comment's inverse regression-test gap still worth addressing.

## Best Possible Solution

Keep this PR open as the implementation candidate for #49864. The best path is to land a focused Telegram plugin change that gives explicit topic `requireMention` precedence over activation state, with regression tests for both `false` and `true` topic overrides against activation state.
