## Summary

Keep this PR open. Current main still computes Telegram inbound `requireMention` with persisted activation state before explicit topic config, so the PR’s intended behavior is not implemented on main. The linked bug #49864 remains open, and this PR is still a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate for #49864. The best path is to update or land a focused Telegram plugin change in `extensions/telegram/src/bot-message-context.ts` so explicit topic `requireMention` wins over activation state, with regression tests for both false and true topic overrides against activation state.
