## Summary

Keep this PR open. Current main still lacks the Active Time overview card, and the latest PR diff is much narrower than the earlier gateway/macOS review comments, but it still has two blocking issues: the clipped-duration path can inflate Throughput under hour filters, and the Control UI i18n generated metadata is not updated with the new source strings.

## What This Changes

This PR adds an Active Time summary card to the Control UI Usage Overview, adds localized copy and tests, and clips usage duration stats to selected day/hour filters.

## Best Possible Solution

Land the Active Time card as a narrow Control UI change, but keep Throughput numerators and denominators on the same scope when filters clip active duration, and regenerate the Control UI i18n outputs so locale metadata and translation memory stay consistent.

## Reproduction Assessment

Yes for the current-main feature gap: source inspection shows no Active Time card while duration stats already exist. The blocking PR issues also have code-level reproduction paths through selected-hour totals versus clipped duration, and through the generated i18n source-hash contract.

## Solution Assessment

No. The feature direction is reasonable, but this PR is not merge-ready until it aligns filtered throughput math and includes the generated i18n metadata/translation-memory changes expected by the repository tooling.

Full review comments:

- **[P2] Keep throughput totals aligned with clipped active time:** `ui/src/ui/views/usage.ts:325-329`
  - body: When an hour filter is active, `displayTotals` still comes from full matching sessions, but this new call passes selected day/hour filters so `durationSumMs` is clipped. Throughput then divides full-session tokens/cost by the clipped duration, inflating the card for partial-session filters; either clip the numerator too or keep throughput on the same full-session scope.
  - confidence: 0.86

  - confidence: 0.84
