## Summary

Keep this PR open, but it is not merge-ready. Current main still lacks the Active Time overview card, so the feature remains useful, but the branch now mixes that UI change with protocol, Gateway, session-cost, and Swift model changes and still has blocking compatibility, performance, and localization issues.

## What This Changes

This PR adds an Active Time card to the Control UI Usage Overview, adds usage tests, and expands usage date interpretation with IANA time-zone support across the web UI, Gateway usage APIs, session-cost bucketing, protocol schema, and Swift protocol models.

## Best Possible Solution

Land a narrow, localized Active Time overview card that reuses existing duration stats with focused UI and metrics coverage, and move IANA time-zone protocol, Gateway, session-cost, and Swift client changes into a separate compatibility-reviewed PR.

## Reproduction Assessment

Not applicable as a bug reproduction for the requested feature; current-main source inspection is enough to show the card is missing. The blocking PR defects have high-confidence code-level reproduction paths: cached legacy gateway state, one-bound date params, per-record formatter allocation, and hardcoded Usage Overview copy.

## Solution Assessment

No. The Active Time card direction is reasonable, but this PR is not the best way to solve it because it couples a localized UI display change to broader protocol, Gateway, session-cost, and Swift client behavior that needs separate compatibility design and validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Re-probe cached legacy gateways before blocking Local mode:** `ui/src/ui/controllers/usage.ts:423-424`
  - body: When the persisted compatibility cache says date-interpretation params are unsupported, this branch throws in Local mode before sending another request. A gateway upgraded at the same URL can therefore keep failing until browser storage is cleared, even though it may now support the fields.
  - confidence: 0.89
- **[P2] Preserve open-ended usage date ranges:** `src/gateway/server-methods/usage.ts:433-435`
  - body: This turns any request with only startDate or only endDate into INVALID_REQUEST. Those fields are still independently optional in the public schema, and current main falls back to the days/default window when one bound is missing, so older clients can start failing after this deploy.
  - confidence: 0.86
- **[P2] Cache the time-zone date formatter in session scans:** `src/infra/session-cost-usage.ts:185-190`
  - body: formatDateKeyInTimeZone creates a new Intl.DateTimeFormat for every transcript entry visited by the usage scanners. Large histories can allocate thousands of formatters per Usage page load; cache one per time zone as the Gateway helper already does.
  - confidence: 0.8
- **[P2] Localize the Active Time overview copy:** `ui/src/ui/views/usage-render-overview.ts:513-517`
  - body: The new Active Time title, hint, and sublabel are hardcoded in the renderer while current Usage Overview copy uses i18n keys. Add English keys and regenerate locale outputs so non-English UI builds can translate the card and raw-copy gates stay green.
  - confidence: 0.84
