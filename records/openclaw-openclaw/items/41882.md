## Summary

Keep this PR open. Current main still serializes same-session Discord inbound jobs through the bundled Discord plugin's outer KeyedAsyncQueue before the core steer/steer-backlog reply logic can run. The PR remains a plausible implementation candidate, but it targets the removed src/discord layout and needs a rebase to extensions/discord plus session-override-aware, plugin-boundary-safe queue resolution. Security review did not show workflow, lockfile, dependency, publishing, or script changes in the provided diff; the meaningful risks are behavioral concurrency and plugin boundary compliance.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase the fix onto extensions/discord, make the Discord inbound worker bypass the outer per-session KeyedAsyncQueue only when the effective queue mode is steer or steer-backlog, preserve serialization for collect/followup/interrupt, honor session-level queue overrides, and expose or reuse a plugin-SDK-safe queue settings helper rather than reaching into core internals from the bundled plugin.

## Risks / Open Questions

- The PR cannot be merged mechanically because it edits the old src/discord layout instead of the current extensions/discord plugin files.
- A config-only bypass can mis-honor per-session /queue overrides, including cases where global Discord config is steer but the session was switched to a serialized mode.
- Bypassing the outer worker changes same-session concurrency; the rebased fix needs coverage for steer, steer-backlog, non-steer serialization, deactivation, replay dedupe, timeout fallback, and run-state status accounting.
- A rebased implementation must avoid importing core auto-reply internals from bundled plugin production code unless a plugin-SDK-safe queue settings seam is added.
- Security review: the provided diff only changes Discord runtime/test files and does not touch workflows, lockfiles, package publishing metadata, dependency sources, generated/vendor assets, or install/build/release scripts.
