## Summary

Current main still serializes Discord inbound jobs through the outer per-session queue before reply queue mode logic can apply `steer` or `steer-backlog`; this PR remains relevant, but its patch targets the old `src/discord` path and needs a current `extensions/discord` rebase or replacement.

## Best Possible Solution

Keep this PR open for maintainer follow-up or replace it with a current implementation under `extensions/discord`. The fix should resolve the effective Discord queue mode early enough to bypass the outer per-session `KeyedAsyncQueue` for `steer` and `steer-backlog`, preserve serialization for non-steer modes, and honor session override precedence where available. Add focused regression coverage for steer, steer-backlog, non-steer serialization, session override precedence, replay dedupe, timeout fallback, deactivation, and busy/run-state accounting.
