## Summary

Keep open. Current main still matches the reported Feishu-specific gap: Feishu admits or drops group messages using config-derived `requireMention` before the session entry written by `/activation mention` can affect the hard routing gate. The existing discussion correctly identified the split and the current workaround of setting Feishu `requireMention` in config, but that does not solve the session-level command behavior.

## What This Changes

The issue asks for Feishu group `/activation mention` to make the current group session require bot mentions, so ordinary unmentioned group messages no longer trigger replies after the command succeeds.

## Best Possible Solution

Implement effective group activation inside the Feishu plugin boundary: keep config-derived `requireMention` as the default, apply persisted per-session `/activation` state as the override before the hard mention gate, and document/test the supported Feishu behavior.
