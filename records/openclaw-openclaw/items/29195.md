## Summary

Keep this issue open. Current main has improved ACP docs, task records, timeouts, and orphan cleanup, but the main ACPX/Codex reliability gaps remain: ACPX still defaults to approve-reads/fail, the runtime receives those settings directly, parent-owned cross-agent ACP child visibility is still gated by generic all+A2A policy, and docs still describe manual stale-process cleanup.

## Best Possible Solution

Keep this issue open as the canonical ACPX/Codex non-interactive reliability umbrella unless maintainers split it into narrower linked follow-ups. The best path is a maintainer-approved ACPX permission profile or spawn preflight with actionable remediation, parent-owned ACP child visibility that does not require broad all+A2A access, and stale harness cleanup/reporting that removes the current manual ps/kill path. The existing docs should stay, but they are not enough to close the issue.
