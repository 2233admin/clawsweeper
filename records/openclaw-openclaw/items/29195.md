## Summary

Keep open: current main documents the ACPX permission workaround and has improved ACP task cleanup/status plumbing, but the core ACPX/Codex reliability gaps remain around noninteractive permission preflight/defaults, parent-owned cross-agent child visibility, and fully automatic stalled-session reporting/cleanup.

## What This Changes

The issue requests more reliable ACP/Codex sessions by fixing noninteractive permission failures, progress/status visibility, stalled-session reporting/cleanup, and permission documentation.

## Best Possible Solution

Keep this open as the canonical ACPX/Codex noninteractive reliability umbrella unless maintainers split it into narrower follow-ups. The best path is a maintainer-approved ACPX permission profile or spawn preflight with actionable remediation, parent-owned ACP child status/history visibility that does not require broad all+A2A access, and stalled-session timeout/cleanup/reporting that replaces the manual ps/kill runbook.
