## Summary

Keep open. Current main still routes every bare numeric Discord outbound target to `channel:<id>` and does not thread `allowFrom` into Discord outbound normalization, while this PR is a scoped implementation for that missing behavior. It still needs maintainer review for the legacy account-level `dm.allowFrom` precedence edge case and normal extension validation before merge.

## What This Changes

The PR updates the Discord plugin to treat bare numeric targets listed in `allowFrom` as DM `user:` targets, applies the same disambiguation in target resolution, prefers top-level `allowFrom` with legacy `dm.allowFrom` fallback, and adds focused Discord tests.

## Best Possible Solution

Keep this PR open, resolve the top-level inherited `allowFrom` versus account-level legacy `dm.allowFrom` precedence question, then land the scoped Discord-plugin fix if that compatibility behavior is accepted. Before merge, run the focused Discord tests/format/lint checks and the extension changed gate.
