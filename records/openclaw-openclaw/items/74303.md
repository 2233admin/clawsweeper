## Summary

Current main does not implement this PR's allowFrom-aware bare numeric Discord DM disambiguation. The related explicit-target fix is already present on main, but that only preserves `user:<id>`/`channel:<id>` kinds and does not cover trusted bare numeric IDs from allowFrom, so this PR should stay open for normal maintainer review.

## What This Changes

The PR updates Discord outbound target normalization, target resolution, config allowFrom accessors, and focused tests so bare numeric IDs explicitly listed in allowFrom resolve as DM user targets while other bare numeric IDs remain channel targets.

## Best Possible Solution

Keep this PR open for normal maintainer review. Current main has the plugin seam needed for the fix and already landed the separate explicit-target preservation, but the allowFrom-aware bare numeric DM case remains missing; the best path is to review the small Discord-plugin patch, resolve the allowFrom precedence question, and run the focused Discord extension validation before merge.
