## Summary

Keep open. Current main still routes drained queued system events into user prompt text, and the linked closing PR #72761 is open and unmerged, so this issue should wait for maintainer review or merge rather than be closed by cleanup.

## What This Changes

Background task updates, deferred maintenance notifications, and heartbeat prompts should be delivered through system-role context or another non-user mechanism instead of polluting user-role prompt text.

## Best Possible Solution

Land a narrowed version of #72761, or an equivalent maintainer-owned patch, that keeps drained runtime-generated system events out of `commandBody`, `followupRun.prompt`, transcript prompt fields, and queued prompt text while preserving the event content in runtime system context and keeping heartbeat policy explicit.

## Reproduction Assessment

Yes. A focused unit reproduction already exists on current main: mock `drainFormattedSystemEvents` to return a `System:` line, call `runPreparedReply`, and inspect the `runReplyAgent` call; the current test asserts that the event appears in `commandBody` instead of runtime system context.

## Solution Assessment

Yes for keeping this issue open until the linked fix lands. The narrow system-event routing approach in #72761 is the right direction, but the PR should drop unrelated plugin CLI changes and maintainers should confirm the heartbeat scope before merge.
