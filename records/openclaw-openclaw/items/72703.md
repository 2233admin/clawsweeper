## Summary

Keep open. Current main still drains queued runtime system events into the user prompt path, and the linked fix PR #72761 is open and unmerged, so cleanup should wait for maintainer review or an equivalent merged fix.

## What This Changes

Background task updates, deferred maintenance notices, and heartbeat/system-event prompts are requested to avoid being delivered as user-role prompt text and instead use system/runtime context.

## Best Possible Solution

Land #72761 or an equivalent narrow change that routes drained runtime-generated system events into `extraSystemPrompt` or another runtime system-context path, keeps them out of `commandBody`, queued prompts, and transcript prompts, and preserves the existing untrusted-event sender downgrade. Heartbeat prompt role semantics should be explicitly accepted as out of scope or handled in a separate maintainer-scoped change.

## Reproduction Assessment

Yes. Current main has a focused source-level reproduction in `get-reply-run.media-only.test.ts`: a mocked `drainFormattedSystemEvents` value is asserted to appear in `commandBody` rather than runtime system context.

## Solution Assessment

Yes for the queued system-event bug: moving drained events into existing runtime system context is the narrowest maintainable fix. The heartbeat prompt portion is less clear because current docs and the linked PR intentionally leave heartbeat prompts as user-message turns.
