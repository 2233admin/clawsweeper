## Summary

Keep open. Current main still routes drained runtime system events into user prompt text, and the open closing PR #72761 is unmerged, so this issue should wait for maintainer review/merge of that fix or an equivalent change.

## What This Changes

The issue requests that background task updates, deferred maintenance notices, and heartbeat/system-event prompts stop being injected as user-role prompt text and instead use system-role/runtime context or another non-user mechanism.

## Best Possible Solution

Land #72761 or an equivalent narrow fix that routes drained runtime-generated system events through runtime system context, keeps them out of commandBody, queued prompts, and transcript prompts, and preserves the existing untrusted-event ownership downgrade plus documented heartbeat prompt behavior unless maintainers intentionally revise that contract.

## Reproduction Assessment

Yes. Current main has a focused source/test reproduction path: mocked drainFormattedSystemEvents output is asserted to appear in commandBody and followupRun.prompt while extraSystemPrompt lacks Runtime System Events in src/auto-reply/reply/get-reply-run.media-only.test.ts.

## Solution Assessment

Yes for the queued runtime-system-event bug: moving drained events into extraSystemPrompt/runtime context is the narrowest maintainable direction. Heartbeat prompt behavior should stay out of scope unless maintainers make a separate product decision.
