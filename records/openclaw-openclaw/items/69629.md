## Summary

Keep open. Current main protects execution after a Discord slash command is selected, but it still does not implement per-guild-channel command picker visibility for multi-bot Discord setups. The remaining work is a Discord plugin/product design decision, not an already-shipped fix or a duplicate of the older execution-routing items.

## What This Changes

Request per-channel Discord slash-command visibility for multi-bot same-guild deployments, so only the bot assigned to a channel appears in that channel’s Discord `/` command picker.

## Best Possible Solution

Keep this open as a Discord plugin feature request. Maintainers should decide whether OpenClaw should manage Discord application-command permissions per account/guild/channel, or instead document supported workarounds such as per-account `commands.native=false`, text commands, or server isolation.
