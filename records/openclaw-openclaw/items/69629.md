## Summary

Keep open. Current main still protects Discord slash-command execution and routing, but it does not implement the requested per-guild-channel Discord command-palette visibility for multi-bot setups. Native commands are still deployed per account from a full command catalog, and channel allowlists are enforced only after Discord sends the interaction.

## Best Possible Solution

Keep this open as a Discord plugin feature request. The best path is maintainer review of Discord's application-command permission model, then a Discord-owned config and reconciliation design for per-account/per-guild-channel native command visibility where feasible. If OpenClaw should not own the required OAuth/Bearer-token flow, update the public Discord and slash-command docs with explicit workarounds such as per-account native command disablement, text commands, or server isolation.
