## Security Review

Status: needs_attention

Summary: The diff does not add supply-chain or dependency risk, but it changes a security-sensitive exec-approval state and socket root and needs maintainer review of the existing Aisle concerns.

Concerns:

- **[medium] Review fresh state-dir approval policy semantics:** `src/infra/exec-approvals.ts:185`
  - body: When `OPENCLAW_STATE_DIR` points to a new directory, the PR intentionally creates a new approvals file rather than migrating an existing stricter `~/.openclaw/exec-approvals.json`. That may be correct state-dir semantics, but maintainers should explicitly accept it or add migration/warning behavior because this store controls host exec approvals.
  - confidence: 0.72
- **[medium] Review socket safety for shared state directories:** `src/infra/exec-approvals.ts:196`
  - body: The default approval socket would move under `OPENCLAW_STATE_DIR`. If operators set that root to a shared or writable directory, a local process may be able to pre-bind the socket path before the approval server; maintainers should decide whether state-dir ownership/mode checks or a private socket subdirectory are required for this path.
  - confidence: 0.64
