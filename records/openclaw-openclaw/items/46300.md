## Summary

Keep this PR open. Current main still has the old read-only behavior for `workspaceAccess: "none"`, so the PR is addressing a real bug, but the proposed patch remains incomplete because it omits the SSH remote FS bridge and leaves the compaction reference docs inconsistent with the changed memory-flush gate.

## What This Changes

The PR changes sandbox Docker mount permissions, local FS bridge writability, config-hash recreation, memory flush gating, docs, and tests so `workspaceAccess: "none"` treats the isolated sandbox workspace as writable.

## Best Possible Solution

Land a corrected sandbox-permission patch that applies the same isolated-writable `none` semantics across Docker mounts, local FS bridge, SSH remote FS bridge, config-hash recreation, memory flush, and public docs, with regression tests and targeted runtime proof for Docker plus the remote shell bridge.
