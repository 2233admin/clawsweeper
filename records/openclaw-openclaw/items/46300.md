## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: `workspaceAccess: "none"` remains read-only in Docker mounts, the local FS bridge, the remote/OpenShell bridge, memory-flush gating, tests, and docs. The PR is a plausible implementation candidate and the author addressed earlier bot review comments in the second commit, but it still needs maintainer review because it broadens sandbox write behavior and appears to miss the remote FS bridge path.

## Best Possible Solution

Review and either land a corrected version of this PR or replace it with a maintainer-owned patch that makes `workspaceAccess: "none"` writable only for the isolated sandbox workspace across Docker mounts, browser creation, local FS bridge, remote/OpenShell FS bridge, memory flush gating, config-hash recreation, docs, and regression tests. Use the related open behavior threads #37634 and #36525 for product context, and include targeted or live proof that the agent workspace remains inaccessible.
