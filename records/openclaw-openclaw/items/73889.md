## Summary

Keep this PR open. Current main already has the earlier Windows policy-close and `schtasks` `Last Result` fixes, but it does not implement this PR's central behavior: normal `openclaw gateway restart` on Windows can still return healthy from Scheduled Task runtime plus port ownership without requiring loopback Gateway reachability. The PR remains a plausible focused candidate for the remaining post-ready HTTP/WebSocket loss path in #63491, though its changelog wording should not close that broader issue unless the full bug is actually fixed.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a focused Windows-only restart-health change that requires loopback Gateway reachability after Scheduled Task runtime and port ownership look ready, preserves the existing exact policy-close allowlist and `schtasks` parser behavior, adds the proposed regression coverage, and adjusts the changelog to reference #63491 unless the broader canonical issue is fully resolved.
