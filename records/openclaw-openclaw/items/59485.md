## Summary

Keep PR #59485 open. Current main still does not implement the PR's observable exec-env plumbing: `createOpenClawCodingTools` has no `paperclipRuntimeEnv` option, `ExecToolDefaults` has no matching default, and the embedded attempt path does not forward such a value. The related Paperclip runtime-auth serialization discussion is partly superseded by the current authStorage design, but that does not prove this PR's exec tool default forwarding is implemented or obsolete enough to auto-close.

## Best Possible Solution

Keep this PR open for maintainer handling. The best path is to either refresh the patch into a generic, typed exec runtime-env forwarding path that covers `ExecToolDefaults`, `createExecTool`, and `run/attempt.ts` with regression tests, or make an explicit maintainer product decision that Paperclip exec env injection no longer belongs in core and close it manually with a canonical replacement or follow-up.
