## Summary

Keep this PR open. Current main still spawns direct `systemctl --user` without synthesized `XDG_RUNTIME_DIR` or `DBUS_SESSION_BUS_ADDRESS`, and the broad missing-systemctl classifier still collides with the bus error this PR addresses. The PR is a focused replacement for the closed, unmerged #64020 approach, credits the #63561 reproduction and prior implementation, and I did not find a concrete blocking correctness or security issue in the patch from read-only review.

## What This Changes

The PR updates daemon systemd helpers to synthesize missing user DBus environment for direct `systemctl --user` calls, preserve root-wrapper target-user scope via machine scope, narrow missing-systemctl classification, add daemon regression tests, and add a changelog entry.

## Best Possible Solution

Keep this PR as the active implementation candidate for #63561, review the synthesized user-bus environment and root-wrapper scope behavior, run the targeted daemon tests plus the changed gate, and land it if validation stays green.

## Reproduction Assessment

Yes. #63561 gives a concrete Ubuntu 24.04 path where `systemctl --user` works with the user bus env in shell but OpenClaw status/install fails, and current-main source inspection confirms the child `systemctl` spawn does not recover those env vars.

## Solution Assessment

Yes, with maintainer validation. Compared with the closed #64020 approach, this PR applies env synthesis only to the direct user-scope path and preserves root-wrapper target scope through machine scope, which is the narrower maintainable direction for the observed bug.
