## Summary

Keep this issue open. Current main still only partially satisfies the request: SearXNG web_search and model-provider request transport have private-network support, but built-in web_fetch has no allowPrivateNetwork policy path and Gateway OpenResponses input_file/input_image URL fetches still force private-network denial.

## What This Changes

The issue requests a default-off private/local network access option for web_fetch, web_search/image tooling, and Gateway URL fetches, with local gateway deployments able to opt in or auto-enable it.

## Best Possible Solution

Add a narrow, documented, default-off private-network policy for tools.web.fetch.ssrfPolicy and Gateway OpenResponses files/images URL fetches, propagate it only into the intended fetchWithSsrFGuard calls with policy-aware cache keys, keep current SearXNG/provider behavior as partial precedent, and require maintainer security sign-off before any local-mode default is enabled.
