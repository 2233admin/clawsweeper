## Summary

Keep this issue open. Current main still strips paragraph/newline boundary text before emitting block chunks, then delivers plain text block replies across text_end flushes without boundary metadata. Coalescing can reinsert a joiner only while chunks remain in the same buffer, so edit-in-place clients that concatenate separate streamed deliveries can still lose Markdown paragraph structure.

## Best Possible Solution

Keep this issue open and fix the shared block-streaming pipeline. Preserve boundary semantics from EmbeddedBlockChunker through block reply delivery, either by carrying explicit paragraph/newline/hard-split metadata or by retaining consumed separators while normalizing dedupe and coalescing behavior. Add a regression test that reconstructs visible text from multiple flushed text_end block deliveries and preserves Markdown like `# Title\n\nFirst paragraph.`.
