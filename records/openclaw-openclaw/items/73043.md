## Summary

Keep open. Current main still does not route simple-completion calls through the `claude-cli` runtime or fail early with a clear unsupported-runtime error; the reported `Connection error` is consistent with the code path handing `cli://claude` to pi-ai's Anthropic provider as a base URL.

## What This Changes

The issue asks OpenClaw to make Lossless/pi-ai simple completions for `claude-cli/claude-sonnet-4-6` either execute through the Claude CLI transport or return a specific unsupported-provider error.

## Best Possible Solution

Define an explicit simple-completion contract for CLI runtime models: either expose a supported non-mutating Claude CLI one-shot helper for context engines, or reject CLI-runtime/simple-completion use early with a clear unsupported-runtime message and docs guidance for Lossless/LCM.

## Reproduction Assessment

Yes. The issue includes a concrete standalone harness, and the source-level reproduction is clear: a `claude-cli` Anthropic model with `baseUrl: "cli://claude"` reaches pi-ai's Anthropic simple stream rather than OpenClaw's Claude CLI backend.

## Solution Assessment

Unclear. Supporting Claude CLI summaries may be valuable, but treating `cli://claude` as a normal pi-ai base URL is not maintainable; the safer direction is a deliberate OpenClaw contract or an explicit unsupported error.
