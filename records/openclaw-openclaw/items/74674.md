## Summary

Keep open. Current main still concatenates matching assistant text blocks before user-facing delivery, the related split from #69737 remains valid, and there is no merged or open referenced fix that establishes a delivery-only pick-one policy.

## What This Changes

The issue requests a delivery-layer policy that chooses one visible assistant text block instead of concatenating multiple text content items for user-facing channel delivery.

## Best Possible Solution

Keep transcript and history extractors concatenating by default, but add an explicit delivery-scoped selector or opt-in extractor option so channel delivery posts one selected visible answer block. Add focused regression coverage for the delivery path and leave TUI, gateway preview, and chat-history behavior unchanged unless intentionally documented.
