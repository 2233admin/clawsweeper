## Summary

Keep this PR open. Current main still does not implement the intended Telegram native `channel_post` command path, the PR is a focused implementation candidate, and its same-author paired auth-semantics issue #49062 remains open.

## Best Possible Solution

Keep this PR open for maintainer review. Refresh it against current main, make built-in and plugin Telegram native command handlers use the correct `ctx.msg`/channel-post message source, align channel-post authorization with existing explicit chat, `commands.allowFrom`, `groupPolicy`, `sender_chat`, and per-chat allowlist semantics, resolve or document #49062, and add regression coverage for allowed and denied `channel_post` commands plus callback/session follow-ups.
