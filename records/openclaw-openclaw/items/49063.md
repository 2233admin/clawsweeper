## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: built-in and plugin Telegram native command handlers read only `ctx.message`, so `channel_post` command updates do not reach the native command auth/execution path. The PR is also tied to open follow-up #49062 and has security-relevant authorization review comments that need maintainer judgment before merge.

## Best Possible Solution

Keep this PR open for maintainer review. Refresh it against current main, make built-in and plugin Telegram native command handlers consume the correct `channel_post` message source, define authorization semantics across explicit chat allow, `commands.allowFrom`, `sender_chat`, group policy, and per-chat `allowFrom`, resolve or document #49062, and add regression tests for allowed and denied channel-post commands plus any interactive callback follow-ups.
