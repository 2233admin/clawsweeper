## Summary

Keep this PR open. Current main still does not implement the PR’s intended native Telegram channel-post command behavior: the native command handlers read only `ctx.message` and return when it is absent, while Telegram `channel_post` messages are only handled by the separate inbound message pipeline. The PR is also paired with open follow-up #49062 and carries security-sensitive authorization review comments that should be resolved before merge rather than closed as obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase or refresh the patch against current main, implement channel-post native commands using grammY `ctx.msg`, preserve explicit chat allowlist and `commands.allowFrom` semantics, decide or document the #49062 sender-scoped policy, and keep regression tests for allowed, denied, `useAccessGroups`, and channel-post authorization cases.
