## Summary

Keep this PR open: current main does not implement the requested per-call `onComplete` exec API, but the proposed branch still has blocking correctness issues around notification routing, node-host behavior, and value validation. The API direction also overlaps with open PR #46926, so this needs maintainer review rather than cleanup closure.

## What This Changes

The PR adds an `onComplete="notify"` exec parameter that auto-backgrounds local commands, forces completion notifications, adds no-poll response text, and threads explicit completion state through bash process sessions.

## Best Possible Solution

Maintain one exec completion-wake design: either land a revised per-call `onComplete` API or the default-notification direction from #46926, with literal validation, routed and unrouted notification semantics, node-host support or rejection, updated docs, and focused tests for quiet success plus wake delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate the no-poll hint on routable notifications:** `src/agents/bash-tools.exec.ts:1581-1583`
  - body: `onCompleteHint` is appended whenever `onComplete="notify"` is used, but `maybeNotifyOnExit` still drops events when `session.sessionKey` is empty. Tool instances without a session key will tell callers not to poll even though no wake can be delivered.
  - confidence: 0.92
- **[P2] Reject or implement onComplete for node-host exec:** `src/agents/bash-tools.exec.ts:1248`
  - body: The patch treats `onComplete="notify"` as a background request, but `host=node` returns through `executeNodeHostCommand` before the local background/session path. Node executions therefore do not get the promised auto-backgrounding or per-call completion wake semantics.
  - confidence: 0.9
- **[P2] Constrain onComplete to supported values:** `src/agents/bash-tools.exec-runtime.ts:168-173`
  - body: The schema accepts any string while the handler only recognizes `"notify"`, so typos such as `"notfiy"` silently no-op. Expose a literal/enum schema or return a warning/error for unsupported values.
  - confidence: 0.86
