## Summary

Current main has automatic notifyOnExit behavior for background execs, but it does not implement this PR's requested per-call `onComplete="notify"` API. The PR remains potentially useful, but the proposed diff still has material review concerns around session routing, node-host behavior, and input validation.

Keep this PR open for review. ## Risks / Open Questions

- The PR's proposed hint can tell agents not to poll even when no `sessionKey` exists, while current `maybeNotifyOnExit` drops notifications without a session key.
- The proposed schema accepts any string for `onComplete`, so typos can silently fall back to default behavior unless validation is tightened.
- The node-host execution path is separate from local `runExecProcess`, so `onComplete="notify"` needs explicit node support or an explicit rejection.
- Related #46926 overlaps the completion-wake problem but is open/unmerged and does not replace this PR's per-call API.
