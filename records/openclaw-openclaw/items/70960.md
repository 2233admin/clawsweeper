## Summary

Keep open. Current main only partially addresses the bundled report: the Control UI agent selector now has a configured-name path, but the topbar still does not render token/cost/cache usage, and the reported QClaw Windows Electron installer/delete flow is not present in this source tree.

## What This Changes

The issue asks for QClaw Windows upgrades not to delete the installed app, for the Control UI agent list to show `agents.list[].name`, and for the Control UI top-right status area to show token/cost/cache usage.

## Best Possible Solution

Split the bundle into focused ownership paths: route the QClaw Windows upgrade deletion to the QClaw installer owner with the supplied logs and registry detail, keep the configured-name selector behavior, and decide whether Control UI should restore a compact current-session usage indicator in the topbar using the existing session usage data path.

## Reproduction Assessment

Partially. The in-repo UI behavior is reproducible by static inspection of current main: the agent selector reads configured names, while the topbar has no usage renderer; the QClaw installer deletion cannot be reproduced from this checkout because that installer surface is absent.

## Solution Assessment

Unclear for the full issue because there is no proposed patch and the report spans different owners. A single automated OpenClaw PR is not the best path; the maintainable route is a split/reroute plus a focused Control UI usage decision if maintainers want that placement restored.
