## Summary

Keep this PR open. Current main still has the Discord chunker behavior the PR is trying to change: omitted maxLines falls back to 17, the acceptance check includes line count, and the Discord UI hint still documents the 17-line default. The PR is not already implemented on main, and its latest head is still separate from main. It also needs review work before merge because the discussion contains unresolved fence-boundary correctness concerns, including possible non-progress loops and malformed chunk rendering; the security pass found no dependency/workflow/secret-surface changes, but those chunking-loop concerns matter for availability and output integrity.
