## Summary

Keep open. Current main still lacks the requested `--global` skills install/update target, while this PR directly implements the open feature request with focused tests and docs and no blocking correctness finding; the remaining work is maintainer UX/product review and making the branch mergeable.

## What This Changes

The PR adds a `--global` option to `openclaw skills install` and `openclaw skills update`, routes those ClawHub operations to the shared managed skills directory, rejects combined `--global`/`--agent` targeting, and updates command tests plus skills/ClawHub docs.

## Best Possible Solution

Land this PR after maintainer acceptance and mergeability cleanup, keeping workspace installs as the default, using the existing `CONFIG_DIR/skills` managed-skill root for explicit global installs, preserving ClawHub install safety checks, and documenting the scope/precedence clearly.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the requested capability: current main's command definitions and docs show `skills install`/`update` are workspace-scoped with no `--global`, while the loader already supports `CONFIG_DIR/skills` as the shared managed root.

## Solution Assessment

Is this the best way to solve the issue? Yes, if maintainers accept the UX: the patch is narrow, reuses the existing ClawHub installer target shape, preserves default behavior, and rejects ambiguous `--global` plus `--agent` targeting instead of adding a broader config migration or new installer path.
