## Summary

Keep open. Current `main` still leaves Pi auto-compaction enabled for safeguard-mode runs unless a context engine separately declares `ownsCompaction`, and the provided GitHub context shows open PR #73839 specifically references and attempts the narrow fix, so the issue should stay open until that PR or an equivalent patch lands.

## What This Changes

The issue reports that safeguard compaction can be blocked by Pi SDK auto-compaction because OpenClaw only disables Pi auto-compaction for context engines that own compaction.

## Best Possible Solution

Land #73839 or an equivalent narrow patch that makes the Pi auto-compaction guard aware of safeguard-owned compaction, threads that mode through embedded and CLI compaction callers, and adds focused tests for safeguard, default/off, context-engine-owned, and unsupported-settings-manager cases.

## Reproduction Assessment

Yes. A focused source-level reproduction is available: configure `agents.defaults.compaction.mode = "safeguard"` with no owning context engine, then observe that the current guard receives only `activeContextEngine?.info` and returns false while the safeguard extension is still enabled.

## Solution Assessment

Yes. The narrow maintainable solution is to keep the policy in `pi-settings` and pass the resolved compaction mode from the embedded runner and CLI compaction paths, instead of adding startup migrations or unrelated session repair behavior.
