## Summary

Keep this issue open. Current main still surfaces ClawHub 429s as the bare `ClawHub ... failed (429): ...` error for skills search/install, and there is an open unmerged closing PR (#56394) that appears to be the implementation candidate.

## Best Possible Solution

Keep this issue open and review/land or deliberately close #56394. The likely code path is `src/infra/clawhub.ts`: carry whether a request was authenticated from `clawhubRequest`, append the ClawHub login tip only for unauthenticated 429s, and add focused regression tests in `src/infra/clawhub.test.ts` for unauthenticated 429, authenticated 429, and non-429 errors.
