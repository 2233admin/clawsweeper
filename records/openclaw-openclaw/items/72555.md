## Summary

Keep open. Current main still lets identical edit replacements pass OpenClaw validation and the edit wrapper delegates to the base edit tool before any no-op rejection; loop detection exists, but it remains disabled by default, so the reported failure mode is only partially covered.

## What This Changes

Request to add edit-tool identical replacement pre-validation, repeated tool-failure circuit breaking, error classification, and agent guidance to reduce retry loops.

## Best Possible Solution

Add an OpenClaw-owned edit guard that rejects any replacement whose `oldText` and `newText` are exactly identical before the base edit tool runs, with focused tests and a clear non-retryable error; keep broad default circuit-breaker policy separate.

## Reproduction Assessment

Yes. A focused source-level reproduction is to call the OpenClaw edit validation path with `edits: [{ oldText: "same", newText: "same" }]`; current validation accepts that payload and the loop guard is disabled unless configured.

## Solution Assessment

No. Current main is only a partial solution: validation and edit recovery improve bad-parameter and mismatch errors, and loop detection can be enabled, but the requested no-op edit pre-validation is still missing.
