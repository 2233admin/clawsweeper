## Summary

Keep open. Current main does not implement the PR’s intended `agent_end.finalLlmOutcome` contract: the hook payload still has only `runId`, `messages`, `success`, `error`, and `durationMs`, and the embedded runner emits only those fields. The adjacent `model_call_ended` telemetry hook is useful but not an equivalent replacement for plugins that need structured terminal outcome metadata on `agent_end`.

## Best Possible Solution

Keep this PR open as a plausible additive plugin API change. The best path is to rebase it onto current main’s `src/plugins/hook-types.ts` split, keep the change focused on the `agent_end` contract and embedded runner emission path, preserve `success` compatibility, document `finalLlmOutcome.ok` versus `success`, and add targeted regression coverage for terminal assistant errors, prompt/build failures, provider HTTP failures, and compaction/runner failure precedence.
