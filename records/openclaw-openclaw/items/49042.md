## Summary

Keep open. Current main still does not implement the PR’s intended `agent_end.finalLlmOutcome` plugin contract. The shipped `agent_end` event remains the legacy run summary shape, and the newer `model_call_ended` hook is adjacent per-call telemetry rather than the requested terminal agent outcome object with stop reason, HTTP status, and sanitized final error metadata.

## Best Possible Solution

Keep this PR open as a focused additive plugin API candidate. The best path is to rebase it onto current main’s `src/plugins/hook-types.ts` split and current harness helpers, implement optional `agent_end.finalLlmOutcome` without changing legacy `success` semantics, document `finalLlmOutcome.ok` versus `success` at `docs/plugins/hooks.md`, add targeted coverage for terminal assistant errors, prompt/build failures, structured provider HTTP failures, provider timeouts, and compaction/runner failure precedence, and remove unrelated test stabilization edits from this PR.
