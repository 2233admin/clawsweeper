Keep open. Current main still uses strict Feishu channel/account schemas without a Feishu stale-config cleanup hook, and an open unmerged PR explicitly links itself as the implementation candidate for this issue. The issue should not be closed before that PR is reviewed, landed, replaced, or rejected.

## What This Changes

The issue asks OpenClaw to confirm and either fix or document upgrade-time Feishu config validation failures caused by legacy keys such as `installs`, `plugins`, `ackReaction`, and `threadSession`.

## Best Possible Solution

Keep this issue open until #63474 is reviewed and either landed, replaced, or closed. The desired end state is a narrow pre-validation legacy config repair path, preferably owned by the Feishu plugin where keys are Feishu-specific and shared only for truly generic channel keys, with regression tests and upgrade documentation for the deprecated fields that should be stripped or migrated.
