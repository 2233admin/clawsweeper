## Summary

Keep this issue open. Current main still does not have an alias-scoped or per-model `contextProfile` for workspace bootstrap filtering, skills prompt visibility, or tool exposure. Existing controls are global, per-agent, provider-compat, or per-run policy knobs, and they do not cover the requested `/model` role/profile workflow.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-designed, backward-compatible model/alias context-profile surface, possibly folded into the broader #59168 model-key/profile design. It should thread the resolved selected model entry through bootstrap file filtering, skills prompt assembly, effective tool/client-tool schema exposure, system prompt rendering/reporting, context commands, zod/generated schemas, Control UI, and public docs, with tests for default behavior, `/model` switching, workspace allowlists, skills suppression, tool suppression, and precedence with existing policy controls.
