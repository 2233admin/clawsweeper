## Summary

Current main still does not implement the requested alias/per-model `contextProfile` for workspace file injection, skills prompt visibility, or tool exposure. Existing controls are global, per-agent, provider/model capability metadata, or runtime allowlists; they do not provide clean `/model`-scoped profiles inside one session.

## Best Possible Solution

Keep this open. Implement a backward-compatible selected-model or alias-scoped `contextProfile` by extending model entry types, zod/generated schemas, docs, and UI; pass the resolved model entry into bootstrap file filtering, skills prompt assembly, and effective tool/schema exposure; add focused tests for defaults, strict validation, workspace-file allowlists, skills suppression, tool suppression, `/model` switching, and precedence with `compat.supportsTools`, `disableTools`, `toolsAllow`, local-model lean mode, and per-agent tool policy.
