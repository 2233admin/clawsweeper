## Summary

Keep this PR open. Current main still lacks the requested sandbox initialization timeout and public config knob, so the work is not obsolete, but the branch is currently dirty against main and has concrete blocking review issues in sandbox cancellation and sandbox-browser security migration handling.

## What This Changes

The PR adds `agents.defaults.sandbox.initTimeoutMs`, wraps sandbox context initialization in a timeout, threads an abort signal through several Docker/browser startup calls, adds timeout tests, and changes one OpenShell plugin e2e availability guard.

## Best Possible Solution

Revise the active PR, or replace it after maintainer approval, so it rebases cleanly on current main, preserves the CDP source-range security epoch, adds and documents the sandbox init timeout config, and makes cancellation a real backend startup contract with focused timeout/abort regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep the latest browser security hash epoch:** `src/agents/sandbox/constants.ts:42`
  - body: This changes `SANDBOX_BROWSER_SECURITY_HASH_EPOCH` back to `2026-02-28-no-sandbox-env`. Current main uses the newer `2026-04-05-cdp-source-range` epoch to force browser container recreation after CDP source-range hardening, so this rollback can leave older browser containers treated as current.
  - confidence: 0.95
- **[P1] Make backend init cancellation required and honored:** `src/agents/sandbox/backend.types.ts:31`
  - body: Adding `abortSignal?: AbortSignal` keeps cancellation optional, and this branch does not update the Docker or SSH backend factories to pass it into their startup work. The timeout can reject the caller while backend initialization keeps running after the timeout.
  - confidence: 0.88
- **[P2] Pass cancellation into prune internals:** `src/agents/sandbox/context.ts:229`
  - body: Racing `maybePruneSandboxes(cfg)` only unblocks the caller; prune itself still receives no signal and its registry removals can keep awaiting Docker/backend work after the init timeout fires. Thread the signal through prune or avoid starting prune once cancellation is active.
  - confidence: 0.84
- **[P2] Keep the OpenShell e2e availability guard precise:** `extensions/openshell/src/backend.e2e.test.ts:126`
  - body: Returning true when the command emits any stdout or stderr means an old or broken OpenClaw command can be treated as available even when it exits nonzero. With the gateway capability check removed, this can run the OpenShell e2e in environments that should skip it.
  - confidence: 0.78
