## Summary

Keep this PR open. Current main still lacks the proposed Swift `.gitignore` generalization, `check:no-large-files`, `check:no-unjustified-exec`, and additional-boundary wiring. The PR is an active hygiene contribution rather than implemented or superseded, but its exec-surface CI guard still needs maintainer/security hardening before it should become a required check.

Keep this PR open for maintainer review. ## Risks / Open Questions

- The proposed exec guard is security-sensitive: the current regex can miss semicolonless `exec` imports, allowing a risky child-process surface to bypass CI.
- The proposed raw `shell: true` scan can block harmless comments or string literals, creating future CI false positives.
- The PR allowlists `src/plugins/bundled-runtime-deps.ts` even though current main has no `shell: true` there, which would hide a future addition in that file.
- The large-file allowlist includes an iOS screenshot that is currently below the 3 MiB threshold, so the allowlist should be refreshed before merge.
