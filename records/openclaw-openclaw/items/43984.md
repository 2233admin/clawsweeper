## Summary

Keep open. The linked gateway bug still appears valid on current main, but this PR is stale against the current `listSessionsFromStore` pipeline: main now filters `entries` by `updatedAt` before row construction, while the PR applies the exemption to the older row-level `sessions` filter.

## What This Changes

The PR adds a primary-agent-session helper, exempts primary agent main sessions from `activeMinutes` filtering in gateway session listing, and adds default/custom `mainKey` regression tests.

## Best Possible Solution

Land a rebased gateway fix that applies the primary-session exemption at the current `entries` active cutoff in `listSessionsFromStore`, preserves the custom `mainKey` guard, and keeps focused regression coverage for stale default-main visibility plus stale legacy custom-key filtering.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with a stale `agent:main:main`, a fresh `agent:heartbeat:main`, and `sessions.list` called with `activeMinutes`, current main drops the stale main row at the `entries` cutoff before the UI dropdown receives rows.

## Solution Assessment

No. The intended approach is narrow and maintainable, but this PR applies it to an older row-level filter; the safer fix is to move the same exemption to current main's pre-row `entries` filter and rerun focused gateway tests.
