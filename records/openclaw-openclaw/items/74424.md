## Summary

Keep open. Current `main` still has the reported source-level path: the gateway tool only persists restart continuations through an emit hook, while the already-in-flight restart branch returns `coalesced: true` before using that hook. The discussion usefully points to open PR #73812 as the intended fix path, but it is not merged on `main`, so this issue should remain open.

## What This Changes

Ensure a gateway restart tool request with `continuationMessage` is delivered or explicitly reported as not queued when the restart request coalesces with an in-flight restart.

## Best Possible Solution

The desired end state is a regression-covered restart contract: if a restart request with `continuationMessage` coalesces after a restart signal is already in flight, OpenClaw either durably queues or merges that continuation exactly once, or returns an explicit result saying no continuation was queued.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to leave an unconsumed SIGUSR1 restart signal active, then call the gateway restart tool with `continuationMessage`; current `scheduleGatewaySigusr1Restart()` returns from the already-in-flight branch before the gateway tool's `beforeEmit` hook can persist the sentinel.

## Solution Assessment

No. Current `main` is not the best solution because it can still return successful coalescing while dropping the promised continuation; the narrow maintainable fix belongs at the gateway tool/restart scheduler/sentinel boundary with a regression test for this branch.
