## Summary

Keep this PR open. Current main still does not implement the PR's intended Discord native system-message audit: `resolveDiscordSystemEvent` handles several system `MessageType` values but still omits `ChannelNameChange`, `ChannelIconChange`, `Call`, `ChannelFollowAdd`, discovery warnings, invite reminders, role/app subscription notices, and similar enum members. The related #49308 is open and narrower, not merged, so it is overlap to coordinate, not shipped replacement evidence. Security review found no supply-chain or permissions concern in the provided PR diff scope; it is Discord mapper/test/changelog work only.

## Best Possible Solution

Keep this PR open and review it as the broader Discord system-message fix, or explicitly choose #49308 as the minimal canonical fix if maintainers only want rename/icon/follow coverage. Before landing this PR, rebase it onto current main, update the enqueue-system-event mock target if needed, and keep focused regression coverage for both the mapper and the preflight drop/enqueue path.
