## Summary

Keep this PR open. Current main still lacks an A2A turn visibility hook, but the proposed patch is stale against the current plugin hook contract layout and introduces a raw conversation-content hook without the existing conversation-access gate, so it needs maintainer API/security review before merge.

## What This Changes

The PR adds a public `agent_to_agent_turn` plugin hook, exports a hook-runner method, and emits A2A ping-pong turn metadata, reply text, flow identity, and delivery target data from `sessions_send` A2A flow.

## Best Possible Solution

Ship a first-class A2A turn visibility hook only after rebasing it onto `src/plugins/hook-types.ts`, adding the hook to the conversation-content access policy, documenting the public contract, and covering loader policy plus A2A turn emission with focused tests.

## Reproduction Assessment

Yes. A static current-main check is enough: intermediate A2A turns run through `runAgentStep` with `deliver: false`, and only the final announce calls gateway `send`, so channel plugins cannot observe the ping-pong transcript through existing delivery hooks.

## Solution Assessment

No. The feature direction is plausible, but this PR is not the best merge shape because it adds the contract in a stale file and does not apply the current raw-conversation-access policy; the safer path is a rebased hook-types/registry/docs/tests change.
