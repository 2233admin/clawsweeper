## Summary

Keep open. Current main still lacks an equivalent turn-level A2A hook, and the discussion shows a concrete DingTalk/channel-plugin use case that existing hooks do not compose. The PR is not merge-ready because its hook contract edits target the old `types.ts` surface and it exposes internal A2A replies without the current conversation-access policy.

## What This Changes

The branch adds an `agent_to_agent_turn` plugin hook contract and emits A2A turn events from the `sessions_send` A2A flow with reply text, flow metadata, and delivery target fields.

## Best Possible Solution

Ship a first-class A2A turn hook after rebasing onto `src/plugins/hook-types.ts`, adding conversation-access gating, documenting the public hook, and covering the A2A event flow with focused loader and sessions tests.
