## Summary

Keep this PR open. Current main still does not implement `agent_to_agent_turn` or an equivalent A2A turn-level plugin seam: the public typed hook list has no A2A turn hook, unknown typed hooks are ignored, intermediate ping-pong replies run as internal agent steps, and only the final announce is delivered through the normal outbound message path. The linked DingTalk @sub-agent work is useful context for a real channel-plugin use case, but this remains unmerged core plugin API work that needs maintainer design review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer/design review. The best path is a first-class typed A2A turn hook emitted from `src/agents/tools/sessions-send-tool.a2a.ts`, registered through the public plugin hook surface, documented at `https://docs.openclaw.ai/plugins/hooks`, and covered by focused tests for metadata, routing context, ordering, timeout behavior, and conversation-access policy.

## Risks / Open Questions

- The proposed hook would expose raw A2A reply text to plugins; maintainers should decide whether it belongs in `CONVERSATION_HOOK_NAMES` and requires `plugins.entries.<id>.hooks.allowConversationAccess=true` like other conversation-content hooks.
- The PR still needs focused regression coverage for hook registration, turn 0 emission, stable flow IDs, speaker/listener metadata, delivery target account/thread fields, ordering, bounded slow-handler behavior, and final announce chronology.
- The linked DingTalk channel work shows a concrete external use case, but the core API shape should be reviewed as a durable third-party plugin contract before merge.
