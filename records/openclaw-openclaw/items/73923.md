## Summary

Keep this PR open. Current main still clears the Control UI Gateway Token for any WebSocket URL edit that is not the exact same trimmed string, while this PR adds the missing normalized gateway-token resolver and targeted regression coverage. The patch is narrow, appears correct, and should be reviewed or rebased as the canonical fix for #41545 rather than closed as obsolete.

## What This Changes

The PR adds a Control UI storage helper for resolving tokens during Gateway Access URL edits, wires Overview to it, expands node/browser UI tests, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent scoped Control UI fix as the canonical resolution for #41545: preserve tokens for same-normalized endpoint edits, reload or clear scoped tokens when the endpoint changes, keep the regression coverage, and then supersede the older duplicate PRs.

## Reproduction Assessment

Yes. The current-main handler can be reproduced by entering a Gateway Token in Overview > Gateway Access and editing the WebSocket URL, including a harmless trailing slash edit; the source shows that any non-identical trimmed URL writes an empty token.

## Solution Assessment

Yes. The PR's normalized-scope helper is a safer fix than always preserving the token because it avoids carrying a gateway-scoped credential to a different normalized endpoint while still preserving harmless same-endpoint edits.
