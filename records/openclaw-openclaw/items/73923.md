## Summary

Current main still has the Overview Gateway Access WebSocket URL handler that clears the Gateway Token whenever the edited URL string differs from the previous trimmed value. This PR addresses the active bug tracked by #41545 and repairs the closed, unmerged source PR #42001 with a more conservative same-normalized-endpoint token-scope helper plus targeted regression coverage, so it should stay open for maintainer review rather than be closed as already implemented.

## Best Possible Solution

Keep this PR open as a likely canonical fix for #41545, review the same-normalized-scope token behavior, and land this PR or an equivalent implementation with the included regression coverage. After landing, close or supersede the older duplicate PRs that target the same bug.
