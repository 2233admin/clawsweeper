## Summary

Keep this PR open. Current main and the latest release still have the underscore-only TUI copy-sensitive guard, so long hyphen-separated identifiers are still normalized by inserting a literal space. The PR is a narrow, relevant implementation for the linked TUI bug and does not introduce an obvious security or supply-chain-sensitive change, but it is unmerged and should go through normal maintainer review with focused regression coverage.

## What This Changes

This PR changes `src/tui/tui-formatters.ts` so `isCopySensitiveToken` treats hyphenated `FILE_LIKE_RE` tokens like underscore tokens, preventing `normalizeLongTokenForDisplay` from injecting spaces into long hyphen-separated identifiers.

## Best Possible Solution

Review and land this narrow formatter fix or an equivalent rebased change with a focused `src/tui/tui-formatters.test.ts` regression for long hyphen-separated package names, while preserving the existing behavior that still chunks generic prose-like long tokens for narrow terminals.
