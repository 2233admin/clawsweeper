## Summary

Keep open. Current main still routes same-agent forced heartbeat and cron session keys literally, while the `session.dmScope=main` single-owner rule is only applied on inbound channel routing; the issue is also paired with open, unmerged PR #72648 using closing syntax.

## What This Changes

Request that `session.dmScope=main` also route exec-event and cron-event heartbeat wakes into the agent main session for single-owner DM setups.

## Best Possible Solution

Review, land, or replace #72648 with a narrow core change that normalizes both system-event enqueue keys and forced heartbeat wake targets for same-agent DM sessions when `dmScope=main` and the existing single-owner allowlist rule is satisfied, while preserving explicit group/channel/custom cron routing.

## Reproduction Assessment

Yes. Configure `dmScope=main` with a single-owner Telegram allowlist, enqueue an exec or cron system event under a same-agent direct-message key such as `agent:main:telegram:default:direct:<userId>`, and wake heartbeat with that forced key; current main resolves and peeks that per-channel queue instead of `agent:main:main`.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix if enqueue and wake routing are normalized together. A heartbeat-only rewrite would be incomplete because pending system events are keyed before preflight peeks and drains them.
