Fix a Discord gateway startup/reconnect race where the bot logs in or appears online but never reaches usable READY/inbound message handling after startup.

## Best Possible Solution

Fix this inside the Discord plugin gateway/lifecycle boundary: surface or retry failed IDENTIFY after limiter waits, preserve READY detection when startup events happen before lifecycle observers, and add focused regression tests under `extensions/discord/src/internal/` and `extensions/discord/src/monitor/`.
