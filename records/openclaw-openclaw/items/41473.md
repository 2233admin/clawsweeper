## Summary

Keep this PR open. Current main still does not expose Slack Socket Mode ping/pong timeout configuration, and the linked open request #14248 remains unsolved. The PR is a plausible implementation candidate, but it should be updated before landing because the current dependency exposes public SocketModeReceiver options and the PR still mutates internal SocketModeClient fields.

## Best Possible Solution

Keep this PR open as the implementation candidate for #14248. Rebase it on current main, add the Slack socketMode type/schema/help/labels/docs/tests, pass clientPingTimeout/serverPingTimeout/pingPongLoggingEnabled through createSlackBoltApp into SocketModeReceiver's public options, remove or revalidate the .secrets.baseline churn, and then land it once targeted Slack/config checks and feasible live Socket Mode verification are clean.
