## Summary

Keep this issue open. Current main still advertises ACP session listing but `AcpGatewayAgent` only exposes the old `unstable_listSessions` handler, while the pinned `@agentclientprotocol/sdk` 0.20.0 dispatcher calls `agent.listSessions` for `session/list`. The direct fix PR is closed unmerged, the adjacent open lineage-metadata PR does not fix the handler name, and the latest release remains affected.

## Best Possible Solution

Keep this issue open and land a narrow ACP bridge fix: expose `listSessions` on `AcpGatewayAgent` for the pinned SDK, keep `unstable_listSessions` only as a delegating compatibility alias if maintainers still want old-SDK support, and add regression coverage for `session/list` through the SDK dispatcher or equivalent handler path. The public ACP docs already describe the desired behavior, so the code and tests should be brought back in line rather than changing the docs to match the bug.
