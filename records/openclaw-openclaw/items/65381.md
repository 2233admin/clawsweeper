## Summary

Keep this PR open. Current main still collects cacheRead/cacheWrite for `/status`, and shared cost estimation already supports those fields, but `buildStatusMessage` still gates and estimates cost from input/output tokens only. The PR remains a focused implementation candidate for the still-open #65109 bug, with no security-sensitive changes visible in the provided diff.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent narrow fix. The implementation should update `src/status/status-message.ts` so `cacheRead` and `cacheWrite` participate in both the cost-usage guard and the `estimateUsageCost` payload, keep focused regression coverage in `src/auto-reply/status.test.ts`, preserve changelog credit, and close #65109 when merged.
