## Summary

Keep this PR open. Current main still collects and displays cacheRead/cacheWrite for /status, but the /status cost-estimation path only treats input/output tokens as cost usage and only passes input/output to estimateUsageCost, so the focused PR remains useful.

## Best Possible Solution

Review and land this focused PR, or merge an equivalent change that updates src/status/status-message.ts to include cacheRead/cacheWrite in the has-usage guard and estimateUsageCost call, keeps the cache-inclusive regression tests in src/auto-reply/status.test.ts, and then closes the linked cache-cost bug #65109 while superseding the broader draft #52329 if appropriate.
