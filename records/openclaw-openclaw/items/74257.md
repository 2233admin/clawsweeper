Keep open. ## What This Changes

The issue asks OpenClaw to prevent Telegram topic approved-exec and internal async/heartbeat followups from leaking HEARTBEAT_OK or orchestration text after context-overflow compaction, while preserving the in-flight turn state.

## Best Possible Solution

Carry a first-class internal delivery intent through approved exec followup runs and overflow-compaction retries, then add a Telegram-topic regression proving internal HEARTBEAT_OK/orchestration text is suppressed while real command output still reaches the originating topic.
