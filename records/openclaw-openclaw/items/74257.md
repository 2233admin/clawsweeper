## Summary

Keep open. Current main still has only adjacent guardrails: approved exec completions resume through a normal agent turn, overflow recovery still compacts and retries the prompt, and simple HEARTBEAT_OK stripping does not prove the combined Telegram topic plus approval plus overflow path is fixed.

## What This Changes

The issue requests that Telegram topic approved-exec followups preserve internal/no-op delivery intent through context-overflow compaction so HEARTBEAT_OK and orchestration text are not sent as user-visible Telegram messages.

## Best Possible Solution

Add a generic runtime delivery-intent contract for internal async/no-op followup turns and carry it through approved exec continuation and compaction retry, with a Telegram forum-topic regression proving internal HEARTBEAT_OK/orchestration text is suppressed while real command output and failures still deliver.
