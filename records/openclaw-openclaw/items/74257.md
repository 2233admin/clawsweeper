## Summary

Keep open. Current main has useful partial guardrails for HEARTBEAT_OK stripping, Telegram topic routing, and busy-lane heartbeat deferral, but the reported combined path is still not proven fixed: approved exec followups still resume through the ordinary Gateway agent path, overflow recovery still compacts and retries the prompt, and current tests do not cover Telegram topic + approved exec followup + context overflow/compaction + internal/no-op delivery suppression as one scenario.

## What This Changes

The issue asks OpenClaw to prevent Telegram topic async exec followup and heartbeat/internal orchestration text from leaking into user-visible chat after context-overflow compaction and approval-expiry failures.

## Best Possible Solution

Keep this open and cluster it with #72918, #73149, and #66126. The best fix is a targeted runtime delivery-state contract for internal async/no-op followups that survives approved exec followup resume and context-overflow compaction retry, with a regression proving Telegram topic delivery does not render HEARTBEAT_OK or internal orchestration text while still preserving real exec output when user delivery is intended.
