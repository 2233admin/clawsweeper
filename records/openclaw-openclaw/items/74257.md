## Summary

Keep open. Current main has partial guardrails for heartbeat ack stripping, internal exec-event prompts, and Telegram topic routing, but the reported combined failure is not proven fixed: exec approval followups still re-enter the normal Gateway agent path, and overflow recovery still retries the prompt after compaction without an evident turn-state ledger for internal async/no-op delivery semantics.

## Best Possible Solution

Keep this open and cluster it with #72918, #73149, and #66126. The best fix is a targeted runtime contract that carries internal async/no-op delivery state through exec approval followups and context-overflow compaction retries, plus a regression test for Telegram topic delivery with an approved exec followup that overflows/compacts and must not render HEARTBEAT_OK or internal orchestration text.
