## Summary

Keep open. The PR targets a real QMD direct-file path bug, and the review discussion supports the intended behavior, but the branch is still stale against current main: the live resolver is package-owned while this PR edits the old source-side path, and the basename pattern still needs literal-safe handling for glob metacharacters.

## What This Changes

The PR detects QMD custom path entries that point to existing files, converts them into parent-directory collections with basename patterns, and adds a regression test in the old source-side host SDK path.

## Best Possible Solution

Implement the fix in the package-owned resolver, keep the source bridge unchanged, make direct-file patterns literal-safe, add package-level regression coverage for normal and metacharacter filenames, and add the user-facing changelog credit.
