## Summary

Keep open. The PR targets a real QMD file-path bug, but it is stale against current main: the resolver is now package-owned under `packages/memory-host-sdk`, while this branch still edits the old source-side file and leaves a literal-pattern gap for filenames containing glob metacharacters.

## What This Changes

The branch detects existing QMD custom path entries that point at files, converts them into parent-directory collections with basename patterns, and adds a regression test for that normalization.

## Best Possible Solution

Update the current package-owned resolver in `packages/memory-host-sdk/src/host/backend-config.ts`, keep the source-side bridge unchanged, add package-level regression coverage for normal and glob-metacharacter direct filenames, and preserve existing directory-path and naming behavior.
