## Summary

Keep open. Current main still has the direct-file QMD path bug, and this PR now targets the package-owned resolver instead of the obsolete source bridge, but the latest patch is still not merge-ready because it stores the raw basename as a QMD glob pattern. That fails the promised exact-file behavior for filenames containing glob metacharacters.

## What This Changes

The PR modifies the package-owned QMD backend config resolver to stat custom paths and convert existing file targets into parent-directory collections with basename patterns.

## Best Possible Solution

Patch the package resolver so direct file targets become parent-directory collections with a literal, glob-safe file pattern; preserve directory-path behavior, naming, and dedupe semantics; add regression coverage for both simple direct files and filenames containing glob metacharacters.

## Reproduction Assessment

Yes. The static reproduction path is a QMD config entry pointing at an existing file: current main emits the file as the collection root, while this PR emits a raw basename pattern that still misbehaves for names such as `notes[1].md`.

## Solution Assessment

No. Moving the fix to the package-owned resolver is the right surface, but the proposed implementation is not the narrowest correct fix until it escapes or otherwise represents direct-file basenames literally and restores package-level regression coverage.

## Review Findings
