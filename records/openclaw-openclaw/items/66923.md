## Summary

Current main already routes the reported OpenAI GPT-4.1 family through the Pi model registry, and the reported v2026.4.7 release already pinned a Pi registry version containing `openai/gpt-4.1`, `openai/gpt-4.1-mini`, and `openai/gpt-4.1-nano` rows with `openai-responses` metadata and 1,047,576-token context windows. The concrete IDs in this issue are managed registry models, not raw unmanaged pass-throughs.

## Best Possible Solution

Close this issue and keep the Pi-registry-backed behavior. If maintainers want a broader warning for deliberately configured custom provider fallback models, that should be a narrower follow-up separate from the GPT-4.1 registry rows.
