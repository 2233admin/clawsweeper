## Summary

Keep open: current main still lacks post-timeout compensation, and this PR remains paired with open bug #68065. The live code still converts an `agent.wait` timeout into a hard `sessions_send` timeout after the target run was accepted, while the discussion includes production confirmation and focused review feedback that should be resolved before landing.

## What This Changes

The PR adds a post-timeout compensation helper in `src/agents/run-wait.ts`, wires `sessions_send` to re-read latest assistant replies before returning timeout, and adds focused run-wait and sessions tests.

## Best Possible Solution

Keep this PR open as the implementation candidate for #68065. The desired end state is explicit `sessions_send` post-timeout handling that returns `ok` for a new post-baseline reply, returns `accepted` only when the send path has already proven the agent run was accepted, preserves hard `timeout` when acceptance is not proven, and keeps focused regression coverage for late replies, accepted-without-reply, compensation failure, and `timeoutSeconds: 0`.
