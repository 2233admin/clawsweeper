## Summary

Keep open. Current main still has the reported pre-aborted signal race, and the issue is paired with open PR #74979 using closing syntax, so cleanup should wait for that PR to be reviewed, merged, replaced, or closed.

## What This Changes

The issue reports that pre-aborted `params.abortSignal` can let `activeSession.prompt()` start a floating Pi Agent loop that continues LLM calls after the embedded attempt exits.

## Best Possible Solution

Review and land #74979, or an equivalent narrow replacement, so the prompt-start boundary checks the run abort signal before every `activeSession.prompt(...)` call and cleanup defensively cancels any escaped active Agent run; defer the configurable LLM-call cap unless maintainers want broader policy hardening.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` test shape, and current main still has the same call-order pattern; this read-only review did not run the test.

## Solution Assessment

Yes. The narrow maintainable fix is to make prompt startup abort-aware and add focused regression coverage; a parallel ClawSweeper repair PR would be counterproductive while #74979 is open.
