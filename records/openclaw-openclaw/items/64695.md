## Summary

Keep this issue open. Current main still defaults both the task registry and Task Flow registry to SQLite-backed stores that call `require("node:sqlite")`; the scheduled sweep rejection leak is mitigated, but Homebrew-style runtimes without `node:sqlite` would still fail to restore or persist durable task state.

## What This Changes

The issue asks OpenClaw to support gateway task and Task Flow persistence on Node runtimes where `node:sqlite` is unavailable, with either a replacement SQLite dependency or a graceful durable fallback.

## Best Possible Solution

Add a maintainer-approved runtime-safe storage selection layer for both task registries: probe `node:sqlite` before default use, keep the existing SQLite path where available, provide a durable fallback or supported dependency where unavailable, update the public storage docs, and cover restore, upsert/update, stale-task maintenance, and Task Flow persistence in regression tests.
