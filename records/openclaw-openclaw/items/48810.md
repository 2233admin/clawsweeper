## Summary

Keep this issue open. Current main reads Pi transcripts as a tree/DAG through SessionManager active-branch traversal, but I did not find the requested compaction-retry integrity guard: retry compaction entries are not superseded, deterministic, or reconciled, and existing tests still accept multiple retry-end events.

## Best Possible Solution

Keep this open for maintainer product/API judgment. The best fix is either a compaction-specific reconciliation contract, such as marking retry-superseded compaction entries or using deterministic retry compaction IDs, plus a regression test for duplicate same-parent compaction entries; or an explicit docs/API decision that downstream readers must use SessionManager active-branch traversal and must not treat parentId as a globally linear list.
