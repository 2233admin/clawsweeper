## Summary

Keep open. Current main still has the silent-success fallback this PR targets: a core `send` result without a `toolResult` is returned as `jsonResult(result.payload)`, and the direct send payload can have no concrete `result.messageId` when delivery produces no result. The author also gave a concrete Feishu production observation, and the related #717 fix only made existing `status:error` tool results count as failures; it did not add the missing-delivery guard proposed here. The PR patch is narrow and not security-sensitive based on the provided file list.

## Best Possible Solution

Keep this PR open, rebase it onto current main, preserve the missing-delivery guard in `src/agents/tools/message-tool.ts`, add focused regression coverage in `src/agents/tools/message-tool.test.ts` for missing delivery, dry-run bypass, and confirmed `messageId`, then run the targeted message-tool test and normal changed gate before merge.
