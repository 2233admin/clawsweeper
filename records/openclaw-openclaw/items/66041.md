## Summary

Keep this PR open. Current main still has the specific Telegram default-scope mismatch the PR targets: unmentioned group messages with `requireMention: false` can continue into processing, but the default `group-mentions` ack gate still rejects them. The linked issue was closed as implemented based on the explicit `messages.ackReactionScope` config surface, but that does not implement this PR's intended default behavior.

## What This Changes

This PR changes Telegram ack handling so default `group-mentions` behaves like `group-all` for Telegram groups with `requireMention: false`, adds a focused regression test, and documents the Telegram behavior.

## Best Possible Solution

Review and land this PR or an equivalent Telegram-owned fix if the default behavior change is intentional: keep the scope promotion local to Telegram, limit it to `group-mentions` plus `requireMention: false`, preserve explicit ack scopes, and retain the regression test and docs note.
