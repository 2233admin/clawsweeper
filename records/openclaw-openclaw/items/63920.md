Keep open. Current main still does not register Slack Bolt shortcut handlers or expose `shortcut` / `message_action` payloads through the bundled Slack channel, while the existing interaction path only covers block actions and modal lifecycle events. ## What This Changes

Expose Slack global shortcut and message-action shortcut events through the bundled Slack channel as sanitized system events for custom plugin workflows.

## Best Possible Solution

Add first-class Slack shortcut handling in the bundled Slack plugin: register Bolt shortcut listeners, parse global and message shortcut payloads into typed sanitized summaries, authorize the actor, resolve stable per-channel or per-user session/context keys, enqueue system events, and document the supported shortcut event types.
