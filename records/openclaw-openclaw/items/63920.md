## Summary

Keep open. Current main still does not expose Slack global shortcut or message-action shortcut payloads through the bundled Slack channel: the monitor registers block actions and modal lifecycle events only, the structured interaction payload type has no shortcut variants, docs omit shortcut events, and the plugin-facing Slack interactive API is still limited to button/select callbacks.

## What This Changes

The issue requests routing Slack global shortcut and message-action shortcut payloads from `channels.slack` into OpenClaw system events for agent/plugin use.

## Best Possible Solution

Add first-class shortcut handling in the bundled Slack plugin, or expose a typed plugin-facing Slack shortcut/raw-interaction API before implementing it externally. The bundled implementation should register Bolt shortcut handlers, acknowledge payloads, authorize the actor, resolve stable session/context keys, enqueue sanitized system events, and document the supported shortcut event types.

Use Bolt `app.shortcut` for both global `type: "shortcut"` and message `type: "message_action"` payloads, acknowledge with `ack()`, drop mismatched app/team payloads, apply existing Slack interactive sender authorization patterns, resolve session keys through `resolveSlackSystemEventSessionKey`, and enqueue sanitized stable system events with context keys.
