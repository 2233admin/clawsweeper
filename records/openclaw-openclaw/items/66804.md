## Summary

Keep open. Current main has useful Active Memory timeout mitigations, but it still does not implement the reported runtime fallback or local recovery path when MiniMax-M2.7 times out with no partial summary.

## What This Changes

Active Memory should complete, fail open, or fall back when MiniMax-M2.7 recall times out instead of returning no injected memory summary.

## Best Possible Solution

Keep the issue open until maintainers choose the intended Active Memory timeout policy: fail open within the configured budget, recover usable partial text when present, and, if desired, add an explicit timeout-fallback or local memory-search recovery setting without overloading the current selection-only modelFallback field.

## Reproduction Assessment

Yes. Configure Active Memory with MiniMax-M2.7 or minimax-portal/MiniMax-M2.7, enable logging, and watch for status=timeout with summaryChars=0; several reporters provided matching logs, though I did not run a live MiniMax call in this read-only review.

## Solution Assessment

No. Current main is a partial mitigation: it bounds the wait, disables thinking by default, can recover partial text, and adds a circuit breaker, but it still does not complete recall or switch to a backup/local path when the selected model times out with no partial text.
