## Summary

Keep open. Current main still lacks an MCP serve scope selector, while this PR is a focused implementation that appears consistent with the requested least-privilege bridge behavior; the remaining decision is maintainer review of auth-scope semantics and whether this supersedes #74030.

## What This Changes

This PR adds a repeatable `openclaw mcp serve --scope` option limited to `operator.read`, `operator.write`, and `operator.approvals`, threads selected scopes through the MCP server and bridge, updates MCP docs/changelog, and adds CLI/server regression tests.

## Best Possible Solution

Land this or an equivalent narrow implementation after maintainer review, preserving the default read/write/approvals behavior when `--scope` is omitted, keeping explicit MCP bridge scopes limited to the intended allowlist, documenting and testing the contract, and then resolving #74030 and any canonical linked report according to the chosen branch.
