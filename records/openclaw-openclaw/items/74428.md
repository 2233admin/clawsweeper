## Summary

Keep open. Current main still lacks configurable `openclaw mcp serve` scope selection, while this draft PR is a focused replacement for #74030 that narrows the proposed flag to the existing MCP bridge read/write/approvals scope set. The patch is not already implemented on main and should go through normal maintainer review, especially for auth-scope semantics and coordination with #74030/#73864.

## What This Changes

This draft PR adds a repeatable `openclaw mcp serve --scope` flag limited to `operator.read`, `operator.write`, and `operator.approvals`, threads selected scopes through the MCP server and bridge to `GatewayClient`, updates MCP docs, and adds CLI/server regression tests.

## Best Possible Solution

Land this or another narrow implementation after maintainer review, keeping the default read/write/approvals behavior unchanged when `--scope` is omitted, limiting explicit MCP bridge scopes to the intended least-privilege set, documenting the flag, adding any required changelog entry, and resolving #74030 as superseded if this PR replaces it.
