## Summary

Keep this PR open. The prompt hardening is still useful because current main lacks the exact-`<location>` skill-path guard, but the submitted patch is incomplete: it only protects the exactly-one-skill branch and leaves the multiple-skill branch and focused prompt assertions unchanged.

## What This Changes

The PR updates `buildSkillsSection()` so the exactly-one-skill instruction tells the agent to use the exact `<location>` value from `<available_skills>` and never guess, fabricate, or hard-code a skill file path.

## Best Possible Solution

Apply the exact-`<location>` / never-guess constraint consistently across both skill-selection branches in `buildSkillsSection()`, update the focused prompt tests for default and cased read-tool wording, and keep the deeper sandbox/discovery bugs tracked separately.

## Reproduction Assessment

Yes. The source-level reproduction is to build or inspect a prompt with `skillsPrompt`: current main lacks the guard, and the PR's unchanged multiple-skill branch would still instruct the agent to choose/read/follow without requiring the selected skill's exact `<location>`.

## Solution Assessment

No. A prompt-level guard is the right narrow layer for this PR, but the current patch is not the best form because it leaves the multiple-skill path and prompt assertions behind.
