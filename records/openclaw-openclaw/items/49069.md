## Summary

Keep this PR open. Current main still does not implement configurable user-facing timestamps in the terminal TUI chat log, and the linked timestamp feature request remains open. The PR is a focused implementation candidate, but the submitted diff is not merge-ready because it introduces an unschematized `tui.showTimestamps` config key, uses a non-existent `theme.muted` formatter, and does not preserve real message timestamps through streaming or history replay.

## Best Possible Solution

Keep this PR open as an implementation candidate for #48708, but require a revision before merge: add a typed/schema-backed and documented setting or settings toggle, use an existing theme formatter such as `theme.dim`, preserve assistant prefixes through streaming and final updates, use persisted transcript timestamps during history replay, and add focused ChatLog/history tests.
