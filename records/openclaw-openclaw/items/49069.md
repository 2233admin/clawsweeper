## Summary

Keep this PR open. Current main still does not implement configurable user-facing timestamps in the TUI chat log, and the linked feature request #48708 remains open. The PR is a focused implementation candidate, but the supplied diff is not merge-ready because it uses an unsupported config key, calls a non-existent theme formatter, and does not preserve/display timestamps consistently through streaming or history replay.

## Best Possible Solution

Keep this PR open as an implementation candidate for #48708. A solid revision should add a typed/schema-backed and documented TUI timestamp setting or settings toggle, use existing theme formatters such as `theme.dim`, preserve assistant prefixes through streaming and final updates, use persisted message timestamps during history replay, and add focused ChatLog/history coverage.
