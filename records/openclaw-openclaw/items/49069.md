## Summary

Keep this PR open. Current main still does not implement configurable TUI chat message timestamps: ChatLog renders system/user/assistant text without timestamp decoration, runTui does not pass any timestamp setting into ChatLog, history replay ignores persisted message timestamps, and the strict config schema has no `tui.showTimestamps` surface. The PR remains a plausible implementation candidate for the still-open linked feature request, but its current diff has known functional review blockers.

## Best Possible Solution

Keep the PR open as an implementation candidate for the TUI timestamp feature. The best path is a revised patch that adds a typed config/docs surface, uses persisted transcript/event timestamps where available, preserves assistant prefixes through streaming and finalize paths, styles timestamps with an existing theme formatter, and adds focused ChatLog/session-history tests.
