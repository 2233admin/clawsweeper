## Summary

Keep open. Current main still implements hard sub-agent run timeouts plus post-timeout partial-progress recovery, but it does not implement the requested configurable pre-timeout warning/grace message for native or ACP sub-agents. The closed partial-result work in #37250 is useful fallback behavior, but it does not give the child agent a chance to checkpoint before the timeout.

## Best Possible Solution

Keep this open for core sub-agent lifecycle design. Add a documented global grace-warning setting and, if accepted, a per-`sessions_spawn` override; schedule a warning before `runTimeoutSeconds` expires; deliver it through a non-disruptive child-session control/message path for native and ACP runs where feasible; and cover config schema, tool schema, native spawn, ACP spawn, timeout race behavior, docs, and existing post-timeout partial-progress recovery with focused tests.
