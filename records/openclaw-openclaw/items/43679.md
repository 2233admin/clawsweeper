## Summary

Keep this PR open. Current main still lacks the PR's shared suspicious-orchestration leakage detector: the shared normalizer only handles empty, silent-token, heartbeat, and sanitizer paths, so route-marker/tool-JSON text can still pass through outbound reply delivery as normal user-visible text. The PR discussion is useful, and the author addressed several false-positive and bypass review comments, but the branch still needs maintainer review and adaptation to current main.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, adapt the narrow suspicious-reply guard to the current shared `normalizeReplyPayload()` flow, and land it with regression tests that preserve existing NO_REPLY, media/channelData, heartbeat, sanitizer, outbound planner, and explanatory-prose behavior.
