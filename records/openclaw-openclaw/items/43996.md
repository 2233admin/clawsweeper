Keep open. Current main still applies Docker `no-new-privileges` unconditionally to sandbox containers, has no config opt-out or fallback retry, and an open unmerged PR already links this issue as its fix candidate.

## What This Changes

The issue requests a Docker sandbox compatibility path for hosts where `--security-opt no-new-privileges` makes sandbox containers fail to execute `/usr/bin/sleep`.

## Best Possible Solution

Review, land, or replace PR #70229 with a secure-default Docker sandbox `noNewPrivileges` setting that is threaded through config types, strict schema, config resolution, Docker create args, dangerous-config/security audit surfaces, docs, and regression tests; close this issue only after that fix merges and ships.
