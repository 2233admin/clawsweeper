## Summary

Keep this PR open. Current main still reloads channel config at whole-channel granularity, while this PR proposes account-scoped planning and dispatch for strict `channels.<channel>.accounts.<accountId>.*` changes. It is also paired with the same author's open issue #43935 via closing syntax, so it remains an implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review/land it if it adds an account-scoped reload plan representation, routes strict `channels.<channel>.accounts.<accountId>.*` changes through `stopChannel(channel, accountId)` and `startChannel(channel, accountId)`, preserves full-channel precedence for channel-global or mixed changes, updates all reload-plan consumers, and adds focused planner plus handler regression tests.
