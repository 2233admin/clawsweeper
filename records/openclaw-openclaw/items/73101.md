## Summary

Keep this PR open. Current main still lacks Slack's configured ACP binding adapter, configured-binding inbound fallback, readiness/drop handling, and regression coverage for unsuffixed configured ACP session keys. The provided PR remains a focused implementation candidate; its listed diff is limited to bundled plugin source/tests and unrelated test-only lint cleanup, with no CI, dependency, release, or package-resolution changes in the supplied file list.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to land a focused Slack-owned fix that registers Slack as a configured ACP binding provider, resolves configured ACP routes only when no runtime conversation binding is active, gates dispatch with `ensureConfiguredBindingRouteReady`, drops unavailable configured ACP targets instead of falling back to `main`, and keeps regression coverage for stable configured ACP session keys across Slack channel/thread cases.

That means the generic configured ACP binding compiler has nothing Slack-specific to call.
