## Summary

Keep this PR open. Current main covers Xiaomi image understanding through the generic image-capable provider path, but it does not implement Xiaomi video understanding, which is part of the PR’s stated purpose. The PR also needs a rebase onto the current Xiaomi plugin and SDK shape before it can be merged safely.

## Best Possible Solution

Keep the PR open and ask for a rebase/rework. The best path is a current Xiaomi media-understanding provider that registers through the modern plugin entry, declares manifest `mediaUnderstandingProviders` metadata, uses the shared OpenAI-compatible video helpers plus `provider-http`, selects `mimo-v2-omni` for Xiaomi image/video understanding instead of the text-only default, and adds focused tests for default Xiaomi image and video routing.
