## Summary

Keep open. Current main partially addresses the reported managed runtime dependency failures for `sqlite-vec` and configured plugin runtime deps after the latest release, but the central local embeddings failure remains: `node-llama-cpp` is still imported as a bare runtime package, is not declared in root dependencies or mirrored runtime deps, and the current docs/error path still points users at a global install that the managed runtime mirror does not reliably resolve.

## What This Changes

Ensure local `memorySearch` runtime dependencies are retained or resolved inside OpenClaw's managed `plugin-runtime-deps` environment instead of requiring unstable manual/global installs.

## Best Possible Solution

Make the local memory provider's runtime contract explicit: when `provider: "local"` is selected, OpenClaw should either install and retain `node-llama-cpp@3.18.1` in the managed runtime-deps plan alongside `chokidar` and `sqlite-vec`, or replace the current global-install advice with a resolver/repair path that is actually reachable from the mirrored runtime root.
