## Summary

Keep open. Current main still lacks the DeepSeek provider thinking-profile hook, so the report matches the source; because open PR #73008 already references this issue with closing syntax and proposes the narrow provider/test/docs fix, this issue should stay open until that PR or an equivalent patch lands.

## What This Changes

Request that the native DeepSeek provider expose a V4 thinking profile so `/think xhigh` and `/think max` are accepted for `deepseek-v4-flash` and `deepseek-v4-pro`.

## Best Possible Solution

Land PR #73008 or an equivalent provider-local patch that adds a DeepSeek V4 `resolveThinkingProfile` for Flash/Pro, leaves non-V4 models on the base profile, and adds regression coverage for exposed levels plus the existing payload mapping.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: selecting `deepseek/deepseek-v4-pro` or `deepseek/deepseek-v4-flash` and requesting `/think max` reaches the base-profile validator before the DeepSeek stream wrapper, so `max` is rejected on current main.

## Solution Assessment

Yes. The narrowest maintainable fix is in the DeepSeek plugin's provider profile, not a core fallback that would advertise `max` for unrelated providers or models.
