## Clownfish Work Prompt

Implement a conservative fix for this issue. Current main live builtin `openclaw memory search` calls `MemoryManager.search()` -> `embedQueryWithTimeout()` -> `provider.embedQuery()` with only a timeout; remote OpenAI-compatible/OpenRouter `fetchRemoteEmbeddingVectors()` and Gemini `withRemoteHttpResponse()` do one guarded POST, while the existing retry classifier only covers rate/5xx-style failures and batch HTTP retry is status-only. Add a narrow transient transport classifier for `fetch failed`, `other side closed`, `ECONNRESET`, `EPIPE`, `UND_ERR_SOCKET`, `socket hang up`, `terminated`, and `Client network socket disconnected before secure TLS connection was established`, then apply bounded retry/backoff to the live single-query remote embedding path. Prefer a shared memory-host helper so OpenAI-compatible/OpenRouter and Gemini use the same policy; keep auth/config/4xx errors immediate. Coordinate with #44166/#44167 batch work, but do not change provider selection, config schema, cache format, QMD lock handling (#66339), voice-call fast context (#71849), or broad fetch-guard/SSRF policy unless focused tests prove it is required. Update `CHANGELOG.md` with a single-line user-facing fix entry crediting @kevinheinrichs and any other reporter credits the maintainer chooses.
