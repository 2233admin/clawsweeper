## Summary

Keep open. Current main still routes live builtin memory search through a single query embedding call with only timeout handling, while the existing retry helper covers batch embedding and the retry classifier still excludes the reported Undici/TLS/socket failure strings. The later issue comments also confirm the same failure class after the latest shipped release, and the related open batch/reindex PR does not by itself cover this live query path.

## What This Changes

The issue asks to make live builtin memory search resilient to intermittent remote embedding transport and socket failures from the SSRF-guarded fetch path across providers.

## Best Possible Solution

Add a conservative shared transient transport-error classifier and bounded retry/backoff around live query embeddings used by builtin memory search, while preserving fail-fast behavior for auth, config, schema, validation, and HTTP 4xx failures. Keep Undici dispatcher tuning and user-facing retry config as follow-up unless targeted evidence shows the narrow retry is insufficient.

## Reproduction Assessment

Yes. The issue provides concrete provider configuration and a repeatable `openclaw memory search` loop, two later comments confirm the same failure class on upgraded installs/providers, and static inspection shows current main still has a single-shot live query embedding path.

## Solution Assessment

Yes. A narrow live-query retry for known transient transport errors is the smallest maintainable first fix; broad dispatcher pool tuning or a new config knob would be riskier without more focused proof.

Current main has live builtin memory search flowing through `MemoryManager.search()` -> `embedQueryWithTimeout()` -> provider `embedQuery()` as a single-shot call; batch embedding uses `runMemoryEmbeddingRetryLoop(...)`, but live query embedding does not. Add a conservative bounded retry/backoff for live query embedding transport failures such as `fetch failed`, `other side closed`, `ECONNRESET`, `EPIPE`, `UND_ERR_SOCKET`, `socket hang up`, `terminated`, and `Client network socket disconnected before secure TLS connection was established`. Coordinate with #44167/#44166: reuse or extend shared classifier code if appropriate, but do not duplicate or broaden the batch split behavior unless needed.
