## Summary

Keep open. The PR is a narrow, apparently correct dashboard fix: current main still lacks a manual-auth recovery hint when tokenized URL delivery fails, and the diff preserves the token-redaction contract while adding focused coverage.

## What This Changes

The PR adds a dashboard manual-auth fallback log when clipboard and browser delivery of a tokenized URL both fail, updates dashboard link tests for redaction and `--no-open`, and adds a changelog entry.

## Best Possible Solution

Keep this PR open, rebase it if maintainers choose it as the canonical fix, and land one CVE-safe dashboard implementation that keeps token values out of logs while giving headless/WSL users enough information to authenticate manually.

## Reproduction Assessment

Yes. Configure a literal gateway token, force clipboard copy to fail, and run a headless/SSH/WSL path where browser open is unavailable; current main logs only the bare URL plus SSH/no-open guidance, while the PR adds a test for that combined-failure path.

## Solution Assessment

Yes. The proposed `fallbackToManualAuth` gate is owner-local and narrow: it only fires when a tokenized URL was intended and neither delivery channel succeeded, while preserving the existing token and `#token=` log redaction contract.
