## Summary

Keep this issue open. Current main preserves inter-session provenance for persistence and later replay, but the live A2A reply loop still feeds the other session's latest reply into the requester/peer run as the active prompt text. The model-visible same-turn input is not converted to a tool result and is not wrapped with a synthetic untrusted per-message envelope before `activeSession.prompt(...)` runs. Existing tests cover provenance metadata and replay annotation, not the reported same-turn prompt-injection boundary.

## Best Possible Solution

Keep the issue open and harden `sessions_send`/`runAgentStep` so foreign-session replies are model-visible as untrusted data before the receiving model call, preferably as a tool-result-like payload or a synthetic inter-session envelope on the active prompt. Add a regression test that captures the requester-side A2A reply step input and proves the foreign reply cannot be submitted as an unmarked current user instruction.

## Risks / Open Questions

- Closing now would treat persisted/replay-time provenance as equivalent to same-turn prompt isolation, which the current code does not prove.
- The affected surface is cross-session agent integrity; a regression test should capture the actual model-bound input for the requester-side A2A reply step before this is considered fixed.
