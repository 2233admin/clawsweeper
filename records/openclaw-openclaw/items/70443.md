## Summary

Keep open. Current main still lacks a Docker setup offline rerun mode: the script has no `--offline` parsing, still builds or pulls the main image before continuing, still builds the optional sandbox image when enabled, and docs/tests do not cover a preloaded-image airgapped workflow. The reporter’s linked patch is useful prior art, but it has not landed in this repository and the related platform/buildx items do not supersede this request.

## What This Changes

Add an explicit `--offline` flag to `scripts/docker/setup.sh` so airgapped operators can rerun setup with preloaded Docker images while preserving env sync, permissions, onboarding, gateway config sync, compose startup, and sandbox handling.

## Best Possible Solution

Implement the narrow `--offline` flag in `scripts/docker/setup.sh`: parse it early, skip main build/pull only in offline mode, require `docker image inspect "$OPENCLAW_IMAGE"` for the main image with clear `docker load` guidance when missing, warn rather than fail for the optional sandbox image, and leave env validation, `.env` upsert, permission fixes, onboarding, gateway config sync, compose startup, and sandbox overlay/config behavior otherwise unchanged. Add focused e2e coverage, Docker docs, and a user-facing changelog entry.
