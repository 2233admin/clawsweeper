## Summary

Keep this open. Current main still has no `--offline` mode for Docker setup: `scripts/docker/setup.sh` does not parse the flag, still unconditionally builds `openclaw:local` or pulls non-local images, and still builds the sandbox image when sandbox setup is enabled. The reporter's linked workaround confirms the desired direction, but it is not the final repo-ready shape because it changes default local-image behavior and adds an unrelated `--force-recreate`.

## What This Changes

Add an explicit `--offline` mode to `scripts/docker/setup.sh` that skips build/pull, verifies a preloaded gateway image, treats the optional sandbox image as a warning-only preload, and leaves the rest of setup intact.

## Best Possible Solution

Implement the flag in the Docker setup script as a narrow, opt-in control: parse only `--offline`, keep default build/pull behavior unchanged, require `docker image inspect "$OPENCLAW_IMAGE"` before setup proceeds, warn rather than fail for a missing optional sandbox image, and document/test the supported `docker load` rerun workflow.

## Reproduction Assessment

Yes. A high-confidence reproduction path exists from current source: run `./scripts/docker/setup.sh --offline` after preloading an image and disabling network, and the script will ignore the flag and still execute the existing build or pull branch before the rest of setup.

## Solution Assessment

Yes, the requested flag is the narrowest maintainable solution if implemented exactly as an opt-in setup mode. The linked workaround is useful prior art, but the repo fix should not change default behavior or include the unrelated gateway `--force-recreate` change.
