## Summary

Keep #70443 open. Current main still lacks a `--offline` or equivalent Docker setup path: `scripts/docker/setup.sh` always builds `openclaw:local` or pulls a non-local `OPENCLAW_IMAGE` before continuing, and sandbox setup still builds the sandbox image when enabled. The related #69143/#69314 context is cross-platform build support and was explicitly scoped separately from this offline rerun request, so it does not supersede this issue.

## Best Possible Solution

Implement the focused `--offline` option in `scripts/docker/setup.sh` with Bash 3.2-compatible argument parsing. When set, skip the main build/pull only after `docker image inspect "$OPENCLAW_IMAGE"` succeeds, fail clearly with `docker load` guidance if it does not, apply equivalent warn-only handling for the optional sandbox image, and leave env synchronization, permission repair, onboarding, gateway config sync, compose startup, and sandbox overlay handling unchanged. Add targeted coverage in `src/docker-setup.e2e.test.ts` plus a short Docker install docs entry.
