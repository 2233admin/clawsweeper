## Summary

Keep open. Current main still lacks a `--offline` Docker setup path: `scripts/docker/setup.sh` always builds `openclaw:local` or pulls a non-local `OPENCLAW_IMAGE` before continuing, sandbox setup still builds the sandbox image when enabled, and neither tests nor Docker docs cover a preloaded-image offline rerun. The related platform/buildx items are closed and do not supersede this request.

## Best Possible Solution

Implement the narrow `--offline` flag in `scripts/docker/setup.sh`: parse it in a Bash 3.2-compatible way, skip main build/pull only when `docker image inspect "$OPENCLAW_IMAGE"` succeeds, fail clearly with `docker load` guidance when the main image is missing, warn rather than fail for the optional sandbox image, and leave the rest of setup unchanged. Add focused `src/docker-setup.e2e.test.ts` coverage, document the flag at `https://docs.openclaw.ai/install/docker`, and add a user-facing changelog entry.

Leave every other setup step intact: env validation, `upsert_env`, data-directory chown, onboarding or `OPENCLAW_SKIP_ONBOARDING`, `sync_gateway_config`, compose startup, sandbox overlay/config behavior, and default behavior when `--offline` is not passed.
