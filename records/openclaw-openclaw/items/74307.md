## Summary

Keep open. Current main intentionally materializes `dist/extensions/node_modules/openclaw/plugin-sdk` as a wrapper directory, not a symlink, but the report still points at a real likely repair gap: doctor’s bundled runtime-deps preflight filters configured plugins through channel/plugin enablement and does not appear to include provider-only bundled plugins such as `anthropic-vertex`, even though that plugin owns `@anthropic-ai/vertex-sdk` as a staged runtime dependency.

## What This Changes

The issue requests that `openclaw doctor --fix` converge the npm/macOS bundled plugin runtime-deps repair so the staged `plugin-sdk` exposure is correct or the repair fails loudly instead of reporting success with a broken staged dependency shape.

## Best Possible Solution

Keep this open and fix the doctor/runtime-deps repair around provider-owned bundled plugins: preserve the current wrapper-directory `plugin-sdk` alias unless maintainers intentionally change that contract, but make `doctor --fix` include configured provider/model owners such as `anthropic-vertex`, install their staged runtime dependencies, and fail loudly if the staged dependency set still cannot satisfy the active provider runtime after repair.
