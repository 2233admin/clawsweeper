## Summary

Current main still does not add the requested Control UI bootstrap or CLI inspection hints to the exec-approval unavailable reply. The useful implementation candidate, PR #53347, was closed unmerged, so this issue should remain open rather than being closed as implemented or duplicate.

## Best Possible Solution

Keep this open. Implement a small core UX change in `src/infra/exec-approval-reply.ts` that preserves channel-specific native approval setup wording while adding concrete recovery steps for `openclaw dashboard --no-open`, `openclaw approvals get`, and the relevant `channels.<channel>.execApprovals.enabled` path. Add focused coverage in `src/infra/exec-approval-reply.test.ts`; PR #53347 is useful prior art but was closed unmerged.
