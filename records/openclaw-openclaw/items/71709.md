## Summary

Current main still matches the reported failure path. The session-memory slug helper calls `runEmbeddedPiAgent` on `sessionKey: "temp:slug-generator"` without `authProfileFailurePolicy: "local"`, while the embedded runner already supports local auth-health scoping and still persists classified billing/auth failures when the policy is not local. This is a narrow, valid fix candidate.

## Best Possible Solution

Keep this open and fix the slug-generator call site by passing `authProfileFailurePolicy: "local"` to `runEmbeddedPiAgent` in `src/hooks/llm-slug-generator.ts`. Add a focused regression test in `src/hooks/llm-slug-generator.test.ts` that asserts the helper passes local auth failure policy, preserving shared auth failure persistence for normal user-facing embedded runs.
