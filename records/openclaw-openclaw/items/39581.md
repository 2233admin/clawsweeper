## Summary

Keep this PR open. Current main still emits the embedded-runner llm_output hook whenever the hook is registered, even if the current attempt produced no assistant output. The PR targets a real bug, but its lastAssistant guard is not quite the right boundary because lastAssistant is full-history state, and the branch also bundles unrelated chat-content parsing changes with review concerns.

## Best Possible Solution

Keep the PR open but request a focused update: guard embedded-runner llm_output on actual current-attempt assistant output, likely using currentAttemptAssistant or non-empty current-attempt assistantTexts rather than full-history lastAssistant; preserve normal-output hook emission; add embedded-runner regression coverage for timeout/no-output and normal-output cases; and split or drop the unrelated chat-content parser change unless it is fixed and justified separately.
