## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] List every approval action in pending text:** `src/infra/exec-approval-reply.ts:282`
  - body: `buildExecApprovalPendingText` now renders only `descriptors[0]` and a full-ID version of that same decision. Text-only approval flows and copied logs no longer show the other allowed actions, especially `deny`, so an operator may be pushed toward approval when they meant to deny or choose another allowed decision.
  - confidence: 0.94
- **[P1] Do not reuse session snapshots across yielded sweeps:** `src/tasks/task-registry.maintenance.ts:381-392`
  - body: `runTaskRegistryMaintenance` creates one session-store cache for the whole sweep, but the loop yields to the event loop between batches. If a backing session appears after a yield, later `shouldMarkLost` calls still read the stale cached store and can mark active tasks `lost`, mutating task state incorrectly.
  - confidence: 0.84
- **[P2] Disable hidden-event stripping for generic rendering:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMarkdownPreprocessor.swift:49-54`
  - body: The preprocessor now strips hidden exec/system lines by default, but `ChatMarkdownRenderer` still calls `preprocess(markdown:)` without passing a role-aware flag. Even though `ChatViewModel` preserves user-authored quoted exec logs during decode, normal rendering will strip those lines from user messages.
  - confidence: 0.88
