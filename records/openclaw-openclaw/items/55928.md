## Summary

Keep this PR open. Current main still serves the exposed HTTP history path without an internal namespace guard, and the latest branch addresses that gap without a blocking correctness or security finding; it should not be closed by cleanup automation.

## What This Changes

The branch adds an internal gateway-session-key predicate, applies it to the HTTP session-history handler, and extends gateway tests for raw/canonical internal keys while preserving WebSocket chat.history.

## Best Possible Solution

Land this focused HTTP-only guard, or an equivalent main-branch patch, while keeping WebSocket history policy separate until maintainers define a narrower caller-trust model that will not break runtime subagent control flows.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction against current main is to seed an internal entry such as agent:main:cron:daily, request /sessions/cron%3Adaily/history with operator.read, and observe that current main canonicalizes and reads the transcript instead of returning 403.

## Solution Assessment

Yes. The PR is a narrow maintainable solution for the HTTP surface: it blocks raw and canonical internal keys before transcript reads, preserves ordinary history reads, and documents the intentionally separate WebSocket behavior with regression coverage.
