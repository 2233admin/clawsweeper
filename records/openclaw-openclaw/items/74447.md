## Summary

Keep open. Current main still has a populated required Knip unused-file allowlist, and this PR is not merge-ready because it moves the stale required entries into the optional allowlist, where they are allowed if reported but are not checked for staleness.

## What This Changes

The PR empties `KNIP_UNUSED_FILE_ALLOWLIST` in `scripts/deadcode-unused-files.allowlist.mjs` and moves the former required paths, plus `src/gateway/test/server-sessions-helpers.ts`, into `KNIP_OPTIONAL_UNUSED_FILE_ALLOWLIST`.

## Best Possible Solution

Keep the deadcode guard strict: delete stale production paths from both allowlists unless a path has a documented platform-dependent Knip finding, keep the optional allowlist narrow, and merge only after deadcode and formatting checks pass on the PR head.
