## Summary

Keep open. Current main still has only workspace-scoped ClawHub skill install/update behavior, while shared skill roots are already supported for loading. The linked #74466 is an open, unmerged closing PR for this exact request, so this issue should remain open until that PR lands or is superseded.

## What This Changes

Add an explicit global/shared install target for `openclaw skills install`, such as `--global` or `--target global`, so ClawHub skills can be installed into `~/.openclaw/skills` instead of an agent workspace.

## Best Possible Solution

Land a narrow, explicit shared-target implementation for native ClawHub skill install/update that writes to the managed shared root, preserves workspace defaults, rejects ambiguous `--agent` combinations, reuses existing slug/path validation and metadata tracking, and updates CLI plus skills docs.

## Reproduction Assessment

Yes. The current-main code path deterministically resolves a workspace and installs under `<workspace>/skills`; the missing `--global` flag is visible in the registered command options and docs without needing a write-producing live install.

## Solution Assessment

Yes. An opt-in CLI flag such as `--global` is the narrowest maintainable direction because shared load roots already exist; the linked PR is the current candidate, but it remains unmerged and should be reviewed before closure.
