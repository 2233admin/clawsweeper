## Summary

Keep open. Current main still has only workspace-targeted ClawHub skill install/update paths, and the linked closing PR #74466 is still open and unmerged, so the issue should remain open until that PR lands or is superseded by an equivalent fix.

## What This Changes

Add an explicit global/shared install target for `openclaw skills install` so ClawHub skills can be installed outside the active agent workspace while preserving the current workspace default.

## Best Possible Solution

Review and land #74466, or replace it with an equivalent narrow PR, adding an explicit opt-in shared install/update target while preserving workspace defaults and covering CLI behavior, installer targeting, lock tracking, docs, and conflict cases.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists on current main: the CLI has no global/target option, passes only `workspaceDir`, and the installer resolves targets under the active workspace skills directory.

## Solution Assessment

Yes. An explicit opt-in shared target is the narrowest maintainable direction; changing the default workspace behavior would be broader and risk surprising existing multi-agent setups.
