## Summary

Keep open. Current main still installs and updates ClawHub skills through the active workspace path only, while shared skill roots are already loadable; the remaining work is the explicit global/shared install target. An open PR, #74466, already references and attempts this fix, so this issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

Request to add an explicit global/shared target, such as `--global`, for `openclaw skills install` so ClawHub skills can be installed under shared skill roots instead of only active agent workspaces.

## Best Possible Solution

Land a reviewed implementation that adds an explicit shared install target for ClawHub skills, keeps workspace installs as the default, updates matching install/update docs and tests, and closes this issue only after the implementation reaches main.

## Reproduction Assessment

Yes. Current-main source inspection is enough: `openclaw skills install` exposes no global/target option and the installer resolves every ClawHub skill target as `<workspace>/skills/<slug>`.

## Solution Assessment

Yes. An opt-in `--global` target for the managed shared skills root is the narrowest maintainable direction; a config default can remain a follow-up after the explicit CLI behavior is merged.
