## Summary

Keep open. Current main still lacks an explicit global/shared install target for `openclaw skills install` or `openclaw skills update`, and the remaining work is already paired with open closing PR #74466, so this issue should stay open until that PR lands or is superseded.

## What This Changes

Add an explicit `openclaw skills install` target such as `--global` or `--target global` so ClawHub skills can install into shared global skill roots instead of only active agent workspaces.

## Best Possible Solution

Land #74466 or an equivalent narrow replacement that adds an explicit opt-in global/shared target for install and update, keeps existing workspace defaults unchanged, writes metadata consistently for shared roots, documents precedence, and covers CLI conflict behavior with focused tests.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction is available from current main: the CLI command registration has no global/target option, and the installer resolves targets under `path.resolve(workspaceDir)/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
