## Summary

Keep open. Current main still exposes only workspace-scoped ClawHub skill install/update behavior, and the linked implementation PR is open, unmerged, and currently dirty, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests an explicit global/shared target for `openclaw skills install` and related updates so ClawHub skills can be installed under shared skill roots such as `~/.openclaw/skills` instead of only the active workspace.

## Best Possible Solution

Land #74466 after review and conflict resolution, or replace it with an equivalent narrow PR that adds an explicit shared/global install and update target while leaving the workspace default unchanged and documenting the new behavior.

## Reproduction Assessment

Yes. A high-confidence non-mutating reproduction is available by inspecting current main: the CLI exposes no global/target option, passes only `workspaceDir`, and the installer resolves install/update targets under `<workspace>/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared flag or target is the narrowest maintainable direction; changing the default workspace target would be broader and risk surprising existing multi-agent setups.
