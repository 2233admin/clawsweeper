## Summary

Keep open. Current main still has only workspace-targeted ClawHub skill install/update behavior, and the open closing PR #74466 is the right implementation candidate to review before this issue can be closed.

## What This Changes

Add an explicit global/shared target for `openclaw skills install` so ClawHub skills can be installed to `~/.openclaw/skills` instead of only the active agent workspace.

## Best Possible Solution

Review and land #74466, or replace it with an equivalent narrow PR, adding an explicit shared install/update target while preserving the current workspace default and covering CLI behavior, installer targeting, docs, and conflict cases.

## Reproduction Assessment

Yes. The gap is reproducible by source inspection on current main: the CLI has no global/target option, passes only `workspaceDir`, and the installer resolves every ClawHub skill target below `<workspace>/skills`.

## Solution Assessment

Yes. An explicit opt-in `--global` or target flag is the narrowest maintainable solution; changing the default workspace behavior would be broader and risk surprising existing multi-agent setups.
