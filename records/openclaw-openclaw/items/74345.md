## What This Changes

The issue reports a 2026.4.27 gateway reliability regression where idle task-registry ACP cleanup loops, holds session-write locks for minutes, saturates the event loop, and stalls Telegram/Discord transports.

## Best Possible Solution

Add a focused gateway reliability fix that bounds task-registry ACP maintenance work: limit or time-box runtime close attempts, avoid retrying the same failed stale ACP close every sweep without cooldown or quarantine, keep clearing only safe stale parent-owned metadata, preserve active persistent bindings, and add regression coverage for failed/hung stale ACP cleanup plus session-lock/liveness behavior.
