## Summary

Keep this PR open. Current main still does not implement `channels.signal.configPath`, and the PR remains a useful feature candidate, but it targets the removed `src/signal/*` layout instead of the current bundled Signal plugin under `extensions/signal`. The latest maintainer discussion also explicitly kept it open for takeover after the contributor said they could not continue.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port the feature to the current bundled Signal plugin by adding optional `channels.signal.configPath` to the Signal config type, strict schema, plugin UI/help metadata, account configured detection, and Signal docs; propagate it through `monitorSignalProvider` into `spawnSignalDaemon`; and add focused tests proving `--config <path>` is emitted only for nonblank values, including account-level config behavior if supported.
