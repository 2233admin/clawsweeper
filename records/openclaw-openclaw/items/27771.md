## Summary

Keep this PR open. Current main still does not implement `channels.signal.configPath` for the bundled Signal plugin, and the submitted branch targets the old `src/signal` layout rather than the current `extensions/signal` plugin. The author has invited maintainer takeover, so the useful path is a port/replacement patch, not cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port optional `channels.signal.configPath` and `channels.signal.accounts.<id>.configPath` into the current Signal plugin by updating the exported config type, strict schema/generated metadata, UI/help hints, docs, configured-account detection, monitor autostart propagation, and daemon argv builder. Add focused tests for top-level, per-account, unset, and blank values, and add a user-facing changelog entry crediting `Thanks @NorseGaud` if accepted.
