## Clownfish Work Prompt

Port optional Signal `configPath` support from the stale PR into current main. Implement both `channels.signal.configPath` and `channels.signal.accounts.<id>.configPath` for the bundled Signal plugin. Keep the change scoped to Signal config/runtime/docs/tests.

Expected behavior: when Signal autostart launches `signal-cli`, a nonblank configured config path is passed as global args before the subcommand, i.e. `signal-cli --config <path> ... daemon ...`; blank/whitespace values must be omitted. Top-level config should apply to the default account, and per-account config should override/merge through the existing account resolution path. The configured path is local Signal account state, so pass it as a separate argv value and avoid logging it unnecessarily.

Do not revive or edit old `src/signal/*` files.

Add tests for schema acceptance at top level and per-account, account configured detection, daemon argv ordering/blank omission, and monitor autostart propagation. If accepted, add a single-line user-facing changelog entry with `Thanks @NorseGaud`.
