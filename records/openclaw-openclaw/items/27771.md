Keep this PR open. Current main still does not implement the requested `channels.signal.configPath` behavior, and the existing discussion already identifies it as a valid Signal plugin feature needing maintainer takeover rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port the feature to the current bundled Signal plugin by adding optional `channels.signal.configPath` to the Signal config type, strict schema, account resolution/configured detection, UI/help metadata, and public Signal docs; propagate it through `monitorSignalProvider` into `spawnSignalDaemon`; and add focused tests proving `--config <path>` is emitted as separate argv elements only for nonblank values, including the multi-account config path if supported.
