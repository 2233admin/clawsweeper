## Summary

Keep this PR open. Current main still lacks `channels.signal.configPath` across the Signal config contract, strict schema, docs, and daemon startup path. The PR is a useful implementation candidate, but it targets older `src/signal/*` paths while current main uses the bundled `extensions/signal/*` plugin layout, so maintainer takeover or a replacement patch is the right path.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port the feature to the current bundled Signal plugin by adding optional `channels.signal.configPath` to the Signal config type, strict schema, generated/channel metadata, UI/help hints, account configured detection, and public Signal docs; propagate it through `monitorSignalProvider` into `spawnSignalDaemon`; and add focused tests proving `--config <path>` is emitted only for nonblank values, including per-account config if supported.
