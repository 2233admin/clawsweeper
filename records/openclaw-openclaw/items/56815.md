## Summary

Keep open. The report is recent, concrete, and current main still does not implement the central fixes: memory embedding retry only recognizes rate-limit/HTTP-status-shaped messages, async memory sync warnings still flatten errors with String(err), and safe reindex still removes the temporary database on any failed build. The related #56879 PR was a relevant attempt but is closed/unmerged, and the later surrogate-pair issue/PR is a different open thread rather than a shipped fix for this fetch-failed/progress-loss path.

## Best Possible Solution

Keep this issue open. The best fix is a memory-core change that classifies nested transport failures such as fetch failed, ECONNRESET, ENOTFOUND, ETIMEDOUT, and UND_ERR_* as bounded retryable embedding errors; wraps remote embedding transport failures with provider/context while preserving cause detail; logs async sync failures with stack/cause-safe formatting; and separately designs a conservative checkpoint/resume or committed embedding-cache path so failed long reindexes do not waste all successful API work.
