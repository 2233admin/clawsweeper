## Summary

Keep open. Current main still does not fully solve the reported large memory reindex failure mode: transient `fetch failed`/transport errors are not classified as retryable, detached memory sync warnings still flatten errors with `String(err)`, and failed atomic reindex still deletes the temp database. Related #44167 is an open implementation candidate for the retry/split subset, while #26772 tracks the checkpoint/resume portion.

## Best Possible Solution

Keep this issue open. Review and land #44167 or an equivalent memory-core change for narrow transient transport retry and split fallback, then address the remaining issue-specific pieces: cause/stack-aware redacted sync warnings and a safe checkpoint/resume or failed-temp preservation design that keeps atomic swap guarantees. Coordinate the resume design with #26772, and close this issue only after those pieces land or a maintainer explicitly splits the remaining work into canonical follow-ups.
