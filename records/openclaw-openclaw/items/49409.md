## Summary

Keep this PR open. Current main still defaults the bundled session-memory hook to LLM-backed slug generation and writes per-session `YYYY-MM-DD-slug.md` files, while the latest release `v2026.4.26` shows the same behavior. The PR is a plausible implementation candidate for canonical daily memory files and nonblocking slugging, but its head patch still has material review blockers around concurrent daily-file header creation, timeout parsing, and removed workspace-routing coverage.

## Best Possible Solution

Keep this PR open as a focused implementation candidate, but require a refreshed patch against current main. The best path is to implement canonical `memory/YYYY-MM-DD.md` appends for the bundled session-memory hook, make LLM slugging opt-in or otherwise nonblocking for `/new` and `/reset`, create the first daily header without a TOCTOU race, preserve workspaceDir/display-session-key coverage, update `docs/automation/hooks.md` and `src/hooks/bundled/session-memory/HOOK.md`, and validate with targeted session-memory plus fs-safe tests before the changed gate.
