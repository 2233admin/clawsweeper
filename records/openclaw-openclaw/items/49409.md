## Summary

Keep this PR open. Current main still has the slugged session-memory writer and default LLM slug path that the PR is trying to change; main only has reader-side compatibility for exact daily files plus slugged artifacts. The PR remains a plausible implementation candidate, but the submitted patch is not ready as-is because review comments identify a daily-header race, timeout fallback bug, and a removed workspace-routing regression path.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require a refreshed patch against current main. The best path is to implement canonical `memory/YYYY-MM-DD.md` writer appends, make LLM slugging opt-in or otherwise nonblocking for /new and /reset, handle first daily header creation without the TOCTOU race, define timeout semantics clearly, preserve workspace routing coverage, update docs/HOOK metadata, and validate with targeted session-memory/fs-safe tests plus the appropriate changed gate.

## Risks / Open Questions

- The submitted daily-file implementation can duplicate the top-level daily header when concurrent /new or /reset hooks both probe before either writes.
- The submitted timeout logic needs an explicit zero and invalid-value policy before adding OPENCLAW_SESSION_MEMORY_SLUG_TIMEOUT_MS or slugTimeoutMs.
- Changing appendFileWithinRoot return semantics must preserve symlink, hardlink, alias, and root-boundary protections in the shared safe-write helper.
- A refreshed patch should preserve the current workspaceDir/display-session-key regression coverage for multi-agent workspace routing.
