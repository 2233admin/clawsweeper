## Summary

Keep this PR open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still does not fully implement the PR’s intended batch-b sendPayload coverage: Discord and Slack now have current-main sendPayload implementations, but Google Chat, Mattermost, Microsoft Teams, and Synology Chat still expose only the older sendText/sendMedia outbound paths. The PR is partly obsolete and needs a rebase/narrowing, but it is not safe to close as implemented.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rebase on current main, drop or reconcile the already-shipped Discord and Slack pieces, implement current-pattern sendPayload support for Google Chat, Mattermost, Microsoft Teams, and Synology Chat through the existing plugin SDK payload helpers where appropriate, and add focused adapter/contract tests for text-only, media-only, multi-media, channelData-only, chunking/formatting, reply-to fanout, and fallback delivery behavior.
