## Summary

Current main still lacks the durable subagent completion spool/inbox requested here. The code has partial direct/fallback hardening, but completion delivery remains direct-first with conditional queue fallback, retry-limit give-up, and orphan pruning before any self-contained pending completion payload can survive restart/drain gaps.

## Best Possible Solution

Keep this issue open as the planning thread. Review #65037 for the narrow retry-state preservation slice, then design the broader core fix around a self-contained persisted completion spool: create it before cleanup can give up, make direct and queued delivery consume/update it, keep task delivery pending while the spool is pending, and change restore/orphan reconciliation to retry or deliberately expire pending spooled completions instead of pruning them.
