## Summary

Keep open. Current main has partial subagent announce hardening, but the requested durable completion spool/restart/orphan preservation is still not implemented. Source, tests, docs, and the issue discussion all show the broader survivable completion-inbox design remains open, with #65037 as an unmerged partial candidate.

## Best Possible Solution

Keep this issue open as the planning thread. The best path is to review #65037 for the retry-state preservation slice, then implement the broader core fix in the subagent registry/announce lifecycle: persist a self-contained completion spool before cleanup can give up, make direct/queue delivery consume and update that spool, keep task delivery pending while the spool is pending, and change restore/orphan reconciliation so pending spooled completions are retried or expired deliberately instead of pruned.
