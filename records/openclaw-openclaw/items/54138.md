## Summary

Keep open. Current main still leaves heartbeat final reply payloads on the `isHeartbeat` pass-through path in `buildReplyPayloads`, the shared bracket sanitizer remains narrower than the reported same-line/non-JSON `[TOOL_CALL]...[/TOOL_CALL]` format, and the focused payload-builder tests still lack heartbeat bracket-block coverage. The issue is also paired with open closing-syntax PR #54165, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that heartbeat-mode reply payloads strip uppercase `[TOOL_CALL]...[/TOOL_CALL]` and `[TOOL_RESULT]...[/TOOL_RESULT]` bracket-format blocks before Telegram/Discord or other user-facing delivery.

## Best Possible Solution

Review and either land PR #54165 or land an equivalent focused fix in `src/auto-reply/reply/agent-runner-payloads.ts` that strips uppercase bracket tool-call/result blocks from heartbeat reply text before final channel delivery, with regression coverage in `src/auto-reply/reply/agent-runner-payloads.test.ts`. Track the broader interactive/XML leak separately if it is still reproducible.
