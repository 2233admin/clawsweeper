## Summary

Keep open. Current main implements the broad MCP exposure path for embedded Pi/main agent runs, but it does not implement the specific behavior reported here: the built-in `image` tool only falls back across configured image models/providers, not to an arbitrary configured MCP `understand_image` tool. Because this issue is fresh, concrete, and follows up on a narrower gap than the earlier closed report, it should not be closed as already implemented.

## What This Changes

Request that MCP servers configured under `mcp.servers` be usable by the main OpenClaw agent, specifically by falling back to an MCP `understand_image` tool when the built-in `image` tool fails.

## Best Possible Solution

Maintainers should decide the intended product contract: either document that users must call the exposed prefixed MCP tool name directly, or add a generic media-understanding adapter/fallback contract for configured MCP tools without hardcoding a MiniMax-specific server or schema into core.
