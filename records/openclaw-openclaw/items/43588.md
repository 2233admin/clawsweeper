## Summary

Keep this PR open, but do not merge it as-is. The requested Slack `allowBots: "mentions"` mode is still missing on current `main`, while the branch implements the runtime change in the removed `src/slack` path and leaves the active `extensions/slack` plugin/config metadata boolean-only.

## What This Changes

The branch widens Slack `allowBots` to accept `"mentions"`, adds mention-gated bot handling and tests in the old Slack runtime path, and updates core config schema/help text.

## Best Possible Solution

Ship the feature through the active Slack plugin: add a tri-state Slack bot policy in `extensions/slack`, preserve own-bot filtering, allowlists, DM behavior, and `thread.requireExplicitMention`, widen the public Slack config schema/types/metadata/UI hints, document the mode, add a user-facing changelog credit, and cover explicit mention, implicit-thread mention, DM, `true`, and `false` behavior in active plugin tests.

## Reproduction Assessment

Yes. The current-main reproduction path is a static config/runtime check: Slack validation/types reject `allowBots: "mentions"`, and active Slack message preparation only has a boolean bot gate.

## Solution Assessment

No. The PR's intended behavior is right, but the implementation is not the best current solution because it patches the removed Slack runtime path; the safer fix is a narrow port into `extensions/slack` plus current metadata/docs/tests.
