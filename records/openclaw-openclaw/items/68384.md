## Summary

Keep open. Current main still returns full trimmed non-JSON stdout for single-id exec providers, and this branch adds a narrowly scoped fail-closed guard plus regression coverage that is not otherwise present. Because it changes CODEOWNERS-protected secrets behavior and is linked to an open security bug, the remaining step is maintainer/secops review rather than cleanup closure.

## What This Changes

The PR changes `parseExecValues()` to reject multi-line non-JSON stdout for single-id `jsonOnly: false` exec providers and adds a POSIX regression test in `src/secrets/resolve.test.ts`.

## Best Possible Solution

Have secops review the fail-closed parser contract, then land it with a short docs/changelog update if accepted. Keep #64821 open until the same-line corruption scenario is either fixed by this path with evidence or tracked by a narrower follow-up.
