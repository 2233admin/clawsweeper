## Summary

Keep open. Current main still auto-starts the bundled Signal plugin's `signal-cli` daemon without a non-mutating account-file preflight for `registered:false`, without automatic backup/recovery behavior, and without docs for the destructive failure mode; the report remains concrete and actionable even though the exact mutating root cause is still unproven.

## What This Changes

Investigate and guard the bundled Signal plugin so OpenClaw update/restart does not auto-start `signal-cli` against account data whose `registered` flag has flipped false, with appropriate backup or recovery guidance.

## Best Possible Solution

Keep the issue open and handle the fix in the bundled Signal plugin: add a Signal-owned, non-mutating startup preflight before daemon auto-start where the account file can be located, fail closed with a clear warning when `registered:false` is detected, add mocked regression coverage, and document backup/recovery guidance on the public Signal channel page.
