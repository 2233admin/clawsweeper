## Summary

Keep open. Current main still lacks a Signal-owned non-mutating preflight before daemon autostart for a `signal-cli` account file with `registered:false`, and the report is recent, concrete, and destructive enough to need maintainer follow-up rather than stale cleanup.

## What This Changes

The issue asks OpenClaw to stop Signal update/restart flows from corrupting or acting on `signal-cli` account state, preserve or back up account data, and warn before startup when `registered:false` is detected.

## Best Possible Solution

Keep the issue open and handle the fix in the bundled Signal plugin: add a Signal-owned, non-mutating startup preflight before daemon autostart where the account file can be located, fail closed with a clear warning on `registered:false`, add mocked regression coverage, and document backup/recovery guidance on the public Signal channel page.
