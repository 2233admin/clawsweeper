## Summary

Keep this PR open. Current main still implements `openclaw logs` as a gateway-file-log-only command, so the PR’s intended `--source gateway|llm|all` and cache-trace-backed LLM trace output are not obsolete or implemented. The PR remains a focused implementation candidate, but it still needs maintainer/contributor follow-up on the existing functional review comments and the cache-trace permission/redaction contract before merge.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up: rebase onto current main, fix the existing review comments, decide and document the permission/redaction contract for cache-trace-backed LLM logs, update the public `openclaw logs --source gateway|llm|all` docs and protocol docs, then rerun the targeted CLI/gateway tests plus the appropriate changed gate before merge.
