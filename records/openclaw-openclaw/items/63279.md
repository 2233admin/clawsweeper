## Summary

Keep open. Current main has partial fixes for the reported cascade, especially compaction timeout handling and subagent announce queueing, but it still does not fully satisfy the issue. GatewayDrainingError is still a user-visible restart failure path rather than an automatic retry path, and fallback config/session override behavior remains nuanced enough that a maintainer should scope the remaining work before closure.

## Best Possible Solution

Keep this issue open as the umbrella until maintainers either split the remaining work into narrower canonical items or land a focused follow-up. The best path is to route GatewayDrainingError retry/user-visible behavior through #55412 or a successor, decide whether repeated compaction failures need an additional circuit breaker or timeoutAction policy, and clarify or adjust fallback-chain reload semantics for active sessions during drain windows.
