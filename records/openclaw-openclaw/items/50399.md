## Summary

Keep open. Current main still does not implement automatic Feishu audio/video duration propagation on the normal message or reply media send path; it only forwards a duration when a caller already supplies one. The open PR is also not a conservative close candidate because its discussion and diff still show regression-prone rewrites around Feishu media routing/download metadata plus security-sensitive process probing in the outbound path. The remaining user-facing bug is also tracked by #53798.

## Best Possible Solution

Keep this PR open for maintainer follow-up or replacement. The best fix is a smaller Feishu-only patch that preserves `extractFeishuDownloadMetadata` and `resolveFeishuOutboundMediaKind`, computes duration after media normalization with the shared `runFfprobe`/`execFile`-based helper, passes duration only for proven Feishu Opus/MP4 uploads, and adds focused tests for duration propagation plus the existing routing and metadata behavior. Coordinate the final fix with #53798.
