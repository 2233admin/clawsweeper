## Summary

Keep this PR open. Current main only has a partial duration sink in `uploadFileFeishu`; the normal Feishu outbound media path still does not compute or pass a duration when uploading Opus/MP4 media, so the PR's intended fix is not implemented on main. The PR is also not landable as-is based on the maintainer review and the current guardrail/security-sensitive surfaces it touches.

## Best Possible Solution

Keep the PR open for now, but require a rework or replacement patch: preserve `resolveFeishuOutboundMediaKind` and download metadata extraction, use the existing `runFfprobe`/`execFile`-style safe helper with normalized/temp media inputs, compute duration for proven Feishu Opus/MP4 uploads, pass it into `uploadFileFeishu`, and add focused tests that assert `duration` is included while MIME fallback, metadata extraction, and MP3 generic-file fallback still work. Coordinate with the open Feishu duration bug report as the canonical user problem.
