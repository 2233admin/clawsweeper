## Summary

Keep this PR open. Current main still does not implement the PR's intended Feishu duration behavior: `uploadFileFeishu` can forward a `duration` value when one is supplied, but the normal `sendMediaFeishu` path never computes or passes duration for Opus/MP4 media. The PR also remains non-landable as-is because its diff is security/guardrail-sensitive and the maintainer discussion already called for a smaller rework rather than merging this patch.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but require a rework or replacement patch: preserve `extractFeishuDownloadMetadata` and `resolveFeishuOutboundMediaKind`, derive duration after media normalization/preparation with the existing `runFfprobe`/`execFile`-based media runtime helper, pass duration only for proven Feishu Opus/MP4 uploads, and add focused Feishu tests. Coordinate the final fix with #53798 as the canonical user-facing bug.
