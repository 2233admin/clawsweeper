## Summary

Keep open. Current main still omits built-in agent tool calls from `/v1/responses` output, while built-in tool start events are available on the agent event stream; the issue is also paired with open closing PR #75075, so it should stay open for normal maintainer review rather than being closed or queued separately.

## What This Changes

The issue requests an opt-in `gateway.http.endpoints.responses.exposeBuiltInToolCalls` flag that would surface built-in agent tool calls as `function_call` output items in JSON and SSE `/v1/responses` responses.

## Best Possible Solution

Review and land PR #75075, or replace it with an equivalent narrow implementation that updates the gateway config schema, JSON/SSE Responses behavior, tests, docs, and changelog together while keeping the behavior opt-in.

## Reproduction Assessment

Yes. Static inspection of current main gives a high-confidence reproduction path: run a `/v1/responses` request whose agent uses built-in tools, and the response output will be built from assistant text plus pending client-tool calls only, while built-in `tool` events remain outside the response.

## Solution Assessment

Unclear until the paired PR is reviewed. The proposed default-off audit flag is a plausible narrow solution, but maintainers should verify security/redaction behavior and ordering semantics in the implementation before merge.
