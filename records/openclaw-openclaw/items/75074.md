## Summary

Current main still only surfaces Responses `function_call` output for caller-provided client tools, while built-in agent tool events remain outside the `/v1/responses` output contract. This issue also has an open closing PR, so it should stay open for normal PR review rather than being closed early.

## What This Changes

Request an opt-in `/v1/responses` mode that exposes built-in agent tool invocations as audit-only `function_call` output items in both JSON and SSE responses.

## Best Possible Solution

Keep this issue open until PR #75075 is reviewed and either landed or replaced with an equivalent narrow implementation that updates the gateway config schema, JSON/SSE Responses behavior, tests, docs, and changelog together.

## Reproduction Assessment

Yes. Static inspection of current main is enough to reproduce the gap: built-in tools emit agent `tool` events, but `/v1/responses` only converts assistant output and pending client-tool calls into Responses output items.

## Solution Assessment

Unclear until PR review. The default-off audit flag proposed here is a plausible narrow design, but maintainers should verify output ordering, redaction boundaries, and client-tool delegation interactions in the paired implementation.
