## Summary

Keep this PR open. Current main still does not implement Feishu chat-history listing through the shared message action surface, and the PR is a plausible small implementation, but the proposed patch has a concrete thread-id/chat-id bug and overlaps with another open Feishu list-action PR that targets a different tool surface.

## What This Changes

The PR adds a Feishu shared `message` tool `list` action, wiring chat-history listing through Feishu action discovery, runtime exports, a new `im.message.list` chat helper, shared action metadata, and Feishu action-surface tests.

## Best Possible Solution

Keep a Feishu-owned chat-history implementation, but first remove the thread-id fallback or route thread IDs to the existing thread-history path, add targeted unit coverage for chat-id listing, pagination, and invalid limits, document required Feishu scopes and privacy implications, then reconcile the chosen surface with #41306.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Remove the threadId fallback from chat history listing:** `extensions/feishu/src/channel.ts:758-761`
  - body: The new `list` branch can pass `ctx.params.threadId` as `chatId`, but `listFeishuChatMessages` calls `im.message.list` with `container_id_type: "chat"`. Feishu thread IDs and chat IDs are different containers, and current main uses `container_id_type: "thread"` for thread history, so this fallback will fail for thread callers.
  - confidence: 0.9
- **[P3] Clamp list limits before sending page_size:** `extensions/feishu/src/send.ts:457`
  - body: `limit` is accepted as any number and then used in `Math.min(limit, 50)`, so values like `0` or negative numbers can be sent to Feishu as an invalid `page_size`, while values above 50 only return a single capped page. Normalize to a valid 1-50 page size and rely on the returned page token for additional pages.
  - confidence: 0.78
