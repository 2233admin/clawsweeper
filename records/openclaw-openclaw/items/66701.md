## Summary

Keep open. Current main still preserves non-empty preamble text after stripping a trailing silent token, and the linked target-repo PRs that propose fixes are open and unmerged, so this issue should remain the canonical tracker until maintainers choose and land the intended suppression policy.

## What This Changes

Suppress channel-visible replies when a model prepends reasoning or think text before a final `NO_REPLY` silent-token signal.

## Best Possible Solution

Keep this issue open as the tracker, choose one suppression policy, then land a focused fix with regression coverage for bare `think`, `<think>...</think>`, whitespace-delimited trailing tokens, glued or doubled final-token cases, existing substantive-reply preservation, and the relevant final/block delivery path.
