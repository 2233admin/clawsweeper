## Summary

Keep open. Current main still does not expose MiniMax speech-to-text: the MiniMax media provider is image-only, while OpenClaw already has a shared audio transcription contract that could host this if MiniMax has an official supported ASR endpoint. The blocker is upstream confirmation: the current official MiniMax docs I checked list T2A and related speech APIs, but not a MiniMax ASR endpoint, and the OpenAI-compatible page is text/chat oriented. 

## What This Changes

The issue requests adding MiniMax speech-to-text transcription support to the bundled MiniMax plugin.

## Best Possible Solution

Confirm MiniMax’s official ASR endpoint and default transcription model first. If confirmed, add MiniMax batch STT as an audio media-understanding provider inside the MiniMax plugin using the existing OpenClaw transcription contract, then update manifest metadata, provider docs, the media matrix, and targeted provider tests; if the endpoint is unofficial, keep it out of the bundled MiniMax plugin and handle it as an external plugin.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The absence is verifiable by source/docs inspection: current MiniMax provider metadata is image-only and the user-facing media matrix does not list MiniMax as an STT provider.

## Solution Assessment

Unclear. Adding audio support to the bundled MiniMax plugin is the narrow maintainable path only if MiniMax has an official supported ASR API; without that confirmation, the safest solution is maintainer/upstream review rather than coding against the reported endpoint.
