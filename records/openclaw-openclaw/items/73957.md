## Summary

Keep open. Current main still does not expose MiniMax speech-to-text: the bundled MiniMax media-understanding provider and manifest are image-only, while the docs advertise MiniMax TTS/media generation but leave STT blank. The existing audio transcription provider seam is available, but MiniMax's public ASR endpoint/model contract is not confirmed enough for an automated implementation.

## What This Changes

Request to add MiniMax ASR/speech-to-text support to the bundled MiniMax plugin so OpenClaw can transcribe audio through MiniMax.

## Best Possible Solution

Confirm MiniMax's supported ASR contract from official docs, vendor confirmation, or live proof. If confirmed and maintainers want it bundled, implement it inside `extensions/minimax` through the existing media-understanding audio provider contract, update manifest metadata and MiniMax/media docs, and add focused registration plus request-shape tests; otherwise document MiniMax as TTS-only for speech in core.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification is high-confidence: source, manifest metadata, tests/contracts, and docs all show MiniMax media understanding is image-only and MiniMax STT is not advertised.

## Solution Assessment

Unclear. A provider-local implementation using `MediaUnderstandingProvider.transcribeAudio` is the narrowest maintainable path if MiniMax ASR is confirmed, but coding against the reporter's unverified `/v1/audio/transcriptions` assumption would be speculative.
