## Summary

Keep open. Current main still registers MiniMax media understanding as image-only and documents MiniMax with no STT capability, while the shared media-understanding API already has the right batch audio transcription seam. The requested direction belongs in the bundled MiniMax plugin if MiniMax officially supports the ASR endpoint, but official MiniMax docs inspected for this review did not confirm an ASR/transcriptions API, so this needs maintainer or vendor-contract confirmation before implementation.

## What This Changes

Request to expose MiniMax speech-to-text/ASR support in the bundled MiniMax plugin so OpenClaw can transcribe audio through the shared media audio pipeline.

## Best Possible Solution

Verify MiniMax's official ASR contract first. If MiniMax supports a stable batch transcription endpoint, add it inside `extensions/minimax` by extending the MiniMax media-understanding provider with audio capability, default model metadata, manifest metadata, focused provider tests, and provider docs; if not, document that MiniMax STT is not currently supported.

## Reproduction Assessment

Not applicable as a runtime bug report. The high-confidence check is source and docs inspection: current main has MiniMax image media understanding and TTS, but no MiniMax `transcribeAudio` implementation or documented STT capability.

## Solution Assessment

Unclear until the vendor contract is confirmed. If MiniMax exposes an OpenAI-compatible ASR endpoint, the narrowest maintainable solution is provider-local registration through the existing media-understanding `transcribeAudio` path rather than a new core API.
