## Summary

Keep open. Current main still renders Dream Diary prose as escaped Lit text after a flattening pass, so inline markdown such as bold or italic markers will remain literal. The linked fix PR had the right direction, but it was closed unmerged and current main has since changed the diary-body path, so this remains a valid narrow UI bug.

## What This Changes

Render Markdown formatting in the Control UI Dream Diary tab instead of showing Markdown syntax literally.

## Best Possible Solution

Render diary prose through the existing sanitized Markdown pipeline at the Dream Diary boundary while preserving the current body-flattening cleanup for structured backfill entries and citations. Add focused view-test coverage for rendered bold/italic Markdown and keep styling scoped to `.dreams-diary__prose`.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to render a Dream Diary entry containing `**bold**`: current main passes the string through `flattenDiaryBody` and Lit text interpolation, so the asterisks remain visible instead of producing a `<strong>` element.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse `toSanitizedMarkdownHtml()` with Lit `unsafeHTML`, matching the existing chat renderer pattern, while keeping the diary-specific flattening behavior; adding a new parser or raw HTML path would be riskier.
