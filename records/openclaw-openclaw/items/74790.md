## Summary

Keep this PR open. Current main still lacks a web_fetch private-network config opt-in, but the proposed branch is not merge-ready because its policy assembly omits the already-present IPv6 ULA SSRF opt-in and the change intentionally expands a security-sensitive network surface that needs maintainer review and docs alignment.

## What This Changes

The PR adds `tools.web.fetch.ssrfPolicy.dangerouslyAllowPrivateNetwork` config plumbing, generated config metadata, cache-key handling, and web_fetch SSRF tests for private-network and loopback access.

## Best Possible Solution

Land or replace this with a single canonical `tools.web.fetch.ssrfPolicy` implementation that adds an explicit private-network opt-in, preserves RFC2544 and IPv6 ULA policy behavior, partitions cache entries by every SSRF policy flag, updates generated config/public docs/changelog, and documents the operator threat model.

Not applicable as a feature PR; there is no runtime bug reproduction required to keep it open. The behavior gap and merge risk are source-verifiable from the strict schema, web_fetch runtime policy assembly, and existing SSRF tests on current main.

## Solution Assessment
