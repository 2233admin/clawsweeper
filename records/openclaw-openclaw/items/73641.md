## Summary

Keep this PR open. Current main still uses `agents.defaults.model` for the Telegram no-session-override fallback, and that value drives the model-list check mark. The current PR head is now focused on the Telegram runtime and regression test, so it remains a valid implementation candidate rather than something to close as obsolete.

## Best Possible Solution

Keep this PR open and review it as the focused implementation candidate. The best fix is the PR's current direction: compute `defaultModel = resolveDefaultModelForAgent({ cfg: runtimeCfg, agentId: route.agentId })` once in `resolveTelegramSessionState`, use it for stored-override default provider resolution, return the provider-qualified default model when no session override exists, and keep the bound-agent regression test.
