## Summary

Keep this PR open. Current main has the gateway-side subscriber-scoped `sessions.changed` event, but the TUI still does not subscribe to it, dispatch it, or clear stale run/stream state for an active session reset from another client. The provided PR context also shows a recent maintainer repair and merge-ready labeling, so this remains the active implementation candidate for #38966 rather than obsolete work.

## Best Possible Solution

Review and land the repaired branch if its current diff and checks are acceptable: add a TUI backend wrapper for `sessions.subscribe`, subscribe on Gateway connection, dispatch `sessions.changed`, and for active-session `new`/`reset` events clear stale run, stream, watchdog, BTW, and local-run state before reloading history/session info. After merge, #38966 can close as implemented.
