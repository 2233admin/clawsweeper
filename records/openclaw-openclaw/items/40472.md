## Summary

Keep this PR open. Current main already has the gateway-side `sessions.changed` subscription contract and emits that event after `sessions.reset`, but the TUI still does not subscribe to that event, dispatch it, or clear same-session stale run state on external `/new` or `/reset`. The linked #38966 remains actionable until this PR or an equivalent TUI-side patch lands.

## Best Possible Solution

Review and land this PR, or an equivalent TUI-side implementation, so the TUI subscribes to `sessions.changed` on connection/reconnection, dispatches active-session `new`/`reset` events, clears stale run/stream/local BTW state, and reloads history/session info. After that ships, #38966 can close as implemented.
