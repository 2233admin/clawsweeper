## Summary

Keep this PR open. Current main has the gateway-side `sessions.changed` lifecycle event, but the TUI still does not subscribe to it or handle it, so the user-visible reset-refresh bug tracked by #38966 remains unresolved on main. The latest PR context shows a maintainer pushed a narrow repair to this branch and marked it merge-ready, making review/land the right cleanup path rather than closing it.

## Best Possible Solution

Review and land the repaired branch if the current diff matches the described direction: have the Gateway-backed TUI call `sessions.subscribe` on connect, dispatch `sessions.changed`, and for active-session `new`/`reset` events clear run/stream/watchdog/local-run state before reloading history and session info. After that lands on main, #38966 can close as implemented.
