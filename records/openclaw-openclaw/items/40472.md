## Summary

Keep this PR open. Current main has the gateway-side `sessions.changed` event and subscription contract, but it still does not implement the TUI-side subscription, dispatch, handler, or reset-state cleanup that this PR is trying to add. The linked TUI external-reset bug therefore remains actionable until this PR or an equivalent patch lands.

## Best Possible Solution

Review and land this PR, or an equivalent TUI-side implementation, so the TUI subscribes to `sessions.changed` on connection/reconnection, handles active-session `new`/`reset` events, clears stale run/stream/local BTW state, and reloads history/session info. After that ships, #38966 can close as implemented.
