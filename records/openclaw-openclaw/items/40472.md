## Summary

Keep this PR open. Current main already has the gateway-side `sessions.changed` subscription contract and emits that event after `sessions.reset`, but the TUI still does not subscribe, dispatch `sessions.changed`, or clear same-session stale run state after an external `/new` or `/reset`. The PR remains an implementation candidate for the linked #38966 flow, and the provided discussion shows a maintainer recently repaired this branch to preserve the contributor path.

## Best Possible Solution

Review and land this PR, or an equivalent TUI-side patch, so the TUI subscribes to `sessions.changed` on connection/reconnection, dispatches active-session `new`/`reset` events, clears stale active run/stream/local BTW state, suppresses late old-run events where feasible, and reloads history/session info. After that ships, #38966 can close as implemented.
