## Summary

Keep this PR open. Current main still has the TUI-side gap this PR targets: the gateway now emits and advertises `sessions.changed`, but the TUI does not subscribe to session lifecycle events, dispatch `sessions.changed`, or clear stale same-session run/stream state after an external `/new` or `/reset`. The PR is also the active implementation candidate for the still-open linked report, with a recent maintainer repair preserving the contributor path.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent TUI-side patch. The fix should add a TUI backend wrapper for `sessions.subscribe`, subscribe on connect/reconnect, dispatch `sessions.changed`, handle active-session `new`/`reset` events by clearing stale run/stream/local BTW state, reload history/session info, keep targeted regression tests, and then close the linked external-reset report once it ships.
