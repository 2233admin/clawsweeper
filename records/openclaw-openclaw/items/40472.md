## Summary

Keep this PR open. Current main has the gateway-side `sessions.changed` lifecycle event and advertised `sessions.subscribe` method, but the TUI still does not subscribe to that event, dispatch it, or clear stale in-memory run state when the active session is reset externally. The provided PR context also shows a recent maintainer repair on the branch, merge-ready labels, and closing syntax for the still-open #38966, so this remains the active implementation candidate rather than obsolete work.

## Best Possible Solution

Review and land the repaired branch if its current diff matches the intended narrow fix: add a TUI backend wrapper for `sessions.subscribe`, subscribe on Gateway connection, dispatch `sessions.changed`, and for active-session `new`/`reset` events clear stale run/stream/watchdog/local-run state before reloading history/session info. After it lands, #38966 can close as implemented.
