## Summary

Keep this issue open. Current main still drops `allowUnsafeExternalContent` in the direct `POST /hooks/agent` normalization path, while the downstream dispatch and isolated cron runner already honor the field when it is present.

## Best Possible Solution

Fix the direct `/hooks/agent` path if this per-request trust opt-out is intended: add `allowUnsafeExternalContent?: boolean` to `HookAgentPayload`, copy `payload.allowUnsafeExternalContent === true` in `normalizeAgentPayload()`, include it in replay/idempotency scope if needed, and add regression coverage for a direct `POST /hooks/agent` request that verifies the cron payload or final prompt is not externally wrapped. If maintainers do not want request-body opt-out, update the endpoint docs and validation to reject or ignore it explicitly rather than silently dropping it.
