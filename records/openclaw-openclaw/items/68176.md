## Summary

Keep open. Current main still resolves the CLI version commit from live git before baked build metadata and has no build-time injected commit source, so the PR is not obsolete. The related bug report is already closed, but the code gap it describes remains on main and this PR is the active implementation candidate.

## What This Changes

This PR adds a tsdown build-time git commit define, reads it in `resolveCommitHash` before live git after explicit env overrides, and adds resolver tests for priority, fallback, and cache behavior.

## Best Possible Solution

Land this PR or an equivalent narrow change after maintainer review: preserve explicit `GIT_COMMIT`/`GIT_SHA` overrides, bake a sanitized build commit into node build outputs, consult it before live git, and keep regression coverage for the resolver priority and caching behavior.
