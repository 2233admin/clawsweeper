## Summary

Keep this PR open. Current main still persists `systemSent: true` inside `ensureSkillSnapshot()` before the LLM run, and neither the normal reply nor queued followup success path has the proposed post-success replacement. The PR remains relevant to the linked tool-dispatch/session-corruption report, but its latest head still has reviewable issues around best-effort persistence and followup reset state.

## Best Possible Solution

Keep this PR open for maintainer and author follow-up. The best path is a shared post-success `systemSent` persistence helper that removes the pre-run disk write from `ensureSkillSnapshot()`, writes best-effort after successful normal and followup runs, uses the active session entry after resets/compaction, treats silent and messaging-tool successes as delivered, skips errors/aborts, and keeps regression coverage for failed first calls plus successful retries.
