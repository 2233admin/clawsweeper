## Summary

Keep this PR open. Current main still persists `systemSent: true` inside `ensureSkillSnapshot()` before the LLM response and has no production post-success `systemSent` writer in the reply or followup runner paths. The PR remains a focused implementation candidate for #41462's corrupted session-state path, though it still needs maintainer review for queued/final followups and successful payloadless or messaging-tool-only completions before landing.

## Best Possible Solution

Keep this PR open. The best path is to finish the fix by separating skill snapshot persistence from delivered-system-prompt state, then persist `systemSent: true` through a shared post-success helper used by normal replies, silent or `NO_REPLY` completions, messaging-tool completions, and queued/final followup completions, while skipping provider/meta-error outcomes. Add targeted regression coverage for failed first call, successful retry with `isNewSession=false`, payloadless success, messaging-tool success, and queued/final followup success.
