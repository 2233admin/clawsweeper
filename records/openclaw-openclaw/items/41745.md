## Summary

Keep this PR open. Current main still persists `systemSent: true` inside `ensureSkillSnapshot()` before the model run, and I did not find an equivalent post-success persistence path in the normal reply or followup runners. The PR remains a focused candidate for the linked session-state corruption bug, but recent review comments still need maintainer/author follow-up before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a shared post-success `systemSent` persistence helper that removes the premature write from `ensureSkillSnapshot()`, treats the followup and normal reply paths consistently, logs and continues on store-write failures, uses the active session entry after resets, and keeps regression coverage for failed first calls, successful retries, silent successes, messaging-tool sends, and followup success.
