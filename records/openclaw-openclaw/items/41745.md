## Summary

Keep open. Current main still persists `systemSent: true` inside `ensureSkillSnapshot()` before the LLM run, and neither normal reply nor queued followup success paths have the post-success replacement proposed here. The latest PR head is still an active implementation candidate, but current review comments flag follow-up issues around best-effort persistence and active session state after resets/compaction.

## Best Possible Solution

Keep this PR open for maintainer and author follow-up. The best path is to remove the pre-run disk write from `ensureSkillSnapshot()`, add a shared post-success marker helper used by normal and followup runs, make that marker persistence best-effort, use the active session entry after reset/compaction, treat silent and messaging-tool successes as delivered, skip errors/aborts, and keep regression coverage for failed first calls plus successful retries.
