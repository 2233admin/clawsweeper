## Summary

Keep this PR open. Current main still persists `systemSent: true` inside `ensureSkillSnapshot()` before the model call, and I did not find an equivalent post-success `systemSent` writer on the normal reply or queued followup paths. The latest release tag available in the checkout has the same pre-response assignment, so current main does not implement the PR’s intended change.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best fix is to separate skill-snapshot persistence from delivered-system-prompt state, remove the pre-response `systemSent` write from `ensureSkillSnapshot()`, and persist `systemSent: true` through a shared post-success helper used by normal replies and queued/final followups. Add targeted regression coverage for failed first call, successful retry with `isNewSession=false`, payloadless or `NO_REPLY` success, messaging-tool-only success, and followup success.
