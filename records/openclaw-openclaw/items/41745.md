## Summary

Keep this PR open. Current main still writes `systemSent: true` during `ensureSkillSnapshot()` before the model call can succeed, and I did not find an equivalent post-success persistence path on main. The latest PR head remains a focused implementation of the missing behavior, including normal reply and followup paths plus regression coverage for silent/messaging-tool successes.

## Best Possible Solution

Keep this PR open for maintainer review and landing work. The best implementation direction is the current PR shape: separate skill snapshot persistence from delivered-system-prompt state, keep the in-memory first-turn signal, persist `systemSent: true` only through a shared post-success helper used by normal replies and followups, and keep regression coverage for failed first calls, successful retries, silent successes, messaging-tool sends, and followup success.
