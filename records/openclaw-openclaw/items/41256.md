## Summary

Keep this issue open. Current main still drops Discord upload MIME types and still routes WebP/non-GIF images through the optimizer, and the issue is paired with open unmerged implementation PRs that explicitly target the same bug.

## Best Possible Solution

Keep the issue open and review/consolidate the existing implementation PRs. The best fix is a narrow patch that passes detected `contentType` into Discord Blob construction across normal sends, component sends, and native-command replies, preserves WebP and animated media in the shared loader, and adds regression tests for Blob MIME type, static WebP preservation, and animated image preservation. Close this issue only after one fix lands on main.
