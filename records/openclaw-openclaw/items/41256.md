## Summary

Keep this issue open. Current main still constructs Discord upload Blobs without the detected MIME type in normal sends, component sends, and native-command replies, and shared web media still sends non-GIF images, including WebP, through an optimizer that returns PNG or JPEG. The issue is already paired with open closing PR #41265, while related PRs #41419 and #41714 are closed unmerged, so the maintainer path is review/consolidation rather than closure or a duplicate repair job.

## What This Changes

Fix Discord media uploads so Blobs carry detected content type and WebP or animated images are not optimized into JPEG/PNG stills.

## Best Possible Solution

Keep this issue open and review or consolidate #41265. The selected patch should thread detected `contentType` into all Discord Blob construction paths, preserve static WebP and animated images in `src/media/web-media.ts` without silent JPEG/PNG conversion, keep existing safety and size-cap checks intact, and add focused regression tests for normal sends, component sends, native-command replies, WebP preservation, and animated-media preservation.
