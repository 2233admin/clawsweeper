## Summary

Keep open. Current main still does not implement the PR's intended macOS `sandbox-exec` support, the same-author paired feature issue #58730 is still open, and this security-boundary change needs maintainer product/security review rather than conservative cleanup. The proposed patch also has concrete correctness and security issues that should block merge as written.

## What This Changes

The PR adds a macOS `sandboxProfile` exec parameter, inline `sandbox-exec` wrapping in `runExecProcess`, a new Seatbelt wrapper module, and tests for profile generation.

## Best Possible Solution

Keep this PR and #58730 open until maintainers decide whether native macOS Seatbelt support belongs in core. If accepted, implement it as a sandbox backend or backend capability under `agents.defaults.sandbox`, with config docs, policy-owned defaults, and regression coverage; if rejected, close the PR and paired issue together after that explicit decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Deny file operations before allowlisting paths:** `src/agents/bash-tools.exec-runtime.ts:130`
  - body: The default profile starts with `(allow default)` and never denies file operations before adding path-specific allow rules, so `sandboxProfile: "default"` would not enforce the advertised read/write restriction. For a security boundary, the profile needs to deny by default and then explicitly allow the required paths.
  - confidence: 0.83
- **[P2] Export the profile helper used by the new test:** `src/agents/bash-tools.sandbox-exec.test.ts:3`
  - body: The new test imports `getSandboxProfileString` from `bash-tools.exec-runtime.js`, but the helper added in the runtime file is declared as a local function. This will fail typecheck/test collection until the helper is exported or the test exercises a public API.
  - confidence: 0.96

## Security Review

Status: needs_attention

Summary: The PR touches host command execution and introduces concrete sandbox policy concerns that need maintainer/security review before merge.

Concerns:

- **[high] Default profile does not restrict files:** `src/agents/bash-tools.exec-runtime.ts:130`
  - body: The advertised restricted profile allows default operations and lacks an initial file deny rule, so the path-specific file allows do not provide the promised filesystem confinement.
  - confidence: 0.83
- **[medium] Sandbox strength is selected per tool call:** `src/agents/bash-tools.exec.ts:1687`
  - body: The proposed `sandboxProfile` is model/user-call selectable, including `permissive`, rather than enforced by maintainer-owned sandbox configuration; that is not a reliable boundary for malicious or buggy instructions.
  - confidence: 0.74
