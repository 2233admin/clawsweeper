## Summary

Keep open. Current main has stronger Anthropic tool-use repair than the original report described, but the user-facing formatter gap remains: a wrapped Anthropic `invalid_request_error` for `tool_use`/`tool_result` mismatch still falls through to the generic `LLM request rejected: ...` branch. The issue is also paired with open, unmerged PR #45393 using closing syntax, so it should stay open until that PR is reviewed/landed or replaced.

## Best Possible Solution

Keep this issue open and resolve it through review/landing or replacement of PR #45393. The narrow fix is a mismatch-specific guard in `src/agents/pi-embedded-helpers/errors.ts` before the generic `invalid_request_error` fallback, with focused regression tests in `src/agents/pi-embedded-helpers.formatassistanterrortext.test.ts`; preserve current broader Anthropic repair logic and add exact final-message tests only if maintainers want that proof.
