## Summary

Keep this issue open. Current main still exposes `asVoice` on the shared message tool schema but does not carry it through the shared send path, and open PR #73483 references this issue with closing syntax as the implementation candidate.

## Best Possible Solution

Keep this issue open and review/land or replace PR #73483. The product fix should read `asVoice` in the shared message action runner, map it to `audioAsVoice`, preserve it through `executeSendAction`, `sendMessage`, gateway send schema/handler, CLI/docs, and add focused regression coverage while preserving the existing Feishu direct-action behavior.
