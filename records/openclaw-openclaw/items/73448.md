## Summary

Keep open. Current main still has the reported propagation gap: the shared message tool schema accepts `asVoice`, but the core send runner, `executeSendAction`, `sendMessage`, and gateway send contract do not carry a voice-intent field into outbound delivery. PR #73483 is open, unmerged, and linked for closure, so this should remain open until that PR is reviewed and merged or replaced.

## What This Changes

The issue reports that the shared `message` send path accepts `asVoice` in the tool schema but drops it before outbound media delivery can treat audio as a native voice message.

## Best Possible Solution

Review and land PR #73483 if it is still correct, or replace it with an equivalent narrow fix that maps `asVoice` to downstream `audioAsVoice` through the shared action runner, outbound send service, `sendMessage`, gateway send schema/handler, generated clients, docs, and focused regression tests.
