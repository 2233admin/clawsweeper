## Summary

Keep open. Current main still has the reported local credential precedence and adjacent diagnostics, but I did not find a doctor/status/gateway-status warning that reads the state-dir `.env` and warns when `OPENCLAW_GATEWAY_TOKEN` shadows `gateway.auth.token`. The linked implementation PR #74433 is open and unmerged with closing syntax for this issue, so this issue should remain open pending review or merge of that PR.

## What This Changes

The issue requests a redacted diagnostic warning when a non-empty state-dir `OPENCLAW_GATEWAY_TOKEN` overrides or conflicts with configured `gateway.auth.token` in local gateway mode.

## Best Possible Solution

Review #74433 against the full requested diagnostic surface, then land or revise a narrow implementation that reads the trusted state-dir `.env`, redacts all credential values, warns only for local mode with configured `gateway.auth.token` plus non-empty `OPENCLAW_GATEWAY_TOKEN`, and documents the remediation in the relevant CLI docs and changelog.
