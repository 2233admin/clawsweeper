## Summary

## What This Changes

Add redacted doctor/status/gateway-status diagnostics when local gateway mode has `gateway.auth.token` configured and `~/.openclaw/.env` contains a non-empty, mismatched `OPENCLAW_GATEWAY_TOKEN`.

## Best Possible Solution

Keep this issue as the canonical report and resolve it by reviewing PR #74433 or a replacement that adds one shared, redacted diagnostic: parse the state-dir `.env`, compare non-empty `OPENCLAW_GATEWAY_TOKEN` with configured local `gateway.auth.token`, explain env precedence, and suggest removing or syncing the env key plus restarting the gateway without printing secrets.

## Reproduction Assessment

Yes. A high-confidence reproduction path is local gateway mode with `gateway.auth.token` set in `~/.openclaw/openclaw.json` and a different non-empty `OPENCLAW_GATEWAY_TOKEN` in `~/.openclaw/.env`; current dotenv loading and credential precedence tests explain the auth disagreement.

## Solution Assessment

Yes for the requested direction. A warning is safer than deleting a possibly intentional env credential, and the maintainable fix should read the state-dir `.env` source directly rather than relying only on `process.env`.
