## Summary

Keep this open. Current main still treats Feishu/Lark sticker content as generic media or plain text only, and no current code, docs, tests, or related item proves that custom bracket-style sticker labels are preserved for the agent.

## What This Changes

Request to preserve inbound Feishu/Lark custom stickers by converting them to Unicode emoji equivalents or bracketed text labels in the agent-visible message body.

## Best Possible Solution

Fix this in the Feishu plugin inbound parser by preserving verified sticker labels or stable text fallbacks from the raw event payload. Add focused parser and dispatch tests for mixed text plus sticker metadata, standalone `sticker` messages, rich-text `emotion` elements, Chinese bracket labels, and unknown-label fallback behavior.

## Reproduction Assessment

Unclear for the exact live Feishu event because the report has no raw payload sample. Source-level reproduction is clear enough: current main returns only `parsed.text` for text messages and `<media:sticker>` for standalone sticker messages.

## Solution Assessment

Yes, a Feishu plugin parser fix is the right maintainable direction and should not require a core channel API change. The safest implementation is label/text preservation first, with Unicode mapping only for verified sticker names.
