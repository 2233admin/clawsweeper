## Summary

Keep this issue open. Current main still drops Feishu custom sticker context on the plain text inbound path: text messages return only `parsed.text`, standalone `sticker` messages become a generic `<media:sticker>` placeholder, and the rich-text `emotion` renderer only helps when Feishu sends a `post` payload with `emoji`, `text`, or `emoji_type` fields. The earlier ClawSweeper comment correctly identified this as a bundled Feishu plugin gap rather than a duplicate of the Feishu reaction API work.

## Best Possible Solution

Keep this open and fix it in the bundled Feishu plugin. Capture the actual Feishu mixed text/custom-sticker event payload, then extend the inbound parser to preserve a stable bracket/text representation or map known sticker labels to Unicode only where safe. Add focused tests for plain text plus sticker metadata, standalone sticker placeholders, rich-text `emotion` elements, and Chinese bracket labels.
