## Summary

Keep this open. Current main and the latest tagged release still do not preserve Feishu/Lark custom sticker labels in the inbound agent-visible message body, and the related Feishu emoji/reaction items cover outbound reaction normalization rather than this inbound sticker-content path.

## What This Changes

The issue requests preserving Feishu/Lark custom sticker content in agent-visible inbound messages by converting sticker labels to Unicode emoji or text fallbacks.

## Best Possible Solution

Implement this in the Feishu plugin inbound parser by preserving verified custom-sticker labels or stable text fallbacks from raw event payloads. Add focused parser and dispatch tests for mixed text plus sticker metadata, standalone sticker messages, rich-text emotion elements, Chinese bracket labels, and unknown-label fallback behavior.

## Reproduction Assessment

Unclear for the exact live Feishu event because the report does not include raw payload JSON. Source-level verification is high-confidence for current behavior: text messages return only parsed.text, and standalone sticker messages become <media:sticker>.

## Solution Assessment

Yes, the right boundary is a Feishu plugin parser/content-normalization fix rather than a core channel API change. The safest solution is label/text preservation first, with Unicode conversion only for verified sticker names.
