## Summary

Keep open. Current main still does not implement VALUE.md as a first-class workspace context file, and the linked implementation PR #45392 was closed unmerged. The runtime bootstrap contract, workspace seeding/loading, subagent filtering, gateway file API, sandbox seeding, docs, and packaged template smoke check all still enumerate the existing bootstrap files without VALUE.md.

## Best Possible Solution

Keep this issue open for maintainer product judgment. If maintainers accept VALUE.md as a core first-class workspace file, implement it across the full workspace surface in one focused PR: filename constant/type, default template, new-workspace seeding, default loading and Project Context order/guidance, minimal/subagent inclusion, heartbeat lightweight behavior unchanged, gateway agents.files list/get/set, sandbox and dev workspace seeding, docs, package template smoke checks, and focused tests. If maintainers do not want another hard-coded file, use this issue to decide whether SOUL.md/AGENTS.md remains the intended home for value guidance or whether a configurable bootstrap-file API is the better direction.
