## Summary

Current main still has the reported Control UI gap. The Overview insecure-HTTP recommendation still tells users to set `gateway.controlUi.allowInsecureAuth: true` manually, while the existing before/after diff and base-hash guarded config writes remain scoped to Config/Quick Settings surfaces and are not wired into Overview recommendation callouts.

## Best Possible Solution

Keep this issue open until Control UI recommendation callouts can stage narrow base-hash guarded config changes, show before/after values, offer accept/dismiss controls, apply through the existing config write path, and confirm the saved value. A shared recommendation/apply component reused by the insecure HTTP Overview hint and future config-hardening suggestions is the likely direction, with security review for `allowInsecureAuth` and focused UI/controller tests.
