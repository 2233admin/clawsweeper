## Summary

Keep open. Current main contains the server-side part of the ACP stdout hygiene fix, but this PR is not fully obsolete: its stated CLI-mode guard before command registration is still absent from `run-main`, and the PR discussion contains unresolved scoping concerns around `acp client` and standalone server console capture.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it with a corrected main change that covers ACP bridge stdout hygiene from CLI startup through `serveAcpGateway`, explicitly excludes `openclaw acp client`, and decides whether the standalone ACP server entrypoint also needs console capture. If maintainers accept the current server-only fix as sufficient, close the PR manually with that narrower rationale rather than auto-closing it as already implemented.
