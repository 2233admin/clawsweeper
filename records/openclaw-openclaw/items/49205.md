## Summary

Keep this issue open. Current main still has the same product gap described by the reporter: `chat.send` writes/broadcasts through OpenClaw transcript and Gateway chat paths, while Open WebUI-visible history is only covered by the separate Open WebUI Docker smoke in the Open WebUI -> OpenClaw direction. I found no current `chat-open-webui-sync` helper, `webui.db` update path, `chat.chat` JSON mirror, or `chat_message` insertion path for external Control UI turns.

## Best Possible Solution

Keep this open for maintainer follow-up. The best path is a supported post-turn sync design for successful external `chat.send` into WebUI-backed UUID sessions: either implement a narrow, tested Open WebUI mirror that uses client id to skip only native `webchat-ui`, or define a small plugin/API seam so Open WebUI-specific storage sync lives outside generic gateway core. Add regression coverage for both OpenClaw transcript context and Open WebUI visible history.
