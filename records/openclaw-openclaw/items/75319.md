## Summary

Keep open. The PR is not mergeable as-is: it adds security-sensitive at-rest encryption, but the patch only covers one transcript append helper, introduces CommonJS `require` calls into ESM runtime paths, writes invalid Pi session JSONL entries after a file is encrypted, and uses key material protection that needs maintainer security review. The linked security umbrella #7604 is still open, so this should remain in maintainer review rather than be closed or auto-repaired.

## What This Changes

The PR adds session encryption helpers, encrypted `sessions.json` read/write support, a new encrypted transcript manager for one assistant transcript append path, and encrypted transcript readers in gateway session utilities.

## Best Possible Solution

Design session-at-rest encryption around a central session persistence seam that covers all Pi `SessionManager` reads/writes, preserves the existing JSONL tree contract, has explicit key management and recovery semantics, and ships with migration tests, docs, changelog, and a maintainer-approved security model.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: calling the PR's new `EncryptedSessionManager.open` from the changed transcript append path would execute `require` in an ESM package, and normal agent runs can be traced to direct `SessionManager.open` calls that bypass the new encryption wrapper.

## Solution Assessment

No. The proposed branch is not the best way to solve the issue because it patches selected readers and one append helper instead of the central persistence boundary, and it changes at-rest security semantics without a reviewed key-management design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Replace the new require calls in ESM modules:** `src/config/sessions/encrypted-session-manager.ts:219`
  - body: OpenClaw is an ESM package, but this new runtime path calls `require` when the transcript append helper opens the wrapper. In Node ESM this raises `ReferenceError: require is not defined`, so delivery-mirror transcript appends fail before encryption is used; the same pattern was also added to gateway read paths.
  - confidence: 0.93
- **[P2] Route primary session writes through encryption:** `src/config/sessions/transcript.ts:281`
  - body: This only swaps the delivery-mirror append helper. Current main still creates the normal agent session manager with `SessionManager.open(params.sessionFile)` in the embedded run and compaction paths, so the ordinary session JSONL files named in the PR continue to be written as plaintext.
  - confidence: 0.88
- **[P1] Preserve the Pi JSONL entry shape for encrypted appends:** `src/config/sessions/encrypted-session-manager.ts:72-75`
  - body: After a transcript is encrypted, `appendEncryptedMessage` appends `{id,message}` directly instead of using Pi's `type: "message"`, `parentId`, and `timestamp` entry shape. Reopened sessions, branch traversal, compaction, and readers that look for tree entries will not treat those appended lines as valid session entries.
  - confidence: 0.9
- **[P2] Fail closed on line encryption errors:** `src/config/sessions/encrypted-session-manager.ts:118-125`
  - body: `convertFileToEncrypted` catches per-line encryption failures, writes the original plaintext line, and then marks the file encrypted. That can leave sensitive transcript content in plaintext while callers believe migration succeeded.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR changes at-rest session security and introduces concrete key-management and plaintext-downgrade concerns that need maintainer security review before merge.

Concerns:

- **[high] Predictable host metadata protects the master key:** `src/config/sessions/encryption.ts:43`
  - body: The key wrapper derives a passphrase from hostname, username, homedir, OS, arch, USER, and HOME, then stores the encrypted master key under OpenClaw state. An attacker who can read local state often has enough of that metadata to derive the same key, so this does not provide the claimed defense-in-depth boundary.
  - confidence: 0.88
- **[medium] Migration can silently keep plaintext transcript lines:** `src/config/sessions/encrypted-session-manager.ts:122`
  - body: The migration/conversion code logs encryption failures but preserves the plaintext line and continues, which can produce mixed encrypted/plaintext transcripts without making the migration fail closed.
  - confidence: 0.86
