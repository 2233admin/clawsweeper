## Summary

Keep this issue open. Current main still persists provider onboarding auth results to one resolved agentDir, Anthropic setup-token paths still write or return one profile for that agent, and the existing sibling-sync helper is not wired into production onboarding. Runtime main-agent fallback helps missing stores, but stale non-main profiles still override main on conflicts, so it does not fully resolve the reported stale/silent-fallback case. The related implementation PR #44644 was closed unmerged; broader auth-sync issue #47684 overlaps but does not supersede this specific onboarding report.

## Best Possible Solution

Keep this issue open and make an explicit maintainer decision on multi-agent auth semantics: either intentionally sync selected onboarding credentials to configured agent dirs with clear SecretRef-safe behavior, or keep per-agent isolation and add a first-class auth-sync/repair/warning path for stale shadowing. The implementation should cover Anthropic setup-token/tokenRef, API-key SecretRef, missing stores, stale conflicting non-main profiles, and docs alignment.
