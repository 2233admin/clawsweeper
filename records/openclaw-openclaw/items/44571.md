## Summary

Keep this issue open. Current main still persists provider onboarding auth profiles to one resolved agent directory, while the sibling-sync helper remains opt-in and is not wired into production onboarding. Runtime main-agent fallback helps missing non-main stores and some OAuth freshness cases, but token/API-key profiles in an agent store can still override the main fallback, so the reported Anthropic setup-token/tokenRef case remains actionable. The related implementation PR #44644 was closed unmerged, so there is no shipped fix candidate to close against.

## Best Possible Solution

Keep this issue open and fix it in the provider-auth/onboarding owner path. The safest product direction is either to intentionally sync supported provider auth profiles to every configured agents.list agent during onboarding, or to document per-agent isolation and add a repair/warning path for stale non-main auth profiles that shadow main. Regression coverage should include Anthropic setup-token/tokenRef, API-key SecretRef, missing non-main stores, and stale conflicting non-main profiles.
