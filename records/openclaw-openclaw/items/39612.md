## Summary

Keep open. Current main still forwards sanitized provider/runner failure details to external direct chats when `/verbose` is on, and the docs/tests preserve that behavior. The Anthropic/Bedrock thinking-block corruption path has partial shipped mitigation, but the central request to never expose raw LLM/API failure details to WhatsApp/Telegram users is not implemented.

## Best Possible Solution

Keep this open for an explicit maintainer decision on external-channel failure policy. A focused implementation would keep generic direct-chat fallback copy and silent non-direct behavior, but stop sending provider or runner exception details to external chat users even when `/verbose` is enabled; detailed errors should remain in logs, Control UI, TUI/operator-only surfaces, or exported diagnostics. Multi-agent auto-escalation and deeper session migration/recovery should be separate follow-ups if maintainers want them in scope.
