## Summary

Keep open. Current main partially addresses the reported Anthropic thinking-block failure class with transcript hygiene and a one-time recovery retry, but it still intentionally forwards formatted provider/runner failure details to external direct chat users when `/verbose` is `on` or `full`. That conflicts with the issue’s central request to never expose raw LLM/API failure details through WhatsApp, Telegram, or similar user-facing channels.

## Best Possible Solution

Keep this open for explicit maintainer product judgment on external-channel failure policy. A likely core fix is to keep generic default direct-chat copy and silent group/channel behavior, but stop sending provider/runner exception details to external chat users even under `/verbose`; reserve details for logs, Control UI, or operator-only surfaces. Add regression coverage for WhatsApp and Telegram direct verbose LLM failures. Treat automatic multi-agent escalation and deeper session migration as narrower follow-ups if maintainers want them in scope.
