## Summary

Keep this issue open. Current main still preserves configured provider headers as static strings, the Fireworks plugin does not add session-affinity headers, and the docs still describe `models.providers.*.headers` as static; the closed implementation PRs linked from the issue did not merge. The requested runtime interpolation for `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` remains actionable.

## What This Changes

Add dynamic interpolation for configured custom-provider headers so values like `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` can be resolved per model request.

## Best Possible Solution

Implement a narrow, documented runtime header-template resolver for `models.providers.*.headers` and `models.providers.*.request.headers`. Resolve only explicit placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at request dispatch for main turns and compaction while preserving static headers, SecretRef handling, auth injection, header precedence, retries, attribution headers, and existing provider-owned hooks.
