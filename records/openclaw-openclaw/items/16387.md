## Summary

Keep this issue open. Current main still treats custom provider headers as literal strings: provider-level and request-level headers are sanitized, merged, and attached without runtime interpolation, Fireworks has no `x-session-affinity` behavior, and the existing provider hook surface is provider-owned rather than a resolver for arbitrary `models.providers.*.headers` templates. The related implementation PRs #16448 and #18213 are closed unmerged, so no canonical fix has landed.

## Best Possible Solution

Implement a narrow, documented runtime header-template resolver for `models.providers.*.headers` and `models.providers.*.request.headers`. Resolve only explicit placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at request dispatch, preserve static headers and SecretRef behavior, cover main turns, compaction, retries, and OpenAI-compatible transports, and document Fireworks and LiteLLM/Langfuse examples.
