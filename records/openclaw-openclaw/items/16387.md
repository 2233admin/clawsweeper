## Summary

Keep this issue open. Current main still treats configured provider headers as literal static strings, the Fireworks plugin does not supply session-affinity headers, and the docs still describe provider headers as static. The linked implementation attempts are closed unmerged, while the latest maintainer discussion already scopes the remaining work as a narrow runtime interpolation feature.

## What This Changes

Add dynamic interpolation for custom-provider header values so placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` can be resolved per model request.

## Best Possible Solution

Implement a narrow documented resolver for `models.providers.*.headers` and `models.providers.*.request.headers` that supports only explicit placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at model-request dispatch, preserving static header behavior, auth injection, SecretRef resolution, retries, attribution headers, and provider-owned stream hooks.
