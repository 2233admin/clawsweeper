## Summary

Keep this issue open. Current main still treats configured model-provider headers as literal strings, the Fireworks plugin does not add session-affinity headers, and the newer provider turn-state hook only covers provider-owned hooks rather than config-level header placeholder interpolation. The two implementation PRs linked from the discussion are closed unmerged, so the requested user-visible capability remains missing.

## What This Changes

Request dynamic interpolation for configured custom-provider headers so placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` can be resolved per model request.

## Best Possible Solution

Land a narrow documented runtime resolver for configured model-provider headers at model-request dispatch, covering both `models.providers.*.headers` and `models.providers.*.request.headers` with explicit placeholders while preserving existing static-header behavior, SecretRef resolution, auth injection, attribution headers, retries, and provider-owned stream hooks.
