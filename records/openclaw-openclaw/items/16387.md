## Summary

Keep this issue open. Current main still treats configured custom-provider headers as literal strings, and the newer provider-plugin per-turn header hook does not provide the requested one-line config templating for arbitrary custom providers. The latest maintainer discussion already scopes this as a narrow runtime interpolation feature, and the prior implementation PRs are closed unmerged.

## What This Changes

The issue requests dynamic interpolation for configured custom-provider HTTP headers so runtime session, agent, or run identifiers can be sent to providers such as Fireworks.ai and LiteLLM.

## Best Possible Solution

Add a narrow documented resolver at model-request dispatch for `models.providers.*.headers` and `models.providers.*.request.headers`, supporting only explicit placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` while preserving current static-header behavior and provider-plugin hooks.
