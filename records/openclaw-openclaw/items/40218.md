## Summary

Keep open. Current main still lacks stable cron job IDs and duplicate-id validation, so this PR is not obsolete; the latest diff is the active implementation candidate but still has a source-compatibility bug in the generated Swift protocol initializer.

## What This Changes

The PR adds optional stable cron job IDs across cron service creation, duplicate-id validation, gateway schema, CLI and agent-tool inputs, docs, tests, and generated Swift protocol models.

## Best Possible Solution

Preserve the stable cron-id feature while making the Swift generated protocol surface source-compatible, preferably by teaching the Swift generator to default optional initializer parameters to `nil` and regenerating both Swift model copies.

## Review Findings
