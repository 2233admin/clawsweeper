## Summary

Keep open. Current main still does not implement this PR's Gmail watcher `--exclude-labels` plumbing: the runtime config has no `excludeLabels`, the strict config schema rejects that field, and the actual `gog gmail watch serve` args omit the flag. The linked regression issue remains open, the `gogcli` dependency supports the flag, and this PR is still a plausible implementation candidate, though maintainers should coordinate it with the overlapping open PRs #56720 and #56726.

## Best Possible Solution

Keep this PR open for maintainer review, but resolve it as part of one canonical fix for #56635. The best implementation should thread `excludeLabels` through config types, strict Zod schema, runtime config, CLI overrides if needed, and `buildGogWatchServeArgs()`, with regression tests and generated config schema/docs updates. After one PR merges, close the overlapping PRs as superseded by the merged fix.
