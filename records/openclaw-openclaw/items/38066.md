## Summary

Current main still does not expose Telegram `streamThrottleMs` or `minInitialChars` through config, docs metadata, or dispatcher wiring. The internal draft stream accepts those parameters, but production dispatch still hardcodes the 30-character initial threshold and omits `throttleMs`; open PR #38255 remains the active unmerged implementation path.

## What This Changes

Expose Telegram preview/draft streaming cadence settings, especially `streamThrottleMs` and optionally `minInitialChars`, through top-level and per-account Telegram config for smoother native draft streaming.

## Best Possible Solution

Keep this issue open until PR #38255, or an equivalent maintainer-approved change, lands. The fix should add documented top-level and per-account Telegram preview cadence settings, validate them in the strict config schema, update generated config metadata and public Telegram docs, merge account overrides through the existing Telegram config path, pass resolved values into `createTelegramDraftStream`, and cover defaults, validation, fallback/final flush behavior, and account overrides with focused tests.
