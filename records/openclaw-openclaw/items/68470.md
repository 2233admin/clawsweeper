## Summary

Keep open. Current main still does not recognize MiniMax `prompt_cache_hit_tokens` in the shared usage normalizer, the timeout-compaction path still derives its high-context ratio from normalized prompt usage, and #68750 remains an open closing PR for the exact fix. The separate comment about raw MiniMax tool-call markup appears to be a different integration bug, not evidence that this token-accounting issue is resolved.

## What This Changes

The issue requests MiniMax prompt-cache hit usage normalization so cached prompt tokens are not double-counted in context accounting and premature timeout compaction.

## Best Possible Solution

Review and land #68750 or an equivalent narrow fix that adds `prompt_cache_hit_tokens` to the usage contract, cache-read extraction, OpenAI-style prompt-total de-duplication, and focused regression coverage; then let the merged PR close this issue.
