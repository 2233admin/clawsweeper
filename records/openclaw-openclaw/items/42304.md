## Summary

Keep this PR open. Current main still does not implement Android inline Adaptive Card parsing or Jetpack Compose rendering: Android chat displays text through ChatMarkdown and non-text parts as base64 images, while the Android message model/history parser has no structured card, template-data, action, or presentation field. The PR is also paired with the same author's still-open iOS/macOS companion #42350, and the proposed client-side renderer still needs product/security review before it is either merged or deliberately moved behind a plugin/client-renderer API.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If native Android rich cards are desired, rework it around the current MessagePresentation direction or a documented Android client-renderer extension path, preserve ordered mixed text/card content, support multiple cards and template data, add defensive render limits plus URL/action policy, and add focused Android parser/render tests. If maintainers decide this belongs outside core, first define the missing Android client-renderer API that would let the external openclaw-adaptive-cards plugin provide native chat rendering, then close this together with the companion client PRs.
