## Summary

Keep this PR open. Current main still does not implement Android inline Adaptive Card rendering: Android chat renders text through Markdown and non-text parts as base64 images, with no Adaptive Card parser/composable, structured card model, template data path, or MessagePresentation client renderer. The PR also has a same-author open iOS/macOS companion, #42350, and links an external openclaw-adaptive-cards plugin, so this needs maintainer product and security review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer product and security review. If native Android rich chat is desired, rework it around MessagePresentation or a documented client-renderer extension path, preserve ordered mixed text/card content, support multiple cards and template data, add defensive render limits and URL policy, wire or explicitly disable actions, and add focused Android parser/render tests. If maintainers decide Adaptive Cards should stay outside core, close this together with #42350 after documenting the missing client-renderer API needed by the external openclaw-adaptive-cards plugin.
