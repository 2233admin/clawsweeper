## Summary

Keep open. Current main still bypasses webhook session reuse for isolated hook-originated agent runs, and this PR is paired with open bug #70894 via closing syntax, so the cleanup path is maintainer review/landing or a narrow equivalent rather than closure.

## What This Changes

The PR changes isolated cron agent preparation to avoid forcing a fresh session for hook/webhook-sourced isolated runs and adds a regression test for webhook session reuse.

## Best Possible Solution

Land this PR or a narrow equivalent that derives `forceNew` from true scheduled isolated-cron semantics, permits policy-approved hook/webhook isolated turns to reuse fresh stable sessions, and keeps regression coverage proving both webhook reuse and normal isolated cron freshness.
