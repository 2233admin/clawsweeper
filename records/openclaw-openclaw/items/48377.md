## Summary

Keep this PR open. Current main still does not implement create-with-content for `feishu_doc`: the create schema has no `content` field, and the runtime create branch only calls `createDoc`. The PR remains a focused implementation candidate for #48373, with no obvious security-sensitive scope creep in the provided diff.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #48373. The maintainer-quality path is to add create-with-content regression coverage, tighten empty-content semantics, and either accept the existing `writeDoc` reuse or switch to a populate/append helper that avoids clearing a newly created document.
