## Summary

Keep open. The PR addresses a real ACP/acpx orphan-process gap that is still present on current main and is related to the existing orphan-process reports, but the proposed patch is not ready to merge because the new cleanup path can make cancel/close fail when process listing is unavailable, and the wrapper's forced-kill fallback is scheduled after an immediate exit so it will not run.

## What This Changes

The PR adds an acpx process reaper, wires it into Gateway startup and acpx cancel/close, hardens generated ACP wrappers to signal child process groups, and updates tests, docs, and the changelog.

## Best Possible Solution

Keep the reaper scoped to the acpx plugin, make runtime process cleanup strictly best-effort and platform-aware, keep the wrapper alive until the SIGKILL fallback has either fired or the child has exited, add tests for process-list failure and SIGTERM-resistant children, and add the required changelog attribution before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep cleanup best-effort when process listing fails:** `extensions/acpx/src/runtime.ts:531`
  - body: `resolveCleanupContextFromRecord` already catches failures from `listProcesses`, but this line turns the empty fallback back into `undefined`, so `cleanupOpenClawOwnedAcpxProcessTree` calls `listUnixProcesses()` again. On Windows or any host where `ps` is unavailable, `/acp cancel` and `/acp close` can now reject from the `finally` block after the delegate operation succeeded. Pass the empty list through or swallow cleanup errors so lifecycle commands remain best-effort.
  - confidence: 0.9
- **[P2] Let the orphan SIGKILL timer run before exiting:** `extensions/acpx/src/codex-auth-bridge.ts:213`
  - body: The orphan watcher sends SIGTERM, schedules a SIGKILL fallback, and then exits the wrapper immediately. Once the wrapper process exits, the unref'd timeout will not run, so a child process group that ignores SIGTERM can still be left behind. Keep the wrapper alive until the child exits or the force-kill timer fires.
  - confidence: 0.88
- **[P3] Credit the changelog entry:** `CHANGELOG.md:35`
  - body: Root changelog policy requires every added user-facing entry to include at least one external `Thanks @...` attribution. This new entry has no credit, so it should be amended before merge.
  - confidence: 0.95
