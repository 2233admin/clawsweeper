## Summary

Keep open. Current main still lacks acpx/Codex ACP process-tree cleanup and the PR addresses a real orphan-process problem backed by live v2026.4.29 evidence, but the latest head still has a blocking wrapper-root mismatch plus best-effort cleanup gaps before merge.

## What This Changes

The PR adds an acpx process reaper, hooks startup/cancel/close cleanup for recorded OpenClaw-owned Codex ACP process trees, hardens generated Codex/Claude ACP wrappers, and updates tests, docs, and the changelog.

## Best Possible Solution

Keep the fix inside the acpx plugin, pass the generated wrapper root used by prepareAcpxCodexAuthConfig into startup reaping, preserve a recorded-root kill fallback when process enumeration is unavailable, and make unsupported startup scans quiet best-effort behavior before landing with the focused tests and changed gate.

Yes. ## Solution Assessment

No, not as-is. The acpx-local ownership-limited reaper is the right boundary, but the current PR must repair the wrapper-root mismatch, process-list failure fallback, and unsupported-scan warning behavior before it is the best maintainable fix.

  - confidence: 0.8
