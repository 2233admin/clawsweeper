## Summary

Keep open. Current main still has the acpx cleanup gap this PR targets, but the latest PR head is not merge-ready because startup reaping still filters generated wrappers with the configurable acpx session-store directory instead of the OpenClaw wrapper root, and unsupported Unix process scans still warn on startup.

## What This Changes

The branch adds an acpx process reaper, invokes it on service startup and runtime cancel/close, hardens generated Codex/Claude ACP wrappers, and updates acpx tests, docs, and the changelog.

## Best Possible Solution

Keep the fix in the acpx plugin boundary, but pass the generated wrapper root derived from ctx.stateDir into startup ownership checks, make unsupported startup process enumeration a quiet no-op, keep recorded pid cleanup scoped to OpenClaw-owned commands, then rerun the focused acpx tests and changed gate before maintainer review.

## Reproduction Assessment

Yes. Current main reproduces the cleanup gap by source inspection, and the PR-specific startup bug reproduces with a custom acpx stateDir because wrappers are generated under ctx.stateDir/acpx while the PR filters with pluginConfig.stateDir.

## Solution Assessment

No, not as-is. The acpx plugin is the right owner boundary and the acpx 0.6.1 session record contract supports pid cleanup, but startup ownership checks need the generated wrapper root and unsupported Unix process scans should not produce routine startup warnings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the generated wrapper state dir for startup reaping:** `extensions/acpx/src/service.ts:222`
  - body: The wrapper is generated under ctx.stateDir/acpx, but startup reaping filters wrapper ownership with pluginConfig.stateDir. With a custom acpx session-store stateDir, stale OpenClaw-generated wrappers no longer match the configured ownership path and remain orphaned. Pass the service state dir or generated wrapper root here instead of the session-store directory.
  - confidence: 0.9
- **[P3] Skip unsupported startup process scans quietly:** `extensions/acpx/src/service.ts:220-232`
  - body: Startup calls the stale-process reaper unconditionally, and the default lister shells out to ps -axo. On Windows or minimal hosts without ps, every acpx service start logs a warning even though PPID=1 orphan scanning is Unix-specific. Guard this path or make unsupported process listing return an empty result without warning.
  - confidence: 0.82
