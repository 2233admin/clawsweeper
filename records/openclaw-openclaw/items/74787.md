## Summary

Keep open. The PR is aimed at a real acpx orphan-process cleanup gap, but the latest head still has a blocking source-level mismatch: startup reaping filters against the configured acpx stateDir while generated wrappers are written under the plugin service stateDir, so custom stateDir setups can still skip stale wrappers. Two smaller cleanup robustness issues also remain.

## What This Changes

The PR adds acpx-owned process-tree reaping for generated Codex/Claude ACP wrappers and plugin-runtime-deps processes, wires cancel/close and startup cleanup, hardens generated wrappers, and updates tests, docs, and the changelog.

## Best Possible Solution

Keep the cleanup inside the acpx plugin boundary, pass the actual generated wrapper root or ctx.stateDir into startup reaping, add a direct recorded-root kill fallback when process listing is unavailable, make unsupported startup scans quietly no-op, and add focused tests for custom stateDir and list-failure cases before normal CI/review.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure acpx with a custom plugin stateDir, start the service so wrappers are generated under ctx.stateDir/acpx, then leave a stale PPID=1 generated wrapper there; the PR head filters startup reaping against pluginConfig.stateDir/acpx and skips it.

## Solution Assessment

No, not as-is. The owner boundary and general cleanup direction are maintainable, but the current diff should use the actual generated wrapper state root and preserve a recorded-root fallback before it reliably solves the orphan cleanup problem.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use the generated wrapper state dir for startup reaping:** `extensions/acpx/src/service.ts:222`
  - body: Startup writes Codex/Claude ACP wrappers under `ctx.stateDir/acpx`, but this call filters orphan wrappers against `pluginConfig.stateDir`. If acpx config uses a custom stateDir, stale generated wrappers under `ctx.stateDir/acpx` are skipped on startup; pass `ctx.stateDir` or the wrapper root into the reaper.
  - confidence: 0.94
- **[P3] Preserve recorded-root cleanup when listing fails:** `extensions/acpx/src/runtime.ts:511-512`
  - body: When process listing throws, this path stores an empty process snapshot and cleanup later walks that empty tree. Even with an owned recorded PID and `rootCommand`, cancel/close sends no signal because the root process is absent from the snapshot; add a direct-root fallback.
  - confidence: 0.86
- **[P3] Quiet unsupported startup process scans:** `extensions/acpx/src/service.ts:220-223`
  - body: Startup now always invokes the Unix `ps` scanner and logs failures as warnings. On Windows or hosts without that scanner, every acpx startup can emit an avoidable best-effort cleanup warning; guard unsupported platforms or make unsupported listing return an empty result.
  - confidence: 0.8
