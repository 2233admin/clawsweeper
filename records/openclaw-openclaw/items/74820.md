## Summary

Keep open for normal maintainer review. Current main still lacks WSL2 auto-disable in the Bonjour advertiser, while the PR is a narrow, documented, tested change at the existing disable control point with no blocking correctness or security finding identified in the diff.

## What This Changes

The PR adds `isWSL2Sync()` to the Bonjour advertiser auto-disable path, preserves `OPENCLAW_DISABLE_BONJOUR=0` opt-in behavior, and updates Bonjour/discovery tests and docs for WSL2.

## Best Possible Solution

Merge the WSL2 auto-disable behavior after maintainer review and targeted validation, while leaving #74778 and #74209 to cover probing-loop recovery and broader bundled-plugin default policy separately.

## Reproduction Assessment

Yes. On current main, a WSL2 environment with no explicit `OPENCLAW_DISABLE_BONJOUR` override still falls through `isDisabledByEnv()` and would proceed to create the ciao service; the PR’s mocked WSL2 unit test is the focused reproduction path.

## Solution Assessment

Yes. The PR uses an existing public runtime-env helper at the existing Bonjour disable boundary, keeps the explicit opt-in override ahead of environment auto-disable, and updates the relevant docs and tests without moving policy into core.
