## Summary

Keep open. Current main still does not auto-disable the Bonjour advertiser on WSL2, while the proposed code change is narrow, uses an existing public runtime-env helper, and fits the existing container auto-disable pattern. I found one low-priority docs follow-up because the current discovery docs describe automatic disablement as container-only behavior.

## What This Changes

The PR adds WSL2 detection to the Bonjour advertiser auto-disable path and adds unit coverage for WSL2 auto-disable plus explicit opt-in with `OPENCLAW_DISABLE_BONJOUR=0`.

## Best Possible Solution

Merge the narrow Bonjour WSL2 auto-disable behavior after aligning the public Bonjour/discovery docs with the new default and validating the targeted advertiser tests; #74778 can still be reviewed independently because it addresses ciao probing-state recovery rather than WSL2 environment skipping.
