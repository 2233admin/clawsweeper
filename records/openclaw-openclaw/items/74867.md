## Summary

Keep open. Current main still lacks the SDK-facing environment discovery RPCs and still documents/tests `oc.environments.list/status` as unsupported, so this PR is not obsolete; it is an active additive Gateway/SDK protocol implementation tied to maintainer issue #74708 and should get maintainer API review plus duplicate-branch reconciliation with #74852.

## What This Changes

The PR adds typed read-only `environments.list` and `environments.status` Gateway RPCs with SDK call-through/types, `operator.read` scope classification, protocol schemas/generated Swift models, docs, changelog, and tests while leaving create/delete unsupported.

## Best Possible Solution

Land one canonical additive implementation for read-only environment discovery after maintainer review, keeping the protocol typed, generated clients/docs/tests in sync, and preserving unsupported create/delete plus per-run environment selection until provisioning semantics are designed.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction: the SDK methods throw explicit unsupported errors, the SDK test asserts those errors, and Gateway neither advertises nor registers `environments.*` handlers.

## Solution Assessment

Yes, with maintainer review. The proposed Gateway handler plus protocol schema, SDK call-through, generated clients, docs, and tests is the narrow maintainable shape for the requested read-only surface, but maintainers should settle vocabulary and choose between this PR and #74852.
