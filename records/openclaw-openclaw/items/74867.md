## Summary

Keep open. Current main still leaves SDK environment discovery unsupported and has no Gateway environment RPC registration, while this PR is a relevant active implementation for the maintainer-labeled environment discovery request. The remaining work is maintainer API review and choosing the canonical implementation against the overlapping open PR, not conservative cleanup closure.

## What This Changes

The PR adds read-only `environments.list` and `environments.status` Gateway RPCs with SDK call-through types, `operator.read` scope classification, protocol schemas/generated Swift models, docs, changelog, and focused tests while leaving create/delete unsupported.

## Best Possible Solution

Land one narrow additive read-only environment discovery contract if maintainers accept the API shape: typed Gateway schemas and handlers, SDK call-through, least-privilege read scoping, generated native models, docs, and tests, with provisioning, create/delete, and run-time environment selection kept unsupported until their semantics are ready.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction: the SDK throws explicit unsupported errors for `oc.environments.list()` and `oc.environments.status(...)`, and the Gateway method registry/handler map lacks matching `environments.*` methods.

## Solution Assessment

Yes, with maintainer review. The PR follows the narrow additive direction for read-only discovery, but maintainers still need to decide the final environment summary vocabulary and canonical branch because #74852 overlaps the same request.
