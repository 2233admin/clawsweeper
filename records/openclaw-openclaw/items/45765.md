Current main still has the reported nesting behavior: both state-dir and config-dir resolution append `.openclaw` beneath the effective `OPENCLAW_HOME`. The issue also has an open related fix PR, so it should stay open until that PR lands or is deliberately replaced.

Keep this issue open and review #45784 as the active fix candidate, or replace it with a complete PR that covers both `resolveStateDir()` and `resolveConfigDir()`, adds regression tests for `OPENCLAW_HOME=~/.openclaw` and legacy state dir names, preserves the non-state-dir `OPENCLAW_HOME=/srv/app` behavior unless maintainers intentionally change the contract, and updates docs/changelog if the user-visible semantics change.
