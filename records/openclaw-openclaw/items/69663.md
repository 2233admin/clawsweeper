## Summary

Current main still uses shallow skill discovery, so this PR is not obsolete. The linked #56915 user problem remains reproducible from code inspection: nested layouts such as `skills/coze/koze-retrieval/SKILL.md` are skipped unless the first child directory itself has `SKILL.md`.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land one focused recursive skill-discovery implementation for #56915 after resolving the minor review comments and verifying traversal containment, loop protection, and limits; close overlapping PRs only after maintainers choose or merge the canonical fix.
