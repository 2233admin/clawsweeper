## Summary

Keep open. Current main still does not discover grouped workspace skills for the reported layout, and this PR is still relevant to #56915, but the proposed recursive scanner is not ready because it can traverse arbitrarily large contained subtrees before existing skill-loading limits apply. There is also now an overlapping bounded implementation candidate in #72534, so maintainer judgment is needed before landing or deduping either branch.

## What This Changes

The PR makes local skill discovery recursive, lets workspace skill roots fall through into child directories that lack `SKILL.md`, and adds a nested workspace skill regression test.

## Best Possible Solution

Land one bounded grouped-skill discovery fix for #56915 that preserves root-containment, symlink-loop protection, file-size limits, and candidate caps, with regression coverage and docs/changelog. If #72534 becomes the canonical branch, close or supersede this PR after that path is accepted; if this PR is used instead, it should first replace unbounded recursion with an explicitly bounded traversal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Bound recursive skill discovery by configured limits:** `src/agents/skills/local-loader.ts:152`
  - body: The new walker recurses through every contained directory before `loadWorkspaceSkillEntries()` can apply `maxCandidatesPerRoot` or `maxSkillsLoadedPerSource`. A configured skill root with a large nested tree can make startup or `openclaw skills list` traverse the whole subtree, so please bound the walk by depth/candidate count or keep discovery to the grouped layout being fixed.
  - confidence: 0.84
