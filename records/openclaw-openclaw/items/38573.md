## Summary

Current main still forces isolated cron deliveries with a resolved thread/topic id through the direct outbound-delivery path, and the public cron delivery contract still has no supported selector for text-only topic/forum completions to opt into the main-session announce/agent-turn route. The related subagent announce work exists, but it is separate from isolated cron completion delivery.

## What This Changes

Request adding a backward-compatible config or per-cron option that lets text-only topic/forum cron completions use the main-session announce/agent-turn flow instead of always using direct delivery.

## Best Possible Solution

Keep this open for maintainer product/API design. A good solution should preserve direct delivery as the default for thread/topic targets and structured payloads, add a backward-compatible opt-in for text-only isolated cron completions to use the main-session announce/agent-turn flow, define explicit fail-closed versus direct-fallback behavior, and update cron types/normalization, Gateway protocol schema, agent cron tool schema, CLI/UI surfaces if applicable, docs, and regression tests.
