## Summary

Current main still forces isolated cron deliveries with a resolved thread/topic id through direct outbound delivery, and there is still no supported cron option for text-only topic/forum completions to opt into the main-session announce/agent-turn route. The related subagent announce work and Telegram topic routing fixes are real context, but they do not solve this cron completion-route feature request.

## Best Possible Solution

Keep this open for maintainer product/API design. A good fix would preserve direct delivery as the default for thread/topic targets, add a backward-compatible cron completion-route option for text-only isolated cron completions, keep structured payloads on direct delivery, define fail-closed versus direct-fallback behavior, and update cron types/normalization, gateway schema, agent tool schema, CLI/UI if applicable, docs, and regression tests.
