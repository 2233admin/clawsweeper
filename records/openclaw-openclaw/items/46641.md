## Summary

Keep this issue open. Current main has opt-in compaction lifecycle notices and a silent pre-compaction memory flush, but it still does not implement the requested user-facing pre-threshold prompt, custom prompt/threshold config, or response window before flush/compaction.

## Best Possible Solution

Keep the issue open and design a core compaction UX feature under `agents.defaults.compaction`: add an opt-in user-notification prompt with configurable message, threshold, and timeout; deliver it before memory flush or preflight compaction; capture a same-conversation response when available; feed that response into the memory flush/compaction context; keep existing `notifyUser` lifecycle notices separate; and update schema/help/docs/tests.
