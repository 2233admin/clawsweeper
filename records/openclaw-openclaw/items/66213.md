## Risks / Open Questions

- The PR touches GitHub Actions release automation and CODEOWNERS, so it should get release-owner review before merge even though it is small.
- The polling loop currently masks gh api auth, rate-limit, and GitHub outage errors as zero assets, which can produce misleading 45-minute failures.
- The loop sleeps after the last poll and then fails without a final asset recheck, so assets appearing during the final minute could be reported missing.
- The tag regex is prefix-only and looser than .github/workflows/macos-release.yml, and writing the tag to GITHUB_OUTPUT should be paired with strict full-string validation.
- Whether missing appcast.xml should warn or fail is a release policy decision; docs describe appcast advancement as part of stable macOS readiness.
- The CODEOWNERS addition names individual maintainers for /appcast.xml rather than the existing openclaw-release-managers team, so ownership should be confirmed.
