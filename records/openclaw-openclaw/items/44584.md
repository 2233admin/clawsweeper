## Clownfish Work Prompt

Fix the open Discord thread metadata + CLI routing regression described here. Scope the PR narrowly.

Observable bugs:
1. Discord thread starter/history and volatile inbound metadata can still be reinjected on follow-up turns.
2. `openclaw agent --channel discord --to <thread-id>` / `--to channel:<thread-id>` resolves from `--to` alone and can land in `agent:*:main` instead of an existing Discord thread-bound session.

Expected implementation boundary:
- In `src/agents/command/session.ts`, extend `resolveSessionKeyForRequest` (and callers) with optional delivery context inputs such as `channel`, `accountId`, and normalized target. Reuse an existing stored session when its `deliveryContext` or `lastChannel`/`lastTo`/`lastAccountId` matches the requested delivery context. Normalize Discord bare snowflakes and `channel:<id>` to the same thread/channel target shape used by inbound Discord sessions. Keep old behavior for new sessions when no existing match is found.
- In `src/commands/agent-via-gateway.ts` and embedded/local agent command paths, pass the normalized channel/account context into session resolution where appropriate.
- For the metadata half, either fix the repeated Discord thread starter/history behavior in `extensions/discord/src/monitor/message-handler.process.ts`, `src/auto-reply/reply/get-reply-run.ts`, and `src/auto-reply/reply/inbound-meta.ts`, or explicitly leave only the metadata-only portion to #41355 with tests proving the CLI routing fix does not regress.
- Update `docs/cli/agent.md` if `--channel` now participates in existing-session reuse; preserve the distinction that delivery overrides should not blindly create unrelated session keys.
- Do not edit compiled `dist/` bundles or generated/baseline files.
- Changelog: add a user-facing fix entry only if runtime behavior changes, with a real reporter credit, not `@codex`.

Related refs: this issue, #41355, #44447, #44449, #37047.

Validation to run: targeted tests for session resolution, CLI agent command/gateway dispatch, Discord thread context handling, and inbound metadata; then changed gate in Testbox per repo policy if code changes are made.
