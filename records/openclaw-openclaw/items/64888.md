## Summary

Keep open. Current main has some adjacent lifecycle hardening, but the two concrete requests here are not fully implemented: the embedded-run liveness union still has no `cancelled` state, user abort/no-payload outcomes still resolve to `blocked`, and the main backend attempt await is not wrapped in a catch path that records fallback terminal lifecycle metadata before rethrowing.

## Best Possible Solution

Keep this issue open for a focused agents runtime patch. The likely fix is to add `cancelled` to `EmbeddedRunLivenessState`, return it for user aborts that are not timeouts, update lifecycle/wait/downstream consumers and tests, and add an error-boundary path around the backend attempt or lower-level attempt lifecycle so fallback terminal metadata is recorded before rethrowing uncaught failures.
