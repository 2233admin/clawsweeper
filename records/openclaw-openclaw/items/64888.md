## Summary

Keep open. Current main and the latest release still lack the requested cancelled liveness state, and uncaught embedded-attempt errors still have paths that rethrow without terminal replay/liveness metadata. The related merged harness-seam work is useful context but explicitly left this cancellation/liveness cleanup to focused follow-up work.

## What This Changes

Request to make the embedded runner emit terminal lifecycle metadata on uncaught attempt errors and distinguish user cancellation with a cancelled liveness state.

## Best Possible Solution

Add cancelled to the embedded liveness contract, return it for user-initiated aborts that are not timeouts, preserve blocked for hard failures/timeouts, and ensure every embedded rethrow/error path emits explicit replay/liveness terminal metadata with focused regression coverage.
