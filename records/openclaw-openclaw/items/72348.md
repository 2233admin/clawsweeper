## Summary

Keep this PR open. Current main still advertises Nextcloud Talk reactions and contains the HTTP reaction sender, but the Nextcloud Talk plugin still does not register a message actions adapter, so the shared `message` tool has no production dispatch path for `react`. The PR remains a focused implementation candidate for linked bug #70110, and the supplied diff is consistent with that scope.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent Nextcloud Talk-owned fix that adds a `ChannelMessageActionAdapter`, registers it on the plugin, dispatches `react` to `sendReactionNextcloudTalk`, preserves the existing SSRF-guarded sender behavior, and keeps focused adapter plus sender tests. The linked bug #70110 should remain open until this PR or an equivalent fix merges.
