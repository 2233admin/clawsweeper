## Summary

Keep this issue open. Current main is still a Node-hosted TypeScript/Gateway runtime with in-process plugin and agent-harness extension points, not a runtime-agnostic agent-core protocol for bare-metal, C/Rust/WASM, or ESP32-class ports. MimiClaw is useful external prior art, and the earlier maintainer/Codex comment already kept the issue open; related #45496 was narrower browser/WASM work and is closed, so it is not a canonical tracker for this embedded/no-Node request.

## Best Possible Solution

Keep this open for explicit maintainer product/API judgment. A strong path would define a stable runtime-agnostic agent-core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, platform adapters, and conformance tests or a small reference implementation that proves an OpenClaw-compatible loop can run outside the Node-hosted Gateway. MimiClaw should stay linked as prior art; #45496 should not be treated as the canonical tracker.
