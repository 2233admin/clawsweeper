## Summary

Keep this issue open. Current main is still a Node-hosted TypeScript/Gateway runtime built around the embedded Pi runner and in-process plugin harnesses; it does not define a runtime-agnostic agent-core protocol for bare-metal, C/Rust/WASM, or ESP32-class ports. MimiClaw remains useful prior art, and the existing maintainer/Codex discussion already kept this open rather than treating the narrower closed browser/WASM item as canonical.

## Best Possible Solution

Keep this open for explicit maintainer product/API judgment. A strong path would define a stable runtime-agnostic agent-core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, platform adapters, and conformance tests or a small reference implementation proving an OpenClaw-compatible loop can run outside the Node-hosted Gateway. MimiClaw should stay linked as prior art; #45496 should not be treated as the canonical tracker.
