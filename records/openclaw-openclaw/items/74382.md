## Summary

Keep open. Current `main` still has the reported onboarding UX ambiguity, and two open unmerged PRs already target this issue directly, so the right cleanup is PR review/deduplication rather than closing the issue before a fix lands.

## What This Changes

Make `openclaw onboard` avoid implying the keyless local `openai-whisper` skill needs OpenAI or ElevenLabs API keys when later prompting for separate API-backed skills.

## Best Possible Solution

Land or consolidate one onboarding UX fix that clearly separates local dependency installation from API credential setup, keeps local `openai-whisper` keyless, names API-backed `openai-whisper-api` and `sag` in the credential phase, and adds regression coverage for that mixed path.
