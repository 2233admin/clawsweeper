## Summary

Keep open. Current main still lacks `before_persistence_write` or an equivalent durable-write context. The public hook surface covers generic tool calls, transcript/message persistence, and an observer-only `cron_changed` lifecycle hook, but durable memory, cron, and config writes still do not expose a stable pre-write target/provenance/veto event.

## Best Possible Solution

Keep this issue open for focused core/plugin-SDK design. The best path is an additive `before_persistence_write` hook or explicit durable-write context on `before_tool_call`, with stable target/session/path/provenance metadata, at least blocking semantics, and targeted SDK/docs/tests covering memory, file, cron, config, and preference writes. Coordinate the design with #48503, #48517, and #48519.
