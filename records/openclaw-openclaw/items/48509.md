## Summary

Keep open. Current main still exposes only the existing tool/transcript hooks (`before_tool_call`, `after_tool_call`, `tool_result_persist`, and `before_message_write`), and those do not provide the requested durable-write event for memory/file/cron/config/preference changes. The related security/provenance proposals remain open, so this is not implemented, stale, or safely superseded.

## Best Possible Solution

Keep this open for a focused core/plugin-SDK design. The likely path is an additive hook or explicit `before_tool_call` persistence context that fires in the memory, file, cron, config, and preference write paths before durable commit, supports blocking, and lands with SDK types, docs, and targeted tests coordinated with #48503 and #48519.
