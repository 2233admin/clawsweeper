## Summary

Keep open. Current main still leaves a configured placeholder literal in resolved logger settings and log tailing, so the PR addresses a real logging surface gap, but the current patch has timezone-portability test failures and an adjacent status-display mismatch to handle before merge.

## What This Changes

The PR resolves `YYYY-MM-DD` placeholders in logger file paths, updates rolling-path recognition, and adds regression coverage for resolved logger settings and log tailing.

## Best Possible Solution

Resolve the placeholder support with host-local date-safe tests, share or explicitly scope the configured-log-file resolver used by status output, then land after the targeted logging tests and changed gate are green.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use a host-local date in the log-tail test:** `src/logging/log-tail.test.ts:60-63`
  - body: The logger formats dates with local `getFullYear()`/`getMonth()`/`getDate()`. Freezing the test at `2026-04-27T02:03:04.000Z` makes `America/Los_Angeles` resolve `2026-04-26`, but the test writes and expects `openclaw-2026-04-27.log`, so the new test fails on non-UTC hosts. Construct the fake time from local components or use a local-noon instant before asserting the filename.
  - confidence: 0.92
- **[P3] Use a host-local date in the settings test:** `src/logging/logger-settings.test.ts:54-61`
  - body: This test has the same local-date contract. `2026-03-09T08:30:00.000Z` happens to stay March 9 in many zones, but it becomes March 8 in zones such as `Pacific/Honolulu`, making the expected hard-coded date wrong. Use a local `new Date(2026, 2, 9, 12, 0, 0)` style fixture so the test matches the logger contract everywhere.
  - confidence: 0.82
- **[P3] Share placeholder resolution with status output:** `src/logging/logger.ts:690-697`
  - body: The new resolver stays private to `logger.ts`, while daemon status still reports `resolveConfiguredLogFilePath(cliCfg)`, whose helper returns `config.logging.file` unchanged. With a placeholder config, logging and tailing can use the dated path while status still shows the literal placeholder. Share the resolver with `src/logging/log-file-path.ts` or make the status boundary explicit.
  - confidence: 0.84
