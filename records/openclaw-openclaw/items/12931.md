## Summary

Keep open. Current main partially addresses the token-cost problem for ordinary bare text-channel resets by returning a static acknowledgement, but it still lacks the requested public reset-greeting policy and the gateway agent reset path still builds a hardcoded greeting prompt and dispatches an agent run using the normal request/session model selection path.

## What This Changes

Request to add configuration for bare `/new` and `/reset` so the reset greeting can use a cheaper model or be suppressed entirely.

## Best Possible Solution

Ship one typed public reset-greeting policy that covers static chat acknowledgements, reply-producing reset turns, and gateway agent resets, with schema/types/help/docs/tests aligned and existing `/new <message>` and `/reset <message>` behavior preserved.
