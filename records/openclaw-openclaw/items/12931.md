## Summary

Current main still only partially addresses the request. Bare chat `/new` and `/reset` now return static acknowledgements without invoking the embedded model, but gateway/CLI-style `agent` resets still build and dispatch a Session Startup greeting prompt, and the strict `session` config surface still has no `resetGreeting`, `resetGreetingModel`, or equivalent greeting policy/model override.

## Best Possible Solution

Keep this issue open for maintainer product follow-up. Either document that static acknowledgements are intentionally chat-only and gateway `agent` resets remain model-run Session Startup greetings, or add a typed reset-greeting policy/model override and apply it consistently across chat and gateway agent reset paths with schema, docs, and focused tests.
