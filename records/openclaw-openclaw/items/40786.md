## Summary

Keep open: current main still lacks native `.gitignore`-style `--exclude` or `--exclude-file` support for `openclaw backup create`. The Keep My Claw plugin linked in the discussion is a useful workaround, but it does not close the native CLI feature gap, and open PR #44288 explicitly targets this issue.

## What This Changes

The issue requests native `.gitignore`-style exclusions for `openclaw backup create`, including repeatable exclude patterns, an exclude-file option, and sensible defaults for dependency, cache, log, and secret-like files.

## Best Possible Solution

Keep this as the user-facing tracker for native backup exclude support. The desired end state is a core backup implementation with repeatable patterns, exclude-file parsing, clear default and protected-path semantics, manifest behavior that avoids leaking sensitive filenames, regression coverage, and public docs at https://docs.openclaw.ai/cli/backup.
