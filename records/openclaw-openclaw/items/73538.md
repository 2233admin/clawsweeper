## Summary

Keep this PR open. Current main does not already implement the PR’s intended Signal group-id allowlist behavior: the Signal group access path still compares `groupAllowFrom` entries only through the sender-identity comparator, and the event handler does not pass the inbound `groupId` into `resolveSignalAccessState`. The linked Signal bug report remains open, so this PR is still an implementation candidate rather than obsolete maintainer-cleanup material.

## Best Possible Solution

Keep this PR open for Signal maintainer review. The best path is to decide the intended Signal semantics, then either land a Signal-local fix that wires `groupId` into the access callback with regression tests and docs/config wording updates, or revise the PR/closing syntax so the remaining `requireMention` part of #53308 stays tracked separately.
