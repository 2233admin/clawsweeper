## Summary

Keep open. Current main still implements and documents default gateway/node host exec as `security="full"` with `ask="off"`, and the approval predicate does not prompt in that mode. The code has narrow guards for `/approve` and interactive `openclaw channels login`, but I did not find a hard guard or confirmation path for package-manager self-removal commands such as uninstalling OpenClaw itself. Related #54416 is adjacent exec-safety work, not a full superseding tracker for this specific self-uninstall case.

## What This Changes

The issue requests that OpenClaw avoid confusing similarly named packages with OpenClaw and require explicit confirmation before destructive host exec commands, especially uninstalling OpenClaw itself.

## Best Possible Solution

Add an explicit execution-time safety boundary for high-risk host commands, at minimum package-manager removal of OpenClaw itself and confusing near-name uninstall targets, or document that YOLO intentionally permits that risk. The fix should live in the shared exec approval/analysis path so gateway and node host behavior stay consistent, with tests covering default full/off, allowlist/on-miss, strict modes, and related prompt-routing surfaces.
