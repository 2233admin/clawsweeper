## Summary

Keep this issue open. Current main still only restarts the OpenClaw-managed systemd user service during `openclaw update`; system-level `/etc/systemd/system` gateway units are detectable elsewhere for doctor/status, but the updater does not use that path to restart them or print the correct `sudo systemctl restart ...` hint. The related ClawSweeper items cover user-systemd or LaunchAgent restart behavior, not this system-wide systemd update path.

## Best Possible Solution

Add system-level systemd handling to the update restart path: reuse the existing system service detector, identify active OpenClaw gateway system units when no managed user unit is found, avoid installing a duplicate user unit, and either restart the active system unit with an explicit privilege flow or print the exact `sudo systemctl restart <unit>` command. Add focused updater tests for an active `/etc/systemd/system` service plus docs wording in the update guide.
