## Repair Work Prompt

Implement a conservative config-first fix for https://github.com/openclaw/openclaw/issues/45210. Add `session.maintenance.pinnedKeys?: string[]` as a generic core maintenance setting. Pinned keys should be normalized consistently with existing session-key handling and should be exempt from `pruneAfter`, `maxEntries`, and disk-budget session-entry eviction. For `maxEntries`, pinned sessions should not count against the unpinned retention cap. Preserve the existing active-session protection behavior, but do not add a `/pin` chat/native command in this PR. Update config types, zod/schema/help/labels/generated config metadata as required, session maintenance/runtime cleanup, cleanup dry-run/action reporting, docs, tests, and changelog with `Thanks @YorrickBao`. Do not make Telegram-specific cleanup policy; Telegram topics should work through their existing topic-qualified session keys.
