## Summary

Keep open. Current main has partial bootstrap and recovery paths, but normal Gateway startup still intentionally blocks missing or non-local `gateway.mode`, and the WebUI no-provider crash plus Raw mode complaint lack enough concrete reproduction detail to choose a narrow code fix.

## What This Changes

The issue requests that Gateway and Control UI startup work without requiring all configuration, model, or provider settings, and adds that Raw mode is not working.

## Best Possible Solution

Maintain the existing safety guard unless maintainers choose a new configless normal-start policy; then split the WebUI/provider and Raw mode claims into focused reproductions with regression tests for the exact startup, model-list, config snapshot, and Raw editor states.

## Reproduction Assessment

Unclear. Current source and tests reproduce the intentional Gateway blocking behavior, but the report does not provide enough detail to reproduce a WebUI no-provider crash or a Raw mode regression.

## Solution Assessment

Unclear. A blanket removal of startup/config guards is not clearly the best fix; the maintainable path is a maintainer decision on Gateway bootstrap policy plus narrow regressions for any proven WebUI or Raw mode failure.
