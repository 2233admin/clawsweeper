## Summary

Keep open. Current main still only accepts the global `*` wildcard or exact normalized origins for `gateway.controlUi.allowedOrigins`; it does not support `chrome-extension://*`. The related open PR #38290 is narrower exact-origin work, and the direct wildcard PR #39867 is closed unmerged.

## Best Possible Solution

Keep this issue open for explicit gateway product/security judgment. If accepted, implement a narrow `chrome-extension://*` matcher in `src/gateway/origin-check.ts`, preserve literal `Origin: null` rejection, exact HTTP(S) matches, global `*`, host-header fallback, and local-loopback behavior, then add focused unit and WebSocket handshake tests plus docs/security guidance. Coordinate with #38290, but do not close this wildcard request until wildcard support is merged or a maintainer explicitly rejects the capability.
