## Summary

Current main still does not support `chrome-extension://*` or any scheme-level wildcard in `gateway.controlUi.allowedOrigins`. The issue remains valid, but it should stay open for explicit gateway/security product judgment because this is an origin allowlist policy change, not a mechanical cleanup close.

## Best Possible Solution

Keep this issue open for explicit gateway product/security judgment. If accepted, implement a narrow `chrome-extension://*` matcher in `src/gateway/origin-check.ts`, preserve literal `Origin: null` rejection, exact HTTP(S) matching, global `*`, host-header fallback, and local-loopback behavior, then add focused unit and WebSocket handshake tests plus docs/security guidance. Coordinate with #38290, but do not close the wildcard request until wildcard support is merged or a maintainer explicitly rejects it.
