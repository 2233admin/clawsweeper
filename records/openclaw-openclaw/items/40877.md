## Summary

Keep open. Current main still has the synchronous iOS permission snapshot path that this PR is trying to replace: `makeConnectOptions` still calls `currentPermissions()` directly, and `currentPermissions()` still calls `SFSpeechRecognizer.authorizationStatus()`, creates a fresh `CLLocationManager()`, and calls `CLLocationManager.locationServicesEnabled()` on that path. The PR’s async permission snapshot/reconnect guard work and new regression tests have not been absorbed into main, and related #51433 was closed unmerged rather than superseding the full speech + location warning cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or a narrowed equivalent that moves speech/CoreLocation permission snapshot work off the warning-prone main-actor path, avoids throwaway CoreLocation permission checks, preserves gateway auth/TLS/reconnect behavior across async gaps, and keeps focused regression tests plus live iOS warning validation.
