## Summary

Keep open. Current main still has the synchronous iOS permission snapshot path this PR is trying to change: `makeConnectOptions` calls `currentPermissions()` synchronously, and `currentPermissions()` still calls `SFSpeechRecognizer.authorizationStatus()`, constructs a throwaway `CLLocationManager()`, and calls `CLLocationManager.locationServicesEnabled()` on the main-actor controller. The PR is recent, unmerged contributor work with focused iOS controller/test changes, and the narrower related #51433 was closed unmerged rather than superseding this work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent narrowed implementation that moves the speech permission probe off the main actor, stops the throwaway CoreLocation permission snapshot behavior, preserves gateway auth/TLS/reconnect semantics across async gaps, and keeps focused regression tests for the permission snapshot and reconnect/refresh race cases.
