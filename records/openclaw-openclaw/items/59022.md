## Summary

Keep open. Current main has the Gateway-side A2UI route and scoped canvas auth, and an unauthenticated direct `/__openclaw__/a2ui/` returning 401 is expected. The reported macOS node failure still has a live source gap: `MacNodeRuntime` resolves A2UI through `GatewayConnection.shared.canvasHostUrl()` and returns `A2UI_HOST_UNAVAILABLE` after one readiness poll, while iOS and Android retry by refreshing the node canvas capability.

## Best Possible Solution

Keep this issue open and fix the macOS node A2UI path: pass or expose the node `GatewayNodeSession` scoped canvas host URL to `MacNodeRuntime`, retry failed A2UI readiness with `refreshNodeCanvasCapability()` before returning `A2UI_HOST_UNAVAILABLE`, and add macOS regression coverage for stale or missing scoped A2UI capability. A small docs or diagnostic clarification that unauthenticated direct `/__openclaw__/a2ui/` returns 401 would also reduce confusion.
