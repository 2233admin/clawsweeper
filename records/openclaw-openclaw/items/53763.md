## Summary

Keep open. Current main has a bundled browser plugin with managed headless launch, remote CDP, Chrome MCP existing-session attach, and optional Docker browser installation, but normal local launch still resolves only explicit/default/system Chrome/Brave/Edge/Chromium executables and reports an error when none exists. The central request for a first-class browser runtime that works without a host-installed browser remains unresolved.

## What This Changes

Request to bundle a Playwright/Puppeteer Chromium runtime behind the existing browser tool so agents can access JS-heavy and login-required pages without relying on host Chrome or third-party web APIs.

## Best Possible Solution

Define a maintainer-approved browser-runtime packaging path, then wire executable discovery, doctor repair, Docker/install docs, and update policy around that path so managed browser profiles can start in headless/server deployments without requiring a preinstalled host browser.
