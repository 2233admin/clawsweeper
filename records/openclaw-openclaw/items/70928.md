## Summary

Keep open. Current main still leaves WebChat, Gateway-backed TUI, and embedded TUI final-delivery paths outside the hook-aware outbound seams, and the open related umbrella/narrow issues do not supersede this default-surface reproduction.

## What This Changes

The issue asks for WebChat and openclaw-tui assistant replies to fire outbound lifecycle hooks such as `message_sending`, `message_sent`, and `before_message_write` consistently with other delivery paths.

## Best Possible Solution

WebChat and TUI final delivery should invoke the applicable pre-send and post-send hooks exactly once with stable session/run context, apply rewrites or cancellations consistently, and route Gateway-injected assistant transcript appends through the same `before_message_write` guard used by normal persisted messages.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: WebChat and Gateway TUI route through `chat.send`, that path currently bypasses the hook-aware dispatcher and final sent-hook emitters, and Gateway-injected assistant transcript appends bypass `runBeforeMessageWrite`; embedded TUI finals also emit directly.

## Solution Assessment

Yes. The best next step is a focused chat-surface parity fix using existing hook seams; the broader outbound observer design can remain tracked by #50126 rather than blocking this default-surface repair.
