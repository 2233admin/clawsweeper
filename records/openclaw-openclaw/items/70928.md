## Summary

Current main still matches the reported default-surface hook gap. The WebChat/Gateway TUI `chat.send` path still builds a raw reply dispatcher and bypasses the hook-aware dispatch wrapper, the dispatcher has no `message_sent` emission, and Gateway-injected assistant transcript appends still write through `SessionManager.appendMessage` without the existing `before_message_write` hook wrapper. Related hook work is still open or closed unmerged, so this is not obsolete.

## What This Changes

The issue requests outbound lifecycle hooks such as `message_sending`, `message_sent`, and `before_message_write` to fire for WebChat and openclaw-tui assistant reply paths.

## Best Possible Solution

Keep this issue open until the Gateway `chat.send` and TUI reply paths use a hook-aware outbound lifecycle seam or equivalent local handling. The end state should fire `message_sending` before visible delivery, emit `message_sent` after success/failure with session/run context, apply `before_message_write` before Gateway-injected assistant transcript appends, and document any intentionally unsupported streaming/internal-hook semantics.
