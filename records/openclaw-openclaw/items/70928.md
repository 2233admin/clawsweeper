## Summary

Current main still matches the reported hook coverage gap for the WebChat and Gateway-backed TUI `chat.send` outbound path. The code still builds a raw reply dispatcher and calls `dispatchInboundMessage` directly, so it bypasses the wrapper that wires `message_sending`; the reply dispatcher does not emit `message_sent`; and Gateway-injected assistant transcript appends still use raw `SessionManager.appendMessage` without the `before_message_write` wrapper. The related hook-wiring PR is closed unmerged, while the broader umbrella issue remains open and does not supersede this default-surface bug.

## Best Possible Solution

Keep this issue open. Fix the Gateway `chat.send` path used by WebChat and Gateway-backed TUI by routing assistant reply emission through a shared hook-aware outbound observer/dispatcher, or by adding equivalent `message_sending` and `message_sent` handling around the current internal UI delivery path. Apply `before_message_write` to Gateway-injected assistant transcript appends, and add focused regression tests for WebChat `chat.send` plus Gateway TUI `sendChat` coverage. Coordinate broader unification with #50126, but do not close this specific default-surface regression until code and tests cover it.
