## Summary

Keep #39961 open. Current main still does not implement the proposed streaming text-repetition guard, and the existing shipped loop guard remains scoped to tool-call loops. The PR is unique, unmerged core agent-streaming work with useful review discussion, not an obsolete or duplicate cleanup candidate.

## Best Possible Solution

Keep #39961 open for maintainer engineering review. The best path is to land a robust core streaming text-repetition guard with config/schema/docs/tests covering abort behavior, queued assistant events, chunked and non-chunked replies, final flush, and lifecycle fallback behavior, or have a maintainer explicitly decide OpenClaw should not add this core guard.
