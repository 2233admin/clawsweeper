## Summary

Keep this open. Current main still has the host-side scheduling shape described in the report: bootstrap maintenance runs as foreground, only turn maintenance can be deferred to the background worker, and deferred compaction execution is only enabled for background runs. The hot-cache details look plugin-owned, but the OpenClaw host contract still lacks a bootstrap/reconcile debt-consuming follow-up path.

## What This Changes

The issue asks OpenClaw to prevent bootstrap/reconcile and hot-cache maintenance paths from leaving deferred context-engine compaction debt recorded but unconsumed.

## Best Possible Solution

Add a generic host-side maintenance contract or scheduler path so bootstrap/reconcile-created deferred compaction debt can request and receive a background debt-consuming follow-up, with regression coverage proving the bootstrap path gets an allowed pass. Keep Lossless Claw cache policy in the plugin and only adjust OpenClaw core where the lifecycle contract is generic.
