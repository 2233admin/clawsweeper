## Summary

Keep open. Current main only partially overlaps with the PR: non-batch memory indexing concurrency is now configurable through `memorySearch.remote.nonBatchConcurrency`, but embedding retry attempts and backoff remain hardcoded. The PR is still relevant to #68812, but it is not merge-ready because it reads an invalid top-level config path, bypasses the typed/validated memory config contract, would need to preserve current-main concurrency behavior after rebase, and adds an unrelated root runtime log file.

## What This Changes

The PR exports memory embedding retry/concurrency defaults, adds overridable getters in the embedding manager, overrides them from `cfg.memorySearch.embedding.*`, and adds a root `openclaw-2026-04-15.log` file.

## Best Possible Solution

Revise this PR or replace it with a narrow implementation that adds typed, schema-validated, agent-scoped retry/backoff settings to the existing resolved memory config path, keeps the shipped `remote.nonBatchConcurrency` behavior for concurrency, updates docs/help/generated schema and regression tests, and removes the log artifact.
