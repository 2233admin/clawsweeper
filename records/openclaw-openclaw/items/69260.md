## Summary

Keep open. Current main has partial ACP env hardening for the default OpenClaw ACP client bridge, but the ACPX harness path still lacks the requested explicit auth contract, Gemini-specific env scrub policy, and fail-closed auth preflight hook; this remains a real auth-boundary design issue rather than an implemented cleanup close.

## What This Changes

The issue requests auth-contract metadata, provider auth env scrubbing, fail-closed preflight hooks, and mixed-credential documentation for ACP-backed Gemini agent integrations.

## Best Possible Solution

Keep this open for a maintainer-designed ACP auth contract: add a generic, typed way for ACP agents to declare expected auth mode, fallback policy, env-scrub requirements, and preflight checks, then implement the Gemini policy through that contract with focused tests and mixed-credential docs.
