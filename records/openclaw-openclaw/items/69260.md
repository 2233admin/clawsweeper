## Summary

Keep this issue open. Current main still does not implement the requested ACP auth-contract surface: ACP runtime/config types lack expected-auth/fallback/scrub/preflight metadata, the ACP manager does not pass an auth policy into runtime.ensureSession, and the acpx wrapper still delegates non-Codex agents such as Gemini without Gemini/GenAI env scrubbing or fail-closed auth preflight. Existing generic ACP client env stripping covers the OpenClaw ACP bridge spawn path, but not the embedded acpx Gemini harness path described by the report.

## Best Possible Solution

Keep this open for maintainer product/security review. The best path is an additive ACP auth-contract seam shared by global ACP config, per-agent runtime defaults, persistent bindings, and runtime.ensureSession; then acpx can implement Gemini-specific Gemini/GenAI env scrubbing and fail-closed preflight through that seam, with mixed-credential docs and disposable-state regression coverage.
