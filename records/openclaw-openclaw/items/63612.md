## Summary

Keep open. Current main still sends pre-prompt compaction history through the pinned upstream token estimator without malformed-message guards, replay normalization still happens after that precheck, and PR #63636 remains open and unmerged as the direct implementation candidate.

## What This Changes

A long-lived main session can crash before provider dispatch when malformed replay history reaches compaction token estimation and upstream `estimateTokens()` reads missing `.length` fields.

## Best Possible Solution

Add a shared guarded token-estimation path, or normalize all history before every compaction and pre-prompt estimation call, with focused regression tests for malformed assistant, toolResult/custom, bashExecution, and summary messages.
