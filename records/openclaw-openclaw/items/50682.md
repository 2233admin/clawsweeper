## Risks / Open Questions

- Merging as-is could warn or leave groups that are intentionally covered by wildcard `channels.telegram.groups: {"*": ...}` if `group-policy-allowlist-empty` is treated as an unknown chat instead of a sender-authorization failure.
- Native slash commands can bypass the proposed unknown-group action path and keep using the current fixed rejection behavior.
- Forum General-topic warning sends need `buildTelegramThreadParams`; directly passing `message_thread_id=1` can fail before a leave action sends its warning.
- The proposed cooldown tracker should be bounded or prune expired entries, and `warn` without `unknownGroupMessage` should produce validation or startup diagnostics.
- No supply-chain regression was visible in the diff, but automatic `leaveChat` is a destructive channel action and needs precise classification and tests before landing.
