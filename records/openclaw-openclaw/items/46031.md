## Summary

Keep open. Current main still matches the reported behavior, and an open PR already references this issue as its fix candidate, so closing or queueing duplicate automated work would be premature.

## What This Changes

The issue asks for implicit GitHub Copilot profile selection to honor `auth.order` instead of selecting the first stored `auth.profiles` entry.

## Best Possible Solution

Fix the plugin-owned `resolveFirstGithubToken` path to iterate order-aware eligible Copilot profile IDs while preserving environment-token precedence, and cover both implicit catalog discovery and memory embeddings with focused regression tests.

## Reproduction Assessment

Yes. Configure two stored `github-copilot` token profiles, set `auth.order.github-copilot` to prefer the non-first profile, and run the implicit Copilot helper path with no env token; current main still selects `profileIds[0]`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the GitHub Copilot plugin helper using the existing order-aware auth seam, not in a new core provider-specific path.
