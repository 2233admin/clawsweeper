## Summary

Keep open. Current main still uses the bundled GitHub Copilot plugin's `resolveFirstGithubToken` path, which enumerates profiles with `listProfilesForProvider` and selects `profileIds[0]` instead of the order-aware SDK helpers, so `auth.order` can still be ignored. The issue is also paired with open PR #48682 using closing syntax, so the right maintainer path is to review or rework that PR rather than close this issue early.

## Best Possible Solution

Keep the issue open and rework/review PR #48682 as the implementation lane. The fix should move into the current GitHub Copilot plugin helper (`extensions/github-copilot/auth.ts`) using the order-aware plugin SDK helpers, cover catalog discovery and the embedding caller if they share the helper, and add focused tests for non-first `auth.order`, unresolved ordered tokenRef fallback, env precedence, and unchanged explicit provider behavior.
