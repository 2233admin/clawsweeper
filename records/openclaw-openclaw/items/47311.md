## Summary

Keep this issue open. Current main has adjacent infrastructure: WebChat can upload audio files, Gateway `chat.send` can stage non-image attachments into media paths, and media understanding can transcribe audio with `tools.media.audio`. The requested WebChat server-side dictation path is still missing: there is no MediaRecorder/getUserMedia batch-STT control that records browser microphone audio and inserts the server transcript into the chat draft/input.

## Best Possible Solution

Implement a Control UI/WebChat dictation path that records browser microphone audio with `getUserMedia` plus `MediaRecorder`, sends the recording through an authenticated Gateway path that reuses `tools.media.audio`/media-understanding transcription, and inserts the returned transcript into the draft. Keep this distinct from realtime Talk and generic file attachments, and update Control UI/WebChat/audio docs and focused tests.

## Clownfish Work Prompt

Implement the server-side STT dictation path requested by the open WebChat voice-input issue. Current main already supports audio file attachments and `tools.media.audio` transcription, but WebChat has no MediaRecorder microphone control that records browser audio and inserts the server transcript into the draft.

Expected boundary:
- Add a Control UI/WebChat microphone dictation control separate from realtime Talk.
- Use `navigator.mediaDevices.getUserMedia({ audio: true })` plus `MediaRecorder` for a completed recording.
- Send the recording through an authenticated Gateway path that reuses existing media-understanding/audio transcription (`tools.media.audio`) and returns text. Prefer a narrow additive RPC if `chat.send` would incorrectly start an agent run; do not overload realtime Talk.
- Insert the returned transcript into the composer draft; do not auto-send unless existing UI patterns clearly require it.
- Reuse existing attachment size/MIME validation, cleanup, auth scopes, and error presentation patterns.
- Update docs to distinguish browser Web Speech/browser dictation, server-side STT dictation, generic audio file attachments, and realtime Talk.
- Add a user-facing changelog entry with thanks to @jmomford and @martin5858 if the repo’s active changelog policy requires it.

Related refs: this issue is canonical; #73634 was closed as duplicate; #67465 and #40242 are Talk/realtime voice context and should not be treated as implementing this batch STT request.

Likely files: `ui/src/ui/views/chat.ts`, `ui/src/ui/controllers/chat.ts`, `ui/src/ui/app.ts`, `ui/src/ui/chat/attachment-support.ts` or a new `ui/src/ui/chat/server-dictation.ts`, `src/gateway/server-methods/chat.ts` or a new server-method module, `src/gateway/protocol/schema/*` if adding an RPC, `src/media-understanding/runtime.ts`, `docs/web/control-ui.md`, `docs/web/webchat.md`, `docs/nodes/audio.md`, and focused colocated tests.
