## Summary

Keep open. Current main still has the reported Mattermost draft-preview overwrite path, and the issue is paired with open closing PR #75256, so policy says to review/land or revise that PR instead of closing this issue early.

## What This Changes

The issue requests a Mattermost preview streaming mode that can split draft preview content into separate posts at reasoning, assistant, and tool boundaries instead of repeatedly editing one Mattermost post.

## Best Possible Solution

Keep the fix Mattermost-local: add a validated Mattermost preview streaming mode that preserves current edit-in-place behavior by default and, in block mode, splits the draft preview at assistant, reasoning, and tool boundaries with focused schema, account-resolution, draft-stream, and monitor tests. Coordinate the final shape with #75256 and #73252 so one coherent config contract lands.

## Reproduction Assessment

Yes. The code path is reproducible from current main: phase callbacks call `draftStream.update(...)`, the draft stream edits an existing `streamPostId`, and nothing in the Mattermost monitor calls `forceNewMessage()` at those boundaries.

## Solution Assessment

Yes. The requested Mattermost-local split mode matches existing preview-streaming seams used by Discord and avoids changing shared core dispatch; the only caveat is coordinating the overlapping open PRs before landing.
