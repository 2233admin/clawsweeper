## Summary

Keep open. Current main still routes Mattermost draft-preview phase changes through one editable preview post, lacks a Mattermost preview `streaming.mode` schema/runtime gate, and the issue is already paired with open closing PR #75256 plus overlapping streaming-mode work in #73252.

## What This Changes

Add a Mattermost preview streaming mode that can split draft-preview phase transitions into separate posts so prior visible preview content is preserved.

## Best Possible Solution

Keep the fix Mattermost-local: accept and validate the canonical preview streaming mode, preserve current edit-in-place behavior by default, and in block mode split draft previews at assistant, reasoning, tool-start, and tool-end boundaries with focused Mattermost schema, account-resolution, draft-stream, and monitor tests.

## Reproduction Assessment

Yes. The issue gives a concrete Mattermost DM/tool-call reproduction path, and current source confirms the code path: each transition updates the same draft stream post until `forceNewMessage()` or cleanup clears the post id.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix because Discord already has a channel-local boundary-splitting seam, and Mattermost can match that behavior without changing shared dispatch semantics; the implementation should be reviewed together with #75256 and #73252.
