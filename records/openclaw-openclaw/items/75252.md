## Summary

Keep open. Current main still routes Mattermost reasoning, partial-reply, and tool-start draft previews into the same editable post and does not call the existing `forceNewMessage()` seam at transition boundaries. The issue is now paired with open PR #75256 and overlaps the older open streaming-mode PR #73252, so it should remain open until maintainers land or replace the canonical fix.

## What This Changes

Mattermost draft preview should support a boundary-splitting streaming mode so thinking, partial replies, and tool-status transitions do not overwrite prior visible content in the same post.

## Best Possible Solution

Keep the fix Mattermost-local: land one canonical implementation that resolves the effective preview streaming mode from Mattermost config, preserves current edit-in-place behavior by default, and in block mode splits the draft stream before phase changes can overwrite prior preview posts. The fix should align schema/types/docs/generated metadata and add focused Mattermost draft-stream, account/config, and monitor tests.

## Reproduction Assessment

Yes. The issue provides a live Mattermost DM/tool-call path, and current source gives a high-confidence code reproduction: successive lifecycle callbacks all call `draftStream.update(...)`, which edits the existing `streamPostId` until `forceNewMessage()` or cleanup clears it.

## Solution Assessment

Yes, the requested Mattermost-local streaming-mode split is the narrow maintainable direction. The best current path is to review and reconcile the open closing PR #75256 with overlapping PR #73252 rather than queueing a separate automated repair.
