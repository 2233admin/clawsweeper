## Summary

Keep open. Current main still has the unguarded assistant `surface_error` throw path, and the reported `failoverFailure=false` with classified `failoverReason` shape is supported by the current runner, policy, and helper contracts. The PR is a focused source/test fix with no visible security-sensitive surface changes.

## What This Changes

This PR adds a `failoverFailure` guard to assistant `surface_error` throwing and adds a regression test for successful turns that retain stale classified `errorMessage` text.

## Best Possible Solution

Land this PR or an equivalent minimal change that requires `params.failoverFailure` before throwing from the assistant `surface_error` branch, while preserving #70848/#70124 behavior for genuine provider failures and keeping the new regression test.
