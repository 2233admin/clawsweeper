## Summary

Current main still does not implement onboarding-time multi-model selection. The existing model system supports primary models, fallbacks, allowlists, and configure-time multi-select, but the first-run setup wizard still prompts for one default model and applies only that primary model.

## What This Changes

Request onboarding support for selecting multiple configured models while choosing one default model to reduce cost and switching friction.

## Best Possible Solution

Keep this open for maintainer product direction. A narrow implementation would reuse `promptModelAllowlist`, `applyModelFallbacksFromSelection`, and `applyModelAllowlist` in `src/wizard/setup.ts` after provider authentication/default-model selection, preserving the chosen primary model while adding onboarding tests and docs updates. If maintainers want true multi-provider endpoint setup during first run, that should be split into a broader onboarding UX design.
