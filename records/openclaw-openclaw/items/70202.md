## Summary

Keep this issue open. Current main has the underlying model infrastructure: primary model plus fallbacks, an allowlist-backed `/model` picker, chat/UI model switching, and `openclaw configure` multi-select support. The specific onboarding request is still missing: `src/wizard/setup.ts` calls the single default-model picker and applies only `applyPrimaryModel`, while the multi-select allowlist/fallback helpers are wired into configure instead of onboarding.

## Best Possible Solution

Keep this open for maintainer product direction. The narrow implementation path is to reuse `promptModelAllowlist`, `applyModelFallbacksFromSelection`, and `applyModelAllowlist` in `src/wizard/setup.ts` after provider auth/default-model selection, preserve the chosen primary model, add onboarding tests, and update onboarding docs. If maintainers want true multi-provider endpoint setup during first run, split that into a broader onboarding UX design rather than hiding it inside the allowlist/fallback step.
