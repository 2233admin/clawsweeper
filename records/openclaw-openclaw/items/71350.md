## Summary

Current main still only partially covers the request. Per-agent `thinkingDefault` and flat `agents.list[].params` are implemented and documented, but the proposed per-agent per-model `agents.list[].models` map and rules/escalation/AGENTS.md-backed `thinkingStrategy` surface are still absent, and the related open items do not fully supersede this broader product/API request.

## What This Changes

Request per-agent thinking defaults, per-agent model/generation parameter overrides, and adaptive thinking strategies based on message rules, failures, or AGENTS.md policy.

## Best Possible Solution

Keep the issue open as the broader product/API discussion, while preserving the shipped `agents.list[].thinkingDefault` and flat `agents.list[].params` behavior. Maintainers should decide whether the remaining work belongs in core as `agents.list[].models` plus a `thinkingStrategy` policy API, or should be split so #47852/#32496 handle generation-parameter forwarding and a separate design thread handles adaptive thinking rules and escalation.
