## Summary

Keep open. Current main implements the first slice of the request with per-agent `thinkingDefault` and flat `agents.list[].params`, but the broader public API is still missing: there is no per-agent `agents.list[].models` map and no `thinkingStrategy` rules, escalation, or AGENTS.md-sourced policy surface. The related open issue and PR overlap but do not fully supersede this broader design request.

## What This Changes

Request to add per-agent thinking defaults, per-agent generation/model-parameter overrides, and adaptive thinking strategies for rules, failure escalation, and AGENTS.md-driven policy.

## Best Possible Solution

Keep this open as the maintainer design tracker or split it into narrower canonical issues: preserve the shipped per-agent `thinkingDefault` and flat `params` behavior, decide whether per-agent per-model maps belong in core, and define any adaptive thinking strategy contract separately with versioned schema, runtime semantics, docs, and tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is statically reproducible on current main by inspecting the strict agent schema, public config types, runtime params merge, docs, and tests: `agents.list[].models` and `thinkingStrategy` are not supported surfaces.

## Solution Assessment

Unclear. The current flat `agents.list[].params` implementation is a narrow maintainable answer for per-agent temperature-style overrides, but the requested per-model map and adaptive strategies need maintainer product/API direction before an implementation can be judged best.
