## Summary

Keep this PR open. Current main still serializes ACP session rows through the generic session model path, does not read `entry.acp.runtimeOptions.model`, and still falls back to `DEFAULT_MODEL` for rows with no resolved model. The PR targets a real remaining bug, but it needs review/rebase because current main has newer session model-provider preservation work and the PR has an unresolved P1 automated review comment on that exact provider edge case.

## Best Possible Solution

Keep this PR open. The best path is to rebase/update it onto current main, keep explicit `modelProvider` authoritative for stored runtime models, add ACP-specific regression coverage for `entry.acp.runtimeOptions.model`, and ensure ACP rows without an explicit ACP model do not inherit the configured default model.
