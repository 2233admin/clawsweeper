## Summary

Keep open. Current main still routes MiniMax through the generic hybrid Anthropic/OpenAI replay hooks, so MiniMax `anthropic-messages` still inherits strict tool-call ID sanitization. The PR is a narrow provider-owned fix with a matching regression-test change, and I found no blocking correctness or security issue in the provided diff.

## What This Changes

The PR wraps MiniMax's hybrid replay hooks to disable `sanitizeToolCallIds` for `anthropic-messages` while preserving strict sanitization for OpenAI-compatible MiniMax paths, and updates the MiniMax provider hook test expectation.

## Best Possible Solution

Land this PR or an equivalent MiniMax-owned replay-policy hook after maintainer review, keeping the override scoped to MiniMax `anthropic-messages` while leaving OpenAI-compatible MiniMax replay strict. A shared factory option can be a later cleanup if more providers need the same knob.
