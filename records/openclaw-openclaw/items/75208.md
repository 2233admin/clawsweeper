## Summary

Keep open. Current main still lacks the artifact RPCs, and this active public Gateway/App SDK API PR still has blocking correctness issues plus overlap with the open maintainer artifact track in #74706/#74926, so it needs maintainer reconciliation rather than cleanup closure.

## What This Changes

The PR adds Gateway and SDK artifact list/get/download RPCs with cursor pagination, type filtering, schemas, generated Swift models, docs and tests, plus browser Hyprland screenshot/config changes and a task-status lookup helper.

## Best Possible Solution

Reconcile the artifact API through the maintainer-owned #74706/#74926 path, fix or drop cursor pagination until it has a unique stable artifact position, keep provenance/download guardrails aligned with the canonical implementation, and split the Hyprland browser work into a separately wired and reviewed change if it is still wanted.

## Reproduction Assessment

Yes. For the artifact bug, create one transcript message with two artifact blocks, call `artifacts.list` with `limit: 1`, then request the next page with the returned cursor; the second artifact is skipped because the cursor stores only `messageSeq`. For the Hyprland bug, set `browser.hyprlandCapture: true` through normal config resolution and observe that `resolveBrowserConfig` does not populate `resolved.hyprlandCapture`.

## Solution Assessment

No. The artifact RPC direction matches the maintainer request, but this patch is not the best final form while pagination is message-sequence-only, the browser side change is unrelated and not wired through real config, and the public contract overlaps the open maintainer hardening PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve artifacts that share a message sequence:** `src/gateway/server-methods/artifacts.ts:388`
  - body: The cursor only encodes `messageSeq`. If one transcript message contains more artifacts than the page limit, `nextCursor` encodes that message's sequence and the next request filters with `messageSeq > cursorSeq`, so the remaining artifacts from the same message are never returned. Include a tiebreaker such as `messageSeq` plus `contentIndex` or artifact id and add a page-split regression test.
  - confidence: 0.95
- **[P2] Wire the Hyprland toggle through resolved config:** `extensions/browser/src/browser/routes/agent.snapshot.ts:493`
  - body: The route checks `ctx.state().resolved.hyprlandCapture`, but the PR only adds the field to the resolved type/schema; `resolveBrowserConfig` never copies the raw browser setting into the returned config, and the raw `BrowserConfig` type is not updated. In a real gateway this condition stays falsey, so the new capture path cannot run outside the test harness.
  - confidence: 0.9
- **[P3] Resolve Hyprland tools from trusted paths:** `extensions/browser/src/browser/hyprland-capture.ts:8`
  - body: `collectSpawnStdout` runs `hyprctl` and `grim` through the process `PATH`. Since this is a new Gateway subprocess path, resolve those binaries with the existing trusted system-bin helper before spawning so a shadowed PATH entry is not executed when the feature is enabled.
  - confidence: 0.78
