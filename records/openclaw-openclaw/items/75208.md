## Summary

Keep open. Current main still lacks SDK-facing artifact RPCs, so this PR remains relevant, but the proposed patch has blocking pagination and browser-config issues and overlaps the open maintainer artifact API track in #74706/#74926.

## What This Changes

The branch adds Gateway and SDK artifact list/get/download RPCs with cursor pagination, type filtering, schemas, generated Swift models, docs and tests, plus an unrelated Hyprland browser capture path and config schema changes.

## Best Possible Solution

Land one maintainer-approved artifact RPC implementation that keeps stable per-artifact pagination, preserves provenance and download guardrails from the maintainer track, updates protocol/SDK/generated/docs surfaces together, and splits Hyprland capture into a separately wired browser change if still wanted.

## Reproduction Assessment

Yes. For the artifact bug, create one transcript message containing two artifact blocks, call `artifacts.list` with `limit: 1`, then request the next cursor page; the second artifact is skipped because the cursor only records `messageSeq`. For the Hyprland bug, enable `browser.hyprlandCapture` through normal config and inspect `resolveBrowserConfig`, which never returns that resolved field.

## Solution Assessment

No. The artifact RPC direction matches the maintainer request, but this patch is not the best merge path while pagination lacks a per-artifact tiebreaker, the Hyprland flag is not wired through real resolved config, and the branch overlaps the maintainer hardening PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve artifacts that share a message sequence:** `src/gateway/server-methods/artifacts.ts:388`
  - body: The cursor only encodes `messageSeq`. If one transcript message contains more artifacts than the page limit, the next request filters with `messageSeq > cursorSeq`, so remaining artifacts from that same message are never returned. Include a tiebreaker such as content index or artifact id and add a page-split regression test.
  - confidence: 0.95
- **[P2] Wire the Hyprland flag through resolved config:** `extensions/browser/src/browser/routes/agent.snapshot.ts:496`
  - body: The route checks `ctx.state().resolved.hyprlandCapture`, but the resolver never copies a raw browser setting into the returned config. In a real gateway this stays falsey, so the new capture path cannot run outside the test harness.
  - confidence: 0.9
- **[P3] Resolve Hyprland tools from trusted system paths:** `extensions/browser/src/browser/hyprland-capture.ts:9`
  - body: `collectSpawnStdout` runs `hyprctl` and `grim` through the process `PATH`. Since this is a new Gateway subprocess path, resolve those binaries with the existing trusted system-bin helper before spawning so a shadowed PATH entry is not executed when the feature is enabled.
  - confidence: 0.78

## Security Review
