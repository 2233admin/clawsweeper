## Summary

Keep open. Current main has a later generic ACP dispatch retarget that likely covers the normal follow-up-message failure, but it does not fully implement this PR’s intended Telegram route/native-command/session-state fix: Telegram DM-topic paths still derive `:thread:<chatId>:<topicId>` keys after route resolution, and native command targets still prefer that derived key. The useful fix now needs to be ported from the old `src/telegram` paths to `extensions/telegram/src`.

## What This Changes

The PR adds a bound-route suffix guard to Telegram conversation routing, applies it to inbound, native-command, and session-state key derivation, and adds bound DM-topic regression coverage.

## Best Possible Solution

Port the bound-route suffix guard into the current Telegram extension code, keep unbound DM-topic isolation unchanged, and add regression coverage proving bound DM-topic messages, native commands, and session-state lookups preserve the exact ACP target session key.
