## Summary

Keep this PR open, but it needs changes before merge: the reported Telegram ACP DM-topic bug is still source-reproducible on current main, while the PR edits retired `src/telegram` paths that are absent from the active tree. The narrow guard is a good solution direction once ported to the bundled Telegram plugin under `extensions/telegram/src`.

## What This Changes

The PR adds a Telegram route-level flag and tests so ACP-bound Telegram DM-topic messages and native commands keep the already-bound session key instead of appending another DM-thread suffix.

## Best Possible Solution

Port the route-level bound-session authority signal into `extensions/telegram/src/conversation-route.ts` and consume it at every active Telegram DM-topic session-key derivation site, with regressions proving bound DM topics keep the ACP target key while unbound DM topics retain isolated thread sessions.

Yes. The PR body gives concrete Telegram steps, and current main has a source-level reproduction: runtime binding can replace `route.sessionKey` with an ACP target key, then active DM-topic paths append `:thread:<chatId>:<topicId>` after route resolution.

## Solution Assessment

No, not as written. The route-level guard is the narrow maintainable fix, but it must be applied to the active `extensions/telegram/src` plugin tree rather than retired `src/telegram` files.
