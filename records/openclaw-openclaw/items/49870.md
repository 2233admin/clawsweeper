## Summary

Keep open. Current main partially moved inbound conversation normalization into channel-owned resolvers, but it does not implement this PR's central agent-hook contract: `PluginHookAgentContext` still has no `conversationId` or `accountId`, and the embedded runner hook call sites still omit those fields. The PR also has unresolved review findings around its added `after_agent_complete` hook, so the useful path is a refreshed, narrower patch rather than cleanup closure.

## Best Possible Solution

Keep this PR open and ask for a refreshed patch on current main: preserve the channel-owned inbound conversation resolver direction, add `conversationId` and `accountId` to the current `src/plugins/hook-types.ts` `PluginHookAgentContext`/agent harness context, wire those fields through every agent hook call site including `before_model_resolve`, `before_prompt_build`, `before_agent_start`, `llm_input`, `llm_output`, `before_agent_finalize`, and `agent_end`, and replace or drop the PR's `after_agent_complete` design in favor of the current `before_agent_finalize`/`reply_dispatch` hook surfaces with focused regression tests.
