## Summary

Keep open. Current main has moved the inbound conversation normalization toward channel-owned resolvers, but it still does not implement the PR's central agent-hook contract: agent hook contexts do not expose conversationId or accountId, and the embedded runner still omits those values at the relevant hook call sites. The PR also carries unresolved functional blockers around its added after_agent_complete design, so it needs a refresh rather than cleanup closure.

## Best Possible Solution

Keep this PR open and ask for a current-main refresh: preserve the channel-owned inbound resolver approach already present, add conversationId and accountId to src/plugins/hook-types.ts PluginHookAgentContext and src/agents/harness/hook-context.ts AgentHarnessHookContext, wire params.messageTo and params.agentAccountId through all relevant embedded agent hook phases, and add focused regression tests for before_model_resolve/before_prompt_build/llm_input/llm_output/agent_end/before_agent_finalize context fields. The PR should replace or drop after_agent_complete in favor of the existing before_agent_finalize and reply_dispatch surfaces unless a maintainer explicitly wants a new hook design.
