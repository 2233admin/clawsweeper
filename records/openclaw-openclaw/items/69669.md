## Summary

Keep open. Current main still routes bound ACP conversation follow-ups directly into the ACP prompt path, and the regression coverage asserts the raw shorthand text reaches the ACP runtime before normal parent reply resolution. The paired same-author PR #69675 is still open and explicitly docs/policy-only, so it should be reviewed before deciding whether runtime semantics should change.

## What This Changes

Request that thread-bound ACP follow-ups preserve child session continuity while parent-orchestrating shorthand into explicit child tasks by default, with verbatim relay only on explicit request.

## Best Possible Solution

Keep the issue open while maintainers review #69675 and decide the ACP contract. If parent orchestration is accepted, implement a focused runtime distinction between shorthand parent-owned continuation and explicit verbatim ACP relay, then update sessions.send/bound dispatch behavior, accepted-note wording, docs, and regression tests together.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the bound ACP dispatch regression sends BodyForAgent "continue" and asserts runtime.runTurn receives that raw text while parent reply resolution is skipped.

## Solution Assessment

Unclear. The requested behavior may be the right UX for parent-orchestrated coding workflows, but current docs and tests encode direct ACP routing, so a maintainer semantics decision is needed before a runtime patch; the open docs-only PR is not sufficient to solve the observed runtime behavior.
