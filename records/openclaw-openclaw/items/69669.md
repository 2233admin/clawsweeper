## Summary

Keep open. Current main still routes bound ACP follow-ups directly into the ACP prompt path, and regression coverage asserts the raw follow-up text reaches the ACP runtime. The paired same-author PR #69675 is still described in the provided context as open and docs/policy-only, so this needs maintainer semantics review before any cleanup closure.

## What This Changes

Request to change ACP thread-bound continuation semantics so the parent rewrites shorthand follow-ups into explicit child prompts by default, with raw relay only on explicit request.

## Best Possible Solution

Keep this issue open, review the paired open PR #69675 first, and make an explicit maintainer decision on ACP follow-up semantics. If parent-orchestrated follow-ups are intended, implement a focused runtime change that distinguishes shorthand continuation from explicit raw relay, then align sessions.send, bound dispatch, accepted-note wording, docs, and regression tests.
