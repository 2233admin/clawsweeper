## Summary

Keep open. Current main has the overflow diagnostic/retry path, memory promotion preview/apply primitives, compaction rotation knobs, and session archive/delete support, but it still does not wire repeated overflow pressure into a deduped promote-then-rotate lifecycle suggestion with dry-run candidates and operator confirmation.

## What This Changes

Request to surface a one-time operator suggestion after repeated session context overflow events, precompute dry-run memory promotion candidates, and offer confirmed promote/rotate actions.

## Best Possible Solution

Approve a small lifecycle design, then implement a per-session overflow-pressure tracker that emits one deduped structured suggestion with a stable id and dry-run candidate count, exposes it through operator-visible CLI/UI surfaces, and requires explicit confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The existing overflow loop test gives a high-confidence simulated path for repeated context overflow, and current source inspection shows that path compacts/retries or returns `context_overflow` without any promote-then-rotate suggestion.

## Solution Assessment

Unclear until maintainers settle the lifecycle design. The issue's opt-in suggestion and confirmation model is safer than automatic promotion/rotation, but the exact state storage, dry-run API boundary, main-session rotation semantics, and UI/CLI surfaces need owner review first.
