## Summary

Keep this open. Current main has the overflow diagnostic, auto-compaction/retry path, `memory promote` dry-run/apply primitives, and transcript archive primitives, but it does not wire those into a repeated-overflow lifecycle suggestion, operator notification, CLI prompt, or promote-then-rotate confirmation flow. Because the proposed flow can mutate `MEMORY.md` and archive/rotate active sessions, it needs maintainer product review before a focused implementation PR.

## Best Possible Solution

Keep this issue open and define a maintainer-approved lifecycle design: track repeated overflow/failed compaction per session, emit a deduped structured suggestion with a dry-run candidate count, surface it in `openclaw sessions` and `openclaw memory status`, and require explicit operator confirmation before applying promotions or archiving/rotating the session.
