## Summary

Keep open. Current main has the underlying overflow diagnostics, bounded auto-compaction, pre-compaction memory flush, memory promotion preview/apply, REM preview, and session archive/delete primitives, but it still does not connect repeated overflow events to a deduped promote-and-rotate lifecycle suggestion or operator confirmation flow.

## What This Changes

Request to surface a one-time promote-then-rotate suggestion after repeated session context overflows, including dry-run memory candidates and explicit operator actions to promote, archive, and rotate the session.

## Best Possible Solution

Add a maintainer-approved lifecycle suggestion seam: persist repeated overflow state per session, compute a dry-run memory promotion preview through a public memory API, emit one deduped structured suggestion, surface it in `openclaw sessions` and `openclaw memory status`, and require explicit confirmation before promotion or session rotation.

## Reproduction Assessment

Yes for the feature gap: current-main inspection shows the overflow path logs and retries compaction, then returns reset/new advice, while targeted searches found no promote-and-rotate suggestion surface. A live long-session overflow would exercise the same code path, but the missing wiring is already reproducible from source.

## Solution Assessment

Unclear until maintainer design review. The requested direction is plausible and uses existing primitives, but the best first implementation should be suggestion-only with dry-run candidates and explicit confirmation because it coordinates memory writes, notifications, and session rotation.
