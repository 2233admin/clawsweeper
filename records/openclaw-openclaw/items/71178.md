## Summary

Keep open. Current main still has the `update.run` producer gap: restart continuations are consumed when present, and the agent restart path can create one, but `update.run` still does not accept or write a `continuation`, and tests still assert that field is absent. The closed, unmerged #71573 is useful implementation context rather than proof that this is fixed.

## Best Possible Solution

Keep this issue open and land a narrow, additive `update.run` continuation fix: accept optional `continuationMessage` in the update protocol, parse it through the shared restart request helper, write `buildRestartSuccessContinuation({ sessionKey, continuationMessage })` into successful update sentinels only when a restart will be scheduled, forward the field from the agent gateway tool, and update the tests that currently assert `continuation` is absent. After that lands, maintainers can decide whether the remaining graceful-drain and post-reset bridge pieces stay here or move to narrower follow-ups such as #73918.
