## Summary

Keep #71178 open. The current checkout at 998e37f still lacks the update.run restart-continuation producer path that the maintainer explicitly called out, and the current tests still assert that update.run sentinel payloads do not include a continuation. Main has partial restart deferral/recovery support, but the core update.run gap remains unshipped and PR #71573 is the related open fix in the supplied context.

## Best Possible Solution

Keep #71178 open until the update.run continuation fix lands on main. The narrow implementation should add continuationMessage to UpdateRunParamsSchema, plumb it through restart-request parsing, build a restart success continuation in src/gateway/server-methods/update.ts on successful updates with a sessionKey, forward continuationMessage from src/agents/tools/gateway-tool.ts, and update the tests that currently assert continuation is undefined. After that lands, maintainers should explicitly scope or split the remaining graceful-drain and post-reset bridge gaps.
