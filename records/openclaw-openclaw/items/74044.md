## What This Changes

The issue reports a Raspberry Pi arm64 Discord channel startup regression where enabling Discord wedges the gateway near 100% CPU and makes CLI/WebSocket probes time out until Discord is disabled.

## Best Possible Solution

Validate current main or the next packaged build on the reported Raspberry Pi arm64 setup with a real Discord bot token. If it still wedges, keep the fix inside the Discord plugin startup, gateway lifecycle, or voice surface, add a targeted regression for the narrowed failure mode, and document channels.discord.voice.enabled=false only if live testing confirms it is a reliable workaround.
