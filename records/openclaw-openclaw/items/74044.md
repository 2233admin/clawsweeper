## Summary

Keep this issue open. The report is fresh and concrete, and current main has related Discord gateway hardening, but I did not find direct source, test, release, or live-environment proof that it fixes the Raspberry Pi arm64 100% CPU Discord startup wedge under the default Discord configuration.

## What This Changes

The issue reports that enabling the Discord channel on OpenClaw 2026.4.26 can wedge the gateway at about 100% CPU on Raspberry Pi arm64, while disabling Discord makes gateway probes healthy again.

## Best Possible Solution

Validate current main or a nightly build on the reported Raspberry Pi arm64 setup with a real Discord bot token. If it still wedges, keep the fix inside the Discord plugin startup/lifecycle/voice surface, add targeted regression coverage for the narrowed failure mode, and document `channels.discord.voice.enabled=false` only as a confirmed workaround if live testing proves it avoids the spin.

## Reproduction Assessment

Unclear for current main. The issue gives a detailed reproduction path for OpenClaw 2026.4.26, but reproducing the current behavior requires Raspberry Pi arm64 plus live Discord credentials, and this environment has no Discord token/profile entry.

## Solution Assessment

Unclear. The existing current-main mitigations are in the right Discord gateway area, but they are not enough evidence to close; the safer path is live validation first, then a narrow Discord plugin fix if the wedge remains.
