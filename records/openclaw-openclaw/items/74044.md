## Summary

Keep this issue open. The report is fresh and concrete, and current main only shows adjacent Discord gateway lifecycle mitigations; it does not provide source, test, release, or live Raspberry Pi arm64 proof that the reported Discord startup CPU wedge is fixed.

## What This Changes

The issue reports a Raspberry Pi arm64 Discord channel startup regression where enabling Discord wedges the gateway at about 100% CPU while disabling Discord restores gateway health.

## Best Possible Solution

Validate current main or the next build on the reported Raspberry Pi arm64 setup with a real Discord bot token. If it still wedges, keep the fix inside the Discord plugin startup, lifecycle, or voice surface, add targeted regression coverage for the narrowed failure mode, and document channels.discord.voice.enabled=false only as a confirmed workaround if live testing proves it avoids the spin.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the reported release because the issue gives a concrete Pi arm64, Node 22.22.2, Discord-token startup path and a Discord-disabled recovery check; no for current main in this review because the required live hardware and credentials were not exercised.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The current lifecycle guards are in the right area but are not enough evidence to close; the safer path is live validation first, then a narrow Discord plugin fix if the wedge remains.
