## Summary

Keep PR #71257 open. Current main at 998e37fcb3d5b937fcf70d7509b677efba939f10 still reads assistant transcript text and writes it into session-memory files without artifact sanitization, so the linked core hook bug is not implemented on main. The PR is a plausible focused fix candidate, though it needs cleanup of generated artifacts and maintainer review against the overlapping open PR #69961.

## Best Possible Solution

Keep this open and have a maintainer choose or consolidate #71257 with #69961. The shipped fix should live in the bundled `session-memory` hook, reuse existing sanitizer/directive/media helpers where practical, sanitize before memory persistence and reuse, remove generated artifacts, and add regression coverage for both `getRecentSessionContent()` and the handler write path.
