## Summary

Keep open. Current main still has the reported Slack blind spot: Slack DM/main sessions and channel sessions are separate buckets, outbound Slack `message` sends are mirrored only into the destination session, and default session-tool visibility does not expose sibling channel sessions back to a DM turn. The recent maintainer review also explicitly kept this open and treated #64427 as adjacent rather than canonical.

## What This Changes

Expose an agent's own outbound Slack channel messages to DM/main sessions so it can recall and follow up on channel activity from a DM context.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp. It should expose only the agent's own outbound channel messages to DM/main sessions without merging full DM and channel transcripts or widening general session visibility.
