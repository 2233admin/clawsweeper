## Summary

Keep open. Current main still preserves the reported Slack cross-session blind spot: Slack DMs and channels use separate session buckets, outbound message-tool sends are mirrored only into the destination session, and default session-history visibility does not expose sibling channel sessions back to a DM/main session. The existing maintainer discussion also kept this open and treated #64427 as adjacent rather than a replacement.

## What This Changes

The issue requests Slack DM sessions to recall the agent's own outbound messages sent into Slack channels so the agent can answer follow-up questions about its channel activity.

## Best Possible Solution

Add a narrow Slack self-activity recall path, such as an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp. It should expose only the agent's own outbound channel messages to DM/main sessions without merging full DM and channel transcripts or widening general session visibility.
