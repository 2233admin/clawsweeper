## Summary

Current main still has the reported Slack visibility gap. Slack DM/main sessions and Slack channel sessions resolve to different session keys, outbound message-tool sends are mirrored into the resolved destination session, and the default session-tools visibility remains the current session tree rather than a same-agent cross-channel self-activity view. The adjacent open #64427 covers Slack DM history hydration, but it does not replace this cross-session own-channel-send recall request.

## Best Possible Solution

Keep this open and design a scoped Slack self-activity recall path. The maintainable shape is an outbound activity ledger or session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, exposed from DM/main context only for the agent's own outbound Slack sends. Add regression coverage for a Slack DM/main turn that posts to a Slack channel through the message tool and later recalls that sent message without broad cross-session transcript access.
