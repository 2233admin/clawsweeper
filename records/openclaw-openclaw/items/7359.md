## Summary

Current main still preserves the reported Slack visibility gap. Slack DMs and channels use different session keys, message-tool sends are mirrored to the destination Slack channel session, and the default session-history visibility policy does not give a DM session a scoped same-agent view of the agent's own outbound Slack channel activity. The open #64427 DM-history report is adjacent, but it does not replace this DM-to-channel self-send recall request.

## Best Possible Solution

Keep this open and implement a scoped Slack self-activity recall path. The best direction is a small outbound activity ledger or narrowly scoped session-history bridge keyed by agent id, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposed to DM/main sessions without merging Slack DM and channel sessions or making arbitrary channel transcripts visible.
