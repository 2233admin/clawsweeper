## Summary

Current main still separates Slack DM/main sessions from Slack channel sessions, and message-tool outbound sends are mirrored into the resolved destination session rather than a DM-visible self-activity record. The default sessions tool visibility remains the current session tree, so this is still a real product gap rather than an already-implemented behavior.

## Best Possible Solution

Keep this open and implement a scoped Slack self-activity recall path. The maintainable shape is an outbound activity ledger or session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, exposed from DM/main context only for the agent's own outbound Slack sends. Add regression coverage for a Slack DM/main turn that posts to a Slack channel via the message tool and later recalls that sent message without broad cross-session transcript access.
