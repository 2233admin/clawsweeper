## Summary

Keep open. Current main has addressed some audit bullets, but the broad Control UI UX audit is still only partially implemented: Instances remains presence-list/refresh oriented, Sessions still lacks an explicit key-copy affordance and keeps dense per-row advanced controls, Config still presents Save/Apply/Update without explanatory labels/tooltips, global shortcuts are limited to the command palette, and the event log is gateway-event based rather than a general action feedback/audit trail. The related #39780 is a narrower Config hardening request, not a replacement for this dashboard-wide audit.

## Best Possible Solution

Keep this open unless a maintainer intentionally splits it. The best path is focused Control UI follow-up work for per-instance actions and clearer presence/reason affordances, session key copy/truncation plus less repetitive advanced controls, clearer Config action labels/help, dashboard shortcut/help UX, and durable action feedback or activity history. Treat #39780 as a narrower Config hardening tracker only.
