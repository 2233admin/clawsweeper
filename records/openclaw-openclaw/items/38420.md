## Summary

Keep open. Current main has addressed several concrete UX audit bullets, including breadcrumb/header context, scrollable sidebar navigation, Overview token masking, Config invalid-state banner, Usage loading/error/empty states, and session-delete confirmation. The broad dashboard audit is still only partially implemented: Instances remains a presence list with refresh only, Sessions still lacks an explicit key-copy affordance and keeps dense per-row advanced controls, Config still exposes bare Save/Apply/Update actions, global shortcuts are limited to command-palette navigation, and the Overview event log is gateway-event based rather than a durable action feedback/audit trail. The related #39780 covers narrower Config hardening and does not supersede the remaining dashboard-wide work.

## Best Possible Solution

Keep this open unless a maintainer intentionally splits it. The best path is focused Control UI follow-up work for per-instance actions and clearer presence/reason affordances, session key copy/truncation plus less repetitive advanced controls, clearer Config action labels/help, dashboard shortcut/help UX, and durable action feedback or activity history. Treat #39780 as a narrower Config hardening tracker only.
