## Summary

Keep this PR open. Current main still has the raw transcript append gaps the PR targets, and the proposed branch is a relevant implementation candidate for #73565, but the latest diff still needs maintainer/security review and fixes for config threading and guarded append no-op handling before merge.

## What This Changes

The PR wraps delivery mirror, CLI/ACP transcript persistence, gateway transcript injection, and Codex app-server transcript mirror writes with the shared session transcript guard, adds update-mode support, exposes the guard through the agent harness runtime SDK barrel, and adds redaction/event/idempotency tests.

## Best Possible Solution

Land a corrected, maintainer-reviewed security fix that applies the shared transcript guard to every raw transcript append path, threads the active OpenClawConfig wherever the caller has one, handles guard-blocked writes explicitly, preserves single transcript update semantics, and keeps any SDK exposure intentionally approved and documented.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Thread config through delivery mirror callers:** `src/config/sessions/transcript.ts:168`
  - body: Adding `config` to the transcript helper does not make the delivery mirror paths honor it. Both `src/infra/outbound/deliver.ts` and `src/infra/outbound/outbound-send-service.ts` already have `cfg`, but the PR does not update their `appendAssistantMessageToSessionTranscript(...)` calls. After this patch those newly guarded paths still fall back to default redaction, so `logging.redactSensitive="off"` is silently ignored for delivered-message transcript mirrors.
  - confidence: 0.88
- **[P2] Handle blocked guard appends before returning success:** `src/config/sessions/transcript.ts:283-288`
  - body: Once this helper wraps the session manager, `appendMessage` can return `undefined` when a `before_message_write` hook blocks the write. The function still returns `{ ok: true, messageId }`, violating its success contract and making callers believe a transcript message was persisted. Check the append result and return an explicit no-op/failure result before reporting success.
  - confidence: 0.81
