Keep this PR open. The PR is a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Review this PR as a security fix candidate: keep the scoped guard-at-append approach, verify the SDK export is the narrowest acceptable plugin boundary for the Codex mirror, add or retain regression tests for the affected transcript sinks, then land it or a maintainer-owned equivalent and let it close #73565.
