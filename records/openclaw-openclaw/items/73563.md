## Summary

Keep this PR open. It addresses a real transcript-redaction gap, and the latest head appears to have fixed the earlier updateMode double-emit/suppression issue, but the patch still has merge-blocking security/API correctness problems: production mirror callers still omit active config, blocked guard writes can be reported as successful, and the internal guard is promoted through a public SDK barrel without a narrow documented contract.

## What This Changes

This PR wraps several session transcript append paths with guardSessionManager, threads redaction config/update-mode options through agent and Codex transcript mirrors, exports the guard from agent-harness-runtime, and adds transcript redaction/idempotency/update tests.

## Best Possible Solution

Land a corrected, maintainer-reviewed security fix that applies the shared transcript guard to every raw transcript append path, preserves existing transcript update semantics, threads the active OpenClawConfig through all production transcript mirror callers, treats guard-blocked appends as non-successful writes, and keeps any SDK exposure narrow, documented, and versioned.

## Reproduction Assessment

Yes. Current main shows raw SessionManager transcript append paths, and the PR-head regressions can be reproduced with a before_message_write hook that blocks an assistant write or with custom logging.redactPatterns/redactSensitive settings on delivery mirror paths whose callers still omit active config.

## Solution Assessment

No. The direction is right, but this PR head is not yet the best maintainable solution because it leaves production config passthrough incomplete, reports success for blocked writes, and promotes internal guard plumbing through a public SDK barrel.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Thread config through delivery mirror callers:** `src/config/sessions/transcript.ts:168`
  - body: Adding `config` to the transcript helper is not enough while production delivery mirror callers still omit `params.cfg` / `params.ctx.cfg`. Those mirrored transcript writes keep using default redaction, so custom `logging.redactPatterns` and `logging.redactSensitive: "off"` are ignored for delivered-message transcript persistence.
  - confidence: 0.88
- **[P2] Handle blocked guard appends before returning success:** `src/config/sessions/transcript.ts:290-291`
  - body: After wrapping this helper with `guardSessionManager`, `appendMessage` can return `undefined` when a `before_message_write` hook blocks the message or the guard drops it. Returning `{ ok: true, messageId }` here can report a successful transcript write with no persisted entry and no valid id.
  - confidence: 0.9
- **[P2] Keep transcript guard out of the public runtime barrel:** `src/plugin-sdk/agent-harness-runtime.ts:129`
  - body: Exporting `guardSessionManager` from `openclaw/plugin-sdk/agent-harness-runtime` promotes mutable SessionManager redaction/update plumbing as a third-party Plugin SDK contract. Use a narrow bundled/internal seam, or document and version this explicitly before syncing the public baseline.
  - confidence: 0.82
