## Summary

Keep this PR open. Current main still has the raw transcript append gaps this PR is trying to close, and the linked security issue remains open, but the latest patch still has merge-blocking correctness/API concerns around active config propagation, blocked guard writes, and public SDK exposure of security-sensitive transcript plumbing.

## What This Changes

The PR wraps several session transcript append paths with `guardSessionManager`, adds guard update-mode support, threads config into some CLI/ACP/Codex mirror paths, exports the guard through the agent harness runtime barrel, and adds transcript redaction/idempotency/update tests.

## Best Possible Solution

Land a corrected, maintainer-reviewed security fix that applies the shared transcript guard to every raw transcript append path, preserves existing transcript update semantics, threads the active runtime config through all production mirror callers, handles guard-blocked writes explicitly, and either keeps the guard internal to bundled code or documents/version-locks it as a supported Plugin SDK contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Thread config through mirror callers:** `src/config/sessions/transcript.ts:168`
  - body: Adding a `config` parameter to the transcript helper is not enough while the production outbound mirror callers still omit their active `cfg`. Those delivered-message transcript mirrors will keep using default redaction, so `logging.redactSensitive="off"` and custom `logging.redactPatterns` are silently ignored. Thread the active config through the mirror type and both outbound call paths.
  - confidence: 0.87
- **[P2] Handle blocked guard appends before returning success:** `src/config/sessions/transcript.ts:288-289`
  - body: After this helper wraps the session manager, `appendMessage` can return `undefined` when a before-write hook blocks the message or the guard drops it. The helper still returns `{ ok: true, messageId }`, so callers can observe success with no persisted transcript entry and an undefined id. Check the append result before returning success.
  - confidence: 0.84
- **[P2] Keep the transcript guard out of the public runtime barrel:** `src/plugin-sdk/agent-harness-runtime.ts:129`
  - body: Exporting `guardSessionManager` from `openclaw/plugin-sdk/agent-harness-runtime` promotes internal SessionManager redaction/update plumbing as public Plugin SDK API. The SDK boundary treats these subpaths as third-party contracts, so use a narrow internal/bundled seam or document and version this as a supported contract before syncing the baseline.
  - confidence: 0.8
