## Summary

Keep this PR open. Current main still has the transcript append paths that the PR targets writing through raw `SessionManager.open(...).appendMessage(...)` or a manual before-write hook path rather than the shared guard, so the linked security gap is not implemented on main. The PR is a focused implementation candidate for the linked tracking issue, but it needs normal maintainer/security review rather than automated cleanup.

## What This Changes

The PR wraps session transcript append paths in core, gateway, CLI/ACP, and the bundled Codex mirror with the existing session transcript guard, threads config/update-mode context, adds an internal SDK subpath for the bundled Codex use case, and expands redaction/idempotency coverage.

## Best Possible Solution

Keep this PR open as the focused fix candidate for #73565. Review and land this PR, or a maintainer-owned equivalent, that routes all raw transcript append paths through the shared guard while preserving config-driven redaction, `before_message_write` blocking, idempotency, update-mode semantics, and the intended SDK boundary for the bundled Codex plugin.
