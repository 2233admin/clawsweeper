## Summary

Keep this PR open. Current main still has the unguarded session transcript append paths the PR targets, while the shared guard already contains the redaction/hook behavior needed for persisted transcript writes. The PR is an active implementation candidate for the open same-author security issue #73565, not obsolete cleanup.

## Best Possible Solution

Keep this PR open as the focused security fix candidate. Review and refine the guard-at-append approach, ensure every new guard call passes the right agentId/sessionKey/config semantics, preserve transcript update and idempotency behavior, choose an acceptable SDK boundary for the Codex mirror, then land this PR or a maintainer-owned equivalent to close #73565 while leaving broader redaction work under #64046 as needed.
