## Summary

Keep open. Current main still ignores `ANTHROPIC_BASE_URL` in both the Anthropic Messages streaming URL helper and the native Anthropic PDF request path, so the PR targets a real bug and is not obsolete. The branch needs a small follow-up before merge because its PDF change mishandles whitespace-only env values and `/v1`-suffixed env URLs, both already called out in review comments.

## What This Changes

The PR exports and tests `resolveAnthropicMessagesUrl`, adds `ANTHROPIC_BASE_URL` fallback for Anthropic Messages streaming, and adds the same env fallback to the native Anthropic PDF request path.

## Best Possible Solution

Keep the PR open and update it to use one shared Anthropic Messages URL resolver, or equivalent identical normalization, for both streaming and native PDF requests. The finished behavior should preserve explicit `baseUrl` precedence, honor trusted `ANTHROPIC_BASE_URL` only when non-empty, avoid duplicate `/v1`, add PDF-specific regression tests, and land with the normal changed-surface validation.
