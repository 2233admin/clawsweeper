## Summary

Keep #69374 open. Current main still drops WhatsApp forward attribution before plugin hooks: Baileys exposes `contextInfo.forwardingScore` and `contextInfo.isForwarded`, but the WhatsApp inbound message type, queue builder, finalized canonical hook context, and `inbound_claim` / `message_received` event metadata do not carry either field. This is not a ClawHub-only item because a plugin cannot recover data that the bundled WhatsApp adapter discards before hook delivery.

## Best Possible Solution

Keep the issue open and implement it as an additive WhatsApp-owned hook contract improvement: extract `contextInfo.forwardingScore` and `contextInfo.isForwarded` in the WhatsApp inbound normalization path, carry optional fields through `WebInboundMessage` and the finalized/canonical hook context, expose them on `inbound_claim` and `message_received` metadata or a maintainer-approved typed event field, and add focused extractor/monitor/mapper tests plus plugin hook docs if the public metadata contract changes.
