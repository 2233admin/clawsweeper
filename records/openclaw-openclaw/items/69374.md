## Summary

Current main still does not expose WhatsApp forwarding metadata to plugin inbound hooks. Baileys has `IContextInfo.forwardingScore` and `IContextInfo.isForwarded`, and OpenClaw's WhatsApp extractor can find `contextInfo`, but the normalized WhatsApp inbound payload, finalized context bridge, canonical hook context, and `inbound_claim` / `message_received` metadata do not carry those fields. The request is valid, narrow, and suitable for a focused fix PR.

## Best Possible Solution

Keep this issue open and implement a narrow additive bridge: extract `contextInfo.forwardingScore` and `contextInfo.isForwarded` from WhatsApp inbound messages, normalize `isForwarded` to true when Baileys says true or `forwardingScore >= 1`, carry both through `WebInboundMessage`, `FinalizedMsgContext`, `CanonicalInboundMessageHookContext`, and expose them in `inbound_claim` and `message_received` metadata. Add focused WhatsApp extraction/dispatch and hook-mapper tests, and document the channel-specific metadata keys for plugin authors.
