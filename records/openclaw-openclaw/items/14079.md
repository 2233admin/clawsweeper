## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the reserved core `system.notify` node command: the headless node host does not advertise or dispatch the command, docs and CLI still describe notification support as macOS/non-headless behavior, and the plugin registry prevents an external plugin from registering the same command surface.

## What This Changes

Add Linux desktop notification support for OpenClaw node hosts, primarily via `notify-send` with graceful handling for headless Linux sessions and missing notification backends.

## Best Possible Solution

Implement a focused core Linux headless node-host `system.notify` path with desktop-session detection, safe backend invocation, structured unavailable results for missing display/backend, docs and CLI updates, and tests proving advertisement plus invocation behavior without changing macOS, iOS, or Android notification behavior.

Implement Linux desktop notification support for the core `system.notify` node-host command tracked by https://github.com/openclaw/openclaw/issues/14079. Keep the fix focused on the headless node host: advertise `system.notify` on Linux only when appropriate, dispatch it safely, prefer `notify-send` using argv-only spawning, and return structured unavailable-style results for missing `DISPLAY`/`WAYLAND_DISPLAY` or missing backend.
