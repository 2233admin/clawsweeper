## Summary

Keep this issue open. Current main still does not implement Linux desktop notification delivery for the existing `system.notify` / `openclaw nodes notify` surface: Linux is allowlisted at the gateway policy layer, but the headless node host neither advertises nor handles `system.notify`, docs still describe headless node hosts as `system.run` / `system.which` only, and the plugin node-host API reserves `system.notify` for core rather than letting an external plugin fill the exact gap.

## Best Possible Solution

Implement a focused Linux headless node-host `system.notify` path in core: detect a usable desktop session (`DISPLAY` or `WAYLAND_DISPLAY`) plus a resolved `notify-send` backend, advertise `system.notify` only when delivery is available, spawn without a shell using sanitized arguments/environment and a timeout, return clear structured unavailable/invalid errors, keep macOS/iOS/Android behavior unchanged, and update CLI help, node docs, targeted tests, and changelog. If maintainers prefer plugin ownership later, first add a documented extension seam for reserved node-host commands.
