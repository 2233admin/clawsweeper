## Summary

Keep open. Current main still does not implement Linux desktop notification delivery for the existing `system.notify` node surface: Linux is allowed at gateway policy level, but the headless node host neither advertises nor handles `system.notify`, docs and CLI still describe the split as macOS-only/headless-run-only, and plugins cannot fill the exact gap because `system.notify` is reserved by core.

## What This Changes

Add Linux desktop notification support for node `system.notify`, using a Linux desktop notification backend such as `notify-send` with graceful headless degradation.

## Best Possible Solution

Implement a focused core Linux headless node-host `system.notify` path. Detect a desktop session and available backend before advertising the command, return structured unavailable/permission failures if delivery cannot proceed, keep `system.notify` core-owned, update CLI/docs/changelog, and cover the node-host advertisement and invocation behavior with targeted tests.
