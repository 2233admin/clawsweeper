## Summary

PR #39899 should remain open. Current main has partial surrogate mitigation for transport text payloads and filters failed assistant turns from replay, but it still lacks the PR's surrogate-specific error detector, friendly formatter branch, and runner-level reset path for provider JSON rejection. A plausible unsanitized replay path remains for persisted tool-call arguments, especially OpenAI Responses function-call argument strings or structured Anthropic tool_use input values containing lone surrogates.

## Best Possible Solution

Keep the PR open for maintainer review. Rebase it onto current main, add focused regression tests for surrogate-bearing provider error formatting, runner reset behavior, OpenAI Responses tool-call arguments, and Anthropic tool_use input replay, then land the smallest recovery fix or replace it with a broader sanitizer if the tests prove reset recovery is unnecessary.
