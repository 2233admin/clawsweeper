## Summary

Current main still does not automatically render a bare image URL typed into the Control UI chat as an inline image. The implemented paths cover explicit chat attachments, structured image blocks, MEDIA directives, data-image markdown, and managed assistant media, while ordinary user text remains text/link content under a CSP that blocks arbitrary remote image fetches.

## Best Possible Solution

Keep this open for an explicit Control UI/Gateway feature design: detect eligible bare image URLs in chat text, ingest or proxy them through guarded same-origin managed media instead of hotlinking, preserve text/link fallback, and add tests for composer/send, chat.send history projection, grouped rendering, markdown sanitization, managed media, and Control UI CSP behavior.

## Risks / Open Questions

- A direct regex-to-img implementation would conflict with the current Control UI CSP and privacy model by enabling arbitrary browser image fetches from chat text.
- A safe implementation likely needs Gateway-managed fetching or managed media storage with SSRF/public-network guards, MIME and size validation, cache/error states, and focused tests across send, history, rendering, markdown sanitization, managed media, and CSP behavior.
