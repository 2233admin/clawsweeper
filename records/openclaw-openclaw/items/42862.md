## Summary

Current main still does not implement automatic inline rendering for a bare image URL typed into the Control UI chat. The existing behavior supports uploaded attachments, structured image blocks, data-image markdown, and managed assistant media, while plain text image URLs remain text/link content under a CSP that intentionally blocks remote image hotlinking.

## Best Possible Solution

Keep this issue open for an explicit Control UI/Gateway feature design: detect eligible bare image URLs in chat text, route previews through guarded managed media or a same-origin proxy instead of hotlinking, preserve text fallback, and add tests across composer/send, chat.send history projection, grouped rendering, markdown sanitization, managed media, and Control UI CSP behavior.

## Risks / Open Questions

- A direct regex-to-img implementation would conflict with the current Control UI CSP and privacy model by allowing arbitrary remote browser image fetches from chat text.
- A safe implementation likely needs Gateway-managed fetching or managed media storage with SSRF/public-network guards, MIME and size validation, caching/error states, and focused send/history/rendering/CSP tests.
