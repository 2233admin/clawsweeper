## Summary

Keep open. Current main has adjacent safeguard summary-quality retries and compaction checkpoint snapshots, but it still does not implement this PR's intended pure post-compaction validator, guard-signal contract, validator reason codes, or `shouldRecommendReset` result. The PR is also a draft stacked on open same-author loop-aware compaction work, including #48238 and #48335, so the right cleanup path is review/rebase or explicit supersession rather than closing it as obsolete.

## Best Possible Solution

Keep this PR open as an implementation candidate in the loop-aware compaction stack. The best path is to resolve or supersede the earlier stack PRs, rebase onto current main, and review a focused validator that either integrates with the existing compaction checkpoint snapshots or clearly documents why validation remains diagnostic-only. Reset delivery should stay separate and require explicit maintainer product/security review.
