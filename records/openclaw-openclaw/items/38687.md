## Summary

Keep this PR open. Current main still does not implement the PR's intended target-aware media-runner behavior: the media runner creates an env-proxy fetch before it resolves the configured audio/video backend baseUrl, and the shared target-aware NO_PROXY helper is not wired into that injected multipart STT fetch path.

## Best Possible Solution

Keep this PR open and adapt it to current main: resolve the media provider execution context before choosing the injected fetch function, reuse or deliberately extend the shared `src/infra/net/proxy-env.ts` target-aware proxy semantics, make the CIDR/loopback bypass contract explicit from a security standpoint, and add focused media-runner tests for NO_PROXY-covered self-hosted audio backends plus private/internal backends that must remain proxied when not excluded.
