## Summary

Keep this PR open. Current main still does not implement the PR's target-aware media proxy behavior: the media runner resolves the env-proxy fetch before it resolves the configured audio/video backend baseUrl, and the env-proxy fetch helper still has no target URL parameter. The PR remains a plausible focused fix, but it needs maintainer review because it changes network egress/proxy bypass behavior.

## Best Possible Solution

Keep this PR open and adapt it to current main: resolve the media provider execution context before choosing the injected fetch function, reuse or deliberately extend the shared target-aware proxy semantics, document the loopback/CIDR NO_PROXY contract from a security standpoint, and add focused media-runner tests for NO_PROXY-covered self-hosted audio backends plus private/internal backends that must remain proxied when not excluded.
