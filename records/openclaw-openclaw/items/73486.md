## Summary

Keep this PR open. Current main still starts the optional model-pricing refresh during initial Gateway runtime setup, while this PR remains a focused implementation candidate that moves that work behind the post-attach/sidecars-ready path and addresses the deferred-sidecars review feedback.

## What This Changes

The PR defers Gateway model-pricing refresh out of initial runtime setup and starts it from scheduled-service activation after post-attach and sidecar readiness, with matching lifecycle tests.

## Best Possible Solution

Review and land or adapt this PR as the narrow startup-order fix: keep pricing enabled by default, preserve plugin lookup metadata, store the deferred stop handle on runtime state, and leave #73323 plus #75067 to track broader pricing/network resilience work.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: `startGatewayRuntimeServices()` starts pricing before the post-attach sidecars-ready transition, and `deferSidecars` can return before sidecars finish.

## Solution Assessment

Yes for this PR’s scope. Moving optional pricing refresh into the post-attach scheduled-services activation, gated on both post-attach return and sidecar readiness, is the narrow maintainable fix; timeout reduction, disk cache, and circuit breaking belong in the separate broader PR.
