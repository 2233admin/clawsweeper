## Summary

Keep this PR open. Current main still starts the optional model-pricing refresh in initial Gateway runtime setup, before HTTP bind, post-attach work, and sidecar/channel readiness. The PR remains a focused implementation candidate for the startup-order part of #73323, and the author has already pushed follow-up changes addressing the deferred-sidecars and early-stop coverage feedback.

## Best Possible Solution

Review and land or adapt this PR if maintainer review and CI pass. The best product direction is to keep pricing enabled by default, but start the optional pricing refresh only after post-attach work and sidecars/channels reach the ready path, store the returned stop handle on runtime state, preserve plugin lookup metadata, and leave #73323 open for the broader runtime/network investigation.
