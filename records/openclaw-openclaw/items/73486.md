## Summary

Keep this PR open. Current main still has the lifecycle behavior the PR targets: the model-pricing refresh is started from initial Gateway runtime setup, before the Gateway reaches the ready/post-attach scheduled-services path. The PR is a focused, active candidate for an open Windows startup degradation report, and I did not find an implemented-on-main, duplicate, ClawHub, or not-actionable close reason.

## Best Possible Solution

Review and land or adapt this PR if CI and maintainer review pass. The best product direction is to start optional model-pricing refresh only from the post-ready scheduled-services activation path, preserve plugin lookup metadata, store the stop handle on runtime state, and keep #73323 open for the broader Windows runtime/network investigation. Coordinate separately with #72306 if timer unref or in-flight abort-on-stop remains unaddressed.
