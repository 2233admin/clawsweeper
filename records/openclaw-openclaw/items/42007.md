## Summary

Keep open. Current main still does not implement Feishu `mentionTriggers`: the Feishu schema, reply policy, inbound gate, tests, and docs only support native @mention gating. The PR targets a real bundled Feishu plugin gap, but the current patch needs revision before merge because review comments correctly identify broadcast dispatch and empty-trigger problems, and the new public config field lacks tests/docs coverage.

## Best Possible Solution

Keep this PR open and request a Feishu plugin revision. A mergeable version should validate normalized non-empty trigger strings, resolve trigger lists consistently across top-level/account/wildcard-group/exact-group config, treat a keyword match as an effective mention for single-agent and broadcast dispatch, update `activeAgentId` and `WasMentioned` semantics, and add focused tests plus Feishu docs/config metadata updates.

## Risks / Open Questions

- Merging the PR as-is can silently fail in Feishu broadcast groups because keyword-only messages would pass the new early gate while `activeAgentId` remains null and no active Feishu reply dispatcher is selected.
- The proposed `mentionTriggers: z.array(z.string())` accepts empty strings; `ctx.content.includes("")` is always true, effectively bypassing `requireMention` for every group message if configured.
- The proposed public config field lacks tests, docs, and config metadata coverage, and the intended precedence across top-level, account, wildcard group, and exact group config is not defined in the patch.
