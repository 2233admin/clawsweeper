## Risks / Open Questions

- Merging the PR as-is would likely fail in broadcast-configured Feishu groups because keyword-triggered messages would pass the early gate but still leave `activeAgentId` null and `WasMentioned` false.
- The proposed `z.array(z.string())` allows empty strings; `ctx.content.includes("")` is always true, so an empty trigger would effectively bypass `requireMention` for every group message.
- The config contract is incomplete: top-level `mentionTriggers` overlaps with existing shared `mentionPatterns`, and the PR does not define account/group precedence or per-group triggers alongside existing per-group `requireMention`.
- The PR has no focused tests or Feishu docs/config metadata updates for the new behavior.
- Security review found no new supply-chain or CI execution surface in the diff, but the empty-trigger behavior is an access-control-like expansion of group reply scope and needs validation before merge.
