## Summary

Keep this PR open. Current main still does not implement Feishu text-based mention triggers, and the PR remains a focused Feishu plugin implementation candidate, but the provided patch is not merge-ready because the broadcast reply path, effective WasMentioned propagation, empty-trigger validation, config precedence, docs, and tests still need maintainer-quality follow-through.

## Best Possible Solution

Keep this PR open and request a Feishu plugin revision. A mergeable version should choose the final config contract, preferably reusing the existing shared mentionPatterns helpers unless maintainers explicitly want a Feishu-specific field; normalize and reject empty triggers; resolve top-level/account/group precedence alongside requireMention; treat keyword matches as effective mentions for single-agent and broadcast dispatch; update activeAgentId and WasMentioned; and add focused schema, policy, bot, broadcast, and Feishu docs coverage.

## Risks / Open Questions

- Merging the PR as-is could let keyword-only messages pass the early gate but still fail to create a real reply dispatcher in Feishu broadcast groups because activeAgentId remains tied to ctx.mentionedBot.
- The proposed z.array(z.string()) plus String.includes trigger check accepts empty strings; an empty trigger would match every group message and bypass requireMention.
- The config contract is unsettled because Feishu-specific mentionTriggers overlaps with the existing shared messages.groupChat.mentionPatterns / agents.list[].groupChat.mentionPatterns model used by other channels.
- No current tests or docs cover Feishu keyword triggers, group/account precedence, empty-trigger rejection, or broadcast effective-mention propagation.
