## Summary

Keep open. Current main still documents and implements agent workspaces as default working directories with opt-in filesystem guards and optional sandboxing, not as a hard default per-agent confinement boundary across filesystem, exec/process, plugin, MCP, and media surfaces.

## What This Changes

Requests hard per-agent workspace confinement so a PM/Feishu agent and installed skills cannot read or modify files outside that agent's own workspace.

## Best Possible Solution

Define a maintainer-approved per-agent confinement profile or onboarding option that can be applied to agents like the PM/Feishu bot: enable workspace-scoped filesystem tools, deny or sandbox runtime tools, cover plugin/MCP/media path surfaces, surface the active workspace to skills, and add regression coverage for cross-workspace reads/writes while preserving explicit trusted-host workflows.
