## Summary

Keep open. The report is fresh and credible, current main still relies on instruction-based and bounded identifier/file-operation preservation rather than deterministic exact path-literal carry-forward, and PR #75093 is already the direct open fix candidate with closing syntax.

## What This Changes

Preserve slash-separated file paths exactly across context compaction so later tool calls do not receive corrupted paths with spaces inserted after slashes.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard change, so path-like literals needed for future tool calls are collected from model-facing compaction input and appended deterministically without exposing hidden tool-result details.
