## Summary

Keep open. The report is fresh and credible, current main still lacks a deterministic exact path-literal preservation layer for compaction, and open PR #75093 is already the direct fix candidate with closing syntax.

## What This Changes

The issue reports that context compaction can rewrite slash-separated file paths, causing later read tool calls to use paths with spaces and fail.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard change, so path-like literals needed for future tool calls are collected from model-facing compaction input and appended deterministically with regression coverage proving slash-separated paths survive unchanged.

## Reproduction Assessment

No. There is not a high-confidence end-to-end reproduction from the checkout alone because it needs the original long transcript or a deterministic MiniMax compaction replay, though the attached tool logs and source review make the failure credible.

## Solution Assessment

Yes. Deterministic path-literal preservation in the compaction/safeguard pipeline is narrower and more reliable than relying on provider prompt adherence alone; the linked PR is the right implementation path to review.
