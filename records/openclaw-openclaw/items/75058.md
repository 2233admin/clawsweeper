## Summary

Keep open. The report is fresh and credible, current main still does not deterministically carry every path-like literal through compaction, and open PR #75093 is already the direct closing fix candidate, so this issue should wait for that PR to be reviewed, landed, or closed.

## What This Changes

The issue reports that context compaction can corrupt slash-separated file paths by inserting spaces after `/`, causing later tool calls to use invalid paths.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction/safeguard hardening change, so path-like literals needed for future tool calls are extracted from model-facing compaction inputs and appended deterministically with regression coverage for unchanged slash-separated paths.

## Reproduction Assessment

No. The provided raw tool-parameter logs are credible and current source review shows a plausible preservation gap, but a high-confidence end-to-end reproduction would need the original long transcript or a deterministic MiniMax compaction replay.

## Solution Assessment

Yes. A narrow deterministic path-literal preservation layer with targeted regression tests is the maintainable direction; more prompt wording alone would still leave correctness dependent on provider/model summarization behavior.
