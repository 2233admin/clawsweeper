## Summary

Keep this issue open. Current main still does not inject a sibling-agent roster into the session system prompt or expose a requester-specific collaboration roster through `agents_list`; the existing surfaces remain split between spawn allowlists, Gateway control-plane agent rows, and separate A2A session-policy enforcement. The KeepMyClaw comment is useful context for stale external state, but there is no corresponding in-repo live roster surface yet.

## Best Possible Solution

Keep this open and implement a focused model-visible roster surface. The conservative path is to extend `agents_list` with additive identity and requester-specific A2A fields, or add a dedicated live roster tool, then update prompt guidance to direct agents to that authoritative tool rather than injecting a static full roster into every prompt.
