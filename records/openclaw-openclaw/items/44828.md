## Summary

Keep this PR open. Current main still lacks a Feishu-local API-boundary silent-token guard in the outbound adapter and reply dispatcher, while the submitted patch is incomplete and not safe to land as-is. Related open PR #56167 covers part of the same Feishu NO_REPLY surface, so maintainers should consolidate the attempts instead of closing this as implemented.

## Best Possible Solution

Keep this PR open but do not land it as-is. The best path is a refreshed Feishu patch against current main that suppresses exact silent tokens at the Feishu API boundary for sendText, sendMedia captions/fallbacks, and reply-dispatcher delivery where needed; preserves media delivery; closes streaming state correctly; uses a narrow SDK/runtime surface; adds focused Feishu regression tests; and reconciles the overlapping work in #56167.
