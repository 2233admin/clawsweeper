## Summary

Keep this PR open. Current main partially covers the legacy cron-id repair through the doctor migration path, but it does not implement the PR's health snapshot budget or per-channel account parallelization; `getHealthSnapshot` still probes accounts sequentially and has no `maxSnapshotMs`/deadline behavior. The PR is not obsolete, but it needs rebasing and cleanup before it can be considered merge-ready.

## Best Possible Solution

Keep this PR open or replace it with a rebased, focused patch: implement bounded/parallel health snapshots in current `src/commands/health.ts`, preserve the existing doctor-based cron ID repair unless a runtime migration is explicitly wanted, add tests for budget exhaustion and parallel account probes, and fix the bot review/changelog attribution before merge.
