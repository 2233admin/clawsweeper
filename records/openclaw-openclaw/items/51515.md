## Summary

Keep this PR open. Current main still lacks the PR's central bounded, parallel live health-snapshot behavior, and the cron-id repair that exists today is doctor-managed rather than the runtime load-time normalization proposed by this PR.

## Best Possible Solution

Keep this PR open or replace it with a rebased focused patch. The best path is to implement bounded parallel live health snapshots in current `src/commands/health.ts` and the gateway refresh/post-connect paths, add regression coverage for concurrent account probes and deadline exhaustion, and make an explicit maintainer decision on whether malformed cron ids should remain doctor-only repair or also be synthesized/coerced during runtime load.
