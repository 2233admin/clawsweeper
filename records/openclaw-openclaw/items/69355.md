## Summary

Keep this PR open. Current main still does not implement a warn-only `openclaw doctor` check that compares local `dist/build-info.json` metadata with npm publish times, and the PR is the active unmerged implementation proposal. It is useful, but it needs maintainer review and a refresh for the current lazy doctor flow plus the existing review findings around bounded registry fetches, stale cache behavior, metadata parity, malformed metadata reasons, and changelog attribution.

## What This Changes

This PR adds a warn-only `openclaw doctor` rebuild guard that reads local build metadata, fetches and caches npm registry publish times, warns for rebuilt/unreleased/corrupt installs, wires the check into doctor, and adds tests, docs, and a changelog entry.

## Best Possible Solution

Keep this PR open and request an update. The best path is to adapt the guard to the current lazy doctor flow, use a bounded npm registry fetch, keep stale-cache behavior conservative, distinguish missing from malformed metadata, compare package and build-info versions, preserve focused tests/docs, and fix changelog attribution before maintainer review and merge.
