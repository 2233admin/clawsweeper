## Summary

Keep #69355 open. Current main still does not implement a doctor check that compares local dist/build-info.json metadata to npm publish time, and the provided PR is a plausible core doctor/install-posture feature rather than ClawHub or external administration work. It is not covered by the title-overlap related items, but the PR should address the existing review comments before merge.

Keep the PR open as the implementation track. The best path is a core doctor install-posture check that reuses dist/build-info.json, compares against npm registry publish time with a bounded timeout and TTL-respecting cache, stays warn-only/silent when inconclusive, flags malformed and mismatched local metadata as corrupt, documents the behavior on the public doctor page, and lands with focused tests plus a compliant changelog entry.
