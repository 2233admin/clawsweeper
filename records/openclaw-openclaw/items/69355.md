## Risks / Open Questions

- The PR adds a normal `openclaw doctor` network fetch to the npm registry without a timeout, so slow or blackholed outbound HTTPS can stall an otherwise non-essential diagnostic path.
- The proposed fallback reads stale cache after rejecting stale cache earlier, which can produce false `unreleased` warnings when current registry data is unavailable.
- The proposed local metadata reader does not compare `package.json.version` with `dist/build-info.json.version`, so a partial overlay can evaluate the wrong version instead of reporting corrupt metadata.
- Malformed `build-info.json` is reported as missing metadata, making the doctor output less actionable.
- The proposed changelog entry still contains `#TBD` and lacks the required `Thanks @...` attribution.
