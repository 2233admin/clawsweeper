## Summary

Keep this issue open. Current main still does not expose WhatsApp Click-to-WhatsApp ad metadata: the WhatsApp plugin can read `ContextInfo`, but the inbound extraction/enqueue path only carries text, location, contact cards, reply context, media, mentions, sender/session fields, and existing untrusted structured context. It does not extract Baileys `contextInfo.externalAdReply.ctwaClid` or source fields into the inbound message, agent context, plugin hook context, or persisted session metadata. No related implementation PR was found, and the reporter recently bumped the issue, so stale cleanup is not appropriate.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin. Extract `contextInfo.externalAdReply` from the first inbound Baileys message, normalize a minimal untrusted CTWA payload containing `ctwaClid`, `sourceUrl`, `sourceType`, `sourceId`, and title/headline-like fields, carry it through `WebInboundMessage` into `buildWhatsAppInboundContext`, and add focused tests for extraction plus prompt/context visibility. If durable retrieval is required, add a narrowly scoped session metadata field or documented plugin-access path rather than overloading generic origin metadata.
