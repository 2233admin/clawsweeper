## Summary

Keep #39037 open. Current main still does not extract or propagate Click-to-WhatsApp attribution data from WhatsApp/Baileys inbound messages. The existing WhatsApp contextInfo path is used for mentions and quoted replies, and the only structured context currently queued into the prompt is contact-card metadata, so ctwaClid/referral data would still be dropped.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin. Add a focused extractor for Baileys/WhatsApp CTWA referral data, normalize ctwaClid/ctwa_clid plus sourceUrl, sourceType, sourceId, headline/title, and related ad metadata into untrusted structured context, and decide whether durable conversion attribution also needs explicit hook/session metadata. Cover extraction, WebInboundMessage queueing, prompt rendering, and message_received hook/session behavior with targeted tests.
