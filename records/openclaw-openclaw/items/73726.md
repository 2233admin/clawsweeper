## What This Changes

The issue asks for `deleteMessageTelegram` to fail soft on benign Telegram Bot API delete no-ops and surface a structured warning through the Telegram action runtime.

Implement a plugin-local Telegram fix: classify only the known benign delete no-op messages, suppress shared ERROR logging for those cases via the existing `shouldLog` seam, return a warning union from `deleteMessageTelegram`, surface `deleted: false` in the action runtime, add focused regression tests, and add a user-facing changelog entry credited to the reporter.

## Repair Work Prompt

Fix the Telegram delete-message fail-soft bug from https://github.com/openclaw/openclaw/issues/73726. Current main still lets benign Telegram Bot API delete errors such as `message to delete not found`, `message can't be deleted`, `MESSAGE_ID_INVALID`, and `MESSAGE_DELETE_FORBIDDEN` propagate through `deleteMessageTelegram`, causing shared ERROR logging and a hard tool/action failure. Keep the fix inside the Telegram plugin boundary. Update `extensions/telegram/src/send.ts` so `deleteMessageTelegram` uses the existing local patterns from `reactMessageTelegram` and `editMessageReplyMarkupTelegram`: match only the known benign delete no-op strings, suppress shared API error logging for those cases via the `shouldLog` seam, return `{ ok: false, warning }`, and keep auth, network, malformed target, and other non-benign errors throwing. Update `extensions/telegram/src/action-runtime.ts` so the `deleteMessage` action surfaces a soft result as `deleted: false` with the warning instead of unconditionally reporting `deleted: true`. Add focused regression tests in the Telegram send/action-runtime/proxy coverage as appropriate, including at least the reported benign strings and a non-benign error that still throws. Add a user-facing `CHANGELOG.md` fix entry credited to `@Avicennasis`. Do not change core delivery-recovery policy, Telegram credentials/auth resolution, retry semantics beyond the delete-specific benign handling, or unrelated Telegram delete paths.
