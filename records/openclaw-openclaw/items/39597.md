## Summary

Current main still has the reported product gap. Gateway clients already send structured metadata that can distinguish Control UI/browser, WebChat, and native macOS/iOS/Android surfaces, and the server stores it on the connection. The agent runtime prompt still only exposes the channel/capability runtime fields, while chat.send maps only client id/displayName into sender metadata rather than a trusted client context agents can branch on.

## Best Possible Solution

Keep this issue open and implement a narrow trusted Gateway client-context path in core: sanitize selected fields from connect.client, carry them through chat.send and queued/follow-up run params into RuntimeInfoInput, render deterministic Runtime fields such as client=openclaw-macos, client_mode=ui, and client_platform=macos/web, and document the privacy-safe contract for WebChat/Control UI clients.
