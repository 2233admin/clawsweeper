## Summary

Keep open. This PR has the protected `maintainer` label, and current `main` still does not implement the intended Copilot picker/catalog behavior: Copilot runtime catalog discovery returns an empty model list, the picker does not use static manifest catalog rows for the preferred provider, and manifest auth choices do not carry `modelAllowlist` metadata.

## What This Changes

The PR reuses existing GitHub Copilot auth profiles, stores Copilot login credentials in the selected agent directory, publishes a static GitHub Copilot model catalog in the plugin manifest, and makes configure/model picker prefer manifest-owned provider catalogs before falling back to runtime catalog loading.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review and refine the focused implementation, especially the static Copilot catalog contents, manifest `modelAllowlist` contract, preferred-provider picker shortcut, and agent-scoped auth-profile reuse, then land it or request changes with targeted tests for those surfaces.
