## Summary

Keep this PR open. Current main still has the cache invalidation gap: the fingerprint helper rebuilds dependency paths from the repo root by name, while staging can copy a plugin-local resolved `realRoot`, and the reuse path skips restaging when the stale stamp matches. The PR aligns the fingerprint with the selected dependency records and adds focused plugin-local regression coverage.

## What This Changes

This PR changes the installed runtime closure fingerprint to hash the selected dependency records by `realRoot` and adds a regression test for plugin-local installed runtime dependency content changes.

## Best Possible Solution

Land a narrow patch that fingerprints the same resolved runtime dependency roots that staging copies, with regression coverage for plugin-local installed dependency contents, then run the targeted staging test and changed-file gate.
