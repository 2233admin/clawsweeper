## Summary

Keep this issue open. Current main still routes ACP parent relay progress, stall, and resumed notices into the parent session as system events, and open PR #55896 is an unmerged implementation candidate that directly targets this bug. Per the cleanup policy, the issue should remain open until that PR or an equivalent fix is merged or explicitly closed.

## Best Possible Solution

Review and either land PR #55896 or replace it with an equivalent narrow fix in the ACP parent relay/spawn wiring. The fix should suppress intermediate relay chatter for visible Discord thread parent sessions, preserve start/done/error/timeout notices and non-Discord behavior, and add targeted regression coverage for the relay and spawn seams.
