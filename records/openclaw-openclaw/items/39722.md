## Summary

Keep this issue open. Current main still has the reported TUI/session-title path: the TUI prefers derived transcript titles over displayName, title derivation still only sees the inner session entry, transcript title extraction still does not strip inbound metadata, and the provided context shows #39790 is still an open implementation candidate that explicitly targets this issue.

## Best Possible Solution

Keep this issue open until #39790 or an equivalent maintainer-reviewed fix lands. The fix should pass the computed outer displayName into title derivation, strip inbound metadata before first-user-message and last-message-preview extraction where appropriate, preserve existing subject/message/session-id fallbacks, and add regression coverage for external displayName precedence plus metadata stripping.
