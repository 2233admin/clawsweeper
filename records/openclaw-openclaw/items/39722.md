## Summary

Keep this issue open. Current main still has the reported TUI/session title behavior: `/sessions` prefers `derivedTitle` over `displayName`, `deriveSessionTitle` only receives the inner session entry, and transcript title extraction still does not strip the gateway-injected inbound metadata block. The provided GitHub context also shows #39790 is an open, unmerged implementation candidate that explicitly targets this issue.

## Best Possible Solution

Keep this issue open until #39790 or an equivalent maintainer-reviewed fix lands. The fix should pass the computed outer displayName into title derivation, strip inbound metadata before first-user-message and last-message-preview extraction where appropriate, preserve existing subject/message/session-id fallbacks, and add focused regression coverage for external displayName precedence plus metadata stripping.
