## Summary

Keep open. Current main still has the two audit-boundary behaviors reported here: the CLI audit path leaves plugin security collector loading enabled, collector discovery reaches the metadata snapshot path without disabling bundled runtime-deps staging, and both installed-plugin audit scanners accept hidden backup directories. The older related close for #73605 covered security command startup, and #75429 covers a different repeated runtime-deps restaging loop; neither resolves this narrower audit collector/backup-directory scan issue.

## What This Changes

Report that `openclaw security audit --json` can hang because read-only audit collector discovery stages bundled plugin runtime dependencies and installed-plugin scans treat hidden backup directories as plugins.

## Best Possible Solution

Keep this open and make the security audit path genuinely read-only: collector discovery should avoid bundled runtime-deps staging while still running intended collectors, and installed-plugin audit scans should ignore hidden, backup, and non-plugin directories under the extensions state directory.

## Reproduction Assessment

Yes for the core issue. The current source path shows `security audit --json` can reach collector discovery with runtime-deps staging enabled, and the installed-plugin audit helpers still list hidden backup directories as plugin candidates; the full high-CPU/orphan-process symptom was not live-run in this read-only review.

## Solution Assessment

Yes. A narrow audit-scoped fix is the best path: pass or add an `installBundledRuntimeDeps: false` style option for collector discovery, add hidden/backup-dir filtering to the audit scanners, and cover both with targeted tests rather than waiting for the broader public runtime-deps opt-out tracked elsewhere.

## Repair Work Prompt

Fix openclaw/openclaw#75456. Make `openclaw security audit --json` remain bounded/read-only when discovering plugin security audit collectors, and stop treating hidden/backup extension directories as installed plugins during audit scans. In the collector discovery path, add or thread an explicit runtime-deps opt-out such as `installBundledRuntimeDeps: false` through `loadPluginMetadataRegistrySnapshot`/`buildPluginRuntimeLoadOptions`, then have `collectPluginSecurityAuditFindings` use it while preserving intended `securityAuditCollectors` execution. Do not broadly disable plugin security collectors unless tests show that is the intended product behavior. In `src/security/audit-extra.async.ts` and `src/security/audit-plugins-trust.ts`, filter installed plugin directory candidates to skip hidden names, `node_modules`, and known install-backup directories such as `.openclaw-install-backups`; keep legitimate plugin directories scanable. Add focused regression tests proving the audit metadata snapshot call disables bundled runtime-deps staging and proving hidden backup dirs are skipped by both code-safety/trust scan paths. Add a single-line `CHANGELOG.md` fix entry crediting `Thanks @liushuaiiu`. Do not change dependency versions, package-manager behavior, public runtime-deps config policy, or unrelated plugin activation rules.
