## Summary

Keep this issue open. Current main still lacks activity-based heartbeat idle timeout behavior: the config/types/schema/docs expose interval, active-hours, routing, timeout, light-context, isolation, and reasoning controls, while the runner remains interval/wake driven and does not gate heartbeats on recent allowed inbound user activity. The provided related context also keeps this as the canonical tracker, with #59567 closed as a duplicate.

## Best Possible Solution

Keep this issue open as the canonical heartbeat idle-timeout tracker. A proper fix should add a maintainer-approved heartbeat activity/idle config key to exported types, strict schema, schema labels/help/generated docs, config-honor inventory, and https://docs.openclaw.ai/gateway/heartbeat. Runtime should gate heartbeat scheduling or execution on recent allowed inbound user/channel activity at the correct agent/session/channel boundary, likely using lastInteractionAt or a dedicated activity ledger, while preserving existing interval, active-hours, wake, requests-in-flight retry, lightContext, isolatedSession, delivery, and per-agent merge behavior.
