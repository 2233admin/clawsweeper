## Summary

Keep this issue open. Current main still has periodic/wake-driven heartbeat behavior, strict heartbeat config has no activity idle-timeout key, and the docs/tests describe session idle freshness without implementing a heartbeat pause-until-recent-allowed-activity mode. Related #59567 is closed as a duplicate, so this remains the canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical heartbeat activity idle-timeout tracker. A maintainer-approved fix should add a heartbeat activity/idle config key to exported types, strict schema, config honor inventory, generated/help docs, and https://docs.openclaw.ai/gateway/heartbeat, then gate heartbeat scheduling or execution on recent allowed inbound user activity at the correct agent/session/channel boundary while preserving existing interval, activeHours, wake, requests-in-flight retry, lightContext, isolatedSession, delivery, and per-agent merge behavior.
