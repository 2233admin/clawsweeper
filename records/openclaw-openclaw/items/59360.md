## Summary

Keep open. Current main still emits the Feishu card note in the reported colon-delimited form and renders it as Feishu card markdown, with no Feishu config option to suppress or reformat that note. Related #60748 and #66759 track broader footer customization, but they do not supersede this narrower Markdown parsing bug.

## What This Changes

The issue requests avoiding or disabling Feishu card note footer text that Feishu Markdown may parse as definition-list syntax.

## Best Possible Solution

Keep the fix in the Feishu plugin and change the generated card note to a Feishu-safe format that avoids `key: value` definition-list parsing, with focused regression coverage for streaming start, streaming close, and static structured-card sends. Broader footer customization should remain with #60748/#66759 unless maintainers choose to expand scope.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available by exercising `createFeishuReplyDispatcher` in card/streaming mode: current tests already assert that the generated card note is `Agent: agent`, and the send layer renders that note as Feishu markdown.

## Solution Assessment

Yes, a narrow Feishu-owned formatting fix is the best first step. Adding a broad footer configuration API would overlap #60748/#66759 and is not necessary to solve the reported Markdown parsing bug.
