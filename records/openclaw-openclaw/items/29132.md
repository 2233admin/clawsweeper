## Summary

Keep open. Current main still does not expose a public `tool_result_before_model`, `transform_tool_result`, or `before_context_send` hook for external plugins. The closest implementation is the bundled-only agent tool-result middleware, which runs at the requested post-execution/pre-model timing, but both source and docs intentionally block non-bundled plugins from registering it. The Agent14 comment is useful production validation of the need, and open related work (#64907, #62123, #72733) should be reviewed rather than using cleanup to close this issue now.

## What This Changes

Add a synchronous public plugin hook that can transform completed tool results before same-turn model visibility and before transcript persistence.

## Best Possible Solution

Keep this issue open until maintainers either land a documented, trust-gated public post-execution/pre-model tool-result mutation hook or explicitly decide that same-turn model-visible result rewriting must remain bundled-only. The design should coordinate with #64907, #62123, and #72733 and specify ordering relative to `before_tool_call`, internal middleware, `after_tool_call`, `tool_result_persist`, and `before_message_write`, plus failure/timeout behavior, SDK types, docs, and Pi/Codex tests.
