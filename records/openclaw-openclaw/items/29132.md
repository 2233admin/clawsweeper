## Summary

Keep open. Current main still does not expose `tool_result_before_model`, `transform_tool_result`, `before_context_send`, or an equivalent public post-execution/pre-model tool-result mutation hook. There is an internal bundled-plugin middleware path that can rewrite tool results before the model sees them, but docs and registry checks keep that surface bundled-only, so the external plugin API gap described here remains.

## Best Possible Solution

Keep this issue open until maintainers either design and land a documented, trust-gated public post-execution/pre-model tool-result mutation hook, or explicitly decide that same-turn model-visible tool-result rewriting must remain bundled-only. A complete solution should define ordering, sync/async behavior, failure behavior, SDK types/docs, and tests for both Pi and Codex model-visible tool output.
