## Summary

Keep open. Current main still has the generic compat-driven schema cleanup, but the affected Volcengine/Ark/Kimi catalog rows do not declare the compat metadata that would trigger stripping of unsupported validation keywords. The direct fix PRs are closed unmerged, and the open Kimi compaction PR tracks a different failure mode.

## What This Changes

The issue asks for Volcengine/Ark-hosted Kimi tool schemas to avoid unsupported JSON Schema validation keywords such as `minLength`, `maxLength`, and `minItems`.

## Best Possible Solution

Add provider-owned `compat.unsupportedToolSchemaKeywords` metadata to the affected Volcengine/Ark/Kimi catalog rows, and include BytePlus sibling rows only if the same backend keyword rejection applies, while leaving the core normalization path generic and compat-driven.
