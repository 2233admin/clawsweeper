## Summary

Keep this issue open. Current main still strips unsupported tool-schema keywords only when resolved model compat declares them, and the bundled Volcengine/Ark/Kimi catalog/provider does not currently declare `unsupportedToolSchemaKeywords` or a provider-owned tool-schema hook. The two linked fix PRs are closed and unmerged, so the user-visible Volcengine/Kimi tool-call failure still needs a narrow provider-owned fix.

## Best Possible Solution

Implement provider-owned compatibility for bundled Volcengine/Ark/Kimi models. Prefer adding `compat.unsupportedToolSchemaKeywords` to the relevant Volcengine manifest catalog rows or contributing resolved-model compat from the Volcengine provider; use a provider `normalizeToolSchemas` hook only if manifest/model compat is not sufficient. Add focused regression coverage for Volcengine standard, Volcengine plan/Ark, and Kimi rows, plus a negative case proving unrelated providers keep validation keywords.
