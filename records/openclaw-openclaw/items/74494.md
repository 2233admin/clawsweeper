## Summary

Keep this PR open. The mobile chat settings problem is still partly real on current main, but the proposed patch introduces blocking regressions: expanded tool cards are capped globally instead of only on mobile, and the viewport change disables page zoom across the Control UI.

## What This Changes

This PR changes the Control UI mobile chat menu into a full-screen settings overlay with session/model/thinking selects, refresh/thinking/tool/focus/cron buttons, safe-area and keyboard viewport handling, expanded tool-card overflow containment, and updated browser render coverage.

## Best Possible Solution

Keep the mobile overlay and parity direction, but revise the patch so mobile-specific overflow containment does not change desktop expanded tool cards, remove the global page-zoom cap, localize new visible Control UI labels, and then run focused Control UI browser tests plus mobile visual validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve desktop expanded tool cards:** `ui/src/styles/chat/tool-cards.css:20-22`
  - body: This selector is global, so every expanded tool card now gets capped to `min(40vh, 400px)` and forced into nested scrolling on desktop. Current main intentionally lets expanded cards grow with `max-height: none; overflow: visible`, and the PR states desktop layout is out of scope, so the height cap should be moved under the mobile breakpoint or otherwise scoped to the mobile overflow case.
  - confidence: 0.9
- **[P2] Do not cap mobile page zoom:** `ui/index.html:4`
  - body: Adding `maximum-scale=1.0` applies to the entire Control UI, not just the settings dropdown, and can prevent users from pinch-zooming the app in mobile browsers or webviews. That is an accessibility regression; keep the viewport-fit/keyboard behavior if needed, but do not restrict user zoom.
  - confidence: 0.82
- **[P3] Localize the new settings copy:** `ui/src/ui/app-render.helpers.ts:496-497`
  - body: The new mobile overlay adds visible English-only strings such as `Chat Settings` and the select labels while the Control UI supports browser locale selection. Please route the new copy through the existing i18n keys/locales so non-English users do not get mixed-language chat controls.
  - confidence: 0.74
