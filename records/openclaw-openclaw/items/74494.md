## Summary

Keep this PR open for revision. Current main has already merged the Lit-owned mobile dropdown state work and the separate mobile cron control, but it has not implemented the full overlay/model/refresh/tool-card overflow changes; the PR branch still introduces blocking UI/accessibility regressions that should be fixed before maintainer review can proceed.

## What This Changes

The PR changes the Control UI mobile chat menu into a full-viewport settings overlay with session/model/thinking selectors, refresh/thinking/tool/focus/cron buttons, safe-area and keyboard viewport handling, expanded tool-card overflow containment, and updated browser render coverage.

## Best Possible Solution

Preserve the useful mobile parity direction, but rebase onto current main and revise the patch so it uses setChatMobileControlsOpen for all open/close paths, scopes tool-card overflow containment to the mobile case, keeps user zoom available, routes new visible strings through Control UI i18n, and then validates the focused browser tests plus real mobile layout behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Close the dropdown through Lit state:** `ui/src/ui/app-render.helpers.ts:515`
  - body: The close button removes the `open` class directly, but current main owns this disclosure through `chatMobileControlsOpen` and `setChatMobileControlsOpen`. After the next render, the stale true state can restore the class and `aria-expanded`, so the X close path is unreliable.
  - confidence: 0.9
- **[P2] Scope expanded-card scroll limits to mobile:** `ui/src/styles/chat/tool-cards.css:20-22`
  - body: This selector is global, so every expanded tool card now gets capped to `min(40vh, 400px)` and forced into nested scrolling on desktop. Current main leaves expanded cards uncapped, and the PR says desktop layout is out of scope, so this cap should be moved under the mobile breakpoint or otherwise scoped to the mobile overflow case.
  - confidence: 0.9
- **[P2] Keep mobile zoom available:** `ui/index.html:4`
  - body: Adding `maximum-scale=1.0` applies to the entire Control UI, not just the settings dropdown, and can prevent users from pinch-zooming the app in mobile browsers or webviews. Keep the viewport-fit or keyboard behavior if needed, but do not restrict user zoom.
  - confidence: 0.84
- **[P3] Localize the new settings copy:** `ui/src/ui/app-render.helpers.ts:501-507`
  - body: The overlay adds visible English-only copy such as `Chat Settings`, `Close settings`, and select labels while the Control UI supports browser locale selection. Please add the new strings through the existing i18n keys/locales instead of hardcoding them in markup/data-labels.
  - confidence: 0.78
