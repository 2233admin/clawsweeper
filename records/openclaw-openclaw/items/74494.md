## Summary

Keep this PR open for revision. Current main still lacks several requested mobile chat-menu improvements, but the branch is stale against recent Control UI work and the proposed patch has blocking regressions around refresh state, dropdown state ownership, desktop tool-card expansion, and mobile zoom accessibility.

## What This Changes

The PR changes the Control UI mobile chat settings menu into a full-viewport overlay, adds mobile refresh/model/select controls and safe-area CSS, constrains tool-card overflow, and updates a browser render test.

## Best Possible Solution

Revise the PR in place: preserve the mobile overlay direction, but rebase over the Lit-state dropdown and refresh busy-state commits, share or mirror the desktop refresh guard, scope tool-card clipping to the mobile overflow case, remove the global zoom cap, route new visible copy through Control UI i18n, and then validate with focused browser tests plus a mobile visual smoke.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the structural dropdown gaps by inspecting current main's mobile render path and browser render tests; Safari clipping, native picker, and tool-card overflow still need visual/device smoke to confirm after the patch is revised.

## Solution Assessment

Is this the best way to solve the issue? No. The product direction is plausible, but the current patch is not the narrowest maintainable fix because it duplicates stale state/refresh logic, changes desktop tool-card behavior, and applies global viewport zoom restrictions.
