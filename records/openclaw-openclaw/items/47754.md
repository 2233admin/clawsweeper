## Summary

Keep this PR open. Current main still builds keyword search snippets by selecting the full FTS row text and truncating `row.text` from the beginning, so the intended match-centered `snippet()` behavior has not landed. The PR is directionally useful, but it now targets the old `src/memory/manager-search.ts` path and needs a rebase onto `extensions/memory-core/src/memory/manager-search.ts` plus coverage for deep matches and the current trigram substring fallback path.

## Best Possible Solution

Keep this PR open as a valid implementation candidate, but update it for the current memory-core plugin path. The best fix is to use SQLite FTS5 `snippet()` for MATCH-backed keyword results, explicitly clamp tokens to 1..64 with visible ellipsis markers, preserve the substring-only trigram fallback, and add a focused regression test where the keyword appears deep in a large chunk. Because this is user-visible memory search behavior, the eventual merge should include an active changelog fix entry crediting the contributor.
