## Summary

Keep this PR open, but it needs changes before merge. The reported memory-search snippet bug is still valid on current main, and the PR's approach is directionally useful, but the patch targets the old `src/memory/manager-search.ts` path while current memory search lives in `extensions/memory-core/src/memory/manager-search.ts`.

## What This Changes

The PR changes legacy memory keyword search to return SQLite FTS5 `snippet()` output with clamped token count and visible ellipsis markers instead of selecting full text and truncating from the beginning.

## Best Possible Solution

Port the FTS5 snippet behavior into the active memory-core keyword MATCH branch, keep LIKE/trigram fallback behavior intact, avoid resurrecting legacy `src/memory/*` search files, and add a regression test where the matched term appears beyond the old 700-character prefix window.

## Reproduction Assessment

Yes. The linked bug gives a concrete CLI path, and current-main source plus an in-memory FTS5 check show that a keyword match past the prefix window is found but omitted from the returned snippet.

## Solution Assessment

No. Using SQLite FTS5 snippets is the right narrow direction, but this PR applies it to the wrong path for current main; the maintainable fix belongs in `extensions/memory-core/src/memory/manager-search.ts` with targeted tests.
