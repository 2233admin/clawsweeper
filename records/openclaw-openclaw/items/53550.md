## Summary

Keep open. Current main still leaves the reported gateway-to-DM recall path unresolved because builtin session memory, QMD transcript export, and session-hit visibility are separate controls, and the issue discussion has not been superseded by a merged fix or narrower canonical issue.

## What This Changes

The issue requests that `experimental.sessionMemory` with `sources: ["memory", "sessions"]` make `memory_search` surface gateway-dispatched session transcript content across sessions, including QMD/Gemini-backed setups.

## Best Possible Solution

Define the intended contract explicitly: either make QMD honor `experimental.sessionMemory` plus `sources: ["sessions"]` by exporting/indexing session transcript collections with clear logging, or keep QMD opt-in separate but surface an actionable diagnostic/docs path requiring `memory.qmd.sessions.enabled: true` and a broader `tools.sessions.visibility` such as `agent` for same-agent gateway-to-DM recall. Add a regression test for a gateway-style same-agent session queried from an unrelated DM key.
