## Summary

Keep open. Current main still flattens remote Control UI markdown image URLs to alt text, and that behavior is now backed by tests, CSP, docs, and changelog entries describing intentional no-remote-fetch hardening. Open PR #41025 directly targets this issue with closing syntax, so the next step is maintainer product/security review rather than cleanup closure.

## What This Changes

The issue requests that Control UI chat markdown render remote `http`/`https` image URLs inline instead of downgrading them to alt text.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision. If remote images should be supported, implement it coherently across renderer, sanitizer expectations, CSP, docs, and tests with a reviewed safety model such as opt-in, allowlist, or managed proxy; otherwise keep the current hardening and close after maintainer review.

## Reproduction Assessment

Yes. The static reproduction is high-confidence: `toSanitizedMarkdownHtml("![Alt text](https://example.com/img.png)")` reaches the renderer branch that returns escaped alt text, and the colocated test asserts no `<img>` output for that case.

## Solution Assessment

Unclear. A renderer-only change is not the best solution while current CSP, docs, and tests intentionally block remote image fetches; the maintainable path is either a policy-backed implementation or an explicit decision to preserve the hardening.

## Security Review

Status: needs_attention

Summary: The requested behavior is security-sensitive because rendered chat content could cause the operator browser to fetch arbitrary remote image URLs.

Concerns:

- **[medium] Remote images would trigger browser fetches:** `docs/web/control-ui.md:348`
  - body: Allowing `http(s)` markdown image URLs inline would let model- or user-supplied chat content cause the operator browser to request third-party hosts, conflicting with the documented Control UI no-remote-fetch policy unless maintainers add an explicit control model.
  - confidence: 0.93
- **[medium] Current CSP intentionally excludes remote image sources:** `src/gateway/control-ui-csp.ts:47`
  - body: The Control UI CSP omits `http:` and `https:` from `img-src`. A renderer-only change would either fail at runtime or require broadening CSP without the reviewed remote-media safety model this issue needs.
  - confidence: 0.91
