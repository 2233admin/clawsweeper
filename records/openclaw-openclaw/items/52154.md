## Summary

Keep this PR open. Current main still does not implement `commands.shellProfile` or an equivalent custom shell-profile loader for exec commands, and the PR remains the active implementation candidate for the linked shell-profile feature request.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a deliberately scoped exec-profile configuration surface, documented in the public exec/config docs, wired through the intended exec surfaces, and tested for PowerShell/POSIX command composition plus approval, allowlist, PTY, gateway, sandbox/node, and chat bash behavior.
