## Summary

Keep this PR open. Current main still does not implement the PR’s intended WhatsApp forwarding metadata path: WhatsApp inbound extraction reads Baileys contextInfo for mentions/replies but not isForwarded/forwardingScore, WebInboundMessage has no forwarding fields, buildWhatsAppInboundContext does not populate ForwardedFrom, and the shared prompt renderer has no forwarding_score output.

## Best Possible Solution

Keep this PR open for maintainer review. The best current path is a small current-main update in the bundled WhatsApp plugin: extract contextInfo.isForwarded and contextInfo.forwardingScore in the inbound extractor, thread them through WebInboundMessage or the newer UntrustedStructuredContext path, populate forwarded context in buildWhatsAppInboundContext, render forwarding_score only inside untrusted forwarded-message metadata, and add WhatsApp forwarded/non-forwarded regression tests. Do not close this contribution unless maintainers decide the generic structured-context variant should replace the proposed ForwardingScore field.
