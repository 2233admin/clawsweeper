## Summary

Keep open. Current main still does not implement the PR’s intended WhatsApp forwarding metadata path: WhatsApp inbound code does not extract Baileys `contextInfo.isForwarded` or `contextInfo.forwardingScore`, the WhatsApp inbound message contract has no forwarding fields, and the shared prompt renderer still has no `forwarding_score` output.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a small current-main update in the bundled WhatsApp plugin: extract `contextInfo.isForwarded` and `contextInfo.forwardingScore`, thread the data through `WebInboundMessage` into `buildWhatsAppInboundContext()`, render any score only in untrusted forwarded-message metadata, and add forwarded/non-forwarded WhatsApp regression tests. If maintainers prefer the newer generic `UntrustedStructuredContext` path over a shared `ForwardingScore` field, request that adjustment rather than closing the contribution.
