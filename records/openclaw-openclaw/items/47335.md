## Summary

Current main does not implement the reported compaction-retry delivery fix. The runner flushes the block reply pipeline before waiting for compaction retry completion, but there is no second flush after the wait resolves. The lifecycle handler also resolves the compaction retry before awaiting the async channel flush, and an existing test explicitly locks that ordering. The related reporter PR is closed/unmerged, while the open final-payload PR is related but does not specifically close this race.

## Best Possible Solution

Keep this issue open and fix the compaction retry ordering in the embedded Pi lifecycle/runner path. The narrowest likely fix is to ensure the retry wait cannot unblock until the retry's block reply pipeline flush has completed, either by awaiting `onBlockReplyFlush()` before `resolveCompactionRetry()` or by adding an idempotent post-wait flush in the runner, with a regression test for pending compaction retry plus async channel flush.

## Clownfish Work Prompt

Fix the compaction retry reply-delivery race described by this issue. Current main flushes `onBlockReplyFlush()` before `waitForCompactionRetryWithAggregateTimeout()`, but a retry's `agent_end` can resolve the compaction wait before the retry block-reply pipeline has finished async flushing. Keep the fix narrowly scoped to the embedded Pi runner/lifecycle delivery ordering. Prefer making the retry wait resolve only after the retry channel flush has drained, or add an idempotent post-wait flush if that proves safer; do not change channel-specific delivery policy or broad final-payload dedupe unless a test shows it is required. Add/adjust regression tests so pending compaction retry plus async `onBlockReplyFlush()` cannot unblock the attempt before the flush completes. Update the changelog under the active release fixes section, crediting the reporter/commenter appropriately, because this is user-facing silent reply loss. Validate with targeted repo test wrappers, then run the changed gate in Testbox per OpenClaw policy if promoted.
