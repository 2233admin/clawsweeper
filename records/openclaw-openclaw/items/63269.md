## Summary

Keep open. Current main still has Mattermost WebSocket `posted` parsing and a channel/group dispatch path, but I found no source, test, release, or related-PR proof that the reported `chatmode: "oncall"` plus `groupPolicy: "open"` public/group @mention regression is fixed. The later duplicate report was closed back to this canonical issue, and the adjacent Mattermost WebSocket liveness work does not specifically prove channel @mention delivery.

## What This Changes

Mattermost bots should receive and process public/group WebSocket messages, especially @mentioned `oncall` posts when `groupPolicy` is `open`, instead of silently handling only DMs after the 2026.4.8 regression.

## Best Possible Solution

Mattermost should reliably process allowed public, private, and group `posted` WebSocket frames, including @mentioned `oncall` posts with open group policy, with focused regression coverage and redacted diagnostics when a frame is filtered or never reaches the parser.
