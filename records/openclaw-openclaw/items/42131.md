## Summary

Current main still compares safe-bin trusted directories with exact resolved casing while doctor lowercases configured safeBins before resolution, so the mixed-case false warning remains plausible. This PR is a focused core exec-policy fix candidate, but its latest global temp-dir filesystem probe still needs trust-boundary review before merge or closure.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a shared, testable safe-bin path comparison helper that compares configured trusted dirs and resolved executable dirs consistently on genuinely case-insensitive paths, preserves distinct paths on case-sensitive or mixed-sensitivity volumes, and adds deterministic regression coverage for doctor/runtime mixed-case matching plus the case-sensitive sibling trust boundary.

## Risks / Open Questions

- Closing this PR would drop a plausible fix for a current core exec-policy bug on case-insensitive filesystems.
- The latest PR implementation may weaken the safe-bin trust boundary on systems where `os.tmpdir()` is case-insensitive but a configured trusted directory is on a case-sensitive volume, because distinct sibling paths can collapse to one lowercase key.
- The added PR tests, as described in the provided diff, mirror the runtime filesystem probe and may not deterministically prove both the intended case-insensitive match and the case-sensitive sibling non-trust boundary.
