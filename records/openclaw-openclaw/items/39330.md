## Summary

Keep open. Current main still appends the internal unscheduled-reminder note and carries those guarded payloads into final delivery; the latest release has the same behavior. This PR remains a relevant fix path, but it still needs revision so stripping is limited to text OpenClaw injected, not every assistant payload that happens to end with the same sentence. The PR security pass found only auto-reply source/test changes, with no workflow, dependency, lockfile, package, install, release, secret, or artifact-download changes.

## Best Possible Solution

Keep this PR open and revise it so final delivery never exposes OpenClaw's internally injected reminder-guard note while preserving legitimate model output. The narrowest fix is to track which payload appendUnscheduledReminderNote actually modified, or refactor the guard so it records the unscheduled-reminder condition without appending user-visible text, then update the existing reminder-guard tests for no-cron, existing-cron, unrelated-cron, disabled-cron, missing-sessionKey, and cron-store-read-failure cases.
