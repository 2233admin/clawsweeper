## Summary

Keep open. Current main still appends the unscheduled reminder guard note into the reply payload path, and the latest release has the same behavior. The PR is useful and narrowly scoped, but its current approach strips the exact note from every non-error text payload before delivery, which can remove legitimate model-authored text and leaves the append-then-strip flow confused.

## What This Changes

The PR adds a helper to strip the unscheduled reminder guard note from auto-reply payload text before typing/final delivery and updates reminder-guard tests to expect clean user-visible replies.

## Best Possible Solution

Revise the PR so the internally injected reminder note never reaches typing or final delivery while preserving model-authored content. The cleanest path is to avoid appending the internal note to deliverable payloads at all, or to track exactly which payload was internally modified and strip only that injected suffix; keep the #47663 config-flag discussion separate.
