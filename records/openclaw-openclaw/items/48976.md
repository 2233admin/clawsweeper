## Summary

Keep this PR open. Current main still does not normalize Feishu `mentions` returned by `im.message.get` for quoted/replied message content, so the intended fix is not implemented on main. The PR is also too broad to land as-is: the discussion and diff show unrelated dependency, command authorization, browser auth, cron cleanup, health, device bootstrap, and package changes mixed into a Feishu mention fix, with security-sensitive review comments still unresolved.

## Best Possible Solution

Keep the work open but request a narrow replacement or revised PR: update the Feishu fetched-message path in `extensions/feishu/src/send.ts` to model API `mentions` and reuse the existing mention-normalization behavior for `getMessageFeishu()`, add focused `send.test.ts` regression coverage for quoted text/post messages with `@_user_N` placeholders, and drop the unrelated package, command authorization, browser auth, cron, health, and device-bootstrap changes from this PR.
