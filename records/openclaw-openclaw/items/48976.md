## Summary

Current main still lacks the Feishu quoted/replied-message @mention normalization this PR targets. The PR is a useful implementation candidate for the linked Feishu bug, but it should not be closed as implemented or merged as-is because the branch also carries unrelated and review-commented security-sensitive changes.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but request a narrow revision or replacement: limit the change to the Feishu plugin, model the im.message.get mentions array in extensions/feishu/src/send.ts, reuse or share the existing normalizeMentions behavior safely for getMessageFeishu, add focused send.test.ts coverage for quoted text/post content with @_user_N placeholders, and remove the unrelated dependency, command authorization, browser auth, cron, health, and device-bootstrap changes.
