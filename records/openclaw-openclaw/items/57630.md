## Summary

Current main still lacks the requested `appendContext` and `prompt` hook result fields. The public hook result type, merge logic, embedded runner, CLI runner, harness helper, docs, and tests all remain limited to `prependContext`, `systemPrompt`, `prependSystemContext`, and `appendSystemContext`. This is a plausible core/plugin SDK seam rather than a ClawHub-only plugin request, though the PR likely needs a refresh and additional tests before merge.

## Best Possible Solution

Keep the PR open, but refresh it into a current-main plugin SDK change: add `appendContext` and `prompt` to the public hook result type and mutation field list, define deterministic merge semantics, apply the fields consistently in embedded runner, CLI runner, and harness prompt-build paths, update docs at `docs/plugins/hooks.md` and related concept pages, and add focused tests for priority order, empty prompt replacement, empty append handling, bootstrap warning preservation, and legacy hook merging.
