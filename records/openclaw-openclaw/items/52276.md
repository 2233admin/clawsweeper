## Summary

Keep open. Current main still uses text-only embedded message-tool duplicate suppression, while this PR’s route-scoped direction is useful but not mergeable because the branch calls an undefined routed-provider resolver and still drops the Telegram origin thread before suppression matching.

## What This Changes

The PR makes embedded messaging-tool duplicate suppression route-aware by threading provider, target, account, and current-route context through the embedded runner/subscriber paths and expanding message-send extraction tests.

## Best Possible Solution

Land a focused repair or replacement that keeps the route-scoped dedupe goal, defines the routed-provider resolver, carries origin provider/target/account/thread context through the embedded subscription boundary, and aligns message-send target tracking with outbound delivery normalization before suppressing final block replies.
