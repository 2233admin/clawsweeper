## Summary

Keep this PR open for maintainer review. The linked `/compact` cancellation bug still exists on main, and the PR is a relevant implementation candidate, but it is currently dirty against base and includes an unrelated launchd/profile change that regresses custom `OPENCLAW_LAUNCHD_LABEL` override behavior.

## What This Changes

The PR wires queued embedded compaction into the active-run abort path, exports embedded-run handle helpers, adds a drain guard to `/compact`, and also changes CLI profile handling/tests for launchd labels.

## Best Possible Solution

Keep the useful compact fix focused: adapt it to current main so queued manual compaction registers an abortable active-run handle when the task starts, forwards cancellation into compaction, cascades abort to any active reply operation, cleans up in `finally`, and lands with targeted cancellation coverage. Handle the launchd/profile label issue separately with maintainer-reviewed semantics for stale inherited labels versus explicit operator overrides.

## Review Findings

  - body: This block deletes any `OPENCLAW_LAUNCHD_LABEL` that differs from the profile-derived label. Split this from the compact fix and only clear labels that are known to be stale inherited service labels.
