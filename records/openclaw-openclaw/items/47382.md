## Summary

Keep open. Current main still lacks the requested configurable channel silence watchdog keyed to operator-visible output, and the adjacent typing, preview streaming, tool-progress, run-state health, and outbound delivery hooks do not provide the same timer/reset/repeat/max-beacon contract. A broader same-author proposal, #47591, is also open and should be triaged with this rather than closing this issue in isolation.

## What This Changes

Requested a configurable gateway-level channel silence watchdog that sends operator-visible status beacons during active work after a period of visible-output silence.

## Best Possible Solution

Define an opt-in, default-off generic runtime watchdog with documented operator-visible delivery timestamps, reset/cancel/repeat semantics, per-channel config/schema/docs, and focused coverage for outbound delivery, preview/block streaming, active run lifecycle, max-beacon behavior, and interaction with #47591’s broader delivery-confirmation design.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The high-confidence check is source/docs search on current main, which shows the requested watchdog contract is absent while adjacent liveness mechanisms exist.

## Solution Assessment

Unclear until maintainer approval. The maintainable direction is a generic shared runtime seam tied to real visible delivery, not per-channel ad hoc messages or model prompt rules.
