## Summary

Current main does not implement the requested channel silence watchdog. There are adjacent mechanisms for block/preview streaming, tool-progress previews, typing indicators, channel health, and outbound delivery diagnostics, but they are not a configurable runtime beacon keyed to operator-visible silence during active work.

## Best Possible Solution

Keep this open for maintainer product/design review. The likely path is an opt-in, default-off generic channel/runtime silence watchdog that observes active reply lifecycle and a well-defined operator-visible delivery timestamp updated by outbound delivery, block/preview streaming, tool-progress previews, and agent-emitted visible messages, with docs and focused tests for reset/repeat/max-beacon behavior.
