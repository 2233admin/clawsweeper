## Summary

Keep open. Current main still does not implement Slack Canvas content reading: the bundled Slack plugin exposes message/file/pin/reaction/member/emoji actions only, `download-file` still goes through `files.info` plus private file URLs, setup/docs do not include `canvases:read`, and the related implementation PR #18752 was closed unmerged.

## Best Possible Solution

Keep this open for bundled Slack plugin work. The best fix is a bounded Slack Canvas read action or dedicated Canvas tool that uses the existing Slack account/token model, adds explicit `canvases:read` setup/docs guidance, calls Slack Canvas read APIs, returns bounded markdown/section data with clear errors, updates the shared message action contract if a new action name is chosen, and adds Slack discovery/dispatch/runtime/docs/tests plus live Slack validation.
