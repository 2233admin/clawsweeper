## Summary

Current `main` does not implement the requested dynamic context-window guard budget yet. The guard still exports and defaults to fixed 16k/32k thresholds, the embedded runner still passes those fixed overrides, and the focused tests still assert fixed-threshold behavior. There is also an open implementation candidate, #72958, whose body uses closing syntax for this issue, so repository policy says this issue should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review #72958, or an equivalent focused patch, against current `main`. The implementation should derive guard warning/block thresholds from the effective model context window, stop passing fixed 16k/32k overrides through the embedded runner, update the focused guard/runtime tests, and document or changelog the user-facing behavior if it ships.
