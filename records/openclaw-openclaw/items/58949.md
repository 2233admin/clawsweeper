## Summary

Keep this PR open for maintainer security review. The reported invalid-input fast path still exists on current main, but the proposed patch replaces the current padded byte comparison with SHA-256 digest comparison and a CodeQL suppression, undoing the direction main just adopted for credential comparison.

## What This Changes

The PR normalizes null/undefined inputs before comparing SHA-256 digests in `safeEqualSecret` and adds null/undefined-versus-empty regression cases.

## Best Possible Solution

Keep the current padded timing-safe byte comparison as the equality primitive, add a reviewed invalid-input path that performs comparable dummy work before returning false, and retain the new null/undefined-versus-empty regression cases.

## Reproduction Assessment

Yes. Current main visibly returns before buffer allocation and `timingSafeEqual` for non-string inputs, while the PR diff visibly normalizes invalid inputs and executes digest comparison; no timing benchmark was run because this was a read-only review.

## Solution Assessment

No. The PR addresses a real hardening target and adds useful tests, but replacing the CodeQL-remediated padded byte comparison with SHA-256 digest comparison plus a suppression is not the safest maintainable solution.
