## Summary

Keep open. Current main still only offers real cron run/run-history flows, skills inspection/install/update, and normal tool execution after hooks, while related #41418 remains open for global dry-run semantics; the requested dry-run/test framework therefore still needs maintainer design.

## What This Changes

The issue requests side-effect-free dry-run/test and validation support for cron jobs and skills, including mock/read-only tool interception and CI-friendly output.

## Best Possible Solution

Design an additive implementation with cron test/validate RPC plus CLI, a skill test runner with explicit input and JSON/exit-code semantics, and a shared dry-run/mock/read-only interception contract aligned with #41418, then document and test the new CLI/protocol/runtime behavior.
