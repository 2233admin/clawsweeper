## Summary

Keep this issue open. Current main still lacks the requested side-effect-free cron and skills testing framework: cron has run/history/inspection commands but no test/validate command, skills has discovery/install/status commands but no input-driven test runner, and the shared tool adapter still executes real tools without a global dry-run interception layer. The related global tool dry-run design remains tracked separately in open #41418, so this needs maintainer design and likely splitting before implementation.

## Best Possible Solution

Keep this issue open until maintainers either ship an additive side-effect-free testing surface or split it into narrower canonical issues. The likely path is a cron test/validate Gateway protocol and CLI surface, an explicit skill test-runner decision, structured JSON/exit-code behavior for CI, and alignment with the global dry-run/tool-interception design tracked by #41418.
