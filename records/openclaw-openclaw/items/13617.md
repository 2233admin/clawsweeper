## Summary

Keep this issue open. Current main still lacks side-effect-free cron test/validate commands, an input-driven skill test runner, and a global dry-run interception layer for tool/API calls; the prior maintainer-side Codex review also explicitly kept this open and linked the unresolved global dry-run design in #41418.

## What This Changes

Add a dry-run/test framework for cron jobs and skills, including cron test/validate commands, a skill test runner, mocked tool/API execution, CI-friendly JSON and exit codes, and related configuration.

## Best Possible Solution

Keep this open for maintainer design or splitting. The likely path is an additive cron test/validate Gateway protocol plus CLI, an explicit skills test-runner model, CI-friendly JSON/exit-code behavior, docs/tests, and alignment with #41418 for any global tool interception or read-only/mock semantics.
