## Summary

Keep open. Current main still has real cron run queuing, no cron test or validate command, no input-driven skill test command, and no global/mockable tool interception layer. The issue discussion also already has a maintainer-side keep-open review tying the shared tool-interception portion to the open #41418 design.

## What This Changes

The issue requests side-effect-free cron and skill testing commands with dry-run validation, mocked tool calls, CI-friendly output, and JSON/exit-code behavior.

## Best Possible Solution

Design an additive contract that adds cron test/validate RPC plus CLI, a skill test runner with explicit input and JSON/exit-code semantics, and a shared dry-run or mock/read-only interception layer coordinated with #41418, with docs and focused coverage for each affected surface.
