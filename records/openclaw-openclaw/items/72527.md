## Summary

Keep open. The report is valid, current main still emits only the short future-version warning on ordinary config reads, and a paired open PR explicitly proposes the downgrade-warning improvement for maintainer review.

## What This Changes

The issue asks for clearer detection, warning, and repair guidance when a config written by OpenClaw 2026.4.x is read by OpenClaw 2026.3.2 and contains newer keys or value formats.

## Best Possible Solution

Keep the issue open while maintainers review PR #72535. The desired end state is a documented downgrade/recovery policy with actionable future-version guidance and exact manual fixes, plus doctor support only for known safe rewrites.

## Reproduction Assessment

Yes. The issue gives concrete versioned steps, and current source confirms the relevant behavior: configs are stamped, ordinary reads warn tersely, unknown keys can be stripped, and current channel streaming uses the newer object shape.

## Solution Assessment

Unclear. The paired PR appears to be a narrow guidance improvement, but it does not implement full automatic downgrade migration; maintainers should decide whether warning-plus-manual-guidance is the right policy.
