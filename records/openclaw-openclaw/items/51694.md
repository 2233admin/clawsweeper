## Summary

Keep this issue open. Current main still has the Google Chat config-contract mismatch: `appPrincipal` and `channels.googlechat.accounts` remain optional in schema/types/generated metadata and docs/setup omit `appPrincipal`, while the app-url add-on token path rejects requests when no principal binding is configured. The closed related PR #51817 was not merged, and current tests support a conditional requirement rather than a blanket global required flag.

## Best Possible Solution

Keep this open and make the Google Chat config contract match runtime deliberately. The safest direction is to document and validate the conditional requirement for app-url add-on token delivery without making `appPrincipal` globally required for project-number, and to update schema/help/generated metadata, setup surfaces, docs, and tests together once maintainers choose the exact contract.
