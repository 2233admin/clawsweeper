## Summary

Keep this issue open. Current main still has the Google Chat app-url add-on principal mismatch: `appPrincipal` and `channels.googlechat.accounts` remain optional in schema/types/generated metadata and docs examples, while the app-url add-on token path still rejects missing principal bindings. The prior related fix PR #51817 is closed unmerged, and the existing warning/logging is useful but does not make the config contract or docs match runtime behavior.

## Best Possible Solution

Keep this issue open and make the config contract match runtime deliberately: add app-url-specific validation or metadata that requires `appPrincipal` when Google Chat uses `audienceType: "app-url"` add-on delivery, update `GoogleChatAccountConfig`, generated config metadata, setup prompts if needed, and the public Google Chat docs together, while preserving the project-number path unless maintainers intentionally choose a broader requirement.
