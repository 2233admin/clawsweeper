## Summary

Keep open. Current main still implements a single combined chat session picker on desktop and mobile; the code groups sessions by agent internally but does not expose separate Agent and Agent Session controls. The related open items are complementary or partial mitigations rather than a full implementation of this request.

## What This Changes

Split the Control UI chat session picker into separate Agent and Agent Session controls while keeping Model and Thinking overrides scoped to the active session.

## Best Possible Solution

Implement the split in the Control UI chat surfaces: render an Agent selector from agentsList, render an Agent Session selector filtered to sessions whose parseAgentSessionKey(...).agentId matches the selected agent, switch agent changes to a deterministic session for that agent, and keep Model/Thinking sessions.patch calls targeted at the active state.sessionKey. Mirror the behavior in the mobile chat settings dropdown and coordinate with #68959 and #70851 without treating either as a complete replacement.
