## Summary

Keep open. Current main still renders one combined chat session selector on desktop and mobile, grouped by agent internally but not split into Agent and Agent Session controls. The related open agent-switcher issue is complementary, and the open internal-session filtering PR is only a partial picker-noise mitigation, not a superseding implementation of this request.

## What This Changes

Request to split the Control UI chat controls from one combined session picker into separate Agent and Agent Session selectors while keeping Model and Thinking scoped to the active session.

## Best Possible Solution

Implement the split in the Control UI chat surfaces: render an Agent selector from agentsList, render an Agent Session selector filtered to parseAgentSessionKey(...).agentId for the selected agent, switch agent changes to that agent's most recent updatedAt session or buildAgentMainSessionKey({ agentId }) as fallback, and leave Model/Thinking sessions.patch calls targeted at state.sessionKey. Mirror desktop behavior in the mobile chat settings dropdown and coordinate with #68959 and #70851 without treating either as a complete replacement.
