## Summary

Keep this issue open. Current `main` still skips `MEDIA:` directives inside Markdown fenced code blocks, current tests assert that fenced media remains visible text, and the related implementation PR #41980 was closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open. Maintainers should choose the intended media-directive contract, then update `src/media/parse.ts` and `src/media/parse.test.ts` either to extract valid own-line fenced `MEDIA:` directives through the existing validation path, or to preserve the skip behavior while adding a visible warning plus clearer system-prompt and docs guidance. Closed PR #41980 is useful prior art but needs review or replacement.

## Risks / Open Questions

- Extracting `MEDIA:` from code fences could accidentally treat illustrative documentation snippets as attachments unless it stays limited to valid own-line directives and uses the existing validation/trust-boundary path.
- If maintainers keep fenced `MEDIA:` unsupported by design, the silent-failure part still needs an observable warning and clearer assistant/docs guidance.
