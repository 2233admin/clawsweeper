## Summary

Keep this issue open. Current `main` still skips `MEDIA:` directives inside Markdown fenced code blocks before token validation/extraction, tests still assert that fenced media remains visible text, and the related implementation PR #41980 was closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open. The best path is to choose the intended media-directive contract, then update `src/media/parse.ts` and `src/media/parse.test.ts` either to extract valid own-line fenced `MEDIA:` directives through the existing validation path, or to keep fenced directives unsupported while adding a visible warning and updating assistant/docs guidance. The closed #41980 PR is useful prior art but should be reviewed or replaced before this issue is closed.
