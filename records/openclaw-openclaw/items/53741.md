## Summary

Keep open. Current main still lacks the requested Discord voice text-session routing, and the PR is useful but not mergeable as-is because routed delivery currently lacks an explicit Discord target, validation can leave an untracked voice connection, and the session-channel security boundary is still incomplete.

## What This Changes

The PR adds optional Discord voice `autoJoin.sessionChannelId` config/schema/generated-baseline support, routes auto-joined voice sessions through that channel, enables delivered text replies for routed voice turns, and adds session-channel validation tests.

## Best Possible Solution

Land a revised narrow implementation only if maintainers want this config option: validate a same-guild messageable target before or with guaranteed cleanup, pass an explicit Discord delivery target/run context, keep text delivery best-effort relative to voice playback, preserve routing during DAVE recovery, document the option in the Discord docs, and cover these paths with focused Discord voice and delivery tests. If #73699 becomes the chosen design, supersede this PR with that broader bridge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass a Discord reply target when enabling delivery:** `extensions/discord/src/voice/manager.ts:666`
  - body: When `sessionChannelId` is set, this call turns on delivery but still omits `to`, `replyTo`, and `runContext.currentChannelId`. The delivery layer treats `messageChannel` as the turn source and has no recipient, so Discord target normalization fails before the TTS path can run.
  - confidence: 0.93
- **[P1] Destroy the joined connection on validation failure:** `extensions/discord/src/voice/manager.ts:420-428`
  - body: `sessionChannelId` validation now happens after the voice connection reaches Ready. These new error returns leave that live connection undisposed and untracked by `this.sessions`, so later `leave` or status flows cannot clean it up.
  - confidence: 0.9
- **[P1] Require session channels to belong to the guild:** `extensions/discord/src/voice/manager.ts:425-428`
  - body: The same-guild check only rejects a mismatched `guildId`; channel objects without a guild id, such as DMs/group DMs, fall through and become the routed session target. Reject non-guild targets explicitly and allow only messageable guild channel types.
  - confidence: 0.86
- **[P2] Preserve routed sessions during DAVE recovery:** `extensions/discord/src/voice/manager.ts:788`
  - body: Decrypt recovery rejoins with only the voice `channelId`, so a session intentionally routed through `sessionChannelId` falls back to voice-channel routing after the first recovery and stops using the paired text channel.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The PR adds a new Discord text-delivery surface for voice sessions and still has concrete target-validation and mention-policy concerns.

Concerns:

  - body: `sessionGuildId` is optional in the new check, so non-guild channel objects can pass validation and become the delivery/session target. That can route voice-derived content outside the intended guild boundary if a DM/group-DM or otherwise non-guild channel id is configured.
