## Risks / Open Questions

- The latest PR design still needs maintainer review for empty directory assets: create can produce a checksum for an empty directory, while verify currently errors when no non-directory entries are found for a checksummed asset.
- The latest PR design still needs maintainer review for symlink/link-like entries: create-side hashing ignores symlinks when walking extracted files, while verify hashes every non-directory tar entry, which can cause mismatches or inconsistent coverage.
- The create path adds a tar extraction/repack staging step. It is based on a self-created archive, but link and path-safety behavior should be reviewed before merging a security-hardening feature.
