## Summary

Keep open. Current main still drops configured thinking budgets before calling pi-ai, so the PR is a relevant implementation candidate, but the proposed validation still forwards malformed token budgets that can reach provider request construction.

## What This Changes

The PR adds a `thinkingBudgets` resolver in the embedded Pi extra-params wrapper, forwards valid configured budgets into `SimpleStreamOptions`, and adds a MiniMax Anthropic-compatible regression test.

## Best Possible Solution

Amend the resolver to accept only positive integer token budgets, keep the pass-through regression, add invalid-budget coverage if feasible, then merge after targeted extra-params tests and the changed gate are green.
