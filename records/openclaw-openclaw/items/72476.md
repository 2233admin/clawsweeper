## Summary

Keep open. Current main still drops configured `thinkingBudgets` before calling pi-ai, so the PR addresses a real runtime gap, but the proposed resolver still forwards zero, negative, and fractional token budgets despite the PR contract to keep malformed data out of the request path.

## What This Changes

The PR adds a `thinkingBudgets` resolver in the Pi embedded runner, forwards configured budgets into pi-ai stream options, and adds MiniMax Anthropic-compatible regression coverage.

## Best Possible Solution

Land a narrow version of this PR after the resolver accepts only positive integer token budgets and adds focused coverage for valid pass-through plus invalid-value filtering, without changing unrelated provider defaults or widening the config schema.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `agents.defaults.models[model].params.thinkingBudgets`: current main preserves that value in effective extra params but never maps it into `SimpleStreamOptions`.

## Solution Assessment

No, not as written. Forwarding `thinkingBudgets` in this wrapper is the right fix boundary, but the validation should reject non-positive and non-integer token counts before merge.
