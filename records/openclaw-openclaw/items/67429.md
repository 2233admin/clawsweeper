## Summary

Keep open. Current main still lacks the requested cron.update null-clear behavior, and the PR is useful, but the current branch preserves null in a shared create/patch normalization path and would regress cron.add validation for callers that serialize optional override fields as null.

## What This Changes

The PR adds null-as-clear handling for cron.update agent-turn payload model, fallbacks, and thinking through cron patch types, normalization, gateway schema, service merge logic, and regression tests.

## Best Possible Solution

Land a revised version that defines null-clear as a cron.update patch-only contract, preserves create-time optional-null tolerance, deletes stored model/fallbacks/thinking overrides when patch values are null, and covers both the normalize-to-apply patch path and cron.add create compatibility in tests.
