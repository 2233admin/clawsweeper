## Summary

Keep this PR open. Current main and the latest release still lack dynamic `require()`/`import()` scanner rules, so the proposed security hardening is not obsolete, but the submitted regex-only implementation remains incorrect for a security-sensitive install-blocking scanner path.

## What This Changes

The PR adds two critical `LINE_RULES` in `src/security/skill-scanner.ts` to flag `require()` and `import()` calls whose first argument does not begin with a single or double quote.

## Best Possible Solution

Implement argument-aware module-load scanning that classifies the complete `require()` or dynamic `import()` specifier, then cover variables, concatenation, template literals with and without interpolation, comments around the callee and argument, multiline literals, and ordinary static literals before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Classify the entire module specifier:** `src/security/skill-scanner.ts:181-188`
  - body: The new regexes only inspect the first non-whitespace character after `(`. Calls such as `require("./" + name)` and `import("./" + name)` begin with a quote, so they skip the critical rule even though the resolved module target is runtime-controlled.
  - confidence: 0.93
- **[P1] Allow comments between the callee and call paren:** `src/security/skill-scanner.ts:181-188`
  - body: JavaScript permits comments between the callee and `(`, but these patterns only allow whitespace there. `require/*x*/(userInput)` and the matching `import/*x*/(userInput)` form bypass the new critical rule while still dynamically loading modules.
  - confidence: 0.9
- **[P2] Skip comments before literal import specifiers:** `src/security/skill-scanner.ts:188`
  - body: A static import such as `import(/* webpackChunkName: "x" */ "./mod")` starts with `/` after the paren, so the new `dynamic-import` rule reports a critical dynamic load even though the module specifier is a literal after the comment.
  - confidence: 0.87
- **[P2] Treat constant template specifiers as static:** `src/security/skill-scanner.ts:181-188`
  - body: Because the lookahead excludes only single and double quotes, constant backtick specifiers like `import(`./fixed.js`)` or `require(`./fixed.js`)` are reported as critical even when they contain no interpolation, creating install-blocking false positives.
  - confidence: 0.88
