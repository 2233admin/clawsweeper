Keep this PR open. Current main still does not strip inbound metadata before gateway session-title extraction, and the main gateway session row path still computes an outer displayName but does not pass it into deriveSessionTitle. The linked #39722 remains open, and #48942 is overlapping test/title work but has not superseded this fix.

## Best Possible Solution

Keep this PR or a successor open as the implementation candidate for #39722. The best fix is to strip inbound metadata before gateway session title/preview text extraction, pass the computed row displayName into deriveSessionTitle or equivalent precedence logic, and add focused regression tests for both behaviors while coordinating with #48942 if that refactor proceeds.
