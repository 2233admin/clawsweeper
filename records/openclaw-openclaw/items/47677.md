## Summary

Keep this open. Current main still treats Telegram reactions as received notification/context events rather than first-class wake or execution triggers, and the issue is paired with open PR #47687 plus the narrower open wake-trigger bug #64752.

## What This Changes

Request first-class Telegram reaction handling so standard and custom emoji reactions can wake the agent, map to configured operator semantics, and expose useful routing/debug state.

## Best Possible Solution

Review #47687 against #64752 and either land it or replace it with a narrower Telegram-plugin implementation that normalizes standard and custom emoji, maps configured reaction actions, wakes only authorized mapped reactions, logs ignored/rejected/executed decisions, and updates focused tests plus the public Telegram docs.
