## Summary

Keep open. Current main still lacks the requested broad Control UI project tree, composer route/fanout targets, selected-message split/move triage, and seed-elsewhere workflow. Existing code has narrower primitives only: grouped flat session selection, parentSessionKey/childSessions metadata, checkpoint branching, and display-only reply pills. The related #55349 quote-reply request is narrower and does not supersede this issue.

## Best Possible Solution

Keep this open as a broad Control UI product feature. A complete implementation should define additive project/session/tree metadata, add a Control UI project tree navigator, add composer route targets or fanout semantics, define selected-message split/move transcript behavior, and support seed/copy branch workflows with Gateway protocol, server, UI, docs, and tests. Reuse parentSessionKey, childSessions, checkpoint branching, slash-command steering, and reply-pill primitives where they fit, while keeping #55349 separate for quote-reply UX.
