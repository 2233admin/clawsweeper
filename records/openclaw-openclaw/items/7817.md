## Summary

Current main still lacks the requested Control UI project tree, composer routing/fanout, selected-message split/move triage, and seed-elsewhere workflow. Existing support covers useful but narrower primitives: flat/grouped session selection, parentSessionKey/childSessions metadata, checkpoint branching, and display-only reply pills. The related #55349 quote-reply request is narrower and does not supersede this broader product feature.

## Best Possible Solution

Keep this open as the broad Control UI product feature. A complete implementation should add additive session/project/tree metadata, a Control UI project tree navigator, composer route targets or fanout, selected-message split/move transcript semantics, and seed/copy branch behavior, with focused Gateway protocol, server, UI, and docs coverage. Reuse parentSessionKey, childSessions, checkpoint branching, slash-command steering, and reply-pill primitives where they fit, while keeping #55349 separate for quote-reply UX.
