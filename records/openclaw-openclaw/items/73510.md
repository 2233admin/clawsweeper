## Summary

Keep this issue open. Current main still only warns/emits `session.stuck` diagnostics and does not automatically abort the wedged run, clear the affected queue, or reset the session. Main has adjacent partial work for manual aborts, restart-aborted main-session recovery, pre-run transcript byte compaction, Feishu probe timeouts, and trajectory write caps, but those do not close the central permanent gateway hang report or the bundled startup-compaction/Feishu/resource-control follow-ups.

## Best Possible Solution

Keep this issue open. The best path is to review and land or replace PR #73243 for diagnostics-owned stuck-session abort/queue cleanup, then split or link the remaining startup compaction and large-transcript work to #64585/#71555/#60858, verify Feishu liveness behavior separately, and decide whether Linux gateway memory limits should be defaults, doctor repair, or documentation.
