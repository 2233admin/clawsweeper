## Summary

Keep this issue open. Current main has daily/idle rollover, heartbeat-aware freshness, heartbeat isolated sessions, pre-compaction memory flushes, and session lifecycle hooks, but it still does not expose the requested configurable normal-session max-turn TTL or explicit `sessionMaxAge`/`sessionMaxTurns` style policy. The remaining work also overlaps with open same-author issue #42867 and open session-memory reset PRs, so it needs maintainer consolidation rather than automatic closure.

## Best Possible Solution

Keep this open for maintainer consolidation. The likely path is to make #42867 the canonical turn-count/heartbeat slice or keep this issue as the umbrella, then design a normal-session TTL policy that counts real conversation turns, optionally supports absolute max age, excludes or explicitly handles system-event traffic, emits/logs lifecycle rotation reasons, and coordinates safe memory persistence through the existing lifecycle hooks or the open session-memory reset PRs.
