## Summary

Keep this open. Current main has daily/idle reset freshness that ignores heartbeat/system-event writes, plus heartbeat isolation options, but it still lacks the requested configurable normal-session maximum age and maximum turn-count rotation policy; the memory-safety part also overlaps several still-open automatic rollover hook PRs.

## What This Changes

Request configurable agent session TTL and max-turn rotation that saves memory, starts a fresh session, and logs rotation when a normal session exceeds age or turn limits.

## Best Possible Solution

Define one generic session reset contract for normal sessions, likely under the existing session.reset surface, that supports max age and max conversation turns, persists any needed counters, deliberately excludes or includes maintenance traffic, emits typed rollover reasons, and coordinates pre-rotation memory persistence through the existing lifecycle/hook system.

## Reproduction Assessment

Not applicable as a feature request. Static verification against current main is high-confidence: the schema, types, runtime reset gate, docs, and targeted source searches show daily/idle freshness but no max-age or max-turn normal-session policy.

## Solution Assessment

Unclear until maintainer/API review. The requested direction is valid, but the best maintainable fix is a unified reset policy and rollover-memory contract rather than adding ad hoc keys that bypass the existing session.reset model.
