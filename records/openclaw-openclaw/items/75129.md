## Summary

Keep open. Current main still resolves the CLI compaction context engine before registering built-in engines, so the PR addresses a real remaining cold-registry path, but the PR head is not mergeable because its new regression fixture fails the repo's test typecheck.

## What This Changes

The PR injects `ensureContextEnginesInitialized` into CLI compaction deps, calls it before resolving the compaction context engine, and adds a Vitest ordering regression test.

## Best Possible Solution

Keep the narrow initializer-before-resolver runtime change, amend the regression test to construct a valid `SessionEntry` with `updatedAt` and the existing token snapshot fields, then land after the targeted test, test typecheck, and changed gate are green.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a cold CLI-backed run that reaches `runCliTurnCompactionLifecycle` with a persisted session file and context token budget before any context-engine initializer has run; current main then resolves the default engine against an empty registry.

## Solution Assessment

No, not as currently patched. The runtime change is the right narrow solution, but the PR should fix the invalid `SessionEntry` fixture before merge; no broader core redesign is needed for this call site.
