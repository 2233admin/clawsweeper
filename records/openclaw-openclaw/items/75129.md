## Summary

Keep this PR open: the reported cold-registry bug is real on current main and the runtime fix is directionally correct, but the added regression test uses an invalid `SessionEntry` fixture that should block merge until amended and typecheck/CI complete.

## What This Changes

The branch injects and calls `ensureContextEnginesInitialized()` before CLI compaction resolves the context engine, and adds a Vitest ordering regression test for the cold CLI compaction path.

## Best Possible Solution

Keep the narrow initializer-before-resolver runtime change, amend the regression test fixture to satisfy the `SessionEntry` type, then land after the targeted test and changed gate are green.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a cold CLI-backed run that reaches `runCliTurnCompactionLifecycle` with a persisted session entry before any context-engine initializer has run; current main resolves the engine first, and the registry throws the reported default-engine-not-registered error when empty.

## Solution Assessment

No, not as currently patched. The runtime change is the right narrow fix, but the PR should use valid `SessionEntry` fields in the regression test before it is mergeable.
