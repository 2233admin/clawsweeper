## Summary

Keep open for maintainer review: current main still has the CLI compaction resolve-before-init path, and this PR is a focused source/test fix for that live behavior. I did not find a blocking correctness or security issue in the patch, but the head SHA still needs normal CI/changed-gate validation before merge.

## What This Changes

This PR imports the built-in context-engine initializer into CLI compaction, injects it through test deps, calls it before resolving the context engine, and adds a cold-registry ordering regression test.

## Best Possible Solution

Land this PR or an equivalent initializer-before-resolver change for CLI compaction after CI/changed-gate proof, while keeping #73904 as the separate native subagent-spawn fix. That keeps the lazy context-engine registry contract explicit at each cold resolver boundary instead of adding broader top-level side effects.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a cold CLI-backed run that reaches `runCliTurnCompactionLifecycle` before any context-engine initializer has run; the new injected mock test matches that path by throwing the production unregistered-legacy error unless initialization happens first.

## Solution Assessment

Yes. The proposed fix is the narrowest maintainable solution for this call site: call the existing idempotent built-in initializer immediately before resolving the context engine, matching adjacent initialized-first paths without changing registry side effects or plugin loading policy.
