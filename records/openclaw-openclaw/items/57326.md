## Summary

Current main still does not implement the requested helper-path CLI dispatch fix. The main reply runner has the correct pattern, but queued followups, `models status --probe`, LLM slug generation, and cron canonical-provider runs still have direct embedded/API paths or check `isCliProvider(...)` before resolving the configured CLI runtime. The issue also has same-author open PR #72499 using GitHub closing syntax, so this issue should stay open until that PR or an equivalent fix is reviewed and landed or rejected.

## Best Possible Solution

Keep this issue open and review, rebase, land, or reject PR #72499, or land an equivalent scoped patch. The fix should resolve configured CLI runtimes before isCliProvider(...) in queued followups, models status probes, slug generation, and cron; route CLI-backed runs through runCliAgent(...); preserve auth/session/routing/cancellation context; and add focused regression coverage for each helper path. Treat the later stale persisted harness-pin discussion as adjacent hardening unless maintainers deliberately fold it into the same patch.
