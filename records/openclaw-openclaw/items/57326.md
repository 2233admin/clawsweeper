## Summary

Keep this issue open. Current main still has direct embedded/API helper paths for queued followups, models status probes, and LLM slug generation, and the cron executor still checks CLI-ness before resolving canonical providers such as anthropic to a CLI runtime. The same-author PR #72499 is still the open implementation candidate and its body uses closing syntax for this issue, so this should not be closed before maintainers review, land, or reject that PR.

## Best Possible Solution

Keep this issue open and review or rebase PR #72499, or land an equivalent scoped fix. The implementation should resolve configured CLI runtimes before isCliProvider(...) in queued followups, models status probes, slug generation, and cron; route CLI-backed helper runs through runCliAgent(...); preserve auth/session/routing/cancellation context; and add focused regression coverage. The newer stale agentHarnessId pin report should be split into a narrower follow-up if maintainers want to harden persisted-session compatibility separately.
