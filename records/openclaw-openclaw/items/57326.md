## Summary

Current main still does not implement the requested helper-path CLI dispatch fix. The primary reply runner and direct agent attempt path now resolve configured CLI runtimes before checking `isCliProvider(...)`, but queued followups, `models status --probe`, LLM slug generation, and cron canonical-provider runs still have the remaining bypass shape described in this issue. The issue also has same-author open PR #72499 with closing syntax, so it should stay open until that PR or an equivalent fix is reviewed and landed or rejected.

## Best Possible Solution

Keep this issue open and review, rebase, land, or reject PR #72499, or land an equivalent scoped patch. The fix should resolve configured CLI runtimes before `isCliProvider(...)` in queued followups, models status probes, slug generation, and cron; route CLI-backed runs through `runCliAgent(...)`; preserve auth/session/routing/cancellation context; and add focused regression coverage for each helper path. Treat the stale persisted `agentHarnessId`/CLI-runtime-alias discussion as a separate compatibility hardening track unless maintainers choose to fold it in.
