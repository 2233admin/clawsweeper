## Summary

Keep this issue open. Current main still has the reported helper gaps: queued followups, models auth probes, slug generation, and cron canonical-provider runs can still bypass CLI runtime resolution or call the embedded runner directly. There is also an open same-author implementation candidate, PR #72499, that explicitly targets this issue, so the right maintainer path is to review/land or close that PR before closing the issue.

## Best Possible Solution

Keep this issue open and review, rebase, land, or reject PR #72499. The eventual fix should resolve configured CLI runtimes before isCliProvider(...) in queued followups, models status probes, slug generation, and cron; dispatch CLI-backed attempts through runCliAgent(...); preserve auth/session/routing/cancellation context; and add focused regression coverage. The stale persisted harness-pin concern from the latest comments should either be folded in intentionally with tests or split into a narrower follow-up issue.
