## Summary

This PR should stay open. Current main has partial hardening for model replay, but it does not implement the PR's full consumer-path goal: delivery-mirror artifacts are still written to transcripts and still flow through `chat.history` and `sessions.get` without a provider/model filter before visible pagination or API slicing. The PR is also the maintainer-designated canonical Track B candidate under #69208 and was actively repaired on April 28, 2026.

## Best Possible Solution

Keep this PR open as the canonical Track B implementation candidate under #69208, or replace it with an equivalent narrow rebased patch. The best implementation is a shared transcript-only artifact filter applied before model-context assembly, WebChat `chat.history` projection/limits, and `sessions.get` API slicing while preserving raw JSONL audit entries and adding focused regression tests for each consumer path.
