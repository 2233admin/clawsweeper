## Summary

Keep this PR open. Current main only partially covers the reported delivery-mirror leakage: provider replay/model context filtering exists, but visible/API consumer paths such as `chat.history` and `sessions.get` still consume raw transcript messages without a shared transcript-only OpenClaw artifact filter. A maintainer also explicitly kept this PR as the canonical Track B assistant-message-shape candidate under #69208.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent narrow rebased patch under #69208 Track B. The best implementation is a shared transcript-only OpenClaw assistant artifact predicate covering at least `delivery-mirror` and `gateway-injected`, applied before pagination and byte budgeting in visible/API consumer paths such as `chat.history`, `sessions.get`, and any session-history REST/SSE surfaces maintainers classify as user-facing. Preserve raw JSONL audit entries and add focused regression tests for the affected consumer paths.
