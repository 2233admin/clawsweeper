## Summary

Keep open. Current main still preserves structurally valid cross-root absolute sessionFile paths, including the same-agent stale-root shape this PR targets, so the intended behavior is not implemented on main. The paired same-author bug #61321 is still open, and this PR remains a plausible implementation candidate but has an unresolved custom session.store/sessionsDir concern that should be fixed before merge.

## Best Possible Solution

Keep this PR open and revise/review it. The implementation should remap stale same-agent absolute sessionFile paths relative to the active sessionsDir/session.store, preserve existing cross-agent fallback behavior, and add regression coverage for both default state-root and custom session.store cases before merge.
