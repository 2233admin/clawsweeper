## Summary

Keep this PR open: current main still implements the opposite precedence in both native subagent and isolated cron model selection, while the docs support the PR's global subagent-default behavior. The patch is targeted and I did not find a concrete blocking bug in its diff, but maintainers should choose between this PR and overlapping open PR #72877 before closing or landing it.

## What This Changes

This PR swaps subagent model precedence in the core helper and isolated cron path so per-agent `subagents.model` wins first, then `agents.defaults.subagents.model`, then the agent primary model, and updates related tests.

## Best Possible Solution

Settle the subagent model contract and land one canonical implementation that aligns code, tests, and docs. If the documented behavior is intended, land this PR or #72877/equivalent; if named-agent primary precedence is intended, update the docs and close the paired bug report with that decision.
