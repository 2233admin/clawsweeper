## Summary

Keep this PR open. Current main still does not implement the PR's intended Unix service-mode cleanup for npm-global `node ... openclaw ... gateway` listeners: the helper exists, but on Unix it still gets candidates from the command-name-filtered stale-pid path, so `lsof` entries reported as `cnode` are dropped before argv verification. The submitted branch also remains a poor merge candidate as-is because the PR diff bundles unrelated Slack/json changes and a runtime log artifact; a fresh focused patch is the best path.

## Best Possible Solution

Keep this PR open as an implementation candidate, but do not merge the current branch as-is. The best maintainer path is a fresh focused change on current main: collect raw listener PIDs for the gateway port on Unix, verify each candidate by argv (`/proc/<pid>/cmdline` on Linux and `ps` on macOS) before signaling, wire that into the shared service-mode stale cleanup path, and add a regression test where `lsof` reports `cnode` while argv identifies `node .../openclaw/dist/entry.js gateway --port 18789`. Drop the unrelated Slack/json/log changes or handle them in separate PRs.
