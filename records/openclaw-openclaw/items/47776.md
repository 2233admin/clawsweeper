## Summary

Current main does not implement this PR’s intended service-mode cleanup for npm-global `node ... openclaw ... gateway` listeners. The helper exists now, but the Unix discovery path still starts from `lsof -Fpc` command-name filtering, so `node` command listeners are not discovered before argv verification. The PR also remains stale/dirty relative to current main, with unrelated Slack/json/log changes in the supplied diff, so it should not be merged as-is.

## Best Possible Solution

Keep the PR open only as an implementation candidate. The best fix is a fresh, focused change on current main that makes Unix listener discovery collect raw port-listener PIDs and then verify argv before killing, wires that into service-mode startup or the shared stale-cleanup path, and adds a regression test for an npm-global `node ... openclaw ... gateway` listener. Drop the unrelated Slack/json/log changes from this PR.
