## Summary

Keep open. Current main still hardcodes the reported 128k/8192 limits for LiteLLM and Copilot Proxy onboarding, but this PR is not merge-ready because its LiteLLM changes target legacy command files that are gone on current main and the latest diff still has a concrete correctness bug around partially populated LiteLLM model metadata.

## What This Changes

The PR adds LiteLLM-compatible `/model/info` probing during onboarding so discovered `max_input_tokens` and `max_output_tokens` can replace static Copilot Proxy and LiteLLM model limits.

## Best Possible Solution

Land a refreshed provider-owned implementation under `extensions/litellm/` and `extensions/copilot-proxy/` that probes the configured LiteLLM-compatible model-info endpoint, authenticates when a LiteLLM key is available, preserves existing limits on probe failure or incomplete metadata, and adds focused tests for discovered limits, fallback, authenticated probes, trailing `/v1`, and re-auth preservation.
