## Summary

Keep this PR open. Current main still hardcodes 128k/8192 model limits for both LiteLLM and Copilot Proxy onboarding, and I found no current `/v1/model/info` or `/model/info` discovery path for those plugins. The PR is stale against the current plugin layout, but the intended bug fix is not implemented on main.

## Best Possible Solution

Keep the PR open as useful but request a rebase/port to the current plugin-owned surfaces: implement the `/model/info` capability probe in `extensions/litellm/onboard.ts` and `extensions/copilot-proxy/index.ts` or an appropriate generic plugin SDK seam, add focused tests for discovered limits, fallback behavior, trailing `/v1`, authenticated proxies, and preserving existing discovered limits, then re-run the changed provider/onboarding checks.
