## Summary

Keep this issue open. Current main has adjacent source labels, session visibility filtering, untrusted prompt framing, Memory Wiki provenance, and optional memory-lancedb prompt-injection mitigations, but it still does not implement durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval across the default memory path.

## Best Possible Solution

Keep open for maintainer/product/security design. A complete solution should define structured source trust metadata at ingestion, persist it through memory-core, QMD, session memory, Active Memory, Memory Wiki bridge, and SDK contracts, document trust-policy config, enforce retrieval-time filtering/downweighting/review gates, support expiration by trust tier, preserve provenance through compaction/dreaming flows, and add regression coverage across default and optional memory paths. HMAKT99's AKF comment is useful prior art to evaluate, not shipped behavior.
