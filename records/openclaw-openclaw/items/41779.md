## Summary

Keep this issue open. Current main still exposes `buffer`, `filename`, `contentType`, and `mimeType` on the shared `message.send` tool schema, but the shared outbound hydration helper still excludes `send`, the send runner still forwards only URL/path-style media, and Telegram still only receives `mediaUrl` for `sendMessage`. The open related PRs #42674 and #63578 remain implementation candidates rather than proof that the bug is fixed on main.

## Best Possible Solution

Keep this issue open until main either supports inline `buffer`/`filename`/`mimeType` attachments for `message.send` through the shared outbound path and Telegram plugin, or rejects those fields explicitly for `send` with a clear error. The practical maintainer path is to review, land, or replace #42674/#63578 with focused shared outbound and Telegram regression coverage.
