## Summary

Keep open. Current main still does not implement `workspaceConfig.allowedExternalPaths`: the config types and strict schemas omit the field, bootstrap context loading remains rooted to the workspace, and `agents.files.get/set` still use root-scoped safe file helpers. The PR is not obsolete, but it changes core file-boundary semantics and write behavior, so it needs maintainer/security review rather than automatic closure or a repair-lane replacement.

## What This Changes

The PR adds an opt-in default and per-agent `workspaceConfig.allowedExternalPaths` allowlist, wiring trusted external symlink targets through agent workspace context loading, extra bootstrap files, gateway `agents.files.*` reads/writes, docs, and regression tests.

## Best Possible Solution

Make an explicit core product/security decision. If OpenClaw should support shared external symlinked workspace context, land a reviewed implementation that covers config types/schemas, default-plus-per-agent allowlist merging, bootstrap and extra-bootstrap reads, `agents.files.get/set`, absolute realpath-prefix enforcement, in-workspace alias write rejection, docs, and focused tests. If core should not support it, document that external symlinked workspace context remains unsupported.
