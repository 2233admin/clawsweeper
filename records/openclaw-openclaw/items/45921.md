## Summary

Keep open. Current main still does not implement this PR's Prompt Inspector layer: there is no Prompt Inspector client/config, webhook requests still dispatch after ordinary validation, cron/external-content handling remains rule-and-wrapper based, and web tool results are not scanned by an ML/API detector. The PR is also security-sensitive because it adds third-party outbound detection calls, a new API-key env surface, and blocking behavior in gateway/agent hot paths, so it needs maintainer product/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The strongest path is either to harden the core feature with focused tests, docs, privacy review, config/secret handling, and final security review, or to split out a generic pre-dispatch/external-content inspection plugin API and publish Prompt Inspector itself as a plugin or ClawHub package.

## Risks / Open Questions

- The PR adds outbound HTTP(S) calls to promptinspector.io or an arbitrary PMTINSP_BASE_URL from gateway/agent paths, so privacy, data-retention, SSRF/proxy policy, and timeout behavior need explicit maintainer security review.
- The PR introduces a new PMTINSP_API_KEY secret/env surface and PMTINSP_ON_UNSAFE blocking policy; redaction, env precedence, fail-open behavior, and diagnostics need tests on the final branch.
- Closing as ClawHub would be premature unless OpenClaw first exposes a generic gateway/pre-dispatch inspection seam that can block built-in hook requests with the intended HTTP response.
