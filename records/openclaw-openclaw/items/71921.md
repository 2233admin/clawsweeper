## Summary

Keep open. Current main still preserves stale Bedrock explicit/static model rows without `input`, and the image gate still treats missing `input` as text-only, so the reported Bedrock Claude vision failure remains valid. PR #72404 is an open linked implementation candidate with closing syntax, so this issue should stay open until that PR or a replacement lands.

## What This Changes

Mark Bedrock-hosted Claude Opus/Sonnet 4.x model entries, including stale explicit/static catalog rows, as image-capable so image attachments are not rejected.

## Best Possible Solution

Land PR #72404 or an equivalent narrow replacement that refreshes missing Bedrock Claude `input` capability metadata for stale explicit/static rows, preserves intentional explicit opt-outs, and adds regression coverage for foundation, regional, and global inference-profile ids.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a non-empty `models.providers.amazon-bedrock.models` list containing a Claude 4.x row with no `input`: current merge code preserves that row, and the image gate returns false when `input` is missing.

Unclear. The linked PR is a plausible focused fix for stale explicit rows, but the most maintainable solution may be provider-owned capability refresh plus coverage for the discovery/runtime-dependency race that seeds stale catalogs.
