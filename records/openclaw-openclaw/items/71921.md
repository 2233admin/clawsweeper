## Summary

Keep open. Current main still preserves stale explicit Bedrock model rows without `input`, and missing `input` is still treated as no image support. The issue is also paired with open PR #72404, which the provided timeline shows references this issue as the implementation candidate, so the issue should remain open until that PR or a replacement lands.

## What This Changes

Mark Bedrock Claude Opus/Sonnet 4.x model metadata as image-capable and repair the stale explicit catalog path that leaves those models without `input: ["text", "image"]`.

## Best Possible Solution

Review and land PR #72404 or a provider-owned replacement that refreshes or inherits missing Bedrock `input` capability metadata for stale explicit/static rows after discovery succeeds, while preserving intentional explicit text-only opt-outs and covering foundation, regional, and global Claude ids.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a non-empty `models.providers.amazon-bedrock.models` list containing a Claude 4.x row with no `input`: current merge code preserves that row and `modelSupportsImages()` returns false for missing `input`.

## Solution Assessment

No for the original registry-only framing. Updating a few registry rows would not repair already-persisted static catalogs; the narrower maintainable fix should address stale explicit catalog metadata or replacement when discovery succeeds.
