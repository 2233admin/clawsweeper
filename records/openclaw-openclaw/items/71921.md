## Summary

Current main still has the reported stale Bedrock catalog path: successful Bedrock discovery can produce `input: ["text", "image"]`, but non-empty explicit/static `amazon-bedrock` model lists are preserved, and explicit-only model rows without `input` remain unchanged. Since image handling still treats missing `input` as no vision and open PR #72404 references this issue with closing syntax, this issue should stay open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open while #72404 is reviewed, or replace it with a provider-owned fix that refreshes stale explicit/static `amazon-bedrock` model rows from successful Bedrock discovery. The fix should include regression coverage for Bedrock Claude Opus/Sonnet 4.x explicit rows missing `input` or stuck at `["text"]`, including regional and global inference profile id variants.
