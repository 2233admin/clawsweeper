## Summary

Keep this open: current main and the latest release still normalize `kimi-coding` to `kimi`, the tests still encode that collapse, and an open candidate PR already references the issue with closing syntax.

## What This Changes

The issue asks OpenClaw to preserve provider-namespaced model refs such as `kimi-coding/k2p5` instead of collapsing the provider namespace to `kimi`.

## Best Possible Solution

Keep the issue open until PR #74315 or a replacement lands; the final fix should preserve exact configured provider namespaces where intended while retaining explicit legacy Kimi compatibility only where it is part of the supported contract.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: `normalizeProviderId("kimi-coding")` returns `kimi`, and runtime default and override paths call `normalizeModelRef` on those provider strings.

## Solution Assessment

Unclear. Removing `kimi-coding` from the global alias list is a plausible minimal fix, but an exact configured provider lookup before alias fallback may be safer if maintainers want to preserve legacy `kimi-code` behavior without collapsing custom namespaces.
