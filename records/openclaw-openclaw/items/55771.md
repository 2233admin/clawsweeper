## Summary

Keep this issue open. Current main still exposes `before_model_resolve` as an override-only plugin hook, while the requested `block`/`blockReason` contract and a blocked model-resolution path are still missing. The related budget-guard context is a real plugin API gap, not ClawHub-only work, because external plugins cannot implement this uniformly without a core hook result and runner path.

## Best Possible Solution

Add an additive core plugin API change: extend `PluginHookBeforeModelResolveResult` with `block?: boolean` and `blockReason?: string`, merge it with terminal sticky-block semantics matching `before_tool_call`, propagate a blocked outcome through `resolveHookModelSelection` and embedded/CLI runner paths before `resolveModelAsync`, surface the reason to the user, and update hook merger tests, model-selection tests, public SDK types, and plugin docs.

## Risks / Open Questions

- A partial workaround through `before_agent_reply` would not cover all model-call surfaces and would not give `before_model_resolve` the requested typed guard semantics.
- A correct fix needs a user-visible blocked-model outcome before provider dispatch, plus SDK/docs/tests so existing override-only plugins remain compatible.
