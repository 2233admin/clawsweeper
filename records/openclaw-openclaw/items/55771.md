## Summary

Keep this issue open. Current main still does not provide `block`/`blockReason` on `PluginHookBeforeModelResolveResult`, and the model-resolution path only consumes provider/model overrides. The linked budget-guard context is a real plugin use case, not a ClawHub-only request, because it needs a core plugin hook contract to abort model dispatch cleanly.

## Best Possible Solution

Add an additive core plugin API change: extend `PluginHookBeforeModelResolveResult` with `block?: boolean` and `blockReason?: string`, make the hook merger use terminal sticky-block semantics like `before_tool_call`, propagate a blocked model-resolution result through `resolveHookModelSelection`/the embedded runner before provider dispatch, surface a clear user-facing reason, and update focused hook, runner, SDK baseline, and docs coverage.
