## Summary

Keep this PR open. The underlying bug is still present on current main, but the latest PR shape is not ready to close or merge because the explicit-account guard still appears to affect internal account scans used for Discord action discovery.

## What This Changes

The PR changes Discord token/account resolution and regression tests so explicitly supplied non-default Discord accountIds do not inherit the channel/default token when the account token is missing, while preserving implicit default-account fallback behavior.

## Best Possible Solution

Keep the strict token resolution for actual explicit non-default outbound targeting, but make internal/default-account/discovery scans opt into implicit resolution and cover that distinction with token, account, and channel-action regression tests.
