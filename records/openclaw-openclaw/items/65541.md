## Summary

Keep this PR open. Current main at 213f92a9 still does not implement the proposed behavior: disabled block streaming buffers text-only and captioned media block replies for final delivery, tests assert that behavior, and docs still define blockStreamingDefault: "off" as final-reply-only. The PR is not obsolete, but it needs maintainer product judgment because it changes documented streaming semantics and intersects with the same-channel queued-delivery gap tracked in #32868.

## Best Possible Solution

Keep this PR open for maintainer judgment. If OpenClaw should support progressive, naturally bounded text delivery while block streaming is disabled, update reply-delivery.ts, disabled-streaming tests, streaming docs, final-payload dedupe semantics, and the same-channel dispatcher await/fallback contract together. If blockStreamingDefault: "off" should remain final-only, close this PR with that product decision and keep #32868 as the canonical same-channel delivery-ordering bug.
