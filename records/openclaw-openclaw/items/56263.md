## Summary

Keep open. Current main still enforces owner-only permissions for OpenClaw-managed state/config files and has no supported config or environment surface for `0640`/`0750` or skip-chmod behavior. The related closed bug #68709 moved in the opposite hardening direction: it treats group-readable config as a security finding, not as a supported multi-user mode.

## Best Possible Solution

Keep this issue open as core product/security work. A good implementation would add a documented config/env policy for OpenClaw-managed state file and directory modes, centralize mode resolution in the shared write helpers, update config schema/help/docs, and adjust security audit/doctor/fix so intentional `0640`/`0750` setups are distinguished from accidental exposure.

## Risks / Open Questions

- Changing write modes without a full policy update would conflict with current security audit, doctor, fix, docs, and tests that assume owner-only files and directories.
- A supported multi-user mode would need explicit product/security semantics for which state files may be group-readable, because config, auth profiles, credentials, and transcripts can contain sensitive data.
