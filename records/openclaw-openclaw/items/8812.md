## Summary

Current main still does not auto-linkify URLs inside Control UI or native WebChat tool-output cards. Normal chat markdown linkifies URLs, but tool-card rendering bypasses that path and renders output as Lit/SwiftUI text. Closed PR #8880 is useful prior art but was not merged or shipped.

## Best Possible Solution

Keep this issue open and land a focused implementation PR. Add a shared safe URL-aware renderer for Control UI tool output that preserves JSON/plain-text formatting, escapes arbitrary text before anchor creation, supports only safe http/https URLs, handles trailing punctuation, and covers expanded, raw, collapsed, inline, grouped, and sidebar tool-output paths. If native WebChat remains in scope, add equivalent SwiftUI URL rendering using AttributedString or Link handling and focused OpenClawKit tests. Treat PR #8880 as useful prior art, not shipped behavior.
