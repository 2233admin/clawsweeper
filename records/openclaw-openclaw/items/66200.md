## Summary

Keep this PR open. The hardening goal is valid for BlueBubbles, but the submitted patch is not merge-ready because it wraps only the legacy envelope `Body`; current `main` still sends raw BlueBubbles text through `BodyForAgent`, which is the field the reply pipeline prefers for the model-facing prompt.

## What This Changes

The PR imports `wrapExternalContent` into the BlueBubbles monitor and wraps the `baseBody` value passed to `formatInboundEnvelope(...)`.

## Best Possible Solution

Revise or replace the patch so BlueBubbles hardens the actual prompt-bound field, likely by using a guarded wrapped `BodyForAgent` or structured `UntrustedContext`, while keeping raw command/dedupe fields intact and adding targeted monitor tests.
