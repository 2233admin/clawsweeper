## Summary

Keep open. Current main has shipped a useful Gemini structured-input path for image/audio memory files, but the public modality contract still excludes PDF and video, and there is no clear persisted user-sent-media ingestion workflow for attachments that should become searchable memory.

## What This Changes

Request multimodal Gemini Embedding 2 memory indexing for raw images, video, audio, and PDFs, including media sent to an agent and marked worth remembering.

## Best Possible Solution

Keep the shipped image/audio foundation, then split the remaining work into a public PDF/video modality contract and a separate safe persisted-media ingestion path for attachments that users explicitly ask OpenClaw to remember.
