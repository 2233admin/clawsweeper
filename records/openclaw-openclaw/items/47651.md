## Summary

Keep open. Current main partially implements the request: Gemini Embedding 2 can index raw image/audio files as structured multimodal memory inputs and return them through memory search. The issue still asks for meaningful unimplemented scope: PDFs, video, and an inbound-media workflow where attachments marked worth remembering become searchable without first being placed in configured memory paths.

## Best Possible Solution

Keep this issue open, but narrow it to the remaining product gap: decide whether OpenClaw should support PDF/video raw media embeddings and define a safe remembered-inbound-media ingestion path. The existing Gemini image/audio extra-path implementation should remain the foundation, while #44734 can handle the narrower current audio-format regression.
