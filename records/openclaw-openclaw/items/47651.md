## Summary

Keep open. Current main implements a narrower Gemini Embedding 2 path for opt-in image and audio files under memorySearch.extraPaths, but the requested PDF/video modalities and explicit user-sent-media remember workflow are still missing and need maintainer product/security design.

## What This Changes

Request native multimodal memory indexing for images, video, audio, and PDFs with Gemini Embedding 2, including raw media sent to an agent and marked worth remembering.

## Best Possible Solution

Keep the shipped image/audio extraPaths foundation, then split the remaining work into a PDF/video multimodal memory contract and a separate explicit-remember attachment ingestion design with clear consent, storage, deletion, and provider-upload behavior.
