## Summary

Keep this issue open. Current main still registers a single default Telegram command menu with one description per command, and the relevant config, docs, and tests do not provide a language-keyed description surface. The broad i18n tracker #3460 is closed as not planned, but this report is narrower: user-supplied localized Telegram menu descriptions via the Telegram plugin, without requiring OpenClaw to maintain translation catalogs.

## What This Changes

Add configuration and runtime support for localized Telegram Bot command-menu descriptions, especially Chinese descriptions for built-in commands such as `/tts` and `/help`.

## Best Possible Solution

Add an additive Telegram-plugin config surface for language-keyed command menu descriptions, then register both the fallback command list and per-language lists via Telegram `language_code` while preserving stable command tokens, native routing, custom-command conflict rules, command count/text-budget limits, generated config docs, and focused tests.
