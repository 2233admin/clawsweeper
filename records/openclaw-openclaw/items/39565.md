## Summary

Keep this PR open. Current main still has the pre-PR TUI formatter behavior: punctuation stripping is computed but most copy-sensitive checks still classify the raw token, asterisks are not stripped, dotted IDs without underscores are still chunked with spaces, and the PR's regression tests are not present. The linked bug #39505 is still the right canonical bug, and this PR remains a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review, or land a rebased/cherry-picked equivalent. The fix belongs in src/tui/tui-formatters.ts with focused regression coverage in src/tui/tui-formatters.test.ts: classify the punctuation-stripped candidate, include '*' in edge punctuation, preserve dotted IDs and digit-bearing hyphenated identifiers, keep punctuation-only and prose-like long tokens chunkable, and consider dropping the unrelated lifecycle.test.ts formatting hunk.
