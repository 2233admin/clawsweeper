## Summary

Keep #59736 open. Current main has only a minimal root Makefile with a single `build` target and no self-documenting `help`, `test`, `check`, `dev`, or broader common-task aliases. Existing docs and scripts document many pnpm commands, but they do not satisfy the issue's requested discoverable Makefile/justfile workflow or acceptance criteria.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. The best implementation would either expand the root Makefile into a curated self-documenting wrapper over the existing pnpm lanes (`help`, `build`, `dev`, `test`, `test-fast`, `test-e2e`, `test-channels`, `test-extensions`, `check`, `lint`, `lint-fix`, `format`, etc.) and document it in contributor/development docs, or explicitly decide that OpenClaw prefers pnpm/docs-only discovery and close the request with that policy.
