## Summary

Keep open. Current main still has no SDK-facing task ledger RPCs and still documents/locks `oc.tasks.*` as unsupported, while this active PR is a relevant implementation for the linked app-SDK task-ledger request. It needs normal maintainer review rather than cleanup closure because it touches Gateway protocol, operator scopes, SDK docs/types, generated Swift client models, and overlaps the separate open draft PR #74815.

## What This Changes

This PR adds `tasks.list`, `tasks.get`, and `tasks.cancel` Gateway RPC schemas, handlers, method scopes, SDK methods/types, App SDK docs, generated Swift protocol models, and regression tests over the existing task registry.

## Best Possible Solution

Land one polished additive Gateway task-ledger contract after maintainer review confirms the metadata boundary, operator scopes, status mapping, cancel result semantics, generated native model shape, docs, and regression coverage; then let the linked task-ledger issue close through the merged implementation and retire the duplicate branch.

## Reproduction Assessment

Yes. Current main can be reproduced by static inspection: the SDK task helpers throw unsupported errors, the SDK test expects those errors, the docs list them as unsupported, and Gateway method discovery omits `tasks.*`.

## Solution Assessment

Yes overall, with a small native-client caveat. Exposing the existing durable task registry through additive Gateway RPCs and SDK methods is the narrow maintainable direction, but the Swift cancel result should preserve `TaskSummary?` before native clients rely on typed responses.
