## Summary

Keep open. Current main still lacks the SDK-facing task ledger RPC surface, and this active contributor PR is a relevant implementation for member-authored #74707; it should go through normal maintainer API review, with the native cancel-result typing caveat fixed or accepted before merge.

## What This Changes

This PR adds Gateway `tasks.list`, `tasks.get`, and `tasks.cancel` RPCs with protocol schemas, operator scopes, SDK methods/types, generated Swift models, docs, and regression coverage.

## Best Possible Solution

Land one polished additive task-ledger Gateway contract after maintainer review confirms the metadata boundary, status mapping, cancel result semantics, generated native model shape, and regression coverage; then let the linked task-ledger issue close through the merged implementation.

## Reproduction Assessment

Yes. Current main can be reproduced by static inspection and existing tests: `oc.tasks.list/get/cancel` still throw unsupported errors, docs list them as unsupported, and Gateway discovery/handlers/scopes do not include `tasks.*`.

## Solution Assessment

Yes overall, with a small native-client caveat. Exposing the existing durable task registry through additive Gateway RPCs and SDK methods is the narrow maintainable direction, but the Swift cancel result should preserve `TaskSummary?` before relying on typed native responses.

## Review Findings
