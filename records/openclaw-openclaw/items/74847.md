## Summary

Keep open. Current main still lacks SDK-facing task ledger RPCs and explicitly treats `oc.tasks.list/get/cancel` as unsupported, while this PR is an active implementation for the linked member-authored API request. The branch is relevant and broadly aligned with the additive Gateway/SDK direction; the remaining work is normal maintainer review, canonical-branch choice against overlapping #74815, CI completion, and one low-severity generated Swift model fix.

## What This Changes

The PR adds Gateway `tasks.list`, `tasks.get`, and `tasks.cancel` RPCs, SDK task methods/types, protocol schemas, operator scopes, generated Swift models, docs, and regression tests over the existing task registry.

## Best Possible Solution

Land one polished additive task-ledger Gateway contract after maintainer review confirms the metadata boundary, status mapping, cancel result semantics, generated native model shape, and regression coverage; then let #74707 close through the merged implementation and retire any duplicate PR.

## Reproduction Assessment

Yes. Current main can be reproduced by static inspection and existing tests: `oc.tasks.list/get/cancel` throw unsupported errors, docs list them as unsupported, and Gateway discovery/scopes do not include `tasks.*`.

## Solution Assessment

Yes overall, with a small native-client caveat. Exposing the existing durable task registry through additive Gateway RPCs and SDK methods is the narrow maintainable direction, but the Swift cancel result should preserve `TaskSummary?` before native clients rely on typed responses.
