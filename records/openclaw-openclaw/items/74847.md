## Summary

Keep open. Current main still lacks SDK-facing task ledger RPCs, while this PR is an active implementation for the member-requested #74707 surface; the remaining action is maintainer protocol/native-client review, not conservative cleanup closure.

## What This Changes

The PR adds `tasks.list`, `tasks.get`, and `tasks.cancel` Gateway RPCs over the existing task registry, SDK methods/types, protocol schemas, operator scopes, generated Swift models, docs, and regression tests.

## Best Possible Solution

Land one canonical additive task-ledger Gateway contract after maintainer review confirms the metadata boundary, status mapping, cancel result semantics, generated native model shape, and regression coverage; then let #74707 close through the merged implementation and retire any duplicate PR.

## Reproduction Assessment

Yes. Current main can be reproduced by static inspection: the SDK helpers throw unsupported errors, the SDK test and docs say they are unsupported, and Gateway discovery/scopes do not include `tasks.*`.

## Solution Assessment

Yes overall, with a native-client caveat. Exposing the existing durable task registry through additive Gateway RPCs and SDK methods is the narrow maintainable solution, but `TasksCancelResult.task` should be generated as `TaskSummary?` for Swift clients before merge.
