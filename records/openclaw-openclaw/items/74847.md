## Summary

Keep open. Current main still lacks the requested task ledger RPCs, and this PR is an active implementation for the open member-authored API request in #74707. Because it changes the public Gateway/App SDK/native protocol surface and overlaps the open draft PR #74815, maintainers should choose the canonical branch and address the remaining native model typing issue before merge.

## What This Changes

The PR adds `tasks.list`, `tasks.get`, and `tasks.cancel` Gateway RPCs, SDK methods/types, protocol schemas, scopes, docs, generated Swift models, and regression coverage over the existing task registry.

## Best Possible Solution

Land one polished additive Gateway task-ledger contract that preserves the intended TypeScript and native client types, keeps `tasks.list`/`tasks.get` behind `operator.read` and `tasks.cancel` behind `operator.write`, updates the SDK docs, and lets #74707 close through the merged implementation.

## Reproduction Assessment

Yes. Current main can be reproduced by static inspection: SDK task helpers still throw unsupported errors, docs/tests say they are unsupported, and Gateway method discovery does not advertise `tasks.*`.

## Solution Assessment

Yes overall, with a native-client caveat. Exposing the existing durable task registry through additive Gateway RPCs and SDK methods is the narrow maintainable direction, but the Swift cancel result should preserve `TaskSummary?` and the duplicate branch should be reconciled first.
