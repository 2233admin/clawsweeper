## Summary

Current main still does not support job-level model/provider selection for cron jobs that use `sessionTarget="main"` with `payload.kind="systemEvent"`. The public cron schema and TypeScript types keep `systemEvent` payloads text-only, normalization still strips model/provider-like fields from system events, main-session execution only enqueues text and wakes heartbeat, and the CLI/UI/docs continue to expose model selection only for agent-turn cron jobs. The related implementation PR #42708 was closed unmerged, so it is useful design context but not a shipped fix.

## Best Possible Solution

Keep this issue open. The best path is an additive cron contract for per-job model selection on main-session `systemEvent` jobs, using existing allowed-model validation and transient runtime plumbing so the cron run can use the requested provider/model without permanently changing the shared main-session selection. PR #42708 is useful prior design context, but it was closed unmerged and should not be treated as a shipped fix.
