## Summary

Current main still does not implement job-level model/provider overrides for cron jobs that use `sessionTarget: "main"` with `payload.kind: "systemEvent"`. The active contract keeps model selection on isolated/current/custom `agentTurn` payloads; main-session system events remain text-only and the runtime wake path has no field to carry a per-job model/provider override. The related implementation PR #42708 is closed unmerged, so it is design context rather than shipped behavior.

## Best Possible Solution

Keep this issue open. The best path is an additive cron contract for per-job model/provider selection on `main + systemEvent` jobs, validated against the existing allowed-model behavior and carried transiently through the main-session cron wake/system-event path. Use #42708 as prior design context, but do not treat it as shipped.

## Risks / Open Questions

- Closing now would hide a still-current feature gap for cheaper recurring main-session cron runs without manual session metadata edits.
- A future implementation needs to avoid persisting a cron job's transient model/provider choice into the shared interactive main-session defaults.
- The fix is cross-surface: protocol schema, runtime types, normalization, main-session system-event/heartbeat plumbing, CLI, UI, docs, and regression tests need to move together.
