## Summary

Keep open. Current main contains the related text-only Discord fix for #73709, but the reported default voice-enabled path is still active: voice defaults to enabled, `GuildVoiceStates` is still requested in that path, Carbon `VoicePlugin` is installed, and `DiscordVoiceManager` plus `DiscordVoiceReadyListener` still load when voice is enabled. I found no current source, test, docs, changelog, or history evidence that the default voice-enabled idle CPU spin has been fixed.

## Best Possible Solution

Keep this issue open and investigate in the Discord plugin: reproduce/profile gateway idle CPU with `channels.discord.voice.enabled=true`, isolate whether the loop sits in Carbon `VoicePlugin`, `@discordjs/voice`, `DiscordVoiceManager` lifecycle/listeners, auto-join/session recovery, or gateway voice-state handling, then make enabled voice idle safely or change the runtime/default explicitly. Add focused lifecycle or perf regression coverage for the identified loop and update Discord docs/changelog if defaults or behavior change.
