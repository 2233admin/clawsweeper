## Summary

Keep open. Current main contains the related text-only Discord fix for #73709, but this issue is about the default voice-enabled runtime path. That path is still active: voice defaults to enabled, `GuildVoiceStates` is still requested, Carbon `VoicePlugin` is still installed, and `DiscordVoiceManager` plus `DiscordVoiceReadyListener` still load when voice is enabled. I found no source, test, docs, changelog, or history evidence that the default voice-enabled idle CPU spin has been fixed.

## Best Possible Solution

Keep this issue open and investigate in the Discord plugin. Reproduce/profile gateway idle CPU with `channels.discord.voice.enabled=true`, isolate whether the loop sits in Carbon `VoicePlugin`, `@discordjs/voice`, `DiscordVoiceManager` lifecycle/listeners, auto-join/session recovery, or gateway voice-state handling, then make enabled voice idle safely or explicitly change the runtime/default with maintainer review. Add focused lifecycle or performance regression coverage for the identified loop and update Discord docs/changelog if defaults or behavior change.
