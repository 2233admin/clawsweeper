## Summary

Keep open. Current main still intentionally enables the Discord voice runtime by default, still requests `GuildVoiceStates` on that default path, and the shipped related fix only covers the text-only `voice.enabled=false` intent-selection bug rather than the enabled-voice idle CPU spin reported here.

## What This Changes

The issue reports sustained `openclaw-gateway` CPU spin when bundled Discord voice remains enabled after embedded Discord bot accounts connect.

## Best Possible Solution

Profile the bundled Discord plugin on current main with `channels.discord.voice.enabled=true` after real bot connections, then land a focused runtime fix or an explicit maintainer-approved voice-default change with Discord regression coverage, docs, and a changelog entry.

## Reproduction Assessment

Unclear: the report gives a concrete A/B path on v2026.4.26 and source still follows the implicated enabled-voice path, but this review did not live-profile current main with real Discord bot accounts.

## Solution Assessment

Not applicable because there is no patch. Closing via the #73709 text-only intent fix would not solve this enabled-voice report; the maintainable path is profiling first, then fixing the actual loop or making a deliberate default change.
