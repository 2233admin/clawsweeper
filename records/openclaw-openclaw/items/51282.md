## Summary

Keep open. Current main does not implement `gateway.handshakeTimeoutMs` as a config-file key; gateway pre-auth timeout still resolves from `OPENCLAW_HANDSHAKE_TIMEOUT_MS`, the test-only env fallback, or the 10s default. This PR is still conceptually relevant, but it needs to be rebased and completed against the current `handshake-timeouts.ts` path and the full config schema/help/label/test surface.

## Best Possible Solution

Keep the PR open but request a rebase and a complete implementation against current main: add the config field to `GatewayConfig`, `OpenClawSchema`, schema help, schema labels, generated schema/docs if required by the repo flow, and focused tests proving priority order `OPENCLAW_HANDSHAKE_TIMEOUT_MS` > `gateway.handshakeTimeoutMs` > 10s default. The runtime implementation should use the current `handshake-timeouts.ts`/gateway startup or runtime-config snapshot path rather than loading config from the unauthenticated socket hot path.
