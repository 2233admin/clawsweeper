## Summary

Keep this PR open. Current main still lacks the PR's central guidance: the macOS VM setup doc only tells users to enable Remote Login before running headlessly, and the LaunchAgent bootstrap error still points users to signing into the desktop or using a custom LaunchDaemon without mentioning VM auto-login. The PR is still useful, but it needs a refresh for current docs policy, security-sensitive wording, and test coverage.

## Best Possible Solution

Refresh this PR instead of closing it: keep the English macOS VM auto-login step in the post-setup/headless flow, add a narrowly scoped auto-login hint to throwBootstrapGuiSessionError, update the launchd guidance test, and remove the obsolete localized-doc edit unless the current i18n pipeline requires glossary-only changes.
