## Summary

Keep #57971 open. Current main still writes the macOS gateway LaunchAgent plist with 0644, and the linked PR is a plausible focused fix, but it needs the existing launchd test expectation updated from 0644 to 0600 before it can be considered complete.

## Best Possible Solution

Keep the PR open or ask the contributor/maintainer to update it so both `src/daemon/launchd.ts` and `src/daemon/launchd.test.ts` expect `0o600` for the LaunchAgent plist, then run the changed gateway/daemon test lane before merge. The core code path is the right place for this because OpenClaw itself writes and chmods the bundled macOS gateway LaunchAgent plist.
