## Summary

Keep #39680 open. Current main still has the reported value-only base64 heuristic, still calls value validation without env-var key context, and the current test suite still expects a warning for a non-credential key. The related fix PRs in the provided context were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep this issue open. Tighten `src/agents/sandbox/sanitize-env-vars.ts` by making the base64 warning structurally aware and/or env-key aware, then add regression tests for legitimate long alphanumeric, hex-like, and JWT-segment-like values while preserving explicit secret blocking and null-byte blocking behavior.
