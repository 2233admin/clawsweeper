## Summary

Current main still has the reported sandbox env-var false positive: `validateEnvVarValue` warns on any 80+ character value in the base64 character class, with no key context or structural base64 check, and the existing test still expects a warning for non-credential key `SAFE_TEXT`. Closed related PRs attempted this fix but did not merge; the open adjacent PR #53209 is about skill-declared sensitive env keys, not this false-positive bug.

## Best Possible Solution

Keep this issue open. The best fix is a narrow sanitizer policy change in `src/agents/sandbox/sanitize-env-vars.ts`, with regression tests that allow legitimate long alphanumeric, hex-like, and JWT-segment-like values while preserving explicit secret-name blocking, null-byte blocking, max-length behavior, strict-mode behavior, and warnings for genuinely suspicious base64 credential-shaped values. Because this is a sandbox/security warning path, secops should choose the exact heuristic before implementation.
