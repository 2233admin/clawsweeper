## Summary

Current main does not contain this PR's focused Nemotron empty-turn regression tests, so the PR is not obsolete. However, current main now treats OpenAI-compatible empty stop turns as retry-eligible, and the bundled NVIDIA provider is declared as `openai-completions`, so the PR's single-attempt/no-retry expectation may need updating before merge.

## Best Possible Solution

Keep this PR open, but ask for a small update before merge: either align the tests with current main's OpenAI-compatible retry policy by mocking NVIDIA as `api: "openai-completions"`, or document and test why NVIDIA/Nemotron should remain a no-retry exception despite the provider catalog. Keep the change test-only unless maintainer review decides the production retry policy needs a targeted adjustment.
