## Summary

Keep PR #70647 open. Current main already has the runtime empty-turn fallback behavior, but it does not contain the PR's specific non-strict-agentic Nemotron regression coverage for unsigned thinking-only `stop` turns or fully empty `content: []` `stop` turns. The PR is recent, focused, test-only, and tied to concrete prior NemoClaw hang reports; the only review feedback in the provided discussion is a minor issue-reference naming nit.

## Best Possible Solution

Keep the PR open and either merge it after the small issue-reference cleanup, or land equivalent focused regression coverage on main for non-strict `nvidia/nemotron-3-super-120b-a12b` turns that end with unsigned thinking-only `stop` and fully empty `content: []` `stop` responses. No production code change is needed unless that coverage exposes a behavioral drift.
