## Summary

Current main still has the behavior this PR is meant to fix. `openclaw models list` builds configured/default/fallback rows through `resolveConfiguredEntries`, and that path still resolves bare model keys and bare fallback refs with `DEFAULT_PROVIDER` instead of using the existing unique-provider inference helper. The PR should stay open for maintainer follow-up, especially because the PR review discussion also identifies an alias-index alignment issue that the submitted patch should address before landing.

## Best Possible Solution

Keep this PR open and either revise/land it or implement the same fix on main by centralizing provider inference for bare configured-list refs, including alias-index alignment. Add focused regression tests for bare configured keys, bare fallback entries, ambiguous duplicate model IDs, explicit provider prefixes, and aliases on inferred bare keys.
