## Summary

Keep this issue open. Current main still has the explicit `openclaw crestodian` no-TTY branch print the error and return normally, so `runCommandWithRuntime` has no thrown error to convert into exit 1. The prior implementation candidate, #73654, is closed and unmerged, so there is no landed fix to close against.

## Best Possible Solution

Keep the issue open and land #73654's narrow fix or an equivalent patch on main: make the explicit `runCrestodian()` no-TTY path throw or otherwise set exit 1, add a regression test for the explicit no-TTY path, and preserve the working `--message` automation path.
