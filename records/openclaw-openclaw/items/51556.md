## Summary

Keep this PR open for maintainer handling. The reported sync stat-error inconsistency still exists on current main, but the branch targets the old inline `pairing-store.ts` reader and its read-file change would make sync reads throw while the async helper still falls back, so the patch is not mergeable as-is.

## What This Changes

The PR changes two sync allowlist reader catch branches in `src/pairing/pairing-store.ts` to rethrow non-ENOENT stat and read errors instead of returning an empty missing allowlist.

## Best Possible Solution

Refresh the fix into `src/pairing/allow-from-store-file.ts`, add focused tests for non-ENOENT stat failures, and make an explicit maintainer decision on whether read-file failures should keep falling back or throw consistently across sync and async paths.
