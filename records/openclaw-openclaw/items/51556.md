## Summary

Keep open. Current main still swallows non-ENOENT errors in the sync allow-from reader, so the intended bug is not implemented on main. The PR should be refreshed because the code moved into the shared allow-from helper, and the readFileSync part needs product/behavior review because the async read-file path still falls back instead of throwing.

## Best Possible Solution

Keep the PR open and ask for a refreshed patch against `src/pairing/allow-from-store-file.ts`. The safest path is to add targeted regression tests for non-ENOENT stat failures, decide explicitly whether read-file failures should throw or remain fallback-compatible across both async and sync paths, and then update the shared helper so plugin SDK and internal callers stay aligned.
