## Summary

Keep this PR open. Current main still pre-fills the interactive `Workspace directory` prompt from `resolveAgentWorkspaceDir`, and that resolver still nests non-default agents under `agents.defaults.workspace`. The PR remains a plausible implementation candidate for open bug #71889, but it overlaps #71899 and #71941, so maintainers should choose the canonical fix before closing duplicate implementation attempts.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to select one canonical implementation among this PR, #71899, and #71941, then land a narrow regression-tested wizard-prefill fix that suggests the documented peer-level workspace for new agents while preserving existing runtime resolution and intentional shared-base configurations. The final fix should include direct `agents add` wizard coverage, resolver/helper coverage for the selected semantics, and a single changelog entry; after merge, close the duplicate implementation PRs and let #71889 close through the merged fix.
