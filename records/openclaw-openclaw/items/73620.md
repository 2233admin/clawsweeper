## Summary

Keep this PR open. Current main still seeds the interactive `openclaw agents add <id>` workspace prompt from `resolveAgentWorkspaceDir(cfg, agentId)`, which can produce the nested `agents.defaults.workspace/<id>` layout after normal setup, while docs still describe peer-level `workspace-<agentId>` defaults. The PR is a plausible focused fix, but it overlaps with open implementation candidates #71899 and #71941, so maintainers should choose a canonical path rather than auto-close this one.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose one canonical implementation among this PR, #71899, and #71941, land a narrow regression-tested wizard-prefill fix that suggests peer-level `workspace-<id>` for new agents while preserving existing runtime resolution for configured/nested workspaces, then close the duplicate PRs after the selected fix merges.
