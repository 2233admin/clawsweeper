## Summary

Keep open. Current main does not implement the requested canonical macOS LaunchAgent PATH: the service PATH helper still includes existing Volta/asdf/fnm/pnpm-style user directories, and the install plan writes that computed PATH into the service environment. This needs maintainer policy review because current tests and docs intentionally describe the opposite behavior.

## What This Changes

Request to make the macOS gateway LaunchAgent installer write a canonical minimal PATH instead of preserving or synthesizing Node version-manager and package-manager directories that can affect update-time Node resolution.

## Best Possible Solution

Define one maintainer-approved service PATH policy for macOS LaunchAgents, then update the installer, doctor repair/audit logic, docs, and service-env tests together. A likely end state is to separate the explicit gateway Node executable from subprocess search PATH and only include system dirs plus deliberately configured OpenClaw/service wrapper dirs.

## Reproduction Assessment

Yes for the core PATH behavior: current source and tests show the macOS service PATH helper includes the reported version-manager/package-manager directories when present. The broader timeout and gateway version-mismatch symptoms still need macOS live confirmation.

## Solution Assessment

Unclear. The requested canonical PATH is plausible for deterministic updates, but current main intentionally supports version-manager and user-bin dirs, so the best fix depends on a maintainer decision about LaunchAgent PATH compatibility versus determinism.
