## Summary

Keep open. Current main still routes Signal group allowlist checks through sender matching, and group mention gating still falls back to `requireMention: true` without an explicit override. The provided GitHub context shows open PR #73538 is already paired with this issue via closing syntax, so this should wait for maintainer review, merge, revision, or replacement rather than being closed or duplicated.

## What This Changes

The issue asks OpenClaw to make Signal group allowlists match inbound Signal group IDs and to avoid requiring a mention by default for explicitly configured Signal groups.

## Best Possible Solution

Review and land or revise #73538 for the Signal-local group-id allowlist match with regression tests for group IDs and sender-shaped allowlist entries. Handle the `requireMention` default as an explicit maintainer policy decision, with matching updates to group-policy tests and public Signal/groups docs if the default changes.
