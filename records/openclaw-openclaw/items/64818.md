## Summary

Keep open. Current main still has the reported updater failure shape: `src/canvas-host/a2ui/.bundle.hash` is tracked while ignored/generated, `pnpm build` still runs the A2UI bundler first, that bundler can rewrite the tracked hash file, and the dev update preflight loop still reuses one disposable worktree across fallback candidates with plain `git checkout --detach` and no reset/clean before the next candidate. The related minimal PR #64822 is closed unmerged, and #67547 tracks A2UI reproducibility rather than the updater checkout-conflict resilience bug.

## Best Possible Solution

Fix this in OpenClaw core. Add a regression test where dev update preflight dirties a tracked generated file and then must continue to an older fallback candidate, harden the disposable preflight worktree by resetting/cleaning before each candidate checkout, and separately either untrack `src/canvas-host/a2ui/.bundle.hash` again or make the A2UI bundler/hash contract avoid dirtying tracked files unexpectedly. Treat #67547 as related A2UI reproducibility work, not as a replacement for the updater resilience fix.
