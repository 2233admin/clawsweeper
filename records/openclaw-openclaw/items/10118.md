## Summary

Keep this issue open. Current main has partial multiline composition through the pinned `pi-tui` editor, but OpenClaw still does not document newline composition in the TUI docs or `/help`, and it exposes no CLI/config bridge for configurable send/newline keybindings. The related implementation PR is closed and unmerged, so it does not supersede the remaining work.

## Best Possible Solution

Keep the issue open and narrow it to the remaining product work: document multiline composition, terminal limitations, and fallback behavior in `docs/web/tui.md` and TUI `/help`; decide whether OpenClaw should expose a config/CLI bridge into `pi-tui` keybindings for Enter/Ctrl+Enter/Shift+Enter preferences; then add focused tests for help visibility and option wiring if configuration is added.
