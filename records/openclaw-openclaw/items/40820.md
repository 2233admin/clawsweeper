## Summary

Keep open. Current main still does not have a Telegram `/settings` inline keyboard panel, so the contribution is not obsolete, but the branch is not merge-ready against current main because it targets the pre-plugin `src/telegram` layout and writes legacy scalar Telegram streaming config.

## What This Changes

The PR adds a Telegram-only `/settings` inline keyboard panel for DM policy, group policy, streaming, and allowlist display, plus callback auth/config-write helpers, native command registration changes, and focused tests.

## Best Possible Solution

Port the feature into `extensions/telegram/src` using current Telegram plugin command/menu and callback patterns, keep Telegram-specific behavior out of core where existing plugin metadata can express it, write canonical `streaming.mode` paths for default and named accounts, and add matching Telegram tests plus user-facing docs/changelog coverage before maintainer merge review.
