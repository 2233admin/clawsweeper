## Summary

Keep this PR open. Current main still lacks `neverReply` or an equivalent passive group-observation mode that records group context while guaranteeing no model run and no outbound reply. The linked `openclaw-listen-only` plugin direction is useful, but current main still does not broadcast unbound `inbound_claim` hooks, so the plugin path does not clearly supersede this PR yet.

## Best Possible Solution

Keep this PR open for maintainer product and architecture review. The best path is either to finish a generic, tested passive group-observation design across the affected channel owners, or first land the missing plugin hook/API behavior needed by the linked `openclaw-listen-only` plugin and then explicitly supersede this PR with that plugin direction.
