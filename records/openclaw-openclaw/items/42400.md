## Summary

Keep this PR open. Current main still does not implement `neverReply` or an equivalent passive group-observation mode that records group context while guaranteeing no model run and no outbound reply. The linked plugin direction is useful, including the `openclaw-listen-only` plugin mentioned in discussion, but main still lacks the global unbound `inbound_claim` broadcast needed for that plugin path to supersede this PR.

## Best Possible Solution

Keep this PR open for maintainer product and architecture review. The best path is either to finish a generic, well-tested passive group observation design across the affected channel owners, or first land the missing plugin hook/API behavior needed by the linked listen-only plugin and then explicitly supersede this PR with that plugin direction.
