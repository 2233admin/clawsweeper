## Summary

Keep this PR open. Current main still has no `neverReply` config or equivalent no-LLM/no-outbound group observation path, and the plugin-hook alternative discussed in the PR is not yet a complete replacement because current dispatch explicitly does not broadcast `inbound_claim` for unbound conversations. The PR is also not a conservative cleanup close candidate because its latest diff mixes the channel feature with plugin loader/path-trust changes that need maintainer security review or splitting.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review. The best path is either a focused core `neverReply` implementation with schema/docs/tests and consistent cross-channel pending-history behavior, or an explicit maintainer decision to add the missing plugin hook/API seam and move listen-only behavior into a plugin such as `https://github.com/flowolforg/openclaw-listen-only`. If this PR continues, split the plugin loader/path-trust changes from the channel feature before merge review.
