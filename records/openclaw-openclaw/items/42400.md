## Summary

Keep this PR open. Current main still does not ship `neverReply`, `resolveNeverReply`, or an equivalent documented channel/default config that suppresses group replies while preserving passive context. The newer `before_dispatch` hook is relevant and may be the right plugin architecture, but the linked public listen-only implementation and discussion do not yet make this PR obsolete without explicit maintainer product/architecture judgment.

## Best Possible Solution

Keep this PR open for maintainer product and architecture review. The best path is either to finish a generic, well-tested passive group-observation design across the relevant channel owners, or explicitly supersede it with a documented ClawHub/plugin implementation built on `before_dispatch` that covers the same no-model/no-outbound use case and context-retention story.
