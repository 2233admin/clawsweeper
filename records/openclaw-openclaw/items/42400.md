## Summary

Keep this PR open. Current main still does not implement the proposed `neverReply` config/resolver or an equivalent zero-LLM passive group observation mode across group-capable channels. Main has adjacent behavior, including mention-skip pending history/ingest and default group final-reply suppression, but mentioned group messages can still reach the agent and explicit message-tool sends remain available. The plugin direction from the discussion is useful, especially the linked `openclaw-listen-only` approach, but current main still does not broadcast `inbound_claim` to unbound plugins, so that is not a complete replacement today.

## Best Possible Solution

Keep the PR open for maintainer product judgment. The clean path is either to finish a generic, well-tested suppression/ingest design across the affected channel owners, or first land the missing plugin API needed for a ClawHub listen-only plugin and then supersede this PR with that plugin direction. In either path, the shipped behavior should clearly separate pending context accumulation, LLM invocation, and transport-level outbound blocking.
