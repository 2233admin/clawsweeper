## Summary

Keep this PR open. Current main still does not ship `neverReply`, `resolveNeverReply`, or an equivalent documented channel/default config that suppresses group replies while preserving passive context without an LLM call. The linked `flowolforg/openclaw-listen-only` plugin and the merged `before_dispatch` hook are real, relevant architecture context, but they do not make this active core PR obsolete without explicit maintainer product/architecture judgment.

## Best Possible Solution

Keep this PR open for maintainer product, architecture, and security review. The best path is either a focused core implementation of passive group observation with tests/docs across the relevant channel owners, or an explicit maintainer decision to supersede it with a documented ClawHub/plugin implementation such as the linked listen-only plugin. Before any merge, split or rebase away unrelated install/plugin-discovery/trash-root changes so the feature review stays auditable.
