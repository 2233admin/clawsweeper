## Summary

Keep this PR open. Current main still has no `neverReply` config or equivalent passive group-observation mode that records group context while guaranteeing no LLM invocation and no outbound reply. Main has adjacent mechanisms, but visible-reply suppression and `sendPolicy` still process inbound messages, and global `inbound_claim` broadcast is still not available for an external listen-only plugin to fully supersede the PR.

## Best Possible Solution

Keep this PR open for maintainer product and architecture review. The best path is either to finish a generic, well-tested passive group observation design across the affected channel owners, or first land the missing plugin API/broadcast behavior needed by the linked listen-only plugin and then explicitly supersede this PR with that plugin/ClawHub direction. The shipped design should clearly separate pending context accumulation, LLM invocation, automatic final-reply suppression, explicit message-tool sends, and transport-level outbound blocking.
