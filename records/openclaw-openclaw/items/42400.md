## Summary

Keep this PR open. Current main still does not implement the requested `neverReply` passive group-observation config, and the existing group `visibleReplies`, mention-skip pending history, send-policy, and plugin hook surfaces do not fully replace the PR’s stated account/channel/default config behavior without maintainer product judgment. The PR also still needs normal review because its current diff includes unrelated security-sensitive plugin-loader, installer, and trash-root changes alongside the channel feature.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review. The best path is either a focused core implementation of passive group observation with schemas, docs, and tests across supported channels, or an explicit maintainer decision to supersede it with a documented plugin/ClawHub approach such as the linked listen-only plugin. If this branch continues, it should be rebased and the unrelated installer, plugin-loader, and trash-policy changes should be split away first.
