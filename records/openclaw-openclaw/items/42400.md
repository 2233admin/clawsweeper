## Summary

Keep open. Current main still does not implement `neverReply` or an equivalent no-LLM/no-outbound group observation mode. Existing mention-gating stores only unmentioned messages, while `sendPolicy` and visible group reply controls still run the agent turn. The plugin-hook alternative discussed in the PR is plausible, especially the linked external listen-only plugin, but current main still does not broadcast unbound `inbound_claim` hooks, so it cannot yet replace this PR without a maintainer architecture decision.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review. The best path is either a focused core `neverReply` implementation with schema/docs/tests and consistent account-scoped pending-history write/read/clear semantics across supported channels, or an explicit maintainer decision to broaden the inbound-claim/plugin seam and move listen-only behavior into a plugin such as `https://github.com/flowolforg/openclaw-listen-only`.
