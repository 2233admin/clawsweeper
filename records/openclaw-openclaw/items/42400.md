## Summary

Keep this PR open. Current main still does not implement `neverReply` or an equivalent passive group-observation mode that guarantees no model run and no outbound reply while retaining group context. The linked plugin approach is relevant, but current main still does not broadcast unbound `inbound_claim` hooks, so a ClawHub/plugin replacement does not yet supersede this PR.

## Best Possible Solution

Keep this PR open for maintainer product and architecture review. The best path is either to finish a generic, well-tested `neverReply`/passive group-observation design across the relevant channel owners, or first land and document the missing plugin hook/API behavior needed by the linked `openclaw-listen-only` plugin and then explicitly supersede this PR with that plugin direction.
