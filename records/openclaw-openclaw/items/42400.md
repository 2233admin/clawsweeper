## Summary

Keep this PR open. Current main still does not ship `neverReply`, `resolveNeverReply`, or an equivalent documented channel/default config that suppresses group replies while preserving passive context without an LLM call. The linked external listen-only/plugin discussion is relevant, but it does not make this active PR obsolete without maintainer architecture judgment. Security/maintainer review is still needed because the current PR also carries unrelated install/plugin-discovery changes outside the stated channel feature.

## Best Possible Solution

Keep this PR open for maintainer product, architecture, and security review. The best path is either a focused core implementation of passive group observation with tests/docs across the relevant channel owners, or an explicit supersession by a documented ClawHub/plugin implementation that covers the same no-model/no-outbound/context-retention contract. Before merge, split or rebase away unrelated install/plugin-discovery changes so the feature review stays auditable.
