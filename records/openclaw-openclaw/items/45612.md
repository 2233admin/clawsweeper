## Summary

Keep this PR open. Current main still generates backup archive roots with the shared session archive timestamp formatter, which is UTC-based and emits the trailing `Z` form. The docs and CLI help also still show UTC-style backup filenames, so the PR remains a focused implementation candidate rather than work already implemented on main.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review or land a scoped backup-specific timestamp formatter that uses local machine time with an explicit UTC offset. Keep the shared session archive formatter unchanged unless maintainers also want that broader behavior change, and align backup docs/help plus regression tests with the final filename contract.
