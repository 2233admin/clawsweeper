## Summary

Keep this PR open. Current main still has the two Nostr DM delivery failures that make the PR useful: the Nostr gateway account startup returns immediately, which the generic gateway supervisor treats as an exited channel and restarts, and the DM subscription still passes an array where pinned `nostr-tools` 2.23.3 expects a single `Filter`. The PR should not merge as-is because the lifecycle code moved from the older inline `channel.ts` path into `gateway.ts`, and its `sendMedia` fallback would mark Nostr as media-capable despite current docs and metadata saying media attachments are unsupported.

## Best Possible Solution

Keep this PR open as a useful but stale implementation candidate. The maintainer path is a refreshed Nostr-owned patch on current main: make `startNostrGatewayAccount()` stay pending until `ctx.abortSignal` with idempotent cleanup, pass the single `Filter` object required by pinned `nostr-tools` 2.23.3 in the DM subscription path, add focused lifecycle and subscription-shape regression coverage, and leave media unsupported unless a public-URL-only media design is explicitly added. Open issue #53858 should remain the canonical user-facing restart-loop report until that corrected fix lands.
