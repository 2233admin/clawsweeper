## Summary

Keep open. Current main still has the two important Nostr DM delivery failures this PR targets: the Nostr gateway start task resolves immediately, and the DM subscription still passes an array-wrapped filter to `nostr-tools`. The PR is useful but stale against current main, and its `sendMedia` fallback conflicts with the current no-media Nostr contract.

## Best Possible Solution

Keep this PR open as a useful but stale implementation candidate. The maintainer path is a refreshed Nostr-owned patch on current main: make `startNostrGatewayAccount()` stay pending until `ctx.abortSignal` using `runPassiveAccountLifecycle` or equivalent idempotent cleanup, pass the single `Filter` object required by pinned `nostr-tools` 2.23.3 in the DM subscription path, add focused lifecycle and subscription-shape tests, and leave media unsupported unless a separate public-URL-only media design updates the channel metadata, tests, and docs. Open #53858 should remain the canonical user-facing restart-loop report until that corrected fix lands.
