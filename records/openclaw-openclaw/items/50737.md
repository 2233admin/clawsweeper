## Summary

Keep this PR open. Current main still does not make custom Anthropic onboarding verification honor `models.providers.*.authHeader`; the Anthropic probe path always builds `x-api-key` headers and the prompt flow never passes provider config or an auth-header choice into verification. Main has a broader runtime `authHeader` mechanism, so the PR should be rebased/reconciled with that code rather than closed as obsolete.

## Best Possible Solution

Rebase this PR onto current main and narrow it around the remaining gap: make Anthropic-compatible custom-provider verification and runtime behavior consistently honor `authHeader: true`, while preserving native Anthropic defaults and explicit false overrides. Add focused tests in the onboarding config/prompt path and runtime auth path, then resolve the existing review comments before merge.
