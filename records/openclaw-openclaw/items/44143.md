## Summary

Keep this PR open. Current main still does not implement a per-channel/account/recipient outbound delivery serializer: live sends write their own queue entry, claim only that entry id, and then call the core delivery path directly. The PR contains useful reproduction notes, mock E2E results, and focused serializer tests, but it is not obsolete and is not safe to cleanup-close as implemented. It also needs maintainer/contributor rework before merge because the diff serializes after queue I/O, derives the key from raw target fields, and includes unrelated Mattermost dependency churn.

## Best Possible Solution

Keep this PR open for maintainer/contributor rework. The best path is to rebase on current main, remove or split the unrelated Mattermost dependency change, place serialization at the correct boundary before queue I/O can invert same-target calls, derive an unambiguous key from normalized account/recipient/thread identity, preserve abort/cancellation behavior while waiting, prevent hung sends from wedging a destination queue indefinitely, and add regression coverage for ordering, normalized-key equivalence, thread separation, cancellation, failure cleanup, and queue/recovery interaction.

## Risks / Open Questions

- Closing this PR as implemented would be incorrect because current main still lacks a per-destination outbound serializer.
- Merging the PR as-is may still fail true FIFO-by-call-initiation because the proposed lock is acquired after enqueueDelivery, so queue I/O latency can invert same-target sends before they enter the serializer.
- The proposed serializer key uses raw channel/account/recipient fields; current code normalizes account ids, Signal strips signal: prefixes, and Discord can route by threadId, so equivalent real destinations can bypass the lock while distinct threaded destinations can be unnecessarily blocked.
- A per-recipient serializer can introduce head-of-line blocking if a channel send hangs; current Discord webhook sending awaits fetch without an explicit timeout or abort signal.
- The unrelated Mattermost @sinclair/typebox dependency and lockfile addition broadens package/supply-chain surface outside the stated outbound ordering fix.
