## Summary

Keep this issue open. Current main still has the reported macOS `screen.snapshot` behavior, and the paired fix PR is open and unmerged, so this should stay tracked until that PR or an equivalent fix lands.

## What This Changes

The issue asks for macOS `screen.snapshot` to reject malformed `paramsJSON` with `INVALID_REQUEST` and reject oversized snapshot payloads before base64 JSON emission.

## Best Possible Solution

Review and land #68186, or apply an equivalent macOS runtime fix that preserves defaults for omitted params, rejects malformed params before capture, enforces the gateway payload budget before returning inline base64, and adds focused `MacNodeRuntimeTests` coverage.
