## Summary

Keep open. Current main still has the provider-auth default-model clobber path for real provider patches: configure opts into preserving the existing primary, but provider `configPatch` output is merged before preservation reads the prior primary, and xAI/OpenRouter still emit patches that can already contain the new primary. Merged #70793 established the intended preserve-vs-explicit-default contract, and the Keep My Claw comment is useful recovery context, but neither removes this remaining OpenClaw source behavior.

## What This Changes

The issue requests that provider onboarding/configuration preserve an existing working `agents.defaults.model.primary` unless the user explicitly opts into changing the default model.

## Best Possible Solution

Carry the pre-auth primary model through the provider-auth application path and use that original value for preservation decisions before merging or evaluating provider-owned patches. Only replace the primary when there was no prior primary or the caller explicitly requested a default-model change, and lock the behavior with real xAI/OpenRouter-style provider-patch coverage while keeping explicit model-setting commands unchanged.
