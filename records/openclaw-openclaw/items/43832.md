## Summary

Keep this PR open. Current main still does not implement the PR's intended graceful cron drain: hot reload stops the old cron service synchronously, `CronService` has no `stopGraceful()` API, and cron state does not track a stopping flag or in-flight timer/manual run promises. Main has related force-reload-before-persist protections, but those are a partial stale-write mitigation rather than the stronger product behavior this PR proposes.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refresh it against current main, preserve the timer and manual-run drain coverage, add a bounded drain or explicit fallback for no-timeout runs, update the cron service contract and gateway reload tests, and then use the graceful stop path in `src/gateway/server-reload-handlers.ts` if maintainers want the stronger guarantee. If maintainers decide the current force-reload merge behavior is the intended product fix instead, close the PR only after documenting that decision against the linked cron state-overwrite report.
