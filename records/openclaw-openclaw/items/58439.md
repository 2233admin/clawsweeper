## Summary

Keep PR #58439 open. Current main still has the Mattermost `handleAction` bug: the shared outbound path computes and dispatches `mediaLocalRoots`, and `sendMessageMattermost` consumes them, but Mattermost's action handler does not destructure or forward them on `action: "send"`. The PR is a focused fix for a real remaining behavior gap, not superseded by related PR #67796, which covers separate `filePath`/`path` media fallback handling.

## Best Possible Solution

Keep #58439 open and land it, or an equivalent maintainer patch, by destructuring `mediaLocalRoots` in `extensions/mattermost/src/channel.ts`, passing it to `sendMessageMattermost`, and adding the proposed `handleAction` regression test. Treat #67796 as a separate Mattermost media parameter fallback improvement.
