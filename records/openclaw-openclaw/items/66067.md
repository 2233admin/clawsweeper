## Summary

Keep this PR open: current main still has the reported TUI thinking-fragment spacing bug, and the PR is a focused implementation candidate, but its new normalizer mishandles CRLF/bare-CR thinking text and needs a narrow fix before merge.

## What This Changes

The PR adds a TUI thinking-whitespace normalizer, applies it to thinking extraction paths, and adds formatter plus stream-assembler regression tests for readable thinking fragments.

## Best Possible Solution

Keep the change scoped to TUI thinking text: normalize all newline variants to LF before collapsing single thinking-fragment boundaries to spaces, preserve blank-line paragraph breaks, add CRLF/bare-CR regression coverage, and leave assistant answer text plus non-thinking block joining unchanged.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize CRLF before collapsing thinking newlines:** `src/tui/tui-formatters.ts:250`
  - body: The new helper replaces only `\n`. For CRLF or bare-CR thinking text, this can leave raw `\r` in rendered terminal output and can turn CRLF paragraph breaks into a malformed single paragraph. Normalize `\r\n?` to `\n` before collapsing single newlines, and cover that case in the formatter tests.
  - confidence: 0.9
