## Summary

Keep this PR open. Current main still has the reported TUI thinking-fragment whitespace behavior, and the PR targets the right formatter and stream-assembler path, but the proposed LF-only normalizer should not merge until it canonicalizes CRLF and bare-CR input and adds that regression coverage.

## What This Changes

The PR adds a TUI thinking-whitespace normalizer that converts single-newline thinking boundaries to spaces, applies it in formatter extraction paths, and adds formatter plus stream-assembler regression tests.

## Best Possible Solution

Keep the PR focused on TUI thinking text only: normalize all newline variants to LF before collapsing single thinking-fragment boundaries to spaces, preserve blank-line paragraph breaks, add CRLF and bare-CR regression coverage, and leave assistant answer text plus non-thinking block joining unchanged.
