## Summary

Keep open. Current main still rotates stale daily/idle sessions by archiving the prior transcript and firing plugin lifecycle hooks without running the compaction memory-flush path; session-memory remains command-only, and reset config still lacks off/onReset controls.

## Best Possible Solution

Keep this issue open. The best product/code path is to handle stale daily/idle rollover in core session reset logic before transcript archival: run the existing agents.defaults.compaction.memoryFlush silent turn or an accepted reset-time equivalent, add explicit reset behavior controls such as onReset: "flush" | "archive-only" and an off/manual-only mode if maintainers approve, bridge or emit the needed internal/plugin hook events consistently, and add focused daily/idle/manual reset tests plus schema/docs updates.
