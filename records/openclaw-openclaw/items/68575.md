## Summary

Keep open. Current main has related context-overflow recovery, including embedded-error session reset and compaction-failure reset copy, but it does not already implement the PR’s thrown non-compaction context-overflow reset. The PR also has an unresolved P1 review concern: the new thrown-error reset is driven by broad string classification and can reset a session for mixed fallback failures where context overflow is only one of several terminal causes.

## What This Changes

The PR adds `resetSessionAfterCompactionFailure` handling and regression tests for thrown and final embedded context-overflow paths in `runAgentTurnWithFallback`.

## Best Possible Solution

Keep the source PR as useful work, but update it so session reset happens only for pure context-overflow exhaustion, uses the shared recovery-copy helper, and has tests that cover both the safe reset case and the mixed-fallback no-reset case before landing against #68494.
