## Summary

Current main still has the reported Control UI behavior: `resolveChannelOrder` returns `channelMeta` or `channelOrder` exactly when present, and the backend still builds those arrays from loaded channel plugins. A custom-only channel snapshot can therefore still hide unconfigured built-in channel cards. Open PR #44165 directly targets this issue with closing syntax, so this issue should remain open until that PR or an equivalent fix merges.

## Best Possible Solution

Keep this issue open until PR #44165 or an equivalent focused Control UI fix merges. The fix should update `ui/src/ui/views/channels.ts` so snapshot-provided custom/loaded ids are merged with the built-in channel order, deduped while preserving custom ids first, and still promoted by enabled state. Add regression coverage in `ui/src/ui/views/channels.test.ts` for custom-only `channelMeta`, custom-only `channelOrder`, and no-snapshot fallback. Backend `channels.status` behavior can remain unchanged unless maintainers prefer a catalog-backed built-in order.
