## Summary

Current main still has the reported Control UI behavior: the Channels view treats `channelMeta` or `channelOrder` from `channels.status` as the complete render list, while the backend status payload is built from loaded channel plugins only. The issue should remain open because PR #44165 is an open, unmerged implementation candidate that specifically addresses this issue.

## What This Changes

The issue requests that the Control UI Channels settings page render built-in channel cards even when the gateway snapshot only lists custom or loaded channel plugins.

## Best Possible Solution

Keep this issue open and review/land PR #44165, or an equivalent Control UI fix, so `resolveChannelOrder()` merges snapshot-provided custom/loaded channel IDs with the built-in channel order, dedupes while preserving snapshot order first, keeps enabled-channel promotion, and adds regression coverage for custom-only `channelMeta`, custom-only `channelOrder`, and no-snapshot fallback.
