## Summary

Keep this open. The report is recent, concrete, and tied to the core macOS WebChat UI; current main and the latest tagged release still use the same native SwiftUI/AppKit WebChat surface with transparent windows, active visual-effect material, material/gradient backgrounds, shadows, and visible scroll animations, with no source, docs, tests, changelog, or release evidence that the WindowServer compositor CPU issue has been reproduced or fixed.

## What This Changes

Investigate and fix very high WindowServer CPU when the macOS OpenClaw chat interface is visible on an Intel Retina iMac.

## Best Possible Solution

Keep this as a macOS WebChat rendering performance investigation: profile current native WebChat on affected Intel Retina hardware, compare opaque/no-material/no-shadow/no-scroll-animation variants, verify Reduce Motion behavior, and add a measured low-compositor fallback for older Intel or high-resolution displays if the regression is confirmed.
