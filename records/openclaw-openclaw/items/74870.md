## Summary

Keep this PR open. The Homebrew skip-and-hint direction is relevant to the linked auto-update report and current main does not already implement it, but the proposed branch has a concrete notification-state bug for the supported `update.checkOnStart: false` auto-update mode.

## What This Changes

The PR adds Homebrew install detection, skips the gateway auto-update child for Homebrew-managed package roots with a `brew upgrade openclaw` log hint, adds focused infra tests, and records the change in the changelog.

## Best Possible Solution

Keep the branch focused on explicit Homebrew unsupported-auto-apply feedback, but persist the Homebrew skip notification state independent of startup hints and add a regression test for `update.checkOnStart: false` before merge.

## Reproduction Assessment

Yes. The current tests show `checkOnStart: false` with auto-update enabled is supported; applying the PR logic to that mode logs the Homebrew skip hint and writes state without updating `lastNotifiedVersion` or `lastNotifiedTag`, so the same hint repeats on later checks for the same version/tag.

## Solution Assessment

No as proposed. The skip-and-hint approach is a narrow maintainable fix, but the notification-state update should happen inside the Homebrew branch whenever it logs the hint, not only in the separate startup-hint path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Persist the Homebrew skip notification:** `src/infra/update-startup.ts:430-440`
  - body: When `update.checkOnStart` is false, the earlier update-hint branch is skipped, so `nextState.lastNotifiedVersion` and `lastNotifiedTag` are never updated. This new branch still logs the Homebrew skip hint and immediately writes `nextState`, so every later auto-update check for the same version/tag emits the same hint instead of the promised one-shot notification. Set the last-notified fields when this hint is logged and cover the `checkOnStart: false` case.
