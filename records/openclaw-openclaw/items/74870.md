## Summary

Keep this PR open. It targets a real Homebrew auto-update gap from #74754, but the proposed Homebrew skip branch has a blocking correctness issue: it can log the claimed one-shot hint without persisting the notified version/tag when startup hints are disabled.

## What This Changes

The PR adds Homebrew-managed package-root detection, skips the gateway auto-update child for those installs with a brew-upgrade log hint, adds infra tests, and updates CHANGELOG.md.

## Best Possible Solution

Keep the PR focused on Homebrew-managed install feedback, but amend the new skip branch so any logged Homebrew hint also records lastNotifiedVersion and lastNotifiedTag, with regression coverage for update.checkOnStart: false.

## Reproduction Assessment

Yes. Set update.checkOnStart to false, enable auto-update, and use a Homebrew package root on the PR branch; the new branch can log the skip hint and write state without recording the version/tag as notified.

## Solution Assessment

No as proposed. Skipping the doomed npm auto-apply and telling users to run brew upgrade is a maintainable fix, but the implementation must persist notification state inside that new branch before merge.
