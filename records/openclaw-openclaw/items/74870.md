## Summary

Keep open: current main still lacks Homebrew-specific auto-update handling, and the linked bug remains credible, but this branch should not merge until the skip hint persists its notification state when startup hints are disabled.

## What This Changes

The PR adds Homebrew package-root detection, skips the gateway auto-update child for Homebrew-managed installs with a `brew upgrade openclaw` log hint, adds infra tests, and updates the changelog.

## Best Possible Solution

Keep the PR focused on Homebrew-aware auto-update feedback, but persist `lastNotifiedVersion` and `lastNotifiedTag` whenever the Homebrew skip hint is logged, and add regression coverage for `update.checkOnStart: false`.

## Reproduction Assessment

Yes. Current main has a focused test proving auto-update still runs when `update.checkOnStart` is false; in the PR branch, that mode can log the Homebrew skip hint and write state without recording the version/tag as notified.

## Solution Assessment

No as proposed. The skip-and-hint approach is maintainable if maintainers choose that Homebrew contract, but the branch needs notification-state persistence before it is safe to merge.
