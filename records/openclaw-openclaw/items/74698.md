## Summary

Keep open. Current main still matches the reported Telegram group `/status` no-reply path, and the linked implementation candidate PR #74894 is still separate from `main`, so this issue should remain open until that PR lands or is superseded.

## What This Changes

Telegram group slash commands such as `/status` should produce a visible Gateway status reply in configured group sessions where normal text messages already route successfully.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent narrow Telegram text-command gate fix, so configured group control commands reply in the originating group/topic while the documented group sender authorization boundary remains fail-closed.

## Reproduction Assessment

Yes. Current main has a high-confidence code-level reproduction: a Telegram group where normal text passes under per-group open policy but the effective command allowlist has no entries causes `/status` to be denied and dropped before a reply is sent.

## Solution Assessment

Yes. A focused Telegram message-body command gate change is the narrowest maintainable solution; changing the shared command-gating contract globally would risk broadening command access for other channels.
