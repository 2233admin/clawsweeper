## Summary

Keep open. Current main still has the Telegram text-command gate that can drop a whole-message group `/status` before the normal reply path when the effective group command allowlist is empty, and the linked closing PR #74894 remains open and unmerged.

## What This Changes

The issue requests restoring visible Telegram group `/status` replies when normal group messages already route successfully.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram fix, so configured group `/status` commands reply in the originating group or topic while sender allowlists, owner-only commands, and disabled group/topic policies still fail closed.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a Telegram group that passes normal group-message access but has no effective command allowlist entries; `/status` then reaches the current control-command gate with `allowForCommands.hasEntries === false` and is dropped before a visible reply.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Telegram message-level command gate, not in core command semantics or native command auth, because the failure is the pre-dispatch group gate blocking commands before the broader command handler can decide authorization.
