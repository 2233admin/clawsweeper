## Summary

Keep open. Current main still contains a Telegram text-command gate that can drop a whole-message `/status` in an otherwise working configured group when no effective group command allowlist is present, and PR #74894 is still the provided open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue reports that Telegram group `/status` messages on OpenClaw 2026.4.27 get no visible reply while normal Telegram group messages still reach and receive agent replies.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram patch, so configured open group/topic commands reply in the originating Telegram conversation while explicit sender allowlists, disabled groups/topics, `commands.allowFrom`, and owner-only command checks continue to fail closed.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a configured Telegram group where normal messages pass with no effective group sender allowlist, then a standalone `/status` hits the current `resolveControlCommandGate` path and is dropped before reply delivery.

## Solution Assessment

Yes. Keeping this issue open until PR #74894 or an equivalent narrow Telegram fix lands is the right path; closing before the linked implementation merges would lose the active regression tracker.
