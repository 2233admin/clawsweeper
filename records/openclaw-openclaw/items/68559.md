## Summary

Current main still does not implement this PR's stated memory-host SDK behavior. Both session-file discovery copies still index usage-counted transcript files, current tests still expect reset/deleted transcript archive variants to be listed, and the changelog documents the opposite QMD recall behavior under the 2026.4.5 release section. The PR remains a plausible product/quality change, but it is not obsolete and needs normal maintainer review, especially because the current head includes broader plugin, gateway, browser, doctor, channel, and Signal-related changes beyond the title.

## Best Possible Solution

Keep this PR open for normal maintainer review. If maintainers accept primary-transcript-only memory indexing, land a scoped version that updates both `src/memory-host-sdk` and `packages/memory-host-sdk`, preserves legitimate dotted session IDs while excluding real checkpoint artifacts, updates tests, and splits or clearly justifies the unrelated plugin, gateway, browser, doctor, channel, and Signal changes. If maintainers want the documented reset/deleted QMD recall behavior to remain, close or revise the PR on that explicit product basis.
