## Summary

Keep this PR open. Current main does not already implement the PR’s intended change: the memory-host-sdk session listing still uses the usage-counted transcript helper and the current test suite explicitly expects reset/deleted transcript variants to be included. The PR is focused, but the live review comments identify real follow-up work: the proposed broad `.checkpoint.` substring guard can exclude legitimate dotted session ids, and the test should directly assert reset/deleted exclusion if that product behavior is desired.

## Best Possible Solution

Keep this PR open for maintainer/product review rather than closing it as obsolete. If OpenClaw wants QMD memory indexing to ignore reset/deleted archives, make that policy explicit in the memory listing path, use exact artifact helpers for compaction checkpoints instead of a broad substring match, add tests for reset/deleted exclusion, exact checkpoint exclusion, and valid dotted session ids, and keep the source/package memory-host-sdk surfaces in sync.
