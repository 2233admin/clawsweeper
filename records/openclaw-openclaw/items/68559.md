## Summary

Current main still does not implement this PR's stated memory-host SDK behavior. The source and package copies still list usage-counted transcripts, the tests still expect reset/deleted transcript archives to be included, and the shipped changelog documents that QMD session export intentionally keeps reset/deleted transcripts for historical recall. The PR remains a plausible product change, but it is not obsolete and needs normal maintainer review, especially because its current diff spans memory, plugin loader/discovery, gateway, browser, doctor, channel, and Signal installer surfaces.

## Best Possible Solution

Keep this PR open for normal maintainer review. If maintainers accept primary-transcript-only memory indexing, land a scoped version that updates both `src/memory-host-sdk` and `packages/memory-host-sdk`, keeps exact checkpoint artifact filtering without dropping legitimate dotted session IDs, updates tests and changelog/docs as needed, and splits or clearly justifies unrelated plugin/gateway/browser/Signal changes. If maintainers want the shipped reset/deleted QMD recall behavior to remain, close or revise the PR on that explicit product basis.
