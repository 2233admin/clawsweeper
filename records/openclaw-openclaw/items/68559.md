## Summary

Keep this PR open. Current main still indexes usage-counted session transcript files, including reset and deleted archive variants, while this PR proposes a real behavior change to index only primary transcripts. The latest PR context also shows the author addressed earlier reset/deleted and broad-checkpoint test concerns, but the branch still needs normal maintainer review because it bundles unrelated plugin loader, doctor, gateway, browser test-isolation, and bundled-plugin trust changes.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept primary-transcript-only memory indexing, land a scoped version that updates both `src/memory-host-sdk` and `packages/memory-host-sdk`, keeps exact compaction-checkpoint filename filtering, preserves the added reset/deleted regression coverage, and splits or explicitly justifies the unrelated plugin loader, doctor, gateway, browser, and bundled-plugin trust changes. If maintainers prefer shipped recall behavior that includes reset/deleted usage-counted transcripts, close or revise the PR on that product basis.
