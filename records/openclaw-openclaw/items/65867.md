## Summary

Keep this issue open. The issue has an open contributor PR, #66225, that explicitly closes it, and current main still contains the narrow `<final>` regex in the streaming/sanitizer paths that the PR is trying to align. Recent issue discussion also reports the leak still reproduces on v2026.4.26, while the companion persistence PR #69685 was closed unmerged.

## Best Possible Solution

Keep the issue open. Maintainers should review #66225 against current main, decide whether the self-closing regex alignment is still needed, and separately decide whether the Control UI persistence/replay behavior from #69685 needs a new implementation. After the relevant PR lands and current main or a release verifies the Control UI no longer leaks the tags, the issue can close as implemented.
