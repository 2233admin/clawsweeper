## Summary

Keep open. Current main has implemented the snapshot path for normal/deep status scans and setup compatibility prompts, but the human `openclaw status --all` report still calls the diagnostics-backed compatibility path. This PR also carries unrelated setup, changelog, and CI/test-runner changes, so the best maintainer path is a narrow rework rather than closing it as already implemented.

## Best Possible Solution

Rework this as a narrow PR against current main: either make `status --all` use a compatibility snapshot path that preserves documented hook-only and `before_agent_start` notices, or add metadata-backed compatibility notices before switching the call-site. Keep setup as it is on main, and remove the unrelated changelog, setup migration/config-write, provider runtime, timeout, and CI/test-runner changes from this PR.
