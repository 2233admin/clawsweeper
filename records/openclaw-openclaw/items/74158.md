## Risks / Open Questions

- The PR adds user-facing config fields but does not update docs/concepts/active-memory.md or CHANGELOG.md; OpenClaw policy expects behavior/config docs and a user-facing changelog entry for this kind of change.
- The proposed breaker is module-level and keyed by agent/provider/model, so one slow conversation can suppress Active Memory recall for other conversations using the same agent/model during the cooldown. That may be intended, but it needs maintainer agreement.
- Client-side timeout/circuit breaking can reduce repeated attempts, but it cannot guarantee upstream providers stop billing tokens after accepting a request. Docs and logs should avoid implying token refunds or cancellation guarantees.
