## Summary

Keep this PR open. Current main still lacks the ACP token-only/device-identity opt-out the PR is trying to add: `openclaw acp` does not register a `--no-device-identity` or `--skip-device-identity` option, `serveAcpGateway()` constructs `GatewayClient` without `deviceIdentity: null`, and the ACP docs list no such option. The linked feature request is closed, but this open PR remains the implementation candidate rather than something already shipped.

## Best Possible Solution

Keep the PR open and either finish it as the ACP implementation candidate or replace it with an equivalent maintainer patch. The implementation should add a documented ACP CLI flag, pass `deviceIdentity: null` only when explicitly requested, preserve the existing `undefined` auto-load behavior, and include focused startup tests for both branches. Because the linked feature issue is already closed, landing or closing this PR is the right place to resolve the remaining work.
