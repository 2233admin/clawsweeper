## Summary

Keep open. Current main still does not expose an ACP device-identity suppression flag, and this PR is a focused implementation of the token-only ACP auth path. It should not be closed as implemented or duplicate, but it still needs the public docs gap fixed and gateway-auth owner review before merge.

## What This Changes

The PR adds a `noDeviceIdentity` ACP server option, exposes `openclaw acp --skip-device-identity`, passes `deviceIdentity: null` to `GatewayClient` when set, and adds startup tests for null versus undefined behavior.

## Best Possible Solution

Finalize the public flag name, add the ACP CLI reference entry, keep the server/Commander/standalone parser behavior aligned, preserve the null-vs-undefined tests, and merge this PR or an equivalent maintainer patch after gateway-auth review.
