## Summary

Keep open: the patch is useful but not merge-ready because Discord still pre-resolves group delivery with only `ChatType` and passes `message_tool_only` as an explicit request, so the new `WasMentioned` resolver branch would not make mentioned Discord guild replies visible. It also changes the documented group/channel visible-reply contract without docs or changelog updates.

## What This Changes

The PR adds `WasMentioned` to the source reply delivery context, makes mentioned group/channel resolver calls return `automatic`, and adds resolver/visibility policy unit tests.

## Best Possible Solution

Decide the public contract first. If mentions should become a visible-reply exception, implement it consistently at channel pre-resolution and dispatch boundaries, update Discord/Slack/core coverage, and document the exception with a changelog entry; otherwise rework or close the PR against the current documented tool-only behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: current main has a Discord process test for `effectiveWasMentioned: true` that still expects `message_tool_only`, and the source path shows that explicit requested mode suppresses automatic final delivery.

## Solution Assessment

Is this the best way to solve the issue? No: the resolver-only patch is too narrow because some channels pre-resolve and pass an explicit mode before dispatch, and the user-facing contract needs docs/changelog alignment if the behavior is accepted.
