## Summary

Keep open. The reported behavior is plausible and related history supports it, but this patch does not fix the main Discord guild path: Discord pre-resolves `message_tool_only` before dispatch, and the resolver still returns requested modes before any `WasMentioned` check. The user-facing behavior change also needs docs and changelog coverage if maintainers accept the @mention exception.

## What This Changes

The PR adds `WasMentioned` to source reply delivery mode resolution and unit tests so directly mentioned group/channel turns resolve to automatic delivery.

## Best Possible Solution

Implement the @mention exception end-to-end only if it is the accepted group/channel contract: carry mention state through channel pre-resolution and dispatch, preserve tool-only behavior for non-mentioned room traffic, and update docs, changelog, and Discord/Slack/core regression coverage together.

## Reproduction Assessment

Yes. The current Discord process test models a guild turn with `effectiveWasMentioned: true` and still expects `message_tool_only`; dispatch then suppresses normal final replies for that mode.

## Solution Assessment

No. The PR changes the core resolver but misses callers that pre-resolve and pass an explicit delivery mode, so at least the Discord guild path remains broken; a safer fix must include the channel pre-resolution seams and public docs.
