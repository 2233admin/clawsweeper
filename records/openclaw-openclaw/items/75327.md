## Summary

Keep open: the PR targets a plausible group/channel mention-delivery problem, but the current patch is not mergeable because at least the Discord guild path pre-resolves `message_tool_only` before dispatch and passes it as an explicit requested mode, bypassing the new resolver branch. The change also needs maintainer product approval because current docs intentionally separate mention triggering from visible final-reply delivery.

## What This Changes

The PR adds `WasMentioned`-aware automatic group/channel reply delivery in the source reply mode resolver and adds resolver/visibility-policy unit coverage.

## Best Possible Solution

Maintainers should first decide the intended visible-reply contract for @mentions. If accepted, the implementation should carry mention state into channel pre-resolution and dispatch consistently, update Discord and Slack coverage, and document the exception with a changelog entry.

## Reproduction Assessment

Yes. Current main has a Discord process test with `effectiveWasMentioned: true` that still expects `message_tool_only`, and the source path shows Discord pre-resolves delivery without `WasMentioned` before passing the explicit mode into dispatch.

## Solution Assessment

No. A resolver-only override is not the narrowest maintainable fix because explicit requested modes bypass it; the safer fix is to propagate mention state at the channel pre-resolution seam or avoid pre-resolving when dispatch has richer context.
