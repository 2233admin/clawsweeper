## Summary

Keep this PR open. Current main still has the narrow final-tag regexes in the streaming subscriber and user-facing sanitizer, so the self-closing `<final/>` leak described by this PR and linked #65867 is not implemented on main or in v2026.4.26. The PR remains a relevant implementation candidate, but it still needs maintainer review around strict-mode parsing and overlap with #48593.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this fix or an equivalent shared final-tag parser that strips `<final/>`, `<final />`, and ordinary final tags in both streaming and user-facing sanitizer paths, while preserving strict-mode fail-closed behavior for malformed tags. Add focused regression coverage for sanitizer stripping, streaming with enforcement off, streaming with enforcement on, slash-containing attributes, and malformed attribute text, then close #65867 and dedupe overlapping #48593 as appropriate.
