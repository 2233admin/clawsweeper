## Summary

Keep #57256 open. Current main still contains the reported false-negative paths: the status overview renders `enabled (plugin <slot>) · unavailable` whenever the local memory snapshot is null, the snapshot resolver can return null before asking the active plugin manager, and the gateway memory probe still treats `doctor.memory.status.embedding.ok` as the readiness signal. The supplied discussion and related PR context show #59685/#60777 target this exact bug, but those fixes are not present in the checked-out main SHA.

## Best Possible Solution

Keep the issue open until #59685, #60777, or an equivalent scoped fix lands on main. The best fix should make `openclaw status --deep` and `doctor.memory.status` distinguish live memory runtime health from embedding probe health, reuse the active gateway/plugin runtime where appropriate, preserve mixed-version behavior, and add regression coverage for external memory plugins such as mem0, Honcho, and qdrant-style plugins.
