## Summary

Keep this PR open. Current main still lacks transport-close eviction for cached bundled MCP runtimes, so the PR addresses a real stale-runtime path, but the current PR head still has a grounded functional review concern: its unguarded `transport.onclose` callback can also fire during intentional cleanup, including partial multi-server startup failure. This should be updated and reviewed rather than closed as implemented, duplicate, or out of scope.

## Best Possible Solution

Keep this PR open and update it in place: distinguish unexpected transport closure from intentional cleanup, preserve the child-death eviction regression test, add a multi-server partial-startup regression test for the intentional-close path, and then run the targeted bundled MCP runtime test plus the normal changed validation before maintainer review/landing.
