## Summary

Keep open. Current main still does not implement Control UI math rendering, and the open canonical feature request remains relevant, but this PR is not mergeable as submitted because it removes existing markdown helper definitions, broadens sanitizer behavior, lacks the lockfile update for a new dependency, and has delimiter parsing regressions.

## What This Changes

This PR adds KaTeX to the Control UI shared markdown renderer and tests so `$...$` and `$$...$$` chat content can render as math HTML.

## Best Possible Solution

Rework this PR, or a successor, against current main so math rendering is added in the shared Control UI markdown renderer while preserving cache and DOMPurify link hardening, using robust delimiter rules, narrowing KaTeX-specific style handling, updating the lockfile, and covering inline math, display math, invalid LaTeX, currency prose, and code-span/code-fence cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Restore the removed markdown helper functions:** `ui/src/ui/markdown.ts:83-91`
  - body: This replacement removes `installHooks`, `getCachedMarkdown`, and `setCachedMarkdown`, but `toSanitizedMarkdownHtml` still calls them. The branch will fail to typecheck or render non-empty markdown, and it also drops the existing DOMPurify link-hardening hook and cache behavior.
  - confidence: 0.98
- **[P1] Avoid treating currency ranges as inline math:** `ui/src/ui/markdown.ts:214-217`
  - body: The inline scanner accepts any later `$` as the closing delimiter, so text like `between $5 and $10` is converted into KaTeX instead of staying plain prose. Please add delimiter-boundary rules before enabling this in the default chat renderer.
  - confidence: 0.9
- **[P2] Skip multi-backtick code spans during math extraction:** `ui/src/ui/markdown.ts:177-180`
  - body: The code-span bypass only finds the next single backtick. Markdown spans delimited with two or more backticks can still have `$...$` extracted from inside code, changing literal examples into rendered math.
  - confidence: 0.88
- **[P2] Update the pnpm lockfile with the new dependency:** `ui/package.json:17`
  - body: This adds `katex` to `ui/package.json` but does not update `pnpm-lock.yaml`, so frozen installs will not have a resolved package entry for the new dependency.
  - confidence: 0.94
