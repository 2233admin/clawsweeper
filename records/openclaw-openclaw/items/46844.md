## Summary

Keep open. Current main still lacks the requested between-turn Talk Mode idle timeout: Voice Wake can activate Talk Mode directly, but Talk only uses the short transcript-finalization silence window and then resumes listening after replies or failures. The related heartbeat/session-cleanup reports are broader lifecycle requests and do not replace this Talk-specific wake path.

## What This Changes

The issue requests a configurable between-turn Talk Mode idle timeout after Voice Wake activation so Talk deactivates after a period with no user interaction instead of staying active until the hard stop or manual exit.

## Best Possible Solution

Add an additive Talk lifecycle setting that is validated in config, included in the gateway talk.config payload, documented on the public Talk page, and honored by macOS Talk so idle sessions return to wake listening while silenceTimeoutMs remains scoped to transcript finalization.
