## Summary

Keep open. Current main still lacks a between-turn Talk idle timeout: Voice Wake can activate Talk Mode directly, but the macOS Talk runtime only has the short transcript-finalization silence window and then resumes listening after replies or failures. The related heartbeat/session-cleanup reports are broader lifecycle requests and do not cover this Voice Wake-triggered Talk gap.

## What This Changes

The issue requests a configurable Talk Mode idle timeout that deactivates Voice Wake-triggered Talk sessions after a between-turn period with no user interaction.

## Best Possible Solution

Add an additive Talk lifecycle setting that is validated in config, exposed through `talk.config`, documented on the public Talk docs, and honored by macOS Talk so idle sessions return to wake listening while keeping `silenceTimeoutMs` scoped to transcript-finalization pauses.
