## Summary

Keep this PR open. Current main still rejects gateway `agent` requests with `send blocked by session policy` before checking whether the request asked for external delivery, while the runtime agent command path already enforces the same policy only when `deliver === true`. The PR is a focused implementation candidate for the still-open #73381 bug and includes the missing deny/non-delivery regression coverage.

## Best Possible Solution

Review and land this PR, or an equivalent maintainer patch, by moving the gateway `resolveSendPolicy` deny check under `request.deliver === true` while retaining the existing runtime guard in `agent-command.ts`. Keep regression coverage for both non-delivery allow under deny policy and `deliver: true` rejection, then let #73381 close after the fix merges.
