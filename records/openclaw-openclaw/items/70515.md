## Summary

Keep open. Current main still has the backup-restore false-success behavior in both async and sync config suspicious-read recovery: copy failures are swallowed, the warning still says the config was restored, and the observe audit record is written with valid: true. This PR remains a focused implementation candidate and is more complete than overlapping #62665 because it adds audit error fields plus async and sync regression tests. Security review found the PR limited to config runtime/audit code and tests, with no workflow, dependency, lockfile, script, or package publishing changes.

## Best Possible Solution

Keep this PR open for maintainer review as the likely canonical fix, or explicitly choose #62665 and port the audit-field plus async/sync regression-test coverage before closing this PR. The final merged fix should make failed async and sync backup copies log a failure, mark the observe audit record invalid, and persist structured restore error details without broadening unrelated recovery behavior.
