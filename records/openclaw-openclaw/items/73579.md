## Summary

Keep open. Current main still has the behaviors this PR is trying to change: suppressed group/channel delivery returns before trusted TTS voice media can be sent, OpenAI-compatible PCM TTS is returned as raw `.pcm`, and WhatsApp voice-note sends still duplicate visible text. The PR is a focused implementation candidate with security-sensitive delivery-policy changes that need maintainer review, not cleanup closure.

## Best Possible Solution

Continue reviewing this PR as the implementation candidate. The best path is to harden and validate the `trustedLocalMedia`-gated voice-media exception, keep ordinary text and untrusted media suppressed in `message_tool_only` groups, stop duplicate WhatsApp text sends for voice notes, wrap declared PCM into a deliverable container with observable mismatch handling, preserve the targeted regression tests, and close only after this PR merges or a newer canonical fix supersedes it.
