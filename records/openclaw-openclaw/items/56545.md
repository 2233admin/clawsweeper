## Summary

Keep this PR open. The intended `failureAlert.mode: "agent-turn"` behavior is still not implemented on current main: cron config, gateway protocol validation, CLI parsing, delivery planning, and the failure-alert sender still only accept `announce` or `webhook`. The PR is a plausible implementation candidate, but the review thread also flags real follow-up needed around protocol/schema sync, CLI acceptance, failureDestination semantics, targeted heartbeat wakeups, and system-event trust handling.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a complete implementation that adds `agent-turn` consistently across cron config types, Zod config, gateway protocol schema, CLI parsing/help, runtime failure-alert dispatch, and tests/docs, while addressing the review comments about targeted heartbeat wakeups and failureDestination semantics.
