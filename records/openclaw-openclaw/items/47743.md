## Summary

Keep open: current main still lacks an explicit trust-tier model, and this PR is not ready because it computes the tier from raw manifest format while the loader normalizes missing formats to `openclaw`, which can under-report native-capable plugins as `content`. The change is security-facing public CLI/docs metadata, so it needs a branch fix plus maintainer product/security review.

## What This Changes

This PR adds a plugin trust-tier type and resolver, stores `trustTier` on plugin records, prints it in plugin CLI output, and adds docs/tests for `content`, `sandboxed`, and `native` tiers.

## Best Possible Solution

If maintainers want this taxonomy, land a rebased version that computes tiers from the normalized effective plugin format, defines the supported public CLI/JSON/status contract, documents only current capabilities plus clearly reserved future behavior, and covers native, bundle, missing-format fallback, list/info output, and inspect/status behavior in tests.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Resolve trust tiers from the normalized plugin format:** `src/plugins/loader.ts:849-853`
  - body: The loader stores missing formats as `openclaw` via `createPluginRecord`, but this new call passes raw `manifestRecord.format` into `resolveTrustTier`.
