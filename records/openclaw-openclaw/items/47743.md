## Summary

Keep open: current main does not implement plugin trust tiers, and the PR still targets a real plugin security/CLI metadata gap, but the proposed patch is not merge-ready because it can label omitted-format native plugins as content-tier while the loader records and executes them as native-format plugins.

## What This Changes

This PR adds plugin trust-tier types and resolution, stores the tier on plugin records, shows it in plugin CLI output, and documents content, sandboxed, and native tiers.

## Best Possible Solution

Land a rebased version only if maintainers want this public taxonomy, with trust tiers computed from the same normalized effective plugin format used in PluginRecord, docs that distinguish current capabilities from reserved future behavior, and tests covering native, bundle, omitted-format fallback, loader record paths, and CLI list/info output.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Resolve trust tiers from the normalized plugin format:** `src/plugins/loader.ts:849-853`
