## Summary

Current main at a820a307dfeb950fa633c0ae6fb2386a35a91991 still does not implement a persistent `agents.defaults` usage-footer default. The config schema/types lack `usageDefault`, `responseUsageDefault`, or `responseUsage` under `agents.defaults`; reply formatting reads only session `responseUsage`; `/usage` and gateway patching still mutate or clear the session-local field; and docs still describe `/usage` as per-session. The issue also has open implementation candidates, including #69618 with closing syntax, so it should remain open until an accepted PR lands or is closed.

## Best Possible Solution

Keep this issue open. The maintainer path is to either review and land an accepted implementation candidate such as #69618 or #48635, or make a smaller core config change that adds one public `agents.defaults` key, updates schema/types/generated help/docs, resolves session override before inherited default, distinguishes explicit `off` from clear-to-inherit, and adds regression coverage for inherited default, explicit off, clearing to inherit, and per-session override behavior.
