## Summary

Current main still does not implement a persistent usage-footer default. The agent defaults schema/types have comparable defaults but no `usageDefault`, `responseUsageDefault`, or `responseUsage` default; reply formatting reads only session-local `responseUsage`; `/usage` and Gateway patching still mutate/delete the session field; and docs still describe `/usage` as per-session. An open PR with closing syntax (#69618) and an earlier open candidate (#48635) are already tracking implementation work, so this issue should remain open until an accepted PR lands or is closed.

## Best Possible Solution

Keep this issue open. The best path is to review and either land or close the active implementation candidate #69618, or choose a smaller core change that adds one stable `agents.defaults` key, updates schema/types/generated help/docs, resolves inherited defaults in the reply runner, preserves explicit per-session overrides, treats clear-to-inherit distinctly, and adds focused regression tests for inherited default, explicit off, clearing, and per-session override behavior.
