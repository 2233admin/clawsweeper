## Summary

Current main still has the reported Mattermost preview-streaming mismatch: docs advertise `channels.mattermost.streaming` modes, but Mattermost schema/types/metadata omit that key and the inbound runtime still creates a draft preview without resolving `off`, `block`, or `progress`. The visual-spam portion is partly improved by same-post draft finalization, but the documented config and notification-mode behavior remain unresolved.

## What This Changes

The issue requests that Mattermost implement the documented `channels.mattermost.streaming` modes and fix preview/final notification UX for draft streaming.

## Best Possible Solution

Keep this issue open as the focused tracker for the Mattermost preview-streaming contract. Implement `channels.mattermost.streaming` in the bundled Mattermost plugin end to end: schema/types/generated metadata, config/doctor compatibility, account resolution, runtime handling for `off | partial | block | progress`, and focused config/runtime tests, with `progress` delivering the final answer through a notifying final post.
