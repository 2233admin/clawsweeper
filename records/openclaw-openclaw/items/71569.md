## Summary

Keep this open. Current main still has the Mattermost streaming contract mismatch: docs advertise `channels.mattermost.streaming` with `off | partial | block | progress`, but the strict Mattermost config schema/type surface omits `streaming`, generated config metadata still only exposes legacy `blockStreaming` fields for Mattermost, and inbound runtime still unconditionally creates draft previews and disables generic block streaming. Open PR #73252 is the active related fix candidate, so this should stay open for maintainer review instead of spawning duplicate cleanup work.

## What This Changes

The issue requests Mattermost support for the documented `channels.mattermost.streaming` preview modes and a fix for draft-preview notification/visual-noise behavior when final answers are delivered as edits.

## Best Possible Solution

Keep this as the Mattermost streaming-contract tracker until PR #73252 or an equivalent Mattermost-local fix lands. The end state should accept documented `streaming` config at root and account scope, preserve legacy block-streaming compatibility, resolve `off | partial | block | progress`, skip draft-preview callbacks/finalization for `off`, define behavior for the other modes, and add focused schema/runtime/docs coverage.

## Reproduction Assessment

Yes for the documented config mismatch: current main’s strict Mattermost schema/type/metadata omit `streaming` while docs advertise it, and inbound runtime still wires draft-preview updates unconditionally. The notification symptom has credible reporter evidence, but no live Mattermost notification reproduction was run in this read-only review.

## Solution Assessment

No current-main solution exists yet. The best visible path is to review, land, or revise PR #73252 as the canonical `streaming.mode` fix, then decide whether final notifications need a narrower follow-up.
