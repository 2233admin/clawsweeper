## Summary

Current main still has the central fixed tool-schema overhead described here. The message tool still builds a broad cross-channel schema, effective tools are still converted into model-facing custom tool definitions with descriptions and parameters, the system prompt still emits a Tooling list, and the cron tool still carries long always-sent guidance; the latest maintainer discussion also keeps this as the canonical tracker rather than superseding it with narrower related items.

## What This Changes

Request to reduce fixed per-session tool schema token overhead through channel-aware schema pruning, lazy or slim tool loading, shorter tool descriptions, and splitting or narrowing high-footprint tools.

## Best Possible Solution

Keep this issue open as the canonical tracker and pursue incremental core changes: remove or make optional duplicated Tooling text, avoid sending tool definitions when the effective tool list is empty, prune high-footprint message fields by active channel and capability without breaking cross-channel cron/isolated use, shorten long tool descriptions, and design a compatible slim or lazy schema mode for plugins and declarative tools.
