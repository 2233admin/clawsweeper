## Summary

Keep this issue open. Current main still has the central fixed overhead described here: message tool schemas remain broad, effective tools are still converted into full custom tool definitions, the system prompt still emits Tooling text, and cron still carries a long always-sent description. Existing allow/deny/profile controls and related partial work are useful mitigations, not a full fix.

## Best Possible Solution

Keep this issue open as the canonical tool-schema overhead tracker. The best path is incremental core work: remove or make optional duplicated Tooling text, avoid schema payloads when the effective tool list is empty, prune message parameters by active channel/capability without breaking cross-channel jobs, shorten always-sent cron guidance, and design a compatible lazy or slim schema mode for plugin and declarative tools. Keep narrower work such as #14619, #52861, #58405, and #58434 linked rather than treating them as superseding this broader tracker.
