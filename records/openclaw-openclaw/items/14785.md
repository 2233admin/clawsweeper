## Summary

Keep this issue open. Current main still has the central fixed tool-schema overhead: broad message parameters remain, full effective tool schemas are still passed to model-facing custom tool definitions, the system prompt still includes a Tooling list, and the cron tool still embeds long tutorial text. Related work is partial or still open, and the latest maintainer review explicitly kept this as the canonical tracker.

## What This Changes

Reduce fixed per-session tool-schema token overhead by pruning channel-specific parameters, avoiding unnecessary tool payloads, lazy-loading or splitting large tools, moving long guidance out of always-sent descriptions, and compressing repeated descriptions.

## Best Possible Solution

Keep this issue open as the canonical tool-schema overhead tracker. The best path is incremental core work: remove or make optional duplicated Tooling text, avoid sending schemas when the effective tool list is empty, prune message parameters by active channel/capability while preserving isolated/cron cross-channel behavior, shorten or move cron tutorial text out of the always-sent description, and design a compatible slim/lazy schema mode for plugin and declarative tools. Keep narrower refs such as #14619, #52861, #58405, and #58434 linked rather than treating them as superseding the broader tracker.
