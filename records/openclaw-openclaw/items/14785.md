## Summary

Keep open. Current main still has the fixed tool-schema and prompt overhead at the center of the report, and the latest discussion treats this as the canonical multi-step performance tracker rather than a completed or superseded item.

## What This Changes

Request to reduce fixed per-session tool schema token overhead through channel-aware pruning, lazy or slim tool loading, shorter descriptions, and splitting high-footprint tools.

## Best Possible Solution

Keep this as the canonical performance tracker and sequence incremental core changes with measurement: remove or make optional duplicated Tooling text, avoid emitting tool definitions when no effective tools remain, prune high-footprint message fields by active channel and capability, shorten long descriptions where behavior remains covered, and design a compatible slim or lazy schema mode for plugins and declarative tools.
