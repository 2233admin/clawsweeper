## Summary

Keep open. Current main has not implemented PR #58796's behavior: manifest registry output still has only `plugins` and `diagnostics`, distinct duplicate plugin ids still emit warn-level diagnostics, and loader lookup is still keyed from the deduped `plugins` rows rather than an all-roots manifest map.

## Best Possible Solution

Keep #58796 open for maintainer review or replacement by an equivalent patch. The best implementation direction is the PR's narrower policy: suppress only benign bundled shadows of non-installed global/workspace copies, preserve warnings for config overrides and explicitly installed globals, and add an all-root manifest index so loader duplicate-precedence rows remain visible after `plugins[]` dedupe, with targeted registry and loader tests.
