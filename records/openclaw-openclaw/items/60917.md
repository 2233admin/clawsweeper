## Summary

Keep open. Current main still has the same failure shape: the Control UI form save serializes the cloned redacted snapshot wholesale, there is no UI-side helper to drop unresolved redaction sentinels absent from the original raw/source config, and the gateway write path still rejects a redacted sentinel at sensitive paths such as `gateway.remote.token` when no original value can be restored. The linked fix PR #60971 was closed unmerged per the provided GitHub context, and the relevant current-main files have not changed since the prior review point.

## Best Possible Solution

Keep this issue open and land a current-main equivalent of #60971: in the Control UI form-mode save path, drop unresolved `__OPENCLAW_REDACTED__` placeholders that were not present in the original raw/source config, preserve existing real redacted secret paths, and add regression coverage for saving unrelated settings when `gateway.remote.token` is absent from the original config.
