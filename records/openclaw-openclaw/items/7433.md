## Repair Work Prompt

Implement the remaining WhatsApp/Baileys group reliability hooks requested by this issue. Current main already has the #73732 reconnect-time group metadata fallback; do not remove it. Add a bounded, account-scoped, in-memory recent-message store suitable for Baileys `getMessage` retry/decryption support, wire `cachedGroupMetadata` into `makeWASocket`, and add inbound event handling for `groups.upsert`, `groups.update`, `group-participants.update`, and `lid-mapping.update` where it fits the existing WhatsApp session/inbound-monitor boundaries. Be careful with privacy: do not persist message contents to disk unless a maintainer explicitly approves it, and avoid adding a stale parallel LID map if Baileys' `sock.signalRepository.lidMapping` already owns persistence. Keep the patch inside the WhatsApp plugin unless a generic seam is demonstrably required. Update or add focused tests for socket options and group/LID event handling. Add a single-line user-facing changelog fix entry if runtime behavior changes, crediting `Thanks @octopuslabs-fl` and any carried-forward related reporters as appropriate.
