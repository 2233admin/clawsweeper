## Summary

Keep open. Current main still routes `/v1/chat/completions` through the normal agent ingress path without an HTTP light-context, prompt-mode, tool/skill suppression, or per-agent HTTP prompt policy, and the only stable-session work linked from the discussion remains an open partial PR.

## What This Changes

The issue asks for low-latency `/v1/chat/completions` support, including HTTP light-context/voice mode, stable agent-scoped sessions, and optional per-agent HTTP system prompt overrides.

## Best Possible Solution

Keep this issue as the umbrella tracker for an explicit OpenAI HTTP prompt policy. The end state should deliberately define how low-latency callers opt into reduced bootstrap context, prompt sections, tools, skills, and memory behavior, while separately landing or rejecting the stable-session/QMD warm-cache work in #69060.
