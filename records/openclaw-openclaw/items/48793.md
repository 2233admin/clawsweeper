## Summary

Keep this issue open. Current main has per-channel async start/stop hooks, abort tracking, runtime contexts, and generic plugin runtime lifecycle cleanup, but it does not implement a centralized PluginResourceManager-style resource ownership API for channel plugins. The Feishu cleanup pattern cited by the issue is still visible on main, PR #48588 remains open and Feishu-specific, and #49804 overlaps the broader lifecycle design rather than proving this item is resolved.

## Best Possible Solution

Keep this open for maintainer design review. The best path is to first decide whether the public plugin SDK should grow an account-scoped async resource registration/lifecycle contract, then migrate one or two high-risk channel resources such as Feishu HTTP servers as contract-test examples. The Feishu-specific leak can still be handled by #48588/#48183, while this issue or #49804 should track the broader cross-plugin lifecycle design.
