## Summary

Keep open. Current main still lacks the requested gateway-owned, account-scoped async resource manager for channel plugin resources; the remaining work is a public plugin SDK/channel lifecycle design decision, not conservative cleanup.

## What This Changes

The issue requests a centralized PluginResourceManager-style interface with async start, stop, and cleanup semantics for channel plugin resources.

## Best Possible Solution

Maintainers should make an explicit API decision: either add a backward-compatible, account-scoped async lifecycle seam to the plugin SDK/gateway with drain, timeout, and forced-dispose semantics, then migrate Feishu plus at least one other high-risk channel with contract tests; or intentionally consolidate this channel-specific request into #49804 after the Feishu-specific cleanup bug is handled.
