## Summary

Keep open. Current main still lacks a gateway-owned, account-scoped async resource manager for channel plugins; the implemented surfaces are per-account start/stop hooks, opt-in lifecycle helpers, synchronous runtime-context leases, and plugin-owned cleanup. The motivating Feishu HTTP server close-before-delete pattern is still present, while the narrow Feishu bug/PR and the broader async resource lifecycle tracker remain open.

## What This Changes

The issue requests a centralized `PluginResourceManager` interface so channel plugins can consistently start, stop, and fully clean up async resources before gateway restart.

## Best Possible Solution

Make an explicit maintainer API decision: either add a backward-compatible account-scoped async resource lifecycle seam to the plugin SDK and prove it by migrating Feishu plus at least one other high-risk channel with contract tests, or consolidate this channel-specific request into #49804 after the Feishu-specific cleanup bug is handled separately.
