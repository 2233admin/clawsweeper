## Summary

Keep open. Current main has a newer, better embedded-agent outcome classifier that covers empty/reasoning-only/planning-only GPT-5 terminal results and falls back when callers pass that classifier, but it does not fully implement this PR's stated change. The generic `runWithModelFallback` path still only treats successful results as failures when `classifyResult` is supplied, there is no `EmptyResponseError`/`isRawProviderResult` equivalent on main, and Telegram's current fallback routing is still generic rather than typed by rate-limit vs empty-response vs other dispatch failure.

## Best Possible Solution

Keep this PR open for maintainer/contributor judgment rather than auto-closing. The best path is to rebase and narrow it: drop the unrelated `.gitignore` changes, adapt Telegram work to `extensions/telegram`, and decide whether raw provider empty-response detection belongs as a generic `runWithModelFallback` classifier contract or caller-owned classifier. If a newer focused PR lands the remaining raw-response and typed-routing behavior, this PR can then be closed as superseded or implemented.
