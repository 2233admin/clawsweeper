## Summary

Keep open. Current main still executes the Bun installer from `https://bun.sh/install` in the root Dockerfile, while CI pins Bun and the Dockerfile pins Node base images by digest. An open closing PR already targets this exact request, so the issue should remain open until that PR or an equivalent maintainer-approved supply-chain hardening change lands.

## What This Changes

Request to harden the Docker build by replacing the unverified Bun `curl | bash` install with a pinned and verified Bun source aligned with CI.

## Best Possible Solution

Land PR #74359 or an equivalent maintainer-approved hardening change that removes the unverified Bun installer from Docker builds, pins or verifies the Bun source according to repository supply-chain policy, keeps Docker and CI Bun versions aligned, updates Docker docs, and validates the Docker image smoke path.
