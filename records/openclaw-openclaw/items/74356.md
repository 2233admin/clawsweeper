## Summary

Keep open. Current main still executes the unverified Bun installer in the root Dockerfile while the same Dockerfile pins Node base images by digest, and PR #74359 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

Request to replace the root Dockerfile's unverified Bun curl-to-bash install with a pinned or verified Bun acquisition path aligned with CI and Docker image pinning.

## Best Possible Solution

Land PR #74359 or an equivalent maintainer-approved patch that removes Docker build-time execution of the remote Bun installer, pins or verifies the Bun source, adds regression coverage for all relevant Docker stages, updates Docker docs, and keeps Dependabot or another update process responsible for reviewed digest refreshes.

## Reproduction Assessment

Yes. This is reproducible by source inspection on current main: `Dockerfile:47` still pipes `https://bun.sh/install` to `bash`, while the same file pins Node image refs by SHA256 digest.

## Solution Assessment

Unclear. Removing `curl | bash` and using a pinned Bun source is the right direction, but the exact source, digest policy, and docs update should be reviewed by maintainers through PR #74359 or a similarly scoped replacement.

## Security Review

Status: needs_attention

Summary: The issue reports a concrete Docker supply-chain hardening gap that remains present on current main.

Concerns:

- **[high] Unverified Bun installer executes during Docker builds:** `Dockerfile:47`
  - body: The root Dockerfile fetches `https://bun.sh/install` and pipes it to `bash` during the build stage. A compromised upstream script or CDN response would execute as root in the build container and could affect the shipped runtime image.
  - confidence: 0.95
- **[low] Docker docs repeat the unsafe installer pattern:** `docs/install/docker.md:364`
  - body: The Docker faster-rebuild example still tells users to run the same unverified Bun installer and PATH setup, so the public guidance should be updated with the code fix.
  - confidence: 0.9
