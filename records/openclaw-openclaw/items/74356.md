## Summary

Keep open. Current main still contains the reported Docker build supply-chain gap, the public Docker docs repeat the same Bun installer pattern, and #74359 is already an open unmerged implementation PR with closing syntax for this issue, so the issue should remain open for maintainer security review rather than cleanup closure.

## What This Changes

The issue requests replacing the root Dockerfile's unverified Bun curl-pipe installer with a pinned or verified Bun source aligned with CI and the repository's Docker digest-pinning policy.

## Best Possible Solution

Land #74359 or an equivalent reviewed change that removes build-time execution of the remote Bun installer, pins or verifies the Bun source in line with the CI Bun version, updates the Docker docs and digest coverage as needed, and validates the root Dockerfile smoke path before this issue is closed.

## Reproduction Assessment

Yes. Static inspection of current main is enough to reproduce the issue: `Dockerfile:47` pipes `https://bun.sh/install` into `bash`, and `docs/install/docker.md:364` repeats the same pattern in public guidance.

## Solution Assessment

Yes for the direction, but it is not implemented on main yet. Replacing the remote shell installer with a digest-pinned or otherwise verified Bun source is the narrow maintainable fix; the exact source and update policy should be settled in the linked PR review.
