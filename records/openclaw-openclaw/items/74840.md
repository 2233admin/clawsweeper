## Summary

Keep open. The linked cron summary bug is valid, but this PR is not merge-ready because it only changes fallback text selection while the current fatal-error path still overrides summary, output, and delivery with the last structured error payload.

## What This Changes

This PR changes `resolveCronPayloadOutcome` so `finalAssistantVisibleText` is considered before first-payload text when deriving fallback cron summary and output text.

## Best Possible Solution

Repair the cron and embedded-runner outcome boundary so the final assistant-visible reply wins only for a proven recovered self-corrected mutating-tool failure, while real unresolved failures remain fatal, and cover that boundary with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is `resolveCronPayloadOutcome` with only `isError: true` payloads, `finalAssistantVisibleText`, and `preferFinalAssistantVisibleText: true`; current tests assert that shape returns the raw last error, and this PR's fallback-only change does not reach the fatal override.

## Solution Assessment

No. This is not the best fix because the reported behavior is decided by `hasFatalStructuredErrorPayload` and `fatalDeliveryText`, so the safer solution is to classify recovered versus unresolved mutating failures before fatal payloads replace the cron summary and delivery text.
