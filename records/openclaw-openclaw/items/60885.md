## Summary

Current main still only partially addresses the reported LaunchAgent downtime path. The merged launchd supervision work improves restart and repair behavior, but the generated plist still uses ThrottleInterval=1, no ExitTimeOut is emitted, and bootout remains in stop/install lifecycle paths rather than being uninstall-only.

## Best Possible Solution

Keep this issue open and land a focused macOS LaunchAgent lifecycle fix: decide and test the intended throttle/backoff policy, add an appropriate ExitTimeOut to the generated plist, and audit package-update/install/stop paths so bootout is either uninstall-only or always paired with a reliable rollback/rebootstrap path. Preserve the #72616 restart/repair improvements while covering the remaining auto-update downtime scenario.
