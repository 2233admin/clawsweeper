## Summary

Keep open. Current main still lacks this PR's intended Control UI streaming-segment dedupe behavior, and this PR is not merge-ready as-is. The PR addresses a real remaining UI path: current main stores full cumulative stream text at tool boundaries and then renders the full live stream after stored segments, so cumulative gateway deltas can still produce duplicate growing bubbles around tool calls. The PR also carries an unrelated binary workspace artifact and has an unresolved review finding about whitespace-only deltas being dropped.

## Best Possible Solution

Keep this PR open until maintainers either fix and land it or choose a corrected canonical implementation, such as the still-open overlapping PR #47377. The best product/code path is a narrow current-main patch that tracks committed stream length, stores only newly streamed suffixes at tool boundaries, renders only the live suffix after committed segments, preserves whitespace-only separators, removes the unrelated binary artifact, and adds regression coverage for cumulative text before and after one or more tool calls.
