## Summary

Keep open. Current main partially addresses general Control UI localization, but Telegram native command registration still uses one English command catalog and does not register per-language command descriptions; the request also raises command-token compatibility questions that need maintainer scope decisions rather than cleanup closure.

## What This Changes

Add configurable internationalization for Telegram bot command names or descriptions, plus related Web UI command labels/help text, so non-English users can see localized command surfaces.

## Best Possible Solution

Add a backwards-compatible localized command metadata path: keep English command tokens stable by default, support language-keyed Telegram command descriptions through setMyCommands language_code, and route Control UI slash-command descriptions through the existing UI locale layer or localized runtime command metadata.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The missing behavior is high-confidence from code inspection: current main publishes one default Telegram command list and the Web UI command palette copies unlocalized command descriptions.

## Solution Assessment

Unclear for the full request. A narrow description-localization design is maintainable, but translating command names and making one global locale config drive every command surface needs maintainer product/API judgment.
