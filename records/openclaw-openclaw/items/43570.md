## Summary

Keep this issue open. Current main still has the reported Discord `thread-reply` attachment alias gap: the handler reads only `media`, while Discord `send` accepts `media`, `path`, and `filePath`. The same-author PR #43574 remains open and unmerged with closing syntax, so it should be reviewed, landed, replaced, or closed before this issue is closed.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #43574. The implementation should make the Discord `thread-reply` handler resolve `media`, then `path`, then `filePath` with the same precedence as `send`, and add focused regression coverage for those aliases and precedence in the Discord action path.
