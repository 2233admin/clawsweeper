## Summary

Keep this PR open. Current main still routes ordinary MP3/WAV/M4A/FLAC/AAC Feishu media sends as generic files unless the newer `audioAsVoice` path explicitly transcodes them, so the PR's central issue is not implemented on main. The branch is useful but needs maintainer rework because current main has since added explicit transcoding and the PR adds a new media-parsing dependency plus a direct non-Opus-to-`file_type=opus` behavior choice.

## Best Possible Solution

Review and rework this PR rather than close it. Maintainers should decide whether general Feishu audio media should be sent as playable audio by direct `file_type=opus` mapping, by reusing the existing transcode path, or only behind an explicit voice-intent flag; then update `extensions/feishu/src/media.ts` and focused Feishu media tests, with duration handling aligned with the existing media runtime or the related duration follow-up.
