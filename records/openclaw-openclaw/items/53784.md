## Summary

Keep open. Current main still keeps ordinary MP3/WAV/M4A-style Feishu audio attachments as files unless explicit voice delivery requests transcoding, so this PR is not obsolete. It should not merge as-is because it changes the documented Feishu audio contract and still needs docs, lockfile, and maintainer policy alignment.

## What This Changes

The PR broadens Feishu outbound audio routing to treat common audio extensions and audio MIME types as playable audio messages, maps those uploads to `file_type: "opus"`, adds `music-metadata` duration parsing, and updates Feishu media tests/package metadata.

## Best Possible Solution

Decide the Feishu audio contract explicitly. If direct playable bubbles for common formats are desired, land a rebased narrow change that updates routing, duration handling, tests, docs, package metadata, and lockfile together; otherwise keep the shipped explicit voice-transcode path and close or supersede this branch with that policy stated.
