## Summary

Keep this PR open. Current main does not implement `tools.exec.allowedHosts` or per-host `exec-approvals.json` allowlist buckets, so the PR is not obsolete. Main only has the newer `host=auto` routing model plus per-agent `tools.exec.host`/`node` overrides, and concrete host mismatches are still rejected. The PR remains a plausible implementation candidate, but it needs maintainer/security review because the supplied review context includes unresolved command-execution safety concerns around sandbox wildcard allowlists and node-host fallback behavior.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, preserve the useful `tools.exec.allowedHosts` and per-host approval design, then address the unresolved P1/P2 review items with focused tests for gateway-to-sandbox requests, approval-prompt preservation, node-host allowlist compatibility, migration/doctor behavior, and docs/schema generation.
