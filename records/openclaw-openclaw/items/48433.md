## Summary

Keep this PR open. Current main still does not implement `tools.exec.allowedHosts` or per-host `exec-approvals.json` allowlist buckets, and the existing `host=auto` routing is not equivalent to the PR’s gateway-default plus sandbox-request use case. The PR remains a plausible implementation candidate, but it touches command execution routing and approval persistence and still has security-sensitive review concerns that need maintainer follow-up before merge.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main’s `host=auto` model, decide whether maintainers still want explicit `tools.exec.allowedHosts`, and address the remaining security-sensitive review items with focused tests for gateway-default plus sandbox-request routing, sandbox wildcard approvals, node-host compatibility, per-host approval persistence, doctor migration, schema, and docs.
