## Summary

Keep this issue open. Current main requests Telegram Business update names via grammY/default fallback coverage, but it still has no `business_message` or `business_connection` runtime handlers, no Business fields in dedupe/sequential keys, and no `business_connection_id` support for outbound replies. PR #53629 remains open and explicitly targets this issue, so the right cleanup path is maintainer review/landing of that PR or an equivalent Telegram plugin fix.

## Best Possible Solution

Keep this issue open as the canonical Telegram Business support tracker. Review and land PR #53629, or an equivalent Telegram plugin fix, that registers `business_connection` and `business_message` handlers before the normal message handler, routes `business_message` through the existing inbound pipeline, extends dedupe/sequential key handling, tracks and persists `business_connection_id` for outbound replies where Telegram supports it, and adds focused tests plus Telegram docs.
