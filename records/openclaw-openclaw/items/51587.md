## Summary

Keep open. Current main still literal-checks the cron minute field as `"0"`, so `*/15 * * * *`, `0,30 * * * *`, and `0-30 * * * *` still do not receive the default top-of-hour stagger. The issue is also paired with open PR #51594 using closing syntax, so the right maintainer path is to review that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open and review PR #51594. The fix should define Croner-aligned minute-zero semantics, update `src/cron/stagger.ts` for both 5-field and 6-field cron expressions, and add focused regression tests in `src/cron/stagger.test.ts` plus create/normalization coverage where needed. Do not close this issue until the linked PR or an equivalent fix lands.
