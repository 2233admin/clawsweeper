## Summary

Keep open. Current main still lacks a doctor diagnostic for stale persisted skill snapshots: doctor state-integrity checks session stores and transcripts, while the CLI and gateway paths continue to reuse cached `skillsSnapshot` values based on snapshot version and skill-filter state rather than validating cached bundled-skill paths against the active runtime root.

## What This Changes

Add an `openclaw doctor` check that detects stale cached session skill snapshots after runtime-root changes and offers safe remediation.

## Best Possible Solution

Add a focused doctor state diagnostic that resolves the active bundled skills directory, scans persisted session `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills[].filePath` values for missing, inactive, or temp-backed bundled runtime roots, reports cached-session metadata drift separately from live runtime health, and in repair mode backs up the session store before invalidating only affected snapshot fields.

## Repair Work Prompt

Implement a focused fix for https://github.com/openclaw/openclaw/issues/47534. Add an `openclaw doctor` state diagnostic that detects stale cached session skill snapshots after runtime root changes. The diagnostic should resolve the active bundled skills directory, scan persisted session entries for `skillsSnapshot.prompt` locations and `skillsSnapshot.resolvedSkills[].filePath` values that point at missing roots, inactive bundled runtime roots, `/tmp`, or `/private/tmp`, and report this as stale cached session metadata while distinguishing it from live runtime health. In `--fix` / `--repair` mode, back up the session store and delete only the affected `skillsSnapshot` fields so they rebuild on the next run; do not wipe whole sessions or change broad snapshot refresh semantics. Keep this separate from open draft PR #71497's restart-version work. Add focused tests for detection, non-detection for current bundled roots, temp/missing-root handling, and safe repair behavior. Update `docs/gateway/doctor.md`, `docs/cli/doctor.md`, and `CHANGELOG.md` with a user-facing entry crediting `Thanks @sene1337` and, if mentioning the reported edge case context, `Thanks @darbsllim`.
