## Summary

Keep open. Current main still lacks the requested doctor diagnostic/remediation for stale persisted skill snapshots: doctor state integrity checks session-store/transcript health, while CLI and gateway runs continue to reuse cached skill snapshots based on version and skill-filter state rather than validating cached bundled-skill paths against the active runtime root.

## What This Changes

The issue requests an `openclaw doctor` check that detects stale session-cached skill snapshot paths after runtime-root changes and offers safe snapshot invalidation.

## Best Possible Solution

Add a focused doctor state-integrity diagnostic that resolves the active bundled skills directory, scans persisted `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills[].filePath` values for inactive, missing, or temp-backed bundled runtime roots, reports cached-session metadata drift separately from live runtime health, and in repair mode invalidates only affected snapshots after a backup.
