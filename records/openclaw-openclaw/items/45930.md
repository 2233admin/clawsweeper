## Summary

Keep open: current main does not implement the polling-frequency reduction, and this PR is not mergeable as-is because slowing the Logs tab poll to 5 seconds can skip log output under the existing bounded tail contract. The work also overlaps open #69454, so maintainers should either request a narrower revision or explicitly supersede this with the broader Control UI churn fix.

## What This Changes

The PR extracts Control UI polling interval literals into constants and changes nodes/logs/debug polling from 5s/2s/3s to 30s/5s/10s in `ui/src/ui/app-polling.ts`.

## Best Possible Solution

Reduce unnecessary Control UI gateway churn without weakening live-tail semantics: preserve the short Logs tab cadence unless the tail contract is made lossless for slower polling, and handle nodes/debug load through tab-scoped polling or the coordinated #69454 changes.
