## Summary

Keep this PR open. Current main still only scopes non-live implicit provider discovery from explicit `providerDiscoveryProviderIds`; it does not derive a non-live plugin filter from configured agent model/provider hints, so the PR remains relevant. It is not ready for cleanup closure because it overlaps #63552 and #73557 and still has the review-noted fallback test gap.

## What This Changes

The PR adds configured-model/provider hint collection to `src/agents/models-config.providers.implicit.ts` for non-live provider discovery scoping and adds live-filter regression tests for scoped and broad fallback behavior.

## Best Possible Solution

Land or adapt one canonical implementation that scopes non-live implicit provider discovery from configured provider and model hints through the existing ownership seams, preserves broad discovery when hints are absent or ownership cannot be proven, fixes fallback coverage, and coordinates with the explicit-provider and replace-mode PRs.

## Reproduction Assessment

Not applicable as a performance/behavior PR rather than a crash report. The current-main source check is high-confidence: non-live discovery still returns `undefined` unless explicit provider ids are supplied, while the PR diff adds the missing configured-model/provider scope derivation.

## Solution Assessment

Unclear that this exact patch is the best final form. The direction is right, but the safest solution should coordinate with #63552 and #73557, fix the fallback test, and make unresolved-hint behavior explicitly conservative before merge.
