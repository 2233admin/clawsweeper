## Summary

Keep this PR open. Current main still scopes implicit provider discovery only from explicit provider ids or live-provider env filters; it does not collect configured model references from agent defaults, fallbacks, subagents, or named model entries to narrow non-live discovery. The PR remains relevant, especially with related gateway-stall report #74135, but it needs maintainer review because it overlaps #63552 and has a review-noted fallback test gap.

## Best Possible Solution

Keep this PR open for maintainer review. Adapt or land one coordinated implementation with #63552 that scopes non-live implicit provider discovery from configured provider and model hints using the existing owner-resolution seams, preserves broad discovery when hints are absent or unresolved, fixes the fallback test to exercise an unresolved model ref, and validates the startup discovery path relevant to #74135.
