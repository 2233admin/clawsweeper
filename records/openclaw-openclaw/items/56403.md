Keep this PR open. Current main still ships tool-call loop detection disabled by default, and the docs, generated config help, and tests still describe/assert that disabled default. The PR is not implemented on main, and the newer overlapping #70877 is still open rather than merged or clearly canonical.

## Best Possible Solution

Keep this PR open until maintainers choose and land a canonical loop-detection-default change. The likely path is to rebase one focused PR, keep opt-out support, update runtime default plus docs/types/generated schema metadata together, preserve the existing stable exec-result hashing tests, and explicitly decide whether generic_repeat should remain warning-only or gain a consecutive no-progress critical path.
