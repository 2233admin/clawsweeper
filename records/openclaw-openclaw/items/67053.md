## Summary

Keep open. Current main has stale-streaming recovery and a 30s watchdog, but it still lacks the requested prompt transition after visible assistant content finishes; normal TUI idle still depends on the later `final` chat event from gateway lifecycle finalization.

## What This Changes

The issue asks for the TUI streaming indicator to stop or move to a finishing state promptly after the last visible response content token is delivered.

## Best Possible Solution

Add a narrow TUI/gateway completion UX path that separates visible assistant-content completion from full run lifecycle completion, preferably through an additive content-complete or finalizing signal, while keeping the watchdog as recovery and preserving concurrent-run and tool-event behavior.
