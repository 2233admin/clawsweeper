## Summary

Keep open. Current main does not implement this PR's intended Discord teardown/managed-identity drain: the Discord provider only deactivates the message handler during teardown, the current message handler/run queue do not expose a waitForIdle drain, KeyedAsyncQueue has no waitForIdle helper, and Discord inbound context does not include SenderManagedAccountId. The earlier #54270 was closed unmerged and is superseded by this PR, so it is not a canonical merged fix.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is to rebase the Discord teardown fix onto the current message-run-queue architecture, add a narrow idle/drain lifecycle method if still needed, define managed-account attribution without a fragile global single-slot map, and keep the tests focused on deactivate/drain/cleanup ordering plus SenderManagedAccountId context. The independent Telegram button-schema issue appears already handled elsewhere and should not drive this PR.
