## Summary

Keep open because current main still does not implement the requested Discord teardown drain, but the proposed patch is not merge-ready after source review: it turns fatal bot identity resolution into a best-effort path and needs to be adapted to the current refactored cleanup/run-queue shape.

## What This Changes

This PR adds Discord managed bot identity tracking and idle-drain teardown plumbing, extends related tests, and also changes Telegram message-tool schema plus plugin loader/hook type surfaces.

## Best Possible Solution

Rebase and narrow the work to a Discord-owned teardown fix that preserves fatal bot identity startup failures, adds a small idle-drain lifecycle only where current main needs it, and splits unrelated Telegram/plugin contract edits into separate PRs if they are still desired.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve bot identity fetch failures:** `extensions/discord/src/monitor/provider.ts:1043-1053`
  - body: The patch catches `fetchUser("@me")` failures and continues startup with `botUserId` left undefined. Current Discord startup treats a missing bot identity as fatal, and the handler only filters the bot's own messages when `botUserId` is set, so this can let the monitor process its own messages instead of failing startup. Keep using `fetchDiscordBotIdentity` or rethrow after logging.
  - confidence: 0.86
