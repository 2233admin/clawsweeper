## Summary

Keep this PR open. Current main still renders Overview recent sessions with `displayName` before `label`, so the PR’s intended label-first UI fix is not implemented on main. The related #45130 work added a label-first helper and tests for the chat/session dropdown surface, but Overview still uses its own inline fallback.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to make Overview recent sessions use the same label-first, key-aware display semantics as the chat session dropdown, preferably by reusing or extracting `resolveSessionDisplayName`, and add focused render/unit coverage for `label > displayName > key` plus key-equality edge cases.
