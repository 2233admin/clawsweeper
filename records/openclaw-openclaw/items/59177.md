Current main still does not expose Discord sender client platform status in inbound conversation metadata, while the existing Discord presence cache and GuildPresences wiring make the PR’s approach plausible. The PR is focused, recent, non-maintainer-authored, and has only minor bot review suggestions rather than evidence that it is obsolete or superseded.

Keep the PR open and ask for a small maintainer-quality update: rebase on current main, add the empty-object guard from the review, avoid redundant `?? undefined`, and add focused coverage that proves Discord sender `client_status` is omitted when unavailable and rendered in user-role inbound metadata when present.
