## Summary

Keep this PR open. Current main has related live dispatch and replay normalization for provider-prefixed tool names, but it still does not implement this PR's central sanitizer/persisted-transcript behavior for leading `functions ` tool-call IDs, prefixed tool-result names, or prefixed `sessions_spawn` redaction. The predecessor #39091 is closed unmerged, so this PR remains the active implementation candidate rather than cleanup-closeable.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow agent-side fix that normalizes only leading `functions ` versus `functions.` ID equivalence where sanitizer matching needs it, strips `functions.` / `functions ` prefixes before transcript-repair allowed-name checks and `sessions_spawn` redaction, preserves raw IDs for pending-state and replay-safe thinking behavior, and keeps the regression tests. Review or split the bundled host=node exec `workdirExplicit` approval-path change before merging.
