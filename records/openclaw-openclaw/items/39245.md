## Summary

Keep this PR open. Current main still does not implement the PR's central agent fix: leading `functions ` tool-call IDs are not normalized for sanitizer matching, and `functions.` / `functions ` prefixed tool names are not normalized in transcript repair or `sessions_spawn` redaction. The older related PR #39091 is closed unmerged, so this PR remains the active implementation candidate, with one security-sensitive bundled node-exec approval-path change that should get maintainer review or be split.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow agent-side fix that normalizes only the leading `functions ` versus `functions.` ID equivalence where matching needs it, strips `functions.` / `functions ` prefixes before transcript-repair allowed-name checks and `sessions_spawn` redaction, preserves raw IDs for pending-state and replay-safe thinking behavior, and keeps the regression tests. Review or split the bundled host=node exec workdir/approval change before merging.
