## Summary

Keep this PR open. Current main has related native Kimi dot-ID preservation and live/replay tool-name normalization, but it still does not implement this PR's central `functions ` versus `functions.` ID-equivalence behavior in the sanitizer, nor does persisted transcript repair strip `functions.` / `functions ` prefixes before allowlist checks, `sessions_spawn` redaction, or tool-result name normalization. The predecessor #39091 is closed unmerged, and the contributor has kept this PR active, so it remains the live implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt a narrow agent-side fix that treats only leading `functions ` and `functions.` tool-call IDs as equivalent where sanitizer pairing needs it, strips `functions.` / `functions ` prefixes before persisted transcript repair and tool-result name normalization, preserves raw IDs for pending-state and replay-safe thinking behavior, keeps the regression tests, and separately reviews or splits the bundled node-host exec explicit-workdir approval-path change.
