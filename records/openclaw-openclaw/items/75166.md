## Summary

Keep open. Current main still contains the compact provider-inventory bypass for `image_generate` and `video_generate`, and the Discord guild-channel path can still deliver the resulting text as a visible tool result. The related #70912 discussion is useful context for this leak class, but that closed item does not cover this separate provider-inventory output path.

## What This Changes

Prevent `image_generate` and `video_generate` provider inventory from being delivered as visible Discord group/guild-channel tool output by default.

## Best Possible Solution

Keep the full provider inventory available to the agent/tool context, but stop emitting the compact `action=list` inventory as external chat tool output by default. The likely fix belongs in the shared embedded tool-output gate, with focused coverage for image/video list results and a Discord guild-channel or generic non-direct delivery assertion; generated media attachment delivery should remain unchanged.

## Reproduction Assessment

Yes. Without running live Discord, current source plus existing tests give a high-confidence reproduction path: produce an `image_generate` or `video_generate` list result with `details.providers`, observe `emitToolOutput` despite `shouldEmitToolOutput()` being false, then follow the Discord guild-channel dispatch path to visible delivery.

## Solution Assessment

Yes, the suggested direction is the right class of fix. The narrow maintainable solution is to remove or surface-gate the compact provider-inventory bypass while preserving internal tool results and media delivery; redaction alone is less robust unless maintainer policy requires showing a minimal inventory.
