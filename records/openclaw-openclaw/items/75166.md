## Summary

Keep open. Current main still contains the compact image/video provider-inventory emission path, the existing media handler test asserts that it emits even when normal tool output is disabled, and Discord guild messages are treated as channel surfaces where tool summaries can be delivered. The related #70912 fix covers Slack verbose progress traces, not this separate provider-inventory output path.

## What This Changes

The issue asks OpenClaw to stop exposing `image_generate` and `video_generate` provider inventory output as visible tool output in Discord group/channel chats by default.

## Best Possible Solution

Keep the full provider inventory available to the agent/tool context, but remove or explicitly surface-gate the compact inventory emission for `action=list` results on shared external chat surfaces. Generated media attachment delivery should stay on the existing structured media path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by returning an `image_generate` or `video_generate` list result with `details.providers` while `shouldEmitToolOutput()` is false; current tests already expect visible emission for that case, and Discord guild turns use a channel surface that can deliver tool summaries.

## Solution Assessment

Yes. The requested fix direction is the narrowest maintainable class of solution: suppress or explicitly gate the compact inventory bypass before external delivery, rather than changing provider list internals or generated media delivery.
