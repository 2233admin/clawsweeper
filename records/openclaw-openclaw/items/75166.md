## Summary

Keep open. Current main still has the compact provider-inventory emission bypass for image/video list results, and Discord guild-channel delivery can still pass that text to participants; the related #70912 thread is useful context but does not cover this separate provider-inventory path.

## What This Changes

The issue asks OpenClaw to stop exposing `image_generate` and `video_generate` provider inventory details as visible Discord group/channel tool output by default.

## Best Possible Solution

Keep full provider inventory in the internal tool result for agent reasoning, but stop emitting `action=list` inventory as external chat tool output by default. The narrow fix belongs in the shared embedded tool-output gate, with coverage for image/video list results and Discord guild-channel or generic non-direct delivery behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: send an image/video list result with `details.providers` while `shouldEmitToolOutput()` is false, observe `emitToolOutput`, then follow the Discord guild-channel dispatch path to visible tool-result delivery.

## Solution Assessment

Yes. The issue’s suggested direction is the right class of fix: remove or surface-gate the compact inventory bypass while preserving internal tool results and generated media delivery; redaction alone is less robust unless maintainers intentionally want a minimal visible inventory.
