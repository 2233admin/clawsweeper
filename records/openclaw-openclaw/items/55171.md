## Summary

Keep this PR open for revision. The stale Feishu binding cleanup goal is valid, but the proposed agent-mismatch rule is too broad and would remove intentional cross-agent ACP/focus/subagent bindings; current main also routes this path through the shared runtime binding helper, so the branch needs a targeted rewrite.

## What This Changes

The PR changes Feishu inbound routing to unbind runtime conversation bindings when the bound session agent differs from the resolved route agent, adds a regression test, and updates Feishu lifecycle fixtures to use main-agent session keys.

## Best Possible Solution

Implement Feishu stale cleanup with a narrow route-shaped rule: clean up only legacy session bindings whose parsed session route matches the current route except for agent id, while preserving explicit ACP, subagent, plugin-owned, and metadata-backed conversation bindings.
