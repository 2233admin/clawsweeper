## Summary

Keep this issue open. Current main still has Matrix outbound draft/live preview behavior and accurate `WasMentioned` metadata, but it does not expose inbound streaming/finality/edit-relation state to the agent context, prompt metadata, or plugin message hook contract. The prior ClawSweeper keep-open comment remains accurate against current main SHA `7e5c3753f6f797d4faf7e22eefe5a11f5a3a17ed`.

## Best Possible Solution

Keep this open and make a narrow maintainer-approved Matrix/core inbound contract. The likely paths are either to suppress live/partial Matrix events from agent routing until a final replacement is available, or to expose typed fields such as `MessageStreamingState`, `IsStreaming`, `IsFinal`, and relation target metadata through finalized inbound context and plugin hook mapping. Cover Matrix `m.replace`, MSC4357 live drafts, `blockStreaming` bot-to-bot rooms, `allowBots` mention gating, replies, and threads in Matrix monitor and hook-mapper tests.
