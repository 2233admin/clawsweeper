## Summary

Keep this issue open. Current main still has Matrix outbound draft/live preview behavior and accurate `WasMentioned` mention metadata, but it does not expose streaming/finality/edit-relation state to the inbound agent context, prompt metadata, or message hook contract. The prior ClawSweeper keep-open comment remains accurate against current `main`; the relevant Matrix and shared context paths are unchanged since that review.

## Best Possible Solution

Keep this open and implement a narrow Matrix/core inbound contract for streaming state. The safest maintainer path is either to suppress live/partial Matrix events from agent routing until finalization, or to add typed fields such as `MessageStreamingState`, `IsStreaming`, `IsFinal`, and relation target metadata to finalized inbound context and plugin hook mapping. Cover Matrix `m.replace`, MSC4357 live drafts, `blockStreaming` bot-to-bot rooms, `allowBots` mention gating, and thread/reply handling in Matrix monitor tests.
