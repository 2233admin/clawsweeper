## Summary

Keep open. Current main still has adjacent Matrix outbound preview streaming fixes, including MSC4357 live markers and block-streaming alignment, but it does not expose inbound streaming, finality, or edit-relation state to agents or message hooks. Matrix inbound handling still drops replacement edits before routing, and the shared agent/hook context types have no field for the requested state.

## What This Changes

The issue requests agent-visible metadata for Matrix inbound streaming, finality, or edit relation state so multi-agent rooms can ignore partial draft chunks.

## Best Possible Solution

Add a narrow typed inbound streaming/relation contract, or gate Matrix live draft events until finalization, then propagate the chosen state through Matrix inbound routing, FinalizedMsgContext, and plugin/internal message hook mapping. Coverage should include MSC4357 live initial events, m.replace edits, allowBots with requireMention rooms, and thread/reply cases.

## Reproduction Assessment

Unclear. The issue gives a concrete Matrix config and event sequence, and current source proves the requested metadata is absent, but this read-only review did not run a live Matrix multi-agent room to reproduce the truncated first-chunk response end to end.

## Solution Assessment

Unclear as proposed. The requested is_streaming/is_final or raw m.relates_to fields address the gap, but the most maintainable fix should be a typed Matrix/core inbound state or a routing suppression policy chosen by maintainers before field names become public contract.
