## Summary

Keep open. Current main still appends the workspace HEARTBEAT.md path hint to configured heartbeat prompts whenever the prompt text mentions HEARTBEAT.md, and the issue is already paired with open unmerged PR #40756 using closing syntax.

## What This Changes

The issue asks OpenClaw to preserve user-configured heartbeat prompt text instead of appending the workspace HEARTBEAT.md path hint to custom prompts.

## Best Possible Solution

Land PR #40756 or an equivalent narrow fix that preserves user-authored heartbeat prompts verbatim, keeps the workspace-path hint for built-in/default and internally generated heartbeat prompts, and adds regression coverage for a configured prompt that mentions HEARTBEAT.md.
