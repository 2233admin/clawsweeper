## Summary

Keep open. Current main still honors configured heartbeat prompts only during resolution, then unconditionally appends the workspace HEARTBEAT.md hint based on prompt text, which conflicts with the documented verbatim custom-prompt behavior; open PR #40756 remains the active unmerged implementation candidate using closing syntax.

## What This Changes

Honor user-configured heartbeat prompts verbatim by not appending the workspace HEARTBEAT.md path hint to custom heartbeat prompts.

## Best Possible Solution

Land a focused heartbeat-runner fix that carries or computes prompt-origin information, skips the workspace-path hint for non-empty user-authored heartbeat prompts, preserves the hint for built-in/internal prompts, and adds regression coverage for default-level and per-agent custom prompts.

## Reproduction Assessment

Yes. A config with agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt containing HEARTBEAT.md reaches resolveHeartbeatPrompt, then resolveHeartbeatRunPrompt appends the workspace hint because the helper only inspects prompt text.

## Solution Assessment

Yes. The narrow maintainable solution is a custom-prompt guard around hint injection while preserving the default/internal prompt behavior; PR #40756 targets that direction, but the issue should remain open until that PR or an equivalent replacement merges.
