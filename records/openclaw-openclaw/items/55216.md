## Summary

Keep open. Current main still matches the Telegram album bug: recoverable media-group download failures are logged and skipped, then the remaining media are sent onward without a user-visible warning. The related batching and polling-offset reports are adjacent, not superseding this specific skipped-item notification gap.

## What This Changes

Warn Telegram users when a media group loses one or more images to recoverable download or media-resolution failures instead of silently forwarding only the successful media.

## Best Possible Solution

Implement the fix inside the Telegram plugin: count recoverable exceptions and null media results during processMediaGroup, process successfully downloaded media as today, and send one concise Telegram warning such as “Received N of M images; K could not be fetched” for skipped album items. Keep timer configurability and corrupted-file validation as separate follow-ups unless maintainers choose to broaden scope.
