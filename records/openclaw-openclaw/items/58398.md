## Summary

Keep open. Current main has related pieces, including context pruning, staged/split summarization, preserved recent-turn suffixes, post-compaction AGENTS.md refresh, and pluggable compaction providers. It does not yet ship the central requested built-in compaction architecture: no analysis scratchpad/strip phase, no compaction-specific microCompact pre-pass, no full-transcript recovery pointer in the compaction replacement message, no all-user-message preservation requirement, and no automatic restoration of recently accessed file or skill content. PR #58455 is an open, unmerged implementation candidate for part of the request, but it is not on main and does not cover the whole umbrella.

## Best Possible Solution

Keep #58398 open for maintainer/product handling, or split it into concrete subtasks. Review or supersede PR #58455 for the microCompact, scratchpad, and transcript-reference pieces; separately decide whether built-in safeguard compaction should add recent-file/skill restoration, all-user-message preservation, anti-tool-call guardrails, and exact partial-compaction modes. Plugin/context-engine APIs are useful for optional alternative compactors, but the requested built-in safeguard behavior still belongs in core if maintainers want it.
