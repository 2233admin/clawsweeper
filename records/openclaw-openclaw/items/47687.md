## Summary

Keep this PR open. Current main still only queues generic Telegram emoji reaction system events; it does not implement `reactionSemantics`, stable `reaction_key` text, Telegram `custom_emoji` reaction handling, or heartbeat wake for mapped reactions. The PR is also explicitly paired with still-open #47677 and relates to still-open wake bug #64752, so it remains a live implementation candidate rather than cleanup to close.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #47677 and related wake report #64752. Maintainers should review the Telegram-owned config/runtime design, settle the SDK seam for heartbeat wake access, and land a refined version with schema/docs/tests covering standard emoji, Telegram custom emoji, `wake|queue|ignore`, authorization, duplicate suppression, and queue-only generic notifications.
