## Summary

Keep open: current main has several adjacent resilience mechanisms, but it still does not implement the requested Gateway-wide process watchdog, configurable recovery-policy surface, richer HTTP health contract, or generalized degraded startup mode. The related open PR #53716 appears to cover part of the recovery/watchdog space for #53684, but it is not merged and does not close this broader issue.

## What This Changes

Requests built-in Gateway resilience features: a process watchdog with crash-loop limits, global error recovery strategies, richer health details, and degraded startup behavior.

## Best Possible Solution

Keep this as a Gateway resilience design item and split it into maintainer-approved follow-ups: process crash-loop watchdog and notification semantics, explicit recovery-policy hooks for agent/provider/runtime errors, additive health schema expansion, and a defined degraded startup mode that composes with diagnostics, restart sentinel, doctor, channel monitor, and config recovery.
