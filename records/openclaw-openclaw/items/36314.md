## Summary

Keep this open. Current main has adjacent Gateway reliability work, but it still does not provide the requested first-class process watchdog, configurable restart-window auto-stop policy, user-configurable global error recovery strategies, richer /health process counters, or general degraded-startup mode.

## Best Possible Solution

Keep this open as a maintainer-owned reliability umbrella, or split it into focused issues: process watchdog config/schema/docs and supervisor behavior; global error handling policy and whether user-configurable recovery strategies are desirable; privacy-safe health/readiness counters; and explicit degraded-startup semantics that build on existing last-known-good config recovery and channel health monitoring.
