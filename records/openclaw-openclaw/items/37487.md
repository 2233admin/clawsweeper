## Summary

Keep open. Current main still supports named isolated browser profiles and a global `browser.extraArgs` proxy workaround, but it does not implement the requested first-class `browser.profiles.<name>.proxy` contract or `agents.defaults.browser.profile` / `agents.list[].browser.profile` routing for concurrent agent browser runs. Related PRs #37534 and #42806 are closed unmerged, and the linked `m13v/browser-lock` project is useful prior art rather than shipped OpenClaw behavior.

## What This Changes

Request first-class support for concurrently running isolated per-agent browser profiles with profile-specific proxy configuration and agent/subagent browser-profile defaults.

## Best Possible Solution

Keep this open for maintainer-reviewed design. The strongest path is a secret-aware `browser.profiles.<name>.proxy` contract for locally managed profiles, redacted profile/status diagnostics, explicit agent and sub-agent browser profile inheritance rules, docs/examples, and focused tests that cover concurrent isolated profiles while preserving current remote CDP and existing-session attach semantics.
