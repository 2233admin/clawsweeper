## Summary

Current main at b85edb3 still lacks the central requested contract: first-class per-profile browser proxy configuration and per-agent default browser profile binding for concurrent browser runs. Named browser profiles, separate CDP/user-data settings, explicit per-call `profile`, and global `browser.extraArgs` proxy flags exist, but there is still no `browser.profiles.<name>.proxy`, no `agents.defaults.browser.profile` / `agents.list[].browser.profile`, and no profile-level `proxyConfigured` diagnostic. Related PR #42806 is closed unmerged, and the `m13v/browser-lock` comment is useful prior art rather than shipped OpenClaw behavior.

## Best Possible Solution

Keep this issue open for maintainer-reviewed design and implementation. The best path is to add a secret-aware `browser.profiles.<name>.proxy` contract for locally managed profiles, apply it only during managed browser launch, expose redacted diagnostics such as `proxyConfigured`, add `agents.defaults.browser.profile` and `agents.list[].browser.profile` with clear sub-agent inheritance rules, and document/test concurrent isolated profiles. Use PR #42806 and `m13v/browser-lock` as prior art, not as shipped fixes.
