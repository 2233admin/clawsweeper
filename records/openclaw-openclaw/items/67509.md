## Summary

Keep this PR open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still does not implement the real/effective UID root guard that this PR adds, and the linked root-execution corruption report remains open with a concrete DigitalOcean reproduction. The PR remains the active focused implementation candidate; its described diff is limited to CLI source and tests and does not introduce an obvious supply-chain or CI/security-sensitive change.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both src/entry.ts and src/index.ts before state/config side effects, checks both real and effective UID, preserves intentional container forwarding only, includes focused regression tests, and updates DigitalOcean/install guidance or wrapper behavior for root SSH users.
