## Summary

Keep open. Current main still has no root guard, and this is an active focused implementation for the root-execution bug tracked by #67478. The patch should not merge as-is because it blocks non-help/version root CLI invocations while the DigitalOcean setup docs still instruct users to SSH as root and run `openclaw onboard --install-daemon`, so maintainer sign-off on the root policy and matching docs/install flow is still needed.

## What This Changes

The PR adds a CLI root guard with real/effective UID checks and escape hatches, covers it with tests, and wires it into both modern and legacy CLI entrypoints.

## Best Possible Solution

Land this patch or an equivalent core fix after maintainer sign-off, with DigitalOcean/1-Click guidance updated to the chosen non-root workflow while preserving the early modern and legacy entrypoint guard, real/effective UID coverage, and intentional container-forwarding bypass.
