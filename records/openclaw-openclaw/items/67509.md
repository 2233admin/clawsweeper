## Summary

Keep this PR open. Current main at 64bd2a2cbe805ddd850dbcbd210c1c8cb6499db6 still has no CLI-wide root guard, the linked corruption report #67478 remains open, and the PR is a focused implementation candidate. It needs maintainer review because refusing root CLI execution is product-visible and currently conflicts with root-based DigitalOcean setup docs.

## What This Changes

The PR adds a CLI root guard and tests, then wires it into the modern and legacy CLI entrypoints to block uid/euid 0 unless an explicit root override or the container-forwarding bypass is present.

## Best Possible Solution

Keep this PR open for maintainer review.
