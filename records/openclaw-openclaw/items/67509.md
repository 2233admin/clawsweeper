## Summary

Keep open. Current main still has no CLI root guard, and this PR is a focused implementation for the linked root-state corruption bug; however, the proposed root refusal still needs maintainer sign-off because it would make the current DigitalOcean/VPS docs' root-shell setup commands fail unless the install guidance or policy is updated with the same change.

## What This Changes

The PR adds a CLI root guard with tests and wires it into the modern and legacy CLI entrypoints to block real or effective UID 0 unless an explicit root override or container-forwarding hints are present.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with matching install guidance: guard both CLI entrypoints before mutable state/config side effects, check both real and effective UID, preserve intentional container forwarding, and document the supported non-root service-user workflow plus any explicit root override.
