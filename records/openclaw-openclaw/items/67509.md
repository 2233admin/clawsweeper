## Summary

Keep open. Current main still lacks the requested root-execution guard, and the PR is a focused implementation for the root-state corruption reported in #67478, but the proposed behavior conflicts with current DigitalOcean setup docs that still run `openclaw` from a root SSH session.

## What This Changes

The PR adds a CLI `assertNotRoot` guard with real/effective UID checks and explicit env/container bypasses, wires it into modern and legacy CLI entrypoints, and adds unit coverage.

## Best Possible Solution

Adopt one maintainer-approved root-execution policy across the CLI, DigitalOcean/1-Click flows, and docs: refuse unsafe root state writes before startup side effects, preserve intentional container/CI use, and document the supported non-root service-user workflow plus explicit override.
