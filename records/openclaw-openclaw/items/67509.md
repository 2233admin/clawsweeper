## Summary

Keep open. Current main still has no CLI root-execution guard, and the PR is a focused implementation for the linked DigitalOcean state-corruption bug, but it should not merge as-is until maintainers reconcile the new root refusal with the existing DigitalOcean/VPS setup docs and root-policy expectations.

## What This Changes

The PR adds a CLI root guard with real/effective UID checks and escape hatches, wires it into the modern and legacy CLI entrypoints, and adds unit coverage for root, non-root, Windows, container-forwarded, and setuid-root cases.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with both CLI entrypoints guarded before mutable state/config side effects, real and effective UID coverage retained, intentional container forwarding preserved, and DigitalOcean/1-Click guidance updated to the chosen non-root service-user or wrapper workflow.
