## Summary

Keep open. Current main still lacks an equivalent CLI root/effective-root guard in both the modern and legacy entrypoints, the linked #67478 bug remains open, and this PR remains the focused implementation candidate. The recent mainline systemd hardening only narrows sudo user-scope behavior; it does not prevent direct root CLI startup or root-scoped state. Security review found no CI, dependency, lockfile, release, install-script, or package-resolution changes in the provided PR file list.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both src/entry.ts and src/index.ts before state/config side effects, checks both real and effective UID, preserves intentional container forwarding, includes focused regression tests, and updates DigitalOcean/install guidance or wrapper flow so root SSH users get a clear supported command path.
