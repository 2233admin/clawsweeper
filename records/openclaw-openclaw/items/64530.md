## Repair Work Prompt

Implement a first-party append-only reopen/startup transcript recovery seam for issue 64530. The fix should inspect only the active transcript tail before prompt build/session resume, preserve existing transcript entries, append parentId-valid synthetic error `toolResult` rows for eligible unresolved tail `toolCall`s, append exactly one explicit assistant recovery marker for a persisted orphan-user tail, be idempotent across repeated reopen/startup, and never re-execute tools. Use `SessionManager.appendMessage(...)` or an equivalent parentId-aware wrapper; do not raw-append JSONL message rows. Coordinate with `installSessionToolResultGuard`, `repairToolUseResultPairing`, and `main-session-restart-recovery` so live guard flushing and replay sanitization do not duplicate durable synthetic rows. Decide and document the policy for `stopReason: "error"` / `"aborted"` assistant tool-call turns, since current code intentionally avoids synthesizing for those in live/replay paths. Add focused tests for missing tail toolResult, partial tail results, orphan-user tail marker append, repeated reopen no-op, and healthy transcript no-op. Update docs/reference transcript hygiene or session management docs if behavior changes, and add a single-line changelog fix entry crediting `Thanks @glfruit`. Do not broaden into a write-atomicity redesign or historical transcript rewrite.
