## Summary

Keep open. Current main still resolves default exec approvals file and socket storage through home-relative constants, and the affected CLI/gateway helpers inherit that path. The focused repair PR #74002 is open and unmerged, so closure would be premature.

## What This Changes

The issue asks OpenClaw to store exec approvals under the active state root/profile instead of the home-backed default path.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives the approvals JSON path, socket path, and effective-policy host reporting from the active state-dir/profile contract, preserves the legacy fallback when no override is active, and updates focused runtime/CLI tests plus public approvals docs.
