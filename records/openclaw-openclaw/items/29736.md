## Summary

Keep this issue open. Current main still resolves exec approvals file/socket defaults through home-expanded `~/.openclaw` paths instead of the active `OPENCLAW_STATE_DIR` state root, and that resolver feeds local CLI storage, gateway RPCs, node-host RPCs, and host exec enforcement. Open PR #74002 remains the focused implementation candidate, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that exec approvals storage and socket paths honor the active OpenClaw state root/profile semantics instead of defaulting to `~/.openclaw`.

## Best Possible Solution

Keep this issue open and route it to #74002 or a maintainer-chosen equivalent. The final fix should derive exec approvals JSON, default socket, and display paths from active state-dir/profile semantics, preserve the unset home-backed fallback, keep path-safety checks sound, update docs and focused regression tests, and validate local CLI, gateway, node-host, and `system.run` enforcement before closing as implemented.
