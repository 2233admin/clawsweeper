## Summary

Keep open. Current main still resolves exec approvals file, socket, CLI/gateway access, and effective-policy host-source display through home-backed defaults, while the focused implementation candidate in #74002 is open and unmerged.

## What This Changes

The issue requests that exec approvals file and socket storage follow the active OpenClaw state root/profile instead of defaulting to home-backed paths.

## Best Possible Solution

Land #74002 or a maintainer-chosen equivalent that derives the approvals JSON path, default socket path, and effective-policy default host path from the active state-dir/profile contract, preserves the legacy home fallback when no override is active, and updates focused CLI/gateway/runtime tests plus public approvals docs.
