## Summary

Keep open. Current main still resolves exec approvals file and socket defaults through home-expanded `.openclaw` paths rather than the active `OPENCLAW_STATE_DIR` or profile-derived state root. The same helper feeds local CLI reads/writes, gateway approvals RPCs, node-host approvals RPCs, and `system.run` policy enforcement. There is an open implementation candidate, #74002, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and route it to #74002 or an equivalent canonical PR. The fix should derive exec approvals file and socket defaults from the active state root/profile semantics, preserve a safe legacy fallback or migration path for existing home-backed approval files, update tests/docs, and validate the gateway, node-host, and local CLI surfaces before closing as implemented.
