## Summary

Keep this PR open. Current main has absorbed several Telegram fixes, but the batch is still only partially implemented in the current plugin layout, and the PR itself targets stale pre-plugin `src/telegram` paths. A direct merge would be risky, but closing it would drop useful unported work.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as one stale batch. Split and port the still-relevant fixes into `extensions/telegram`, especially getFile failure surfacing, command-menu config hash invalidation, mention-gating verbose logging, broader media-group item skipping, explicit-network fallback behavior, and approval auth edge cases. Preserve the newer main behavior for reply parameters, ACP placement, proxy transport, word-boundary chunking, and conservative non-idempotent send retries.
