## Summary

Keep this PR open. Current main has absorbed several Telegram fixes through the newer plugin layout, but the batch is not implemented end to end and a direct merge would be stale/risky because the PR targets older `src/telegram` paths while current Telegram code lives under `extensions/telegram`.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as a single stale batch. Split and port the still-relevant fixes into `extensions/telegram`, especially getFile failure surfacing, command-menu config hash invalidation, mention-gating verbose logging, broader media-group item skipping, and any remaining draft/metadata sanitizer coverage. Preserve the newer main behavior for reply parameters, ACP placement, proxy transport, word-boundary chunking, and conservative non-idempotent send retries.
