## Summary

Keep this PR open for maintainer review, but do not merge it as-is. Current main still lacks cron multi-target delivery, so the branch is useful source work; however, the diff still has delivery-mode and session-routing bugs that can silently skip configured targets or route `channel: "last"` fan-out through the wrong session.

## What This Changes

The PR adds cron `delivery.additionalTargets` types, gateway schema support, normalization, CLI edit flags, isolated-run fan-out dispatch and tests, plus lazy channel-runtime lookups in `commands-session.ts`.

## Best Possible Solution

Land one canonical cron fan-out implementation that explicitly defines supported delivery modes, resolves `channel: "last"` from the job delivery session context, keeps schema/normalization/CLI/runtime/docs/tests aligned, and either implements webhook fan-out from the gateway webhook completion path or rejects `additionalTargets` outside supported announce delivery.

## Reproduction Assessment

Yes for the review findings. Static inspection is sufficient: a webhook delivery with `additionalTargets` validates through the shared schema but cannot reach the direct-send `sendOccurred` gate, and a `channel: "last"` additional target is resolved with `params.agentSessionKey` rather than the job delivery session key.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best mergeable solution until it fixes unsupported delivery-mode behavior and passes the correct session context into additional-target resolution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject or deliver non-announce fan-out:** `src/gateway/protocol/schema/cron.ts:191`
  - body: Adding `additionalTargets` to the shared delivery properties makes webhook and none deliveries accept and persist targets, but the runtime only fans out when `dispatchCronDelivery` reports a direct announce send. Webhook delivery is posted later by gateway notifications, so configured extra targets are silently skipped. Restrict the field to supported modes or trigger fan-out from the webhook completion path.
  - confidence: 0.9
- **[P2] Resolve last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: For additional targets with `channel: "last"`, this call passes the fresh cron run session key into `resolveDeliveryTarget`. Primary cron delivery resolves from the job's preserved delivery session, so session-bound jobs can route fan-out to the wrong conversation or fail unexpectedly. Pass the job delivery session key and cover it with a regression test.
  - confidence: 0.86
