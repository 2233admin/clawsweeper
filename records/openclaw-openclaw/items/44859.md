## Summary

Keep this PR open. Current main still has single-target cron delivery, so the contribution is not obsolete, but the patch should not merge as-is because it accepts additional targets for delivery modes that never fan out and can resolve `channel: "last"` fan-out against the wrong cron run session.

## What This Changes

The PR adds `delivery.additionalTargets` cron fan-out across cron types, gateway schema, normalization, runtime delivery dispatch, CLI edit flags, and tests, plus lazy channel-runtime access in `commands-session.ts`.

## Best Possible Solution

Land one canonical, documented cron fan-out implementation that keeps schema, normalization, delivery planning, runtime dispatch, trace/status reporting, CLI/UI surfaces, docs, changelog, and regression tests aligned, with explicit behavior for announce, webhook, no-delivery, primary-success gating, `channel: "last"`, and per-target failure isolation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/cron/isolated-agent/run.ts:770-772`
  - body: `additionalTargets` is added to the shared delivery schema, so webhook-mode cron jobs can now accept and persist extra targets. The runtime fan-out only runs when `dispatchCronDelivery` reports `sendOccurred`, but webhook delivery is posted later by the gateway notification path, so webhook jobs silently skip all additional targets. Please reject `additionalTargets` outside supported modes or trigger fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve last fan-out targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:762-767`
  - body: Additional targets can use `channel: "last"`, but this helper passes the cron run's `agentSessionKey` into `resolveDeliveryTarget`. Primary delivery resolves against the job's configured session key, so session-bound jobs can route fan-out from the wrong last channel/thread or fail closed. Pass the job delivery session key into this helper and cover `channel: "last"` fan-out with a regression test.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces one concrete routing-sensitive outbound delivery concern before merge.

Concerns:

- **[medium] Fan-out can resolve the wrong last recipient:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: `channel: "last"` additional targets are resolved with the cron run session key instead of the job delivery session key. For session-bound cron jobs, that can send cron output to the wrong prior channel/thread or expose output to an unintended recipient.
  - confidence: 0.84
