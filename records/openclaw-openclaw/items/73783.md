## Summary

Keep this PR open. Current main still has the reported subagent announce race, the PR is a plausible targeted fix with focused tests, and the remaining maintainer decision is how to handle its overlap with the broader open raw-tool-output fix in #74340.

## What This Changes

The PR adds compaction-marker handling to subagent announce output selection, keeps timeout partial-progress announcements working while paused, and adds regression tests for the compaction limbo, recovery, and timeout paths.

## Best Possible Solution

Land one consolidated subagent announce fix that both recognizes compaction markers as a continuation boundary and prevents successful completion announces from treating raw tool results as final replies, with regression coverage from this PR and #74340 plus a user-facing changelog entry if maintainers accept it as a shipped bug fix.

## Reproduction Assessment

Yes. The PR body gives a concrete production timeline, and current main can be checked deterministically with the same history shape: toolResult text before a synthetic compaction marker leaves `latestRawText` available for selection before a real assistant answer arrives.

## Solution Assessment

Yes for the narrow compaction-window bug: the PR uses the existing synthetic marker contract and mirrors the `sessions_yield` pause mechanism. As the final project direction, it is unclear whether maintainers should merge this as-is or combine it with #74340's broader raw-tool-result suppression.
