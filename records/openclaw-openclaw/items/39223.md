## Summary

Current main still reaches the `Where will the Gateway run?` select before any `--section model` handling, so the reported configure path can still block before Anthropic auth setup. Direct `openclaw models auth ...` commands are documented and useful workarounds, but they do not fix the wizard path itself.

## Best Possible Solution

Keep this open and fix the configure path itself. The narrow implementation should let section-filtered model/auth configuration reach `promptAuthConfig` without first requiring the gateway-location select when the selected sections do not need gateway, daemon, channel, or health work. Add an early actionable non-interactive TTY guard that points users to `openclaw models auth setup-token --provider anthropic` and `openclaw models auth paste-token --provider anthropic`, and update configure docs/tests so direct auth commands are documented as workarounds rather than the closing fix.

## Clownfish Work Prompt

Fix the configure wizard hang reported for section-filtered model/auth setup. On current main, `runConfigureWizard` asks `Where will the Gateway run?` before it handles `opts.sections`, so `openclaw configure --section model` can block before Anthropic auth setup. Update the wizard so section-filtered model/auth configuration can call `promptAuthConfig` without first requiring the gateway-location select when the requested sections do not need gateway, daemon, channel, or health work. Preserve existing behavior for full `openclaw configure` and for sections that genuinely need gateway mode. Add an early actionable non-interactive TTY guard for interactive configure paths that points Anthropic users to `openclaw models auth setup-token --provider anthropic` and `openclaw models auth paste-token --provider anthropic`. Add regression tests in the configure wizard tests proving `sections: ["model"]` reaches `promptAuthConfig` without consuming a gateway select, and keep or update tests for gateway/channel/daemon/health paths that should still request gateway mode. Update `docs/cli/configure.md` if behavior changes; keep direct `models auth` commands documented as workarounds, not as the fix itself. Do not change provider auth semantics, token storage, or unrelated model selection behavior.
