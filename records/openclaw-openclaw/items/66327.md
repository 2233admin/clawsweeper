## Summary

Keep open. Current main still lacks an MS Teams outbound `sendPayload` path, and this PR targets that real Teams approval-card gap, but the latest patch still needs normal review before merge because its card-media path bypasses the existing outbound dependency seam.

## What This Changes

The PR adds MS Teams `sendPayload` support that renders interactive reply buttons or prebuilt `channelData.msteams.card` payloads as Adaptive Cards, wires the runtime channel adapter, and adds outbound unit tests.

## Best Possible Solution

Land a rebased Teams-plugin-owned `sendPayload` implementation that renders approval buttons as Adaptive Cards, preserves media, respects `ctx.deps`, and includes focused tests for no-button fallback, prebuilt-card media, interactive-card media with injected sender, and messageBack `/approve` delivery.
