## Summary

Keep this issue open. Current `main` still has the reported config contract mismatch, and the linked open PR is the active implementation candidate that should be reviewed or landed before this issue closes.

## What This Changes

The issue asks OpenClaw to accept or explicitly de-scope per-agent `verboseDefault` and `elevatedDefault` under `agents.list[]` so config validation matches the runtime/type surfaces.

## Best Possible Solution

Land PR #74643 or an equivalent narrow fix that aligns `AgentEntrySchema`, config metadata/help, public types, docs, and parser-level regression tests around the intended per-agent override contract.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: `agents.list` is validated through strict `AgentEntrySchema`, and that schema omits both reported keys, so config validation will reject them as unrecognized. I did not run the CLI because this review is read-only.

## Solution Assessment

Yes. Accepting the two keys in the per-agent schema and documenting their precedence is the narrowest maintainable fix if the existing runtime readers are intentional; otherwise the PR review should explicitly decide to remove the per-agent-facing runtime/type surface.
