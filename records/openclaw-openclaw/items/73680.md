## Summary

Keep this issue open. Current main still has the reported schema/runtime mismatch: `agents.list[]` is validated with a strict `AgentEntrySchema` that accepts `thinkingDefault`, `reasoningDefault`, and `fastModeDefault` but not `verboseDefault` or `elevatedDefault`, while runtime resolution already reads per-agent `verboseDefault` and several directive/status paths read `elevatedDefault`. Docs also only document `verboseDefault` and `elevatedDefault` under `agents.defaults`, not per-agent entries.

## Best Possible Solution

Fix the schema/runtime contract instead of closing: add `verboseDefault` and `elevatedDefault` to `AgentEntrySchema`, align `AgentConfig`/`ResolvedAgentConfig` typing for both keys, add focused schema tests proving `OpenClawSchema` accepts `agents.list[].verboseDefault` and `agents.list[].elevatedDefault`, and update the generated config schema/help plus `docs.openclaw.ai/gateway/config-agents` and elevated/verbose precedence docs to describe per-agent override precedence.
