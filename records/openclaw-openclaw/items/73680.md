## What This Changes

The issue asks OpenClaw to accept and document `verboseDefault` and `elevatedDefault` on `agents.list[]` entries, or remove the runtime/type surfaces that imply per-agent support.

## Best Possible Solution

Review and land PR #74643 or an equivalent narrow fix that aligns `AgentEntrySchema`, public config types, generated schema/help metadata, docs, and focused validation tests; then close this issue as implemented.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: `agents.list` is validated through strict `AgentEntrySchema`, and that schema omits the two reported keys while global defaults accept them. I did not run the CLI because this review is read-only.

## Solution Assessment

Yes. Accepting the two fields in the per-agent schema and documenting their precedence is the narrowest maintainable solution because the resolver, status, and directive paths already treat them as per-agent-capable values.
