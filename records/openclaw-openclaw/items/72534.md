## Summary

Keep this PR open. Current main still skips `skills/<group>/<skill>/SKILL.md` because workspace skill discovery only loads immediate child directories that already contain `SKILL.md`. The PR is still a valid implementation candidate for the linked open bug, and the supplied PR discussion shows maintainer repair plus merge-ready/human-review labels rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and review or land this branch, or an equivalent patch, as the canonical fix for #56915. The best implementation should add bounded one-level grouped discovery in `src/agents/skills/workspace.ts`, preserve direct-skill behavior and existing safety limits, and ship the regression tests plus docs/changelog update represented in the PR context.
