## Summary

Current main still uses shallow workspace skill discovery for ordinary skill roots, so the grouped layout this PR fixes is not implemented. The linked regression #56915 remains matched by current code: `skills/<group>/<skill>/SKILL.md` is skipped unless the group directory itself has `SKILL.md`. Prior discussion also points toward keeping this PR as the active candidate: Greptile found no correctness/security blocker, and a maintainer pushed a narrow repair to this branch to preserve the contributor path.

## Best Possible Solution

Keep this PR open for maintainer review and land it, or an equivalent patch, as the canonical fix for grouped skill discovery. The implementation should add bounded one-level discovery under configured skill roots, preserve direct skill behavior, keep containment and scan limits intact, and ship the regression tests plus docs/changelog update.
