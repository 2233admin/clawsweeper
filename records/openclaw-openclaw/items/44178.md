Keep this PR open. Current main still does not implement the PR's intended `feishu_doc` comment actions; the overlap is limited to `feishu_drive` comment listing/adding/replying, and prior review comments still flag a real pagination problem in the PR implementation. The diff is plugin-scoped, but it adds Feishu comment write/update/delete network capability, so it needs maintainer API and security review rather than automated cleanup.

## Best Possible Solution

Keep the PR open for maintainer review.
