## Summary

Keep this PR open. Current main has overlapping Feishu Drive comment support, but it does not implement the PR's intended `feishu_doc` comment actions or the full operation set, especially public get/update/delete comment actions. The PR also still needs correctness and security review because prior review flagged incomplete pagination and the diff adds new comment mutation capability.

## Best Possible Solution

Do not auto-close. Maintainers should either adapt the useful parts into the existing `feishu_drive` comment surface or intentionally add a compatible `feishu_doc` wrapper for document-comment workflows. Any updated PR should address pagination, avoid untyped SDK bypasses where possible, add tests for page tokens plus get/update/delete behavior, and document the required Feishu comment permission.

## Risks / Open Questions

- The PR adds Feishu comment write/update/delete capability and a new `docx:document.comment` permission, so it needs explicit maintainer review of permission scope and destructive-comment behavior before merge.
- The proposed implementation uses untyped SDK access in the provided diff (`client.docx as any`). Current main's Drive comment implementation uses explicit OpenAPI request helpers instead, so a revived implementation should avoid an untyped dependency bypass where possible.
- Directly merging this PR would likely duplicate or diverge from the newer `feishu_drive` comment API unless maintainers intentionally reconcile the two tool surfaces.
- The provided review comments identify incomplete pagination in the PR's `list_comments` path, which could silently omit comments on active documents.
