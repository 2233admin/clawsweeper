## Summary

Keep open. Current main does not implement the PR's direct `feishu_doc` comment actions; the shipped `feishu_drive` comment support overlaps but is a different surface and does not cover the PR's proposed get/update/delete actions. Existing review findings also identify pagination and return-shape issues, so this needs Feishu maintainer API/product review rather than cleanup closure.

## What This Changes

The PR adds Feishu `feishu_doc` schemas, implementation, skill documentation, and tests for listing, getting, creating, updating, and deleting document comments.

## Best Possible Solution

Pick one supported Feishu comment API surface. Either extend the existing `feishu_drive` comment actions to cover the missing operations, or accept direct `feishu_doc` actions after aligning naming and return shapes, handling pagination correctly, avoiding or documenting raw SDK bypasses, updating permission docs, and adding focused tests for pagination, update, delete, and block-scoped behavior.
