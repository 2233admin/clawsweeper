## Summary

Keep open. Current main has the broader chat model-ref fixes, but it does not fully implement this PR's agents overview picker change: the overview picker still builds configured options only from `agents.defaults.models`, so configured primary model refs can still fall back to a synthetic `Current (...)` option instead of a normal selectable provider-qualified option.

## Best Possible Solution

Refresh or reimplement the narrow remaining change on current main: make the agents overview picker build configured options from the same broader configured-model collector used for cron suggestions, preserve `agents.defaults.models` alias labels, and add the provider-qualified overview regression test. The chat-header part should be reconciled with the existing `chat-model-ref` and `chat-model-select-state` implementation rather than reapplied wholesale.
