## Summary

Current main still appears to have the reported gap: Slack advertises `download-file` and its dispatch/runtime paths require `fileId`, but the shared `message` tool schema does not expose a `fileId` parameter. PR #45626 is already open as a focused fix candidate and references this issue with closing syntax, so this issue should stay open until that PR or an equivalent fix merges.

## Best Possible Solution

Keep this issue open and review/land PR #45626 or an equivalent narrow fix that exposes `fileId` as an optional message-tool parameter, with tests covering the schema and Slack `download-file` dispatch/runtime path. Close as implemented only after that fix reaches main.
