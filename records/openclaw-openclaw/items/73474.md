## Summary

Keep this PR open. Current main still implements and documents a literal-loopback-IP-only Gateway control-plane proxy bypass, while this PR is a focused active contributor fix for the Windows/WSL inherited-proxy `localhost` case. It is not implemented on main and needs maintainer product/security review rather than cleanup closure.

## Best Possible Solution

Review this as an active Gateway/proxy bug-fix PR. If maintainers accept `localhost` as safe for the Gateway control-plane direct bypass, align the client and proxy guard on `isLoopbackHost`, preserve proxy env snapshot/restore semantics, add regressions for `ws://localhost` and env restoration, and update the public network proxy docs. If maintainers want to preserve the existing security contract, close the PR explicitly with the documented workaround of using `ws://127.0.0.1:18789` or `ws://[::1]:18789` for local direct Gateway connections.
