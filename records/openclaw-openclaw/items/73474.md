## Summary

Keep this PR open. Current main still deliberately limits the Gateway control-plane direct proxy bypass to literal loopback IP URLs, while this PR proposes broadening that boundary to `localhost` and clearing inherited proxy env during inactive managed-proxy state. The PR is focused and relevant to WSL/proxy reports, but it changes security-sensitive routing behavior and still needs maintainer/secops judgment, docs alignment, and explicit env restore regression coverage before merge.

## Best Possible Solution

Keep this PR open for maintainer/secops review. If maintainers accept `localhost` as safe for the Gateway control-plane bypass, align the client and proxy guard, preserve exact proxy env snapshot/restore semantics, add regressions for `localhost` plus inherited env restoration, and update the public network-proxy docs. If maintainers want to keep the current boundary, close the PR with the documented workaround of using `ws://127.0.0.1:18789` or `ws://[::1]:18789`.
