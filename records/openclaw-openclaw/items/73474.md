## Summary

Keep this PR open. Current main still documents and tests a literal-loopback-IP-only Gateway proxy bypass, so the `localhost`/WSL proxy case is not implemented on main. The PR is a focused contributor bug-fix candidate, with a related open WSL gateway report, and needs maintainer review rather than cleanup closure.

## Best Possible Solution

Review this as an active Gateway/proxy bug-fix PR. If maintainers accept `localhost` as safe for the Gateway control-plane direct bypass, align the client and proxy guard on one loopback-host helper, preserve proxy environment snapshot/restore semantics, add regressions for `ws://localhost` and env restoration, and update `docs/security/network-proxy.md`. If maintainers want to preserve the existing security contract, close the PR with the current documented workaround of using `ws://127.0.0.1:18789` or `ws://[::1]:18789` for local direct Gateway connections.
