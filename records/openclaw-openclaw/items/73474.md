## Summary

Keep this PR open. Current main still implements and documents a literal-loopback-IP-only Gateway control-plane proxy bypass, while this PR is an active focused attempt to fix WSL/Windows inherited proxy behavior for localhost Gateway connections. The current PR head appears to address the earlier Greptile defects, but it broadens a documented network-proxy security boundary and lacks the matching tests/docs needed before merge.

## Best Possible Solution

Keep this PR open for maintainer and secops review. If maintainers accept `localhost` as safe for the Gateway control-plane bypass, align the client and proxy guard on `isLoopbackHost`, preserve env snapshot/restore behavior, add regressions for `localhost` plus proxy env restoration, and update `docs/security/network-proxy.md` / the public network-proxy docs.
