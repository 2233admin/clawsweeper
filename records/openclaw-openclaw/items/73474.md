## Summary

Keep this PR open. Current main still has a literal-loopback-IP-only Gateway control-plane proxy bypass, while this PR is a focused attempt to include `localhost` and clear inherited proxy env during inactive managed-proxy state. The change is security-sensitive because it broadens the documented network-proxy bypass boundary, and the PR still needs maintainer/secops review plus matching docs and env-restore regression coverage before merge.

## Best Possible Solution

Keep this PR open for maintainer/secops review. If maintainers accept `localhost` as safe for the Gateway control-plane bypass, align the client and proxy guard, preserve exact proxy env snapshot/restore semantics, add regressions for localhost plus inherited-env restoration, and update the public network-proxy docs. If maintainers want to preserve the current boundary, close the PR with the documented workaround of using `ws://127.0.0.1:18789` or `ws://[::1]:18789`.
