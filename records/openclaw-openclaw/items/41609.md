## Summary

Keep open. Current main still does not parse direct inbound Feishu interactive-card message events. The shipped work on main improves fetched/read/quoted interactive-card parsing, and the merged maintainer PR explicitly scoped itself away from the broader inbound event path. The open follow-up PRs are also limited to `send.ts` fetched-message parser gaps and do not close this issue before merge.

## Best Possible Solution

Keep this issue open and land a focused Feishu plugin fix. Share or move the existing interactive-card parser into the live inbound path so `parseMessageContent()` or `parseFeishuMessageEvent()` handles direct `interactive` and `interactive_card` message types, preserves useful card/action JSON where appropriate, surfaces readable content through `RawBody` and agent context, adds direct inbound regression tests, and updates Feishu receive docs when support is real.
