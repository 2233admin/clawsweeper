## Summary

Keep open. Current main still does not parse direct inbound Feishu `message_type: "interactive"` / interactive-card message events into readable agent content. The existing interactive-card parser is on the fetched/read-message path, not the direct inbound event parser, and current docs still list interactive cards as send-only. The issue discussion also has a second user confirming the impact, and the provided related context shows open PR #56795 as a still-unmerged implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open and review, land, or replace #56795 with a focused Feishu plugin fix. The fix should add direct inbound `interactive` / `interactive_card` handling in the inbound event path, reuse or share the existing card parser where appropriate, preserve/fetch raw card JSON when needed, expose readable card content to `RawBody`/agent context, add targeted inbound tests, and update `docs/channels/feishu.md` once receive support is real.
