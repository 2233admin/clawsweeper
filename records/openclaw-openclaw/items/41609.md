## Summary

Keep open. Current main still does not implement direct inbound Feishu interactive-card parsing for this issue. The shipped parser improvements cover fetched/read/quoted message content, but the direct inbound event path still feeds `message_type: "interactive"` through `parseMessageContent()`, which has no interactive-card branch and falls back to the raw payload. Docs also still list interactive cards as send-only, and related open PR context shows this remains an implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open and review, land, or replace the focused Feishu inbound parsing PR. The fix should route direct inbound `interactive`/`interactive_card` message events through a shared card parser, preserve useful raw card JSON when available, populate readable `RawBody`/agent context, add targeted direct inbound tests in the Feishu bot path, and update the Feishu docs once receive support is real.
