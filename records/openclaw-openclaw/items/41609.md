## Summary

Keep open. Current main still does not implement direct inbound Feishu interactive-card parsing for this issue. The fetched/read/thread message path has an interactive-card parser, but `im.message.receive_v1` events still pass `message_type: "interactive"` through the bot event parser without an interactive-card branch, so card JSON can still reach agent context as raw/unreadable content.

## Best Possible Solution

Keep this issue open. The best path is to review, land, or replace the focused Feishu inbound parsing work: route direct `interactive` and `interactive_card` message events through a shared card parser, preserve useful raw card JSON where appropriate, ensure readable content reaches `RawBody`/agent context, add direct inbound bot-event tests, and update the Feishu docs once receive support is real.
