Keep open. This is a fresh WSL-specific v2026.4.26 loopback Gateway regression with concrete logs, and current main does not provide strong evidence that the reported bind-then-WebSocket-handshake-timeout path is fixed. The adjacent timeout-config and Windows/WSL proxy PRs are still open, and the earlier sidecar-startup report was closed after a non-repro on main rather than as a shipped WSL fix for this report.
