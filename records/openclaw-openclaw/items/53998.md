## Summary

Keep open. Current main still has the reported audit false-positive paths for bare `$VAR` values and auth-profile refs, and the mixed auth-profile type mismatch remains a fatal apply/configure error. The linked #54141 PR has open closing intent for this issue, so the issue should stay open until that PR or a replacement lands.

## What This Changes

The issue reports that `openclaw secrets audit --check` treats environment-variable secret references as plaintext and that `secrets configure` can abort on mixed auth profile types instead of leaving operators with a clean migration path.

## Best Possible Solution

Ship a focused fix that makes `secrets audit` distinguish supported env references from real plaintext across config and auth-profile stores, prevents auth-profile refs from double-reporting as plaintext, and gives `secrets configure/apply` a non-fatal path or clear scoped skip for OAuth profiles while preserving the documented SecretRef/OAuth boundary and scrub/rotation guidance.
