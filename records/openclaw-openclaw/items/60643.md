## Summary

Keep open. Current main does not contain this prompt work, and the PR needs changes before it is safe to merge because the new prompt text weakens existing memory-recall behavior and adds overly broad side-effect guidance for installs, commits, and file writes.

## What This Changes

The PR adds cognitive-processing prompt text to the agent system prompt, rewrites the memory-core prompt section and tests, and expands the MS Teams feedback reflection prompt and length test.

## Best Possible Solution

Revise the PR so cognitive framing is internal and concise, preserve the existing explicit memory retrieval trigger for search-enabled memory modes, and remove or narrow the install/commit/file-write guidance to stay approval-aware before running the targeted prompt tests.

## Reproduction Assessment

Yes for review purposes: compare the rendered prompt strings or the final PR diff. The memory regression is visible by rendering buildPromptSection with memory_search enabled and observing that the previous before-answer recall trigger is no longer present.

## Solution Assessment

No. The proposed direction may be useful, but this version is not the narrowest maintainable solution because it weakens recall instructions and overgeneralizes side-effect safety; the safer path is to layer the new framing without relaxing existing memory and approval contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep pressure naming internal:** `src/agents/system-prompt.ts:427`
  - body: The global system prompt says to name the pressure before responding. Without saying this is an internal self-check, ordinary replies can start exposing labels like speed or hedging, which conflicts with the existing concise user-facing narration style. Reword this to silently identify the pressure before answering.
  - confidence: 0.75
- **[P2] Restore the mandatory memory recall trigger:** `extensions/memory-core/src/prompt-section.ts:55-57`
  - body: The search-enabled memory branches now only describe what to do when searching memory; they drop the existing instruction to search before answering about prior work, decisions, dates, people, preferences, or todos. This makes memory use optional in the most important cases, so keep an explicit before-answer retrieval trigger in the rewritten prompt.
  - confidence: 0.84
- **[P2] Do not treat installs and commits as reversible:** `src/agents/system-prompt.ts:499`
  - body: This line tells agents to execute file writes, git commits, and installs without hesitation. Installs can run package lifecycle code and commits mutate repository state, so this weakens approval-aware tool behavior; split truly low-risk edits from package/code-execution and VCS operations.
  - confidence: 0.87
