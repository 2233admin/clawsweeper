## Summary

Keep open. The PR targets a real Feishu feedback gap, but the proposed patch is not safe to merge because it sends a visible Typing reaction before eligibility checks and outside the existing typing lifecycle/configuration, while related #69572 may change the desired API direction.

## What This Changes

The PR adds a direct Feishu `im.messageReaction.create` call in `handleFeishuMessage()` to add a `Typing` emoji reaction immediately after logging receipt of an inbound message.

## Best Possible Solution

If maintainers want immediate Feishu feedback, extend the existing Feishu typing lifecycle or native typing path after reply eligibility is known, behind `typingIndicator`, with cleanup and focused tests for disabled groups, allowlists, mention-required groups, and the disabled typing setting.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Move ack reaction after reply eligibility checks:** `extensions/feishu/src/bot.ts:327-336`
  - body: The new `Typing` reaction runs before disabled-group, group allowlist, sender allowlist, and mention-required early returns. Messages the bot is configured to ignore would still get a visible processing reaction with no reply, especially in mention-required group chats.
  - confidence: 0.93
- **[P2] Clean up the immediate typing reaction:** `extensions/feishu/src/bot.ts:330-334`
  - body: This direct create call does not keep the returned reaction id or pair it with a delete call. Feishu typing reactions persist until removed in the existing lifecycle, so eligible messages would keep a stale Typing reaction even after the bot replies.
  - confidence: 0.88
- **[P2] Honor the typingIndicator setting:** `extensions/feishu/src/bot.ts:330-334`
  - body: The new path unconditionally sends a Typing reaction, while the existing Feishu typing path skips reaction calls when `account.config.typingIndicator` is false. Users who disabled typing indicators would still get this API call and visible reaction.
  - confidence: 0.9
