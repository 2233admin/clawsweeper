## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Scope sidecar writes to one agent by default:** `extensions/memory-core/src/cli.runtime.ts:2255`
  - body: The sidecar write commands call `resolveAgentIds(cfg, opts.agent)`, which expands an omitted `--agent` to every configured agent. Since the command help says `--agent` defaults to the default agent, a plain `memory sidecar pin/status/salience/supersede` can unintentionally mutate multiple agents' sidecar databases. Use the single-agent resolver for mutating commands unless the operator explicitly requests fan-out.
  - confidence: 0.9
- **[P2] Open sidecar stats and list without schema writes:** `extensions/memory-core/src/cli.runtime.ts:2000`
  - body: `stats` and `list` are described as read-only, but their collectors call `openSidecarDatabase`, which runs schema initialization and writes the meta row or ALTERs an existing DB. That makes inspection mutate local state and fail on read-only filesystems. Add a read-only opener/probe path for these commands.
  - confidence: 0.86
- **[P2] Return non-zero when sidecar input is rejected:** `extensions/memory-core/src/cli.runtime.ts:2289`
  - body: Invalid or missing positional inputs log a warning and then return without setting `process.exitCode`, and unresolved or ambiguous ref prefixes are rendered as skipped writes without making the command fail. Scripts will treat those no-op mutations as success. Set a non-zero exit for rejected arguments and rejected write resolutions.
  - confidence: 0.84
- **[P2] Record shadow touches after the final result limit:** `extensions/memory-core/src/memory-v2/rerank/wrapper.ts:90-100`
  - body: The rerank wrapper records shadow touches by iterating the full pre-sort/pre-limit `results` array. `memory_search` applies the final sort and limit after the wrapper returns, so rows that were never surfaced can still get `last_accessed_at` refreshed and bias later recency rerank. Move shadow touching to the final surfaced result list or pass that list into the touch path.
  - confidence: 0.78
- **[P2] Make duplicate-location signal choice deterministic:** `extensions/memory-core/src/memory-v2/rerank/lookup.ts:31-35`
  - body: `loadSidecarSignalsByLocations` queries duplicate `location_id` rows without an `ORDER BY`, and `preferCandidate` has no final tie-break when salience and last-accessed values are equal. SQLite can return tied rows in different orders, so rerank may pick different pinned/status signals across runs. Add deterministic SQL ordering or a stable final tie-break.
  - confidence: 0.76

## Security Review

Status: needs_attention

Summary: The diff introduces persistent memory sidecar storage and mutating CLI paths with concrete data-integrity concerns that need maintainer attention before merge.

Concerns:

- **[medium] Unscoped sidecar writers can mutate unrelated agents:** `extensions/memory-core/src/cli.runtime.ts:2255`
  - body: Because mutating sidecar commands use the all-agent resolver when `--agent` is omitted, one operator command can change pin/status/salience/supersede state across multiple configured agents' local memory databases. That broadens the data mutation scope beyond the CLI contract.
  - confidence: 0.9
- **[low] Read-only inspections perform schema writes:** `extensions/memory-core/src/cli.runtime.ts:2000`
  - body: The stats/list path opens existing sidecar databases through the schema-initializing opener, so commands presented as read-only can write meta/schema changes. This matters for auditability and read-only/least-privilege operator contexts.
  - confidence: 0.86
