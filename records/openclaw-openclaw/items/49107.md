## Summary

Keep this PR open. Current main still does not implement the proposed non-interactive onboarding flags for trusted-proxy auth, trusted proxy IPs, trusted-proxy allowUsers, or Control UI allowed origins. The gateway config/schema and interactive configure flow already support the underlying trusted-proxy fields, so the remaining work is focused CLI/onboarding plumbing rather than obsolete or out of scope.

## Best Possible Solution

Keep this PR open or replace it with an updated focused PR that adds non-interactive onboarding support for `gateway.auth.mode="trusted-proxy"`, `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, `requiredHeaders`, `allowUsers`, and `gateway.controlUi.allowedOrigins`, with targeted tests and docs. Coordinate with #69066 for trust-model wording, but do not close this PR as implemented or superseded because the concrete CLI behavior is still missing on main.
