## Summary

Keep this PR open. Current main still lacks the standardized fs permission_denied helper and write/edit payload guards, so the work is not obsolete; however, the PR is a security-sensitive file-mutation hardening change with unresolved review blockers and a same-author dependent open PR #67332.

## Best Possible Solution

Keep this PR open and request changes. The maintainer path is to fix the classifier for "escapes allowed mounts", preserve path-miss/ENOENT semantics for sandbox reads, add focused regression coverage for both, decide whether the 2 MB and NUL-byte mutation limits are the desired product contract, and review it together with dependent #67332.

## Risks / Open Questions

- Landing the PR as-is may misclassify sandbox mount escapes as reason=unknown because "escapes allowed mounts" is not recognized by the new classifier.
- Landing the PR as-is may regress missing sandbox reads by surfacing E_FS_PERMISSION_DENIED instead of ordinary ENOENT/not-found behavior.
- The 2 MB and NUL-byte write/edit guards are plausible hardening, but they intentionally change file mutation behavior and need maintainer product/security review. The PR does not add third-party dependencies, CI workflows, release scripts, secrets access, or package resolution changes.
