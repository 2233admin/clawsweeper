## Summary

Keep this PR open. Current main and the latest release still support regex-based inline, same-line block, and bare TTS directive behavior, while this PR changes that public directive contract and bundles unrelated Slack/config-validation changes. A maintainer has already asked for the work to be split and reworked rather than merged as-is.

## What This Changes

This PR refactors TTS directive parsing to a line-by-line standalone directive scanner with new TTS contract tests, and also changes Slack Bolt socket-mode `ignoreSelf` handling plus config plugin validation known-ID handling.

## Best Possible Solution

Rework this into focused changes: first decide the intended TTS directive contract, then update the parser, contract tests, unit tests, runtime prompt hints if needed, public TTS docs, and changelog together; split any Slack or config-validation fixes into separate PRs with their own validation.
