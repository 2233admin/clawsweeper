## Summary

Keep this PR open. Current main still lacks the Feishu legacy `botName` migration: the account schema is strict and supports `name`, runtime account resolution reads only `name`, and Feishu does not expose a doctor compatibility normalizer through either its contract API or channel plugin. The linked regression #65177 remains open, and the PR is a focused Feishu-owned fix; the review comments show the arrow-style nit and blank-name edge case were both addressed on the PR branch.

## What This Changes

The PR adds a Feishu doctor migration that rewrites `channels.feishu.accounts.<id>.botName` to `name`, exports the doctor contract, wires it into the Feishu channel plugin, and adds channel legacy migration test coverage.

## Best Possible Solution

Land this PR or an equivalent Feishu-owned doctor migration after validation. The finished fix should stay inside the Feishu plugin boundary, expose `legacyConfigRules` and `normalizeCompatibilityConfig`, wire the Feishu channel doctor adapter, and cover normal migration, existing-name conflicts, and blank-name handling; after merge, #65177 can close as implemented.
