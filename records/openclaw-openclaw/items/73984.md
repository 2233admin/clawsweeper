## Summary

Keep this PR open. Current main still does not hydrate TUI slash-command autocomplete from the runtime command inventory: TUI autocomplete is built from `getSlashCommands(...)`, `TuiBackend` has no `listCommands` seam, and `src/tui` has no `commands.list` consumer. The PR remains a relevant implementation candidate for surfacing plugin commands such as `/active-memory` in TUI autocomplete.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to review the current `commands.list` hydration design for both Gateway and embedded TUI modes, verify the forced-refresh race fix, preserve built-in argument completions on command name collisions, run the targeted TUI tests, manually confirm `/active-memory` appears after typing `/act`, and add a user-facing changelog entry if the PR lands.
