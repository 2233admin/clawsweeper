## Summary

Keep this PR open. Current main still builds TUI slash autocomplete from the local `getSlashCommands(...)` path and has no `TuiBackend.listCommands` seam, while the existing Gateway `commands.list` inventory already includes plugin commands such as `/active-memory`. The PR targets the right surface and remains a focused implementation candidate; it should go through normal maintainer review and targeted TUI validation rather than be closed by cleanup.
