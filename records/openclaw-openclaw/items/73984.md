## Summary

Keep this PR open. Current main still builds TUI slash autocomplete from the static local helper and has no TUI command-list backend seam, while the existing Gateway command inventory already includes plugin commands; the PR targets that gap directly, and the latest head appears to address the earlier stale forced-refresh review concern.

## What This Changes

The PR adds TUI backend command-listing for gateway and embedded modes, merges remote command inventory into slash-command autocomplete, refreshes it across agent/model/session/connect changes, and updates focused TUI tests.

## Best Possible Solution

Review and land this focused TUI hydration path after CI rerun and targeted TUI validation, keeping local built-in argument completions ahead of remote command-name collisions and preserving the existing Gateway `commands.list` contract as the source of runtime command inventory.

## Solution Assessment
