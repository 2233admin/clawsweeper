## Summary

Keep open. Current main still does not hydrate TUI slash-command autocomplete from the runtime command inventory: the TUI builds suggestions from `getSlashCommands(...)`, `TuiBackend` has no `listCommands` seam, and `src/tui` has no `commands.list`/`listPluginCommands` usage. The runtime inventory already includes plugin commands, including Active Memory, so this open PR is still a relevant implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to review the latest `commands.list` hydration design for both Gateway and embedded TUI modes, verify the forced-refresh race fix with a regression test or convincing existing coverage, preserve built-in argument completions on command name collisions, run the targeted TUI tests, perform the manual `/active-memory` autocomplete check, and add a user-facing changelog entry if the PR lands.
