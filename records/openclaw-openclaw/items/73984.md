## Summary

Keep open. Current main still does not hydrate TUI autocomplete from runtime plugin commands, while this PR is a focused implementation with green latest check-runs and no blocking review findings from this read-only pass.

## What This Changes

The PR adds TUI command-list hydration for gateway and embedded modes, merges runtime command entries into slash-command autocomplete, refreshes that inventory across command-context changes, and updates focused TUI tests.

## Best Possible Solution

Review and land this focused TUI hydration path if maintainers are satisfied with the behavior, keeping commands.list as the runtime command inventory and preserving built-in argument completions on local/remote name collisions.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence reproduction path: enable a plugin command such as active-memory, run the TUI, type a prefix like /act, and autocomplete is still built from the static/shared chat-command list rather than commands.list/plugin command inventory.

## Solution Assessment

Yes. Reusing commands.list in gateway mode and adding an embedded backend equivalent is the narrow maintainable fix; it avoids duplicating plugin-command discovery inside the autocomplete layer while preserving local built-in command behavior.
