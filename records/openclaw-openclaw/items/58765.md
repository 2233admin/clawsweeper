## Summary

Current main accepts `outputDimensionality` in memory-search config and carries it into provider creation, but the built-in local GGUF provider does not read or apply it. Local embeddings are returned at full vector length after normalization, local runtime cache keys do not include dimensions, and the docs/tests only cover local model path/cache/context settings. The requested local truncation and reindex behavior remains unimplemented.

## Best Possible Solution

Keep this open and implement local GGUF dimensionality truncation in the memory embedding path. A focused fix should add a shared helper or local-provider handling for positive `outputDimensionality`, apply it consistently to `embedQuery` and `embedBatch`, include the resolved dimension in the local provider runtime cache key so dimension changes force a full reindex, and update the local memory-config docs plus tests. Treat `queryInstruction` as a separate follow-up unless maintainers want to expand the same config surface now.

## Risks / Open Questions

- Implementation needs a deliberate vector operation order: for cosine search, local truncation should likely slice first and then normalize the truncated vector rather than normalize the full vector and then slice.
- Dimension changes must be included in provider/cache metadata so existing indexes are rebuilt instead of mixing vectors with incompatible lengths.
- The bonus `queryInstruction` request is adjacent but separable; it should not block the core local-dimension truncation fix.
