## Summary

Keep open. Current main at a820a307dfeb950fa633c0ae6fb2386a35a91991 still does not implement per-spawn tool restrictions for `sessions_spawn`: the tool schema, native spawn params, child session metadata, session schema, policy resolver, tests, and docs only support global `tools.subagents.tools` plus depth/session-role restrictions. The Claw Drive comment and closed unmerged PR #23506 remain useful implementation context, while #23906 is a related open feature request rather than a shipped replacement.

## Best Possible Solution

Keep this issue open as the canonical `sessions_spawn` core API/security gap. Implement an additive per-spawn policy parameter, likely `tools: { allow?: string[]; deny?: string[] }` plus consideration for `tools: "none"`; thread it through schema parsing, `SpawnSubagentParams`, immutable child-session metadata, `SessionEntry`/protocol schema, and `resolveSubagentToolPolicyForSession`; add focused unit/e2e coverage; update https://docs.openclaw.ai/tools/subagents.
