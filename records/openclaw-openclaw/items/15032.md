## Summary

Current main still does not implement per-spawn tool restrictions for native `sessions_spawn`. The current schema has no `tools`, `allow`, `deny`, `allowTools`, or `denyTools` parameter; native spawn params do not carry a policy; child session metadata only persists depth/role/control/model state; and the subagent tool-policy resolver still reads only global `tools.subagents.tools` plus stored subagent role. The linked Claw Drive use case and closed unmerged PR #23506 remain useful implementation context, while #23906 is a related open feature request rather than a shipped replacement.

## Best Possible Solution

Keep this open as the canonical native `sessions_spawn` per-spawn tool-policy gap. The best implementation is an additive, backward-compatible API such as `tools: { allow?: string[]; deny?: string[] }` plus a considered `tools: "none"` path if maintainers want to cover #23906 at the same time. Thread the policy through `createSessionsSpawnToolSchema`, `SpawnSubagentParams`, immutable child session metadata and protocol schema, and `resolveSubagentToolPolicyForSession`; then add focused unit/e2e tests and update https://docs.openclaw.ai/tools/subagents.
