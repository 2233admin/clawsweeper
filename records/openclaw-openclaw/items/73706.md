## Summary

Keep this PR open. Current main still omits `sessionKey` from the shared outbound `message_sending` hook context even though that value is already available in `deliverOutboundPayloadsCore` and the plugin hook context type already supports it. The PR is a focused additive implementation candidate with tests, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the focused shared-outbound fix after normal validation, and decide during review whether to extend the same `sessionKey` threading to the Telegram direct delivery path or track that as a separate follow-up.
