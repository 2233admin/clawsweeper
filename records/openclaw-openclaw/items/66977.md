## Summary

Keep open. Current main still opens builtin memory with `node:sqlite` and loads `sqlite-vec` through `enableLoadExtension()`/`loadExtension()`, so the reported macOS runtime boundary is not fixed. Main does mitigate part of the user-visible damage by storing embeddings and falling back to in-process cosine ranking, and the QMD discussion/workaround is now reflected in QMD docs, but that is degraded behavior rather than native sqlite-vec support for builtin memory on macOS.

## Best Possible Solution

Keep this issue open until maintainers either implement a supported builtin macOS sqlite-vec load path using a SQLite runtime that permits extensions, or explicitly document native sqlite-vec acceleration as unsupported on affected macOS Node runtimes while preserving the current in-process cosine fallback as degraded behavior.
