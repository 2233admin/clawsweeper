## Summary

Keep open. Current main still routes builtin memory through node:sqlite and the sqlite-vec loadable-extension path, with degraded fallback when loading fails; the code and docs do not contain a macOS-specific native sqlite-vec repair or a supported workaround for the reported Node/SQLite runtime limitation. The related unmerged PR #74435 was narrower than the reported compile/runtime boundary and does not supersede this tracker.

## What This Changes

The issue asks OpenClaw to make sqlite-vec vector search work on macOS under node:sqlite, or to provide a supported workaround/documented limitation instead of silently degrading recall.

## Best Possible Solution

Keep this as the canonical macOS sqlite-vec/node:sqlite tracker until maintainers either ship a SQLite loading strategy that supports sqlite-vec on macOS or explicitly document the macOS limitation and supported workaround while preserving the fallback as degraded behavior.
