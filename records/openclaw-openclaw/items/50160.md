## Summary

Keep open. Current main does not implement this PR's stated Discord delivery behavior: both the bot send-result helper and webhook send path still return messageId "unknown" when Discord returns no message id, and the cron/direct delivery layer still treats any returned delivery result as a successful delivery. The PR also should not be merged as-is because the provided review context flags an incomplete webhook fix, a duplicate const compile blocker in the PR branch, and an unrelated schtasks reliability regression; the branch also bundles unrelated Matrix, model-selection, Ollama, Windows daemon, and Discord changes.

## Best Possible Solution

Keep this PR open for maintainer handling, but do not merge it as-is. The best path is a fresh, scoped Discord plugin fix in extensions/discord/src/send.outbound.ts that defines the expected semantics for missing Discord ids, covers both bot and webhook sends, accounts for webhook wait=false behavior, and adds regression tests showing cron/direct delivery reports failure instead of delivered when Discord returns no usable message id.
