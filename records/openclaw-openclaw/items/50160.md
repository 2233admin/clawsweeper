## Summary

Keep this PR open. Current main still does not implement the PR's stated Discord missing-message-id behavior, so the work is not obsolete. The PR should not merge as-is because it targets the old `src/discord` path while current Discord code lives under `extensions/discord`, leaves webhook/component send surfaces with the same `"unknown"` fallback, and bundles unrelated Matrix, agent model-selection, Ollama logging, and Windows service-management changes.

## Best Possible Solution

Revise or replace this PR with a fresh, scoped Discord plugin patch under `extensions/discord`. The best fix should define the missing-message-id contract, cover bot, webhook, and component send paths, handle Discord webhook `wait=false` semantics deliberately, and add regression coverage proving failed Discord delivery is not recorded as delivered by cron/direct outbound delivery.
