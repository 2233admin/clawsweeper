## Summary

Keep this PR open. Current main still has the central cleanup-hint bug: doctor and gateway status report detected extra services but render generic cleanup commands for the current profile instead of the actual detected service. The SecretRef half is only partially adjacent to current behavior, and the PR needs a refresh against the current doctor-health flow plus the existing review concerns before it is safe to close or land.

## Best Possible Solution

Refresh the PR against current main, keep the #52510 fix paired with this PR, and implement the cleanup-hint change as a service-specific contract used by both doctor and gateway status. Add focused regression tests for active gateway protection, status cleanup hints, legacy-service manual guidance, and user/system scope cleanup commands. For SecretRefs, prefer the existing inspect/warning diagnostics pattern and keep fail-closed runtime semantics intact.
