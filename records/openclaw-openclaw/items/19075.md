## Summary

Keep open. Current main still has the Brave Search runtime pinned to fixed Brave API URLs, the Brave plugin config schema and UI hints expose only apiKey and mode, and the docs do not document a Brave baseUrl. Related implementation attempts, including #18167 and #45843, are closed/unmerged, while live issue search found no open PR for this tracker or Brave baseUrl. A maintainer comment on April 26 also explicitly kept this issue open as the canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical Brave Search baseUrl tracker. Implement the option in the Brave plugin-owned config surface, preferably plugins.entries.brave.config.webSearch.baseUrl, update manifest UI hints/schema/docs, normalize and validate the URL, derive both /res/v1/web/search and /res/v1/llm/context from it while preserving the default Brave endpoint, include the resolved base in cache keys, and add focused runtime/config-schema tests.
