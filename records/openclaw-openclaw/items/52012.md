## Summary

Keep this PR open. Current main still does not implement `agents.defaults.compaction.modelFallbacks` or an equivalent compaction-specific fallback path. The linked issue was previously closed as superseded by this PR, but the PR itself remains the canonical implementation candidate and is not obsolete or implemented on main. The supplied review context also shows substantive late P1/P2 concerns in the PR diff, so the right path is request changes or supersede with a tighter PR, not conservative cleanup closure.

## Best Possible Solution

Keep the PR open but require changes before merge: add `agents.defaults.compaction.modelFallbacks` consistently to the public type, strict schema, generated/help metadata, docs, and direct/queued compaction paths; preserve existing prompt/runtime/checkpoint behavior; and add focused tests for unset behavior, model resolution fallback, auth fallback, runtime compaction failure fallback, and all-fallbacks-fail diagnostics. If maintainers prefer a smaller implementation, land a narrower replacement and then close this PR as superseded.
