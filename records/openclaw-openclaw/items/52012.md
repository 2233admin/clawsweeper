## Summary

Keep this PR open. Current main still lacks `agents.defaults.compaction.modelFallbacks` or an equivalent compaction-specific fallback path, and related issue #52011 was closed as superseded by this still-open PR. The PR is not obsolete; it needs maintainer review or a narrower replacement because the implementation touches security-sensitive compaction auth and provider transport behavior.

## Best Possible Solution

Keep the PR open, but require maintainer review and likely a smaller or repaired implementation. The best path is to add `agents.defaults.compaction.modelFallbacks` consistently across public types, strict schema, generated/help metadata, docs, direct compaction, and queued/context-engine compaction, while preserving existing checkpoint, prompt override, heartbeat, stream wrapper, auth-profile, runtime-auth exchange, and thinking-level fallback behavior. Add focused tests for unset behavior, resolution fallback, auth fallback, runtime compaction failure fallback, all-fallbacks-fail diagnostics, and current behavior that must not regress.
