## Summary

Keep this PR open. Current main still only clears `lastToolError` when the successful mutating call has the exact same action fingerprint, and the current regression test still asserts that a same-tool success on a different target keeps the prior `write` error. The PR remains a focused implementation candidate for changing that product behavior, not something already implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether a successful same tool/action-type retry on a different target should clear the earlier warning; if yes, land a narrowed implementation with the added regression tests, and if no, close it with an explicit rationale that exact target fingerprints are the intended warning contract.
