## Summary

Keep open. Current main still routes Feishu scan-to-create through compact terminal QR output, the latest shipped release still has the same compact setting, and the documented onboarding path still depends on scanning that QR with the Feishu/Lark mobile app. The only linked fix candidate, PR #67450, was closed unmerged and also had review concerns, so there is no merged implementation that resolves the reported Windows onboarding failure.

## What This Changes

The issue reports that the Feishu QR code shown during `openclaw onboard` on Windows 10 cannot be scanned by the Feishu mobile app, preventing Feishu channel setup.

## Best Possible Solution

Fix this in the Feishu plugin-owned setup path by making scan-to-create output reliably scannable while preserving narrow-terminal behavior, preferably with larger QR rendering when it fits and a clear fallback when terminal QR scanning fails. Add focused regression coverage around Feishu QR render options and keep user-facing docs or changelog aligned if the fallback changes the setup experience.
