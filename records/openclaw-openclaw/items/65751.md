## Summary

Current main still sends the Slack setup manifest through the framed wizard note path, so the reported copy/paste bug remains. The issue should also stay open because open PR #65773 explicitly targets it with closing syntax and needs review or replacement before the issue is resolved.

## What This Changes

The issue asks for `openclaw channels add` Slack setup to output the Slack app manifest as copyable JSON without ASCII note/frame characters.

## Best Possible Solution

Keep the issue open and review #65773 or a replacement that moves the manifest to a copyable prompter-visible output path for both CLI and session-backed wizards, preserves env-shortcut suppression, and adds focused Slack setup regression coverage.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce with high confidence: Slack setup builds the manifest into `introNote.lines`, the shared setup runner calls `prompter.note`, and the CLI note implementation uses clack's framed note output.

## Solution Assessment

No, the issue is not solved on main yet. The narrow maintainable fix should separate the manifest from the framed note through a wizard/prompter-visible plain or copyable output path rather than closing the issue before the linked fix PR lands.
