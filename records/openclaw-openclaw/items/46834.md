## Summary

Keep open. Current main still does not implement this PR's intended direct final visible ACP output delivery for non-threaded direct chats. The current relay emits system events/progress/completion notices, and ordinary successful ACP output does not become a task terminal summary. The PR remains functionally relevant, but it needs a current-main rework and maintainer review because it changes direct channel delivery of ACP child output.

## Best Possible Solution

Keep this PR open for maintainer review or rework against current main. Choose one delivery owner for final ACP child results: either extend the task-completion announce path to carry the final visible result safely, or add a direct parent-relay path with duplicate suppression, current sanitization/parent-rewrite policy, non-threaded direct-chat coverage, threaded exclusion, error/truncation tests, and whitespace-preserving delta accumulation.

## Risks / Open Questions

- Applying the PR as-is may duplicate or conflict with the newer task terminal notification path unless ownership and idempotency are designed explicitly.
- Direct raw ACP child output delivery through routeReply could bypass current parent rewrite, sanitization, silent-delivery, or channel routing policy.
- The provided review comments flagged whitespace-only delta corruption and best-effort startup behavior; startup appears addressed by the second PR commit, but whitespace-preserving accumulation still needs verification if this approach is revived.
- Current docs describe a parent rewrite completion path, while current code/tests show ordinary successful ACP output is not carried as terminalSummary; maintainers should reconcile intended behavior before closing or merging.
