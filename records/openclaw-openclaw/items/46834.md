## Summary

Keep open. Current main has ACP task-completion notifications and a parent progress relay, but it still does not implement this PR's intended non-threaded direct-chat delivery of the child agent's final visible ACP output. The current behavior is generic task terminal delivery, while raw ACP child output is deliberately kept out of direct run-mode channel delivery.

## Best Possible Solution

Keep this PR open for maintainer review or a current-main rework. The clean path is to choose one owner for ACP completion delivery: either extend task completion to carry the final visible ACP result safely, or add a direct parent-relay path with duplicate suppression, current sanitization policy, whitespace-preserving delta accumulation, threaded-context exclusion, error handling, truncation coverage, and regression tests for non-threaded direct chats.
