## Summary

Keep this PR open, but do not merge it as-is. The underlying ACP direct-chat completion gap is still a real product question, but the branch adds a raw direct `routeReply` path that conflicts with current main's task-completion notifier and drops whitespace-only streaming chunks in the new direct buffer.

## What This Changes

The PR adds routeReply-based final/error direct delivery from the ACP parent stream relay for non-threaded parent sessions, with tests for direct-chat success, threaded suppression, lookup failure, and error delivery.

## Best Possible Solution

Choose one owner for ACP completion delivery. Prefer extending the existing task-completion or parent handoff path to carry the final visible child result with sanitization, thread policy, and duplicate suppression; if maintainers explicitly want a direct relay, rework it to preserve streamed whitespace, suppress the generic terminal notification for the same run, use the current relay delivery context, and cover success, error, truncation, threaded, and duplicate paths in tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve whitespace-only assistant deltas:** `src/agents/acp-spawn-parent-stream.ts:367`
  - body: The new direct-reply buffer is appended after the existing `!delta.trim()` early return, so standalone space or newline chunks are discarded before direct delivery. Current assistant event emitters can derive deltas from `cleanedText.slice(...)`, so a stream split like `"hello"`, `" "`, `"world"` can become `"helloworld"` in the user-visible final reply.
  - confidence: 0.86
- **[P2] Suppress terminal task delivery before direct sending:** `src/agents/acp-spawn-parent-stream.ts:396-400`
  - body: Current main already sends the same ACP run's terminal task notification to the requester origin. Adding this `routeReply` on lifecycle end sends the raw child result while the task registry can still send the generic completion message, so non-threaded chats can receive duplicate completion messages for one child run.
  - confidence: 0.8
