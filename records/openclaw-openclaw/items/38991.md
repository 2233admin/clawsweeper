## Summary

Keep this PR open. Current main still does not implement the intended legacy cron timestamp backfill: cron store loading can synthesize `updatedAtMs`, but it still does not synthesize or sanitize missing `createdAtMs`, while `cron.list` and `listPage` return stored jobs directly. The PR is still relevant, but it needs rebase/scope cleanup because its branch also contains unrelated secret-baseline and iOS Fastfile changes.

## Best Possible Solution

Keep this PR open or replace it with an equivalent scoped patch. Rebase onto current main, implement finite non-negative `createdAtMs` and `updatedAtMs` backfill during cron store/service load, persist only when legacy storage is actually mutated, preserve legacy job visibility through `cron.list` and `listPage`, and update tests for both the in-memory response and the split persisted files. Drop or split the unrelated secret-baseline and iOS Fastfile changes before merge.
