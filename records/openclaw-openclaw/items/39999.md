## Summary

Keep open. Current main has staged global package update hardening and post-update doctor execution, but it still does not implement this PR's intended previous-version reinstall rollback when doctor fails after a package update. The PR is plausible core update-safety work, but it needs rebasing into the shared package update helper and should address the review-noted rollback guard gap.

## Best Possible Solution

Keep this PR open for contributor or maintainer revision. The best implementation is to move rollback support into `runGlobalPackageUpdateSteps`, carry the known `beforeVersion`, attempt `packageName@beforeVersion` rollback only after a successful live replacement/direct install followed by post-update doctor failure, rerun doctor when the restored entrypoint exists, preserve the overall error result, and add focused helper/CLI tests for rollback and no-rollback cases.
