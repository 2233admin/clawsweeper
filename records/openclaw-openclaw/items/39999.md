## Summary

Keep this PR open. Current main has staged global package update hardening and post-update doctor execution, but it still does not implement the PR's intended previous-version reinstall rollback when doctor fails after a package update. The proposed direction is plausible, but the current PR needs revision around the rollback guard and likely belongs in the shared package-update helper rather than only the CLI wrapper.

## Best Possible Solution

Keep this PR open for contributor or maintainer revision. The best implementation is to add rollback support at the shared package-update-helper boundary, carry the known previous version, attempt `packageName@beforeVersion` rollback only after a successful package replacement followed by post-update doctor failure, rerun doctor when the restored entrypoint exists, preserve the overall error result, and add focused tests for rollback and no-rollback cases. Update docs/changelog only if the behavior ships.
