## Summary

Keep open. Current main still only partially hardens deleted-agent sessions; it does not provide a first-class agent reset path, does not clean cron jobs that target the deleted agent, and does not route agent deletion through the broader session/plugin cleanup lifecycle needed for plugin or context-engine-owned state.

## What This Changes

Request that deleting or resetting an agent clear all related session, cron, and plugin/context-engine state and surface recoverable session-corruption errors.

## Best Possible Solution

Keep this open until agent deletion or a new agent reset command performs a coherent cleanup plan: enumerate affected sessions through the session mutation lifecycle, clean or explicitly report cron jobs for the agent, and give plugin/context-engine owners a documented cleanup hook for session-owned persistence. Add regression coverage for CLI deletion, Gateway deletion, cron job references, plugin cleanup, and the reported corruption path.
