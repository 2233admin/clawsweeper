## Summary

Keep open. Current main includes a useful partial fix for stale agent-scoped sessions after deletion, but it still does not provide an agent-scoped reset command, route agent deletion through the shared session/plugin cleanup lifecycle, or remove/disable/report cron jobs that still target the deleted agent.

## What This Changes

The issue asks for agent deletion or a new agent reset path to clear all persistent state tied to an agent, including session files, session-store/plugin state, and cron job references.

## Best Possible Solution

Keep this open until agent deletion or a new agent reset command has a coherent cleanup contract: enumerate/delete or reset the affected sessions through the shared session mutation lifecycle, remove/disable/report cron jobs that target the deleted agent, and invoke a documented plugin/context-engine cleanup hook for agent-owned persistent state. Add regression coverage for CLI deletion, Gateway deletion, cron references, plugin cleanup, and the reported recovery path.
