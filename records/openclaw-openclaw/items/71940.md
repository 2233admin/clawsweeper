## Summary

Keep open. Current main still routes normal browser screenshots through CDP and has no Hyprland/grim fallback, so the PR is not obsolete; however, the latest head still has merge-blocking issues in the new Hyprland setup cleanup and Plugin SDK API baseline, and the PR body still points at an unrelated Windows Gmail issue.

## What This Changes

The PR branch adds an opt-in Hyprland/Wayland `grim` viewport screenshot fallback for headed browser sessions, plus browser config/schema/docs/tests and a new `openclaw/plugin-sdk/resolve-system-bin` subpath.

## Best Possible Solution

Revise the PR in place: keep the Hyprland fallback tied to the headed viewport screenshot bug, make virtual-output removal symmetric for every setup/capture failure, regenerate and commit the Plugin SDK API baseline for the new subpath or avoid adding the subpath, update the PR body to close the Hyprland bug instead of the Windows Gmail bug, and then run the targeted browser/config/SDK checks plus the changed gate in Testbox.

## Reproduction Assessment

Yes. The related Hyprland bug has concrete steps on Omarchy/Hyprland/Wayland with headed Chromium and specific pages, and the setup-cleanup defect is reproducible by source tracing a successful `create-output` followed by a failing monitor lookup or workspace dispatch.

## Solution Assessment

No. The feature direction is plausible and current main lacks it, but this patch is not yet the best mergeable form because it can leak a virtual output during setup failure, leaves the Plugin SDK API baseline stale, and carries a stale closing issue/body mismatch.
