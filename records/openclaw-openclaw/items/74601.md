## Summary

Keep open. Current main still implements Skill Workshop as a workspace-skill-only bundled plugin, and the requested shared-skill targeting plus cross-workspace promotion remains absent. The request is valid, but it expands write authority and reviewer scope across agents, so it needs maintainer product and security direction before it is a safe automated repair lane. The related #35272 and #15717 context is orthogonal: those cover shared-skill visibility and per-agent allowlists, not workshop-driven shared-skill modification.

## What This Changes

Add opt-in Skill Workshop support for shared managed/local skill proposals, cross-workspace pattern promotion, and an explicit promotion action while keeping safety gates in place.

## Best Possible Solution

Stage this inside `extensions/skill-workshop`: first add a narrowly scoped, opt-in shared-target model with separate target metadata/storage, strict root validation, approval-only shared applies, docs, and focused tests; then evaluate cross-workspace semantic promotion after privacy and evidence boundaries are defined.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main check is high-confidence: the tool schema, config, reviewer, store, docs, and write path all remain workspace-only.

## Solution Assessment

Unclear until maintainer review. A small `promote_to_shared` or explicit `target` v1 may be the maintainable first step, while the periodic cross-workspace reviewer needs product and security design before implementation.
