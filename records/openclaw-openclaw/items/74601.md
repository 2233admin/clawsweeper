## Summary

Keep open. Current main still scopes Skill Workshop proposals, storage, reviewer context, tool actions, and writes to workspace skills only, while the requested shared-skill targeting and cross-workspace promotion would expand a bundled experimental plugin across shared write and privacy boundaries that need maintainer product/security direction.

## What This Changes

Request to let the Skill Workshop plugin target shared skills and promote repeated workspace skill proposals into shared skill candidates through opt-in config, explicit actions, and approval gates.

## Best Possible Solution

Stage this inside `extensions/skill-workshop`: first design explicit shared-target metadata, strict shared-root validation, separated proposal storage, manual-only shared applies, docs, and focused tests; then handle cross-workspace semantic promotion as a follow-up after the privacy model is settled.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Static inspection gives high confidence in the current limitation because the source, docs, tests, tool schema, reviewer, and store are all workspace-scoped on current main.

## Solution Assessment

Unclear. Shared targeting is plausible, but the proposed A/B/C package combines a narrow target expansion with broader cross-workspace review, so the maintainable path should split those pieces and get maintainer product/security approval first.
