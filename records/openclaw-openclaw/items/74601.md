## Summary

Keep open. Current main still scopes Skill Workshop proposals, review context, proposal state, and file writes to workspace skills only, while the request would add shared-skill write authority and cross-workspace semantic review that need maintainer product and security direction before a narrow repair lane is safe.

## What This Changes

The issue requests opt-in Skill Workshop support for proposing edits to shared skills, promoting workspace skills to shared candidates, and detecting cross-workspace skill patterns.

## Best Possible Solution

Stage the work inside `extensions/skill-workshop`: first define an explicit shared-target model with target metadata/storage, root validation, manual-approval-only shared applies, docs, and focused tests; then consider cross-workspace semantic promotion as a separate follow-up after privacy and evidence limits are settled.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The missing capability is high-confidence by static inspection: current main has no shared target config, no tool `target` or `promote_to_shared` action, and writes only under the active workspace skills directory.

## Solution Assessment

Unclear as a complete implementation plan. The proposed direction is plausible, but the safest maintainable path is a staged design rather than one automated patch that combines shared writes, manual promotion, and cross-workspace LLM review.
