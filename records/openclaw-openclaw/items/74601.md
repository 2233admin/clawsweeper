## Summary

Keep open. Current main still implements Skill Workshop as a workspace-skill-only plugin, and the requested shared-skill targeting plus cross-workspace promotion is a real missing capability rather than a duplicate or already-shipped behavior. Because it would let one agent propose or apply changes to skills visible to other agents and would add cross-workspace review, this needs maintainer product and security direction before it is a safe automated repair lane.

## What This Changes

Request opt-in Skill Workshop support for proposing updates to shared skills, detecting cross-workspace skill patterns, and promoting workspace skills into shared skill candidates.

## Best Possible Solution

Keep this as a maintainer-designed Skill Workshop enhancement: first define an explicit shared-target model with separate target metadata/storage, root validation, manual-approval-only shared applies, docs, and focused tests; then treat cross-workspace semantic promotion as a follow-up after privacy and evidence limits are agreed.

## Reproduction Assessment

Not applicable as a feature request. The current-main source check gives a high-confidence verification path for the gap: the tool schema has no shared target and all writes/review context resolve through `<workspace>/skills`.

## Solution Assessment

Unclear. The requested direction is plausible, especially the staged opt-in model, but the full A/B/C proposal is not yet the narrowest maintainable solution until maintainers settle the shared-write and cross-workspace review security model.
