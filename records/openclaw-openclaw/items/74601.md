## Summary

Keep open. Current main still scopes Skill Workshop proposals, review context, storage, and writes to workspace skills only, while this request intentionally expands a bundled experimental plugin into shared-skill writes and cross-workspace review. That is plausible product work, but it needs maintainer security/product design before it is a safe cleanup close or automated repair lane.

## What This Changes

Request optional Skill Workshop support for shared-skill targeting, cross-workspace pattern promotion, and a promote-to-shared workflow for applied workspace skills.

## Best Possible Solution

Stage this inside `extensions/skill-workshop`: first design an explicit shared-target model with target metadata/storage, root validation, manual-approval-only shared applies, docs, and focused tests; then treat cross-workspace semantic promotion as a separate follow-up after privacy, similarity, and evidence limits are agreed.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current limitation is high-confidence from static inspection: writes, reviewer context, tool schema, proposal storage, and docs are all workspace-scoped on current main.

## Solution Assessment

Unclear. The requested direction is plausible, but the maintainable path is likely staged: a minimal explicit shared-targeting/promote design first, with cross-workspace semantic promotion deferred until the security and privacy model is settled.
