## Summary

Keep this PR open. Current main still uses slug-only local-file source identities in memory-wiki ingest, so distinct files with the same normalized title still target the same source page; the PR is a focused fix with relevant regression coverage and no blocking review findings from this read-only pass.

The PR updates memory-wiki ingest to generate new local-file source page IDs and paths from a readable slug plus a 12-hex canonical-path hash, reuse matching legacy source pages, and add focused ingest collision tests.
