## Summary

Keep this issue open. Current main still does not implement the unsubscribe-time fallback terminal lifecycle event, and live GitHub data shows #73105 is open, unmerged, and explicitly targets this bug with closing syntax. Per policy, the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review #73105 as the direct implementation candidate. The best fix is a backend contract guarantee in subscribeEmbeddedPiSession: emit exactly one terminal lifecycle/chat state during subscription teardown when agent_end was missed, before removing the listener. UI-side stale-run recovery can remain defense in depth but should not replace the backend lifecycle guarantee.
