## Summary

Keep this issue open. Current main still has the embedded timeout lifecycle gap: timeout abort is fire-and-forget, subscription teardown unsubscribes without synthesizing a terminal lifecycle event, Gateway chat finalization relies on terminal lifecycle after an agent run starts, and Control UI clears Stop only from terminal chat events. The open closing PR #73105 is the implementation candidate, not a reason to close this issue before merge.

## Best Possible Solution

Keep this issue open and review #73105 as the primary backend fix candidate. The best fix is for subscribeEmbeddedPiSession teardown to guarantee exactly one terminal lifecycle/chat finalization event when agent_end is missed, with regression coverage for unsubscribe-before-agent_end and idempotency when agent_end arrives normally. Treat #42258 as an optional Control UI recovery mitigation rather than the canonical backend fix unless maintainers choose that product direction.
