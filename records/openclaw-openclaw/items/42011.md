## Summary

Keep this issue open. Current main at d22a851253b0225e72847dc6b1a7b74d8035e214 still has the lifecycle gap described in the report: timeout abort is fire-and-forget, cleanup can unsubscribe without synthesizing a terminal lifecycle event, terminal emission is still driven by agent_end, and Gateway/Control UI clear the Stop state only after terminal chat events. The provided context also shows #73105 is open and uses closing syntax for this issue, so the right next step is to review and either land or close that implementation candidate before closing the issue.

## Best Possible Solution

Keep this issue open and review #73105 as the direct backend fix candidate. The best fix is to make the embedded subscription teardown path guarantee exactly one terminal lifecycle/chat finalization event when agent_end is missed, with regression coverage for unsubscribe-before-agent_end and idempotency when agent_end arrives normally. Treat #42258 as a separate UI recovery mitigation unless maintainers intentionally choose that path.
