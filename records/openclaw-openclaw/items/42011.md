## Summary

Keep this issue open. Current main still does not guarantee a terminal lifecycle event on embedded subscription teardown when `agent_end` is missed, and open PR #73105 explicitly targets this failure with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review #73105 as the direct backend implementation candidate. The best fix is to guarantee exactly one terminal lifecycle/chat state during embedded timeout or abort teardown even when `agent_end` is missed; the separate Control UI watchdog work in #42258 can remain defense in depth for stale client state.
