## Summary

Keep this issue open. Current main still has the embedded timeout/abort race described by the reporter: the timeout path aborts the active session and then teardown removes the embedded subscription, but subscription teardown does not synthesize a lifecycle `end` or `error` event when `agent_end` is missed. The provided context also shows open PR #73105 explicitly closes this issue, so policy says the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review #73105 as the direct backend implementation candidate. The best fix is to guarantee exactly one terminal lifecycle/chat state during embedded timeout or abort teardown even when `agent_end` is missed; UI-side recovery can remain defense in depth but should not replace the backend lifecycle guarantee.
