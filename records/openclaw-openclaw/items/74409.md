## Summary

Keep open. Current main still gates empty/non-visible assistant-turn silence behind group/channel chat detection, so a direct chat with `silentReply.direct: "allow"` can receive the direct-chat silent prompt and outbound silent-token handling, but a clean empty or reasoning-only stop never gets the runner flag that would suppress the empty-response retry and incomplete-turn fallback.

## What This Changes

Allow direct chats configured with `silentReplyPolicy="allow"` to treat clean empty or non-visible assistant replies as intentional silent replies instead of failed empty responses.

## Best Possible Solution

Keep the direct-chat default as disallow/rewrite, but make an explicit `silentReply.direct: "allow"` opt into the same clean empty/reasoning-only silent absorption already supported by the runner, with tests and docs clarifying the default versus opt-in behavior.
