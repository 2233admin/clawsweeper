## Summary

Keep this PR open. Current main still lacks the proposed `ltm reindex` and `ltm drop-index` maintenance surface, and the PR uses supported LanceDB APIs in the plugin-owned module, but the branch still needs revision before merge because the new user-facing commands are undocumented and `--timeout` parsing still accepts partial numeric strings.

## What This Changes

The PR adds `MemoryDB.reindex`, `MemoryDB.dropIndexes`, `openclaw ltm reindex --timeout`, and `openclaw ltm drop-index` to the `memory-lancedb` plugin using LanceDB index creation/list/drop APIs.

## Best Possible Solution

Land a plugin-owned update that keeps the maintenance behavior inside `memory-lancedb`, documents the new CLI commands and destructive index-drop semantics, reuses strict option parsing, and adds focused mocked coverage for the LanceDB index calls before release.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. Source inspection gives a high-confidence check: current main lacks the commands, while the PR diff adds them and exposes the remaining docs/parser gaps.

## Solution Assessment

No, not yet. The owner-module location and LanceDB APIs are appropriate, and the updated PR fixed the original `NaN` timeout case, but the maintainable solution should also document the commands, reject partial timeout values, and add focused tests.

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Document the new ltm maintenance commands:** `extensions/memory-lancedb/index.ts:830-850`
  - body: The patch registers `ltm reindex` and `ltm drop-index`, but the memory-lancedb docs still list only `ltm list`, `ltm search`, and `ltm stats`. These are user-facing maintenance commands, and `drop-index` removes all table indexes, so users need the timeout/default and drop semantics documented before merge.

  - confidence: 0.78
