## Summary

Keep this PR open. Current main still does not implement the proposed configurable model API request delay: the model config type, strict Zod schema, generated config metadata, docs, and embedded runner/compaction stream setup all lack `models.requestDelayMs` or a per-provider `requestDelayMs` override. The PR remains a plausible implementation candidate for #49077, but it needs maintainer review and follow-up on the existing review comments before merge.

## Best Possible Solution

Review and update this PR, or replace it with an equivalent maintainer-approved patch, so OpenClaw gets a validated `models.requestDelayMs` plus per-provider override across config types, strict schema, generated config metadata, docs, embedded runs, and compaction. The implementation should keep one limiter history per normalized provider, handle config changes without a burst window or stale limiter growth, test explicit per-provider `0`, and add a changelog entry with the required attribution.
