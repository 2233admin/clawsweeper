## Summary

Keep this issue open. Current main still matches the reported lifecycle bug: the stop-timeout branch marks the channel account as running and returns before task/abort cleanup, while the start path still refuses to start when the stale task entry remains. PR #70056 is an open, unmerged closing PR for the same fix, so the issue should stay open until that PR or an equivalent patch lands.

## What This Changes

Fix `stopChannel` timeout handling so timed-out channel stops clear task/abort state, mark the account stopped, and allow subsequent restarts.

## Best Possible Solution

Keep this issue open and review/land PR #70056, or land an equivalent focused fix in `src/gateway/server-channels.ts` plus `src/gateway/server-channels.test.ts` that clears stale task/abort entries on stop timeout, records `running: false` and `lastStopAt`, and proves a restart can run after timeout while preserving existing lifecycle race safety.
