## Summary

Current main still matches the reported channel lifecycle bug: stop-timeout marks the account running, skips task/abort cleanup, and leaves a stale task entry that blocks restart. PR #70056 is open and unmerged as the direct implementation candidate, so this issue should stay open until that PR or an equivalent fix is resolved.

## Best Possible Solution

Keep this issue open and review/land PR #70056, or land an equivalent focused fix that marks stop timeouts as not running, records lastStopAt, clears the stale task/abort entries, and updates the existing regression test to prove a restart can run after the timeout.
