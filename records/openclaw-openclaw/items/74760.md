Keep open. ## What This Changes

The issue reports that after updating to OpenClaw 2026.4.27, a POPO group-chat mention reaches the agent but produces no visible reply while direct P2P chat still replies.

## Best Possible Solution

Clarify support guidance for 2026.4.27 POPO users: confirm that the core visible-reply rule is platform-independent, recommend messages.groupChat.visibleReplies: "automatic" when legacy group auto-posting is desired, and only open a code follow-up if maintainers want explicit mentions to bypass the message-tool privacy default across group/channel turns.

## Reproduction Assessment

Yes for the core delivery signature: the current dispatch regression test reproduces a group/channel turn reaching the reply resolver while queuing no final reply by default. No live POPO-specific reproduction is available because the POPO implementation is not bundled here.

## Solution Assessment

Unclear. The documented config restore path is the narrowest support answer if the private-by-default policy is intended; changing explicit mentions to auto-post would be a broader privacy/product decision that should not be queued as an automatic fix.
