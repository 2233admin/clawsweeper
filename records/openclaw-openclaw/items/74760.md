## Summary

Keep open. Current main still matches the reported group-chat no-visible-final signature, and the behavior appears platform-independent; deciding whether external POPO explicit mentions should auto-post, use the documented compatibility config, or get channel-specific support needs maintainer judgment.

## What This Changes

The reporter asks for POPO group-chat bot mentions to produce visible replies again in v2026.4.27, while direct chats already work.

## Best Possible Solution

Clarify support guidance for v2026.4.27 POPO users: confirm that group/channel visible-reply behavior is platform-independent, recommend messages.groupChat.visibleReplies: "automatic" when legacy group auto-posting is desired, and open a code follow-up only if maintainers want explicit mentions to bypass the message-tool privacy default.
