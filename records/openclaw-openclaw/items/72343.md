## Summary

Keep this PR open. Current main still routes bare `openclaw` directly to Crestodian without checking whether the active config is missing or a valid empty object, while this PR remains a focused implementation candidate for changing fresh CLI installs to classic onboarding first.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept the UX change, the best path is the PR's scoped CLI route: check the config snapshot before the bare-root Crestodian branch, send missing or valid-empty authored configs to classic onboarding, preserve Crestodian for configured installs and invalid configs needing repair, and keep the paired unit, Docker smoke, and docs updates.
