## Summary

Keep this PR open. Current main still sends bare `openclaw` to Crestodian through an argv-only route, with docs and smoke coverage that continue to describe/assert Crestodian as the no-command first-run behavior, so the PR is not obsolete or implemented on main.

## What This Changes

The PR adds a fresh-install config snapshot gate for bare `openclaw`, routes missing or valid empty configs into classic onboarding, preserves Crestodian for configured/invalid configs and `onboard --modern`, and updates related CLI docs and smoke tests.

## Best Possible Solution

Keep the PR open for maintainer review of the first-run UX. If accepted, land the scoped config-snapshot gate before the bare-root Crestodian branch, preserve Crestodian for configured and invalid configs, keep the docs/tests aligned, and require targeted CLI tests plus Docker/Testbox first-run proof.
