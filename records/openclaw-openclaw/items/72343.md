## Summary

Keep this PR open. Current main still sends bare `openclaw` directly to Crestodian without checking whether the active config is missing or a valid empty object, while the PR remains a focused implementation candidate for fresh CLI installs. The prior Greptile concern about the Docker smoke assertion was addressed in the PR context, so the remaining question is maintainer/product review rather than obsolescence.

## Best Possible Solution

Keep this PR open for maintainer review. If the UX change is accepted, the best path is the PR's scoped CLI route: inspect the active config snapshot before the bare-root Crestodian branch, send missing or valid-empty authored configs to classic onboarding, preserve Crestodian for configured installs and invalid configs needing repair, and keep the paired unit, Docker smoke, and docs updates.
