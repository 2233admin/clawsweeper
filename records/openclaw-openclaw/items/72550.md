## Summary

Keep open. Current main still has a plausible shared-core false-success path: raw channel send helpers can return `ok: false` plus an error as data, while outbound delivery and isolated cron delivery count any returned result as delivered. The WeChat session guard itself is external-plugin behavior, but the generic OpenClaw result contract still needs a core fix before this can be closed.

## What This Changes

The issue asks cron announce delivery for `openclaw-weixin` to mark failed WeChat outbound sends as failed instead of recording `delivered: true`.

## Best Possible Solution

Keep this issue open and fix the channel-agnostic outbound result contract so returned failed send results cannot be counted as successful cron delivery. WeChat token/session recovery should remain in the external plugin, with only generic OpenClaw SDK/outbound seams changed here.

## Reproduction Assessment

Yes for the core path: a focused test can register or mock a channel adapter whose text send returns `{ ok: false, messageId: '', error: 'session expired' }` and verify cron does not persist `delivered`. The live WeChat reproduction is plausible from the report but depends on external plugin behavior not present in this repo.

## Solution Assessment

Yes: the narrowest maintainable repair is to make outbound delivery reject or filter failed raw send results before cron counts returned results. A WeChat-specific branch in core would cross the documented plugin ownership boundary.
