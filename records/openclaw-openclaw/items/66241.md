## Summary

Keep open. Current main still does not persist Discord component/modal registry state across gateway restarts; the registry remains process-global Maps only, and registry misses still produce expired-component replies. The PR is a plausible implementation candidate, but it touches security-sensitive Discord routing and allowlist state, and the safest path is maintainer review or rework against the existing state-dir, lock, and atomic JSON patterns, or the related #70626 helper if that lands first.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rework the Discord registry persistence to use the active OpenClaw state directory and existing atomic/locked JSON patterns, or adapt it to `openclaw/plugin-sdk/persistent-keyed-store` after #70626 lands. The final change should preserve `sessionKey`/`agentId`/`accountId` route overrides and `allowedUsers`, validate persisted records before loading, prune expired entries, bound growth, and add focused restart round-trip, malformed-file, expired-prune, and authorization/routing regression tests.
