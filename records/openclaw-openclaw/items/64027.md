## Summary

Keep open. Current main still lacks the PR's central Feishu behavior: card-action envelopes cannot carry replacement card content, the card-action payload parser does not expose the clicked card's `message_id`, and the handler never calls the existing Feishu card patch helper. The PR remains a plausible implementation candidate, but the branch should be reviewed or narrowed because it also carries unrelated Feishu reply-dispatcher and test churn.

## What This Changes

The PR adds Feishu card-action update-card wiring through a `uc` envelope/context payload, `message_id` parsing, `updateCardFeishu` calls from the card-action handler, and additional Feishu reply-dispatcher/test changes.

## Best Possible Solution

Keep the Feishu card-update behavior in this PR or a focused replacement: add the `uc` update-card payload to the Feishu card interaction contract, parse the clicked card `message_id`, patch the original card with isolated failure handling during cancel/confirm/quick actions, and cover the new paths with targeted Feishu tests before merge.
