## Summary

Current main still does not fully satisfy the requested internal-only `sessions_send` behavior. The initial target-agent turn is injected with `deliver:false` on the internal channel, but the follow-up A2A announce flow can still generate a non-`ANNOUNCE_SKIP` reply and call the gateway `send` method with the resolved target channel/thread. The public `sessions.send` schema also has no `silent` or internal-only flag, and existing tests still assert visible announce delivery including thread IDs.

## Best Possible Solution

Keep this open and define the product contract for inter-agent delivery. The strongest fix is to enforce internal-only A2A `sessions_send` delivery by default while preserving the target session transcript, or add an explicit `silent`/internal-only API option with safe defaults. The implementation should prevent the announce path from calling user-facing `send` for internal coordination and add regressions for Slack channel-bound and thread-bound targets.
