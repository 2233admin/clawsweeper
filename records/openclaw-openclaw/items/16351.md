## Summary

Keep this issue open. Current main has implemented substantial parts of the request: `openclaw config validate` exists with JSON output, config writes validate before commit, and write-back has env-placeholder restoration for existing `${ENV}` values. The remaining central request is still unresolved: bare `openclaw config` is intentionally documented and implemented as a guided configure wizard entrypoint rather than help on already configured systems.

## Best Possible Solution

Keep the issue open and narrow the remaining work to the root command behavior and secret-safe value-mode UX: decide whether bare `openclaw config` should show help on valid/fully configured systems, preserve `--section`/`openclaw configure` as the wizard path, and add a focused test/docs update. Keep the existing `config validate`, write validation, SecretRef builder, and env-placeholder restoration behavior.
