## Summary

Keep open. Current main still has automatic per-channel restart paths for config hot reload, secrets reload, and health monitoring, but it does not expose the requested manual `openclaw channels reload <provider>` command or a `channels.reload` Gateway RPC. The existing `channels.start` method is not an equivalent reload path for already-running accounts.

## What This Changes

The issue requests a manual per-channel reload command or Gateway action that reconnects one named channel without restarting the whole gateway.

## Best Possible Solution

Add an admin-scoped `channels.reload` Gateway method plus an `openclaw channels reload --channel <provider> [--account <id>]` CLI wrapper that reuses the existing stop/start lifecycle, preserves unrelated channels and sessions, returns structured status, and updates protocol schemas, method scopes, CLI/Gateway docs, and focused tests.
