## Summary

Keep open. Current main still does not capture OpenAI-compatible proxy-resolved `response.model` or `chunk.model`, persist separate requested-vs-resolved backend identity fields, or expose those fields through `session_status`. The existing selected-vs-active fallback surfaces are adjacent, but they cover OpenClaw-managed fallback state rather than LiteLLM/OpenRouter routing metadata.

## Best Possible Solution

Keep this open and implement the missing OpenAI-compatible transport/session pipeline: capture returned `response.model` and streamed `chunk.model` when present, thread requested and resolved identities through assistant output and `agentMeta`, persist requested-vs-resolved state on sessions, expose it in `GatewaySessionRow`, `session_status`, and status output, and add focused LiteLLM/OpenRouter-style tests proving silent proxy routing is visible in-band.
