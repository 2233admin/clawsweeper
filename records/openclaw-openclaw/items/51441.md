## Summary

Keep open. Current main still has only selected-vs-active OpenClaw fallback metadata; it does not capture OpenAI-compatible proxy-returned `response.model` or `chunk.model`, persist requested-vs-resolved backend fields, or expose those fields through `session_status` / status output.

## Best Possible Solution

Keep this open and implement an additive OpenAI-compatible transport/session pipeline: capture returned `response.model` and streamed `chunk.model` when present, preserve requested and resolved identities separately through assistant output and `agentMeta`, persist resolved backend metadata on sessions, expose it in `GatewaySessionRow`, `session_status`, and status output, and add focused LiteLLM/OpenRouter-style tests for silent proxy routing.
