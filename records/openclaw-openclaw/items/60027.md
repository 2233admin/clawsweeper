## Summary

Keep this PR open. Current main partially addresses the startup bottleneck by moving primary-model prewarm off the channel-start path, bounding it, and scoping provider discovery to the configured provider, but it does not implement the requested `OPENCLAW_SKIP_MODEL_WARMUP` escape hatch. The PR remains useful, though it needs rebasing onto the current `server-startup-post-attach.ts` startup path.

## Best Possible Solution

Rebase the PR onto current main by adding the env check to `prewarmConfiguredPrimaryModel` in `src/gateway/server-startup-post-attach.ts`, register the key in gateway test env helpers, and add focused coverage proving the warmup path returns without calling `ensureOpenClawModelsJson` when `OPENCLAW_SKIP_MODEL_WARMUP` is truthy. Keep the existing background/scoped warmup behavior as the default.
