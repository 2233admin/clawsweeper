## Summary

Keep this issue open. Current main has partial hook timeout support, but legacy `before_agent_start` handlers without per-registration `timeoutMs` still have no default/global timeout and are still awaited on both model-resolution and prompt-build paths. The related upstream OpenViking issue and closed unmerged PR #48741 are useful prior context, but they do not close the remaining OpenClaw core guardrail gap.

## Best Possible Solution

Add a fail-open default timeout for legacy `before_agent_start` in the hook runner, preserving per-registration `timeoutMs` overrides. The warning should identify plugin id, hook name, elapsed budget, and run id when available, and the run should continue. Add regression tests for hung legacy handlers on both model-resolution and prompt-build paths, and align plugin-hook docs with the actual default policy.

## Risks / Open Questions

- Legacy or external plugins that register `before_agent_start` without `opts.timeoutMs` can still block an agent run before the provider call starts.
