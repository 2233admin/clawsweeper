## Summary

Keep open. Current main still lacks Telegram native subagent lifecycle hook registration, so Telegram `sessions_spawn({ mode: "session", thread: true })` cannot prove `threadBindingReady` through the core hook gate. Telegram has conversation/thread binding infrastructure and ACP topic binding, but those do not satisfy the requested native subagent session path. PR #42707 remains the focused open implementation candidate.

## Best Possible Solution

Keep this issue open and review/land PR #42707 or an equivalent Telegram plugin-boundary implementation. The fix should add Telegram `registerFull` wiring for `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`, reuse the existing Telegram thread-binding manager, honor `channels.telegram.threadBindings.spawnSubagentSessions`, clean up bindings on failed starts/end, and add focused tests/docs that keep ACP topic binding distinct from native subagent thread binding.
