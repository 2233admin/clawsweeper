## Summary

Keep open. Current main still does not wire Telegram native subagent lifecycle hooks, while core requires a registered `subagent_spawning` hook before `sessions_spawn({ mode: "session", thread: true })` can bind a channel thread. Telegram has adjacent topic/current-conversation binding pieces and ACP topic binding, but the requested native subagent session binding remains unshipped; PR #42707 is still the focused open implementation candidate.

## Best Possible Solution

Keep this issue open and review/land PR #42707 or an equivalent focused implementation in the Telegram plugin boundary. The fix should add Telegram `registerFull` hook wiring, implement `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`, reuse the existing Telegram thread-binding manager, honor `channels.telegram.threadBindings.spawnSubagentSessions`, clean up bindings on failed starts/end, and add native subagent docs/tests so ACP topic binding and native subagent thread binding stay distinct.
