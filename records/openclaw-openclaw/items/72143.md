## Summary

Keep open. Current main still exposes the internal resume-failure prefix for denied and other non-success direct fallbacks, and the session follow-up path still makes a single resume attempt before falling back without the requested retry/backoff or internal-only resume failure log.

## What This Changes

The issue requests retry/backoff for exec approval follow-up session resume failures, silent direct fallback delivery without an internal prefix, and sanitized internal logging when fallback is used.

## Best Possible Solution

Land a small agents-focused fix that removes the resume-failure prefix from every direct fallback path, adds bounded retry/backoff around the `agent` follow-up call, and logs sanitized resume-failure metadata only after retries are exhausted.

## Reproduction Assessment

Yes. A high-confidence unit-level reproduction exists by making `callGatewayTool` reject in the denied resume-failure path; the current colocated test still expects the internal prefix, and source inspection shows only one session resume attempt.

## Solution Assessment

Yes. The requested direction is the narrow maintainable solution: retry the session follow-up briefly, then silently deliver sanitized result content directly and keep resume diagnostics in logs rather than user chat.
