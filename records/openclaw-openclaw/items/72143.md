## Summary

Keep open. Current main still only partially addresses the report: successful code-0 resume-fallback deliveries omit the internal prefix, but denied/non-success fallback deliveries still prepend it, the session follow-up call is attempted once with no backoff, and exhausted resume failures are not logged internally when direct fallback succeeds.

## Best Possible Solution

Land a narrow exec approval follow-up fix: retry the `agent` session follow-up call with short bounded backoff, silently direct-deliver sanitized fallback content without the internal resume-failure prefix for every result kind after retries are exhausted, and log the exhausted resume failure internally with safe metadata. Preserve the current code-0 success fallback behavior and the existing denied-followup suppression for cron/subagent sessions.

## Clownfish Work Prompt

Fix this issue on current main. Observable bug: async exec approval follow-up tries to resume the original session once; if that resume/follow-up fails, direct fallback still exposes the internal `Automatic session resume failed, so sending the status directly.` prefix for denied and non-success results. Expected behavior: retry the `agent` session follow-up with short bounded backoff before falling back; if retries are exhausted, direct-deliver the sanitized command result/status content without the internal prefix for every result kind; log the exhausted resume failure internally with safe metadata such as approvalId, sessionKey, channel, to/account/thread identifiers, and error message. Do not log raw command output, secrets, or full user message bodies. Keep the existing `exec-approval-followup:<approvalId>` idempotency key, preserve current code-0 success fallback behavior, and preserve denied-followup suppression for cron/subagent sessions. Scope the implementation to `src/agents/bash-tools.exec-approval-followup.ts` and colocated tests unless a tiny local helper is clearly needed. Do not change exec approval decision policy, allowlist behavior, command execution, delivery target resolution, or channel authorization. Add/update focused tests for retry count/backoff, prefix-free denied fallback, prefix-free non-code-0 fallback, preserved success fallback, and internal logging after exhausted retries. Add a single-line user-facing changelog fix entry thanking @ahmednasr999; mention #67181 only as related if useful.
