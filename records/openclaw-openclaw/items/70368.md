## Summary

Keep open. Current main and the latest tagged release still report `gateway.trusted_proxies_missing` as `warn` only for loopback Control UI configs, so this PR is not implemented on main; the PR also remains security-sensitive because its shared loopback-equivalence model can suppress critical gateway audit findings for ambiguous `auto` and fallback-prone `custom` binds.

## What This Changes

This PR changes gateway security-audit classification by adding a shared loopback-equivalent bind helper, changing trusted-proxy and sibling Control UI findings to use that classification, and adding regression tests for loopback, custom, auto, and non-loopback bind cases.

## Best Possible Solution

Adopt a conservative effective-bind audit model: downgrade only definitely local-only exposure, fail closed for ambiguous `auto` and fallback-prone `custom` binds, keep `trusted_proxies_missing`, `bind_no_auth`, `loopback_no_auth`, and Control UI origin findings coherent, and update regression tests plus the public audit-check docs if severity semantics change.
