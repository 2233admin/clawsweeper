## Summary

Keep open. Current main still does not cover the exact Anthropic `specified API usage limits` / `regain access` wording, but this PR needs maintainer rework before merge because the billing matcher is broader than the reported error and the reply-dedupe change bypasses the current same-target guard.

## What This Changes

The PR updates failover billing/rate-limit classification and assistant error formatting for Anthropic usage-limit messages, adds billing classification tests, and changes messaging-tool final-reply dedupe in `agent-runner-payloads`.

## Best Possible Solution

Split or rework the PR: add a narrow Anthropic billing matcher for the exact `specified API usage limits` plus `regain access` wording while preserving periodic usage-limit rate-limit behavior, and solve the duplicate-reply case through identity-aware target matching via the existing channel suppression/normalization seams rather than global text-only dedupe.
