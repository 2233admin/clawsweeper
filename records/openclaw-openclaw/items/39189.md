## Summary

Keep this PR open. Current main still lacks the exact Anthropic `specified API usage limits` / `regain access` billing matcher, but this PR as written still has the two unresolved functional blockers from review: the billing matcher is too broad for ordinary usage-window limits, and unconditional text dedupe can suppress legitimate same-text replies sent to a different target.

## What This Changes

The PR adds Anthropic usage-limit billing classification coverage and changes messaging-tool final-reply dedupe so sent text can suppress final replies even when the message-tool target differs from the originating target.

## Best Possible Solution

Keep this PR open, but rework or split it. The failover fix should add a narrow Anthropic billing matcher and regression tests for the exact `specified API usage limits` plus `regain access` wording without shadowing periodic usage-window limits. The reply-dedupe fix should use identity-aware target matching, likely through the existing channel suppression hook or target normalization for iMessage/BlueBubbles aliases, while preserving same-text cross-target delivery tests.
