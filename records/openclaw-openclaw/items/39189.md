## Summary

Current main still does not implement this PR’s intended behavior. The Anthropic `specified API usage limits` / `regain access` wording is still not covered by billing matching and can hit the broad `usage limit` rate-limit path, while the reply-dedupe fix in the PR still conflicts with current same-text cross-target preservation tests.

## Best Possible Solution

Keep this PR open, but rework or split it. The failover half should add a narrow Anthropic billing matcher and regression tests for the exact `specified API usage limits` plus `regain access` wording without shadowing periodic usage-window limits. The reply-dedupe half should use identity-aware target matching, likely through the existing channel suppression hook for BlueBubbles/iMessage aliases, with positive same-person email/phone coverage and negative same-text cross-target coverage.
