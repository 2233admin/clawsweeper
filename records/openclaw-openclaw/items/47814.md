## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject collapsed empty segments in named-DM parsing:** `src/sessions/session-key-utils.ts:180-185`
  - body: `parseNamedDmSessionKey` delegates to `parseAgentSessionKey`, which removes empty `:` segments before this shape check. A malformed key such as `agent:main:dm-named:123::work` can be parsed as a valid named-DM key; validate the raw suffix or reject empty segments before returning a parsed result.
  - confidence: 0.92
- **[P2] Fail closed for malformed stored active names:** `src/gateway/session-utils.ts:866-870`
  - body: `activeNamedSession` is persisted session-store data, but `getActiveNamedSessionKey` passes it straight into `buildNamedDmSessionKey`. A stored value containing `:` will throw during lookup despite the helper's `string | null` contract; validate the stored name and return `null` or repair it before building the key.
  - confidence: 0.9
- **[P2] Prioritize dm-named before arbitrary name tokens:** `src/sessions/session-key-utils.ts:50`
  - body: Adding `dm-named` to the direct branch after the `group` and `channel` token checks still misclassifies valid named DM keys whose user-chosen name is `group` or `channel`. That can make send-policy fallback apply group/channel rules to a DM session, so recognize the named-DM shape before scanning arbitrary name tokens.
  - confidence: 0.91
- **[P2] Keep the changelog entry in the active section only:** `CHANGELOG.md:109`
  - body: The branch repeats the named-DM changelog bullet in historical release sections as well as `Unreleased`. That rewrites old release history and can confuse release-note tooling; keep a single credited entry in the active changelog section for this change.
  - confidence: 0.96

## Security Review

Status: needs_attention

Summary: The diff does not add dependencies, CI, secrets, or package execution paths, but the named-DM chat-type change can misapply send-policy rules for user-controlled names.

Concerns:

- **[medium] Named-DM names can hit group/channel send policies:** `src/sessions/session-key-utils.ts:50`
  - body: Because `dm-named` is checked only after `group` and `channel` tokens, a DM session named `group` or `channel` can be classified as that chat type when policy falls back to key-derived chat type. That can apply the wrong outbound send-policy rules to a direct-message session.
