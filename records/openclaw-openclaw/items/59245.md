## Summary

Keep this open. Current main has outbound voice-call primitives, DTMF, realtime voice, and manual call controls, but it still lacks the central autonomous task-call workflow requested here: objective-driven call orchestration, IVR/hold handling, owner escalation, authorized-information policy, timeout/reschedule behavior, and callback continuity. A maintainer already reviewed the same feature area and explicitly kept it open; pchero's VoIPBin/media-offloading comment is useful design input, not a reason to close the OpenClaw product request.

## What This Changes

The issue requests an outbound task-call capability where an agent can place phone calls on the user's behalf, handle holds and IVRs, escalate decisions to the user, share only authorized information, and resume or reschedule calls as needed.

## Best Possible Solution

Define and implement a voice-call plugin task-call orchestration layer on top of the existing outbound call, DTMF, realtime voice, and status primitives. The design should include a task objective schema, call state machine, IVR/DTMF strategy, hold detection or provider hold events, owner escalation with timeout behavior, allowed-information policy, max duration/cost controls, transcript/reporting, and callback/reschedule semantics; media offloading can be evaluated as a provider strategy inside that contract.
