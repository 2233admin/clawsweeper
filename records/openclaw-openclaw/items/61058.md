## Summary

Keep open. Current main still does not implement the requested safe env-reference audit behavior: the audit report contract has no ENV_REF-style classification, $VAR is not parsed as a SecretRef, and several audit paths still emit PLAINTEXT_FOUND for non-empty strings. The earlier comment pointing to #61207 is useful context, but the provided timeline shows that PR closed unmerged; #64344/#54141 remain the plausible implementation paths rather than shipped fixes.

## Best Possible Solution

Keep this issue open and route it through #64344, #54141, or a successor fix. Maintainers should decide the supported syntax and audit contract, then update src/config/types.secrets.ts and src/secrets/audit.ts with focused regression coverage in src/secrets/audit.test.ts / src/config/types.secrets.resolution.test.ts and align docs/cli/secrets.md plus docs/reference/secretref-credential-surface.md.
