## Summary

Keep open. Current main still forwards iMessage reply context only when the `imsg` payload already contains resolved `reply_to_*` fields; it does not accept GUID-only inline-reply metadata or enrich from `chat.db`. Open PR #42683 remains the closest implementation path and needs maintainer review rather than a duplicate repair lane.

## What This Changes

Request that incoming iMessage inline replies include the referenced message context by resolving reply/thread metadata into `ReplyToId`, `ReplyToBody`, and `ReplyToSender` before dispatch.

## Best Possible Solution

Land or replace the iMessage-focused monitor change so the bundled iMessage plugin resolves reliable inline-reply origin metadata into the existing `ReplyTo*` dispatch fields, scoped to `extensions/imessage/src/monitor/*` with regression coverage for GUID-only replies and failure modes.
