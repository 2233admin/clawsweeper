## Summary

Keep open. Checked-out main at 28ff82dc still allows a hook-provided bootstrap file with a path but missing or empty name to reach the context report as `file.name`, and `/context` renders that value directly. Live GitHub API data also shows open PR #48021 still unmerged with closing syntax for this issue, so the right path is to review/land or close that PR before resolving this issue.

## Best Possible Solution

Keep this issue open and review #48021 or an equivalent focused fix. The fix should normalize bootstrap stat labels in `buildBootstrapInjectionStats` from a non-empty `name`, then path basename, then a stable placeholder, and add regression coverage for hook-injected virtual bootstrap files with a valid path and missing or empty name. The broader session-memory echo behavior should remain tracked separately by #39013.
