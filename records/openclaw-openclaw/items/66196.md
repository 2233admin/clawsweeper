## Summary

Keep this PR open. Current main still routes WhatsApp inbound DM text and audio transcripts through raw model-facing fields, so the security-hardening goal remains relevant, but this patch is not merge-ready because it wraps legacy envelope strings rather than the active prompt path and would make combined echo keys nondeterministic.

## What This Changes

The PR imports `wrapExternalContent` and wraps WhatsApp inbound envelope bodies in `message-line.ts` and group-history envelope entries in `process-message.ts`.

## Best Possible Solution

Revise or replace the PR with a WhatsApp-specific prompt-boundary fix that wraps or isolates the actual model-facing user body, audio transcript, reply-context text, and group-history text, while keeping operator-configured prefix/system context trusted and leaving echo keys based on deterministic unwrapped content. Add focused WhatsApp regression tests for direct DMs, audio transcripts, reply context, group history, prefix handling, and duplicate suppression.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wrap the model-facing WhatsApp body:** `extensions/whatsapp/src/auto-reply/monitor/message-line.ts:43`
  - body: The changed envelope `body` is not the primary text the agent sees. `processMessage` still passes raw `msgForAgent.body` into `BodyForAgent`, and session setup derives the prompt body from `BodyForAgent` before the legacy envelope `Body`, so direct DM text and audio transcripts remain unwrapped despite this patch.
  - confidence: 0.93
- **[P1] Keep combined echo keys deterministic:** `extensions/whatsapp/src/auto-reply/monitor/message-line.ts:43`
  - body: `buildInboundLine` feeds `combinedBody`, which is used to build and remember the combined echo key. Since `wrapExternalContent` adds a fresh random marker id on every call, duplicate WhatsApp deliveries of the same inbound message would no longer match the remembered combined key and could receive duplicate replies.
  - confidence: 0.9
- **[P2] Keep trusted prefix text outside the untrusted block:** `extensions/whatsapp/src/auto-reply/monitor/message-line.ts:43`
  - body: `baseLine` contains the operator-configured `messagePrefix` plus reply-context framing before the user body. Wrapping the whole string marks that trusted prefix and structure as untrusted content, which can make the agent discount operator-supplied context that should remain outside the external-content markers.
  - confidence: 0.82
