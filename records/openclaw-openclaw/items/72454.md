## Summary

Keep open. Current main still lacks the exact MiniMax text-only classifier this PR adds: explicit HTTP 529 and raw strings beginning with 529 classify as overloaded, but the reported raw message `High traffic detected. For a more stable experience, upgrade to our Plus plan` has no matching overload pattern on main. The PR is narrow and plausible, with no security-sensitive diff surface found, but it should add the focused regression test already requested in review before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to add a focused unit test, likely in `src/agents/pi-embedded-helpers/failover-matches.test.ts`, asserting `High traffic detected. For a more stable experience, upgrade to our Plus plan` returns true from `isOverloadedErrorMessage` and false from `isBillingErrorMessage`, then land this PR or an equivalent tested fix on main.
