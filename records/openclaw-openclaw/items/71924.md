## Summary

Current main does not implement this PR’s intended behavior. The GitHub Copilot catalog still returns `models: []`, `gpt-5.5` is not in the Copilot xhigh thinking list, and unknown `github-copilot/gpt-5.5` still falls through to synthetic 128k/8192 metadata rather than Copilot `/models` capability metadata. The PR is a focused implementation candidate for the already-linked GPT-5.5 catalog issue, so it should remain open for maintainer review/landing rather than cleanup closure.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to land this PR or an equivalent provider-owned patch that fetches GitHub Copilot `/models`, maps capability fields into `ModelDefinitionConfig`, includes `gpt-5.5` in Copilot xhigh thinking profiles, and keeps focused coverage in `extensions/github-copilot/index.test.ts` plus the existing Copilot model tests.
