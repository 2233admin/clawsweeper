## Summary

Keep open. Current main still does not expose or forward Jina-style embedding `task` fields, and the same-author open PR #74444 uses closing syntax for this issue, so the issue should remain open until that implementation candidate lands or is replaced.

## What This Changes

The issue requests configurable Jina Embeddings v5 task parameters for memory embedding requests, ideally with query/document-specific values.

## Best Possible Solution

Review and land PR #74444 if its diff matches the stated scope, or replace it with a narrow implementation that adds typed `queryTask` and `documentTask` memorySearch config, forwards `task` for query and document embedding requests, includes task values in cache identity, and updates schema docs, tests, generated config docs, and changelog.

## Reproduction Assessment

Not applicable as a runtime bug report. The feature gap is reproducible by source inspection: current main has typed `input_type` config and forwarding, but no accepted config or request-body path for Jina's separate `task` parameter.

## Solution Assessment

Yes, the typed query/document task direction is the narrowest maintainable solution for Jina v5 adapters. The issue should stay open while PR #74444 is reviewed, because an open closing PR is an implementation candidate rather than a reason to close early.
