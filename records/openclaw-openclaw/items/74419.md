## Summary

Keep open. Current main still lacks any `task`, `queryTask`, `documentTask`, or embedding-body passthrough for OpenAI-compatible memory embeddings, while Jina's current API schema supports a v5 `task` field. The issue is also paired with open PR #74444, which uses closing syntax for this feature, so this issue should stay open until that PR is reviewed and merged or closed. 

## What This Changes

Request to add configurable Jina v5 `task` forwarding for OpenAI-compatible memory embedding requests, preferably with separate query and document task settings.

## Best Possible Solution

Review and land PR #74444 if its typed `queryTask`/`documentTask` implementation passes the focused memory embedding tests, docs generation checks, and changed gate; otherwise implement the same narrow typed task surface in the OpenAI-compatible memory embedding path rather than a broad arbitrary body passthrough.

## Reproduction Assessment

Not applicable as a feature request, but the gap has a high-confidence source-level reproduction: a config containing `memorySearch.queryTask` or `documentTask` has no schema/type/resolution path on current main, and the request builders never emit `task`.

## Solution Assessment

Yes, the typed query/document task direction is the narrowest maintainable way to solve the issue because Jina retrieval uses different query and passage adapters. A generic `extraBody` passthrough would be broader and should require explicit product approval.
