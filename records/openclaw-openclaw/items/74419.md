## Summary

Keep open. Current main supports OpenAI-compatible `input_type` forwarding for memory embeddings, including separate query/document values, but it does not expose or forward Jina's separate `task` request field. The request is narrow, current code and docs confirm the gap, and the upstream Jina v5 model docs support task-specific adapters, so this is a valid focused feature rather than a duplicate or stale cleanup item.

## What This Changes

Request support for configuring Jina Embeddings v5 `task` values on memory embedding requests, ideally with query/document-specific settings.

## Best Possible Solution

Add typed OpenAI-compatible memory embedding task configuration that forwards `task` for query and document embedding requests, includes the task values in cache identity, and updates config schema, docs, tests, and changelog without weakening existing `input_type` behavior.

## Repair Work Prompt

Implement typed Jina/OpenAI-compatible memory embedding task forwarding for this issue. Add config fields under `agents.defaults.memorySearch` and per-agent overrides, preferably `task`, `queryTask`, and `documentTask`, where query embeddings send `queryTask ?? task` as request body `task` and document/index/batch embeddings send `documentTask ?? task` as request body `task`. Extend the resolved memorySearch config, public config types, zod schema, plugin SDK memory embedding option types, and memory-core provider request plumbing. In `extensions/openai`, forward the task value in both inline `createOpenAiEmbeddingProvider` requests and OpenAI batch embedding requests, and include document task values in runtime/cache identity so changing tasks causes reindex/cache separation. Preserve existing `inputType`/`queryInputType`/`documentInputType` behavior. Do not add generic arbitrary `extraBody` passthrough unless a maintainer explicitly asks for that broader contract. Update `docs/reference/memory-config.md`, `docs/providers/openai.md`, generated schema/help/labels as required by repo tooling, and add a user-facing changelog entry crediting `@seoseo-ai`. Add focused regression tests for query task, document task, batch task, fallback/shared task, and existing `input_type` coexistence.
