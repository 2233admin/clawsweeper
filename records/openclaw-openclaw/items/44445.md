## Summary

Keep open. Current main still copies raw user transcript text into session-memory output without applying the existing inbound metadata, envelope, or message-id cleanup. The PR addresses a real bug, but it is not merge-ready as-is because it also changes scheduled stale-session rollover into command hook dispatch, which can run command:new/before_reset hooks for ordinary messages.

## Best Possible Solution

Keep this PR open, but ask for a narrower rebase. The best fix is a session-memory-specific user text sanitizer in src/hooks/bundled/session-memory/transcript.ts that reuses the existing inbound metadata, envelope, and message-id helpers while preserving real user-authored content. Add focused regression tests for Feishu metadata blocks, timestamp-prefixed transcripts, message-id hint lines, slash-prefixed real content, and user-authored message-id-looking lines. Keep scheduled rollover persistence separate, preferably through the existing session_end/lifecycle path rather than treating a stale-session message as a command:new event.
