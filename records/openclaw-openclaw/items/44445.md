## Summary

Keep this PR open. Current main still lacks session-memory-specific cleanup for channel-injected user metadata, so the central bug is not implemented or superseded. The PR is also not merge-ready as-is because it mixes the metadata fix with scheduled-reset hook dispatch and has unresolved review feedback around message-id fidelity, slash-prefixed user content, timestamp-prefixed transcripts, and command-hook fanout for ordinary stale-session messages.

## Best Possible Solution

Keep this PR open, but ask for a narrower rebase. The best path is to implement a session-memory-specific user transcript sanitizer in `src/hooks/bundled/session-memory/transcript.ts`, reuse existing metadata/envelope/message-id helpers where their contracts fit, add regression tests for Feishu-style metadata and fidelity edge cases, and handle scheduled rollover persistence through a dedicated lifecycle/session-end hook instead of routing ordinary stale-session messages through `command:new` reset hooks.
