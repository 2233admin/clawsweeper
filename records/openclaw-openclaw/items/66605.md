## Summary

Keep open. Current main still lacks the requested subscribeEmbeddedPiSession-level markdown-table integration test, and the underlying block chunking paths are still fence-aware rather than table-aware. Open PR #66344 explicitly declares it will close this issue and #66614, and #73981 is another overlapping open candidate, so this issue should remain open while maintainers choose the PR path.

## What This Changes

Add subscribeEmbeddedPiSession integration coverage proving paragraph-preference block streaming emits Intro, the full markdown table, and Outro as separate block replies without splitting the table.

## Best Possible Solution

Land or deliberately close a canonical PR that adds table-span protection to EmbeddedBlockChunker and chunkByParagraph, then adds the subscribeEmbeddedPiSession table integration case so the full table stays in one block reply between Intro and Outro.
