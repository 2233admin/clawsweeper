## Summary

Keep this issue open. Current main has SecretRef runtime, CLI, redaction/restore, preflight, and documentation foundations, but the requested first-class Control UI/config UX is still missing: form rendering still shows structured SecretRefs as generic read-only placeholders, the schema analyzer still collapses SecretInput unions to the string editor path, and the docs describe that limitation rather than env/config/unset source-aware editing.

## What This Changes

Request first-class .env/environment-backed secret configuration UX that shows secret source state, preserves env references, and improves migration guidance in CLI and Control UI.

## Best Possible Solution

Add a source-aware SecretInput renderer and metadata flow on top of the existing SecretRef contracts, with masked display, safe source labels, env variable identity when appropriate, resolved/unresolved status, explicit config/env/unset editing, and aligned docs/tests for the Control UI and existing secrets CLI workflows.
