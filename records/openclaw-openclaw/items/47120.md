## Summary

Keep open. Current main has a backend session-list preview contract, but the Control UI Sessions list still neither requests nor renders last-message previews, and open PR #47243 is already paired with this work through closing syntax.

## What This Changes

The issue requests that the Control UI Sessions list show a timestamp and last-message preview so sessions are easier to distinguish.

## Best Possible Solution

Land a small Control UI change that requests bounded preview data through the existing `sessions.list` `includeLastMessage` contract, extends the UI row type, renders timestamp plus preview in the Sessions list, and adds focused controller/view coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is current-main source inspection or opening the Sessions page with multiple sessions: the Control UI list request omits `includeLastMessage`, and rows render only relative updated time.

## Solution Assessment

Yes, the best direction is to reuse the existing backend `includeLastMessage` list contract rather than adding per-row frontend `sessions.preview` RPC calls. The open PRs should be reviewed or replaced against that current-main shape.
