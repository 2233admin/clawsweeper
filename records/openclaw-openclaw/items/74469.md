## Summary

Close as implemented on main: current main no longer runs the heavy security audit for plain human `openclaw status`, and the text report now renders a skipped-audit note instead. The fix is present after `v2026.4.26` but is not included in a tagged release yet.

## What This Changes

The issue reports that `openclaw status` human table output in `v2026.4.26` takes about 43 seconds while `--json` output takes about 3 seconds, pointing to CLI-side status formatting/rendering.

## Best Possible Solution

Keep the current fast default status path, with expensive audit/probe work behind explicit deep/all commands. Any remaining deep-mode table rendering bottleneck should be handled as a narrower follow-up with current-main timing evidence and a profile of the slow section.
