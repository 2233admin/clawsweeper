## Summary

Keep this PR open. Current main does not implement the PR's intended Feishu message recall/delete/list tool: the Feishu plugin still lacks `extensions/feishu/src/message.ts` and does not advertise or handle Feishu `delete`/`recall` message actions. The PR remains potentially useful, but it needs a current-main refresh and maintainer review because later review comments still identify schema, config-schema, and multi-account routing blockers for a destructive message-management feature.

## Best Possible Solution

Keep the PR open and ask for a refresh against current main. The best implementation direction is likely to wire Feishu recall/delete through the existing generic message action adapter, or clearly justify a separate Feishu tool, then use JSON Schema/TypeBox parameters, add `tools.message` to the strict config schema if that toggle remains, route through the active Feishu account, add focused tests for delete/recall/list behavior and multi-account routing, and document the user-facing Feishu message action surface.
