## Summary

Keep this PR open. Current main only has the schema/status half of per-agent contextTokens from #71247; the embedded runner still resolves runtime context budgets from model metadata and agents.defaults.contextTokens, not agents.list[].contextTokens. The submitted PR is also incomplete as-is because review discussion identifies missing compaction/extension call sites and an override-precedence bug.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a corrected rebase: make agents.list[].contextTokens an authoritative per-agent override over agents.defaults.contextTokens while still respecting the resolved model window, thread the active agent id or resolved agent config through embedded run setup, direct compaction, queued/context-engine compaction, and extension factory setup, then add focused tests for main run, compaction, extension/runtime budgeting, and the global-default override case.
