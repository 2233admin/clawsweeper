## Summary

Keep open. Current main has the schema/status portion of per-agent `contextTokens`, but it does not implement the runtime context-window override this PR is trying to add. The submitted PR is also incomplete as-is: the review discussion correctly points out missing compaction/extension paths and an override-precedence bug.

## Best Possible Solution

Keep this PR open and ask for a corrected rebase: make per-agent `contextTokens` an authoritative override over `agents.defaults.contextTokens`, thread the active agent id or resolved agent config through embedded run setup, compaction, queued/context-engine compaction, and extension factory setup, then add focused tests for main run, compaction, extension/runtime budgeting, and the global-default override case.
