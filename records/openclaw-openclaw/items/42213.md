## Summary

Keep this issue open. Current main still uses `chatRunId` as the Control UI busy latch, normal follow-up messages still queue while that latch is set, and I did not find a connected Control UI `agent.wait` watchdog/recovery path. The same-author implementation candidate #42258 is open and unmerged with closing syntax for this issue, so the maintainer path is to review/land or close that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open and review #42258 as the implementation candidate. Land that PR or an equivalent Control UI recovery that probes `agent.wait` for the active `chatRunId` after inactivity, clears busy state only on terminal wait results, and includes regression coverage for stale-finished recovery, still-active timeout behavior, watchdog reset/cancellation, and queued-message flushing.
