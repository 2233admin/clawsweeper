## Summary

Keep this PR open. Current main still scopes Control UI session tokens by the exact parsed gateway host, so the loopback alias sharing proposed here is not implemented; however, the PR should not merge unchanged until maintainers resolve whether collapsing all numeric 127/8 hosts is an acceptable bearer-token boundary.

## What This Changes

The PR adds loopback-host normalization to Control UI gateway token/settings scope, adds storage tests for localhost/127/8/[::1] sharing and isolation boundaries, and adds a changelog entry.

## Best Possible Solution

Land this PR or an equivalent focused change after maintainers explicitly choose the loopback credential boundary. The safer default is to normalize common true aliases such as localhost, 127.0.0.1, and [::1], preserve protocol/port/path/DNS-host isolation, keep the positive and negative tests, and leave #69647 as a separate reverse-proxy path bug.

## Reproduction Assessment

Yes. Current-main behavior can be reproduced by saving a session token for one gateway URL and observing that normalizeGatewayTokenScope keeps localhost, 127.0.0.1, 127.255.255.255, and [::1] as distinct parsed hosts; the PR's added tests cover the intended same-storage path.

## Solution Assessment

Unclear as written. The helper-and-test shape is a narrow maintainable place to fix Control UI token scope, but treating all 127/8 addresses as interchangeable is broader than true aliasing unless maintainers deliberately accept that local trust boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Narrow loopback token sharing to true aliases:** `ui/src/ui/storage.ts:26-27`
  - body: The helper treats every numeric 127/8 address as the same token scope. Two local gateways can be bound to different loopback addresses on the same port/path, so a token saved for `127.0.0.1:18789` would be reused when the UI is pointed at `127.0.0.2:18789`, despite the PR claiming distinct gateways stay isolated. Please narrow the normalized IPv4 set or make the broader 127/8 trust boundary explicit.
  - confidence: 0.74
