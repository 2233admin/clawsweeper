## Summary

Keep this PR open. Current main still scopes Control UI gateway tokens by the exact parsed gateway host, so the localhost/127.0.0.1/[::1] token-sharing behavior proposed here is not implemented. The PR is a focused repair of the closed, unmerged #43254 branch, but it needs maintainer/security review because the submitted token-scope change intentionally shares a bearer token across all numeric 127/8 loopback aliases on the same protocol, port, and path.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land this PR or an equivalent focused change after resolving the 127/8 token-sharing security question, preserving protocol/port/path boundaries, keeping DNS hostnames isolated, and retaining the positive and negative storage tests. #69647 should remain separate because it tracks reverse-proxy path persistence, not loopback alias token scope.
