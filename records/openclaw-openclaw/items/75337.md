## Summary

Keep open. The PR has a protected maintainer label and the code-side timeout increases are coherent, but the proposed docs sentence overstates the behavior for other bundled hosted image providers that still fall back to the shared 60s HTTP default when no request/config timeout is supplied.

## What This Changes

The PR raises Google, OpenRouter, and xAI image-generation provider defaults to 180 seconds, adds OpenRouter/xAI timeout assertions, and documents hosted image-provider defaults.

## Best Possible Solution

Land a corrected PR that keeps the timeout behavior and docs aligned: either narrow the docs to the providers this patch actually changes, or intentionally standardize the remaining hosted providers with matching targeted tests.

## Reproduction Assessment

Not applicable as a user bug report. The review has a high-confidence static reproduction path: inspect each provider's timeout fallback and the shared HTTP helper's 60s default on current main, then compare that to the PR's added docs claim.

## Solution Assessment

No. The provider timeout increases are a reasonable narrow fix, but the proposed docs wording is broader than the implementation; the safer solution is to narrow that sentence unless maintainers want a universal hosted-provider timeout change.
