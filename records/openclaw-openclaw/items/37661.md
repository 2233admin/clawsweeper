## Summary

Keep open. Current main still lacks an assistant text repetition or n-gram guard, and the existing duplicate protections only cover tool-call loops, repeated text_end/full-content cases, or exact block-reply payload duplicates. The April 11 comment reports pure assistant text repetition on 2026.4.10 outside subagent spawning, so closed #34574 and #39536 do not fully cover the remaining bug.

## Best Possible Solution

Keep this issue open and implement a conservative assistant-output repetition guard in the streaming/final reply path, plus a shared per-turn outbound duplicate/message cap before channel delivery. Keep #34574 scoped to tool-call loops and treat closed #39536 as heartbeat-specific context rather than the generic fix.
