## Summary

Keep this open. Current main still lets inline Control UI/WebChat HEIC or HEIF attachments keep their resolved MIME through the current-run prompt path, and #50320 remains an open unmerged implementation candidate with closing syntax for this issue.

## What This Changes

Normalize HEIC/HEIF images uploaded through the Control UI/WebChat before the current-run provider call, not only in later persistence or history paths.

## Best Possible Solution

Land a narrow fix, from #50320 or a split/replacement, that converts HEIC/HEIF inline chat attachments to JPEG before `parsedImages` reaches dispatch and `activeSession.prompt`, rechecks the converted size against image limits, preserves non-HEIC behavior, and adds regression coverage for the chat attachment and prompt dispatch path.
