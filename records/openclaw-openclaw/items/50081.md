## Summary

Keep open. Current main still allows inline WebChat/Control UI HEIC or HEIF attachments to remain current-run prompt images without JPEG normalization, and open PR #50320 already uses closing syntax for this issue but remains unmerged.

## What This Changes

Normalize HEIC/HEIF Control UI and WebChat current-run prompt images to JPEG before provider prompt submission so OpenAI vision models do not receive `image/heic` payloads.

## Best Possible Solution

Land a narrow fix, from #50320 or a split replacement, that converts HEIC/HEIF inline chat attachments to JPEG before `parsed.images` reaches `activeSession.prompt`, rechecks converted output against image limits, preserves non-HEIC and offloaded-media behavior, and adds focused regression coverage for chat attachment parsing and prompt dispatch.
