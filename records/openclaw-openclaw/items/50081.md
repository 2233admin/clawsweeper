## Summary

Keep this issue open. Current main still preserves inline Control UI/WebChat HEIC or HEIF attachments as current-run prompt images, while HEIC-to-JPEG normalization exists on other input/media paths. The provided GitHub context also shows open PR #50320 uses closing syntax for this issue, so maintainer follow-up should happen there before this issue is closed.

## What This Changes

The issue requests converting Control UI/WebChat HEIC and HEIF uploads to JPEG before current-run prompt images are sent to OpenAI vision models.

## Best Possible Solution

Land a narrow fix, from #50320 or a split/replacement, that converts HEIC/HEIF inline chat attachments to JPEG before `parsed.images` reaches `activeSession.prompt`, rechecks the converted output against image limits, preserves non-HEIC behavior, and adds regression coverage for chat attachment parsing and prompt dispatch.
