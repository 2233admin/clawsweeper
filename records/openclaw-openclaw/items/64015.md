## Summary

Keep open. Preserve the current fixes for Discord health/status, plugin allowlist, and Claude CLI gating with focused regressions.

## Repair Work Prompt

Fix the remaining diagnostics false positive from this issue: `openclaw doctor --deep` reports `Gateway token is managed via SecretRef and is currently unavailable` even when `gateway.auth.token` is a resolvable SecretRef and `openclaw secrets audit` would report zero unresolved refs. Current source shows `runGatewayAuthHealth` in `src/flows/doctor-health-contributions.ts` calls sync `resolveGatewayAuth`, while `src/gateway/auth-resolve.ts` deliberately ignores SecretRef values. Use the existing async gateway auth SecretRef helpers in `src/gateway/auth-config-utils.ts` or the same resolver contract used by secrets audit/status so doctor suppresses the warning when the applicable SecretRef resolves in the command path. Keep doctor read-only: do not persist resolved plaintext, do not generate replacement tokens for SecretRef-managed auth, and keep the existing warning when the applicable ref is unresolved. Preserve password/none/trusted-proxy behavior. Add focused regression coverage for a resolvable env SecretRef token and the existing unresolved case. Also add or update a small assertion only if needed to ensure disabled `memory-core` no longer produces a bundled public-surface access error; do not refactor plugin loading, channel status, or Discord runtime code. Add a user-facing changelog Fixes entry crediting `Thanks @RotoDude93` unless maintainers classify this as internal-only.
