## Summary

Keep open. Current main still has the OpenResponses SSE mismatch this PR targets: streamed pending-tool-call responses build a response resource with `status: "incomplete"` but emit a `response.completed` terminal event, and the current schema still lacks `response.incomplete`. The PR is a valid small bug fix, but it is stale relative to current main and should be reviewed/rebased rather than closed.

## What This Changes

This PR corrects OpenResponses reference URLs, adds a `response.incomplete` streaming event type, changes streamed pending-tool-call responses to emit `response.incomplete`, and adds regression coverage for that SSE path.

## Best Possible Solution

Update the OpenResponses handler, schema, docs, and current gateway regression tests so streamed pending-tool-call responses terminate with `response.incomplete` while ordinary completed streams continue to emit `response.completed`.
