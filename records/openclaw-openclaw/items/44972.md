## Summary

Keep open. Current main still emits `response.completed` while carrying an `incomplete` response for pending tool-call streams, and the schema, test, and public docs still reflect the old terminal event; this PR targets the right gateway path but needs maintainer review/rebase and a small docs update before merge.

## What This Changes

The PR corrects OpenResponses doc-comment URLs, adds a `response.incomplete` streaming event schema/type, changes the pending-tool-call SSE terminal event to `response.incomplete`, and adds gateway regression assertions.

## Best Possible Solution

Land a rebased version that keeps the handler, streaming schema/types, public SSE docs, and regression tests aligned so pending-tool-call streams terminate with `response.incomplete` while normal completed streams continue to emit `response.completed`.
