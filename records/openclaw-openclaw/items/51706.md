## Summary

Keep this PR open. Current main already shipped the original runtime-model display fix, but this branch is stale against the current `src/status` split and its fallback-preservation attempt can still hide active fallback state for channel/group-selected sessions whose selected model is not the global default.

## What This Changes

This PR changes `session_status` and status rendering tests so persisted runtime session models are shown when no explicit session override exists, then adds a fallback-preservation attempt.

## Best Possible Solution

Land a focused current-main replacement that separates display-only runtime identity from selected-model semantics in the `src/status` APIs, keeps ordinary persisted runtime model display, preserves unknown-provider display, and keeps active fallback visibility correct for explicit overrides plus channel/group-selected sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use the stored fallback-selected model:** `src/agents/tools/session-status-tool.ts:456-464`
  - body: When fallback notice fields exist, this branch forces `configured.provider`/`configured.model`. That only matches fallback state for sessions selected by the global default; channel or group routing can persist a different `fallbackNoticeSelectedModel`, so `buildStatusMessage` will fail the selected-vs-active match and hide the active fallback line. Resolve the selected ref from the stored fallback notice instead of the global default.
  - confidence: 0.91
