## Summary

Keep open. Current main still exposes the relevant compaction token settings as absolute numeric values only, and the runtime path that receives the active context window only uses it to cap the existing numeric reserve floor, not to parse percentage strings.

## What This Changes

Request support for percentage-valued compaction token settings so values like reserve headroom and recent-tail budget scale across model context windows.

## Best Possible Solution

Add a small typed compaction token-budget value contract that preserves current numeric behavior, accepts documented percentage strings for the relevant compaction token fields, resolves them against the active context window at runtime, and updates schema generation, docs, changelog, and focused tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but the static verification path is high confidence: current types, zod schema, generated schema, docs, and runtime resolver all show number-only behavior with no percentage parser.

## Solution Assessment

Is this the best way to solve the issue? Yes: supporting explicit percentage strings is narrower and safer than auto-scaling existing numeric values, because it preserves backward-compatible absolute-token configs while adding portable model-relative config.
