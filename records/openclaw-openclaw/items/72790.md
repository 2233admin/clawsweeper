## Summary

Keep open. Current main has related context-window-aware capping for the numeric reserve floor, but it still does not accept percentage-based compaction config values in the public type, Zod schema, generated schema, docs, or runtime resolver.

## What This Changes

The issue requests percentage-based compaction token settings so values such as `"10%"` and `"15%"` scale against the active model context window instead of staying absolute token counts.

## Best Possible Solution

Add a typed token-budget value contract that preserves existing numeric configs, accepts documented percentage strings for the compaction token settings, resolves them against the effective model context budget at runtime, and updates schema generation, docs, changelog, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence static verification path. Current main's type, Zod schema, generated schema, docs, and runtime resolver all accept numbers only for these settings.

## Solution Assessment

Yes. Explicit percentage-string support is the narrowest maintainable direction because it keeps existing numeric configs backward-compatible; silently auto-scaling existing absolute numbers would change established behavior.
