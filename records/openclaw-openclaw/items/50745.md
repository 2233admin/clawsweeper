## Summary

Keep this PR open. Current main still does not expose `lookupPnLidEntry` or `PnLidEntry` on the WhatsApp active-listener/runtime API, so the PR is not obsolete or implemented on main. The proposed feature is plausible, but the PR still needs maintainer review and fixes for the authDir LID fallback, stale/destructive changelog edit, and the privacy implications of exposing phone-number resolution to plugins.

## Best Possible Solution

Keep the PR open as a focused WhatsApp implementation candidate. The best path is to rebase it on current main, preserve existing changelog entries, implement the lookup through the existing LID normalization/fallback semantics (`authDir` reverse mapping before Baileys lookup where applicable), add tests for `@lid`, `@hosted.lid`, phone JID, plain phone, and authDir/cold-cache fallback, and get maintainer product review on the plugin-visible privacy boundary before merging.

- The PR changelog patch is stale against current main and reportedly removed unrelated release notes; it should be rebased and changed to an append-only entry before merge.
