## Summary

Keep this PR open. Current main still builds outbound delivery payloads from raw reply text, so the user-visible metadata leak tracked by #39847 is not implemented on main. The PR remains a plausible fix candidate, but it overlaps with same-author open PR #51653 and still needs maintainer choice on sanitizer scope.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Choose one canonical implementation between this PR and #51653, rebase it onto the current createOutboundPayloadPlanEntry() flow, strip echoed inbound metadata before directive parsing with the narrowest outbound-safe sanitizer, and add regression tests proving metadata echoes are removed while timestamp-prefixed normal replies and intentionally quoted metadata examples are preserved. After one patch lands, close the duplicate PR and #39847 as implemented.
