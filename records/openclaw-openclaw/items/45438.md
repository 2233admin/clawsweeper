## Summary

Current main fixed the reported session-store cache/load structuredClone hot path, but auth-profile cache and runtime auth snapshot clone helpers still call structuredClone, and maintainer discussion explicitly kept the issue open for that remaining path plus benchmark or soak confirmation.

## What This Changes

The issue requests replacing structuredClone-based cloning in session-store and auth-profile store/cache paths to avoid native memory growth and gateway OOMs under large session workloads.

## Best Possible Solution

Finish or explicitly defer the auth-profile clone cleanup in the auth-profile owner area, add focused mutation-isolation/no-structuredClone coverage for persisted and runtime auth stores, and validate with the reported large-store benchmark or a production-shaped soak before closing. Keep broader gateway memory accumulation on #54155 and #51264.
