## Summary

Keep open. Current main has removed the reported session-store cache/load `structuredClone` hot path, but the auth-profile cache, runtime snapshot, and external OAuth overlay paths still clone auth stores with `structuredClone`, and maintainer discussion explicitly kept the issue open for that remaining work plus benchmark/soak confirmation.

## What This Changes

The issue asks to replace `structuredClone` in session-store and auth-profile clone hot paths with JSON/serialized-data cloning to reduce native memory growth in large gateway workloads.

## Best Possible Solution

Finish or explicitly defer the auth-profile clone cleanup in the auth-profile owner area, add focused mutation-isolation and no-`structuredClone` coverage for persisted/runtime auth stores, then validate with the reported large-store benchmark or a gateway soak before closing. Keep broader gateway memory growth on #54155 and #51264.
