## Summary

Keep open. Current main has landed the session-store portion of the fix, but the auth-profile clone path still uses structuredClone and a maintainer explicitly kept this issue open for that remaining path plus benchmark/soak confirmation.

## What This Changes

The issue asks to eliminate structuredClone-based cloning from session-store cache/load and auth-profile store hot paths because large stores can drive native memory growth and gateway OOMs.

## Best Possible Solution

Keep this issue open until maintainers finish or explicitly defer the auth-profile structuredClone cleanup, add targeted auth-profile mutation-isolation coverage, and run the reported benchmark/soak workload after the session-store fix. Broader session accumulation and all-agent loading concerns should stay tracked on #54155 and #51264 rather than being folded into this close decision.
