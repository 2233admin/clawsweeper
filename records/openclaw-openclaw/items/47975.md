## Summary

Keep open. Current main has partial mitigations for stale child-link display and task lost-state reconciliation, but the central requested behavior is still not implemented: ordinary run-mode subagent sessions default to retained history, keep-mode runs do not get archive timers, and unfiltered session-list/Control UI paths still enumerate stored completed child rows.

## What This Changes

The issue requests that completed one-shot subagent sessions be cleaned up or hidden promptly so stale child sessions do not make the main session or Control UI appear unresponsive after subagent completion.

## Best Possible Solution

Define the intended contract for one-shot subagent history: whether terminal run-mode child sessions are hidden, archived, or pruned by default, how cleanup=keep remains inspectable, and how Control UI/status surfaces distinguish active work from retained history. Then implement that contract consistently across subagent registry cleanup, sessions.list defaults, task reconciliation, attachment retention, docs, and focused UI/session tests, leaving lane-starvation work to narrower trackers such as #64253 and #54488.
