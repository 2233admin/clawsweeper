## Summary

Keep open. Current main still retains default one-shot subagent sessions and exposes stored completed child rows through normal session listing and Control UI paths. Recent liveness and task reconciliation work reduces some stale display and lost-task symptoms, but it does not settle the cleanup, retention, and visibility contract reported here.

## What This Changes

The issue asks for completed one-shot subagent sessions to be cleaned up or hidden so stale child sessions do not leave the main session or Dashboard appearing unresponsive.

## Best Possible Solution

Define the intended contract for terminal one-shot subagent history: whether completed run-mode child sessions are hidden, archived, or pruned by default; how cleanup="keep" remains inspectable; and how status surfaces distinguish active work from retained history. Then implement that contract consistently across subagent registry cleanup, sessions.list defaults, Control UI filtering/status display, task reconciliation, attachment retention, docs, and focused regression tests.
