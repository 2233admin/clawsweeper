## Summary

Keep open. Current main still retains one-shot subagent session rows by default, `sessions.list` still returns terminal subagent rows, the Control UI renders those rows, and task reconciliation can still treat a stale subagent task as backed when the child session row exists. Related reports narrow parts of the problem, but none fully supersedes this broader session persistence and responsiveness issue.

## Best Possible Solution

Keep this open as the broader subagent lifecycle/responsiveness tracker. Maintainers should decompose it into a product decision for one-shot subagent retention, a focused implementation for hiding or pruning terminal run-mode child sessions, task reconciliation that can mark stale running subagent tasks terminal when no live run backs them, and scheduler/announce-path fixes tracked by the narrower related issues.
