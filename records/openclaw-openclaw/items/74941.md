## Summary

Keep open. The underlying current-main bug is real, but the PR should not merge as-is because the new restore helper appends deferred inspected events after any tail events queued during the heartbeat run, which can reorder the system-event FIFO contract.

## What This Changes

The PR adds a system-event restore helper, changes exec-heartbeat cleanup to restore non-exec inspected events, and adds regression tests for mixed exec/non-exec heartbeat queues plus duplicate deferred reminders.

## Best Possible Solution

Keep the narrow deferred-event preservation fix, but restore deferred inspected entries ahead of any uninspected tail only after the inspected prefix is actually consumed, while preserving duplicate reminders, `MAX_EVENTS` trimming, and queue metadata semantics with focused tests.

## Reproduction Assessment

Yes. A deterministic unit path is to snapshot `[Reminder A, exec E]`, enqueue `Reminder B` before cleanup, then consume the inspected prefix and restore deferred entries; the PR's append-based helper yields `[Reminder B, Reminder A]` instead of preserving FIFO order.

## Solution Assessment

No. Restoring deferred entries is the right direction, but appending them after the current queue is not the best implementation; a safer helper should rebuild or prepend the deferred inspected entries ahead of the uninspected tail and guard on successful prefix consumption.

## Review Findings
