## Summary

Keep open. The PR addresses a real remaining current-main bug where exec-event heartbeat cleanup consumes non-exec system events, but the proposed restore helper appends deferred older events after any uninspected events queued during the heartbeat run, which can reorder reminders and other system events.

## What This Changes

The PR adds a `restoreSystemEventEntries` queue helper, changes exec-event heartbeat cleanup to restore deferred non-exec events after consuming the inspected prefix, and adds regression tests for mixed exec/non-exec queues and duplicate deferred reminders.

## Best Possible Solution

Keep the narrow deferred-event preservation fix, but make restoration preserve FIFO order relative to any uninspected tail and avoid restoring if prefix consumption did not actually remove the inspected events. Add a focused test for a later queued event so the queue contract stays explicit.

## Reproduction Assessment

Yes. A deterministic unit path is to snapshot `[Reminder A, exec E]`, enqueue `Reminder B` before cleanup, then consume the inspected prefix and restore deferred entries; the PR's append-based helper yields `[Reminder B, Reminder A]` instead of preserving the older deferred reminder first.

## Solution Assessment

No. Restoring deferred entries is the right direction, but appending them after the current queue is not the best implementation; the safer fix should rebuild or prepend the deferred inspected entries ahead of the uninspected tail while preserving duplicate reminders and `MAX_EVENTS` semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve deferred-event FIFO order:** `src/infra/system-events.ts:191`
  - body: `restoreSystemEventEntries` appends restored entries after whatever remains in the queue. Because heartbeat cleanup runs after the model call, another system event can be queued after preflight but before cleanup; `consumeSystemEventEntries` intentionally leaves that uninspected tail in place. This patch then turns `[older reminder, exec]` plus later `new reminder` into `[new reminder, older reminder]`, so newer events can be delivered before older deferred ones. Restore the deferred prefix ahead of the tail, and only restore after a successful consume.
  - confidence: 0.86
