## Summary

Current `main` still does not implement the dedicated real-time Agent Activity dashboard proposed by this draft PR. The Control UI has no Activity tab, route, controller, styles, or render branch; the gateway event list has no dedicated activity event family; and the existing live agent UI remains scoped to chat tool cards, thinking stream display, and lower-level tool/session events rather than an execution tree, timeline, and metrics dashboard. The PR is still a plausible implementation thread for remaining activity-observer work, especially with #12831 and #54577 still open, so it should not be closed as obsolete or implemented.

## Best Possible Solution

Keep this draft PR open as a possible implementation thread, but require maintainer design and security review before merge. The best path is to split or rework it into a narrow additive gateway activity contract plus a focused Control UI Activity panel, align remaining scope with #12831 and #54577, avoid sensitive metadata persistence by default, harden or remove `unsafeHTML`, regenerate i18n through the normal pipeline, and close this PR only after a reviewed implementation lands or a clearer canonical replacement is opened.
