## Summary

Keep this issue open. Current main still routes Discord native commands and Slack slash commands through config captured at provider startup, and `resolveAgentRoute` still derives DM session keys from the supplied `cfg.session.dmScope`. Telegram has the fresher runtime-config pattern, but Discord and Slack do not. The issue is also paired with open closing-syntax PR #39617, so the issue should stay open until that PR is merged, closed, or replaced.

## Best Possible Solution

Keep this issue open and review, rebase, and land PR #39617 or an equivalent maintainer-approved fix. The narrow fix should refresh routing config for Discord native command and Slack slash command route resolution, preserve Telegram’s fresh-config behavior, and add focused regression tests proving native/slash DM commands honor `session.dmScope` without recreating the provider.
