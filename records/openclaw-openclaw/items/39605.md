## Summary

Keep open. Current main still resolves Discord native command interactions and Slack slash commands with provider-startup config, while Telegram native commands already reload runtime config before route resolution. The supplied timeline also shows open PR #39617 is an unmerged implementation candidate linked to this issue, so the issue should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

Native slash commands in Discord, Telegram, and Slack DMs should respect `session.dmScope` instead of routing to the main session.

## Best Possible Solution

Review, land, or replace #39617 with a narrow channel-owned fix that refreshes route-sensitive config for Discord and Slack native command paths, leaves Telegram’s already-fresh behavior intact, and adds regression tests proving non-main `session.dmScope` direct-message session keys are used.
