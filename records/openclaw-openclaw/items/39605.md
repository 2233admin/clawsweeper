## Summary

Keep open. Current main still routes Discord native command interactions and Slack slash commands through provider-startup config even though route selection depends on `cfg.session.dmScope`; Telegram already uses a fresh runtime config path. The issue is also paired with open PR #39617, so it should remain open until that PR is reviewed, landed, or replaced.

## What This Changes

The issue asks native Discord, Telegram, and Slack slash commands in DMs to respect `session.dmScope` and route to the same per-peer session keys as normal messages.

## Best Possible Solution

The end state should be a narrow channel-owned fix that refreshes route-sensitive config for Discord and Slack native/slash command route resolution, keeps Telegram’s already-fresh path intact, and adds regression coverage for non-main DM session keys.
