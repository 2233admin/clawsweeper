## Summary

Keep this issue open. Current main still routes Discord native command interactions and Slack slash commands through provider-startup config objects, while Telegram native commands already reload runtime config before route resolution; the issue is also paired with open PR #39617, so it should remain open until that PR is reviewed, replaced, or merged.

## What This Changes

The issue requests Discord, Telegram, and Slack native slash commands in DMs to respect fresh `session.dmScope` routing instead of falling back to the main session.

## Best Possible Solution

A narrow channel-owned fix should resolve Discord and Slack native-command DM targets with fresh runtime config, preserve Telegram's already-fresh path, and add regression coverage for non-main `session.dmScope` values.

## Reproduction Assessment

Yes. The issue gives concrete channel steps, and static current-main verification shows `resolveAgentRoute` depends on `cfg.session.dmScope` while Discord and Slack native command paths still pass startup config into route resolution.

## Solution Assessment

Yes. Reloading route-sensitive runtime config at native command handling time for Discord and Slack is the narrowest maintainable fix; this does not need a core migration or broader routing redesign.
