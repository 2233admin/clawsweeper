## Summary

Keep this issue open. Current main still does not implement the reported `volcengine-plan/doubao-seed-2.0-pro` imageModel path: the explicit imageModel route is still honored, but the bundled Volcengine coding provider remains text-only, the exact reported model is absent from the Volcengine catalog, and there is no Volcengine-owned media-understanding adapter proving this Ark coding image endpoint works through OpenClaw. The related #55688 remains an open partial catalog-capability candidate, not a merged fix for the exact model and endpoint.

## What This Changes

The issue reports that configuring `agents.defaults.imageModel.primary` as `volcengine-plan/doubao-seed-2.0-pro` causes OpenClaw image handling to fail against the Volcengine Ark coding API even though direct API calls to the same endpoint succeed.

## Best Possible Solution

Keep the issue open until maintainers verify the exact Volcengine Ark coding image endpoint and model ID through OpenClaw. Then either add Volcengine-owned image-capable catalog/media-understanding support with regression coverage, or explicitly document `volcengine-plan` imageModel as unsupported and guide users to a supported image-capable `volcengine/*` model or a custom provider config with `input: ["text", "image"]`.
