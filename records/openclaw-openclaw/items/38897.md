Current main still does not implement this PR’s intended runtime-aware Mission Control surface. ## What This Changes

This PR adds Gateway and Control UI runtime-ops surfaces for dashboard summary/delta, incidents, tool activity, execution queue visibility, autonomy posture, and related polling/pivots, while also touching unrelated workspace ignores, local note files, scripts, Docker/test helpers, and UI navigation/style files.

## Best Possible Solution

The best path is a narrower runtime-ops PR that deliberately adds `dashboard.summary`/`dashboard.delta`, a global tool activity registry, process visibility with finished-session pivots, and autonomy posture based on existing `tools.effective` and exec-approval policy data; it should fix the known `requestContext` and `isError` blockers, integrate with the background task ledger where appropriate, add focused Gateway/UI/docs tests, and drop unrelated workspace/script/test churn.
