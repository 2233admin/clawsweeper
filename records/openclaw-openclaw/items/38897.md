## Summary

Current main still lacks this PR's central runtime-aware Mission Control surface. The Gateway contract has no `dashboard.summary`, `incident.*`, or `dashboard.delta`; the request context has no dashboard registries; finished background process records still do not retain `sessionKey`; and the Control UI overview remains the existing health/usage/sessions/skills/cron/auth/logs surface rather than Tool Runtime, Execution Queue, and Autonomy Posture. The PR also remains broad and has unresolved review/security concerns, so it should stay open for maintainer judgment or contributor rework.

## Best Possible Solution

Keep this PR open for maintainer judgment or contributor rework. The best path is a narrower runtime-ops PR that adds a deliberate `dashboard.summary`/`dashboard.delta` contract, a global tool activity registry, execution/process visibility with finished-session pivots, and an autonomy posture view backed by existing `tools.effective` and exec-approval policy data. It should fix the known `isError` and startup `requestContext` bugs, reuse or clearly integrate with the existing background task ledger and `session.tool` events, add focused Gateway/UI/docs tests, and drop unrelated workspace/script/test churn.
