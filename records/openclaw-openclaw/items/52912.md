## Summary

Keep open: the PR is not merge-ready because it broadens credential propagation for all plugin OAuth credentials while leaving the linked models auth token entrypoints unresolved; current main still preserves explicit OAuth copy opt-in and main-agent read-through semantics.

## What This Changes

The PR routes plugin provider auth profile writes through a shared helper, syncs Anthropic token and OAuth profiles to sibling agent directories, adds Anthropic token sibling-sync coverage, and adds DeepSeek to the provider contract registry.

## Best Possible Solution

Land a narrower maintainer-approved auth fix that covers Anthropic setup-token/manual-token persistence across interactive plugin auth, models setup-token, paste-token, and non-interactive setup where needed; preserve OAuth copy as explicit opt-in; and split unrelated contract-registry cleanup into its own change.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: the PR predicate returns true for all OAuth credentials, while current-main models auth and Anthropic non-interactive token paths still write only one agentDir. No runtime tests were run because this review was read-only.

## Solution Assessment

No. The PR is not the best way to solve the linked issue because it expands OAuth refresh-token copying and leaves reported token entrypoints untouched; a safer fix should centralize only the intended Anthropic token persistence behavior and keep OAuth portability explicit.
