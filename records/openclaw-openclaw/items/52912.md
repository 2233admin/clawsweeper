## Summary

Keep this PR open: the branch is a useful partial fix for Anthropic setup-token sibling sync, but it is not mergeable as-is because it broadens OAuth refresh-token propagation across sibling agents and does not cover the `models auth setup-token` / `paste-token` paths described by the linked regression.

## What This Changes

The PR routes plugin provider auth profile writes through a new sibling-sync helper, syncs Anthropic token and OAuth credentials to sibling agent directories, adds Anthropic token sibling-sync coverage, and adds DeepSeek to the bundled provider contract registry.

## Best Possible Solution

Land a narrower maintainer-approved fix that centralizes Anthropic token profile persistence for the intended setup-token and manual-token contexts, covers plugin auth-choice plus `models auth setup-token`, `models auth paste-token`, and non-interactive setup-token as appropriate, keeps OAuth copying explicit via provider/context opt-in, and separates the DeepSeek registry cleanup from the auth fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restrict OAuth auth-profile sync to explicit opt-ins:** `src/plugins/provider-auth-helpers.ts:222-223`
  - body: `shouldSyncSiblingAgentsForCredential` returns true for every OAuth credential, so plugin OAuth logins would copy refresh material into all sibling agent stores. That conflicts with the documented default that OAuth profiles are non-portable unless explicitly opted in and weakens per-agent credential isolation; gate sibling sync to the Anthropic token case or a provider/context opt-in.
  - confidence: 0.92
- **[P2] Cover the models auth token entrypoints:** `src/plugins/provider-auth-choice.ts:133-135`
  - body: The linked regression uses `models auth setup-token` and `paste-token`, but this patch only changes `runProviderPluginAuthMethod`. Current `models auth` and Anthropic non-interactive token writers still persist to one agent store, so the reported multi-agent failure can remain after merging this branch.
  - confidence: 0.86
