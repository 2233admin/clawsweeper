## Summary

Keep this PR open. The reported `sessions.list` bug is still present on current `main`, and the PR contains useful targeted work, but the latest branch is stale/dirty against the current session-store split and still has two correctness defects before it can merge.

## What This Changes

The PR changes gateway session-store combination so literal `agents/<id>/sessions/sessions.json` paths discover sibling per-agent stores scoped to that agents root, with regression tests for the literal-path case.

## Best Possible Solution

Adapt the fix to `src/config/sessions/combined-store-gateway.ts`: literal paths that resolve to `agents/<id>/sessions/sessions.json` should discover sibling stores under that same agents root, return the resolved literal store path for downstream transcript/usage lookups, derive literal-layout target agent ids from disk paths, and preserve single-store behavior for non-agent-layout literal paths.

## Review Findings

  - body: Literal agent-layout stores now skip the single-store branch, but the multi-store return still uses the raw `storeConfig` string. If another agent appears first in `agents.list`, this loop canonicalizes legacy keys from the main store under that other agent id. Use the agent id encoded in the store path or discovered directory for literal-layout targets.
  - confidence: 0.86
