## Summary

Keep this PR open. Current main still only applies the Claude-name image-support fallback to `claude-cli`, while direct `anthropic` Claude models remain a documented/default product path, and the PR is a small focused fix with matching regression tests.

## What This Changes

The PR extends gateway image-support detection so direct `anthropic` Claude model names share the existing stale/missing-catalog fallback used for `claude-cli`, and adds two resolver regression tests.

## Best Possible Solution

Review this focused PR and either land the direct `anthropic` Claude-name fallback with its two regression tests, or replace it with a provider-owned capability resolver that still preserves images for direct Anthropic Claude models when catalog metadata is stale or absent.
