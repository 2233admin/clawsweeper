## Summary

Keep open. Current `main` still renders assistant copy/expand controls separately from the bubble class that reserves right padding, and the latest release tag contains the same mismatch. The existing issue comment's CSS/render diagnosis matches the current source, while closed #66155 is related but does not provide an open canonical fix.

## What This Changes

The issue requests fixing WebChat/Control UI message bubbles so copy/expand actions do not cover or truncate message text after the 2026.3.31 upgrade.

## Best Possible Solution

Apply the existing action-spacing contract when message actions render, or replace it with equivalent action-aware CSS, then add a focused renderer/layout test that proves assistant bubbles with copy/expand controls reserve space on hover and no-hover surfaces.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to render any assistant message with markdown and copy/expand actions enabled; current DOM renders `.chat-bubble-actions` while the bubble lacks `.has-copy`, and no-hover devices keep those actions visible.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the Control UI grouped message renderer and its colocated tests/styles; no gateway, protocol, or core behavior change is needed.
