## Summary

Keep open. Current main still does not implement this PR's intended legacy OAuth default canonicalization: steady-state automatic session overrides remain pinned to the stored profile id, and the existing legacy-default helper is only used in repair and OAuth refresh-error fallback paths. The PR remains potentially useful, but it needs a rebase onto the current split auth-profile resolver and maintainer review of the OAuth identity-matching semantics raised in review.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, port the canonicalization into the current `src/agents/auth-profiles/session-override.ts` split imports, reuse or explicitly align with the existing OAuth identity helpers, preserve the negative identity-mismatch and rotation-progress regression tests from the PR discussion, then run the targeted auth-profile test lane and the appropriate changed gate before maintainer review.

## Risks / Open Questions

- Auth routing is security-sensitive: an overly permissive canonicalization rule could silently switch a session to a different OAuth account. The prior review comments already identified identity-mismatch edge cases, so maintainer review should align any port with `src/agents/auth-profiles/oauth-identity.ts`.
- The PR branch predates the current split auth-profile module shape, so it should be rebased and ported rather than merged as-is.
- No supply-chain, workflow, dependency, release, or package-publishing changes are visible in the provided PR file list; the security review concern is credential-routing semantics, not CI or dependency execution.
