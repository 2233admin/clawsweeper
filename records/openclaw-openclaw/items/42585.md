## Summary

Current main still does not implement this PR’s intended legacy OAuth default canonicalization. The session override resolver keeps an existing automatic `provider:default` override pinned unless a normal rotation trigger fires, while the existing legacy-default helper is only used in repair/OAuth refresh fallback paths. The PR remains a plausible focused auth-routing fix, but it needs rebase and maintainer review of identity-matching semantics before landing.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, port the canonicalization into the current split auth-profile resolver, reuse or explicitly align with existing OAuth identity helpers, preserve the negative identity-mismatch and rotation-progress regression tests from the PR discussion, then run the targeted auth-profile test lane and the appropriate changed gate before maintainer review.
