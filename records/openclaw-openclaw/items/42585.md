## Summary

Keep open. Current main still does not implement this PR's intended steady-state canonicalization from a legacy OAuth `provider:default` session override to a canonical non-default OAuth profile. The PR remains useful, but it needs a current-main rebase/port and careful maintainer review of identity-matching semantics before merge.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, port the canonicalization into the current `src/agents/auth-profiles/session-override.ts` shape, reuse or explicitly align with the existing OAuth identity helpers, preserve the negative identity-mismatch and rotation-progress regression tests from the PR discussion, then run the targeted auth-profile test lane and changed gate before maintainer review.
