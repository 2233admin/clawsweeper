## Summary

Keep open. Current main has provider setup docs, provider-owned model catalogs, per-model catalog listing status, plugin-contract deprecation policy, and live auth/probe status, but it still does not publish the requested provider x model-family x capability compatibility matrix with lifecycle deadlines, canonical issue links, generated docs, or doctor/models lifecycle output. A maintainer-side Codex review on 2026-04-26 also explicitly kept this issue open; the related Matrix regression #52899 is useful motivation but was plugin-specific and closed, not a canonical replacement for this broader policy proposal.

## Best Possible Solution

Keep this issue open until maintainers approve the provider support policy shape. The likely implementation path is to extend the existing modelCatalog/provider-index control plane or add a new lifecycle registry keyed by provider, model family, capability, status, deadline, replacement, and canonical issue link; generate a public docs page from it; and surface concise lifecycle state in openclaw doctor or openclaw models status while keeping live auth/probe health separate.
