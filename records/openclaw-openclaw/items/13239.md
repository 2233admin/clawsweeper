## Summary

Current main still does not publish the requested provider x model-family x capability compatibility matrix, lifecycle/deprecation policy, generated docs page, or doctor/models lifecycle output. Existing docs and CLI surfaces cover provider setup, provider-owned catalogs, auth status, live probes, and plugin compatibility contracts, but they do not provide the broader support-lifecycle policy requested here. The related Matrix plugin regression #52899 is useful motivation, but it was a closed plugin-specific issue and does not supersede this broader provider policy proposal.

## Best Possible Solution

Keep this issue open until maintainers approve the policy shape and ship a machine-readable provider lifecycle registry keyed by provider, model family, capability, status, deadline, replacement, and canonical issue link. Generate a public docs page from that registry and surface concise lifecycle state in `openclaw doctor` or `openclaw models status` while keeping live auth/probe health separate.
