## Summary

Current main does not implement the PR's intended terminal-field preference in `parseCliJsonl`. The parser still only special-cases Claude `result` events and generic `parsed.item.text` fallback, while current tests only cover Claude JSONL/error cases. Related phase-aware fixes on main cover other reply/history surfaces, not this CLI JSONL parser path, so this PR remains a valid implementation candidate.

## Best Possible Solution

Keep this PR open for normal maintainer review and landing. The best path is a narrow parser fix that prefers `task_complete.last_agent_message`, falls back to `phase: "final_answer"`, preserves existing Claude JSONL behavior, handles commentary fallback intentionally, and keeps focused regression tests in `src/agents/cli-output.test.ts`.
