## Summary

Keep this PR open. Current main still lacks the terminal-field preference this PR targets: `parseCliJsonl()` handles Claude result JSONL and generic `item.text` fallback, but does not read `task_complete.last_agent_message`, `phase: "final_answer"`, or suppress commentary-phase fallback text. The PR remains the active implementation candidate, though it is currently dirty against latest main.

## Best Possible Solution

Keep this PR open for maintainer review and landing. The best path is a narrow parser fix in `src/agents/cli-output.ts` that prefers `task_complete.last_agent_message`, falls back to assistant content tagged `phase: "final_answer"`, preserves Claude stream-json behavior, handles commentary-only structured output intentionally, and keeps focused regression coverage in `src/agents/cli-output.test.ts`. Refresh the branch against current main, then run the targeted parser tests plus the changed gate before merging.
