## Summary

Keep this PR open. Current main still only normalizes Homebrew Cellar Node paths, so the Volta tools/image path problem remains, and the PR is still a relevant implementation candidate. It should not be closed as obsolete, but it still needs the unresolved custom VOLTA_HOME and Windows layout gaps addressed before landing.

## Best Possible Solution

Keep this PR open and update it, or land a replacement, with Volta path resolution that handles default `~/.volta`, custom `VOLTA_HOME` roots such as `/opt/volta`, Windows Volta layouts, `.exe` shims, and inaccessible-shim fallback. Add focused coverage in `src/infra/stable-node-path.test.ts` and `src/daemon/runtime-paths.test.ts`, then use that implementation to resolve #52184.
