## Summary

Keep open. Current main does not implement the PR's intended `before_tools_resolve` hook or an equivalent list-level plugin API for filtering the final tool list before it reaches the model. Main still has per-tool execution hooks plus static/final tool policy, so this remains a real plugin API proposal and should get maintainer/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If accepted, adapt it to current `src/plugins/hook-types.ts` / `src/plugins/hooks.ts`, evaluate the hook once over the complete final tool set in both normal runs and compaction, include core, client, bundled MCP, and LSP tools, normalize equivalent tool names consistently, decide fail-closed behavior for hook errors, and add docs plus regression tests for allow intersection and bypass cases.
