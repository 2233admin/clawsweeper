## Summary

Keep open. Current main does not implement the PR's intended cron/main-session constraints: main-session cron events still embed rich reminder prose into heartbeat prompts, main systemEvent payloads are not limited to short wake tokens, and the agent/tool/docs surfaces still describe main-session systemEvent reminders as valid. The supplied PR diff is a plausible implementation candidate, but it also has unresolved functional review comments that should be addressed before landing.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase or adopt it, resolve the no-relay and legacy-job review comments, preserve isolated agentTurn reminder flows, update the cron docs/system prompt/tool guidance, and land with focused regression tests for main-session cron tokenization and legacy job operations.
