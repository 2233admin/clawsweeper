## Summary

Keep this PR open. Current main still has the cron/main-session behavior this PR is trying to remediate: rich cron system-event text is embedded into main-session heartbeat prompts, main-session systemEvent payloads are accepted without text-shape validation, and docs/tool guidance still encourage reminder prose on the main-session path. The PR remains a plausible implementation candidate, but its open no-relay and legacy-job editability review comments should be resolved before landing.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase or adopt it onto current main, replace main-session cron reminder prose with short internal wake tokens, constrain or safely degrade rich main-session systemEvent text, preserve isolated agentTurn reminder flows, update the agent/tool/docs guidance, and add focused regression coverage for tokenized main-session cron prompts plus legacy job editability.
