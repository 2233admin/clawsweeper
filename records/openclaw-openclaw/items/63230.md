## Summary

Keep this PR open. Current main still only implements Slack `thread.requireExplicitMention` at the account/global Slack config level; the per-channel override proposed here is not present in the strict schema, exported config type, channel resolver, mention gate, docs, or tests. The PR is a focused implementation candidate, but it needs contract cleanup before merge.

## Best Possible Solution

Keep this PR open and request a focused revision: add a narrow channel-level thread config type/schema that only supports `requireExplicitMention`, resolve exact channel then wildcard then account default in `resolveSlackChannelConfig`, use that resolved value in Slack mention gating, add resolver and message-preparation tests for both override directions plus wildcard fallback, and update Slack docs, generated config metadata, exported config types, and changelog for the new public config surface.
