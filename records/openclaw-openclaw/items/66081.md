## Summary

Keep open. Current main still finalizes direct codex-cli runs from stdout, so the PR is addressing a real gap, but the proposed patch is not merge-ready because it broadens generic JSONL parsing outside codex-cli, adds a new pre-cleanup artifact allocation failure path, and logs the generated artifact path in verbose argv output.

## What This Changes

The branch adds Codex output-last-message temp artifact creation, readback, cleanup, artifact-over-stdout final text selection, broader JSONL text extraction, and focused parser/runner tests.

## Best Possible Solution

Land a narrow codex-cli artifact finalization path that keeps stdout-derived session and usage telemetry, prefers a non-empty last-message artifact only for final reply text, scopes Codex-specific JSONL content parsing, keeps all temp resources under cleanup coverage, masks generated paths in logs, and preserves focused artifact precedence and fallback tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Scope the JSONL fallback to Codex:** `src/agents/cli-output.ts:486-493`
  - body: The new fallback extracts message/content/result/response for every JSONL backend. OpenClaw supports custom JSONL CLI backends, so status or tool-result records from non-Codex providers can now be folded into the assistant reply. Guard this block with providerId === "codex-cli" or an explicit JSONL dialect.
  - confidence: 0.9
- **[P2] Create the artifact inside cleanup coverage:** `src/agents/cli-runner/execute.ts:270-272`
  - body: prepareCodexLastMessageArtifact is awaited after system prompt, image, and skills-plugin resources are created but before the outer finally begins. If mkdtemp fails, those earlier resources bypass cleanup. Move this allocation under the existing guarded cleanup path, or start the try/finally before the resource allocations.
  - confidence: 0.82
- **[P3] Mask the artifact path in argv logs:** `src/agents/cli-runner/execute.ts:275-279`
  - body: The new --output-last-message argument is included in verbose cli argv logs, but buildCliLogArgs does not mask that flag's value. This exposes the generated temp directory and run artifact filename; mask or abbreviate it like the existing prompt and image path arguments.
  - confidence: 0.84
