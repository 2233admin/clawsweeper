## Summary

Keep open. Current main still lacks the requested Codex final-message artifact path, and this PR is a useful implementation candidate, but the branch is not merge-ready because it broadens JSONL reply parsing beyond `codex-cli` and leaves command-argument/logging concerns in a command execution path.

## What This Changes

The PR adds Codex `--output-last-message` temp artifact creation, readback, cleanup, artifact-over-stdout final text selection, and focused parser/runner tests for the `codex-cli` path.

## Best Possible Solution

Land a narrowly scoped `codex-cli` artifact finalization path that inserts the flag before positional prompt handling, prefers a non-empty artifact only for final reply text, preserves stdout-derived session/usage telemetry and fallback behavior, scopes any Codex JSONL content fallback to Codex or an explicit dialect, masks generated artifact paths in logs, and keeps focused tests for artifact precedence and empty-artifact fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Scope the JSONL fallback to Codex:** `src/agents/cli-output.ts:503-511`
  - body: The new fallback extracts `message`, `content`, `result`, and `response` from every JSONL backend. Since OpenClaw supports custom JSONL CLI backends, status or tool-result records from non-Codex providers can now be folded into the final assistant reply, contrary to the PR's stated scope. Guard this block with `providerId === "codex-cli"` or an explicit dialect.
  - confidence: 0.88
- **[P2] Insert the artifact flag before the prompt:** `src/agents/cli-runner/execute.ts:275`
  - body: This appends `--output-last-message` after `buildCliArgs()` has already appended the positional prompt for `input: "arg"`. Codex documents options before `[PROMPT]`, so placing the new option after the prompt is fragile and may be consumed as prompt text. Add the flag to the base args before prompt placement, or splice it before the final prompt arg.
  - confidence: 0.76
- **[P3] Mask the artifact path in CLI argv logs:** `src/agents/cli-runner/execute.ts:275`
  - body: Adding the artifact argument means verbose `cli argv` logs can expose the generated temp path because `buildCliLogArgs` has no masking case for `--output-last-message`. Mask or abbreviate that value like the existing system prompt and image path arguments.
  - confidence: 0.81
