## Summary

Keep open. Current main still has a source-backed path where an isolated cron run with only error payloads plus final assistant text is classified as fatal and announced as the raw tool error, and there is an open closing PR (#74840) that should be corrected or replaced before this issue closes.

## What This Changes

The issue requests that isolated cron `agentTurn` summaries prefer the final self-corrected assistant reply over an intermediate tool error when the run actually recovered.

## Best Possible Solution

Update or replace #74840 so cron classifies proven recovered mutating-tool failures separately from unresolved fatal failures, then cover the helper, payload builder, and message-tool policy boundary with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is `resolveCronPayloadOutcome` with only `isError: true` payloads, `finalAssistantVisibleText`, and `preferFinalAssistantVisibleText: true`; current tests assert that this returns the raw last error.

## Solution Assessment

No for the current open PR shape. The requested direction is right, but a blanket fallback preference is not enough because fatal structured-error precedence still wins; the safer fix is a narrow recovered-versus-unresolved classification before fatal delivery is chosen.
