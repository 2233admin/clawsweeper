## Summary

Keep open. Current main still has the reported classifier behavior: recovered-looking final assistant text is ignored when the payload set is all errors, and the issue is paired with open unmerged PR #74840, so it should remain open until that PR or a narrow equivalent lands.

## What This Changes

The issue asks isolated cron `agentTurn` summaries to prefer the final self-corrected assistant reply over intermediate tool-error payloads when announcing the run outcome.

## Best Possible Solution

Adjust the cron outcome classifier so recovered text-only final assistant output can win over stale intermediate tool-error payloads, while preserving fatal handling for real run failures, denial signals, structured deliveries, and unresolved mutating failures.

## Reproduction Assessment

Yes. The helper test at `src/cron/isolated-agent.helpers.test.ts:265` reproduces the core classifier behavior with all-error payloads plus `finalAssistantVisibleText` and expects the last error to be selected.

## Solution Assessment

Yes, a targeted classifier change is the right direction; a blanket rule that always trusts the final assistant message would be too broad because real fatal tool and run errors must still surface.
