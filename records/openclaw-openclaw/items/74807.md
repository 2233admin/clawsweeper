## Summary

Keep open. Current main still lets fatal structured error payloads override `finalAssistantVisibleText` in the isolated cron outcome classifier, and the issue is already paired with open unmerged PR #74840 using closing syntax, so it should stay open until that PR or a narrow equivalent lands.

## What This Changes

Cron `agentTurn` summaries should prefer a recovered final assistant reply over an intermediate tool-error payload when building the final announcement.

## Best Possible Solution

Review and land PR #74840 or a narrow equivalent that lets recovered assistant output drive cron summary/output selection while preserving fatal reporting for run-level errors, denial signals, and genuinely unresolved mutating-tool failures.

## Reproduction Assessment

Yes. The helper boundary reproduces the classifier choosing an error payload over `finalAssistantVisibleText`, and the payload tests show how a success-looking assistant reply can be followed by a mutating tool warning.

## Solution Assessment

No, not as a blanket rule. The maintainable fix should prefer final assistant text for recovered or presentation-only errors, while keeping structured fatal errors and unresolved mutating-tool failures fatal.
