## Summary

Keep open. Current main still lets fatal structured error payloads override `finalAssistantVisibleText`, and the issue is already paired with open unmerged PR #74840 as the implementation candidate.

## What This Changes

The issue asks for isolated cron `agentTurn` announcements to use the agent's final self-corrected assistant reply instead of an intermediate failed tool result.

## Best Possible Solution

Review and land PR #74840 or a narrow equivalent that lets recovered final assistant output drive cron summaries while preserving fatal reporting for real run-level failures and unresolved unsafe tool failures.

## Reproduction Assessment

Yes. A direct helper case with only error payloads plus `finalAssistantVisibleText: "Recovered final answer"` reproduces the classifier choosing the last error, and payload tests show how a mutating tool warning can trail successful assistant text.

## Solution Assessment

No, not as a blanket “always trust the last assistant reply” rule. The maintainable fix should prefer final assistant text only for recovered or presentation-only error cases while keeping structured fatal failures authoritative.
