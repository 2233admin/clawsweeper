## Summary

Keep open. Current main has some final-assistant preference for cron announce delivery, but the reported failure mode is still plausible because unresolved mutating tool errors are converted into trailing error payloads and `resolveCronPayloadOutcome` treats those trailing non-message errors as fatal ahead of the final assistant text.

## What This Changes

The issue asks for isolated cron `agentTurn` summaries to prefer the final self-corrected assistant reply over an intermediate tool error when the agent ultimately succeeds.

## Best Possible Solution

Teach the cron/embedded-runner boundary to distinguish an actually recovered mutating tool failure from a still-unresolved one, then announce the final assistant-visible result only for the recovered case while keeping fatal reporting for real trailing errors.
