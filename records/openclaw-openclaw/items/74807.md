## Summary

Keep open. Current main still has a plausible path where a trailing error payload overrides final assistant-visible text for isolated cron announce output, and this issue is already paired with open PR #74840 using closing syntax, so it should remain open until that PR is reviewed and merged or closed.

## What This Changes

Cron isolated `agentTurn` summaries should report the agent's final self-corrected assistant reply instead of an intermediate tool-error payload after recovery.

## Best Possible Solution

Review and land or revise #74840 so cron distinguishes recovered mutating-tool failures from unresolved trailing failures, using final assistant-visible text for recovered textual announce delivery while keeping existing fatal-error semantics for real failures.

## Reproduction Assessment

Yes. A focused source-level reproduction is available by passing a successful assistant text followed by an `isError` tool payload into `resolveCronPayloadOutcome` with final-assistant preference enabled; current code routes that through the fatal trailing-error branch.

## Solution Assessment

Unclear for the open PR until its exact diff is reviewed, but the broad rule in the issue is not sufficient by itself. The maintainable fix should be a narrow recovered-failure classification rather than blindly trusting every final assistant success claim.
