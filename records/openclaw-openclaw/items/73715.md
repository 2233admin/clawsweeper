Current main still preserves explicit mixed-case `infer model run --model` values on local and gateway paths, and two open, unmerged PRs already reference this issue with closing syntax, so the issue should remain open until maintainers review and land or close those implementation candidates.

## What This Changes

The issue requests that `openclaw infer model run --model <provider>/<model>` reject or canonicalize case-mismatched model ids instead of forwarding them to the provider and surfacing a misleading empty-output error.

## Best Possible Solution

Review and consolidate the linked implementation PRs, then land a catalog-aware fix that canonicalizes unique known model matches before both local and gateway dispatch while preserving or clearly rejecting unmatched custom refs, with regression coverage in `src/cli/capability-cli.test.ts`.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current source inspection reproduces the mechanism without a live provider: local dispatch forwards raw `params.model`, gateway dispatch preserves sliced casing, and Anthropic runtime resolution can keep the mixed-case model identity.

## Solution Assessment

Unclear because no fix has merged yet and the two open PRs differ. A unique catalog-match canonicalization path is the safer maintainable solution; broad lowercasing is riskier for custom mixed-case ids.
