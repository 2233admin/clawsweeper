## Summary

Keep open. Current main still preserves mixed-case model ids on the `infer model run` local and gateway paths, and the GitHub context shows two open closing PRs already targeting this issue, so the issue should remain open until maintainers review, consolidate, land, or close those candidates.

## What This Changes

The issue requests that `openclaw infer model run --model <provider>/<model>` either canonicalize case-only catalog model matches before dispatch or reject case-mismatched model ids with a clear unknown-model error.

## Best Possible Solution

Land a catalog-aware fix that canonicalizes a unique known model match before both local and gateway dispatch, or otherwise rejects case-only mismatches before provider calls with a clear unknown-model error, with focused regression coverage in `src/cli/capability-cli.test.ts` and preservation of unmatched custom refs.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current source inspection reproduces the mechanism without a live provider: the local path forwards raw `params.model`, the gateway parser preserves model casing, and the Anthropic resolver can keep the mixed-case model identity.

## Solution Assessment

Unclear until a PR is chosen. A unique catalog-match canonicalization path is the safer maintainable fix; broad lowercasing is riskier for custom or provider-specific mixed-case model ids.
