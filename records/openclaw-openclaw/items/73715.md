## Summary

Keep open. Current main still preserves user-supplied model casing on both the local and gateway `infer model run` paths, and two open unmerged PRs already target this issue, so maintainers should review or consolidate those candidates rather than close it now.

## What This Changes

`infer model run --model <provider>/<model>` should either canonicalize case-only catalog model matches before dispatch or reject case-mismatched model ids with a clear unknown-model error.

## Best Possible Solution

Review and consolidate #73717 and #74945, then land a narrow fix that canonicalizes unique known catalog matches before both local and gateway dispatch, while preserving or clearly rejecting unmatched custom refs and adding regression coverage in `src/cli/capability-cli.test.ts`.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current source inspection confirms the same path without live provider access: local dispatch forwards raw `params.model`, gateway parsing preserves the sliced model casing, and the Anthropic runtime can retain the mixed-case model id after matching.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. A catalog-aware unique-match canonicalization is safer than unconditional lowercasing because it fixes known OpenClaw catalog refs without changing arbitrary custom mixed-case ids.
