## Summary

Keep open. Current main at f6d23ab still lacks the provider-scoped external CLI overlay path proposed by this PR: runtime auth overlays still evaluate every built-in external CLI reader without provider eligibility, `models.authStatus` still loads the overlaid auth store before computing configured refreshable providers, and macOS Claude CLI Keychain reads remain prompt-eligible unless explicitly disabled. The PR is also not superseded by #74033 because that related open PR only adds a narrower profile-exists guard.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this provider-scoped overlay fix, or an equivalent canonical fix, after reconciling it with #74033 and adding/keeping regression coverage for api-key-only `models.authStatus`, scoped external CLI readers, forwarded `allowKeychainPrompt`, and intended Claude CLI configured-provider behavior. The fix should stay limited to external CLI overlay eligibility and prompt policy, without changing supported credential formats, persistence rules, or broader provider discovery.
