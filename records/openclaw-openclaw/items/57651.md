## Summary

Keep open. Current main still uses an in-memory ACP bridge session store, so the PR's central bridge-session persistence request is not implemented on main. The submitted branch is not mergeable as-is because the PR body is an untouched template, the diff bundles unrelated enhanced-compaction and gateway auto-abort work, and the existing review context reports P1 defects in the proposed persistence implementation. Those are reasons to require a focused rework, not enough for one of the allowed auto-close reasons because the underlying ACP bridge persistence gap appears to remain real.

## Best Possible Solution

Ask for or prepare a much smaller ACP bridge PR that only persists the ACP client sessionId to Gateway sessionKey mapping, validates deserialized JSON, preserves current loadSession/listSessions semantics, and adds restart-focused tests. Leave the unrelated enhanced compaction and gateway auto-abort changes out of that PR; if maintainers decide bridge-side persistence is not desired, close this manually with that product decision rather than via automated cleanup.
