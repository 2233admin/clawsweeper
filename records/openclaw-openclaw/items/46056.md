## Summary

Keep this issue open. Current main still requires every chained shell segment to satisfy allowlist evaluation, and there is no current shell-builtin satisfaction path for `cd`/`export`/similar builtins. The related closed #40478 covers durable `allow-always` exact-command reuse after a prompt, but it does not solve preconfigured allowlist mode for builtins in a fresh command chain.

## Best Possible Solution

Keep this issue open for maintainer direction. The minimum safe path is to document explicitly that POSIX shell builtins are not PATH-resolved allowlist targets and to recommend forms like `git -C <path>` instead of `cd <path> && git ...`. If maintainers want behavior changes, design an opt-in builtin policy with tests for cwd/env mutation effects rather than silently auto-allowing builtins in the existing allowlist evaluator.
