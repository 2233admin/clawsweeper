## Summary

Keep open. Current main and the latest release still let `sessions cleanup --fix-missing` prune from a single resolved transcript path, while the linked implementation PR remains open and unmerged; producer-side rollover fixes do not protect already-stale store entries from cleanup.

## What This Changes

The issue asks for `sessions cleanup --fix-missing` to preserve valid sessions when stored `sessionFile` metadata is stale but the canonical `<sessionId>.jsonl` transcript exists.

## Best Possible Solution

Review and land PR #69954 if its behavior is acceptable, or replace it with a focused cleanup fix that checks the canonical `<sessionId>.jsonl` candidate before pruning, repairs stale generated `sessionFile` metadata when the canonical transcript exists, preserves valid custom transcript paths and true-missing pruning, and keeps broader doctor/orphan archive semantics separate from this issue.
