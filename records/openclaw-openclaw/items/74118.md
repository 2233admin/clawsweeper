## Summary

Keep open. The PR targets a real hot-path cache miss, but the proposed guard is broader than the current loader contract and can skip required plugin loads once any active registry exists, including registries from a different workspace/config or placeholder/scoped registries.

## What This Changes

The PR adds a fast path in `src/agents/runtime-plugins.ts` that returns when `getActivePluginRegistry()` is truthy and updates `src/agents/runtime-plugins.test.ts` to expect no registry resolution in that case.

## Best Possible Solution

Keep the performance fix, but make the fast path prove that the active registry satisfies the requested load options. A maintainable direction is to teach the loader a subset-compatible gateway-startup match, or add a loaded/compatible active-registry predicate, while preserving misses for different workspaces, configs, plugin scopes, and placeholder registries.

## Reproduction Assessment

Yes. A focused unit test can set an active registry for one workspace or an empty placeholder, call `ensureRuntimePluginsLoaded` with a different configured workspace, and assert that the resolver still runs; the PR would currently return early.

## Solution Assessment

No. The helper-level short-circuit is the right performance boundary, but this implementation is too broad because it bypasses the existing compatibility checks instead of narrowing them to the reported gateway-startup case.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Prove the active registry matches the requested load:** `src/agents/runtime-plugins.ts:31-33`
  - body: Returning for any active registry ignores the `config`, `workspaceDir`, and runtime options computed below. Those inputs are part of the loader compatibility key, and plugin discovery includes `<workspace>/.openclaw/extensions`; in a long-lived process, an active registry from another workspace, a scoped load, or an empty placeholder would now prevent the requested plugins from loading. Please make the fast path verify that the active registry satisfies these load options before returning.
