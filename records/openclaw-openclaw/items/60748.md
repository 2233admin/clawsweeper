## Summary

Keep open. Current main only has a narrow mitigation that omits the generic `main` Feishu card header when no identity name is configured. The broader request remains unimplemented: no configurable card header modes, no reply-to-context header, no elapsed/status footer rendering, no footer hiding, and no per-field footer toggles.

## Best Possible Solution

Keep this issue open as the canonical Feishu card customization tracker. The best path is a Feishu plugin change that adds backward-compatible top-level and per-account config for header mode (`hidden`, `custom`, `reply-to`) and footer fields (`enabled`, `elapsed`, `status`, `agent`, `model`, `provider`), wires it through both streaming and static card reply paths, updates the plugin manifest/generated config metadata and `docs/channels/feishu.md`, and adds focused schema/dispatcher tests for the new modes.
