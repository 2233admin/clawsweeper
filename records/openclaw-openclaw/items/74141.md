## Summary

Keep open. Current main still has the reported `session_status` resolution gap, and there are open linked implementation PRs, so the issue should stay open until maintainers review, merge, or replace those fixes.

## What This Changes

The issue requests that `session_status` resolve `sessionKey="current"` to the requester session for channel-plugin-driven agents instead of throwing `Unknown sessionKey: current`.

## Best Possible Solution

Review and consolidate #74142 and #72306, land one narrow fix that adds a channel-style requester regression test for `sessionKey="current"`, preserves literal-current precedence, and keeps visibility self-target handling intact.

## Reproduction Assessment

Yes. The issue provides concrete gateway logs, and current main can be traced deterministically from `sessionKey="current"` through the skipped non-sessionId resolution to the `Unknown sessionKey: current` throw when no store entry matches.

## Solution Assessment

Yes. A final `session_status` fallback after the literal-current key/sessionId paths is the narrowest maintainable direction, provided the selected PR keeps existing strict unknown-key and visibility behavior unchanged.
