## Summary

Keep open. Current main still has the documented contract mismatch for channel-plugin requesters, and two open, unmerged PRs already target the same bug, so the issue should stay open until one canonical fix lands.

## What This Changes

The issue asks `session_status` to resolve explicit `sessionKey="current"` to the requester session for channel-plugin agents instead of throwing and causing retry latency.

## Best Possible Solution

Review and land one canonical fix PR, preserving literal `current` key/session-id precedence while adding a final requester-session fallback for explicit `sessionKey="current"` plus a focused channel-style requester regression test.

## Reproduction Assessment

Yes. The issue provides a concrete channel-style requester key and logs, and current main still has a static path where explicit `sessionKey="current"` can miss store/gateway lookup and throw `Unknown sessionKey`.

## Solution Assessment

Yes. A narrow fallback in `session-status-tool.ts` after existing literal-`current` resolution is the best maintainable direction; the open PRs should be reviewed rather than starting another fix branch.
