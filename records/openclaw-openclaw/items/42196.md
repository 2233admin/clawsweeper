## Summary

Keep open. Current main still allows `openclaw cron add` without `--agent`, passes `agentId` as `undefined`, and the Gateway resolves missing cron agents to the default agent. Open PR #42245 remains the implementation candidate with closing syntax for this issue, so closing before that work merges would be premature.

## Best Possible Solution

Review and refresh PR #42245, or apply an equivalent focused change in `registerCronAddCommand`: warn for agent-turn `cron add` when `--agent` is omitted, keep system-event jobs quiet, avoid contaminating JSON stdout, and add regression coverage for missing `--agent`, explicit `--agent`, and `--session-key` without `--agent`. Close this issue only after that implementation merges or maintainers intentionally choose a different shipped fix.
