## Summary

Keep this issue open. Current main does not yet require `--agent` or warn when an agent-turn cron job is added without it, and there is still an open implementation PR with closing syntax for this issue.

## Best Possible Solution

Review and refresh the open cron warning PR, or apply an equivalent fix directly in `registerCronAddCommand`: warn for agent-turn `cron add` jobs when `--agent` is omitted, keep system-event jobs quiet, avoid breaking JSON stdout, and add regression coverage for missing `--agent`, explicit `--agent`, and `--session-key` without `--agent`. Close this issue only after that implementation is merged or the PR is intentionally closed with another shipped fix.
