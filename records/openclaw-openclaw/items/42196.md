## Summary

Keep open. Current main still allows agent-turn `openclaw cron add` without `--agent`, forwards `agentId` as absent/undefined, and the Gateway resolves missing cron agents to the configured default agent. The provided GitHub context also shows open PR #42245 with closing syntax for this issue, so the best maintainer path is to review/refresh that implementation candidate rather than close the issue before merge.

## Best Possible Solution

Review and refresh PR #42245, or apply an equivalent focused change in `registerCronAddCommand`: warn for agent-turn `cron add` when `--agent` is omitted, keep system-event jobs quiet, avoid contaminating JSON stdout, and add regression coverage for omitted `--agent`, explicit `--agent`, and `--session-key` without `--agent`. Close this issue only after that implementation merges or maintainers explicitly choose a different shipped behavior.
