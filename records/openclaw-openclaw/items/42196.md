## Summary

Keep open. Current main still allows agent-turn `openclaw cron add` jobs without `--agent`, passes `agentId: undefined` through the add RPC, stores the missing agent id, and resolves execution to the configured default agent. The provided discussion also shows open PR #42245 as the implementation candidate, so this issue should remain open until that PR or an equivalent fix merges.

## Best Possible Solution

Review and land PR #42245, or land an equivalent focused change in `src/cli/cron-cli/register.cron-add.ts` that warns on stderr when an agent-turn `cron add` omits `--agent`, preserves explicit `--agent` and `--session-key` behavior, keeps system-event jobs quiet, and adds regression coverage for omitted and explicit agent cases. Close this issue only after that implementation merges or maintainers choose a different shipped behavior.
