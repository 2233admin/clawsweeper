## Summary

Keep open. Current main still has the runtime-deps predicate gap for manifest-less bundled library packages, and `media-understanding-core` is exactly such a package: it has `stageRuntimeDependencies: true`, depends on `sharp`, and has no `openclaw.plugin.json`. The PR is relevant, and Greptile's comment correctly identifies the patch as narrow, but the current one-file patch may not cover the `doctor`/repair path because that path first filters candidates to manifest-backed plugin ids and scoped scans bypass the predicate this PR changes.

## Best Possible Solution

Finish this PR rather than close it: make manifest-less bundled packages that set `openclaw.bundle.stageRuntimeDependencies: true` participate in the intended runtime-deps scan/repair path while preserving global plugin disable, denylist, allowlist, and explicit disabled-entry gates. Add focused regression coverage for the package-only library case and then run the targeted plugin/doctor tests before maintainer review and merge.
