## Repair Work Prompt

Finish the runtime-deps fix for this PR. Current main and the PR patch cover `isBundledPluginConfiguredForRuntimeDeps`, but `openclaw doctor --fix` still builds `effectivePluginIds` from manifest-backed plugins only and can miss manifest-less library packages such as `media-understanding-core`.

Implement a narrow fix so bundled extension directories without `openclaw.plugin.json` but with `package.json#openclaw.bundle.stageRuntimeDependencies === true` participate in both `scanBundledPluginRuntimeDeps` and `maybeRepairBundledPluginRuntimeDeps` repair selection. Use the directory name as the plugin id for package-only libraries. Preserve existing behavior for manifest-backed channel/provider/default-enabled plugins, global `plugins.enabled: false`, denylist, restrictive allowlist, explicit `plugins.entries.<id>.enabled === false`, dependency name/version normalization, external stage dirs, and `npm install --ignore-scripts`. Do not change workflows, lockfiles, package versions, npm runner args, or public plugin APIs.

Add regression tests for: unscoped/configured scan includes a manifest-less staged package; denylist and explicit disabled entries suppress it; restrictive allowlist behavior is intentional and covered; doctor non-interactive repair installs `sharp@^0.34.5` for a manifest-less staged package; doctor skips it when denied or explicitly disabled. Add a user-facing changelog fix entry if this branch is expected to carry release notes, credited to `@kybrcore` per repo policy.
