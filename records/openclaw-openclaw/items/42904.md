## Summary

Keep this PR open. Current main still does not implement the PR's channel-specific active log fallback: `openclaw channels logs` reads the configured log file directly, while the safer rolling-log resolver exists only on the generic `openclaw logs` / `logs.tail` path. Main has fixed writer-side daily rollover, but that is not the same as this command's missing-file fallback. The PR also still needs cleanup before merge because it bundles unrelated cron warning work and its fallback design can read the wrong local log file for custom/non-rolling `logging.file` setups.

## Best Possible Solution

Keep this PR open for maintainer rework: split the cron warning commit back to #42245, then make `openclaw channels logs` reuse `readConfiguredLogTail` or a shared exact-pattern rolling-log resolver before applying channel filtering. Add focused tests for missing current-day files, existing current-day precedence, custom/non-rolling `logging.file`, and channel filtering against the resolved fallback file.
