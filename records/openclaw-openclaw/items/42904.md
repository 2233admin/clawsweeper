## Summary

Keep this PR open. Current main still reads `openclaw channels logs` from the exact configured file and returns no lines if that file is missing, while the shared rolling-log resolver is only used by the generic `logs.tail` path. The latest PR patch is narrow, reuses that existing resolver for channel logs, adds focused regression tests, and does not touch security-sensitive supply-chain or CI surfaces.

## Best Possible Solution

Keep this PR open for maintainer review and landing. The best path is to merge the narrow current branch direction: export and reuse the existing rolling-log resolver in `channelsLogsCommand`, keep fallback limited to default dated rolling log files, preserve custom non-rolling `logging.file` behavior, and land the focused regression tests plus changelog credit.
