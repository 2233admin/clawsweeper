## Summary

Keep this PR open. Current main has generic `logs.tail` fallback and writer-side daily rollover, but `openclaw channels logs` still reads the exact configured logger file and returns no lines when that file is missing. The latest PR context shows a narrow repair that reuses the shared rolling-log resolver and adds focused channel-log regression tests, with recent maintainer follow-up on the branch.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the narrow current branch if CI stays green: export/reuse the shared rolling-log resolver in `channelsLogsCommand`, keep the focused regression tests, and preserve the contributor credits in the changelog. If maintainers decide writer-side rollover is sufficient and command-level fallback parity is not wanted, that should be an explicit maintainer product decision rather than an automated cleanup close.
