## Summary

Keep this PR open. Current main has the rolling-log fallback for the generic `openclaw logs` / `logs.tail` path and writer-side daily rollover, but `openclaw channels logs` still reads the configured log file directly. The PR also bundles unrelated cron warning work that is separately tracked by the same author in open #42245, and its fallback design has a remaining correctness/privacy risk for custom logging paths.

## Best Possible Solution

Keep the PR open for maintainer rework: split out the cron warning in favor of #42245, then replace the logging change with a focused `openclaw channels logs` implementation that reuses the shared exact rolling-log resolver and adds targeted regression coverage for missing current-day files, existing current-day precedence, custom `logging.file`, and channel filtering across the fallback file.

## Risks / Open Questions

- Merging as-is would mix a channel-log bug fix with unrelated cron warning work already represented by open #42245, making review and rollback harder.
