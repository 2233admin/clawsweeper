## Summary

Keep this PR open. Current main has the Telegram delivery normalization half, but it still does not implement the parser-order fix this PR added for same-line reply tags before `MEDIA:` directives. The remaining useful work is small but real.

## Best Possible Solution

Refresh the PR onto the current `extensions/telegram` layout, keep the already-shipped delivery-plan normalization, and add a narrow parser fix that strips reply tags before `MEDIA:` extraction only when needed. Add a regression test for `[[reply_to:<id>]] MEDIA:...` and a Telegram delivery test proving media delivery with the reply target. No supply-chain or secret-handling concern was visible in the provided diff; the remaining review focus is functional correctness and whitespace preservation.
