## Summary

Keep this open. Current main still allows WhatsApp visible auto-replies to be delivered through the inbound message's bound reply/sendMedia closures, and the existing owner pinning only protects main-session lastRoute metadata; it does not block or reroute implicit replies from allowlisted non-owner DMs.

## What This Changes

The issue requests an opt-in WhatsApp routing safeguard that blocks or reroutes implicit plain-text replies when the last inbound DM sender is not the session owner.

## Best Possible Solution

Keep this open for maintainer product/security review. The likely fix belongs in the WhatsApp plugin visible auto-reply path before deliverWebReply sends payloads, with an opt-in typed root/account config policy, schema/help/docs updates, and regression tests for owner DMs, allowlisted non-owner DMs, and explicit message-tool sends that should remain intentional.
