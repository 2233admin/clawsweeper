## Summary

Keep open. Current main still creates a real Google Chat typing message before dispatch, only clears it from the delivery callback, and exact silent `NO_REPLY` finals can be skipped before any delivery. The directly related fix PR #39891 remains open, so the issue is not implemented on main yet.

## Best Possible Solution

Keep #39843 open until PR #39891 or an equivalent Google Chat monitor fix lands. The fix should delete any remaining `typingMessageName` after `dispatchReplyWithBufferedBlockDispatcher` completes without delivery, or wire `onTypingCleanup`, and add a focused regression test for the Google Chat emoji reaction plus exact `NO_REPLY` case.
