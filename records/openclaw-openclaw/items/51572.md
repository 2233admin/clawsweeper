## Summary

Current main still does not implement the requested reset/prune memory preservation behavior. The bundled session-memory hook remains registered and filtered to command:new/command:reset, stale daily/idle rollover only emits typed session_end/session_start plugin hooks, and store maintenance deletes pruneAfter entries without internal hook or before_reset dispatch. Open PRs #61675 and #71792 are relevant daily/idle implementation candidates, but the prune path remains unaddressed, so this issue should stay open or be explicitly split.

## Best Possible Solution

Keep this issue open, or split it only if maintainers create a canonical prune-specific tracker. The daily/idle half should be resolved by reviewing and landing #61675, #71792, or a replacement. The pruneAfter half needs a small core/session-store design: dispatch a documented lifecycle or session-memory-compatible hook before stale entries are deleted, pass the removed SessionEntry plus stable transcript path, and add regression coverage for maintenance pruning.
