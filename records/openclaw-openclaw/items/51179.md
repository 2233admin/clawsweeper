## Summary

Keep this PR open. Current main still silently truncates `allowFrom` to two entries in the live `channels status --probe` formatter and in the channel summary helper, and the existing summary test still asserts the truncated output. The PR context is useful because later commits address both the live formatter gap and the status-summary privacy concern raised in review; #51180 overlaps but is a competing policy choice, not a clear supersedence.

## Best Possible Solution

Keep this PR open for maintainer review or consolidate it deliberately with #51180. The best path is to remove silent two-entry truncation from the live `formatGatewayChannelsStatusLines()` path and the intended summary helper, add regression coverage for three-or-more entries, and explicitly preserve the non-admin/redacted status-summary boundary by tying full allowlist detail to `includeSensitive` or another documented policy.
