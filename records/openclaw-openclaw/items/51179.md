## Summary

Keep open. Current main still truncates `allowFrom` entries in both the live `openclaw channels status --probe` formatter and the channel-summary helper, so this PR is not obsolete or already implemented. The linked bug #51174 remains a real diagnostic issue, and the PR discussion correctly called out the privacy-sensitive status-summary path that should stay part of the final fix.

## Best Possible Solution

Keep this PR open for maintainer review or consolidate it with #51180. The best code path is a small shared formatting fix that removes hard truncation from both `channels status --probe` and the channel-summary fallback, adds regression coverage for three-or-more entries, and gates full allowlist display behind sensitive/admin status summaries.
