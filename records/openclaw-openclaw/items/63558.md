## Summary

Keep this issue open. Current main still has no Dreaming-tab agent selector, and the Dreaming UI/controller/gateway path remains scoped to the default agent rather than a selected agent context; related #63829 and #68959 overlap but do not supersede this narrower Dreaming inspection/action gap.

## What This Changes

Add a Control UI Dreaming-tab agent selector so users can inspect and act on Dreaming status, diary, and memory artifacts for non-default agents.

## Best Possible Solution

Add selected-agent state to the Dreaming tab, render a selector backed by agents.list, pass the selected agentId through Dreaming status/diary/action RPCs, and update doctor.memory handlers to honor an optional validated agentId with default-agent fallback. Keep the broader memory-wiki vault isolation in #63829 and the general web UI agent switcher in #68959 separate.
