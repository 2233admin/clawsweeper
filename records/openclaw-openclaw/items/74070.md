## Summary

Keep open. Current main and the latest release still contain the reported ACP background-task summary behavior, and open PR #74103 already references this issue with closing syntax, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The report asks for ACP background-task completions to fall back from successful progress output into `terminalSummary` and to raise the progress-summary cap from 240 characters to roughly 1200.

## Best Possible Solution

Land PR #74103 after sanitization and regression coverage are verified, or replace it with the same narrow ACP manager fix: copy cleaned successful progress into `terminalSummary`, preserve blocked-outcome handling, raise the bounded progress cap, and cover both useful summaries and suppressed internal/control text.

## Reproduction Assessment

Yes. The static current-main path is high confidence: non-thought ACP output populates `taskProgressSummary`, ordinary non-blocked success returns `{}`, completion stores `terminalSummary: null`, and delivery formats only `terminalSummary` while the progress cap remains 240.

## Solution Assessment

Yes on the direction, with a testing caveat. Falling back from cleaned progress output and increasing the bounded cap is the narrow maintainable fix, but the final patch should explicitly preserve sanitizer/no-leak behavior and add focused regression coverage.
