## Summary

Keep open: current main and the latest release still contain the reported ACP background-task summary behavior. Open PR #74103 explicitly targets this issue with closing syntax, but it remains unmerged and the prior review identified a needed cleanup/test blocker, so this issue should stay open until that PR or an equivalent sanitized fix lands.

## What This Changes

Request to make ACP background-task completions preserve useful progress output in terminalSummary and raise the progressSummary cap from 240 to about 1200 characters.

## Best Possible Solution

Land PR #74103 after addressing the control-token cleanup and regression coverage, or replace it with an equivalent narrow ACP manager patch that copies cleaned non-blocked progress into terminalSummary, preserves blocked-outcome handling, and keeps terminal delivery sanitized.

## Reproduction Assessment

Yes. The static current-main path is high-confidence: non-thought ACP output populates taskProgressSummary, ordinary non-blocked success returns {}, terminal finalization stores null terminalSummary, and delivery formats only terminalSummary.

## Solution Assessment

Yes on the requested direction, but not as a blind copy. Falling back from cleaned non-blocked progress and raising a bounded cap is the narrow maintainable fix, while the final patch should strip silent-control tokens and add focused tests.
