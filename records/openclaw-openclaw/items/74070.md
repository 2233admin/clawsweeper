## Summary

Keep open: current main and the latest release still contain the reported ACP background-task summary behavior, and open PR #74103 already references this issue with closing syntax. The issue should remain open until that PR, or an equivalent sanitized and tested fix, lands.

## What This Changes

The issue requests ACP background-task completions to copy useful cleaned progress text into terminalSummary for successful non-blocked tasks and raise the progress summary cap from 240 to about 1200 characters.

## Best Possible Solution

Land PR #74103 after review, or replace it with an equivalent narrow ACP/task patch that raises the bounded progress cap, derives terminalSummary only from cleaned user-safe successful progress, preserves blocked outcome handling, and adds regression coverage for both useful successful summaries and suppressed internal/control text.

## Reproduction Assessment

Yes. The static current-main path is high-confidence: non-thought ACP output fills taskProgressSummary, ordinary non-blocked success returns {}, terminal finalization stores a null summary, and delivery formats only terminalSummary.

## Solution Assessment

Yes on the fix boundary, with a safety caveat. Falling back from cleaned progress output and increasing the cap is the maintainable direction, but the final patch should include sanitization and focused tests before this issue closes.
