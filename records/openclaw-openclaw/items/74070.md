Keep open: current main still has the reported ACP background-task behavior, and an open fix PR already references this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue asks for ACP background-task completions to reuse useful progress output as terminalSummary and to raise the progress summary cap from 240 characters.

## Best Possible Solution

Land a narrow ACP fix that derives a sanitized terminal summary for useful successful output, preserves blocked-outcome detection, raises the bounded progress cap, and adds regression coverage for both useful summaries and suppressed internal chatter.

## Reproduction Assessment

Yes. The source path is enough to reproduce: successful non-blocked progress reaches resolveBackgroundTaskTerminalResult, returns {}, is stored as null terminalSummary, and formatTaskTerminalMessage emits only the bare completion label.

## Solution Assessment

Yes in direction, but the safest implementation is a sanitized fallback plus tests rather than an unconditional raw pass-through; PR #74103 or an equivalent narrow patch should be reviewed against the existing no-leak behavior.
