## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: the gateway create/patch protocol, session config, persisted session entry, transcript creation/history paths, bootstrap-context handling, and Control UI session row do not carry a private/no-persist contract. The prior discussion correctly broadens this from a demo UI toggle into a program-level privacy guarantee, and that guarantee is still not implemented.

## What This Changes

The issue requests a Mission Control Private Mode that suppresses workspace context and prevents durable session history or memory/workspace reads and writes for private sessions.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito mode tracker and design an additive private/ephemeral session contract across Gateway protocol, SessionEntry/session store, JSONL transcript/history APIs, bootstrap and memory loading/writing, tool/media persistence, channel triggers, and Control UI indicators, with regression tests proving private sessions neither read private workspace/memory context nor write durable history.
