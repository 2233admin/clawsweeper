## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: there is no Gateway/session flag, no SessionEntry enforcement bit, session creation still ensures a JSONL transcript, normal chat/agent paths still append transcript messages, and the existing contextInjection knob is only a global bootstrap-injection control. Related #12242 is already closed as a duplicate of this request, while #52478 was closed unmerged and explicitly deferred runtime filesystem/session-history enforcement.

## Best Possible Solution

Keep this open as the canonical product request. A complete implementation should add an explicit private/ephemeral session mode through Gateway protocol and SessionEntry/session state, enforce no transcript/media/session-history writes, suppress workspace bootstrap and memory reads/writes, expose Mission Control and per-channel controls, and add regression tests proving private sessions neither read workspace/memory context nor write persistent history.
