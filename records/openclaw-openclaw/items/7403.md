## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: there is no protocol/session-store bit for private/no-persist behavior, session creation still creates JSONL transcripts, history still reads persisted transcripts, and the existing `agents.defaults.contextInjection: "never"` is only a global bootstrap-context mitigation. Provided context also shows #12242 was closed as a duplicate of this issue, while #52478 was closed unmerged after explicitly deferring runtime filesystem/session-history enforcement.

## Best Possible Solution

Keep this issue open as the canonical Private/Secret/Incognito mode tracker. A complete implementation should add an explicit private/ephemeral mode to the Gateway protocol and `SessionEntry`, enforce no transcript/media/session-history persistence, suppress workspace bootstrap and memory reads/writes, expose Mission Control and per-channel controls, and add regression tests proving private sessions neither read private workspace/memory context nor write persistent history.
