## Summary

Keep open. Current main still does not implement an enforceable per-session Private/Secret/Incognito mode. A global `agents.defaults.contextInjection: "never"` setting can suppress workspace bootstrap/context-file injection, but the requested product guarantee also needs session/protocol state, no transcript or media persistence, memory read/write suppression, and Mission Control/channel controls. Related #12242 is already closed as a duplicate of this issue, and #52478 was closed unmerged after explicitly deferring filesystem/session-history enforcement.

## Best Possible Solution

Keep this issue open as the canonical product request. A complete implementation should add an explicit private/ephemeral mode through Gateway protocol and `SessionEntry`, enforce no transcript/media/session-history writes, suppress workspace bootstrap and memory reads/writes, expose Mission Control and per-channel controls, and add regression tests proving private sessions neither read private workspace/memory context nor write persistent history.
