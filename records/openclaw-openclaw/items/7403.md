## Summary

Keep open. Current main still lacks an enforceable per-session private/ephemeral mode: the Gateway protocol, session config, persisted session entry, transcript creation, history APIs, bootstrap context, memory flush, and Control UI row model do not carry a private/no-persist bit. The prior discussion also makes this the canonical tracker for #12242, while the only related implementation PR #52478 was closed unmerged.

## What This Changes

Add a Mission Control private mode that suppresses workspace/persona/memory context and prevents durable session history or workspace mutation for demo, content creation, and shared-channel use cases.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito mode tracker and design an additive private-session contract across Gateway protocol, SessionEntry/session store, transcripts/history APIs, bootstrap and memory loading/writing, tool/media persistence, channel triggers, and Control UI indicators, with regression tests proving private sessions do not read protected context or write durable history.
