## Summary

Keep open. Current main supports Telegram voice-note download, batch preflight transcription, and voice-note TTS delivery, but it does not implement the requested Telegram voice optimization program: no voice-specific priority lane, no streaming STT/TTS pipeline, no transcription-result or TTS-preset cache, no dedicated voice/text concurrency policy, and no stage-boundary timing snapshots. Related PR #39075 is adjacent performance work, but it is open, unmerged, and narrower than this issue.

## Best Possible Solution

Keep this open as the canonical Telegram voice pipeline performance tracker. The best next step is to add stage timing/state snapshots first, benchmark current batch STT and final-buffer TTS latency, then design a voice-aware priority/concurrency policy that preserves Telegram chat/topic ordering. Streaming and caching should be added through shared media-understanding and speech-provider contracts where providers support them, with #39075 treated as adjacent prior work rather than a shipped fix.
