## Summary

Keep open. Current main still does not implement loader-time SIE envelope/signature verification, warn/strict unsigned handling, trust-root configuration, or docs/tests for that behavior. The related implementation PR is closed unmerged, and the current ClawHub SHA-256 fingerprint helper is an install/content hash primitive rather than issuer/signature verification for loaded skill instructions.

## What This Changes

The issue requests an optional, feature-flagged Signed Instruction Envelope verification gate in the skill loader with warn/strict handling for unsigned skills and rejection of invalid signatures.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, with deterministic reason codes, config/schema/docs support, documented trust-root and revocation policy, warn/strict unsigned behavior, and invalid-signature fail-closed behavior that composes clearly with ClawHub install/update flows.
