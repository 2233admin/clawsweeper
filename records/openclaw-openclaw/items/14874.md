## Summary

Keep open. Current main still lacks a skill-loader SIE envelope/signature verification path, the related implementation PR is closed unmerged, and the adjacent SHA-256 integrity proposal is not equivalent to issuer/signature verification.

## What This Changes

The issue requests an optional, feature-flagged SIE verification gate for skills with warn/strict handling for unsigned skills and rejection for invalid signatures.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, with deterministic reason codes, config/schema/docs support, documented trust-root and revocation policy, warn/strict unsigned behavior, and invalid-signature fail-closed behavior that composes clearly with ClawHub install/update flows.
