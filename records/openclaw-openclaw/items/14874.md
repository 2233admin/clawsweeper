## Summary

Keep open. Current main still has skill path, size, frontmatter, precedence, visibility, and ClawHub download-integrity plumbing, but no optional SIE envelope/signature verification contract in the TypeScript skill loader/config path. The linked SIE implementation PR #14893 is closed unmerged, and open PR #56256 is adjacent SHA-256 integrity work rather than the same signed-instruction verification design.

## What This Changes

Add optional feature-flagged SIE verification in the skill loader with warn/strict handling for unsigned skills and rejection for invalid signatures.

## Best Possible Solution

Keep this issue open for maintainer security design. The best path is a default-off TypeScript loader/config contract for SIE verification, deterministic machine-readable reason codes, warn/strict behavior for unsigned skills, reject behavior for invalid signatures, coverage across bundled/managed/workspace/extra/plugin skill roots, and docs explaining how SIE relates to ClawHub scans, archive hashes, lockfiles, install/update flows, trust roots, and revocation.
