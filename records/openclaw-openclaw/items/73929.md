## Summary

Keep this PR open. Current main still treats any non-empty replay-history array as prompt content, so blank replay entries can bypass the empty prompt/history/images skip guard. The PR is a focused fix for the linked open bug #73926, with scoped helper/test/changelog changes and no evident security-sensitive diff surface.

## Best Possible Solution

Review and land this PR or an equivalent focused fix that changes `hasPromptSubmissionContent` from replay-array presence to model-visible replay content, preserves structured non-text replay such as images, and keeps the regression tests plus changelog attribution for #73926.
