## Summary

Keep this issue open. The discussion narrowed the bug to malformed persisted cron rows with missing `schedule` or incomplete/missing `payload`, and current main still allows those rows to reach Control UI code paths that dereference `schedule.kind` and `payload.kind` directly.

## Best Possible Solution

Keep open and fix the malformed persisted-row boundary. Harden `src/cron/service/store.ts` and `src/cron/normalize.ts` so rows missing required `schedule` or a valid `payload.kind` are inferred when safe or quarantined/skipped with clear diagnostics. Then make the Control UI defensive in `ui/src/ui/controllers/cron.ts`, `ui/src/ui/views/cron.ts`, and `ui/src/ui/app-render.ts`, and add regression coverage for cron store/list behavior plus Control UI rendering with malformed `jobs.json` rows.
