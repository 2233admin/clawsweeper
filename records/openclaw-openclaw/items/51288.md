## Summary

Keep this PR open. Current main still does not implement the PR’s intended user-message `<relevant-memories>` sanitization: the Control UI extraction and rendering paths strip inbound metadata/envelopes but not XML memory blocks, while `memory-lancedb` still injects those blocks through `prependContext`. The PR is relevant, but it needs rebase and hardening before merge because current main’s renderer also uses `normalizeMessage`, the gateway history projection has a separate sanitizer path, and review feedback flags user-authored tag and malformed-prefix edge cases.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but consolidate it with the overlapping open attempts (#53588 and #59697) or pick one canonical PR. The fix should add a user-visible sanitizer for plugin-injected leading relevant-memory scaffolding in the shared gateway/UI display paths (`chat-sanitize`, `message-normalizer`, and `message-extract` as needed), preserve user-authored literal tags outside the injected-prefix shape, fail open on malformed user text, and add regression tests for string and array content in both chat history and Control UI rendering.
