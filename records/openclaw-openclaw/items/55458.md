## Summary

Keep open. Current main still hides memory probe failures, but this PR should not merge as-is because it mixes the useful status fix with stale and unrelated Docker, Fly, workflow, SDK, docs, and runtime-config changes that introduce blocking correctness and security concerns.

## What This Changes

The PR surfaces memory probe errors in status output while also adding Fly deployment workflow/scripts, Docker runtime changes, workflow-sanity filters, plugin-SDK outbound helper re-exports, bundled channel import updates, docs edits, and sync-runtime-config tests.

## Best Possible Solution

Land a focused current-main replacement that captures memory probe failures in the shared status snapshot, renders them in the memory status line, and adds a regression test for the failing-probe path; leave Fly deployment, Docker image policy, workflow, SDK boundary, and template changes to separate reviewed PRs.

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve OpenAI provider selections:** `scripts/sync-runtime-config.mjs:164`
  - body: `availableProviders` skips `openai` once `openai-codex` consumes `OPENAI_API_KEY`, so an explicit `openai/...` primary model is treated as unavailable and rewritten to Codex on startup. That breaks the script's stated preserve-user-overrides behavior.
  - confidence: 0.91
- **[P1] Restore the non-root runtime user:** `Dockerfile:262`
  - body: The Dockerfile removes the existing `USER node` hardening and adds a root entrypoint without switching back, so the gateway and entrypoint run as root. That broadens the impact of any gateway, plugin, or dependency compromise and should not be bundled into a status diagnostics PR.
  - confidence: 0.9
- **[P2] Fix Docker GPG awk quoting:** `Dockerfile:238`
  - body: The awk program is single-quoted, so the embedded escaped quotes are passed literally and cause an awk parse error when `OPENCLAW_INSTALL_DOCKER_CLI` is enabled. That aborts the Docker build before key fingerprint verification completes.
  - confidence: 0.87
- **[P3] Keep empty-message probe failures visible:** `src/commands/status.scan.ts:183`
  - body: The conditional spread drops `probeError` when the probe throws `new Error("")`, so that failure remains hidden. Preserve the field based on presence, and render based on presence, so every thrown probe failure is visible.
  - confidence: 0.78
