## Summary

Keep open. Current main still loses pending approvals on gateway restart and still exposes the stale approval miss through user-facing approval surfaces: `/approve` formats the raw gateway sentinel, while Telegram inline approval callbacks rethrow the not-found error as retryable failure. The related narrow fix PRs are closed unmerged, and no open PR in the provided context or API search specifically closes this issue.

## What This Changes

The issue asks OpenClaw to make stale Telegram approval buttons after gateway restart understandable, either by persisting pending approvals or by translating missing approval IDs into clear re-request guidance.

## Best Possible Solution

Implement a narrow UX fix that maps approval-not-found failures to actionable re-request guidance in both the generic `/approve` reply path and Telegram inline callback path, with focused regression tests. Leave disk-backed pending approval persistence for a separate product/storage follow-up.
