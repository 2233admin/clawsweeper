## Summary

Current main still replays missed cron jobs during gateway startup and does not expose a global, per-job, or max-age/window policy to skip stale missed runs. Related issue #42039 overlaps on exposing missed-job limits/staggering, but it does not fully cover the requested skip/window semantics unless maintainers broaden it.

## Best Possible Solution

Keep this open and implement a backward-compatible cron missed-run policy: preserve replay by default, add explicit global and optionally per-job skip/max-age controls, define maxMissedJobsPerRestart=0 semantics if that setting is exposed, and update runtime, config schema/help, gateway protocol, CLI add/edit surfaces, docs, and restart catch-up tests while coordinating with #42039.
