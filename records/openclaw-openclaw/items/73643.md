## Summary

Current main still runs missed cron jobs during Gateway startup and does not expose a global, per-job, or max-age/window policy to skip stale missed runs. Related issue #42039 overlaps on missed-job throttling/exposure, but it does not fully cover this issue's catch-up policy semantics, so this should stay open for maintainer direction.

## Best Possible Solution

Keep this issue open. The best implementation is a backward-compatible cron-owned missed-run policy: preserve replay by default, add an explicit global opt-out and/or max-age window, optionally add a per-job override for time-sensitive jobs, make any maxMissedJobsPerRestart=0 semantics explicit if exposed, update config/schema/help/docs and CLI/protocol surfaces, and add restart catch-up tests for default replay, global skip, per-job override, and stale-window behavior. Coordinate scope with #42039 rather than closing this automatically.
