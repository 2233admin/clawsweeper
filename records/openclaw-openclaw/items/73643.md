## Summary

Current main still replays missed cron jobs during Gateway startup and does not expose a global, per-job, or max-age/window policy to skip stale missed runs. Related issue #42039 overlaps on missed-job throttling/exposure, but it is not a complete duplicate because the current deferred path would still schedule missed jobs after restart unless catch-up policy semantics change.

## Best Possible Solution

Keep this issue open. Implement a backward-compatible missed-run policy in the cron-owned surface: keep replay as the default, add a global cron default plus optional per-job override and/or max-age window, make maxMissedJobsPerRestart=0 semantics explicit if used for skipping, update config/schema/help/docs and CLI add/edit surfaces, and add restart catch-up tests for default replay, global skip, per-job override, and stale-window behavior. Coordinate wording/scope with #42039 rather than closing this automatically.
