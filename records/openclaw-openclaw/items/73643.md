## Summary

Current main still runs startup catch-up for missed cron jobs and does not expose the requested global, per-job, or max-age/window policy to skip stale missed runs. This issue is recent, concrete, unprotected, and actionable in the cron scheduler/config/CLI/UI/docs surfaces, so it should stay open for maintainer follow-up.

## Best Possible Solution

Keep this issue open. Implement a backward-compatible missed-run policy in the cron-owned surface, likely a global default under cron plus optional per-job override and/or max-age window, then wire it through CronConfig, zod/schema help, cron job types, CLI/UI edit surfaces, docs, and runMissedJobs planning with tests for default replay, global skip, per-job override, and stale-window behavior.

## Risks / Open Questions

- Changing the default globally could make users miss useful recovery/reminder jobs after short outages, so any implementation should preserve current replay behavior unless explicitly configured otherwise.
- The requested policy crosses cron storage, config validation, CLI/UI edit surfaces, docs, and scheduler semantics; maintainers should choose the smallest backward-compatible public contract before implementation.
