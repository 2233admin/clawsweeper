## Summary

Keep open. Current main still ships the unclear shared status reaction defaults reported here, including `stallSoft: "🥱"` and `stallHard: "😨"`; Discord and Slack inherit the shared defaults when config overrides are absent, and Telegram still resolves/falls back through variants that start with the same stall emoji. The latest release tag also contains the old defaults, so this UX request is not implemented yet.

## Best Possible Solution

Keep this issue open and land a focused UX PR that updates the shared status reaction defaults, at minimum replacing the confusing stall defaults with clearer indicators, while preserving `messages.statusReactions.emojis` overrides. The fix should update Telegram-safe fallback variants, adjust core/Discord/Slack/Telegram regression coverage, and add user-facing docs/changelog notes for the default behavior change.
