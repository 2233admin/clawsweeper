## Summary

Keep open. Current main still ships the reported shared status reaction defaults, including `stallSoft: "🥱"` and `stallHard: "😨"`; Discord and Slack inherit those defaults unless users configure overrides, and Telegram still resolves/falls back through variants that start with the same stall emoji. The latest release tag also contains the old defaults, so this UX request is not implemented yet.

## Best Possible Solution

Keep this issue open. Make a small shared channel UX change in `src/channels/status-reactions.ts` to use clearer defaults, update Discord/Slack/Telegram coverage, and choose Telegram-safe fallback variants so unsupported proposed emoji do not collapse back to the old confusing stall indicators. Keep `messages.statusReactions.emojis` as the override path and align config help/docs with the shipped defaults.
