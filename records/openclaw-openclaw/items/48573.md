## Summary

Keep open. Current main has partial ACP/subagent cleanup, but the reported stale-session class remains possible for ACP child sessions: task maintenance treats a persisted ACP child session row as authoritative backing state and does not verify a live process, runtime handle, lease, heartbeat, or parent liveness. Prior discussion already narrowed the most concrete gap to ACP sessions, and open PR #68718 is a broader possible implementation path rather than a merged fix.

## Best Possible Solution

Keep this issue open until a merged fix adds process-aware or lease/heartbeat-based reconciliation for ACP child sessions, and ideally aligns native subagent handling under the same parent-child lifecycle contract. The fix should cover parent abort, timeout, gateway crash/restart, cascade cancellation, and still-present stale ACP session metadata, with regression tests proving such rows no longer keep tasks alive indefinitely. Review #68718 as the broader implementation path, or land a narrower task-maintenance/ACP-session-liveness fix if that PR is not the chosen direction.
