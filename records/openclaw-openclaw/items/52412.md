## Summary

Current main still does not expose the requested `api.runtime.agent.abort({ runId })` plugin runtime API. The runId prerequisite is present and gateway-level abort plumbing for agent RPC runs exists, but the plugin runtime type, runtime builder, SDK test mock, and docs do not include the requested method. The adjacent #67586 work fixed `chat.abort` for agent RPC runs and #66532 is related embedded-run API work, but neither replaces this plugin-facing API request.

## Best Possible Solution

Keep this open for maintainer API review. The likely implementation is an additive, documented `api.runtime.agent.abort({ runId, sessionKey? })` runtime seam in `src/plugins/runtime/types-core.ts` and `src/plugins/runtime/runtime-agent.ts`, backed by the existing gateway abort infrastructure with explicit sessionKey/authorization semantics, plus SDK docs, runtime mock/API baseline coverage, and targeted tests for success, missing run, cross-session denial, and cleanup behavior. Maintainers should decide whether to keep it separate from #66532 or fold both into one broader plugin cancellation API plan.
