## Summary

Keep open. Current main still renders Control UI chat history by a fixed 200-message window only, while this PR adds the missing character-budget windowing and related tests for large tool-result payloads. The PR is still an implementation candidate, but it should get maintainer review after the latest repair and the security review note about bounded recursive content estimation.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and have the UI maintainers review the repaired branch. The best implementation path is to land the character-budgeted `buildChatItems` windowing with tests for count caps, large string content, tool_result `content`, nested tool_result content, and hidden tool messages, while making the estimator bounded/cycle-safe and preserving the current server-side chat.history contract.
