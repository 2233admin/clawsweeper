## Summary

Keep open. Current main has complementary Gateway `chat.history` payload caps, but it does not implement this PR's UI-side aggregate render-character budget. The Control UI still selects the last 200 history messages by count only, then renders that window; there is no `CHAT_HISTORY_RENDER_CHAR_BUDGET`, `estimateMessageRenderChars`, or equivalent current-main test coverage. The PR discussion also shows the author addressed the bot/maintainer review findings in later commits, so this remains a plausible focused PR after rebase.

## Best Possible Solution

Keep the PR open. Rebase the char-budget selection into `ui/src/ui/chat/build-chat-items.ts`, keep Gateway `chat.history` truncation as a complementary safeguard, and add focused `build-chat-items` tests for the count cap, aggregate char budget, tool-result `content`, hidden tool messages, raw-walk cap, and accurate visible history notice.
