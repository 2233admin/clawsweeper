## Risks / Open Questions

- The PR is not obsolete: current main does not implement the requested UI render character budget, so closing it would drop a still-relevant candidate fix for large-session Control UI freezes.
- The branch's `estimateRawContentChars` recursively traverses raw nested content without an explicit depth, node, cycle, or remaining-budget bound. Chat history is normally JSON, but the added synchronous walk is in the same client-side DoS/performance surface this PR is trying to reduce.
- The branch adds extra pre-render `normalizeMessage` calls in the history trimming path. Existing code also normalizes during rendering, but malformed or non-object entries should be considered before merge because the prepass can fail before any visible item is built.
- The PR base SHA is behind the reviewed checkout, so it should be rebased or refreshed against current main before any merge decision.
