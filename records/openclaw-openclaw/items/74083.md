## Summary

Keep this PR open. Current main still does not pass `--init` when building Docker sandbox `create` arguments, and the standard sandbox container still runs `sleep infinity` as the long-lived command. The PR is a focused implementation candidate for the same remaining sandbox-zombie problem tracked in #68691, but it still needs maintainer/secops review and a quick decision on how existing containers pick up the new create flag.

## Best Possible Solution

Keep this PR open for maintainer/secops review. The best fix is to add Docker `--init` through `buildSandboxCreateArgs`, keep the focused unit coverage asserting its placement after `create`, and decide whether to force container recreation through the sandbox hash/version path or document `openclaw sandbox recreate` for existing runtimes. Validate with the targeted sandbox create-args test and the normal changed gate before merge.
