## Summary

Keep this PR open. Current main still does not pass Docker `--init` in `buildSandboxCreateArgs`, and the standard sandbox container still runs `sleep infinity`, so the PR is not obsolete. The diff is narrow and directly addresses the zombie-process problem tracked in #68691, but it touches the security-owned sandbox startup path and needs maintainer/secops review plus a decision on how existing containers pick up the new create flag.

## Best Possible Solution

Keep this PR open for maintainer/secops review. The best fix is to add Docker `--init` through `buildSandboxCreateArgs`, keep focused unit coverage asserting it appears after `create` and before the image, and decide whether to force container recreation through the sandbox hash/version path or document `openclaw sandbox recreate` for existing runtimes. Validate with the targeted sandbox create-args test and the normal changed gate before merge.
