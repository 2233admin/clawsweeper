## Summary

Keep open. Current main still lacks Docker `--init` for sandbox containers, and the related zombie-process report remains open; the PR is useful but still needs maintainer/secops review because it changes the Docker sandbox startup path and leaves an explicit rollout choice for already-created containers.

## What This Changes

The PR adds Docker `--init` to sandbox container creation arguments and adds unit coverage asserting the flag is present after `docker create`.

## Best Possible Solution

Ship a secops-reviewed sandbox fix that enables Docker init for new sandbox containers and makes the rollout behavior explicit for existing runtimes, either by forcing recreation through the existing hash/epoch mechanism or by documenting the required recreate path clearly enough for affected users.

## Reproduction Assessment

Unclear for a minimal live reproduction: the linked report provides credible PPID=1 zombie observations, and current main confirms Docker sandboxes run `sleep infinity` without `--init`, but this read-only pass did not verify a live command sequence that accumulates zombies.

## Solution Assessment

Unclear as submitted: adding `--init` in `buildSandboxCreateArgs` is the narrow right code path for new containers, but the best fix should also settle how already-created matching containers receive the new startup flag.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Handle rollout for existing sandbox containers:** `src/agents/sandbox/docker.ts:404-408`
  - body: `ensureSandboxContainer` only recreates an existing Docker sandbox when its stored config hash differs, and the hash is based on Docker config, workspace paths/access, and the mount format version. Adding `--init` only to the create args means upgraded users with a matching existing container keep the old `sleep infinity` PID 1 until manual recreate/prune or another hash change, so the zombie-process fix may not apply to the long-running sandboxes that need it. Please bump/include a sandbox runtime epoch in the hash or make the manual recreate rollout explicit.
  - confidence: 0.86
