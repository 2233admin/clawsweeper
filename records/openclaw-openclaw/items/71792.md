## Summary

Keep open. Current main still lacks an automatic rollover command hook for bundled session-memory, and this PR is a narrow, plausible fix with coverage for daily, idle, and fresh no-op paths. I found no blocking diff bug; remaining work is maintainer review, CI cleanup, and choosing a canonical branch among overlapping open attempts.

## What This Changes

The PR emits a synthetic internal `command:reset` hook during automatic daily/idle stale-session rollover, passes previous-session and workspace context, adds regression tests, and updates memory/hooks docs plus the changelog.

## Best Possible Solution

Land one narrow implementation that reuses the existing internal `command:reset` contract for automatic daily/idle rollover, preserves previous-session and workspace context for session-memory, and keeps broader shutdown/prune or agentic memory-flush ideas in their separate feature threads.

## Reproduction Assessment

Yes. On current main, enable `session-memory`, create a stored session that is stale by daily `sessionStartedAt` or idle `lastInteractionAt`, then send a normal message: `initSessionState` rotates the session but the command hook path is gated on explicit `resetTriggered`.

## Solution Assessment

Yes. Reusing the existing `command:reset` hook is the narrowest maintainable fix for the reported bundled hook loss because it avoids a new public lifecycle API while keeping existing listeners working; broader lifecycle events can be considered separately for shutdown/prune cases.
