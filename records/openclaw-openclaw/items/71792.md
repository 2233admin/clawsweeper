## Summary

Keep this PR open. Current main still does not implement the PR's intended automatic daily/idle rollover memory persistence: stale rollover archives the previous transcript and emits typed session lifecycle hooks, but it does not emit the internal command reset hook that bundled session-memory consumes. The PR is active, closes #11960, has maintainer repair context, and is a better landing/review candidate than a cleanup close.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing unless maintainers explicitly choose another overlapping PR as canonical. The best implementation direction is the narrow one described here: emit the existing internal command:reset hook during non-command stale daily/idle rollover only when a previousSessionEntry exists, pass cfg/workspaceDir/sessionEntry/previousSessionEntry consistently with manual reset, use explicit commandSource values such as auto:daily and auto:idle, keep typed session_end/session_start behavior intact, and retain focused daily, idle, and fresh-session no-op regression coverage plus docs/changelog credit.
