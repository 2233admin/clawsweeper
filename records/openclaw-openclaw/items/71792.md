## Summary

Current main still does not implement this PR's intended automatic daily/idle rollover memory persistence. The stale rollover path creates a new session, archives/cleans the previous session, and emits typed plugin `session_end` / `session_start` hooks, but it does not emit the internal `command:reset` event that the bundled `session-memory` hook consumes. The PR remains an active, focused implementation candidate with maintainer repair comments and validation reported on the branch, so conservative cleanup should leave it open for normal review/landing.

## Best Possible Solution

Keep this PR open and review or land it as the focused canonical fix for automatic daily/idle rollover memory persistence. The right product/code path is to emit the existing internal `command:reset` event for non-command stale rollovers with clear `commandSource` values like `auto:daily` and `auto:idle`, pass `previousSessionEntry` and resolved workspace context for `session-memory`, retain daily/idle/fresh no-op coverage, and add docs only if maintainers want the expanded command-event semantics documented.
