## Summary

Keep this PR open. Current main still does not implement the PR's intended automatic daily/idle rollover memory persistence: stale rollover archives the old transcript and emits typed session lifecycle hooks, but it does not emit the internal command reset hook that bundled session-memory consumes. The PR is active, linked to open #11960 with closing syntax, and has maintainer repair context, so it remains an implementation candidate rather than cleanup to close.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing as the narrow contributor path unless maintainers explicitly choose another overlapping PR as canonical. The best code direction is to emit the existing internal command:reset hook during non-command stale daily/idle rollover when a previousSessionEntry exists, pass cfg/workspaceDir/sessionEntry/previousSessionEntry consistently with manual reset, use explicit commandSource values such as auto:daily and auto:idle, keep typed session_end/session_start behavior intact, and retain focused daily, idle, and fresh-session no-op regression coverage plus docs/changelog credit.
