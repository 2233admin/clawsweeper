## Summary

Keep open. Current main still has the reported isolated-cron behavior: text delivery waits for active descendant subagents with the full cron timeout budget and suppresses parent delivery without checking `deliveryBestEffort`. The related implementation PR #46160 was closed unmerged, so the likely fix path remains available but not shipped.

## Best Possible Solution

Fix `src/cron/isolated-agent/delivery-dispatch.ts` so text-only isolated cron delivery with `deliveryBestEffort: true` skips the active-descendant wait and active-descendant suppression path, delivering the parent output immediately while preserving non-bestEffort descendant wait/suppression behavior. Add a regression test in `src/cron/isolated-agent/delivery-dispatch.double-announce.test.ts` for active descendants plus `deliveryBestEffort: true`; PR #46160 is the closest reference but was not merged.
