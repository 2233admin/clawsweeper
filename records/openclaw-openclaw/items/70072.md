## Summary

Keep open. Current main still lacks the requested deep-dreaming introspection: scheduled deep reports only get aggregate ranked/promoted counts, verbose scoring is still logger-only, rejected candidates are filtered before callers can explain them, and the memory CLI has adjacent single-candidate/REM preview tools but no aggregate dreaming doctor or gate-reachability diagnostic. The request remains a distinct additive follow-up to #68882, not a duplicate of that closed calibration bug.

## What This Changes

Expose per-candidate deep-dreaming ranking, promotion, and rejection reasoning in reports, structured verbose artifacts, and an aggregate memory CLI diagnostic.

## Best Possible Solution

Keep this open and implement a shared memory-core deep-candidate evaluation surface that returns bounded considered candidates with score components, threshold pass/fail state, promotion status, and rejection reasons without changing default promotion policy. Use that surface to extend scheduled deep reports, optionally write structured verbose JSONL when verboseLogging is enabled, and add an aggregate CLI diagnostic for gate reachability, with docs and focused tests.
