## Summary

Keep open. The PR targets a valid current-main bug in the `agents add` auth/model write path, but it is not merge-ready because it removes every deferred provider `agentModelOverride`; that would lose the selected provider model when there is no inherited `agents.defaults.model.primary` to fall back to.

## What This Changes

The PR changes provider auth/default-model handling and related auth-choice tests so `setDefaultModel: false` no longer returns provider default models as per-agent overrides during agent auth setup.

## Best Possible Solution

Keep the PR open and narrow the implementation: suppress `agentModelOverride` only when the config has an inherited `agents.defaults.model.primary`, preserve the selected provider default as the agent override when no inherited primary exists, and cover string/object global defaults plus the no-default case.

## Reproduction Assessment

Yes. Current main is source-reproducible: deferred provider auth returns `agentModelOverride`, and `agents add` writes it into `agents.list[]`; the PR regression is also source-reproducible by following a deferred provider setup with no global primary.

## Solution Assessment

No, not as-is. The requested direction is right for the configured-default case, but the PR’s blanket removal is broader than the model-selection contract; the safer fix is conditional suppression based on whether an inherited primary exists.
