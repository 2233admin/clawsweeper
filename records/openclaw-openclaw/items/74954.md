## Summary

Keep open. Current main still contains the reported write-time model override path, and this PR is aimed at the right files, but the patch is not merge-ready because it drops the provider default override even when there is no inherited default model and GitHub reports the branch as dirty/unmergeable.

## What This Changes

The PR changes provider auth-choice/default-model handling and related tests so `setDefaultModel: false` no longer returns a provider `defaultModel` as `agentModelOverride` for new-agent auth setup.

## Best Possible Solution

Land a narrow fix that suppresses `agentModelOverride` only when a configured inherited `agents.defaults.model.primary` exists, keeps the provider default path when no inherited model exists, and adds focused tests for both cases before closing the older duplicate PRs.

## Reproduction Assessment

Yes. The current-main source path is clear: `applyAuthChoice*` can return `agentModelOverride` with `setDefaultModel: false`, and `agents add` writes that override into `agents.list[]`; the #24170 steps exercise that path with a configured global primary.

## Solution Assessment

No. The proposed direction is right for the configured-default case, but the PR is too broad because it removes the no-default fallback behavior and is currently dirty against main; a conditional override is the safer maintainable fix.

## Review Findings
