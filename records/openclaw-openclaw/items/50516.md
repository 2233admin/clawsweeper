## Summary

Keep this PR open. Current main still has the exact Nextcloud Talk verifier behavior the PR aims to harden: an early return on signature-length mismatch followed by a manual XOR loop. The PR is a focused security improvement, not obsolete cleanup, though its current mismatch-branch dummy comparison should be tightened before merge.

## Best Possible Solution

Keep this PR open and have the contributor revise the length-mismatch branch to match the repo's established timing-safe comparison patterns, then land the focused production change with the new signature tests after targeted Nextcloud Talk verification and the normal changed gate.
