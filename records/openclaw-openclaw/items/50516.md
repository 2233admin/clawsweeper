## Summary

Keep this PR open. Current main still verifies Nextcloud Talk webhook signatures with an immediate length-mismatch return and a manual XOR loop, so the hardening this PR targets is not implemented. The PR is narrowly scoped and useful, but it should be revised before landing to avoid the self-comparison dummy `timingSafeEqual(expBuf, expBuf)` pattern called out in review.

## Best Possible Solution

Keep this PR open. Ask the contributor to update the mismatch branch to follow the existing LINE/shared timing-safe comparison pattern, retain the focused Nextcloud Talk signature tests, then run the targeted Nextcloud Talk test lane and the normal changed gate before landing.
