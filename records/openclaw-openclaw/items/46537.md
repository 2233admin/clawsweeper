## Summary

Keep this PR open. Current main still creates ensure-derived ACP identities as pending even when the handle already has stable ACP/session IDs, so the PR's intended behavior is not implemented. The PR also still needs maintainer review for the merge-precedence concern: making ensure results resolved can make stale ensure IDs overwrite fresher resolved/status IDs unless merge authority is tightened.

## Best Possible Solution

Keep this PR open and revise it. The best path is to implement ensure identity promotion for stable IDs while tightening merge authority so ensure-sourced values cannot regress current resolved/status-sourced identifiers, then keep focused tests for stable-ID ensure, record-only ensure, startup reconcile skipping, and stale ensure-vs-status behavior.
