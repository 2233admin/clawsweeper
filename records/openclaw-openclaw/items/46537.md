## Summary

Current main still does not implement this PR's intended ACP identity behavior: ensure-derived identities are always created as pending even when stable ACP/session IDs are present, and startup reconciliation still processes pending identities with stable IDs. The PR remains a plausible fix candidate, but it also has a real unresolved merge-precedence risk where marking ensure results as resolved could let stale ensure IDs overwrite fresher resolved IDs.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to implement the intended ACP ensure identity promotion while preserving merge precedence so ensure results cannot regress fresher resolved IDs, then keep focused tests for stable-ID ensure, record-only ensure, startup reconcile skipping, and stale ensure-vs-status behavior.
