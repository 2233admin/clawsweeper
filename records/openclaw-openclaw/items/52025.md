## Summary

Keep this PR open. Current main has adjacent image/media support, but it still does not implement the PR’s intended behavior for direct tool-result image blocks shaped as `{ type: "image", data, mimeType }`. The PR discussion is useful: the Greptile summary and follow-up review comments identified real edge cases around image-only results, nested tool-result content, and images hidden behind collapsed tool output.

## Best Possible Solution

Keep this PR open as a focused implementation candidate for the open tool-returned image bug tracked by #50779, but require an updated patch against current main. The best fix should handle direct and nested `{ type: "image", data, mimeType }` tool-result blocks, image-only results, accompanying text/captions, and default visibility outside collapsed tool details, with regression coverage in the real grouped renderer. It should also account for live-stream and `chat.history` behavior, preferably by converting trusted tool image blocks into bounded managed media references rather than relying on raw base64 surviving projection.
