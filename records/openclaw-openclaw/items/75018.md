## Best Possible Solution

Keep the implementation provider-owned in `extensions/gradium`, fix the provider-page Voice Call example so copied config actually enables streaming, then merge after the draft is ready and maintainer validation confirms the Gradium ASR protocol and live-test behavior.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug. There is a high-confidence static check that current main lacks Gradium realtime transcription registration, and the PR body provides concrete unit and live verification steps for the added behavior.

## Solution Assessment

Mostly yes, but not as currently documented. The provider-owned implementation using the existing realtime transcription SDK seam is the narrow maintainable path; the Gradium provider docs should add `streaming.enabled: true` before merge.
