## Summary

Keep open. Current main has moved the memory code into the bundled memory-core plugin, but it still implements the exact provider/model filter that this PR is trying to remove from FTS keyword search. The linked bug remains open, and the PR still represents useful work, though it needs to be rebased onto the current `extensions/memory-core` paths with regression coverage there.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate, but ask for a rebase/update against `extensions/memory-core/src/memory/manager-search.ts`, `extensions/memory-core/src/memory/manager.ts`, and `extensions/memory-core/src/memory/manager-search.test.ts`. The best fix is to stop passing/applying provider model filters in FTS keyword search, preserve vector model filtering, and add the old-model/new-model keyword regression in the current memory-core test file.
