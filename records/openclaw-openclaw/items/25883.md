## Summary

Keep open. Current main still uses process-local command lanes, Gateway-local cron scheduling, file-backed cron state, and local SQLite task/flow ledgers; it does not implement the requested Redis/BullMQ-style distributed queue, shared worker claiming, dead-letter handling, or horizontal worker coordination. The existing maintainer comment also explicitly keeps this as architecture input, with #13613 covering only the narrower multi-gateway cron coordination slice.

## What This Changes

Request to add a Redis/BullMQ-style horizontal scaling architecture for background jobs, delayed jobs, scheduled jobs, retries/backoff, failure handling, workers, and observability.

## Best Possible Solution

Keep open for maintainer architecture review. Decide whether OpenClaw should add a generic distributed scheduler/job backend abstraction, possibly plugin-capable, with shared queue/state, worker claiming, retries/backoff, dead-letter semantics, cron support, and observability; otherwise document Redis/BullMQ-style orchestration as an external deployment or plugin pattern while preserving the ScalyClaw prototype as prior art. Coordinate the cron HA slice with #13613.
