## Summary

Keep this open. Current main still has local/process-bound cron, command queue, task ledger, and Task Flow surfaces; it does not implement the requested Redis/BullMQ-style distributed job orchestration or horizontal worker coordination. A maintainer comment already kept this open as architecture input, and the open related #13613 only covers the narrower cron HA slice, not the broader background-job orchestration request.

## Best Possible Solution

Keep this open for maintainer architecture review. The best path is to decide whether OpenClaw should add a generic distributed scheduler/job backend abstraction, possibly plugin-capable, with shared queue/state, worker claiming, retries/backoff, failure/dead-letter semantics, cron support, and observability; otherwise document Redis/BullMQ-style orchestration as an external deployment pattern while preserving the ScalyClaw prototype as prior art. Coordinate any narrower cron HA work with #13613.
