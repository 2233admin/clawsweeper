## Summary

Keep open. Current main still ignores the child-process exit signal in the node-host exec path, so signal-killed subprocesses still resolve as `exitCode: undefined` with `error: null`; the provided PR discussion shows the earlier timeout-semantics and test-isolation concerns were addressed, and I found no blocking correctness or security issue in the proposed diff.

## What This Changes

The PR adds node-host exit-result handling that reports non-timeout signal exits as `process killed by signal ...` errors and adds focused unit coverage for signal, normal-exit, and timeout-kill cases.

## Best Possible Solution

Land a focused node-host exec fix that reads `(code, signal)`, reports non-timeout signal terminations in the structured result, keeps intentional timeout kills as `timedOut: true` with `error: null`, and retains regression coverage close to the helper or run-command path.
