## Summary

Keep open. Current main still ignores the signal parameter on node-host exec exits, and the PR's latest diff appears to address that gap while preserving timeout semantics; prior review feedback has been incorporated, with rebase/green CI as the remaining merge blocker.

## What This Changes

The PR adds a tested `resolveExitResult` helper and uses it in `src/node-host/invoke.ts` so non-timeout signal-terminated child processes produce an error string while timeouts and numeric exits keep their existing result shape.

## Best Possible Solution

Land this PR or an equivalent narrow change after rebase and green CI, keeping timeout kills as `timedOut: true, error: null` while surfacing non-timeout signal kills through `RunResult.error` and exec-finished output.
