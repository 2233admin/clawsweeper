## Summary

Keep this PR open. Current main still does not pass requester sender context into channel-owned agent tool factories, and the PR remains a narrow additive implementation candidate for a public channel plugin SDK gap. The supplied PR diff is small and functionally aligned with existing trusted requester semantics; it is not obsolete on main.

## Best Possible Solution

Review and land this PR, or an equivalent additive SDK/core change, with a focused regression test in the agent tool composition area proving senderId and senderIsOwner are forwarded into a ChannelAgentToolFactory. Keep the fields optional and documented as trusted server-injected context, matching the existing channel message-action discovery semantics.
