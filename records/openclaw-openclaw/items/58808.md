## Summary

Keep this PR open. Current main still does not pass requester sender context into channel-owned agent tool factories, and the PR remains a narrow implementation candidate for a real plugin SDK gap rather than obsolete work.

## Best Possible Solution

Review and land this PR, or an equivalent additive SDK/core change, with a focused regression test around createOpenClawCodingTools forwarding senderId and senderIsOwner into a channel agentTools factory. Keep the trusted sender semantics aligned with the existing message-action discovery context.
