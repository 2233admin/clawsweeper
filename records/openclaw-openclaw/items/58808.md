## Summary

Current main still does not implement this PR's intended additive channel-plugin SDK seam. The PR remains useful: the linked report and PR body describe a concrete DMWork channel-plugin need for sender-aware agent tools, and the current code already forwards trusted sender context to adjacent message-action/core-tool paths but not to ChannelAgentToolFactory.

## Best Possible Solution

Review and land this PR, or an equivalent narrow core SDK change, with a regression test. The implementation should add optional `requesterSenderId` and `senderIsOwner` fields to `ChannelAgentToolFactory` and `listChannelAgentTools`, thread `options.senderId` and `options.senderIsOwner` from `createOpenClawCodingTools`, and keep the trusted server-injected wording aligned with the existing message-action context.
