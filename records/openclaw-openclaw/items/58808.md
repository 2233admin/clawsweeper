## Summary

Keep open. Current main still has the gap described by the PR: channel-owned agent-tool factories are typed and called with only cfg, while the adjacent core tool path receives requester sender context. The proposed patch is a small additive public channel/plugin SDK change and appears directionally correct, but it needs maintainer review for contract shape, regression coverage, and docs/changelog expectations before merge.

## What This Changes

The PR adds optional requesterSenderId and senderIsOwner fields to ChannelAgentToolFactory/listChannelAgentTools and forwards createOpenClawCodingTools sender context into channel-owned agent-tool factories.

## Best Possible Solution

Land this PR or an equivalent narrow patch with regression coverage for trusted sender-context propagation into ChannelAgentToolFactory, and update the plugin SDK docs/changelog if maintainers treat the new fields as user-facing.
