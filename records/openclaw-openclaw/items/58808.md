## Summary

Current main still does not implement this PR's intended additive channel-plugin SDK seam. The PR remains useful: the body and linked discussion describe a concrete DMWork channel-plugin need for sender-aware agent tools, and current main already carries trusted sender context through adjacent message-action/core-tool paths but not through ChannelAgentToolFactory.

## Best Possible Solution

Review and land this PR, or an equivalent narrow core SDK change, with regression coverage. The implementation should add optional requesterSenderId and senderIsOwner fields to ChannelAgentToolFactory/listChannelAgentTools, thread options.senderId/options.senderIsOwner from createOpenClawCodingTools, and keep the trusted inbound context wording aligned with the existing message-action context.
