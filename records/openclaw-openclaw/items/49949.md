## Summary

Keep open. Current main does not implement Anthropic native server-side web search as a `tools.web.search.provider: "anthropic"` option: the Anthropic bundled plugin registers its model provider, CLI backend, and media-understanding provider only, while the web-search docs and provider artifacts list the existing managed providers plus native OpenAI/Codex behavior, not Anthropic native search. The issue is also explicitly paired with open PR #50692 using closing syntax, so it should remain open until that PR is reviewed, merged, or closed. Broader Anthropic server-side tool support is also being tracked in #23353.

## Best Possible Solution

Keep this issue open and route work through review of PR #50692 or a focused successor PR. The implementation should live in the Anthropic bundled plugin/provider surfaces, add the manifest/config/docs/tests needed for `tools.web.search.provider: "anthropic"`, and coordinate with #23353 if the same request-payload and response-block handling should support future Anthropic `web_fetch` or `code_execution` server tools.
