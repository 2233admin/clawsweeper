## Summary

Keep open. Current main still lacks a legacy `channels.imessage` auto-acknowledgment or typing-feedback path before agent dispatch. BlueBubbles has automatic typing support, but it is a separate recommended channel and does not implement this request for the legacy imsg channel.

## Best Possible Solution

Keep this open until maintainers either implement or explicitly decline legacy `channels.imessage` feedback. If implemented, keep it in `extensions/imessage`: add typed config/schema/docs for `autoAck` or a native-typing-with-text-fallback policy, send feedback after access and mention gating but before `dispatchInboundMessage`, honor `dmOnly`, and add regression coverage proving the acknowledgment is not re-ingested as inbound user text. Continue recommending BlueBubbles for new iMessage deployments.

## Risks / Open Questions

- A text auto-ack is a real outbound iMessage and must be remembered in the sent-message cache so it is not reflected back into the agent as user text.
