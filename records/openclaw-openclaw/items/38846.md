## Summary

Keep open. Current main has not implemented the PR’s proposed exported validator, null/control-character checks, or PowerShell-pattern check, so the branch is not obsolete; however, the PR as written is not safe to merge because it applies an over-broad blacklist to the Windows cmd.exe wrapper path and would reject ordinary Windows paths, scoped package names, slash switches, equals-style npm flags, and would echo full argument values in errors.

## What This Changes

This PR modifies `src/process/exec.ts` to broaden Windows cmd.exe argument rejection, add PowerShell-pattern/null/control-character validation, call that validation from `escapeForCmdExe`, and export `validateWindowsCommandArgument`.

## Best Possible Solution

Replace or revise this with a narrower Windows command-hardening change that targets real cmd.exe wrapper metacharacter hazards, preserves path separators, scoped package specs, slash switches, and equals flags, avoids PowerShell-specific policy outside PowerShell execution, redacts argument values in errors, and adds focused regression tests for Windows wrapper behavior.
