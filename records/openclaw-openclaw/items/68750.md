## Summary

Keep open. Current main does not recognize `prompt_cache_hit_tokens`, the linked MiniMax bug remains open, and the related alternative PR was closed unmerged, so this PR is not obsolete or implemented. The diff is small and limited to usage normalization, but it should get maintainer review because the exact MiniMax response shape needs confirmation and the PR currently lacks regression coverage.

## What This Changes

The PR adds MiniMax `prompt_cache_hit_tokens` handling to `src/agents/usage.ts` so cache-hit prompt tokens can be extracted as `cacheRead` and subtracted from OpenAI-style prompt totals before compaction math.

## Best Possible Solution

Confirm the affected raw MiniMax usage payload, then either land a narrow normalization change with a focused `normalizeUsage` regression test or redirect the fix to the actual provider field/path if `prompt_cache_hit_tokens` is not emitted by supported MiniMax traffic.
