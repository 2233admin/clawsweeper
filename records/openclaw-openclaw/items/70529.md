## Summary

Keep open. Current main retires the legacy EasyClaw/Chrome extension relay path, but it still leaves the Desktop `user` profile on the default Chrome MCP launch path that depends on ambient `npx`, which matches the remaining packaged-app failure described in the issue and the maintainer triage comment.

## What This Changes

The issue asks OpenClaw Desktop to support existing signed-in Chrome sessions without EasyClaw OAuth failure or a bundled-app `spawn npx ENOENT` failure for the built-in `user` browser profile.

## Best Possible Solution

Keep the issue open and make the packaged Desktop/browser runtime handle the built-in `user` profile without relying on ambient `npx`: either ship or resolve a pinned Chrome MCP runner through the app boundary, or fail early with a structured missing-runner diagnostic that points to the managed `openclaw` profile and explicit `mcpCommand` override. Keep the removed EasyClaw relay path retired.

## Reproduction Assessment

Yes for the remaining Desktop `user` profile failure: current main defaults `user` to `existing-session`, builds the default Chrome MCP command as `npx -y chrome-devtools-mcp@latest`, and passes it to the stdio transport. The EasyClaw relay half is obsolete on current main because the extension relay lifecycle is now a no-op.

## Solution Assessment

Unclear because there is no proposed patch and the right fix requires a maintainer packaging strategy. The existing `mcpCommand` override is a useful workaround, but it is not a complete Desktop fix for the built-in profile users expect to work out of the box.
