## Summary

Keep open. Current main still routes the built-in `user` browser profile through Chrome MCP existing-session attach, and that path still defaults to spawning ambient `npx -y chrome-devtools-mcp@latest`. The removed EasyClaw/extension relay part is obsolete, but the Desktop-specific packaged-app failure remains valid and was explicitly scoped by a maintainer as separate from the broader browser attach and login UX issues.

## Best Possible Solution

Keep this issue open and decide the Desktop/browser runtime strategy: make the built-in `user` existing-session profile work without ambient `npx` in the packaged app by bundling or resolving a pinned Chrome MCP executable, or add a structured missing-runner diagnostic that points to the managed `openclaw` profile and explicit `mcpCommand` override. The change belongs in the browser plugin/Desktop packaging boundary, with docs and targeted browser config/Chrome MCP tests updated.
