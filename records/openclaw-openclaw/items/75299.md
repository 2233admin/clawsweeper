## Summary

Keep open. Current main still has FIFO-only command-queue draining with no priority API, so the PR targets a real remaining feature gap. The force-pushed head appears to address the earlier starvation and task-timeout concerns, but it is not merge-ready because `check-lint` is failing on mechanical lint violations in the changed queue code and tests.

This PR adds command-queue priority levels and optional enqueue priority, changes lane draining to select the highest effective-priority queued entry with aged-entry promotion, and adds process queue scheduling tests.
