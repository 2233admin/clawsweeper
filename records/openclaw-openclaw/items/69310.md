## Summary

Keep this PR open. It addresses a real current-main media-drop gap, but the latest head still has a blocking correctness issue: when every media item fails normalization, the branch keeps the legacy generic warning and also delivers the new dropped-media notice, so users receive duplicate failure messages. The earlier `reason` vs `code` test-shape finding appears fixed in the latest head.

## What This Changes

The PR adds dropped-media tracking for reply media normalization failures, user/transcript notices, outbound JSON/transport preservation, warning logs, and focused tests.

## Best Possible Solution

Remove or replace the legacy `formatBlockedReplyMediaWarning` text once `droppedMedia` is populated, so a failed attachment produces exactly the sanitized dropped-media notice across text, media, structured, fallback, and transcript paths; then rerun the focused dropped-media/media-path/outbound tests and the changed gate before normal maintainer review.
