## Summary

Keep open: the user problem is still real on current main and this branch is the active fix, but the latest head still fails CI and introduces merge-blocking issues around runtime type compatibility, silent NO_REPLY media failures, lint, and duplicate all-dropped warnings.

## What This Changes

The PR adds dropped-media tracking and user-visible notices for reply media normalization failures across reply normalization, outbound delivery, transcript mirroring, JSON projection, routing/follow-up paths, and tests.

## Best Possible Solution

Keep the dropped-media direction, but update the copied runtime payload contracts, preserve silent NO_REPLY suppression for failed media, remove lint-blocking unused imports, replace the legacy generic all-failed warning with the new structured notice, and rerun the focused reply/outbound tests plus typecheck/lint before merge.
