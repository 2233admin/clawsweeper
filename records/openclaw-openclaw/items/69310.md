## Summary

Keep this PR open. Current main still only emits a generic warning when every normalized media item is dropped, and it still lacks the PR's intended structured dropped-media pipeline for partial drops, catastrophic normalization failures, outbound JSON/delivery, transcript mirroring, routing, and followup delivery. The related #70109 candidate is closed and unmerged, so it does not supersede this active implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review. The best outcome is to land this PR after validating the current head, or replace it with an equivalent maintainer-approved implementation that preserves sanitized per-item dropped-media details through normalization, catastrophic fallback, block/routing/followup filters, outbound JSON/delivery, and transcript mirroring.
