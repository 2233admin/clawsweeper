## Summary

Current main still does not implement this PR's intended dropped-media visibility contract. Main only emits a generic warning when every normalized media item fails; partial drops, catastrophic normalization failures, outbound JSON, dropped-media-only routing/followups, and transcript mirroring still lack the PR's structured `droppedMedia` pipeline. The related #70109 PR is closed and unmerged, so it does not supersede this active contribution.

## Best Possible Solution

Keep this PR open. The best path is a maintainer-reviewed implementation that preserves sanitized per-item dropped-media details through reply media normalization, catastrophic fallback, block/routing/followup filters, outbound JSON/delivery, and transcript mirroring, while retaining the privacy and delivery-safety fixes already worked through in the review thread.

- The supplied diff snippets show some tests using `reason` while the introduced type uses `code`; that may be stale context, but the PR needs a typecheck/test pass before merge.
