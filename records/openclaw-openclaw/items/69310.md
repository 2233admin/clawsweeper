## Summary

Keep this PR open. Current main at 64bd2a2cbe805ddd850dbcbd210c1c8cb6499db6 still lacks the structured dropped-media diagnostics contract this PR is trying to add: partial media normalization failures are still continued after verbose logging, catastrophic normalization still clears media, outbound payload/json/mirror types cannot carry dropped-media diagnostics, and delivery has no attachment-failure notice path. The PR is an active implementation candidate, but it changes privacy-sensitive logging, user-visible channel text, routing, hooks, result identity, and transcript audit behavior, so it needs maintainer review rather than cleanup closure.

## What This Changes

The PR adds DroppedMedia types/helpers and threads dropped-media entries through reply media normalization, routing, outbound payload projection, delivery notices, transcript mirroring, and tests.

## Best Possible Solution

Keep this PR open for maintainer review. The best outcome is to land this PR, or an equivalent maintainer-owned implementation, after validating that sanitized dropped-media diagnostics survive per-item normalization, catastrophic fallback, block/followup routing, outbound JSON/delivery, media-unsupported fallback, hooks/result-id preservation, and transcript mirroring without leaking sensitive source details.
