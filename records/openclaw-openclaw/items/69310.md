## Summary

Keep this PR open. Current main still only emits a generic warning when all reply media is dropped, still silently loses partial drops and catastrophic normalization failures, and still has no DroppedMedia contract through routing, outbound JSON, delivery, or transcript mirroring. The related #70109 candidate is closed and unmerged, so it does not supersede this active PR.

## Best Possible Solution

Keep this PR open for maintainer review. The best outcome is to land this PR, or an equivalent maintainer-owned implementation, after fixing current head cleanup and validating that sanitized per-item dropped-media diagnostics survive per-item normalization, catastrophic fallback, routing/followup/block filters, outbound JSON/delivery, hook/audit behavior, and transcript mirroring without exposing paths or secrets.
