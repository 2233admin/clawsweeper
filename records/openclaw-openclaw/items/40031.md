## Summary

Keep #40031 open. Current main has landed useful isolated cron direct-delivery behavior, but the PR's remaining core requests are not obsolete: main-session announce delivery is still rejected or cleared, the CLI still blocks the requested main announce/account flow, Feishu `on_`/`union_id` target resolution is still missing, and `OPENCLAW_HOST_STATE_DIR` media-path rewriting is absent. The branch is broad and should be split/rebased rather than closed.

## Best Possible Solution

Keep the PR open, but request a narrowed rebase or split follow-ups: one core cron PR for main-session announce delivery and heartbeat override propagation with CLI/account/failure-destination tests; one Feishu plugin PR for `on_`/`union_id` target normalization and send-target tests; and one Docker/media PR only if maintainers accept `OPENCLAW_HOST_STATE_DIR` as a supported contract. Drop the already-covered isolated direct-delivery work and unrelated baseline, Dockerfile, skipped-test, and CI-trigger churn.
