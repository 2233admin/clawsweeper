## Summary

Keep open. Current main has a replay-history stripping mechanism, but the active OpenAI-compatible policy still scopes `dropReasoningFromHistory` to Gemma 4; the reported Qwen/oMLX-style path can still preserve signed provider reasoning blocks across turns. The related attempted fixes are closed and unmerged, so the report remains valid and actionable.

## What This Changes

Prevent Qwen/oMLX `reasoning_content` from being replayed into follow-up OpenAI-compatible conversation history in a way that breaks subsequent turns.

## Best Possible Solution

Add a targeted replay-history policy for affected Qwen/local self-hosted OpenAI-compatible backends that removes or downgrades historical provider reasoning thinking blocks before later user turns, while preserving same-turn tool-call continuations and provider contracts that explicitly require replayed reasoning.

Add targeted replay-history behavior for Qwen-style/local self-hosted OpenAI-compatible backends, including a custom provider id like `omlx` with model id `Qwen3.5-122B-A10B-8bit`, so historical provider reasoning thinking blocks are removed or downgraded before later user turns.
