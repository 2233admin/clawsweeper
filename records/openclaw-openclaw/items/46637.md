## Summary

Keep open. Current main has a generic `dropReasoningFromHistory` mechanism, but it is only selected for Gemma 4 OpenAI-compatible model ids, not for Qwen/oMLX-style `openai-completions` history. The current stream path still records `reasoning_content` as assistant thinking metadata, and same-model replay preserves signed thinking blocks unless a provider replay policy explicitly strips them. The linked attempted fixes for this report are closed and unmerged, so there is not enough evidence to close this as implemented or superseded.

## Best Possible Solution

Keep this open and land a narrow replay-policy fix for affected Qwen/oMLX-style `openai-completions` histories. The fix should strip or downgrade prior assistant `reasoning_content` thinking blocks before follow-up requests for the affected local/self-hosted Qwen path, while preserving provider-owned replay contracts for DeepSeek, Z.AI, Venice, and current same-turn tool continuations. Add focused policy and sanitize-session-history tests plus a transcript-hygiene docs note.

## Clownfish Work Prompt

Fix the openai-completions replay bug where Qwen/oMLX-style `reasoning_content` from a prior assistant turn is replayed into follow-up request history and can make strict local OpenAI-compatible backends reject the payload. Current main captures completion deltas `reasoning_content`/`reasoning`/`reasoning_text` as assistant thinking blocks in `src/agents/openai-transport-stream.ts`, preserves same-model signed thinking in `src/agents/transport-message-transform.ts`, and only enables `dropReasoningFromHistory` for Gemma 4 in `src/agents/transcript-policy.ts`. Add a narrow policy/provider hook for affected Qwen/oMLX or local Qwen OpenAI-compatible replay that strips/downgrades prior assistant reasoning blocks before follow-up requests while preserving the current tool-call continuation exception in `dropReasoningFromHistory`. Add focused coverage in `src/agents/transcript-policy.test.ts` and `src/agents/pi-embedded-runner.sanitize-session-history.test.ts` for an `openai-completions` Qwen/oMLX model with `thinkingSignature: "reasoning_content"` followed by another user turn.
