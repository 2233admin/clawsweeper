## Summary

Keep open. Current main has a replay-history stripping mechanism, but the policy still appears scoped to Gemma 4 OpenAI-compatible replay or provider-owned hooks; the reported Qwen/oMLX-style OpenAI-compatible path can still preserve provider reasoning thinking blocks across turns. The issue is recent, the reporter confirmed the symptom after the original report, and the linked fix PRs are closed unmerged, so this is not safe to close as implemented or stale.

## What This Changes

The issue requests that OpenClaw stop replaying or otherwise sanitize prior assistant `reasoning_content` from Qwen/oMLX OpenAI-compatible conversation history so follow-up turns do not fail JSON parsing.

## Best Possible Solution

Add a targeted replay-history policy for affected local/self-hosted Qwen-style OpenAI-compatible backends that removes or downgrades historical provider reasoning thinking blocks before follow-up requests, while preserving providers and same-turn continuations that intentionally require replayed reasoning.
