## Summary

Current main has useful noise mitigation for dreaming, but the requested semantic-importance/material-density selection layer is still missing. REM reflections remain concept-tag count based, and Dream Diary REM inputs still come from those reflections/candidate truths or the first recent snippets rather than a filter that prefers decisions, preferences, conclusions, or unusual events.

## What This Changes

The issue requests that memory-core dreaming select REM and Dream Diary material by semantic importance instead of repeated word or concept frequency.

## Best Possible Solution

Keep this open and fix it inside memory-core with an internal semantic-importance/material-density filter before REM reflections and Dream Diary REM narrative inputs. Preserve existing transcript hygiene and role/scaffolding tag filters, prefer snippets with decisions, preferences, conclusions, or unusual events over repeated routine chatter, and avoid adding a public selection config unless maintainers choose that surface.
