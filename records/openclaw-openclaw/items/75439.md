## Summary

Keep open: current main still has the heartbeat exec-event cooldown bypass and this PR is on the right implementation seam, but the PR head introduces one delivery regression for blocked task followups and is missing the required user-facing changelog entry.

## What This Changes

The PR adds a heartbeat cooldown helper and tests, routes targeted and broadcast heartbeat wakes through centralized deferral in `src/infra/heartbeat-runner.ts`, and tracks recent per-agent run starts to limit runaway non-interval wakes.

## Best Possible Solution

Land the centralized cooldown approach after preserving both task-delivery wake reasons that need immediate heartbeat delivery, then add the required `CHANGELOG.md` fix entry and let CI gate the exact head.

## Reproduction Assessment

Yes. Current main has a source-level reproduction for the original bug because scoped `exec-event` wakes bypass `nextDueMs`; the PR also supplies a focused scheduler test where five `exec-event` wakes inside a 30-minute cadence should produce one run.

## Solution Assessment

No, not yet. The shared cooldown helper is the right maintainable seam, but the exact immediate-reason policy needs to include the blocked-task followup path or the fix trades one feedback-loop bug for delayed task-followup delivery.
