## Summary

Keep this issue open. Current main still lacks a hard per-session ephemeral/no-persistence contract: session config/state/RPC schemas have no ephemeral flag, session creation and transcript append paths still persist durable files, memory flush still runs through normal persistence, and the plugin/context-engine contract has no isEphemeral or persistence-policy signal. The related #62436 is open but narrower HTTP passthrough work, not a replacement for secure chat across all channels and session layers.

## Best Possible Solution

Keep this issue open for explicit maintainer design. The best path is an additive ephemeral session contract across config, Gateway protocol, SessionEntry/session creation and patch flows; hard no-write gates in transcript, session-store, compaction, context-engine, memory-flush, and plugin persistence paths; a documented plugin SDK/session-context persistence signal; visible status/inspection support; and focused tests proving ephemeral sessions leave no durable transcript, session metadata, memory files, compaction summaries, or searchable index entries. Treat #62436 as a narrower companion for HTTP passthrough, not the canonical replacement.
