## Summary

Keep this PR open. Current main still rejects generic unknown config keys through the strict root schema and the raw/plugin-aware validation path, and gateway startup still treats an invalid config snapshot as fatal unless a separate recovery path succeeds. The PR is not implemented on main, and the related implementation attempts #40198 and #47654 are closed unmerged, so this remains useful maintainer-review work rather than conservative cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase or rewrite it narrowly against current main, then land one implementation that strips or ignores unknown read/startup keys with sanitized warnings while preserving fail-closed behavior for invalid known fields, legacy keys, and union/discriminator mismatches. Review the `.secrets.baseline` delta and run targeted config validation/write/legacy tests plus the required changed gate for the final touched files.
