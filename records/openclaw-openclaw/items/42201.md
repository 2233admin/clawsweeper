## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: unknown config keys remain strict-schema validation failures on the read/startup path, and the gateway still rejects invalid config snapshots instead of continuing after stripping unknown keys. Related #40268 remains open as a bug report, while #47654 was closed unmerged, so there is no shipped or canonical superseding implementation.

## Best Possible Solution

Keep this PR open for maintainer judgment. The best path is to rebase or consolidate it with #40268, then land one implementation that tolerates unknown keys on the read/startup path while preserving fail-closed behavior for invalid known fields, legacy keys, and union/discriminator mismatches. Before merge, review the .secrets.baseline change and run targeted config validation, config write, legacy migration tests, plus the required changed gate for the final touched files.
