## Summary

Keep this PR open. Current main still matches the linked Telegram echoTranscript regression: Telegram preflight marks the audio as already transcribed, but does not carry the raw transcript into the finalized context for the shared echo path, and applyMediaUnderstanding only calls sendTranscriptEcho for newly produced audio transcription outputs. The PR is a plausible implementation candidate for #73615, but it also bundles unrelated Pi compaction changes for #73003 that should be split or removed before maintainer review.

## Best Possible Solution

Keep this PR open as a focused implementation candidate for #73615, but ask the author to split out the unrelated #73003 compaction commits and tighten the media fix/test so Telegram preflight audio echoes once, does not rerun STT, and does not create a generic ctx.Transcript double-echo path for unrelated contexts.
