## Summary

Current main still intentionally preserves explicit user and legacy session `modelOverride` values across `/new`, `/reset`, `sessions.reset`, and gateway restarts. Auto fallback stickiness has been addressed, but the issue’s main UX gap remains: config default changes can still be hidden by a user/legacy session override, `/status` still does not label session override versus config default, and there is no obvious documented bulk clear command.

## Best Possible Solution

Keep this open for a focused maintainer product/code decision. Either make `/new`, `/reset`, and `sessions.reset` clear explicit user and legacy `providerOverride`/`modelOverride`, or preserve pins but add first-class UX: `/status` labels for config default versus session override, a documented `/model clear` or `/model inherit` command, and a bulk clear path for stored session model overrides.
