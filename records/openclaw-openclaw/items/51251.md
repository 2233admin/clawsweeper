## Summary

Current main still preserves explicit user and legacy session model overrides across gateway restarts, `/new`, `/reset`, and `sessions.reset`. Auto-sourced fallback pins are cleared, and there are per-session clear surfaces, but the issue's central UX gap remains: config default changes can still be hidden by a persisted user/legacy override and `/status` still does not label session override versus config default.

## Best Possible Solution

Keep this open for a maintainer product/code decision. Either change `/new`, `/reset`, and `sessions.reset` to clear explicit user and legacy `providerOverride`/`modelOverride`, or preserve pins but make the UX first-class: `/status` source labels for config default versus session override, a documented `/model clear` or `/model inherit` chat command, and a bulk clear path for stored session model overrides.
