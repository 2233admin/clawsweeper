## Summary

Keep open. Current main still matches the reported subagent lifecycle gaps: clean `gateway closed (1000...)` is not classified as recoverable, `agent.wait` `pending` still drops out of `waitForSubagentCompletion` without rearming supervision, and the stale active sweep can still mark a run lost after seeing a non-terminal backing session unless it has `abortedLastRun` set. The related closed #73064 and open #74131 overlap lifecycle hardening, but they do not supersede this narrower three-part report.

## What This Changes

The issue requests subagent lifecycle recovery so clean gateway close, pending wait results, and missing in-memory context with a still-running backing session do not falsely fail or lose runs.

## Best Possible Solution

Implement a focused `src/agents` fix: classify clean gateway-close wait interruptions as recoverable for subagent supervision, rearm `waitForSubagentCompletion` after `pending`, and re-wait or reattach during stale active sweep when the child session store still says `running`, while preserving existing terminal, missing-session, stale, and drifted-session failure semantics.
