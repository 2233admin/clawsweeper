## Summary

Keep open. Current main still matches the reported subagent lifecycle gaps: clean gateway close 1000 is not classified as recoverable, a pending wait result drops supervision instead of rearming, and the stale active-run sweep can still terminally fail an ordinary still-running backing child session. Related #73064 is broader and #74131 is adjacent but does not cover the missing run-wait/run-manager behavior.

## What This Changes

The issue requests subagent lifecycle recovery for clean gateway close, pending agent.wait results, and stale active runs whose backing session is still running.

## Best Possible Solution

Implement a focused src/agents reliability fix that treats clean gateway-close wait interruptions as recoverable for subagent supervision, rearms waitForSubagentCompletion after pending, and re-waits or reattaches when a stale active run still has a running child session, while preserving existing terminal, missing-session, timeout, and aborted-restart semantics.
