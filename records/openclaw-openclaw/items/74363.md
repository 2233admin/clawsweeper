## Summary

Keep open. Current main still matches the reported subagent lifecycle gaps: clean `gateway closed (1000...)` is not treated as recoverable by the wait classifier, `agent.wait` `pending` returns without rearming supervision, and the stale-run sweeper can still mark a run lost even after finding a non-terminal backing child session. Related #73064 and #74131 overlap, but neither is a closing-syntax or complete fix for this narrower report.

## What This Changes

The issue requests subagent lifecycle recovery so clean gateway closes, pending waits, and still-running backing sessions do not falsely terminalize runs as failed or lost.

## Best Possible Solution

Implement a focused `src/agents` fix: classify clean gateway-close wait interruptions as recoverable for subagent wait supervision, rearm `waitForSubagentCompletion` after `pending`, and have stale active sweep re-wait or reattach when the child session store still says `running`, while keeping existing terminal/missing-session failure semantics and adding regression coverage for each branch.
