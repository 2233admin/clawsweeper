## Summary

Keep open. Current main still has the reported subagent lifecycle gaps: clean gateway close `1000` is not classified as recoverable, `pending` waits still stop supervision, and stale active sweep can still mark a run lost while the backing session is merely `running`. #73064 is now closed in favor of this narrower report, and #74131 is an adjacent open PR rather than a complete fix for these branches.

## What This Changes

The issue requests hardened subagent lifecycle recovery for clean gateway close `1000`, `agent.wait` `pending` results, and missing in-memory run context while the backing child session is still running.

## Best Possible Solution

Implement a focused `src/agents` fix that treats clean gateway-close wait interruptions as recoverable for subagent supervision, rearms `waitForSubagentCompletion` after `pending`, and re-waits or reattaches when the child session store still says `running`, while preserving existing terminal, missing-session, stale, and drifted-session failure semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by stubbing `agent.wait` to return `pending` or throw `gateway closed (1000 normal closure)`, and by seeding a stale active subagent run whose child session store entry is still `running`; current branches drop supervision or mark the run failed/lost.

## Solution Assessment

Yes. The narrowest maintainable fix is in the wait classifier, subagent run manager, and stale-run sweep logic, with focused tests; broad gateway readiness, cleanup retry, or product-policy changes should remain separate unless required by the regression tests.
