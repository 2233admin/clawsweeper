## Summary

Keep open. Current `main` still matches the reported subagent lifecycle gaps: clean gateway close `1000` is not recoverable, `pending` wait exits supervision without a follow-up waiter, and a stale active run can still be failed as lost even when the child session entry remains non-terminal.

## What This Changes

The issue requests subagent wait/lifecycle recovery so clean gateway closes, pending waits, and still-running backing sessions do not falsely mark subagent runs failed or lost.

## Best Possible Solution

Implement a focused `src/agents` fix that classifies clean gateway-close wait interruptions as recoverable for subagent supervision, rearms wait supervision after `pending`, and re-waits or reattaches when the child session store still says `running`, while preserving fail-closed behavior for missing, terminal, stale, or drifted session entries.
