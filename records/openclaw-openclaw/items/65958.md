## Summary

Keep open. Current main still has the reported UX gap: the GitHub skill requires `gh` and offers dependency install recipes, but the setup text, docs, gateway-backed install success result, and Skills UI do not clearly tell users to start a new session or run `openclaw gateway restart` when a newly installed GitHub CLI is not visible yet.

## Best Possible Solution

Keep this issue open and fix the narrow skills dependency UX path: add explicit GitHub CLI setup guidance, make successful CLI/bin dependency installs mention starting a new session and using `openclaw gateway restart` when the binary is still missing, update the public skills docs, and cover the behavior with targeted tests. Leave automatic restart as a separate product decision.

## Clownfish Work Prompt

Fix the UX gap from openclaw/openclaw#65958. Current main has a bundled GitHub skill that requires `gh` and can install it, but after install/auth the user is not clearly told to start a new session or run `openclaw gateway restart` when the running Gateway cannot see the new CLI.

Scope:
- Update `skills/github/SKILL.md` setup guidance to mention installing/authenticating GitHub CLI, starting a new session after setup, and using `openclaw gateway restart` if `gh` is still unavailable to OpenClaw after install/auth.

- Update targeted tests for the installer result and UI forwarding path. Do not implement automatic gateway restart in this PR.
