## Summary

Keep open. Current main still implements TUI Ctrl+C as clear/warn/exit only, while Esc and `/abort` are the active-run abort paths. This PR’s intended behavior (Ctrl+C abort priority, stale exit-confirmation reset, abort-pending/stale-run guards, and preserving partial assistant text on aborted runs) is not on main, and related open work #38501/#38502/#38661 represents a competing lower-risk UX direction rather than an implementation of this PR.

## Best Possible Solution

Keep this PR open until maintainers decide the intended TUI Ctrl+C contract. If Ctrl+C should abort active runs, land this PR or an equivalent focused patch in src/tui/tui.ts, src/tui/tui-session-actions.ts, and src/tui/tui-event-handlers.ts with the included regression coverage and update the public TUI docs. If Ctrl+C should remain clear/exit only, prefer the narrower hint path in #38502/#38661 and close or supersede this PR after that decision is explicit.
