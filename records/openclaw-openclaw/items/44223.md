## Summary

Keep open. Current main still treats Ctrl+C in the TUI as clear/warn/exit only, while Esc and /abort are the implemented abort paths; this PR changes that contract to make Ctrl+C abort active runs and also tightens stale/pending abort handling. Because related #38501/#38502 propose a different hint-only contract, this should stay open for maintainer review rather than be closed as implemented or duplicate cleanup.

## What This Changes

The PR changes TUI Ctrl+C handling to abort active runs before draft clearing or exit, track pending/stale abort responses, preserve partial assistant text on aborted runs, and add focused TUI regression tests.

## Best Possible Solution

Decide and document the intended TUI Ctrl+C contract. If Ctrl+C should abort active runs, land this PR or an equivalent focused patch with the regression tests and an update to the public TUI docs; if Ctrl+C should remain clear/exit, land or supersede the narrower active-run hint path in #38501/#38502 and then close this PR as superseded.
