## Summary

Keep open. Current main at 20e21173715ef488850023a90f72585a01cc379f still implements TUI Ctrl+C as clear/warn/exit only, with Esc and `/abort` as the active-run abort paths, so this PR is not obsolete or implemented on main. The remaining question is maintainer product review: choose this PR's broader Ctrl+C-aborts-active-run contract or the narrower active-run hint path tracked by #38501/#38502.

## What This Changes

The PR changes TUI Ctrl+C handling to prioritize aborting active runs, track pending/stale abort responses, preserve partial assistant text on aborted runs, and adds focused TUI regression tests.

## Best Possible Solution

Keep this PR open for maintainer review. The best outcome is to decide the TUI Ctrl+C contract: either land this PR or an equivalent focused patch that makes Ctrl+C abort active runs, update https://docs.openclaw.ai/web/tui, and validate the TUI regressions; or explicitly keep Ctrl+C as clear/exit and resolve the user confusion through the narrower active-run hint path in #38501/#38502, then close or supersede this PR.
