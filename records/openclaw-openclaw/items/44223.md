## Summary

Keep this PR open. Current main still implements Ctrl+C in the TUI as clear/warn/exit only, while Esc and `/abort` are the active-run abort paths. This PR’s intended behavior, prioritizing active-run abort on Ctrl+C, clearing stale exit state, handling pending/stale aborts, and preserving partial assistant text on abort, is not implemented on main. The related #38502/#38501 path is an open alternative UX decision, not a shipped superseding fix. Security review found the PR scope limited to TUI TypeScript and tests, with no dependency, workflow, package, release, secret, or generated/vendor surface changes in the provided diff.

## Best Possible Solution

Keep this PR open until maintainers decide the intended TUI Ctrl+C product behavior. If Ctrl+C should abort active runs, land this PR or an equivalent focused patch in src/tui/tui.ts, src/tui/tui-session-actions.ts, and src/tui/tui-event-handlers.ts with its regression coverage and docs updates. If Ctrl+C should remain clear/exit only, proceed with the #38502/#38501 status-hint path and then close or supersede this PR based on that explicit decision.
