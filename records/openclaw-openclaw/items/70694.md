## Summary

Keep open. The report is recent, specific, and describes an intermittent native `sessions_spawn` failure that is not conclusively fixed or superseded on current `main`. Current source does forward the task into the child run, and the token stats line may partly reflect cache accounting rather than literal delivered prompt size, but there is no targeted regression test, issue-specific fix, or live proof that the reported silent task-loss behavior is gone.

## Best Possible Solution

Keep the issue open and investigate with a focused reproduction or instrumentation across the full native path: `sessions_spawn` tool args -> `spawnSubagentDirect` -> gateway `agent` request -> provider/harness payload -> persisted usage. Add a regression test or live smoke using a long unique task string, and make the completion/diagnostic surface distinguish uncached input from cached prompt tokens clearly. If the investigation proves this was only the already-fixed Codex cached-input accounting issue, close later with that narrower evidence and commit link.
