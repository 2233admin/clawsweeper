## Summary

Keep open. Current main still leaves the reported CLI gap: `openclaw message send --media` supports `--account`, but it does not expose or thread an agent selector, so media-root resolution remains tied to the default agent rather than the workspace for another configured agent/account.

## What This Changes

The issue requests that CLI `message send --media` resolve local media paths against the relevant agent workspace instead of only default media roots/default-agent roots.

## Best Possible Solution

Add an explicit `openclaw message ... --agent <id>` selector, validate it against configured agents, thread it through the CLI helper and `messageCommand` into `runMessageAction`, and use that selected agent workspace for media roots; account-to-agent inference should only be added if unique, deterministic, and documented.
