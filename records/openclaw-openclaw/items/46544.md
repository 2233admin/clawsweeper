## Summary

Keep open. Current main still has the reported CLI gap: `openclaw message send --media` accepts `--account`, but it does not expose or thread an agent selector, while the downstream media path already scopes local roots by whichever agent id it receives. The adjacent media-root PRs in the provided context remain open and do not provide a merged current-main CLI fix.

## What This Changes

Request that CLI `message send --media` use agent-scoped workspace/media roots, or an equivalent explicit allowlist, instead of only the default media roots/default agent context.

## Best Possible Solution

Add a documented `openclaw message ... --agent <id>` selector, validate it against configured agents, thread it through the CLI helper and `messageCommand` into `runMessageAction`, and use that selected agent workspace for media roots. Keep `--account` scoped to channel account selection; only add account-to-agent inference later if it is unique, deterministic, documented, and tested.
