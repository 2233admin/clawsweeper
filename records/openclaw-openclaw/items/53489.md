## Summary

Current main already prevents default thread/topic context bleed through shipped channel routing changes rather than by deleting the generic fork helper. Slack and Discord thread sessions omit `ParentSessionKey` by default, Telegram forum topics use topic-scoped session keys without parent forking, and parent transcript inheritance remains a documented opt-in via `thread.inheritParent`. The PR's one-file change is therefore obsolete and would remove an intentional opt-in behavior now covered by tests and docs.

## Best Possible Solution

Close this PR as already covered by the shipped current-main design: thread/topic sessions are isolated by default, channel-specific starter/history context is injected without copying parent transcripts, and explicit parent transcript inheritance remains available only through documented opt-in configuration.
