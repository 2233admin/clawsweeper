## Summary

Keep this PR open. Current main still does not implement configurable fallback default agent IDs: bootstrap paths, config schema/types, default-agent resolution, session-key helpers, docs, and focused tests still preserve the hardcoded `main` fallback. The linked feature issue #17477 is closed stale, so this PR remains the active implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to centralize a normalized default-agent resolver that covers config, env, bootstrap, sessions, store paths, provider auth/model paths, and CLI runtime paths; then update schema/docs/focused tests and address the env-path sanitization issue before landing.
