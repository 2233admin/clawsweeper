## Summary

Keep this PR open. Current main still does not implement the requested configurable fallback default agent ID: bootstrap paths, config contract/schema, default-agent resolution, session-key helpers, docs, and tests still use `main` as the no-config fallback. The linked feature issue #17477 is closed stale, so this PR remains the active implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for configurable fallback default agent IDs. The best path is to rebase against current main, route config/env/bootstrap/session/store behavior through one normalized default-agent resolver, sanitize env/config IDs before filesystem use, add focused tests and docs for the public contract, and resolve the remaining review concerns before landing.
