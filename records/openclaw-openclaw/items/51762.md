## Summary

Keep this PR open. Current main still does not implement configurable fallback default agent IDs: bootstrap agent-dir resolution, the config type/schema, default-agent resolution, session-key helpers, tests, and docs all continue to fall back to `main`. The linked issue #17477 is closed stale, but this PR remains the active implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for configurable fallback default agent IDs. The best path is to rebase against current main, route config/env/bootstrap/session/store behavior through one normalized default-agent resolver, sanitize env/config IDs before filesystem use, add focused tests and docs for the contract, and remove or justify unrelated test churn before landing.
