## Summary

Keep this PR open. Current main still hard-disables the Android Voice Tab manual mic path when `SpeechRecognizer.isRecognitionAvailable(context)` is false, and there is still no configured OEM recognizer fallback. The PR addresses a real remaining gap with useful Xiaomi validation, but it is not merge-ready because it targets the old Android package path and still needs a no-backend retry safety path.

## Best Possible Solution

Keep this PR open for revision. Rebase it onto `apps/android/app/src/main/java/ai/openclaw/app/voice/MicCaptureManager.kt`, preserve the generic configured-service fallback using the device `voice_recognition_service` setting plus `ComponentName.unflattenFromString` and `SpeechRecognizer.createSpeechRecognizer(context, component)`, and add a no-backend safety path that disables manual mic or avoids retrying when no usable recognizer exists. Add focused Android/Robolectric coverage around the recognizer creation and error handling seam if practical; otherwise retain the Xiaomi manual proof and document why the platform seam is not reliably mockable.
