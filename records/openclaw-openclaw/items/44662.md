## Summary

Keep this PR open. Current main still does not implement the PR’s intended lower-level write-size guard: host pinned writes verify identity/root only, local and remote sandbox writes do not pass an expected byte count, and the regression coverage described by the PR is absent. Related PR #67202 overlaps at the user-facing write-verification layer, but it is open and does not make this lower-level atomic-write hardening obsolete on current main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a carefully reviewed lower-level expected-size guard with regression tests for host pinned writes, local sandbox temp writes, remote sandbox writes, encoded byte counts, and rollback behavior, or explicitly supersede it only after a merged canonical PR covers the same lower-level atomic-write failure mode on main.
