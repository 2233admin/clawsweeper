## Summary

Keep this issue open. Current main still routes session-memory slug generation through the default/effective agent model and the embedded runner's `session:temp:slug-generator` lane, and the open #65134 PR remains the focused implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests moving LLM slug generation off the default agent primary model, or otherwise preventing slow heavy slug-generation calls from congesting queued agent lanes.

## Best Possible Solution

Keep this issue open and review/land #65134 or an equivalent focused fix. The fix should move slug generation off `runEmbeddedPiAgent` and `session:temp:slug-generator`, use `completeSimple` or an approved lightweight slug-model selection contract, preserve `llmSlug: false` and timestamp fallback, and add regression coverage proving the slug path no longer queues on the embedded-runner temp session lane. If maintainers choose a new `slugModel` config surface, update schema/help/docs/tests together.
