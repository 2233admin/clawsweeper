## Summary

Keep open. Current main still lacks the PR's intended server-side WebSocket protocol ping in the Gateway connection handler. Main has an application-level `tick` keepalive, including a recent fix to make tick broadcasts non-droppable, but that is not the same as adding `socket.ping()` frames per connection. The provided PR diff remains a focused implementation candidate, and the security review found no unexplained supply-chain, workflow, dependency, secret, or permission changes in the one-file patch.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best path is to add or accept an equivalent Gateway-owned fix in `src/gateway/server/ws-connection.ts` that sends server-side WebSocket protocol pings per connection, clears the timer on all close paths, and ideally adds a small fake-timer test proving `socket.ping()` and cleanup behavior. Any remaining stop-button or `/stop` failures from #70991 should be tracked separately if they reproduce after the connection stays alive.
