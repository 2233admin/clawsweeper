## Summary

Keep open. Current main does not implement cron `delivery.additionalTargets`: the public cron delivery type/schema/docs still expose a single target, and the isolated run path resolves and dispatches one delivery target. The PR is also not safe to close as obsolete because the provided review discussion identifies remaining functional gaps in the PR itself.

## Best Possible Solution

Keep this PR open or ask for a revised implementation against current main: add a typed cron delivery target array to the public config/schema, normalize it with the existing delivery helpers, preserve it through cron update/patch merging, dispatch secondary targets independently after primary delivery without failing the whole run on one optional target, and cover protocol, normalization, merge, docs, and isolated-run delivery behavior with focused tests.
