## Summary

Keep this PR open. Current main still exposes a single-target cron delivery contract and dispatch path; the PR is still relevant, but the submitted implementation is incomplete because it accepts/resolves secondary targets without actually delivering to them or preserving them through updates.

## Best Possible Solution

Keep the PR open for a revised implementation against current main: add the public CronDeliveryTarget/delivery.additionalTargets contract, normalize entries with the same helpers as the primary route, preserve and patch the field through cron updates, resolve and dispatch each secondary target independently after the primary path, isolate optional target failures, and add protocol, normalization, merge, dispatch, docs, and relevant CLI/UI coverage.

## Risks / Open Questions

- Merging the current PR head would expose a public additionalTargets contract before the delivery path actually sends to those targets.
- Resolving optional secondary targets with Promise.all before dispatch can make one bad secondary route fail an otherwise valid cron run.
- Cron fanout has privacy and audit implications because one output is sent to multiple channels; the final implementation should make each route explicit in validation, docs, and run traces.
- The current PR lacks focused tests/docs/CLI or UI coverage for normalization, update preservation, dispatch behavior, partial failures, and user-facing configuration.
