## Summary

Keep this PR open. Current main still does not implement configurable Feishu `streamingThrottleMs`: the streaming card runtime uses a fixed 160 ms throttle, the dispatcher does not read or pass a config value, and the strict Feishu config schemas, plugin manifest, docs, and tests omit the proposed setting. The linked feature issue #31136 is closed as stale, but this PR remains a plausible focused implementation candidate that needs Feishu-owner review because its 100 ms default no longer matches current main's fixed 160 ms behavior.

## Best Possible Solution

Keep this PR open for Feishu plugin-owner review. A mergeable version should add `streamingThrottleMs` to the shared Feishu config schema and plugin manifest metadata, preserve top-level/account merge semantics, pass the resolved value into `FeishuStreamingSession`, document it on the public Feishu channel page, and add tests for default behavior plus configured throttle plumbing. Maintainers should explicitly choose whether the default follows current main's 160 ms behavior or the PR's older 100 ms compatibility argument.
