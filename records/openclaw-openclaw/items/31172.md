## Summary

Keep this PR open. Current main at 20e21173715ef488850023a90f72585a01cc379f still uses a fixed 160 ms Feishu streaming-card throttle and does not expose `streamingThrottleMs` through the schema, manifest, docs, tests, or dispatcher plumbing. The PR remains a useful focused implementation candidate, but it is incomplete against the current public config surface and its 100 ms default no longer matches current main behavior.

## What This Changes

The PR adds a Feishu `streamingThrottleMs` config field and passes it into `FeishuStreamingSession` so users can tune streaming card update frequency, with a 100 ms default.

## Best Possible Solution

Keep this PR open for Feishu plugin-owner review. A mergeable version should add `streamingThrottleMs` to the shared Feishu config schema and plugin manifest metadata, preserve top-level/account merge semantics, pass the resolved value into `FeishuStreamingSession`, document it on https://docs.openclaw.ai/channels/feishu, and add tests for default behavior plus configured throttle plumbing. Maintainers should decide whether the default follows current main's 160 ms behavior or the PR's older 100 ms compatibility argument.
