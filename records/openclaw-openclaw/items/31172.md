## Summary

Keep this PR open. Current main still uses a fixed 160 ms Feishu streaming-card throttle and does not expose `streamingThrottleMs` through the Feishu schema, manifest, docs, tests, or dispatcher plumbing. The PR is a useful focused implementation candidate, but it is incomplete against the current public config surface and its 100 ms default no longer matches current main behavior.

## What This Changes

The PR adds a Feishu `streamingThrottleMs` config field, passes the configured value into `FeishuStreamingSession`, and changes the streaming-card throttle initialization to a constructor parameter with a 100 ms fallback.

## Best Possible Solution

A mergeable version should add `streamingThrottleMs` to the shared Feishu config schema and plugin manifest metadata, preserve top-level/account merge semantics, pass the resolved value into `FeishuStreamingSession`, document it on https://docs.openclaw.ai/channels/feishu, and add tests for default behavior plus configured throttle plumbing. Feishu maintainers should decide whether the default follows current main's 160 ms behavior or the PR's older 100 ms compatibility argument.
