## Summary

Keep this PR open. Current main still preserves an explicit `--history-limit 0`, passes it into TUI history loading, forwards it to `chat.history`, and the Gateway protocol still requires `limit >= 1`. The PR’s TUI-side no-history guard and regression test are still relevant; the diff is limited to TUI source/test code and does not introduce a security or supply-chain concern.

## Best Possible Solution

Keep this PR open and review/rebase/land the narrow TUI-side guard, or recreate the same behavior on main. The Gateway protocol should continue rejecting `chat.history.limit: 0`; the TUI should treat an explicit `--history-limit 0` as local no-history mode and skip the gateway history request, with the regression test preserved.
