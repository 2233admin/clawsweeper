## Summary

Keep this PR open. Current main still blanket-suppresses final reply payloads when same-target messaging-tool metadata matches, and the current tests still assert suppression even when the final text is not a duplicate. The PR is also paired with the still-open bug #59743 via closing syntax, so it remains an active implementation candidate rather than cleanup.

## Best Possible Solution

Keep this PR open for maintainer review, or supersede it with a focused main-branch fix that removes blanket same-target suppression in both agent-runner and followup delivery paths, keeps content-level text/media dedupe, preserves account-scoped routing fallbacks, and adds regression coverage for same-target media-only sends with distinct final text. After that lands, #59743 can close as implemented.
