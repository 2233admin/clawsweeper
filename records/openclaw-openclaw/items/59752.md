## Summary

Keep this PR open. The affected reply paths on current main still use blanket same-target suppression, and current tests still assert the old suppress-all behavior for non-duplicate final replies. The PR is an active implementation candidate for the linked open bug, with contributor and maintainer follow-up, rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for https://github.com/openclaw/openclaw/issues/59743. The best path is to land or request changes on a narrowed fix that removes blanket same-target reply suppression, keeps content-level text/media dedupe, preserves account-scoped routing behavior, and keeps regression coverage in `agent-runner-payloads`, `followup-delivery`, and `followup-runner` tests. After it merges, the linked bug can close as implemented.
