## Summary

Keep #59125 open. Current main has schema/write-time validation for persisted `approvals.exec.mode`, but the linked fix PR #59187 was closed unmerged and the approval forwarder still has no runtime `normalizeMode` validation or invalid-mode fallback. If an invalid runtime config value reaches the forwarder, target resolution still skips all `session`/`targets`/`both` branches and returns no forwarded approval targets.

## Best Possible Solution

Keep the issue open but narrow the fix: add runtime validation in `src/infra/exec-approval-forwarder.ts` so invalid `approvals.exec.mode` values warn and fall back to `session`, add the missing regression test described by #59187, and ask for or link a concrete subagent approval reproduction if failures remain after the invalid-mode path is covered.
