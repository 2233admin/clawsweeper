## Summary

Keep open. Current main already implements the attachment button/file-picker path and the chat header model picker, and it also exposes skills through slash-command/command-palette surfaces. The requested discoverable skill/capability menu in the WebChat composer, with user-facing capability categories and routing semantics, is still not clearly implemented or superseded by a narrower canonical item.

## What This Changes

Request to enhance WebChat with a composer file upload button, a model selection dropdown, and a skill/capability selection menu.

## Best Possible Solution

Maintain the shipped attachment and model-picker work, then decide whether WebChat should add a first-class composer capability button backed by existing `commands.list`, skill metadata, and effective-tool/session state, or explicitly document that slash commands and the Skills panel are the intended capability surface.
