## Summary

Keep open. Current main covers the attachment button/file picker, drag/drop/paste upload path, non-video attachment serialization, and the chat-header model picker. ## What This Changes

The issue requests WebChat UI controls for file uploads, model switching, and an intuitive skill/capability selection menu.

## Best Possible Solution

Keep the shipped attachment and model-picker work, keep #30759 focused on document-processing behavior, and have maintainers decide whether the remaining WebChat surface should be a small composer button over commands.list/user-invocable skills or a broader category-based model/tool routing design.
