## Summary

Keep open. Current main still does not implement the PR's slash-format Telegram forum-topic cron delivery or the PR's delivery-specific error propagation, and the PR head still has unresolved review findings that need maintainer or contributor follow-up.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, or replacement. The best path is a refreshed fix that adds slash support to Telegram target parsing, pre-parses `:topic:`, slash, and bare `chatId:topicId` forum-topic formats before isolated cron plugin bootstrap, threads config into any plugin-capable fallback deliberately, propagates delivery-specific errors into `lastDeliveryError`, addresses the existing review comments, and keeps regression coverage for final assistant-visible Telegram topic delivery.
