## Summary

Keep this PR open. Current main has Telegram cron announce support for `:topic:` targets and explicit `delivery.threadId`, but it still does not implement this PR’s slash-format parsing, pre-bootstrap extraction for all topic target forms, cfg-backed explicit-target parsing, or delivery-specific error propagation. The PR head also has unresolved review feedback for the bare `chatId:topicId` shorthand path.

## Best Possible Solution

Keep this PR open for maintainer review, revision, or replacement. The best fix should parse all supported Telegram forum-topic target formats before isolated cron plugin bootstrap, add slash support to the Telegram parser, intentionally handle cfg-backed plugin fallback, propagate delivery-specific errors into `lastDeliveryError`, and add regression coverage for slash, `:topic:`, and bare `chatId:topicId` announce delivery paths.
