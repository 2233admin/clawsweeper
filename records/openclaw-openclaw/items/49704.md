## Summary

Keep this PR open. Current main has partial Telegram forum-topic cron support for the explicit `:topic:` form, but it still lacks this PR’s slash-format support and delivery-error propagation, and the PR itself has unresolved review findings that need a focused rebase rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or a focused rebase. The best path is to preserve the useful slash-format parsing and deliveryError propagation, add the missing `chatId:topicId` isolated pre-parse case, keep helpers below imports, and ensure threaded Telegram cron delivery still goes through the final-text/subagent completion path before merge.
