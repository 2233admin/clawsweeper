## Summary

Keep this PR open. Current main still has the explicit `openclaw crestodian` non-TTY path returning after logging the error, and this PR targets the real remaining bug, but it needs the test cleanup noted in review and overlaps with another open fix PR for the same linked bug.

## What This Changes

The PR sets `process.exitCode = 1` in `runCrestodian()` when stdin/stdout are non-TTY and adds two regression tests for the stdin and stdout non-TTY cases.

## Best Possible Solution

Address the test cleanup, then land one canonical narrow fix that makes the explicit Crestodian non-TTY guard exit with code 1, keeps focused stdin/stdout regression coverage, and preserves `--message`, `--json`, and the interactive TUI path.

## Reproduction Assessment

Yes. The high-confidence path is `openclaw crestodian` through `registerCrestodianCommand()` into `runCrestodian()` with non-TTY stdin or stdout; current main logs the error and returns normally while the wrapper only exits non-zero on thrown errors.

## Solution Assessment

Yes after cleanup. Setting `process.exitCode = 1` at the existing Crestodian guard is a narrow maintainable fix that matches the nearby bare-root behavior, but the test cleanup should be addressed and maintainers should choose between this PR and #74059.
