## Summary

Keep open. Current main still lacks the Android chat parity behavior, and this PR is active source work with useful follow-up fixes and validation evidence, but the branch still needs maintainer review because it spans Android UI/runtime plus gateway auth/session behavior and has an actionable authorization bug in the new model/thinking persistence path.

## What This Changes

This PR expands Android chat with richer agent/session controls, new/delete chat flows, timeline/status rendering, model and thinking controls, cleaner session naming, gateway title sanitization, admin-scoped delete support, and related Android/gateway tests.

## Best Possible Solution

Land a maintainer-approved Android chat parity slice that keeps routine operator connections non-admin, routes admin-only session mutations through an explicit elevated path or a narrower gateway API, preserves the cleaner Telegram/thread naming behavior, and ships with focused Android/gateway tests plus the changed gate green.

## Reproduction Assessment

Yes for the reviewer finding: with the PR defaults, Android connects as an operator with read/write/talk.secrets, then selecting a model or persisting thinking calls sessions.patch, which current gateway scopes require operator.admin for. For the broader parity request, reproduction is a current-main comparison rather than a bug repro: main still has the 24-hour cutoff and simpler chat UI.

## Solution Assessment

No, not as-is. The general PR direction targets the right Android parity surfaces, but model/thinking persistence should not call an admin-only RPC through the baseline non-admin session and silently swallow the failure; use the existing elevated session pattern or add a narrower authorized gateway method.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Route session patch through an authorized path:** `apps/android/app/src/main/java/ai/openclaw/app/chat/ChatController.kt:528-545`
  - body: The PR keeps the normal Android operator connection on read/write/talk.secrets scopes, but both thinking and model changes call `sessions.patch` through that normal session. `sessions.patch` is currently an `operator.admin` method, and both calls swallow failures, so model selection and persisted thinking defaults will appear to do nothing for normally paired Android clients. Use the elevated scoped path, or add a narrower non-admin API for these chat settings.
  - confidence: 0.88
