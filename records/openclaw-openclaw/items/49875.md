## Summary

Keep open. Current `main` does not implement the PR's intended broadcast `runInboundClaim` dispatch for regular channel messages; it still only invokes the targeted plugin-bound claim path. The PR also still needs the review issues addressed before it would be safe to merge.

## Best Possible Solution

Keep this PR open as useful but incomplete implementation work. The best path is to update it so regular, non-plugin-bound messages can be claimed by broadcast `runInboundClaim` before agent reply resolution, while preserving plugin-owned binding precedence, skipping generic broadcast after plugin-bound fallback, preserving fast-abort handling ahead of claim hooks, calling `markIdle` on claimed early return, and replacing the old no-broadcast test with regression coverage for both claimed and unclaimed regular messages.
