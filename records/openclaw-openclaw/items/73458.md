## Summary

Keep this PR open. Current main still lacks ACP lineage serialization, and the proposed path appears focused and additive, but it intentionally expands ACP-visible session metadata and should get maintainer review before merge.

## What This Changes

The PR adds an ACP session-lineage metadata normalizer, wires it into ACP session listings and session info snapshots, updates ACP docs/changelog, and adds targeted ACP tests.

## Best Possible Solution

If maintainers accept the ACP data contract, land a narrow bridge update that serializes validated Gateway lineage under `_meta`, documents that graph keys use the Gateway session-key namespace, and explicitly settles the workspace path exposure policy.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence contract check: mocked Gateway rows with lineage currently lose that data in ACP list results and session info snapshots, while the PR adds targeted serialization tests for those paths.

## Solution Assessment

Unclear pending maintainer data-scope approval. The serializer is the narrowest maintainable code path and ACP SDK 0.21.0 permits `_meta`, but raw `spawnedWorkspaceDir` exposure may need narrowing before merge.
