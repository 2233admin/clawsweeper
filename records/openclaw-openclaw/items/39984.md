## Summary

Keep this PR open. The requested runtime guidance is still missing on current main, but the PR needs a refresh because web_search missing-key payloads are now provider-owned rather than in the old core wrapper path the PR edits.

## Best Possible Solution

Keep the PR open and ask for a refresh against current main: update the provider-owned web-search runtime missing-key payloads for Brave, Perplexity, Grok/xAI, Gemini, and Kimi; preserve the real environment variable names and current plugin config paths; drop unrelated baseline/generated churn; and add focused coverage for the updated messages where practical.
