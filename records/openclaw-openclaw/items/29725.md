## Summary

Keep open. Current main still has adjacent Discord presence support through static config, auto-presence, and the gated `setPresence` message action, but it does not expose the requested code-plugin helper `api.runtime.channel.discord.updatePresence()`. The remaining question is a public plugin-runtime API decision, not conservative cleanup.

## Best Possible Solution

Keep this open for Discord/plugin-runtime owner review. The best path is either a narrow Discord-owned, documented `updatePresence` helper that reuses the existing gateway registry and presence validation path, or an explicit docs/API decision that the existing gated `setPresence` message action is the supported code-plugin equivalent. The implementation should stay in the Discord plugin/runtime surface unless maintainers decide a generic plugin-runtime capability is needed.
