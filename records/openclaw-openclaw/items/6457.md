## Summary

Keep this PR open. Current main partially absorbed the auth/diagnostic topic-threading part, but it still does not register or clear Telegram native commands for the `all_group_chats` scope, and the plugin-command `Command not found.` fallback still sends without topic thread params. The PR also targets the old `src/telegram` layout, while current main has this code under `extensions/telegram`, so it needs a rebase or replacement implementation rather than cleanup closure.

## Best Possible Solution

Keep this PR open for a focused rebase or replacement in `extensions/telegram`: register and clear Telegram commands for both default and `all_group_chats` scopes, thread the plugin-command `Command not found.` fallback through the existing Telegram thread helpers, and add regression tests for dual registration, dual cleanup, cache/retry behavior, and General-topic id 1 handling.
