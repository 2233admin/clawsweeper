## Summary

Keep open. Current main has already absorbed the auth-reply thread-param part of the idea, but the two central remaining behaviors are still missing: Telegram menu registration/cleanup only uses the default Bot API command scope, and the plugin-command `Command not found.` fallback still sends without thread params. The PR is useful source work, but it targets the old `src/telegram` layout and needs a replacement/rebase against `extensions/telegram`, matching the maintainer review already posted on the PR.

## What This Changes

The PR branch modifies the legacy `src/telegram` native command handler and test to register and clear Telegram commands in both default and `all_group_chats` scopes, and to preserve Telegram topic thread params for native auth and command-not-found replies.

## Best Possible Solution

Keep this PR open as useful source material, but land the fix in the current Telegram plugin: make command-menu sync register and clear both default and `all_group_chats` scopes, preserve thread params for the plugin-command not-found fallback through the existing `TelegramThreadSpec` helpers, and cover the behavior with focused extension tests plus a user-facing changelog entry.
