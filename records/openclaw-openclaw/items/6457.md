## Summary

Keep this PR open. Current main partially implements the topic-thread/auth reply side, but it still lacks the PR's central Telegram group-scope command registration and cleanup, and the plugin-command "Command not found." fallback is still sent without thread params. The PR itself is stale because it targets the old `src/telegram` layout while current main uses `extensions/telegram`.

## Best Possible Solution

Keep this PR open and rebase or replace it with a focused `extensions/telegram` patch: register and clear both default and `all_group_chats` command scopes, preserve topic params for the plugin-command not-found fallback using `resolveTelegramThreadSpec` and `buildTelegramThreadParams`, and add regression tests for dual registration, dual cleanup, retry/cache failure behavior, and General-topic id 1 handling.
