## Summary

Keep this PR open. Current main has partial Telegram topic-thread handling for native-command auth replies, but it still does not implement the central group-scope command registration/cleanup and still has an unthreaded plugin-command not-found fallback. The PR is useful but stale against the current `extensions/telegram` layout.

## Best Possible Solution

Keep this PR open and rebase or replace it with a focused `extensions/telegram` patch: register and clear both default and `all_group_chats` command scopes, preserve topic params for the plugin-command not-found fallback via `resolveTelegramThreadSpec` and `buildTelegramThreadParams`, and add regression tests for dual registration, dual cleanup, retry/cache failure behavior, and General-topic id 1 handling.
