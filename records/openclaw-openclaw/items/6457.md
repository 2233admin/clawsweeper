## Summary

Current main only partially supersedes this PR. The provided maintainer review also explicitly kept this open for the same remaining gaps. Security pass found no supply-chain/workflow/dependency changes in the PR context, but the rebase should preserve existing auth gates while broadening only Telegram command-menu visibility.

## Best Possible Solution
