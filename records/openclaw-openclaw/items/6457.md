## Summary

Keep this PR open. Current main has absorbed part of the thread-auth behavior, but the central Telegram group command-scope work is still missing: command menu sync still registers/clears only the default scope, explicit native-command disable clears only default commands, and the plugin-command not-found fallback still sends without topic thread params. The PR itself is stale against the old `src/telegram` layout, so the useful path is a focused rebase or replacement under `extensions/telegram`.

## Best Possible Solution

Keep the PR open for a rebase or replacement under `extensions/telegram`: add default plus `all_group_chats` registration and cleanup in the current command-menu/native-disable paths, thread the plugin-command not-found fallback through `resolveTelegramThreadSpec` and `buildTelegramThreadParams`, and add regression tests for dual registration, dual cleanup, retry/cache behavior, and General-topic id 1 handling.
