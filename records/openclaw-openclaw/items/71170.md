## Summary

Keep this PR open. Current main still lacks the proposed per-agent/defaults reminder-guard opt-out, and the current reminder guard still only recognizes built-in cron success or an existing same-session cron job, so the plugin-managed scheduler false-positive remains unresolved on main.

## What This Changes

The PR adds optional `agents.defaults.reminderGuard` and per-agent `reminderGuard` config, threads it through agent config resolution, uses it to bypass the unscheduled-reminder note, updates generated config help/baselines, adds tests, and adds a changelog entry.

## Best Possible Solution

Keep this PR open for maintainer review. The desired end state is a default-on, documented opt-out for agents using external scheduling, with schema/docs/tests/changelog aligned; if maintainers prefer direct plugin scheduling signals, that should be a separate SDK contract proposal rather than a blocker for this narrow config fix.
