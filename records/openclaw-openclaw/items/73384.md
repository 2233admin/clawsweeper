## Summary

Keep open. Current main still lacks the requested workflow seams, so this PR remains the active implementation candidate, but the inspected head has a blocking authorization regression: read-scoped plugin session actions reject write-scoped clients even though existing Gateway authorization treats `operator.write` as read-compatible.

## What This Changes

The PR adds plugin SDK workflow APIs, Gateway protocol support, host-mediated session actions, attachments, scheduled turns, finalize retry, advanced workflow fixtures, and QA stabilization.

## Best Possible Solution

Keep this as the canonical implementation PR, fix the action-level scope compatibility by reusing or mirroring the existing Gateway read/write scope semantics, add a regression test for `operator.write` invoking a read-scoped session action, then rerun exact-head SDK/Gateway/protocol validation before merge.

## Reproduction Assessment

Yes for the PR regression. Register a session action with `requiredScopes: [READ_SCOPE]` and call `plugins.sessionAction` as a client whose scopes are `[WRITE_SCOPE]`; static inspection shows the method-level pre-auth allows the call but the handler rejects it at the exact-match action check.

## Solution Assessment

No, the current patch is not the best mergeable version as-is. The maintainable fix is a small shared action-scope helper or direct reuse of the existing Gateway rule that treats write as satisfying read, plus coverage in the existing host-hooks contract tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Honor write scope for read-only actions:** `src/gateway/server-methods/plugin-host-hooks.ts:102`
  - body: The new per-action check only accepts exact required scopes, so a client connected with `operator.write` is rejected for an action that declares `requiredScopes: ["operator.read"]`. The rest of Gateway auth treats write as read-compatible, so read-only plugin actions will fail for write-capable clients until this check mirrors the existing scope semantics.
  - confidence: 0.95
