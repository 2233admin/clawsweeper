## Summary

Keep this PR open for maintainer review. Current main still does not expose or run a `resolve_exec_env` hook, and the proposed hook remains a security-sensitive public SDK seam because plugin-returned environment values would be merged into exec after the existing host/sandbox env construction path.

## What This Changes

This PR adds a typed `resolve_exec_env` plugin hook, runs it from the exec tool after environment construction, merges plugin-returned env vars, and adds focused hook tests.

## Best Possible Solution

Keep the PR open until maintainers decide whether this should be an exec-env hook at all. If accepted, land it as a narrow typed plugin SDK contract with documented host/sandbox/node behavior, a reserved-key or trusted-plugin policy, channel/account/thread context where needed, public docs/API coverage, and focused security/error-isolation tests; otherwise redirect the design to a narrower exec/tool policy hook.
