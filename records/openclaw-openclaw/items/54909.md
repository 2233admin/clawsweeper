## Summary

Keep open because current main still forwards generic Telegram inline-button callbacks as raw payload text, which conflicts with the documented `callback_data: <value>` contract. The linked PR #54962 remains open and unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to route Telegram inline-button callback data to the agent in the documented prefixed form so agent instructions can trigger the configured HTTP tool instead of fabricating a confirmation.

## Best Possible Solution

Review and land PR #54962, or apply the same narrow change in `extensions/telegram/src/bot-handlers.runtime.ts` so only generic callback fallback data is forwarded as `callback_data: ${data}`. Add a regression test for generic callback prefixing and keep the internal callback handlers unchanged.
