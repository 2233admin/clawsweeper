## Summary

Keep this PR open, but it should not merge as-is. The underlying symlink-warning problem is real and current main has not implemented this cross-root allowance, but the proposed trust list includes bundled skills without preserving bundled source semantics, which can bypass `skills.allowBundled`.

## What This Changes

The PR adds cross-configured-root realpath allowances for symlinked skills and regression tests for workspace-to-project-agent and managed-to-extra-dir symlinks.

## Best Possible Solution

Implement one source-preserving symlink policy for configured skill roots: allow cross-root symlinks among explicitly trusted non-bundled roots, keep external escapes rejected, and either exclude bundled targets from non-bundled aliases or relabel them so `skills.allowBundled` still applies. Cover cross-root success, external rejection, bundled allowlist enforcement, and the current grouped-skill loader shape in tests and docs.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a bundled skill excluded by `skills.allowBundled`, plus a workspace or managed symlink pointing at that bundled skill; this PR would accept the realpath while the local loader keeps the source as non-bundled.

## Solution Assessment

No. Allowing realpaths into other configured roots may be a reasonable direction for non-bundled roots, but including `bundledSkillsDir` without preserving bundled source semantics is not the narrowest maintainable fix.
