## Summary

Current main still rejects skill symlinks whose realpath leaves the currently scanned root, so this PR is not obsolete or implemented on main. The PR remains potentially useful for the configured-root symlink gap, but it should not be closed or landed as-is because the provided review discussion identifies an unresolved bundled-skill policy bypass risk around `skills.allowBundled`.

## Best Possible Solution

Keep this PR open for maintainer review, but require a revised implementation. The best path is one consolidated skills symlink fix, coordinated with #59688 and #44051, that allows explicitly trusted non-bundled configured-root symlink targets while preserving the target root's source policy. Add regression coverage for cross-root symlinks, unknown external symlinks still warning/skipping, and the `skills.allowBundled` bypass case before merge.

## Risks / Open Questions

- The PR changes a security-sensitive loader boundary and needs a revised design that either excludes bundled roots from non-bundled symlink targets or preserves/reapplies the target root's source policy before `skills.allowBundled` filtering.
- The submitted tests in the provided diff appear to look for the older warning text, while current main emits `Skipping escaped skill path outside its configured root`, so the PR likely needs a rebase/test refresh.
- Related PR #59688 and issue #44051 overlap the managed-symlink warning problem but are still open, so there is no merged canonical fix to close this PR against.
