## Summary

Keep open. Current main still creates the MS Teams personal-chat streaminfo/informative-status path from conversation type alone, the strict config schema exposes no server-side DM opt-out, and the related Microsoft tracker lists this as remaining work rather than an implementation.

## What This Changes

Add an MS Teams config option to disable Copilot-style informative stream status in personal DMs and use native typing plus normal final-message delivery instead.

## Best Possible Solution

Add a default-preserving MS Teams plugin config option for personal-chat stream status, wire it through the dispatcher/stream controller so the stream is not created and onPartialReply is not installed when disabled, keep typingIndicator semantics for native typing, and document the fallback behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: create a personal MS Teams dispatcher with typingIndicator=false and current main still sends the informative update; the existing test asserts that behavior.

## Solution Assessment

Yes. A plugin-owned config option is the narrowest maintainable solution because the behavior is MS Teams-specific and the affected surfaces are already local to the bundled plugin config, dispatcher, docs, and tests.
