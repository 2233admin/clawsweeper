## Summary

Keep open. Current main still creates the MS Teams personal-chat streaminfo/informative status path unconditionally and the strict MS Teams config schema has no accepted server-side option to disable it for DMs; the open Microsoft tracker also lists this as remaining work.

## What This Changes

Add an MS Teams plugin config option to disable Copilot-style informative stream status in 1:1 DMs and fall back to native typing plus final-message delivery.

## Best Possible Solution

Add a default-on MS Teams plugin setting that disables the personal-DM streaminfo/informative status transport when requested, preserves current behavior for existing installs, and documents the native typing plus final-message fallback path.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: create a personal MS Teams dispatcher with `typingIndicator: false`, call reply start, and current tests still expect `sendInformativeUpdate` while only suppressing native typing keepalive.

## Solution Assessment

Yes. A plugin-owned, default-preserving opt-out is the narrowest maintainable solution because the behavior is specific to the bundled MS Teams plugin and does not require a core channel policy change.
