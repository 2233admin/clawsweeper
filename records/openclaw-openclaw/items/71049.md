## Repair Work Prompt

Implement a default-preserving MS Teams plugin option to disable the personal-DM streaminfo/informative status UI. The observable bug is that DMs always create `TeamsHttpStream` and send an informative update even when `typingIndicator: false`; users need a server-side config key that keeps the default current behavior but can switch DMs to native typing plus one final message. Keep the change owned by the MS Teams plugin/config surface, not core channel policy. Likely implementation: add a typed config field in `src/config/types.msteams.ts` and `MSTeamsConfigSchema`, plumb it through `createMSTeamsReplyDispatcher` into `createTeamsReplyStreamController`, skip creating/installing the stream controller when disabled for personal chats, and ensure `typingIndicator` still controls native typing. Add/adjust tests for default personal-chat informative status, disabled personal-chat no `sendInformativeUpdate`/no `onPartialReply`, native typing fallback, schema acceptance/rejection, and block streaming interaction. Update `docs/channels/msteams.md` and `CHANGELOG.md` with user-facing wording and attribution. Do not change non-personal Teams behavior, core streaming policy, or unrelated channel schemas.
