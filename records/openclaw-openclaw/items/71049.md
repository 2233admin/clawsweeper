## Summary

Current main still hardcodes the MS Teams personal-chat streaminfo path and does not expose a config option to disable the Copilot-style informative stream status in DMs. `typingIndicator` only controls the native typing keepalive, `blockStreaming` only controls block delivery, and tests currently assert that `typingIndicator: false` still sends the informative update. The issue is valid and narrow enough for a focused MS Teams plugin fix.

## Best Possible Solution

Keep this issue open and implement a narrow MS Teams plugin-owned config option, defaulting to current behavior, that lets personal chats disable the informative streaminfo status and use native typing plus normal final delivery. The fix should stay inside the MS Teams plugin/config surface, avoid core owner-specific policy, and preserve the long-tool-chain keepalive behavior.
