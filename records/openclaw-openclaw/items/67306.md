## Summary

Keep this issue open. Current main improved `infer model run` so it is now a lean raw model probe, but it still lacks the requested JSON Schema CLI contract, schema validation, repair/retry semantics, and structured schema-failure reporting on the core inference surfaces.

## What This Changes

The issue requests first-class JSON Schema input support, output validation, and defined invalid-output handling for OpenClaw inference CLI surfaces such as `openclaw infer model run` and `openclaw agent`.

## Best Possible Solution

Implement a narrow provider-neutral JSON Schema contract on `openclaw infer model run` first: accept a schema from a file or inline JSON, validate final JSON output with shared structured errors, define bounded repair or no-repair semantics clearly, keep local/gateway behavior consistent, and update docs/tests. Broader `openclaw agent` exposure should follow only after the core contract is stable.

## Repair Work Prompt

Implement first-class JSON Schema support for `openclaw infer model run` for openclaw/openclaw#67306. Scope the first PR narrowly: add a provider-neutral `--json-schema <path-or-inline-json>` option on `infer model run`, parse file paths or inline JSON safely, require/parse final model text as JSON when a schema is supplied, validate it with the repo's existing Ajv dependency or a small shared validator helper, and return machine-readable validation/parse failures in `--json` mode.
