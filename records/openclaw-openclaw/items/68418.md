## Summary

Keep this PR open. Current main still sends any truthy `choice.delta.content` directly through the text delta path, and there is no typed-block `delta.content` unpacker or regression coverage for the Mistral reasoning array shape. The linked bug remains open, while related PRs #67292 and #68855 are closed and unmerged, so this PR is still a useful implementation candidate rather than obsolete.

## What This Changes

The PR branch changes the OpenAI completions stream parser to unpack ordered typed-block `delta.content` arrays into text/thinking deltas, adds regression tests for Mistral-style array content and fall-through behavior, and adds a changelog entry.

## Best Possible Solution

Land a rebased version of this fix, or an equivalent maintainer patch, in `src/agents/openai-transport-stream.ts`: preserve the string fast path, unpack ordered `text` and `thinking` typed blocks, fall through for unsupported array content so reasoning/tool-call fields still process, and keep focused regression coverage plus a policy-compliant changelog entry.
