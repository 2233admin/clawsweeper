## Summary

Keep open. Current main still leaves the dedicated Control UI Logs tab on a raw `logs.tail` display path: the parser preserves ANSI-bearing message text and the view renders that message directly. The Overview gateway log tail has a local strip helper, but the related fix PR #64431 was closed unmerged and #45001 was closed as a duplicate pointing back to this remaining issue.

## What This Changes

The issue asks for Control UI gateway log output to stop showing raw ANSI escape sequences and instead strip or render them for readable display.

## Best Possible Solution

Land a focused Control UI logging fix that applies one ANSI display strategy to every `logs.tail` UI consumer, normalizes displayed message/subsystem/fallback text, leaves raw export behavior intentional, and adds regression coverage for ANSI-bearing structured and plain log lines.
