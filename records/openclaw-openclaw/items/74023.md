## Summary

Keep open. Current main still does not wire `openclaw tasks maintenance` to prune stale cron run session registry rows, and this PR is the active implementation path for #73867. The patch looks focused and I found no blocking correctness findings, but it still needs normal maintainer review and changed-gate validation before merge.

## What This Changes

The PR adds a `tasks maintenance` session-registry sweep that prunes stale cron run session rows, preserves currently running cron jobs and non-cron sessions, reports JSON/text counts, adds command coverage, and documents the behavior.

## Best Possible Solution

Ship a focused `tasks maintenance` implementation that uses existing session-store discovery and pruning primitives, preserves non-cron rows and rows for cron jobs still marked running in durable state, reports stable preview/apply counts, and documents the command behavior.

## Reproduction Assessment

Yes. The high-confidence reproduction is to seed `sessions.json` with stale/recent cron run rows and a non-cron row, seed `cron/jobs-state.json` with one running job, then run `tasksMaintenanceCommand` in apply mode; the PR adds that focused regression test.

## Solution Assessment

Yes, pending validation. The PR is the narrowest maintainable direction because it reuses current session discovery, cron store loading, and locked session-store mutation rather than changing scheduler or run execution behavior.
