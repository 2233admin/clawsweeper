## Summary

Keep open. Current main still trims oversized bootstrap files by deterministic prefix/suffix slices, so the central bootstrap relevance request remains unresolved. The related compaction and observation-extraction parts are partly covered by existing plugin/context-engine seams, but this issue bundles several architecture decisions that need maintainer scoping before implementation.

## What This Changes

The issue requests relevance-aware bootstrap file selection, incremental compaction architecture, and plugin-style observation extraction from existing LLM responses.

## Best Possible Solution

Split this into narrower trackers: design relevance- or structure-aware bootstrap selection in the bootstrap context builder, keep compaction changes aligned with the existing compaction-provider/context-engine contracts, and leave observation extraction plugin-owned unless a specific missing SDK hook is identified.
