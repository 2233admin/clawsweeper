## Summary

Keep open. Current main still uses fixed positional bootstrap head/tail slicing for oversized workspace files, and the active run path, tests, and docs continue to protect/report truncation rather than selecting relevant or structured middle content. Incremental compaction and observation extraction now have plausible plugin/context-engine paths, but those do not close the core bootstrap relevance request; the prior April 26, 2026 maintainer/Codex review remains consistent with current main.

## Best Possible Solution

Keep this issue open, or split it into narrower trackers: bootstrap relevance should remain an OpenClaw core issue around buildBootstrapContextFiles and the embedded prompt injection path; incremental compaction should follow the context-engine/Lossless Claw/#58398/#58455 path; observation extraction should remain plugin/hook-driven unless a concrete missing SDK API is identified.
