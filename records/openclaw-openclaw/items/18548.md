Keep open. Current main still uses fixed prefix/suffix bootstrap truncation and the embedded run path still injects that output; the compaction and observation parts have plugin/context-engine surfaces and related open trackers, but they do not resolve the core bootstrap relevance request.

Request relevance-aware bootstrap file extraction, more incremental compaction, and observation extraction from existing model outputs.
