## Summary

Keep open. Current main still trims large bootstrap files by fixed prefix/suffix selection, so the central bootstrap relevance request is not implemented. Compaction and observation-extraction have useful plugin surfaces now, but the issue bundles three separate architecture topics and needs maintainer scoping before implementation.

## What This Changes

Request relevance-aware bootstrap file selection, incremental or safer compaction behavior, and lower-overhead observation extraction from existing LLM responses.

## Best Possible Solution

Split the work into narrower trackers: implement relevance- or structure-aware bootstrap selection around buildBootstrapContextFiles and embedded bootstrap injection, align any incremental compaction changes with the existing context-engine and compaction-provider contracts, and keep observation extraction plugin-driven unless a concrete missing SDK/API seam is identified.
