## Summary

Keep open. Current main has TUI streaming/history mitigations, but it still does not implement the reported TUI viewport behavior: no scroll anchor, no auto-follow-only-when-at-bottom policy, no jump-to-latest affordance, and no TUI auto-scroll setting. The related #7648 is a Control UI/WebChat request, and the older #11269 was closed without covering this remaining TUI-specific UX gap.

## Best Possible Solution

Keep this TUI-specific issue open and fix it in `src/tui` with explicit chat viewport state: preserve the user's reading position, auto-follow only when already near the bottom, provide a clear jump-to-latest affordance/shortcut, and consider a TUI setting for follow behavior. Add focused regression coverage for streaming deltas, final-event history reloads, and user-scrolled state. Keep #7648 scoped to Control UI/WebChat unless maintainers intentionally create a shared cross-surface tracking issue.
