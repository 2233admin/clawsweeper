## Summary

Keep open. Current main still does not implement a scroll-aware TUI chat viewport: ChatLog appends/prunes components, the TUI mounts it directly in the root layout, history reloads rebuild the log, and settings/docs still have no TUI auto-scroll, paging, or jump-to-latest behavior. Related PR #51128 is open and appears to be the closest implementation candidate, so the issue should remain open until that PR, or an equivalent rebased fix, lands.

## Best Possible Solution

Keep this issue open and route the fix through the TUI code path. The best near-term path is to review/rebase/land #51128 if it still satisfies the desired behavior on current main: preserve a user-scrolled transcript, auto-follow only when already at latest, support clear paging or jump-to-latest behavior, and preserve the newer pending-user/history/streaming fixes. If #51128 is superseded, implement the equivalent in src/tui/components/chat-log.ts, src/tui/tui.ts, src/tui/components/custom-editor.ts, tests, docs/web/tui.md, and the changelog without changing Control UI/WebChat scope from #7648.
