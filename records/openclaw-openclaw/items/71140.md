## Summary

Keep this issue open. Current main still lacks the reaction directive parser/normalization and Telegram execution path, while the media/GIF side is already mostly implemented; the same reporter also has draft PR #71141 open as the linked implementation path.

## What This Changes

Telegram outbound replies should consume `[[react:...]]` / `[[react_to_current:...]]` and `MEDIA:` directives into native Telegram reactions/media instead of leaking or dropping directive text.

## Best Possible Solution

Review and land PR #71141 if it is correct, or replace it with a maintainer-owned patch that extends shared directive parsing, carries structured `channelData.telegram.reaction`, permits reaction-only Telegram replies with a valid target, applies reactions through the existing Telegram primitive, and adds normalization, delivery, and adapter coverage.
