## Summary

Keep #57825 open. Current main still has the core behavior this PR tries to change: generic session persistence deletes every configured cleared field when that field is absent from the incoming entry, and the agent-command wrapper still passes model/provider override fields into that deletion path unconditionally. The PR patch is stale against current file layout, but the underlying fix is still useful and should be adapted rather than closed.

## Best Possible Solution

Keep the PR open and adapt it to current main: make generic persistSessionEntry preserve omitted cleared fields by default, add an explicit opt-in for reset/delete paths that truly need deletion, and add focused regression tests for both preserving providerOverride/modelOverride on partial writes and explicitly clearing them on reset.
