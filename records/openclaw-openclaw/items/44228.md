## Summary

Keep this PR open. Current main still does not implement the PR’s central Windows reply-media dedupe behavior in the shared dedupe helper, and current regression coverage remains POSIX-only. The PR is useful but needs a refresh: the implementation moved to the split dedupe file, review already identified missing drive-letter case folding, and the branch includes unrelated config/protocol changes.

## Best Possible Solution

Refresh this PR against current main, drop the unrelated push/config protocol changes, implement Windows drive-letter normalization in `src/auto-reply/reply/reply-payloads-dedupe.ts`, fold drive-letter case consistently, and add regression tests for `file:///C:/...`, `C:/...`, `C:\...`, and mixed drive-letter casing.
