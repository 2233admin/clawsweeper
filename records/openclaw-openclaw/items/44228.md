## Summary

Keep this PR open. Current main still does not normalize Windows drive-letter local paths and `file:///C:/...` URLs into the same reply-media dedupe key. The PR targets a real bug, but it needs a refresh for the current split helper, drive-letter case folding, and removal of unrelated push/config protocol changes noted in review.

## Best Possible Solution

Refresh this PR against current main, drop the unrelated push/config protocol changes, implement Windows drive-letter normalization in `src/auto-reply/reply/reply-payloads-dedupe.ts`, fold drive-letter case consistently, and add regression tests for `file:///C:/...`, `file:///c:/...`, `C:/...`, and `C:\...` forms. The fix should stay limited to reply-media dedupe unless maintainers intentionally split the push/config updates into a separate PR.
