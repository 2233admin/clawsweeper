## Summary

Keep open. Current main still does not implement the requested value-first first-run onboarding: QuickStart/default setup still configures auth, workspace/bootstrap, gateway, channels, search, and skills before the first runnable chat/task moment, and macOS onboarding still starts the bootstrap/persona chat after setup pages. Because this is a cross-surface activation UX change rather than a narrow defect, it should stay with maintainer product triage.

## What This Changes

Request a first-run onboarding flow that shows runnable templates or example tasks before optional agent name, reply style, symbols/tone, persona, and other customization steps.

## Best Possible Solution

Design a first-run path that performs only the minimum safe model/session setup, presents a small set of runnable templates or example tasks, lets the user complete one useful task, and then offers optional agent identity, reply style, persona, channel, search, skills, and other customization steps across CLI, macOS, Control UI, docs, and focused onboarding tests.
