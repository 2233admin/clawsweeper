## Summary

Keep open. Current main still does setup, auth, workspace/bootstrap, channels, search/skills, and identity-oriented bootstrapping before the first ordinary useful task, while the chat-first Crestodian path remains an opt-in preview rather than the default onboarding contract.

## What This Changes

Request to redesign first-run onboarding so users can run a useful starter task before optional personalization such as agent name, reply style, symbols, or persona preferences.

## Best Possible Solution

Define a maintainer-approved first-run contract that keeps only mandatory model/session/security prerequisites before the first result, offers a small set of useful starter tasks, runs one immediately, and moves identity, persona, style, channel, search, skills, and other customization into optional follow-up steps across CLI, macOS, Control UI, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current main still sequences setup and bootstrapping before the first ordinary useful task.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer UX and security judgment about required prerequisites and whether classic CLI onboarding, macOS onboarding, Control UI, and Crestodian should converge or keep separate flows.
