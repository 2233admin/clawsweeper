## Summary

Current main still does not provide the requested value-first first-run onboarding. QuickStart and Control UI welcome suggestions reduce friction, but the primary CLI and macOS onboarding paths still complete setup/bootstrap/persona work before offering a runnable example-task/template experience.

## Best Possible Solution

Keep this open for maintainer UX/product triage. The best implementation would add a first-run path that performs only the minimum required model/session setup, presents a small set of bootstrap templates or example tasks, lets the user run one useful task immediately, and then offers optional agent name, reply style, symbols/avatar, persona, channel, search, skill, and other customization steps.
