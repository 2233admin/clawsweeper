## Summary

Keep open. Current main still has a setup-first first-run experience across the classic CLI and macOS flows, while Crestodian only partially overlaps as an opt-in conversational setup preview. The requested value-first sequence needs maintainer UX/product direction before it is safe to hand to an autonomous repair lane.

## What This Changes

The issue requests reordering first-run onboarding so users can see templates or run a useful task before optional agent identity, reply style, symbols, tone, or persona customization.

## Best Possible Solution

Define a maintainer-approved first-run contract that performs only the minimum required model/session setup, presents a small set of useful example tasks or templates, runs one task immediately, and moves identity, persona, channel, style, search, skill, and other customization into optional follow-up steps with aligned CLI, macOS, Control UI, docs, and tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows the current CLI and macOS first-run paths still put setup and bootstrap/persona work before the first useful task/chat outcome.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer UX judgment about required auth/setup prerequisites and whether Crestodian, classic CLI, macOS, and Control UI should share a new flow or expose surface-specific variants.
