## Summary

Keep open. Current main still has a setup-first first-run experience in the classic CLI and macOS flows, with only partial overlap from Crestodian as a modern preview; this needs maintainer UX/product direction rather than conservative cleanup closure.

## What This Changes

The issue requests a value-first onboarding flow that lets first-time users run or see a useful task before optional personalization such as agent name, reply style, symbols, tone, or persona preferences.

## Best Possible Solution

Define a maintainer-approved first-run contract that performs only the minimum required model/session and security setup, presents a small set of useful example tasks or templates, runs one task or first chat immediately, and moves identity, persona, channel, style, search, skill, and other customization into optional follow-up steps with aligned CLI, macOS, Control UI, docs, and tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current main still routes users through setup/bootstrap/persona work before the first useful task or chat result.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer UX judgment about required prerequisites and whether Crestodian, classic CLI, macOS onboarding, and Control UI should share a new flow or expose surface-specific variants.
