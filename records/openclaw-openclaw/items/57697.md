## Summary

Keep PR #57697 open. Current main still has the target-side announce routing behavior this PR is trying to fix: `runSessionsSendA2AFlow()` resolves the final announce target from `targetSessionKey`/`displayKey`, and the existing sessions_send test still expects the final announce send to go to `group:target` rather than the requester target. The supplied PR discussion is useful evidence too: reviewers identified channel-metadata and fallback concerns, and the author pushed follow-up fixes for those concerns, so this remains plausible focused work rather than an obsolete cleanup PR.

## Best Possible Solution

Keep PR #57697 open. The best path is a targeted sessions_send A2A fix on current main that resolves the final announce delivery target from the requester session when available, preserves target-side channel metadata for prompts/provenance, keeps the requester-less fallback, and lands with the requester-target regression test plus the existing threadId/session-lookup coverage.
