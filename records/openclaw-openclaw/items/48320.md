## Summary

Keep this PR open. Current main still does not implement the intended Windows spawn fix: the shared plugin SDK helper still omits `windowsHide: true` for Win32 direct executable resolution and explicit shell fallback, and the ACP client forwards that missing value into `child_process.spawn()`. The branch is stale against the current ACPX test layout, but the core fix remains focused and actionable.

## Best Possible Solution

Keep the PR open for maintainer follow-up and ask for a rebase. The focused fix should update `src/plugin-sdk/windows-spawn.ts` so Win32 `direct` and explicit `shell-fallback` outcomes return `windowsHide: true`, add direct `.exe` regression coverage in `src/plugin-sdk/windows-spawn.test.ts`, update the shell-fallback expectation, preserve non-Windows `undefined` behavior and fail-closed unresolved wrappers, and refresh or drop stale ACPX test-only edits. Consider mirroring the behavior in the memory-host SDK copy if maintainers want consistent Windows process handling there.
