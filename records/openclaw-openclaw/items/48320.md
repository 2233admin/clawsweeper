## Summary

Keep this PR open. Current main still omits `windowsHide: true` from the shared Windows spawn helper’s Windows `direct` and explicit `shell-fallback` outcomes, and downstream ACP spawn paths still forward that helper value into `child_process.spawn()`. The PR is stale against current file layout, but the core behavior it targets is not implemented or superseded.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a rebase onto current main. The focused implementation should now update `src/plugin-sdk/windows-spawn.ts` and `src/plugin-sdk/windows-spawn.test.ts`, add explicit direct `.exe` Windows coverage, change shell-fallback coverage to expect `windowsHide: true`, and refresh or drop stale ACPX test-only edits.
