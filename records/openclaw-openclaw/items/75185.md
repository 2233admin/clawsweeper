## Summary

Keep open. Current main still has the Telegram DM no-visible-response fallback that can turn a synthetic `NO_REPLY` into visible filler text, and this PR is a narrow Telegram-owned fix with a matching regression update. There is an older open PR for the same bug, but this branch is updated against current main and should be reviewed as the likely replacement rather than swept closed.

## What This Changes

The PR changes Telegram dispatch to skip synthetic `NO_REPLY` fallback delivery for successful no-visible direct-message turns, treats that path as handled for status reactions, updates the Telegram dispatch regression test, and adds a changelog entry.

## Best Possible Solution

Land one narrow Telegram fix that keeps successful no-visible DM turns silent while preserving real error and non-silent delivery-failure fallbacks, then close the linked bug and supersede the older duplicate PR if this branch is chosen.

## Reproduction Assessment

Yes. The current-main test and source path reproduce the issue without live Telegram: `queuedFinal: false`, no dispatch error, and no prior delivery cause a synthetic `NO_REPLY` to flow through direct silent-reply rewriting.

## Solution Assessment

Yes. The proposed fix is scoped to the Telegram adapter branch that fabricates the fallback, keeps existing error fallback logic intact, and updates the inverted regression test; the only maintainer choice is whether this updated branch should supersede #70630.
