## Summary

Keep open. Current main still does not implement the PR's central Android assistant launch hardening: app-action prompts are trimmed but not capped, assistant launch handling is not fingerprinted/restored through saved instance state, and MainViewModel launch routing state is not SavedStateHandle-backed. The PR remains useful, but it is stale against the shipped #70714 Android/security behavior that keeps ASK_OPENCLAW prompts draft-only instead of auto-sending them.

## Best Possible Solution

Keep the PR open and ask for a current-main update: preserve `ASK_OPENCLAW` as `autoSend = false`, apply bounded prompt sanitization to the draft path, add restored-launch fingerprinting without the transient instance-field contract, persist launch routing state through `SavedStateHandle`, update tests for draft-only app actions plus restoration behavior, then run the targeted Android assistant unit tests and Kotlin compile lane.
