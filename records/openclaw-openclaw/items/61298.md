## Summary

Keep open. Current main still lacks the prompt-length cap and assistant-launch restoration work, but this stale branch is not merge-ready because it conflicts with the shipped draft-only security behavior for exported `ASK_OPENCLAW` App Actions.

## What This Changes

The PR branch adds Android assistant prompt trimming/capping, restored-launch fingerprint handling, `SavedStateHandle`-backed chat routing state, and unit coverage for those paths.

## Best Possible Solution

Update this PR or replace it with a narrow Android assistant hardening patch that keeps `ASK_OPENCLAW` prompts draft-only, caps the draft prompt, persists transient launch routing through saved state, passes restored launch fingerprints explicitly, and adds tests for draft-only capped prompts plus restoration behavior.

## Reproduction Assessment

Yes. The source/unit reproduction path is clear: current main has no sanitizer, restored-launch fingerprinting, or `SavedStateHandle` routing state for this surface, while the PR adds tests for those cases; the blocking regression is visible in the PR test expectation that `ASK_OPENCLAW` auto-sends.

## Solution Assessment

No. The hardening direction is appropriate, but this branch is not the best mergeable solution because it predates and conflicts with #70714; the safer path is to preserve `autoSend = false` for external app-action prompts while adding the lifecycle and prompt-boundary hardening.
