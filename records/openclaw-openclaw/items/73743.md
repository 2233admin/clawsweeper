## Summary

Keep this issue open. The report is recent, detailed, and points at current core/plugin-runtime surfaces that are not proven fixed on current main. The repo does not contain the reported `recipes`/`kitchen` plugins, but current main still exposes the raw timeout runner to plugins and still relies on plugin CLI activation metadata to avoid broad command-registration work, so this needs maintainer investigation rather than cleanup closure.

## Best Possible Solution

Keep the issue open and investigate with the reported plugin set installed on macOS. Profile `openclaw recipes list` and `recipes workflows worker-tick` through the CLI startup trace/plugin profiler, verify the external plugins declare `activation.onCommands` and lightweight `cli-metadata` descriptors, and harden the plugin-facing subprocess path so timeout or unparseable JSON cannot be cached as success. Add launchd jitter or deployment guidance only after the baseline single-command startup cost is understood.
