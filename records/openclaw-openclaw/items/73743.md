## Summary

Keep open. The report is current, detailed, and not implemented on current main: the named recipes/kitchen plugin sources are not bundled here, but the core subprocess and plugin-CLI paths still leave the reported startup/fan-out and timeout-cache hardening work unresolved.

## What This Changes

The issue reports slow OpenClaw CLI subprocess startup on macOS that compounds under many launchd workflow ticks and causes ClawKitchen to cache partial timed-out `recipes list` output as success.

## Best Possible Solution

Keep this open as a core/plugin-startup investigation: profile the reported `recipes` plugin command paths on macOS, narrow plugin CLI initialization for single-plugin commands with manifest descriptors/activation metadata, add a lightweight worker-tick path or launchd jitter if needed, and harden subprocess timeout results so partial stdout cannot be treated as successful JSON.

## Reproduction Assessment

Yes for a maintainer with the matching macOS setup: the issue includes concrete versions, launchd shape, commands, timings, process/RSS observations, and the downstream cache symptom. It was not live-reproduced in this Linux read-only checkout, and the external `recipes`/`kitchen` sources are not available here.

## Solution Assessment

No patch is proposed here, and current main is not a completed solution. The safer direction is to split the work into measured startup optimization for the plugin command path plus focused subprocess-timeout and JSON-output hardening.
