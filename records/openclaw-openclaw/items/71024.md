## Summary

Keep open. Current main still has the Mattermost status failure path: Mattermost has no read-only `inspectAccount`, its resolver still uses strict SecretRef resolution, and its status snapshot does not preserve `botTokenStatus`. The PR remains a focused fix for linked #70942 rather than obsolete cleanup.

## Best Possible Solution

Keep #71024 open. The best path is to merge this PR after maintainer and CI verification, or apply the same Mattermost-owned fix on main: add a read-only `inspectMattermostAccount`, wire it through the Mattermost config adapter, preserve `botTokenStatus: configured_unavailable` in status snapshots, keep runtime resolution strict, and retain the new regression tests.
