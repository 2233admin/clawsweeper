## Summary

Keep open for maintainer review. Current main still performs package-update restart verification from the parent update process, while this PR moves that verification into the updated install and appears to address the prior timeout and auth-env review findings with focused tests and green CI.

## What This Changes

The branch updates the CLI package-update restart flow so restart health/version verification runs through a hidden mode in the freshly installed OpenClaw CLI, with stale-PID retry, an expanded timeout budget, caller gateway auth-env preservation, and targeted tests.

## Best Possible Solution

Land this implementation or a maintainer-approved equivalent after normal review and final gate validation so package-update restart verification consistently runs from the updated install while keeping stale-PID recovery, expected-version checks, and auth-enabled probes intact.

## Reproduction Assessment

Yes. Current main has a high-confidence code and unit-test reproduction path: package updates run the updated restart command but still perform the health/version probe from the parent process. A live package-update smoke would increase confidence in host service-manager timing but is not required to understand the bug.

## Solution Assessment

Yes. Delegating only restart health/version verification to the updated CLI is the narrowest maintainable fix for stale parent modules, and the PR preserves the existing in-process fallback for non-package update modes.
