## Summary

Keep open. The PR addresses a real package-update restart verification gap that current main still has, but the proposed patch introduces a blocking false-failure path because the parent allows only 70 seconds for a delegated verifier that can legitimately run stale-PID cleanup, an updated-install gateway restart, and a second health wait.

## What This Changes

The PR changes the update CLI to delegate package-update restart verification to the freshly installed OpenClaw entrypoint via hidden restart-verification environment mode and updates the CLI tests around restart/version/plugin-error verification.

## Best Possible Solution

The update CLI should keep the updated-install verification design, but give the delegated verifier a timeout budget that covers the full recovery path, preferably derived from the update timeout or explicit child-step budgets, with coverage for stale-PID cleanup and successful delayed recovery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Raise delegated restart verification timeout budget:** `src/cli/update-cli/update-command.ts:554`
  - body: The parent limits the child `openclaw update` restart verifier to 70s, but the verifier can legitimately spend up to one health wait, stale-PID cleanup, a 60s `gateway restart`, and another health wait. The parent can therefore kill a valid recovery path early and report a package update failure even though the gateway would recover shortly after.
  - confidence: 0.89
