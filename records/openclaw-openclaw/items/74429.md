## Summary

Keep open. Current main does not implement the proposed CLI `config apply` command, and the branch is an active implementation candidate rather than obsolete cleanup. It still needs maintainer review because it mutates config from file/stdin, has unresolved bot-review findings, and uses `config apply` for partial patches while gateway docs currently reserve `config.apply` for full-config replacement.

## What This Changes

The PR adds an `openclaw config apply` subcommand for JSON5 config patches from files or stdin, with recursive object patching, null deletes, `--replace-path`, shared validation/dry-run write handling, tests, and related CLI/channel/install docs.

## Best Possible Solution

If maintainers want this capability, land a reviewed version that preserves the existing validation, dry-run, SecretRef, unsupported-surface, and atomic write safeguards while resolving the CLI/RPC naming contract, validating every requested `--replace-path`, and adding clear stdin size/TTY handling before updating setup docs to recommend it.
