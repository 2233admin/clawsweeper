## Summary

Keep this open. Current main still has useful external-content markers, immediate web_fetch metadata, message-scoped provenance, plugin hook blocking, exec approvals, and outbound hooks, but it does not implement the requested end-to-end model: persistent taint through derived content, default content-derived execution denial, exec risk classes, or provenance-aware secret/outbound enforcement. The related issue list is useful context for decomposition, not proof that this umbrella is shipped or superseded.

## What This Changes

Request an end-to-end external-content security model with persistent taint/provenance, content-derived execution guardrails, exec risk classes, and provenance-aware secret and outbound-action controls.

## Best Possible Solution

Keep this as the core security architecture umbrella unless maintainers explicitly split it into canonical implementation issues. The desired end state is structured provenance/taint metadata for external inputs and tool outputs, propagation through summaries and persisted state, stable action/risk/provenance fields in hook and approval payloads, and core enforcement that blocks untrusted content from authorizing execution, secret access, or outbound actions without explicit guarded review.
