## Summary

Keep this PR open. Current main already keeps Mattermost direct messages out of thread-scoped sessions, but it does not implement the PR's other observable behavior: preserving DM reply context as structured metadata, fetching the replied post, root typing parent_id handling, direct-only typing TTL behavior, or websocket silence heartbeat handling. The provided discussion also shows the author addressed earlier review findings and reported live validation, so this is a real Mattermost bugfix candidate, not conservative cleanup material.

## Best Possible Solution

Keep the PR open as a Mattermost plugin bugfix candidate. The best path is to review or update it so Mattermost DM replies keep a single direct session while preserving structured `ReplyToBody`/`ReplyToSender` context, preserving explicit outbound reply targets, gating replied-post fetches to DMs, avoiding raw body mutation and attachment downloads, and either splitting or separately validating the websocket heartbeat and typing changes.
