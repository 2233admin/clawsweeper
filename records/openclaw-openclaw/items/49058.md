## Summary

Keep this PR open. Current main implements the single-session half for Mattermost DMs, but it still does not implement the PR's remaining observable behavior: structured replied-post context, direct-only replied-post fetching, explicit DM reply-target preservation, root typing parent_id compatibility, direct-only typing TTL override, or websocket silence heartbeat handling. The PR also has meaningful review history and a late P1 functional concern, so it remains a real Mattermost bugfix candidate rather than cleanup material.

## Best Possible Solution

Keep this PR open as a Mattermost plugin bugfix candidate. The best path is to review or update it so Mattermost DMs keep one OpenClaw session while preserving structured ReplyToBody/ReplyToSender context, fetching replied posts only for direct messages, preserving explicit outbound reply targets where intended, and avoiding raw body mutation or attachment downloads. The websocket heartbeat and typing changes should either be split or validated separately with focused tests/live Mattermost proof.
