## Summary

Current main still lacks the requested cross-platform message-tool multi-select contract. The shared presentation schema, public SDK export, docs, and channel renderers support a portable single select plus buttons, while adjacent Discord/Slack/Telegram native paths do not provide the requested min/max multi-select and one structured selected-values callback across channels. Related PR #18286 proposed a dialog/multiselect path but is closed unmerged and its dialog API is not present on main.

## Best Possible Solution

Keep this issue open until OpenClaw adds an additive portable batch-selection contract in the shared message presentation/message tool surface, exports it through openclaw/plugin-sdk/interactive-runtime, documents it on docs.openclaw.ai/plugins/message-presentation, and implements channel-specific render plus callback handling for Slack, Discord, Telegram, and fallback text paths. Related PR #18286 can be mined for dialog/data-collection ideas, but it does not supersede the current missing message-tool feature because it was closed unmerged.
