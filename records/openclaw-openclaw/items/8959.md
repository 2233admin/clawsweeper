## Summary

Keep this issue open. Current main still has only a portable single-select presentation/message-tool contract, while the requested cross-platform multi-select contract with min/max selections and one selected-values callback is not implemented. Related PR #18286 proposed a broader dialog/multiselect path, but it is closed unmerged and current main does not contain that dialog API.

## Best Possible Solution

Keep this issue open until OpenClaw adds an additive portable batch-selection contract in the shared message presentation/message tool surface, exports it through openclaw/plugin-sdk/interactive-runtime, documents it on docs.openclaw.ai/plugins/message-presentation, and implements channel-specific render plus callback handling for Slack, Discord, Telegram, and fallback text paths. Related PR #18286 can be mined for dialog/data-collection ideas, but it does not supersede the missing message-tool feature because it was closed unmerged.
