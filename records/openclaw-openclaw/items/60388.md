## Summary

Keep this issue open. Current main still lacks the requested distinction between the embedded run's own timeout budget and retryable timeout-shaped provider failures, and open PR #62682 remains the focused implementation candidate.

## Best Possible Solution

Keep the issue open and review #62682, or land an equivalent focused change that carries terminal run-abort context into the model fallback decision. The fix should fail immediately for the embedded runner's own timeout budget while preserving fallback for provider timeouts, rate limits, auth failures, and overloads.
