## Summary

Keep open. Current main still lacks a caller-selectable `sessions_spawn` lane: the tool schema has no `lane` field, native and ACP child dispatch both hard-code `subagent`, and config/docs still expose one scalar subagent concurrency cap. The adjacent per-agent lane work is not a replacement because it covers inbound agent runs, not sub-agent lanes spawned through this tool.

## What This Changes

The issue requests an optional `sessions_spawn.lane` parameter with per-lane subagent concurrency configuration and a default fallback to the existing `subagent` lane.

## Best Possible Solution

Add a backward-compatible `sessions_spawn.lane` API that validates a bounded lane name, defaults omitted or blank lanes to `subagent`, threads the resolved lane through native and ACP child dispatch, applies compatible per-lane subagent concurrency config, and updates docs plus regression tests while preserving existing callers.
