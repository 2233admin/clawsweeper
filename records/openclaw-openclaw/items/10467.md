## Summary

Keep this issue open. Current main still lacks multi-lane sessions_spawn support: the tool schema exposes no lane parameter, both native and ACP spawn paths dispatch agent runs on the fixed subagent lane, and config/docs/tests still model a single global agents.defaults.subagents.maxConcurrent cap. The April 26 maintainer-side Codex comment remains accurate on the current main checkout, and the nearby open delegate PR explicitly does not change sessions_spawn lanes.

## What This Changes

Add an optional sessions_spawn lane parameter with independent per-lane subagent queue concurrency, defaulting to the existing subagent lane.

## Best Possible Solution

Keep this issue open until OpenClaw adds an additive sessions_spawn.lane parameter that defaults to subagent, validates lane names, forwards the resolved lane through native and ACP spawn paths, applies per-lane subagent concurrency while preserving the existing scalar default, and updates config schema, docs, and regression tests.

## Repair Work Prompt

Implement multi-lane concurrency support for `sessions_spawn`.

Observable problem: `sessions_spawn` currently has no `lane` parameter, and both native and ACP child `agent` calls always run on the fixed `subagent` lane. `agents.defaults.subagents.maxConcurrent` is a single global cap, so slow/background subagents can starve unrelated higher-priority subagent work.

Expected fix boundary:
- Add an optional `lane` parameter to the `sessions_spawn` tool schema and execution path.
- Default omitted/blank lane to the existing `subagent` lane.
- Validate lane names with a conservative allowlist/normalizer; reject unsafe names rather than creating arbitrary queue keys from untrusted input.
- Forward the resolved lane through `spawnSubagentDirect` and `spawnAcpDirect` into their gateway `agent` calls.
- Add per-lane subagent concurrency configuration while preserving the existing scalar `agents.defaults.subagents.maxConcurrent` behavior as the default/fallback for `subagent` and unspecified lanes.
- Keep existing depth, maxChildrenPerAgent, sandbox, ACP policy, lifecycle, and cleanup behavior intact.
- Update config types, zod schema, labels/help/docs, and focused regression tests.
- Do not change unrelated `sessions_delegate` behavior or broaden channel delivery params for `sessions_spawn`.

Related refs: this issue, #4355 for lock/contention background, and #64281 as adjacent but explicitly non-superseding work.

Changelog: add a user-facing Fixes/Changes entry only if the repo’s current changelog policy indicates enhancement entries are expected for this surface; include non-bot reporter credit if adding one.
