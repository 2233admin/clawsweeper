## Summary

Current main still does not implement multi-lane `sessions_spawn` support. The tool schema has no `lane` parameter, neither the native nor ACP spawn path forwards a lane, both dispatch child `agent` runs on the fixed `subagent` lane, and config/docs/tests still model sub-agent concurrency as one global `agents.defaults.subagents.maxConcurrent` scalar.

## Best Possible Solution

Keep this issue open until OpenClaw adds an additive `sessions_spawn.lane` parameter that defaults to `subagent`, validates custom lane names, forwards the resolved lane through both native and ACP spawn paths, supports a maintainer-approved per-lane subagent concurrency config while preserving `agents.defaults.subagents.maxConcurrent` for the default lane, and updates docs plus regression tests.
