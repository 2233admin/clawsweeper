## Summary

Keep this issue open. Current main still implements `sessions_spawn` with one fixed `subagent` queue lane: the tool schema has no `lane` parameter, native and ACP spawn paths both dispatch with `AGENT_LANE_SUBAGENT`, config/schema still expose only the scalar `agents.defaults.subagents.maxConcurrent`, and docs/tests explicitly describe/assert the single global lane. The maintainer-side discussion from April 26, 2026 remains accurate against current main.

## What This Changes

The issue requests an optional `sessions_spawn.lane` parameter with default `subagent`, independent queue lanes, and per-lane subagent concurrency configuration.

## Best Possible Solution

Add an additive `sessions_spawn.lane` parameter that defaults to `subagent`, validates lane names, forwards the resolved lane through both native and ACP spawn paths, applies compatible per-lane subagent concurrency configuration, and updates schema, docs, and regression tests without changing existing callers.
