## Summary

Keep open. Current main still lacks the requested Anthropic context-management/native passthrough: headers can be configured through existing Anthropic beta handling, but there is no Anthropic `context_management` payload shaping, no compaction/context-editing stream replay or pause handling, and no provider-defined `memory_20250818` tool path. Related #10213 remains open for the narrower compaction slice, #10214 was closed unmerged, and closed #48558 only resolved the memory-tool-only request as plugin/ClawHub scope, so they do not supersede this broader provider design issue.

## What This Changes

The issue requests native or explicit passthrough support for Anthropic beta context-management surfaces, including Memory Tool, server-side compaction, context editing, beta headers, and provider-defined tool objects.

## Best Possible Solution

Keep this open for bundled Anthropic provider design. The desired end state is a documented, opt-in Anthropic context-management surface that validates and sends `context_management.edits`, handles beta headers, preserves/replays compaction and context-editing semantics in the stream/history model, supports pause/resume behavior, and deliberately routes `memory_20250818` either through a provider-owned implementation or a generic provider-defined-tool SDK seam.
