## Summary

Keep this issue open. Current main still has memory-search query tuning for max results, min score, and hybrid ranking, plus adjacent timeout/cache behavior in QMD, Active Memory, and embedding batch code. It still does not expose schema-backed `agents.defaults.memorySearch.query.timeoutMs`, `cacheTtlMs`, or `retry` through the main per-agent memory search config/runtime path.

## Best Possible Solution

Implement the requested controls in the core memory-search config boundary: add typed and zod-validated `agents.defaults.memorySearch.query.timeoutMs`, `cacheTtlMs`, and `retry`; resolve them through `src/agents/memory-search.ts`; wire them into the builtin memory search execution path with bounded timeout, short-lived in-session query-result caching, transient retry behavior, and graceful fallback; add focused schema/runtime tests; and document the public options in the memory configuration reference. Keep #49056 and #49233 as adjacent durability/provider context, not replacements for this query reliability work.
