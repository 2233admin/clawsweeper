## Summary

Keep open. The report is still a concrete npm-global downgrade failure, and current main still lacks proof that `openclaw update --tag 2026.4.25` can pass staged npm verification from a 2026.4.26 install; the verifier remains stricter than the repo’s documented/package-acceptance compatibility window for already-published 2026.4.25 artifacts.

## What This Changes

Fix npm global downgrades from OpenClaw 2026.4.26 to 2026.4.25 so `openclaw update --tag 2026.4.25` completes instead of failing `global install verify`.

## Best Possible Solution

Add bounded installed-package verifier compatibility matching the already-published package window, especially 2026.4.25 private-QA inventory omissions, then add an exact npm downgrade regression and real Linux npm-global proof for 2026.4.26 to 2026.4.25 without bypassing version or dist integrity checks.
