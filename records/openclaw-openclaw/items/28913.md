## Summary

Keep open: current main still lacks the requested source- or wrapper-aware autoCapture filter. The memory-lancedb config parser and manifest do not accept `captureFilter.excludeWrapped`, the auto-capture hook still sends extracted user text directly through `shouldCapture`, and the existing tests cover trigger/prompt-injection heuristics but not EXTERNAL_UNTRUSTED_CONTENT-wrapped trigger text. Related #29442 and #48519 are broader security/provenance discussions, not replacements for this narrow plugin option.

## What This Changes

Add a backward-compatible `captureFilter.excludeWrapped` option to memory-lancedb so `autoCapture` skips configured wrapped external-content blocks before applying capture triggers.

## Best Possible Solution

Keep this issue open and implement a backward-compatible memory-lancedb `captureFilter.excludeWrapped` option. The fix should update the plugin config parser, manifest schema/UI hints, docs, changelog, and agent_end auto-capture path, with focused tests proving EXTERNAL_UNTRUSTED_CONTENT-wrapped trigger text is skipped while ordinary direct user preference capture still works. If robust marker detection needs shared logic, add or reuse a small public plugin-SDK security helper instead of importing core internals directly.
