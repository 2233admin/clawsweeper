## Summary

Current main still lacks source- or wrapper-aware filtering for memory-lancedb autoCapture. The requested captureFilter.excludeWrapped option is absent from config parsing, manifest/UI metadata, docs, and the agent_end capture path, while external-content boundary markers still exist elsewhere in core.

## Best Possible Solution

Keep this open. Implement a backward-compatible memory-lancedb captureFilter.excludeWrapped option in the plugin-owned config parser, manifest schema/UI hints, docs, and agent_end auto-capture path. Add focused tests proving wrapped EXTERNAL_UNTRUSTED_CONTENT with trigger words is skipped while ordinary direct preference capture still works, and keep broader provenance/privilege-separation design in #29442 and #48519.

## Risks / Open Questions

- Current autoCapture can still persist wrapped external text if it reaches a user message and matches broad trigger words such as always, never, or important without matching the existing prompt-injection regexes.
- A naive marker string check could drift from the randomized EXTERNAL_UNTRUSTED_CONTENT marker format or create spoofing edge cases; the implementation should deliberately mirror the current marker semantics while respecting the extension/core boundary.
- Adding only runtime filtering would leave config validation, manifest/UI hints, docs, and tests out of sync.
- This narrow plugin filter should not be treated as a complete provenance or privilege-separation solution; #29442 and #48519 remain broader open architecture discussions.
