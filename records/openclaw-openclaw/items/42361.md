## Summary

Keep this PR open. Current main still does not implement the headline web_fetch diagnostic improvement: direct network failures are still rethrown after provider fallback misses, so the agent can still see a bare `fetch failed` without requested URL or underlying cause. The PR contains a useful web_fetch patch, but it is bundled with unrelated Feishu, Telegram, config/model/table, tool-result, and package override changes, including security- and install-sensitive risks.

## Best Possible Solution

Keep this PR open for maintainer rework or replacement. The best path is a narrow web_fetch-only patch that wraps direct network-level failures only after provider fallback misses, extracts `error.cause.message` or `error.cause.code`, preserves the original error as the cause, includes safe requested-URL context, and adds a regression test for the no-fallback network-error case. Split or drop the unrelated Feishu, Telegram, config/schema, model/table, image-helper, tool-result, and package override changes.
