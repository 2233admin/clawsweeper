## Summary

Keep #69975 open. Current main still exposes the older cron-only `--tz` help in `cron add` and `cron edit`, while current runtime code and tests confirm `--tz` does apply to offset-less ISO `--at` values. The PR is not obsolete, but its exact proposed wording needs amendment because it says `--tz` defaults to UTC even though cron schedules without `--tz` use the gateway host timezone.

## Best Possible Solution

Keep the PR open and ask for a small wording revision. The best fix is still in `src/cli/cron-cli/register.cron-add.ts` and `src/cli/cron-cli/register.cron-edit.ts`: clarify that `--tz` applies to `--cron` and offset-less ISO `--at` datetimes, but avoid a single `default: UTC` phrase that misstates cron behavior. A safe wording should distinguish cron's host-timezone fallback from offset-less `--at` UTC behavior, aligning with `docs/automation/cron-jobs.md` and the existing CLI tests.
