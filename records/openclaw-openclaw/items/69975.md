## Summary

Keep this PR open. Current main still only partially covers the intended CLI help cleanup: `cron add --at` mentions `--tz`, but `cron edit --at` does not, and both `--tz` help strings still describe cron expressions only. The PR is narrow and security-benign, but it should not merge unchanged because its `(default: UTC)` wording is wrong for recurring cron schedules, which default to the Gateway host timezone when `tz` is omitted.

## Best Possible Solution

Keep the PR open and request a narrow wording refresh. The final help text should make `cron edit --at` mention offset-less datetimes with `--tz`, and should update the `--tz` description to cover both recurring cron expressions and offset-less one-shot datetimes without claiming a single UTC default for both. Validate with the existing cron CLI help tests or rendered `openclaw cron add --help` / `openclaw cron edit --help` output.
