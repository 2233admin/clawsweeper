## Summary

Keep open. Current main still has the reported session metadata behavior, and this PR is a focused implementation for the linked open bug; it should proceed through normal maintainer review after addressing the low-severity test portability finding and validation.

## What This Changes

The PR adds a preserve-activity option to session-store entry updates, applies it to compaction and memory-flush metadata writes, removes explicit `updatedAt` writes from compaction bookkeeping, and adds regression tests for legacy daily/idle reset behavior.

## Best Possible Solution

Land a narrow version of this fix after switching the new fixture root to `os.tmpdir()` and validating the touched session-store, compaction, and memory-flush paths. The reset policy should remain unchanged, with metadata-only writers explicitly opting into preserve-activity behavior.

## Reproduction Assessment

Yes. The linked bug gives concrete daily and idle legacy-row scenarios, and current main source confirms the reproduction path: compaction/flush metadata can advance `updatedAt` while reset freshness falls back to `updatedAt` when lifecycle timestamps are missing.

## Solution Assessment

Mostly yes. The proposed preserve-activity option is the narrowest maintainable fix because it reuses the existing merge policy and limits behavior changes to metadata-only writers; the safer adjustment is to make the new test fixture root platform-neutral before merge.
