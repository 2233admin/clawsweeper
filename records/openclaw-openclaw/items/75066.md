## Summary

Keep open. Current main still has the reported session freshness bug, and this PR is a focused implementation candidate for the linked open bug; it should not be closed before merge. The patch direction is sound, with two low-severity pre-merge fixes: make the new temp fixture cross-platform and add the required changelog entry for the user-visible fix.

## What This Changes

The branch adds a preserve-activity option to session-store entry updates, applies it to compaction and memory-flush metadata writes, removes compaction bookkeeping updatedAt writes, and adds regression coverage for legacy daily/idle reset behavior.

## Best Possible Solution

Land the narrow preserve-activity fix after making the new fixture root use the platform temp directory and adding a single-line Unreleased Fixes changelog entry. The core reset policy should stay unchanged; only metadata-only compaction and memory-flush writers should opt into activity preservation.

## Reproduction Assessment

Yes. The linked bug provides concrete daily and idle legacy-row scenarios, and current main source confirms both paths: metadata writes can advance updatedAt while reset freshness falls back to updatedAt when explicit lifecycle timestamps are missing.

## Solution Assessment

Mostly yes. Reusing the existing preserve-activity merge policy for selected metadata-only writers is the narrowest maintainable code path; the safer pre-merge version also fixes the cross-platform test fixture and satisfies changelog policy.
