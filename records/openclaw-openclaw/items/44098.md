## Summary

Keep this PR open. Current main still creates sandbox Docker config without a default PID limit, and the linked security issue remains open. The branch is a plausible hardening fix, but it needs maintainer/security review because it changes the sandbox compatibility contract, lacks matching docs/test updates, and the PR head has failing checks.

## What This Changes

The PR adds a sandbox PID-limit constant and uses it as the fallback value in Docker sandbox config resolution when neither agent nor global Docker config sets `pidsLimit`.

## Best Possible Solution

Land a maintained sandbox hardening change that documents and tests the default PID-limit behavior, preserves explicit configured overrides, clarifies whether unlimited is still supported, and coordinates with #60827 for any broader CPU or memory defaults.
