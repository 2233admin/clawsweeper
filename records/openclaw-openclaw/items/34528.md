## Summary

Keep open. Current main still creates and preserves Feishu synthetic reaction message IDs with the reported `:reaction:<emoji>:<uuid>` suffix, and raw message IDs still flow into Feishu API path parameters. The provided GitHub context also shows open PR #39558 with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #39558 with a narrow Feishu plugin fix that strips the synthetic `:reaction:<emoji>:<uuid>` suffix at every Feishu API-boundary `message_id` path parameter while preserving suffixed IDs for internal dedupe, session routing, and event identity. Regression coverage should include reply dispatch, typing reactions, explicit reaction actions, media reply/download paths, message get/edit, and streaming-card reply start/update paths.
