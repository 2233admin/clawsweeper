## Summary

Keep open. Current main at a820a307dfeb950fa633c0ae6fb2386a35a91991 still creates Feishu synthetic reaction message IDs with the reported :reaction:<emoji>:<uuid> suffix, preserves that value in ctx.messageId, and can pass it unnormalized into Feishu API path parameters. The provided issue context also shows open PR #39558 with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, rebase, land, or replace PR #39558 with a narrow Feishu plugin fix that adds a tested reaction message ID normalizer at every Feishu API-boundary message_id path parameter while preserving suffixed synthetic IDs for dedupe and session routing. Regression coverage should include reply dispatch, typing reactions, Feishu reaction helpers, media replies/downloads, message get/edit, and streaming card start/update paths.
