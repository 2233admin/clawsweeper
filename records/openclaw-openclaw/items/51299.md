## Summary

Keep open. Current main still does not provide a proven SIP-enabled Sequoia/Tahoe TCC approval path for legacy `imsg` under a bare LaunchAgent, and the runtime/probe still treat the reported Full Disk Access and AppleEvents failures as generic `imsg rpc` errors. BlueBubbles is documented as the recommended iMessage path, but it is a separate channel and does not answer the legacy LaunchAgent/imsg question.

## Best Possible Solution

Keep this issue open for explicit maintainer/platform guidance. The best outcome is either a tested Sequoia/Tahoe-safe legacy `imsg` LaunchAgent procedure, or a clear statement that bare LaunchAgent `imsg` is not reliable under modern macOS TCC/SIP, with docs and status/probe diagnostics pointing users to BlueBubbles, a GUI-user Mac setup, or another supported path. If support remains, diagnostics should classify `permissionDenied`, AppleEvents `-1743`, Full Disk Access, Automation, and LaunchAgent context failures into actionable text.
