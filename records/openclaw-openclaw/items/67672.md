## Summary

Keep open. Current main still does not explicitly await Slack SocketModeClient.disconnect() before stopping the Bolt app, and the stale-socket restart path described by the PR still exists. The PR remains a plausible narrow bug fix, but it should be rebased onto the current provider-support.ts layout and gain a focused regression test.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, put the Slack-owned cleanup in extensions/slack/src/monitor/provider-support.ts and/or reconnect-policy.ts, await the underlying SocketModeClient.disconnect() before app.stop(), preserve the removal of the unnecessary gateway.stopAccount hook, and add a small regression test for disconnect-before-stop/awaited cleanup.
