## Summary

Current main does not implement this PR's intended Linux brew-to-apt fallback. `installSkill` still returns the brew-missing failure immediately when a `kind: "brew"` installer has no Homebrew executable, while only `kind: "go"` has an apt fallback. The linked Docker/Homebrew issue (#14593) and alternative Docker/Linuxbrew PR (#42303) are both still open, so this PR is not safely closeable as implemented or superseded.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either revise the apt fallback so it is explicitly gated, allowlisted, or limited to trusted/bundled installers with regression tests, or decide that the Docker/Linuxbrew approach in #42303 is the canonical fix for #14593 and close this PR only after that decision is explicit.

- The related #42303 PR is an alternative Docker/Linuxbrew approach, but it is also open and not a clear merged or maintainer-designated superseding fix.
