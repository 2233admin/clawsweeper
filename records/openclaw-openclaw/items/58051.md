## Summary

Keep open. Current main still has the shared transient cooldown probe slot behavior that this PR changes: a primary cooldown probe can set `transientProbeProviderForAttempt`, and a later failed attempt can add that provider to `cooldownProbeUsedProviders`, causing later same-provider fallbacks to be skipped. The PR remains a focused, useful fix with added regression coverage from the review discussion, not an implemented or superseded change.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. The best path is to land an equivalent `runWithModelFallback` change that prevents primary cooldown probes from consuming the transient same-provider fallback slot while preserving non-primary same-provider probe deduplication, with the PR's two regression tests or equivalent coverage.
