## Summary

Keep open. The branch is useful but still incomplete: it recovers labels only in the raw JSONL fallback while current v3 transcripts take the active-branch reader first, so hidden runtime-context rows are ignored there and sender labels can still be lost.

## What This Changes

The PR adds gateway session-history helpers and regression tests to recover `senderLabel` from hidden `openclaw.runtime-context` rows while keeping those rows out of `readSessionMessages()` output.

## Best Possible Solution

Handle hidden runtime-context `custom_message` entries consistently in both the active-branch and raw JSONL readers, keep those rows out of visible history, preserve the clean model-facing prompt/body boundary, and add focused v3 active-branch regression coverage.

## Reproduction Assessment

Yes. A v3 transcript with a user message followed by a hidden `openclaw.runtime-context` `custom_message` entry will enter `SessionManager.getBranch()` and return before this PR’s raw-loop recovery runs, so the visible user message still lacks `senderLabel`.

## Solution Assessment

No. Recovering from raw JSONL rows is a useful partial fix, but the maintainable solution is to share the runtime-context filtering/label attachment with the active-branch path and cover both storage shapes.
