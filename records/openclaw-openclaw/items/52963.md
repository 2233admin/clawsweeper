## Summary

Keep this issue open. Current main still documents and implements Slack Socket Mode around bot-event subscriptions and bot-token reply delivery; `userToken` is present for reads/action helpers and limited write fallback, but not as an inbound user-event identity for DM routing.

## What This Changes

The issue requests Slack user-level `message.im` event support so DMs to a configured Slack user account can be routed through OpenClaw and answered with the configured xoxp user token.

## Best Possible Solution

Implement the feature in the bundled Slack plugin only after maintainer product/security direction: document `user_events` setup for `message.im`, resolve the configured xoxp identity at monitor startup, distinguish bot-level and user-level DM events, select the user token only for eligible opt-in replies, prevent self-loops, and cover manifest/docs, routing, token selection, and live Slack behavior with regression tests.
