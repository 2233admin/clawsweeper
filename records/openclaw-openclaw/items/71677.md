## Summary

Keep open. Current main still treats Feishu `card.action.trigger` as a void card-action path: the handler calls `handleFeishuCardAction` and returns nothing, quick/confirmation actions still await synthetic command dispatch, and the `ocf1` envelope has no disable-state metadata. The SDK plumbing described in the report is real, but OpenClaw does not yet use it to return an immediate transitional card or toast.

## What This Changes

Request support for immediate Feishu interactive-card callback responses so clicked buttons can show a disabled or processing state while command handling continues.

## Best Possible Solution

Implement the feature inside the bundled Feishu plugin: add an opt-in typed interaction metadata field for immediate disable/processing state, return a valid callback response for structured card button clicks, and keep final message/card edits on the existing outbound path.
