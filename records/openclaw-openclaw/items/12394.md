## Summary

Keep this issue open. Current main still lacks a channel-level WhatsApp heartbeat `target`/`targets` recipient contract: the channel heartbeat schema and types are visibility-only, WhatsApp reuses that strict shape, recipient resolution only considers explicit runtime opts, session recipients, `allowFrom`, and pairing-store entries, and the existing agent-level `heartbeat.target`/`to` route both OK and alert payloads rather than OK-only admin acknowledgments.

## Best Possible Solution

Keep this open until maintainers implement or explicitly reject an additive WhatsApp/channel heartbeat recipient contract. A complete fix should extend schema/types/generated config docs for `channels.whatsapp.heartbeat.target` and/or `targets`, validate normalized recipients against the selected WhatsApp account and allowlist/pairing policy, apply those recipients only to `HEARTBEAT_OK` delivery when `showOk` is enabled, preserve alert/content routing, and add coverage for omitted-target compatibility, single target, multiple targets, per-account overrides, and invalid targets.
