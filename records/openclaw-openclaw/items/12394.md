## Summary

Keep this issue open. Current main still does not implement per-user/channel-level WhatsApp heartbeat OK routing: the channel heartbeat config remains visibility-only, WhatsApp reuses that strict shape, recipient resolution ignores `channels.whatsapp.heartbeat.target`/`targets`, and the existing `agents.defaults.heartbeat.target`/`to` route the whole heartbeat delivery rather than OK-only admin acknowledgments while preserving alert routing.

## Best Possible Solution

Keep this open until maintainers implement or explicitly reject an additive WhatsApp/channel heartbeat recipient contract. A complete fix should extend schema/types/generated docs for `channels.whatsapp.heartbeat.target` or `targets`, validate normalized recipients against the selected WhatsApp account and allowlist/pairing policy, apply those recipients only to `HEARTBEAT_OK` delivery when `showOk` is enabled, preserve alert/content routing, and add coverage for omitted-target compatibility, single target, multiple targets, per-account overrides, and invalid targets.
