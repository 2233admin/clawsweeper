## Summary

Keep this PR open. Current main still requires `models.providers.<id>.models` in the source Zod schema, generated schema, and exported config type, while the memory embedding runtime would use `models.providers.<id>.baseUrl` if validation accepted credential/baseUrl-only provider entries. The linked bug #39589 remains open, and related PRs #46980 and #71636 were closed unmerged rather than superseding this PR.

## Best Possible Solution

Keep this PR open, or land an equivalent focused fix, after aligning the source Zod schema, generated config schema, and exported `ModelProviderConfig` contract so config input can omit `models` while parsed/runtime code still has a safe array. Add focused regression coverage for config parsing plus OpenAI embedding `models.providers.openai.baseUrl` fallback behavior, stub DNS/SSRF lookups in tests, and explicitly review the existing remote embedding SSRF policy before merge.
