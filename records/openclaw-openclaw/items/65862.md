## Summary

Current main still appears to exclude DuckDuckGo from the interactive web-search setup list: the setup flow filters on `onboardingScopes`, the real DuckDuckGo runtime/contract metadata still does not provide that scope, and the docs still present DuckDuckGo as a selectable key-free provider. Because open PR #65940 references this issue with closing syntax, this issue should stay open for that PR to be reviewed or refreshed rather than being auto-closed.

## What This Changes

DuckDuckGo should appear as a selectable key-free web-search provider in the interactive `openclaw configure --section web` setup flow.

## Best Possible Solution

Keep the issue open until the linked fix path lands. The durable implementation should put `onboardingScopes: ["text-inference"]` in `extensions/duckduckgo/src/ddg-search-provider.shared.ts` so runtime and lightweight contract surfaces stay aligned, with focused regression coverage that resolves the real bundled DuckDuckGo provider through setup-mode discovery.

## Reproduction Assessment

Yes. A source-level reproduction path is high confidence: setup resolves web-search providers in `mode: "setup"`, filters by `onboardingScopes`, and the real DuckDuckGo provider metadata currently lacks the required `"text-inference"` scope.

## Solution Assessment

Yes, adding the setup scope to the shared DuckDuckGo provider metadata is the narrowest maintainable fix. Adding the field separately to runtime and contract files would work locally but is more drift-prone than updating the shared base.
