This should stay open for maintainer review of the paired PR rather than be auto-closed.

## Best Possible Solution

Keep this issue open and review or refresh PR #65940. The best fix is to add `onboardingScopes: ["text-inference"]` to the shared DuckDuckGo web-search provider metadata so both runtime and lightweight contract surfaces inherit it, then add a focused regression assertion that the real bundled DuckDuckGo provider appears in setup/web-search options when plugin policy allows it.
