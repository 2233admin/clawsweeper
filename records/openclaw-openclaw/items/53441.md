## Summary

Current main does not implement this PR's intended behavior. The Synology Chat webhook handler still rejects HEAD requests with 405 and still ACKs accepted POSTs with 204 No Content, while the tests assert that 204 behavior. This is a focused implementation candidate for the linked open Synology Chat issue, so it should remain open for maintainer review or landing rather than being closed by cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate for the linked Synology Chat issue. The likely maintainer path is to review and, if accepted, land the scoped extension fix: return 200 with {"success":true} for accepted webhook ACKs, return 200/no body for Synology HEAD probes, update the 204 assertions and HEAD regression coverage, and optionally document the provider-specific ACK expectation before closing the paired issue.
