## Summary

Current main still leaves heartbeat responsePrefix on the raw-string path, so templated values such as `{model}` can still be prepended literally. This also has an open same-author implementation candidate, #43065, that targets the same behavior and references this issue for closure, so it should stay open until that PR is reviewed, landed, or rejected.

## Best Possible Solution

Keep this issue open and review #43065 as the primary implementation candidate. The fix should wire the heartbeat runner into the shared reply-prefix context path, pass onModelSelected through getReplyFromConfig, resolve the prefix after model selection, and add focused regression coverage for templated heartbeat alert prefixes plus prefixed HEARTBEAT_OK handling.
