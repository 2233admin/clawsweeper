## Summary

Current main still appears to have the reported heartbeat prefix bug: heartbeat reads the raw responsePrefix, never wires the reply-prefix model-selection callback into getReplyFromConfig, and passes the raw prefix into heartbeat reply normalization. The issue also has an open same-author implementation candidate, #43065, with closing syntax, so it should stay open until that PR is reviewed, landed, or explicitly rejected.

## Best Possible Solution

Keep this issue open and review #43065 as the primary implementation candidate. The fix should route heartbeat runs through the shared reply-prefix context, pass onModelSelected into getReplyFromConfig, resolve the prefix after model selection before normalizeHeartbeatReply, and add focused regression coverage for templated heartbeat alert prefixes plus prefixed HEARTBEAT_OK handling.
