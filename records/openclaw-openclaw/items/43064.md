## Summary

Current main still leaves heartbeat replies on the raw responsePrefix path, so the reported literal `{model}` behavior remains plausible. The issue also has an open same-author PR, #43065, with closing syntax and a focused implementation candidate, so it should stay open until that PR is reviewed, landed, or explicitly rejected.

## Best Possible Solution

Keep this issue open and review #43065 as the primary implementation candidate. The fix should wire heartbeat runs through the shared reply-prefix context path, pass onModelSelected to getReplyFromConfig, resolve the prefix after model selection, and add focused regression coverage for templated heartbeat alert prefixes plus prefixed HEARTBEAT_OK handling.
