## Summary

Current main still has the reported Control UI config usability gap. The Advanced Settings path uses the accordion layout, but the visible search control is only rendered for the non-accordion tab layout, and the underlying form filter still scopes by active section before applying search.

## Best Possible Solution

Keep this open and make a focused Control UI fix: expose search/filter controls in Advanced Settings accordion mode, make searches global across visible config sections instead of local to the selected group, and preserve the newer Quick Settings and scoped top-level tabs unless maintainers choose a broader v2 redesign.
