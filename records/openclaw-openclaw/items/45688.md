## Summary

Keep open: current main still has the reported Control UI config usability regression. Advanced Settings is still rendered through the accordion layout, that branch still lacks visible search/filter controls, and the form renderer still applies active-section scoping before search matching, so the v2 UI remains unable to search globally across the included Advanced Settings sections.

## What This Changes

The issue asks to restore v1-level Config usability in the v2 Control UI, especially a clear searchable/filterable settings list rather than only grouped top-level sections.

## Best Possible Solution

Implement a focused Control UI repair: Advanced Settings should expose search/filter controls in accordion mode, and nonempty searches should operate across the included Advanced Settings sections while preserving Quick Settings, raw mode, and the existing scoped settings tabs.

## Reproduction Assessment

Yes. Static inspection against current main is enough to reproduce the reported gap: Advanced Settings selects accordion layout, the accordion branch lacks the search input, and form filtering applies activeSection before search matching.

## Solution Assessment

Yes for a narrow repair. Restoring accordion-mode search/filter and making search global across the included Advanced Settings sections is the smallest maintainable fix; reverting the whole v2 grouping to the v1 flat list should remain a separate product decision.
