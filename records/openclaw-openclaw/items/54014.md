## Summary

Keep open. Current main partially honors OPENCLAW_HOME for state/config and agent defaults, but the reported broader contract is still not implemented for the dev/git checkout default, installer bootstrap workspace/config checks, and docs alignment. Related #65536 is adjacent sandbox tilde work, not a fix for these remaining gaps.

## What This Changes

The issue requests that OPENCLAW_HOME seed OpenClaw internal state, config, and git/install defaults unless OPENCLAW_STATE_DIR, OPENCLAW_CONFIG_PATH, or OPENCLAW_GIT_DIR are explicitly set.

## Best Possible Solution

Define the OPENCLAW_HOME contract as an effective home for internal OpenClaw defaults, update dev/git checkout and installer bootstrap defaults to derive from it unless specific path env vars are set, add focused tests, and align public docs with explicit OS-home exceptions.
