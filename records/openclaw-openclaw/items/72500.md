## Summary

Keep open. Current main still has the reported two-state Control UI behavior: the app renders the credentials gate whenever `connected` is false, and the gateway clears `connected` at reconnect start and on active socket close. PR #72522 remains open and unmerged with closing syntax for this issue, so the issue should remain open until that PR is reviewed and either merged or closed.

## What This Changes

The issue reports that Control UI WebChat briefly shows the full credentials gate during transient reconnects after tab refocus instead of preserving the authenticated chat UI.

## Best Possible Solution

Review PR #72522 as the active implementation candidate. The desired end state is to preserve the authenticated chat UI across transient disconnects while restoring the credentials gate on first connection, explicit auth failure, pairing failure, or identity failure, with focused coverage in the gateway/render state tests.
