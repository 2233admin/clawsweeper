## Summary

Keep open. Current main still has the reported Feishu topic mismatch: group tool-only prompting steers visible output to `message(action=send)`, while the Feishu message action handler only supplies `replyInThread` and a reply target for explicit `thread-reply`; the automatic inbound reply path is topic-aware, so the remaining bug is specifically tool-driven visible replies in `group_topic` and `group_topic_sender` sessions.

## What This Changes

The issue requests that Feishu group topic sessions post assistant visible replies back into the original topic thread instead of as plain group messages.

## Best Possible Solution

Add Feishu-owned topic-aware tool-delivery behavior so same-conversation visible sends from `group_topic` and `group_topic_sender` sessions carry the topic root into Feishu `reply_in_thread` transport, while ordinary group sends keep the existing top-level behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: trigger a Feishu `group_topic` session in `message_tool_only` mode, observe the prompt asks for `message(action=send)`, and verify the Feishu `send` action does not set `replyInThread` or a topic reply target.

## Solution Assessment

Yes. The narrow maintainable fix should stay at the Feishu/plugin threading boundary, likely by adding a Feishu threading tool context or action-resolution path; changing the generic group prompt for every channel would be broader and less robust.

Keep the fix Feishu-owned: add a Feishu threading/tool-context/action-resolution path so same-conversation visible sends from topic sessions use the topic root (`om_*`) and Feishu `reply_in_thread` semantics. Do not change generic group prompting for every channel unless it is only a small supplementary hint.
