## Summary

Keep open. Current main still matches the reported Feishu topic-session mismatch: tool-only group prompting directs visible output to `message(action=send)`, while Feishu only applies `replyInThread` and a reply target for explicit `thread-reply`; the automatic inbound reply path is topic-aware, so the remaining gap is specifically message-tool visible replies from `group_topic` and `group_topic_sender` sessions.

## What This Changes

The issue requests that Feishu visible replies from `group_topic` and `group_topic_sender` sessions post into the original Feishu/Lark topic thread instead of as plain group messages.

## Best Possible Solution

Add a Feishu-owned threading/tool-context or action-resolution path so same-conversation `send` calls from `group_topic` and `group_topic_sender` sessions carry the topic root (`om_*`) into Feishu `reply_in_thread` transport, while ordinary group sends remain top-level.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: run a Feishu `group_topic` turn in message-tool-only group delivery, observe the prompt asks for `message(action=send)`, and trace that Feishu `send` does not set `replyInThread` or a topic reply target.

## Solution Assessment

Yes. The narrow maintainable fix should stay at the Feishu/plugin threading boundary; changing the generic group prompt for every channel would be broader and would not guarantee Feishu transport-level topic delivery.
