## Summary

Keep open. Current main still matches the reported Feishu topic-session mismatch, and PR #75159 is an open implementation candidate that explicitly references this issue, so the issue should remain open until that PR is reviewed and either merged or superseded.

## What This Changes

The issue asks for Feishu `group_topic` and `group_topic_sender` sessions to post visible assistant replies back into the originating Feishu topic thread instead of the top-level group chat.

## Best Possible Solution

Review PR #75159 against the Feishu-owned topic/session boundary, then merge it or replace it with an equivalent Feishu plugin fix that makes topic-scoped visible replies use the originating topic root while preserving ordinary group sends.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: in a Feishu `group_topic` or `group_topic_sender` session, the shared group prompt steers visible output to `message(action=send)`, while the Feishu send handler only enables topic/thread delivery for `thread-reply`.

## Solution Assessment

Yes. The narrow maintainable direction is a Feishu-owned topic-aware reply fix rather than a broad generic group prompt change for every channel; the open linked PR follows that general boundary and needs normal review.
