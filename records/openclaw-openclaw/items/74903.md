## Summary

Keep open. Current main still has the reported Feishu topic/tool-send mismatch, and the issue is now explicitly covered by open closing PRs #75364 and #75159, so it should remain open until one of those fixes lands or is replaced.

## What This Changes

Feishu group_topic and group_topic_sender message-tool replies should stay in the native Feishu topic thread instead of being sent as top-level group messages.

## Best Possible Solution

Keep this issue open and land a Feishu-owned runtime fix that makes same-conversation message(action=send) from group_topic and group_topic_sender sessions carry the inbound topic root into Feishu reply_in_thread transport, while preserving plain group send behavior and adding focused channel action tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: in a Feishu group_topic message-tool-only turn, the prompt asks for message(action=send), and the Feishu action handler sends that action with no replyToMessageId and replyInThread false.

## Solution Assessment

Yes, the best direction is a narrow Feishu plugin runtime fix like #75364, because it fixes the delivery invariant even if the model follows the existing generic action=send prompt. A prompt-only nudge like #75159 is useful but more model-dependent.
