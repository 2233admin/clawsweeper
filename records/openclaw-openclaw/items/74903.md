## Summary

Keep open. Current main still has the reported Feishu topic mismatch: tool-only group prompting steers visible output to `message(action=send)`, while the Feishu `send` action only supplies topic/thread transport for explicit `thread-reply`; automatic inbound replies are topic-aware, so the remaining gap is specifically tool-driven visible replies in `group_topic` and `group_topic_sender` sessions.

## What This Changes

Feishu group-topic sessions should keep tool-driven visible assistant replies inside the native topic thread instead of sending them as top-level group messages.

## Best Possible Solution

Add Feishu-owned topic-aware tool delivery so same-conversation visible sends from `group_topic` and `group_topic_sender` carry the topic root into Feishu `reply_in_thread` transport, while ordinary group sends keep the existing top-level behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: trigger a Feishu `group_topic` session in tool-only visible-reply mode, observe the prompt asks for `message(action=send)`, and verify the Feishu `send` action does not set the topic reply target or `replyInThread`.

## Solution Assessment

Yes. The narrow maintainable fix should stay at the Feishu/plugin threading boundary, likely by adding Feishu topic-aware tool context or action resolution; changing the generic group prompt for every channel would be broader and less robust.
