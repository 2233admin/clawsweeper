## Summary

Keep open. Current main still has the reported Feishu topic mismatch: message-tool-only group prompting steers visible output to `message(action=send)`, while the Feishu plugin only enables native topic/thread delivery for explicit `thread-reply`. An open contributor PR, #75159, targets this bug but is not yet a merge-ready fix.

## What This Changes

The issue requests that Feishu `group_topic` and `group_topic_sender` message-tool replies post into the native topic thread instead of as top-level group messages.

## Best Possible Solution

Land a Feishu-owned, delivery-mode-aware fix, either by revising #75159 or replacing it with an equivalent narrow patch. The final behavior should make same-conversation topic message-tool sends use the topic root and `reply_in_thread`, while automatic Feishu topic replies and ordinary group sends keep their current routing.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: trigger a Feishu `group_topic` message-tool-only turn, follow the prompt to `message(action=send)`, then trace the Feishu action handler where `replyInThread` remains false unless the action is explicitly `thread-reply`.

## Solution Assessment

Yes for a Feishu-owned fix, but not for the currently open PR as written. The safer route is to revise the fix so topic-thread guidance or action resolution applies only when the message tool is responsible for visible delivery.
