## Summary

Keep open. Current main still has the reported Feishu topic mismatch: group tool-only prompting steers visible output to `message(action=send)`, while Feishu only sets thread reply transport for explicit `thread-reply`, so visible message-tool replies from topic-scoped sessions can leave the native topic thread.

## What This Changes

The issue requests that Feishu `group_topic` and `group_topic_sender` sessions preserve native topic threading when the assistant sends visible output through the message tool.

## Best Possible Solution

Add Feishu-owned topic-aware message-tool delivery so same-conversation `send` calls from `group_topic` and `group_topic_sender` sessions carry the topic root into Feishu `reply_in_thread` transport, while ordinary group sends keep top-level behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: trigger a Feishu topic-scoped group session in message-tool-only mode, observe the prompt asks for `message(action=send)`, and verify the Feishu `send` handler does not set `replyInThread` or a topic reply target.

## Solution Assessment

Yes. The narrow maintainable solution is to preserve Feishu topic context at the plugin/threading boundary or action-resolution path; changing the generic group prompt for all channels would be broader and less reliable.
