## Summary

Keep open. Current main still has the reported Feishu topic mismatch: tool-only group prompting steers visible output to `message(action=send)`, while Feishu `send` handling does not turn the current topic root into a threaded reply; only explicit `thread-reply` gets `replyInThread: true`.

## What This Changes

Feishu `group_topic` and `group_topic_sender` sessions should post tool-driven visible replies back into the triggering topic thread instead of sending a plain top-level group message.

## Best Possible Solution

Add Feishu-owned topic-aware message-tool delivery so same-conversation visible sends from `group_topic` and `group_topic_sender` sessions carry the topic root into Feishu reply transport with `reply_in_thread`, while non-topic group sends remain top-level.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: trigger a Feishu `group_topic` session in `message_tool_only` mode, observe the prompt asks for `message(action=send)`, then follow the Feishu send action path where no topic reply target or `replyInThread` flag is applied.

## Solution Assessment

Yes for keeping this open and fixing it at the Feishu/plugin threading boundary. A broad generic prompt change would be less maintainable than teaching Feishu `send` to resolve same-topic tool delivery through the existing threading context/action params.
