## Summary

Keep open. Current main still renders the Control UI Agents page as a single dropdown rather than a visible agent list/sidebar, and the Set Default action still stages `agents.defaultId`, while the current config contract and resolver use `agents.list[].default`. Related #61887 was closed as a duplicate in favor of this broader tracker, and #54178 remains an adjacent automation issue rather than a fix.

## Best Possible Solution

Keep this issue open. The best fix belongs in the Control UI Agents page: replace the desktop single dropdown with a visible agent list/sidebar or comparable low-friction navigation, keep a responsive mobile selector if needed, and change Set Default to persist through the supported `agents.list[].default` config contract with focused UI/controller tests.
