## Summary

Keep this issue open. Current main has useful token-efficiency mitigations, but the central requests remain active: workspace bootstrap files are still assembled from a fixed default set with coarse filtering/caps rather than declarative per-file loading conditions, and compaction summaries are still reintroduced as model-visible summary context rather than title-only/lazy expansions. The later production-use comment reinforces that conditional workspace loading is still needed, not obsolete.

## Best Possible Solution

Keep the issue open for maintainer triage, then split it into focused follow-ups: declarative per-bootstrap-file loading/filtering for default context assembly, lazy or bounded compaction-summary reintroduction, and any remaining provider-usage or marketplace-discovery gaps. Mark already-shipped mitigations with links to the current docs/code so future implementation work focuses on the unresolved core token-efficiency requests.
