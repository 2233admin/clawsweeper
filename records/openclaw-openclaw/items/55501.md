## Summary

This issue is a broad umbrella with several already-improved areas, but current main does not fully implement the central high-priority requests. The repository now has bootstrap caps, context inspection, heartbeat light-context options, cron retry/failure alerting, config schema output, and subagent metadata/tool-policy documentation. However, per-file conditional workspace loading, lazy/collapsible compaction summary injection, and per-check heartbeat intervals still do not appear to be implemented or superseded by a single canonical issue.

## Best Possible Solution

Keep this issue open for maintainer triage, then split it into focused follow-ups: one for declarative per-file bootstrap/context loading, one for lazy or bounded compaction summary injection, and one for per-check heartbeat scheduling if that remains desired. Mark the already-covered sub-items with links to the current docs/code, and treat plugin marketplace clarity as ClawHub/product-discovery work unless a specific OpenClaw plugin API gap is identified.
