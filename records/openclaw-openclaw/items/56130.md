## Summary

Keep open. Current main does not implement the visionOS node app, its docs, or the companion visionOS command-policy support. This remains a substantial implementation candidate, paired with same-author open PR #55944, and the provided review context plus current gateway contracts show protocol/security-sensitive issues that still need maintainer review before any cleanup close would be safe.

## Best Possible Solution

Keep this PR open for maintainer review. If OpenClaw wants a first-party visionOS node app, the best path is to resolve or merge the paired platform-policy work in #55944, fix the remaining node protocol/reconnect review findings against current gateway contracts, add the needed app/docs/labeler surfaces, and require native-app security plus real-device validation. If maintainers decide visionOS should live outside core, that should be an explicit product decision rather than an automated cleanup close.
