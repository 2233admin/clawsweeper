## Summary

Keep this PR open. Current main still lacks the proposed Pi tool adapter conformance harness, and the adapter still does not pre-check already-aborted execute signals before agent-tool hook execution or client-tool delegation. The PR remains relevant, but it should be revised before merge because it has unresolved abort-ordering review feedback, overlaps current client-tool parameter coercion, and imports a TypeBox package name that is not the repo-declared dependency.

## Best Possible Solution

Keep this PR open for contributor or maintainer revision. The right path is to rebase onto current main, preserve client-tool parameter coercion, add the adapter conformance harness, fix the abort-ordering issue, use the repo-declared TypeBox package import, and then run the focused Pi adapter tests plus the changed gate before merge consideration.
