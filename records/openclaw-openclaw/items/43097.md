## Summary

Keep this PR open. Current main still lacks the proposed Pi tool adapter conformance harness, and the adapter still does not pre-check already-aborted execute signals before agent-tool hook execution or client-tool delegation. The PR remains useful, but it needs revision before merge to address the client-tool abort-ordering review feedback, preserve current client-tool parameter coercion, and align the new test with the repo-declared TypeBox import.

## Best Possible Solution

Keep this PR open for contributor or maintainer revision. The best path is to rebase onto current main, add the adapter conformance harness, implement already-aborted signal handling with safe client-tool ordering, preserve client param coercion, align the TypeBox import with repo package metadata, and then run the focused Pi adapter tests plus the changed gate before merge consideration.
