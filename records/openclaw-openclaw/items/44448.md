## Summary

Keep open. Current main still does not implement the central request: resolving `openclaw agent --channel discord --to ...` by existing live channel/thread delivery context before falling back to the main session. The CLI still chooses a session key before channel/account/thread delivery metadata is considered, and the resolver does not accept or scan stored `deliveryContext` fields.

## Best Possible Solution

Implement or explicitly reject delivery-context session routing for `openclaw agent`. If implemented, normalize Discord raw numeric IDs and `channel:<id>` targets with the shared target parser, pass channel/account/thread context into session resolution, search stored sessions by `(channel, to, accountId, threadId)` before main fallback, preserve explicit selector precedence, and add regression tests for raw numeric and channel-prefixed Discord targets.
