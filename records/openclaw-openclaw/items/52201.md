## Summary

Keep this issue open. Current main still resolves normal route bindings to an agent and session only; `bindings[].workspace` is not typed, is rejected by the strict route-binding schema, is absent from generated config docs, and the reply/runtime workspace path is still selected from agent-level config. Related implementation PRs are closed unmerged, and the current ACP/spawned-run workspace overrides are narrower surfaces that do not satisfy normal per-conversation route binding workspace isolation.

## Best Possible Solution

Keep this open for maintainer product review and a fresh focused PR. If accepted, add a typed and documented `bindings[].workspace` field for normal route bindings, trim/validate it in schema and generated config docs, carry the matched override through route resolution and inbound message contexts, apply it consistently to bootstrap/context/memory/tool/plugin workspace selection, persist it only for outbound interactions that require continuity, clear it on reroute/rebind paths, and cover precedence plus stale-override behavior with targeted routing/channel tests.
