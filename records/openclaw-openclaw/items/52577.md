## Summary

Keep open. Current main has stronger Discord gateway readiness/reconnect handling and queues events after MESSAGE_CREATE is delivered, but it still does not fetch/replay recent Discord channel history after READY or reconnect. The prior ClawSweeper comment remains accurate: there have been no Discord monitor/source changes since that review, only CHANGELOG updates.

## Best Possible Solution

Keep this open and implement the fix inside the Discord plugin. The best path is a Discord-owned READY/reconnect catch-up pass: maintain per-account/channel last-seen cursors, fetch recent messages through Discord REST after READY/reconnect, dedupe against the existing inbound replay key shape, and feed eligible replayed events through the same preflight/createDiscordMessageHandler path as live MESSAGE_CREATE events. Add regression coverage for startup gap, runtime reconnect gap, dedupe with live delivery, and authorization-preserving replay.
