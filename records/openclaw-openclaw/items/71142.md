## Summary

Keep open. Current main has removed the old hardcoded 5 MB parser cap by routing chat uploads through `agents.defaults.mediaMaxMb`, but the requested Control UI upload policy is still incomplete: images still hit a separate 6 MiB server cap, the bootstrap contract does not expose an effective upload limit to the browser, the browser reads files before size validation, and the Control UI docs do not describe the effective upload or WebSocket/base64 ceilings. The discussion also has two contributors offering scoped plans and explicitly waiting on maintainer direction for the public config surface, so this should not be auto-closed.

## What This Changes

The issue requests a configurable Control UI upload-size limit so browser uploads can accept larger media consistently with agent media limits and channel behavior.

## Best Possible Solution

Implement a focused Control UI upload policy that aligns server image acceptance with the approved configured limit, exposes resolved raw-byte and practical WebSocket/base64 ceilings through the Control UI bootstrap contract, rejects oversize paste/file/drop inputs before `FileReader.readAsDataURL`, documents the behavior on `https://docs.openclaw.ai/web/control-ui`, and adds targeted Gateway/UI tests for default rejection, raised-limit acceptance, bootstrap exposure, browser-side rejection, and transport-boundary behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to configure `agents.defaults.mediaMaxMb` above 6, then submit a Control UI image larger than 6 MiB: `chat.send` uses the raised parser cap, but `parseMessageWithAttachments` still rejects images above `MAX_IMAGE_BYTES`, and the browser reads the file before any configured-size check.

## Solution Assessment

Unclear until a maintainer chooses the config surface. Reusing `agents.defaults.mediaMaxMb` is the narrowest current implementation path, but a Control UI-specific knob may be the better product boundary if browser uploads should differ from channel or agent media handling.
