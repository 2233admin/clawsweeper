## Summary

Keep this PR open. Current main still does not implement the requested Discord `channelEdit` `appliedTags` path: the edit payload type has no `appliedTags`, the shared channel-edit parser only reads `availableTags`, and the REST PATCH body never sends `applied_tags`. The PR remains a relevant implementation candidate, but it is stale against the current Discord plugin runtime layout and still needs empty-array preservation through dispatch, not only in the REST body guard.

## Best Possible Solution

Keep this PR open and request a current-main rebase. The best implementation is to add `appliedTags?: string[]` to `DiscordChannelEdit`, extend `readDiscordChannelEditParams` so camelCase and snake_case inputs reach both guild and guild-admin paths, preserve explicit `appliedTags: []` through parsing for tag clearing, map `appliedTags` to `applied_tags` in `editChannelDiscord`, and add focused tests for non-empty tag updates plus empty-array tag clearing.
