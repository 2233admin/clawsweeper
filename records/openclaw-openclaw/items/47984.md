## Summary

Keep this PR open. Current main still does not implement the requested Discord `channelEdit` `appliedTags` path: the edit type has `availableTags` only, the edit REST body maps `available_tags` only, and the current guild action parser forwards `readDiscordChannelEditParams` without applied tag parsing. The PR remains useful but needs a rebase onto the newer Discord plugin runtime layout and should preserve explicit empty arrays so agents can clear all thread tags.

## Best Possible Solution

Keep this PR open and ask for a rebase onto the current Discord plugin action runtime. The implementation should add `appliedTags` to `DiscordChannelEdit`, map it to Discord `applied_tags` in `editChannelDiscord`, parse it through the current `readDiscordChannelEditParams`/guild-admin paths, and add focused coverage for both non-empty tag updates and `appliedTags: []` tag clearing.
