## Summary

Keep open. Current main still supports only a single built-in compaction model override and has no `agents.defaults.compaction.fallbacks` or `summaryModelFallbacks` surface. The related fallback work confirms the generic chat fallback machinery exists, but it does not supersede this compaction/LCM-specific request.

## Best Possible Solution

Keep this issue open. A good fix would add a first-class compaction fallback contract in config types, schema/help, docs, and runtime, then thread the ordered candidates through built-in compaction with the same transient-failure semantics as chat fallback. For Lossless Claw, coordinate with the plugin or expose a generic context-engine fallback chain so plugin-owned summary models can opt in without hard-coding that plugin in core.

## Risks / Open Questions

- The LCM `summaryModel` surface appears owned by the external Lossless Claw plugin, so the right fix may require plugin changes, a generic context-engine runtime fallback field, or both.
- There is already model fallback behavior for normal agent turns and other model surfaces; the implementation should avoid accidentally applying chat fallbacks to explicit compaction overrides unless that is the chosen product contract.
