## Summary

Keep open. Current main still accepts explicit `x-openclaw-session-key` values unchanged and forwards them through the OpenAI-compatible and OpenResponses HTTP paths, so the PR’s reserved internal-session override guard has not shipped.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best implementation path is a narrow guard in the HTTP request-context/session-key path for `subagent:`, `acp:`, and `cron:` variants, preserving ordinary custom session keys and mapping only the dedicated reserved-session override error to a 400 `invalid_request_error` in the OpenAI-compatible and OpenResponses endpoints.
