## Summary

Keep this PR open. Current main still accepts explicit `x-openclaw-session-key` values unchanged in the OpenAI-compatible and OpenResponses HTTP request-context path, then forwards that value into agent command execution. The PR's intended guard is not implemented on main, and the tests on main still only cover normal custom session keys, not rejection of `subagent:`, `acp:`, or `cron:` overrides.

## Best Possible Solution

Keep the PR open for maintainer review or landing. The best path is to merge or recreate the narrow guard on current main for `resolveGatewayRequestContext` callers in the OpenAI-compatible and OpenResponses HTTP endpoints, preserve normal custom session keys, test `subagent:`, `acp:`, and `cron:` cases, and keep unexpected context/config errors on the internal-error path rather than exposing them as client input errors.
