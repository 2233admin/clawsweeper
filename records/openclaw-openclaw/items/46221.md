## Summary

Keep this PR open. Current main still drains formatted system events only from the active session key, while the documented and implemented wake/system-event paths still enqueue default events on the canonical main session. The PR is not obsolete, but its head patch still needs rework before landing because it drains the config-default main key for every non-main session, which can cross agent boundaries in multi-agent setups.

## Best Possible Solution

Keep this PR open for a focused rework. Implement the receiver-side merge in `src/auto-reply/reply/session-system-events.ts`, but derive the fallback main key from the active session's agent id using `resolveAgentIdFromSessionKey(params.sessionKey)` plus `resolveAgentMainSessionKey(...)`. Add regression tests for per-channel-peer delivery, all intended non-main scopes, and multi-agent isolation, then encode the intended scope in comments/tests before landing.
