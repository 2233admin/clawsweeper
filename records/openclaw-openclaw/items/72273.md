## Summary

Keep open. Current main still hardcodes the Discord gateway startup READY timeout to 15s and runtime READY timeout to 30s, while multi-account Discord startup is staggered by 10s per account. The existing Discord config/schema/docs expose `gatewayInfoTimeoutMs` for metadata lookup, but not the READY timeouts requested here.

## Best Possible Solution

Implement Discord plugin-owned READY timeout configurability on current main: add inherited `channels.discord.readyTimeoutMs` and `channels.discord.runtimeReadyTimeoutMs` fields with per-account overrides, optional env fallback only when config is unset, pass the resolved values into the lifecycle startup wait and runtime READY observer, and update schema/types/UI hints/generated metadata/docs plus focused lifecycle tests.
