## Summary

Keep open. Current main still sends local `infer model run --model` through the agent-aware simple-completion preparation path, and open PR #74028 is an active closing PR for the same narrow repair, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks for `openclaw infer model run --model github-copilot/<model>` to honor the requested Copilot provider/model instead of routing to the configured Anthropic default.

## Best Possible Solution

Review and land PR #74028 if it stays correct against current main, or close it with a replacement plan that keeps the same narrow boundary: qualified local model overrides should prepare the requested provider/model directly, while unqualified refs should continue using the existing agent-aware selection path.

## Solution Assessment

Yes. The narrow local-path split in PR #74028 is the best-shaped fix because it preserves unqualified agent selection and the already-explicit gateway forwarding behavior while adding focused CLI regression coverage.
