## Summary

Keep open. Current main still routes local `infer model run --model` through the agent-aware simple-completion preparation path, and open PR #74028 explicitly targets this same scoped bug with closing syntax; this issue should stay open until that PR lands or is closed.

## What This Changes

The issue requests that `openclaw infer model run --model <provider/model>` honor fully qualified Copilot model overrides instead of routing local probes to the default model.

## Best Possible Solution

Review PR #74028 and either land it after focused validation or close it with an equivalent replacement plan. The desired end state is a local `runModelRun` path that prepares fully qualified provider/model overrides directly while preserving the existing agent-aware selection behavior for unqualified refs.

Yes. The issue provides concrete commands, and the current-main code path can be traced from the CLI option into local `runModelRun`; I did not execute live Copilot calls during this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to special-case fully qualified local overrides and call the direct simple-completion preparation path only for that case, leaving gateway forwarding and unqualified agent-aware defaults unchanged.
