## Summary

Keep open. Current main still has the pre-fix local `infer model run` implementation, and the issue is paired with open PR #74028 via closing syntax, so this should stay open until that PR lands or is replaced.

## What This Changes

The issue asks for `openclaw infer model run --model <provider/model>` to honor fully qualified local Copilot model overrides instead of routing through the agent/default model path.

## Best Possible Solution

Review and land PR #74028, or close it with a replacement that keeps the same narrow end state: fully qualified local model-run overrides are prepared directly, unqualified refs keep using the agent-aware selection path, and gateway override forwarding stays explicit.

## Reproduction Assessment

Yes. The issue provides concrete live commands and metadata, and current source still shows the local `runModelRun` path using `prepareSimpleCompletionModelForAgent` for the supplied override; I did not run a live Copilot request in this read-only sweep.

## Solution Assessment

Yes. The active PR's direct-preparation split for qualified local overrides is the narrowest maintainable fix; case canonicalization and broader OpenRouter behavior are separate follow-ups in #73717 and #69527.
