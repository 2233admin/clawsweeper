## Summary

Keep open. The report is fresh and concrete, current main still sends local `infer model run --model` through the agent-aware simple-completion preparation path, and open PR #74028 explicitly targets this issue; the issue should stay open until that PR is merged or closed.

## What This Changes

The issue reports that local `openclaw infer model run --model <provider/model>` ignores GitHub Copilot model overrides and falls back to the configured Anthropic default instead of probing the requested model.

## Best Possible Solution

Keep the issue open until a focused fix lands that makes local `model run --model <provider/model>` prepare the requested provider/model directly, with regression coverage for GitHub Copilot refs and no behavior change for unqualified refs or gateway dispatch.

## Reproduction Assessment

Yes. The issue provides concrete CLI commands, the docs define full provider/model refs as a supported local smoke path, and current main still exposes the local preparation path that needs coverage; live provider confirmation requires Copilot auth.

## Solution Assessment

Yes. The narrow maintainable fix is to special-case fully qualified local overrides into direct provider/model preparation while leaving unqualified agent-default selection and gateway forwarding unchanged; PR #74028 appears to target that boundary.
