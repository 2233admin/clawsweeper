## Summary

Keep open. Current checked-out main still routes local `infer model run --model` through the agent-aware simple-completion preparation path, and the GitHub context shows open PR #74028 explicitly closes this issue with the focused repair, so the issue should remain open until that PR lands or is superseded.

## What This Changes

The issue reports that `openclaw infer model run --model <copilot-provider/model>` ignores the fully qualified Copilot override and routes local model probes to the default Anthropic model.

## Best Possible Solution

Review and either land PR #74028 or replace it with the same narrow shape: fully qualified local provider/model overrides should prepare that exact provider/model directly, while unqualified refs continue through the existing agent-aware default-selection path and gateway forwarding remains explicit.

## Reproduction Assessment

Yes. The reporter provided concrete commands and response metadata, and current source gives a high-confidence seam reproduction: local `infer model run --model github-copilot/...` enters `runModelRun` and still delegates the override through `prepareSimpleCompletionModelForAgent`; I did not execute a live Copilot call.

## Solution Assessment

Yes. The best fix is the focused local-path split described by the open linked PR, with regression tests in `src/cli/capability-cli.test.ts`; broader canonicalization or provider catalog behavior should stay in the related issues/PRs that already track those concerns.
