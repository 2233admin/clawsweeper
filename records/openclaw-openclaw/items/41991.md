## Summary

Keep this PR open. Current main still collapses Gemini CLI OAuth client-config extraction failures into the same generic setup error, while the PR is a focused diagnostics and regression-test update for the active bundled Google plugin. The related security issue #54289 could supersede this only if maintainers remove or gate automatic extraction; it has not made this diagnostics work obsolete yet.

## Best Possible Solution

Keep this PR open for maintainer review. If Gemini CLI OAuth credential extraction remains supported, land a narrow diagnostics fix in extensions/google/oauth.credentials.ts with resolveOAuthClientConfig() regression coverage for missing binary, missing credential source, parse failures, and read/extraction exceptions. If #54289 changes the product/security direction by removing or gating extraction, close this PR as superseded by that decision.
