## Summary

Keep open. Current main has absorbed some related command palette and slash-menu accessibility work, but it still does not implement several central pieces from this PR, including skip navigation, raw config editor labeling/height constraints, generated config segmented-control state, chat footer keyboard/touch visibility, chat rotor headings, pinned-list semantics, and sessions aria-sort. The PR is useful but broad and stale against a fast-moving Control UI surface, so it needs maintainer rebase/split judgment rather than cleanup closure.

This PR updates Control UI CSS and Lit components with screen-reader labels and semantics, keyboard focus affordances, raw config editor constraints, chat and command menu ARIA states, sessions sort announcements, and a Shift+Escape chat-focus shortcut.

## Best Possible Solution

Ship focused, rebased Control UI accessibility patches for the app shell, config form/raw editor, chat controls/message navigation, command surfaces, and sessions table, with DOM-level accessibility assertions where practical and a manual Safari/VoiceOver pass.
