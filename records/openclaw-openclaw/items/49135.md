## Summary

Keep this PR open. Current main still enables Telegram block streaming but does not expose `streaming.blockStreamingCoalesceDefaults`, while the shared resolver still reads those defaults from the loaded channel plugin before falling back to the global 800-character minimum. The PR remains a focused implementation candidate for the linked Telegram newline/message-splitting regression, and its second commit addressed the earlier review concern by adding `"streaming"` to the return `Pick` type.

## Best Possible Solution

Keep this PR open, rebase it onto current main, preserve the `streaming` return type addition, add `blockStreamingCoalesceDefaults: { minChars: 1500, idleMs: 1000 }` through `createTelegramPluginBase`, and add a small regression test before maintainer review/landing.
