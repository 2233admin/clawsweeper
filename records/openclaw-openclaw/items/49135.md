## Summary

Keep this PR open. Current main still does not implement the Telegram block-streaming coalesce default: Telegram’s plugin base does not expose a `streaming` adapter, while the shared block-streaming resolver still reads provider defaults from `plugin.streaming.blockStreamingCoalesceDefaults`. Comparable channels already set `{ minChars: 1500, idleMs: 1000 }`, so the PR remains a focused implementation candidate for the linked Telegram newline-splitting regression #47454.

## Best Possible Solution

Keep this PR open, rebase it onto current main, preserve the `streaming` return type addition, add `blockStreamingCoalesceDefaults: { minChars: 1500, idleMs: 1000 }` through `createTelegramPluginBase`, and add a targeted regression test before maintainer review/landing.
