## Summary

Keep open. The PR addresses a real current-main stale timestamp bug, but its proposed matcher introduces a concrete regression by replacing user-authored cron reminder lines that start with `Current time:`; the existing review comment is still valid and should be addressed before merge.

## What This Changes

The PR changes `appendCronStyleCurrentTimeLine` to refresh or collapse existing `Current time:` lines and adds colocated Vitest coverage for empty input, append, refresh, and duplicate-collapse cases.

## Best Possible Solution

Keep the shared-helper refresh boundary, but replace only OpenClaw-generated timestamp lines or add a backward-compatible internal marker, while preserving user-authored `Current time:` content in cron/reminder/workspace text.

## Reproduction Assessment

Yes. The original bug is reproducible by feeding a helper output with an old generated `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the PR regression is reproducible with cron reminder content containing a user-authored line like `Current time: check the server clock`.

## Solution Assessment

No. The helper-level approach is the right narrow fix, but the current PR matcher is too broad because it cannot distinguish OpenClaw’s injected timestamp from user-authored prompt content.

## Review Findings
