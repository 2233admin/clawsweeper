## Summary

Keep open: current main partially addresses the unsafe blind-switchback concern with sticky auto fallback state and per-run cooldown probes, but it does not implement the requested configurable background health-probe recovery that marks the primary healthy for future work and coordinates recovery notifications.

## What This Changes

Request health-probe-based recovery from fallback models back to the primary model for new sessions and subagents while existing sessions remain sticky on fallback.

## Best Possible Solution

Design a core fallback-recovery state machine that reuses existing auth/model probe contracts, keeps existing sessions pinned to their active fallback, routes new sessions and subagents to the primary only after a successful probe, and coordinates user-visible recovery messaging with #44941.
