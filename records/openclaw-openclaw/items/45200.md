## Summary

Keep open. The underlying silent give-up bug still appears valid on current main, but this patch should not merge as-is because it routes the last-resort notice through the gateway `agent` method, which starts a requester model turn and delivers that generated output instead of the recovered notice verbatim. The current implementation has also moved this lifecycle surface into split modules, so the useful idea needs a targeted port rather than applying the old monolithic-file change directly.

## What This Changes

The PR adds a `notifyAnnounceGiveUp` helper to send a best-effort requester notification after subagent announce retries are exhausted, and calls it from resumed and live retry-exhaustion cleanup paths.

## Best Possible Solution

Implement the give-up notice in the current lifecycle and announce-delivery layers using direct outbound `sendMessage` delivery when a deliverable requester origin exists, with focused tests for resumed and live retry exhaustion, missing origin, killed-run suppression, idempotency, and failed last-resort delivery logging.
