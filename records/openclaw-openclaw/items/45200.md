## Summary

Keep open. Current main still has the subagent announce give-up gap on the `sessions_spawn`/subagent cleanup surface: retry-limit paths log, mark task delivery failed, and complete cleanup without a user-visible recovered-result notification. The PR addresses a real bug from #44925, but the proposed fallback is not merge-ready because it routes through the gateway `agent` method, which starts another requester-session model turn rather than a raw outbound notice, and the branch also needs to be ported onto the current lifecycle split.

## What This Changes

The PR adds a last-resort `notifyAnnounceGiveUp` path in `src/agents/subagent-registry.ts` that attempts to notify the requester when subagent announce retries are exhausted.

## Best Possible Solution

Implement the last-resort give-up notice on current main in the lifecycle/announce-delivery layer, using a non-model outbound delivery path for the recovered result when a deliverable requester origin exists, while preserving swallowed failure semantics, killed-run suppression, idempotency, and task cleanup behavior.
