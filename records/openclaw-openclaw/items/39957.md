## Summary

Keep open. Current main still does not implement the PR's narrow guard: `handleMessageEnd` can emit `message_end` block replies for assistant `stopReason: "toolUse"` turns that contain structured tool-call blocks, and current provider conversion can still create that message shape. The PR remains a focused implementation candidate, though maintainers should keep the landed diff scoped to the handler/test change.

## Best Possible Solution

Keep this PR open and land this narrow suppression, or an equivalent current-main patch, in `handleMessageEnd`: suppress only `message_end` block replies for assistant turns with `stopReason === "toolUse"` and a structured tool-call block, while preserving normal `stop` turns, `toolUse` turns without tool-call blocks, and `text_end` block-reply behavior. Add/keep the regression cases from the PR, then run the targeted agent and Telegram tests plus the appropriate changed gate before merge.
