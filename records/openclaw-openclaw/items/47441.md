## Summary

Keep this issue open. Current main still depends on `sharp` for bundled media/image attachment resizing through `media-understanding-core`, while the locked `sharp@0.34.5` platform matrix has Linux/macOS/Windows/wasm prebuilts but no `android-arm64` runtime. The update path only retries npm installs with `--omit=optional`; it does not detect Termux, rebuild `sharp` with a Termux-native toolchain, or preserve raw image blocks when resize falls back to a missing `sharp` runtime.

## Best Possible Solution

Keep this open and fix the image path first: add a bounded graceful fallback for unsupported/missing `sharp` runtimes so ordinary image attachments are preserved when safe instead of being omitted, while leaving a broader Termux-native rebuild utility or update repair policy as a separate maintainer decision if still needed.
