## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: `QmdMemoryManager.search()` uses mcporter when enabled and then immediately normalizes whatever results came back, with no direct `qmd query` retry when mcporter returns an empty result set. The PR remains an active implementation candidate, but it has also grown broader than the memory-core fallback and needs maintainer review or splitting before merge.

## Best Possible Solution

Keep the PR open for maintainer review.
