## Summary

Keep this PR open. Current main still does not implement the proposed memory-core fallback: mcporter empty results flow straight into result normalization, and the only direct `qmd query` compatibility fallback is disabled when mcporter is enabled. The live PR is still open and its current diff adds the intended fallback plus regression tests, but it also includes broader plugin/browser/Signal/gateway-adjacent changes that need maintainer review or splitting before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to split or explicitly accept the non-memory-core changes, rebase/resolve the dirty state, then land the focused memory-core fallback with touched-surface validation if maintainers want the mcporter empty-result recovery behavior.
