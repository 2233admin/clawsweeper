## Summary

Keep this PR open. Current main still has the core behavior the PR is trying to change: wake-triggered heartbeats can resolve through the originating forced session before consulting configured `heartbeat.session`, and main does not contain the PR's cross-session event migration infrastructure or regression test file. The PR also has an unresolved data-isolation review concern around canonicalizing the source session before queue migration, so the best path is review/fix/land or replace with an equivalent maintainer patch, not close as obsolete.

## Best Possible Solution

Keep this PR open as an implementation candidate for #41165. The best maintainer path is to finish review by canonicalizing and agent-scoping the source session before any migration, keep the trusted/dedup/skip-restore protections from the latest PR discussion, and land focused regression tests for configured heartbeat session precedence plus cross-session event migration. After that merges, #41165 can close through the normal linked-issue flow.
