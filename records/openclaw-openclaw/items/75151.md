## Summary

Keep open. Current main still allows the active session store to be updated to a successor transcript path before that JSONL is durable, and the history reader still returns an empty list when none of the mapped transcript candidates exists.

## What This Changes

The issue requests that context-overflow/session-reset rotation never point `sessions.json` at a nonexistent transcript and instead preserve or recover usable session continuity.

## Best Possible Solution

Enforce a generic reset invariant in the embedded/direct session path: create or prewarm the successor JSONL before updating the active store, keep the previous valid mapping if successor creation fails, refresh queued follow-up state only after durability is established, and add focused regression coverage for missing, empty, assistant-only, and normal replay cases.

## Reproduction Assessment

Yes. A focused unit reproduction can reset a direct session whose prior transcript is missing or has no replayable user-starting tail, then assert the persisted successor `sessionFile` exists and `readSessionMessages` cannot silently lose continuity because the mapped file is absent.

## Solution Assessment

Yes. The narrow maintainable solution is to enforce transcript durability at the reset boundary; doctor/cleanup diagnostics are useful but too late for an active-session data-integrity invariant.
