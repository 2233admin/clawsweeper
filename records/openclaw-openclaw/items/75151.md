## Summary

Keep open. Current main still has the reported durability gap: the reset path can persist a new active session entry before the successor transcript exists, the replay helper can return without creating that transcript, and history reads return an empty list when no candidate file exists.

## What This Changes

The issue requests that context-overflow/session-reset rotation never update `sessions.json` to a nonexistent transcript and instead preserve or recover valid session continuity.

## Best Possible Solution

Add a generic reset invariant in the embedded/direct session path: create or prewarm the successor JSONL before updating the active store, keep the prior valid mapping if creation fails, and make history/recovery avoid silently returning empty continuity when a mapped successor transcript is absent.

## Reproduction Assessment

Yes. A focused unit reproduction can reset a direct session whose prior transcript is missing or has no replayable user-starting tail, then assert the persisted successor `sessionFile` exists and `readSessionMessages` does not silently lose continuity because the mapped file is absent.

## Solution Assessment

Yes. Enforcing transcript durability at the reset boundary is the narrowest maintainable fix; cleanup/doctor pruning remains useful diagnostics but is too late for an active-session data-integrity invariant.

Fix the session reset durability bug reported in https://github.com/openclaw/openclaw/issues/75151. In the embedded/direct reset path, do not persist a new active `sessionFile` in `sessions.json` unless the successor JSONL has been durably created or prewarmed. Preserve the previous valid mapping if successor creation/replay fails, and refresh queued follow-up state only after the successor file is durable. Keep the fix generic to reset/session infrastructure; do not add Telegram-specific policy. Add regression tests covering: missing previous transcript, empty/no-replayable previous transcript, assistant-only tail, and normal replay. Ensure `readSessionMessages` cannot silently return empty solely because reset mapped to a missing successor file.
