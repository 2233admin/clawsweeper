## Summary

Keep open. Current main still has the unsafe reset ordering and no durable-successor invariant: the active session store can be switched to a new transcript path before that JSONL exists, replay can no-op without creating the target file, and history lookup returns an empty list when no mapped candidate exists.

## What This Changes

The issue requests a data-integrity fix so context-overflow or compaction reset cannot leave `sessions.json` pointing at a nonexistent transcript while the real session history remains orphaned.

## Best Possible Solution

Enforce the reset invariant in the embedded/direct session path: create or prewarm the successor JSONL before updating the active store, keep the previous valid mapping if successor creation fails, refresh queued follow-up state only after durability is established, and add regression coverage for missing-source, empty-source, assistant-only-tail, and normal replay cases.

## Reproduction Assessment

Yes. A focused unit reproduction can reset a direct session whose prior transcript is missing or has no replayable user-starting tail, then assert the persisted successor `sessionFile` exists and `readSessionMessages` cannot silently lose continuity because the mapped file is absent.

## Solution Assessment

Yes. The narrow maintainable fix is to enforce transcript durability at the reset boundary; cleanup/doctor diagnostics are useful but too late for this active-session data-integrity invariant.

## Repair Work Prompt

Fix the session reset durability bug reported in https://github.com/openclaw/openclaw/issues/75151. In `resetReplyRunSession`, do not persist or publish a successor `sessionFile` until the successor transcript JSONL has been durably created/prewarmed. Update the replay helper or reset flow so missing, empty, and assistant-only source transcripts still produce a valid successor session header, or else keep the previous valid mapping and surface an explicit recovery warning. Refresh queued follow-up state only after the durable transcript invariant is established. Add focused regression tests for missing source, empty/no replayable source, assistant-only retained tail, and normal replay. Keep the PR scoped to this reset/replay/history durability invariant; do not implement broad transcript size caps or unrelated compaction policy. Add a user-facing `CHANGELOG.md` fix entry with `Thanks @MickeMG`.
