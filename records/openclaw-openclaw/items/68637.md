## Summary

Keep this PR open. Current `main` at `acae48b790fac67c3f46afae310bbba837f5e678` still rejects empty or whitespace-only `write.content`: the canonical `write` required-param group lacks `allowEmpty`, and the validator rejects blank strings unless that flag is set. The related issue was closed as superseded by this PR, and the older related PR was closed as superseded by this same current fix path, so this PR remains the active implementation candidate rather than obsolete.

## What This Changes

The PR adds `allowEmpty: true` to the `content` entry in `REQUIRED_PARAM_GROUPS.write` and adds focused tests for empty, newline-only, whitespace-only, missing, non-string, and missing-path write parameters.

## Best Possible Solution

Land this PR, or an equivalent narrow patch, so the write tool can create or truncate empty and whitespace-only files while still rejecting missing, null, undefined, non-string `content`, and missing `path`. If maintainers intentionally want empty writes rejected, the product contract should be made explicit in tests and user-facing error behavior instead of leaving the current retry-prone validation path.
