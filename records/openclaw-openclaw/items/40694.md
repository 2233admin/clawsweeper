## Summary

Keep this issue open. Current main has useful browser tab cleanup for idle/capped primary sessions and for lifecycle/reset paths, but it still does not close temporary primary-agent browser tabs at ordinary task or turn completion, and the tracked-tab model has no ephemeral versus preserved/user-owned marker.

## Best Possible Solution

Keep this issue open for a focused browser lifecycle change. Add task-scoped ephemeral/preserved metadata to tracked browser tabs, ensure all browser open paths participate, expose an explicit keep-open/debug preservation path, close only ephemeral primary-agent tabs on successful task or turn completion, preserve existing-session/user-owned/labeled or pinned tabs, and add regression coverage for primary completion cleanup, keep-open preservation, and existing-session safety.

## Risks / Open Questions

- A naive close-on-completion implementation could close existing-session or user-intended tabs unless the registry gains explicit ephemeral versus preserved/user-owned state.
- Current proxy-routed browser opens appear to bypass the direct tracking call, so a future completion cleanup path should verify all browser-opened temporary tabs are actually tracked.
- The UX needs a clear keep-open/debug preservation path before changing primary-session defaults, because docs currently describe primary tabs as reusable across activity until idle/cap cleanup.
