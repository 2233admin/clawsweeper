## Summary

Keep this issue open. Current main has useful browser tab cleanup for idle/cap sweeps, session reset/delete, isolated cron, and subagent lifecycle paths, but it still does not implement the requested default close-on-task-completion behavior for ordinary primary chat or agent browser work. The related closed #56271 is narrower and does not supersede this broader remaining gap.

## Best Possible Solution

Implement task-scoped browser tab ownership in the bundled browser plugin and gateway run lifecycle: mark automation-opened tabs as temporary or preserved, track direct and node/proxy opens consistently, run best-effort cleanup after successful primary chat/agent completion, preserve explicit keep-open/debug/user-owned tabs, and add focused regression tests plus docs/config updates.

## Risks / Open Questions

- A naive completion hook could close user-owned, existing-session, labeled, debug, or intentionally preserved browser tabs unless the registry first gains explicit temporary/preserved ownership semantics.
- Proxy-routed browser opens currently bypass direct registry tracking, so any completion cleanup implementation would be incomplete unless node/proxy opens are tracked consistently.
