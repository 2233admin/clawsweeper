## Summary

Keep open. Current main has useful cleanup for idle/excess primary tabs plus lifecycle cleanup for session reset, cron, ACP, and subagent sessions, but it still does not close temporary primary-agent browser tabs/windows at ordinary task or turn completion.

## Best Possible Solution

Keep this issue open for a focused browser lifecycle change. Add task-scoped ephemeral versus preserved/user-owned metadata to tracked tabs, expose an explicit keep-open/debug preservation path, close only ephemeral primary-agent tabs on successful task or turn completion, preserve existing-session/user-owned/labeled or pinned tabs, and keep the current idle/excess sweep plus subagent/cron/session-reset cleanup as fallback. Add regression coverage for primary-agent completion cleanup, keep-open preservation, and existing-session safety.

## Risks / Open Questions

- A naive close-on-completion implementation could close tabs users intended to reuse across turns unless the tracked tab model distinguishes ephemeral automation tabs from preserved or user-owned tabs.
- Existing-session and attach-only browser profiles can point at a user's running browser, so completion cleanup must avoid closing manually opened or explicitly preserved tabs.
