## Summary

Current main still does not implement automatic close-on-task-completion for ordinary primary browser work. It has tracked tab cleanup for idle/cap sweeps and lifecycle teardown paths such as subagents, isolated cron jobs, session reset/delete, and some reply-session rollover, but primary chat and agent completion still finish without closing browser tabs opened during the task. The related prior PR #43666 closed unmerged, and the related issue #56271 is closed without replacing the remaining primary-session gap.

## Best Possible Solution

Keep this issue open for maintainer judgment on the primary-session default. The best implementation is task-scoped browser tab ownership in the bundled browser plugin and gateway run lifecycle: mark automation-opened tabs as temporary or preserved, track direct and node/proxy opens consistently, run best-effort cleanup after successful primary chat/agent completion when appropriate, preserve explicit keep-open/debug/user-owned tabs, and update focused tests plus browser docs/config.

## Risks / Open Questions

- Changing the primary-session default from reusable tabs to close-on-completion is a product behavior change and may surprise users who rely on active tabs persisting after a chat turn.
- The implementation needs a clear preserved/user-owned/debug tab semantic before broad automatic closing, otherwise it can close tabs the user expected to keep.
- Node/proxy browser opens are not tracked through the same local path as direct opens, so a fix limited to direct tabs would leave inconsistent cleanup behavior.
