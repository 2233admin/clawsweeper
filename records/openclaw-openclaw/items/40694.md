## Summary

Keep open. Current main has idle/excess cleanup for tracked primary-agent browser tabs and lifecycle cleanup for session reset, cron, ACP, and subagent sessions, but it still does not close temporary primary-agent tabs/windows at ordinary task or turn completion. The tracked tab model also has no ephemeral/preserved/user-owned metadata, so the requested default behavior remains unimplemented.

## Best Possible Solution

Keep this issue open for a focused browser lifecycle change in the bundled browser plugin plus the agent turn lifecycle. Add task-scoped ephemeral versus preserved/user-owned metadata to tracked tabs, expose an explicit keep-open/debug preservation path, close only ephemeral primary-agent tabs on successful task or turn completion, preserve existing-session/user-owned/labeled or pinned tabs, and add regression coverage for primary completion cleanup, keep-open preservation, and existing-session safety.

## Risks / Open Questions

- A naive close-on-completion change could close tabs users intended to reuse across turns unless the tracked tab model distinguishes ephemeral automation tabs from preserved or user-owned tabs.
- Existing-session and attach-only browser profiles can point at a user's running browser, so completion cleanup needs an explicit preservation/debug path and should avoid manually opened or pinned tabs.
- Current tests cover idle/cap sweep and lifecycle cleanup for reset/subagent/cron paths, but not primary-agent successful task or turn completion cleanup.
