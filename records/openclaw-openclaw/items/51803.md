## Summary

Keep open. Current main still does not implement durable followup-queue persistence or drain-window inbound-message replay, and the linked bug remains open. This PR is still an implementation candidate, but the provided diff needs maintainer review because it bundles unrelated UI work and includes a committed runtime log artifact plus functional replay risks around routing, wakeup, media/command context, queue semantics, and consecutive restarts.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to remove the committed log artifact, split the unrelated /model command fix, and implement durable restart recovery through normal followup/dispatch semantics that preserve routing, untrusted user content, media and command context, overflow summary state, queue limits, wakeup behavior, malformed persisted-data handling, and consecutive restarts. If maintainers choose #51623 or another PR as canonical, close this PR as superseded only after that direction is explicit.
