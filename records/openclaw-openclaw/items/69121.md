## Summary

Keep #69121 open. Current main still has the pre-restart deferral path, including pending reply counts before SIGUSR1/restart shutdown, but it does not implement this PR's second-stage bounded drain inside `server.close()` or its active chat-run abort behavior. The provided PR discussion shows recent, targeted contributor work for a real gateway restart correctness gap, not an obsolete or superseded change.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrow gateway fix that passes the remaining restart drain budget into `server.close()`, drains pending reply dispatchers and active chat runs before sockets/channels are torn down during restart shutdown, aborts still-active chat runs when the bounded budget is exhausted, and keeps unrelated qa-lab changes out unless they are required for the regression scenario.
