## Summary

Current main at 6186ed2c074be70928ca4351ca6fc2fb87e8a142 still does not implement this PR's intended close-time restart drain. Main has pre-restart deferral that counts pending replies, but `runGatewayLoop` still calls `server.close()` with only reason/restart metadata, `GatewayServer.close` has no drain-budget option, and `createGatewayCloseHandler` does not poll pending replies or abort active chat runs before socket/channel teardown. The open PR remains an active implementation candidate, not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review as the active implementation candidate. The best path is a narrow gateway fix that passes the remaining restart drain budget into `server.close()`, drains pending reply dispatchers and active chat runs before socket/channel teardown, aborts remaining active chat runs when the bounded budget is exhausted including zero-budget restarts, preserves normal stop semantics, and either removes or justifies the qa-lab mock-provider changes.
