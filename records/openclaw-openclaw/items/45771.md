## Summary

Keep open. Current main has adjacent provider usage-window visibility and subagent fan-out safety limits, but it still does not implement the requested pace-aware autonomous-agent policy: no Anthropic inference response-header capture, no projected pace tiers, no `agents.defaults.pace` config, and no budget-aware prompt/throttle behavior for subagents or other non-interactive work.

## What This Changes

The issue requests built-in pace-aware rate-limit tracking, status/session visibility, and autonomous-agent throttling based on Anthropic 5-hour and 7-day usage windows.

## Best Possible Solution

Define a provider-aware pace design before implementation: usage sources and freshness, per-profile persisted windows, observed request-cost calibration, structured status/session output, agent prompt context, notifications, and explicit subagent/non-interactive enforcement rules without hard-coding an Anthropic-only policy into unrelated core paths.
