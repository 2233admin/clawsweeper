## Summary

Keep this PR open. Current main still has the false plugin allowlist diagnostic path for arbitrary missing primary commands, but the branch is based on the older run-main layout and needs to be adapted to the current run-main-policy split while preserving manifest command-alias diagnostics.

## What This Changes

The PR branch makes the missing-plugin CLI diagnostic async, probes bundled plugin CLI metadata with a force-enabled detection config, updates the runCli call site, and adds regression coverage for unknown roots, aliased CLI roots, denylist-neutral probing, and probe failures.

## Best Possible Solution

Land one canonical CLI diagnostic fix that updates src/cli/run-main-policy.ts so plugins.allow and plugins.entries.<id>.enabled=false messages are emitted only for real bundled plugin CLI roots or known manifest command aliases, preserves runtime slash-command and parent-plugin guidance, returns null for unknown roots such as run, and adds focused regression coverage.

Keep the real-root check additive to the alias handling.
