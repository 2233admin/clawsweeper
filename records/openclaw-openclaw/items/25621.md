## Summary

Keep open. Current main still throws for OpenBSD and other platforms outside darwin/linux/win32 in `resolveGatewayService()`, and both `openclaw status` and `openclaw gateway status` resolve that service before their status degradation paths can render. The related implementation PR #25690 was closed unmerged, so the requested fix remains actionable.

## What This Changes

The issue asks for unsupported platforms such as OpenBSD to keep `openclaw status` and related status commands usable instead of crashing when managed gateway service installation is unavailable.

## Best Possible Solution

Land a narrow gateway service/status fix: either return a conservative unsupported-platform `GatewayService` adapter whose read-only methods report safe unknown/not-installed state while mutating methods reject clearly, or guard status-only call sites before resolving managed service adapters. Cover both `openclaw status` and `openclaw gateway status` on an unsupported `process.platform`, and do not add OpenBSD service-manager support unless that is designed separately.
