## Summary

Keep open. Current main still supports managed Gateway service adapters only for macOS, Linux, and Windows, and unsupported platforms still throw from `resolveGatewayService()`. Both top-level `openclaw status` and `openclaw gateway status` reach that resolver before their degraded status paths can render. The related PR #25690 is useful prior art but is closed and unmerged, so it does not supersede this issue.

## Best Possible Solution

Land a narrow Gateway service/status fix: either return a conservative unsupported-platform `GatewayService` adapter or guard read-only status paths so unsupported platforms can render degraded diagnostics, while managed lifecycle operations still fail with clear unsupported-platform errors. Add regression coverage for unsupported-platform status behavior and update docs/changelog if the visible behavior changes.
