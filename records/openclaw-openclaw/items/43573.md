## Summary

Keep open: current main still has the reported ACP history gap, and this PR is useful source work but not merge-ready because it can overstate task delivery and mixes workflow/security-policy changes into the ACP fix.

## What This Changes

The PR seeds ACP `sessions_spawn` transcripts with the initial task, forwards provenance into the first ACP dispatch, dedupes the replayed prompt, adjusts ACP prompt-only transcript initialization, adds tests, updates the changelog, and changes CI/labeler workflows.

## Best Possible Solution

Land a narrow, rebased ACP-only fix that writes the initial spawn task to the child transcript before returning success, dedupes only the seeded first replay by provenance or marker, reports delivery truthfully, preserves current main's newer ACP spawn behavior, and moves CI/labeler policy changes to a separate maintainer-reviewed PR if still needed.

## Reproduction Assessment

Yes. Spawn an ACP session with `sessions_spawn(runtime: "acp", task: ...)`, inspect `sessions_history` before the first assistant reply, and current main’s code shows history reads the transcript file while ACP spawn only dispatches the task and persists the turn later.

## Solution Assessment

No as-is. The seed/dedupe direction is the right product fix, but this patch should be rebased, should not claim delivery when seeding failed or was skipped, and should split unrelated workflow/security policy changes.
