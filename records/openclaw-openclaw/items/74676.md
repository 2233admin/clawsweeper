## Summary

Keep open. Current main still wires the plugin API resolver to user-path resolution instead of the plugin root while the public SDK docs promise root-relative behavior, and plugin services can observe that same captured API during later startup. There is also an open same-author closing PR for this bug, so the issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

The issue asks OpenClaw to make `api.resolvePath('.')` usable during extension plugin service startup by resolving relative paths against the plugin install root instead of returning an unusable value.

## Best Possible Solution

Review and land PR #74718 or an equivalent narrow fix that roots relative `api.resolvePath` values under `record.rootDir` while preserving existing absolute, home-relative, and empty-input behavior; then close this issue after the implementation is merged.
