## Summary

Keep open. The underlying Discord reset race is still not implemented on current main, but this PR branch has blocking one-shot behavior bugs and drops newer active-session-id resolution, so it needs maintainer consolidation with the overlapping reset-window work before merge.

## What This Changes

The PR adds a persisted `postRotationStartupUntilMs` session window and uses it to steer Discord post-reset queued owner messages in `agent-runner.ts`, `session.ts`, and `SessionEntry` typing.

## Best Possible Solution

Consolidate this PR with #49001 into one durable Discord reset-window implementation that applies only to the first qualifying owner turn, clears after both normal and embedded-queue consumption paths, preserves active session-id resolution, and includes focused channel and thread reset regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Clear the window only after an owner steer is consumed:** `src/auto-reply/reply/agent-runner.ts:850-857`
  - body: `startedRun` is true for any run, but the startup steer only applies to owner Discord messages. If non-owner channel traffic starts a run before the owner's first post-reset turn, this clears `postRotationStartupUntilMs` and the owner never receives the recovery prompt.
  - confidence: 0.88
- **[P2] Clear the window before returning from embedded steer enqueue:** `src/auto-reply/reply/agent-runner.ts:273-277`
  - body: When `queueEmbeddedPiMessage` succeeds, this early return exits before the new clear path runs. The marker remains active, so later owner messages inside the window can be wrapped again instead of preserving one-shot behavior.
  - confidence: 0.86
- **[P2] Preserve the active session id when queueing the steer prompt:** `src/auto-reply/reply/agent-runner.ts:273`
  - body: Current main resolves the active run id through `replyRunRegistry.resolveSessionId(sessionKey)` before queueing an embedded steer. This branch queues with `followupRun.run.sessionId`, which can point at a stale pre-reset id after session rotation.
  - confidence: 0.74
