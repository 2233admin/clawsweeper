## Summary

Keep this PR open. Current `main` still has the narrow Telegram safe-send retry classifier that only accepts pre-connect error codes, while the PR targets an open, linked Telegram delivery bug where grammY emits the plain `Network request for '<method>' failed!` envelope around a fetch-level failure. The diff is small and scoped to Telegram retry classification plus tests, but it needs normal maintainer review because broadening retries for non-idempotent sends has duplicate-delivery safety implications.

## Best Possible Solution

Keep this PR open for normal maintainer review against #74203. The best path is to land a narrowly scoped Telegram helper/test change if maintainers agree the plain grammY envelope plus nested fetch-level failure is safe for bounded non-idempotent send retry, while preserving the existing strict guard against broad network-message retries.
