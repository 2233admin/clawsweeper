## Summary

Keep open. Current `main` does not implement Anthropic native web search, and the remaining work belongs in the existing Anthropic provider rather than being auto-closed. The PR also still needs functional and security cleanup before it could be merged.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best implementation path is provider-owned work in `extensions/anthropic`: gate native web-search injection on `tools.web.search.enabled` and explicit/auto Anthropic provider selection, pass configured Anthropic search options into the payload builder, suppress the managed `web_search` tool only when native search is active, add manifest/config/docs/onboarding coverage, and remove unrelated log artifacts. If maintainers decide this must be externalized instead, the missing generic SDK seam should be designed first rather than closing this PR as ClawHub-ready.

## Risks / Open Questions

- The latest PR head appears to inject Anthropic server web search for every direct Anthropic request, regardless of `tools.web.search.enabled` or the selected search provider, which could trigger unexpected outbound searches and costs.
- The latest PR head constructs the wrapper without passing search config, so requested domain filters, `maxUses`, `toolVersion`, and `userLocation` would be silently ignored.
- The PR includes unrelated OpenClaw log files; these should be removed and audited for secrets or local operational data before review continues.
- A separate related issue, #49949, remains open and tracks the same user request, so closing this PR as implemented or obsolete would lose a still-unshipped implementation candidate.
