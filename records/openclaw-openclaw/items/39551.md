## Summary

Keep this PR open. Current main still rejects `sessions_send` calls that include both `sessionKey` and `label`, which is the behavior this PR is meant to fix. The PR is also the active maintainer-backed implementation candidate after a narrow repair was pushed to its branch, so it should be reviewed and landed or replaced by a merged canonical patch before cleanup.

## Best Possible Solution

Review and land this PR, or close it only after maintainers merge a different canonical patch. The best implementation is a narrow `sessions_send` selector precedence rule: normalize an explicit non-empty `sessionKey` or accepted session ID as authoritative, ignore redundant `label` and `agentId` for lookup, preserve label-only behavior and downstream visibility/A2A checks, update the tool description and changelog, and keep regression coverage proving mixed inputs use the exact-key target without calling `sessions.resolve`. After the canonical patch lands, close or supersede the overlapping open PRs for the same bug.
