Keep this PR open. Current main still rejects `sessions_send` calls that include both `sessionKey` and `label`, which is the bug this PR is meant to fix. The latest PR context also shows a maintainer-backed narrow repair on the contributor branch, marked for human review and merge-ready, so the right cleanup path is review/land this canonical PR rather than close it as obsolete.

## Best Possible Solution

Review and land this PR, or close it only if maintainers pick and merge a different canonical patch first. The best implementation is the narrow `sessions_send` selector precedence rule: normalize an explicit non-empty `sessionKey` or accepted session ID as authoritative, ignore redundant `label` and `agentId` for lookup, preserve label-only behavior and downstream visibility/A2A checks, update the tool description/changelog, and keep regression coverage proving mixed inputs use the exact-key target without calling `sessions.resolve`.
