## Summary

Current main respects `contextPruning.mode: "off"` for Anthropic smart defaults and documents that pruning is separate from compaction, but the remaining user problem is not implemented or proved obsolete: public config still has no `compaction.mode: "off"` or hard-limit-only mode, and there is no current regression covering the reported Anthropic API-key + Claude Opus 4.6 + 1M-context low-usage compaction case. Related #73839 is open and targets #73003, not this issue.

## Best Possible Solution

Keep this open as a focused config/runtime follow-up: decide whether OpenClaw should support a documented `compaction.mode: "off"` or hard-limit-only mode, add regression coverage for the Anthropic API-key + Claude Opus 4.6/1M context + large reserve-token setup, and align docs so `contextPruning` and compaction controls cannot be confused.
