## Summary

Current main still only supports `${VAR}` config substitution, not `${VAR:-default}` fallbacks. This PR is a focused implementation candidate and should not be closed as obsolete or implemented-on-main.

## Best Possible Solution

Keep this PR open. The best path is to review and land a scoped implementation that updates `src/config/env-substitution.ts`, `src/config/env-preserve.ts`, config-env chaining behavior if needed, regression tests, and public docs for `${VAR:-default}` semantics. It is not implemented on current main and is not superseded by a canonical target-repo item in the provided context.

## Evidence
