## Summary

Keep open. Current main still does not implement this PR's explicit global/per-agent `tools.exec.shell` config contract, and the proposed diff remains incomplete for PTY supervision and gateway approval follow-up execution. The work is plausible, but it should be reviewed or consolidated with the canonical open design issue #49931 rather than closed as shipped.

## Best Possible Solution

Keep this PR open for maintainer review or explicitly consolidate it into #49931. A complete implementation should define the supported shell override contract, add typed global and per-agent config plus strict schema/docs/help, propagate the resolved shell consistently through direct exec, PTY supervision, gateway approval follow-up runs, and sandbox/node policy boundaries where applicable, and add focused tests for precedence, fallback, Windows argument style, validation, approval paths, and config/docs drift.
