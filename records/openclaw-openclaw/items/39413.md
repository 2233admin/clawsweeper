## Summary

Current main still does not implement the PR's explicit `tools.exec.shell` global/per-agent config contract. The PR remains a plausible contribution, but it is incomplete for PTY supervision and gateway approval follow-up runs, and the related open design issue #49931 is still the better anchor for the remaining product/security decisions.

## Best Possible Solution

Keep this PR open for maintainer review or explicitly consolidate it into #49931. A complete implementation should define the supported shell override contract, add typed global and per-agent config plus strict schema/docs/help, propagate the resolved shell consistently through direct exec, PTY supervision, gateway approval follow-up runs, and node/sandbox policy boundaries where applicable, and add focused tests for precedence, fallback, Windows argument style, validation, approval paths, and config/docs drift.
