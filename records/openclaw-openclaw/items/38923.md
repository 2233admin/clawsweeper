## Summary

Keep open. Current main still does not satisfy the PR’s intended fix for Markdown backtick-wrapped `IDENTITY.md` avatar values, and the Windows namespace/drive-letter containment gap remains relevant. The PR has useful targeted work, but it is not ready as-is because it broadens into unrelated `onPayload` wrapper casts and, per the supplied diff, replaces shared avatar policy in a way that risks losing max-size and policy consistency.

## What This Changes

The PR changes agent identity parsing and avatar resolution/tests to support workspace `IDENTITY.md` avatars and backtick-wrapped values for `/avatar/{agentId}`, adds a changelog entry, and applies unrelated `onPayload` type casts across agent runner/provider wrappers.

## Best Possible Solution

Land a narrow avatar-only fix: unwrap exactly single-backtick Markdown values in `parseIdentityMarkdown`, normalize Windows namespace/drive-letter comparisons in shared avatar policy before containment checks, keep `AVATAR_MAX_BYTES` and `resolveSafeAvatarFile` behavior aligned, add parser/resolver/control-ui policy regression tests, and remove or split the unrelated `onPayload` casts.
