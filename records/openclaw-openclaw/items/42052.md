## Summary

Keep this PR open. Current main still has the reported backup output-path behavior, and the PR's useful fix is stale against the current implementation location: backup creation now resolves output paths in `src/infra/backup-create.ts`, while the PR patch edits the older command wrapper path.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase the useful output-path behavior into `src/infra/backup-create.ts`: treat missing extensionless output paths as destination directories, keep existing files and non-empty suffixes as explicit archive filenames, preserve `.tar`/custom suffix expectations from the PR discussion, rethrow non-ENOENT `fs.stat` errors, update `docs.openclaw.ai/cli/backup`/CLI help to document the classification, and add focused regression tests for extensionless missing paths, explicit archive filenames, `.tar`, `.bak`, and stat-error handling.
