## Summary

Current main still has the reported backup output-path behavior: a missing extensionless `--output` path is returned as the archive filename instead of being treated as a destination directory. The PR is not obsolete, but its patch targets the old command-file layout and needs to be rebased into `src/infra/backup-create.ts` with focused tests/docs.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase the useful output-path behavior onto `src/infra/backup-create.ts`, preserve the existing containment and no-overwrite protections, document the `--output` classification in CLI help and https://docs.openclaw.ai/cli/backup, and add focused regression tests for missing extensionless directories, explicit archive filenames, `.tar`, custom suffixes such as `.bak`, and non-ENOENT `fs.stat` errors.
