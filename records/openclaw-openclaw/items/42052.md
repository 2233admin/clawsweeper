## Summary

Keep this PR open. Current main still treats a missing extensionless backup `--output` path as the final archive filename, and the PR’s useful patch is stale against the active implementation location: backup output resolution now lives in `src/infra/backup-create.ts`, while the PR changes `src/commands/backup.ts`.

## Best Possible Solution

Rebase the useful behavior into `src/infra/backup-create.ts`: treat missing extensionless output paths as destination directories, keep existing files and non-empty suffixes such as `.tar` and `.bak` as explicit archive filenames, rethrow non-ENOENT stat errors, update `docs/cli/backup.md` and CLI help to document the classification, and add focused tests for the directory/file heuristic and stat-error handling.

- Applying this PR as-is would likely be ineffective because it patches the old command wrapper instead of the current infra resolver.

- Security review: the PR does not touch workflows, dependencies, lockfiles, release scripts, permissions, or secret handling. The relevant security-sensitive surface is filesystem destination selection for backups, which needs focused regression coverage before merge.
