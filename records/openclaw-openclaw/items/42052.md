## Summary

Keep open. Current main still treats a missing extensionless backup output path as the final archive filename, while this PR’s patch targets the old `src/commands/backup.ts` resolver that no longer exists on main. The linked issue was closed for the separate atomic temp-file stall behavior, not for this output directory heuristic.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase the intended behavior into `src/infra/backup-create.ts`: treat missing extensionless output paths as destination directories, preserve existing files and non-empty suffixes such as `.tar` and `.bak` as explicit archive filenames, rethrow non-ENOENT `fs.stat` errors, add focused resolver regression tests, and update the CLI docs/help if this heuristic is product policy.
