## Summary

Keep open. Current main has a bundled Xiaomi provider, but it only wires provider catalog/auth usage plus TTS; it does not expose Xiaomi webSearch settings or append Xiaomi's provider-native web_search tool. The reporter's live follow-up and Xiaomi's public docs make this a concrete, narrow Xiaomi provider-plugin feature rather than stale cleanup.

## What This Changes

Add Xiaomi MiMo provider-native web_search support, including configuration and request payload passthrough for supported Xiaomi chat models.

## Best Possible Solution

Add Xiaomi-owned provider-plugin support: declare plugin-scoped webSearch settings, gate native tool injection to Xiaomi's official OpenAI-compatible endpoint and supported chat models, append the native web_search tool without dropping existing function tools, document activation/billing/key behavior, and add focused Xiaomi provider tests.

## Reproduction Assessment

Not applicable as a bug reproduction, but there is a high-confidence absence check: current main has no Xiaomi webSearch config schema and no Xiaomi runtime payload hook for adding the native web_search tool. Xiaomi's public docs and the reporter's live example define the expected request and response behavior.

## Solution Assessment

Yes. The narrow maintainable solution is provider-owned Xiaomi support through the existing manifest/config and wrapStreamFn request-patching surfaces; a broader core abstraction should wait until this hook proves insufficient or multiple providers need the same generic native-tool contract.
