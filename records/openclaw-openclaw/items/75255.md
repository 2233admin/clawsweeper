## Summary

Keep open. Current main has a bundled Xiaomi provider, but it still only covers provider auth/catalog metadata and Xiaomi TTS; it does not declare Xiaomi webSearch config or append Xiaomi's provider-native web_search tool to Xiaomi chat-completions requests. The reporter supplied concrete live API details, and the related-context matches are about other providers or surfaces rather than a canonical Xiaomi fix.

## What This Changes

The issue requests Xiaomi MiMo provider-native web_search support so OpenClaw can append Xiaomi's native web_search tool configuration to Xiaomi chat-completions requests when enabled.

## Best Possible Solution

Add the feature inside the Xiaomi plugin: declare plugin-scoped webSearch settings, gate native tool injection to supported Xiaomi MiMo chat models and official endpoint/key behavior, append Xiaomi's native web_search tool while preserving ordinary function tools, document activation/billing/annotation behavior, and cover the request payload path with focused Xiaomi provider tests.
