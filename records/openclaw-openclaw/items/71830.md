## Summary

Keep this PR open. Current main still lacks the central WhatsApp behavior: XML-style tool/function-call blocks are not sanitized on the WhatsApp outbound paths, there is no `channels.whatsapp.exposeErrorText` config surface, and text-only error payloads are not routed through channel `sendPayload` for WhatsApp-owned suppression. The linked production report #71715 is closed, but this PR remains the active implementation candidate rather than obsolete.

## Best Possible Solution

Keep this PR open for normal maintainer review and landing, or replace it with an equivalent focused patch that sanitizes WhatsApp-visible XML tool/function-call payloads across outbound, auto-reply, and sendPayload paths; adds documented `channels.whatsapp.exposeErrorText` with account inheritance; and treats suppressed WhatsApp error payloads as no-op deliveries without false message-sent bookkeeping.
