## Summary

Keep this PR open. Current main does not implement `sandbox.docker.volumes`: the public sandbox Docker type, Zod schema, generated config schema, Docker create-args path, browser sandbox path, tests, and docs still expose only bind-string support via `sandbox.docker.binds`. The PR remains a plausible implementation candidate, with security review discussion already focused on reserved target enforcement for named/ephemeral volumes.

## Best Possible Solution

Keep the PR open as the active implementation candidate. A maintainer should review the current head with special attention to Docker `--mount` construction, reserved target checks for all strategies, browser sandbox parity, inherited override validation, generated schema/API drift, and targeted sandbox/config tests before deciding whether to land or request changes.
