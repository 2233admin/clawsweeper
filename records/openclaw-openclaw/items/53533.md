## Summary

Keep open. Current main still lacks Feishu transport lifecycle reporting, but this PR branch is not ready to merge because the sink is not wired from gateway startup, the WebSocket changes are based on the pre-reconnect implementation, and the branch predates current webhook signature hardening. Open PR #39489 overlaps the same health-monitor surface and should be canonicalized by a maintainer.

## What This Changes

The PR branch adds Feishu monitor status-sink plumbing, a WebSocket lifecycle logger and logger passthrough, webhook connect/disconnect status updates, and targeted Feishu monitor/client tests.

## Best Possible Solution

Land one canonical Feishu-owned health-monitor implementation that passes the gateway account status sink from startAccount through monitorFeishuProvider, monitorSingleAccount, and both transports; reports connection lifecycle and inbound activity timestamps; preserves current WebSocket reconnect/heartbeat behavior and raw-body webhook signature verification; then close or rebase overlapping Feishu health PRs.
