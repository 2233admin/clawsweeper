## Summary

Keep open. Current main still suppresses QMD embed maintenance in lexical `search` mode before either `embedInterval` or forced embedding can take effect, and the reporter's config omits `searchMode`, which resolves to that default. Open PR #73512 already targets this scheduler behavior with a narrow implementation, so the issue should stay open for that PR review rather than being closed or duplicated.

## What This Changes

The issue requests that QMD automatically run embedding maintenance according to configured `memory.qmd.update.interval` or `memory.qmd.update.embedInterval` after OpenClaw updates memory documents.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The desired end state should either treat an explicit `memory.qmd.update.embedInterval` as an embedding opt-in while preserving conservative defaults, or make the required `searchMode: "vsearch"`/`"query"` guidance visible where users encounter pending embeddings.

## Reproduction Assessment

Yes. A high-confidence unit-level path already exists: configure QMD with `searchMode: "search"` and `embedInterval: "5m"`, advance fake timers, and current main expects no QMD `update` or `embed` calls.

## Solution Assessment

Unclear. The proposed direction in PR #73512 is narrow and plausible, but it changes behavior that current docs and tests present as intentional lexical-mode suppression, so maintainer product judgment is still needed.
