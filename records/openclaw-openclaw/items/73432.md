## Summary

Keep open. Current main still matches the reported behavior: QMD defaults to lexical `search` mode, and the embed run/timer paths are gated off for that mode even when `memory.qmd.update.embedInterval` is configured. The linked open PR #73512 uses closing syntax and is an implementation candidate, so this issue should not be closed before that PR is reviewed, merged, or explicitly declined.

## Best Possible Solution

Keep this issue open and review #73512 as the concrete candidate fix. If maintainers agree that an explicit `memory.qmd.update.embedInterval` should request embedding maintenance even in default `search` mode, land that PR or an equivalent memory-core change with the regression test. If they prefer the existing lexical-mode behavior, update runtime status/config diagnostics so users understand that embeddings require `searchMode: "query"` or `"vsearch"`.
