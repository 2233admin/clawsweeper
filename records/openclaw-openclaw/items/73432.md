## Summary

Keep open. Current main still gates QMD embedding maintenance off in the default lexical `search` mode before it considers `memory.qmd.update.embedInterval`, and the issue is already paired with open PR #73512 using closing syntax, so the tracker should remain open until that implementation is reviewed and merged or maintainers choose a docs/diagnostic-only path.

## What This Changes

The issue requests that QMD automatically run embedding on the configured `memory.qmd.update.embedInterval` cadence after OpenClaw changes memory files, even when `memory.qmd.searchMode` is omitted and defaults to lexical search.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The preferred end state should either honor an explicit `memory.qmd.update.embedInterval` as an embedding opt-in while preserving conservative defaults, or make the lexical-mode skip unmistakable in status/errors so users know how to enable automatic embeddings.
