## Summary

Keep this issue open. Current main still appears to miss the reported Slack binding shape: Slack inbound routing supplies raw Slack IDs, while route-binding peer IDs are normalized with trim/stringify only and then compared literally. A binding using `channel:C...` can still fall through to the default agent and produce `agent:main:...` session keys. The related PR #42162 was closed unmerged, so it is not shipped evidence of a fix.

## Best Possible Solution

Normalize Slack route-binding peer IDs before generic route matching while preserving the canonical session key shape. A conservative fix should allow raw IDs and target-style IDs such as `channel:C123`, `user:U123`, and `<@U123>` to match the raw IDs produced by Slack events, then add focused regression tests for Slack channel/group and DM bindings. If maintainers want raw IDs as the documented preferred form, add config help or doctor warnings while keeping compatibility for existing target-style configs.
