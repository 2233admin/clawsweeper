## Summary

Current main still appears to miss Slack route bindings when the configured peer id uses Slack target syntax such as `channel:C...` or `user:U...`. Slack inbound routing passes raw Slack ids into the generic matcher, while route binding peer ids are only trim-normalized and then compared exactly. The related implementation candidate, PR #42162, is closed and unmerged, so the issue should remain open and is a good focused fix-PR candidate.

## Best Possible Solution

Keep this issue open and land a narrow Slack routing fix. Normalize Slack route-binding peer ids before generic matching so target-style forms such as `channel:C123`, `user:U123`, and `<@U123>` match the raw Slack ids received at runtime, while preserving existing raw-id configs and canonical raw ids in session keys. Add focused regression coverage for channel and direct bindings plus config help or warnings if raw Slack ids remain the documented preferred form.

## Clownfish Work Prompt

Fix Slack route binding peer-id normalization for this issue. Current main passes raw Slack `message.channel` / `message.user` ids into `resolveAgentRoute`, while configured route bindings can contain Slack target-style ids like `channel:C123`, `user:U123`, or `<@U123>`; those are trim-normalized only and then exact-matched, so they miss and fall back to the default `main` agent. Implement a narrow fix that makes Slack route-binding peer ids in target-style forms match the raw Slack ids received at runtime, while preserving existing raw Slack id configs and keeping canonical raw ids in session keys. Prefer a Slack-owned normalization path or a small generic plugin-facing seam; do not hard-code broad Slack-specific policy into unrelated channel behavior. Add regression tests for Slack channel binding `channel:C123` matching inbound `C123`, direct binding `user:U123` / `<@U123>` matching inbound `U123`, and existing raw-id bindings continuing to work. If config help or warnings are touched, keep them non-fatal and aligned with generated schema/docs expectations.
