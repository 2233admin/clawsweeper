## Summary

Keep open. Current main has adjacent model fallback, provider/auth-profile cooldowns, and restart recovery retry caps, but it still does not implement the requested persisted session-level circuit breaker that suppresses or backs off synthetic auto-resume injection after consecutive model errors. The supplied discussion narrows this to the March 22 Gemini 429 loop where resume prompts were repeatedly injected into the same failing session; related PR #42933 remains open and unmerged, so it is an implementation candidate rather than proof that this issue is fixed.

## Best Possible Solution

Keep this issue open and implement a generic persisted session-level model-error breaker around automatic resume attempts. It should count consecutive model errors for main-session and subagent recovery, suppress or exponentially back off further synthetic resume injection after a threshold, reset on the first successful model response or explicit operator reset, and add focused tests for restart/orphan recovery plus model-fallback exhaustion. Coordinate with #42933 and #45389 so the breaker state is shared rather than duplicated.
