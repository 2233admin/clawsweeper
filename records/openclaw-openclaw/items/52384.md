## Summary

Keep open. Current main has adjacent provider/model fallback, auth-profile cooldown probes, and restart recovery retry caps, but it still lacks the requested persisted session-level circuit breaker that suppresses or backs off synthetic auto-resume injection after consecutive model errors. The related implementation PR #42933 is open and unmerged, so it is a candidate to review rather than proof this issue is fixed.

## Best Possible Solution

Keep this issue open and implement a generic persisted session-level model-error breaker around automatic resume attempts. The breaker should count consecutive model errors for main-session and subagent recovery, suppress or exponentially back off further synthetic resume injection after a threshold, reset on the first successful model response or explicit operator reset, and add focused tests for restart/orphan recovery plus fallback exhaustion. Coordinate with #42933 and #45389 so breaker state is shared rather than duplicated.
