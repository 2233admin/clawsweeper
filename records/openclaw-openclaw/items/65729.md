## Summary

Current main still does not implement the requested Mattermost reliability bundle. DMs remain explicitly non-threaded and history-disabled, WebSocket ping/pong keepalive is absent, thread-history backfill is not present, and mention-only messages still drop after normalization. Related #51104, #53186, and #59981 cover parts of the area but do not supersede this broader request.

## Best Possible Solution

Keep this open and scope it as Mattermost-plugin-owned reliability work. Preserve flat DMs by default, add an explicit DM/per-chat-type threading option, add configurable WebSocket ping/pong keepalive without removing the existing update_at poll, safely backfill empty thread-scoped histories from Mattermost after access-control rules are defined, handle mention-only invocations, and update Mattermost tests, config metadata, and public docs.
