## Summary

Keep open. The PR is a useful feature proposal, but it is stale against current main: the active directive parser, runtime prefs, SDK facade, and Discord voice path now live under speech-core/plugin SDK paths, while the branch adds persistence to older surfaces. Current main also does not already implement provider-keyed persistent voices, so there is no cleanup-close reason.

## What This Changes

The PR adds provider-keyed TTS voice preferences, a `persist=true` TTS directive, best-effort post-synthesis preference writes, and tests across the older core TTS, telephony, and Discord voice paths.

## Best Possible Solution

Port the feature to the current provider-aware TTS architecture: add a typed persistence signal to `src/tts/directives.ts` and `TtsDirectiveParseResult`, add provider-keyed voice prefs plus runtime/facade APIs in speech-core and `src/plugin-sdk/tts-runtime*`, resolve persisted voices during normal and telephony synthesis, write prefs only after successful synthesis as best-effort, wire the current Discord voice path, and document the durable directive behavior.
