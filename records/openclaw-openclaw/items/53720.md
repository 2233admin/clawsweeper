## Summary

Keep this PR open. Current main still does not implement the intended per-agent enabled/disabled lifecycle control: there is no typed `agents.list[].enabled` field, no `openclaw agents disable`/`enable` commands, no `[disabled]` list status, and current routing/cron/heartbeat paths still treat configured agents as active. The PR remains the canonical implementation thread for the already-closed #21680 request, but it needs maintainer review/revision because later review comments identify unresolved behavior gaps in all-disabled routing fallback, cron/heartbeat suspension, and clean `--json` output.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main. The best implementation path is a typed `agents.list[].enabled?: boolean` contract in config types, strict schema, generated config baseline, docs, and CLI; separate helpers for all configured agents versus active agents; routing, cron, heartbeat, and channel polling that exclude disabled agents; management commands that still operate on disabled agents; clean JSON output; and regression tests covering data preservation, all-disabled fallback, scheduler suspension, and CLI round trips.
