## Summary

Keep this issue open. Current main still has no native WhatsApp per-peer outbound-only mode: direct-peer config is prompt-only, the strict schema has no inbound/outbound direction fields, inbound admission is still controlled by dmPolicy/allowFrom/pairing, accepted DMs continue into read/enqueue/session/reply handling, and outbound authorization reuses allowFrom rather than a separate outbound-only permission surface.

## Best Possible Solution

Keep this open for a maintainer-designed WhatsApp plugin/config change: add a directional per-peer inbound policy for direct WhatsApp peers, enforce inbound-disabled immediately after DM normalization/admission and before read receipts, hooks, enqueueing, session metadata, agent execution, or fallback replies, while preserving outbound sends for explicitly allowed notification-only peers. Update WhatsApp types, strict schema, generated config metadata/help, docs, and focused tests for an outbound-allowed peer whose inbound messages are ignored.
