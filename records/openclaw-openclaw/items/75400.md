## Summary

Keep this PR open. It targets a real current-main OpenRouter/Anthropic prefill gap, but the patch should not treat explicit OpenAI-compatible disabled reasoning values as thinking-enabled, and it still needs the required user-facing changelog entry before merge.

## What This Changes

The PR adds OpenRouter Anthropic prefill stripping in the OpenRouter stream wrapper, broadens the shared stripping helper to OpenAI-compatible reasoning fields, and adds focused provider-stream tests.

## Best Possible Solution

Land the OpenRouter prefill-strip fix at the stream-wrapper boundary, but make the shared helper respect explicit disabled OpenAI-compatible reasoning values and record the user-facing regression fix in the Unreleased Fixes changelog.

## Reproduction Assessment

Yes. The linked bug report gives concrete OpenRouter Anthropic configuration and provider 400 logs, and current main source inspection shows the OpenRouter wrapper lacks the shared prefill-strip call. The PR-specific regression can be reproduced with a focused unit case using an Anthropic OpenRouter payload with trailing assistant prefill and `reasoning: "none"`.

## Solution Assessment

No, not as-is. Reusing the existing stripper in the OpenRouter wrapper is the right narrow boundary, but the helper should mirror the existing disabled `off`/`none` semantics before stripping and the PR should add the required changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Respect disabled reasoning before stripping prefill:** `src/plugin-sdk/provider-stream-shared.ts:166`
  - body: This makes any truthy `reasoning`/`reasoning_effort` payload count as Anthropic thinking. OpenAI-compatible requests already use string values like `reasoning: "none"` to disable reasoning; because the OpenRouter wrapper preserves `reasoning` when thinking is off, an Anthropic OpenRouter request with legal non-thinking prefill would now drop that prefill. Please mirror the existing `off`/`none` handling before stripping.
  - confidence: 0.82
