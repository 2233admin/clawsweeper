## Summary

Keep open. The PR targets a valid current-main gap for #74385, but the patch introduces a reconnect-state regression by unconditionally replacing a restored active-run `streaming` status with `initializing`, then potentially `idle`. That should be fixed before merge.

## What This Changes

The PR adds an `initializing` TUI activity status, treats it as busy in the status renderer, sets it during post-connect agent/history initialization, and adds unit coverage for the busy-status helper.

## Best Possible Solution

Keep the narrow initialization loader change, but preserve reconnect semantics: show `initializing` only for startup work that is not already covered by an active run state, clear it to `idle` only when no auto-message or active run has taken over, and add a regression test for reconnecting during an active streamed run.

## Reproduction Assessment

Yes. For the original issue, the source path is `openclaw onboard` -> Terminal hatch -> local TUI connect, where current main renders `local ready | idle` while awaiting `refreshAgents()` and `loadHistory()` before the auto-message can start. For the PR regression, reconnecting with an active tracked run follows `reconnectStreamingWatchdog()` to `streaming`, then the new line overwrites it with `initializing`.

## Solution Assessment

No, not as currently patched. The helper and initialization status are the right direction, but the unconditional status transition should be guarded so it does not clobber active reconnect streaming state or clear it back to idle.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve streaming state on reconnect:** `src/tui/tui.ts:1112`
  - body: On reconnect, `reconnectStreamingWatchdog()` can restore an active run to `streaming`, but this line immediately overwrites that with `initializing`. If the user reconnects while a run is still active and no new event arrives before history loading completes, the post-init block can then flip the status to `idle`, regressing the reconnect watchdog/status recovery path. Gate initialization status to first-connect/no-active-run cases, or avoid clearing it when a run is tracked.
