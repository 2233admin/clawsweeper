The PR changes outbound message policy to default-deny same-provider cross-context sends, updates config help/tests/changelog, adds Telegram topic same-context handling, and also includes unrelated failover-classification and UI scroll fallback changes.

## Best Possible Solution

Refresh and narrow the branch to the outbound message-policy hardening: keep same-provider forwarding opt-in by default, preserve the legacy explicit override, compare Telegram/topic targets through the loaded channel route parser, require topic identity when a session is bound to a Telegram topic, update generated config help plus changelog attribution, drop unrelated changes, and then run focused outbound/Telegram coverage before the changed gate.
