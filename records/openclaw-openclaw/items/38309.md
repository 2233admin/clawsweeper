## Summary

Keep this issue open. Current main still does not support `SecretRef` objects in `sandbox.docker.env`: the schema, public type, generated schema, runtime resolution path, secret collectors, and SecretRef target registry remain string-only or omit the Docker env surface. The paired implementation PR #38310 was closed unmerged, so the requested behavior still needs sandbox/secrets maintainer review.

## What This Changes

Allow `SecretRef` values in `sandbox.docker.env` for default and per-agent Docker sandbox environment variables while preserving resolved Docker env as `Record<string, string>`.

## Best Possible Solution

Keep this issue open and route it through sandbox/secrets ownership. The best implementation is an end-to-end SecretInput addition for Docker sandbox env: accept `SecretInput` in Zod and generated schemas, type unresolved `SandboxDockerSettings.env` as `Record<string, SecretInput>` while preserving resolved `SandboxDockerConfig.env` as `Record<string, string>`, collect `agents.defaults.sandbox.docker.env.*` and `agents.list[].sandbox.docker.env.*` through the secrets runtime and target registry, resolve before Docker create args, preserve plain string behavior, update public docs, and add focused schema/runtime/registry/merge/create-arg tests.
