## Summary

Keep open. Current main at f6d23ab5c2d8e7cbf50522b6c8137241524b6646 still does not support SecretRef objects in sandbox.docker.env: the Zod schema, public config type, generated schema, and core secret collectors remain string-only or SSH-only for sandbox secrets. The linked implementation PR #38310 is closed unmerged, so it is useful design context but not a superseding fix.

## Best Possible Solution

Keep this issue open and implement SecretInput support end-to-end for Docker sandbox env after maintainer review: accept SecretRef values in Zod and generated schemas, type unresolved sandbox docker env as Record<string, SecretInput> while keeping resolved SandboxDockerConfig.env as Record<string, string>, collect defaults and per-agent env entries through the secrets runtime and target registry, preserve plaintext string behavior, and add focused schema/runtime/create-arg regression tests. Use closed PR #38310 as design context, but land a reviewed current-main implementation.
