## Summary

Keep this issue open. Current main still routes GitHub Copilot Gemini catalog rows and forward-compatible Gemini refs to `openai-responses`, matching the reported unsupported Responses API failure; the manual completions-header symptom appears partly addressed in shared transport code, but the default Gemini route remains wrong.

## What This Changes

The issue requests a GitHub Copilot provider fix so Gemini/chat-completions-only Copilot models use the completions endpoint with required Copilot IDE headers instead of falling back after Responses API failures.

## Best Possible Solution

Land a focused GitHub Copilot provider fix: route known Gemini and forward-compatible Gemini Copilot IDs to `openai-completions`, keep or add regression proof that Copilot IDE headers are present on that completions path, update the provider docs and changelog, and leave fallback-cost warning UX for a separate issue.

## Solution Assessment
