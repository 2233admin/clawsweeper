## Summary

Keep this issue open. Current main supports thinking effort and reasoning visibility, but I found no config, session/API field, UI control, docs surface, or runtime path for selecting the language of visible reasoning output.

## Best Possible Solution

Keep this open for product/design work. A good implementation would add a documented reasoning-language preference, likely under agent defaults plus per-agent/session override, wire it through config schema/types/help, Control UI and session APIs, and the agent runtime/provider request path, with tests proving final response language stays independent from visible reasoning language.
