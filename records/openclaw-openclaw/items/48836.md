## Summary

Keep this PR open. The contribution is still plausible, but it needs a refresh to the current generated Control UI i18n workflow and should resolve the bot's unused `closeSessionDetails` review note. Security review found no CI, dependency, package, secret, network, or command-execution surface in the PR diff.

## Best Possible Solution

Keep the PR open for a contributor or maintainer refresh: move the remaining hardcoded chat labels to `t(...)`, add the English source keys, update the current `run-controls.ts` surface, remove or wire `closeSessionDetails`, then run the Control UI i18n sync/check workflow so zh-CN and the other generated locale artifacts stay structurally aligned.
