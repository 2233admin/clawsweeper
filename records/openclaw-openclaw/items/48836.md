## Summary

Keep this PR open. Current main still does not implement the PR’s central request: several chat UI labels remain hardcoded in English, the proposed chat translation keys are absent from the active locale maps, and the chat run controls have since moved into a separate module that the PR does not update. The PR is still a coherent localization candidate, but it needs a refresh for the current Control UI i18n workflow.

## Best Possible Solution

Keep the PR open and refresh it against current main: move the remaining hardcoded chat labels in ui/src/ui/views/chat.ts and ui/src/ui/chat/run-controls.ts to t(...) calls, add the English source keys, remove or wire closeSessionDetails, regenerate locale bundles with the Control UI i18n sync workflow, and update the run-controls tests to assert the localized labels through the current i18n path.
