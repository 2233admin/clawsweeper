## Summary

Keep open. Current main still prepends full fenced Conversation info and Sender JSON metadata blocks before direct Telegram/external-channel user text, and #72749 is the open unmerged closing PR for this work, so cleanup closure would be premature.

## What This Changes

The issue requests reducing verbose inline Telegram inbound metadata or moving it out of user message content so models see the actual user text more clearly.

## Best Possible Solution

Review and land #72749, or an equivalent narrow config-backed prompt-builder change, so direct Telegram-style chats can opt into compact metadata while preserving full untrusted context for groups, replies, forwards, locations, and richer history surfaces.

## Reproduction Assessment

Yes. A direct Telegram context with `ChatType: "direct"`, `OriginatingChannel: "telegram"`, `OriginatingTo`, message id, and sender fields is enough for `buildInboundUserContextPrefix` to render the full Conversation info and Sender JSON blocks, which `get-reply-run.ts` prepends before the user body.

## Solution Assessment

Yes. Keeping this issue open while the paired closing PR is reviewed is the right handling; current main has not implemented the requested compact mode, and closing before merge would hide remaining work.
