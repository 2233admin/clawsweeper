## Summary

Keep open. Current main still prepends external direct-channel messages with full fenced Conversation info and Sender JSON blocks, and the linked closing PR #72749 remains open/unmerged, so the issue should not be closed before that implementation is reviewed and landed or replaced.

## What This Changes

The issue requests reducing or structurally separating Telegram inbound metadata so short user messages are not preceded by large inline JSON metadata blocks.

## Best Possible Solution

Review and land #72749, or an equivalent narrow prompt-builder/config change, so direct Telegram-style chats can opt into compact metadata while preserving full untrusted context for groups, replies, forwards, locations, and history with sanitizer and config/docs coverage.
