## Summary

Keep open. Current main still prepends full fenced JSON Conversation info and Sender blocks to direct external-channel user prompts, and #72749 is an open, unmerged PR with closing syntax for this issue, so the issue should stay open until that PR is reviewed and landed or replaced.

## What This Changes

The issue asks OpenClaw to reduce or move Telegram/direct-channel inbound metadata out of user-message text because full fenced JSON metadata blocks make short prompts noisy.

## Best Possible Solution

Review and land #72749, or an equivalent narrow prompt-builder/config change, so direct external-channel messages can use a compact metadata shape while preserving full untrusted context for groups, replies, forwards, locations, and history; include config/schema/docs and sanitizer tests before closing this issue.

## Reproduction Assessment

Yes. A direct external-channel context with `ChatType: "direct"`, `OriginatingChannel: "telegram"` or another non-webchat channel, and sender fields reaches `buildInboundUserContextPrefix`, which current source and tests show emits full Conversation info and Sender JSON blocks before the user body.

## Solution Assessment

Yes. The narrowest maintainable direction is a prompt-builder/config change like #72749, because provider-specific structured metadata fields are not portable across all supported model providers and the existing safety framing still needs to be preserved for richer contexts.
