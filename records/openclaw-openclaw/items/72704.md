## Summary

Keep open because current main still emits the reported full fenced inbound metadata in user-role prompt text, and the explicitly linked implementation PR is still open and unmerged. The issue should remain open until that PR, or an equivalent narrow fix, lands.

## What This Changes

Request compact or structured handling of Telegram inbound metadata so short direct user messages are not preceded by large repeated JSON Conversation info and Sender blocks.

## Best Possible Solution

Land #72749 or an equivalent narrow change that adds a compact direct-chat metadata mode through the prompt builder, config schema, docs, and sanitizer tests while preserving full untrusted metadata for groups, replies, forwards, locations, and history.

## Reproduction Assessment

Yes. A Telegram direct message context sets `ChatType: "direct"`, `OriginatingChannel: "telegram"`, sender fields, and message ids, which current main feeds into `buildInboundUserContextPrefix` before the actual user body.

## Solution Assessment

Yes. Keeping this open while reviewing the linked closing PR is the right handling because current main has not implemented the compact mode and closing before merge would hide remaining work.
