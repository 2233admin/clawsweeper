## Summary

Keep open. Current main still prepends full fenced Conversation info and Sender JSON metadata ahead of direct Telegram user text, and #72749 is already the open closing implementation candidate, so cleanup closure before that PR lands would be premature.

## What This Changes

Request to reduce Telegram direct-message prompt noise by moving or compacting repeated inline Conversation info and Sender JSON metadata ahead of user text.

## Best Possible Solution

Review and land #72749, or an equivalent narrow config-backed prompt-builder and sanitizer change, so direct Telegram-style chats can use compact metadata while groups, replies, forwards, locations, and richer history surfaces keep the fuller untrusted context.

## Reproduction Assessment

Yes. A Telegram DM finalizes context with ChatType direct, Telegram provider/surface, message id, destination, and sender fields; current prompt assembly then prepends buildInboundUserContextPrefix output before the user body, and that builder emits the full fenced Conversation info and Sender blocks.

## Solution Assessment

Yes for triage: keeping this issue open while the linked closing PR is reviewed is the right handling. The implementation should stay narrow in inbound prompt construction/config and avoid dropping richer metadata where it is still needed.
