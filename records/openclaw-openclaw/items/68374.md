## Summary

Keep open. Current `main` still drops the requested Claude CLI reasoning path before it reaches the HTTP API: the Claude stream-json parser accepts only `text_delta`, `/v1/responses` emits/returns assistant text only, `/v1/chat/completions` also resolves only assistant text, and the public OpenResponses docs still say `reasoning` is accepted but ignored. Related PR #56674 is an open implementation candidate, not a merged fix.

## What This Changes

Expose Claude CLI thinking blocks as structured reasoning output on `/v1/responses` and `/v1/chat/completions`, gated by existing reasoning settings or an explicit HTTP opt-in.

## Best Possible Solution

Keep this issue open while maintainers review #56674 or replace it with a refreshed implementation that preserves Claude `thinking_delta` as agent reasoning, gates HTTP exposure through `reasoningDefault` or an explicit request option, emits structured reasoning for streaming and non-streaming `/v1/responses`, defines the `/v1/chat/completions` compatibility behavior, and adds focused parser/gateway docs and tests.
