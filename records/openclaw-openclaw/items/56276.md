## Summary

Keep open. Current main still does not show a hostname in the TUI footer/status bar; it only exposes the connection URL in the header. The related reports in the provided context cover model/status-bar behavior and busy state, not hostname display.

## Best Possible Solution

Implement a small TUI-owned host label in the footer/status bar. For remote Gateway connections, derive a friendly label from the configured/connected URL hostname; for local or embedded mode, use a stable local hostname/display label if maintainers want that behavior. Add a focused formatter/rendering test and update the public TUI docs at https://docs.openclaw.ai/web/tui.
