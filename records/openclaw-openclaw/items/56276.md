## Summary

Keep open. Current main still does not render host context in the TUI footer/status bar; the adjacent header and backend expose connection context, so this remains a narrow valid TUI enhancement rather than stale cleanup.

## What This Changes

Add the current hostname to the OpenClaw TUI status/footer bar so users can distinguish multiple machines, especially over Tailscale.

## Best Possible Solution

Add a compact TUI-only `host <label>` segment after the agent label, deriving Gateway mode from the parsed connection URL hostname without credentials and using the existing machine display-name helper or a conservative local label for embedded mode; update TUI tests, docs, and changelog.

## Reproduction Assessment

Yes. Current-main static inspection is enough to reproduce the missing behavior: `updateFooter` has no host part, while the docs also omit hostname from the footer description.

## Solution Assessment

Yes. A narrow TUI renderer/helper change is the best fit; changing the Gateway protocol would be unnecessary unless maintainers want server-authored host identity instead of connection/local display context.
