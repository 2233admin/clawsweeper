## Summary

Keep open. Current main still does not implement the requested `/memory` Control UI surface, but this stacked PR is not merge-ready because the diff introduces definite build/protocol regressions and confirmed Gateway memory RPC bugs that should block merge.

## What This Changes

The PR adds a stacked Control UI `/memory` view with memory status/sources/jobs/search/excerpt UI, new scoped `memory.*` Gateway RPCs and schemas, generated Swift models, i18n strings, and several memory-core indexing/scope changes.

## Best Possible Solution

Keep the PR open as the UI layer of the vector-memory stack, land or close the prerequisite PRs first, then rebase/split this branch so it only adds the memory page and required RPC wiring while preserving existing plugin/session/node protocol contracts and fixing the review findings.

## Reproduction Assessment

Yes. This is reproducible by static inspection of current main and PR head: main lacks the `/memory` route and handlers, while the PR head has a `Set.length` compile error, removed protocol exports used by existing files/tests, an unsafe path sort, and an unbounded job Map.

## Solution Assessment

No. A scoped memory management view is a plausible direction, but this PR is not the best mergeable form until it is rebased after the stack, restores existing public Gateway surfaces, fixes the package/RPC defects, and passes targeted Gateway/protocol/UI validation.

  - body: `readColumns()` returns a `Set<string>`, so `existingFtsColumns.length` is a type error and also never detects legacy FTS columns at runtime. The package source is what the bridge exports, so this blocks the memory schema path until it uses `.size`.
  - confidence: 0.96
- **[P1] Restore the removed protocol exports:** `src/gateway/protocol/index.ts:262-280`
  - body: The PR removes existing session/plugin/node protocol validators and types from the protocol barrel, but current server files and protocol tests still import names such as `validateSessionsPluginPatchParams`, `validatePluginsUiDescriptorsParams`, `validateNodeEventResult`, and `validateNodePresenceAlivePayload`. This will break typecheck/tests and drops public client contract surfaces unrelated to the memory view.
  - confidence: 0.92
- **[P2] Keep plugin host handlers registered:** `src/gateway/server-methods.ts:77`
  - body: Adding `memoryHandlers` also drops `pluginHostHookHandlers` from `coreGatewayHandlers`, so the existing `plugins.uiDescriptors` RPC becomes unknown even though plugins can still register Control UI descriptors. Restore that handler group alongside the new memory handlers.
  - confidence: 0.9
- **[P2] Keep package error handling package-local:** `packages/memory-host-sdk/src/host/memory-schema.ts:2`
  - body: This package source previously used its local `./error-utils.js`; importing `../../../../src/infra/errors.js` crosses the memory-host-sdk package boundary and risks boundary/packaging failures. Keep the package helper self-contained or expose a sanctioned package-level helper.
  - confidence: 0.86
- **[P1] Guard pathless memory search hits before sorting:** `src/gateway/server-methods/memory.ts:443`
  - body: `decorateSearchHit()` can leave `path` undefined, so tied scores will call `left.path.localeCompare(...)` and throw for real pathless hits. Sort with null-safe strings or filter/normalize the path before comparing.
  - confidence: 0.94
- **[P2] Bound retained memory index jobs:** `src/gateway/server-methods/memory.ts:52`
  - body: `indexJobs` is process-global and never pruned, while `memory.index.run` adds entries over time. A long-running Gateway can grow this Map indefinitely; cap it or evict completed/failed jobs after a TTL.
  - confidence: 0.88
