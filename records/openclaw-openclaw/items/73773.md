Overall confidence: 0.9

Full review comments:

- **[P1] Restore existing Gateway and protocol surfaces:** `src/gateway/server-methods.ts:74`
  - body: The patch adds `memoryHandlers` while dropping existing plugin/session/protocol registrations such as `plugins.uiDescriptors`, `sessions.pluginPatch`, `NodeEventResult`, and `NodePresenceAlivePayload`. These are present on current main and are unrelated public Control UI/plugin surfaces, so the memory RPC additions need to be additive.
  - confidence: 0.9
- **[P1] Guard pathless search hits before sorting:** `src/gateway/server-methods/memory.ts:442-444`
  - body: `decorateSearchHit` can leave `path` undefined, but the equal-score sort fallback calls `left.path.localeCompare(right.path)`. A pathless hit with the same score will throw and fail `memory.search.debug`; normalize both paths before comparing.
  - confidence: 0.93
- **[P2] Bound the index job registry:** `src/gateway/server-methods/memory.ts:50-52`
  - body: `indexJobs` is process-global and entries are added by `memory.index.run` without TTL, pruning, or a size cap. A long-running Gateway that receives repeated index requests will retain completed or failed jobs indefinitely; evict old jobs or cap the map.
  - confidence: 0.88
- **[P2] Keep the memory host SDK self-contained:** `packages/memory-host-sdk/src/host/memory-schema.ts:2`
  - body: The package-local schema file now imports `../../../../src/infra/errors.js`. Published package consumers should not need the repository root `src` tree at runtime; keep the local helper or expose this through a supported package boundary.
  - confidence: 0.84
- **[P3] Document the new user-facing memory tab:** `ui/src/ui/navigation.ts:9`
  - body: This is a user-facing `feat(control-ui)` change, but the PR file list does not update `CHANGELOG.md` or the Control UI docs. Add an Unreleased changelog entry and update the Control UI capability docs if the new tab remains in the final diff.
  - confidence: 0.82
