## Summary

Keep open. Current main still reports Telegram runtime, credential, webhook/polling, and group-membership status problems, but it does not inspect Telegram persisted JSON state artifacts for readability, size, parse/schema health, or provide a safe backup/reset recovery path. The discussion’s Keep My Claw link is useful backup context, but it was explicitly framed as a safety net rather than the requested Telegram status/recovery UX fix.

## What This Changes

Add Telegram status/recovery diagnostics for oversized, malformed, unreadable, or otherwise poisoned persisted Telegram session/state JSON, with clear operator guidance for safe remediation.

## Best Possible Solution

Add a Telegram-owned persisted-state diagnostic that feeds channel status and, if appropriate, doctor warnings. It should identify account-scoped Telegram state artifacts, report path/readability/size/parse/schema health and likely impact, and recommend a backup-before-reset recovery path; destructive reset or auto-quarantine should be explicit follow-up work.
