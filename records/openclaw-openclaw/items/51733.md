## Summary

Keep open. Current main appears to have superseded the PR's watcher-glob approach by watching skill root directories directly, but it still does not implement the PR's loader-side goal for repo-style nested bundles under managed/workspace roots such as `skills/<repo>/skills/<skill>/SKILL.md`.

## Best Possible Solution

Keep this PR open as a potentially useful implementation candidate, but narrow it to the still-missing loader behavior: discover repo-style nested skill bundles under workspace, project-agent, personal-agent, and managed skill roots, with regression tests for those roots. Drop or adapt the watcher-glob changes to current main's root-watch model, and address the remaining hybrid-bundle precedence review before merge.
