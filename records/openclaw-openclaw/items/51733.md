## Summary

Keep open. Current main no longer needs the PR's watcher-glob change because it watches skill root directories, but it still does not load repo-style nested bundles placed under managed, workspace, project-agent, or personal-agent skill roots such as `skills/<repo>/skills/<skill>/SKILL.md`. The PR remains a useful implementation candidate after rebasing and narrowing to the loader behavior.

## Best Possible Solution

Keep this PR open as a focused implementation candidate, but rebase it onto current main and narrow it to loading repo-style nested bundles under workspace, project-agent, personal-agent, and managed skill roots. Add targeted regression tests for those roots, keep the current root-watch model, and resolve hybrid bundle precedence before merge.

## Risks / Open Questions

- The PR needs a current-main rebase because the watcher side now uses root-directory watching rather than explicit `SKILL.md` globs.
- The loader-side change broadens filesystem discovery; final review should verify containment, symlink-cycle handling, `maxCandidatesPerRoot`, `maxSkillsLoadedPerSource`, and `maxSkillFileBytes`.
- Hybrid bundle precedence remains an open product/code question when a repo contains both top-level `SKILL.md` and nested `skills/*/SKILL.md`.
