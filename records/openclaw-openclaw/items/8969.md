## Summary

Keep open. Current main implements the first quick win as `openclaw skills check`, and the reporter's local validation scripts are useful context, but the two remaining requested operator surfaces are still missing: manifest-registry diagnostics are not exposed through status, and the non-tool skill command prompt-rewrite path has no operator-visible warning. The April 26 maintainer/Codex review in the issue discussion already reached the same conclusion.

## Best Possible Solution

Keep this issue open or split the two remaining quick wins into focused follow-ups. Preserve the shipped `openclaw skills check` behavior, then surface manifest-registry diagnostics in `openclaw status`, `openclaw status --json`, and `openclaw status --all`; add an operator-visible warning/log for the non-tool skill command branch that rewrites the request for model-directed execution. Cover the status text/JSON/all surfaces and the prompt-rewrite branch with focused tests, then update the public status and skills docs.
