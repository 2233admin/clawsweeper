## Summary

Keep open. Current main implements the skills validation quick win as `openclaw skills check`, but the remaining requested operator surfaces are still missing: manifest-registry diagnostics are not exposed through status/status JSON/status --all, and the non-tool skill command prompt-rewrite path still rewrites the request without an operator-visible warning.

## Best Possible Solution

Keep this issue open or split the two remaining quick wins into focused maintainer-scoped issues. Preserve the shipped `openclaw skills check` behavior, then expose manifest-registry diagnostics through `openclaw status`, `openclaw status --json`, and `openclaw status --all`; separately add an operator-visible signal for the non-tool skill command path that rewrites the request for model-directed execution. Cover the status text/JSON/all surfaces and prompt-rewrite branch with focused tests, and update the public status and slash-command/skills docs once the surfaces are chosen.
