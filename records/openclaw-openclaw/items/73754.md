## Summary

Keep open. Current main contains the Teams JWT CommonJS/ESM unwrap fix and regression coverage, but the reported user-facing problem is that the latest shipped release still missed that fix. The v2026.4.26 tag has the broken dynamic-import destructuring, there is no v2026.4.27 tag in the remote tag check, and the bundled @openclaw/msteams package metadata on current main still reports 2026.4.25, so a release/publish action remains before this can be closed as implemented for users.

## Best Possible Solution

Keep this issue open until maintainers cut and verify a release or patch release that includes the current Teams JWT loader fix in the built OpenClaw package and the published/bundled @openclaw/msteams artifact. Before closing, confirm the shipped artifact contains the default-export unwrap, and consider the requested warning-level diagnostic for unexpected JWT validator exceptions.
