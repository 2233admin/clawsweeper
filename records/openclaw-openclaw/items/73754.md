## Summary

Keep open. Current main has the Microsoft Teams JWT CommonJS/ESM unwrap and a regression-shaped test, but the reported problem is release-facing: the latest provided release remains v2026.4.26, that tag still has the broken dynamic-import destructuring, and the bundled @openclaw/msteams metadata on main still reports 2026.4.25. The silent validation catch also remains as a diagnostic follow-up.

## Best Possible Solution

Keep this open until maintainers cut and verify a stable correction or v2026.4.27-style release whose built OpenClaw package and bundled/published @openclaw/msteams artifact contain the jsonwebtoken default-export unwrap. Before closing, verify the shipped artifact rather than only the TypeScript source, and either implement the warning/error logging request or split it into a narrower follow-up.
