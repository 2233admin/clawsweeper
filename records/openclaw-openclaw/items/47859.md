## Summary

Keep open. Current main does not implement the PR's intended TUI image attachment feature: `/image` is not registered or documented, the TUI send path passes only text, and the TUI backend type has no attachment field even though the Gateway already accepts `chat.send` attachments. The PR is still potentially useful, but it needs rebasing and review fixes before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to ask the contributor to rebase and fix the reviewed bugs, or to supersede it with a small TUI-focused implementation that adds image extraction/loading, forwards attachments through the current `TuiBackend` abstraction, preserves non-image prompt text exactly, avoids raw path leakage, updates TUI docs, and adds focused tests.
