## Summary

Keep this PR open with requested changes. The deleted-cwd helper change is useful but does not cover the current startup paths that still call raw `process.cwd()`, and the branch also carries unrelated Telegram transient-error behavior that can permanently suppress typing cues after a temporary network outage.

## What This Changes

The PR adds a `safeCwd()` fallback to `os.tmpdir()` in `src/infra/home-dir.ts` with a unit test, and also changes Telegram `sendChatAction` handling to back off, swallow, and eventually suspend transient network errors.

## Best Possible Solution

Revise or supersede this with a narrow deleted-cwd startup fix that guards entry/main-module detection plus CLI dotenv preflight/loading, preserves existing `.env` precedence when cwd exists, and adds focused regression coverage for the actual `openclaw tui` startup path. The Telegram transient-error behavior should be split into its own PR or removed unless maintainers explicitly want that policy change.

## Reproduction Assessment

Yes. The linked bug has concrete steps, and current main still has deterministic raw `process.cwd()` calls in `isMainModule`, `shouldLoadCliDotEnv`, and `loadCliDotEnv` that match the `uv_cwd` failure path; I did not run a live repro because this review is read-only.

## Solution Assessment

No. The home-dir fallback is a useful partial helper change, but it is not the narrowest complete fix for the reported startup crash, and the unrelated Telegram suspension change introduces a separate runtime regression risk.

## Review Findings

A temporary Telegram/network outage can therefore disable all future typing cues for the account until restart, unlike the intentional 401 token circuit breaker.
  - confidence: 0.87
