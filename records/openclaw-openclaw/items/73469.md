## Summary

Keep open. Current main still sends Feishu streaming cards as an initial placeholder/card-reference message, then applies the final mention-bearing text through Card Kit updates; the existing mention plumbing only prepends known inbound `mentionTargets` at stream close, so agent-authored `<at user_id="...">...</at>` text is not routed through a notification-capable initial send. The related #60103 issue overlaps on syntax normalization, but it does not cover this first-notification timing problem.

## What This Changes

Feishu streaming-card replies should send push notifications when an agent-authored response contains `<at user_id="...">...</at>` mentions.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug and design a tested outbound strategy that preserves notification semantics for mention-bearing agent replies while making the streaming tradeoff explicit. The end state should cover agent-authored `<at user_id="...">...</at>` tags, existing inbound `mentionTargets`, card/text syntax conversion, topic replies, duplicate-final suppression, and a clear fallback when Feishu cannot notify from Card Kit updates.
