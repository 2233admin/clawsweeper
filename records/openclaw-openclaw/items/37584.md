## Summary

Keep open. Current main still has root/plugin-scoped web search settings, but no per-agent `agents.list[].tools.web.search.*` schema/type contract or runtime path. The reporter refreshed the issue on 2026-04-20, and the prior ClawSweeper review in the provided context reached the same unresolved conclusion.

## Best Possible Solution

Keep this open for maintainer design. The best path is to define and document an agent-scoped web-search settings contract, decide merge precedence with plugin-owned provider config and env fallbacks, add schema/types for `agents.list[].tools.web.search`, thread the effective per-agent search config through `createOpenClawTools()`, `createWebSearchTool()`, `runWebSearch()`, and provider contexts, then add regression tests and docs. If maintainers want provider plugins to own this entirely, add a generic SDK/runtime seam for resolving per-agent web-search credentials safely before closing the issue.
