## Summary

Keep open. Current main still has only global/plugin-owned web-search configuration and does not provide an agent-scoped `agents.list[].tools.web.search.*` schema, type contract, help/docs surface, or runtime merge path. The reporter also confirmed the request is still valid on 2026-04-20, so this is active maintainer design work rather than stale cleanup.

## What This Changes

Request to allow `agents.list[].tools.web.search` and similar nested tool settings to override global web-search provider, key, timeout, and related configuration per agent.

## Best Possible Solution

Keep this open for maintainer design. Define the accepted per-agent web-search settings contract and precedence model, document it, add schema/types/help/labels, thread the effective per-agent search config through tool creation and `runWebSearch()`, and add focused tests covering root/plugin/env fallback, SecretRef behavior, provider selection, and multi-agent isolation.
