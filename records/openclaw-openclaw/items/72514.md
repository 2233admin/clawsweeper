## Summary

Keep open. Current main has useful primitives for session visibility, parent-to-child sends, subagent list/steer/kill, restart-scoped orphan recovery, and cross-session memory/plugin options, but it still does not implement the issue’s full requested behavior: explicit retry/resume controls for aborted subagents and a seamless QClaw/WebChat-to-WeChat continuity path remain product and boundary questions.

## What This Changes

The issue requests better cross-session and cross-agent continuity across WebChat, WeChat, main agents, and spawned subagents, including communication, history/status visibility, abort recovery, and handoff behavior.

## Best Possible Solution

Keep this open as an owner-reviewed session/subagent UX task: reproduce the QClaw WebChat-WeChat flow on current main, decide which behavior belongs in core versus the WeChat plugin, and design an explicit parent-visible status/retry/resume path for interrupted subagent runs without weakening session isolation.
