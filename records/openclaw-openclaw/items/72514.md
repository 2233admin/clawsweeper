## Summary

Keep open. Current main has partial support for the reported area, including session tools, tree-scoped spawned-session visibility, subagent list/steer/kill, and bounded orphan recovery after restart-aborted runs, but it does not provide the explicit subagent retry/resume/status API or prove the QClaw WebChat-to-WeChat continuity flow against the external WeChat plugin.

## What This Changes

Request cross-session and cross-agent continuity improvements for parent/subagent communication, abort recovery, and WebChat-to-WeChat task handoff.

## Best Possible Solution

Keep this as an owner-reviewed session/subagent UX task: reproduce the QClaw WebChat-WeChat flow on current main with the external WeChat plugin, decide whether a first-class retry/resume/status API is needed beyond steer and bounded orphan recovery, and document the supported continuity path for dmScope, identityLinks, bindings, memory, and plugin-owned channel behavior.

## Reproduction Assessment

Unclear. The report gives a concrete user scenario, but it lacks current-main configuration, plugin/QClaw versions, logs, and transcript evidence needed to reproduce the WebChat-WeChat and abort paths with high confidence from this repository alone.

## Solution Assessment

No, closing as implemented is not the best way to solve the issue. Current main covers important primitives, but the remaining retry/resume/status and cross-channel continuity behavior needs maintainer product review before a narrow implementation can be chosen.
