## Summary

Keep open for maintainer review: current main already covers the image-side resolver behavior, but the PDF execution path still uses the registry-only resolver, and this PR overlaps an open narrower PDF fix while carrying an unresolved review finding in its own fallback guard.

## What This Changes

The PR exports the unknown-model helper, removes the registry-only media resolver, switches image and PDF model resolution toward `resolveModelWithRegistry`, and adds image/PDF regression tests for configured providers and fallback behavior.

## Best Possible Solution

Preserve the current image implementation on main, choose one canonical PDF-only resolver path, and land focused PDF coverage proving config-defined custom providers and configured provider `baseUrl` overrides reach the execution-time model used by native PDF providers and extraction fallback.
