## Summary

Keep this PR open. Current main still does not implement the PR's central behavior for Cloudflare Zero Trust WebSocket upgrade headers: the node run/install CLI has no repeatable header flag, the GatewayClient cannot accept or forward custom upgrade headers, node host config does not store headers, and the installed service path does not propagate OPENCLAW_NODE_HEADERS or CF_ACCESS_CLIENT_* environment values. The PR remains useful but needs maintainer/security review before merge because it handles credential-bearing headers across config, service environment, and status surfaces.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #42792. Maintainers should review the credential-handling model, confirm header precedence and redaction semantics, fix the changelog attribution, then run targeted node CLI, node daemon install, node-host, and GatewayClient tests before merging or requesting changes.
