## Summary

Keep open. Current main has transcript-backed usage and cost summaries plus provider quota/status surfaces, but it still does not implement the requested native append-only per-call usage ledger; the issue discussion and adjacent open PRs explicitly treat transcript summary work as separate from the ledger request.

## What This Changes

The issue requests a native per-model/per-call usage log for cost tracking with model, session, source, token, cache, and cost attribution so users do not have to parse session transcripts.

## Best Possible Solution

Add a core typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer the ledger while preserving transcript scanning for older data.
