## Summary

Keep this open. Current main still has transcript-backed usage summaries, per-session logs, provider quota snapshots, and Control UI usage aggregation, but it does not implement the requested native append-only per-call usage ledger with durable event IDs, source/job/run attribution, idempotent writes, backfill semantics, or ledger-first readers. The latest maintainer discussion also explicitly treats #49064 and #70205 as adjacent work, not closure for this ledger request.

## What This Changes

Request a native append-only per-call usage log or ledger with per-model token/cost fields and session/source attribution for cost tracking and model-mix optimization.

## Best Possible Solution

Keep this open until core adds a typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, with durable event IDs, provider/model, agent/session/run/source/job attribution, token/cache/cost fields, status/latency/cost metadata, and no prompt/completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill command prefer the ledger while preserving transcript scanning as a compatibility fallback; #49064 and #70205 should remain scoped as adjacent transcript-summary fixes unless retargeted.
