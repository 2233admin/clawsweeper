## Summary

Keep open. Current main still has transcript-backed token/cost summaries, provider quota snapshots, and UI aggregation, but it does not implement the requested native append-only per-call usage ledger with durable event IDs, source/run/job attribution, idempotent writes, backfill, or ledger-first readers. Maintainer discussion also treats the adjacent CLI and gateway PRs as non-closure for this ledger request.

## What This Changes

Request native per-model/per-call usage logging so users can track model mix, token use, and cost by session, source, cron/job, and time range without parsing all session transcripts.

## Best Possible Solution

Add a core typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer that ledger while preserving transcript scanning for older data and keeping provider quota snapshots separate.
