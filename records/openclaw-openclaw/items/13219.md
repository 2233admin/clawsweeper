## Summary

Keep this open. Current main still provides transcript-backed cost/session summaries, provider quota snapshots, and UI aggregation, but it does not implement the requested native append-only per-call usage ledger with durable event IDs, source/run/job attribution, idempotent writes, backfill, or ledger-first readers. The linked CLI and gateway PRs are adjacent work, and the latest maintainer discussion explicitly treats them as non-closure for this request.

## What This Changes

The issue requests native per-model usage logging through a dedicated usage ledger/log for each LLM call so users can track cost, tokens, model mix, sessions, and automated-job attribution.

## Best Possible Solution

Add a core typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt/completion content. Then make usage.cost, CLI/UI summaries, and a backfill command prefer that ledger while preserving transcript scanning for older data.
