## Summary

Keep open. Current main has supervised exec runs, scoped process records, process-tree kill fallbacks, and subagent kill plumbing, but it still does not expose a session-wide Gateway/RPC/CLI operator surface to list and terminate all active exec children for an arbitrary stored session. The closed #39960 appears to cover active stop/abort behavior, while open #38721 is adjacent shutdown cleanup work and does not supersede this broader operator request.

## What This Changes

The issue requests a session-level operator command/API that can list and terminate all exec child processes associated with a session.

## Best Possible Solution

Add an explicit additive operator contract that resolves the canonical session key, lists active supervised exec records for that session scope, terminates matching active runs through supervisor cancellation with process-tree fallback where appropriate, returns per-run status, and documents authorization/isolation behavior without silently changing default sessions.abort semantics.
