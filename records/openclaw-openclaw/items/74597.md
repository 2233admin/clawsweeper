## Summary

Keep open. The reporter narrowed the original incident to a manual source-level global install, but current main still lacks the requested guard that distinguishes a package install root from its realpath when it resolves into a mutable source checkout; open PR #74920 appears to be the active implementation candidate and is still unmerged.

## What This Changes

Add updater/status/doctor safeguards so npm-global package installs are not silently left or reported as source-checkout symlinks, and expose reliable diagnostics when install roots, service entrypoints, and runtime layout disagree.

## Best Possible Solution

Add a shared install-layout diagnostic that preserves both lexical package roots and realpaths, detects npm-global roots resolving into git/source checkouts, and surfaces clear status, doctor, update pre/post verification, and repair guidance while keeping intentional dev-channel source installs explicit.

## Reproduction Assessment

Yes. The issue includes direct journalctl evidence for npm install -g /opt/openclaw-source, and current main has a focused update-check test that reproduces the classification gap by expecting a symlinked git root to report as installKind git.

## Solution Assessment

Yes. The guard/diagnostic direction is the narrowest maintainable fix; removing dev-channel source installs would conflict with documented behavior, while preserving lexical root plus realpath lets status, doctor, and update distinguish intentional source installs from mixed package/source layouts.
