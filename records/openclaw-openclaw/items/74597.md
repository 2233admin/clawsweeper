## Summary

Keep open. The original incident was narrowed to a manual source-level global install, but current main still lacks the requested guard that detects an npm-global package root resolving into a mutable source checkout; an open maintainer-labeled PR appears to address this, but it has not landed.

## What This Changes

Request to diagnose and guard against npm-global OpenClaw installs whose package root resolves through a symlink to a source checkout, especially across update/status/doctor surfaces.

## Best Possible Solution

Land or split a focused implementation that records lexical package roots and realpaths, detects npm-global roots resolving into source checkouts, and surfaces actionable status/doctor/update verification without removing intentional dev-channel checkout behavior.

## Reproduction Assessment

Yes. The existing update-check unit fixture shows a symlinked package root resolving to a git repo is classified as a git install, and the reporter supplied host journal evidence for the mixed-layout failure mode.

## Solution Assessment

Yes, the diagnostic/guard direction is the right maintainable solution. Removing dev-channel source installs would conflict with documented behavior; the safer path is explicit layout detection, warnings, and post-update verification.
