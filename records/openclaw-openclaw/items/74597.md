## Summary

Keep open. The reporter narrowed the exact incident to a manual source-level global install, but the remaining guard request is still valid on current main: update/status still collapse a symlinked package root into git/source mode, and the status/doctor service-layout diagnostic appears to be covered by open PR #74920 rather than shipped code.

## What This Changes

The issue asks OpenClaw to detect and guard against npm-global package installs whose global `node_modules/openclaw` root resolves to a mutable source checkout symlink, especially in update/status/doctor flows.

## Best Possible Solution

Add a shared install-layout diagnostic that preserves both the lexical package root and its realpath, detects package roots that resolve into source checkouts, and surfaces clear status, doctor, update pre/post verification, and repair guidance while keeping intentional dev-channel source installs explicit.

## Reproduction Assessment

Yes. The issue includes direct journal evidence for the source checkout global install, and current source/tests show a deterministic reproduction of the diagnostic gap: a symlinked OpenClaw root pointing at a git repo is expected to report `installKind: "git"`.

## Solution Assessment

Yes. A layout guard/diagnostic is the narrowest maintainable direction; removing dev-channel source installs would conflict with documented behavior, while preserving lexical root plus realpath lets status, doctor, and update distinguish intended source installs from mixed package/source layouts.
