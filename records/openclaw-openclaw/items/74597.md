## Summary

Keep open. Current main still lacks the requested package-root/source-checkout guard: update status collapses a symlinked package root that resolves to a git checkout into `installKind: "git"`, and doctor only compares service entrypoint realpaths rather than reporting the mixed npm-global/source layout. Related PR #74920 appears to be the active implementation path, but it is still open and unmerged in the provided context, so there is no closeable fix on main yet.

## What This Changes

Detect and guard against npm-global OpenClaw package roots that resolve to mutable git/source checkouts so update, status, doctor, and gateway repair do not leave mixed package/source layouts.

## Best Possible Solution

Add a shared install-layout diagnostic that records both the lexical package root and its realpath, detects package roots that resolve into source checkouts, and surfaces actionable status/doctor/update guidance while preserving the documented dev-channel source install path.

## Reproduction Assessment

Yes. The remaining diagnostic gap has a high-confidence unit-level reproduction: create a symlinked OpenClaw root that points at a git checkout and `checkUpdateStatus` currently reports it as `installKind: "git"`; the existing test already asserts that behavior.

## Solution Assessment

Yes. A focused layout diagnostic/guard is the narrow maintainable direction; removing the dev-channel source install flow would conflict with documented behavior, and current main does not yet provide the needed package-root/source-root distinction.
