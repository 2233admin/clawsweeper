## Summary

Keep open. Current main still does not contain this PR's Docker WhatsApp helper scripts, compose/setup env passthrough for bundled plugins and Playwright cache, or the proposed UID/GID runtime-user wiring. The PR is also not safe to treat as obsolete because its Docker permission and teardown changes are security/ops-sensitive and still need maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review. A mergeable version should resolve the Compose interpolation and teardown-safety concerns, add focused coverage for compose env passthrough, setup persistence, UID/GID ownership behavior, and Playwright cache handling, then coordinate with #41624 or explicitly supersede it. If maintainers decide a dedicated WhatsApp Docker helper is not a supported surface, close it manually with that product decision rather than as already implemented.
