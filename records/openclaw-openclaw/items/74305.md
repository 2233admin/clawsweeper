## Summary

Keep open. Current main still preserves configured Codex ACP adapter arguments and then unconditionally appends runtime model/thinking `-c` overrides, so the reported duplicate-key path is still plausible; the latest release also lacks a duplicate-key guard. Open PR #74339 is explicitly paired to this issue and should be reviewed, landed, or closed before the issue is resolved.

## What This Changes

The issue requests that ACPX Codex worker runs honor configured model/thinking overrides without crashing, or return a specific validation error when those overrides are unsupported.

## Best Possible Solution

The desired end state is an ACPX-owned command-normalization fix that preserves configured Codex ACP adapter args, skips appending runtime `-c` keys already present in the configured command, covers full and partial overlap in tests, and closes this only after PR #74339 or an equivalent focused fix lands.

Yes. The report provides concrete operator steps, and current main statically reproduces the risky command construction: configured Codex ACP `-c` args are preserved, while `sessions_spawn` model/thinking overrides can append the same keys again.

## Solution Assessment
