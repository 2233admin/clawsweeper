## Summary

Keep this issue open. Current main has several related ACPX/Codex improvements after v2026.4.26, including the bundled Codex ACP 0.12.0 dependency, wrapper preservation of configured adapter args, and Codex ACP model/thinking startup mapping. However, the exact reported configured command path with `gpt-5.5` plus `model_reasoning_effort=medium` is not live-verified or directly covered, and current code still relies on the adapter to accept configured `-c` values rather than guaranteeing a specific validation error.

## What This Changes

The issue requests that ACPX Codex worker runs handle configured Codex model and reasoning overrides successfully, or fail with a specific validation error instead of a generic internal ACP runtime error.

## Best Possible Solution

Keep the issue open until maintainers run a live current-main ACPX Codex smoke with Codex auth and the configured command `-c model=gpt-5.5 -c model_reasoning_effort=medium`, or add a targeted OpenClaw validation/error-translation fix plus tests covering configured command args for both `model` and `model_reasoning_effort`.
