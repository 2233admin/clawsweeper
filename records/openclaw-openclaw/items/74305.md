## Summary

Keep open. Current main still preserves configured Codex ACP adapter arguments and then appends runtime `model` and `model_reasoning_effort` `-c` overrides without checking whether those keys are already present, matching the reported failure path. Open PR #74339 explicitly references this issue as its fix, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to make ACPX Codex worker runs succeed, or fail with a specific validation error, when Codex model/thinking overrides are configured.

## Best Possible Solution

Land a narrow ACPX runtime fix that detects existing configured Codex ACP `-c model` and `-c model_reasoning_effort` keys before appending runtime overrides, with coverage for full overlap, partial overlap, and wrapper-command variants; then close this issue after the fix reaches main or a release.

## Reproduction Assessment

Yes. The issue provides concrete operator steps, and current main statically reproduces the risky construction path: configured Codex ACP args are preserved, then runtime model/thinking overrides can append the same config keys before session startup.

## Solution Assessment

Yes. The narrow maintainable solution belongs in the ACPX Codex command-normalization/runtime override path, while `sessions_spawn` should continue forwarding generic ACP model and thinking options.
