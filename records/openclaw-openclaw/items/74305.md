## Summary

Keep this issue open. The report is recent and actionable, current main still has a plausible unmerged duplicate `-c model` / `-c model_reasoning_effort` append path for Codex ACP runtime overrides, the latest tagged release also lacks the guard, and open PR #74339 explicitly references this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or rejected.

## What This Changes

The issue reports that ACPX Codex worker runs fail with `AcpRuntimeError: Internal error` when the configured Codex ACP command includes model and reasoning-effort `-c` overrides.

## Best Possible Solution

Land a focused ACPX runtime fix, likely via #74339, that preserves configured Codex ACP args without duplicating model/thinking config keys, adds regression coverage for configured and runtime override combinations, and then close this issue after merge or after a live current-main smoke proves the configured-command path no longer fails.

## Reproduction Assessment

Yes. The issue gives a concrete OpenClaw version, command line, `sessions_spawn` shape, model/thinking settings, and a control case where removing the configured `-c` args makes the workflow succeed; current-main source also shows the remaining duplicate-append path that can reproduce the same class of failure.

## Solution Assessment

Yes, the narrowest maintainable fix appears to be in the ACPX runtime command-normalization path, not core agent spawning. PR #74339’s direction of detecting existing Codex ACP `-c` config keys before appending runtime overrides matches that boundary, but it still needs normal maintainer review and validation.
