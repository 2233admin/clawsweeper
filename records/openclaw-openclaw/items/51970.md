## Summary

Keep this PR open. Current main still has the suffix-revealing `maskApiKey` behavior, so the PR is not obsolete or implemented on main. The PR is a focused candidate for the remaining credential-display problem, but it still needs maintainer revision because the proposed `> 4` branch exposes more of 5- and 6-character secrets than current main, and one new test assertion is in the wrong test block.

## Best Possible Solution

Keep this PR open and request a focused revision: choose a masking policy that never exposes suffixes and does not increase exposure for 5- or 6-character secrets, update `maskApiKey`, move the misplaced test assertion, add explicit short-key and suffix-nondisclosure regression coverage, then run the targeted `mask-api-key` tests plus the models auth overview/chat auth label tests that cover the affected display surfaces.
