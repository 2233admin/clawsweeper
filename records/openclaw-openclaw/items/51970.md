Keep this PR open. Current main still implements and tests the older suffix-revealing `maskApiKey` behavior, so the PR is not obsolete or implemented on main. The PR is a narrow candidate for the linked credential-display problem, but it still needs a policy tweak for very short secrets before merge.

## Best Possible Solution

Keep the PR open and revise it before merge.
