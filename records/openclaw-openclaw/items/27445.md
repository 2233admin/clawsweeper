## Summary

Current main still does not implement the requested native `announceTarget` option for `sessions_spawn` or `agents.defaults.subagents`. Native subagent completion routing remains driven by requester depth and current delivery state: top-level requesters still use external delivery, while nested subagent requesters get internal `deliver=false` injection. ACP `streamTo="parent"` is separate and explicitly ignored for native subagent spawns.

## Best Possible Solution

Keep this open and implement a focused, backward-compatible core subagent announce-routing feature: add `announceTarget: "channel" | "parent"` to native `sessions_spawn` and `agents.defaults.subagents`, default to current channel behavior, route `parent` completions into the requester session with `deliver=false` and no direct channel announce, keep ACP `streamTo` separate, and update schema/docs/changelog/tests.
