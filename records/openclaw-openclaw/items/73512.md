## Summary

Keep this PR open. Current main still does not implement the proposed behavior, and the existing tests/docs currently assert the opposite semantics for QMD `searchMode: "search"`. The PR also needs revision because `embedIntervalMs` is already defaulted during config resolution, so it cannot by itself prove that `update.embedInterval` was explicitly configured.

## Best Possible Solution

Keep this PR open for maintainer review/revision. If maintainers want explicit `memory.qmd.update.embedInterval` to opt lexical/default QMD setups into embedding, preserve whether that field was explicitly configured in the resolved QMD update config, update both `shouldRunEmbed()` and `shouldScheduleEmbedTimer()`, revise the conflicting lexical-mode tests, and update the memory docs. If maintainers want `searchMode: "search"` to remain strictly BM25-only, close the linked bug and this PR with that documented behavior explained.
