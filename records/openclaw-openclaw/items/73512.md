## Summary

Keep this PR open for maintainer review. The linked user report is real enough to consider, but the proposed patch is not merge-ready because it changes the documented lexical QMD contract and uses resolved `embedIntervalMs` as if it proves the user explicitly configured `embedInterval`.

## What This Changes

The PR changes QMD embed execution and timer scheduling so a positive resolved `embedIntervalMs` bypasses the vector-search-mode guard, and adds a regression test for `searchMode: "search"` with `embedInterval: "5m"`.

## Best Possible Solution

Decide the QMD lexical-mode contract first. If explicit `memory.qmd.update.embedInterval` should opt lexical setups into embedding, carry an explicit-config flag through QMD config resolution, use that flag in both embed guards, update or split the lexical-mode tests, and revise the memory config docs; otherwise keep the shipped BM25-only contract and close the paired bug/PR with that behavior called out.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Track explicit embedInterval before bypassing search mode:** `extensions/memory-core/src/memory/qmd-manager.ts:1620`
  - body: The new guard treats `embedIntervalMs > 0` as proof that the user configured `embedInterval`, but `resolveEmbedIntervalMs(undefined)` returns the default 60-minute value. That makes default lexical QMD configs eligible for forced/boot embedding and can arm the embed timer when regular updates are disabled; preserve a raw-config or explicit flag before bypassing `qmdUsesVectors`.
  - confidence: 0.93
- **[P2] Reconcile the lexical-mode contract test:** `extensions/memory-core/src/memory/qmd-manager.test.ts:3609`
  - body: The added test expects `update` and `embed` for `searchMode: "search"` plus `embedInterval: "5m"`, but current main already has a nearby lexical-mode test with the same effective setup expecting no calls. The suite should distinguish default versus explicitly opted-in embedding, or replace the old contract if the product behavior is changing.
  - confidence: 0.9
