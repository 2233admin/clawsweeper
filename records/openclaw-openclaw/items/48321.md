## Summary

Keep open. Current main still does not add image-specific timestamp, sender, or thread position/total annotations for Slack thread images. Slack thread history gets message-level envelopes and file placeholders, while thread-starter media can still be sent through the generic media note path without temporal image metadata. The directly related implementation PR #48725 was closed unmerged.

## Best Possible Solution

Keep this issue open and implement a focused Slack-owned fix. Annotate image-bearing Slack thread context with sanitized sent time, sender, and message position/total when that image is included or referenced in model context. Keep core extension-agnostic; only extract a shared plugin SDK helper later if another bundled channel needs the same behavior.

## Clownfish Work Prompt

Implement a focused Slack fix for this issue: when Slack thread context includes or references image-bearing prior thread messages, add sanitized metadata so the model can tell when the image was sent, who sent it, and where it sits in the fetched thread history. Current main formats `filteredThreadHistory` in `extensions/slack/src/monitor/message-handler/prepare-thread-context.ts` from `historyMsg.text` and ignores `historyMsg.files`; `resolveSlackThreadHistory` in `extensions/slack/src/monitor/thread.ts` preserves `files` and turns file-only messages into `[attached: ...]`; thread-starter media can flow through `MediaPath(s)` in `extensions/slack/src/monitor/message-handler/prepare.ts` and the generic `src/auto-reply/media-note.ts` path without temporal metadata. Keep the behavior Slack-owned: do not import Slack extension production code from core `src/**`, and do not add bundled channel ids or Slack policy to generic core. Detect images by Slack file MIME/filetype/name extension, sanitize sender/file fields, and mark the annotation text as untrusted context. Include sent time from Slack `ts`, sender label/role, and message position/total within the filtered history. If maintainers choose to add an opt-out, use `channels.slack.thread.annotateImages`, update `src/config/types.slack.ts`, `src/config/zod-schema.providers-core.ts`, config help/labels, and `docs/channels/slack.md`; otherwise document the default behavior without adding config. Add focused tests for image and non-image files, file-only messages, allowlist-filtered history position counts, bot/self filtering, and thread-starter media metadata if included. Add a changelog entry because this is user-facing; credit the reporter and the prior PR author, not Codex.
