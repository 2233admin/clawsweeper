## Summary

Current main has the compaction and memory-flush model override pieces, but the requested flush-then-reset mode is still missing. The issue should stay open and be narrowed to `agents.defaults.compaction.memoryFlush.resetAfterFlush` or an equivalent maintainer-approved reset policy.

## Best Possible Solution

Keep this issue open, but narrow the remaining scope to a maintainer-reviewed `memoryFlush.resetAfterFlush` design. The best path is to add an explicit config/schema/docs surface, reset only after a verified successful memory flush, skip redundant compaction when configured, reuse existing session reset or successor-transcript machinery, and cover queue/session metadata, transcript replay, active runs, subagents/ACP, and channel delivery behavior with focused tests.
