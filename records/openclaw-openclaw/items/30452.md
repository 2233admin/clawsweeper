## Summary

Keep open. Current main implements the compaction and memory-flush model override portions, but the requested flush-then-reset/reset-after-flush mode is still missing and needs maintainer lifecycle design before a safe implementation.

## What This Changes

Add configurable compaction and memory-flush model overrides plus an optional mode that flushes durable memory and resets the session instead of continuing into compaction.

## Best Possible Solution

Keep this open but narrow it to a maintainer-approved reset-after-flush design: add an explicit config/schema/docs surface only after deciding lifecycle semantics, reset only after a verified successful memory flush, skip redundant compaction when configured, reuse the existing reset/session replay machinery where appropriate, and cover metadata, queued follow-ups, active runs, subagents/ACP, and channel delivery with focused tests.
