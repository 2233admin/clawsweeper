## Summary

Keep PR #71950 open. Current main still has the unsafe gateway/tailnet network-interface paths that the PR targets, and the existing tests still lock in throwing behavior when `os.networkInterfaces()` fails. The provided PR discussion also has maintainer signal that this is the right fix direction after preserving status diagnostics.

## Best Possible Solution

Keep #71950 open and land this PR or an equivalent targeted patch: switch gateway LAN and tailnet discovery off direct `readNetworkInterfaces()` calls, preserve diagnostic error reporting for status/display surfaces, and update the existing gateway/tailnet/network-interface tests plus changelog credit for #71721.
