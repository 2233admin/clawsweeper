## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the proposed `targetProvider` short-circuit, so the PR is not obsolete, but the patch has blocking cache and security issues in the new fast path and is also dirty against current main’s provider-discovery options.

## What This Changes

The PR adds a `targetProvider`/`targetModel` option to `ensureOpenClawModelsJson`, a disk-vs-config short-circuit for matching provider entries in `models.json`, and focused tests for match, drift, and cache behavior.

## Best Possible Solution

Rebase the PR onto current main, preserve the current provider-discovery option contract, and make the fast path prove only what it caches: either keep target-specific state separate from global readiness, or validate the whole file before populating `readyCache`. The short-circuit should also fail closed for any request-shaping disk drift, use descriptor-based no-follow file handling, and then be threaded into the actual PI call sites with regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is: configure two providers, let the `targetProvider: "openai"` short-circuit match only the OpenAI disk entry, then call `ensureOpenClawModelsJson` for another provider or with no target; the second call can hit the global `readyCache` before validating or rewriting the other provider.

## Solution Assessment

No. The direction is useful, but the current implementation is not the narrowest safe solution because it turns a provider-local validation into global readiness, leaves request-shaping model fields trusted from disk, and loses current-main discovery options during rebase.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Do not cache a provider-only match as global readiness:** `src/agents/models-config.ts:455`
  - body: This cache entry is keyed only by path and the existing fingerprint, not by `targetProvider`. After a short-circuit validates one provider, a later call for another provider or an untargeted caller can return from `readyCache` before checking that provider, leaving missing, stale, or tampered `models.json` data in place.
  - confidence: 0.94
- **[P1] Compare model transport fields before trusting disk:** `src/agents/models-config.ts:275-279`
  - body: The short-circuit intentionally ignores `models[]`, but the PI runtime later uses `discoveredModel.baseUrl` when provider-level baseUrl is absent. A disk entry with a matching provider apiKey/baseUrl/headers/auth but a malicious per-model baseUrl can therefore bypass planning and keep an attacker-controlled request destination.
  - confidence: 0.9
- **[P1] Read the same models file that was checked:** `src/agents/models-config.ts:315`
  - body: The lstat size/symlink check is separated from `fs.readFile`, so a writable `models.json` path can be swapped between check and use. Use an opened descriptor with no-follow plus `fstat`/descriptor read, and avoid path-based chmod on the checked file.
  - confidence: 0.85
- **[P2] Preserve providerDiscoveryEntriesOnly in the options type:** `src/agents/models-config.ts:177-178`
  - body: Current main passes `providerDiscoveryEntriesOnly: true` for startup warmup to keep discovery metadata-only. The new exported options type and forwarding logic omit that option, so a rebase would either fail typechecking at current call sites or broaden startup provider discovery again.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff changes security-sensitive model/auth readiness logic and still has concrete trust-boundary issues before it can safely merge.

Concerns:

- **[high] Provider-local proof poisons global readiness cache:** `src/agents/models-config.ts:455`
  - body: A successful short-circuit for one provider populates the global `readyCache`; later callers can skip validation for different providers and keep stale or tampered disk settings active.
  - confidence: 0.94
- **[high] Per-model request destinations remain trusted from disk:** `src/agents/models-config.ts:275`
  - body: The short-circuit excludes `models[]` from comparison even though current PI runtime can use `discoveredModel.baseUrl` for outbound requests, allowing model-level destination drift to survive.
  - confidence: 0.9
- **[high] Path check and path read are raceable:** `src/agents/models-config.ts:315`
  - body: The code checks `models.json` with `lstat` and then reopens the path with `readFile`, leaving a TOCTOU window for symlink or file replacement before the trusted read/chmod path.
  - confidence: 0.85
