## Summary

Keep this PR open. Current main still does not fully implement either half of the PR: skill discovery remains immediate-child based except for a narrow `dir/skills/*/SKILL.md` heuristic, and gateway startup still does not warn when the selected provider lacks usable auth. The PR is also not safe to merge as written because its recursive scanner and startup auth check need security and correctness rework.

## Best Possible Solution

Keep this PR open for maintainer rework or contributor split. The recursive skill-scanning half should align with or defer to #69663 and preserve bounded traversal, realpath containment, watcher behavior, docs, and regression tests. The startup-auth warning should be a separate best-effort diagnostic using the same non-throwing, read-only runtime auth contract as `hasAvailableAuthForProvider`, or be explicitly dropped if maintainers decide current onboarding/default-model behavior already resolves the original user symptom.
