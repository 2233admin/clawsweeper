## Summary

Keep this PR open. Current main still scopes control-plane write budgets by `deviceId|clientIp` whenever either identity signal is present, while the PR proposes appending `connId` for every connected session. That behavior is not implemented on main and changes throttling for gateway config/update writes, so it needs maintainer policy/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer gateway policy review. If per-connection control-plane write budgets are desired, rebase onto current main, preserve the prune/cap memory safeguards, add same-device/IP multi-connection regression coverage plus bucket-bound coverage, update the gateway docs/changelog, and complete the security-impact section. If maintainers prefer aggregate `deviceId+clientIp` throttling, close the PR manually with that product/security rationale.
