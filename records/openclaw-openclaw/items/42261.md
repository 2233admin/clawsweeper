## Summary

Current main still matches the reported bug. Live reasoning is enabled only when an `onReasoningStream` callback exists, and the `stream: "thinking"` WebSocket agent-event broadcast is still behind that callback guard. The related fix attempts mentioned in the discussion are closed and unmerged, so this remains actionable in the source tree.

## Best Possible Solution

Keep this issue open. Fix `src/agents/pi-embedded-subscribe.ts` so live reasoning state is based on stream mode plus the existing visibility gates, emit `stream: "thinking"` before optionally invoking `params.onReasoningStream`, and add a callback-absent regression test proving WS/agent-event clients receive thinking events. Then update `/reasoning stream` docs and acknowledgement text, and add Control UI handling if WebChat display is part of the intended surface.
