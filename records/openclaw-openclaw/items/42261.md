## Summary

Current main still matches the reported bug: live reasoning is only enabled when `onReasoningStream` exists, and the `stream: "thinking"` agent-event broadcast is still behind that same optional callback guard. Webchat/Control UI `chat.send` remains callback-less, while the gateway transport would carry non-tool agent events if they were emitted. The issue should stay open.

## Best Possible Solution

Keep this issue open. Fix `src/agents/pi-embedded-subscribe.ts` so `streamReasoning` is enabled from stream mode plus the existing visibility gates rather than callback presence, emit `stream: "thinking"` before optionally invoking `params.onReasoningStream`, and add a callback-absent regression test proving WS/agent-event clients receive live thinking. Then update the `/reasoning stream` docs and acknowledgement if Webchat/Control UI support is intended user-facing behavior.
