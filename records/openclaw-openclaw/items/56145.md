## Summary

Keep this PR open. Current main still trims each non-streaming reply payload before recombining `/btw` side results and still routes final payload display/transcript text through helpers that trim payload text, so the PR’s central gateway-layer indentation fix is not implemented on main. The merged #55960 work covers the adjacent directive-tag parser layer, but it does not supersede this gateway recombination bug.

## Best Possible Solution

Keep the PR open as a plausible focused bug fix, rebase it onto current main, and implement the combiner at the shared gateway final-display/transcript and `/btw` side-result recombination points. Preserve authored chunk boundaries, trim only outer line-break boundaries including CRLF/CR/LF, and keep targeted regression tests for multi-payload YAML/list indentation, `/btw`, single-newline preservation, and CRLF boundaries.
