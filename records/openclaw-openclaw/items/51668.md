## Summary

Keep this PR open. Current main still lacks the PR’s intended gateway-layer outbound secret sanitization with named reference handles, and the PR remains a relevant security-boundary implementation candidate. The earlier Greptile P0/P1 findings were addressed in the PR discussion, but the remaining hook, structured-payload, config-contract, and credential-surface concerns still need maintainer review before this should land.

## Best Possible Solution

Keep the PR open for maintainer security review or a revised patch. The strongest path is to sanitize at the final outbound send boundary after hooks, rendering, and normalization; cover text/caption, presentation, interactive, and channelData string surfaces; derive named handles from the canonical SecretRef and resolved runtime/auth-profile credential surfaces; add any sanitizeSecrets setting to schema, exported types, docs, and generated config help; and add regression tests for hook rewrites, structured payload delivery, key=value/JSON tokenization, and low-entropy configured secrets.
