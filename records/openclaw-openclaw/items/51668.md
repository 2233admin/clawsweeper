## Summary

Keep open. Current main does not implement this PR’s intended gateway-layer outbound secret sanitization, and the PR still needs design/security fixes before it would be safe to merge.

## Best Possible Solution

Keep this PR open for maintainer review or ask for a revised patch that implements the sanitizer at the final outbound send boundary, covers text/presentation/interactive/channelData surfaces, builds credential handles from the canonical secret/config metadata rather than a shallow config walk, adds the `messages.sanitizeSecrets` schema/type/docs contract if that option remains, and includes regression tests for hook rewrites and structured payload delivery.
