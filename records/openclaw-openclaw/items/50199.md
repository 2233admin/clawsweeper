## Summary

Keep open. Current main still has deterministic skill source precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and truncation warnings, but it has no user-configurable skill priority/ordering contract and no `openclaw skills priority` CLI surface. The closed unmerged priority PR and open truncation-related issue are useful context, but neither implements or supersedes the broader request.

## What This Changes

Add user-configurable skill priority/ordering so overlapping skills can be preferred or used as fallbacks consistently, with config support and CLI management.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema and docs, apply it only after eligibility and per-agent filters are resolved and before prompt count/character limits, preserve deterministic non-priority ordering for prompt-cache stability, and wire the same order through snapshots, runtime materialization, CLI inspection, docs, and the related truncation/pinning work.
