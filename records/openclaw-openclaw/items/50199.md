## Summary

Keep open. Current main has deterministic skill source precedence, agent allowlists, per-skill enabled/env/apiKey config, and alphabetical prompt ordering, but it does not provide a user priority/ordering field or `openclaw skills priority` CLI surface. The request is understandable and not old enough for stale-insufficient-info cleanup, but it needs maintainer product/API judgment before a fix PR because priority could mean prompt ordering, filtering, slash-command ordering, or backend-specific resolver behavior.

## Best Possible Solution

Keep this open for maintainer design. The best path is to decide whether skill priority belongs in the existing `~/.openclaw/openclaw.json` `skills` schema, what surfaces it affects, and how it composes with source precedence and agent allowlists; then update the loader, prompt/snapshot ordering, CLI/docs, schema/help, and focused skill tests together.
