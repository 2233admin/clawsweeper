## Summary

Keep open. Current main still has deterministic skill precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and truncation warnings, but it does not provide a user-configurable skill priority/ordering contract or an `openclaw skills priority` CLI surface. The closed priority PR is unmerged and the open truncation issue is narrower, so neither supersedes this broader feature request.

## What This Changes

Add user-configurable skill priority or ordering so overlapping skills can be preferred consistently through config and CLI management.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema and docs, apply it after eligibility and per-agent filters but before prompt count/character limits, keep non-priority fallback ordering deterministic, and wire the same order through snapshots, runtime materialization, CLI inspection, docs, and related truncation/pinning work.
