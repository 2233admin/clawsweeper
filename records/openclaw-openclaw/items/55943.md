## Summary

Keep this issue open. Current main has fixed the related UTC filename problem, but the bundled session-memory hook is still registered and implemented only for `command:new` and `command:reset`; subagent completion goes through the typed plugin `subagent_ended` lifecycle path and does not call the bundled memory writer. The related combined report was closed as a duplicate pointing back here, the direct fix PR is closed unmerged, and I did not find an open PR with closing syntax for this issue.

## What This Changes

The issue requests that the bundled session-memory hook save a memory Markdown file when a subagent session created through `sessions_spawn` or secretary-style flows ends.

## Best Possible Solution

Implement a narrow lifecycle bridge that resolves the completed child session key, workspace, session entry, and transcript, then reuses the bundled session-memory writer so subagent completion creates the same dated Markdown artifact without changing existing `/new` or `/reset` behavior or writing duplicate memory files.
