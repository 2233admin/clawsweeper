## Summary

Keep this PR open. Current main still strips mixed-message `/model <provider/model> <message>` and clears the model directive before the reply pipeline can apply it, so the PR’s intended one-shot model override is not implemented. The PR is a relevant implementation candidate, but it still needs updates for unresolved review findings around stored session model overrides and auth-profile validation/mutation.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require an update that preserves mixed-message `/model` as one-shot state, applies provider/model/auth profile only to the current run, validates any one-shot auth profile through the existing provider/cooldown rules, avoids mutating persistent session model/auth-profile overrides, recomputes thinking/reasoning defaults without reapplying stored session overrides, and adds regressions for prior persisted model/profile overrides plus follow-up turn reversion.
