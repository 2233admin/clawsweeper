## Summary

Keep this PR open. Current main still parses and strips mixed-message `/model <provider/model> <message>` but clears the model directive before model selection can apply it, so the PR's intended behavior is not implemented on main. The PR remains a relevant implementation candidate, but its auth-profile and session-override review findings should be addressed before merge.

## Best Possible Solution

Keep the PR open as the focused implementation candidate. Update it to preserve mixed-message `/model` as one-shot state, apply provider/model/auth profile only to the current run, avoid writing or clearing persistent session overrides, validate one-shot auth profiles against provider/cooldown rules, recompute thinking/reasoning defaults from the effective one-shot model, and add regression tests for persisted override interactions plus follow-up turn reversion.
