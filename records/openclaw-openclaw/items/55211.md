## Summary

Keep open. Current main still dispatches internal hooks without a same-chain re-entrancy guard, while this PR remains a clean, green implementation candidate with no definite blocking bug from the read-only review. The remaining decision is maintainer semantics and priority review, especially around AsyncLocalStorage inheritance for spawned async work.

## What This Changes

The PR adds an AsyncLocalStorage-backed same-chain re-entrancy guard to `triggerInternalHook` and regression tests for same-event recursion, sequential dispatch, cross-session dispatch, and cross-event dispatch.

## Best Possible Solution

Land a narrow internal-hook recursion guard if maintainers accept the AsyncLocalStorage same-chain semantics, preferably with an explicit concurrent same-key test before merge. Keep reset-specific duplicate-dispatch handling in the reset command/session reset paths instead of using this guard as the only dedupe mechanism.
