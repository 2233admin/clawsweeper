## Summary

Current main has not fully implemented this PR's intended fix. The default group/channel path now prefers message-tool-only replies, but the supported legacy `automatic` mode still injects the blanket same-group `message` tool prohibition, so attachment/file/image workflows can still hit the prompt conflict this PR is trying to refine.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a narrow rebase that preserves the current `message_tool_only` prompt, changes only the `automatic` branch to say text replies are sent directly while files/images/attachments may use `message(action=send)` targeting the same group, and updates the existing prompt tests without weakening the current untrusted-metadata structure.
