## Summary

Keep this PR open. Current main only allows same-group message-tool sends in the newer `message_tool_only` group delivery mode; the legacy/configured `automatic` group mode still injects the blanket "Do not use the message tool" instruction that this PR is trying to refine for attachment workflows.

## Best Possible Solution

Keep this PR open and ask for a rebase onto current main. The maintainer path is to update only the automatic-mode branch of `buildGroupChatContext` so direct text replies remain automatic while files/images/attachments may use the message tool targeting the same group, preserve the existing `message_tool_only` prompt, update current prompt/tests/fixtures, and keep group subject/member metadata in the existing untrusted metadata path.
