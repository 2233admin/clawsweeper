## Summary

Keep this PR open. Current main moved the default group/channel reply mode to message-tool-only, which reduces the default impact, but the configured legacy `automatic` mode still injects the blanket same-group message-tool prohibition this PR is trying to refine for attachment/file/image workflows.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a rebase onto current main that updates only the automatic-mode branch of `buildGroupChatContext`: keep direct text replies automatic, allow files/images/attachments to use `message(action=send)` targeting the same group, preserve the current `message_tool_only` prompt, and update the current prompt tests/fixtures without moving group subject or member metadata out of the untrusted metadata path.
