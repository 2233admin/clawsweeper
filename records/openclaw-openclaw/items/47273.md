## Summary

Keep open. Current main still excludes Darwin in the affected doctor startup optimization helper, and the issue is paired with open PR #47291 as the implementation path, so it should remain open until that PR or an equivalent patch lands.

## What This Changes

The issue asks OpenClaw to include macOS Darwin hosts in memory/startup optimization detection so available system memory information is not skipped by Linux-only gating.

## Best Possible Solution

Review and land #47291, or replace it with an equivalent focused patch that includes Darwin ARM/low-memory hosts in `noteStartupOptimizationHints` and adds Darwin regression tests. Keep broader agent memory telemetry separate unless maintainers decide that product surface is also required.

## Reproduction Assessment

Yes. A direct unit-level call to `noteStartupOptimizationHints` with `platform: "darwin"`, `arch: "arm64"`, low `totalMemBytes`, and a `noteFn` reproduces the skip because current main returns before calling `noteFn`.

## Solution Assessment

Yes for the proven bug surface. The narrow maintainable solution is to update the doctor helper platform predicate and colocated startup optimization tests; changing unrelated prompt, memory-file, or agent telemetry paths would exceed the evidence in this report.
