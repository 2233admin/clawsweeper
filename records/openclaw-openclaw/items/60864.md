## Summary

Keep open. Current main has substantial adjacent restart recovery, including one-shot gateway restart continuations, durable session-delivery queue handoff, main-session restart recovery, subagent orphan recovery, and Task Flow durable state. It still does not implement this issue's first-class task checkpoint lifecycle with paused_for_restart/resuming/resumed/failed_resume states, plan cursor persistence, original/new run linkage, and automatic resume for restart-required config work.

## Best Possible Solution

Keep this issue open as the canonical RFC for first-class task checkpoint and auto-resume semantics. The best implementation path is to build on the shipped restart sentinel/session-delivery queue for routed wakeup, Task Flow for durable step state where applicable, and the task registry for observable lifecycle, then add an explicit secret-safe checkpoint contract with idempotent claim/resume, failure states, plan cursor data, and original/new run linkage.
