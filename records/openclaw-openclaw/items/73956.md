## Summary

Keep open. Current main still has the agent-run assistant-media append path that can persist a supplemental `:assistant-media` assistant transcript from fallback text when media references do not resolve, and two open PRs already target this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to stop Control UI/WebChat from persisting duplicate gateway-injected `:assistant-media` assistant replies when stale or unplayable media/TTS references fall back to the final answer text.

## Best Possible Solution

Land one narrow gateway fix, choosing one canonical PR, that appends supplemental agent-run assistant-media transcript entries only when resolved non-text media content exists while preserving normal model transcripts and non-agent command/TTS fallback behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an agent-run final or block payload with visible text plus a stale or unembeddable media URL: the media-bearing guard passes, media resolution returns no media message, and the append path can persist fallback text with an `:assistant-media` idempotency key.

## Solution Assessment

Yes. Guarding the supplemental append on resolved media content is the narrowest maintainable fix; PR #73962 and PR #74502 both target that direction, so maintainers should review and land one canonical fix rather than open another repair lane.
