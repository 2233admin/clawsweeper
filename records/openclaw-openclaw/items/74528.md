## Summary

Keep this PR open. Current main still enforces a 30s timeout on Codex app-server dynamic tool calls plus an independent 30s client fallback, while `image_generate` already supports longer configured and per-call timeouts. The PR is a narrow, applicable fix with focused regression coverage, and I did not find a blocking correctness or security issue in the diff.

## What This Changes

The PR raises the Codex app-server dynamic-tool timeout ceiling/fallback to 600s and resolves dynamic tool deadlines from per-call `timeoutMs` or configured `imageGenerationModel.timeoutMs`, with regression tests.

## Best Possible Solution

Review, validate, and merge this PR or an equivalent owner-approved patch so Codex dynamic tool calls honor advertised tool timeout budgets while retaining a bounded maximum and fallback.
