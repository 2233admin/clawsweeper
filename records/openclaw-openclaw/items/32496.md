## Summary

Keep this issue open. Current main still accepts arbitrary model params, but the OpenAI/OpenAI-compatible stream paths do not normalize or forward direct frequency_penalty/frequencyPenalty or presence_penalty/presencePenalty params; the documented extra_body escape hatch only covers api: "openai-completions" payloads and does not cover the requested direct params or the Responses/WebSocket paths. The linked fix PRs were closed without merging, and the broader per-agent generation-parameter issues do not supersede this narrow request.

## What This Changes

Support direct frequency_penalty and presence_penalty model params for OpenAI-compatible providers so configured sampling penalties are forwarded to provider requests.

## Best Possible Solution

Keep this issue open as the canonical narrow request for OpenAI/OpenAI-compatible sampling-penalty passthrough. Implement a focused transport change that normalizes finite numeric frequency_penalty/frequencyPenalty and presence_penalty/presencePenalty from the existing merged params sources, forwards them through OpenAI Responses, OpenAI Completions, and OpenAI WebSocket payload builders where supported, adds targeted regression tests, and documents the supported scope without folding in the broader #47852 or #71350 programs.
