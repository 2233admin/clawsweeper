## Summary

Keep open. Current main has adjacent retry caps, provider/profile cooldown and fallback, manual abort/reset, and bounded restart/orphan recovery, but it does not provide the requested durable session-level unhealthy-state circuit breaker that pauses or rotates a failing session after consecutive failures with operator-visible state.

## What This Changes

The issue requests a gateway/runtime session-level circuit breaker that stops automatic retries after repeated same-session failures, marks the session unhealthy, and exposes operator-visible recovery actions.

## Best Possible Solution

Add a documented gateway/session-owned breaker with persisted per-session failure state, configurable defaults and per-agent overrides, reset-on-success/manual-resume semantics, clear operator-visible session state, and integration across interactive replies, heartbeat, cron, and automatic resume paths while reusing existing session reset/abort services where appropriate.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. The high-confidence check is source inspection: current main lacks the requested persisted breaker state, config surface, and enforcement points even though adjacent retry/fallback guards exist.

## Solution Assessment

Unclear for the exact requested policy shape. A persisted session-level breaker is the maintainable direction, but pause versus reset versus fresh-session rotation and UI/alert semantics should be consolidated with #42933, #45389, #52384, and #59161 before implementation.
