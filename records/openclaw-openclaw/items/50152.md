## Summary

Keep open. Current main has an opt-in `compat.requiresStringContent` path that covers part of this problem, but it does not implement this PR’s requested NVIDIA/default OpenAI-compatible provider behavior. The bundled NVIDIA catalog still declares `api: "openai-completions"` without `compat.requiresStringContent`, so the linked #50107 default configuration is not clearly fixed on main.

## Best Possible Solution

Keep the PR open, but steer it toward the current compat architecture: either set `compat.requiresStringContent: true` for NVIDIA and any other bundled string-only OpenAI-compatible models, or deliberately broaden normalization with maintainer approval. Reuse `flattenCompletionMessagesToStringContent` or an equivalent shared helper, add NVIDIA/#50107 regression coverage, update provider docs if behavior/config changes, and resolve the review concerns about annotated blocks, signature metadata, mixed content, and text-block separators.
