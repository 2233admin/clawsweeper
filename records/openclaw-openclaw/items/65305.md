## Summary

Keep this issue open. Current main still rejects `messages.groupChat.historyLimit: 0` through `GroupChatSchema`, while docs and sibling history-limit schemas treat `0` as the disable value. PR #65359 is still the open unmerged implementation candidate, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Allow `messages.groupChat.historyLimit` and per-agent `groupChat.historyLimit` to accept `0` consistently with other history-limit config fields.

## Best Possible Solution

Update `GroupChatSchema.historyLimit` to use `.min(0)`, refresh generated schema artifacts if required by the repo workflow, and add focused regression coverage that accepts zero and positive values while rejecting negative values.
