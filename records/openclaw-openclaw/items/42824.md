Keep open. Current main and v2026.4.27 still leave unset IRC block streaming as `disableBlockStreaming: undefined`, so IRC can inherit the global block-streaming default; the PR remains a focused implementation candidate with no definite correctness or security defect found in the proposed diff.

## What This Changes

The PR modifies `extensions/irc/src/inbound.ts` to make IRC block streaming explicit opt-in and adds `extensions/irc/src/inbound.policy.test.ts` regression coverage for unset, false, and true account config values.

## Best Possible Solution

Merge or port the IRC-owned default so final IRC replies are preserved unless `channels.irc.blockStreaming` or an account override is explicitly true, keeping the unset/false/true regression coverage with the IRC plugin tests.
