## Summary

Keep this PR open. Current main still has the run-side typing TTL gap: keepalive ticks call the typing callback through `triggerTyping()`, but they do not refresh `createTypingController`'s own TTL, so the controller can still clean up during a long active run. The PR is a narrow, relevant fix with a direct regression test, and the security pass found no supply-chain or permission-sensitive changes.

## What This Changes

The PR refreshes the typing controller TTL on each keepalive tick and adds a regression test that advances beyond the original TTL while typing ticks continue.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing, unless maintainers choose to fold the same one-line TTL refresh and regression test into the broader #65919 active-run presence work. The implementation should preserve cleanup after run completion, dispatch idle, and sealed controllers.
