## Summary

Keep this PR open. Current main still has the header-bound ByteString crash class for pasted non-Latin1 secret values, so the PR is not obsolete. The proposed patch is still too broad because it rewrites generic SecretInput normalization, which can mutate valid Unicode credentials and change configured-presence checks outside HTTP-header usage.

## Best Possible Solution

Keep this PR open, but request changes rather than merging the generic normalization patch. Preserve trim-only semantics for normalizeSecretInputString, hasConfiguredSecretInput, and non-header credentials. Apply normalizeSecretInput or a clearly named header-value-safe helper only at HTTP-header assembly boundaries, starting with Comfy Cloud X-API-Key and shared provider request header/auth assembly. Add regression coverage for pasted U+2502 in a header-bound path plus guard coverage proving Unicode Matrix/gateway password values and configured-presence checks remain unchanged.
