## Summary

Current main has chunking, preview streaming, tool-call loop detection, and abort plumbing, but it does not implement the requested assistant-output repetition circuit breaker with halt-and-confirm behavior. This is still a valid core safety/product request rather than a cleanup close.

## Best Possible Solution

Keep this issue open for maintainer product/design review. The best implementation would likely be an opt-in core assistant-output loop guard near the embedded stream subscription or stream wrapper layer, with conservative detection, clear config, provider-safe abort behavior, a single user-facing alert, and an explicit decision on whether approval resumes a queued follow-up or only cancels/retries the turn.
