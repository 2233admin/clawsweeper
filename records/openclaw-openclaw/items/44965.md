## Summary

Keep open. Current main has streaming chunking, duplicate final-event suppression, tool-call loop detection, and idle timeouts, but no assistant-output repetition circuit breaker that halts a repeating model stream, alerts the user, and waits for confirmation.

## What This Changes

Request a core streaming safeguard that detects repeated assistant output, halts the LLM stream, alerts the user, and requires approve/cancel before continuing.

## Best Possible Solution

Add an opt-in core assistant-output loop guard near the stream subscription or stream-wrapper boundary, with conservative phrase/window detection, provider-safe abort behavior, one clear user-facing alert, config/docs/tests, and explicit semantics for what confirmation does after a stopped stream.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. Source inspection gives a high-confidence absence check: repeated assistant deltas would pass through the existing stream/chunk path without a repetition threshold or halt-and-confirm flow.

## Solution Assessment

Unclear until maintainers define the product semantics. The requested core stream guard is the right area, but pause/resume confirmation needs design because provider streams are typically abortable rather than resumable.
