## Summary

Keep this issue open. Current main still requests Telegram edited-message updates but does not register an inbound `edited_message` handler, and open PR #73104 is already the paired implementation candidate using closing syntax.

## What This Changes

Request to add debounced Telegram `edited_message` inbound handling so streaming bot edits are coalesced and the final edited message reaches the agent.

## Best Possible Solution

Keep the fix extension-owned in the Telegram plugin: register an `edited_message` handler that reuses the normal inbound pipeline, debounces rapid edits by chat/message id, skips bot-authored echoes, and covers registration/coalescing/dispatch behavior with focused Telegram tests.

## Reproduction Assessment

Yes. A synthetic grammY `edited_message` update or focused unit test against current main would reproduce the drop because Telegram can request that update type while no registered handler dispatches it.

## Solution Assessment

Yes for the implementation direction, but not yet complete on main. The narrow maintainable path is the Telegram plugin handler with debounce coverage; review and land or replace #73104 rather than starting a parallel fix.
