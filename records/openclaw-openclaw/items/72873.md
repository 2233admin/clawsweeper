## Summary

Keep open. Current main still requests Telegram edited-message update types but does not register an `edited_message` handler, so edited Telegram messages still have no path into the inbound pipeline. The linked open PR #73104 is the active implementation candidate and should be reviewed, landed, or replaced before this issue closes.

## What This Changes

Add Telegram `edited_message` handling so streamed bot-message edits are coalesced and processed as inbound content instead of leaving OpenClaw with only the initial partial message.

## Best Possible Solution

Keep the fix owned by the Telegram plugin: add an `edited_message` handler that reuses the inbound path, debounces rapid edits by chat/message id, avoids bot-authored echoes, and covers registration, coalescing, dedupe, and dispatch behavior with focused tests.

## Reproduction Assessment

Yes. A synthetic grammY `edited_message` update is enough to reproduce the source-level gap because current main asks Telegram for that update type but has no registered handler that forwards it into the inbound pipeline.

## Solution Assessment

Yes. The narrow maintainable solution is a Telegram runtime handler with debounce and dedupe coverage; the open closing PR should be reviewed, landed, or replaced before this tracker closes.
