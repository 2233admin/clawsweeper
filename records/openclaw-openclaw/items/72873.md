## Summary

Keep open. Current main still requests/accepts Telegram `edited_message` updates but does not register a Telegram runtime handler for them, and open PR #73104 is the active closing implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue asks the Telegram plugin to process `edited_message` updates, with debounce and dedupe handling, so streaming bot edits are delivered as final inbound content instead of only the initial partial message.

## Best Possible Solution

Keep the fix extension-owned in the Telegram plugin: add an `edited_message` handler that normalizes edited messages through the existing inbound pipeline, coalesces rapid edits by account/chat/message id, skips bot-authored echoes, and lands with focused registration, debounce, dedupe, and dispatch tests.

## Reproduction Assessment

Yes. A synthetic grammY update containing `edited_message` reproduces the drop because current main requests that update type but registers no handler that can pass it into the inbound Telegram message pipeline.

## Solution Assessment

Yes for the implementation direction. A narrow Telegram plugin handler with debounce coverage is the maintainable fix boundary; this issue should close only after PR #73104 or an equivalent replacement lands.
