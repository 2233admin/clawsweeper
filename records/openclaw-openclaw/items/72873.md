## Summary

Keep open. Current main still subscribes to Telegram edited-message update types but has no Telegram `edited_message` runtime handler, and open PR #73104 is already the active unmerged closing implementation candidate.

## What This Changes

Handle Telegram `edited_message` updates so edited or streaming Telegram messages are debounced and processed instead of leaving OpenClaw with only the initial partial message.

## Best Possible Solution

Keep the fix inside the Telegram plugin: register `edited_message`, reuse the inbound message path, coalesce rapid edits by chat and message id, skip bot-authored echoes, and cover registration, debounce, dedupe, and dispatch behavior with focused Telegram tests.

## Reproduction Assessment

Yes. A synthetic grammY `edited_message` update reproduces the source-level gap because current main asks for that update type but has no registered handler to send it through the inbound pipeline.

## Solution Assessment

Yes. A narrow Telegram plugin fix with debounce and dedupe coverage is the maintainable approach; because PR #73104 is already open with closing syntax, this issue should remain open until that PR lands or is replaced.
