## What This Changes

The PR changes web_fetch provider resolution so non-sandboxed explicit and runtime-selected providers can come from non-bundled plugins, preserves bundled-only sandboxed fetch resolution, and updates/adds runtime regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow fix after targeted tests and CI, keeping sandboxed web_fetch restricted to bundled providers while allowing non-sandboxed web_fetch to honor explicit and runtime-selected non-bundled plugin providers.

## Reproduction Assessment

Yes. The linked issue gives concrete external-plugin setup steps, and current main has a source-level reproduction because resolveWebFetchDefinition filters candidates to bundled providers before configured or runtime-selected provider ids can match.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix I found: use the existing runtime-provider resolver for non-sandboxed runtime-preferred lookup, allow explicit non-bundled provider lookup, and keep the sandboxed bundled-only branch.
