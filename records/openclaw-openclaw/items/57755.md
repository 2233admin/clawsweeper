## Summary

Keep open. Current main still emits only the payload/meta JSON envelope for `openclaw agent --json`, then performs delivery without surfacing a machine-readable delivery outcome. The requested `deliveryStatus` implementation and tests live only in the active draft PR stack, and the author explicitly marked #57755 as waiting on #57843/#53961 rather than abandoned or superseded.

## Best Possible Solution

Keep #57755 open as useful active delivery-status work. After #57843 lands or is otherwise resolved, rebase the stack, consume the delivery outcome metadata for partial/cancelled sends, decide whether the JSON surfacing belongs in #53961 or #57755, and keep the `--json --deliver` envelope tests that prove automation gets structured status without plain-text stdout pollution.
