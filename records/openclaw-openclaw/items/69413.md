## Summary

Keep PR #69413 open. Current main still hydrates Signal native mention placeholders into @uuid/@phone text and then evaluates `wasMentioned` only through configured/display-name regexes, so the reported `requireMention` failure remains plausible. The PR is a focused Signal plugin bug fix, and the provided PR context shows the span-validation review concern was addressed in the follow-up commit.

## Best Possible Solution

Keep the PR open and review it as a real Signal plugin bug fix. The best path is to land or adapt the native mention-target detection in `extensions/signal/src/monitor/mentions.ts` and `event-handler.ts`, keep the span validation from the follow-up commit, add the UUID/phone regression tests, and update Signal docs that currently say native mentions are unsupported. Consider either fixing `canDetectMention` for native-only detection in this PR or tracking that as a narrow follow-up.
