## Summary

Keep this PR open. Current main still hydrates Signal native mention placeholders into @uuid/@phone text and then decides wasMentioned only through configured or derived text mention regexes, so a native Signal mention of the bot account can still miss when the regex is based on the bot display name. The PR is a focused Signal plugin fix with relevant tests, and its second commit addresses the security review concern about forged or stale mention spans.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or rework the focused Signal plugin fix after confirming the span-validated native mention detection, adding or adjusting Signal docs for native mention behavior, and deciding whether the metadata-only canDetectMention gap belongs in this PR or a follow-up.
