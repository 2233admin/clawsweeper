## Summary

Keep open. Current main has adjacent agent-to-agent building blocks, including `openclaw agent --to --deliver`, `sessions_send`, inter-session provenance, and a bounded A2A reply/announce path, but it does not implement the requested teleconference lifecycle primitive, meeting ledger, named CLI verbs, or transcript surface. The maintainer comment explicitly narrows this into an OpenClaw agent-to-agent coordination feature layered above #63789, and #63789 is still an open draft docs-only proposal rather than shipped code.

## What This Changes

The issue requests a minimum-viable agent teleconference CLI primitive layered over existing agent/session primitives, with explicit dial/pickup/send/wrap/list/show/transcript verbs, visible channel frames, and a durable file-backed meeting transcript ledger.

## Best Possible Solution

Keep the RFC open as the focused tracker for a first-party agent-to-agent meeting lifecycle. The maintainable end state is a small implementation layered on the settled #63789 handoff contract, with explicit lifecycle verbs, a stable meeting/transcript artifact, visible frames, and tests for state transitions and transcript rendering, while leaving Google Meet, external protocol interop, dashboards, and broad workflow orchestration out of the first slice.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a runtime bug. The current-main verification path is a code/docs search plus inspection of the CLI and session tool implementation, which confirms the requested teleconference surface is absent while adjacent A2A primitives exist.

## Solution Assessment

Unclear until the #63789 handoff contract is settled, but the maintainer-suggested direction is the narrowest plausible path: add a lifecycle layer above the existing `sessions_send`/agent delivery path instead of a separate workflow engine or external meeting integration.
