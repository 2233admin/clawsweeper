## Summary

Keep open. Current main still does not define or dispatch the requested internal `session:end` hook; the existing `session_end` surface is a separate plugin hook, and open PRs #56672 and #61675 reference this issue with closing syntax, so this should stay open until those implementation candidates are reviewed, landed, or explicitly rejected.

## Best Possible Solution

Keep this issue open until maintainers review and either land or close #56672 and #61675, or explicitly decide that the typed plugin `session_end` hook is the supported replacement and document that in the internal hooks docs. A direct implementation should add a typed internal `session:end` contract/guard, dispatch it from the relevant cron, webhook/gateway, reset, and subagent completion paths, and update internal hook tests plus `docs/automation/hooks.md`.
