## Summary

Keep this PR open. The underlying Telegram bug is still present on current main and the production change is pointed at the right Telegram-owned seam, but the proposed patch introduces a concrete test typecheck failure because the new regression calls `sendMessageTelegram` without the required `cfg` option.

## What This Changes

The PR changes Telegram send chunk planning to resolve `channels.telegram.textChunkLimit`, clamp it to Telegram's 4096-character maximum, and adds send-path regression coverage for the 4096 case.

## Best Possible Solution

Amend this PR in place: keep the Telegram send-path resolver/clamp approach, pass a valid `cfg` object into the new regression or otherwise satisfy `TelegramSendOpts`, and preferably add one lower-limit regression before running the targeted Telegram send test and changed gate.
