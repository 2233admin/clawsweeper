## Summary

Keep this PR open. Current main still has direct-first completion dispatch, no persisted per-run delivery claim/user-delivery payload fields, and iMessage outbound sanitization has not gained the OpenClaw internal-context stripping or fence normalization proposed here. The PR is also still an active implementation candidate with related open follow-up context, not something current main has made obsolete.

## Risks / Open Questions

- The PR is security-sensitive because it changes user-visible delivery of internal subagent payloads and iMessage sanitization; it needs maintainer review and targeted regression proof before merge.
- The current PR diff begins a persisted delivery claim before the already-aborted signal return; as a review comment noted, that can leave a claimed run in-flight until the lease expires unless moved under the cleanup/finally path.
- The current PR diff also normalizes iMessage text in sendMessageIMessage after monitor chunking, which can append code-fence closures per chunk and corrupt long fenced responses.
- The PR is broad for this area: 14 files and 720 additions across core agent delivery, registry state, tests, and iMessage delivery. It should be validated against the linked reconnect/retry and iMessage leak scenarios, not merged solely from static reasoning.
