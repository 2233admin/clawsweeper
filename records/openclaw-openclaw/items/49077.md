## Summary

Current `main` still does not expose a configurable delay between consecutive model/API request starts. The model config types, strict schema, provider transport, embedded run, compaction path, docs, and nearby tests cover timeouts, transport policy, retry-after handling, and rate-limit failover, but not proactive request pacing. PR #49221 remains the matching open implementation candidate, so this issue should stay open for maintainer review or an equivalent replacement design.

## Best Possible Solution

Keep this issue open and review/land PR #49221, or replace it with an equivalent core implementation that adds validated global and per-provider model request delay config, applies request-start pacing to embedded agent runs and compaction, documents the public config surface, and adds focused tests for provider-scoped spacing plus zero/override behavior.
