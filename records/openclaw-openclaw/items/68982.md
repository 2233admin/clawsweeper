## Summary

Keep open. The latest PR is still not a complete live reasoning streaming fix: the branch only changes Control UI renderer/item-building files, while current main still emits chat delta/final payloads as text-only messages and collapses live chat events into string state before rendering. The PR also introduces a reasoning visibility regression by falling back from `/reasoning` visibility to `/think` effort.

## What This Changes

This PR changes Control UI chat item building and grouped streaming rendering to accept structured stream message payloads, render reasoning-only assistant content, thread reasoning visibility into streaming groups, and add grouped-render regression coverage.

## Best Possible Solution

Land one consolidated implementation that carries structured assistant reasoning/text payloads from gateway agent events through chat delta/final payloads, controller/app stream state, item building, and streaming rendering, while keeping `/reasoning` visibility independent from `/think` effort and preserving the reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing current main plus this diff: gateway chat events are text-only and handleChatEvent stores only extractText(payload.message) into string chatStream state, so a reasoning-only live delta cannot reach the new object-stream renderer branch.

## Solution Assessment

No. Renderer-side support is not the best fix by itself because the gateway event projection and Control UI stream state still need structured payload plumbing, and visibility must continue to follow `/reasoning` rather than `/think`.
