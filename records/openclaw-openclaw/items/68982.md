## Summary

Keep open. The latest PR branch is still not correct enough to merge: it adds renderer-side structured stream handling, but current main still emits/stores live chat streams as text before the renderer sees them, and the PR also risks hiding the progress indicator and widening reasoning visibility.

## What This Changes

The PR updates Control UI chat item building and grouped streaming rendering to accept optional structured stream message payloads, pass reasoning visibility into streaming groups, and add grouped-render regression coverage.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured thinking/text payloads from agent events through gateway chat delta/final payloads, Control UI state, item building, and streaming rendering while preserving /reasoning visibility semantics and the reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits/stores live chat deltas as text-only payloads, so a reasoning-only live delta cannot reach the PR's new object-stream renderer branch.

## Solution Assessment

No. Renderer-side handling alone is not the best fix because the source live payload is still collapsed before rendering; the safer path is end-to-end structured stream plumbing, likely coordinated with #59086.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured payloads before rendering:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts object payloads, but the real live path still cannot supply one: gateway chat delta/final payloads are text-only, `ChatState.chatStream` and `ChatProps.stream` remain `string | null`, and `handleChatEvent` stores `extractText(payload.message)`. Reasoning-only deltas are still dropped before this branch can run, so live reasoning streaming remains unfixed.
  - confidence: 0.94
- **[P2] Preserve the reading indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:296-301`
  - body: The new object-stream path emits a stream item even when it has no visible text. With reasoning hidden, `renderGroupedMessage` suppresses the thinking block and returns nothing, so the previous reading indicator is replaced by an empty assistant row until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:198`
  - body: `resolveStreamingReasoningLevel` falls back from a missing session reasoning level to `props.thinkingLevel`, but `/think` controls model effort while `/reasoning` controls whether reasoning is shown and falls back to `off`. A session with thinking enabled but no reasoning visibility override can show reasoning unexpectedly.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff has no dependency, workflow, or supply-chain changes, but it can expose reasoning content by treating thinking effort as reasoning visibility.

  - body: `resolveStreamingReasoningLevel` uses `thinkingLevel` when no session reasoning level is set. The documented /reasoning fallback is off, so sessions with thinking effort enabled could display reasoning blocks contrary to the visibility setting.
