## Summary

Keep open, but do not merge this head yet. The PR is working on a real Control UI gap, but the current branch only adds renderer/item support for structured stream objects while current main still emits and stores live chat deltas as text-only strings, so reasoning-only live deltas cannot reach the new path. It also introduces reasoning visibility regressions around the `/reasoning` fallback and hidden-reasoning loading state.

## What This Changes

The PR updates Control UI chat item building and grouped streaming rendering to accept structured stream payloads with thinking/text blocks, passes reasoning visibility into streaming groups, and adds grouped-render regression tests.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured thinking/text payloads from agent events through gateway chat delta/final payloads, Control UI stream state, item building, and grouped rendering while preserving `/reasoning` visibility semantics and the reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits chat delta/final payloads as text-only blocks and `handleChatEvent` stores `extractText(payload.message)` into a string `chatStream`, so a thinking-only live delta is dropped before the PR's object-stream rendering path can run.

## Solution Assessment

No. Renderer-side support alone is not the best fix because the source live payload is still collapsed before rendering; the safer path is an end-to-end structured stream implementation coordinated with #59086.
