## Summary

Keep open. The latest PR is still a renderer-side slice rather than an end-to-end live reasoning stream fix: current main emits and stores live chat deltas as text, while the PR only adds object handling in Control UI item building/rendering. It also introduces visibility regressions for hidden reasoning streams.

## What This Changes

The PR widens Control UI stream item rendering to accept structured assistant message payloads, passes reasoning visibility into streaming bubbles, renders reasoning-only assistant bubbles, and adds grouped-render/buildChatItems regression tests.

## Best Possible Solution

Land one consolidated implementation that carries structured assistant stream payloads from gateway chat events through controller/app state, stream segments, item building, and rendering, while preserving `/reasoning` visibility semantics and the reading indicator for hidden reasoning-only streams.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing a live `assistant` event through `emitChatDelta`, `handleChatEvent`, `state.chatStream`, `buildChatItems`, and `renderStreamingGroup`; the current pipeline collapses the message to text before the PR's structured branch can run.

## Solution Assessment

No. The PR is a useful Control UI rendering slice, but the narrow maintainable fix needs end-to-end structured payload plumbing and visibility-aware rendering rather than only widening renderer inputs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured stream payloads end to end:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: This widens `BuildChatItemsProps.stream` to accept objects, but real live deltas still do not pass objects here: gateway deltas are emitted as text-only assistant messages, `handleChatEvent` stores `extractText(payload.message)` in `chatStream: string | null`, and callers pass that string into `buildChatItems`. Reasoning-only deltas are therefore dropped before reaching this new branch.
  - confidence: 0.94
- **[P2] Keep hidden reasoning-only streams on the indicator path:** `ui/src/ui/chat/build-chat-items.ts:300`
  - body: The new object-stream path emits a stream item for any object even when it has no visible text. With `showReasoning` false, `renderGroupedMessage` returns `nothing`, so the reading indicator is replaced by an empty assistant group until text arrives. Treat structured streams as renderable only when they have visible content, or fall back to the indicator.
  - confidence: 0.88
- **[P2] Do not treat thinking level as reasoning visibility:** `ui/src/ui/views/chat.ts:193`
  - body: `resolveStreamingReasoningLevel` falls back from `reasoningLevel` to `thinkingLevel`, but `/think` controls model effort while `/reasoning` controls whether thinking blocks are displayed and falls back to off. A session with `thinkingLevel: "low"` and no reasoning visibility override can now show reasoning whenever the UI's Show Thinking toggle is enabled.
  - confidence: 0.84
