## Summary

Keep open. The latest patch is still a UI-only rendering slice, while current main emits and stores live chat deltas as text-only strings before this new structured stream branch can run. The diff also introduces reasoning-visibility regressions that need revision before merge, and related broader work remains open in #59086.

## What This Changes

The PR widens Control UI stream item rendering to accept structured assistant payloads, passes reasoning visibility into streaming bubbles, renders reasoning-only assistant bubbles, and adds grouped-render/buildChatItems regression coverage.

## Best Possible Solution

Land one consolidated implementation that carries structured assistant stream payloads from gateway chat events through controller/app state, stream segments, item building, and rendering, while preserving /reasoning visibility semantics and the reading indicator for hidden reasoning-only streams.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing a live assistant event through emitChatDelta, handleChatEvent, state.chatStream, buildChatItems, and renderStreamingGroup; current main collapses the message to text before the PR's object-stream branch can run.

## Solution Assessment

No. The PR is a useful renderer slice, but the maintainable fix needs end-to-end structured payload plumbing plus visibility-aware item building rather than only widening renderer inputs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured stream payloads through chat state:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: This widens `BuildChatItemsProps.stream` to accept objects, but real live deltas still cannot pass objects here: gateway chat deltas are emitted as text-only assistant messages, `handleChatEvent` stores `extractText(payload.message)` in `chatStream: string | null`, and callers pass that string into `buildChatItems`. Reasoning-only deltas are dropped before reaching this new branch.
  - confidence: 0.94
- **[P2] Keep hidden reasoning streams on the indicator path:** `ui/src/ui/chat/build-chat-items.ts:300`
  - body: The new object-stream path emits a stream item for any structured object even when it has no visible text. With `showReasoning` false, `renderGroupedMessage` returns `nothing`, so the reading indicator is replaced by an empty assistant group until text arrives.
  - confidence: 0.88
- **[P2] Do not infer reasoning visibility from thinking effort:** `ui/src/ui/views/chat.ts:194`
  - body: `/think` controls model effort, while `/reasoning` controls whether thinking blocks are displayed and falls back to off. Falling back from absent `reasoningLevel` to `props.thinkingLevel` can show reasoning for sessions with thinking enabled but no reasoning visibility override.
  - confidence: 0.84
