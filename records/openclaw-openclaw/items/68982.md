## Summary

Keep open. Current main still does not implement live reasoning in chat, and the latest PR head is not yet a complete fix: its structured stream renderer path is not reached by the current gateway/controller pipeline, and reasoning-only hidden streams can skip the reading indicator. This needs maintainer review rather than cleanup closure.

## What This Changes

The PR updates Control UI chat item building/rendering to accept structured stream message payloads, pass reasoning visibility into streaming bubbles, render reasoning-only assistant content, and add grouped-render regression tests.

## Best Possible Solution

Land a single end-to-end implementation that carries structured assistant stream payloads from gateway chat events through Control UI state, stream segments, item building, and rendering while preserving reasoning visibility settings and reading indicators for hidden reasoning-only streams.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: follow a live thinking event from `pi-embedded-subscribe` into `server-chat`, then through `handleChatEvent` and `renderStreamingGroup`; the current pipeline only forwards assistant text and stores stream state as a string.

## Solution Assessment

No. The current PR is a useful renderer-side slice, but it is not the narrowest complete fix because it adds object-stream handling without updating the gateway/controller/app state that must supply those objects in real runs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Plumb structured stream payloads through chat state:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: This adds an object-valued stream path, but the live pipeline still narrows every chat delta to text: gateway deltas/finals currently emit text blocks and `handleChatEvent` stores `extractText(payload.message)` into `chatStream: string | null`. Real reasoning-only deltas therefore never reach this new branch, so live reasoning still will not render until the structured assistant message is carried through gateway events, app state/props, and stream segments.
  - confidence: 0.93
- **[P2] Keep hidden reasoning streams on the indicator path:** `ui/src/ui/chat/build-chat-items.ts:298`
  - body: When `props.stream` is an object, this condition emits a stream item even if the object has no visible text. If that payload is reasoning-only and `showReasoning` is false, `renderGroupedMessage` returns no bubble content, so the old reading indicator is skipped and the user sees an empty assistant row until visible text arrives. Treat hidden reasoning-only payloads as non-renderable or keep the reading indicator until visible content exists.
  - confidence: 0.87
