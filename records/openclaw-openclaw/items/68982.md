## Summary

Keep open. The latest PR head is a focused Control UI renderer change, but current main still emits live chat deltas/finals as text-only payloads and the Control UI controller still collapses incoming messages to a string, so the central live reasoning-streaming behavior is not fixed end to end. The patch also widens reasoning visibility by falling back from `/reasoning` to `/think`, which is a correctness and privacy-sensitive regression before merge.

## What This Changes

The PR changes Control UI chat item building and grouped streaming rendering to carry optional structured stream payloads, render reasoning-only assistant content, pass reasoning visibility into streaming groups, and add grouped-render regression tests.

## Best Possible Solution

Land one consolidated end-to-end implementation that preserves structured assistant thinking/text payloads from agent events through gateway chat delta/final payloads, Control UI state, item building, and streaming rendering, while keeping `/reasoning` visibility independent from `/think` effort and preserving the reading indicator when hidden reasoning is the only live content.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits live chat payloads as text-only messages and handleChatEvent stores only extractText(payload.message), so a reasoning-only live delta cannot reach the object-stream renderer branch added by this PR.

## Solution Assessment

No. Renderer-side support alone is not the best fix because the source payload is still collapsed before rendering; the safer path is end-to-end structured payload plumbing, likely coordinated with #59086, while preserving the documented reasoning visibility fallback.

Full review comments:

- **[P1] Plumb structured payloads through live chat state:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts object payloads, but the real live path still cannot supply one: gateway chat delta/final payloads are text-only, `ChatState.chatStream` and `ChatProps.stream` remain `string | null`, and `handleChatEvent` stores `extractText(payload.message)`. Reasoning-only deltas are still dropped before this renderer branch can run, so the advertised live reasoning stream remains unfixed.
  - confidence: 0.94
- **[P2] Preserve the progress indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:299-300`
  - body: Any object stream now becomes a stream item even when it has no visible text. With `showReasoning` false, `renderGroupedMessage` suppresses the thinking block and returns nothing, replacing the previous reading indicator with an empty assistant group until answer text arrives.
  - confidence: 0.88
- **[P2] Keep /think effort separate from reasoning visibility:** `ui/src/ui/views/chat.ts:197-199`
  - body: `resolveStreamingReasoningLevel` falls back from a missing session reasoning level to `props.thinkingLevel`, but `/think` controls model effort while `/reasoning` controls whether reasoning is shown and falls back to `off`. A session with thinking enabled but no reasoning visibility override can show reasoning unexpectedly.
  - confidence: 0.9
