## Summary

Keep open. The latest PR diff is still a Control UI-only renderer/item-builder change, while current main emits live chat delta/final payloads as text-only assistant messages and stores live stream state as extracted text. That means reasoning-only or structured thinking/text payloads still cannot reach the new renderer path in real live chat, and the PR also widens reasoning visibility by falling back from `/reasoning` state to `/think` effort.

## What This Changes

The PR branch updates Control UI chat item building and grouped streaming rendering to accept structured stream payloads with thinking/text blocks, passes reasoning visibility into streaming groups, and adds grouped-render regression tests.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured thinking/text payloads from agent events through gateway chat delta/final payloads, Control UI stream state, item building, and grouped rendering while preserving `/reasoning` visibility semantics and the reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits and stores live chat deltas as text-only strings, so a reasoning-only live delta cannot reach the PR's object-stream rendering path.

## Solution Assessment

No. Renderer-side support alone is not the best fix because the source live payload is still collapsed before rendering; the safer path is an end-to-end structured stream implementation coordinated with #59086.
