## Summary

Keep open. Current main does not implement the proposed manifest permission model, and the PR is security-sensitive, currently unmergeable, and still has concrete audit correctness bugs around manifest path resolution and enabled-plugin scoping.

## What This Changes

The PR adds an optional `capabilities` field and normalizers to plugin manifests, adds a legacy-unrestricted plugin security audit warning, and introduces an RFC for phased capability and budget enforcement.

## Best Possible Solution

Rebase the work onto current main, have maintainers settle the manifest capability schema and phased security semantics, then implement the legacy-mode audit in `src/security/audit-plugins-trust.ts` using installed plugin root paths and the enabled plugin set, with manifest docs and focused audit/manifest tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: install a plugin under `stateDir/extensions/some-plugin/openclaw.plugin.json` while running from another cwd, and the PR's `loadPluginManifest('some-plugin')` lookup misses it; disabling the plugin also shows the raw `pluginDirs` loop can warn on inactive plugins.

## Solution Assessment

No. The declaration-only direction may be reasonable, but this PR is not the best current implementation until it is rebased onto the current audit split, scopes findings to enabled plugins, uses installed extension paths, and gets maintainer review for the new security contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Join plugin dirs before loading manifests:** `src/security/audit-extra.async.ts:833`
  - body: `pluginDirs` are entry names from the extensions directory, while `loadPluginManifest` resolves `<rootDir>/openclaw.plugin.json`. Passing the bare name makes the audit look under the process cwd instead of `stateDir/extensions/<plugin>`, so installed legacy plugins are usually skipped and the new security finding does not fire.
  - confidence: 0.94
- **[P2] Scope legacy capability warnings to enabled plugins:** `src/security/audit-extra.async.ts:831`
  - body: This loop scans every discovered plugin directory, including plugins disabled globally, denied, or marked `enabled: false`. The finding says those plugins run without declared capabilities, which creates false-positive security warnings for plugins that are not loaded; reuse the enabled plugin set or relabel this as installed inventory.
  - confidence: 0.88
