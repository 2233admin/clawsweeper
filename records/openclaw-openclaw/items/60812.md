## Summary

Keep this issue open. Current main still has the Telegram delivery fallback anchor, but Discord, Slack, Signal, and WhatsApp do not implement the requested user-visible fallback for reply-delivery errors or no-visible-output cases. The related same-author PR is still described as open in the provided context and does not cover this broader channel parity work.

## Best Possible Solution

Keep the issue open and implement a shared delivery-fallback helper or plugin-SDK reply-runtime utility. Discord, Slack, Signal, WhatsApp, and eventually LINE should track delivery errors/skips, inspect delivered/queued/failed counts after dispatch settles, choose typed user-facing messages through the existing failover classification path, and add focused channel tests for provider failure, empty response, delivery throw, and no-visible-output cases.
