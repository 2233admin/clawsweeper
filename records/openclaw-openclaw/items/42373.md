## Summary

Current main still does not implement `costCurrency` or an equivalent currency metadata/display contract. Pricing config, docs, runtime usage structs, session fields, chat/CLI/Control UI/macOS cost displays, and tests remain USD/dollar-oriented, so the requested feature remains actionable core pricing/config work.

## Best Possible Solution

Keep this open for a real core pricing-currency contract. Maintainers should choose the boundary first, likely global, per-provider, per-model, or explicit USD storage plus display conversion, then update config schema/types/generated docs, runtime pricing lookup, transcript/session/diagnostic fields, CLI/chat/status output, Control UI/macOS cost displays, tests, and mixed-currency aggregation behavior.
