## Summary

Keep this issue open. Current main still exposes `session_status` as a rendered `/status`-style card with minimal details, while raw structured token/cost usage remains on separate gateway usage RPCs rather than an agent-facing `session_status` usage contract. Related #39400 is adjacent but narrower; it does not supersede this broader session/last-turn usage API request.

## Best Possible Solution

Keep the issue open until OpenClaw adds a documented structured agent-facing usage surface. The likely path is to extend `session_status.details` with stable `usage.session`, `usage.lastTurn`, cost, budget/window availability, source/unavailable metadata, and tests, or add an equivalent supported tool that reuses `sessions.usage` and transcript usage parsers while preserving current visibility checks.
