## Summary

Keep this PR open. Current main has implemented the central continuation behavior under the public `continueAgent` field, but the PR is not fully superseded because the current handler does not preserve the PR's nullish-result guard and the public command JSDoc still describes plugin commands as bypassing the agent. The PR should not merge as-is because its API name is now stale (`shouldContinue` vs `continueAgent`), but it still contains review-addressed work worth porting or explicitly rejecting.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a rebase onto current main that ports only the still-useful parts: guard nullish plugin command results, update the `registerCommand` JSDoc, and align all examples/tests with the current `continueAgent` API instead of adding a second `shouldContinue` field.
