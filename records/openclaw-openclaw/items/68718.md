## Summary

Keep open. Current main does not implement the durable minions queue, CLI, protocol, or SDK entrypoint from this PR; it only contains the narrower detached-task runtime seam and plugin registration contract that maintainers asked future work to build on. The discussion also explicitly says the core contract landed in #68915 but that the durable queue/retry/stall engine did not, so this PR needs maintainer-guided rework rather than conservative cleanup closure.

## What This Changes

The PR branch adds a SQLite-backed minions job queue subsystem with worker lifecycle, gateway startup wiring, task shadow-sync, protocol schemas, CLI/docs, plugin SDK exports, and a separate pnpm workspace symlink change in the plugin install security scanner.

## Best Possible Solution

Keep Tasks and TaskFlow as the canonical core state model, then rebuild any durable executor as a smaller follow-up against the detached-runtime registration contract from #68886 and #68915. The install security-scan change should be split into its own focused fix if still needed.
