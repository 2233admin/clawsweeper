## Summary

Keep open. Current main has meaningful partial mitigation: strict-agentic GPT-5/OpenAI-Codex runs can retry plan-only replies, and reminder-style commitments get an explicit “I did not schedule a reminder” note when no cron add or existing session cron backs them. The provided discussion also has a positive 2026.4.23 reporter datapoint and related reports #50341/#57263 were closed. But the specific issue is broader than those mitigations: the original reproduction involved one `memory_search` followed by “I need to read ... and then I’ll come back,” and current code does not clearly guard that exact unbacked follow-up promise. The reminder guard patterns do not include “come back,” and the plan-only retry path deliberately skips lone unclassified tools; `memory_search` is not in the single-action retry-safe tool set.

## Best Possible Solution

Keep the issue open until the runtime guard or strict-agentic retry path explicitly covers generic unbacked follow-up commitments. A good fix would add tests for the original `memory_search` plus “come back/send a follow-up” transcript, treat safe read-only recall tools such as `memory_search` as eligible for single-action continuation when no side effect occurred, and/or add a post-processing guard that checks promised follow-up language against real mechanisms like cron, `sessions_spawn`, queued follow-up, or background process state.
