## Summary

Keep open. Current `main` has not absorbed this PR’s intended Windows status fix: `readScheduledTaskRuntime()` still performs the broad `schtasks /Query` availability preflight before the task-scoped runtime query, and the current tests do not include the PR’s single-call or thrown-`execSchtasks` fallback coverage.

## Best Possible Solution

Keep this PR open for maintainer review or port the narrow fix onto current `main`: remove the broad availability preflight from `readScheduledTaskRuntime()`, query the task-scoped `schtasks /Query /TN ... /V /FO LIST` directly, preserve Startup-folder fallback and unknown-status behavior when that query fails or throws, and land the focused regression tests from the PR. If #49187 was closed administratively, this PR can still be the implementation vehicle for the remaining code change.
