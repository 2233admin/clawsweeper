## Summary

Keep open. Current main preserves Feishu slash-command text and mention-stripped command probing, but the shared command authorization path still denies Feishu group commands when commands.useAccessGroups is true and no command allowlist is configured. Open PR #49958 explicitly targets this issue, so maintainers should review and land or close that implementation candidate instead of auto-closing the issue now.

## Best Possible Solution

Keep this issue open while maintainers review PR #49958 or an equivalent focused fix. The likely implementation belongs in src/channels/command-gating.ts with regression coverage for Feishu group slash commands when no allowFrom is configured, while preserving fail-closed behavior when an allowlist is configured and the sender is not allowed.
