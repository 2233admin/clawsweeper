## Summary

Keep this issue open. Current main still runs the TTS transform on normal reply and ACP delivery paths, but isolated cron announce delivery still resolves final text into plain payloads and sends them through outbound delivery without calling `maybeApplyTtsToPayload`. The prior maintainer/bot comments remain accurate; #33082 is related but separate because it covered tool-produced `MEDIA:` voice payloads, not model-emitted `[[tts]]` tags in cron announce text.

## Best Possible Solution

Implement a cron-owned pre-delivery TTS transform for isolated `delivery.mode: "announce"` before `deliverOutboundPayloads`, reusing the existing `maybeApplyTtsToPayload` helper with `cfgWithAgentDefaults`, resolved delivery channel/account, agent id, final reply kind, inbound-audio false, and preserved session `ttsAuto`. Add a focused regression test proving cron announce text containing `[[tts]]` or `[[tts:text]]` is stripped from visible text and delivered as synthesized voice/media. Keep #33082 treated as related but separate.
