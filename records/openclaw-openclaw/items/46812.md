## Summary

Keep open. Current main still does not implement a pinned facts editor, structured pinned-facts storage, or an every-turn context-injection guarantee; the existing surfaces are whole-file workspace Markdown editing, browser-local pinned chat message indexes, and a narrow plugin UI descriptor projection. The active #71736 discussion and 1fanwang’s manifest-first panel prototype are useful related context, but neither is a merged complete fix for this issue.

## What This Changes

The issue requests a Control UI panel for user-defined persistent facts with inline editing and guaranteed injection into agent context on every turn.

## Best Possible Solution

Keep this open until maintainers either split or approve a design that includes both pieces: a safe manifest-first Control UI contribution/panel contract and a pinned facts editor backed by a documented structured store or USER.md section, with tests proving how those facts enter normal agent context under the intended budget and mode rules.
