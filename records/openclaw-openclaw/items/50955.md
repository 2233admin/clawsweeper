## Summary

Keep open. Current main only partially overlaps this PR and does not definitely implement its intended ACP routing change: native `sessions_spawn(runtime:"acp", thread:true)` still uses channel default placement, so Discord requests with an existing thread context can still bind with `placement: "child"` rather than matching `/acp spawn --thread here`. The PR also has unresolved review findings around Feishu action regressions, bootstrap routing, announce-session classification, and current-binding ownership.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up rather than closing it as implemented. The best path is to rebase or replace it with a narrower current-main patch that makes native ACP placement match `/acp spawn` for Discord threads/DMs and Telegram topics/DMs, preserves Feishu action handlers, mirrors the current-binding `boundBy` ownership gate, and adds focused regression tests for bootstrap delivery, bound completion routing, and threaded `sessions_send` announce routing. If maintainers prefer cleanup, close it manually only after a fresh canonical PR carries the remaining work.
