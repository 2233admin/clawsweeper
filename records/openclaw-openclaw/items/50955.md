## Summary

Keep this PR open. Current main has picked up adjacent work, especially plugin-owned session-key parsing and threadId propagation for sessions_send announce delivery, plus native Telegram current-topic/DM ACP coverage. But the PR’s central Discord/native ACP placement and binding-aware reverse-delivery work is still not implemented on main, and the PR still carries review blockers around Feishu action handling and current-binding ownership.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best path is a fresh rebase or narrower replacement that shares one origin-aware placement resolver between /acp spawn and native sessions_spawn, preserves Feishu action handlers, enforces the current-binding boundBy ownership gate, adds binding-aware reverse delivery for bootstrap/completion routing, and keeps the current sessions_send thread propagation work. Open PR #64322 is complementary for Discord per-thread session keys, not a superseding fix for this broader placement and delivery-routing scope.
