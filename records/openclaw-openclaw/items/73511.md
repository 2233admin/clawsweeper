## Summary

Keep open. Current main still returns `URL.origin` directly during Gateway browser-origin normalization, so custom-scheme origins such as `tauri://localhost` still normalize to the opaque string `"null"` and cannot match an explicit `gateway.controlUi.allowedOrigins` entry. This PR remains a focused implementation candidate for the linked open bug #46520.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #46520. The best path is maintainer review of the narrow origin-normalization change, CI on the added regression cases, and merge if the authorization semantics remain constrained to explicit allowlist matches.
