## Summary

Keep open. Current main still normalizes custom-scheme browser origins through URL.origin, which becomes "null" for tauri:// and electron:// origins, so explicit gateway.controlUi.allowedOrigins entries cannot match. This PR is a focused implementation candidate for the linked open bug #46520 and should continue through maintainer review and CI.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #46520. The best path is maintainer review of the narrow src/gateway/origin-check.ts normalization change, CI on the added regression cases, and merge if the explicit-allowlist-only authorization semantics hold.
