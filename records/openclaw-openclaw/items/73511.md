## Summary

Keep open. Current main still normalizes browser Origin through URL.origin, which becomes the opaque string "null" for custom-scheme origins such as tauri://localhost, so explicit gateway.controlUi.allowedOrigins entries still cannot match. The linked bug #46520 remains open, and this PR is a narrow implementation candidate rather than work that current main has already shipped.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #46520. The best path is maintainer review of the narrow src/gateway/origin-check.ts normalization change, CI on the added src/gateway/origin-check.test.ts regression cases, and merge if the authorization semantics remain constrained to explicit allowlist matches.
