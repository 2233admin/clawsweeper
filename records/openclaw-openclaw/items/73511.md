## Summary

Keep open. The checked-out main revision still normalizes browser Origin through `URL.origin`, which becomes the opaque string `"null"` for custom-scheme origins like `tauri://localhost`; explicit `gateway.controlUi.allowedOrigins` entries therefore still cannot match. This PR remains a focused implementation candidate for linked open bug #46520.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #46520. The best path is maintainer review of the narrow `src/gateway/origin-check.ts` normalization change, CI on the added `src/gateway/origin-check.test.ts` regression cases, and merge if the authorization semantics remain constrained to explicit allowlist matches.
