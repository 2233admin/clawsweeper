## Summary

Keep open. Current main still does not implement this PR's intended custom-scheme Control UI origin behavior: the gateway normalizes the browser Origin through URL.origin, which is "null" for non-special schemes such as tauri:// and electron://, so an explicit gateway.controlUi.allowedOrigins entry cannot match. The PR is a focused implementation candidate for the linked open bug and touches the right source, test, and changelog surfaces.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the linked custom-scheme Control UI origin bug. The best path is maintainer review plus CI on the PR head, then merge a narrow fix in src/gateway/origin-check.ts with the added regression cases in src/gateway/origin-check.test.ts and the changelog credit.
