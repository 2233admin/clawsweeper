## Summary

Keep this PR open. Current main still normalizes browser Origin through `URL.origin`, which stays `"null"` for custom schemes such as `tauri://` and `electron://`, so explicit `gateway.controlUi.allowedOrigins` entries for those desktop-wrapper origins still cannot match. The PR remains a focused implementation candidate for the linked open bug.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the linked custom-scheme Control UI origin bug. The best path is maintainer review plus CI on the PR head, then merge a narrow `src/gateway/origin-check.ts` fix with the added regression cases in `src/gateway/origin-check.test.ts` and the changelog credit.
