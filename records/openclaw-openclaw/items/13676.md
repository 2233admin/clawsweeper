Current main still has adjacent sessions, agents, workspace paths, skill allowlists, sandbox/tool policy, and subagent primitives, but it does not implement the central first-class Project model, Projects dashboard surface, or Gateway projects protocol/API. ## What This Changes

Request to add first-class Projects to the Gateway/Control UI dashboard, including project/session/workspace bindings, scoped skills/tools, runtime constraints, and fast project switching.

Keep this as the canonical core/product request until OpenClaw has an explicit Project domain model, additive Gateway projects.* protocol methods, a Control UI Projects list/detail/switching surface, project-to-home-session and workstream bindings, workspace/doc links, and clearly labeled project-scoped constraint display or enforcement built on the existing agents, sessions, workspaces, skills, sandbox, and subagent primitives.
