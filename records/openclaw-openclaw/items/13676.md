## Summary

Keep open. Current main still has adjacent agents, sessions, workspaces, skill allowlists, and sandbox/tool policy, but it does not have a first-class Project object, Projects dashboard surface, or Gateway `projects.*` protocol/API. The recent maintainer/Codex comment in the issue also explicitly keeps this as the canonical core/product dashboard request rather than treating the narrower closed related issues as replacements.

## What This Changes

Add first-class Projects to the Gateway Dashboard, including durable project storage, project-to-session and project-to-workspace bindings, and project-scoped tool/skill/runtime constraints.

## Best Possible Solution

Keep this open as the canonical core/product dashboard request. The best path is an explicit Project storage model plus additive Gateway `projects.*` protocol methods, a Control UI Projects list/detail/switching surface, project-to-home-session and workstream bindings, one-or-more workspace/doc links, and project-scoped tool/skill/runtime constraint display or warn-only enforcement while reusing existing agents, sessions, workspaces, skill allowlists, sandbox settings, and sub-agent metadata where appropriate.
