## Summary

Keep open. Current `origin/main` still lacks the requested first-class Project model, Projects dashboard route, and Gateway project API; the existing sessions, agents, workspaces, skills, and sandbox/tool policies are adjacent building blocks but do not satisfy the core product request. The issue discussion also already treats this as the canonical broader request rather than a duplicate of the narrower closed project-session reports.

## What This Changes

The issue requests a first-class Project concept in the Gateway dashboard with project/session/workspace bindings plus project-scoped skills, tools, and runtime constraints.

## Best Possible Solution

Keep this as the canonical core/product request until OpenClaw has an explicit Project domain model, additive Gateway project protocol, Control UI project list/detail/switching surface, project-to-home-session and workstream bindings, workspace/doc links, and project-scoped tool/skill/runtime constraint display or warning behavior that reuses existing agents, sessions, workspaces, skill allowlists, sandbox settings, and subagent metadata where appropriate.
