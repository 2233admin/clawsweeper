## Summary

Keep this issue open. Current main still lacks the requested first-class Project object, Projects dashboard surface, projects.* Gateway protocol, project-to-session/home-session binding, multi-workspace project model, and project-scoped tool/skill/runtime policy. Existing sessions, agents, per-agent workspaces, skill allowlists, sandbox settings, and sub-agent sessions are adjacent primitives, but they do not satisfy the durable Project model requested here. The discussion also adds a useful project-scoped port/service registry idea and the recent maintainer/Codex comment explicitly kept this open rather than treating #12333 or #13535 as replacements.

## Best Possible Solution

Continue tracking this as the canonical core/product dashboard request. A complete solution should design explicit Project storage and Gateway protocol methods, expose a Projects list/route in Control UI, bind projects to a home session plus optional workstreams, associate one or more workspace paths and docs links, and surface project-scoped tool/skill/runtime constraints while reusing existing agents, sessions, workspaces, skill allowlists, tool policies, sandbox settings, and sub-agent primitives where they fit. The port/service registry idea from the discussion should be considered during project-detail design or tracked as a focused follow-up.
