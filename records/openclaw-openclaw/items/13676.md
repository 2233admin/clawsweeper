## Summary

Keep open. Current main still has adjacent agents, sessions, workspaces, skill allowlists, and sandbox/tool policy, but it does not have a first-class Project object, Projects dashboard surface, or Gateway `projects.*` protocol/API. The issue discussion also includes a recent maintainer/Codex comment keeping this as the canonical core/product dashboard request rather than treating narrower related issues as replacements.

## What This Changes

Add a first-class Project model in the Gateway Dashboard with project lists, session and workspace bindings, scoped skills/tools, and runtime constraint metadata.

## Best Possible Solution

Keep this open as the canonical core/product dashboard request until OpenClaw has an explicit Project storage model, additive Gateway `projects.*` protocol methods, a Control UI Projects list/detail/switching surface, project-to-home-session and workstream bindings, workspace/doc links, and project-scoped tool/skill/runtime constraint display or warn-only enforcement that reuses existing agents, sessions, workspaces, skill allowlists, sandbox settings, and sub-agent metadata where appropriate.
