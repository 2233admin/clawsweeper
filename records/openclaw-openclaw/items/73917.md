## Summary

Keep open. Current main still has the `bootstrap-extra-files` basename allowlist, so the feature is not implemented, and the proposed patch has two P2 behavior regressions: bracket-only globs are skipped and explicit configured paths are reordered before prompt-budget application. The branch also overlaps active #74735, so it needs maintainer coordination before merge.

## What This Changes

The PR removes the fixed basename allowlist from `bootstrap-extra-files`, widens bootstrap file names to strings, updates docs/changelog copy, and adds tests for arbitrary workspace-contained extra files.

## Best Possible Solution

Land one coherent `bootstrap-extra-files` update that removes or scopes the basename allowlist according to the chosen product contract while preserving workspace containment, realpath checks, per-file and total prompt budgets, configured literal path order, deterministic per-glob ordering, subagent filtering, docs, and changelog coverage.

## Reproduction Assessment

Yes. Current main is reproducible by configuring `bootstrap-extra-files` with an arbitrary basename such as `docs/PROJECT.md`, which source inspection shows is rejected as `invalid-bootstrap-filename`; the PR regressions are reproducible by the read-only Node glob/order probes described in evidence.

## Solution Assessment

No, not as currently written. Removing the allowlist is a plausible narrow fix for #39120, but the patch should treat `[` as glob syntax and preserve configured literal order, or maintainers should choose the #74735 session-scoped design instead.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Treat bracket-only patterns as globs:** `src/agents/workspace.ts:186-188`
  - body: `hasGlobPattern()` does not check for `[`, so a valid Node glob such as `docs/[ab].md` is treated as a literal path unless it also contains `*`, `?`, or `{`. That makes bracket-only configured patterns silently report missing instead of expanding while this PR is broadening extra file loading.
  - confidence: 0.92
- **[P2] Preserve configured path order:** `src/agents/workspace.ts:714`
  - body: Sorting all resolved paths reorders explicit `paths` entries before they are loaded. `buildBootstrapContextFiles()` consumes files in order and spends the total bootstrap budget as it goes, so a user-provided priority order like `notes/z.md` before `notes/a.md` can be reversed and change what context survives truncation; keep config order and sort only each glob expansion.
  - confidence: 0.88
