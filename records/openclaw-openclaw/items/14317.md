## Summary

Keep open. Current main still lacks Slack `groupPolicy: "private"` across the shared config contract, Slack schema/docs/metadata, normal message routing, slash-command routing, and tests. The existing maintainer review comment that kept this open is still accurate after rechecking the current checkout.

## What This Changes

Add a Slack `groupPolicy: "private"` option that allows invited private Slack channels while ignoring public channels unless explicitly allowlisted.

## Best Possible Solution

If maintainers approve the trust boundary, implement a Slack-scoped private group policy through config types/schema/metadata, normal and slash-command routing, security audit guidance, docs, and focused tests while preserving the current fail-closed allowlist default and documenting Slack Connect caveats.
