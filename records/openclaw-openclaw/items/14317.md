## Summary

Keep open. Current main still lacks Slack `groupPolicy: "private"`: the shared config contract rejects it, Slack message and slash-command routing only evaluate the existing policy modes, and docs/metadata expose only `open`, `disabled`, and `allowlist`. The request is concrete, but it changes Slack access-control and audit semantics, so it needs maintainer/security approval before implementation.

## What This Changes

Add a Slack-only `groupPolicy: "private"` mode that allows invited private channels, blocks public channels by default, and still permits explicit public-channel allowlist entries.

## Best Possible Solution

If maintainers approve the trust boundary, implement a Slack-scoped private policy end to end: config type/schema/metadata, normal and slash-command routing using Slack channel privacy metadata, explicit public-channel allowlist fallback, security-audit classification, docs with Slack Connect caveats, and focused tests.
