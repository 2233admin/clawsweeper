## Summary

Keep open. Current main still does not implement Slack `groupPolicy: "private"`: the shared type/schema, Slack config metadata/docs, normal message routing, slash-command routing, and tests still only model `open`, `allowlist`, and `disabled`. Slack channel discovery does capture private-channel metadata for allowlist resolution, but the access-control paths do not consume it.

## What This Changes

Add a Slack-scoped `groupPolicy: "private"` option that allows invited private channels, still permits explicitly allowlisted public channels, and updates security audit guidance, docs, and tests.

## Best Possible Solution

Keep this issue open for maintainer/security product review. If approved, implement a Slack-scoped `groupPolicy: "private"` through the Slack plugin config type/schema/metadata, normal and slash-command routing, security audit guidance, docs, and tests; preserve the current fail-closed `allowlist` default and document Slack Connect caveats.
