## Summary

Keep this issue open. Current main at b85edb3f0cf6c2a8bec06537af9211a044266a6c still does not implement Slack `groupPolicy: "private"`: the shared policy type/schema, Slack config schema, generated metadata, runtime message routing, slash routing, docs, and tests still model only `open`, `allowlist`, and `disabled`. The earlier default-open motivation is obsolete because Slack now defaults/falls back to `allowlist`, but the requested private-channel trust boundary remains missing.

## Best Possible Solution

Keep this issue open and implement a Slack-scoped `groupPolicy: "private"` option in the bundled Slack plugin after maintainer confirmation of the trust boundary. The fix should accept the config value, evaluate Slack `is_private` for message and slash routing, allow private channels where the bot is present, preserve explicit public-channel allowlist and explicit deny behavior, update security audit/fix guidance so `private` is not treated like `open`, and document/test Slack Connect caveats. Do not change the current Slack default from `allowlist` as part of this issue without maintainer product approval.
