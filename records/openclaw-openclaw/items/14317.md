## Summary

Keep open. Current main still does not accept or implement Slack `groupPolicy: "private"`: the shared config contract and Slack schema/docs/metadata omit the value, normal and slash-command routing still evaluate only open/disabled/allowlist allowlist state, and the exposure audit only classifies the current `open` case. The request is concrete, but it changes Slack access-control semantics and should wait for maintainer/security approval of the private-channel trust boundary.

## What This Changes

Add a Slack-only `groupPolicy: "private"` mode that allows invited private channels, keeps public channels blocked unless explicitly allowlisted, and updates security guidance accordingly.

## Best Possible Solution

After maintainer/security approval, implement a Slack-scoped private group policy end to end: config type/schema/metadata support, normal and slash-command routing that carries Slack privacy metadata, public-channel allowlist fallback, exposure-audit classification, public Slack docs with Slack Connect caveats, and focused regression tests while preserving the current fail-closed default.
