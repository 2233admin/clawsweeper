## Summary

Keep open. Current main still only appends token/cache/cost/session data to the per-response `/usage` footer, while `/status` and the GitHub Copilot plugin already have the generic Premium/Chat provider-usage window plumbing. The direct implementation PR #43098 is closed unmerged, and open #69618 overlaps footer work but does not claim this quota-window request.

## What This Changes

Add GitHub Copilot Premium/Chat provider quota percentages to the per-response `/usage` footer when available, similar to `/status`.

## Best Possible Solution

Implement a generic reply-footer enhancement that, when `/usage tokens` or `/usage full` is enabled and the active provider has usage windows, appends a compact `formatUsageWindowSummary` result to the existing footer while keeping provider-specific fetching inside plugins. Update token-use docs and changelog, and add regression coverage for the reply-runner/footer path.
