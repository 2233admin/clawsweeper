## Summary

Keep open. Current main still keeps provider quota display separate from the per-response `/usage` footer: `/status` loads and formats provider usage windows, and the GitHub Copilot plugin supplies Premium/Chat windows, but the reply footer only formats turn token/cache/cost data plus the full-mode session key. The related PR #43098 is closed unmerged, so it is useful prior art but not an implementation on main.

## Best Possible Solution

Keep this issue open. The best path is a focused reply-footer change that reuses the active provider's cached or timeout-bounded provider usage summary for GitHub Copilot OAuth sessions, appends formatted Premium/Chat quota when available, degrades silently on auth/fetch errors, and adds a regression test in `src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts`.

## Risks / Open Questions

- A naive implementation could add a provider-usage network call to every reply and delay final response delivery; it should reuse cached data or a short timeout.
- The implementation needs to preserve `/usage tokens` versus `/usage full` semantics and the current OAuth/API-key cost hiding rules.
- PR #43098 is useful prior art, but it is closed unmerged and would need current-main review/rebase before reuse.
