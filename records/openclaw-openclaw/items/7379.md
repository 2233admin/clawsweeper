## Summary

Keep open. Current main still does not bridge provider quota windows into the per-response `/usage` footer: the reply path formats token/cache/cost/session data only, while `/status` already loads the active provider usage summary and the GitHub Copilot plugin already exposes the requested Premium/Chat windows. The direct fix PR #43098 is closed unmerged, and open PR #69618 overlaps footer plumbing but does not specifically claim this provider-quota footer behavior.

## What This Changes

The issue requests adding GitHub Copilot Premium/Chat provider quota percentages to the per-response `/usage` footer.

## Best Possible Solution

Implement a generic reply-footer enhancement that appends a compact `formatUsageWindowSummary` result for the active provider's usage windows when `/usage tokens` or `/usage full` is enabled, while keeping provider-specific fetch/auth behavior inside plugins. Add reply-runner regression coverage, update `docs/reference/token-use.md`, and include a user-facing changelog entry.
