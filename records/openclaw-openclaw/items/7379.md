## Summary

Keep this issue open. Current main still builds the per-response `/usage` footer from turn token/cache/cost data only, while `/status` already has generic provider-usage loading and the GitHub Copilot plugin already exposes Premium/Chat windows. The linked implementation PR #43098 is closed unmerged, so there is no merged fix to close against.

## What This Changes

Requested adding GitHub Copilot Premium/Chat quota percentages to the per-response `/usage` footer when `/usage tokens` or `/usage full` is enabled.

## Best Possible Solution

Implement a focused generic reply-footer enhancement: when `/usage tokens` or `/usage full` is enabled and the active provider has provider usage windows, append a compact `formatUsageWindowSummary` result to the existing footer without blocking final reply delivery. Keep provider-specific fetching inside plugins, update `docs/reference/token-use.md` and changelog as needed, and add regression coverage for the footer path.
