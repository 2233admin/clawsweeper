## Summary

Keep this issue open. Current main still formats the per-response `/usage` footer from turn token/cache/cost data only; provider quota windows are available through the status/provider-usage path and the GitHub Copilot plugin, but they are not wired into the reply footer. PR #43098 directly attempted the requested change, but it is closed unmerged and no open PR currently references this issue.

## Best Possible Solution

Implement a focused reply-footer enhancement: when `/usage tokens` or `/usage full` is enabled and the active provider has generic provider usage windows, append a compact formatted quota summary such as `Premium 86% left Â· Chat 100% left` to the existing footer. Keep provider-specific fetching in plugins, avoid blocking final replies on usage failures, update the token-use docs/changelog, and add regression coverage for the footer path.
