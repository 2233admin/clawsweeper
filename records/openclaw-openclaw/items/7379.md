## Summary

Keep open. Current main still keeps provider quota display separate from the per-response `/usage` footer: `/status` loads and formats provider usage windows, and the GitHub Copilot plugin supplies Premium/Chat windows, but the reply footer only formats turn token/cache/cost data plus the full-mode session key. The related PR #43098 is closed unmerged, so it is useful prior art but not an implementation on main.

## Best Possible Solution

Keep this issue open and implement a focused reply-footer enhancement: when the active model provider has a provider usage window and the session has `/usage tokens` or `/usage full` enabled, append a formatted Premium/Chat quota summary to the existing footer for GitHub Copilot OAuth sessions. The change should stay generic at the provider-usage-summary boundary, keep Copilot-specific fetching in the plugin, avoid blocking final replies on quota fetch failures, and add a regression test for the per-response footer.

## Risks / Open Questions

- A naive implementation could add a provider-usage network call to every reply and delay final response delivery; it should use cached data when available or a short timeout and silently omit quota on failure.
- The implementation needs to preserve `/usage tokens` versus `/usage full` semantics and the current OAuth/API-key cost hiding rules.
- The existing `loadProviderUsageSummary` helper does not appear to cache by itself, so reusing it directly in the hot reply path needs care.
- PR #43098 is useful prior art, but it is closed unmerged and would need current-main review/rebase before reuse.
