## Summary

Current main still has the reported shape: gateway startup can pass a scoped `onlyPluginIds` set into plugin loading, but discovery, metadata snapshot creation, and manifest-registry materialization still read/validate the bundled plugin candidate set before the final import/registration scope is applied. The earlier ClawSweeper keep-open comment remains consistent with current `6186ed2c`.

## Best Possible Solution

Keep this issue open and implement scoped cold discovery/manifest materialization: derive a conservative startup/plugin scope before bundled manifest registry creation, pass that scope into discovery and installed-index manifest loading, skip out-of-scope bundled directories before `package.json`/`openclaw.plugin.json`/schema validation where safe, and add regression tests proving skipped bundled plugin manifests are not read while existing explicit channel/provider/slot activation semantics still work.
