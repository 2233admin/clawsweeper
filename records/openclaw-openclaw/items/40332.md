## Summary

Keep open. Current main still supports ACPX `permissionMode` and `nonInteractivePermissions` only as global plugin config; the typed agent runtime config, strict binding schema, configured binding materialization, persistent binding metadata, runtime ensure contract, and docs still omit scoped permission policy. This remains valid but security-sensitive, so it needs maintainer design before an automated fix lane.

## What This Changes

Request ACPX `permissionMode` and related non-interactive permission overrides at `bindings[].acp` and `agents.list[].runtime.acp`, with binding > agent runtime > global plugin precedence.

## Best Possible Solution

Add a typed scoped ACP permission contract on agent runtime and binding config, persist and rehydrate it for configured bindings, pass the effective policy into ACP runtime startup, resolve precedence as binding > agent runtime > global ACPX plugin config, and update schema/help/docs plus regression tests for both `permissionMode` and `nonInteractivePermissions`.
