## Summary

Current main still has the reported sanitizer gap: final tags are stripped as plain text, duplicate collapse only handles adjacent blank-line blocks, and outbound replies still flow through that sanitizer. The closed same-line duplicate PRs were not merged, and the open adjacent PR targets code-block indentation rather than final-tag or same-line duplicate removal.

## What This Changes

Improve `sanitizeUserFacingText()` so same-line, single-newline, partial, or final-tag-boundary duplicate assistant output is detected without corrupting intentional formatting.

## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued `</final>` repeat and any deliberately supported adjacent single-newline duplicate form, with regression tests that preserve ordinary repeated wording, normal single-newline formatting, trailing whitespace, and code-block indentation.
