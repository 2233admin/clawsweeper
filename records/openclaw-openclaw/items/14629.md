## Summary

Current main still has the reported sanitizer gap: `<final>` tags are stripped textually, duplicate collapse only handles exact adjacent blank-line-separated blocks, and outbound replies still use this sanitizer. Prior same-line duplicate fix PRs #16813 and #19113 are closed unmerged, while open PR #41851 is adjacent indentation work rather than a fix for this issue.

## What This Changes

Improve `sanitizeUserFacingText()` duplicate handling so same-line, single-newline, and conservative partial duplicates left by final-tag stripping are removed without harming normal output.

## Best Possible Solution

Keep this issue open and implement a narrow sanitizer fix in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` with focused regression coverage in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts`. The fix should handle the concrete glued `</final>` repeat and any deliberately supported adjacent single-newline duplicate shape while preserving ordinary repeated wording, normal single-newline formatting, trailing whitespace, and code-block indentation.
