## Summary

Keep this issue open. Current main still does not implement the broad Anthropic native server-tools request: config and strict schema only cover managed web search/fetch plus OpenAI/Codex-native search, Anthropic transport still emits client-side tool definitions, stream parsing still lacks server-tool/result block preservation, and the Anthropic plugin manifest still has no web search/fetch or server-tool contract. Prior PRs #23469 and #23573 are closed unmerged, while open #50692 is scoped to #49949's narrower native web_search provider request and does not cover web_fetch, code_execution/PTC, allowed_callers, or server content-block handling.

## What This Changes

The issue requests opt-in Anthropic-native server-side web_search, web_fetch, code_execution, Programmatic Tool Calling, allowed_callers, and server response block support in OpenClaw.

## Best Possible Solution

Keep this issue open as the broad Anthropic native server-tools tracker. Let #49949/#50692 cover the narrower native web_search provider if that PR passes review, then separately design and implement web_fetch, code_execution/PTC, allowed_callers, server content-block preservation/rendering, docs, tests, and security defaults in the Anthropic plugin/transport boundary.
