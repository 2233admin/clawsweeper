## Summary

Keep this issue open. Current `main` still does not implement the broad Anthropic native server-tools request: there is no Anthropic `serverTools`, PTC, or `allowed_callers` config/schema surface; the Anthropic payload path only converts OpenClaw client tools into Anthropic client tool definitions; and the Anthropic stream parser still lacks explicit server-tool result block handling. Related PRs #23469 and #23573 are closed unmerged reference work, while open PR #50692 only targets the narrower #49949 native `web_search` provider.

## Best Possible Solution

Keep this issue open as the broad Anthropic native server-tools tracker. Let #49949/#50692 cover the narrower Anthropic native web_search provider if it passes review, then separately design web_fetch, code_execution/PTC, `allowed_callers`, server-tool content-block preservation/rendering, docs, tests, and security defaults for the broader request.
