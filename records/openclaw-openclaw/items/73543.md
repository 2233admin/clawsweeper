## Summary

Keep open. Current main still has the PR's target gap: path-shaped `tools.exec.allowlist` matching checks only `resolution.resolvedPath`, while execution planning can rewrite argv[0] to `resolvedRealPath`. The PR is a plausible focused fix with tests and no apparent workflow/dependency/supply-chain diff, but it is not obsolete on main and the linked #45595 is broader than this single matcher change.

## Best Possible Solution

Review this PR as a focused bug-fix candidate for path-shaped `tools.exec.allowlist` matching. Before landing, either expand it to cover #45595's remaining realpath trust/persistence surfaces or narrow the closing and changelog language so the broader issue remains open for follow-up. The maintainer review should explicitly decide the threat model for symlink-pattern allowlists.
