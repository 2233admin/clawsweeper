## What This Changes

The issue reports a Feishu/Lark regression after upgrading to OpenClaw 2026.4.26 where multi-account Feishu dispatch fails because a legacy npm-installed Feishu plugin expects `createScopedPairingAccess` from `openclaw/plugin-sdk/feishu`.

## Best Possible Solution

Review and land one canonical compatibility repair that restores the legacy `openclaw/plugin-sdk/feishu` facade needed by published Feishu packages, adds a focused SDK contract regression test, and then closes this issue after merge or release proof.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the published `@openclaw/feishu@2026.3.13` package imports and calls `createScopedPairingAccess` from `openclaw/plugin-sdk/feishu`, while current main and `v2026.4.27` do not expose that subpath.

## Solution Assessment

Yes, the likely best fix class is a narrow backward-compatible facade plus contract coverage, not a broader Feishu runtime rewrite. The issue should stay open while maintainers review #74219/#74223 or a consolidated replacement.
