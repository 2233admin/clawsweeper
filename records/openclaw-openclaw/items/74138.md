## Summary

Keep open. The reported `createScopedPairingAccess` failure matches a real compatibility gap between the old published `@openclaw/feishu` npm package and the OpenClaw 2026.4.26 plugin SDK surface, and current main does not clearly repair existing npm-installed Feishu configs.

## Best Possible Solution

Keep this open and choose a compatibility repair: either restore a backwards-compatible `openclaw/plugin-sdk/feishu` export for `createScopedPairingAccess` until old external Feishu packages are no longer loadable, publish/update `@openclaw/feishu` to match the current SDK, or migrate existing npm-installed Feishu configs back to the bundled plugin during update. Add a regression test or fixture that models an installed `@openclaw/feishu` package importing the old facade against the current host SDK.
