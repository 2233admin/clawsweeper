## Summary

Current main still lacks the requested built-in, configurable tool-level confirmation gate. OpenClaw has useful lower-level primitives, especially plugin `before_tool_call.requireApproval` from #55339 and host exec approvals, but there is no `tools.confirmation` config/schema surface, no core risk-tier classifier, and no model-response-level gate that can inspect same-response assistant text before the first tool executes.

## What This Changes

Add a configurable tool confirmation gate that can pause tool execution before first execution based on tool risk level and same-response assistant confirmation text.

## Best Possible Solution

Keep this issue open for core agent tool-execution design. The best path is a documented, backward-compatible `tools.confirmation` config/schema/help surface; deterministic tool-risk classification; access to same-response assistant text before the first tool executes; user-visible text flush before holding tools; explicit approve/deny/timeout behavior; transcript handling for skipped tool calls; and regression coverage that preserves plugin `before_tool_call.requireApproval` as the lower-level custom policy hook.
