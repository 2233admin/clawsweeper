## Summary

Keep open. Current main still lacks the requested opt-in Kubernetes ServiceAccount Trust / TokenReview authentication path for autoscaled worker or node pods. The repository now has a minimal single-pod Kubernetes/Kustomize deployment using shared token auth, but PR #12434 is closed unmerged and the broader distributed-runtime RFC #42026 does not replace this concrete auth/pairing request.

## Best Possible Solution

Keep this issue open for maintainer security/product review. If accepted, the implementation should be an explicit opt-in core gateway/node auth feature aligned with distributed runtime planning: add config types/schema/docs, protocol ConnectParams and GatewayClient/node-host support, fail-closed TokenReview validation with allowlists, pairing policy that only skips for validated node identities, focused tests, and Kubernetes deployment guidance. If maintainers choose a broader distributed-runtime canonical item, this request should be linked there only after the SA Trust requirement is explicitly covered.
