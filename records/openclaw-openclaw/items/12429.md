## Summary

Keep this issue open. Current main has a minimal single-pod Kubernetes/Kustomize deployment and existing token/password/device-pairing auth, but it does not implement the requested opt-in Kubernetes ServiceAccount Trust / TokenReview path for autoscaled node or worker pods. The linked implementation PR #12434 is closed unmerged, and the broader distributed runtime RFC #42026 does not specifically replace this authentication request.

## Best Possible Solution

Keep this issue open until OpenClaw either implements an opt-in Kubernetes ServiceAccount Trust path or maintainers choose a narrower canonical K8s auth/distributed-runtime item. A complete solution likely needs gateway config/types/schema, protocol ConnectParams and GatewayClient auth payloads, node-host credential handling, fail-closed TokenReview validation with namespace/service account/audience allowlisting, explicit pairing-skip policy only for validated node identities, tests, docs, and deployment assets.
