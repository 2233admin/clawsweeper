## Summary

Keep open. Current main has a shared CLI progress helper and applies it to several slow command paths, but the broad loading-state request is not fully satisfied because ClawHub skill search/install/update and plugin install/update flows still run network/download/install work through plain loggers or lifecycle tracing rather than a consistent progress wrapper.

## What This Changes

The issue requests loading/progress indicators for slow OpenClaw CLI commands so users can distinguish active work from a hang.

## Best Possible Solution

Keep this as a scoped CLI UX tracking item: pick the slow command families and delay/TTY semantics, then apply the existing progress helper consistently at those command boundaries while preserving JSON/plain output and adding focused tests/docs for the chosen surfaces.

## Reproduction Assessment

Unclear. The issue does not provide a concrete command/timing reproduction, but static inspection gives high confidence that some current ClawHub skill/plugin network and install paths still lack the shared progress wrapper.

## Solution Assessment

No proposed patch is under review. The best direction is not a new loader framework, but targeted expansion of the existing progress helper at slow command boundaries after maintainers define the command set and output semantics.
