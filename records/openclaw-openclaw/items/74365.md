## Summary

Keep this issue open. Current main already has a shared CLI progress helper and several slow command paths use it, but the broad loading-state request is not complete because ClawHub skill search/install/update and plugin install/update paths still run through direct ClawHub/npm/archive work with ordinary logging or lifecycle tracing rather than a consistent progress wrapper.

## What This Changes

Request to add loading or progress indicators to slow OpenClaw CLI commands so users can tell commands are still progressing.

## Best Possible Solution

Define a CLI progress policy for slow command families, delay thresholds, and output modes, then apply the existing progress helper at those command boundaries while preserving JSON/plain output and adding focused command tests/docs.
