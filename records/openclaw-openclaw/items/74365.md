## Summary

Keep open. Current main has a shared CLI progress helper and uses it on several slow paths, but the broad CLI loading-state request is still only partially covered: skills and plugin install/update/search flows still do ClawHub, archive, npm, and config work through ordinary log/tracing paths rather than a consistent progress wrapper.

## What This Changes

Request to add CLI loading/progress indicators for slow OpenClaw commands so users can tell whether a command is stuck or still working.

## Best Possible Solution

Keep this as a scoped CLI UX tracking item: choose the slow command families and output semantics, then apply the existing progress helper at those command boundaries while preserving JSON/plain output and adding targeted tests/docs.

## Reproduction Assessment

No high-confidence runtime reproduction is available from the report because it does not name a specific command or environment. Static inspection does identify concrete uncovered paths in skills and plugin install/update/search flows.

## Solution Assessment

Unclear as stated. The maintainable direction is not “add loaders everywhere,” but to extend the existing shared progress helper to selected slow command boundaries with explicit TTY, JSON, plain, and tracing behavior.
