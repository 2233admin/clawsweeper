## Summary

Keep this issue open. Current main has a shared CLI progress helper and progress coverage on several slow paths, but the broad request is not fully implemented because ClawHub skills and plugin install/update/search flows still rely on ordinary log lines or tracing rather than a consistent progress wrapper.

## What This Changes

The issue requests loading or progress indicators for slow OpenClaw CLI commands so users can tell whether commands are still working.

## Best Possible Solution

Keep this as a scoped CLI UX tracking item: choose the slow command families and delay/TTY semantics, then apply the existing progress helper consistently at those command boundaries while preserving JSON/plain output and adding targeted tests/docs.
