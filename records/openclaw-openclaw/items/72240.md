## Summary

Keep open. Current main has partial internal exec/process telemetry, but the foreground exec result and process poll surfaces still do not expose enough structured termination metadata for this macOS SIGKILL diagnostics report, and the related SIGKILL/supervisor reports remain open rather than merged or shipped fixes.

## What This Changes

The issue asks OpenClaw to surface actionable diagnostics when macOS exec/process commands are killed with SIGKILL, including timeout/cancellation cause, output tail, resource/log correlation, and safer handling for broad state/log scans.

## Best Possible Solution

Add a focused diagnostics contract improvement that carries existing supervisor/runtime termination state into foreground exec and background process results, with bounded stdout/stderr tails and a stable run/session/log correlation field. Keep signal-escalation policy changes in #66399 unless maintainers intentionally merge the scopes.

## Reproduction Assessment

Unclear: the report describes a credible recurring macOS pattern, but it does not provide a deterministic command that reliably SIGKILLs on current main. The missing metadata is reproducible by inspecting and unit-testing current timeout, no-output timeout, manual-cancel, and signal-result paths.

## Solution Assessment

Yes for a first repair: enriching result metadata is the narrowest maintainable fix for the observable diagnostics gap. A broader change to SIGTERM/SIGKILL escalation or scan prevention should be handled separately unless maintainers approve expanding the scope.
