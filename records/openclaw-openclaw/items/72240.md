## Summary

Keep open. Current main still has the user-facing exec diagnostics gap: foreground failures omit structured signal/failure metadata, SIGKILL without a timeout remains a generic signal message, and the internal diagnostic event is not enough to give the agent a run/log correlation point or distinguish supervisor cancellation from OS/process termination.

## Best Possible Solution

Keep this issue open and land a focused exec diagnostics improvement: add additive structured fields such as exitSignal, failureKind, terminationReason, timedOut/noOutputTimedOut, timeoutSec, durationMs, and a safe run/log correlation id to foreground exec failures and background process results; preserve useful output tail; and document the new diagnostics. Leave graceful SIGTERM escalation and process-drain policy to #66399 unless maintainers decide to fold the work together.
