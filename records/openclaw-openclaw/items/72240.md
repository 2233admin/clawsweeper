## Summary

Keep open. Current main still has a real user-facing diagnostics gap: exec and process paths track some signal, timeout, and exit-reason data internally, but foreground tool results and background poll details still do not expose enough structured metadata to distinguish timeout, no-output timeout, manual cancellation, supervisor SIGKILL, or OS SIGKILL. The related open issues overlap, but none is a merged/shipped fix for this macOS diagnostics request.

## What This Changes

The issue requests better diagnostics for intermittent macOS exec SIGKILL failures, including structured termination metadata, buffered output tails, cause hints, and log/run correlation.

## Best Possible Solution

Add a focused diagnostics contract improvement that propagates termination metadata through foreground exec and background process results: exitSignal, failureKind, exitReason/terminationReason, timedOut/noOutputTimedOut flags and limits, run/session/log correlation, duration, and bounded stdout/stderr tails. Keep signal escalation and process-drain behavior scoped to #66399 unless maintainers explicitly merge the work.

## Reproduction Assessment

Unclear: the report describes a credible recurring macOS pattern, but it does not include a deterministic current-main command that reliably SIGKILLs. The missing metadata is reproducible by source inspection and unit tests around simulated signal, timeout, no-output timeout, and cancellation outcomes.

## Solution Assessment

Yes for the first repair step: enriching exec/process result metadata is the narrowest maintainable fix for the observable diagnostics gap. A broader change to graceful SIGTERM/SIGKILL escalation should remain in #66399 unless maintainers approve expanding scope.
