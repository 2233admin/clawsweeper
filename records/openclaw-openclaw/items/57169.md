## Summary

Current main still has the reported policy mismatch: the macOS node declares and implements `screen.record`, but the gateway default allowlist keeps `screen.record` gated as a dangerous command and the test suite explicitly asserts that behavior. The issue is also paired with open, unmerged PR #57192, so it should stay open for maintainer review of the default-allow versus explicit-opt-in decision.

## Best Possible Solution

Keep this issue open and resolve it through PR #57192 or an equivalent maintainer decision. If macOS screen recording should be callable by default, update the gateway policy, regression tests, and docs/security messaging together. If explicit opt-in is intended, close or retarget PR #57192 and make the macOS node docs and CLI-facing guidance clearly say `screen.record` is supported but gated by `gateway.nodes.allowCommands`.
