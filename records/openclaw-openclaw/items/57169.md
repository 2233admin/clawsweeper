## Summary

Current main still has the reported policy mismatch: the macOS node declares and implements `screen.record`, while the gateway default platform policy treats `screen.record` as a dangerous command that requires explicit `gateway.nodes.allowCommands`. The issue should stay open because open PR #57192 already uses closing syntax for this issue and needs maintainer review of the privacy/security default-allow decision before any closure.

## Best Possible Solution

Keep this issue open and resolve it through review of PR #57192 or an equivalent maintainer-owned decision. If macOS screen recording should be callable by default, update the gateway policy, regression tests, docs, and security/audit messaging together. If explicit opt-in remains intended, close or retarget the PR and make the macOS node docs/CLI-facing guidance clearly say `screen.record` is supported but gated by `gateway.nodes.allowCommands`.
