## Summary

Keep this PR open. Current main still lacks the requested shared log redaction defaults on the `logs.tail` path, and the PR is a focused implementation candidate for the linked bug, but it needs a small correctness amendment before merge because the proposed Basic auth regex can leave padding outside the masked span.

## What This Changes

The PR adds default redaction regexes and tests for generic JWTs, Basic auth headers, `X-OpenClaw-Token`, `x-pomerium-jwt-assertion`, `X-Api-Key`, and `X-Auth-Token`.

## Best Possible Solution

Keep this PR as the implementation candidate for #66832, but amend it so shared default redaction fully covers padded Basic auth, standalone JWTs, and the named security headers, with focused `redactSensitiveText`, `redactSensitiveLines`, and `logs.tail` regression coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Mask Basic auth padding with the credential:** `src/logging/redact.ts:44`
  - body: The trailing `\b` after the base64 character class can make the match stop before padded `=` characters. In `Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=`, the padding remains outside the replacement span, so `logs.tail` can still return part of the raw credential string. Drop the word boundary or replace it with a delimiter that still includes padding.
  - confidence: 0.91
- **[P3] Exercise the standalone JWT pattern directly:** `src/logging/redact.test.ts:212-216`
  - body: This test puts the JWT behind `Authorization: Bearer`, which is already covered by the existing Bearer regex. It would still pass if the new generic JWT pattern stopped working, so use a standalone JWT or non-Bearer header sample to protect the behavior this PR adds.
  - confidence: 0.86
