## Summary

Current main still lacks the requested concurrent session-send tool, so this PR is not obsolete. It should stay open, but the branch is not merge-ready because it conflicts with current session/A2A contracts: main now uses per-session nested lanes, baseline-aware reply reads, repo-local `typebox` imports, and an ACP child A2A guard that this branch does not fully mirror.

## What This Changes

This PR adds a new `sessions_send_concurrent` agent tool with docs/tests, tool catalog and OpenClaw tool registration, a concurrent A2A helper, and gateway nested-lane concurrency changes.

## Best Possible Solution

Keep the feature candidate open, but rework it against current main by reusing existing `sessions_send` validation and A2A/run-wait helpers where possible, routing each target through the current per-session nested lane model, leaving the shared gateway `Nested` lane serialized, and adding focused tests for ambiguous targets, reply correlation, ACP child skip behavior, and concurrent per-session scheduling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Import the repo's typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: Current main imports schemas from the direct `typebox` dependency. The new tool imports `@sinclair/typebox` without adding that package as a direct dependency, so a clean install can fail to resolve the module. Use the repo's existing `typebox` import contract or deliberately update dependencies with maintainer approval.
  - confidence: 0.9
- **[P2] Keep the shared nested lane serialized:** `src/gateway/server-lanes.ts:10`
  - body: Current main uses per-session nested lanes for agent sends and tests that the shared `CommandLane.Nested` lane stays at default concurrency. Raising the shared lane here can interleave unrelated nested flows and conflicts with the current model; the batch tool should schedule each target on its resolved per-session lane instead.
  - confidence: 0.88
- **[P2] Reject ambiguous target references:** `src/agents/tools/sessions-send-concurrent-tool.ts:131-134`
  - body: A target can provide both `sessionKey` and `label`, and this implementation resolves the `sessionKey` path while ignoring the label. `sessions_send` rejects that combination to avoid generated payloads silently routing to a different session than intended, so the batch tool should enforce the same validation before sending.
  - confidence: 0.9
- **[P2] Route each target through its session lane:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: The batch send uses the shared `AGENT_LANE_NESTED` lane for target execution. Current main scopes nested agent work with `resolveNestedAgentLaneForSession(...)`, which lets different sessions run independently while preserving order for repeated sends to the same session. This branch loses that ordering and concurrency contract.
  - confidence: 0.86
- **[P2] Read the reply produced by this run:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, the tool reads the latest assistant reply from session history without a baseline or run correlation. If that session already had a reply or another send completes nearby, ping-pong and announce can operate on stale or unrelated text instead of the reply from this target run.
  - confidence: 0.84
- **[P2] Mirror the owned ACP child A2A skip:** `src/agents/tools/sessions-send-concurrent-tool.ts:404-416`
  - body: Current `sessions_send` skips ping-pong/announce when the requester owns its background ACP child, because that child already has a completion channel. The concurrent tool starts A2A follow-up unconditionally, which can re-enter the parent/child path that current main avoids.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: Security review needs attention because the new batch send can bypass the current owned ACP child A2A skip and re-enter a guarded parent/child session boundary.

Concerns:

- **[medium] Preserve the owned ACP child A2A guard:** `src/agents/tools/sessions-send-concurrent-tool.ts:404`
  - body: The PR starts A2A follow-up for accepted targets without mirroring the current `sessions_send` guard for parent-owned background ACP child sessions. That path can wake the parent and feed a parent response back into the child, crossing the session-control boundary that current main intentionally avoids.
