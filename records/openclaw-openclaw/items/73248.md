## Summary

Keep open. Current main has useful mitigations for large skills prompts, including prompt character/count limits, compact formatting, truncation notices, and per-agent allowlists, but it still uses a flat `<available_skills>` catalog and asks the model to scan descriptions before choosing a skill. The issue’s central request for hierarchical, tagged, or lazy skill discovery remains unimplemented and is not fully covered by the related truncation fix.

## Best Possible Solution

Keep this issue open for a core skills prompt/discovery design. A good implementation would preserve existing prompt budgets, allowlists, plugin-bundled skills, and SKILL.md on-demand loading, while adding a structured discovery layer such as skill categories/tags or lazy retrieval, with config schema, docs, and tests proving reduced prompt size and stable model-facing selection behavior.
