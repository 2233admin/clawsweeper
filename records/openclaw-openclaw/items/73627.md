## Summary

Keep this PR open. Current main does not include the PR's central embedded cron `sandboxSessionKey` handoff or the new regression test, and the linked bug report #73586 remains the active user-facing report. The PR is a plausible implementation candidate, but it still needs maintainer review and tighter coverage for the actual cron/heartbeat/sub-agent Docker-probe path.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73586. The best path is to verify the real caller path, pass the policy session key where needed, add a regression that exercises `runCronIsolatedAgentTurn` or the embedded runner call boundary with `agents.defaults.sandbox.mode: "off"` and proves Docker/backend initialization is not reached, then confirm heartbeat/sub-agent coverage before merge. The duplicated context guard should either be dropped or justified in review.
