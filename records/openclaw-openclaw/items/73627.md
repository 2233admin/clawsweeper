## Summary

Keep this PR open. Current main still lacks the PR's central `sandboxSessionKey` handoff for embedded isolated cron runs, and the linked bug #73586 remains open. This is an active implementation candidate, not conservative cleanup-closeable.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73586. The best path is to review the narrow `sandboxSessionKey: params.agentSessionKey` handoff, keep or add a regression test proving `sandbox.mode: "off"` avoids sandbox backend initialization for isolated cron runs, verify heartbeat/sub-agent coverage, and confirm `non-main`/`all` sandbox semantics before merge.
