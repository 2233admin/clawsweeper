## Summary

Keep this PR open for maintainer review. Current main still lacks the cron `sandboxSessionKey` handoff and the added regression file, while the linked bug report is recent, detailed, and still open. The PR is not obsolete, but its current regression test appears too narrow because it exercises the existing `resolveSandboxContext` off-mode guard rather than the cron/heartbeat/sub-agent path reported to bypass that guard.

## Best Possible Solution

Keep the PR open and review it as the active implementation candidate for #73586. The best path is to tighten the regression so it exercises the automated-session caller path that currently omits `sandboxSessionKey`, prove Docker/backend initialization is not reached with `agents.defaults.sandbox.mode: "off"`, and then either keep the explicit cron handoff or extend the same policy-session-key fix to heartbeat/sub-agent paths if they share the bug. Drop or justify the context rewrite if it remains behaviorally equivalent to the existing guard.

## Risks / Open Questions

- The added regression test mocks `resolveSandboxRuntimeStatus()` to return `sandboxed: false` and calls `resolveSandboxContext()` directly; that appears to pass against current main's existing resolver guard, so it may not prove the reported cron/heartbeat/sub-agent Docker-probe bypass.

- The bug report also mentions heartbeat and sub-agent sessions, while the PR only changes the cron executor. Maintainers should confirm whether the same missing policy-session-key handoff exists on those paths before merging as a complete fix.
