## Summary

Keep this open. Current main still resolves bootstrap context from workspaceDir only, with no agentDir parameter or misplaced-bootstrap warning in the resolver or the main runtime call sites. PR #72871 is open and unmerged with an Option B warning implementation that explicitly targets this issue, while earlier linked attempts are closed unmerged.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review and either land or supersede PR #72871 with a maintained Option B warning, or explicitly define a tested Option A agentDir/workspace merge or override contract. The final fix should thread agentDir through every bootstrap-context call path, add focused regression tests for misplaced bootstrap files, and update the public workspace/config docs to make the workspace-vs-agentDir boundary explicit.
