## Summary

Keep this PR open. Current `main` at `f256eeba431b5f59e6085b7c11c9c0a309e408b7` still does not explicitly reject null bytes in archive entry paths, while the PR remains a narrow security hardening change to the shared archive validation path. The prior Greptile and ClawSweeper comments correctly identify the same placement and scope.

## Best Possible Solution

Review and land this PR, or an equivalent minimal patch, by adding an explicit `entryPath.includes("\0")` rejection in `src/infra/archive-path.ts` immediately after the empty-like path return and before drive-path/normalization checks, with the focused regression coverage in `src/infra/archive-path.test.ts`.
