## Summary

Keep this PR open. Current main still does not explicitly reject null bytes in `validateArchiveEntryPath()`, and the archive extraction paths for ZIP and TAR rely on that helper before resolving output paths. The live PR is narrowly scoped to adding the missing guard and two focused tests, with no workflow, dependency, lockfile, install script, or publishing-surface changes observed in the diff.

## Best Possible Solution

Review and land this PR, or an equivalent minimal change, by adding explicit null-byte rejection in `src/infra/archive-path.ts` before normalization plus focused regression coverage in `src/infra/archive-path.test.ts`.
