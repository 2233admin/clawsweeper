## Summary

Keep this issue open. Current main still accepts Chrome internal CDP targets in the browser plugin's tab listing and default selection paths, and the related fix PR is closed unmerged, so the reported behavior remains actionable.

## What This Changes

Requests that CDP browser control ignore Chrome internal targets such as chrome://omnibox-popup.top-chrome/ and chrome://tab-search.top-chrome/ so navigation uses real web content tabs.

## Best Possible Solution

Add a shared browser-plugin predicate for selectable web-content CDP targets, use it consistently in raw CDP listing, Playwright-backed listing, implicit/default target selection, explicit target resolution, and tab cleanup, and fall back to opening/selecting about:blank when only Chrome internal UI targets are visible.
