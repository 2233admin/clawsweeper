## Summary

Keep open. Current main has prompt-cache retention, cache-TTL pruning, prompt-cache observability, and heartbeat-based cache warming, but it does not implement the requested invisible non-turn Anthropic keep-warm ping with cost caps and lifecycle cancellation. The related foundation PRs are useful context, but the remaining behavior is a product/design decision around hidden billable provider calls rather than conservative cleanup closure.

## What This Changes

Request an opt-in Anthropic prompt-cache keep-warm mechanism that sends invisible minimal non-turn pings between user messages to avoid TTL-driven cache misses.

## Best Possible Solution

Decide the product boundary first, then either add a documented opt-in Anthropic keep-warm maintenance mode that reuses the existing payload/cache telemetry path with hard cost and lifecycle safeguards, or explicitly document heartbeat/cache-TTL pruning as the supported workaround if hidden pings are out of scope.

## Reproduction Assessment

Not applicable as a feature request. The current-main check is high-confidence for absence of the requested surface because targeted code/config/docs searches found no keep-warm ping mode, while the only documented keep-warm path is heartbeat.

## Solution Assessment

Unclear. The proposed non-turn ping design is plausible, but it is not automatically the best fix until maintainers decide how OpenClaw should handle hidden billable API calls versus the existing heartbeat and cache-TTL pruning controls.
