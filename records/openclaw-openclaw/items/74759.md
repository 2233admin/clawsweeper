## Summary

Keep this open. Current main has an eager configured SecretRef system with env/file/exec providers, including 1Password and Vault-style exec examples, but it does not provide lazy agent-callable `secret.get`/`secret.has`/`secret.list` tools or the trusted pre-model redaction and session-scope contract that would make those tools safe.

## What This Changes

Request a generic secret broker plugin/tool surface that resolves named secrets from Bitwarden, macOS Keychain, 1Password, or Vault at agent call time with redaction and session-trust guards.

## Best Possible Solution

Define a trusted secret capability contract on top of the existing SecretRef provider foundation, including redacted or non-persistent result semantics and a session-scope gate, then implement the first backend either as a trusted bundled plugin or as an external plugin after the plugin API can safely express that trust boundary.

## Reproduction Assessment

Not applicable as a feature and security-design request. The current-main gap is high-confidence from source and docs inspection: configured SecretRefs exist, while lazy agent-callable `secret.*` tools and a safe redaction/session-scope API are absent.

## Solution Assessment

Unclear. The requested generic plugin direction is not sufficient by itself because external plugins cannot currently register the trusted pre-model result middleware; the safer path is a maintainer-approved bundled implementation or a core plugin-API extension first.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because it proposes returning live credential material through agent tools before OpenClaw has a safe secret-result and session-trust contract for that surface.

Concerns:

- **[high] Define redacted secret result semantics before get:** `src/plugins/registry.ts:369`
  - body: A `secret.get` tool would intentionally return credential material. Current external plugin tools return normal tool content, while pre-model tool-result rewriting is restricted to bundled plugins, so core needs a trusted redacted/non-persistent result contract before any backend returns secrets.
  - confidence: 0.93
- **[high] Gate secret tools by trusted session scope:** `docs/channels/groups.md:112`
  - body: OpenClaw has distinct group/channel sessions and group authorization behavior. Secret tools should be hidden or blocked for group and otherwise untrusted sessions before execution, not only redacted after the tool result is produced.
  - confidence: 0.86
