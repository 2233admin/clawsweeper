## Summary

Current main still uses Baileys `useMultiFileAuthState` for the bundled WhatsApp plugin. The shipped hardening around `creds.json` atomic writes and backup recovery does not replace the multi-file signal-key store with a SQL/consolidated production auth store, and the related implementation PR was closed unmerged.

## Best Possible Solution

Keep this issue open until the bundled WhatsApp plugin replaces Baileys `useMultiFileAuthState` with a production auth store, safely migrates existing multi-file credentials and signal keys, preserves login/logout/status/config behavior, updates the public WhatsApp docs and config help, and adds focused tests for fresh install, migration, corruption recovery, and key persistence. PR #66863 can inform the design but should not be treated as a completed fix.
