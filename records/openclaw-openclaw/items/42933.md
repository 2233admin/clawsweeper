## Summary

Keep open. Current main still does not implement the PR's intended configurable session-level circuit breaker. Main has adjacent protections such as heartbeat duplicate suppression, cron job error/backoff/failure-alert state, auth-profile cooldowns, and context-overflow recovery/reset paths, but it lacks the requested `agents.defaults.circuitBreaker` / per-agent config, strict schema support, persisted `cb*` session state, and session-level pause/reset/alert integration across heartbeat, cron, and interactive reply paths.

## Best Possible Solution

Keep this PR open and ask for a focused revision. The best path is a core-owned session-level breaker with typed `agents.defaults.circuitBreaker` and per-agent schema, persisted per-session breaker state, structured model-failure metadata from interactive reply, heartbeat, and cron execution, correct isolated-heartbeat session targeting, `/new`-equivalent reset behavior, sanitized/validated external alert delivery, and targeted tests for heartbeat, cron, and interactive reply paths.
