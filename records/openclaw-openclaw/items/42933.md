## Summary

Keep open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still does not implement this PR's intended configurable session-level circuit breaker. Main has adjacent mitigations such as provider/auth-profile cooldowns, cron retry backoff/failure alerts, tool-loop detection, heartbeat duplicate suppression, and high-context timeout compaction/reset recovery, but it lacks `agents.defaults.circuitBreaker` / per-agent config, strict schema support, persisted per-session `cb*` breaker state, and pause/reset/alert enforcement across heartbeat, cron, and interactive reply paths.

## Best Possible Solution

Keep this PR open for maintainer follow-up or a superseding core implementation. The best path is a core-owned session-level breaker with documented `agents.defaults.circuitBreaker` and per-agent schema, persisted per-session breaker state, structured model-failure metadata from interactive reply, heartbeat, and cron execution, correct isolated-heartbeat session targeting, `/new`-equivalent reset behavior, sanitized external alert delivery, and targeted tests for heartbeat, cron, and interactive reply paths.
