## Summary

Keep open. Current main still lacks the requested text/thinking repetition breaker and loop-aborted replay quarantine. The idle watchdog remains chunk-activity based, the existing loop detector is scoped to tool calls, and #39961 is open/unmerged as an implementation candidate rather than shipped behavior.

## What This Changes

Request to add a semantic streaming repetition breaker plus context sanitization so continuous repetitive model output cannot evade idle timeout or poison later turns.

## Best Possible Solution

Review, land, adapt, or replace #39961 with a core assistant-stream guard that detects repeated text/thinking while chunks are still arriving, aborts with explicit loop metadata, suppresses any buffered repetitive final drain, and excludes loop-aborted assistant output from future provider replay while keeping transcript visibility intact.
