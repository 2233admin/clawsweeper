## Summary

Keep this issue open because current main still treats model stream idleness as absence of chunks, has no semantic repetition guard for assistant text/thinking streams, and only ships loop detection for tool calls. The related #39961 is an open implementation candidate rather than merged behavior, so the central request remains unresolved.

## What This Changes

The issue requests a semantic streaming repetition breaker and loop-aware context sanitization so active but repetitive model streams cannot bypass idle timeout or poison future session context.

## Best Possible Solution

Implement or land a core stream guard near the assistant stream subscription path that detects repeated text/thinking while chunks are still arriving, aborts with explicit loop metadata, suppresses any buffered repetitive final drain, and quarantines loop-aborted assistant output from future model replay while keeping transcript visibility intact.
