## Summary

Do not close #57901 yet. Current main has a shipped partial fix for manual `/compact` and context-engine compaction paths, but the initialized Pi safeguard path still registers the session model and the `session_before_compact` handler still prefers `ctx.model`, so the exact active-session safeguard concern is not conclusively implemented.

## Best Possible Solution

Keep the issue open and fix the initialized safeguard path explicitly: resolve `agents.defaults.compaction.model` before `setCompactionSafeguardRuntime()` for active Pi runs, or carry a dedicated compaction target model in runtime and prefer it in `session_before_compact`. Add regression coverage where `ctx.model` is the primary/session model and the configured compaction model is different, while preserving the existing manual `/compact` and context-engine override behavior.
