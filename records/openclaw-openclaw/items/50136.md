## Summary

Keep this PR open. Current main still has the Windows restart cleanup behavior the PR is trying to harden: Windows restart enables unknown-listener fallback, unknown listener PIDs can flow into `staleGatewayPids`, and that same list is terminated before retrying. Current main also still classifies any listener text containing `openclaw` as a gateway. The PR is not merge-ready as submitted because review comments identify warning-array and test-isolation issues, but it is not obsolete.

## Best Possible Solution

Do not auto-close. Maintainers should either ask the contributor to rebase and address the review findings, or replace it with a focused maintainer patch that separates diagnostic unknown listener PIDs from verified actionable gateway PIDs, tightens listener classification using parsed argv/executable checks, preserves the existing runtime-lag probe success behavior, and adds targeted Windows restart-health and stale-cleanup tests. If #66627 is the preferred canonical thread for the remaining unknown-listener work, close or supersede this PR only after that relationship is confirmed and communicated.
