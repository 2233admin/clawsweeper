## Summary

Current main still rejects whitespace in Docker setup paths, so the reported Windows Docker install regression is not implemented. The provided GitHub context also shows open PR #44671 referencing this issue with closing syntax, so the issue should remain open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Keep this issue open and review or close PR #44671. The likely fix belongs in scripts/docker/setup.sh: allow ordinary spaces in OPENCLAW_CONFIG_DIR and OPENCLAW_WORKSPACE_DIR, continue rejecting control characters and unsafe mount syntax, quote generated compose volume entries, and add focused e2e coverage for spaced config/workspace paths.
