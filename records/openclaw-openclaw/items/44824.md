## Summary

Keep open. Current main now has a Node-based docs formatter, so the original Unix `xargs` pipeline is partly addressed, but I would not close this PR as implemented yet: `format:docs:check` still sends every docs file as one `oxfmt` invocation using absolute temp paths, which is likely to exceed Windows command-line limits under normal native PowerShell temp paths. The submitted PR also still has unresolved review feedback around Windows `.cmd` shim execution and launch-error diagnostics, so this needs amendment or consolidation rather than cleanup closure.

## Best Possible Solution

Keep this PR open for amendment or consolidate it with the canonical fix for #44293. The best implementation is a single docs formatter wrapper on main that removes `xargs`, enumerates tracked docs with a structured `git ls-files` call, chunks formatter invocations for Windows command-line limits, invokes repo-local `oxfmt` without relying on `.cmd` shim lookup, reports child-process launch errors, and is verified in native PowerShell plus the existing Linux/macOS path.
