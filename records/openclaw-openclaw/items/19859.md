## Summary

Current main still hardcodes plain Enter as send in the WebChat/Control UI composer and does not expose a send-shortcut preference in UI settings, gateway config, or docs. The same remaining work is already tracked by open canonical issue #8147 and open maintainer-labeled PR #73390, so this should stay open without creating a parallel repair lane.

## What This Changes

The issue requests a WebChat/Control UI option to choose whether Enter sends immediately or inserts a newline, with a modifier shortcut for sending.

## Best Possible Solution

Keep this open until the canonical WebChat send-shortcut work lands or maintainers explicitly consolidate it. The finished behavior should support Enter-as-newline with Ctrl/Cmd+Enter send, preserve Shift+Enter newline and IME behavior, and update the relevant persistence/config/docs plus focused composer tests.
