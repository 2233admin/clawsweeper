## Summary

Current main still does not provide a config option to suppress only the restart-sentinel outbound notification. The restart-sentinel startup path still removes the sentinel, wakes the session, and unconditionally calls outbound notice delivery whenever routing resolves, while the exposed config schema/docs cover generic internal hooks only and do not define a restart-sentinel notify/disable field.

## Best Possible Solution

Keep this issue open. The best implementation is a first-class restart-sentinel config flag in the gateway/config schema that gates only the call to deliverRestartSentinelNotice after routing resolution, with docs/help coverage and focused tests in src/gateway/server-restart-sentinel.test.ts proving wake and continuation still run when outbound notification is disabled.
