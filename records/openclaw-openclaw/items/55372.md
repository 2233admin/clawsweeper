## Summary

Current main still attempts an outbound restart-sentinel notice whenever startup consumes a sentinel with a resolvable delivery target, and there is no current config/schema/docs path to disable that notification while preserving the session wake. The linked merged PR #53940 fixed wake/routing and explicitly left config changes out of scope, so this remains an actionable core gateway/config request rather than a duplicate cleanup close.

## Best Possible Solution

Keep this open. The best implementation is a narrow core gateway config flag, wired near restart-sentinel startup handling, that suppresses deliverRestartSentinelNotice while leaving sentinel consumption, system-event wake, heartbeat wake, and restart continuation delivery intact. Add focused coverage in src/gateway/server-restart-sentinel.test.ts for the disabled-notification path.
