## Summary

Keep open. Current main has partial mitigations for related gateway responsiveness cases, but it still runs agent prompt/context/tool preparation inside the gateway process and the central system-prompt path remains synchronous, so there is no high-confidence evidence that the reported RPC starvation during agent prep is fixed. The open companion report #75329 should be triaged with this issue rather than treating this as stale or implemented.

## What This Changes

Report requests investigation and a fix for gateway API hangs, 100% event-loop utilization, and multi-second RPC delays during agent preparation in OpenClaw 2026.4.29.

## Best Possible Solution

Treat this issue and #75329 as one gateway performance incident: profile packaged 2026.4.29 and current main during an agent run while polling Gateway RPCs, then reduce, yield, cache, or isolate the long prompt/context/tool prep path so the gateway remains responsive throughout preparation.
