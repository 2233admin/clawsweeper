## Summary

Keep open. The report is current, concrete, and not implemented on current main: the embedded-run prep path still performs system-prompt construction inside the gateway process, the prompt builder and skill/frontmatter loading paths remain synchronous, and the event-loop readiness code is diagnostic/client-side rather than a mitigation for RPC starvation during agent prep.

## What This Changes

The issue requests that gateway agent preparation stop starving the Node event loop so RPC endpoints remain responsive during prompt, tool, and session setup.

## Best Possible Solution

Treat #75329 and this issue as one gateway performance incident: profile packaged 2026.4.29 and current main on a comparable Linux setup, then remove or break up the long synchronous prep work so prompt, skill/frontmatter, plugin-tool, and session-resource setup cannot starve gateway RPC handling.
