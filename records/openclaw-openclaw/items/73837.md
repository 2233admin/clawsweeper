## Summary

Keep open. Current main has installer-source hardening for the reported gum ioctl and apt noninteractive paths, but the live hosted https://openclaw.ai/install.sh still differs from source and still contains the reporter's v22/ioctl-era behavior, so the user-facing installer path is not proven fixed.

## Best Possible Solution

Keep this open and have maintainers sync or redeploy the hosted openclaw.ai installer from current `scripts/install.sh`, then add a release/publish guard that proves the public installer matches the intended source or release artifact. The fix should specifically verify the WSL2/Ubuntu manual-download path where Node is absent: gum ioctl fallback runs the underlying command, apt uses noninteractive wrappers, and the script does not print Node install success unless `node` is actually available and satisfies the version check.
