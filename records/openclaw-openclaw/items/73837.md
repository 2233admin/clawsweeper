## Summary

Keep open. Current source and the v2026.4.27 tag contain installer hardening for noninteractive apt and gum ioctl fallback, but the live hosted installer still matches the stale v22/raw-apt behavior reported by the user, and open PR #73855 still explicitly claims to close this issue. ## What This Changes

The issue asks the installer to detect Node.js auto-install failures accurately on WSL2 Ubuntu instead of printing Node.js success and then crashing because Node is still unavailable.

## Best Possible Solution

Keep this issue open until maintainers redeploy or sync the hosted `install.sh` from the hardened source, then land a reviewed hosted-installer drift guard such as #73855 if it fits the workflow policy. Final proof should compare hosted/source markers or hashes and ideally cover WSL2 Ubuntu with no preinstalled Node so success is reported only after a usable `node` is on PATH.
