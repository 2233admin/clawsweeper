Keep open. Current source and the latest release tag include installer hardening for noninteractive apt and gum ioctl fallback, but the live hosted installer still serves the older v22/raw-apt script that matches the report, and an open linked PR still uses closing syntax for this issue.

## What This Changes

The issue asks for the Linux/WSL installer to detect Node.js auto-install failures accurately instead of printing Node.js install success and then crashing when `node` is unavailable.

## Best Possible Solution

Redeploy `https://openclaw.ai/install.sh` from the hardened `scripts/install.sh`, then review and either land or replace #73855 with a durable hosted-installer drift guard. Close this issue only after the hosted script matches source and the no-preinstalled-Node WSL path cannot print success unless `node` is actually usable.
