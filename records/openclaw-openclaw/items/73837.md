## Summary

Keep open. Current main and the v2026.4.26 tag contain source-side installer hardening for gum ioctl fallback and noninteractive apt use, but the documented hosted https://openclaw.ai/install.sh still serves an older v22-era script that matches the reporter's failure mode. The remaining user-visible problem is hosted installer drift/redeployment, not lack of source intent.

## Best Possible Solution

Keep this open until maintainers sync or redeploy the hosted openclaw.ai installer from the hardened source/release artifact, then add a release/publish guard that proves the public installer matches the intended source or at least contains the gum ioctl fallback, noninteractive apt wrapper, and Node 24/default-version behavior. The verification path should cover WSL2/Ubuntu with no preinstalled Node and ensure the script does not print Node install success unless `node` is actually available and satisfies the required version check.
