## Summary

Keep open. Current main still uses the low-alpha `--accent-subtle` token for global browser text selection in the Control UI/WebChat stylesheet, and PR #60854 remains the open implementation candidate that explicitly targets this issue with closing syntax.

## Best Possible Solution

Keep the issue open and review/land PR #60854, or apply an equivalent Control UI CSS fix that uses a visibly higher-contrast `::selection` background/text pair and confirms the result with a quick visual accessibility check or lightweight CSS contract coverage.
