## Summary

Keep open. Current main still has skill metadata gates and `openclaw skills` search/install/update/list/info/check commands, but it does not define a skill capability manifest or an `openclaw skills inspect` command. The RFC is coherent, recent, and security-adjacent, with overlap against open permission-manifest/security discussions that need maintainer product/security judgment before implementation.

## What This Changes

Request to add an optional skills-first capability manifest and an `openclaw skills inspect` command that displays tools/files/exec declarations without changing runtime behavior in v0.

## Best Possible Solution

Make an explicit maintainer decision on the public skill capability vocabulary and its relationship to #12219/#10827; if accepted, add a typed schema, parser/status propagation, CLI rendering, docs, and targeted tests with clear inspect-only wording.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a bug report. The current-main check is high confidence: source and docs inspection show no capability manifest field or `openclaw skills inspect` command.

## Solution Assessment

Unclear until maintainers choose the permission vocabulary and consolidation path. The proposed inspect-only v0 is plausibly narrow, but the security UX must avoid implying enforcement and may belong with the broader open manifest/security design.
