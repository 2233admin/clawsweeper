## Summary

Keep open. Current main and the latest release still lack the proposed skill capability manifest shape and `openclaw skills inspect` flow, while the related open threads cover broader permission/security designs rather than a resolved implementation of this narrower inspect-only slice.

## What This Changes

The issue proposes optional skill capability manifests and an `openclaw skills inspect` command that displays declared tools, file, and exec access without changing runtime enforcement.

## Best Possible Solution

Maintainers should decide whether this inspect-only vocabulary is accepted as a separate first phase or folded into #12219/#10827. If accepted, implement a typed schema, parser/status propagation, human and JSON CLI rendering, docs, and targeted tests with explicit inspect-only language.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a runtime bug. Source, docs, tests, and release-tag inspection give a high-confidence verification path that the proposed schema and command are absent.

## Solution Assessment

Unclear. The proposed v0 is a plausible narrow first step, but the best maintainable solution depends on maintainer agreement about the public permission vocabulary and how it should relate to the broader security-manifest work.
