## Summary

Keep open. Current main still lacks the requested skill capability manifest and `openclaw skills inspect` command, while the related threads are broader prior art rather than a merged or canonical replacement for this narrow inspect-only RFC. Because the proposal defines a public permission vocabulary, it needs maintainer product/security review before implementation.

## What This Changes

The issue proposes an optional skills-first capability metadata block for tools, files, and exec plus an `openclaw skills inspect <skill-id>` command, with no runtime enforcement in v0.

## Best Possible Solution

Make an explicit maintainer product/security decision on the skills-first capability vocabulary and its relationship to #12219 and #10827. If accepted, implement a typed manifest schema, parser/status propagation, CLI renderer, docs, and focused tests with clear inspect-only wording.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a runtime bug. Code, docs, and test inspection gives high-confidence verification that current main does not expose the requested manifest or `skills inspect` flow.

## Solution Assessment

Unclear. The proposed v0 is a plausible narrow first step, but the best maintainable path depends on maintainer agreement about the permission vocabulary and whether it should be split from or folded into the broader manifest/security threads.
