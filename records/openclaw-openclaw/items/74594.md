## Summary

Keep open. Current main has skill load-time metadata and `openclaw skills info/list/check`, but it does not define an inspectable capability manifest or an `openclaw skills inspect` command; the RFC is coherent and touches a public skill/security contract that needs maintainer product and security review.

## What This Changes

Add an optional skill capability manifest and an inspect command that displays requested tool, file, and exec access without changing runtime enforcement in v0.

## Best Possible Solution

Make an explicit maintainer decision on the public skill capability vocabulary, then either define the typed schema in the skill metadata path with CLI/docs/tests or close this in favor of a canonical security/design issue if that is the intended tracking path.
