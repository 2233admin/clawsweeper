## Summary

Keep open. Current main still does not auto-select the only enabled account for omitted-account message sends. The shared message runner only preserves explicit/tool-default/agent-binding account IDs, channel selection only chooses a channel, and WhatsApp still falls back to its default account resolver when no account is passed. The related fix PR #20801 is closed unmerged, so this remains a valid narrow fix candidate.

## Best Possible Solution

Keep this issue open and implement a shared outbound message-account resolver after channel selection. It should use the selected channel plugin's config adapter to preserve explicit accountId, input.defaultAccountId, and agent binding first; then prefer an enabled configured default account, auto-select exactly one enabled/configured account, and return clear zero/multiple-enabled-account errors. Add focused shared-runner/channel-selection tests, WhatsApp coverage for the omitted-account path, docs updates for routing/message behavior, and a user-facing changelog fix entry.
