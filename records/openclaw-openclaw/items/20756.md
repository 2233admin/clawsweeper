## Summary

Keep open. Current main still does not auto-select the only enabled account when the message tool omits `accountId`; the shared runner only preserves explicit/default/bound account IDs, WhatsApp omitted sends still fall back to the default-account helper, and the linked implementation PR #20801 is closed unmerged.

## What This Changes

The issue requests that `message` sends without `accountId` prefer an enabled default account, otherwise auto-select exactly one enabled account, and require explicit selection only when multiple accounts are enabled.

## Best Possible Solution

Implement a shared outbound message-account resolver after channel selection. Preserve explicit/default/bound account IDs first; then use the selected channel plugin config adapter to prefer an enabled configured default account, auto-select exactly one enabled/configured account, and throw clear zero-or-multiple-enabled-account errors. Add shared runner/channel-selection tests, WhatsApp regression coverage, routing/message docs, and a user-facing changelog fix entry.
