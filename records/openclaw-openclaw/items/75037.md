## Summary

Keep open. Current main still has compaction and memory-flush metadata paths that can refresh `updatedAt`, while freshness evaluation still falls back to `updatedAt` for legacy rows; the linked closing PR #75066 is open and unmerged, so this issue should remain the tracker until that PR or an equivalent fix lands.

## What This Changes

The issue requests that compaction and memory-flush bookkeeping preserve activity timestamps so legacy daily and idle reset evaluation is not skipped after metadata-only writes.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that makes compaction and memory-flush bookkeeping preserve activity timestamps, with regression coverage for legacy rows missing `sessionStartedAt` and `lastInteractionAt`.

## Reproduction Assessment

Yes. A high-confidence reproduction is a legacy session-store row missing lifecycle timestamps, followed by current compaction or memory-flush metadata persistence and then `initSessionState`/`evaluateSessionFreshness` on the next real user message.

## Solution Assessment

Yes. Applying the existing preserve-activity policy to metadata-only compaction and flush writes is the narrowest maintainable fix, and it matches the prior `updateLastRoute` guard rather than broadening reset fallback semantics globally.
