## Summary

Keep open. Current main still has both reported metadata paths refreshing `updatedAt`, and the linked fix PR remains open and unmerged, so this issue should stay as the tracker until PR #75066 or an equivalent patch lands.

## What This Changes

The issue asks OpenClaw to stop compaction and memory-flush bookkeeping from advancing activity timestamps in a way that can suppress daily or idle resets for legacy session rows.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that applies preserve-activity semantics to compaction and memory-flush bookkeeping while keeping real user or agent turns responsible for activity timestamps, with regression coverage for legacy rows missing `sessionStartedAt` and `lastInteractionAt`.

## Reproduction Assessment

Yes. The issue provides a concrete legacy session-store reproduction, and source inspection confirms the same path: metadata writes refresh `updatedAt`, then freshness evaluation can use that refreshed timestamp when lifecycle timestamps are absent.

## Solution Assessment

Yes. Preserving activity timestamps at the metadata write sites is the narrowest maintainable fix; changing the global freshness fallback would be broader and could affect other legacy-session behavior.
