## Summary

Keep open. Current main still has the reported compaction and memory-flush metadata paths that can refresh `updatedAt`, while reset freshness still falls back to `updatedAt` for legacy rows; the targeted closing PR is open and unmerged, so this issue should remain the tracker.

## What This Changes

The issue requests that compaction count updates and memory-flush metadata persistence preserve activity timestamps so legacy daily and idle session resets still fire correctly.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that makes compaction and memory-flush bookkeeping use the preserve-activity merge path, with regression coverage for legacy rows missing `sessionStartedAt` and `lastInteractionAt`.

## Reproduction Assessment

Yes. A high-confidence reproduction is a legacy session-store row missing lifecycle timestamps, followed by current compaction or memory-flush metadata persistence and then `initSessionState`/`evaluateSessionFreshness` on the next real user message.

## Solution Assessment

Yes. Applying the existing preserve-activity policy to these metadata-only writes is the narrowest maintainable fix; changing the global legacy fallback would be broader and could alter unrelated session behavior.
