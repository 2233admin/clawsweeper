## Summary

Keep open. Current main still lets compaction and memory-flush bookkeeping refresh `updatedAt`, while legacy daily/idle freshness still falls back to `updatedAt`; PR #75066 is open and unmerged, so this issue should remain the tracker until that fix or an equivalent patch lands.

## What This Changes

The issue requests that compaction and memory-flush metadata writes preserve activity timestamps so legacy daily and idle session reset decisions are not skipped.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that makes compaction and memory-flush bookkeeping preserve activity timestamps, with regression coverage for legacy rows missing `sessionStartedAt` and `lastInteractionAt`; avoid broad reset-policy rewrites unless a separate behavior gap is found.

## Reproduction Assessment

Yes. The high-confidence path is a legacy session-store row missing lifecycle timestamps, followed by current compaction or memory-flush metadata persistence and then `initSessionState`/`evaluateSessionFreshness` on the next real user message.

## Solution Assessment

Yes. Extending the existing preserve-activity policy to metadata-only compaction and flush writes is the narrowest maintainable fix; changing the global legacy fallback would have a wider behavioral blast radius.
