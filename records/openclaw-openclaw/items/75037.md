## Summary

Keep open. Current main still has both reported metadata-write paths that can refresh `updatedAt`, and the linked fix PR is open and unmerged, so this issue should remain the tracking item until that PR or an equivalent narrow fix lands.

## What This Changes

Compaction and memory-flush metadata writes should preserve activity timestamps so legacy daily and idle session reset decisions are based on real activity rather than bookkeeping writes.

## Best Possible Solution

Review and land PR #75066 or an equivalent narrow fix that makes compaction-count and memory-flush bookkeeping preserve activity timestamps, while keeping real user or agent turns responsible for activity updates and adding regression coverage for legacy rows missing lifecycle fields.

## Reproduction Assessment

Yes. A high-confidence reproduction is a legacy session-store row missing `sessionStartedAt` and, for idle mode, `lastInteractionAt`, followed by current compaction or flush metadata persistence and then `initSessionState`/`evaluateSessionFreshness` on the next real message.

## Solution Assessment

Yes. A targeted preserve-activity path for compaction and flush metadata is the narrowest maintainable fix; changing reset fallback semantics globally would be broader and could disturb other legacy-session behavior.
