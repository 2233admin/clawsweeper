## Summary

Keep open for maintainer/security review. Current main still lacks the Discord guild-admin requester authorization that this PR is trying to add, so the PR is not obsolete; however the branch is unmergeable against current main and has an unresolved P1 review concern that the proposed fail-closed runtime guard would regress trusted local CLI guild-admin actions that do not carry requester sender identity.

## What This Changes

The PR adds Discord guild-admin trusted-requester gating, threads `requesterSenderId` into guild-admin action dispatch as `senderUserId`, adds runtime Discord permission checks for privileged guild mutations, and expands Discord regression tests.

## Best Possible Solution

Land a narrow Discord-owned fix that requires trusted requester identity and matching Discord permissions for tool/requester-driven guild-admin mutations, while preserving or explicitly redesigning trusted local CLI/manual operator behavior, with regression coverage for both unauthorized guild members and owner CLI paths.
