## Summary

Keep open. Current main still lacks the Signal ReplyToId and outbound quote-parameter behavior this PR is trying to add, so it is not obsolete; however, the latest PR diff still has two merge-blocking P2 correctness gaps around Signal group quote visibility and group outbound quote-author propagation. The author’s latest mediaReadFile fix is present in the diff, but it does not address those remaining quote-context issues.

## What This Changes

The PR branch adds Signal inbound ReplyToId propagation, reaction-triggered heartbeat wakeups, outbound Signal quote-timestamp/quote-author support, first-chunk-only quote handling across Signal send paths, a focused heartbeat SDK export, and one status JSON test expectation update.

## Best Possible Solution

Update the branch so Signal quote sender resolution accepts authorNumber, then carry a verified quote author through the inbound reply payload and every Signal outbound send path that needs quote-timestamp/quote-author, while preserving visibility policy, remap guards, lazy send-runtime boundaries, and first-chunk-only behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is an allowlisted Signal group quote payload with id/text/authorNumber but no author/authorUuid; the PR type accepts authorNumber, but the visibility resolver ignores it, so ReplyToId can be dropped. A second path is a Signal group reply with replyToId only: group targets cannot infer quoteAuthor, so send.ts emits no quote fields.

## Solution Assessment

No. The feature direction is useful and current main does not implement it, but this PR is not yet the best mergeable fix because it forwards replyToId without the author metadata Signal needs for group quoted replies.
