## Summary

Keep open. Current main still appears to route ZAI image understanding through the configured provider base URL without a ZAI-specific rewrite from Coding Plan endpoints to the standard multimodal endpoints, and open PR #70915 already uses closing syntax for this issue, so this should remain open until that PR lands or is replaced.

## What This Changes

The issue requests fixing the built-in image tool so `agents.defaults.imageModel = "zai/glm-4.6v"` works with ZAI Coding Plan endpoints instead of returning 404.

## Best Possible Solution

Review and either land #70915 or replace it with an equivalent narrow ZAI-owned fix that routes only known ZAI vision models from Coding Plan base URLs to the matching standard multimodal endpoint, while keeping text/coding models on the configured Coding Plan endpoint and adding regression coverage for both global and CN routes.

## Reproduction Assessment

Yes. Maintainers with a ZAI key can configure `zai-coding-global` or `zai-coding-cn`, set `agents.defaults.imageModel` to `zai/glm-4.6v`, and call the `image` tool; this review confirms the current source path still leaves the Coding Plan base URL on the resolved image model.

## Solution Assessment

Yes for triage: keeping this issue open while #70915 is open is the best current action. The code fix should stay in the ZAI provider/media-understanding boundary or a narrow endpoint-routing seam, not a broad core migration.
