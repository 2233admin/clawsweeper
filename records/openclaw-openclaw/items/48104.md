## Summary

Keep open. Current main has adjacent execution-bias, strict-agentic plan-only retry, exec-approval denial, diagnostics, and trajectory-export work, but it still does not implement the central request: a first-class distinction between real SSH/tool failure, authorization or approval denial, and model-level soft refusal for explicitly authorized read-only operational diagnostics.

## Best Possible Solution

Keep this issue open for a core agent-runtime observability and policy design pass. The best path is to add first-class run metadata/events that classify tool execution failure, authorization/approval denial, and model/tool-routing soft refusal; refine runtime guidance for explicitly authorized read-only infrastructure diagnostics; and add regression coverage with a simulated provider response that refuses SSH/log inspection instead of using the available exec path.
