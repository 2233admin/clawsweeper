## Summary

Keep open. Current main partially supports recursive memory files through recursive collection, but nested dated memory still misses temporal decay, built-in dirty searches still sync in the background, QMD watcher paths still include glob patterns under chokidar 5, and user-facing copy still advertises flat `memory/*.md` files. The reporter also has paired open PR #72748 targeting the same remaining gaps, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Request recursive `memory/**/*.md` memory search support so markdown files in nested `memory/` subdirectories are indexed, watched, classified, and described correctly.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing #72748, or an equivalent focused memory-core PR, that recognizes nested dated memory paths for temporal decay, awaits dirty built-in search sync before querying, fixes QMD watcher root/pattern handling under chokidar 5, and updates remaining `memory/*.md` user-facing copy to recursive `memory/**/*.md`. The fix should include targeted tests for temporal decay, first-search freshness, watcher path/filter behavior, and QMD nested-file behavior.
