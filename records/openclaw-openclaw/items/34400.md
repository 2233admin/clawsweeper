## Summary

Keep open. Current main still only partially supports the requested recursive memory search behavior: recursive file collection exists, but nested dated memory paths still miss temporal decay, built-in dirty search sync remains fire-and-forget, QMD watcher paths still combine collection paths with glob patterns, and multiple user-facing strings still advertise flat `memory/*.md`. The reporter also has open PR #72748 targeting these same gaps, so closing now would be premature.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing #72748, or an equivalent focused memory-core PR, that fixes nested dated-path temporal decay, first-search freshness after dirty sync, QMD watcher root/pattern handling compatible with chokidar 5, and the remaining `memory/**/*.md` user-facing copy. The fix should include targeted tests around temporal decay, built-in async search sync, watcher path construction/filtering, and QMD nested-file behavior.
