## Summary

Keep this PR open. The underlying Feishu multi-account webhook bug still exists on current main, but this branch is not merge-ready because it is stale against current webhook hardening and would regress raw-body signature validation, pooled error propagation, rate limiting, and ambiguous route handling.

## What This Changes

The branch adds a Feishu webhook server pool keyed by host:port, with per-account routes, buffered token/app_id dispatch, request replay, and pooled cleanup state.

## Best Possible Solution

Implement the shared host:port server inside the Feishu plugin, but rebase it onto current main and preserve the raw-body signature contract, 64 KiB / 5 s pre-auth body budget, per-account throttling semantics, joined-account error propagation, unambiguous route matching, and focused multi-account webhook tests.

## Reproduction Assessment

Yes. Configure two enabled Feishu accounts in webhook mode with the same webhookHost:webhookPort; current main creates and listens a separate HTTP server for each account, so the second listener has a clear EADDRINUSE path.

## Solution Assessment

No. A Feishu-plugin-local pooled server is the right boundary, but this branch is not the best mergeable fix until it is rebased over current webhook security behavior and the pooled lifecycle, throttling, and route-disambiguation bugs are addressed.

## Security Review

Status: needs_attention

Summary: The diff changes unauthenticated Feishu webhook ingress and has concrete regressions around signature validation, request throttling, route isolation, and body-budget hardening.

Concerns:

- **[medium] Raw-body signature hardening can be lost:** `extensions/feishu/src/monitor.transport.ts:365`
  - body: The branch validates signatures using parsed JSON serialization and direct string equality, while current main validates the exact raw body with `safeEqualSecret`. The shared-server implementation must preserve the current raw-body contract.
  - confidence: 0.87
- **[medium] Unroutable pooled requests bypass throttling:** `extensions/feishu/src/monitor.transport.ts:428`
  - body: The pooled dispatcher buffers and parses before route resolution without using the existing rate limiter, and route misses return before per-account guards run. Invalid-token traffic can consume request resources without being counted.
  - confidence: 0.82
- **[medium] Ambiguous token-only routing can cross accounts:** `extensions/feishu/src/monitor.transport.ts:519`
  - body: The token-only fallback selects the first matching route when multiple accounts share a verification token, which can misdeliver webhook payloads across accounts when app_id is absent or ambiguous.
  - confidence: 0.8
- **[low] Stale body budget is broader than current main:** `extensions/feishu/src/monitor.state.ts:31`
  - body: The branch carries 1 MiB and 30 s webhook body limits, while current main uses 64 KiB and 5 s. A rebase should keep the tighter unauthenticated request budget unless maintainers intentionally change it.
  - confidence: 0.78
