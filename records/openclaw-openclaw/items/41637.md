## Summary

Keep this PR open. The underlying Feishu multi-account webhook bind bug still exists on current main, but this branch is stale and not merge-ready because it regresses current webhook signature and pre-auth body-limit hardening and has unresolved pooled-server lifecycle/error handling issues.

## What This Changes

The PR adds a Feishu webhook server pool keyed by host:port, per-account route registration, pooled request dispatch by token/app_id, request body replay, and pool-aware cleanup.

## Best Possible Solution

Land a refreshed Feishu plugin fix that shares one webhook listener per host:port while preserving current raw-body signature validation, timing-safe comparison, 64 KiB / 5 second pre-auth limits, per-account abuse controls, explicit joined-account error propagation, and focused tests for startup coalescing, dispatch, cleanup, malformed bodies, and server-error paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve raw-body signature verification:** `extensions/feishu/src/monitor.transport.ts:365-369`
  - body: The PR head validates Feishu signatures over `JSON.stringify` of the parsed body and compares with `===`. Current main signs the exact raw request body with a timing-safe comparison before JSON parsing, so this can reject valid signed requests whose JSON formatting differs and loses the current hardening.
  - confidence: 0.9
- **[P1] Keep the tightened pre-auth body limits:** `extensions/feishu/src/monitor.state.ts:31-32`
  - body: The PR head sets Feishu webhook limits to 1 MiB and 30 seconds, while current main uses 64 KiB and 5 seconds before authentication. The pooled path buffers unauthenticated bodies before it can route to an account, so relaxing these limits increases exposure and regresses existing security coverage.
  - confidence: 0.88
- **[P1] Propagate shared-server errors to joined accounts:** `extensions/feishu/src/monitor.transport.ts:282-295`
  - body: When the pooled server emits `error`, this handler clears state for every route but only rejects the creator account's monitor promise. Joined accounts only resolve on abort, so they can remain marked as running after their listener state has been torn down and no restart/backoff path is triggered.
  - confidence: 0.87
- **[P2] Return malformed pooled JSON as 400:** `extensions/feishu/src/monitor.transport.ts:473-477`
  - body: In multi-account mode, malformed JSON makes route resolution return no route and the dispatcher maps that to 404. The current account handler returns 400 Invalid JSON, so pooled mode changes webhook error semantics and hides malformed payloads as routing misses.
  - confidence: 0.82

## Security Review

  - confidence: 0.9

  - body: The PR head uses 1 MiB and 30 second pre-auth body limits while current main tightened Feishu webhook limits to 64 KiB and 5 seconds with regression coverage.
  - confidence: 0.88
