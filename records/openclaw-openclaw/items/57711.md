## Summary

Keep open. Current main still has the reported hybrid memory-search gap: provider-backed hybrid search sends the raw cleaned conversational query to FTS, while `buildFtsQuery` AND-joins every token. Keyword extraction exists and is used in the no-provider FTS-only fallback, but not before the hybrid keyword lookup. The PR discussion is substantive, and the author addressed bot review feedback, so this remains a plausible focused bug fix rather than conservative cleanup material.

## Best Possible Solution

Keep #57711 open as an actionable memory-core bug-fix candidate. The best path is to choose and land one bounded hybrid FTS strategy: either apply `extractKeywords` before provider-backed hybrid keyword search as this PR proposes, or adopt a per-keyword fallback like the related #39555 approach with explicit fan-out limits. In either case, add focused regression coverage for conversational queries in hybrid mode before closing either PR as resolved or superseded.
