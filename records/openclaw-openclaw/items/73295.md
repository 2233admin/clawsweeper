## Summary

Keep open. Current main still lacks this Discord-specific breaker, and this PR is the active successor to the earlier closed attempts, but the latest patch is not ready to merge because the recovery path can drain before the aborted run has actually cleared and the per-queue timer lifecycle can fire against a later run on the same queue key.

## What This Changes

The PR adds a Discord-only stuck-session circuit breaker for queued follow-ups, wires run-state and abort hooks through the reply runner, exports the new queue helper, and adds focused Vitest coverage.

## Best Possible Solution

Land a breaker only if it reuses or mirrors the existing abort-and-drain/force-clear recovery contract, ties timer lifetime to the current queued run instead of a stale queue-key entry, and covers non-draining abort plus same-queue reuse with focused tests before any live Discord restart window.

## Reproduction Assessment

Yes. The static reproduction path is high-confidence: queue a follow-up behind a non-streaming active Discord run, have abortActiveRun return before the session key clears, and the scheduled drain can collide with ReplyRunAlreadyActiveError; a second same-key run before the old timer fires exercises the stale timer bug.

## Solution Assessment

No, not as written. The feature direction is plausible, but the patch should wait for abort settlement or force-clear through the existing helper before draining, and should clear or replace stale timers when the same queue key is reused.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Replace stale breaker timers when the queue key is reused:** `src/auto-reply/reply/queue/circuit-breaker.ts:72-74`
  - body: The breaker keeps one timer per queueKey and returns when an old timer is still present, but normal follow-up drain completion does not clear that timer. A later Discord run on the same queue can inherit the old timer and be aborted before its own five-minute window, or fail to get a fresh breaker. Clear the timer when the queue drains or replace it for the current active run.
  - confidence: 0.82
- **[P2] Wait for abort to settle before draining recovery work:** `src/auto-reply/reply/queue/circuit-breaker.ts:112`
  - body: The breaker schedules the follow-up drain immediately after abortActiveRun returns true, but the current abort helper only signals cancellation and the session key remains active until the run actually clears. Recovery work can therefore hit ReplyRunAlreadyActiveError and keep the queue blocked; wait for drain/force-clear before scheduling recovery.
  - confidence: 0.87
