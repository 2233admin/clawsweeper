## Summary

Keep open. Current main still does not implement the Discord stuck-session circuit breaker proposed by this PR: follow-up messages can be enqueued behind an active run, but main has no Discord-specific delayed breaker, recovery notice, or non-streaming active-run abort for queued followups. The earlier related PRs #73233 and #73239 are closed unmerged, so this PR remains the active implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review. The likely path is to review the proposed Discord followup circuit breaker, require focused queue tests plus Discord-safe operational proof if maintainers want it, and decide whether the threshold/recovery notice belongs as fixed channel behavior or configurable policy.
