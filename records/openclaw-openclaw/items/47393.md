## Summary

Keep this PR open. Current main still lacks the PR’s intended Discord cross-account route-owner guard and still drops bot identity entirely when fetchUser("@me") fails, so the PR is not obsolete. The supplied PR discussion also identifies unresolved correctness issues that should be handled before merge.

## Best Possible Solution

Keep this PR open as a real Discord bug-fix candidate. The best path is to refresh it onto current main, implement route-owner resolution inside extensions/discord using plugin SDK routing exports, compare only active/configured Discord accounts, preserve botUserId with an applicationId fallback when fetchUser("@me") fails, decide or skip owner blocking for DMs, and add focused regression tests for cross-account channel ownership, explicit alternate-bot mentions, inactive account bindings, and identity-fetch failure.

## Risks / Open Questions

- Merging the supplied implementation as-is could silently drop direct messages in multi-account DM configurations because the owner check runs where explicit-mention bypass is unavailable.
- The supplied owner resolver considers account IDs from bindings, so stale or non-running accounts could suppress the live account that would otherwise respond.
- The supplied production diff should be refreshed to respect the bundled plugin boundary, using openclaw/plugin-sdk exports or an added SDK seam instead of direct extension imports from core src internals.
