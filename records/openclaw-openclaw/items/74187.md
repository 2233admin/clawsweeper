## Risks / Open Questions

- The submitted `chat.history` change appears to break the current test that non-default agent sessions inherit their owning agent's `thinkingDefault`.

- Changing Feishu `New session` from a confirmation-card flow to direct `/new` execution removes an intentional user confirmation step; that may be fine for `/new`, but it should be explicit maintainer/product review rather than an incidental latency fix.
- The PR overlaps open Feishu startup and WebSocket lifecycle work in #63257, #67466, and #73998, so maintainers should consolidate or sequence the fixes before landing this slice.
