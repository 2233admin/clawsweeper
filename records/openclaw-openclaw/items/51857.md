## Summary

Keep this issue open. Current main has real point fixes for image-model routing, Gemini usage mapping, and MS Teams Graph media diagnostics, but the issue still names the unresolved Moonshot/Kimi duplicate tool-call-id failure in WhatsApp group chats, and current code still lacks a canonical cross-channel content-verification contract for inbound media, model fallback diagnostics, and missing-vs-zero provider usage semantics.

## Best Possible Solution

Keep this issue open but narrow it. Treat the imageModel, Gemini usage, and MS Teams Graph media items as progress; leave #51593 as the concrete Moonshot/WhatsApp tracker; and define a specific cross-channel verification contract: expected media fetches should produce non-empty inspectable payloads or structured diagnostics, image/PDF routing should log fallback decisions, provider usage should distinguish missing metadata from true zero usage, and each channel/provider path should have targeted regression coverage.
