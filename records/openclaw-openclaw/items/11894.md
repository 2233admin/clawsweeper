## Summary

Keep this issue open. Current main has transcript-backed token/cost summaries and auth-profile rotation/fallback routing, but completed model-call usage is still parsed and aggregated by provider/model/session, not by the resolved auth profile that paid for each call. The burn0/#55379 discussion is a useful interim per-request cost workaround, but it does not replace native OpenClaw auth-profile attribution.

## Best Possible Solution

Keep this issue open until OpenClaw records the resolved authProfileId for each completed model call in safe metadata, exposes it through transcript/usage/status or diagnostics, aggregates token/cost totals by auth profile, and emits a clear profile rotation/fallback event. burn0/#55379 can remain an external cost workaround, but it is not a native replacement for OpenClaw auth-profile attribution.
