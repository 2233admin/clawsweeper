## Summary

Keep open. Current main still lacks the intermediate task states and lifecycle formatting, and this PR remains the active implementation candidate, but the latest patch is not mergeable as-is because the new statuses are not accepted by the task status normalizer used by create/update paths.

## What This Changes

The PR adds `awaiting_approval` and `waiting_external` task statuses, propagates active-state handling through task registry, Task Flow, audit, CLI, and auto-reply surfaces, and adds shared lifecycle/operational formatting with focused tests.

## Best Possible Solution

Make the intermediate statuses first-class everywhere status values are parsed, normalized, displayed, audited, and documented while preserving the Task Flow projection compatibility, then land this PR or an equivalent patch after focused task registry/status tests and the changed gate pass.

## Reproduction Assessment

Yes. On PR head, inspect or exercise `createTaskRecord`/status update paths with `awaiting_approval` or `waiting_external`; they pass through `normalizeTaskStatus`, which still only accepts the old status set and maps the new values to `queued`.

## Solution Assessment

No, not in the current form. The overall direction is maintainable, but the implementation must update status normalization and user-facing docs before it is the narrowest safe solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the new statuses in normalization:** `src/tasks/task-registry.types.ts:7-8`
  - body: Adding `awaiting_approval` and `waiting_external` to `TaskStatus` is not enough because `normalizeTaskStatus` in `task-registry.ts` still accepts only the old values and falls back to `queued`. Any create/update path that receives one of the new statuses through params will silently store `queued`, and the new tests that seed `awaiting_approval` directly through `createTaskRecord` will not exercise the state they expect. Add both statuses to the normalizer or centralize normalization on the canonical status set.
  - confidence: 0.95
