## Summary

Keep open. Current main mitigates the misleading dispatcher timeout wording, but the upload-with-trigger path remains stale-prone: `/hooks/file-chooser` still arms the file upload and then clicks the original `ref` without re-resolution and without forwarding timeout or abort context into that trigger click.

## What This Changes

Make browser upload/file-chooser flows robust by using an atomic upload-and-trigger path and preserving stale/action-specific errors instead of browser-control reachability timeout wording.

## Best Possible Solution

Land a focused browser plugin fix for upload-with-trigger: make `/hooks/file-chooser` with `ref` behave as one robust request, propagate timeout and AbortSignal context into trigger activation, re-snapshot or re-resolve stale trigger refs where current metadata supports it, preserve stale/action-specific errors, and add regression tests. Keep #51395 separate because it covers non-standard file-input fallbacks.
