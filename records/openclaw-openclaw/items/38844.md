## Summary

Keep open. Current main has mitigations for this report, including upload-with-ref/inputRef/element support, selector/batch action support, and dispatcher timeout wording that preserves local action context. The remaining file-chooser bug is still present: the managed `/hooks/file-chooser` path arms upload and then clicks the same stale-prone `ref` without internal re-snapshot/re-resolution and without passing the route timeout or abort signal through the trigger click.

## Best Possible Solution

Keep this issue open and land a focused browser plugin fix. The best path is to make upload-with-trigger robust as one request: arm upload and activate the trigger together, propagate route timeout/AbortSignal context into trigger activation, re-snapshot or re-resolve stale trigger refs where the current ref metadata supports it, preserve stale-ref/action errors, add focused regression tests, and update browser docs/tool guidance. Keep the separate non-standard file-input fallback in #51395 out of this change.
