## Summary

Keep open. Current main does not contain the PR's reduced pricing-fetch timeout or env timeout override, but the branch is not ready to merge because main has since added an overlapping documented pricing opt-out and a timeout-warning test that still expects the old 60s value.

## What This Changes

The PR lowers the Gateway model-pricing catalog fetch timeout to 15s with an env override and adds `OPENCLAW_DISABLE_PRICING_REFRESH=1` to skip scheduling the remote pricing refresh.

## Best Possible Solution

If the reduced timeout and env-level override are still desired, rebase onto current main, keep `models.pricing.enabled` as the documented config path, update the pricing tests and docs/changelog for the new env knobs, and tighten invalid env parsing before merge.

## Reproduction Assessment

Yes for the merge-readiness issue: current main has a test that simulates pricing fetch timeouts and asserts `timeout 60s`, while the PR changes the default timeout used in that log path to 15s. The original Windows TUI `gateway closed (1000)` report was not live-reproduced here and was already closed as a duplicate/fixed startup-path issue.

## Solution Assessment

Unclear. The shorter timeout is a narrow QoL improvement, but the offline opt-out overlaps with the current documented `models.pricing.enabled` config surface, so maintainer product judgment is needed before carrying both knobs forward.
