## Summary

Keep open. Current main still renders the live markdown text directly through markdown-it without the unclosed-fence preprocessing proposed by #73485, and the linked PR is closed rather than merged.

## Best Possible Solution

Keep this issue open. Restore or reimplement the narrow #73485 approach in `ui/src/ui/markdown.ts`, but address the review edge cases for indented fence openers and streams ending on a fence opener, then add colocated regression tests in `ui/src/ui/markdown.test.ts` proving streamed/unclosed fenced code preserves leading indentation before the final history refresh.
