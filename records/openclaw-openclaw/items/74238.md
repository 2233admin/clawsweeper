## Summary

Keep open, but do not merge as-is: current main still has the duplicated Control UI page-header composition this PR targets, while the PR is broad, currently dirty against main, and removes useful per-section context such as the sessions store path and Debug card labels.

## What This Changes

The PR moves tab subtitles into the Control UI dashboard breadcrumb, removes non-chat page and card header blocks across several UI views, adjusts Config/Usage header surfaces, adds small breadcrumb CSS, and updates one cron view test.

## Best Possible Solution

Rebase and narrow the cleanup so only duplicate page-level title/subtitle surfaces move into the shared breadcrumb, while dynamic data and meaningful section/card labels remain visible; then add or update focused UI render checks and perform desktop/mobile Control UI smoke for Sessions, Usage, Skills, Logs, Config, Debug, and Chat.

## Reproduction Assessment

Yes. Source inspection on current main shows the topbar dashboard header plus a separate content-header/page-sub path, and the PR diff shows the proposed removal path; final merge still needs browser visual smoke because this is a UI layout change.

## Solution Assessment

No, not as submitted. Consolidating page-level headers is a reasonable direction, but this patch also removes non-duplicative context and is dirty against current main; a smaller rebase preserving dynamic/session and section labels is safer.
