## Summary

Keep this PR open. Current main still has the documented Telegram SOCKS5 media-download gap, and this branch is the active implementation candidate, but the patch does not yet fix the guarded media path and overclaims SOCKS schemes that undici 8.1.0 rejects.

## What This Changes

The PR branch adds SOCKS scheme detection to the shared proxy fetch helper and Telegram explicit-proxy dispatcher, imports undici Socks5ProxyAgent, and adds unit coverage for SOCKS proxy fetch routing and metadata reuse.

## Best Possible Solution

Rework the patch around the current main architecture: support dependency-backed `socks5://` and `socks://` for the documented Telegram explicit-proxy media path by updating the guarded explicit-proxy policy/runtime dispatcher, keep proxy-host validation and Telegram target allowlisting intact, preserve Telegram HTTP/1 pool and TLS lookup behavior, and remove unsupported SOCKS4/SOCKS5H claims unless separately implemented.

## Review Findings

Overall correctness: patch is incorrect

  - confidence: 0.9
