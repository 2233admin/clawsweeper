## Summary

Keep open. Current main still derives Mattermost native slash callback URLs as plain HTTP when no explicit callback URL is configured, still registers after only warning on loopback HTTP callbacks, and still authorizes callbacks by account-level token membership before trusting the caller-supplied slash trigger. The issue is also paired with open, unmerged closing PR #65655, so it should stay open until that PR or an equivalent maintainer-approved fix lands.

## What This Changes

Mattermost native slash commands should fail closed on non-HTTPS callback URLs and bind each Mattermost-issued command token to the exact registered slash trigger.

## Best Possible Solution

Ship a Mattermost plugin fix that rejects non-HTTPS resolved native slash callback URLs before command registration, carries token-to-trigger bindings from registered commands through slash state into HTTP dispatch, documents the HTTPS callback requirement, and adds regression tests for both HTTPS enforcement and cross-trigger token replay.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: omit `commands.callbackUrl` to get a derived `http://...` callback, then send the HTTP handler any registered token with a different `payload.command`; the code checks token membership before using the supplied trigger.

## Solution Assessment

Yes. Enforcing HTTPS at Mattermost slash registration and binding tokens to triggers inside the Mattermost plugin is the narrowest maintainable fix; it does not require a core gateway or plugin API change.

## Security Review

Status: needs_attention

Summary: The issue describes a concrete security-sensitive Mattermost slash callback weakness that is still present on current main.

Concerns:

- **[high] Reject cleartext derived slash callback URLs:** `extensions/mattermost/src/mattermost/slash-commands.ts:592`
  - body: When no explicit callback URL is configured, current main derives an `http://` Mattermost slash callback and proceeds to registration. That can transport reusable command tokens in cleartext on the Mattermost-to-gateway path.
  - confidence: 0.96
- **[high] Bind command tokens to their registered triggers:** `extensions/mattermost/src/mattermost/slash-http.ts:263`
  - body: The callback path accepts any registered token for the account, then trusts `payload.command` to choose the trigger. A captured token for one registered slash command can therefore be replayed against a different trigger unless token-to-trigger binding is added.
  - confidence: 0.94
