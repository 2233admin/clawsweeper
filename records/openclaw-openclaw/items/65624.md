## Summary

Keep this issue open. Current `origin/main` at `213f92a9ef0eb54972ef6cbb0e8b900adc8699b0` still derives plain-HTTP Mattermost slash callback URLs when `commands.callbackUrl` is omitted, still registers those callbacks after a warning rather than failing closed on HTTPS, and still validates callbacks by token-set membership rather than token-to-trigger binding. The issue also has an open, unmerged implementation candidate, #65655, with closing linkage, so the best cleanup path is to review/land or explicitly reject that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and review #65655, or land an equivalent maintainer-approved Mattermost plugin fix. The fix should reject non-HTTPS resolved native slash callback URLs before registration, preserve token-to-trigger bindings through slash activation and HTTP dispatch, update the Mattermost docs for the HTTPS callback requirement, and add focused regression tests for both HTTPS enforcement and cross-trigger token replay.
