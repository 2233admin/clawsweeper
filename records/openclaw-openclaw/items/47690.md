## Summary

Keep open. Current main still routes each Telegram inbound message to one route for the receiving account, drops named-account group fallback without an explicit binding, and has no `forwardToAccounts`, `broadcastToAccounts`, or equivalent Telegram fan-out config. The stale label is not enough to close this because the requested cross-account routing behavior remains a concrete product gap.

## What This Changes

The issue requests opt-in Telegram multi-account routing so a group message received through one bot account can be forwarded or broadcast to other configured bot accounts or agents while preserving separate bot identities.

## Best Possible Solution

Design an additive, opt-in Telegram cross-account or broadcast routing model with explicit destination account authorization, identity preservation, loop prevention, bot-authored message handling, docs, and focused multi-account group coverage; if maintainers decide not to support this pattern, document the supported bridge/workaround clearly.

Status: needs_attention

Summary: No patch was reviewed, but the requested feature is security-sensitive because it would let one Telegram account cause other accounts or agents to process group traffic.
