## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior for providers that return valid all-zero usage; the current token-usage pipeline continues to distinguish usage by nonzero counters, so zero-valued usage is treated like missing usage before it can become fresh status data.

## Best Possible Solution

Keep the PR open for author or maintainer follow-up, or replace it with a complete current-main fix. The implementation should track whether usage was observed independently from whether counters are nonzero, return/persist zero-valued totals when appropriate, keep missing-usage stale fallback behavior intact, and add focused regression coverage through the subscription, runner metadata, session store, and status surfaces.
