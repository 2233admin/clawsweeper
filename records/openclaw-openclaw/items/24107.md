## Summary

Keep open. Current main has shorter reactive cooldowns and provider usage-window reporting, but it still lacks the requested generic RPM/RPD quota contract, persisted quota counters, and pre-request quota-aware fallback skipping.

## What This Changes

The issue requests generic per-provider RPM/RPD quota configuration and quota-aware model fallback cooldowns that avoid overlong reactive backoff after rate-limit windows reset.

## Best Possible Solution

Define a generic quota contract, persist minute/day quota state with exact reset-time calculation, integrate pre-request fallback skipping with cooldownUntil under the existing auth-profile store locks, and update config schema/help/docs plus focused auth-profile and model-fallback tests.
