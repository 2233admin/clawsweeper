## Summary

Keep open. Current main still implements reactive auth-profile cooldowns and model fallback, but not the requested generic provider-level RPM/RPD quota contract, persisted minute/day quota counters, or pre-request quota skip that computes exact cooldown windows.

## Best Possible Solution

Keep this open as a core model fallback/auth-profile enhancement. The best path is maintainer agreement on a provider quota contract, then implementation of persisted quota window state, exact reset-time calculation, pre-request skip that writes cooldownUntil, schema/help/docs updates, and focused tests for RPM and RPD windows alongside existing cooldown/probe coverage.
