## Summary

Current main still has no generic RPM/RPD provider quota contract: provider config/schema do not accept quota fields, auth-profile usage state only tracks cooldown/failure fields, and fallback preflight only reacts after every profile is already in cooldown. The existing WHAM reset probe is scoped to `openai-codex`, and the related usage-aware routing issue remains open rather than an agreed replacement, so this should stay open for maintainer API/product review.

## What This Changes

The issue requests per-provider RPM/RPD quota settings that compute exact cooldownUntil windows and pre-request fallback skipping instead of relying on generic reactive cooldown backoff.

## Best Possible Solution

Define a generic quota contract, persist minute/day quota state with exact reset-time calculation, integrate pre-request fallback skipping with cooldownUntil under the existing auth-profile store lock, and update config schema/help/docs plus focused auth-profile and model-fallback tests.
