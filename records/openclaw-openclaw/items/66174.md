## Summary

Keep open. Current main still lacks the requested save-time rewrite, and this PR is useful source work, but its implementation reuses a compatibility resolver that can preserve structurally valid cross-root session paths, so it does not fully fix the reported persistence bug.

## What This Changes

The PR adds a save-time session-store normalization helper, first-class `transcriptPath` on `SessionEntry`, and an integration test for rewriting persisted transcript metadata into the active sessions directory.

## Best Possible Solution

Land a corrected session-store fix that computes persisted metadata paths from the active store directory and validated session IDs, updates both `sessionFile` and the legacy `transcriptPath` alias when present, preserves archive/deletion safeguards, and covers structurally valid cross-root paths in regression tests.

## Review Findings

Overall correctness: patch is incorrect

- **[P1] Force save canonicalization to the active sessions dir:** `src/config/sessions/store.ts:109`
