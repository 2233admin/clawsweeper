## Summary

Keep this PR open. Current main still lacks save-time canonicalization for persisted session transcript paths, and the PR is a focused implementation candidate, but the proposed helper uses the compatibility resolver that can preserve structurally valid cross-root paths instead of forcing the active store's sessions directory.

## What This Changes

The PR adds a save-time session-store helper that rewrites persisted `sessionFile` and `transcriptPath` values, adds `transcriptPath` to `SessionEntry`, and covers a simple outside-path case in the pruning integration tests.

## Best Possible Solution

Revise the PR so save-time normalization derives the target from `path.dirname(path.resolve(storePath))` plus a validated session id, persists both `sessionFile` and the legacy `transcriptPath` alias when present, adds a regression case for structurally valid cross-root paths, and leaves the existing external-transcript deletion safeguards intact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Force save canonicalization to the active sessions directory:** `src/config/sessions/store.ts:109`
  - body: `resolveSessionFilePath` is a compatibility resolver, not a force-current-dir canonicalizer: it can return structurally valid `.../agents/<agentId>/sessions/<file>` paths from another root unchanged. A restored entry with that shape will survive this save pass and can still be treated as missing/orphaned, so derive the save target from `path.dirname(storePath)` plus a validated `sessionId` instead.
  - confidence: 0.94
