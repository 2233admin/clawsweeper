## Summary

Keep this issue open. Current main still writes macOS launchd Gateway stdout/stderr to fixed `gateway.log` and `gateway.err.log` paths through `StandardOutPath` / `StandardErrorPath`, with no OpenClaw-managed size cap or rotation for those supervisor files. The structured JSONL logger has `logging.maxFileBytes` rotation, but that is a separate log surface and does not bound launchd stderr growth.

## Best Possible Solution

Add bounded retention for macOS launchd supervisor logs (`gateway.log` and `gateway.err.log`) or route launchd stdout/stderr through an OpenClaw-managed rotating sink. Cover install and restart plist rewrite paths with targeted tests, document the retention behavior, and separately trace/rate-limit the heartbeat-timeout producer once identified.
