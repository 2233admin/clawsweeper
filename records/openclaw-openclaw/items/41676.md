## Summary

Keep this issue open. Current main rotates the structured Gateway JSONL logger, but the reported macOS launchd supervisor stderr file still resolves to a fixed `gateway.err.log` path and is written through `StandardErrorPath` with no OpenClaw-managed size cap, truncation, or rotation policy.

## Best Possible Solution

Add bounded retention for macOS launchd supervisor logs (`gateway.log` and `gateway.err.log`) or route launchd stdout/stderr through an OpenClaw-managed rotating sink. Keep the existing structured logger rotation, add targeted launchd log cap/truncation tests and docs, and separately rate-limit or summarize the heartbeat-timeout producer once it is identified.
