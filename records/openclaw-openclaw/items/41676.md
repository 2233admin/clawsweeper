## Summary

Keep this issue open. Current main rotates the structured Gateway file logger, but the reported macOS launchd stderr file is still a fixed `gateway.err.log` path wired through `StandardErrorPath` with no OpenClaw-managed max-size, truncation, or rotation policy on that supervisor-log surface.

## Best Possible Solution

Add bounded retention for macOS launchd supervisor logs (`gateway.log` and `gateway.err.log`) or route launchd stdout/stderr through an OpenClaw-managed rotating sink. Keep structured file logger rotation as-is, add targeted launchd log cap/truncation tests and docs, and separately rate-limit or summarize the heartbeat-timeout producer once it is identified.
