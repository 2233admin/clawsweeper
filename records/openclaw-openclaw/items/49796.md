## Summary

Keep this PR open. Current main partially mitigates the doctor orphan warning by comparing transcript realpaths, but it still does not implement the PR's core behavior: preserving and persisting caller-visible logical session paths when the sessions directory is reached through a symlink.

## Best Possible Solution

Keep the PR open and review/adapt it against current main. The likely best fix is still in src/config/sessions/paths.ts: use realpaths only for containment checks, return logical paths under the caller-visible sessions directory, persist those logical paths through resolveAndPersistSessionFile, and keep/extend regression coverage for symlinked sessions dirs, legacy real-path remapping, sibling fallback, and doctor orphan detection.
