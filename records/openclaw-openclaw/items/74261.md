## Summary

Keep open. Current main still does not expose an agent-callable WhatsApp group creation path: the bundled WhatsApp plugin registers only the QR login tool as a channel-owned agent tool, the message/action surface covers reactions and polls rather than group creation, and the active linked-session API has no group-create wrapper. The request is plausible, but inviting phone-number participants through a linked WhatsApp identity is privacy/security-sensitive and needs maintainer product/security review before implementation.

## What This Changes

Add a WhatsApp-owned agent tool named `whatsapp_create_group` that accepts a group name and participant phone numbers and creates a WhatsApp group for an agent workflow.

## Best Possible Solution

If maintainers accept the product/security tradeoff, implement this inside the WhatsApp-owned plugin boundary or first add a narrow documented runtime seam. The end state should gate the tool appropriately, normalize participant phone numbers to WhatsApp JIDs, call the active linked session through a typed wrapper, redact identifiers in logs, record outbound activity, document the behavior, and cover the tool/runtime path with focused WhatsApp tests.
