## Summary

Keep open. Current main still lacks a WhatsApp group-creation agent tool or safe runtime wrapper, and the existing discussion correctly treats agent-initiated group creation as plausible but security-sensitive rather than ready for automated cleanup.

## What This Changes

The issue requests a new WhatsApp agent tool that creates a WhatsApp group from a supplied group name and participant phone numbers.

## Best Possible Solution

If maintainers accept the product and security tradeoff, implement this in the WhatsApp-owned plugin behind an owner-only or equivalent gate, normalize phone numbers to WhatsApp JIDs, call Baileys through the active listener/send API boundary, redact participant identifiers in logs, record outbound activity, add focused tests, and update the WhatsApp docs. If the runtime surface should be reusable by third-party plugins, add a narrow documented WhatsApp runtime API first.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but yes for verifying the missing feature: inspect the WhatsApp channel registration, active listener, send API, runtime API, docs, and latest release tag and observe that none expose group creation.

## Solution Assessment

Is this the best way to solve the issue? Unclear pending maintainer approval; the proposed Baileys `groupCreate` direction is technically plausible, but the maintainable solution must first settle authorization, logging/audit, and whether the surface is internal to the WhatsApp plugin or a documented runtime helper.
