## Summary

Keep open. Current main still lacks a WhatsApp group-creation tool or supported active-session API, and this should not be closed as ClawHub-only because the existing WhatsApp runtime surface does not expose group creation on the linked Gateway-owned socket.

## What This Changes

Add a WhatsApp agent tool that creates a group from a group name and participant phone-number list using the linked WhatsApp session.

## Best Possible Solution

If maintainers approve the capability, implement it inside the WhatsApp plugin as a gated owner/trusted-requester tool backed by a narrow active-listener method, with E.164-to-JID normalization, redacted logs, outbound activity tracking, focused WhatsApp tests, and docs. If maintainers reject agent-created group invites, document the limitation in the WhatsApp channel docs.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence: exact symbol searches plus source reads show no `whatsapp_create_group` tool, no active-listener group method, and no documented create-group surface.

## Solution Assessment

Unclear until maintainer product/security review. If approved, a WhatsApp-owned gated runtime/tool change is the narrow maintainable path; a separate ClawHub plugin cannot currently solve this through the supported active-listener API.
