## Summary

Keep open. Current main does not expose a WhatsApp group-creation agent tool or message action: the bundled WhatsApp plugin currently registers only `whatsapp_login` as a channel agent tool, advertises WhatsApp message actions for reactions and polls, and the active listener IPC surface has send/poll/reaction/typing but no group-create operation. The request is still potentially valid because Baileys exposes `groupCreate(subject, participants)`, but adding an AI-callable group creator that invites phone numbers needs maintainer product/security review and a clear owner-only/policy boundary. The issue body also mixes the group-creation request with unrelated `responsePrefix` examples, so scope should be clarified before implementation.

## Best Possible Solution

Keep this open for maintainer review. If accepted, the best implementation likely belongs in the bundled WhatsApp plugin as a tightly gated owner-only tool or channel action that uses the existing linked account, validates/normalizes participant JIDs, records outbound activity, redacts identifiers in logs, and documents the security boundary. If maintainers prefer this outside core, first add a narrow WhatsApp/plugin API seam that lets an external ClawHub plugin use the existing linked WhatsApp session safely.
