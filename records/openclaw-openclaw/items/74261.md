## Summary

Keep open. Current main still has no WhatsApp group-creation agent tool, message action, active-listener method, or send/runtime wrapper; Baileys does expose `groupCreate`, but OpenClaw does not currently expose group management through the linked WhatsApp session. The newest discussion adds credible operational/security concern around automated repeated group creation, so this needs maintainer product/security review before any implementation.

## What This Changes

Request to add a `whatsapp_create_group` WhatsApp agent tool accepting `groupName` and participant phone numbers, backed by Baileys `groupCreate`, so an agent can create listing-specific WhatsApp groups.

## Best Possible Solution

If maintainers approve the capability, implement it inside the WhatsApp plugin as a narrowly gated owner/trusted-requester operation: normalize participant phone numbers to WhatsApp JIDs, expose a small runtime/listener method around Baileys `groupCreate`, redact identifiers in logs, record outbound activity, add focused WhatsApp tests, and document the risk and limitations. If maintainers reject it, document that OpenClaw intentionally does not automate WhatsApp group creation.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence: source, docs, and API searches plus direct inspection show login/send/poll/reaction/typing support only, with no group-create surface.

## Solution Assessment

Unclear until maintainer product/security review. Calling Baileys directly from an ungated agent tool is not the safest shape; the maintainable path would be a WhatsApp-owned, policy-gated runtime surface rather than a generic core helper or ClawHub-only plugin.
