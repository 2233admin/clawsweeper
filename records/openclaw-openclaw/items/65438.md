## Summary

Keep open. Current main still lacks configurable bootstrap dynamic-file classification and configurable bootstrap file ordering; the relevant prompt assembly, config schema, and docs only expose the existing hardcoded order, `HEARTBEAT.md` dynamic split, context-injection mode, and bootstrap size limits. Related open issues #59050 and #67419 overlap, but neither supersedes the dynamic-cache-boundary request.

## What This Changes

The issue asks for config-driven bootstrap file ordering and dynamic-file placement below the system-prompt cache boundary to keep volatile workspace files from invalidating the stable prompt prefix.

## Best Possible Solution

Add a documented, backwards-compatible bootstrap prompt configuration surface that preserves the existing default order and `HEARTBEAT.md` dynamic behavior, lets operators opt additional basenames into the dynamic group below `OPENCLAW_CACHE_BOUNDARY`, and optionally lets them override ordering while keeping deterministic fallback sorting for unlisted files.
