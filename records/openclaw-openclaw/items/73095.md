## Summary

Keep open. Current main still has the source-level resolve-before-init path for native `sessions_spawn`, and the provided GitHub context shows open PR #73904 is the paired closing fix candidate rather than a merged fix. Item URL: https://github.com/openclaw/openclaw/issues/73095

## What This Changes

Initialize built-in context engines before native subagent spawn preparation resolves the configured context engine on cliBackend-only cold starts.

## Best Possible Solution

Review and land PR #73904, or an equivalent narrow change, so native `sessions_spawn` initializes built-in context engines before resolving the configured engine; keep the separate cli-compaction pattern in #75129 on its own review track.

## Reproduction Assessment

Yes. A cold process can enter `sessions_spawn` -> `spawnSubagentDirect()` -> `prepareContextEngineSubagentSpawn()` and resolve the default `legacy` context engine before `ensureContextEnginesInitialized()` registers it, which matches the reported error and the independent reproduction comments.

## Solution Assessment

Yes. Initializing via `ensureContextEnginesInitialized()` at the spawn preparation boundary is the narrowest maintainable fix for this issue; moving legacy registration to a top-level side effect would be broader and riskier.
