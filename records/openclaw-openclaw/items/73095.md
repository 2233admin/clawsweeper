## Summary

Keep open. Current main still has the reported resolve-before-init path in native subagent spawn preparation, and open PR #73904 uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that cliBackend-only cold starts can fail `sessions_spawn` because `prepareContextEngineSubagentSpawn()` resolves the default context engine before registering built-in engines.

## Best Possible Solution

Land #73904 or an equivalent narrow change that initializes built-in context engines at the native subagent spawn preparation boundary before resolving the context engine; review #75129 separately for the related cli-compaction site.

## Reproduction Assessment

Yes. A cold cliBackend-only process can reach `sessions_spawn` through `spawnSubagentDirect()` without PI runner initialization; current main then resolves the default `legacy` context engine before registration and hits the explicit unregistered-engine throw path.

## Solution Assessment

Yes. Initializing with `ensureContextEnginesInitialized()` at the spawn-side resolution boundary is the narrow maintainable fix for this report; making registration a top-level import side effect would be broader and riskier.
