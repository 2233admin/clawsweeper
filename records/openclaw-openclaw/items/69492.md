## Summary

Keep open. Current main still has no SystemEvent audience/classification primitive; system events are still shaped as text/timestamp/context/delivery/trust and exec/heartbeat paths classify with routing flags and text-pattern filters. The related implementation PR is open and explicitly partial, with separate open tactical heartbeat PRs, so this still needs maintainer design follow-up rather than cleanup closure.

## What This Changes

Request to add a per-system-event audience/classification primitive so internal exec, cron, and heartbeat events do not leak through user-facing consumer paths.

## Best Possible Solution

Keep this open as the canonical maintainer discussion. Review #72201 together with the tactical heartbeat PRs #72218 and #72253, decide whether OpenClaw wants a SystemEvent audience primitive or only case-by-case producer/consumer suppression, then close this after the accepted design lands or after the remaining work is superseded by narrower canonical issues.
