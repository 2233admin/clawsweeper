## Summary

Keep this PR open. Current main still does not implement a durable observable `session-state.json` snapshot, and the live PR is an unmerged focused implementation candidate. The diff is not obsolete or duplicated, but it still needs maintainer/product review and contributor revision around sidecar write isolation, supported custom/shared store layouts, and the new durable metadata artifact.

## Best Possible Solution

Keep this PR open for maintainer judgment and contributor revision. If OpenClaw wants a core filesystem snapshot, make the snapshot write best-effort, derive per-entry agent identity from canonical session keys, choose collision-free snapshot paths for custom and shared store layouts, add tests for those supported layouts, and document the new artifact. If maintainers prefer existing Gateway/CLI/Control UI inspection instead of a durable sidecar file, close the PR with that explicit product rationale.

## Risks / Open Questions

- The current PR diff can make a successful primary `sessions.json` write appear failed if the secondary `session-state.json` write throws after `writeTextAtomic` completes.
- The current PR path strategy still risks collisions for supported custom store templates that put multiple store files in one directory, and it derives every entry `agentId` from the store path rather than the canonical session key for shared stores.
- The PR creates a new durable copy of session metadata, including human-readable session fields and spawned workspace paths; permissions are restrictive, but artifact documentation, retention, and product expectations are not settled.
