## Summary

Keep this PR open. Current main still has no equivalent observable session snapshot, but the current PR head is still the two-commit `a6d5c1e58d86` version with concrete sidecar write, path-collision, and shared-store attribution defects that should block merge until revised.

## What This Changes

The PR adds `src/config/sessions/observable-state.ts`, tests for the snapshot, and a `writeSessionStoreAtomic` hook that writes a `session-state.json` sidecar from persisted session-store data.

## Best Possible Solution

If OpenClaw wants a filesystem snapshot, keep it best-effort, collision-free for every supported store path, derive per-entry agent identity from canonical session keys, and document the artifact's privacy and retention expectations. If maintainers prefer existing Gateway, CLI, or Control UI inspection surfaces, close the PR with that product rationale.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Isolate snapshot write failures from store saves:** `src/config/sessions/store.ts:540-543`
  - body: `saveObservableSessionState` runs after `sessions.json` has already been written and caches updated, but it writes through `saveJsonFile`, which can throw on filesystem errors. A failed sidecar write would make callers treat an already-committed primary session-store save as failed, so this auxiliary write needs to be best-effort or otherwise isolated.
  - confidence: 0.92
- **[P2] Scope sidecar names to each source store file:** `src/config/sessions/observable-state.ts:41-42`
  - body: `resolveObservableSessionStatePathForStore` always writes `session-state.json` in the parent directory. Supported templates such as `/tmp/sessions-{agentId}.json` put multiple store files in the same directory, so different agents would overwrite one another's observable snapshots.
  - confidence: 0.88
- **[P2] Derive each entry agent from its session key:** `src/config/sessions/observable-state.ts:77`
  - body: The snapshot passes one `agentId` from the store path into every entry. Shared stores are supported when `session.store` omits `{agentId}`, so entries like `agent:ops:*` in a shared file would be emitted as `agentId: "main"`, breaking filters and attribution.
  - confidence: 0.9
