## Summary

Keep this PR open. Current main still does not implement the PR’s intended Python and shell coverage for the built-in skill scanner: the scanner remains JS/TS-extension-only, Python/shell rule IDs are absent, and directory plus explicit include scanning still skip non-scannable extensions. This is core security-gate work rather than ClawHub-only work, but the PR should be revised before landing because the supplied review context still has unresolved scanner-quality concerns around extensionless shell entrypoints and non-literal Python command execution.

## Best Possible Solution

Keep this PR open for revision, or replace it with an equivalent focused core scanner-hardening patch. The best path is to extend scannable Python and shell coverage, explicitly decide the extensionless shebang/script boundary, add include/exclude rule guards without regressing JS/TS behavior, fix the remaining Python command-execution false negatives, and land regression tests for both malicious-pattern coverage and false-positive control.
