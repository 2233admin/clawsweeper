## Summary

Keep open. Current main still does not implement the PR's intended Python and shell coverage for the built-in skill/plugin code scanner. The requested work belongs in OpenClaw core because the scanner is used by install and deep security-audit paths, and the PR still has unresolved detection-quality review comments that need revision rather than cleanup closure.

## Best Possible Solution

Keep this PR open for revision, or replace it with an equivalent focused core scanner-hardening patch. The best path is to extend scannable Python/shell coverage, decide and test the extensionless shebang boundary, add include/exclude rule guards without regressing JS/TS behavior, fix the remaining Python command-exec false negatives, and land regression tests for both the earlier false-positive fixes and the later review comments.
