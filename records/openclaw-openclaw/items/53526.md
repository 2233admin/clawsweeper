## Summary

Keep open. Current main still defaults session reset policy to `daily` for group/thread sessions when no explicit per-type policy exists, while this PR is trying to change that behavior. The PR is still useful but not ready as-is: the provided review context flags a real guard bug in the patch, and current main has moved the implementation into `src/config/sessions/reset-policy.ts`, not the older wrapper file shown in the PR diff.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main: implement the type-aware default in `src/config/sessions/reset-policy.ts`, scope the explicit-policy guard to the current reset type, add exact tests for no-config group/thread defaults and unrelated `dm`/`direct` overrides, and update user-facing session docs/changelog if maintainers accept the new default.
