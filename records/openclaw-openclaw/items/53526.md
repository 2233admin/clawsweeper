## Summary

Keep open. The requested behavior is not implemented on current main, and the PR is not merge-ready because the branch's default-selection guard still leaves group/thread sessions on daily reset when an unrelated direct/DM override exists. The change also alters default context retention in shared channels, so it needs explicit maintainer product/privacy approval before a refreshed fix is pursued.

## What This Changes

This PR changes implicit session reset defaults so group and thread sessions use idle reset with a 10080-minute window while direct/DM sessions keep daily reset, and updates reset-policy tests.

## Best Possible Solution

If maintainers accept the behavior change, implement the type-aware implicit default in src/config/sessions/reset-policy.ts, keep explicit base/type/channel overrides authoritative, add exact 10080 coverage for no-config and unrelated-override group/thread paths, and update user-facing session docs plus the changelog.
