## Summary

Keep this issue open. Current main has the narrow late-success repair from #45493, but it is still a compaction-count-only reconciliation path plus separate run-finalization accounting. The broader requested convergence across token snapshots, status/UI state, memory-flush/auth-profile dependent fields, and post-timeout completion semantics remains unimplemented.

## Best Possible Solution

Keep the issue open as a design/implementation follow-up. The best path is to promote the existing successful-compaction hook into an authoritative, idempotent reconciliation boundary that deliberately converges compactionCount, post-compaction token snapshots/freshness, session rotation metadata, and dependent session fields, with regression coverage for timeout-late success and duplicate completion signals.
