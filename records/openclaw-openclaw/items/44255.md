## Summary

Keep open. Current main does not implement outbound message redaction, and the PR remains security-relevant but not merge-ready: the current delivery paths still pass raw send/poll params to plugin handlers, existing docs/tests cover logging and tool payload redaction rather than external channel sends, and the related #72733 guardrail is still open rather than a merged superseding fix.

## What This Changes

The PR adds dynamic sensitive-value tracking and config/env scanning, initializes that redaction from CLI startup, applies redaction to outbound send/poll handling, and adds docs plus logging redaction tests.

## Best Possible Solution

Keep the work open for maintainer/security design or an explicit replacement. The desired end state is a reviewed outbound guardrail that sanitizes normalized send and poll params before both plugin dispatch and core fallback delivery, preserves existing logging redaction semantics, avoids broad unreviewed config/env scraping, and has regression coverage for generic outbound, gateway/plugin dispatch, Slack, Telegram, and Discord send/poll paths.
