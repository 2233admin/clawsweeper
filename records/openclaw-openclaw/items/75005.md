## Summary

Keep open. Current main still has the related media-provider missing-key failure path, and this PR adds a useful media-provider hook, but the patch is not mergeable as-is because the no-provider-config fast path can bypass existing model-auth profile/env precedence for plugin-only providers.

## What This Changes

The PR adds a `resolveSyntheticAuth` media provider hook, passes the provider into media runner auth resolution, adds local/no-auth audio and video regression tests, and documents the hook for local or self-hosted media plugins.

## Best Possible Solution

Keep the hook, but move media synthetic auth behind the existing model-auth resolver: literal provider API keys should still win, explicit profile errors must remain hard failures, env/profile credentials should be tried before synthetic markers, and the media hook should only be used after the generic missing-key case. Add focused tests for plugin-only providers with an explicit missing profile and with real env/profile auth present.

## Reproduction Assessment

Yes. The original bug has a clear current-main reproduction through a configured audio media provider with no model auth, and the PR regression can be reproduced by extending the new plugin-only synthetic test with `entry.profile: "missing-profile"` or a real env/profile credential and no `models.providers` entry.

## Solution Assessment

No as-is. `resolveSyntheticAuth` is the right seam, but this implementation should not run it before the established model-auth resolver for plugin-only providers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve explicit auth before synthetic fallback:** `src/media-understanding/runner.entries.ts:446-456`
  - body: Line 446 returns the provider hook's synthetic key whenever there is no `models.providers` entry, before `resolveApiKeyForProvider(...)` runs. For plugin-only providers this bypasses explicit `entry.profile`/`preferredProfile` failures and any real env/profile credentials, so a configured credential can be ignored and the request runs with the synthetic marker instead. Please run model auth first and use the hook only as a fallback for the generic missing-key case.
  - confidence: 0.91
