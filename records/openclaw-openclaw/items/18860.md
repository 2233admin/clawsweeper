## Summary

Keep this PR open. Current main still lacks the requested `after_tools_resolved` hook and does not provide an equivalent plugin-facing API for per-attempt resolved tool/schema discovery. The related feature issue #21330 is closed, but it points back to this PR as the proposed implementation path, and the latest maintainer review comment also says to keep this PR open for follow-up.

## Best Possible Solution

Keep this PR open as the active implementation/design path. A maintainer-quality path is to finish review and either land an additive `after_tools_resolved` hook with docs, SDK baseline updates, and focused tests, or explicitly supersede it with a chosen pull-based resolved-tools API that safely exposes final per-attempt schemas.
