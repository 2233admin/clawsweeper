## Best Possible Solution

## Risks / Open Questions

- The proposed API would expose the full post-policy per-attempt tool list and schemas, including tools that are never called, to loaded plugins; maintainers should confirm that fits the trusted-plugin boundary.
- The PR body says schema `parameters` are passed by reference. If those are live provider-facing schema objects, hook handlers could accidentally mutate later provider/session behavior unless the final contract clones, freezes, or clearly enforces read-only handling.
- Fire-and-forget per-attempt emissions can repeat on retries; consumers may need documented dedupe expectations or a stable fingerprint.
