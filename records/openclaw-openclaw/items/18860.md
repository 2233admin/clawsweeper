The related feature issue #21330 is closed, but it points back to this PR as the implementation path, and the latest maintainer review comment also says to keep this PR open for follow-up.

## Best Possible Solution

A maintainer-quality resolution is to finish review and either land an additive `after_tools_resolved` hook with docs, SDK baseline updates, and focused tests, or explicitly supersede it with a pull-based resolved-tools API that safely exposes final per-attempt schemas.
