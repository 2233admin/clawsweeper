## Summary

Keep this PR open. Current main still lacks the requested `after_tools_resolved` hook and does not provide an equivalent plugin-facing API for resolved tool/schema discovery. The related issue is closed, but its body points back to this PR as the proposed implementation path, and the maintainer comment in the provided discussion already identifies this PR as the active design/review path.

## Best Possible Solution

Keep this PR open as the active implementation/design path for plugin-facing resolved tool and schema discovery. A maintainer-quality path is to finish review and either land an additive `after_tools_resolved` hook with docs, SDK baseline, and focused tests, or explicitly supersede it with a chosen pull-based resolved-tools API that exposes final per-attempt schemas safely.
