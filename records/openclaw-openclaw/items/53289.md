## Summary

Keep open. Current main still lacks the useful tools-vs-CLI clarification this PR is trying to add, but the PR is not ready to merge as-is because it documents invalid `openclaw sessions list` usage and duplicates the built-in-tools intro sentence. The diff is docs-only and has no observed security-sensitive surface.

## Best Possible Solution

Revise this PR, or apply an equivalent docs update in `docs/tools/index.md`: add a concise callout that `sessions_list`, `sessions_spawn`, and `agents_list` are agent/runtime tool identifiers rather than standalone shell commands; point shell users to `openclaw --help` and the CLI reference; use `openclaw sessions` and `openclaw agents list` as the examples; and remove the duplicate built-in-tools sentence before merge.
