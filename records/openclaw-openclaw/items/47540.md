## Summary

Keep open: current main still lacks the requested pricing-threshold split, but the proposed branch should not merge as-is because it is stale against current Control UI bootstrap auth, context freshness, and generated-doc artifact contracts.

## What This Changes

The PR adds `gateway.controlUi.chat.showPricingThreshold`, wires it through Gateway bootstrap and Control UI state, and changes chat context notices/styles/tests to show pricing-threshold hints separately from model context limits.

## Best Possible Solution

Rebase or replace the branch on current main, add the chat setting inside the already-authorized bootstrap response, implement the threshold split in the current `ui/src/ui/chat/context-notice.ts` view-model path, use a fresh context/request usage signal for threshold claims, and update only the current generated-doc hash artifact.

## Reproduction Assessment

Yes. The auth issue is reproducible by comparing the PR-head bootstrap response path with current main’s authorized handler, and the pricing false positive is reproducible with high cumulative `inputTokens` but low or stale fresh `totalTokens`.

## Solution Assessment

No. The requested direction is useful, but this branch is not the best current solution because it applies the behavior to stale Gateway/UI paths, bases pricing claims on aggregate session input, and carries obsolete generated artifacts instead of integrating with current main’s auth and freshness model.
