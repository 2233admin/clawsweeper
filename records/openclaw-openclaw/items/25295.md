## Summary

Current main still does not implement the intended Mattermost message-tool edit/delete actions. Main has Mattermost post update/delete helpers for preview streaming and cleanup, but the shared Mattermost action adapter still advertises and supports only send/react, so this PR remains useful. It is not merge-ready as provided because the diff mixes unrelated infra/provider/loader test changes with the Mattermost feature and crosses the extension boundary by importing a core internal helper.

## Best Possible Solution

Keep this PR open and request a focused refresh against current main. The best path is to wire Mattermost `edit` and `delete` through `describeMattermostMessageTool()`, `supportsAction()`, and `handleAction()`; reuse the existing Mattermost update/delete helpers through a local runtime seam or SDK-approved helper imports; validate non-empty edit text; add targeted Mattermost action tests and docs; and remove unrelated infra/provider/loader/changelog changes before maintainer review.
