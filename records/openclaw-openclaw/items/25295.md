## Summary

Keep this PR open. Current main still does not expose Mattermost `edit` or `delete` through the message action adapter: discovery advertises only `send`/`react`, `supportsAction` only accepts those actions, and `handleAction` throws for anything else. The PR remains a useful implementation candidate, but the current branch needs a refresh because it adds a destructive delete path, imports a core-internal helper from extension production code, and includes unrelated infra/provider/plugin test changes.

## Best Possible Solution

Keep this PR open as an implementation candidate, but refresh and narrow it against current main. The best path is to wire Mattermost `edit` and `delete` through `describeMattermostMessageTool`, `supportsAction`, and `handleAction`; route API calls through an SDK-safe lazy Mattermost runtime/helper seam; validate `messageId`, account/target context, requester trust, and non-empty edit content; add focused Mattermost action tests and public docs; and drop unrelated infra/provider/plugin changes from the branch.
