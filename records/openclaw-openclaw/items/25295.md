## Summary

Current main still does not implement the Mattermost message-tool edit/delete actions this PR is trying to add. Mattermost has internal post update/delete helpers for preview streaming and cleanup, but the public channel action adapter still advertises and supports only send/react, and the dispatcher enforces that support gate. The PR remains useful implementation work, but it should stay open for a focused refresh because the provided diff mixes unrelated infra/provider/loader test changes and crosses the extension boundary by importing a core internal helper.

## Best Possible Solution

Keep this PR open and ask for a focused refresh against current main. The best path is to wire Mattermost edit/delete through describeMattermostMessageTool(), supportsAction(), and handleAction(); reuse existing Mattermost update/delete helpers via a local runtime seam or SDK-approved helper imports; reject empty edit text; add targeted Mattermost action tests and docs; and drop unrelated infra/provider/loader/changelog changes before maintainer review.
