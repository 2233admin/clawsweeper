## Summary

Keep open. Current main still does not implement this PR's intended Mattermost message-tool edit/delete actions: Mattermost only advertises and supports send/react, and edit/delete helpers are currently internal to preview streaming rather than exposed through the message action adapter. The PR remains useful but needs a current-main refresh, boundary cleanup, tests, docs, and a security review for the destructive delete path.

## Best Possible Solution

Keep this PR open as an implementation candidate, but refresh and narrow it against current main: wire Mattermost edit/delete through describeMattermostMessageTool, supportsAction, and handleAction; route API calls through an SDK-safe lazy runtime/helper seam; validate messageId, target/account, requester trust, and non-empty edit content; add focused Mattermost edit/delete tests and public docs; and remove unrelated infra/provider/loader changes from the branch.
