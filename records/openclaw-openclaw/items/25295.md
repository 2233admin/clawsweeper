## Summary

Current main still does not implement Mattermost message-tool edit/delete actions. Mattermost has internal update/delete post helpers for preview streaming, but the user-facing action adapter only discovers and handles send/react, so this PR remains a live implementation candidate rather than something to close as implemented.

## Best Possible Solution

Keep this PR open as the implementation candidate, but ask for a focused refresh against current main: wire Mattermost `edit` and `delete` through `describeMattermostMessageTool()`, `supportsAction()`, and `handleAction()`; expose any needed update/delete runtime seam through the Mattermost plugin boundary or SDK-safe helpers; validate `messageId`, target/account, and non-empty edit text; add Mattermost action tests and user-facing docs; remove unrelated infra/provider/loader/changelog changes; and review destructive delete semantics explicitly.
