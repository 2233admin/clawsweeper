## Summary

Current main still does not implement the Mattermost message-tool edit/delete actions this PR targets. Mattermost has internal update/delete helpers for preview streaming cleanup, but the public action adapter advertises and supports only send/react, and the dispatcher blocks unsupported actions before handleAction can run. The PR remains useful implementation work, but it needs a focused refresh before maintainer review because the provided diff mixes unrelated changes and crosses the extension boundary.

## Best Possible Solution

Keep this PR open and ask for a focused refresh against current main. The best implementation path is to wire Mattermost edit/delete through describeMattermostMessageTool(), supportsAction(), and handleAction(); expose any needed post update/delete helper through a local runtime seam or approved SDK-facing helper; validate messageId/target/account and non-empty edit text; add targeted Mattermost action tests and docs; and drop unrelated infra/provider/loader/changelog changes before review.
