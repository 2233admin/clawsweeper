## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the requested stale recovery expiry and direct reply dedupe, while the PR head is stale against current outbound recovery safeguards and introduces security-sensitive coalescing across per-call delivery boundaries.

## What This Changes

The PR adds 10-minute queued outbound recovery expiry, a 1.5-second direct reply/mirror outbound dedupe cache, outbound tests, one channel setup type loosening, and broad Control UI render formatting churn.

## Best Possible Solution

Prepare a narrow outbound-only replacement or revision: add age-based recovery expiry only after claiming and re-reading the current queue entry, preserve all queued replay metadata, and split direct-reply dedupe into a separate design unless it can maintain per-call delivery semantics and recovery queue identity.

  - body: The dedupe key omits session, gatewayClientScopes, abortSignal, and per-call callbacks, but cache and inflight hits return before core delivery runs. A distinct caller can skip its own media scoping, hooks, adapter scope behavior, cancellation handling, and callbacks.

  - confidence: 0.9

  - body: The stale-expiry check moves entries from the startup snapshot. Current main claims and re-reads each pending queue file before acting, and expiry needs that same ownership guard to avoid racing reconnect or live delivery drains.
  - confidence: 0.86
- **[P2] Replay stored session and reply metadata:** `src/infra/outbound/delivery-queue-recovery.ts:65-81`
  - body: buildRecoveryDeliverParams omits stored replyToMode, formatting, and session metadata that current main forwards during recovery. Recovered messages can route, scope media, or fire hooks differently from the original queued send.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR introduces outbound coalescing and recovery movement that do not preserve all caller/session and queue ownership boundaries.

Concerns:

- **[high] Dedupe crosses caller scope boundaries:** `src/infra/outbound/deliver.ts:368`
  - body: The direct outbound cache key omits gatewayClientScopes and session while delivery passes both into adapters, media access scoping, and hooks. A cache or inflight hit can reuse a result produced under another caller or session context.
  - confidence: 0.88
- **[medium] Recovery replay can acknowledge unsent messages:** `src/infra/outbound/deliver.ts:641`
  - body: Queued recovery calls are eligible for the new inflight/recent-result dedupe path with no queue entry identity in the key, so duplicate stored records can be treated as delivered without a second adapter send.
  - confidence: 0.9
- **[medium] Startup expiry lacks ownership guard:** `src/infra/outbound/delivery-queue-recovery.ts:193`
  - body: The stale-expiry path moves entries based on the initial recovery snapshot rather than the claimed and re-read queue entry, so it can race another drain that has already owned or acked the same queue file.
  - confidence: 0.84
