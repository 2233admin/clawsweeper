## Summary

Keep this PR open. Current main does not implement either headline behavior: stale queued deliveries are still handled by max-retry/backoff recovery only, and direct outbound sends do not have the proposed short-window reply dedupe path. The PR branch also is not a clean landing candidate as-is because it carries broad unrelated UI formatting churn and unresolved outbound-dedupe review concerns around recovery replay, session/scope isolation, aborts, callbacks, and payload identity.

## Best Possible Solution

Keep the PR open for maintainer review, but do not land it as-is. The best path is a cleaned-up outbound-only change, likely split into a low-risk stale-recovery-expiry patch and a separately reviewed direct-reply dedupe design that includes all behavior-changing context in the key or deliberately narrows when dedupe is allowed. The unrelated UI churn should be dropped from this branch or moved to a separate formatting PR.
