## Summary

Keep open. Current main has several bundled runtime-deps mitigations, but the same remaining failure shape is still plausible: PID-alive runtime-deps locks are still trusted when start-time evidence is unavailable, and gateway HTTP binding still happens after plugin bootstrap, so a runtime-deps stall can keep `/healthz` unavailable. The issue is also paired with open PR #74950, which should be reviewed or replaced before this issue is closed.

## What This Changes

The issue asks for Docker/Unraid gateway startup to become healthy again and for `openclaw doctor --fix` to repair bundled runtime-deps state or exit cleanly instead of hanging.

## Best Possible Solution

Keep this issue open until #74950 or an equivalent stale Docker runtime-deps lock fix lands, then validate Docker/Unraid health and doctor exit behavior. Any remaining plugin-loader hot-loop or event-loop starvation should stay tracked under the narrower open issue #73532.
