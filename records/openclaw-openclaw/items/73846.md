## Summary

Keep this issue open. Current main has useful unreleased mitigations for plugin runtime-deps and manifest/mirror hot paths, but it still does pre-listen plugin bootstrap and bundled runtime-deps staging before `/healthz` can answer, and the reported post-install `doctor --fix` hang is not proven fixed by source, docs, tests, or release provenance.

## Best Possible Solution

Keep this issue open and route it through the gateway startup plus bundled plugin runtime-deps path. The next maintainer step should reproduce on current main or a 2026.4.27 candidate in Docker/Unraid, check whether pre-listen runtime-deps staging or plugin bootstrap is blocking `/healthz`, and inspect which handle or child process keeps `openclaw-doctor` alive after `repairBundledRuntimeDepsInstallRootAsync()` completes.
