## Summary

Keep open. Current main has a partial preservation path for `agents.defaults.model`, but it still returns `agentModelOverride` when `setDefaultModel=false`; `openclaw agents add` then writes that override into the new agent entry, so the provider default can still become a per-agent primary instead of inheriting the configured default. Current tests also assert the old override behavior, so this PR is not safely obsolete yet.

## Best Possible Solution

Keep the PR open or ask for a fresh rebased replacement. The implementation should preserve provider config/auth-profile/allowlist updates, restore `agents.defaults.model`, and suppress `agentModelOverride` plus the agent-scoped default-model note when `setDefaultModel=false` and an inherited primary model already exists; only fall back to a provider default override when there is no inherited primary model. Add regression coverage for `agents add`/plugin auth choices and address the existing test-review comment before merge.
