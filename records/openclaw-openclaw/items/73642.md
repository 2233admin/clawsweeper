## Summary

Keep this PR open. Current main still does not implement `openclaw infer model run --thinking <level>`: the infer model-run command has no `--thinking` option, local model-run completion does not accept or forward a reasoning override, and the Gateway model-run call omits the already-supported `thinking` request field. The PR remains an active implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Review and land this change, or an equivalent one, after confirming the `completeSimple` behavior for local provider probes and preserving coverage for local forwarding, gateway forwarding, provider-level normalization, and invalid input rejection.
