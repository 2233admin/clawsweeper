## Summary

Keep open. Current main has moved this out of the old generic failure text, but it still treats `GatewayDrainingError` as a terminal reply failure and returns a user-visible gateway-restarting message instead of waiting and retrying. The linked fix attempt was closed unmerged, and the newer restart/session-loss reports are adjacent rather than canonical replacements for this narrow retry gap.

## What This Changes

Request that `GatewayDrainingError` during gateway restart drain be handled as a transient condition with bounded auto-retry instead of surfacing a failure message to the user.

## Best Possible Solution

Add a bounded gateway-drain retry at the agent-runner boundary, reusing the existing transient-error loop shape where practical, with tests proving that the first drain error waits and retries without sending a user-visible failure while repeated drain failure exits predictably.
