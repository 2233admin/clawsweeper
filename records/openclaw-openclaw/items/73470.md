## Summary

Keep open. Current main still has advisory prompt guidance plus bounded read/fetch truncation handling, but I did not find a mandatory guardrail that broadly covers read/search operations or blocks finalization when evidence is partial. The issue is also paired with open unmerged PR #73618, which uses closing syntax for this work, so the issue should stay open until maintainers review, land, or close that PR.

## What This Changes

The issue requests a mandatory completeness-scan guardrail for read and search operations before an agent finalizes from potentially partial evidence.

## Best Possible Solution

Keep this open while maintainers define the enforcement scope and review PR #73618. The durable implementation should specify which read/search tools are covered, expose partial/truncated/no-result state where needed, and add tests showing the agent either performs a follow-up scan or clearly reports blocked/partial evidence before finalizing.
