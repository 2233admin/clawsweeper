## Summary

Keep open. Current main has advisory prompt guidance and a bounded file-read auto-paging helper, but I did not find a mandatory cross-surface completeness guardrail for read/search operations. The issue is recent and title-only, and the related Discord read/search items are narrower than this broader guardrail request.

## Best Possible Solution

Keep this issue open for maintainer product clarification. Define the read/search surfaces in scope, add a shared completeness contract where tools can report exhausted/truncated/partial/unknown plus next-cursor metadata, and enforce finalization by requiring a follow-up scan or an explicit partial/blocked answer. Route Discord-specific hangs through #73431/#73504 rather than closing this broader guardrail early.
