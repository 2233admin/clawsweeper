## Summary

Keep this PR open. Current main still does not implement the PR's intended cron lifecycle hook system: there is no `cron.hooks` config, no per-job hook field, no Gateway schema support for hooks, and scheduled/catch-up/manual cron execution paths still call core execution directly. Main does have a related `cron_changed` plugin observation hook from PR #72773, but that is after-the-fact observation and not a substitute for `beforeRun` aborts or configured script lifecycle dispatch.

## Best Possible Solution

Keep the PR open as an implementation candidate. Maintainers should decide whether scheduler-level cron lifecycle hooks are a core cron API or should be expressed through a plugin/API extension; if the feature proceeds, the PR should harden per-job validation, settle hook import caching/reload behavior, and update config schema, Gateway protocol, docs, and tests together.
