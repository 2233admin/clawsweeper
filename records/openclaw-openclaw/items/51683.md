## Summary

Keep this PR open. Current main still does not implement the PR’s intended `session_status` behavior: `session_status` delegates to the shared status builder without passing a time line, the shared status message only renders `args.timeLine` when a caller supplies it, and there is no `formatUserLocalIsoTimestamp`/`local ISO` implementation or regression test on main. The linked reminder issue may be closed, but this PR remains a plausible focused implementation candidate after rebasing onto the current status rendering split.

## Best Possible Solution

Rebase and rework this PR against current main: add or reuse a small timezone-aware local ISO formatter, pass a machine-readable local ISO plus UTC timestamp into the shared status card for `session_status`, and add a targeted regression test that pins the clock and a configured user timezone. Keep the change scoped away from system prompt injection and scheduler behavior.
