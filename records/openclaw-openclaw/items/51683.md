## Summary

Keep this PR open. Current main still does not implement the PR’s intended `session_status` behavior: there is renderer support for an optional `timeLine`, but the `session_status` tool and shared status builder never compute or pass a machine-readable local ISO plus UTC timestamp. The PR remains a focused implementation candidate, but it needs a rebase/rework against the current shared status modules.

## Best Possible Solution

Rebase and rework this PR against current main. Add or reuse a small timezone-aware formatter, thread a machine-readable local ISO timestamp plus UTC timestamp through the shared status card path used by `session_status`, and add a pinned-clock regression test for a configured non-UTC user timezone. Keep the change scoped away from system prompt injection, cron scheduler behavior, auth, CI, and dependency surfaces.
