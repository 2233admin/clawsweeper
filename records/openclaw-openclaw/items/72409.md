## Summary

Kept open. This PR is maintainer-authored and carries the protected `maintainer` label, so this cleanup lane should not auto-close it. Current `main` does not already implement the PR’s intended policy: the manifest registry still emits duplicate plugin-id diagnostics for distinct-root cross-origin conflicts, while suppressing only same-physical-root duplicates and intentional installed-global/bundled duplicates. The best next step is maintainer review of the warning policy and the security-visibility concern raised on the PR, not automated closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide the plugin duplicate-diagnostic policy explicitly: suppress only false positives for the same physical plugin root, keep useful warnings for distinct-root or trust-boundary shadowing, reconcile that policy with #42099 and #42192, then land the focused registry tests and changelog only after the active security concern and CI failure are addressed.
