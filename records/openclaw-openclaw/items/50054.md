## Summary

Keep this PR open. Current main does not implement the PR’s intended distributed ACP session locking, Redis-backed fail-closed behavior, `acp_execution_locked` skip reason, or env/documentation surface. The PR is not obsolete, but it also should not be treated as merge-ready without maintainer review because the later review context identified remaining correctness gaps around preserving queued turns, renewal failure behavior, and applying the lock to `/acp steer`. The author did address earlier Redis timeout and generic `REDIS_URL` fallback comments in the PR branch, which is useful context for a rework.

## Best Possible Solution

Keep the PR open and ask for a focused current-main rework. The best implementation should put the distributed lock at a shared ACP turn boundary so normal bound-message dispatch and `/acp steer` are covered, preserve current queued-turn semantics or document an explicit new policy, fail closed only when `OPENCLAW_ACP_SESSION_LOCK_REDIS_URL` is explicitly configured, handle command timeouts/socket cleanup/owner-checked renew and release, and add targeted tests plus `.env.example` and public ACP docs updates.
