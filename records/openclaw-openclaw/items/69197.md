## Summary

Keep #69197 open. Current main still lacks the PR's poll-active suppression path: `maybeNotifyOnExit()` emits from the exec completion promise without checking whether `process(poll)` is actively waiting, while the poll path waits on a 250 ms timer loop and does not mark the notification as consumed. The PR is a focused, plausible bug fix with useful reproduction/live-validation discussion, but it still needs maintainer review for the overlapping-poll review comment rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open. The best path is for a maintainer to review the active-poll suppression design against current main, likely require a small rebase plus either a poll refcount/token or an explicit decision that overlapping polls are unsupported, then land with the regression tests if the race is confirmed.
