## Summary

Current main and the latest release still do not implement the PR's active-poll suppression for background exec completion notifications. The PR should stay open as a plausible focused fix, but it needs maintainer review/amendment because the boolean `pollActive` design has an overlapping-poll race and the branch touches the exec/process runtime.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to amend or justify the active-poll design, preferably with an in-flight poll counter/token unless maintainers can prove same-session process polls are serialized. The final patch should preserve normal `notifyOnExit` behavior for unpolled background exits, suppress the active-poll ghost, keep timeout/later-exit and kill behavior intact, rebase cleanly over current poll wait/abort handling, format the try block, and add direct system-event/heartbeat regression coverage.
