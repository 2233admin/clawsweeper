## Summary

Keep this issue open. Current main has meaningful mitigations around ACP availability and Feishu current-conversation binding, but Cursor is still an advertised ACP harness route when ACP spawning is available, and I did not find a deterministic guard or regression test proving that a plain Cursor-related link in a custom Feishu session cannot trigger `sessions_spawn` with `runtime: "acp"` and `agentId: "cursor"`.

## Best Possible Solution

Keep this open and add a targeted ACP intent-routing guard plus regression coverage. A Feishu/custom-agent message containing a Cursor GitHub link or incidental Cursor text should remain in the current custom-agent session and must not call `sessions_spawn(runtime: "acp", agentId: "cursor")` unless the user explicitly asks to run, use, open, or continue work in Cursor/ACP. The likely implementation belongs around `src/agents/system-prompt.ts`, `extensions/acpx/skills/acp-router/SKILL.md`, and a focused prompt/tool-routing or Feishu dispatch test.
