## Summary

Keep this PR open. Current main still lacks the inbound-media read guard this PR proposes: the read tool returns image-sanitized results without classifying `media/inbound/*` paths or wrapping text blocks as untrusted prompt data. The linked #11207 security issue remains current and actionable, and the PR discussion shows the author addressed multiple review findings rather than proving the work is obsolete.

## Best Possible Solution

Keep this PR open and have a maintainer review or land a maintained version of the inbound-media read guard. The fix should live in the read-tool path, wrap only text blocks from staged inbound media with the existing untrusted-data wrapper, sanitize path labels, preserve image blocks, propagate `containerWorkdir`, and include focused regression coverage. Keep #11207 open until a reviewed fix merges.
