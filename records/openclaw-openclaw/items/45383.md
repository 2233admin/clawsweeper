## Summary

Keep this PR open. Current main still does not implement the inbound-media read guard: the read tool returns sanitized image-normalized results without classifying `media/inbound/*` paths or wrapping text blocks as untrusted data. The linked #11207 security issue remains open and actionable, and the PR discussion shows the author addressed multiple review findings rather than proving the work is obsolete.

## Best Possible Solution

Keep this PR open and review or land a maintained version of the inbound-media read guard. The implementation should stay in `src/agents/pi-tools.read.ts`, wrap only text blocks from staged inbound media with the existing untrusted-data wrapper, sanitize path labels, preserve image blocks, propagate sandbox `containerWorkdir`, and include focused regression coverage. Keep #11207 open until a reviewed fix merges.
