## Summary

Keep this PR open. Current main still does not implement the intended top-level `openclaw usage` command with custom ranges, session ranking, by-model summaries, and JSON output. Main has related usage infrastructure, Control UI usage views, chat `/usage cost`, and the narrower `openclaw gateway usage-cost`, but that does not make this PR obsolete. The submitted implementation also has unresolved functional review blockers that should be addressed before maintainer consideration.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main and implement `openclaw usage` as a thin CLI wrapper over the existing `usage.cost` and `sessions.usage` APIs, with tests for Commander options, conflicting date flags, local/UTC semantics, complete aggregation before display limiting, shared token formatting, and JSON output. If maintainers decide `openclaw gateway usage-cost` plus the Control UI is sufficient, that should be an explicit product decision or a canonical follow-up, not an automatic cleanup close.

## Risks / Open Questions

- As submitted, custom date ranges can be wrong outside UTC because `--from` and `--to` are parsed in different timezone modes.
- As submitted, `--days` can silently override more explicit range flags, producing surprising summaries.
- The submitted session/model breakdown path can exclude older high-cost sessions if it applies the display limit before aggregating all in-range sessions.
- Landing this without refit could create a second aggregation path that drifts from the existing `usage.cost` and `sessions.usage` Gateway behavior.
