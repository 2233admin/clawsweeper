## Summary

Keep this issue open. Current main has improved adjacent pieces, including bundled DeepSeek onboarding, additive config patch behavior, configured model views, and unmanaged gateway restart fallback, but the full model-switch UX report is not implemented because explicit allowlists still intentionally hide configured provider models and active sessions keep their selected/runtime model until a user or tool changes it.

## What This Changes

The issue requests a smoother model-provider switch flow after moving from `bailian/qwen3.6-plus` to `deepseek/deepseek-v4-pro`, covering config patching, dual config visibility, allowlist warnings, gateway restart behavior, and active-session model refresh.

## Best Possible Solution

Split this into narrower maintainer decisions: keep the bundled DeepSeek onboarding path, add high-signal `models status`/doctor warnings for hidden configured providers or defaults outside `agents.defaults.models`, and decide explicitly whether a first-class `openclaw models add/set` flow should mutate provider config, allowlists, generated `models.json`, and existing session overrides.

## Reproduction Assessment

Partially. The report has concrete symptoms and environment data, and current source/tests reproduce the allowlist precedence plus explicit-session-switch behavior; the missing exact patch payload and full config prevent a high-confidence replay of all five steps end to end.

## Solution Assessment

Unclear. The requested automatic allowlist expansion and active-session re-evaluation may not be the safest default because they can bypass intentional scoping; warnings plus explicit model setup/switch commands look like a narrower maintainable direction.
