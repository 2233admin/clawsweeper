## Summary

Keep open. Current main has Japanese support for the browser Control UI and a separate Japanese docs translation pipeline, but it still does not provide first-class core UX localization for CLI onboarding, help, workspace templates, or shared runtime labels.

## What This Changes

Add key-based Japanese localization for OpenClaw core UX with English fallback, starting with onboarding, help, templates, and common labels.

## Best Possible Solution

Keep this issue open until maintainers define the core i18n contract outside the browser UI: locale resolution for CLI/wizard contexts, English fallback semantics, ja-JP resources for agreed onboarding/help/template/common-label surfaces, and focused tests for wizard prompts, CLI help rendering, fallback, and workspace template generation.
