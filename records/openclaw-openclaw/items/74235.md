## Summary

Keep this PR open. The underlying Google Chat threading bug is real on current main, and the PR is pointed at the right owner boundary, but the latest head is draft and still has a concrete media-thread regression that should block merge.

## What This Changes

The PR adds Google Chat thread resource validation, carries inbound thread metadata through webhook/reply/message-tool paths, normalizes outbound thread targets at the Google Chat boundary, and expands core plus Google Chat regression tests.

## Best Possible Solution

Revise the Google Chat outbound adapter so valid thread resource names are preserved consistently for text and media paths, keep the resource-name normalization inside the plugin boundary, then take the PR out of draft only after isolated Google Chat gateway validation passes on the exact head.

## Reproduction Assessment

Yes. Current main is source-reproducible from the missing MessageThreadId, raw replyToId delivery, and unvalidated thread.name path; the PR regression is reproducible by calling googlechatOutboundAdapter.attachedResults.sendMedia with threadId set to a valid spaces/.../threads/... resource and no replyToId.

## Solution Assessment

No, not as currently patched. The boundary-placement is maintainable, but sendMedia should preserve explicit valid threadId-only delivery the same way sendText does, and the draft hold needs exact-head live validation before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve explicit media thread targets:** `extensions/googlechat/src/channel.adapters.ts:362`
  - body: `sendText` preserves a valid explicit `threadId` by passing `allowThreadIdOnly: true`, but `sendMedia` calls the same resolver without that option. A media send with `threadId: "spaces/.../threads/..."` and no `replyToId` will upload and then send with `thread: undefined`, posting top-level instead of in the requested thread. Preserve valid thread resources here and add a media regression test.
  - confidence: 0.89
