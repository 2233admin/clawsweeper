## Summary

Keep this PR open. Current main still has a real Google Chat threading gap, and the PR is aimed at the right surface, but the latest patch is not merge-ready because it can reintroduce an inbound thread after shared reply handling has stripped threading for `replyToMode: "off"`.

## What This Changes

The PR adds Google Chat thread-resource validation, carries inbound `MessageThreadId` through webhook/reply/tool paths, normalizes outbound thread targets, and expands core plus Google Chat regression tests.

## Best Possible Solution

Revise the PR so Google Chat preserves valid inbound thread resources for normal threaded replies and explicit reply tags, while still honoring `replyToMode: "off"` for implicit fallbacks. Add focused off-mode regression coverage for direct delivery and routed/follow-up delivery, then land and reconcile #39554, #69422, and #67055 as appropriate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor replyToMode before using the inbound thread:** `extensions/googlechat/src/monitor-reply-delivery.ts:38`
  - body: `replyToMode: "off"` strips `payload.replyToId` in the shared reply filter, but this new unconditional fallback turns a missing `replyToId` back into `params.inboundThreadId`. That makes normal Google Chat text/media replies land in-thread even when the account config asks for unthreaded replies; only fall back when threading is allowed or when the payload carried an explicit message-resource reply target.
  - confidence: 0.88
- **[P2] Preserve off-mode in routed Google Chat replies:** `extensions/googlechat/src/channel.adapters.ts:143-144`
  - body: `resolveReplyTransport` now promotes any valid Chat `threadId` to `replyToId`. Routed and follow-up replies pass the session `MessageThreadId` separately from the filtered payload, so a reply whose `replyToId` was removed for `replyToMode: "off"` will still be sent threaded. Gate the `threadId` fallback on the account reply mode while preserving explicit thread-shaped `replyToId` values.
  - confidence: 0.82
