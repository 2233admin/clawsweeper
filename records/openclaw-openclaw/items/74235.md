## Summary

Keep open. Current main still has the Google Chat threading gap, and this PR is the right active implementation lane, but the latest head still has a concrete media-thread regression that should be fixed before merge.

## What This Changes

The PR preserves Google Chat thread resource names through inbound context, reply delivery, routed outbound sends, and message-tool sends, adds resource-name validation, and expands core plus Google Chat regression tests.

## Best Possible Solution

Revise this PR so Google Chat resource-name normalization stays inside the plugin boundary while preserving valid explicit thread resources consistently for text and media paths. Then land with a media-thread regression test and reconcile #39554, #69422, #42510, and #67055 against the shipped behavior.

## Reproduction Assessment

Yes. The original bug is source-reproducible on current main from the missing `MessageThreadId` plus raw `replyToId` delivery, and the PR regression is reproducible by sending Google Chat media through the generic outbound adapter with `threadId: "spaces/.../threads/..."` and no `replyToId`.

## Solution Assessment

No, not as currently patched. The overall direction is maintainable, but the media path should preserve explicit valid thread resources the same way the text path does while still suppressing unsafe implicit fallback thread ids.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve explicit media thread targets:** `extensions/googlechat/src/channel.adapters.ts:362`
  - body: `sendMedia` now calls `resolveGoogleChatOutboundThread` without `allowThreadIdOnly`, unlike `sendText`. A direct Google Chat media send with a valid `threadId: "spaces/.../threads/..."` and no `replyToId` will upload successfully but call `sendGoogleChatMessage` with `thread: undefined`, so the attachment lands top-level instead of in the requested thread. Preserve valid explicit thread resources here and add a media regression test.
  - confidence: 0.88
