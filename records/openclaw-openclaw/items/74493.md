The PR changes Claude CLI token and token auth-profile auth-epoch encoding to ignore rotating token material, bumps the auth-epoch version to 5, adds targeted regression tests, and adds a changelog fix for #74312.

## Best Possible Solution

Ship a narrow auth-epoch fix that treats token rotation and partial Claude keychain reads as the same identity, preserves invalidation when real identity fields change, keeps the version bump for stored-binding migration, and lands with the targeted regression coverage after the branch is rebased and validated.
