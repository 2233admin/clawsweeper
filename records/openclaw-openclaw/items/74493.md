## Summary

Keep open. Current main still has the Claude CLI auth-epoch reset path, while this PR is an active, relevant fix with no blocking code-review findings from this pass; it should proceed through maintainer review and exact-head CI before merge.

## What This Changes

The PR makes Claude CLI token-shaped credentials, token auth-profile credentials, and Claude null local credential reads use identity-only auth-epoch hashing, bumps the auth-epoch version to 5, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent narrow patch after maintainer review accepts the identity-only token epoch contract and required checks are green; keep deeper parser semantics as a separate follow-up unless maintainers want to expand scope.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main can parse a transient Claude keychain read as `type: "token"`, current token hashing changes the epoch when the token changes, and same-version epoch mismatches reset reusable CLI sessions with `reason=auth-epoch`.

## Solution Assessment

Yes, with a maintainer contract decision. The PR fixes the reset at the epoch layer, includes the version bump needed for migration, and adds regression tests for both token rotation stability and real identity changes; parser hardening is not required for the reported symptom.
