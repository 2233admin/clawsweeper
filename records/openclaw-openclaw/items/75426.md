## Summary

Current main already fixes the reported Signal SSE receive wedge: the Signal monitor now disables the bounded 10s deadline only for the long-lived receive stream, while RPC/check calls keep their normal deadlines. The issue's logs match the closed #74741 failure mode, and current main contains regression coverage for idle signal-cli 0.14.3 event streams.

## What This Changes

The issue requests reliable Signal inbound DM handling and prompt `channels status` behavior when outbound Signal sends and gateway health probes are otherwise working.

## Best Possible Solution

Keep the current main implementation that separates bounded Signal RPC/check deadlines from the long-lived receive SSE stream, then include it in the next shipped release for users on `v2026.4.29`.

## Reproduction Assessment

Yes. The report's repeated `Signal SSE connection timed out after 10000ms` logs on signal-cli 0.14.3 match #74741, and current tests reproduce the idle event-stream case without needing a live Signal account.

## Solution Assessment

Yes. The implemented fix is narrowly scoped to the receive SSE monitor: it disables the deadline only for the long-lived stream while preserving Signal RPC/check timeouts, reconnect/backoff behavior, access control, and message handling.
