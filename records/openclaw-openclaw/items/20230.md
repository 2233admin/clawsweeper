## Summary

Keep open. Current main covers the active assistant name/avatar path and gives user bubbles a distinct style, but it still does not implement the central request: chat history groups have no assistant agent/style attribution, assistant rendering uses one active assistant identity for all assistant groups, and there is no validated per-agent chat bubble color/style setting or render hook. The existing maintainer comment also explicitly kept this open for the same remaining Control UI contract work.

## What This Changes

Request to add distinct and configurable per-agent chat bubble styling plus configured agent-name display in the WebUI chat, especially for multi-agent conversations.

## Best Possible Solution

Keep this issue open until maintainers define and implement, or explicitly reject, a Control UI chat identity/style contract. A complete implementation should add assistant attribution at message or group level, define a validated optional per-agent bubble color/style setting or safe render hook, render assistant-specific bubbles in CSS, update docs/config if a public key is added, and add focused tests for active-session, mixed-agent, and historical chat rendering.
