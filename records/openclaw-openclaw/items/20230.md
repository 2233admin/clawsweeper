## Summary

Current main partially addresses the display-name side by resolving one active assistant identity for the selected session and rendering user-vs-assistant bubble styling, but it still does not implement the central request: per-agent bubble color/style configuration or message/group-level assistant attribution for mixed-agent chat history.

## Best Possible Solution

Keep this open until maintainers either implement or explicitly reject a Control UI chat identity/style contract. A complete implementation should add message or group-level assistant attribution, a validated optional per-agent bubble color/style setting or render hook, renderer/CSS support for assistant-specific bubbles, and focused UI tests for mixed-agent and historical sessions.
