## Summary

Keep open. Current main does not implement this PR's intended ordering change: both prompt-side and assistant-side rotate-profile paths still await the auth-profile failure marker before calling `advanceAuthProfile()`, and that marker still awaits `markAuthProfileFailure()`, which acquires the auth-store file lock. The paired bug report remains open and this PR is the implementation candidate.

## Best Possible Solution

Keep this PR open and either rebase it onto current main or implement the same conservative fix in the current failover surfaces: rotate first, then record cooldown for the failed profile without blocking the rotation path, with regression coverage for rate-limit rotation ordering, timeout no-cooldown behavior, prompt-side failover, and eventual cooldown persistence. Then review it together with the linked open bug report.
