## Summary

Keep this PR open because current main still lacks the requested Google Chat `sessionThread` config and per-thread session partitioning, so the branch is not obsolete. The branch is useful, but it needs maintainer review before merge because current main has since refactored the Google Chat turn/reply path and the PR still has a blocking changelog policy issue plus a stale JSDoc detail.

## What This Changes

The PR adds an optional Google Chat `sessionThread` config, derives thread-specific session keys, preserves inbound thread routing for replies, adds schema/docs/changelog updates, and adds a focused session-key unit test.

## Best Possible Solution

Land a rebased Google Chat plugin change that keeps route matching space-based, adds an additive `channels.googlechat.sessionThread` option, partitions only the session key per inbound thread, carries the case-preserved thread name through context/delivery metadata, updates schema/docs/changelog accurately, and leaves broader streamed/follow-up thread propagation to #69422.
