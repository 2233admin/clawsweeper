## Summary

Keep this PR open. Current main still does not implement the PR’s intended channel-startup behavior: startup enumerates only loaded channel plugins, while direct lookup can resolve bundled channel plugins that are absent from the active registry snapshot. The PR is recent, contributor-authored, has no protected labels, and the supplied related context does not show a merged superseding implementation.

## Best Possible Solution

Review and land or revise this focused PR rather than closing it. The likely best path is to merge configured channel ids from runtime config into the startup plugin set via the existing `getChannelPlugin()` fallback, dedupe loaded and configured entries, keep startup hot-path behavior intentional, and run targeted gateway/QA-channel tests plus the changed gate before merge.
