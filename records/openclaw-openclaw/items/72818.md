## Summary

Current main still does not implement this PR’s intended channel-startup change. `startChannels()` still enumerates only loaded channel plugins, while direct channel lookup can resolve bundled plugins absent from that registry snapshot. The related registry-pinning reports are about a different already-fixed desync path, and #71858 is closed unmerged, so this PR remains the active implementation candidate rather than a cleanup close.

## Best Possible Solution

Keep this PR open for maintainer review. The likely best path is to land or revise this focused fix: build the startup plugin set from loaded channel plugins plus configured channel ids resolved through `getChannelPlugin()`, dedupe by plugin id, preserve the gateway hot-path constraints, keep the diagnostic `context.assembled` output stable, and retain the QA-channel thread roundtrip coverage.
