## Summary

Keep this PR open. Current main does not implement the PR’s intended channel-startup change: gateway startup still enumerates only `listChannelPlugins()`, while the registry API shows that `listChannelPlugins()` is loaded-registry-only and `getChannelPlugin()` has the bundled fallback the PR wants to use for configured channels absent from the active registry snapshot. The PR is recent, focused, and split from a closed broader PR; it is not maintainer-authored and has no protected labels.

## Best Possible Solution

Keep this PR open for review. The best path is to review the narrow startup enumeration change, confirm it does not accidentally materialize unwanted bundled channel runtime, and run the targeted gateway/QA channel tests plus the changed gate before merge.
