## Summary

Keep open. The PR targets a real SDK event-stream bug, but the latest branch still drops later chat-only delta projections after normalizing the first one, so chat-only clients can receive stale UI updates until final.

## What This Changes

The PR changes `@openclaw/sdk` run event streaming to suppress duplicate raw Gateway `chat` projection frames, normalize chat-only projections, add focused SDK unit tests, and add a changelog entry.

## Best Possible Solution

Keep the SDK contract centered on stable per-run events: suppress raw chat projections only when canonical agent assistant or terminal events already exist, preserve every chat-only `delta` projection as `assistant.delta` until final, keep `oc.rawEvents()` as the raw escape hatch, and lock the behavior with duplicate-suppression plus multi-delta fallback tests.

## Review Findings

- **[P2] Preserve later chat-only deltas:** `packages/sdk/src/client.ts:344-345`

Keep canonical-agent detection separate from fallback chat deltas.
