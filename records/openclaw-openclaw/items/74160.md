## Summary

Keep open; this PR is not ready to merge. The tail-drop direction addresses a real disk-repair gap on current main, but the PR head still stops at non-message tail entries even though SessionManager can use those entries as the leaf while replaying the assistant message beneath them.

## What This Changes

The PR adds a session-file repair pass that drops trailing errored assistant entries, reports a new drop counter, expands regression tests, and documents the transcript-hygiene rule.

## Best Possible Solution

Revise the repair pass to preserve metadata rows but treat non-replay metadata as transparent when finding the last replay-visible turn, then drop trailing errored assistant entries beneath them and add regression coverage for `user -> errored assistant -> model_change` or `thinking_level_change`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a JSONL session ending `user -> errored assistant -> model_change` or another non-message metadata entry. The PR stops at the metadata row, while SessionManager uses that row as the leaf and still emits the assistant beneath it as the final replay-visible message.

## Solution Assessment

No. Disk repair is the right narrow surface and the WeakSet counter fix is sound, but stopping at every non-message tail is not the best fix because metadata rows should be preserved while scanning past them to the replay-visible tail.
