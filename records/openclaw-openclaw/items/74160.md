## Summary

Keep this PR open. Current main still rewrites trailing empty errored assistant entries to the fallback sentinel in `repairSessionFileIfNeeded`, and current tests explicitly lock that trailing rewrite/no-op behavior. Main has separate outbound replay guards that strip failed/trailing assistant turns before provider calls, so older consecutive-error PRs were reasonably closed, but this PR targets the on-disk session repair loop and observable backup/log churn that is not implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a focused current-main fix in `repairSessionFileIfNeeded`: add a tail-aware drop pass for errored empty or already-sentinel assistant entries, keep mid-transcript rewrite behavior, report/log the dropped count, add the PR's regression coverage, and update transcript-hygiene docs if the persisted repair contract changes.
