## Summary

Keep open. The report is actionable and already has open PR #74060 as a paired implementation candidate; current main has account-aware Feishu resolution in the health path, but the Feishu setup/status surface still probes credentials from top-level `channels.feishu` only.

## What This Changes

Make Feishu health/setup status treat `channels.feishu.defaultAccount` plus `accounts.*.appId/appSecret` credentials as configured and probeable instead of requiring top-level credentials.

## Best Possible Solution

Keep the fix Feishu-plugin-owned: resolve setup/status probes through the selected Feishu account and add focused regression coverage for account-scoped credentials, without adding Feishu-specific behavior to core.

## Reproduction Assessment

Unclear for the exact live health table because no Feishu credentials were available in env/profile during this read-only review. The narrower source-level defect is reproducible by following `resolveStatusLines`, where account-only credentials make Feishu configured but are not passed to the setup/status probe.

## Solution Assessment

Yes. Reviewing and landing PR #74060, or an equivalent Feishu-plugin-owned change, is the narrowest maintainable path because it reuses existing account resolution and keeps the fix out of core.
