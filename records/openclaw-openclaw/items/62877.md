## Summary

Keep open. Current main still returns the raw `Usage: /btw <side question>` text for a missing BTW question, the test suite still expects that raw placeholder, and the outbound sanitizer path confirms this is better fixed at the command producer rather than by globally escaping channel output.

## What This Changes

The issue asks for channel-facing slash-command usage text such as `/btw <side question>` to avoid raw angle-bracket placeholders that markdown or HTML renderers strip.

## Best Possible Solution

Change producer-owned command usage replies to render placeholders safely, preferably by formatting the whole command usage as inline code, update the colocated tests, and audit nearby auto-reply usage strings with raw angle-bracket placeholders without weakening the sanitizer's autolink behavior.
