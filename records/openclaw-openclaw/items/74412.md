## What This Changes

The PR updates cron delivery inference to emit `user:<id>` for Discord `direct`/`dm` session keys and adds four cron-tool tests covering Discord direct, legacy dm, group, and channel inference.

## Best Possible Solution

Land this narrow cron-tool fix, or an equivalent maintainer patch, so inferred Discord direct/dm announce delivery is explicit as `user:<id>` while preserving existing current-context precedence and group/channel behavior.

## Reproduction Assessment

Yes. A Discord direct/dm session key for an agent-turn cron add with announce delivery and no explicit target currently infers a bare numeric `to`, and the Discord normalizer then maps that bare ID to `channel:<id>`.

## Solution Assessment

Yes. The PR fixes the ambiguity at the inference point that creates the malformed target, stays scoped to Discord direct/dm session keys, and adds regression coverage for both DM marker forms and non-DM cases.
