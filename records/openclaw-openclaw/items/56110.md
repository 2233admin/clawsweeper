## Summary

Keep this issue open. Current main does not auto-load `STATE.md`, does not seed a `STATE.md` template, and the bundled configurable extra-bootstrap hook still rejects unrecognized basenames, so the requested first-class workspace state file remains unimplemented. Related #59050 is about load order, and #39120 could become a broader path for arbitrary bootstrap files, but neither currently resolves the default `STATE.md` product decision.

## Best Possible Solution

Keep this issue open until maintainers choose the product direction. A first-class implementation would add STATE.md to the workspace bootstrap contract in src/agents/workspace.ts, update agent file listing/control surfaces, docs, templates if seeding is intended, and regression tests for normal runs, missing markers, truncation, and sub-agent exclusion. If maintainers prefer configurability, resolve it through the broader arbitrary bootstrap files work in #39120 and explicitly document how STATE.md should be configured.
