## Summary

Keep open. The PR addresses the earlier thrown first-send gap, but it still leaves permanent `bestEffort` per-payload failures on the generic retry path, so a permanent error can remain pending and replay successful sibling payloads later; the branch also currently reports a dirty merge state.

## What This Changes

The PR expands outbound delivery permanent-error regexes, applies the classifier to live send cleanup, adds recovery/live-send regression tests, and adds a changelog entry.

## Best Possible Solution

Revise this PR so permanent classification applies to both thrown live-send failures and `bestEffort` per-payload failures, while preserving retry/backoff for transient errors. Maintainers can still choose whether the conservative regex expansion is acceptable now or whether this should become a channel-owned classifier seam before landing.

## Reproduction Assessment

Yes. A focused unit scenario can call `deliverOutboundPayloads` with `bestEffort: true`, make the first payload reject with `message is too long`, and verify the queue still goes through the generic partial-failure `failDelivery` path instead of `moveToFailed`.

## Solution Assessment

No. The second commit makes the shared-pattern approach much closer, but the patch is not complete until the permanent decision is propagated out of suppressed `bestEffort` payload errors and the dirty branch state is resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Classify best-effort permanent failures before retrying:** `src/infra/outbound/deliver.ts:903-908`
  - body: The new permanent check only runs in the outer catch. With `bestEffort`, per-payload send errors are caught inside `deliverOutboundPayloadsCore`, reported through `onError`, and the wrapper later calls `failDelivery(queueId, "partial delivery failure (bestEffort)")`. A permanent `message is too long` in a multi-payload delivery can therefore stay pending and replay successful sibling payloads later. Track whether any best-effort error is permanent and move the entry to `failed/` instead.
  - confidence: 0.84
