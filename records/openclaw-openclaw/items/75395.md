## Summary

Keep open. Current main still does not apply the shared Anthropic assistant-prefill sanitizer in the OpenRouter openai-completions stream path, while the direct Anthropic and Cloudflare Anthropic routes already do. Open PR #75400 explicitly closes this issue and should be reviewed or closed before the issue is resolved.

## What This Changes

Request OpenRouter-routed Anthropic models using the openai-completions adapter to strip trailing assistant prefill turns when reasoning or thinking is enabled.

## Best Possible Solution

Review and land #75400, or an equivalent provider-local OpenRouter fix, that reuses the shared Anthropic prefill sanitizer only for Anthropic-routed OpenRouter payloads with reasoning/thinking enabled and preserves non-Anthropic OpenRouter behavior.

## Reproduction Assessment

Yes. The report includes concrete OpenRouter configuration and repeated 400 logs, and current-main source inspection shows the OpenRouter stream path lacks the sanitizer used by direct Anthropic and Cloudflare Anthropic routes.

## Solution Assessment

Yes. A provider-local OpenRouter wrapper around the existing shared sanitizer is the narrow maintainable direction; a broad mutation in the generic openai-completions builder would risk unrelated providers.
