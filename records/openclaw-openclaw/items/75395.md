## Summary

Keep open. Current main still does not apply the shared Anthropic thinking-prefill sanitizer in the OpenRouter `openai-completions` stream path, and the linked fix PR is open and explicitly closes this issue, so the issue should stay open until that PR is reviewed and merged.

## What This Changes

OpenRouter Anthropic models using the `openai-completions` adapter should strip trailing assistant prefill turns when thinking/reasoning is enabled, matching direct Anthropic and Cloudflare AI Gateway routes.

## Best Possible Solution

Review and land the linked OpenRouter provider fix if its tests prove the sanitizer is applied only for Anthropic-routed OpenRouter payloads while preserving existing OpenRouter routing, reasoning, attribution, cache-control, and Gemini replay behavior.

## Reproduction Assessment

Yes. The issue provides a concrete OpenRouter configuration and repeated 400 logs, and current-main source inspection shows the OpenRouter stream path lacks the sanitizer that direct Anthropic and Cloudflare AI Gateway paths already use.

## Solution Assessment

Yes. A provider-local OpenRouter wrapper that reuses the shared Anthropic thinking-prefill sanitizer is narrower and more maintainable than a broad core `openai-completions` mutation that could affect unrelated providers.
