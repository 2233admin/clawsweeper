## Summary

Keep open. The PR targets a real runtime-deps lock edge case, but the latest diff still makes the mtime fallback apply to every live-PID owner missing `starttime`, including current locks with `createdAtMs` on platforms where starttime cannot be read, which violates the existing active-lock safety invariant.

## What This Changes

The PR adds an mtime-based stale fallback in `shouldRemoveRuntimeDepsLock()` for PID-alive runtime-deps locks without starttime evidence and updates bundled runtime-deps tests for the legacy Docker lock case.

## Best Possible Solution

Preserve the active-lock invariant while adding a narrow fallback for true legacy owner records, with tests covering both stale no-starttime/no-createdAt lock removal and createdAt-bearing live locks staying held. If createdAt-bearing pre-starttime Docker locks also need recovery, use a more specific design that proves the owner is from an older process incarnation instead of expiring all no-starttime live locks by mtime.

## Reproduction Assessment

Yes. The target bug is reproducible at function level with a live PID, no `starttime`, no `createdAtMs`, and stale lock mtimes; the PR regression is reproducible by adding `createdAtMs` to a no-starttime live-PID owner while keeping stale mtimes.

## Solution Assessment

No. The direction is plausible for true legacy records, but the current condition is broader than the stated lock shape and can remove live active install locks on platforms without starttime support.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep createdAt-bearing live locks held:** `src/plugins/bundled-runtime-deps-lock.ts:123`
  - body: Because this condition only checks for a missing `starttime`, the mtime fallback also runs for current locks that have `createdAtMs` but no `starttime` on platforms where `/proc` starttime is unavailable. After 10 minutes, a second process can delete a live install lock while the owner PID is still alive. Gate this fallback to the true legacy shape, or otherwise prove the lock is from an old incarnation, and add the createdAt-bearing regression case.
  - confidence: 0.9
