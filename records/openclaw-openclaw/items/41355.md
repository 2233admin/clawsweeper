## Summary

Keep open. Current main still lets Discord supply ThreadStarterBody on follow-up thread turns, and the prompt path can still prepend it as `[Thread starter - for context]`. The same-author PR #50522 remains an open implementation candidate with closing syntax, so maintainers should review, land, or supersede that PR before closing this issue.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review and land #50522 or an equivalent Discord adapter fix that only supplies ThreadStarterBody for the first thread session turn, then add focused Discord regression coverage for existing thread sessions and separately evaluate whether the generic session fallback should ignore ThreadStarterBody on non-new sessions.
