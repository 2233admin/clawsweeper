## Summary

Keep this PR open. Current main still does not implement the PR's central Docker `.env` handling: the root `docker-setup.sh` is only a shim, the active setup logic in `scripts/docker/setup.sh` only reads project `.env` for the gateway token fallback, `upsert_env` still rewrites managed keys as bare `KEY=value`, and the setup build still does not pass `OPENCLAW_INSTALL_BROWSER` despite Dockerfile support. The PR remains useful but needs to be rebased/ported to the active script with focused regression tests and shell-parser security review.

## Best Possible Solution

Keep the PR open and ask the author to rebase/port the fix onto `scripts/docker/setup.sh`. The best implementation is a small non-executing dotenv loader that applies project `.env` values only when the caller has not already set them, a sourceable `upsert_env` writer for whitespace/quotes/backslashes, and focused Docker setup e2e coverage for `.env` defaults, precedence, parser edge cases, and `OPENCLAW_INSTALL_BROWSER` build-arg propagation. Drop the unrelated `.gitignore` change unless maintainers explicitly want it in this PR.
