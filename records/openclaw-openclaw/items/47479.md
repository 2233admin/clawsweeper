## Summary

Keep open. The reported Docker `.env` handling problem still exists on current `main`, but this PR now patches the stale root entrypoint instead of the active Docker setup implementation, so it should not merge as-is.

## What This Changes

The PR branch modifies root `docker-setup.sh` to parse `.env` defaults, quote dotenv rewrites, pass `OPENCLAW_INSTALL_BROWSER` into Docker builds, and ignore `docker-compose.override.yml`.

## Best Possible Solution

Move the useful dotenv parsing, sourceable `.env` writing, caller-environment precedence, and browser build-arg wiring into `scripts/docker/setup.sh`, then add focused Docker setup e2e coverage for quoted package lists and browser build args.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence path: put `OPENCLAW_DOCKER_APT_PACKAGES="jq ripgrep"` or `OPENCLAW_INSTALL_BROWSER=1` in root `.env` and run the active setup path; those values are not generally loaded, and managed whitespace values are still rewritten bare.

## Solution Assessment

No. The PR addresses the right user problem, but the implementation is no longer on the maintained surface; the narrow maintainable fix is to update `scripts/docker/setup.sh` and its e2e tests.
