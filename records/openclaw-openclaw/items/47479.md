## Summary

Keep this PR open. Current main still has not implemented the central Docker `.env` handling changes: the active setup logic lives in `scripts/docker/setup.sh`, it computes Docker setup defaults from the process environment before any project `.env` load, only reads `.env` for the gateway token fallback, still rewrites managed `.env` keys as bare `KEY=value`, and still does not pass `OPENCLAW_INSTALL_BROWSER` through the setup build. The PR needs to be ported to the current script with focused regression tests and a careful shell-parser/security review.

## Best Possible Solution

Keep the PR open and ask the author to rebase/port the fix onto `scripts/docker/setup.sh`. The best implementation is a small non-executing dotenv loader that applies project `.env` values only when the caller has not already set them, a sourceable `upsert_env` writer for whitespace/quotes/backslashes, and focused e2e coverage for the Docker setup behavior. The already-present `.gitignore` change should be dropped from the PR during the port.

## Risks / Open Questions
