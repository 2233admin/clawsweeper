## Summary

Keep open: current main still applies symmetric jitter to a `retryAfterMs`-based delay and then clamps only to `minDelayMs`, so the reported lower-bound bug remains. The issue is also paired with open PR #68543 using closing syntax, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for `retryAsync` to preserve server-supplied `retryAfterMs` as a lower bound when applying jitter.

## Best Possible Solution

Review and land #68543 or an equivalent narrow patch that enforces the retry-after lower bound after jitter, adds deterministic regression coverage for jitter plus `retryAfterMs`, and audits the copied memory host retry helper for the same invariant.
