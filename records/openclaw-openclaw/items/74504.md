## Summary

Keep open. Current main still matches the reported Matrix/MAS cross-signing failure path, and same-author implementation PR #74509 is open, so this issue should stay open until that PR lands or is superseded.

## What This Changes

The issue requests Matrix cross-signing bootstrap support for MAS-fronted Synapse/PAT bots by honoring MSC3967 first-publish behavior, parsing UIA challenges, preserving UIA sessions, and surfacing the MAS cross-signing reset URL instead of suggesting password fallback.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses Matrix UIA 401 challenges, carries the UIA session, surfaces a typed MAS reset URL for operator approval, preserves the MSC3967 first-publish upload path, and updates focused crypto-bootstrap tests plus Matrix operator docs.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists by making the Matrix SDK auth callback reject no-auth and dummy auth without a configured password; current tests assert the same misleading password fallback error, and the issue provides a concrete live MAS 401 response body.

## Solution Assessment

Yes at the design level, but it is not implemented on current main. Handling this inside the Matrix plugin's existing cross-signing auth callback and MatrixClient capability probing is the narrowest maintainable fix; the open PR should be reviewed, landed, or replaced.
