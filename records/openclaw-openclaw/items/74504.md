## Summary

Keep open. Current main and the latest tagged release still match the reported Matrix/MAS failure path: the cross-signing UIA callback does not inspect UIA 401 bodies, does not thread the UIA session, and has no MAS cross-signing-reset handling; the same-author implementation PR #74509 remains open, so closing this issue before that PR lands or is superseded would be premature.

## What This Changes

The issue requests a Matrix plugin fix for MAS-fronted Synapse cross-signing bootstrap so PAT-authenticated bots can use MSC3967 first-publish, surface MAS reset URLs, preserve UIA sessions, and avoid misleading password-only recovery guidance.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned change that parses Matrix UIA 401 challenges, preserves the UIA session, surfaces a typed MAS reset URL for operator approval, keeps MSC3967 first-publish uploads available, and updates focused crypto-bootstrap tests plus Matrix operator docs.

## Reproduction Assessment

Yes. The code-level reproduction is high-confidence: make the Matrix SDK auth callback reject no-auth and dummy UIA with no configured password, and current tests assert the same misleading password fallback error; the issue also provides a concrete MAS 401 response shape from a live deployment.

## Solution Assessment

Yes at the design level, but it is not implemented on main yet. Handling this in the Matrix plugin through the existing cross-signing auth callback and a homeserver capability probe is the narrowest maintainable path; the open PR should be reviewed, landed, or superseded.
