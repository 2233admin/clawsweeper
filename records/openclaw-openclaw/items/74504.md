Keep open. Current main still matches the reported Matrix MAS cross-signing failure, and open PR #74509 is already the paired implementation path, so this issue should stay open for maintainer review, landing, or supersession rather than being closed independently.

## What This Changes

The issue requests Matrix cross-signing bootstrap support for MAS-fronted Synapse by preserving MSC3967 first-publish uploads, surfacing MAS reset UIA URLs, threading UIA sessions, and replacing the misleading password-only fallback.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses Matrix UIA 401 challenges, preserves the UIA `session`, surfaces a typed MAS reset URL for operator approval, keeps MSC3967 first-publish uploads available, and updates focused crypto-bootstrap tests plus Matrix verification docs.

Yes. Yes at the design level. Handling this in `extensions/matrix` through the existing matrix-js-sdk `authUploadDeviceSigningKeys` callback is the narrowest maintainable path; #74509 is the active implementation vehicle but still needs maintainer review and patch iteration before merge.
