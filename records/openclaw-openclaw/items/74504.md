## Summary

Keep open. Current main still has the reported Matrix cross-signing UIA behavior: the upload auth callback only tries no-auth, dummy, and password stages, does not inspect the UIA challenge body, and does not preserve the UIA session. A same-author companion PR, #74509, is open for this bug, so this issue should stay open until that PR or a replacement fix lands.

## What This Changes

The issue requests Matrix cross-signing bootstrap support for MAS-fronted Synapse/PAT bot accounts by honoring MSC3967 first-publish, parsing UIA stages and sessions, and surfacing the MAS reset URL.

## Best Possible Solution

Land a Matrix plugin fix that parses Matrix UIA challenge bodies, preserves and resends `session`, surfaces a structured MAS cross-signing reset URL, keeps the MSC3967 first-publish upload path unblocked, uses the existing guarded Matrix transport for any auth metadata probe, and updates focused tests plus Matrix operator docs. Then close this issue after #74509 or an equivalent replacement fix merges.
