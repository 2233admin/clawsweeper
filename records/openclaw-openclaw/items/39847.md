## Summary

Keep #39847 open. Current `main` still lacks a shared outbound metadata-stripping safety net: the outbound plan parses raw reply text before any `stripInboundMetadata()`/`sanitizeUserFacingText()` call, delivery only projects that plan plus optional adapter sanitizers, and the Discord adapter does not provide a metadata sanitizer. The common assistant-reply path is partly mitigated upstream, but direct/shared outbound planning can still preserve echoed `Conversation info` / `Sender` metadata.

## Best Possible Solution

Keep #39847 open as the canonical bug. Land one focused source-level fix in `src/infra/outbound/payloads.ts` that strips inbound metadata before directive parsing/formatting in the shared outbound plan, with regression coverage in `src/infra/outbound/payloads.test.ts` for `Conversation info` and `Sender` metadata while preserving MEDIA directives, reply tags, `audioAsVoice`, silent replies, presentation/interactive payloads, and `channelData`. Then close or supersede the duplicate open fix PRs against that landed fix.
