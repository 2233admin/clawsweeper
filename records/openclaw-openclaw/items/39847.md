## Summary

Keep this issue open. Current main still does not strip `Conversation info` / `Sender` inbound metadata in the shared outbound payload planner before directive parsing, and multiple open implementation PRs still name this issue as their closing target.

## Best Possible Solution

Keep this issue open as the canonical bug. Review and land one focused source-level fix in `src/infra/outbound/payloads.ts` that strips inbound metadata before `parseReplyDirectives()` in the shared outbound plan, with regression coverage in `src/infra/outbound/payloads.test.ts` for Conversation/Sender metadata plus preservation of media directives, reply tags, silent replies, `audioAsVoice`, presentation/interactive payloads, and `channelData`. Then close or supersede the duplicate open fix PRs with a maintainer rationale.
