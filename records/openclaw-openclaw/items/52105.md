## Summary

Keep this issue open. Current main still clears local slash-command input before dispatch, still returns silently when the Control UI has no browser client, and still awaits local slash-command execution without converting unexpected dispatch failures into an inline system message. The `/think` executor handles normal RPC failures, so the remaining bug surface is the Control UI dispatch/injection path, matching the discussion and the closed unmerged #52113.

## Best Possible Solution

Land a focused Control UI fix in `ui/src/ui/app-chat.ts`: when a local slash command cannot run because the browser client is unavailable, inject a visible system message; wrap unexpected `executeSlashCommand` failures so they render inline after input clearing instead of escaping to the console; keep `/think` executor/provider semantics unchanged; add `app-chat` regression tests for missing client and rejected local dispatch. #52113 is a useful reference, but the fix still needs to land on main with coverage.
