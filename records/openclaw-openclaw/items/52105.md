## Summary

Keep this issue open. Current main still routes `/think` through the Control UI local slash-command dispatch path, clears the composer before dispatch, returns silently when the browser client is missing, and awaits `executeSlashCommand` without a catch wrapper that would render unexpected dispatch failures inline. The executor itself handles normal `/think` RPC failures, so the remaining gap is still the Control UI dispatch/injection layer described in the discussion and in the closed, unmerged #52113.

## Best Possible Solution

Keep the issue open and land a focused Control UI dispatch fix in `ui/src/ui/app-chat.ts`: inject a visible system message when a local slash command cannot run because the browser client is unavailable, wrap unexpected `executeSlashCommand` failures so they render inline instead of escaping after input clearing, and add `app-chat` regression tests for missing client and rejected dispatch using `/think` or another local command. #52113 is a useful reference, but the fix still needs to land on main with coverage.
