## Summary

Keep this PR open. Current main still has the finite WhatsApp reconnect default and terminal max-attempt stop path the PR is trying to change, so the intended behavior is not implemented on main. The PR remains a plausible fix direction, but it must be refreshed for the current WhatsApp plugin layout under `extensions/whatsapp/src` rather than the old `src/web` paths.

## Best Possible Solution

Refresh this PR against the current WhatsApp plugin implementation. The likely maintainer path is to update `extensions/whatsapp/src/reconnect.ts`, `extensions/whatsapp/src/connection-controller.ts`, and `extensions/whatsapp/src/auto-reply/monitor.ts` so retryable disconnects do not permanently kill the monitor, while logged-out, session-conflict, and intentionally terminal auth-unstable states remain explicit. Add focused tests for default unlimited retries, bounded degraded-mode retry, non-retryable closes, and auth-unstable behavior, then align `web.reconnect.maxAttempts` schema/help/docs/generated metadata.
