## Summary

Keep this PR open. Current main still has the bounded WhatsApp reconnect default and terminal stop path this PR is trying to change, but the PR targets obsolete `src/web/*` files instead of the current WhatsApp plugin under `extensions/whatsapp/src/*`. The best path is a refresh against the current plugin implementation, not maintainer cleanup closure.

## Best Possible Solution

Refresh this PR against the current WhatsApp plugin. The likely maintainer path is to update `extensions/whatsapp/src/reconnect.ts`, `extensions/whatsapp/src/connection-controller.ts`, and `extensions/whatsapp/src/auto-reply/monitor.ts` so retryable disconnects do not permanently kill the monitor, while logged-out, session-conflict, and intentionally terminal auth-unstable states remain explicit. Add focused tests for default unlimited retries, bounded degraded-mode retry, non-retryable closes, and auth-unstable behavior, then align `web.reconnect.maxAttempts` schema/help/docs/generated metadata.
