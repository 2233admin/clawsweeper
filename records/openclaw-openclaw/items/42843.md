## Summary

Keep this PR open. Current main still has the bounded WhatsApp reconnect default and terminal stop path this PR is trying to change, and the PR’s code now targets old `src/web/*` paths instead of the current WhatsApp plugin implementation under `extensions/whatsapp/src/*`. The best outcome is a refresh against the current plugin layout, not closure.

## Best Possible Solution

Refresh this PR against the current WhatsApp plugin. The likely maintainer path is to update `extensions/whatsapp/src/reconnect.ts`, `extensions/whatsapp/src/connection-controller.ts`, and `extensions/whatsapp/src/auto-reply/monitor.ts` so retryable disconnects do not permanently kill the monitor, while logged-out, session-conflict, and intentionally terminal auth-unstable states remain explicit. Add focused tests for default unlimited retries, bounded degraded-mode retry, non-retryable closes, and auth-unstable behavior, then align the `web.reconnect.maxAttempts` schema/help/docs/generated metadata.
