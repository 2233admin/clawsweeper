## Summary

Current main still leaves the reported `/model` round-trip bug open. The no-argument Control UI `/model` command prints `Available:` from bare `ModelCatalogEntry.id` values, while the backend parser treats slash-containing input as `provider/model` and only infers a provider for inputs without `/`. The chat picker has provider-qualified helpers, and related PR #47249 directly targeted this issue but is closed unmerged.

## Best Possible Solution

Keep this issue open and land a focused Control UI fix. Format `Current model` and every `/model` `Available:` entry as the canonical value accepted by `sessions.patch`, using the catalog provider plus model id for nested ids. Keep the existing backend bare-id inference for compatibility and add regression tests for copy-pasting an advertised value back into `/model`.
