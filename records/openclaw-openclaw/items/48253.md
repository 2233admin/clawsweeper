## Summary

Current main still does not expose the requested host-owned Memory Plugin API for file scan/watch/change events. The memory slot/runtime API, memory-host helper exports, and bundled memory plugins exist, but startup scan, chokidar watching, hash-based incremental sync, and stale cleanup remain private to memory-core. The linked memory-lancedb-pro extraPaths request is useful evidence that external memory plugins need this surface, not evidence that core can close this item yet.

## Best Possible Solution

Keep this open for maintainer design of an additive memory file-indexing capability: host-owned scan/watch/change delivery for the active memory plugin, built on the existing list/chunk helpers where possible, with memory-core dogfooding, public SDK/docs, and contract tests before third-party memory plugins depend on it.
