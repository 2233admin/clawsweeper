## Summary

Keep this PR open. Current main still lacks a native Windows no-Docker/WSL host-hardening fallback guide, and the contribution is a focused docs-only attempt backed by an external Windows reference implementation. It should not be auto-closed, but it needs maintainer/secops review because it documents security-sensitive PowerShell ACL/runas/rollback procedures and the latest PR head still has at least one verification concern plus rebase drift against current Windows and docs navigation content.

## What This Changes

This PR adds a new Windows no-Docker/WSL hardening guide and cross-links it from the Windows platform page, sandboxing docs, and docs navigation.

## Best Possible Solution

Have a docs/secops reviewer decide whether OpenClaw core docs should carry this native Windows fallback.
