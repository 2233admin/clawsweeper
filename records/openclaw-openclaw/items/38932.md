## What This Changes

The PR adds a new Windows host-hardening fallback guide for native Windows without Docker/WSL, registers it in docs navigation, and cross-links it from sandboxing and Windows platform docs.

## Best Possible Solution

Land a rebased, docs-only version only after the verification section tests a user-chosen blocked sibling path from the ACL step and the page is integrated with the current Windows, sandboxing, and security docs without presenting the fallback as equivalent to Docker/WSL sandboxing.

## Reproduction Assessment

Not applicable as a runtime bug reproduction for the requested docs addition. The remaining review finding is high-confidence from static inspection: the PR's forbidden-path test targets `$env:USERPROFILE` instead of a denied sibling path introduced by the guide's ACL step.

## Solution Assessment

Unclear as-is. A dedicated fallback page is a reasonable docs direction for native-Windows users who cannot use Docker/WSL, but the current PR needs the forbidden-path check fixed and a maintainer security-doc review before it is the narrowest maintainable solution.
