## Summary

Keep this PR open. Current main still lacks the native Windows no-Docker/WSL host-hardening guide this PR proposes, and the existing Windows, sandboxing, security, and daemon surfaces do not provide an equivalent dedicated-user plus NTFS ACL fallback. The change is docs-only, but it is security-adjacent and still needs maintainer/secops review plus native Windows validation of the PowerShell and ACL examples before merge.

## What This Changes

This PR adds a new Windows no-Docker/WSL hardening guide plus docs navigation and Windows/sandboxing cross-links for native Windows users, centered on a dedicated low-privilege user, dedicated state/workspace paths, NTFS ACL allowlists, verification, and rollback steps.

## Best Possible Solution

Keep this PR open for maintainer docs and secops review. Rebase it onto current main, keep WSL2/Docker/tool sandboxing as the stronger recommendation, fix or explicitly revalidate the PowerShell runas, ACL grant/deny, forbidden-path, and rollback examples on native Windows, then land the fallback guide and cross-links only if secops agrees with the threat-model framing.
