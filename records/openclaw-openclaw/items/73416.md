## Summary

Keep this PR open. Current main still gates `createGoogleThinkingPayloadWrapper` on only `model.api === "google-generative-ai"`, while existing Google CLI and Vertex-related model resolution can produce `google-gemini-cli` models that bypass that guard. The linked regression issue remains open and this PR is a narrow implementation candidate, not cleanup-close material.

## Best Possible Solution

Keep the PR open as a focused bug-fix candidate for the open Google Vertex/Gemini CLI regression. Best path is to revise the misleading test comment, confirm the Vertex API-shape assumption, keep the non-Google passthrough test, run the targeted wrapper tests, and then have maintainers land this or an equivalent guard widening on main.
