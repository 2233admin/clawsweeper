## Summary

Keep open. Current main still has the narrow Google thinking payload guard, while this PR is a focused, plausible fix for the Vertex/Gemini CLI bypass path and the earlier comment-scope nit has been addressed in the latest diff.

## What This Changes

This PR widens the Google thinking payload wrapper guard to cover `google-generative-ai`, `google-vertex`, and `google-gemini-cli`, and adds wrapper regression tests for those API shapes plus non-Google passthrough.

## Best Possible Solution

Land this guard widening, or an equivalent main-branch patch, after normal maintainer review and CI confirmation; keep #38327 open until a fix merges, and handle any remaining gaxios/native-fetch failure as a separate dependency-patch decision.

## Reproduction Assessment

Yes. The focused reproduction is to run the wrapper with `thinkingConfig: { thinkingBudget: -1 }` and `model.api` set to `google-vertex` or `google-gemini-cli`; current main leaves the config intact, while the PR's added tests assert it is stripped.

## Solution Assessment

Yes. Widening the existing Google-family guard is the narrowest maintainable fix for the stated bypass, preserves non-Google passthrough, and avoids mixing in the separate gaxios dependency patch.
