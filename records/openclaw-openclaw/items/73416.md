## Summary

Keep this PR open. Current main has not implemented the PR's central change: `createGoogleThinkingPayloadWrapper` still sanitizes only when `model.api === "google-generative-ai"`, while current Google CLI/Vertex-related model resolution can produce non-`google-generative-ai` shapes. The PR is a focused implementation candidate for open regression #38327, not cleanup-close material.

## Best Possible Solution

Keep the PR open as a focused bug-fix candidate for #38327. Best path is to revise the misleading test comment, confirm whether `google-vertex` is a current or imminent `model.api` value, keep the non-Google negative case, run the targeted wrapper tests, and then have maintainers review/land or explicitly supersede it with an equivalent guard widening on main.
