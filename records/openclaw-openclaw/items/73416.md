## Summary

Keep open. Current main still gates `createGoogleThinkingPayloadWrapper` on only `google-generative-ai`, while current Google provider code has `google-vertex` and `google-gemini-cli` API shapes that can reach the same shared wrapper path. The PR is a focused, plausible fix for the #38327 regression; it needs maintainer review, a small test-comment clarification, rebase over current main, and targeted validation rather than cleanup closure.

## What This Changes

This PR widens the Google thinking payload wrapper guard to include `google-generative-ai`, `google-vertex`, and `google-gemini-cli`, and adds wrapper regression tests for those Google API shapes plus non-Google passthrough.

## Best Possible Solution

Land this guard widening, or an equivalent main-branch patch, after correcting the scope comment, rebasing over current main, and rerunning the focused Google wrapper and extra-params tests; keep #38327 as the canonical user regression until a fix merges.
