## Summary

Keep open. Current main does not implement the PR's central Debug page event-log scroll behavior: scroll utilities and lifecycle scheduling cover chat/log streams only, the Debug event log has no scroll handler or sticky-at-bottom state, and incoming gateway events are still prepended to the shared event buffer. The related #69520 work is chat-scroll focused and does not supersede this Debug event-log fix. The PR discussion also shows a focused review concern was addressed in commit 7b4232285f7fa150f87d6ce0702a6cc94113210b, so this remains useful maintainer-reviewable UI work rather than cleanup-closeable.

## Best Possible Solution

Keep #70105 open for maintainer review or adaptation. The best product fix belongs in the Control UI: add Debug event-log scroll state and a scroll handler, schedule auto-follow only on Debug tab entry or when the viewer is already at the bottom, make the Debug event-log container independently scrollable, preserve Overview's recent-first presentation, and keep focused `ui/src/ui/app-scroll.test.ts` coverage for released and sticky-at-bottom behavior.
