## Summary

Keep #70541 open. Current main still has the reported inconsistent post-compaction path: `runReplyAgent` reads post-compaction context from `process.cwd()` after an auto-compaction count, while the preflight compaction helper already uses `followupRun.run.workspaceDir`. The public config docs describe post-compaction/heartbeat context rebuild behavior, so this remains a plausible product bug rather than a stale or duplicate report.

## Best Possible Solution

Fix `src/auto-reply/reply/agent-runner.ts` to pass `followupRun.run.workspaceDir` into `readPostCompactionContext` in the `autoCompactionCount > 0` hook, preserving the existing agentId options and adding a targeted regression test for the auto-compaction branch where cwd and run workspace differ.
