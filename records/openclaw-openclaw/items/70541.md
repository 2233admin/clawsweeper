## Summary

Keep this issue open. Current main still has the reported inconsistent post-compaction caller: the auto-compaction system-event hook passes `process.cwd()` to `readPostCompactionContext`, while the adjacent preflight compaction refresh uses the queued run's required `workspaceDir`. The report is narrow, source-confirmed, and suitable for one focused regression-fix PR.

## Best Possible Solution

Fix the auto-compaction system-event hook in `src/auto-reply/reply/agent-runner.ts` to call `readPostCompactionContext` with `followupRun.run.workspaceDir` instead of `process.cwd()`, preserving the existing enqueue behavior and agent-id handling. Add a focused regression test where `process.cwd()` differs from the queued run workspace and only the workspace `AGENTS.md` content is injected.
