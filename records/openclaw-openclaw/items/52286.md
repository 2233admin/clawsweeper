## Summary

Current main still has the reported Telegram forum-topic routing gap. Topic context is preserved when the message tool targets the current Telegram chat or omits a target, but an explicit positive numeric Telegram user ID is kept as the outbound target, parsed as a direct chat, and does not inherit the originating supergroup chat_id plus topic thread_id.

## Best Possible Solution

Keep this issue open as the canonical Telegram forum-topic file/media routing bug. Implement a deterministic current-conversation attachment path or a Telegram-specific message-tool guard that maps accidental current sender user-id targets back to the originating group chat plus MessageThreadId, while preserving an explicit way to intentionally DM a user. Add focused regression coverage for same-topic and sender-user-id cases.
