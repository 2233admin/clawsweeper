## Summary

Keep open. Current main covers the direct Block Kit slices: authored Slack blocks, message-tool blocks, presentation blocks, and opt-in interactive controls. The issue discussion still tracks richer Slack output beyond raw block passthrough, and current main still sends ordinary markdown replies through the text/mrkdwn path while coercing `markdown.tables: "block"` back to code rendering, so the table/rich-output umbrella is not fully resolved.

## What This Changes

The issue requests richer Slack agent messages through Block Kit payloads, including structured sections, context, dividers, buttons, selects, and better table-style output.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until native markdown-table Block Kit delivery lands or maintainers explicitly split that scope out; leave broader automatic Slack `rich_text` conversion to the dedicated RFC while preserving the shipped raw-block and interactive-directive paths.
