## Summary

Keep open. Current main supports authored Slack Block Kit payloads and opt-in interactive reply controls, but the umbrella request is not complete because ordinary markdown replies still use mrkdwn text chunking and `markdown.tables: "block"` is still coerced back to code rendering; the discussion also identifies #57717 and #57708 as active remaining follow-ups.

## What This Changes

The issue requests richer Slack agent messages using Block Kit payloads, including structured sections, tables, buttons, selects, and context metadata.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until the native markdown-table Block Kit send path lands or maintainers explicitly split that scope out, while leaving broader rich_text conversion design in #57708 and preserving the shipped raw-block and interactive-directive support.
