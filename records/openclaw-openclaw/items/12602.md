## Summary

Keep this issue open. Current main supports authored Slack Block Kit payloads, presentation blocks, and opt-in interactive reply directives, but the ordinary Slack markdown reply path still renders text chunks and `markdown.tables: "block"` is still coerced back to `code`. The latest maintainer review and open follow-ups #57717 and #57708 also point to remaining Slack richer-output work rather than cleanup closure.

## What This Changes

Allow Slack agents to compose and send richer Block Kit messages, including structured sections, dividers, buttons, selects, context metadata, and table-style outputs instead of plain text-only replies.

## Best Possible Solution

Keep this issue open as the Slack richer-output umbrella. Land or replace #57717 for markdown-table Block Kit delivery in the bundled Slack plugin, keep broader Markdown-to-Slack rich_text conversion tracked under #57708, preserve safe text/code fallbacks, and document the public Slack behavior at https://docs.openclaw.ai/channels/slack when the user-visible path changes.
