## Summary

Keep this issue open. Current main still has a single cron `delivery` contract (`none`, `announce`, `webhook`) and no deterministic `output.targets` contract for file writes, channel fan-out, or named/main-session result injection independent of prompt behavior. The prior ClawSweeper comment reached the same conclusion, and current `main` still matches that shape.

## Best Possible Solution

Keep this open for maintainer design. The best path is an additive cron output contract, likely `output.targets` or a versioned extension of `delivery`, covering validated file append/write targets, channel/webhook delivery, explicit main/named-session context injection, silent/discard mode, per-target run-log status, idempotency, mixed-failure behavior, CLI/Control UI editing, protocol schema, docs, and focused runtime tests.
