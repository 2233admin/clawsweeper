## Summary

Keep this PR open. Current main still has the Slack `download-file` schema gap: Slack advertises `download-file` and requires `fileId`, but `describeSlackMessageTool` contributes no `fileId` schema fragment. The PR is directionally valid, but it needs changes before merge because the new friendly guard throws a plain `Error` instead of preserving `ToolInputError` input-error semantics, and the model-visible action lists in the error/schema copy drift from the actual Slack actions. The security pass found no CI, dependency, lockfile, package-script, publishing, or third-party code-execution changes; the diff is limited to Slack source and tests.

## Best Possible Solution

Keep the PR open and revise it into the canonical Slack-owned fix: add action-scoped Slack `fileId`/`messageId` schema contributions, keep the targeted `download-file` misuse hint as a `ToolInputError` or equivalent input error, align all model-visible action lists with actual Slack discovery, and then land one fix that can close or supersede the older #45574/#45626 tracking pair.
