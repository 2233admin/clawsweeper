## Summary

Keep this PR open. Current main still has the Slack `download-file` schema gap: Slack advertises `download-file` and the dispatcher requires `fileId`, but `describeSlackMessageTool` returns only actions/capabilities and contributes no `fileId` schema. The PR is a plausible plugin-owned fix, but it is not ready to merge because CI reports lint failures in `message-tool-api.ts`, and the model-visible action copy needs to be aligned with Slack's actual supported messageId actions before landing. Security review found no supply-chain, workflow, dependency, secret, or package-publishing changes in the PR diff.

## Best Possible Solution

Keep the PR open, fix the lint/style failures, then align the schema descriptions and friendly error text with the actual Slack messageId actions before review. Prefer the plugin-owned `describeMessageTool` schema contribution approach here over a generic global `fileId` field, then close or supersede the older related #45574/#45626 pair once one canonical fix lands.
