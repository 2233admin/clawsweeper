## Summary

Keep open. Current main still lacks this PR's targeted tolerance for transient pre-hello clean WebSocket closes in both the one-shot gateway call wrapper and the GatewayClient connect path. The PR branch remains the canonical active implementation candidate per maintainer follow-up, and it also addresses the Greptile review by switching from string matching to a structured close error.

## Best Possible Solution

Review, rebase if needed, and land this PR as the focused canonical fix if validation remains green. Keep the structured `GatewayClientCloseError` approach and focused positive/negative regression tests, then resolve #54774 as duplicate or superseded and leave the broader cron quiet/retry scope in #49109 for separate maintainer judgment.
