## Summary

Keep open: current main still has the Gateway reasoning gap, but this PR is not mergeable as-is because the branch mixes the valid Gateway fix with unrelated scheduled-reset/session-memory changes and introduces blocking correctness plus trust-boundary regressions.

## What This Changes

This PR changes embedded Pi reasoning stream gating, replaces the commands agent re-export with a copied implementation that forwards session reasoning, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused Gateway reasoning fix in the owner modules: pass stored `sessionEntry.reasoningLevel` into `runEmbeddedPiAgent()`, allow callback-less Gateway `thinking` agent events while preserving reasoning visibility and silent-reply gates, and split scheduled-reset/session-memory behavior into a separate reviewed change.
