## Summary

Keep this PR open. Current main still has the core no-callback reasoning-stream gap for default Gateway WebSocket chat sends, so the intended gateway fix is not implemented on main. The PR also is not safe to merge as-is because it mixes the gateway fix with unrelated command/session-memory changes that touch trust boundaries and have review-identified regressions.

## Best Possible Solution

Keep the PR open for now, but the best path is a refreshed, narrowly scoped gateway fix: decouple `streamReasoning` and `emitReasoningStream()` from the channel callback, preserve the current agent-command ingress guards, add regression coverage for default `chat.send` WebSocket clients receiving `thinking` agent events after `sessions.patch({ reasoningLevel: "stream" })`, and split or drop the unrelated scheduled-reset/session-memory changes. The narrower open reasoning-stream work in #47613 is a useful comparison point for the gateway-only patch.
