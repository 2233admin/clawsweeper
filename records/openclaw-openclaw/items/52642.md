Keep this PR open for maintainer handling. Current main still has the no-callback reasoning-stream gap in `subscribeEmbeddedPiSession`, so this cannot close as implemented. The branch should not be merged as-is: it mixes the gateway reasoning fix with unrelated session-memory changes and replaces the current agent-command wrapper with a large copied implementation that drops current trust-boundary checks.

## Best Possible Solution

Keep this PR open for maintainer action, but do not merge the branch as-is. Any session-memory/reset-hook work should be split and reviewed separately, and related #47613 should be considered as the narrower gateway fix candidate.
