## Summary

Keep open: the gateway reasoning bug is still present on current main, but this PR is not mergeable as-is. It mixes the focused WebSocket reasoning fix with unrelated scheduled-reset/session-memory work and introduces compile/runtime, model-selection, authorization, and reasoning-visibility regressions already visible from the diff and current main comparisons.

## What This Changes

This PR replaces the `src/commands/agent.ts` barrel with a copied agent command implementation that forwards stored session reasoning, relaxes embedded Pi reasoning stream callback gating, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused gateway reasoning fix in the owner modules: forward stored `sessionEntry.reasoningLevel` into `runEmbeddedPiAgent()`, allow callback-less `emitAgentEvent({ stream: "thinking" })` for WebSocket clients while preserving `canShowReasoning` and silent-reply gates, and add targeted embedded-subscribe plus gateway chat coverage. Keep scheduled-reset/session-memory changes split from this PR.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to set a WebSocket session to `reasoningLevel: "stream"` via `sessions.patch` and then send through `chat.send`; current main stores the setting but neither forwards it into the embedded run nor supplies `onReasoningStream` from the gateway chat path.

## Solution Assessment

No. The PR addresses the right user-visible gap, but this is not the best fix because it copies a large command implementation, loses current model/ingress safeguards, weakens reasoning visibility gates, and bundles unrelated session-memory work; a small patch in `src/agents/command/attempt-execution.ts` and `src/agents/pi-embedded-subscribe.ts` is safer.
