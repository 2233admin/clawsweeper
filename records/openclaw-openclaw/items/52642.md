## Summary

Keep open for maintainer review: current main still appears to have the reported gateway reasoning gap, but this branch is not mergeable because it mixes unrelated session-memory work and introduces clear P1/P2 regressions in module loading, per-run model overrides, and ingress trust-boundary checks.

## What This Changes

This PR changes embedded Pi reasoning stream gating, replaces the `src/commands/agent.ts` barrel with a copied agent-command implementation that forwards session reasoning, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused gateway reasoning fix in the owner modules: pass stored `reasoningLevel` from the session entry into the embedded runner, allow callback-less gateway `thinking` agent events while preserving `silentExpected` and `canShowReasoning`, add targeted gateway/subscriber coverage, and keep the #43524 session-memory work separate.
