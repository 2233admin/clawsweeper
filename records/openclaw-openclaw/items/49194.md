## Summary

Keep this PR open. Current main still does not implement the intended Google Vertex ADC fix: ADC resolves to the non-secret `gcp-vertex-credentials` marker, but runtime paths still classify and store that marker as `api-key` auth, and the Google transport serializes any non-JSON string as `x-goog-api-key`. The PR also targets the old `<authenticated>` sentinel and includes an unrelated `skills/bird` addition, so it needs a cleaned rebase rather than closure as implemented.

## Best Possible Solution

Keep this PR open as a candidate, but ask for a cleaned rebase onto current main: remove `skills/bird`, update the fix for the current `gcp-vertex-credentials` marker flow, represent Google Vertex ADC as provider-owned OAuth/no literal API key before runtime/provider calls, and add targeted tests covering main chat, compaction, simple completion, and Google transport headers.
