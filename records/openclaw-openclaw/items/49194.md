## Summary

Keep this PR open. Current main does not clearly implement the PR’s intended Google Vertex ADC fix: the old `"<authenticated>"` sentinel has been replaced by OpenClaw’s `gcp-vertex-credentials` marker, but the current runtime path still treats that marker as an `api-key` credential and can store it into pi runtime auth. The PR also contains an unrelated bundled `bird` skill addition, which should be split or removed before any focused auth fix is considered.

## Best Possible Solution

Keep this PR open only as a candidate for a cleaned, current-main fix. The best path is to remove the unrelated `skills/bird` change, update the Google Vertex ADC fix for the current `gcp-vertex-credentials` marker flow, ensure ADC is represented as provider-owned OAuth/no literal API key before pi/provider transport calls, and add focused regression tests covering main run and compaction/provider-call behavior. If maintainers consider the linked bug already closed incorrectly, reopen or file a narrow current-main follow-up instead of closing this PR as implemented.
