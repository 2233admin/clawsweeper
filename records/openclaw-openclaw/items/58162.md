## Summary

Keep #58162 open. Current main still does not include default redaction patterns or regression tests for standalone `AKID`, `LTAI`, `hf_`, or `r8_` credentials, and `logs.tail` consumes the shared default redaction pipeline. The PR remains a plausible focused security hardening change, with the final provided patch addressing the earlier mixed-case Tencent review concern, but the secret-scanning notice in the PR discussion needs maintainer verification before merge.

## Best Possible Solution

Keep the PR open. The best path is to land or reapply the focused default-redaction change on main with regression tests for standalone `AKID`, `LTAI`, `hf_`, and `r8_` values, preserve the vendor-neutral production comment style, and resolve the secret-scanning/rotation concern before merge.
