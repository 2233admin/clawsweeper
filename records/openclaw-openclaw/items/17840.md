## Summary

Keep open. Current main still has passive reaction notification plumbing and no documented opt-in mode that wakes agent turns on reactions. The existing issue remains the canonical cross-channel tracker; #56708 and #11747 are closed overlaps, while #73681 and #38714 cover narrower Google Chat and Hooks gaps.

## Best Possible Solution

Keep this issue open as the canonical cross-channel product request. Implement a documented opt-in reaction wake mode, such as reactionTriggersTurn: "off" | "own" | "all" | "allowlist" or a narrowly scoped per-message awaitReaction flag, while preserving the current passive default. The implementation should enqueue the reaction event, then request a scoped heartbeat or agent turn only after the existing reaction notification filters and channel authorization checks pass; it should update config schemas/generated metadata, channel docs, and focused tests for the bundled reaction-capable channels.
