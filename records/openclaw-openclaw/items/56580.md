## Summary

Keep open. Current main still models Discord native slash commands with only default names/descriptions and does not expose or pass `description_localizations` through to Carbon/Discord deploy, so the reported localization drift remains actionable.

## Best Possible Solution

Keep this issue open and implement localization as an additive command-contract change: add typed localization metadata to native command definitions/specs, propagate it through built-in/plugin/skill command generation, set Carbon `descriptionLocalizations` on Discord command objects, include option localization where needed, and add regression tests for serialization plus redeploy/reconcile behavior.
