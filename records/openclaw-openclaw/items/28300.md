## Summary

Keep open. Current main implements a narrower Control UI theming model: Claw/Knot/Dash plus one browser-local tweakcn custom-theme import slot. It does not implement the requested six-preset gallery, hue-based generator, gradient controls, grouped editor, or per-variable override/reset studio, and the related discussion plus open #49788 show this is still an active product/design decision rather than conservative cleanup.

## What This Changes

The issue requests a Control UI theme customization system with six preset themes, a Custom Theme Studio, hue-generated OKLCH variables, dark/light controls, gradient toggles, per-variable overrides, persistence, and i18n coverage.

## Best Possible Solution

Define the intended Control UI theming model first. If approved, implement it consistently across the closed theme contract, CSS token families, browser-local persistence, Appearance and Quick Settings UI, generated/i18n copy, public docs, and focused UI regression tests while coordinating the narrower preset request in #49788.
