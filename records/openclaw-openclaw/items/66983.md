## Summary

Keep this issue open. Current main still lacks first-class browser-tab A2UI canvas node support: `web` is not an explicit node platform, device-less `role: "node"` sessions still reject even with shared auth, browser-origin node auto-approval is excluded, and #68497 is a narrower browser action-bridge request rather than a replacement for node registration/pairing/auth design.

## Best Possible Solution

Keep this open as the canonical design request for browser tabs acting as A2UI canvas node surfaces. A good path is an explicit `web` node platform policy with narrow canvas/A2UI commands, a secure browser node auth/pairing model for reverse-proxied deployments, regression tests for browser Origin and node-role auth behavior, and public docs for the supported web canvas-node pattern. Coordinate with #68497 for action delivery, but do not treat it as a replacement.
