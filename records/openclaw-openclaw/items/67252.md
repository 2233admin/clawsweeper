## Summary

Keep open. Current main still clears Feishu document content with one non-paginated `documentBlock.list` call before `feishu_doc` write inserts replacement content, while the bundled tool skill documents write as replacing the entire document. EronFan's comment and external fork PR are useful prior art, but the fix is not present in `openclaw/openclaw` main.

## Best Possible Solution

Keep this issue open and land a focused Feishu plugin fix in `extensions/feishu/src/docx.ts`: paginate `client.docx.documentBlock.list` inside `clearDocumentContent` until no next `page_token` remains, delete the full top-level child range, and add a regression test where `feishu_doc` write receives multiple list pages and clears all old top-level blocks before inserting replacement content. EronFan's comment and external fork PR can be used as prior art after review.
