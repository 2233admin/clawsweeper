## Summary

Keep open. Current main still clears Feishu Docx content with a single non-paginated block-list request before inserting replacement content, so the reported large-document write bug remains valid; the latest tagged release has the same clear helper, and the related external fork PR is open but not merged into this repository.

## What This Changes

The issue requests that `feishu_doc` `write` fully replace large Feishu documents by paginating `clearDocumentContent` before deleting existing top-level blocks.

## Best Possible Solution

Land a focused Feishu plugin fix in `extensions/feishu/src/docx.ts` that paginates `client.docx.documentBlock.list` inside `clearDocumentContent` until no next page token remains, deletes the full top-level range, and adds a regression test where `feishu_doc` write receives multiple block-list pages before inserting replacement content.
