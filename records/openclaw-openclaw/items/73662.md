## Summary

Keep open. Current main still uses the live `channels.status` payload only for channel issue overlays, while the visible `openclaw status` Channels rows come from read-only config/plugin discovery. This PR directly targets open #73525 by adding gateway-snapshot row synthesis and regression coverage, and the diff does not show security or supply-chain sensitive side changes.

## Best Possible Solution

Review this PR as the likely implementation path for #73525: make `openclaw status` augment or populate visible Channels rows from the reachable `channels.status` account snapshot, keep the current read-only discovery table as the unreachable-gateway fallback, and retain the proposed regression coverage that proves live Telegram account rows flow into `result.channels.rows`.
