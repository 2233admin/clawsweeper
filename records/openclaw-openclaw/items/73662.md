## Summary

Keep this PR open: current main still lacks the gateway-snapshot row synthesis, but the proposed wiring does not actually fix the reported plain `openclaw status` path because that caller still disables live channel status and setup fallback by default.

## What This Changes

The PR adds a helper to synthesize missing Channels table rows from `channels.status.channelAccounts`, wires it into `collectStatusScanOverview`, adds focused tests, and records the fix in the changelog.

## Best Possible Solution

Adjust the PR so the command that users reported, plain `openclaw status`, gets a populated Channels table while preserving the fast-status contract. The narrowest maintainable path is either to make read-only/setup discovery reliably produce configured channel rows for the default path, or explicitly allow a bounded gateway `channels.status` row source there and cover it with a `scanStatus({ json: false })` regression test.

## Reproduction Assessment

Yes. #73525 gives concrete default-command steps, and the PR head source shows the default `scanStatus` caller still disables the only live payload that the new merge helper needs.

## Solution Assessment

No. The helper is a reasonable augmentation for `--all`, `--deep`, or direct overview callers, but the PR as wired misses the plain `openclaw status` path that the bug report and changelog claim to fix.
