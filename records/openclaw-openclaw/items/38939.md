## Summary

Keep open. Current main still lacks cross-account Feishu group mention arbitration for ordinary inbound dispatch, and this PR is useful but not merge-ready because review context identifies a hard-coded coordinator account and substring fallback that can route messages incorrectly.

## What This Changes

The PR branch adds Feishu group mention extraction and arbitration across bot accounts, modifies inbound receive dispatch, extends Feishu mention tests, and adds a short design note.

## Best Possible Solution

Implement a narrow Feishu-owned arbitration layer in the receive path that collects structured mentioned bot open_ids, selects the coordinator from configured default/routing account state, preserves existing ordinary and broadcast dedupe semantics, and covers the specialist-only, coordinator-plus-specialist, and multi-specialist group cases with regression tests.
