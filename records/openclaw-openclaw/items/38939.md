## Summary

Keep this PR open. Current main still lacks Feishu dispatch-time arbitration across sibling bot accounts, and the PR is useful but should not merge unchanged because it hardcodes `main` as the coordinator and adds an untested substring-based text mention fallback.

## What This Changes

The PR branch adds Feishu group mention arbitration in the inbound receive path, refactors mention extraction for text/post events, adds regression tests, and adds a short design note.

## Best Possible Solution

Keep this PR open and request changes, or create a narrow replacement PR. The corrected implementation should arbitrate Feishu group events before dispatch using structured mentioned bot open_ids from text/post payloads, derive the coordinator from configured Feishu routing/default account rather than a literal `main`, avoid substring-only implicit mention matching, preserve existing account-scoped ordinary dedupe and broadcast dedupe, and add focused regression tests for specialist-only, coordinator-plus-specialist, and multi-specialist mentions.
