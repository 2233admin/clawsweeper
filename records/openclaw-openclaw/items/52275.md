## Summary

Keep open. Current main still lacks the legacy queue canonicalization this PR is trying to add, but the PR should not merge as-is because it introduces a malformed gateway startup block, calls undefined readiness helpers, and normalizes queued delivery records in a way that would drop current-main replay metadata.

## What This Changes

The PR adds legacy outbound delivery-queue normalization and regression tests for preserved mirror/media metadata, plus unrelated gateway agent-create runtime readiness refresh changes.

## Best Possible Solution

Land a narrow outbound storage/recovery change: normalize legacy queue files on load and before failure bookkeeping, preserve every queued replay option used by recovery, add focused legacy-file regression tests, and remove the gateway readiness/secrets refresh changes or move them to a separate gateway-owned PR.

## Reproduction Assessment

Yes. A legacy queue JSON containing payload/target/attempt/createdAt can be checked against current main by following loadPendingDeliveries or failDelivery; current source casts the raw JSON and only backfills lastAttemptAt, so the legacy fields are not canonicalized before replay or retry bookkeeping.

## Solution Assessment

No. The storage normalizer is the right boundary, but this PR is not the best mergeable implementation because it mixes unrelated gateway runtime changes, has undefined readiness APIs, and would drop current queued replay metadata.

  - body: The inserted block leaves refreshRuntimeConfigFromDisk inside the secrets reload error branch and later places a bare return false inside the startGatewaySidecars call. This makes server.impl.ts structurally invalid, so the gateway cannot parse or start with this patch.
  - confidence: 0.98

  - body: agents.create now calls waitForAgentReady and resolveAgentCreateReadyTimeoutMs, but the diff does not define or import those symbols or add the context contract used by refreshRuntimeConfigFromDisk. The handler would fail type checking and throw before returning a create response.
  - confidence: 0.96
- **[P1] Preserve all queued replay options during normalization:** `src/infra/outbound/delivery-queue-storage.ts:213-226`
  - body: normalizeQueuedDelivery rebuilds queued records without replyToMode, formatting, session, or gatewayClientScopes. Because failDelivery normalizes and rewrites entries, a retry can silently drop reply formatting, media/session policy, and caller scopes before recovery replays the delivery.
  - confidence: 0.93
- **[P2] Add the missing readiness test mocks:** `src/gateway/server-methods/agents-mutate.test.ts:307-328`
  - body: The added tests call mocks.refreshRuntimeConfigFromDisk and mocks.state.*, but those fields are not added to the hoisted mocks or passed through makeCall. The new tests will fail before exercising the readiness behavior they are meant to cover.
  - confidence: 0.9
