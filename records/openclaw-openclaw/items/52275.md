## Summary

Keep open. Current main preserves the relevant metadata for newly enqueued delivery records and forwards it during recovery, but it still does not implement this PR's central legacy queue normalization: old `target`/`payload`/`attempt`/`createdAt` records are parsed as-is, and the only legacy migration today is `lastAttemptAt` backfill. The PR should not be closed as implemented, and the provided PR diff also has unrelated, high-blast-radius gateway changes with unresolved review findings, so it needs cleanup/rebase before maintainer review or landing.

## Best Possible Solution

Keep this PR open for maintainer/contributor cleanup: split out the unrelated gateway readiness changes, rebase onto current main, implement focused legacy queue canonicalization in `src/infra/outbound/delivery-queue-storage.ts`, preserve all recovery metadata including `forceDocument`, `mirror.idempotencyKey` and group context, `session`, and `gatewayClientScopes`, and add storage/recovery regression tests for legacy queue files before review.
