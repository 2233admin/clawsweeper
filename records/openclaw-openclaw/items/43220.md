## Summary

Keep open. Current main has adjacent backup, cross-system migration, ClawHub, and bundle surfaces, but none implements the requested portable OpenClaw agent package that separates shareable identity/skills/knowledge from runtime state, logs, secrets, and machine-local paths. The related target-repo PR #53520 was closed unmerged, so it does not supersede this issue.

## What This Changes

Request to define a safe, reproducible OpenClaw agent packaging and pull/run workflow that distributes stable agent identity, skills, rules, and knowledge while leaving memory, logs, credentials, and machine-local state out of the package.

## Best Possible Solution

Define a canonical agent package contract that explicitly separates portable identity, skills, curated knowledge, dependency metadata, and compatibility from instance-local memory, sessions, logs, credentials, and machine paths. Reuse ClawHub bundles/plugins where they fit, and add a core export/import or install workflow only for gaps that need first-class OpenClaw semantics.

## Reproduction Assessment

Not applicable as a feature/design request. The current-main check is source/docs inspection: backup and migration commands exist, but no implemented OpenClaw agent package export/import or pull/run workflow exists.

## Solution Assessment

No current implementation is the best complete solution yet. Full backups are too stateful and sensitive for sharing, while cross-system migration providers solve a different import problem; the safer path is a maintainer-scoped design that composes existing plugin/bundle surfaces with any needed core contract.
