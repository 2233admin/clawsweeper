## Summary

Keep open. Current main has useful Active Memory timeout mitigations, but it does not prove the reported Cerebras gpt-oss-120b reasoning path aborts cleanly or preserves gateway health, and the docs still recommend the affected model for Active Memory.

## What This Changes

The issue reports Active Memory timing out and leaving the gateway CPU-bound when configured with Cerebras gpt-oss-120b in a Discord channel session.

## Best Possible Solution

Keep this as a focused Active Memory/Cerebras reliability regression until a maintainer can reproduce it live or with a deterministic provider harness, prove abort cleanup and gateway responsiveness during timeout, and then either keep the docs recommendation with evidence or change the recommendation/workaround.

## Solution Assessment
