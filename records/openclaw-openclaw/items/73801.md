## Summary

Keep open. Current main has generic Active Memory timeout mitigations, including a hard watchdog, partial timeout recovery, and a per-agent/model circuit breaker, but it still does not prove the documented Cerebras `gpt-oss-120b` Active Memory path aborts cleanly or preserves gateway responsiveness under the reported reasoning-model stream behavior.

## What This Changes

Active Memory should fail fast and preserve gateway responsiveness when configured with `cerebras/gpt-oss-120b` in interactive Discord channel sessions.

## Best Possible Solution

Keep this as a focused Active Memory/Cerebras regression until maintainers can reproduce or simulate the `gpt-oss-120b` reasoning/tool-call stream in the embedded recall path, prove abort cleanup and gateway responsiveness during and after timeout, and then either fix provider/runtime handling or qualify the Cerebras Active Memory recommendation.
