## Summary

Current main still does not implement the PR's intended bare Telegram `group:<numeric>` normalization in the active Telegram plugin. The PR also targets obsolete `src/telegram/*` paths and is functionally incomplete because the shared parser used by send and approval routing remains the key entry point. It should be reworked or superseded, not closed as implemented.

## Best Possible Solution

Keep this open until it is reworked or superseded. The right fix belongs in `extensions/telegram/src/targets.ts`: normalize bare `group:<numeric>` before topic and generic colon parsing in `parseTelegramTarget`, while preserving explicit `telegram:group:` and topic behavior. Add focused regression coverage for strip/normalize/parse/chat-type behavior, send delivery without lookup, active plugin target normalization/session routing, and exec approval button routing. After that lands on main, close this PR as superseded or implemented.
