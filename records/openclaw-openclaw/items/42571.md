## Summary

Current main still does not implement this PR's intended bare `group:<numeric>` Telegram delivery normalization in the active plugin. The prior review concern remains valid: the shared parser still preserves bare `group:` prefixes, current tests assert that behavior, and send/approval paths depend on that parser. Related PR #51688 described the right root area, but the current checked-out main at `7a23b2d945cc2d2e5c452e4641213e71de8dcc66` still has the unfixed behavior.

## Best Possible Solution

Keep this PR open until it is reworked or superseded. The durable fix should land in `extensions/telegram/src/targets.ts` by normalizing bare `group:<numeric>` before topic and generic colon parsing in `parseTelegramTarget`, while preserving explicit `telegram:group:` and topic behavior. Add focused coverage for strip/normalize/parse/chat-type behavior, send delivery without lookup, active plugin target normalization/session routing, and exec approval button routing. After a corrected implementation lands on main, close this PR as superseded or implemented.
