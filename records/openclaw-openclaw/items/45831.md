## Summary

Keep this PR open. Current main has partial local hardening in the attachment cache, but it still does not implement the PR’s intended shared `readLocalFileSafely()` path and still writes temp attachment files without an explicit private mode.

## Best Possible Solution

Port the PR onto current main by either replacing `readLocalBuffer()` with `readLocalFileSafely()` or making the custom helper match the shared safe-read identity guarantees. Keep the `too-large` error mapping, add `{ mode: 0o600 }` to temp writes, and add focused tests for safe-read behavior and temp-file mode.

- The PR is stale against current main because `attachments.cache.ts` now has a custom `readLocalBuffer()` implementation; it likely needs a rebase or port rather than direct merge.
