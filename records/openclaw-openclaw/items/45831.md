## Summary

Keep this PR open. Current main only partially overlaps with the hardening: the attachment cache now has a custom nofollow local reader, but it still does not use the shared `readLocalFileSafely()` path with its opened-file identity checks, and temp attachment cache writes still lack an explicit `0o600` mode. The PR is still a plausible implementation candidate, though it needs to be ported to the current attachment-cache shape.

## Best Possible Solution

Port the PR onto current main by replacing or refactoring `readLocalBuffer()` to use the shared `readLocalFileSafely()` path while preserving attachment-cache error semantics, add `{ mode: 0o600 }` to temp writes in `getPath()`, and add targeted tests for the shared safe-read behavior and temp-file mode. Then run the focused media-understanding attachment tests and the changed gate before merge.
