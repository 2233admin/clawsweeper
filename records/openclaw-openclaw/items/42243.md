## Summary

Keep open. Current main appears to sanitize some provider/run-level cron failures, but it still converts fatal cron outcomes into `isError: true` delivery payloads and sends them through normal announce delivery. That leaves the central request unmet: failed runs should be recorded in run history and failure-alert paths, not posted as ordinary announce output. The related implementation PR #46236 is closed unmerged.

## Best Possible Solution

Keep this issue open. Add a guard at the isolated cron finalize or delivery-dispatch boundary so fatal/error-status runs skip normal `delivery.mode: "announce"` delivery, while still recording `lastRunStatus: "error"`, preserving error counters/backoff/run history, and allowing only explicit failure-alert/failure-destination paths to send sanitized failure messages. Add regression coverage for provider/run-level errors with announce delivery and `bestEffort: false` asserting no outbound announce send occurs.
