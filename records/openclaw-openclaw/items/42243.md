## Summary

Keep open. Current main still turns fatal cron/provider failures into `isError: true` delivery payloads and routes them through normal announce delivery; newer sanitization appears to reduce raw JSON leakage, but it does not implement the requested failed-run announce suppression.

## Best Possible Solution

Keep this issue open. Add a guard at the isolated cron finalize or `dispatchCronDelivery` boundary so fatal/error-status isolated cron runs skip normal `delivery.mode: "announce"` fallback delivery, while still recording the failed run and sending configured failure alerts through the separate failure-notification path. Add focused regression coverage for provider/run-level failures with announce delivery and `bestEffort: false` asserting no outbound announce send occurs, plus coverage that explicit failure alerts still send sanitized messages.
