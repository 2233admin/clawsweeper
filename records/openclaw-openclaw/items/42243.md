## Summary

Keep open. Current main still converts fatal isolated cron failures into `isError: true` delivery payloads and sends them through the normal announce delivery path. The raw JSON formatting appears partly sanitized now, but the central behavior remains: failed cron runs can still post an error message to the configured channel instead of only recording the failure and using the separate failure-alert path.

## Best Possible Solution

Keep this issue open. Add a guard at the isolated cron finalize or delivery-dispatch boundary so fatal/error-status runs skip normal `delivery.mode: "announce"` delivery, while still recording `lastRunStatus: "error"`, run history, counters/backoff, and configured failure-alert/failure-destination notifications. Add focused regression coverage for provider/run-level errors with announce delivery and `bestEffort: false` asserting no outbound announce send occurs, plus coverage that explicit failure alerts still send sanitized messages.
