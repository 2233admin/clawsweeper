## Summary

Keep this issue open. Current main still nulls redacted raw text on unsafe round-trip fallback, and Control UI still treats `raw: null` as Raw mode unavailable. The older #59330/#59336 thread tracks an excessive-fallback root cause, but this issue’s requested display-versus-editability contract remains distinct and is not implemented.

## Best Possible Solution

Keep this issue open unless maintainers explicitly decide that unsafe raw round-trips must hide Raw mode entirely. A durable fix should separate raw display availability from raw editability in the config snapshot/UI contract, preserve safely redacted raw text for read-only viewing when possible, expose an explicit unsafe-round-trip or editability reason, keep save/apply blocked while unsafe, and update gateway, Control UI, docs, and regression coverage. Continue tracking the narrower excessive-fallback root cause in #59330/#59336.
