## Summary

Keep this issue open. Current main at 20e21173715ef488850023a90f72585a01cc379f still reads session-memory transcript text unchanged and writes it directly under `## Conversation Summary`; the latest release tag v2026.4.26 has the same relevant code. The issue is also paired with two open, unmerged closing PRs, #69961 and #73768, so the right maintainer action is to review or consolidate those fixes rather than close the issue before merge.

## What This Changes

The issue reports that the bundled session-memory hook persists raw chat-template tokens, raw tool-call XML, orphan role markers, and NO_REPLY-style malformed turns into memory files, then re-injects them across new sessions.

## Best Possible Solution

Keep this issue open and review or consolidate #69961 and #73768 into one canonical fix. The fix should sanitize session-memory content before persistence, cover model chat-template tokens, raw or dangling tool-call XML, orphan role markers, and NO_REPLY-only malformed turns, and add focused regression tests around transcript extraction and handler persistence. After that PR merges, close this issue as implemented by the merge.
