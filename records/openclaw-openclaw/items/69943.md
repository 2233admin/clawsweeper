## Summary

Keep #69943 open. Current main still extracts raw user/assistant text from session JSONL and writes the resulting sessionContent into the session-memory Markdown file without a session-memory sanitization pass. The related fix PR #69961 is the right active implementation path, but it is open/unmerged in the provided GitHub context and its sanitizer files are not present on main.

## Best Possible Solution

Keep #69943 open until PR #69961 or an equivalent maintainer-approved fix lands on main. The fix should sanitize the session-memory transcript read path and/or handler write path, add regression tests for leaked chat-template tokens, raw `<tool_call>` XML, role-marker leakage, and NO_REPLY-only turns, and consider reusing the existing shared assistant-visible/internal-scaffolding sanitizer where its behavior matches session-memory's persistence needs.
