## Summary

Keep open. Current main still contains the reported compaction-failure recovery behavior that rotates the active session mapping, and the provided GitHub context shows PR #70479 is an open, unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review or land PR #70479, or an equivalent fix. The fix should preserve the active agent:main:main session mapping when overflow recovery or compaction fails, avoid silent session rotation, provide explicit retry/manual session-control guidance, and keep regression coverage for embedded overflow payloads, thrown compaction failures, and the next WebChat message continuing under the same sessionId.
