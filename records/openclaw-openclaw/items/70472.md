## Summary

Keep open. Current main still sends embedded context-overflow payloads and thrown compaction failures through a recovery path that resets the active session to a newly generated session id, and #70479 remains open and unmerged with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to preserve the active WebChat `agent:main:main` session mapping across context overflow or compaction failures and make any recovery or rotation explicit and auditable.

## Best Possible Solution

Land or rework #70479, or an equivalent focused fix, so overflow and compaction-failure recovery preserves the active WebChat session id by default, avoids hidden `agent:main:main` remapping, records any intentional rotation, and keeps regression coverage for embedded overflow payloads, thrown compaction failures, and the next WebChat message continuing under the same session id.
