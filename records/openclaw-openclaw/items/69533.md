## Summary

Keep #69533 open. Current main still hardcodes the Dream Diary narrative system prompt and prompt text in `memory-core`, while the documented/configured Dreaming surface covers enablement, cadence, timezone, storage, execution, and phase thresholds but not a user-editable prompt/template. The related prompt issues/PRs in the provided context are adjacent but do not implement or supersede this Dreaming-specific request.

## Best Possible Solution

Keep #69533 open. The best product path is a memory-core-owned Dreaming prompt/template configuration design, likely under `plugins.entries.memory-core.config.dreaming`, with schema validation, documented placeholders, safe defaults, UI support in the Dreaming/settings surface, and focused tests proving custom prompt text reaches narrative generation without breaking diary storage or promotion safety.
