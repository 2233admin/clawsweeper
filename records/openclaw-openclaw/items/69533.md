## Summary

Keep this issue open. Current main still hardcodes the Dream Diary system prompt and user prompt, the resolved memory-core Dreaming config/schema/docs do not include a prompt/template/language field, and the Dreams UI only exposes status, diary actions, and the enable toggle. The language portion overlaps with #65097 and the closed duplicate #69354, but this issue is broader because it asks for user-customizable prompt/template output with placeholders.

## Best Possible Solution

Keep this open and implement a memory-core-owned Dream Diary prompt/template contract: add a documented schema-bounded field under plugins.entries.memory-core.config.dreaming, preserve the current prompt as the default, support a small explicit placeholder set, thread it through Dream Diary narrative generation, expose it through the existing config/UI surface, update docs/changelog, and add focused resolver/schema/narrative/UI coverage.
