## Summary

Keep open. Current main still hardcodes the Dream Diary system prompt and user prompt, and the memory-core config schema, resolver, docs, and Dreams UI expose model/cadence/enablement controls but no prompt/template/language placeholder contract. Related #65097 covers the narrower language/locale request and #69354 was closed as that subset, but this issue asks for broader user-customizable prompt formatting.

## What This Changes

The issue requests a user-configurable Dreaming prompt/template field with placeholder support and a Settings UI editor for Dream Diary output.

## Best Possible Solution

Implement a memory-core-owned Dream Diary prompt/template contract under plugins.entries.memory-core.config.dreaming. Preserve the current prompt and prompt-builder output as defaults, support a small documented placeholder set, thread the resolved config through light, REM, and deep narrative generation, expose it through the existing config/UI surface, and update docs, changelog, and focused backend/UI tests.
