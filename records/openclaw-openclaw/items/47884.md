## Summary

Current main still does not implement the reported custom OpenAI-compatible memory embedding path end to end. The runtime can look up a custom provider's configured api id, but `api: "openai-completions"` does not resolve to the OpenAI memory adapter, and the OpenAI memory adapter still forces endpoint/auth lookup through `openai` instead of preserving `bailian-embedding`.

## Best Possible Solution

Keep this issue open and implement a focused memory embedding fix: when `agents.defaults.memorySearch.provider` points at a custom `models.providers.<id>` whose `api` is `openai-completions`, resolve the OpenAI-compatible memory adapter while preserving the requested custom provider id for endpoint, auth, headers, and model-prefix normalization. Add regression coverage for the reported Bailian/DashScope-style config and keep #39766 separate unless the schema change is still desired for credential-only provider entries.
