## Summary

Keep PR #58445 open. Current main still has no `before_block_reply` hook, no block-reply flush trigger in the public/plugin-facing context, and no equivalent existing hook that can distinguish `tool_start` narration from normal text/message/run flushes before block delivery. The PR addresses a real core plugin API gap, though the branch still needs maintainer review and fixes for the coalesced-trigger behavior raised in review.

## Best Possible Solution

Keep the PR open and ask for a maintainer-quality port onto current main. The best implementation is an additive `before_block_reply` plugin hook in the public hook types/runner, a `BlockReplyFlushTrigger` on block-reply context, trigger threading through embedded subscribe, delivery, and the block-reply pipeline including coalescer storage/flush, and a cancellation path that does not mark unsent content as delivered. It should include docs and focused regression tests for tool-start narration, text/message/run-end triggers, coalesced payload trigger preservation, and final-answer dedupe.
