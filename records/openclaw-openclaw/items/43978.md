## Summary

Keep open. Current main and v2026.4.26 still use the old WhatsApp auto-reply retry window and have no final-delivery failure system event, while this PR targets removed `src/web/auto-reply/*` paths and is currently unmergeable. A newer open PR overlaps the retry half, but it does not fully supersede the structured final-delivery failure signal requested here.

## What This Changes

The PR widens WhatsApp Web auto-reply send retries from three short retries to a longer five-attempt window and enqueues a system event when final reply delivery fails.

## Best Possible Solution

Keep this PR open as useful source work, but do not merge it as-is. The best path is a narrow current-main adaptation in `extensions/whatsapp`: widen or otherwise cover the reconnect-window retry behavior in the plugin-owned send path, add a final-only sanitized system event for unrecovered final reply delivery failures through the plugin SDK seam, coordinate with open retry work in #54183, update the WhatsApp tests, and add a user-facing changelog fix entry with reporter/contributor attribution.
