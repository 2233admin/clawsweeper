## Summary

Keep open. Current main and the latest release still use the old WhatsApp auto-reply retry window and do not surface unrecovered final delivery failures as system events, while recent PR discussion reports the same production failure class on v2026.4.26. The PR is useful source work, but it targets removed `src/web/auto-reply` files and needs a current-main adaptation in the plugin-owned WhatsApp path before merge.

## What This Changes

The PR modifies old `src/web/auto-reply` WhatsApp delivery code and tests to increase send retry attempts/backoff and enqueue a system event when delivery fails.

## Best Possible Solution

Carry the useful behavior into `extensions/whatsapp`: extend or replace the plugin-owned retry window for transient reconnect failures, add a final-only sanitized system event for unrecovered final reply delivery failures through the existing reply-dispatch SDK seam, update focused WhatsApp tests, and resolve overlap with #54183 before landing.
