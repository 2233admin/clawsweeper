## Summary

Keep open. Current main still has no `acp.fallbacks` config key or ACP backend failover loop, and this PR is the active implementation attempt. It should not be closed as implemented, but it needs maintainer changes before merge because the PR branch still has ACP turn-loop correctness issues, an unrelated closing reference to #4698, and unrelated install-smoke/runtime-deps changes in security-sensitive execution/package-resolution surfaces.

## Best Possible Solution

Keep this PR open and request changes. The ACP failover work should be narrowed to the control-plane/config surface, derive candidates from resolved session metadata, close or retire old handles before switching, avoid failover after output or tool effects start, define init-stage `ACP_SESSION_INIT_FAILED` behavior, use precise retry classification, preserve failure bookkeeping/observability, add focused manager tests, and split the unrelated install-smoke/runtime-deps work plus correct the wrong closing reference.
