## Summary

Keep open. Current main still renders tool-enabled guidance and raw skills/report data for explicit empty effective tool lists, so PR #59898 is not implemented on main. Related PR #70790 is open and narrower: it targets empty `tools`/`tool_choice` provider payloads, not the prompt/report/provider-context behavior described here.

## Best Possible Solution

Keep #59898 open or ask for a narrowed revision that preserves the central fix: distinguish omitted tool lists from explicit empty effective tool lists, suppress tool-only prompt sections and skills/report entries for true no-tools sessions, and pass final tool/client-tool availability into provider prompt overlays. Coordinate separately with #70790 for provider payload stripping.
