## Summary

Keep open. Current main still does not implement cron `delivery.mode: "session"`: cron delivery remains `announce`/`webhook`/`none`, while internal session delivery is still available only through the `sessions_send` agent tool path with its own visibility and agent-to-agent policy checks.

## Best Possible Solution

Keep this open for maintainer product/API follow-up. If accepted, implement it as a core cron delivery feature: add `delivery.mode: "session"` to cron types, Gateway protocol schemas, normalizers, planner, CLI/UI/docs, and tests; add a framework-owned dispatch path that sends the final cron summary to a target session using sessions_send-equivalent session resolution, visibility, A2A policy, provenance, idempotency, and diagnostics without posting to a visible chat channel.
