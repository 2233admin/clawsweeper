## Summary

Keep open. Current main still does not implement cron `delivery.mode: "session"`; cron delivery remains `announce`/`webhook`/`none`, while `sessions_send` is still only available through the agent tool path with visibility and agent-to-agent policy checks.

## Best Possible Solution

Keep this issue open for maintainer product/API follow-up. If maintainers want the behavior, implement it as a core cron delivery feature: extend cron delivery types, Gateway protocol schema, normalizer, planner, CLI/UI/docs, and tests; add a framework-owned dispatch path that sends the final cron summary to a target session using sessions_send-equivalent provenance, visibility, and agent-to-agent policy checks; record delivery success/failure diagnostics without posting to a visible chat channel.
