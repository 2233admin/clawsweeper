## Summary

Keep open. Current main still preserves a host-built skills snapshot prompt for sandboxed non-rw attempt and compaction runs, even though sandbox setup mirrors eligible skills into the sandbox workspace. The focused implementation path remains the open #38789 PR; #74161 is related prompt hardening but does not address the stale snapshot remapping by itself.

## What This Changes

Sandboxed non-main sessions should expose sandbox-readable skill paths in the available-skills prompt instead of host-managed skill locations.

## Best Possible Solution

Implement a sandbox-aware skills prompt path that rebuilds or remaps available-skills locations from sandbox-readable entries whenever a non-rw sandbox run receives a host-built snapshot. The fix should cover both attempt and compaction paths while preserving skill filters, exposure metadata, remote eligibility notes, environment override behavior, and empty-sandbox behavior.
