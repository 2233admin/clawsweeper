## Summary

Current main still preserves the reported sandbox skill prompt mismatch. Non-rw sandbox setup mirrors eligible skills into the sandbox workspace, but embedded attempt and compaction paths still pass the host-built session skill snapshot through; when that snapshot has a prompt/resolved skills, live sandbox skill loading is skipped and the host-style prompt can win. #38789 remains the focused open implementation candidate.

## Best Possible Solution

Keep this issue open and review or land #38789, or implement the same fix directly on main. The fix should rebuild or remap the available-skills prompt from sandbox-readable skill entries for non-rw sandbox attempt and compaction runs instead of returning the host-built snapshot prompt, while preserving skill filters, agent allowlists, environment override behavior, remote eligibility notes, and empty-sandbox behavior. Add regression coverage for snapshot-present attempt and compaction runs.
