## Summary

Keep open. Current main still builds the skills snapshot from the agent workspace, then non-rw sandbox attempt and compaction paths switch to the sandbox workspace while reusing the original snapshot prompt and skipping live sandbox skill loading when resolved snapshot skills exist. Open #38789 remains the focused root-fix PR; #74161 is related prompt hardening and does not by itself rebuild or remap stale host paths.

## What This Changes

Fix sandboxed non-main sessions so skill paths exposed in the model prompt resolve inside the sandbox workspace instead of pointing at host-only skill locations.

## Best Possible Solution

Land a root fix that rebuilds or remaps available-skills prompt entries from sandbox-readable skill entries whenever a non-rw sandbox run receives a host-built snapshot, covering both attempt and compaction while preserving normal snapshot reuse, skill allowlists, exposure metadata, remote eligibility notes, and skill env overrides.
