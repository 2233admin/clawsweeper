## Summary

Keep open. Current main has bootstrap truncation defenses, so the reported overflow/freeze mechanism needs logs or reproduction, but the core memory concern remains real: `memory-core` deep promotion still appends to root `MEMORY.md` without a durable-file size budget, compaction, or archive layer.

## Best Possible Solution

Keep this issue open and fix it in the bundled `memory-core` owner surface: add a budget-aware durable promotion policy before writing `MEMORY.md`, with compaction or an archive layer for older promoted entries, status/doctor visibility for oversized root memory, and regression tests for near-limit and over-limit `MEMORY.md` promotion runs. Treat the Gateway freeze claim as a separate verification point unless logs reproduce it.
