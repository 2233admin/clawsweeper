## Summary

Keep open. Current main still has the bundled memory-core deep promotion path appending to root MEMORY.md without a durable on-disk budget, compaction, or archive layer, and the linked PR #74088 is still an open, unmerged closing PR rather than shipped behavior.

## What This Changes

The issue asks memory-core/dreaming to keep root MEMORY.md within bootstrap-safe bounds instead of appending promoted long-term entries indefinitely.

## Best Possible Solution

Review and land PR #74088, or replace it with an equivalent memory-core fix that enforces a durable write budget before promotion appends, preserves user-authored MEMORY.md content, and adds regression coverage for near-limit and over-limit promotion runs.

## Reproduction Assessment

Partially: the unbounded disk growth is reproducible from the current append-only applyShortTermPromotions path and repeated dreaming deep promotions. The full Gateway freeze/session-write-lock timeout still lacks logs or a focused reproduction against current main.

## Solution Assessment

Yes for the repair boundary: the narrow maintainable fix belongs in the bundled memory-core promotion writer, with budget-aware compaction or archival before writing MEMORY.md. PR #74088 appears to target that boundary, but it still needs review before this issue can close.
