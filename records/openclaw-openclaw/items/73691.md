## Summary

Keep open because current main still has the unbounded memory-core promotion write path, while the linked fix candidate remains an open, unmerged PR with closing syntax. The issue is recent, has no protected maintainer/blocked labels, and points at a valid bundled plugin bug rather than stale support noise.

## What This Changes

Bound memory-core deep dreaming promotions so `MEMORY.md` cannot grow past bootstrap limits and contribute to Gateway session stalls.

## Best Possible Solution

Land PR #74088 after review, or replace it with an equivalent memory-core fix that applies a durable pre-write budget, preserves user-authored `MEMORY.md` content, and adds regression coverage for near-limit and over-limit promotion runs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: the unbounded disk growth path is clear from the current append-only promotion writer and repeated distinct promotion candidates should reproduce growth, but the full Gateway freeze/session-write-lock timeout still lacks logs or a focused end-to-end reproduction.

## Solution Assessment

Is this the best way to solve the issue? Yes on ownership and boundary: the narrow fix belongs in bundled `memory-core` promotion writing, not in bootstrap truncation. The exact compaction/archive policy should be settled in review of PR #74088 or an equivalent replacement.
