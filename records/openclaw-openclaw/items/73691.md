## Summary

Keep open. Current main still has the memory-core promotion writer appending promoted dreaming entries to `MEMORY.md` without a durable size budget, compaction, or archive layer, and the open paired fix PR remains unmerged.

## What This Changes

The issue reports that memory-core dreaming promotions can grow `MEMORY.md` beyond bootstrap limits and eventually wedge Gateway sessions, and requests bounded promotion, compaction, archival, or structured memory storage.

## Best Possible Solution

Land PR #74088 after review, or replace it with an equivalent memory-core fix that enforces a durable budget before promotion writes, preserves user-authored `MEMORY.md` content, and adds regression tests for near-limit and over-limit promotion runs.

## Reproduction Assessment

Partially: repeated dreaming/deep promotion runs against current main can reproduce unbounded `MEMORY.md` growth from the append-only writer. The final Gateway freeze and session-write-lock timeout still need logs or a focused runtime reproduction.

## Solution Assessment

Unclear for the open PR until it is reviewed, but the right repair boundary is the bundled memory-core promotion writer. A bootstrap-only workaround or naive truncation would be less maintainable because it would leave disk growth or risk deleting durable user memory.
