## Summary

Keep open: current main still has the append-only memory-core promotion path without a durable `MEMORY.md` file-size budget, and PR #74088 remains the paired open implementation candidate.

## What This Changes

The issue asks OpenClaw to bound dreaming promotions so `MEMORY.md` cannot grow past bootstrap budgets and contribute to Gateway stalls.

## Best Possible Solution

Land PR #74088 or an equivalent memory-core change that enforces a durable size budget before promotion writes, preserves user-authored `MEMORY.md` content, and adds regression coverage for repeated near-limit and over-limit promotion runs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for unbounded `MEMORY.md` growth: repeated successful `applyShortTermPromotions` calls append new promotion sections to existing content without a size guard. The exact Gateway freeze mechanism is unclear because no lock-timeout logs or focused gateway reproduction were provided.

## Solution Assessment

Is this the best way to solve the issue? Yes on boundary: the durable guard belongs in `memory-core`'s promotion writer, not only in bootstrap truncation. The precise compaction/archive policy should be settled while reviewing PR #74088 or an equivalent replacement.
