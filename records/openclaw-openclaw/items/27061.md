## Summary

Keep this PR open. Current main still has the validator gap this PR targets: `validate_skill()` trims `name` and `description`, but only validates them inside truthy guards, so empty or whitespace-only values can still pass. The related hardening PRs in the provided context were closed unmerged, and a maintainer review already identified the same remaining issue before the contributor refreshed the PR description.

## Best Possible Solution

Keep this PR open and review or land it, or apply an equivalent maintainer patch scoped to `skills/skill-creator/scripts/quick_validate.py` and `skills/skill-creator/scripts/test_quick_validate.py`. The implementation should reject empty or whitespace-only `name` and `description` immediately after trimming and keep regression coverage for both fields.
