## Summary

Keep open. Current main still does not implement `/total_usage`, but the PR should not merge as-is because the latest diff still removes bounded transcript/discovery filtering, aggregates model totals by model id only, and makes `CostUsageSummary.models` required without updating an existing typed fixture.

## What This Changes

The PR adds a `/total_usage` chat command with range parsing, Telegram quick-select buttons, per-model aggregate usage output, usage summary type changes, focused tests, and slash-command documentation.

## Best Possible Solution

Revise this PR in place so `/total_usage` keeps the existing bounded file filters, uses provider-aware model buckets, updates typed fixtures for the new summary shape, and lands with focused command plus session-usage tests.

## Reproduction Assessment

Yes. The PR diff itself reproduces the blockers: the new model aggregate key is model-only, both bounded file filters are removed, and an existing typed `CostUsageSummary` fixture still omits the newly required `models` field.

## Solution Assessment

No. The feature direction fits the existing usage surfaces, but the current patch is not the narrowest maintainable solution until it preserves current provider-aware aggregation and bounded filtering contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Partition model totals by provider and model:** `src/infra/session-cost-usage.ts:426-431`
  - body: The new aggregate key is only `entry.model`, while each bucket stores a single provider from the first matching entry. If two providers emit the same model id, `/total_usage by-model` will merge their tokens and costs and label the row with whichever provider was seen first; use the provider-plus-model key shape already used by `sessions.usage`.
  - confidence: 0.9
- **[P2] Restore bounded transcript prefiltering:** `src/infra/session-cost-usage.ts:399-402`
  - body: Removing the `mtimeMs < sinceTime` prefilter makes bounded queries such as `/total_usage 5h` enumerate and scan every historical transcript before `onEntry` drops old records. Long-lived installations can turn a small range command into a full-history I/O scan, so keep the lower-bound file filter for bounded ranges.
  - confidence: 0.84
- **[P2] Keep discovery filtering bounded by the requested start:** `src/infra/session-cost-usage.ts:497-500`
  - body: `discoverAllSessions()` no longer honors `params.startMs`, but `sessions.usage` still sorts and slices discovered sessions before loading per-session summaries. Out-of-range transcript files can consume the limit and push in-range sessions out of the response or totals; restore the start-bound filter during discovery.
  - confidence: 0.78
- **[P2] Update CostUsageSummary fixtures:** `src/infra/session-cost-usage.types.ts:68`
  - body: Adding `models` as a required `CostUsageSummary` field changes existing typed fixtures. `commands-session-usage.test.ts` still returns summary objects without `models`, so the test-type lane needs those fixtures updated or the field made backward-compatible.
  - confidence: 0.78
