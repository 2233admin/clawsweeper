## Summary

Keep open. Current main still lacks `/total_usage`, so the PR is not obsolete, but the proposed patch has merge-blocking correctness and validation issues around usage aggregation keys, bounded transcript scanning, discovered-session range filtering, and typed test fixtures.

## What This Changes

The PR adds `/total_usage` command registration and handler wiring, hour/day/all range parsing, Telegram range buttons, per-model aggregate usage data, focused tests, and slash-command documentation.

## Best Possible Solution

Update this PR in place: keep the `/total_usage` feature, preserve bounded transcript and discovery filtering for ranged queries, aggregate model totals by provider plus model, update all `CostUsageSummary` fixtures or make the new field compatible, then run the focused usage/gateway tests and the changed gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Partition model totals by provider and model:** `src/infra/session-cost-usage.ts:426-431`
  - body: The new aggregate key is only `entry.model`, while each bucket stores a single provider from the first matching entry. If two providers emit the same model id, `/total_usage by-model` will merge their tokens/costs and label the row with whichever provider was seen first; use the provider-plus-model key shape already used by `sessions.usage`.
  - confidence: 0.9
- **[P2] Restore transcript prefiltering for range summaries:** `src/infra/session-cost-usage.ts:399-402`
  - body: Removing the `mtimeMs < sinceTime` prefilter makes bounded queries such as `/total_usage 5h` enumerate and scan every historical transcript before `onEntry` drops old records. Long-lived installations can turn a small range command into a full-history I/O scan, so keep the lower-bound file filter for bounded ranges.
  - confidence: 0.84
- **[P2] Keep discovery filtering bounded by the requested start:** `src/infra/session-cost-usage.ts:497-500`
  - body: `discoverAllSessions()` no longer honors `params.startMs`, but `sessions.usage` still sorts and slices discovered sessions before loading per-session summaries. Out-of-range transcript files can therefore consume the limit and push in-range sessions out of the response/totals; restore the start-bound filter during discovery.
  - confidence: 0.78
- **[P2] Update CostUsageSummary test fixtures:** `src/infra/session-cost-usage.types.ts:69`
  - body: Adding `models` as a required `CostUsageSummary` field changes existing typed fixtures. `commands-session-usage.test.ts` declares a `vi.fn<() => Promise<CostUsageSummary>>` and returns objects without `models`, so the test-type lane needs those fixtures updated or the new field made backward-compatible.
  - confidence: 0.78
