## Summary

Keep open. Current main still lacks the central LINE sticker support requested here: there is no shared sticker payload or STICKER directive path, LINE rich-message parsing excludes stickers, outbound LINE delivery has no native sticker send branch, and inbound LINE stickers are reduced to text-only context. The discussion is active and split-scope, so there is no merged implementation to close against.

## What This Changes

Request LINE bot send/receive support for native stickers, including directive parsing, payload/pipeline delivery, inbound packageId/stickerId context, and a Skill-based sticker catalog/editor.

## Best Possible Solution

Review and land or revise the narrow LINE-native send path in #73378, keep any core change limited to a generic channelData delivery fix, and then track inbound sticker metadata plus any cross-channel abstraction separately. The sticker catalog/editor should remain a ClawHub skill or bundle unless maintainers explicitly choose bundled promotion.
