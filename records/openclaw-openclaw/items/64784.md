## Summary

Keep this issue open. Current main still lacks native LINE sticker sending, a STICKER directive path, and structured packageId/stickerId exposure for inbound stickers; the only current inbound handling turns stickers into bracketed text. The issue also has an active narrower related PR (#73378) from the same contributor, so the right maintainer action is to review that PR rather than close or spawn a competing repair.

## What This Changes

Request to support LINE sticker send/receive, including a sticker directive, LINE Messaging API delivery, inbound sticker handling, and a catalog-driven skill for AI sticker selection.

## Best Possible Solution

Land or revise the focused LINE-native send path in #73378, keep any generic core change limited to a channelData-only delivery seam, then split follow-up tracking for inbound sticker metadata and any cross-channel sticker abstraction. The sticker catalog/editor should live as a ClawHub skill or bundle unless maintainers intentionally promote it.
