## Summary

Keep this PR open. Current main still exposes `sessions_send` with `timeoutSeconds` as the only fire-and-forget control; the proposed `async` boolean alias is not implemented in schema, runtime handling, docs, or tests. The PR is narrow and the discussion already addressed the main conflict case, so it remains a valid implementation candidate pending maintainer product review and focused coverage.

## Best Possible Solution

Keep this PR open as an implementation candidate. If maintainers want the DX alias, rebase it onto current main, update `SessionsSendToolSchema`, timeout resolution, shared `describeSessionsSendTool()` text, and display/detail surfaces as appropriate, then add focused tests for the new schema field, accepted fire-and-forget behavior, and contradictory parameter rejection. If maintainers decide the existing `timeoutSeconds: 0` contract is sufficient, close with that product rationale rather than as already implemented.
