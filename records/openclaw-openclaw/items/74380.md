## Summary

Keep open. Current main and v2026.4.29 still contain the reported onboarding install failure path for `model-usage` and `summarize`, and PR #74408 remains the active open fix candidate rather than a reason to close this issue before merge.

## What This Changes

`openclaw onboard` should install the selected bundled skill dependencies successfully on macOS instead of failing on the CodexBar and summarize Homebrew recipes.

## Best Possible Solution

Revise and land PR #74408, or an equivalent narrow fix, so skill install metadata can express Homebrew casks end to end, `model-usage` installs CodexBar through the macOS cask path, and `summarize` uses a maintained installer source with focused parser, command, status, and onboarding regression coverage.

## Reproduction Assessment

Yes. On macOS with missing `codexbar` and `summarize`, current `setupSkills()` will offer these missing dependencies and `installSkill()` will build formula-only Homebrew commands that match the reported failures.

## Solution Assessment

Partially. Adding structured cask support and switching `summarize` away from the removed Homebrew formula is the narrow maintainable direction, but the active PR needs the cask parser follow-through and maintainer validation of the npm source before it is merge-ready.
