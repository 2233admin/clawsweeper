## Summary

Keep open. Current main still has a live onboarding failure path for the macOS `model-usage` skill because its metadata advertises a Homebrew cask install while the installer can only run formula installs. The linked open PR #74408 directly targets this issue with closing syntax, so this issue should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue asks for `openclaw onboard` to stop failing when installing bundled missing skill dependencies such as `model-usage` and `summarize` on macOS.

## Best Possible Solution

Review and land #74408 or an equivalent narrow fix that makes bundled skill installer metadata match working install commands, especially structured macOS cask handling for `model-usage`; after merge, let the issue close through the linked fix.
