## Summary

Keep open. Current main still has the reported skill dependency install failures, and open PR #74408 uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for `openclaw onboard` to stop failing when installing bundled skill dependencies such as `model-usage` and `summarize`.

## Best Possible Solution

Revise and land PR #74408, or an equivalent narrow fix, so cask-backed Homebrew installs are a structured skill installer contract, `model-usage` uses that contract, `summarize` points at a working maintained installer, and focused parser/status/install/onboarding tests plus a user-facing changelog entry cover the regression.

## Reproduction Assessment

Yes. The reporter's `openclaw onboard` path maps directly to `setupSkills()` and `installSkill()`, and current main still builds the exact failing Homebrew formula installs while upstream confirms the CodexBar formula/cask mismatch and removed summarize formula.

## Solution Assessment

Yes, the narrow maintainable solution is to fix the skill installer metadata contract and the two affected bundled skill manifests. The currently open PR is the right vehicle, but it needs parser follow-through for boolean `cask` metadata and regression coverage before it can fully solve the issue.
