## Summary

Keep open. Current main still has the reported onboarding failure path: `model-usage` advertises a macOS CodexBar install but the installer can only run `brew install <formula>`, while the upstream formula is Linux-only and the macOS binary is delivered as a cask; `summarize` still points at a deleted Homebrew tap formula even though the maintained npm package exists. Related #57555 covers the `summarize` subset, but this issue is broader because it also covers `model-usage`/CodexBar onboarding.

## What This Changes

The issue asks for `openclaw onboard` bundled skill dependency installation to stop failing for selected skills such as `model-usage` and `summarize` on macOS.

## Best Possible Solution

Repair the bundled skill installer metadata and contract so onboarding selects working installers: add a safe structured cask path or stop offering `model-usage` as installable on macOS until supported, replace `summarize`'s deleted formula with the current `@steipete/summarize` node installer, audit the other selected skills, and cover metadata parsing, installer command generation, status selection, and onboarding failure handling with focused tests plus a user-facing changelog entry.
