## Summary

Keep open. Current main still contains the reported onboarding failure path: bundled `model-usage` is labeled as a brew cask but still has only a `kind: "brew"` formula installer, `summarize` still points at the removed Homebrew formula, and the core brew installer still emits formula-only `brew install <formula>`. Open PR #74408 is already the active closing fix candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw onboard` fails installing bundled skill dependencies because `model-usage` uses a Homebrew formula path for CodexBar and `summarize` references a removed Homebrew formula.

## Best Possible Solution

Review and land PR #74408, or an equivalent narrow fix, so skill install metadata can express a Homebrew cask, `model-usage` installs CodexBar through the cask path, `summarize` uses the maintained npm package, and installer/status/security-scan tests cover the new metadata.

## Reproduction Assessment

Yes. On macOS with missing `codexbar` and `summarize`, current main will surface these as installable missing skill dependencies during onboarding and then run the checked-in brew formula commands through `installSkill()`, matching the reported failures.

## Solution Assessment

Yes. The linked repair direction is the narrowest maintainable fix: add structured cask support instead of label-only metadata, switch `summarize` to the maintained node installer, and keep installer security metadata aligned.
