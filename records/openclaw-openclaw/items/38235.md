## Summary

Current main still has the reported Slack draft-preview identity gap. Normal Slack delivery resolves and forwards agent identity into `sendMessageSlack`, but `createSlackDraftStream` still has no identity parameter and its initial `send()` call omits identity. The issue is also paired with open PR #38237 using closing syntax, so it should stay open until that PR or an equivalent fix merges.

## Best Possible Solution

Keep this issue open and review, rebase, or adapt PR #38237. The focused fix should add optional `SlackSendIdentity` support to `createSlackDraftStream`, pass the resolved `slackIdentity` from `dispatchPreparedSlackMessage`, forward it into the initial draft `sendMessageSlack` call, and add a regression test proving the first draft `chat.postMessage` receives identity. Separately verify the native Slack streaming API contract and either add identity support there if available or document the upstream limitation.
