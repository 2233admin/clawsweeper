## Summary

Keep this PR open. Current main still intentionally persists generated gateway install tokens with token auth, gateway auth defaults to token mode, and docs describe auth as required by default; the PR is a gateway auth product/security policy change rather than conservative cleanup.

## What This Changes

The PR changes generated gateway install-token persistence to write `gateway.auth.mode: "none"` when no auth mode exists, and adds a test expecting that persisted config.

## Best Possible Solution

Keep token-auth persistence as the current default unless maintainers intentionally design a narrower local-only no-auth path that proves effective loopback/private ingress, preserves protection for daemon installs and Tailscale/HTTP exposure, updates docs, and adds regression coverage across loopback, non-loopback, generated-token persistence, and Serve/Funnel cases.
