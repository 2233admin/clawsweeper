## Summary

Keep open. Current `origin/main` already implements most runtime behavior through manifest-driven provider auth metadata and tests, but the public provider docs still omit the `GOOGLE_CLOUD_API_KEY` path, and the PR branch now contains stale core/static env-list edits that should not be merged without a rebase.

## What This Changes

The PR adds `GOOGLE_CLOUD_API_KEY` recognition for the `google-vertex` provider across auth detection, secret scrubbing/shell-env discovery, tests, and model-provider documentation.

## Best Possible Solution

Rebase this PR onto current `origin/main`, drop the obsolete core/static env-list changes, keep the provider metadata in the Google plugin manifest, and retain only the remaining useful docs update plus any narrowly relevant regression coverage maintainers still want.
