## Summary

This should stay open: current main still lacks OpenAI Codex active-workspace validation or recovery for a deactivated ChatGPT workspace, and the related open PR only covers failover classification for the `deactivated_workspace` marker. The reporter’s analysis is also corroborated by a second user on v2026.4.8 seeing the same runtime error.

## What This Changes

Request to make OpenAI Codex OAuth avoid, recover from, or clearly remediate binding to a deactivated ChatGPT workspace when a ChatGPT account has multiple workspaces.

## Best Possible Solution

Keep the issue open and treat #55893 as a partial mitigation to review, then add an OpenAI-plugin-owned fix that either persists a verified active `accountId` after OAuth/device-code login or detects `deactivated_workspace` at runtime, retries once with a usable active workspace, persists that selection, and documents a clear remediation path when recovery is impossible.
