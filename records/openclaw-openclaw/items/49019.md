## Summary

Keep open. Current main still exposes `POST /hooks/agent` as async run-id-only ingress, with no opt-in wait/result-return request fields, no blocking dispatcher contract, no result-bearing HTTP response, and no docs/tests for the requested mode. Open PR #67433 is a direct implementation candidate and remains unmerged in the provided context, so this issue should stay open for maintainer review rather than be closed by cleanup.

## Best Possible Solution

Keep this issue open and review PR #67433, or an equivalent maintainer-owned implementation. The accepted change should add opt-in blocking/result-return behavior to `POST /hooks/agent`, keep the current async run-id-only behavior by default, return a narrow deterministic status/result shape, document the new fields, and add focused hook tests for default async behavior, success/error/timeout, idempotency, delivery suppression, and hook security boundaries.
