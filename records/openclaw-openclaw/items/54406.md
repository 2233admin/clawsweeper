## Summary

Keep open. Current main still does not implement this PR's central compaction event observability: compaction start events carry only `phase: "start"`, and compaction end events carry `phase`, `willRetry`, and `completed` but not `hasResult` or sanitized `errorMessage`. The PR is also stale against main because the upstream event names are now `compaction_start`/`compaction_end`, while the PR diff targets the older `auto_compaction_*` names, and the posted review correctly flags the missing `hasResult` field and noisy `warn` logging.

## Best Possible Solution

Keep this PR open, but require a rebase onto current main and a small contract-focused update: use the current `compaction_start`/`compaction_end` names, pass any available start reason through safely, emit `completed`, `hasResult`, `willRetry`, and sanitized `errorMessage` on end events, keep routine lifecycle logging below `warn` unless retry/abort/error, and add focused tests for the ambiguous cases called out in review.
