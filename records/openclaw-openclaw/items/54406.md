## Summary

Keep open. Current main still has the narrower compaction event contract, so the PR's core observability goal is not implemented or superseded; however, the submitted branch is stale against the current `compaction_start`/`compaction_end` naming and still has blocking review issues around the missing `hasResult` payload field and noisy `warn` lifecycle logs.

## What This Changes

The PR branch adds compaction start/end event metadata, sanitized compaction end error text, structured lifecycle logs, dispatcher forwarding of the start event object, and focused compaction payload tests.

## Best Possible Solution

Rework the change on current main so the embedded compaction dispatcher passes the upstream start event into the handler, start events expose a safe `reason`, end events expose `completed`, `hasResult`, `willRetry`, and sanitized `errorMessage`, and existing compaction count, retry, reconciliation, hook, liveness, and UI behavior remain unchanged except for the richer observability contract.
