## Summary

Keep open. Current main still lacks the richer compaction event payload this PR is trying to add, but the submitted branch is stale and incomplete: it targets the old `auto_compaction_*` dispatcher shape, omits `hasResult` from emitted end events, and promotes routine lifecycle logs to `warn`. The prior Greptile and ClawSweeper comments match the current source evidence, so this remains useful source work but is not merge-ready as-is.

## What This Changes

The PR branch adds compaction start/end event metadata, structured compaction lifecycle logging, dispatcher forwarding for the start event object, and focused tests for compaction event payloads.

## Best Possible Solution

Rework the change on current main so the embedded compaction dispatcher passes the `compaction_start` event into the handler, start events expose a safe `reason`, end events expose `completed`, `hasResult`, `willRetry`, and sanitized `errorMessage`, and compaction count, retry, hook, liveness, UI, and `/status` behavior stay otherwise unchanged.
