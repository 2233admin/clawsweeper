## Summary

Keep this PR open. Current main still lacks the PR's central proactive cron-budget behavior: there is no `beforeAttempt` hook in `runWithModelFallback`, no `stop_before_candidate` decision, and no cron `deadlineAtMs` propagation into isolated fallback execution. The PR remains a plausible implementation candidate, but it needs maintainer review because the branch has unresolved review comments and a timeout-policy DoS concern raised in the discussion.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a scoped implementation that threads a real cron execution deadline into isolated runs, adds a generic pre-attempt fallback gate, preserves the original failure or explicit stop reason correctly, and includes focused tests for near-deadline fallback skipping. Before merge, the timeout-policy DoS concerns should be fixed or deliberately split to a tracked security follow-up, and the guard should be based on the actual per-attempt budget rather than only the outer cron safety ceiling.
