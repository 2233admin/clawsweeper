## Summary

Keep this PR open. Current main still connects bundled MCP servers with the configured `connectionTimeoutMs`, but bundled MCP tool execution calls `client.callTool(...)` without the third `RequestOptions` argument, so the timeout fix proposed here is not implemented on main and still matches the open linked bug report.

## Best Possible Solution

Review and land a narrow fix in `src/agents/pi-bundle-mcp-runtime.ts` that carries the resolved MCP timeout into `Client.callTool(..., undefined, { timeout })`, preferably with a focused regression test for the third positional SDK argument and wording aligned to the documented `connectionTimeoutMs` config key. After merge, the linked bug can close as implemented.
