## Summary

Keep this issue open. Current main has related strict OpenAI-compatible backend mitigations, but it does not fully address the Mindie 422 path discussed here: the suggested `supportsStreamOptions` / `supportsToolChoice` keys remain unsupported, `stream_options` is still seeded unconditionally for Chat Completions payloads, and there is no narrow `tool_choice` suppression control while preserving tools.

## Best Possible Solution

Keep this open and fix the generic OpenAI-compatible transport path. First, make `compat.supportsUsageInStreaming: false` actually omit `stream_options` in `buildOpenAICompletionsParams`, with tests for a strict custom backend. Then decide whether a narrow documented `tool_choice` suppression compat flag is needed, or whether `compat.supportsTools: false` is the intended supported mitigation for Mindie.
