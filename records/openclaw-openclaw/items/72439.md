## Summary

Keep this PR open. Current main still conflates a non-matching session-kill path with a matched path whose session key cannot be decoded, so malformed route-shaped requests can fall through instead of returning a deterministic Gateway API error. The PR is narrow and has a focused regression test; it likely needs a small rebase because current main moved the runtime config import after the PR branch was cut.

## What This Changes

This PR changes the Gateway `POST /sessions/:sessionKey/kill` path parser to distinguish malformed matched session keys from non-matching paths, return `400 invalid_request_error` before auth/session lookup, and add a regression test for `/sessions/%zz/kill`.

## Best Possible Solution

Rebase the narrow handler change onto current main, keep the current `getRuntimeConfig` import path, and land the malformed-key guard plus regression test without changing valid session auth, ownership, kill, or RPC semantics.
