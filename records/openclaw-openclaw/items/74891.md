## Summary

Keep this PR open. Current main still prompts for every missing skill with a primary env var, while the PR’s selection guard is a narrow fix for the linked onboarding bug and I found no blocking functional or security finding in the provided diff.

## What This Changes

The PR records which skills were selected in the onboarding dependency multiselect, skips API-key prompts for unselected bin-backed skills, and adds focused regression coverage in `src/commands/onboard-skills.test.ts`.

## Best Possible Solution

Land this PR or an equivalent narrow change after maintainer review, keeping the behavior in the shared onboarding skill prompt flow and leaving skill manifests unchanged unless maintainers want separate copy follow-up.

## Solution Assessment
