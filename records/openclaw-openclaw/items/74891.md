## What This Changes

The PR records which onboarding skill dependency installs were selected, skips primary environment-variable prompts for skipped bin-backed skills, and adds tests for local Whisper versus Whisper API behavior.

## Best Possible Solution

Land this PR or an equivalent narrow change after maintainer review, keeping the fix in the shared onboarding skills prompt flow and leaving skill manifests unchanged unless maintainers want a separate UX-copy follow-up.

## Solution Assessment

Yes. The proposed selection guard is the narrowest maintainable fix I found: it preserves env-only prompts and selected bin-backed skill prompts while suppressing credentials for bin-backed skills the user skipped.
