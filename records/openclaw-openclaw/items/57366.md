## Summary

Keep this PR open. Current main still treats every non-cid HTML <img> in Teams attachments as an inline image candidate, so a pure Teams emoji CDN img tag still becomes a <media:image> placeholder while the HTML text extractor strips the img tag and does not preserve alt text. The PR remains a focused implementation candidate for this bug, and the discussion shows the author addressed the review concerns about mixed non-HTML attachments, single-pass src/alt validation, and URL host parsing.

## Best Possible Solution

Keep the PR open and review it as the active fix candidate. The best implementation is a narrowly scoped Teams attachment helper that extracts unicode alt text only when all HTML img tags are verified Teams emoji CDN URLs, bails out to existing placeholder logic for mixed or non-HTML attachments, and keeps regression tests for pure emoji, mixed real media, missing src/alt, and CDN-host false positives.
