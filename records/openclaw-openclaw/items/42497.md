## Summary

Keep this PR open. The underlying WebSocket self-contention problem is still not implemented on current main, but this branch is not safe to merge: the local-dispatch early return bypasses important gateway request-contract behavior, and GitHub currently reports the PR as dirty against main.

## What This Changes

The PR adds a local in-process gateway dispatch registry, registers `dispatchGatewayMethod` during gateway startup, makes agent gateway-tool calls prefer that local path for non-`expectFinal` calls, and adds focused tests for the new registry and fallback behavior.

## Best Possible Solution

Keep the in-process-dispatch direction, but implement it as an explicit gateway request adapter that preserves WebSocket-equivalent behavior: resolved token intent, method-specific or explicit scopes, default-deny unknown methods, bounded timeout handling, deliberate final-response behavior, and JSON-frame-equivalent params normalization.

## Reproduction Assessment

Yes. A focused reproduction is to register a local dispatcher in the gateway tool test harness and call `callGatewayTool` with `gatewayToken`, `timeoutMs`, explicit scopes, or params containing optional `undefined` fields; the PR path returns before the current WebSocket contract is applied.

## Solution Assessment

No. The proposed registry is a useful direction, but returning before the existing gateway request contract is preserved is not the narrowest maintainable fix for this security-sensitive control-plane path.

  - body: The local branch returns before `callGatewayTool` computes explicit or least-privilege scopes. `dispatchGatewayMethod` then authorizes from the ambient request client or a synthetic operator client, so embedded calls no longer match the WebSocket path's scope and default-deny behavior.

  - confidence: 0.88

  - body: The local path forwards the live `params` object directly, while the WebSocket path sends a JSON frame. Optional properties set to `undefined` can therefore reach validators locally even though they would be omitted over WebSocket.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff adds an in-process gateway control-plane path without preserving the existing auth, scope, token, timeout, and transport-normalization contract.

Concerns:

- **[high] Local dispatch bypasses auth and scope resolution:** `src/agents/tools/gateway.ts:154`
  - body: `callGatewayTool` can return from local dispatch before resolving explicit gateway tokens, least-privilege method scopes, or explicit scope overrides, so embedded calls may execute under ambient or synthetic gateway identity rather than the caller's intended authorization contract.
  - confidence: 0.88
- **[medium] Timeout bypass can hang privileged operations:** `src/agents/tools/gateway.ts:154`
  - body: Because `timeoutMs` is not applied on the local path, a stuck in-process handler can block an active agent turn instead of failing under the bounded timeout behavior used by the WebSocket request path.
  - confidence: 0.84
