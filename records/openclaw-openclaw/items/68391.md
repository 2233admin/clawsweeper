## Summary

Keep open. Current `main` still has the silent scope drop: `models` exposes `--agent`, but the `set` and `set-image` handlers ignore the inherited option and call global-default write paths. The related fix PR #68399 is closed and unmerged, so there is still actionable work.

## What This Changes

The issue reports that `openclaw models --agent <id> set <model>` accepts an agent scope but writes `agents.defaults.model.primary` instead of a per-agent model override.

## Best Possible Solution

Prevent the silent global write by either honoring `--agent` with an explicit `agents.list` update or, more conservatively, rejecting `--agent` for `models set` and `models set-image` with a clear error until per-agent setters are intentionally designed.
