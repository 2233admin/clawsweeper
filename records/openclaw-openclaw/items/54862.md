## Summary

Keep open. Current main does not implement `autoRouter.allowedModels`, and the requested OpenRouter Auto Router payload is supported upstream, but the proposed branch still needs maintainer review because it applies the new allowlist path to fixed OpenRouter models and crosses the core/plugin test boundary.

## What This Changes

The PR adds an OpenRouter `params.autoRouter.allowedModels` option that injects an `auto-router` plugin payload, adjusts reasoning suppression for allowlists, exports the helper, and adds OpenRouter/provider extra-param tests.

## Best Possible Solution

Revise the PR so `autoRouter.allowedModels` is honored only for the OpenRouter Auto model, expose any helper needed by core tests through `extensions/openrouter/api.ts` or keep the coverage in OpenRouter plugin tests, handle x-ai/Grok wildcard reasoning conservatively, and add the docs/changelog entry if maintainers accept the config shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard auto-router injection to Auto requests:** `extensions/openrouter/stream.ts:99-107`
  - body: `autoRouter.allowedModels` is described as an `openrouter/auto` constraint, but this block runs for every OpenRouter model when the params are present. Since shared/default params are merged before provider wrapping, a shared `autoRouter` config can add an Auto Router plugin payload, and possibly suppress reasoning, on fixed-model calls. Gate this block to the Auto model path before injecting the plugin or changing reasoning behavior.
  - confidence: 0.9
- **[P2] Expose the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` from the OpenRouter extension entrypoint. The scoped extension rules require core tests to use `api.ts` or an SDK seam for bundled plugin helpers, otherwise entrypoint refactors can break unrelated core tests. Export the helper from `extensions/openrouter/api.ts` or keep this coverage inside the OpenRouter plugin tests.
  - confidence: 0.86
- **[P2] Treat broad Grok wildcards as reasoning-unsafe:** `extensions/openrouter/stream.ts:57-59`
  - body: OpenRouter allowed-model patterns support provider-agnostic wildcards, so patterns like `*/grok-*` can route Auto Router to x-ai/Grok models. This helper only flags entries containing `x-ai/`, leaving reasoning injection enabled and reintroducing the unsupported x-ai reasoning payload failure. Mark broad Grok/x-ai-matching wildcards as unsafe or disable reasoning for unresolved broad patterns.
  - confidence: 0.82
