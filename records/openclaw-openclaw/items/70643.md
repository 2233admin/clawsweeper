## Summary

Keep #70643 open. Current main has safer temp-file-and-rename writes and recovery paths, but it still does not serialize the shared openclaw.json read-modify-write path with an advisory lock. The related PR #70655 was closed unmerged, and current source/tests do not show an implemented concurrent config-write fix.

## Best Possible Solution

Implement a shared advisory lock for openclaw.json that covers the full config read-modify-write operation, not just the final rename. The likely path is to reuse `withFileLock` with an `openclaw.json.lock` sidecar in the config IO/mutation layer, ensure gateway, CLI, doctor, plugin/runtime config writes all route through it, and add a regression test with concurrent writers that proves both changes survive and the final file remains valid JSON.
