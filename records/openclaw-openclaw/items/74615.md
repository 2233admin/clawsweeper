## Summary

Keep open. Current main still has static, per-message group mention gating with narrow implicit reply/thread and command bypasses; it does not implement a configurable timer that lets unmentioned follow-up messages trigger after an initial mention. The related reports cover mention detection, command bypass, or model/context routing, but they do not supersede this cross-channel attention-window request.

## What This Changes

Request an opt-in group-chat attention window that temporarily processes unmentioned follow-up messages after the agent is mentioned.

## Best Possible Solution

Define an opt-in shared attention-window contract with per-conversation expiry semantics, explicit defaults, platform limitations, tests, and docs, then wire channel plugins through that contract while preserving existing authorization and mention-gating boundaries.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure a group/channel with `requireMention: true`, send one mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The user problem is valid, but the proposed per-channel `attentionWindow` field needs maintainer design because the maintainable fix likely spans shared policy, plugin contracts, timer state, privacy semantics, and platform-specific delivery constraints.
