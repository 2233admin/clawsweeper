## Summary

Keep open. Current main still implements static per-message mention gating plus narrow implicit mention and command bypasses; it does not implement a configurable timer that lets unmentioned group follow-ups trigger after an initial mention. The prior ClawSweeper comment remains consistent with current source, docs, and schema inspection, and the related reports cover distinct mention-detection, command-bypass, or routing gaps rather than this attention-window behavior.

## What This Changes

Add a configurable attention window for group/channel chats so unmentioned follow-up messages can trigger for a limited duration after a bot mention, then revert to mention-only behavior.

## Best Possible Solution

Define an opt-in shared attention-window policy with per-conversation expiry, reset-on-mention behavior, clear defaults, and channel-specific delivery limits, then expose it through the config schema, SDK/channel contracts, bundled channel adapters, docs, and targeted regression tests.

## Reproduction Assessment

Yes. Configure a group/channel with requireMention true, send one mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the no-mention skip/store path in the shared gate and channel adapters.

## Solution Assessment

Unclear. The user problem is coherent, but the example channels.slack.attentionWindow shape is not enough for a cross-channel feature; the safer direction is a shared opt-in policy plus channel-specific wiring rather than a Slack-only config field.
