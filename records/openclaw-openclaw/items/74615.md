## Summary

Keep open. Current main implements static group mention gating plus narrow explicit/implicit mention bypasses, but it does not implement a configurable, time-limited attention window after a mention.

## What This Changes

The issue requests a configurable group-chat attention window that temporarily processes follow-up messages after an initial mention before returning to mention-only mode.

## Best Possible Solution

Define a shared, opt-in group attention model that preserves existing authorization gates, has explicit per-conversation expiry semantics, documents config and defaults, and is wired through the channel/plugin contract only after maintainers choose the ownership boundary.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure a group/channel with `requireMention: true`, send one mentioned message, then send a follow-up without an explicit/implicit mention; current main evaluates only the follow-up message facts and takes the no-mention skip/store path.

## Solution Assessment

No current solution is present on main. The requested direction is plausible, but the best implementation needs maintainer design first because it changes cross-channel privacy, token usage, config schema, and plugin contract behavior.
