## Summary

Keep open. Current main still has static, per-message group mention gating with explicit/implicit mention and command bypasses, but no configurable attention window or remembered prior-mention state. The related reports cover command bypasses, mention detection, or model/context routing rather than this time-limited cross-channel behavior.

## What This Changes

Request an opt-in attention window that processes unmentioned group-chat follow-ups for a configurable duration after the agent is mentioned, then returns to mention-only behavior.

## Best Possible Solution

Define an opt-in shared attention-window contract with per-conversation expiry state, explicit config/defaults, plugin SDK guidance, and channel-specific limitations, then wire it through the shared mention policy and affected channel plugins while preserving existing authorization and visible-reply safeguards.

## Reproduction Assessment

Yes. Configure a group/channel with `requireMention: true`, send a mentioned message, then send an unmentioned follow-up; current main evaluates the follow-up only from its own mention facts and takes the no-mention skip/store path.

## Solution Assessment

Unclear. The requested behavior is valid, but the proposed single `channels.slack.attentionWindow` shape is not yet proven to be the best solution because the maintainable design likely needs shared cross-channel state, config, SDK, and privacy semantics.
