## Summary

Keep open. Current main has static mention gating plus implicit reply/thread exceptions, but no configurable timed attention window that temporarily accepts all group messages after a mention.

## What This Changes

The issue requests a configurable group-chat attention window that temporarily treats a room as listening to all messages after the bot is mentioned.

## Best Possible Solution

Add an opt-in, bounded attention-window policy through the shared mention-gating/channel policy layer, then wire only the channels that can actually observe unmentioned group traffic and document platform limits. The implementation should keep static `requireMention` as the default and include focused tests for mention start, reset, expiry, and pending-history behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The absence is high-confidence from current-main source/docs/config inspection: after a mention, later unmentioned group messages still follow the same `requireMention` gate and are skipped or stored as pending history unless another existing implicit mention rule applies.

## Solution Assessment

Unclear. The requested behavior is useful, but the best fix is likely an additive shared policy/state seam plus channel-specific capability checks, not ad hoc per-channel toggles or making `requireMention: false` the workaround.
