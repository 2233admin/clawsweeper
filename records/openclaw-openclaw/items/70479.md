## Summary

Keep open. Current main still rotates the active session mapping for embedded overflow payloads and thrown compaction failures; the later main-side tail-replay mitigation keeps some continuity after rotation but does not implement this PR’s preserve-the-existing-session behavior. This PR remains a focused implementation candidate for the linked WebChat/session orphaning bug, pending normal maintainer review and rebase.

## What This Changes

This PR removes the compaction-failure session-reset callback from auto-reply execution, returns explicit retry, `/compact`, and `/new` guidance, and adds docs, changelog, and regression coverage for embedded overflow and thrown compaction-failure paths.

## Best Possible Solution

Land this PR or an equivalent rebased fix that stops compaction/overflow recovery failures from silently rewriting the active WebChat session mapping, keeps explicit operator-controlled retry, `/compact`, and `/new` guidance, and preserves regression coverage for both embedded error payloads and thrown compaction failures while keeping role-ordering reset behavior intact.
