## Summary

Keep this PR open. Current main still has the compaction-failure path that rotates the active auto-reply/WebChat session mapping, while this PR is an active, focused implementation candidate for the linked #70472 bug and is not superseded by the related overflow-budget work.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate for #70472. Maintainers should review and land this PR, or an equivalent fix, that preserves the active WebChat session mapping when overflow recovery or auto-compaction fails, surfaces explicit retry /compact /new guidance, keeps role-ordering reset behavior separate, and retains regression coverage for embedded overflow payloads, thrown compaction failures, and unchanged session-store/sessionId state.
