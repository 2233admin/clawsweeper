## Summary

Keep open. Current main still lacks the full embedded tool-result `ReplyPayload` preservation contract this PR targets, so the work is not obsolete; however, the branch should not merge as-is because its final payload assembly can suppress the explicit timeout error when a timed-out run only buffered tool-result payloads.

## What This Changes

The PR buffers embedded Pi runner tool-result `ReplyPayload` values, resets the buffer across compaction retries, prepends buffered payloads for non-live final consumers, and adds regression tests plus a changelog entry.

## Best Possible Solution

Rebase the useful payload-preservation work onto current main, preserve full `ReplyPayload` metadata for non-live final consumers, avoid replaying live-streamed tool results, clear buffered data across compaction retries, make timeout synthesis depend on assistant/final reply production rather than buffered tool callbacks, and keep the changelog entry credited per repo policy.
