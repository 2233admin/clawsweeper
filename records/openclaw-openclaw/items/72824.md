Keep open. Current main still has the reported Copilot capability gap, and same-author PR #72825 is already the focused implementation path, so this should stay open until that PR or a canonical equivalent lands.

## What This Changes

## Best Possible Solution

Review and land #72825 or an equivalent provider-owned fix that keeps Copilot dynamic model resolution, seeded metadata, thinking profiles, and session/status context lookup consistent for the two Claude Opus 1m variants; broader live /models discovery can remain a separate follow-up decision.

Yes. Source inspection on current main is enough: selecting github-copilot/claude-opus-4.7-1m-internal reaches 128k/8192 synthetic/default Copilot metadata, lacks xhigh in the provider thinking profile, and has no Copilot 1m context shortcut for status/session lookup.

## Solution Assessment

Yes for the narrow bug. A Copilot-local known-capability override plus targeted context/status coverage is the smallest maintainable fix; live upstream discovery would be cleaner long term but is not required before fixing these known divergent ids.
