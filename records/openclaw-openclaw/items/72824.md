## Summary

Keep open. Current main still has the reported GitHub Copilot capability mismatch: unknown Claude Opus 1m IDs fall through to the synthetic 128k/8192-token model path, and the xhigh thinking allowlist does not include the Claude 4.7 1m internal ID. The issue is also paired with open PR #72825 from the reporter, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks the GitHub Copilot plugin to expose 1,000,000-token context, 64,000-token output, and xhigh support where applicable for Claude Opus 4.6/4.7 1m Copilot model IDs.

## Best Possible Solution

Keep this issue open and resolve it through #72825 or an equivalent GitHub Copilot plugin change that keeps dynamic model resolution, seeded defaults, and thinking-level support aligned for the Claude Opus 1m IDs.

## Reproduction Assessment

Yes. A static reproduction path exists on current main: `resolveCopilotForwardCompatModel` for an unregistered `claude-opus-4.7-1m-internal` or `claude-opus-4.6-1m` falls through to 128k/8192 synthetic metadata, and xhigh is omitted because the Claude ID is not in the Copilot xhigh allowlist.

## Solution Assessment

Unclear. The linked PR’s curated table is a narrow maintainable patch for the reported IDs, but the issue body also identifies provider discovery from Copilot’s /models endpoint as the more complete long-term direction.
