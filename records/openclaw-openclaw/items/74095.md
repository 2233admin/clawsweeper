## Summary

This issue should stay open. Current main still has task and TaskFlow lifecycle statuses, delivery statuses, audit finding codes, and raw error/summary text, but it does not provide the requested shared blocker/failure taxonomy for reusable machine-readable classes such as approval waits, external waits, provider/config/transport/tool errors, timeouts, interruptions, and unknown failures. Related #64413 is still open and narrower to delivery-failure reasons, so it complements rather than supersedes this broader task taxonomy request.

## Best Possible Solution

Keep this issue open for maintainer-owned additive design. The likely path is a shared task blocker/failure classification type plus mapping helpers in the task domain, optional fields on TaskRecord and TaskFlow metadata, propagation into task audit JSON, summaries, CLI/status output, plugin runtime DTOs, and docs. Related #64413 should remain linked as narrower delivery-reason input, not as a complete replacement for this broader taxonomy request.
