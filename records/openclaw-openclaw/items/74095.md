## Summary

Keep this issue open. Current main still has coarse task statuses, Task Flow waiting/blocked state, operational audit codes, and free-form error/summary text, but it does not have a shared machine-readable blocker/failure taxonomy; related #64413 is narrower delivery-failure work and does not supersede this broader task-system request.

## What This Changes

Request to add a shared task blocker/failure taxonomy and reuse it across task audits, summaries, retry decisions, and user-facing status output.

## Best Possible Solution

Define an additive shared task blocker/failure classification contract, map existing task/task-flow/cron/agent-run conditions into it, expose it through persisted records and DTOs where appropriate, and document how consumers should handle unknown or unclassified cases while using #64413 as delivery-specific input.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs inspection plus targeted search showing current main has statuses and audit codes but no shared task blocker/failure taxonomy.

## Solution Assessment

Yes, a centralized additive taxonomy is the maintainable direction, but the exact names, persistence fields, DTO compatibility, and rollout order need maintainer design rather than an unreviewed one-off worker patch.
