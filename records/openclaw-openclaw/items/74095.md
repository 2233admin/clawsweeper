## Summary

Keep this open. Current main still has coarse task statuses, Task Flow waiting/blocked state, delivery statuses, audit codes, and prose summary/error fields, but not the requested shared machine-readable blocker/failure taxonomy; related #64413 is narrower delivery-failure work and does not supersede this broader task-system contract request.

## What This Changes

Introduce a shared machine-readable blocker and failure taxonomy for tasks so audits, summaries, retry decisions, and status output can distinguish waiting, blocked, failed, and transient recoverable failures consistently.

## Best Possible Solution

Define an additive shared task blocker/failure classification contract with explicit unknown handling, map task, Task Flow, delivery, audit, and agent-run outcomes into it, and expose it through compatible persisted records, runtime DTOs, CLI JSON, user-facing status, and docs while using #64413 as delivery-specific input.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs inspection, which shows current main has status/prose fields and operational audit codes but no shared blocker/failure taxonomy.

## Solution Assessment

Yes, a centralized additive taxonomy is the maintainable direction, but the exact contract and rollout boundaries should be chosen before implementation so consumers do not branch on another ad hoc field.
