## Summary

Keep this open. Current main still exposes task status, delivery status, Task Flow blocked/waiting state, audit codes, and prose summaries, but it does not have the requested shared machine-readable blocker/failure taxonomy; related #64413 is narrower delivery-failure work, not a superseding fix.

## What This Changes

Request to add a centralized blocker/failure classification contract for task records, audits, summaries, user-facing status output, and future retry logic.

## Best Possible Solution

Define an additive shared task blocker/failure classification contract, map existing task, Task Flow, audit, delivery, and agent-run conditions into it, expose it through persisted records and runtime DTOs where compatible, document unknown/unclassified handling, and use #64413 as delivery-specific input rather than the whole solution.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs inspection plus focused search showing current main has coarse statuses, audit codes, and prose fields but no shared task blocker/failure taxonomy.

## Solution Assessment

Yes, a centralized additive taxonomy is the maintainable direction, but the exact contract and rollout boundaries should be chosen before implementation so consumers do not branch on another ad hoc field.
