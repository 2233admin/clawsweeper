## Summary

Keep this issue open. Current main still exposes coarse task statuses, Task Flow waiting/blocked state, operational audit codes, and raw error/summary text, but it does not provide the requested shared blocker/failure taxonomy for approval waits, external waits, provider/transport/config errors, timeouts, interruptions, unknown failures, or transient/recoverable classification. Related #64413 remains open and narrower to delivery failure reasons, so it complements rather than supersedes this broader task taxonomy request.

## Best Possible Solution

Keep this open for a maintainer-owned additive design: define shared task blocker/failure classification types and mapping helpers, add optional persisted/DTO fields where appropriate, map known task/flow/cron/agent-run conditions, and reuse the taxonomy in audits, task summaries, CLI/status output, plugin runtime DTOs, and docs. Keep #64413 linked as narrower delivery-failure input rather than the canonical replacement.
