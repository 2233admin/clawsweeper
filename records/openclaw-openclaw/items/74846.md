## Summary

Keep open. This is a focused contributor PR for a real Moonshot/Kimi thinking-mode replay bug, and current main does not yet add `reasoning_content` to assistant tool-call messages before Moonshot requests. I found no blocking correctness finding in the proposed diff; the remaining action is normal maintainer review and validation.

## What This Changes

The PR adds a Moonshot payload helper that inserts empty `reasoning_content` on assistant messages with `tool_calls` when Moonshot thinking is enabled, plus regression tests for enabled, disabled, and pre-existing reasoning-content cases.

## Best Possible Solution

Land a narrow Moonshot-only outgoing payload repair that preserves existing `reasoning_content`, adds an empty placeholder only for assistant tool-call messages when final Moonshot thinking is enabled, and keeps generic OpenAI-compatible replay behavior unchanged.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a Moonshot/Kimi request with thinking enabled and replayed assistant `tool_calls` lacking top-level `reasoning_content`; current main does not fill that field, and the official Kimi docs require it for multi-step tool calling.

## Solution Assessment

Yes, with validation. The proposed provider-wrapper patch is the narrowest maintainable fix I found because it runs at the final outgoing request boundary, preserves existing reasoning content, and mirrors the existing DeepSeek compatibility pattern.
