## Summary

Current main still lacks the DeepSeek provider `resolveThinkingProfile` hook that this PR adds. DeepSeek V4 transport already maps `xhigh` and `max` to DeepSeek `reasoning_effort: "max"`, but command/directive validation resolves allowed levels from the provider thinking profile first; without a DeepSeek profile, current main falls back to base levels and rejects `xhigh|max`. The PR remains a focused implementation candidate for the linked DeepSeek report, with a narrow provider/test/docs diff and no security-sensitive changes in the provided file set.

## Best Possible Solution

Keep this PR open for maintainer review and likely merge after resolving any desired helper deduplication. The best path is to land the provider-owned DeepSeek V4 `resolveThinkingProfile`, preserve regression coverage proving V4 exposes `xhigh|max` while non-V4 DeepSeek models do not, update the DeepSeek/thinking docs, and then close the linked DeepSeek report as implemented after merge.
