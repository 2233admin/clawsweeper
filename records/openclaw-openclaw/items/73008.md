## Summary

Keep open for maintainer review. Current main still lacks a DeepSeek provider `resolveThinkingProfile`, so `/think xhigh` and `/think max` can be rejected before transport even though the DeepSeek V4 stream wrapper already maps those levels correctly. The PR is focused on the provider-owned hook, local matcher reuse, regression coverage, and docs; I found no blocking correctness or security issue in the proposed patch.

## What This Changes

This PR adds a DeepSeek V4 provider thinking profile with `xhigh` and `max` support, shares the V4 model matcher between provider and stream code, adds DeepSeek regression coverage, and documents the supported levels.

## Best Possible Solution

Land the provider-owned DeepSeek V4 thinking profile with the shared local matcher, keep the positive V4 and negative non-V4 regression coverage, publish the docs updates, then close the linked DeepSeek bug while leaving the later closed duplicate PR as superseded by this implementation.
