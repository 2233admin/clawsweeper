## Summary

Current main still lacks the DeepSeek provider `resolveThinkingProfile` hook that this PR adds. The existing DeepSeek V4 stream wrapper can map `xhigh` and `max` to DeepSeek `reasoning_effort: "max"`, but current directive validation resolves supported levels from the provider thinking profile first, falls back to the base `off|minimal|low|medium|high` set, and rejects explicit `xhigh|max` before transport. The PR remains a focused provider/test/docs implementation candidate for #72893; after the latest update it also shares the V4 model matcher between provider registration and stream wrapping. The security pass found no workflow, dependency, lockfile, install script, secret, or package-publishing changes in the PR file set.

## Best Possible Solution

Keep this PR open for maintainer review and likely merge. The best path is to land the provider-owned DeepSeek V4 `resolveThinkingProfile`, keep the shared local V4 model matcher, preserve regression coverage proving V4 exposes `xhigh|max` while non-V4 DeepSeek models do not, update the DeepSeek and thinking docs, then close #72893 as implemented after the PR merges.
