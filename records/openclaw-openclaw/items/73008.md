## Summary

Current main still lacks the DeepSeek provider thinking-profile hook, so native DeepSeek V4 models still fall back to the base thinking levels and reject explicit `xhigh`/`max` before request construction. This PR remains a focused implementation candidate, with no security-sensitive file changes, but the latest head has failing CI checks that need review before merge.

## What This Changes

This PR adds a DeepSeek V4 `resolveThinkingProfile` exposing `off|minimal|low|medium|high|xhigh|max`, shares the V4 model-id matcher, updates DeepSeek/thinking docs, and adds provider regression coverage.

## Best Possible Solution

Keep this PR open for maintainer review and CI follow-up. The best path is to land the provider-owned DeepSeek V4 `resolveThinkingProfile`, keep the shared local V4 matcher, preserve the V4-positive/non-V4-negative regression test, update the DeepSeek and thinking docs, and then close #72893 after merge.
