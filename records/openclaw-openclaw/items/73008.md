## Summary

Keep open. Current main still lacks a DeepSeek provider thinking profile, so explicit `/think xhigh` and `/think max` validation still falls back to the base levels even though the DeepSeek V4 transport already maps those levels correctly. The PR is a narrow provider-owned fix for the linked DeepSeek bug, with docs/tests in the right surface; it needs maintainer review and current-head validation rather than cleanup closure.

## What This Changes

The PR adds a DeepSeek V4 provider thinking profile and shared V4 model matcher, updates DeepSeek/thinking docs, and tests that V4 models expose `xhigh`/`max` while older DeepSeek models do not.

## Best Possible Solution

Land a provider-owned DeepSeek V4 `resolveThinkingProfile` that advertises `off|minimal|low|medium|high|xhigh|max`, keep the shared local V4 matcher, preserve positive V4 and negative non-V4 regression coverage, update the public DeepSeek and thinking docs, then close #72893 and related duplicate reports as implemented.
