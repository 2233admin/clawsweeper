## Summary

Keep open for maintainer review. Current `main` still lacks the DeepSeek provider `resolveThinkingProfile` hook, while the PR addresses a valid narrow gap: validation rejects `xhigh`/`max` before the existing DeepSeek V4 stream wrapper can map those levels to `reasoning_effort: "max"`. I did not find a concrete correctness or security blocker in the proposed patch.

## What This Changes

This PR adds a DeepSeek V4 provider thinking profile, a shared DeepSeek V4 model matcher, regression coverage, and docs so native V4 Flash/Pro expose `/think xhigh` and `/think max` while non-V4 DeepSeek models stay on the base profile.

## Best Possible Solution

Land the provider-owned DeepSeek V4 `resolveThinkingProfile` with `off|minimal|low|medium|high|xhigh|max`, keep the shared local V4 matcher used by provider registration and stream patching, preserve positive V4 and negative non-V4 regression coverage, update the public DeepSeek/thinking docs, then close the matching DeepSeek profile-exposure reports as implemented.
