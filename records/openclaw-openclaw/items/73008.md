## Summary

Keep this PR open. Current main still lacks the DeepSeek provider `resolveThinkingProfile` hook, so native DeepSeek V4 `/think xhigh` and `/think max` remain rejected before the existing V4 stream wrapper can map them to `reasoning_effort: "max"`. The PR is a narrow, still-relevant implementation candidate for #72893 with matching tests/docs and no security-sensitive diff surface.

## Best Possible Solution

Keep the PR open for maintainer review and likely merge. The best path is to land the DeepSeek-owned `resolveThinkingProfile` for V4 Flash/Pro, keep the regression test proving V4 exposes `xhigh|max` while non-V4 DeepSeek models do not, keep the docs updates, and optionally deduplicate the V4 model-id helper or add a changelog entry if maintainers want the user-visible fix recorded. After merge, #72893 can close as implemented.
