## Summary

Current main at 610e575844b21ca9f1501c53f1255af37a1d7d34 still lacks the DeepSeek provider `resolveThinkingProfile` hook, so native DeepSeek V4 models still fall back to the base thinking levels and explicit `/think xhigh` or `/think max` remains rejected before the existing V4 request wrapper can map the payload. This PR is still a focused implementation candidate for the linked DeepSeek issue, with docs and regression coverage, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the provider-owned DeepSeek V4 `resolveThinkingProfile`, keep the regression test proving V4 exposes `xhigh|max` while non-V4 DeepSeek models do not, update the DeepSeek/thinking docs, and optionally deduplicate the V4 model-id helper before merge. After merge, #72893 can close as implemented.
