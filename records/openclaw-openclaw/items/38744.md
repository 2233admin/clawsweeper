## Summary

Current main still does not implement this PR's intended native iOS ElevenLabs realtime WebSocket STT path for Talk Mode. iOS continuous and push-to-talk capture remain Apple SFSpeechRecognizer-backed; adjacent ElevenLabs realtime transcription exists for Voice Call and gateway/browser realtime voice sessions, but those surfaces do not supersede this iOS Talk Mode PR. Keep it open for maintainer architecture, reliability, and security review.

## What This Changes

The PR adds native iOS ElevenLabs realtime and batch STT clients, a WAV PCM conversion helper, TalkModeManager integration for ElevenLabs STT in continuous and push-to-talk capture, STT model/language settings UI, and Swift source list entries.

## Best Possible Solution

Keep this PR open for maintainer/product review. If the feature proceeds, first choose the architecture: harden a native iOS ElevenLabs realtime STT client, or route iOS Talk through a provider-generic gateway realtime transcription/voice path. Before merge, address WebSocket readiness, PTT committed-transcript waiting, fallback permission and recovery behavior, settings persistence, docs, security review, and focused iOS coverage.
