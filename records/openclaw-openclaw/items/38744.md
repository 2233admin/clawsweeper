## Summary

Keep open. Current main still does not implement this PR's native iOS ElevenLabs realtime WebSocket STT path: iOS Talk Mode remains Apple SFSpeechRecognizer-backed, while the existing ElevenLabs realtime work is scoped to Voice Call/gateway realtime surfaces. The PR also has unresolved reliability and credential-boundary review comments, so it needs maintainer architecture and security review before merge.

## What This Changes

The PR adds native iOS ElevenLabs realtime and batch STT clients, a WAV PCM conversion helper, TalkModeManager integration for continuous and push-to-talk capture, STT model/language settings UI, and Swift source-list entries.

## Best Possible Solution

Keep the PR open until maintainers choose the architecture for iOS Talk STT. A mergeable direction should either harden the native iOS ElevenLabs client with robust readiness, fallback, PTT, settings, docs, and device validation, or replace it with a gateway/provider-generic realtime Talk path that keeps vendor credentials and provider behavior gateway-owned.
