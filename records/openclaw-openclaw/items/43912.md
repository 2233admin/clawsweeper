## Summary

Keep this PR open. Current main still returns `null` before bundled lookup for scoped/non-bare install specs, and the current helper test explicitly asserts that `@openclaw/voice-call` skips bundled pre-NPM resolution. The PR is not implemented or superseded; it remains a focused resolver change that needs maintainer/product-docs alignment. Security review of the provided PR diff found no workflow, dependency, lockfile, script, package-publishing, permissions, or secrets-handling changes; the main risk is resolver semantics, not new code execution surface.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers want exact bundled `npmSpec` matches to win before ClawHub/npm, land or adapt the narrow helper change: trim the raw spec, keep plugin-id lookup for bare specs, use exact `npmSpec` lookup for scoped/non-bare specs, let non-matches continue to ClawHub/npm, preserve `npm:` as the explicit npm-only path, update helper and CLI install tests, and align public plugin install docs.
