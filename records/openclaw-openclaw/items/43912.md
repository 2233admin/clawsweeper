## Summary

Keep open. Current main still skips bundled pre-NPM planning for scoped npm specs, while the PR addresses that narrow install-flow gap by reusing the existing exact bundled npmSpec lookup. The patch appears functionally sound, with only a low-priority warning-text issue before merge.

## What This Changes

The PR changes the CLI bundled pre-NPM plugin install planner so scoped/non-bare specs can resolve by exact bundled npmSpec, and updates helper tests for matching and non-matching scoped specs.

## Best Possible Solution

Land or adapt the narrow resolver change so exact bundled npmSpec matches win before ClawHub/npm while unmatched scoped/versioned specs still fall through, runtime duplicate precedence remains unchanged, and bypass guidance points users to npm:<package>.
