## Summary

Keep this PR open. Current main still does not implement the central behavior: QMD session transcript exports are configured and refreshed during QMD updates, but QMD does not subscribe to transcript-update events and its file watcher deliberately skips session collections. The PR remains a plausible implementation candidate, especially because the earlier PR #65017 was closed as superseded by this PR, but the latest PR context shows a broad mixed-surface diff that needs narrowing or splitting before merge.

## Best Possible Solution

Keep this PR open, but ask for a narrower memory-core patch or a clear split. The best implementation is to subscribe QMD session exports to canonical transcript-update events when `memory.qmd.sessions.enabled` is active, filter updates to the current agent's sessions directory, debounce refreshes, use the forced queued QMD update path so transcript writes during an in-flight update are not dropped, clean up listener/timer state on close, and cover the behavior with focused QMD tests. Review or remove the unrelated bundled-plugin trust/loader, doctor, gateway, browser, and config changes separately.
