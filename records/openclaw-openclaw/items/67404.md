## Summary

Keep this PR open. Current main still does not implement the central behavior: QMD session exports are configured and exported during QMD updates, but QMD does not subscribe to transcript-update events, and its filesystem watcher deliberately skips session collections. The PR remains a plausible implementation candidate, though it needs maintainer review or splitting because the provided PR context shows unrelated plugin-loader, browser trash, Signal installer, doctor, channel, and gateway/test changes riding along with the memory-core fix.

## Best Possible Solution

Keep this PR open, but ask for a narrower memory-core patch or a clear split. The best implementation is to subscribe QMD session exports to canonical transcript-update events when qmd.sessions.enabled is active, filter events to the current agent's sessions directory, debounce refreshes, clean up listener/timer state on close, and use the forced queued session-delta update path so transcript writes during an in-flight qmd update are not dropped. Separately review or remove the unrelated security-sensitive and loader/test changes before merge.
