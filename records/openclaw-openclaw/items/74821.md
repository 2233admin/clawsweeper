## Summary

Keep open. This is a member-authored PR with the protected `maintainer` label, and current `main` does not already include the requested gateway PR quality guard. The diff is directionally aligned with the existing CodeQL quality workflow, but it still needs maintainer review because it changes CI topology and appears to fan gateway-only PRs into the existing plugin quality jobs too.

## What This Changes

The PR adds gateway protocol/server-method pull-request paths and a manual `gateway-runtime-boundary` profile to the CodeQL Critical Quality workflow, expands the gateway CodeQL config to cover top-level gateway dispatch files, and updates the CI docs.

## Best Possible Solution

The desired end state is a documented CodeQL Critical Quality PR guard that covers gateway protocol/server-method changes with explicit shard scope, no unintended plugin-shard fan-out, workflow sanity proof, and observed PR-run proof before landing.
