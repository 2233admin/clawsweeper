## Summary

Keep open. Current main still lacks a directory-backed cron job store, the issue remains the canonical tracker, and open PR #41206 is already the active implementation candidate that should be reviewed or replaced before this issue is closed.

## What This Changes

Request support for one cron job definition per JSON file in a jobs.d/jobs directory so NixOS and GitOps setups can declaratively manage cron jobs without merging a single jobs.json file.

## Best Possible Solution

Land or rework #41206, or an equivalent additive implementation, so existing cron.store and jobs.json users remain compatible while a documented directory-store mode deterministically loads per-job JSON definitions, safely handles lifecycle operations, and updates schema/help, doctor, docs, and focused cron store/Gateway tests.
