## Summary

Current main still does not implement directory-backed cron job definitions. Cron definitions are loaded and saved through one aggregate `cron.store` path, defaulting to `cron/jobs.json`, with runtime state split into `jobs-state.json`. This issue should stay open as the product request while open PR #41206 remains the implementation candidate; #53581 only solved the complementary runtime-state split.

## Best Possible Solution

Keep this issue open as the canonical product request. The best path is to review and either land #41206 or replace it with an additive implementation that preserves existing `cron.store`/`jobs.json` compatibility, adds a documented directory-store option, deterministically loads per-job JSON files, keeps runtime state compatible with the current sidecar model, and updates config schema/help/docs/doctor plus focused cron store and gateway tests.
