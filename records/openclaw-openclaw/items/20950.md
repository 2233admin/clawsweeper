## Summary

Keep open. Current main still implements cron persistence as one aggregate definition file plus a runtime-state sidecar, and the requested per-job directory store is not present in config, schema, Gateway startup, store load/save logic, or docs. The provided discussion also identifies open PR #41206 as the active implementation candidate, so this issue should remain the canonical tracker until that PR or an equivalent replacement lands.

## What This Changes

Add support for a cron jobs.d/per-job JSON directory so NixOS and GitOps users can manage individual cron job definitions without merging a single jobs.json file.

## Best Possible Solution

Ship an additive, documented directory-store mode for cron definitions that keeps existing cron.store and jobs.json users compatible, deterministically loads per-job JSON files, preserves the runtime-state sidecar model, and updates config schema/help, doctor behavior, docs, and focused cron store/Gateway tests.
