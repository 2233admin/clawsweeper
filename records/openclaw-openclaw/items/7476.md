## Summary

Keep this issue open. Current main still receives WhatsApp stickers as inbound media but does not provide an outbound WhatsApp sticker send path: WebP media is still treated as a generic image, no `asSticker`-style send option reaches the active listener, WhatsApp action dispatch only supports reactions/polls, and docs only describe inbound sticker placeholders. The related implementation PR #7502 is closed and unmerged, so there is no merged replacement to close against.

## Best Possible Solution

Implement outbound sticker support in the bundled WhatsApp plugin with an explicit sticker intent, preserve intended WebP sticker media before generic image optimization, route the active send API through Baileys `{ sticker: buffer }`, advertise/dispatch the WhatsApp sticker capability consistently, and update focused tests plus WhatsApp/message docs. Use related PR #7502 and the reporter's local patch as implementation context, but avoid relying only on empty-caption detection.
