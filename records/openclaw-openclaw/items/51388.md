## Summary

Keep this PR open. Current main does not implement the PR’s intended fix: `alsoAllow` is still not part of `ToolPolicyLike` or `SandboxToolPolicy`, `collectExplicitAllowlist()` still reads only `allow`, and both subagent policy constructors still drop `alsoAllow` in the `alsoAllow`-only case. The PR is a focused implementation candidate for a still-current plugin-tool allowlist gap, and the provided review discussion correctly identified the same subagent/session and HTTP-path risks that the later PR diff attempts to address.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to validate the bug with a focused regression test around an optional plugin tool allowed only through `tools.subagents.tools.alsoAllow`, then land a fix that preserves `alsoAllow` through subagent policy construction and includes it in `collectExplicitAllowlist()` for both PI and gateway HTTP tool resolution.
