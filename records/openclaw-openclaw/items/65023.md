## Summary

Keep open. Current main does not implement the proposed unified `google-genai` provider/auth mode, and this PR is still an active implementation vehicle, but the patch has blocking runtime, dependency, and security-sensitive provider-boundary issues before it can merge.

## What This Changes

The PR adds a core `google-genai` provider backed by `@google/genai`, a `google-genai-sdk` auth mode, Vertex project/location config fields, embedded-runner routing, live tests, and a new Google GenAI auth doc.

## Best Possible Solution

Revise the work so unified Google GenAI and Vertex auth live in the bundled Google provider/plugin architecture, or narrow it to the current `google-vertex` ADC/proxy failures, while preserving transport parity, dependency ownership, network policy controls, docs, and focused regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass Vertex config into runtime models:** `src/agents/google-genai-stream.ts:35-36`
  - body: `model.vertexai` is read here, but the existing runtime model builders only forward known fields such as baseUrl, api, headers, request, and params. A user who sets `models.providers.google-genai.vertexai.project` as documented will still reach this stream without `vertexai` values unless the same values are also in the environment.
  - confidence: 0.9
- **[P1] Allow keyless auth on every run surface:** `src/agents/model-auth.ts:419-421`
  - body: This new auth mode returns no API key, but only embedded-runner checks were updated to allow it. `/btw` still exempts only `aws-sdk` before calling `requireApiKey`, so a `google-genai-sdk` model fails before the new stream can run there.
  - confidence: 0.91
- **[P1] Declare the SDK where core imports it:** `src/agents/google-genai-stream.ts:1`
  - body: This core file imports `@google/genai`, but the root package does not declare that dependency; current main declares it as a Google plugin-local runtime dependency. Published installs and strict package resolution should not rely on a plugin-local dependency for core code.
  - confidence: 0.87
- **[P2] Forward tools and run options to the SDK:** `src/agents/google-genai-stream.ts:91-95`
  - body: The request sent to `generateContentStream` includes only model, contents, and an optional system instruction. It drops `context.tools`, tool choice, max-token and temperature controls, and the SDK-supported abort signal, so tool-dependent runs and caller limits do not behave like other transports.
  - confidence: 0.92
- **[P2] Preserve provider request policy for SDK calls:** `src/agents/google-genai-stream.ts:40`
  - body: Creating `new GoogleGenAI` and using the SDK's internal request path bypasses the existing Google transport's guarded fetch path, including configured proxy/TLS policy, capture hooks, and timeout handling. This provider should keep those controls or stay inside a plugin-owned runtime seam that explicitly handles them.
  - confidence: 0.84
- **[P2] Map streaming finish and usage metadata:** `src/agents/google-genai-stream.ts:99-124`
  - body: The streaming loop consumes only `chunk.text`, leaving usage at zero and `stopReason` as `stop` for all non-error completions. Max-token, safety, and tool-use finishes will be misclassified, and cost/usage accounting will be wrong for this provider.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces security-sensitive Google SDK network execution from core and changes dependency ownership without preserving existing provider request controls.

Concerns:

- **[medium] Direct SDK calls bypass provider network policy:** `src/agents/google-genai-stream.ts:40`
  - body: `new GoogleGenAI(...)` and `ai.models.generateContentStream(...)` use the SDK's internal network stack instead of OpenClaw's guarded provider fetch path, so configured proxy/TLS, capture, timeout, and network-policy controls may not apply to this provider.
  - confidence: 0.84
- **[medium] Core import relies on plugin-local dependency:** `src/agents/google-genai-stream.ts:1`
  - body: The new core stream imports `@google/genai`, but current main declares that package under the bundled Google plugin rather than root dependencies; this changes the supply-chain boundary and can break published package resolution unless dependency ownership is deliberate.
  - confidence: 0.87
