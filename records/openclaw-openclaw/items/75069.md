## Summary

Keep open. Current main contains useful partial mitigations for runtime-deps reinstall loops, dist-mirror reuse, and hardlink churn, but the bundled runtime mirror/load path is still synchronous, and an open closing PR is already paired with this issue for the broader fix.

## What This Changes

The issue asks OpenClaw to stop repeated synchronous bundled plugin runtime-deps and mirror preparation from blocking the Gateway during startup and pi-agent/tool invocation paths.

## Best Possible Solution

Keep this issue open until PR #75183 or an equivalent staged fix lands, then close only after packaged/global-install profiling shows bounded Gateway readiness and first-agent event-loop lag for bundled runtime-deps and mirror preparation.

## Solution Assessment
