## Summary

Keep open. Current main contains important mitigations for the bundled runtime mirror regression, but the remaining runtime-deps/mirror path is still partly synchronous and the issue is already paired with the open fix PR #75183, so it should not be closed before that PR lands or is explicitly replaced.

## What This Changes

The issue asks OpenClaw to stop bundled plugin runtime dependency and mirror preparation from synchronously blocking Gateway and pi-agent execution on packaged installs.

## Best Possible Solution

Keep this issue open and use #75183, or a narrower equivalent replacement, as the active implementation path: preserve the current hardlink and dist-mirror cache fixes, prevent repeated read-only/status/plugin paths from staging runtime deps, and validate packaged Gateway readiness and first-agent-run event-loop lag before closing.

## Reproduction Assessment

Yes. The issue has a high-confidence released-version reproduction with packaged install steps, lag output, CPU-profile attribution, and a smaller acpx-only startup-readiness repro in the discussion; current-main timing still needs a packaged profile rerun after the partial mitigations.

## Solution Assessment

Unclear. A full async migration would address event-loop starvation but crosses several runtime loader contracts; the active staged repair/caching PR is the right thing to review first, with profile-backed validation before deciding whether deeper async work is still required.
