## Summary

Keep this issue open. Current main has per-provider runtime discovery toggles for Bedrock, GitHub Copilot, Hugging Face, and Ollama, but the public docs still only document the Bedrock config path, there is no typed global model-discovery disable control, and the linked implementation PR #33365 is closed unmerged.

## Best Possible Solution

Keep this issue open. Document the shipped plugin-owned config paths in public docs (`plugins.entries.github-copilot.config.discovery.enabled`, `plugins.entries.huggingface.config.discovery.enabled`, and `plugins.entries.ollama.config.discovery.enabled`) and clarify the legacy `models.*Discovery.enabled` aliases while they remain supported. Then either add a typed global model-discovery disable control checked before `resolveImplicitProviders`, or split that global kill switch and `models.mode` interaction into a focused follow-up with tests.
