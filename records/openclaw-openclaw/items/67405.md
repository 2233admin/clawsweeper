## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Render timestamps outside assistant Markdown:** `src/tui/components/assistant-message.ts:13`
  - body: Prepending the timestamp to the assistant Markdown string means replies that start with Markdown-at-line-start syntax, such as fenced code blocks, headings, or lists, no longer start at column 0 and can render incorrectly. Keep the timestamp as a separate TUI element instead of mutating the Markdown payload.
  - confidence: 0.92
- **[P1] Require agent-qualified session matches:** `src/tui/tui-session-actions.ts:143`
  - body: `isCurrentSessionEntry` compares only the parsed rest key, so an event for `agent:ops:main` can match a TUI currently on `agent:main:main`; the event path can then switch `currentSessionKey` to the wrong agent/session. Match exact keys or compare both agent and rest.
  - confidence: 0.91
- **[P2] Preserve configured default sessions when omitted:** `src/tui/tui.ts:224`
  - body: When `--session` is omitted, this generated project key prevents `resolveTuiSessionKey` from seeing the empty input that activates configured `global` scope or main-key defaults. Users with those defaults will be moved into per-directory sessions unless they pass `--session` manually.
  - confidence: 0.9
- **[P2] Keep timestamps on pending user updates:** `src/tui/components/user-message.ts:7`
  - body: `UserMessageComponent` adds the timestamp only in the constructor, but `ChatLog.addPendingUser` updates an existing pending row via inherited `setText(text)`, replacing the content with raw text and dropping the timestamp. Preserve the prefix on updates or render it separately.
  - confidence: 0.87
- **[P2] Force deterministic 24-hour timestamps:** `src/tui/components/timestamp.ts:2-6`
  - body: `toLocaleTimeString` inherits the host hour cycle, so locales such as `en-US` can render `02:30:45 PM` instead of the intended fixed `HH:MM:SS` width. Pass `hour12: false` or use a deterministic formatter.
  - confidence: 0.8
- **[P3] Avoid duplicating the waiting status label:** `src/tui/tui-waiting.ts:46`
  - body: The template hardcodes `working` after the phrase, so the `working on the request` phrase renders as `working on the request... • working 3s`. Use a neutral label such as `elapsed` that cannot duplicate the chosen phrase.
  - confidence: 0.78
