## Summary

Keep open. Current main still has the reported macOS direct `wss://` first-use TOFU path, still does not plumb `gateway.remote.tlsFingerprint` into the macOS direct node session, and the closing PR is open and unmerged rather than a shipped fix.

## What This Changes

The issue asks OpenClaw to stop the macOS node direct `wss://` path from trusting an arbitrary first-use TLS certificate and to support configured remote TLS fingerprints for that path.

## Best Possible Solution

Harden the macOS node direct `wss://` path so first connection fails closed unless normal trust evaluation succeeds or an explicit configured/confirmed fingerprint matches, wire `gateway.remote.tlsFingerprint` through the macOS direct remote endpoint into TLS session construction, and add focused Swift/macOS IPC coverage before landing PR #50643 or a maintainer-selected replacement.
