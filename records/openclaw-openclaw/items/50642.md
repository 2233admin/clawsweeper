## Summary

Keep open. Current main still has the reported macOS direct `wss://` first-use TOFU path, the macOS config path still does not carry `gateway.remote.tlsFingerprint` into that session, and an open unmerged closing PR already exists for maintainer/security review.

## What This Changes

The issue requests hardening macOS node direct `wss://` gateway connections so a first-use certificate cannot be auto-trusted and used by a rogue gateway to control the node.

## Best Possible Solution

Harden the macOS node direct `wss://` path so first connection fails closed unless normal certificate trust succeeds or an explicit configured/confirmed fingerprint matches, carry `gateway.remote.tlsFingerprint` into the macOS session construction, and add focused Swift coverage for first-use TLS and configured fingerprint behavior.
