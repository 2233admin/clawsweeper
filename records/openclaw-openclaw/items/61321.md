## Summary

Keep this issue open. Current main still preserves structurally valid same-agent cross-root absolute sessionFile paths, and the same-author implementation PR remains open with closing syntax and an unresolved custom sessionsDir review concern.

## Best Possible Solution

Review and revise PR #61322 rather than closing this issue. The fix should remap stale same-agent agents/<agent>/sessions/<file>.jsonl paths relative to the active sessionsDir before transcript writes, preserve current cross-agent and unsafe-path behavior, and add regression coverage for both default state-dir and custom session.store cases.
