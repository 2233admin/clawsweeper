## Summary

Keep open. The PR addresses real CLI help and gateway guidance gaps, but the proposed patch is not merge-ready: the launcher change still misses common `--help`/`-h` paths, the gateway lock-error output would give conflicting stop/restart advice, and the gateway status output would duplicate auth diagnostics for unreachable targets.

## What This Changes

The PR updates CLI docs/help examples, tweaks lazy-subcommand help behavior, clarifies gateway service/runtime guidance, and adds gateway config/auth details to status output.

## Best Possible Solution

Revise the PR so help-trigger eager registration covers both `--help` and `-h` for command groups and nested help, gateway lock failures show one consistent supervised-service action, and gateway status auth diagnostics appear once in the appropriate warning or target context. Keep the docs/help improvements once the behavior matches them.

## Reproduction Assessment

Yes. The review findings are reproducible by source trace: argv length and help-flag checks show the nested help gap, and the gateway output paths show the duplicated/conflicting messages. I did not run live CLI commands because this review was required to keep the checkout read-only.

## Solution Assessment

No. The requested direction is maintainable, but this patch is not the best final form until the help trigger is centralized or broadened, the stale stop hint is removed or replaced, and the inline auth hint is guarded to avoid duplicate warnings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Expand all help aliases before lazy registration:** `openclaw.mjs:167`
  - body: The launcher guard only fires when argv length is greater than 4 and the literal `--help` is present. `openclaw models --help` has length 4, and `openclaw models auth -h` uses `-h`, so both still go through lazy placeholders and miss the nested subcommands this PR is trying to expose.
  - confidence: 0.9
- **[P2] Remove the conflicting supervised-gateway hint:** `src/cli/gateway-cli/run.ts:591-595`
  - body: This path would print the existing `stop it with openclaw gateway stop` line and then append the new `restart it with openclaw gateway restart` hint for the same lock failure. Users get opposite service-control actions in one error, so drop or replace the stale stop advice before adding the new hint.
  - confidence: 0.92
- **[P3] Avoid duplicating auth diagnostics for unreachable targets:** `src/commands/gateway-status/output.ts:244-248`
  - body: `buildGatewayStatusWarnings` already promotes auth diagnostics for unreachable probes into the top-level warning block. The new unconditional per-target `Auth hint` repeats the same diagnostic for those targets; guard it to reachable probes or otherwise suppress duplicates.
  - confidence: 0.88
