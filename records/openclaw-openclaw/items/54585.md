## Summary

Keep open. Current main does not implement this PR's requested compaction modes: the public config contract still only accepts `default` and `safeguard`, the embedded runner still has generic overflow precheck/recovery rather than user-selectable `warn`/`error`/`none` behavior, and overflow recovery still auto-compacts after provider context-overflow errors. The PR also has unresolved correctness review points and an unrelated bundled skill addition that needs maintainer review before merge.

## Best Possible Solution

Keep this PR open for maintainer and author iteration. A mergeable version should implement the new modes consistently across exported config types, Zod/generated schema, help text, defaults, embedded runner mode resolution, Pi auto-compaction disabling, and overflow recovery; route warn/error exits through normal hook/finalization semantics; reuse sanitized token accounting with safety margin and bounded reserve thresholds; remove or separately justify the unrelated bundled skill; fix the changelog attribution; and add focused config plus runner/overflow tests.
