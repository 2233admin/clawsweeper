## Summary

Keep open. Current main does not implement the proposed compaction modes, and the PR branch still has blocking correctness issues in warn-mode control flow and none-mode overflow recovery, plus smaller changelog and scope concerns that should be addressed before merge.

## What This Changes

This PR extends agent compaction config/schema/help with `warn`, `error`, and `none` modes, adds a Pi pre-prompt token guard and mode-aware auto-compaction disabling, lowers the Pi reserve floor, updates the changelog, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

A mergeable version should settle the semantics for each mode first, then implement them consistently across exported config types, strict/generated schemas, docs/help, Pi mode resolution, auto-compaction disablement, pre-prompt guard behavior, overflow recovery, hook/finalization behavior, and focused tests, with unrelated skill changes split out.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2929-2932`
  - body: The `warn` branch only emits the warning through `onBlockReply` and `assistantTexts.push`; it never sets `skipPromptForCompactionGuard`. Execution therefore reaches the later `activeSession.prompt(...)` block, so near-limit warn-mode turns still call the model even though the new docs/body say this mode stops before provider overflow.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:110`
  - body: Adding `none` here disables only Pi's built-in auto-compaction. The separate OpenClaw overflow-recovery loop still calls `contextEngine.compact()` and retries after context overflow, so `mode: "none"` can still auto-compact instead of failing at the model limit as the new help text promises.
  - confidence: 0.89
- **[P2] Preserve the default reserve floor:** `src/agents/pi-settings.ts:4`
  - body: Changing `DEFAULT_PI_COMPACTION_RESERVE_TOKENS_FLOOR` from 20,000 to 1,000 alters existing default/safeguard behavior for large-context models, despite the stated scope boundary. Current main already caps the 20,000 floor against the active context window, so keep that default or isolate any smaller threshold to the new guard logic.
  - confidence: 0.84
- **[P2] Apply the estimator safety margin:** `src/agents/pi-embedded-runner/run/attempt.ts:2923`
  - body: The new guard compares raw `estimateTokens` totals to the threshold. Existing preemptive compaction multiplies estimates by `SAFETY_MARGIN` because the estimator can undercount, so warn/error mode can still miss near-limit prompts and hit provider overflow without the same margin.
  - confidence: 0.78
- **[P3] Add the required changelog attribution:** `CHANGELOG.md:11`
  - body: This user-facing changelog entry omits the required `Thanks @...` attribution for added bullets. Add the credited GitHub username or remove the entry until the feature is ready.
  - confidence: 0.86
