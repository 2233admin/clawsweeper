## Summary

Keep open. Current main still accepts zero and negative Telegram reply-to message IDs, while this PR narrowly rejects those values in the Telegram plugin normalizer and adds focused regression coverage. The same fix is not present in the latest release or current main, and the PR discussion/reviews point to normal maintainer review rather than cleanup closure.

## What This Changes

This PR changes Telegram outbound reply-to message ID normalization to require positive integer numeric and string inputs, with outbound-parameter tests for zero, negative, positive, non-finite, and non-numeric values while preserving thread ID parsing.

## Best Possible Solution

Land the narrow Telegram plugin validation fix: require reply-to message IDs to be positive integers in the shared normalizer, keep negative Telegram group/thread ID parsing unchanged, and keep the regression coverage focused on reply-id normalization.
