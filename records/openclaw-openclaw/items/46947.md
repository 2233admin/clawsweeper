## Summary

Keep open. Current main still leaves sidebar markdown inline images at `max-width: 100%` and `max-height: 420px`, while this PR applies the same 420px/320px cap already used by the main chat markdown image rule. The linked regression remains open and the patch has no blocking correctness finding, so the remaining action is maintainer review, fresh validation, and merge judgment.

## What This Changes

This PR changes `ui/src/styles/chat/sidebar.css` so `.sidebar-markdown .markdown-inline-image` uses `max-width: min(100%, 420px)` and `max-height: 320px` instead of `100%` and `420px`.

## Best Possible Solution

The desired end state is a constrained sidebar markdown image rule, or an equivalent shared markdown image sizing rule if maintainers prefer deduplicating chat and sidebar styling, verified against the sidebar/webchat image regression path.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: render a base64 data URI markdown image in the markdown sidebar on a wide sidebar; current main emits `markdown-inline-image` and the sidebar CSS allows `100%` width / `420px` height while the main chat rule is capped.

## Solution Assessment

Yes. The proposed CSS-only change is the narrowest maintainable fix for the observed sidebar inconsistency; a shared rule could reduce duplication later but is not required to fix this behavior.
