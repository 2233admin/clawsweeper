## Summary

Keep this open. Current main still uses fixed token/overlap memory chunking with no heading-aware or section-aware strategy exposed through config, indexing, reindex metadata, or docs. The added issue comments describe the same production failure mode, and #68019 is an open, unmerged implementation candidate rather than shipped behavior.

## What This Changes

Request heading-aware or section-aware memory chunking to avoid blending adjacent Markdown sections in memory search, with entity extraction and graph memory as broader follow-up work.

## Best Possible Solution

Ship a maintained heading or section-aware strategy in the memory-host SDK path used by memory-core, expose it through config types/schema/help/docs, persist the effective chunking strategy for reindex invalidation, and add focused tests proving adjacent `##`/`###` sections no longer blend into one chunk. Treat entity extraction, graph memory, reranking, and nightly consolidation as explicit follow-ups unless maintainers intentionally keep this as an umbrella.
