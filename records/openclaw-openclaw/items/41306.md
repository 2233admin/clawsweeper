## Summary

Keep open. Current main still lacks the PR’s central `feishu_chat` behavior: no `list` action, no `accountId` override, no list pagination/sort params, and the chat tool still routes through the first enabled Feishu account. The PR is directionally useful, but it needs a rebase and the remaining disabled-account routing check before merge.

## Best Possible Solution

Keep this PR open for Feishu maintainer follow-up. The merge-ready path is to rebase onto current main, preserve `member_info` and the current `typebox` import shape, add the `feishu_chat` `list` action with pagination/sort forwarding, switch chat tool registration to the existing per-agent Feishu tool-account factory pattern, reject both disabled accounts and per-account `tools.chat: false`, and add focused tests for list behavior, routing priority, explicit override, disabled-account rejection, and disabled chat-tool rejection.
