## Summary

Keep this PR open. Current main still does not implement the proposed `feishu_chat` chat-directory `list` action or per-agent account routing for that tool, while adjacent Feishu tools already use the requested routing seam. The PR is not superseded by #68443, which targets message-history listing rather than chat-directory listing and tool-account routing. It still needs rebase/security cleanup before merge, especially around explicit disabled-account execution.

## Best Possible Solution

Keep this PR open for Feishu owner review and rebase. The best path is to add `list` to `feishu_chat`, route through the existing Feishu tool-account seam with `ctx.agentAccountId` and explicit `accountId` override, preserve current `member_info`, reject disabled accounts and disabled per-account chat tools at execution time, and add focused tests for listing, pagination/sort forwarding, contextual routing, explicit override, disabled-account rejection, and disabled-tool rejection. Treat #68443 as separate message-history work.
