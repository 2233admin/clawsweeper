## Summary

Keep this PR open. Current main still does not implement the proposed `feishu_chat` `list` action or per-agent account routing for that tool, and the related open Feishu PR tracks a different message-history surface rather than replacing this work.

## Best Possible Solution

Keep the PR open for Feishu owner review and rebase. The best path is to preserve current `member_info` behavior and `typebox` usage, add `list` to `feishu_chat`, register chat through the same factory/account-routing seam used by other Feishu tools, route by `ctx.agentAccountId` with explicit `accountId` override, reject disabled accounts and accounts with `tools.chat: false`, and add focused tests for listing, pagination/sort parameters, contextual routing, explicit override, disabled-account rejection, and disabled-tool rejection.
