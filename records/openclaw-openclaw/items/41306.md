## Summary

Keep this PR open. Current main still does not implement the proposed `feishu_chat` chat-list action or per-agent account routing for that tool, and the PR remains useful Feishu extension work. It should be rebased and tightened before merge, especially preserving current `member_info` behavior and rejecting disabled accounts on explicit `accountId` routing.

## Best Possible Solution

Keep the PR open for Feishu owner review and rebase. The best path is to preserve current `member_info` and `typebox` usage, add a `list` action to `feishu_chat`, register the chat tool through the same factory/account-routing seam used by wiki/drive/perm/bitable tools, route by `ctx.agentAccountId` with explicit `accountId` override, reject disabled accounts and disabled chat tools at execution time, and add focused tests for listing, pagination/sort parameters, per-agent routing, explicit override, disabled-account rejection, and disabled-tool rejection.
