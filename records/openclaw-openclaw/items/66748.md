## Summary

Keep open. Current main still queues exec-event heartbeat wakes for subagent session keys in both exec completion paths, while heartbeat resolution still maps forced subagent keys back to the main session. The linked open implementation PR is the right path to review rather than closing this issue early.

## What This Changes

The issue asks OpenClaw to stop background exec completions from subagent sessions from waking the parent/main heartbeat runner and producing spurious HEARTBEAT_OK traffic.

## Best Possible Solution

Keep this issue open until a fix lands. The desired end state is to skip requestHeartbeatNow for isSubagentSessionKey(sessionKey) in both maybeNotifyOnExit and emitExecSystemEvent, preserve enqueueSystemEvent/process-poll behavior, and add regression coverage for subagent session keys on both paths.
