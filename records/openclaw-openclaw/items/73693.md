## Summary

Keep this PR open. Current main still has the ACP timeout mirror and command paths that the PR targets, still hard-fails runtime-control application instead of fail-soft handling unsupported controls, still omits JSON-RPC data.details from ACP error text, and still drops the handle-aware getCapabilities input in the acpx wrapper. The PR is open, draft, and unmerged, so it remains a live ACP bug-fix candidate rather than cleanup.

## Best Possible Solution

Keep the PR open as the focused fix candidate. Review and tighten the unsupported-control classifier if needed, align the acpx dependency story with upstream openclaw/acpx#282 or a later published acpx release, preserve the local timeout runtime-options path, and land this PR or a cleaner superseding PR only after ACP manager/runtime/commands/acpx tests plus build or changed-gate proof and an ACP smoke verify that timeout is no longer sent as session/set_config_option.
