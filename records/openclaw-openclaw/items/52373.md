## Summary

Keep this issue open. Current main still does not implement a `configUrl`/`configButton` SKILL.md metadata contract, does not expose those fields through `skills.status`, and does not render a dedicated configuration action in the Control UI Skills list. This remains actionable OpenClaw core/dashboard work because an external skill or bundle can ship a page, but cannot make OpenClaw parse, validate, transport, and render new metadata without a core contract.

## Best Possible Solution

Keep the issue open and implement a small reviewed core/dashboard contract: add optional validated configuration-link metadata to SKILL.md parsing/types, include it in `skills.status` and UI types, render a safe external configuration action in the Skills list or detail view, and update the skills docs plus parser/status/UI tests. External skills can then use the surface without adding skill-specific behavior to core.
