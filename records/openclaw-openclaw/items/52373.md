## Summary

Keep this issue open. Current main still does not implement a `configUrl`/`configButton` SKILL.md metadata contract, does not include those fields in `skills.status`, and does not render a dedicated configuration action in the Control UI Skills list or detail view. The earlier ClawSweeper comment reached the same conclusion on a previous SHA; this review rechecked the current main SHA and the gap remains.

## Best Possible Solution

Keep this issue open and implement a small core/dashboard contract: add optional validated configuration-link metadata to SKILL.md parsing/types, include it in `skills.status` and UI types, render a safe external configuration action in the Skills list or detail view with default button text, and update the skills docs plus parser/status/UI tests. This belongs in OpenClaw core because external skills cannot make the dashboard parse, transport, and render new metadata by themselves.
