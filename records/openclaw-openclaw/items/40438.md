## Summary

Keep this PR open, but do not merge the branch as-is. The feature request is still valid on current main, but the branch is stale against the current memory host SDK/memory-core split and still has concrete merge blockers, including an unrelated process-wide DNS helper and an exclusion gap for excluded extra path roots.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config field across schema/types/help/status, filters directory basenames during memory file discovery, adds tests, and adds an unrelated root `ipv4-fix.cjs` helper.

## Best Possible Solution

Land a focused current-main implementation that defines additive, documented memory indexing exclusions across config schema/types/help/generated metadata, builtin discovery, sync/watch/status/reindex behavior, docs, and tests; remove the IPv4 helper and explicitly decide whether QMD needs matching exclusion semantics or a separate follow-up tied to #54463.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Remove the DNS override from this PR:** `ipv4-fix.cjs:1-4`
  - body: `ipv4-fix.cjs` changes process-wide DNS/address-family behavior if loaded, but this PR is about memory indexing. Keeping this unrelated root script creates an ops-sensitive side effect path that should not ship with this feature.
  - confidence: 0.9
- **[P2] Rebase onto the current memory host paths:** `src/memory/internal.ts:56`
  - body: The PR wires exclusion logic through `src/memory/internal.ts`, but current main's active discovery path is `packages/memory-host-sdk/src/host/internal.ts` with sync in `extensions/memory-core`. As written, the branch does not update the current production path and is likely to conflict or reintroduce stale files.
  - confidence: 0.88
- **[P2] Skip excluded extraPaths roots before walking:** `src/memory/internal.ts:122`
  - body: The exclusion check only runs for children inside `walkDir`, so an `extraPaths` entry whose basename is itself excluded, such as `node_modules` or `.venv`, is still walked and indexed. Check the normalized extra path root before calling `walkDir`.
  - confidence: 0.86
- **[P3] Normalize user excludeDirs entries:** `src/agents/memory-search.ts:218-225`
  - body: User-provided `excludeDirs` are merged without trimming or dropping blank strings, unlike `extraPaths`. A config value such as `" tmp "` will not match a `tmp` directory, so normalize entries before adding them to the resolved set.
  - confidence: 0.74
