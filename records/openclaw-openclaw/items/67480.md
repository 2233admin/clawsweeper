## Summary

Keep this PR open. Current main still has the nested skills-root bug, and the PR is a focused contributor fix with a relevant regression test, but the candidate-window edge case in the new early-exit heuristic needs maintainer review before merge.

## What This Changes

The PR updates skills discovery to avoid redirecting a configured skills root into its nested `skills` directory when the configured root already contains a loadable root-level skill, and adds a regression test for that layout.

## Best Possible Solution

Land a reviewed version of this PR that validates an already-loadable configured skills root before applying the repo-style nested-root heuristic, while preserving nested repo compatibility, containment checks, `SKILL.md` size caps, and regression coverage for the root-plus-nested layout.
