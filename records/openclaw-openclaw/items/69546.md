## Summary

Keep this issue open. Current main still routes QQBot inbound messages through the startup `ctx.cfg` snapshot, the generic router caches evaluated bindings by config and bindings references, and bindings are explicitly planned as no-restart config. PR #73567 is an open, unmerged closing PR for the same QQBot dynamic-binding bug, so the right maintainer action is to review/land or close that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and route maintainer attention to PR #73567. If that PR lands, close this issue as implemented by the merge; if it closes unmerged, the replacement fix should stay narrow: make QQBot resolve the active runtime config per inbound routing decision and add focused regression coverage for account-binding to peer-binding reroute without a gateway restart.
