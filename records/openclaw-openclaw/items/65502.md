## Summary

Keep open. Current main still lacks configurable per-candidate retry/backoff and a fallback context safe/strict mode; the active behavior tries each model candidate once, rejects the proposed model fallback policy shape in config validation, and keeps context-overflow recovery inside the current model’s runner instead of compacting for a smaller fallback candidate.

## What This Changes

The issue requests configurable per-candidate model fallback retries plus a safe/strict context mode that can compact history before falling back to a smaller-context model.

## Best Possible Solution

Keep the issue open and design a narrow agents fallback policy on the existing model config surface: bounded retries only for classified transient failures before side effects, observable retry metadata, and explicit safe/strict context behavior that compacts or refuses smaller-context candidates using the embedded runner’s existing compaction machinery.
