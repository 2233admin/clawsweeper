## Summary

Keep this issue open. Current main still has the reported failure class: recovered/fallback agent runs can complete successfully and return payloads while the delivery helper does not return structured delivery status, the Gateway finalizes success from agent completion, and skipped/empty lower-level delivery outcomes are not surfaced to the caller. The related PR #53961 is still open/draft in the provided context, so it is an implementation thread rather than a merged fix.

## Best Possible Solution

Keep this issue open as the canonical bug for recovered/fallback agent-run delivery drops. The best path is to land or revise #53961, or an equivalent structured delivery-status contract, then add a focused post-condition/retry at the `agentCommandFromIngress()` / `deliverAgentCommandResult()` boundary that re-resolves fresh session delivery context when delivery was requested but not attempted or did not complete. Add regression coverage for overloaded-error recovery and model-fallback races, including duplicate-send edge cases around empty delivery results and hook cancellations.
