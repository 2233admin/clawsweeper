## Summary

Keep open: current main and the latest release still use the narrow memory-wiki lint resolver, and the linked open PR #73722 is the active implementation candidate that explicitly closes this issue.

## What This Changes

The issue requests that memory-wiki lint resolve Obsidian wikilinks by title, slug/path/basename variants, fragment-stripped targets, and imported source-path suffixes.

## Best Possible Solution

Land PR #73722, or an equivalent memory-wiki-owned fix, that expands lint target lookup keys, strips fragments and queries consistently, preserves imported evidence provenance, and adds focused regression coverage for the reported link forms.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a page titled Alpha Database linked as [[Alpha Database#Evidence]] leaves the fragment-bearing target in linkTargets, while current lint only accepts syntheses/alpha-db-style paths and basenames.

## Solution Assessment

Yes. Fixing the memory-wiki lint resolver and its colocated tests is the narrow maintainable path; rewriting imported source evidence would damage provenance and should not be required.
