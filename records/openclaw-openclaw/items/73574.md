## Summary

Current main still has the reported memory-wiki lint gap. The lint resolver compares raw wikilink targets only against exact extensionless relative paths and basenames, while Obsidian links are extracted without fragment/case/slug normalization. The requested title, slug, fragment, and source-path suffix handling is not implemented, so this issue should stay open.

## Best Possible Solution

Keep this issue open. Implement the resolver in the memory-wiki plugin by extracting/shared normalized lookup helpers, then add title, slugified title/path basename, fragment-stripped Obsidian targets, and imported source-path/relative-path suffix keys in `extensions/memory-wiki/src/lint.ts`. Add focused positive and negative coverage in `extensions/memory-wiki/src/lint.test.ts`.
