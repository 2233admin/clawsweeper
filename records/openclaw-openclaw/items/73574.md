## Summary

Current main still does not implement the requested memory-wiki lint normalization. The resolver only accepts exact extensionless relative paths and basenames, while Obsidian wikilinks are extracted raw and no tests cover title, slug, fragment, or source-path suffix variants.

## Best Possible Solution

Keep this issue open. Implement a shared normalized wikilink lookup helper in `extensions/memory-wiki/src/lint.ts` or a small memory-wiki-local helper reused by compile/lint: strip fragments and queries, normalize slash/case/extension variants, add page title and `slugifyWikiSegment(title)` keys, path and basename keys, and imported `sourcePath`/`bridgeRelativePath`/`unsafeLocalRelativePath` suffix keys. Add focused positive and negative coverage in `extensions/memory-wiki/src/lint.test.ts` for title-style Obsidian links, slug variants, `[[Page#Fragment]]`, and imported source-path suffix references.
