## Summary

Keep this open because current main still has the narrow Memory Wiki broken-link resolver, and open PR #73722 is the active implementation candidate that uses closing syntax for this issue.

## What This Changes

The issue requests that `openclaw wiki lint` resolve title-style, slug, fragment-stripped, path/basename, and imported source-path Obsidian link variants without rewriting imported evidence.

## Best Possible Solution

Land PR #73722 or an equivalent Memory Wiki-owned fix that expands lint target lookup keys, strips fragments and queries consistently, preserves imported provenance, and adds focused regression coverage for the reported link forms.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a page at `syntheses/alpha-db.md` titled `Alpha Database` linked as `[[Alpha Database#Evidence]]` remains a raw Obsidian target, while current lint only accepts `syntheses/alpha-db` and `alpha-db`.

## Solution Assessment

Yes. The narrowest maintainable solution is to fix `extensions/memory-wiki/src/lint.ts` and its colocated tests; rewriting imported source evidence would damage provenance and solve the symptom in the wrong layer.
