## Summary

Keep this PR open. Current main still starts a new session on implicit daily/idle rollover with resetTriggered=false, and the only model/provider preservation path in initSessionState remains inside the explicit reset guard. The PR's original bug report is still valid, but the patch needs rebasing/adaptation to current main's user-vs-auto override policy.

## Best Possible Solution

Keep this PR open and rebase/adapt it onto current main. The likely fix belongs in initSessionState's stale rollover path: preserve explicit user or legacy model/provider selection outside the resetTriggered guard, preferably through resolveResetPreservedSelection or a sibling implicit-rollover helper, while avoiding auto-sourced fallback pins. Add a regression test simulating a post-4am daily rollover after a pre-4am session with user and/or legacy modelOverride/providerOverride.
