## Summary

Keep this PR open. Current main still creates a fresh session on implicit daily/idle rollover with resetTriggered=false, and the only model/provider preservation path in initSessionState is still inside the explicit reset guard. The PR’s original idea remains valid, but the patch now needs adaptation to current main’s user-vs-auto override policy.

## Best Possible Solution

Keep this PR open and rebase/adapt it onto current main. The likely fix belongs in initSessionState’s stale rollover branch: preserve explicit user or legacy model/provider selection outside the resetTriggered guard, preferably by reusing resolveResetPreservedSelection or a sibling implicit-rollover helper, then add a regression test for a post-4am daily rollover after a pre-4am session with user or legacy modelOverride/providerOverride.
