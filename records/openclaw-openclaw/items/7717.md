## Summary

Keep open. Current main still does not implement Discord role-mention triggers: role mentions are collected as generic mention metadata, but no guild/channel config field exists and the mention gate does not turn configured role IDs into `wasMentioned`. The closed related #59151 is adjacent, but local current-main code still lacks the requested behavior.

## Best Possible Solution

Implement the feature in the bundled Discord plugin as an additive guild/channel config surface, likely `triggerOnRoles?: string[]`, with channel-level override/fallback semantics matching nearby Discord config. Preflight should normalize mentioned role IDs from Discord metadata and treat a configured match as an explicit wake mention while leaving the existing sender `roles` allowlist untouched. Add focused schema, metadata/docs, and preflight tests.

Do not change or repurpose the existing `roles` config key; it is sender allowlisting.
