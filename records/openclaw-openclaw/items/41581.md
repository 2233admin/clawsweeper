Keep open. Current main still forces Telegram DM partial answer previews through the message-preview path, while the lower-level draft stream still supports DM-preferring `sendMessageDraft`; the closed fix PRs were not merged, and the latest release tag still contains the forced message transport.

Restore smooth `sendMessageDraft`-style partial streaming in Telegram DMs after the regression to coarser `sendMessage`/`editMessageText` preview updates.

Implement a targeted Telegram plugin fix that lets normal DM answer partial previews use draft-preferring auto transport again, while preserving final materialization, duplicate-final suppression, stale-preview cleanup, native-quote exclusions, and the intended reasoning/tool-progress behavior.
