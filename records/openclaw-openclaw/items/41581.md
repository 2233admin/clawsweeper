## Summary

Keep open. Current main and the latest release still force Telegram DM answer previews through message transport, and current tests/docs assert the reported message-edit path. The lower-level draft stream and final-materialization helpers still support native DM draft preview, but the dispatcher bypasses that path; #47631 remains the focused open fix candidate, while #62207 and #62210 are closed unmerged.

## What This Changes

Restore smooth Telegram DM partial streaming for `channels.telegram.streaming = "partial"` by using draft-style `sendMessageDraft` preview instead of coarse `editMessageText` message updates.

## Best Possible Solution

Review, update, and land #47631 or an equivalent Telegram plugin patch that lets normal DM answer partial previews use draft-preferring auto transport again, keeps reasoning/tool-progress behavior stable, materializes final text without duplicate replies, preserves fallback paths for media/buttons/errors, and updates Telegram streaming tests plus docs to make the intended DM transport explicit.
