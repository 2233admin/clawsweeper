## Summary

Keep open. Current main still sends the Feishu websocket/startup/status probe through the SDK-backed `openclaw_bot/ping` path that matches the reported Axios timeout, and the related Feishu timeout repair PR is still open rather than merged. The webhook workaround is useful operational evidence, but websocket mode remains a documented supported surface.

## What This Changes

The issue reports Feishu websocket-mode gateway startup/API probing timing out through the bundled `@larksuiteoapi/node-sdk` Axios path on Alibaba Cloud ECS with Node.js 24, while native HTTPS works and webhook mode avoids the failure.

## Best Possible Solution

Fix this inside the Feishu plugin with one consistent transport and timeout policy for startup identity probing, setup/status probing, and SDK HTTP calls. Preserve gateway lifecycle ownership, keep webhook mode as an operational workaround, and use #67466 or a replacement PR only after the startup/status fix is lifecycle-safe and validated against a deterministic SDK transport fixture or redacted live Feishu proof.
