## Summary

Keep open. Current main still lacks a first-class global setup hook for credentialed tool-only plugins: the manifest setup contract, descriptor lookup, setup registry, and public plugin API are provider/CLI-backend shaped, while the generic plugin config wizard skips sensitive fields and only naturally surfaces enabled or enabled-by-default plugins. The related #61551 runtime hook-runner bug is closed and does not supersede this plugin SDK/setup API gap.

## What This Changes

Add a manifest-first setup/onboarding hook so credentialed tool-only plugins can participate in global `openclaw setup` and persist plugin-owned credentials/config.

## Best Possible Solution

Define a backward-compatible tool-plugin setup contract that adds descriptor-first manifest metadata, a setup-time registration/helper API for SecretInput-backed plugin credentials, safe persistence under documented plugin-owned config, and matching SDK docs plus contract tests.
