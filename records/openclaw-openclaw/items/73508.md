## Summary

Keep this PR open. The docs-only direction matches the linked Telegram report and current runtime behavior, but the submitted callouts use `!!!` admonition syntax that is not used by the Mintlify docs tree and is likely to render as literal text instead of styled warnings.

## What This Changes

The PR adds Telegram documentation notes and a changelog entry explaining that native Telegram quote replies prevent the answer-lane preview stream used for tool-progress status lines.

## Best Possible Solution

Revise the added Telegram notes to use Mintlify `<Warning>` and `<Info>` components, keep the changelog attribution, and land the focused docs fix; any doctor warning or runtime fallback should remain a separate maintainer decision.

## Reproduction Assessment

Yes. The linked report gives concrete Telegram configuration steps and a live trace, and current main source/tests provide a focused code-level reproduction for quote replies disabling the answer preview stream while `replyToMode: "off"` allows it.

## Solution Assessment

No, not as submitted. A docs-only fix is a reasonable minimum for #73487, but the submitted admonition syntax is not the maintainable docs-site solution; the safer fix is to use the repo’s existing Mintlify callout components.
