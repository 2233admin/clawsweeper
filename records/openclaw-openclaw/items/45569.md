## Summary

Keep open. Current main still has the reported two-Enter path: `/status` is offered in the slash menu but is not `executeLocal`; pressing Enter while the menu is open selects/fills it, and the normal send only happens on the next Enter. There is also an open PR, #45602, using closing syntax for this issue, so the issue should stay open until that PR is reviewed/landed or replaced.

## Best Possible Solution

Keep this issue open and review #45602 or replace it with a fix that matches the current gateway-backed `/status` design. The likely fix should make selecting a no-argument gateway command from the Control UI slash menu submit it on the first Enter, or add a proper client-side `/status` implementation plus regression coverage; it should not blindly flip `executeLocal` without handling `/status`.
