## Summary

Keep open. Current main still has the reported Control UI behavior: `/status` appears in the slash-command menu, but it is not marked for immediate local execution, so Enter selects/fills the command instead of submitting it. The issue is also paired with open PR #45602 using closing syntax, so it should remain open until that implementation is reviewed, rebased or replaced, and landed.

## What This Changes

Make the Control UI `/status` slash command execute after a single Enter from the slash-command menu instead of requiring a second Enter.

## Best Possible Solution

Review, rebase, or replace PR #45602 with a focused Control UI fix that makes `/status` submit from the slash menu on the first Enter while preserving the documented local status behavior and adding regression coverage for the menu-selection path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: type `/status` so the slash menu is open, press Enter, and the current selection path fills the draft because `/status` is not `executeLocal`; a second Enter is then needed to submit.
