## Summary

Keep this issue open. Current main has partial Codex OAuth and quota support, including automatic OAuth refresh, Codex WHAM usage-window parsing, Control UI quota hints, and doctor guidance for permanent refresh-token failures, but it still lacks the requested native diagnostic state contract and bounded brokered reauth action. The prior ClawSweeper review comment still matches current main.

## What This Changes

Request native Codex/ChatGPT OAuth diagnosis that distinguishes quota windows from true credential failure, displays Codex 5-hour and weekly limits, and can invoke a bounded host-side reauth adapter when reauthentication is required.

## Best Possible Solution

Add an additive provider auth/usage diagnostic contract with states such as ok, expiring_soon, quota_wait, and reauth_required; keep existing OAuth refresh and WHAM parsing; expose local-time 5-hour and weekly Codex renewal details in Control UI; and design a narrow, auditable broker API that can only run after reauth_required is diagnosed.
