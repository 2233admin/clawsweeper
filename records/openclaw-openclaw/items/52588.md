## Summary

Keep this PR open. Current main does not implement the requested TUI user-message color overrides: the palette still selects only the built-in light/dark palette, the public environment docs only mention OPENCLAW_THEME/COLORFGBG, and there are no current-main tests or changelog entries for OPENCLAW_TUI_USER_BG or OPENCLAW_TUI_USER_TEXT. The PR is focused and relevant, but the existing review comments identify an actionable readability issue for background-only overrides that should be addressed before landing.

## Best Possible Solution

Keep this PR open and have the contributor or maintainer finish it in the TUI theme layer: add strict color parsing for OPENCLAW_TUI_USER_BG and OPENCLAW_TUI_USER_TEXT, preserve defaults for invalid input, document the vars, add tests, and address the background-only contrast issue raised in review. The change is not obsolete on current main and does not appear to belong in ClawHub or another canonical item.
