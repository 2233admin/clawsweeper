## Summary

Keep this PR open. Current main still lacks the proposed TUI user-message color env vars, and the branch is narrow, but the patch introduces a concrete readability regression for the advertised background-only override path that needs maintainer or author follow-up before merge.

## What This Changes

The PR adds `OPENCLAW_TUI_USER_BG` and `OPENCLAW_TUI_USER_TEXT` TUI palette overrides with strict hex validation, theme tests for valid and invalid values, and environment/TUI documentation updates.

## Best Possible Solution

Land the customization only after the theme layer preserves readable user-message text for single-variable overrides, defines the behavior for low-contrast paired values, keeps malformed input falling back to defaults, and documents the final policy consistently.

- **[P2] Derive readable text for background-only overrides:** `src/tui/theme/theme.ts:140`
  - body: The docs advertise setting `OPENCLAW_TUI_USER_BG` and/or text, but this line replaces only `userBg` when the background env var is set. Derive or require a contrasting text color, or reject background-only overrides that fail readability.
