## Summary

Keep this PR open. Current main still does not implement the intended Discord media fix: Discord upload blobs are still constructed without MIME types, regular/component/native Discord media paths still drop `contentType`, and shared web media still optimizes non-GIF images into JPEG/PNG with no WebP/AVIF/animated preservation path. The PR is paired with the same author's still-open bug #41256 and overlaps open implementation candidates #41419 and #41714, so maintainers should reconcile the canonical fix rather than close this PR as obsolete.

## Best Possible Solution

Keep this PR open while maintainers choose the canonical implementation for #41256 and reconcile overlap with #41419 and #41714. The final fix should preserve verified `media.contentType` in regular Discord sends, component uploads, and native interaction replies; preserve WebP and genuinely animated formats without flattening frames; keep Discord media caps and remote fetch caps aligned; avoid non-Discord compression regressions; and add focused regression tests for MIME type, WebP preservation, animation scope, and cap behavior.
