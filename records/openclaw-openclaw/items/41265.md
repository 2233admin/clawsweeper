## Summary

Keep this PR open. Current main still does not implement the PR's intended Discord media fix: Discord upload blobs remain typeless, Discord send/component/native paths still drop detected content types, and shared web media still optimizes non-GIF images into JPEG/PNG with no WebP/AVIF/animated preservation option. The paired bug #41256 and overlapping implementation candidates #41419 and #41714 are still open, so maintainers should reconcile the canonical fix instead of closing this PR as obsolete.

## Best Possible Solution

Keep this PR open while maintainers choose the canonical implementation for #41256 and reconcile overlap with #41419 and #41714. The final fix should preserve verified `media.contentType` in regular Discord sends, component uploads, and native interaction replies; preserve WebP and genuinely animated formats without flattening frames; keep Discord media caps and remote fetch caps aligned; avoid non-Discord compression regressions; and add focused regression tests for MIME type, WebP preservation, animation scope, and cap behavior.
