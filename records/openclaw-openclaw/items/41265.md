## Summary

Keep this PR open. Current main still has the reported Discord/media behavior: Discord upload blobs are typeless, Discord send paths do not pass loaded media MIME types, the shared web-media loader still only skips optimization for GIF or explicit optimizeImages=false, and the plugin SDK has no preserveWebp/preserveAvif options. The PR is also paired with the still-open same-author bug #41256 and has recent maintainer context saying a narrow repair was pushed so the original contributor path can stay canonical.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate for #41256. Maintainers should review the repaired branch, resolve the remaining media-cap and shared-loader review points, consolidate or close overlapping open PRs #41419 and #41714, then land one fix that preserves Discord upload MIME types and avoids flattening WebP/APNG/animated media across regular sends, component sends, and native interaction replies.
