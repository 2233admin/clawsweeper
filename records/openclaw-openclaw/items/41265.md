## Summary

Keep this PR open. Current main still does not implement the PR’s intended Discord media fix: regular sends wrap media in typeless Blobs, component sends do the same, interaction replies pass raw buffers, and the shared web-media / SDK surfaces still lack preserve-WebP, preserve-AVIF, or animated-image preservation controls. The PR is also paired with the still-open linked bug and appears to be the maintainer-blessed canonical contributor path after the latest branch repair, so cleanup closure would be premature.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate for the still-open linked Discord media bug. The best path is maintainer review of the latest repaired branch, with special attention to Discord regular sends, component sends, native interaction replies, shared media preservation flags, APNG/WebP animation detection, per-account media caps, remote fetch limits, SDK compatibility, and the added regression tests. If this branch lands, the linked bug and overlapping PRs can then be closed or superseded from the merged implementation.
