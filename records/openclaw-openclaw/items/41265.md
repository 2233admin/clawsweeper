## Summary

Keep this PR open. Current main still does not implement the intended Discord media fix: upload blobs are still constructed without MIME type, regular/component/native Discord media paths still drop the detected content type, and shared web media still optimizes non-GIF images to PNG/JPEG with no WebP/AVIF/animated preservation path. The paired bug #41256 and overlapping PR candidates #41419 and #41714 are still open, so maintainers should reconcile the canonical fix rather than close this PR as obsolete.

## Best Possible Solution

Keep this PR open while maintainers choose the canonical implementation for #41256 and reconcile overlap with #41419 and #41714. The final fix should preserve verified media.contentType in regular Discord sends, component uploads, and native interaction replies; add an explicit shared media preservation contract for WebP/AVIF and genuinely animated formats; keep Discord account media caps and remote fetch caps aligned; and add focused regression tests for MIME type, WebP preservation, animated true-path/APNG behavior, and cap handling.
