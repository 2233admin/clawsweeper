## Summary

Keep this PR open. Current main still does not implement the intended Discord media fix: upload blobs are still typeless, regular/component/native interaction media paths do not thread detected MIME types, and the shared media loader plus SDK outbound-media surface still lack WebP/AVIF or animated-image preservation controls. The PR is also paired with the still-open linked bug and was recently repaired by a maintainer as the canonical contributor path, so cleanup closure would be premature.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate for the linked Discord media bug. The best path is maintainer review of the repaired branch, with attention to Discord regular sends, component sends, native interaction replies, shared media preservation flags, APNG/WebP animation detection, per-account media caps, remote fetch limits, SDK compatibility, and the added regression tests. If this branch lands, close the linked bug and supersede the overlapping PRs from the merged implementation.
