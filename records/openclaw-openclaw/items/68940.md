## Summary

Current main still has the reported docs-vs-CLI mismatch. The CLI Backends quick start still tells users to run `openclaw agent --message ... --model ...`, but the top-level `openclaw agent` command does not register `--model`, and its CLI option type/path does not pass a model override into the gateway call. Lower-level model override plumbing exists, and the Codex CLI default backend setup has improved, but the observable user problem remains unresolved.

## Best Possible Solution

Keep the issue open until maintainers either land a real `openclaw agent --model <provider/model>` path or change the CLI Backends quick start to a supported command. The implementation path should register `--model` on `openclaw agent`, include it in `AgentCliOpts`, forward it through `agentCliCommand`/`agentViaGatewayCommand` with authorized model override semantics, add Commander/gateway pass-through tests, and then update `docs/gateway/cli-backends.md`. If maintainers prefer not to expose per-turn agent model overrides, the docs should be corrected to use a supported model-run command instead.
