## Summary

Keep open. Current main still uses raw `${provider}/${modelId}` lookup in `resolveExtraParams`, so provider-prefixed model IDs such as `openrouter/auto` still miss canonical per-model params. The PR is an active replacement for the closed, unmerged source PR #44319 by @HenryXiaoYang and appears narrowly scoped to the intended agent config fix plus regression coverage.

## Best Possible Solution

Keep this PR open for maintainer review and merge if CI/review are clean. The best fix is the narrow one proposed here: resolve extra params with `modelKey` plus `legacyModelKey` fallback, keep regression coverage for canonical `openrouter/auto` and legacy double-prefixed keys, preserve the changelog credit for @HenryXiaoYang and source PR #44319, and avoid unrelated workflow or dependency changes.
