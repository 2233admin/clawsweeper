## Summary

Keep open. Current main still lacks a runtime-default model sentinel for cron and subagent stored overrides, and this PR is a plausible implementation path, but the patch has merge-blocking consistency and type-safety issues plus missing docs/changelog follow-through.

## What This Changes

The PR adds a shared `@default` model sentinel for stored cron/subagent model overrides and persisted model refs, with sentinel-aware normalization and new cron/subagent tests.

## Best Possible Solution

Land a revised version only if maintainers accept the `@default` contract, with sentinel semantics applied consistently to cron model selection, cron fallback policy, subagent session metadata, persisted override resolution, public docs, changelog, and focused regression/type coverage; then use the related runtime-default cron model issue as the canonical follow-up to close after merge.

## Reproduction Assessment

Yes. The PR defects are reproducible with focused static/type checks: the added object-shaped subagent test conflicts with the current string-typed plan API, `modelOverride: "@default"` records user metadata, and `resolveCronFallbacksOverride` differs for omitted `payload.model` versus `payload.model: "@default"`.

## Solution Assessment

No, not as currently patched. The shared sentinel helper is a reasonable narrow approach, but the implementation needs the type contract, metadata predicate, cron fallback policy, docs, and changelog tightened before it is the best maintainable solution.
