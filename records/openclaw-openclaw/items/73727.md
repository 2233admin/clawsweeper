## Summary

Keep open. Current main still has no `@default` sentinel, so the PR is addressing a real gap, but the proposed patch is not merge-ready because two downstream model-source/fallback paths still treat the sentinel as an explicit stored model override.

## What This Changes

The PR adds a stored `@default` model sentinel and tests so cron and subagent model overrides can resolve to the live `agents.defaults.model.primary` instead of pinning a concrete model string.

## Best Possible Solution

Revise the PR so all stored-override readers share the same sentinel semantics, including selection, fallback-policy detection, and persisted session source metadata; add focused tests for cron fallback override behavior and subagent `modelOverrideSource`, then document the operator-facing `@default` behavior before merge.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists on the PR diff: `resolveSubagentModelAndThinkingPlan` with `modelOverride: "@default"` records `modelOverrideSource: "user"`, and `resolveCronFallbacksOverride` with `payload.model: "@default"` still treats the payload as an explicit model override.

## Solution Assessment

No. The sentinel helper is a reasonable direction, but the PR is not yet the narrowest maintainable fix because downstream fallback/source metadata paths still branch on raw non-empty model strings instead of the sentinel-aware normalized value.
