## Summary

Keep this issue open. Current main still does not implement one-call Feishu document creation with initial content: the create branch only calls createDoc, and there is no regression test for create plus content. More importantly, open PR #48377 is an unmerged implementation candidate that explicitly targets this issue, so the issue should stay open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Review PR #48377 as the active implementation candidate, or close it with a maintainer decision to reject content during create. The final fix should either internally chain createDoc and writeDoc with regression coverage, or validate/reject content on create with a clear error so agents cannot get a successful empty document after asking for initial content.
