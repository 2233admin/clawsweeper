## Summary

Keep open. Current main still does not implement WhatsApp missed-message backfill after reconnect: inbound handling is event-driven, stale `append` history is explicitly skipped, full Baileys history sync is disabled, and reconnect handling only drains pending outbound deliveries. The BlueBubbles catchup work is useful related context but is channel-specific and does not supersede this WhatsApp request.

## What This Changes

Request to backfill WhatsApp messages missed during Web connection outages by fetching bounded history for monitored chats after reconnect.

## Best Possible Solution

Implement bounded reconnect recovery inside the WhatsApp plugin: persist last-seen state per account and eligible chat, validate a Baileys history mechanism for monitored chats, cap lookback and replay volume, dedupe by message ID before enqueueing, and emit a clear warning when provider backfill is unavailable or untrusted.
