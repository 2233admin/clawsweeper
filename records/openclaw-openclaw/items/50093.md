## Summary

Keep open. Current main still lacks the requested WhatsApp reconnect backfill: the plugin consumes live Baileys upserts, skips stale history append events outside a short grace window, disables full history sync, and reconnect handling only drains pending outbound deliveries. The related BlueBubbles catch-up item is useful context but does not supersede this WhatsApp-specific gap.

## What This Changes

Request to backfill WhatsApp group messages missed while the Web connection was down, using a reconnect-time history/cursor mechanism or at least an explicit missed-message warning.

## Best Possible Solution

Implement bounded reconnect recovery inside the WhatsApp plugin: persist last-seen state per account and eligible chat, validate a Baileys/WhatsApp Web history mechanism for monitored chats, cap lookback and replay volume, dedupe by message ID before enqueueing, and emit a clear gap warning when provider backfill cannot be trusted or is unavailable.
