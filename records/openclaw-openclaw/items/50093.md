## Summary

Keep open. Current main still does not implement WhatsApp missed-message backfill after reconnect: the WhatsApp plugin remains event-driven, skips stale append history, disables full socket history sync, and only drains pending outbound deliveries after reconnect. The closed BlueBubbles catchup work is useful related context but is channel-specific and does not supersede this WhatsApp gap.

## What This Changes

Request bounded WhatsApp reconnect recovery for monitored chats or groups so inbound messages sent while the WhatsApp Web connection is down are fetched or otherwise surfaced after reconnect.

## Best Possible Solution

Implement bounded reconnect recovery inside the WhatsApp plugin: persist last-seen state per account and eligible chat, validate the Baileys history mechanism for monitored chats, cap lookback and replay volume, dedupe by message ID before enqueueing, and emit a clear warning when provider backfill cannot be trusted or is unavailable.
