## Summary

Current main has related restart hardening, but it does not implement the requested two-phase macOS LaunchAgent stop/wait/start sequencing. The normal restart path still uses `launchctl kickstart -k` without `bootout`/`bootstrap` first, and tests explicitly assert that current behavior. The issue is recent, actionable in this repo, and not protected by author association or labels, so it should stay open for maintainer judgment and a targeted fix if desired.

## Best Possible Solution

Keep the issue open. The best maintainer path is to decide whether macOS explicit LaunchAgent restart should move back to a true two-phase launchd sequence, then implement it in `src/daemon/launchd.ts` and `src/daemon/launchd-restart-handoff.ts` with tests that assert ordering: capture prior launchd PID/state, stop or bootout, wait until the old process/port is gone, then bootstrap/start or kickstart without breaking installed-but-unloaded repair and detached self-restart.
