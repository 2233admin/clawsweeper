## Summary

Current main still does not implement the requested two-phase macOS LaunchAgent restart. The normal restart path enables the service and runs `launchctl kickstart -k` directly, the unit test still asserts no `bootout` or `bootstrap` on the healthy restart path, and the related merged work in #72616 only covers installed-but-unloaded supervision recovery while explicitly referencing this issue as adjacent.

## Best Possible Solution

Keep this issue open for maintainer judgment on the macOS LaunchAgent restart contract. The best implementation path is to decide whether normal `openclaw gateway restart` should move away from the intentionally introduced `kickstart -k` contract, then update `src/daemon/launchd.ts`, `src/daemon/launchd-restart-handoff.ts`, and their tests to prove old-service exit-before-start ordering without regressing installed-but-unloaded recovery or self-restart handoff behavior. Live macOS launchd validation should be part of landing because the behavior depends on launchd service-state edge cases.
