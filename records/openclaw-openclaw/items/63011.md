## Summary

Keep open. Current main still leaves outbound sends split: gateway send still treats hook-cancelled empty delivery as a missing result, message_sending cancellation still does not produce a structured blocked outcome, hook context still lacks the requested outbound identity, and the same-author draft PR #63007 remains the focused implementation candidate.

## What This Changes

The issue requests one canonical outbound message.send path that runs message_sending hooks, returns real delivery results, surfaces guarded denials, and preserves agent/session/request identity for audit correlation.

## Best Possible Solution

Define and document a maintainer-approved canonical outbound-send contract that threads available agent/session/request identity into message_sending, returns real provider delivery results on success, returns explicit guarded-denial outcomes on cancellation, and aligns or intentionally documents direct CLI, gateway send, and plugin-action behavior.
