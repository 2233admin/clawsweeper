## Summary

Keep this PR open. The reported behavior is still missing on current main: proxied OpenAI Responses models with `compat.supportsStore: true` still do not force `store: true`, and the open PR is now dirty because the store decision moved into the centralized payload policy path.

## What This Changes

The PR adds a `compat.supportsStore: true` opt-in for proxied OpenAI Responses store forcing, decouples server compaction defaults from that opt-in, and adds helper-level unit tests.

## Best Possible Solution

Implement the proxy opt-in in the centralized OpenAI Responses payload capability/policy layer, preserve the provider allowlist and `supportsStore: false` stripping, keep proxy store opt-in separate from default server compaction, and document/changelog the user-facing fix.
