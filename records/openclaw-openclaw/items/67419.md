## Summary

Current main still defaults workspace bootstrap context injection to `always`, docs/schema/tests still encode that default, `MEMORY.md` is still injected when present, and the configurable dynamic bootstrap-file controls requested here are still absent. The open closing PRs are unmerged implementation candidates, and the maintainer umbrella explicitly preserves concrete reports with unique evidence.

## Best Possible Solution

Keep this issue open. The maintainer path is to decide whether the default should change from `always` to `continuation-skip`, then update `src/agents/bootstrap-files.ts`, config schema/help/docs, UI config defaults if applicable, and regression tests together. Treat the `MEMORY.md` lazy-loading/stub behavior and dynamic bootstrap-file ordering as scoped follow-up work or link them to #65438 if that remains the canonical tracker. Review or close the open README-only closing PRs separately rather than allowing closing syntax to resolve this runtime bug.
