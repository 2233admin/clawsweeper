## Summary

Keep open. Current main still defaults bootstrap context injection to per-turn `always`, keeps `continuation-skip` opt-in, still injects `MEMORY.md` as normal bootstrap context, and has no configurable dynamic bootstrap-file surface. The latest discussion also adds a concrete restart case where configured `continuation-skip` can still reinject, so this is not obsolete or fully covered by the umbrella tracker.

## What This Changes

The issue requests reducing repeated bootstrap-file context injection by changing the default context-injection mode, adding or documenting MEMORY.md lazy loading, and supporting configurable dynamic bootstrap files.

## Best Possible Solution

Keep this issue open until maintainers choose the default bootstrap reinjection policy and split the remaining work cleanly: default/runtime/config/docs/UI/test alignment in the agent bootstrap path, a separate decision for MEMORY.md stub or lazy-load behavior, and configurable dynamic bootstrap-file ordering under #65438 or a linked follow-up. The restart case should be investigated against the completion-marker/session-file logic so `continuation-skip` behaves consistently across gateway restarts.
