## Summary

Keep open. Current main still only injects OpenRouter `cache_control` markers for Anthropic model refs, and there is no Qwen cache-family predicate, test coverage, or prompt-caching documentation on main. This PR is a focused implementation candidate, not obsolete or superseded; the remaining maintainer question is the upstream OpenRouter/DashScope contract and E2E proof for Qwen `cache_control` behavior.

## Best Possible Solution

Keep this PR open for normal maintainer review. Verify the OpenRouter/DashScope Qwen `cache_control` contract with upstream docs or a live E2E request, then land a narrow tested change in the existing OpenRouter cache-wrapper path and update prompt-caching docs if the behavior is accepted.
