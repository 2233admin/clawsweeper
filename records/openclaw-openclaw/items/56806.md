## Summary

Keep open. Current main still does not expose `knownLongFlags` through the `safeBinProfiles` config schema, fixture type, normalization path, generated schema, or docs. The validator already has internal `knownLongFlags` support, so this PR remains a focused implementation candidate for the config/schema gap reported in #56805. The provided PR discussion also shows the security review concern about denied-flag prefix bypass was addressed in the contributor branch.

## Best Possible Solution

Keep this PR open for maintainer review, rebase it onto current main, and land this PR or an equivalent change that adds `knownLongFlags` to the Zod schema, fixture/normalization/compile path, generated schema, focused regression tests, and user-facing docs while preserving the denied-prefix safety check.
