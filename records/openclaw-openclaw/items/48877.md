## Summary

Keep this PR open. Current main still treats Telegram `customCommands` as BotFather/native menu entries with only `command` and `description`; it does not expose `menus`/`routes` or route menu callbacks without the agent. The PR targets a real missing bundled Telegram feature, but it still deserves normal maintainer review, especially around callback ownership and `callback_data` validation.

## Best Possible Solution

Keep the PR open as an implementation candidate. The best path is to finish it in the Telegram plugin/config surface with typed `menus`/`routes`, authorization parity with existing native commands, route handling scoped to the originating validated custom command, callback payload validation using the existing Telegram 64-byte sanitizer/limit, and focused tests/docs for menu commands, callback fallback, duplicates/conflicts, and malformed buttons.
