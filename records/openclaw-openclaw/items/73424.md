## Summary

Keep open. Current main still contains the exact shared media preprocessing failure path, and the linked PR #73451 is an open implementation candidate with closing syntax for this issue. The issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and use PR #73451 as the implementation candidate, or replace it with an equivalent shared-core fix in `src/media/web-media.ts`. The conservative fix is to fall back to the original valid image when optimization fails and the original is already within `maxBytes`, preserve/log the underlying optimizer cause, and add regression coverage around `loadWebMedia` or the explicit image tool plus a shared channel media path. After that lands, close as implemented.
