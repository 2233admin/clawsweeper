## Summary

Keep this PR open. Current main still lacks the requested abort threshold and typed abort telemetry, and the linked bug remains real, but this branch changes high-impact gateway recovery semantics and still needs maintainer review before merge.

## What This Changes

This PR adds `diagnostics.stuckSessionAbortMs`, `session.aborted` diagnostics/OTel handling, heartbeat-driven stuck-session abort recovery, config schema/reload updates, and focused diagnostics/config tests.

## Best Possible Solution

Ship a documented, typed stuck-session abort threshold that recovers only the intended stale session, preserves normal run timeout and `/stop` semantics, and emits abort telemetry only after the recovery race guard proves the diagnostic state still matches.
