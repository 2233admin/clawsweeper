## Summary

Keep this PR open for maintainer review. Current main still has the unscoped `usage.cost` path, while the PR directly addresses the multi-agent aggregation mismatch discussed in #41986 and supersedes the closed unmerged #42063 with broader cache, explicit-agent, partial-failure, and UI date-label coverage.

## What This Changes

The PR changes gateway `usage.cost` to resolve configured agent scopes, aggregate and cache transcript-backed cost summaries across those agents, validate explicit agent requests, and preserve date-only usage labels in the Control UI.

## Best Possible Solution

Land this PR or an equivalent focused fix after maintainer review, keeping the multi-agent aggregation, explicit-agent validation, cache scoping, partial-failure behavior, and date-only label regression coverage together while leaving broader usage logging and provider-billed-cost work to their separate issues.
