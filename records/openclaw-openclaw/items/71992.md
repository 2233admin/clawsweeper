## Summary

Keep #71992 open. Current main still has the Control UI chat event ordering identified in the issue: `handleChatEvent` appends final/aborted assistant messages to `chatMessages` before clearing `chatStream`, while the renderer can append a live stream item whenever `stream` is non-null. The provided discussion also has a same-day v2026.4.24 reproduction and the focused fix PR #72004 is still described as open, so this is current and actionable rather than stale or already implemented.

## Best Possible Solution

Keep #71992 open until #72004 or an equivalent focused patch lands on main. The likely fix belongs in `ui/src/ui/controllers/chat.ts`: snapshot any needed fallback stream text, clear `chatStream`/`chatRunId`/`chatStreamStartedAt` before mutating `chatMessages` in both `final` and `aborted`, and add a regression test that observes the stream is already null when `chatMessages` changes. Keep broader persisted-transcript duplicate cases coordinated under #69208 rather than using them to close this webchat rendering race.
