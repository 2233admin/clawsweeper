## Summary

Keep open. Current main still has the Control UI/WebChat final and aborted event ordering described in the report, and the renderer still combines committed chat history with a non-null live stream. The supplied GitHub context also shows open PR #72004 with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #72004 or an equivalent patch that snapshots fallback stream text, clears `chatStream`/`chatRunId`/`chatStreamStartedAt` before mutating `chatMessages` in both `final` and `aborted`, and adds an order-sensitive regression test. Track persisted JSONL duplicate cases separately under #69208 rather than closing this focused Control UI rendering race early.
