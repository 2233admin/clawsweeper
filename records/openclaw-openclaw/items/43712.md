## Summary

Keep this issue open: current main still lets source git updates build inside the existing checkout, while the A2UI bundle script excludes ignored renderer output from hashing but does not clean or reject that stale output before running the renderer TypeScript compile.

## What This Changes

The issue reports an in-place source checkout update under `/usr/lib/node_modules/openclaw` failing during `pnpm build` with A2UI renderer `TS5055` and lit `TS2742` errors, while the same tag built successfully in a clean checkout.

## Best Possible Solution

Add a source-build guard around A2UI renderer compilation that removes stale ignored renderer output when safe, or fails with a precise recovery message when cleanup cannot proceed, and document the source-checkout recovery path for in-place updates.
