## Summary

Keep this issue open. Current main still lacks a typed, schema-backed, or documented config-level default task notification policy, and the main ACP, subagent, Task Flow, and ACP control-plane task creation paths still rely on hardcoded `done_only` behavior unless a caller supplies `notifyPolicy`.

## What This Changes

The issue requests a configuration-level default notification policy for background tasks so users can suppress raw ACP and subagent completion notifications without racing per-task updates.

## Best Possible Solution

Add a backwards-compatible core config surface, likely under `agents.defaults.tasks`, that resolves the effective default before ACP, subagent, and Task Flow records are created. Preserve current behavior when unset, keep cron/CLI/media silent defaults intact, let explicit task or flow `notifyPolicy` win, and document the user-facing behavior.
