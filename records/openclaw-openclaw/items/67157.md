## Summary

Keep this PR open. Current main still does not include the proposed launcher-backed npm bin, local stable tarball upgrade script, live memory-pro host smoke, or launcher e2e coverage. The PR is also not conservative cleanup because it changes the published npm executable path and adds global-install/live-execution scripts while concrete release-preflight, entry-bootstrap, gateway-detection, and regression-script reliability issues remain unresolved.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a revised implementation that either keeps `openclaw.mjs` as the canonical launcher and adds local tarball/memory-pro regression coverage around it, or changes the bin target while updating release guards, cross-OS/release smoke assumptions, entry-bootstrap parity, gateway argv detection, and the live/local script reliability findings together.

## Risks / Open Questions

- Merging as-is would make the package bin target disagree with the release metadata guard and release-check tests.
- The proposed direct run-main launcher can bypass startup behavior in `src/entry.ts` unless reworked for bootstrap parity.
- Gateway lock/liveness detection can misclassify processes whose command line exposes the proposed launcher path.
- The added live/local scripts still have review-identified reliability risks around async error handling, output parsing, tarball selection, and verifying the intended installed binary.
- This is an npm install surface change; even without new dependencies or workflows, it needs maintainer validation before merge.
