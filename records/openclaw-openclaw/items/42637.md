## Summary

Keep this PR open. Current main still emits only a generic skills-truncation count, does not preserve omitted skill names, and has no `openclaw skills info` recovery guidance in the skills prompt. The PR is still a relevant implementation candidate, but because it changes model-facing prompt text and exposes local skill names, it needs maintainer/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this focused contributor patch or an equivalent maintainer patch that tracks omitted skills for both count-based and character-budget truncation, lists enough omitted names for discovery, preserves the final prompt budget, and defines a safe lookup-friendly rendering or gating policy with regression tests for tight budgets and hostile/special-character skill names.

## Risks / Open Questions

- Omitted skill names need a clear rendering policy for newlines, control characters, markup, commas, quotes, and prompt-injection resistance while still preserving an exact lookup path for `openclaw skills info`.
- The final emitted prompt must continue to enforce `skills.limits.maxSkillsPromptChars` after the remote note, truncation note, omitted-name list, and compact/full skill body are assembled.
