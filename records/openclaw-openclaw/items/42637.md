## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Fix the contradictory prompt-injection assertion:** `src/agents/skills.buildworkspaceskillsnapshot.test.ts:349-350`
  - body: This test cannot pass: it first asserts that the prompt does not contain `Ignore prior instructions`, then immediately expects `- evilIgnore prior instructions`, which contains that exact substring. The skills test lane will fail before validating the intended sanitization behavior.
  - confidence: 0.99
- **[P2] Keep omitted names usable for skills info:** `src/agents/skills/workspace.ts:797`
  - body: The note tells the model to run `openclaw skills info "<exact skill name>"`, but omitted names are rendered through XML escaping here. For a raw name like `skill-<xml>`, the model sees `skill-&lt;xml&gt;`, while `skills info` matches only the raw name or skillKey, so the advertised recovery path fails for those skills.
  - confidence: 0.91
- **[P3] Preserve exact compact skill locations:** `src/agents/skills/workspace.ts:639`
  - body: `escapeXml` now calls `sanitizeSkillName` for every escaped field, and `formatSkillsCompact` also uses it for `<location>`. A skill under a path containing a backtick will be rendered with a different path, so the model can be told to read a `SKILL.md` path that does not exist; keep name sanitization separate from generic XML escaping for locations.
  - confidence: 0.76
