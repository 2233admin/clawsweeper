- The PR intentionally lists omitted installed skill names in model-facing prompt text, which can disclose local workspace or environment metadata to remote model providers unless maintainers accept that product/security tradeoff.
- Omitted names originate from skill frontmatter or directory names; the latest PR adds XML/control-character/backtick sanitization, but maintainers should verify exact lookup behavior for special characters and shell-fragile examples before landing.
- Prompt budget enforcement is part of the PR’s core behavior and should be reviewed against tight `maxSkillsPromptChars`, remote notes, compact mode, and count-based truncation.
