## Summary

Keep this PR open. Current main still drops prompt-truncated skills without preserving or listing omitted skill names, and the PR remains a focused implementation candidate. It should not be closed as implemented because main only emits the generic `included N of M` warning and has no `openclaw skills info` recovery guidance in the model-facing skills prompt. The security pass also favors maintainer review rather than cleanup closure because the PR changes system-prompt text and renders local skill names from skill metadata.

## Best Possible Solution

Keep this PR open for maintainer/security review. The best path is to land this focused patch or an equivalent maintainer patch that tracks omitted skills for both count-based and character-budget truncation, lists enough omitted names for discovery, preserves the final prompt budget, and defines a safe exact-lookup rendering or gating policy with regression tests for tight budgets and hostile/special-character skill names.

## Risks / Open Questions

- The PR would add local installed skill names to model-facing system prompt text, which may disclose project or environment metadata to remote model providers unless maintainers accept or gate that behavior.
- Skill names come from skill metadata and need a rendering policy that preserves exact `openclaw skills info` lookup keys while neutralizing newlines, control characters, markup, quotes, commas, and prompt-injection-like text.
- The final prompt must still enforce `skills.limits.maxSkillsPromptChars` after adding remote notes, truncation warnings, omitted-name lists, and compact/full skill bodies.
