## Risks / Open Questions

- The PR changes system-prompt text sent to models; listing omitted local skill names may disclose local/project metadata to remote providers unless maintainers explicitly accept or gate that behavior.
- Escaping omitted names for prompt safety can conflict with exact openclaw skills info lookup unless the implementation preserves a lookup-safe representation.
- Skill names are not slug-restricted on current main, so omitted-name rendering needs a clear policy for control characters, newlines, commas, quotes, and prompt-injection resistance.
- The final emitted prompt must still enforce maxSkillsPromptChars after any truncation note, omitted-name list, remote note, and compact/full skill body are assembled.
