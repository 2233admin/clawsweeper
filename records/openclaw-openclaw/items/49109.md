## Summary

Keep this PR open for maintainer judgment, but do not merge it as-is. Current main still lacks both the cron helper quieting and the CLI retry behavior from this PR, while the newer #54475 is a narrower open implementation candidate for the gateway transient-close half. This PR has also drifted into a 66-file, mergeable-false branch with unrelated runtime, plugin, SSRF, generated-doc, and channel-test changes.

## Best Possible Solution

Split the useful work. Let maintainers review or land #54475 for the gateway pre-hello transient close behavior, then apply the cron helper quieting as a small focused PR against current main with tests for `--json`, quiet probe mode, and `expectFinal: false`. Drop or separately justify the unrelated SSRF, plugin registry, build entry, status fast-path, generated-baseline, and channel-test churn before this branch is mergeable.
