## Summary

Keep open. Current main still has the Signal attachment cap bug: Signal RPC responses are capped at 1 MiB before the decoded attachment is checked against `channels.signal.mediaMaxMb`. This PR is the active implementation candidate for the still-open paired issue #73564, and it is not merged or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review, add or accept a focused monitor/getAttachment threading test, then land it as the implementation candidate for #73564 once mergeability and CI are clean. After merge, GitHub can close the paired issue as implemented.
