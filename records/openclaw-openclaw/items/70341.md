## Summary

Keep this PR open. Current main has related Windows exec approval improvements and exact-command durable trust, but it does not implement this PR's specific PowerShell-cmdlet allowlist fallback, and the PR still carries a material execution-path concern: if a bare cmdlet becomes allowlist-satisfied, the Windows node-host rewrite path can drop the PowerShell wrapper and try to spawn the cmdlet token directly.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to close it as superseded if exact-command durable approvals are the intended resolution for the linked Windows Telegram exec report, or revise the PR so bare cmdlet allowlist matches preserve the PowerShell host at execution time and add regression coverage for allowlist evaluation, allow-always persistence, and Windows node-host execution.
