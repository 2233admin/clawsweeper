## Summary

Keep open. Current main has native updater primitives, a generic config Update button, an update-available banner, skipped/error feedback, and a narrow post-restart sentinel check, but it still does not provide the requested first-class Control UI Update Manager with operator-facing status, channel explanations, dry-run, release notes, safe-update controls, persisted update history, and health summaries.

## What This Changes

Request a first-class Control UI Update Manager that surfaces and orchestrates OpenClaw's existing native update engine, auto-update config, safe rollout controls, and post-update feedback.

## Best Possible Solution

Add a first-class Update Manager under Settings or Infrastructure, backed by additive gateway APIs that expose update availability/status, install kind, active channel, last check/attempt/success/failure, release-note links, and dry-run/update-now operations. Wire auto-update toggles through the existing config schema and orchestrate existing update, backup, doctor/health, plugin sync, and restart-sentinel primitives instead of building a second updater.
