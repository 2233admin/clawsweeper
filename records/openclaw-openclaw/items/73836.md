## Summary

Keep this issue open. It is recent, has concrete reproduction details for v2026.4.26, and current main still has source-backed evidence for at least one reported symptom: outbound media mirroring collapses media URLs into filename text before appending a text-only delivery-mirror transcript message, while Control UI image previews require structured attachment/image content. The broader Control UI/gateway responsiveness and Telegram typing symptoms are not proven fixed, and related active-run responsiveness work remains open in #65522.

## Best Possible Solution

Keep this issue open and treat it as an actionable regression bundle. The best maintainer path is to reproduce the active-run stall under Docker/WSL2 or comparable gateway load, profile whether control-plane RPCs such as `tools.effective`, `models.list`, usage, memory/wiki, or session history block reconnect/bootstrap during runs, split independent root causes where needed, fix delivery mirroring so successful outbound image/media sends persist as structured or managed attachments for Control UI previews, and verify heartbeat prompt entries and Telegram typing keepalive behavior with targeted regression coverage. Coordinate with #65522 for the active-run refresh/stall portion rather than closing this report now.
