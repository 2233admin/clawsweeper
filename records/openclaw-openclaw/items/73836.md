## Summary

Keep this issue open. It is recent, has concrete reproduction details, and current main still has source-backed evidence for at least one reported symptom: outbound media mirroring resolves media URLs to filename text before writing the delivery-mirror transcript entry, while the Control UI needs structured attachment/image content to render a preview. The broader Control UI/gateway responsiveness and Telegram typing symptoms are not proven fixed either, and related responsiveness reports remain open rather than resolved canonical fixes.

## Best Possible Solution

Keep this issue open and treat it as an actionable regression bundle. The best maintainer path is to reproduce the active-run stall under a Docker/WSL2 or comparable gateway load, split any independent root causes, fix delivery mirroring so successful outbound image/media sends are persisted as structured/managed attachments for Control UI previews, and verify heartbeat poll prompts are hidden or isolated from visible chat history without breaking heartbeat behavior. Coordinate with the existing open responsiveness reports #72365 and #65522 rather than closing this report now.
