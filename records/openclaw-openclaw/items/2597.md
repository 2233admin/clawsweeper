## Summary

Current main still does not implement the requested passive agent-visible `context=X%` field in the system-prompt Runtime line. The available status, `/context`, `session_status`, post-compaction refresh, and community `openclaw-workspace-sync` plugin are useful mitigations, but they are explicit inspection or recovery paths rather than the per-turn Runtime-line primitive requested here. The latest maintainer-facing comment also kept this issue open.

## Best Possible Solution

Keep this issue open until OpenClaw either implements bounded context-usage plumbing from the latest reliable session token snapshot into embedded-runner params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused tests/docs, or a maintainer explicitly decides that existing status commands, UI warnings, context/memory plugins, and `openclaw-workspace-sync` are the intended product boundary.

Add a bounded `contextPercent?: number` carrier for embedded Pi runs only, derived from the latest reliable session prompt/context snapshot (`SessionEntry.totalTokens` when fresh and `contextTokens` when available).
