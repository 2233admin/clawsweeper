Keep open. Current main and the latest release still silently swallow the three gateway revocation close failures, while this PR is a focused log-then-terminate hardening with regression tests and no concrete correctness finding from the diff review. The PR changes gateway shared-auth and device revocation disconnect paths to warn when socket.close throws, fall back to socket.terminate, thread revocation loggers through reload paths, and add focused unit coverage.

Land a narrow version of this hardening after secops/maintainer review, keeping the behavior limited to gateway revocation close-failure logging plus terminate fallback and focused tests for shared-auth rotation and device removal.
