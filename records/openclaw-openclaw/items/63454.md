## Summary

Current main still does not implement the requested behavior: the bundled session-memory hook formats its filename date, fallback HHMM slug, and `# Session` header from `process.env.TZ` or the host default timezone, not `agents.defaults.userTimezone`. The closed PR #63583 remains a useful unmerged reference, so this issue is valid and narrow enough for a follow-up fix PR.

## What This Changes

Make session-memory memory-file names and `# Session` header timestamps respect `agents.defaults.userTimezone` or an explicit memory timestamp timezone instead of UTC or host-local time.

## Best Possible Solution

Update the bundled session-memory hook to resolve `agents.defaults.userTimezone` through the existing timezone helper, format the date, time, fallback HHMM slug, and header in that resolved IANA timezone, keep host-timezone fallback for unset or invalid config, and document the behavior in the hook docs.
