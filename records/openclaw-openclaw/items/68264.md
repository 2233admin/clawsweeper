## Summary

Keep open. Current main still treats canvas present/navigate and browser navigate as state-changing controls, while visual confirmation remains a separate snapshot/screenshot action; the remaining decision is the intended product contract for chat-visible confirmation and no-node guidance.

## What This Changes

The issue asks for canvas/browser presentation or navigation to reliably show a visible chat-side confirmation instead of returning node-selection or validation-style errors.

## Best Possible Solution

Define the chat-visible contract for canvas and browser state changes, then either add a preview-producing result or optional follow-up snapshot for the selected actions, or tighten tool descriptions, docs, and errors so users are guided to select/pair a node and use the explicit preview action. Add focused regression coverage for the chosen behavior.
