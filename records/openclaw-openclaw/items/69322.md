## Summary

Keep PR #69322 open. Current main still exposes `doctor.memory.status` for only the default agent and does not include the proposed `perAgent` array in source, docs, or tests. The PR remains relevant to the closed related feature request, but it should address the existing review note about isolating secondary-agent manager acquisition failures before merge.

## Best Possible Solution

Keep the PR open and finish it as an additive gateway protocol change: add a documented `perAgent` array to `DoctorMemoryStatusPayload`, populate it for the default agent first and all configured agents after that, preserve the existing embedding/dreaming response shape, and add tests for default-only, multi-agent, unavailable manager, and per-agent acquisition-error isolation.
