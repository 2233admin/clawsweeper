## Summary

Keep this PR open. Current main still returns `doctor.memory.status` for the active default agent only, with no `perAgent` response field or per-agent aggregation. The PR’s intended additive gateway payload is therefore not obsolete; it needs maintainer review/rebase and should address the open review concern around isolating secondary-agent manager acquisition failures.

## Best Possible Solution

Keep this PR open and have maintainers review a rebased revision that adds an additive `perAgent` field to `doctor.memory.status`, keeps the default agent first, isolates each non-default agent’s manager acquisition/status errors into that agent’s entry, updates the protocol docs, and covers the default/no-manager/multi-agent paths in `src/gateway/server-methods/doctor.test.ts`. The already-closed feature request #69295 does not make this PR obsolete because current main still lacks the proposed RPC field.
