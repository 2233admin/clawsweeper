## Summary

Keep open. Current main still uses the blocking `expectFinal: true` path, so the PR is not obsolete, but it also has an unresolved behavioral concern: switching to accepted-only acknowledgement would stop `sendExecApprovalFollowup` from seeing later gateway dispatch failures and could skip the direct fallback that currently surfaces the exec completion.

## What This Changes

The PR switches exec approval follow-up session resume calls from `expectFinal: true` to `expectFinal: false` and adds regression assertions for accepted-only gateway follow-ups.

## Best Possible Solution

Update this PR so exec approval follow-ups do not block on a full resumed agent turn while still preserving an observable fallback for detectable dispatch failures after acceptance, with tests covering both accepted-only success and post-accept failure behavior.
