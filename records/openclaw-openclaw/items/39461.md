## Summary

Keep this open. Current main still does not honor `QQBOT_DATA_DIR` for QQBot persistent storage: the helper always builds `~/.openclaw/qqbot`, diagnostics still recommends an override that is not implemented, and the docs/metadata do not advertise the variable.

## What This Changes

The issue requests that the QQBot plugin read `QQBOT_DATA_DIR` and use it as the plugin data directory instead of always using the home-derived `~/.openclaw/qqbot` path.

## Best Possible Solution

Implement `QQBOT_DATA_DIR` in the QQBot plugin’s central persistent-storage helper with trimmed/non-empty env handling, default-path compatibility, subpath joining, and lazy directory creation preserved. Update diagnostics to report the effective data dir and suppress the override recommendation when it is already active, then add focused tests plus QQ Bot docs, plugin metadata, and changelog coverage crediting the reporter.
