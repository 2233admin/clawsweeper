## Summary

Current main still does not implement the reported QQ Bot `QQBOT_DATA_DIR` override. The central QQ Bot data helper still derives storage from `getHomeDir()/.openclaw/qqbot`, diagnostics still recommends `QQBOT_DATA_DIR`, and runtime state paths for sessions/users/refs/credential backups still flow through that helper. This remains a valid, narrow behavior bug and is a good focused fix-PR candidate.

## Best Possible Solution

Keep this issue open and implement `QQBOT_DATA_DIR` in the QQ Bot plugin's central persistent-storage helper, most likely `extensions/qqbot/src/engine/utils/platform.ts:getQQBotDataPath()`. Preserve subpath joining and lazy creation, update diagnostics so the override is reflected and not recommended when already set, add focused tests for default and override behavior, and document the env var in the QQ Bot channel docs with a user-facing changelog fix entry crediting the reporter.
