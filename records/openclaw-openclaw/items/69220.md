## Summary

Current main still matches the reported failure mode: Google/Gemini resolves to tagged reasoning-output mode while the direct Gemini transport sends native thinking payloads. The related empty-payload PR is adjacent, but it does not remove the tagged-vs-native conflict.

## What This Changes

The issue requests changing Google/Gemini reasoning output so native Gemini thinking is not combined with injected `<think>/<final>` text-tag instructions.

## Best Possible Solution

Update the Google/Gemini provider reasoning contract so paths that send native Gemini thinking payloads resolve to native output mode, while preserving Gemini replay sanitation and thought-signature behavior.

## Reproduction Assessment

Yes. The current-main source path deterministically combines tagged prompt instructions with native Gemini `thinkingConfig`; the issue discussion adds multiple matching real-session reproductions, though I did not run a paid live Gemini loop in this read-only pass.

## Solution Assessment

Yes. Changing the Google/Gemini reasoning-output contract is narrower and more maintainable than treating only the downstream empty-payload symptom, provided the fix keeps existing Gemini replay and transport sanitation intact.
