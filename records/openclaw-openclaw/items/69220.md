## Summary

Current main still matches the reported Gemini failure mode: Google/Gemini can resolve to tagged reasoning-output mode, which injects <think>/<final> prompt requirements, while the direct Gemini transport still sends native thinkingConfig payloads with includeThoughts. The issue is valid, not superseded by the adjacent empty-payload PR, and is narrow enough for a focused repair PR.

## What This Changes

The issue requests removing the Google/Gemini tagged reasoning-output default where native Gemini thinking payloads are used, to avoid unclosed <think>/<final> prompt conflicts and empty post-tool assistant turns.

## Best Possible Solution

Fix the bundled Google/Gemini reasoning-output contract so API paths that send native Gemini thinking payloads resolve to native output mode and do not inject <think>/<final> directives. Keep Gemini replay sanitation and thought-signature handling intact, update focused provider/runtime tests, and review #67781 separately as adjacent empty-payload handling.
