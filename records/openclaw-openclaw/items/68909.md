## Summary

Keep open. Current main still reports systemd runtime through the user manager only and still computes stale gateway PIDs without any systemd cgroup attribution, so the PR is not obsolete. The PR targets a real headless Linux system-service gap, but the existing review comments identify behavior that should be fixed before merge.

## What This Changes

This PR adds a Linux system-bus systemd runtime fallback, cgroup-aware stale PID filtering, status/hint output updates, and tests for headless system-level gateway and node services.

## Best Possible Solution

Keep the PR open and update it so the system-bus fallback explicitly reports only gateway units as the primary runtime, keeps node units as auxiliary context, narrows cgroup-based stale PID suppression to known safe OpenClaw sibling services, and preserves the existing user-bus behavior with targeted tests.
