## Summary

Keep open. Current main does not implement this PR's intended Node.js SEA packaging path: the SEA entrypoint, build/fetch scripts, Makefile, generated config, docs page, dist-sea artifact path, and package scripts are absent. The current container path remains a normal Node runtime image that copies dist, node_modules, openclaw.mjs, docs, plugins, and skills into a node:24-bookworm-slim runtime and starts with node. The PR also has unresolved functional and security-sensitive review concerns, so this should stay in maintainer/product review rather than be closed as cleanup.

## Best Possible Solution

Keep this PR open for maintainer/product review. If OpenClaw wants SEA support in core, revise the implementation around the supported Node SEA flow, verify downloaded Node binaries, keep generated local config out of git, remove dead bundling code, update Docker/container docs, and add focused artifact or container smoke coverage. If the existing Docker image/tarball packaging path is preferred, a maintainer should make that product call explicitly and close the PR after that decision.

## Risks / Open Questions

- The PR's proposed Node binary download path is supply-chain sensitive and needs checksum verification, preferably signature verification, before producing production container executables.
- The proposed sea-config.json contains developer-local absolute paths and should not be committed as machine-specific generated config.
- The proposed SEA build flow appears incompatible with OpenClaw's supported Node 22.14+ runtime contract unless revised to the supported Node SEA configuration/injection flow.
- The unused alternate bundler script and CJS/import.meta patching make the packaging path fragile without focused build or container smoke coverage.
