## Summary

Current main still does not implement this PR's intended TUI commentary fallback, but the proposed one-line shared-helper fallback remains too broad for automatic cleanup because the same helper is used by gateway previews, history, and Control UI paths that intentionally suppress commentary-only assistant text.

## Best Possible Solution

Keep this PR open for maintainer review, but do not merge the current global fallback as-is. The best path is a scoped TUI/live-stream projection change with regression coverage, or a provider/phase-normalization fix that turns actual completed assistant output into `final_answer` while preserving commentary suppression in history, previews, gateway live chat, and Control UI.
