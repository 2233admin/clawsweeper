## Summary

Close as implemented on current main. The gateway now merges explicit `models.providers.*.models` rows into the model catalog, preserves their `input` metadata, resolves image capability through that catalog, and WebChat passes the resulting image-support flag into attachment parsing. The Qwen Coding Plan built-in catalog still suppresses `qwen3.6-plus`, but explicit configured model rows are now the supported opt-in path. This is on main/Unreleased and is not proven shipped in `v2026.4.26`.

## What This Changes

The issue asks the gateway to honor explicitly configured image-capable Qwen/custom provider models when the built-in model catalog omits them, so WebChat image attachments are not dropped.

## Best Possible Solution

Keep the current catalog-based implementation: explicit provider model metadata should remain the source of truth when discovery omits a model, while Qwen Coding Plan built-in suppression continues to prevent advertising unsupported defaults. Users on `v2026.4.26` need a later main build or the next release containing the Unreleased entries.
