## Summary

Keep this issue open. Current main still has the macOS Skills UI call the gateway `skills.install` path, still defaults to Homebrew-first installer selection when `brew` exists, and still reports nonzero installer exits through the generic `Install failed (...)` formatter without CLT/macOS 26 detection, fallback, or remediation. The earlier ClawSweeper comment remains consistent with the current checkout at db40ec404a9178294dba3121758ff59459983a11.

## Best Possible Solution

Keep the issue open and fix the gateway-backed skill dependency installer path: detect Homebrew/Xcode CLT unsupported-macOS errors, return actionable remediation, avoid preferring brew when the host brew toolchain is known unusable, fall back to uv/node/go/download when the selected skill metadata provides a viable alternative, and add focused tests for both the failure classifier and installer-selection behavior.
