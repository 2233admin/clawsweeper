## Summary

Keep this issue open. The reported macOS app flow still uses the gateway `skills.install` dependency-installer path, current main still prefers Homebrew when `brew` exists, and installer failures still surface through a generic `Install failed (...)` formatter with no CLT/macOS 26 classifier, remediation, or fallback after a selected brew installer fails. The prior ClawSweeper comment remains consistent with current main at 5de284c2e30f915b769fac08dbeab45f57749702.

## Best Possible Solution

Fix the gateway-backed skill dependency installer path: classify Homebrew/Xcode CLT unsupported-macOS errors, return actionable remediation, avoid selecting brew when the host brew toolchain is known unusable, fall back to uv/node/go/download when the selected skill metadata provides a viable alternative, and add focused tests for both the failure classifier and installer-selection behavior.
