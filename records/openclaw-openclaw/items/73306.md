## Summary

Keep open. Current main still has the Active Memory timeout-budget mismatch at the center of this report: the hook and outer watchdog include setup grace, but the embedded runner still receives the raw configured timeout, and an open closing PR already targets that exact remaining gap.

## What This Changes

The issue reports Active Memory timing out with zero summary output after v2026.4.25/v2026.4.26, with multiple provider and channel confirmations plus observed context/token bloat when recall fails.

## Best Possible Solution

Land #74480 or an equivalent narrow fix that computes one effective Active Memory recall timeout and passes it consistently through the hook, watchdog, and embedded runner, then validate the timeout/no-summary logs before closing this issue.

## Reproduction Assessment

Yes. The issue has several independent log-based reproductions, and current-main inspection gives a focused synthetic path: enable setup grace, trigger Active Memory, and observe the embedded runner still receives the raw timeout that drives its abort timer.
