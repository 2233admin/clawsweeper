## Summary

Keep open. Current main still has partial effective tool visibility through `tools.effective` and the Control UI, but it does not implement strict declared-vs-effective inheritance or a general per-layer clipping/removal trace.

## Best Possible Solution

Keep this issue open as the umbrella for tool capability determinism. The best path is to design a generic policy-resolution trace that records declared inputs, effective tools, removed tools, and removal layer/reason; expose it through `tools.effective`, `/tools` or debug/status surfaces, Control UI, and docs; then add strict inheritance only after maintainers define fail-closed behavior for runtime/security layers that cannot honor parity.
