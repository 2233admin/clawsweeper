## Summary

Keep this PR open. Current main still lacks the proposed reusable orphan-transcript pruning primitive; existing cleanup paths cover stale/capped store entries, missing transcript entries, disk-budget pressure, or interactive doctor archival, but not this opt-in direct orphan JSONL sweep. The PR has already addressed review feedback around checkpoint preservation, bounded I/O, and symlink containment, so the remaining action is maintainer review of the deletion contract and whether to merge it as a staged utility before CLI wiring.

## What This Changes

This PR adds an opt-in `pruneOrphanedTranscripts` utility in session store maintenance plus unit coverage for warn/enforce modes, preserved paths, old orphan JSONL files, nested traversal, checkpoint skips, symlink containment, and unrelated JSONL files.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether the utility is acceptable as a staged building block, then either merge it with focused Testbox validation or ask for same-branch CLI wiring that builds the full preserved-path union across neighboring stores, topic transcripts, explicit session files, and compaction checkpoint metadata.
