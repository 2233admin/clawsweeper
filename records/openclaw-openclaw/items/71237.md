## Summary

Keep this PR open. Current main still lacks the proposed explicit orphan-transcript prune utility; existing cleanup is adjacent but either removes missing store entries, archives top-level orphans via doctor, or removes unreferenced artifacts only under disk-budget pressure. The PR is useful source work, but it still needs maintainer review and has one low-priority robustness issue in the recursive directory walk.

## What This Changes

The PR adds an opt-in `pruneOrphanedTranscripts` session-maintenance helper plus unit tests for warning, enforcing, preserved paths, recursive orphan discovery, checkpoint preservation, and symlink containment.

## Best Possible Solution

Accept the staged utility only if maintainers want a reusable primitive before CLI wiring; otherwise fold the helper into the `openclaw sessions cleanup` implementation that computes the full preserved-path set and previews/enforces orphan cleanup safely. The recursive traversal should be made resilient before merge so one unreadable descendant cannot disable the whole sweep.

## Reproduction Assessment

Yes for the PR utility and the review finding. The PR tests cover old orphan JSONL files and preserved live paths, and the recursive-walk issue can be reproduced by placing an unreadable descendant under `sessionsDir` so Node rejects the whole recursive `readdir` before otherwise-prunable files are considered.

## Solution Assessment

Mostly yes, with one small fix needed. An opt-in, age-gated, header-gated helper is a maintainable building block, but a resilient walker and later CLI wiring with a complete preserved-path union are safer than relying on raw recursive `readdir` and caller discipline alone.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.84

Full review comments:

- **[P3] Handle unreadable subtrees without aborting the sweep:** `src/config/sessions/store-maintenance.ts:473-478`
  - body: `fs.promises.readdir(..., { recursive: true })` rejects the whole traversal when any descendant cannot be read. Because this catch returns the empty result, a single unreadable or problematic subdirectory under `sessionsDir` prevents pruning even readable top-level orphans. Walk directories per subtree, skipping failed descendants, so one bad child does not disable cleanup.
  - confidence: 0.82
