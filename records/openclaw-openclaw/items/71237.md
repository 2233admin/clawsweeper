## Summary

Keep open. Current main does not contain the proposed helper, and the adjacent cleanup paths only cover missing-index entries, pressure-based disk-budget cleanup, or interactive doctor archival, so this PR is still useful source work rather than obsolete. The latest diff looks broadly sound, with one low-severity resilience issue for maintainers to decide before merge.

## What This Changes

The branch adds an opt-in `pruneOrphanedTranscripts` session-maintenance utility plus unit coverage for warn/enforce behavior, preserved paths, recursive orphan discovery, checkpoint handling, and symlink containment.

## Best Possible Solution

Accept the helper once the resilient-walk edge case is addressed and maintainers agree the utility should land before CLI wiring; otherwise fold the same safety gates into the dedicated sessions cleanup command that builds the full preserved-path union.

## Reproduction Assessment

Yes. The PR utility behavior is covered by focused tests in the branch, and the remaining review finding is reproducible with a sessions directory containing one readable orphan plus an unreadable descendant that makes recursive `readdir` reject.

## Solution Assessment

Mostly yes. An opt-in, age-gated, header-gated, canonical-path-scoped primitive is a maintainable building block, but a per-subtree walker is safer than relying on a single recursive `readdir` call.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.84

Full review comments:

- **[P3] Skip unreadable subtrees without aborting pruning:** `src/config/sessions/store-maintenance.ts:473-478`
  - body: `fs.promises.readdir(..., { recursive: true })` rejects the whole traversal when one descendant cannot be read. Because this catch returns the empty result, one unreadable child directory prevents pruning otherwise readable orphans; walk per subtree and skip failed descendants instead.
  - confidence: 0.82
