## Summary

Keep this PR open. Current main still installs Docker Playwright browsers under `/home/node/.cache/ms-playwright`, current docs still tell users to persist that cache path, and the same-author paired bug #42819 remains open. The PR remains a plausible in-repo implementation candidate, but its current Dockerfile patch still needs correctness fixes before merge.

## Best Possible Solution

Keep this PR open for maintainer review and contributor fixes. The best path is to rebase onto current main, set `/opt/playwright` as the Docker browser registry path, create it with `node` ownership for default builds, recursively chown after preinstall, remove apt-list deletion from the BuildKit cache mount, create the Chromium symlink only after a successful install, update both Docker and browser-control docs, and add focused Dockerfile assertions for path, ownership, symlink, and cache behavior.
