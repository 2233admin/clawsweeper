## Summary

Keep this issue open. Current main still does not implement the requested Web Control UI display priority of `bindings[].comment` -> agent name -> technical identifier for Feishu/open_id-style sessions. There are partial mitigations through Feishu sender-name lookup, stored session labels/display names, and agent identity rendering, but route binding comments are accepted in config and then not propagated into the session rows or UI labels.

## Best Possible Solution

Keep this issue open. The best fix is a single Control UI session display-label contract that resolves the matched route binding `comment` first, then agent identity/name, then the existing row label/displayName or technical session key. That likely belongs in backend session-row construction or a shared display-label resolver so `sessions.list`, the chat session selector, and the sessions table all agree, with regression coverage for a Feishu `ou_xxx` binding comment.
