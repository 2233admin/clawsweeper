## Summary

Current main still lacks the requested manually configured model rate limiter. The code supports adjacent reactive behavior for provider 429s, Retry-After caps, auth-profile cooldowns, key/profile rotation, and fallback models, but there is no provider/model config schema, docs surface, or shared preflight scheduler for tokens-per-interval or requests-per-interval budgets. The OpenAI x-ratelimit header suggestion remains useful context, not implemented enforcement.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is a documented core model request budgeting design with per-provider/per-model and likely per-auth-profile request/token budgets, interval windows, preflight scheduling before shared model calls, post-response and streaming usage reconciliation, optional provider-header ingestion such as OpenAI x-ratelimit-*, and focused tests/docs for scheduler, fallback, retry, streaming, and auth-profile interactions.
