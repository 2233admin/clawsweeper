## Summary

Keep this PR open, but it is not merge-ready. Current main has no Nacos config-source implementation, and the PR still contains blocking behavior gaps: gateway startup initializes the source but then reads through a file-backed metadata snapshot helper, and Nacos in-process writes do not notify the hot-reload path. The earlier listener-protocol review feedback is reflected in the current head, but the remaining issues plus the remote-config trust boundary need maintainer review.

## What This Changes

The PR adds an optional Nacos-backed gateway config source with file/Nacos adapters, env-based source resolution, Nacos GET/long-poll subscription, gateway reload/startup/CLI/doctor wiring, docs, changelog, and tests.

## Best Possible Solution

If maintainers want remote config in core, land a generic source seam with file as the default and Nacos as an explicit, validated adapter that routes every startup/read/write/reload path through the active source, preserves source/runtime snapshots, emits correct reload notifications, documents persistence limits, and has real Nacos validation. If this belongs outside core, define the required pre-start config-source plugin API before moving the Nacos adapter to plugin space.

  - body: This initializes the Nacos source, but `loadGatewayStartupConfigSnapshot()` immediately calls `readConfigFileSnapshotWithPluginMetadata()`, which still constructs `createConfigIO()` and reads the local file. In Nacos mode, `startGatewayServer()` therefore boots from local/default config instead of the Nacos payload, while later reload wiring subscribes to Nacos.
  - confidence: 0.9
- **[P2] Notify reload listeners after Nacos writes:** `src/config/io.ts:2481-2501`
  - body: In the Nacos branch, `writeConfigFile()` updates the runtime snapshot and returns without emitting a config write notification, and subscribe-mode reload does not consume `subscribeToWrites`. Successful `config.apply`/`config.patch` calls can update `config.get` while hooks, cron, channels, or other hot-reloadable services keep running the old config until a remote Nacos edit or restart.
  - confidence: 0.87
