## Summary

Keep this PR open. Current main still lacks the Nacos-backed config source, environment surface, and subscribe-based external reload path proposed here; the branch remains an unmerged implementation candidate, and the review thread contains substantive correctness, CLI semantics, and security/product questions rather than evidence that the work is obsolete.

## What This Changes

The PR adds an optional Nacos-backed gateway config source with file/Nacos source abstraction, Nacos long-poll reload, `OPENCLAW_CONFIG_SOURCE=nacos` and `NACOS_*` env handling, CLI/gateway integration, tests, docs, and a changelog entry.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If remote config belongs in core, land a generic, tested config-source seam with file as the default, explicit Nacos behavior, correct listener/auth/error handling, source/runtime snapshot preservation, consistent CLI and gateway semantics, clear persistent-versus-in-memory write behavior, docs/changelog updates, and targeted or live Nacos validation. If maintainers want this outside core, first define the missing pre-start config-source plugin API and then move the Nacos implementation to a plugin/ClawHub path.
