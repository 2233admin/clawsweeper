## Summary

Keep this PR open. Current main still does not implement a Nacos-backed config source, the proposed Nacos env surface, or subscribe-based external config reload. The PR remains the only live OpenClaw item found for this capability, and the diff introduces a security-sensitive remote config trust boundary that needs maintainer product/security review rather than automated cleanup.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If remote config belongs in core, land a generic, tested config-source seam with file as the default, explicit Nacos support, correct listener/auth/error handling, source/runtime snapshot preservation, CLI and gateway consistency, clear persistent-versus-in-memory write semantics, docs/changelog updates, and targeted or live Nacos validation. If maintainers decide this belongs outside core, define the missing pre-start config-source plugin API first and then move a Nacos implementation to ClawHub.
