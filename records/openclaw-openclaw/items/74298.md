## Summary

Keep open. Current main does not show an OpenClaw rsync-related `.env` mutation, and the state-dir dotenv loader would accept Telegram/Discord token keys when present. However, the requested migration hardening is not already covered: the migration docs tell users to copy the state directory and run doctor/status, but there is no dedicated `.env` token integrity checklist or doctor preflight for expected channel env keys.

## What This Changes

Request to investigate missing Telegram/Discord bot token lines after rsync migration and add doctor/docs migration checks for expected `.env` channel secrets.

## Best Possible Solution

Keep the issue open for maintainer judgment on a narrow migration hardening path: document state-dir `.env` verification in the migration guide and recommend `openclaw status --all` / channel probes after cutover; only add doctor logic where OpenClaw can know a token is expected, such as explicit env SecretRefs or service-managed env keys, without warning on every absent optional channel env var.
