## Summary

Keep open. Current main still has the reported behavior: `generic_repeat` can only warn, warning-level loop detections do not block tool execution, and `process` with `action: "list"` is still outside the known-poll path. PR #60248 is an open implementation candidate rather than merged or shipped behavior.

## What This Changes

The issue asks for `tools.loopDetection` to let `generic_repeat` escalate to `critical` at `criticalThreshold`, with a related proposal to treat `process(action="list")` as known polling.

## Best Possible Solution

Land or replace the scoped loop-detection fix so `generic_repeat` can block at `criticalThreshold` when repeated calls show no progress, update the unit/e2e expectations and public docs/config help, and handle `process(action="list")` plus any broader tool-only-turn circuit breaker as explicit follow-up decisions.
