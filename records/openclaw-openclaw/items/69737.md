## Summary

Keep open. Current main still has the reported user-facing path: unclassified assistant error text can be returned verbatim by the formatter and then emitted as an error reply payload. The related narrow fix candidate #41803 is open but not merged, and the multi-text behavior has already been split to #74674, so this issue remains valid and cleanly scoped.

## What This Changes

The issue asks OpenClaw to stop delivering raw provider errorMessage text to users when an assistant message terminates with stopReason=error.

## Best Possible Solution

Land #41803 or an equivalent narrow fix that makes unclassified assistant errors resolve to a generic user-facing message or suppression, keeps classified friendly errors intact, and preserves raw provider diagnostics only in redacted/admin observability paths with focused regression coverage for the OpenAI 5xx/help URL/request ID shape.
