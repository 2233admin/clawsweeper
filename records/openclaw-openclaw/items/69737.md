## Summary

Keep open. Current main still has the two reported behaviors: terminal assistant errors can still surface unclassified provider prose through the delivery/lifecycle path, and assistant visible-text extraction still joins multiple matching text blocks rather than applying a delivery-only pick-one policy.

## Best Possible Solution

Keep this issue open. Land a small delivery/lifecycle hardening change for terminal assistant errors so unknown provider prose emits a generic user-facing fallback or no chat reply, with a regression for the OpenAI 500/help-link/request-id shape. Split or explicitly scope the multi-text behavior as a delivery-only policy after owner review, because the shared extractors currently join text blocks for multiple surfaces.
