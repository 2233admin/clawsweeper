## Summary

Keep open. Current main still has a user-facing path where unclassified assistant errors are returned as raw provider text, and the relevant open fix candidate has not merged. The split-out multi-text behavior is now tracked separately in #74674, so this issue is cleanly scoped to the raw error fallback.

## What This Changes

The issue asks OpenClaw to suppress or replace raw unclassified provider errorMessage text in user-facing delivery when an assistant message ends with stopReason=error.

## Best Possible Solution

Land #41803 or an equivalent narrow fix that makes unclassified assistant errors resolve to a generic user-facing message or suppression, while preserving the existing classified friendly messages and keeping raw provider diagnostics only in redacted/admin observability paths.
