## Summary

Keep open. Current main has some generic media-root widening for concrete local media sources, but it does not fully resolve this issue: the Feishu direct outbound adapter still catches media-send failures and falls back to a text attachment link, and Feishu's current config/schema/docs do not expose a supported `channels.feishu.mediaLocalRoots` setting. The related Feishu PR #51374 was closed unmerged, and the later related PR #63479 is scoped to gateway-delivery staging rather than Feishu's direct adapter behavior.

## Best Possible Solution

Keep this issue open and implement a focused outbound-media fix. The best path is to decide the intended product contract for CLI `--media`: either stage user-supplied local paths into managed media before Feishu sends, or make Feishu fail fast on `LocalMediaAccessError` with clear remediation and a non-zero CLI result. If `channels.feishu.mediaLocalRoots` is intended to be supported, add it to the Feishu schema, account merge semantics, generated metadata, docs, and tests; otherwise document the supported staging roots and ensure unsupported config is not silently implied. PR #51374 is useful prior art but did not land.
