## Summary

Keep open. Current main still does not define or document `channels.feishu.mediaLocalRoots`, and Feishu media-send errors are still caught and converted into a successful text/link fallback that can be formatted as a successful send. The related fix PR #51374 was closed unmerged, so it is not a shipped implementation to close against.

## Best Possible Solution

Keep this issue open and implement a focused Feishu outbound media contract. Either add supported `channels.feishu.mediaLocalRoots` schema/generated metadata/docs, including account merge semantics and security wording, or document that Feishu CLI sends only use core-managed media roots. In both cases, required media access/upload failures should propagate as a clear non-zero CLI failure or an explicit partial-delivery result, with tests around Feishu send action handling and CLI formatting.

## Risks / Open Questions

- Adding per-channel media roots would widen host file-read scope, so the supported config surface needs explicit schema, docs, merge order, and security semantics.
- Maintainers still need to choose whether Feishu media failure should fail the whole send or return an explicit partial-delivery result instead of a success summary.
- The exact `/tmp` reproduction can vary under current filesystem root-expansion policy, so regression coverage should include restricted policy behavior as well as CLI output formatting.
