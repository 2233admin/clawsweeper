## Risks / Open Questions

- Functional risk in the PR as written: wildcarding inside shared `parseBindingSpecs` would also affect `agents unbind`, so legacy channel-only bindings on multi-account channels may become harder to remove by bare channel shorthand.
- Documentation accuracy risk: wording that says omitted `accountId` follows `channels.<channel>.defaultAccount` needs to be reconciled with current routing, which indexes omitted binding account patterns under `DEFAULT_ACCOUNT_ID`.
- Coverage gap: current main has tests for default-only and wildcard routing, but no current-main test for the PR's proposed multi-account auto-wildcard creation behavior or for preserving channel-only unbind compatibility.
