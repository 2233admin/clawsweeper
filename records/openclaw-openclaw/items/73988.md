## Summary

Keep this PR open for maintainer/security follow-up. Current main still rejects out-of-workspace bootstrap symlink targets, so the user-facing bug remains, but this PR’s fallback broadens bootstrap reads to arbitrary readable local files outside the workspace and the supplied Aisle/Greptile reviews identify that as a concrete local-file-read risk.

## What This Changes

The PR adds a fixed-bootstrap-only symlink fallback in the agent workspace reader, adds workspace bootstrap symlink regression tests, and records the change in the changelog.

## Best Possible Solution

Consolidate this into one maintainer-reviewed implementation for fixed bootstrap symlink support that preserves extra/glob boundaries and only reads external targets under a defensible trust model, such as explicit operator opt-in or an allowlist, with regression tests for shared-vault symlinks and secret-path rejection.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce the original bug with a fixed bootstrap file symlink resolving outside `workspaceDir`, and the PR security regression is reproducible by pointing a top-level `AGENTS.md` symlink at another readable local file.

## Solution Assessment

No. The PR is functionally aimed at the right bug, but a deny-list of OpenClaw state paths is not the narrowest maintainable solution for reading outside the workspace; a containment rule or explicit trusted-external-path opt-in is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Require an explicit boundary before following bootstrap symlinks:** `src/agents/workspace.ts:210-224`
  - body: The fallback now opens the symlink target for any fixed top-level bootstrap filename and only rejects known OpenClaw state paths. A workspace can point `AGENTS.md` at another readable local secret such as an SSH or cloud credential file, and that content is then cached and injected as bootstrap context. Please require workspace containment or an explicit operator allowlist/opt-in before reading external targets.
