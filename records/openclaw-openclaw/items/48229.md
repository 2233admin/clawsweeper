## Summary

Keep open. Current main still clears requested WebSocket operator scopes for ordinary device-less shared-token clients, the HTTP default-scope fix discussed in the thread is a separate path, and open PR #57889 references this issue with closing syntax but appears focused on Tailscale/trusted-proxy rather than the regular Docker token-auth case.

## Best Possible Solution

Keep this issue open. Review PR #57889, but require it to either cover the reported Docker LAN token-auth WebSocket path or be split/re-scoped; the eventual fix should explicitly define the contract for token-authenticated device-less WebSocket operator clients, preserve the intended device-identity hardening, and add a regression around bind: "lan", token auth, requested operator.admin/operator.write, and a write-scoped RPC.
