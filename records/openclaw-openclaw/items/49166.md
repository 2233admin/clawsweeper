## Summary

Keep open. This PR still targets a real gap in the `session_status(model=...)` override path. Current `main` has allowlist-based bare-model provider inference in the shared model-selection resolver used by `sessions.patch` and Control UI model switching, but `session_status` still bypasses that resolver and resolves bare model names against the current session provider before allowlist validation.

## Best Possible Solution

Keep this PR open or replace it with a maintainer patch that makes `session_status(model=...)` use the same allowlist-aware provider inference as `sessions.patch`. The implementation should either call the shared allowed-model resolver or apply the same unique-provider inference before `resolveModelRefFromString()`, add the PR's `openai-codex` to bare `deepseek-chat` regression test in `src/agents/openclaw-tools.session-status.test.ts`, and address the harmless bot nit by using `splitTrailingAuthProfile(raw).model.trim()` without optional chaining.
