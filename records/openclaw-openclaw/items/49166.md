## Summary

Keep this PR open. Current main still resolves `session_status(model=...)` bare model overrides against the current session provider before allowlist validation, while the shared model-selection resolver and `sessions.patch` path already infer a unique configured provider for bare refs. The PR addresses a real gap, but the maintainer version should likely reuse `resolveAllowedModelRef` and keep the regression test rather than duplicate inference logic.

## Best Possible Solution

Keep this PR open, or replace it with a maintainer patch that makes `session_status(model=...)` use the shared allowlist-aware `resolveAllowedModelRef` path. Land the cross-provider bare-model regression test and clean up the optional-chaining nit before merge.
