## Summary

Keep open. The branch contains useful memory-search status UX work, but it is not merge-ready against current main: the active implementation is now the `memory-core` plugin, while the PR edits legacy core paths that are absent from current main. The active plugin still omits the requested human status rows and tool payload field, and existing review discussion already identified duplicate fallback output in the proposed status renderer.

## What This Changes

The branch adds memory status display rows in legacy CLI code and changes the legacy `memory_search` unavailable-action hint for Ollama auto-mode provider errors.

## Best Possible Solution

Port the useful behavior into `extensions/memory-core`: show search mode and provider issue in human `openclaw memory status`, consolidate fallback into one readable row, include `providerUnavailableReason` in successful `memory_search` results when present, and add the specific Ollama auto-mode unavailable hint with focused CLI/tool tests.
