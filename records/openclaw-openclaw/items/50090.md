## Summary

Keep open. Current main has shipped major pieces of the skill ecosystem, including workspace/managed/extra-dir discovery, non-glob watcher roots, native ClawHub search/install/update, scanner support for gateway-backed skill installers, and a generic before_install hook. The umbrella still contains live OpenClaw-side security/product work: direct ClawHub skill install/update bypasses the gateway installer scan/hook path, permission manifests/signing/trust are not implemented, post-install lifecycle/audit hooks are absent, and trigger-quality/linting fields are not part of the skill contract. Narrower related trackers exist, but there is no single maintainer-designated canonical superseder for the whole issue.

## Best Possible Solution

Keep this issue open for maintainer consolidation. Mark the discovery and watcher portions as shipped, route standalone ClawHub CLI/service publish bugs to openclaw/clawhub, and keep or link concrete OpenClaw trackers for direct ClawHub install scanning, permission manifests/provenance, lifecycle hooks including post-install audit events, and trigger-quality/linting if maintainers want those in the core skill contract.
