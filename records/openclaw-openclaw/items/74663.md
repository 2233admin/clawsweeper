## Summary

Keep open. The PR addresses a real Telegram voice-note propagation gap, but it only updates the exported legacy outbound adapter; current main's registered Telegram plugin outbound path still builds send options without `asVoice`, so normal plugin delivery can still drop `audioAsVoice`.

## What This Changes

The PR forwards `audioAsVoice` to `sendMessageTelegram` as `asVoice` in `extensions/telegram/src/outbound-adapter.ts` and adds three legacy adapter unit tests.

## Best Possible Solution

Preserve Telegram voice-note intent through one shared send-options path so both the exported legacy adapter and the registered `telegramPlugin.outbound.base` path derive `asVoice` from `ctx.audioAsVoice` or `payload.audioAsVoice`, with focused regression coverage on the registered plugin path.

## Review Findings

This can pass the new tests while normal plugin delivery still sends TTS audio as an audio/document attachment.
