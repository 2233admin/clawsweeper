## Summary

Keep open. The PR addresses a real Telegram `audioAsVoice` propagation gap, but the patch only updates the legacy exported adapter; current `main` still drops the same option on the registered Telegram plugin outbound path that normal delivery uses, so the proposed fix is incomplete before merge.

## What This Changes

The PR forwards `audioAsVoice` as `asVoice` in `extensions/telegram/src/outbound-adapter.ts` and adds legacy outbound adapter tests for `sendMedia` and `sendPayload`.

## Best Possible Solution

Propagate Telegram voice-note intent through the shared registered outbound implementation: derive `asVoice` from `ctx.audioAsVoice` or `payload.audioAsVoice` in the registered `telegramPlugin` send paths, keep the legacy adapter aligned, and cover the registered path with a focused regression test.
