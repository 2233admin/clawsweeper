## Summary

Keep this PR open. Current main still does not implement `channels.msteams.webhook.host`: the MS Teams runtime listens with `expressApp.listen(port)`, the public config type and strict Zod schema expose only `port` and `path`, and the docs omit the host setting. The PR is still a relevant implementation candidate for #62766, but it should address the existing review comments before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Rebase it onto current main, preserve backward-compatible default behavior when `host` is omitted, add `host` consistently to the MS Teams schema/types/runtime/docs/tests, and address the review comments by keeping the quick-start externally reachable and logging the actual or explicitly unspecified bind address.
