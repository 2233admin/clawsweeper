## Summary

Keep this PR open. Current main still does not expose `fileId` in the shared `message` tool schema or a Slack-owned schema contribution, while Slack still advertises `download-file` and requires `fileId` at dispatch/runtime. The PR is still a plausible narrow fix for the linked open bug; it is not implemented on main or superseded by the related Slack media-type PR.

## Best Possible Solution

Review and land a scoped schema fix for Slack `download-file`, preferably with a regression test that builds the Slack-scoped message tool and asserts `fileId` is present while `download-file` remains advertised. The current one-field PR is aligned with the root cause; a plugin-owned schema contribution would also be acceptable if maintainers want provider-scoped parameters.
