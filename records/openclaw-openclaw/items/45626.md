## Summary

Keep this PR open. Current main still advertises Slack `download-file` but does not expose `fileId` in the shared message tool schema or a Slack-owned schema contribution, so the PR still addresses a real schema gap for linked bug #45574.

## Best Possible Solution

Keep this PR open as a plausible narrow fix for #45574. Review and land a scoped schema fix for Slack `download-file`, preferably with a regression test; if maintainers prefer provider-owned params, move the `fileId` schema addition into Slack discovery via the existing plugin schema contribution seam.
