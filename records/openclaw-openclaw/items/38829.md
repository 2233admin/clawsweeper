## Summary

Keep open. Current checked-out main still has the Gateway transcript subscription API and tests, but the bundled TUI does not call `sessions.messages.subscribe`/`unsubscribe` and does not route `session.message` events into the active chat view. The linked maintainer PR #73402 is the right implementation path and remains the reason not to close this issue before merge.

## Best Possible Solution

Keep this issue open while #73402 is reviewed. The best fix is the focused bundled TUI change described there: add active-session subscribe/unsubscribe support in the TUI gateway client/backend, subscribe after connect and session switches, route `session.message` for the active session without disrupting local streams, ignore other sessions, clean up stale subscriptions, and add targeted TUI regression tests. If #73402 merges, close this as implemented by that merge; if it closes unmerged, keep this issue as the canonical bug.
