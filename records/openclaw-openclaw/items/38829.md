## Summary

Keep open. Current main still has the Gateway transcript subscription API, but the bundled TUI does not subscribe to active-session `session.message` updates or route those events into the chat view. The open closing PR #73402 is the active implementation path, so this should remain open until that PR is reviewed and merged or closed unmerged.

## What This Changes

Make `openclaw tui` display new messages appended from the web UI or other channels in real time instead of only after exiting and re-entering the terminal.

## Best Possible Solution

Keep this issue open while #73402 is reviewed. The best fix is the focused TUI change described there: add active-session subscribe/unsubscribe support in the Gateway TUI backend/client, subscribe after connect and session switches, route `session.message` for the active session without disrupting local streams, ignore non-active sessions, clean up stale subscriptions, and add targeted TUI regression tests. If #73402 merges, close this as implemented by that merge; if it closes unmerged, keep this issue as the canonical bug.
