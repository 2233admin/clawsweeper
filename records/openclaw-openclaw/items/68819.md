## Summary

Keep this PR open. Current main still has the reported Windows Claude CLI spawn gap: the bundled backend invokes `claude`, but the child supervisor only recognizes npm-family `.cmd` shims and main has no resolver that rewrites npm-style `.cmd` wrappers to their underlying executable or `node.exe <cli.js>`. The PR addresses a real remaining command-spawn path, but it touches Windows command execution and has review history around PATH/PATHEXT behavior, so it needs maintainer/security review rather than cleanup closure.

## What This Changes

The PR adds Windows `.cmd` shim target resolution for child-supervisor spawns, wires that resolution into `createChildAdapter`, includes `claude` in shim handling, and expands `windows-command` regression coverage.

## Best Possible Solution

Land this PR or a narrow maintainer replacement that resolves Windows npm-style `.cmd` shims for supervisor child spawns without shelling through `cmd.exe`, honors the child environment and Windows command precedence, preserves non-Windows behavior and custom shim fallthrough, and adds focused coverage for `claude.cmd`, `.exe` precedence, missing PATH, and `.js` targets.
