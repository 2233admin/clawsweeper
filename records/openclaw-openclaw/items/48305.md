## Summary

Keep open. Current main has general restart-reason plumbing, but the PR’s central config-reload propagation is not implemented: the reload handler still computes restart reasons and then calls the restart emitter/deferrer without passing them through. The PR is not already shipped on main, and it needs a refresh with tests and review-comment cleanup rather than conservative closure.

## Best Possible Solution

Keep this PR open as a still-relevant contributor fix, but ask for a focused refresh on current main: pass `config-reload: ${reasons}` through both deferred and immediate config-reload restart paths, add a regression test that observes the propagated reason, and resolve the stack-trace/no-reason test-noise concern before merge.
