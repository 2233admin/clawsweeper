## Summary

Keep this PR open. Current main has generic restart-reason plumbing, but the config reload caller still computes restart reasons and then drops them before calling the restart deferrer/emitter. The PR remains useful but needs a current-main refresh, a regression test for propagated config-reload reasons, and review of the proposed missing-reason stack trace diagnostic.

## Best Possible Solution

Refresh the PR onto current main, keep it focused on restart-reason propagation, pass `config-reload: ${reasons}` through both `deferGatewayRestartUntilIdle({ reason })` and `emitGatewayRestart(reason)`, add a regression test that observes the propagated reason, and either remove or tightly constrain the missing-reason stack trace diagnostic. If maintainers want to fully eliminate the original `reason=none` log, also make the gateway restart tool require or synthesize a meaningful reason.
