## Summary

Keep this issue open. Current main still advertises Telegram inline buttons to models as legacy `buttons=[[{text,callback_data,style?}]]`, while the actual message-tool contract exposes `presentation.blocks[{type:"buttons"}]`. Telegram can render structured presentation buttons, but final assistant text containing `[[ {"text":...,"callback_data":...} ]]` is still parsed only as normal reply text/directives, not converted to inline keyboards.

## Best Possible Solution

Keep the issue open and fix it as a contract-alignment bug. Update the agent prompt and Telegram docs to steer models to `message(action="send")` with `presentation.blocks[{type:"buttons"}]`, keep the existing Telegram presentation renderer, and add regression coverage for the prompt/tool schema plus final delivery behavior. Only add a legacy `[[...callback_data...]]` parser if maintainers explicitly want compatibility, and validate it tightly.
