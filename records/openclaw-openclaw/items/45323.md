## Summary

Keep open. Current main has slash-command autocomplete and an existing agents.list RPC that could supply agent metadata, but it does not implement @mention autocomplete, mention chips in the composer, validation, or mention-chip rendering in chat history.

## Best Possible Solution

Keep this open for product/UI follow-up. The best path is likely to reuse the existing agents.list data for a Control UI @mention dropdown, then decide whether the composer should remain plain text with autocomplete insertion or move to a tested rich-text/contenteditable chip model with matching history rendering and accessibility coverage.
