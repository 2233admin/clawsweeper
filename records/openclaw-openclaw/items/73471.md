## Summary

Keep open. Current main still defines orphan primary transcripts as files absent from the active `sessions.json` map and still presents the repair as safe archival; the deleted-transcript discovery and interactive-confirmation mitigations reduce impact but do not fix the classifier or the broader doctor safety concerns.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` from archiving valid historical session transcripts as orphans and to harden doctor repair recommendations around user-content state changes.

## Best Possible Solution

Define a real orphan transcript in the doctor state-integrity path without using absence from the active session-key map as sufficient proof. Preserve valid historical primary transcripts with focused regression coverage, keep sidecar exclusions, and split restore UX, prompt wording, corrupt-store warnings, and doctor recommendation CI hardening into explicit follow-ups if maintainers want them tracked separately.

## Reproduction Assessment

Yes. A source-level reproduction is clear: create a sessions directory with one active `sessions.json` entry plus an older valid `<sessionId>.jsonl`; current `noteStateIntegrity` will classify the older file as orphan because it is primary JSONL and absent from `referencedTranscriptPaths`.

## Solution Assessment

Unclear for the full issue because the requested outcome spans classifier correctness, repair UX, recovery commands, store-load diagnostics, and CI policy. The best first fix is the narrow classifier and regression-test change that prevents valid historical transcripts from being offered for archival.
