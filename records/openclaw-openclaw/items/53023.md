## Summary

Keep open. Current main still lacks the requested config surfaces: session lanes are still created with a default concurrency of one, gateway startup/reload only applies configured concurrency to cron/main/subagent lanes, the default lane wait warning remains hardcoded at 2 seconds unless an enqueue caller passes an override, and yield behavior remains explicit through the `sessions_yield` tool. The related #63864 PR is open and covers only the session-concurrency slice; #14747 separately tracks the warning-threshold slice, and auto-yield still needs design.

## What This Changes

The issue requests configurable per-session lane concurrency, automatic yielding after a configured tool-execution interval, and a configurable lane wait warning threshold.

## Best Possible Solution

Keep this as the umbrella tracker. Review or rework #63864 for the per-session concurrency slice, decide whether #14747 is the canonical lane-wait-threshold tracker, and design auto-yield in the embedded runner/tool loop with clear transcript, cancellation, and user-message ordering semantics before implementation.
