## Summary

Keep this PR open. Current main has overlapping CLI startup work, especially built-in-primary plugin-registration skipping, but it still does not implement the PR's two concrete changes: reusing the already validated config snapshot for plugin CLI registration and adding a metadata-entry path usable by lazy plugin CLI registration. The PR discussion also shows the initial behavior concern was addressed by narrowing the metadata-only path to known core command roots, so this should get maintainer review/rebase rather than cleanup closure.

## What This Changes

The PR reuses the validated config snapshot during plugin CLI registration, adds metadata-entry helpers for plugin CLI command groups, routes lazy known-core command startup through that metadata-only path, and adds focused regression tests.

## Best Possible Solution

Keep the PR open for maintainer review and rebase. The best end state is a current-main patch that validates `snapshot.config` reuse independently, preserves full-loader fallback for plugin commands without activation metadata, and only keeps the metadata-entry optimization where current CLI routing still benefits from it.
