## Summary

Keep this PR open. Current main still has the main hardening gaps the PR targets, while the PR head is stale against the active Docker setup path and has security/portability concerns in its added secure Compose file.

## Best Possible Solution

Keep the PR open for maintainer follow-up or replace it with a focused rebased hardening PR: move token masking into scripts/docker/setup.sh with a copyable safe health-check flow, make a deliberate Docker exposure decision and update docs, pin or policy-audit current CI action refs, and drop or repair docker-compose.secure.yml before merge.

## Risks / Open Questions

- Merging the PR as-is would mix useful hardening with a non-portable docker-compose.secure.yml containing host-specific defaults and a hardcoded VPN IP.
- The GitHub Actions changes touch CI code execution paths; any SHA pinning should be updated against current action versions and reviewed as a workflow security change.
- Changing Docker published ports from LAN-friendly defaults to loopback-only may break documented Docker access flows unless the intended exposure model is updated in code and docs together.
