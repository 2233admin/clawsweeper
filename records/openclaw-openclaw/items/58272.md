## Summary

Keep open. The issue is recent, has a concrete macOS/oMLX reproduction with mixed CJK+ASCII path segments, and current main does not show a source-backed fix or regression coverage for the reported path mutation. The title-similar related items are macOS PATH/cwd issues, not duplicates of read-tool path argument corruption.

## Best Possible Solution

Keep the issue open. A maintainer fix should add focused read-tool coverage for mixed CJK+ASCII directory names and filenames, then address the actual failure layer: preserve path tool arguments across provider transports, add diagnostics that show the raw tool argument versus resolved path, and only consider safe repair heuristics when the filesystem can unambiguously match the intended existing path.
