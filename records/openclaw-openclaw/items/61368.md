## Summary

Keep open. Current main does not implement the requested full workspace file-management API or Control UI file manager, and an open implementation PR explicitly closes this issue, so this should stay open until that PR or a reviewed replacement lands.

## What This Changes

Request to add an `agents.workspace.*` gateway API and Control UI workspace file manager for browsing, uploading, downloading, editing, moving, deleting, and organizing agent workspace files in remote/container deployments.

## Best Possible Solution

Keep this as the canonical feature tracker while maintainers review #62417 or a replacement implementation that adds an additive typed gateway protocol, reuses the hardened `fs-safe` helpers, includes focused server/UI tests for traversal and resource limits, and updates user-facing remote workspace docs.

## Reproduction Assessment

Yes for the capability gap. On current main, source inspection shows no `agents.workspace.*` methods and the only public agent file API rejects anything outside the bootstrap/MEMORY allowlist.

## Solution Assessment

Unclear. A full gateway RPC plus Control UI file manager is a plausible direction, but the exact security model and the current open PR’s correctness issues need maintainer review before calling it the best solution.
