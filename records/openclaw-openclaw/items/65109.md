## Summary

Keep open. Current `main` still displays cache read/write tokens in `/status`, but the displayed cost is calculated from only input/output tokens; the cache-aware pricing support exists lower in the estimator and docs, so the remaining gap is the `/status` caller. The open #65381 PR is the active implementation candidate, while #52329 was closed unmerged.

## What This Changes

The issue reports that `/status` displays cache read tokens but omits their `cacheRead` pricing from the total cost estimate.

## Best Possible Solution

Keep this issue open until maintainers land #65381 or an equivalent narrow patch that includes `cacheRead`/`cacheWrite` in the `/status` `estimateUsageCost` usage object and adds a focused regression in `src/auto-reply/status.test.ts`.
