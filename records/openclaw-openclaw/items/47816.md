## Summary

Keep this PR open. Current main still does not include the proposed VirtualBox/Vagrant install guide, `openclaw-vm` wrapper, Vagrantfile, or provisioning scripts, and the PR head still contains unresolved functional and security-sensitive setup-script concerns. It is therefore not implemented on main and not a conservative cleanup close candidate.

## Best Possible Solution

Keep the PR open for maintainer review or contributor rework. If OpenClaw wants official VirtualBox support, it should land as source-owned scripts plus a docs navigation entry, with installer behavior fixed, shell/env parsing hardened, gateway bind/auth/origin instructions aligned with current source behavior, and focused manual VM proof or tests. If maintainers do not want to own VirtualBox/Vagrant as official setup UX, redirect it to a clearly unofficial community path rather than merging the scripts and docs into core.

## Risks / Open Questions

- The PR introduces executable host and guest setup paths, including package-manager changes and `curl | bash`, so supply-chain and host-execution behavior need maintainer review before merge.
- The current PR head can mislead users because the advertised one-line installer and headless SSH command do not match the script behavior.
- The proposed docs still risk overpromising host WebChat/remote access unless gateway bind mode, auth, Control UI origins, and Vagrant networking are verified together.
- Merging this would make VirtualBox/Vagrant an official OpenClaw-supported install path without current main having maintained navigation, tests, or live VM proof for that surface.
