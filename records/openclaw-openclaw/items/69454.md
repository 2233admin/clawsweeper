## Summary

Keep open. Current main still has the Control UI churn and stale index caching behavior this PR is intended to change, and the PR has a concrete unresolved functional review finding around deferred `sessions.changed` reloads after tab switches. The related #45930 polling PR is overlapping but narrower, and does not supersede this PR's session-list bounds, session-change debounce, node/device preload scoping, or `index.html` cache hardening.

## What This Changes

This PR bounds Control UI chat/session list refreshes, scopes node/device polling and preloads to the Nodes flow, debounces `sessions.changed` reloads away from Chat, serves Control UI `index.html` with `Cache-Control: no-store`, and adds targeted Gateway/UI tests.

## Best Possible Solution

Keep this PR open, update it against current main or create a narrow replacement branch, address the deferred `sessions.changed` tab-switch review finding, and rerun the targeted Control UI/Gateway tests. Keep #45930 coordinated as a separate interval-policy change rather than treating it as a superseding fix for this broader churn/cache-control work.
