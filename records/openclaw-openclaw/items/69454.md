## Summary

Keep open. Current main still has the Control UI churn and cache behavior this PR is trying to change: chat/session refreshes remain unbounded, sessions.changed still reloads immediately, node polling/preloads still run outside the Nodes flow, and Control UI index responses still use no-cache rather than no-store. The PR also has an unresolved review finding about deferred session reloads after tab switches.

## Best Possible Solution

Keep this PR open, update it against current main, address the deferred session reload tab-switch review comment, and rerun the targeted Control UI/Gateway tests listed in the PR body. If maintainers want smaller changes, split the index.html no-store hardening from the Control UI churn reductions and coordinate the polling policy with #45930.
