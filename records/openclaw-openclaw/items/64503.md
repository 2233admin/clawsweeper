## Summary

Keep open. Current main still drops empty-text Teams messageBack payloads, and this PR is the active narrow implementation for that gap. The branch should not be cleanup-closed, but it needs a small blocking correction because the new catch block calls the optional `warn` logger unguarded.

## What This Changes

The branch updates the Microsoft Teams message handler to serialize empty-text `activity.value` payloads as `[CARD_ACTION] ...` and adds adaptive-card messageBack tests.

## Best Possible Solution

Keep this PR as the active inbound Teams messageBack fix, guard the optional logger call, align the serialized card-action contract with Teams interactive-payload work, then land after targeted Teams validation.
