## Summary

Keep this PR open. Current main still drops empty-text Microsoft Teams messageBack card actions before considering `activity.value`, while this PR is the narrow active implementation candidate for that bundled plugin behavior. The branch is not ready to merge as-is because the new catch path calls an optional logger method directly, and maintainers still need to confirm the agent-facing `[CARD_ACTION]`/command-body contract.

## What This Changes

The PR changes the Microsoft Teams message handler to serialize empty-text `activity.value` payloads as `[CARD_ACTION] ...` before the empty-message drop, and adds adaptive-card messageBack tests for object, string, and primitive payloads.

## Best Possible Solution

Keep this PR as the active inbound Teams messageBack fix, guard the optional logger call, make an explicit maintainer decision on the serialized card-action contract, then land the branch or request focused changes after targeted Teams validation.
