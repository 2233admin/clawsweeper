## Summary

Keep open. Current main still has the reported DeepSeek wrapper-ordering gap, and two open unmerged PRs already use closing syntax for this issue; the issue should remain open until a maintainer lands or closes the appropriate fix.

## What This Changes

The issue asks OpenClaw to avoid sending stale DeepSeek V4 `reasoning_content` in replay history when thinking is disabled through `params.thinking` or `extra_body`.

## Best Possible Solution

Land one reviewed fix path that derives the effective DeepSeek disabled-thinking state after config and `extra_body` are combined, strips replayed `reasoning_content` whenever the final DeepSeek payload disables thinking, preserves enabled-thinking backfill behavior, and adds regression coverage for the reported `params.thinking=false` plus `extra_body.thinking.type=disabled` configuration.
