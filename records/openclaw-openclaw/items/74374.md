## Summary

Keep open. Current main still leaves the reported DeepSeek V4 disabled-thinking path plausible: the provider wrapper strips `reasoning_content` only when the resolved OpenClaw thinking level is already disabled, while `extra_body.thinking.type=disabled` is applied later and can overwrite the final request after the DeepSeek wrapper has backfilled replay history. Two open, unmerged PRs already reference this issue with closing syntax, so the issue should remain open until maintainers review and land or reject those fixes.

## What This Changes

The issue asks OpenClaw to stop sending DeepSeek V4 `reasoning_content` in replay history when thinking is disabled through `extra_body` or equivalent model configuration.

## Best Possible Solution

Land one narrow fix that derives a single effective disabled-thinking decision before the final DeepSeek V4 payload is sent, covering OpenClaw `off` or `none`, accepted disabled aliases, and `extra_body.thinking.type=disabled`, then strips replayed `reasoning_content` only for disabled-thinking requests while preserving enabled-thinking backfill behavior.
