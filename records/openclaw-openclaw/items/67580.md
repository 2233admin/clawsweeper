## Summary

Keep open. Current main still writes rehydrated promotion snippets verbatim into MEMORY.md, the contamination detector still misses the reported inline-metadata-before-Candidate shape, and rehydration still has no managed-dreaming fence overlap guard. The linked open PR #68774 appears to target the narrow leak path, while the broader no-distillation concern is still active in #67363.

## What This Changes

The issue reports that memory-core dreaming promotes raw short-term Candidate metadata into MEMORY.md instead of distilled long-term memory.

## Best Possible Solution

Keep the issue open until the narrow leak is fixed by landing PR #68774 or an equivalent patch, then leave any broader raw-verbatim/no-distillation policy work to #67363. The desired end state is that managed dreaming artifacts are excluded at recording and rehydration, and MEMORY.md receives only durable, human-readable promoted entries with regression tests for inline metadata, fenced ranges, and zero-real-recall edge cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: feed the current detector an inline `status/confidence/evidence/recalls` prefix before `Candidate:` and it does not satisfy the lead check; rehydration can also return a range from a managed dreaming fence because no fence guard exists there.

## Solution Assessment

No for current main. The linked PR direction looks like the narrowest maintainable fix for the reported artifact leak, but it is not merged and it does not settle the larger distillation/threshold behavior.
