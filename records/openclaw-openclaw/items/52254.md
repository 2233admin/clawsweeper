## Summary

Keep this issue open. Current main still has zh-CN locale loading and translated tab labels, but the reported lazy Control UI views continue to render many inner UI strings as hardcoded English literals, so the user-visible problem remains unresolved.

## Best Possible Solution

Keep this issue open. The best fix is to move remaining user-facing strings in the Agents, Skills, Channels, Sessions, Nodes, Debug lazy views and child panels into ui/src/i18n/locales/en.ts, replace direct render text with t(...) keys, regenerate non-English locale bundles through the Control UI i18n pipeline, and add representative zh-CN render or i18n coverage for the affected tabs.
