## Summary

Current main still has the reported Control UI zh-CN gap. Locale loading and translated tab labels are present, but the affected lazy views and child panels still render many inner UI strings as hardcoded English literals, so this should stay open.

## Best Possible Solution

Keep this issue open. The best fix is to move remaining user-facing strings in the Agents, Skills, Channels, Sessions, Nodes, Debug lazy views and their child panels into ui/src/i18n/locales/en.ts, replace direct render text with t(...) keys, regenerate non-English locale bundles through pnpm ui:i18n:sync, and add representative zh-CN render or i18n coverage for the affected tabs.
