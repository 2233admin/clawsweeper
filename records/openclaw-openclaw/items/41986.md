## Summary

Keep open. Current main still has the Usage dashboard split described in the report: the header totals come from `sessions.usage`, while the daily cost chart comes from `usage.cost`, and the backend `usage.cost` path still loads a single unresolved/default transcript scope instead of the configured agent set that `sessions.usage` discovers. The open PR #70205 remains the implementation candidate; #42063 was closed unmerged and #68066 tracks a different provider billing-accuracy problem.

## Best Possible Solution

Keep this issue open and review/land #70205 or an equivalent core fix. The fix should make unscoped `usage.cost` aggregate over the same configured gateway agent set as `sessions.usage`, preserve explicit agent scoping, include the resolved scope in cache keys, keep cost units documented as USD, and add regression coverage proving header totals and daily bars agree for multi-agent date ranges.
