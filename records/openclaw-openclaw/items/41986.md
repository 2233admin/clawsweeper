## Summary

Keep open. Current main still has the split described in the report: the Usage UI fetches `sessions.usage` and `usage.cost` separately, the header renders `sessions.usage` totals, and unscoped `usage.cost` still loads a single transcript scope instead of aggregating over the configured agents that `sessions.usage` scans. Live PR metadata also shows #70205 remains open and unmerged as the active fix candidate, while #42063 was closed unmerged.

## Best Possible Solution

Keep this issue open and review/land #70205 or an equivalent gateway/UI fix. The fix should make unscoped `usage.cost` aggregate over the same configured-agent scope as `sessions.usage`, preserve explicit agent scoping, include the resolved scope in cache keys, keep costs in USD, and add regression coverage proving header totals and daily bars agree for multi-agent date ranges. After that, validate the activity-by-time token discrepancy separately.
