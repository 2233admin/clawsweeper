## Summary

Keep this issue open. Current main still preserves the Usage dashboard split described in the report: header totals come from `sessions.usage`, daily bars come from `usage.cost`, and `usage.cost` still loads one transcript scope instead of the configured agent set that `sessions.usage` discovers. The related open PR #70205 remains the right implementation candidate; #42063 was closed unmerged and #68066 tracks a different provider billing-accuracy bug.

## Best Possible Solution

Keep this issue open and review/land #70205 or an equivalent core fix. The fix should make unscoped `usage.cost` aggregate over the same configured gateway agent set as `sessions.usage`, preserve explicit agent scoping if added, include the resolved scope in cache keys, keep cost units documented as USD, and add regression coverage proving header totals and daily bars agree for multi-agent date ranges.
