## Summary

Keep open. Current main still has the reported Usage dashboard split: header totals come from `sessions.usage`, while daily bars come from `usage.cost`, and `usage.cost` still loads one default transcript scope instead of the configured agent set that `sessions.usage` discovers. Open PR #70205 remains the implementation candidate; #42063 was closed unmerged and #68066 tracks a different provider billing-accuracy problem.

## Best Possible Solution

Keep this issue open and review or land PR #70205, or an equivalent gateway/UI fix. The fix should make unscoped `usage.cost` aggregate over the same configured gateway agent set as `sessions.usage`, preserve explicit agent scoping, include the resolved scope in cache keys, keep cost units as USD, and add regression coverage proving header totals and daily bars agree for multi-agent date ranges.
