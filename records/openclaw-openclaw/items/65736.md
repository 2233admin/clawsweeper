## Summary

Keep this PR open. Current main still resolves exec approvals JSON and socket defaults through home-relative `~/.openclaw` paths, while `OPENCLAW_STATE_DIR` remains the documented mutable state-root override used by hosted deployments. This PR is also active after maintainer repair and merge-ready labeling, so it is an implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or explicitly supersede it with #62938 if maintainers choose that branch. The shipped fix should make exec approvals JSON/socket defaults honor an explicit `OPENCLAW_STATE_DIR`, preserve the legacy `~/.openclaw` fallback when unset, update effective-policy reporting and docs, keep path-safety checks intact, and include focused regression coverage.
