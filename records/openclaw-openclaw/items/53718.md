## Summary

Keep open. Current main still resolves shared user paths with tilde-only expansion, so the reported `${XDG_CONFIG_HOME}` ClawHub install bug is not implemented on main; this PR is a relevant fix candidate but overlaps #58281 and needs maintainer selection of the canonical resolver semantics before merge.

## What This Changes

The PR adds `$VAR` and `${VAR}` expansion to shared home-relative path resolution and adds resolver tests for braced, bare, unknown, inherited-key, empty-value, and tilde-valued placeholders.

## Best Possible Solution

Land one focused shared path resolver update that expands env placeholders consistently before filesystem resolution, preserves intentional passthrough for unresolved or unsafe placeholders, covers both resolver helpers and the ClawHub install path, then close #53628 and retire the overlapping unchosen PR.
