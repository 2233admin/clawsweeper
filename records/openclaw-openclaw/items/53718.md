Keep open. Current main still preserves `$VAR` and `${VAR}` placeholders literally in the shared user-path resolver, so this PR is still relevant to the linked ClawHub install bug; it needs maintainer consolidation with the overlapping open resolver PR rather than cleanup closure.

Land one focused shared resolver update that expands supported env placeholders before filesystem resolution, preserves deliberate passthrough behavior for unresolved or unsafe placeholders, covers both resolver helpers and the active ClawHub install path, then close #53628 and retire the unchosen overlapping PR.
