## Summary

Keep open. Current `main` still resolves only leading `~` in the shared path helpers, so configured paths containing `$VAR` or `${VAR}` still flow through as literal path segments; the PR remains a relevant implementation for the linked ClawHub install bug. The latest PR patch addresses the earlier functional review problems, and the remaining decision is maintainer selection/review across this PR and the overlapping #58281 rather than cleanup closure.

## What This Changes

The PR adds environment-placeholder expansion to `resolveHomeRelativePath` and `resolveOsHomeRelativePath` in `src/infra/home-dir.ts` and extends `src/infra/home-dir.test.ts` for braced, bare, unknown, inherited, empty, and tilde-valued placeholders.

## Best Possible Solution

Land one focused shared resolver update that expands env placeholders consistently before filesystem resolution, preserves deliberate passthrough behavior for unresolved or unsafe placeholders, covers both resolver helpers and the ClawHub install path, then close #53628 and retire the unchosen overlapping PR.
