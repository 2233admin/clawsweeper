## Summary

Keep this PR open. Current main still short-circuits fresh persisted token totals before the token-threshold compaction gate when transcript-byte compaction is inactive, so the intended behavior is not implemented. The PR is a focused fix candidate, but it now needs maintainer consolidation or rebase because overlapping fresh-token compaction work remains open.

## What This Changes

The PR changes the preflight compaction fast path to skip only when fresh persisted tokens are below threshold, adds focused regression tests, and adds a changelog entry for the auto-reply memory fix.

## Best Possible Solution

Land one consolidated fix in `runPreflightCompactionIfNeeded` that lets fresh persisted totals at or above `contextWindowTokens - reserveTokensFloor - softThresholdTokens` reach `shouldRunPreflightCompaction`, keeps below-threshold fresh totals on the cheap fast path, preserves transcript-byte compaction, and adds runtime regression coverage for both fresh above-threshold and below-threshold cases.
