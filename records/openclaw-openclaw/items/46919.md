## Summary

Keep open. Current main still does not implement the requested config flag to disable tokenized dashboard URLs plus direct tailnet/IP Control UI access with no token, no HTTPS, and no pairing. The implemented paths remain narrower: loopback-only `gateway.auth.mode: "none"`, verified Tailscale Serve identity-header auth for the Control UI WebSocket, and trusted-proxy auth.

## Best Possible Solution

Keep this open for maintainer product/security judgment. If accepted, the design should be a narrowly named trusted-tailnet/private-ingress mode with explicit warnings, clear HTTP API scope behavior, origin handling, and tests for Control UI WebSocket, HTTP endpoints, pairing, device identity, direct tailnet/IP, Tailscale Serve, and trusted-proxy interactions. If maintainers reject direct non-loopback no-auth as policy, close manually with that rationale and point users to Tailscale Serve identity-header auth or trusted-proxy auth.
