## Summary

Keep this issue open. Current main still lacks the requested Chrome OptGuide/WasmTts model-download suppression: the managed browser launcher only disables Translate/MediaRouter, headless mode only adds normal headless/GPU flags, the sandbox browser entrypoint has no disable-features list, and the public config/docs still expose only generic browser.extraArgs as a workaround. The related PR #41213 is closed unmerged, so it is not a landed fix.

## Best Possible Solution

Keep this issue open. The best fix is to centralize the bundled browser plugin's disabled Chrome feature list, apply the OptGuide/WasmTts model-download suppression to managed headless/server launches and the sandbox browser entrypoint, reconcile sandbox docs with actual entrypoint defaults, and add focused regression coverage for buildOpenClawChromeLaunchArgs plus a lightweight sandbox-entrypoint default check. The closed PR #41213 is useful reference material, but it is not landed.
