## Summary

Keep this issue open. Current main still does not implement default Chrome OptGuide/WasmTts model-download suppression: the managed browser launcher only disables Translate/MediaRouter, headless mode only adds normal headless/GPU flags, the sandbox browser entrypoint has no disable-features flag, and config/docs expose only the generic browser.extraArgs workaround. The related implementation PR #41213 is closed unmerged in the provided context, so it is not a superseding fix.

## Best Possible Solution

Keep this issue open. The best fix is to centralize the bundled browser plugin's disabled Chrome feature list, apply OptGuide/WasmTts model-download suppression to managed headless/server launches and the sandbox browser entrypoint, reconcile sandbox docs with actual entrypoint defaults, and add focused regression coverage for `buildOpenClawChromeLaunchArgs` plus a lightweight sandbox-entrypoint default check. The closed PR #41213 can be used as reference, but it is not landed.
