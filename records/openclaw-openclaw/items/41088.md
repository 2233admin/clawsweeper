## Summary

Current main still does not implement the requested Chrome OptGuide/WasmTts model-download suppression. The managed browser launcher still disables only Translate and MediaRouter, the headless resolver only decides whether to add headless/GPU flags, the sandbox entrypoint has no requested disabled-feature list, and there is no first-class browser.disableModelDownloads-style config surface. The related PR #41213 directly attempted the fix, but it is closed and unmerged after becoming stale against the moved browser plugin path.

## Best Possible Solution

Keep this issue open and land a focused replacement PR on current main: centralize or explicitly list the Chrome background ML/optimization feature disables in the browser plugin launcher, apply the same safe defaults to the sandbox browser entrypoint if it still launches Chromium directly, add launch-argument regression coverage, update user-facing browser/sandbox/config docs, and add a changelog entry with reporter attribution. Use PR #41213 as reference only; redo it against extensions/browser/src/browser/chrome.ts and avoid .secrets.baseline/generated churn.
