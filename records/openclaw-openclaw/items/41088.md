## Summary

Current main still does not implement Chrome OptGuide/WasmTts model-download suppression. The managed browser launcher only disables Translate and MediaRouter, headless mode only adds headless/GPU flags, the sandbox browser entrypoint has no `--disable-features` argument, and config/docs expose only the generic `browser.extraArgs` workaround rather than a default or dedicated model-download switch. The related PR #41213 is closed unmerged in the provided context, so it is not a superseding implementation.

## Best Possible Solution

Keep this issue open. The best fix is to centralize the disabled Chrome feature list in the bundled browser plugin, apply the OptGuide/WasmTts model-download suppression to managed headless/server launches and the sandbox browser entrypoint, reconcile the sandbox docs with the actual entrypoint defaults, and add regression coverage for `buildOpenClawChromeLaunchArgs` plus a lightweight sandbox-entrypoint default check.
