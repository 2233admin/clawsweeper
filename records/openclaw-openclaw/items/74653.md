## Summary

Keep open. The PR now addresses the earlier adapter bug by propagating the gateway payload's skipped-probe flag, but the renderer still treats every `checked: false` key-optional provider result as silent. Current main also uses `checked: false` for gateway memory probe timeouts, so the patch would hide an existing diagnostic for ollama/lmstudio when `doctor.memory.status` times out rather than only suppressing the deliberately skipped probe case.

## What This Changes

The PR propagates the gateway embedding probe `checked` flag through `probeGatewayMemoryStatus`, suppresses key-optional memory-provider doctor warnings when the probe is unchecked, and updates gateway/renderer tests for skipped-probe cases.

## Best Possible Solution

Keep the adapter propagation, but model skipped-probe as distinct from timeout/unavailable probe failures and suppress only the deliberate skipped-probe payload. Preserve a key-optional-provider timeout regression test so doctor still reports when the gateway cannot confirm memory readiness for reasons other than a planned non-deep check.
