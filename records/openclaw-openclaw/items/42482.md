## Summary

Keep this PR open. Current main has already shipped the deferred isolated-cron execution-timeout behavior, but this PR still carries a distinct abort/fallback isolation concern from the discussion and field reports. The remaining work is now stale against the refactored cron executor and should be rebased or replaced with a narrower patch plus regression coverage.

## Best Possible Solution

Keep this PR, or a rebased canonical replacement, open. The best maintainer path is to drop the already-shipped deferred-timeout portion, rebase the remaining abort/fallback isolation work onto src/cron/isolated-agent/run-executor.ts and runWithModelFallback, and add focused regression coverage proving retryable cron fallback failures do not poison later candidates while true cron budget aborts still stop the whole run.
