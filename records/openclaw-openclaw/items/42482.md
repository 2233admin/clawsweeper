## Summary

Keep this PR open. Current main ships the deferred isolated-cron execution-timeout behavior, but the per-attempt AbortController/fallback-signal isolation described in the PR is still missing from the current cron fallback executor.

## Best Possible Solution

Keep this PR, or a rebased canonical replacement, open. The best maintainer path is to drop the already-shipped deferred-timeout portion, rebase the remaining work onto src/cron/isolated-agent/run-executor.ts and runWithModelFallback, and add regression coverage proving later cron fallback candidates are not killed by stale attempt state while terminal cron budget aborts still stop the whole run.
