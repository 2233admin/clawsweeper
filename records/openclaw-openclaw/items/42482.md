## Summary

Keep this PR open. Current main has shipped the deferred isolated-cron execution-timeout behavior, but the per-attempt cron fallback abort isolation described by the PR is still not implemented in the current cron prompt executor.

## Best Possible Solution

Keep this PR, or a rebased canonical replacement, open. The best maintainer path is to drop the already-shipped deferred-timeout portion, rebase the remaining work onto src/cron/isolated-agent/run-executor.ts and runWithModelFallback, and add focused regression coverage proving later cron fallback candidates are not killed by stale attempt state while true cron budget aborts still stop the whole run.

- The PR is stale against current main: the deferred-timeout half has already shipped, while the remaining abort isolation now belongs in src/cron/isolated-agent/run-executor.ts rather than the older run.ts hunk.
- There is overlapping open design work around terminal aborts versus retryable failures, so maintainers should choose the canonical contract before landing another fallback-timeout patch.
- A per-attempt abort fix must preserve true cron budget cancellation; otherwise isolated cron jobs could run past the intended terminal timeout.
