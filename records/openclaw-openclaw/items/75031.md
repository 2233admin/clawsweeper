## Summary

Keep this PR open for maintainer review. Current main still stages backup archives under `os.tmpdir()` while the macOS service environment can set `TMPDIR` inside the OpenClaw state tree, and the PR is a scoped implementation for the linked duplicate-manifest bug with no definite blocking findings from the read-only diff review.

## What This Changes

The PR adds a backup temp-root containment guard, stages backup manifests under a safe temp root instead of direct `os.tmpdir()`, and adds regression tests for duplicate `manifest.json` entries and unsafe fallback roots.

## Best Possible Solution

Land this PR after maintainer and CI review so backup staging always uses a temp root outside selected backup sources, or fails before tar creation when no safe staging root is available.

## Reproduction Assessment

Yes. The high-confidence reproduction is the linked macOS LaunchAgent path: service `TMPDIR` resolves under the state directory, current main writes `manifest.json` there, and tar then sees that same file while walking the state payload.

## Solution Assessment

Yes. The PR keeps the fix inside backup archive creation, validates both preferred and fallback temp roots against the planned sources, and avoids changing global service `TMPDIR` behavior or tar semantics.
