## Summary

Keep this PR open. The linked backup bug is valid, but the latest diff still falls back to unchecked `os.tmpdir()` after detecting a state-local preferred root; under the macOS service environment that fallback is also state-local, so the duplicate-manifest bug can remain.

## What This Changes

This PR changes backup archive staging to call `resolvePreferredOpenClawTmpDir()` through a new backup temp-root guard and adds a regression test for a preferred temp root inside the state directory.

## Best Possible Solution

Keep the PR open and make backup staging choose only a temp root that has been checked against every included backup source. If no preferred or fallback candidate is outside the payload, fail before archiving with an actionable error, and add a regression where both the preferred resolver and `os.tmpdir()` resolve under the state directory.

## Reproduction Assessment

Yes. The high-confidence reproduction is the linked macOS LaunchAgent path: `TMPDIR` resolves to `<stateDir>/tmp`, the backup includes the state directory, and staging `manifest.json` below that tree lets tar include it once explicitly and once during the state walk.

## Solution Assessment

No. The PR starts in the right module, but returning unchecked `os.tmpdir()` after the preferred root fails containment is not the narrowest maintainable fix; every fallback candidate needs the same source-containment check or the archive should fail safely.
