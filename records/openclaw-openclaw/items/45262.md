## Summary

Keep open. Current main has not implemented `slashCommand.contextAware`, but this PR should not merge as-is because it is stale against the current Slack plugin layout and routes recent channel history into prompt fields that can become directive input.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config option and changes empty slash-command invocations to fetch recent Slack channel history and use it as the agent prompt.

## Best Possible Solution

Rework the feature in the current Slack plugin as an opt-in config that carries recent history as filtered untrusted or supplemental context, keeps command/directive parsing tied to the invoker's actual slash text, and updates schema, metadata, docs, and targeted Slack/directive tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep recent Slack history out of directive parsing:** `src/slack/monitor/slash.ts:597-598`
  - body: When the empty slash invocation builds `effectivePrompt` from channel history, the command source remains empty. Current directive resolution falls back to `BodyForAgent` when that happens, so prior Slack messages containing directives such as `/think`, `/reasoning`, `/elevated`, `/exec`, or `/model` can be applied to the invoker's turn. Carry the history as filtered untrusted context and keep a non-history command source for directive parsing.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete prompt/directive injection risk by treating recent Slack channel history as the slash-command prompt body.

Concerns:

- **[medium] Recent channel history can drive directives:** `src/slack/monitor/slash.ts:597`
  - body: Any channel participant could place directive-like text in recent history; an authorized user invoking the empty context-aware slash command would then send that history through `BodyForAgent`, which current directive parsing can fall back to when the slash text is empty.
  - confidence: 0.88
