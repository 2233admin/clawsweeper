## Summary

Keep open. The report targets the bundled Feishu document tool path, and current main still sends write/append/insert Markdown directly through Feishu `document.convert` without any special handling for `$$...$$` or `<equation>` formula content. I found no current source or tests that preserve LaTeX backslashes inside formula blocks, so this is not implemented on main and is a plausible narrow Feishu plugin bug.

## Best Possible Solution

Implement a Feishu-plugin-local conversion path that preserves raw LaTeX inside `$$...$$` blocks and `<equation>...</equation>` tags during `feishu_doc` write/append/insert, preferably by using the official Feishu equation block contract or a verified escaping strategy. Add regression tests for the complex formula from the report, keep the change out of core Markdown behavior, and document any remaining Feishu API limitation if full preservation is not possible.

## Clownfish Work Prompt

Observable bug: when `feishu_doc` write/append/insert receives Markdown containing `$$...$$` or `<equation>...</equation>`, complex LaTeX commands such as `\mathbb`, `\sim`, `\pi`, `\nabla`, `\log`, and `\theta` must survive into the Feishu document instead of being stripped by Markdown escape handling. Scope the fix to the bundled Feishu plugin document path; do not change core Markdown behavior or other channel plugins. Start in `extensions/feishu/src/docx.ts` around `convertMarkdown`, `chunkedConvertMarkdown`, and the write/append/insert callers. Read the official Feishu/Lark Docx equation block contract or SDK types first; prefer constructing explicit equation blocks for formula segments if supported, otherwise use a verified escaping strategy that does not regress simple formulas like `\frac{a}{b}`. Add focused regression tests in `extensions/feishu/src/docx.test.ts` using the complex formula from the issue and, if supporting `<equation>`, a tag-form case. Update `extensions/feishu/src/docx-types.ts` only if needed for a typed equation payload. Update the Feishu doc skill reference if behavior or limitations change. Add a user-facing changelog Fixes entry with `Thanks @gaohongkui` unless maintainers decide plugin-only bug fixes are being batched elsewhere. Validate with `pnpm test extensions/feishu/src/docx.test.ts`, targeted oxfmt check, and the normal changed gate in Testbox before handoff. Do not touch unrelated Feishu channel messaging, auth, table, image, or core plugin-sdk code.
