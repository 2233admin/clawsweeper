## Summary

Keep open. Current main still sends Feishu document write, append, and insert content through Feishu's Markdown conversion endpoint without an equation-aware preservation path, and the issue includes a concrete formula that maps directly to that bundled plugin surface.

## What This Changes

The issue requests that Feishu document creation/update preserve LaTeX backslashes inside `$$...$$` block formulas and `<equation>` tags.

## Best Possible Solution

Implement this in the Feishu plugin by detecting raw `$$...$$` and `<equation>` formula content before Markdown conversion, preserving LaTeX verbatim, and emitting the supported Feishu equation representation with regression coverage for the reported formula.

## Reproduction Assessment

Yes. The issue gives exact Markdown input, and current main has a deterministic source path that sends that content to Feishu's Markdown converter without equation-specific preservation; live Feishu credentials would complete rendered-output confirmation.

## Solution Assessment

Yes. A Feishu-plugin-local repair is the narrow maintainable direction because the failure is in the bundled Feishu document write surface, not a cross-plugin core Markdown contract.
