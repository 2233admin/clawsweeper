## Summary

Keep this PR open. Current main does not implement the central WhatsApp inbound poll/reaction/event/RSVP support, and the PR is not cleanly superseded because the related open reaction PR only covers a narrower subset. The PR also carries unresolved functional review concerns and unrelated core/build changes, so it should not be closed as implemented or merged without maintainer review and a tighter patch.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up, but ask for a focused WhatsApp-owned implementation: handle Baileys upsert/update/reaction payloads through the inbound pipeline, preserve voter/reactor identity and selected poll options, cover event/RSVP variants from the pinned Baileys contract, add targeted tests, and split unrelated core/build fixes into separate PRs or drop them if main already covers them. Coordinate the reaction-only subset with #46881 and the remaining reaction-querying work with #11460.
