## Summary

Keep this PR open. Current main still does not implement the central WhatsApp inbound poll, reaction, event, RSVP, or gated-message hook behavior, and the open reaction-only PR is a narrower overlap rather than a full replacement. The PR also needs maintainer review and narrowing before it is mergeable.

## Best Possible Solution

Keep the PR open, but ask for a rebased and narrowed WhatsApp-owned patch. It should route Baileys messages.upsert, messages.update, and messages.reaction through the inbound pipeline with correct voter/reactor identity, original message IDs, selected poll options, reaction add/remove state, event time/location, RSVP states, named listener cleanup, and targeted tests against Baileys 7.0.0-rc.9. Split or drop unrelated gateway, poll-parameter, and build-entry changes, and coordinate the reaction-only overlap with #46881 plus historical/query work in #11460.
