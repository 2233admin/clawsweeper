## Summary

Keep this issue open. Current main still lacks a restart-sentinel channel-notification suppression policy/config surface, and BOOT.md startup still appends the fixed `NO_REPLY` footer after workspace content. The Control UI model read-back portion appears covered on current main, but two of the three reported bugs remain actionable gateway/runtime work.

## Best Possible Solution

Keep this issue open and narrow the remaining work around gateway/runtime behavior: add a documented restart-sentinel channel-notification policy/config key and guard the outbound notice path in `src/gateway/server-restart-sentinel.ts`; then settle and test the BOOT.md silent-token contract in `src/gateway/boot.ts`. Preserve the current Control UI effective-model read-back behavior unless a fresh reproduction shows a remaining UI regression.
