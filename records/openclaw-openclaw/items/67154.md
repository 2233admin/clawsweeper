## Summary

Keep this open. Current main still has no dedicated `exec-runtime.json` store: usage metadata remains in the exec allowlist schema, and gateway, node-host, and macOS allowlist execution still records `lastUsed*` fields by saving back to `exec-approvals.json`. The related corruption class has an atomic temp-file rename mitigation, but that does not provide the requested immutable-allowlist/runtime-metadata split.

## What This Changes

The issue requests moving mutable exec usage metadata such as `lastUsedAt` and `lastUsedCommand` out of `exec-approvals.json` into a dedicated `exec-runtime.json` file while preserving existing allowlist behavior.

## Best Possible Solution

Add a reviewed core hardening change that introduces a backward-compatible `exec-runtime.json` usage store, stops ordinary gateway, node-host, and macOS allowlist execution from writing usage metadata to `exec-approvals.json`, preserves existing allowlist reads, and updates protocol validators, CLI, Control UI, macOS UI, docs, and tests to show merged legacy/runtime usage metadata without broadening exec policy.
