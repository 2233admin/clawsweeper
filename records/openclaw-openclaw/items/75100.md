## Summary

Keep this PR open for author or maintainer revision. Current main still lacks a Feishu segment delivery mode, but the proposed implementation can violate its ordering guarantee because media-only payloads bypass the segment delivery queue while a partial text flush is still in flight.

## What This Changes

The PR adds an opt-in Feishu `streamingMode: "segment"` config, segment flushing in the Feishu reply dispatcher, generated config metadata, docs, and Feishu dispatcher/schema tests while leaving CardKit streaming as the default.

## Best Possible Solution

Land the Feishu segment streaming option after every segment-mode outbound payload, including media-only sends, is queued behind or explicitly awaits `segmentDeliveryQueue`, with a regression test that defers a partial text segment and then delivers a media-only payload. Keep CardKit streaming as the default and keep generated metadata/docs aligned.

## Reproduction Assessment

Yes. A focused unit test can defer the first `sendMessageFeishu` segment flush from `onToolStart` or `onIdle`, then deliver a payload with `mediaUrls` and no text; on the PR branch, that media send reaches `sendMediaReplies` without waiting for the queued text segment.

## Solution Assessment

No. The feature belongs in the Feishu plugin and the config shape is narrow, but the current implementation is not the best merge shape until media-only delivery uses the same ordered segment path as text and tool payloads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Queue media-only sends behind segment flushes:** `extensions/feishu/src/reply-dispatcher.ts:663-664`
  - body: In segment mode, media-only payloads with `shouldDeliverText` false reach this branch and call `sendMediaReplies` without waiting for `segmentDeliveryQueue`. If a tool-start or idle partial flush is still sending, the attachment can arrive before the text segment, breaking the ordering guarantee this mode is adding. Queue or await media sends on the same segment delivery path.
  - confidence: 0.86
