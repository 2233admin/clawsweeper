## Summary

Keep open. Current main still has no first-class path that imports a loose `.jsonl` session backup into the session store so it appears in the sessions menu. The current UI and Gateway listing paths are backed by `sessions.json` entries, and the existing CLI/session APIs cover list, create-empty, cleanup, export, preview, patch, reset, delete, and Claude CLI history augmentation, not arbitrary OpenClaw JSONL transcript import.

## Best Possible Solution

Keep this issue open and implement a first-class import path. The conservative shape is `openclaw sessions import <path>` backed by a shared helper that validates an OpenClaw/PI JSONL transcript, derives or accepts a safe key/label/sessionId, copies the transcript into the selected agent's sessions directory, writes the matching `sessions.json` row, and then lets `sessions.list`, `sessions.preview`, `chat.history`, CLI listing, and Control UI render it through the existing store-backed path. Document it on the public sessions docs and add focused malformed-file, duplicate, path-safety, and list/history visibility coverage.
