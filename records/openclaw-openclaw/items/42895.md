## Summary

Keep this PR open. Current main and the latest release still use the old home-directory fallback chain with no guarded Android/Termux PREFIX fallback, while the PR remains a focused implementation candidate for the linked Termux crash and has addressed the earlier review concerns around tilde expansion and generic Android chroots.

## Best Possible Solution

Review and land this focused PR, or a maintainer-equivalent implementation, in src/infra/home-dir.ts. Preserve OPENCLAW_HOME, HOME, USERPROFILE, OPENCLAW_STATE_DIR, and OPENCLAW_CONFIG_PATH precedence; add a guarded Termux PREFIX fallback for both normal home resolution and tilde expansion; keep the regression tests and docs/changelog update; and either align the memory-host SDK resolver or track that explicitly as a follow-up.
