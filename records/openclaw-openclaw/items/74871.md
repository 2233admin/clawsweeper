## Summary

Keep this PR open, but do not land it as-is. The diff removes the current `openai-codex/gpt-5.4-mini` suppression even though current main intentionally treats that PI Codex OAuth route as unsupported in source, docs, tests, and changelog; the PR body's live evidence is for the separate Codex app-server `codex/gpt-5.4-mini` route.

## What This Changes

The PR deletes the `openai-codex/gpt-5.4-mini` `modelCatalog.suppressions` entry from the OpenAI plugin manifest.

## Best Possible Solution

Keep the suppression unless maintainers can live-verify `openai-codex/gpt-5.4-mini` through the PI `openai-codex-responses` route; if verified, update the manifest, resolver/provider tests, docs, and changelog together so OpenClaw has one consistent route contract.
