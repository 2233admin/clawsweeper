## Summary

Keep open. Current main does not implement the TUI `/upload` command or an equivalent interactive local-file context flow. The PR is a plausible focused feature, but it is not obsolete or shipped, and the latest PR review context still identifies safety/lifecycle issues to fix before merge.

## Best Possible Solution

Keep #58636 open. The best path is to continue the PR or rework it on main with a scoped TUI `/upload` command, a tested file-browser selector, regular-file validation, binary detection, size/lifecycle safeguards, and regression tests for stale pending context and skipped files. WebChat image attachments are not a substitute for this TUI file-context workflow.

## Risks / Open Questions

- Merging the PR as-is could leave stale pending upload context queued when all newly selected paths are skipped or unreadable.
- The proposed browser should filter or validate regular files before reading so FIFOs/devices cannot freeze the TUI.
- A maintainer should decide whether TUI file context should be prepended as text or flow through the existing gateway attachment contract before final polish.
