## Summary

Keep this issue open. Current main still creates `.jsonl.reset.*` archives and has improved daily-reset freshness semantics, but it does not provide the requested bulk restore/re-index utility, and archived reset transcripts are still not first-class in `sessions_history`, `chat.history`, or `sessions.list`. The timeline also points at open same-author implementation work (#73883, and earlier #60409), so the best path is maintainer review of that recovery design rather than cleanup closure.

## Best Possible Solution

Keep this issue open until maintainers choose the recovery path: either add a conservative first-party restore/re-index command or doctor repair for `.jsonl.reset.*` archives, or explicitly supersede the script request once archived reset sessions are first-class in `chat.history`, `sessions_history`, and session listing/UI. The design should preserve stable session identity, original/restored paths, timestamps, reset/archive timestamp, and enough metadata for downstream history viewers.
