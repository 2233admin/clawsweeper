## Summary

Current main only partially overlaps the request. It has hybrid vector/BM25 ranking, optional exponential temporal decay, MMR, and weighted dreaming promotion, but it does not implement write-time 1-10 memory importance scoring, persisted importance/access metadata, access boosting, Weibull importance decay, or retrieval ranking by similarity times importance. The related #71590 PR is open/draft and only covers a Weibull temporal-decay model, while #71656 is an adjacent diagnostics/scoring issue rather than a canonical replacement.

## Best Possible Solution

Keep #57307 open as the canonical feature request. The best maintainer path is a focused memory-core design/PR that adds a backward-compatible importance/access metadata contract, configurable decay model including the requested Weibull shape if accepted, access-boost semantics, QMD/builtin behavior alignment, config/docs updates, and retrieval tests proving important older memories can outrank trivial stale matches without regressing current hybrid vector/BM25 scoring.
