## Summary

Keep this PR open. Current main still restores `openclaw.json.bak` in the suspicious-read recovery path without checking the parsed backup for redacted secret placeholders, while the linked #68423 remains open. The PR is a relevant implementation candidate, but it needs maintainer review because current main now has adjacent last-known-good placeholder detection and the PR review thread raised concrete detector coverage gaps.

## What This Changes

This PR adds a new config backup-pollution detector with tests and calls it from both async and sync suspicious-read recovery before copying `.bak` over the live config.

## Best Possible Solution

Keep this PR as the implementation candidate for #68423, rebase it onto current main, share one redacted-placeholder detection contract across `.bak` and last-known-good recovery, and land it with async plus sync regression tests that prove polluted backups are refused before any restore copy.
