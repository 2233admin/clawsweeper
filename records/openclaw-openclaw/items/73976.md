## Summary

Keep this PR open. Current main still lacks the provider-backed hybrid per-keyword FTS fallback: the no-provider path fans out through extracted keywords, but the provider-backed hybrid path still performs a single full-query `searchKeyword(cleaned, ...)` call. The PR is an active, narrow repair for #39484/#39555, but it still needs normal maintainer review around fallback scoring semantics and test durability before merge.

## What This Changes

The PR adds a bounded per-keyword FTS fallback helper for memory-core hybrid search, applies it to both FTS-only and provider-backed hybrid paths, unskips hybrid keyword tests, adds a bounded fan-out regression test, and updates the changelog.

## Best Possible Solution

Keep this PR open, preserve the #39555/@yangzi33 credit, keep the per-keyword fallback bounded, and have a memory-core maintainer either approve or revise the fallback scoring floor before running targeted memory-core tests plus `pnpm check:changed`.
