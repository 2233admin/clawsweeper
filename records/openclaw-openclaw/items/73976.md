## Summary

Keep this PR open. ## What This Changes

This PR adds a bounded `searchKeywordWithFallback` path for memory-core FTS-only and provider-backed hybrid search, re-enables skipped hybrid keyword tests, adds a fan-out bound test, and records the fix in the changelog.

## Best Possible Solution

Land a bounded provider-backed hybrid FTS fallback in `memory-core` if maintainers accept the recall/scoring semantics, preserving the #39555/@yangzi33 credit and keeping the fan-out cap covered by tests.

## Reproduction Assessment

Yes. The source-level reproduction path is clear: on current main, provider-backed hybrid search sends the full cleaned query into a single FTS AND query, while the two intended hybrid keyword tests are still skipped; the PR changes that exact path and adds a bounded fan-out regression test.

## Solution Assessment

Unclear until maintainer review. The capped helper is a narrow maintainable direction for the zero-hit bug, but the fallback score floor change from the current text-weight floor to `0` should be explicitly approved or adjusted before merge.
