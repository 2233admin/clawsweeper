## Summary

Keep this PR open. Current main still has the cron normalization bug the PR targets: implicit nested `payload.text` plus `model` is inferred as `systemEvent` and prunes the model, and the legacy top-level `text` plus `model` path still infers `systemEvent` before copying agent-turn fields. The provided PR context shows the branch now addresses both paths and has maintainer follow-up plus merge-ready labels, so it is an active implementation candidate rather than cleanup to close.

## Best Possible Solution

Keep this PR open, review the current branch, and land it if the targeted cron normalization tests and changed gate remain green. After merge, the linked stale cron model report can be closed by the PR’s closing syntax or normal implemented-on-main cleanup.
