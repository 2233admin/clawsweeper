## Summary

Keep open. Current main still lacks the requested `forcePruneRatio` config/runtime/schema/docs/test surface, and the existing context-pruning hook still exits during a fresh TTL window before estimating context usage. Related PR #13572 is closed unmerged, so there is no landed implementation to close against.

## What This Changes

The issue requests an optional `contextPruning.forcePruneRatio` setting that bypasses context-pruning TTL when estimated context-window usage is above a configured threshold.

## Best Possible Solution

Implement this as an additive context-pruning capability: accept and document `agents.defaults.contextPruning.forcePruneRatio`, carry it through effective settings and embedded runtime setup, estimate usage before the fresh-TTL early return, bypass TTL only when a finite known usage ratio meets the configured threshold, and preserve the current cache-TTL behavior otherwise.
