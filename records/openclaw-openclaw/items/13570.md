## Summary

Keep this issue open. Current main at b85edb3 still has no `contextPruning.forcePruneRatio` public config field or equivalent high-usage TTL bypass, and the context-pruning hook still returns during a fresh TTL window before the pruner can inspect context usage. The related PR #13572 is closed unmerged and remains useful reference material, not a shipped fix.

## Best Possible Solution

Implement the feature in current `main`: add `agents.defaults.contextPruning.forcePruneRatio` to the public config type, strict Zod schema, generated schema/help, and docs; thread it through `computeEffectiveSettings()` and runtime settings; estimate context usage before the fresh-TTL early return; bypass TTL only when usage is at or above the configured threshold; preserve existing TTL behavior below threshold or when the context window is unknown; and add focused regression tests for bypass and non-bypass paths. Use #13572 as reference only because it is closed unmerged.
