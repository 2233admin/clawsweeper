## Summary

Keep open. Current main still lacks the requested Apple Silicon Metal-safe local embedding path: the built-in local provider loads node-llama-cpp in the gateway process without a CPU/GPU/backend selector, local batches still call one embedding context concurrently, and the public docs do not define a supported CPU-only or safe Metal configuration. The fresh reproduction comments and still-open #60347 show this is active follow-up work, not cleanup-ready.

## What This Changes

The issue asks for official guidance and safer behavior for Apple Silicon local memory embeddings that can crash in node-llama-cpp/ggml-metal, including backend selection, safer batching, and gateway crash containment.

## Best Possible Solution

Keep this as the canonical Apple Silicon local-embedding stability tracker until OpenClaw has a documented macOS support matrix for built-in local and QMD embeddings, a validated node-llama-cpp CPU/GPU/backend config path, bounded or serialized local GGUF embedding calls, and a gateway-safe failure boundary or fallback for native embedding crashes.
