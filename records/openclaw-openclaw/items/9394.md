## Summary

Keep this issue open. Current main still does not provide outbound rich iMessage style spans, text-size attributes, underline/strikethrough handling, or the requested Shake/Nod/Explode/Ripple/Bloom/Jitter effects; legacy iMessage remains a plain imsg text path, while BlueBubbles has only a narrower effectId action and strips Markdown on normal sends. The issue discussion already includes maintainer-side context keeping this open for product/API follow-up, and that remains accurate against the current checkout.

## What This Changes

Request iMessage outbound support for rich text formatting, text-size styling, and Shake/Nod/Explode/Ripple/Bloom/Jitter effects instead of plain-text-only delivery.

## Best Possible Solution

Define a typed outbound capability model for iMessage and BlueBubbles formatting/effects, implement only the upstream-backed subset per transport, preserve plain-text fallback where rich formatting is unsupported, and document legacy imsg limitations in the public channel docs.
