## Summary

Keep this issue open. Current main still does not implement iMessage rich text spans, text sizing, or the requested newer text animations. Legacy iMessage sends a plain `text` payload through `imsg`, iMessage outbound delivery stays in plain-text chunking mode, and BlueBubbles only exposes a narrow `effectId` path while stripping Markdown before send. A maintainer-side Codex review already kept this open on 2026-04-26, and the current checkout still matches that assessment.

## Best Possible Solution

Keep this issue open for maintainer product/API direction. The likely path is a typed iMessage/BlueBubbles rich-text and text-effect send surface covering supported style spans, text size, and effect/animation enums, wired through outbound delivery and message actions with tests and public docs. Legacy `imsg` should either gain upstream-supported capability handling or document these attributes as unsupported.
