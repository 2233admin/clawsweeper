## Summary

Keep open. Current main still has only shallow HTTP liveness/readiness probes plus the WS/CLI health snapshot; it does not implement the requested native, auth-gated, no-inference `/health/deep` model-contract endpoint. The issue is recent, contributor-authored, and explicitly asks for maintainer API/security direction before a PR, so it should remain in the maintainer queue.

## What This Changes

Request a native `/health/deep` Gateway HTTP probe that reports the active provider/model/fallback contract and bounded routing/error telemetry without invoking inference.

## Best Possible Solution

Keep the issue open for maintainer product/API/security direction, then implement an additive no-inference Gateway probe or authenticated readiness-detail extension backed by existing model/fallback resolution plus bounded telemetry hooks. The implementation should document the public contract and cover local versus remote auth, unauthorized no-body/no-leak behavior, redaction, status semantics, no inference, and root-mounted Control UI routing.
