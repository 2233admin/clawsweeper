## Summary

Current main still lacks the requested native, auth-gated `/health/deep` no-inference model-contract endpoint. The existing HTTP probes are limited to shallow liveness/readiness, readiness details do not include provider/model/fallback/routing/error telemetry, and `/v1/models` is an agent-target list rather than backend model introspection. This is a recent contributor design proposal with concrete external overlay prior art and security/API questions, so it remains actionable maintainer work rather than cleanup-close material.

## Best Possible Solution

Keep this issue open for maintainer product/API/security direction. The likely path is an additive native gateway probe or authenticated readiness-detail extension backed by existing configured model/fallback resolution plus bounded routing/error telemetry hooks, with tests for local versus remote auth, unauthorized no-body/no-leak behavior, no-inference operation, redaction, status semantics, route reservation, and public health/model-contract docs. If maintainers prefer a plugin route, first document or add the readiness-detail auth/redaction contract and routing/error telemetry helper needed to implement it outside core.
