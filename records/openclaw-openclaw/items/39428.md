## Summary

Keep this issue open. Current main still routes Discord inbound accountId through the shared resolver, whose ordered match tiers are explicit bindings followed by default-agent fallback; there is still no implicit Discord account-name to agent-id match. The provided GitHub context also shows open PR #39485 with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review or land PR #39485, or an equivalent maintainer-approved fix, that adds a narrow Discord-only fallback after explicit binding tiers and before default-agent fallback. The fix should skip accountId "default", require the candidate agent to exist in agents.list, expose an unambiguous match reason such as implicit.account, and add regression tests for positive matching, default-account behavior, and the empty-agent-list guard.
