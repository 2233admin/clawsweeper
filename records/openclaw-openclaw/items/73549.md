## Summary

Keep this issue open. Current main supports plaintext SKILL.md discovery, per-agent skill filtering, watcher/version-based snapshot refresh, and persisted session skill snapshots, but it does not implement a signed/encrypted Markdown wrapper or a trusted commercial-source policy; open PR #74035 also uses closing syntax for this issue and is still unmerged.

## What This Changes

The issue requests persistent skill/.md configuration support for commercial use, including stronger skill persistence and a verified/signed or encrypted wrapper model for protected skill content.

## Best Possible Solution

Keep the issue open while maintainers review PR #74035 or define a narrower signed/private ClawHub skill-bundle design that preserves operator inspection, provenance, dangerous-code scan auditability, and documented commercial trust boundaries.

## Reproduction Assessment

Not applicable as a feature and product/security policy request. Source inspection gives a high-confidence check of current plaintext skill loading and snapshot behavior, but the report does not provide concrete steps to reproduce skills being forgotten or ignored.

## Solution Assessment

Unclear. A paid encrypted wrapper or trusted-source scan bypass is broader than a small persistence flag; the safer maintainable path is maintainer product/security review of PR #74035 or a signed/private ClawHub bundle model before treating this as the right fix.
