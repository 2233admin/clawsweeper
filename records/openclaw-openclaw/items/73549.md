## Summary

Keep this issue open. Current main already has persistent plain `SKILL.md` loading, per-skill config, ClawHub install/update flows, and session skill snapshots, but the issue’s remaining request is larger: official paid/verified or encrypted Markdown skill wrapping plus a change to commercial/security-review policy. That is not implemented or clearly covered by current plugin/ClawHub surfaces, and it needs maintainer product/security judgment rather than an automated cleanup close.

## Best Possible Solution

A maintainer should either narrow this into two tracks or respond with product direction: keep the existing plain-skill persistence path as-is, and separately decide whether commercial IP protection belongs in ClawHub/private packages, official publisher verification, or a new core/plugin API for signed/encrypted skill bundles. If the desired outcome is only stronger persistence, identify a concrete repro against current snapshot/config behavior; if the desired outcome is paid encrypted skills, open a design discussion with the required trust model and loader/API changes.
