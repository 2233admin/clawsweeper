## Summary

Keep open. Current main already has plaintext SKILL.md loading, persisted skill snapshots, watcher/config invalidation, ClawHub distribution, and fail-closed install scanning, but it does not implement the requested signed/encrypted Markdown wrapper, a `persist` skill flag, or a `trustedSources` scan-bypass contract. Open PR #74035 still references this issue with closing syntax, so the issue should remain open until that PR is reviewed/merged or maintainers choose a different product/security direction.

## What This Changes

Request asks OpenClaw to support persistent commercial skill/.md usage, including a verified or encrypted Markdown wrapper and less restrictive commercial skill security-review handling.

## Best Possible Solution

Keep the existing plaintext skill persistence path, then have maintainers either review a narrowly safe version of PR #74035 or define a signed/private ClawHub skill-bundle model with clear provenance, scanning, audit logging, and operator controls.

## Reproduction Assessment

Not applicable as a feature and product/security policy request. Current-main inspection gives a high-confidence check of existing plaintext skill persistence, but the issue does not provide concrete steps for reproducing skills being forgotten or ignored.

## Solution Assessment

Unclear. A paid encrypted wrapper or trusted-source scan bypass is broader than a small config flag; the maintainable direction needs explicit maintainer trust-model review, with ClawHub provenance/signing likely safer than a broad scan bypass.
