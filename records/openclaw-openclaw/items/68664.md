## Summary

Keep open. Current main already applies runtime legacy migrations before validation, so the broad ordering claim is partly outdated, but the concrete `agents.defaults.memorySearch` nested-key failure remains: the migration only moves top-level `memorySearch`, while the strict current schema still rejects obsolete root-level `chunkSize`, `chunkOverlap`, and `maxResults`. The issue is also paired with open fix PR #68685, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests startup-safe legacy config cleanup for obsolete `agents.defaults.memorySearch` keys so schema changes do not crash gateway startup after upgrades.

## Best Possible Solution

Review and land #68685 or an equivalent narrow migration that handles obsolete `memorySearch` keys before strict validation, with regression coverage for configs already using `agents.defaults.memorySearch`; then close this issue as implemented.
