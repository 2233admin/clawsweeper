## Summary

Keep this issue open. Current main still only remaps `containerWorkdir` paths through the read-tool workspace-root guard, while `workspaceAccess: "ro"` documents and mounts the agent workspace at `/agent`; current coverage still expects `/agent/secret.txt` to fail under the workspace-only guard. The related fix PRs were closed unmerged, and the merged adjacent sandbox handoff PR explicitly did not address `/agent` read-guard remapping.

## Best Possible Solution

Keep this issue open. The best fix is to make the sandboxed read workspace-root guard validate through the sandbox mount table, or pass an explicit `/agent` mapping only when the sandbox context has `workspaceAccess: "ro"` and that mount exists. Add regression coverage for both the guard helper and `createOpenClawCodingTools` proving `/agent/...` reads succeed while write/edit/apply_patch remain blocked; track `/tmp` reads separately.
