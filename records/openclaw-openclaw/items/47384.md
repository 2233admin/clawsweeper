## Summary

Current main still does not implement this PR's intended Telegram polling fallback. `deleteWebhook` recoverable failures still go straight to retry without checking `getWebhookInfo`, and current tests still assert retry behavior rather than already-absent continuation. The PR should remain open for maintainer review/rebase.

## Best Possible Solution

Keep this PR open. The best path is to review and rebase the narrow Telegram plugin change, then land a fallback that checks `getWebhookInfo` only after a recoverable `deleteWebhook` failure, continues polling when the webhook URL is empty, preserves retry behavior when a webhook remains configured or confirmation also has a recoverable failure, and keeps the regression test.
