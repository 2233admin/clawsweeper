## Summary

Keep open. Current main still emits `llm_output` once after the embedded attempt with accumulated usage, while per-assistant-message handling records and commits usage without a per-call output hook. The related PR #45990 remains open, so it is an implementation candidate rather than shipped behavior.

## What This Changes

Request per-LLM-call observability in agentic loops by emitting per-call `llm_output` events or adding an `llm_call` hook with iteration usage and run/session/model identifiers.

## Best Possible Solution

Land an additive, documented per-call observability contract in the embedded runner/subscription path. It should expose iteration usage plus provider/model/run/session/agent identifiers, clearly preserve or replace aggregate semantics, and include focused multi-round tests plus plugin hook documentation.
