## Summary

Keep this PR open. Current main still does not implement the PR's intended first-class Azure Claude AI Foundry onboarding path: there is no `anthropic-azure-api-key` choice, no dedicated `--anthropic-azure-*` flags or Azure Claude env vars, and the current Microsoft Foundry plugin is OpenAI/Responses or Chat Completions oriented rather than Anthropic `/anthropic`. The PR is still a focused implementation candidate, but it needs maintainer review/rework before merge.

## Best Possible Solution

Keep this PR open for maintainer review or ask the author to rebase and rework it as provider-owned Azure Claude/Microsoft Foundry support. The concrete path should add a first-class `anthropic-azure-api-key` or equivalent provider-plugin auth choice with interactive and non-interactive setup, validated `/anthropic` endpoint/resource normalization, `ANTHROPIC_FOUNDRY_*` / `AZURE_CLAUDE_*` env-var support, auth-profile metadata, docs, and regression tests, while resolving the outstanding endpoint-validation comments and avoiding core hardcoded provider behavior where the plugin API is sufficient.
