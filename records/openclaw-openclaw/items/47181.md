## Summary

Keep this PR open. Current main still does not implement the PR's intended first-class Azure Claude / Azure AI Foundry onboarding path: there is no `anthropic-azure-api-key` auth choice, no dedicated `--anthropic-azure-*` flags, no `ANTHROPIC_FOUNDRY_*` or `AZURE_CLAUDE_*` env-var handling, and the existing Microsoft Foundry path is Azure OpenAI / Responses / Chat Completions oriented rather than Anthropic `/anthropic`. The PR remains a plausible implementation candidate, but it needs maintainer review and likely provider-owned rework before merge.

## Best Possible Solution

Keep this PR open for maintainer review or ask the author to rebase and rework it as provider-owned Azure Claude support, likely in the Microsoft Foundry provider plugin or a dedicated `anthropic-azure` provider plugin. The implementation should add the auth choice/CLI flags/env vars through plugin manifest/auth APIs, validate `/anthropic` endpoint and Azure resource normalization strictly, store auth-profile metadata, add docs and regression tests, and resolve the outstanding endpoint-validation review comments before merge.
