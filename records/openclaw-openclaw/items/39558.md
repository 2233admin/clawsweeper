## Summary

Keep this PR open. Current main still creates Feishu reaction synthetic message IDs with `:reaction:<emoji>:<uuid>`, still lacks the proposed `stripFeishuReactionSuffix` helper, and still passes caller-provided message IDs directly into Feishu message, reaction, typing, media, pin, and streaming-card API boundaries. The linked bug #34528 is still open, and live PR metadata shows this PR is open, unmerged, and recently repaired to preserve the contributor path. Security pass found the PR file list scoped to Feishu plugin code/tests plus changelog, with no CI, dependency, lockfile, install, release, secret, or permission-surface changes.

## Best Possible Solution

Keep this PR open and review or land it as the canonical fix for #34528. The right path is a Feishu-plugin-local `stripFeishuReactionSuffix()` helper applied at every external Feishu API boundary that accepts a message ID, while preserving suffixed synthetic IDs internally for dedupe/session identity, with focused tests for valid suffix stripping and malformed/partial suffix passthrough.
