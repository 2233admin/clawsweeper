## Summary

Keep this PR open. Current main still generates Feishu synthetic reaction message IDs with the reported `:reaction:<emoji>:<uuid>` suffix, still has no `stripFeishuReactionSuffix` helper, and still passes raw message IDs into Feishu message, reaction, typing, media, pin, and streaming-card API calls. The linked bug #34528 remains the matching open issue, and this PR is the active contributor path for that fix.

## Best Possible Solution

Keep this PR open and review or land it as the canonical fix for #34528. The best implementation remains a Feishu-plugin-local message ID normalizer applied at every external Feishu API boundary that accepts a message ID, while preserving suffixed synthetic IDs internally for dedupe/session identity and covering valid, clean, malformed, and partial suffix cases with focused tests.
