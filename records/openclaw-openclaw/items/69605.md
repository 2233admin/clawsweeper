## Summary

Keep this issue open. Current main at 213f92a9ef0eb54972ef6cbb0e8b900adc8699b0 still has the relevant Control UI behavior: code-block copy buttons store payloads in data-code, the delegated chat handler copies that dataset value, and assistant bubbles with message-level copy/expand actions still do not add the existing has-copy spacing class. The issue is also paired with open, unmerged in-repo fix PRs #69634 and #73633 that use closing syntax, so it should stay open until maintainers review and land or close the canonical fix.

## Best Possible Solution

Keep this issue open and review the linked fix PRs. The best product fix is one canonical Control UI patch that prevents message-level action overlap and makes the code-block Copy handler copy only the rendered block content, with focused regression coverage for assistant bubbles with actions, multiple fenced blocks, special characters, and code-block click targets.
