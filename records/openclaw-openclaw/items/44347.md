## Summary

Keep this issue open. Current main covers the threaded outbound reply path, but it still does not implement the second requested capability: receiving non-mention Google Chat space messages via user OAuth or an equivalent event subscription mode. Related PR #68967 is open and only addresses thread/session behavior, while adjacent issue #9764 tracks broader Google Chat user OAuth work that would likely share the same credential foundation.

## Best Possible Solution

Keep this issue open, or split it after #68967 is resolved. The best path is extension-owned: add a Google Chat user-OAuth or verified event-subscription mode for non-mention space messages, keep the existing service-account webhook path fail-closed, extend config/schema/docs with secret-aware credentials and delivery-mode behavior, and add focused tests for routing, mention gating, redaction, and token selection.
