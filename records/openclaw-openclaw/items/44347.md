## Summary

Keep this issue open. Current main implements threaded Google Chat replies for messages that are delivered to OpenClaw, but the second requested capability, optional receipt of all non-mention messages in Google Chat spaces, is still missing from the service-account webhook, auth, config, setup, and docs surfaces.

## What This Changes

The issue requests threaded outbound replies for Google Chat plus an optional way to receive all non-mention messages in spaces.

## Best Possible Solution

Keep this issue open, or split it after the threaded-reply half is fully resolved. The remaining best path is Google Chat plugin-owned: design an optional user-OAuth or Workspace Events delivery mode for non-mention space messages, keep the service-account webhook path fail-closed, add secret-aware config/schema/setup/docs, and coordinate with #9764 for shared OAuth foundations.
