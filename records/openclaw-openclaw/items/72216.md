## Summary

Keep open. Current main still creates the Nostr setup-only wizard through `createDelegatedSetupWizardProxy`, and that proxy delegates status resolution through `loadWizard`, which imports the full `setup-surface.js`. The full surface imports `nostr-key-utils`, which imports `nostr-tools`, so the setup-status import path this PR targets is still present on main. The PR is a narrow implementation with a focused regression test and no apparent security-sensitive scope expansion.

## Best Possible Solution

Keep this PR open for maintainer review and either land it or apply an equivalent patch: keep Nostr setup status resolution local to `extensions/nostr/src/channel.setup.ts`, keep the full `setup-surface.js` loaded only for real setup/runtime work, retain the regression test proving status checks do not import the full surface, and keep the changelog attribution if the fix lands for a user-facing release.
