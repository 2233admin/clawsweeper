## Summary

Keep open. Current main still has the Nostr setup-status path this PR is meant to fix, and the functional approach is narrow, but the branch now has a blocking changelog placement issue after v2026.4.29 shipped.

## What This Changes

The PR replaces Nostr's setup-only delegated wizard proxy with a local lightweight setup-status wizard, adds a regression test that fails if `setup-surface.js` is imported during status checks, and adds a changelog entry for #72182.

## Best Possible Solution

Land this PR or an equivalent owner-local patch after moving the changelog entry to `## Unreleased` / `### Fixes`: keep Nostr setup status resolution inside `extensions/nostr/src/channel.setup.ts`, reserve the full `setup-surface.js` for real setup/runtime work, and keep the regression test proving status checks do not import the full surface.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. The current-main source path is enough: setup status collection calls the Nostr wizard status, the delegated proxy loads `setup-surface.js`, and that surface reaches `nostr-tools` through `nostr-key-utils`.

## Solution Assessment

Is this the best way to solve the issue? Functionally yes: changing only the Nostr setup-only surface is narrower than weakening the shared delegated proxy for every plugin. The changelog part needs one repair because the entry now targets a shipped release section instead of the active Unreleased section.
