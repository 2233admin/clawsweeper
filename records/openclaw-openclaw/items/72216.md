## Summary

Keep open for maintainer review. Current main still has the Nostr setup-status delegation that can load the full setup surface, while this PR directly removes that status-time import path and adds targeted regression coverage; I did not find a blocking correctness or security issue in the proposed diff.

## What This Changes

The PR replaces Nostr's setup-only delegated wizard proxy with a local lightweight status wizard, adds a regression test that fails if `setup-surface.js` is loaded during status checks, and adds a changelog entry for #72182.

## Best Possible Solution

Land this PR or an equivalent owner-local patch after rebasing: keep Nostr setup status resolution inside `extensions/nostr/src/channel.setup.ts`, reserve `setup-surface.js` for full setup/runtime work, and keep the regression test proving status checks do not import the full surface.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main status collection calls the Nostr setup wizard status, the delegated proxy loads `setup-surface.js`, and that surface reaches `nostr-tools` through `nostr-key-utils`.

## Solution Assessment

Yes. This is the narrowest maintainable fix because it changes only the Nostr setup-only status surface and avoids weakening the shared delegated proxy behavior used by other plugins; the branch still needs normal maintainer rebase/review before landing.
