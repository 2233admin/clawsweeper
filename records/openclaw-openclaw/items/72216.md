## Summary

Keep open. Current main still builds the Nostr setup-only wizard with `createDelegatedSetupWizardProxy`, and that proxy delegates status resolvers through `loadWizard`, which imports the full `setup-surface.js`. The full surface imports `nostr-key-utils`, which imports `nostr-tools`, so the #72182 failure path remains present on main. The PR is a focused implementation with a regression test and no security-sensitive scope expansion.

## Best Possible Solution

Keep this PR open for maintainer review and either land it or apply an equivalent patch: keep Nostr setup status resolution local to `extensions/nostr/src/channel.setup.ts`, keep the full `setup-surface.js` loaded only for real setup/runtime work, retain the regression test proving status checks do not import the full surface, and keep the changelog attribution if the fix lands for a user-facing release.
