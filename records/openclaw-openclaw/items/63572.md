## Summary

Keep open. Current main still lacks both requested remedies: `openclaw doctor` has no stale cross-OS workspace-path detector/repair, and the migration guide still does not warn users to rewrite absolute workspace paths after moving between OS home layouts. The provided GitHub context also shows #63803 is an open, unmerged PR with closing syntax for this issue, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that OpenClaw either detect and repair stale absolute `agents.defaults.workspace` and per-agent `workspace` paths during `openclaw doctor`, or document the required path rewrite step for cross-OS migrations.

## Best Possible Solution

Review #63803 as the focused implementation candidate. The desired end state is a doctor-owned diagnostic/repair for stale home-shaped absolute workspace paths in `agents.defaults.workspace` and per-agent `workspace`, plus a migration-guide warning that points users to the repair path; if that PR is not accepted, an equivalent doctor/docs fix should land before closing this issue.

## Reproduction Assessment

Yes. The reporter provided a concrete WSL/Linux to macOS migration path and stack trace, and current code still resolves configured workspace paths directly into `fs.mkdir(dir, { recursive: true })` without a doctor detector or migration-doc warning.

## Solution Assessment

Unclear until a fix is merged. The #63803 direction appears to be the narrowest maintainable path because it keeps legacy config repair in doctor and documents the migration footgun, while avoiding startup-time config mutation.
