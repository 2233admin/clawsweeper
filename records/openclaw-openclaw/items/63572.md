## Summary

Keep open. Current main still lacks both requested remedies: the migration guide does not warn about cross-OS absolute workspace paths, and doctor does not scan or repair stale `agents.defaults.workspace` or per-agent `workspace` values. The provided context also shows #63803 is an open, unmerged implementation PR with closing syntax for this issue, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a migration-guide warning or `openclaw doctor` detection/repair for stale absolute agent workspace paths after moving OpenClaw state between OS home-directory layouts.

## Best Possible Solution

Land #63803 or an equivalent focused fix that keeps legacy config repair in doctor: detect stale home-shaped absolute paths in `agents.defaults.workspace` and per-agent `workspace`, offer a conservative repair to portable `~`-relative paths or clear manual instructions, add regression coverage, and document the cross-OS migration pitfall on the public migration guide.
