## Summary

Keep this PR open. Current main still keeps `/status` off the Control UI local-command path: selecting it from the slash menu only fills the draft, and the next Enter sends it. The linked bug remains unresolved on main, while this PR is still a plausible implementation candidate with unresolved correctness concerns around duplicating the richer agent-side status path.

## Best Possible Solution

Keep this PR open, but revise the implementation. The safest path is either to make selecting a no-argument agent-routed command like `/status` submit with one Enter while preserving the existing agent-side status builder, or to add a gateway-backed local status endpoint that reuses the same status inputs and resolves per-session thinking and fast-mode defaults accurately. Add focused Control UI regression coverage for the single-Enter slash-menu flow plus status correctness for configured per-model defaults.
