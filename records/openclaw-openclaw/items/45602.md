## Summary

Keep this PR open. Current main still does not implement the PR’s central Web UI workflow: selecting `/status` from the slash menu still follows the non-local agent-routed path, so the first Enter selects/fills the command instead of submitting it. The PR is a plausible implementation candidate, but its current local-status approach also has acknowledged correctness concerns versus the existing agent-side status builder.

## Best Possible Solution

Keep this PR open, but revise the implementation. The safest direction is either to make selecting a no-argument agent-routed command like `/status` submit with one Enter while preserving the existing agent-side status builder, or to add a gateway-backed local status endpoint that reuses the same status inputs and resolves per-session thinking and fast-mode defaults accurately. Add focused Web UI regression coverage for the single-Enter slash-menu flow and status correctness.
