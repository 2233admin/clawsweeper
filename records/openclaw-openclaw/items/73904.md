## Summary

Keep this PR open. Current main still lacks the initializer call in the subagent spawn preparation path, and the PR is a narrow, source-matched fix with focused regression coverage for the linked cold-registry failure.

## What This Changes

The PR adds `ensureContextEnginesInitialized()` to the subagent spawn dependency path, calls it before `resolveContextEngine()`, and extends the subagent spawn test helper plus context-mode tests to cover cold registry ordering.

## Best Possible Solution

Merge this branch, or an equivalent initializer-before-resolve patch with the same cold-registry regression test, after CI confirms the agent runtime boundary still builds cleanly. The linked regression reports can then close through the normal implemented path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a cold process where cliBackend chat has not initialized context engines, followed by `sessions_spawn`; current main then resolves the default `legacy` engine against an empty registry and returns the reported spawn-preparation error.

## Solution Assessment

Yes. Calling the existing idempotent built-in initializer immediately before spawn preparation resolves the context engine is the narrowest maintainable fix and matches adjacent initialized-first paths; making registration a broader top-level side effect would have wider bundling/runtime implications.
