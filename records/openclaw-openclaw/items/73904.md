## Summary

Keep this PR open. Current main still has the cold-registry ordering bug, and the branch applies the narrow initializer-before-resolver fix with focused regression coverage; I did not find a blocking correctness issue in the proposed patch.

## What This Changes

The branch adds `ensureContextEnginesInitialized()` to the subagent spawn dependency path, calls it before `sessions_spawn` resolves the context engine, and extends the subagent spawn tests/helpers with a cold-registry ordering regression.

## Best Possible Solution

Land this branch, or an equivalent one-line initializer call with the same regression coverage, after CI confirms the lazy runtime boundary still builds cleanly. Then the linked regression reports can close as implemented.

## Reproduction Assessment

Yes. The high-confidence source-level reproduction is a cold process where cliBackend chat never initializes context engines, followed by `sessions_spawn`; current main then resolves the default `legacy` engine against an empty registry and returns the reported error.

## Solution Assessment

Yes. Calling the idempotent built-in initializer immediately before spawn preparation resolves the context engine is the narrowest maintainable fix and matches existing initialized-first agent paths; moving registration to top-level module side effects would have broader bundling implications.
