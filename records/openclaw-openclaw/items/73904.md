## Summary

Keep open. Current main still has the spawn-side cold-registry failure, and the PR’s initializer-before-resolve change is the right narrow fix with focused regression coverage, but the branch still needs the required changelog entry before merge.

## What This Changes

The PR adds `ensureContextEnginesInitialized()` to the subagent spawn runtime dependency path, calls it before `prepareContextEngineSubagentSpawn()` resolves the context engine, and extends the context-mode test helper/test coverage for the cold-registry ordering case.

## Best Possible Solution

Land this PR, or an equivalent initializer-before-resolve patch with the same regression test, after adding the changelog entry and validating the exact head. Keep #75129 as the separate cli-compaction follow-up unless a maintainer intentionally combines both fixes.

## Reproduction Assessment

Yes. A cold cliBackend-only process can reach `sessions_spawn` without any prior context-engine initialization; current main then resolves the default `legacy` engine against an empty registry and returns the reported `Context engine "legacy" is not registered` error.

## Solution Assessment

Yes for the spawn endpoint. Calling the idempotent built-in initializer immediately before resolving the context engine is the narrowest maintainable fix and matches the adjacent subagent registry ordering; the separate cli-compaction site should be handled by #75129 or an equivalent follow-up.
