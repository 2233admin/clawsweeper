## Summary

Keep open. Current main has newer Bonjour recovery and model-prewarm mitigations, but the reported pre-sidecar hang is not proven fixed: `runtime.early` still completes before HTTP binding, and it still awaits discovery service startup without a generic timeout. A never-settling discovery service can still prevent `/healthz` from being served.

## Best Possible Solution

Keep this open and make pre-listen startup robust at the generic discovery seam: move local discovery off the blocking pre-bind path or wrap each `gatewayDiscoveryServices[].advertise()` call in a short bounded timeout that logs and continues. Add a regression test with a never-resolving discovery service and ask the reporter to retry the latest image with `OPENCLAW_DISABLE_BONJOUR=1` plus startup trace to confirm the suspected path.
