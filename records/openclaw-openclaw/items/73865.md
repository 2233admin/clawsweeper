## Summary

Keep open. Current main has bounded HTTP listener EADDRINUSE handling, but the reported symptom is specifically before sidecars and before /healthz responds. The current startup path still awaits `runtime.early` work, including gateway discovery/Bonjour startup, before calling `startListening()`, and that discovery service path has no core timeout or background handoff.

## Best Possible Solution

Keep this open and make pre-listen startup robust: move local discovery/Bonjour advertisement off the blocking `runtime.early` path or wrap each `gatewayDiscoveryServices[].advertise()` call in a short bounded timeout that logs and continues. Add a regression test where a discovery service never resolves and assert the Gateway still binds `/healthz` or fails with a bounded error, plus ask the reporter to retry with the latest image and with `OPENCLAW_DISABLE_BONJOUR=1` to confirm the suspected path.
