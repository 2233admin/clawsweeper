## Summary

Keep open. Current main still has no supported Feishu `ignoreFileContent` or attachment-content opt-out, and the Feishu inbound file path still downloads file/post media into `MediaPaths`, where shared media-understanding can extract file text and append it to the agent-facing prompt.

## Best Possible Solution

Keep this issue open and implement a Feishu-owned inbound attachment-content opt-out. Add a documented per-account/top-level Feishu setting, apply it before `resolveFeishuMediaList` downloads file/document resources or before those entries enter `mediaPayload`, preserve placeholder text such as `<media:document> (name)`, and add focused Feishu plus shared media-understanding regression tests. Update Feishu schema/config metadata/help and the public Feishu docs page.
