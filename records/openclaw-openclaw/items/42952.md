## Summary

Keep open. Current main still does not support Feishu `ignoreFileContent` or an equivalent inbound attachment-content opt-out, and the Feishu file/post media path still downloads attachments into `MediaPaths` where shared media-understanding can extract text and append it to the agent prompt.

## Best Possible Solution

Implement a Feishu-owned inbound attachment-content opt-out. Add a documented top-level and per-account Feishu setting, apply it before `resolveFeishuMediaList` downloads file/document resources or before those entries enter `mediaPayload`, preserve placeholder text such as `<media:document>`, and add focused Feishu plus shared media-understanding regression tests. Update Feishu schema/config metadata/help and the public Feishu docs page.
