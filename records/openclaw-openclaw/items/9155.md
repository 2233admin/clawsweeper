## Summary

Keep this issue open. Current main still does not expose `cron.defaults.delivery` or an equivalent primary cron delivery default; the only global cron delivery-like configuration is failure-notification-specific. Job creation and normalization still preserve explicit per-job delivery or fall back to `{ mode: "announce" }` without a default channel/target. The related #57638 is broader and open, while #41164 was closed back to this narrower tracker.

## Best Possible Solution

Keep this issue open as the focused tracker for a primary cron delivery default. Maintainers should decide whether to implement delivery-only `cron.defaults.delivery` now or fold it into the broader #57638 cron-defaults design; the implementation should add config types/schema/help/docs, service/gateway normalization, CLI and Control UI support, and regression coverage for precedence: per-job delivery first, then the global default, then the existing fallback behavior.
