## Summary

Keep this PR open. Current main still does not implement the PR's intended Slack allowlist behavior: `resolveSlackChannelConfig` only tries bare channel ID case variants and optional name keys, while the shared matcher does exact key lookup without adding `channel:`-prefixed aliases. The PR is a focused implementation candidate with regression tests, and the related earlier PR addressed a different Slack route-persistence hypothesis rather than this unresolved key-match gap.

## Best Possible Solution

Review and either land this focused Slack resolver/test change or apply the same behavior on main: add `channel:<channelId>` candidate variants in `extensions/slack/src/monitor/channel-config.ts` and keep the two regression tests in `extensions/slack/src/monitor/monitor.test.ts`. If maintainers want a broader canonical target syntax policy, document that separately, but the runtime bug should be fixed in the Slack plugin rather than closed as already implemented.
