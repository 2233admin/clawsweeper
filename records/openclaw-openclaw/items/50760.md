## Summary

Keep open. Current main has a shipped background-task ledger, CLI inspection, and chat-native `/tasks`, plus separate Control UI Sessions and Cron panels, but it does not implement this PR's intended Control UI Task Board tab/page/service/components. The PR remains net-new UI work rather than obsolete or already shipped.

## Best Possible Solution

Keep this PR open if maintainers still want a browser Control UI Task Board. The best path is to rebase it onto current main, decide whether the page should read from the shipped task ledger rather than merging sessions and cron data directly, wire the tab through navigation/state/i18n/rendering, address the existing review findings, and add focused Control UI tests. Do not close it as implemented by the chat-native `/tasks` command.
