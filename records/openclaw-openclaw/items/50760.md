## Summary

Keep this PR open. Current main still lacks the browser Control UI Task Board tab, route, state, service, adapters, and components that this PR proposes. Main does ship a background-task ledger, CLI `openclaw tasks`, and chat `/tasks`, but those are not equivalent to the PR's intended read-only Control UI page.

## Best Possible Solution

Keep this PR open if maintainers still want a browser Control UI Task Board. The best path is to rebase it onto current main, decide whether the page should read from the shipped task ledger through a proper Gateway/UI contract, wire the tab through navigation/state/i18n/rendering, and address the existing review findings with focused Control UI tests.
