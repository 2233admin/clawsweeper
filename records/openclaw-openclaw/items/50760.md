## Summary

Keep this PR open. Current main still does not implement the PR's intended browser Control UI Task Board tab/page: the shipped task system is available through CLI and chat/status surfaces, while Control UI has separate Sessions and Cron panels and no unified `/tasks` route, app state, renderer branch, or task-board module.

## Best Possible Solution

Keep this PR open if maintainers still want a browser Control UI Task Board. Rebase it onto current main, align the page with the shipped task ledger or an explicit Gateway/UI task contract, wire the `tasks` tab through navigation/state/i18n/rendering, address the existing adapter/page review findings, and add focused UI tests.
