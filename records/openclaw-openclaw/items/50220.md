## Summary

Current main preserves explicit `host=node` workdirs and no longer resolves `/Users/...` on the gateway filesystem before forwarding to the node. That matches the reported Linux-gateway to macOS-node failure shape, is covered by a `/Users/vv` regression test, and shipped in `v2026.3.31` via #50961.

## Best Possible Solution

Close as implemented by #50961 and keep the current boundary: the gateway must forward explicit `host=node` workdirs unchanged, while the node remains responsible for validating its own canonical cwd. A new report would be appropriate only if a current `v2026.3.31+` build still fails after confirming the node receives the intended `/Users/...` cwd unchanged.
