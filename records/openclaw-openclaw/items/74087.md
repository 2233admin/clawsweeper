## Summary

Keep open. Current main partially addresses the report with packaged-launcher Node compile-cache enablement, documented `NODE_COMPILE_CACHE`/`OPENCLAW_NO_RESPAWN=1` tuning, and a respawn escape hatch, but it still does not provide a first-class `openclaw warm-cache` command, shipped pre-warmed cache artifact, or postinstall cache priming. The remaining shape needs maintainer product/security judgment.

## Best Possible Solution

Keep this issue open and have maintainers choose the accepted product shape. The safest likely direction is an explicit, documented `openclaw warm-cache --output <dir>` image-build helper with focused tests and VPS/container docs, while treating shipped cache artifacts or automatic postinstall prewarming as separate product/security decisions. The existing `OPENCLAW_NO_RESPAWN=1` env can remain the public respawn escape hatch unless maintainers want a CLI flag alias.
