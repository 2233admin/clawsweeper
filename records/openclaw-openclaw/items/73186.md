## Summary

Current main still leaves the reported cron announce reasoning-leak path unresolved, and an open PR with closing syntax is already paired to this issue. Cleanup should wait for maintainer review and canonical fix selection rather than closing this issue now.

## What This Changes

The issue reports that Matrix and Feishu cron announce delivery can expose model thinking/reasoning text and sometimes duplicate delivered content when isolated cron runs use thinking-enabled models.

## Best Possible Solution

Land one canonical fix that preserves isReasoning through embedded-runner payloads and filters reasoning payloads in the shared cron outcome path before summary selection, fallback synthesized text, deliverable payload selection, recovery checks, awareness events, and direct Matrix/Feishu announce delivery, with focused regression tests for mixed reasoning/final-answer and reasoning-only cases.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning reply items are created in buildEmbeddedRunPayloads, the returned payloads drop isReasoning, and the cron announce selectors and fallback delivery path can select non-error text without a reasoning guard.

## Solution Assessment

Unclear pending maintainer choice. The right boundary appears to be the shared embedded-payload and cron outcome path; a Matrix- or Feishu-only filter would leave shared cron selectors and other direct announce paths exposed.
