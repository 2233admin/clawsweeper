## Summary

Keep open. Current main still has the reported cron announce leak path: the embedded runner creates reasoning payload items but does not copy `isReasoning` into the returned payload object, and shared cron delivery selection still filters only `isError`. An open closing PR already exists, so this should wait for maintainer review instead of being closed or routed to a competing repair job.

## What This Changes

The issue requests preventing reasoning/thinking payloads from appearing in Matrix and Feishu cron announce delivery, including duplicate delivery caused by mixed visible and reasoning payloads.

## Best Possible Solution

Land one canonical, rebased fix that preserves `isReasoning` through embedded-runner payloads and filters reasoning payloads in the shared cron outcome path before summary, output text, single- and multi-payload delivery, awareness/error recovery decisions, and Matrix/Feishu direct announce delivery. The final patch should include focused regression tests for marker propagation, picker selection, deliverable payload selection, and the selected reasoning-only fallback policy.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning reply items are created, the returned payload drops `isReasoning`, and cron announce helpers can select the resulting text because they only skip `isError`.

## Solution Assessment

Yes on boundary, with one unresolved policy choice. The maintainable fix belongs in shared embedded-payload propagation and cron outcome filtering; channel-only Matrix/Feishu guards would be partial, and maintainers still need to decide whether reasoning-only payloads should ever fall back to delivery.
