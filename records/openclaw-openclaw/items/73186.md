## Summary

Keep open. Current main still has the reasoning-marker propagation and cron announce picker gaps, and an open closing PR already exists, so cleanup should wait for maintainer review and merge of one canonical fix.

## What This Changes

The issue reports leaked thinking/reasoning text and occasional duplicate delivery in cron announce output for Matrix and Feishu.

## Best Possible Solution

Land one canonical, rebased fix that preserves `isReasoning` through embedded-runner payloads and filters reasoning payloads in the shared cron outcome/picker path before summary, output text, deliverable payloads, awareness, and fallback delivery can reach Matrix or Feishu. The final patch should include focused regression tests for payload marker propagation and cron announce selection, and it should explicitly settle the reasoning-only fallback policy.

## Reproduction Assessment

Yes. The source path is enough to reproduce at high confidence: a reasoning payload is created with `isReasoning: true`, the returned payload drops that marker, and the cron announce helpers currently select text/deliverable payloads while skipping only `isError`.

## Solution Assessment

Unclear pending maintainer choice. PR #73363 is a focused implementation candidate, but its reasoning-only fallback behavior differs from the stricter no-reasoning fallback requested here and described by #40534.
