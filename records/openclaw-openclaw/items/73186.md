## Summary

Keep open. Current main still drops the embedded-runner `isReasoning` marker before cron outcome selection, the shared cron selectors still filter only error payloads, and open PR #73363 already uses closing syntax for this issue with #40534 overlapping.

## What This Changes

Prevent model thinking/reasoning payloads from being included or duplicated in Matrix and Feishu isolated cron announce deliveries.

## Best Possible Solution

Land one canonical fix that preserves `isReasoning` through embedded-runner payloads and filters reasoning payloads in the shared cron outcome and direct announce paths, with focused regression tests for Matrix/Feishu cron announce behavior and a clear decision on whether duplicate delivery is handled in the same change or split.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a reasoning reply item is created, the returned payload drops the marker, and cron announce selectors can then choose that text as summary/output/delivery content for Matrix or Feishu.

## Solution Assessment

Unclear pending maintainer selection of the canonical PR. The best boundary is shared embedded-payload plus cron outcome filtering; a Matrix- or Feishu-only patch would leave the cron selectors and other announce paths exposed.
