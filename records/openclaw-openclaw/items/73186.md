## Summary

Keep open. Current main still has the reported leak path: the embedded runner creates reasoning payloads but drops the marker before returning them, and cron announce selection still treats reasoning text like ordinary deliverable text. An open closing PR already exists, so cleanup should wait for maintainer review rather than close the issue early.

## What This Changes

The issue requests preventing reasoning/thinking payloads from appearing in Matrix/Feishu cron announce delivery and avoiding duplicate announce content.

## Best Possible Solution

Land one canonical, rebased fix that preserves `isReasoning` through embedded-runner payloads and filters reasoning payloads in the shared cron outcome/picker path before summary, output text, deliverable payloads, awareness events, and direct Matrix/Feishu announce delivery. The final patch should include focused regression tests for marker propagation, picker selection, deliverable payloads, and the chosen reasoning-only fallback policy.

## Reproduction Assessment

Yes. The source path is enough for high-confidence reproduction: a reasoning reply item is created, the returned payload drops `isReasoning`, and cron announce helpers then select non-error text/deliverable payloads without a reasoning guard.

## Solution Assessment

Unclear pending maintainer choice. PR #73363 is a focused implementation candidate, but its reasoning-only fallback differs from the stricter behavior described by #40534 and may not fully satisfy the no-reasoning-delivery expectation.
