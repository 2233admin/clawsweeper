## Summary

Keep open. Current main still has the shared cron announce reasoning-leak path, and open PR #73363 already uses closing syntax for this issue, with #40534 covering the broader overlapping fix shape.

## What This Changes

The issue reports that Matrix and Feishu cron announce delivery can expose reasoning content and sometimes duplicate delivered content when thinking-enabled models run isolated cron announce jobs.

## Best Possible Solution

Land one canonical fix that preserves `isReasoning` through embedded-runner payloads and excludes reasoning payloads in the shared cron outcome/delivery selectors, including Matrix and Feishu announce coverage and focused regression tests. Then close this issue through the merged PR.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a reasoning reply item is created, the returned payload drops the marker, and cron announce helpers select non-error text or deliverable payloads without a reasoning guard.

## Solution Assessment

Yes for the proposed boundary, but not yet implemented on main. The maintainable fix belongs in the shared embedded-payload and cron outcome path; a Matrix- or Feishu-only filter would leave other cron selectors and direct announce paths exposed.
