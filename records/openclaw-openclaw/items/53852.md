## Summary

Keep this issue open. Current main still routes isolated `agentTurn` cron jobs into the embedded OpenAI-compatible stream path, and that stream still appends each `function.arguments` chunk into `partialArgs` before parsing. The current repair wrapper covers fragmented arguments, provider preambles, and tiny trailing junk, but still rejects a second complete JSON object as unsafe trailing content, so the reported duplicated complete-JSON shape is not fixed.

## Best Possible Solution

Fix this at the OpenAI-compatible streaming/tool-call boundary: detect repeated or cumulative complete `function.arguments` payloads before appending or before parsing, safely collapse identical balanced JSON duplicates, strip scratch `partialArgs` from completed tool-call blocks, and add a focused regression test for an isolated `agentTurn` cron/OpenRouter-Kimi-shaped stream that emits the same complete arguments twice.
