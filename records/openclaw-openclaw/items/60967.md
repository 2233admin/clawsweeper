## Summary

Keep open. Current main still calls bundled MCP tools without passing MCP RequestOptions.timeout, while connectionTimeoutMs is only applied to the connection path. PR #73536 is an open, unmerged implementation candidate with closing syntax for this issue, so repo policy says to leave the issue open until that PR, or an equivalent fix, lands.

## Best Possible Solution

Keep this issue open and review PR #73536 as the canonical implementation candidate. The likely fix belongs in src/agents/pi-bundle-mcp-runtime.ts by carrying a resolved per-server request timeout into BundleMcpSession and passing it as MCP RequestOptions.timeout to client.callTool(); if maintainers prefer separate connection and request limits, add a documented config field with schema/type/docs/test coverage.
