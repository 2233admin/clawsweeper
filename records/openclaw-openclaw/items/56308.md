## Summary

Keep open. Current main still does not satisfy the user-facing request: `openclaw sessions` lists technical keys, the CLI row model drops stored label/display-title fields, and there is no `/title` slash command. Main does have useful partial plumbing in Gateway/TUI/Control UI, and the provided timeline points to external Clawdi-AI PR attempts, but that work is not shipped in this repository and does not make the OpenClaw issue obsolete.

## Best Possible Solution

Keep this issue open. A conservative implementation would add title/label display to `openclaw sessions` text and JSON output, expose a user-facing title command such as `/title` backed by the existing session patch/store validation, and decide whether automatic titles should be persisted or remain a bounded derived-title read from transcripts. Coordinate with #55249 for the narrower CLI label workflow.
