## Summary

Keep this issue open. Current main has meaningful partial session management, but the requested first-class topic/session management surface is still incomplete: Control UI has a Sessions table and chat session selector, TUI has a session picker, Gateway has create/list/search/patch/delete RPCs, and CLI has listing/cleanup. The broader requested lifecycle remains missing: no Control UI topic sidebar/create workflow, no CLI create/rename/show/export/delete lifecycle commands, no description/tags/message-count session contract, and no deleted-session archive restore/import/export API. The latest discussion also includes an active implementation offer for focused CLI lifecycle work, so this is not obsolete cleanup.

## Best Possible Solution

Keep this issue open as the umbrella tracker, then split implementation into focused PRs: Control UI topic/session creation and switching around existing Gateway RPCs, a maintainer decision on session metadata fields such as description/tags/messageCount, accepted CLI lifecycle commands such as list/show/export/delete, and a separate design for deleted-session archive restore plus import/export.
