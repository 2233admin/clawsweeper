## Summary

Current main still lacks the requested configurable lane wait diagnostic threshold: the queue warns from the per-entry `warnAfterMs`, missing options still default to `2_000`, diagnostics config/types/docs expose `stuckSessionWarnMs` but no lane wait setting, and the related implementation PRs are closed unmerged. This is a narrow, valid core follow-up rather than a stale or duplicate item.

## What This Changes

Add a documented configuration option for lane wait diagnostic warning thresholds so long-running cron jobs can avoid noisy hardcoded 2s wait warnings.

## Best Possible Solution

Keep this open and land a focused core patch adding documented `diagnostics.laneWaitWarnMs` as a positive millisecond value. Preserve the current 2,000ms default when unset, preserve explicit per-call `warnAfterMs` overrides, decide and test how the manual cron 5,000ms override interacts with the new config, wire reload/current-config behavior for future enqueues, and update schema/types/help/generated docs, queue/config docs, tests, and changelog. Treat #53023 as broader concurrency/auto-yield follow-up, not the canonical fix for this request.
