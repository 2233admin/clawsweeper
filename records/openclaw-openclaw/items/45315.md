## Summary

Keep this PR open. Current main at 7a23b2d945cc2d2e5c452e4641213e71de8dcc66 still does not implement the PR’s intended change: response-prefix context is seeded only with identityName, the fast-abort path returns before normal model selection populates model/thinking metadata, and unresolved template variables remain literal. The linked same-author bug report #45314 is still open, and draft PR #45454 is an implementation candidate rather than shipped proof.

## Best Possible Solution

Keep this PR open until maintainers choose a canonical fix for #45314. The landed implementation should populate response-prefix context before stop/abort reply normalization, include focused regression coverage for templated abort prefixes, and either fix or explicitly scope follow-up work for session overrides, channel overrides, routed replies, and catalog-backed thinking defaults. If #45454 becomes the preferred implementation, close this PR only after that work lands or a maintainer explicitly marks this one superseded.
