## Summary

Keep open. The remaining reply-to approval UX is still relevant because current main does not implement reply-only `/approve <decision>`, but the latest patch should not merge yet because the new extractor can read the wrong approval ID from plugin approval text.

## What This Changes

The PR branch adds reply-only `/approve <decision>` parsing from `ReplyToBody`, exports parser/extractor tests, and updates exec/plugin approval prompt text to mention reply and ID-prefix shortcuts.

## Best Possible Solution

Merge the reply-to approval UX after the extractor reads the canonical rendered approval ID as a full trimmed line, ignores earlier freeform `ID:` lines, and adds regression coverage for embedded `ID:` lines and whitespace-bearing custom exec IDs. Keep relying on main’s existing prefix lookup.

## Reproduction Assessment

Yes. A source-level reproduction is a plugin approval whose title or description contains a newline such as `ID: plugin:other` before the canonical rendered request ID, then replying with `/approve allow-once`; the PR regex captures the earlier line. A second focused check is an exec approval with a custom ID containing whitespace, which the PR extractor truncates at the first token.

## Solution Assessment

No. The prefix part is already solved on main by `lookupPendingId`; the reply-to part should parse the known approval-message structure and capture the canonical full ID line rather than the first `ID:` token in the reply body.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Capture the canonical full ID line:** `src/auto-reply/reply/commands-approve.ts:40`
  - body: `APPROVAL_ID_RE` captures only `\S+` from the first line named `ID:`. Plugin approval messages render freeform title/description before the canonical request ID, and those schema fields permit newlines, so a reply-only approval can route to a spoofed or truncated ID. Parse the canonical rendered request ID as a full trimmed line and cover embedded `ID:` lines.
  - confidence: 0.9
