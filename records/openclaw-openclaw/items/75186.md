## Summary

Keep open. Current main still lacks Gateway plugin management RPCs, and this active contributor PR is the implementation path rather than obsolete cleanup; latest head adds the intended surface but still has actionable review findings before it should merge.

## What This Changes

The PR adds typed Gateway plugins.* management RPC schemas, handlers, scopes, rate limits, shared plugin-management services, Swift protocol models, docs, and tests for remote plugin list, inspect, doctor, registry, install, update, uninstall, enable, and disable operations.

## Best Possible Solution

Land this direction only after fixing the install-error mapping, adding the required changelog entry, and getting maintainer signoff that the admin-scoped remote plugin-management contract is acceptable for Gateway clients.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The concrete review finding is reproducible by calling plugins.install with source path and a nonexistent Gateway-host path: current lower-level installers return uncoded local-input failures, and PR head maps that missing code to UNAVAILABLE.

## Solution Assessment

No, not as-is. The typed protocol plus shared service layer is a maintainable direction, but deterministic install input errors should map to INVALID_REQUEST or carry explicit client-error codes, and the user-facing API change needs release-note coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
