## Summary

Keep this issue open. Current main carries session lifecycle data through the Gateway and Control UI state, but the Sessions table still does not render an active/completed/timed-out status badge or allow sorting/filtering by lifecycle status. The overlapping #46792 is closed, but it does not supersede this remaining UI gap.

## Best Possible Solution

Implement a compact lifecycle Status column or badge in ui/src/ui/views/sessions.ts using GatewaySessionRow.status, subagentRunState, hasActiveSubagentRun, startedAt, endedAt, and runtimeMs. If scope allows, extend sorting/search or filtering for status, update SessionsProps/state wiring as needed, and add rendering coverage in ui/src/ui/views/sessions.test.ts for running, done, failed/killed, and timeout rows.
