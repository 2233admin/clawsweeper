## Summary

Keep this open. Current main carries session lifecycle status and timing through the Gateway and Control UI state, but the Sessions table still only renders kind, updated time, tokens, compaction, and per-session override controls. The prior maintainer review and the closed overlapping #46792 both point to the same remaining UI gap rather than superseding it.

## Best Possible Solution

Implement a compact lifecycle Status column or badge in the Control UI Sessions table, using the existing GatewaySessionRow lifecycle fields. Add rendering coverage for running, done, failed/killed, timeout, and missing-status rows; update sort/search or filtering only if it stays focused; adjust responsive table CSS and add a user-facing changelog entry if the PR ships the visible UI change.
