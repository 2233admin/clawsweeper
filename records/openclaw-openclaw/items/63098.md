## Summary

Current main still matches the reported Discord voice failure family: Discord voice join waits for @discordjs/voice Ready with a fixed 15s timeout, destroys the connection on Ready failure, and post-join Disconnected recovery still waits only 5s for Signalling/Connecting before silently destroying the session. The requested timeout/retry/logging/config path is not implemented in source, config schema, UI hints, tests, or docs, and the issue discussion includes confirmations through v2026.4.26.

## Best Possible Solution

Keep this issue open and fix the Discord plugin voice join/recovery path in the Discord plugin owner module. A focused fix should add regression coverage for initial Ready aborts and disconnected recovery failure logging, use a longer bounded default and/or documented `channels.discord.voice.connectTimeoutMs` or recovery timeout control, and avoid moving Discord-specific policy into core. If the final implementation fully covers #39825 or #65039, close those from the implementation PR; keep #63125 as the broader regression umbrella and #73978 as a separate slash-ack issue unless directly fixed.
