## Summary

Current main still lacks a supported cross-channel Plugin SDK hook for passive, read-only observation before requireMention drops group messages. The reporter's public archive plugin can use existing standard hooks, but full-fidelity mode still needs the missing core/plugin API seam rather than ClawHub-only work.

## Best Possible Solution

Keep this open and design a documented Plugin SDK seam for read-only early inbound observation. It should fire before requireMention drops, carry stable channel/account/conversation/thread/sender/message metadata, be explicitly opt-in for privacy, avoid session/reply creation by default, and add regression coverage across the affected bundled plugins.
