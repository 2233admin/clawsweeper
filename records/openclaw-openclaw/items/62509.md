## Summary

Keep open. Current main still runs the reported installer finalization probes without an internal timeout, and controlled read-only harness checks show both the npm-prefix path and daemon-status path can hang until killed by an outer timeout. The cross-referenced #73814 is useful related installer-hang context, but it appears to describe a separate piped-stdin/truncated-function failure rather than a canonical fix for this issue.

## What This Changes

Bound the installer finalization probes so `npm prefix -g` and `openclaw daemon status --json` cannot hang `install.sh` indefinitely.

## Best Possible Solution

Add a small portable timeout guard in `scripts/install.sh` for nonessential finalization probes, use it around npm prefix/config lookups and daemon-status detection, warn and continue when a probe times out, and cover the behavior with focused installer helper tests.

## Reproduction Assessment

Yes. A read-only harness that sources `scripts/install.sh` and stubs either `npm prefix` or the daemon-status command to sleep hangs until killed by an outer `timeout`, matching the reported finalization hang class.

## Solution Assessment

Yes. The narrow maintainable fix is installer-level bounding of best-effort probes, optionally also passing `--no-probe` and a short `--timeout` to daemon status, while keeping install success independent of those checks.
