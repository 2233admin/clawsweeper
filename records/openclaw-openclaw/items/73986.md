## Summary

Keep open. Current main still calls Feishu `messageResource.get` once with the caller-provided `type` and has no HTTP 502 `file` to `media` fallback, while the linked Feishu/Lark report gives concrete iOS video reproduction evidence. The live PR is a narrow implementation candidate with focused tests, limited changed files, and no obvious security-sensitive surface in the diff; it should go through normal maintainer review/landing rather than be closed as already implemented.

## Best Possible Solution

Keep this PR open for normal maintainer review and landing. The best path is to merge this narrow Feishu plugin fix, or an equivalent maintainer-approved patch, that retries `downloadMessageResourceFeishu` as `media` only after a file resource download fails with HTTP 502, preserves the original 502 if the retry fails, keeps the focused tests in `extensions/feishu/src/media.test.ts`, and keeps the changelog credit for @alex-xuweilong. After merge, #49855 can close as fixed and duplicate replacement PRs such as #73937 can be closed or redirected to the landed commit.
