## Summary

Keep open. Current main still downloads Feishu message resources with a single `messageResource.get` call using the caller-provided `type` and has no 502-to-`media` fallback, while related issue #49855 provides a concrete Feishu/Lark iOS-video reproduction. This PR is a narrow implementation candidate that carries forward #50164 and supersedes the earlier ProjectClownfish replacement #73937; it should be reviewed/landed or closed by maintainer judgment, not auto-closed as already implemented.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to land this narrow Feishu plugin fix or an equivalent maintainer-approved patch that retries `downloadMessageResourceFeishu` as `media` only for file downloads that fail with HTTP 502, preserves the original error on retry failure, adds focused regression tests in `extensions/feishu/src/media.test.ts`, and includes the changelog credit for @alex-xuweilong. After merge, #49855 can close as fixed and duplicate replacement PRs like #73937 can be closed against this canonical implementation.
