## Summary

Keep open. Current main still has explicit sub-agent spawning and repetitive loop detection, but it does not implement per-channel max-per-turn tool budgets, channel-scoped onExceed policies, or gateway-level auto-delegation after a budget is exceeded. The prior maintainer/Codex comment also reached the keep-open conclusion, and the current main checks did not find a newer implementation.

## Best Possible Solution

Keep this open for maintainer design. A good path is a generic runtime tool-budget policy that first aligns with #9912, defines config precedence across global/agent/channel/group scopes, and implements warn/block semantics before considering auto-delegation. Auto-delegation should reuse sessions_spawn/subagent limits but needs explicit child-task and result-delivery semantics.

## Risks / Open Questions

- Automatic delegation after a denied N+1 tool call needs a reliable remaining-task prompt and result path; the current before-tool-call surface mainly sees tool name, params, and run/session context.
- Config precedence needs product design because the request overlaps global #9912, tools.loopDetection, per-agent tool policy, and channel/group tool policy.
- Hard warn/block behavior can affect provider transcript validity and tool-result replay, so it needs embedded-runner and channel-routed session tests.
- Auto-spawning sub-agents can increase gateway resource pressure unless concurrency, child limits, run timeouts, and failed delivery behavior are part of the design.
