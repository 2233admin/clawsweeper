## Summary

Keep open. Current main still has adjacent loop detection, channel/group allow-deny tool policy, plugin before-tool hooks, and explicit `sessions_spawn`, but it does not implement channel-scoped max-per-turn tool-call budgets or the requested `warn`/`block`/`auto-delegate` overflow behavior. Recent comments and current HEAD checks both point to this needing maintainer design rather than cleanup closure.

## What This Changes

Requests a config-enforced per-channel tool-call limit with `warn`, `block`, and `auto-delegate` overflow behavior for deployments spanning channels such as Discord and Telegram.

## Best Possible Solution

Define a generic scoped tool-call budget model that resolves alongside existing effective tool policy. Ship deterministic run-scoped `warn` and `block` behavior first in the trusted before-tool path, then design `auto-delegate` as a separate extension of `sessions_spawn` with explicit authority inheritance and completion handoff semantics.
