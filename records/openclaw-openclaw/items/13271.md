## Summary

Keep open. Current main still has adjacent loop-detection, channel/group tool allow/deny policy, and explicit `sessions_spawn`, but it does not implement a channel-scoped max-per-turn tool-call budget or the requested `warn`/`block`/`auto-delegate` overflow behavior. The existing issue discussion also already kept this open for maintainer design follow-up.

## What This Changes

Request a per-channel tool-call budget that enforces a maximum number of tool calls per turn and handles overflow by warning, blocking, or auto-delegating to a subagent.

## Best Possible Solution

Define a generic scoped tool-call budget model that composes with the existing tool-policy pipeline. Land deterministic `warn` and `block` enforcement first in the trusted before-tool-call path, then design `auto-delegate` separately around `sessions_spawn`, child-run tracking, sandbox/tool inheritance, and completion handoff.
