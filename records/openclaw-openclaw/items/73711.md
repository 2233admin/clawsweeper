## Summary

Keep this PR open. Current main still has the old focused iOS composer attachment flow: the only PhotosPicker and picker loader are inside the toolbar that is hidden when the standard composer is focused, and the attachment row is still a capsule filename strip without a persistent add-more tile. The PR is open, draft, unmerged, and directly targets the missing UI behavior; the author also marked it draft pending rebase and real-device validation rather than abandoning it.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to rebase it onto current main, rebuild the iOS app, validate focused composer add-via-plus-tile, remove, and send flows on a physical iPhone or representative simulator, attach screenshots, and then have maintainers decide the layout polish and whether the duplicated picker `.onChange` belongs on the tile or an always-present parent. Keep #73710 separate because it owns resize/EXIF processing rather than this composer UI affordance.
