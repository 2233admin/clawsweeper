## Summary

Keep this PR open. Current main still has the old chat composer attachment UI: the PhotosPicker is only rendered inside the toolbar, the toolbar is hidden when the standard iOS composer is focused/compacted, and the existing attachment strip is still the capsule/file-name row without a persistent trailing add-more tile. The PR remains the active implementation candidate, with Greptile's focused-state review finding addressed in the latest head commit.

## Best Possible Solution

Keep the PR open as the focused implementation candidate. Maintainers should review the single-file SwiftUI change, validate the focused iOS composer on device or simulator, and decide whether the mirrored onChange handler is acceptable or should be centralized on an always-present view. Keep related #73710 separate because it handles image processing, not this composer UI affordance.
