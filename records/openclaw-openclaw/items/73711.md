## Summary

Keep this PR open. Current main still has the old shared chat composer attachment UI: the PhotosPicker is only in the toolbar, the focused iOS state still hides that toolbar, and the attachment strip is still capsule-style rows without a persistent trailing add-more tile. The PR remains the focused implementation candidate, with the remaining bar being maintainer review/device validation rather than cleanup closure.

## Best Possible Solution

Review this PR as the active implementation candidate for the shared OpenClawChatUI composer. Before landing, validate the focused iOS composer path on device or simulator and either prove the dual `.onChange` path cannot double-load attachments or centralize the PhotosPicker loader on an always-present view. The related image-processing PR #73710 should stay separate; this PR is the UI affordance work.
