## Summary

Keep this PR open. Current main still lacks the thumbnail strip and persistent add-more attachment affordance, so the branch is not obsolete, but the latest fetched head still has the duplicate shared `PhotosPicker` observer bug that prior review discussion identified as unsafe before merge.

## What This Changes

The PR changes the shared SwiftUI chat composer attachment row from filename capsules to image thumbnail tiles with remove overlays and a trailing add-more picker tile.

## Best Possible Solution

Keep this PR as the canonical implementation candidate, but update the branch so `pickerItems` is observed exactly once from an always-mounted composer-level view with an empty-selection guard, then validate expanded and focused iPhone add-more, multi-select, remove, and send flows without duplicate attachments.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on the latest PR head: after at least one attachment exists and the expanded composer mounts both picker views, one picker selection changes shared `pickerItems` and both observers can run before the async loader clears the state.

## Solution Assessment

No. The feature direction is appropriate, but the current implementation is not the narrowest maintainable fix because duplicated per-picker observers can duplicate attachments; a single stable observer is safer.
