## Summary

Keep this issue open. The report is concrete, current, and not implemented on main: the source tree has no locale path-preservation handler, the docs publish pipeline owns generated locale navigation, and live docs still do not serve standalone root docs assets like the attempted PRs relied on. The prior PR closure also explicitly says the user-facing bug may still be real and points to a docs publish/config path or proven Mintlify-supported custom-script integration, so this issue is a useful tracker rather than stale, duplicate, or not actionable.

## Best Possible Solution

Keep this issue as the canonical tracker. The best path is for maintainers to decide between an upstream Mintlify report and a repo-owned publish/config integration; if OpenClaw implements a workaround, wire it through the docs publish pipeline, derive or generate locale mappings from `GENERATED_LOCALES`, and verify the behavior in a Mintlify preview or live deployment before closing.
