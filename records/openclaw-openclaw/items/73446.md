## Summary

Keep open. Current main still lacks a durable path-preserving locale switcher or a verified Mintlify custom-script deployment hook, and the prior standalone-script PRs were closed unmerged because that hook was not proven in this repo's publish pipeline.

## What This Changes

The issue asks for the Mintlify docs language selector to preserve the current page path when switching locales, such as `/install/docker` to `/zh-CN/install/docker`.

## Best Possible Solution

Keep this as the canonical tracker until maintainers choose the boundary. If OpenClaw owns a workaround, it should use a Mintlify-supported deployment hook, derive locale mapping from `GENERATED_LOCALES` or generated publish config, and be verified in a Mintlify preview or live docs deployment; otherwise the right end state is an upstream Mintlify bug report for path-preserving language switching.
