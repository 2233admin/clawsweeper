## Summary

Keep this issue open. Current main still has no repo-owned locale path-preservation implementation, the attempted standalone JS approach was closed unmerged because it is not wired into Mintlify deployment, and the live docs still do not serve root docs JS/CSS assets directly. The issue is recent, concrete, contributor-authored, has no protected labels, and no open PR currently appears to close it.

## Best Possible Solution

Keep this as the canonical tracker. Maintainers should either report/follow up with Mintlify for path-preserving language switching, or implement a repo-owned workaround through the docs publish/config path so locale mappings are derived from `GENERATED_LOCALES` and verified in a Mintlify preview or live deployment before closing.
