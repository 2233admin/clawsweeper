## Summary

Current main still does not implement the requested Gateway-owned per-session activity API or WebSocket statechange contract. The tree has adjacent channel/account busy counters, active-run abort tracking, task/subagent status, and session change/message events, but no `sessions.activity.get/list`, no `sessions.activity.statechange`, no exported activity schemas/types, no explicit attention state model, no stale-run counter on a session activity payload, and no bounded child rollup API. Related PRs #39438 and #41399 remain useful prior art, but both are closed and unmerged, so this issue should stay open as the canonical core Gateway API feature.

## Best Possible Solution

Keep this issue open as the canonical core Gateway API feature. The best path is an additive `sessions.activity.get`, `sessions.activity.list`, and `sessions.activity.statechange` contract with exported schemas/types, `operator.read` authorization, run-id-set based activity tracking, stale-run cleanup, explicit attention transitions, bounded cross-agent child rollups, reconnect resync semantics, public Gateway protocol docs, and focused Gateway tests. Use #39438 and #41399 as prior art, not as shipped or superseding work.
