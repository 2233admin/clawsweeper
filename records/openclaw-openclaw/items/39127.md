## Summary

Current main still lacks the requested Gateway-owned per-session activity RPCs and WebSocket statechange contract. Adjacent channel busy counters, chat run tracking, subagent child metadata, and background task records exist, but they do not provide `sessions.activity.get/list`, explicit attention states, monotonic per-session versions, stale-run accounting, or a bounded activity child rollup. Closed unmerged PRs #39438 and #41399 remain useful prior art, not shipped fixes.

## Best Possible Solution

Keep this issue open as the canonical Gateway API feature. The best path is an additive `sessions.activity.get`, `sessions.activity.list`, and `sessions.activity.statechange` contract with exported schemas/types, `operator.read` authorization, run-id-set based activity tracking, stale-run cleanup, explicit attention transitions, bounded cross-agent child rollups, reconnect resync semantics, public Gateway protocol docs, and focused Gateway tests. Use #39438 and #41399 as prior art rather than treating them as shipped or superseding work.
