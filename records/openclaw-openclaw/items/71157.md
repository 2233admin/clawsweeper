## Summary

Keep this issue open. Current main still has no WhatsApp disappearing-message expiration config, no resolved account value, and no Baileys `ephemeralExpiration` forwarding in the text/media/voice-note or poll send paths. The direct implementation attempts in #73080 and #73083 are useful context, but both are closed and unmerged, and I found no open PR carrying this work.

## What This Changes

Add a WhatsApp channel/account config option that applies a default disappearing-message expiration to outbound replies and forwards it through Baileys send options.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin as a backward-compatible opt-in: add channel-level and per-account `disappearingMessagesSeconds`, validate it in the strict config schema and generated metadata/help/docs, resolve a positive effective value with account override precedence, and pass it as Baileys `ephemeralExpiration` for all outbound message-generation paths that can produce an agent reply.
