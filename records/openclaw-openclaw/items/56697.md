## Summary

Keep this PR open. Current main has related validation improvements, but it does not implement the PR’s central request: tool-specific parameter examples on missing-parameter errors, especially for empty `{}` payloads. The PR is still potentially useful, but it needs to be rebased and updated for the current canonical tool schemas before it could land.

## Best Possible Solution

Keep the PR open and ask for a rebase/update: add accurate examples for current canonical schemas, especially edit({"path":"...","edits":[{"oldText":"...","newText":"..."}]}), cover the empty-object read/write/edit failures in tests, and remove or wire any dead acceptedKeys logic. Broader retry-loop prevention should remain tracked separately.
