## Summary

## What This Changes

This PR adds an optional `messages.unscheduledReminderNote` boolean gate around the unscheduled-reminder note, adds a Zod schema field and label, and adds reminder-guard unit tests.

## Best Possible Solution

Settle the reminder-guard UX first, then land one complete implementation: either a fully documented/generated `messages.unscheduledReminderNote` config with runtime wiring, public types, schema/help metadata, docs/changelog, and integration coverage, or the competing delivery-stripping/system-annotation approach that prevents internal reminder text from appearing as agent-authored content.
