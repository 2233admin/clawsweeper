## Summary

Keep open. Current main still has the reported same-model OpenAI Responses replay gap: OpenClaw pins `@mariozechner/pi-ai` 0.70.6, OpenAI-family replay policy disables generic tool-call-id sanitization for Responses APIs, the OpenClaw wrapper only handles reasoning-pair downgrade, and the pi-ai converter still skips tool-call-id normalization for same-model messages before splitting `call_id|item_id` into Responses payload fields.

## What This Changes

Normalize OpenAI Responses and OpenAI Codex Responses replay tool identifiers so `function_call.call_id`, matching `function_call_output.call_id`, and `function_call.id` remain provider-valid after tool use.

## Best Possible Solution

Add a Responses-specific replay identifier normalization step that sanitizes and caps the call id, coerces present item ids into a replay-safe `fc_` shape, rewrites matching tool results to the same normalized call id, and preserves canonical reasoning-paired `call_id|fc_id` replay behavior.
