## Summary

Keep open. Current main still appears to have the reported OpenAI Responses replay gap, and an open closing PR now tracks the narrow fix, so this issue should stay open until that PR lands or is explicitly rejected.

## What This Changes

The issue asks OpenClaw to normalize OpenAI Responses replay tool identifiers so overlong or malformed stored `call_id|item_id` pairs are not replayed as invalid `function_call.id` or `function_call_output.call_id` values.

## Best Possible Solution

Review and land #74850, or an equivalent targeted patch, that adds Responses-specific deterministic id normalization before pi-ai converts replay messages, with regression coverage for overlong `call_id` values and malformed function-call item ids.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture reproduction, and source inspection shows current main plus `@mariozechner/pi-ai@0.70.6` can still pass same-model stored ids through to the Responses payload without the requested normalization.

## Solution Assessment

Yes. A Responses-specific normalizer before pi-ai payload conversion is the narrow maintainable fix; a broad generic sanitizer would be riskier because OpenAI Responses reasoning replay depends on preserving valid `call_id|fc_id` pairs.
