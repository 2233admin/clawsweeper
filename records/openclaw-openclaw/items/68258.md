## Summary

Keep open. Current main still has no top-level auth/auth.profiles reload rule, so auth.profiles.* remains unmatched and still enters the conservative gateway-restart path. The provided discussion also shows independent production confirmation and a same-author open implementation PR, so the maintainer path is to review/land or reject #68257 rather than close this issue early.

## What This Changes

Request that OAuth auth profile config changes, especially auth.profiles.* token-refresh diffs, be treated as dynamic/no-op reload paths instead of gateway restart triggers while preserving gateway.auth.* restart behavior.

## Best Possible Solution

Add a scoped reload-planner rule and regression tests so auth.profiles.* OAuth/profile metadata updates are classified as dynamic/no-op config changes, while gateway.auth.* continues to require gateway restart. #68257 is the immediate source PR to review for that end state.
