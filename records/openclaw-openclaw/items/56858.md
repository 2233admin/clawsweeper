## Summary

Keep this PR open. Current main still strips pruned image blocks to text markers and does not implement the PR’s opt-in `cacheMedia` setting, disk cache writer, `[media cached: ...]` marker handling, or image rehydration path.

## Best Possible Solution

Keep the PR open as the current implementation candidate. The best path is to rebase/port it onto `src/agents/pi-hooks`, preserve the opt-in default-off behavior, add focused tests for multi-image and soft-trim plus hard-clear cases, update generated config/docs, and complete maintainer security review of the disk-cache and marker-read path.
