## Summary

Keep open. Current main still uses strict trimmed equality for `isAnnounceSkip`, so the reported multi-line final-token case remains unfixed, and open PR #74136 explicitly closes this issue and should be reviewed rather than bypassed by an autoclose or competing repair job.

## What This Changes

The issue asks for `isAnnounceSkip` to suppress sub-agent announce output when a multi-line reply ends with `ANNOUNCE_SKIP` on its own final line.

## Best Possible Solution

Land PR #74136 or an equivalent narrow patch so `isAnnounceSkip` accepts the canonical exact token plus a standalone final-line `ANNOUNCE_SKIP`, while preserving `REPLY_SKIP` and `NO_REPLY` semantics and adding focused helper plus announce-flow regression coverage.
