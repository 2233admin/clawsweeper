## Summary

Keep open. Current main still implements `isAnnounceSkip` as strict trimmed equality, so the reported multi-line final-line `ANNOUNCE_SKIP` shape remains reproducible; the issue is also paired with open PR #74136, which specifically targets this behavior and should be reviewed before this issue closes.

## What This Changes

The request is to make announce suppression recognize multi-line sub-agent output whose standalone final line is `ANNOUNCE_SKIP`, instead of only accepting output that is exactly the token.

## Best Possible Solution

Review and land PR #74136, or an equivalent narrow patch, so `isAnnounceSkip` accepts the exact token plus a standalone final-line `ANNOUNCE_SKIP` while preserving `REPLY_SKIP` and `NO_REPLY` semantics; include focused helper coverage, announce-flow regression coverage, and docs or prompt updates if the contract changes.

## Reproduction Assessment

Yes. The current helper can be reproduced with a direct predicate check: exact `ANNOUNCE_SKIP` returns true, while multi-line text ending in `ANNOUNCE_SKIP` returns false, and both announce capture and delivery paths depend on that helper.

## Solution Assessment

Yes, a narrow helper-level change with focused tests is the best maintainable direction. PR #74136 appears to target that route, but maintainers should verify the final-line matching semantics and decide whether docs and prompt wording should also change.
