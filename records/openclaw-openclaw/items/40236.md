## Summary

Keep open. Current main has adjacent cron safeguards and plugin-level cron lifecycle observability, but it still lacks the requested persisted job-level lifecycle contract for timeout, automatic cleanup by run/time/status, and per-job start/complete/failure hooks. The related open PR #45465 may address part of the hook story, but it is unmerged and does not cover the timeout and cleanup portions.

## What This Changes

The issue requests cron job-level lifecycle support, including configurable job timeout, retry/automatic cleanup policy, and start/complete/failure hooks.

## Best Possible Solution

Keep this as a cron product/API design item. A complete solution should add an explicit, additive lifecycle contract across the Gateway protocol, stored job normalization/types, scheduler timeout and cleanup behavior, CLI/UI/docs, tests, and security boundaries while preserving existing payload.timeoutSeconds, deleteAfterRun, retry/backoff, failureAlert, session retention, run-log pruning, and plugin cron_changed behavior.
