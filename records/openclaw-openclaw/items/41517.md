## Summary

Keep this PR open. Current main still does not implement the intended resolver behavior for `provider="google"` with `modelId="google/gemini-2.5-pro"` against a bare configured Google model, and the overlapping #41398 remains open rather than shipped.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to land one narrow fix, either here or in #41398, that strips a duplicated active-provider prefix only when it is truly redundant, preserves OpenRouter and nested provider-native ids such as NVIDIA catalog ids, adds the Google inline/configured resolver regression test, and then closes the competing implementation after the chosen fix ships.
