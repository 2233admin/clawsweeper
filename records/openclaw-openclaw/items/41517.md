## Summary

Keep this PR open. Current main still does not implement the PR’s intended resolver-level fix for a Google same-provider model ref such as provider `google` with model ID `google/gemini-2.5-pro`, and the PR is a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and request a current-main revision. The best fix is a narrow resolver-level normalization that strips a duplicated same-provider prefix only when it resolves to the intended configured or discovered model for that provider, while preserving OpenRouter pass-through and provider-native nested IDs. Add regression coverage for Google same-provider inline/configured resolution, OpenRouter pass-through, and at least one provider-native nested ID case.
