## Summary

Keep this PR open. Current main still does not implement the intended resolver-level fix for `provider=google` with `modelId=google/gemini-2.5-pro`, and the related Google-prefix PR #41398 is also open/unmerged rather than a shipped canonical replacement. This PR is a focused implementation candidate, but it needs a current-main revision and maintainer review before merge.

## Best Possible Solution

Keep this PR open and ask for a current-main revision. The best fix is a narrow Google-owned or resolver-level normalization path that strips duplicated `google/` only when it is truly a provider/model duplication, preserves pass-through and provider-native nested IDs, and adds regression coverage for Google inline/configured resolution plus nested-ID preservation. Review it alongside #41398 and close the unlanded duplicate only after one implementation is merged.
