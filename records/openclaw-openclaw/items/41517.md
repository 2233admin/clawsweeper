## Summary

Keep this PR open. Current main at 5de284c2e30f915b769fac08dbeab45f57749702 still does not implement the PR’s intended resolver-level fix for `provider='google'` with `modelId='google/gemini-2.5-pro'`. The PR is not safe to merge as-is because the generic same-provider stripping approach can affect provider-native nested model IDs, but it remains a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and request a current-main revision. The best fix is narrow provider-owned or resolver-level normalization that strips duplicated Google same-provider refs only when safe, preserves OpenRouter pass-through and provider-native nested IDs, and adds regression coverage for Google inline/configured resolution plus at least one nested-ID preservation case. Maintainers should compare this with open PR #41398 and land one clean, scoped fix before closing the other as superseded.
