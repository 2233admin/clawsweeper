## Summary

Keep open. Current main still lacks explicit SIGHUP handling and bounded shutdown for the TUI exit path, and an open unmerged PR is already linked with closing syntax as the active implementation candidate.

## What This Changes

The issue reports that `openclaw tui --session ...` can survive terminal closure or gateway restart, peg CPU, resist SIGTERM, and consume memory until killed.

## Best Possible Solution

Keep this issue open while the narrow TUI lifecycle fix is reviewed: handle SIGHUP, make shutdown idempotent and bounded, stop loader/status timers, and tolerate closed terminal streams without changing gateway reconnect policy. If PR #75294 lands, this issue can close through that merged implementation.

## Reproduction Assessment

Yes. The report gives concrete macOS reproduction steps, and current main has matching source evidence: no SIGHUP listener, a then-only drain/stop exit path, and active TUI status timers not explicitly stopped on exit.

## Solution Assessment

Yes. The best path is to review and land PR #75294 or an equivalent narrow TUI shutdown patch; keeping the issue open until that merge preserves the GitHub closing workflow.
