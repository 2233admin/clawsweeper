## Summary

Keep open. Current main still matches the reported TUI shutdown failure surface: `runTui` handles SIGINT/SIGTERM but not SIGHUP, exit completion depends on terminal drain/stop resolving, and status loader/timer cleanup is not tied to the signal-exit path. PR #75294 is open, unmerged, and uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for `openclaw tui` to exit cleanly on terminal hangup, SIGTERM, and gateway restart without leaving high-CPU, high-RSS orphan processes.

## Best Possible Solution

Keep this issue as the canonical tracker for the terminal-hangup/SIGTERM orphan path, then close it through PR #75294 or an equivalent narrow TUI lifecycle patch that handles SIGHUP, bounds exit cleanup, stops status UI timers, and tolerates closed terminal streams.

## Reproduction Assessment

Yes. The issue provides concrete macOS reproduction steps, and current main has matching source evidence: no SIGHUP listener, a then-only drain/stop exit path, and status loader timers outside signal-exit cleanup.

## Solution Assessment

Yes. The narrow maintainable path is to review and land PR #75294 after addressing its remaining shutdown race, rather than closing this issue before the linked implementation merges.
