## Summary

Keep open. Current main has adjacent hardening for Codex route selection, session harness pinning, stale status totals, cumulative usage projection, and OAuth/API-key isolation, but it still does not prove the reported route-switch regression is fixed end to end. The remaining gap is concrete: native Codex compaction completion still does not return a parsed post-compaction token count, and no current-main test or live trace covers the full switch from PI/OAuth to native Codex with auth state and context isolation together.

## What This Changes

The issue reports that switching an agent from `openai-codex/gpt-5.4` to native `codex/gpt-5.4` can trigger runaway context growth, off-task workspace contamination, and Codex auth-mode drift instead of failing clearly or preserving stable session state.

## Best Possible Solution

Keep this as a focused child under #66251 until maintainers capture or add a current-main route-switch proof. The desired end state is a clean-session switch path with stable harness/thread binding, isolated OAuth/API-key selection, and status that shows fresh context usage or explicitly unknown usage after native compaction.
