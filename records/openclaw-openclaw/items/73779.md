## Summary

Keep this PR open. Current main still performs the eager session reload, so the PR is not obsolete, but the submitted patch needs changes because it defers a call that existing tests still assert synchronously and it adds a pending timer without teardown.

## What This Changes

The branch adds Control UI session reload debounce state and replaces the immediate `sessions.list` reconciliation after `sessions.changed` with a 5-second timeout.

## Best Possible Solution

Land a repaired Control UI-only debounce that keeps the incremental `sessions.changed` update immediate, coalesces only the full `sessions.list` reconciliation, clears any pending timer during teardown, and covers delayed/coalesced reload behavior with focused fake-timer tests.

## Reproduction Assessment

Yes. The source check is high-confidence: current main calls `loadSessions` immediately on `sessions.changed`, and the PR defers it with `setTimeout` while the existing unit test still asserts the immediate call.

## Solution Assessment

No, not as submitted. The debounce direction is narrow and maintainable, but the best implementation needs lifecycle cleanup and updated timer tests before merge.
