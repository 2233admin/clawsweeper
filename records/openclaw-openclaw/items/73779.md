## Summary

Keep this PR open. Current main still handles `sessions.changed` by applying the incremental update and immediately calling `loadSessions`; no debounce timer or teardown path exists. The PR is a plausible focused optimization, but the submitted diff still needs fake-timer/coalescing test updates and timer cleanup before it is merge-ready.

## Best Possible Solution

Keep the PR open and refine it: add a named debounce constant for full `sessions.list` reconciliation after `sessions.changed`, keep the immediate `applySessionsChangedEvent` update, add fake-timer tests for delay and coalescing, initialize the timer in the test host, and clear the pending timer in Control UI lifecycle cleanup before merge.
