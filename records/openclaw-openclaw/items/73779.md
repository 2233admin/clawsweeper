## Summary

Keep this PR open. Current main still reloads `sessions.list` immediately for every `sessions.changed` event, and the existing unit test explicitly asserts that immediate reload behavior. The PR is a focused UI change, not implemented on main, and the supplied related items do not supersede the same `sessions.changed` reload-throttling work.

## Best Possible Solution

Review and refine this PR rather than closing it: implement the debounced `sessions.list` reconciliation in the Control UI event path, update `app-gateway.sessions.node.test.ts` to assert delayed/coalesced reload behavior, and handle timer cleanup in the UI lifecycle or reconnect path if the timer is stored on the app host.
