## Summary

Keep open. Current main still only provides manual schema printing and preserves an existing root `$schema`; it does not automatically create or refresh `openclaw_schema.json`, default a missing schema reference, or expose the requested update opt-out. The issue is also paired with open PR #55390, so it should remain open until that implementation candidate is reviewed, landed, replaced, or closed.

## What This Changes

Request automatic generation and refresh of a local `openclaw_schema.json` beside `openclaw.json`, defaulting missing `$schema` values to it and adding an update opt-out plus docs and tests.

## Best Possible Solution

A narrow implementation should hook the existing config/startup/update flow to idempotently maintain `openclaw_schema.json`, add the requested schema-version metadata and `update.maintain_config_json_schema` default, preserve custom schema URIs, and document/test the opt-out and no-op rerun behavior.
