## Summary

Keep this PR open. Current main still validates cron announce channels only, not explicit announce delivery account IDs, so the intended bug fix is not implemented. The PR is plausible for #69513, but it is stale against current main and needs maintainer review around the gateway hot path and plugin-cache boundary before landing or replacement.

## What This Changes

This PR adds gateway cron.add/cron.update validation for explicit announce delivery and failure-destination account IDs, plus gateway integration tests for invalid, default, normalized, and inferred-channel account cases.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase/port the account validation onto current main, validate explicit `delivery.accountId` and `delivery.failureDestination.accountId` for explicit resolved announce channels, preserve omitted-channel/`last`, normalized ID, default/unlisted account behavior, and use a lightweight configured-account seam instead of broad plugin runtime loading in the gateway hot path. Close #69513 only after the implementation lands.
