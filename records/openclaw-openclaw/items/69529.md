## Summary

Keep #69529 open. Current main has not implemented the central cross-provider account-id fix, the gateway allowlist `source` schema change, or the two skill description updates. The PR is still potentially useful, but it should get maintainer review/rework because it bundles unrelated fixes and the runner-only account suppression appears incomplete for the real message-tool path that pre-populates `params.accountId` before calling `runMessageAction`.

## Best Possible Solution

Keep the item open, but rework or split it. Land #69525 as a focused fix that suppresses the source session default for cross-provider sends in both `createMessageTool` and `runMessageAction`, while preserving a truly user-provided target `accountId`, and add a regression test that exercises the message tool entry point plus the target plugin context. Handle the gateway `source` schema/protocol test separately for #69482, and keep the skill description cleanup tied to #69475 or a separate skill-only PR.
