## Summary

Keep open. Current main still contains the reported early return in the gateway chat lifecycle finalizer, the registry contract still permits `shift()` to miss after `peek()`, and the linked fix PR #74678 is closed without being merged.

## What This Changes

A terminal lifecycle event should emit a final chat event via the already-resolved session/run fallback when the chat run registry entry disappears between `peek()` and `shift()`.

## Best Possible Solution

Update the gateway lifecycle finalizer so a `shift()` miss after a successful `peek()` falls back to the already-resolved `sessionKey` and `eventRunId`, then add a focused regression test for that branch and leave broader missing-terminal-event timeout symptoms to their own tracking unless separately proven fixed.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to enter `finalizeLifecycleEvent` with `chatLink` truthy and make `chatRunState.registry.shift(evt.runId)` return `undefined`; current main then returns before emitting the terminal `chat` event.

## Solution Assessment

Yes. The narrow gateway fallback is the best current direction because it fixes the producer-side suppressed final event instead of adding another Control UI or TUI workaround after the event has already been dropped.
