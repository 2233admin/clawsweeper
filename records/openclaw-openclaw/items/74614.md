## Summary

Keep open. Current `main` still contains the reported early return, the registry API can return `undefined` from `shift()`, and the linked fix PR #74678 is closed without merge, so no cleanup close reason applies.

## What This Changes

The issue reports that gateway chat lifecycle finalization can skip the terminal `chat` event when `peek()` finds a chat run link but `shift()` later misses.

## Best Possible Solution

Make the gateway lifecycle finalizer fall back to the already-resolved `sessionKey` and `eventRunId` when `registry.shift(evt.runId)` misses, then add a focused gateway regression test for that exact branch while leaving broader timeout/no-terminal-event symptoms to their own tracking.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to enter `finalizeLifecycleEvent` with a truthy `chatLink` and force `chatRunState.registry.shift(evt.runId)` to return `undefined`; current `main` then returns before emitting the terminal `chat` event.

## Solution Assessment

Yes. The narrow producer-side fallback is the best current direction because it restores the missing terminal event at the gateway instead of adding a client workaround after finalization has already been suppressed.
