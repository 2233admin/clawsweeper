## Summary

Keep open. Current main has adjacent reliability cleanup for terminal lifecycle and stale lane release, but it still does not fully satisfy this report: active embedded/reply/lane work remains warning-only by default, the diagnostic heartbeat does not opt into active abort, and normal Discord direct pre-reply failures still use generic failure copy rather than a timeout/partial-success state.

## What This Changes

The issue asks OpenClaw to make Discord-routed embedded-agent timeouts unambiguous by marking, rotating, or failing the affected session and distinguishing possible partial-success late failures from total failure.

## Best Possible Solution

Add a shared terminal timeout/session-health transition for embedded agent failures that marks or rotates the affected session before later routing, plus channel-appropriate timeout copy that warns previous side effects may have completed. Keep the fix shared across channels and avoid a Discord-only workaround unless the chosen policy explicitly requires channel-specific presentation.

## Reproduction Assessment

Yes for source-level reproduction: current tests still assert generic Discord direct failure copy, and diagnostic stuck-session recovery still skips active runs by default. A live Discord plus Claude CLI timeout would strengthen final policy validation, but the current-main gap is visible without mutating the checkout.

## Solution Assessment

Unclear because no patch is proposed and the remaining change depends on maintainer policy. The narrow maintainable direction is a shared session-health transition plus explicit timeout/partial-success copy after deciding whether active side-effectful runs should be aborted, rotated, or marked dead.
