## Summary

Keep open. Current main and the latest shipped tag still resolve Docker sandbox pidsLimit only from agent/global config, and Docker create args only emit --pids-limit when that value is explicitly positive. The issue is also paired with open PR #44098, which explicitly proposes the same default-limit fix, so this should stay open until that PR or an equivalent maintainer-approved security fix lands and ships.

## What This Changes

Request adding a default Docker sandbox pidsLimit and documentation guidance so sandbox containers are not left with unbounded process creation when users leave pidsLimit unset.

## Best Possible Solution

Keep this issue open and review/land PR #44098 or an equivalent security-owned fix. The fix should add a maintainer-approved default in resolveSandboxDockerConfig, preserve explicit global and per-agent overrides, prove buildSandboxCreateArgs emits the resolved default, update public sandbox/config/security docs if runtime behavior changes, and close this issue only after the fix merges and ships.
