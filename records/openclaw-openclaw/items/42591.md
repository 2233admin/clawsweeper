## Summary

Keep open. Current main still has the requested installer responsibilities in one large public Bash installer, and the docs, tests, and CI smoke coverage still treat `scripts/install.sh` and `https://openclaw.ai/install.sh` as the stable surface. The request is valid, but it is a broad public installer refactor that needs maintainer agreement on module boundaries, sourcing/fetching strategy, compatibility, and release validation before it is safe to implement.

## What This Changes

Request to split the large public Bash installer into a `scripts/install/` module tree with separate UI, download, platform, npm, git, and build-tool helpers.

## Best Possible Solution

Keep `scripts/install.sh` and the public `https://openclaw.ai/install.sh` entrypoint stable, then extract narrowly scoped sourced helpers behind that wrapper with matching installer docs, unit coverage, Docker smoke coverage, and release-validation updates after maintainers agree on the helper layout and delivery model.
