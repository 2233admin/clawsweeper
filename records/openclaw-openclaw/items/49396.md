## Summary

Keep this PR open. Current main still computes Peak Error Hours by proportionally spreading each session's total errors and messages across time slices, and the backend usage summary does not expose precise hourly or quarter-hour message buckets. The PR is not obsolete, and the discussion shows active maintainer feedback followed by an amended contributor response.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent fix that adds backend per-bucket message/error counts, uses them in buildPeakErrorHours for Local and UTC views, preserves the legacy fallback, and keeps the new backend/frontend regression tests green.
