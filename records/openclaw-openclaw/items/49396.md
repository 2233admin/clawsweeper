## Summary

Keep this PR open. Current `main` still computes Peak Error Hours by proportionally allocating each session's session-wide message and error totals across hour slices, and the backend usage summary still has no precise hourly or quarter-hour message-count contract. The PR remains an active, focused implementation candidate with maintainer feedback already addressed by the contributor.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent fix that adds backend per-bucket message/error counts, uses them in `buildPeakErrorHours` for both Local and UTC views, preserves the legacy fallback for older payloads, keeps the backend/frontend regression tests, and confirms the changed gate before merge.
