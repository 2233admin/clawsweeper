## Summary

Keep #58800 open. Current main still implements Slack custom identity for normal sends, but the edit paths involved in the linked regression still drop `username` and avatar fields: `editSlackMessage` has no identity option, draft-stream edits do not forward identity, preview finalization does not accept identity, and the `status_final` completion path still uses a raw `chat.update`. The PR is a focused fix for real remaining Slack behavior, though the submitted diff needs maintainer follow-up for the bot-review findings before merge.

## Best Possible Solution

Keep the PR or linked bug open and ask for a refreshed Slack fix: reuse the existing `SlackSendIdentity` shape for edits, add best-effort `chat.update` retry without custom identity on missing `chat:write.customize`, thread resolved `slackIdentity` through draft stream, preview finalization, and `status_final` update paths, and add focused regression tests for send/edit identity propagation and fallback behavior.
