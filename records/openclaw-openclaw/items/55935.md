## Summary

Keep open. Current main still formats heartbeat/cron-style `Current time:` with the locale-style `formatUserTime(...)` string plus a separate non-ISO UTC suffix, and heartbeat still passes the heartbeat runner's `startedAt` into that line. The related implementation PR #55957 was closed unmerged, and the latest release tag still contains the same formatter.

## Best Possible Solution

Keep this issue open and implement the timestamp contract in the shared cron-style time helper: emit an ISO 8601 local timestamp with offset for `Current time`, decide whether heartbeat should also include a distinct `Heartbeat triggered at` value from `startedAt`, and update the heartbeat/cron/reset/compaction tests plus timestamp double-stamp detection/docs together. The open related temporal-grounding issue #63181 is adjacent, but it does not supersede this heartbeat/cron formatting request.
