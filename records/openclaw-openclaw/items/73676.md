## Summary

Keep open. Current main still has unguarded startup and dotenv `process.cwd()` call sites that can throw when the working directory has been deleted, and PR #74994 is an open closing implementation candidate rather than a merged fix.

## What This Changes

The issue asks OpenClaw CLI startup to tolerate a deleted current working directory instead of crashing with `ENOENT: no such file or directory, uv_cwd`.

## Best Possible Solution

Review and land PR #74994 or an equivalent shared safe-cwd startup fix that covers main-module detection, CLI/runtime dotenv loading, and required home-dir fallbacks, then close this issue after merge.

## Reproduction Assessment

Yes. The reporter's create/cd/delete/run path maps directly to current startup code that still evaluates raw `process.cwd()` before command-specific recovery.

## Solution Assessment

Yes, the linked shared-helper direction is the narrowest maintainable fix shape. Closing this issue before that implementation merges would leave the reported current-main behavior unresolved.
