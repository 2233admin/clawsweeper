## Summary

Keep open. The PR targets a real open report, but the patch conflicts with current tested and documented memory-core behavior for daily-ingestion and grounded backfill promotion evidence, so it needs maintainer policy review before merge.

## What This Changes

The PR changes memory-core short-term promotion to compare `minRecallCount` against `recallCount` alone in both candidate ranking and apply filtering, while leaving aggregate `signalCount` for scoring.

## Best Possible Solution

Define the memory-core promotion policy explicitly: real user recalls should be separated from automated daily/session signals where needed, but grounded backfill and dreaming-only promotion should either keep a supported threshold path or be deliberately retired with matching tests and docs.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: with this PR, the existing daily-only and grounded-only promotion tests would no longer pass because those candidates have `recallCount: 0` and now fail the default `minRecallCount` gate before ranking or applying.

## Solution Assessment

No, not as-is. The recall-only gate is a narrow fix for #72021, but the best maintainable solution must also reconcile daily/grounded promotion semantics and update the affected tests/docs in the same change.
