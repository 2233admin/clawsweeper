## Summary

Keep this PR open. Current main has an overlapping fix for metadata-only exec completion noise, but it does not fully implement this PR's no-op sentinel handling for heartbeat delivery, especially exact `NO_REPLY`, `NO_NEW_AUDIO`, and `SESSION_WATCHDOG_OK` replies. The branch should be rebased and reviewed against the newer heartbeat exec-completion logic rather than closed as implemented.

## What This Changes

This PR adds shared no-op sentinel reply detection, applies it during heartbeat reply normalization for heartbeat and system-event outputs, and adds heartbeat-runner regression tests for silent sentinels and exec system-event replies.

## Best Possible Solution

Rebase this PR onto current main and keep the narrow heartbeat-level sentinel suppression, with tests for exact no-op tokens, exact exec `HEARTBEAT_OK`, and meaningful exec summaries. During conflict resolution, preserve the current `hasRelayableExecCompletion` distinction so metadata-only successful exec completions stay silent without hiding relayable command output.

## Reproduction Assessment

Yes. Current main's heartbeat reply normalization can be traced to show exact non-heartbeat sentinels are not handled before outbound delivery, and the PR adds focused `runHeartbeatOnce` tests that exercise the leak and the intended no-send behavior.

## Solution Assessment

Yes, with rebase. A centralized sentinel helper used before heartbeat token stripping is a narrow maintainable fix, but it should be integrated with current main's newer relayable-exec gating rather than replacing it with the older exec-completion exception model.
