Keep this PR open. Current main still lacks the PR's centralized no-op sentinel detection and still has the exec-completion fallback that can re-deliver an exact HEARTBEAT_OK reply, so the linked no-op leak remains until this PR or an equivalent fix lands.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73149. The best path is to review and land the focused sentinel-suppression change, with targeted coverage that exact no-op tokens are suppressed before outbound delivery for heartbeat/system-event handoffs while meaningful exec summaries still deliver.
