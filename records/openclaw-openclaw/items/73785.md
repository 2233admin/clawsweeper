## Summary

Keep this PR open. Current main does not implement the PR’s intended no-op sentinel suppression for heartbeat/system-event replies: `HEARTBEAT_OK` is still forced visible for exec-completion events, `NO_NEW_AUDIO` and `SESSION_WATCHDOG_OK` are not recognized centrally, and the proposed regression tests are not present.

## Best Possible Solution

Keep the PR open as an active implementation candidate. The best path is maintainer review of the focused sentinel-suppression approach, ensuring exact no-op tokens are suppressed before outbound delivery for heartbeat/system-event handoffs while preserving meaningful exec summaries, then landing with the added regression coverage and normal targeted gates.
