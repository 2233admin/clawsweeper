## Summary

Keep this PR open, but do not merge it as-is. The exact sentinel suppression direction is useful and matches the linked no-op leak, but the patch also changes all system-event heartbeat-token stripping to message mode, which can re-expose short no-op summaries ending in HEARTBEAT_OK that current heartbeat-mode suppression would skip.

## What This Changes

The PR adds centralized exact no-op sentinel detection for heartbeat/system-event replies, suppresses NO_REPLY, NO_NEW_AUDIO, SESSION_WATCHDOG_OK, and HEARTBEAT_OK in selected paths, and adds heartbeat-runner regression tests for silent sentinels and exec system events.

## Best Possible Solution

Land a narrow heartbeat/system-event delivery fix that centralizes exact no-op sentinel detection, keeps current heartbeat-mode suppression for short no-op summaries, and only preserves system-event summaries when they contain genuinely user-facing exec or cron content.

## Reproduction Assessment

Yes. A focused heartbeat-runner test can enqueue a cron/system event, mock the reply as a short no-op summary ending in HEARTBEAT_OK, and assert no outbound send; by code inspection this patch would strip the token in message mode and deliver the remaining summary.

## Solution Assessment

No. The exact sentinel detector is the right local seam, but changing all system events to message-mode stripping is too broad because it bypasses the existing ackMaxChars no-op suppression for short system-event summaries.
