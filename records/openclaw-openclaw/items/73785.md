## Summary

Keep open. Current main still lacks the PR's centralized no-op sentinel helper and still has the exec-completion fallback that can re-deliver an exact HEARTBEAT_OK from a system-event heartbeat handoff. The PR is a focused implementation candidate for #73149, with no security-sensitive diff surface found in the current patch.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent focused fix. The best path is to suppress exact runtime no-op sentinels for heartbeat/system-event delivery while preserving meaningful exec summaries, then leave any broader first-class no-reply contract for #73149 or a narrower follow-up if maintainers want that larger API.
