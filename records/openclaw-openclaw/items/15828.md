## Summary

Keep open. Current main still keeps bundled session-memory persistence tied to explicit `/new` and `/reset` command events; adjacent idle/daily rollover and gateway shutdown hook infrastructure exists, but it does not implement the requested idle, shutdown, message-count checkpoint, trigger-config, or summary-output behavior. The related open PRs are partial daily/idle implementations, not merged fixes for the broader request.

## What This Changes

Request automatic bundled session-memory checkpoints on idle, shutdown, periodic message-count, and explicit command triggers, with configurable trigger settings and optional concise LLM summaries.

## Best Possible Solution

Keep this as the broad session-memory automation tracker. The end state should extend the bundled hook or a documented replacement path with explicit trigger config, shutdown-safe checkpointing, message-count checkpoints, summary-output semantics, docs, and focused regression tests while preserving existing `/new` and `/reset` behavior.
