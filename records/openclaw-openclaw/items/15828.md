## Summary

Keep this issue open. Current main has some adjacent infrastructure, including gateway shutdown hook events and plugin session lifecycle hooks for stale rollovers, but the bundled session-memory hook still only saves on explicit `/new` and `/reset`; the requested idle/shutdown/message-count trigger config and summary-output mode remain unimplemented, with daily/idle only covered by still-open partial PRs.

## What This Changes

The issue requests automatic session-memory checkpoints for idle, shutdown, message-count, and command resets, plus optional LLM summarization instead of raw transcript storage.

## Best Possible Solution

Keep this issue open as the broader session-memory automation tracker. Maintainers should choose or consolidate the daily/idle path from #61675 and #71792, then separately scope shutdown checkpointing, message-count checkpointing, and summary-output config with docs and targeted tests.
