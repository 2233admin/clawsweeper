## Summary

Keep open. Current main still has adjacent session rollover and gateway lifecycle infrastructure, but the bundled session-memory hook remains registered and implemented only for explicit `/new` and `/reset`; idle/shutdown/message-count trigger config and LLM summary-output mode are still absent. The related daily/idle PRs remain open partial implementations, so they are not a merged fix for the broader request.

## What This Changes

The issue requests automatic bundled session-memory checkpoints on idle rollover, gateway shutdown, and periodic message counts, plus optional concise LLM-generated summary output.

## Best Possible Solution

Keep this as the broader session-memory automation tracker. Maintainers should consolidate the daily/idle PR path, then scope the remaining shutdown checkpoint, message-count checkpoint, and summary-output config with docs and targeted tests.
