## Summary

Keep open. Current main has partial mitigations, but it still leaves billing-disabled same-provider fallback chains blocked until near expiry and the configure/plugin auth path still lacks the same stale cooldown clearing added for `models auth login`.

## What This Changes

The issue requests that OpenClaw recover provider auth profiles after billing recovery or successful re-auth instead of persisting long `auth-state.json` cooldown or disabled windows.

## Best Possible Solution

Keep persisted backoff for real provider protection, but treat same-provider shared-profile billing fallback chains as having no effective alternate for recovery probes, and clear provider/profile-scoped stale usage state after successful configure/onboarding auth refresh while preserving unrelated profile history.

## Reproduction Assessment

Yes. The issue provides concrete Anthropic and GitHub Copilot steps, and current source/tests corroborate the remaining billing path without needing live provider credentials: far-from-expiry billing disables with fallbacks are still skipped.

## Solution Assessment

Yes, with a narrower implementation than removing persisted state entirely. Targeted billing probes plus provider-scoped cooldown clearing after successful auth refresh fix recovery while preserving the backoff contract for real repeated failures.
