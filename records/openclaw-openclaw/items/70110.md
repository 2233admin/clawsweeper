## Summary

Current main still has the reported Nextcloud Talk reaction wiring gap: the plugin advertises reactions and has an HTTP reaction sender, but it does not register a message action adapter, so shared `message` tool `react` dispatch returns `null`. Two open PRs, #72348 and #72679, already reference this issue with closing syntax, so the issue should remain open until one lands or an equivalent fix is merged.

## What This Changes

The issue asks for Nextcloud Talk agents to send emoji reactions by wiring the existing reaction sender into the channel action path.

## Best Possible Solution

Land one canonical Nextcloud Talk action-adapter fix, from #72348, #72679, or an equivalent patch, that registers a `ChannelMessageActionAdapter`, dispatches `react` to `sendReactionNextcloudTalk` with the correct account/config/message identifiers, and adds focused regression coverage for discovery and dispatch.
