## Summary

Current main still has the reported Nextcloud Talk reaction wiring gap. The plugin advertises reactions and contains the HTTP sender, but it does not register a channel message action adapter, so shared `message` tool `react` dispatch cannot reach `sendReactionNextcloudTalk`. Two open PRs, #72348 and #72679, are implementation candidates for the same issue, so this should remain open until one lands or an equivalent fix is merged.

## Best Possible Solution

Keep this issue open and review or land #72348, #72679, or an equivalent fix that adds a Nextcloud Talk `ChannelMessageActionAdapter`, registers it from the channel plugin, dispatches `react` to `sendReactionNextcloudTalk`, and adds regression coverage for discovery plus dispatch.
