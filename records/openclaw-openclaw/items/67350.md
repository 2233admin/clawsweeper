## Summary

Keep open. Current main still does not implement the PR's central stability remediation, and the PR head has concrete blocking correctness and privacy-sensitive delivery bugs that need rework before merge.

## What This Changes

The PR ports broad installed-dist stability changes into source across session routing, session-store cleanup, lock payloads, Telegram polling and protected-DM handling, overflow compaction recovery, health/status reporting, SDK exports, and regression tests.

## Best Possible Solution

Rework this PR into a mergeable implementation that keeps the broad stability intent but fixes the health helper, covers account-scoped Telegram direct-session protections, justifies or narrows any new public SDK runtime exports, and preserves focused regression coverage for routing, degraded health, structured locks, Telegram poller liveness, heartbeat/task delivery protection, and overflow compaction provenance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Define the health issue resolver before calling it:** `src/commands/health.ts:353`
  - body: `getHealthSnapshot()` now calls `resolveOperationalHealthIssues(...)`, but that function is not defined or imported in this module. Any health snapshot that reaches an account will throw instead of returning a health payload.
  - confidence: 0.95
- **[P1] Match account-scoped Telegram direct session keys:** `src/tasks/task-registry.ts:95-98`
  - body: This guard only recognizes `agent:<id>:telegram:direct:<peer>`. With `dmScope="per-account-channel-peer"`, the owner key is `agent:<id>:telegram:<account>:direct:<peer>`, so protected ACP completion delivery can bypass the primary-DM protection this patch is adding.
  - confidence: 0.9
- **[P1] Protect account-scoped Telegram heartbeat sessions:** `src/infra/heartbeat-runner.ts:653`
  - body: The heartbeat protection regex has the same accountless-only shape, so account-scoped Telegram direct sessions can skip the protected-primary-DM branch and still relay heartbeat output to a conversation this remediation is meant to block.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR introduces privacy-sensitive Telegram delivery protection logic, but the account-scoped direct-session gaps can allow protected primary-DM relays to bypass the new guard.

Concerns:

- **[high] Account-scoped protected DM bypass:** `src/tasks/task-registry.ts:95`
  - body: The protected Telegram direct-session guard only matches accountless direct keys. Supported multi-account deployments use `agent:<id>:telegram:<account>:direct:<peer>`, so ACP completion delivery can bypass the new fail-closed path.
  - confidence: 0.9

  - confidence: 0.88
