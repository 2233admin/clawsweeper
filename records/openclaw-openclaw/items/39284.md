## Summary

Keep this PR open. Current main partially solves the linked workspace-context problem through automatic bootstrap injection and `lightContext`, but it still does not implement this PR's explicit `sessions_spawn.contextFiles` or `sessions_spawn.inheritContext` API. The proposed diff also has unresolved requester-vs-target workspace semantics and security/resource concerns, so it needs maintainer product review rather than auto-close.

## Best Possible Solution

Keep this PR open for maintainer product judgment. If explicit spawn-time file selection is still desired, rework it on top of current bootstrap plumbing with documented requester-vs-target workspace semantics, bounded `contextFiles` schema, prompt-size limits, same-agent and cross-agent regression tests, docs updates, and removal or separation of the unrelated plugin logger type change. If automatic bootstrap injection plus `lightContext` is the intended product direction, maintainers should close this PR manually with that rationale.
