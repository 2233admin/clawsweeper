## Summary

Keep open. Current main still has the audit/loader mismatch this PR targets, and the proposed patch is a plausible narrow fix rather than obsolete cleanup; because it changes a security audit boundary, it should stay open for maintainer/security review instead of being auto-closed.

## What This Changes

The PR changes the workspace skill symlink audit to compare `SKILL.md` realpaths against the resolved `workspace/skills` directory, updates finding/docs/changelog wording, and adds regression coverage for in-workspace but outside-skills symlink targets.

## Best Possible Solution

Land this narrow fix, or a maintainer-adjusted variant, after confirming the `workspace/skills` root semantics, audit wording, docs copy, changelog credit, and regression test match the intended loader boundary.

## Reproduction Assessment

Yes. The high-confidence reproduction path is a `workspace/skills/linked/SKILL.md` symlink whose real target is inside the workspace but outside `workspace/skills`; the existing loader test rejects that path while current main's audit accepts it because it checks the broader workspace root.

## Solution Assessment

Yes, with maintainer/security confirmation. The proposed source change is the narrowest maintainable fix I found because it aligns the audit containment boundary with the loader's configured root and updates the single check/test/docs surface; a maintainer-adjusted variant would only be needed if audit semantics are intentionally different from loader semantics.
