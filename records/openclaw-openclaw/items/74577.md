## Summary

Keep open. Current main still has the reported config-contract mismatch: the strict `browser` config schema omits `tabCleanup`, while runtime resolution, public types, help labels, and docs all expose/read that path. The linked open PR #74588 uses closing syntax for this issue, so the right next step is maintainer review of that PR rather than closing this report early.

## What This Changes

Allow operators to configure documented `browser.tabCleanup` fields in `openclaw.json` instead of rejecting them as unrecognized.

## Best Possible Solution

Land a focused schema fix, likely via #74588 if it passes review, that adds an optional strict `browser.tabCleanup` object to the root config schema, regenerates checked-in schema/docs outputs, and adds regression coverage for valid documented overrides plus invalid nested keys.
