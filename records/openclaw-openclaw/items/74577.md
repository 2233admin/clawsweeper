## Summary

Current main still has the reported mismatch: the browser runtime and docs recognize `browser.tabCleanup`, but the strict root config Zod schema does not declare it, so this remains a valid config-validation bug rather than a cleanup-close candidate.

## What This Changes

Accept and validate the documented `browser.tabCleanup` config block so operators can tune browser tab cleanup from `openclaw.json`.

## Best Possible Solution

Add a first-class `browser.tabCleanup` Zod schema with optional `enabled`, `idleMinutes`, `maxTabsPerSession`, and `sweepMinutes` fields, wire it into the strict browser config object, regenerate the checked-in schema/doc baselines, and add regression tests that accept valid cleanup config and reject malformed or unknown nested keys.
