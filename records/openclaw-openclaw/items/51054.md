## Summary

Current main still leaves the reported Media Understanding credential-source gap open. The Media Understanding page shows provider model entries with provider/model/profile fields and says configured models.providers entries participate in auto-detection, but it still does not explicitly document that provider credentials resolve through auth profiles, environment variables, or models.providers.<provider>.apiKey, and current schema/runtime still do not support inline apiKey on tools.media model entries. The related docs PR #51089 proposed the right clarification but is closed and unmerged.

## Best Possible Solution

Keep this issue open and land a small docs-only patch to docs/nodes/media-understanding.md near Config overview or Model entries. It should state that tools.media provider entries do not accept inline apiKey, credentials resolve through auth profiles, provider environment variables, or models.providers.<provider>.apiKey, and include a short models.providers example or link to the public authentication/config docs.
