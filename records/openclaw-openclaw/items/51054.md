## Summary

Keep open. Current main still leaves the Media Understanding credential-source documentation gap unresolved: the docs page shows provider model entries without explaining where credentials come from, runtime still resolves provider keys outside the media entry, and the strict media model schema still has no inline apiKey field. The related docs PR #51089 proposed the right clarification but is closed and unmerged.

## Best Possible Solution

Land a small docs-only patch in `docs/nodes/media-understanding.md` near `Config overview`, `Model entries`, or `Provider auth`. It should state that `tools.media` provider entries do not accept inline `apiKey`, credentials resolve through auth profiles/profile selection, provider environment variables, and `models.providers.<provider>.apiKey` or secret refs, and should link to the public auth/config docs or include a short `models.providers` example. No runtime change is needed unless maintainers deliberately choose to add an inline media credential surface.
