## Summary

Keep open. Current main still has no REST-native session reset/status/compact/model/reasoning endpoints for HTTP API integrations; the underlying Gateway session RPC methods exist, but only WebSocket/RPC and adjacent session kill/history HTTP helpers are present. The closed plugin-SDK reset work and the closed unmerged REST PR do not resolve the requested HTTP API surface, and the recent maintainer review explicitly kept this as the specific tracker.

## What This Changes

The issue requests authenticated REST endpoints for HTTP API integrations to reset, inspect, compact, and update session model/reasoning settings without re-enabling slash-command dispatch in `/v1/chat/completions`.

## Best Possible Solution

Keep this issue open and implement a focused authenticated Gateway HTTP session-management surface that maps reset/status/compact/model/reasoning to the existing session RPC/shared helper semantics, preserves default-deny slash-command handling for `/v1/chat/completions`, adds route/auth/request/response tests, updates the public OpenAI-compatible HTTP docs, and coordinates but does not collapse this specific tracker into the broader #47826 read-only REST endpoint request.

## Repair Work Prompt

Implement this issue as a focused Gateway HTTP session-management REST surface on current main. Add authenticated endpoints for reset/status/compact/model/reasoning under `/v1/sessions/...` or the maintainer-approved path shape from the issue, mapping to existing Gateway session semantics rather than duplicating business logic. Preserve `/v1/chat/completions` default-deny slash-command behavior; do not set `CommandAuthorized` or reinterpret `/new`, `/status`, or `/compact` chat text. Use existing Gateway HTTP auth helpers and operator scope checks appropriate for `sessions.reset`, `sessions.compact`, and `sessions.patch`. Avoid the pitfalls from closed #23941: do not pass `key` into strict `sessions.list` params, do not use a broad `/v1/sessions` prefix match that catches unrelated paths, and validate reset `reason` against `new|reset`. Add route/auth/request/response tests, update `docs/gateway/openai-http-api.md`, and add a user-facing changelog entry crediting the reporter and any source PR author if code/design is reused. Do not change unrelated OpenAI HTTP behavior or plugin SDK contracts.
