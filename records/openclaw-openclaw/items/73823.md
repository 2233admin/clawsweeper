## Summary

Keep this PR open. Current main still lacks the submitted content gate: WhatsApp inbound normalization only filters missing/status/broadcast JIDs and recent outbound echoes before calling `checkInboundAccessControl`, so the linked pairing side-effect bug remains plausible on non-content Baileys upserts. The PR is a focused implementation candidate for the still-open linked bug rather than obsolete work.

## Best Possible Solution

Keep this PR open for normal review. The best path is to review or refine the proposed WhatsApp inbound content gate, land it with regression coverage for non-content Baileys upserts and interactive responses, and then let the linked WhatsApp pairing issue close through the merged fix.
