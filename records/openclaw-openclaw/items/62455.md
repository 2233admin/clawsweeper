## Best Possible Solution

Keep Discord per-channel agent routing canonical through `bindings[]`, but add a Discord-owned validation/doctor path that detects channel-level `agentId`, gives a specific binding example, and repairs only unambiguous cases without adding hidden core migrations.

## Solution Assessment

Yes for a narrow fix: a plugin-owned diagnostic/doctor migration toward `bindings[]` is the safest maintainable path. Adding channel-level routing support should be treated as a separate maintainer product decision, especially with adjacent account-level `agentId` work open.
