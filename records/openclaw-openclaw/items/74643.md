## Summary

Keep open. The latest branch fixes useful schema/type/docs pieces, but it still leaves the normal reply directive and directive-status paths reading verbose/elevated defaults from global `agents.defaults` instead of the per-agent entry, so the linked behavior remains only partially fixed and overlaps the still-open #73696.

## What This Changes

The PR adds per-agent `verboseDefault` and `elevatedDefault` config schema/type/generated metadata/docs, extends resolved agent config for `elevatedDefault`, and adds schema plus agent-scope regression tests.

## Best Possible Solution

Land one consolidated fix for #73680 that keeps the strict per-agent schema/type/docs metadata aligned and also makes normal reply directive resolution, directive-only status/current-level handling, and the resolved agent config path prefer `agents.list[]` verbose/elevated values before `agents.defaults`, with regression coverage for both parsing and runtime precedence.

## Reproduction Assessment

Yes. The original validation failure is source-reproducible from current main's strict `AgentEntrySchema`, and the remaining PR defect is source-reproducible by following a normal reply path where `agentCfg` is `cfg.agents?.defaults` and verbose/elevated defaults are not read from `agentEntry`.

## Solution Assessment

No. Adding the schema/type/docs entries is necessary, but this is not the best complete fix until the runtime directive and current-level helpers consume the accepted per-agent values and prove the precedence in tests.
