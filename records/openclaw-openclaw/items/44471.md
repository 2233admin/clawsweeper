## Summary

Keep open. Current main still emits the reported `ready for user delivery` wording in the subagent completion handoff path, tests still assert that wording, and the requested `available for review` plus external-final-state verification guard is not present. Related cleanup issues #44472 and #44587 are closed and point back to this issue as the remaining canonical source-level bug.

## Best Possible Solution

Keep this issue open as the canonical source-level bug. The focused fix should update `src/agents/subagent-announce.ts` so subagent completion handoff wording says the child result is available for review and explicitly tells the requester to verify whether the child produced only text/analysis or actually applied and verified an external final state before saying done/ready/complete. Update the colocated subagent announce formatting tests and `docs/tools/subagents.md` to match.
