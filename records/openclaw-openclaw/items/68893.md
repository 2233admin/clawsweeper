## Summary

Current main still has the reported behavior: the Homebrew prompt is gated only by `process.platform !== "win32"`, so FreeBSD still reaches the prompt when a selected skill has a brew installer and `brew` is missing. This should stay open because multiple open PRs already use closing syntax for the same narrow fix, and the issue should close only after one lands.

## What This Changes

Stop `openclaw onboard` from recommending Homebrew during skill dependency setup on FreeBSD and other unsupported platforms.

## Best Possible Solution

Land one narrow fix that replaces the Homebrew prompt blocklist with an explicit supported-platform check, adds FreeBSD regression coverage, preserves expected macOS/Linux behavior, and then closes the duplicate PRs and this issue through the merged PR.
