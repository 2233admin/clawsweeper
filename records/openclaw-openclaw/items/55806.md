## Summary

Keep open. Current main has overlapping partial observability work: tool execution diagnostics now carry duration, low-cardinality parameter shape, and error category for OTEL/Prometheus, and Control UI agent events sanitize tool args/results. That does not fully implement this issue’s requested surface: structured toolResult error fields are still absent, INFO/file-log sanitized argument summaries are not present, and only apply_patch has a narrow patch-summary event rather than a general context writeback audit trail. The related PR #55846 was closed unmerged, so it is not a canonical implementation to close against.

## Best Possible Solution

Keep the issue open and implement it as an extension of the existing diagnostics/tool event model: add explicit sanitized argument summaries or opt-in bounded content fields where appropriate, define stable toolResult error metadata if transcript persistence should carry it, and add a generic writeback/audit event contract that can be emitted by mutating tools such as apply_patch, config/session mutations, memory plugins, and file-writing tools.
