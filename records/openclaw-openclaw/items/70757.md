## Summary

Keep this PR open. Current main still authenticates BlueBubbles inbound webhooks with the BlueBubbles API password, still documents password-bearing webhook URLs, and still has setup/schema/tests built around password-based webhook auth. The PR is the active same-author implementation candidate for open linked bug #70755, and the provided review thread shows the author addressed the bot-reviewed setup, validation inheritance, and helper-duplication gaps in the latest PR commit.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent maintainer-approved fix. The best product path is to add a dedicated BlueBubbles webhookSecret, authenticate inbound webhooks only against that secret, enforce that it is present and distinct from the API password across top-level and inherited account configs, expose it through setup/CLI metadata and SecretRef surfaces, and update docs plus regression tests. After merge, close #70755 as implemented.
