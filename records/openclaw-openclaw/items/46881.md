## Summary

Keep this PR open. Current main still does not implement the PR’s intended WhatsApp inbound reaction forwarding: the monitor only subscribes to `messages.upsert` and `connection.update`, `WebInboundMessage` still has no reaction metadata, and the docs/tests only cover outbound/ack reaction behavior. The PR remains a plausible implementation candidate, but it needs a rebase, tests, and resolution of the access-control/dedupe review concerns before merge.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Rebase it onto current main, reconcile it with #57288 and overlapping #48570, then land a WhatsApp-owned `messages.reaction` path with targeted tests and explicit access-control/dedupe/listener-cleanup behavior.
