## Summary

Keep this PR open. Current main still resolves a mixed non-empty `command` plus non-empty `url` MCP server as stdio before considering HTTP, while the docs already say that shape is rejected; the PR is a focused fix with targeted coverage and no blocking review findings from the read-only pass.

## What This Changes

The PR adds an early MCP transport-config guard that rejects mixed non-empty `command` and `url` server definitions, plus tests for diagnostics, whitespace-only URLs, and log sanitization.

## Best Possible Solution

Land this focused guard and its tests, or apply the same change on main, so mixed MCP transport definitions fail early with a clear sanitized diagnostic while pure stdio and pure HTTP configurations keep their existing behavior.
