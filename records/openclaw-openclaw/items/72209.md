## Summary

Keep this PR open. Current main does not already implement passive session recovery checkpoints, and the PR is useful but incomplete: it writes checkpoint state yet does not wire recovered bundles into the real prompt creation path, and it adds hot-path synchronous persistence plus overly broad free-text redaction.

## What This Changes

The PR adds session-recovery prompt discipline, local Task Ledger and Recovery Bundle persistence under the OpenClaw state directory, and checkpoint writes from `update_plan`, `sessions_yield`, and `sessions_spawn` with focused tests.

## Best Possible Solution

Keep the PR open for author and maintainer iteration: load the right Recovery Bundle when a session is actually recreated, pass it through embedded and CLI prompt builders, preserve the passive non-authorizing discipline, move checkpoint persistence off the tool hot path, and tighten redaction to avoid corrupting normal recovery summaries.

## Reproduction Assessment

Yes for the review findings. Static inspection of the PR patch and current main is enough to show the missing runtime bundle-to-prompt wiring, the broad redaction pattern, and the synchronous checkpoint writes.

## Solution Assessment

No. The direction is plausible, but this patch is not yet the best solution because it records recovery data without delivering it to live recreated sessions and adds blocking best-effort persistence on common tool paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wire recovered bundles into real prompt creation:** `src/agents/system-prompt.ts:502`
  - body: This adds `sessionRecovery` only to the leaf prompt builder, but the live embedded and CLI prompt wrappers are not changed and no runtime path loads a saved Recovery Bundle for a recreated session. As written, checkpoints are persisted but never appear in live prompts, so the advertised recovery context is unavailable.
  - confidence: 0.93
- **[P2] Avoid redacting normal recovery prose:** `src/agents/session-recovery-state.ts:115`
  - body: The bare `token|secret|authorization|password` pattern treats ordinary summaries like `authorization: waiting for review` as secret material and stores them as redacted text. That silently damages the recovered context this feature is meant to preserve; keep the high-confidence secret patterns or require stronger value shapes.
  - confidence: 0.86
- **[P2] Move checkpoint writes off the tool hot path:** `src/agents/session-recovery-state.ts:205-206`
  - body: Every enabled `update_plan`, `sessions_yield`, and `sessions_spawn` checkpoint calls synchronous filesystem writes before returning the tool result. Since these checkpoints are best-effort, slow local or network-backed state directories can block unrelated agent progress; queue the write or use async persistence.
  - confidence: 0.88
