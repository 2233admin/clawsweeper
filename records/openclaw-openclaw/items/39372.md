## Summary

Keep this PR open. Current main still rejects unthreaded `mode="session"` in both native subagent and ACP spawn paths, so the PR is not obsolete; however, the supplied branch still needs maintainer review because the ACP inline-delivery concern was acknowledged in discussion and is not addressed by the shown diff.

## What This Changes

The PR removes the native subagent and ACP `mode="session"`/`thread=true` guards, adds headless-session accepted notes and native failed-start cleanup, and changes spawn-hook coverage to expect unthreaded persistent sessions to proceed.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require an update before merge: decouple persistence from binding only if maintainers want that product direction, make ACP inline delivery depend on an actual bound conversation/thread, preserve cleanup behavior, keep current `thread=true` behavior intact, and update the public subagent and ACP docs.
