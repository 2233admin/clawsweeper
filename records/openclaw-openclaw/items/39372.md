## Summary

This PR should stay open. Current main still rejects `mode="session"` without `thread=true` in both native subagent and ACP spawn paths, so the PR is not obsolete or implemented on main. The branch is relevant, but it still needs maintainer product review and the acknowledged ACP delivery fix before it is merge-ready.

## What This Changes

The PR removes the native subagent and ACP `mode="session"`/`thread=true` guards, adds headless-session accepted notes and native failed-start cleanup, and updates spawn hook coverage to expect headless persistent sessions to proceed.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require an update before merge. If maintainers want headless persistent sessions, the fix should remove both guards, make ACP inline delivery require an actual bound conversation/thread, preserve failed-start cleanup, emit accurate headless accepted notes, and update tests plus the public subagent and ACP docs. If maintainers prefer the current thread-bound-only model, close the PR with that product rationale and leave the broader discussion in #53548.
