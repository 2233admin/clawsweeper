## Summary

Keep this issue open. Current main still injects a resolved skills prompt containing every eligible skill until count/character limits apply; compact mode removes descriptions but still keeps one XML block per skill with name and SKILL.md location. I found no task-matched lazy loading, names-only catalog mode, or configurable skills prompt visibility mode that would satisfy the requested optimization.

## Best Possible Solution

Keep this open as a core skills prompt optimization. The likely path is to add an explicit names-only and/or guarded auto prompt visibility mode in the skills prompt builder, config types, zod schema, generated/help docs, and tests, while preserving the current full/compact behavior as the compatible default and covering truncation, per-agent overrides, prompt reporting, and skill-selection guidance.

## Risks / Open Questions

- A naive automatic skill matcher could hide the relevant skill and regress discovery; any auto mode needs a conservative fallback.
- Changing the <available_skills> XML shape may affect existing prompt tests, prompt-report parsing, and model guidance that currently expects <skill> blocks with locations.
- Per-agent skill allowlists and maxSkillsPromptChars help configured users, but they are manual policy controls and do not solve automatic prompt overhead for large installed skill sets.
