## Summary

Keep PR #58732 open. Current main still does not support `AbortSignal` in `runTasksWithConcurrency`, so the PR is not obsolete or implemented, but the submitted implementation needs a contract fix before merge because aborting would return partial results while reporting success.

## Best Possible Solution

Keep the PR open, but require a revised cancellation contract before merge. The best path is still to add optional `AbortSignal` support, with deterministic tests for pre-aborted and mid-run aborts, while surfacing cancellation as an abort error or explicit aborted result instead of returning sparse `results` with `hasError: false`.

## Risks / Open Questions

- Merging the submitted patch as-is could make cancellation indistinguishable from success for callers that only gate on `hasError`.
- Because the helper is exported through `openclaw/plugin-sdk/infra-runtime`, the abort contract should be explicit and covered by tests before it becomes a plugin-facing API.
