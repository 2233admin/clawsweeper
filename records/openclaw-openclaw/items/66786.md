## Summary

Keep open. Current main still ignores OPENCLAW_WORKSPACE_DIR in the default agent workspace resolver, and the system prompt prints that resolved workspace value. The linked implementation candidate, PR #66831, was closed unmerged, so there is no implemented or superseding fix to close against.

## What This Changes

The issue requests that default agent workspace resolution and the system prompt honor OPENCLAW_WORKSPACE_DIR when no explicit agent workspace is configured.

## Best Possible Solution

Implement the env fallback in the shared default workspace resolver so non-empty OPENCLAW_WORKSPACE_DIR wins before OPENCLAW_PROFILE/OPENCLAW_HOME default-path derivation, while explicit per-agent workspace and agents.defaults.workspace continue to take precedence. Add focused resolver, agent-scope, run-workspace, and prompt regression coverage.
