## Summary

Keep this PR open. Current main still lacks the PR's exact hook-context threading for the unwrapped `toToolDefinitions()` fallback path: core tools are wrapped earlier, but bundled MCP/LSP tools can still reach `splitSdkTools() -> toToolDefinitions()` unwrapped, where `runBeforeToolCallHook()` is called without `ctx`. The PR is a plausible targeted fix and should be reviewed or refreshed rather than closed as implemented.

## Best Possible Solution

Keep this PR open for maintainer review or a refreshed patch. The best implementation is to add optional `HookContext` threading through `toToolDefinitions()` and `splitSdkTools()`, pass session metadata from both embedded attempt and compaction paths, consider adding `loopDetection` for parity, and keep the focused regression test proving `ctx.agentId` and session fields reach `before_tool_call` on the unwrapped fallback path.
