## Summary

Keep this issue open. Current main still carries `sessionKey` and turn-source metadata through the exec approval request path, but the macOS native exec approval popup still renders only cwd, agent, executable, host, security, and ask mode. It also does not decode or display higher-level requester context such as channel, sender, or thread, while the Control UI already shows the session row.

## Best Possible Solution

Add a focused macOS prompt fix in `apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift`: trim and render a non-empty Session detail row, then decode/display existing safe turn-source fields such as channel, target/account, and thread where available. Keep approval policy, auth, command sanitization, timeout, and allow/deny behavior unchanged; treat sender display names or tool-reason text as follow-up product/API work unless existing metadata can support it safely.
