## Summary

Keep this PR open. Current `main` still does not implement the followup delivery cross-target dedupe guard: the followup helper dedupes text/media before checking whether the messaging-tool send targeted the origin conversation. The PR contains a plausible fix and regression coverage, but it also bundles unrelated model retry, outbound serialization, and dependency changes with unresolved behavioral/security review concerns, so the best path is a narrowed PR rather than closure.

## Best Possible Solution

Keep this PR open, but ask for a narrow version: put the same-target dedupe guard in `resolveFollowupDeliveryPayloads` or the followup-specific payload path, add followup text/media cross-target regression tests, and preserve legacy no-target dedupe behavior. Split the same-model transient retry, outbound `DeliverySerializer`, and Mattermost dependency changes into separate PRs with their own functional and security review.
