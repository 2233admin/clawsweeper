## Summary

Keep this PR open. Current main still has the undefined-params crash path: `hashToolCall` passes `params: unknown` through a serializer typed as `string`, but `JSON.stringify(undefined)` returns the JavaScript value `undefined`, which then reaches `createHash().update(...)` and throws `ERR_INVALID_ARG_TYPE`. The PR is narrow and directionally correct, but it still needs focused regression coverage before landing.

## Best Possible Solution

Keep this PR open as a valid narrow bug fix. The best path is to add focused regression coverage in `src/agents/tool-loop-detection.test.ts` for undefined tool params and the undefined non-Error formatting path, then land the minimal serialization guard after the targeted agent test lane passes.
