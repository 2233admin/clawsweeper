## Summary

Keep this issue open. Current main has compaction checkpoint snapshot recovery and context-engine compaction seams, but it still lacks the requested native checkpoint engine APIs, deterministic replay packets, task-boundary contract, checkpoint diffing, and missing-field validation hooks.

## Best Possible Solution

Keep this open as the native checkpoint/replay umbrella. A complete solution should define a versioned Gateway/session checkpoint API with checkpoint.create/list/resume/diff, a structured done/saved/dropped/next task contract, deterministic replay packet generation, diff/audit data, missing-field validation hooks, UI/docs/tests, and compatibility with the existing compaction checkpoint branch/restore surface. If this should move to a plugin, first design the core plugin/API seam needed to support it.
