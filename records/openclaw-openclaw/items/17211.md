## Summary

Keep open. Current main has compaction checkpoint snapshot recovery and a pluggable context-engine compact hook, but it does not implement the requested native checkpoint engine with checkpoint.create/list/resume/diff APIs, deterministic replay packets, task-boundary contracts, diffing, or missing-field validation hooks. The existing maintainer/Codex comment also explicitly kept this umbrella open.

## Best Possible Solution

Keep this open as the native checkpoint/replay umbrella. A complete solution likely belongs in versioned Gateway/session protocol plus session persistence/runtime/UI/docs/tests: checkpoint.create/list/resume/diff, a structured done/saved/dropped/next contract, deterministic replay packet generation, diff/audit data, validation hooks, and compatibility with the current compaction checkpoint branch/restore surface. If maintainers want plugin ownership, first define the missing checkpoint/context-engine API so a plugin can implement it without core internals.
