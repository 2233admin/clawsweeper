## Summary

Keep this PR open. For https://github.com/openclaw/openclaw/pull/43810, the affected surface is the shared DM pairing challenge helper used by multiple channel plugins. Current main still preserves the no-resend contract, while the branch would resend pending pairing codes and make creation-only side effects fire for repeat start messages; that is not ready to merge without maintainer product/security policy.

## What This Changes

The PR changes `issuePairingChallenge` so a pending sender with an existing pairing code falls through to send another pairing reply instead of returning early when the pairing request was not newly created.

## Best Possible Solution

If repeat pairing reminders are desired, implement them as an explicit resend path that reuses only valid pending codes, keeps first-created and resent outcomes distinct, preserves creation-only side effects, documents the resend policy, and updates shared plus channel-specific tests together.
