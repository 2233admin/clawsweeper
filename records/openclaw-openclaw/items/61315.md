## Summary

Keep this PR open. Current main still does not implement the intended usage aggregation changes: sessions.usage totals and aggregates are still scoped to limited returned sessions, channel/agent rankings remain cost-first, the result type has no overview scope metadata, and the UI still recomputes overview insights from returned rows while displaying Top Agents/Top Channels cost first. The PR remains useful, but its review threads identify unresolved limit, scope, fallback, and I/O concerns that need maintainer rework before merge.

## Best Possible Solution

Keep this PR open and rework it into a bounded usage fix. The best path is to define the sessions.usage scope contract clearly, keep limit restricted to returned rows, make totals/aggregates/UI insight denominators use the same documented scope, preserve fallback behavior for gateways without overview metadata, rank zero-cost channel and agent results by tokens while preserving cost-first behavior when cost exists, bound or cache full matched-session summary I/O, and land focused regressions for zero-cost ranking and limit-scoped aggregate correctness.
