## Summary

Current main still does not implement this PR's intended usage aggregation behavior. The gateway still scopes sessions.usage totals and aggregates to the limited returned rows, channel/agent rankings remain cost-first in zero-cost cases, the result type has no overview scope metadata, and the UI still recomputes/display-ranks usage insights from returned rows. The PR remains a plausible fix candidate, but the existing review discussion shows scope, fallback, and I/O concerns that need maintainer follow-up before merge.

## Best Possible Solution

Keep this PR open and rework it into a bounded usage fix. The target contract should make sessions.usage explicit about matched-session versus returned-session scope, keep limit restricted to returned rows, use one documented scope for totals/aggregates/insight denominators, preserve a fallback limit warning for gateways without overview metadata, rank zero-cost channel and agent results by tokens while preserving cost-first behavior when real cost exists, and add focused regressions for zero-cost ranking plus limit-scoped aggregate correctness.
