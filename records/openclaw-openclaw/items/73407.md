Keep this open: current main and the latest stable release still carry an active WhatsApp plugin dependency path from @openclaw/whatsapp to Baileys to GPL-3.0 @whiskeysockets/libsignal-node, and deciding whether to replace, isolate, document, or stop publishing that path is a maintainer/legal distribution decision rather than a conservative cleanup close.

## Best Possible Solution
