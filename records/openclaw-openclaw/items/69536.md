## Summary

Current main still has the reported mismatch for the dreaming/backfill path. The session-memory hook intentionally writes `memory/YYYY-MM-DD-slug.md`, while memory-core daily ingestion, CLI `memory rem-backfill`, and the Control UI dream-diary backfill only discover exact `YYYY-MM-DD.md` files. A related startup-context fix loads slugged files for `/new` and `/reset`, but that does not make them visible to the dreaming promotion/backfill pipeline.

## Best Possible Solution

Keep the issue open and implement one shared memory-file discovery/normalization policy for memory-core daily ingestion, CLI `memory rem-backfill`, and Control UI backfill. Either make the session-memory hook append to canonical `memory/YYYY-MM-DD.md`, or deliberately ingest bounded `memory/YYYY-MM-DD-*.md` session artifacts with regression tests and docs updates for the chosen contract.
