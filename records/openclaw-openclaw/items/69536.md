## Summary

Current main still has the reported mismatch. The bundled session-memory hook writes `memory/YYYY-MM-DD-slug.md`, while memory-core CLI historical discovery, live daily ingestion, historical seeding, and Gateway Dream Diary backfill still accept only exact `YYYY-MM-DD.md` files. The related startup-context work covers `/new` and `/reset` prompt preloading, but not dreaming promotion or grounded backfill.

## What This Changes

Fix the inconsistency where slugged `session-memory` artifacts are not discovered by dreaming and grounded backfill paths that only accept date-only daily memory filenames.

## Best Possible Solution

Keep this open and fix memory-core daily-file discovery with a shared dated-memory-file matcher that accepts canonical date-only files and slugged session-memory artifacts. Use it in CLI `rem-harness`/`rem-backfill`, live dreaming daily ingestion, historical grounded seeding, and Gateway Dream Diary backfill; update focused tests and docs while preserving deterministic ordering and source paths.
