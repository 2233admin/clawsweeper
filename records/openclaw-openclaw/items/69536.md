## Summary

Current main still has the reported mismatch for the dreaming/backfill path. The session-memory hook and docs still produce `memory/YYYY-MM-DD-slug.md`, while memory-core live daily ingestion, CLI `rem-harness`/`rem-backfill`, and Gateway Dream Diary backfill still filter for exact `YYYY-MM-DD.md` files. Startup context now reads slugged artifacts, which partially addresses reset/new context visibility, but it does not make those files participate in dreaming promotion/backfill.

## Best Possible Solution

Keep this open and fix memory-core daily-file discovery so existing session-memory artifacts can participate in dreaming/backfill. A focused implementation should add a shared dated-memory-file matcher that accepts both `YYYY-MM-DD.md` and `YYYY-MM-DD-*.md`, use it in live daily ingestion, CLI historical rem-harness/rem-backfill, and Gateway Dream Diary backfill, then update tests and docs while preserving deterministic ordering and source attribution.
