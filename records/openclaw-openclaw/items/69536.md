## Summary

Keep open. Current main still has the reported cross-system mismatch: the bundled hook writes slugged dated memory artifacts, but the memory-core CLI backfill, live daily ingestion, historical seeding, and Gateway Dream Diary backfill paths still filter for exact date-only daily files. The startup-context fix covers first-turn prompt loading, but it does not make those slugged artifacts participate in dreaming promotion or grounded backfill.

## What This Changes

The issue requests a consistent memory-file contract so `session-memory` artifacts written as `memory/YYYY-MM-DD-slug.md` are visible to dreaming and grounded backfill paths that currently expect `memory/YYYY-MM-DD.md`.

## Best Possible Solution

Add a shared dated-memory-file matcher that accepts canonical date-only files and slugged session-memory artifacts, then reuse it in memory-core CLI historical discovery, live daily ingestion, historical grounded seeding, and Gateway Dream Diary backfill. Keep the writer behavior unchanged unless maintainers separately choose to land a broader canonical-daily writer change.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: a file like `memory/2026-04-21-ai-assistant-config.md` is produced by the hook shape, but the current `rem-backfill`, daily ingestion, historical seeding, and Gateway backfill regexes all reject it because they only match `YYYY-MM-DD.md`.

## Solution Assessment

Yes. A constrained slugged-file ingestion fix is the narrowest backward-compatible solution because it preserves existing session-memory artifacts and avoids making the hook writer change the migration mechanism; changing only the writer would still leave already-created slugged files invisible.
