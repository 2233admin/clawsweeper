## Summary

Keep open. Current main still has the Mattermost `replyToMode: "off"` thread-delivery bug this PR targets: existing thread roots are suppressed into `effectiveReplyToId: undefined`, and the Mattermost inbound context, typing indicators, draft preview, and reply delivery paths still reuse that same value for delivery routing. Related PR #55186 was closed as superseded by this decoupled approach, not because the bug is fixed; dependency PR #57970 also remains absent from current main.

## Best Possible Solution

Keep #57565 open as the canonical decoupled Mattermost fix: preserve `effectiveReplyToId` for session routing, add/use a separate delivery thread id for existing channel/group thread roots, update all Mattermost context/typing/draft/reply call sites, include the `resolveThreadFlag` guard for non-forked channel sessions, and land or split out #57970's `ChatType: "unknown"` prerequisite before rebasing.

There is no separate `messageThreadId` on current main to preserve the inbound thread root while keeping the session key top-level.
