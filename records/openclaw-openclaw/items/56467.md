## Summary

Keep open. Current main tracks subagent terminal outcomes and background task failure status, but it does not implement N-strike enforcement: there is no durable consecutive failure count on subagent run records, no `agents.defaults.failureThreshold` or `agents.defaults.onFailure` config, and no automatic escalate/disable/notify action after repeated agent or subagent failures. The provided related items are only title-overlap context around agents/subagents and do not supersede this request.

## Best Possible Solution

Keep this open for a focused design/implementation. The best path is to add explicit failure semantics, durable consecutive counters keyed per agent/subagent, config/schema/help for threshold and action, enforcement or notification behavior, and regression tests for reset-on-success plus `error`/`timeout`/`killed` handling. If maintainers prefer this as plugin-owned behavior, first define the missing plugin API surface rather than closing it as already externalizable.
