## Summary

Keep open. Current main still does not implement the requested two-strike/N-strike agent or subagent enforcement: subagent state stores per-run outcomes, completion persists a single terminal outcome/reason, config schema/types/docs have no failure-threshold policy, and the plugin hook event still lacks consecutive-failure context. The earlier ClawSweeper keep-open review remains accurate against the current main SHA.

## What This Changes

Request to add configurable consecutive-failure tracking and threshold actions for agents/subagents, including `failureThreshold`, `onFailure`, failure counts in run records, and richer `subagent_ended` hook context.

## Best Possible Solution

Keep this open for maintainer product/API review. If accepted, the end state should be a typed subagent failure-policy contract with explicit outcome classification, config schema/types/docs, hook context for plugins, and tests for counter reset, threshold crossing, and chosen enforcement actions.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. There is a high-confidence static verification path: inspect the current subagent lifecycle, config schema/types/docs, hook event type, and targeted identifier search to confirm the requested counters and policy are absent.

## Solution Assessment

Unclear. Core enforcement may be the right direction if OpenClaw owns this safety policy, but the requested config/action shape should wait for maintainer decisions on failure classification and action semantics.
