## Summary

Keep open. Current main still renders native chat bubble bodies as plain SwiftUI `VStack` / Android Compose `Column` content, with scrolling only at the transcript/list level. The latest release tag has the same gap, and the existing maintainer review comment already reached the same conclusion.

## Best Possible Solution

Keep this issue open for a focused native chat UI fix. Define the current equivalent of `.large` or tall bubbles in the SwiftUI and Android chat bubble components, wrap the message body area in a bounded vertical scroll container for those bubbles, preserve transcript-level scrolling and auto-scroll behavior, and add focused render coverage or documented manual mobile verification for tall-but-not-truncated content.
