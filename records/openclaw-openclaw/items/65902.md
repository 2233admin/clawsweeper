## Summary

Keep this issue open. Current main still permits the reported mismatch: WhatsApp direct sessions can be persisted and visible through status/store-backed paths, while agent-side session tools apply the default `tools.sessions.visibility="tree"` guard and can hide or forbid same-agent WhatsApp DM sessions that are neither the current session nor spawned by it.

## Best Possible Solution

Keep this open for maintainer policy and implementation follow-up. The best fix is a generic deliverable-session discovery or target-resolution path that makes existing same-agent channel DMs usable for on-demand automation only within the intended `tools.sessions.visibility` and sandbox rules, with WhatsApp direct coverage and docs explaining the required visibility setting.
