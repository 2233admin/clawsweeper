## Summary

Keep this PR open. Current main still filters explicit OpenAI/OpenAI-Codex cache retention out before stream options, so the user-facing gap remains, but the PR branch is stale against the current resolver shape and would regress model-aware cache eligibility if applied as-is.

## What This Changes

The PR extracts cache-retention resolution into a provider-neutral helper, forwards explicit `cacheRetention` for OpenAI and OpenAI-Codex without adding defaults, keeps Anthropic behavior, and adds focused unit tests.

## Best Possible Solution

Add explicit OpenAI/OpenAI-Codex passthrough to the current `src/agents/pi-embedded-runner/prompt-cache-retention.ts` resolver with no implicit defaults, preserve the existing Anthropic, Bedrock, custom Anthropic, and Google rules, and add regression coverage for both config-to-stream-options and direct OpenAI Responses payload mapping.
