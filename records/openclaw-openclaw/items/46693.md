## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: explicit `cacheRetention` for `openai` / `openai-codex` is filtered out before it reaches stream options, even though the downstream OpenAI Responses transport can emit `prompt_cache_retention: "24h"` when it receives `cacheRetention: "long"`.

## Best Possible Solution

Keep this PR open and rework it onto current main by extending `src/agents/pi-embedded-runner/prompt-cache-retention.ts` to pass through explicit `cacheRetention` for `openai` and `openai-codex` without adding defaults. Add focused regression tests showing configured OpenAI/OpenAI-Codex retention reaches stream options, plus a direct OpenAI Responses payload test proving `cacheRetention: "long"` produces `prompt_cache_retention: "24h"` on direct OpenAI hosts.
