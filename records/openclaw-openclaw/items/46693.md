## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: explicit `cacheRetention` for `openai` and `openai-codex` is filtered out before it reaches stream options, while the downstream OpenAI Responses transport can map an already-present `cacheRetention: "long"` to `prompt_cache_retention: "24h"` for direct OpenAI hosts. The prior discussion in #27515 and the PR body identify a real remaining gap, but the patch needs rework onto the current `prompt-cache-retention.ts` layout.

## Best Possible Solution

Keep this PR open and rework it onto current main by extending `src/agents/pi-embedded-runner/prompt-cache-retention.ts` to allow explicit `cacheRetention` passthrough for `openai` and `openai-codex` without adding defaults. Add focused regression tests proving configured OpenAI/OpenAI-Codex retention reaches stream options, plus an OpenAI Responses payload test showing `cacheRetention: "long"` maps to `prompt_cache_retention: "24h"` on direct OpenAI hosts.
