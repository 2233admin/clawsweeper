## Summary

Keep open. Current main still has the IRC reliability and NickServ fallback-nick gaps this PR targets, but the proposed patch is not mergeable as-is because it regresses graceful QUIT delivery and leaves delayed NickServ reclaim timers outside the client lifetime.

## What This Changes

The PR modifies bundled IRC client/send/test code to make empty or closed outbound sends fail visibly, delay transient disconnects, use account-qualified NickServ IDENTIFY, and reclaim fallback nicks with delayed GHOST/NICK.

## Best Possible Solution

Merge a repaired IRC plugin change that keeps the useful visibility, closed-socket detection, transient drain delay, account-qualified IDENTIFY, and fallback-nick reclaim behavior while preserving graceful QUIT delivery and binding delayed reclaim work to client shutdown.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:166-168`
  - body: The new `sendRaw` guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before it sends `QUIT` through `sendRaw`. Every graceful shutdown now swallows that error and closes the socket without sending the QUIT frame. Move the closed transition until after QUIT is written, or let the shutdown frame bypass this guard.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:343-360`
  - body: The delayed NickServ GHOST/NICK timers are not stored, cleared, or `unref()`'d. Once scheduled, they can keep one-shot IRC sends alive after `quit()` and can fire against a client that has already been closed. Track them and clear or unref them during `quit()`/`close()`.
  - confidence: 0.86
- **[P3] Avoid duplicate NickServ reclaim attempts:** `extensions/irc/src/client.ts:343-360`
  - body: The delayed reclaim block can run after `tryRecoverNickCollision()` already sent GHOST/NICK during collision recovery. Gate the post-identify reclaim path on that attempt state so slow or rejected first attempts do not send duplicate NickServ GHOST commands unnecessarily.
  - confidence: 0.74
