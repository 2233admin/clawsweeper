## Summary

Keep open. The PR addresses real IRC gaps that still exist on current main, but the proposed patch should not merge as-is because the new closed-send guard suppresses graceful QUIT and the delayed NickServ reclaim timers are not tied to client lifetime.

## What This Changes

The PR modifies the bundled IRC plugin client, transient send helper, and NickServ tests to surface empty or closed sends, delay transient disconnects, use account-qualified NickServ IDENTIFY, and schedule fallback-nick GHOST/NICK reclaim.

## Best Possible Solution

Merge a repaired IRC plugin change that keeps the useful reliability and NickServ fixes while preserving graceful QUIT, making fallback reclaim timers part of the client lifecycle, and adding focused client/send coverage for the changed behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new `sendRaw` guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before it calls `sendRaw("QUIT ...")`. Every graceful shutdown now catches that error and closes the socket without writing a QUIT frame. Move the closed transition until after QUIT is written, or let the shutdown frame bypass this guard.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:354-369`
  - body: The delayed NickServ GHOST/NICK `setTimeout` calls are not stored, cleared, or unref'd. Once scheduled, they can keep one-shot IRC sends alive after `quit()` and can fire against a client that has already been closed. Track these timers and clear or unref them as part of shutdown.
  - confidence: 0.86
