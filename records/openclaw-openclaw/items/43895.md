## Summary

Keep open. The reported bug is still real on current main: aggregate status defaults are resolved once without agent context even though the status runtime already has an agent-aware resolver. The PR is useful but stale against the current status overview refactor because it patches an older inline `status.command.ts` row, while current human output is rendered through the shared overview helpers.

## What This Changes

The PR derives status session defaults from each agent’s effective model, adds an optional `variesByAgent` marker to status and TUI types, renders `varies by agent`, and adds divergent/uniform regression tests.

## Best Possible Solution

Implement the fix against the current status summary/runtime shape: compute `sessions.defaults` from each configured agent’s effective read-only default model, expose a backward-compatible variation marker through CLI JSON and TUI types, render that marker in `buildStatusSessionsOverviewValue()` and TUI summary output, and add divergent/uniform tests for both summary construction and overview formatting.
