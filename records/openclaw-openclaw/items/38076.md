Current main still parses `init_skill.py --resources` case-sensitively: `parse_resources` strips tokens but validates the original casing, so mixed-case resource names are still rejected. The related implementation PRs in the supplied context are closed and unmerged, and there is still no focused regression test for `parse_resources`.
