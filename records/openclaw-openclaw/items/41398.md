## Summary

Keep open. Current main has partial session-boundary cleanup for duplicated provider prefixes, but it does not implement this PR's intended Google-owned transport/model normalization. Google model helpers and manifest policy still do not strip `google/` or `models/`, the native Gemini and PDF paths still build URLs from unnormalized IDs, and OpenAI-compatible payloads still send `model.id` verbatim if a prefixed Google model reaches transport. The related #41517 is an open implementation candidate for one resolveModel path, but it does not cover the broader Google transport/PDF hardening in this PR.

## Best Possible Solution

Keep this PR open, but rework it against current main: implement Google-owned prefix cleanup through `extensions/google` manifest/helper/runtime seams, add focused tests for Google OpenAI-compatible chat plus native Gemini/PDF URL construction, preserve slash-containing model IDs for non-Google providers, and coordinate with #41517 if that PR supplies the generic same-provider resolveModel piece.
