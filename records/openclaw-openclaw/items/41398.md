## Summary

Keep this PR open. Current main still lacks the Google-owned prefix stripping that the PR is trying to add in the Google helper, manifest policy, native Google transport URL construction, and Gemini PDF path. The overlapping #41517 PR only covers the embedded-runner same-provider resolver path, while this PR still contains useful Google transport/PDF work plus unrelated security-sensitive diffs that should be split or explicitly reviewed.

## Best Possible Solution

Keep the PR open for maintainer follow-up or fold the useful pieces into a focused Google-owned fix: add Google manifest/helper prefix stripping, normalize and validate native Google transport and `geminiAnalyzePdf()` before provider calls, preserve slash-containing model IDs for non-Google pass-through providers, add regression tests for OpenAI-compatible Google chat plus native Gemini/PDF paths, coordinate with #41517, and split unrelated Signal/catalog/bundled-dir changes into separate PRs.
