## Summary

Keep open. Current main still lacks the Google-owned prefix stripping for the native Google transport and Gemini PDF path, and this PR still contains useful regression coverage, but it is not merge-ready because it mixes unrelated security-sensitive changes and introduces a bundled plugin override regression.

## What This Changes

The PR adds Google model prefix stripping in the Google helper, plugin manifest, native Google transport, and Gemini PDF path with tests, while also changing browser trash roots, Signal installer fetching, channel catalog priority, and bundled plugin directory resolution.

## Best Possible Solution

Land a focused Google-owned fix that strips `google/`, `gemini/`, and `models/` prefixes through the Google manifest/helper and validates native Google transport plus Gemini PDF URL construction, with regression tests for OpenAI-compatible Google chat, native Gemini streaming, and PDF analysis; split the browser, Signal, catalog, and bundled-dir changes into separate reviewed PRs or drop them from this branch.

## Solution Assessment

No. The Google prefix-stripping direction is appropriate, but this PR is not the best mergeable form because it combines unrelated browser, Signal, catalog, and bundled-dir changes, and the bundled-dir hunk introduces a plugin code-loading trust regression.
