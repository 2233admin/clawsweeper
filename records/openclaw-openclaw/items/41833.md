## Summary

Keep this issue open. Current main still has useful partial mitigations for deferred work, including cron scheduling, task records, Task Flow, standing orders, compaction preservation, and a reminder guard, but it does not provide the requested native commitment primitive with `openclaw commit add/list/done`, durable commitment records, completion reconciliation, overdue escalation, or startup/session surfacing.

## Best Possible Solution

Keep the issue open and design a native commitment layer above cron/tasks/Task Flow: durable records with due time, owner/channel target, optional linked cron/task/flow run, `add/list/done/cancel/status` operations, reconciliation from cron/task outcomes, overdue or failed-run escalation, and startup/session surfacing for uncompleted commitments. Reuse the existing scheduler and task ledger rather than building a parallel runner.

- Escalation needs clear routing, privacy, authority, and failure semantics across channels, sessions, cron, tasks, and Task Flow.
