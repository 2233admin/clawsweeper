## Summary

Keep open. Current main still has cron scheduling, cron-backed task completion tracking, failure alerts, compaction preservation of promised follow-ups, and a narrow reminder guard, but it does not implement the requested native commitment/deferred-task primitive with `commit add/list/done`, generic commitment records, overdue escalation, or startup/session surfacing for non-reminder promises. The related PENDING.md items remain workaround/plugin-pattern discussions and do not supersede this core product request.

## Best Possible Solution

Keep this issue open and scope a first-class commitment record with due time, linked cron/task run, done/cancel/list/status operations, reconciliation from cron/task outcomes, overdue or failed-run escalation, and startup/session surfacing for uncompleted commitments. PENDING.md or skill-style patterns can remain optional workarounds, but they should not be treated as a replacement for the requested native primitive.

## Risks / Open Questions

- Native commitment creation from natural-language promises needs product design for false positives and user intent.
- Overdue escalation and channel notification behavior need explicit authority, consent, and routing boundaries.
- The eventual implementation should reuse cron run history, task-ledger reconciliation, failure alerts, and the existing reminder guard rather than adding a parallel scheduler.
