## Summary

Keep open. Current main still has cron scheduling, task-run tracking, cron failure alerts, deferred follow-up guidance, compaction preservation, and a narrow reminder guard, but it does not implement the requested native commitment primitive with `openclaw commit add/list/done`, generic commitment records, completion reconciliation, overdue escalation, or startup/session surfacing for unscheduled promises. The related PENDING.md reports are workaround/plugin-pattern discussions and do not supersede this core product request.

## Best Possible Solution

Keep this issue open and scope a first-class commitment record with due time, linked cron/task run, done/cancel/list/status operations, reconciliation from cron/task outcomes, overdue or failed-run escalation, and startup/session surfacing for uncompleted commitments. PENDING.md or skill-style patterns can remain optional workarounds, but they should not be treated as a replacement for the requested native primitive.

## Risks / Open Questions

- Natural-language commitment capture needs product design for false positives and user consent.
- Overdue escalation needs explicit channel routing, authority, and notification boundaries.
- The implementation should reuse cron run history, task-ledger reconciliation, failure alerts, and the reminder guard instead of adding a parallel scheduler.
