## Summary

Current main still has adjacent automation pieces, but not the requested native commitment/deferred-task primitive. Cron schedules jobs, tasks track detached runs, Task Flow tracks multi-step flows, standing orders document discipline, and the reminder guard warns for a narrow reminder-like promise without a cron add. None provide durable commitment records, `openclaw commit add/list/done`, completion reconciliation, overdue escalation, or startup/session surfacing for unresolved conversational commitments.

## Best Possible Solution

Keep this open for maintainer design. The best path is a native commitment layer above the existing scheduler and task ledger: durable records with due time, owner/channel target, optional linked cron/task/flow run, add/list/done/cancel/status operations, reconciliation from cron/task outcomes, overdue or failed-run escalation, and startup/session surfacing for unresolved commitments. Reuse cron/tasks/Task Flow where possible, and make natural-language capture explicit enough to avoid accidental durable state.
