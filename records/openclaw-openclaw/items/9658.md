## Summary

Current main has partial provider-side tool-call streaming/repair, but it still does not expose streaming, stdin, file-reference, or blob-style arguments for `exec`, `write`, `process`, or `apply_patch`. The request remains a real core/tool-transport enhancement, not an implemented-on-main cleanup candidate.

## Best Possible Solution

Keep this open for maintainer design. The best path is an additive, security-reviewed large-argument transport for core tools, likely starting with explicit file/blob/stdin references for `exec`/`process` and `write`/`apply_patch`, plus clear limits/errors and docs. It should not be treated as a small one-file fix because it affects tool schemas, provider/tool-call handling, filesystem boundaries, approvals, sandbox/node execution, and transcript/debug surfaces.
