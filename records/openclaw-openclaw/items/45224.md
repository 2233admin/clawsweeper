## Repair Work Prompt

Fix the browser/Gateway crash reported in openclaw/openclaw#45224. Current main only suppresses Playwright dialog-close races; add a browser-plugin-owned recovery/handler for Playwright CDP assertion unhandled rejections that match the reported `CRSession._onMessage` / `CRConnection._onMessage` stack, and consider the related `CRBrowser._onAttachedToTarget` shared-worker assertion from #52051 if the same scoped predicate can safely cover it. Keep the change in the browser plugin if possible; do not broadly suppress all `Assertion error` failures in core, do not change the Playwright dependency, and do not add startup/load-time migrations. The handler should log/degrade browser capability rather than let an idle browser CDP failure terminate unrelated Gateway sessions. Add focused tests in `extensions/browser/src/browser/runtime-lifecycle.unhandled-rejections.test.ts` or a nearby colocated test proving the synthetic reported error shape is handled, while unrelated generic assertions remain unhandled. Changelog: add a user-facing fix entry only if the repair changes shipped behavior, with reporter credit per repo rules.
