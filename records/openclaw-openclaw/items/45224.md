## Summary

Keep open. Current main still leaves the reported Playwright `CRSession._onMessage` assertion as an unclassified `Error: Assertion error`: the browser plugin only registers a dialog-race classifier, while the Gateway's global unhandled-rejection fallback still exits for unknown errors. The merged #73983 work is useful adjacent context but only covers `Page.handleJavaScriptDialog` / `No dialog is showing`, and the related worker-target PR #46259 remains closed unmerged.

## What This Changes

The issue requests that Playwright CDP assertion failures from `CRSession._onMessage` be isolated or recovered so they do not crash the Gateway and terminate unrelated sessions or cron work.

## Best Possible Solution

Add a browser-owned stability boundary for narrowly identified Playwright CDP session assertion rejections: classify the specific `CRSession._onMessage` assertion shape, log it, evict or reset the affected browser connection as needed, and preserve core's fatal fallback for unknown unhandled rejections. Cover direct and wrapped error shapes with focused browser lifecycle tests, and only touch core if a generic SDK seam is missing.
