## Summary

Keep this PR open. Current main still indexes memory files through markdown/multimodal-only discovery and fixed-window `chunkMarkdown()` calls, so the PR's central feature, code-file memory indexing with declaration-boundary chunking, is not implemented. The contribution is still potentially useful, but it now needs a rebase from the old `src/memory/*` paths to the current memory-host SDK plus `extensions/memory-core` layout, and it should resolve the remaining watcher consistency concern from review before maintainer merge consideration.
