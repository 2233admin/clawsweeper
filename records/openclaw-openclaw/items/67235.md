## Summary

Keep open. Current main still resolves the requested agent for the non-probe `models status` overview, but the probe pipeline does not receive that resolved `agentDir` or agent context. `buildProbeTargets()` still reads the default auth profile store, and probe execution still resolves the default agent, so the reported per-agent auth-order mismatch remains valid. The directly related PR #67263 is closed and unmerged, but its approach is useful follow-up context.

## Best Possible Solution

Keep this issue open and land a narrow fix that threads the already-resolved `agentDir` and target agent execution context from `modelsStatusCommand()` through `runAuthProbes()`, `buildProbeTargets()`, and `runTargetsWithConcurrency()`/`probeTarget()`. Add regression tests proving `models status --probe --agent <id>` reads the target agent auth store and execution context while the no-`--agent` path preserves default-store behavior. #67263 is the closest prior implementation, but it needs to be reapplied or reviewed against current main before closing this issue.

## Clownfish Work Prompt

Fix the `models status --probe --agent <id>` auth-store mismatch tracked by issue #67235. Current main resolves `agentDir` in `modelsStatusCommand()` and uses it for the non-probe auth overview, but the probe path drops it: `runAuthProbes()` has no agent context, `buildProbeTargets()` calls `ensureAuthProfileStore()` with no `agentDir`, and `runTargetsWithConcurrency()` resolves the default agent/dir/workspace/session internally. Reapply the narrow idea from closed PR #67263 against current main, but make sure both target planning and live probe execution use the requested agent. Likely change `runAuthProbes()` to accept the resolved agent context from `modelsStatusCommand()` and pass `agentDir` into `buildProbeTargets()`/`ensureAuthProfileStore(agentDir)`, and pass the target `agentId`, `agentDir`, workspace dir, and session dir into the execution side instead of resolving defaults there. Preserve no-`--agent` behavior. Add regression tests in `src/commands/models/list.probe.targets.test.ts` for forwarding `agentDir` to `ensureAuthProfileStore(agentDir)` and preserving `undefined` for the default path, and add/update `src/commands/models/list.status.test.ts` to prove `--agent <id>` passes the resolved agent context into probes. Do not change auth-order semantics, provider selection, or unrelated model-list behavior. This is user-facing, so add a single-line changelog fix entry in the active release section with `Thanks @ginishuh` and, if reusing #67263's approach, `@hclsys`. Validate with targeted tests first, then the changed gate through the repo's normal Testbox path if promoted by a maintainer.
