## Summary

Keep open. Current main still resolves the requested agent for the non-probe status/auth overview, but the probe pipeline does not receive that resolved agent directory. `buildProbeTargets()` still opens the auth profile store with no `agentDir`, and probe execution still resolves the default agent context, so the reported per-agent auth-order mismatch remains possible. The directly related fix PR #67263 is closed and unmerged.

## Best Possible Solution

Keep this issue open and review/apply the #67263 approach on current main, extended through the whole probe pipeline: pass the resolved `agentDir` and likely `agentId`/workspace/session context from `modelsStatusCommand()` into `runAuthProbes()`, `buildProbeTargets()`, and probe execution. Add regression tests proving `--agent <id>` forwards the target agent directory into `ensureAuthProfileStore(agentDir)` while the no-`--agent` path preserves default-store behavior.
