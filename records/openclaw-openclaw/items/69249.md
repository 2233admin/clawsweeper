## Summary

Keep this issue open. Current main still has only partial restart recovery: Discord abort handling returns without sending a recovery notice, main-session restart recovery resumes internally with `deliver: false`, and the channel-visible restart-sentinel path only runs when a sentinel was written before restart.

## Best Possible Solution

Keep this open and implement a generic post-restart, channel-visible interruption recovery path. On startup, detect abandoned channel-originated running or restart-aborted sessions, use persisted delivery context/last route to post a Discord-visible recovery notice, and only auto-resume when the transcript tail is explicitly safe. Reuse the restart-sentinel outbound machinery where possible and keep #38836 in view so the core behavior is not Discord-only.
