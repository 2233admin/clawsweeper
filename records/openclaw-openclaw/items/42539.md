## Summary

Keep this issue open. Current main still lacks a configurable Telegram TTS delivery mode: `messages.tts` has no delivery/caption policy field, auto-TTS still returns a single visible-text-plus-audio payload, and Telegram still uses short/medium text as a voice/audio caption, splitting only caption overflow or caption-too-long fallback cases.

## Best Possible Solution

Keep this issue open. Implement a Telegram-owned delivery policy or shared `messages.tts.delivery: "bundled" | "separate"` contract, update strict config types/schema, generated/public docs, Telegram bot and direct send paths, and focused tests. In `separate` mode, send visible text as a normal Telegram message first, then send the voice note without a caption.
