## Summary

Keep this issue open. Current main still has no configurable Telegram TTS delivery mode: shared TTS config rejects unknown delivery/caption-policy keys, auto-TTS keeps visible text and generated audio in one reply payload, and Telegram still sends normal-length voice/audio text as a caption rather than as a separate preceding message.

## Best Possible Solution

Keep this issue open. Implement a Telegram-owned delivery policy or shared messages.tts.delivery contract with bundled/separate modes, update strict config types/schema/generated docs, and add focused Telegram bot plus direct-send tests. In separate mode, Telegram should send visible text as a normal message first, then send the voice note without a caption.
