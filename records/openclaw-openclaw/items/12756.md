## Summary

Keep open. Current main still lacks Telegram support for the shared message `read` action across discovery, adapter mapping, runtime dispatch, config gating, docs, and contract tests. The related inbound-store PR is closed unmerged and the duplicate report is closed, so this issue remains the live tracker.

## What This Changes

Add Telegram support for the shared `openclaw message read` / message tool `read` action so agents can retrieve recent Telegram chat or group messages.

## Best Possible Solution

Implement Telegram `message read` inside the Telegram plugin with explicit observed-message semantics, a bounded privacy-reviewed inbound cache or equivalent runtime store, action discovery/mapping/runtime/config/docs/test coverage, and clear docs that this is not full historical chat retrieval.
