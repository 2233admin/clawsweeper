## Summary

Keep this open. Current main still has prompt guidance and a protected first-class gateway config mutation path, but the raw `exec` path does not require confirmation for model-issued OpenClaw self-management commands such as `openclaw config set`, `openclaw models set`, or `openclaw gateway restart` when gateway/node host exec resolves to `security=full` and `ask=off`.

## What This Changes

The issue requests a hard guard so agents do not automatically execute example OpenClaw config, model-switch, or gateway restart commands without explicit user confirmation.

## Best Possible Solution

Add a hard exec-boundary policy that requires explicit user-approved intent before model-initiated OpenClaw self-management commands can run through raw shell execution. Keep first-class gateway/config tools as the preferred path, add regression coverage for model-switch/example-command prompts, document the boundary, and coordinate the broader destructive-command policy with #65307.
