Keep open. Current main still does not publish a linux/riscv64 Docker release image: the Docker release workflow builds only linux/amd64 and linux/arm64, the manifest consumes only those two digests, and attestation verification checks only those two platforms. Closed PR #12033 remains useful prior art, but it is closed unmerged and no replacement target-repo PR is open.

Keep this issue open until a maintainer lands a current Docker release workflow change that adds a non-blocking linux/riscv64 build, includes the riscv64 digest in the multi-platform manifest when available, extends release validation/docs as needed, and handles Dockerfile prerequisites such as Bun and native addons on riscv64. Closed PR #12033 is useful prior art, not a canonical superseding item.
