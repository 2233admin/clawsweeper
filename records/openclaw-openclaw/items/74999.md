## Summary

Keep open: the feature is useful, but the PR still loses the new namespace scope on interactive model-list surfaces, especially Telegram, and the slash-command docs are not updated for the new drilldown syntax.

## What This Changes

The PR adds `/models <provider>/<group>` parsing, grouped output for namespaced model catalogs, scoped model listings, and colocated command tests.

## Best Possible Solution

Keep the namespace grouping behavior, but make the channel-rendering path namespace-aware or intentionally bypass interactive rendering for grouped/scoped views. The fix should include Telegram callback/pagination coverage and update the public slash-command docs for `/models <provider>/<group>`.

## Reproduction Assessment

Yes. A high-confidence static reproduction is Telegram `/models litellm` on the PR branch with several namespaced groups: the command can return provider-only interactive channel data before grouped text is built, and scoped pagination encodes only provider/page.

## Solution Assessment

No. The text fallback implementation is close, but it is not the narrowest complete solution because channels with `buildModelsListChannelData` can skip the grouped output and lose namespace scope during pagination.
