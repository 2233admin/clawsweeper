## Summary

Keep this issue open. Current main still classifies Feishu inbound routing from `chat_type` alone, so a topic/open-chat delivery reported as `chat_type: "p2p"` can still take the direct-message route. The provided GitHub context also shows open PR #52342 explicitly closes this issue, so repository policy says the issue should remain open until that PR is reviewed, landed, or replaced.

## Best Possible Solution

Keep this issue open and review PR #52342, or replace it with a focused Feishu plugin patch that derives group/topic routing for anomalous `p2p` topic/open-chat deliveries from reliable context such as verified chat metadata, `thread_id`, `root_id`, and/or safe `chat_id` semantics. The fix should update both bot routing and mention-forward classification, and add regression coverage for the reported `p2p` topic/open-chat case plus real p2p DM/card-action cases.
