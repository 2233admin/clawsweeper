## Summary

Keep this issue open. Current main still classifies Feishu topic/open-chat events solely from normalized `chat_type`, so a reported `chat_type: "p2p"` event with `oc_...` chat/thread context remains on the direct-message routing path. The supplied GitHub context also shows open PR #52342 is an unmerged implementation candidate that explicitly links this issue, so the tracker should stay open until that PR is reviewed, landed, or replaced.

## Best Possible Solution

Keep this issue open. Review and land PR #52342 if it safely handles the reported event shape, or replace it with a focused Feishu plugin patch that derives topic/group routing for p2p topic/open-chat deliveries from reliable context such as `thread_id`, `root_id`, and/or verified chat metadata. Update both bot routing and mention-forward classification, and add regression coverage for the p2p topic/open-chat case plus real p2p DM and card-action cases.
