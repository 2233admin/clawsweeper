## Summary

Keep open. Current main has iOS deep-link forwarding and the gateway `/hooks/agent` endpoint, but it does not ship a native iOS Siri/App Intent that accepts a question parameter and returns a spoken dialog. Open PR #40874 is related because it adds an `OpenTalkModeIntent`, but that PR is still open and only opens Talk Mode rather than implementing the requested “Ask OpenClaw” parameterized Siri flow.

## Best Possible Solution

Keep this issue open. The best path is for iOS maintainers to review #40874 and either land/split its Talk Mode AppIntent work, then add a focused `Ask OpenClaw` AppIntent that accepts a message, respects the existing gateway/deep-link trust policy, and returns a Siri/Shortcuts dialog when safe, with device or simulator proof.
