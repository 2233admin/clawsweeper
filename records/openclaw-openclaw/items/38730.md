## Summary

Keep open. Current main includes meaningful cron/session retention mitigation, but the requested high-frequency cron schedule guardrail is still missing from the creation and update paths, and the same-author implementation PR #38731 remains open with closing syntax for this issue.

## What This Changes

The issue requests guardrails for cron session accumulation, including cron run-session TTL cleanup, context/session pressure warnings, high-frequency schedule warnings, and stronger session cleanup behavior.

## Best Possible Solution

Keep this issue open and treat #38731 as the active implementation candidate. The best fix is a shared high-frequency `every` schedule advisory applied consistently to CLI add/edit, Gateway add/update, Control UI add/edit, and agent-tool-created cron jobs, while keeping short intervals valid and non-breaking; close this only after that PR or an equivalent fix lands with SHA and release provenance.
