## Summary

Keep this PR open. Current main still does not implement the proposed Feishu document ownership-transfer action: the `feishu_doc` schema and dispatch omit `transfer`, current Feishu permission tooling only supports list/add/remove member permissions, and docs/skills/changelog have no shipped ownership-transfer entry. The PR remains a focused Feishu plugin implementation candidate, with a security-sensitive ownership-transfer path that should get maintainer review before landing.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing after confirming Feishu ownership-transfer authorization semantics, preserving the `old_owner_perm` default regression coverage, updating the Feishu doc skill/docs and scope guidance, and rebasing/validating the focused plugin changes against current main.
