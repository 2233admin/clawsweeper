## Summary

Keep open. Current main still intentionally omits the delegated task from the first visible subagent user message and locks that behavior in tests as the #72019 token-dedup contract, so the reported UX/diagnostic gap remains real but needs maintainer prompt-contract judgment before changing.

## What This Changes

The issue requests that native subagent spawns include the delegated task text in the first visible user message as an auditable fallback while still keeping the task in the subagent system prompt.

## Best Possible Solution

Define the subagent task placement contract explicitly. The maintainable end state should either expose the delegated task through channel/transcript diagnostics without sending a second full task copy to the model, or deliberately move the canonical assignment into the visible first user turn and update the source, tests, changelog/docs, and token-cost expectations together.

## Reproduction Assessment

Yes. Source and test inspection on current main is a high-confidence reproduction: the child `message` omits the delegated task, `extraSystemPrompt` carries it under `## Your Role`, and tests assert that the task appears only once across the model payload.

## Solution Assessment

Unclear. The suggested patch solves the visible transcript fallback, but it duplicates the task into the first model-visible user message while retaining the system prompt copy; a transcript-only diagnostic surface may be safer unless maintainers choose to revise the #72019 prompt contract.
