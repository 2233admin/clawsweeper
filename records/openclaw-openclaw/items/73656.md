## Summary

Keep open. Current main still has the reported visible-transcript behavior, and tests/changelog now enforce it as the #72019 single-source prompt contract, so this is a real UX/auditability gap that needs maintainer direction rather than conservative cleanup closure.

## What This Changes

Include the delegated task text in the first visible native subagent user message as a diagnostic fail-safe while keeping the canonical task in the subagent system prompt.

## Best Possible Solution

Define the subagent task visibility contract explicitly: either add a transcript/channel diagnostic that exposes the assignment without duplicating model input, or intentionally move/change the canonical prompt placement with source, tests, docs, and changelog updated together.

## Reproduction Assessment

Yes. Read-only source and test inspection on current main gives a high-confidence reproduction path: spawn builds the task into `extraSystemPrompt`, while the first visible `message` is produced by a helper that cannot receive task text and tests assert that omission.

## Solution Assessment

Unclear. The suggested patch is localized, but it conflicts with the shipped #72019 token-dedup contract; a transcript-only diagnostic or maintainer-approved prompt-contract change may be safer than simply adding duplicate task text to the visible user turn.
