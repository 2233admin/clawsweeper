## Summary

Keep open. Current main still has the reported visible-transcript behavior, but source, regression tests, and changelog show that behavior is the intentional #72019 single-source prompt contract, so changing it needs maintainer prompt-contract judgment rather than conservative cleanup.

## What This Changes

Request to include delegated native subagent task text in the first visible kickoff message so channel transcripts are auditable even when the canonical task remains in the system prompt.

## Best Possible Solution

Define the subagent task visibility contract explicitly: either add an audit-friendly transcript/channel diagnostic that does not duplicate model-visible tokens, or deliberately copy or move the task into the kickoff message and update source, tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the symptom: buildSubagentInitialUserMessage omits the task, while buildSubagentSystemPrompt places the delegated task under ## Your Role and tests assert that single-source behavior.

## Solution Assessment

Unclear. The suggested local patch is narrow, but it is not clearly the best fix because it reverses the shipped #72019 dedupe contract; a transcript-only diagnostic may satisfy auditability with less token and visibility impact.
