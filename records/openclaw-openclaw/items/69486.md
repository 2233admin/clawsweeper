## Summary

Keep this PR open. Current main still has the branch-and-reappend transcript rewrite behavior without the PR's ancestry-aware on-disk cleanup or leaf-branch regression coverage, so the work is not obsolete. The PR is a plausible implementation candidate with useful production-derived repro context, but it still needs maintainer review/update because the cleanup currently guards `parentId` ancestry while review comments correctly point out other load-bearing references such as `label.targetId`.

## Best Possible Solution

Keep this PR open and revise the cleanup so it removes only entries abandoned by the specific rewrite while preserving every load-bearing reference from surviving entries, not only `parentId` ancestry. The likely fix belongs in `src/agents/pi-embedded-runner/transcript-rewrite.ts` with focused regression tests for linear rewrites, repeated rewrites, sibling branches, label target references, compaction keep markers, and the production-derived lineage shape. Keep compaction-chain reader/reset semantics in #68765 or another narrower follow-up.
