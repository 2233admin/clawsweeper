## Summary

Keep this PR open. Current main has partial mitigation for abandoned rewrite entries by reading active transcript branches in chat history, but it does not implement this PR's on-disk ancestry-aware cleanup, and the PR still has a material data-integrity concern around non-parent ID references such as label.targetId.

## What This Changes

This PR adds transcript rewrite cleanup that snapshots active branch IDs before and after a rewrite, removes abandoned JSONL entries after a parentId ancestry filter, and adds leaf-branch plus production-lineage regression coverage.

## Best Possible Solution

Keep this PR open and revise the cleanup to preserve every load-bearing entry ID reference from surviving entries, not only parentId ancestry. The fix should stay in the runner transcript rewrite boundary, rebase against current main's active-branch history reader and compaction duplicate pruning, and add regression coverage for label.targetId plus existing sibling, repeated rewrite, compaction keep-marker, and production-lineage cases. Keep compaction-chain reader/reset work in #68765 or another narrow follow-up.
