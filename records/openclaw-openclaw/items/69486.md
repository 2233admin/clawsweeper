## Summary

Keep PR #69486 open. Current main still rewrites transcripts by branching to the first replaced entry's parent and re-appending the suffix, while the gateway history reader still scans every JSONL message row without branch filtering. The PR is a focused, recent bug-fix attempt with production-derived repro evidence, not a duplicate of #68765 and not implemented on main yet. It also has real review feedback around non-parent references such as label targetId that should be resolved before merge, rather than closing the PR.

## Best Possible Solution

Keep the PR open for maintainer review and require a final patch that centralizes ancestry-aware cleanup for all file-backed rewrite paths, preserves non-parent references such as label.targetId and branch_summary metadata, and lands the leaf-branch plus production-derived regression tests. Keep #68765 separate because it tracks compaction-checkpoint history reading, not raw rewrite debris cleanup.
