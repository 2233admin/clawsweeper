## Summary

Keep open. Current main still makes `/subagents list` depend on a registry snapshot filtered by an exact requester/controller session key, and I did not find source, test, release, or discussion proof that the reported empty-list plus parent-completion symptom is fixed. The issue has concrete post-release follow-up, and the adjacent subagent fixes in the changelog do not specifically cover this read-path failure.

## What This Changes

The issue reports a 2026.4.23 regression where `/subagents list` returns no active or recent runs even though gateway logs show active/completed subagents, with follow-up evidence that parent completion delivery may also miss finished subagents.

## Best Possible Solution

Keep this issue open until maintainers add targeted diagnostics and a regression test for the full boundary: spawn/register a subagent, verify the same requester key can see it through `/subagents list`, and verify completion delivery reaches the parent. The fix should land in the command requester-key wiring, registry persistence/read path, or packaging/module-instance boundary that the test exposes.
