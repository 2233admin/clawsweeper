## Summary

Keep open: the PR addresses a real subagent-targeting reliability problem, but the proposed shared schema/runtime requirement still regresses the documented ACP default-agent fallback path before ACP dispatch can resolve it.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the shared tool schema/runtime reader and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving native subagent target reliability through the existing `requireAgentId` configuration, clearer runtime-specific guidance, or an additive runtime-specific schema path that only requires `agentId` where omission is not a supported contract.
