## Summary

Keep open. Current main has not implemented this PR’s intended change: `sessions_spawn.agentId` is still optional in the tool schema and execution path, and tests still cover omitted-agentId behavior. Main does have a config-level `agents.defaults.subagents.requireAgentId` path that addresses the related strict-hierarchy issues, but landing this PR as-is would change ACP behavior that current code, docs, and prompt guidance still support via `acp.defaultAgent`. The PR also changes runtime validation to require `agentId`, not only the schema, so it needs maintainer product/compatibility review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review, but do not land it unchanged. The best path is to preserve ACP `acp.defaultAgent` compatibility while improving native subagent explicit-target reliability, likely by building on the shipped `requireAgentId` config path, tool-description/prompt guidance, or a runtime-specific schema/tool split rather than making `agentId` globally required.
