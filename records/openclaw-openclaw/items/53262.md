## Summary

Keep open. The reliability goal is valid, but current main still intentionally supports omitted `agentId` for ACP via `acp.defaultAgent`, while native subagents already have a configurable `requireAgentId` gate. The PR’s unconditional required reader/schema would regress that shared ACP/native contract and needs maintainer direction before merge.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the tool schema/runtime reader and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit target reliability for native subagents, either by strengthening the existing `requireAgentId` config/prompt path or by introducing runtime-specific schema behavior so only the paths that cannot safely omit `agentId` require it.
