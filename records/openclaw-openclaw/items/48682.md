## Summary

Keep this PR open. Current main still has the reported GitHub Copilot auth-order gap: implicit Copilot discovery still goes through a helper that picks the first stored profile, while the order-aware auth helper exists but is not used on that path. The PR is useful conceptually for #46031, but its patch is stale against the current plugin-owned discovery layout and has an unresolved plugin-disablement concern.

## Best Possible Solution

Keep the PR open and ask for a rebase/rework against current main: implement order-aware Copilot token selection in `extensions/github-copilot/auth.ts` or via the existing provider-auth SDK helper, keep env-token precedence, iterate/fall through unresolved ordered profiles, add focused regression coverage for `auth.order` and stale/unresolved fallbacks, and preserve plugin discovery disablement.
