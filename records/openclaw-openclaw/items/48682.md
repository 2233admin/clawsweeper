## Summary

Keep this PR open. Current main still does not implement the intended behavior: implicit GitHub Copilot discovery uses the plugin-owned `resolveFirstGithubToken` path, and that resolver still selects `profileIds[0]` without consulting `auth.order`. The PR remains useful, but its patch is stale against the current plugin catalog layout and needs a rebase/rework rather than cleanup closure.

## Best Possible Solution

Keep this PR open as an implementation candidate, but ask for a rebase/rework against current main. The fix should live in `extensions/github-copilot/auth.ts` and the plugin catalog path, use the existing order-aware auth-profile seam, preserve env-token precedence plus explicit provider config and plugin discovery disablement, and add focused regression coverage for ordered profiles, unresolved ordered tokenRefs, stale order handling, and discovery/plugin disablement.
