## Summary

Keep open. Current main still returns `null` when macOS default Chromium browser detection recognizes Edge's LaunchServices bundle ID but `osascript` or `defaults` cannot resolve the app path, which then lets the generic fallback choose Chrome first. The PR is a focused and directionally correct fix, but it is currently dirty against main and lacks regression coverage for the failure branch it is meant to change.

## What This Changes

The PR adds a macOS known bundle-ID-to-executable fallback in `extensions/browser/src/browser/chrome.executables.ts`, including system and user Applications paths, when dynamic default-browser path resolution fails.

## Best Possible Solution

Land a rebased version of this browser-plugin fallback with focused macOS tests covering Edge known-path success after failed dynamic lookup and missing Edge known-path fallback to the ordinary Chrome-first candidate list.
