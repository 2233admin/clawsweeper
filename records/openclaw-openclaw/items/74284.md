## Summary

Keep open. Current main still allows heartbeat runs without an explicit `heartbeat.model` to reuse stored session model overrides, and the linked implementation PR is still open and unmerged, so closing this issue now would be premature.

## What This Changes

The issue asks for heartbeat sessions to stop pinning an old auto-fallback model after `agents.defaults.model.primary` changes across gateway restarts.

## Best Possible Solution

Review, revise if needed, and land or supersede PR #74932 with a narrow heartbeat/default-model-change rule that clears only stale direct auto-fallback overrides and documents the behavior.

## Reproduction Assessment

Yes. Seed an `agent:main:main` session entry with `providerOverride`, `modelOverride`, and `modelOverrideSource: "auto"` from an older default, then run heartbeat selection without `heartbeat.model` after changing the configured default; current main applies the stored override instead of the new default.

## Solution Assessment

No for current main. The maintainable direction is a targeted fix on the linked PR path, but the current shipped code still preserves the stale stored override behavior.
