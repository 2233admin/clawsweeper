## Summary

Keep open. Current main still applies stored session model overrides on heartbeat runs without an explicit `heartbeat.model`, and the linked fix PR is open and unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks heartbeat runs for `agent:main:main` to stop reusing an auto-persisted fallback model after `agents.defaults.model.primary` changes across gateway restarts.

## Best Possible Solution

Land or supersede #74932 with a targeted heartbeat/default-model-change rule that clears only stale direct auto fallback overrides while leaving the broader session override contract intact.

## Reproduction Assessment

Yes. The source-level reproduction is to create an `agent:main:main` session entry with `modelOverrideSource: "auto"` from an older default, then run heartbeat model selection without `heartbeat.model`; current main will apply that stored override instead of the new default.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: detect stale direct auto-sourced heartbeat overrides when the recorded selected model no longer matches the current default, then clear or skip only that override. The issue should stay open while the linked PR is reviewed because the fix has not merged.
