## Summary

Keep open. Current main still lets heartbeat runs without an explicit heartbeat model apply stored session model overrides, and the targeted PR that claims to fix this issue is open and unmerged, so closing now would be premature.

## What This Changes

The issue asks for heartbeat sessions to stop pinning an old auto fallback model after `agents.defaults.model.primary` changes across gateway restarts.

## Best Possible Solution

Land or supersede PR #74932 with a narrow heartbeat-only stale auto fallback override rule, backed by focused model-selection tests and docs that explain the default-change behavior.

## Reproduction Assessment

Yes. Seed an `agent:main:main` session entry with a direct `modelOverrideSource: "auto"` override from an older default, then run heartbeat selection without `heartbeat.model` after changing the configured default; current main applies the stored override instead of the new default.

## Solution Assessment

Yes in direction, but it is not on current main yet. The narrow maintainable fix is to clear only stale direct heartbeat auto fallback overrides while preserving explicit heartbeat/user/legacy/parent override semantics.
