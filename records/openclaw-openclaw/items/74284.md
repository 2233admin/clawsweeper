## Summary

Keep open. Current main still applies stored session model overrides on heartbeat runs without an explicit `heartbeat.model`, including auto-fallback overrides, and the linked closing PR #74932 is still the appropriate implementation path rather than a reason to close early.

## What This Changes

The issue requests that heartbeat sessions stop reusing a persisted auto-fallback model override after `agents.defaults.model.primary` changes, so later heartbeat runs follow the updated default unless an explicit heartbeat or user override applies.

## Best Possible Solution

Land or supersede #74932 with a targeted heartbeat/model-selection rule that clears only stale direct auto-fallback overrides after a configured default change while preserving the broader session override contract.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to seed `agent:main:main` with a direct `modelOverrideSource: "auto"` fallback override from an older default, run heartbeat selection without `heartbeat.model`, and observe current main applying the stored override instead of the updated default.

## Solution Assessment

Yes in direction, but not yet on current main. The narrow maintainable solution is a heartbeat-specific stale-auto-override check, not a broad change to all stored session override behavior.
