## Summary

Keep open. Current main still lets heartbeat runs without an explicit `heartbeat.model` apply the stored session model override, and the open paired implementation PR #74932 uses closing syntax for this issue but has not merged, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks heartbeat turns without explicit `heartbeat.model` to stop reusing stale session-stored auto fallback model overrides after `agents.defaults.model.primary` changes.

## Best Possible Solution

Land or supersede #74932 with a narrow heartbeat/default-model-change rule that clears only stale direct auto fallback overrides while preserving the broader session override contract.

## Reproduction Assessment

Yes. The source-level reproduction is to seed an `agent:main:main` session entry with a stored auto model override from an older default, then run heartbeat model selection without `heartbeat.model`; current main applies that stored override instead of the new configured default.

## Solution Assessment

Yes in direction, but not yet on main. The narrow maintainable fix is to detect stale direct heartbeat auto fallback overrides after the default changes and clear or skip only that direct stale override, leaving user, legacy, parent, and unchanged-default behavior intact.
