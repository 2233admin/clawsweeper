## Summary

Keep open. Current main does not implement the proposed hook agent metadata length limits, and this PR still contains useful security hardening work. It should not be closed as obsolete, but the patch needs changes because it only validates direct `/hooks/agent` payloads while mapped hook agent dispatches can still render and dispatch oversized metadata fields.

## Best Possible Solution

Keep this PR open for maintainer review and revise it into a shared hook-agent metadata validator used by both direct `/hooks/agent` normalization and mapped `/hooks/<name>` agent dispatch. Add targeted tests for direct and mapped over-limit rejection, exact-boundary acceptance including `agentId` and `sessionKey`, and no regression to the 256 KB body limit behavior.
