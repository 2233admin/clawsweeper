## Summary

Keep this PR open. Current main still normalizes explicit heartbeat targets before plugin-only channel IDs can be bootstrapped, so the generic load-order gap described by the PR remains. Telegram itself is less representative now because it is a bundled channel, but external/plugin-only heartbeat targets are still documented and validated.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a focused successor. The fix should bootstrap or read configured plugin channel metadata before deliverable-channel recognition, retry normalization after registry initialization, and add regression coverage for `agents.defaults.heartbeat.target` pointing at a plugin-only channel while the active registry is initially empty. Keep the bootstrap side effect narrow and avoid unrelated workflow, dependency, or release changes.
