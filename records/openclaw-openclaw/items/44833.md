## Summary

Keep open. Current main still appears to skip the reported Telegram path: captionless image messages are converted to a media placeholder and have media paths available, but group mention matching is computed from Telegram text/caption only, so a photo with no caption cannot satisfy a catch-all mention pattern before the requireMention gate drops it.

## What This Changes

Request to make Telegram group photo-only messages trigger image understanding consistently when the group is configured to allow the bot to respond.

## Best Possible Solution

Update Telegram group activation so media-only image messages are eligible for the same configured activation policy as captioned images, while keeping ordinary unmentioned group media silent by default. Add regression coverage that a captionless photo with an intentional catch-all mention policy reaches the agent with `MediaPaths`, and that normal bot-name mention patterns still do not auto-trigger unrelated media-only group messages.

## Repair Work Prompt

Fix the Telegram group captionless-photo activation gap reported in the current issue. In current main, `resolveTelegramInboundBody` builds `<media:image>` and keeps `MediaPaths`, but computes `wasMentioned` from Telegram text/caption only; with a captionless photo and `messages.groupChat.mentionPatterns: [".*"]`, `matchesMentionWithExplicit` sees empty text and the requireMention gate skips the turn before image understanding can run.

Expected boundary: change Telegram inbound mention matching so media-only image messages can satisfy intentionally broad configured activation policy without requiring a caption, while preserving default group safety. Do not make all captionless group media trigger for ordinary bot-name mention patterns, and do not change Telegram DM access, media download limits, or group allowlist semantics. Prefer the smallest Telegram-owned change; only adjust the shared mention helper if the API shape stays backward-compatible and all current callers remain safe.

Add focused regression coverage, likely in `extensions/telegram/src/bot-message-context.body.test.ts` or the Telegram bot harness, for: a captionless group photo with downloaded image media, `requireMention: true`, and `mentionPatterns: [".*"]` reaches the agent with `effectiveWasMentioned: true` / media paths; a captionless group photo with a normal bot-name pattern remains skipped. Add a user-facing changelog Fixes entry with `Thanks @h12n0924` if behavior changes. Validate with targeted Telegram tests and formatter checks; use the repo changed-gate policy before handoff.
