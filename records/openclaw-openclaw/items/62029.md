## Summary

Keep this open. Current main still has threshold-only deep promotion: candidates below `minScore` are dropped during ranking and rejected again before writing to `MEMORY.md`. The related dreaming fixes improve daily/phase reinforcement, but they do not add wildcard or sub-threshold exploration candidates.

## What This Changes

Request to add a bounded wildcard sampling path so deep dreaming can occasionally promote eligible but sub-threshold short-term memory candidates.

## Best Possible Solution

Decide whether deep dreaming should have an explicit exploration lane. If accepted, implement it in `memory-core` as a bounded, testable policy with deterministic or seedable selection, visible candidate metadata, aligned apply-time validation, and docs for any new config surface.

## Reproduction Assessment

Not applicable as a feature request, but current behavior is high-confidence from source inspection: below-threshold candidates are filtered in ranking and apply. A focused test could create a candidate that passes signal/diversity gates but scores below `minScore` and confirm it is absent.

## Solution Assessment

Unclear. A raw rank-only `wildcardRatio` is probably not the best final shape because `applyShortTermPromotions` rechecks `minScore`, and random sampling could conflict with deterministic promotion expectations; a bounded deterministic exploration lane would be safer if maintainers want this behavior.
