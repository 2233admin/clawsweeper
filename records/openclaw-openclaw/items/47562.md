## Summary

Keep this PR open. Current main still lacks Telegram custom emoji reaction support, but the PR is not merge-ready because it targets obsolete current-main paths and the action-layer custom-only call remains blocked by existing emoji validation.

## What This Changes

The PR adds Telegram custom emoji reaction support for inbound reaction updates, outbound reaction sending, Telegram action plumbing, and focused bot/send tests.

## Best Possible Solution

Port the reaction feature into current main's Telegram plugin surfaces: add `customEmojiId` to the shared reaction schema and parser contract, accept either `emoji` or `customEmojiId` for add calls, send Telegram `custom_emoji` reaction payloads from `reactMessageTelegram`, emit inbound custom emoji reaction system events after existing gates, and update focused tests plus user-facing docs.

## Reproduction Assessment

Yes. The PR bug is reproducible by calling the proposed Telegram react action with `chatId`, `messageId`, and `customEmojiId` but no `emoji`; `readReactionParams` still requires the `emoji` key before the PR reads `customEmojiId`. Current-main verification also shows no custom emoji schema, send option, or inbound handling.

## Solution Assessment

No. The feature direction is valid, but this patch is not the best merge path because it targets removed current-main files and misses the shared parser/schema changes needed for custom-only action calls; a narrow replacement port to the current runtime files is safer.
