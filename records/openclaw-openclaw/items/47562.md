## Summary

Keep this PR open, but it should not merge as-is. Current main still lacks Telegram custom emoji reaction support, while the PR is stale against renamed/current Telegram files and still leaves the action-layer custom-only flow blocked by required `emoji` validation.

## What This Changes

The PR adds Telegram custom emoji reaction handling for inbound `message_reaction` updates and outbound reaction sends/actions, plus bot and send tests.

## Best Possible Solution

Port the narrow feature into the current Telegram plugin surfaces: add a `customEmojiId` reaction input to the shared tool schema and Telegram action runtime, allow custom-only reaction calls without requiring `emoji`, send Telegram `custom_emoji` reaction payloads from `reactMessageTelegram`, emit inbound custom emoji reaction system events after the existing gates, and document the Telegram-specific behavior.
