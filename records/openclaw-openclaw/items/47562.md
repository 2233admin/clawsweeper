## Summary

Keep this PR open. Current main still lacks Telegram custom emoji reaction support, and the PR is useful source work, but it is not merge-ready because it targets stale Telegram files and its action-layer parsing still rejects the advertised custom-only `customEmojiId` flow before the send helper can use it.

## What This Changes

The PR adds inbound Telegram custom emoji reaction events, outbound `customEmojiId` reaction payload support, Telegram action plumbing, and bot/send tests.

## Best Possible Solution

Port the narrow reaction feature into the current Telegram plugin surfaces: add `customEmojiId` to the shared `message` tool reaction schema and parser, allow either `emoji` or `customEmojiId` for add calls, send Telegram `{ type: "custom_emoji", custom_emoji_id }` payloads from `reactMessageTelegram`, emit inbound custom emoji reaction system events after the existing notification and authorization gates, and update focused tests plus user-facing docs.
