## Summary

Keep open. The underlying Feishu multi-bot NO_REPLY bug is not fixed on current main, but this PR is not merge-ready because its preserve-all approach regresses the direct-chat slash-command path; the narrower open #72661 replacement better matches the existing command boundary.

## What This Changes

The PR changes Feishu mention normalization to preserve the receiving bot's own `<at>` tag everywhere and updates Feishu tests/comments to expect preserved bot self-mentions.

## Best Possible Solution

Land a narrow Feishu plugin fix that strips only the leading addressing self-mention while preserving inline or mid-sentence self-mentions for multi-bot group context, with regression coverage for the multi-bot NO_REPLY case plus direct and group slash commands.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: with this PR, a direct `@_bot_1 /model` event normalizes to a leading `<at>` tag, and the direct command path does not strip that tag before requiring slash-leading command normalization.

## Solution Assessment

No. Making `botStripId` a global no-op is broader than the maintainable fix; the safer boundary is to preserve semantic inline self-mentions while keeping addressing prefixes clean for command dispatch.
