## Summary

This PR should stay open. Current main and the latest shipped release still await provider auth resolution before the existing provider-usage timeout boundary, so the PR is not obsolete; the remaining question is maintainer review of the proposed non-cancelling timeout and JSON-command stderr warning behavior.

## What This Changes

This PR wraps provider usage auth discovery in `withTimeout` inside `src/infra/provider-usage.load.ts` and adds a loader test for a never-settling auth resolver returning an empty provider list.

## Best Possible Solution

Keep this PR open for focused maintainer review. Validate the #74085 non-TTY subprocess repro on current main, then either land a narrow provider-auth timeout with regression coverage and accepted diagnostic behavior, or adjust the fix so auth discovery is cancellable or per-provider bounded if lingering work can still keep the CLI alive.
