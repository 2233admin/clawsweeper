## Summary

Keep this issue open. Current main has useful adjacent memory work, especially optional Active Memory and improved memory-core recall guidance, but it still does not provide a default deterministic project/task/focus recall contract for the reported `openclaw agent --session-id` workflow, and the long-session-id prompt-cache-key path still forwards raw session ids into OpenAI transport without a visible validation/normalization layer.

## Best Possible Solution

Keep this open as maintainer-owned work. The best path is to split it into concrete follow-ups: add a repo-contained memory/task recall gate, decide and document whether OpenClaw owns a canonical project/task/focus workspace-state contract, make default recall prefer current canonical state over stale semantic hits where such state exists, define open-task retrieval semantics for any OpenClaw-owned task surface, and validate or normalize provider prompt-cache/session keys before OpenAI transport.
