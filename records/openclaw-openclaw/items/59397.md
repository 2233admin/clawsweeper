## Summary

Keep open. Current main still has the nodes CLI timeout validation gap: the runtime path for `openclaw nodes` still converts `opts.timeout` with raw `Number(...)` before calling the Gateway, and there is no focused nodes CLI regression test for rejecting invalid `--timeout` values. The PR remains plausibly useful, but it needs adaptation to the current `rpc.ts` / `rpc.runtime.ts` split and should address the review comment about `parseInt`-style partial parsing before merge.

## Best Possible Solution

Keep the PR open and request an adapted patch for current main: validate `openclaw nodes --timeout` on the actual runtime path, add the focused invalid-timeout regression test, preserve existing command defaults, and either tighten the shared parser semantics or add a strict nodes-specific wrapper so partial numeric strings are not silently reinterpreted.
