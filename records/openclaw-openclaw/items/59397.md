## Summary

Keep this PR open. The nodes CLI timeout bug is still present on current main and in the latest release, but this branch is stale against the current runtime split and its parser switch would still accept partial/scientific strings through `parseInt`; it also needs the required changelog entry before merge.

## What This Changes

The PR replaces nodes CLI gateway timeout parsing in `src/cli/nodes-cli/rpc.ts` with `parseTimeoutMsWithFallback(..., { invalidType: "error" })` and adds a regression test for `--timeout 0`.

## Best Possible Solution

Adapt the fix to current main by validating `opts.timeout` in `src/cli/nodes-cli/rpc.runtime.ts`, tightening the shared timeout helper or adding an equivalent whole-string positive-integer parser, covering invalid and valid cases in focused tests, and adding a single active changelog fix entry crediting the contributor.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main passes `Number(opts.timeout ?? 10_000)` into `callGateway`, and `callGateway` treats non-finite values as unset while the proposed helper would reinterpret `1e3` and `123abc` through `parseInt`.

## Solution Assessment

No. The direction is correct, but this exact patch is not the best fix until it is moved to the current runtime helper and rejects partial or scientific timeout strings instead of reinterpreting them.
