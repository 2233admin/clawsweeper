## Summary

Keep this PR open. Current main still rejects `message` tool `action: "send"` calls when non-empty poll creation params are present, so the PR is not obsolete; it is an open implementation candidate that needs maintainer review, a rebase onto current main, and coordination with the broader overlapping PR #46455.

## What This Changes

The PR strips poll-creation fields from `message` tool `action: "send"` params instead of throwing, adds a poll-param stripping helper, and updates focused poll/send tests to expect the send path to proceed.

## Best Possible Solution

Land a current-main implementation that strips stray poll-creation params only for `action: "send"`, preserves all `action: "poll"` validation and plugin poll dispatch behavior, remains symmetric with today's poll-param detection including snake_case and channel-specific `poll*` keys, and coordinates the overlap with #46455 rather than leaving parallel fixes to drift.
