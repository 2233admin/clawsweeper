Keep open. The report is fresh, concrete, and still lacks current-main macOS direct-install timing proof against LM Studio; current main contains several adjacent latency mitigations, but none proves this specific 10-15s per-message pre-dispatch delay is fixed for both WebChat and Telegram.

## What This Changes

The issue reports a regression where WebChat/UI and Telegram messages on macOS direct install wait 10-15 seconds before OpenClaw dispatches normal turns to an LM Studio OpenAI-compatible endpoint.

## Best Possible Solution

Keep this issue open until current main or the next release is tested on macOS direct install with LM Studio.
