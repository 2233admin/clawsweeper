## Summary

Current main still has a sanitizer gap for the reported workflow leak shape. The shared assistant-visible sanitizer strips JSON-looking `<function_calls>` blocks, but plural XML-style `<function_calls><invoke ...>` remains preserved by both source logic and regression tests, and there is no `<function_response>` handling. Because Telegram message-end delivery uses the same visible extraction path, the issue remains valid and narrow enough for a focused fix PR.

## Best Possible Solution

Keep this open and land a focused sanitizer fix. Extend the shared assistant-visible text sanitizer to strip standalone/internal-looking plural XML `<function_calls>` blocks with nested `<invoke>` payloads and paired `<function_response>` blocks, add regression coverage in the shared sanitizer plus one agent/Telegram-facing delivery path test, and preserve fenced code plus literal inline prose examples.
