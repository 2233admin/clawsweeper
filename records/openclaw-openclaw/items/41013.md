## Summary

Keep open. Current main still does not strip leading `<session-recap>` / `<session_recap>` blocks from the shared inbound user-message sanitizer, and the linked issue remains open. The PR is useful source work, but the proposed helper has two concrete correctness bugs before it should merge.

## What This Changes

The PR adds a session-recap fast path, a `stripLeadingSessionRecapBlock` helper, and colocated tests for removing leading session recap blocks from inbound user-visible messages.

## Best Possible Solution

Land a rebased sanitizer fix on current main that strips only complete leading `<session-recap>` / `<session_recap>` blocks in both full and leading-only display paths, preserves malformed and mid-message literals, keeps current timestamp and active-memory behavior, and adds regression coverage for malformed/no-close and whitespace-preservation cases before resolving #40948.
