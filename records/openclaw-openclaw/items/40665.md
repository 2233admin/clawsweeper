## Summary

Keep open. Current main still routes Discord voice autoJoin through a READY-only listener, has no RESUMED listener/export/registration, and calls @discordjs/voice connection.destroy() without a destroyed-state guard in the relevant cleanup paths. The issue is also paired with open PR #40676 using closing syntax, so it should stay open until that PR is reviewed, ported/replaced for the current extension paths, merged, or closed.

## What This Changes

The issue requests reliable Discord voice autoJoin after health-monitor account restarts by handling RESUMED events and stale or already-destroyed @discordjs/voice connections.

## Best Possible Solution

Keep this as the canonical Discord voice reliability report. The fix should live in extensions/discord: trigger voice autoJoin on READY and RESUMED, clean up or ignore stale destroyed @discordjs/voice connections before joining or destroying, preserve same-guild and multi-account behavior, and add focused regression tests for RESUMED autoJoin plus stale destroyed-connection cleanup.
