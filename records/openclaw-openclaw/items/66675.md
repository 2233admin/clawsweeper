## Summary

Keep open. Current main still has the two implicated surfaces: a completed Linux/systemd service restart is followed by a health check that can make the command return failure, and expired approval followup dispatch errors are still logged as warnings. The same-author PR #66685 remains open and explicitly scopes itself to warning-noise cleanup, not the full false non-zero restart behavior.

## What This Changes

Make `openclaw gateway restart` on Linux/systemd report success when the service restart completes and the gateway is healthy afterward, while avoiding benign expired exec approval followup warning noise.

## Best Possible Solution

Keep the issue open until maintainers land a targeted fix or regression proof for the restart exit status. A complete fix should make the post-restart path treat an actually healthy gateway as success, and may separately adopt the existing approval-not-found classifier to suppress benign expired approval followup noise.
