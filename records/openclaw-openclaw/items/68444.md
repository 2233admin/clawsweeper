## Summary

Keep open. Current main still hardwires Feishu mention forwarding into both agent prompt construction and outbound reply dispatch, and it has no `mentionForwardMode` config seam. This PR directly matches the remaining Feishu-specific work from #68357, with a Feishu-local docs/tests/runtime diff and no unexplained security-sensitive surface in the reviewed file list.

## What This Changes

The PR adds a Feishu `mentionForwardMode` config at shared/account/group scope, threads it through reply policy resolution, uses it to suppress prompt hints and outbound forwarded mentions when disabled, and updates Feishu docs plus schema/policy/bot tests.

## Best Possible Solution

Review and either merge this Feishu-local `mentionForwardMode` implementation or a direct successor after rebase, preserving the default `auto` behavior, the explicit `none` opt-out, docs coverage, and schema/policy/bot regression tests.
