## Summary

Keep this open. Current main still has a plausible, source-backed Telegram-versus-WebChat divergence for skill-backed tool availability: Telegram can pass DM/group/topic skill filters into the reply path, including an explicitly empty filter that removes all skills, while WebChat `chat.send` dispatches as internal `webchat` without a channel skill filter. I found no merged fix, canonical duplicate, or maintainer resolution for the reported vLLM + himalaya mismatch.

## Best Possible Solution

Keep the issue open and reproduce on current main using the same agent/session, vLLM `gpt-oss-20b` model, himalaya skill, and Telegram route. Capture Telegram DM/group/topic config, `replyOptions.skillFilter`, merged skill filter, `skillsSnapshot`, effective runtime tools, provider/model metadata, and structured-versus-text tool-call events for both WebChat `chat.send` and Telegram. If Telegram is unintentionally losing himalaya through route skill filtering, fix Telegram filter inheritance or diagnostics and add a Telegram dispatch regression test; if vLLM emits text-only or empty structured tool calls, document the exact provider/model setup and diagnostics.
