## Summary

Keep this open. Current main still has a source-backed Telegram/WebChat divergence for skill-backed tool availability: Telegram group/topic configuration can pass an explicit skill filter, including `[]` meaning no skills, into the reply path, while WebChat `chat.send` dispatches on the internal `webchat` surface without a channel skill filter. I found no relevant code diff since the prior ClawSweeper review and no supplied maintainer resolution, merged fix, or canonical duplicate for the vLLM + himalaya mismatch.

## Best Possible Solution

Keep the issue open and reproduce on current main using the same agent/session, vLLM `gpt-oss-20b` model, himalaya skill, and Telegram route. Capture Telegram DM/group/topic config, `replyOptions.skillFilter`, merged skill filter, `skillsSnapshot`, effective runtime tools, provider/model metadata, and structured-versus-text tool-call events for both WebChat `chat.send` and Telegram. If Telegram is unintentionally losing himalaya through route skill filtering, fix Telegram filter inheritance or diagnostics and add a Telegram dispatch regression test; if vLLM emits empty or text-only tool calls, document the exact provider/model setup and diagnostics.
