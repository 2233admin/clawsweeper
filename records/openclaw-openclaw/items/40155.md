## Summary

Keep this issue open. Current main still has a credible Telegram-versus-WebUI divergence path for skill-backed tool availability: Telegram can apply DM/group/topic skill filters, including an explicitly empty filter, while WebUI `chat.send` uses the internal webchat path without a channel skill filter. The provided discussion and current checkout do not show a merged fix, maintainer confirmation, or canonical duplicate that resolves the reported vLLM + himalaya mismatch.

## Best Possible Solution

Keep this issue open and reproduce on current main with the same agent/session, vLLM `gpt-oss-20b` model, himalaya skill, and Telegram route. Capture Telegram DM/group/topic config, `replyOptions.skillFilter`, merged skill filter, `skillsSnapshot`, effective runtime tools, provider/model metadata, and structured-versus-text tool-call events for both WebUI `chat.send` and Telegram. If Telegram is unintentionally losing himalaya through skill filtering, fix Telegram filter inheritance or diagnostics and add a Telegram dispatch regression test; if vLLM emits text-only or empty structured tool calls, document the exact provider/model setup and diagnostics.
