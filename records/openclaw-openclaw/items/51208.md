## Summary

Keep this issue open. Current main still hardcodes hook-dispatched agent jobs as isolated, mapped hook config still has no continuation/sessionTarget field, and the CLI resume path still suppresses stored CLI session IDs when the cron session is new. The GitHub context also shows an open fork PR that says it closes this issue, so the maintainer path is to review, land, or close that candidate before resolving this report.

## Best Possible Solution

Keep this issue open and review the open implementation candidate. The concrete product/code path is to add a supported mapped-hook continuation control such as `hooks.mappings[].sessionTarget`, plumb it through config types, zod schema, generated help, mapping normalization, transform overrides, hook replay scope, and dispatch, then add regression coverage proving stable hook sessions with stored `cliSessionIds.codex-cli` invoke CLI resume while default isolated cron and hook jobs still start fresh and clear stale identity.
