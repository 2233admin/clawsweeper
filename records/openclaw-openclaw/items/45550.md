## Summary

Keep open. Current main and the latest release still carry the pre-GA Anthropic 1M beta-header behavior in code, tests, and docs; open PR #45613 remains the active unmerged implementation candidate, so this should stay open until maintainers land, refresh, or replace that PR.

## What This Changes

Migrate Anthropic 1M context handling from beta to GA by stopping the retired beta-header and OAuth skip behavior while preserving `context1m` as an explicit opt-in and updating docs/tests.

## Best Possible Solution

Review, refresh, and land PR #45613 or a narrow replacement that removes emission of the retired `context-1m-2025-08-07` beta, preserves required Anthropic OAuth betas, keeps `context1m` as the explicit context-window sizing opt-in, and updates the Anthropic provider/token-use/troubleshooting docs plus focused wrapper coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure `context1m: true` for an eligible Anthropic Opus/Sonnet model and the active wrapper still emits the retired beta for API-key auth while stripping it for `sk-ant-oat` tokens; tests and docs also assert that old behavior.

## Solution Assessment

Yes. The Phase 1 direction is the narrowest maintainable solution: remove stale header/OAuth special-casing, keep explicit sizing and cost control through `context1m`, and leave default-enabling 1M context as a separate product decision.
