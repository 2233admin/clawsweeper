## Summary

Keep this PR open. The branch is useful source work and its latest commit addresses the earlier base-directory and aggregate-size review comments, but it still does not make reference content available through every supported skill-read path; current main also has no implementation of the feature.

## What This Changes

The PR adds skill metadata parsing, docs/changelog coverage, tests, and workspace loading that appends allowed skill-local `references/*.md` content to `SKILL.md` during sandbox skill sync.

## Best Possible Solution

Land the feature only after OpenClaw has one consistent selected-skill read contract that exposes `SKILL.md` plus allowed `references/` content in normal runs, sandbox sync, and `workspaceAccess: rw`, with docs and changelog updated to match that shipped behavior.
