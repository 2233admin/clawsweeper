## Summary

Keep this PR open. Current main at b85edb3f0cf6c2a8bec06537af9211a044266a6c still does not implement skill `references/` auto-load support, and the PR remains a useful implementation candidate. The latest patch addresses earlier path/aggregate-size review feedback, but it still needs maintainer revision because references are only appended during sandbox skill sync while normal and `workspaceAccess: rw` runs still point agents at the original `SKILL.md`.

## Best Possible Solution

Keep this PR open and ask for a focused revision: make `references/` content available wherever the runtime points agents at a selected skill file, not only after sandbox sync. Preserve the latest path traversal and aggregate-size protections, and add regression coverage for default alphabetical loading, selective `references.autoLoad`, normal/non-sandbox reads, sandbox sync reads, path escape rejection, non-markdown rejection, and combined-size limits.
