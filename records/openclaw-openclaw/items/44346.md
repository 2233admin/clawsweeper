## Summary

Keep this PR open. Current main still does not implement skill `references/` auto-load support, and the latest PR is not obsolete, but it remains functionally incomplete because it only appends reference content during sandbox skill sync while common non-sandbox and `workspaceAccess: rw` runs still direct agents to the original `SKILL.md`.

## Best Possible Solution

Keep this PR open and ask for a focused revision: make `references/` content available wherever the runtime points agents at a skill file, not only after sandbox sync. Add regression coverage for default alphabetical loading, selective `references.autoLoad`, non-sandbox reads, sandbox sync reads, path escape rejection, non-markdown rejection, and combined-size limits.

## Risks / Open Questions

- Merging the PR as-is would document references auto-load as general skill behavior, but common non-sandbox and `workspaceAccess: rw` runs would still read the original `SKILL.md` and miss the appended reference content.
