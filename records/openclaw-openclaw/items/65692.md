## Summary

Keep open. Current main still lacks the requested context-pruning micro-compression surface and the bundled public-surface loader follow-up, and the narrower tokenjuice plugin does not implement the same pruning-time behavior. The linked umbrella issue’s ClawHub close plus unresolved review comments make this a maintainer judgment item rather than conservative cleanup closure.

## What This Changes

This PR adds in-memory micro-compression for prunable text tool results, config/schema/docs for `contextPruning.microCompress.*`, a lazy pruner import, and a bundled public-surface source-artifact loader change to prefer Jiti.

## Best Possible Solution

Keep this PR open for maintainer scope and code review. If the core behavior is wanted, land a refined version that clearly separates pruning-time text normalization from optional tokenjuice-style tool-result compaction, resolves the remaining rewrite/threshold concerns, and either includes or splits the bundled public-surface loader fix.
