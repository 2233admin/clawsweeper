## Summary

Keep this issue open. Current main has adjacent DashScope/Qwen streaming token-accounting fixes, but it still does not implement DashScope/Qwen provider quota usage snapshots: `qwen`/`dashscope` are absent from the provider-usage id list, the bundled Qwen plugin has no `resolveUsageAuth` or `fetchUsageSnapshot` hook, and docs list only Anthropic, Copilot, Gemini CLI, OpenAI Codex, MiniMax, Xiaomi, and z.ai for provider usage windows.

## Best Possible Solution

Keep open and implement this in the bundled Qwen provider boundary. Add provider-usage support for Qwen/DashScope aliases, preferably with `extensions/qwen` owning `resolveUsageAuth` and `fetchUsageSnapshot`; add the smallest generic core mapping/registration change needed for plugin usage ids; cover standard and Coding Plan endpoints plus `QWEN_API_KEY`/`MODELSTUDIO_API_KEY`/`DASHSCOPE_API_KEY`; then update usage-tracking and Qwen docs with focused regression tests for status/UI provider usage loading.
