## Summary

Keep this issue open. Current main still lacks DashScope/Qwen provider-usage quota snapshots, so the reported usage-window gap remains real rather than implemented or obsolete.

## Best Possible Solution

Implement this in the bundled Qwen provider boundary: add usage-provider id or alias resolution for qwen, qwencloud, modelstudio, and dashscope; add Qwen-owned resolveUsageAuth and fetchUsageSnapshot hooks using QWEN_API_KEY, MODELSTUDIO_API_KEY, DASHSCOPE_API_KEY, config, and auth profiles; handle Standard and Coding Plan usage endpoints as applicable; then add focused provider-usage/status tests and update the Qwen and usage-tracking docs.
