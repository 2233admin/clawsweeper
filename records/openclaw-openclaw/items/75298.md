## Summary

Keep this open. Current main still has strong safeguards against duplicate HTTP route table entries, but I did not find code, tests, docs, or release evidence proving that repeated webhooks plugin registration under event-loop saturation is fixed or expected.

## What This Changes

The issue reports repeated webhooks plugin route registration under event-loop saturation and asks whether webhooks plugin register() should be guarded per gateway lifecycle.

## Best Possible Solution

Trace the plugin-load or registry-reload caller during the reported saturation windows, then add a focused regression that webhooks routes register once per gateway/plugin lifecycle unless an explicit config or gateway restart/reload occurs.
