## Summary

Keep open. Current main does not implement the PR branch's repaired policy, and it also does not implement the original title/body request for `messages.suppressToolErrors` to suppress mutating tool failures. Main still short-circuits `suppressToolErrorWarnings` before the mutating-tool guard, while `messages.suppressToolErrors` remains after that guard. The PR head is still an active implementation candidate with recent maintainer repair activity, tests, docs/schema updates, and merge-ready/human-review labels in the provided discussion.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers agree with the repaired policy, land the current branch after aligning the PR title/body/changelog to say that non-mutating heartbeat/run warning payloads can be suppressed while unresolved write/edit/message failures remain visible unless explicitly acknowledged. If maintainers want the original all-tools suppression behavior instead, that needs an explicit product/safety decision and corresponding tests/docs.
