## Summary

Keep open. Current main still has the Control UI chat composer shaped as a single bordered input area with a short textarea above a separate toolbar, and the composer controls remain small icon-only buttons with tooltip/title text rather than visible labels. That matches meaningful parts of the reported v2 usability regression, so this is not implemented or stale-insufficient-info cleanup.

## Best Possible Solution

Keep this issue open and fix the Control UI composer directly: make the text-entry target visually and interactively unambiguous, improve action discoverability and hit size, preserve existing send/queue/stop and Shift+Enter behavior, and add focused regression coverage. Broader v2 UI redesign work should remain separate from this composer-specific bug.
