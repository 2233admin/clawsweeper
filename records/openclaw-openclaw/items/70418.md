## Summary

Keep #70418 open. Current main has prompt-cache retention knobs and heartbeat keep-warm guidance, but it does not implement the proposed dedicated Cache TTL Warmer, and the existing plugin/runtime surfaces do not expose enough of the internal conversation request builder to make this an obvious ClawHub-only item. The related heartbeat divergence thread/PR is adjacent, not a replacement for this design proposal.

## Best Possible Solution

Keep this open for maintainer product/API design. A real solution likely belongs in core or behind a new explicit plugin API: add an opt-in cacheWarmer config/schema/docs surface, schedule per active agent with idle-skip behavior, reuse the normal conversation request builder through the same cache boundary/tool/system/message shape, send a bounded minimal provider request, consume/discard the response, avoid transcript/session/channel delivery side effects, and add tests proving request-shape parity plus no persistence/delivery.
