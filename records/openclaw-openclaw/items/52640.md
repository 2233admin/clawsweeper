## Summary

Keep open. Current main has adjacent pieces: Discord edit-based preview streaming, typing/status reactions, and a durable task ledger with maintenance. Those do not yet implement the requested first-class persistent per-run/channel-turn status surface with explicit terminal/unknown semantics. The task docs explicitly exclude normal interactive chat turns from task creation, and the Discord implementation is a transient preview/final-reply path rather than an authoritative durable status object. Related PR #59719 is still open and covers a background exec/CLI-task slice, while #71630 is a separate Web UI indicator request.

## Best Possible Solution

Keep this issue open as the canonical channel-turn status-surface request. The best path is a small generic, best-effort run-status transport keyed by session/thread/run, with Discord editable-message support behind a delay/config gate, lifecycle/tool/finalizer events feeding meaningful state updates, and persisted ownership sufficient to mark uncertain restart/disconnect cases as lost/unknown/interrupted. Review or land narrower slices such as #59719 for background exec liveness separately, and keep #71630 as the Web UI consumer/sibling rather than treating either as a full replacement.
