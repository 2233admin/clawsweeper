## Summary

Keep open. Current main has substantially better durable task tracking, `/tasks`, task audit/maintenance, and Discord preview/tool-progress edits, but it still does not provide the requested persistent, per-run channel status surface for normal long-running Discord/chat turns with explicit terminal/unknown semantics. The related open PR #59719 looks like a useful CLI/background-liveness slice, not a full replacement for this feature request.

## Best Possible Solution

Keep this issue open as the canonical product/design request for a durable, best-effort channel-turn status surface. Review #59719 as a partial CLI/background liveness slice, then design an additive runtime/channel status transport that can create/update/finalize one Discord status message for normal long-running turns, backed by enough ownership state to degrade to lost/unknown after restart or uncertain continuity.
