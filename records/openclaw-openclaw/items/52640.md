## Summary

Keep open. Current main has adjacent task-ledger terminal states, Discord preview edits/tool-progress, status reactions, and internal channel busy snapshots, but it still lacks the requested single persistent editable status surface for normal long-running channel turns with explicit done, failed, unknown, or interrupted semantics. The stale label should not drive closure because the issue is recent, has active follow-up context, and related #59719/#71630 are adjacent rather than superseding.

## What This Changes

The issue requests one persistent, session-scoped editable status surface for long-running channel turns, initially Discord, with explicit lifecycle states and best-effort delivery failure handling.

## Best Possible Solution

Define an additive runtime/channel status transport that can create, update, finalize, and best-effort clear one Discord status message for a normal long-running turn, backed by enough ownership state to fail closed to unknown or interrupted after restart, disconnect, or uncertain continuity.
