## Summary

Keep open. The report is current, concrete, and still matches current-main code paths: completion output can be frozen from the current captured snapshot, announce stats still read only live session-store token fields and format missing values as zero, and the newly added duplicate-emission repro is not resolved by a shipped fix.

## What This Changes

The issue requests hardening subagent completion announces so successful subagent runs reliably deliver their final result payload, accurate token stats, and no duplicate completion events.

## Best Possible Solution

Keep this issue open as the canonical ASCval reproduction cluster. The maintainable end state is an announce path that captures final child output only after the terminal child state is durable, recovers token usage from transcript/session fallback data before formatting stats, and makes completion delivery idempotent across retry/resume paths.

## Reproduction Assessment

Yes for triage, though not as a deterministic unit test yet. The issue gives four detailed live reproductions, and current main has source-backed paths that can freeze a partial result and report zero tokens when session-store usage is missing.

## Solution Assessment

No. Current main is not the best solution yet; the safer path is to land or replace the active payload-hardening PRs where they apply, then add a focused token-usage fallback and duplicate-emission guard with regression coverage.
