## Summary

Keep open. Current main still ships the broad mandatory memory recall wording in both the memory prompt section and the `memory_search` tool description, while the live-check guidance elsewhere does not explicitly exempt current system state, configuration, service status, or time questions from memory recall first.

## What This Changes

The issue asks to refine memory recall prompt and tool guidance so current system-state, configuration, time, and service-status questions use direct live verification instead of defaulting to memory search.

## Best Possible Solution

Refine `memory-core` prompt and tool wording so mandatory recall applies to historical or durable memory questions, add an explicit live-verification-first exception for current system state/configuration/time/service questions, and lock the intended precedence with focused prompt regression tests.
