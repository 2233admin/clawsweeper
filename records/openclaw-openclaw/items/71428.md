## Summary

Keep open. Current main still models conversation-content hook access as host config only, the native plugin manifest does not parse or preserve `permissions.conversationAccess`, and open PR #72690 is already the linked implementation candidate that should be reviewed before this issue closes.

## What This Changes

The issue requests an additive `permissions.conversationAccess` plugin manifest field so plugin authors can declare conversation-hook access needs without patching the host config schemas.

## Best Possible Solution

Review and land PR #72690 or an equivalent narrow patch that adds manifest-level conversation-access request metadata, preserves it through manifest registry/status surfaces, keeps explicit operator approval as the effective grant, and updates manifest/hooks/config docs plus loader and registry tests.

## Reproduction Assessment

Yes. A code-level reproduction is a non-bundled plugin registering `llm_input`, `llm_output`, `before_agent_finalize`, or `agent_end`: current code and tests show those hooks are blocked without host config and accepted only with `plugins.entries.<id>.hooks.allowConversationAccess=true`, while manifest parsing ignores `permissions.conversationAccess`.

## Solution Assessment

Unclear for the RFC sketch as written. A plugin-owned manifest value should not grant raw conversation access by itself; the safer maintainable solution is manifest request metadata plus explicit host approval and documented precedence.

## Security Review

Status: needs_attention

Summary: This issue changes metadata and approval semantics for non-bundled plugin access to raw conversation-content hooks.

Concerns:

- **[medium] Preserve operator approval for transcript access:** `src/plugins/registry.ts:1956`
  - body: If `permissions.conversationAccess` becomes an effective grant from plugin-owned manifest data, an untrusted non-bundled plugin could self-authorize raw transcript hooks. The implementation should keep host config approval or an equivalent operator-controlled grant as the effective access decision.
  - confidence: 0.9
