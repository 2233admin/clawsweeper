## Summary

Keep this issue open. Current main appears to have addressed two reported symptoms elsewhere: the OpenAI/Codex websocket fallback path and Discord InteractionEventListener blocking. The primary `memory-lancedb-pro` `sessionStrategy` behavior is not fully resolved in this repository, and there is no single canonical OpenClaw issue or merged PR that clearly covers that remaining external-plugin/session-strategy regression.

## Best Possible Solution

Leave this issue open or split it into a focused canonical follow-up for the remaining `memory-lancedb-pro` active-memory/session-strategy behavior. Maintainers should ask the reporter to retest current main/latest release with `plugins.slots.memory` explicitly set to the intended memory plugin and confirm whether the external plugin still resolves `sessionStrategy` as `none`; if it still reproduces, the fix likely belongs either in the external plugin's config/default handling or in a generic OpenClaw plugin SDK/runtime config propagation seam, not in the already-fixed websocket or Discord listener paths.

- **No OpenClaw implementation owns `sessionStrategy`:** A repository-wide search found no OpenClaw source, docs, or tests for `sessionStrategy`, `session-strategy`, `memoryReflection`, or `memory-reflection`, which means the central behavior is not directly implemented or fixed in current core code.
