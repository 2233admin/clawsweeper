## Summary

Keep open. Current main still does not implement a delayed Feishu direct-message acknowledgment, and this PR is a plausible Feishu-local implementation candidate, but the current PR head is not mergeable and the patch still has two P2 defects that should block merge.

## What This Changes

The PR adds an 8-second delayed acknowledgment for slow Feishu p2p/private-message dispatches and adds Feishu bot tests for slow, fast, and group-chat delayed-ack behavior.

## Best Possible Solution

Land a Feishu-local revision that integrates with the current runPrepared dispatch path, sends the acknowledgment only for p2p/private chats after a real no-visible-output interval, suppresses it as soon as visible Feishu reply delivery starts, preserves the final reply flow, and covers slow DM, fast DM, group chat, and early-visible-reply cancellation deterministically.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Cancel delayed ack when visible replies start:** `extensions/feishu/src/bot.ts:203-210`
  - body: The timer only gets canceled after `withReplyDispatcher` settles, but Feishu can start streaming/card/plain-text/media output before settlement. A long run that already produced visible output can still send the delayed acknowledgment at 8 seconds, violating the PR body's no-user-visible-reply condition and creating duplicate/confusing messages.
  - confidence: 0.9
- **[P2] Use a fast dispatcher in the fast-path test:** `extensions/feishu/src/bot.test.ts:536-537`
  - body: This test advances fake timers by only 7,999 ms and then awaits `pending`, while the shared dispatch mock resolves after 9,000 ms. Under fake timers the promise cannot settle there, so the test can hang instead of validating that the ack was canceled before the threshold.
  - confidence: 0.88
