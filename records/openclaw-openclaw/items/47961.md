## Summary

Keep this PR open. Current main still lacks the proposed Feishu delayed acknowledgment behavior, but the PR is not merge-ready: the timer is canceled only when dispatch settles rather than when visible output starts, one fake-timer negative test can hang, and the branch is currently merge-dirty against main.

## What This Changes

The PR branch adds a Feishu-local 8-second delayed acknowledgment timer for slow p2p/private-message dispatches and bot tests for slow, fast, and group-chat paths.

## Best Possible Solution

Land a rebased Feishu-local implementation that sends the delayed acknowledgment only for p2p/private chats after a real no-visible-output interval, cancels or suppresses it as soon as any user-visible Feishu reply starts or is delivered, preserves the final reply flow, and covers slow DM, fast DM, group chat, and early-visible-reply cancellation deterministically.

## Reproduction Assessment

Yes. The review defects reproduce by inspection with a long p2p Feishu run that starts visible output before 8 seconds but settles after 8 seconds, and with the fake-timer fast-path test that awaits a 9,000 ms mock after advancing only 7,999 ms.

## Solution Assessment

No. The Feishu-local direction is appropriate, but the current patch ties cancellation to dispatcher settlement instead of visible-output start and leaves one negative test nondeterministic; the safer fix is to cancel from the reply lifecycle/delivery path and make the fast-path test actually settle before the threshold.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P2] Cancel delayed ack when visible replies start:** `extensions/feishu/src/bot.ts:1835-1861`
  - body: The timer is only canceled after `withReplyDispatcher` settles. Feishu can start streaming/card/plain-text/media output from the reply lifecycle before settlement, so a long run that already produced visible output can still send the extra acknowledgment after 8 seconds, contrary to the no-user-visible-reply condition.
  - confidence: 0.9

  - body: This test advances fake timers by only 7,999 ms and then awaits `pending`, but the shared dispatch mock resolves after 9,000 ms. Under fake timers the promise cannot settle there, so the test can hang instead of proving the ack was canceled before the threshold.
  - confidence: 0.88
