## Summary

Keep this PR open. Current `main` does not implement the delayed Feishu DM acknowledgment: the single-agent path still calls `withReplyDispatcher` directly, and a repository search finds no delayed-ack helper or acknowledgment text. The PR is a plausible localized feature, but it still needs review/update because later review feedback flags duplicate acknowledgments after an early visible reply and a fake-timer test that can hang.

## Best Possible Solution

Keep the existing PR as the implementation candidate, rebase it onto current `main`, then update the delayed-ack logic so it cancels once any user-visible reply starts or is delivered. Fix the fake-timer fast-path test, keep the group/fast negative coverage, and validate with the targeted Feishu bot/reply tests before merge.
