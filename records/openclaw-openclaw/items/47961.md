## Summary

Keep this PR open, but it needs changes before merge. Current main still lacks the delayed Feishu direct-message acknowledgment, and the PR is scoped to the right Feishu plugin path, but the proposed timer can still fire after a visible reply has already started and one added fake-timer test can hang instead of proving the fast path.

## What This Changes

The PR adds an 8-second delayed acknowledgment timer for Feishu p2p/private-message dispatches and bot tests for slow, fast, and group-chat behavior.

## Best Possible Solution

Land a Feishu-local revision that sends the delayed acknowledgment only for p2p/private chats after a real no-visible-output interval, cancels or suppresses it as soon as any user-visible Feishu reply starts or is delivered, preserves normal final reply flow, and covers slow DM, fast DM, group chat, and early-visible-reply cancellation deterministically.
