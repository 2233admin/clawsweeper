## Summary

Keep open. The branch targets a real continuation-skip gap, but the proposed detector is too broad: it would treat a heartbeat-only assistant reply as completed bootstrap evidence and can skip full bootstrap context on the next normal turn.

## What This Changes

The PR changes `hasCompletedBootstrapTurn` so any assistant message after the latest compaction counts as completed bootstrap evidence, and updates bootstrap-file tests for assistant-only continuation tails.

## Best Possible Solution

Amend the PR so assistant-message compatibility remains available for ordinary completed turns, but recognized heartbeat-only user/assistant ack pairs do not count as bootstrap-complete evidence. Add regression coverage for a shared-session `heartbeat.lightContext` run followed by a normal continuation-skip turn.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a session tail containing the heartbeat prompt and a `HEARTBEAT_OK` assistant reply, no full-bootstrap marker, `contextInjection: "continuation-skip"`, and then a normal follow-up run; the PR fallback would make the detector return true before full bootstrap context exists.

## Solution Assessment

No, not as written. The detector is the right place to fix the missing-marker case, but counting every assistant row is broader than the heartbeat/lightweight bootstrap contract; the safer solution is to keep scanning past heartbeat-only pairs or require non-heartbeat assistant evidence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Do not count heartbeat-only replies as bootstrapped:** `src/agents/bootstrap-files.ts:120-121`
  - body: When a shared-session heartbeat uses `heartbeat.lightContext: true`, OpenClaw injects only `HEARTBEAT.md` and deliberately records no full-bootstrap marker. This fallback would count the resulting `HEARTBEAT_OK` assistant row as bootstrap-complete evidence, so the next normal continuation-skip turn can skip AGENTS/SOUL/USER before any full bootstrap context exists. Please ignore recognized heartbeat pairs or continue scanning for a non-heartbeat assistant/marker.
  - confidence: 0.82
