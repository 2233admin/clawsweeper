## Summary

Keep open. The PR targets a real continuation-skip gap that current main still has, but the proposed assistant-message fallback is too broad and can make a lightweight heartbeat-only turn suppress full workspace bootstrap on the next normal turn.

## What This Changes

The branch changes `hasCompletedBootstrapTurn` to treat assistant messages without a later compaction as completed continuation evidence and updates bootstrap regression tests for assistant-only and post-compaction transcript tails.

## Best Possible Solution

Amend the continuation detector so ordinary completed assistant turns can satisfy continuation-skip, while recognized heartbeat-only user/assistant pairs are ignored or skipped over and compaction-after-latest-real-turn still forces reinjection.

## Reproduction Assessment

Yes. A source-level reproduction is a session tail with a heartbeat user prompt and `HEARTBEAT_OK` assistant response, no full-bootstrap marker, `contextInjection: "continuation-skip"`, and a following non-heartbeat run; the PR fallback would return true and skip workspace bootstrap injection.

## Solution Assessment

No. The helper is the right place to fix the marker-only gap, but treating every assistant row as safe continuation evidence is broader than the heartbeat/lightweight bootstrap contract; the safer fix is to distinguish non-heartbeat assistant turns or continue scanning past heartbeat-only pairs.

## Review Findings

Overall correctness: patch is incorrect
