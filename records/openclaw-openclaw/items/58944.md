## Summary

Current main still treats `background: false` the same as omitting `background`: only `params.background === true` changes `yieldWindow`, so `false` still gets the default/yieldMs auto-background timer. The schema, tool description, docs, and tests also do not document or cover explicit synchronous execution. This PR is not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open or have a maintainer rebase/adapt it onto current main. The likely product path is to add `synchronousRequested = params.background === false`, make that set `yieldWindow = null`, warn when `yieldMs` is also supplied, update the exec schema/tool description/docs, and add the targeted backgroundMs=10 regression test so the per-command synchronous override is covered.
