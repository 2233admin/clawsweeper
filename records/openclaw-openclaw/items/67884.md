## Summary

Keep this PR open. Current main does not implement the PR's intended outbound hook conversation-id normalization, but the submitted patch is not ready to close or merge because it conflicts with current resolver behavior that intentionally preserves typed ids such as `user:` and `channel:` for channels like Discord.

## Best Possible Solution

Keep this PR open for revision. The best path is to define one resolver-aware conversation-id contract, preserve typed ids where they are canonical for channels like Discord, and add regression tests that compare `inbound_claim`, `message_received`, and `message_sent` contexts for both fallback-prefixed ids and plugin resolver outputs.
