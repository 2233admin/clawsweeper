## Summary

Current main still does not satisfy the requested default, hard-to-miss compaction/reset signal plus assistant-side user disclosure. Existing compaction notices remain opt-in, tests and docs preserve silent default behavior, post-compaction recovery is internal model guidance, Control UI status is transient, and channel behavior is still surface-specific.

## Best Possible Solution

Keep this open for maintainer product work. A complete solution should define the default compaction/reset visibility contract, decide which durable UI or channel indicators are required, add assistant-side disclosure and re-grounding/reload-offer behavior after compaction, preserve safe suppression controls for customer-facing deployments, and update tests/docs for `agents.defaults.compaction.notifyUser`, post-compaction recovery, Control UI status, and supported chat surfaces.
