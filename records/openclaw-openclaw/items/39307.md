## Summary

Current main still does not implement the requested default mid-thread compaction/reset disclosure. It has partial pieces: opt-in compaction notices, internal post-compaction guidance, transient Control UI status, persisted compaction dividers, and channel-specific callbacks. Those do not satisfy the default user-visible signal plus assistant-side self-disclosure/reload-offer contract, and related #72043 shows the default/suppression policy needs maintainer product judgment.

## What This Changes

Surface mid-thread compaction/context reset to users and make post-compaction assistant recovery disclosure and re-grounding behavior explicit.

## Best Possible Solution

Keep this open for maintainer product work. Define the default compaction/reset visibility and suppression contract, then implement a durable user-visible event plus assistant-side disclosure/re-grounding/reload-offer behavior across the reply runner, post-compaction recovery, Control UI, and supported chat surfaces, with docs and targeted tests updated together. Coordinate the direction with related #72043 before changing defaults.
