## Summary

Keep this PR open. Current main still does not implement the attributed-prefix normalization this PR is trying to add: iMessage echo-cache text keys only normalize CRLF and trim whitespace, so a reflected body like `\x93b\x02Cached reply` still misses the clean outbound cache entry. The PR remains a focused implementation candidate with relevant regression coverage and no obvious security-sensitive file changes in the supplied diff.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is maintainer review and merge, or an equivalent narrow patch, preserving the echo-cache and self-chat regression tests and validating with the targeted iMessage monitor tests plus the changed gate.
