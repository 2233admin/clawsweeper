## Summary

Current main still does not expose manual Telegram `pin` and `unpin` message actions. The shared action names and low-level Telegram helpers exist, and Telegram send-time delivery pinning works, but Telegram action discovery, the action map, runtime aliases, runtime dispatch, config action gates, and docs still omit manual pin/unpin for existing messages.

## Best Possible Solution

Keep this issue open and implement the missing behavior in the bundled Telegram plugin: advertise `pin` and `unpin`, map them to Telegram runtime actions, add runtime aliases and dispatch branches that call `pinMessageTelegram` and `unpinMessageTelegram`, add action gates/config schema/docs if maintainers want parity with delete/edit controls, and cover discovery, runtime, helper, and docs behavior with focused tests.

## Clownfish Work Prompt

Implement manual Telegram message `pin` and `unpin` actions for this issue. Keep the change scoped to the bundled Telegram plugin and directly related config, docs, and tests.

Expected behavior:
- `openclaw message pin --channel telegram --target <chat> --message-id <id>` and agent message-tool action `{ action: "pin", ... }` should pin an existing Telegram message via `pinMessageTelegram`.
- `openclaw message unpin --channel telegram --target <chat> --message-id <id>` and agent message-tool action `{ action: "unpin", ... }` should unpin an existing Telegram message via `unpinMessageTelegram`.
- Preserve existing send-time `--pin` / `delivery.pin` behavior for `send`; do not conflate it with manual pin/unpin.
- Do not implement `list-pins` unless maintainers explicitly ask; Telegram does not expose the same cross-channel list-pins surface.

Implementation boundary:
- Add `pin` and `unpin` to Telegram message action discovery and `TELEGRAM_MESSAGE_ACTION_MAP` in `extensions/telegram/src/channel-actions.ts`.
- Import/expose `unpinMessageTelegram` in `extensions/telegram/src/action-runtime.ts`, add runtime aliases such as `pin`, `pinMessage`, `unpin`, and `unpinMessage`, and add action branches that parse `chatId`/`to` plus `messageId` and return explicit JSON success.
- Add Telegram action gates such as `pinMessage?: boolean` and `unpinMessage?: boolean` only if maintainers want parity with delete/edit gates; if added, update `src/config/types.telegram.ts`, `src/config/zod-schema.providers-core.ts`, generated config metadata/checks as required, and Telegram docs.
- Keep owner-specific behavior in `extensions/telegram`; avoid core changes unless a shared contract test exposes an existing generic gap.
- Update docs to list Telegram manual pin/unpin actions and explicitly distinguish them from send-time delivery pinning.
- Add a user-facing changelog entry if repo convention requires one for this enhancement, crediting the reporter as `Thanks @dan-osss`.

Do not change unrelated Telegram send/media behavior, plugin loading, or shared message action names.
