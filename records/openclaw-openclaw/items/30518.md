## Summary

Keep this issue open. Current main still has shared `pin`/`unpin` action names and Telegram Bot API transport helpers, but the bundled Telegram plugin does not expose manual `pin`/`unpin` through discovery, action mapping, runtime aliases, runtime dispatch, config gating, docs, or contract tests.

## What This Changes

Expose Telegram `pin` and `unpin` message actions through the message tool, backed by Telegram Bot API `pinChatMessage` and `unpinChatMessage`.

## Best Possible Solution

Implement manual Telegram `pin` and `unpin` in the bundled Telegram plugin: add discovery/action mapping, runtime aliases and dispatch to the existing helpers, add explicit action gates if maintainers want them, keep send-time `delivery.pin` behavior distinct, update Telegram and message CLI docs, add focused tests, and include a user-facing changelog entry crediting the reporter.
