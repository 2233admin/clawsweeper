## Summary

Keep this PR open. Current main still does not implement the cron chaining fields and still has the smaller cron UI i18n/test-registration gaps, but the proposed branch introduces a concrete manual-run chaining bug and needs maintainer product judgment because current docs route durable multi-step pipelines to Task Flow.

## What This Changes

This PR adds cron success/failure job chaining across the cron model, gateway schema, runtime, generated Swift protocol models, and Control UI, while also fixing cron UI i18n strings and cron UI test registration.

## Best Possible Solution

Either rebase this PR and make cron-native chaining consistent across scheduled, queued manual, and direct service manual runs, or split out the narrow cron UI i18n/test-registration fixes and route durable pipeline orchestration through Task Flow instead.
