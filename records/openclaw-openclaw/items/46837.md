## Summary

Keep open. Current main still lacks the requested missing-`extraPaths` doctor warning, so the PR is not obsolete; however, the branch still needs revision because its builtin-backend path check can diverge from the runtime resolver for `~`-prefixed entries and produce a false healthy result.

## What This Changes

The PR adds an `openclaw doctor` memory-search diagnostic for configured `memorySearch.extraPaths` that do not exist, plus unit coverage for missing, existing, and tilde-prefixed paths.

## Best Possible Solution

Revise the PR so the doctor diagnostic reuses or exactly mirrors the active backend resolver. For builtin memory, check the same normalized workspace-relative paths that indexing and memory reads use; keep any QMD-specific tilde behavior backend-specific, and update tests only if the product intentionally changes the documented path contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Align extraPaths checks with builtin runtime resolution:** `src/commands/doctor-memory-search.ts:271-273`
  - body: The new check resolves `~` entries with `resolveUserPath`, but builtin memory indexes `extraPaths` through `normalizeExtraMemoryPaths`, where every non-absolute entry is resolved under the workspace. If `$HOME/notes` exists while `<workspace>/~/notes` is absent, doctor can report healthy even though runtime indexes nothing, hiding the stale path this warning is meant to catch.
  - confidence: 0.87
