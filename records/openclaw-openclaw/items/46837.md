## Summary

Keep open. Current main still lacks the requested missing-`extraPaths` doctor diagnostic, but the PR branch currently misaligns the new check with builtin memory runtime path resolution by home-expanding `~` paths that runtime treats as workspace-relative.

## What This Changes

The PR adds an `openclaw doctor` memory-search warning for missing configured `extraPaths` and unit coverage for missing, existing, and tilde-prefixed paths.

## Best Possible Solution

Revise the PR so the doctor diagnostic shares or faithfully mirrors the active backend resolver: builtin should check the same workspace-relative normalized paths that runtime indexes, while any QMD-specific behavior should remain backend-specific. Keep focused coverage for absolute, workspace-relative, missing, existing, default/per-agent merged paths, and tilde-prefix cases without changing the documented path contract accidentally.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Align extraPaths checks with builtin runtime resolution:** `src/commands/doctor-memory-search.ts:271-273`
  - body: The new check treats `~/...` entries as home-relative, and the new test requires that behavior, but builtin memory resolves every non-absolute `extraPaths` entry under the workspace. If `$HOME` contains the target while the workspace-relative path is absent, doctor reports healthy while runtime indexes nothing. Reuse the builtin normalizer or branch the check by active backend semantics.
  - confidence: 0.87
