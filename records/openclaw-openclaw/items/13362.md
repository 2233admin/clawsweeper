## Summary

Keep open. Current main still does not implement a first-class `openclaw doctor --dry-run`: the doctor command lacks the flag, `DoctorOptions` cannot carry it, docs omit it, and several doctor repair paths can still write config, install staged plugin dependencies, rewrite cron state, or restart/repair services. The related global dry-run issue and open `--check-config` PR are adjacent but do not supersede this scoped doctor safety request.

## What This Changes

The issue requests an `openclaw doctor --dry-run` mode that validates and previews doctor repairs without writing config or performing other side effects.

## Best Possible Solution

Add a native doctor dry-run contract: register and type the flag, propagate it through doctor prompter/config/health flows, render the same repair plan without prompts or mutations, suppress non-config repair side effects, define exit semantics for detected issues, and update the public doctor docs plus changelog.
