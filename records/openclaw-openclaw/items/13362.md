## Summary

Keep this issue open. Current main still lacks a native `openclaw doctor --dry-run` surface: the doctor CLI has no `--dry-run` option, `DoctorOptions` has no dry-run state, the doctor docs omit it, and the existing preview/confirmation flow can still write config when accepted. Related #41418 and #47302 cover different dry-run/check-config scopes rather than superseding this request.

## Best Possible Solution

Keep this issue open and define a native `openclaw doctor --dry-run` contract. The best implementation should add a doctor option and docs, run validation and recommendation collection without prompts or writes, block all doctor repair/write side effects, optionally show a structured change plan or diff, and exit non-zero when actionable issues or would-change repairs are found. Coordinate wording with `openclaw config set --dry-run`, while keeping #41418 as the separate global agent-tool dry-run scope and #47302 as the separate deep `--check-config` validation scope.
