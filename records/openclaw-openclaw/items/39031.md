## Summary

Keep open. Current main still does not forward legacy iMessage/imsg tapback reactions to the agent: the watcher only handles ordinary `message` notifications, subscribes only with attachment options, the imsg payload/parser/config types have no reaction fields or opt-in, and inbound context construction remains text-message oriented. The issue is also paired with open implementation/filtering work in #39322, #60677, and #60274, so maintainer review should consolidate those rather than close this issue.

## What This Changes

Request to forward legacy iMessage/imsg tapback reactions to the agent with target message id, reaction emoji, sender, and timestamp, with possible cross-channel reaction support later.

## Best Possible Solution

Keep this issue open as the canonical legacy imsg tapback-to-agent request. Review, rebase, or supersede #39322 with a deliberate opt-in implementation that verifies the imsg reaction RPC contract, parses reaction metadata, routes reactions as a typed system event or accepted summary format, and reconciles #60677/#60274 filtering behavior with focused config, docs, and iMessage tests.
