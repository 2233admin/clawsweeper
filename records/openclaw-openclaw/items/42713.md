## Summary

Keep open. Current main still does not fully solve the reported structured logging gap, and an open PR explicitly links this issue with GitHub closing syntax. The shipped code carries provider/model/profile details for individual fallback attempts, but the exhausted fallback wrapper and model fallback decision logging path still lack normalized request-origin metadata such as sessionId, sessionKey, session_id, or lane.

## Best Possible Solution

Keep this issue open. Review PR #73506 and either land, revise, or close it; the eventual fix should pass normalized request-origin metadata such as sessionId/sessionKey or lane through the model-fallback runner, FailoverError description/coercion, FallbackSummaryError or equivalent wrapper logging, and model_fallback_decision logs, with focused tests for exhausted multi-provider fallback attribution.
