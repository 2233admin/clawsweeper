## Summary

Keep this issue open. Current main still carries provider/model attribution for individual fallback attempts, but the exhausted fallback wrapper and model_fallback_decision logging surface still have no normalized session_id/sessionKey/lane metadata to tie wrapper failures back to the originating session.

## Best Possible Solution

Keep open and implement a narrow observability fix in the model-fallback path: add normalized request-origin metadata such as sessionKey/sessionId or lane to runWithModelFallback, carry it through FallbackSummaryError and logModelFallbackDecision, wire the main reply, follow-up, memory-flush, and cron call sites to pass it, and add focused tests for structured decision/error logs on exhausted multi-provider fallback.
