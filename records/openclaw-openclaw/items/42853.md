## Risks / Open Questions

- The PR is stale against current main: it targets src/wizard/onboarding.gateway-config.ts, while current setup code lives in src/wizard/setup.gateway-config.ts.
- The proposed NODE_OPTIONS handling can preserve or append arbitrary ambient startup flags such as --require, conflicting with current daemon environment hardening.
- The heap-flag detector should account for Node's accepted underscore spelling, --max_old_space_size, before adding a second heap flag.
- The 85% of total system RAM default is a runtime policy choice that may be too aggressive for containers, shared hosts, or memory-constrained machines.
