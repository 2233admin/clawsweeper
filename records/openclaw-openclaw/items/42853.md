## Risks / Open Questions

- The PR should not preserve or append arbitrary install-time `NODE_OPTIONS`; current main treats that variable as blocked startup-control input because flags such as `--require` can execute code before OpenClaw starts.
- The heap-flag detector should handle both `--max-old-space-size` and Node's accepted underscore spelling `--max_old_space_size` before adding a new heap flag.
- The 85% of total system RAM default is a product/runtime policy decision and may be too aggressive on shared hosts, containers, or memory-constrained machines.
- The branch is stale against current main: gateway wizard code moved to `src/wizard/setup.gateway-config.ts`, and the node install CLI path still needs end-to-end config threading.
