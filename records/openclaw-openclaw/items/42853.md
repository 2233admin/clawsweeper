## Summary

Keep this PR open. Current main still does not implement the proposed `gateway.nodeMaxOldSpaceMb` config, daemon service heap-limit injection, node-service threading, or setup wizard prompt. The PR is still a plausible implementation path for the open Gateway OOM report in #52092, but it needs maintainer review and refresh against the current setup flow and NODE_OPTIONS hardening before it could merge.

## Best Possible Solution

Keep this PR open for maintainer review or an equivalent replacement implementation. The best path is to add a documented `gateway.nodeMaxOldSpaceMb` config contract, generate a deterministic heap-only startup setting for gateway and node services without importing ambient `NODE_OPTIONS`, handle both heap-flag spellings, update the current setup wizard path, and add focused tests for gateway service env, node service env, schema/help/docs, wizard blank-to-auto behavior, hostile NODE_OPTIONS, and the related cron/OOM path.
