## Summary

Keep open. Current main still lacks the requested Gateway artifact RPC surface, and this PR is a relevant implementation for the linked SDK-facing artifact request, but the branch has a concrete run/task scoping bug in transcript-derived artifacts and still needs normal maintainer review before it can safely close the request.

## What This Changes

The PR adds Gateway `artifacts.list`, `artifacts.get`, and `artifacts.download` protocol schemas, read-scope discovery, transcript-backed handlers/tests, and generated Swift protocol models.

## Best Possible Solution

Keep the PR open and revise it into an additive artifact contract that only returns run/task-scoped artifacts when transcript or ledger provenance proves the association, treats broader transcript scraping as session-scoped, tightens URL/download source policy, and either wires `@openclaw/sdk` artifact helpers or leaves #74706 open with an explicit SDK follow-up.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: on current main transcript messages only get `__openclaw.id` and `seq`, while the PR’s `collectArtifactsFromMessages` returns untagged transcript artifacts for a `runId` or `taskId` query and labels them with the requested id.

## Solution Assessment

No. The direction is useful, but the proposed handler is not the narrowest maintainable solution until run/task provenance is explicit and the SDK-facing closure boundary is settled.

## Review Findings

Full review comments:

- **[P2] Require explicit run/task provenance before returning artifacts:** `src/gateway/server-methods/artifacts.ts:211-216`
