## Summary

Keep this issue open: current main has partial active-registry reuse for plugin tools, but the provider-auth path still rebuilds manifest-backed candidate and evidence maps on config/workspace-aware lookups, and the report includes concrete production CPU-profile evidence plus an independent macOS data point.

## What This Changes

The issue requests eliminating Gateway event-loop stalls by avoiding repeated synchronous plugin registry rebuilds and provider-auth env-var resolution during OpenClaw tool creation.

## Best Possible Solution

Fix the hot path with scoped reuse: tool construction should reuse the compatible active Gateway/plugin registry where freshness allows, and provider-auth candidate/evidence maps should be computed once per stable config/workspace/environment snapshot and passed through repeated model/tool auth checks without adding an unbounded stale global metadata cache.
