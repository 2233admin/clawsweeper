## Summary

Keep open. Current main still only prepends model fallback and fallback-cleared reply notices when verbose mode is enabled, the requested independent opt-in config is absent from config and docs, and the existing e2e coverage still asserts verbose-off suppression. The issue is also paired with open implementation PR #69399, so it should remain open until that PR lands or maintainers choose a different implementation shape.

## What This Changes

The issue requests a default-off configuration option that lets model fallback and fallback-cleared notices appear in normal replies without enabling verbose mode.

## Best Possible Solution

Review #69399 and either land it or request a revised implementation that adds a default-false agent-defaults option, updates config type/schema/help/docs, and lets `runReplyAgent` prepend fallback and fallback-cleared notices when verbose mode is enabled or the opt-in is true while preserving once-per-active-fallback behavior.
