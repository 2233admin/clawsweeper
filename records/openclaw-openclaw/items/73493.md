## Summary

Keep this PR open. It is maintainer-authored and carries the protected `maintainer` label, and current `main` still lacks the bounded startup retry that this PR proposes for memory-core dreaming cron reconciliation. The linked bug #72841 and overlapping implementation PR #73170 are still open, so the right path is maintainer review and consolidation rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or consolidate one memory-core-owned bounded startup retry in `extensions/memory-core/src/dreaming.ts`, keep the regression test proving cron can become available after `gateway_start` without heartbeat traffic, verify timer cleanup through `gateway_stop`, and then close #72841 while superseding #73170 if this implementation is chosen.
