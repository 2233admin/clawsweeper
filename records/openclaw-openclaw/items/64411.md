## Summary

Keep this issue open. Current main still awaits bundle MCP runtime setup and tool materialization on the assistant reply path, only bounds MCP connection setup, and has no catalog/tool-call budget, circuit/cooldown state, degraded empty-tool fallback, or status surface that separates channel health from external MCP health.

## What This Changes

Request bounded fail-fast degradation for unhealthy MCP/plugin capabilities so optional external failures do not block the main assistant reply path.

## Best Possible Solution

Add a core resilience contract for bundle MCP and reply-path plugin hooks: bounded catalog and tool-call budgets, per-server failure cooldown/circuit state, degraded empty-tool materialization for optional unhealthy MCP capability, and diagnostics that clearly distinguish channel health from external capability health.
