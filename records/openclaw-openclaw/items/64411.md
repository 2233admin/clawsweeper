## Summary

Keep this issue open. Current main still materializes bundle MCP tools on the main reply path, only time-bounds MCP connection setup, and lacks catalog/tool-call budgets, circuit state, degraded status, or a fast empty-tool fallback for unhealthy optional MCP servers. Plugin hook timeout coverage is also only partial.

## What This Changes

Request time-bounded, optional degradation for unhealthy external MCP/plugin capabilities so normal assistant replies continue when those integrations stall.

## Best Possible Solution

Add a core resilience contract for bundle MCP and reply-path plugin hooks: bounded catalog and tool-call budgets, per-server failure cooldown/circuit state, degraded empty-tool materialization for optional unhealthy MCP capability, diagnostics that distinguish channel health from external capability health, and focused regression tests for slow catalog listing, slow tool calls, repeated failure cooldown, and blocking hook phases.

## Repair Work Prompt

Implement a focused fix for this issue. Current main awaits bundle MCP catalog materialization on the embedded Pi reply path, only time-bounds MCP connection setup, directly awaits MCP listTools/callTool, rethrows catalog failures, and only has partial plugin hook timeout defaults. Add a conservative resilience layer for external bundle MCP work: short configurable catalog and tool-call budgets, per-server failure accounting/cooldown, degraded empty-tool materialization for optional unhealthy MCP servers, and high-signal logs/status data that makes external MCP degradation distinguishable from channel/model health. Preserve explicit tool allowlists and intentionally required tool behavior; do not silently hide failures when the current turn/config explicitly requires the MCP tool. Add regression tests for slow listTools, slow callTool, repeated-failure cooldown, catalog failure fast-degrade, and any hook timeout default you change. Update config types/schema/docs if new knobs are added, and add a changelog fix entry with reporter credit. Do not change bundled tool profile defaults, unrelated provider timeouts, or plugin install behavior.
