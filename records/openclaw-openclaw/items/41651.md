## Summary

Current main still emits the escaped-symlink warning before skill records are merged by name, and no merged implementation suppresses the duplicate trusted symlink case. The issue should remain open while maintainers decide between the open implementation candidates or a narrower replacement that preserves the skill-root security boundary and bundled allowlist policy.

## What This Changes

Avoid repeated warn-level skill-loader noise when personal skill-root symlinks point at bundled or otherwise trusted skill roots that are already loaded from another source.

## Best Possible Solution

Keep this issue open. Maintainers should review and either land a revised #45912 or #59688, close the unsuitable candidates, or implement an equivalent focused fix in src/agents/skills/workspace.ts that suppresses repeated trusted duplicate symlink warnings while preserving strict handling for unknown escapes and preserving bundled-source allowlist semantics. Add regression tests for personal .agents symlinks to bundled/configured roots, unknown external symlinks still warning/skipping, and skills.allowBundled not being bypassed; update https://docs.openclaw.ai/tools/skills if the containment contract changes.
