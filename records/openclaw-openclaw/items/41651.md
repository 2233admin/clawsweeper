## Summary

Keep open. Current main still emits a warn-level skills log whenever a skill path realpath escapes its configured root, including personal `.agents/skills` symlinks, and the duplicate-by-name merge happens only after all sources are loaded. The provided GitHub context also has an open unmerged implementation candidate with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open. Review haroldfabla2-hue/openclaw#2 or land an equivalent focused change in `src/agents/skills/workspace.ts` that keeps the realpath containment check but suppresses or downgrades the warning when a personal `.agents/skills` symlink points at a skill already loaded from a trusted bundled or managed source. Add a regression test for that duplicate trusted-source symlink case before closing.
