Keep open. Current main still performs per-source realpath containment before duplicate-by-name merging, still emits a warn-level escaped-symlink message, and docs/tests still assert strict resolved-realpath containment. The related open implementation candidates should be reviewed before this issue is closed.

Keep this issue open. Review and either land or close #45912 and the linked haroldfabla2-hue/openclaw#2 candidate, or implement an equivalent focused change in src/agents/skills/workspace.ts that preserves strict containment for untrusted escapes while avoiding repeated warn-level noise for duplicate trusted symlink cases.
