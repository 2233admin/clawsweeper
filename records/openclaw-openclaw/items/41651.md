## Summary

Keep open. Current main still realpath-checks each skill source against only that source root, still warns on symlink escapes before duplicate-by-name merging, and related implementation candidates remain unresolved. The remaining fix is real but security-sensitive because bundled-skill source policy and `skills.allowBundled` need to be preserved.

## Best Possible Solution

Keep this issue open. Review and either land or close #45912 and the linked haroldfabla2-hue/openclaw#2 candidate, or implement an equivalent focused change in `src/agents/skills/workspace.ts` that preserves strict containment for untrusted escapes while avoiding repeated warn-level noise for duplicate trusted symlink cases. Add regression coverage for personal `.agents/skills` symlinks pointing at bundled or configured trusted skill roots, and update https://docs.openclaw.ai/tools/skills if the containment contract changes.
