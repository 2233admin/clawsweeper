## Summary

Keep this PR open. Current main still lacks the proposed `rescueWatchdog` cron payload, protocol schema, runner, and gateway cron wiring. Related #40791 is an open external script/docs approach, not a replacement for this core cron/service layer.

## Best Possible Solution

Keep this PR open for maintainer review or replacement by a narrower core implementation. The best path is to rebase and validate the `rescueWatchdog` cron/service layer, optionally split daemon hardening from the cron runner, and only supersede this PR if a newer core PR implements the same payload, runner, gateway wiring, and service-repair behavior.

- This PR changes local service-control and command-execution surfaces, including cron execution, gateway probing, launchd/systemd/schtasks restart paths, and process execution, so it needs maintainer security review and current CI/Testbox validation before merge.
- Current main has overlapping rescue-bot documentation and some daemon hardening, but not the core `rescueWatchdog` payload/runner/wiring; closing as implemented would lose still-relevant work.
- The related open #40791 external watchdog proposal may be useful operationally, but it does not supersede this core cron/service-layer implementation.
