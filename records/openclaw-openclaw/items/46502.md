## Summary

Keep this PR open. Current main still does not implement the PR's central core rescue-watchdog layer: there is no `rescueWatchdog` cron payload/schema, no rescue runner dependency, no gateway cron wiring, and no source/docs/changelog evidence that the feature shipped. Related #40791 is an open external script/docs approach and explicitly does not add core gateway runtime behavior, so it does not supersede this PR.

## Best Possible Solution

Keep this PR open for maintainer review or replacement by a narrower core implementation. The best path is to rebase it onto current main, split the daemon/service hardening from the `rescueWatchdog` cron runner if that helps review, and validate the payload schema, runner, gateway wiring, service repair behavior, and security-sensitive process/service-control changes before merge.
