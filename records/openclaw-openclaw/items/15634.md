## Summary

Keep this issue open. Current main has completed many Mattermost parity subscopes, including markdown-aware delivery, reactions, threading, typing indicators, media, streaming, and interactive buttons, but Mattermost read receipts / user presence are still not implemented or documented. The prior maintainer review reached the same conclusion, and there were no Mattermost plugin/doc changes between that review SHA and current main.

## Best Possible Solution

Keep this issue open as the Mattermost parity umbrella until read receipts / user presence are implemented in the Mattermost plugin, explicitly documented as descoped, or split into a narrower canonical Mattermost issue. Any implementation should update Mattermost WebSocket/client handling, config/types/docs, and focused tests for the chosen event/action/privacy contract.
