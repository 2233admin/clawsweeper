## Summary

Keep open. Current main still permits the reported overlap: session transcript indexing and workspace memory indexing can both run, the bundled session-memory hook still writes session-derived Markdown into the workspace memory directory, and search deduplication is keyed to chunk ids rather than shared session provenance. Related PR #40618 is open and only a partial mitigation, so this remains a valid narrow docs/design issue.

## What This Changes

The issue asks OpenClaw to document or prevent duplicate memory indexing when `memorySearch` indexes both `memory` and `sessions` while the session-memory hook writes session summaries into workspace memory.

## Best Possible Solution

Add public docs guidance in the memory config/search docs and session-memory hook docs explaining that transcript indexing and the session-memory hook overlap. The guidance should recommend using `sources: ["memory"]` when relying on the hook, or disabling/redirecting the hook when indexing raw session transcripts; a later runtime fix can add provenance-aware dedupe or a configurable output directory.
