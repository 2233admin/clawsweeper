## Summary

Keep this PR open for maintainer review. Current main still has the reported groundedCount-based badge and Daily Log section behavior, so the PR is not implemented; however the discussion and open #72552 show the standalone patch is only part of #68904 and the desired grounded-only badge semantics are still unsettled.

## What This Changes

The PR changes `describeWaitingEntryOrigin` in `ui/src/ui/views/dreaming.ts` so the daily-log origin badge is keyed from `dailyCount > 0` instead of `groundedCount > 0`, while leaving mixed-origin detection based on grounded replay plus live support.

## Best Possible Solution

Resolve #68904 with one coherent Control UI change that aligns the Daily Log title, summary count, staged section, waiting/promoted badges, and metadata with the chosen semantics, plus a focused badge/section matrix test. After that canonical PR lands or maintainers choose which PR should carry the fix, close the overlapping PR accordingly.
