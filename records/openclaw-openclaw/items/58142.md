## Summary

Keep #58142 open. Current main still does not implement per-skill `model`/`modelProfile` routing in `SKILL.md` frontmatter, and the existing `before_model_resolve` plugin hook is only a run-level pre-resolution hook, not a skill invocation contract. The PR is a focused candidate implementation for the still-open #43260 feature and has substantive author verification plus follow-up fixes in the discussion, so it should not be auto-closed as obsolete or out of scope.

## Best Possible Solution

Keep the PR open for maintainer review or replace it with a narrower implementation that explicitly designs the `SKILL.md` `model`/`modelProfile` contract, updates skill metadata/types/docs/config expectations, handles cached snapshots without rescanning unnecessarily, and covers the active read/invocation routing behavior with focused tests. If maintainers prefer plugin-driven routing instead, #43260 should be redirected to a concrete new plugin API that exposes skill selection/invocation context; the current run-level `before_model_resolve` hook alone is not enough for this specific frontmatter request.
