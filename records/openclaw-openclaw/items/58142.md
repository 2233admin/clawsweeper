## Summary

Keep this PR open. Current main still lacks the requested SKILL.md model-routing contract, so the PR is not obsolete; however, the proposed implementation is not merge-ready because it changes only the model object while keeping the stream transport and credential lookup bound to the original session provider, and it introduces silent provider/cost routing from untrusted skill metadata without a settled policy.

## What This Changes

The PR adds optional `model` and `modelProfile` SKILL.md metadata, rebuilds skill entries from cached snapshots, wraps read-tool detection and stream calls for per-skill model routing, and adds focused unit tests.

## Best Possible Solution

Land a maintained implementation only after the SKILL.md routing contract is explicit: parse and preserve the metadata through cached snapshots, route at a well-defined skill activation boundary, re-resolve stream transport and credentials for the selected provider/model, define user/session override precedence and fallback behavior, document privacy and cost implications, and cover parser, cached-snapshot, cross-provider, alias/path, fallback, and plugin-hook interactions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Re-resolve stream transport for routed models:** `src/agents/skills/model-router.ts:210`
  - body: `wrapStreamFnSkillModelRouter()` only swaps the `Model` object passed to the already-built stream function. That stream function was resolved earlier from the session model and, on main, provider streams and API-key lookup are bound to `params.model.provider`; a skill that routes from the default provider to another configured provider will still use the original provider transport/auth path, so the cross-provider use case can fail or hit the wrong endpoint. Rebuild or select the stream/auth path from the active routed model instead of only replacing the argument.
  - confidence: 0.9
- **[P2] Only route after a successful canonical read:** `src/agents/skills/model-router.ts:164-171`
  - body: The wrapper treats `file` and `filename` as SKILL.md path aliases and sets `ctx.activeModel` before delegating, but current OpenClaw read tools validate only the canonical `path` parameter. If a model calls `read` with one of these unsupported aliases, the underlying read fails while the next LLM turn can still be routed as though the skill loaded. Either normalize the delegated args and prove the read succeeds, or only update the active model after a successful canonical SKILL.md read.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces silent skill-controlled model/provider switching, which is security- and cost-sensitive for untrusted third-party skills.

Concerns:

- **[medium] Add policy for untrusted skill model overrides:** `src/agents/skills/model-router.ts:210`
  - body: OpenClaw docs tell users to treat third-party skills as untrusted, but this PR lets SKILL.md metadata silently route subsequent prompt/session content to another configured provider or a more expensive model using existing credentials. Before merge, the feature needs an explicit user/agent policy, precedence rules, audit/logging behavior, and docs for privacy and cost impact.
  - confidence: 0.86
