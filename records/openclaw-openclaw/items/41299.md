## Summary

Keep this PR open. Current main still joins inbound metadata and user content with blank lines, has no `---openclaw:user-msg---` marker or marker stripper, and the supplied PR diff is still a useful implementation candidate but needs maintainer rework before it is safe to land.

## Best Possible Solution

Keep this PR open for maintainer review and rework. The best path is to centralize inbound metadata/body composition in a helper used by normal text, captionless-media fallback, raw-body tests, and prompt-composition fixtures; inject a unique separator only at that boundary; strip only the injected separator associated with removed OpenClaw metadata; preserve user-authored marker text; include generated metadata labels such as location/structured context in the contract; then run changed gates for auto-reply, gateway/UI sanitization, session-cost/session export, and prompt fixture surfaces.
