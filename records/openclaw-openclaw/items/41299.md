## Summary

Keep this PR open. Current main still joins OpenClaw inbound metadata and the user body with only blank lines, and no `---openclaw:user-msg---` implementation or stripper exists on main. The PR remains a plausible fix candidate, but the current proposed diff still needs maintainer review/rework around media-only prompt parity, scoped separator stripping, and prompt-composition test alignment.

## Best Possible Solution

Keep this PR open for maintainer review and rework. The best path is to introduce one shared helper for composing metadata plus user body, use it for both normal text and media-only fallback prompts, strip only the injected separator adjacent to removed OpenClaw metadata, preserve user-authored marker text, update prompt-composition/raw-body tests to match runtime behavior, and then run the changed OpenClaw gates outside this read-only review.
