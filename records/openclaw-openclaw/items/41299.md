## Summary

Keep this PR open. Current main still joins inbound metadata and user content with blank lines, and there is no `---openclaw:user-msg---` marker or marker stripper on main. The supplied PR is a plausible narrow fix candidate, but it still needs maintainer rework before landing because the current diff does not cover every prompt-composition and metadata-stripping path.

## Best Possible Solution

Keep this PR open for maintainer review and rework. The best path is to centralize inbound metadata/body composition in one helper used by normal text, captionless-media fallback, raw-body tests, and prompt-composition fixtures; inject a unique separator only at that boundary; strip only the separator associated with removed OpenClaw metadata; preserve user-authored marker text; align sanitizer recognition with all generated metadata labels, including location and structured context; then validate the auto-reply, gateway/UI sanitization, replay-history, session-cost, and prompt fixture surfaces.
