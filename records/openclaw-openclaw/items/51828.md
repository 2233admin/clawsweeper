## Summary

Keep open. Current main has partial mitigations for the cron startup flood and channel health-monitor restart loops, and the KeepMyClaw comment is useful external snapshot context, but the concrete subagent session-retention gap remains and is still best tracked through the narrower open #51767 thread.

## Best Possible Solution

Keep this issue open until maintainers resolve #51767 or intentionally split this umbrella into narrower canonical issues. The concrete next step is to decide and document the default subagent/session retention contract: either archive/delete default keep-mode subagent sessions after agents.defaults.subagents.archiveAfterMinutes, introduce a separate subagent/session maintenance policy, or update docs to make keep-mode persistence explicit. Then align src/agents tests and the public subagent docs with that decision.
