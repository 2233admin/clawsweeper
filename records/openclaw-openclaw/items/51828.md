## Summary

Keep open. Current main has mitigations for cron startup catch-up and channel health-monitor restart churn, and the KeepMyClaw comment is useful external snapshot context, but the concrete subagent session-retention gap remains: default keep-mode subagent runs still avoid archive/delete, while the cron session reaper still only targets cron run sessions. The narrower #51767 thread remains the best canonical place for the remaining implementation decision.

## Best Possible Solution

Keep this issue open until maintainers either resolve #51767 or explicitly split/close this umbrella as superseded by narrower canonical issues. The concrete remaining code/docs decision is the subagent session-retention contract: either archive/delete default keep-mode subagent sessions after agents.defaults.subagents.archiveAfterMinutes, add a separate subagent/session retention policy, or update docs/tests to make default persistence explicit.
