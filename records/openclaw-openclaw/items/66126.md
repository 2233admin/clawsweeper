## Summary

Keep open. Current main still retries the original embedded-runner prompt after overflow compaction and does not carry a durable current-turn identity or inbound-message ledger through the retry path, so the central compaction-resume bug remains valid.

## What This Changes

The issue reports that overflow compaction retry treats cron/system prompts as fresh input, which can re-run deterministic failing tool calls and grow context again.

## Best Possible Solution

Define and implement a stable compaction-resume contract for embedded runs: preserve current-turn identity and original inbound message metadata across compaction, distinguish resumed attempts from fresh user/system/cron input, avoid replaying deterministic failed tool cascades as new work, and cover cron/system-event overflow cases with regression tests tied to #59618 and #63216.
