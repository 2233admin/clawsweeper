## Summary

Keep open. Current main still has the same-PID legacy lock reclamation path this PR is trying to fix, so the work is not obsolete; however, the latest diff still has a blocking correctness issue when current process starttime cannot be read, because it can reclaim a live same-PID lock before falling back to the PID-alive guard.

## What This Changes

The PR changes session write-lock self-orphan detection to use starttime and PID-liveness checks, and skips the legacy same-PID/no-starttime reclamation test.

## Best Possible Solution

Refine the branch in place so the no-verifiable-starttime path is conservative whenever the PID is alive, including stored-starttime locks where current starttime cannot be read. Replace the skipped test with explicit regression coverage for live same-PID/no-starttime and null-current-starttime/stored-starttime cases, while preserving dead-PID, recycled-PID, and payload-less orphan cleanup coverage.
