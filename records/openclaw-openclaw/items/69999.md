## Summary

Current main still does not implement the macOS wallboard shell proposed by this PR. The PR remains the active implementation candidate for REL-433, but it needs maintainer review and small fixes before merge, especially around packaged-app bundle lookup, WebKit private/KVC settings, deprecated activation, and re-open reload behavior.

## Best Possible Solution

Keep this PR open. If maintainers want the REL-433 wallboard direction, update the PR to use safe main-bundle-first resource lookup, avoid reloading the placeholder when re-presenting an existing controller, replace the deprecated activation call, and remove or explicitly justify the private/undocumented WebKit settings. Then validate with the Wallboard smoke tests, a macOS Swift build/test pass, and a manual --wallboard launch with and without an external display where hardware is available.
