## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior of pre-creating `memory/YYYY-MM-DD.md` before gateway or CLI agent runs. Main has partial mitigations for startup-context and `memory_get`, but they do not create the daily memory file, and the submitted PR is stale against the current CLI entrypoint.

## Best Possible Solution

Keep the PR open for maintainer follow-up, but rebase or replace it against current main. The best fix should derive the target from the registered memory flush plan, respect disabled memory flush and non-writable workspace policy, wire both `getReplyFromConfig` and `src/agents/agent-command.ts`, preserve existing daily memory content, treat concurrent creation as success, and add focused tests for fresh workspaces, existing files, disabled memory flush, read-only/sandboxed workspaces, and concurrent creation. Remove the unrelated manifest-registry test from this PR.
