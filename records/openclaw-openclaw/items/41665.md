## Summary

Keep this PR open. Current main still has the Windows onboarding problem the PR is trying to fix: installer OS filtering exists and is tested, but many bundled skill manifests still expose Homebrew-only install specs without `os` metadata. The PR discussion includes a concrete Windows Git Bash onboarding repro and the diff is directionally useful, but it is stale against current main and needs a rebase/regeneration before it can land.

## Best Possible Solution

Keep the PR open and ask for a rebase/update against current main. The best fix is a manifest-only refresh that adds `os` to every current brew-only install spec, uses top-level `metadata.openclaw.os` when the whole skill is platform-specific, preserves valid Windows-capable non-brew installers, and keeps the xurl fallback on the parser-supported `node` kind. A small manifest audit test would make this less likely to regress.
