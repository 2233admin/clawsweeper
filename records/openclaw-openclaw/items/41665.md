## Summary

Keep this PR open. Current main still has the Windows onboarding problem the contributor is trying to fix: installer OS filtering exists and is documented, but many bundled skill manifests still expose unscoped Homebrew install specs, so this PR is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. Ask the contributor to rebase and refresh the manifest audit against current main, add `os` metadata to every current brew-only install spec or top-level `metadata.openclaw.os` for whole-skill platform restrictions, preserve or add valid Windows-capable installers where they exist, handle the `xurl` node installer only with intentional reviewer approval, and consider adding a small manifest audit test for unscoped brew installers.
