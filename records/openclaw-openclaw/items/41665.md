## Summary

Keep this PR open. Current main still has the Windows onboarding problem described by the contributor: installer OS filtering exists and is documented, but many bundled skill manifests still expose Homebrew-only install specs without `os` metadata, so this PR is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open and ask for a rebase plus a refreshed manifest audit against current main. The best fix is a metadata-only update that adds `os` to every current brew-only install spec, uses top-level `metadata.openclaw.os` when an entire skill is platform-specific, preserves valid Windows-capable non-brew installers, changes xurl’s fallback to the parser-supported `node` kind only after reviewer approval, and ideally adds a small manifest audit test to prevent unscoped brew-only installers from regressing.
