## Summary

Keep this issue open. Current main still has trace provenance, browser-local identity, `senderLabel` display hints, and a recent `spawnedBy` chat event field, but it does not provide the requested durable human/main-agent/sub-agent sender identity contract across `chat.send`, `sessions.send`/`sessions.steer`, subagent completion delivery, transcript history/live events, and Control UI avatars. The prior PRs that explicitly targeted this space are closed unmerged, while #52498, #58110, #47216, and #72951 remain narrower or broader related work rather than replacements.

## What This Changes

The issue requests durable multi-role sender identity in the Chat UI so human, main-agent, and sub-agent messages remain distinguishable in multi-agent workflows.

## Best Possible Solution

Keep this open as the focused durable sender-identity tracker. The best path is an additive per-message sender metadata contract with sender id, sender kind, display name, source session/agent, and renderable avatar/color metadata; persist it in transcripts; expose it through `chat.history` and live chat/session events; carry it through `chat.send`, `sessions.send`, `sessions.steer`, and subagent announce delivery; and update Control UI grouping/avatar rendering. Coordinate with #52498, #58110, #47216, and #72951 without treating any of them as replacements.
