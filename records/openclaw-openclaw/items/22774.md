## Summary

Keep this issue open. Current main has useful partial provenance and browser-local UI identity support, but it still does not provide a durable human/main-agent/sub-agent sender identity contract through chat.send, sessions.send/sessions.steer, inter-session and subagent completion delivery, transcript persistence, chat.history/live events, and Control UI rendering.

## Best Possible Solution

Keep this issue open as the focused tracker for durable multi-agent sender attribution. The best path is an additive sender identity contract carrying sender id, sender kind, display name, source session/agent, and renderable avatar/color metadata through chat.send, sessions.send/sessions.steer, subagent announce delivery, transcript persistence, chat.history/live events, and Control UI rendering. Coordinate with #52498 for the session_send display slice and #58110 for broader multi-agent chat UX, but neither currently replaces this issue.
