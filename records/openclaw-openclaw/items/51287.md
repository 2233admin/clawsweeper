## Summary

Current main and the latest release mitigate the original two-minute pain by using a 30-minute default, but exec approval expiration is still not configurable. The request/wait path still sends hardcoded defaults, `approvals` and `tools.exec` schemas have no supported timeout key, and the related implementation PR remains open/incomplete while the earlier closing PR was closed unmerged.

## Best Possible Solution

Keep this issue open until OpenClaw exposes a supported exec approval expiration setting, validates it in the config schema, resolves it from the chosen config surface, threads it through `ExecToolDefaults` / `resolveExecConfig` / host approval request creation into the `exec.approval.request` payload and wait budgets, covers gateway and node approval paths, and documents the behavior. The practical path is to review and complete #25789 or replace it with an end-to-end PR.
