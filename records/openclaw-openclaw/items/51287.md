## Summary

Keep this issue open. Current main and the latest release use a 30-minute exec approval default, but the timeout is still hardcoded through the agent request/wait path and there is no supported config field in either `approvals` or `tools.exec`. Related PR #25789 is still open and incomplete, while #51292 was closed unmerged, so there is no merged implementation to close this against.

## Best Possible Solution

Keep this issue open until OpenClaw exposes a supported exec approval expiration setting, validates it in the config schema, resolves it from the chosen config surface, threads it through `ExecToolDefaults` / `pi-tools` into the exec approval request payload and wait RPC budgets, covers gateway/CLI/node approval paths, and documents the behavior. The practical path is to complete or replace #25789 with end-to-end wiring and tests.
