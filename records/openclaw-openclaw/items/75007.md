## Summary

Keep open. Current main still stages the backup manifest under ambient `TMPDIR`, while macOS service environments intentionally set `TMPDIR` under the OpenClaw state directory; PR #75031 is an open unmerged closing implementation candidate, so this issue should stay open until that fix or an equivalent lands.

## What This Changes

Fix `openclaw backup create` so backup staging cannot live under the archived OpenClaw state payload when macOS service `TMPDIR` points to `~/.openclaw/tmp`.

## Best Possible Solution

Land PR #75031 or an equivalent focused change that stages backup metadata in a trusted temp root outside every archived source, and keep a regression test asserting exactly one root `manifest.json` when service `TMPDIR` points under the state directory.

## Reproduction Assessment

Yes. The current-main static path is high-confidence: macOS service env sets `TMPDIR` under state, backup creation writes `manifest.json` under `os.tmpdir()`, and tar receives both that manifest and the recursive state source. I did not create a live archive because this review was required to keep the checkout read-only.

## Solution Assessment

Yes in direction. Replacing the ambient `os.tmpdir()` dependency with `resolvePreferredOpenClawTmpDir()` or a backup-specific trusted temp root outside included sources is the narrow maintainable fix; PR #75031 appears to target that boundary.
