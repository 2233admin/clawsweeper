## Summary

Keep open. Current main still stages the backup manifest under ambient `os.tmpdir()` while macOS service startup can set `TMPDIR` inside the OpenClaw state tree, and the active open PR #75031 is the focused implementation candidate with closing syntax.

## What This Changes

The issue reports that `openclaw backup create` can package `manifest.json` twice when macOS LaunchAgent `TMPDIR` points under `~/.openclaw/tmp`.

## Best Possible Solution

Land PR #75031 or an equivalent focused fix that creates backup staging directories outside every included backup source and adds a regression asserting one root manifest when service `TMPDIR` points under state.

## Reproduction Assessment

Yes. The static reproduction path is high-confidence on current main: macOS service env sets `TMPDIR` under state, backup creation writes `manifest.json` under `os.tmpdir()`, and tar packs both the explicit manifest and the recursive state source.

## Solution Assessment

Yes in direction. Replacing the ambient `os.tmpdir()` dependency with a trusted OpenClaw temp root, plus a guard that staging is outside included sources, is the narrow maintainable fix.
