## Summary

Keep open. Current main still stages backup metadata under `os.tmpdir()` while macOS service startup sets `TMPDIR` under the OpenClaw state directory, and PR #75031 is open with closing syntax as the focused implementation candidate.

## What This Changes

The issue reports that `openclaw backup create` can archive `manifest.json` twice when `TMPDIR` is set to `~/.openclaw/tmp` on macOS.

## Best Possible Solution

Review and land PR #75031 or an equivalent focused fix that stages backup metadata outside every included source tree, fails before archiving when no safe temp root is available, and proves a state-local `TMPDIR` produces exactly one root manifest entry.

## Reproduction Assessment

Yes. Current main gives a high-confidence static reproduction path: macOS service setup places `TMPDIR` under state, backup creation stages `manifest.json` under `os.tmpdir()`, and tar archives both that explicit file and the state source tree.

## Solution Assessment

Yes in direction. The narrow maintainable fix belongs in backup archive creation by using `resolvePreferredOpenClawTmpDir()` or a backup-specific guarded staging root outside all planned assets, with regression coverage for the state-local `TMPDIR` case.
