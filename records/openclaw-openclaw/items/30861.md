## Summary

Keep this issue open. Current main still does not provide the requested persistent active-agent plus workspace indicator in the Control UI chat header. The existing UI has a grouped session selector and the backend already exposes agent workspace metadata, but the header has no always-visible workspace label/tooltip and session switches do not produce explicit confirmation feedback. The directly related implementation PR #31602 is closed unmerged, and no open PR currently mentions this issue.

## Best Possible Solution

Keep this issue open as the Control UI tracker. Implement a minimal persistent chat-header context indicator derived from the active sessionKey plus agents.list, showing agent id/name and workspace path with a full-path tooltip, updating through switchChatSession and session-selection paths, surviving reload through existing URL/session state, adding clear switch feedback, and staying quiet for single-agent setups. Closed PR #31602 is useful prior art but is not shipped or canonical; related #69668 should remain separate because it asks for broader team/project hierarchy.
