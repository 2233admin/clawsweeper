## Summary

Keep open. Current `main` still has the legacy SQLite migration gap described by the reporter, and the same-author PR #65767 is an open implementation candidate rather than shipped behavior.

## What This Changes

The issue reports that the webhooks plugin `create_flow` action should create a managed Task Flow owned by the route-bound `sessionKey`, but fails on legacy SQLite `flow_runs` tables that still require `owner_session_key`.

## Best Possible Solution

Keep this issue open and review PR #65767, or land an equivalent maintainer patch, that rebuilds or otherwise normalizes legacy `flow_runs` tables so `owner_session_key` no longer blocks new inserts, preserves existing owner data, and adds regression coverage for legacy and partially migrated schemas plus managed webhook flow creation.
