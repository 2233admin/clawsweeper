## Summary

Current main still appears affected. The TaskFlow SQLite migration copies legacy `owner_session_key` values into `owner_key` but leaves a legacy `owner_session_key TEXT NOT NULL` column in place, while new flow inserts only write `owner_key`. The webhooks `create_flow` path still creates managed flows from the route-bound session, and open PR #65767 is a same-author implementation candidate for this exact migration gap.

## Best Possible Solution

Keep this issue open and review PR #65767, or land an equivalent maintainer patch, that rebuilds or otherwise normalizes legacy `flow_runs` tables so `owner_session_key` no longer blocks new inserts, preserves existing owner data in `owner_key`, and adds focused regression coverage for the legacy SQLite schema plus managed flow creation.
