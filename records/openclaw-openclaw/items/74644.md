## Summary

Keep open. Current main still requires provider auth before invoking audio/video media-understanding providers, and the issue is paired with open, unmerged PR #75005, which explicitly references this issue as its fix.

## What This Changes

Allow media-understanding audio/STT providers that explicitly represent local or no-auth services to run without auth-store credentials.

## Best Possible Solution

Review and land or revise PR #75005 as the canonical fix: add an explicit provider-owned synthetic/no-auth media auth hook, cover local audio/video providers, and keep ordinary providers without credentials failing closed.

## Reproduction Assessment

Yes. Current main reproduces from source: configure a plugin media audio provider without credentials and runProviderEntry reaches requireApiKey before calling transcribeAudio.

## Solution Assessment

Yes in direction, but not yet merged. An explicit opt-in synthetic auth hook is narrower and safer than bypassing media auth globally because it preserves the existing auth gate for providers that do not opt in.
