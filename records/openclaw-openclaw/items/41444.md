## Summary

Keep this issue open. Current main still omits the Anthropic OAuth beta wrapper when only model-level `anthropic-beta` headers are configured, and the direct Anthropic transport can still let model headers replace the OAuth-required beta. The same-author implementation PR #41461 is also still open and unmerged with closing intent, so the issue should wait for that PR to land, be replaced, or be closed.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #41461. The fix should preserve `oauth-2025-04-20` independently of `params.context1m`, merge or protect OAuth-required Anthropic betas when provider/model headers include `anthropic-beta`, continue stripping `context-1m-*` for OAuth/subscription tokens per current docs, and add a regression test for OAuth plus model-level `anthropic-beta` with no `params.context1m`.
