## Summary

This issue should stay open. The supplied discussion shows an open same-author implementation PR with closing syntax, and current main still leaves the reported model-header path unresolved: Anthropic OAuth beta preservation only runs inside a conditional wrapper, while model/provider headers are still a supported request surface and can override the OAuth default `anthropic-beta` header before transport send.

## Best Possible Solution

Keep this issue open and review/land #41461 or replace it with an equivalent maintainer patch. The fix likely belongs in the Anthropic wrapper or Anthropic transport header merge path: merge Anthropic beta lists instead of replacing OAuth-required defaults, strip unsupported `context-1m-*` for OAuth/subscription tokens, and add regression coverage for model-level `anthropic-beta` plus `sk-ant-oat` auth with no `params.context1m`.
