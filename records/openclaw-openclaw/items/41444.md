## Summary

Keep this issue open. Current main still allows a model-level `anthropic-beta` header to replace the OAuth-required Anthropic beta because the beta wrapper only runs when extra params produce betas, and the transport header merge is last-wins. There is also an open same-author implementation PR with closing syntax for this issue, so cleanup policy says not to close it before that PR or an equivalent fix is resolved.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #41461. The fix should merge Anthropic beta lists at the Anthropic wrapper or direct transport header boundary so OAuth-required betas survive model/provider headers, and it should add regression coverage for model-level `anthropic-beta: context-1m-2025-08-07` with `sk-ant-oat` auth and no `params.context1m`.
