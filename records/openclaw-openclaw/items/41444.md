## Summary

Keep this issue open. Current main still gates Anthropic OAuth beta preservation behind extra-param beta resolution, so a model-level `anthropic-beta` header can still replace the OAuth-required default. There is also an open same-author implementation PR with closing syntax for this issue, so cleanup policy says the issue should remain open until that PR or an equivalent fix is resolved.

## Best Possible Solution

Keep this issue open and review or replace PR #41461 with a maintainer-approved patch. The fix should live in the Anthropic wrapper or Anthropic transport header merge path, merging Anthropic beta lists instead of replacing OAuth-required defaults, and should add regression coverage for model-level `anthropic-beta` plus `sk-ant-oat` auth without `params.context1m`.
