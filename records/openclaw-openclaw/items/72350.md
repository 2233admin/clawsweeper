## Summary

Keep open for maintainer review. Current main and the latest release snapshot still warm the context-window cache for canonical `openclaw tui`, the related #75104 was closed unmerged rather than superseding this work, and the PR patch appears narrow with no blocking correctness finding; the remaining decision is whether the TUI aliases should share the same startup behavior.

## What This Changes

This PR adds `tui` to the eager context-window warmup skip list and adds a regression assertion for `shouldEagerWarmContextWindowCache(["node", "openclaw", "tui"])`.

## Best Possible Solution

Keep the branch under maintainer review and land the narrow skip-list/test change if canonical `openclaw tui` should avoid eager model/plugin discovery during startup; handle any `chat` and `terminal` alias behavior as an explicit maintainer decision in this PR or a follow-up.

## Reproduction Assessment

Yes for the gate-level behavior: current main parses `tui` as the primary command, does not include it in the skip set, and calls the warmup loader at import time when the gate is true. The human-visible input-lag timing itself was not reproduced during this read-only pass.

## Solution Assessment

Yes for canonical `openclaw tui`: adding `tui` to the existing skip list is the narrowest maintainable fix and the added assertion covers that boundary. It is unclear whether the same patch should include `chat` and `terminal`, which is a separate product/maintainer scope call.
