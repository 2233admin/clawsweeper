## Summary

Keep this PR open. Current main and the latest tagged release still omit `tui` from the eager-warmup skip list, while the PR is a narrow implementation that appears correct; the remaining action is maintainer review of the TUI startup boundary, especially whether aliases should share the same behavior.

## What This Changes

The PR adds `tui` to `SKIP_EAGER_WARMUP_PRIMARY_COMMANDS` and adds a regression assertion that `shouldEagerWarmContextWindowCache(["node", "openclaw", "tui"])` returns false.

## Best Possible Solution

Land the narrow skip-list and regression-test change if maintainers want canonical `openclaw tui` to avoid startup warmup, then decide separately whether `chat` and `terminal` should be added to the same skip set or intentionally keep eager metadata warmup.

## Reproduction Assessment

Yes for the gate-level behavior: current main has `tui` registered as a primary CLI command and absent from the skip set, so the warmup gate remains enabled for `openclaw tui`. A live PTY timing reproduction was not run in this read-only review.

## Solution Assessment

Yes for the canonical command: adding `tui` to the existing skip set is the narrowest maintainable fix and the added assertion covers that behavior. It is unclear whether aliases should be included in the same PR, which is a maintainer/product boundary decision rather than a correctness defect in this patch.
