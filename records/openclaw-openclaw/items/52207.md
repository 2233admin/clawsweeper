## Summary

Keep open. Current main already shipped the SearXNG provider portion, but the Tavily generic `web_search` freshness support is still missing, and this mixed-scope PR now also carries unrelated provider-runtime changes with concrete review findings.

## What This Changes

The PR branch adds SearXNG web-search plugin/docs/config/onboarding wiring, adds Tavily freshness-to-days support for generic web search, updates generated config/audit surfaces, and changes Pi embedded runner provider normalization tests/code.

## Best Possible Solution

Keep the shipped SearXNG implementation from #57317, then land Tavily freshness as a narrow Tavily-plugin change that adds a generic `web_search` freshness field, maps supported values to Tavily `days`, includes the value in request/cache behavior, and updates focused Tavily/web-search tests and docs only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Keep custom OpenRouter endpoints on their configured API:** `src/agents/pi-embedded-runner/model.provider-normalization.ts:25-38`
  - body: The new `isOpenRouter` branch rewrites every OpenRouter model using `openai-completions` to `openai-responses`, even when the user configured a custom/proxy `baseUrl`. Existing proxies may only implement Chat Completions, so this unrelated change can break working OpenRouter deployments.
  - confidence: 0.86

  - confidence: 0.8
