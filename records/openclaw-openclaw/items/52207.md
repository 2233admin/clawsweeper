## Summary

Keep open. Current main has already shipped the SearXNG provider portion via #57317, but it does not implement the PR's Tavily freshness support for the generic `web_search` provider. The branch still contains a useful Tavily idea, but it is mixed with superseded SearXNG work and unrelated provider-runtime/config changes that introduce blocking regressions.

## What This Changes

The PR branch adds a bundled SearXNG web-search plugin with docs/config/onboarding wiring, adds Tavily generic `web_search` freshness-to-days support, updates generated config/security surfaces, and changes Pi embedded provider normalization code.

## Best Possible Solution

Keep the shipped SearXNG implementation from #57317. If Tavily freshness is still desired, land it as a narrow Tavily-plugin change against current main: add the generic freshness field using OpenClaw's current `day/week/month/year` shape, map it to the appropriate Tavily request field, include it in cache behavior, and update focused Tavily docs/tests without adding legacy config paths or unrelated provider normalization changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Preserve custom OpenRouter endpoints:** `src/agents/pi-embedded-runner/model.provider-normalization.ts:25-38`
  - body: The new `isOpenRouter` branch rewrites every OpenRouter `openai-completions` model to responses transport, even when a user configured a custom/proxied `baseUrl`. Existing OpenRouter-compatible proxies may only implement chat completions, so this unrelated change can break working deployments.
  - confidence: 0.86
- **[P2] Do not accept unread Tavily config paths:** `src/config/zod-schema.agent-runtime.ts:319-325`
  - body: Adding `tools.web.search.tavily.apiKey` to the strict runtime schema makes that config validate, but Tavily credential resolution still reads the plugin-owned path or `TAVILY_API_KEY`. Users can follow the accepted config and still get missing-key errors at runtime.
  - confidence: 0.82
- **[P2] Count keyless SearXNG configs in audits:** `src/security/audit-extra.sync.ts:335-339`
  - body: SearXNG is intentionally usable with only a base URL, but this audit heuristic treats web search as configured only when an API key or key env var exists. Active keyless SearXNG search can therefore be underreported in exposure auditing.
  - confidence: 0.78
- **[P2] Avoid defaulting SearXNG to blocked localhost:** `extensions/searxng/src/config.ts:5`
  - body: The provider defaults to `http://localhost:8080` while private-network access defaults to false and is enforced by the guard. A default or onboarding-style SearXNG selection can therefore point at localhost and reject its own request until a separate flag is set.
  - confidence: 0.74
