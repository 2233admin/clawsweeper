## Summary

Current main supersedes the SearXNG portion through the merged bundled SearXNG provider, but it does not implement this PR’s Tavily `web_search` freshness support. The Tavily generic web-search provider still exposes only `query` and `count`, `runTavilySearch` has no `days`/freshness input or request body field, and the docs explicitly say Tavily only supports `query` and `count` through `web_search`. This PR also includes unrelated PI/OpenRouter model-runner changes, so it should not be auto-closed as fully implemented or merged as-is.

## Best Possible Solution

Keep this PR open for maintainer judgment or ask the contributor to split it. The SearXNG provider work should be considered superseded by #57317/current main. The remaining useful path is a focused Tavily `web_search` freshness change that updates the provider schema, maps generic freshness to Tavily’s supported recency parameter, includes that value in cache keys and request bodies, updates docs, and adds targeted tests, with the unrelated PI/OpenRouter changes removed.
