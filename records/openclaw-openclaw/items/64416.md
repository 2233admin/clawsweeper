## Summary

Keep open. Current main still lacks the PR's ACP capability-normalization prompt and task-level parent-spawned ACP silent-wake path, while the branch has blocking prompt-boundary and restart-persistence defects and overlaps the still-open CompletionDeliveryGate PR.

## What This Changes

The PR adds task-level spawnedBy metadata and ACP silent parent wake routing, changes subagent completion prompt/output handling, suppresses deferred farewell delivery, and appends ACP capability-normalization guidance to gateway agent turns.

## Best Possible Solution

Revise or consolidate the ACP completion-delivery design so structured internal events remain authoritative, any child output used in plain prompts is delimiter-escaped, parent lineage is restart-safe, and regression coverage includes delimiter injection plus restored ACP task delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Escape completion text before building steer messages:** `src/agents/subagent-announce.ts:209`
  - body: For completion announces, the new steer message interpolates `findings` directly into prompt text. Child output is untrusted; if it contains an OpenClaw internal-context delimiter on its own line, `prependInternalEventContext()` treats the body as already wrapped and skips the real structured `internalEvents`, letting child text masquerade as runtime context.
  - confidence: 0.88
- **[P2] Persist spawnedBy before relying on delivery routing:** `src/tasks/task-registry.ts:1027`
  - body: The new silent-wake branch depends on `latest.spawnedBy`, but the task SQLite row/schema/select/upsert paths in this PR do not persist that field. After a gateway restart, restored ACP task records lose the metadata and can fall back to the visible requester delivery path this branch is meant to avoid.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete prompt-boundary concern by placing unescaped child completion text in a steer message that is later combined with structured internal events.

Concerns:

- **[medium] Unescaped child output can spoof runtime context:** `src/agents/subagent-announce.ts:209`
  - body: The PR's completion steer prompt includes untrusted child output before internal event context is prepended. Because current runtime-context detection skips prepending when it sees an internal-context delimiter, injected child text can be treated as runtime context instead of data.
  - confidence: 0.88
