## Summary

Keep open. Current main still does not contain the PR's ACP capability-normalization prompt or task-level parent-spawned ACP silent-wake delivery path, and the PR overlaps the still-open CompletionDeliveryGate PR while introducing blocking prompt-boundary and restart-safety issues.

## What This Changes

The PR changes ACP/subagent completion delivery by adding task-level spawnedBy metadata and silent parent wake logic, changing completion announce prompts to prefer live output/structured events, suppressing a deferred farewell hook, and appending ACP capability-normalization guidance in gateway agent runs.

## Best Possible Solution

Consolidate this work with the CompletionDeliveryGate direction or revise this PR into a focused fix that preserves structured internal events as the authoritative completion source, escapes any child output used in plain prompts, persists or derives the lineage needed for restart-safe silent wake routing, and adds regression coverage for delimiter injection plus restart-restored ACP tasks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Escape completion text before building steer messages:** `src/agents/subagent-announce.ts:209`
  - body: For completion announces, the new steer message interpolates `findings` directly into the prompt. Child output is untrusted; if it contains `<<<BEGIN_OPENCLAW_INTERNAL_CONTEXT>>>` on its own line, `prependInternalEventContext()` treats the body as already wrapped and skips the real structured `internalEvents`, letting child text masquerade as runtime context. Escape internal-context delimiters or build the visible prompt from the sanitized internal-event formatter.
  - confidence: 0.88
- **[P2] Persist spawnedBy before relying on it for delivery:** `src/tasks/task-registry.ts:1027`
  - body: The new silent-wake branch depends on `latest.spawnedBy`, but the task SQLite row/schema/select/upsert paths in this PR do not persist that field. After a gateway restart, restored ACP task records lose the metadata and fall back to the old visible requester delivery path, reintroducing the duplicate/noisy completion behavior this branch is meant to prevent.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete prompt-boundary concern by placing unescaped child completion text in an agent steer message that is later combined with structured internal events.

Concerns:

- **[medium] Unescaped child output can spoof runtime context:** `src/agents/subagent-announce.ts:209`
  - body: The PR's completion steer prompt includes untrusted child output before internal event context is prepended. Because current runtime-context detection skips prepending when it sees an internal-context delimiter, a child can inject that delimiter and cause its text to be treated as runtime context instead of data.
  - confidence: 0.88
