## Summary

Keep this PR open. Current main at 7a23b2d945cc2d2e5c452e4641213e71de8dcc66 still has the responsePrefix early-abort gap: prefix context is seeded only with identity metadata, fast-abort replies return before onModelSelected can populate model/thinking fields, and the linked bug plus alternate implementation PR #45315 remain open rather than shipped.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased equivalent, and review it alongside #45315. The best fix is to seed early responsePrefix context from accurate session/runtime metadata before fast-abort delivery, thread the needed sessionKey or context through current extension/gateway call sites, keep onModelSelected as the authoritative later override, and add regression coverage for early abort plus normal post-selection behavior before closing #45314.
