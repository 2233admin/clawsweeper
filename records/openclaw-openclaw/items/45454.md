## Summary

Keep this PR open. Current main still initializes responsePrefix context with only identity metadata, fast-abort replies still return before model selection populates model/thinking fields, and the linked bug #45314 plus alternate PR #45315 remain open rather than shipped.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased equivalent, and review it alongside #45315. The best fix is to seed early responsePrefix context from accurate session/runtime metadata before fast-abort delivery, thread the needed sessionKey or context through current channel/plugin/gateway call sites, preserve onModelSelected as the later authoritative override, and add regression coverage for fast-abort plus normal post-selection behavior before closing #45314.
