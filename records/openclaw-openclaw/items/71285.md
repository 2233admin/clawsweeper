## Summary

Keep open. Current main still lacks a memory-search exclusion contract, and the default memory indexing/watch paths can still include `memory/dreaming/**`; the related merged dreaming hardening only blocks promotion/self-ingestion, not search indexing or watcher scope.

## What This Changes

Request adds a configurable `memorySearch.ignorePaths` exclusion so dreaming reports and other chosen directories can be omitted from memory search indexing and file watching.

## Best Possible Solution

Introduce one documented `memorySearch.ignorePaths` contract with consistent glob semantics, feed it through config schema/types/resolution, builtin file listing, builtin and QMD watcher/update paths, reindex scope hashing, generated config docs, and focused regression tests. Keep dreaming exclusion opt-in unless maintainers explicitly choose a default.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place a markdown file under `memory/dreaming/light/` and follow the default memory indexing path: builtin `listMemoryFiles` recurses through `memory/`, and QMD uses `memory/**/*.md`; no live write/run was performed because this review was read-only.

## Solution Assessment

Yes. A config-based exclusion is the narrowest maintainable direction for this issue, provided the implementation uses shared glob matching across indexing and watching instead of a hardcoded `dreaming` special case or literal directory-only extraction.
