## Summary

Current main still lacks the requested configurable memory-search exclusion. The config schema/types only expose `extraPaths`, builtin indexing still recursively walks `memory/`, QMD default collections still include `memory/**/*.md`, and the builtin watcher only has hardcoded dependency/cache directory ignores, so `memory/dreaming/**` can still be indexed and watched. The related dreaming PRs address promotion/self-ingestion, not search index or watcher scope.

## What This Changes

Add a configurable memory-search ignore-path mechanism so users can exclude `memory/dreaming/**` and other directories from memory search indexing and watch scope.

## Best Possible Solution

Add an additive, documented memory-search exclusion contract, likely `memorySearch.ignorePaths`, that applies consistently to builtin file enumeration, chokidar watching, QMD default/custom collection scope, and reindex scope hashing, with `memory/dreaming/**` covered by focused opt-in tests rather than an undocumented one-off special case.
