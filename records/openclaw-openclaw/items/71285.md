## Summary

Keep open. Current main still lacks a configurable memory-search exclusion contract, and the merged dreaming hardening only covers promotion/self-ingestion paths rather than built-in indexing, watchers, or QMD default memory collections. The issue is narrow enough for a focused repair lane.

## What This Changes

Request adds an opt-in `memorySearch.ignorePaths` config so users can exclude `memory/dreaming/**` and other directories from memory search indexing and file watching.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` contract with shared matching semantics across schema/types, resolved agent config, built-in file listing, built-in watcher filtering, QMD collection/update behavior where feasible, reindex-state hashing, generated config docs, and focused regression tests. Keep `memory/dreaming/**` opt-in unless maintainers explicitly choose a default exclusion.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to add a markdown file under `memory/dreaming/light/` and use current defaults: built-in `listMemoryFiles` recursively walks `memory/`, and QMD default collections bind `memory` with `**/*.md`.

## Solution Assessment

Yes. The requested config-based exclusion is the narrowest maintainable direction, provided the implementation uses shared glob semantics and wires the setting through all memory backends instead of hardcoding `dreaming` or only changing one watcher predicate.
