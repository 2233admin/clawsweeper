## Summary

Keep open. Current main still lacks a configurable memory-search exclusion contract, and the adjacent dreaming fixes only block promotion/self-ingestion; they do not prevent `memory/dreaming/**` markdown from being indexed, watched, or returned by memory search.

## What This Changes

The issue requests a configurable `memorySearch.ignorePaths` exclusion so `memory/dreaming/**` and other ignored directories can be omitted from memory search indexing and file watching.

## Best Possible Solution

Add a single documented memory-search exclusion contract, likely `memorySearch.ignorePaths`, with shared glob matching and consistent propagation through config types/schema, builtin file listing, memory-core watcher filtering, QMD collection/search behavior where feasible, reindex-state hashing, generated config docs, and focused regression tests. Keep `memory/dreaming/**` opt-in unless maintainers explicitly choose a default exclusion.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create a markdown file under `memory/dreaming/light/` and use the default memory indexing path: current `listMemoryFiles` walks `memory/` recursively, and the watcher ignores only a hardcoded dependency/cache directory list.

## Solution Assessment

Yes. A config-based exclusion is the narrowest maintainable direction for this report, but the implementation should not reuse a narrow literal-directory matcher if the documented contract says glob patterns; shared matcher behavior across builtin and watcher paths is safer.
