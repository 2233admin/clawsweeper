## Summary

Keep open. Current main has several relevant mitigations after the reported 2026.4.26 build, but they are partial: readiness now reports event-loop health when it can respond, restart deferral has a default cap, task inspection can project stale records as lost, and local log tailing has a loopback fallback. The core reported failure, a CPU-hot gateway main thread starving `/readyz` and control-plane traffic under orchestration/plugin load, is not proven fixed because those paths still run in the same Node event loop and plugin prompt hooks still execute in-process.

## What This Changes

The issue requests gateway control-plane responsiveness under orchestration load, bounded reload deferral/stale active-work accounting, plugin hook offload or guidance, diagnostics for event-loop stalls, and resilient `openclaw logs --follow` behavior.

## Best Possible Solution

Keep this as a core reliability follow-up: preserve the shipped mitigations, then define a maintainer-owned plan for control-plane survivability when agent/plugin work saturates the main event loop, plus inspectable/reapable active-work blockers. Route the log-follow-only behavior through #74583 and the larger runtime split through #42026 rather than closing this concrete incident against either one.
