## Summary

Keep this issue open. Current main has useful new mitigations for this report, including event-loop health in detailed readiness and a default timeout for `before_prompt_build` hooks, but the central failure is not fully closed: `/readyz` still runs on the same Node event loop, reload/restart deferral can still wait indefinitely by default, stale active task accounting is only partly reconciled, and `openclaw logs --follow` still exits on gateway RPC errors instead of reconnecting.

## Best Possible Solution

Keep this issue open as a gateway reliability bug. The best path is a focused maintainer fix that captures current-main stall profiles, makes readiness/stability expose event-loop pressure without pretending the event loop is isolated, adds abortable or offloaded bounds for prompt/runtime plugin work, makes reload deferral safe by default or gives operators an explicit reaper/force path with blocker details, and changes `openclaw logs --follow` to reconnect/resubscribe across normal Gateway close or reload churn. Add regression tests around stale active task bookkeeping, reload deferral timeout/reaper behavior, prompt hook timeout semantics, and logs-follow reconnect behavior.
