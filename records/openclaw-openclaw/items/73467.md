## Summary

Keep this issue open. The report is recent, concrete, and still maps to current main: existing code has useful mitigations and diagnostics, but it does not prove the reported gateway main-thread starvation and indefinite reload deferral are fixed. The relevant paths still run in the same Node gateway process, restart deferral can wait indefinitely by default, and prompt-mutating plugin hooks can still execute sequentially without a core timeout.

## Best Possible Solution

Keep the issue open and treat it as a gateway reliability bug. The best path is to capture current-main stability/CPU-profile evidence, then add a focused fix in the gateway/agent/plugin runtime: expose event-loop pressure in readiness/stability, add bounded or abortable prompt-build plugin hook execution, provide an inspectable active-work blocker source, and make reload/restart deferral safe by default or clearly operator-configurable. Add regression coverage for blocked prompt hooks or stale active bookkeeping plus the expected /readyz/reload behavior.
