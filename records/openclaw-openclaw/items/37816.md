## Summary

Keep this issue open. Current main still does not provide the requested opt-in Telegram mode that mirrors the Web UI by sending each agent activity step as a durable separate message. Telegram has preview streaming, optional edited tool-progress text, `/reasoning` visibility, and `/verbose` tool summaries in some runtimes, but the public config/schema still only expose preview modes `off | partial | block | progress`, with `progress` mapped back to `partial` on Telegram.

## Best Possible Solution

Keep this open as bundled Telegram plugin/product work. The best path is an explicit opt-in `channels.telegram` mode or flag for durable activity messages, with schema/docs/tests, default behavior unchanged, NO_REPLY/internal-run suppression, rate-limit controls, and sensitive-output safeguards. Reuse current replyOptions callbacks where sufficient; add a narrow generic channel/plugin activity event seam only if current callbacks cannot represent the Web UI-style events needed.
