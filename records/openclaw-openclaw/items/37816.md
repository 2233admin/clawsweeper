## Summary

Keep this issue open. Current main still lacks the requested Telegram mode for durable, separate, Web-UI-style agent activity messages. Telegram has preview edits, tool-progress lines inside the preview, separate reasoning behavior for `/reasoning`, and final-only suppression paths, but there is no public `full` streaming mode, `activityMessages` flag, or durable per-activity message delivery contract. The related PR I found was closed unmerged and only added an issue-bootstrap note, so it does not resolve the request.

## What This Changes

Request an opt-in Telegram mode or flag that renders Web-UI-like agent activity, reasoning, tool calls, memory updates, and the final answer as durable separate messages while excluding internal `NO_REPLY` housekeeping turns.

## Best Possible Solution

Keep this open as bundled Telegram plugin/product work. The best path is an explicit opt-in `channels.telegram` mode or flag for durable activity messages, with defaults unchanged, internal/`NO_REPLY` run suppression, sensitive-output safeguards, coalescing/rate limiting, schema/config UI/docs updates, and focused Telegram dispatch tests. Reuse existing reply option callbacks if they can represent the needed events; add a narrow generic channel/plugin activity-event seam only if current callbacks are insufficient.
