## Summary

Keep open. Current main has custom per-session labels in Control UI, but the requested account-config-driven default label for newly created channel sessions is still missing. Existing channel account configs commonly expose `name`, not `displayLabel`, and that account label is surfaced for channel status/account UI rather than applied to `session.label` during inbound session creation.

## Best Possible Solution

Keep this open as a valid core/channel-session feature request. The likely implementation should reuse or explicitly alias the existing channel account `name` field, or add a documented generic `displayLabel` if maintainers prefer that public name, then wire inbound session metadata creation to resolve the account label from `ctx.Provider`/`ctx.AccountId` and set `session.label` only for new or unlabeled sessions. Add tests around `recordSessionMetaFromInbound`/gateway session rows and update channel/config docs so users do not have to edit `sessions.json` manually.

## Risks / Open Questions

- There is already an account-level `name` concept; adding a separate `displayLabel` without an alias/migration decision could create two overlapping display-name fields.
- Any implementation should avoid overwriting user-edited `session.label` values and should only apply the account-derived default when creating or first labeling an unlabeled session.
