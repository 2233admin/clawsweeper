## Summary

Current main still does not implement the requested account-config-driven default label for newly created channel sessions. It supports manual per-session labels and channel account `name` metadata for account/status UI, but inbound session metadata does not look up the configured account `name` or a `displayLabel` field and does not seed `session.label` or `displayName` from channel account config.

## Best Possible Solution

Keep this open as a valid core/channel-session enhancement. The conservative implementation path is to reuse the existing channel account `name` as the default session label, or add `displayLabel` as a documented alias if maintainers prefer that public spelling, then update inbound session metadata to resolve the provider/account from `ctx.AccountId` and seed `session.label` or `displayName` only for new or unlabeled sessions. Add focused coverage around `recordSessionMetaFromInbound`, gateway session-row display data, and the Control UI session picker, plus docs for the chosen account-label configuration.

## Risks / Open Questions

- Adding a separate `displayLabel` could duplicate the existing account `name` concept unless maintainers choose one spelling or define a clear alias/migration policy.
- The implementation should only seed labels for new or unlabeled sessions so user-edited `session.label` values are preserved.
- This crosses channel config, session metadata, gateway session rows, and Control UI display behavior, so it needs coordinated contract docs and regression tests.
