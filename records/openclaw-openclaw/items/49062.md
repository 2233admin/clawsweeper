## Summary

Keep this issue open. Current main still has no defined Telegram `channel_post` policy for sender-scoped `groups[chatId].allowFrom`: native command auth is still built around `msg.from` sender identity, the Telegram docs describe `groupAllowFrom` / per-group `allowFrom` as sender filters without a `channel_post` caveat, and the related same-author PR #49063 is open, unmerged, and explicitly leaves this semantics question out of scope.

## Best Possible Solution

Keep the issue open until maintainers define the product/security policy for Telegram `channel_post` with sender-scoped `groups[chatId].allowFrom`: either document and implement explicit-chat-only authorization that ignores sender-scoped overrides on channel posts, or fail closed with a clear unsupported-configuration message and docs. Review/resolve the adjacent PR #49063 separately because it intentionally does not settle this follow-up semantics question.
