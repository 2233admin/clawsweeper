## Summary

Keep this PR open. Current main still has the transcript-header behavior the PR targets: new mirrored session transcript headers are written with `cwd: process.cwd()` instead of an explicit target agent workspace. The linked bug report remains open and this PR uses closing syntax for it, so this is still an implementation candidate rather than a conservative cleanup close.

## Best Possible Solution

Keep this PR open or supersede it with a cleaned-up fix. The best implementation is to write transcript headers with an explicit workspace derived from already-available session or agent context, avoid ambient config reloads in the post-delivery mirror path, forward a consistent resolved agent id through session-file resolution, and add the focused per-agent cwd isolation regression test.

## Risks / Open Questions

- The submitted patch likely needs revision before merge because the review comment identifies a real reliability risk: loading config inside post-delivery transcript mirroring could turn a bookkeeping failure into a delivery failure after the user already received the message.
- The Greptile summary flags another correctness gap in the submitted patch: the locally resolved/defaulted `agentId` was used for header cwd but not forwarded to session-file resolution, which could leave header and file-path context divergent in edge cases.
