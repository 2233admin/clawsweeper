## Summary

Current main still has the reported exec-approval identity mismatch: executable resolution stores both the PATH/symlink path and the realpath, allowlist path matching still compares path-shaped patterns against `resolvedPath` only, and approval-backed execution pins to `resolvedRealPath` when available. The GitHub context also shows open PR #73543 uses closing syntax for this issue, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while maintainers review #73543. The best fix is either to land that PR with a clearly scoped matcher fix and leave any remaining realpath-trust surfaces tracked, or expand it into a coherent executable-identity model across allowlist matching, safeBins, skill auto-allow, Allow Always persistence, audit metadata, and diagnostics, with regression tests for realpath allowlisting through symlinks and symlink target drift.
