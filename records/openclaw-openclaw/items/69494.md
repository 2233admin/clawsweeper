## Summary

Keep open. Current main still resolves the macOS browser-proxy local gateway token/password path as plain strings only, so a structured SecretRef for `gateway.auth.token` or `gateway.auth.password` is ignored. The PR is active and targets the implicated code path with tests requested in review discussion.

## Best Possible Solution

Keep #69494 open and review or land this PR, or an equivalent patch, after final macOS validation. The best implementation remains an app-side SecretInput resolver seam for local `gateway.auth.token` and `gateway.auth.password`, with remote auth behavior unchanged unless explicitly product-approved, keychain lookup kept off actor/main refresh paths, and regression coverage for the actual `localConfig()` path used by `MacNodeBrowserProxy`.
