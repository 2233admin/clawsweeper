## Summary

Current main still lacks the macOS desktop browser-proxy SecretRef auth behavior targeted by this PR. The PR remains relevant: `GatewayEndpointStore.localConfig()` still resolves local `gateway.auth.token` and `gateway.auth.password` only from strings/launchd/env, while `MacNodeBrowserProxy.defaultEndpoint()` forwards those unresolved values into browser proxy requests. The PR diff addresses the root bug with an injected keychain resolver and tests, but it also adds a security-sensitive `/usr/bin/security` subprocess path that should be reviewed before merge.

## Best Possible Solution

Keep this PR open for maintainer review and land it, or land an equivalent patch. The fix belongs in the macOS app `GatewayEndpointStore` local auth path: resolve supported local `gateway.auth.token` and `gateway.auth.password` SecretRefs through a small injected keychain resolver seam, preserve env and launchd precedence, keep remote auth behavior unchanged unless separately designed, cover both helper resolution and the actual `localConfig()` browser-proxy path, and bound/review the keychain subprocess behavior.
