## Summary

Keep open. Current main still has the stale raw-tool-output path around compaction markers, and this issue is already paired with open overlapping fixes, including PR #74113 with closing syntax, so it should not be closed before maintainers consolidate and land a fix.

## What This Changes

The issue requests preventing subagent announce and completion capture from returning stale raw tool output when auto-compaction delays the final assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats compaction system markers as continuation boundaries for both announce output selection and frozen completion capture, preserves timeout partial-progress behavior, and adds regression coverage for toolResult-then-compaction limbo, delayed assistant recovery, and frozen completion capture.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: a child history containing a toolResult, then the synthetic compaction system marker, and no later assistant text leaves latestRawText selectable as the subagent output.

## Solution Assessment

No for the narrow unified-retry-only direction. The safer maintainable fix should cover the output summarizer’s compaction marker handling and the captureSubagentCompletionReplyUsing freeze path, not only recheck delayed assistant text in runSubagentAnnounceFlow.
