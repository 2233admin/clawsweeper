## Summary

Keep open. Current main still has a raw tool-output fallback after synthetic compaction markers, and the issue is already paired with open overlapping fix PRs, including one with closing syntax, so it should not be closed before a coordinated fix lands.

## What This Changes

The issue reports that subagent auto-compaction can make `readSubagentOutput` return stale raw tool-result JSON as the parent-facing subagent announcement or frozen completion result instead of waiting for the post-compaction assistant reply.

## Best Possible Solution

Keep this open until maintainers consolidate the open fixes into one path that treats synthetic compaction markers as continuation boundaries for both announce output selection and frozen completion capture, while preserving timeout partial-progress behavior and adding focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: a child history with a toolResult followed by the synthetic compaction system marker and no later assistant text leaves `latestRawText` selected as the subagent output.

## Solution Assessment

No for the suggested retry alone. A unified retry helps the announce branch, but the maintainable fix also needs to cover the compaction marker boundary and `captureSubagentCompletionReplyUsing` so frozen completion capture cannot lock in raw tool JSON.
