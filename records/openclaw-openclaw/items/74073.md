## Summary

Keep this open. Current main still has a concrete source-level path where a subagent history ending with raw tool output plus a synthetic compaction marker can select `latestRawText` as the completion, and the issue is already linked to open fix PRs that need maintainer review rather than premature closure.

## What This Changes

The issue reports that subagent auto-compaction can cause stale raw tool-result JSON to be announced to the parent instead of the delayed post-compaction assistant reply.

## Best Possible Solution

Land or combine a fix that treats compaction markers as an in-progress continuation before raw tool-result fallback is allowed, covering both final announce delivery and frozen-result capture, with regression tests for the compaction limbo window, post-compaction recovery, and timeout partial-progress behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a `chat.history` snapshot with an assistant tool call, a toolResult containing raw Tavily-style JSON, a synthetic compaction marker, and no later assistant reply; current `readSubagentOutput` can select the raw tool result in that state.

## Solution Assessment

No, the one-shot retry suggested by #74113 is probably not sufficient by itself because frozen-result capture also flows through `captureSubagentCompletionReply` and `readSubagentOutput`. The safer fix is to make compaction markers pause raw-output selection at the shared output-selection layer, then keep retry logic only as a secondary guard.
