## Summary

Keep open. Current main still has a source-level path where a subagent history ending in tool output plus a synthetic compaction marker can select the raw tool text as the completion result, and this issue is already paired with open implementation PRs, including an open closing PR, so it should remain open until one coordinated fix lands.

## What This Changes

The issue reports that subagent completion/announce output can race auto-compaction and forward stale raw tool JSON instead of waiting for the post-compaction assistant reply.

## Best Possible Solution

Land one consolidated fix that treats synthetic compaction markers as continuation boundaries and prevents successful subagent completion capture from using raw tool output while a post-compaction assistant reply may still arrive. The fix should cover both announce output selection and frozen completion capture, preserve timeout partial-progress behavior, and include regression coverage for the compaction limbo window and delayed assistant recovery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a child history with a raw `toolResult`, then a synthetic compaction system marker, and no later assistant text leaves `latestRawText` selected on current main.

## Solution Assessment

Unclear. The issue's suggested unified retry addresses part of the announce race, but the safer maintainable fix also needs marker-aware output selection and frozen completion capture coverage, as the linked PR discussion points out.
