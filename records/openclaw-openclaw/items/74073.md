## Summary

Keep this issue open. Current main still has the stale raw-output selection shape, and an open unmerged PR already uses closing syntax for this report, so the issue should remain open until a maintainer lands or rejects the linked fix path.

## What This Changes

The issue requests that subagent completion announcement capture wait for post-compaction assistant output instead of treating stale raw tool JSON as the child reply.

## Best Possible Solution

Land one focused agents fix, or a coordinated combination of the existing PRs, that treats auto-compaction as a continuation window for both announcement selection and completion-result freezing, prefers delayed assistant or silent-token output over raw tool text, and adds regression coverage for the compaction limbo window.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: a unit-level `chat.history` with a tool/toolResult payload, a synthetic compaction marker, and no later assistant reply still follows the current `latestRawText` fallback path; the live Grok/Tavily trigger is useful evidence but not required to demonstrate the selection bug.

## Solution Assessment

Is this the best way to solve the issue? Unclear: a one-shot retry helps part of the report, but the safer fix likely also needs compaction-marker continuation handling in output selection and completion capture so non-empty stale raw output cannot short-circuit the retry.
