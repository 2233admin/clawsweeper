## Summary

Keep open. Current main still has a plausible raw-tool-output capture path, and an open PR with closing syntax is already linked, so this issue should remain open until maintainers review and merge or reject the active fix path.

## What This Changes

The issue reports that subagent auto-compaction can let stale raw tool-result JSON be captured as the completion announcement before the delayed post-compaction assistant reply is available.

## Best Possible Solution

Land one coherent fix that treats compaction markers as a continuation window, suppresses raw tool-result fallback while a post-compaction assistant reply may still arrive, and adds focused regression coverage for both direct announce output and frozen completion capture.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available by mocking `chat.history` with a latest toolResult plus the synthetic compaction system marker and no later assistant reply; current `readSubagentOutput` can select the raw tool text in that state.

## Solution Assessment

Unclear. A one-shot retry in `runSubagentAnnounceFlow` may help, but the safer solution must also cover `captureSubagentCompletionReply` and the registry frozen-result path called out in the issue comments.
