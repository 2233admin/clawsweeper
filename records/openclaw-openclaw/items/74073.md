## Summary

Keep open. Current main still has the reported race shape: `readSubagentOutput` can return non-empty raw tool output before consulting the latest assistant reply, and `runSubagentAnnounceFlow` only enters the retry path when that first capture is empty. The linked #74113 is open and unmerged with closing syntax for this issue, so the next step is PR review rather than cleanup closure.

## What This Changes

The issue asks for a unified post-compaction retry in subagent announce output selection so delayed assistant replies replace stale raw tool JSON before completion announcements.

## Best Possible Solution

Land a bounded post-wait refresh or polling step that prefers the latest visible assistant or silent reply over stale tool output across both child-completion-findings and no-child-completion-findings announce paths, with focused regression tests for delayed post-compaction output.
