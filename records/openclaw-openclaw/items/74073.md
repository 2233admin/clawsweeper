## Summary

Keep open: current main still has a plausible raw tool-result capture path during subagent compaction, and this issue is already paired with open fix PRs that need maintainer consolidation rather than premature cleanup.

## What This Changes

The issue reports that subagent announce completion can capture raw tool-result JSON during auto-compaction before the delayed final assistant reply is available.

## Best Possible Solution

Land one maintainer-reviewed fix that treats compaction markers as a continuation/pause signal in subagent output selection, avoids raw toolResult fallback for successful completion capture, and retries or refreshes post-compaction assistant output across both direct announce and frozen-result paths, with focused regression tests for the limbo window, recovery, and timeout partial-progress behavior.

## Reproduction Assessment

Yes. The issue body gives a live provider/tool reproduction, and current main reduces to a deterministic unit shape: a raw toolResult followed by the synthetic compaction system marker leaves `latestRawText` selected before any delayed assistant reply is checked.

## Solution Assessment

Unclear as written. The suggested unified retry is directionally useful, but the later discussion identifies a second capture/frozen-result path, so the best fix should cover both output selection and capture refresh rather than only adding a retry in one announce branch.
