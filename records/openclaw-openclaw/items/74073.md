## Summary

Keep this issue open. Current main still has the reported subagent announce/output race shape: `readSubagentOutput` can select a latest raw tool/toolResult text when no assistant reply is visible yet, and `runSubagentAnnounceFlow` only polls again when the reply is empty. The related compaction issues #67750 and #70334 are open but cover different post-compaction failure modes, so they do not supersede this report.

## Best Possible Solution

Keep this issue open and fix the subagent announce/output layer: after the child wait/output read settles, do a bounded post-wait refresh that prefers a delayed latest assistant reply or silent token over raw tool output before constructing the completion event, covering both no-child-findings and child-findings paths where applicable. Add focused regression tests for raw tool JSON followed by delayed post-compaction assistant text, and preserve the current partial-progress and sessions_yield tests.
