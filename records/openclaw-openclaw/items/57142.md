## Summary

Keep open. Current main has adjacent chat-composer affordances, including slash-command completion, fixed welcome prompt buttons, input history, Talk mode, and a browser STT helper, but it does not implement configurable custom-term autocomplete/suggestions for the WebChat/Control UI chat input. The supplied related items are adjacent only and do not canonicalize this request.

## Best Possible Solution

Keep the issue open as a focused Control UI/WebChat feature request. A good implementation would add a small explicit custom-term source, wire a suggestion menu into `ui/src/ui/views/chat.ts` beside the existing slash menu and draft updates, validate/persist the source through config or a documented workspace/browser-local mechanism, and update WebChat/Control UI docs. If maintainers want this outside core, first define a deliberate plugin/UI composer extension API rather than closing this request as already externalizable.
