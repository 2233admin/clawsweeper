## Summary

Keep this PR open. Current main still does not implement the BlueBubbles reply-context API fallback: inbound replies only hydrate from webhook metadata and the process-local reply cache, with no shipped config flag, helper, docs, or tests for API fallback. The live PR is scoped and addresses prior SSRF/inheritance/dead-code review feedback, but it remains an implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose this PR or #73241 as the canonical BlueBubbles-owned reply-context fallback, keep the implementation behind a clear config/default policy, preserve account inheritance and SSRF-guarded fetch behavior, add public BlueBubbles docs for the new flag if this PR lands, and run the scoped BlueBubbles tests plus the changed gate before merge.
