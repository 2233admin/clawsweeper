## Summary

Keep this PR open. Current main at 7a23b2d945cc2d2e5c452e4641213e71de8dcc66 still does not implement the PR’s intended normal auto-reply behavior: own-line trailing `NO_REPLY` is stripped and the preceding text remains deliverable, and the regression test still asserts that old behavior. The PR is also paired with same-author open follow-up #46626 and overlaps open #69394, so maintainer coordination is still needed rather than auto-closing it.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The best path is a rebased shared silent-token detector, or equivalent trailing/own-line guard, in `src/auto-reply/tokens.ts` and `src/auto-reply/reply/normalize-reply.ts`, with tests flipping the current own-line trailing `NO_REPLY` expectation while preserving inline stripping, JSON-envelope suppression, glued-leading-token handling, case-insensitivity, media/channelData behavior, direct silent-reply policy, outbound delivery behavior, and cron-specific suppression. Coordinate with #46626 and #69394 before closing or landing duplicates.
