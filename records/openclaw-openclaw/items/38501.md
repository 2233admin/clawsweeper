## Summary

Current main still lacks the requested active-run Ctrl+C hint. The TUI Ctrl+C path only resolves clear/warn/exit status text and does not inspect activeChatRunId, while Esc and /abort remain the abort paths. This issue should stay open because #38502 is still an open closing PR for the narrow text fix, with #44223 open for a broader Ctrl+C abort-policy change.

## Best Possible Solution

Keep this issue open until a focused TUI fix lands. The narrow path is to review and land #38502 or an equivalent patch that updates src/tui/tui.ts Ctrl+C status text when activeChatRunId is set and adds a regression test in src/tui/tui.test.ts. If maintainers prefer Ctrl+C to abort active runs instead, reconcile that policy with #44223 and close or supersede the text-only PR accordingly.
