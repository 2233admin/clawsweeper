## Summary

Current main still has the reported TUI Ctrl+C UX gap: Ctrl+C only clears input, warns, or exits, while active-run cancellation remains on Esc and `/abort`. This issue should stay open because focused implementation work is already open in #38502, with a broader competing Ctrl+C policy proposal in #44223.

## What This Changes

Request active-run Ctrl+C status text in the TUI so users know the run is still active and can be stopped with Esc or `/abort`.

## Best Possible Solution

Keep this issue open until #38502 or an equivalent focused patch lands with regression coverage for active-run Ctrl+C status text. If maintainers prefer Ctrl+C to abort active runs instead, reconcile that direction through #44223, update docs/tests accordingly, and explicitly supersede the text-only path.
