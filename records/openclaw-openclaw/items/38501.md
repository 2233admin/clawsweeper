## Summary

Keep open: current main still has Ctrl+C clear/warn/exit handling without active-run-aware feedback, while Escape and `/abort` use the separate abort path. The issue is also paired with open focused PR #38502 and broader PR #44223, so maintainers should resolve the linked TUI policy work rather than close this issue now.

## What This Changes

The issue requests clearer TUI Ctrl+C status text when a chat run is active, so users know the run is still active and can stop it with Esc or `/abort`.

## Best Possible Solution

Choose the intended TUI Ctrl+C policy and land one matching patch with focused regression coverage. If Ctrl+C remains a clear/exit shortcut, a status-hint fix like #38502 should close this after merge; if Ctrl+C should abort active runs, reconcile #44223 with docs and tests.
