## Summary

Current checked-out main still has the reported TUI Ctrl+C UX gap: Ctrl+C only clears, warns, or exits without checking active runs, while Esc and `/abort` use the separate abort path. This should stay open because #38502 already proposes the focused closing fix and #44223 raises a broader Ctrl+C policy change for maintainer review.

## What This Changes

The issue asks for TUI Ctrl+C feedback during an active run to clearly say the run is still active and can be stopped with Esc or `/abort`.

## Best Possible Solution

Choose the intended Ctrl+C policy and land the matching TUI patch with focused regression coverage. If current keybinding semantics stay, #38502 or an equivalent status-text patch should close this after merge; if Ctrl+C should abort active runs, #44223’s broader behavior change should be reconciled with tests and docs.
