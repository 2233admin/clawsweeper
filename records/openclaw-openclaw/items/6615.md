## Summary

Keep this issue open. Current main still lacks first-class exec-approvals denylist support: the core policy enums, approvals file model, normalizer, runtime decision path, schema, CLI, Control UI, docs, and tests remain allowlist-oriented. Related implementation PRs were closed unmerged, and the linked SafeClaw/CRE plugin work is useful prior art but does not ship the requested OpenClaw core exec-approvals surface.

## Best Possible Solution

Keep this issue open until OpenClaw either implements first-class exec-approval denylist support across config/types/schema, approvals normalization and storage, gateway/node/system.run policy, CLI, Control UI, docs, and tests, or maintainers explicitly redirect the design to a ClawHub/plugin policy path and track any missing core authorization hooks separately.
