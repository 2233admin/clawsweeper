## Summary

Keep this open. Current main still lacks native exec-approvals denylist mode, a denylist collection, `ask: "on-match"`, and any deny-pattern evaluation path; the issue also has recent maintainer discussion explicitly keeping the native policy design open.

## What This Changes

Add native exec-approvals denylist support so users can broadly allow host exec while prompting or blocking specific command and argument patterns.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes configuration in CLI and Control UI, documents the security model, and coordinates overlap with #41140.
