## Summary

Keep this issue open. Current main still lacks first-class exec-approvals denylist support: policy enums, approvals storage, normalization, runtime decision logic, config schema, CLI, Control UI, docs, and tests remain centered on deny/allowlist/full plus allowlist entries. Related implementation PRs are closed unmerged, and the linked SafeClaw/CRE plugin work is useful prior art but does not ship the requested OpenClaw core approval surface.

## Best Possible Solution

Keep this issue open until OpenClaw either implements first-class exec-approval denylist support across config/types/schema, approvals normalization/storage, gateway and node system.run policy, CLI, Control UI, docs, and tests, or maintainers explicitly choose a ClawHub/plugin policy direction and track any missing pre-exec authorization hooks separately. Related #41140, #71097, and #48532 should be reviewed as part of the design so this does not fragment into incompatible permission models.

## Risks / Open Questions

- This is an exec authorization/security-policy change. Maintainers should decide precedence and semantics before implementation: denylist mode versus a denylist field, deny versus require approval, askFallback behavior, whether denylist applies before allowlist/full, and how argument or full-command regex matching is represented.
- The related open reports (#41140, #71097, #48532) overlap but are not identical. A fix should either deliberately cover the shared core policy gap or split the broader permission-learning/security-by-intent work into follow-ups.
- Prior implementation PRs were closed unmerged, and one attempted PR mixed a standalone Python package/root README churn with the requested OpenClaw change. A new implementation should stay inside the existing TypeScript exec-approvals, CLI, UI, docs, and tests surfaces.
- The CRE/SafeClaw plugin prior art may help some users, but redirecting this core approval policy to plugins would need explicit maintainer judgment and a check that current plugin hooks can enforce gateway/node system.run before execution.
