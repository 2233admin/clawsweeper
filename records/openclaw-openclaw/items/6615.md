## Summary

Keep this open: current main still lacks a native exec-approvals denylist mode, denylist collection, `ask: "on-match"`, or deny-match evaluation path. The SafeClaw and CRE links in the discussion are useful prior art, and the closed implementation PRs are relevant history, but they do not replace a merged OpenClaw policy change; a maintainer comment also explicitly kept this open for follow-up.

## What This Changes

The issue requests a native exec approvals denylist mode and denylist array with argument-aware matching and prompt-on-match behavior so users can allow most commands while prompting or blocking dangerous matches.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes configuration in CLI and Control UI, documents the security model at https://docs.openclaw.ai/tools/exec-approvals, and coordinates the overlap with #41140 rather than treating external plugins as the core fix.
