## Summary

Keep this issue open. Current main still has no first-class exec-approvals denylist mode: exec policy enums, approvals storage, normalization, runtime approval decisions, node system.run policy, config schema, Control UI, docs, and tests remain centered on deny, allowlist, full plus allowlist entries. The SafeClaw and CRE links are useful prior art, but they do not replace the requested OpenClaw core approval surface.

## Best Possible Solution

Keep this issue open until OpenClaw either implements first-class exec-approval denylist support across config/types/schema, approvals normalization/storage, runtime prompt decisions, gateway and node system.run policy, CLI, Control UI, docs, and tests, or maintainers explicitly choose a plugin/ClawHub policy direction and track any missing pre-exec authorization hooks separately. Review #41140, #71097, #48532, and the closed implementation attempts #43222, #60684, and #72466 together before accepting another implementation PR.
