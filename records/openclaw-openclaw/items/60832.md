## Summary

Keep this issue open. Current main still labels the Control UI composer plus action as “New session”, confirms it with mixed “start new session/reset current chat” wording, and routes the button through `/new` on the active session rather than the existing `sessions.create` API that creates a separate session entry.

## Best Possible Solution

Keep the issue open. The conservative fix belongs in the Control UI: rename the composer plus tooltip/aria/confirmation to reset-context wording and update tests/i18n as needed. If maintainers want a true “new session entry” action, add a separate UI control that calls `sessions.create` and keep reset-context as a distinct action.
