## Summary

Keep open. Current main still has a parent-fallback resolver for Discord channel config, but normal guild-message preflight only supplies parent metadata for thread channels, not ordinary text channels under a Discord category. The docs and tests still describe channel-level allowlists plus thread-parent inheritance, not category-level allowlist inheritance.

## Best Possible Solution

Keep this issue open and implement a documented Discord category allowlist path in the Discord plugin. The likely fix is to pass non-thread fetched channel `parentId`/parent metadata into the existing fallback resolver, decide the public config shape (`categories`, bare category IDs in `channels`, or `category:<id>`), preserve direct channel overrides over category defaults, and add schema/docs/preflight tests proving new child channels inherit without per-channel config edits or a gateway restart.
