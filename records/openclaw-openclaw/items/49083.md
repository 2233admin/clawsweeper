## Summary

Keep this PR open. Current main still uses ordinary string inequality for both Gateway TLS fingerprint comparisons that the PR targets, while the PR is a narrow security hardening change against a documented Gateway TLS pinning path.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, replace both Gateway TLS fingerprint comparisons in `src/gateway/client.ts` with `safeEqualSecret(fingerprint, expected)`, and add a focused GatewayClient regression test or narrow spy proving both TLS fingerprint paths use the constant-time helper.
