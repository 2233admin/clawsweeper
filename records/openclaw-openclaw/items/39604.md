## Summary

Keep this issue open. Current main still has no web_fetch private-network opt-in: the strict runtime schema, exported config type, runtime cache/policy forwarding, tests, docs, and v2026.4.26 changelog only show the narrower RFC2544 fake-IP proxy exception. The issue is also paired with open closing PR #39630, so it should stay open until maintainers land, replace, or close that implementation candidate.

## Best Possible Solution

Keep this issue open and have maintainers review, update, land, replace, or close PR #39630. The best code path is a default-off web_fetch private-network policy surface aligned with the existing tools.web.fetch.ssrfPolicy shape, with strict schema/types/help/docs updates, runtime forwarding, policy-aware cache keys, and SSRF regression tests for private literals, localhost, DNS-to-private, redirects, provider fallback boundaries, and default-block behavior.
