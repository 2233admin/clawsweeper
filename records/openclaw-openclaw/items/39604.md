## Summary

Keep this issue open. Current main still does not expose a `web_fetch` private-network opt-in, and #39630 is an open same-author implementation candidate that declares it closes this issue. The shipped current behavior only adds the narrower RFC2544 fake-IP proxy allowance, which does not cover localhost or private LAN access requested here.

## Best Possible Solution

Keep this issue open while maintainers review #39630 or a replacement design. The likely core path is a default-off `web_fetch` SSRF policy option, preferably under `tools.web.fetch.ssrfPolicy` with explicit dangerous naming and/or narrower hostname controls, wired through the strict schema, exported config types, generated labels/help, runtime policy forwarding, cache isolation, public docs, and focused SSRF regression tests.
