## Summary

Keep this PR open. The target checkout still has the reported CSS-only fallback bug on current main, and the submitted PR head is based on the older single `marked` renderer path. It remains useful, but it needs a current-main refresh that covers both fenced and indented code-block renderers and resolves the sanitizer-scope review concern.

## Best Possible Solution

Keep the PR open and ask for a current-main refresh: set the default hidden state for generated `.code-block-copy__done` labels in both fenced and indented renderers, toggle `HTMLElement.hidden` symmetrically in `renderChat`, and add regression tests for both code-block forms. Prefer avoiding a global DOMPurify `hidden` allowlist unless maintainers explicitly accept that sanitizer-scope change.
