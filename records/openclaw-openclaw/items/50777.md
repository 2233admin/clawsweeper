## Summary

Keep this PR open. Current main still has the CSS-only copy-button label behavior the PR is trying to fix, and the proposed patch is not already implemented on main or in the latest release. The PR remains plausible, but it needs current-main refinement because there are now separate fenced and indented code-block renderers, plus a maintainer decision on whether to avoid a global DOMPurify `hidden` allowlist.

## Best Possible Solution

Keep the PR open and ask for a current-main patch that applies the hidden fallback to both fenced and indented code-block copy buttons, toggles `HTMLElement.hidden` alongside the copied state in `renderChat`, and adds regression coverage for the default label visibility. Prefer avoiding a global DOMPurify `hidden` allowlist unless maintainers explicitly accept that sanitizer behavior.
