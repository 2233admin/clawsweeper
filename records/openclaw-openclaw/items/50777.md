## Summary

Keep this PR open. Current main still relies on CSS alone to hide the code-block `Copied!` span, and the copy handler only toggles the `copied` class. The PR’s intended behavior is not implemented on main or in the latest release. The PR likely needs a rebase/refinement because current main has separate fenced and indented code-block renderers, and the sanitizer allowlist change deserves a targeted security review.

## Best Possible Solution

Keep the PR open and rebase or replace it with a current-main patch that applies the non-CSS hidden fallback to both fenced and indented code-block copy buttons, updates `renderChat` to toggle `HTMLElement.hidden` with the copied state, and adds regression tests for the default hidden state. Prefer a targeted approach that avoids globally allowlisting `hidden`, or document why the allowlist is acceptable given the raw-HTML escaping path.

- The PR patch shown in the provided context predates current main’s two code-block renderers; if applied without rebasing, it could fix fenced code blocks while leaving indented code blocks with the same CSS-only label behavior.
