## Summary

Current main still contains the duplicate Usage header composition that this PR targets: the Usage view renders its own title/subtitle block while the app shell also renders the shared tab header/subtitle. The PR is not obsolete and remains a plausible implementation candidate for the open Usage-header regression tracked by #45709, with remaining review work around i18n regeneration, mobile behavior, and regression coverage.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for #45709. The merge path should remove the duplicate local Usage page header, preserve the Filters card label, regenerate i18n outputs from the English source including Thai, make an explicit maintainer decision on mobile page-heading behavior, and add or intentionally waive a focused UI render regression test.
