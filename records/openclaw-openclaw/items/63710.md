## Summary

Keep open. Current main still excludes explicit HTTP(S) prompt image URLs from native image detection, so the mixed inline-image plus remote-image behavior this PR targets is not implemented. The PR is paired with the same-author open bug #62514 and has unresolved security/behavior review feedback around broad outbound fetching and SSRF-policy threading, so it should remain in review rather than be closed as cleanup.

## What This Changes

The PR changes native prompt image detection/loading to treat HTTP(S) image URLs as prompt image refs, preload them with `loadWebMedia`, merge them with inline/offloaded images for vision-capable models, and adds regression tests in `src/agents/pi-embedded-runner/run/images.test.ts`.

## Best Possible Solution

Keep this PR and #62514 open for maintainer review or contributor rework. A mergeable fix should preserve mixed inline plus remote image delivery to the first native vision request while constraining URL detection, threading the existing SSRF policy, handling punctuation/dedupe edge cases, and covering mixed-input, non-image URL no-fetch, and SSRF-policy propagation cases.
