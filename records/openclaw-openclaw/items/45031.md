## Summary

Keep open. Current main has partial hardening, but not the requested default protection for ClawHub skill installation/update/loading: gateway-backed skill dependency installs run the built-in scanner, while the native ClawHub skill install/update path still downloads and copies skill folders into the workspace without invoking that scan or the install policy hook. The AgentShield and ClawGuard comments are useful external implementations, but they do not satisfy the requested built-in/default enforcement.

## What This Changes

Request to add built-in security scanning around skill install, update/load, and ClawHub publish flows, with AgentShield suggested as one possible scanner.

## Best Possible Solution

Define a first-class skill supply-chain policy for ClawHub install/update and possibly load/publish: surface registry scan state in native OpenClaw flows, run or consume a trusted scanner before workspace install/update, block critical findings by default with an explicit recovery path, and preserve the existing plugin hook surface for optional scanners such as AgentShield or ClawGuard.
