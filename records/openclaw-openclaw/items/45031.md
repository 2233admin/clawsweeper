## Summary

Keep open. Current main has registry-side ClawHub scan summaries and a built-in scanner for gateway-backed skill dependency installs, but native ClawHub skill install and update still use a download/extract/copy path without a local scan or policy gate. The remaining work is security-sensitive product design, so no conservative close reason applies.

## What This Changes

The issue requests built-in security scanning for skill installation, including ClawHub install, skill loading, and ClawHub publish paths, with discussion expanding the scope to update scans, provenance, and per-agent policy.

## Best Possible Solution

Define a first-class OpenClaw skill supply-chain policy for ClawHub install and update: local scan execution, registry scan-state handling, critical-finding blocking, operator overrides, provenance/version evidence, UI/CLI/Gateway result surfaces, and docs/threat-model alignment. External scanners can remain optional integrations unless maintainers choose one as the default backend.
