## Summary

Keep open. Current main does not implement the proposed short-command respawn allowlist, and the PR still has a blocking startup-environment regression: the widened skip predicate also bypasses the planner path that injects `NODE_EXTRA_CA_CERTS` before Node startup. The existing ClawSweeper discussion identified the same concern, and current source, tests, and docs still support keeping this for maintainer-guided rework rather than cleanup closure.

## What This Changes

The PR changes CLI respawn policy so only `gateway`, `daemon`, `agent`, and `memory` commands reexec, adds a dedicated respawn-policy test file, and removes the old respawn-policy assertions from `cli-utils.test.ts`.

## Best Possible Solution

Split the respawn planner so command policy can skip ExperimentalWarning-only suppression for verified short-lived commands while mandatory startup-environment reexec still runs when needed. Preserve or explicitly revise the foreground gateway run policy, then cover both command policy and startup TLS behavior in tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve startup TLS respawn when skipping warnings:** `src/cli/respawn-policy.ts:41`
  - body: `buildCliRespawnPlan` exits as soon as `shouldSkipRespawnForArgv` returns true. With this allowlist, commands like `status`, `config`, or `models` will also skip the Linux/nvm `NODE_EXTRA_CA_CERTS` reexec, even though the docs say the CLI sets that before Node startup. Split the policy so only ExperimentalWarning suppression is command-gated while required startup environment respawn still runs.
  - confidence: 0.91
