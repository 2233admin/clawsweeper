## Summary

Keep open. Current main does not implement this PR's intended change: `status` and other short-lived commands still keep the respawn path, and the current respawn path now also initializes `NODE_EXTRA_CA_CERTS`, so the PR needs a rebase/review against the newer startup contract rather than cleanup closure.

## Best Possible Solution

Keep this draft PR open, but rebase/rework it against current main by separating experimental-warning suppression policy from mandatory startup environment re-exec. A good fix would preserve `NODE_EXTRA_CA_CERTS` readiness while avoiding unnecessary warning-only respawns for safe short-lived commands, with tests covering both policies.
