## Summary

Keep open. Current main repairs tool_use/tool_result pairing in memory before provider replay and has a partial on-disk session-file repair before SessionManager.open, but that on-disk repair only drops malformed JSONL lines and rewrites empty assistant error messages. It does not run repairToolUseResultPairing or rewrite orphaned/displaced toolResult entries, which is the durable restart self-healing requested in #58608.

## Best Possible Solution

Keep #58608 open for a focused core fix: extend the pre-SessionManager.open repair path, or add a new locked transcript load-repair pass, to parse the active JSONL branch, apply repairToolUseResultPairing to message entries, and rewrite the session file with a backup when orphaned, duplicated, displaced, or missing tool results are found. Add regression coverage in session-file-repair tests for orphaned toolResult persistence, displaced toolResult reordering, missing synthetic result insertion policy, and no-op behavior for clean transcripts; consider a doctor repair path for existing corrupted files.
