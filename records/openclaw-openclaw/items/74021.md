## Summary

Keep this issue open. Current main has partial runtime support for OpenAI-compatible reasoning deltas, non-visible/reasoning-only retry recovery, and vLLM Nemotron visible-content enforcement when thinking is off, but it still lacks the requested oc llm benchmark/reporting schema, extraction-source counters, separate scoring modes, and model/profile policy knobs. The reporter also clarified that this broader product/runtime/reporting work should remain separate from the narrower scorer issue #74017.

## Best Possible Solution

Keep this open for maintainer design review. The safest path is to split it into a measurement/reporting change that records content/reasoning presence, extraction source, and separate visible-content versus reasoning-aware scores, followed by a runtime/provider-profile contract for opt-in reasoning-field and visible-content policy. Keep #74017 as the separate narrower scorer-bug history.
