## Summary

Keep this issue open. Current main has useful partial runtime support for OpenAI-compatible reasoning deltas, non-visible/reasoning-only retry recovery, and vLLM/Nemotron visible-content enforcement when thinking is off, but it still lacks the requested benchmark/reporting schema, separate visible-content versus reasoning-aware scores, extraction-source counters, and first-class model/profile policy knobs. The reporter also clarified that this broader product/runtime/reporting work should stay separate from #74017.

## Best Possible Solution

Keep this open for maintainer design review. The safest path is to split the work into a reporting/measurement layer that records content/reasoning presence, extraction source, compatibility counters, and separate visible-content versus reasoning-aware scores, followed by a runtime/provider-profile contract for opt-in reasoning-field use and visible-content policy. Keep #74017 as the separate scorer-bug history.
