## Summary

Keep open. Current main has useful partial runtime/provider handling for reasoning-only and vLLM/Nemotron visibility cases, but it still lacks the requested first-class benchmark/reporting schema, extraction-source counters, separate reasoning-aware versus visible-content scores, and explicit profile policy knobs; the issue discussion also confirms this broader work should remain separate from the closed scorer bug.

## What This Changes

Request to add opt-in reasoning-field ingestion, visible final-answer enforcement, extraction-source reporting, separate scoring modes, and model/profile policy for native reasoning model outputs.

## Best Possible Solution

Keep the issue open for maintainer design. The best end state is a reporting-first slice that records content/reasoning presence, extraction source, compatibility counters, and separate visible-content versus reasoning-aware scores with redacted fixtures, followed by an explicit opt-in provider/profile contract for reasoning-field use and visible-content requirements.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature/reporting request. We do have a high-confidence current-main verification path through focused source/docs searches; the exact benchmark deltas require the reporter's local artifacts or redacted fixtures.

## Solution Assessment

Yes, keeping this open for design is the best current handling. The existing runtime retries and vLLM request shaping are useful partial mitigations, but a complete fix needs explicit reporting semantics and privacy policy before broader runtime/profile behavior is added.
