## Summary

Keep this PR open. Current main still generates zsh leaf-command completions from option specs only, so Commander positional arguments like `openclaw wiki ingest <path>` are not represented. The supplied PR context shows this branch is the active canonical implementation candidate after duplicate attempts were closed and a maintainer pushed a narrow repair to preserve the contributor path.

## Best Possible Solution

Keep this PR open for normal maintainer review and landing. The best path is to merge a focused zsh completion fix that reads Commander registered arguments for leaf commands, preserves subcommand dispatch, avoids generic config-path file suggestions, and keeps the new regression coverage in `src/cli/completion-cli.test.ts`.
