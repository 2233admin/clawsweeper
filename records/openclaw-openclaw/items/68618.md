## Summary

Current main still lacks a native Feishu sender-name alias map. The Feishu config schema exposes `resolveSenderNames` but no map option, the runtime path still resolves names only through the Feishu contact API/cache, and the docs do not document a mapping surface. This is a valid bundled Feishu plugin request, but an adjacent open PR is already changing the same display-name resolver path, so it should stay open for maintainer sequencing rather than be auto-closed.

## What This Changes

Add a Feishu channel configuration map from sender IDs to custom display names for fallback or override behavior when sender-name resolution is unavailable or undesirable.

## Best Possible Solution

Implement this in the Feishu plugin as an owner-controlled attribution feature, likely with top-level and account-level config support, tests for API success/failure and `resolveSenderNames` interactions, docs at the public Feishu channel page, and no effect on ID-based allowlist decisions.
