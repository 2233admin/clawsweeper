## Summary

Keep this issue open. Current main has background task records, `/subagents` inspection/control commands, task list/show/cancel/notify, terminal subagent hooks, and push-style task notifications, but it still does not provide the requested durable per-subagent lifecycle event ledger, persisted tool/warning/error timeline, artifact-pointer completion payloads, first-failure/recommended-action diagnostics, or idempotent audit-logged cancel/steer/resume supervision surface. The related closed #62478 background-spawn work and closed-unmerged #24925 subagent hook PR are complementary, not replacements for this broader observability/control scope.

## What This Changes

Request deterministic subagent lifecycle observability and async supervision controls, including timestamped lifecycle/tool/error events, persisted summaries with artifact pointers, first-failure diagnostics, and audit-logged cancel/steer/resume actions.

## Best Possible Solution

Keep this open for maintainer design and likely split it into narrower implementation issues: add a durable per-run event ledger for subagent/task spawn, queued/running, tool start/end, warning/error, completion, and supervision actions; persist artifact pointers and terminal first-failure/recommended-action diagnostics; expose the data through `/subagents info/log`, `openclaw tasks list/show --json`, gateway/control UI surfaces, and docs; and make cancel, steer, and resume deterministic, idempotent, and audit-logged. Treat #62478 as shipped complementary background-spawn work and #24925/current terminal hooks as useful narrower context, not the canonical replacement.
