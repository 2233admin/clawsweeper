## Summary

Current main still generates the Windows Startup-folder login item as a `.cmd` launcher containing `start "" /min cmd.exe /d /c <gateway.cmd>`. There is no `.vbs`/`wscript` sidecar generation or cleanup, and the current startup-fallback test still asserts the minimized-cmd launcher shape, so the reported login-time flash remains plausible and actionable.

## Best Possible Solution

Keep this issue open and fix the core Windows daemon Startup-folder fallback: keep the existing `OpenClaw Gateway.cmd` path/name for detection, launch the gateway through a truly hidden sidecar or equivalent no-window mechanism, remove any sidecar on uninstall, update startup-fallback tests, and add a user-facing changelog fix entry.
