## Summary

Keep open. Current main still generates the Windows Startup-folder fallback as a `.cmd` launcher containing `start "" /min cmd.exe /d /c ...`; there is no `wscript.exe`/`.vbs` companion launcher, and the existing Startup fallback test explicitly asserts the old minimized-cmd behavior. The related closed #57682 addresses the separate `openclaw update` restart helper path, not this login-item fallback.

## Best Possible Solution

Keep #70788 open. The likely fix belongs in `src/daemon/schtasks.ts`: replace the Startup-folder fallback's minimized `cmd.exe` launcher with a tested no-window Windows launcher, keep the existing `.cmd` path/name for detection, write and remove any companion launcher during install/uninstall, and update `src/daemon/schtasks.startup-fallback.test.ts` plus Windows gateway docs if user-visible behavior changes.
