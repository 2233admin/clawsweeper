## Summary

Keep open. Current main still does not implement this PR’s intended silent ACPX exit classification: active-turn plain `Error("acpx exited with code 1")` is wrapped as `ACP_TURN_FAILED`, and the current regression test asserts that behavior. The PR is also stale against the current ACPX wrapper, which now delegates turn/session behavior to the published `acpx/runtime` package rather than the older in-repo process-spawn paths touched by the PR.

## Best Possible Solution

Keep this PR open as a focused implementation candidate, but ask for a current-design rework. The best path is to implement structured silent-exit classification at the `acpx/runtime` boundary and consume a released ACPX version here, or add a narrow OpenClaw adapter that maps only genuinely silent prompt exits to `ACP_BACKEND_UNAVAILABLE` while preserving permission-denied, session-init, non-silent stderr, control-command, and fresh-handle recovery behavior. Regression tests should be updated for the current delegate shape and should cover manager fallback, runtime event classification, user-facing error text, and the protected session-init/control paths.
