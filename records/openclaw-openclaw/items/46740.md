## Summary

Keep open. The requested behavior is not implemented on current main, but this PR is not merge-ready: main now delegates ACPX turns to the published `acpx/runtime` package, current manager tests still assert `acpx exited with code 1` as `ACP_TURN_FAILED`, and the PR branch is dirty against main while introducing P2 misclassification regressions.

## What This Changes

The PR adds ACPX/ACP error-mapping helpers and tests so silent non-zero `acpx` exits surface as `ACP_BACKEND_UNAVAILABLE` with `/acp doctor` guidance while preserving permission-denied and session-init handling.

## Best Possible Solution

Rework this at the current ACPX boundary so only genuinely silent prompt/backend exits become `ACP_BACKEND_UNAVAILABLE`, while preserving permission-denied, session-init, non-silent stderr, control-command, and fresh-handle recovery behavior. The durable path is likely an `acpx/runtime` fix plus an OpenClaw dependency bump, with OpenClaw adapter code only if maintainers choose to keep the mapping local.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard prompt exit remapping behind empty stderr:** `extensions/acpx/src/runtime.ts:679`
  - body: This sets `ACP_BACKEND_UNAVAILABLE` for any non-zero prompt exit without a structured error event. The same path still includes stderr in `formatAcpxExitMessage`, so normal prompt or validation failures that write stderr would be routed to `/acp doctor` instead of turn-failure recovery. Only truly silent exits should be upgraded.
  - confidence: 0.86
- **[P2] Preserve control-command failures with stderr:** `extensions/acpx/src/runtime.ts:1041-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` upgrades every non-permission-denied control exit whose fallback is `ACP_TURN_FAILED`, without checking whether the command wrote stderr or whether the failure came from status, cancel, close, mode, or config. Those user/control failures can be mislabeled as backend outages.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery for silent init exits:** `extensions/acpx/src/runtime.ts:483-484`
  - body: The early rethrow bypasses `recoverEnsureFailure` for a silent `sessions ensure` exit with no events, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
