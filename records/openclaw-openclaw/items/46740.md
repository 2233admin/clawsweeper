## Summary

Keep open. Current main still does not implement this PR’s intended silent ACPX exit classification, and the PR is stale against the current ACPX wrapper shape. The OpenClaw wrapper now delegates turn execution to the published `acpx/runtime` package, manager fallback still uses `ACP_TURN_FAILED` for active-turn plain errors, and current tests assert `Error("acpx exited with code 1")` remains `ACP_TURN_FAILED`. The provided PR discussion also shows the author addressed the first session-init concern, but later review comments still flag control/stderr/recovery regressions that need a current-design rework.

## Best Possible Solution

Keep this PR open, but ask for a current-design rework. The best path is to implement silent prompt-exit classification in `openclaw/acpx` and consume a released `acpx` version here, or add a narrow OpenClaw adapter around the delegate that maps only genuinely silent prompt/backend exits to `ACP_BACKEND_UNAVAILABLE` while preserving permission-denied, session-init, non-silent stderr, control-command, and fresh-handle recovery behavior. Update regression coverage for the current delegate shape, manager fallback, user-facing error text, and protected session-init/control paths.
