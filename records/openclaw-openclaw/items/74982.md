## Summary

Keep open. Current main still drops the Telegram `audioAsVoice` delivery hint before it reaches the low-level sender, while this PR wires that existing hint to Telegram's existing `asVoice` option in the relevant plugin-owned paths; the remaining action is maintainer review and targeted validation, not cleanup closure.

## What This Changes

The PR maps Telegram outbound `audioAsVoice` contexts to `asVoice` in the legacy `sendMedia`/`sendPayload` paths and the registered plugin `sendPayload` path, with legacy adapter regression tests.

## Best Possible Solution

Review and land this plugin-boundary fix after targeted validation so the existing voice-note hint reaches Telegram's existing sender option without adding Telegram-specific logic to shared core delivery.

## Reproduction Assessment

Yes. A voice-marked payload reaches shared outbound delivery with `audioAsVoice`, but current main drops that hint in both Telegram outbound adapters before `send.ts`, where `sendVoice` depends on `asVoice`.

## Solution Assessment

Yes. The patch is the narrowest maintainable approach: preserve an existing outbound context flag inside the owning Telegram plugin paths and pass it to the existing Telegram sender option; adding a registered-path regression test would strengthen proof but is not a different solution.
