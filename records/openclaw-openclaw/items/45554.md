## Summary

Keep open. Current main routes new direct DMs with `session.dmScope: "main"` to the canonical main key, but it still does not reconcile, mark inactive, warn about, or safely clean up existing peer-keyed direct-session rows such as `agent:main:telegram:direct:*`. The stale label should not drive closure; the issue comments remain accurate and the related items are adjacent rather than canonical fixes.

## Best Possible Solution

Keep this open and add a dmScope-aware session health/cleanup path. The safest direction is a dry-run-first `openclaw sessions cleanup` or doctor-style audit that detects peer-keyed direct rows when current config routes that conversation class to main, then archives/marks/retires those rows without silently deleting history. If maintainers want automatic startup reconciliation, it should use the same tested helper and preserve transcripts.
