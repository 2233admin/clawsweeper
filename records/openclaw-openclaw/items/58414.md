## Summary

Keep open. Current main has addressed several reported compounding factors, including `gateway run` routing, supervised lock retry/stale-lock handling, and package-update restart health checks, but the concrete LaunchAgent request from this issue is still not implemented: generated macOS plists do not include `StandardInPath=/dev/null`. The related launchd hang tracker #46153 is also still open, so this recent core macOS gateway report should be narrowed or consolidated by a maintainer rather than auto-closed.

## Best Possible Solution

Keep this open and narrow it. The best maintainer path is to either add `StandardInPath=/dev/null` in `buildLaunchAgentPlist` with launchd plist unit coverage, or explicitly consolidate this issue into #46153 after preserving the self-update trigger, routing regression history, and workaround details. Do not close as implemented while the requested plist key is absent.
