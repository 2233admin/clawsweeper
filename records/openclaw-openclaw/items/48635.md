## Summary

Keep open. Current main still lacks both central changes: the per-response usage formatter does not include model or thinking metadata, and responseUsage remains session-scoped with no agents.defaults.responseUsageDefault config surface. The PR is useful work toward open usage-footer requests, but it needs a rebase/fix pass before it is mergeable.

## Best Possible Solution

Keep this PR open as a useful implementation candidate. The best path is to rebase on current main, drop the root log artifact, add responseUsageDefault to the typed config and schema, wire it so explicit session /usage off still overrides the default, decide and test whether model/thinking appear only in full mode, use the resolved run thinking level, and update the usage docs/tests. That would directly address #12578 and part of the model-footer requests without closing the broader context-usage work in #40215 prematurely.

## Risks / Open Questions

- The PR head includes an unrelated root log artifact, openclaw-2026-03-17.log, with local runtime paths; that should be removed before merge.
- As drafted, falling back to responseUsageDefault with nullish coalescing conflicts with current /usage off behavior, because the command and gateway patch paths delete responseUsage for off; a non-off default would re-enable the footer instead of preserving a session-level off override.
- The review comments identify real correctness gaps: model/thinking metadata should be gated to the intended footer mode or the contract should be changed, and thinking should come from the resolved run state rather than only activeSessionEntry.
- The PR needs tests and docs for the new config default, session override precedence, and model/thinking footer output before maintainers can safely land it.
