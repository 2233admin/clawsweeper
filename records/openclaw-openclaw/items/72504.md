## Summary

Keep open. Current main still strips every Feishu self-mention before the agent-facing prompt, so the reported multi-bot prompt-loss path remains live. Open PR #72661 explicitly closes this issue and open PR #72532 is an alternate fix, so the conservative maintainer-cleanup result is to leave this issue open until one of those PRs is reviewed and merged or rejected.

## What This Changes

Feishu should preserve the receiving bot's model-visible self-mention context in multi-bot group messages so explicitly mentioned bots do not return NO_REPLY.

## Best Possible Solution

Land one focused Feishu fix that strips only the leading addressing prefix needed for slash-command routing while preserving inline or repeated self-mentions for the agent prompt, with regression coverage for multi-bot group messages; close this issue after the PR merges.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu group text event with mentions for Bot A and Bot B while botOpenId is Bot A; current normalizeMentions removes Bot A's placeholder before buildFeishuAgentBody constructs the model-facing prompt.

## Solution Assessment

No for the issue's broad suggested removal of all botStripId behavior, because current code and changelog history show that self-strip also protects #35994 slash-command routing. The narrower leading-prefix-only approach in open PR #72661 appears to be the best maintainable fix path.
