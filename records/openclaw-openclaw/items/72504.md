## Summary

Keep this open. Current main still strips every matching Feishu bot self-mention from normalized inbound content, and an open PR with closing syntax (#72661) is already tracking the narrower fix; per policy this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks Feishu group messages that mention multiple bots to preserve each receiving bot's own `<at>` tag so the model does not mis-route the turn as `NO_REPLY`.

## Best Possible Solution

Land or adapt the #72661-style Feishu-owned fix: strip only a leading self-addressing mention, preserve inline or mid-sentence self `<at>` tags, and keep regression coverage for both multi-bot groups and slash commands. Close this issue after that implementation merges.

## Reproduction Assessment

Yes. A unit-level reproduction is enough: parse a Feishu group text like `Hey @_bot_a @_bot_b please coordinate` with Bot A's open id, and current main deterministically removes Bot A's own placeholder before the agent body is built.

## Solution Assessment

Yes, the narrow leading-prefix strip is the best direction. Removing `botStripId` entirely would likely regress the documented #35994 slash-command behavior, while the open PR's approach keeps that behavior and restores the model-visible self mention for multi-bot routing.
