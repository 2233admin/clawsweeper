## Summary

Keep open. Current main still passes the receiving Feishu bot open id into `normalizeMentions()`, and that helper still replaces every matching self-mention placeholder with an empty string before the agent-facing body is built. Open PR #72661 explicitly closes this issue, so the conservative cleanup outcome is to leave this issue open until that PR is reviewed and merged or rejected.

## What This Changes

Feishu group chats with multiple @mentioned bots should preserve each receiving bot's own mention in the agent prompt so mention-gated agents do not silently return `NO_REPLY`.

## Best Possible Solution

Review and either land or reject the focused PR that strips only the leading Feishu bot addressing prefix while preserving inline self-mentions as `<at>` tags, with regression coverage for multi-bot and repeated self-mention messages.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: parse a Feishu group text event whose `mentions[]` contains the receiving bot plus another bot, and current `normalizeMentions()` removes every placeholder for the receiving bot before `buildFeishuAgentBody()` builds the prompt.

## Solution Assessment

No for the issue's suggested full removal of `botStripId`; the narrower leading-prefix approach in #72661 is safer because it preserves the #35994 slash-command behavior while keeping inline self-mentions visible to the model.
