## Summary

Keep this issue open. Current main still strips the receiving Feishu bot's own mention before building the agent-facing body, and an open implementation PR with closing syntax already tracks the focused fix, so the right cleanup path is maintainer review of that PR rather than closing early.

## What This Changes

The issue asks Feishu group message handling to preserve the receiving bot's own `<at>` mention in multi-bot prompts so agents do not return `NO_REPLY` after being explicitly mentioned.

## Best Possible Solution

Keep this issue as the bug tracker until a Feishu-owned fix lands that preserves #35994 leading-address slash-command behavior while keeping inline or mid-sentence self-mentions visible to the agent, with regression tests for multi-bot group prompts.
