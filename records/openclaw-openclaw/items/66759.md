## Summary

Keep open. Current main still builds Feishu card notes from fixed Agent/Model/Provider fields, routes both streaming and non-streaming cards through that fixed note, has no Feishu `cardFooter`/`showContext` config, and has no `[[card_footer: ...]]` directive. Existing `/status` and `/usage` surfaces expose context or usage data, but `/usage` appends text to the reply body rather than populating the Feishu card footer.

## Best Possible Solution

Keep this open and implement it in the bundled Feishu plugin. A conservative path is a typed `channels.feishu.cardFooter` option for built-in fields such as context usage and compactions, composed in `extensions/feishu/src/reply-dispatcher.ts` for both streaming and non-streaming card paths, with tests and Feishu docs. If maintainers want an assistant directive too, add it through the shared directive parser with strict sanitization.
