## Summary

Keep open. Current main still builds Feishu card footers from fixed Agent/Model/Provider fields, has no Feishu cardFooter/showContext config, and does not parse a card_footer directive or route context/compaction stats into the card note.

## Best Possible Solution

Implement this in the bundled Feishu plugin with a typed, documented channels.feishu.cardFooter option for built-in fields such as context usage and compactions, compose it through resolveCardNote for streaming and non-streaming card paths, and add focused schema/dispatcher/docs tests. Defer any assistant-supplied footer directive unless maintainers explicitly want that extra model-output surface.

## Clownfish Work Prompt

Implement Feishu message-card footer customization for the bundled Feishu plugin. The observable gap is that Feishu card notes currently show only Agent/Model/Provider and users cannot add context-health fields such as context usage or compaction count.

Expected boundary:
- Prefer a typed config-driven solution, e.g. `channels.feishu.cardFooter`, with built-in fields such as `contextUsage` and `compactions` plus existing Agent/Model/Provider behavior preserved by default.
- Compose the footer in `extensions/feishu/src/reply-dispatcher.ts` for both streaming cards and non-streaming structured cards.
- If session metadata is needed, pass the minimum generic data into the Feishu dispatcher from `extensions/feishu/src/bot.ts`; avoid broad core behavior changes unless a small generic reply metadata seam is unavoidable.
- Do not implement an assistant/model-supplied `[[card_footer: ...]]` directive in this PR unless maintainers explicitly request it; if touched, it must have strict sanitization and length limits.
- Keep core extension-agnostic and avoid hard-coded Feishu behavior outside the plugin.
- Update Feishu docs and focused tests. Add a changelog entry only if maintainers treat the new config as user-facing release-note material.

Related ref: https://github.com/openclaw/openclaw/issues/66759
