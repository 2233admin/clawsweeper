## Summary

Keep open. Current main still lacks a migration that strips obsolete flat `memorySearch` keys before strict config validation, so the PR is not implemented or superseded. The branch is on the right surface for #68664 and its follow-up commit addresses the per-agent gap raised in review, but it still needs maintainer review because one added test preserves `memorySearch.limit`, which current `MemorySearchSchema` does not accept.

## What This Changes

The branch adds a shared legacy config migration and regression tests to remove obsolete flat `memorySearch` keys from `agents.defaults.memorySearch` and `agents.list[].memorySearch` before strict config validation.

## Best Possible Solution

Land a revised version that intentionally strips or maps the obsolete flat `chunkSize`, `chunkOverlap`, and `maxResults` keys for both defaults and per-agent memorySearch entries in the shared runtime/doctor migration path, uses only valid modern memorySearch fields in tests, and includes startup/read-path coverage before closing #68664.
