## Summary

Keep PR #72033 open. Current main still does not implement `diagnostics.pricing`: only `diagnostics.stability` is registered, scoped, advertised, tested, and documented. The existing pricing-cache metadata helper and bootstrap timeout warning path are present, so the PR remains a focused Gateway observability change rather than obsolete cleanup. The PR discussion also supports continued review: bot reviewers found the approach useful, and the author addressed the concrete review comments in follow-up commits.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers want this operator diagnostic surface, the best path is the PR's small core Gateway change: add `diagnostics.pricing` to the diagnostics handler, READ_SCOPE table, advertised method list, focused cache-state tests, and public Gateway protocol/operator docs if the RPC is intended to be supported.
