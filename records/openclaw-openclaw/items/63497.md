## Summary

Keep this PR open. Current main has shared media-reference helpers and direct managed-inbound tests, but the production image-tool path still rewrites a preserved `media://inbound/<id>` URI through `workspaceDir` before `loadWebMedia` can hydrate it, so the linked user-visible bug remains. The branch is useful source work, but should be rebased onto the newer shared helpers rather than merged as older duplicate parsing.

## What This Changes

The PR teaches the image tool to accept bare and `MEDIA:`-prefixed `media://inbound/<id>` references, adds image-tool regression tests, and adds a changelog fix entry.

## Best Possible Solution

Update `src/agents/tools/image-tool.ts` so media-store URLs are detected before workspace-relative resolution, preferably by using `resolveMediaReferenceLocalPath` or by excluding `refInfo.isMediaStoreUrl` from the `workspaceDir` branch. Add regression coverage through `createImageTool` or `createOpenClawTools` with `workspaceDir` and `fsPolicy: { workspaceOnly: true }` for both bare and `MEDIA:`-prefixed inbound refs, while preserving the shared inbound-media safety checks and coordinating with the sandbox-specific #74186 work.
