## Summary

Keep open. The report is fresh, non-maintainer-authored, has no protected labels or linked implementation candidate, and current main still has a plausible stale-binding path: Discord preflight resolves the configured route, then lets a persisted/current conversation binding override both session key and agent id from the stored target session key. Existing startup cleanup is scoped to stale ACP thread bindings and does not cover generic/current Discord channel bindings or routing/default-account drift. Related ClawSweeper reports cover different Discord lifecycle, DM-key, PluralKit, or macOS-service issues and do not supersede this routing/persona mismatch.

## Best Possible Solution

Keep this issue open and fix it in the Discord ingress/conversation-binding path. The best direction is to add a freshness check or stored route fingerprint for Discord current-conversation bindings, compare the stored target agent/session against the current effective route when channel/defaultAccount/agent bindings change, and either ignore/unbind stale bindings or emit a clear diagnostic. Add focused tests around Discord preflight with a stale agent:<old>:discord:channel:<id> binding plus a current route to a different agent/account, while preserving intentional ACP/thread-bound sessions.
