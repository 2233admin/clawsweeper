## Summary

Keep this PR open for maintainer review. Current main still does not retry Telegram native command-menu `setMyCommands` after 429/`retry_after` responses, while this PR is a narrow replacement for the closed, unmerged contributor work in #49569; the open #67782 is related Telegram command-menu work but addresses the separate delete-before-set optimization.

## What This Changes

This PR adds Telegram native command-menu retry handling for structured and fallback `retry_after` 429 errors, adds focused command-menu tests including no-cache-on-set-failure coverage, and credits the source contributor work in the changelog.

## Best Possible Solution

Land a narrow Telegram plugin fix that retries `setMyCommands` on Telegram 429 `retry_after` responses, preserves the existing `BOT_COMMANDS_TOO_MUCH` shrink behavior, avoids caching unsynced command hashes after failures, keeps #67782 separate unless maintainers explicitly combine it, and keeps the changelog credit for @andyliu/#49569.

## Solution Assessment

Yes. The PR keeps the fix in the Telegram plugin, uses the dependency-backed error shapes, adds focused regression tests, and does not broaden core/plugin contracts or mix in the separate skip-delete change.
