## Summary

Keep open. Current main still lacks a chunk-table text fallback when FTS is unavailable; the merged #74175 fix only covers FTS MATCH failures when the FTS table exists, and this PR remains useful but not merge-ready because the branch is stale and still needs semantic/performance review.

## What This Changes

The PR adds a memory-core chunk-scan text-search fallback for FTS-unavailable SQLite builds, re-enables related memory regression tests, skips FTS-dependent helper tests on unsupported runtimes, and includes stale-base CLI/gateway/doctor test fixture adjustments.

## Best Possible Solution

Keep the useful fallback idea, but land it as a narrow memory-core change: add a bounded chunk-table text fallback only for FTS-unavailable search paths, preserve provider vector-only and hybrid ranking semantics, keep source/model filters intact, and add focused regression coverage for populated chunks with unavailable FTS.
