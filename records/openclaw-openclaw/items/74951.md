## Summary

Keep this PR open, but it is not merge-ready as submitted. The PR addresses a real agent-turn blocking problem, yet it changes process-denied exec semantics so a backgrounded command can run with no process tool and no automatic timeout when callers pass `timeout: 0`.

## What This Changes

The PR enables exec auto-backgrounding even when `process` is denied, adds hidden-process follow-up wording and docs, and adds regression coverage for a long process-denied exec call.

## Best Possible Solution

Land a corrected version only if maintainers accept the new process-denied contract: keep the improved hidden-process text and docs, but preserve a finite timeout or reject timeoutless backgrounding whenever the process tool is unavailable.

## Reproduction Assessment

Yes. The focused path is `createOpenClawCodingTools` with `tools.deny: ["process"]`, then a long `exec` call with a small `yieldMs`; current main source proves that path is synchronous, while the PR adds coverage expecting `status: "running"`.

## Solution Assessment

No, not as submitted. The narrow wiring is plausible, but the mergeable solution should not let process-denied background sessions opt out of the only remaining lifecycle timeout.

## Review Findings
