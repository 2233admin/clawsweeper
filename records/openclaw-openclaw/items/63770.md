## Summary

Current main still ignores deleteAfterRun for non-at schedules because applyJobResult keeps the at-only shouldDelete guard. The issue should stay open because PR #64375 is open, unmerged, and explicitly proposes the narrow fix with closing syntax; the right path is maintainer review of that PR rather than auto-closing the issue.

## Best Possible Solution

Review PR #64375 as the canonical implementation candidate. If maintainers agree with recurring jobs self-deleting when deleteAfterRun is explicitly true, land the narrow guard removal with regression tests for every/cron successful deletion and non-ok retention, plus a user-facing changelog entry. If maintainers reject that semantic change, close or revise the PR and record the decision on this issue.
