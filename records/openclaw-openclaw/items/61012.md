## Summary

Keep open. Current main still does not route omitted-account Telegram message actions through the Telegram default-account resolver. The direct Telegram send/account resolver path knows about `channels.telegram.defaultAccount`, but the CLI/gateway message-action path can still hand an undefined account into `extensions/telegram/src/action-runtime.ts`, where token lookup normalizes that to the implicit `default` account instead of the configured default account.

## Best Possible Solution

Fix the Telegram-owned action path so omitted-account token-gated actions resolve a `ResolvedTelegramAccount` via `resolveTelegramAccount({ cfg, accountId })` before token use, then pass the resolved token and resolved account ID into send, poll, react, edit, delete, sticker, and forum-topic operations. Add focused regression tests for `channels.telegram.defaultAccount` pointing to a named account with no top-level/default token, plus explicit-account tests to preserve multi-bot routing and unknown-account fail-closed behavior.
