## Summary

Keep this PR open. Current main still does not implement the Feishu native post `at` element fix: structured mentions are converted into pseudo-XML text and sent as one `md` element, which is the behavior this PR is meant to replace. The PR is also an active implementation candidate with a maintainer-repaired head and merge-ready context, so the best cleanup path is review and landing rather than closure.

## Best Possible Solution

Keep this PR open for maintainer review and landing. The desired implementation belongs in the Feishu plugin: pass structured mention targets into `buildFeishuPostMessagePayload`, emit native Feishu post `at` elements with optional `user_name`, keep literal body `<at ...>` text from being parsed as an outbound mention, and retain the regression tests and changelog credit from the repaired branch.
