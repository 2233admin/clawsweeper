## Summary

Keep this PR open. Current main still lacks table-aware protection in `EmbeddedBlockChunker`, and the newer related PR #73981 plus broader #63460 are both open and unmerged, so there is no shipped superseding fix yet; this branch is still relevant but has blocking table-streaming edge cases to resolve before merge.

## What This Changes

The PR adds a Markdown table span parser, wires table spans into `EmbeddedBlockChunker` break protection alongside code fences, and adds parser/chunker unit coverage.

## Best Possible Solution

Land one canonical table-aware block-streaming fix that protects Markdown tables without emitting partial streaming tables early, covers no-edge-pipe table syntax, includes `subscribeEmbeddedPiSession` integration coverage, and then close whichever open PRs are superseded by the landed SHA.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Defer emitting table spans at the streaming tail:** `src/agents/pi-embedded-block-chunker.ts:455`
  - body: When `drain({ force: false })` sees a partial table longer than `maxChars`, `parseTableSpans` can report the current buffer tail as the table end. Returning that end emits the early rows immediately, so later `append()` calls continue the same table in a new message and the new protection still breaks incremental streaming tables.
  - confidence: 0.9
- **[P2] Expand no-edge pipe rows into table spans:** `src/markdown/table-spans.ts:76-86`
  - body: For valid two-column Markdown tables without outer pipes, header/body rows contain only one `|`, but the level-1 expansion requires `countPipes(...) >= 2`. The parser then protects only the delimiter row, leaving the header and body available for chunk splits.
  - confidence: 0.88
