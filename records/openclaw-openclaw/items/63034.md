## Summary

Keep open. Current main still leaves OpenRouter Anthropic cache markers system/developer-only, while the direct Anthropic path still marks the trailing user turn. The issue is also paired with open PR #63062 using closing syntax, so it should stay open until that implementation is reviewed, landed, or rejected.

## What This Changes

The issue asks OpenRouter Anthropic requests to apply `cache_control` to conversation messages, especially the trailing user message, instead of only system/developer prompt blocks.

## Best Possible Solution

Review #63062 as the canonical implementation candidate. If maintainers accept the behavior, land a focused change that reuses the existing trailing-user cache-control helper on the verified OpenRouter Anthropic path and updates tests plus public prompt-caching/OpenRouter docs; if not, document the intentional system/developer-only policy and close or retitle accordingly.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is the existing OpenRouter wrapper test path with a system message plus a user message: current code tags the system block and leaves the user content unchanged, and the test suite currently asserts that behavior.

## Solution Assessment

Unclear as a final product decision, but the narrow technical direction is maintainable. Reusing the existing trailing-user helper is the smallest likely code change, while docs and shared-helper consumers need maintainer review before the paired PR lands.
