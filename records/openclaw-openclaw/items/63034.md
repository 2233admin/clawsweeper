## Summary

Current main still has the reported OpenRouter gap: the verified OpenRouter Anthropic wrapper applies the helper that marks only system/developer messages and strips assistant thinking-block markers, while direct Anthropic still tags the trailing user message. Tests and docs currently encode system/developer-only OpenRouter cache markers, and open PR #63062 is already linked with closing syntax, so this issue should remain open until that PR is reviewed, landed, or rejected.

## What This Changes

OpenRouter Anthropic requests should apply Anthropic `cache_control` markers to conversation messages, especially the trailing user message, instead of only system/developer prompt blocks.

## Best Possible Solution

End state should be an explicit maintainer decision for OpenRouter Anthropic cache semantics: either land the focused PR-style change that reuses the existing trailing-user cache-control helper with regression coverage and docs updates, or document that OpenRouter intentionally remains system/developer-only and close or retitle accordingly.
