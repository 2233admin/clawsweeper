## Summary

Keep open. Current main still has the reported observability gap in internal-hook error logging, still contains plausible unguarded string-method paths matching the reported TypeError, and two open unmerged PRs explicitly target this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to stop repeated `f.toLowerCase is not a function` errors during `agent:bootstrap` and make hook failures diagnosable with stack/context instead of message-only logs.

## Best Possible Solution

Keep this issue open until maintainers review the open implementation candidates and land a narrow fix that both guards runtime model/session override normalization at JSON/plugin boundaries and emits redacted stack plus event context for internal-hook failures.
