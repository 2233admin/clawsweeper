## Summary

Keep this issue open. Current main at b85edb3f0cf6c2a8bec06537af9211a044266a6c still lacks a Feishu outbound minimum-interval/rate-limit config field, and the visible Feishu reply, comment, message-tool, card/media, fallback, and streaming-card send paths are not routed through a shared configurable per-account pacing helper. The related #13615 is still a broader external API throttling tracker and does not supersede this Feishu-specific channel request.

## Best Possible Solution

Keep this issue open and implement Feishu-owned outbound pacing in the bundled Feishu plugin. Add a documented top-level and per-account minimum outbound interval setting, expose it through schema/manifest/docs, and route normal replies, comment replies, message-tool text/card/media sends, fallback sends, pairing notifications, and streaming-card create/update/close calls through one per-account pacing helper. If #13615 later adds a generic limiter contract, Feishu can adopt it underneath, but this issue should close only once the Feishu setting exists and is applied.
