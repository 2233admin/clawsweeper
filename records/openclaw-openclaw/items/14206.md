## Summary

Keep this issue open. Current main still has no Feishu outbound minimum-interval/rate-limit config, and the Feishu reply, comment, outbound adapter, media, and streaming-card send paths still call Feishu APIs directly or use only a hard-coded streaming update throttle. The open broader core tracker #13615 and retry/backoff tracker #70879 are related but do not supersede this proactive Feishu outbound pacing request.

## What This Changes

The issue requests configurable outbound message rate limiting/minimum-send intervals for the Feishu plugin, including how chunked LLM responses are segmented and sent.

## Best Possible Solution

Implement Feishu-owned outbound pacing in the bundled Feishu plugin: add documented top-level and per-account minimum outbound interval config, expose it through schema and manifest, and route normal replies, comment replies, message-tool text/card/media sends, fallbacks, pairing notifications, and streaming-card create/update/close calls through one per-account pacing helper. Keep #13615 as the broader core limiter tracker and #70879 as the separate retry/backoff follow-up unless a PR intentionally covers those scopes too.
