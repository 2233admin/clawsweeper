## Summary

Keep open. Current main still lacks a Feishu outbound minimum-interval or pacing configuration, and the central reply, comment, message-tool, media, and streaming-card send paths can still call Feishu APIs without a shared per-account outbound pacing layer. Related #13615 and #70879 are useful sibling trackers, but they do not supersede this narrower proactive Feishu pacing request.

## What This Changes

Request to add configurable minimum-interval message pacing for the Feishu channel, preferably also influencing how long LLM responses are split into outbound messages.

## Best Possible Solution

Add a Feishu-owned, per-account outbound pacing layer that resolves top-level and per-account configuration, gates every Feishu API path that can burst visible outbound messages, documents the setting in the Feishu channel docs and manifest/schema, and keeps #13615 for broad cross-provider limits and #70879 for retry/backoff handling.
