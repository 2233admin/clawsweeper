## Summary

Keep open. Current main still lacks the requested BlueBubbles trainer/supervised owner-approval mode, and the issue is fresh, concrete, and scoped to an existing bundled plugin rather than stale or already implemented work.

## What This Changes

Add an optional BlueBubbles trainer/supervised mode that drafts DM replies, forwards them to the owner for approval or correction, and prevents delivery to the original sender without owner sign-off.

## Best Possible Solution

Keep this BlueBubbles-owned: add schema-backed trainer/supervised settings, intercept DM final delivery before BlueBubbles sends, route drafts to the configured owner, handle approval/correction commands with TTL expiry, log corrections only under approved privacy semantics, update docs, and cover the delivery and command paths with focused tests.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence current-main verification path: the requested config/docs are absent and the DM reply path still sends assistant output directly through sendMessageBlueBubbles.

## Solution Assessment

Unclear until BlueBubbles owner/product review. The proposed plugin-local boundary is the right starting point, but command grammar, correction logging, AI tagging, and privacy defaults need approval before a repair PR is safe to queue.
