## Summary

Keep open. Current main still lacks the requested BlueBubbles trainer/supervised reply mode, and the feature is actionable but needs BlueBubbles maintainer, product, and privacy review before an automated implementation should be queued.

## What This Changes

Add an optional BlueBubbles trainer/supervised mode that captures DM agent replies, routes drafts to the owner for approval or correction through iMessage, logs corrections, and only sends approved replies.

## Best Possible Solution

Keep the work BlueBubbles-owned unless implementation proves a reusable channel approval seam is needed: add schema-backed trainerMode/trainerNotifyNumber/agentTag, intercept DM final delivery before sendMessageBlueBubbles/sendBlueBubblesMedia, manage pending drafts with TTL, parse owner approve/correct commands, deliver only approved supervised replies, define privacy-safe correction logging, and update docs plus focused tests.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence current-main verification path. Source, docs, tests, and config searches show the requested fields are absent, and the DM reply path still sends final agent output directly to the resolved BlueBubbles target.

## Solution Assessment

Unclear until BlueBubbles owner/product review. The proposed plugin-local boundary is the right starting point, but the exact owner command grammar, correction logging, AI tagging, and privacy defaults should be settled before an implementation PR is queued.
