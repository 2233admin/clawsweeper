## Summary

Keep open. Current main still lacks the requested BlueBubbles trainer/supervised owner-approval mode, and the request is fresh, actionable, and aimed at an existing bundled plugin rather than an external-only ClawHub idea.

## What This Changes

Add optional BlueBubbles trainer/supervised modes that let the agent draft replies but require owner approval or correction over iMessage before anything is sent to the original DM sender.

## Best Possible Solution

Keep the work BlueBubbles-owned: add typed and schema-backed trainerMode, trainerNotifyNumber, optional agentTag, pending-message TTL, owner command parsing, outbound-send interception, docs, and focused tests under the BlueBubbles plugin while reusing existing session/memory surfaces where possible.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence current-main verification path: source and docs searches show the requested config/API is absent, and the active DM reply path still sends directly to the resolved outbound target.

## Solution Assessment

Unclear until owner/product review, but the plugin-local direction is the right starting point because the behavior is specific to BlueBubbles/iMessage DM delivery and current evidence does not show a missing core SDK seam.
