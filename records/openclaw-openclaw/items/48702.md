## Summary

Keep this PR open. Current main still does not implement the PR's intended `/status` clarification: reasoning is omitted when it is `off`, and enabled reasoning still displays raw values like `on` instead of user-facing labels like `visible`. The PR is stale against the newer status-rendering layout, but the focused UX change remains useful and is not superseded by current main.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main. Move the display change into `src/status/status-message.ts`, always render a reasoning visibility line, map canonical values to `hidden`/`visible`/`stream`, normalize resolved and persisted reasoning inputs safely, and update the current status and system-prompt tests around the new file layout.
