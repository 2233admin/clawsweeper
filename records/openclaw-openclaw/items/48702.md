## Summary

Keep this PR open. Current main still does not implement the requested /status clarification: the status renderer hides reasoning when it is off and displays raw values like `on` instead of user-facing labels like `visible`. The PR is stale against the newer status file layout and should be rebased, but its core UX change remains useful. Security pass found no CI, dependency, package, secret, release, or code-execution surface in the supplied four-file diff.

## Best Possible Solution

Keep the PR open, but ask for a rebase onto current main and move the display change into `src/status/status-message.ts`. The implementation should always render a reasoning visibility line, map canonical values to `hidden`/`visible`/`stream`, normalize persisted/resolved reasoning values safely, and update the existing status and system-prompt tests around the new file layout.
