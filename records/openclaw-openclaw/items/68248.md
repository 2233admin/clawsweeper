## Summary

Keep open. Current main implements the core agent/avatar plumbing, but the full request is not complete: favicon links are still static, agent/session selector surfaces remain text-only, and the Control UI personal display-name path is inconsistent because docs/storage/chat support a local name while Quick Settings now intentionally renders a fixed "You" and exposes only avatar controls.

## What This Changes

The issue requests per-agent avatar images, per-agent/browser-tab favicon customization, and user display-name overrides in the Control UI.

## Best Possible Solution

Keep the shipped avatar route/model, then narrow the remaining work to an explicit Control UI decision: either restore a browser-local display-name editor and pass it as client displayName, add safe active-agent favicon updates, and replace text-only agent selectors where avatars are required, or update docs/tests to state that those parts are intentionally out of scope.
