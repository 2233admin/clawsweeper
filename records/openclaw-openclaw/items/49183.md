## Summary

Keep this issue open. Current main has adjacent global, per-agent, provider, group, sender, subagent, sandbox, and owner-only tool policy layers, but it still does not implement the requested operator-scope-to-agent-tool-policy bridge or a `toolsByScope` config surface.

## Best Possible Solution

Keep this RFC open for maintainer product/security design. The best path is a typed, documented session or operator-scope-aware policy input in the shared tool-policy resolver, with authenticated gateway scopes and trusted sender identity threaded into agent/tool resolution, owner-only gates preserved, schema/docs updated, and tests proving narrower operator scopes cannot access denied tools while admin/full-operator sessions keep intended access. Reconcile that design with #43673, #53530, #53496, and #58434 before implementation.
