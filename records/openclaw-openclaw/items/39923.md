## Summary

Keep #39923 open. Current main still implements the numbered config backup ring, still treats non-numbered `.bak.*` entries as orphans, and tests continue to assert `.bak.1` through `.bak.4` behavior. The linked PRs #39982 and #40392 are still unmerged implementation candidates, while #46353 was closed unmerged; #20460 is related but broader and does not make this narrower actionable issue obsolete by itself.

## Best Possible Solution

Keep #39923 open until maintainers either merge a datetime-suffix backup implementation or explicitly decide to retain the numbered ring. The likely code path is `src/config/backup-rotation.ts` with matching `src/config/config.backup-rotation.test.ts` coverage for bounded timestamp pruning, permission scanning, collision handling, and legacy `.bak.N` migration; PRs #39982 and #40392 remain the relevant implementation candidates.
