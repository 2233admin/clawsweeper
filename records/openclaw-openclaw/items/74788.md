## Summary

Keep open. Current main still lacks an OpenRouter-owned thinking profile or supported-effort metadata for `xhigh`, and open PR #74839 is already the active closing implementation path, so this issue should not be closed before that PR merges or is replaced.

## What This Changes

Requests that the OpenRouter provider advertise `xhigh` thinking for DeepSeek V4 Pro and Flash so `/think xhigh` is accepted before the OpenRouter transport sends the supported reasoning payload.

## Best Possible Solution

Add the level exposure in the OpenRouter plugin boundary, with regression coverage for DeepSeek V4 Pro/Flash positive cases and non-V4 or unsupported-route negative cases, so directives, UI/session validation, catalog metadata, and transport behavior agree without a core API change.

## Reproduction Assessment

Yes. The source-level path is high confidence: current main leaves OpenRouter on the base thinking profile, so an explicit `/think xhigh` for `openrouter/deepseek/deepseek-v4-pro` reaches the unsupported-level rejection before transport wrapping can send `reasoning.effort`.

## Solution Assessment

Yes. A plugin-local OpenRouter `resolveThinkingProfile` or equivalent compat metadata is the narrowest maintainable fix because provider docs say plugins own model-specific level sets and the OpenRouter transport already handles accepted non-off levels.
