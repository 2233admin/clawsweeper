## Summary

Keep open. Current main still lacks an OpenRouter-owned thinking profile or xhigh hook, so OpenRouter DeepSeek V4 models fall back to the base thinking levels before the stream wrapper can forward `reasoning.effort`. The issue is also paired with open closing PR #74839, so cleanup should wait for that PR or a replacement to land.

## What This Changes

Allow `/think xhigh` for `openrouter/deepseek/deepseek-v4-pro` and `openrouter/deepseek/deepseek-v4-flash` by advertising the supported OpenRouter provider thinking levels.

## Best Possible Solution

Land a narrow OpenRouter-owned provider profile fix, or an equivalent replacement, that advertises the supported DeepSeek V4 thinking levels and adds regression coverage for both positive V4 models and at least one negative OpenRouter model.

## Reproduction Assessment

Yes. The source-level path is high confidence: with provider `openrouter` and model `deepseek/deepseek-v4-pro` or `deepseek/deepseek-v4-flash`, current main reaches thinking validation without an OpenRouter profile, compat effort list, or legacy xhigh hook, so `/think xhigh` is rejected before transport shaping.

## Solution Assessment

Yes. A provider-owned `resolveThinkingProfile` or equivalent OpenRouter hook is the narrow maintainable solution; the open PR should be reviewed to ensure the profile is limited to models with documented xhigh support rather than the generic OpenRouter `reasoning` boolean alone.
