## Summary

Keep this PR open. Current main still does not implement the PR's intended observability hardening, and the submitted patch is not safe to close as obsolete or ready: its logging calls can throw in formerly non-throwing cleanup/fallback paths, the TLS chmod warning is proposed at info severity, and the pairing allow-from code has moved behind a shared helper that the PR no longer updates.

## Best Possible Solution

Keep this PR open for author or maintainer revision. A safe version should rebase onto current main, move malformed allow-from JSON logging into `src/pairing/allow-from-store-file.ts`, log cleanup failures without passing metadata objects as `RuntimeEnv`, use warning severity for TLS chmod failures, preserve non-throwing fallback behavior, and add focused regression tests for these error paths.
