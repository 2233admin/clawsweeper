## Summary

Keep this PR open. Current main still has the Matrix malformed percent-decoding bug, while the proposed branch is a narrow parser guard with focused regression coverage and no blocking review findings from this read-only pass.

## What This Changes

The PR adds guarded Matrix geo URI parameter decoding that returns an invalid location on malformed percent-encoding and adds a colocated Matrix location parser test file.

## Best Possible Solution

Land a small Matrix-owned parser guard with regression coverage that treats malformed percent-encoded `geo:` parameters as invalid input while preserving valid encoded accuracy parsing and the existing inbound location flow.

## Reproduction Assessment

Yes. On current main, an inbound Matrix location payload such as `geo:45.5,9.2;u=%` reaches `resolveMatrixLocation`, and the raw parameter value is passed to `decodeURIComponent`, which throws `URIError` for `%`.

## Solution Assessment

Yes. Guarding `decodeURIComponent` inside the Matrix geo URI parser and returning `null` for malformed parameters is the narrowest maintainable fix; it avoids core changes and preserves valid event behavior.
