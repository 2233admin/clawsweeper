## Summary

Keep open. The PR targets a plausible Ollama tool-call normalization gap tied to #74487, but the proposed helper over-strips bare `functions`/`tools` prefixes and can corrupt legitimate tool names. Current main also already has generic embedded-runner normalization for provider-prefixed live and replayed tool calls, so the remaining decision is whether Ollama needs this lower-level normalization after the regex is tightened.

## What This Changes

The branch adds an Ollama-local tool-call name normalizer in stream conversion/reconstruction and regression tests for prefixed `functions.exec`/`tools.exec` names.

## Best Possible Solution

Tighten Ollama-local normalization to strip only explicit provider prefixes with a separator, add negative tests for legitimate names beginning with `functions` or `tools`, and then decide whether the Ollama boundary coverage is still useful beyond the existing generic runner normalization.

## Review Findings
