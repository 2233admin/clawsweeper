## Summary

Keep open. Current main still only extracts Discord embed title/description from the first embed for inbound messages, forwarded snapshots, and thread starters, so this PR’s central behavior is not implemented. The PR is focused on the Discord plugin path, adds relevant tests, and the bot-review test gap was addressed by the second PR commit. Security review did not find supply-chain or infra-sensitive changes in the PR diff.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent Discord plugin change: expand the embed-like type to include fields/footer, extract title/description/fields/footer, iterate all embeds in inbound message, forwarded snapshot, and thread-starter paths, keep the existing fallback ordering, and run the targeted Discord monitor tests plus the changed gate.
