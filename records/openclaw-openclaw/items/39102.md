## Summary

Current main still does not implement this PR’s intended per-agent outbound A2A allowlist override. The shipped policy remains global-only, per-agent `tools.agentToAgent` is neither typed nor accepted by the strict agent tools schema, session tool denial paths still reference only global `tools.agentToAgent`, and deletion pruning only scrubs the global allowlist. The PR remains a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or an author refresh. The best path is to implement the override in the shared A2A policy, add typed and strict-schema support for `agents.list[].tools.agentToAgent.allow`, document the config surface, preserve global participation plus outbound-only requester semantics, prune deleted agent IDs immutably, and validate with targeted session/A2A tests plus the changed gate.
