## Summary

Keep open. Current main still only partially addresses the report: some image limits are configurable, but the requested shared image byte/input-pixel/dimension contract is still absent across several sanitizer, media, gateway, sandbox, and ACP layers.

## What This Changes

The issue requests a configurable image media limit contract for byte limits, input-pixel limits, resize dimensions, and per-request totals across OpenClaw media handling layers.

## Best Possible Solution

Define a backwards-compatible, documented image limit contract with global defaults and scoped overrides, then thread it through agent image sanitization, media image-ops, generic inbound media, media-store and sandbox staging, gateway attachment parsing, ACP attachment reads, and OpenAI-compatible HTTP paths while preserving existing aliases where appropriate.
