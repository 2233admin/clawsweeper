## Summary

Keep this issue open. Current main still closes the long-lived WhatsApp inbound listener and the shared WhatsApp socket helper through raw WebSocket close calls instead of Baileys `sock.end(error)`, the pinned Baileys version exposes the requested API, and the same-author fix PR is closed unmerged.

## Best Possible Solution

Keep this issue open and fix it in the WhatsApp plugin. Widen the inbound and shared close-helper socket types to include `end(error?: Error)`, prefer `sock.end(new Error("OpenClaw listener close"))` when available, preserve `sock.ws?.close?.()` as a compatibility fallback, and update existing teardown tests to assert both Baileys-capable and fallback socket behavior. Review #52443 as prior art, but it is closed unmerged and does not implement the fix on main.
