## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: Discord partial preview streaming cannot opt into assistant-boundary preview rotation. The PR remains useful, but it needs to be ported to the current pluginized Discord layout and reconciled with the nested streaming config shape plus the unresolved in-flight send failure review comment.

## Best Possible Solution

Keep the PR open as a useful implementation candidate. The best path is to port the option into the current Discord plugin surface, preferably under the existing nested streaming preview config shape, wire `message-handler.process.ts` to resolve that option instead of `discordStreamMode === "block"`, add strict schema/UI/docs coverage, add a partial-mode boundary-rotation regression test, and fix the stale in-flight send failure path before review or merge.
