## Summary

Keep this PR open. Current main still does not expose an independent Discord partial-mode assistant-boundary preview rotation option: the runtime split gate remains tied to block mode, schema/types/docs have no split field, and tests only cover block-mode boundary rotation. The PR remains a useful implementation candidate, but it needs a port to the current `extensions/discord` plugin layout and nested `streaming.preview` config shape.

## Best Possible Solution

Keep this PR open as an implementation candidate. Port the option into the current Discord plugin surface under the existing nested streaming config shape, add a resolver that preserves block-mode defaults while allowing partial-mode opt-in, wire `extensions/discord/src/monitor/message-handler.process.ts` away from the hardcoded block-only split gate, update strict schema/types/config UI hints/docs, add a partial-mode boundary-rotation regression test, and finish the stale in-flight send failure handling before merge.
