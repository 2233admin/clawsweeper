## Summary

Keep this PR open. Current main at 7a23b2d945cc2d2e5c452e4641213e71de8dcc66 still lacks the proposed Discord partial-mode assistant-boundary preview rotation option: the runtime only splits previews in block mode, the resolver/schema/types/docs have no boundary-split field, and tests only cover block-mode rotation. The PR remains useful, but it needs a port to the current pluginized Discord layout and the nested streaming config shape.

## Best Possible Solution

Keep the PR open as an implementation candidate. Port the option into the current Discord plugin surface, preferably under the existing nested `channels.discord.streaming.preview` shape, add a resolver that preserves block-mode defaults while allowing partial-mode opt-in, wire `extensions/discord/src/monitor/message-handler.process.ts` away from the hardcoded block-only split gate, update strict schema/types/config UI hints/docs, add a partial-mode boundary-rotation regression test, and fix the stale in-flight send failure path before review or merge.
