## Summary

Keep this PR open. Current main still scopes Control UI token/session storage by exact gateway host, so the central loopback-alias token fragmentation is not implemented on main. The PR targets the right storage surface and the author addressed the earlier IPv6 test/dead-code review, but the submitted implementation still needs a security/correctness revision before it should land because `host.startsWith("127.")` also matches ordinary DNS hostnames that are not loopback IP literals.

## Best Possible Solution

Keep this PR open and request a focused revision in `ui/src/ui/storage.ts`: normalize only `localhost` plus actual numeric IPv4/IPv6 loopback literals, preserve protocol/port/path boundaries, add positive tests for localhost/127/[::1] same-port sharing, add negative tests for DNS hostnames such as `127.example.com` and `127.0.0.1.nip.io`, update the fixtures for the current `UiSettings` shape, then run the targeted Control UI storage test/check before landing.
