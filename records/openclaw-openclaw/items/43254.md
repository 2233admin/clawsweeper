## Summary

Keep this PR open. Current main still scopes Control UI token/session storage by the exact parsed gateway host, so `localhost`, `127.0.0.1`, and `[::1]` remain separate token scopes. The PR targets the right storage surface and the author addressed the initial IPv6/dead-code review, but the PR head still has an unresolved security/correctness issue because `host.startsWith("127.")` also matches DNS hostnames like `127.example.com` and `127.0.0.1.nip.io`, which could collide with a real loopback token scope.

## Best Possible Solution

Keep this PR open and request a focused revision: normalize only `localhost` plus actual numeric loopback IP literals, preserve protocol/port/path boundaries, add negative tests for DNS hostnames such as `127.example.com` and `127.0.0.1.nip.io`, update the tests for the current `UiSettings` fields, then run the targeted Control UI storage check before landing.
