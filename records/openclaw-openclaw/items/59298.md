## Summary

Keep open: current main does not implement the requested update-hint command, so this PR is still the active implementation candidate. The patch is not mergeable as-is because the advertised automatic skill notification is added as fenced Markdown in skill files, while current skill handling advertises and materializes SKILL.md files but does not execute Markdown code blocks before a skill runs.

## What This Changes

The branch adds an `openclaw update hint` CLI command with tests and inserts cached-update bash hint blocks into seven `.agents/skills/*/SKILL.md` files.

## Best Possible Solution

If maintainers want this feature, keep the small cached-state CLI helper but surface update availability through a shared executed runtime hook or documented generic skill prelude, with tests proving the hint appears for eligible skill invocations without hand-editing individual skill files.
