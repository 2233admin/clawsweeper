## Summary

Keep open. Current main partially addresses the report by carrying `stopReason` through the pi-embedded terminal lifecycle metadata, but the normal `lifecycle:end` emitter still has no `aborted` field and the fallback payload that includes it remains guarded off after the subscribe lifecycle path emits the terminal event.

## What This Changes

The issue requests adding `aborted` and `stopReason` to pi-embedded `lifecycle:end` event payloads so event subscribers can distinguish cancellation from natural completion.

## Best Possible Solution

Extend the existing terminal lifecycle metadata seam to carry an explicit `aborted` boolean from runner result metadata into both global `emitAgentEvent` and direct `onAgentEvent` `lifecycle:end` payloads, with focused regression coverage for normal completion, explicit abort, and timeout/fallback semantics.
