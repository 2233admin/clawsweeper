## Summary

Keep open. Current main still appears vulnerable to the reported Control UI duplicate-final path, so this PR is not obsolete. However, the submitted branch is dirty and security-sensitive beyond the stated UI fix, and review comments identify correctness gaps in the PR’s own dedupe implementation.

## Best Possible Solution

Keep this PR open for maintainer review, but do not merge the current branch as-is. The best path is a minimal UI-only fix or replacement PR that deduplicates terminal chat events by runId across active-run completion, tool-history reloads, and sub-agent final paths, with focused tests in `ui/src/ui/controllers/chat.test.ts` and `ui/src/ui/app-gateway.node.test.ts`, while dropping the unrelated security-sensitive changes.
