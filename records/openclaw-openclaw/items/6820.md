## Summary

Current main still does not implement the requested OpenAI Codex base `gpt-5.2` xhigh support. The provider recognizes `openai-codex/gpt-5.2` as a modern model, but `OPENAI_CODEX_XHIGH_MODEL_IDS` still omits base `gpt-5.2`, so `resolveThinkingProfile` does not expose `xhigh` for that provider/model pair. The related one-off fix PRs are closed unmerged, and open RFC #37804 is broader design context rather than a canonical replacement for this concrete missing-model request.

## Best Possible Solution

Keep this open until the OpenAI Codex provider either adds base `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage for `listThinkingLevels("openai-codex", "gpt-5.2")` or `resolveThinkingProfile`, or maintainers explicitly document that base OpenAI Codex GPT-5.2 is high-only while `gpt-5.2-codex` is xhigh-capable. Treat #37804 as broader design context, not a replacement for this concrete missing model case.
