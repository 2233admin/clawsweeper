## Summary

Keep open. Current main still lacks numeric-bound enrichment for config validation errors, and this PR is the active implementation candidate for the linked issue, but the patch should not merge as-is because it can report an invalid exclusive lower bound as allowed for existing positive-number config fields.

## What This Changes

The PR adds generic numeric min/max hints to mapped config validation messages and adds regression coverage for an over-limit `session.agentToAgent.maxPingPongTurns` value.

## Best Possible Solution

Keep this PR open and update the central mapper to honor Zod `inclusive` for both upper and lower bounds before appending guidance. Add regression coverage for the reported inclusive max case plus an existing exclusive lower-bound case such as a `.positive()` config field set to `0`, then rerun the targeted config validation test and the required changed gate.

## Reproduction Assessment

Yes. The linked issue reproduces by validating `session.agentToAgent.maxPingPongTurns: 10` against the current `.max(5)` schema, and the PR finding reproduces conceptually with any existing `.positive()` config field set to `0`, which Zod reports as `too_small` with `inclusive: false`.

## Solution Assessment

No, not as written. The central-mapper approach is the right place for this UX improvement, but the implementation needs to branch on `record.inclusive` so strict bounds do not tell operators that the excluded boundary value is allowed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Respect exclusive Zod bounds:** `src/config/validation.ts:497-503`
  - body: Current config schemas use `z.number().positive()`; Zod implements that as `gt(0)`, emitting `too_small` with `minimum: 0` and `inclusive: false`. This helper ignores `inclusive`, so a value of `0` would be reported with `(minimum allowed value is 0)` even though `0` is still invalid. Please branch on `record.inclusive === false` for both lower and upper bounds before appending these hints.
  - confidence: 0.92
