## Summary

Keep open. Current main still follows the synthetic restart-recovery path for resumable channel-shaped sessions, while this PR adds the missing manual-recovery guard and focused regression coverage. I found no blocking correctness or security finding in the proposed patch.

## What This Changes

The PR adds channel-backed session detection to main-session restart recovery, marks matching restart-aborted sessions failed for manual recovery, and adds a regression test for that behavior.

## Best Possible Solution

Land a narrow restart-recovery fix that prevents synthetic replay for sessions with durable channel delivery metadata, keeps automatic recovery for local non-channel main sessions, and preserves the focused agents-core regression coverage.
