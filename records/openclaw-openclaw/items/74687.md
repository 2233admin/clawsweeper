## Summary

Keep open. Current main still auto-resumes restart-aborted main sessions with resumable transcript tails, and the PR is a narrow active fix with focused coverage; the remaining step is maintainer review and targeted validation rather than cleanup closure.

## What This Changes

This PR adds channel-backed main-session detection to restart recovery, marks matching aborted sessions failed for manual recovery before synthetic replay, and adds a regression test for that behavior.

## Best Possible Solution

Land a narrow restart-recovery fix that prevents synthetic replay for sessions with durable channel delivery metadata, keeps automatic recovery for local non-channel main sessions, and preserves focused agents-core regression coverage.

## Reproduction Assessment

Yes. The PR's unit-test shape is a high-confidence reproduction: seed a running `abortedLastRun` channel-backed session with a resumable tool-result tail and verify current main would call the synthetic gateway resume path while the patch marks it failed for manual recovery.

## Solution Assessment

Yes. The proposed guard is placed at the restart-recovery decision point before external channel delivery can be re-entered, while leaving the existing non-channel resumable path intact; the only maintainer judgment is whether the metadata classifier is intentionally broad enough.
