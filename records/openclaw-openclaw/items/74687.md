Keep open. Current main still resumes restart-aborted main sessions without a channel-delivery guard, while this active PR adds a narrow guard and focused regression coverage. ## What This Changes

The PR adds channel-backed session detection to main-session restart recovery, marks matching aborted sessions failed for manual recovery before synthetic replay, and adds a regression test for that path.

## Best Possible Solution

Land a focused restart-recovery fix that prevents synthetic hidden replay for sessions with durable channel delivery metadata, preserves automatic recovery for local non-channel main sessions, and keeps the behavior locked by agents-core regression coverage.

## Reproduction Assessment

Yes. A high-confidence reproduction is to seed a running abortedLastRun main session with channel delivery metadata and a resumable transcript tail, then invoke restart recovery and observe current main call the synthetic gateway resume path instead of failing the session for manual recovery.

## Solution Assessment

Yes, pending maintainer confirmation of the metadata predicate. The guard sits at the restart-recovery decision point before replay and leaves the existing non-channel path intact, which is the narrowest maintainable place to enforce this behavior.
