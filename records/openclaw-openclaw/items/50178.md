## Summary

Keep open. Current main has generic OpenAI-compatible reasoning capture plus provider-specific `reasoning_content` replay/backfill fixes for DeepSeek/Venice/Z.AI-style paths, but the Qianfan plugin still has no replay policy or stream wrapper that would preserve/backfill `reasoning_content` for the reported Qianfan Coding Plan tool-call replay error. The reported `baiduqianfancodingplan/qianfan-code-latest` route is also absent from the bundled Qianfan catalog/docs, so maintainers still need to decide whether this belongs in the Qianfan plugin, a custom OpenAI-compatible compatibility option, or documentation of a retired route.

## Best Possible Solution

Keep this open. Add provider-owned replay support for Qianfan-like OpenAI-compatible thinking models: either wire a Qianfan `wrapStreamFn`/replay hook using the shared `reasoning_content` preservation/backfill helper when thinking is enabled, or add a documented custom-provider compatibility option for providers with this contract. Include focused regression coverage for Qianfan/custom OpenAI-compatible tool-call replay and update https://docs.openclaw.ai/providers/qianfan if the old Coding Plan route is retired or renamed.
