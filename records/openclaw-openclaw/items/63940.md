## Summary

Keep open. Current main has not incorporated the relaxed skill-selection behavior: the source, prompt tests, and docs still describe the stricter one-skill read flow. The PR remains a focused implementation candidate, with final wording needing maintainer review because it overlaps the open Skill-tool prompt request in #8813.

## What This Changes

The PR changes the built-in skills prompt to allow outcome-based skill matching and reading one to three likely candidate SKILL.md files, with matching prompt test updates.

## Best Possible Solution

Adopt a single agreed skills-loading prompt that supports outcome-based candidate selection while preserving targeted reads and, if #8813 is accepted, preferring a registered Skill tool; keep source, prompt tests, and user-facing docs aligned.
