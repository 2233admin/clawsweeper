## Summary

Keep this PR open. Current main still does not implement the PR's intended `jobs.json` watcher or an equivalent idle reload path: cron force-reloads from disk only inside timer ticks, and `armTimer` still exits without a timer when there are no enabled jobs or next run times. The linked #52772 issue is closed as duplicate/superseded by this PR, but the PR itself is not implemented on main and still needs rework before merge.

## Best Possible Solution

Keep the PR open, but rework it before merge. The best fix is a cron-owned watcher or bounded idle poll that flows through the existing locked reload/recompute/timer path, wakes or re-arms scheduling after external changes, suppresses self-write invalidations from `persist()`, handles missing-file and rename semantics, and adds focused cron service tests for direct `jobs.json` writes after an empty startup.
