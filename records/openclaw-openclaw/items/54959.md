## Summary

Keep open. Current main still lacks an equivalent OpenShell platform deployment page, and the existing OpenShell documentation covers using OpenShell as a host-side sandbox backend rather than running OpenClaw inside an OpenShell sandbox. The branch is not obsolete, but it still needs maintainer docs/security review because parts of the guide overlap with current proxy/fetch-guard behavior and present security-sensitive operator workarounds.

## What This Changes

The PR adds a new `docs/platforms/openshell.md` deployment guide, registers it in the Platforms navigation, and adds OpenShell-related zh-CN glossary entries.

## Best Possible Solution

Keep the contribution under maintainer docs/security review, then either merge a dedicated `platforms/openshell` guide after reconciling it with `docs/gateway/openshell.md`, `docs/security/network-proxy.md`, current fetch-guard behavior, and related proxy work, or fold the durable operator guidance into the canonical OpenShell docs. The duplicate glossary entries should be cleaned before merge.

## Security Review

Status: needs_attention

Summary: The diff is docs-only, but it introduces security-sensitive operator guidance that should be reviewed before publication.

Concerns:

- **[medium] Custom preload and module monkey-patch guidance:** `docs/platforms/openshell.md:328`
  - body: The guide tells operators to use `NODE_OPTIONS` to preload JavaScript that installs a process-wide proxy dispatcher and monkey-patches `Module._load` for `ws`. That is code-execution and egress-routing guidance, not a direct repository code change, but it should be reconciled with OpenClaw's supported proxy lifecycle and reviewed before being published as deployment advice.
  - confidence: 0.84
- **[low] Supervisor binary replacement instructions:** `docs/platforms/openshell.md:744`
  - body: The advanced section documents replacing the OpenShell supervisor binary inside the cluster container with `docker cp` and `kill -9`. That may be useful for upstream testing, but it normalizes swapping a trusted runtime binary outside a release process and needs clearer scope, rollback, and trust warnings before merge.
  - confidence: 0.76
