## Summary

Keep open. Current main does not implement the PR's shared memory store behavior: the active memory configuration and CLI still expose per-agent `extraPaths` and no `--shared` memory commands, while related reports were already closed as superseded by this PR. The branch is also stale against the current `memory-core` plugin architecture and still has security-relevant review concerns that need maintainer judgment before merge.

## What This Changes

The PR adds an opt-in shared memory store using `memorySearch.sharedPaths`, a reserved `_shared` memory namespace, merged shared search results, CLI `--shared` controls, config validation, and tests around shared path indexing/exclusion.

## Best Possible Solution

Keep this as the canonical shared-memory-store candidate, but rework it against the current `memory-core` plugin and host SDK design with explicit opt-in shared corpus semantics, strict path validation, deterministic store/status behavior, docs/config updates, and focused tests for cross-agent access boundaries.
