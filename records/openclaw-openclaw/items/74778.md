## Summary

Keep this PR open for normal maintainer review. Current main still lets a ciao service in `probing` fall through to the watchdog re-advertise path and only logs conflict rename events, while the PR is a focused Bonjour-plugin fix with targeted tests and docs and no blocking review findings found.

## What This Changes

The PR updates the Bonjour advertiser watchdog to treat ciao probing/announcing and fresh conflict rename windows as in-progress states, adds targeted advertiser tests, and documents the watchdog behavior.

## Best Possible Solution

Land a focused Bonjour plugin fix that treats ciao negotiation and fresh conflict renames as in-progress, keeps bounded disable behavior for genuinely stuck advertisers, and leaves broader bundled-plugin default policy to #74209 and environment-specific handling to #74820.

## Reproduction Assessment

Yes. A unit-level reproduction can hold the mocked ciao service in `probing`; on current main the watchdog does not skip that state, so it reaches the non-announced re-advertise path before eventual recreation.

## Solution Assessment
