## Summary

Keep open. Current main has per-tool collapsed rows and a show/hide toggle, but it does not implement the requested aggregate summary mode that replaces a multi-tool turn with one expandable “N tool calls” control.

## What This Changes

Add a configurable WebChat/Control UI mode that summarizes multiple tool executions into one collapsible tool-call summary instead of showing each tool execution as its own row.

## Best Possible Solution

Implement a UI-level summary mode that groups the tool calls/results for one agent turn behind a single accessible summary control, while preserving the current detailed per-tool rendering as an explicit mode and keeping Gateway protocol/tool-event payloads unchanged unless a real data gap appears.

## Reproduction Assessment

Not applicable as a feature request. Code and test inspection provide a high-confidence verification path: render multiple tool messages/cards through `buildChatItems` and `renderMessageGroup`, and current main produces separate collapsed rows rather than one aggregate summary.

## Solution Assessment

Yes, the maintainable path is a focused Control UI/WebChat rendering and browser-local settings change using the existing chat item, grouping, and expansion-state seams. A safer implementation should avoid backend protocol changes and preserve search/export behavior for expanded details.
