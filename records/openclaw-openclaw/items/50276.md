## Summary

Keep open. Current main still does not route parsed Slack `[[reply_to_current]]` into outbound message-action `threadId`, so the PR is not obsolete, but the PR branch is not mergeable as-is because it introduces compile/test defects and conflicts with the documented Slack `replyToMode="off"` policy.

## What This Changes

The PR adds Slack-specific `[[reply_to_current]]` handling in outbound message-action sends and adds same-channel and cross-channel Slack threading tests.

## Best Possible Solution

Rework this PR through the existing outbound threading seam, avoid redeclaring imported helpers, add runnable coverage for same-channel, cross-channel, explicit `threadId`/`replyTo` precedence, and `first`-mode accounting, then update Slack docs and plugin metadata if maintainers intentionally change `off` mode behavior.
