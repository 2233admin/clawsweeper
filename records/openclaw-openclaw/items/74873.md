## Summary

Keep this PR open. Current main still does not have a shutdown drain manifest, and the PR has addressed the earlier run-id, Linear-specific metadata, and owner-only atomic write concerns, but it is not merge-ready while the new manifest reader accepts unbounded and weakly validated local state for a future boot recovery path.

## What This Changes

The PR adds a gateway shutdown drain manifest with ChatRunRegistry snapshots, drain-manifest read/write/delete helpers, close-handler wiring, tests, and a small system-event trust flag change.

## Best Possible Solution

Land a generic, validated shutdown manifest contract only after the reader rejects malformed, oversized, and unexpected filesystem objects, then let the boot-time recovery PR consume that contract with explicit TTL and recovery behavior.

## Reproduction Assessment

Yes. A source review of the PR head reproduces the remaining blocker: readDrainManifest uses readFileSync and JSON.parse on the manifest path, then returns a type assertion after only shallow top-level checks.

## Solution Assessment

No. The shutdown snapshot seam and atomic owner-only write direction are reasonable, but the recovery-state reader should mirror the existing restart-intent regular-file/max-size guard and validate each returned field before this becomes a boot contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Bound and validate manifest reads:** `src/gateway/server-drain-manifest.ts:119-125`
  - body: `readDrainManifest()` reads whatever is at the manifest path without a regular-file or size check and then casts the parsed sessions after only checking that `sessions` is an array. A stale, corrupt, symlinked, or oversized state file can become a valid-looking recovery manifest or make the boot consumer parse unbounded local state; mirror the restart-intent lstat/max-size guard and validate `writtenAt`, `runId`, `sessionKey`, and `clientRunId` before returning.
  - confidence: 0.86

## Security Review

Status: needs_attention
