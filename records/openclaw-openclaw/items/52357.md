## Summary

Keep open. Current main has newer zh-TW key parity, but it still contains the Mainland-style terms this PR is trying to replace, and the related in-repo follow-up PRs are closed/unmerged rather than a canonical open replacement.

## Best Possible Solution

Keep this PR open as useful contributor work, but do not land it in its old hand-edited form. The maintainable path is to port the terminology decisions into the current Control UI i18n pipeline, likely by expanding `ui/src/i18n/.i18n/glossary.zh-TW.json`, regenerating `ui/src/i18n/locales/zh-TW.ts` against the current 763-key English source, and crediting the prior rebase/baseline work from #59706 where reused.
