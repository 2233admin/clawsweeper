## Summary

Current main still does not implement one-click Anthropic browser OAuth onboarding. The Anthropic provider supports Claude CLI reuse, manual setup-token paste, API-key auth, and runtime compatibility for existing `sk-ant-oat` token traffic, but there is no provider auth choice or runtime method that opens a browser, completes an Anthropic OAuth exchange, detects account type, and configures the auth profile automatically.

## What This Changes

The issue requests a one-click Anthropic browser OAuth authentication flow that detects account type and configures OpenClaw without manual token or API-key setup.

## Best Possible Solution

Keep this issue open for maintainer/product follow-up. If accepted, implement the flow inside `extensions/anthropic` as a provider-owned auth method exposed through `providerAuthChoices`, using the existing `ProviderAuthContext.openUrl`, VPS-aware OAuth handlers, and `buildOauthProviderAuthResult` where applicable; then persist the profile safely, explicitly choose subscription-vs-API defaults, and update onboarding docs and tests. If Anthropic cannot support a sanctioned browser OAuth flow, split this into a smaller token-prefix/API-key auto-detection improvement and a separate blocked/vendor-contract discussion.
