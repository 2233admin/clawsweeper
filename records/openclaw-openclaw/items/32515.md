## Summary

Current main still does not implement one-click Anthropic browser OAuth onboarding. Anthropic auth remains Claude CLI reuse, manual setup-token paste, or API-key auth; existing `sk-ant-oat` handling is runtime compatibility for already-supplied tokens, not a sign-in flow.

## Best Possible Solution

Keep this open for maintainer/product handling. If accepted, implement an Anthropic provider-owned browser OAuth auth method in `extensions/anthropic`, expose it through `providerAuthChoices`, use `ProviderAuthContext.openUrl` and the existing OAuth helper seams, persist the resulting auth profile safely, select subscription-vs-API defaults explicitly, update onboarding/configure/docs, and add focused Anthropic auth-choice/runtime coverage. If Anthropic cannot support a sanctioned browser OAuth flow, retitle this around supported Claude CLI/setup-token/API-key UX and track token-prefix auto-detection as a narrower separate improvement.
