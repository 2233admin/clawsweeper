## Summary

Keep open. Current main still has the reported behavior: `gateway status` runs the service config audit, the audit compares the LaunchAgent PATH against the full Darwin minimal-service-path superset, and the warning text still says `Gateway service PATH missing required dirs` without filtering nonexistent user-bin directories.

## Best Possible Solution

Fix the gateway service PATH audit in `src/daemon/service-audit.ts` or the Darwin expected-path resolver so macOS only warns for meaningful missing entries, such as existing directories or env-configured directories, and update the wording away from `required dirs` if the audit remains advisory. Add a focused Darwin regression test for a healthy LaunchAgent PATH that omits nonexistent user-bin/version-manager directories.
