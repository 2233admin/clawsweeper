## Summary

Keep open. Current main has partial fixes for the original noise path, including scoped external CLI discovery and `models list` avoiding external overlays, but it still has no documented config or environment opt-out for external CLI auth-profile discovery.

## What This Changes

Request a first-class way to disable or scope external CLI auth-profile discovery, such as an `auth.externalCliSync` config field or environment override.

## Best Possible Solution

Add a documented, default-on external CLI auth discovery policy under `auth` config, with optional environment override support, and thread it through external auth overlay/scope resolution without changing runtime-only credential persistence or existing scoped discovery defaults.

## Reproduction Assessment

Yes for the remaining feature request: current main source, schema, docs, and tests show no user-facing config or environment opt-out. The exact old double `models list` info-log reproduction is no longer high confidence because current main avoids external overlays in that command and moved the bootstrap message to debug.

## Solution Assessment

No: the current scoped discovery is a useful partial fix, but it is not the requested first-class opt-out. The maintainable path is a narrow config/env gate layered onto the existing external CLI scope and overlay path.
