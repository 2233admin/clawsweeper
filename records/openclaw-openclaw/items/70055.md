## Summary

Keep open. Current main partially reduces the reported noise by scoping external CLI credential discovery and keeping `models list` off external auth overlays, but it still lacks the requested user-facing config or environment opt-out for external CLI auth-profile discovery.

## What This Changes

Request a default-on config, environment, or logging surface that lets users disable external CLI auth-profile sync/discovery without patching installed OpenClaw files.

## Best Possible Solution

Expose a documented, default-on external CLI auth discovery policy under `auth`, apply it consistently where external auth overlays are resolved, and preserve the current scoped-discovery and runtime-only persistence behavior.

## Reproduction Assessment

Yes for the remaining feature request: current-main source, schema, and docs show no config or environment opt-out. The original `models list` duplicate-info-log repro is no longer the best reproduction because that command now avoids external auth overlays.

## Solution Assessment

Yes, a narrow config/env gate threaded through the existing external CLI overlay path is the maintainable solution. A log-level downgrade alone would reduce noise but would not let users disable external credential discovery.
