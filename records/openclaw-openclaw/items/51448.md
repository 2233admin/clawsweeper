## Summary

Keep this PR open. Current main still does not forward configured `tool_choice`/`toolChoice` from agent/model extra params into the embedded runner's stream options, while the downstream OpenAI transports do honor `options.toolChoice` once it is present. The PR is directionally useful, but it needs to be rebased and adjusted for the alias-precedence review comment before it is merge-ready. Security pass found no CI, dependency, lockfile, release, package-script, or generated/vendor changes in the PR file list.

## Best Possible Solution

Keep the PR open and update it against current main. The best fix is to canonicalize `tool_choice`/`toolChoice` with the same precedence model used for other aliased params, preserve runtime overrides over inherited config, forward the resolved value into `streamParams.toolChoice`, and add regression coverage for both configured forwarding and cross-alias override precedence. The PR's file scope is appropriate and does not raise a supply-chain/security concern.
