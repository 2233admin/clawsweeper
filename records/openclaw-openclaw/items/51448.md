## Summary

Keep this PR open. Current main still does not forward configured `tool_choice`/`toolChoice` from agent model params into embedded runner stream options, although the downstream OpenAI transports already honor a direct `toolChoice` option.

## Best Possible Solution

Keep this PR open and ask for an update against current main. The best fix belongs in `src/agents/pi-embedded-runner/extra-params.ts`: resolve `tool_choice`/`toolChoice` with source-ordered alias precedence across inherited config and overrides, validate or warn on invalid values, forward the resolved value into stream options, and add current-layout regression coverage for configured forwarding plus cross-alias override precedence.
