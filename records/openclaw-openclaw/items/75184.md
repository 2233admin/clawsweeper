## Summary

Keep open. Current main still wires `agents.defaults.skipBootstrap` to workspace file creation, not runtime prompt injection, and an open PR is already linked as the active fix candidate.

## What This Changes

The issue asks OpenClaw to either make `agents.defaults.skipBootstrap: true` suppress workspace bootstrap prompt injection or clarify/remove that config surface so users do not treat it as a prompt-size control.

## Best Possible Solution

Resolve this through the already-linked PR or a maintainer-approved schema/docs clarification: either make `skipBootstrap` suppress runtime injection with tests and generated schema help, or keep creation-only semantics and explicitly point prompt-size users to `contextInjection: "never"` and the bootstrap budget knobs.

## Reproduction Assessment

Yes. The issue has deterministic CLI steps, and current-main source inspection reproduces the root cause statically: `skipBootstrap` is read before workspace creation but not in the bootstrap-file injection resolver.

## Solution Assessment

Unclear. The linked PR's direct runtime semantic change may satisfy the reporter, but current docs define `skipBootstrap` as creation-only, so maintainers should choose between that semantic change and a narrower schema/help clarification.
