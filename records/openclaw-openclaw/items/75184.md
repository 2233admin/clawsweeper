## Summary

Keep open. Current main still does not make `agents.defaults.skipBootstrap` suppress workspace bootstrap injection, and the generated config schema still exposes it as a bare boolean with no title or description. The runtime and docs show the original behavior is creation-only, so the narrow actionable fix is to make the schema/help surface explicit and point prompt-size users to the supported injection controls, not to auto-close this as implemented.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap: true` to either skip workspace bootstrap file injection for normal agent runs or be clearly scoped/renamed/documented so it is not mistaken for a prompt-size control.

## Best Possible Solution

Keep the original creation-only semantics for `skipBootstrap`, add generated schema label/help that says so explicitly, and route users who want no prompt injection to `agents.defaults.contextInjection: "never"` or a future tiered-bootstrap design. A runtime semantic change to make `skipBootstrap` skip injection would risk breaking pre-seeded workspaces that use the flag only to avoid file creation.

## Reproduction Assessment

Yes. The issue provides deterministic CLI steps, and current source inspection shows `skipBootstrap` is read only for workspace file creation while the prompt injection path never reads it.

## Solution Assessment

No. Treating `skipBootstrap` as the injection control is not the best fix because its original PR and current docs define creation-only behavior; the safer solution is schema/help clarification plus the existing `contextInjection: "never"` pointer.
