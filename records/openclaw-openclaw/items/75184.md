## Summary

Keep open. Current main still does not make `agents.defaults.skipBootstrap` suppress runtime workspace bootstrap injection, and the issue is paired with open closing PR #75217, so the issue should remain open until that PR is reviewed, merged, or rejected.

## What This Changes

Make `agents.defaults.skipBootstrap` either suppress runtime workspace bootstrap injection or clarify/remove/rename the flag so prompt-size controls are unambiguous.

## Best Possible Solution

The config surface should end with one tested contract: either `skipBootstrap` empties runtime bootstrap/context-file injection and the system prompt report reflects that, or the creation-only flag is clearly labeled in generated schema/help and users are directed to `contextInjection: "never"` plus bootstrap budgets for prompt-size control.

## Reproduction Assessment

Yes. The issue gives deterministic CLI steps, and current-main source inspection reproduces the root cause statically: existing workspace bootstrap files remain eligible for injection because the runtime resolver does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a plausible narrow implementation of the reporter's requested behavior, while a schema/help clarification would be safer if `skipBootstrap` is meant to remain creation-only.
