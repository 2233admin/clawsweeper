## Summary

Keep open. Current main still treats `agents.defaults.skipBootstrap` as a workspace-file creation flag, not as a runtime injection skip, and the issue is paired with open closing PR #75217 so it should remain open until that PR is reviewed, merged, or rejected.

## What This Changes

The issue requests that `agents.defaults.skipBootstrap` either suppress runtime workspace bootstrap injection or be renamed/documented to reflect a creation-only scope.

## Best Possible Solution

End with one tested config contract: either `skipBootstrap` makes runtime bootstrap resolution return no injected workspace files and the system prompt report reflects that, or the generated schema/help clearly label it as creation-only and direct prompt-size control to `contextInjection: "never"` plus bootstrap budgets.

## Reproduction Assessment

Yes. The issue gives deterministic CLI steps, and current-main source inspection reproduces the root cause statically: existing workspace bootstrap files remain eligible for injection because the runtime resolver does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended contract. PR #75217 is a plausible narrow implementation of the reporter's requested behavior, while a schema/help clarification is safer if `skipBootstrap` is meant to remain creation-only.
