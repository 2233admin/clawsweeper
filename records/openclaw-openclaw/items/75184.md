## Summary

Keep open. Current main still leaves the reported behavior unresolved, and an open closing PR is already paired with this issue, so the right cleanup path is to review and either land or reject that PR rather than close the issue or queue duplicate repair work.

## What This Changes

The issue reports that `agents.defaults.skipBootstrap: true` does not suppress runtime workspace bootstrap file injection for normal agent runs.

## Best Possible Solution

The config surface should have one unambiguous, tested contract across runtime behavior, schema/help, docs, and `systemPromptReport`: either `skipBootstrap` suppresses runtime workspace bootstrap injection, or it is clearly documented and labeled as creation-only with prompt-size users pointed to the runtime injection controls.

## Reproduction Assessment

Yes. The issue provides deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction: existing workspace bootstrap files remain eligible for injection because the resolver path does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers review the paired PR and decide the intended contract. PR #75217 is a plausible narrow implementation of the reporter's expected behavior; the safer compatibility path would be schema/help clarification if `skipBootstrap` is intended to stay creation-only.
