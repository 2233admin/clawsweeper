## Summary

Keep open. Current main fixes the narrower ACP-only `streamTo`/`resumeSessionId` subagent failure, but it does not implement the requested per-agent tool schema parameter filtering; the config schema has only whole-tool allow/deny policy, and the provider handoff still forwards each tool's full parameter schema.

## What This Changes

Request to let per-agent config hide or allowlist individual tool schema parameters, especially for `sessions_spawn` in dispatcher/subagent workflows.

## Best Possible Solution

Add a backwards-compatible, documented per-agent tool schema visibility contract that prunes `parameters.properties` before provider handoff, preserves required fields, updates generated config schema/help/docs, and adds focused tests proving `sessions_spawn` can be exposed with only the intended parameters for a selected agent.

## Reproduction Assessment

Not applicable as a feature request rather than a runtime bug. Source and docs inspection provide high-confidence proof that the requested config surface and schema-pruning behavior are absent on current main.

## Solution Assessment

Unclear until maintainers choose the public config shape. The direction is plausible and narrower than broad provider-specific prompt work, but the implementation should be framed as schema slimming/model guidance and not as an access-control boundary.
