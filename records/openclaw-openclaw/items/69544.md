## Summary

Keep open. Current main still derives fallback mention regexes from `identity.emoji`, Matrix still treats those shared regex matches as mentions, and current schema/docs expose only full `mentionPatterns` overrides rather than an emoji-only opt-out. The adjacent scoped mention-policy PR #70864 is open but the provided discussion says it does not cover this fallback-control gap, while the focused fix PR #74211 was closed unmerged.

## What This Changes

The issue requests a configuration switch to control whether Matrix/shared group-chat mention detection treats an agent identity emoji as a mention trigger.

## Best Possible Solution

Add a backward-compatible shared group-chat setting with global and per-agent precedence that defaults to current behavior but can exclude only `identity.emoji` from identity-derived fallback regexes. Native Matrix mentions, name-derived fallback matching, explicit `groupChat.mentionPatterns`, schema/types/help metadata, docs, and targeted tests should be updated together.

## Reproduction Assessment

Yes. Configure an agent with `identity.emoji` and no explicit `groupChat.mentionPatterns`; `buildMentionRegexes` will include the escaped emoji, and Matrix `resolveMentions` will mark a room message containing that emoji as mentioned through the shared regex fallback path.

## Solution Assessment

No for the Matrix-only switch as proposed in the title; the behavior is shared before Matrix consumes it. The better fix is a shared derived-mention fallback control with clear global/per-agent precedence, coordinated with but not hidden inside #70864 unless maintainers choose that path.
