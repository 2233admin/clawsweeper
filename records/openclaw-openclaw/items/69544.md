## Summary

Keep open. Current main still derives mention regexes from both `identity.name` and `identity.emoji`, Matrix uses those shared regexes for room mention gating, and there is no Matrix or shared per-agent emoji-specific opt-out. The overlapping scoped mention-policy PR #70864 is still open/unmerged and broader than this emoji-only Matrix request, so this needs maintainer follow-up rather than cleanup closure.

## Best Possible Solution

Keep this issue open. The best path is for maintainers to decide whether #70864 should be adapted to explicitly cover identity-derived emoji mention control, or whether a focused follow-up should add a backward-compatible opt-out that leaves native Matrix mentions and name-derived mention behavior intact while excluding `identity.emoji` from derived regexes when disabled. Update the config schema/types, generated config help, Matrix/group-chat docs, and targeted mention-gating tests with whichever policy is chosen.
