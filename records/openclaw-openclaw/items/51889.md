## Summary

Keep this PR open. Current main does not implement the proposed `/restore` chat command or the archive-list/restore helper; the only shipped restore path is for compaction checkpoints, not `/new`/`/reset` archived transcripts. The PR is still a plausible implementation candidate, but it needs maintainer review of the session-key isolation and restore-path concerns raised in review before landing.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate for restoring `/new` and `/reset` archived transcripts. The best product path is a first-party session restore flow that is sessionKey-safe, restores into the active session entry's resolved transcript path, preserves expected reset semantics/hooks, and includes focused tests for multi-channel isolation, topic/thread sessions, legacy archives, corrupt headers, and restore selection races. It can also be cross-checked against the related open session-history recovery requests before landing or being replaced by a newer canonical PR.
