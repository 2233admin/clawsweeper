## Summary

Keep this issue open. Current main has partial coverage for prompt-token snapshots, transcript usage metadata, and preflight compaction estimates, but it still does not provide the requested shared fallback that estimates context from raw session messages when provider usage is missing or all-zero, marks it as estimated, and renders it consistently in `/status`, `/usage`, session rows, and context notices.

## What This Changes

Request local transcript-based context token estimation when model providers return missing or all-zero usage, with status/usage visibility and compaction threshold support.

## Best Possible Solution

Add a shared estimated context snapshot contract that derives prompt/context tokens from stored session messages only when provider/prompt snapshots are absent, carries an explicit estimated/source flag, preserves fresh provider snapshots as authoritative, renders the estimate clearly in `/status`, `/usage`, session rows, and context notices, and lets compaction thresholds consume the estimate with conservative headroom.
