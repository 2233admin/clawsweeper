## Summary

Keep this PR open. Current main still only repairs one trailing orphaned user leaf through the existing merge strategy, while this PR targets the broader multi-leaf retry/image recovery problem. The proposed patch is useful but not merge-ready because recovered/current images are deduped before the normal image sanitization pipeline, so one logical retry image can still be attached twice after sanitization.

## What This Changes

The PR adds embedded-runner orphaned user-turn recovery that walks trailing user leaves, carries recovered text and image payloads into the next prompt, dedupes retry echoes, and adds focused helper tests.

## Best Possible Solution

Extend the current message-merge strategy or an adjacent helper on main to recover all trailing orphaned user leaves, preserve repeated real turns, dedupe only true newest retry echoes against effective and raw prompts, carry recovered images in chronological order, and canonicalize image dedupe after sanitization.

- **[P2] Deduplicate recovered images after sanitization:** `src/agents/pi-embedded-runner/run/attempt.ts:363`
  - body: `mergeRecoveredPromptImages` dedupes by raw `mimeType` and `data` before the prompt image pipeline sanitizes/resizes images.
