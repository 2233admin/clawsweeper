## Summary

Keep this PR open. Current main has a later, narrower orphaned-user repair path that carries one active-turn user leaf into the next prompt, but it does not implement this PR’s full chain-walk recovery for multiple trailing orphaned turns or the repeated-turn/image retry edge cases described in the PR discussion.

## Best Possible Solution

Keep the PR open, but refresh it against current main by extending the existing orphaned-user repair seam to walk all trailing user leaves, remove all repaired leaves, preserve real repeated identical turns, dedupe only true prompt echoes against both effective and raw prompts, and carry recovered image payloads in chronological order with post-sanitization dedupe. Add focused tests in `src/agents/pi-embedded-runner/run/attempt.test.ts` and the runtime contract tests for multi-leaf, repeated-text, hook-prepended raw prompt, and image ordering/dedupe cases.
