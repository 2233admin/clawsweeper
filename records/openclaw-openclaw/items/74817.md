## What This Changes

The issue requests Feishu handling for broadcast `@所有人` mentions and a built-in way to prevent multi-bot group echo loops when `requireMention` is disabled.

## Best Possible Solution

Split the remaining work by Feishu semantics: keep #75001 as the canonical `@all` handling discussion, and have maintainers decide whether standard OpenClaw should add conservative app-sender filtering, document the platform limitation, or support a relay-style Feishu feature. If filtering is chosen, carry `sender_type` into `FeishuMessageContext`, add an explicit account/group policy with a compatibility-preserving default, and cover schema, dispatch, docs, and tests.
