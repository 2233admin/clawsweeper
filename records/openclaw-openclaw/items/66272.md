## Summary

Keep open. Current main still emits and tests the reported `gateway.trusted_proxies_missing` warning, while this PR is stale against the current split gateway-audit collector and its raw `bind !== "loopback"` predicate is too coarse for effective bind-host and same-host reverse-proxy cases.

## What This Changes

This PR changes the gateway security audit to suppress `gateway.trusted_proxies_missing` for literal loopback binds, warn for non-loopback binds, and update the loopback logging regression test.

## Best Possible Solution

Revise this PR against `src/security/audit-gateway-config.ts` so `gateway.trusted_proxies_missing` is based on effective gateway exposure and explicit reverse-proxy context, with regression coverage for loopback, LAN, custom loopback, custom exposed, auto/container behavior, and same-host proxy configurations.
