## Summary

Keep this open. Current main has useful adjacent fixes for WhatsApp error payload hiding and Telegram delivery-error throttling, but billing and rate-limit provider failures still become normal final reply text before channel delivery, and the requested suppress/custom/owner-only policy is not implemented across WhatsApp and Telegram.

## What This Changes

Request a per-channel policy to suppress, customize, or owner-route LLM/provider/runtime error messages before WhatsApp or Telegram delivery.

## Best Possible Solution

Add a shared outbound provider/runtime error-reply policy before channel delivery. It should classify billing, rate-limit, auth, quota, and generic runtime failures; support per channel/account/group/topic/DM choices for normal friendly reply, silent suppression, custom text, and owner-only routing; keep full diagnostics in logs/control surfaces; and align WhatsApp and Telegram schema, docs, and tests.
