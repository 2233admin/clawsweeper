## Summary

Keep this PR open. Current main at 7a23b2d945cc2d2e5c452e4641213e71de8dcc66 still does not implement the Feishu `replyToMode` config surface or runtime behavior, while the PR is a focused Feishu-owned implementation attempt. It needs a current rebase plus tests/docs/metadata alignment before it is mergeable; it is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is a rebased Feishu-owned implementation that adds `replyToMode` to top-level, per-account, and per-group config; resolves precedence against `replyInThread`, topic sessions, `ctx.replyTargetMessageId`, and `suppressReplyTarget`; applies one reply-target policy across streaming start, text/card chunks, media sends, and typing behavior; updates manifest/generated metadata/docs; and adds regression coverage for `off`, `first`, and `all`.
