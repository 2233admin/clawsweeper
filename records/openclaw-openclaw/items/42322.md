## Summary

Keep this PR open. Current main still does not implement the requested Feishu `replyToMode` config surface or runtime behavior. Feishu schemas, manifest metadata, docs, dispatcher params, and tests still only cover fixed `replyInThread`/reply-target behavior, while Discord/Slack/Telegram have `replyToMode` plumbing elsewhere. The PR is still useful but needs a current rebase and complete Feishu-owned handling, especially for `first` mode across streaming/card/text/media paths.

## Best Possible Solution

Keep this PR open and ask for a rebased Feishu-owned implementation. The best path is to add `replyToMode` to strict Feishu top-level, per-account, and per-group config schemas; update plugin manifest, generated config metadata, and public Feishu docs; define clear precedence among group/account/top-level settings; and apply one reply-target policy across streaming start, chunked text/card sends, media sends, and typing behavior. Add focused tests for `off`, `first`, and `all`, including a streamed response with media so `first` mode only threads once.
