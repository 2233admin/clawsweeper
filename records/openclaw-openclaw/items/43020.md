## Summary

Keep this issue open. Current main still routes Mattermost partial replies through one draft preview stream, disables core block streaming for that run, and only resets local partial-text state at assistant/reasoning boundaries instead of flushing or rotating the Mattermost preview post. The related fix attempts #43041 and #46330 are closed and unmerged in the provided GitHub context.

## Best Possible Solution

Fix this in the Mattermost plugin inbound streaming path. Track whether a draft has streamed visible text, flush or preserve the current draft at assistant/tool boundaries, rotate to a fresh Mattermost post before the next streamed text segment, and skip final payloads already delivered through preview streaming. Add a focused Mattermost regression test for first partial text, tool/assistant boundary, second partial text, and final delivery without duplicates.
