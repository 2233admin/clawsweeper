## Summary

Keep this issue open. Current main still has the Mattermost behavior described by the report: the inbound path creates one draft preview stream for the whole run, disables core block streaming, and only clears local partial-text state at assistant/reasoning boundaries. It does not flush, seal, or rotate the Mattermost preview post between tool-call turns, and the prior fix PRs in the provided context are closed unmerged.

## Best Possible Solution

Fix this in the Mattermost plugin inbound streaming path. When a draft has visible text, flush or seal it at assistant/tool boundaries, rotate to a fresh Mattermost draft post before the next streamed assistant segment, and suppress final payloads already delivered through preview streaming. Add a focused Mattermost regression test for first partial text, tool boundary, second partial text, and final delivery without duplicate messages.
