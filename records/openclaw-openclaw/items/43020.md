## Summary

Keep this issue open. Current main still disables core block streaming for Mattermost draft-preview delivery, uses one Mattermost draft stream for the whole inbound run, and only clears local partial text at assistant boundaries instead of preserving/rotating the preview message. The two related fix PRs in the provided GitHub context, #43041 and #46330, are both closed and unmerged.

## Best Possible Solution

Fix this in the Mattermost plugin inbound streaming path. On assistant/tool boundaries, flush or preserve the current draft preview, rotate to a fresh Mattermost post before the next streamed text segment, and suppress duplicate final replies for segments already delivered through preview streaming. Add a focused Mattermost regression test for partial text, a tool/assistant boundary, second partial text, and final delivery.
