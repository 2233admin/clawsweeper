## Summary

Current main still has the reported zero `cacheRead` and `cacheWrite` pricing for the bundled Volcengine/Doubao and BytePlus catalogs, and the runtime cost estimator uses those fields directly. This should remain open because PR #55913 is already the open, unmerged implementation path that references this issue with closing syntax.

## What This Changes

The issue requests vendor-backed nonzero context-cache pricing for Doubao/Volcengine and BytePlus model catalogs so cached token usage is not reported as free.

## Best Possible Solution

Review, retarget, or replace PR #55913 as the implementation path. The merged fix should update the bundled Volcengine and BytePlus plugin model catalogs with vendor-backed USD-per-million-token cache prices, keep compatibility constants aligned, and add focused regression coverage so cache pricing cannot silently return to zero.
