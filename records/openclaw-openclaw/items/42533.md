## Summary

Keep this issue open. Current main still matches the reported macOS speech-input failure mode: PTT, Talk Mode, Voice Wake runtime, and the Voice Wake tester all tap the hardware input format and append captured buffers directly to `SFSpeechAudioBufferRecognitionRequest`, with no shared downmix/conversion layer for multi-channel input devices.

## Best Possible Solution

Keep this issue open and fix the macOS app speech capture pipeline. Add a shared macOS voice-input helper that converts multi-channel hardware `AVAudioPCMBuffer` input into a Speech-compatible mono format before appending to `SFSpeechAudioBufferRecognitionRequest`, wire it through `VoicePushToTalk`, `TalkModeRuntime`, `VoiceWakeRuntime`, and `VoiceWakeTester`, then add focused conversion/downmix coverage plus manual validation on a >2-channel macOS input device.
