## Summary

Keep open. Current main still lacks the requested guardrail: configured provider model inputs accept and preserve `audio`/`video`, while the documented and runtime-facing model-input behavior remains centered on text/image plus separate manifest/document handling. Tightening this is a public config/schema/type contract decision, not conservative cleanup.

## What This Changes

Add validation guardrails, tests, CI coverage, and docs so unsupported `models.providers.*.models[*].input` modalities are rejected before deployment.

## Best Possible Solution

Define one configured-model input contract and enforce it consistently across source schema, exported types, generated schema, catalog normalization, runtime capability checks, tests, and public docs: either reject unsupported values such as `audio`/`video`, or implement and document their runtime meaning.
