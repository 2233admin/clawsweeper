## Summary

Keep open. Current main still accepts and preserves `audio` and `video` for configured provider model inputs, while the operator docs and representative runtime model-turn paths only define text/image behavior; tightening this now would change a public config contract and an explicit preservation test, so it needs maintainer product/API direction.

## What This Changes

The issue requests stricter validation for `models.providers.*.models[*].input`, tests for valid and invalid modality configs, early CI failure, and updated contributor/operator docs.

## Best Possible Solution

Define one explicit configured-model input contract and enforce it consistently across schema, exported types, generated schema, catalog normalization, tests, and docs: either reject unsupported values such as `audio`/`video`, or implement and document their runtime meaning so they are no longer silently accepted metadata.
