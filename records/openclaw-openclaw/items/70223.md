## Summary

Keep open. Current main still does not provide a default WorkingDirectory for the packaged gateway launchd install path, and configured relative agentDir values still resolve against process cwd rather than the config file/state directory. The provided issue comments correctly identify the current code path; the related items do not provide a canonical shipped fix for this specific ENOENT regression.

## Best Possible Solution

Keep the issue open and fix it in core. The narrow fix should set a deterministic workingDirectory for packaged gateway launchd installs before rendering the plist, likely the user home, state dir, or config directory. A defensive follow-up should make relative agents.list[].agentDir resolve relative to the loaded openclaw.json directory or reject/repair legacy relative values through doctor with tests for launchd cwd=/ behavior.
