## Summary

Keep open. Current main has relevant lower-level sub-agent, session-history, context-budget, compaction, and context-engine seams, but it still does not answer the product/design question here: how OpenClaw should preserve useful cross-agent conversational context while bounding token growth and budget risk at larger multi-agent scales.

## What This Changes

The issue asks for maintainer/community design input on reducing token-cost growth in conversational multi-agent coordination without losing emergent cross-agent context sharing.

## Best Possible Solution

Keep the design discussion open until maintainers choose the boundary. A good end state would document whether OpenClaw should provide a generic coordination/cost-governance primitive, extend context-engine or sub-agent hooks for tiered delivery and budget reservation, or explicitly leave these policies to plugins and scenario harnesses.
