## Summary

Keep open. Current main has isolated sub-agent spawning, bounded session history, usage reporting, context-engine hooks, and fan-out/depth controls, but it does not answer the design question raised here: how OpenClaw should preserve useful conversational cross-agent context while bounding token growth and budget risk at larger multi-agent scales.

## What This Changes

The issue requests maintainer and community design input on reducing token-cost growth in conversational multi-agent coordination without losing useful cross-agent context sharing.

## Best Possible Solution

Keep the discussion open until maintainers decide the boundary. A good end state would document whether OpenClaw should provide a generic conversational coordination and cost-governance primitive, extend context-engine/sub-agent hooks for tiered delivery and budget reservation, or explicitly leave these policies to external plugins and scenario harnesses.

## Reproduction Assessment

No high-confidence repo-local reproduction exists. The issue gives a plausible manual setup and later observed run data, while current source/docs confirm that repeated full-history coordination would grow context and cost structurally, but there are no runnable scripts or token traces in the repository.

## Solution Assessment

Not applicable as a patch because no implementation is proposed. Existing primitives are useful guardrails, but the narrowest maintainable next step is maintainer architecture/product direction before code changes.
