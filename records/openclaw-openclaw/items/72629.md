## Summary

Keep open. Current main has adjacent sub-agent isolation, usage reporting, context-engine, and plugin-hook primitives, but it still does not settle the design question raised here: preserving useful emergent cross-agent context while bounding token growth and budget risk in conversational multi-agent coordination.

## What This Changes

The issue asks maintainers to decide how conversational multi-agent coordination should preserve emergent cross-agent context while reducing quadratic token-cost growth.

## Best Possible Solution

Keep the design thread open until maintainers choose the boundary. A good end state would document whether OpenClaw should provide a generic coordination/cost-governance primitive, extend context-engine or sub-agent hooks for tiered delivery and budget reservation, or explicitly leave these policies to plugins and scenario harnesses.

No, not as a high-confidence repo-local reproduction. The issue gives a plausible manual setup and later cost observations, but no runnable scripts or token traces; current source and docs do confirm the structural risk when an orchestrator repeatedly replays full history.
