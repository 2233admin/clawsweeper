## Summary

Keep open. The discussion is a current design question with concrete cost observations, and current main exposes lower-level sub-agent, session, usage, and context-engine primitives but does not settle how OpenClaw should preserve cross-agent conversational context while bounding token and budget growth at larger multi-agent scales.

## What This Changes

The issue asks for maintainer/community design input on reducing token-cost growth in conversational multi-agent coordination without losing emergent cross-agent context sharing.

## Best Possible Solution

Keep the design discussion open until maintainers decide the product boundary. A good end state would document whether OpenClaw should provide a generic coordination/cost-governance primitive, extend context-engine or sub-agent hooks for tiered delivery and budget reservation, or explicitly leave these policies to plugins and scenario harnesses.

No high-confidence repo-local reproduction exists. The issue gives a plausible manual setup and later cost observations, but not runnable scripts or token traces; current source and docs do confirm that repeated full-history coordination would structurally increase context and cost.
