## Summary

Keep this open. Current main still has partial safety scanners, but not the shared prompt-bearing Markdown validation contract requested here. Workspace bootstrap files and extra bootstrap files are read and injected as content with only boundary/size checks; skills are discovered with path/frontmatter guards and then exposed for model-driven `SKILL.md` reads; native ClawHub skill installs still copy the downloaded skill tree without invoking a Markdown policy validator. The existing Skill Workshop scanner is useful, but scoped to generated workspace skill proposals and has its own narrow rule list rather than a reusable contract across all prompt-bearing Markdown surfaces.

## Best Possible Solution

Keep this issue open until maintainers define a shared prompt-bearing Markdown validation contract. A good implementation should reuse or align with the existing Skill Workshop scanner, make the rule/severity/override policy explicit, and apply it consistently to workspace bootstrap ingestion, extra bootstrap files, model-visible SKILL.md reads/discovery, native ClawHub skill install/update handling, and ClawHub publish-side coordination, with focused tests and docs for the chosen policy.
