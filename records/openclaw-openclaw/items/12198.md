## Summary

Keep this open. Current main still has path, size, symlink, code-scan, ClawHub install hardening, and optional Skill Workshop scanning, but it does not provide a shared prompt-bearing Markdown validation contract for workspace bootstrap files, extra bootstrap files, normal SKILL.md reads, or native ClawHub skill install/update. The recent maintainer/Codex comments reached the same conclusion, and current main still matches that gap.

## What This Changes

Markdown files used as prompt inputs should be linted for prompt-injection or security-dangerous patterns, including at least HTML comments, across skill and bootstrap ingestion and ClawHub publishing/install paths.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract, then apply it consistently to workspace bootstrap ingestion, extra bootstrap files, normal SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior. The contract should document severity, override, warning/tagging, false-positive recovery, and compatibility handling for existing generated skills.
