## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract across workspace bootstrap files, extra bootstrap files, model-readable `SKILL.md` paths, and native ClawHub skill installs; recent maintainer and ClawSweeper comments reached the same conclusion, and the current source still matches that gap.

## What This Changes

The issue requests validation for Markdown files that are automatically exposed to agent prompts, especially skill definitions and workspace prompt files, to detect prompt-injection patterns such as HTML comments.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract, then apply it consistently before workspace bootstrap files, extra bootstrap files, normal `SKILL.md` reads, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior expose content to the model. The contract should document severities, overrides, warnings or quarantine, ClawHub tagging, and focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place an HTML comment or prompt-injection marker in `AGENTS.md`, `BOOTSTRAP.md`, an extra bootstrap file, a local `SKILL.md`, or a ClawHub-installed skill; current loaders read and expose that Markdown without a shared validator.

## Solution Assessment

Unclear. A shared validation contract is the right direction, but an HTML-comment-only rule would be under-scoped; maintainers should settle the reusable policy and ClawHub behavior before a narrow implementation PR.
