## Summary

Keep open. Current main still has path, size, symlink, frontmatter, ClawHub archive, and code-scanner protections, plus the optional Skill Workshop scanner, but it still lacks one shared validation policy for Markdown that becomes model-visible through workspace bootstrap files, extra bootstrap files, SKILL.md reads, and native ClawHub skill installs. The recent maintainer/Codex comments reached the same conclusion, and the current source still matches that gap.

## What This Changes

Request to add mandatory or configurable validation for prompt-bearing Markdown files, especially workspace bootstrap files and skills, to catch prompt-injection and security-risk patterns before model exposure or ClawHub publish/install flows.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently before workspace bootstrap files, extra bootstrap files, normal SKILL.md reads, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior expose content to the model, with documented severities, overrides, warnings or quarantine behavior, and focused tests.
