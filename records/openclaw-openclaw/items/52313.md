## Summary

Keep this open. Current main still respawns unmanaged gateway restarts by reusing `process.execArgv`, `process.argv.slice(1)`, and `process.execPath`; it has update health fallback hardening, but not the requested stable `openclaw.mjs` or pnpm-realpath rewrite. The related implementation PR #52319 is closed unmerged, so the central behavior remains missing.

## Best Possible Solution

Implement a narrow respawn entrypoint planner in `src/infra/process-respawn.ts`: detect pnpm versioned OpenClaw realpaths, choose a stable package-root or symlinked `openclaw.mjs` entrypoint when appropriate, preserve dev/source entrypoints, and use the planner from both `restartGatewayProcessWithFreshPid()` and `respawnGatewayProcessForUpdate()`. Add focused regression tests for pnpm realpaths, package installs, source/dev runs, Bun/Node behavior, and update fallback behavior; treat PR #52319 as prior art rather than a landed fix.

## Clownfish Work Prompt

Fix the gateway restart/update stable-entrypoint bug tracked by this issue. Current main respawns unmanaged gateways with `process.execPath` and `process.execArgv + process.argv.slice(1)`, which can reuse a pnpm versioned realpath that disappears after self-update. Implement a narrow respawn entrypoint planner in `src/infra/process-respawn.ts` and use it from both `restartGatewayProcessWithFreshPid()` and `respawnGatewayProcessForUpdate()`. Detect OpenClaw pnpm versioned realpaths under `node_modules/.pnpm/openclaw@*/node_modules/openclaw/...` and prefer a stable package-root or symlinked `openclaw.mjs` entrypoint when available; if the package root is known, prefer `<packageRoot>/openclaw.mjs`; preserve direct dev/source entrypoints such as `src/entry.ts` and avoid changing supervisor policy. Reuse or extract existing stable-entrypoint helpers from the service-install path only if the dependency direction stays clean; do not import daemon internals into infra if that violates local architecture. Treat PR #52319 as prior art, but do not copy its broad `scripts/run-node.mjs` and unrelated wrapper/runtime changes unless a focused test proves they are required. Add focused regression coverage in `src/infra/process-respawn.test.ts` for pnpm realpath rewrite, package-root wrapper selection, source/dev passthrough, update respawn, and Node/Bun-relevant behavior. Changelog only if maintainers consider this user-facing enough; otherwise no changelog for a narrow internal restart hardening PR.
