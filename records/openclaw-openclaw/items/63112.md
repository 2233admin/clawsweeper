## Summary

Keep this PR open. Current main still lacks the proposed cron CLI warning, the linked #63107 report remains open, and the PR is a focused implementation candidate; review found one low-priority edit lookup gap but no blocking functional or security issue.

## What This Changes

The PR adds a shared shell-command detector, non-blocking stderr warnings for main-session systemEvent cron add/edit payloads that look like shell commands, and focused CLI/shared tests.

## Best Possible Solution

Land a narrow cron CLI fix that warns before the gateway RPC for both add and edit, including existing-main edits where --session is omitted, with regression tests and no change to cron runtime execution semantics.
