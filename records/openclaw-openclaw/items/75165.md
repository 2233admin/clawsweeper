## Summary

Keep open. Current main does not implement this agents termination API, and the PR is still an open XL core-surface proposal with a concrete correctness bug in `ReplyPattern` after the force-push.

## What This Changes

The PR adds a composable `TerminationCondition` algebra, GSAR groundedness scoring helpers, an optional A2A loop termination hook, and related agent tests.

## Best Possible Solution

Keep the PR open for maintainer API review. If maintainers accept the direction, first make regex matching stateless, then keep the initial surface internal and narrow until OpenClaw has a reviewed scorer/evidence contract or a clear SDK/plugin extension point.

## Reproduction Assessment

Yes for the review finding: PR head calls `.test()` on a retained `RegExp`, and a Node probe reproduces alternating true/false results for global and sticky regexes on identical text. For the broader feature, reproduction is not applicable because current main does not expose this API.

## Solution Assessment

No for the current patch as-is: the termination algebra direction is plausible, but stateful regex handling makes the branch incorrect and the GSAR/core API surface still needs maintainer product review. The safer path is to fix `ReplyPattern` and decide whether GSAR belongs in core, SDK, or a plugin/skill surface before merge.
