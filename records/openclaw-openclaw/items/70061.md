## Summary

Current main still has the reported Active Memory classifier gap. Telegram DM topic sessions can still produce `agent:main:main:thread:{chatId}:{topicId}`, while `resolveChatType` only recognizes explicit/group/channel/direct tokens or an unthreaded 3-part main session key, so this threaded default-main DM shape still resolves to no chat type and is skipped before recall or transcript persistence.

## What This Changes

Fix Active Memory so Telegram DM topic session keys like `agent:main:main:thread:{chatId}:{topicId}` are classified as direct-message sessions instead of being silently skipped.

## Best Possible Solution

Fix the Active Memory session classifier in `extensions/active-memory/index.ts` so default-main Telegram DM topic sessions like `agent:main:main:thread:{chatId}:{topicId}` classify as `direct` when provider/channel context indicates an interactive direct-message channel. Prefer stripping the generic `:thread:<id>` suffix with the existing routing helper before applying the current rules, add focused regression coverage in `extensions/active-memory/index.test.ts`, and avoid changing Telegram session-key generation or core routing semantics.
