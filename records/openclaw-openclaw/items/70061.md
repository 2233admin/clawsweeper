## Summary

Keep this issue open. Current `origin/main` at `213f92a9ef0eb54972ef6cbb0e8b900adc8699b0` still has the reported Active Memory classifier gap: Telegram DM topic sessions can produce `agent:main:main:thread:{chatId}:{topicId}`, but `resolveChatType` only recognizes explicit/group/channel/direct tokens or unthreaded 3-part main sessions, so the threaded main DM key still resolves to no chat type and is skipped before recall or transcript persistence.

## Best Possible Solution

Fix the Active Memory session classifier in `extensions/active-memory/index.ts` so Telegram default-main DM topic sessions like `agent:main:main:thread:{chatId}:{topicId}` are classified as `direct` when the provider/channel context indicates an interactive direct-message style session. Prefer stripping the generic `:thread:<id>` suffix before applying the existing rules, add regression coverage in `extensions/active-memory/index.test.ts`, and avoid changing Telegram session-key generation or core routing semantics.
