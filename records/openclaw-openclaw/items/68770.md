Keep open. Current `main` still lacks Telegram media-send success logs, and the linked open PR is a relevant implementation candidate with closing syntax, so this issue should remain open until that PR is reviewed and merged, replaced, or closed.

## What This Changes

Request that the Telegram plugin emit success log lines for successful outbound media sends such as voice, audio, document, video, photo, and animation sends.

## Best Possible Solution

Keep this issue open until a fix lands that emits consistent media-kind success logs after Telegram returns a message id for both reply media delivery and direct outbound media sends, with focused regression assertions for the logged success cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current `main` has media send branches that resolve successful Telegram message ids without a corresponding success `runtime.log`, while text sends do log success.

## Solution Assessment

Yes, adding `runtime.log?.(...)` immediately after successful media API calls or message-id resolution is the narrow maintainable fix. The open PR is relevant, but maintainers should verify or expand it so the direct outbound media path is covered too.
