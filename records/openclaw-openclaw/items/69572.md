## Summary

Current main still implements Feishu reply typing as a persistent `Typing` message reaction, and the docs/tests continue to describe and assert reaction-based typing. The related older Feishu typing cleanup only improves the reaction lifecycle; it does not satisfy this newer request for native Feishu/Lark typing/status behavior.

## Best Possible Solution

Keep this open as a Feishu bundled-plugin behavior bug. A maintainer should confirm the official Feishu/Lark native typing or follow-up/status API, then update the Feishu-owned typing lifecycle in `extensions/feishu/src/typing.ts` and `extensions/feishu/src/reply-dispatcher.ts` to use native behavior when supported, while preserving config gating/backoff and updating tests plus `https://docs.openclaw.ai/channels/feishu`. If Feishu has no native bot typing API, document that limitation and stop presenting message reactions as native typing.
