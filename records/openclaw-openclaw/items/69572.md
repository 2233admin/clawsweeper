## Summary

Keep #69572 open. Current main still implements Feishu reply typing as a `Typing` message reaction, and docs/tests still describe and assert reaction-based typing. Earlier cleanup of #8722/#43084 confirms the reaction feature and keepalive behavior, but does not resolve this newer report asking for native Feishu typing/status behavior instead of reactions.

## Best Possible Solution

Keep the issue open as a Feishu bundled-plugin behavior bug. The likely fix belongs in `extensions/feishu/src/typing.ts` and `extensions/feishu/src/reply-dispatcher.ts`: replace message-reaction typing with the correct native Feishu/Lark typing or follow-up status API after verifying endpoint semantics, preserve `typingIndicator` gating and backoff/cleanup behavior, and update Feishu tests plus `https://docs.openclaw.ai/channels/feishu` so the documented behavior no longer says typing reactions.
