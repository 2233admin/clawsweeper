## Summary

Keep open. Current main still implements Feishu typing feedback with Feishu message and comment reactions, the dispatcher and tests still exercise that reaction helper path, and the latest release tag also contains reaction-based typing rather than a native typing/status replacement.

## What This Changes

The issue requests replacing Feishu typing feedback implemented through `im.messageReaction.create/delete` and the `Typing` emoji with a native Feishu/Lark typing/status API.

## Best Possible Solution

Keep this issue open as a bundled Feishu plugin behavior bug. A maintainer should confirm the official Feishu/Lark native typing/status API and required permissions, then update the Feishu-owned typing lifecycle in `extensions/feishu/src/typing.ts` and `extensions/feishu/src/reply-dispatcher.ts`, plus `extensions/feishu/src/comment-reaction.ts` if comment typing shares the user-facing setting. Tests and the Feishu channel docs should distinguish native typing from any reaction fallback. If Feishu has no bot-native typing API, document that limitation and stop presenting message reactions as native typing.
