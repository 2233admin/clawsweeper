## Summary

Keep this PR open. Current main still validates cron webhook URLs with only string trimming, URL parsing, and http(s) protocol checks, so private/internal literal hosts can still be stored in cron job delivery config. Runtime delivery remains SSRF-guarded, but this PR's pre-persistence defense-in-depth change is not implemented or superseded on main.

## Best Possible Solution

Review and land this focused PR, or reimplement the same small fix on main: call `isBlockedHostnameOrIp(parsed.hostname)` inside `normalizeHttpWebhookUrl`, add direct `src/cron/webhook-url.test.ts` coverage for reserved hostnames, private/special-use IPs, `0.0.0.0`, and IPv4-mapped IPv6 literals, and keep `fetchWithSsrFGuard` as the DNS-aware runtime guard.
