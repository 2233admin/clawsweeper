## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: aborted embedded assistant turns are not treated as clean error payloads, and non-empty partial assistant text can still flow through the payload builder instead of triggering the timeout-only fallback. The provided PR diff is a focused implementation candidate and also appears to address the Greptile reasoning-text concern with regression coverage.

## Best Possible Solution

Keep this PR open and refresh it, or apply the same focused fix directly on main: make `buildEmbeddedRunPayloads` treat aborted last-assistant turns as needing a clean error surface, suppress aborted answer and reasoning artifacts, keep the regression coverage for both visible partial text and reasoning text, and then run the targeted payload tests plus the normal changed gate.
