## Summary

Keep this issue open. Current main still has the reported 3-minute CLI resume no-output watchdog cap and the 180-second fresh watchdog floor, and the resolver/schema still require numeric watchdog values rather than supporting a null or 0 disable mode. The issue also has an open, unmerged implementation candidate, #41022, with closing syntax, so it should stay open until that PR is reviewed and merged or explicitly rejected.

## Best Possible Solution

Review #41022 against current main, require focused regression coverage for resolveCliNoOutputTimeoutMs and default CLI backend behavior, then land it if the 30-minute default policy is acceptable. If maintainers want the full disable mode from the issue, implement it deliberately across the config types, zod schema, resolver, supervisor/live-session contracts, and tests instead of treating the open PR as complete for that part.
