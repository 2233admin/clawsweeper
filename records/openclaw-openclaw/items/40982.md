## Summary

Keep this issue open. Current main still has the reported 180,000 ms resumed CLI watchdog cap and the fresh watchdog floor remains 180,000 ms; the resolver still returns a bounded numeric no-output timeout and the config schema still rejects null/0 disable-style values. There is also an open unmerged implementation candidate, #41022, with closing syntax for this issue, so maintainers should review or supersede that PR before closing the issue.

## Best Possible Solution

Keep the issue open and review #41022 against current main. If the 30-minute default policy is acceptable, land it with focused regression coverage for resolveCliNoOutputTimeoutMs and bundled CLI backend defaults; if maintainers want a disable mode, implement null/0 no-cap behavior deliberately across config types, zod schema, resolver, supervisor/live-session contracts, and tests.
