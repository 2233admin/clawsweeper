## Summary

Keep this issue open. Current main still has the reported 180,000 ms resume no-output watchdog cap and the 180,000 ms fresh watchdog floor, and the config/schema path still does not support null or 0 as a disable mode. There is also an open implementation candidate, #41022, with closing syntax for this issue, so the issue should stay open until that PR is reviewed and merged or rejected.

## Best Possible Solution

Review #41022 against current main, add or request focused regression coverage for resolveCliNoOutputTimeoutMs and default CLI backend behavior, then land it if the 30-minute default policy is acceptable. If maintainers want the full disable mode from the issue, implement it deliberately across the zod schema, CliBackendConfig types, resolver, supervisor/live-session contracts, and tests rather than treating the open PR as complete for that part.
