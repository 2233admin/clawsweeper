Keep open. Current main still preserves the reported 180-second resumed CLI no-output cap, the resolver still guarantees a positive watchdog timeout, and the latest release commit carries the same behavior; open PR #41022 remains the active closing-linked implementation path rather than a reason to close the issue before merge.

## What This Changes

Request to raise, remove, or make configurable the CLI no-output watchdog cap so long-running CLI-backed agent requests are not terminated after about three minutes of silence.

## Best Possible Solution

Land or revise #41022, or a narrow successor, so shared CLI watchdog defaults no longer terminate legitimate long-running CLI turns at 180 seconds; if disable/configuration semantics are accepted, make them explicit in schema, resolver behavior, and regression coverage while keeping the global timeout backstop intact.
