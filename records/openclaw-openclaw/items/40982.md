## Summary

Keep this issue open. Current main still contains the reported 180-second resume no-output watchdog cap, still does not allow `0` or `null` to disable the cap, and open PR #41022 explicitly references this issue as its implementation candidate.

## Best Possible Solution

Keep the issue open while maintainers review PR #41022 or replace it with a current-main patch. The likely code path is `src/agents/cli-watchdog-defaults.ts` for raised defaults, plus `src/agents/cli-runner/reliability.ts`, `src/config/zod-schema.core.ts`, generated schema/docs, and focused tests if maintainers want `0`/`null` disable semantics.
