## Summary

Keep this issue open. Current main still has the reported 180,000 ms resume no-output watchdog cap, the fresh watchdog floor is still 180,000 ms, and the schema/resolver do not support null/0 as a disable mode. An open implementation candidate, #41022, explicitly references this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or rejected.

## Best Possible Solution

Review #41022 against current main, add or request focused regression coverage for resolveCliNoOutputTimeoutMs/default backend behavior, then either land it to satisfy the central 3-minute cap problem or close it with a maintainer-approved alternative. If maintainers want a true disable mode, implement it deliberately in the watchdog schema, resolver, and process-supervisor contract rather than treating the open PR as complete for that part.
