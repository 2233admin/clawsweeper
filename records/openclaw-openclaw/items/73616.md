## Summary

Keep this PR open. Current main still has the QQBot slash-command auth, pre-dispatch registration, clear-storage path, and file-delivery gaps this PR targets. The PR is a plausible implementation candidate, but its current head still has unresolved review findings around `/bot-version`, `dm` private-chat semantics, and missing regression coverage.

## Best Possible Solution

Keep this PR open as the broader implementation candidate. Update it to remove the unintended `/bot-version` private-chat restriction unless maintainers explicitly want that change, decide and test whether `dm` counts as private chat for `c2cOnly`, add focused tests for auth resolution, registry matching, private/group gating, clear-storage directory resolution, and file-delivery options, then review it as the canonical QQBot fix and coordinate the narrower #71158 afterward.
