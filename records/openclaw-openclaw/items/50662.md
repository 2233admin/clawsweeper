## Summary

Keep open. Current main does not implement the PR’s combined behavior: restart health still builds probe auth only from the calling process environment, and the embedded-runner failover policy still prefers auth-profile rotation before model fallback for `rate_limit` when a fallback model is configured. The PR is therefore not obsolete or implemented on main, though it would need a rebase/refactor because current main has moved the failover decision logic into `src/agents/pi-embedded-runner/run/failover-policy.ts` and `run/assistant-failover.ts`.

## Best Possible Solution

Keep the PR open for maintainer review. If this behavior is still desired, rebase it onto current main, implement the rate-limit decision in `src/agents/pi-embedded-runner/run/failover-policy.ts` and `run/assistant-failover.ts`, update the existing tests that currently assert rotation-first behavior, and either wire restart-health through the existing config-aware probe auth resolver or explicitly document why the current auth-close reachability allowlist is sufficient.
