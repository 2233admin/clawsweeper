## Summary

Keep open. Current main still rejects top-level Telegram ACP bindings whose peer id is a plain positive direct-chat id, and public docs still describe Telegram persistent ACP bindings as forum-topic-only. The PR targets a real remaining gap, but it needs maintainer review and current-main cleanup before landing.

## What This Changes

This PR changes Telegram’s configured ACP binding adapter to accept plain positive direct-chat ids, adds focused adapter tests for direct chats/topics/negative groups, and updates ACP binding documentation.

## Best Possible Solution

Land a current-main version of the Telegram plugin change that compiles positive direct-chat ids, matches inbound DM conversation refs, keeps bare negative group ids rejected, updates all public Telegram ACP binding docs, and adds both adapter-level and full DM route coverage while preserving the existing DM access gate.
