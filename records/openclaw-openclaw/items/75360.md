## Summary

Keep open. This PR is protected by the maintainer label and the latency direction is plausible, but the diff introduces concrete tool-contract regressions: web_search loses provider-specific schemas, and media/PDF tools become visible even when no usable backing model/provider is configured.

## What This Changes

The PR caches targeted provider runtime hook resolution, defers media/PDF/web provider resolution until tool execution, and makes typing indicators asynchronous to reduce Telegram reply cold-start latency.

## Best Possible Solution

Keep the cold-start optimization, but preserve the existing tool contracts: avoid broad provider scans using lightweight/static descriptors, keep supported web_search parameters visible, and either keep unavailable media/PDF tools hidden or make an intentional docs/tests/product change with maintainer sign-off.

## Reproduction Assessment

Partially. The Telegram latency improvement itself was not reproduced in this read-only review, but the review findings are reproducible by comparing the PR diff against current docs, tests, and provider schemas on main.

## Solution Assessment

No. The targeted runtime lookup and async typing changes may be the right direction, but replacing provider schemas with a two-field web_search schema and exposing unavailable media tools is not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve provider web_search schemas:** `src/agents/tools/web-search.ts:63`
  - body: The new generic `WebSearchSchema` only advertises `query` and `count`, but existing providers expose useful parameters such as Brave's country/language/date filters and Perplexity's domain/token controls. `runWebSearch` still passes those arguments through at execution, so this change makes supported functionality invisible to the model and breaks runtime provider custom schemas. Please use a lightweight superset/static artifact instead of dropping provider parameters.
  - confidence: 0.91
- **[P2] Keep unavailable media tools hidden:** `src/agents/tools/pdf-tool.ts:303-306`
  - body: Moving PDF model resolution into `execute` means `createPdfTool` now returns a tool whenever `agentDir` exists, even when no PDF-capable model/auth can be resolved. Current docs say the `pdf` tool is not exposed in that state, and the same pattern affects the media tool availability contract. Please preserve the hidden-unavailable behavior without reintroducing broad cold-start scans.
  - confidence: 0.86
