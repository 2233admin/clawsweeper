## Summary

Keep this PR open. Current `main` already preserves default-enabled Feishu streaming and streams auto-mode card/markdown replies, but it still does not implement the PR's boolean-or-object `streaming` config, config-driven `throttleMs`, or config-driven card title/header behavior; the PR head also still risks regressing omitted `streaming` from enabled to disabled.

## Best Possible Solution

Keep the PR open for focused rework or replacement. Add a typed Feishu streaming config resolver that accepts boolean and object forms while preserving `undefined` as default-enabled and `false` / `{ enabled: false }` as disabled; wire bounded `throttleMs` and optional validated title/header behavior through `FeishuStreamingSession`; update manifest/docs/config metadata; and add regression tests for default, disabled, object config, custom throttle/title, and auto-mode card/plain-text routing.
