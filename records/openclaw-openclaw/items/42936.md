## Summary

Keep this PR open. Current main already streams Feishu auto-mode markdown/card replies and preserves boolean/default streaming behavior, but it still does not implement object-form Feishu `streaming` config, config-driven `throttleMs`, or config-driven `title`. The PR head is also not ready to close as implemented because its predicate disables streaming when the setting is omitted, which regresses current main's default-enabled behavior.

## Best Possible Solution

Keep this PR open for focused rework or replacement. The best path is to add a typed Feishu streaming config resolver that accepts boolean and object forms while preserving `undefined` as default-enabled and `false` / `{ enabled: false }` as disabled, wire bounded `throttleMs` and optional title/header behavior on top of the existing identity-derived header/note behavior, update manifest/docs/config metadata, and add regression tests for default, disabled, object config, custom throttle/title, and auto-mode card/plain-text routing.
