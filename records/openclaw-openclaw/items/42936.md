## Summary

Keep this PR open. Current main already streams Feishu auto-mode markdown/card replies and preserves boolean/default streaming behavior, but it still does not accept object-form Feishu `streaming` config or wire config-driven `throttleMs` and `title`. The PR is also not ready to close as implemented or merge as-is because its current head still appears to disable streaming when the setting is omitted and it does not update the public config metadata/docs surfaces.

## Best Possible Solution

Keep this PR open for focused rework or replacement. The best path is to rebase onto current main, add a typed Feishu streaming config resolver that accepts boolean and object forms while preserving `undefined` as default-enabled and `false` / `{ enabled: false }` as disabled, wire bounded `throttleMs` and optional title/header behavior on top of the existing identity-derived header/note behavior, update manifest/docs/config metadata, and add regression tests for default, disabled, object config, custom throttle/title, and auto-mode card/plain-text routing.
