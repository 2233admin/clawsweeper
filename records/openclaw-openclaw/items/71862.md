## Summary

Keep open. Current main still lacks the requested hook auth-profile preference, so the PR is not obsolete, but the proposed patch remains incomplete: the new resolver depends on a legacy compaction marker that the normal queued reply path does not forward, which can incorrectly lock migrated automatic auth profiles as user-pinned.

## What This Changes

The PR adds `authProfileOverride` to model-resolve hook results, merges it through hook priority, applies hook-selected auth profiles during embedded runner setup, forwards direct-attempt legacy markers, and updates related docs, SDK types, and tests.

## Best Possible Solution

Keep the hook/API direction, but preserve or normalize legacy auth-profile source metadata across every path that can call `runEmbeddedPiAgent`, especially normal queued chat replies. Add a regression for a migrated session with `authProfileOverride`, `authProfileOverrideCompactionCount`, and no source proving a trusted hook can still prefer a different automatic profile while explicit user-pinned profiles remain locked.

## Reproduction Assessment

Yes. A high-confidence static reproduction path is clear: create a migrated chat session entry with `authProfileOverride` plus `authProfileOverrideCompactionCount` and no source, enter the normal queued reply path, and the PR helper receives no compaction count and treats the profile as user-pinned.

## Solution Assessment

No. The proposed public hook shape is a narrow maintainable direction, but the implementation is not complete until legacy auth-profile marker metadata is carried or normalized through the queued reply runner path and covered by regression tests.

## Review Findings

- **[P2] Thread legacy auto auth markers through queued runs:** `src/agents/pi-embedded-runner/run/setup.ts:112-124`

A legacy chat session with `authProfileOverrideCompactionCount` and no source is therefore treated as user-pinned, so hook-selected auth profiles cannot rotate it; thread the marker through followup/run-base params or normalize the source before runner setup.
