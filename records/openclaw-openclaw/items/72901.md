## Summary

Keep open for PR revision. The latest patch fixes the common stored-field case, but it still diverges from current session reset behavior for legacy session rows whose `sessionStartedAt` must be recovered from the transcript header; in that case `initSessionState` can reset to an empty transcript while this Slack gate treats the entry as fresh and skips thread history.

## What This Changes

This PR changes Slack thread-context preparation to fetch thread history when a stored thread session is stale under daily or idle reset policy, and adds regression coverage for the freshness gate.

## Best Possible Solution

Land a narrow Slack-owned fix after the thread-history gate uses the same lifecycle timestamp resolver contract as session initialization, including JSONL header fallback for legacy rows, with a focused legacy `sessionFile` regression test. Track the active-session assistant-turn omission separately if it reproduces.

## Reproduction Assessment

Yes. The original bug is reproducible with a Slack thread session whose stored timestamp survives a daily or idle reset, and the remaining PR gap is reproducible with a legacy row whose recent `updatedAt` lacks `sessionStartedAt` while its transcript header predates the reset boundary.

## Solution Assessment

No, not yet. The PR direction is the right narrow Slack fix, but the implementation should reuse or expose `resolveSessionLifecycleTimestamps` instead of passing only direct store-entry fields into `evaluateSessionFreshness`.
