## Summary

Keep open. Current main still injects the exact Model Aliases prompt guidance reported here and still builds that prompt from every configured `agents.defaults.models` alias, so fallback/provider-overlap aliases can be visible to the parent LLM. The later sessions_spawn alias-resolution work fixes raw aliases after they are supplied to the tool, but it does not prevent the prompt-side provider substitution described in the report.

## What This Changes

The issue requests preventing Model Aliases prompt guidance from causing an LLM to replace an explicit `provider/model` subagent model request with an alias from a different provider.

## Best Possible Solution

Keep aliases as exact shorthand only, make the prompt explicitly preserve any supplied full `provider/model` string verbatim, and consider omitting fallback-only aliases from the agent-visible Model Aliases block while leaving CLI/gateway alias resolution intact.

## Reproduction Assessment

Partially yes: current main provides a high-confidence source-level reproduction of the risky prompt exposure and fallback alias visibility. The final LLM substitution is nondeterministic and was not live-run during this read-only review.

## Solution Assessment

No current implementation is not the best solution because it still encourages alias use before an explicit provider/model tool argument is formed. The narrow maintainable fix is prompt/filtering behavior, not changing the sessions_spawn resolver that already preserves qualified refs.

## Repair Work Prompt

Fix the prompt-side provider substitution risk from https://github.com/openclaw/openclaw/issues/61949. Current main still injects `## Model Aliases` with `Prefer aliases when specifying model overrides; full provider/model is also accepted.`, and `buildModelAliasLines` lists every alias from `agents.defaults.models`, including aliases for fallback providers. Make a narrow fix in the agent prompt/model-alias surface so an explicit full `provider/model` string is preserved verbatim and aliases are presented only as exact shorthand when no provider/model was specified. Consider suppressing aliases that are only present as configured fallback models from the agent-visible Model Aliases block, but do not remove CLI/gateway alias resolution or the #59681 behavior that resolves bare aliases passed to `sessions_spawn`. Add focused tests for the prompt wording and, if filtering is added, for fallback-only alias exclusion with a multi-provider overlap. Keep docs/changelog judgment conservative; add a changelog fix entry only if the user-facing prompt behavior changes. Do not touch provider transport, Bedrock thinking behavior, or broad model fallback policy.
