## Summary

Keep open. Current main still routes `sessions_send` through an A2A ping-pong/announce path that can inject the target reply back into the requester session after returning it inline, and existing tests assert multiple inter-session agent steps for one send. Related #53145 is important context and should inform the fix, but it is not an exact replacement for this persistent sub-agent duplicate-delivery report.

## Best Possible Solution

Keep this issue open and fix the `sessions_send` A2A/sub-agent delivery path. The best implementation should choose one parent-visible delivery path for parent-owned persistent sub-agent replies, suppress or dedupe repeated same-run inter-session replies before transcript/UI injection, treat control replies such as `NO_REPLY`, `ANNOUNCE_SKIP`, and `REPLY_SKIP` as non-deliverable where appropriate, and add focused regression coverage for `runtime="subagent"`, `mode="session"`, and `sessions_send` parent delivery. Coordinate the shared control-token and placeholder cleanup with #53145.
