## Summary

Keep open. Current main still has a plausible native persistent sub-agent duplicate-delivery path: the sessions_send A2A skip only covers parent-owned background ACP sessions, while waited sends can still return the child reply inline and also schedule the A2A announce flow that resolves delivery from the target session's stored deliveryContext. Related #53145 is useful coordination context, but it is broader, still open, and not a terminal duplicate for this native sub-agent report.

## What This Changes

The issue reports duplicated parent-visible replies when sessions_send talks to a persistent native sub-agent session that carries inherited deliveryContext.

## Best Possible Solution

Fix the sessions-send delivery contract so parent-owned native persistent sub-agent replies have exactly one parent-visible route, likely by detecting own-child native sub-agent sends and suppressing the redundant A2A/announce path while leaving peer A2A behavior and the existing ACP guard intact.

## Reproduction Assessment

Yes. The issue gives concrete steps for persistent native sub-agent sessions, and current source shows a high-confidence code-level reproduction: a waited sessions_send can return the target reply inline while still scheduling A2A announce delivery through the target session's stored deliveryContext.

## Solution Assessment

No. Current main's ACP-only skip is not the best complete solution for this report because native persistent sub-agent entries are not covered. The narrow maintainable solution is to choose one delivery path for parent-owned native child sends and keep unrelated peer A2A semantics unchanged.
