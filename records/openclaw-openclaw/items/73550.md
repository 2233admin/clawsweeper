## Summary

Keep open. Current main still leaves the native persistent sub-agent path on the general sessions_send A2A reply/announce flow, so a waited send can return the child reply inline while a follow-up reply/announce path remains scheduled and can target the child session's stored deliveryContext. Related #53145 is useful coordination context for the same inter-session feedback-loop family, but this issue has distinct persistent sub-agent deliveryContext evidence and should remain open.

## What This Changes

The issue requests that sessions_send deliver persistent native sub-agent replies to the parent exactly once and avoid deliveryContext/announce feedback loops.

## Best Possible Solution

Make the sessions_send delivery contract choose exactly one parent-visible path for parent-owned persistent native sub-agent targets: suppress same-run duplicate transcript/UI injection, avoid announcing through the child session's inherited deliveryContext, keep legitimate peer A2A messaging intact, and share control-token filtering work with #53145.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists from the issue steps plus current source: a waited sessions_send can return the target reply inline and still schedule A2A reply/announce work that uses the target session deliveryContext. I did not run a live UI reproduction because this review is read-only.

## Solution Assessment

No. Current main only skips A2A for requester-owned background ACP sessions, while the reported native persistent sub-agent path still follows pending announce delivery. The narrower maintainable fix is to detect parent-owned persistent native sub-agent sends and select one delivery route with regression coverage for the deliveryContext loop.

## Repair Work Prompt

Fix sessions_send duplicate delivery for parent-owned persistent native sub-agent sessions. The observable bug is that a parent sending to a persistent native sub-agent can see the same reply more than once through inline reply, A2A reply injection, and/or announce delivery through the child session's inherited deliveryContext. Keep the change narrow to the sessions_send/sub-agent delivery contract. Detect parent-owned persistent native sub-agent targets separately from the existing requester-owned background ACP skip, choose exactly one parent-visible delivery path, and prevent the child session deliveryContext from causing an announce loop back to the parent or child. Preserve ordinary peer agent-to-agent messaging and ACP streamTo="parent" behavior. Coordinate suppression of NO_REPLY, ANNOUNCE_SKIP, REPLY_SKIP, and placeholder/control text with #53145. Add regression tests for waited and fire-and-forget sessions_send against a parent-owned persistent native sub-agent session with stored deliveryContext. Do not change unrelated channel routing, provider behavior, release metadata, or broad session visibility policy. Add a CHANGELOG.md Fixes entry only if the resulting behavior is user-facing under the active version, crediting @sylviazhang2006-design and @TarahAssistant if #53145 is covered too.
