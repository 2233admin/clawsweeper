## Summary

Current main still has the WebChat cumulative-stream duplication path that this PR is trying to fix. The gateway broadcasts cumulative assistant text, the UI stores the latest cumulative delta as `chatStream`, tool-start handling commits the full `chatStream` into `chatStreamSegments`, and `buildChatItems` renders stored segments plus the live stream without suffix dedupe. The related #47377 is open and unmerged, so it is not a superseding landed fix.

## Best Possible Solution

Keep this PR open while maintainers choose the implementation path. The best fix is a focused main-branch change that stores only the uncommitted suffix when a tool starts, dedupes the live stream preview against already committed stream segments, and keeps regression coverage for repeated identical segments plus multi text/tool/text cycles. If #47377 becomes the preferred replacement, close this PR only after that replacement lands or a maintainer explicitly marks it superseded.
