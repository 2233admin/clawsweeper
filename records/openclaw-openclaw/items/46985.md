## Summary

Keep this PR open. Current main still appears to have the WebChat cumulative-stream duplication path: gateway chat deltas are cumulative snapshots, the UI stores each latest snapshot as `chatStream`, tool-start handling commits the full snapshot into `chatStreamSegments`, and the renderer then renders stored segments plus the raw live stream. The PR is a focused UI fix with regression tests and no obvious security/supply-chain-sensitive file changes in the latest provided diff; the related #47377 is still open/unmerged, so it is not a landed superseding fix.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a focused WebChat fix that stores only the uncommitted suffix at tool boundaries, dedupes the live stream preview against committed stream segments, and keeps regression coverage for repeated identical segments plus multi text/tool/text cycles. If #47377 becomes the preferred replacement, close this PR only after that replacement lands or a maintainer explicitly marks it superseded.
