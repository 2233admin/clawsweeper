## Summary

Keep open. Current main still lacks a generic doctor/config warning for default or per-agent primary model refs, so the contribution targets a real gap, but the branch is not ready to merge because the proposed diagnostic can miss the linked misconfiguration, can false-warn documented alias configs, and prints config-derived warning text without sanitization.

## What This Changes

The PR branch adds `collectModelConfigWarnings`, wires a new `doctor:model-config` health contribution, and adds tests for warning when `agents.defaults.model.primary` or `agents.list[].model` is absent from the loaded model catalog.

## Best Possible Solution

Land a focused doctor/config diagnostic that resolves configured model refs through the same alias-aware path as runtime/model-picker code, distinguishes provider/static/runtime catalog evidence from user-declared config rows, sanitizes terminal output, and adds regression coverage for #39811 plus alias-based configs without changing runtime failover behavior in this PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate against non-config catalog rows:** `src/flows/doctor-health-contributions.ts:407`
  - body: The new doctor check passes `loadModelCatalog({ config: ctx.cfg })` into the warning helper, but that catalog appends `models.providers.*.models` from the same config. In the #39811 reproduction, the bogus model is declared there first, so this can treat it as present and emit no warning even though provider/runtime calls can still fail. Use provider/static/runtime-origin evidence or origin metadata for this diagnostic.
  - confidence: 0.88
- **[P2] Resolve aliases before checking primaries:** `src/config/validation.ts:1341`
  - body: `collectModelConfigWarnings` calls `resolveModelRefFromString` without `cfg` or an alias index, so a valid alias from `agents.defaults.models` can be parsed under the default provider and warned as missing. Doctor should match the documented runtime/model-picker alias resolution path before checking catalog membership.
  - confidence: 0.9
- **[P3] Sanitize model warning text before printing it:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning path and message include model identifiers derived from local config and are passed directly to `note()`. Route them through `sanitizeTerminalText` or the existing config issue formatter so control characters cannot spoof or obscure doctor output.
  - confidence: 0.82
