## Summary

Keep this PR open. Current main still has the Gateway dedupe cleanup under-eviction bug, and the production diff is a narrow fix, but the added regression test only covers a one-entry overflow that already passes against the buggy loop. The PR also overlaps with open PR #45979, so maintainers should choose one canonical patch before landing.

## What This Changes

The PR precomputes the Gateway dedupe overflow count, deletes timestamp-sorted oldest keys, and adds a maintenance test for timestamp-based eviction after reinsertion.

## Best Possible Solution

Land one canonical Gateway maintenance patch that precomputes the excess count and deletes a fixed list of timestamp-sorted oldest dedupe keys, with a regression test that overflows by at least two entries and proves the cache trims exactly to `DEDUPE_MAX`.

## Reproduction Assessment

Yes. The current-main loop can be reproduced with a read-only simulation using `DEDUPE_MAX + 10` entries, which leaves the map at 1005 entries because the loop bound shrinks during deletion.

## Solution Assessment

Unclear as-is. The production code change is the narrow maintainable fix, but the PR should strengthen its regression test and be reconciled with #45979 before it is the best landing package.
