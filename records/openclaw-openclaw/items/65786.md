Current main still rejects the reporter's Feishu config shape: Feishu `streaming` remains boolean-only, `contextVisibility` is read at runtime but not accepted by the strict Feishu schema, and `openclaw doctor --fix` still has no Feishu compatibility normalizer for legacy scalar streaming values such as `"progress"`.

## What This Changes

The reporter wants Feishu config validation and `openclaw doctor --fix` to handle a previously working upgraded config containing `streaming: "progress"` and `contextVisibility` instead of rejecting `channels.feishu` as invalid.

## Best Possible Solution

Keep this issue open and fix it in the Feishu plugin/config contract: accept the runtime-supported `contextVisibility` shape where intended, add a Feishu doctor compatibility normalizer for legacy scalar `streaming` values such as `"progress"`, regenerate bundled channel config metadata, update https://docs.openclaw.ai/channels/feishu, and add regression coverage using the reporter's config plus a doctor repair path.
