## Summary

Keep this issue open. Current main still has the reported behavior: cron task records are explicitly silent, while subagent task records omit a notify policy and therefore fall through to the task registry default of `done_only`. The current config types, zod schema, and subagent docs expose model/thinking/timeout/archive/allowlist defaults but no configurable subagent notify policy or parent-inheritance option. The provided related items are adjacent cron/task feature reports, not a canonical duplicate or shipped fix for this request.

## Best Possible Solution

Add a backwards-compatible subagent task notify default, preserving current `done_only` behavior when unset. A focused fix should let `agents.defaults.subagents.notifyPolicy` choose `done_only`, `silent`, or `inherit_from_parent`; for inheritance, cron-run parent sessions should suppress subagent task completion notifications in line with the parent cron task while ordinary sessions can retain the current default unless a real parent task policy is available. Update config types/schema, docs, and targeted tests for cron-spawned subagents.

## Clownfish Work Prompt

Implement a focused fix for https://github.com/openclaw/openclaw/issues/73905. Current behavior: cron task records are silent, but subagent task records created from cron-run sessions omit `notifyPolicy` and default to `done_only`, causing completion notifications from subagents spawned inside otherwise silent cron jobs.

Expected boundary:
- Add a backwards-compatible configurable default for subagent task notify policy under `agents.defaults.subagents.notifyPolicy` with values `"done_only"`, `"silent"`, and `"inherit_from_parent"`.
- Preserve current behavior when unset: normal subagent tasks still default to `done_only`.
- For `"silent"`, create subagent task records with `notifyPolicy: "silent"`.
- For `"done_only"`, create them with `notifyPolicy: "done_only"`.
- For `"inherit_from_parent"`, make canonical cron run parent sessions (`agent:<agentId>:cron:<jobId>:run:<runId>`) inherit the parent cron quiet behavior and use `silent`; for ordinary parent sessions, keep current `done_only` unless there is an existing task/flow parent policy available without inventing a broad new session policy.
- Do not change cron failure alert routing, cron delivery modes, subagent announce formatting, task lifecycle statuses, or non-subagent task defaults.
- Update config types, zod schema, generated config docs/schema artifacts as required by the repo, and public docs (`docs/automation/tasks.md`, `docs/tools/subagents.md`, `docs/gateway/config-tools.md`, and configuration reference if generated).
- Add targeted tests proving configured `silent`, configured `done_only`, unset default, and `inherit_from_parent` for a cron-run requester session.
- Add a user-facing changelog entry if the repo’s active changelog requires one; credit `Thanks @arlen8411`.

Suggested validation: run the focused tests for subagent spawn/task policy and config schema, then the changed-file config/docs generation check. Do not broaden into unrelated cron or task refactors.
