  - body: This workflow invokes `openai/codex-action@v1` while passing an OpenAI API key and granting PR write access.   - body: The autofix job only checks for a failed push CI run, so failed push CI on any repository branch can run a write-token Codex autofix and open a PR. Add the documented `head_branch == 'main'` gate, current-main provenance checks, and a secret guard before granting write permissions.
  - confidence: 0.92
- **[P1] Preserve knip's exit status before truncating output:** `.github/workflows/dead-code-sweep.yml:41`
  - body: `pnpm knip` is piped into `head` without `pipefail`, so the captured exit code is usually `head`'s status rather than Knip's. Dead-code findings can be treated as clean runs, which skips the report path and may run the clean-sweep close path incorrectly.
  - confidence: 0.9
- **[P2] Keep actionlint ignores under paths:** `.github/actionlint.yaml:13`
  - body: The new top-level `ignore` block does not match the current repo config shape used by actionlint. Workflow Sanity runs `actionlint`, so this config change can make workflow linting fail before it checks the workflow files.
  - confidence: 0.86
- **[P2] Skip PR review when secrets are unavailable:** `.github/workflows/codex-pr-review.yml:25-28`
  - body: The PR review job runs on `pull_request` but does not skip forks or missing `OPENAI_API_KEY`. Forked PRs normally do not receive repository secrets, so this check will fail instead of cleanly no-oping for external contributors.
  - confidence: 0.88
- **[P2] Avoid write-token comment calls on forked PRs:** `.github/workflows/bundle-size.yml:136-141`
  - body: The bundle-size report runs on `pull_request` and always tries to create or update PR comments. Fork-triggered runs usually have a read-only token, so external-contributor PRs can fail with `Resource not accessible by integration`; use a trusted reporting pattern or skip the comment step when write permission is unavailable.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces secret-bearing and write-token GitHub Actions workflows with unpinned third-party action refs and insufficient trusted-event/fork guards.

Concerns:

- **[high] Unpinned third-party action receives secrets:** `.github/workflows/codex-pr-review.yml:43`
  - body: `openai/codex-action@v1` is used in a workflow that receives `OPENAI_API_KEY` and has PR write permission. Current main's Codex automation pins the action to a full commit SHA; this PR should do the same before secrets are passed to the action.
  - confidence: 0.9
- **[high] Autofix write token is not limited to main:** `.github/workflows/codex-autofix.yml:22`
  - body: The workflow_run job grants contents and pull-requests write but gates only on failed push CI, not the main branch or exact current main SHA. That is broader than the PR description and weaker than the hardened main-only Codex workflow pattern in current main.
  - confidence: 0.92
- **[medium] PR workflows can fail or misbehave for forks:** `.github/workflows/codex-pr-review.yml:25`
  - body: The AI review and bundle-size workflows run on `pull_request` while depending on unavailable secrets or write-token comment permissions. Without fork/permission guards, external contributor PRs can get red checks instead of skipped optional automation.
  - confidence: 0.86
- **[medium] Dead-code issue mutation can affect PRs:** `.github/workflows/dead-code-sweep.yml:41`
  - body: The dead-code sweep uses issue APIs to update or close matching open items without filtering out pull requests returned by issue listing endpoints, and the Knip pipeline can mask findings. That can mutate or close a PR labeled `dead-code`.
  - confidence: 0.88
