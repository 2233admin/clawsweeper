## Summary

Keep open. The reported bug is still present on current main: `sessions_spawn` declares and reads `cwd`, forwards it to ACP, but the native subagent path still calls `spawnSubagentDirect` with only `opts?.workspaceDir`. The PR is not merge-ready as-is because review already identified attachment/workspace regressions in its follow-up changes, but the underlying subagent cwd behavior still needs a targeted fix and tests.

## Best Possible Solution

Keep the PR open or supersede it with a focused patch that forwards `sessions_spawn.cwd` to native subagent same-agent workspace inheritance, preserves target-agent workspace behavior for cross-agent spawns, aligns attachment materialization with the actual child runtime workspace, and adds regression tests for subagent cwd, ACP parity, attachments with cwd, and cross-agent attachments.
