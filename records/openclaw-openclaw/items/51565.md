## Summary

Keep open. Current main still does not implement the PR’s intended WhatsApp per-agent ack emoji fallback: the WhatsApp plugin reads only `channels.whatsapp.ackReaction.emoji`, while the generic identity resolver still assumes channel/account ack reactions are strings. The PR is a focused, plausible fix, but it still needs maintainer review/revision for the unresolved test lint/type concern and the plugin-SDK export surface.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The best path is a cleaned implementation that preserves WhatsApp’s no-global-ack behavior, adds per-agent identity fallback only when WhatsApp ack config is intentionally present, covers account/channel/object edge cases with typed tests, and treats any plugin-SDK export as an intentional public contract.
