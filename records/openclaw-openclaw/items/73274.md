## Summary

Keep open. Current main still exposes only session store/path helpers on the plugin runtime surface, while the durable transcript append helper remains internal. The linked cron-context workaround is concrete evidence of a real plugin need, but making durable cross-session transcript writes public needs maintainer SDK and security design first.

## What This Changes

Request an additive Plugin Runtime API that lets trusted plugins persist assistant messages into another session transcript by sessionKey.

## Best Possible Solution

Design an additive, host-owned runtime capability for durable transcript mirroring that preserves assistant-only delivery-mirror semantics, idempotency, session normalization, and transcript update events, with explicit operator/trusted-plugin authorization. Maintainers should also decide whether isolated cron announce delivery should mirror directly into target channel sessions without requiring a third-party plugin.

## Reproduction Assessment

Not applicable as a feature/API request. The high-confidence check is static: current main lacks `api.runtime.agent.session` transcript append support while the internal append helper, ephemeral system event queue, and durable-but-not-transcript next-turn injection path are all present.

## Solution Assessment

Unclear. A persistent helper is plausible, but directly exposing the internal helper with arbitrary session keys or user-role writes is not yet the narrowest secure design; an assistant-only, opt-in delivery-mirror capability is the safer starting point.

## Security Review

Status: needs_attention

Summary: The requested API is security-sensitive because it would let plugin code durably write content into other sessions' transcripts.

  - confidence: 0.9

  - body: The internal exact helper rejects non-assistant messages, while the proposed API includes a `user` role. Public exposure should keep assistant-only delivery-mirror semantics unless maintainers define user attribution and replay rules.
  - confidence: 0.86

  - body: The mirror helper writes supplied text after trim/media-name projection and emits transcript updates. A public plugin API needs explicit redaction, audit, and prompt-injection policy before durable plugin-supplied content is persisted.
