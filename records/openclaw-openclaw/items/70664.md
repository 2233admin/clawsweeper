## Summary

Keep open. Current main still contains the reported Unix/macOS bug shape: the stale PID parser classifies gateway listeners from `lsof -Fpc` `c` command tokens, so a real Node-based gateway reported as `cnode` is dropped before cleanup or port-free polling can act. The linked fix PR #70681 appears to track the right work, but it is still open and not merged into main, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep #70664 open until PR #70681 or an equivalent fix lands. The best code path is to make Unix lsof parsing return raw listener PIDs after self/ancestor exclusion, verify gateway ownership via full argv (`ps` on macOS and `/proc/<pid>/cmdline` or equivalent on Linux) using the existing `isGatewayArgv` contract, apply that verification in both `findGatewayPidsOnPortSync` and `pollPortOnce`, and update regression tests so `lsof` reporting `cnode` with `ps` showing `openclaw-gateway` is treated as a verified stale gateway listener.
