## Summary

Current main still has the reported Unix/macOS behavior: `findGatewayPidsOnPortSync` runs `lsof -Fpc` and `parsePidsFromLsofOutput` drops any listener whose `c` command token does not contain `openclaw`, before the existing argv verifier can inspect the process. Tests still assert that `cnode`/non-OpenClaw command names are excluded. There is also an open paired fix PR, #70681, with closing linkage, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land #70681 or an equivalent narrow fix. The fix should stop using lsof `c`/`p_comm` as the Unix gateway classifier, preserve self and ancestor exclusions, verify Unix listener PIDs through full argv inspection before killing or treating the port as busy, and update coverage for macOS-style `cnode` plus unrelated non-gateway listeners.
