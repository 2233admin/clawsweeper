## Summary

Keep this PR open. Current main still registers `openclaw cron runs` as a `--id`-only command, docs still show only the flag form, and current CLI tests do not cover the proposed positional alias. The PR remains a focused implementation candidate for the linked CLI UX bug.

## Best Possible Solution

Keep this PR open as the focused CLI fix candidate for the linked bug. Rebase it on current main, remove the unreachable returns after `command.error()`, add the missing no-ID test, update the cron CLI docs to document the accepted forms, then run the targeted cron CLI test and the appropriate changed gate before merge.
