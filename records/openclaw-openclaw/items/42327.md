## Summary

Keep open. Current main still registers `openclaw cron runs` as a flag-only command requiring `--id`, while this PR is a focused implementation candidate for the linked cron CLI UX bug #42300. The remaining work is review cleanup, a missing no-id regression test, and docs updates.

## Best Possible Solution

Keep this PR open as the focused fix candidate. Rebase it on current main, address the existing review comments, add the missing no-id test in `src/cli/cron-cli.test.ts`, update the public cron docs to document both positional and `--id` forms, then run the targeted cron CLI test plus the appropriate changed gate before merge.
