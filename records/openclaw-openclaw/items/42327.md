## Summary

Keep this PR open. Current main still does not implement the intended `openclaw cron runs <jobId>` positional alias: the CLI command remains `--id`-only, docs still document only the flag form, and the current cron CLI tests have no `cron runs` coverage. The PR remains a focused implementation candidate for the linked CLI UX bug, with small review cleanup needed before merge.

## Best Possible Solution

Keep this PR open as the focused CLI fix candidate. Rebase it on current main, address the two review comments, update the cron CLI docs to document accepted forms, then run the targeted cron CLI test and the appropriate changed gate before merge.
