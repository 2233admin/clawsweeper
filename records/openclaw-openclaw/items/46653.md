## Summary

Keep this PR open. Current main has transient SSH tunneling for `openclaw gateway probe --ssh` and macOS app remote mode, but it still does not implement the PR's persistent `openclaw tunnel up/down/status` CLI, PID-backed state, or CLI docs page. The PR is a plausible feature candidate, but it needs a refresh against the current descriptor-based CLI registration plus fixes for the process/state safety issues found in review.

## Best Possible Solution

Keep the PR open for contributor or maintainer follow-up. The best path is to refresh it onto the current descriptor-based CLI registration, reuse or extend `src/infra/ssh-tunnel.ts` for readiness and target validation, add ownership-safe persistent state for `up/down/status`, and add targeted tests for stale PID, reused PID, port collision, invalid target, and fresh state directory behavior.
