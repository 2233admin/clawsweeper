## Summary

Keep this PR open. Current main still does not implement the PR's intended persistent `openclaw tunnel up/down/status` command or its PID-backed lifecycle state; the only current SSH CLI surface is temporary `gateway probe --ssh` forwarding, and docs still tell CLI users to create SSH tunnels manually. The PR remains a plausible feature candidate, but it needs a refresh for the current descriptor-based CLI registration plus fixes for unresolved process/state safety issues from review.

## Best Possible Solution

Keep the PR open for contributor or maintainer follow-up. The best path is to rebase/refactor it onto the current descriptor-based CLI registration, reuse or extend `src/infra/ssh-tunnel.ts` for target validation and readiness, add ownership-safe persistent state for `up/down/status`, and cover stale PID, reused PID, port collision, invalid target, and fresh state-directory behavior with targeted tests.
