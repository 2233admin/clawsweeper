## Summary

Keep open. Current main still leaves `openclaw tasks maintenance` scoped to task-registry and Task Flow maintenance, while the linked implementation PR #74023 remains open and unmerged with closing syntax for this issue.

## What This Changes

Add conservative `openclaw tasks maintenance` preview/apply pruning for stale cron run session-registry entries older than seven days while protecting jobs currently running in `jobs-state.json`.

## Best Possible Solution

Land PR #74023 or an equivalent focused change that adds session-registry maintenance to `tasksMaintenanceCommand`, reports it in text and JSON output, protects currently running cron jobs via durable job state, updates the relevant docs, and covers preview/apply behavior with targeted command tests.

Yes. Source inspection on current main shows `tasksMaintenanceCommand` only invokes task-registry and Task Flow maintenance, while stale cron run session rows are only handled by the separate cron timer reaper path.

## Solution Assessment

Yes. A command-level sweep with jobs-state protection is the narrowest maintainable direction; the already-linked PR #74023 is the right implementation candidate to review instead of opening a duplicate repair lane.
