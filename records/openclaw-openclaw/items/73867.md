## Summary

Keep open. Current main still does not wire a session-registry sweep into `openclaw tasks maintenance --apply`; that command only runs task-registry and Task Flow maintenance. Existing session cleanup and cron-run session reaping are separate paths and do not implement the requested conservative tasks-maintenance sweep with a durable `jobs-state.json` active-job guard.

## Best Possible Solution

Implement a focused `tasks maintenance` session-registry sweep on current main. Add a small session-registry maintenance helper that previews/applies pruning of old cron-run session entries, protects active cron jobs derived from durable cron state, exposes additive JSON/text counts beside `tasks` and `taskFlows`, reuses existing session-store transcript cleanup behavior where possible, and documents the new task-maintenance behavior.

## Clownfish Work Prompt

Implement the fix for openclaw/openclaw issue #73867: `openclaw tasks maintenance --apply` should also preview/apply a conservative session-registry sweep.

Scope:
- Add an additive session-registry maintenance phase to `tasksMaintenanceCommand` beside the existing `tasks` and `taskFlows` blocks.
- Target stale cron-run session entries in configured agent `sessions.json` stores. Do not prune general chat, ACP, subagent, or base cron session keys.
- Use conservative criteria: old cron-run session entries older than 7 days, excluding entries protected by durable cron active state. Derive active jobs from `loadCronStoreSync(resolveCronStorePath(cfg.cron?.store))` / merged `jobs-state.json` state. Because active state is job-level (`runningAtMs`) while run keys include a generated session id, protect the base cron session key and matching `agent:<agentId>:cron:<jobId>:run:*` prefix for jobs with `runningAtMs`, unless you find a stronger existing mapping.
- Reuse existing session-store update/archive/cleanup primitives where feasible; do not duplicate unsafe file deletion logic.
- JSON output should stay additive, for example `maintenance.sessionRegistry`, and text output should include a concise session-registry count in preview/apply mode.
- Preserve dry-run behavior for `openclaw tasks maintenance` and only mutate stores under `--apply`.
- Update docs for `openclaw tasks maintenance` and add a changelog Fixes entry if this is user-facing in the active changelog section. Credit `@dknoodle`; credit `@hclsys` too if the implementation uses the sketch from the discussion.
- Do not change `session.maintenance` defaults, cron retention defaults, startup/load-time migrations, `jobs.json`/`jobs-state.json` persistence semantics, or unrelated task reconciliation behavior.
