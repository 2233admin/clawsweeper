## Summary

Keep this PR open. Current main has already handled the Slack `ignoreSelf` behavior and supports inline TTS directives, but I do not see the PR’s central config-validation change implemented on main: plugin validation still builds `knownIds` only from the loaded manifest registry, and the specific missing-bundled-dir/provider-ID validation tests from the PR are not present.

## Best Possible Solution

Keep the PR open, but do not merge it as-is. The best path is to rebase or port only the config-validation portion: centralize known plugin ID resolution for validation, include bundled alias/auto-enable metadata when manifest discovery is incomplete, and add focused tests for `plugins.allow`, `plugins.entries.*`, and `plugins.slots.memory` under a missing bundled plugin directory. Drop or separately close the Slack and TTS changes because current main already covers those surfaces.
