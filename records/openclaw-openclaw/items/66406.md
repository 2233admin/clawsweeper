## Summary

Keep open. Current main still exposes Feishu `react`/`reactions` and passes the user-provided emoji string directly into Feishu reaction API parameters. The latest shipped tag still has the same behavior, and open PR #66478 remains the direct linked implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #66478, or an equivalent Feishu-owned fix, that normalizes Unicode emoji and accepted Feishu type strings before add/remove/list reaction API calls. The fix should include focused tests proving `message action=react emoji="👍"` sends a Feishu type such as `THUMBSUP`, then this issue can close after the implementation merges.
