## Summary

Keep this PR open. Current main has not implemented the intended behavior: `resolveCommandAuthorizedFromAuthorizers` still denies `useAccessGroups=true` when every authorizer is unconfigured, and the Feishu group test still locks the reported no-`allowFrom` case to `CommandAuthorized: false`. The PR is a plausible fix candidate for #49915, but the posted review comments identify real security/contract risks in the PR as written, so the right maintainer path is to revise or review the PR rather than close it as obsolete.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the linked Feishu group command bug, but require a scoped fix before merge: allow the no-`allowFrom` fallback only for the intended group command flow, preserve the open-DM no-allowlist contract, align the root SDK CJS shim or its generation path, and add targeted regression tests for Feishu group commands plus the DM non-authorization case.

## Risks / Open Questions

- The PR as described broadens command authorization in `resolveCommandAuthorizedFromAuthorizers`; if applied unconditionally, it can authorize open-DM command flows with empty allowlists, contrary to existing tests and the command authorization security model.
