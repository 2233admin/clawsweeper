## Summary

Keep open. Current main still has a concrete Google Vertex ADC credential-routing bug: the ADC path resolves to the non-secret `gcp-vertex-credentials` marker, no Google provider runtime-auth hook exchanges it, and the Google transport serializes raw auth strings as `x-goog-api-key`. The issue is recent, has another affected-user confirmation, and the separate Node 25/gaxios crash still needs verification rather than cleanup closure.

## What This Changes

The issue reports that `google-vertex` requests fail on Node 25+ and that ADC credentials are passed downstream as a raw marker/API key instead of being used as Vertex auth.

## Best Possible Solution

Keep the issue open and fix Google Vertex ADC in the provider/auth path: exchange or use ADC before request time, avoid storing or forwarding `gcp-vertex-credentials` as an API key, target the Vertex endpoint with project/location, and add regression coverage proving the marker is never emitted as `x-goog-api-key`. Verify the Node 25/gaxios failure separately against the current dependency/runtime path.
