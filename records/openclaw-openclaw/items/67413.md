## Summary

Keep open. Current main still exposes dreaming as global memory-core config only, still resolves all configured agent workspaces for automatic dreaming, and still documents one managed sweep schedule. The issue is also paired with open implementation PR #67420 using closing syntax, so it should remain open until that PR is reviewed, merged, or closed.

## What This Changes

Request per-agent dreaming enable/disable and schedule overrides so memory-core does not run every agent workspace on one global cron.

## Best Possible Solution

Add a documented, schema-backed agents.list[].dreaming contract for per-agent enablement, wire it through the config types and workspace resolver, and explicitly decide whether per-agent cron schedules require separate scoped managed jobs or a narrower follow-up.
