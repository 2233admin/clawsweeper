## Summary

Keep open. Current main still prepares and can rewrite the active agent `models.json` during gateway startup, and merge mode still refreshes matching provider model arrays from generated config/catalog data while preserving only selected credential/base URL fields. The docs now describe that behavior, but the product contract for direct user pruning of per-agent model lists is still unresolved.

## What This Changes

The issue asks for user-edited per-agent `models.json` model lists to persist across gateway restarts instead of being restored from default, cached, or generated model registry data.

## Best Possible Solution

Maintainers should decide the intended contract for per-agent `models.json`. If direct pruning is supported, add a gateway-restart regression and preserve deletions through an explicit tombstone or replace mechanism; if it is generated state, make CLI, doctor, setup, and docs steer persistent pruning through `models.providers` or `agents.defaults.models` with the intended merge/replace flow.
