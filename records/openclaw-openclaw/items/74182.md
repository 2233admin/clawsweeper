## Summary

Keep open. Current main and the latest release still have the inline embedded-run abort wrapper that matches the suspected retention surface, and open draft PR #75008 is the direct implementation candidate rather than a merged fix.

## What This Changes

The issue requests cleanup of embedded LLM run timeout and abort paths so abandoned provider promises, listeners, registries, buffers, and run closures do not retain memory in the single-process Gateway.

## Best Possible Solution

Review and land PR #75008 or an equivalent replacement that extracts the abort wrapper out of the run scope, adds the hung-provider abort retention regression test, and profiles the remaining subagent announce and wait-for-idle surfaces before closing this issue.

## Reproduction Assessment

Unclear. The report gives a credible live reproduction path over several hours with provider instability and memory growth, but current main does not include a deterministic heap/RSS harness; source inspection still shows a plausible retained-promise path.

## Solution Assessment

Yes for keeping this open. The narrowest maintainable path is to review the existing draft PR and require targeted retention proof, rather than closing the issue or starting a separate repair while that PR is active.
