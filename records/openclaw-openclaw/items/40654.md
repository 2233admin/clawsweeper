## Summary

Keep this PR open. Current main still closes ACP sessions and unbinds conversation bindings without archiving Discord threads, so the central behavior is not implemented; however, the PR is stale against the current plugin layout and would need to be ported behind an extension-safe Discord/plugin seam before it can merge.

## What This Changes

The PR records ACP-created Discord thread ownership metadata, makes `/acp close` request archive-after-reply behavior for owned Discord threads, adds a Discord REST archive helper, and covers ACP close, outbound payload, and thread-binding metadata behavior with tests.

## Best Possible Solution

Land the behavior on current main by adding a generic close-time reply/payload hook or equivalent channel-binding contract, storing ACP-created child-thread ownership metadata in the binding record, and letting `extensions/discord` archive only those owned threads after the close reply while preserving runtime notices and partial-failure messaging.
