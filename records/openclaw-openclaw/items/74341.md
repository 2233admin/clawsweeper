## What This Changes

The PR validates forced plugin harness support before pinning, broadens the Codex harness default provider ids, adds selection and Codex harness tests, and adds a changelog entry.

Review a narrowed fix that validates forced and session-pinned plugin harnesses with `supports()` before pinning, while preserving the documented auto-mode boundary.
