## Summary

Keep this PR open. Current main does not implement the PR’s intended all-channel `operator.admin` requirement: it only blocks `/phone arm` and `/phone disarm` when `gatewayClientScopes` is present without `operator.admin`, while preserving external channel calls that have no gateway scope vector. The PR remains a focused security/product decision rather than obsolete work.

## Best Possible Solution

Do not close this PR as implemented or obsolete. The best path is explicit maintainer security/product review of the desired `/phone arm|disarm` authority model: either land an updated version that requires `operator.admin` for all mutation paths and flips the external no-scope tests, or revise the PR to match the current internal-gateway-scope boundary while documenting why external authorized senders remain allowed.

## Risks / Open Questions

- This is a security-sensitive authorization change, but it also changes current channel-owner behavior: external authorized channel commands normally have no `gatewayClientScopes`, so requiring `operator.admin` universally would block those current paths unless another owner/admin signal is supplied.
- Current main has only the narrower non-webchat-with-operator.write regression covered. If maintainers want universal admin gating for phone-control mutations, the external no-scope tests need to be flipped as this PR proposes.
