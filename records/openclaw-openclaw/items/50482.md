## Summary

Keep open. Current main still loses the Telegram voice-note origin before rendering agent-visible metadata, so agents cannot distinguish dictated voice from typed text or generic audio. Related #40838 is broader Telegram metadata work and does not supersede this narrower request.

## What This Changes

Request to expose whether an inbound agent turn originated from a voice note, preferably through a generic message type or modality field in inbound context metadata.

## Best Possible Solution

Add a backwards-compatible generic inbound message-origin or modality field to the shared context contract, populate it from Telegram before msg.voice is normalized to audio, render it for agents in the dynamic conversation metadata, and document the field with regression coverage for voice notes versus ordinary audio and text.
