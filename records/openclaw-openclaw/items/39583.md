## Summary

Keep open. Current main and the latest release still return string chat content before content-block normalization, so JSON-stringified vLLM/OpenAI-compatible content-block arrays remain raw strings. This PR is still relevant, but the latest type-only JSON-array guard from the PR discussion needs a narrower chat-block shape check before merge.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best fix is a conservative shared normalization helper that parses strings only when they are recognizable chat content-block arrays, mirrors real-array extraction for text-only and mixed media/text blocks, preserves arbitrary JSON strings, and adds regression tests for vLLM text-only blocks, mixed blocks, invalid JSON fallback, and non-chat typed JSON arrays. After that, review whether gateway prompt and streaming boundaries should call the same helper or need a separate output-normalization step.
