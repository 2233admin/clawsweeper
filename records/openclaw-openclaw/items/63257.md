## Summary

Keep open. Current main still has a bundled Feishu startup bot-identity probe with a 30s default budget, and the Feishu account monitor still resolves bot identity before the WebSocket monitor starts. The tests currently cover "timeout; continuing startup" rather than proving the timeout no longer delays Feishu account startup.

## Best Possible Solution

Keep this open and fix it in the bundled Feishu plugin startup path. The best direction is to start the Feishu account monitor/WebSocket promptly even when the bot-identity/API probe is slow, move identity lookup/recovery to a bounded background path, keep requireMention group handling fail-closed until bot identity is known, and preserve sequential/rate-limit protections for multi-account recovery.
