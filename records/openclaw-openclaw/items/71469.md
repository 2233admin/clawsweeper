## Summary

Keep open. Current main and the latest tagged release still only print the Gemini CLI OAuth URL when `ctx.openUrl(authUrl)` throws, while the provider auth wrapper treats core opener `false` results as resolved calls, so the silent browser-launch hang described by #70964 remains possible. The supplied PR diff is narrow, has maintainer support in the discussion, and does not appear superseded on main.

## What This Changes

The PR moves Gemini CLI OAuth URL logging before the best-effort browser launch and adds a focused local OAuth regression test for log-before-open ordering.

## Best Possible Solution

Land this PR or an equivalent focused Google plugin change so local Gemini CLI OAuth always prints the authorization URL before waiting for the localhost callback, while preserving the existing browser launch, manual fallback, and token exchange behavior.
