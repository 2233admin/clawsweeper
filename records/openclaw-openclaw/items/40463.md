## Summary

Keep this PR open. Current main has absorbed parts of the Teams media work, including hostedContents /$value fallback, Bot Framework DM routing, and exact smba.trafficmanager.net auth forwarding, but it still does not implement the PR's channel /chats fallback, DM channelData.chatId preference, or fully bounded hosted-content body read.

## Best Possible Solution

Keep the PR open for Microsoft Teams maintainer review. Rebase it onto current main, preserve the shipped Bot Framework DM routing, exact smba.trafficmanager.net auth allowlist, and <attachment id> fallback gate, then land or explicitly reject the channel /chats/{threadId}/messages fallback with Microsoft Graph evidence. The hosted-content /$value path should use readResponseWithLimit and focused tests before merge.
