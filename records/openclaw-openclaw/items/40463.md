## Summary

Keep this PR open. Current main has absorbed several Microsoft Teams media fixes, including hosted-content `$value` fetching, Bot Framework DM attachment routing, and a narrowed auth allowlist, but it still does not implement the PR's channel `/chats/{threadId}/messages` fallback and the current hosted-content `$value` path still buffers unbounded when `Content-Length` is absent.

## Best Possible Solution

Keep this PR open for maintainer cleanup. Rebase it onto current main, retain the already-shipped hosted-content and DM media fixes, add or explicitly reject the channel `/chats/{threadId}/messages` fallback with tests, move the bounded `$value` read into the current `graph.ts` implementation, and replace the tenant-token overload with a typed provider capability or remove it.
