## Summary

Keep this PR open. Current main has absorbed several Microsoft Teams media fixes, including hosted-content `$value` fetching, Bot Framework DM attachment routing, and exact `smba.trafficmanager.net` auth forwarding, but it still does not implement the PR's channel `/chats/{threadId}/messages` fallback and the current `$value` path still buffers unbounded when `Content-Length` is absent. The PR also still needs maintainer cleanup around the tenant-scoped token-provider contract before it is safe to merge.

## Best Possible Solution

Keep this PR open for maintainer cleanup. Rebase it onto current main, keep the already-shipped hosted-content, Bot Framework DM, diagnostics, and exact-auth-allowlist fixes, then either implement or explicitly reject the channel `/chats/{threadId}/messages` fallback with focused tests. Move the hosted-content `$value` body read to `readResponseWithLimit`, and replace the tenant-token overload with a typed provider capability or remove that path.
