## Summary

Keep this PR open. Current main has related Control UI i18n startup and transcript-repair work, but it still does not definitely implement this PR’s two stated changes: a first-render i18n readiness gate and a tool_use/tool_result mismatch recovery hint if that provider rejection reaches error formatting.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with two focused updates: add a deterministic Control UI i18n readiness path that loads the initial locale before first render and applies settings locale authoritatively, and add a narrowly scoped tool_result/tool_use mismatch recovery classifier only if current transcript repair still leaves a provider-rejection path. Address the existing review comments and add regression tests before merge.
