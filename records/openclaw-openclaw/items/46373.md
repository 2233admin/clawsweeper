## Summary

Keep this PR open. Current main has related transcript-repair mitigations and saved-locale loading, but it still does not definitely implement this PR’s two intended changes: a first-render Control UI i18n readiness gate and a mismatch-specific tool_use/tool_result recovery hint. The existing review comments also identify issues that would need fixing before this PR could land as-is.

## Best Possible Solution

Keep this PR open for maintainer follow-up, or replace it with two focused changes: add a deterministic Control UI i18n readiness path that awaits initial locale loading and applies settings locale authoritatively before first render, and add a narrowly scoped tool_result/tool_use mismatch classifier only if current transcript repair can still surface that provider rejection. Address the existing review comments and add regression tests before merge.
