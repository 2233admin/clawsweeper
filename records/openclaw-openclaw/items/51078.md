## Summary

Keep this PR open. Current main still does not implement the PR's two intended Android behavior fixes: the Screen tab still renders no first-visit content in the active-tab branch, and initial chat load still uses the default 50-session bootstrap path. The PR is not obsolete, but it still has actionable review work before merge.

## Best Possible Solution

Keep this PR open and route it through normal Android review: keep the chat change on the single bootstrap `sessions.list` path with the higher initial limit, revise the Screen tab first-open implementation so it composes in one stable keep-alive slot without WebView teardown/flicker, and add focused regression coverage or explicit Android manual verification for both behaviors.
