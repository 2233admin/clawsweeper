## Summary

Keep open. Current main has changed the local model-run path since the report, but the reported OpenRouter/status and fallback visibility gaps are still present: `model.run` JSON still returns empty attempts, `models status` can still build the double-prefixed OpenRouter label, and the directly related fix PRs were closed or remain separate open review items rather than merged fixes.

## What This Changes

The issue asks for `openclaw infer model run` to honor a configured OpenRouter primary such as `openrouter/auto` or `openrouter/@preset/...` and to show fallback attempt details when a fallback model is used.

## Best Possible Solution

Define the intended `infer model run` contract for default-model probes, fallback use, and diagnostics; then implement it consistently. The end state should honor configured OpenRouter refs, keep model/status labels round-trippable, and either surface sanitized fallback attempts for both local and gateway transports or document that local raw probes do not use fallback chains.
