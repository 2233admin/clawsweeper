## Summary

Keep open. Current main still renders the Raw config surface as a native textarea and has no CodeMirror/Monaco/Ace/jsoneditor integration, folding support, syntax highlighting, or JSON validation implementation. The related PR #40732 is open and appears to improve readability/status context, but it does not supersede the requested structured JSON code editor with folding/highlighting/validation on main.

## Best Possible Solution

Keep the issue open. Implement a first-class Raw config JSON/JSON5 editor in the Control UI, likely by introducing a focused component under `ui/src/ui/components` or the existing config view layer, backed by CodeMirror v6 or an equivalent editor. Preserve `rawAvailable`, sensitive-value reveal/redaction, `onRawChange`, Reset to saved, `baseHash`, and `config.set` / `config.apply` semantics, then update `ui/src/styles/config.css`, `ui/src/ui/views/config.browser.test.ts`, and controller tests to cover folding/highlighting/formatting/validation without regressing existing raw save/apply behavior.
