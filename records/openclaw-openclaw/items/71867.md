## Summary

Keep open. Current main still reports doctor auth health from the implicit/default auth store with generic note titles, while the related implementation PR #71923 is open, unmerged, and still has review feedback to resolve.

## What This Changes

Request that `openclaw doctor` identify which per-agent auth store it is reporting when multiple agent stores have divergent auth state.

## Best Possible Solution

Review and land #71923 after its remaining review feedback is addressed, or land an equivalent narrow doctor/auth fix that enumerates configured agent stores, labels multi-agent diagnostics, keeps single-agent output stable, and adds regression coverage for divergent auth stores.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agents with divergent `auth-profiles.json` stores, then current main loads the unscoped/default auth profile store and emits generic auth note titles.

## Solution Assessment

Yes. A focused doctor/auth reporting fix is the right shape; it should reuse existing agent-scope and auth-store helpers rather than changing auth inheritance, credential selection, or provider runtime semantics.
