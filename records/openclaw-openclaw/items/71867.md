## Summary

Keep open. Current main still uses an implicit default auth store and generic doctor auth note titles, and #71923 is an open implementation candidate for this exact bug, so there is no conservative close reason yet.

## What This Changes

Have `openclaw doctor` identify which agent store each model-auth, auth-cooldown, and OAuth-refresh diagnostic comes from when multiple agent auth stores exist.

## Best Possible Solution

Review and land #71923, or an equivalent narrow fix, so doctor enumerates the relevant agent stores, labels multi-agent auth diagnostics, preserves stable single-agent output, scopes refresh attempts to the inspected store, and adds divergent-store regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agents with divergent `auth-profiles.json` stores, then current main runs doctor auth health through `ensureAuthProfileStore(undefined, ...)` and reports generic auth note titles.

## Solution Assessment

Yes for the fix direction. The maintainable solution is a focused doctor/auth reporting change that reuses existing agent-scope helpers and read-only auth-store loading, without changing auth inheritance or provider runtime semantics.
