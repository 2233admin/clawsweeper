## Summary

Keep open. Current main still reports doctor model-auth health through one unlabeled auth-store path, while the related open PR #71923 is the active implementation candidate and should be reviewed or landed before this issue closes.

## What This Changes

The issue requests that `openclaw doctor` identify which agent's auth store it is reporting, or at least flag divergent per-agent auth state.

## Best Possible Solution

Review and land #71923 or an equivalent narrow doctor/auth change that enumerates configured agents, resolves each agent directory, labels multi-agent auth diagnostics, preserves the existing single-agent output, and adds regression coverage for divergent stores.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agents with divergent `auth-profiles.json` stores, then `openclaw doctor` reaches `noteAuthProfileHealth` once with no agent directory and emits generic auth note titles.

## Solution Assessment

Yes. The best fix is a focused doctor/auth reporting change using existing agent-scope helpers, not a change to core auth inheritance, credential validation, or provider runtime semantics.
