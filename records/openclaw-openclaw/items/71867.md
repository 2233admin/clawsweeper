## Summary

Keep open. Current main still runs doctor model-auth health against an implicit/default auth profile store and emits generic auth note titles, while the already-linked open PR #71923 directly targets this bug and should be reviewed, landed, repaired, or closed by maintainers.

## What This Changes

Request that `openclaw doctor` identify the agent store behind model-auth, cooldown, and OAuth refresh diagnostics when multiple per-agent auth stores exist.

## Best Possible Solution

Keep this issue open until #71923 or an equivalent focused fix lands. The desired end state is doctor enumerating relevant agent stores, labeling multi-agent auth diagnostics by agent, preserving stable single-agent output, scoping refresh attempts to the inspected store, and adding divergent-store regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agents with divergent `auth-profiles.json` stores, then current main loads doctor auth health through `ensureAuthProfileStore(undefined, ...)` and reports generic note titles.

## Solution Assessment

Yes. The narrow maintainable solution is a doctor/auth reporting fix that reuses existing agent-scope helpers and read-only auth-store loading, without changing auth inheritance, credential selection, or provider runtime semantics.
