## Summary

Keep open. Current main still reports doctor auth health from one implicit/default auth store with generic note titles, and the open related PR #71923 is an active implementation candidate rather than merged proof.

## What This Changes

The issue requests that `openclaw doctor` identify which agent store produced auth health output when multiple per-agent auth stores exist.

## Best Possible Solution

Review and land #71923, or an equivalent narrow doctor/auth fix that enumerates configured agents, resolves each agent directory, labels multi-agent auth diagnostics, keeps single-agent output stable, and adds regression coverage for divergent auth stores.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agents with divergent `auth-profiles.json` stores, then the current doctor auth flow loads `undefined` agent context and emits generic auth titles.

## Solution Assessment

Yes. A focused doctor/auth reporting fix is the right shape; it should use the existing agent-scope helpers and auth store APIs rather than changing core auth inheritance or credential semantics.
