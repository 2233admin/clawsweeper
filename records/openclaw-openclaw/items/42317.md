## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: runtime config continues to rewrite a configured session.mainKey to "main", core route builders still hard-code DEFAULT_MAIN_KEY, and the current Telegram/WhatsApp plugin paths still need the configured key ported in. The PR remains a relevant implementation candidate for the linked feature request, but it is not merge-ready without a rebase/port and a delivery-guard fix.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, port the Telegram/WhatsApp hunks to the current extensions/ paths, preserve configured session.mainKey through runtime defaults and every route/session-key builder, update delivery guards to recognize configured custom main keys and legacy aliases, align docs/schema help, and add focused config/routing/plugin regression coverage before maintainer review or merge.
