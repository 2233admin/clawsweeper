## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: runtime materialization rewrites a configured `session.mainKey` to `"main"`, core route builders still hard-code `DEFAULT_MAIN_KEY`, and current Telegram/WhatsApp plugin paths still do not pass a configured main key through their route/session-key builders. The PR is still a relevant implementation candidate, but it needs a rebase/port onto current main plus delivery-guard and docs/schema alignment before maintainer review or merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, port the Telegram/WhatsApp hunks to the current plugin paths, preserve configured `session.mainKey` through runtime defaults and every route/session-key builder, update delivery guards to recognize configured custom main keys and legacy aliases, align docs/schema help, and add focused config/routing/plugin regression coverage before maintainer review or merge.
