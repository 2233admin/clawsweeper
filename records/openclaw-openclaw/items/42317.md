## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: runtime materialization still rewrites configured `session.mainKey` to `"main"`, core route builders still hard-code `DEFAULT_MAIN_KEY`, outbound routing cannot pass a configured main key, and the current Telegram/WhatsApp plugin paths still need the configured key ported. The PR remains a relevant implementation candidate for the linked custom-main-session feature, but it needs a rebase/port plus the delivery-guard fix before maintainer review or merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, port the Telegram/WhatsApp hunks to the current `extensions/` paths, preserve configured `session.mainKey` through runtime defaults and every route/session-key builder, update delivery guards to recognize configured custom main keys and legacy aliases, align docs/schema help, and add focused config/routing/plugin regression coverage before maintainer review or merge.
