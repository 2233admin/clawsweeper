## Summary

Keep open. Current main still does not implement this PR’s central doctor behavior: when `cron.webhook` is unset, `notify: true` is warned about and left in place. The PR remains potentially useful, but it needs a focused rebase/split because the provided diff also bundles unrelated provider secret-normalization and test-drive-by changes.

## Best Possible Solution

Keep the PR open, but ask for a focused rebase or replacement PR against current main: update `migrateLegacyNotifyFallback` to remove legacy `notify` when `cron.webhook` is unset and delivery can be preserved, keep warnings for announce/unknown modes as appropriate, add targeted `src/commands/doctor-cron.test.ts` coverage, and update doctor/cron docs if behavior changes. Split or drop the provider secret flip-flop work into a separate security-reviewed PR.
