## Summary

Keep open. The manager-level registration change is plausibly aimed at the reported cold-provider failure, but the PR still carries a stale source-bridge rewrite and lacks the required user-facing changelog entry; there is also overlapping open CLI-focused work in #73769 that maintainers should reconcile before landing one canonical fix.

## What This Changes

This PR adds lazy built-in memory embedding provider registration before `MemoryIndexManager` creates a provider, adds an `ensureBuiltInMemoryEmbeddingProvidersRegistered` helper, and rewrites the source memory-host SDK embeddings bridge.

## Best Possible Solution

Land one canonical cold-registry fix for local memory embeddings, preserve `src/memory-host-sdk/engine-embeddings.ts` as the package bridge, add targeted regression coverage for the accepted entrypoint, and include a `CHANGELOG.md` fix entry crediting the contributor.

## Reproduction Assessment

Unclear against current `main`. The linked issue gives a high-quality installed-package reproduction using `openclaw memory status --deep` with `memorySearch.provider = "local"`, but I did not live-run that path and current `main` has a manifest/capability fallback that could change the outcome.

## Solution Assessment

Unclear as submitted. Manager-level registration is a plausible narrow fix for all manager callers, but the source bridge rewrite should be removed and maintainers should choose between this manager-level approach and the CLI-focused #73769 path with tests.
