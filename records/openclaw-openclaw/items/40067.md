## Summary

Keep #40067 open. Current main still has no Playwright dialog-race suppression, so the reported crash remains plausible, but the PR’s current shape hard-codes Playwright-specific handling in generic core unhandled-rejection infra. The prior maintainer comment is still aligned with the code: this should be rewritten as browser-plugin-owned handling through the existing plugin SDK rejection-handler seam.

## Best Possible Solution

Keep the PR open for a rewrite. Move the Playwright dialog-race predicate into the browser plugin, register it when the browser control/runtime service starts, unregister it on stop, and add browser-plugin tests for the exact `Protocol error (Page.handleJavaScriptDialog): No dialog is showing` message plus wrapped or nested error forms. Core unhandled-rejection infra should remain generic.
