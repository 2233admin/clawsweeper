## Summary

Keep this PR open for maintainer review. Current main still lacks the requested doctor/FAQ clarification, but the proposed auto-mode guidance conflicts with the runtime contract by telling users that `provider = "auto"` can use an `hf:` model URI; runtime auto-selection only accepts an existing local file for local embeddings.

## What This Changes

The PR updates `openclaw doctor` local memory-search guidance and FAQ text for node-llama-cpp/GGUF embeddings, adds regression assertions, and reformats one subagent workspace test block.

## Best Possible Solution

Land a narrow version that aligns doctor guidance, helper behavior, tests, and FAQ text with the memory embedding contract: explicit `provider = "local"` may use the default downloadable GGUF or `hf:` refs, `provider = "auto"` should only present local as ready for an existing local file, Ollama remains explicit, and pnpm build-approval guidance stays scoped to node-llama-cpp.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
