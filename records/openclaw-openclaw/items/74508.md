## Summary

Keep open. Current main still stops Telegram draft previews once rendered text exceeds the Telegram limit, and this draft PR targets the right Telegram plugin surfaces, but the proposed patch is not merge-ready because completed chained reasoning chunks are routed into cleanup deletion and requested lane rotation can be lost during 429 backoff.

## What This Changes

The PR updates Telegram draft streaming to split overlong reasoning previews into continuation messages, label reasoning continuations, add shared edit throttling and 429 backoff handling, suppress partial reasoning-tag leakage, and extend Telegram streaming tests.

## Best Possible Solution

Revise the Telegram draft-stream design so completed overflow chunks have an explicit retained lifecycle separate from stale/superseded previews, and make rate-limit backoff preserve any requested lane rotation until it can be applied. The revised PR should include dispatch-level regression coverage for final cleanup after chained reasoning and fake-timer coverage for 429 plus lane rotation.

## Reproduction Assessment

Yes. The current-main overflow behavior can be reproduced by driving createTelegramDraftStream with rendered text over maxChars; the PR-specific cleanup regression can be reproduced with a dispatch test that overflows reasoning, finalizes the turn, and observes archivedReasoningPreviewIds deletion. The 429 regression can be reproduced with fake timers by triggering forceNewMessage while rateLimitedUntilMs is active.

## Solution Assessment

No. The PR is aimed at the narrow Telegram plugin surfaces, but this patch is not the best mergeable solution because it reuses stale-preview cleanup semantics for retained chained chunks and drops requested rotation during 429 backoff. A safer solution needs explicit retained-preview state and a pending-rotation flag or equivalent replay path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Retain chained reasoning chunks during cleanup:** `extensions/telegram/src/draft-stream.ts:379-382`
  - body: The overflow path reports intentionally completed reasoning chunks through onSupersededPreview, but Telegram dispatch records reasoning previews from that callback in archivedReasoningPreviewIds and deletes them during final cleanup. Long reasoning would briefly chain, then remove completed chunks except the active last one; give overflow chunks a retained lifecycle or exclude them from stale-preview cleanup.
  - confidence: 0.91
- **[P2] Preserve lane rotation through 429 backoff:** `extensions/telegram/src/draft-stream.ts:543-549`
  - body: forceNewMessage returns during active 429 backoff without recording that rotation is still needed. onReasoningStream then resets the lane and clears splitReasoningOnNextStream, so after backoff the retry edits the old reasoning message instead of starting the requested new one.
  - confidence: 0.84
