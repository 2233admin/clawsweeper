## Summary

Keep open. Current main still stops Telegram draft previews when the rendered text exceeds the Telegram message limit, so this PR is not obsolete; however, the current PR head still has a blocking 429 retry-state bug that should be fixed before merge.

## What This Changes

This draft PR updates the Telegram plugin's draft streaming path to chain overflowing reasoning previews into continuation messages, label reasoning continuations, suppress partial reasoning-tag answer leakage, add shared edit/typing throttles with 429 backoff, and extend Telegram tests.

## Best Possible Solution

Keep the overflow-chaining direction, but revise the branch so retryable 429 failures do not poison duplicate-send state, then add a regression test where a rate-limited draft send backs off and actually sends or edits the same pending preview text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Make 429 backoff actually resend the pending preview:** `extensions/telegram/src/draft-stream.ts:455-465`
  - body: The 429 path sleeps and returns `false`, so `createDraftStreamLoop` restores the same pending text for a later flush. But this function already set `lastSentText` and parse mode before Telegram rejected the call, so retrying identical text can hit the duplicate check and exit without sending or editing. Move those markers after a successful API call or clear them before returning retryable failure, and cover the retry case.
