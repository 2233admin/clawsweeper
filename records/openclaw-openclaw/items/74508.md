## Summary

Keep open. Current main still stops Telegram draft previews when the rendered text exceeds the Telegram message limit, so the PR addresses a real gap, but the current branch has a blocking cleanup regression: the new overflow path reports intentional reasoning chunks through the superseded-preview callback, and Telegram dispatch deletes superseded reasoning preview IDs at turn cleanup. The existing ClawSweeper comment already flagged this P2 interaction, and I confirmed it against PR head `bb346e33297463722df5aa55f08e7a7dd6b22b44` and current main `cab86dc32541a664b8048f62bd47a0ada294fc41`.

## What This Changes

The PR changes Telegram draft streaming to split overlong reasoning previews across continuation messages, add a reasoning continuation renderer, throttle and rate-limit concurrent draft lanes with 429 retry handling, and add Telegram streaming tests.

## Best Possible Solution

Revise the PR so intentional reasoning overflow chunks are retained while true stale or superseded previews are still cleaned up, preferably by separating overflow-retention from superseded-preview cleanup or by passing an explicit retention signal. Add a dispatch-level regression that drives a long reasoning stream through cleanup and verifies all chained reasoning messages remain visible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve chained reasoning chunks during cleanup:** `extensions/telegram/src/draft-stream.ts:379-382`
  - body: The overflow path now reports an intentionally completed reasoning chunk through `onSupersededPreview`. Telegram dispatch treats reasoning previews from that callback as archived cleanup targets and deletes them in the final cleanup loop, so a long reasoning stream would briefly chain and then lose every completed chunk except the active last one. Keep intentional overflow chunks retained, or use a separate callback/flag, and cover it with a dispatch cleanup regression.
  - confidence: 0.9
