## Summary

Keep open. Current main still stops Telegram draft previews once the rendered text exceeds Telegram's 4096-character limit, so the PR addresses a real remaining gap, but the latest diff still has blocking overflow-state and cleanup regressions before it should merge.

## What This Changes

This PR changes Telegram draft streaming to split oversized previews into chained messages, label reasoning continuation previews, suppress partial reasoning-tag answer leakage, scale draft throttles for concurrent lanes, and add Telegram tests.

## Best Possible Solution

Revise the branch so overflow-internal rotations preserve the active split offset, public fresh-message rotations reset it for a new logical preview, and overflow-created preview messages are either tracked for cleanup or scoped to the reasoning lane with explicit retention semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reset overflow offsets for new logical previews:** `extensions/telegram/src/draft-stream.ts:323`
  - body: After an overflow chain, `textBaseOffset` remains nonzero. Existing callers use `forceNewMessage()` to start unrelated fresh previews, so the next update can be sliced from the old offset and lose the beginning of the new message, or send nothing if it is shorter than that offset. Split the internal overflow rotation from the public fresh-message reset, and cover overflow followed by `forceNewMessage()` plus a short update.
  - confidence: 0.9
- **[P2] Track chained answer previews before final cleanup:** `extensions/telegram/src/draft-stream.ts:345`
  - body: This generic overflow path can also run for the answer lane. When it calls `forceNewMessage()` internally, the previous preview message ID is dropped from the stream but is not added to `archivedAnswerPreviews`; final cleanup only clears the active stream and known archives, so long answer drafts can leave earlier preview chunks in chat after the final answer is sent. Either make overflow rotations observable to lane cleanup or restrict chaining to reasoning messages that are intentionally retained.
  - confidence: 0.82
