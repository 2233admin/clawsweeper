## Summary

Keep open: current `main` still has the reported Telegram preview overflow stop, and the PR is directionally useful, but the latest patch is not merge-ready because intentionally chained reasoning chunks are routed through the existing superseded-preview cleanup path and will be deleted at turn cleanup.

## What This Changes

This draft PR updates Telegram draft streaming to split overlong reasoning previews into continuation messages, adds continuation rendering and 429 retry handling, and extends Telegram streaming tests.

## Best Possible Solution

Revise the PR so intentional reasoning overflow chunks are retained while true superseded or stale previews are still cleaned up, then add a regression that drives a long reasoning stream through dispatch cleanup and verifies all chained messages remain visible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve chained reasoning chunks during cleanup:** `extensions/telegram/src/draft-stream.ts:379-383`
  - body: The overflow path now calls `onSupersededPreview` for an intentionally completed chunk. Telegram dispatch records reasoning previews from that callback in `archivedReasoningPreviewIds` and deletes them in the finally cleanup, so a long reasoning stream will lose each completed chunk at turn end and leave only the last slice visible. Keep intentional reasoning overflow chunks retained, or use a distinct callback/retention flag, and add a dispatch cleanup regression.
  - confidence: 0.9
