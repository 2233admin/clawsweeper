## Summary

Keep this PR open. Current main still does not implement the requested `agent.wait` usage/cost metadata: the terminal snapshot and both `agent.wait` response paths only return status/timing/error and newer liveness fields. The underlying agent runtime already carries `usage` and `lastCallUsage`, so this remains a real gateway wire-protocol gap rather than an obsolete PR. The PR should be refreshed and re-reviewed against current main before landing.

## Best Possible Solution

Keep the PR open and refresh it against current main. The best implementation is an additive gateway protocol change that threads existing `EmbeddedPiAgentMeta` usage through `AgentWaitTerminalSnapshot`, preserves current terminal fields, makes cost calculation best-effort, adds regression tests for cached and lifecycle/dedupe race paths, and updates the public gateway/agent-loop docs.

## Risks / Open Questions

- The PR branch is older than current main and the wait snapshot has since gained `stopReason`, `livenessState`, and `yielded`; a refreshed implementation should preserve those fields and add focused tests for both cached and in-flight wait paths.
- Review comments in the provided context flagged nondeterministic metadata when lifecycle wins the wait race and a concern that optional cost metadata must not turn a successful agent run into an error response; those should be addressed before merge.
- Related open issue #68066 questions cost accuracy for some OpenRouter tier-priced models, so this PR should treat `costUsd` as estimated metadata or coordinate with that follow-up rather than promising authoritative billing.
