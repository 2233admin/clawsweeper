## Risks / Open Questions

- The PR is stale against current main: current `agent.wait` now forwards `stopReason`, `livenessState`, and `yielded`, while the PR diff does not preserve those fields in the added response shapes.
- The proposed cost calculation calls config/pricing utilities in the successful run path; metadata should remain best-effort so a cost lookup/config failure cannot turn a completed run into an error response.
- The PR appears to add `meta` to the direct `agent` terminal response as well as `agent.wait`; that may be reasonable, but it is a protocol/API broadening that needs docs and tests.
- Security review found no supply-chain or secret-handling changes in the diff, but exposing `sessionId`, provider/model, token counts, and cost over gateway responses should remain limited to the intended authenticated/operator surface and be documented.
- Related #68066 shows cost semantics can be subtle for OpenRouter/tiered billing; the exposed `costUsd` field should be documented as estimated unless it is made authoritative.
