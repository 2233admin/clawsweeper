## Summary

Keep this PR open. Current main still does not expose per-run usage, cost, provider/model, or session metadata from `agent.wait`; the PR remains a focused implementation candidate for the closed feature request #49494, but it needs refresh against current terminal fields and the existing review concerns before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is an additive gateway protocol update that threads existing `EmbeddedPiAgentMeta` into `AgentWaitTerminalSnapshot`, returns optional session/usage/last-call/provider/model/estimated-cost metadata from both cached and lifecycle-race `agent.wait` paths, keeps cost estimation best-effort, preserves current terminal fields, and adds focused tests plus docs for the new response shape.

## Risks / Open Questions

- The PR branch must be refreshed to preserve current main terminal fields such as `stopReason`, `livenessState`, and `yielded` while adding metadata.
- Existing review comments identify important correctness risks: lifecycle-vs-dedupe races can make metadata nondeterministic, and optional cost estimation must not turn a successful run into an error response.
- The additive gateway wire-contract change should land with focused regression tests and protocol-facing docs; plugin runtime wrappers may also need a deliberate decision if `waitForRun` should expose the new metadata.
