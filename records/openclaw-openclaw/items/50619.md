## Summary

Keep open. Current main still switches the session model and acknowledges it before any target-model context/rate-limit risk comparison; existing context notices and pre-prompt compaction happen elsewhere and do not satisfy the requested switch-time warning.

## What This Changes

Request `/model` to estimate current session context and warn or compact before switching to a target model that may have tighter context or rate-limit budgets.

## Best Possible Solution

Add a shared model-switch context-risk check used by WebChat and messaging directives: resolve the target model's effective contextTokens/contextWindow/maxTokens, compare it with the freshest available prompt/session usage, and recommend `/compact` or `/new` when the current session is near or above a conservative threshold. Treat automatic compaction and provider/account-specific TPM prediction as separate maintainer decisions.
