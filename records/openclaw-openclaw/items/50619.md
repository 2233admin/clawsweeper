## Summary

Keep open. Current main still changes the session model and acknowledges `/model` without estimating the existing session context against the selected model's effective budget or warning about the reported large-request TPM/rate-limit case. Existing context safeguards run later in the agent path, and the WebChat context notice is tied to the current active-session snapshot rather than a target-model switch preflight.

## What This Changes

The issue requests a `/model` switch-time check that estimates current session context, compares it with target model limits and rate-limit risk, then warns or offers compaction before switching.

## Best Possible Solution

Add a shared model-switch context-risk warning used by WebChat and messaging directives: resolve the selected model's effective `contextTokens` and output reserve, compare that with the freshest available session prompt usage, and recommend `/compact` or `/new` when the session is near or above a conservative threshold. Keep automatic compaction and provider/account-specific TPM prediction as explicit maintainer decisions or follow-ups.
