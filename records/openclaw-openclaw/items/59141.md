## Summary

Keep open. Current main still has memory-lancedb recall/search ordered by LanceDB vector similarity only, with no freshness-sensitive candidate widening, recency reranking, timestamp/provenance hints, or stale-vs-newer regression coverage. The related bug #59130 remains a real memory-lancedb behavior gap, while this PR is plausible but still needs review-comment resolution before it could land.

## Best Possible Solution

Keep the PR open and either revise it into a narrower memory-lancedb freshness fix or land an equivalent implementation on main: detect freshness-sensitive recall, widen candidates only for those prompts, rerank with createdAt/source timestamps while preserving semantic relevance, surface timestamp/provenance cues in injected/tool recall results, and add regression tests for stale-vs-newer ordering plus non-freshness fallback ordering.
