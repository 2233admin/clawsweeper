## Summary

Keep open. Current main still matches the reported behavior: allowed elevated sessions fall back to `on`, omitted per-call `elevated` inherits that active elevated default, and elevated exec target resolution routes out of the sandbox to `gateway` unless the target is already `node`. The docs and tests now describe that as the current contract, and the related default-off PR is closed unmerged, so this needs maintainer product/security review rather than cleanup closure.

## What This Changes

The issue requests that enabling `tools.elevated` preserve sandbox exec as the default and make gateway/node elevation opt-in per `exec(elevated: true)` call, with `agents.defaults.elevatedDefault` defaulting to `off`.

## Best Possible Solution

Maintainers should explicitly choose the elevated-mode contract. If sandbox-default opt-in elevation is accepted, update directive resolution, exec elevated-mode handling, prompt/status wording, public docs, and regression tests so enabling `tools.elevated` alone does not route omitted-elevated exec calls to the host. If default-on elevated sessions are intentional, document that security decision clearly before closing.
