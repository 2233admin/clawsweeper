## Summary

Keep open. Current main still matches the reported routing chain: an allowed elevated session with no explicit directive, session state, or `agents.defaults.elevatedDefault` falls back to `on`; omitted `exec` calls inherit that elevated default; and elevated target resolution routes to `gateway` unless the selected target is already `node`. The linked default-off PR was closed unmerged, while docs and tests currently encode default-on and `/elevated ask` as an alias for `on`, so this needs maintainer product/security judgment rather than stale cleanup.

## What This Changes

The issue requests sandbox-default elevated exec semantics: enabling `tools.elevated` should not route non-elevated `exec` calls to the gateway, and elevation should be opt-in per call or explicit session directive.

## Best Possible Solution

Maintainers should decide the elevated-mode contract explicitly. If sandbox-default opt-in elevation is desired, update directive resolution, session persistence/status, exec default inheritance, prompt wording, docs, and regression tests together. If default-on host exec is intentional, keep the implementation but make the exec docs and security guidance state the operator tradeoff clearly.
