## Summary

Current main still does not implement the central request. `agent.wait` can still return a plain timeout without phase attribution, the protocol and dedupe snapshot types have no queue/preflight/provider/post-turn timeout fields, embedded-run state does not track the requested timing phases, and auth-profile failure policy suppresses every timeout rather than counting only provider-started timeouts. The related implementation PR #65511 is closed unmerged, so there is no open implementation candidate to let GitHub resolve this issue automatically.

## Best Possible Solution

Keep this open. Implement a focused core/gateway change that records embedded-run timing phases, carries a phase-aware timeout reason through lifecycle, dedupe, and `agent.wait`, and changes auth-profile failure accounting so only provider-started timeouts affect shared cooldown/rotation. Add regression coverage for queued-never-started runs, provider-started timeout fallback/cooldown behavior, and terminal snapshot preference; update auth/failover docs if the public contract changes.
