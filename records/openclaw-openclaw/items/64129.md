Keep open. Current main still has the reported paid-background-use path: Google API-key setup can make Gemini the primary default, and heartbeat inherits the resolved default model unless the user has already configured a heartbeat-specific model. ## What This Changes

The issue requests that configuring a paid Google Gemini API key not silently turn existing heartbeat runs into paid Gemini background usage.

## Best Possible Solution

Add an explicit paid-background-use boundary: provider setup should preserve an existing primary model unless the user explicitly confirms switching it, and heartbeat should not inherit a newly configured paid external provider silently. Cover Google API-key setup and populated HEARTBEAT.md heartbeat inheritance with regression tests.

## Repair Work Prompt

Fix the paid Gemini heartbeat inheritance bug reported here. Current main can configure Google Gemini API-key auth, patch agents.defaults.model to the Google Gemini default, and then heartbeat inherits that default when no heartbeat.model is configured. Implement the conservative boundary that provider setup must not replace an existing primary model with a newly configured paid provider unless the user explicitly selects or confirms the switch. Make sure Google's API-key applyConfig/configPatch path cannot bypass preserveExistingDefaultModel; simply passing the flag through setup is likely insufficient unless the configPatch path preserves the original primary too. It is acceptable for setup to set Google as primary when no primary model exists. Keep Google-specific policy in the Google plugin/provider-auth seam; do not add bundled Google ids to unrelated core paths. Add regression coverage for applyAuthChoice with gemini-api-key plus an existing Anthropic/OpenAI primary, setup wizard with an existing primary, and the heartbeat inheritance expectation that no heartbeat.model means heartbeat uses the current default. Add a user-facing changelog Fixes entry crediting Thanks @vegapunk9527 unless maintainers decide the final patch is internal-only.
