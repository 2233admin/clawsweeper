## Summary

Keep open. Current main still has the reported path: Google API-key setup can install Gemini as the primary default model, and heartbeat runs use the resolved default model unless a heartbeat-specific override is configured. Current docs/templates reduce the chance for brand-new empty HEARTBEAT.md workspaces, but they do not protect existing populated heartbeat files or require explicit confirmation before paid background usage.

## What This Changes

Prevent configuring a paid Google Gemini key from silently moving existing heartbeat/background turns onto a paid Gemini default before explicit user intent.

## Best Possible Solution

Add an explicit paid-background-use boundary: Google/provider setup should preserve an existing primary model unless the user explicitly confirms switching it, Google applyConfig must not bypass that preservation, and heartbeat should avoid silently inheriting a newly configured paid external provider for existing populated HEARTBEAT.md workspaces. Cover the setup and heartbeat inheritance path with targeted regression tests.
