## Summary

Current main does not implement this PR’s intended fix. The linked #73460 bug remains plausible on the browser-direct Google Live transport: interruption handling only resets the playback cursor, while scheduled AudioBufferSourceNodes are not retained or stopped. The PR is also paired with an open same-author issue using closing syntax, so it should stay open for maintainer review rather than be swept closed.

## Best Possible Solution

Keep this PR open as the implementation candidate for #73460. The best path is to review the small browser audio-source lifecycle change, preferably add a focused regression test that schedules output then sends serverContent.interrupted, and land it if CI confirms the Google Live browser Talk path behaves like the Gateway relay clear path.
