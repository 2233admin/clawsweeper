## Summary

Keep open. Current main still lacks the PR's intended startup/provider-registry reductions: gateway startup does not pass workspace-scoped entry-only provider discovery through models.json, models-json planning does not carry an entry-only discovery flag, and capability-provider fallback registry loads still omit `onlyPluginIds`. The PR is active, paired with open performance reports #73835 and #73729, and intentionally complements the separate open web-provider cache PR #73847.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review it alongside #73847, verify the focused startup/provider tests and changed gate, then either land the scoped fix for #73835/#73729 or request targeted changes around the load-scope/cache behavior if needed.
