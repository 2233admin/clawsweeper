## Summary

Keep open. Current main still uses a module-local sandbox backend registry, while plugin-facing registration is exposed through `openclaw/plugin-sdk/sandbox` and core later resolves the backend through `requireSandboxBackendFactory`. The same-author fix PR #67623 is still reported open/unmerged in the supplied GitHub context, so this issue should remain open for maintainer review rather than being closed by cleanup.

## Best Possible Solution

Keep this issue open and review #67623, or a maintainer-authored equivalent. The fix should make sandbox backend registrations shared across plugin/core module instances, preferably with a direct process-global registry appropriate for mutable live state, and add a regression test proving registration through `openclaw/plugin-sdk/sandbox` is visible to core sandbox context resolution. Route the change through sandbox/security owners.
