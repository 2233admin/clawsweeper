## Summary

Current main still uses a module-local sandbox backend registry, while sandbox backends are exposed to plugins through `openclaw/plugin-sdk/sandbox` and resolved later by core. The same-author implementation candidate #67623 is still open and unmerged, so this issue should remain open for maintainer review.

## Best Possible Solution

Keep this issue open. Review #67623, or a maintainer-authored equivalent, to move sandbox backend registrations to a process-global registry that is shared across plugin/core module instances. Add a regression test that registers a backend through `openclaw/plugin-sdk/sandbox` and verifies core sandbox context resolution can require that backend.
