## Summary

Keep open. Current main still treats Feishu group `/think medium` as unauthorized unless the group sender is allowed by Feishu command authorization, and unauthorized directive-only or command-only messages still produce no visible rejection. The reporter confirmed the authorization workaround fixes execution, but the remaining UX problem is real and the related PR #49821 is closed unmerged.

## Best Possible Solution

Keep this open and implement a focused Feishu/group command UX fix: when an explicitly mentioned Feishu group command or directive-only message such as `/think medium` is blocked by command authorization, send a non-sensitive visible rejection that points users toward `commands.allowFrom`, `channels.feishu.allowFrom`, or `channels.feishu.groups.<chat_id>.allowFrom`. Preserve existing authorization semantics and update targeted tests/docs.
