## Summary

Keep this open. Current main still has chat lifecycle state and abort/queue controls, but it does not render the requested compact In progress/Interrupted/Done status chip in the Control UI composer, and the broader gateway activity API work in #39127 is still separate open work rather than a shipped substitute.

## What This Changes

Add a compact, accessible, session-scoped runtime status indicator beside the Web UI composer actions showing in-progress, interrupted, and done states without adding transcript messages.

## Best Possible Solution

Implement a small Control UI composer status chip that uses existing chat lifecycle signals now, fails closed to Interrupted when continuity is lost, and can later swap to the gateway-owned session activity API without changing the visible UI contract.

## Repair Work Prompt

Implement the compact runtime status indicator requested by openclaw/openclaw#71630 as a narrow Control UI change. Add a stable composer toolbar chip near the attachment/Talk controls, not a transcript message. Support at least In progress, Interrupted, and Done. Drive it from explicit client lifecycle state, not assistant text: show In progress for the current session while chatSending/chatRunId or finalization is active, show Done briefly after a successful terminal final for the current run, and show Interrupted for abort/error/cancel or reconnect/seq-gap/gateway restart cases where app-gateway clears an active run because continuity was lost. Keep the state session-scoped and reset appropriately on new run/session changes. Do not implement the broader #39127 sessions.activity API in this PR, but leave the UI contract easy to wire to it later. Keep desktop/mobile layout stable, collapse to icon-only or otherwise avoid wrapping on narrow widths, and add aria-label/title text with useful state/timestamp or elapsed detail when available. Update focused tests for rendering, terminal transitions, abort/error/reconnect interruption, and layout/status accessibility.
