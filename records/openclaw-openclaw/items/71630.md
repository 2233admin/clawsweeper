## Summary

Keep this open. Current main still lacks the requested always-visible composer runtime chip, and the existing UI only exposes action controls, Talk-specific status text, and transient compaction/fallback indicators; the broader authoritative activity API is still separately tracked in #39127.

## What This Changes

Request to add a compact Web UI composer status indicator showing In progress, Interrupted, and Done for the active session without adding transcript messages.

## Best Possible Solution

Add a narrow Control UI composer status chip driven by explicit local chat lifecycle state now, fail closed to Interrupted when reconnect or lost-terminal-event continuity is uncertain, and leave the richer gateway-owned activity model to #39127.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Static verification against current main is high-confidence because the composer render path, run controls, reconnect handling, and method list do not expose the requested status indicator.

## Solution Assessment

Yes, a narrow Control UI implementation is the best first step if it uses existing lifecycle signals and does not create transcript messages. The safer long-term source of truth remains the gateway activity API tracked in #39127 rather than bundling that broader API into this UI change.
