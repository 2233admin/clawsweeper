## Summary

Keep open. Current main has useful partial mitigations for session write-lock cleanup and failover classification, but the requested public timeout/stale configuration is still absent and the embedded attempt lock can still cover the model prompt lifecycle. The fresh v2026.4.26 operator report and open overlapping PR #69159 make this a maintainer staging/design item rather than a cleanup close.

## What This Changes

Request to make session write-lock timeout and stale thresholds configurable, narrow the lock lifetime around session-file mutation, and surface lock contention separately from model/provider failover.

## Best Possible Solution

Add a supported session write-lock configuration contract, wire it into embedded attempts and session-store/transcript mutation paths, preserve the typed lock-timeout failover suppression, and split any deeper single-writer or lock-lifecycle redesign into an explicit follow-up if maintainers want to stage the operator-unblocking config first.
