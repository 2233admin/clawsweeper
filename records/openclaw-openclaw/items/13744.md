## Summary

Keep open. Current main has partial improvements: session lock timeout errors are typed and excluded from model-timeout failover, and embedded lock acquisition moved later. The remaining requested behavior is still not implemented: there is no public config/env surface for session write-lock timeout/stale values, and the embedded runner still holds the session write lock across `activeSession.prompt()` until attempt cleanup.

## Best Possible Solution

Keep this issue open until OpenClaw adds a supported session write-lock timeout/stale configuration surface, threads it through embedded run, compaction, transcript rewrite/truncation, and session-store mutation paths as appropriate, and either narrows the lifecycle lock to real transcript critical sections or introduces a documented per-session single-writer queue. Preserve the current `SessionWriteLockTimeoutError` failover suppression and add focused concurrency/config regression coverage.
