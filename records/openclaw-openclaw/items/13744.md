## Summary

Keep open. Current main has partial mitigations for lock cleanup and failover classification, but the requested public timeout/stale configuration is still absent and the embedded attempt still holds the session write lock through prompt execution and end-of-attempt cleanup.

## What This Changes

The issue requests configurable session write-lock timeout/stale settings, a narrower embedded-run lock lifetime, and lock-timeout errors that do not masquerade as model/provider failover.

## Best Possible Solution

Add a supported session write-lock configuration contract with schema, exported types, generated help/metadata, and docs; wire it consistently into embedded attempts and other transcript/session-store mutation paths; keep the typed lock-timeout failover suppression; and split any deeper single-writer or prompt-lifecycle refactor into a clearly scoped follow-up if needed.
