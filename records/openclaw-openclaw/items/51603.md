## Summary

Keep this PR open. Current main already has the chat model picker sending provider-qualified model refs, and it has custom provider setup through `models.providers` plus onboarding helpers, but it does not implement the PR's central top-level `llm` shorthand config key. Because this is a PR, it should only be closed if main definitely implements the intended change; it does not.

## Best Possible Solution

Keep the PR open or ask for a focused revision. The best path is to decide whether OpenClaw wants a top-level `llm` shorthand at all; if yes, implement it in the config owner area with strict schema/type support, SecretInput handling, model-ref-safe provider IDs, supported provider/API validation, non-destructive merging that appends or preserves existing provider models correctly, tests for the existing-provider and fallback cases, and docs for the public config surface. The already-shipped UI model picker behavior can be retained as current-main evidence and does not need to be reintroduced by this PR.
