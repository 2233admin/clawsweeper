## Summary

Keep this issue open. Current main has provider-scoped request timeout plumbing and a model stream idle watchdog, but it still lacks a typed global or per-model total generation timeout for active text-generation loops. The proposed `agents.defaults.models[...]` timeout keys are still rejected by strict config schemas, and the embedded runner still schedules the total abort timer from the run-level timeout rather than an effective per-model generation budget.

## Best Possible Solution

Implement a typed global and per-model text-generation timeout contract. Resolve the effective budget per fallback candidate, enforce it with a timer that does not reset when chunks arrive, abort/log/classify it as a timeout for fallback handling, and update config types, zod/generated schema, help/docs, changelog, and focused regression tests. Keep first-token timeout as a separate follow-up unless maintainers intentionally include it.

## Clownfish Work Prompt

Implement per-model and global total generation timeout support for text agent model calls. Current main only has whole-run `agents.defaults.timeoutSeconds`, provider-scoped `models.providers.<id>.timeoutSeconds`, and an idle/no-chunk watchdog. Add a typed config contract for a total generation budget that does not reset when chunks arrive, with a global default and per-model override under `agents.defaults.models["provider/model"]`. Prefer a clear name that cannot be confused with provider request timeout, such as `generationTimeoutSeconds`, unless the repo already has a duration-string helper suitable for `generationTimeout`. Do not repurpose `models.providers.*.timeoutSeconds` and do not bundle first-token timeout unless explicitly scoped. Resolve the effective budget per fallback candidate, enforce it in the embedded attempt/runtime path, abort with timeout classification so fallback/auth-profile handling works, log the timeout with provider/model/run context, and update config types, zod/generated schema, schema help/labels, docs, changelog, and focused tests. Preserve existing behavior when the new field is unset.
