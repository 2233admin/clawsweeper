## Summary

Keep open because current main still lacks the requested typed global/per-model non-resetting generation timeout. The existing agent run timeout, provider request timeout, and stream idle watchdog are adjacent controls, but they do not provide a model-specific total generation budget for a model that keeps streaming reasoning or planning chunks.

## What This Changes

The issue requests global and per-model generation timeout configuration to abort runaway model generations and optionally allow fallback routing.

## Best Possible Solution

Add a backwards-compatible, documented generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective non-resetting budget for each selected fallback candidate, enforce that budget during active generation with clear timeout classification/logging, and leave first-token/provider-skip behavior to separate follow-up work.
