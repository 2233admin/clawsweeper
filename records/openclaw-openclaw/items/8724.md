## Summary

Keep this issue open. Current main has adjacent provider request and idle timeout controls, but it still does not expose or enforce the requested typed global/per-model non-resetting generation timeout for active model output loops.

## What This Changes

Request a global and per-model generation timeout config that aborts long-running model generation loops, reports/logs the timeout, and can feed fallback handling.

## Best Possible Solution

Add a documented generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective budget per fallback candidate, enforce it with a non-resetting timer during active generation, and preserve first-token/provider-skip behavior as separate work.

## Reproduction Assessment

Yes for the missing feature: the proposed per-model key is outside the strict schema, and the current stream watchdog is reset-on-chunk rather than total-duration based. A live Gemini infinite-planning loop is not needed to prove the config/runtime gap.

## Solution Assessment

Yes, a narrow core config/schema plus embedded-runner enforcement change is the best fit. Provider-specific workarounds or first-token timeout changes would solve related but different failure modes.
