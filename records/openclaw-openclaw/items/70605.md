## Summary

Keep open. Current main still does not strip strikethrough from injected context files, so the PR addresses a real prompt-context bug, but the latest PR regex strips raw Markdown before respecting code regions and can remove literal `~~...~~` examples from fenced or inline code.

## What This Changes

This PR updates `sanitizeContextFileContentForPrompt()` to remove Markdown and HTML strikethrough content from injected context files and adds system prompt tests for inline, multiline, tilde-fence, and attributed-HTML cases.

## Best Possible Solution

Revise the sanitizer to remove rendered strikethrough while preserving Markdown code fences and inline code, preferably by masking code regions with the existing `src/shared/text/code-regions.ts` helper or by using a Markdown-aware token pass. Keep the HTML attribute coverage and add regressions for backtick fences, inline code, tilde fences, multiline strikethrough, and attributed HTML tags before merge.
