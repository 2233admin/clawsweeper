## Summary

Keep this PR open. Current main still forwards context file content through a sanitizer that only removes the heartbeat policy block and collapses blank lines; it does not remove markdown or HTML strikethrough content. The PR is a focused implementation candidate for that missing behavior and has recent author follow-up, so it should receive normal maintainer review rather than be closed as obsolete.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Review the sanitizer approach against real Markdown behavior, require regression coverage for markdown strikethrough, HTML tags with attributes, multiline cases, tilde-fenced code, and malformed/unpaired delimiters, then merge or request changes. Track broader context compression/checkpointing work separately under #47668.
