## Summary

Keep this PR open. Current main still does not include the answering provider/model in the per-response usage footer, and the PR remains a focused implementation candidate for the same user problem tracked by open issue #65477.

## Best Possible Solution

Review and rebase this PR, or land an equivalent maintainer patch: extend `formatResponseUsageLine` with optional provider/model inputs, format them through `formatProviderModelRef`, pass `providerUsed`/`modelUsed` from `runReplyAgent`, add focused footer tests, and update the public usage docs if the footer format changes.
