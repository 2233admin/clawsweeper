## Summary

Keep open. Current main has partial generic guards and an opt-in efficient browser snapshot mode, but it still does not implement the browser-specific, task-aware context budgeting requested here. Default/explicit AI snapshots can still carry broad page content into the browser tool result, efficient mode is config- or caller-driven, and no current browser path adapts snapshot detail to remaining model context or fetches details on demand.

## Best Possible Solution

Implement a browser-owned context-budget policy: default agent-facing snapshots to bounded compact/interactive output where appropriate, apply budget limits consistently across AI, role, Chrome MCP, and efficient paths, tighten browser output under context pressure, support selector/ref/detail fetch-on-demand so full page state stays outside the main prompt, and add focused regression tests for browser-tool.actions, resolveSnapshotPlan, /snapshot routing, and repeated browser tool loops.
