## Summary

Keep open. Current main has partial mitigations: bounded AI snapshots, an opt-in/config-driven efficient snapshot mode, selector/frame snapshot options, and generic overflow recovery for oversized tool results. It still does not implement the browser-specific, task-aware context budgeting requested here: explicit AI snapshots remain broad, efficient mode is not universal, and browser output does not adapt to remaining model context or keep full page state out-of-band by default.

## Best Possible Solution

Implement a browser-owned context-budget policy: default agent-facing snapshots to bounded compact/interactive output where appropriate, apply budget limits consistently across AI, role, Chrome MCP, and efficient paths, tighten snapshot detail under context pressure, support selector/ref/detail fetch-on-demand so full page state stays outside the main prompt, and add regression coverage for browser-tool.actions, resolveSnapshotPlan, /snapshot routing, and repeated browser tool loops.
