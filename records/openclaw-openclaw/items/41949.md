## Summary

Keep open. Current main has useful generic tool-result/context guards, but the browser path still does not implement the browser-specific, task-aware context budgeting requested here: default AI snapshots can still carry a broad page dump, efficient mode is opt-in/config-driven, and explicit AI snapshots bypass the efficient default.

## Best Possible Solution

Implement a browser-owned context-budget policy: make agent-facing snapshots default to bounded compact/interactive output where appropriate, apply budget limits consistently across AI, role, Chrome MCP, and efficient paths, dynamically tighten browser output under context pressure, support selector/ref/detail fetch-on-demand so full page state stays outside the main prompt, and add focused regression tests for browser-tool.actions, resolveSnapshotPlan, /snapshot routing, and repeated browser tool loops.
