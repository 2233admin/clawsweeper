## Summary

Keep open. Current main still has a deterministic path for the reported noise: `openclaw doctor` asks the gateway for `doctor.memory.status` with `probe: false`, the gateway returns the skipped embedding probe as `checked: false`, but the CLI wrapper treats any successful RPC payload as `checked: true`; the memory doctor then warns for key-optional providers such as `ollama` when the probe is not ready. This is narrower than the shipped timeout fix for #44426 and remains a valid current-main bug.

## Best Possible Solution

Keep this issue open and land a narrow doctor/memory fix. Preserve `payload.embedding.checked === false` through `probeGatewayMemoryStatus`, treat skipped/not-checked or timeout/inconclusive gateway memory probes as non-actionable for key-optional providers such as `ollama` and `lmstudio`, and keep warnings for explicit checked failures. Add focused regression coverage for the skipped payload and the explicit failure path, plus a short changelog fix entry.

## Clownfish Work Prompt

Fix the doctor memory-readiness false warning reported in openclaw/openclaw#73882. Current `openclaw doctor` calls `doctor.memory.status` with `probe: false`; when the gateway has no cached embedding readiness, `src/gateway/server-methods/doctor.ts` returns `embedding.checked === false` with `memory embedding readiness not checked`, but `src/commands/doctor-gateway-health.ts` maps every successful payload to `checked: true`. For key-optional providers such as `ollama`/`lmstudio`, `src/commands/doctor-memory-search.ts` then emits a doctor warning even though no live probe was performed.

Expected fix boundary: preserve skipped/inconclusive state from `DoctorMemoryStatusPayload.embedding.checked === false` in `probeGatewayMemoryStatus`, and ensure skipped/not-checked or timeout/inconclusive gateway probes do not surface as provider-not-ready doctor warnings for key-optional providers. Preserve warnings for explicit checked failures such as `connection refused`, missing runtime dependencies, or other concrete not-ready results. Do not make normal `openclaw doctor` perform a live embedding probe by default unless existing docs/tests clearly require that change.

Add focused regression coverage in `src/commands/doctor-gateway-health.test.ts` for a successful `doctor.memory.status` response with `embedding.checked === false`, and in `src/commands/doctor-memory-search.test.ts` for an `ollama` or `lmstudio` skipped/not-checked probe producing no Memory search warning while checked explicit failures still warn. Update `CHANGELOG.md` under Unreleased Fixes with a single-line user-facing entry crediting `Thanks @islandpreneur007`. Docs likely do not need changes unless behavior text is adjusted.

Do not touch unrelated memory backends, QMD behavior, plugin loading, release metadata, lockfiles, or generated files.
