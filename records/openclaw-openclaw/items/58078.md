## Summary

Keep #58078 open. Current main and the latest release still do not pre-install `iproute2` in the official Docker runtime image. The supported path today is an opt-in `OPENCLAW_DOCKER_APT_PACKAGES` build arg for locally built/custom images, which is useful but does not satisfy the request to avoid rebuilding a custom image for Synology/macvlan routing use cases.

## Best Possible Solution

Keep the issue open for maintainer product judgment. A complete fix would either add `iproute2` to the default runtime package list in `Dockerfile` with Dockerfile/docs/test updates, or explicitly document a supported Synology/macvlan routing recipe that covers the required image package, user, and Linux capability settings without implying the official prebuilt image already supports runtime route replacement.

## Risks / Open Questions

- Adding `iproute2` alone may not be enough for the reported `ip route replace` workflow: route changes normally also depend on container user/capability settings such as root and `NET_ADMIN`, while the shipped Compose file explicitly drops `NET_ADMIN` for the CLI helper service.
- Maintainers should decide whether the official image should remain minimal/generic or include this small networking tool because the existing Docker architecture already supports local apt additions for optional tools.
