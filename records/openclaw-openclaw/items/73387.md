## Summary

Current main still has the Discord implicit channel allowlist behavior this issue reports: a non-empty guild `channels` map makes unmatched channels resolve to `allowed: false`, and message preflight then drops those guild messages even when the broader Discord `groupPolicy` is `open`. Existing docs and tests still describe/assert that behavior, while related reports are broader or adjacent rather than a clear canonical replacement.

## Best Possible Solution

Keep this issue open for bundled Discord plugin work. Update Discord channel policy resolution so `guilds.*.channels` entries can act as additive per-channel overrides under `channels.discord.groupPolicy: "open"`, with unlisted channels inheriting guild defaults such as `requireMention`, sender restrictions, tools, skills, and prompts. Preserve explicit channel denial and allowlist-mode deny semantics, document precedence, and add regression coverage for listed overrides, unlisted inherited channels, and explicit disabled channels.

## Risks / Open Questions

- Changing this can widen Discord guild access for existing configs that rely on `guilds.*.channels` as an implicit allowlist, so maintainers need an explicit compatibility and migration decision.
- The implementation needs to distinguish unmatched channels that inherit guild defaults from explicit disables such as `enabled: false`, and should preserve deny-by-default behavior for `groupPolicy: "allowlist"`.
