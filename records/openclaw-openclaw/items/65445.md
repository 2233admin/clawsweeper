## Summary

Keep open. Current main has partial cleanup for old `.deleted` and `.reset` archives, and pressure-based disk-budget cleanup can remove archive artifacts including `.bak`, but it still does not implement the requested default garbage-collection/recycle policy for orphaned `.jsonl.bak.*` files or `sessions_saved.json`. The earlier ClawSweeper review remains directionally correct, with current main adding useful overlap but not closing the central request.

## What This Changes

The issue requests session maintenance garbage collection for orphaned `.reset`, `.deleted`, `.bak`, and `sessions_saved.json` files, preferably through configurable recycle/retention settings.

## Best Possible Solution

Define one session-maintenance policy for orphaned archive artifacts: either add documented recycle/retention config under `session.maintenance` or explicitly extend the existing archive cleanup to cover `.jsonl.bak.*` and any accepted legacy `sessions_saved.json` behavior, with schema/docs/tests aligned.
