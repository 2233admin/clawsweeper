## Summary

Keep open. Current main has useful timeout plumbing, but it still does not prove a complete fix for the updated large tool-parameter failure: OpenRouter models still resolve through the OpenAI-compatible transport, direct Anthropic fine-grained tool streaming is gated to native Anthropic routes, and the embedded runner still applies a chunk-idle watchdog when the transport withholds tool-call argument chunks.

## What This Changes

The issue requests that embedded runner LLM calls survive, retry, or fail clearly when slow models generate large tool-call parameters, especially on the OpenRouter to Anthropic path.

## Best Possible Solution

Keep this as the canonical tracker for OpenRouter/Anthropic large tool-parameter generation failures. The durable fix should add a synthetic OpenAI-compatible regression that withholds tool-call argument chunks long enough to exercise the idle/request-timeout path, then implement the chosen runtime policy for supported tool-argument streaming, safe timeout behavior, or avoiding single huge tool payloads.
