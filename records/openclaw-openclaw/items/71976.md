## Summary

Keep open. Current main still contains the reported Memory Dreaming mechanics, and open PR #74051 is linked with closing syntax for this issue, so the conservative maintainer path is to review or land that PR rather than close the issue early or queue duplicate repair work.

## What This Changes

The issue asks OpenClaw to make Memory Dreaming prefer real recall signals over daily/session recency, keep daily-only signals out of recall-count promotion gates, and widen short-term promotion rehydration.

## Best Possible Solution

Land #74051 or a focused replacement that separates recall-gate eligibility from daily/session reinforcement, preserves intentional daily and grounded-evidence scoring, widens and tightens rehydration as needed, and updates tests/docs to define the corrected contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: daily/session ingestion refreshes lastRecalledAt, Light sorts by that timestamp before recallCount, and promotion ranking/apply use combined signalCount for minRecallCount; the current tests also assert daily-only signals can clear the default promotion gate.

## Solution Assessment

No for the original sort-only snippet by itself. The maintainable fix should address recall-gate semantics, scoring contract, rehydration behavior, and docs/tests together, which is the direction already represented by open PR #74051.
