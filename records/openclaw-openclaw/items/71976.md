## Summary

Keep open. Current main still matches the reported memory-core behavior: Light dreaming sorts by refreshed recency before real recall count, deep promotion counts daily signals toward the default recall gate, and live rehydration still uses the narrow span. The provided GitHub context also shows an open unmerged implementation PR with closing syntax, so the issue should wait for that PR or a maintainer-approved replacement.

## What This Changes

The issue asks memory dreaming to prioritize real recall signals over daily/session recency signals and widen rehydration so valid short-term recall candidates can promote.

## Best Possible Solution

Land or replace a focused memory-core fix that makes real recall signals drive Light selection and default promotion eligibility, keeps daily/session ingestion from satisfying recall gates by itself while preserving intended scoring signals, widens robust snippet rehydration, and updates the tests/docs that define the signal contract.
