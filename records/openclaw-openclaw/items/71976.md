## Summary

Keep open. Current main still contains the reported Memory Dreaming behavior, and open PR #74051 is linked with closing syntax for this issue, so the conservative maintainer path is to review or land that PR rather than close the issue early.

## What This Changes

The issue reports that Memory Dreaming Light sorting and promotion gates let daily/session signals hide real recall entries, causing valid short-term recall data to miss promotion.

## Best Possible Solution

Review and land #74051, or a focused replacement, so real recall signals drive default Light selection and recall-gate eligibility, daily/session ingestion can still contribute scoring context without satisfying recall gates by itself, rehydration handles shifted snippets, and tests/docs define the corrected signal contract.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence reproduction path: repeated daily signals can produce a default-ranked candidate with recallCount 0, Light sorts by lastRecalledAt before recallCount, and apply/rank gates use combined signal count.

## Solution Assessment

Yes for the triage outcome: keeping this open while #74051 is reviewed is the narrowest safe path. A sort-only fix is not enough because the remaining behavior also spans recall-gate semantics, scoring averages, apply filtering, and rehydration span.
