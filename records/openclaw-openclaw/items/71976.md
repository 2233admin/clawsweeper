Keep open. Current main still has the reported memory-core behavior, and open PR #74051 is already linked with closing syntax, so this issue should wait for that PR or a focused replacement to land.

## What This Changes

The issue asks Memory Dreaming to prioritize real recall signals over daily/session ingestion recency, keep daily-only signals out of recall gates, and widen rehydration for shifted snippets.

## Best Possible Solution

Review and land #74051, or a focused replacement, so real recall signals drive default Light selection and recall-gate eligibility, daily/session ingestion can still contribute non-gating context, rehydration handles shifted snippets, and tests/docs define that contract.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence reproduction path: daily/session ingestion refreshes lastRecalledAt, Light sorts by that timestamp before recallCount, deep promotion gates on combined signalCount, and relocation still uses the narrow span.

## Solution Assessment
