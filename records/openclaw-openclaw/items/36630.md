## Summary

Keep open. Current main at 7a23b2d945cc2d2e5c452e4641213e71de8dcc66 still does not implement the PR's central bidirectional Signal quote-reply behavior: Signal sends do not emit signal-cli quote parameters, Signal outbound adapters do not forward quote metadata, monitor replies are unquoted, and inbound quote context still lacks the Signal quote timestamp as ReplyToId. The PR remains a plausible implementation candidate, but the current PR evidence includes build, boundary, and quote-metadata lifecycle risks that need maintainer cleanup before it can land.

## Best Possible Solution

Keep this PR open and rework it into a smaller, hardened Signal quote-reply change: implement Signal-owned quote timestamp/author support through plugin-safe SDK seams, surface structured inbound ReplyToId and richer quote payload fields, preserve quote metadata correctly across chunk/media/queued/recovered delivery, remove conflict markers and core deep imports, and add targeted regression coverage for formatted sends, monitor replies, group quote-author lookup, inbound quote context, and partial-failure reply consumption.
