## Summary

Keep this PR open. Current main still lacks the central bidirectional Signal quote-reply behavior: Signal sends do not emit native `quote-timestamp`/`quote-author` parameters, the Signal outbound adapter does not pass reply metadata into formatted or attached sends, monitor-delivered replies are unquoted, and inbound Signal quote context is still partial without `ReplyToId` or richer quote payload fields. The PR remains a plausible implementation candidate, but the provided PR context shows unresolved cleanup/security issues that need maintainer review before landing.

## Best Possible Solution

Keep this PR open and rework it into a smaller, hardened Signal quote-reply implementation: add native Signal quote timestamp/author support through the Signal send and formatted delivery paths, resolve group quote authors from trusted inbound message metadata where possible, surface inbound `ReplyToId` and richer quote payload fields, keep bundled Signal production imports inside plugin SDK/local boundaries, remove conflict markers, and add targeted regression coverage for formatted sends, monitor-delivered replies, chunk/media delivery, debounced inbound quote context, and queued/recovered delivery.
