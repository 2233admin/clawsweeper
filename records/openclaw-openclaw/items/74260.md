## Summary

Keep open. This PR is protected by maintainer routing, and current main still does not implement the intended Control UI behavior: the docs describe the assistant avatar override as browser-local, but Quick Settings still derives the override from config and stale missing IDENTITY.md metadata can still force the fallback/File not found path before a browser-local override wins. The PR patch is scoped to Control UI TypeScript and tests, with no security-sensitive infrastructure or supply-chain changes observed.

## What This Changes

The PR updates Control UI assistant avatar handling so a browser-local assistant avatar override wins over stale missing IDENTITY.md metadata in Quick Settings and chat rendering, with regression tests.

## Best Possible Solution

Keep this PR open in normal maintainer review.
