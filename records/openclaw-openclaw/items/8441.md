## Summary

Keep this issue open. Current main still does not implement first-class `skills.entries.<skill>.thinking` or `skills.entries.<skill>.model`; the config type/schema omit those fields, strict validation still rejects unknown top-level per-skill keys, docs only document `enabled`/`env`/`apiKey` plus custom `config`, and skill command invocation only rewrites the prompt body without carrying model or thinking defaults into model selection. The related implementation PR #8455 is closed and unmerged, and the narrower #40536 was closed while referencing this still-open request rather than superseding it.

## Best Possible Solution

Keep this issue open. Implement the feature in core skills config and invocation plumbing: add validated optional `model` and `thinking` fields to `SkillConfig`/`SkillEntrySchema`, carry resolved per-skill defaults through `SkillCommandSpec` or equivalent invocation context, apply them as per-turn defaults with clear precedence against explicit directives/session overrides/agent defaults, add schema and invocation/model-selection tests, and update the public skills config docs.
