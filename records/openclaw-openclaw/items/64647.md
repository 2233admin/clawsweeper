## Summary

Keep this PR open. Current main still does not implement per-agent memory backend overrides, and the discussion shows this is the active implementation candidate for the staged rollout request, but the branch needs maintainer refresh before merge because the memory host SDK moved and the new per-agent schema appears to lag newer QMD update fields.

## What This Changes

The PR adds `agents.list[].memory` for per-agent memory backend and QMD overrides, updates config schema/types/docs, and adds regression/runtime coverage while keeping `memory.citations` global.

## Best Possible Solution

Land this PR or an equivalent patch after rebasing onto current main, implementing the override in both the source config surface and the package memory-host-sdk resolver/config mirror, sharing or fully mirroring the root QMD schema, keeping citations global, updating generated schema/docs, and preserving focused schema/runtime regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Mirror root QMD update fields in agent schema:** `src/config/zod-schema.agent-runtime.ts:804-815`
  - body: The new strict per-agent QMD update schema omits `startup` and `startupDelayMs`, even though current root `memory.qmd.update` accepts them and the resolver has runtime behavior for them. A per-agent canary that needs to tune gateway-start refresh settings would be rejected by validation despite the PR promising `memory.qmd.*` overrides. Add those fields or reuse the root QMD schema.
  - confidence: 0.88
- **[P2] Move resolver changes into the package SDK:** `src/memory-host-sdk/host/backend-config.ts:363-375`
  - body: Current main has moved the memory host implementation to `packages/memory-host-sdk/src/host/backend-config.ts`, while `src/memory-host-sdk/host/backend-config.ts` is only a compatibility re-export. The PR diff applies the override merge logic to the source-wrapper path, so after a rebase the package resolver and SDK mirror would still read only the root `memory` config unless the package files are updated too.
  - confidence: 0.84
