## Summary

Keep open. Current main still sends WhatsApp connected/disconnected lifecycle notices straight into the resolved session system-event queue, and the shared queue only suppresses exact consecutive duplicate text. There is no threshold, verbosity option, reconnect coalescing, or surface-aware filter, and the related reports cited in discussion cover different warning or health-monitor surfaces rather than this user-visible lifecycle-notification policy.

## What This Changes

The issue requests a configurable way to suppress, throttle, or surface-filter transient channel connection status system events.

## Best Possible Solution

Implement a tested WhatsApp-owned lifecycle notification policy first, such as a statusEvents mode and/or notifyAfterMs threshold with reconnect coalescing for outages that recover quickly. Promote a generic plugin SDK helper only if multiple channel plugins need the same behavior.
