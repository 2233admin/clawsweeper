## Summary

Keep this PR open. Current main still decodes canvas request paths with `decodeURIComponent` without a malformed-percent guard, and the canvas host request handler maps that exception class to a 500 response. The proposed branch is a narrow resolver-level fix with a focused regression test, and I did not find a concrete correctness or security blocker in the submitted diff.

## What This Changes

The PR wraps canvas file path normalization in a `URIError` guard that treats malformed URL encoding as an unresolved path and adds a canvas host test expecting 404 `not found` for `%E0%A4%A`.

## Best Possible Solution

Merge a narrow resolver-level malformed-encoding fallback with regression coverage so both canvas and A2UI file serving treat invalid percent-encoded paths as ordinary misses while preserving the existing root, traversal, symlink, and file-open safety behavior.
