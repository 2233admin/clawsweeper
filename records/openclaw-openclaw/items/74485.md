## Summary

Keep this PR open for maintainer review and merge consideration. Current main still has the malformed percent-decoding failure path, while the PR is a narrow resolver-level fix with regression coverage, green remote checks, and no concrete correctness or security finding from this review.

## What This Changes

The PR catches `URIError` from malformed percent decoding in `resolveFileWithinRoot` and adds a canvas host regression test expecting `404 not found` for `%E0%A4%A`.

## Best Possible Solution

Canvas and A2UI static file resolution should treat malformed percent-encoded paths as ordinary unresolvable misses while preserving the existing root, traversal, symlink, method, safe-open, and auth boundaries.

## Reproduction Assessment

Yes. The current-main code path is deterministic: `URL.pathname` preserves the malformed `%E0%A4%A` path, `decodeURIComponent` throws in `normalizeUrlPath`, and the canvas request handler maps that thrown resolver error to a 500 response.

## Solution Assessment

Yes. Returning `null` only for `URIError` during resolver path normalization is the narrowest maintainable fix because it reuses the existing 404 miss path and does not broaden filesystem access or alter auth behavior.
