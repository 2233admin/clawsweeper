## Summary

Keep this PR open. Current main still calls `decodeURIComponent` for canvas file paths without a malformed-encoding guard, and the submitted patch is a narrow, relevant fix that converts that resolver failure into the existing not-found path with focused regression coverage. I did not find a discrete blocking correctness issue in the diff; it needs normal maintainer and CI review before merge.

## What This Changes

The branch wraps canvas file path normalization in a `URIError` guard that returns `null` from `resolveFileWithinRoot` and adds a canvas host regression test expecting malformed percent-encoded paths to return 404 `not found`.

## Best Possible Solution

Merge a resolver-level malformed-encoding fallback with regression coverage so canvas and A2UI file serving treat invalid percent-encoded paths as ordinary misses while preserving the existing root, traversal, symlink, and file-open safety checks.
