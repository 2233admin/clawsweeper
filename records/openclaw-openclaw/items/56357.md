## Summary

Keep open. Current main does not definitely implement this PR’s intended fix for the Control UI logo/favicon regression, and the linked regression issue remains open. The PR is now stale against refactored avatar rendering, but the underlying problem is still plausible on main: the header/login logo still flows through `agentLogoUrl()`, which still returns route-relative `favicon.svg` for an empty base path, and Control UI static responses still receive the global CSP that the PR was trying to narrow for SVG paint references.

## Best Possible Solution

Review or replace this PR with a current-main patch: make the empty-basePath logo URL absolute where appropriate, update the current header/login/welcome/avatar tests, and handle the SVG favicon paint-reference CSP issue with a narrow policy that preserves avatar and direct-SVG security guarantees. If the implementation is no longer the right shape after the `chat-avatar.ts` refactor, keep #55600 as the canonical bug and land a fresh focused fix there.
