## Summary

Keep this PR open. Current main already has the `/usage` context-percentage alignment, but it still lacks the PR’s second intended behavior: refreshing the active chat history after successful compaction. The PR also needs a rebase because its proposed `/usage` guard and sessionless compaction reload condition would regress or over-broaden current behavior.

## Best Possible Solution

Keep the PR open as a possible implementation candidate, but require a rebase onto current main. Preserve the shipped `/usage` total-token logic, then add a narrow compaction-complete refresh path that only reloads the active session when the event is clearly for that session, with app-gateway tests for active-session refresh, unrelated-session ignore, and missing-sessionKey behavior during active runs.
