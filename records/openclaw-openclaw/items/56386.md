## Summary

Keep open. Current main still does not cancel ACP-backed child subagent sessions during subagent stop propagation, so the PR is not obsolete, but the PR head has a blocking availability regression risk and a PR-head test setup failure that should be fixed before merge.

## What This Changes

The PR converts subagent stop propagation to async, awaits stop/reset callers, cancels ACP-backed child sessions, persists child aborted state, and adds fast-abort regression coverage.

## Best Possible Solution

Keep the child-stop cascade direction, but bound child ACP cancellation with the same timeout/fallback style used by gateway ACP reset cleanup, then rebase and cover both successful child cancellation and hung-cancel fallback before merge.

## Reproduction Assessment

Yes. A child ACP runtime whose cancel promise never resolves reproduces the reset hang once the PR makes reset await stopSubagentsForRequester; the PR-head unit-test failure is also source-reproducible from the missing latest-run mock on that branch.

## Solution Assessment

No, not as proposed. Propagating stop into ACP-backed child sessions is the right direction, but the mergeable solution needs bounded cancellation and refreshed tests against current main.

## Review Findings
