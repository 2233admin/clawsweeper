## Summary

Keep this PR open. Current main has adjacent cascade-stop support for native subagents, but it does not implement this PR’s central ACP-backed stop propagation: the stop helper is still synchronous, reset still calls it without awaiting, active ACP child sessions are not cancelled from that helper, and /stop does not contain the proposed secondary requester-key fallback. The PR also needs rework before merge because the review discussion identified a hang risk around unbounded ACP cancellation and a regression-test setup issue.

## Best Possible Solution

Rebase or replace the PR with a focused implementation that makes stop propagation awaitable, cancels active ACP child sessions through a bounded timeout/fallback, persists child `abortedLastRun` state, and adds regression tests for ACP child cancellation, reset cleanup, cascade traversal, and the intended /stop target fallback semantics.

## Risks / Open Questions

- Merging the PR as written could hang stop/reset if an ACP backend never resolves cancellation; the child cancel path should be bounded like the existing session-reset ACP cleanup wrapper.
- The provided review comments flagged a regression-test setup issue around latest-run mocking; that should be fixed or revalidated after rebasing onto current main.
- The proposed /stop fallback conflicts with current target-fallback test expectations, so maintainers should decide the intended target semantics before landing that part.
