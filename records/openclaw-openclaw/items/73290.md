## Summary

Keep this PR open. Current main still has the reported GitHub Copilot GUI/RPC auth gap: the provider auth method short-circuits without a TTY, the device-login command is still stdin/terminal-oriented, and the current contract tests assert that behavior. The PR is an active, focused implementation candidate for the missing wizard/RPC flow, with review feedback reportedly addressed in the latest head commit.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate. The best path is to land a focused version that keeps the CLI terminal login behavior intact, uses the existing provider `prompter`/`openUrl`/returned-profiles contract for GUI/RPC wizard auth, preserves validation around GitHub device-flow URLs/codes, and includes the updated provider-auth contract coverage plus a security check of the raw-fetch allowlist shift.
