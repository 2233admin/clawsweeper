## Summary

Keep this PR open. Current main still has the GitHub Copilot GUI/RPC auth gap: provider auth exits with no profiles when stdin is not a TTY, the device login command is terminal-driven, and the current contract tests assert that old behavior. The PR is a focused implementation candidate, with review feedback in the provided discussion reportedly addressed at head commit 7cc0e63671ea68ac5a47462f050777abc3374b6e.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate. The best path is to land a focused version that preserves the CLI terminal login path, makes provider auth work through `ctx.prompter` and `ctx.openUrl` for GUI/RPC clients, returns credentials through the existing provider-auth result contract so the framework persists them to the selected agent dir, and includes the updated contract coverage plus an auth/security review of the raw-fetch allowlist shift.
