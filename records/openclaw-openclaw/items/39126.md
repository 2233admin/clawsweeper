## Summary

Keep open. Current main still lacks session-based exec trust windows: there are no trust/untrust CLI commands, Gateway RPC methods, protocol schemas, in-memory trust cache, trust audit module, or slash-command handlers. Related open PRs #46956 and #47696 overlap but are not merged or clearly canonical replacements, and this is security-sensitive because it intentionally bypasses exec approval prompts for a bounded period.

## Best Possible Solution

Keep this PR open until maintainers choose a canonical trust-window design. The likely path is to review the narrower core/CLI/Gateway proposal in #46956 first, decide the security model, then either close this PR as superseded after that path is accepted or rebase this broader CLI/Gateway/Discord/TUI/audit implementation onto current main with focused security review and tests.
