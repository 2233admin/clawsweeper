## Summary

Keep open. Current main does not implement session-based exec trust windows, and this PR is still useful but security-sensitive and now overlaps with narrower open trust-window proposals that need maintainer design review before any broad implementation is merged.

## What This Changes

This PR adds time-bounded exec trust windows across exec approvals, CLI trust/untrust commands, Gateway RPC/protocol methods, Discord slash commands, status display, Swift protocol models, and JSONL audit logging.

## Best Possible Solution

Maintainers should choose one canonical trust-window design, settle the enforcement and authorization semantics first, then either rebase this broader branch onto that design or close it after a narrower implementation lands. Discord or cross-surface command work should follow the chosen core model rather than defining a separate trust authority.
