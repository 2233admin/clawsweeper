## Summary

Keep open. Current main does not implement session-based exec trust windows, and this PR is still useful design/review context while overlapping open PRs #46956 and #47696 remain unmerged. Because the change intentionally creates a time-bounded bypass of exec approval prompts, it needs maintainer threat-model and product review rather than automated closure or a replacement repair job.

## What This Changes

The PR branch adds time-bounded exec trust windows across the exec approvals model, CLI trust/untrust commands, Gateway RPC/protocol methods, Discord slash commands, TUI/status display, Swift protocol models, and JSONL audit logging.

## Best Possible Solution

Keep this PR open until maintainers choose one canonical trust-window design. The likely path is to review the narrower core/CLI/Gateway proposal in #46956 first, decide the security model and deny-policy semantics, then either close this broader PR as superseded after a canonical implementation merges or rebase it onto current main with focused security review and tests. #47696 should be considered for any cross-surface slash-command design once the core model is settled.
