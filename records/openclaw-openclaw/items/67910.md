## Summary

Keep this PR open for maintainer review. Current main still has the deep-only `memory status` renderer, and the PR is an active implementation thread for #67868, but its latest patch still gates phase status on the deep-promotion resolver and has reviewable correctness concerns before merge.

## What This Changes

This PR changes memory-core CLI status output to render dreaming phase state from the full dreaming config, adds CLI tests, exposes `memory status --force`, and trims blank timezone labels in `/dreaming status`.

## Best Possible Solution

Keep the PR open and revise or replace it with a narrow memory-core status change that reports configured light/deep/REM phase state while preserving effective deep-promotion disabled reasons such as deep off and `limit=0`. Cover light-only, REM-only, deep-disabled, global-disabled, and limit-zero configurations in targeted tests; keep unrelated `--force` or timezone cleanup only if maintainers explicitly want that scope bundled.

  - confidence: 0.9

  - confidence: 0.84
