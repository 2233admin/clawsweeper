## Summary

Keep open. Current main and release v2026.4.27 still contain the unguarded Feishu `sendMedia` companion-text path, while this PR is a narrow, relevant fix with no concrete correctness or security regression found.

## What This Changes

The PR changes the Feishu outbound `sendMedia` handler so companion text is sent before media only when `mediaUrl` is present, and updates the adjacent comment.

## Best Possible Solution

Land the minimal `mediaUrl` guard in the Feishu outbound `sendMedia` handler, ideally with a regression test proving text plus no `mediaUrl` sends exactly one Feishu text message; keep broader media-upload fallback behavior as separate follow-up work.
