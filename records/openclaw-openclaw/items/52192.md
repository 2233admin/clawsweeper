## Summary

Keep open. Current main still has the Feishu sendMedia duplicate text path this PR targets: non-empty text is sent before the mediaUrl check, then the no-media fallback sends the same text again when mediaUrl is absent. The PR is narrow and security-benign, but should ideally add a focused regression test before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the minimal mediaUrl guard in extensions/feishu/src/outbound.ts and add focused regression coverage in extensions/feishu/src/outbound.test.ts for sendMedia with text but no mediaUrl, while leaving the upload-failure companion-text behavior as a separate Feishu follow-up if desired.
