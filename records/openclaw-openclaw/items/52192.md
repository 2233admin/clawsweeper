## Summary

Keep open. Current main and the latest release still have the duplicate-send path, while this PR is a narrow contributor implementation that addresses the Feishu text-without-media case; it needs maintainer review/rebase and likely a focused regression test, not auto-closure.

## What This Changes

The PR changes the Feishu outbound `sendMedia` handler so companion text is sent before media only when `mediaUrl` is present, and updates the adjacent comment.

## Best Possible Solution

Land the minimal `mediaUrl` guard in the Feishu outbound `sendMedia` handler and add or request a focused regression test proving non-comment text plus no `mediaUrl` sends exactly one Feishu text message; leave broader upload-failure fallback behavior to a separate follow-up if maintainers want to change it.

## Reproduction Assessment

Yes. A direct `feishuOutbound.sendMedia` call with non-empty `text` and no `mediaUrl` reaches the current companion text send and then the no-media text fallback, which is visible from current-main source inspection without a live Feishu account.

## Solution Assessment

Yes. The proposed guard is the narrowest maintainable fix for this duplicate path because it preserves text-plus-media and media-only behavior while letting the existing text-only fallback handle the no-media case once.
