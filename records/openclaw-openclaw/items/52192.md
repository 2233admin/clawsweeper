## Summary

Keep open. Current main still has the Feishu `sendMedia` duplicate-send path this PR targets: it sends non-empty text before checking `mediaUrl`, then sends the text fallback again when `mediaUrl` is absent. The PR is narrow and still useful, but should ideally gain a regression test for the text-only `sendMedia` case before landing or maintainer-apply.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to apply the `text?.trim() && mediaUrl` guard in `extensions/feishu/src/outbound.ts` and add focused regression coverage in `extensions/feishu/src/outbound.test.ts` for `feishuOutbound.sendMedia` with text but no `mediaUrl`, while leaving the upload-failure companion-text follow-up as a separate Feishu issue if desired.
