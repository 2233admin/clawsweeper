## Summary

Keep open. Current main has meaningful prompt-injection hardening for external content and cron, including wrapping, marker/special-token sanitization, suspicious-pattern logging, tool policy, and docs recommending read-only/tool-disabled reader agents. It still does not implement the requested default dual-LLM privilege-separation architecture, strict JSON sanitizer handoff, or a sanitizer continuation hook for privileged manager turns. The linked ClawMoat plugin and closed unmerged PR #30046 are useful prior art, not evidence that the core security/product request is resolved.

## Best Possible Solution

Keep this open as the canonical security/product design item. The best path is maintainer review of either a first-class read-only/tool-disabled reader-worker flow for cron and other external-content ingestion, with strict structured sanitizer output before any privileged manager turn, or a narrow documented modifying hook that lets a plugin replace/block wrapped external content and then continue safely into the model. The design should account for ClawMoat, the closed #30046 proposal, and adjacent memory-source filtering in #28913, with tests proving untrusted external content cannot silently reach privileged cron execution or memory capture as ordinary prompt text.
