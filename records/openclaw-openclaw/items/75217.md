## Summary

Keep open. The PR targets the right shared resolver for #75184, and current main still has the reported split between workspace creation and runtime injection, but the proposed early return is broader than the bug: it skips the documented `agent:bootstrap` hook path and would drop explicitly configured bootstrap-extra files.

## What This Changes

The branch adds a `skipBootstrap` guard to runtime bootstrap resolution, updates agent config docs/JSDoc, and adds regression coverage for pre-existing workspace bootstrap files.

## Best Possible Solution

Keep the resolver-level fix, but treat `skipBootstrap` as an empty default workspace-file set and still run hook override/sanitize logic; add regression coverage for hook-added files under `skipBootstrap`, and align the zod/generated config schema help with the new documented contract.

## Reproduction Assessment

Yes. Current main can be reproduced by leaving `AGENTS.md`/`SOUL.md` in the workspace, setting `agents.defaults.skipBootstrap: true`, and observing `resolveBootstrapContextForRun` still returns context files; the PR regression is reproducible by enabling `bootstrap-extra-files` with `skipBootstrap: true` and seeing hook-added files disappear.

## Solution Assessment

No. The shared resolver is the right surface, but the early return is not the narrowest maintainable fix because it skips explicit hook context; starting from `rawFiles = []` while still running hooks is safer.
