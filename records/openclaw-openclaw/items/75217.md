## Summary

Keep open. The PR addresses the reported runtime no-op for pre-existing workspace bootstrap files and the author added docs/tests after the first review, but the new early return bypasses `agent:bootstrap` hook processing and would drop explicitly configured hook-provided context such as `bootstrap-extra-files`.

## What This Changes

The PR adds a `skipBootstrap` guard to runtime bootstrap resolution and updates agent config docs, type comments, and regression tests for pre-existing workspace bootstrap files.

## Best Possible Solution

Keep the resolver-level runtime fix, but treat `skipBootstrap` as an empty initial workspace-file set and still run `applyBootstrapHookOverrides`; then add coverage for both existing workspace files and hook-added files, and align generated config schema/help if the public schema should explain the expanded scope.

## Reproduction Assessment

Yes. Current main can be reproduced by leaving workspace bootstrap files on disk, setting `agents.defaults.skipBootstrap: true`, and observing that CLI or embedded runtime paths still resolve them through `resolveBootstrapContextForRun`; the PR regression can be checked by enabling `bootstrap-extra-files` with `skipBootstrap: true` and seeing the early return skip hook-added files.

## Solution Assessment

No. The shared resolver is the right surface, but returning before hook execution is broader than the reported bug; a narrower implementation should skip default workspace file loading while preserving explicitly configured bootstrap hooks, or document a deliberate hook-suppression policy.
