## Summary

Keep open. Current main still has the reported skipBootstrap runtime-injection gap, but this PR’s early return skips the shared bootstrap hook path as well, which would drop explicitly configured hook-provided context such as bootstrap-extra-files.

## What This Changes

The PR adds an early agents.defaults.skipBootstrap guard in src/agents/bootstrap-files.ts so runtime bootstrap file resolution returns no files before system-prompt injection.

## Best Possible Solution

Handle skipBootstrap as an empty default workspace-bootstrap file set while still running applyBootstrapHookOverrides, then add resolver/prompt-report coverage and align the docs/schema wording with the chosen semantics.

## Reproduction Assessment

Yes. Current main reproduces the original bug by keeping workspace bootstrap files, setting agents.defaults.skipBootstrap: true, and observing runtime injection through resolveBootstrapContextForRun; the PR regression is reproducible by enabling a bootstrap-extra-files hook under the same flag and seeing the early return prevent hook-provided files.

## Solution Assessment

No. The shared resolver is the right place to fix the runtime path, but returning before hook overrides suppresses explicitly configured bootstrap hooks; a narrower fix should preserve hooks or intentionally document that only contextInjection: "never" disables all injection.

## Review Findings
