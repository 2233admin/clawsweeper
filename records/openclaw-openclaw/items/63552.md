## Summary

Keep this PR open. Current main has a scoped startup-provider option, but ordinary non-live `models.json` generation still does not derive discovery scope from explicitly configured `models.providers`; without `providerDiscoveryProviderIds`, discovery remains broad. The overlapping #72352 is still open, so it is not a merged or maintainer-selected superseding fix.

## What This Changes

The PR updates models-config provider discovery to scope implicit discovery from explicitly configured providers, preserve fallback/live-filter behavior, and add focused regression coverage.

## Best Possible Solution

Land or adapt one canonical implementation that scopes non-live provider discovery from configured provider/model hints, resolves owners through the existing plugin ownership metadata, preserves broad fallback when ownership cannot be proven, and keeps live filter semantics intact. If maintainers select #72352 instead, close this PR only after that replacement path is explicitly chosen or merged.
