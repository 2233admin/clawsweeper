## Summary

Keep this PR open. Current `main` still lets `openclaw message ...` pass the eager context-window warmup gate, and the PR is a focused, low-risk implementation that matches the message CLI's channel-action-only behavior.

## What This Changes

The PR adds `message` to the eager context-window warmup skip list and adds a regression assertion for `openclaw message send --channel telegram`.

## Best Possible Solution

Land the focused skip-list change with regression coverage so message-only CLI invocations avoid unrelated model catalog/auth discovery while preserving eager warmup for commands that actually need model metadata.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Current-main source shows `message` is absent from the skip set while the warmup predicate warms real CLI primary commands not in that set; the PR adds direct predicate coverage for `openclaw message send --channel telegram`.

## Solution Assessment

Is this the best way to solve the issue? Yes. Adding `message` to the existing primary-command skip set is the narrowest maintainable fix, and the message CLI docs and implementation do not show a normal outbound send dependency on model metadata.
