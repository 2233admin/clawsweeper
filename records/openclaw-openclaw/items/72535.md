## Summary

Keep this PR open. The warning text is a useful small improvement, but it does not cover the invalid downgrade path described by the linked bug because current config loading and snapshot reads exit before `warnIfConfigFromFuture` runs when validation fails.

## What This Changes

The PR appends `openclaw doctor --fix` and manual value-format remediation guidance to the future-version warning emitted by `warnIfConfigFromFuture` in config I/O.

## Best Possible Solution

Surface future-version remediation from invalid-config reporting as well as the valid-config warning, using raw/source config `meta.lastTouchedVersion` before validation blocks. Keep the warning text as a narrow improvement only if the linked downgrade issue remains open or the invalid downgrade path gets focused tests and guidance.

## Reproduction Assessment

Yes. The high-confidence reproduction for the review finding is the current-main control flow: invalid config validation exits at `loadConfig` and `readConfigSnapshot` before `warnIfConfigFromFuture`, while the linked downgrade issue supplies concrete invalid-config examples.

## Solution Assessment

No. Adding hints only to `warnIfConfigFromFuture` is safe as a partial improvement, but it is not the best fix for the linked downgrade bug; the safer path is to attach the same future-version remediation to invalid-config errors or narrow the PR's closing scope.
