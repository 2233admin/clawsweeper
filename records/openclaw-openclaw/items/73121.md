## Summary

Keep open. Current main has strong partial reconciliation for stale task records, but the reported CLI/subagent zombie-task problem is not fully closed: the lean routed JSON list still returns raw store rows, the human table prints truncated task/run IDs while lookup/cancel remains exact, and the exact exec-approval plus old subagent persisted-row shapes from the report still need targeted fixtures before maintainers can call this fixed.

## What This Changes

The issue requests automatic terminal reconciliation for dead CLI/subagent task records and clearer cancellation behavior for task IDs shown by `openclaw tasks list`.

## Best Possible Solution

Keep the existing runtime-aware maintenance foundation, then add a focused task-ledger fix that reconciles the JSON and human list behavior, makes printed identifiers usable for show/cancel, and adds regression fixtures for stale exec-approval, chat-backed CLI, blank-child-session, persistent-session, and subagent orphan row shapes.

## Reproduction Assessment

Partly. Current tests reproduce chat-backed CLI rows losing their live run context and stale cron/subagent recovery cases, but there is not yet a high-confidence fixture for the exact exec-approval and older subagent records described in this issue.

## Solution Assessment

No. The current implementation is a good foundation, but closing this now would leave observable operator gaps in JSON listing and cancellation UX; the narrower maintainable solution is one task-registry/CLI PR with targeted fixtures.
