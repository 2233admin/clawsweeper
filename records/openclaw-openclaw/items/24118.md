## Summary

Keep open. Current main still does not expose `channels.signal.linkPreview` in the Signal config type/schema/generated metadata/docs, and the Signal JSON-RPC send path still does not pass `previewUrl`/`previewTitle`/`previewDescription`/`previewImage` or call `updateConfiguration`. The related WebChat preview-card issue #29466 is a separate UI surface and does not supersede this Signal channel request.

## Best Possible Solution

Keep this open for a maintainer-approved Signal plugin design: add the `channels.signal.linkPreview` config contract in types/schema/generated metadata/docs/tests, decide top-level versus per-account behavior, define `updateConfiguration` lifecycle and linked-device error handling, and wire only the safe JSON-RPC send fields unless a separate metadata-fetching security design is approved.
