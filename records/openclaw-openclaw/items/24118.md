## Summary

Keep this issue open. Current main still does not implement `channels.signal.linkPreview`: Signal config types, strict schema, generated metadata, docs, and the Signal JSON-RPC send path all omit the requested setting and preview/updateConfiguration fields. The request is valid, but it should stay maintainer-reviewed because `signal-cli` link-preview configuration is primary-device-sensitive and overlaps the open link-preview prompt-injection concern in #22060.

## What This Changes

Add first-class `channels.signal.linkPreview` configuration and wire Signal link-preview/updateConfiguration parameters into the signal-cli JSON-RPC send path.

## Best Possible Solution

Keep this open for maintainer-reviewed Signal plugin work: add `channels.signal.linkPreview` to the Signal config type, strict schema, generated metadata, docs, and tests; decide top-level/per-account semantics and default behavior; define when `updateConfiguration` should run and how linked-device failures are handled; and pass only explicit safe Signal preview fields through JSON-RPC send unless the separate #22060 security design approves metadata fetching.
