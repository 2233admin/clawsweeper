## Summary

Keep this issue open. Current main still has the shared default-account fallback/status behavior on the Feishu path, the focused Feishu restart/status regression coverage is still missing, and same-author PR #64712 remains the active implementation candidate rather than a merged fix.

## What This Changes

The issue reports that restarting the gateway after configuring multiple Feishu bot accounts creates or shows an unconfigured enabled `default` Feishu account.

## Best Possible Solution

Review and land PR #64712, or a replacement, that distinguishes configured account ids from synthetic fallback ids in the channel summary/status path, preserves diagnostics for genuinely configured but unresolved accounts, and adds Feishu multi-account gateway restart/status regression coverage.
