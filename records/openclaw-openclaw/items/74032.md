## Summary

Keep open. Current main still leaves a plausible Telegram forum-topic `/status` gap: native Telegram slash commands use a separate authenticated command path, current tests cover topic-aware reply delivery, but command-menu sync still registers only the default Telegram command scope and the latest release has the same behavior. Open PR #6457 remains the focused implementation candidate for the group-scope/topic-command fix.

## What This Changes

The issue reports inconsistent Telegram forum-topic `/status` responses even though normal mention-triggered messages reply in the same topic with `requireMention: false`.

## Best Possible Solution

Resolve this in the Telegram plugin by registering and clearing the appropriate Telegram command scopes for group chats while preserving forum-topic thread parameters for native command replies. Core command handling should stay generic; review, update, or replace #6457 as the narrow implementation path.
