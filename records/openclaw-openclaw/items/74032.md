## Summary

Keep open. Current main has topic-aware Telegram routing for ordinary messages and native command replies, but the native slash-command path is still distinct from normal mention/text routing, still applies command authorization separately, and current command-menu sync still registers only the default Telegram command scope. The new report matches a plausible remaining gap, with open PR #6457 already covering the closest Telegram group-scope/topic-command fix candidate.

## Best Possible Solution

Keep this issue open. Reproduce `/status` in a Telegram forum topic on current main with `requireMention:false`, then compare the native slash-command sender against `commands.allowFrom`, channel `allowFrom`, `groupAllowFrom`, per-group/per-topic `allowFrom`, and `groupPolicy`. If the failure is Telegram native command availability or topic fallback, land or replace #6457 in the Telegram plugin with group-scope registration, matching cleanup, and regression coverage. If it is intentional command authorization, improve docs/diagnostics; if delivery is wrong, add a focused `/status` forum-topic regression without mixing in broader `/status` visibility work from #48187.
