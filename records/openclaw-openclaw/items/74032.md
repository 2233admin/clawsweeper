## Summary

Keep this issue open. Current main has topic-aware Telegram native-command plumbing, but the report is new and still matches real current-main gaps between normal Telegram group/topic message routing and the native slash-command path. The most likely remaining causes are Telegram native command group-scope registration/topic fallback work already tracked by #6457, command authorization policy differences for `/status`, or a narrower `/status` delivery regression. None is proven fixed on main.

## Best Possible Solution

Keep this issue open and route it through the Telegram native-command/status owners. Maintainers should first reproduce `/status` in a Telegram forum topic with `requireMention:false` and compare the sender against `commands.allowFrom`, `allowFrom`, `groupAllowFrom`, per-group/per-topic `allowFrom`, and `groupPolicy`. If the failure is Telegram client command availability or thread fallback, land or replace #6457 in `extensions/telegram`; if it is intentional command authorization, update docs and add a targeted diagnostic; if `/status` delivery itself is wrong, add a focused native `/status` forum-topic regression without mixing in the broader `/status` visibility feature from #48187.
