## Summary

Keep open. Current main still has a plausible Telegram forum-topic `/status` gap: native slash commands use a separate authenticated command path, topic-aware reply delivery is partly covered, but Telegram command menu sync and cleanup still only operate on the default command scope instead of group scope. The paired open PR #6457 remains the focused implementation route, though it may need refresh or replacement for the current plugin layout.

## What This Changes

The issue requests reliable Telegram `/status` responses in group forum topics when normal mention-triggered replies already work with `requireMention: false`.

## Best Possible Solution

Resolve this in the Telegram plugin by registering and clearing both the default and `all_group_chats` native command scopes, preserving forum-topic thread parameters for native command replies, and covering the behavior with focused Telegram extension tests. Core command dispatch should stay generic.
