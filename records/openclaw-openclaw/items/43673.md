## Summary

Keep this issue open. Current main has substantial adjacent support for multi-agent routing, per-agent workspaces/auth/session stores, per-agent sandbox/tool restrictions, operator RPC scopes, and channel group/sender tool policies. It still does not ship the central first-class org/team deployment request: a team workspace scaffold CLI, an agents.yaml-style deployment manifest/importer, or a complete per-session/operator-scope RBAC bridge into agent tool policy. The related #49183, #53496, and #53530 items remain narrower RBAC building blocks, not replacements for the umbrella request.

## Best Possible Solution

Keep this open as the umbrella product/design tracker unless maintainers split it. The concrete path is to review or close the focused RBAC work in #49183, #53496, and #53530, then decide whether team workspace scaffolding and deployment manifests belong in core CLI/config, a documented plugin/ClawHub workflow, or a smaller core API that enables external implementation.
