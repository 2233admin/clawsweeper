## Summary

Keep open. Current main has adjacent multi-agent workspaces, per-agent tool policy, group/sender tool overrides, config includes, and documented security guidance, but it still does not provide the requested team workspace scaffold command, deployment manifest, or operator/session-scope RBAC. The issue discussion’s linked policy PRs are useful incremental work, not a complete implementation of the broader org/team deployment request.

## What This Changes

The issue requests first-class org/team deployment support through workspace scaffolding, per-session RBAC-style tool restrictions, declarative deployment manifests, and official org deployment documentation.

## Best Possible Solution

Keep this as a maintainer-scoped umbrella or convert it into a tracked design/RFC, then split implementation into narrower issues for RBAC, workspace/team scaffolding, deployment manifests, and documentation once the desired trust boundary is agreed.
