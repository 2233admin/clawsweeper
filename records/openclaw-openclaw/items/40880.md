## Summary

Keep open. Current main still hardcodes the 5 MB media-store and sandbox staging cap, chat preflight still rejects sandboxed non-image offloads above that same constant, and the strict top-level media config/type surface still has no media.maxBytes key. PR #40950 remains the open unmerged implementation path that explicitly closes this issue, so the issue should stay open until that PR is reviewed, landed, or closed.

## What This Changes

Request to make the hardcoded 5 MB sandbox/media-store staging cap user-configurable instead of requiring patched built bundles after install.

## Best Possible Solution

Land PR #40950 or an equivalent narrow fix that adds a bounded top-level media.maxBytes config with the current 5 MB default, threads the effective value through media store, sandbox staging, and chat preflight, updates schema/help/generated docs as needed, and adds regression coverage for default and configured limits.
