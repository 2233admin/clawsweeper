## Summary

Keep open. Current main has useful session-management pieces, but the WebChat/Control UI New session button still routes through `/new` reset semantics instead of creating and switching to a preserved dashboard session. The directly related fix PR #73725 is still open and unmerged.

## Best Possible Solution

Keep this issue open. Review and land #73725, or an equivalent focused patch, so Control UI/WebChat New Chat calls `sessions.create`, switches to the returned dashboard session key, refreshes `sessions.list`, preserves drafts/attachments/pending state safely, and keeps `/new`/`sessions.reset` as explicit reset behavior with focused UI/controller/backend tests and docs updates.
