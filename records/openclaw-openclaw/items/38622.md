## Summary

Current main still appears to reject workspace bootstrap files that are symlinks to targets outside the workspace boundary, so this issue should stay open until one canonical implementation PR lands. The related context already has active implementation candidates, including #73401 and #73988, so this should not spawn another automated repair lane right now.

## What This Changes

Allow workspace bootstrap files such as SOUL.md and AGENTS.md to be injected when they are symlinks to readable shared files outside the workspace directory.

## Best Possible Solution

Keep this issue open and review or land one canonical implementation PR, preferably the maintainer-routed path in #73401 or a clear successor. The fix should live narrowly in `src/agents/workspace.ts`, follow explicit fixed bootstrap file symlinks to readable regular files, preserve existing safety guards, add regression coverage in `src/agents/workspace.test.ts`, and avoid widening extra/glob bootstrap file boundaries.
