## Summary

Current main still routes fixed workspace bootstrap files through the workspace boundary reader with no symlink fallback. Outside-workspace symlink targets are rejected by boundary resolution, read failures become missing bootstrap records, and the missing records render as the reported `[MISSING] Expected at: ...` text. Keep this open because #73401 and #73988 are open implementation PRs for the same behavior, including a maintainer-labeled path.

## Best Possible Solution

Keep this issue open until a canonical implementation PR lands. The best fix is a narrow src/agents/workspace.ts fallback for explicit fixed workspace bootstrap filenames that follows user-created symlinks to readable regular files while preserving the existing size budget, cache correctness, hardlink/non-regular/dangling protections, and unchanged extra/glob bootstrap boundaries. Review and repair/land #73401 or #73988, or close those PRs with a clear replacement path before closing this issue.
