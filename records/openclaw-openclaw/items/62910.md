## Summary

Keep this PR open. Current main still has the Node v24 DEP0190-triggering `spawn(..., args, { shell: true })` shape in `scripts/ui.js` for Windows command launchers, so the PR is not obsolete. The patch is still not the best merge shape because it manually folds args with `args.join(" ")` instead of using the existing cmd.exe quoting helper, and it lacks regression coverage for the final spawn/spawnSync invocation.

## Best Possible Solution

Keep this PR open and request a revision that routes the Windows command-launcher path through a `shell:false` cmd.exe wrapper using `buildCmdExeCommandLine()` or a shared pnpm-runner-style helper. The revised patch should preserve Program Files launcher paths, quote forwarded args with spaces, keep unsafe metacharacter rejection, and add tests that assert the final spawn/spawnSync call shape avoids `shell:true` with a nonempty args array.
