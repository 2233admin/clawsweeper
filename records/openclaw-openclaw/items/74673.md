## Summary

Keep open. Current main has the compatibility gaps this PR targets, but the branch should not land as-is because it expands Google Vertex ADC evidence into the existing non-secret marker path that current Google transport still serializes as an API-key header rather than converting to ADC/Bearer auth.

## What This Changes

This PR extends Google Vertex ADC manifest auth evidence to accept `GOOGLE_CLOUD_PROJECT_ID` and Windows ADC fallback paths, updates auth-evidence placeholder docs, and adds resolver regression tests.

## Best Possible Solution

Make Google Vertex ADC evidence resolve through a provider-owned request-ready auth path, or defer these manifest compatibility additions until the `gcp-vertex-credentials` marker is no longer passed to the Google transport as an API key.

## Review Findings

- **[P2] Gate ADC evidence on request-ready auth:** `extensions/google/openclaw.plugin.json:84-89`
  - body: The new project-id and Windows fallback entries make additional ADC-only `google-vertex` setups resolve to `gcp-vertex-credentials`. Please wire ADC to a request-ready Bearer/Vertex auth path before broadening this evidence.
