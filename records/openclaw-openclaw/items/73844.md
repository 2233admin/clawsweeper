## What This Changes

This PR changes `isServerErrorMessage` so an exact `status: internal server error` message is classified as a server error after the status phrase is scrubbed away.

## Best Possible Solution

If maintainers accept the semantic change, land the narrow classifier branch with an exact raw-message regression test while preserving the existing conservative behavior for prefixed status prose and the existing timeout behavior for mixed upstream text and explicit `code:500` cases.

## Solution Assessment
