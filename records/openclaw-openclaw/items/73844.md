The PR changes `isServerErrorMessage` so a message that becomes empty after scrubbing `status: internal server error` is classified as a server error.

## Best Possible Solution

If maintainers accept the semantic change, land the narrow classifier update together with an exact `status: internal server error` regression test while preserving the existing conservative behavior for prefixed plain status prose and the existing timeout behavior for mixed upstream text and explicit `code:500` cases.
