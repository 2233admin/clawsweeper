## Summary

Keep this PR open. Current main still has the delete-confirm popover listener leak: Cancel and Delete remove the popover without unregistering the captured document click listener, while only the outside-click path removes it. The PR is a narrow one-file Control UI fix and the security pass did not find CI, dependency, lockfile, script, publishing, secret, or artifact-download changes.

## Best Possible Solution

Keep this PR open for maintainer review and land it or an equivalent current-main patch. The implementation should keep the fix scoped to `ui/src/ui/chat/grouped-render.ts`, route every delete-confirm dismissal path through one cleanup helper, and preferably add a focused regression test in `ui/src/ui/chat/grouped-render.test.ts` that spies on document listener add/remove for Cancel and Delete. No docs or changelog entry is needed for this internal Control UI leak fix.
