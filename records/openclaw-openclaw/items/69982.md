## Summary

Keep this PR open. Current main still leaks the captured document click listener when the delete-confirm popover is dismissed via Cancel/Delete, and the PR patch fixes those paths but still leaves the same-button toggle dismissal path using bare `existing.remove()` before any listener cleanup helper is available.

## Best Possible Solution

Keep this PR open and update it, or land an equivalent current-main patch, so every delete-confirm dismissal path unregisters the captured document click listener: Cancel, Delete, outside click, and clicking the same delete button while the popover is already open. Add focused regression coverage in `ui/src/ui/chat/grouped-render.test.ts`. No docs or changelog entry is needed for this internal Control UI leak fix.
