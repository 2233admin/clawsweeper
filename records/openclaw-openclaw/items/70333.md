## Summary

Current main still persists auto-enable-derived `enabled: true` flags from gateway startup and `doctor --fix`. The older restart-loop hash guard mitigates self-induced restart churn, but the requested runtime-only/non-persistent auto-enable behavior is not implemented.

## Best Possible Solution

Keep this issue open for maintainer product judgment on config canonicalization. If accepted, implement runtime-only auto-enable for gateway startup and doctor diagnostics, do not persist inferred `enabled` flags, preserve explicit user enable/disable writes, keep the existing self-write hash guard, and add regression coverage that startup and `doctor --fix` do not reintroduce the reported keys.
