## Summary

Keep this PR open. Current main still fetches only the first Microsoft Teams Graph replies page for channel-thread context, while this PR is a focused Teams-plugin change that follows `@odata.nextLink`, keeps a bounded recent reply window, preserves already-fetched replies on later-page failures, and adds targeted unit coverage. It is not implemented on main or superseded; it needs normal MS Teams maintainer review and CI/Testbox validation.

## What This Changes

The PR changes the MS Teams Graph thread helper to follow paginated reply pages and keep a bounded recent reply window, with tests for pagination, page caps, and later-page failure behavior.

## Best Possible Solution
