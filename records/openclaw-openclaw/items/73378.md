## Summary

Keep this PR open. Current main still lacks the PR’s intended LINE native sticker send path: there is inbound LINE sticker context, but no `channelData.line.sticker` carrier, no `STICKER:` directive parser, no `pushStickerMessage` helper/export, no outbound sticker dispatch, and the block-reply coalescer still drops payloads with neither text nor media. The PR is also a recent draft implementation candidate paired with same-author open tracker #64784, while the earlier broader #65270 was closed unmerged in favor of this narrower replacement.

## Best Possible Solution

Keep this draft PR open for normal maintainer review. The best path is to decide whether the generic channelData-only coalescer fix should land with the LINE sticker send feature or be split, then review the LINE sticker dispatch, directive parsing, docs, and tests. After a suitable implementation lands, #64784 can remain as the broader send/receive tracker or be narrowed to any remaining inbound/catalog work.
