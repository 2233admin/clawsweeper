## Summary

Keep this PR open. Current main still has the MCP channel bridge leak pattern the PR targets: pending Claude permission and approval entries are stored in raw Maps, cleared only on matching reply/resolved events, and not cleared by `close()`. The supplied PR is a focused implementation with regression tests, is paired with still-open #71646, and its visible security/supply-chain surface is limited to MCP bridge code and tests.

## What This Changes

The PR wraps MCP bridge pending permission/approval entries with timestamps, adds a lazy self-terminating TTL sweeper plus close cleanup and post-close guards, and adds/updates MCP regression tests.

## Best Possible Solution

Land or adapt a focused MCP bridge fix that bounds pending Claude permissions and approvals with stable timestamps, a lazy self-terminating TTL sweeper, close cleanup, post-close guards, and regression tests, while coordinating any cap/FIFO and session-binding follow-ups separately.
