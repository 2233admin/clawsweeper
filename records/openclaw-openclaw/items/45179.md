## Summary

Keep open. Current main has partial temp hardening, but it does not implement a first-class temp artifact provenance/classification model, durable rerouting semantics, janitor-aware cleanup, or broad runtime-critical temp-path validation. The remaining work is broad and security-adjacent, so it needs maintainer and secops design rather than an automated close or one-shot repair PR.

## What This Changes

Request OpenClaw to add default provenance/classification for agent-created temp artifacts, durable-path routing, cleanup awareness, and runtime-critical `/tmp` guardrails.

## Best Possible Solution

Define a first-class temp artifact contract in core, expose a backwards-compatible SDK seam for classification and provenance, migrate OpenClaw-owned temp writers to record it, teach cleanup/doctor/security reporting how to interpret artifact classes, and add explicit runtime-critical temp-path checks at the validation points OpenClaw owns.
