## Summary

Keep open. Current main still has ACP `sessions_spawn` modes limited to `run` and thread-bound `session`; there is no `affinityKey` surface, no non-thread `mode="persistent"` path, and ACP child sessions are still keyed with a random UUID on each spawn. The issue discussion usefully points to a downstream implementation in WuKongAI-CMU/openclaw#3, but that capability has not landed upstream.

## Best Possible Solution

Keep this issue open and either review/adapt the downstream WuKongAI-CMU/openclaw#3 implementation or implement the same upstream contract in `src/agents/tools/sessions-spawn-tool.ts` and `src/agents/acp-spawn.ts`: add `mode: "persistent"` plus `affinityKey`, limit it to cron/hook/orchestrator-owned contexts, derive deterministic ACP session keys scoped by owner identity/backend/cwd/affinityKey, preserve existing affinity sessions when later dispatch fails, and cover the behavior in ACP spawn/tool/docs tests.
