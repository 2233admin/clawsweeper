## Summary

Keep this issue open. Current main still lacks the requested ACP `sessions_spawn` non-thread persistent affinity contract: there is no `affinityKey`, the ACP tool path accepts only `run`/thread-bound `session`, non-thread ACP spawns still allocate random OpenClaw session keys, and docs/tests describe `resumeSessionId` plus configured/bound ACP sessions rather than cron/orchestrator-owned affinity reuse. The linked downstream `WuKongAI-CMU/openclaw#3` implementation remains useful prior art, but I found no upstream implementation in the checked-out main tree.

## Best Possible Solution

Keep the issue open and adapt or reimplement the downstream contract upstream: add `mode: "persistent"` plus `affinityKey` for `sessions_spawn({ runtime: "acp" })`, restrict it to trusted cron/hook/orchestrator contexts, derive deterministic ACP session keys from requester owner identity plus backend, cwd, and affinity key, preserve existing affinity-backed sessions when dispatch fails, and add focused source, tool-schema, docs, and regression coverage.

## Risks / Open Questions

- Implementing this is ownership- and security-sensitive: deterministic ACP reuse needs explicit trusted requester identity, cron/hook/orchestrator authorization, backend/workspace scoping, collision-resistant affinity keys, and cleanup behavior that does not delete an existing affinity-backed session after a later dispatch failure.
