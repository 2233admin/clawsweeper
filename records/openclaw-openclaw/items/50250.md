## Summary

Keep this PR open. Current main still has `HealthSummary.durationMs` and preserves it in `openclaw health --json`, but the non-JSON `openclaw health` renderer still does not print a gateway probe duration line. The paired issue #50245 is open from the same contributor, and this PR remains a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this small change, or an equivalent one, that logs `Gateway probe duration: <n>ms` from `summary.durationMs` in the non-JSON `openclaw health` output, keeps `--json` unchanged, and adds the regression assertion. After merge, #50245 can close normally.
