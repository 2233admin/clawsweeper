## Summary

Keep this PR open. Current main still carries `durationMs` in the health summary and JSON output, but the non-JSON `openclaw health` renderer still logs channel lines and self IDs, then goes straight to `Agents:` without printing the gateway probe duration. The linked issue remains open and this PR is the focused same-author implementation candidate, so it should not be cleanup-closed before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this small change, or an equivalent one, that logs `Gateway probe duration: <n>ms` from `summary.durationMs` in non-JSON `openclaw health` output, keeps `--json` unchanged, and preserves the regression assertion. After it merges, the linked issue can close normally.
