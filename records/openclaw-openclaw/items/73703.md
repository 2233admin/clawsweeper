## Summary

Keep this PR open. Current main does not implement the requested `openclaw/plugin-sdk/discord` subpath, and the PR is a focused implementation candidate for the linked open Discord regression. It should not be closed as cleanup, but it does need maintainer review because current SDK docs and contract tests intentionally keep removed channel-branded SDK aliases such as `discord` out of the public list.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either to revise this PR into an explicitly supported compatibility shim for the published external `@openclaw/discord` package, with SDK contract tests and docs adjusted accordingly, or to fix/release the external Discord plugin so it imports the existing generic subpaths (`plugin-entry`, `config-types`, `runtime-store` or `plugin-runtime`, and `channel-plugin-common`) instead of restoring a channel-branded SDK facade.
