## Summary

Keep open. Current main still has the helper-path gaps this PR targets, and this PR is the active implementation candidate for #57326, but the latest diff still needs maintainer changes because the newly routed canonical-provider cron CLI path can bypass cron job tool restrictions.

## What This Changes

The PR routes queued followups, model auth probes, LLM slug generation, and cron runs with CLI-resolved canonical providers through `runCliAgent()`, adds CLI session/MCP routing metadata and message-tool send tracking, and adds focused regression tests.

## Best Possible Solution

Update this PR so CLI-backed helper paths preserve the same auth, session, routing, cancellation, and cron tool-policy contracts as the embedded paths. In particular, canonical-provider cron runs should not enter the CLI path until `toolsAllow`, message-tool disabling, and related tool restrictions are enforced there; then run focused helper-path and gateway MCP validation before merge.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure an isolated cron job with a canonical provider such as `anthropic`, an agent runtime of `claude-cli`, and a restrictive `--tools` policy; the PR routes that job through the new CLI branch while the CLI params omit the cron tool restrictions that the embedded branch receives.

## Solution Assessment

No. Routing helper paths through `runCliAgent()` is the right direction, but the current PR is not yet the best fix because the new canonical cron CLI path does not preserve the documented cron tool restriction contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Preserve cron tool restrictions on CLI alias runs:** `src/cron/isolated-agent/run-executor.ts:177`
  - body: When a cron job selects a canonical provider that resolves to a CLI runtime, this branch now uses `runCliAgent()`, but it does not forward the job's tool policy (`toolsAllow`, disabled message tool, explicit target requirement) that the embedded cron path receives. A job configured with `--tools read` can therefore get the default CLI MCP tool surface after this change. Add CLI-side tool-policy enforcement or keep those runs on the embedded path until the policy can be preserved.
  - confidence: 0.86
