## Summary

Keep open. Current main still does not implement the requested subagent completion leak fix: tool/toolResult fallback output is copied without checking failed-tool markers, and the newer childCompletionFindings path can bypass the ANNOUNCE_SKIP/silent filtering that is applied to normal replies. The prior direct fix PR #39367 is closed unmerged, while #59649 addressed adjacent internal-context surfaces but not this extractor/announce branch.

## Best Possible Solution

Keep this issue open and land a focused agents fix that covers both current leak paths: filter failed tool/toolResult messages before they populate subagent fallback output, and make the childCompletionFindings branch honor the same ANNOUNCE_SKIP/silent reply handling as the normal reply branch. Add regression tests proving failed tool results are skipped, successful tool-only fallback still works, and child-completion aggregates do not bypass skip/silent output decisions.
