## Summary

Keep this issue open. Current main still has no Telegram `allowBots` config, schema, generated metadata/help, docs, or runtime bot-sender policy for groups/topics. The Plaiground ClawHub skill mentioned in the discussion is a useful Discord workaround, and open PR #68325 is the current implementation candidate, but the requested bundled Telegram behavior is not implemented on main.

## Best Possible Solution

Keep this issue open and review/land PR #68325, or replace it with a fresh Telegram plugin fix that adds `allowBots` consistently across account/group/topic config types, strict schema, generated metadata/help, docs, UI hints, and inbound group/topic runtime filtering. The fix should include regression tests for bot-authored group/topic messages, same-bot echoes, mention gating, and loop-safe defaults.
