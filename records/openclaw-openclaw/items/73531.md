## Summary

Keep open. Current main still performs the duplicated status task reconciliation path, and PR #73650 is an open, unmerged implementation linked with closing syntax, so this issue should remain open until that PR is reviewed, landed, or closed.

## What This Changes

Make `openclaw status` avoid duplicated task reconciliation by sharing or caching one reconciled task snapshot across the registry and audit summaries.

## Best Possible Solution

Review and land PR #73650 if its per-status-call shared snapshot is correct, or merge an equivalent narrow fix that reconciles tasks once inside `getStatusSummary()` and derives both registry and audit summaries from that snapshot while preserving zero-argument helper behavior for other call sites.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main calls both status task helpers in one summary build, and each helper independently calls `reconcileInspectableTasks()` before reaching the clone-and-sort task list path.

## Solution Assessment

No for a module-level process-lifetime cache as the first fix. A per-call shared reconciled snapshot is narrower, avoids invalidation and shared-reference hazards, and matches the open PR direction.
