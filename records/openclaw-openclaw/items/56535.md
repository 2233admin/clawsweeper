## Summary

Keep open. Current main has partial Fake-IP/proxy handling, but it does not fully solve the reported provider-backed image generation failure: shared SSRF still blocks DNS answers in 198.18.0.0/15 by default, provider HTTP auto-upgrade only applies when HTTP_PROXY/HTTPS_PROXY will actually proxy the target, Google image generation needs an explicit broad private-network opt-in, and the direct OpenAI image path does not appear to honor models.providers.openai.request.allowPrivateNetwork. This is a real security-sensitive product bug, not stale cleanup or ClawHub work.

## Best Possible Solution

Keep this issue open and implement a security-reviewed provider HTTP compatibility path: add narrow RFC2544/Fake-IP handling or trusted proxy mode for provider-backed Google/OpenAI image and model requests, make OpenAI and Google honor the same documented provider request policy, add regression tests where generativelanguage.googleapis.com and api.openai.com resolve to 198.18.x.x, and document the safe operator configuration and error message tradeoffs.
