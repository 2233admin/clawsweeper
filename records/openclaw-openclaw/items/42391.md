## Summary

Keep this issue open. Current main still exposes typed plugin hooks for `before_tool_call`, `after_tool_call`, and `agent_end`, but their contracts, runtime emitters, harness/CLI relay paths, tests, and public docs do not provide the requested structured progress `metrics` object (`stepIndex`, `stepTotal`, `percent`, `etaSec`). The request is a concrete plugin API gap for live progress cards, not something a Feishu plugin can reliably solve outside core today.

## Best Possible Solution

Keep this open for maintainer-led additive plugin API design. The best path is either an optional `metrics` object on `before_tool_call`, `after_tool_call`, and `agent_end`, or a dedicated progress hook, emitted only when OpenClaw has a defensible monotonic step model. Update `src/plugins/hook-types.ts`, runtime emitters across Pi and harness paths, focused tests for monotonic/success/failure behavior, and the public plugin hooks docs at https://docs.openclaw.ai/plugins/hooks.
