## Summary

Keep open. Current main still has typed plugin hooks and adjacent sanitized agent event streams, but the requested optional numeric progress metrics are not exposed on `before_tool_call`, `after_tool_call`, or `agent_end` hook payloads, and the emitters do not derive or publish `stepIndex`, `stepTotal`, `percent`, or `etaSec`.

## Best Possible Solution

Keep this open and design an additive progress observation contract. The conservative path is to add a shared optional progress metrics type to the relevant hook events, or introduce a dedicated progress hook, emit metrics only when OpenClaw has a defensible step model, document absent/unknown semantics in the public plugin docs, and add hook contract plus runtime tests for monotonicity, successful completion, failure/cancellation, and compatibility.

## Risks / Open Questions

- A numeric progress contract can mislead users unless unknown totals, dynamic plans, retries, subagents, failures, and cancellations are specified explicitly.
- This is a public plugin SDK/API change, so implementation should include shared types, docs, and focused runtime/contract tests for backward compatibility and terminal success/failure behavior.
