## Summary

Keep open. Current main still exposes typed plugin hook events for tool calls and agent end, but the requested optional numeric progress metrics are not part of the public hook contract and are not derived by the current emitters. The earlier ClawSweeper comment remains accurate against current main SHA 7a23b2d945cc2d2e5c452e4641213e71de8dcc66.

## Best Possible Solution

Keep this issue open and design an additive progress observation contract. The conservative path is to add a shared optional metrics type to the relevant hook events, or introduce a dedicated progress hook, emit metrics only when OpenClaw has a defensible step model, document absent/unknown semantics in the public plugin docs, and add hook contract plus runtime tests for monotonicity, successful completion, failure/cancellation, and backwards compatibility.

## Risks / Open Questions

- A numeric progress contract can mislead users unless unknown totals, dynamic plans, retries, subagents, failures, and cancellations have explicit semantics.
- This is a public plugin SDK/API change, so implementation should include shared types, docs, and focused runtime/contract tests for compatibility and terminal success/failure behavior.
