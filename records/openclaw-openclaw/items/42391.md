## Summary

Keep open. Current main has typed plugin hooks for tool calls and agent completion, plus adjacent structured agent item/plan events, but the public hook contracts and emitters still do not expose the requested optional numeric progress metrics (`stepIndex`, `stepTotal`, `percent`, or `etaSec`). The prior ClawSweeper keep-open comment remains accurate against current main.

## Best Possible Solution

Keep this issue open and design an additive, backward-compatible progress observation contract. The most conservative path is to add a shared optional progress metrics type to the relevant hook events or introduce a dedicated progress hook, emit metrics only when OpenClaw has a defensible step model, document absent/unknown semantics in the public plugin docs, and add focused hook contract plus runtime tests.

## Risks / Open Questions

- A numeric progress contract can mislead users unless it defines unknown totals, dynamic plans, retries, subagents, failures, and cancellations explicitly.
- This is a public plugin SDK change, so implementation should include shared types, docs, and regression coverage for monotonicity, terminal success behavior, failure/cancel behavior, and backward compatibility.
