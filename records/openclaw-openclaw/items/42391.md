## Summary

Keep this issue open. Current main still exposes the relevant hook events, but the plugin-facing contracts and runtime emitters do not provide structured progress metrics such as stepIndex, stepTotal, percent, or etaSec on before_tool_call, after_tool_call, or agent_end. The related context covers adjacent hook/lifecycle observability, not the numeric progress contract requested here.

## Best Possible Solution

Keep this open for an additive plugin-facing progress observation contract. The best path is optional metrics on the requested hook events or a dedicated progress hook, emitted only when OpenClaw has a defensible step model, with documented absent/unknown semantics and tests for monotonicity, success completion, failure/cancellation behavior, and compatibility with existing plugins.

## Risks / Open Questions

- A numeric progress contract can mislead users unless OpenClaw defines unknown totals, changing plans, retries, subagents, failures, cancellations, and terminal success semantics.
- This is public plugin API work, so a fix needs typed contracts, docs, and focused runtime/contract tests rather than a plugin-only workaround.
