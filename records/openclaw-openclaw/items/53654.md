Keep this open. Current main still lacks Discord edit/delete inbound handling, and the related open items only cover the edit half or other channels, so there is no implemented or superseding OpenClaw fix to close against.

## What This Changes

Request support for Discord messageUpdate/messageDelete events so edited source messages can reprocess and deleted source messages can cancel the matching in-progress run.

## Best Possible Solution

Implement this in the bundled Discord plugin with typed update/delete listener support, startup registration, edit-aware replay or supersede handling, and a precise source-message-id cancellation map that aborts only the matching active run. Keep the narrower edit trackers coordinated, but ship both edit-to-reprocess and delete-to-cancel before closing this broader request.
