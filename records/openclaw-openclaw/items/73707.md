## Summary

Keep this PR open. Current main does not implement the proposed `skills.entries.<skill>.persist`, `metadata.openclaw.persist`, or `skills.trustedSources` behavior, and the PR is an active, unmerged contributor implementation for the still-open #73549. The `trustedSources` portion is also security-sensitive because it would skip skill code-safety scanning for configured source classes, so it needs explicit maintainer product/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open and review it with #73549. Maintainers should decide whether to document the existing `metadata.openclaw.always` path, add a schema-backed operator override such as `skills.entries.<skill>.persist`, or design a narrower trust/audit API. The scanner-bypass portion should not land without clear security semantics, warnings, docs, and tests covering source-class trust behavior.
