## Summary

Keep this issue open. Current main still stores cron-friendly labels but does not carry `label`/`displayName` through the status summary contract or render them in CLI `openclaw status`, TUI `/status`, or the TUI footer/statusline. The Control UI dropdown part is already addressed, and open PR candidates #51257 plus the fork PR remain the right implementation path.

## Best Possible Solution

Keep this issue open until maintainers review and land a rebased implementation from #51257 or the fork PR, or close those candidates and implement the same behavior directly. The fix should carry `label` and `displayName` through `SessionStatus` and `GatewayStatusSummary`, populate them from `SessionEntry`, render a sanitized `displayName`/`label` fallback in CLI status, TUI `/status`, the TUI footer/statusline, and the session picker, and add focused tests for cron-labeled sessions plus fallback-to-key and sanitization behavior.
