## Summary

Current main still does not carry session `label` or `displayName` through the `openclaw status` / TUI `/status` summary contract, and the active renderers still print raw session keys. The Control UI/session picker side has friendly-label handling, but the reported status/statusline gap remains and open PR #51257 is already paired with this work.

## Best Possible Solution

Keep this issue open until maintainers review and land a rebased implementation from #51257 or the fork PR, or close those candidates and implement the same behavior directly. The fix should add `label` and `displayName` to `SessionStatus` and `GatewayStatusSummary`, populate them from `SessionEntry`, render a sanitized `label`/`displayName` fallback in CLI status, TUI `/status`, and the footer/statusline, and add focused fallback plus sanitization tests.
