## Summary

Keep this PR open. Current main still lacks the stderr-derived manual repair hint for QMD path/pattern conflicts when `qmd collection list` only exposes names, while the PR is narrow, non-destructive, and aligned with the open related bug. I did not find a blocking correctness or security issue in the latest diff; normal maintainer review and completion of CI/security checks are still needed before landing.

## What This Changes

The PR adds a QMD add-error parser that extracts the stale collection name for unverifiable path/pattern conflicts, logs a manual `qmd collection remove <name>` repair hint without automatic removal, and adds regression coverage for the stderr-only legacy-list case.

## Best Possible Solution

Land this focused memory-core warning if maintainers accept the wording/parser and CI/security checks complete cleanly, while leaving #71783 open for any future verified automatic repair path that can safely reconcile different-name collections with structured metadata.
