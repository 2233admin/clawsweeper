## Summary

Keep open. Current main still has the old opt-in behavior for empty-output successful background exec completions: the exec tool factory and runtime both require `notifyOnExitEmptySuccess === true`, tests still assert the default skips no-op completion events, and config docs/schema still document the default as false. The PR is not implemented on main and is not superseded by the related open `onComplete="notify"` PR or the broader config-policy issue.

## Best Possible Solution

Keep this PR open for maintainer review or rebase. The narrow implementation path is still to make both `createExecTool` and `runExecProcess` treat `notifyOnExitEmptySuccess` as true unless explicitly false, preserve explicit opt-out paths such as gateway/cron suppression, and update tests plus config docs/generated schema consistently. Review #54549 separately because it adds an explicit API but does not replace this default-behavior change.
