## Summary

Keep this PR open. Current main still lacks the active reply-run guard in `runHeartbeatOnce`, while the reply-run registry exposes the state needed to avoid heartbeat/system-event preemption after the lane has drained. The linked bug remains open, the prior identical PR was closed unmerged by automation, and this PR is a narrow implementation candidate with no observed security-sensitive diff.

## Best Possible Solution

Review, rebase if needed, and land this PR or an equivalent focused patch. The fix should check `resolveActiveReplyRunSessionId(sessionKey)` after heartbeat preflight resolves the canonical `sessionKey` and before the session-lane queue-size check, return the existing `requests-in-flight` skip so wake retry behavior is preserved, and keep the regression test for an active `ReplyOperation` with otherwise idle queues. Keep #64810 open until the fix merges.
