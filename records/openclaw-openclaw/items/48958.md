## Summary

Keep this PR open. Current main has not implemented the PR's intended memory-context image filtering: plugin `prependContext` is still prepended into the same prompt text that `detectAndLoadPromptImages()` scans, and the bundled memory plugin still emits `<relevant-memories>` context whose escaped memory text can retain `[media attached: ...]` markers. Main has a newer history-image pruning path, but that only sanitizes old active-session replay messages before hooks run; it does not sanitize hook-injected memory context before image detection. The PR is focused and security-positive, but its open review comments about the case-sensitive fast path and vacuous test should be addressed before landing.

## Best Possible Solution

Keep this PR open and either update it or land an equivalent fix on main. The best implementation is to exclude injected memory/hook context from prompt-local image reference detection, preserve normal current-turn image loading, and add regression coverage that exercises `before_prompt_build` / `<relevant-memories>` with a `[media attached: ...]` marker plus a real current prompt image outside the memory block. Address the existing review comments before merge.
