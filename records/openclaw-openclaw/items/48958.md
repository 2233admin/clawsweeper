## Summary

Keep this PR open. The linked Telegram issue is closed, but this PR’s own memory-context image injection path is still plausible on current main: hook prepended context can still reach prompt image detection, `memory-lancedb` still emits `<relevant-memories>` as `prependContext`, and current main has no sanitizer or regression coverage for this path. The PR is directionally useful but should address the existing review comments before landing.

## Best Possible Solution

Keep this PR open and land an updated version, or an equivalent main-branch fix, that excludes injected hook/memory context from prompt-local image detection while preserving normal current-turn image loading. Add regression coverage with a `before_prompt_build` memory block containing a `[media attached: ...]` marker plus a real current prompt image outside that block, and address the existing review comments before merge.

## Risks / Open Questions

- The PR should not land exactly as supplied until the case-sensitive `text.includes("relevant")` fast path and vacuous `refs.length >= 0` test assertion from the existing review comments are fixed.

- The CLI external-harness path also prepends hook context and then calls `detectImageReferences()` on the prepared prompt, so an equivalent fix may need to cover more than the embedded image loader.
