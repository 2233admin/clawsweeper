## Summary

Keep this PR open. Current main has partial overlap from strict-agentic/update_plan handling and the 120s LLM idle default, but it still does not implement the PR's central dispatch-level plan-stall fail-close, memory session-delta fetch-failure retry/backoff, or memory search fallback behavior. The PR also still carries unresolved review risks and mixes several runtime surfaces, so it should be rebased and split rather than closed as obsolete.

## Best Possible Solution

Keep this PR open and ask for a rebase plus split patches. The useful slices are dispatch-level fail-closed handling for plan/control progress stalls, memory session-delta fetch-failure retry/backoff that preserves pending work, and memory search fallback that preserves existing hybrid/vector semantics. Treat the strict-agentic work and 120s idle timeout on main as partial overlap, not a replacement.
