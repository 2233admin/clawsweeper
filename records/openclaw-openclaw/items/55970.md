## Summary

Keep open. Current main does not implement this PR’s intended Telegram document fallback reason: failed getFile/file-too-big document paths still collapse to null media and the inbound body has no unavailable-reason field. The PR remains potentially useful, but it should address the existing debounce propagation review before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to update the Telegram implementation so failed inbound document downloads return a structured unavailable reason, final body rendering preserves user text plus <media:document> plus the reason, and debounce paths propagate or intentionally handle that reason with focused tests.
