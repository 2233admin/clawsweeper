## Summary

Keep this PR open. Current main still does not surface session label/displayName through the status summary, CLI status table, or TUI /status summary, so the intended change is not implemented on main. The PR is plausible but needs rework against the current status.command-sections pipeline plus the two existing review concerns: prefer a resolvable label over displayName, and sanitize untrusted friendly names before printing them.

## Best Possible Solution

Rework and review this PR against current main: add label/displayName to SessionStatus and GatewayStatusSummary, copy them from SessionEntry in status.summary.ts, render a sanitized friendly session name in status.command-sections.ts and tui-status-summary.ts, prefer the resolvable label over displayName or show both clearly, and add focused CLI/TUI status tests including unsafe control characters. That would make the PR a good implementation candidate for #51184.

## Risks / Open Questions

- The proposed diff would print raw label/displayName values in terminal and TUI output; labels are only trimmed/length-checked today, so control characters or ANSI escapes need sanitization before merge.
- Preferring displayName before label can hide the selector that follow-up commands can actually resolve, because sessions.resolve accepts label but not displayName.
- The PR patch targets the older status.command.ts rendering site; current main routes CLI rows through status.command-sections.ts, so it needs a small rebase/rework rather than direct cleanup closure.
