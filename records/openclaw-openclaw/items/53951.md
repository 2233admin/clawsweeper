## Summary

Keep open. The Defuddle idea may still be useful, but this branch is stale against current main's plugin-owned web-content-extractor architecture and adds the new parser dependency at the root runtime level, so it cannot merge as-is without maintainer product and security direction.

## What This Changes

This PR adds Defuddle as the primary `web_fetch` HTML extractor with Readability fallback, updates extractor tagging/config/docs/tests/scripts, and adds the `defuddle` dependency.

## Best Possible Solution

Rework the feature around the current `webContentExtractors` contract: add a plugin-owned Defuddle extractor with its dependency, ordering, docs, config/help text, and tests, or first extend the plugin runtime/API if maintainers want external ClawHub extractor plugins to participate. Keep extraction local and bounded, including `useAsync: false`, and preserve Readability as fallback only if maintainers choose Defuddle as the preferred default.
