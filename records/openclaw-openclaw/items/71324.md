## Summary

Keep open. The Gateway legacy-history bug is valid and current main does not fully cover the PR's quoted runtime/status shapes, but the current PR head still introduces two correctness regressions: it can drop reset-tail user text after a legacy startup prompt and it applies legacy status stripping to assistant-authored quoted content before any user-role guard.

## What This Changes

The branch adds Gateway chat-history display sanitization for legacy quoted internal-context wrappers, background-task and gateway-restart status rows, empty-row filtering, regression tests, and a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection approach, but make the legacy cleanup role-aware, strip only known runtime headers/status blocks from legacy user/runtime transcript rows, preserve any following user-authored text, keep raw transcript sequence accounting unchanged, and add regressions for reset-tail preservation plus assistant quote preservation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project a PR-head user row beginning with the legacy startup/reset prompt plus real tail text, or an assistant row quoting a matching `System: [...] Background task ...` line; the current PR code empties or strips those cases before the intended boundaries are enforced.

## Solution Assessment

No. The projection layer is the right place for legacy display cleanup, but this implementation is not yet the narrowest maintainable fix because it over-strips mixed reset rows and assistant-authored content.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve reset-tail text after legacy startup prompts:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any text that starts with the legacy startup/reset prompt. Legacy rows can combine that runtime header with a real reset tail, and current main explicitly keeps reset user notes visible, so this drops user-authored history instead of only removing the runtime header.
  - confidence: 0.87
- **[P2] Gate legacy status stripping by user role:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy background-task and gateway-restart prefixes before the user-role-specific guard is applied. Assistant messages that quote those `System: [...]` lines will be rewritten or emptied in chat history even though this cleanup targets legacy user/runtime transcript rows.
  - confidence: 0.88
