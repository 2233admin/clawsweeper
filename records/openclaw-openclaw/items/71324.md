## Summary

Keep open. The PR targets a real WebChat/history legacy transcript leak that current main does not fully handle, but the latest branch still has merge-blocking sanitizer correctness issues around preserving user-authored prompt tails and avoiding rewrites of legitimate assistant text.

## What This Changes

The PR adds Gateway chat.history display sanitization for legacy quote-wrapped runtime/status transcript rows, expands session-history regression coverage, adjusts raw transcript sequence filtering, and adds a WebChat/history changelog entry.

## Best Possible Solution

Keep the Gateway display-projection approach, but make the sanitizer role-aware and tail-preserving: strip known persisted runtime/status material only from user/runtime legacy rows, preserve following real user text, leave assistant quotations intact, and add regressions for those boundaries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project PR-head history containing a user row that starts with the legacy startup/reset prompt followed by real text, and an assistant row that quotes a legacy System background-task or Gateway restart line.

## Solution Assessment

No. The projection layer is the right place, but the current implementation is not the best fix as written because it deletes prompt-prefixed tails and applies legacy status stripping before role gating.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any row that starts with a known legacy runtime prompt. Legacy reset/startup rows can include that header followed by real user text, and current main explicitly keeps reset tails visible, so this can hide user-authored history instead of only removing runtime text.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before any role gate, and callers pass assistant content through the same helper. Assistant messages that quote or explain those `System:` lines can therefore be rewritten or blanked even though the PR is scoped to persisted user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` checks `text.trimStart()`, but the status stripping helpers test the raw string. A quoted legacy row with leading whitespace can be unwrapped as runtime text and then fail to strip the status prefix, leaving the legacy status visible.
  - confidence: 0.76
