## Summary

Keep open. The PR targets the right Gateway display-projection surface, but the latest head still has blocking sanitizer regressions: it can drop user-authored text after legacy startup headers and can rewrite assistant content because legacy status stripping runs before any role-specific guard.

## What This Changes

The PR expands Gateway chat-history sanitization to unwrap or strip legacy runtime/status wrappers from projected history, filters now-empty user history rows, adds session-history regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the display-projection approach, but make the new legacy sanitizer role-aware and tail-preserving: strip known legacy runtime/status prefixes only from persisted user/runtime rows, preserve any following user-authored text, leave assistant content intact, and add focused regressions for reset-tail, assistant quote, and leading-whitespace status cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project a PR-head user row beginning with the legacy reset/startup prompt followed by real text, and a PR-head assistant row quoting a legacy Background task status line; the current PR code paths show the former is emptied and the latter is stripped before role gating.

## Solution Assessment

No. The projection layer is the narrow maintainable surface, but this implementation is not the best fix as written because it globally strips legacy status/prompt shapes instead of limiting the cleanup to the legacy user/runtime transcript rows it intends to repair.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any row that starts with a legacy reset/startup prompt. Legacy persisted rows can include that header followed by a real reset tail, and current main explicitly keeps reset user notes visible, so this can hide user-authored history instead of only removing runtime text.
  - confidence: 0.87
- **[P2] Limit legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before the `stripUserEnvelope` branch. Assistant messages that quote or explain one of those `System: [...]` lines can therefore be rewritten or emptied even though the PR is scoped to legacy persisted user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-99`
  - body: `looksLikeLegacyRuntimeText` tests status regexes against `text.trimStart()`, but the stripping helpers test the raw string. A legacy row with leading whitespace before `System:` can be classified as runtime text during quote unwrapping while the actual prefix stripping no-ops.
  - confidence: 0.74
