## Summary

Keep open. The PR targets the right Gateway WebChat history projection gap left after the newer write-path fix, but the latest head still has concrete sanitizer regressions that can hide user-authored reset tails and rewrite legitimate assistant text that quotes legacy status lines.

## What This Changes

The PR adds Gateway chat.history sanitization for legacy quote-wrapped runtime/status transcript rows, updates raw history projection tests, and adds a WebChat/history changelog entry.

## Best Possible Solution

Keep the Gateway display-projection approach, but make the legacy sanitizer role-aware and tail-preserving: strip only known legacy runtime/status material from user/runtime legacy rows, preserve following real user text, leave assistant quotations intact, and add regressions for those boundaries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project PR-head history containing a user row that starts with the legacy startup/reset prompt followed by real text, plus an assistant row that quotes a legacy Background task or Gateway restart line.

## Solution Assessment

No. The projection layer is the right place, but this implementation is not the narrowest maintainable fix until legacy prompt/status stripping is gated to the intended user/runtime rows and preserves real text after prompt headers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any row that starts with a known legacy runtime prompt. Legacy reset/startup rows can include that header followed by real user text, and current main keeps reset tails visible, so this can hide user-authored history instead of only removing runtime text.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before any role gate, and callers pass assistant content through the same helper. Assistant messages that quote or explain those `System:` lines can therefore be rewritten or blanked even though the PR is scoped to persisted user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` checks `text.trimStart()`, but the status stripping helpers test the raw string. A quoted legacy row with leading whitespace can be unwrapped as runtime text and then fail to strip the status prefix, leaving the legacy status visible.
  - confidence: 0.76
