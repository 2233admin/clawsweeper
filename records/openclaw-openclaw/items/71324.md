## Summary

This PR should stay open for author changes and maintainer review. The current `main` Gateway history path is the right surface and still does not cover the legacy quote/background/gateway status shapes this branch targets, but the latest PR head still has two blocking display-sanitizer regressions: it can drop reset-tail user text and it applies new legacy status stripping to assistant messages before the user-role guard.

## What This Changes

The PR adds Gateway chat-history display sanitization for legacy quoted runtime/internal context, background-task and gateway-restart status rows, regression tests, and a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection approach, but update it to remove only known legacy runtime headers/status blocks from user/runtime transcript rows, preserve following user-authored text, keep raw transcript sequence accounting unchanged, and add focused regressions for reset-tail preservation and assistant quote preservation.

## Reproduction Assessment

Yes. A high-confidence code-only reproduction is to pass a user row beginning with the legacy startup/reset prompt plus real tail text, or an assistant row quoting a legacy `System: [...] Background task ...` line, through the PR-head projection path; source inspection shows the first is emptied and the second is stripped before any user-role guard.

## Solution Assessment

No, not as written. The projection layer is the narrow right place for legacy cleanup, but this implementation needs role-aware stripping and mixed-row tail preservation before it is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve reset text after legacy startup prompts:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any text that starts with the legacy startup/reset prompt. Legacy rows can combine that runtime header with a real reset tail, and current behavior preserves that tail as transcript text, so this drops user-authored history instead of projecting only the runtime header out.
  - confidence: 0.86
- **[P2] Gate legacy status stripping by user role:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy background-task and gateway-restart prefixes before the user-role-specific guard is applied. Assistant messages that quote those `System: [...]` lines will be rewritten or emptied in chat history even though this repair targets legacy user/runtime transcript rows.
  - confidence: 0.88
