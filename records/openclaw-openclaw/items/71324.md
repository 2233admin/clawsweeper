## Summary

Keep open. Current main still does not handle these legacy WebChat history projection cases, so the PR is addressing a real Gateway bug, but the latest PR head still introduces two correctness regressions in the sanitizer: mixed legacy prompt rows can lose following real text, and assistant-authored messages that quote legacy status lines can be rewritten or emptied.

## What This Changes

The PR adds Gateway `chat.history` display sanitization for legacy quote-wrapped runtime context, background-task and gateway-restart status rows, filters now-empty history rows, adds session-history regression tests, and updates the changelog.

## Best Possible Solution

Land a constrained Gateway display-projection fix that handles already-persisted legacy runtime-only rows, preserves following real user text, and limits legacy runtime/status cleanup to the persisted user/runtime transcript shapes that carried those artifacts. Keep raw sequence preservation and focused `buildSessionHistorySnapshot` plus sanitizer regression coverage.

## Reproduction Assessment

Yes. A high-confidence code-only reproduction is to pass a user row like `A new session was started via /new or /reset.\n<actual user text>` or an assistant row beginning with `System: [timestamp] Background task done: ...` through the PR-head sanitizer paths; I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No, not as written. Gateway history projection is the right layer for legacy display cleanup, but the patch needs mixed-row preservation and role-aware status stripping before it is the narrowest maintainable solution.

Apply these legacy repairs only to the persisted user/runtime transcript shapes they target.
