## Summary

Keep open. Current main still lacks the specific legacy wrapper/status cleanup this PR targets, and the latest PR head still has two blocking correctness issues in the sanitizer: it can erase real reset-tail text after a startup prompt and it strips legacy status-looking text before checking that the message is a user transcript row.

## What This Changes

The PR adds Gateway chat-history display sanitization for legacy quote-wrapped runtime/internal-context rows, background-task and gateway-restart status prefixes, empty visible-row filtering, regression tests, and a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection approach, but make legacy runtime/status cleanup role-aware and prefix-specific, preserve any real user text that follows known runtime headers, and add focused regressions for reset-tail preservation and assistant quote preservation before normal maintainer review.

## Reproduction Assessment

Yes. A high-confidence code-only reproduction is to project a PR-head user row beginning with the legacy startup/reset prompt followed by real text, or an assistant row quoting a legacy System background-task line; source inspection shows the first is emptied and the second is stripped before any user-role guard.

## Solution Assessment

No, not as written. The projection layer is the right narrow surface, but this implementation needs tail preservation and role-aware status stripping before it is the maintainable fix.

  - confidence: 0.88
