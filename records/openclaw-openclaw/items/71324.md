## Summary

Keep open: the legacy-history bug is valid and current main does not cover the new legacy status/quote shapes, but the latest PR head still introduces two blocking display-projection regressions that can hide real reset text and rewrite assistant content.

## What This Changes

The PR adds Gateway chat.history display sanitization for legacy quote-wrapped runtime context, background-task and gateway-restart status rows, filters now-empty visible history rows, adds regression tests, and updates the changelog.

## Best Possible Solution

Update the active PR: keep the Gateway history projection approach, but make the legacy cleanup role-aware for user/runtime transcript rows, preserve real text after startup/reset prompt headers, keep raw transcript sequence accounting unchanged, and add focused regressions for reset-tail preservation and assistant quote preservation.

## Reproduction Assessment

Yes. A high-confidence code-only reproduction is to pass a legacy user row beginning with the reset startup prompt plus real tail text, or an assistant row quoting a legacy System background-task line, through the PR-head sanitizer/projection path; source inspection shows the first is emptied and the second is stripped before any user-role guard.

## Solution Assessment

No, not as written. The display projection layer is the right place for legacy cleanup, but the implementation needs role-aware stripping and mixed-row tail preservation to be the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve reset text after legacy startup prompts:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any text that starts with the legacy startup/reset prompt. Legacy rows can combine that runtime header with a real reset tail, and current behavior preserves that tail as transcript text, so this drops user-authored history instead of projecting only the runtime header out.
  - confidence: 0.86
- **[P2] Gate legacy status stripping by role:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy background-task and gateway-restart prefixes before the user-role-specific guard is applied. Assistant messages that quote those `System: [...]` lines will be rewritten or emptied in chat history even though this repair targets legacy user/runtime transcript rows.
  - confidence: 0.88
