## Summary

Keep open. The PR targets a real legacy WebChat/history leak and uses the right display-projection layer, but the current head still has sanitizer regressions that can hide reset-tail user text and rewrite assistant messages that quote legacy `System:` lines.

## What This Changes

The PR adds Gateway `chat.history` legacy runtime/status transcript sanitization, filters now-empty visible history rows while preserving raw transcript sequence accounting, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection approach, but make the legacy sanitizer role-aware and tail-preserving: strip only known legacy runtime/status material from intended legacy user/runtime rows, preserve following real user text, leave assistant quotations intact, and add focused regressions for those boundaries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to apply the PR-head sanitizer to a legacy user row that starts with the reset/startup prompt followed by real text, and to an assistant row that quotes a legacy Background task or Gateway restart status line.

## Solution Assessment

No. The projection layer is the right place, but this implementation is not the narrowest maintainable fix until legacy prompt/status stripping is gated to the intended rows and preserves real user text after legacy headers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:88-90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` whenever a row starts with the reset/startup prompt. Legacy rows can include that runtime header followed by real reset text, and current main explicitly keeps reset tails visible, so this can drop user-authored history.
  - confidence: 0.9
- **[P2] Restrict legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before any role gate, and callers pass assistant content through this helper too. Assistant messages that quote those `System:` lines can therefore be rewritten or blanked.
  - confidence: 0.88
- **[P3] Trim status text before matching legacy prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: The runtime detector tolerates leading whitespace, but the status-prefix strippers test the raw string. A persisted row with whitespace before `System:` would remain visible even though it is the same legacy status shape.
  - confidence: 0.72
