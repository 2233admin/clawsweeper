## Summary

Keep open. The PR targets a real remaining legacy WebChat/history projection gap after #71229 and supersedes the earlier unmerged #71312 attempt, but the latest head still has blocking sanitizer regressions: it can drop user-authored text after legacy prompt headers and rewrites assistant content that merely quotes legacy status text.

## What This Changes

The PR extends Gateway chat.history/WebChat display sanitization for legacy quote-wrapped runtime context, background-task and gateway-restart status rows, adds session-history regression tests, and updates the changelog.

## Best Possible Solution

Keep the Gateway display-projection approach, but make the legacy sanitizer role-aware and tail-preserving: strip known legacy runtime/status material only from persisted user/runtime transcript rows, preserve following user text, leave assistant quotations intact, and add focused regressions for those boundaries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project PR-head history containing a user row that starts with the legacy reset/startup prompt followed by real text, and an assistant row that quotes a legacy `System: [...] Background task` or Gateway restart line.

## Solution Assessment

No. The projection layer is the right place, but this implementation is not the best fix as written because it deletes prompt-prefixed tails and applies new legacy status/prompt stripping before role gating.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row that starts with one of the legacy runtime prompt strings. Legacy reset/startup rows can include that header followed by real user text, and current main explicitly keeps reset tails visible, so this can hide user-authored history instead of only removing runtime text.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before any role gate, and callers pass assistant content through the same function. Assistant messages that quote or explain those `System:` lines can therefore be rewritten or blanked even though the PR is scoped to persisted user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` matches against `text.trimStart()`, but the status stripping helpers test the raw string. A quoted legacy row with leading whitespace can be unwrapped as runtime text and then fail to strip the status prefix, leaving the legacy status visible.
  - confidence: 0.76
