## Summary

Keep open. The bug direction is valid and current main still lacks this legacy cleanup, but the PR head remains unsafe because its sanitizer can drop real reset-tail text and rewrite assistant content that happens to quote legacy status lines.

## What This Changes

The PR extends Gateway chat-history display sanitization for legacy quote-wrapped runtime context, background-task and gateway-restart status rows, adds session-history regressions, and updates the changelog.

## Best Possible Solution

Keep the Gateway display-projection approach, but make the sanitizer role-aware and tail-preserving: strip only known legacy runtime/status material from persisted user/runtime rows, preserve following user-authored text, leave assistant content intact, and add focused regressions for those boundaries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project a PR-head user row that starts with the legacy reset/startup prompt followed by real user text, and a PR-head assistant row that quotes a legacy Background task or Gateway restart System line.

## Solution Assessment

No. The projection layer is the right place, but this implementation is not the best fix as written because it applies legacy status stripping globally and treats prompt headers as whole-message deletion instead of preserving real transcript tails.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any row that starts with a legacy reset/startup prompt. Legacy rows can include that header followed by real reset-tail text, and current main explicitly keeps reset user notes visible, so this can hide user-authored history instead of only removing runtime text.
  - confidence: 0.87
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before the user-role branch, and it is called for assistant content too. Assistant messages that quote or explain those `System:` lines can therefore be rewritten or blanked even though the PR is scoped to persisted user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` matches against `text.trimStart()`, but the stripping helpers test the raw string. A quoted legacy row with leading whitespace can be unwrapped as runtime text and then fail to strip the status prefix, leaving the legacy status visible.
  - confidence: 0.74
