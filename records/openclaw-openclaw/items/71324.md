## Summary

Keep open. The PR targets a real Gateway/WebChat legacy-history gap that current main does not fully cover, but the latest PR head still has merge-blocking sanitizer correctness issues around tail preservation and role-aware stripping.

## What This Changes

The PR adds Gateway chat-history sanitizers, regression tests, and a changelog entry to hide legacy quote-wrapped runtime/context, background-task, gateway-restart, subagent-completion, and startup-prompt rows from WebChat/history.

## Best Possible Solution

Keep the Gateway display-projection approach, but change the sanitizer so it removes only known legacy runtime/status prefixes from user/runtime rows, preserves any following user-authored text, leaves assistant quotes intact, and adds focused regressions for startup/reset tails, assistant status quotes, and leading-whitespace legacy status prefixes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project a PR-head user row that starts with a legacy startup/reset prompt followed by real user text, or an assistant row quoting a legacy System background-task line; source inspection shows the former is emptied and the latter is stripped before any user-role guard.

## Solution Assessment

No, not as written. The projection layer is the right narrow surface, but the implementation should be role-aware and tail-preserving rather than wholesale-dropping startup-prefixed rows or stripping status prefixes from non-user messages.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any text that starts with a legacy runtime prompt header. Legacy rows can contain that header followed by a real reset/startup tail, and current main explicitly preserves reset tails as transcript-visible text, so this can hide user-authored history instead of only removing the runtime header.
  - confidence: 0.87
- **[P2] Limit legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:124-127`
  - body: `stripVisibleTranscriptText` strips legacy background-task and gateway-restart prefixes before the role-specific branch runs. Assistant messages that quote those `System: [...]` lines can therefore be rewritten or emptied even though this fix targets legacy persisted user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-100`
  - body: The runtime detector trims before checking the `System:` status regexes, but the stripping helpers test the raw string. A legacy row with leading whitespace before the status prefix can remain visible even though it is the same runtime-only status shape this PR is trying to hide.
  - confidence: 0.74
