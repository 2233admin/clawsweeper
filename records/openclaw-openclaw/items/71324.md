## Summary

Keep open. The PR targets the right Gateway display-projection surface, but the current PR head still introduces user-visible regressions by dropping reset-tail user text after legacy startup prompts and by rewriting assistant content that quotes legacy `System:` status lines.

## What This Changes

The PR extends Gateway chat-history sanitization to hide quote-wrapped legacy runtime transcript wrappers and status rows, filters now-empty visible history rows, adds session-history regressions, and adds a changelog entry.

## Best Possible Solution

Keep the projection-layer approach, but make it role-aware and tail-preserving: unwrap known legacy runtime quote envelopes, strip legacy status/prompt prefixes only from persisted user/runtime rows, preserve any following user-authored text, leave assistant prose untouched, and cover those boundaries in focused history tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project a PR-head user row beginning with the legacy reset/startup prompt followed by a real reset note, and an assistant row that quotes a legacy `System: [...] Background task ...` or gateway-restart line.

## Solution Assessment

No as written. The projection layer is the right place to solve the legacy display leak, but this implementation is too broad because it can remove legitimate user tails and assistant content.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row that starts with a legacy reset/startup prompt. Legacy rows can contain that runtime header followed by real reset-tail text, and current main explicitly keeps reset notes visible, so this can hide user-authored history instead of only removing runtime text.
  - confidence: 0.87
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before the user-role branch, and callers pass assistant content through the same function. Assistant messages that quote or explain those `System:` lines can therefore be rewritten or blanked even though the PR is scoped to persisted user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` matches against `text.trimStart()`, but the stripping helpers test the raw string. A quoted legacy row with leading whitespace can be unwrapped as runtime text and then fail to strip the status prefix, leaving the legacy status visible.
  - confidence: 0.74
