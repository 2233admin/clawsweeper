## Summary

Keep this PR open. Current main still accepts explicit agent selectors from the OpenAI-compatible chat and OpenResponses HTTP paths without checking that the selected agent exists, so the PR’s intended gateway hardening has not been implemented or superseded.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best fix belongs in the gateway HTTP layer: validate explicit header/model agent selectors against configured agents on both `/v1/chat/completions` and `/v1/responses`, return a 400 `invalid_request_error` for unknown configured agents, preserve implicit default routing plus `openclaw/default` and explicit `main` compatibility, make the `knownAgentIds` contract unambiguous, and add regression tests for header and model selector rejection.
