## Summary

Keep open. Current main still accepts explicit OpenAI-compatible gateway agent selectors without checking whether the agent exists, while this PR is a focused implementation candidate with unresolved review comments rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The fix should live in the gateway HTTP layer: validate explicit header/model agent selectors against configured agents for both /v1/chat/completions and /v1/responses, return a 400 invalid_request_error for unknown explicit selectors, preserve implicit default routing plus openclaw/default and explicit main compatibility, make the knownAgentIds contract unambiguous, and keep regression coverage for header and model selector rejection.
