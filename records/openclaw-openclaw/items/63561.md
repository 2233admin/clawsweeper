## Summary

Keep open. Current main still routes `systemctl --user` through `execFileUtf8("systemctl", args)` without recovered `XDG_RUNTIME_DIR`/`DBUS_SESSION_BUS_ADDRESS`, still classifies `Failed to connect to bus: Permission denied` as user-bus unavailable, and there is an open unmerged PR (#73888) with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review/land #73888, or an equivalent focused fix in `src/daemon/systemd.ts`, that derives missing Linux user-scope `XDG_RUNTIME_DIR` and `DBUS_SESSION_BUS_ADDRESS` for `systemctl --user` spawns while preserving target-user behavior. Add/keep regression coverage for gateway status and gateway install, then close after the fix merges and release provenance is known.
