## Summary

Keep open. Current main still lacks llama.cpp `/props.n_ctx` runtime-context discovery for self-hosted OpenAI-compatible providers, and #74057 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The report asks the llamacpp/self-hosted provider session context display to use the actual loaded runtime context window instead of the model's trained context metadata.

## Best Possible Solution

Review and land #74057, or an equivalent narrow provider-discovery change, so explicit configured caps stay authoritative while llama.cpp-compatible servers can supply the smaller loaded runtime context via `/props.n_ctx`.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main only discovers self-hosted OpenAI-compatible models through `/models`, and session listing reads stored or configured context tokens without a live llama.cpp runtime-context probe.

## Solution Assessment

Yes. Keeping this issue open while maintainers review the existing guarded `/props.n_ctx` PR is the best path; a duplicate repair branch would add churn before that candidate is resolved.
