## Summary

Keep this issue open. Current main still has no llama.cpp `/props.n_ctx` runtime-context discovery in the self-hosted OpenAI-compatible provider path, and #74057 is an open, unmerged implementation candidate with closing syntax for this exact behavior.

## What This Changes

The issue requests that llama.cpp/self-hosted OpenAI-compatible context accounting use the actual runtime context window instead of the model training context window in session usage denominators.

## Best Possible Solution

Review and land or revise #74057, or an equivalent narrow provider-discovery change, so explicit configured caps remain authoritative while llama.cpp-compatible servers can expose the smaller loaded runtime context through `/props.n_ctx`.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main's self-hosted discovery path only reads `/models`, so a llama.cpp server whose OpenAI-compatible model metadata exposes only `n_ctx_train` cannot provide the runtime `n_ctx` value OpenClaw needs for the session denominator.

## Solution Assessment

Yes. Keeping this issue open while maintainers review the existing guarded `/props.n_ctx` implementation candidate is the best path; opening a duplicate repair branch before #74057 is resolved would add churn.
