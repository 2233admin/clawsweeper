## Summary

Keep this issue open. Current main still lacks llama.cpp runtime-context discovery: self-hosted OpenAI-compatible discovery only reads `/models`, does not parse `meta.n_ctx_train`, and never probes `/props.n_ctx` to set `contextTokens`. PR #74057 is open and explicitly links this issue as its closing target, so the best cleanup action is to review, revise, or land that PR rather than close the issue early.

## Best Possible Solution

Keep this issue open and use PR #74057 as the active implementation path. The fix should land or revise a guarded llama.cpp-compatible discovery path that strips a trailing `/v1` before calling `/props`, reads positive `/props.n_ctx` as `contextTokens`, preserves `meta.n_ctx_train` as native `contextWindow` metadata when useful, and keeps explicit configured caps authoritative. Add regression coverage for `/v1/models` reporting `n_ctx_train: 262144` while `/props` reports `n_ctx: 65536`, then close this issue through the merged PR.
