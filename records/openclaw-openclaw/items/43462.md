## Summary

Keep this PR open. Current main still has the case-sensitive default `memory/` directory-prefix behavior the PR is trying to fix, but the implementation is stale because the touched `src/memory/internal.ts` and `src/memory/temporal-decay.ts` surfaces have moved into the memory host SDK and `memory-core` plugin. The later PR commit addressed the prior bot review concern by lowercasing before the dated-memory regex and adding temporal-decay coverage, so the contribution remains a useful implementation candidate after porting.

## Best Possible Solution

Keep this PR open as a useful stale implementation candidate. Port the behavior to current main by lowercasing only normalized default `memory/` directory-prefix checks in `src/memory-host-sdk/host/internal.ts`, `packages/memory-host-sdk/src/host/internal.ts`, and `extensions/memory-core/src/memory/temporal-decay.ts`; use the lowercased value for dated-memory parsing; consider the matching QMD default-memory gate in `extensions/memory-core/src/memory/qmd-manager.ts`; and add focused regression tests for `MEMORY/notes.md`, `Memory/daily.md`, and `MEMORY/YYYY-MM-DD.md` while preserving canonical root `MEMORY.md` behavior.
