## Summary

Keep open. The reported bug still exists on current main, but this PR is not merge-ready because it targets old `src/memory/*` paths and still misses the earlier dated-path parser that must also lowercase before matching.

## What This Changes

The PR lowercases memory directory prefix checks in the old `src/memory/*` helpers and adds tests for case-variant memory paths and temporal decay.

## Best Possible Solution

Port the narrow path-normalization fix to the active memory-host SDK and memory-core modules, lowercase normalized paths before both directory-prefix checks and dated-path parsing, preserve current canonical `MEMORY.md` semantics, and add focused regression tests for case-variant memory directories.

Yes. Current main can be reproduced with a direct string/regex check: case-variant `MEMORY/` paths miss the `memory/` prefix, and `MEMORY/2025-01-01.md` only matches the dated-memory regex after lowercasing.

## Solution Assessment

No. The direction is right, but this branch is not the best final patch because it targets old files and does not lowercase before the first dated-path parse; a small replacement against the active modules is safer.
