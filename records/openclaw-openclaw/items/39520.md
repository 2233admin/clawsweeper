## Summary

Keep this PR open. Current main still lacks the requested Feishu-owned `channels.feishu.localRoots` and account-scoped `localRoots` support for media uploads; it only forwards caller-provided `mediaLocalRoots` arrays to `loadWebMedia`. The PR remains a focused implementation candidate, but the `"any"` mode deliberately expands local file-read access and needs maintainer review plus user-facing docs before merge.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent Feishu plugin-owned patch: add `localRoots: "any" | string[]` to shared top-level and account Feishu config, resolve it from the merged runtime account in `sendMediaFeishu`, honor explicit empty arrays, inject raw `readFile` only for explicit `"any"`, update Feishu docs/config help with the security implications, and coordinate with #51375 for reply-path `mediaLocalRoots` forwarding.
