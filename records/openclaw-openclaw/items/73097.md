## Summary

Keep open. Current main still requires explicit runtime selection for Claude CLI and does not infer Claude CLI transport from `agents.defaults.cliBackends` alone, so the reported PI/subagent routing asymmetry remains a real contract question rather than a cleanup-close candidate.

## What This Changes

The issue requests that PI, subagent, and related model-dispatch paths consult configured Claude CLI backends so canonical Anthropic OAuth traffic does not split between CLI and direct API transports.

## Best Possible Solution

Maintainers should define one supported contract: either keep Claude CLI execution explicit through `agentRuntime.id` and add docs/doctor diagnostics for misleading `cliBackends`-only configs, or add a deliberate opt-in policy that lets eligible Anthropic PI, subagent, and model-run paths route through a configured Claude CLI backend with focused regression coverage.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. A config with canonical `anthropic/*`, `agents.defaults.cliBackends.claude-cli`, and no `agentRuntime.id` leaves `resolveCliRuntimeExecutionProvider` without a CLI runtime and therefore uses embedded PI/direct provider dispatch; the external billing side needs a live Anthropic account to confirm.

## Solution Assessment

Unclear. Inferring CLI transport from `cliBackends` alone is not obviously the safest fix because current docs make `agentRuntime.id` the executor-selection contract, so a maintainer should choose explicit docs/doctor enforcement or a new opt-in routing policy.
