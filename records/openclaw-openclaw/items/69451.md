## Summary

Keep open. Current main has useful mitigations for session growth and some bounded preview/list behavior, but the core report is still valid: `sessions.json` is still read/parsed and rewritten as one monolithic object, and several history paths still materialize full JSONL transcripts before applying limits or projections. Related #71227, #72165, and open PR #51946 show this is an active session-storage performance cluster rather than a stale report.

## Best Possible Solution

Keep this open and route it into the active session-storage performance cluster. The best path is to land or supersede the directory-backed/per-session metadata work from #51946 for `sessions.json`, then separately replace full-transcript readers used by history/send/status paths with cursor-aware or streaming readers while preserving current maintenance, migration, transcript safety, and docs.
