## Risks / Open Questions

- Native SIGILL in sharp/libvips can terminate openclaw-gateway before JavaScript catch/fallback logic can run.
- The exact failure may depend on CPU features, libc, npm global install layout, bundled-vs-system libvips selection, and the specific image payload, so source inspection alone cannot prove a fix.
- Installer docs default SHARP_IGNORE_GLOBAL_LIBVIPS=1 for install/build behavior, but that is not evidence that the runtime WebChat image-flow crash is fixed.
