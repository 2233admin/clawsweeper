## Summary

Keep open. Current main still has no structural per-channel or per-spawn policy for suppressing sub-agent completion announces; top-level requester delivery continues to resolve an external delivery target and tests intentionally preserve both external `agent` delivery and direct fallback sends. The recent maintainer comment also explicitly keeps this open and ties it to #69492 rather than closing it as implemented or stale.

## What This Changes

Add configuration to suppress or reroute sub-agent completion announcements per channel so Telegram, iMessage, and similar chats can keep normal replies without receiving raw sub-agent completion noise.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior but can mark sub-agent completions as suppressed or internal-only for selected channels and/or spawn calls. Apply it before both the primary announce-agent delivery and direct fallback sends, then update config types, generated schema/help, public docs, and focused delivery tests while coordinating with #69492's audience-classification direction.
