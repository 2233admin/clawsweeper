## Summary

Current main still lacks first-class per-channel or per-spawn sub-agent announce suppression. The strict config schema, exported config types, sessions_spawn schema, docs, delivery code, and tests all preserve the current behavior where top-level requester sessions resolve an external delivery target and deliver completion announces or fallback sends back to the channel. The discussion also has active Telegram and iMessage confirmations plus a recent maintainer comment explicitly keeping this open for coordinated follow-up.

## Best Possible Solution

Keep this open and coordinate it with the related suppress/reroute/audience issues. The best implementation is a documented, backwards-compatible completion delivery policy that preserves current channel delivery by default while allowing per-channel and/or per-spawn internal-only/suppressed delivery before both primary announce-agent delivery and fallback sends. It should update config types, zod/generated schema, public sub-agent docs, and focused delivery tests.
