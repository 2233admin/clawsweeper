## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn suppression policy for sub-agent completion announcements, and the recent maintainer discussion explicitly keeps this work tied to the broader #69492 audience-classification design rather than treating it as implemented or stale.

## What This Changes

Add a configurable way to suppress or reroute sub-agent completion announcements for selected channels or spawn calls while keeping normal interactive replies working.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior but can mark sub-agent completions as suppressed or internal-only for selected channels and/or spawn calls. Apply it before both primary announce-agent delivery and direct fallback sends, then update config schema/types, generated config docs, public Sub-agents docs, and focused delivery tests in coordination with #69492.
