## Summary

Keep open. Current main and the latest tagged release still lack a structural per-channel or per-spawn policy for suppressing or rerouting sub-agent completion announces; existing code and docs still preserve external delivery for top-level requester sessions, with only behavioral `NO_REPLY`/`ANNOUNCE_SKIP` suppression available.

## What This Changes

The issue requests a per-channel or per-spawn policy to suppress or reroute sub-agent completion announcements so chat channels like Telegram and iMessage are not cluttered with raw sub-agent completion output.

## Best Possible Solution

Keep this issue open and coordinate it with #69492. The best path is a backward-compatible completion delivery policy that keeps current external delivery as the default while adding documented per-channel and/or per-spawn internal-only or suppressed delivery before both the primary announce-agent path and direct fallback sends; update config types, generated schema/help, docs, and focused delivery tests.
