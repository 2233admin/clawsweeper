## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn policy to suppress or reroute sub-agent completion announcements, and the recent maintainer discussion keeps the item open pending alignment with broader audience-classification work in #69492.

## What This Changes

Add a config or spawn-time control to suppress or reroute sub-agent completion announcements per channel so chat channels such as Telegram and iMessage are not cluttered by raw child output.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior but can mark sub-agent completions as suppressed, internal-only, or rerouted for selected channels and/or spawn calls. Apply it before both direct announce delivery and direct fallback sends, then update config types, generated schema/help, public docs, and focused delivery tests in line with #69492's audience-classification direction.
