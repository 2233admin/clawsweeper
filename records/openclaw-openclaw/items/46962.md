## Summary

Keep open. Current main still lacks blockquote-depth tracking and still emits bare newlines for soft/hard breaks, so the user-visible Markdown fix is not implemented. The PR is useful and focused, but its current diff is stale against main because it replaces the token-aware loose-list paragraph separator that current list-spacing tests depend on.

## What This Changes

The PR adds blockquote-depth tracking in `markdownToIR` plus regression tests so configured blockquote prefixes are re-emitted on soft breaks, hard breaks, multi-paragraph blockquotes, and nested blockquote continuation lines.

## Best Possible Solution

Integrate blockquote-depth prefix restoration into current main's shared Markdown IR renderer while preserving the token-aware list paragraph separator, then validate blockquote spacing, nested-list spacing, and the broader Markdown renderer before merge.
