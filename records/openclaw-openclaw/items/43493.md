## Summary

Keep this PR open. Current main still does not implement the PR's intended interactive custom-provider metadata prompt for contextWindow/contextTokens/maxTokens/reasoning/input/cost. The existing custom-provider setup flow verifies and saves base URL, key, endpoint compatibility, model ID, endpoint ID, alias, and inferred image input, then writes generic custom defaults of 16,000 context tokens and 4,096 max output tokens. The related custom-provider context UX issue #48338 remains open, so this PR is still a plausible implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open as a plausible implementation path. The best maintainer path is to rebase it onto current main, align the prompt with current ModelDefinitionConfig and ModelProviderConfig fields, keep it scoped to the custom-provider setup path, and land it with focused configure/model-metadata tests. If maintainers prefer a different UX, supersede it with an equivalent fix for #48338 rather than closing the remaining user problem.
