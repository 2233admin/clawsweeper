## Summary

Keep this PR open. Current main still lacks Discord `ackReactionTiming` support: Discord queues the initial ack immediately, config/schema/docs do not expose the field, and the same-author Slack and Telegram companion PRs (#45421 and #46258) are still open rather than shipped.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase/port it onto `extensions/discord`, add a channel-scoped `channels.discord.ackReactionTiming` type/schema/help/docs surface, preserve the guarded deferred lifecycle from the later PR commits, and keep focused regression tests for default immediate behavior, run-start deferral, scope gating, and no status reaction before run start.
