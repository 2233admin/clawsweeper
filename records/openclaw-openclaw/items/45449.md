## Summary

Keep this PR open. Current main at 5de284c2e30f915b769fac08dbeab45f57749702 still does not implement optional Discord run-start ack reaction timing: Discord queues the initial ack reaction immediately, and the config type/schema/help/docs surface has no ackReactionTiming key.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate. The best path is to rebase the work onto current extensions/discord, add a channel-scoped channels.discord.ackReactionTiming type/schema/help/docs surface, preserve the guarded deferred lifecycle from the later PR commits, and keep focused regression tests for default immediate behavior, run-start deferral, scope gating, and no reaction before run start.
