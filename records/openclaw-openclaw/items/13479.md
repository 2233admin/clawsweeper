## Summary

Keep open. Current main still has the data contract and edit/search plumbing for cron job descriptions, and it renders the prompt/task payload in the Control UI job card, but the visible cron card/detail path still does not render the saved description as the requested human-readable summary.

## What This Changes

Show a cron job's prompt/task and human-readable summary in the Web UI cron dashboard when the job is selected or clicked.

## Best Possible Solution

Add a visible Summary or Description field to the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, using the existing CronJob.description value while leaving storage, RPC, payload rendering, and run-history selection semantics intact.

## Reproduction Assessment

Yes. A source-level reproduction is clear: render a CronJob with a non-empty description; current renderJob displays the card and calls renderJobPayload, but neither path reads or renders job.description.

## Solution Assessment

Yes. The narrowest maintainable fix is a Control UI presentation change with focused view tests; no core cron schema, storage, or Gateway RPC change appears necessary.
