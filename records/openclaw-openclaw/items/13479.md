## Summary

Keep open. Current main still stores, edits, searches, and renders cron prompt/task payloads, but the Control UI cron dashboard job card/detail path does not visibly render the saved job description as the requested human-readable summary.

## What This Changes

The issue requests that the Web UI cron dashboard show a cron job’s prompt/task and saved human-readable summary when inspecting a job, avoiding manual config-file inspection.

## Best Possible Solution

Add a visible Summary or Description field to the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, using the existing CronJob.description value while leaving storage, RPC, payload rendering, and run-history selection semantics intact.
