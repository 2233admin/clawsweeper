## Summary

Current main still matches the reported Slack bug. Slack exposes and accepts `dmHistoryLimit`, but the Slack monitor path derives and carries only `historyLimit`; prompt assembly, `InboundHistory`, pending-history recording, and clearing remain gated on `isRoomish`, which excludes 1:1 Slack DMs.

## Best Possible Solution

Keep this issue open. The best path is a focused Slack monitor fix that derives `account.config.dmHistoryLimit`, carries it through `SlackMonitorContext` and prepared-message state, computes an effective room-vs-DM history limit, uses that limit for pending-history recording, prompt `combinedBody`, structured `InboundHistory`, and dispatch clearing, and adds a regression test for a non-threaded 1:1 Slack DM with `dmHistoryLimit > 0`. Broader Slack `conversations.history` hydration and cross-channel parity can remain separate follow-up work, likely under #64427 or a new maintainer-scoped issue.
