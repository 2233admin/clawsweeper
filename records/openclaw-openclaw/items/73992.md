## What This Changes

The issue reports intermittent 10s `openclaw nodes list` loopback gateway timeouts on OpenClaw 2026.4.25, including windows where a follow-up `openclaw gateway probe` also times out.

## Best Possible Solution

Keep the issue open and investigate on current main or the next tagged build with Gateway logs, `/readyz` event-loop diagnostics, timings for `node.pair.list` and `node.list`, and an immediate `gateway probe --json` after any `nodes list` timeout. The end state should either isolate and fix the Gateway responsiveness/pairing-state stall with focused regression coverage, or link a narrower canonical issue if a different confirmed root cause owns the remaining work.
