## Summary

Keep #70616 open. Current main still has the reported memory-core dreaming config gap: the strict plugin schema does not allow `dreaming.execution` or phase-level `execution`, while runtime code already has partial execution-model parsing and the managed dreaming cron/narrative paths still do not carry model fallback overrides end to end.

## Best Possible Solution

Keep the issue open and fix it in the memory-core plugin/core config contract. Decide and document the public `dreaming.execution` shape, update `extensions/memory-core/openclaw.plugin.json`, generated config/docs/tests, and wire resolved execution model settings into the managed dreaming cron payload. If fallback chains are part of the contract, map them to cron `payload.fallbacks` and add a typed subagent fallback override or avoid using subagent paths that cannot honor the configured chain.
