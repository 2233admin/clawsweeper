## Summary

Current main still has the exact docs drift this PR targets: the config type JSDoc, FIELD_HELP, and generated base schema advertise `stallSoftMs`/`stallHardMs` as `25000`/`60000`, while the runtime default source of truth is `10_000`/`30_000`. The PR remains useful, but it should also refresh the generated schema before merge.

## Best Possible Solution

Keep this PR open or apply an equivalent docs-only fix that updates `src/config/types.messages.ts` and `src/config/schema.help.ts` to `10000`/`30000`, then regenerate/check `src/config/schema.base.generated.ts` with the config schema generator before merge. No runtime behavior change is needed.
