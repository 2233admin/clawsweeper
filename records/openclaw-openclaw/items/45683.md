## Summary

Keep this PR open. Current main still documents stale `stallSoftMs`/`stallHardMs` defaults as `25000`/`60000` in the config type JSDoc, `FIELD_HELP`, and the checked-in generated base schema, while the runtime `DEFAULT_TIMING` source of truth is `10_000`/`30_000`. The PR's intended docs-only correction remains valid, but it should be refreshed to include generated-schema output before merge.

## Best Possible Solution

Keep this PR open or apply an equivalent docs-only fix: update `src/config/types.messages.ts` and `src/config/schema.help.ts` to `10000`/`30000`, refresh `src/config/schema.base.generated.ts` through the config schema generator, and run the narrow config schema/docs checks for that surface. No runtime behavior change is needed.
