## Summary

Keep open. Current main still uses exact-only `skills info` lookup, while this PR is the focused implementation for case/separator-tolerant lookup. The PR head is relevant but still has the unresolved case-insensitive ambiguity called out in review, so it should be amended rather than closed.

## Best Possible Solution

Keep this PR open and amend it narrowly: exact matches should still win, but both case-insensitive and separator-normalized fallback paths should collect all matches and resolve only when exactly one candidate remains, otherwise returning an explicit ambiguity/not-found result. Preserve the trimmed/sanitized not-found output, keep the focused regression tests, add an ambiguous case-insensitive regression, then run `pnpm test src/cli/skills-cli.test.ts` and the normal changed gate before landing.
