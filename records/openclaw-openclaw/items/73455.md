## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Default the AI gates to explicit opt-in:** `extensions/matrix/src/matrix/monitor/index.ts:246-255`
  - body: `participationEnabled` and `freshnessEnabled` resolve to true unless explicitly set false, with AI-backed defaults and zero thresholds. Existing Matrix rooms can therefore start sending room context to the configured model without adding the new config blocks; make these controls default-off or add an explicit migration/approval path.
  - confidence: 0.88
- **[P2] Let revise and auto pass the holdback gate:** `extensions/matrix/src/matrix/monitor/handler.ts:2360-2370`
  - body: When a holdback sees newer room activity, this branch suppresses every mode except `send-as-is`. That makes `revise` and `auto` skip the draft/final freshness resolver entirely, contradicting their documented behavior; only `freshnessMode === "suppress"` should return here.
  - confidence: 0.87
- **[P2] Remove the deterministic ai-first fallback:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:378-382`
  - body: `resolveParticipationDecision` still parses raw `text` deterministically when `currentDirective` is absent. That leaves a silent fallback path despite the PR contract that `ai-first` does not fall back; remove the raw text parameter and require callers to pass a resolved directive.
  - confidence: 0.84
- **[P2] Use plugin SDK runtime imports:** `extensions/matrix/src/matrix/monitor/handler.ts:11-17`
  - body: The Matrix plugin production handler imports agents and auto-reply internals via relative `src/**` paths. Bundled plugins should use exported `openclaw/plugin-sdk/*` seams, and current main already exposes simple-completion and reply runtime subpaths for this path.
  - confidence: 0.9
- **[P2] Fix the AI participation schema prompt:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:314`
  - body: The schema string in the system prompt has malformed optional array field names for `includeAgentIds?` and `excludeAgentIds?`, which can make the requested JSON shape ambiguous to the model. Use a syntactically clear schema string for those optional arrays.
  - confidence: 0.78
