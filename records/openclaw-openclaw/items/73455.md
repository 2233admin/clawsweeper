## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor revise and auto during holdback:** `extensions/matrix/src/matrix/monitor/handler.ts:2360-2371`
  - body: When `draftHoldbackMs` sees new room activity, this branch suppresses every freshness mode except `send-as-is`. That means `revise` never gets a draft to rerun and `auto` never consults the configured final-action logic. Restrict this pre-draft suppression to `freshnessMode === "suppress"` and let `revise`/`auto` continue into the freshness-aware delivery path.
  - confidence: 0.91
- **[P2] Remove the deterministic fallback from AI-first decisions:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:384-388`
  - body: `resolveParticipationDecision` parses raw text deterministically when `currentDirective` is absent, so a future AI-first call can silently bypass AI despite the documented no-fallback contract. Drop the `text` parameter from this decision helper and require callers to pass the already resolved directive.
  - confidence: 0.82
- **[P2] Fix the malformed AI schema prompt:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:314`
  - body: The system prompt schema leaves `includeAgentIds?` and `excludeAgentIds?` without closing quotes, which garbles the JSON shape shown to the model and can degrade participation parsing. Use a valid compact schema string for those optional arrays.
  - confidence: 0.86
- **[P2] Keep Matrix production code behind SDK seams:** `extensions/matrix/src/matrix/monitor/handler.ts:10-16`
  - body: The Matrix plugin now deep-imports `src/agents` and `src/auto-reply` from production extension code. That bypasses the plugin boundary and couples this bundled plugin to private core file layout; add or consume documented `openclaw/plugin-sdk/*` helpers instead.
  - confidence: 0.88
