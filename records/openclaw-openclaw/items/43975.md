## Summary

Keep this PR open. Current main still does not implement the intended Windows Scheduled Task UTF-8 bootstrap, and the PR remains a focused candidate for the linked Chinese-username gateway startup failure in #43943.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent change in `src/daemon/schtasks.ts`: emit a deterministic UTF-8 code-page bootstrap immediately after `@echo off`, teach `readScheduledTaskCommand` to ignore that exact bootstrap during read-back, and add Chinese-path regression coverage while preserving current `sourcePath` and environment source metadata behavior.
