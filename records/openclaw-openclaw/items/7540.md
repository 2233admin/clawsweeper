## Summary

Current main still lacks WhatsApp/Baileys call-event support. The bundled WhatsApp plugin subscribes to message and connection events only, exposes message-only plugin hooks/config, and the pinned Baileys version does provide a `call` event that OpenClaw does not consume. Recent maintainer comments also explicitly kept this as valid WhatsApp-plugin-owned work, not a PSTN voice-call or ClawHub item.

## Best Possible Solution

Keep this issue open and implement it inside `extensions/whatsapp`: subscribe to Baileys `call`, normalize/sanitize `WACallEvent` payloads, apply the same account, DM/group, self-chat, allowlist, and privacy boundaries as inbound messages, add opt-in config if maintainers want it, expose the resulting activity through a documented session-event/plugin-hook/tool surface, and cover the behavior with focused tests and docs.
