## Summary

Keep open. Current main now has a later metadata-only model-call hook surface (`model_call_started` / `model_call_ended`) with embedded-run wiring, tests, docs, and changelog coverage, but it deliberately does not expose the per-call request/response payloads requested by #58360 and linked issue #57653. Closing this PR as implemented would skip a real maintainer product/security decision about whether raw provider-call payload observability should exist, or whether the metadata-only API is the intended final scope.

## Best Possible Solution

Keep #58360 open for maintainer judgment. The likely path is either to explicitly supersede it with the shipped `model_call_started` / `model_call_ended` metadata-only API and update/close #57653, or to design a separate trusted raw-payload hook surface with policy gates, docs, and tests if maintainers want plugins to observe provider-facing request and response bodies.
