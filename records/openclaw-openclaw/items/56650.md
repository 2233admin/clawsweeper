## Summary

Keep open. Current main partially addresses headless Copilot setup through non-interactive onboarding, but it does not implement the PR’s two advertised fixes: `models auth paste-token --token`/stdin/env support and PAT-direct GitHub Copilot runtime auth. The PR also has unresolved review concerns around Copilot editor-header propagation and stdin test coverage.

## Best Possible Solution

Keep this PR open and ask for a refresh against the current plugin-owned GitHub Copilot implementation. The best path is to split or tightly scope the work: add generic non-interactive `models auth paste-token` support with stdin/env tests, and separately implement PAT support in the GitHub Copilot plugin using the current runtime request override/header contract, with tests proving PATs skip `/v2/token`, use the intended Copilot endpoint, and attach required headers.
