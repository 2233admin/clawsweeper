## Summary

Keep this issue open. Current main has partially addressed the original Unix-only `xargs` failure by replacing the docs format pipeline with a Node script, but the checker still passes the full docs file list to one `oxfmt` spawn and has no landed native Windows/PowerShell proof. Open PRs #44824 and #49913 also still explicitly reference this issue with closing syntax, so the issue should remain open until a canonical implementation lands or those candidates are resolved.

## Best Possible Solution

Keep this issue open and land one canonical cross-platform docs formatter update. The best fix is to preserve the current Mintlify-safe repair behavior in `scripts/format-docs.mjs`, add batching or another long-argument-safe `oxfmt` invocation strategy for Windows, and add a small regression test or native PowerShell smoke proof. After that merges, close this issue as implemented and close #44824/#49913 or equivalent duplicate PRs as superseded.
