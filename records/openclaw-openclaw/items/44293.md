## Summary

Keep open. Current main no longer uses the literal `xargs` pipeline for docs formatting, but the replacement still sends all tracked docs temp paths through one unchunked `spawnSync` call. With 463 tracked docs files, the native Windows command-line-length concern remains plausible, the latest release still ships the old `xargs` scripts, and the related open PR #48887 remains the right place for maintainer reconciliation.

## What This Changes

Make `pnpm check:docs` work in native Windows PowerShell without relying on Unix-only `xargs` and without requiring Git Bash.

## Best Possible Solution

Keep this issue open while maintainers reconcile the docs formatter path. The best outcome is to add chunking and native PowerShell proof to the current `scripts/format-docs.mjs` implementation, or explicitly close #48887 as superseded with equivalent proof; after that, close this issue as implemented on main or in the first release that ships the fix. Keep #44291 as the broader native PowerShell smoke-coverage follow-up.
