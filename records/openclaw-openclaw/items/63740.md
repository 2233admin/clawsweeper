## Summary

Keep open. Current main has package/install hardening around stale and missing `dist` files, but the installed package integrity checks are still path-based and would not detect a byte-modified expected `dist/run-main-*.js` chunk. Startup still dynamically imports the CLI chunk and reports the same compilation failure shape, so the reported installed-artifact corruption is not fixed or ruled out.

## What This Changes

Investigate and prevent or recover from corruption of installed OpenClaw `dist/run-main-*.js` files that can make the CLI fail at startup with `SyntaxError: Unexpected number`.

## Best Possible Solution

Keep this as a packaging/runtime integrity investigation. Maintainers should audit update, postinstall, restart, logging, and diagnostic paths for writes under the installed package root, then either prove OpenClaw cannot mutate installed core chunks at runtime or add hash-based installed-dist verification with clear startup/doctor repair guidance for corrupted expected chunks.
