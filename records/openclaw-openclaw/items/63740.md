## Summary

Keep open. Current main still verifies installed package `dist` state by expected paths/import closure, not by byte content, so a modified expected `dist/run-main-*.js` chunk would still be imported and fail with the reported startup syntax error shape instead of being caught or repaired first.

## What This Changes

Investigate and prevent or recover from corruption of installed OpenClaw `dist/run-main-*.js` chunks that can make the CLI fail at startup with `SyntaxError: Unexpected number`.

## Best Possible Solution

Keep this as a packaging/runtime integrity investigation: audit update, postinstall, restart, logging, and diagnostic paths for writes under the installed package root, then either prove OpenClaw cannot mutate installed core chunks at runtime or add byte-level installed-dist verification with clear startup and doctor recovery guidance.
