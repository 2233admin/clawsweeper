## Summary

Keep this PR open. Current main still derives direct loose-list paragraph depth from list nesting count alone, so a loose list inside a blockquote remains outside the recognized direct paragraph level; the PR is a narrow follow-up to the merged #74474 loose-list fix and appears to address the remaining blockquote container-depth case without introducing a review finding.

## What This Changes

The PR records each Markdown list's opening token level, uses it to recognize direct loose-list paragraph closes, and adds regression coverage for paragraph breaks inside a loose blockquoted list item.

## Best Possible Solution

Land the narrow Markdown IR fix after standard maintainer review and changed-gate validation, keeping loose-list paragraph detection based on the actual list-opening token level with regression coverage for blockquoted lists.
