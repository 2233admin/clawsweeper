## Summary

Keep this PR open, but it still needs changes before merge. The proposed direction addresses a real persistence/data-loss problem, yet the current patch defaults omitted paths to a hard-coded workspace location that can differ from OpenClaw's active workspace resolution in profiled, multi-agent, configured-workspace, or OPENCLAW_HOME setups.

## What This Changes

The PR makes `skills/skill-creator/scripts/init_skill.py` accept an omitted `--path`, adds a default skills-directory resolver, and updates bundled skill-creator instructions to avoid global or relative output paths.

## Best Possible Solution

Land a narrow fix that keeps explicit `--path` backward-compatible, defaults omitted paths to the same active workspace `skills/` directory used by runtime and `openclaw skills`, removes relative output examples from bundled instructions and the script docstring, and adds focused regression tests for the resolver behavior.
