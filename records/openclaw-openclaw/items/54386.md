## Summary

Keep this PR open. Current main still has the reported skill-creator behavior: `init_skill.py` requires `--path`, and the bundled skill instructions still show relative `skills/public` examples. The proposed fix direction matches OpenClaw's workspace-skill model, but the PR needs to align its default path logic with the canonical workspace/home/profile rules and add targeted Python coverage before it is ready to merge.

## Best Possible Solution

Keep the PR open and ask for a small update: make `--path` optional, default generated skills to the canonical active workspace `skills/` directory, align Python path resolution with `src/agents/workspace-default.ts`/`src/infra/home-dir.ts` semantics, update the skill instructions to avoid relative-path examples, and add focused Python tests for omitted `--path`, explicit `--path`, `OPENCLAW_HOME`, non-default `OPENCLAW_PROFILE`, and tilde expansion for any supported workspace env override.
