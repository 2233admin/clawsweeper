## Summary

Keep open. Current main has useful partial fixes for commentary/final-answer separation and a generic `message_sending` plugin hook, but it does not implement the requested runtime-enforced, execution-state-aware pre-send policy for ack classification, repeated-ack suppression, progress debt, or durable execution-state gating. Ryce's linked keepmyclaw snapshot context is relevant because the requested policy depends on reliable execution state across restarts.

## Best Possible Solution

Keep this open as a design/architecture tracking issue. The best path is to carve a core-owned outbound policy design that builds on current phase-aware extraction and delivery hooks, defines typed message classification, binds allowed acknowledgements to durable execution/task state, records repeat-ack/progress-debt state per session, and adds focused tests across the shared outbound path plus at least one channel integration surface. Ryce's keepmyclaw snapshot pattern should be considered as prior art for making the backing execution state durable enough for the policy to trust.
