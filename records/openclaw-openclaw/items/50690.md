## Summary

Keep open. Current main has partial mitigations, but it still lacks the requested execution-state-backed fake-progress policy across user-facing outbound delivery paths; related open issues track prerequisite outbound-hook architecture but do not supersede this narrower policy request.

## What This Changes

The issue requests a runtime pre-send policy layer that classifies outbound assistant messages and blocks low-value acknowledgements unless they are backed by real execution/progress state.

## Best Possible Solution

Choose one first-class outbound policy boundary and durable execution-state model, then implement core fake-progress enforcement with typed ack/progress/blocker/completion classification, repeat suppression, progress-debt faulting, structured suppression metrics, and shared outbound plus channel/agent coverage.
