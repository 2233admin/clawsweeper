## Summary

Keep open. Current main has useful partial pieces, including outbound rewrite/cancel hooks, phase-aware commentary suppression, and isolated-cron stale-ack retry, but it still lacks the requested runtime policy that classifies outbound assistant messages and gates acknowledgements on durable execution/progress state across user-facing channels.

## What This Changes

The issue requests a runtime-enforced pre-send policy layer that blocks fake-progress acknowledgements, commentary leakage, repeated ack spam, and post-start silence unless backed by real execution/progress state.

## Best Possible Solution

Define a first-class outbound policy boundary, then implement this fake-progress policy on top with durable execution state, typed ack/progress/blocker/completion classification, per-session repeat and progress-debt tracking, structured suppression metrics, and shared outbound plus channel integration coverage.
