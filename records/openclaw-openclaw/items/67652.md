## Summary

Keep this PR open. Current main still does not implement the central behavior: the public reply-payload SDK surface normalizes and resolves only `mediaUrls` and `mediaUrl`, while the proposed loose `media` alias support remains outside main.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. If OpenClaw wants this compatibility behavior, land a refreshed patch in `src/plugin-sdk/reply-payload.ts` that consistently accepts `media` in normalization, media resolution, content detection, sendable-part extraction, and chunked media sending, with regression tests for string and array aliases, blank alias entries, empty `mediaUrl` fallback, and delivery wrapping through `createNormalizedOutboundDeliverer`.
