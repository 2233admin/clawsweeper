## Summary

Keep this issue open. Current main still documents and specifies bundled OpenProse `model:` values as only `sonnet`, `opus`, or `haiku`, so the reporter's `model: openai/gpt-5.2` example remains outside the OpenProse grammar. The underlying OpenClaw `sessions_spawn` path can already preserve explicit provider/model refs, so the remaining fix belongs in the bundled OpenProse skill/spec/docs surface rather than core model selection.

## Best Possible Solution

Fix this in the bundled OpenProse plugin surface: allow canonical `provider/model` refs wherever OpenProse accepts `model:`, keep the existing aliases working, update `extensions/open-prose/skills/prose/prose.md`, `compiler.md`, examples, and `docs/prose.md` consistently, and add focused coverage or a runnable example showing distinct OpenProse agents can pass exact model overrides through to `sessions_spawn.model` without falling back to the default subagent model.
