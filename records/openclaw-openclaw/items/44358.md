## Summary

Keep this issue open. Current main still ships OpenProse as a bundled plugin/skill whose OpenProse grammar and compiler reference only allow the `sonnet`, `opus`, and `haiku` model aliases, so the reporter's `model: openai/gpt-5.2` example remains outside the supported OpenProse input. The generic `sessions_spawn` path can preserve explicit provider/model refs, which points the remaining work at the bundled OpenProse spec/docs/examples or its execution guidance rather than at core subagent model selection.

## Best Possible Solution

Keep this issue open and fix the bundled OpenProse plugin surface. Maintainers should decide whether OpenProse accepts canonical `provider/model` refs directly or intentionally requires documented aliases, then update `extensions/open-prose/skills/prose/prose.md`, `extensions/open-prose/skills/prose/compiler.md`, examples, and `docs/prose.md` consistently. Add focused coverage or a runnable example showing multiple OpenProse agents spawning through `sessions_spawn.model` with distinct exact model overrides and no silent fallback to the default subagent model.
