## Summary

Current main still leaves the reporter's OpenProse `model: openai/gpt-5.2` example outside the bundled OpenProse grammar/compiler docs, while the core `sessions_spawn` path already accepts and forwards arbitrary model strings. This remains a valid, narrow OpenProse plugin/spec/docs bug rather than an implemented-on-main close.

## What This Changes

OpenProse should consistently honor per-agent and per-session `model:` values, including canonical `provider/model` refs such as `openai/gpt-5.2`, instead of only accepting the `sonnet`, `opus`, and `haiku` aliases.

## Best Possible Solution

Keep this issue open and update the bundled OpenProse plugin/spec/docs so `model:` accepts canonical OpenClaw `provider/model` refs wherever aliases are accepted, while preserving the existing aliases. Add a small regression artifact or example showing a provider/model value flowing to `sessions_spawn.model`; avoid core model-selection changes unless the repair uncovers a concrete core API gap.
