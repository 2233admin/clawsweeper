## Summary

Current main still enforces and documents a 0-5 cap for `session.agentToAgent.maxPingPongTurns`, so this PR is not obsolete and is not implemented on main. It remains a focused implementation candidate for the closed related request, but it needs rebase/current-surface updates, generated/help/type updates, boundary tests, and maintainer review before merge.

## Best Possible Solution
