## Summary

Current main does not implement this PR's central change. `sessions_send` still has no `deliveryMode` or `announce` parameter, its result metadata is always announce-oriented except for the owned ACP-child skip path, and the A2A flow still always proceeds to the announce/send path after the private inter-session run. The PR is not obsolete, but it needs maintainer review and likely a rebase/split because its diff also contains unrelated secrets-test stabilization against a path absent from current main.

## Best Possible Solution

Keep this PR open. The best path is for maintainers to review the sessions feature as a real, still-missing capability, ask the contributor to rebase onto current main, and split or justify the unrelated secrets-test change before landing. The implementation should preserve backward compatibility with default announce delivery while adding an explicit private mode, updated prompt guidance, docs, and focused regression tests.
