## Summary

Keep open. Current main still has the explicit `context1m` gate, so the PR is not obsolete; however, the PR branch is not a complete fix because it makes planning/status return 1M without also making the Anthropic request path send the required 1M beta header for that no-flag path.

## What This Changes

The PR changes `resolveContextTokensForModel()` and its tests so Anthropic Opus/Sonnet 4 models resolve to the 1M context window by default, with `context1m: false` as an explicit opt-out.

## Best Possible Solution

Rework the fix so the resolver, Anthropic beta-header injection, OAuth/legacy-token fallback, session/status limits, compaction thresholds, tests, and docs all share the same source of truth for whether the active credential/request path actually has 1M context.
