## Summary

Current main still does not implement per-binding `tools.allow`/`tools.deny` for route or ACP bindings. The strict binding schemas and exported types have no `tools` field, route resolution drops all matched-binding data except agent/session routing, ACP configured-binding records do not preserve a tool policy, and the shared tool-policy pipeline has no binding-scoped step. The latest tagged release snapshot shows the same gap, so this remains valid work rather than a cleanup close.

## Best Possible Solution

Keep this issue open and implement an additive core binding policy surface: add `bindings[].tools.allow`/`deny` to route and ACP schemas/types/help/docs, carry the matched route binding policy through `ResolvedAgentRoute`, preserve ACP binding policy through configured-binding materialization, and apply the policy as an additional deny-wins/narrowing step in the shared tool-policy pipeline before sandbox/subagent restrictions. Cover both validation and runtime behavior for a Telegram group route binding that denies `sessions_spawn` while DMs remain unaffected, plus ACP configured-binding materialization/enforcement.
