## What This Changes

## Best Possible Solution

Land this focused issue-template fix, or make the second field an explicit GitHub `upload` element if maintainers want a file-only control, while preserving shell rendering for pasted logs.

Yes. Current main reproduces the problematic configuration directly in `.github/ISSUE_TEMPLATE/bug_report.yml`: the text asks for screenshot/recording evidence in a textarea that also has `render: shell`, and GitHub's form schema documents that `render` prevents attachment expansion.

## Solution Assessment

Yes. Splitting logs from screenshot/evidence collection is the narrowest maintainable fix for the reported UX problem; using `type: upload` for the second field is a reasonable maintainer alternative, not a required correction.
