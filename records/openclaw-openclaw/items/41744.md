## Summary

Keep this issue open. Current main still lacks the reported fallback from trusted `read` image tool results to original tool-call args when `details.path` is absent, and open PRs #46268 and #50359 already target the same fix path.

## Best Possible Solution

Keep this issue open and review or consolidate PR #46268 and PR #50359. The fix should thread `tool_execution_start` args into embedded tool-result media extraction, fall back to `path` / `file_path` / `filePath` only when trusted image content exists and no explicit media path is present, then verify pending media is merged into final replies and delivered through Feishu `sendMediaFeishu`.
