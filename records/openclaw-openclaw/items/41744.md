## Summary

Keep open. Current main still lacks the reported fallback from trusted read-image tool output to the original tool-call args when `details.path` is absent, and open PRs #46268 and #50359 are implementation candidates for the same bug shape.

## Best Possible Solution

Keep this issue open and review or consolidate PR #46268 and #50359. The fix should pass stored `tool_execution_start` args into embedded media extraction, fall back to `path` / `file_path` / `filePath` only when trusted image content exists and no explicit media path is present, then preserve the resolved media through the existing pending-tool-media merge and Feishu send path with focused regression tests.
