## Summary

Keep this issue open. Current main has shipped the env/file/exec SecretRef foundation and documents 1Password through the exec provider, but it still does not implement native keyring/Keychain, native 1Password, or a plugin-owned SecretRef provider/source seam. The same author has an open paired PR, #72548, that is the active implementation candidate for the remaining architecture and native keyring work, so closing this issue before that PR is reviewed or merged would race the active contribution.

## Best Possible Solution

Review PR #72548 as the active same-author implementation candidate. If it lands, close this issue as implemented for the env/keyring/plugin-seam portion and open or link a narrower follow-up for native 1Password if maintainers still want it bundled. If the PR is declined, narrow this issue to the accepted native keyring/Keychain, native 1Password, or plugin SecretRef provider API work, leaving broader cloud-provider tracking to #13610 or dedicated provider issues.
