## Summary

Keep this issue open. Current main still only supports built-in SecretRef sources env/file/exec, documents 1Password through the generic exec provider, and has no native keyring/Keychain, native 1Password, or plugin-owned SecretRef provider-source seam. Open PR #72548 by the same author is the active implementation candidate for the remaining seam and keyring work, so closing now would race that review.

## Best Possible Solution

Review #72548 as the active implementation candidate. If it lands, close this issue as implemented for the pluggable SecretRef provider-source seam and native keyring/Keychain path, then open or link a focused native 1Password follow-up only if maintainers want that bundled. If maintainers decline that PR, narrow this issue to the accepted keyring/Keychain, native 1Password, or plugin SecretRef provider API work, with broad cloud-provider tracking left to #13610 or dedicated provider issues.
