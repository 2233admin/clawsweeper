## Summary

Keep this issue open. Current main ships the SecretRef foundation and env/file/exec providers, but it still does not ship native keyring/Keychain, native 1Password, or a plugin-owned SecretRef provider source seam. The issue is also paired with same-author open PR #72548 in the provided GitHub context, so closing this issue now would race the active implementation review.

## Best Possible Solution

Review PR #72548 as the active implementation candidate. If it lands, close this issue as implemented for the env/keyring/plugin-seam portion and open or link a focused follow-up for native 1Password if maintainers still want it bundled. If maintainers decline that PR, narrow this issue to the accepted native keyring/Keychain, native 1Password, or plugin SecretRef provider API work, with broader cloud-provider tracking left to #13610 or dedicated provider issues.
