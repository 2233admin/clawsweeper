## Summary

Keep this PR open. Current origin/main still emits browser screenshot and AI snapshot `MEDIA:` lines through `shortenHomePath(...)`, while the shared media parser still rejects `~`-prefixed candidates. The PR is a focused implementation candidate for the linked bug: it changes the three implicated browser CLI emissions to use absolute paths, leaves the human-readable shortened `--out` log alone, and adds regression tests for the machine-readable `MEDIA:` output. Security review found no unexplained supply-chain or CI-sensitive changes in the provided diff.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the linked browser media delivery bug. The best path is maintainer review plus normal CI, then land the producer-side fix so browser `MEDIA:` lines use absolute paths while the shared parser keeps rejecting `~` and traversal-style media sources.
