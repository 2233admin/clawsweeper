## Summary

Keep this PR open. Current main and the latest release still emit browser CLI MEDIA lines through shortenHomePath(...), while the shared parser still rejects home-shortened media paths; this PR remains the focused producer-side fix for #73796.

## Best Possible Solution

Keep this PR open for maintainer review and CI, then land the producer-side browser CLI change if checks pass. The best fix is to emit absolute paths only on machine-readable browser MEDIA lines while keeping human-readable shortening elsewhere and preserving the shared parser's rejection of home-dir shorthand and traversal-style media sources. After merge, #73796 can close as implemented.
