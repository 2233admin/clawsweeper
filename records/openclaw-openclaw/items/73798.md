## Summary

Keep this PR open. Current main still emits browser screenshot and AI snapshot `MEDIA:` lines through `shortenHomePath(...)`, while the shared media parser still rejects `~`-prefixed media paths. The PR remains a focused implementation candidate for the linked open browser media delivery bug, and its diff is scoped to the producer, tests, and changelog without changing the parser security contract.

## Best Possible Solution

Keep this PR open for maintainer review and CI, then land the producer-side browser CLI fix if checks pass. The best product direction is to emit absolute paths in machine-readable browser `MEDIA:` lines while preserving the shared parser's rejection of `~` and traversal-style media sources; after merge, the linked bug can close as implemented.
