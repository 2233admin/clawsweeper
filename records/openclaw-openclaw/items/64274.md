## Summary

Current main still appears to contain the reported agent-specific auth resolution bug. Runtime auth snapshots are still keyed by resolved auth-store path, omitted agentDir still resolves through OPENCLAW_AGENT_DIR, and the MiniMax portal provider still discovers OAuth availability through that affected auth-store lookup. The related PR #64320 was closed unmerged, so there is no shipped implementation to close this against.

## Best Possible Solution

Keep this issue open. Revive PR #64320 or apply an equivalent focused fix that keys runtime auth snapshots by stable runtime identity rather than resolved auth-store path, updates get/has/set/replace/save/load paths consistently, and adds a regression test where OPENCLAW_AGENT_DIR points at the selected agent while main and that agent have distinct minimax-portal profiles.

## Risks / Open Questions

- Leaving this open is appropriate because the current code can still conflate the selected agent and main runtime auth identities when OPENCLAW_AGENT_DIR points at the selected agent.
- A fix should preserve intentional main-agent auth fallback/inheritance for sub-agents while preventing runtime snapshot identity collisions.
- MiniMax portal OAuth is an observable affected surface because catalog discovery still treats minimax-portal profile presence as auth availability.
