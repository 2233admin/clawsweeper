## Summary

Keep this PR open. Current main still lacks native Azure OpenAI request shaping for memory-lancedb embeddings, and the PR remains a relevant implementation candidate, but the submitted patch is not complete enough to close or merge as-is.

## Best Possible Solution

Keep the PR open and ask for a revised implementation. The best path is Azure-aware memory-lancedb embedding transport that reuses parsed-host Azure detection, supports both Azure OpenAI and Azure AI Foundry hosts, exposes an embedding apiVersion setting through runtime config, manifest schema, UI hints, and docs, and adds focused tests for Azure and non-Azure request behavior.
