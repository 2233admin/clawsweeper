## Summary

Keep this PR open. Current main still does not provide native Azure OpenAI support for `memory-lancedb` embeddings: both the direct OpenAI-compatible path and the provider-backed OpenAI embedding path lack Azure `api-key` auth plus `api-version` query handling. The submitted PR is still a relevant implementation candidate, but it needs revision before merge because it hardcodes an API version, detects only one Azure host family, and adds no tests or config/schema/docs coverage.

## Best Possible Solution

Keep the PR open and ask for a revised implementation. The best fix should add Azure-aware embedding transport for `memory-lancedb`, use parsed-host suffix detection consistent with existing Azure helpers, expose an `apiVersion` setting through runtime config, manifest schema, UI hints, and docs, and add focused tests for Azure and non-Azure embedding request behavior.
