## Summary

Keep this PR open. Current main still lacks native Azure request shaping for memory-lancedb embeddings, but the proposed patch is not merge-ready because it only adds an `api-key` header and hard-coded query default while leaving bearer auth, root `/embeddings` routing, incomplete host detection, and no configurable API-version surface.

## What This Changes

The PR modifies the memory-lancedb direct OpenAI-compatible embedding client to detect `.openai.azure.com` base URLs and add Azure `api-key` and `api-version=2024-02-01` defaults.

## Best Possible Solution

Add Azure-aware memory embedding request shaping that parses Azure hostnames, sends `api-key` without bearer auth, normalizes to deployment-scoped embedding URLs, exposes an API-version setting through runtime config, manifest, UI hints, and docs, and covers both direct and provider-backed memory embedding paths where maintainers want parity.

## Reproduction Assessment

Yes. A code-level reproduction is clear: configure memory-lancedb with `embedding.provider: "openai"`, a plugin `apiKey`, and an Azure resource `baseUrl`; current main uses the direct client and posts to `/embeddings`, while the PR would still not normalize the Azure deployment URL or expose API-version configuration.

## Solution Assessment

No. The patch is a useful start, but the narrow maintainable fix should reuse the repo’s existing parsed-host Azure pattern, suppress OpenAI bearer auth, build Azure deployment-scoped embedding URLs, and add a supported API-version config surface instead of hard-coding a query default in the constructor.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Suppress bearer auth for Azure embeddings:** `extensions/memory-lancedb/index.ts:174-179`
  - body: The Azure branch still passes `apiKey` into `new OpenAI(...)` and only adds an `api-key` default header. In openai-node 6.34.0, the base client contributes `Authorization: Bearer <key>` before default headers are merged, so Azure requests can still carry the OpenAI bearer auth shape unless it is explicitly removed or the Azure client/request path is used.
  - confidence: 0.91
- **[P2] Build deployment-scoped Azure embedding URLs:** `extensions/memory-lancedb/index.ts:175-180`
  - body: This leaves `baseURL` unchanged while the existing embed path posts to `/embeddings`. If a user points `baseUrl` at an Azure resource root, the request goes to the wrong root-style endpoint instead of Azure's `/openai/deployments/{deployment}/embeddings` shape.
  - confidence: 0.88
- **[P2] Make the Azure API version configurable:** `extensions/memory-lancedb/index.ts:179-180`
  - body: The patch hard-codes `api-version=2024-02-01`, but memory-lancedb runtime config, manifest schema, and setup hints have no `apiVersion` field. Users needing a different Azure-supported version would still have no supported override path.
  - confidence: 0.9
- **[P2] Parse all supported Azure host suffixes:** `extensions/memory-lancedb/index.ts:173`
  - body: `baseUrl?.includes(".openai.azure.com")` is narrower and less precise than the repo's existing Azure handling. It misses recognized `.services.ai.azure.com` and `.cognitiveservices.azure.com` hosts and can match text outside the parsed hostname.
  - confidence: 0.86
