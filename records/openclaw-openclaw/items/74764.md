## Summary

Current main still has the runtime mismatch described by the PR body: Bedrock Opus 4.7 now advertises `max`, but the active Pi runner maps OpenClaw `max` to Pi `xhigh` before the Bedrock payload is built. The PR adds Amazon Bedrock wrapper logic and tests to preserve Claude Opus 4.7 `/think max` by rewriting the final Bedrock `output_config.effort` to `max` while leaving `xhigh` unchanged.
