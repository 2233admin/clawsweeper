## Summary

Keep open. Current main narrows the Active Memory recall example and new local onboarding defaults to the `coding` profile, but the broader least-privilege default requested here is still not implemented: unset/`full` profiles are unrestricted, absent or empty policies allow tools, missing/empty runtime `toolsAllow` keeps the full tool list, and docs still describe broad default/sub-agent behavior.

## What This Changes

The issue requests disabled-by-default, explicit per-agent tool permissions so narrow agents and embedded memory recall do not receive the full tool catalog unless configured.

## Best Possible Solution

Define explicit least-privilege default semantics across global, per-agent, provider, sub-agent, embedded-runner, plugin, and MCP tool resolution, while preserving an intentional opt-in unrestricted escape hatch such as `tools.profile: "full"`; ship migration or warning behavior, update schema/help/docs, and add regression coverage for the affected runtime paths.
