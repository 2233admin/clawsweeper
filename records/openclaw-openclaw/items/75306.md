## What This Changes

The PR changes the ACPX runtime to drop `timeout` and `timeout_seconds` `setConfigOption` calls before backend-specific handling, updates the Claude timeout regression test, and adds a changelog entry.

## Best Possible Solution

Land this narrow ACPX runtime fix after required CI completes, keeping the manager-level turn timeout as the canonical OpenClaw enforcement point and avoiding dependency, ACP core, or broader runtime-control changes for this regression.

## Reproduction Assessment

Yes. Current main can be traced to a high-confidence failure path: persisted ACP runtime timeout options are replayed as `timeout`, Claude ACP 0.31.1 does not advertise or build that config option, and its handler throws `Unknown config option`; the PR body also includes a live Dash `sessions_spawn` repro.

## Solution Assessment

Yes. Filtering the unsupported timeout keys at the ACPX wrapper is the narrowest maintainable fix for this backend regression because OpenClaw already persists and enforces the turn timeout in the manager; a broader capability-negotiation redesign is not required for this focused bug.
