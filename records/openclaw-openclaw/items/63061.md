## Summary

Keep open. Current main still has the reported CLI/profile mismatch, and this PR is a focused attempt to fix it, but it should not merge until maintainers decide and document the precedence between explicit OPENCLAW_HOME and named profile isolation.

## What This Changes

The PR changes CLI profile state resolution so an explicit OPENCLAW_HOME uses <OPENCLAW_HOME>/.openclaw without a profile suffix, adds a normalized OPENCLAW_HOME helper, and updates profile tests for named, dev, and sentinel cases.

## Best Possible Solution

Define one documented precedence contract across CLI, daemon/service helpers, and docs: either OPENCLAW_HOME owns the physical state root and profiles only affect labels/ports inside that root, or named profiles continue to derive separate state dirs under that root. Then land code, tests, help, and docs that all express the same contract.
