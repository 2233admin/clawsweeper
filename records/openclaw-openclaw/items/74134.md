## Summary

Keep open. This PR has the protected `maintainer` label, current `main` does not already contain the bundled file-transfer plugin, and the latest head still needs maintainer security review because file commands can be reached through raw `node.invoke` surfaces without the plugin gate.

## What This Changes

The PR adds a bundled `extensions/file-transfer` plugin with file and directory fetch/list/write tools, matching node-host commands, file-command policy constants, media-invoke redirects, audit wording, dependencies, and tests.

## Best Possible Solution

Keep the branch open for maintainer security and architecture review; enforce file-transfer authorization at every gateway path that can invoke the new node commands, or block raw file commands outside the dedicated tools, then validate the bundled plugin metadata, tests, and package/build gates before deciding whether it should ship enabled by default.
