## Summary

Current main still does not implement this PR's central lock-loss protection. The watchdog force-releases held session locks without notifying the active run, `acquireSessionWriteLock` has no `onForceRelease` callback or lock-held query, and the attempt path opens a guarded `SessionManager` without any post-lock-loss write guard. The PR remains a plausible core agents fix, but it is unmerged, currently dirty against main, and needs maintainer review/rebase rather than closure.

## Best Possible Solution

Keep this PR open for maintainer review and rebase. The best core direction is to add a stable lock-loss path around `src/agents/session-write-lock.ts` and `src/agents/pi-embedded-runner/run/attempt.ts`: notify the active run before watchdog release, abort the run and active session, reject later transcript writes after ownership loss, cover rewrite paths, and add focused regression tests for watchdog callbacks, lock-held checks, lost-lock write rejection, cleanup ordering, and active-session abort behavior.
