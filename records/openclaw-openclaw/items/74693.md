## Summary

Keep open. Current main mitigates plain `openclaw status` by keeping default text status off the setup runtime fallback, but the fallback loader itself still derives `../../plugins/loader.js` from `import.meta.url`; in a packaged `dist/read-only-*.js` chunk that resolves outside the `openclaw` package. The issue remains actionable because the same fallback is still used by deeper status/channel/doctor paths and should be bundler-safe even when an external plugin is missing `channelConfigs`.

## What This Changes

The issue requests that read-only channel setup fallback stop crashing in packaged installs when resolving the plugin runtime loader for an external channel plugin.

## Best Possible Solution

Land a targeted core fix that gives read-only channel setup fallback a stable source/package-safe way to call `loadOpenClawPlugins`, with regression coverage for packaged `dist/read-only-*.js` resolution; leave Tencent/openclaw-weixin#93 as the separate plugin metadata follow-up.
