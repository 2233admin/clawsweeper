## Summary

Current main now fixes the reported packaged loader crash. The read-only channel setup fallback detects when it is running from a bundled `dist/` chunk, resolves the plugin loader from the installed OpenClaw package's `dist/plugins/*` entries instead of `../../plugins/*`, and the changelog records this exact issue as fixed on main after the `v2026.4.27` release.

## What This Changes

The issue reports that `openclaw status` can crash in the read-only channel setup fallback because the packaged fallback loader resolves `../../plugins/loader.js` from a bundled `dist/read-only-*.js` chunk.

## Best Possible Solution

Keep the main-branch packaged loader fix and let the next release carry it; the Tencent WeChat plugin should still add its `channelConfigs` metadata separately, but OpenClaw core no longer depends on that external fix to avoid this crash.
