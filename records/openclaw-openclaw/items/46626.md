## Summary

Keep open. Current main still does not implement the PR's full own-line NO_REPLY suppression across shared reply normalization, reply directives, and gateway live/history projection, while the cron part is partially superseded by newer scoped cron suppression. The PR is useful but not merge-ready because it has a failing inline-token test expectation and can drop channelData-only deliverables when reclassifying mixed own-line text as silent.

## What This Changes

The PR adds an own-line NO_REPLY helper, applies it across reply normalization, directive parsing, cron delivery/subagent follow-up, gateway streaming, and chat history filtering, and adds token/reply-directive/normalization tests.

## Best Possible Solution

Consolidate this work with #42873, #42472, and the narrower trailing-token alternatives into one maintained fix: add a shared cached own-line/trailing silent-intent helper, apply it consistently to normalization, directive/outbound planning, gateway live/history projection, and cron/subagent paths, and add focused tests that preserve inline text plus media/channelData semantics.
