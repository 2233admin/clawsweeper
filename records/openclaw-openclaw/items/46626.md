## Summary

Keep this PR open. Current main still does not implement the PR’s full own-line `NO_REPLY` suppression behavior, and this PR is paired with the same author’s still-open dependency #42873 plus the still-open canonical feature/bug report #42472. It also is not merge-ready as-is because review comments identify a failing added test and a channelData-preservation regression risk. A later cron-specific fix partially overlaps the patch, so the useful path is a rebase or replacement patch rather than auto-closing it.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to consolidate it with #42873 and #42472, add a shared cached own-line silent-token helper, rebase over the current cron trailing-token fix, update only the still-missing gateway/reply-directive/history surfaces, preserve channelData/media/presentation semantics, and add focused tests for normal reply normalization, reply directives, outbound payload planning, gateway live/history projection, and cron subagent follow-up.
