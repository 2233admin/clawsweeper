## Summary

Keep this PR open. Current main still does not implement the PR’s full own-line `NO_REPLY` suppression across reply normalization, directive parsing, gateway live/history projection, and cron subagent follow-up. A later cron-specific fix overlaps part of the PR, but the broader user-visible leak remains and the PR is also paired with open #42873 and #42472.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to consolidate it with #42873 and #42472, add a shared cached own-line silent-token helper, rebase over the current cron trailing-token fix, update only the still-missing reply-directive, gateway live/history, and cron subagent surfaces, preserve media/channelData/presentation semantics, and add focused regression tests for each delivery path.
