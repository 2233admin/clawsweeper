## Summary

Keep open. Current main still counts every top-level session file ending in `.jsonl` for `openclaw memory status`, while the indexing path uses the shared usage-counted transcript listing that excludes compaction checkpoint snapshots. A checkpoint file can still produce the reported `Indexed: 3/4 files` mismatch.

## Best Possible Solution

Keep the issue open and fix `extensions/memory-core/src/cli.runtime.ts` so `scanSessionFiles` counts the same session transcript corpus that indexing can process, preferably by reusing a shared host/runtime predicate or listing contract. Add focused `memory status` coverage with a normal transcript plus a `.checkpoint.<uuid>.jsonl` snapshot, and include a user-facing changelog fix crediting the reporter.
