## Summary

Keep open. Current main still counts every top-level session `.jsonl` in `openclaw memory status`, while the actual session indexing path uses the shared usage-counted transcript filter that excludes compaction checkpoint snapshots. That preserves the reported `Indexed: 3/4 files` style mismatch when a `.checkpoint.<uuid>.jsonl` exists.

## Best Possible Solution

Keep the issue open and fix `extensions/memory-core/src/cli.runtime.ts` so `scanSessionFiles` uses the same session transcript predicate or shared listing contract as indexing. Add focused coverage for a normal transcript plus a `.checkpoint.<uuid>.jsonl` snapshot, verifying `memory status` reports the sessions total without the checkpoint while the indexed file count remains unchanged.
