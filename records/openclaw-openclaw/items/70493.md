Keep open. Current main still routes sandboxed/isolated browser calls through the sandbox browser bridge and the same Playwright-gated navigate/act/snapshot routes, while the issue discussion has explicit maintainer direction to keep this open until isolated-session coverage exists.

The issue requests consistent Playwright-backed browser navigate, snapshot, and act support in isolated sub-agent sessions, matching the main session behavior.

## Best Possible Solution

Define the browser runtime owner for isolated sessions, implement that path without weakening sandbox boundaries, and add an isolated sub-agent or sandbox-bridge regression test proving navigate or snapshot no longer returns the Playwright-unavailable error when the owning browser runtime supports Playwright.
