## Summary

Keep this open. Current main still has the same unresolved failure path: sandboxed/sub-agent browser tools default to the sandbox bridge, while navigate, act, and AI/Playwright-backed snapshots can still return the Playwright-unavailable 501 from that bridge. The maintainer comment also explicitly says not to close until isolated-session browser coverage exists.

## What This Changes

The issue requests full Playwright-backed browser support for isolated sub-agent sessions so navigate, snapshot, and act work consistently with the main session.

## Best Possible Solution

Keep this issue open until isolated sub-agent browser calls use a Playwright-capable runtime through a clearly owned path and a regression test exercises navigate or snapshot through the real isolated-session tooling without returning the Playwright-unavailable error.
