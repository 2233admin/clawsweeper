## Summary

Keep open. Current main still lacks a `silent` field in the cron failure-alert contract, gateway validation, config schema, UI form state, and runtime delivery plumbing, while the downstream outbound and Telegram delivery layers already support silent messages.

## What This Changes

Add an optional `silent: boolean` setting to cron `failureAlert` so Telegram failure alerts can be delivered without sound or push notification.

## Best Possible Solution

Add a narrow, backward-compatible cron contract field that resolves from per-job and global failure-alert config, passes through only the announce delivery path into outbound delivery, updates the Control UI/CLI/docs/schema surfaces, and leaves Telegram’s existing message-tool behavior unchanged.
