## Summary

Close. Current main already fixes the linked null-params bug with a schema-level `prepareArguments` seam merged in #72673 and shipped in `v2026.4.27`; this PR should not merge because its wrapper-level normalization would run after the validation boundary that was failing.

## What This Changes

The PR adds a before-tool-call parameter normalization helper that converts `null` tool params to `{}` for empty object schemas and uses the normalized params for execution, diagnostic events, adjusted-param tracking, and loop outcomes.

## Best Possible Solution

Keep the shipped schema-level `prepareArguments` implementation from #72673 and close this obsolete PR rather than merging a second, later-bound normalization path.

## Reproduction Assessment

Yes. The linked report gives a concrete `wiki_lint` tool call with `arguments: null`, and current main has a regression that drives that shape through `runAgentLoop` and verifies the tool receives `{}` instead of a validation error.

## Solution Assessment

No for this PR. The maintainable fix is the already-merged `normalizeToolParameters`/`prepareArguments` path because it runs before `pi-ai` argument validation; this PR normalizes inside the wrapped execute path after that validation boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Move null normalization before tool validation:** `src/agents/pi-tools.before-tool-call.ts:484`
  - body: The failing `root: must be object` validation happens before the wrapped tool `execute` handler is entered, so normalizing `outcome.params` here would not run for the reported `null` argument call. The fix needs to use the pre-validation argument preparation seam, as current main now does in `normalizeToolParameters`.
  - confidence: 0.9
- **[P2] Treat empty schema fields as empty:** `src/agents/pi-tools.before-tool-call.ts:109-110`
  - body: `Array.isArray(required)` and `properties !== undefined` both reject common parameterless schemas such as `{ required: [] }` or `{ properties: {} }`. Those shapes are semantically empty, so this helper would still leave `null` unnormalized for the exact TypeBox-style schemas used by many no-param tools.
  - confidence: 0.86
