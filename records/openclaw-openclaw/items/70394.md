Do not close this PR. Current main still keeps the strict plugin runtime/metadata load behavior that #70394 tries to change, and tests still assert throwing on invalid plugin config when `throwOnLoadError` is enabled. The PR needs maintainer review/rework rather than cleanup closure because the linked bug remains live, while the branch also has unrelated changes and unresolved review concerns.

## Best Possible Solution

Keep the PR open for maintainer review, but require a narrowed/rebased implementation. The safest path is a focused plugin-loader fix for #70371/#61683 that lets metadata/doctor/status and non-required plugin config validation report per-plugin errors without crashing, while preserving fail-closed behavior for required/runtime-critical plugins and avoiding caching or activating failed registries silently. Add regression coverage around `loadPluginMetadataRegistrySnapshot`, `ensurePluginRegistryLoaded`, diagnostics, and retry/cache behavior; move unrelated QQBot, Vertex, and reply-media work to separate PRs or drop it if already handled on main.
