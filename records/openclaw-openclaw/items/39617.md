## Summary

Keep PR #39617 open. Current main still has the same unresolved Slack and Discord routing issue: slash/native-command route resolution uses provider-startup config snapshots while the shared router still derives DM session keys from `session.dmScope`. Telegram has since been moved to a fresh runtime config path with regression coverage, so this PR is partially superseded and needs a rebase rather than closure.

## Best Possible Solution

Keep PR #39617 open, or keep #39605 as the canonical bug if maintainers prefer a fresh replacement PR. Rebase the useful remaining fix onto `extensions/discord/src/monitor/native-command.ts`, `extensions/discord/src/monitor/native-command-route.ts`, and `extensions/slack/src/monitor/slash.ts`; load current runtime config for native/slash route resolution and configured binding readiness; preserve Telegram's existing fresh-runtime-config behavior; and add focused Slack/Discord regression tests that simulate startup config followed by changed `session.dmScope` or bindings without provider restart.
