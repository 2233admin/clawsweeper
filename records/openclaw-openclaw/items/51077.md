## Summary

Keep this PR open. Current main only implements part of the requested behavior: there is an `NSWindow` chat path and macOS file drop support, and the Talk Mode crash has been fixed separately, but the status-item left-click path still opens the dismissing panel and the shared chat composer is still capped rather than resizable. The PR also contains unresolved review findings and unrelated security-sensitive direct Gemini image-generation changes, so it is not obsolete or safely closeable as implemented on main.

## Best Possible Solution

Keep this PR open only as potential salvage material. The best maintainer path is to ask for a rebased, much narrower macOS PR: wire the persistent `NSWindow` chat path to the intended menu-bar action, keep drag/drop behavior platform-safe, add a macOS-only resizable composer with iOS unchanged, drop or split the direct Gemini/image-generation work, and address the existing P1/P2 review comments before merge consideration.
