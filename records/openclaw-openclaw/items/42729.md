## Summary

Keep this PR open. Current main still lacks the PR's stale already-executed due-slot guard in the cron runnable path, while restart catch-up still loads persisted nextRunAtMs values before recomputing. The PR remains a focused candidate fix with a matching regression test and no visible security-sensitive drift.

## Best Possible Solution

Review and land this PR, or port the same strict stale-slot guard into current main's isRunnableJob/startup catch-up path together with the regression test. The implementation should preserve missed-slot catch-up and same-millisecond retry behavior.
