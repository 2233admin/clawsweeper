## Summary

Keep this PR open. Current main still lacks the PR's stale past-due slot guard in the cron runnable path, while startup catch-up intentionally loads persisted nextRunAtMs values before recompute. The linked bug is closed, but the PR's narrower stale-slot fix and regression test still cover a real remaining scheduler gap.

## Best Possible Solution

Keep this PR open and review or land it, or port the same strict stale-slot guard into current main's isRunnableJob/startup catch-up path with the PR's regression test. The implementation should explicitly preserve missed-slot catch-up and same-millisecond retry behavior.
