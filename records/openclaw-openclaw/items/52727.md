## Summary

Keep open. Current main still lacks the proposed instruction-file precedence directive, and the PR is the active narrow implementation. The main blocker is not mechanics but trust policy: the added line would tell agents to prefer an active SKILL.md over AGENTS.md, TOOLS.md, and SOUL.md, while current docs treat third-party skills as untrusted.

## What This Changes

The PR adds a one-line `SKILL.md (active skill) > AGENTS.md > TOOLS.md > SOUL.md` precedence directive to the Skills system-prompt section and extends the system-prompt test to assert it.

## Best Possible Solution

Land a maintainer-approved conflict-resolution rule in the Skills prompt section, with tests and docs aligned; if workspace policy should remain above skills, revise the hierarchy before merge so the prompt matches OpenClaw's intended trust model.
