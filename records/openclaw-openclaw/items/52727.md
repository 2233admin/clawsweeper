## Summary

Keep this PR open. Current main does not implement the PR’s intended instruction-precedence chain; it only has general skill-loading guidance and separate skill-location precedence docs. The PR is small and still potentially useful, but the SKILL.md-over-AGENTS.md policy is a prompt-authority/product decision that maintainers should review rather than auto-close.

## Best Possible Solution

Review this PR as a focused prompt-policy change. If maintainers want active skills to outrank workspace bootstrap instructions, land or recreate the one-line prompt change with the full test assertion and consider a small docs note in the skills or system-prompt docs. If the intended policy is different, close the PR with that product/security rationale rather than treating it as already implemented.
