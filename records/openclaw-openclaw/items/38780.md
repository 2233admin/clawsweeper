Full review comments:

- **[P2] Forward the delegate drain flag into embedded tools:** `src/agents/pi-embedded-runner/run/attempt.ts:689-690`
  - body: `runEmbeddedPiAgent` accepts `drainsContinuationDelegateQueue`, but this tool construction forwards only the continuation work/compaction callbacks into `createOpenClawCodingTools`. Since `createOpenClawTools` exposes `continue_delegate` whenever the flag is undefined, command/utility runs can advertise a scheduling tool whose queued delegates are not drained by that path, leading to lost or later replayed work. Thread the flag through `createOpenClawCodingTools` and into `createOpenClawTools`, then gate the tool on an explicit draining run.
  - confidence: 0.86

This hides the exact lazy/static import boundary warning the repo normally treats as a build guard, and this PR changes runtime lazy-loading paths around continuation and embedded runs. Please fix the import graph or add a stable runtime facade instead of suppressing the warning.
  - confidence: 0.74
