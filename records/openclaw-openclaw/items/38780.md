## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Forward the delegate drain flag into embedded tools:** `src/agents/pi-embedded-runner/run/attempt.ts:689-690`
  - body: `runEmbeddedPiAgent` accepts and forwards `drainsContinuationDelegateQueue` to the attempt layer, but this tool construction forwards only the continuation work and compaction callbacks into `createOpenClawCodingTools`. Because `createOpenClawTools` exposes `continue_delegate` when the flag is undefined, embedded command/utility runs can advertise a scheduling tool whose queued delegates are not drained by that path. Thread the flag through `createOpenClawCodingTools` and into `createOpenClawTools`, then gate the tool on an explicit draining run.
  - confidence: 0.86
- **[P2] Fix the pi-embedded dynamic import boundary:** `tsdown.config.ts:63-66`
  - body: The build config now suppresses `INEFFECTIVE_DYNAMIC_IMPORT` for `pi-embedded.ts`. That hides the lazy/static import boundary warning the repo normally treats as a build guard, and this PR changes runtime lazy-loading paths around continuation and embedded runs. Please fix the import graph or add a stable runtime facade instead of suppressing the warning.
  - confidence: 0.74
