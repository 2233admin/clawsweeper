## Summary

Keep open. Current main still lacks the nullish guard this PR proposes for `parseAbsoluteTimeMs`, so the cron parser fix is not obsolete. The PR should not be merged as-is because it also bundles an unrelated `openclaw update` early return that conflicts with documented same-version package refresh and post-update behavior.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but split the work. The cron change should keep the nullish guard, parser tests, and ideally add a targeted CLI regression for malformed `{ kind: "at" }` schedules. The update-command early return should be removed from this PR or handled separately with a design that preserves same-version package refresh, plugin sync, completion refresh, restart, and doctor semantics.
