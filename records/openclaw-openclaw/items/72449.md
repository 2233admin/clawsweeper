## Summary

Keep open. Current main still lacks the nullish guard this PR proposes for `parseAbsoluteTimeMs`, and malformed cron `at` schedules can still reach that parser from CLI formatting. The PR is not safe to close as obsolete, but it also should not be merged as-is because it bundles an unrelated update-command early return that conflicts with current documented update behavior.

## Best Possible Solution

Keep this PR open for maintainer review, but split the work. The cron parser guard and parser tests remain useful; add a targeted CLI regression for malformed `at` schedules. Remove the update-command early return from this PR, or handle that behavior separately with a design that preserves documented package refresh, plugin sync, completion refresh, restart, and doctor semantics.
