## Summary

Keep open. Current main still has the unsafe `parseAbsoluteTimeMs(input: string)` implementation and malformed cron rows can still route an undefined `schedule.at` into that parser. The PR's cron guard is useful, but the branch also bundles an unrelated `openclaw update` early return that conflicts with documented same-version package refresh and post-update behavior.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but split the work. The cron portion should keep the nullish parser guard, parser tests, and add a targeted CLI regression for malformed `{ kind: "at" }` schedules. The update-command early return should be removed from this PR or handled separately with a design that preserves same-version package refresh, plugin sync, completion refresh, restart, and doctor semantics.
