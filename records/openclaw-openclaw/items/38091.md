## Summary

Current main still does not recover an active Control UI/WebChat chat run across a WebSocket reconnect. The reconnect hello path still clears `chatRunId`/stream state, the hello snapshot still has no active chat-run recovery payload, and #41067 remains the open implementation candidate for adding that recovery behavior. #45952 is related but tracks a separate outbound queue/ACK replay problem.

## Best Possible Solution

Keep this issue open as the canonical Control UI/WebChat active-run reconnect recovery report. Review and refresh #41067 or land an equivalent narrow fix: add an additive active chat-run recovery contract in the gateway hello/snapshot or a dedicated reconnect RPC, preserve/reconcile `chatRunId` and busy/streaming state across reconnect, avoid clearing active state until the run is known terminal, and add regression coverage for WebSocket close/reconnect during an active `chat.send`. Keep #45952 separate for outbound message queue/ACK loss.
