## Summary

Keep this PR open. Current main still injects short OpenRouter Anthropic cache markers on verified OpenRouter routes without consulting `cacheRetention`, so the PR's core bug fix remains useful. The patch does need a refresh against current main because the cache-retention code moved since the PR was opened.

## Best Possible Solution

Refresh and review this PR rather than closing it. The best implementation is to resolve `cacheRetention` for OpenRouter Anthropic `openai-completions` routes, pass the resolved value into the OpenRouter system/developer cache-marker wrapper or shared helper, keep default short behavior, suppress markers for `"none"`, emit `{ type: "ephemeral", ttl: "1h" }` for `"long"`, preserve endpoint gating and thinking-block cleanup, and add regression tests for default, `none`, `short`, `long`, non-Anthropic OpenRouter, and custom proxy exclusions.
