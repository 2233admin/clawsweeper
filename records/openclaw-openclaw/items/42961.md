## Summary

Keep this PR open. Current main still has the reported gap: OpenRouter Anthropic payloads get the short `{ type: "ephemeral" }` cache marker unconditionally, while `cacheRetention: "none"` and `cacheRetention: "long"` are not routed into the OpenRouter marker path.

## Best Possible Solution

Refresh and review this PR rather than close it. The best fix is to route effective OpenRouter Anthropic `cacheRetention` into cache-marker application, preserve default short behavior, suppress markers for `"none"`, emit `{ type: "ephemeral", ttl: "1h" }` for `"long"`, keep custom OpenAI-compatible proxy exclusions intact, preserve public SDK helper compatibility, and add regression coverage for default/short/none/long plus non-Anthropic and bundled-proxy consumer cases.
