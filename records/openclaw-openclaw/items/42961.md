## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: OpenRouter Anthropic cache markers are still applied by a post-plugin wrapper with no `cacheRetention` input, the marker helper always emits the short `{ type: "ephemeral" }` form, and the retention resolver still does not classify OpenRouter's `openai-completions` Anthropic route as cache-retention eligible. The PR remains a focused bug-fix candidate, but it needs refresh/review against the current helper split and route-gating code.

## Best Possible Solution

Refresh and review this PR. The best implementation is to route the effective OpenRouter Anthropic `cacheRetention` into cache-marker application, keep default short behavior, suppress markers for `"none"`, emit `{ type: "ephemeral", ttl: "1h" }` for `"long"`, preserve verified-OpenRouter route checks and custom proxy exclusions, and add regression coverage for default/short/none/long plus non-Anthropic and custom-proxy paths.
