## Summary

Keep this PR open. Current main still hard-codes short OpenRouter Anthropic cache markers and does not route the effective `cacheRetention` value into the OpenRouter system-cache wrapper, so the PR’s central bug fix is not implemented yet.

## Best Possible Solution

Refresh and review this PR rather than close it. The best fix is to pass the effective OpenRouter Anthropic `cacheRetention` into cache-marker application, preserve default short behavior, suppress markers for `"none"`, emit `{ type: "ephemeral", ttl: "1h" }` for `"long"`, keep custom OpenAI-compatible proxy exclusions intact, preserve public SDK helper compatibility, and add regression tests for default/short/none/long plus non-Anthropic and custom-proxy cases.
