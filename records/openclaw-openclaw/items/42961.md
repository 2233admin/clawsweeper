## Summary

Keep this PR open. Current main still injects short OpenRouter Anthropic cache markers unconditionally and does not route the effective `cacheRetention` value into the OpenRouter cache-marker path, so the PR’s intended fix is not implemented yet.

## Best Possible Solution

Refresh and review this PR rather than close it. The best fix is to route effective OpenRouter Anthropic `cacheRetention` into cache-marker application, preserve default short behavior, suppress markers for `"none"`, emit `{ type: "ephemeral", ttl: "1h" }` for `"long"`, keep custom OpenAI-compatible proxy exclusions intact, preserve public SDK helper compatibility, and add regression coverage for default/short/none/long plus non-Anthropic and custom-proxy cases.
