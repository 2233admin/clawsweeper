## Summary

Keep open. Current main still has the narrow Windows world SID/principal set, while this focused PR targets a real security-audit severity gap; because it changes security-owned classification policy, it should receive secops maintainer review and targeted validation rather than be closed or replaced automatically.

## What This Changes

The PR expands Windows ACL world-principal and SID classification and adds classifier plus filesystem-audit regression tests for broad Windows principals such as Anonymous Logon, Guests, Interactive, Local, and Network.

## Best Possible Solution

Land a focused security fix, after secops sign-off, that classifies the broad Windows well-known principals into untrustedWorld while preserving trusted SYSTEM, Administrators, and current-user SID handling; keep the classifier and filesystem-audit regressions, refresh the stale guard comment, and add a changelog entry if maintainers treat the severity change as user-visible.
