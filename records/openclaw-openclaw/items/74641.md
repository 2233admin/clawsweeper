## Summary

Keep open: the PR targets a real ACP resolver bug from #66299, but the current branch overcorrects the fallback chain. It fixes the bound-thread case while allowing an unresolved explicit token to continue into requester-session fallback, which can route a bad target to the wrong session instead of returning the documented bad-target error.

## What This Changes

The PR changes ACP target resolution so unresolved explicit session tokens fall through to existing fallbacks, adds a bound-thread `/acp steer` regression test, and adds an Unreleased changelog fix entry.

## Best Possible Solution

Update the resolver to try an explicit token first, then allow only the thread-bound lookup to recover from an unresolved token. If a non-empty token still has not resolved after the thread-bound lookup, return `Unable to resolve session target` before requester-session fallback; keep the bound-thread regression and add no-thread invalid-token coverage.
