## Summary

Keep open. Current main still maps terminal Gateway timeout snapshots with lifecycle metadata to `accepted` unless `stopReason` or `aborted` is present, while the Gateway can return terminal `status: "timeout"` snapshots separately from bare wait-deadline timeouts. The proposed patch is narrow and appears correct, but it still needs normal maintainer review and CI/Testbox validation before merge.

## What This Changes

The PR updates SDK `Run.wait()` status normalization for terminal Gateway timeout snapshots, adds a focused SDK regression test, adds a changelog entry, and adjusts a changed-test routing expectation.

## Best Possible Solution

Merge this PR or an equivalent narrow SDK fix after maintainer review and validation, preserving the documented split between bare wait-deadline `accepted` results and terminal runtime `timed_out` results.
