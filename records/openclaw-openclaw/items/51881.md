## Summary

Keep this issue open. Current main still lets a non-generic sniffed MIME such as `image/apng` win over the provided MIME in gateway attachment parsing, then forwards that value unchanged to Anthropic-style image payloads. Live PR #51964 remains open and unmerged with closing syntax for this exact issue, so the best cleanup path is to review/refresh that PR or replace it with the same focused fix before closing.

## Best Possible Solution

Keep this issue open and review PR #51964, or replace it with a maintainer patch that normalizes provider-facing `image/apng` to `image/png` before `finalMime` is used for inline and offloaded image payloads. Add focused regression coverage in `src/gateway/chat-attachments.test.ts` or a central media MIME test, then close after the fix lands with release provenance.
