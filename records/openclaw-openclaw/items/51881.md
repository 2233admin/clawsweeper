## Summary

Keep this issue open. Current main still allows a sniffed `image/apng` MIME to win attachment MIME resolution and forwards that MIME to Anthropic-style image blocks unchanged. The provided GitHub context also shows open PR #51964 with closing syntax for this exact bug, so the issue should remain open until that implementation candidate is reviewed and merged or explicitly closed.

## Best Possible Solution

Keep this issue open and review #51964, or replace it with a maintainer patch that normalizes provider-facing `image/apng` to `image/png` before `finalMime` is used for inline and offloaded image payloads. Add regression coverage in `src/gateway/chat-attachments.test.ts` and, if normalization is centralized, `src/media/mime.test.ts`; close only after the fix lands with release provenance.
