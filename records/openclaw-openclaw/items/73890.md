## Summary

Keep this PR open. Current main still silently redirects an unwritable Linux npm global prefix to `~/.npm-global` with only an info message, while this PR adds the missing pre-redirection warning, the post-redirection no-sudo update warning, changelog attribution, and a targeted ordering test.

## Best Possible Solution

Keep this PR open for maintainer review and, if validation stays green, merge the focused installer warning/test change. The best implementation boundary is exactly this path: warn before `npm config set prefix "$HOME/.npm-global"`, warn after it that future global updates should use non-sudo `npm i -g openclaw@latest`, preserve #50479/@Sayeem3051 attribution, and avoid broader installer refactors.
