## Summary

Keep open. Current main still has the reported broad sibling-home scan and warning output: Doctor enumerates `/home` on Linux and `/Users` on macOS, probes every non-dot sibling directory for `.openclaw`, and prints discovered paths. I found no current-user scoping, backup-name filtering, redaction for non-current-home paths, or documented env/config opt-out. The related #68752 corroborates the same behavior but is closed, so it does not supersede this issue.

## What This Changes

Constrain or suppress Doctor’s broad sibling-home `.openclaw` scan so it does not expose other users’ state paths or warn on backup/snapshot sibling directories.

## Best Possible Solution

Keep the issue open and change the Doctor state-integrity check so broad sibling-home enumeration is not the default source of user-visible paths. A good endpoint would scope detection to the current effective home/user and explicit alternate state dirs, redact or summarize non-current-home matches if any remain, add focused tests for `/home/alice`/`/home/bob` and backup-sibling layouts, and document any operator suppression control that maintainers approve.
