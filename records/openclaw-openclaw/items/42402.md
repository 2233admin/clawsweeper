## Summary

Keep this PR open. Current main still lacks the runtime type guard in the `env.vars` collection path, so a truthy non-string value parsed from JSON can still flow into `containsEnvVarReference()` and throw before schema validation rejects the malformed config. The updated PR diff is narrow and relevant: it adds the guard, a regression test, and a changelog entry, with no unexplained security-sensitive or supply-chain changes in the provided diff.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to review and land the narrow guard in `src/config/config-env-vars.ts`, keep the mixed-type regression in `src/config/config.env-vars.test.ts`, make an explicit maintainer call on silent skip versus diagnostics, then run the targeted config tests and the appropriate changed gate before merge.
