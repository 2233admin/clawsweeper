## Summary

Keep this PR open. Current main still lacks the proposed runtime type guard in the `env.vars` collection path, so truthy non-string JSON values can still flow into `containsEnvVarReference()` and throw. The patch is stale against the renamed implementation file, but it remains a focused, valid bug-fix candidate with no supply-chain-sensitive diff surface.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Rebase the one-line `typeof value !== "string" || !value.trim()` guard into `src/config/config-env-vars.ts`, add mixed-type regression coverage in `src/config/config.env-vars.test.ts`, decide whether skipped malformed values should warn, then run the targeted config tests and the appropriate changed gate.
