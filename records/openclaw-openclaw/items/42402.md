## Summary

Keep this PR open. Current main still lacks the proposed runtime type guard in the `env.vars` collection path, so truthy non-string JSON values can still be collected and passed into `containsEnvVarReference()`, which still calls `.includes()` on a string-typed value. The linked issue is closed and the earlier broader PR #42377 is also closed unmerged, so there is no merged canonical fix for this remaining `env.vars` edge.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Rebase the `typeof value !== "string" || !value.trim()` guard into `collectConfigEnvVarsByTarget` in `src/config/config-env-vars.ts`, add mixed-type `env.vars` regression coverage in `src/config/config.env-vars.test.ts`, preserve the public string-valued config contract, decide whether skipped malformed values should warn, then run the targeted config tests and the appropriate changed gate.
