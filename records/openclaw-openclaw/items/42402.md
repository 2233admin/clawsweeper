## Summary

Keep this PR open. Current main still lacks the proposed runtime type guard for `env.vars`, so truthy non-string values can still be collected as config env values and passed to `containsEnvVarReference()`, which assumes a string and calls `.includes()`. The PR is stale against the renamed implementation file, but it remains a focused, valid bug-fix candidate rather than implemented or superseded.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Rebase the one-line non-string guard into `src/config/config-env-vars.ts`, add the mixed-type regression coverage to `src/config/config.env-vars.test.ts`, decide whether skipped malformed values should emit a diagnostic, then run the targeted config tests and the appropriate changed gate.
