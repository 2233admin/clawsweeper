## Summary

Current main still does not paginate the bundled Feishu `feishu_wiki` `nodes` action. The implementation calls `client.wiki.spaceNode.list` once with only `parent_node_token`, the `nodes` dispatcher uses that helper directly, the schema/skill docs do not expose pagination behavior, and the latest release tag still contains the same single-page implementation. The related fix PR #37641 is closed unmerged, so this remains a valid narrow plugin bug.

## Best Possible Solution

Keep this issue open and fix it in the bundled Feishu plugin. Update `extensions/feishu/src/wiki.ts` so the `nodes` action requests a documented bounded page size, follows `has_more` and `page_token`, accumulates all node items, handles empty pages that still have `has_more=true`, caps the number of pages, and returns the existing `nodes` array plus lightweight metadata such as `total` and `pages_fetched`. Add focused Feishu wiki tests and update the skill docs if result metadata changes.
