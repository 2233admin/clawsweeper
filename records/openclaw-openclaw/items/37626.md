## Summary

Current main still does not paginate the bundled Feishu `feishu_wiki` `nodes` action. The helper makes one `wiki.spaceNode.list` call, the tool dispatcher returns that helper directly, schema/skill docs expose no pagination or fetch-all behavior, the latest release tag has the same single-page code, and related fix PRs are closed unmerged. This remains a valid narrow Feishu plugin bug and a good automated fix-PR candidate.

## What This Changes

The issue requests automatic pagination for the bundled Feishu `feishu_wiki` `nodes` action so it returns more than the Feishu API default first 20 wiki nodes.

## Best Possible Solution

Keep this issue open and land a focused Feishu plugin fix. Update `extensions/feishu/src/wiki.ts` so the `nodes` action requests a bounded page size, follows `has_more` and response `page_token`, accumulates all node items, handles empty pages that still have `has_more=true`, caps pages, and returns the existing `nodes` array plus lightweight pagination metadata. Add focused tests and update schema/skill docs only if optional pagination inputs or result metadata become part of the tool contract.
