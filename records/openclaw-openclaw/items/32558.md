## Summary

Keep this PR open. Current main still sends Microsoft Teams OneDrive and SharePoint files through direct Microsoft Graph `/content` uploads, with no upload-session fallback or large-file regression coverage. The PR remains a focused implementation candidate, but it needs maintainer review/rebase before merge because current main now requires OpenClaw User-Agent headers and the PR discussion already flags a stall-count off-by-one.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. Rebase it onto current main, preserve current Graph User-Agent behavior, fix the stall-count guard, confirm the upload-session threshold, and land a focused implementation in `extensions/msteams/src/graph-upload.ts` with OneDrive and SharePoint upload-session regression tests.
