## Summary

Keep this PR open. Current main still does not implement the Microsoft Teams Graph upload-session fallback: the OneDrive and SharePoint helpers both perform a single authenticated `/content` PUT with the full buffer, and the current graph-upload tests only assert the direct-upload path. The related earlier PRs #26668 and #26712 are closed unmerged, so this focused PR remains the relevant implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. Rebase it onto current main, add upload-session fallback inside the current `uploadToOneDrive` and `uploadToSharePoint` helpers, preserve current User-Agent/preconnect behavior, fix the stall-count guard, and land focused tests for simple upload, upload-session creation, chunk PUT progression/retry/stall handling, and both OneDrive and SharePoint paths.
