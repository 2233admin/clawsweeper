## Summary

Keep this PR open. Current main still lacks the Microsoft Teams Graph upload-session fallback: OneDrive and SharePoint uploads still perform single direct `/content` PUT requests, outbound Teams file paths still delegate to those helpers, and current tests only assert direct-upload behavior. The PR remains a focused implementation candidate split from the closed broader work, but it needs maintainer review, a rebase onto current User-Agent/preconnect conventions, and the flagged stall-count fix before merge.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. Rebase it onto current main, add the upload-session fallback inside the existing `uploadToOneDrive` and `uploadToSharePoint` helpers while preserving small-file direct `/content` uploads and current User-Agent/preconnect behavior, fix the stall guard, and land focused tests for session creation, chunk progression, retry/stall handling, and both OneDrive and SharePoint paths.
