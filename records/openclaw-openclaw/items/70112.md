## Summary

Keep open. The PR targets a real current-main CJK compaction bug and is the canonical implementation path for closed issue #70052, but it is not merge-ready because the new script-aware overflow estimate is still routed through raw character thresholds for tool-result truncation, and the branch is dirty against current main.

## What This Changes

The PR adds a CJK-aware wrapper around the upstream token estimator, threads it through agent compaction and preemptive compaction estimates, and adds focused regression tests for ASCII, CJK text, and CJK inside toolCall arguments.

## Best Possible Solution

Keep this PR as the active implementation path, but update preemptive truncation routing so the overflow comparison and tool-result recovery are both token-aware or both script-aware, then rebase onto current main without dropping transcript-sanitizing behavior and land with focused compaction regression coverage.
