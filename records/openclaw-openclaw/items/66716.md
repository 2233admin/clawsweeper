## Summary

Keep open. The reported cached-token preflight compaction bug is still present on current main, and the PR is a useful fix candidate, but it now needs maintainer consolidation/rebase because the same function has newer active-transcript byte-trigger work and overlapping open compaction reports.

## What This Changes

This PR changes preflight compaction to evaluate explicitly fresh persisted totalTokens, including cached prompt tokens, against the token threshold and adds focused regression tests for fresh, stale, legacy, and heartbeat paths.

## Best Possible Solution

Land one consolidated fix in the preflight compaction path that checks explicitly fresh persisted totals against the threshold, falls back to transcript estimation for stale or unknown freshness, preserves the active transcript-byte trigger, and adds runtime regression coverage for high cached totals, below-threshold fresh totals, legacy freshness, stale fallback, and heartbeat/CLI skips.
