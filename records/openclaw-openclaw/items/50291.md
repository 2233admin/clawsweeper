## Summary

Keep open. Current main partially addresses the original P0 run-correlation gap by backfilling `runId` onto `before_agent_start` and `agent_end`, and it has useful diagnostic trace/message-hook plumbing, but the public plugin-hook contract still lacks the requested first-class agent inbound identity fields, message_sending event correlation fields, and subagent requester-run hierarchy needed for accurate observability plugins.

## What This Changes

The issue requests richer trace and correlation fields across plugin hook events and contexts, including `messageId`, `senderId`, run IDs on all applicable hooks, span parent fields, and subagent parent/child run relationships.

## Best Possible Solution

Define one backwards-compatible plugin-hook correlation contract, then land focused slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases, with SDK baseline and public docs updates for plugin authors.
