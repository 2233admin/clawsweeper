## Summary

Keep open. Current main has partial hook-correlation work, including runId backfill for legacy agent boundary hooks and trace-aware message-hook context, but the public plugin hook contract still misses core requested fields for agent inbound identity, generic outbound message_sending correlation, and subagent requester-run hierarchy.

## What This Changes

The issue requests a complete plugin-hook correlation contract exposing runId, messageId, senderId, and span parentage across agent, message, compaction, and subagent hook events/contexts for observability plugins.

## Best Possible Solution

Define one backwards-compatible plugin-hook correlation contract, then land focused implementation slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases, with SDK baseline and public plugin docs updated together.
