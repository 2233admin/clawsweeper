## Summary

Keep this PR open. Current main still does not implement the requested automatic 20s cap for broad `find $HOME` or `find /` exec scans, and the matching system-prompt guidance is also absent. The PR remains a focused implementation candidate, but it needs maintainer review/changes because existing review comments identify parser gaps in a security-sensitive shell execution path.

## Best Possible Solution

Keep this PR open and request changes before merge. The best path is to rebase onto current main, implement a shared broad-`find` timeout classifier used by local exec, gateway approval follow-up, and node execution planning, parse all `find` start roots, resolve relative roots against command `workdir`, support `~`, `$HOME`, and `${HOME}`, preserve explicit timeout and real traversal-limiting escape hatches only when they apply to the actual `find`, and add focused regression tests plus prompt/docs guidance.

## Risks / Open Questions

- Current main can still let broad `$HOME` or `/` discovery scans consume the configured default exec timeout when the agent omits an explicit timeout.
- Merging the PR without the parser fixes noted in review could create inconsistent or bypassable timeout behavior in a security-sensitive exec path.
- The final implementation should stay consistent across local exec, gateway approval follow-up, and node `system.run`.
