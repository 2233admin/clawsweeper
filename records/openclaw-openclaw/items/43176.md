## Summary

Keep this PR open. Current main still does not implement the requested 20s cap or prompt guidance for broad `$HOME`/`/` `find` scans, and the PR remains a focused implementation candidate. It needs review changes before merge because the patch touches exec timeout behavior and still has parser/route coverage gaps called out in review.

## Best Possible Solution

Keep this PR open and request changes before merge. The best path is to rebase onto current main, factor a shared broad-`find` timeout classifier used by local exec, gateway approval follow-up, and node execution planning, parse all `find` start roots, resolve relative roots against command `workdir`, support `~`, `$HOME`, and `${HOME}`, preserve explicit timeout and real traversal-limiting escape hatches only when they apply to the actual `find`, and add focused regression tests for local, gateway, node, relative-workdir, multiple-root, and prompt-guidance behavior.
