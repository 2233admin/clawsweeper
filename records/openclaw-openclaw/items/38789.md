## Summary

Keep this PR open. Current main at 7e5c3753 still has the sandbox skill prompt gap this PR targets: non-rw sandbox runs switch to the sandbox workspace and sync skills there, but attempt and compaction still skip live sandbox skill loading when a resolved snapshot exists and resolveSkillsPromptForRun still returns the host-built snapshot prompt first. The PR also has active maintainer repair context from vincentkoc and remains the focused implementation path for open bug #43383.

## Best Possible Solution

Keep this PR open and review or land the repaired branch, or replace it with an equivalent focused agents fix. The final implementation should force-load sandbox-local, config-eligible skill entries for non-rw attempt and compaction runs; rebuild prompts from prompt-visible sandbox entries instead of host snapshots; avoid empty-entry fallback to host prompts; preserve snapshot skill filters and remote eligibility; align env overrides with the prompt-visible entries; and keep the regression coverage. After merge, #43383 can close as implemented.
