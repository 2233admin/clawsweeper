## Summary

Keep this PR open. Current main still has the sandbox skill prompt gap this PR targets: non-rw sandbox runs switch to the sandbox workspace, but resolved skill snapshots still prevent live sandbox entry loading and the prompt helper still prefers the saved host-side snapshot prompt. The PR is also active after a maintainer repair push, with the remaining work tied to open bug #43383 rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and review or land the repaired branch, or replace it with an equivalent focused agents fix. The final implementation should force-load config-eligible sandbox skill entries for non-rw attempt and compaction runs, rebuild prompts from sandbox entries instead of host snapshots, avoid falling back to host snapshot prompts when sandbox entries are empty, preserve snapshot skill filters and remote eligibility, align env overrides with the prompt-visible skills, and keep the focused regression coverage.
