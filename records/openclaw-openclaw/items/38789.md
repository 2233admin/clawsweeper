## Summary

Keep this PR open. Current main still has the sandbox skill prompt behavior this PR targets: non-rw sandbox runs switch to the sandbox workspace and sync eligible skills there, but saved snapshots still prevent live sandbox skill loading and the saved snapshot prompt still wins. The PR also remains the active implementation candidate for open bug #43383, with recent maintainer repair comments and merge-ready labels.

## Best Possible Solution

Keep this PR open and review or land the repaired branch, or replace it with an equivalent focused agents fix. The final implementation should force-load config-eligible sandbox skill entries for non-rw attempt and compaction runs, rebuild prompts from sandbox entries instead of host snapshots, avoid falling back to host snapshot prompts when sandbox entries are empty, preserve snapshot skill filters and remote eligibility, align env overrides with the prompt-visible skills, and keep focused regression coverage. After that lands, #43383 can close as fixed.
