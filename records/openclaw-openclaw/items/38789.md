## Summary

Keep this PR open. Current main still has the sandbox skill prompt bug the PR targets: non-rw sandbox runs switch to the sandbox workspace, but resolved skill snapshots still suppress live sandbox entry loading and `resolveSkillsPromptForRun` still returns the saved host-side snapshot prompt before considering sandbox entries. The PR is active, has a maintainer repair push, and is tied to open bug #43383, so the best cleanup path is review/land this branch or an equivalent focused fix rather than closing it as obsolete.

## Best Possible Solution

Keep this PR open and review or land the repaired branch, or replace it with an equivalent focused agents fix. The final implementation should force-load config-eligible sandbox skill entries for non-rw attempt and compaction runs, rebuild prompts from sandbox entries instead of host snapshots, avoid falling back to host snapshot prompts when sandbox entries are empty, preserve snapshot skill filters and remote eligibility, align env overrides with the prompt-visible skills, and keep the focused regression coverage.
