## Summary

Keep this PR open. Current main still has the reported artifact gap: the gateway watch build profile runs `runtime-postbuild` but does not run the standalone hook-metadata copier, while hook discovery needs `HOOK.md` beside the built handler before a bundled hook can load. The PR is a narrow fix with focused tests and no blocking review findings; it may need a small rebase adjustment to preserve the newer runtime-postbuild phase wrapper on current main.

## What This Changes

The PR adds a runtime postbuild helper that copies bundled hook `HOOK.md` files from `src/hooks/bundled` into `dist/bundled`, calls it from `runRuntimePostBuild`, adds focused runtime-postbuild tests, and records the fix in the changelog.

## Best Possible Solution

Land this PR or an equivalent narrow fix that copies bundled hook manifests during runtime postbuild, with the call adapted to current main's phase-timed `runRuntimePostBuild`. Treat the author’s follow-up concern about silently enabled hooks failing to load as a separate diagnostics/logging improvement unless a maintainer wants to expand scope.

## Reproduction Assessment

Yes. The static reproduction path is high-confidence: the watch profile skips `copy-hook-metadata`, tsdown emits `dist/bundled/<hook>/handler.js`, and hook discovery returns no entry without `HOOK.md`; the author also reported manual before/after validation.

## Solution Assessment

Yes. Folding the existing source-to-dist `HOOK.md` copy into `runRuntimePostBuild` is the narrowest maintainable fix because `gateway:watch` already invokes that postbuild path; the only adaptation needed is to preserve current main's runtime-postbuild phase wrapper.
