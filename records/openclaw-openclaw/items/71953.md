## Summary

Keep this PR open. Current main still has the reported build artifact gap: `gatewayWatch` runs `runtime-postbuild` but skips the standalone hook metadata copier, while bundled hook discovery needs `HOOK.md` before it can load a handler. The PR remains a narrow, relevant fix, with the main remaining maintainer step being rebase/reconciliation with the newer phase-timed runtime postbuild wrapper.

## What This Changes

The PR adds runtime postbuild copying for bundled hook `HOOK.md` files from `src/hooks/bundled` into `dist/bundled`, adds focused tests, and records the fix in the changelog.

## Best Possible Solution

Land this PR or an equivalent narrow fix that copies bundled hook manifests during runtime postbuild, adapted to current main's `runPhase` structure. Keep the author’s follow-up concern about silently enabled hooks failing to load as a separate diagnostics/logging improvement unless a maintainer chooses to expand scope.

## Reproduction Assessment

Yes. The static reproduction path is high-confidence: `gatewayWatch` skips `copy-hook-metadata`, tsdown emits bundled hook handlers under `dist/bundled`, and hook discovery returns no entry when `HOOK.md` is absent. The author also reported manual before/after validation.

## Solution Assessment

Yes. Folding the existing source-to-dist `HOOK.md` copy into `runRuntimePostBuild` is the narrowest maintainable fix because `pnpm gateway:watch` already invokes that postbuild path; it only needs rebase adaptation to the current phase wrapper.
