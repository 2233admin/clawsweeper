## Summary

Keep this PR open. Current main still does not implement the PR's intended source-tagged internal prompt hiding: heartbeat/cron/exec-event turns are not marked with `InputProvenance.kind = "internal_system"`, gateway chat history does not drop user messages by that provenance, and the Control UI defense-in-depth filter does not inspect provenance. The PR remains an active implementation candidate for the linked internal-prompt leak class, with the source-tag pivot matching the safer direction raised in earlier review comments.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate, or land an equivalent narrow fix: tag heartbeat/cron/exec-event internal turns with `InputProvenance.kind = "internal_system"` at trusted origins, persist that through the existing session-manager path, hide those user-role messages in shared gateway history projection and Control UI defense-in-depth, and add regressions for full history snapshots, SSE appends, and normal human messages without that provenance.
