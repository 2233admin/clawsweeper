## Risks / Open Questions

- The PR diff appears to remove existing protocol/API surfaces (`plugins.uiDescriptors`, `sessions.pluginPatch`, `NodePresenceAlive*`, `NodeEventResult`) while adding memory RPCs. That is outside the stated memory-management scope and could break Control UI, plugin-host, or native client compatibility if not rebased or explained.
- The new `memory.source.open` RPC is intended to gate file reads through server-issued `sourceRef`; it should receive focused security review around sourceRef lifetime/scope, manager-provided paths, and same-agent session sharing before merge.
- The PR is stacked on #73769, #73770, and #73771, so the useful review path depends on narrowing/rebasing after prerequisite memory changes rather than closing this PR as obsolete.
