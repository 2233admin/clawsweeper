## Summary

Keep open. Current main still lacks `models status --probe --all`, and the PR is a relevant implementation for the linked per-model health-check feature, but it should be refreshed before merge because the exported probe summary type still omits `probeAll` and the public CLI docs do not describe the new flag.

## What This Changes

The PR adds `--all` plumbing for `openclaw models status --probe`, expands probe targets to every configured candidate model, warns when `--all` is used without `--probe`, and adds focused route/parser and probe-target tests.

## Best Possible Solution

Land a refreshed version of this PR after aligning the exported probe summary type and public CLI docs with the new flag, while preserving the existing one-model-per-provider behavior when `--all` is omitted.
