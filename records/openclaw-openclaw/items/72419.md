## Summary

Keep open for maintainer/security review. Current `main` does not implement this PR's `bootstrap-token` backend self-pairing bypass, and the PR is paired with open issue #72418 by closing syntax. The proposed diff also leaves the shared token/password backend skip intact, which is the core behavior challenged by the linked security report and is currently documented and tested behavior.

## What This Changes

The PR changes the gateway WebSocket handshake helper so direct-local backend clients authenticated with `bootstrap-token` can skip backend self-pairing, with focused test and comment updates in the handshake auth helper files.

## Best Possible Solution

Keep the PR open until maintainers decide the intended security model for backend self-pairing. A final fix should either bind the backend exception to a server-verifiable backend credential/identity and update tests/docs accordingly, or explicitly document that the shared-token loopback backend exception is accepted and adjust this PR's bootstrap-token change to match that policy with current-main locality coverage.
