## Summary

Keep open. Current main still embeds BOOT.md as ordinary prompt text, sends that same body through model fallback attempts, and has no boot-run echo guard before final reply handling or message-tool delivery. The linked PR #55016 is closed unmerged and addressed boot outbound delivery, not literal BOOT.md echo suppression.

## What This Changes

The issue reports that fallback model retries can expose BOOT.md startup instructions verbatim instead of silently executing them.

## Best Possible Solution

Add a narrow internal boot-run context that carries enough prompt identity into agent execution and tool delivery to suppress substantial literal echoes of the generated BOOT.md prompt, while preserving intentional BOOT.md-requested outbound messages and existing fallback retry semantics.
