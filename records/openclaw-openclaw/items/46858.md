## Summary

Keep this PR open. Current main still has the heartbeat responsePrefix divergence described by the PR and linked issue #43064: heartbeat runs read the configured prefix as a raw string, do not capture selected model/provider context, and pass that raw value into HEARTBEAT_OK handling and outbound heartbeat reply normalization. The PR remains a plausible focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open as an implementation candidate for #43064. The best maintainer path is to refresh or rebase the focused heartbeat-runner change, wire heartbeat runs into the existing onModelSelected response-prefix context, resolve the prefix before heartbeat normalization and OK delivery, and keep regression coverage for templated alert replies plus prefixed HEARTBEAT_OK suppression.
