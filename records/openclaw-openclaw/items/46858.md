## Summary

Keep open. Current main still treats heartbeat `messages.responsePrefix` as the raw configured string, while normal reply delivery already supports template interpolation through `onModelSelected` and `resolveResponsePrefixTemplate()`. The linked bug remains open, and this PR is still a plausible focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open as an implementation candidate for #43064. The best maintainer path is to rebase or refresh the focused heartbeat-runner change, wire heartbeat runs into the existing `onModelSelected` response-prefix context, resolve the prefix template before ack suppression and alert delivery, and keep regression coverage for templated heartbeat alert replies plus prefixed `HEARTBEAT_OK` suppression.
