## Summary

Keep open. Current main fixes a related manual allowlist problem, but it does not implement the issue's explicit `safeBins` behavior for compound commands. `isSafeBinUsage()` still matches only the executable name, requires a trusted path, and then requires a safe-bin profile; multi-token entries like `gh pr list` do not participate in matching. Docs and tests now frame `safeBins` as a narrow stdin-only fast path, while the exact compound-safeBins change is only represented by the still-open PR #70571.

## Best Possible Solution

Keep #59600 open for maintainer judgment. If the intended product direction is to support exact compound command trust, refine or review the open #70571 approach with focused security tests. If maintainers instead want the current contract, close manually with docs-backed guidance that `gh`, `curl`, and `python3 -c` belong in exec approvals allowlist entries, not `tools.exec.safeBins`.
