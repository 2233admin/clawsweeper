## Summary

Keep open. Current main partially addresses this bug family by suppressing commentary-phase output and stripping exact `NO_REPLY` text from media-carrying payloads, but it still does not provide a shared outbound guard for raw `to=functions.*`/tool-envelope text or the later `REPLY_SKIP`/`ANNOUNCE_SKIP` control-token variants reported in the discussion.

## What This Changes

The issue requests that Discord and shared outbound delivery stop sending internal control tokens and model tool-call routing artifacts to user-visible channels.

## Best Possible Solution

Add a tested shared outbound/user-facing sanitizer for internal control tokens and model-emitted tool-routing envelopes, wired through normal channel delivery so Discord cannot expose these artifacts while preserving legitimate content and media-only silent replies.
