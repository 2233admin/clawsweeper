## Summary

Keep open. Current main covers the commentary-phase subset and exact `NO_REPLY` normalization better than the original report, but it still lacks a shared outbound/Discord sanitizer for the broader leak family: plain `to=functions.*` routing text, envelope field names, raw tool JSON, and `REPLY_SKIP`/`ANNOUNCE_SKIP`-style control tokens.

## What This Changes

The issue requests that Discord and shared outbound delivery strip or suppress internal tool-call traces, commentary markers, raw tool JSON, and silent/control tokens before messages reach user-visible channels.

## Best Possible Solution

Add a conservative delivery-boundary sanitizer for internal trace/control artifacts, with focused Discord send backstop coverage if needed, while preserving legitimate prose, media-only replies, code fences, channel formatting, and existing exact silent-token policy semantics.
