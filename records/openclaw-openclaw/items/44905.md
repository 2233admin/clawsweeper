## Summary

Keep open. Current main has fixed the commentary-phase subset and the exact media-caption NO_REPLY gap, but the Discord/shared outbound path still does not have a delivery-boundary guard for raw tool-routing lines, envelope field names, generic tool-call JSON, or REPLY_SKIP/ANNOUNCE_SKIP-style control tokens.

## What This Changes

The issue requests suppression of Discord-visible internal tool-call traces, silent/control tokens, and raw tool-routing envelope text before channel delivery.

## Best Possible Solution

Add a conservative shared delivery-boundary sanitizer for internal trace/control artifacts, with a Discord send-level backstop for bypass paths and regression coverage that preserves legitimate prose, code fences, Discord formatting, media-only sends, and the existing silent-reply policy.

## Reproduction Assessment

Yes. A focused static reproduction is a Discord outbound payload containing raw tool-routing text such as `to=functions.exec` or a small tool-call JSON envelope; current main shows that payload reaches the shared delivery and Discord send paths without a matching sanitizer or final guard.

## Solution Assessment

No. Current main is only a partial solution; the narrow maintainable direction is a shared outbound filter plus a Discord backstop after maintainer review of the exact drop/strip policy.
