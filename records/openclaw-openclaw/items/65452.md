## Summary

Keep open. Current main still exposes unquoted SQLite table identifier interpolation through the memory host storage SDK, so the proposed security fix is not obsolete. It needs maintainer/security review and iteration because the live implementation has since moved under `packages/`, the branch lacks focused regression tests, and the proposed dot-splitting table quoting contract may break existing pre-quoted or dotted identifiers.

## What This Changes

The PR adds SQL identifier quoting helpers to the memory schema management files and applies them to dynamic table and column identifiers in SQLite DDL.

## Best Possible Solution

Land a revised memory-schema hardening change that defines the accepted SQLite identifier contract, quotes or validates dynamic identifiers in the live package implementation, keeps the `src/` wrapper and SDK surface aligned, preserves schema-qualified behavior or rejects unsupported forms explicitly, and adds regression tests for malicious and compatibility-sensitive identifiers.
