## Summary

Current main still leaves the reported LINE multi-account webhook path broken. `channels.line.accounts.default` is still ignored for the default LINE account, configured named LINE accounts still default to disabled when `enabled` is omitted, and gateway startup returns before LINE can call `monitorLineProvider` and register webhook routes. The related fix PR #47528 is closed and unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open and implement the #47528 direction on current main: resolve the default LINE account from `channels.line.accounts.default`, treat configured named LINE accounts as enabled unless explicitly disabled while still respecting top-level `channels.line.enabled: false`, and add focused regression coverage for `accounts.default` credentials plus a named account with a custom `webhookPath` reaching route registration.
