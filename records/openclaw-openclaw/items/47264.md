## Summary

Current main does not yet fix the reported LINE multi-account shape. The related fix PR was closed unmerged, and the current resolver still ignores `channels.line.accounts.default` for the default account while leaving named LINE accounts disabled unless they explicitly set `enabled: true`, which prevents webhook route registration for the reported config.

## Best Possible Solution

Keep this issue open and implement the #47528 direction on current main: make `resolveLineAccount` read `channels.line.accounts.default` for the default account, treat configured named LINE accounts as enabled unless explicitly disabled while respecting top-level channel disable, and add regression coverage for `accounts.default` plus a named account with a custom `webhookPath`. A targeted LINE account/startup test should prove both accounts reach route registration.
