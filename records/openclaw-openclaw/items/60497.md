## Summary

Keep open. Current main still suppresses ordinary exec/bash failures at default verbosity before the mutating-tool warning path runs, so a success-looking assistant answer can remain the only user-visible output after a command-not-found failure. The related NemoClaw sandbox symlink fix and #60507 do not implement the remaining OpenClaw fail-closed payload behavior.

## What This Changes

The issue requests that embedded agent runs report failed exec/bash commands, especially command-not-found, instead of delivering fabricated success text or results.

## Best Possible Solution

Add a narrow embedded-run payload fix that surfaces concise exec/bash failure warnings at default verbosity for command-not-found and other failed mutating exec actions, and ensures success-looking assistant text is not the sole delivered result after the matching exec action failed. Preserve explicit heartbeat suppression and verbose-detail behavior.
