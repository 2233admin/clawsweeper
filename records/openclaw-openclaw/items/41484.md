## Summary

Keep this issue open. Current main still has provider/sender-based elevated allowlists and partial cron/heartbeat mitigations, but it does not implement the requested per-job cron elevated flag or heartbeat provenance/explicit elevated marker.

## Best Possible Solution

Keep this open for maintainer/secops design. The best path is an additive trust model: add a persisted cron job elevated/trust field, gate setting or changing it on owner/elevated-approved senders, thread it through cron types, normalization, storage, Gateway schemas, CLI, Control UI, the agent cron tool, docs, and tests, and add heartbeat provenance or an explicit heartbeat elevated marker so HEARTBEAT.md-driven runs do not rely on broad cron-event or heartbeat wildcard allowlists.
