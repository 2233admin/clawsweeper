## Summary

Keep this issue open. Current main has DeepSeek V4 replay support and a newer unreleased blank-backfill fix for plain assistant messages, but the reported long multi-turn tool-call/subagent failure is still not proven fixed: the active wrapper still fills missing assistant reasoning with an empty string rather than recovering real prior reasoning, and the current tests cover short/direct replay cases rather than the reported chained sessions_spawn flow.

## What This Changes

The issue requests a fix for DeepSeek V4 Pro/Flash high-thinking multi-turn tool-call and sessions_spawn flows that fail with 400 errors because required reasoning_content is not replayed correctly.

## Best Possible Solution

Implement provider-level DeepSeek V4 replay recovery that preserves real non-empty reasoning_content across chained assistant/tool turns and only falls back to blank placeholders when there is no native DeepSeek reasoning to replay. Add deterministic multi-tool-chain coverage plus a sessions_spawn-style regression, and keep the docs aligned with the actual supported replay contract.
