## Summary

Keep open for maintainer review. Current main still has the silent group/channel fallback with no runtime warning, while this PR is a narrow one-file warning slice for the broader linked regression and I did not find a concrete blocking bug in the patch.

## What This Changes

The PR adds an auto-reply subsystem logger and a module-level one-shot warning in `resolveSourceReplyDeliveryMode` when group/channel `visibleReplies` config is absent before falling back to `message_tool_only`.

## Best Possible Solution

Land the resolver-level runtime warning if maintainers accept the wording and CI proof, while keeping the linked broader issue open or split for the remaining documentation, config-editing, and migration-policy work.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to call `resolveSourceReplyDeliveryMode` with empty config and `ChatType` set to `group` or `channel`; current tests and dispatch coverage show it becomes `message_tool_only` and suppresses the automatic final reply.

## Solution Assessment

Yes for this PR's warning slice. Placing the one-shot warning in the shared resolver is narrow, channel-agnostic, and fires only when the affected fallback is actually used; it is not a complete solution for the broader linked issue.
