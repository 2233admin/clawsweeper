## What This Changes

This PR adds a one-shot runtime warning in the source reply delivery resolver when group/channel visibleReplies is unset and the resolver falls back to message-tool-only delivery.

## Best Possible Solution

Land the narrow runtime warning after adjusting the wording to say the config can be saved/updated and only needs a manual restart when reload is disabled, then keep or split any remaining linked-issue work that is not covered by the warning.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: call the resolver or dispatch path with empty config and ChatType of group/channel; current tests show the mode becomes message_tool_only and no normal final reply is sent until messages.groupChat.visibleReplies is automatic.

## Solution Assessment

Mostly yes. A resolver-level one-shot warning is a narrow maintainable mitigation because it fires only when the affected runtime path is reached, but the text should match the supported reload contract and a small warning assertion would make the behavior durable.
