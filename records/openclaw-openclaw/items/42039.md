## Summary

Keep this issue open. Current main still has `maxMissedJobsPerRestart` and `missedJobStaggerMs` only as internal cron service dependencies; the public `CronConfig`, strict zod schema, generated config schema/help metadata, gateway construction, and docs do not expose them for `openclaw.json`. The exact implementation PR #42069 is closed and unmerged, #41332 is only a partial adjacent stagger proposal, and #73643 overlaps without superseding this older, more specific tracker.

Keep this issue open as the canonical specific tracker for public cron missed-job catch-up controls. A complete fix should expose backward-compatible config under `cron`, validate it in the strict schema, generate schema/help metadata, wire it into `CronService` construction, document it in cron/config docs, add a changelog entry with reporter credit, and add regression tests for config acceptance, `maxMissedJobsPerRestart: 0`, stagger timing, and startup/interrupted-run behavior while reconciling #41332 and #73643.
