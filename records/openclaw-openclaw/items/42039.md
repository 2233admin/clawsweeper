## Summary

Keep this issue open. Current main still has `maxMissedJobsPerRestart` and `missedJobStaggerMs` only as internal cron service dependency knobs, while the public cron config type, strict config schema, generated schema metadata, gateway wiring, job protocol surface, and cron docs do not expose the requested controls or an equivalent `skipMissedRuns`/`skipCatchUp` option. The related implementation PR #42069 is closed and unmerged, and related issue #48345 is also closed, so there is no active canonical replacement visible in the provided context.

## Best Possible Solution

Implement a narrow public cron catch-up configuration surface. Add either `maxMissedJobsPerRestart` plus `missedJobStaggerMs`, or a clearer `skipMissedRuns`/catch-up policy, to `CronConfig`, zod validation, generated config metadata/help/labels, gateway passthrough into `CronServiceDeps`, and cron docs. Preserve current defaults for compatibility, define `0`/disabled semantics explicitly, and add focused tests for config acceptance, gateway wiring, startup catch-up limits, stagger timing, and skipped catch-up behavior. Consider a per-job control only if maintainers want announcement jobs and monitoring jobs to diverge.
