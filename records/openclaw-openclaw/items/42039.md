## Summary

Keep open. Current main still has `maxMissedJobsPerRestart` and `missedJobStaggerMs` only as internal `CronServiceDeps` controls; they are not exposed through public cron config, strict validation, generated schema/help metadata, docs, or gateway wiring. The related implementation PR #42069 is closed and unmerged, and related issue #48345 is also closed, so there is no active canonical replacement.

## Best Possible Solution

Implement a narrow, documented cron catch-up configuration surface in core. Add the selected fields to `CronConfig`, zod validation, generated schema/help metadata, gateway `CronService` wiring, and public cron/config docs; preserve current defaults and add focused tests for config acceptance, gateway passthrough, startup catch-up limits, stagger timing, and disabled or skipped catch-up behavior.
