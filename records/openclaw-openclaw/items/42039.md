## Summary

Keep this issue open. Current main still has the missed-job catch-up controls only as internal CronService dependency knobs, while public config types, strict zod validation, generated config metadata, gateway wiring, and user docs still omit `maxMissedJobsPerRestart`, `missedJobStaggerMs`, `skipMissedRuns`, and `skipCatchUp`. The related implementation PR #42069 is closed and unmerged, and #48345 is closed, so there is no active canonical replacement visible in the provided context.

## Best Possible Solution

Implement a narrow public cron catch-up configuration surface: add either `maxMissedJobsPerRestart` and `missedJobStaggerMs`, or a clearer `skipMissedRuns`/catch-up control, to `CronConfig`, zod validation, generated schema/help/labels, and cron/gateway docs; wire accepted values from gateway config into `CronService`; define disabled catch-up semantics explicitly; and add focused tests for config acceptance, gateway passthrough, default compatibility, stagger timing, and skipped catch-up behavior.

## Risks / Open Questions

- The desired semantics for `maxMissedJobsPerRestart: 0` still need product definition: skip overdue runs, defer them, or advance recurring jobs to the next future schedule.
- Changing default startup catch-up behavior could regress users who rely on missed jobs running after sleep or downtime, so the conservative path should preserve current defaults and add explicit opt-out/tuning controls.
