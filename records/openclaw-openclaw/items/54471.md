## Summary

Keep this PR open. Current main still handles ACP parent relay output only for `assistant` and `lifecycle` streams, so the proposed `system_event` relay is not implemented on main. The PR is narrowly scoped and security-neutral, but it needs maintainer review/update before landing because the current patch lacks a regression test and does not count relayed `system_event` text as progress for stall detection.

## Best Possible Solution

Keep the PR open and update it rather than closing it. The best landing shape is a narrow change in `src/agents/acp-spawn-parent-stream.ts` that relays non-empty `system_event` text to the parent session, treats it as relay progress by updating the stall timer/resume state, and adds focused coverage in `src/agents/acp-spawn-parent-stream.test.ts`. Because this is user-facing ACP behavior, include a concise changelog fix entry with contributor attribution if the PR is merged.
