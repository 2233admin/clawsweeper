## Summary

Keep open. The PR targets a real ACP parent-relay gap, but the current ACP turn path drops runtime `status` events before the parent relay and the proposed patch only listens for a new AgentEvent stream named `system_event`; it also does not count relayed system text as progress, so it can still produce false no-output notices. The branch needs an in-place update and focused coverage before merge.

## What This Changes

The PR adds a `system_event` branch to `startAcpSpawnParentStreamRelay()` so non-empty system-event text can be emitted to the parent session with a fallback ACP context key.

## Best Possible Solution

Update ACP event plumbing so runtime status/system progress is explicitly translated into parent-relay system events, count every visible progress relay in the stall/resume state, preserve existing assistant/commentary/lifecycle behavior, and add focused tests plus a user-facing changelog entry if merged.
