## Summary

Keep this PR open. The reported bug is still valid, but current main does not implement the PR's intended fix: the active memory host SDK helper still catches and ignores real mkdirSync failures, while this PR edits an obsolete path that is not present on current main.

## Best Possible Solution

Keep the PR open and ask for a focused rebase onto current main: remove the swallowed mkdirSync failure from packages/memory-host-sdk/src/host/internal.ts, keep src/memory-host-sdk/host/internal.ts coherent if that copy remains maintained, and add a narrow Vitest regression proving real mkdir failures such as EACCES, ENOSPC, or EROFS propagate immediately from ensureDir or the memory database open path.
