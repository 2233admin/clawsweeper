## Summary

Keep open. Current main can display configured agent avatars through the authenticated `/avatar/<agentId>` route and has a browser-local Quick Settings assistant-avatar override, but it still lacks the requested Agents-tab upload flow that persists image bytes under the agent workspace and updates `IDENTITY.md`. A closed unmerged PR and the attached gist show useful prior implementation direction, but there is no open paired fix PR and no current-main implementation to close against.

## What This Changes

The issue requests an Agents-tab avatar picker that uploads a local image, stores it under the agent workspace, updates `IDENTITY.md`, and renders it through `/avatar/<agentId>`.

## Best Possible Solution

Add an additive admin-scoped gateway method or authenticated endpoint with typed protocol schema, strict image validation, bounded storage under the selected agent workspace, identity updates through existing helpers, Agents-tab picker/refresh wiring, docs for the persistent workflow, and focused gateway/UI regression coverage.
