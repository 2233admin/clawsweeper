## Summary

Keep open. Current main still filters tools without preserving matched policy layer or rule attribution for later audit logging, and the embedded and HTTP tool-call surfaces still lack the requested policy decision details. The open implementation candidate #55826 explicitly targets this issue and is unmerged, so the issue should stay open for maintainer review rather than being closed or queued as duplicate automated repair work.

## What This Changes

The issue requests audit logging that records each tool policy allow/deny decision, the matched configuration layer, and the matched rule when a tool call is evaluated.

## Best Possible Solution

Land a focused patch, either by updating #55826 or replacing it, that carries matched policy layer and matched rule information through policy evaluation into embedded tool-call and HTTP `/tools/invoke` audit logs, with targeted tests for the pipeline and both consumer surfaces.
