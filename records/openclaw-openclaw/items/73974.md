## Summary

Keep open. Current main still lacks the Mac app LaunchAgent ownership guard for macOS doctor service repair, and this PR targets the still-open #67631 report. The PR is not obsolete, but it needs revision before merge because review found a real `OPENCLAW_PROFILE=mac` label-collision bug in the proposed ownership check.

## Best Possible Solution

Keep this PR open and revise it in place: disambiguate real OpenClaw.app ownership from a CLI `OPENCLAW_PROFILE=mac` LaunchAgent, likely by checking the app plist `ProgramArguments` or another app-specific marker instead of label alone. Preserve the default-profile behavior, add explicit regression tests for `OPENCLAW_PROFILE=mac` and unrelated profiles, keep the #67631/#42573 attribution, and run `pnpm check:changed` before maintainer review.
