## Summary

Current main still has the reported Control UI bug: `timeoutSeconds: 0` is rejected by form validation and would also be omitted from the saved payload. The backend/protocol side already accepts and preserves zero as no-timeout, and there is still an open same-author implementation candidate, #41275, with closing syntax, so this issue should stay open until that PR or an equivalent complete fix lands.

## Best Possible Solution

Keep this issue open and review #41275 or an equivalent complete patch. The fix should allow zero in `validateCronForm`, preserve explicit `timeoutSeconds: 0` in `buildCronPayload`, update Control UI regression tests, and align the source English timeout help/error text with the gateway/runtime contract.
