## Summary

Current main still reproduces the reported Control UI mismatch: the cron form rejects `timeoutSeconds: 0`, while the gateway schema and cron runtime already allow zero as no-timeout. The issue also has an open same-author implementation candidate, #41275, with closing syntax, so this issue should stay open until that PR or an equivalent complete fix lands.

## Best Possible Solution

Keep this issue open and review #41275 or an equivalent complete patch. The fix should allow zero in `validateCronForm`, preserve explicit `timeoutSeconds: 0` in `buildCronPayload`, update Control UI regression tests, and align the source English timeout help/error text with the gateway contract.
