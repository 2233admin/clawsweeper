## Summary

Keep this issue open. Current main still rejects `timeoutSeconds: 0` in the Control UI and drops explicit zero from the submit payload, while the gateway protocol/runtime side already accepts zero as no-timeout. Same-author PR #41275 remains open and unmerged with closing syntax, so this issue should not be closed before that PR or an equivalent complete fix lands.

## Best Possible Solution

Keep this issue open and review, update, or replace PR #41275. A complete fix should allow zero in `validateCronForm`, preserve explicit `timeoutSeconds: 0` in `buildCronPayload`, update Control UI regression tests so zero is accepted and negative values are rejected, and align timeout help/error copy with the gateway/runtime no-timeout contract. No core protocol change appears necessary.
