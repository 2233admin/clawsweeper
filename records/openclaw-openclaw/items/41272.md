## Summary

Current main still has the reported cron Control UI mismatch: the form rejects `timeoutSeconds: 0`, payload construction omits an explicit zero, and the existing UI test still expects zero to be invalid. Backend/protocol cron paths already accept and preserve zero as no-timeout, and open PR #41275 still uses closing syntax for this issue, so this should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #41275 or an equivalent maintainer patch. The fix should update `validateCronForm` to accept an explicit `0` while still rejecting negative and malformed values, update `buildCronPayload` to preserve an explicitly entered zero, and add/update UI controller tests for zero, negative, malformed, empty, and positive timeout inputs.
