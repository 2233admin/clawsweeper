## Summary

Current main still has the reported Control UI bug. `timeoutSeconds: 0` is rejected by `validateCronForm`, explicit zero is dropped by `buildCronPayload`, and the UI test still encodes zero as invalid while the gateway/runtime contract accepts zero as no-timeout. The same-author PR #41275 remains open with closing syntax, so the issue should stay open until that PR or an equivalent complete fix lands.

## Best Possible Solution

Keep this issue open and review #41275 or an equivalent complete patch. The fix should allow zero in `validateCronForm`, preserve explicit `timeoutSeconds: 0` in `buildCronPayload`, update Control UI regression tests, and align the English timeout help/error text with the gateway/runtime no-timeout contract.
