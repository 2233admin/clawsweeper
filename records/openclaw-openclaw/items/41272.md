## Summary

Current main still has the reported Control UI mismatch: the cron form rejects `timeoutSeconds: 0`, drops an explicit zero from the payload path, and the UI regression test still expects zero to be invalid. Backend/protocol cron paths already accept and preserve zero as no-timeout, and open PR #41275 still references this issue with closing syntax, so this should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #41275 or an equivalent maintainer patch. The fix should update `validateCronForm` to accept explicit zero while still rejecting negative and malformed values, update `buildCronPayload` to preserve an explicitly entered zero, and adjust/add UI controller tests for zero, negative, malformed, empty, and positive timeout inputs. Close this only after the paired fix merges.
