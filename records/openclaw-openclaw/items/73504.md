## Summary

Keep open. Current main partially overlaps via #73521, but it still does not expose Discord action execution mode through the registered Discord plugin action wrapper. Because the shared runner defaults missing execution mode to local, the PR’s wrapper-forwarding change remains relevant for the CLI read/search gateway-routing bug.

## Best Possible Solution

Keep this PR open, but ask for a rebase onto current main. The best fix is to retain main’s read/search-only resolveExecutionMode in extensions/discord/src/channel-actions.ts, add the wrapper forwarding in extensions/discord/src/channel.ts, and add a focused contract or plugin-surface test asserting discordPlugin.actions.resolveExecutionMode returns gateway for read/search.
