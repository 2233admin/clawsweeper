## What This Changes

The PR adds model-catalog provider aliases to provider ownership resolution, avoids caching provider hook misses while provider loading is in flight, and adds regression coverage for both paths.

## Best Possible Solution

Land the narrow provider-runtime fix after adding a single-line Unreleased Fixes changelog entry, preserving manifest-first alias ownership and avoiding a return to broad runtime hook scans.

Yes. ## Solution Assessment

Yes, with the changelog repaired. Fixing ownership in `resolveOwningPluginIdsForProvider` and suppressing cache writes only for in-flight hook resolution is narrower and more maintainable than broad-loading provider hooks for every runtime call.
