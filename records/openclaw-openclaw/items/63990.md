## Summary

Keep this issue open. Current main still resolves memory search to one provider/model/fallback/store per agent, and embedding-provider fallback still switches provider and forces a full reindex rather than routing reads between already-built per-model indexes. The closed broader related request #55590 does not supersede this issue's stricter no-mixed-vector-spaces, per-index ingestion, active/fallback index, and observability design.

## What This Changes

Add first-class multi-index embedding memory with per-model indexes, active/fallback index routing, independent ingestion state, observability, and backward-compatible single-index config.

## Best Possible Solution

Design an additive memory contract that preserves current single-index behavior while adding explicit per-model index definitions, a canonical chunk source, per-index ingestion/backfill state, read routing to exactly one selected ready index with controlled failover, dimension/model validation, and CLI/status/metrics that show coverage, lag, and failover state.
