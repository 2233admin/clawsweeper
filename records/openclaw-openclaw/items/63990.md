## Summary

Keep open. Current main still exposes and resolves memory search as one provider/model/store per agent, and the existing embedding fallback switches the provider before rebuilding the same single index rather than routing reads between already-built per-model indexes. The closed broader request #55590 does not cover this issue's stricter per-index ingestion, active/fallback index routing, no mixed vector spaces, and observability requirements.

## What This Changes

Add first-class multi-index memory embeddings so each embedding model has its own index, coordinated ingestion state, explicit active/fallback index selection, and status/metrics without breaking existing single-index configs.

## Best Possible Solution

Design an additive memory contract that keeps existing single-index configs working while adding explicit per-model index definitions, canonical chunks, per-index ingestion/backfill state, dimension/model validation, runtime selection of exactly one ready index per query, controlled failover, and CLI/status/metrics showing coverage, lag, errors, and failover state.
