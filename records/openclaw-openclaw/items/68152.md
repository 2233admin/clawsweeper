## Summary

Keep this PR open. The proposed change is directionally correct and uses an existing translation key, but it only localizes one option in a two-option target selector, leaving non-English users with a mixed-language control and leaving the i18n repair incomplete.

## What This Changes

The PR replaces the hardcoded `Node` option label in the exec approvals target select with `t("nodes.binding.node")`.

## Best Possible Solution

Update the PR so the target selector localizes both options: keep the `Node` change, add a `nodes.binding.gateway` source key, regenerate generated locale bundles and i18n metadata through the Control UI i18n workflow, and add or adjust focused coverage if needed.

## Reproduction Assessment

Yes. On current main, `ui/src/ui/views/nodes-exec-approvals.ts:258` and `:259` are hardcoded English; with this PR applied, zh-CN would render the same select as `Gateway / 节点` because the `Gateway` sibling remains literal English and no `nodes.binding.gateway` key exists.

## Solution Assessment

No. The PR is a valid partial fix, but the narrow maintainable solution is to localize the whole two-option selector in one i18n workflow change so the generated locale set stays structurally aligned.
