## Summary

Keep this issue open. Current main still leaves the core user problem in place for non-Azure custom/OpenAI-compatible providers: new custom models are written with the hard-minimum 16,000-token context window and 4,096 max output tokens, and the configure/onboard setup paths still do not collect or display contextWindow/maxTokens metadata or warn that OpenClaw is using the conservative fallback. The open #43493 PR remains the clearest implementation candidate and is not present on current main.

## Best Possible Solution

Review and land #43493 or an equivalent main-branch change that keeps the fix scoped to custom-provider setup, lets users configure and see effective contextWindow/maxTokens metadata during interactive and non-interactive setup, warns when OpenClaw falls back to conservative defaults, updates docs/tests, and optionally improves runtime overflow copy for custom-provider fallback cases.
