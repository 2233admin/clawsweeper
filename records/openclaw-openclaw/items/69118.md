## Summary

Keep open. Current main still hashes `groupIntro` as part of `extraSystemPromptStatic`, while `groupIntro` is only present on the first group turn or re-intro turn. That means the reported turn-1 to turn-2 `extraSystemPromptHash` drift can still produce `invalidatedReason: "system-prompt"`. The related shipped fix only excludes per-message inbound metadata; it does not remove the one-shot group intro from the reuse hash.

## Best Possible Solution

Keep #69118 open and fix the CLI session reuse identity so one-shot group intro text cannot participate in `extraSystemPromptHash`. The narrow code path is `src/auto-reply/reply/get-reply-run.ts`: remove `groupIntro` from `extraSystemPromptStaticParts` or split stable vs one-turn behavioral prompt parts more explicitly. Add a regression test that simulates two consecutive group turns and asserts the second turn reuses the same CLI binding instead of returning `invalidatedReason: "system-prompt"`.
