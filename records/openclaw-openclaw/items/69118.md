## Summary

Keep open. Current main still builds `groupIntro` only for the first group turn or re-intro, but also includes that turn-scoped text in `extraSystemPromptStaticParts`; the CLI runner hashes that static string and `resolveCliSessionReuse` still resets on hash mismatch, so the reported turn-1 to turn-2 group-channel reset remains valid.

## What This Changes

Fix Claude CLI group-channel session continuity so first-turn-only group intro text does not change the extra system prompt hash used for CLI session reuse.

## Best Possible Solution

Land a narrow CLI-session reuse fix: keep group intro delivery for first group turns and re-intros, but exclude first-turn-only `groupIntro` from `extraSystemPromptStaticParts` so the stored CLI binding is reused on follow-up group turns. Add a regression test proving a group turn-1 to turn-2 path does not produce a system-prompt invalidation, and verify any Claude live-session fingerprinting stays aligned.
