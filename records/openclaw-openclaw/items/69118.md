## Summary

Keep open. Current `main` still builds `groupIntro` only for the first group turn or re-intro, but includes that same turn-scoped text in `extraSystemPromptStatic`, which is what the CLI session reuse hash uses. A normal group turn 1 to turn 2 can still change `extraSystemPromptHash` and hit `invalidatedReason: "system-prompt"`.

## Best Possible Solution

Keep this issue open and land a narrow CLI-session reuse fix: keep injecting `groupIntro` into the actual first-turn/reintro group system prompt, but exclude first-turn-only `groupIntro` from the stable prompt text used for `extraSystemPromptHash`. Add a regression test for a group session turn 1 to turn 2 path proving the stored CLI binding is reused and no `system-prompt` invalidation occurs. Separately inspect whether Claude live-session fingerprinting should align with the same stable prompt identity.
