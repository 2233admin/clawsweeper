## Summary

Keep this PR open. The linked production report still describes a real current-main Discord `thread-reply` attachment parity gap, but the proposed patch is not ready to merge because it misses part of the media-read contract and adds tests under an obsolete path.

## What This Changes

The PR updates Discord `thread-reply` attachment handling to accept `media`, `path`, and `filePath` with alias precedence, forwards media local roots into the action call, and adds parser tests for those aliases.

## Best Possible Solution

Land a current-main Discord extension fix that normalizes `media ?? path ?? filePath` for `thread-reply`, forwards both `mediaLocalRoots` and `mediaReadFile` from the shared action context into the Discord runtime, and covers alias precedence plus callback propagation in the current extension tests so #43570 can close through the merged fix.
