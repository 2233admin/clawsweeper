## Summary

Keep open. Current main still treats macOS app Launch at login as a live launchd boolean, with no persisted user intent, no stale/unloaded/repair-needed state, and no Sparkle post-update reconciliation for the app LaunchAgent. The related #40213 KeepAlive fix is merged and present in the changelog, but it addresses relaunch-after-quit, not the update-time setting drift reported here.

## Best Possible Solution

Keep this issue open and fix the macOS app-owned Launch at login contract: persist the user's intended launch-at-login preference separately from live launchd state, model disabled versus stale/unloaded/repair-needed app LaunchAgent states, reconcile or re-bootstrap `ai.openclaw.mac` on app startup and Sparkle update/relaunch paths, and add focused Swift tests for stale plist, unloaded job, UI reconciliation, explicit disable, and post-update repair behavior.
