## Summary

Keep #58733 open. Current main still sends bundled Grok web_search through the shared web-search timeout default of 30 seconds, and the latest release tag still has the same code path. The linked PR #43609 is relevant and proposes the same 60s Grok default, but it is open/unmerged in the provided context, so the bug is not implemented on main.

## Best Possible Solution

Keep the issue open and use #43609 as reference material, but land or rework a focused fix on main: give the bundled Grok/xAI web_search path a provider-appropriate default such as 60s, or expose a provider-owned webSearch timeout while preserving the shared default for other providers. Add regression coverage that proves unset Grok web_search no longer inherits the 30s shared fallback and that explicit tools.web.search.timeoutSeconds overrides still win.
