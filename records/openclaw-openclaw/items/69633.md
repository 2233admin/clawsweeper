## Summary

Keep this PR open. Current main does not contain the SPA-side WebView2 bridge, native ready handshake, draft-text listener, or OpenClawApp lifecycle cleanup wiring. The linked openclaw/openclaw-windows-node#192 host work is useful context and supports the integration, but the OpenClaw SPA side is still a live implementation candidate rather than obsolete or implemented.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to finish the Control UI SPA bridge with explicit payload validation, focused unit tests for handshake/listener cleanup/no-op/malformed-message behavior, and a draft-text path through handleChatDraftChange so input-history navigation resets consistently, then land it if maintainers accept the Windows WebView2 host integration.
