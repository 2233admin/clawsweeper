## Summary

Keep open. Current main still lacks Bedrock service-tier support, so the PR is useful, but the branch is not ready to merge because it is behind newer Bedrock wrapper behavior on main and its public Plugin SDK export is missing the required API baseline refresh.

## What This Changes

The PR adds Amazon Bedrock `serviceTier`/`service_tier` support by resolving model extra params, injecting top-level Bedrock Converse `serviceTier`, exporting shared helpers through the Plugin SDK, documenting the setting, and adding Bedrock plugin tests.

## Best Possible Solution

Keep the PR open, rebase it onto current main, preserve the current Bedrock Opus 4.7 temperature-omission and failover behavior while adding service-tier wrapping, make an explicit call on the SDK export, refresh the Plugin SDK API baseline if the export remains public, and rerun the changed Bedrock/plugin-SDK gates before merge.
