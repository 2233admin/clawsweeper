## Summary

Keep this PR open. Current main still only reads `IDEProvisioningTeams` from Xcode preferences, while the PR addresses a distinct `IDEProvisioningTeamByIdentifier` layout that was not covered by the merged Xcode 16+ provisioning-profile fallback. The implementation is focused and useful, but the remaining review finding is still valid because the changed smoke test does not execute the new inline Python parser.

## What This Changes

The PR updates `scripts/ios-team-id.sh` to read Xcode team data from both `IDEProvisioningTeamByIdentifier` and `IDEProvisioningTeams`, and adjusts the iOS team-id test fixtures and expectations.

## Best Possible Solution

Land the focused signing-helper support after the regression test runs the real inline Python parser against mocked `defaults export` plist data containing `IDEProvisioningTeamByIdentifier`, while preserving the existing explicit-env, `IDEProvisioningTeams`, legacy defaults, managed-profile, and optional keychain fallback paths.
