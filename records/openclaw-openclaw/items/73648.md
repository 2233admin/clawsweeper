## Summary

Keep this PR open. Current main still has the stop-timeout stale task behavior it targets, #71412 remains open, and the PR is an active focused gateway implementation candidate rather than obsolete or already implemented. The diff is scoped to gateway runtime, tests, and changelog; the main residual question is the known plugin-level I/O cancellation concern from the Aisle comment, which is follow-up scope rather than a reason to close the PR.

## Best Possible Solution

Keep this PR open for maintainer review as the active core gateway fix for #71412. The best path is to land this PR or an equivalent gateway change that releases stale task/abort entries on stop timeout, marks the runtime stopped, ignores stale lifecycle/status writes from replaced tasks, preserves startup instrumentation, and keeps the regression coverage; track stronger plugin-level I/O cancellation separately if maintainers want to close Aisle's residual concurrency concern.
