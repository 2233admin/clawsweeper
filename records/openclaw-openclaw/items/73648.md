## Summary

Current main still has the stop-timeout zombie task failure mode, and this PR remains the active implementation candidate for open bug #71412. It should stay open for maintainer review rather than being closed as implemented, duplicate, stale, or out of scope.

## Best Possible Solution

Keep this PR open and review it as the active gateway fix for #71412. The maintainer path is to land this PR or an equivalent core change that releases stale task/abort entries on stop timeout, prevents stale task callbacks from overwriting the fresh runtime, preserves the startup instrumentation path, and keeps regression coverage for both fresh restart and stale-settlement behavior. If maintainers want to address the remaining low concurrency risk in the same area, add a generation/token contract or stronger per-channel abort handling as follow-up rather than closing this recovery fix.
