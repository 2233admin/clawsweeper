## What This Changes

The issue reports that a selected `openai-completions` fallback provider behind a local OpenAI-compatible proxy still fails real agent completion despite direct minimal chat-completions requests succeeding.

## Best Possible Solution

Keep the issue open until the failing fallback attempt is captured. The maintainable end state is either a documented strict-minimal `openai-completions` compatibility mode, a targeted sanitizer for the captured rejected field, or better opt-in diagnostics that reveal the rejected request shape without leaking secrets.

## Solution Assessment
