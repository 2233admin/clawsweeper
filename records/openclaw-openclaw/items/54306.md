## Summary

Keep this PR open. Current main still creates `.openclaw-config-doc-cache` under the supplied repo root during the isolated config-surface retry and only removes the per-run child directory, so the PR addresses a real remaining repo-root side effect with a narrow cleanup and regression assertion. The earlier review concern about moving the retry to `os.tmpdir()` was addressed in follow-up: the current proposed patch keeps the retry under the repo-root parent for dependency resolution and removes only the now-empty parent.

## Best Possible Solution

Keep this PR open for maintainer review and merge if validation remains green. The best fix is the narrow cleanup already proposed: keep the isolated retry under the repo-root cache parent so dependency resolution still works, remove the empty `.openclaw-config-doc-cache` parent after deleting the per-run copy, tolerate `ENOENT` and `ENOTEMPTY`, and retain regression coverage that a successful retry leaves no repo-root cache residue.
