## Summary

Current main still does not provide a hook result field or runner path that lets plugins replace the submitted user prompt. The additive context and system-prompt hook paths are wired, including related Codex/system-prompt work, but the requested user-prompt override remains missing.

## Best Possible Solution

Keep this issue open until maintainers either accept and implement an explicit user-prompt replacement field or intentionally reject the API. A complete accepted fix should update the public hook result type, prompt-mutation filtering, merge precedence, embedded runner, CLI runner, Codex harness, docs, and focused regression tests for replacement plus prepend/append ordering.
