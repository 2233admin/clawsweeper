## Summary

Keep open. Current main still supports prepending/appending user-context text and system-prompt mutations from `before_prompt_build` / legacy `before_agent_start`, but there is no public hook result field or runner path that lets a plugin replace the submitted user prompt. The related implementation PR #8178 is closed and unmerged; the related `systemPrompt` work is adjacent but does not solve user-prompt replacement.

## Best Possible Solution

Keep this issue open until OpenClaw either implements and documents an explicit user prompt replacement field for `before_prompt_build` and legacy `before_agent_start`, or maintainers explicitly reject that API. A complete fix should update public hook result types, prompt mutation filtering, hook merge precedence, embedded runner, CLI runner, Codex harness, docs, and regression tests for replacement plus prepend/append ordering.
