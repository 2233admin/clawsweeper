## Summary

Keep open. Current main still treats snapshot `channelMeta` or `channelOrder` as the complete Channels page render list, so a custom-only snapshot can still hide built-in channel cards. The PR addresses the right user-facing bug, but its hard-coded built-in list should be refreshed to use the current catalog-backed channel order before merge.

## What This Changes

The PR exports a built-in channel order, updates `resolveChannelOrder()` to append missing built-in channel IDs after snapshot-provided IDs with de-duplication, and adds focused unit coverage for the resolver.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, or equivalent reimplementation. The fix should update `ui/src/ui/views/channels.ts` so snapshot-provided custom/loaded IDs are merged with a catalog-backed built-in/bundled channel order, deduped while preserving snapshot order first, and covered by focused resolver regression tests for custom-only `channelMeta`, mixed `channelOrder`, and no-snapshot fallback. Prefer a UI-safe seam sourced from `src/channels/ids.ts` or generated channel metadata over a hard-coded legacy list, and avoid changing backend plugin loading behavior unless maintainers explicitly choose that direction.
