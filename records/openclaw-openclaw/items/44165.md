## Summary

Keep open. Current main still treats gateway-provided `channelMeta` or `channelOrder` as the complete Channels card list, so a custom-only snapshot can still hide built-in channel cards. The PR targets a real paired issue, but the patch should be refreshed to avoid a hard-coded legacy built-in list that already drifts from current bundled channel metadata.

## What This Changes

The PR exports a hard-coded built-in channel order, changes `resolveChannelOrder()` to return deduped snapshot IDs followed by built-in IDs, and adds resolver tests for custom-only metadata, mixed order, and no-snapshot fallback.

## Best Possible Solution

Land a refreshed UI-only fix that merges snapshot-provided channel IDs with a UI-safe catalog-backed bundled channel order, dedupes while preserving snapshot order first, and keeps backend plugin loading behavior unchanged. Add focused resolver regression coverage for custom-only `channelMeta`, mixed `channelOrder`, and no-snapshot fallback.
