## Summary

Keep open. Current main still has the exact behavior this PR is trying to change: the Control UI Channels page treats snapshot `channelMeta` or `channelOrder` as the complete card list, so a custom-only gateway snapshot still omits the built-in channel cards. The PR also references open feature request #31379, and current main does not yet implement the requested UI behavior.

## Best Possible Solution

Keep this PR open for maintainer review or reimplementation. The desired fix belongs in the Control UI channel ordering path: merge snapshot-provided custom/loaded channel IDs with the built-in channel order, dedupe while preserving snapshot order first, and add focused regression tests for custom-only `channelMeta`, mixed `channelOrder`, and no-snapshot fallback. On current main, source the built-in order from the catalog-backed channel ID surface where feasible instead of adding another independent hard-coded list.
