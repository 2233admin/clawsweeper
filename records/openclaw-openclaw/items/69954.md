## Summary

Keep this PR open. Current main still lacks the canonical `<sessionId>.jsonl` fallback for `sessions cleanup --fix-missing`, while the PR addresses the real cleanup safety bug tracked by #50248. The remaining blocker is the review finding that repair-only runs can rewrite `sessionFile` while still reporting `wouldMutate: false`.

## What This Changes

This PR modifies `src/commands/sessions-cleanup.ts` and `src/commands/sessions-cleanup.test.ts` to check a canonical session transcript before pruning stale `sessionFile` entries and to add regression coverage for that cleanup path.

## Best Possible Solution

Keep this PR open, fix repair-only mutation accounting, then review and land the canonical transcript fallback with focused regression coverage. The final behavior should keep true-missing transcript pruning intact while preventing `sessions cleanup --fix-missing` from deleting entries whose canonical `<sessionId>.jsonl` transcript still exists.

## Repair Work Prompt

Treat this PR as useful source work for the stale `sessionFile` cleanup bug. Update the PR branch if safely editable; otherwise create a narrow replacement PR, credit the source PR, and keep the implementation focused.

Observable bug: `openclaw sessions cleanup --fix-missing` can prune a valid session when stored `entry.sessionFile` is stale/missing but the canonical `<sessionId>.jsonl` transcript exists. The current PR adds the canonical fallback and repairs `entry.sessionFile`, but repair-only runs can still report `wouldMutate: false` because `pruneMissingTranscriptEntries` returns only a prune count.

Expected fix boundary: preserve the canonical transcript fallback and stale-path repair, but change cleanup accounting so repairs are counted as mutations in both dry-run JSON and applied JSON. A reasonable shape is for `pruneMissingTranscriptEntries` to return `{ removed, repaired }` or equivalent, then include repairs in `wouldMutate`. Add a public count field only if tests and docs are updated consistently.

Do not broaden cleanup into transcript rotation/reset/archive behavior, do not change unrelated session pruning rules, and preserve path containment through existing session path helpers. Keep true-missing transcript pruning unchanged.

Tests: add focused coverage in `src/commands/sessions-cleanup.test.ts` for dry-run stale-path/live-canonical repair, applied repair updating `sessionFile` without pruning, repair-only `wouldMutate: true`, and truly missing transcripts still pruning.

Changelog/docs: this is user-facing CLI cleanup safety. Add a single-line changelog fix with reporter/contributor attribution if the active release changelog expects CLI bug fixes. Only update `docs/cli/sessions.md` if a new public JSON field or behavior wording needs documentation.
