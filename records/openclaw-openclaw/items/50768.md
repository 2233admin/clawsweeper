## Summary

Keep open. Current main supports sub-agent defaults for model/thinking/timeout and target-agent discovery, but it does not implement reusable sub-agent templates: `agents.list[].subagents` has no `templates` schema, `sessions_spawn` has no `template` parameter, and native spawn builds the child prompt from the runtime `task` only. The cross-reference to AIpexForge/agent-swarm#19 is useful prior art, but it is not an OpenClaw implementation.

## Best Possible Solution

Keep this issue open for maintainer product/API review. If accepted for core, implement it as an additive `sessions_spawn.template` feature backed by typed `agents.list[].subagents.templates`, with prompt-file resolution relative to the selected agent workspace, explicit tool params overriding template defaults, discovery via `agents_list` or a narrow new tool field, and aligned docs/schema/tests. If maintainers decide this is orchestration policy rather than platform API, redirect it to a ClawHub skill/plugin pattern instead of closing as already implemented.
