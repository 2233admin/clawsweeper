## Summary

Keep open. Current main still supports explicit `sessions_spawn` model/thinking/timeout/label parameters and global/per-agent sub-agent defaults, but it does not provide reusable sub-agent template definitions, a `sessions_spawn.template` input, prompt-file prepending, or template discovery. The remaining work changes public config/tool API and would introduce workspace file reads into model context, so it needs maintainer product/API review rather than stale cleanup or a ClawHub-only close.

## What This Changes

Add reusable sub-agent template config under `agents.list[].subagents` and a `sessions_spawn` template parameter that prepends a prompt file and supplies default model, timeout, and label values.

## Best Possible Solution

Maintainers should decide the product/API boundary first. If accepted for core, implement an additive typed template contract with safe prompt-file resolution relative to the selected agent workspace, bounded reads, explicit override precedence, template discovery, generated schema/docs updates, and focused tests; otherwise document the supported skill/plugin orchestration pattern and leave the built-in tool contract unchanged.
