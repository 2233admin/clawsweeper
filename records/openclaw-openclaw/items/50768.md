## Summary

Keep open. Current main still supports explicit `sessions_spawn` arguments plus global/per-agent sub-agent defaults, but it does not implement reusable sub-agent templates, a `sessions_spawn.template` selector, prompt-file prepending, or template discovery. The issue is concrete, and the existing ClawSweeper comments remain directionally correct; this should be maintainer product/API review rather than stale cleanup because it changes public config/tool contracts and would read workspace files into model context.

## What This Changes

The issue requests reusable sub-agent templates that define a prompt file, model, timeout, and label defaults, plus a `sessions_spawn.template` parameter.

## Best Possible Solution

Maintainers should decide the core API boundary first. If accepted, implement an additive typed template contract with safe prompt-file resolution relative to the selected agent workspace, bounded reads, explicit override precedence, template discovery, schema/docs generation, and focused tests; otherwise document the supported skill/plugin orchestration pattern and leave `sessions_spawn` unchanged.
