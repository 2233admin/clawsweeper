## Summary

Keep open. Current main still does not implement the PR's default-on ACP progress forwarding or `acp.progressForward` opt-out, and the PR remains a potentially useful implementation candidate. It is not merge-ready against current main because the generated config artifact workflow now tracks `docs/.generated/config-baseline.sha256` rather than the JSON/JSONL files this branch updates, and the new user-facing behavior is documented in root `docs.acp.md` rather than the published ACP guide.

## What This Changes

This PR default-enables implicit ACP parent progress forwarding with an `acp.progressForward` opt-out, adds thread-bound forwarding exclusions and regression tests, and updates ACP config schema/help/labels plus ACP docs and generated config artifacts.

## Best Possible Solution

If maintainers want this default, port the branch onto current main, keep `acp.progressForward: false` as the opt-out, preserve the session-binding/session-key/delivery-context thread exclusions, update `docs/tools/acp-agents.md` and `docs/.generated/config-baseline.sha256`, coordinate the relay event contract with #54471, then merge after targeted ACP tests and the changed gate pass.
