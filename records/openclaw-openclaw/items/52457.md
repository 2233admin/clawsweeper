## Summary

Keep this PR open. Current main has explicit ACP parent streaming via `streamTo: "parent"` and a narrower implicit path only for subagent requester sessions with heartbeat plus a session-local relay route. It does not implement this PR's central behavior: default-on progress forwarding for ordinary parent Gateway sessions, an `acp.progressForward` opt-out, or the added thread-bound guard coverage. The PR is a plausible implementation candidate, but it needs maintainer review and a current-main rebase rather than cleanup closure.

## Best Possible Solution

Keep the PR open as an implementation candidate. Rebase it onto current main, preserve the `acp.progressForward !== false` opt-out plus schema/help/labels/generated config updates, move or update docs in the current ACP docs surfaces, and keep regression coverage for ordinary parent sessions, opt-out behavior, active session bindings, session-key thread/topic markers, and delivery-context-only thread IDs. If maintainers decide default-on forwarding is not desired, close after that explicit product decision rather than as already implemented.
