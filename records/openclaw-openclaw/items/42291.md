## Summary

Keep open. Current main still persists `openclaw models auth login` results to the locally resolved agent auth store, and the gateway surface still only exposes model auth status, not a credential-sync mutation. PR #42381 is still the active open implementation candidate and uses closing syntax for this issue, so this should remain open until that PR is reviewed, merged, or rejected.

## Best Possible Solution

Keep this issue open and review PR #42381 as the active implementation candidate. The conservative fix belongs in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts` to block or clearly warn on OAuth login from remote node hosts. If maintainers prefer direct node-host OAuth, first design a secure gateway-side auth-profile mutation API and scopes.
