## Summary

Keep open. Current main still runs `openclaw models auth login` against the locally resolved agent auth store, with no remote node-host block, warning, or gateway credential-sync mutation. PR #42381 is still open and uses closing syntax for this issue, so the tracker should stay open until that PR is reviewed, merged, or rejected.

## Best Possible Solution

Keep this issue open and review PR #42381 as the active conservative fix. The likely acceptable path is to block or clearly warn on OAuth login from remote node hosts in `src/commands/models/auth.ts` and cover it in `src/commands/models/auth.test.ts`; if maintainers want node-host OAuth to work directly, first design a scoped gateway auth-profile mutation API.
