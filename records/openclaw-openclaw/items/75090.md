## Summary

Keep open: current main still has the tmux-only 1Password skill bug, and this PR is the focused fix path, but the new service-account example uses a command that official service-account docs do not list as supported. That should be corrected before merge.

## What This Changes

This PR rewrites the bundled 1password skill and reference notes to choose direct `op` execution or tmux by auth mode, and adds an Unreleased changelog fix entry for the linked bug.

## Best Possible Solution

Merge the auth-mode-aware skill update after replacing the service-account example with supported verification or secret-read commands, keeping the fix limited to the bundled skill/reference/changelog surface and avoiding core runtime changes.

## Reproduction Assessment

Yes. The linked bug gives a concrete macOS desktop-app path where direct `op vault list` works but the tmux-wrapped call fails, and current main still instructs agents to use tmux for every `op` command.

## Solution Assessment

Mostly yes. Updating the bundled skill by auth mode is the narrow maintainable fix, but the service-account example should use a documented supported command such as `op whoami` or `op read` rather than `op vault list`.
