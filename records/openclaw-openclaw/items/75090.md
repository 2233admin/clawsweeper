## Summary

Keep open. The PR targets a real bundled 1Password skill bug, and current main still has the tmux-only guidance, but the proposed docs should correct the platform-specific 1Password IPC explanation before merge.

## What This Changes

The PR rewrites the bundled 1password skill and reference notes to choose direct `op` execution for service-account and desktop-app auth, keep tmux for standalone interactive signin, and adds an Unreleased changelog entry.

## Best Possible Solution

Merge an auth-mode-aware skill update after replacing the socket-specific explanation with official, platform-neutral 1Password desktop app integration wording, while preserving direct exec for desktop-app and service-account flows and keeping tmux only for standalone session-token signin.

## Reproduction Assessment

Yes. The linked bug has a clear macOS desktop-app reproduction, current main contains the tmux-only instructions that trigger it, and 1Password's official docs support direct desktop-app and service-account auth flows.

## Solution Assessment

No, not as written. The auth-mode split is the right narrow fix, but the implementation should correct the desktop-app IPC explanation before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Correct the desktop app IPC explanation:** `skills/1password/SKILL.md:60-61`
  - body: The new desktop-app section says 1Password CLI uses a per-user Unix socket on macOS, and the reference/changelog generalize that socket model across desktop platforms. Please keep the direct-exec guidance but replace this with platform-neutral wording or limit socket detail to Linux-specific troubleshooting.
