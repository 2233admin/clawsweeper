## Summary

Keep this issue open. Current main partially satisfies the request with Control UI cron editing, delete-after-run wiring, run history, Gateway cron finished events, backend/channel failure notification paths, and an inline context-pressure notice with a Compact action. The central remaining product request is still missing: there is no native in-app notification/toast queue, no actionable cron-result popup layer, and no snooze/reschedule controls.

## What This Changes

The issue requests a native Control UI notification/toast system with actionable cron-result popups, inline cron management including one-shot jobs and snooze/reschedule controls, plus context usage monitoring and warning actions.

## Best Possible Solution

Keep this issue open and split or explicitly scope the remaining product work: add a native Control UI in-app notification/toast queue, wire cron finished events into actionable result notifications, define validated snooze/reschedule behavior, and decide whether context pressure stays inline-only or also emits toast-style Compact prompts.
