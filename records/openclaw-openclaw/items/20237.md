## Summary

Keep open. Current `main` still only partially covers the request: cron editing, run history, delete-after-run, inline context pressure UI, and Web Push subscription/test plumbing exist, but there is no generic in-app Control UI toast queue, no actionable cron-result popup layer, and no snooze/reschedule flow. The prior issue comments reached the same conclusion, and the newer related PR is explicitly a narrow Notifications settings-tab visibility fix rather than the broader notification architecture requested here.

## What This Changes

The issue requests a native WebUI notification/toast system with cron-result action popups, inline cron management including one-shot jobs and snooze/reschedule controls, plus context usage monitoring with warning actions.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify persisted snooze/reschedule behavior, and decide whether context pressure should also emit notification prompts before implementing the work in focused slices with tests and docs.
