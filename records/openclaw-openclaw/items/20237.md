## Summary

Keep open. Current main still partially satisfies the request, but the central product surface is missing: there is no generic in-app Control UI toast queue, no actionable cron-result popup layer, and no snooze/reschedule UI flow. The existing issue discussion reached the same conclusion, and related open notification work in #53228 should be coordinated rather than closing this broader request.

## What This Changes

Request a native WebUI notification/toast system with actionable cron job popups, snooze/reschedule controls, inline cron management including delete-after-run, and context usage warnings with one-click flush or compaction.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify snooze/reschedule behavior, and decide whether context pressure remains inline-only or also emits notification prompts. Then implement the selected slices with UI, Gateway, docs, and targeted regression coverage.
