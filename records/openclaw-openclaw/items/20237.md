## Summary

Keep open. Current main partially covers the request with cron editing, delete-after-run UI, run history, Web Push subscription/test plumbing, and an inline context-pressure compact notice, but it still lacks the central in-app toast queue, actionable cron-result popup layer, and snooze/reschedule UI flow. ## What This Changes

Add a native WebUI notification/toast system with cron-result management popups, snooze/reschedule actions, one-shot cron UI exposure, and context usage warning integration.
