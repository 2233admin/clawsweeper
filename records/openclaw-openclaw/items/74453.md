## Summary

Keep open. Current main still passes an empty string to the voice-call in-flight limiter when `req.socket.remoteAddress` is missing, and the PR’s latest one-file patch directly closes that fail-open path. The patch looks functionally correct, but it is not implemented on main yet and the head check suite still has unrelated failures to clear before merge.

## What This Changes

The PR changes the voice-call webhook handler to log missing `req.socket.remoteAddress` and use a constant non-empty fallback key for the pre-auth in-flight limiter.

## Best Possible Solution

Land the focused voice-call fallback-key fix after CI/rebase validation, ideally with a small missing-remote-address regression test if maintainers accept the private-method seam. Keep the shared SDK empty-key opt-out contract and unrelated channel behavior for separate follow-up work.
