## Summary

Keep this issue open. Current main has partial Telegram reaction support and outbound prompt guidance, but it still lacks the requested `reactionGuidance: "off" | "minimal" | "full"` config surface, schema/generated metadata/docs coverage, gating of the guidance by `reactionNotifications`, and incoming-reaction soft-CTA prompt guidance.

## Best Possible Solution

Implement the missing feature in the bundled Telegram plugin plus shared agent prompt composition: add and document `reactionGuidance: "off" | "minimal" | "full"`, define its composition with existing `reactionLevel` and `reactionNotifications`, gate incoming soft-CTA guidance on enabled reaction notifications, update strict schema/generated metadata/docs, and add focused tests for off/minimal/full plus notification gating.

## Clownfish Work Prompt

Implement Telegram reaction prompt guidance for this issue. Current main already supports Telegram incoming reaction events, outbound `message` tool `react`, and `reactionLevel`; do not remove or regress those. Add a public `channels.telegram.reactionGuidance` option with values `off | minimal | full`, default `minimal`, and document how it composes with `reactionNotifications` and existing `reactionLevel`. Prompt guidance should only be injected when Telegram reactions are enabled appropriately: outbound reaction guidance should still respect `reactionLevel`, and incoming soft-CTA guidance should be gated by `reactionNotifications` not being `off`. Update `src/config/types.telegram.ts`, strict Zod schema, generated config metadata/help as required by repo tooling, Telegram docs and reactions docs. Extend `src/agents/system-prompt.ts` to include incoming reaction interpretation guidance for full/minimal as appropriate, including examples such as eyes/interested, fire/approval, thinking/clarify, heart/thumbs-up positive acknowledgement, and laugh/humor, while keeping the prompt concise and deterministic. Add focused tests for `reactionGuidance` off/minimal/full, interaction with `reactionNotifications: off|own|all`, and preservation of current `reactionLevel` action gating. Add a user-facing changelog entry under the active version with `Thanks @sepivip`. Do not refactor unrelated channel/plugin runtime behavior or change non-Telegram reaction semantics.
