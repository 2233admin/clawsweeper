## Summary

Keep open. Current main supports the Android companion-node topology, pairing flow, command/capability registry, and Play vs third-party release-bundle mechanics, but it still does not provide or document Android beta/tester access for an existing-Gateway phone node. The prior Codex review comment reached the same conclusion, and the remaining gap is distribution/access documentation rather than the node protocol itself.

## Best Possible Solution

Keep this issue open until maintainers either publish and document an Android beta/public distribution path for existing-Gateway Android phone nodes, or explicitly route Android beta-access requests to a support/auto-response process with clear public instructions. Android capability, flavor split, signed bundle, and release-hardening work should continue in the Android app and release surfaces separately.
