## Summary

Keep open. Current main still keeps heartbeat `target: "none"` exec/cron prompts internal-only because relay wording is gated only on an outbound delivery target, while this PR adds the missing direct internal WebChat route exception and focused regression coverage. The session route-poisoning half is already largely covered by #66073, so the remaining useful review surface is the prompt-selection fallback and its interaction with recent heartbeat changes on main.

## What This Changes

The PR adds shared main/direct session-key helpers, changes heartbeat prompt relay gating to recognize direct internal WebChat routes when heartbeat delivery target is `none`, and adds targeted heartbeat regression tests for relayable and non-relayable internal routes.

## Best Possible Solution

Keep this PR open as the active implementation candidate, rebase or merge it against current main, preserve the recent heartbeat exec-noise and task-block fixes, and land only the narrow direct internal WebChat `target: none` prompt exception with the focused heartbeat/session tests.
