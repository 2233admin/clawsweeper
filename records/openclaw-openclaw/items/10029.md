## Summary

Keep this issue open. Current main still renders WebChat fenced and indented code blocks as escaped monospace text with copy-button/language/JSON-collapse chrome, but it does not tokenize code or emit syntax-highlighting spans/classes. The closed copy-button work tracked by #30390 is related but separate and does not satisfy this enhancement.

## Best Possible Solution

Implement syntax highlighting in the existing Control UI/WebChat markdown renderer or a narrow helper used by it. The fix should produce sanitization-compatible highlighted markup/styles for fenced code blocks, preserve existing code-block chrome and fallbacks, and add focused tests for highlighted output without regressing copy buttons, JSON collapse, cache behavior, raw HTML escaping, and large-message handling.

- Adding a new syntax-highlighting dependency affects client bundle and supply-chain surface, so the PR should keep the dependency narrow and verify sanitized output.
