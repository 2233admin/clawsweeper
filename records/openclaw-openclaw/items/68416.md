## Summary

Keep this PR open. Current main has already shipped part of the underlying Opus 4.7 fix, but it does not implement the PR's broader intended change for Sonnet 4.7 or future Opus/Sonnet 4.8+ adaptive-thinking classification; the branch still contains useful work, though it likely needs a rebase/port onto the current provider thinking-profile architecture.

## What This Changes

The PR changes Anthropic transport and provider registration logic/tests to replace hardcoded Claude 4.6/4.7 thinking checks with forward-compatible Opus/Sonnet 4.6+ and Opus 4.7+ patterns, adds Sonnet 4.7 dynamic resolution, and adds regression tests for future/date-stamped model IDs.

## Best Possible Solution

Port the useful behavior onto current main by putting the Claude family matching in the shared provider-model helper, using date-stamp-safe minor-version parsing, then wiring direct Anthropic transport/provider resolution and any intended Vertex surface through that shared policy with focused regression tests for Sonnet 4.7, future Opus/Sonnet 4.8, and date-stamped non-matches.
