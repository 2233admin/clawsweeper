## Summary

Keep this PR open. Current main still passes compaction reserveTokens directly into the pinned pi-coding-agent summarizer, whose generateSummary computes API maxTokens as Math.floor(0.8 * reserveTokens). The Anthropic Vertex catalog still has 1M-context Claude Opus/Sonnet 4.6 models with 128K output caps, so the reported high-context overflow remains a real current-main risk. The PR is directionally useful and includes focused tests, but it is stale/non-mergeable and needs a rebase against the current compaction call signature before landing.

## Best Possible Solution

Refresh and land this PR or an equivalent narrow fix: clamp the reserveTokens passed into built-in compaction summarization using the resolved model output limit before pi-coding-agent computes Math.floor(0.8 * reserveTokens), preserve the current headers-aware generateSummary wrapper, keep the focused clamping tests, and keep the user-facing changelog attribution.
