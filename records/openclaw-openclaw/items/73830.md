## Summary

Keep this PR open, but it is not merge-ready. The existing review comments are correct: the proposed `bash -s --` docs change does not prevent child prompts or subprocesses from reading the same piped stdin, so it would misdocument the linked installer hang rather than fix it.

## What This Changes

The PR changes the Getting Started macOS/Linux install command to `curl -fsSL https://openclaw.ai/install.sh | bash -s --` and adds a note saying those flags prevent stdin consumption.

## Best Possible Solution

Validate the linked installer failure against the hosted/current script, then harden `scripts/install.sh` so every interactive prompt or subprocess that needs user input uses terminal-backed stdin or a documented noninteractive path. After that, update all public install snippets consistently and add focused installer stdin regression coverage.

## Reproduction Assessment

Yes for the PR's claimed fix: the Bash manual plus a focused pipe smoke show `bash -s --` still leaves child reads on the script pipe. The full WSL2 installer hang remains reproducible only from the linked report until someone live-runs the current installer in that environment.

## Solution Assessment

No. This is not the best way to solve the installer hang because it changes one docs snippet and adds a false stdin guarantee; a validated installer-side fix or a proven caller-side pattern applied consistently is safer.
