## Summary

Keep this PR open. Current main still documents the raw piped installer command, but the PR’s proposed `bash -s --` change is not a verified fix for the linked stdin-consumption bug and would make the getting-started docs inconsistent with the rest of the install docs. A small shell smoke also supports the existing review concern: `bash -s --` does not stop child processes from reading the same pipe.

## Best Possible Solution

Keep this PR open for requested changes or maintainer review. The best path is to validate the linked WSL/Linux installer hang from #73814, then either harden `scripts/install.sh` so interactive children consistently use terminal-backed stdin where appropriate, or update every public install surface to a tested invocation pattern. Add a focused installer stdin regression/smoke check; consider #73837 separately because it reports Node auto-install failure handling after the workaround path.
