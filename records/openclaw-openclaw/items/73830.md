## Summary

Keep this PR open because the proposed docs change is not a correct fix for the installer stdin problem. The Bash flag claim is false, the change only updates one public install snippet, and the linked WSL2 installer behavior still needs installer-level validation before public guidance changes.

## What This Changes

This PR changes the Getting Started macOS/Linux installer snippet to pipe the hosted installer into `bash -s --` and adds a note saying those flags prevent stdin consumption.

## Best Possible Solution

Validate the WSL2 installer failure, harden `scripts/install.sh` so prompts and subprocesses cannot consume the script stream, then update all public installer examples consistently with focused stdin regression coverage.

## Reproduction Assessment

Yes for the PR's claimed fix: the focused Bash pipe smoke shows `bash -s --` does not prevent a child process from consuming the script stream. The full hosted WSL2 installer hang from #73814 remains unvalidated in this read-only review.

## Solution Assessment

No. The proposed one-page docs change documents a false stdin guarantee; a safer solution is installer-side stdin hardening or a deliberately chosen docs-wide caller pattern backed by installer smoke coverage.
