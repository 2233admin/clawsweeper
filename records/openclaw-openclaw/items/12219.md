## Summary

Keep open. Current main has SKILL.md/frontmatter skills, ClawHub scan visibility, and generic before_install/before_tool_call hooks, but it still does not implement the requested standardized skill.yaml permission manifest, install-time consent review, skills audit/trust grades, signing, or attestation. The latest maintainer discussion explicitly kept the item open for product/security follow-up.

## What This Changes

Request to add a standardized skill.yaml permission manifest for skills, including install-time permission review, audit/trust reporting, signing, and attestation support.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The end state should be a versioned skill permission manifest schema with parser/types, ClawHub and direct install/update integration, visible consent and audit/trust reporting, documented signing/attestation expectations, and a clear relationship to before_install and before_tool_call enforcement hooks.
