## Summary

Keep open. Current main still lacks the requested standardized `skill.yaml`/`skill.yml` permission manifest, install-time consent prompt, `openclaw skills audit`, signing/attestation, and trust grading. The issue has a recent maintainer comment explicitly keeping it open, and current code/docs show only adjacent pieces: SKILL.md metadata, ClawHub scans, dangerous-code install scans, and generic plugin hooks.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. A good path is a versioned skill permission manifest with validation in ClawHub and direct skill install/update flows, visible install-time permission review for `openclaw skills install`, an audit/trust report or equivalent, and a clear relationship to existing `before_install` and `before_tool_call` enforcement hooks. Coordinate with #10827, #12678, and #50090 while using `@ellistevo/openclaw-secure`, ClawHub `shell-security-ultimate`, and `globalcaos/tinkerclaw` as reference implementations.
