## Summary

Keep open. Current main still documents and implements SKILL.md/frontmatter-based skills, ClawHub scan visibility, and generic plugin hooks, but it does not implement a standardized skill.yaml permission manifest, install-time permission consent, skills audit/trust grades, signing, or attestation. The discussion also has recent maintainer signal to keep this as product/security design work rather than cleanup closure.

## What This Changes

Request a standardized skill.yaml permission manifest for skills, with install-time review, auditing/trust scoring, signing, and attestation support.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The desired end state is a versioned skill permission manifest schema with parser/types, ClawHub and direct install/update integration, visible consent and audit/trust reporting, documented signing/attestation expectations, and a clear relationship to before_install and before_tool_call enforcement hooks.
