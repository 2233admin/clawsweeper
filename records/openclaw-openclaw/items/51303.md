## Summary

Keep this PR open. Current main moved child stdout/stderr decoding behind the shared Windows output decoder, so the original child.ts patch is stale, but the PR’s intended UTF-8 split-chunk fix is still missing on the default/non-Windows path used by that shared decoder.

## Best Possible Solution

Keep this PR open, but ask for a port into src/infra/windows-encoding.ts: make the default UTF-8 decoder streaming and boundary-safe, preserve the Windows legacy-codepage fallback, and add focused tests for split UTF-8 chunks when platform is non-Windows or windowsEncoding is null/UTF-8.
