## Summary

Keep open. Current `origin/main` still emits raw `llm_input` once before prompt execution and raw `llm_output` once after attempt completion, so the PR is not obsolete; however, the branch has two merge-blocking defects around pre-stream failures and usage normalization, and the public plugin contract should be settled with #64268 before merge.

## What This Changes

The PR moves raw `llm_input` and `llm_output` plugin hook emission from the outer embedded attempt runner into per-assistant-message stream handling and updates subscription params, hook comments, and tests for per-round behavior.

## Best Possible Solution

Define a stable raw per-call observability contract alongside #64268, then revise the branch so input hooks still fire before provider submission, output usage is normalized, per-call identity is explicit, aggregate/final-run semantics remain preserved or separately labeled, and docs/changelog explain the plugin-visible migration.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Emit input hooks before prompt submission:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:72`
  - body: Moving `llm_input` to `message_start` means provider failures before the first assistant stream event no longer emit an input hook. The current attempt-level hook fires before prompt submission, so tracing plugins can still capture auth, rate-limit, network, or provider-rejection failures; this PR would lose that data for the failing rounds users most need to diagnose.
  - confidence: 0.92
- **[P2] Normalize assistant usage before emitting output:** `src/agents/pi-embedded-subscribe.ts:407-418`
  - body: The new output helper forwards `message.usage` through a narrow cast instead of the existing `normalizeUsage` path. Provider aliases such as `totalTokens`, `prompt_tokens`, cache-token fields, and timing-shaped usage can therefore drop `usage.total` or token breakdowns for per-round `llm_output` consumers.
  - confidence: 0.88
