## Summary

Keep open for maintainer review. Current main still emits raw `llm_input`/`llm_output` at run boundaries, so the requested per-round raw observability is not implemented; this PR is also not merge-ready because it moves input observation after stream start and bypasses the existing usage normalization contract.

## What This Changes

The PR moves `llm_input` and `llm_output` hook emission from embedded attempt start/end into assistant `message_start`/`message_end`, adds subscription hook context fields, and updates tests and hook comments for per-round semantics.

## Best Possible Solution

Define a stable raw per-call observability contract alongside #64268, then revise this branch so input hooks still cover failed provider submissions, per-call output usage is normalized, aggregate/final-run semantics are preserved or separately labeled, and plugin docs/changelog clearly explain the migration.

## Reproduction Assessment

Yes. The blocking findings reproduce by inspection and focused unit tests: make `activeSession.prompt(...)` fail before any assistant `message_start`, and feed assistant usage shaped like `{ totalTokens: 42 }` through the proposed per-round output path.

## Solution Assessment

No. The PR is directionally useful, but moving raw input emission to `message_start` is later than the provider-call boundary and the output helper bypasses the normalized usage contract; a safer fix keeps failed-submission observability and versions or labels per-call versus aggregate output semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Emit input hooks before prompt submission:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:74`
  - body: Moving `llm_input` to `message_start` means provider failures before the first assistant stream event never emit an input hook. Current main emits before prompt submission, so tracing plugins can capture auth, rate-limit, network, or provider-rejection failures; this PR would lose that data for the failing rounds users most need to diagnose.
  - confidence: 0.92
- **[P2] Normalize assistant usage before emitting output:** `src/agents/pi-embedded-subscribe.ts:407-418`
  - body: The new output helper forwards `message.usage` through a narrow cast instead of the existing `normalizeUsage` path. Provider aliases such as `totalTokens`, `prompt_tokens`, cache-token fields, and timing-shaped usage can therefore drop `usage.total` or token breakdowns for per-round `llm_output` consumers.
  - confidence: 0.88
