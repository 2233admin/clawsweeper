## Summary

Keep open. Current main still exposes `tools.sessions.visibility` as one four-value string scope and applies that same scope to send, history, list, and status access; the requested per-capability object form is not present. The adjacent open allowlist issue and A2A-send PR do not supersede this broader permission-model request.

## What This Changes

The issue requests a backward-compatible structured `tools.sessions.visibility` object so send, history, list, and metadata access can be granted independently instead of through one shared scope.

## Best Possible Solution

Define one coherent session-tool capability model: preserve existing string shorthands, add a validated object form for separate send/history/list/metadata permissions, keep sandbox and A2A restrictions fail-closed, update generated schema/help/docs, and add focused tests proving cross-agent send can be allowed without transcript history or broad listing.
