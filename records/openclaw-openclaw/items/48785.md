## Summary

Keep open. Current main still models `tools.sessions.visibility` as one string enum and applies the same shared visibility guard to `sessions_send`, `sessions_history`, `sessions_list`, and `session_status`; the requested object-shaped per-capability knobs are not implemented. Related #57447/#57483 cover a narrower send-without-read fix, and #55420 covers allowlists, but neither fully supersedes this permission-model request.

## Best Possible Solution

Keep this issue open for maintainer decision. The likely path is to first decide whether #57483's narrower per-action A2A send gate is the desired MVP, or whether OpenClaw should add the full backward-compatible structured `tools.sessions.visibility` object with explicit `send`, `history`, `list`, and `metadata` semantics. If implemented, the change belongs in the shared session visibility helper plus config schema/types/docs and targeted session-tool tests.
