## Summary

Keep open. Current main still exposes `tools.sessions.visibility` as a single string scope and feeds that one scope into the shared guard for send, history, list, and status access; the requested per-capability object form is not implemented or superseded by the adjacent allowlist/A2A work.

## What This Changes

The issue requests a backward-compatible structured `tools.sessions.visibility` object with separate send, history, list, and metadata capability controls while preserving the existing string shorthands.

## Best Possible Solution

Define one backward-compatible session-tool capability model: keep the existing string shorthands, add a validated object form for separate send/history/list/metadata permissions, preserve sandbox and A2A restrictions, update generated schema/help/docs, and add tests proving cross-agent send can be allowed without exposing transcript history or broad listing.
