## Summary

Keep open. Current main has Anthropic cache-control and system-prompt cache-boundary work, but it still lacks the requested Anthropic native compaction controls, Anthropic `context_management.edits` request wiring, compaction-block round-trip handling, and the mixed stable-prefix 1h / trailing-conversation 5m TTL policy. Related #10213 and #54041 are still open and the direct implementation PR #65288 was closed unmerged, so this is not implemented or clearly superseded enough to close.

## What This Changes

Request adding opt-in Anthropic native compaction controls plus mixed prompt-cache TTLs that keep stable prefixes at 1h while high-churn conversation, compaction, and user content stays at 5m.

## Best Possible Solution

Keep this open until maintainers either adopt it as the canonical Anthropic native compaction plus mixed-TTL tracker or explicitly fold its mixed-TTL acceptance criteria into #10213/#54041. The desired end state is a documented, opt-in Anthropic transport/cache-policy implementation that gates beta headers only when needed, injects Anthropic `context_management.edits`, preserves compaction blocks through streaming and replay, and uses 1h cache retention only for stable prefix material while trailing conversation content remains default/short TTL.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The current-main verification path is high-confidence: source, docs, tests, and related issue/PR context show the requested params, Anthropic compaction request payload, compaction block handling, and mixed TTL behavior are absent or only partially addressed.

## Solution Assessment

Unclear for an autonomous patch. The Anthropic transport and payload-policy area is the right technical surface, but maintainers should first choose the canonical tracker and public config/API shape across #10213 and #54041 because the prior direct PR was closed unmerged.
