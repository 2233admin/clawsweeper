## Summary

Keep this PR open. Current main still does not implement the intended default TUI behavior: the TUI resolver does not read an active Gateway lock port, the lock payload does not persist the resolved runtime port, and the linked bug #42461 remains open. The PR is a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review as the current implementation candidate for #42461. The best path is to land this or an equivalent focused fix that records the resolved Gateway runtime port in the active lock, reads that verified lock for default local openclaw tui connections, preserves explicit --url and remote-config precedence, and keeps the regression tests and user-facing docs note. Split the unrelated test stabilization only if maintainers want a cleaner patch boundary.
