## Summary

Keep this PR open. Current main still builds default TUI Gateway URLs from the normal env/config/default port path and does not persist or read an active Gateway lock port, while the linked bug #42461 remains open and this PR is the active implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review as the active implementation candidate for #42461. The best path is a focused fix that records the resolved Gateway runtime port in the per-config lock, has default local `openclaw tui` read only a verified active lock port, preserves explicit `--url` and remote-config precedence, and keeps the regression tests plus user-facing docs. Split the unrelated active-memory test tweak if it is not required for this fix.
