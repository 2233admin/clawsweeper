## Summary

Keep this PR open. Current main still does not implement the intended behavior: default `openclaw tui` connections do not read an active Gateway lock port, the Gateway lock payload does not persist the resolved runtime port, and the linked bug #42461 remains open. This PR is still a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review as the current implementation candidate for #42461. The best path is to land a focused fix that records the resolved Gateway runtime port in the active lock, has default local `openclaw tui` read only a verified active Gateway lock, preserves explicit `--url` and remote-config precedence, and keeps the new regression tests plus user-facing docs note.
