## Summary

Keep this PR open. Current main still builds default TUI Gateway connections from explicit URL/env/config/default port data, and the gateway lock payload on main does not persist the active runtime port. The PR is an active implementation candidate for #42461, not obsolete or superseded by the closed unmerged #42496.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land one canonical fix for #42461 that records the resolved runtime Gateway port in the active lock, reads it only from a verified active Gateway owner for default local TUI connections, preserves explicit --url and remote-config precedence, and keeps the new gateway-lock/TUI regression tests plus user-facing docs. Consider splitting the unrelated video-generation test stabilization if it is not needed for this PR's CI.
