## Summary

Keep open. Current main and the latest stable tag still have the malformed-content crash path in the Control UI chat normalizer and extraction helpers, while this PR is a narrow UI-only guard with focused regression coverage. The remaining action is maintainer review, rebase/merge handling, and validation rather than closing the PR as obsolete.

## What This Changes

The PR adds non-null, non-array object guards to Control UI chat message normalization and text/thinking extraction so malformed content-array entries are skipped, with regression tests for null, undefined, and array-valued blocks.

## Best Possible Solution

Land a rebased defensive guard in the Control UI chat helpers that skips invalid content-array entries while preserving all current normalized content variants, with focused tests for null, undefined, and array-valued blocks. Backend transcript persistence and Telegram media delivery should stay out of scope unless a separate bug proves they need changes.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is a message whose `content` array contains `null`, `undefined`, or an array beside a valid text/thinking block; current main still reads `.type` on those malformed entries in `normalizeMessage`, `extractThinking`, and `extractRawText`.

## Solution Assessment

Yes, directionally. A narrow non-null, non-array object guard in the UI normalizer/extractors is the maintainable fix boundary, but the final patch should be applied against current main so newer chat-rendering behavior is retained.
