## Summary

Keep open. Current main still has unguarded dashboard chat content-block paths that can dereference malformed entries, so this PR is not obsolete. The proposed change is narrow, includes regression coverage, and its second commit addresses the earlier array-valued block review concern; it mainly needs maintainer review/rebase against the newer Control UI chat helper code.

## What This Changes

The PR adds guards in the dashboard chat message normalizer and text/thinking extractors to skip null, undefined, and array-valued content entries, plus regression tests for those malformed transcript shapes.

## Best Possible Solution

Land a rebased version of the defensive content-block guard that preserves the current attachment, canvas, and tool-block behavior while adding regression coverage for null, undefined, and array-valued transcript blocks in the normalizer and extractor helpers.
