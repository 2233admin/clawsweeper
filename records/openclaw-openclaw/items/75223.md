## Summary

Keep open for maintainer review: current main still has the CLI status/probe regression, and this PR appears to be a narrow, source-consistent repair with focused regression coverage.

## What This Changes

The PR updates `models status` and `models status --probe` to load auth profiles with scoped external-CLI discovery and adds focused tests for both CLI paths.

## Best Possible Solution

Land this narrow PR, or an equivalent patch, so CLI status and probe store loading use the same scoped, no-keychain external CLI discovery contract as the gateway auth-status path.

## Reproduction Assessment

Yes. Current main can be reproduced by having a file-backed external CLI credential without a persisted OpenClaw profile: unscoped `resolveExternalCliAuthProfiles` omits it, and both CLI status/probe paths currently load the store without the needed scope.

## Solution Assessment

Yes. The proposed fix is the narrowest maintainable path I found: pass the already-known provider/profile or config-derived scope into auth-store loading, matching the documented semantics and the existing gateway implementation.
