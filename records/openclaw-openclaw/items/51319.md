## Summary

Keep this PR open. Current main still has Linux-only zombie detection in the shared PID liveness helper, and the stale session-lock recovery paths still depend on that helper. The latest PR head is a focused implementation candidate, but maintainers should verify the test mock and the new macOS subprocess behavior before merging.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Maintainers should review the latest two-file diff, adjust the Darwin regression test if the `spawnSync` spy does not actually intercept the production call, confirm the `ps` subprocess behavior is acceptable for macOS, then run the targeted PID test and changed gate before merging.

## Risks / Open Questions

- The PR test should be verified carefully: the patch spies on `childProcess.spawnSync` while production imports `spawnSync` as a named binding, so Vitest may not intercept the production call without a different mock/import pattern.
- The PR adds a PATH-resolved `ps` subprocess call in a shared liveness helper. Injection risk is low because the PID is validated and passed as an argument array, but command resolution and timeout behavior still need maintainer review.
- Targeted tests were not run in this read-only review; the next proof should include `pnpm test src/shared/pid-alive.test.ts` and the appropriate changed gate.
