## Summary

Keep this PR open. Current main has newer gateway-lock recovery for macOS via port and argv checks, but the PR’s shared `isPidAlive()` macOS zombie-detection change is not implemented. The PR also cannot be merged as-is because its diff includes unrelated runtime log artifacts with operator-specific data.

## Best Possible Solution

Keep the PR open only as a useful fix candidate. The maintainable path is to remove the committed log files, keep the safe `spawnSync("ps", ["-o", "state=", "-p", pid])` style macOS check, treat any `state` beginning with `Z` as zombie, and add mocked cross-platform regression coverage for darwin zombie states in `src/shared/pid-alive.test.ts`. Gateway-lock-specific work should stay aligned with the existing port/argv recovery already on main.
