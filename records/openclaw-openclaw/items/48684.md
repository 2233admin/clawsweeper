## Summary

Keep open. Current main still has the reported control-path gap: shared inbound debouncing bypasses slash/text commands only, while the abort-aware helper and trigger table already recognize bare phrases such as `stop`, `abort`, and `wait`; the related fix PR is closed unmerged and the debounce-boundary regression test is still absent.

## What This Changes

The issue asks for bare abort phrases such as `stop`, `abort`, and `wait` to bypass inbound message debouncing the same way `/stop` does.

## Best Possible Solution

Use abort-aware command detection for shared and plugin-local debounce bypasses, add focused regression coverage for bare abort phrases, and preserve the documented BlueBubbles same-sender DM coalescing exception.
