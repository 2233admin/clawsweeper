## Summary

Keep open. Current main still has the reported bug: the shared inbound debounce policy bypasses only `hasControlCommand()`, while bare abort phrases are recognized through `isControlCommandMessage()` and `ABORT_TRIGGERS`. The prior fix PR #48916 was closed unmerged, and current tests still lack bare `stop`/`wait` debounce coverage.

## Best Possible Solution

Keep this issue open and land a narrow fix that changes inbound debounce command detection from slash-only `hasControlCommand()` to abort-aware `isControlCommandMessage()`, with regression tests for bare abort phrases and an audit of direct debouncer call sites that still use `hasControlCommand()`. Preserve normal text debouncing, media flush behavior, slash command behavior, and the documented BlueBubbles coalescing exception.
