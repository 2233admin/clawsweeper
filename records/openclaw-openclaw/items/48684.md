## Summary

Keep open. Current `origin/main` still has the reported debounce gap: the shared inbound debounce predicate bypasses only slash/text commands through `hasControlCommand()`, while abort-aware helpers, tests, and docs already recognize standalone abort phrases such as `stop`, `abort`, and `wait`. The linked fix PR #48916 is closed unmerged, so there is no merged implementation to close against.

## What This Changes

The issue requests that standalone abort phrases such as `stop`, `abort`, and `wait` bypass inbound channel debouncing immediately instead of being delayed or merged with later text.

## Best Possible Solution

Use abort-aware command detection for shared and plugin-local debounce bypass decisions, preserve normal text/media debounce behavior and BlueBubbles’ documented coalescing opt-in, and add focused regression tests proving standalone abort phrases flush immediately.

Current `origin/main` still has `shouldDebounceTextInbound()` using `hasControlCommand()`, so standalone abort phrases such as `stop`, `abort`, `wait`, `stop action`, and localized stop phrases can be delayed or merged by channel debounce. Treat PR #48916 as useful source context, but make the current-main fix complete for shared and plugin-local debouncers.
