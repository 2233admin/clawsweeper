## Summary

Keep open. Current main still depends on openai 6.34.0 and routes OpenAI Responses, Chat Completions, and explicit SSE/WebSocket HTTP fallback streams through the OpenAI SDK before OpenClaw can filter chunks. Upstream openai-node v6.34.0 still flushes an event-only SSE frame into an object with empty data and then calls JSON.parse(sse.data), and I found no current OpenClaw guard or regression test for this empty-data SSE case.

## Best Possible Solution

Keep this issue open. The best fix is to upgrade to an OpenAI SDK release that demonstrably ignores empty/whitespace SSE data, or add an OpenClaw-owned fetch/stream guard for OpenAI Responses, Chat Completions, Azure/OpenAI-compatible fallback paths before the SDK parses SSE frames. Add focused regression coverage for event-only SSE, event-only followed by valid data, and CRLF delimiter variants.
