## Summary

Keep open. Current main still does not implement Mermaid rendering in the built-in WebChat/Control UI markdown path, and the linked feature request #48996 is closed in favor of this PR, so this is still the active implementation candidate. The submitted patch should not merge as-is because it introduces clear build/runtime blockers, weakens the sanitizer hook, regresses existing task-list markdown behavior, and bundles a large unrelated CSS refactor into the feature.

## What This Changes

The PR adds Mermaid rendering to WebChat/Control UI markdown by adding a mermaid dependency, changing the shared markdown renderer and app lifecycle render pass, updating the lockfile, and modifying broad Control UI component CSS.

## Best Possible Solution

Keep the feature alive, but require a narrower WebChat/Control UI Mermaid patch that preserves the current markdown-it renderer contracts, DOMPurify URL-scheme hardening, task-list and ordinary code-block behavior, English UI copy, and existing Control UI layout styling except for focused diagram presentation. Add regression coverage for Mermaid fences, sanitizer URL handling, non-browser imports, task lists, and normal fenced code blocks before merge.

## Security Review

Status: needs_attention

Summary: The PR introduces a new Mermaid SVG rendering path and removes existing markdown URL-scheme defense-in-depth, so it needs security review before merge.

Concerns:

- **[medium] URL scheme hardening removed:** `ui/src/ui/markdown.ts:187`
  - body: Current main's DOMPurify hook removes anchor hrefs whose resolved protocol is not http, https, or mailto. The PR deletes that allowlist and sets target/rel for any remaining href, weakening defense-in-depth on chat markdown rendered with unsafeHTML.
  - confidence: 0.9
- **[low] Rendered SVG bypasses markdown DOMPurify pass:** `ui/src/ui/markdown.ts:361`
  - body: The PR writes Mermaid's returned SVG into `element.innerHTML` after the markdown HTML has already been sanitized. Mermaid is configured with strict security level, but this creates a separate trusted SVG insertion boundary that should be explicitly reviewed and tested.
  - confidence: 0.78
