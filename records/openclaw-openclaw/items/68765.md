The PR updates gateway session history reading to chain configured-main-session `compactionCheckpoints[].preCompaction.sessionFile` transcripts before the live transcript, with focused tests and a production-derived fixture.

## Risks / Open Questions

- The branch needs a current-main rebase that preserves active-branch transcript reading; otherwise `chat.history` could re-emit abandoned tree entries that current main filters out.
- Reset and restore semantics are still unresolved: current main intentionally preserves checkpoint metadata across both flows, while the PR's chain walk would replay older snapshots unless it adds an epoch/boundary rule.

- Full snapshot concatenation may duplicate retained post-compaction tail entries unless the final design uses `firstKeptEntryId`, branch DAG state, or another explicit boundary.
