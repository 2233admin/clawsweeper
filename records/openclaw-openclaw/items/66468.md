## Summary

Keep open. Current main still has the stale-status path described here: the fallback loop knows each provider/model candidate, but no live attempt identity is emitted to WebSocket clients, and the CLI TUI footer still renders only the last sessionInfo model state.

## What This Changes

The issue requests real-time TUI footer updates for the provider/model currently being attempted during model fallback instead of showing the last persisted session model.

## Best Possible Solution

Add a typed fallback-attempt status path that is emitted immediately before each actually attempted candidate, carries runId/sessionKey/provider/model/attempt/total, is broadcast through the existing agent event path, and is consumed by the CLI TUI with correct session/run filtering to update the footer without waiting for sessions.list.

## Repair Work Prompt

Fix the stale CLI TUI footer during model fallback. Current main does not emit the provider/model currently being attempted, and the TUI only renders sessionInfo from sessions.list. Implement a narrow typed fallback-attempt status path: emit a live agent event immediately before each actually attempted candidate in runWithModelFallback/agent-command, include runId, sessionKey, provider, model, attempt, and total, and avoid emitting for candidates skipped before runFallbackAttempt. Prefer a generic callback from model-fallback into agent-command rather than importing WebSocket/event infrastructure into the generic fallback helper. In the CLI TUI, handle the new event before any overly-strict unknown-run filter if needed, but require correct sessionKey/run filtering before mutating state.sessionInfo.modelProvider/model. Refresh the footer and render after applying the event. Keep web fallback lifecycle behavior compatible; if reusing existing lifecycle fallback fields, align the CLI and web event shape rather than inventing conflicting names. Add focused tests for fallback emission, skipped-candidate behavior, TUI state update, and rejecting other-session events. Add a single-line changelog fix entry with Thanks @SasugaTV. Do not change provider fallback policy, auth cooldown behavior, model selection semantics, or broad session-list behavior.
