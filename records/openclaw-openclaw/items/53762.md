## Summary

Keep this PR open. Current main still does not contain the PR's intended early-return flush: Telegram final delivery can buffer an answer after a reasoning hint, skip reasoning delivery, then return from the segment branch without flushing the buffered final answer. The PR is narrow and matches the linked bug, but it should address the existing review concern about preserving the internal message-sent hook when the flush finalizes a preview.

## Best Possible Solution

Keep this PR open and review/rebase it or land an equivalent narrow fix in the Telegram dispatcher. The implementation should flush buffered final answers before the segmented final-delivery early return, preserve the `message:sent` hook when that flush finalizes a preview, keep the scope limited to Telegram dispatch/test/changelog, and validate with the targeted `extensions/telegram/src/bot-message-dispatch.test.ts` regression plus the normal Telegram extension test lane.
