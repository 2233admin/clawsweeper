## Summary

Keep this PR open. Current main still lacks the intended Telegram final-answer flush, but the proposed patch has a concrete hook-preservation regression: the new early-return flush can deliver the buffered answer while bypassing the existing internal `message:sent` hook path for preview-finalized deliveries.

## What This Changes

This PR adds a Telegram dispatcher flush for buffered final-answer text before the segmented final-delivery early return, adds a skipped-reasoning regression test, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after `flushBufferedFinalAnswer()` captures or returns the `LaneDeliveryResult` and routes preview-finalized buffered answers through `emitPreviewFinalizedHook()`, while keeping the fix scoped to Telegram dispatch, focused regression coverage, and the changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit the hook when flushing buffered final answers:** `extensions/telegram/src/bot-message-dispatch.ts:703`
  - body: The new segmented-final early return calls `flushBufferedFinalAnswer()` and exits, but that helper discards the `deliverLaneText()` result. When the buffered answer finalizes an existing preview, Telegram shows the answer but `emitPreviewFinalizedHook(...)` never runs, so `sessionKeyForInternalHooks` flows miss the internal `message:sent` hook unlike normal final segment delivery. Capture the result and route preview-finalized deliveries through the hook before resetting state.
  - confidence: 0.88
