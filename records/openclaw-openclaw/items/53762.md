## Summary

Keep open. The PR targets a real Telegram partial-streaming message-loss path that current main and the latest release still lack, but the proposed branch should not merge as-is because the new flush path drops the preview-finalized delivery result and skips the internal message:sent hook.

## What This Changes

This PR adds a Telegram dispatcher flush for buffered final-answer text before the segmented final-delivery early return, adds a regression test for skipped reasoning after a tool boundary, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after flushBufferedFinalAnswer() captures or returns the LaneDeliveryResult and routes preview-finalized buffered answers through emitPreviewFinalizedHook(), while keeping the change scoped to Telegram dispatch, focused regression coverage, and the user-facing changelog entry.

## Reproduction Assessment

Yes. The PR and linked bug give a concrete two-turn Telegram partial-streaming scenario, and current main statically shows that skipped reasoning can leave final answer text buffered before the segmented final branch returns.

## Solution Assessment

No, not as currently proposed. The missing flush is the right narrow fix for the dropped answer, but the implementation must preserve the same preview-finalized hook behavior as normal final segment delivery.

## Review Findings
