## Summary

Keep open. Current main still treats `agent_end` as a raw conversation hook, blocks it for non-bundled plugins unless `allowConversationAccess` is enabled, and has no settled state-only permission or lightweight turn-ended plugin contract; the linked OpenClaw Interaction Bridge use case is valid plugin API feedback but needs maintainer product/security direction.

## What This Changes

Request state-only hook access or a lightweight turn-ended lifecycle signal so non-bundled plugins can track agent completion without granting raw conversation access.

## Best Possible Solution

Raw-content hooks should stay behind `allowConversationAccess`; the durable end state is a documented lifecycle-only turn-completion contract that exposes only run/session/state metadata and is covered by schema/help, SDK docs, hook docs, and focused loader/contract tests.

## Reproduction Assessment

Yes. Source and tests give a high-confidence reproduction path: load a non-bundled plugin that registers `agent_end` without `plugins.entries.<id>.hooks.allowConversationAccess=true`, and the registry drops the hook with a warning while docs and loader tests lock that behavior.

## Solution Assessment

Unclear. The requested `allowStateAccess` flag is plausible, but the safer maintainable direction is for maintainers to choose between a sanitized turn-ended hook/event or blessing/documenting lifecycle subscriptions rather than allowing the existing `agent_end` payload through a weaker permission.
