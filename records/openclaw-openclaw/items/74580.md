## Summary

Keep open. Current main still treats `agent_end` as a raw conversation hook and has no `allowStateAccess` or equivalent state-only turn-ended hook; the existing lifecycle event subscription path may be a workaround, but deciding whether to document that path or add a narrower API is plugin SDK and security-boundary work.

## What This Changes

Request a granular state-only permission or lifecycle event so external plugins can observe agent turn completion/state without granting full conversation access.

## Best Possible Solution

Keep raw-content hooks behind `allowConversationAccess` and add or bless an additive, documented lifecycle-only contract for external plugins. The safe end state is a sanitized turn-ended/state event with only minimal run/session/state metadata, plus aligned SDK docs, config/help schema if a new permission is added, and loader/contract tests proving raw messages remain gated.

## Reproduction Assessment

Yes. A focused reproduction is a non-bundled plugin that registers `agent_end` without `hooks.allowConversationAccess`; current loader tests already assert it registers no hooks and emits the policy diagnostic.

## Solution Assessment

Unclear for implementation. The requested permission split is directionally valid, but the existing `agent_end` payload is not safe to expose as state-only; a sanitized lifecycle event or clearly documented `lifecycle` event subscription is the narrower maintainable path.
