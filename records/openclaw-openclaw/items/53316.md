## Summary

Keep this PR open. Current main still parses `chat_id:` values with `Number.parseInt(value, 10)` in the shared chat-target helper, so the hex-prefix truncation and allowlist false-match reported in #17362 remain possible. The PR’s intended fix is still useful, but the implementation now needs to be ported from the old iMessage helper file to the shared SDK-backed parser.

## Best Possible Solution

Keep this PR open and port the numeric-only `chat_id:` guard into `src/channels/plugins/chat-target-prefixes.ts` for both `parseChatTargetPrefixesOrThrow` and `parseChatAllowTargetPrefixes`. Add the PR’s regression coverage to the current iMessage tests, consider a neutral error message for all non-digit values, and include a user-facing changelog fix entry because this affects message routing and access control.
