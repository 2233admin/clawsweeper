## Summary

Keep open. The reported misrouting bug still exists on current main in the shared chat target parser, and this PR’s intended behavior is valid, but the branch edits the old iMessage helper body that current main has replaced with a Plugin SDK re-export. A maintainer should have the fix ported to the shared parser before merge.

## What This Changes

The PR branch adds decimal-only validation before parsing iMessage chat_id values and adds regression tests for hex chat_id rejection plus allowlist false-match prevention.

## Best Possible Solution

Land a narrow shared-parser fix in src/channels/plugins/chat-target-prefixes.ts that requires decimal-only chat_id values before parsing, throws a neutral invalid-chat_id error for send targets, silently rejects invalid allowlist chat_id entries, adds shared/iMessage and BlueBubbles regression coverage for the misrouting and allowlist cases, and records a user-facing changelog fix with reporter/contributor credit.
