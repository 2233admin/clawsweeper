## Summary

Keep open. The underlying iMessage `chat_id` truncation bug is still present on current `main` and in `v2026.4.27`, but this PR is not merge-ready because it adds the guard to an obsolete iMessage-local helper body while the live code now uses the shared chat-target parser used by both iMessage and BlueBubbles.

## What This Changes

The PR branch adds decimal-only `chat_id` validation to the old iMessage target parsing helper and adds iMessage regression tests for hex target rejection and allowlist false-match prevention.

## Best Possible Solution

Land a narrow shared-parser fix in `src/channels/plugins/chat-target-prefixes.ts` that requires decimal-only `chat_id` values before parsing, throws a neutral invalid-`chat_id` error for send targets, silently rejects invalid allowlist `chat_id` entries, preserves `chat_guid` and `chat_identifier` behavior, adds shared/iMessage/BlueBubbles regression coverage, and includes a user-facing changelog fix with reporter and contributor credit.
