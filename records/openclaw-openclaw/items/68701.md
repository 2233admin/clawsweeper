## Summary

Keep this PR open. The current main branch has broader WebChat route-contamination safeguards, but it does not implement this PR’s specific delivery-context normalization change, and the PR branch still has a functional concern: it preserves any structured `to` prefix for `channel: "webchat"`, including stale external prefixes such as `telegram:` or `channel:`. That needs maintainer review before merge.

## What This Changes

The PR changes `normalizeDeliveryContext` to drop unstructured `to` values for WebChat delivery contexts while preserving structured targets, and adds regression coverage for a preserved WebChat-shaped target plus a dropped phone-number-shaped target.

## Best Possible Solution

Resolve this by defining the WebChat-owned target contract and then either amending this PR or replacing it with a narrow fix that rejects stale external recipients for WebChat without preserving arbitrary structured prefixes or breaking valid internal WebChat/session targets.
