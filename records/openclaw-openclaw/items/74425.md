## Summary

Keep open. Current main still ends the root launcher after importing `dist/entry.(m)js` without an equivalent final `process.exit(process.exitCode ?? 0)`, so the proposed Windows CLI termination behavior is not already implemented. The PR is small and relevant, but it needs normal maintainer review and Windows/pipe-output validation before merge rather than cleanup closure.

## What This Changes

This PR adds an explicit `process.exit(process.exitCode ?? 0)` at the end of `openclaw.mjs` so completed CLI commands force the Node launcher process to terminate on Windows.

## Best Possible Solution

Land this PR or an equivalent launcher-level termination fix only after confirming it preserves intended exit codes, does not break long-running commands, and has at least targeted coverage or Windows smoke proof for a completed command with lingering handles.
