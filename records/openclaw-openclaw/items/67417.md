## Summary

Keep open. Current main still has the reported live-backup race: `backup create` plans source roots once and then lets `tar.c` walk them without an ENOENT-specific per-entry skip path, while session maintenance can still rename deleted transcript files during cleanup. The issue is also paired with an open closing PR, so it should remain open until a narrow or broader backup archiver fix lands.

## What This Changes

Make `openclaw backup create` tolerate session transcript files that are deleted or renamed by session cleanup during archive creation instead of failing the whole backup with ENOENT.

## Best Possible Solution

Land one coherent backup archiver fix in `src/infra/backup-create.ts` that treats expected per-entry deleted-file ENOENT as non-fatal, reports skipped files clearly, and adds a regression test proving backup creation succeeds when a planned session transcript is removed or renamed during archive creation.
