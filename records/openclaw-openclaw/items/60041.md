## Summary

Keep open. Current main appears to have addressed much of the Twitch config/schema/setup mismatch, but the reported post-connect auto-restart loop is still present in the runtime lifecycle: Twitch startAccount awaits a monitor setup function that returns a stop handle immediately, and the gateway treats that clean return as a channel exit and schedules restart attempts.

## Best Possible Solution

Fix the Twitch gateway lifecycle in the Twitch plugin: have startAccount keep the passive monitor task pending until abort, likely using runStoppablePassiveMonitor with a Twitch account status sink, then add a focused regression test proving Twitch startAccount does not resolve after successful monitor setup and only stops on abort. Keep the schema/docs/setup improvements already on main.
