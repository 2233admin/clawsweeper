## Summary

Keep open. Current main still preserves and reuses stale subagent lineage on ordinary session paths, while this PR is a narrow fix for the linked current-main bug and the refreshed diff appears to have resolved the earlier fork-marker review concern.

## What This Changes

The PR gates persisted subagent lineage fields to subagent/ACP session keys, stops ordinary top-level agent runs from reusing stored `spawnedBy`, and adds a focused regression test.

## Best Possible Solution

Land this PR or an equivalent targeted fix after current maintainer review and CI, with an optional direct `initSessionState` regression if the maintainers want fuller coverage; that should resolve the linked #67943 behavior.

## Reproduction Assessment

Yes. A source-level reproduction is clear: seed an ordinary `agent:main` session entry with stale `spawnedBy` or `spawnDepth`, then the current main path preserves that metadata and feeds it into depth resolution before `sessions_spawn` can reject at the wrong depth.

## Solution Assessment

Yes. The PR’s current shape is the narrow maintainable fix: clear subagent-only lineage from non-subagent/ACP sessions while preserving ordinary thread fork metadata, and stop agent-command from treating stored top-level `spawnedBy` as live spawn context.
