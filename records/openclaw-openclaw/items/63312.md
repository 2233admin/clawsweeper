## Summary

Keep this PR open. Current main still lacks the proposed user-managed `.agents/skills` symlink support, and the branch is a narrow implementation candidate, but it changes a security-sensitive skill containment boundary that needs maintainer sign-off and docs before merge.

## What This Changes

The PR adds a logical-path containment fallback for personal and project `.agents/skills` sources, with tests for symlinked personal/project skill loading and managed-source symlink rejection.

## Best Possible Solution

Land this PR or an equivalent narrow patch only if maintainers want personal and project `.agents/skills` symlinked directories to be supported. Preserve strict physical containment for managed, bundled, workspace, extra, and plugin-owned skill roots; keep focused regression tests; and update the public Skills security docs for the accepted exception.
