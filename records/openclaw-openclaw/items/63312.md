## Summary

Keep this PR open. Current main still enforces strict realpath containment for all skill sources, including personal and project `.agents/skills`, so the PR’s intended symlink support is not implemented. The PR is narrow and directly targets the missing behavior, but it intentionally relaxes containment for user-managed skill roots and should get maintainer review before landing.

## Best Possible Solution

Review and either land this PR or land an equivalent narrow change that accepts symlinked personal/project `.agents/skills` entries using logical containment while keeping bundled, managed, workspace, plugin, and extra-dir physical containment strict. Retain the positive personal/project symlink tests and the managed-source negative test, then add a short docs/security clarification for the user-managed exception.
