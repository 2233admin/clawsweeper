## Summary

Keep open. Current main still lacks the requested Bedrock Mantle discovery opt-out, and the issue is paired with open PR #67341 using closing syntax, so it should stay open until that PR or an equivalent fix is merged or explicitly closed.

## What This Changes

Add a Bedrock Mantle plugin `config.discovery.enabled` opt-out so users can suppress implicit IAM token and model discovery without disabling the whole plugin.

## Best Possible Solution

Land PR #67341 if it still applies, or replace it with an equivalent focused fix in `extensions/amazon-bedrock-mantle`: add `discovery.enabled` to the manifest schema and UI hints, pass validated plugin config into Mantle discovery, short-circuit before IAM token generation and `/v1/models` when disabled, document the option on the Bedrock Mantle provider page, and add regression coverage for disabled/default/enabled discovery behavior.
