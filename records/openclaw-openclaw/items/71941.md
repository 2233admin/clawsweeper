## Summary

Keep open. Current main still has the nested workspace prompt behavior this PR targets, and the PR remains a useful implementation candidate, but it has unresolved review feedback, a dirty merge state, and competing open fixes for the same bug that need maintainer selection.

## What This Changes

This PR adds a `suggestPeerAgentWorkspaceDir` helper, uses it for the interactive `agents add` workspace prompt on brand-new agents, re-exports it, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land exactly one focused fix for #71889 that changes the brand-new `agents add` prompt default to the documented peer-level workspace while preserving explicit per-agent workspaces and the #59789 shared-base runtime contract unless a maintainer deliberately changes that contract.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: after setup writes `agents.defaults.workspace` to the default workspace, `agents add <new-id>` seeds the prompt from `resolveAgentWorkspaceDir`, whose non-default fallback appends the id under that workspace.

## Solution Assessment

No as submitted. The scoped wizard-default approach is maintainable, but this PR should not merge until the environment-dependent test is fixed, attribution is clarified, and a maintainer chooses this implementation over the competing open fixes.
