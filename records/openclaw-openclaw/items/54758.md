## Summary

Keep this PR open. Current main still has the reported UI gateway bug: the close handler preserves structured connect-handshake failures for `onClose`, but pending RPC requests are still rejected with a plain `gateway closed (...)` error. The latest PR head matches the provided narrow diff, and the patch has a focused regression test with no blocking correctness or security finding from this read-only review.

## What This Changes

The PR changes `GatewayBrowserClient` so pending requests are flushed with a `GatewayRequestError` built from the stored connect failure when available, and adds a unit test covering `details.code` preservation during a failed connect handshake.

## Best Possible Solution

Land this narrow UI gateway fix, or recreate the same change on current main: reject already-pending Control UI RPCs with the structured connect failure when that failure caused the close, preserve generic close errors for ordinary disconnects, and keep the focused unit regression test.
