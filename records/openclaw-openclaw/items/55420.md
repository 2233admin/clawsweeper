## Summary

Keep open. Current main still only supports `self`, `tree`, `agent`, and `all` for `tools.sessions.visibility`; there is no `tools.sessions.allowlist` config, schema, docs, or runtime matcher for session-key patterns. The related granular-visibility issue is adjacent but does not supersede this narrower allowlist request.

## Best Possible Solution

Keep this issue open and implement it in the core session-tools visibility layer, not as a plugin. The likely path is to add `allowlist` to `SessionToolsVisibility`, add a strict `tools.sessions.allowlist: string[]` config/schema/docs surface, evaluate patterns against canonical resolved session keys in `src/plugin-sdk/session-visibility.ts`, and add targeted tests for list/history/send/status behavior. Cross-agent behavior should either continue to require `tools.agentToAgent` or be explicitly documented if the allowlist is intended to override it.
