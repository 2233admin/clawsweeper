## Summary

Keep open. Current main still does not implement `tools.sessions.visibility="allowlist"` or `tools.sessions.allowlist`; the runtime visibility type/resolver, shared access guard, strict config schema, generated config surface, docs, and focused tests still model only `self`, `tree`, `agent`, and `all`. The related #48785 is adjacent but not a canonical duplicate because it asks for per-capability visibility knobs, not session-key pattern allowlisting.

## Best Possible Solution

Keep this issue open and implement the feature in the shared session-tools visibility layer. Add `allowlist` to runtime types, strict config schema, generated config/types/help, and docs; add `tools.sessions.allowlist: string[]`; match canonical resolved session keys before `sessions_list`, `sessions_history`, `sessions_send`, and `session_status` grant access; and add focused tests for allowed/denied same-agent keys, cross-agent/A2A behavior, sandbox clamping, sessionId resolution, malformed patterns, and overbroad patterns.
