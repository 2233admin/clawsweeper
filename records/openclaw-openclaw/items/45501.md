## Summary

Keep open. Current main still builds the bare `/new`/`/reset` model prompt from a hardcoded `BARE_SESSION_RESET_PROMPT_BASE`, and the config schema does not expose `session.resetPrompt` or an equivalent per-agent override. The discussion also contains concrete scoping and Ollama reproduction evidence, while related #51586 appears to duplicate this older, richer report rather than supersede it.

## Best Possible Solution

Keep this issue open as the canonical narrow feature request for a configurable model-facing reset startup prompt. Implement a full replacement string with fallback to the existing prompt, support the requested global and per-agent precedence, update config schema/help/docs, and keep reset archival/state preservation out of scope. Related #51586 should likely be closed or linked as a duplicate of this issue after maintainers choose the final config shape.
