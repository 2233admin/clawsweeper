## Summary

Keep this issue open. Current main has the intended shared inbound debounce path and Feishu wiring, and the docs correctly distinguish idle inbound batching from active-run `messages.queue.collect`; however, the reporter reproduced the Feishu DM idle-burst case after switching to `messages.inbound.debounceMs`, and current Feishu debounce tests still cover group-style events rather than the reported p2p/DM path. The issue requests that rapid Feishu messages sent within the debounce window be batched into one agent turn instead of being processed as separate replies.
