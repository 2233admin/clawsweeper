## Summary

Keep this PR open. Current main still resolves Telegram reply-target media by media presence alone, then fetches the replied-to file, without checking whether the reply target was authored by the bot. The PR remains a focused implementation candidate for the linked open Telegram bug, and the provided final diff appears narrow and security-neutral.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent current-main patch that adds a numeric bot-id self-authorship guard before Telegram reply-target media fetches, preserves normal user-authored reply-media behavior, keeps regression coverage for the direct and debounce paths, and carries the contributor changelog credit.
