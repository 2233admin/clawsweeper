## Summary

Keep this PR open. Current main still downloads reply-target media after only checking that the replied-to Telegram message has inbound media, so a user reply to the bot's own photo can still inject bot-authored media as the next user attachment. The PR remains a focused implementation candidate for linked #57278, with the earlier undefined-ID review concern addressed in the PR head.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent patch on main that adds the numeric bot-ID guard before Telegram reply-media fetches, preserves user-authored and different-bot reply-media behavior, keeps debounce-path coverage, and carries the changelog credit for linked #57278.
