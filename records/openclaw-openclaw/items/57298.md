## Summary

Keep this PR open. Current main still resolves Telegram reply-target media after only checking that the reply target has inbound media, so a human reply to the bot's own photo can still fetch and inject bot-authored media. The PR head adds the missing numeric bot-ID guard and focused regression coverage, and it remains the active implementation candidate for #57278.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent patch on main that adds the numeric bot-ID guard before Telegram reply-media fetches, preserves user-authored and different-bot reply-media behavior, keeps debounce-path coverage, and carries the changelog credit for #57278.
