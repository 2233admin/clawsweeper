## Summary

Keep this PR open. Current `main` still fetches Telegram reply-target media whenever the replied-to message has inbound media, without checking whether that media was authored by the bot. The PR remains the implementation candidate for the linked Telegram bug, and the current branch context shows the earlier numeric-ID review concern was addressed.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent patch on `main` that adds the numeric bot-ID self-authorship guard before Telegram reply-media fetches, preserves user-authored and other-bot reply-media behavior, keeps debounce-path regression coverage, and carries the changelog credit for the linked Telegram bug.
