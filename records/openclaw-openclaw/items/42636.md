## Summary

Keep this PR open. Current main still requests and renders a 200-message initial Control UI chat window, so the intended 80-message guard is not implemented. The PR remains directionally useful, but it needs a current-main refresh because active render-window logic now lives in `ui/src/ui/chat/build-chat-items.ts`, not only the older view-local path changed by the PR.

## Best Possible Solution

Refresh the PR on current main by introducing one shared initial chat history/render window constant, preferably near the existing `ui/src/ui/chat/history-limits.ts` surface, using it from `ui/src/ui/controllers/chat.ts` and `ui/src/ui/chat/build-chat-items.ts`, deciding whether `ui/src/ui/chat/input-history.ts` should follow it, and updating both controller and direct `buildChatItems()` tests. No Gateway protocol, server, dependency, CI, or security-sensitive change appears necessary.
