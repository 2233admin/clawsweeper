## Summary

Keep this PR open. Current main still recreates `HEARTBEAT.md` whenever the workspace is missing it, including existing workspaces, so the PR’s intended behavior is not implemented. The PR remains a focused fix candidate, but it needs a rebase because current main uses `src/agents/workspace.test.ts` rather than the PR’s stale `src/agents/workspace.e2e.test.ts` path.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Rebase it onto current main, move/refit the regression coverage into `src/agents/workspace.test.ts`, set up the negative fixture with all starter files except `HEARTBEAT.md`, guard `writeFileIfMissing(heartbeatPath, heartbeatTemplate)` behind `isBrandNewWorkspace`, then land after targeted workspace and heartbeat validation.
