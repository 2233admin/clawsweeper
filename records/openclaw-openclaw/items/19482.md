## Summary

Keep this PR open. Current main still recreates HEARTBEAT.md for any workspace missing the file, which conflicts with the heartbeat runner and docs that intentionally distinguish a missing file from an effectively empty file. The PR is a valid narrow fix, but it needs to be rebased or recreated against current main because its tests target the removed src/agents/workspace.e2e.test.ts path and the negative fixture should model a fully initialized workspace with only HEARTBEAT.md missing.

## What This Changes

The PR gates HEARTBEAT.md seeding behind brand-new workspace detection and adds workspace regression coverage for fresh and existing workspaces.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Rebase or recreate it on current main, guard writeFileIfMissing(heartbeatPath, heartbeatTemplate) behind isBrandNewWorkspace in src/agents/workspace.ts, move the regression coverage into src/agents/workspace.test.ts, make the negative fixture fully represent an initialized workspace with only HEARTBEAT.md missing, add a user-facing changelog fix entry with attribution, and run targeted workspace plus heartbeat validation before the changed gate.
