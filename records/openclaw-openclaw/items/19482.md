## Summary

Keep this PR open. Current main still recreates `HEARTBEAT.md` whenever an existing workspace lacks it, so the PR’s intended fix is not implemented. The PR remains a narrow implementation candidate, but it needs a rebase onto the current workspace test layout and the negative fixture should model a fully initialized workspace with only `HEARTBEAT.md` absent.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, move the regression coverage into `src/agents/workspace.test.ts`, tighten the fixture to include `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, and `USER.md` while leaving `HEARTBEAT.md` absent, then land the `isBrandNewWorkspace` guard around the heartbeat template write.
