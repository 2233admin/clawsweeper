## Summary

Keep this PR open. Current main still computes `isBrandNewWorkspace` but writes `HEARTBEAT.md` unconditionally whenever it is missing, so the PR's intended behavior is not implemented. The PR remains a focused candidate for the heartbeat regression, but it needs a rebase onto the current test layout and a tighter negative fixture.

## Best Possible Solution

Keep this PR open as the implementation candidate for the linked heartbeat regression. Rebase it onto current main, move the regression coverage into `src/agents/workspace.test.ts`, tighten the negative fixture to create every canonical workspace file except `HEARTBEAT.md`, then land the `isBrandNewWorkspace` guard around the heartbeat template write.
