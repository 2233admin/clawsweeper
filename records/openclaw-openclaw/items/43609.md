## Summary

Keep this PR open. Current `main` still lacks the central xAI/Grok `web_search` fixes this PR is trying to provide: malformed Responses entries are still not guarded, Grok still inherits the shared 30 second timeout, and timeout AbortErrors can still cross the tool adapter as session-level aborts. The contribution is useful, but it needs a current-layout port and scope cleanup before it is mergeable.

## Best Possible Solution

Keep this PR open as useful contributor work, but ask for a narrow current-main port. Move only the xAI/Grok fixes into `extensions/xai/src/responses-tool-shared.ts`, `extensions/xai/src/web-search-shared.ts`, and `extensions/xai/src/web-search-provider.runtime.ts`; wrap internal timeout aborts with shared `isAbortError()` handling and `{ cause }`; add a Grok-specific 60 second default while preserving explicit user overrides; add focused xAI regression tests; and drop or split the unrelated `src/commands/status*` changes.
