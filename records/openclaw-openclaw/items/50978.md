## Summary

Keep open. Current main still does not implement the PR's intended force-enable behavior: `shouldIncludeSkill` only short-circuits `enabled: false`, then applies `allowBundled` and runtime eligibility checks. The status path also still requires runtime requirements to be satisfied. The linked bug was closed as stale, not fixed, so this PR remains a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Review the semantic change directly. If maintainers want `skills.entries.<key>.enabled: true` to mean force-include, land this PR or an equivalent patch that updates `shouldIncludeSkill`, keeps `allowBundled` precedence, syncs `buildWorkspaceSkillStatus`, adds regression coverage, and updates the skills docs. If maintainers prefer runtime eligibility to remain authoritative, close the PR with that product decision rather than as already implemented.
