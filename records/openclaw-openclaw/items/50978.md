## Summary

Keep this PR open. Current main still lacks the intended `enabled: true` force-enable behavior: `shouldIncludeSkill` only short-circuits `enabled === false`, then applies `skills.allowBundled`, then always calls runtime eligibility checks. The PR remains a focused implementation candidate for the linked skill-gating bug, and its follow-up commit addressed the prior review comments by documenting precedence and syncing status eligibility.

## Best Possible Solution

Keep the PR open for maintainer review. If `enabled: true` is meant to force-include a skill, land this PR or an equivalent patch that updates `shouldIncludeSkill`, keeps `skills.allowBundled` precedence, keeps skill status/listing behavior aligned, and updates the skills docs. If runtime eligibility should remain authoritative, close the PR with that explicit product decision.
