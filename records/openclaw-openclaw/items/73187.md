## Summary

Keep open. Current main still emits and tests the compact Gateway startup log shape, and the requested structured auth/server/model/plugins/log-file/health/heartbeat startup summary is not implemented. The related #14861 is adjacent subsystem-outcome observability work, but it does not supersede this formatted startup display request.

## What This Changes

Request to restore or make configurable a structured Gateway startup summary that reports auth, server URL, agent model, loaded plugins, log path, and health/heartbeat status instead of the current compact lines.

## Best Possible Solution

Add one deterministic, redacted Gateway-owned startup summary on the existing startup path, covering auth mode/source, server URL, agent model, loaded plugins, log file, and scheduled service states; update focused tests, Gateway docs, and the changelog for the user-visible logging change.

Implement the Gateway startup log enhancement requested in https://github.com/openclaw/openclaw/issues/73187. Keep the change scoped to the existing Gateway startup logging path. Add a deterministic, redacted structured startup summary that includes auth mode/source, server URL or bind/port with TLS scheme, agent model, sorted loaded plugin ids/count, log file path, and scheduled service states for channel health monitor, heartbeat, cron, and delivery recovery where those states are known. Reuse existing resolved runtime/auth state such as `ResolvedGatewayAuth.modeSource`; do not log raw tokens, passwords, SecretRef values, or unrelated config.
