## Summary

Keep open. Current `main` still emits the compact Gateway startup shape and does not provide the requested structured startup snapshot; the open #14861 is adjacent for subsystem outcome logging but does not supersede this formatted startup display request.

## What This Changes

Restore or add a configurable structured Gateway startup log that shows auth mode/source, server URL, agent model, loaded plugins, log path, and health-monitor/heartbeat startup status.

## Best Possible Solution

Add one redacted, deterministic Gateway startup summary in the Gateway-owned startup path, reusing resolved auth/runtime state and existing logger behavior, with focused tests and docs; avoid a new config switch unless maintainers explicitly want both minimal and structured modes.
