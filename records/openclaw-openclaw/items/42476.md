## Summary

Keep open. Current main at 5de284c2e30f915b769fac08dbeab45f57749702 still does not implement this PR's intended hardening: the temporary media handler sets only `X-Content-Type-Options: nosniff`, and `src/media` has no `Cache-Control`, `Pragma`, `Expires`, `no-store`, or `must-revalidate` implementation or assertions. The PR remains a focused security-hardening candidate, but it needs to be rebased from the older route shape onto `createMediaRequestHandler` and have broader response-path tests.

## Best Possible Solution

Keep this PR open. Port the hardening onto current `src/media/server.ts` by setting `Cache-Control: no-store, private, max-age=0, must-revalidate`, `Pragma: no-cache`, and `Expires: 0` immediately after a matched `/media/:id` route and before validation/read/error branches. Extend tests so success, HEAD, missing, expired, invalid, oversized, and outside-workspace responses assert the anti-cache headers while preserving TTL expiry, cleanup timing, path validation, active-content download behavior, and file-serving semantics.
