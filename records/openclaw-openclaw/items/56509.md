## Summary

Keep this PR open. The central cron fix is not present on current main: the existing `skipAtIfAlreadyRan` guard is still only enabled for startup catch-up, while the normal timer tick and `runDueJobs` paths still call `collectRunnableJobs` without that option. The PR also bundles unrelated WhatsApp, transcript repair, gateway secrets retry, and exec-env security changes, so the best path is a focused cron fix plus separate review for the other changes.

## Best Possible Solution

Keep this PR open for now, but do not land it as-is. The maintainable path is to extract or replace it with a focused cron patch that passes `skipAtIfAlreadyRan: true` in `onTimer` and `runDueJobs`, adds a regression test for the linked repeated one-shot `at` job scenario, and handles the unrelated WhatsApp, transcript, secrets, and exec-env hardening changes in separate PRs with owner/security review.
