## Summary

Keep open. Current main routes Feishu DMs with the active runtime account and sender Open ID, and the shared resolver prioritizes exact peer bindings, but the supplied collapse-to-one-agent logs are not disproven without the redacted Feishu account and binding config. The related open docs PR #39389 also suggests `match.accountId` semantics remain a likely configuration trap.

## What This Changes

Route each Feishu DM sender or configured binding to its intended independent agent instead of collapsing all Feishu users into one agent session.

## Best Possible Solution

Reproduce this on current main or the latest release with redacted Feishu account IDs, binding entries, sender Open IDs, `defaultAccount`, and app/webhook setup. If the collapse still occurs, fix the Feishu route construction or binding normalization and add a regression test proving two Feishu `ou_` senders or account-scoped bindings route to distinct configured agent IDs; also land or supersede the accountId docs clarification in #39389.
