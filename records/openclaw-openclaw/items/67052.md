## Summary

Keep open. Current main still has the watchdog and stale-final mitigations, but it does not implement the requested prompt transition when visible assistant content is complete; the TUI still presents `streaming` from chat deltas until terminal chat/lifecycle handling or the watchdog path. The merged related PR #73749 explicitly left this issue and #67053 open as content-completion/finalizing timing work.

## What This Changes

The issue requests that `openclaw tui` stop showing `streaming • elapsed` soon after visible assistant content finishes, instead of waiting for delayed gateway final/lifecycle cleanup.

## Best Possible Solution

Add a compatible content-completion or finalizing path that lets the TUI stop labeling visible answer generation as `streaming` once assistant-visible content is done, while keeping the run tracked until terminal final/error/abort events complete. Preserve the existing watchdog as recovery, update protocol/docs if a new event or payload field is added, and cover delayed lifecycle finals, concurrent runs, hidden/tool-only output, local `/btw`, and reconnect fallback behavior.
