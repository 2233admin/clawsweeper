## Summary

Keep this issue open. Current main has partial explicit local embedding support, including a default EmbeddingGemma GGUF URI for provider "local", but the zero-config request is still unresolved: node-llama-cpp remains operator-installed, setup/docs still require native build steps, release checks intentionally prevent bundling it as a dependency, and provider "auto" still skips the default downloadable local model unless a real local file path is configured.

## Best Possible Solution

Keep this open until maintainers either implement true turnkey local embeddings or explicitly decide that local embeddings remain a manual optional runtime. A complete path should settle the node-llama-cpp packaging/prebuilt policy, add first-use GGUF download progress and failure handling, define clean provider "auto" versus explicit "local" semantics, coordinate with the Apple Silicon/runtime stability work, update docs, and add regression coverage for setup errors and provider selection.
