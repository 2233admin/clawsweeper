## Summary

Keep open. The PR addresses real raw `process.cwd()` crash points in dotenv loading and main-module detection, but it does not yet prove or fully cover the linked `openclaw tui` repro: the new startup test uses `status`, which bypasses CLI path setup, while `tui` still goes through cwd-sensitive path setup and TUI session initialization.

## What This Changes

This PR adds a safe `process.cwd()` helper, uses it in CLI/runtime dotenv loading and main-module detection, adds targeted regression tests, and records a changelog fix for deleted-cwd startup.

## Best Possible Solution

Land one consolidated deleted-cwd fix that uses the shared safe-cwd helper across the full startup/repro path, including dotenv, main-module detection, CLI path setup, and TUI workspace/session initialization, with a regression that exercises `openclaw tui` or an equivalent command that goes through `ensureCliPath`.
