## Summary

Keep open. Current main still has unguarded `process.cwd()` calls in the CLI startup dotenv gate, CLI/runtime dotenv loaders, and main-module detection, while this PR directly addresses the linked deleted-cwd bug with a focused helper, tests, and changelog entry. I did not find a blocking correctness or security issue in the proposed patch; the remaining action is normal maintainer review, CI completion, and coordination with overlapping PR #74062.

## What This Changes

The PR adds a safe `process.cwd()` helper, uses it in CLI startup, dotenv loading, and main-module detection to tolerate deleted working directories, and adds targeted regression tests plus a changelog entry.

## Best Possible Solution

Land one narrow deleted-cwd startup fix that uses a shared safe-cwd helper, preserves existing dotenv precedence whenever cwd exists, skips only the unavailable cwd-local `.env`, keeps global/state dotenv loading available, and retains the new regression coverage. After merge, close the linked bug and supersede the older overlapping PR if it is still open.
