## Summary

Keep open. Current main still awaits `reader.cancel()` on max-bytes overflow in the shared bounded response reader, and the current tests do not cover the non-returning cancel case described by this PR. Discord now has a 120s total attachment timeout on main, which reduces the indefinite-stall risk, but it does not implement the PR's intended fail-fast overflow behavior. The PR remains useful but needs maintainer review/rebase because Matrix now delegates to the shared reader instead of carrying the copied helper patched in the PR.

## Best Possible Solution

Keep the PR open and review or rebase it into a current-main patch that fixes `src/media/read-response-with-limit.ts` directly, adds the non-returning `reader.cancel()` regression test, and leaves Matrix on the shared SDK wrapper. The existing contributor PR is a reasonable implementation candidate, but it needs maintainer attention rather than closure as implemented.
