## Summary

Keep open. Current main still awaits `reader.cancel()` before surfacing a streamed overflow, and the unit suite still lacks the never-resolving-cancel regression covered by this PR. The branch is useful but stale against current main because Matrix now delegates through the shared SDK wrapper and the agent import cleanup no longer applies.

## What This Changes

The PR changes the shared bounded-response reader to fire-and-forget cancellation for oversized streamed media, adds a never-resolving-cancel regression test, applies the same pattern to the then-current Matrix helper copy, and removes two agent imports.

## Best Possible Solution

Land a narrow current-main patch in `src/media/read-response-with-limit.ts` that fire-and-forgets overflow cancellation, add the focused never-resolving-cancel unit test in `src/media/read-response-with-limit.test.ts`, let Matrix inherit the behavior through its SDK wrapper, and keep unrelated agent cleanup out of the patch.
