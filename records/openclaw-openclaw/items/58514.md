## Summary

Keep open. Current main still has the Google Chat direct-vs-space classification gap and no coverage for the modern Chat API space fields; the issue is also already paired with an open closing PR, so it should stay open until that PR or a replacement lands.

## What This Changes

The issue requests that Google Chat Space messages create `googlechat:group:<spaceId>` sessions and be processed instead of returning HTTP 200 with no session or response.

## Best Possible Solution

Keep this issue open and resolve it through #58993 or a narrow replacement PR that supports `spaceType` and `singleUserBotDm`, adds focused inbound tests for legacy and modern Google Chat payloads, and verifies a Space event reaches group session routing when group policy and mention gates pass.
