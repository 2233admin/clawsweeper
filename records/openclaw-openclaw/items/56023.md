## Summary

Keep this PR open. Current main does not implement the PR's intended weighted-recall feedback loop or the three SkillFoundry skills, and the existing memory-core weighting is only for short-term promotion into MEMORY.md, not user feedback that changes search ranking. The PR also mixes a memory-core feature with new agent skills that the current project vision says should normally go through ClawHub first, and its review thread still has substantive functional/data-integrity concerns to resolve before maintainers could land it.

## Best Possible Solution

Keep the PR open but ask for it to be split or narrowed: publish the three SkillFoundry skills through ClawHub or a bundle-style plugin unless maintainers approve bundling, and continue the weighted-recall work as a focused memory-core PR with a supported way to discover chunk IDs, correct ranking behavior across hybrid/vector/FTS paths, safe persistence semantics, docs, and targeted tests.

## Risks / Open Questions

- Closing as implemented would be incorrect because current main lacks both the CLI feedback surface and the search-ranking weight persistence proposed by the PR.
- The PR bundles two product surfaces: new agent skills and memory-core ranking behavior. The skill portion appears better suited to ClawHub or a bundle-style plugin unless maintainers explicitly want bundled promotion.
- Functional review concerns remain for the memory feature: feedback IDs need a supported user-facing path, weighted results must preserve IDs and re-sort after score changes, and concurrent weight writes need a safer persistence strategy.
- Security/supply-chain pass: the provided PR file list does not show workflow, dependency, lockfile, package-publishing, or external download changes. The main security-relevant surface is adding repo-root agent skill prompts plus a new file-backed state writer, which needs normal maintainer review but is not an obvious supply-chain escalation from the provided diff.
