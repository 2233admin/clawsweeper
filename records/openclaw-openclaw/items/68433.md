## Summary

Keep this PR open: current main still lacks the outbound prewarm path, so the PR is not obsolete, but the latest patch has a concrete freshness bug that should be addressed before merge. The earlier review concerns about first-turn lifecycle state and agent skill filters appear addressed in the current diff.

## What This Changes

This PR adds a mirrored-session prewarm helper that hydrates skill snapshots after outbound session metadata is recorded, calls it from outbound session entry creation, and adds targeted regression tests, alongside incidental formatting-only changes.

## Best Possible Solution

Keep the PR as the active implementation candidate for #68429, but adjust prewarm to reuse the same snapshot freshness criteria as normal reply execution, including snapshot version checks, while still avoiding `systemSent` changes. Add a stale-snapshot regression test and consider dropping unrelated formatting-only file changes before merge.
