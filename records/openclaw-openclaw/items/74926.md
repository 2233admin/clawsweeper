## Summary

Keep open. Current main still lacks the artifact RPC implementation, and this maintainer-labeled, member-authored public Gateway API PR still has a blocking taskId resolution defect before it should merge.

## What This Changes

The PR adds Gateway `artifacts.list`, `artifacts.get`, and `artifacts.download` RPCs over transcript media, SDK artifact helpers/types, generated Swift protocol models, docs, changelog, and focused handler/SDK tests.

## Best Possible Solution

Land a revised artifact RPC hardening PR that keeps the transcript-provenance and URL guardrail design, resolves task-scoped queries through task records and their owning session, and updates the public Gateway protocol docs before closing the linked artifact API issue.

## Reproduction Assessment

Yes. Create a task record whose `taskId` differs from its `runId`, add a transcript artifact tagged with that task id, then call `artifacts.list({ taskId })`; the PR sends the task id to `resolveSessionKeyForRun` and cannot resolve the owning session.

## Solution Assessment

No, not yet. The overall read-scoped Gateway/SDK artifact RPC direction is appropriate, but task-scoped lookup should use the task registry before transcript filtering, and the public protocol docs should be updated with the new methods.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Resolve taskId queries through task records:** `src/gateway/server-methods/artifacts.ts:271-272`
  - body: `taskId` is advertised as a supported artifact query, but this path sends it to `resolveSessionKeyForRun`, which only resolves run/session IDs. Existing task records keep `taskId` separate from `runId`, so taskId-only queries can return `artifact_scope_not_found` even when the task produced transcript artifacts. Resolve the task record by taskId and use its owning session before filtering artifacts.
  - confidence: 0.9
- **[P3] Document the artifact RPCs in protocol docs:** `src/gateway/server-methods-list.ts:74-76`
  - body: The PR advertises `artifacts.list`, `artifacts.get`, and `artifacts.download` as Gateway methods, but the protocol reference still omits them. Since this is a public protocol addition, add the method contract to `docs/gateway/protocol.md` so SDK and operator clients can discover the surface from the docs.
  - confidence: 0.78
