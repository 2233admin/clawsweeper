## Summary

Keep open. Current main still does not fully implement the requested hardening: OpenShell forwards generated command text into the shared SSH `/bin/sh -c` wrapper without placeholder or malformed-shell preflight, and generic same-tool/same-args loops remain warning-level until the later no-progress circuit breaker. The related #58726 covers the narrower loop-abort piece, but not the OpenShell malformed-command preflight requested here.

## Best Possible Solution

Keep this issue open for a focused runtime hardening patch: add shared SSH/OpenShell preflight before remote shell command construction for obvious generated placeholders and malformed shell structure, then add failure/no-progress-gated hard blocking for repeated malformed or identical exec/tool calls with targeted regression coverage. Keep #58726 as the narrower loop-detector follow-up unless maintainers choose to consolidate the loop portion there.

## Risks / Open Questions

- A generated-command preflight needs careful scoping so legitimate shell redirection, comparisons, command substitution, and literal angle brackets in data are not rejected incorrectly.
- A generic repeat hard-abort should likely be failure/no-progress gated and configurable to avoid blocking legitimate repeated status or polling workflows.
