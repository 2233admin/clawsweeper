## Summary

Keep open. Current main still lacks the requested OpenShell generated-command preflight, and the generic repeat detector still stays warning-level until the later global no-progress breaker; the broader #58726 overlap does not cover the OpenShell validation gap.

## What This Changes

The issue requests OpenShell fail-fast validation for obviously malformed/generated shell commands and hard aborts for repeated identical failing or no-progress tool loops.

## Best Possible Solution

Add a conservative sandbox exec preflight before the shared SSH/OpenShell `/bin/sh -c` wrapper for syntactically incomplete commands and literal generated placeholders, then escalate generic repeats to critical only when the repeated calls are failing or show no progress at the configured threshold.

## Reproduction Assessment

Yes. A static reproduction path is high-confidence: commands like `workflow install <name>` or an unclosed quote reach `buildExecRemoteCommand` and `/bin/sh -c`, while current loop tests assert generic repeats at the critical threshold remain warning-level.

## Solution Assessment

Yes. The narrow maintainable fix is targeted OpenShell/SSH exec validation plus no-progress-gated loop escalation; broad host-exec heuristic filtering would be higher risk and should stay out of this repair unless maintainers explicitly widen scope.
