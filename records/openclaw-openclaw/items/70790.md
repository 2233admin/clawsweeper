## Summary

Keep this PR open. Current main still lacks the embedded-runner payload guard for empty `tools` arrays and orphaned `tool_choice`, and the PR remains a focused implementation candidate, but it needs maintainer review because current main has since added auth/signal wrapping in the same stream-resolution path and the final fix should compose with that behavior plus add direct sanitizer tests.

## What This Changes

The PR wraps embedded runner stream functions with an `onPayload` guard that deletes empty `tools` arrays and orphaned `tool_choice`, and updates the stream-resolution test expectation for wrapped custom streams.

## Best Possible Solution

Keep the PR or a successor open and land a narrow, composable embedded-runner egress sanitizer that strips empty `tools` arrays and orphaned `tool_choice` only at the final payload boundary, preserves current auth/signal wrapping, covers the session HTTP fallback plus boundary-aware/custom paths, and leaves the already-shipped `toolsAllow` forwarding intact.
