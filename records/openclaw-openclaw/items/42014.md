## Summary

Keep this PR open. Current main still does not expose session sync health, pending session byte/message counts, last session sync time/reason, or last sync errors through the memory status payload or `openclaw memory status` output. The PR is stale against the current `extensions/memory-core` plugin layout, but the author addressed the review-reported timestamp and Map aggregation bugs, and the work remains a useful implementation candidate.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into `extensions/memory-core`. The best implementation is a typed session-sync status helper on the memory-core manager, exposed as a backwards-compatible `MemoryProviderStatus.custom.sessionSync` payload, rendered by `openclaw memory status` including JSON and `--deep` output, and covered by focused manager plus CLI tests. Carry forward the PRâ€™s timestamp consistency and Map aggregation fixes, and sanitize error summaries before displaying or logging them.
