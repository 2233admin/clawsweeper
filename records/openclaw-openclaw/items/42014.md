## Summary

Keep this PR open. Current main still does not implement the requested session sync observability in the memory status payload, plain/JSON CLI output, tests, or docs. The PR remains a focused implementation candidate, but it needs to be ported from the old `src/memory` and `src/cli/memory-cli.ts` paths into the current `extensions/memory-core` plugin layout before maintainer review can proceed.

## Best Possible Solution

Keep this PR open and ask for a port into `extensions/memory-core`. The best implementation is an additive typed `MemoryProviderStatus.custom.sessionSync` object, surfaced in JSON and plain `openclaw memory status` output, with focused memory manager and CLI tests plus docs for the new fields. Carry forward the author’s timestamp-consistency and Map aggregation fixes from the PR discussion, and sanitize displayed/logged sync errors before merge.
