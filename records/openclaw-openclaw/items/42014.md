## Summary

Keep this PR open. The requested behavior, session memory sync observability in `openclaw memory status` and JSON status, is still missing on current `main`. The PR remains a useful contribution, but it targets the old `src/memory/*` and `src/cli/memory-cli.ts` layout while current memory status is owned by the `memory-core` plugin under `extensions/memory-core`.

## Best Possible Solution

Keep this PR open and request a port to `extensions/memory-core`. The best path is an additive session sync status object exposed through the active memory provider status JSON and rendered by `openclaw memory status`, with `--deep` showing pending bytes/messages, focused memory manager and CLI tests, and docs at https://docs.openclaw.ai/cli/memory. Carry forward the PR branch's timestamp consistency and `Map` aggregation fixes, and review displayed/logged error text before merge.
