## Summary

Keep this PR open. Current main still does not expose session sync status, pending session byte/message counts, last session sync time/reason, or sync errors through the memory status payload or `openclaw memory status`. The PR is stale against the current `extensions/memory-core` plugin layout, but the author addressed the timestamp and Map aggregation review comments, and the work remains a useful implementation candidate.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into `extensions/memory-core`. The best implementation is an additive, typed session-sync status helper on the memory-core manager, exposed through a backwards-compatible status payload, rendered in `openclaw memory status` including JSON and `--deep` output, and covered by focused manager plus CLI tests. Carry forward the authorâ€™s fixes for timestamp consistency and Map aggregation, and add error sanitization before display or logging.
