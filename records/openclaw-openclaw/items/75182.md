## Summary

Keep open. Current main still has the parser path that can accept the polluted local media source, and the linked open PR is the active implementation candidate, so the issue should remain open until that PR merges or maintainers choose an equivalent fix.

## What This Changes

The issue reports that a `MEDIA:` directive with a local file path followed by serialized JSON can be parsed as one invalid local path and fail outbound delivery.

## Best Possible Solution

Land PR #75202 or an equivalent shared `src/media/parse.ts` parser fix that trims or rejects serialized JSON after a valid local media extension, with focused regression coverage that preserves existing local path variants and traversal/home-dir rejection.

## Reproduction Assessment

Yes. The current source path is high-confidence: a one-line `MEDIA:/path/to/image.png"}],...` payload is captured as one token, accepted as a slash-prefixed local path, and later read as a local file path. I did not run a live channel test because the review was constrained to read-only inspection.

## Solution Assessment

Yes. A shared parser-level fix with a regression test is the narrowest maintainable solution; channel-level guards can be defense in depth, but they should not be the primary fix.
