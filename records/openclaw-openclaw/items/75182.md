## Summary

Keep open. Current main still has the parser shape described by the report, and an open closing PR specifically targets this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to harden shared MEDIA directive parsing so a local media path followed by serialized JSON is trimmed or rejected before outbound channel delivery.

## Best Possible Solution

Land PR #75202 or an equivalent shared parser-level fix that trims or rejects serialized JSON after a valid local media source, with focused `src/media/parse.test.ts` regression coverage and no channel-specific workaround as the primary behavior.

## Reproduction Assessment

Yes. The current source provides a high-confidence reproduction path: a one-line `MEDIA:/path/to/image.png"}],"details":...` payload is captured as one candidate, accepted as a local path, merged into outbound media, and later opened as that full local path.

## Solution Assessment

Yes. The narrow maintainable direction is a shared parser boundary fix with regression coverage; channel-level guards can be added as defense-in-depth but should not replace parser hardening.
