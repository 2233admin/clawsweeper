## Summary

Keep open. Current main still does not provide a `/workspace` alias in the OpenClaw runtime Docker/Podman containers: the image only creates the CLI symlink, Compose/setup/Podman mount the workspace at `/home/node/.openclaw/workspace`, and `/workspace` remains documented/implemented as the sandbox workdir. The related PR #12764 is useful prior art, but it was closed unmerged.

## Best Possible Solution

Keep this issue open and implement a focused runtime-container compatibility alias, preferably an image-level `/workspace -> /home/node/.openclaw/workspace` symlink if validation confirms it works for Docker Compose, Docker setup, Podman, and Quadlet. Update Docker/Podman docs to describe the alias and add a regression test in the existing Dockerfile/setup test surface. If maintainers decide `/workspace` should remain sandbox-only, close the loop by documenting that explicitly instead of leaving the runtime/docs mismatch ambiguous.
