## Summary

Keep open. Current main still does not provide a runtime-container `/workspace` alias for Docker or Podman gateway/CLI containers. The runtime mounts remain `/home/node/.openclaw/workspace`, while the first-party `/workspace` behavior is sandbox-specific. The related PR #12764 was closed unmerged, so it is useful prior art rather than a shipped fix.

## Best Possible Solution

Keep this issue open. Maintainers should either implement a safe runtime compatibility alias or second bind mount for `/workspace -> /home/node/.openclaw/workspace` across Dockerfile, Compose, Docker setup, Podman/Quadlet, and relevant smoke/docs coverage, or explicitly decide that `/workspace` is sandbox-only and update runtime/workspace docs to remove the ambiguity. Treat PR #12764 as prior art, not as a canonical fix.
