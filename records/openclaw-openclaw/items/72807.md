## Summary

Keep open. Current main still has no `gitInit` opt-out on `agents.create` or `ensureAgentWorkspace`, and the PR is a small additive protocol/helper change that appears functionally correct but still needs normal maintainer review and branch validation before merge.

## What This Changes

This PR adds an optional `gitInit` boolean to the `agents.create` protocol and `ensureAgentWorkspace` path, forwards it from the gateway handler, refreshes Swift protocol models, and adds a workspace regression test.

## Best Possible Solution

Land this PR or an equivalent additive default-preserving `gitInit` option after maintainer review, with generated protocol artifacts in sync and targeted workspace/gateway validation passing.

## Reproduction Assessment

Yes. On current main, `agents.create` with `gitInit: false` is rejected by the schema, and direct workspace creation has no opt-out before `ensureGitRepo`; the PR’s added unit test covers the intended helper behavior.

## Solution Assessment

Yes. An optional default-true boolean on the protocol and helper is the narrowest maintainable fix for embedders that need non-git workspaces; the conditional spread in the handler can be simplified, but it is behaviorally equivalent.
