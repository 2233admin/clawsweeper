## Summary

Keep open. Current main still lacks the `gitInit` opt-out, and the PR implements the intended additive behavior without changing the default, but it still needs the required user-facing changelog entry before merge.

## What This Changes

The PR adds an optional `gitInit` boolean to `agents.create` and `ensureAgentWorkspace`, forwards it through the gateway handler, regenerates Swift protocol models, and adds a workspace regression test for skipping automatic `git init`.

## Best Possible Solution

Land the opt-out as an additive protocol/workspace option with default git initialization preserved, after adding the Unreleased changelog entry and keeping schema, handler, tests, and generated protocol models aligned.

## Reproduction Assessment

Not applicable as a bug reproduction. Current-main inspection gives high-confidence behavioral evidence: `gitInit` is absent from the schema/handler, and brand-new bootstrapped workspaces still run `git init` when git is available.

## Solution Assessment

Yes. An optional boolean defaulting to the existing behavior is the narrowest maintainable API shape for embedders that need to suppress automatic repo creation; the remaining fix is release-note coverage and normal validation, not a different product direction.
