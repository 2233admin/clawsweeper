## Summary

Keep open. Current main still does not implement the optional `gitInit` opt-out for `agents.create`: brand-new workspaces still run `git init`, the RPC schema does not accept `gitInit`, the handler does not forward it, and generated Swift protocol models lack the field. This PR remains an active implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Best path is to rebase on current main, drop the stale generated `NodePairRemoveParams` drift, add an additive optional `gitInit?: boolean` defaulting to true through `AgentsCreateParamsSchema`, `agents.create`, `ensureAgentWorkspace`, generated Swift models, and focused regression coverage; update workspace docs if maintainers want the opt-out public.
