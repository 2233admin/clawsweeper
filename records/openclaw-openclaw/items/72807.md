## Summary

Keep open. Current main still does not implement this PR's intended `gitInit` opt-out for `agents.create`: the workspace helper has no parameter, the RPC schema still rejects unknown create fields, the handler does not forward a git preference, and generated Swift protocol models lack the field. The PR remains a focused implementation candidate rather than obsolete cleanup; the only notable cleanup need is rebasing/regenerating to remove generated drift that already exists on main.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Best path is to rebase on current main, remove unrelated generated drift, implement `gitInit?: boolean` as an additive optional field with default `true` through `AgentsCreateParamsSchema`, the `agents.create` handler, `ensureAgentWorkspace`, generated clients, and focused regression tests, then update the public workspace docs if maintainers want this opt-out documented.
