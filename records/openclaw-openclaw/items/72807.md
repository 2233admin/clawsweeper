## Summary

Keep open. Current main still does not implement the PR's intended `gitInit` opt-out: the workspace helper has no parameter, `agents.create` schema and handler do not accept or forward it, and generated Swift protocol models lack the field. The PR remains a focused implementation candidate after rebase/regeneration.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Best path is to rebase on current main, remove unrelated generated drift, implement `gitInit?: boolean` as an additive optional field with default `true` through `AgentsCreateParamsSchema`, the `agents.create` handler, `ensureAgentWorkspace`, generated clients, and focused regression tests, then update the workspace docs if maintainers want the opt-out documented publicly.
