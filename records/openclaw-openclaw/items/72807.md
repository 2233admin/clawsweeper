## Summary

Keep open. Current main still lacks a `gitInit` opt-out for `agents.create`, and the PR is a narrow additive protocol/workspace change that preserves default behavior. I found no blocking correctness or security issue; the only prior review note is a non-blocking style simplification.

## What This Changes

The PR adds an optional `gitInit` boolean to `agents.create` and `ensureAgentWorkspace`, forwards it through the gateway handler, updates generated Swift protocol models, and adds a workspace regression test for `gitInit: false`.

## Best Possible Solution

Accept the additive RPC/workspace option after maintainer review and repo-wrapper validation, keeping the default `git init` behavior unchanged while allowing embedding hosts to opt out explicitly.

## Reproduction Assessment

Yes. A high-confidence path is to create a brand-new temp workspace through `ensureAgentWorkspace({ gitInit: false })` and assert that `.git` is absent, plus a gateway handler check that `agents.create` accepts and forwards the boolean.

## Solution Assessment

Yes. An optional boolean on the existing protocol and workspace helper is the narrowest maintainable shape for this request because it avoids config migration and preserves existing callers when omitted.
