## Summary

Keep open. Current main still does not implement the PR's requested unconditional Feishu root_id-to-MessageThreadId forwarding for normal group replies, and the PR patch is not mergeable as written because it would add duplicate inbound-context keys while leaving the existing guarded MessageThreadId assignment in place.

## Best Possible Solution

Keep this PR open for revision or maintainer decision. The strongest path is to update the existing ReplyToId and MessageThreadId assignments instead of adding duplicate keys, add focused Feishu tests for non-topic and topic-session root_id behavior, and reconcile the broader unconditional metadata request with the narrower Feishu threading work in #60848.
