## Summary

Keep open. Current main still only forwards Feishu `MessageThreadId` for topic-session scopes, while this PR asks for broader `root_id` metadata; the branch also adds duplicate inbound-context keys, so the later guarded `MessageThreadId` still overrides the new value and the patch is not mergeable as written.

## What This Changes

The PR adds Feishu `ctx.rootId` and `ctx.parentId` to the inbound context payload as `MessageThreadId` and `ReplyToId` in `extensions/feishu/src/bot.ts`.

## Best Possible Solution

Feishu should have one tested threading policy: revise the existing inbound-context assignments instead of adding duplicate keys, add focused coverage for normal-group and topic-session `root_id` metadata behavior, and align the result with the overlapping topic-thread fallback work in #60848.
