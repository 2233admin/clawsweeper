## Summary

Keep open. Current main still has the reported false-positive path: the doctor Gateway auth health check detects a SecretRef, asks the generic gateway auth resolver for credentials without resolving configured SecretRef providers, and emits the unavailable warning when token mode still needs a token. The issue is also paired with open implementation candidates, including PR #65229 with closing syntax, so it should remain open until a focused fix lands or those PRs are closed.

## What This Changes

Doctor should stop warning that SecretRef-managed `gateway.auth.token` is unavailable when the configured secret resolves and the gateway is already using it successfully.

## Best Possible Solution

Land a narrow doctor health fix that resolves `gateway.auth.token` through the existing SecretRef-aware token resolver before emitting the unavailable warning, with regression coverage proving a resolvable env/provider SecretRef does not warn and an unresolved SecretRef remains read-only and does not get replaced with plaintext.
