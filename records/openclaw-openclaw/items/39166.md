## Summary

Keep this PR open. Current main has picked up the explicit ACP `runTimeoutSeconds` forwarding path, but it still does not implement the PR's central `acp.defaultRunTimeoutSeconds` config fallback, schema/docs surface, or default-resolution tests.

## What This Changes

The PR adds an optional `acp.defaultRunTimeoutSeconds` config field and wires ACP `sessions_spawn` timeout resolution through explicit input, that config default, then `0`.

## Best Possible Solution

Keep this PR open for maintainer review or a focused rebase. The remaining useful delta is to add `acp.defaultRunTimeoutSeconds` to the ACP config type, strict Zod schema, generated schema metadata, help/labels, and user docs, then resolve ACP timeout as explicit tool value -> ACP config default -> `0` in the ACP spawn path. Regression coverage should include omitted value using config default, explicit nonzero override, explicit `0` overriding a nonzero default, and schema acceptance/rejection for non-negative integer behavior.
