## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the requested ACP config default, while the PR head leaves the resolved timeout out of ACP runtime initialization and omits required generated config artifacts.

## What This Changes

The PR adds optional `acp.defaultRunTimeoutSeconds`, forwards ACP `sessions_spawn` timeout input, resolves ACP timeout as explicit value to config default to `0`, and updates config metadata plus tests.

## Best Possible Solution

Land a rebased ACP-owned implementation that resolves one effective timeout value, applies it consistently to ACP runtime initialization and Gateway dispatch, preserves explicit `0`, updates generated config schema/docs baselines, and documents the ACP-specific default beside the existing `sessions_spawn` timeout docs.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main has no accepted `acp.defaultRunTimeoutSeconds` key, and PR-head source shows the config-resolved timeout is only used for initial Gateway dispatch rather than ACP runtime metadata.

## Solution Assessment

No. The requested direction is narrow and maintainable, but this PR is not the best merge shape until it is rebased and the resolved timeout is wired into both ACP runtime metadata and dispatch with generated config artifacts committed.
