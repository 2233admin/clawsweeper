## Summary

Keep this issue open. Current main still resolves an explicit `--agent` to that agent's main session before `--to` can create per-recipient routing, and PR #73403 is an open, unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review PR #73403 as the active implementation candidate. The fix should define precedence for `--session-key`, `--session-id`, `--agent`, `--channel`, and `--to`; derive agent-scoped per-recipient keys for explicit deliverable-channel targets; preserve intentional main-session behavior where documented; and add resolver, Gateway CLI, Gateway server, embedded/local, and docs coverage for the changed session-routing contract.
