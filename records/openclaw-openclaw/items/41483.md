## Summary

Keep this issue open. Current main still resolves `--agent` as an implicit agent-main session before `--to`/`--channel` can derive a per-recipient session key, and the provided GitHub context shows PR #73403 is an open, unmerged implementation candidate with closing syntax for this bug.

## Best Possible Solution

Keep the issue open and use PR #73403 as the current implementation candidate. The fix should define precedence for `--agent` plus explicit `--channel`/`--to`, preserve explicit `--session-key` and `--session-id` behavior, derive agent-scoped per-recipient keys through the existing routing helpers when isolated direct-message routing is intended, add regression coverage for distinct `cw_111`/`cw_222` targets through Gateway and embedded paths, and update the CLI docs once behavior changes.
