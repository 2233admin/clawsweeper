Keep this issue open. Current main still has the reported routing shape, and an open implementation candidate PR #73403 explicitly references this issue with closing syntax, so the right maintainer path is to review/land or close that PR rather than close the issue early.

## Best Possible Solution

Keep this issue open and use PR #73403 as the current implementation candidate. The best fix should define precedence for `--agent` plus explicit `--channel`/`--to`, derive agent-scoped per-recipient keys through existing helpers such as `buildAgentPeerSessionKey` when the session config calls for isolated direct-message routing, preserve explicit `--session-key` and `--session-id` behavior, add regression coverage for distinct `cw_111`/`cw_222` targets through both Gateway and embedded paths, and update the CLI docs once behavior changes.
