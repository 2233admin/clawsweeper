## Summary

Keep open. Current main has later cron timeout and task-ledger reconciliation hardening, but it does not implement the PR's separate cron `runTimeoutSeconds` payload API or its scheduled-timer proactive ledger-close change as proposed. Because the PR is draft, stale against newer timer behavior, and changes public timeout semantics, this needs maintainer review rather than conservative cleanup closure.

## What This Changes

The PR adds a cron `agentTurn` `runTimeoutSeconds` payload option, a separate wall-clock timeout path in the cron timer, proactive task-ledger finalization on scheduled timer errors, and regression tests.

## Best Possible Solution

Rebase or rework the draft against current main, then decide explicitly whether cron should expose a separate `runTimeoutSeconds` public payload field. If yes, add it consistently across types, normalization, docs/API surfaces, timer behavior, and scheduled/manual/startup regression coverage while preserving the newer durable task recovery and timeout deferral behavior.
