## Summary

Keep open. The report is fresh and concrete, current main still routes `openclaw status --usage --json` into a JSON path that awaits provider usage before writing stdout, and `loadProviderUsageSummary` still awaits provider auth resolution before the existing usage-fetch timeout begins. A linked open PR, #74185, uses closing syntax for this issue, so this issue should stay open until that PR is reviewed and merged or explicitly closed.

## Best Possible Solution

Keep this issue open and review #74185. The right fix belongs in the provider usage/auth path, covering auth/profile/plugin resolution with a bounded timeout and adding regression coverage for never-settling auth plus the non-TTY `openclaw status --usage --json` command path. After that PR merges, this issue can close as implemented.
