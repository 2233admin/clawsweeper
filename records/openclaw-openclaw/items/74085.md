## Summary

Keep open. Current main still routes `openclaw status --usage --json` as a supported machine-readable command, but the usage JSON path still awaits provider auth resolution before any stdout JSON is written. The linked open PR #74185 uses `Fixes #74085`, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review PR #74185 or land an equivalent narrow fix. The fix should bound provider usage auth resolution before JSON output waits on it, return parseable JSON with provider-level degradation instead of empty stdout/hang, and add a regression where `status --usage --json` or the underlying usage loader completes when auth resolution never settles.
