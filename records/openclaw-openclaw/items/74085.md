## Summary

Keep open. Current main and the latest tagged release still await provider usage auth resolution without a timeout before `status --usage --json` can write JSON, and open PR #74185 is the active unmerged fix candidate using closing syntax for this issue.

## What This Changes

The issue reports that `openclaw status --usage --json` hangs or exits without output from non-TTY subprocesses after 2026.4.26, especially on the OpenAI Codex OAuth usage-auth path.

## Best Possible Solution

Land a narrow fix that bounds provider usage auth resolution before JSON status output waits on it, returns parseable degraded JSON when auth resolution cannot complete, and adds a regression test where auth resolution never settles.

## Reproduction Assessment

Yes. The issue includes a concrete Node `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current main has a source path where JSON output waits on unbounded provider auth resolution; I did not run the live CLI because this review is read-only.

## Solution Assessment

Yes for triage: keeping this issue open while #74185 or an equivalent narrow fix is reviewed is the right path. The fix should put the timeout around auth resolution itself, not only around later provider usage fetches.
