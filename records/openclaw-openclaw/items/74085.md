## Summary

Keep open. Current main and latest release v2026.4.29 still await provider usage auth resolution before any timeout boundary can produce JSON output, and an open fix PR already references this issue with closing syntax. The maintainer path is to review, land, or replace PR #74185 rather than close this report early.

## What This Changes

Request that `openclaw status --usage --json` reliably return machine-readable JSON from non-TTY subprocesses after a regression in 2026.4.26.

## Best Possible Solution

Land a narrow fix that bounds provider usage auth resolution before JSON status output waits on it, returns parseable degraded JSON when auth resolution times out, and adds a regression test where auth resolution never settles.

## Reproduction Assessment

Yes. The issue includes a concrete non-TTY Node `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current main still has a source path where JSON output waits on unbounded provider auth resolution.

## Solution Assessment

Yes for triage: keeping this issue open while PR #74185 or an equivalent narrow fix is reviewed is the right path. Current main is not sufficient because its timeout only applies after auth resolution completes.
