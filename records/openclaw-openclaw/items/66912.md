## Summary

Keep this PR open. Current main still lacks the Telegram reply-media self-authored guard, so the reported bug remains real. The PR head appears to add the right runtime guard, but it still needs focused regression coverage and cleanup or justification of unrelated Plugin SDK generated baseline hash churn.

## Best Possible Solution

Keep this PR open and ask for a clean Telegram plugin patch: keep the self-authored reply-target guard wired inside resolveReplyMediaForMessage, add regression coverage in extensions/telegram/src/bot.test.ts for reply_to_message.from.id === ctx.me.id with no getFile/media fetch, preserve ordinary reply-media ingestion, and remove or justify the docs/.generated/plugin-sdk-api-baseline.sha256 change.
