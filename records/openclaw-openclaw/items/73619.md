## Summary

Keep this issue open. Current main still resolves omitted message-tool account IDs from the inherited/default account before considering the calling agent's channel binding, and open PR #74052 is the active unmerged fix path with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review PR #74052 as the canonical fix path. The final implementation should preserve explicit `accountId`, prefer the calling agent's channel account binding when the message tool omits `accountId`, fall back to inherited/default account only when no binding exists, and add the combined precedence regression test.
