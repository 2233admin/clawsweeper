## Summary

Keep this PR open. Current main still has the iOS node emitting `location.update`, but the gateway has no matching handler, heartbeat has no location-event prompt/wake path, and the public node docs still document only `location.get`. The PR is a focused candidate for a real current gap rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for the current iOS-to-gateway gap. The best product path is either to land a sanitized, tested gateway/heartbeat handler for `location.update`, or explicitly remove/disable the iOS passive location event path and docs expectations if maintainers decide not to support passive updates.
