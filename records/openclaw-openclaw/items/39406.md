## Summary

Current main still does not implement the requested normal-chat configurable delivery policy. It partially mitigates the original symptom by clearing same-action retry failures and hiding acknowledged failed-action warnings, but unresolved mutating failures remain visible even with `messages.suppressToolErrors`, and tests explicitly preserve success-looking replies plus a mutating warning payload.

## What This Changes

Requested a normal-chat configuration option to suppress transient tool error warnings when the agent recovers or succeeds, with possible `all` / `final` / `off` delivery modes.

## Best Possible Solution

Keep this issue open for maintainer policy review. The best path is to define one documented normal-chat tool-error delivery policy that explains how `messages.suppressToolErrors`, heartbeat-only `suppressToolErrorWarnings`, mutating failure safety, and any `all` / `final` / `off` style option interact, then update config types, schema/help, public docs, and focused payload/tool-handler tests while preserving visible unresolved mutating failures by default.
