## Summary

Current main still does not implement the requested normal-chat `toolErrorDelivery` or global final/off suppression policy. Mutating tool warnings are still surfaced before `messages.suppressToolErrors` can apply, and tests deliberately preserve visible warnings for success-looking assistant replies. Recent merged work only covers acknowledged failure replies and explicitly leaves this broader policy open.

## Best Possible Solution

Keep this issue open for maintainer policy review. The best path is to decide how `messages.suppressToolErrors`, heartbeat-only `suppressToolErrorWarnings`, and any new normal-chat delivery policy should interact, then implement one documented config/schema/help/test change that preserves visible unresolved mutating failures by default while allowing an explicit managed-deployment suppression mode.
