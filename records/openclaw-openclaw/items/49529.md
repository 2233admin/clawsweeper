## Summary

Keep this PR open. Current main still does not implement configurable Gemini safety settings: the provider config type and Zod schema have no `safetySettings` field, docs only cover Gemini thinking/cache behavior, and the runtime payload wrappers only sanitize Gemini thinking payloads or OpenAI-compatible extras. The PR is still a relevant implementation candidate for the open Gemini safety-settings feature, but it needs maintainer review because the supplied PR diff also includes unrelated Z.AI/test-config changes and unresolved wrapper-scope review concerns.

## Best Possible Solution

Keep this PR or a narrowed successor open as the implementation candidate for Gemini safety settings. The best path is to add a strict `models.providers.*.safetySettings` config contract, generated schema/help/docs, and provider-scoped runtime injection for `google-generative-ai` that preserves explicit request payload values, does not affect `google-gemini-cli`, handles custom provider keys, and includes regression tests for stale wrapper/provider-switch behavior. Split unrelated Z.AI/test-config work before merge or resolve it as a separate PR.
