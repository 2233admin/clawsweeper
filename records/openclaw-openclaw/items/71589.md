## Summary

Keep #71589 open. Current main still suppresses only explicit commentary-phase assistant output in the embedded reply path; it does not suppress unphased assistant turns that continue into tool use via `stopReason: "toolUse"` or tool-call content. The PR remains a focused active fix for the #71588 follow-up gap, and the provided discussion shows the author addressed the review notes about `text_end` coverage and snake_case tool-call block variants.

## Best Possible Solution

Keep #71589 open for maintainer review. The best path is to land or refine the narrow embedded-agent suppression for unphased tool-use continuations, preserve explicit `final_answer` delivery, clear already-buffered suppressed text, and retain regression coverage for both `message_end` and `text_end` block-reply modes.
