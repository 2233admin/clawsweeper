## Summary

Keep open. Current main still has the reported suppression gap, and this PR is a plausible narrow fix, but the proposed patch still needs changes because chunked block streaming can emit prelude text before later phase/tool-use suppression can clear buffers, and the branch also carries unrelated QA/status/lockfile hunks that need maintainer scope review.

## What This Changes

The PR extends embedded agent assistant-output suppression to unphased tool-use messages, clears pending visible buffers, preserves explicit final_answer output, adds regression coverage, and also changes qa-lab mock routing, a status-json test expectation, and a lockfile importer.

## Best Possible Solution

Land a narrow embedded-agent suppression fix that covers unphased tool-use messages before any user-visible block, partial, or final reply can escape; keep explicit final_answer delivery; add message_end, text_end, and chunked block-streaming regressions; and split or justify unrelated QA/status/lockfile changes before merge.

## Reproduction Assessment

Yes. The current-main code path is reproducible with an assistant message that has visible text plus stopReason: toolUse and no commentary phase metadata; the handler’s phase-only guard lets that text reach update/end delivery paths, and the PR’s new test fixture models that input shape.

## Solution Assessment

No, not yet. Extending the suppression predicate and preserving final_answer output is the right direction, but the implementation should also prevent chunked block-stream drains while phase/tool-use classification is still pending and should remove or justify unrelated changes.

## Review Findings

Full review comments:

- **[P2] Hold chunks until tool-use suppression is known:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:560-562`
  - body: When an unphased text_delta arrives before the later text_end/message_end reveals commentary or tool use, this branch does not suppress yet, so the chunk can enter block streaming and be drained before suppressPendingAssistantVisibleOutput clears buffers. Because text_end block streaming can emit as the chunker emits, long preludes or low chunk thresholds can still leak user-visible tool-use text.
