## Summary

Keep open for maintainer review. Current main still has the three zero-minute Control UI refresh paths, and `loadSessions` drops non-positive `activeMinutes`; this PR fixes that specific drift and I found no blocking correctness or security issue in the diff.

## What This Changes

The PR switches three Control UI chat-session refresh paths from `activeMinutes: 0` to `CHAT_SESSIONS_ACTIVE_MINUTES` and adds/updates UI tests for chat refresh and session-switch behavior.

## Best Possible Solution

Land this PR or an equivalent scoped UI fix so every current Control UI chat-session refresh path uses the existing shared active-session window while preserving the constant value, `sessions.list` contract, Dreaming filters, cron filtering, and config/schema behavior.

## Reproduction Assessment

Yes. On current main, inspecting the three refresh callsites plus `loadSessions` shows that `activeMinutes: 0` is dropped before `sessions.list`, and the existing session-switch test still expects the old zero-minute value.

## Solution Assessment
