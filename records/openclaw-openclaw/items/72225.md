## Summary

Current main still has the Control UI behavior this PR targets: `CHAT_SESSIONS_ACTIVE_MINUTES` is 120, but the initial chat refresh, session-switch refresh, and chat header session-controls refresh still pass `activeMinutes: 0`. `loadSessions` strips non-positive active windows before calling `sessions.list`, so the PR remains a focused, plausible UI/test fix rather than something already implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review and merge it, or land an equivalent scoped fix: replace the three remaining Control UI `activeMinutes: 0` chat-session refresh callsites with `CHAT_SESSIONS_ACTIVE_MINUTES`, keep the constant value and sessions API behavior unchanged, and retain regression coverage for initial chat refresh and session-switch refresh.
