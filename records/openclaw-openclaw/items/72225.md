## Summary

Current main still has the Control UI behavior this PR targets: the shared chat-session active-window constant is 120 minutes, but three chat session refresh paths still pass `activeMinutes: 0`, and `loadSessions` drops non-positive active windows before calling `sessions.list`. The PR is a narrow UI/test fix, not implemented on main, so it should stay open for maintainer review or merge.

## Best Possible Solution

Keep this PR open for maintainer review and merge it, or land an equivalent scoped fix: replace the three remaining Control UI `activeMinutes: 0` chat-session refresh callsites with `CHAT_SESSIONS_ACTIVE_MINUTES`, keep the constant value and sessions API behavior unchanged, and retain regression coverage for initial chat refresh and session-switch refresh.
