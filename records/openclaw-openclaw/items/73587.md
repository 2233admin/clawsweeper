## Summary

Keep open. The report is recent, concrete, unprotected, and current main still passes `logging.file` through as a raw string before creating the log directory. The title-search related reports cover different gateway crash-loop causes and do not supersede this issue.

## Best Possible Solution

Keep this issue open and fix the logger path boundary in `src/logging/logger.ts`: resolve `logging.file` through the existing home-relative path helper before rolling/non-rolling detection and directory creation. Add focused regression coverage for both `logging.file: "~/.../gateway.log"` and a rolling `~/.../openclaw-YYYY-MM-DD.log` path, then consider a narrow audit of other config-derived filesystem paths that bypass `resolveUserPath`.
