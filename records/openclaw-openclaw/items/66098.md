## Summary

Keep open. Current main has partly superseded the thinking-default side through session rows/defaults, but it still does not expose or consume an effective reasoning default for Web UI chat rendering. The PR targets a real Gateway/Web UI gap, but the proposed patch is not merge-ready because its reasoning resolution can diverge from runtime defaults and its history-derived UI caches can become stale after session/model updates.

## What This Changes

The PR extends `chat.history` with effective thinking/reasoning values, stores those values in Control UI chat state, uses them for chat rendering/default labels, and adds gateway/UI regression tests.

## Best Possible Solution

Keep the newer session-row/default thinking path, then add a backward-compatible authoritative reasoning default contract for Web UI rendering that matches the runtime resolver exactly. The fix should include focused Gateway tests for per-agent and global reasoning defaults plus UI tests proving reasoning visibility updates correctly after session/model changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Match chat history reasoning defaults to runtime:** `src/gateway/server-methods/chat.ts:1667-1674`
  - body: The new `chat.history` reasoning calculation only considers the persisted entry and per-agent config, then can infer reasoning from model catalog capability. Runtime resolution also honors `agents.defaults.reasoningDefault` and otherwise falls back to `off`, so this response can disagree with actual runs and make the Web UI show or hide reasoning incorrectly.
  - confidence: 0.91
- **[P2] Invalidate cached thinking defaults after model changes:** `ui/src/ui/app-render.helpers.ts:682-687`
  - body: The default-label path prefers `state.chatThinkingLevel`, but that value is only refreshed by `chat.history`. Changing the model through the chat header patches and refreshes session rows without reloading history, so the label can keep showing the previous model's default until a later history refresh.
  - confidence: 0.88
- **[P2] Prefer fresh session reasoning over cached history state:** `ui/src/ui/views/chat.ts:1126`
  - body: Reasoning visibility now prioritizes the history-derived `props.reasoningLevel` over the live session row. Session updates from `sessions.patch` or `sessions.changed` can therefore leave reasoning blocks shown or hidden according to stale history state until history is reloaded.
  - confidence: 0.9
