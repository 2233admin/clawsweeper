## Summary

Keep open. The PR targets a real remaining Web UI/Gateway defaults gap, but current main has only partly superseded the thinking-default side and still lacks the proposed reasoning-default contract. The branch itself has blocking correctness issues: its `chat.history` reasoning default can disagree with runtime resolution, and the UI changes introduce stale history-derived caches for thinking/reasoning display.

## What This Changes

The PR extends `chat.history` with effective thinking/reasoning values, stores those values in Control UI chat state, uses them for chat rendering/default labels, and adds Gateway/UI regression tests.

## Best Possible Solution

Keep the newer session-row thinking default path, then add an authoritative reasoning-default surface that matches runtime resolution exactly: session override, per-agent default, global default, then `off`. The Control UI should consume fresh Gateway session/default state for rendering rather than a stale history cache, with focused Gateway and UI tests covering fresh sessions, model/session updates, and #54919.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Match chat history reasoning defaults to runtime:** `src/gateway/server-methods/chat.ts:1669-1676`
  - body: The added `chat.history` reasoning calculation only considers persisted entry/per-agent config, then can infer `on` from model catalog capability. Runtime resolution also honors the global `agents.defaults.reasoningDefault` and otherwise falls back to `off`, so this response can disagree with actual runs and make the Web UI show or hide reasoning incorrectly.
  - confidence: 0.91
- **[P2] Invalidate cached thinking defaults after model changes:** `ui/src/ui/app-render.helpers.ts:679-681`
  - body: The default-label path prefers `state.chatThinkingLevel`, but that value is only refreshed by `chat.history`. Changing the model through the chat header patches the session and refreshes session rows without reloading history, so the label can keep showing the previous model's default until a later history refresh.
  - confidence: 0.88
- **[P2] Prefer fresh session reasoning over cached history state:** `ui/src/ui/views/chat.ts:1126`
  - body: Reasoning visibility now prioritizes the history-derived `props.reasoningLevel` over the live session row. Session updates from `sessions.patch` or `sessions.changed` can therefore leave reasoning blocks shown or hidden according to stale history state until history is reloaded.
  - confidence: 0.9
