## Summary

Current main still has the reported QQBot auth gap. `/bot-clear-storage` is registered without `requireAuth: true`, which keeps it on the pre-dispatch path, while its `--force` handler deletes files from the QQBot media download area. The related #70706 fix only covered `/bot-approve`; the current test coverage also only asserts `/bot-approve` is auth-gated. This PR is still useful and should not be closed as obsolete.

## Best Possible Solution

Keep the PR open and either land its narrow source/test fix or apply the same change directly on main: add `requireAuth: true` to `bot-clear-storage`, keep the framework-registry regression test, and fix the changelog attribution before merge. Consider a follow-up hardening pass for destructive QQBot maintenance commands if maintainers want explicit allowlist defense in depth.
