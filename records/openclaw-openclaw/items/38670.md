## Summary

Keep open. Current main still lacks the agent-scoped internal hook policy surfaces and dispatch metadata this PR proposes, and the cross-referenced plugin-hook work does not supersede it. The PR remains a coherent implementation candidate, but it changes core hook execution policy and has unresolved review concerns around non-agent hook gating, exact event matching, and hot-path policy resolution, so it needs maintainer review before merge.

## What This Changes

The PR adds optional agent-scoped internal hook policy config at `hooks.internal.events`, `agents.defaults.hooks`, and `agents.list[].hooks`, threads config/agent metadata through internal hook dispatch call sites, and updates hook/config docs and tests.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept agent-scoped internal hook policy, revise it so agent overlays apply only to real agent-scoped events, define and document exact-match versus wildcard `events` semantics, preserve global behavior for gateway/non-agent lifecycle hooks, and add focused tests for precedence, named-hook gating, non-agent hook behavior, hot-path resolution, and backward compatibility. If maintainers reject the product direction, close it explicitly with that rationale.
