## Summary

Keep open. Current main still has global-only internal hook dispatch and strict config schemas without agent-scoped hook overlays, so this PR is not implemented or superseded. The branch is a real implementation attempt, but it changes trusted in-process hook dispatch and already has concrete review concerns around non-agent lifecycle hooks, exact event matching, and hot-path policy resolution, so it needs maintainer review before merge.

## What This Changes

This PR adds agent-scoped internal hook policy overlays to config/types/schema, resolves effective policy during internal hook dispatch, threads config and agent metadata through hook call sites, and updates docs and tests.

## Best Possible Solution

If maintainers accept agent-scoped internal hook policy, revise the PR so overlays apply only to real agent-scoped events, preserve global behavior for gateway and other non-agent lifecycle hooks, document event matching semantics, and add focused tests for precedence, named-hook gating, non-agent behavior, hot-path resolution, and backward compatibility. If maintainers reject the direction, close it explicitly with that rationale.
