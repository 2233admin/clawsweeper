## Summary

Keep this PR open. Current main still does not implement Slack `channel-list` or Slack message `search`, so the contribution is not obsolete, but the PR is not merge-ready because the branch still has runtime/config-contract and output-shape blockers, plus an overlapping open Slack search PR that needs maintainer coordination.

## What This Changes

The PR adds Slack `channel-list` and `search` message actions, wires them through Slack action runtime/dispatch/discovery, and updates Slack action config/generated metadata.

## Best Possible Solution

Keep this work in the Slack plugin, but bring it onto current main with a coherent runtime contract: pass resolved config/token state through action helpers, return Slack search results in the shared formatter envelope, advertise search only for the invokable account context, and expose pagination or truncation metadata for channel discovery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pass cfg through to Slack search token resolution:** `extensions/slack/src/actions.ts:334`
  - body: `resolveUserToken()` calls `loadConfig()` from inside a Slack action helper. Current main’s runtime-action contract resolves config at the command/gateway boundary and passes it through, and the guard rejects ambient `loadConfig()` calls in action paths. Pass the already-resolved config/account data through `SlackActionClientOpts` instead.
  - confidence: 0.94
- **[P2] Return Slack search hits in the results envelope:** `extensions/slack/src/action-runtime.ts:494`
  - body: The shared text formatter renders search rows from `payload.results.messages`, but this returns Slack hits as top-level `messages`/`total`. Successful Slack searches will fall back to the generic summary view instead of the expected search-results table.
  - confidence: 0.9
- **[P2] Advertise search only for the invokable account:** `extensions/slack/src/message-actions.ts:55-63`
  - body: When discovery is not scoped to an `accountId`, this advertises `search` if any Slack account has a user token and search enabled. Invocation without `accountId` resolves the default account, so a non-default search-capable account can cause discovery to list an action the default invocation cannot run.
  - confidence: 0.84
- **[P2] Report channel-list truncation to callers:** `extensions/slack/src/actions.ts:302`
  - body: `channel-list` stops at the default 500-channel cap but returns only `channels`. Larger workspaces will silently omit conversations, leaving agents unable to tell whether discovery is complete or needs another page/cursor.
  - confidence: 0.82
