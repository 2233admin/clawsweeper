## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the requested Slack channel-list/search message actions, while the branch has concrete review blockers around runtime config boundaries, search result shape, and incomplete channel-list pagination semantics, plus overlapping Slack search work in #65691.

## What This Changes

The branch adds Slack `channel-list` and `search` message actions, Slack API helpers, dispatch and action discovery plumbing, Slack action config/schema metadata, generated config baseline updates, and a channel-action contract update.

## Best Possible Solution

Land one coherent Slack plugin implementation: keep `channel-list` in the Slack message action surface with explicit pagination or truncation metadata, add Slack `search` through resolved account/config data, return the shared `results.messages` output contract, update focused Slack tests/docs/generated config metadata, and reconcile the search work with #65691 before merge.

## Reproduction Assessment

Yes for the PR blockers: source-level checks reproduce them without a live Slack token. The branch response shape conflicts with the current search formatter contract, and the runtime config boundary guard is visible in main.

## Solution Assessment

No: the feature direction is useful, but this branch is not the best mergeable form today. The safer solution is a rebased Slack-plugin implementation that respects current runtime boundaries, uses the shared search result envelope, and exposes non-silent pagination/truncation behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pass resolved Slack config into search helpers:** `extensions/slack/src/actions.ts:333-347`
  - body: `resolveUserToken()` calls ambient config loading from a Slack action helper when no token override is passed. Current main's runtime-action guard forbids `loadConfig()` in channel action helpers, so the search path should use the already-resolved config/account data passed through `handleSlackAction()`.
  - confidence: 0.94
- **[P2] Return search hits under the results envelope:** `extensions/slack/src/action-runtime.ts:494`
  - body: `handleSlackAction()` returns Slack search data as top-level `messages`/`total`, but the shared CLI formatter renders search rows from `payload.results.messages`. Successful Slack searches will skip the search-results table unless this returns the shared `results` envelope.
  - confidence: 0.9
- **[P2] Advertise search only for the invokable account:** `extensions/slack/src/message-actions.ts:55-63`
  - body: Unscoped discovery advertises `search` if any Slack account has a user token and search enabled, while invocation without `accountId` resolves the default account. A non-default search-capable account can expose an action the default invocation cannot run.
  - confidence: 0.84
- **[P2] Report channel-list truncation to callers:** `extensions/slack/src/actions.ts:302`
  - body: `channel-list` stops at the default 500-channel cap but returns only `channels`. Larger workspaces will be silently truncated, leaving callers unable to tell whether discovery is complete or needs another page/cursor.
  - confidence: 0.82
