## Summary

Keep open. Current main appears to have addressed the hard failure path by treating `streamTo` as an ACP-only field that is ignored for native subagent spawns, but it does not implement the requested realtime `streamTo="parent"` behavior for `runtime="subagent"`. The docs, schema text, dispatch path, and tests all preserve ACP-only semantics for `streamTo`, while native subagents still run as background tasks with completion announcement rather than parent streaming.

## Best Possible Solution

Keep this open for maintainer design. Preserve the current compatibility behavior that ignores ACP-only `streamTo` for native subagents, then decide whether native subagent progress should be added through task progress notifications, a parent-session heartbeat relay, or a broader gateway event stream with clear visibility and isolation rules.
