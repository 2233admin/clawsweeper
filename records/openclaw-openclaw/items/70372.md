## Summary

Current main still does not implement this PR's Anthropic/Bedrock unphased-thinking fallback. Anthropic streaming emits thinking and text events without assistant phase metadata, while the delivery handler only switches to phase-aware visible-text extraction when `resolveAssistantMessagePhase(...)` returns a phase. Related commentary-leak fixes are adjacent, but they do not supersede this PR.

## Best Possible Solution

Keep this PR open as the implementation vehicle. Rebase it onto current main, scope or prove the derived final-answer fallback so it cannot affect providers with late phase metadata, add a focused subscription regression test for Anthropic/Bedrock-style `thinking_*` events followed by unphased text, and coordinate overlap with #70344 rather than closing this as obsolete.
