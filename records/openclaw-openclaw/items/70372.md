## Summary

Keep #70372 open. Current main has the phase-aware delivery machinery and commentary-phase suppression from related work, but it does not implement this PR's specific `hadThinkingInMessage` fallback for unphased Anthropic/Bedrock thinking streams. The PR also has an unresolved P1 review concern: deriving `final_answer` from any prior thinking event could misclassify providers whose phase metadata arrives later, so the work is not safely obsolete or ready to close as implemented.

## Best Possible Solution

Keep the PR open, but require a tighter implementation before merge: add a regression for Anthropic/Bedrock-style thinking events followed by unphased text, scope any synthetic `final_answer` derivation to providers/transports known not to emit phase metadata, preserve the existing OpenAI late-phase buffering behavior, and coordinate with #70344 so one canonical suppression path owns missing-phase tool-use/prelude leaks.
