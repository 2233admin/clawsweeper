## Summary

Current main does not implement this PR's intended user-visible change: the Sessions page still has no History shortcut, and the Control UI handler has no standalone `/history` or `/history.js` viewer route. The PR is not obsolete, but it should not be auto-closed or auto-landed because the proposed transcript-serving route is security-sensitive and the review thread still flags unresolved data-exposure and robustness concerns.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The useful product direction is a Sessions-page shortcut to history, but transcript data should be served through the existing authenticated session-history path or an equivalent scoped API, with basePath-safe UI wiring, archive-file support, bounded reads before loading file contents, and focused tests.
