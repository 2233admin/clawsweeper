## Summary

Keep open. Current main still recurses into the first object-valued `anyOf`/`oneOf` variant during Control UI config coercion, so this PR's discriminator-aware object variant selection remains useful unmerged work. The PR's follow-up commit also appears to address the review concern about unconstrained variants being treated as discriminator matches.

## Best Possible Solution

Keep this PR open and review or land the discriminator-aware object-variant selection in `ui/src/ui/controllers/config/form-coerce.ts`. The implementation should prefer object variants with matching top-level `const` constraints, fall back to the first object variant when no discriminator match exists, preserve existing primitive union and array fallback behavior, and include focused `anyOf`/`oneOf` regression coverage including the unconstrained-variant edge case raised in review.
