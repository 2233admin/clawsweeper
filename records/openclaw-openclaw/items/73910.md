## Summary

Keep open. Current main has the Codex ACP timeout guard, but the isolated CODEX_HOME path still writes only generated config and still has a regression test asserting source Codex auth is not copied; the remaining auth fix is already tracked by open PR #73943 and should not be closed before merge and security review.

## What This Changes

The issue asks for managed Codex ACP sessions to reuse or bridge Codex authentication for the isolated CODEX_HOME and to avoid timeout config options rejected by Codex ACP.

## Best Possible Solution

Keep the timeout guard on main, then review and land or replace #73943 as a narrow Codex ACP auth fix: bridge only the minimum required auth material or document a supported isolated-home login flow, preserve generated config isolation, and add regression coverage plus ACP docs.

## Reproduction Assessment

Yes. The source-level reproduction is clear: provide a source CODEX_HOME with auth.json, run prepareAcpxCodexAuthConfig, and current main still generates an isolated codex-home/wrapper without bridging auth while tests assert no auth copy.

## Solution Assessment

Unclear until the open auth PR receives maintainer security review. A narrow auth.json bridge or documented isolated-home login flow is the maintainable direction, but it must avoid copying source config hooks or broader Codex state.
