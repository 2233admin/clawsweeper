## Summary

Keep open. Current main has the Codex ACP timeout guard, but the isolated CODEX_HOME auth bridge is still missing, and open PR #73943 explicitly tracks the remaining fix with closing syntax. Per policy, this issue should stay open until that PR or an equivalent maintainer-reviewed auth fix lands.

## What This Changes

The issue requests that managed Codex ACP sessions safely reuse or bridge Codex auth for OpenClaw's isolated CODEX_HOME and stop forwarding unsupported timeout config to Codex ACP.

## Best Possible Solution

Keep the existing Codex ACP timeout guard, then land or replace #73943 with a narrow maintainer-reviewed auth fix: bridge only the minimum Codex auth material or document a supported isolated-home login flow, preserve generated config isolation, add regression coverage, and update the ACP docs.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main creates an isolated Codex ACP home and wrapper without auth.json, and the existing test explicitly asserts no source Codex auth is copied while the timeout guard is already covered.

## Solution Assessment

Unclear until the auth bridge receives maintainer security review. The timeout guard is the right narrow fix for the unsupported config option; the remaining auth part should be solved by a tightly scoped auth.json bridge or supported isolated-home login flow, not by importing the user's full Codex config.
