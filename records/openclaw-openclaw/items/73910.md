## Summary

Keep open. Current main has the Codex ACP timeout guard, but it still launches Codex ACP with an isolated generated CODEX_HOME that contains only OpenClaw's generated config and the tests still assert that source Codex auth is not bridged; the remaining auth work is already paired with open PR #73943.

## What This Changes

OpenClaw-managed Codex ACP should either safely bridge or document Codex authentication for its isolated CODEX_HOME and avoid sending unsupported timeout config options.

## Best Possible Solution

Keep the timeout guard already on main, then review and land or replace PR #73943 with a narrow Codex ACP auth fix that exposes only the minimum required auth material, keeps source config hooks isolated, adds regression coverage, and updates the ACP docs.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: prepareAcpxCodexAuthConfig still generates a wrapper that forces CODEX_HOME to state/acpx/codex-home while the tests assert a source auth.json is not copied or bridged.

## Solution Assessment

Unclear until the auth bridge receives maintainer security review. Current main is only a partial fix because it suppresses unsupported timeout config for Codex ACP but leaves the isolated-home authentication failure intact.
