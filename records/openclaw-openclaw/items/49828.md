## Summary

Keep open. Current main still renders Control UI WebChat agent/session switching as a grouped native dropdown in the chat header, not as a persistent WhatsApp/Telegram-style agent conversation sidebar. The existing chat sidebar is for rendered markdown/canvas/tool content, and the related dropdown fixes do not supersede this distinct multi-agent UX request.

## Best Possible Solution

Keep this issue open for a focused Control UI chat improvement: add a persistent agent/session sidebar fed by existing `agents.list`, `sessions.list`, and optionally `sessions.preview`; switch through the existing session-switch path; keep model/thinking controls intact; define responsive/mobile fallback behavior; and cover multi-agent WebChat behavior with UI tests plus docs/changelog updates.
