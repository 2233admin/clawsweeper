## Summary

Keep open. Current main still has the reported Windows dotenv and registry-helper trust-boundary gap, and the latest PR head only adds the dotenv blocklist entries while preserving env-derived reg.exe selection. Because this is security-sensitive command-resolution work with overlapping open hardening PRs, it needs maintainer/security review and source-branch correction rather than cleanup closure.

## What This Changes

This PR adds SystemRoot/WINDIR workspace dotenv blocklist coverage and adjusts Windows registry-probing tests/call sites around locateWindowsRegExe.

## Best Possible Solution

Land a maintainer-approved security fix that blocks untrusted workspace .env from setting Windows system-root variables and makes registry probing execute only a trusted System32 reg.exe path, while reconciling the overlapping taskkill and ACL hardening work before merge.
