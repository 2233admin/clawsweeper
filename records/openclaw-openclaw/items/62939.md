## Summary

Keep open. Current main still has partial prompt-injection hardening for web/file/hook content and marker spoofing, but it does not implement the requested shared trust-boundary contract for non-owner current message bodies, generic/open-world tool results, or stable system-prompt guidance. The narrow linked implementation PR is closed unmerged, and the broader CaMeL PR remains open rather than shipped.

## What This Changes

The issue requests core prompt-injection hardening that marks externally sourced message and tool-result content as untrusted data, adds stable system-prompt guidance, and validates obvious injection markers before model context injection.

## Best Possible Solution

Implement or explicitly supersede this with a core-owned prompt-injection boundary design: stable pre-cache guidance, tested non-owner current-message marking during prompt assembly, shared classification/wrapping for open-world tool results, and provider-shape regression coverage across OpenAI, Anthropic, and Google while preserving existing external-content sanitizers.
