## Summary

Keep open: current main still lacks the bundled `dist/` template fallback, so the PR targets a real remaining bug, but the branch needs changes because its test seam changes an existing fallback contract and the new regression test can still discover the real repo through `process.argv[1]`.

## What This Changes

The PR changes workspace template directory resolution to try module-relative source and bundled `dist` docs paths before cwd, and adds a bundled-layout resolver regression test.

## Best Possible Solution

Keep the PR open and repair it before merge: preserve the production resolver’s reliable package-root and real-module fallbacks, add a controlled seam for bundled-layout coverage, force root discovery to be null in the new test, and prefer the packaged `../docs/reference/templates` candidate before the source-layout overshoot when running from root `dist` artifacts.

Yes. The original bug can be reproduced by simulating a bundled module URL with package-root and cwd template lookups missing; the PR-branch test defects can be reproduced by walking the candidate order through the existing temp package-root test and the new test’s default `process.argv[1]` fallback.

## Solution Assessment

No. The production direction is narrow and appropriate, but this patch is not yet the best form because the injected `moduleUrl` changes existing fallback semantics and the new regression test does not isolate `argv1`; a small resolver/test-seam adjustment is safer.
