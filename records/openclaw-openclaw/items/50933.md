## Summary

Keep open. Current main still does not implement this PR’s intended session-memory behavior for automatic idle/daily resets: the bundled hook is registered only for manual command events, its handler still ignores non-command events, and automatic stale rollover currently only emits plugin lifecycle hooks after transcript archival. The PR remains potentially useful, but it needs review/rebase work and the latest review concerns resolved before landing.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a focused implementation that emits an automatic reset signal early enough for session-memory to read the previous transcript, updates the bundled hook metadata/handler/docs/tests, and resolves the latest trigger-matching and latency review concerns before merge.
