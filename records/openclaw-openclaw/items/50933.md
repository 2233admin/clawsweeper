## Summary

Keep this PR open. Current main still does not implement this PR's intended automatic session-memory reset behavior: daily/idle rollover creates a new session and archives the old transcript, but it does not emit `session:idle_reset` or `session:daily_reset`, and the bundled `session-memory` hook remains registered and implemented only for manual `command:new` and `command:reset` events.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best product path is a focused implementation that emits an automatic reset signal early enough for `session-memory` to read the previous transcript, updates the bundled hook metadata/docs/tests, avoids duplicate manual-reset side effects, keeps hook execution bounded or fire-and-forget, and either lands here or is explicitly superseded by a selected replacement such as #61675.
