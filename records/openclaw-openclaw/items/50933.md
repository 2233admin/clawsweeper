## Summary

Keep open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still does not implement this PR's intended behavior: automatic daily/idle session rollover does not emit internal session reset hook events, and the bundled session-memory hook remains command-only. The PR remains potentially useful, but it still needs maintainer review or an explicitly chosen superseding implementation before cleanup can close it.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a focused implementation that emits an automatic reset signal early enough for session-memory to read the previous transcript, updates the bundled hook metadata/handler/docs/tests, and resolves the trigger-normalization, text-command, duplicate-execution, and latency concerns before merge. If maintainers choose a newer PR as canonical, close this only after that replacement is explicitly selected or merged.
