## Summary

Keep this issue open. Current main still has wall-clock subagent run timeouts, manual kill/steer controls, stale unended-run cleanup, and restart orphan recovery, but it does not implement last-tool-call activity tracking, a nudge stage, automatic kill escalation, stall config fields, or sessions_spawn stall overrides. The related implementation PR #43017 is closed unmerged, while #39127 and #39141 remain broader open activity/watchdog design work rather than a shipped subagent recovery implementation.

## Best Possible Solution

Keep this issue open until maintainers either implement activity-based subagent stall recovery or explicitly consolidate it into the canonical session activity/watchdog design. A good implementation would add strict config and sessions_spawn override fields, track deterministic child tool/run activity, inject auditable nudges into child sessions, terminate through the existing subagent kill/cancellation path after the kill threshold, document the behavior, and cover config/runtime/docs with focused tests.
