## Summary

Keep this issue open. Current main still provides wall-clock `runTimeoutSeconds`, manual subagent kill/steer paths, and coarse stale-run cleanup, but it does not implement per-subagent last-tool-call tracking, a nudge phase, automatic kill escalation after inactivity, stall config fields, or `sessions_spawn` stall overrides. The related implementation PR #43017 is closed unmerged, while #39127 and #39141 remain broader open activity/watchdog design items rather than shipped behavior.

## What This Changes

Request activity-based two-stage stall recovery for sub-agents, with configurable nudge and kill thresholds plus per-spawn overrides.

## Best Possible Solution

Keep this issue open until maintainers either implement a targeted activity-based subagent stall recovery path or explicitly consolidate it into the canonical #39127/#39141 session activity/watchdog design. A good implementation should add strict config and `sessions_spawn` override fields, track deterministic child activity, inject auditable nudges into child sessions, terminate through the existing subagent kill/cancellation path after the kill threshold, document the behavior, and cover config/runtime/docs with focused tests.
