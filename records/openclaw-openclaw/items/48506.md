## Summary

Current main still does not expose resolved target context on `message_sending`. The hook event type and delivery call sites provide `to`, `content`, typed reply/thread routing, and limited metadata, while target resolution and cross-context policy remain internal outbound helpers. This is a valid public plugin API/security-design request and should stay open for maintainer review, especially alongside related open hook-enrichment work.

## Best Possible Solution

Keep this open for maintainer API/security review. The best implementation path is an additive, optional `PluginHookMessageSendingEvent` target context populated from existing target-resolution and cross-context logic, documented in the plugin hook docs, and tested across generic outbound delivery plus channel-specific delivery paths. Coordinate the shape with #48503 and #48517 rather than closing this as stale.
