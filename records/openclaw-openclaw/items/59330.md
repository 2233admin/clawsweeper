## Summary

Keep this issue open. Current main still has the reported failure path: safe-bin normalization leaves own properties with `undefined`, raw redaction round-trip still compares restored raw text against the materialized runtime config, and Control UI still disables Raw mode when redaction returns `raw: null`. The linked implementation PR #59336 remains open and unmerged, so cleanup should wait for that PR or a maintainer replacement to land.

## Best Possible Solution

Keep this issue open and use #59336, or a maintainer-owned replacement, as the implementation path. The fix should delete empty safe-bin keys instead of assigning `undefined`, compare raw restoration against the authored/source snapshot or an intentionally normalized JSON-serializable view rather than the materialized runtime config, and add focused regression coverage for safe-bin normalization, materialized snapshot differences, and the comment-reported substring redaction corruption.
