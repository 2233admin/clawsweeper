## Summary

Keep open. Current main has Telegram group/supergroup handlers and exact-ID group migration support, but the reported migrated-group path still matches a no-op branch for unconfigured old group IDs, and Telegram group directory output remains config-backed rather than live-discovered. The newer comments also show this is not a one-off setup report: the reporter reproduced it live after BotFather/admin/privacy changes, and another user reports DMs work while ordinary group messages do not trigger replies.

## What This Changes

The issue requests that Telegram group and supergroup chats be discovered, listed, ingested, and responded to when direct Telegram messages already work.

## Best Possible Solution

Define Telegram's intended behavior for newly observed and migrated groups: either keep directories config-backed and improve diagnostics/docs/doctor guidance for the new `-100...` ID, or add a read-only observed-groups surface that does not grant access by itself. Runtime changes should preserve group allowlist and sender authorization boundaries while adding focused coverage for migration, wildcard config, directory output, and group ingress.
