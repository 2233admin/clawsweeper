## Summary

Keep open. Current main supports Telegram group message handling, but the reported group-discovery expectation is not implemented: Telegram directory groups are read from configured group IDs only, and wildcard-only or newly observed groups will still produce an empty directory list. The response failure may still be a real bug, but the issue needs maintainer review or redacted config/log evidence before it can be closed or safely automated.

## What This Changes

The issue requests that Telegram groups be discovered/listed and that group messages be ingested/responded to after the bot is added to a Telegram group.

## Best Possible Solution

Define the desired Telegram product behavior first: either keep group IDs explicit and improve docs/status/doctor guidance for finding numeric group IDs, or add a read-only live/seen-groups directory surface that does not silently allow groups. Separately, reproduce the group-response failure on current main with redacted config and verbose logs before changing ingress policy.
