## Summary

Keep open. Current main still leaves the reported mcporter-backed QMD first-query failure plausible: daemon startup is invoked without discarding stdout, the mcporter call result is parsed with `JSON.parse(result.stdout)`, and tests only cover empty daemon stdout plus clean JSON call stdout.

## Best Possible Solution

Keep the issue open and patch `extensions/memory-core/src/memory/qmd-manager.ts`: expose a `discardOutput` or `discardStdout` option on `runMcporter()`, use it for `daemon start`, and make `runQmdSearchViaMcporter()` consume only the structured mcporter JSON payload when stdout contains leading mcporter log lines. Add a regression test for the first `startDaemon=true` search where daemon startup emits `[mcporter] ...` before a clean `mcporter call --output json` result, plus a malformed-output test so the parser does not silently accept unrelated text.
