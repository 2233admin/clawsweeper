## Summary

Keep this issue open. Current main still uses the builtin SQLite memory backend by default, and forced/full reindexing still builds a fresh random temp database and swaps it only after the whole build succeeds. There is no durable checkpoint cursor, reusable temp DB discovery, resume-on-restart config, incomplete-generation status, or interrupted-reindex regression coverage.

## Best Possible Solution

Keep this issue open until builtin memory-core full reindex has a durable checkpoint/resume design: persisted generation/checkpoint state or safe temp DB reuse, restart resume from committed file/chunk progress, search/status handling that never treats incomplete generations as clean, config/status/docs updates if controls are added, and regression coverage for interrupted forced/full reindex behavior.
