## Summary

Keep open. Current main has partial protection when an explicit sub-agent model ref is split into provider/model, but a Google provider config can still preserve an inline model id like `google/gemini-3-flash-preview` into the native Google transport, where the URL builder would turn it into `models/google/...` and plausibly produce the reported 404. The latest issue comment adds a concrete provider-prefixed Google config, and related prefix-normalization PRs are still open rather than shipped.

## What This Changes

The issue asks OpenClaw to stop Google Gemini provider-prefixed model ids from producing 404s when sub-agents are spawned or sessions are reset.

## Best Possible Solution

Keep this issue open and consolidate the Google prefix work into one narrow fix that strips only redundant active-provider prefixes before Google model matching and native transport URL construction, while preserving valid nested model ids for wrapper providers and adding regression coverage for sub-agent spawn/reset plus inline Google configured models whose ids are written as `google/<model>`.
