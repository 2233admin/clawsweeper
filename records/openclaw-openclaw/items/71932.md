## Summary

Keep open. The current main branch still has a credible Google native-transport path where an inline configured model id shaped as `google/<model>` is preserved into the request URL as `models/google/<model>`, and the newer issue discussion confirms that exact config shape on a recent release. ## What This Changes

Fix Google Generative AI model-id normalization so sub-agents and session reset do not send provider-prefixed Gemini model ids to the native Google API.

## Best Possible Solution

Land one consolidated Google-owned normalization fix that strips only redundant active-provider prefixes before Google model matching and native Google transport URL construction, while preserving legitimate nested model ids for wrapper providers. Add regression coverage for sub-agent spawn/reset and inline `models.providers.google.models[].id` values written as `google/<model>`.
