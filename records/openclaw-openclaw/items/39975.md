## Summary

Keep PR #39975 open. Current main still serializes `message_reference` in the Discord Execute Webhook request body when `replyTo` is set, and current Discord webhook docs still do not list that field for Execute Webhook payloads.

## Best Possible Solution

Keep the PR open or replace it with an equivalent refreshed patch against `extensions/discord/src/send.outbound.ts`: remove `message_reference` only from `sendWebhookMessageDiscord`, preserve bot-token/channel reply behavior in `sendMessageDiscord` and related media/component paths, and add a webhook request-body regression test for `replyTo`.
