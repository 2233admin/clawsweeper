## Summary

Keep open. Current main still lacks the requested per-evaluation precheck observability: the helper returns `route: "fits"` and the requested token-budget fields, but the embedded runner only logs `[context-overflow-precheck]` when it truncates or compacts. The cache-ttl/large-context behavior also remains plausible enough to investigate rather than close as implemented.

## What This Changes

The issue requests reliable preemptive auto-compaction for large-context Anthropic sessions using cache-ttl pruning, plus debug logging for every precheck decision including `route: "fits"`.

## Best Possible Solution

Add a debug-level precheck decision log or structured diagnostic immediately after `shouldPreemptivelyCompactBeforePrompt()` in the embedded runner, including the route and token-budget fields requested here, then use that evidence to decide whether cache-ttl pruning or context assembly needs a separate compaction-threshold change. Coordinate with the open prompt-authority PR so diagnostics continue to describe the prompt surface actually being admitted.
