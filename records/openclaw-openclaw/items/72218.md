## Summary

Keep this PR open. Current main still has the `isHeartbeat` early return in `buildReplyPayloads`, so heartbeat final payloads bypass the existing `HEARTBEAT_OK` stripping path. The PR is a narrow source/test fix for that live gap, while #72253 and #69492 remain separate broader exec/system-event leakage work.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent narrow change in `src/auto-reply/reply/agent-runner-payloads.ts` that applies existing `HEARTBEAT_OK` stripping to heartbeat final payloads, preserves media-bearing payloads after text stripping, keeps Bun socket error formatting scoped to non-heartbeat runs, and adds focused `isHeartbeat: true` regression tests in `src/auto-reply/reply/agent-runner-payloads.test.ts`. Treat #72253 and #69492 as separate broader follow-ups.
