## Summary

Keep #72218 open. Current main still has the `isHeartbeat` early-return in `buildReplyPayloads`, so heartbeat final payloads bypass the existing `HEARTBEAT_OK` token stripping. The dedicated heartbeat runner normalizes its selected reply before outbound delivery, but the shared final-payload helper still violates the documented heartbeat reply contract and lacks the focused regression tests proposed by this PR.

## Best Possible Solution

Keep #72218 open for maintainer review or land an equivalent narrow change in `src/auto-reply/reply/agent-runner-payloads.ts`: remove the heartbeat bypass for `HEARTBEAT_OK` stripping, keep Bun socket error formatting scoped to non-heartbeat runs, preserve media-bearing heartbeat payloads after token stripping, and add focused `isHeartbeat: true` tests in `src/auto-reply/reply/agent-runner-payloads.test.ts`. Treat #71860, #72253, and #69492 as adjacent broader exec/system-event leakage work, not substitutes for this helper fix.
