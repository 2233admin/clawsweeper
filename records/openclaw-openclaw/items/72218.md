## Summary

Keep this PR open. Current main still returns raw heartbeat final payloads from buildReplyPayloads when isHeartbeat is true, so HEARTBEAT_OK skips the existing final-reply token sanitation path. The PR is a narrow source/test change for that live gap, and the separate exec/system-event work remains tracked elsewhere.

Keep this PR open for maintainer review, or land an equivalent narrow change in src/auto-reply/reply/agent-runner-payloads.ts that applies HEARTBEAT_OK stripping to heartbeat final payloads, preserves media-bearing payloads after token stripping, keeps Bun socket error formatting scoped to non-heartbeat runs, and adds focused isHeartbeat true regression tests in src/auto-reply/reply/agent-runner-payloads.test.ts.
