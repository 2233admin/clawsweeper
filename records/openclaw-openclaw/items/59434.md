## Summary

Keep #59434 open. Current main still has the exact guard this PR removes, and the existing regression test still expects the reasoning payload to be sent when the final heartbeat reply is HEARTBEAT_OK. The linked issue #59381 was later closed by the reporter as fixed, but I did not find a corresponding implementation on main; the PR remains a focused candidate fix, with only minor cleanup follow-up noted by Greptile.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch: broaden the heartbeat OK early return to `if (shouldSkipMain)`, update the existing reasoning-plus-HEARTBEAT_OK test to expect zero sends, add the edge case where reasoning text itself contains HEARTBEAT_OK, and optionally remove the now-unreachable `shouldSkipMain` branches before running the changed gate.
