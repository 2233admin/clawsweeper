## Summary

Keep this PR open. Current main has related observability work, including OTEL export, model-call hooks, and trusted W3C traceparent propagation, but it still does not implement the PR's opt-in diagnostics.sessionTracing config or configurable x-session-id / x-session-name style headers on outgoing LLM provider requests. The branch remains relevant and needs maintainer review for config contract, docs/changelog, privacy, and header-hardening details.

## Best Possible Solution

Keep this PR open for maintainer review as a still-relevant narrow feature. If core-level session tracing headers are desired, land them behind an opt-in diagnostics config, reuse the existing provider header normalization/forbidden-header behavior, preserve trusted traceparent precedence, document the privacy tradeoff, add docs/changelog, and keep focused tests for disabled-by-default behavior, caller-header precedence, invalid session keys, subagent/run IDs, and privacy-sensitive fallbacks. If maintainers decide the behavior belongs outside core, route that explicitly to a provider or observability plugin design rather than closing as already implemented.
