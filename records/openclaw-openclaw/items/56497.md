## Summary

Keep this PR open. Current main does not implement Volcengine Ark Responses session caching: the bundled Volcengine plugin has no cache wrapper/tests/docs, and both Volcengine provider catalog entries still default to `openai-completions`. The PR is a focused provider-local feature and uses an existing `wrapStreamFn` seam, but the latest diff and review context still show correctness risks that need normal maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for normal review. The best path is to rework the Volcengine provider-local cache wrapper against the existing `wrapStreamFn` seam, address the open correctness concerns with targeted tests for prompt/tool invalidation, stale response-id retry behavior, and mixed provider/model sessions, then update the English Volcengine docs if the behavior lands.
