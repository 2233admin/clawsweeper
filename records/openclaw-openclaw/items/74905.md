## Summary

Keep open. Current main still has the direct APNs `http2.connect` gap this PR targets, but this draft maintainer/security PR still needs a branch update because the new APNs helper bypasses the shared `NO_PROXY`-aware proxy decision contract. https://github.com/openclaw/openclaw/pull/74905

## What This Changes

The PR replaces direct APNs `http2.connect` with a proxy-aware HTTP CONNECT/TLS helper, adds APNs/tunnel regression tests, and adds docs plus an OpenGrep rule for raw HTTP/2 callsites.

## Best Possible Solution

Keep the narrow APNs HTTP/2 session seam, but gate tunneling with the shared `shouldUseEnvHttpProxyForUrl(authority, env)` decision and add a focused APNs regression for `HTTPS_PROXY` plus matching `NO_PROXY` before normal maintainer/security review decides whether to merge.

## Reproduction Assessment

Yes. On the PR diff, `HTTPS_PROXY=http://proxy.test:8080` with `NO_PROXY=api.push.apple.com` still reaches the CONNECT helper because the new code checks only `resolveEnvHttpProxyUrl`; current main tests establish the shared helper should return false for matching `NO_PROXY`.

## Solution Assessment

No. The helper boundary is the right place for direct APNs HTTP/2 proxying, but the proxy-selection step should reuse the existing NO_PROXY-aware contract before resolving and opening a CONNECT tunnel.
