## Summary

Keep open. The work is still relevant because current main bypasses managed proxy routing for direct APNs HTTP/2, but this draft protected PR is not mergeable as-is: the latest head removes the `node:http2` import from `src/infra/push-apns.ts` while the file still references `http2.constants` in the timeout path.

## What This Changes

The PR adds an APNs HTTP/2 CONNECT/TLS wrapper backed by active managed proxy state, switches direct APNs sends to that wrapper, and adds tunnel/APNs/proxy lifecycle tests plus raw-http2 lint/OpenGrep guardrails and network docs.

## Best Possible Solution

Keep the APNs proxy logic at the low-level session wrapper, restore the missing `http2` constants binding or switch the timeout close call to an imported constant, then have maintainers run the targeted APNs/tunnel tests plus the changed gate before merging.

## Reproduction Assessment

Yes. Current main is reproducible by source inspection: direct APNs sends call `http2.connect(authority)` directly, and the PR-head defect is reproducible by static inspection because `http2.constants.NGHTTP2_CANCEL` remains after the `node:http2` import was removed.

## Solution Assessment

No, not as currently patched. The wrapper/active-managed-proxy direction is the narrow maintainable fix, but the branch first needs the missing `http2` binding repaired and then maintainer/security validation.
