## Summary

Keep open. Current main still lets an explicit root `--profile` coexist with an inherited `OPENCLAW_LAUNCHD_LABEL`; the macOS launchd lifecycle path resolves that inherited label before the profile-derived label and `gateway restart` passes `process.env` into the restart call. The linked fix PR #47594 is closed and unmerged, and #65643 remains open for the same stale-label root cause on `gateway status`.

## What This Changes

The issue requests that `openclaw --profile <name> gateway restart` from an agent session target the requested profile's gateway instead of the current session's inherited gateway identity.

## Best Possible Solution

Explicit root profile selection should produce a lifecycle environment where profile-derived service identity wins for `gateway restart`, `status`, and related service operations, while custom launchd labels remain valid for managed service contexts that did not pass a root profile.
