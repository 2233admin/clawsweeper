## Summary

Keep open. Current main still accepts an explicit root `--profile` but leaves any inherited `OPENCLAW_LAUNCHD_LABEL` in place, and the macOS launchd restart/status helpers still resolve that label before the profile-derived service identity. The earlier fix PR is closed and unmerged, while the related status issue remains open under the same stale-label root cause.

## What This Changes

The issue asks for `openclaw --profile <name> gateway restart` from an agent or gateway session to restart the requested profile’s macOS LaunchAgent instead of the current session’s inherited gateway.

## Best Possible Solution

Explicit root `--profile` selection should make the profile-derived service identity win across launchd lifecycle, status, plist resolution, and detached restart handoff paths, while custom launchd labels remain honored when no explicit root profile was requested.
