## Summary

Keep open. The PR is a useful partial fix, but the status-gated raw-output suppression does not cover the outcome-less frozen-result refresh path, so successful subagent captures can still preserve raw tool output in current main once this diff is applied.

## What This Changes

The PR adds an `ok`-status guard before subagent output selection falls back to `latestRawText` and adds a regression test for successful tool-only histories.

## Best Possible Solution

Land one canonical subagent announce fix that prevents raw tool/toolResult text from becoming a successful completion result across direct announce, initial freeze, and frozen-result refresh paths, while preserving timeout/error partial-progress behavior and updating docs, tests, and changelog for the resulting contract.

## Reproduction Assessment

Yes. A source-level reproduction is a successful tool-only history whose latest message is a tool result: current main selects `latestRawText`, and the PR only suppresses that when an `ok` outcome is passed. The remaining gap is reproducible through `refreshFrozenResultFromSession`, which captures without outcome.

## Solution Assessment

No. The PR is directionally right, but the narrow `outcome?.status === "ok"` guard is not the complete fix because successful capture refreshes can still arrive without outcome; the safer fix is to make the capture/refresh contract carry outcome or otherwise suppress raw fallback for successful completion captures consistently.
