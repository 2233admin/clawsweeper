## Summary

Keep this PR open. Current main still has the behavior the PR targets: `selectSubagentOutputText` falls back to `snapshot.latestRawText` for a successful subagent run with no assistant text, so a tool-only history can still surface raw tool output as the completion result. The PR is a narrow code/test change, not implemented on main, and needs normal maintainer review rather than cleanup closure.

## What This Changes

The PR changes subagent announce output selection so successful runs without assistant completion do not use raw toolResult/tool text, and adds regression coverage for tool-only histories while preserving timeout partial-progress behavior.

## Best Possible Solution

Keep the PR open for normal maintainer review. The likely best fix is the narrow selector change in `src/agents/subagent-announce-output.ts` plus the regression test, with maintainers confirming the success-vs-timeout/error semantics and updating `docs/tools/subagents.md` if successful raw tool fallback is no longer intended.
