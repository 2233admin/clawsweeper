## What This Changes

The PR updates shared agent runtime policy resolution to synthesize legacy `embeddedHarness` into `agentRuntime`, with tests for harness selection, startup plugin activation, and `sessions_spawn` native-versus-ACP routing.

## Best Possible Solution

Land this PR or an equivalent shared-resolver compatibility change after maintainer review, preserving explicit `agentRuntime` precedence, legacy field compatibility, plugin enable/disable semantics, and the native Codex versus explicit ACP routing boundary.

## Reproduction Assessment

Yes. Against current main, a config that only sets `embeddedHarness: { runtime: "codex", fallback: "none" }` reproduces by source inspection because the shared resolver ignores `embeddedHarness`, and both harness selection and startup activation consume that resolver.

## Solution Assessment

Yes. Updating the shared resolver is the narrowest maintainable solution because the affected consumers already centralize runtime policy there, and the PR keeps `sessions_spawn` ACP dispatch explicitly gated on `runtime: "acp"`.
