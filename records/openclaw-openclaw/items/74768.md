## What This Changes

The PR extends agent runtime policy resolution to merge legacy `embeddedHarness` runtime/fallback values into missing `agentRuntime` fields and adds tests for native Codex harness selection, startup plugin activation, and `sessions_spawn` routing.

## Best Possible Solution

Land this PR or an equivalent central resolver compatibility change after maintainer review, preserving explicit `agentRuntime` precedence, legacy fill-missing behavior, plugin disablement semantics, and the native Codex versus explicit ACP routing split.

## Reproduction Assessment

Yes. A config that only sets `agents.defaults.embeddedHarness` or `agents.list[].embeddedHarness` is still accepted by current types/schema and doctor migration, but current main's shared resolver ignores it before harness selection and startup activation consume policy.

## Solution Assessment

Yes. Centralizing compatibility in `resolveAgentRuntimePolicy` is the narrowest maintainable solution because harness selection, model-runtime aliases, startup plugin activation, and agent metadata already depend on that resolver instead of duplicating legacy fallback logic at each call site.
