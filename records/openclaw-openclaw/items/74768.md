## Summary

Keep this PR open for maintainer review. Current main still ignores legacy `embeddedHarness` in the shared agent runtime policy resolver even though the config type, schema, and doctor migration still accept and migrate that legacy surface; the PR is a narrow compatibility fix with no concrete automated-review defect found.

## What This Changes

The branch updates `resolveAgentRuntimePolicy` to merge deprecated `embeddedHarness` runtime/fallback values into missing `agentRuntime` fields and adds tests for harness selection, `sessions_spawn` routing, and gateway startup plugin activation.

## Best Possible Solution

Land this PR or an equivalent central resolver compatibility change after maintainer review, preserving explicit `agentRuntime` precedence, fill-missing legacy semantics, plugin disablement behavior, and the documented native Codex versus explicit ACP route split.

## Reproduction Assessment

Yes. A config using only `agents.defaults.embeddedHarness` or `agents.list[].embeddedHarness` is still accepted by current types/schema and doctor migration, but current main's shared resolver ignores that field before harness selection and startup activation consume policy.

## Solution Assessment

Yes. Centralizing compatibility in `resolveAgentRuntimePolicy` matches the doctor migration semantics and avoids duplicating legacy fallback logic in each downstream consumer; the remaining question is intended blast radius across all resolver callers.
