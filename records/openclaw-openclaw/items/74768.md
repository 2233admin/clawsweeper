## Summary

Keep open. Current main still leaves the legacy `embeddedHarness` runtime policy invisible to the shared resolver, while the schema, help text, doctor migration, and docs still treat that legacy shape as valid compatibility input. The proposed patch is focused on that gap and I found no definite blocking defect in the provided diff.

## What This Changes

The PR updates agent runtime policy resolution to fold legacy `embeddedHarness` into missing `agentRuntime` fields, with tests for harness selection, startup plugin activation, and `sessions_spawn` Codex native routing.

## Best Possible Solution

Land this PR or an equivalent shared-resolver compatibility change after maintainer review, preserving explicit `agentRuntime` precedence, legacy fill-missing behavior, plugin disablement semantics, and the native Codex versus explicit ACP routing split.

## Reproduction Assessment

Yes. A config that only sets `embeddedHarness: { runtime: "codex", fallback: "none" }` reproduces the current-main gap by source inspection: the field is accepted and migrated elsewhere, but the shared runtime resolver ignores it before harness selection and startup activation consume the policy.

## Solution Assessment

Yes. Resolving the compatibility shape once in `resolveAgentRuntimePolicy` is the narrowest maintainable fix because the affected harness-selection, metadata, model-runtime, and startup-plugin paths already centralize on that resolver.
