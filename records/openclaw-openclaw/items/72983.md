## Summary

Keep open. The PR is pointed at a real Control UI streaming-rendering bug and current main still has the unsafeHTML streaming path, but the proposed patch introduces a visible whitespace regression in streaming message text and leaves streaming reasoning whitespace under-specified, so it should be amended before merge.

## What This Changes

The PR changes Control UI chat rendering so streaming assistant markdown and reasoning use Lit text interpolation with a `.chat-text--streaming` CSS rule while completed messages continue through sanitized markdown HTML.

## Best Possible Solution

Amend this PR, or replace it with an equivalent focused Control UI patch, so streaming markdown and reasoning render as stable text without visible template whitespace, final messages still use sanitized markdown HTML, grouped-render regression coverage locks the behavior, and the user-facing fix gets a changelog entry before #68655 is closed.

## Reproduction Assessment

Yes. Current main statically reproduces the risky path because streaming messages still reach `unsafeHTML(toSanitizedMarkdownHtml(markdown))`, and #68655 provides a manual long-markdown streaming repro. The PR regression is reproducible from the diff because `white-space: pre-wrap` preserves the template newlines and indentation around `${markdown}`.

## Solution Assessment

No. The plain-text streaming fallback is a reasonable minimal direction, but this implementation should bind streaming text without surrounding literal whitespace, preserve reasoning newlines intentionally, and add focused tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove preserved indentation from streaming text:** `ui/src/ui/chat/grouped-render.ts:1432`
  - body: The new streaming branch puts `${markdown}` on an indented standalone template line while `.chat-text--streaming` uses `white-space: pre-wrap`. Lit keeps those literal newlines and spaces as text nodes, so streaming replies can show a leading blank/indented line and trailing whitespace until the final markdown render; bind the text directly inside the div, and fix the duplicate regular-message branch too.
  - confidence: 0.91
- **[P3] Preserve reasoning line breaks while streaming:** `ui/src/ui/chat/grouped-render.ts:1409`
  - body: Streaming `reasoningMarkdown` is now raw text inside `.chat-thinking`, but that class does not preserve whitespace. `formatReasoningMarkdown()` joins the reasoning label and lines with newlines, so visible streaming reasoning collapses into one run until final markdown rendering; add a streaming thinking style/helper that preserves newlines without preserving template indentation.
  - confidence: 0.78
