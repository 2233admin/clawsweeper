## Summary

Keep open. Current main still has the streaming `unsafeHTML(toSanitizedMarkdownHtml(...))` path, and this PR is the paired implementation for the open flicker report, but the branch needs a small repair before merge because the new `pre-wrap` streaming branch preserves template indentation and the reasoning stream path loses intended line breaks.

## What This Changes

The PR changes Control UI grouped chat rendering and chat text CSS so streaming assistant markdown and reasoning render as Lit text during streaming, while completed messages continue through sanitized markdown HTML.

## Best Possible Solution

Amend this PR, or replace it with an equivalent focused Control UI patch, so streaming markdown and reasoning update as stable escaped text without visible template whitespace, final messages keep sanitized markdown rendering, and grouped-render coverage plus a changelog entry land before the linked issue is closed.

## Reproduction Assessment

Yes. Current main statically reproduces the risky path because streaming messages are marked with `isStreaming: true` and still render changing markdown through `unsafeHTML`; the PR regression is reproducible from the head file because `pre-wrap` preserves the indented template text around `${markdown}`.

## Solution Assessment

No. The streaming plain-text fallback is a reasonable minimal direction, but this implementation should bind streaming text without surrounding template whitespace, preserve reasoning newlines with a streaming thinking style/helper, and add targeted tests plus the required changelog entry before merge.
