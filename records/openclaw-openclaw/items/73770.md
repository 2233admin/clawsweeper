## Summary

Keep open. Current main still lacks the scoped cross-agent memory_search and provenance behavior, so this PR is not obsolete, but the patch needs maintainer follow-up before merge because it changes the memory visibility boundary and has concrete correctness issues.

## What This Changes

The PR adds a memory_search scope resolver, loops searches over allowed agent ids for coordinator-style agents, decorates results with provenance fields, updates memory tool mocks/tests, and includes a stacked memory CLI provider-registration change.

## Best Possible Solution

Keep this PR as the active implementation candidate, but require an explicit cross-agent access policy, focused tests for ordinary-agent isolation and coordinator searches, partial-failure handling that preserves healthy results, SDK/output-type alignment after rebase, and docs/config coverage before merging.

## Reproduction Assessment

Yes for the patch-level issues: the diff shows the implicit chief default and the early unavailable return, and both can be covered with focused memory-core unit tests using multiple configured agents and one failing scoped manager. Current main itself does not reproduce the new behavior because it still has the single-agent search path.

## Solution Assessment

No, not as currently proposed. The memory-core tool is the right owner for scoped memory_search, but the safer implementation is explicit opt-in for cross-agent access plus a clean serialized output contract and robust partial-error behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Make cross-agent search explicit:** `extensions/memory-core/src/memory-permissions.ts:56-58`
  - body: The resolver grants any requester named `chief` access across configured agents when no `searchScope` is configured. Since current docs describe memory as per-agent storage, this can silently broaden memory visibility for an existing agent id; require explicit coordinator/allowed-agent config before enabling cross-agent search.
  - confidence: 0.86
- **[P2] Keep healthy scoped agents searchable:** `extensions/memory-core/src/tools.ts:237-245`
  - body: This guard returns an unavailable result as soon as any scoped memory manager errors. For coordinator searches, one failing agent suppresses all healthy agents even though the later loop is prepared to skip errored managers; only return unavailable when every scoped memory lookup failed.
  - confidence: 0.9
