## Summary

Keep open. Current main does not implement the requested Copilot Opus 4.7 adaptive-effort clamp: the Anthropic transport still maps minimal/low to `low` and emits `output_config.effort` unchanged whenever a reasoning-enabled adaptive model reaches that path. The common default Copilot Claude path appears mitigated because synthetic/default Copilot Claude models are currently non-reasoning, but that is not the requested fix and the exact linked fix PR #70584 remains open/unmerged. The auth-profile cooldown recommendation also still needs maintainer judgment because current docs say format/invalid-request failures use cooldowns.

## Best Possible Solution

Keep the issue open and handle it together with #70584 or a maintainer-authored replacement. The best fix is a narrow provider/model effort-capability guard for `github-copilot/claude-opus-4.7` on the Anthropic Messages adaptive-thinking path, clamping `minimal`/`low` to `medium` while preserving direct Anthropic Opus 4.7 behavior, with regression tests for Copilot low/minimal and non-Copilot passthrough. Separately decide whether `invalid_reasoning_effort` should be exempt from auth-profile cooldowns or whether the current documented cooldown policy should stand.
