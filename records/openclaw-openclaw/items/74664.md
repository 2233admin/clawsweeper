## Summary

Keep open: the report remains valid on current main. OpenClaw accepts and uses `compat.supportsTools: false` to suppress normal tool registration, but the active OpenAI-compatible payload builder still ignores that compat field and can emit `tools` or the tool-history `tools: []` fallback.

## What This Changes

The issue requests that OpenAI-compatible chat-completions models configured with `compat.supportsTools: false` omit all tool-related fields from outbound payloads.

## Best Possible Solution

Honor `compat.supportsTools === false` inside the OpenAI Completions payload builder itself: carry the flag through compat resolution, skip `tools`, `tool_choice`, and the tool-history `tools: []` fallback when disabled, and add regression coverage for non-empty tools plus replayed tool history while preserving default behavior when the flag is true or omitted.
