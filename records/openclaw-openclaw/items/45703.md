## Summary

Keep open. Current main has improved adjacent symptoms, but the central Agents view regression remains: the picker is still a native label-only dropdown, and the Overview panel still does not surface the selected agent's emoji/avatar identity in the main agent context.

## What This Changes

The issue asks to restore the Control UI Agents section after dashboard v2 replaced a richer agent selector/list and identity display with a plain dropdown, missing emoji/metadata, poor visual treatment, and broken controls/model selection.

## Best Possible Solution

Repair the Control UI Agents page with an accessible rich selector, list, or sidebar that shows avatar/emoji, display name, id/default state, and useful metadata, and add the selected agent's identity to the Overview panel while preserving existing Copy ID, Set Default, refresh, tabs, and provider-qualified model picker behavior.

## Reproduction Assessment

Yes. Rendering `renderAgents` on current main with a non-empty agent list produces `.agents-select` label-only options, and `renderAgentOverview` does not render selected-agent identity/avatar/emoji in the primary overview grid.

## Solution Assessment

Yes. A focused Control UI Agents-view repair is the narrowest maintainable solution; a broad dashboard-v2 rollback or core/plugin API change is unnecessary for the remaining observable problem.
