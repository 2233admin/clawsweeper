## Summary

Current main still does not fully implement the requested Slack parity: Slack message actions expose neither `channel-info` nor `channel-list` through Slack action discovery, dispatch, runtime helpers, public exports, tests, or docs. The issue is also paired with open PR #53199 via closing syntax, so it should remain open until that candidate is reviewed, completed, and merged or replaced.

## Best Possible Solution

Keep this issue open and review the linked implementation candidate. The best fix belongs in `extensions/slack`: expose both Slack `channel-info` and `channel-list` through `listSlackMessageActions`, route both in `handleSlackMessageAction`, add runtime/action helpers around Slack `conversations.info` and paginated `conversations.list`, gate them consistently with Slack action config, and update focused Slack action tests plus `docs/channels/slack.md`. If maintainers intentionally split the work, keep this issue open for `channel-info` or create a narrower canonical follow-up before closing it.
