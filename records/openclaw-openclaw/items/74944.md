## Summary

Keep open. The report is concrete, unprotected, and still maps to a narrow updater reliability gap: current global package-update code can invoke the owning Homebrew npm shim but does not build a launchd-minimal PATH that guarantees `env node` can resolve the active Homebrew Node runtime.

## What This Changes

Ensure macOS launchd-triggered self-update/global npm update subprocesses can resolve Homebrew Node/npm shims under launchd's minimal PATH and avoid update retry/crash loops.

## Best Possible Solution

Add a shared update subprocess env builder and use it from gateway and CLI package-update paths so minimal launchd environments get the active runtime bin, owning npm prefix bin, Homebrew Node/bin locations, and system dirs without adding project-local bins or weakening service PATH hardening.

## Reproduction Assessment

Yes. A macOS Homebrew global npm install with PATH `/usr/bin:/bin:/usr/sbin:/sbin`, package root `/opt/homebrew/lib/node_modules/openclaw`, and owning npm `/opt/homebrew/bin/npm` should exercise the failing `env node` shim path; current code inspection shows no update env repair for that case.

## Solution Assessment

No. The issue's hard-coded shell `export PATH=...` workaround would likely avoid the immediate failure, but the maintainable fix is a shared, tested update env helper used by every global package-update subprocess path.

## Repair Work Prompt

Fix the macOS Homebrew launchd update failure reported at https://github.com/openclaw/openclaw/issues/74944. Current main can invoke the owning absolute npm binary, but global package-update subprocess env construction does not guarantee that npm's `env node` shebang can resolve Homebrew Node when PATH is launchd-minimal. Implement a shared update subprocess env helper used by `createGlobalInstallEnv`/gateway and CLI global package update paths. It should preserve existing env, prepend or otherwise include the active runtime bin from `process.execPath` when safe, the owning npm prefix bin derived from the package root/install target, Homebrew Node/bin locations already used by repo helpers, and system dirs; do not add project-local `node_modules/.bin` or broaden trusted exec safe-bin policy. Add regression coverage for a darwin global npm install with PATH `/usr/bin:/bin:/usr/sbin:/sbin`, package root `/opt/homebrew/lib/node_modules/openclaw`, npm `/opt/homebrew/bin/npm`, and Node under `/opt/homebrew/opt/node/bin/node`, asserting both primary and fallback update steps receive a PATH that resolves `node`. Add a single-line `CHANGELOG.md` fix entry with `Fixes #74944` and `Thanks @jrsdhr`. Do not change release metadata or unrelated service hardening policy.
