## Summary

Keep open. Current main still does not repair the reported launchd/Homebrew npm shim PATH failure: the update path now avoids restarting after a failed update, and service startup has separate PATH helpers, but the global package update environment still does not add the active Node or Homebrew Node directories needed when launchd supplies only `/usr/bin:/bin:/usr/sbin:/sbin`.

## What This Changes

The issue asks OpenClaw to make macOS launchd-submitted self-updates work for Homebrew global npm installs whose npm shim needs `node` resolvable on PATH.

## Best Possible Solution

Centralize update subprocess PATH construction so package-manager updates and any launchd update handoff include the active runtime directory, the owning package-manager prefix bin, Homebrew Node/bin locations, and safe system dirs, while preserving service PATH hardening; add regression tests and a changelog fix crediting the reporter.

## Reproduction Assessment

Yes. The issue includes a concrete macOS Homebrew LaunchAgent setup, minimal launchd PATH, exact npm exit 127 log signature, and current source still matches the PATH failure mode; live macOS confirmation remains part of fix validation.

## Solution Assessment

Unclear. The reporter's `export PATH=...` workaround is directionally correct, but the maintainable fix should be a shared update environment helper used by CLI/Gateway update paths and any launchd handoff rather than a one-off shell snippet.
