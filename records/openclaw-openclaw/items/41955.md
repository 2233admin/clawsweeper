## Summary

Keep this PR open. Current main still builds the Control UI session dropdown from persisted session rows plus the active key, so configured agents without session-store rows are still missing. The submitted patch is stale because the active resolver is now `resolveSessionOptionGroups()` in `ui/src/ui/chat/session-controls.ts`, while the PR targets the older `resolveSessionOptions()` helper. Related open PR #48671 targets the newer resolver, so the best maintainer path is consolidation or review rather than auto-closing this PR.

## Best Possible Solution

Refresh this PR or consolidate it with #48671. The implementation should update `resolveSessionOptionGroups()` to merge `state.agentsList.agents` after stored rows and the active key, dedupe against normalized keys, preserve configured names, respect `agents.list.scope` and custom `mainKey`, and add regression tests for sessionless configured agents plus global-scope behavior.
