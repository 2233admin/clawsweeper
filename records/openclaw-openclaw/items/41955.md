## Summary

Keep this PR open for maintainer follow-up. Current main still does not implement the intended behavior: the Control UI session dropdown is built from `sessions.list` rows plus the active session key, and configured agents without stored sessions are not synthesized into the options. The PR’s original patch is also stale because current main uses `resolveSessionOptionGroups()` in `ui/src/ui/chat/session-controls.ts`, while the PR patch targets the removed `resolveSessionOptions()` helper. Related open PR #48671 appears to be the better current-code implementation candidate, so the path is review/consolidation rather than auto-closing this PR.

## Best Possible Solution

Refresh this PR against current main or consolidate it with open PR #48671. The current-code fix should update `resolveSessionOptionGroups()` to merge `state.agentsList.agents` after stored rows and the active key, dedupe against normalized keys, preserve configured names, respect `agents.list.scope` and custom `mainKey`, and add regression tests for sessionless configured agents plus global-scope behavior.
