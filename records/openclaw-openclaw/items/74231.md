## Summary

Keep open. Current main does not have the requested diagnostic, and the PR remains the focused active implementation rather than a duplicate or obsolete change; the main unresolved point is maintainer acceptance of the configured-root disclosure tradeoff.

## What This Changes

This PR appends a bounded allowed-roots hint to both `path-not-allowed` throws in `assertLocalMediaAllowed` and adds regression tests for short and overflow root lists.

## Best Possible Solution

Land a bounded, tested diagnostic only after maintainers explicitly accept the root-path disclosure across error-message surfaces, preserving `LocalMediaAccessError.code` and the rejected-path contract.

## Reproduction Assessment

Yes. A direct source check on current main shows both `assertLocalMediaAllowed` rejection paths still emit the bare message; calling it with an out-of-root path and explicit allowed root would reproduce the missing hint.

## Solution Assessment

Yes, subject to maintainer approval of the disclosure tradeoff. The proposed shared formatter and cap are the narrowest maintainable shape for the requested diagnostic; a safer alternative would be a generic hint or sanitized roots, but that would not solve the operator guidance problem as directly.
