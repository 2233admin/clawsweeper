## Summary

Keep open. Current main has auto-compaction, pre-compaction memory flush, successor transcript rotation, and queued-followup retargeting, but it does not implement the requested configurable token-threshold session split lifecycle with channel routing rollover and continuity payloads. Same-author #53330 is already closed as the duplicate, so this issue remains the canonical tracker.

## What This Changes

Request a configurable auto-split policy that creates a new session near context limits, carries summaries/task/confirmation state forward, and routes later channel messages to it.

## Best Possible Solution

Keep this as the canonical core session-lifecycle tracker and design a supported auto-split policy that reuses existing compaction, memory flush, transcript rotation, queue retargeting, and session hooks while adding explicit threshold configuration, channel binding rollover, continuity payload handling, docs, and regression coverage.
