## Summary

Keep open. Current main has substantial context-pressure mitigation, including preflight compaction, memory flush, successor transcript rotation, queued-followup retargeting, and overflow fallback/reset paths. It still does not implement the requested first-class auto session split lifecycle: configurable session-level token/window thresholds, active-run/tool-call boundary handling, invisible route rollover, and continuity payloads for task or pending-confirmation state.

## Best Possible Solution

Keep this issue open as the canonical core product/design tracker. The best path is a supported session.autoSplit-style policy layered on existing memory flush, compaction, successor transcript rotation, queue retargeting, and session store machinery, with explicit config thresholds, active-run/tool-call boundary rules, route rollover, continuity payload handling for summaries/tasks/pending confirmations, docs, and focused regression tests.

## Risks / Open Questions
