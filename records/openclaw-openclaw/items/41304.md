## Summary

Keep open. Current main has a real embedded tool-registration/dispatch path and several related guardrails, but the active AgentMail/GitHub/Composio reports describe a broader class where the assistant claims write/action success without a matching tool execution. The current code mostly diagnoses or partially prevents specific failure modes; it does not prove the user-facing hallucinated-success bug is fixed across the reported providers/plugins/models.

## Best Possible Solution

Keep the issue open and ask for a current-main `/export-trajectory` or `/diagnostics` bundle plus provider/model id, plugin list, effective tool policy, and one minimal side-effect reproduction. If reproduced, add a focused guardrail that detects claimed write/action success without a matching tool execution event and reports a clear model/provider/plugin-policy error instead of accepting the assistant's success text.
