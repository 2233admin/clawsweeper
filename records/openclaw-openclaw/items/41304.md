## Summary

Keep this issue open. Current main has shipped narrower fixes for missing allowlisted tools and plain-text pseudo tool-call diagnostics, but the discussion also reports plugins/MCP tools being registered while the model still produces prose or simulated success instead of a structured tool call. I did not find source, test, or release evidence that current main fully fixes that broader registered-tool/non-invocation path.

## Best Possible Solution

Keep this issue open and ask for a trajectory export from a current failing run. Maintainers should use it to classify the failure as: no callable tools after policy filtering, callable tools present but no structured tool call emitted by the model/provider, a structured call blocked or normalized incorrectly, or a real tool execution whose external side effect failed. Then add a targeted regression or diagnostic for the confirmed path.
