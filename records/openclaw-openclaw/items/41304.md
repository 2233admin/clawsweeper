## Summary

Keep this open. Current main has meaningful partial fixes for missing allowlisted tools and proxy/local OpenAI-compatible models emitting plain-text tool directives, but the thread also reports plugins/MCP tools being registered while the model still produces prose or simulated success instead of a structured tool call. I did not find source, test, or release evidence that current main fully fixes that broader registered-tool/non-invocation path.

## Best Possible Solution

Keep this issue open and ask for a redacted trajectory export from a current failing run on current main or the latest release. Maintainers should classify the failure as: no callable tools after policy filtering, tools sent but no structured tool call emitted by the model/provider, a structured call blocked or normalized incorrectly, or a real tool execution whose external side effect failed. If the only remaining repro is a provider-specific local-model parser/tool-choice problem, split or close against the narrower provider/docs fix after confirmation.
