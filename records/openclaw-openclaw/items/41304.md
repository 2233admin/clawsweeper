## Summary

Keep this issue open. Current main has useful mitigations and diagnostics for subsets of this failure, but it still does not generally enforce real tool invocation when a model answers with prose or JSON/XML/ReAct-looking text instead of a structured tool call. The discussion also spans AgentMail, GitHub, Composio, and custom plugin tools, so this remains broader than one plugin setup issue.

## Best Possible Solution

Keep this issue open and ask for a current redacted trajectory export from latest main or the latest release. Maintainers should classify the failure as one of: no callable tools after policy filtering, tools sent but no structured tool call emitted, structured call blocked/normalized incorrectly, real tool execution succeeded but the external side effect failed, or provider-specific local-model behavior. If it reduces to the same remaining enforcement gap as #45049, consolidate there after maintainer review; otherwise add a guarded, provider-aware retry/tool-choice path without blindly executing assistant text.
