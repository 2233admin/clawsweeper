## Summary

Keep this issue open. Current main has several meaningful partial fixes for missing allowlisted tools and proxy/local OpenAI-compatible models that emit plain-text tool directives, but the discussion still reports registered plugin/MCP tools producing prose or simulated success instead of structured tool calls. I did not find current source, test, docs, or release evidence that the broader registered-tool/non-invocation path is fully fixed.

## Best Possible Solution

Keep this issue open and ask for a redacted trajectory export from a current failing run on current main or the latest release. Maintainers should classify the failure as: no callable tools after policy filtering, tools sent but no structured tool call emitted by the model/provider, a structured call blocked or normalized incorrectly, or a real tool execution whose external side effect failed. If the remaining repro reduces to a provider-specific local-model parser/tool-choice problem, split or close against that narrower provider/docs fix after confirmation.
