## Summary

Keep open. Current main still follows the empty `transcriptPrompt` to runtime-only blank prompt path, and an open linked fix PR is already tracking the narrow repair, so this issue should not be closed before that PR is reviewed and resolved.

## What This Changes

The issue asks OpenClaw to stop sending an empty provider-visible user message during pre-compaction memory flushes, which can make Anthropic-backed Telegram sessions fail and reset.

## Best Possible Solution

Preserve hidden/silent memory-flush semantics while making the provider-facing flush turn non-empty and valid, then close this issue only after the linked fix PR is merged or the maintainer chooses a different narrow patch.

## Reproduction Assessment

Yes. A source-level reproduction path is high-confidence: memory flush passes `transcriptPrompt: ""`, runtime prompt resolution turns it into `prompt: ""`, and the embedded runner submits that blank prompt for a runtime-only turn.

## Solution Assessment

No. Current main is not the best solution because it keeps the blank provider-visible submission; the maintainable fix is a focused non-empty flush prompt path that still avoids user-visible delivery.
