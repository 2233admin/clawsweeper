## Summary

Keep open. Current main still routes pre-compaction memory flush through a runtime-only turn whose provider-facing prompt is empty, and the existing tests still encode that blank prompt behavior, so this remains a valid narrow bug rather than cleanup-close material.

## What This Changes

The issue requests fixing pre-compaction memory flush so it does not submit an empty provider-facing user prompt to Anthropic while preserving silent auto-compaction behavior.

## Best Possible Solution

Make the runtime-only memory flush path produce a valid non-empty provider-facing turn while keeping the flush instructions in hidden runtime/system context and preserving silent delivery, then add focused regression coverage through the memory-flush handoff and embedded attempt path.

## Reproduction Assessment

Yes for the OpenClaw-side path: current main sends `transcriptPrompt: ""`, resolves that to `prompt: ""` with `runtimeOnly: true`, bypasses the blank-prompt skip, and submits it to the active session. The exact live Anthropic 400 should be confirmed by a targeted request-shape regression test or live probe during the fix.

## Solution Assessment

No; the current blank runtime-only submission is not the best provider-facing shape because strict providers can reject or mishandle empty user turns. The safer maintainable fix is a non-empty housekeeping prompt or equivalent provider-safe sentinel that remains hidden from transcript and channel delivery.
