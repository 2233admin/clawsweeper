## Summary

Keep open. Current main still does not expose `tools.fs.allowedRoots`, so the feature is not implemented outside this branch. The branch is the active implementation attempt, but it should not merge yet because the diff still introduces blocking filesystem-boundary regressions in `apply_patch` deletes and sandbox bind-mount handling.

## What This Changes

This PR adds a `tools.fs.allowedRoots` config field and threads additional filesystem roots through read/write/edit, `apply_patch`, media, prompt-image, and sandbox path checks with regression tests.

## Best Possible Solution

The desired end state is one canonical path-policy implementation where read/write/edit, `apply_patch`, and media tools resolve sandbox container and host paths through the bridge or mount table before comparing against configured roots, with pinned filesystem helpers retained for root-contained mutations and schema/help/tests aligned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore pinned removal for apply_patch deletes:** `src/agents/apply-patch.ts:311`
  - body: The workspace-only delete path now validates with `assertSandboxPath()` and then calls `fs.rm()` on the resolved string. Current main uses `removePathWithinRoot()` so an intermediate parent cannot be swapped after validation; this change reopens that race and can delete outside the workspace or allowed root if a concurrent process replaces a parent with a symlink. Use the pinned remove helper with `sandboxResult.matchedRoot` and `sandboxResult.relative`.
  - confidence: 0.9
- **[P2] Validate allowed roots after sandbox mount resolution:** `src/agents/pi-tools.read.ts:601`
  - body: In sandbox runs, this wrapper checks the raw/container path against host-configured `additionalRoots` before `SandboxFsBridge.resolvePath` can map Docker bind mounts. A bind such as `/host/shared:/shared` with `allowedRoots: ["/host/shared"]` still rejects `read`/`write`/`edit` on `/shared/file`, so the new setting fails for the sandbox bind-mount case. Resolve through the bridge or mount table first, or compare against corresponding container roots.
  - confidence: 0.87
