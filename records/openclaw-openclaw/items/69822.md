## Summary

Keep open. Current main still has the socket.drain/session-message subscriber gap, and the latest PR is the canonical active fix attempt, but the branch still needs a small correctness update before merge because its new teardown helper assumes a request-context method that at least one existing direct session test context does not provide.

## Best Possible Solution

Keep this PR as the focused implementation candidate, fix the remaining request-context coverage gap, document socket.drain if it is a public advertised event, then rerun the targeted gateway tests and changed gate before normal maintainer review.

## Reproduction Assessment

Yes. Applying the latest PR diff and running the existing server.sessions.store-rpc.test.ts reset/delete coverage should reproduce the context-method failure, because that direct context lacks getSessionMessageSubscriberConnIds while the new teardown path calls it.

No, not yet. The feature direction is plausible and the earlier duplicate binding appears fixed, but the implementation should either update every direct GatewayRequestContext fixture or narrow the helper contract so teardown paths cannot crash outside the full server context.
