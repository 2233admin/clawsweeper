## Summary

Keep open. The PR targets the canonical remaining socket.drain work from #63374, but the current diff is not mergeable because it imports `emitSessionsChanged` into `src/gateway/server-methods/agent.ts` while leaving the existing local function with the same name in place.

## What This Changes

The PR registers `socket.drain` as a read-scoped gateway event, exposes session message subscriber IDs to request contexts, emits drain telemetry for reset/delete/new session changes, changes teardown `sessions.changed` slow-consumer behavior, and adds gateway tests.

## Best Possible Solution

Keep this PR as the canonical review surface, but land only a focused version that uses a single shared `emitSessionsChanged` binding, preserves read-scope filtering, documents the new gateway event if it is public, and proves reset/delete/new delivery to event and message subscribers with targeted gateway tests plus the changed gate.

## Reproduction Assessment

Yes. Applying the PR patch is enough to reproduce the issue statically: `agent.ts` gains an imported `emitSessionsChanged` while retaining a local function of the same name, which will fail parsing/typechecking before runtime behavior can be tested.

## Solution Assessment

No. The feature direction is plausible, but this implementation is not mergeable until the stale local helper is removed or the shared import is avoided; after that, the targeted gateway tests and changed validation should be rerun.
