## What This Changes

This PR adds a read-scoped `socket.drain` gateway event for session reset/delete/new teardowns, routes it to both session event and session message subscribers, exposes message-subscriber lookup on `GatewayRequestContext`, adjusts teardown slow-consumer handling, and adds gateway coverage.

## Best Possible Solution

Keep this PR as the active implementation surface, then land a focused version after maintainer review confirms the public event contract, targeted gateway tests pass, and the changed gate is green; add protocol docs if `socket.drain` is intended as a supported client event.

## Reproduction Assessment

Yes. The source-level reproduction is clear: on current main, `sessions.messages.subscribe` clients are tracked separately, while `emitSessionsChanged` only targets session event subscribers and `socket.drain` is not a registered broadcast event.

## Solution Assessment

Yes, with validation pending. Moving teardown emission into the session mutation helper and using the existing subscriber registries and scope guard is a narrow maintainable approach; the main open question is whether the new discovered event needs protocol documentation before merge.
