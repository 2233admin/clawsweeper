## Summary

Keep this PR open. Current main still does not implement the proposed socket.drain routing for sessions.messages.subscribe-only clients: session teardown emits only sessions.changed to sessions.subscribe clients, socket.drain is not a registered/guarded gateway event, and GatewayRequestContext does not expose the message-subscriber registry to session mutation handlers. The PR is an active focused implementation candidate for the closed related feature request #63374 and a cleaner successor to closed unmerged PR #65539, so cleanup closure would be premature.
