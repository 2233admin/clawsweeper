## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background LLM agent turns: provider runtime contracts are stream-oriented, cron/session/subagent/heartbeat surfaces have no batch opt-in, and the batch support present today is scoped to memory embeddings or media/transcription paths rather than agent turns. The related closed reports #49326, #56126, and #70606 are useful narrower context for this canonical tracker, not shipped fixes.

## What This Changes

The issue requests Batch API support for non-real-time background LLM work such as cron agent turns, spawned background agents, heartbeat checks, and scheduled maintenance tasks.

## Best Possible Solution

Keep this as the canonical product/API tracker. The desired end state is an additive delayed/batch agent-turn contract in the provider/runtime and public Plugin SDK, opt-in protocol/config fields for cron agentTurn jobs, sessions_send, sessions_spawn/subagents, and heartbeat/background runs, plus OpenAI and Anthropic provider implementations with documented usage, accounting, fallback, cancellation, and delivery semantics.
