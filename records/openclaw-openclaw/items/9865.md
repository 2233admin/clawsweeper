## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background agent turns: cron, sessions_send, sessions_spawn/subagents, heartbeat, and provider runtime surfaces are still immediate or stream-oriented, while the existing Batch API code is scoped to OpenAI memory embeddings. The prior discussion also treats this as the canonical tracker, with the related closed reports as narrower context rather than shipped fixes.

## What This Changes

The issue requests opt-in provider Batch API routing for non-real-time background agent work such as cron jobs, subagents, scheduled maintenance, and heartbeats to reduce token cost.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract: public SDK/provider runtime support, protocol/config opt-ins for cron, sessions_send, sessions_spawn/subagents, and heartbeat/background runs, plus OpenAI and Anthropic provider implementations and docs for fallback, cancellation, accounting, provider eligibility, and delayed delivery.
