## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background agent turns, and the existing discussion treats this as the canonical tracker while the related closed reports are narrower context rather than shipped fixes.

## What This Changes

The issue requests opt-in provider Batch API support for non-real-time background OpenClaw work such as cron jobs, heartbeat checks, scheduled maintenance, and spawned sub-agents.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract: public SDK/provider runtime support, protocol/config opt-ins for cron, sessions_send, sessions_spawn/subagents, and heartbeat/background runs, plus OpenAI and Anthropic provider implementations and docs for fallback, cancellation, accounting, and delayed delivery.
