## Summary

Keep open. Current main still does not implement the intended Gmail watcher exclude-label plumbing: `buildGogWatchServeArgs()` omits `--exclude-labels`, the Gmail runtime/config schema has no `excludeLabels` field, and both Gateway auto-start and `openclaw webhooks gmail run` use that same arg builder. The linked bug report #56635 remains a live implementation target, and this PR is a plausible focused fix rather than an obsolete or already-shipped change.

## Best Possible Solution

Keep this PR open as an implementation candidate for #56635, or choose one canonical duplicate PR and close the others after the canonical fix lands. The best fix belongs in the Gmail hook owner path: add `hooks.gmail.excludeLabels`, resolve a default of `SPAM,TRASH,DRAFT,SENT`, pass it through both Gateway and CLI watcher args, and keep the regression tests for default, custom, and empty-label behavior.
