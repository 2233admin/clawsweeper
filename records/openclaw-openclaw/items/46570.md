## Summary

Current main indexes workspace memory files and can source-filter memory_search tool calls, but I do not see high-confidence proof that the reported CLI path is fixed. The CLI still calls manager.search without a memory/session source filter, and the builtin hybrid path only relaxes keyword-only hits when there are no strict hits at all, so mixed session results can still plausibly crowd out exact memory-file hits. Keep this open as a narrow memory-core fix candidate.

## Best Possible Solution

Keep this issue open and add a focused memory-core regression for builtin hybrid search with `sources: ["memory", "sessions"]`: an exact phrase present only in `memory/*.md` should appear in `openclaw memory search` / `MemoryIndexManager.search` results even when session transcript chunks are indexed. Then adjust ranking or result selection so keyword-backed memory hits are not dropped solely because strict session-vector hits exist. Consider a CLI `--corpus memory|sessions|all` only if needed, but the default mixed search should still include relevant memory-file hits.
