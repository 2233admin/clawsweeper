## Summary

Keep this issue open. Current main still takes the active `enqueue-followup` path, drains only when the active-run snapshot is stale, and then unconditionally calls `typing.cleanup()`, so a followup queued behind a live run still loses its typing/liveness signal.

## What This Changes

The issue requests that queued followups behind an active run preserve visible typing/liveness instead of immediately cleaning up their typing controller.

## Best Possible Solution

Update the `enqueue-followup` path to compute post-enqueue liveness once: immediately drain when the active snapshot is stale, but when the followup is truly queued behind a live run, keep or refresh typing and let normal queued-turn or terminal cleanup paths own shutdown.
