## Summary

Keep this open. Current main does not contain a literal hardcoded `wangtao` path, but the actionable bug from the thread still exists: default workspace values are resolved to host-absolute paths and then persisted into config by onboarding/setup/configure flows. The related #56810 PR targeted the same portability problem but is closed and unmerged.

## Best Possible Solution

Keep the issue open and land a focused fix like #56810: when onboarding/setup/configure/migration flows persist the canonical default workspace, save a portable `~/.openclaw/workspace` value or omit the default, while continuing to resolve to absolute paths at runtime and preserving explicit custom absolute workspace values. Add regression coverage for classic onboarding, non-interactive onboarding, setup, configure, and profile-aware defaults.
