## Summary

Keep open. Current main still surfaces provider stopReason:"length" but does not implement the requested main-session recovery guard before tool execution. The runner has adjacent malformed-tool-call repair and incomplete/empty-turn fallbacks, but no policy that blocks suspect parsed tool calls or injects a model-facing truncation recovery prompt. The related open PR #66146 is only a partial max-token forwarding mitigation and does not supersede this issue.

## Best Possible Solution

Keep this open and implement a focused main-session Phase 1 fix: preserve provider stopReason:"length" through the stream/result path, detect length-terminated assistant turns with tool-call activity before dispatch, prevent suspect tool execution, and steer the model with a recovery/continuation message. Coordinate with #63188 for subagent mode:"run" semantics and #66146 for max-token forwarding, but do not treat either as a replacement for main-session truncation recovery.
