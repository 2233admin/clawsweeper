Current main still lacks the requested missing-plugin-id guard: `plugins enable <id>` and `plugins disable <id>` both mutate config from the raw CLI argument before any registry lookup. This PR remains a valid implementation candidate for #73551; the overlapping related PRs #73932 and #74016 are closed and unmerged, so they do not supersede it.
