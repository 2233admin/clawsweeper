## Summary

The PR should remain open because current main does not already implement the requested main-agent self-inspection exception, and the proposed patch still has blocking command-execution boundary defects that need maintainer/security review before merge.

## What This Changes

The PR adds a main-agent exec exception for selected host-level OpenClaw self-inspection commands under full trust and adds path tests for allowed, mixed, mutating, and wrapper command forms.

## Best Possible Solution

The desired end state is a maintainer-approved policy for a narrow main-agent self-inspection exception, implemented with parser/segment-based OpenClaw command detection, cross-platform binary recognition, strict read-only argv validation, and regression coverage for pipes, Linux paths, doctor mutation flags, and memory status mutation flags.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Detect piped OpenClaw invocations before allowing execution:** `src/agents/bash-tools.exec.ts:178-181`
  - body: `MAIN_HOST_LEVEL_OPENCLAW_COMMAND` does not treat a single pipe as a command boundary, so `echo ok | openclaw update` skips `assertMainHostLevelOpenClawBoundary` entirely under full/off. That leaves a mutating OpenClaw command outside the new main-host restriction.
  - confidence: 0.92
- **[P1] Match host OpenClaw paths across platforms:** `src/agents/bash-tools.exec.ts:178-185`
  - body: The absolute path patterns only match `/Users/...` paths. Linux invocations such as `/home/user/.npm-global/bin/openclaw` or `/home/user/.openclaw/scripts/ops-host-openclaw.sh` will not trigger the boundary regex, so the new restriction can be skipped on Linux hosts.
  - confidence: 0.9
- **[P1] Reject all mutating doctor modes:** `src/agents/bash-tools.exec.ts:219-222`
  - body: The doctor allowlist only rejects the exact `--fix` token, but current CLI registration also exposes mutating `--repair`, `--force`, and `--generate-gateway-token` modes, with `--fix` as an alias. Those forms can still pass through the proposed self-inspection exception.
  - confidence: 0.96
- **[P1] Reject mutating memory status flags:** `src/agents/bash-tools.exec.ts:228-230`
  - body: `memory status` is allowed with arbitrary trailing arguments, but current docs describe `memory status --fix` as repairing metadata and `--index` as rebuilding the index. This makes the supposedly read-only exception permit mutating memory operations.
  - confidence: 0.95

## Security Review

Status: needs_attention

Summary: The PR changes a host command-execution boundary and currently lets concrete mutating or bypassed OpenClaw commands through the new exception.

Concerns:

- **[high] Mutating doctor modes can pass the allowlist:** `src/agents/bash-tools.exec.ts:219`
  - body: The proposed doctor guard blocks only exact `--fix`, while current CLI options include `--repair`, `--force`, and `--generate-gateway-token`; this weakens the intended main-agent safety boundary.
  - confidence: 0.96
- **[high] Mutating memory status flags can pass the allowlist:** `src/agents/bash-tools.exec.ts:228`
  - body: The proposed `memory status` allowance accepts arbitrary trailing args, including documented repair and reindex modes such as `--fix` and `--index`.
  - confidence: 0.95
- **[high] Boundary trigger can be skipped by pipes and Linux paths:** `src/agents/bash-tools.exec.ts:178`
  - body: The trigger regex omits pipe boundaries and hard-codes macOS absolute paths, allowing pipe-right OpenClaw commands and Linux absolute OpenClaw invocations to avoid the new enforcement path.
  - confidence: 0.9
