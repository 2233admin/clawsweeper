## Summary

Keep this PR open. Current main at 7a23b2d945cc2d2e5c452e4641213e71de8dcc66 still does not implement the PR’s central hardening across the managed session directory creation paths. There is a small partial improvement in the shared Pi session header helper, but the main store, transcript, write-lock, setup/onboarding, model probe, and Gateway agent-create paths still use plain recursive mkdir without a managed/custom split, post-create chmod hardening, or symlink-ancestor guard.

## Best Possible Solution

Keep the PR open as an implementation candidate. The best path is a focused, maintainer-reviewed shared helper in the session path/store layer that creates managed session directories privately, hardens existing managed dirs safely, rejects unsafe symlinked managed path components before permission mutation, preserves plain mkdir behavior for arbitrary custom store/transcript paths, wires setup/onboarding/agents.create/store/transcript/lock/model-probe/shared runner paths through it, and adds targeted regression tests for default roots, templated session.store roots, custom-path non-regression, and symlink rejection.
