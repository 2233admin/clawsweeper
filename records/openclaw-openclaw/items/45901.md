## Summary

Keep this PR open. Current main still does not implement the PR's central security hardening: managed session store, transcript, setup/onboarding, lock, and agent-create paths still create session directories with plain recursive mkdir and no managed/custom split, chmod hardening, or symlink guard. The PR remains a plausible implementation candidate, but it is XL and security-sensitive enough to need maintainer review rather than automated closure.

## Best Possible Solution

Keep the PR open as an implementation candidate, but require a maintainer-reviewed update against current main. The best path is a focused shared helper in the session path/store layer that privately creates and hardens managed session directories, preserves plain mkdir behavior for arbitrary custom store/transcript paths, validates symlinked managed path components before permission mutation, wires setup/onboarding/agents.create/store/transcript/lock/model-probe/Pi runner paths through it, and adds targeted regression tests for default roots, configured {agentId} store templates, custom-path non-regression, and symlink rejection.
