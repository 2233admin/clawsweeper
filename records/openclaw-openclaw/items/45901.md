## Summary

Keep open. Current main still creates managed session store, transcript, lock, setup, onboarding, and Gateway agent session directories through plain recursive mkdir paths, so the hardening request is not implemented on main. The PR is useful source work, but it should not be merged or auto-closed yet because the current PR head still has a TypeScript type error in the new helper and the filesystem hardening needs maintainer review.

## What This Changes

The PR adds a private managed session-directory helper, wires session store/transcript/setup/onboarding/agent-create/write-lock/probe paths through it, preserves plain mkdir behavior for custom paths, and adds permission plus symlink regression tests.

## Best Possible Solution

Land a focused, maintainer-reviewed implementation that centralizes managed session directory creation with private POSIX permissions, preserves custom-path behavior, rejects unsafe symlink traversal before permission mutation, fixes the FileHandle type import, and validates the touched session/setup/Gateway paths before merge.
