## What This Changes

The PR preserves configured workspace directories during state-only uninstall, keeps workspace deletion destructive when selected, adds regression coverage, and updates uninstall docs.

## Solution Assessment

Yes, with one merge-hygiene fix. Selectively deleting state contents while preserving configured workspace paths is narrower and more complete than skipping state deletion entirely, and the PR keeps `--workspace`/`--all` intentionally destructive.
