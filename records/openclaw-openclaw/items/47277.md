## Summary

Keep this PR open. Current main still uses shared memory discovery/read behavior and does not implement user-specific memory isolation, while the PR remains a focused but security-sensitive implementation candidate that needs maintainer review and rework before it can land.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a revised memory-core and memory-host-sdk implementation with regression tests for builtin and QMD backends, read/search scoping, session-key and requesterSenderId identity derivation, unsafe path and userId inputs, disabled-isolation compatibility, config schema/docs, and manager cache/watcher lifecycle.

## Risks / Open Questions

- This PR changes a privacy boundary for shared-agent memory, so memory_search, memory_get, builtin indexing, QMD indexing/fallback, path normalization, and identity derivation need consistent enforcement before merge.
- Default-on isolation is a config and behavior contract change; it needs schema, docs, compatibility, and migration review.
- Per-user managers or cache keys can create long-lived watcher/database resource risks if implemented without bounded lifecycle management.
