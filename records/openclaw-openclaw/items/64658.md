## Summary

Keep open. Current main still lacks a documented native `sessions_spawn` option to choose silent, parent-only, or channel completion delivery; the existing native path still defaults to completion announcement delivery for top-level channel-bound requester sessions, while ACP-only `streamTo: "parent"` and the hidden raw `expectsCompletionMessage` boolean do not satisfy the requested public per-spawn policy.

## What This Changes

The issue requests a `sessions_spawn` option to suppress or redirect sub-agent completion announcements instead of always delivering them to the inherited requester channel.

## Best Possible Solution

Add a documented native sub-agent completion delivery policy that keeps current channel delivery as the default while supporting parent-only internal injection and silent completion handling; coordinate naming and config semantics with #13911 so per-call and per-channel suppression remain consistent.
