## Summary

Keep open. Current main still lacks configurable `sessions_spawn` label templates, while this PR is a relevant but incomplete implementation with unresolved target-agent/requester-config semantics plus missing tests, docs, changelog, and generated schema follow-through.

## What This Changes

The PR adds an optional `labelTemplate` field under `agents.defaults.subagents` and `agents.list[].subagents`, then uses it to generate a spawned session label when `sessions_spawn.label` is omitted.

## Best Possible Solution

Land a complete label-template implementation that preserves explicit-label precedence, computes fallback labels after requester and target agent resolution, applies requester-agent template overrides with defaults fallback, covers native and ACP spawn paths consistently, and updates config types, strict schemas, generated schema artifacts, docs, changelog, and regression tests.
