## Summary

Keep this issue open. Current main has useful nearby mitigations, including active-run cleanup on session reset and Anthropic thinking-block replay recovery, but bare `/new` and `sessions.reset` still preserve user-selected model and thinking/reasoning state. That means the reported Opus/Kimi reasoning-loop recovery problem is not proven fixed.

## Best Possible Solution

Keep this issue open for a deliberate recovery design. The likely product path is an explicit stuck-run recovery command or reset mode that can abort the active run and optionally clear thinking/reasoning and model overrides, while preserving current `/new` semantics. If maintainers decide current behavior is intended, docs and command UX should state that bare `/new` preserves selected model/thinking and recommend `/think off`, `/new <model>`, or a model switch for reasoning-loop recovery.
