## Summary

Keep this issue open. Current `main` has useful reset recovery pieces, including active-run cleanup and clearing auto fallback model pins, but the reported central problem is still not implemented: a bare `/new` or `sessions.reset` intentionally preserves user-selected model plus session thinking/reasoning overrides, so it is not proven to recover an Opus/Kimi reasoning loop without a model or thinking change.

## Best Possible Solution

Keep this open for a deliberate stuck-run recovery design in core session/run controls. The likely path is an explicit reset/recovery option that aborts the active run and can clear selected model, thinking, and reasoning overrides back to defaults, while preserving existing `/new` semantics. If maintainers decide current behavior is intended, document the recovery path clearly, such as `/think off`, `/new <model>`, or switching away from the affected model before resetting.
