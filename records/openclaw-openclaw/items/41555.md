## Summary

Keep open. Current main still treats `/new`, `/reset`, and `sessions.reset` as session/transcript resets that preserve explicit model selection plus sticky thinking/reasoning settings. That does not implement the reported recovery behavior where a bare `/new` clears a stuck Opus or Kimi thinking state without a model or thinking-mode change.

## Best Possible Solution

Keep this open for an explicit stuck-run recovery design. A conservative path is a separate recovery/reset mode that aborts active work and can clear selected model plus thinking/reasoning overrides back to defaults, while preserving current `/new` semantics unless maintainers intentionally change them. If current behavior is intended, document the recovery path clearly, such as `/think off`, `/new <model>`, `/model`, or a stronger reset command that includes model/thinking cleanup.
