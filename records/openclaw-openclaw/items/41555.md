## Summary

Keep this issue open. Current main has useful recovery pieces, including active-run cleanup on reset and Anthropic thinking-block replay recovery, but bare `/new` and `sessions.reset` still preserve the selected model plus session thinking/reasoning overrides. The reported Opus/Kimi reasoning-loop recovery problem is therefore not proven fixed by current reset behavior.

## Best Possible Solution

Keep this issue open for a deliberate stuck-run recovery design. The likely path is an explicit command or reset option that aborts the active run and can clear model, thinking, and reasoning overrides, while preserving current `/new` semantics. If maintainers decide current behavior is intended, docs and command UX should clearly recommend `/think off`, `/new <model>`, or a model switch for reasoning-loop recovery.
