## Summary

Keep open. Current main still lacks the requested WhatsApp pre-built link preview pass-through, has no opt-in WhatsApp socket/config setting for Baileys high-quality preview fetching, and exposes only the existing single-message Gateway send RPC. The related reports and PRs are useful context, but #43397 was closed unmerged, #18414 is not reflected in the current send path, and #6722 was closed as superseded by this broader tracker.

## What This Changes

The issue requests WhatsApp outbound support for pre-built Baileys link previews, an opt-in high-quality preview fetch setting, and a batch send Gateway RPC for bulk message workflows.

## Best Possible Solution

Split the work into staged changes: first add a typed WhatsApp-owned pre-built linkPreview pass-through through the outbound/Gateway send path with tests and docs, then handle automatic URL-preview fetching and batch Gateway sending as separate security and protocol designs.
