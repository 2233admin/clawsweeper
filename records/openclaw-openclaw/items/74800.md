## What This Changes

The issue asks OpenClaw to handle Telegram/plugin command handlers that return undefined without throwing or surfacing Telegram's generic error reply.

## Best Possible Solution

Normalize nullish and non-object plugin command handler results at the shared executor boundary, preserve existing continueAgent semantics, add regression coverage for the shared text-command path and Telegram native plugin commands, then close the duplicate fix PRs that are not selected.

## Reproduction Assessment

Yes. A plugin command handler that omits an explicit return reproduces the source-level path: executePluginCommand() returns undefined, handlePluginCommand reads result.continueAgent, and Telegram dispatch converts the thrown TypeError into the reported generic fallback.

## Solution Assessment

Yes. The requested nullish-result guard is the right narrow fix, but the most maintainable form is central normalization plus focused caller-side regression tests rather than a Telegram-only patch.
