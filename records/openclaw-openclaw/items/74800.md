## Summary

Keep open: current main still has the reported nullish plugin-command result crash path, and this issue is already paired with open closing PRs. The maintainer path is to review and land one canonical fix, not close this issue before the fix merges.

## What This Changes

Make Telegram/plugin command handling tolerate a plugin command handler that returns undefined instead of throwing and surfacing Telegram's generic error reply.

## Best Possible Solution

Land one focused implementation that normalizes nullish or invalid plugin command handler results at the shared executePluginCommand boundary, preserves continueAgent behavior, and adds regression coverage for the shared text command path plus Telegram native command handling.

## Reproduction Assessment

Yes. A synthetic registered plugin command whose handler omits a return can reproduce the source-backed failure path: executePluginCommand returns undefined, then handlePluginCommand reads result.continueAgent and Telegram surfaces the generic dispatch fallback.

## Solution Assessment

Yes. The narrowest maintainable fix is shared result normalization in src/plugins/commands.ts, with defensive coverage at the two current consumers only if needed; fixing only the Telegram caller would leave other plugin command consumers exposed.
