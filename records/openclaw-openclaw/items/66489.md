## Summary

Keep open. Current main still has the local-media allowlist rejection and the delivery-queue permanent-error classifier still does not match it, so queued failures continue through the retry path rather than moving directly to failed.

## What This Changes

Classify deterministic outbound delivery failures such as `Local media path is not under an allowed directory` as permanent queue failures instead of retryable delivery errors.

## Best Possible Solution

Add a focused permanent delivery classification for the exact local-media allowlist rejection in the delivery-queue policy, with policy and recovery coverage showing that poisoned queued entries move directly to `failed/`. Treat enqueue-time validation, `lastErrorClass`, and Telegram overlong-message handling as follow-ups unless they are separately verified in the same PR.
