## Summary

Keep this PR open. Current main still wires the ACP stdio stream directly into `AgentSideConnection`, and the pinned ACP SDK validates `initialize.params.protocolVersion` as a uint16 before `AcpGatewayAgent.initialize()` runs. There is no current-main coercion for MCP date-string protocol versions such as `"2025-11-25"`, so the PR still addresses the linked open bug rather than being obsolete.

## Best Possible Solution

Keep the PR open as the active implementation candidate for the linked ACP handshake bug. Best path is to review or update the PR with a small regression test around inbound `initialize` protocol-version coercion, address the two minor review comments, then land this or an equivalent `src/acp/server.ts` stream wrapper so the linked issue closes through the merge.
