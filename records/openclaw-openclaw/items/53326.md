## Summary

Current main has not implemented this PR’s intended behavior: workspace setup state is still resolved and tested at `<workspace>/.openclaw/workspace-state.json`. The PR is a plausible focused fix for the dot-prefixed workspace-state problem and the nested path reported in #44783, but it still needs maintainer review and likely revision for the review-commented edge cases before it should land.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The likely path is to keep the workspace-state move in `src/agents/workspace.ts`, but add focused tests and harden the design: use a collision-resistant non-dot state filename or foreign-file guard, treat legacy `.openclaw/workspace-state.json` reads as best-effort on dot-restricted filesystems, and promote legacy state to the new root path once read.
