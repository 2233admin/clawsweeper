Keep open. The thread remains the canonical tracker for the plugin-loader/runtime-deps gateway starvation cluster: current main contains substantial mitigations, but the latest discussion includes post-v2026.4.27 residual CPU, event-loop delay, and RSS-growth evidence, and I did not find affected-host proof that the broader regression is fully resolved.

Fix the bundled plugin loader/runtime-deps hot loop that repeatedly prepares runtime mirrors, reparses manifests or restages dependencies, saturates CPU, and starves Gateway RPC/channel handling in packaged installs.
