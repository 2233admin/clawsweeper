## Summary

Keep open. Current main contains narrow mitigations for repeated packaged dist-mirror preparation and manifest/file-decision caching, but it still retains the specs-only runtime-deps manifest and scanner/retained-staging paths reported in the issue. The issue is also paired with open PR #74246, which is linked as the intended broader fix, so it should stay open until that PR is reviewed, landed, or closed.

## What This Changes

Fix the plugin loader/runtime-deps hot loop that repeatedly prepares bundled plugin dist mirrors, reparses manifests or restages deps, saturates CPU, and starves the Gateway event loop in packaged installs.

## Best Possible Solution

Keep this issue open while maintainers review, land, or close PR #74246. If that PR lands, retest affected packaged installs and then close this issue as implemented with the merge SHA and release provenance. If it does not land, the best fix remains a focused plugin/runtime-deps change that replaces specs-only retained state with durable plan/hash install state, removes runtime JS mirror dependency inference from Gateway startup, preserves source-checkout refresh behavior, and proves idle Gateway/channel/agent turns stop rewalking or reinstalling runtime deps.
