## Summary

Keep open. Current main still only partially covers the reported Feishu compatibility break: `groupPolicy: "allowall"` is normalized and legacy `topicSessionMode` still maps to topic routing, but legacy `botId` is absent from the strict Feishu schemas, credential resolution, setup/status detection, manifest metadata, doctor compatibility normalizer, and docs. The linked #52885 is a closed Weixin/plugin-SDK export issue, not a canonical tracker for this Feishu config migration.

## What This Changes

Request backward-compatible handling and migration for changed channel configuration fields, especially Feishu `botId` to `appId`, `topicSessionMode` to `groupSessionScope`, and `groupPolicy: "allowall"` to `"open"`.

## Best Possible Solution

Add a Feishu-owned compatibility path that accepts legacy top-level and per-account `botId`, normalizes it to canonical `appId`, exposes an `openclaw doctor --fix` migration for stored configs, keeps existing `allowall` and `topicSessionMode` behavior intact, and documents the upgrade path without adding a startup-time core migration.
