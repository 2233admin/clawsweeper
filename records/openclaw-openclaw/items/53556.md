## Summary

Keep this open. Current main only partially covers the requested Feishu compatibility: legacy `groupPolicy: "allowall"` and `topicSessionMode` are handled, but `botId` -> `appId` remains missing from schema, credential/status resolution, docs/metadata, and doctor migration hooks. The linked #52885 is a closed Weixin plugin-SDK issue, not a canonical Feishu config migration tracker.

## Best Possible Solution

Keep this open and implement the remaining Feishu-owned compatibility: accept and normalize top-level and per-account `botId` to `appId`, preserve `appId` precedence when both fields exist, wire `openclaw doctor --fix` through Feishu `legacyConfigRules` or `normalizeCompatibilityConfig`, update credential/status/setup tests, and document the upgrade path on the public Feishu channel docs.
