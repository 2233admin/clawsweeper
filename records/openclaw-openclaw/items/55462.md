## Summary

Keep open. Current main still does not expose the source Telegram callback message_id to the generic agent callback path, and PR #55548 is still the unmerged implementation candidate linked with closing intent for this issue.

## Best Possible Solution

Keep this issue open until PR #55548 or an equivalent maintainer-reviewed fix lands. The fix should carry callbackMessage.message_id and chat_id from Telegram callback_query into agent-visible metadata or tool context, add focused regression coverage for inline-button callbacks driving editMessage, and preserve existing MessageSid/dedupe semantics unless maintainers intentionally change them.
