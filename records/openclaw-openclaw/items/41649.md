## Summary

Keep this PR open. Current main still lacks the model-change transcript archive, session-id rotation, and one-shot sanitized user-message handoff that this PR is trying to add. Model-related config changes currently reset caches/heartbeat only, while persisted session runtime model fields still take precedence over configured defaults. The narrower related PR #41907 is closed unmerged and does not supersede this work.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is a focused core session/gateway change that detects default-model config changes, preserves explicit user overrides, clears stale runtime model/context metadata, archives or rotates affected transcripts, stores any handoff data with clear privacy and retention semantics, and delivers sanitized one-shot user context only when an agent run will actually proceed. Add coverage for hot reload, restart reload, missing transcript files, multi-session retention, explicit overrides, and first-turn handoff delivery.
