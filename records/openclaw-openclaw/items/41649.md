## Risks / Open Questions

- The proposed handoff JSON stores recent user-message excerpts separately from the transcript, so privacy, file mode, retention, cleanup, and support-bundle/export interactions need explicit maintainer review.
- The PR's final one-shot consumption path reads and deletes the handoff before later early-return guards in runPreparedReply, so a first-turn event that does not reach a model run can lose the handoff.
- The proposed cleanup keeps only five handoff files per sessions directory, which can delete fresh handoffs for active sessions when more than five sessions are affected by one model switch.
- The startup-side reset clears persisted model fields on every gateway start rather than only after a detected model-default change; that may be too broad without a persisted previous-config comparison.
