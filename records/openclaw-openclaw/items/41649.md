## Summary

Keep this PR open. Current main still does not implement the PR's intended model-change transcript archiving or handoff behavior, and it also lacks the narrower `reset-session-models` hot-reload action from the earlier version. The related narrower PR #41907 is closed unmerged, so there is no canonical merged replacement to close this PR against.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is a focused core session/gateway implementation that detects default-model config changes, preserves explicit user model overrides, clears stale runtime model/context metadata, rotates or archives affected transcripts, and delivers sanitized one-shot user-context handoff only after a run will actually proceed, with tests covering hot reload, restart reload, multi-session retention, and first-turn delivery.
