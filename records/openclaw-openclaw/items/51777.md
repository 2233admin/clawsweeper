## Summary

Keep this PR open. Current main still has the exact Feishu onboarding persistence gap: the DM policy prompt skips `setPolicy` when the selected default equals the fallback current value, Feishu still reports missing `dmPolicy` as `"pairing"`, and config writes intentionally do not persist schema/runtime defaults. The PR is focused and low-risk from a security/supply-chain perspective, but it needs review because the existing discussion correctly notes that a global unconditional `setPolicy` can mutate non-idempotent channel settings.

## Best Possible Solution

Keep the PR open and either land a focused Feishu fix with a regression test, or introduce a small generic missing-policy detection seam so onboarding can persist an explicitly selected default only when the backing config key is absent. Avoid a blanket unconditional `setPolicy` call until non-idempotent channel setters such as Twitch are fixed or isolated.
