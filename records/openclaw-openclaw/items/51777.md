## Summary

Keep this PR open. Current main still has the Feishu explicit-default persistence gap, but the PR now targets an old implementation path, so it needs to be rebased/adapted to the current channel setup flow and covered with a regression test rather than closed.

## Best Possible Solution

Keep the PR open for maintainer review. Rebase it onto current main, implement the fix in `src/flows/channel-setup.prompts.ts` either as a focused Feishu missing-key write or a small generic explicit-policy-presence seam, and add a regression test for fresh Feishu config selecting `pairing`. Avoid a global unconditional `setPolicy` call until non-idempotent channel setters are handled.
