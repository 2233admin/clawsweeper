## Summary

Keep this issue open. Current main implements a useful subset of the request, but it does not yet solve the central conversation-history attachment workflow: session transcript indexing still flattens messages to text and ignores persisted media path/type fields. Video and PDF are also not part of the current memory multimodal modality set, and current upstream Gemini Embedding 2 docs now describe a broader stable model surface than the repo’s preview-only config/docs.

## Best Possible Solution

Keep this issue open and narrow it to the missing session-memory attachment path. Extend session transcript indexing to collect eligible persisted media path/type fields, resolve them through existing managed-media and sandbox safety boundaries, and pass structured embedding inputs for supported attachments while preserving the shipped Gemini Embedding 2 preview, output-dimensionality, and extra-path image/audio indexing behavior. Add regression coverage for chat or Telegram attachment recall and update docs to distinguish extra-path media indexing from conversation-history attachment indexing, plus any future video/PDF support.
