## Summary

Keep this issue open. Current main already implements Gemini Embedding 2 support, configurable output dimensionality, and opt-in image/audio multimodal indexing for files under `memorySearch.extraPaths`, but it still does not implement the requested conversation-history attachment indexing workflow for chat/Telegram media. Session-memory indexing flattens transcripts to sanitized text and never turns `MediaPath`/`MediaPaths` into structured embedding inputs.

## Best Possible Solution

Keep this open and narrow the remaining work to conversation-history attachment indexing. The best path is to extend session-memory indexing to discover eligible `MediaPath`/`MediaPaths` plus MIME metadata from transcript messages, resolve only safe managed-media/sandbox paths, build structured `EmbeddingInput` chunks for provider-supported attachments, add focused regression coverage for chat/Telegram attachment recall, and document the difference between `extraPaths` multimodal memory and session transcript attachment indexing.
