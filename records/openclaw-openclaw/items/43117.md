## Summary

Keep this issue open. Current main already supports part of the request: Gemini Embedding 2, `outputDimensionality`, and structured image/audio embedding for memory files under configured extra paths. The central conversation-history attachment workflow is still missing because session transcript indexing flattens stored messages to text and does not consume persisted media path/type fields. Video and PDF are also outside the current multimodal memory modality set.

## Best Possible Solution

Keep this issue open and narrow it to the missing session-memory attachment path. Extend session transcript indexing to collect eligible persisted `MediaPath`/`MediaPaths` plus `MediaType`/`MediaTypes`, resolve those paths through existing media safety boundaries, and pass structured Gemini embedding inputs for supported attachments while preserving the shipped Gemini Embedding 2, `outputDimensionality`, and extra-path image/audio indexing behavior. Add regression coverage for chat or Telegram attachment recall and update docs to distinguish memory-file media indexing from conversation-history attachment indexing and any future video/PDF support.
