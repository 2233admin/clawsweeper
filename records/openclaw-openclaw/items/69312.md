## Summary

Keep this PR open. Current main still lacks protection for indented-code `MEDIA:` lines, so the PR addresses a real bug, but the latest patch should not merge until MEDIA case semantics are aligned with existing block-reply and streaming behavior.

## What This Changes

The PR changes media parsing to skip `MEDIA:` directives in indented code blocks, preserve retained code indentation during cleanup, tighten fenced-code closing detection, remove case-insensitive media matching, and add media/fence regression tests.

## Best Possible Solution

Land a focused version after maintainers decide the MEDIA case contract and the patch updates every affected parser surface consistently. Keep the indented-code protection and code-indentation preservation, validate shared fence consumers, and add a user-facing changelog fix entry if merged.
