## Summary

Keep this PR open. Current main still has the reported MEDIA extraction behavior for 4-space/tab indented code blocks, and the unmerged PR remains the active implementation candidate rather than cleanup material. The linked issue #69311 being closed does not make this PR obsolete because the intended parser and regression-test changes are not on main.

## Best Possible Solution

Keep this PR open for maintainer review or landing of a corrected version. The best path is to add indented-code protection in `src/media/parse.ts`, preserve code indentation during cleaned-text normalization, include the media and fence regression tests, and explicitly align MEDIA case handling between `splitMediaFromOutput` and streaming directive buffering before merge.
