## Summary

Keep this PR open. Current main still has the MEDIA false-positive path the PR targets: `splitMediaFromOutput` only protects fenced code spans, trims indentation before checking for `MEDIA:`, and has no indented-code-block state. The PR is open, active, unmerged, and its parser/test changes still appear relevant rather than obsolete.

## Best Possible Solution

Keep the PR open for maintainer review or a corrected maintainer-apply. The likely path is to add indented-code protection in `src/media/parse.ts`, preserve code indentation during media cleanup, add focused media/fence regression tests, and decide the MEDIA case-sensitivity contract consistently across `splitMediaFromOutput`, streaming directive buffering, and reply delivery tests.
