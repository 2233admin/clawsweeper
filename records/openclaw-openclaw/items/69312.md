## Summary

Keep this PR open. Current main still lacks indented-code protection in `splitMediaFromOutput`, so the central bug is real, but the PR needs maintainer review because it also changes lowercase `media:` compatibility and shared fence parsing behavior used outside the media parser.

## What This Changes

The PR updates media parsing to skip `MEDIA:` directives inside indented code blocks, preserves retained code indentation during cleanup, tightens markdown fence closing validation, removes case-insensitive `MEDIA:` matching, and adds parser/fence regression tests.

## Best Possible Solution

Keep this PR open for maintainer review.
