## Summary

Keep this PR open. Current main still has the false-positive path the PR targets: `splitMediaFromOutput` only protects fenced code spans, then `line.trimStart()` lets 4-space or tab-indented `MEDIA:` lines enter extraction. The PR is open, unmerged, and remains a plausible implementation candidate; the closed linked issue does not make the unmerged PR obsolete.

## Best Possible Solution

Keep the PR open for maintainer review or a corrected merge. The best path is to land an indented-code guard in `src/media/parse.ts`, preserve code indentation during cleaned-text normalization, add the media and fence regression tests, and explicitly align streaming-directive MEDIA case handling with the parser before merging.
