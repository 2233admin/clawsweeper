## Summary

Keep this PR open. Current main still has the reported indented-code MEDIA extraction bug, and the PR is a relevant implementation candidate, but the latest patch still introduces inconsistent lowercase MEDIA handling and can strip tab indentation from retained code after `[[audio_as_voice]]` cleanup.

## What This Changes

The PR updates media and fence parsing plus regression tests to skip `MEDIA:` extraction in indented code blocks, preserve retained code indentation, reject invalid fence closers with trailing text, and make MEDIA token matching case-sensitive.

## Best Possible Solution

Land a focused version that keeps the indented-code protection and retained-code indentation handling, but first makes MEDIA case semantics consistent across parser, block-reply, streaming buffering, docs, and tests, and reuses code-line state when normalizing text after `[[audio_as_voice]]`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep MEDIA case semantics aligned:** `src/media/parse.ts:17`
  - body: Removing the case-insensitive flag makes `splitMediaFromOutput` ignore lowercase `media:`, but current block-reply coverage still expects `media: ./report.pdf` to produce `mediaUrls`, and streaming still buffers lowercase/mixed-case MEDIA prefixes with `/i`. This can expose lowercase directives as text in one path while buffering or dropping them in another, so update all directive surfaces/tests together or preserve the existing case-insensitive contract.
  - confidence: 0.9
- **[P2] Preserve tab code indentation after audio tags:** `src/media/parse.ts:738`
  - body: The `audio_as_voice` cleanup path does not reuse `firstLineIsCode` or `keptLineIsCode`, so `[[audio_as_voice]]` followed by a tab-indented code line can lose its leading tab during post-processing. That turns retained code like `\tMEDIA:/tmp/code.png` into top-level-looking `MEDIA:` text and undercuts the indentation-preservation behavior this patch adds.
  - confidence: 0.86
