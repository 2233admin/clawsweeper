## Summary

Keep this PR open. Current main still has the indented-code false-positive path this PR targets: `splitMediaFromOutput` only protects fenced spans, trims indentation before the `MEDIA:` gate, and lacks the proposed indented-code regression coverage. The PR remains a relevant implementation candidate, but the shared fence parser change and MEDIA case-sensitivity behavior still need maintainer review before merge.

## Best Possible Solution

Keep this PR open for maintainer review or maintainer-apply. The best path is to land a scoped fix in `src/media/parse.ts` that protects CommonMark-style indented code lines and preserves retained code indentation, add focused `src/media/parse.test.ts` and `src/markdown/fences.test.ts` coverage, and explicitly decide whether MEDIA directives are case-sensitive across `splitMediaFromOutput`, streaming buffering, and reply delivery. If the shared fence parser correction is considered too broad for this bug, split or review it as a separate parser behavior change rather than dropping the media fix.
