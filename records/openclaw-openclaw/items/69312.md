## Summary

Keep open. Current main still has the reported indented-code MEDIA extraction bug, and this PR is useful work toward fixing it, but the latest patch is not ready because it regresses existing lowercase MEDIA compatibility and still has a code-indentation loss path after audio tag stripping.

## What This Changes

The PR updates media and fence parsing plus regression tests to skip MEDIA directives in indented code blocks, preserve retained code indentation, validate fence closing lines more strictly, and make MEDIA token matching case-sensitive.

## Best Possible Solution

Land a focused version that keeps the indented-code protection and code-aware whitespace preservation, either preserves lowercase MEDIA compatibility or intentionally updates every parser, streaming, docs, and test surface together, and reuses the same code-line-aware leading normalization after audio tag removal. If the fence-closing change stays in scope, validate the shared chunking, code-span, canvas, and PI embedded consumers before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep MEDIA case semantics aligned:** `src/media/parse.ts:17`
  - body: Removing the `i` flag makes `splitMediaFromOutput` ignore lowercase `media:`, but block-reply parsing still enters directive parsing for `/media:/i` and current coverage expects lowercase directives to become media. Streaming buffering also remains case-insensitive, so lowercase directives can be buffered or parsed inconsistently instead of producing the existing `mediaUrls` behavior.
  - confidence: 0.9
- **[P2] Preserve code indentation after audio tags:** `src/media/parse.ts:738`
  - body: The audio-tag cleanup path still applies its own leading-normalization regex after `[[audio_as_voice]]` is stripped, without checking whether the first surviving content line is code. Inputs such as `[[audio_as_voice]]\n\n\tMEDIA:/tmp/code.png` can lose the leading tab, undermining the indentation preservation this patch is adding.
  - confidence: 0.86
