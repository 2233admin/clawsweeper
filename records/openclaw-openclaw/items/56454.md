## Summary

Current main already implements this PR's user-visible fix through the shipped media-understanding config surface: `tools.media.image.timeoutSeconds` and matching per-model image `timeoutSeconds` entries. That path is documented, tested, and shipped in `v2026.4.26`; the PR's proposed `agents.defaults.imageTimeoutSeconds` key is not the canonical current API.

## Best Possible Solution

Close this PR as implemented on main and keep the shipped media-understanding config surface. Users with slow vision providers should use `tools.media.image.timeoutSeconds` or a matching per-model `tools.media.image.models[].timeoutSeconds`; related reports such as #56419 and #64212 should be triaged against `v2026.4.26` or newer.
