## Summary

Keep open. Current main still only partially satisfies the request: backend `sessions.create`/`sessions.patch`, labels, existing-session switching, and the separate Sessions page exist, but the visible WebChat New Session button still resets the current transcript via `/new` instead of creating and switching to a new labeled dashboard session. Open maintainer-labeled PR #73725 remains the active candidate for the focused WebChat create/switch slice, while agent-controlled create/relabel and broader tab/sidebar/rename/delete scope still need maintainer scoping.

## What This Changes

Add multiple isolated labeled WebChat sessions with create/switch UI, custom labels, and an agent-facing session label/create API while preserving the default main session.

## Best Possible Solution

Keep this issue open as the canonical WebChat multi-session tracker. Review, adjust if needed, and land or close PR #73725 for the focused New Chat -> `sessions.create` create/switch flow. After that, explicitly split or scope the remaining agent-controlled create/relabel API plus broader rename/delete/sidebar/tab session-management work.
