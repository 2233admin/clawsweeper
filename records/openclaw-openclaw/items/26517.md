## Summary

Current main still does not implement the central request. Backend session labels plus `sessions.create`/`sessions.patch` exist, and WebChat can switch among existing sessions, but the visible WebChat New Session button still sends `/new`, which current code and tests define as resetting the current transcript. Open maintainer-labeled PR #73725 remains the active implementation candidate for the narrow WebChat create/switch slice, while agent-controlled create/relabel and broader tab/sidebar/rename/delete session management remain unresolved.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review, adjust if needed, and land or close PR #73725 for the narrow WebChat New Session -> `sessions.create` create/switch flow. After that, explicitly split or scope the remaining agent-controlled create/relabel API plus broader rename/delete/sidebar/tab session-management work.
