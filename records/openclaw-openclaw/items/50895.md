## Summary

Keep this PR open. Current main still does not implement Brazilian WhatsApp ninth-digit equivalence, and the PR remains a relevant fix candidate for the linked #20187 behavior, but it needs a careful port/revision before it can land.

## Best Possible Solution

Keep this PR open and ask for a rebase/port onto current main. The revised fix should add a narrow WhatsApp direct-target equivalence helper in the bundled plugin, use it for allowlist, command, self-chat, main-last-route, and heartbeat authorization checks, preserve the original explicit chat target for reactions, quote lookup, outbound delivery, and per-peer session routing, and add focused regression tests under the current `extensions/whatsapp` paths. Avoid expanding the public SDK unless maintainers confirm a third-party plugin API need.
