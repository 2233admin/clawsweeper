## Summary

Current main still treats browser `act` evaluate input as a parenthesized JavaScript expression in the managed Playwright path, so statement bodies beginning with `var`, `let`, or `const` still fail before execution. The public surface remains `kind: "evaluate"` with an `fn` string and no statement mode or replacement `browser.exec`; the related closed #55598 does not supersede this narrower request.

## Best Possible Solution

Keep this issue open and implement statement support in the browser plugin evaluate path, likely around `evaluateViaPlaywright`, by detecting expression versus statement bodies or adding a compatible explicit mode. Preserve current expression/function/ref behavior and return values, keep timeout/abort/navigation guards intact, and add focused tests for page and ref evaluate with `var`, `let`, `const`, async statement bodies, and existing expression/function inputs.
