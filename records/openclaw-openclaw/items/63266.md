Keep open. Current `origin/main` still has the legacy workspace doctor hook, but `detectLegacyWorkspaceDirs()` returns an empty list unconditionally, so the reported `workspace-local` / `workspace-local/local` split is not detected, warned about, repaired, or specifically documented. The issue is recent, unlabelled, author association is `NONE`, and there is no open PR found by a read-only search that references it.

## Best Possible Solution

Keep this open and implement a narrow doctor/docs fix: detect the `workspace-local` parent plus active `workspace-local/local` child layout when parent-level operational files such as `secrets/`, workspace bootstrap files, or plugin/workspace state exist; report both paths clearly; document canonical Docker-to-host workspace and secret/config locations. Do not add startup-time migration, and do not move/copy/delete secrets without explicit maintainer-approved `doctor --fix` behavior.
