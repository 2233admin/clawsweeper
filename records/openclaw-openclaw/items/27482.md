## Summary

Current main still does not support direct video upload to LLM chat models: the Control UI filters video files, tests assert that MP4 is rejected, Gateway native prompt attachments are image-only plus generic offloaded media, and the model catalog input type does not include video. Open PR #74082 now directly targets this feature, so the issue should stay open while maintainers review or land that PR rather than queueing duplicate repair work.

## What This Changes

Request direct MP4/WebM/MOV video uploads in chat/message tooling, with validation, preview, and delivery to LLMs that support native video input.

## Best Possible Solution

Keep this issue open as the product tracker and review/land #74082, or a maintainer-approved equivalent, that updates Control UI/WebChat video allowlists and previews, Gateway video validation and native/fallback attachment parsing, model-catalog video capability metadata, provider dispatch, docs, and regression tests.

- Open PR #74082 changes upload acceptance and provider payload behavior, so it needs normal maintainer/security review before this issue can be considered implemented.
