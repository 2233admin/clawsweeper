## Summary

Keep open. Current main still leaves the Control UI shell, advanced Config form, and Sessions table using the available content width, and the related chat-width issue/PR only covers a narrower chat readability path rather than the broader forms/tables ultrawide request.

## What This Changes

Issue requests ultrawide-aware Control UI layout patterns for forms and tables instead of stretching primary content across the full viewport.

## Best Possible Solution

Define a Control UI-wide responsive layout policy with shared tokens/patterns, then apply it per view: cap readable form/detail content, keep form actions visually tied to their sections, and handle data tables with constrained containers, useful side panels, or intentional density controls instead of default full-viewport stretch.

## Reproduction Assessment

Yes. Open the Control UI Config or Sessions view at a viewport wider than 2560px; source inspection shows the shared shell, Config layout, and Sessions table still consume the available content width.

## Solution Assessment

Unclear. The requested direction is valid, but the best maintainable fix is probably not a blanket max-width; it should combine shared layout primitives with view-specific decisions for Config, Sessions/data tables, Chat, Usage, and Quick Settings.
