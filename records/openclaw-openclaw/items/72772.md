## Summary

Keep open. Current main still lacks a shared ultrawide-aware Control UI layout policy for the shell, advanced Config form, and Sessions table; the related open chat-width issue and PR only cover chat readability and do not supersede this broader forms/tables request.

## What This Changes

Requested ultrawide-aware Control UI layouts for forms and tables, including capped main content widths, side-panel patterns, and action controls that stay visually tied to the relevant content.

## Best Possible Solution

Define reusable Control UI wide-screen layout primitives, then apply them per surface: cap readable form/detail content, keep section actions visually attached, and redesign wide tables with constrained containers, side filters, detail panels, or density-specific layouts where extra horizontal space adds value.

## Reproduction Assessment

Yes. Open the Control UI advanced Config form or Sessions view at a viewport wider than 2560px; source inspection shows those non-chat surfaces still fill the available content width while the shared shell lacks an ultrawide cap.

## Solution Assessment

Unclear. The requested direction is sound, but the best implementation is probably shared layout primitives plus targeted Config/Sessions/table changes, not a blanket max-width across every Control UI view.
