## Summary

Keep open. Current main still leaves the shared Control UI shell, Config editor, and Sessions table expanding across the available content column, so the app-wide ultrawide layout request is not implemented broadly enough to close.

## What This Changes

Request ultrawide-aware Control UI layouts that cap or restructure forms and tables instead of stretching primary content across very wide viewports.

## Best Possible Solution

Define a Control UI-wide responsive layout policy: cap readable form/detail content, intentionally handle data-dense tables with either constrained containers or useful side panels, and reconcile chat width preferences with the broader shell/table/form behavior.

## Reproduction Assessment

Yes. Open the Control UI Config or Sessions view at a viewport wider than 2560px and inspect the current `.content`, `.config-layout`, `.config-actions`, and `.data-table` rules; source inspection shows they still consume the available content column.

## Solution Assessment

Unclear. The proposed direction is valid, but the best maintainable fix is not a blanket max-width; it needs shared layout tokens/patterns plus view-specific decisions for Config, Sessions tables, chat, and Quick Settings.
