## Summary

Keep open. Current main still only partially addresses the request: non-verbose chat-visible tool warnings generally omit the underlying failure reason, and same-action retry success clears internal failure state without producing the requested user-visible confirmation. The remaining work is tied to chat/channel privacy policy rather than a narrow already-implemented bug fix.

## What This Changes

The issue requests chat-visible tool failure messages that include a failure reason and a later confirmation when a retry succeeds.

## Best Possible Solution

Define sanitized, allowlisted failure summaries by tool and channel, then add a retry-success confirmation only when a previously surfaced failure is resolved by a matching successful mutating action. Keep the broader diagnostic direction coordinated with #60946, the external-channel privacy constraints from #54757, and the narrower edit-mismatch work in #67276.

## Reproduction Assessment

Yes. A high-confidence unit-level reproduction exists by building a verbose-off payload with a mutating `lastToolError` and observing that the warning omits the detail, then simulating a failed edit followed by a matching successful edit and observing that only `lastToolError` state is cleared.

## Solution Assessment

No current implementation fully solves the issue. The closed unmerged PR #46592 was not the best complete fix because it exposed raw/truncated reasons without retry confirmation or channel privacy policy; the safer direction is sanitized reason summaries plus explicit retry-success semantics.
