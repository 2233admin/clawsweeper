## Summary

Keep open. Current main still only partially addresses the report: ordinary non-verbose tool warnings do not include the failure reason, and a matching successful retry clears internal error state without emitting a chat-visible retry-success confirmation. Expanding these diagnostics touches channel privacy policy, so this needs maintainer review before implementation.

## What This Changes

The issue requests that chat-surfaced tool failures include an actionable failure reason and that successful retries visibly confirm the failed operation was resolved.

## Best Possible Solution

Define a safe diagnostics policy for chat-visible tool failures, including redacted or allowlisted reason summaries by tool and channel, then emit retry-success confirmation only when a previously surfaced failure is resolved by the same successful mutating action.
