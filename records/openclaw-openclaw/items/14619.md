## Summary

Keep open. Current main still includes the human-readable Tooling list in the generated system prompt, still sends callable tool schemas separately via customTools, and still has no scoped skipToolListText/toolListText config option. The prior maintainer-side review in the issue reached the same conclusion, and current main still matches it.

## Best Possible Solution

Keep this open until OpenClaw adds a scoped agent/default option, for example agents.defaults.skipToolListText, that suppresses only the human-readable Tooling list while preserving callable tool schemas. The implementation should update prompt construction, run/config plumbing, config types/Zod/generated schema/help/docs, context reporting, and focused tests proving the prompt text is omitted while customTools remain available. Coordinate with the broader open #14785 work, but keep this issue as the specific tool-list-text optimization unless a maintainer intentionally makes another issue canonical.
