## Summary

Current main still renders the human-readable Tooling list in the system prompt, still submits callable tool schemas separately via customTools, and still has no scoped skipToolListText/toolListText config option. The narrower request is not superseded by the broader open tool-schema overhead work.

## Best Possible Solution

Keep this open and add an opt-in scoped agent/default setting, such as agents.defaults.skipToolListText with a per-agent override if maintainers want parity, that omits only the human-readable Tooling list from generated prompts while still passing callable tool schemas through customTools. Document the tradeoff and keep #14785 as the broader schema-overhead follow-up.
