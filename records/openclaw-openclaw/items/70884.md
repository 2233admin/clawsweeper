## Summary

Keep this PR open. Current main still has a real gateway documentation gap: the trusted-proxy docs and protocol docs do not clearly separate HTTP `x-openclaw-scopes` behavior from WebSocket/device-identity scope behavior, while the runtime does clear unbound scopes for device-less trusted-proxy WebSocket sessions. The PR is docs-only and addresses that gap rather than being superseded or implemented on main.

## Best Possible Solution

Review and land this PR, or land an equivalent docs update in docs/gateway/trusted-proxy-auth.md and docs/gateway/protocol.md that clearly says `x-openclaw-scopes` is for HTTP API endpoints, documents WebSocket scope clearing without device identity, preserves the local `allowInsecureAuth` token/password exception, and keeps `dangerouslyDisableDeviceAuth` framed as break-glass only.
