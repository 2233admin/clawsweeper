## Summary

Keep open. Current main still has the trusted-proxy/WebSocket docs gap this PR targets, while the gateway runtime and tests confirm the behavior being documented. The current PR diff is docs-only, the prior review comments appear addressed in the latest PR commits, and the remaining action is normal maintainer docs review rather than cleanup closure.

## What This Changes

This PR updates trusted-proxy and gateway protocol docs to clarify that `x-openclaw-scopes` applies to HTTP API requests, that device-less trusted-proxy WebSocket sessions have self-declared scopes cleared, and how operators should handle the resulting `missing scope` errors.

## Best Possible Solution

Review and land this PR, or land an equivalent docs update that clearly separates HTTP `x-openclaw-scopes` behavior from WebSocket handshake scope binding, documents scope clearing for device-less trusted-proxy WebSocket sessions, preserves the local token/password `allowInsecureAuth` exception, and keeps `dangerouslyDisableDeviceAuth` framed as a critical break-glass workaround.
