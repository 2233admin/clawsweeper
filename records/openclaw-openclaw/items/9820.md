## Summary

Keep open. Current main still does not implement native JSONL support for builtin memorySearch.extraPaths: file discovery admits Markdown plus enabled multimodal files, memory indexing still chunks non-session text as Markdown, memory_get rejects non-.md files, watcher filtering remains Markdown/multimodal-oriented, and docs describe extra paths as Markdown-only. The existing session transcript JSONL support is a separate opt-in session-memory path, not arbitrary JSONL ingestion from extraPaths.

## Best Possible Solution

Keep this open until maintainers either add native JSONL support to builtin memorySearch.extraPaths or explicitly document the supported alternative. A complete core fix should update file admission, JSONL parsing/chunking with line mapping, watcher filtering, memory_get behavior, config help/docs, and focused tests. If raw hook logs are out of scope for builtin memory, document the intended QMD/session-memory/converter-plugin workflow and close against that product decision.
