## Summary

Keep open. Current main still does not implement native JSONL support for builtin `memorySearch.extraPaths`: discovery admits Markdown plus enabled multimodal files, indexing chunks ordinary memory entries as Markdown text, `memory_get` rejects non-`.md` targets, watcher filtering excludes `.jsonl`, and docs still describe extra paths as Markdown-only. Existing session transcript JSONL support is separate opt-in session-memory handling, not arbitrary JSONL ingestion from extra paths.

## Best Possible Solution

Keep this open until maintainers either add native JSONL support to builtin `memorySearch.extraPaths` or explicitly document the supported alternative. A complete core fix should update file admission, JSONL parsing/chunking with line mapping, watcher filtering, `memory_get`, config help/docs, and focused tests. If raw hook logs are out of scope for builtin memory, document the intended QMD, session-memory, or converter-plugin workflow before closing.

## Clownfish Work Prompt

Implement native JSONL support for builtin memorySearch.extraPaths for issue https://github.com/openclaw/openclaw/issues/9820. Current main only indexes `.md` plus enabled multimodal files; arbitrary `.jsonl` files in extraPaths are ignored and `memory_get` rejects them. Keep the fix scoped to builtin memory extra paths, not QMD internals or session transcript behavior.

Expected boundary:
- Add a typed supported-text-file concept for builtin memory extra paths, including `.md` and `.jsonl` (and only add `.txt` if maintainers explicitly want it; the issue proposes it but the core ask is JSONL).
- Update `src/memory-host-sdk/host/internal.ts` so explicit extra `.jsonl` files and `.jsonl` files under extra directories are admitted while default workspace memory roots stay Markdown-only unless explicitly intended otherwise.
- Extend file entries/chunking so `.jsonl` entries are chunked line-aware. Prefer one JSONL record per logical line/chunk where feasible, preserve original line numbers for citations, tolerate blank/invalid JSON lines without crashing, and do not reuse OpenClaw session transcript sanitization unless the schema is actually a session transcript.
- Update `memory_get`/`readMemoryFile` to allow `.jsonl` paths only when they are inside configured extraPaths or are explicit configured extra files; keep the existing symlink/path-boundary protections.
- Update watcher filtering so `.jsonl` changes under configured extra paths trigger sync, without broadening ignored directories or multimodal behavior.
- Update config help and docs (`docs/reference/memory-config.md`, and related memory docs if needed) to state that builtin extra paths support Markdown and JSONL, with any limitations.
- Add focused tests for discovery, indexing/chunk line mapping, watcher filtering, and `memory_get` JSONL reads. Preserve existing session-memory JSONL tests.
- Add a changelog entry if this is user-facing; credit `Thanks @matskevich` and do not credit Codex/OpenClaw/steipete.

Do not change provider selection, QMD behavior, session transcript indexing semantics, dependency pins, or broad memory architecture. Validate with the targeted tests first, then use the normal changed gate/Testbox path before handoff if the PR changes production code.
