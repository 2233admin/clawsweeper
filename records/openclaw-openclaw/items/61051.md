## Summary

Keep this PR open. Current main has related approval-forwarding and Telegram native-approval infrastructure, but it still does not implement the PR’s central target-owned routing behavior: the shared gateway path broadcasts origin/operator approval events before forwarding, turn-source routing is not approval-kind or destination aware, and Telegram target recipients are still treated as exec approval authority in current tests.

## Best Possible Solution

Keep this PR open for maintainer review, but ask for a rebase and a narrower approval-routing patch. The best path is to move generic route selection into the shared gateway approval handler, add exec and plugin regression tests for targets-only forwarding and same-origin suppression, keep Telegram delivery targets separate from approval authority, and split or drop the already-shipped guarded-fetch changes.
