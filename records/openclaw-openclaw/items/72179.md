## Summary

Keep this PR open. Current main now has an overlapping typed `cron_changed` plugin hook for gateway cron lifecycle events, with gateway emission, docs, and focused tests, but it does not definitely implement this PR's intended public API: the PR adds a distinct `cron_lifecycle` hook and explicitly includes `agentId`, while current main exposes `cron_changed` and omits `agentId` from the public cron job snapshot.

## Best Possible Solution

Keep this PR open for maintainer review and rework it against current main. The clean path is to decide whether `cron_changed` is the accepted public hook, then either close this PR as superseded by that API or land a narrower follow-up that adds accepted missing fields such as `agentId` to the existing hook with docs and tests, avoiding a duplicate cron lifecycle hook unless maintainers deliberately want a new public name.
