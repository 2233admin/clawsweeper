## Summary

Keep this PR open. It targets the right OpenAI-compatible stream boundary for the Mistral `[object Object]` corruption still present on current main, but the patch is not merge-ready because it misses a nested Mistral thinking-block shape already covered in the related parser work and it lacks the required changelog entry.

## What This Changes

The branch adds a structured `delta.content` normalizer to the OpenAI completions stream, routes normalized text/thinking parts through existing append paths, extends reasoning-field fallback normalization, and adds unit tests for the normalizer.

## Best Possible Solution

Land one parser-side OpenAI-completions fix that normalizes structured `delta.content` arrays, including nested Mistral thinking sub-block arrays, preserves source order, falls through for unsupported/empty content so same-chunk reasoning and tool calls still process, and includes focused regression tests plus the required changelog credit.

## Reproduction Assessment

Yes. The linked issue gives concrete Mistral-with-thinking steps, and current main provides a high-confidence mocked reproduction path: feed the completions parser a `delta.content` array with thinking/text objects and observe that the existing string append path can coerce it into `[object Object]`.

## Solution Assessment

Mostly yes, but not as submitted. The stream parser is the right layer for this fix; the safer version should also flatten nested `thinking` arrays and add release-note coverage before merge.
