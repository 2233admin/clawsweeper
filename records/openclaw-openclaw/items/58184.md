## Summary

Keep PR #58184 open. Current main still serves the permissive `connect-src 'self' ws: wss:` header, so the underlying hardening request is not implemented. The PR discussion also surfaced a real product constraint: Control UI still supports configured cross-origin `gatewayUrl` targets, including documented `ws://` remote dev usage, so the final fix needs a more careful CSP design than simply closing this as obsolete.

## Best Possible Solution

Keep the PR open and ask for a revised gateway CSP design. The best path is to remove blanket `ws:`/`wss:` for the default served Control UI while preserving documented remote gateway usage through explicit, narrowly derived connect sources or an intentional product/docs change. Add regression coverage for `connect-src`, arbitrary external WebSocket blocking, and the supported `gatewayUrl` cases.
