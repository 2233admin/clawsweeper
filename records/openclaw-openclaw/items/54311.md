## Summary

Keep this issue open. Current main still ships `gh-issues` with the reported `user-invocable: true` frontmatter, the skill remains a single 34,800-byte `SKILL.md`, and the skill-creator validator still rejects frontmatter keys outside its narrow allowlist. The description has been improved since the original report, but the validation mismatch and size/refactor requests are still unresolved.

## Best Possible Solution

Keep this issue open and fix it with a focused bundled-skill cleanup. The immediate path is either to remove the redundant `user-invocable: true` from `skills/gh-issues/SKILL.md` or intentionally update the skill-creator validator/guidance to accept the runtime-documented optional keys. The follow-up should split the large orchestration details into `references/` files while keeping only concise triggers, key constraints, and navigation in the top-level `SKILL.md`.
