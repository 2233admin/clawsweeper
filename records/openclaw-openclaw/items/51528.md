## Summary

Keep this PR open. Current main still builds the chat model picker in catalog/default/current order and has no browser-local MRU model storage or custom-provider prioritization, so the PR’s intended behavior is not implemented on main. The work is stale against the current UI split, but still maps to a real missing Control UI behavior.

## Best Possible Solution

Keep this PR open as useful but stale work. The best path is to port the MRU/custom-provider ordering into the current chat-model-select-state.ts and chat/session-controls.ts split, using getSafeLocalStorage and best-effort writes, preserving the pinned Default option and active missing overrides, storing canonical resolved refs only after successful sessions.patch, skipping MRU updates when clearing to Default, and adding focused tests beside chat-model-select-state.test.ts and the chat view/session-control tests.
