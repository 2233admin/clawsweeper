The PR updates Crestodian’s non-TTY guard to call `runtime.exit(1)` and adds regression tests for stdin, stdout, and explicit non-interactive failures.

## Best Possible Solution

Land one canonical fix that makes the explicit `openclaw crestodian` non-TTY path exit non-zero at the Crestodian guard, preserves `--message`, `--json`, and interactive TUI behavior, and keeps focused regression coverage for stdin/stdout non-TTY cases.

## Solution Assessment
