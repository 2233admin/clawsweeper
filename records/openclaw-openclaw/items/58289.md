## Summary

Keep open. Current main still has the reported gap: an empty provider plan returns `skip`, `ensureOpenClawModelsJson` then returns without creating `models.json`, and `agents.create` creates the agent workspace/config without generating that file. An open closing PR already targets this issue, so the issue should stay open pending review/merge of that PR.

## What This Changes

Prevent newly created non-main agents from reaching runtime without an agent-local `models.json` when provider discovery resolves no providers.

## Best Possible Solution

Review and land PR #58373 or an equivalent narrow fix that handles only the non-main missing-file plus empty-provider-plan path, preserves existing per-agent registries, and avoids broad global model sync.

## Reproduction Assessment

Yes. A focused temp-directory check can reproduce it by creating a non-main agent dir without `models.json`, making provider resolution empty, calling `ensureOpenClawModelsJson`, and observing the skip path leaves the file missing before runtime model resolution.

## Solution Assessment

Yes, the narrow one-time bootstrap or an early actionable failure is the right direction. It addresses the reported runtime failure without reopening the broader global model-sync request from #22747.
