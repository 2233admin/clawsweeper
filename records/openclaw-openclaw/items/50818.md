The PR adds a `resolveEventSessionKey` helper, extends `scopedHeartbeatWakeOptions` with optional `mainKey` cron-run remapping, threads that through async exec, ACP, CLI watchdog, and gateway node-event enqueue/wake paths, and adds session-key and exec-completion classification tests.

## Best Possible Solution

Land a rebased slim fix that keeps enqueue-side and wake-side keys aligned for ephemeral cron-run async completion events using one main-key-aware helper, while leaving `target: "none"` delivery semantics and global-scope cron behavior to separately reviewed follow-up work.
