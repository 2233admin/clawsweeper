## Summary

Keep this PR open. Current main still sends Telegram polls but does not persist poll metadata or route `poll_answer` updates into sessions, so the PR is relevant; however, the final patch still has a blocking correctness issue because a registry write failure after a successful Telegram send would make the action fail as if the poll was not sent.

## What This Changes

The PR adds Telegram public-poll answer routing by persisting poll metadata on public poll sends, including `poll_answer` in allowed updates, adding a `poll_answer` bot handler, and covering the registry/action/bot paths with tests.

## Best Possible Solution

Keep the PR as the active implementation candidate, but make registry persistence best-effort after a successful Telegram send, preserve the final diff's restored CODEOWNERS state, and validate the Telegram action/registry/bot tests before maintainer review.
