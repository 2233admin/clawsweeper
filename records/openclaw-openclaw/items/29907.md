## Summary

Keep this issue open. Current main still only supports cron payloads for `systemEvent` and `agentTurn`; there is no first-class `exec` or `lobster` cron payload, no `condition`/`onCondition` contract, and no CLI/UI/docs path for deterministic cron runs that avoid the LLM until a condition matches. The related direct-exec request in #18160 is narrower and does not cover the Lobster pipeline plus conditional handoff design here.

## Best Possible Solution

Keep this open for maintainer design. The best path is either an additive native cron executor contract covering `exec` and Lobster payloads, condition evaluation, silent false-condition runs, bounded output capture, failure visibility, CLI/Control UI/docs, and protocol/runtime tests, or an explicit ClawHub/plugin plan that uses existing cron lifecycle hooks, reply short-circuiting, context injection, Lobster, and process-runtime helpers. Coordinate with #18160 as the narrower direct-exec subset rather than treating it as a full duplicate.
