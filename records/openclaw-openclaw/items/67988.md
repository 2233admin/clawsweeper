## Summary

Keep open. Current main still builds the Agents/Control UI primary-model dropdown from configured aliases or raw ids before catalog display labels can apply, so the reported `opus` and `sonnet` labels remain possible even though Anthropic runtime support exists.

## What This Changes

The issue asks for the Control UI model dropdown to show full display names for configured Claude Opus 4.7 and Claude Sonnet 4.6 models instead of alias/key labels.

## Best Possible Solution

Unify the Agents primary-model option builder with the catalog display-name lookup used by the chat picker, and add focused regression coverage for configured Anthropic model refs with default aliases.
