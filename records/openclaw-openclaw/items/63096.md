## Summary

Keep this PR open because current main still lacks the completion-wake prompt synthesis the PR is trying to add, but the patch is not merge-ready against current main. The branch still diverges from current prompt assembly for direct/dm and channel sessions, and its tests assert stale inbound metadata and group-context text.

## What This Changes

This PR adds gateway `agent` prompt synthesis for inter-session `task_completion` wakes from persisted session metadata, preserves explicit `extraSystemPrompt`, and adds gateway seam tests.

## Best Possible Solution

Refresh the branch onto current main and make completion-wake synthesis reuse the same current session-derived prompt shape as ordinary turns, preferably through a shared helper so it cannot drift again: inbound metadata v2, direct/dm context, group/channel context, explicit `extraSystemPrompt` pass-through, and no synthesized metadata for metadata-free sessions. Also decide whether the PR should stop closing #63030 unless it expands to cover the heartbeat/chat portions of that broader report.
