## Summary

Keep this PR open. Current main still has the completion-wake prompt-context gap, and the PR targets the right gateway `agent` seam, but it is not merge-ready against current main: the proposed helper does not yet synthesize the direct-chat prompt section now used by normal chat turns, and the added tests still assert the older inbound metadata and group-context wording.

## What This Changes

This PR adds gateway `agent` prompt synthesis for inter-session `task_completion` wakes from persisted session delivery metadata, preserves explicit `extraSystemPrompt`, and adds gateway seam tests for synthesis and pass-through behavior.

## Best Possible Solution

Refresh this PR onto current main and make the gateway completion-wake path reuse the same current session-derived prompt sections as ordinary chat turns: trusted inbound metadata v2, direct-chat context for direct/dm sessions, group-chat context only for group sessions, explicit `extraSystemPrompt` pass-through, and no synthesized metadata for metadata-free sessions. Keep the broader heartbeat/cache-digest work in #63030 open unless this PR expands to cover those paths too.
