## Summary

Current main still has `openclaw logs` using the old UTC-by-default behavior unless `--local-time` is passed or `TZ` is set to a valid timezone. This PR’s intended change, default-local timestamps plus a `--utc` opt-out, has not shipped, so it remains useful implementation work for the linked open bug #46748.

## Best Possible Solution

Keep this PR open as an implementation candidate for #46748. The best path is maintainer review of the focused CLI behavior change, with added/updated tests for default-local timestamps and the UTC opt-out, docs updates for `openclaw logs`, and an explicit compatibility decision for the existing `--local-time` flag.
