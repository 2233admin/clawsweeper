## Summary

Keep this PR open. Current main still does not implement the PR’s intended default-local `openclaw logs` behavior: the CLI registers `--local-time` instead of `--utc`, computes local rendering as an opt-in via `--local-time` or valid `TZ`, and tests/docs still describe the old contract. The PR remains a focused implementation candidate for open bug #46748, but it should not land as-is without tests/docs and an explicit compatibility decision for the existing `--local-time` flag.

## Best Possible Solution

Keep this PR open as an implementation candidate for #46748. The best path is maintainer review of the CLI contract, add or update tests for default-local output and `--utc`, update `docs/cli/logs.md` and `docs/logging.md`, and decide whether `--local-time` remains as a compatibility alias before merging or closing competing attempts.
