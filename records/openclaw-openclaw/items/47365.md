## Summary

Keep this PR open, but do not merge it as-is. Current main still does not implement the requested default-local `openclaw logs` behavior, and the PR introduces a blocking CLI compatibility regression by removing the released `--local-time` flag instead of preserving it while adding a UTC opt-out.

## What This Changes

This PR changes `openclaw logs` timestamp display to default to local time by replacing the existing `--local-time` option with a new `--utc` option in `src/cli/logs-cli.ts`.

## Best Possible Solution

Revise this PR or a replacement implementation so human-readable `openclaw logs` output defaults to local time, `--utc` opts back into UTC, the released `--local-time` flag remains accepted as a compatibility alias or gets a maintainer-approved deprecation path, and the CLI tests plus `docs/cli/logs.md` and `docs/logging.md` describe the final contract before closing #46748.
