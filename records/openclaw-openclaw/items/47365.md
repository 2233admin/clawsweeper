## Summary

Keep open. Current main has not implemented default-local `openclaw logs` timestamps, and this PR is a useful implementation direction, but the branch removes a shipped CLI flag that current docs, tests, and the latest release still expose.

## What This Changes

The PR changes `src/cli/logs-cli.ts` so human-readable `openclaw logs` output defaults to local time, removes `--local-time`, and adds `--utc` for UTC display.

## Best Possible Solution

Land one coordinated implementation that defaults human-readable `openclaw logs` output to local time, keeps `--local-time` accepted as a compatibility alias/no-op or follows an explicit deprecation path, exposes one clear UTC opt-out, updates tests and docs, then resolves the overlapping PR and closes #46748.

## Reproduction Assessment

Yes. Current main shows the existing behavior in source and tests: without `--local-time` or a valid `TZ`, `localTime` stays false, while this branch removes the currently accepted `--local-time` parser option.

## Solution Assessment

No. Defaulting to local time is a reasonable fix direction, but removing the released `--local-time` flag is not the narrowest maintainable solution; preserving it while adding a UTC opt-out is safer.
