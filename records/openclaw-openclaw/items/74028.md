## Summary

Keep this PR open for normal maintainer review. Current main still sends local fully qualified model probes through agent-aware model selection, and this PR is the focused implementation candidate for the linked open CLI bug.

## What This Changes

The PR changes the infer CLI model-run path so local `--model provider/model` overrides use direct simple-completion preparation while unqualified refs keep the existing agent-aware path, and it adds CLI unit tests for local override preparation plus gateway forwarding.

## Best Possible Solution

Land or refine the focused CLI fix after maintainer confirmation of the local probe boundary: qualified local `--model <provider/model>` refs should bypass agent alias/default selection, unqualified refs should keep the existing agent path, profile suffix handling should remain intact, and the linked bug should close only after this or an equivalent fix merges.

## Reproduction Assessment

Yes. Current main shows the local `runModelRun` implementation always passing `params.model` into `prepareSimpleCompletionModelForAgent`, and #73921 provides concrete `github-copilot/...` commands with observed provider metadata mismatch.

## Solution Assessment

Yes, pending maintainer confirmation of the slash-form boundary. The proposed approach is narrow because it changes only the local fully qualified override path while preserving unqualified agent-aware selection and existing gateway forwarding.
