## Summary

Keep this issue open. Current main still has a real core onboarding gap: non-interactive onboarding only accepts token/password gateway auth, while trusted-proxy is already supported by runtime config, startup validation, interactive onboarding, docs, and tests. The same-author draft PR #49107 is an implementation candidate, and #73639 looks like a newer duplicate rather than a reason to close this older tracker.

## Best Possible Solution

Keep this issue open and implement the missing core onboarding surface, preferably by reviewing or replacing #49107. The fix should accept `--gateway-auth trusted-proxy`, add explicit non-interactive flags for trusted proxy IPs, user header, required headers, allowUsers, allowLoopback, and Control UI allowed origins, update `docs/cli/onboard.md`, and add focused tests for valid config writes plus missing or unsafe security fields. Treat #73639 as the newer duplicate unless maintainers choose to make it canonical.
