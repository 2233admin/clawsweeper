## Summary

Keep this issue open. Current main still rejects `--gateway-auth trusted-proxy` in non-interactive onboarding even though runtime config and the interactive gateway flow support trusted-proxy auth. The same author also has open draft PR #49107 targeting this gap, so the right maintainer action is to review, land, or replace that implementation rather than auto-close this issue.

## Best Possible Solution

Keep this issue open and implement the feature in core onboarding. Review, land, or replace #49107 with a focused change that accepts `--gateway-auth trusted-proxy`, adds explicit non-interactive flags for `gateway.trustedProxies` and `gateway.auth.trustedProxy.*`, handles Control UI allowed origins where needed, updates CLI docs, and adds tests for valid config writes plus invalid or missing trusted-proxy inputs.
