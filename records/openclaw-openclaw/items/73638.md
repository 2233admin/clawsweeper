## Summary

Current main still lacks non-interactive trusted-proxy onboarding. The runtime config, gateway startup path, docs, and interactive gateway wizard all support `gateway.auth.mode: "trusted-proxy"`, but `openclaw onboard --non-interactive` still exposes and accepts only token/password gateway auth. The same-author draft PR #49107 is an implementation candidate, so this issue should stay open until that PR is merged, replaced, or closed.

## Best Possible Solution

Keep this issue open and review, land, or replace #49107. The core onboarding fix should add non-interactive trusted-proxy gateway auth flags, write `gateway.trustedProxies`, `gateway.auth.trustedProxy.*`, and Control UI allowed origins safely, update `docs/cli/onboard.md`, add focused tests for valid writes and missing/unsafe security fields, and then close the newer duplicate-looking #73639 if appropriate.
