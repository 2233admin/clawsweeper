## Summary

Keep open. Current main still enables the packaged CLI Node compile cache without an OpenClaw install-root/version-scoped cache directory, and this open PR is the active implementation candidate for that cache-hygiene change. The latest PR patch addresses the prior review direction by deriving a scoped cache directory in both bootstrap paths, passing a string cacheDir, adding tests, and documenting NODE_COMPILE_CACHE as a base directory; it still needs normal maintainer review and complete current-head validation rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open and review/land it or an equivalent patch after current-head validation. The desired fix is to derive one OpenClaw-scoped compile-cache directory per packaged install root and OpenClaw version in both bootstrap paths, keep source-checkout cache disable/respawn behavior intact, pass Node-compatible string cacheDir values, and document NODE_COMPILE_CACHE as the base for those scoped entries while leaving #71348 as a reference unless the root cause is proven.
