## Summary

Keep open. Current main fixed the linked @mention command parsing problem, but it does not implement this PR's stated no-mention Feishu slash-command bypass: the Feishu handler still returns at the requireMention gate before command probing. The PR also needs maintainer review because its proposed bypass is broader than the shared Discord/Telegram authorization-gated mention-bypass pattern.

## What This Changes

The PR modifies Feishu group handling so recognized slash commands are detected before requireMention gating and can activate the routed single-agent or broadcast-agent path without an @mention.

## Best Possible Solution

Decide explicitly whether Feishu text slash commands should bypass requireMention. If yes, implement it in the Feishu handler using the shared command-authorization semantics and cover authorized, unauthorized, single-agent, and broadcast group cases; if no, close this PR manually as outside the intended Feishu group activation model because the linked @mention parsing bug is already shipped.
