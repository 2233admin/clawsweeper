## Summary

Current main still leaves the reported heartbeat timestamp footgun reachable: the default AGENTS template documents raw Unix epoch values in memory/heartbeat-state.json with no instruction to obtain them through a tool, while the current-time helper does not inject epoch seconds. Safer HEARTBEAT.md task mode exists and stores last-run state internally, but the template still points users at manual JSON timestamp tracking.

## Best Possible Solution

Keep this open and update the core workspace guidance. The narrowest good fix is to change docs/reference/templates/AGENTS.md so interval heartbeat checks prefer HEARTBEAT.md tasks blocks and session-backed heartbeatTaskState; if memory/heartbeat-state.json remains documented, the template should explicitly say to obtain Unix timestamps through a tool such as date +%s and never calculate epochs manually. A small follow-up can consider adding epoch seconds to current-time injection or defensively ignoring future manual heartbeat timestamps where such state is read.
