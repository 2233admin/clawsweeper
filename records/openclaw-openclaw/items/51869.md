## Summary

Current main implements the requested custom-provider vision input support. Interactive onboarding now infers common vision-capable model IDs, prompts only for unknown models, passes the result into config generation, and non-interactive onboarding has explicit image/text override flags. Regression tests and docs/changelog cover the behavior. The implementation landed in 59162379627bc0a2c26af78df4823926dfa4f9fc and is contained by the checked-out main SHA, but it is not in the latest published release yet.

## Best Possible Solution

Close as implemented on current main. Keep the shipped approach: shared inference plus an explicit prompt/override path, with docs explaining how users can mark unknown custom vision models as image-capable. Let the next OpenClaw release carry the already-landed fix.
