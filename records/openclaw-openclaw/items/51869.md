Current main implements the requested custom-provider vision handling. Interactive custom onboarding now infers known vision model IDs, asks for image support only when the model ID is unknown, passes that choice into custom provider config generation, and documents non-interactive `--custom-image-input` / `--custom-text-input` overrides. The fix is on current main at commit 59162379627bc0a2c26af78df4823926dfa4f9fc but is not in v2026.4.26.

## Best Possible Solution

Close this issue as implemented on current main. Keep the main-branch implementation and let the next release ship it; users on v2026.4.26 or older still need to re-onboard with the fixed version, pass `--custom-image-input`, or manually update existing custom model `input` metadata until that release is available.
