## Summary

Keep open. Current main adds an implicit-loopback file-log fallback for closed/timeout Gateway RPC failures, but the `logs --follow` loop still exits on non-fallback errors and the tests explicitly preserve fatal behavior for explicit Gateway URLs, so the requested reconnect/backoff behavior is not fully implemented.

## What This Changes

Request `openclaw logs --follow` to keep tailing across transient Gateway disconnects by warning, retrying with backoff, and exiting only for non-recoverable errors.

## Best Possible Solution

Add bounded reconnect/backoff behavior inside the `logs --follow` polling loop for transient Gateway transport failures, while preserving fatal handling for auth/configuration failures and the existing implicit-loopback local file fallback. Cover default loopback and explicit URL cases in focused CLI tests and update the logs docs/changelog if behavior changes.
