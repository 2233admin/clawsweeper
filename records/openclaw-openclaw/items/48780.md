## Summary

Keep this open. Current main still has no implemented sanitizer for the reported `</arg_value>>` suffix, the active exec/read paths forward string arguments unchanged, and an open implementation PR already references this issue with closing syntax, so cleanup closure would be premature.

## What This Changes

The issue reports Windows/Qwen-DashScope tool calls appending `</arg_value>>` to exec command strings and read paths, corrupting command execution and file reads.

## Best Possible Solution

Keep this issue open until PR #48792 or a replacement lands a narrow fix at the active tool-argument boundary, with regression coverage for Qwen/DashScope suffixes and negative coverage that preserves legitimate XML-like content.
