## Summary

Keep this issue open. Current main still has no `arg_value` / XML-suffix sanitizer for tool arguments, and the active `exec` and `read` paths pass string arguments through without cleanup. There is also an open implementation PR, #48792, that explicitly targets this issue, so the issue should stay open until that PR is reviewed, reworked for current main, and merged or closed.

## Best Possible Solution

Keep this issue open and treat #48792 as the current implementation candidate. The best fix is a current-main rework that adds a narrowly scoped sanitizer at the active tool-argument boundary, covers read path-like args and exec string params intentionally, and adds regression tests for Qwen/DashScope `</arg_value>` suffixes plus negative tests proving legitimate file content and edit replacement text are preserved.
