## Summary

Keep this PR open. Current main still reads `logging.file` directly from the active config file, passes the configured path through `resolveActiveLogFile()`, and returns non-rolling paths verbatim before `fs.mkdirSync(path.dirname(activeFile))`. The PR remains a focused implementation candidate for #73587, with one small test-helper cleanup still worth addressing.

## Best Possible Solution

Keep this PR open as the implementation candidate for #73587. The best path is to update the test helper to reuse `resolveEffectiveHomeDir()`, keep the narrow `expandHomePrefix()` call inside `resolveActiveLogFile()`, then land with focused logging regression coverage and the normal changed-gate validation.
