## Summary

Keep this PR open. Current main still has the reported `logging.file` tilde behavior: `resolveActiveLogFile()` returns non-rolling paths verbatim, then `buildLogger()` calls `fs.mkdirSync(path.dirname(activeFile), { recursive: true })`, so a configured `~/.openclaw/logs/gateway.log` can still reach `mkdirSync` as a literal `~` path. The PR is a focused implementation candidate for the still-open linked regression, with only a small test-helper nit noted by review.

## Best Possible Solution

Keep the PR open as the implementation candidate for the linked open regression. The best path is to update the test helper to reuse `resolveEffectiveHomeDir()`, then land the focused logger change with the regression tests and changelog entry after the normal targeted logging checks pass.
