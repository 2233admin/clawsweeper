## Summary

Keep this PR open. Current main still reads `logging.file` directly from the active config file, carries the configured path through `resolveSettings()`, returns non-rolling paths verbatim in `resolveActiveLogFile()`, and calls `fs.mkdirSync(path.dirname(activeFile))` on that result. The latest release tag has the same behavior, so the PR is still a focused implementation candidate for #73587. The PR diff is small and stays inside logging source, logging tests, and changelog; the remaining review item is the test helper nit already raised in the PR discussion.

Keep this PR open as the implementation candidate for #73587. The best path is to update the PR test helper to reuse `resolveEffectiveHomeDir()`, keep the narrow `expandHomePrefix()` call inside `resolveActiveLogFile()`, and land with focused logging regression coverage plus the normal changed-gate validation. Keep #73587 open until this PR or an equivalent fix merges.
