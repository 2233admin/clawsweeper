## Summary

Keep open. Current main still rejects the reported MSYS2/MINGW `install.sh` path, the installer still persists and warns about PATH using bash/zsh-only guidance, and the Windows docs still route users to WSL2 or the PowerShell installer. The related Microsoft tracker is open coordination, not an implementation.

## What This Changes

Support running the Unix installer from Windows 11 MSYS2/MINGW and add fish shell-specific PATH persistence or guidance.

## Best Possible Solution

Maintain this as an open platform-support decision: either add tested MSYS2/MINGW support to `scripts/install.sh` with Windows/MSYS-safe install behavior, fish-aware PATH handling, docs, and smoke coverage, or explicitly document that Windows Unix-like environments should use WSL2 or the PowerShell installer.
