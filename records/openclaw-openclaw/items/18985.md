Keep open. Current main still rejects the reported Windows 11 MSYS2/MINGW `install.sh` path and still gives bash/zsh-only PATH persistence guidance; the related Microsoft tracker is open tracking platform work rather than a merged implementation.

## What This Changes

The issue requests that `install.sh` run successfully under Windows 11 MSYS2/MINGW and that installer PATH setup/warnings include fish shell guidance.

## Best Possible Solution

Keep this open until maintainers either explicitly support MSYS2/MINGW in `scripts/install.sh` with tested OS detection, Windows/MSYS-safe install decisions, fish-aware PATH persistence or guidance, docs, and smoke coverage; or document that Windows Unix-like environments should use WSL2 or `install.ps1` instead.
