## Summary

Keep this issue open. Current main has partial password/secret reveal support in the web Control UI, but the requested cross-platform behavior is still missing from the web Skills API-key editor and several macOS, iOS, and Android credential entry surfaces. The related PR #7124 is closed and unmerged in the provided context, so there is no shipped implementation to close against.

## Best Possible Solution

Keep the issue open and implement a reusable sensitive-input reveal pattern across the relevant surfaces: extend the existing Control UI reveal pattern to Skills/API-key fields, add SwiftUI reveal wrappers for macOS and iOS SecureField credential entries, add Android Compose password fields using password visual transformation plus trailing visibility icons, and cover the touched web/native surfaces with focused UI or view tests.
