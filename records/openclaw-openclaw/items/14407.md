## Summary

Keep this issue open. Current main has partial password/secret reveal support in the Control UI, but the requested cross-platform toggle behavior is still incomplete: the Skills API-key editor lacks a reveal control, macOS/iOS still have direct SecureField credential surfaces without adjacent toggles, and Android connection/onboarding password fields are plain text fields rather than masked fields with reveal buttons. The related PR #7124 is closed and unmerged in the provided context, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this open and implement a reusable sensitive-input reveal pattern across the relevant surfaces: extend the existing Control UI config/Overview/login reveal pattern to the Skills API-key editor, add a small SwiftUI revealable secure-field wrapper for macOS and iOS credential entries, convert Android password inputs to masked Compose fields with trailing visibility icons, and cover the touched web/native surfaces with focused tests or render smoke coverage. Do not persist reveal preference across sessions unless maintainers explicitly approve that product/security behavior.
