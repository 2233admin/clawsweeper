## Summary

Keep open. Current `main` still has direct `os.homedir()` calls in the sandbox resolver and sandbox error shortening path that this PR changes, and the latest release tag has the same behavior. The PR is still a narrow implementation candidate, but it should likely be revised to reuse the central OS-home helper and add focused sandbox regression coverage. One important dependency check: Node documents and a local probe show `os.homedir()` already honors `$HOME` on POSIX, so the reported `$HOME`-set divergence needs a precise reproducer; the remaining repo-consistent delta is aligning sandbox code with the existing `HOME -> USERPROFILE -> os.homedir()` helper contract.

## Best Possible Solution

Keep this PR open as the active implementation candidate, but ask for a small revision before merge: reuse the central `resolveOsHomeDir()`/`expandHomePrefix()` contract in `src/agents/sandbox-paths.ts`, cover both `expandPath` consumers and `shortPath`, add sandbox resolver tests for HOME/USERPROFILE/os.homedir divergence plus `OPENCLAW_HOME` separation, and keep the broader `OPENCLAW_HOME` policy issue (#54014) separate from this OS-home tilde expansion fix.
