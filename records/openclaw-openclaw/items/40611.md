## Summary

Current main still matches the reported heartbeat contention path: `requests-in-flight` skips are requeued through the shared 1 second wake retry, and there is still no heartbeat-specific busy-session retry/backoff config in schema, types, docs, or tests. The focused fix PR #40629 is closed unmerged in the provided GitHub context; #13744 and #14051 remain broader follow-ups, not a shipped fix for this issue.

## Best Possible Solution

Keep this issue open and fix the periodic heartbeat busy-session retry path directly. A narrow implementation should add a heartbeat-specific `requests-in-flight` retry delay or backoff separate from generic wake error retry, choose a conservative default, expose it under `agents.defaults.heartbeat` and per-agent heartbeat config, and update schema/types, config-honor inventory, generated config metadata/help, public heartbeat docs, and scheduler/wake regression tests. Leave #13744 for broader session-lock behavior and #14051 for activity-based heartbeat semantics.
