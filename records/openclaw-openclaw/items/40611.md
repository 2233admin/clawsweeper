## Summary

Keep this issue open. Current main still routes a `requests-in-flight` heartbeat skip through the shared 1 second wake retry path, and there is still no heartbeat-specific busy-session retry/backoff setting in the heartbeat schema, generated config metadata, tests, or public heartbeat docs. The focused fixing PR #40629 is closed unmerged, while #13744 and #14051 remain broader designs rather than a shipped fix for this regression.

## Best Possible Solution

Keep this issue open and fix the periodic heartbeat busy-session retry path. The most direct maintainer path is to revive the useful parts of #40629: add a heartbeat-specific `requests-in-flight` retry delay separate from the shared wake error retry, decide the default and configurability under `agents.defaults.heartbeat` and per-agent heartbeat config, then update the strict schema, typed config, generated metadata/help, public heartbeat docs, and scheduler/wake regression tests. Leave #13744 for broader session-lock scope/configuration and #14051 for activity-based heartbeat semantics.
