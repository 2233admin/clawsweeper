## Summary

Keep open. Current main still lacks a supported cross-channel config and runtime path for `streaming.preview.toolProgress: false` on channels such as LINE and WhatsApp, and the remaining work is privacy-sensitive enough to need maintainer/API judgment rather than cleanup closure.

## What This Changes

The issue requests preserving and honoring `streaming.preview.toolProgress` for channel configs that do not already declare `streaming.mode`, especially on WhatsApp, LINE, Slack, and Sendblue-style surfaces.

## Best Possible Solution

Define a documented, schema-backed delivery contract for suppressing tool/progress messages independently of preview streaming, expose it in the affected channel config surfaces and generated metadata/docs, and wire each runtime to the shared suppression option or an equivalent explicit delivery policy with regression coverage.

## Reproduction Assessment

Yes. The config-strip part has a high-confidence source reproduction: WhatsApp and LINE strict schemas do not accept the nested `streaming.preview.toolProgress` key, and LINE’s runtime does not pass the shared suppression flag. I did not run live reload because this review was read-only.

## Solution Assessment

No current fix is present on main. Preserving the config key alone would be incomplete; the maintainable fix also has to translate the effective setting into channel runtime delivery policy for every affected channel.
