## Summary

Keep open. Current main still exposes `streaming.preview.toolProgress` only through preview-streaming channel schemas such as Telegram, Discord, Slack, and Matrix; WhatsApp and LINE still lack a supported nested `streaming.preview.toolProgress` config path and do not pass `suppressDefaultToolProgressMessages` into generic reply dispatch. The related open Slack PR is narrower and does not cover the cross-channel config contract gap reported here.

## Best Possible Solution

Keep this issue open. The best fix is to add a supported, documented suppression path for non-preview-streaming channels: either extend the shared channel config contract so WhatsApp, LINE, Sendblue, and similar channels can preserve and honor `streaming.preview.toolProgress: false`, or add a documented global/default setting with clear per-channel precedence. The implementation should update strict schemas, generated channel metadata, docs, and regression coverage for config validation/round-trip plus outbound progress delivery on at least WhatsApp and LINE.
