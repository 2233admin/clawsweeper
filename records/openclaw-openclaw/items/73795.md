## Summary

Keep this issue open. Current main still lacks a supported way to preserve or apply `streaming.preview.toolProgress: false` for WhatsApp and LINE, while the shared setting exists for preview-streaming channels. The related open Slack PR is narrower and does not cover the cross-channel config gap described here.

## Best Possible Solution

Keep this issue open. The best fix is to add a supported suppression path for non-preview-streaming channels, either by extending the shared channel config contract so WhatsApp/LINE can preserve and honor `streaming.preview.toolProgress: false`, or by adding a documented global `agents.defaults.streaming.preview.toolProgress` with clear per-channel precedence. Update strict schemas, generated channel metadata/docs, and add regression coverage for config round-tripping plus outbound tool/progress delivery behavior on WhatsApp and LINE.
