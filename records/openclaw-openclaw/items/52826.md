## Summary

Keep this issue open. Current main still has generic agent, plan, item, lifecycle, tool, session.message, and session.tool events, but it does not expose the requested durable session-level process.* / step.* Gateway contract with stable step identity and related message/tool/artifact ids.

## Best Possible Solution

Keep this issue open and design an additive Gateway protocol surface for durable session process events, likely a typed session.process or process/step event family plus a replayable per-session/per-run ledger. It should derive from existing lifecycle, plan, item, tool, transcript, task, and artifact signals where possible, include the requested stable fields, update docs and schema, and add coverage for live streaming plus reconnect/reload recovery.
