## Summary

Current main still does not provide non-interactive onboarding for trusted-proxy gateway auth. The onboard CLI, option types, docs, and implementation remain token/password-only, while trusted-proxy is supported at the gateway/runtime and interactive configure layers. The same-author draft PR #49107 is still open as the implementation candidate, so this issue should stay open for maintainer review rather than be swept closed.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #49107 with a focused implementation that adds non-interactive onboard flags for `gateway.auth.mode="trusted-proxy"`, `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, `requiredHeaders`, `allowUsers`, an explicit `allowLoopback` decision, and any needed Control UI allowed-origin automation, with targeted tests and docs aligned to the trusted-proxy auth guide.
