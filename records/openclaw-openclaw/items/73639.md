## Summary

Current main still has the reported gap: non-interactive onboarding accepts only token/password gateway auth, while trusted-proxy is supported by core gateway config/runtime and the interactive gateway flow. The issue also has a same-author open draft implementation PR (#49107) and a duplicate-looking same-author issue (#73638), so it should stay open for maintainer triage rather than conservative cleanup closure.

## Best Possible Solution

Keep this issue open. Maintainers should review, land, or replace PR #49107 with a focused core onboarding change that accepts --gateway-auth trusted-proxy, adds explicit non-interactive flags for gateway.trustedProxies and gateway.auth.trustedProxy.*, handles Control UI allowed origins where needed, updates public onboarding docs, and adds targeted tests for valid config writes plus unsafe or missing security fields. After the implementation path is chosen, deliberately dedupe or close #73638 if it is the canonical duplicate.
