## Summary

Keep this PR open. Current main still has the docs gap the PR is trying to fill, but the proposed entries publish stale file-tool contracts, so the patch is useful direction but not mergeable as written.

## What This Changes

The PR adds dedicated `read`, `write`, and `edit` entries to `docs/tools/index.md` with descriptions, parameters, and usage notes for the built-in file operation tools.

## Best Possible Solution

Revise the docs-only patch so the tool inventory matches current main: canonical `path`, string `content`, `edits: [{ oldText, newText }]`, adaptive read paging/caps, and image reads as image content blocks with no guaranteed `MEDIA:` token.

## Reproduction Assessment

Not applicable as a docs-only PR. There is a high-confidence verification path by comparing the proposed text against current runtime validation, read execution, media extraction, and colocated tests on main.

## Solution Assessment

No, not as written. Adding dedicated inventory entries is the right direction, but the current wording would publish incorrect API guidance; correcting the docs is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Document adaptive read paging:** `docs/tools/index.md:190-202`
  - body: The added `read` docs state text output is always truncated at `2000 lines or 50KB`, but current read execution computes an adaptive budget and auto-pages when context allows. Please describe the adaptive cap and continuation behavior instead of a fixed 50KB limit.
  - confidence: 0.94
- **[P2] Remove unsupported file_path aliases:** `docs/tools/index.md:194`
  - body: The new entries document `path or file_path`, but runtime validation only accepts canonical `path` and tests reject `file_path`. Keeping this alias in the public docs will make copy-paste tool calls fail.
  - confidence: 0.93
- **[P2] Describe edit as an edits array:** `docs/tools/index.md:222-224`
  - body: The `edit` section documents top-level `oldText`/`newText` and snake_case aliases, but current validation requires `edits: [{ oldText, newText }]` and supports multiple replacements. Users following this shape will get a missing `edits` error.
  - confidence: 0.95
- **[P2] Document image reads as image blocks:** `docs/tools/index.md:202`
  - body: The `read` notes promise `MEDIA:<path>`, but current read results are image content blocks and may have no media path. Please avoid documenting a guaranteed `MEDIA:` token for image reads.
  - confidence: 0.91
