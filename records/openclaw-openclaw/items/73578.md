## Summary

Keep this issue open. Current main still does not establish a separate current-working-set prompt-token snapshot for the claude-cli status path, so the reported cumulative-token numerator cannot be closed as implemented.

## Best Possible Solution

Add a CLI/status contract that carries current working-set prompt tokens separately from provider/session usage, for example `agentMeta.promptTokens` or structured `context.used` for CLI runners. Render `Context:` from that snapshot, show cumulative session tokens only as a clearly labeled separate metric if desired, and add a claude-cli `/status`/`session_status` regression where cumulative usage exceeds the context window but the latest prompt snapshot does not.
