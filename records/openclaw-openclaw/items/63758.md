## Summary

Keep open. Current main still matches the reported cron producer gap: task storage and status rendering support live progress summaries, but scheduled, startup catch-up, and manual cron task runs still create running task rows without `progressSummary` and only write terminal summaries at finish.

## What This Changes

The issue requests live progress summaries for managed cron task runs so `/tasks` and `session_status` show useful in-flight cron activity instead of staying quiet until completion.

## Best Possible Solution

Add cron-owned progress reporting: seed a coarse `progressSummary` when scheduled, startup catch-up, and manual cron task rows are created; update it through the existing detached-task progress writer at meaningful execution phases; preserve cron's silent notification default; and add focused regression coverage proving active cron work exposes useful status before `terminalSummary` is written.

Fix the managed cron task progress-summary gap from https://github.com/openclaw/openclaw/issues/63758.
