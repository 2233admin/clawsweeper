## Summary

Keep open. Current main has useful mitigations, including app-owned TCC surfaces, a local-prefix installer, stable Homebrew Node resolution, and an optional service wrapper, but the supported macOS Gateway path still defaults to an external CLI and shared Node runtime rather than a bundled or dedicated OpenClaw-owned TCC identity.

## What This Changes

Request to ship or default to a dedicated/bundled macOS Node runtime so Full Disk Access and Accessibility grants are scoped to OpenClaw instead of a shared node executable.

## Best Possible Solution

Keep this as core macOS security and packaging architecture work. The desired end state is a supported macOS Gateway/FDA runtime with a stable OpenClaw-owned identity by default, plus migration, doctor/onboarding checks, docs, and LaunchAgent reinstall behavior coordinated with the related TCC issues.
