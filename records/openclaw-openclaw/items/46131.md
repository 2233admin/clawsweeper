## Summary

Keep this PR open as a still-valid bug-fix candidate, but do not land it as-is. Current main still does not apply Anthropic provider-policy API defaults inside the inline provider helper, while the PR branch is stale and includes an unrelated Telegram retry change that should be split or dropped.

## Best Possible Solution

Keep this PR open, but require a rebase/split before landing. The model-only fix should move into `src/agents/pi-embedded-runner/model.inline-provider.ts`, apply the current provider-policy normalization before resolving inline transport, and add regression coverage for Anthropic with no explicit provider API, explicit provider API precedence, and non-Anthropic providers. The Telegram retry change should be dropped from this PR or redesigned separately under the existing non-idempotent send safety constraints.
