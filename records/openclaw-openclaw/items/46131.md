## Summary

Keep this PR open as a useful bug-fix candidate, but do not land it as-is. Current main still builds inline provider models from raw `model.api ?? entry.api`, so Anthropic provider-policy defaults are not applied on that inline path. The PR also carries an unrelated Telegram retry commit with a non-idempotent send retry risk, so the right maintainer path is a rebase/split and a current-code regression test.

## Best Possible Solution

Keep the PR open, but require a rebase/split before landing: move the model-only fix into current `src/agents/pi-embedded-runner/model.inline-provider.ts`, apply the current provider-policy/defaulting path for inline provider configs, add regression coverage for Anthropic with no explicit provider API plus explicit-provider and non-Anthropic cases, and drop or separately redesign the Telegram retry commit under the existing non-idempotent send safety constraints.
