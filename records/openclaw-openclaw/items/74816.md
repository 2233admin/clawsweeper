## Summary

Keep open. Current main still treats `stopReason: "aborted"` assistant turns as eligible for answer and reasoning payloads, while this PR adds a narrow payload-builder guard and focused regression tests for the linked timeout leak. I found no blocking review finding; the remaining step is maintainer review and validation.

## What This Changes

The PR updates embedded run payload assembly to surface aborted assistant turns as formatted errors, suppress aborted answer/reasoning text, and add regression tests for timeout/abort leakage.

## Best Possible Solution

Land this PR or an equivalent abort-aware guard in `buildEmbeddedRunPayloads` after green targeted tests and required checks, preserving current media/final-answer selection while preventing aborted assistant turns from delivering partial answer or reasoning artifacts.

## Reproduction Assessment

Yes. A focused static reproduction is to call `buildEmbeddedRunPayloads` with non-empty `assistantTexts`, `lastAssistant.stopReason: "aborted"`, and a timeout-like `errorMessage`; current main can emit the partial text instead of only a clean error payload.

## Solution Assessment

Yes. The PR is a narrow maintainable fix because it changes the payload assembly decision point, reuses the existing assistant error formatter, and leaves the current media/raw-answer preference logic intact for non-aborted turns.
