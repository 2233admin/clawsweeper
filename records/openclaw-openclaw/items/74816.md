## Summary

Keep open. Current main still lets aborted assistant turns bypass the embedded payload error-surface path, so aborted answer or reasoning text can become user-facing output before the runner's no-payload timeout fallback can run. The PR is a narrow implementation of the missing abort guard, and I did not find a concrete correctness or security regression in the diff.

## What This Changes

The PR updates embedded run payload assembly to treat aborted assistant turns as error surfaces, suppress aborted answer and reasoning text, and add regression coverage for timeout/abort leakage of reply directives and reasoning content.

## Best Possible Solution

Land this PR or an equivalent abort-aware guard in `buildEmbeddedRunPayloads` after green targeted tests and required checks, preserving the current final-answer/media behavior while ensuring aborted assistant turns cannot deliver partial answer or reasoning artifacts.

## Reproduction Assessment

Yes. A focused static reproduction is to call `buildEmbeddedRunPayloads` with non-empty `assistantTexts`, `lastAssistant.stopReason: "aborted"`, and `errorMessage: "request timed out"`; current main does not enter the error path and can emit the partial text, while the PR adds the missing suppression and clean error surface.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix I found: it reuses the existing assistant error formatter, keeps the current media/final-answer and `thinkingLevel !== "off"` branches, and only adds aborted-turn suppression where payload assembly decides user-visible output.
