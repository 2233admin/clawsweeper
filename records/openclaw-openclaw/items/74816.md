## Summary

Keep open. Current main still treats only `stopReason === "error"` as an assistant error surface, so aborted assistant text and reasoning can still become user-facing payloads before the runner's no-payload timeout fallback runs. This rebased PR directly adds the missing aborted-turn guard while preserving current final-answer/media selection logic, and I did not find a concrete correctness or security regression in the diff.

## What This Changes

The PR updates embedded run payload assembly to treat aborted assistant turns as error surfaces, suppress aborted answer and reasoning text, and adds regression tests for timeout/abort leakage of reply directives and reasoning content.

## Best Possible Solution

Land this PR or an equivalent abort-aware guard in `buildEmbeddedRunPayloads` after green targeted tests and required checks, preserving the current final-answer/media behavior while ensuring aborted assistant turns cannot deliver partial answer or reasoning artifacts.
