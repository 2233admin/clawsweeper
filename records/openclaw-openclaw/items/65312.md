## Summary

Keep open. Current main still enforces the 2 MB local avatar cap and returns oversized local avatars as unavailable, while the CLI and gateway agent docs still describe avatar sources without documenting that size limit. The previously linked closing PRs are now closed and unmerged, so this remains a narrow docs gap.

## What This Changes

Document the existing 2 MB size limit for local agent avatar image files in the agent identity docs.

## Best Possible Solution

Add a short user-facing note in `docs/cli/agents.md` and `docs/gateway/config-agents.md` beside the avatar path descriptions, keeping the existing runtime cap unchanged unless maintainers separately choose to revisit the security policy.
