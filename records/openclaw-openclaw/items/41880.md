## Summary

Keep this PR open. Current main still sends Feishu text posts as a raw `md` element, so the linked underscore-URL bug is not implemented on main. The PR remains a plausible narrow fix with no blocking review or security finding, but it is dirty against current main and overlaps with the broader open formatter PR #68514, so maintainers should choose the canonical approach and rebase/review rather than close it as cleanup.

## What This Changes

This PR changes Feishu post payload construction so single bare HTTP(S) URL messages are emitted as explicit rich-text link elements and adds regression coverage for underscore-containing URLs while preserving normal Markdown payloads.

## Best Possible Solution

Land one Feishu plugin-local URL preservation fix, then close #41860 and explicitly supersede the overlapping Feishu URL PRs. If maintainers only want the reported URL-only command case, this PR's explicit link-element approach is sufficient after rebase; if embedded bare URLs in longer Markdown messages are in scope, #68514 should be repaired and reviewed as the canonical formatter fix.
