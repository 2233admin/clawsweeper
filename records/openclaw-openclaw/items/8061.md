## Summary

Keep this issue open. Current main still supports only a global subagent concurrency cap at agents.defaults.subagents.maxConcurrent. Per-agent config entries have subagent allow/model/thinking/require-agent controls, but no typed, validated, documented, or enforced per-target-agent maxConcurrent setting, and both native and ACP subagent dispatch still use the single global subagent lane.

## Best Possible Solution

Keep this issue open and implement a focused core config/runtime change: add a validated per-target subagent concurrency setting, preferably agents.list[].subagents.maxConcurrent, document it, and enforce it alongside the existing global subagent lane for native and ACP sessions_spawn.
