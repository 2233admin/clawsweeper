## Summary

Keep open. Current main still has only the generic Cron Control UI surface: no Reminders tab/route, no reminder-specific render branch, no cron job classifier/display metadata in the wire or stored job type, and no backend cron.list filter that can separate reminder one-shot jobs from ordinary cron jobs. The prior maintainer review comment also kept this open for the same missing core UI/protocol work, and current main has not closed that gap.

## Best Possible Solution

Keep this open as a focused core Control UI plus Gateway cron protocol enhancement: add optional cron classification/display metadata for reminders, have reminder-producing paths set it, add typed reminder/non-reminder filtering to cron.list/listPage, hide or filter reminders from the Cron tab, and add a Reminders tab with cleaned reminder text, fire time, channel/recipient, and delete actions.
