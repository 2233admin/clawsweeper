Keep open. Current main still has the reported MCP channel bridge retention pattern, and the linked closing PR remains open/unmerged, so this should wait for maintainer review or an equivalent landed fix.

## What This Changes

Request bounded cleanup for the MCP channel bridge pending Claude permission and approval maps to avoid long-running `openclaw mcp serve` memory growth.

## Best Possible Solution

Land #71648 or an equivalent maintainer-approved MCP bridge patch that clears both pending maps on close, prevents post-close writes, expires stale Claude permission and approval entries, preserves the MCP protocol surface, and adds focused regression tests. Yes. Yes for keeping this issue open. The narrow maintainable fix belongs in `src/mcp/channel-bridge.ts` with MCP regression tests, and the already-linked closing PR is the right path to review, rebase, land, or replace instead of queuing duplicate automation.
