## Summary

Keep open. Current main still has the reported MCP bridge retention pattern: both pending maps are unbounded, close only clears waiters, Claude permission cleanup depends on a matching channel reply, and approval cleanup depends on a matching resolved event. The linked closing PR remains open and unmerged, so the issue should stay open until that or an equivalent fix lands.

## What This Changes

The issue asks to add bounded lifetime and cleanup for MCP bridge pending Claude permission and approval maps so long-running `openclaw mcp serve` processes do not retain missed or unresolved entries indefinitely.

## Best Possible Solution

Land #71648 or an equivalent maintainer-approved MCP bridge patch that clears both pending maps on close, avoids post-close writes, expires stale Claude permission and approval entries, preserves the MCP tool/notification protocol, and adds focused regression tests. If burst caps are considered a separate hardening step, track that explicitly before closing this issue.
