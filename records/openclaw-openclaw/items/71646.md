## Summary

Current main still has the reported unbounded MCP channel-bridge pending maps, and the paired open PR that would fix them has not landed. The issue should remain open pending review and merge of #71648 or an equivalent maintainer-approved patch.

## What This Changes

Request to bound `OpenClawChannelBridge` pending Claude permissions and pending approvals with TTL cleanup, close-time clearing, and a cap to prevent long-running MCP memory growth.

## Best Possible Solution

Land #71648 or an equivalent patch that clears both pending maps on bridge close, avoids post-close writes, expires stale Claude permission and approval entries using explicit TTL or `expiresAtMs`, and adds focused regression tests. If maintainers want burst protection beyond the slow-leak fix, track or include a narrow cap/FIFO guard explicitly.
