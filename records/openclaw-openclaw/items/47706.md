## Summary

Keep this PR open. Current main still lacks the adaptive resource-profile behavior, but the patch is not ready to merge because it is broad, stale against current plugin paths, and still has concrete correctness/security review findings in the proposed changes.

## What This Changes

This PR adds resource-profile detection and applies low-memory/ARM defaults across FFmpeg and image processing, memory embedding and LanceDB search, systemd cgroup rendering, plugin-loader test setup, and Raspberry Pi documentation.

## Best Possible Solution

Revive this as a maintainer-reviewed resource-policy change or replace it with a narrower branch ported to the current plugin layout, with fixed test cleanup, clamped low-profile image grids, validated systemd directive rendering, matching Raspberry Pi docs, and focused coverage for media, memory-core, memory-lancedb, browser screenshots, plugin loading, and systemd behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Restore the global process stub after the ARM test:** `src/infra/platform-profile.test.ts:34`
  - body: `vi.stubGlobal("process", ...)` is not undone by `vi.restoreAllMocks()`, so after this test runs the worker can keep the mocked ARM `process` for later tests. Add `vi.unstubAllGlobals()` in teardown or avoid replacing the whole process object.
  - confidence: 0.93
- **[P2] Clamp low-profile resize grids before the first encode:** `src/media/image-ops.ts:28`
  - body: The low-profile grid still starts with `sideStart`, so a 2000px input still performs the expensive 2000px encode pass before trying the reduced sizes. Clamp the first low-profile entry to the low ceiling, for example `Math.min(sideStart, 1200)`, so the optimization actually starts lower.
  - confidence: 0.82
- **[P3] Validate systemd memory limit values before rendering:** `src/daemon/systemd-unit.ts:53-58`
  - body: The new `resourceLimits` strings are appended to the unit without the CR/LF checks used for description, args, env, and EnvironmentFile values. Keep these values as a closed set or validate them before rendering so future callers cannot inject extra unit lines.
  - confidence: 0.74
