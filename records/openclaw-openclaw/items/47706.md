## Summary

Keep this PR open. Current main still lacks the requested adaptive resource-profile behavior, but the proposed patch is broad, stale against current memory/browser plugin layout, and still has concrete review findings that should be addressed before merge.

## What This Changes

This PR adds resource-profile detection and applies low-memory/ARM defaults across FFmpeg and image compression, memory embedding and LanceDB search, plugin-loader cache behavior, systemd cgroup rendering, and Raspberry Pi docs.

## Best Possible Solution

Revive this as a maintainer-reviewed resource-policy change or replace it with a freshly rebased branch that ports the helper into the current core/plugin layout, fixes the test isolation and image-grid findings, validates systemd directive rendering, and keeps Raspberry Pi docs aligned with the shipped behavior.

## Reproduction Assessment

Not applicable as a user bug reproduction; this is a feature PR review. The patch issues are reproducible by static inspection of the PR diff, and current-main inspection shows the requested adaptive behavior is not already implemented.

## Solution Assessment

No, not in the current form. The direction is useful, but the narrowest maintainable path is a fresh or carefully rebased patch that fixes the P2 findings, ports moved memory/browser paths, and treats systemd resource limits as validated policy rather than freeform strings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Restore the global process stub after the ARM test:** `src/infra/platform-profile.test.ts:34`
  - body: `vi.stubGlobal("process", ...)` is not undone by `vi.restoreAllMocks()`, so later tests in the same worker can keep the mocked ARM process. Add `vi.unstubAllGlobals()` in teardown or avoid replacing the whole process object.
  - confidence: 0.93
- **[P2] Clamp low-profile resize grids before the first encode:** `src/media/image-ops.ts:28-29`
  - body: The low-profile grid still starts with `sideStart`, so a 2000px input still performs the expensive 2000px encode pass before trying reduced sizes. Clamp the first low-profile entry to the low ceiling, for example `Math.min(sideStart, 1200)`.
  - confidence: 0.82
- **[P3] Validate systemd memory limit values before rendering:** `src/daemon/systemd-unit.ts:53-58`
  - body: The new `resourceLimits` strings are appended to the unit without the CR/LF checks used for description, args, env, and EnvironmentFile values. Keep these values as a closed set or validate them before rendering so future callers cannot inject extra unit lines.
  - confidence: 0.78
