## Summary

Keep open. Current main still provides immediate model setting, manual fallback management, OpenRouter-specific scan/probe support, and runtime failover, but not the requested staged migration flow with generic verification, promotion, old-primary preservation, and operator rollback. The issue also has explicit maintainer keep-open guidance in the discussion.

## What This Changes

The issue requests a staged model migration workflow that lets users add and test a new model before promoting it to primary, keeps the old primary as fallback, and supports rollback.

## Best Possible Solution

Add a maintainer-designed `openclaw models` migration flow that stages any provider/model candidate, verifies it before promotion, preserves the old primary in the fallback chain, records enough rollback state for a one-command restore, and documents/tests the resulting config transitions.
