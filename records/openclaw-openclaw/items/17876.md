## Summary

Keep this open. Current main still provides immediate default-model setting, manual fallback management, OpenRouter scan/probe helpers, auth/status probes, and runtime failover, but it does not provide the requested staged model migration workflow with arbitrary candidate testing, promote-with-old-primary-fallback, and user-facing rollback. The issue discussion also already contains explicit maintainer keep-open guidance from steipete.

## Best Possible Solution

Keep this open for a maintainer-designed core `openclaw models` migration workflow. The best implementation should build on existing status/probe, fallback, auth, and scan primitives: stage an arbitrary candidate as fallback, test it deeply enough to cover sub-agents and tool use, promote it only after verification while preserving the old primary as fallback, persist previous config for rollback, and add docs/tests for those config transitions.
