## Summary

Keep this issue open. Current main still has immediate default model setting, manual fallback management, OpenRouter scan/probe helpers, and auth/status probes, but not the requested staged model migration workflow with arbitrary candidate testing, promotion, old-primary fallback insertion, and rollback. The issue discussion also includes explicit maintainer guidance to keep this open.

## Best Possible Solution

Keep this open for a maintainer-designed core `openclaw models` workflow. The best implementation should build on existing status/probe, fallback, auth, and scan primitives: stage a candidate as fallback, test an arbitrary provider/model ref deeply enough to cover sub-agents and tool use, promote only after verification while preserving the old primary as fallback, persist previous config for rollback, and add docs/tests for the config transitions.
