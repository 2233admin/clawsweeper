## Summary

Keep open. Current main still documents the affected `curl | bash` installer path, and the installer still has spinner/child-command paths that can inherit the script pipe as stdin while gum remains enabled from stdout/stderr or `/dev/tty`; the linked open PR is useful context but has not merged and is docs-only.

## What This Changes

The issue asks OpenClaw to stop the documented piped installer from hanging or truncating `install.sh` when interactive installer UI consumes the script stream on WSL2/Linux.

## Best Possible Solution

Keep this issue open until the installer itself prevents gum/spinner/subprocess paths from reading the piped script stream, likely by redirecting interactive UI to `/dev/tty` and noninteractive subprocess stdin to `/dev/null` or an equivalent safe fd strategy, then update the public install docs and smoke coverage around the official one-liner.

## Reproduction Assessment

Yes. The issue gives a concrete WSL2 Ubuntu 24.04 command and repeated failure, and current main still has a matching static path where gum/spinner commands can inherit piped stdin; I did not run the destructive full installer locally.

## Solution Assessment

No. The currently linked docs-only `bash -s --` approach is not the best complete fix because it does not change the installer processes that may read from stdin; the safer solution belongs in `scripts/install.sh`, with docs updated after the behavior is safe.
