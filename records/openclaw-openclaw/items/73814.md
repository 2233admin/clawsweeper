## Summary

Keep open. Current main still publishes the reported piped installer command, the issue is paired with open closing PR #73830, and the review did not prove the WSL2 stdin-corruption hang is fixed end to end.

## What This Changes

Prevent the macOS/Linux installer from hanging or corrupting streamed input when run through the documented `curl | bash` install path on WSL2 Ubuntu.

## Best Possible Solution

Keep this open until #73830 is reviewed and either landed with a correct fix or replaced by a narrower installer/docs change. The desired end state is a public install path that cannot let prompts consume the streamed script, with docs and an installer smoke or unit check covering that behavior while #73837 tracks the separate Node auto-installer failure.

## Reproduction Assessment

Unclear. The report gives a concrete WSL2 Ubuntu command and current docs still publish that path, and a minimal shell check confirms the stdin-consumption mechanism is plausible; this review did not run the full mutating installer.

## Solution Assessment

No/unclear for the linked docs-only direction. `bash -s --` is useful for argument passing but does not isolate stdin by itself, so the safer fix is to ensure interactive installer reads/subprocesses use `/dev/tty` or to document an invocation that separates script input from prompt input.
