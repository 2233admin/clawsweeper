## Summary

Keep this open. Current main still parses Anthropic streamed `tool_use` argument buffers through `parseStreamingJson` without the unsafe-integer guard that exists for the Ollama path, so large numeric IDs can still be rounded before tool execution.

## What This Changes

Anthropic Messages streaming should preserve large integer `tool_use.input` values instead of coercing them through JavaScript number parsing.

## Best Possible Solution

Add a shared unsafe-integer-preserving JSON helper and use it before the Anthropic streamed tool argument parser can coerce large integer literals; keep Ollama coverage aligned and add Anthropic regression coverage for safe integers, unsafe integers, nested values, and split deltas.

## Reproduction Assessment

Yes. The source path from Anthropic `input_json_delta` to `parseStreamingJson` is visible on current main, and a parser-level probe reproduces rounding for the reported Discord snowflake-sized value.

## Solution Assessment

Yes. A neutral parser guard before numeric coercion is the narrow maintainable fix; monkey-patching vendored SDK or dependency internals would be more fragile.
