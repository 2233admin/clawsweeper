## Summary

Current main still has an Anthropic Messages tool-use argument path that parses accumulated `input_json_delta` JSON through `parseStreamingJson`, which ultimately uses `JSON.parse`/`partial-json` without the unsafe-integer quoting guard that exists in the Ollama plugin. The exact old Anthropic SDK `MessageStream` root cause has shifted in current code, but the user-visible precision-loss problem remains valid for Anthropic-style tool input.

## Best Possible Solution

Keep this issue open and fix the Anthropic Messages tool-use input parser so unsafe integer literals in raw `input_json_delta` buffers are preserved as exact strings before any `JSON.parse`/partial parser can round them. The best code direction is a shared unsafe-integer-preserving JSON helper or Anthropic-local wrapper with focused regression tests; do not move core code across the Ollama plugin boundary unless extracting a neutral shared helper.
