## Summary

Keep this issue open. Current main still defaults `commands.nativeSkills` to `"auto"`, still resolves that through Discord/Telegram plugin metadata as enabled, and still documents/generated-publishes the auto default. Existing Discord/Telegram overflow handling reduces startup failure risk, but it does not make native skill command registration opt-in; the narrower diagnostic-warning path is still represented by open PR #50191.

## Best Possible Solution

Keep this open for maintainer judgment. The concrete path is to review and either land or close #50191 for targeted overflow guidance. If maintainers choose the default change, update schema defaults, generated schema/config metadata, public docs, resolver tests, Discord/Telegram behavior expectations, and changelog/migration notes together; if they keep `auto`, make overflow diagnostics consistently point at the narrower `nativeSkills` setting.
