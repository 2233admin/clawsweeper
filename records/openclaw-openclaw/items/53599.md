## Summary

Keep open. Current main still intentionally removes the Chrome extension relay path, rejects the legacy extension profile driver, and documents only host-local Chrome MCP, raw CDP, or node-host proxy alternatives; the managed-hosting cross-machine extension relay workflow described by the reporter and confirmed by commenters remains unserved.

## What This Changes

Restore or replace the removed Chrome extension browser relay so remote OpenClaw deployments can control a user's local Chrome across machines without requiring a local Node.js host or manual remote-CDP setup.

## Best Possible Solution

Keep this open until maintainers choose a deliberate browser strategy: either a hardened opt-in Chrome extension relay owned by the browser plugin, or an additive public browser profile-driver/transport API that lets an external plugin implement the relay without private internals, with schema, docs, tests, and a threat model.
