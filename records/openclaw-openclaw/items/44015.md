## Summary

Keep open. The requested hardening is still absent on current main, but the proposed patch should not merge as-is because it applies the byte cap after NFC normalization, allowing oversized decomposed input to be accepted under the documented request-body limit. The schema addition also needs contract cleanup because JSON Schema `maxLength` is character-based, not UTF-8 byte-based.

## What This Changes

The PR adds a 512 KiB UTF-8 byte limit for gateway `chat.send` messages in the sanitizer and protocol schema, expands sanitizer tests, and documents the gateway protocol limit.

## Best Possible Solution

Revise this PR to define the byte policy explicitly, enforce the 512 KiB cap in the actual `chat.send` path before expensive normalization when the policy is request-size protection, make the schema guard clearly advisory or replace it with a contract that cannot be mistaken for byte enforcement, and align docs plus ASCII, multi-byte, and decomposed-Unicode regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Check raw message bytes before normalization:** `src/gateway/server-methods/chat.ts:238`
  - body: The new check measures `Buffer.byteLength(normalized, "utf8")` after NFC normalization. A decomposed payload such as `e\u0301` repeated enough times can be over 512 KiB on input, normalize below the cap, and still be accepted despite the documented request-body limit. Check the incoming string before normalization, or explicitly document and test normalized-message semantics.
  - confidence: 0.9
- **[P3] Clarify that the schema limit is character-based:** `src/gateway/protocol/schema/logs-chat.ts:35-36`
  - body: The added TypeBox `maxLength` is a JSON Schema string-length guard, not UTF-8 byte enforcement. Multi-byte messages can pass this layer well above 512 KiB and rely on the sanitizer, so the comment should say this is advisory/ASCII-aligned or the schema guard should be removed or changed.
  - confidence: 0.82
