## Summary

Keep open. Current main still does not implement the requested WhatsApp read-only DM observation mode: the config contract and docs have no `observe` policy, blocked DMs are dropped before any persistence path, and admitted DMs still proceed into hooks, session routing, and agent-turn dispatch. The earlier ClawSweeper review remains accurate on the checked surfaces, and the remaining work needs maintainer privacy/security design before implementation.

## What This Changes

The issue requests a WhatsApp `dmPolicy: "observe"` mode that records incoming DMs for later read-only agent access without creating sessions, triggering agent turns, or exposing reply paths.

## Best Possible Solution

If maintainers approve the privacy model, implement this inside the WhatsApp plugin as a pre-routing observe policy accepted by the shared config schema, persist bounded observation records before session resolution, skip hooks/session/agent dispatch for observed DMs, and expose a guarded read tool with account, sender, time-range, retention, deletion, and redaction controls.

## Reproduction Assessment

Yes for source-level verification, not as a live WhatsApp runtime repro. Current main rejects `dmPolicy: "observe"` through the schema, drops unauthorized or disabled DMs before persistence, and routes allowed DMs through `runInboundReplyTurn`.

## Solution Assessment

Unclear pending maintainer product/security review. A WhatsApp-owned pre-routing observe branch is the narrowest direction that matches the requested mechanical safety; `session.sendPolicy: "deny"` and the current generic `observeOnly` turn seam are not sufficient by themselves.

## Security Review

Status: needs_attention

Summary: The issue is privacy-sensitive because the requested feature would persist and expose WhatsApp DM content outside normal session turns.

Concerns:

- **[medium] Define observation retention and access controls:**
  - body: The proposed store would contain WhatsApp message bodies, phone numbers, sender names, account IDs, and timestamps. A safe design needs bounded retention, deletion behavior, redaction rules, account scoping, and explicit tool-access authorization before agents can query it.
  - confidence: 0.9
- **[medium] Keep observed DMs out of hooks and turns:** `extensions/whatsapp/src/auto-reply/monitor/process-message.ts:435`
  - body: Current WhatsApp processing emits message hooks and calls `runInboundReplyTurn` for admitted messages. An observe implementation should intercept before session resolution so observed DMs cannot trigger hooks, model turns, reply dispatch, or message-tool reply affordances.
  - confidence: 0.86
