## Summary

Keep open. Current main does not implement a WhatsApp `dmPolicy: "observe"` mode: the shared DM policy type/schema and WhatsApp docs only allow `pairing`, `allowlist`, `open`, and `disabled`. Existing `session.sendPolicy: "deny"` is not equivalent because the dispatcher explicitly still processes the inbound message through the agent while only suppressing outbound delivery; this issue asks for DMs to be logged without creating/routing an agent session or firing a turn.

## Best Possible Solution

Keep this issue open for maintainer product/security review. The best implementation would likely be a WhatsApp-owned `dmPolicy: "observe"` path that accepts the new enum in the shared config contract, intercepts authorized direct-message inbound after parsing but before route/session resolution, appends a bounded privacy-aware observation record, and exposes a guarded query tool only if maintainers approve the data-access model. If the needed pre-routing observation/query surface should be generic, add it as a documented plugin SDK seam rather than hard-coding a one-off core bypass.
