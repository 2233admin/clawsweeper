## Summary

Keep open. Current main still does not implement a WhatsApp `dmPolicy: "observe"` mode: the config contract rejects it, blocked DMs are dropped before persistence, and admitted DMs still enter the session/agent turn path. Existing send suppression and generic `observeOnly` turn plumbing are not equivalent to the requested no-session, no-turn observation log, and the remaining design needs maintainer privacy/security review.

## What This Changes

The issue requests a WhatsApp `dmPolicy: "observe"` mode that records direct messages for later read-only querying without creating sessions, triggering agent turns, or exposing a reply path.

## Best Possible Solution

If maintainers approve the privacy model, implement this inside the WhatsApp plugin as a pre-routing observe policy accepted by the shared config schema, append bounded observation records before session resolution, skip hooks/route/session/agent dispatch for observed DMs, and expose a guarded read tool with account, sender, time-range, retention, and redaction controls plus docs and targeted tests.

## Reproduction Assessment

Yes for source-level verification, not a live WhatsApp runtime repro. Current main rejects `dmPolicy: "observe"` through the schema, drops unauthorized/disabled DMs before persistence, and routes allowed DMs through `runInboundReplyTurn`.

## Solution Assessment

Unclear pending maintainer product/security review. A WhatsApp-owned pre-routing observe branch is the narrowest direction that matches the mechanical safety requirement; `session.sendPolicy: "deny"` and the current generic `observeOnly` turn seam are not sufficient by themselves.

## Security Review

Status: needs_attention

Summary: The issue is privacy-sensitive because the requested feature would persist and expose WhatsApp DM content outside normal session turns.

Concerns:

- **[medium] Define observation retention and access controls:**
  - body: The proposed store would contain WhatsApp message bodies, phone numbers, sender names, account IDs, and timestamps. A safe design needs bounded retention, deletion behavior, redaction rules, account scoping, and explicit tool-access authorization before agents can query it.
  - confidence: 0.9
- **[medium] Keep observed DMs out of hooks and turns:** `extensions/whatsapp/src/auto-reply/monitor/process-message.ts:457`
  - body: Current WhatsApp processing emits message hooks and calls `runInboundReplyTurn` for admitted messages. An observe implementation should intercept before session resolution so observed DMs cannot trigger hooks, model turns, reply dispatch, or message-tool reply affordances.
  - confidence: 0.86
