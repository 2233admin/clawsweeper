## Summary

Keep open. Current main does not yet rewrite the QMD config index during managed-collection repair, and the proposed implementation has a likely repair-path regression: it can skip the remove/add reset when QMD lists matching collections, which is the recovery behavior current code and tests still protect.

## What This Changes

This PR adds an `index.yml` rewrite before QMD collection bootstrap and changes managed QMD collection repair to call `ensureCollections()` instead of the existing unconditional remove/add loop, with test expectations updated accordingly.

## Best Possible Solution

Keep the useful index-rewrite idea, but implement it as a small helper that writes a quoted/escaped QMD config file and call it before both normal collection bootstrap and the existing unconditional repair remove/add loop.
