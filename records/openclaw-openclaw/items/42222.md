## Summary

Keep this PR open. Current main still does not implement the requested Discord `allowBots=false` diagnostic: the active Discord plugin path only emits the old terse verbose message before dropping bot-authored messages. The PR remains a narrow implementation candidate, but it needs to be ported from the removed `src/discord` layout to `extensions/discord`.

## Best Possible Solution

Keep this PR open as a useful focused implementation candidate. The maintainer path is to port the behavior to `extensions/discord/src/monitor/message-handler.preflight.ts` and `extensions/discord/src/monitor/message-handler.preflight.test.ts`: add a `logDebug` diagnostic plus a clearer `logVerbose` hint on the existing `allowBotsMode === "off"` bot-drop branch, preserve default filtering behavior, and add a regression test for that exact log path.
