Keep this PR open. Current main still does not implement the requested Discord `allowBots=false` diagnostic: the active Discord plugin still drops default-blocked bot messages with only the old terse verbose log, while this PR’s patch targets the removed `src/discord` layout instead of the current `extensions/discord` plugin files.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. The maintainer path is to port the same behavior to `extensions/discord/src/monitor/message-handler.preflight.ts` and `extensions/discord/src/monitor/message-handler.preflight.test.ts`: add a `logDebug` diagnostic and clearer `logVerbose` hint on the existing `allowBotsMode === "off"` bot-drop branch, preserve the default filtering behavior, and add a regression test for that exact log path.
