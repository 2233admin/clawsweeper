## Summary

Keep this PR open. The intended Discord `allowBots=false` diagnostic is still missing from current `main`: the active Discord plugin still drops default-blocked bot messages with only the terse verbose log. The original patch was narrowly scoped and Greptile's review correctly found no runtime behavior change in that old layout, but current `main` uses `extensions/discord/...`, while this PR edits the removed `src/discord/...` path.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Port the same behavior to `extensions/discord/src/monitor/message-handler.preflight.ts` and `extensions/discord/src/monitor/message-handler.preflight.test.ts`: add a `logDebug` diagnostic and clearer `logVerbose` hint on the existing `allowBotsMode === "off"` bot-drop branch, preserve the current default filtering behavior, and add a regression test for that exact log path.
