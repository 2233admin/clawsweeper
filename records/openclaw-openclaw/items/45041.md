## Summary

Keep open. Current main still does not expose a global or per-agent `outputFilters`-style config surface, and the existing delivery sanitizers cover tagged reasoning, `isReasoning` payloads, tool/runtime scaffolding, and provider artifacts rather than configurable untagged meta-planning preambles. The open related PR #60551 is the nearest implementation candidate, so this needs maintainer product review rather than a competing automated repair.

## What This Changes

The issue requests an optional global or per-agent output filter that strips leaked reasoning or meta-planning text from assistant messages before channel delivery.

## Best Possible Solution

Keep this as the product/design tracker until maintainers settle on one path: first-class opt-in config filters, a conservative shared sanitizer, or a documented plugin/API hook. Any implementation should align config types, zod schema, generated docs/help, delivery ordering, TTS/media behavior, and regression coverage for English and CJK preambles plus false-positive cases.
