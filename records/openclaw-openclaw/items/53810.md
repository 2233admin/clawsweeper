## Summary

Keep open. Current main still lacks Qwen/ModelStudio tagged tool-call normalization for reasoning/thinking output, so the prior ClawSweeper keep-open comments remain accurate after rechecking the current SHA. The bundled affected models are still user-facing, the Qwen wrapper only patches thinking request fields, the generic OpenAI-compatible stream path still only creates tool calls from structured `delta.tool_calls`, and the existing tagged-marker parser remains Kimi-plugin-local and text-block-only.

## What This Changes

The issue requests provider/model-specific handling for subagent tool calls emitted as tagged marker text inside thinking/reasoning blocks so subagent completion is not announced before tool work runs.

## Best Possible Solution

Add provider-owned Qwen/ModelStudio stream normalization, or a narrowly gated shared OpenAI-compatible helper, that extracts only validated standalone tagged tool-call sections from reasoning/thinking content, emits structured `toolCall` blocks, and changes `stopReason` to `toolUse` only after extraction succeeds. Keep the existing Kimi behavior covered while adding explicit regression coverage for Qwen/ModelStudio marker text in thinking blocks and normal structured tool-call streams.
