## Summary

Keep open: current main does not already implement the PR's update-run continuation path. The existing restart startup code can drain a sentinel continuation when present, but `update.run` currently rejects/omits `continuationMessage`, does not write `payload.continuation`, and the main-branch tests still assert update sentinels have no continuation. Security review did not find unexplained supply-chain or permissions changes in the supplied PR metadata, but the linked issue is broader than this one-shot continuation fix.

## What This Changes

This PR adds `continuationMessage` to `update.run` protocol params, forwards it from the agent gateway tool, writes an `agentTurn` continuation into successful update restart sentinels, and updates docs, tests, changelog, and generated Swift protocol models.

## Best Possible Solution

Land or revise a narrow additive protocol/server/agent-tool change that writes an agent-turn continuation only for successful `update.run` restarts, keeps failed/skipped updates without a continuation, and either narrows the closing language or leaves follow-up tracking for the broader message-loss cases in #71178.
