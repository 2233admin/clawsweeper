## Summary

Keep open. Current main still lacks the `update.run` restart-continuation producer/protocol path, while this PR is an active narrow implementation for that gap rather than obsolete work. The remaining decision is maintainer review of the additive protocol/client sync, CI, trust boundary, and whether the changelog should avoid implying that all of #71178 is fixed.

## What This Changes

This PR adds `continuationMessage` to `update.run`, forwards it from the gateway agent tool, writes an `agentTurn` continuation into successful update restart sentinels, and updates tests, generated Swift protocol models, docs, and changelog.

## Best Possible Solution

Land or revise a narrow additive Gateway protocol/server/agent-tool change that queues one follow-up `agentTurn` only after successful `update.run` restarts, leaves failed and skipped updates without continuations, keeps generated protocol clients/docs/tests in sync, and preserves #71178 as the broader follow-up unless maintainers decide the remaining root causes are covered elsewhere.
