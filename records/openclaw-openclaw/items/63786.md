## Summary

Keep this open. Current main still has the Copilot Claude replay gap the branch targets, but the proposed patch still needs revision because it selects the Copilot sanitizer from raw provider strings and relaxes security-adjacent guard scripts.

## What This Changes

The branch adds a GitHub Copilot-specific thinking-block sanitizer, routes session replay, tool-continuation replay, and context-pruning estimates through it, updates replay tests, and includes ACP/thread-binding guard-script/test changes.

## Best Possible Solution

Model the replay sanitizer as a normalized provider/policy strategy, apply it consistently to session-history replay, tool-continuation replay, and context-pruning estimates, keep the non-Copilot Anthropic latest-turn behavior intact, and split or explicitly justify the ACP/guard-script changes before maintainer review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize provider before selecting Copilot sanitizer:** `src/agents/pi-embedded-runner/replay-history.ts:428`
  - body: `resolveTranscriptPolicy()` normalizes provider ids before setting `dropThinkingBlocks`, but this branch chooses the stricter Copilot sanitizer with a raw `params.provider === "github-copilot"` check. A Copilot run that enters with a differently cased or otherwise unnormalized provider id can still get the Copilot policy but fall back to the generic helper, preserving latest-turn `thinking` blocks and reproducing the follow-up failure. Derive this from the normalized provider or an explicit policy strategy in all three call sites.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: No dependency, secret, or third-party execution change was found, but the PR weakens guard-script enforcement in security-adjacent areas.

Concerns:

- **[low] Avoid broad tmpdir guard allowlists:** `scripts/check-no-random-messaging-tmp.mjs:20`
  - body: The PR adds many plugin/runtime paths to the `os.tmpdir()` guard allowlist instead of migrating or documenting each callsite. That guard exists to keep messaging/channel runtime code off host-default tmp paths, so these exemptions should be narrowed, justified, or split from the Copilot replay fix.
