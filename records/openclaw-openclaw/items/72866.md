## Summary

Keep this PR open. Current main still has the exact asymmetry described: `hasUserBody` treats non-empty `InboundHistory` as valid input, but the `isBareSessionReset` prompt branch still omits `inboundUserContext`, so the context that allowed the run is not passed to the agent. This PR is a plausible implementation candidate, not obsolete on main.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to update the regression test so it forces the bare reset branch, then review and land the one-line prompt assembly fix if the focused tests pass.
