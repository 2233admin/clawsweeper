## Summary

Keep this issue open. Current main still uses the Baileys 7.0.0-rc.9 WhatsApp Web backend and does not include the unified-session handshake patch proposed for the reported silent QR-pairing failure. An open PR, #74349, explicitly targets this issue, so the issue should stay open until that PR is reviewed and merged or closed.

## What This Changes

The issue reports that WhatsApp QR pairing on macOS ARM64 Docker shows a QR and logs `WhatsApp QR received.`, but never reaches `connection.update` open or a surfaced failure after the phone scans it.

## Best Possible Solution

Review #74349 as the current implementation candidate. If maintainers accept the Baileys `sendUnifiedSession` approach, land it with changelog credit and live QR-pairing proof from QR receipt through `connection.update` open; otherwise keep the fix owned in `extensions/whatsapp` with an equivalent backend or dependency strategy that surfaces failures instead of silently waiting.
