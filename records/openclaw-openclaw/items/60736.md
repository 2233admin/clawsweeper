## Summary

Keep this issue open. Current main has partial Google Vertex metadata and credential detection, but it still does not implement the requested standalone google-vertex runtime path, project/location endpoint construction, CLOUD_ML_REGION/default location handling, or onboarding/auth choice. The provided GitHub context also shows open implementation PRs #60860 and #62103 that explicitly close this issue, including a same-author PR, so this should remain open until a canonical PR is reviewed and merged or rejected.

## Best Possible Solution

Keep this issue open while maintainers review a canonical Google Vertex implementation, likely #60860 or a consolidated successor. The fix should live in the Google plugin: register a real google-vertex provider, resolve project/location from GOOGLE_CLOUD_PROJECT or GCLOUD_PROJECT plus GOOGLE_CLOUD_LOCATION or CLOUD_ML_REGION with a safe default, route native Gemini requests to the Vertex AI project/location endpoint, handle ADC/OAuth bearer auth, expose onboarding/docs, and add focused route/auth tests.
