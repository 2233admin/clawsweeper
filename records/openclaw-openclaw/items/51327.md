## Summary

Keep open. Current main does not implement this PR's central behavior: after `agents.delete` with `deleteFiles=false`, transcripts can remain on disk, but `sessions.usage` still discovers only configured gateway agents and does not scan disk-backed deleted-agent transcript directories.

## Best Possible Solution

Keep this PR open or replace it with a maintainer-owned follow-up that updates `sessions.usage` discovery to include validated disk-backed transcript directories for deleted agents, preserves the real `sessionsDir`, uses async/non-fatal discovery, and adds regression coverage for `agents.delete` with `deleteFiles=false` across list, specific-key, timeseries, and logs paths.
