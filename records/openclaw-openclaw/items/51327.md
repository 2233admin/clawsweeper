## Summary

Keep this PR open. Current main still does not implement the PR’s intended usage-only discovery path for preserved deleted-agent transcript directories: `sessions.usage` still enumerates configured gateway agents only, while `agents.delete` with `deleteFiles=false` removes the agent from config and purges its session-store entries without deleting transcript files.

## Best Possible Solution

Keep this PR open, or replace it with a maintainer-owned successor that adds a validated read-only usage discovery target for preserved deleted-agent transcript directories. The implementation should keep `listAgentsForGateway()` config-scoped, pass the real `sessionsDir` through list, specific-key, timeseries, and logs usage paths, tolerate expected filesystem errors, and add regression coverage for `agents.delete` with `deleteFiles=false`.
