## Summary

Keep open. Current main already has scoped non-fatal handling for DBus-unavailable install probes, but the PR broadens the generic service-loaded contract and can turn an unknown transport failure into a false “not loaded” result for lifecycle callers.

## What This Changes

The PR branch adds `isSystemctlBusUnavailable(detail)` to the non-fatal condition in `isSystemdServiceEnabled` in `src/daemon/systemd.ts`.

## Best Possible Solution

Keep the install-specific non-fatal handling unless maintainers intentionally change the broader service-loaded contract. If they accept that broader change, lifecycle callers should distinguish disabled, missing, and unknown transport states, with targeted coverage for install, status, reset, and uninstall behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve DBus probe failures as errors:** `src/daemon/systemd.ts:635`
  - body: Returning `false` here turns an exhausted DBus/user-bus transport failure into “service not loaded.” Callers such as `openclaw uninstall --service` treat `false` as a successful not-loaded state and skip stop/uninstall, so an installed service can remain after an unavailable bus. Keep this as an error path or update lifecycle callers/tests to distinguish unknown from not loaded.
  - confidence: 0.88
