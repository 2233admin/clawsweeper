## Summary

Keep this PR open. Current main does not implement the proposed change: `isSystemdServiceEnabled` still returns false only for missing `systemctl` or disabled/not-found unit states, and DBus/user-bus failures still fall through to `systemctl is-enabled unavailable`. The PR is source-only and not a supply-chain concern, but the one-line behavior change needs maintainer review because lifecycle callers treat `false` as “service disabled/missing,” which can skip uninstall work.

## Best Possible Solution

Keep the PR open for maintainer review. If the behavior is desired, update it with tests and either preserve lifecycle error semantics or narrow the non-fatal handling to the install/status surface that actually needs it; otherwise close the PR after maintainer confirmation that DBus transport failures should remain errors for `isLoaded`.
