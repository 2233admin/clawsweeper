## Summary

Current main still does not implement persistent multi-turn webhook hook sessions. Direct and mapped agent hooks can carry a stable sessionKey, and resolveCronSession can reuse a fresh stored session when forceNew is false, but /hooks/agent dispatch still creates an isolated cron-style job and runCronIsolatedAgentTurn still passes forceNew: true for isolated jobs. The hook payload, mapping, transform, TypeScript config type, and strict Zod schema still have no sessionMode/reuseSession/targetExistingSession field. Prior implementation PRs #15084, #27123, and #27193 are closed unmerged; related #42621 was closed as superseded by this issue, not as a fix.

## Best Possible Solution

Keep this issue open and implement an explicit, backward-compatible persistent hook-session option in core. Add sessionMode: "isolated" | "persistent" for direct /hooks/agent payloads, hook mappings, and transform results; default to isolated; preserve all existing session-key security gates; and only pass forceNew: false for the explicit persistent hook-agent path so stable sessionKey values reuse fresh existing session ids without changing cron isolation.
