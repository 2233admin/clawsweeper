## Summary

Keep open. Current main still enforces the 5-turn ceiling, so this PR addresses a real linked request, but the branch is not correct as-is because it would drop newer session-tool docs and misses the current tracked config-doc hash and gateway config doc updates.

## What This Changes

The PR raises `session.agentToAgent.maxPingPongTurns` from 5 to 20 across runtime clamping, config validation, generated config schema/help metadata, docs, and helper tests while keeping the default at 5.

## Best Possible Solution

Adopt one maintainer-approved path for #52382, then update the runtime clamp, Zod schema, generated base schema, tracked `docs/.generated/config-baseline.sha256`, public config help/types, session and gateway docs, and focused tests together while preserving the default unless maintainers intentionally change it.

## Reproduction Assessment

Yes. Current main statically reproduces the cap: schema validation rejects values above 5, the resolver clamps configured values to 5, and the A2A loop consumes that capped value.

## Solution Assessment

No, not as currently proposed. Raising the cap is a plausible narrow fix, but this branch needs a rebase plus docs and generated-artifact corrections before it is the safest maintainable solution.
