## Summary

Keep this issue open. Current main still constructs and starts a fresh `GatewayClient` inside every `probeGateway()` call, with no per-URL/module-level `DEVICE_IDENTITY_REQUIRED` short-circuit before the WebSocket is opened. The reporter’s corrected RCA and closed-unmerged #63446 still match current source: reconnect is paused for one client instance, but repeated short-lived CLI/status/cron probes can create new probe instances.

## What This Changes

Add exponential/backoff-style suppression for repeated CLI WebSocket probes rejected with `device-required`, with a suggested source-level per-gateway cache and optional health endpoint alternative.

## Best Possible Solution

Keep this issue open until a source-level fix lands. The narrowest implementation is in `src/gateway/probe.ts`: track recent `DEVICE_IDENTITY_REQUIRED` probe failures per gateway URL, short-circuit after a small threshold/TTL without constructing `GatewayClient`, clear on successful connection, and add focused regression tests proving the fourth repeated device-required probe does not instantiate the mock client. If #68280 is used as the vehicle, it should explicitly cover this repeated device-required probe storm before this issue closes.

## Repair Work Prompt

Implement a focused fix for the repeated CLI WebSocket device-required probe storm. Current main still constructs a fresh `GatewayClient` for every `probeGateway()` call, so repeated short-lived CLI/status/cron probes can keep opening WebSockets after `DEVICE_IDENTITY_REQUIRED` closes. In `src/gateway/probe.ts`, add a module-level per-URL cache for recent device-required probe failures, using #63446 as guidance: after 3 device-required failures within a 5 minute TTL, return a synthetic failed `GatewayProbeResult` without constructing `GatewayClient`; include close code 1008, reason `device identity required`, and a hint that the probe was short-circuited; clear the cache on successful connection; keep caches independent per URL. Do not change gateway auth policy, generated dist files, or add a new health endpoint. Add focused tests in `src/gateway/probe.test.ts` proving the first three device-required closes instantiate the mock client, the fourth does not, non-device-required closes do not populate the cache, success clears the cache, and URLs are isolated. Coordinate with #68280 and avoid duplicating its local auth fail-fast work; if #68280 has landed when this starts, re-evaluate current main behavior first. Add a single-line user-facing changelog Fixes entry crediting `Thanks @djimit`; also credit `@hclsys` if the prior PR shape is reused.
