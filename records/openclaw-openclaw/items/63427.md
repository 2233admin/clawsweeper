## Summary

Keep this issue open. Current main still constructs a fresh probe GatewayClient for every probeGateway() call and has no per-URL/module-level DEVICE_IDENTITY_REQUIRED short-circuit, so the cross-session CLI/status/cron burst case described by the reporter is not fixed on main.

## Best Possible Solution

Keep this issue open until a source-level fix lands. The most direct path is to revive the #63446 approach in src/gateway/probe.ts: track recent DEVICE_IDENTITY_REQUIRED probe failures per gateway URL, short-circuit after a small threshold/TTL without constructing GatewayClient, clear the cache on successful connection, and add focused regression tests proving the fourth repeated device-required probe does not instantiate GatewayClient. If #68280 is intended to supersede this, it should explicitly cover the repeated device-required probe storm with equivalent tests before this issue closes.

## Clownfish Work Prompt

Implement a focused fix for the repeated CLI WebSocket device-required probe storm. Current main still constructs a fresh GatewayClient for every probeGateway() call, so repeated short-lived CLI/status/cron probes can keep opening WebSockets after DEVICE_IDENTITY_REQUIRED closes. In src/gateway/probe.ts, add a module-level per-URL cache for recent device-required probe failures, using #63446 as guidance: after 3 device-required failures within a 5 minute TTL, return a synthetic failed GatewayProbeResult without constructing GatewayClient; include close code 1008, reason 'device identity required', and a hint that the probe was short-circuited; clear the cache on successful connection; keep caches independent per URL. Do not change auth resolution, gateway connect policy, generated dist files, or add a new health endpoint. Add focused tests in src/gateway/probe.test.ts proving the first three device-required closes instantiate the mock client, the fourth does not, non-device-required closes do not populate the cache, success clears the cache, and URLs are isolated. Coordinate with #68280 and avoid duplicating its local auth fail-fast work. Add a user-facing changelog Fixes entry crediting Thanks @djimit and @hclsys if the prior PR shape is reused.
