## Summary

Keep this PR open. Current main still does not redirect gateway root `/` to the configured Control UI `basePath`; the root request falls through the Control UI classifier and then the gateway returns `404 Not Found`. The PR remains a narrow implementation candidate for the linked #34298 workflow, and the provided discussion shows the query-string preservation review note was addressed.

## Best Possible Solution

Keep this PR open for maintainer review and rebase or reapply it onto current main. The best implementation is the narrow gateway routing change in `classifyControlUiRequest`: when `basePath` is non-empty and `pathname === "/"`, redirect `GET` and `HEAD` to `${basePath}/${search}`, preserve non-read fallthrough for plugin/webhook routes, and keep regression tests for root redirect, query preservation, and write-method fallthrough. Also resolve the basePath validation/documentation question before merge.
