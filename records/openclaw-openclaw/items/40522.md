## Summary

Keep this PR open. Current main still does not redirect gateway root `/` to the configured Control UI `basePath`: the classifier returns `not-control-ui` for `/` when `basePath` is `/openclaw`, and the HTTP server then falls through to the generic `404 Not Found`. The provided PR remains a focused implementation candidate, and its later diff addresses the prior query-string preservation review note.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase or reapply the focused gateway routing change so `GET` and `HEAD` requests to `/` redirect to `${basePath}/${search}` when `gateway.controlUi.basePath` is set, while non-read root requests still fall through for plugin or webhook handling and regression tests cover root redirect, query preservation, and write-method fallthrough.
