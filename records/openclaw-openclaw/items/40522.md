## Summary

Keep this PR open. Current main still returns `not-control-ui` for `GET /` or `HEAD /` when `gateway.controlUi.basePath` is set, so the request falls through to the generic gateway 404 instead of redirecting to the Control UI base path. The PR remains a focused implementation candidate, and the provided discussion shows the query-string review gap was addressed.

## Best Possible Solution

Keep this PR open for maintainer review and land or reapply the focused routing change: when `gateway.controlUi.basePath` is set, `GET /` and `HEAD /` should redirect to `${basePath}/${search}`, while non-read root requests continue to fall through. Keep regression coverage for root redirect, query preservation, and write-method fallthrough, then run the targeted gateway routing test.
