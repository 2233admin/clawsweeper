## Summary

Keep this PR open. Current main still does not redirect gateway root `/` to the configured Control UI `basePath`; the request falls through Control UI routing and then becomes a gateway `404 Not Found`. The PR remains a focused implementation candidate for the linked Control UI workflow, with the query-string review note already addressed in the PR discussion.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase or reapply the narrow gateway routing change on current main: when `basePath` is non-empty and `pathname === "/"`, redirect `GET` and `HEAD` to `${basePath}/${search}`, preserve non-read fallthrough for plugin/webhook routes, keep regression tests for root redirect, query preservation, and write-method fallthrough, and resolve the basePath validation/documentation question before merge.
