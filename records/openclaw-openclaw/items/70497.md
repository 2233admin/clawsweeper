## Summary

Keep PR #70497 open. Current main still preserves `if`, `then`, and `else` in Gemini tool schemas because the shared Gemini cleaner's unsupported-keyword set omits those keywords, and the cleaner only skips keys present in that set. The PR's small change still addresses a real remaining gap and is in the right shared cleaner path.

## Best Possible Solution

Keep #70497 open and integrate the focused fix in `src/agents/schema/clean-for-gemini.ts`, with the regression test from the PR. That preserves the shared Gemini cleaner as the single owner for provider-specific schema keyword stripping.
