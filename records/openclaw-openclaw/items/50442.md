## Summary

Keep open. Current main still writes `openclaw backup create` output through an adjacent random `.tmp` file and only removes that scoped file when the async archive path reaches normal `finally` cleanup; I found no backup-owned SIGINT, SIGTERM, timeout, or AbortSignal cleanup path for externally interrupted runs. The discussion confirms the reporter is relying on a cleanup workaround, not a shipped OpenClaw fix.

## What This Changes

The issue asks for `openclaw backup create` to clean up large temporary `.tmp` archive files when a timeout or interruption kills the backup process.

## Best Possible Solution

Backup creation should own the active temporary archive lifecycle: track the exact temp archive during the write window, clean it on graceful interruption or abort, preserve the existing no-overwrite final publication behavior, avoid deleting unrelated `*.tmp` files, and document unavoidable hard-kill limits.
