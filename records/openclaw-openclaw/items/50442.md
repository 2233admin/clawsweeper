## Summary

Keep open. Current main still creates the backup archive through an adjacent random `.tmp` file and only removes it during normal JavaScript unwinding; there is no backup-scoped SIGINT/SIGTERM, AbortSignal, or timeout cleanup path for an externally interrupted `openclaw backup create`. The workaround comments are useful, but they confirm a manual cleanup mitigation rather than a shipped fix.

## What This Changes

The issue asks for `openclaw backup create` to clean up temporary archive files on timeout or interruption while preserving atomic successful publication.

## Best Possible Solution

Backup creation should own the active temporary archive lifecycle: track the exact temp archive path, clean it up on graceful termination or abort during the archive-write window, preserve no-overwrite final publication, and document the unavoidable crash/SIGKILL limits.
