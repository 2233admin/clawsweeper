## Summary

Keep open. Current main still has only narrow noise controls, while the reported broader confirmation spam remains unaddressed across direct skill/tool command replies, cron announce defaults, skill-install acknowledgements, and the public messages config surface.

## What This Changes

The issue requests a default-silent or configurable suppression/grouping policy for routine auto-generated chat confirmations such as cron creation, skill install, and generic done messages.

## Best Possible Solution

Define and implement a coherent notification/confirmation policy, likely under `messages` or `commands`, that suppresses or groups routine success confirmations across command dispatch, cron delivery, skill installs, and generic done replies while preserving explicit alerts and the existing narrow controls for exec, heartbeat, and cron delivery.
