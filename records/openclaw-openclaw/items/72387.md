## Summary

Keep this PR open. Current main still validates `agents.defaults.contextPruning.softTrimRatio` and `hardClearRatio` as 0..1, but `docs/gateway/config-agents.md` only shows example values and says ratios are character-based; it does not document the accepted range. The PR is a focused one-line docs update retargeted from closed #72280, and its diff has no security-sensitive surface.

## Best Possible Solution

Keep the PR open for maintainer review and merge the focused documentation update if docs checks pass. The best product path is to document the existing `contextPruning` ratio bounds in `docs/gateway/config-agents.md`; no core behavior or plugin API change is needed.
