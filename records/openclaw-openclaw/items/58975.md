## Summary

Keep PR #58975 open. The linked regression remains plausible on current main, and the PR targets the right area, but the latest PR context shows the patch is incomplete: current main has no shipped fix, and the PR's added return flag was reported as captured but not actually used to suppress the user-visible compaction side effects. The existing current-main tests also still exercise the old runMemoryFlushIfNeeded return shape, so the PR needs changes before it can land or be closed as implemented.

## Best Possible Solution

Keep the PR open and request changes. The best path is to preserve the focused bug-fix direction, but wire the memory-flush result into the actual compaction side-effect emission path, update every runMemoryFlushIfNeeded consumer for the new return contract, set performedMemoryFlush at the correct point in the flush lifecycle, and replace boolean-only tests with runner-path regression coverage that proves system events and verbose notices are suppressed for memory flushes without suppressing real foreground compaction context refresh.
