## Summary

Keep open. Current main still lacks the requested pre-RPC cron CLI warning for shell-looking text in main-session systemEvent jobs, and #63112 remains an open, unmerged implementation candidate that directly targets this behavior.

## What This Changes

Add a non-blocking stderr warning before cron add/edit submits a main-session systemEvent cron job whose payload text looks like a shell command.

## Best Possible Solution

Cron add and cron edit should share one small helper that detects shell-looking text only for resolved main-session systemEvent payloads, emits a non-blocking stderr warning before cron.add/cron.update, and covers warning and non-warning cases in focused CLI tests while preserving existing systemEvent delivery semantics.
