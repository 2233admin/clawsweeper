## Summary

Keep open. Current main has partial backend lineage metadata and session control primitives, but it still lacks the requested browser Control UI orchestration board: no dedicated route, no agent/session/subagent hierarchy rendering, no active-first status filters or virtualization, and the selected-row bulk UI only deletes sessions. The linked #54797 and #69364 reports are adjacent narrower/broader surfaces rather than replacements for this runtime subagent operations view.

## What This Changes

Request a scalable Control UI orchestration surface for multi-agent and subagent work, including hierarchy, active-first filters, bulk kill/steer/retry/cleanup actions, per-row runtime metadata, and large-session-count readability.

## Best Possible Solution

Keep this as the umbrella Control UI runtime orchestration request and split implementation into reviewed slices: first render existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions. Keep #54797 for narrow completed-session cleanup and #69364 for the separate durable independent-agent graph discussion.
