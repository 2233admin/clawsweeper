## Summary

Keep open. Current main has useful session/subagent metadata, chat slash-command controls, `/tasks` task-status primitives, and a flat Sessions table, but it still does not implement the requested Control UI orchestration board: no dedicated Mission Board/Orchestration route, no agent -> main session -> subagent/task hierarchy, no active/failed/completed orchestration filters, no virtualized large-run view, and no bulk kill/steer/retry/cleanup controls. Open #54797 is a narrower completed-subagent cleanup request and does not supersede this umbrella UI/product work.

## Best Possible Solution

Keep this issue open as the umbrella Control UI orchestration request. The best path is a dedicated Control UI mission/orchestration surface that consumes existing session hierarchy/status/model/runtime fields and task records where useful, renders a scalable active-first tree grouped by agent -> main session -> subagent/task, and wires scoped multi-select actions for abort/kill, steer, retry where semantics are explicit, and completed-session cleanup. Keep #54797 as a narrower cleanup follow-up, not as a replacement.
