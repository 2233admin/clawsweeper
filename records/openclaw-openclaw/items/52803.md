## Summary

Keep open. Current main still lacks the requested browser Control UI orchestration surface: session lineage and chat-side subagent controls exist, but the UI has no orchestration route, the Sessions tab is still a flat paginated table, and selected-row bulk actions only delete/archive sessions. The discussion also distinguishes #69364 and #54797 as adjacent, not replacements.

## What This Changes

Request to add a scalable Control UI orchestration view for agent/session/subagent hierarchy, active-first filtering, per-row run metadata, and bulk kill/steer/retry/cleanup operations.

## Best Possible Solution

Keep this as the umbrella Control UI orchestration request and split it into reviewed slices: first render existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add authorized bulk intervention and cleanup actions with clear confirmation and partial-failure behavior.
