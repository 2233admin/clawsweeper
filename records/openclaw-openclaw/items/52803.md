## Summary

Keep open. Current main has the backend/session primitives, chat slash-command controls, and a `/tasks` chat board that help with subagent visibility, but it still does not implement the requested Control UI orchestration board with hierarchy, active/failed/completed filters, scalable/virtualized rendering, or bulk kill/steer/retry/cleanup controls.

## Best Possible Solution

Keep this issue open as the umbrella product/UI request. The best implementation path is a dedicated Control UI orchestration surface that consumes existing sessions.list hierarchy/status/model/runtime fields plus task records where useful, renders a virtualized tree grouped by agent -> main session -> subagent/task, adds active/failed/completed and history controls, and wires scoped multi-select actions to chat.abort, sessions.steer or command equivalents, sessions.delete/task cancellation, and explicit retry semantics. Keep #54797 as a narrower cleanup follow-up, not as a replacement for this broader work.

## Risks / Open Questions

- The request is broad enough that maintainers may want to split it into separate UI hierarchy, backend query/aggregation, virtualization, and bulk-action follow-ups before implementation.
- Bulk kill/steer/retry/cleanup needs explicit authorization, scope, confirmation, and recovery semantics so a Control UI operator cannot accidentally interrupt unrelated sessions or lose useful transcripts.
- Retry semantics are not defined by the existing Sessions table; they likely need product decisions around whether retry means sessions.steer redirect, task requeue, subagent respawn, or transcript branch/replay.
