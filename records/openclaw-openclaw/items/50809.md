## Summary

Keep open. Current main still lacks the requested Android `sms.read` invoke command and lacks an SMS channel ingress path; the related implementation PR is closed unmerged, and the remaining work needs maintainer product/security decisions before implementation.

## What This Changes

The issue requests an Android `sms.read` invoke command plus a full SMS messaging channel that routes allowlisted inbound texts through a connected Android node and replies via SMS.

## Best Possible Solution

Keep the issue open and split the work into maintainer-approved slices: define a narrow, security-reviewed Android `sms.read` command contract first, then separately decide whether SMS ingress belongs in a bundled channel, a plugin-backed channel with any missing node-event seam, or outside OpenClaw core.
