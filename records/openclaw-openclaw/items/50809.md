## Summary

Keep open. Current main still lacks the requested Android `sms.read` invoke command and the Android-node-mediated SMS channel ingress path, and the linked implementation PR #50808 is closed unmerged. The work is real but crosses Android SMS permissions, node command policy, gateway event ingress, channel registration, and allowlist/privacy decisions, so it needs maintainer product/security review before an automated repair lane.

## What This Changes

Request to add an Android `sms.read` invoke command for ID/thread reads and make SMS a node-mediated messaging channel with allowlist-gated inbound texts and SMS replies.

## Best Possible Solution

Keep the issue open and split the path: first define a narrow, security-reviewed Android `sms.read` command contract, then separately decide whether Android-node SMS ingress belongs as a bundled channel, an external/plugin-backed channel with a generic node-event seam, or outside OpenClaw core.
