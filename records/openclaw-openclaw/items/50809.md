## Summary

Keep this issue open. Current main has Android node `sms.send` and `sms.search`, but it still lacks the requested `sms.read` command, id/threadId read semantics, Android inbound SMS receiver/event plumbing, and an SMS messaging channel registration. The linked implementation PR #50808 is closed unmerged, so it is prior context rather than a shipped fix.

## Best Possible Solution

Keep this issue open and split the work before implementation: first add a focused Android node read contract for `sms.read` with id/threadId semantics and tests, then separately design the SMS channel ingress path with RECEIVE_SMS/BroadcastReceiver forwarding, gateway `sms.received` handling, allowlist configuration, docs, and `.github/labeler.yml`/label updates for the new channel surface. If maintainers prefer external channel ownership, add the missing generic Android node/event APIs first so a plugin can implement the channel without core bypasses.
