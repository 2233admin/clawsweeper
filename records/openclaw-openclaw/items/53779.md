## Summary

Keep open: the underlying provider-isolation bug is real and is supported by the related report plus the April 11 confirmation comment, but this PR does not cover the strongest missing-apiKey PI semantic failure and introduces unsanitized config-derived warning output. It needs maintainer rework rather than conservative cleanup.

## What This Changes

The PR adds a config-validation pre-pass and tests so invalid `models.providers` entries are warned about and dropped while other providers continue validating.

## Best Possible Solution

Implement provider isolation at the models.json generation or runtime discovery boundary with PI-equivalent per-provider semantic validation. Valid providers should preserve `baseUrl`, auth, headers, models, overrides, and metadata; invalid providers should be skipped with sanitized diagnostics and regression coverage for mixed providers, all-invalid providers, missing-apiKey custom providers, invalid API enums, and generated implicit providers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Recover from PI semantic provider failures:** `src/config/validation.ts:251`
  - body: The pre-pass only calls `ModelProviderSchema.safeParse`, where `apiKey` is optional, and it runs before generated or merged providers are written to `models.json`. The related repro fails later in PI when a non-built-in provider has `models` but no `apiKey`, so this patch can still leave the bad provider in place and PI will reject the whole custom models file, dropping unrelated overrides.
  - confidence: 0.9
- **[P2] Sanitize provider warning text:** `src/config/validation.ts:264`
  - body: This warning interpolates the provider key and validation reason directly from user config into `console.warn`. Existing config diagnostics sanitize path and message text before terminal output; without that, crafted provider ids or issue text can inject newlines or terminal control characters into logs.
  - confidence: 0.88

## Security Review
