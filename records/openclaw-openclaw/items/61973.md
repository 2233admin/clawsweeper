## Summary

Keep this PR open for normal maintainer review. Current main does not implement `openclaw update review`, and the proposed feature is relevant to recent upgrade-safety discussions, but the PR still has concrete correctness issues in git-install state handling that should be fixed before merge.

## What This Changes

The PR adds a read-only `openclaw update review` subcommand, tests for its risk/recommendation states, CLI wiring, and update documentation.

## Best Possible Solution

Keep the feature work alive, but align `update review` with the updater's actual state model before merge: preserve git `behind: 0` as a known-current signal, scope downgrade assessment to package installs, rebase onto current update docs/help, and add regression tests for git-current plus registry-unavailable and git-local-newer cases.

## Reproduction Assessment

Yes. The first finding reproduces by mocking a git install with `fetchOk: true`, `behind: 0`, and an unavailable registry/dist-tag lookup; the PR returns `checkUnavailable: true` instead of a known-current git state. The second reproduces by mocking a git install where local `VERSION` compares newer than the npm target; the PR reports a downgrade even though current `openclaw update` only evaluates downgrade risk for package installs.

## Solution Assessment

No. The proposed command is a useful direction, but the current implementation is not the best mergeable solution because it models git update state through npm-target comparisons in cases where the real updater uses git state or package-only downgrade rules.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve known-current git status:** `src/cli/update-cli/review.ts:249-254`
  - body: For git installs, `behind: 0` is a meaningful result: fetch succeeded and the checkout is current. This code collapses that to `null`, so if the registry or dev dist-tag lookup is unavailable, `checkUnavailable` becomes true and the command tells users to review even though the git update check proved there is no git update. Keep a tri-state distinction between `0`, positive behind counts, and unknown.
  - confidence: 0.89
- **[P2] Scope downgrade detection to package installs:** `src/cli/update-cli/review.ts:260-261`
  - body: `isDowngrade` is computed from the npm target for every install kind, but current `openclaw update` only performs downgrade handling in the package-install path. A git checkout whose local `VERSION` is newer than the dev/beta dist-tag would be reported as a downgrade even though the git updater would fetch/rebase/checkout git refs instead. Gate this branch to package installs or mirror the updater's mode-specific target logic.
  - confidence: 0.87
