## Summary

Keep this issue open. Current main already blocks Telegram groups that are not allowed by `channels.telegram.groups`/`groupPolicy`, but it only skips/logs the update. There is no Telegram config, schema/type surface, docs, or runtime path for `unknownGroupAction`, warning cooldowns, or `leaveChat` auto-leave behavior.

## Best Possible Solution

Keep this open for a Telegram-plugin-owned implementation or maintainer-approved design. The likely shape is an opt-in `unknownGroupAction: "ignore" | "warn" | "leave"` plus optional message/cooldown fields on the Telegram account config, enforced in the existing preflight skip path for `group-chat-not-allowed` before assistant execution, with account-aware cooldowns, `bot.api.sendMessage`/`bot.api.leaveChat` handling, config schema/types/UI hints/docs, and targeted Telegram group-access tests.
