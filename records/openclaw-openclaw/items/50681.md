## Summary

Keep open. Current main still only denies unknown Telegram groups through the existing allowlist path; it does not provide the requested configurable ignore, warn, cooldown, or leave behavior, and the open paired PR has not merged.

## What This Changes

Request Telegram fail-closed handling for unknown groups with configurable ignore, warning, cooldown, and optional auto-leave behavior before assistant execution.

## Best Possible Solution

Implement this in the bundled Telegram plugin preflight with public config schema/types/docs, shared unknown-group handling for normal inbound messages and native commands, topic-preserving warnings, cooldown tests, and explicit coverage that approved groups continue to run normally.
