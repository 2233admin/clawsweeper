## Summary

Keep open. Current main still rejects unknown Telegram groups only by dropping/logging normal messages or sending a static native-command denial; it does not provide the requested configurable ignore, warn, cooldown, or auto-leave behavior, and open PR #50682 remains the paired implementation path.

## What This Changes

Add Telegram unknown-group handling that can ignore, warn, or leave unapproved groups before assistant execution, with optional warning text and cooldown config.

## Best Possible Solution

Land PR #50682 or a narrow successor that implements the unknown-group action in Telegram preflight, updates public config types/schema/generated metadata/docs, shares behavior across normal messages and native commands, and validates cooldown plus forum-topic delivery before this issue is closed.
