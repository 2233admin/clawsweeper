## Summary

Keep this PR open. Current main still does not implement the central Telegram behavior: `/model` in a Telegram topic/thread only updates the active session, there are no parent-chat future-thread default fields, and new Telegram thread sessions are not seeded from a parent default. The PR contains relevant implementation work, but the branch is broad and still has unresolved functional and security-sensitive review concerns that need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open, but ask for a scoped revision or replacement focused on Telegram model inheritance. The implementation should persist a same-chat parent future-thread default from both directive-only and inline Telegram `/model` paths, seed only genuinely new Telegram topic/DM-thread sessions from that default, prevent cross-chat leakage for main-scoped DM keys, handle `direct_messages_topic` payloads if needed, and cover the edge cases with focused tests. Split memory, gateway, recovery-script, and optional live E2E tooling into separate maintainer-reviewed PRs.
