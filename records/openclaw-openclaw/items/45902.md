## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: a Telegram `/model` change in one topic/thread is not persisted as a parent-chat default for future Telegram threads. The PR contains relevant implementation work, but the supplied diff and review discussion show it is too broad and still needs scoping and bug fixes before it is mergeable.

## Best Possible Solution

Keep this PR open, but request a scoped revision or replacement focused on Telegram thread model inheritance. The best path is to persist a same-chat parent future-thread default from Telegram `/model`, seed only new Telegram topic/DM-thread sessions from that default, cover directive-only and inline/native command paths, handle pre-created metadata entries and main-scoped DM keys without cross-chat leakage, and split the memory, gateway, and optional live E2E tooling into separately reviewed changes.
