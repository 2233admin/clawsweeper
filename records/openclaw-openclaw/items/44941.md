## Summary

Keep open. Current main has fallback transition tracking, lifecycle events, verbose-mode reply notices, /status visibility, and Control UI indicators, but the requested default-on per-session/topic activation and recovery notifications are still absent when verbose mode is off and there is no dedicated disable/config knob.

## What This Changes

Request default-on per-session/topic notifications when model fallback activates and when the primary model recovers, with a config option to disable them.

## Best Possible Solution

Define a fallback notification policy separate from verbose mode: default-on activation and recovery notices delivered to the affected session/topic, a documented disable knob, privacy-safe reason formatting, throttling/coalescing rules, and tests proving verbose-off chat delivery plus one-time recovery clearing.
