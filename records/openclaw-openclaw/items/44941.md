## Summary

Keep open. Current main has adjacent fallback state, lifecycle events, and verbose-mode fallback/cleared notices, but it does not satisfy the central request for default-on per-session/per-topic user notifications with a disable switch; the recovery half also overlaps the same author’s still-open #44935.

## What This Changes

Request default-on per-session/per-topic chat notifications when model fallback activates and when the primary model recovers, with configuration to disable them.

## Best Possible Solution

Define a maintainer-approved fallback notification policy that reuses the existing fallback transition state and lifecycle events to emit one activation and one recovery notice to the affected session/topic, with dedupe/throttling, documentation, tests, and an explicit disable setting.
