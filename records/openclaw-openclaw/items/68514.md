## Summary

Keep open. Current main still sends Feishu post `md` payloads with the original `messageText` unchanged, so the URL-with-underscore behavior this PR targets is not implemented. The PR is a focused Feishu plugin fix with tests and no supply-chain-sensitive file changes, but it still has an unresolved review concern around masking existing markdown links with nested brackets. Related #41880 is open for a narrower URL-only variant, so maintainers should consolidate or choose the best implementation rather than close this PR now.

## Best Possible Solution

Keep this PR open. The best path is to resolve the nested-bracket review concern, compare it with #41880 and the linked Feishu URL-only issue, then land one plugin-local Feishu post rendering fix with regression coverage for embedded URLs, already-linked markdown, code regions, punctuation, and parentheses.
