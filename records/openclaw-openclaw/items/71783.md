## Summary

Keep open. Current main still leaves the reported metadata-free, different-name QMD conflict unresolved: verified path/pattern conflicts can be rebound, but name-only conflicts still fall through to a generic skipped-add warning while searches continue to target the configured managed collection name.

## What This Changes

The issue requests safer boot-time handling for QMD collection conflicts when an existing same-path/pattern collection has a different name than the configured managed collection.

## Best Possible Solution

Keep this open until a merged memory-core change either safely reconciles verified same-path/pattern different-name conflicts or surfaces a targeted manual repair hint for unverified name-only conflicts, with regression coverage that prevents destructive removal without metadata.

## Reproduction Assessment

Yes. The issue provides concrete QMD stderr for the `life` versus `life-main` conflict, and current tests model the metadata-free add-conflict path that still produces the skipped-add warning.

## Solution Assessment

No for current main. The narrow maintainable direction is metadata-gated automatic rebind plus a clear manual repair hint for unverified conflicts; current main still leaves the user with a generic warning and later searches against the missing configured collection.
