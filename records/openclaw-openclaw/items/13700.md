## Summary

Keep open. Current main has adjacent compaction checkpoint branch/restore and native subagent transcript fork support, but it still does not provide the requested named `/session save` and `/session load` workflow, reusable session templates, or restore/import from `.reset` and `.deleted` transcript archives; the recent maintainer comment also explicitly kept this issue open for design follow-up.

## What This Changes

The issue requests native session snapshots with explicit save/load, restore, and branching semantics for long-running OpenClaw sessions.

## Best Possible Solution

Define a native session snapshot design that builds on SessionManager, session store metadata, compaction checkpoint artifacts, reset/deleted transcript archives, and subagent fork support, then add coherent save/list/load/branch semantics across slash commands, Gateway protocol, CLI/Control UI where appropriate, docs, and focused tests.
