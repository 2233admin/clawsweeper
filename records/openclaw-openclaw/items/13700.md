## Summary

Keep open. Current main has adjacent compaction checkpoint branch/restore and native subagent transcript-fork primitives, but it still does not provide the requested named `/session save` and `/session load` workflow, reusable session templates, or restore/import from `.reset` and `.deleted` transcript archives; the recent maintainer comment also explicitly kept the item open for design follow-up.

## What This Changes

The issue requests named session snapshots with save/load/fork semantics so users can restore or branch a conversation context checkpoint.

## Best Possible Solution

Define a native session snapshot design that builds on SessionManager, session store metadata, compaction checkpoint artifacts, reset/deleted archive handling, and subagent fork support, then add save/list/load/branch semantics across slash commands, Gateway protocol, CLI/UI where appropriate, docs, and focused tests.
