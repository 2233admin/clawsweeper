## Summary

Keep open. Current main has adjacent compaction checkpoint branch/restore and subagent fork primitives, but it still does not provide the requested named `/session save` and `/session load` workflow, reusable session templates, or restore/import from `.reset` and `.deleted` transcript archives; the 2026-04-26 maintainer comment also explicitly kept this open.

## What This Changes

Add explicit session snapshot commands and supporting restore/fork semantics so users can save, load, branch, and reuse conversation context checkpoints.

## Best Possible Solution

Keep this open for maintainer design. The best path is a native session snapshot design that builds on SessionManager, the session store, compaction checkpoint metadata, reset/deleted archive handling, and subagent fork primitives, then defines save/list/load/branch semantics plus slash, Gateway, CLI, docs, and focused tests.
