## Summary

Current main has adjacent pieces, including native sub-agent `context: "fork"` and Gateway compaction checkpoint branch/restore RPCs, but it does not implement the requested user-facing `/session save` and `/session load` workflow, named reusable snapshots, or general restore from `.reset`/`.deleted` archives. The issue should stay open for maintainer product/design follow-up.

## Best Possible Solution

Keep this open and define a native session snapshot design that builds on the existing SessionManager/compaction-branch primitives: add explicit save/list/load/branch semantics, user-facing slash/CLI or Gateway surfaces, docs, and tests, while deciding whether `.reset`/`.deleted` archive restore and prompt-cache-preserving sub-agent forks are in or out of scope for the first PR.
