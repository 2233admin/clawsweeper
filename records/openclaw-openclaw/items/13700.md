## Summary

Keep open. Current main has useful adjacent primitives for compaction checkpoint branch/restore and native subagent transcript forks, but it still does not provide the requested named `/session save` and `/session load` workflow, reusable session templates, or restore/import from `.reset` and `.deleted` transcript archives. The issue discussion also expanded the use case beyond compaction recovery into context-poison rollback, prompt-cache-sensitive forks, batch reusable snapshots, and restart recovery, and a maintainer comment already kept this open for design follow-up.

## What This Changes

The issue requests explicit session snapshots that can save, load, restore, and fork a session's messages, context state, model choice, and checkpoint data.

## Best Possible Solution

Define a native session snapshot design on top of SessionManager, session store metadata, compaction checkpoint artifacts, reset/deleted transcript archives, and subagent fork support, then add coherent save/list/load/branch semantics across slash commands, Gateway protocol, CLI/Control UI where appropriate, docs, and focused tests.
