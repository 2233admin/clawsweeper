## Summary

Keep open: current main still preserves `memorySearch.outputDimensionality` through config resolution but the OpenAI embedding adapter does not place it into inline OpenAI request bodies, batch JSONL request bodies, or provider cache identity. The issue is also paired with open, unmerged PR #55507 using closing syntax, so the issue should remain open until that PR or a replacement lands.

## What This Changes

The issue asks for `memorySearch.outputDimensionality` to be passed to OpenAI embeddings as `dimensions` so reduced-dimensional OpenAI memory vectors work.

## Best Possible Solution

Land an updated #55507 or replacement patch that threads `memorySearch.outputDimensionality` into the OpenAI embedding client, sends OpenAI `dimensions` for both inline and batch `/v1/embeddings` calls when configured, includes the dimension in provider cache/reindex identity, and adds regression tests for all three paths.
