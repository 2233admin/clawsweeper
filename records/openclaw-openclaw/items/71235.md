## Summary

Keep open for maintainer review. Current main does not already provide the requested `python` shim, and the PR branch now applies a narrow conditional alias for images that already have `python3`; the separate question of shipping `python3` in the default gateway image remains tracked by the related open issue.

## What This Changes

The PR adds a conditional `python` -> resolved `python3` symlink to the gateway and sandbox Dockerfiles when `python3` is present and no `python` command is already on PATH.

## Best Possible Solution

Merge or request small scope adjustments for this narrow alias if maintainers want `python` to work wherever the shipped gateway/sandbox images already contain `python3`; keep the separate default-gateway `python3` installation decision in #75041.

## Reproduction Assessment

Yes. Build a current sandbox image and compare `python3 --version` with `python --version`; source shows `python3` is installed in the sandbox images while current main has no unversioned shim, and the PR adds that shim conditionally.

## Solution Assessment

Yes for the narrow alias problem: resolving the target with `command -v python3` and skipping when any `python` command already exists is the smallest maintainable Dockerfile change. It is not the complete solution for images that lack `python3` entirely, which should stay separate.
