## Summary

Keep this issue open. Current main still allows same-session heartbeat runs to use `agents.defaults.heartbeat.model`, persist the actual runtime `modelProvider`/`model` snapshot into the session store, and let `session_status` treat that persisted runtime identity as the selected card model when no explicit session override exists. The docs acknowledge the heartbeat-model bleed workaround with `isolatedSession`, and related PR #51706 remains open/unmerged, so this is not implemented or safely superseded.

## Best Possible Solution

Keep this issue open. The best fix is to preserve explicit user/session model selection as the selected model and treat heartbeat/internal runtime identity as display-only metadata, or prevent heartbeat/internal runs from overwriting the user-facing runtime snapshot in shared sessions. The fix should coordinate with #51706, preserve ordinary runtime-model and fallback visibility, and add focused regression coverage for same-session heartbeat model overrides with no session override and empty fallback lists.
