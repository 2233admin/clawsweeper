## Summary

Keep this PR open. Current main still does not implement the intended `modelContextWindowOverrides` Amazon Bedrock discovery config surface: runtime discovery only uses the built-in known-model table or `defaultContextWindow`, the plugin/config schemas do not expose the key, and docs/tests do not cover regex overrides. The linked bug #73328 remains open, and competing PR #73382 proposes a different family/table fallback, so this needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review alongside #73328 and #73382. The best path is to choose one canonical Bedrock context-window policy, then align runtime lookup, cache-key determinism, plugin manifest schema/UI hints, config type/schema compatibility, Bedrock docs, and focused discovery tests, with an explicit regex-safety decision if configurable patterns are accepted.
