## Summary

Keep this PR open. Current main does not implement the PR's intended `modelContextWindowOverrides` Bedrock discovery config surface: runtime discovery still falls back from `KNOWN_CONTEXT_WINDOWS` to `defaultContextWindow`, the plugin manifest and core config schemas do not include the new key, and the Bedrock docs/tests only cover defaults and known-model lookup. The linked #73328 remains open, and #73382 is a competing open approach, so this needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review alongside #73328 and #73382. If maintainers choose the configurable override policy, the implementation should align runtime lookup, deterministic cache keys, plugin manifest schema/ui hints, config type/schema compatibility, Bedrock docs, and focused discovery tests, with an explicit decision on regex safety.
