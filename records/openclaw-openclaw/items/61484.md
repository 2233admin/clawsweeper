## Summary

Keep this issue open. Current main still exposes `sessions_send` as a text-only agent tool, while the lower Gateway `sessions.send` path can already carry generic attachments, so the remaining gap is the agent/session-tool media contract and A2A/native delivery behavior.

## What This Changes

Request `sessions_send` support for image/media transfer between agents, including local media inputs and native channel media delivery where available.

## Best Possible Solution

Define a media-capable `sessions_send` contract that reuses existing chat attachment normalization and media safety limits, preserves inter-session provenance/redaction, and extends A2A announce delivery to carry structured media through the existing channel send path when supported.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: on current main the agent tool schema and execution path accept only `message`, while the Gateway RPC beneath it already forwards optional attachments.

## Solution Assessment

Unclear as an implementation plan. The requested surface is valid, but the safest solution should be structured attachments/media refs through existing Gateway media handling rather than channel-specific bypasses or unrestricted local path reads.
