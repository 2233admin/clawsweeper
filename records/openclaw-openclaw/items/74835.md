## Summary

Keep open. Current main supports per-provider model request overrides, including explicit proxy and `allowPrivateNetwork`, but there is no schema, type, docs, or runtime merge path for a global/default model-provider request policy. The related #74809 top-level proxy bug and #56535 Fake-IP SSRF report reinforce that this is a real operator problem, but they do not supersede this narrower config-default request.

## What This Changes

Request a global/default model-provider request policy so proxy and `allowPrivateNetwork` settings can be configured once, inherited by remote model providers, overridden per provider, and inspected effectively.

## Best Possible Solution

Design a security-reviewed model-provider request-default layer, likely under the `models` config contract, that merges before per-provider `request` overrides, is visible in effective config, has schema/help/docs/secret-redaction coverage, and clearly scopes private-network allowance or trusted-proxy behavior so local and self-hosted providers can opt out safely.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: configure only a proposed global default and omit `models.providers.<id>.request`; the strict model config schema has no global request default field, and the runtime reads only `providerConfig?.request` for provider request policy.

## Solution Assessment

Unclear. The requested direction is plausible and narrower than relying on the currently problematic top-level proxy, but the exact config shape and SSRF inheritance semantics need maintainer review before this is a safe implementation path.
