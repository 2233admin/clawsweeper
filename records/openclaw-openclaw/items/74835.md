## Summary

Keep open. Current main still supports provider request proxy and private-network SSRF overrides only per model provider; there is no schema, type, runtime merge path, or docs surface for a global/default model-provider request policy. The related proxy and Fake-IP reports support the operator problem but do not supersede this narrower config-default request.

## What This Changes

The issue requests a global/default model-provider request policy so proxy routing and `allowPrivateNetwork` can be configured once and overridden per provider.

## Best Possible Solution

Design a security-reviewed `models`-level provider request-default layer that merges before per-provider `request` overrides, is visible in effective config, preserves explicit per-provider opt-out, handles local/self-hosted providers deliberately, and updates schema/help/docs/redaction plus focused config and resolver tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure only a proposed global default such as `models.defaults.request` and omit `models.providers.<id>.request`; the strict schema has no such field and runtime resolution reads only the selected provider request block.

## Solution Assessment

Unclear. A model-provider request-default layer is a plausible narrow direction, but the safe solution depends on maintainer decisions for config shape, inheritance scope, private-network opt-in semantics, and audit visibility.
