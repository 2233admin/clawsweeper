## Summary

Keep open: current main still lacks a global/default model-provider request policy; only per-provider request blocks are accepted and merged. The related Fake-IP/top-level proxy reports and the shipped per-provider opt-in are useful operator evidence, but they do not supersede this narrower ergonomics and security-design request.

## What This Changes

The issue requests a global/default model-provider request policy so proxy routing and private-network SSRF settings can be configured once and inherited by providers unless overridden.

## Best Possible Solution

Design a models-scoped provider request defaults layer that merges before models.providers.<id>.request, preserves per-provider overrides, handles local/self-hosted providers explicitly, appears in effective config inspection, and includes SSRF threat-model docs plus schema, redaction, and runtime tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Configure one of the proposed global default paths and omit models.providers.<id>.request; the strict schema has no accepted default field, and the runtime only reads the selected provider's request block.

## Solution Assessment

Is this the best way to solve the issue? Unclear. A global defaults layer is plausible and narrower than relying on process-wide proxy routing, but the config shape and private-network inheritance semantics need maintainer/security design before implementation is the best fix.
