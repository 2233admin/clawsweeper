## Summary

Keep open. Current main still only supports model-provider request proxy and private-network SSRF settings on each `models.providers.<id>.request` block; there is no global/default model-provider request policy in schema, types, docs, or the runtime merge path. The related Fake-IP and top-level proxy reports are useful context, but they do not supersede this narrower config-default request.

## What This Changes

Request a global/default model-provider request policy so proxy and `allowPrivateNetwork` settings can be configured once and overridden per provider.

## Best Possible Solution

Design a security-reviewed model-provider request-default layer, probably under the `models` config contract, that merges before per-provider `request` overrides, remains visible in effective config, preserves explicit provider opt-out, and updates schema/help/docs/redaction/tests together.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: configure only a proposed global default such as `models.defaults.request`, omit `models.providers.<id>.request`, and current strict schema/runtime have no field or merge path that would apply it.

## Solution Assessment

Unclear. The requested direction is plausible and narrower than relying on the currently problematic top-level proxy path, but the exact config shape and SSRF inheritance semantics need maintainer security/product review before implementation.
