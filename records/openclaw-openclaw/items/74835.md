## Summary

Keep open. Current main still only supports per-provider model request overrides, while this asks for a global/default provider request policy with override semantics and effective-config visibility. The issue is valid but security-sensitive, so the config shape and SSRF inheritance rules need maintainer design before implementation.

## What This Changes

Add a global/default model-provider request policy so proxy routing and private-network allowance can be configured once and inherited by providers unless overridden.

## Best Possible Solution

Add a security-reviewed model-provider request-default layer, likely under the `models` config contract, that merges before per-provider `request` overrides, remains visible in effective config, preserves explicit per-provider opt-out, and keeps local/self-hosted provider behavior deliberate.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: configure only a proposed global default and omit `models.providers.<id>.request`; the strict model config schema has no global request-default field, and the runtime reads only the selected provider's `request` block.

## Solution Assessment

Unclear. The requested direction is plausible and narrower than relying on the problematic process-wide proxy, but the exact config shape and SSRF inheritance semantics should be chosen by maintainers before a fix PR implements it.
