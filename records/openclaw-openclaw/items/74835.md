## Summary

Keep open. Current main still exposes model-provider request proxy/TLS/auth and allowPrivateNetwork only under each models.providers.<id>.request block; the schema, types, docs, and runtime do not provide or merge a global/default provider request policy. The request is valid, but the global SSRF/proxy inheritance semantics need maintainer/security design before implementation.

## What This Changes

The issue requests a global/default request policy for model providers so proxy routing and private-network SSRF settings can be configured once and inherited by providers unless overridden.

## Best Possible Solution

Design a models-scoped provider request defaults layer that merges before models.providers.<id>.request, preserves per-provider overrides, handles local/self-hosted providers explicitly, appears in effective config inspection, and includes SSRF threat-model docs plus schema/redaction/runtime tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure one of the proposed global default paths and omit models.providers.<id>.request; the strict schema has no accepted default field and the runtime only reads the selected provider's request block.

## Solution Assessment

Unclear. A global defaults layer is a plausible direction and narrower than relying on process-wide proxy routing, but the config shape and private-network inheritance semantics are security-sensitive and need maintainer/security design first.
