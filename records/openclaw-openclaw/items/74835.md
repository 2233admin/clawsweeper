## Summary

Keep open. Current main still exposes model-provider request proxy/TLS/auth and `allowPrivateNetwork` only under individual `models.providers.<id>.request` entries, and neither the schema nor runtime merges a global/default request policy into provider calls.

## What This Changes

Request to add a global/default model-provider request policy so operators can configure proxy routing and fake-IP/private-network allowance once while preserving per-provider overrides.

## Best Possible Solution

Keep this open for maintainer/security design of a `models`-scoped provider request defaults layer that merges before `models.providers.<id>.request`, preserves per-provider override and opt-out behavior, handles local providers explicitly, appears in effective config inspection, and ships with schema, redaction, runtime, and docs coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure one of the proposed default paths and omit `models.providers.<id>.request`; the strict schema has no accepted default field, and current model resolution only reads the selected provider's request block.

## Solution Assessment

Unclear. The direction is plausible and narrower than relying on the process-wide proxy path, but the config shape and global private-network inheritance semantics need maintainer/security design before implementation is the best fix.
