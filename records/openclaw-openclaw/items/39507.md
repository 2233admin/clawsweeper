## Summary

Keep this PR open. Current `main` still does not implement the intended behavior: in normal Feishu group sessions, inbound `thread_id` or `root_id` still sets `threadReply: true`, and the reply dispatcher still turns that into `reply_in_thread` delivery even when `replyInThread` is configured as `"disabled"`. The PR is stale against the current file split and overlaps with open PR #40147, but it remains a live Feishu behavior decision rather than something safely closeable as implemented or obsolete.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Maintainers should decide whether `replyInThread: "disabled"` is a hard off switch in normal groups. If yes, port the change into `extensions/feishu/src/bot-content.ts`, preserve topic-session anchoring for `group_topic` and `group_topic_sender`, and update the regression test that currently expects forced thread replies. If the existing follow-thread behavior is intended, update schema/docs/test wording and close this PR with that explicit product decision, coordinated with #40147.
