## Summary

Keep this PR open. Current main still forces Feishu thread replies from inbound `thread_id` or `root_id` even when `replyInThread` is configured as `"disabled"`, and a current regression test explicitly expects that opposite behavior. The submitted PR is stale against the current file split and overlaps with #40147, so this needs maintainer product judgment rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Maintainers should decide whether `replyInThread: "disabled"` is a hard off switch in normal Feishu groups. If yes, port the fix into `extensions/feishu/src/bot-content.ts`, preserve topic-session anchoring for `group_topic` and `group_topic_sender`, and update the current regression test. If the existing follow-thread behavior is intended, update schema/docs/test wording and close this PR with that explicit decision, coordinated with #40147.
