## Summary

Close this PR. Current main already fixes the linked Copilot replay failure with a narrower shipped implementation: Copilot reasoning IDs are preserved instead of replaced, encrypted reasoning payloads are now round-tripped through stored signatures, and the linked bug was closed after a live Copilot matrix confirmed the remaining synthetic function-call ID path succeeds once the reasoning ID is original. The PR's broader delete-all-ID approach is obsolete, and its decode-only encrypted-content change was incomplete compared with current main.

## What This Changes

The PR changes the GitHub Copilot OpenAI Responses payload sanitizer to delete opaque replay item IDs for message, function_call, and reasoning items, and adds reasoning replay tests for encrypted content and summary preservation.

## Best Possible Solution

Keep the shipped current-main behavior: preserve provider-issued Copilot reasoning IDs, keep non-reasoning normalization bounded to the verified-safe path, and round-trip encrypted reasoning payloads through the Responses replay signature. Reopen a narrower follow-up only if a live Copilot failure shows that message or function-call IDs also need deletion.

## Reproduction Assessment

Yes. The linked bug includes concrete Copilot tool-call replay steps and a curl wire-format matrix, and the maintainer comment on #72602 live-verified the failure mode and current-main success path against Copilot `/responses`.

## Solution Assessment

No for this PR. The current main solution is narrower and better supported: it preserves the reasoning ID that Copilot validates, keeps the verified-safe function-call normalization, and separately fixes encrypted reasoning encode/decode; deleting every opaque ID is broader than the proven failure requires.
