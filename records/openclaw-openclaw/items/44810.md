## Summary

Keep open. Current main still has the reported user-visible failure mode: ACP child runs can emit the generic no-output/interactive-input stall notice, and the parent relay still does not distinguish prompt-submitted/no-first-output/network failure states. Adjacent ACPX adapter pinning and proxy-routing work helps related startup/network setups, but it does not implement the requested stall classification or diagnostics.

## Best Possible Solution

Keep this issue open. The best fix is a narrow ACP event/relay change: forward prompt/runtime status events with the active run id, track prompt-submitted and first-assistant-output state, change the watchdog copy for post-prompt silence to a runtime/network/no-response classification, add redacted proxy-env and stream-log diagnostics, and cover it with targeted relay, dispatch, and ACPX-runtime tests. PR #44949 is the closest prior implementation candidate and should be reviewed, revived, or replaced by a smaller patch on current main.
