## Summary

Current main still has the reported path: unset Telegram `reactionLevel` resolves to `minimal`, the Telegram plugin exposes that as prompt `reactionGuidance`, and the shared system prompt renders the exact MINIMAL reaction sentence. This remains a valid current behavior/design issue rather than an implemented fix, stale report, or full duplicate of #9835.

## What This Changes

The issue asks OpenClaw to make Telegram reaction prompt defaults explicit and stable so GPT-backed Telegram sessions do not appear to switch to MINIMAL reaction mode without a user config change.

## Best Possible Solution

Keep this issue open for maintainer direction on Telegram reaction default semantics. The best path is to make prompt-affecting reaction guidance explicit and stable, either by adding a documented `reactionGuidance` contract that composes with `reactionLevel`, or by adding an upgrade/migration/doctor path that pins existing unset installs so live prompts do not appear to switch modes without user intent. Runtime code, tests, generated config metadata, TypeScript comments, and public Telegram docs should be updated together.
