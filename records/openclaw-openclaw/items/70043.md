## Summary

Current main still has the reported behavior path: unset Telegram `reactionLevel` resolves to `minimal`, Telegram exposes that as prompt `reactionGuidance`, and the shared system prompt emits the exact `Reactions are enabled for Telegram in MINIMAL mode.` text. This is not stale or already fixed; it needs maintainer/product direction because docs, type comments, and existing-session stability are still out of alignment.

## Best Possible Solution

Keep this issue open for maintainer direction on Telegram reaction default semantics. The best path is to make prompt-affecting reaction guidance explicit and stable: either introduce a separate documented `reactionGuidance` config/default contract, or add an upgrade/migration/doctor path that pins existing unset installs so live prompts do not appear to switch modes without user intent. The chosen behavior should update Telegram runtime code, tests, generated config metadata, TypeScript comments, and the public Telegram docs together.
