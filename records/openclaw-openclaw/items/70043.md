## Summary

Keep open. Current main still resolves unset Telegram `reactionLevel` to `minimal`, exposes that value as Telegram prompt reaction guidance, and emits the reported `MINIMAL mode` prompt text during embedded runs; the related open enhancement #9835 is broader and does not supersede this defaulting/stability bug.

## What This Changes

The issue reports that GPT-backed Telegram DM sessions can start showing MINIMAL reaction guidance in the live prompt even though the user did not explicitly set Telegram reaction configuration.

## Best Possible Solution

Make Telegram reaction prompt behavior explicit and stable: either split prompt guidance into a documented config/default contract that composes with `reactionLevel`, or add an upgrade/doctor path that pins existing unset installs so live prompts do not appear to change without user intent. Keep runtime code, tests, generated config metadata, TypeScript comments, and public Telegram docs aligned.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: use Telegram config with no `reactionLevel`, resolve Telegram reaction guidance, then build the system prompt; the source path and existing tests show it reaches the exact `MINIMAL mode` prompt text.

## Solution Assessment

No. The current behavior is internally explainable, but it is not the best user-facing contract because a prompt-affecting default can surface as a silent mode change and the docs/type comments disagree about the default.
