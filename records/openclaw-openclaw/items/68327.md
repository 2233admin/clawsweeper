## Summary

Keep this PR open. Current main has absorbed or independently implemented some adjacent startup work, but it does not fully implement the PR's central secrets-runtime shortcut for the gateway-auth SecretRef preflight path. The PR also still needs maintainer review because its branch includes unguarded production profiling logs and broad behavior changes outside the narrow latency fix.

## Best Possible Solution

Keep the PR open, but narrow it before merge: implement the secrets-runtime optimization against current main with a regression test proving gateway auth SecretRef preflight does not load external provider plugins; remove or gate the profiling logs behind the existing startup trace mechanism; split or justify the async-lock timeout and Windows mDNS policy changes separately; then validate the gateway/secrets/scripts touched surface.

## Risks / Open Questions

- The PR mixes the targeted secrets-runtime latency fix with unrelated startup profiling, async-lock timeout, Windows mDNS policy, and UI spawn changes, which raises review and regression risk.
- The PR adds a 30s default timeout to createAsyncLock, a shared infra helper used by multiple pairing/push/write paths, without current-main evidence that every lock holder is safe under that default.
- The Windows mDNS default-disable change affects documented Bonjour discovery behavior and likely needs product/docs/test follow-through if kept.
