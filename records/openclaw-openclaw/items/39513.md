## Summary

Keep this PR open. Current main still has the behavior the PR documents: main-session cron/system-event wakes use the specialized cron-event prompt and do not include the default heartbeat instruction to read HEARTBEAT.md, while the current cron docs and AGENTS.md template still lack the proposed workaround guidance. The PR is docs/comment-only and does not introduce an observable security or supply-chain risk; the remaining question is maintainer review of the wording and whether the AGENTS template text should be direct agent-facing guidance.

## What This Changes

The PR adds cron documentation, AGENTS.md template guidance, and a source comment explaining that main-session cron system events use the cron-event prompt without the default HEARTBEAT.md instruction and showing an AGENTS.md workaround.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent docs-only patch. The best path is to document that main-session cron system events use a self-contained cron-event prompt without the default HEARTBEAT.md instruction, add the workaround to docs/automation/cron-jobs.md, and make docs/reference/templates/AGENTS.md contain direct agent-facing cron handling guidance. Runtime behavior changes should be tracked separately.
