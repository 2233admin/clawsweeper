## Summary

Keep open. Current main still treats CLI silence as a no-output watchdog failure, including the claude-cli live-session path, and I found no Monitor or pending-tool expected-idle exemption. The related cap-raising work in #40982/#41022 may reduce false positives but does not clearly solve the reported Monitor-specific session loss.

## What This Changes

The issue asks for the CLI no-output watchdog to stop killing otherwise healthy agent sessions that are idle while waiting for a long-running Monitor/background task.

## Best Possible Solution

Add a tested expected-idle contract for CLI runs waiting on pending tool/background work: suspend or renew the no-output watchdog only while that state is active, preserve the watchdog for truly silent stalled processes, and keep cap/config tuning as a fallback rather than the only protection.

## Reproduction Assessment

Yes. The issue provides concrete logs and a Monitor wait example, and current-main source/tests show a silent CLI path still becomes a no-output watchdog timeout with the same failure text.

## Solution Assessment

Unclear. The requested direction is valid, but the best implementation needs maintainer choice between a Monitor-state-aware idle signal, a heartbeat from the wrapper, and broader timeout cap/config tuning.
