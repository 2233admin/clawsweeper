## Summary

Keep open. Current main still applies the CLI no-output watchdog to silent Claude CLI live-session turns, including turns that may be legitimately idle while a native Monitor task is pending; the related open cap-raising PR is only a mitigation and does not add Monitor-aware expected-idle handling.

## What This Changes

The issue asks OpenClaw to stop the CLI no-output watchdog from killing a Claude CLI session that is silent because a native Monitor task is pending.

## Best Possible Solution

Add a bounded proof-of-life or expected-idle contract for Claude CLI native Monitor waits so healthy long-running background work does not kill the supervising session, while keeping no-output and overall timeout protection for genuinely stalled backends.

## Reproduction Assessment

Yes. The issue gives a concrete Monitor wait plus watchdog log sequence, and current-main source still shows silent Claude CLI live-session turns hitting the no-output timeout path without a Monitor-aware exemption.

## Solution Assessment

Unclear. Raising watchdog caps can reduce false kills, but the most maintainable fix is likely a semantic Monitor/pending-tool expected-idle contract or bounded heartbeat after the Claude stream-json event shape is verified.
