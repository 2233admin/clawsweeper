## Summary

Keep this issue open. Current main has automatic Discord typing plumbing for inbound/heartbeat-style delivery, but it still does not expose a `typing` or `typing-indicator` action through the shared `message` tool, CLI action enum, target-mode map, or Discord message-action discovery/dispatcher. The related implementation attempt #30535 was closed unmerged, so no merged fix resolves the remaining manual message-tool request.

## Best Possible Solution

Keep this open until maintainers either add a focused shared `typing`/`typing-indicator` message-tool action for Discord, or explicitly document that automatic inbound/component/heartbeat typing is the intended OpenClaw boundary. A complete narrow fix should wire the shared action name, target mode, Discord discovery, Discord dispatcher, existing `sendTypingDiscord` helper, CLI/tool docs, changelog, and targeted tests without changing automatic typing lifecycle behavior or unrelated providers.
