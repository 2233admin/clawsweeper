## Summary

Keep this issue open. Current main has automatic Discord typing plumbing for inbound replies, component replies, and heartbeat delivery targets, but it still does not expose a `typing` or `typing-indicator` action through the shared `message` tool, CLI action enum, target-mode map, or Discord message-action discovery/dispatcher. A prior implementation attempt, #30535, was closed unmerged, so there is no merged candidate that resolves the remaining request.

## Best Possible Solution

Keep this open until maintainers either add a shared `typing` or `typing-indicator` message-tool action, or explicitly document that automatic inbound/component/heartbeat typing is the intended OpenClaw boundary. A complete fix likely touches the shared action enum, target-mode/action normalization, agent message-tool schema/description, CLI docs, Discord discovery and dispatch, mapping to `sendTypingDiscord`, and focused tests for rejection/dispatch/lifecycle behavior.
