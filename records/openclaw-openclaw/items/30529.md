## Summary

Keep this issue open. Current main has Discord typing infrastructure for automatic inbound replies and heartbeat runs, but it still does not expose the requested agent-callable Discord typing action through the shared `message` tool. The related comments point to broader cross-provider typing and previously closed lifecycle bugs, not to a canonical duplicate or shipped replacement for `message(action="typing", ...)`.

## Best Possible Solution

Keep the issue open until maintainers either add a shared `typing` or `typing-indicator` message-tool action, or explicitly decide automatic typing is sufficient. A concrete implementation should add the shared action name and schema fields, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to `sendTypingDiscord` with account and target resolution, define start/refresh/stop or one-shot semantics, add Discord regression coverage, and update the public message/typing docs. If cross-provider parity is desired, design the action so Telegram, Slack, Discord, and similar plugins can opt in through the channel plugin contract.
