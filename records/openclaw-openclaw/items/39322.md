## Summary

Keep this PR open. Current main at 28ff82dcdae7f7fb11491deefe48bd82b68eb0bb still does not implement legacy imsg tapback reaction forwarding: there is no includeReactions config/schema field, no reaction payload/parser, no reaction notification branch, and watch.subscribe still sends only attachments. The paired same-author feature issue #39031 remains open, and the related open #60677 is a separate tapback filtering repair rather than this forwarding feature.

## Best Possible Solution

Keep this PR open alongside #39031 for maintainer review. The best path is to rebase/refresh the implementation on current main, verify the imsg reaction notification and subscription contract, add includeReactions consistently across config types, schema, generated metadata, docs, and tests, and make an explicit decision for group reaction behavior under mention gating. If maintainers decide legacy imsg should not gain this feature, close or supersede both the PR and #39031 with that product decision.
