## Summary

Keep open. Current main still has no supported `models.providers.<id>.enabled` contract: the provider type and strict runtime schema omit the field, explicit providers are merged without an enabled gate, fallback/model catalog paths still read every configured provider, and `openclaw models status` has no disabled-provider output. The related implementation PR #25415 was closed unmerged, so this remains the active tracker.

## Best Possible Solution

Keep this issue open until OpenClaw adds a documented `models.providers.<provider>.enabled?: boolean` contract that defaults to enabled, preserves disabled config blocks, excludes disabled providers from generated `models.json`, configured model catalogs, fallback candidate selection, and status/probe paths, and shows disabled state in `openclaw models status` rich and JSON output. Reuse #25415 as implementation context, but rebase or reimplement it against current main with focused tests and docs.
