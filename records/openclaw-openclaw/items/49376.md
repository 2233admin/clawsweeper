## Summary

Keep this issue open. Current main still has transient error classification, auth-profile rotation/fallback controls, and cron retry prior art, but it does not expose a configurable main LLM retry policy with retry classes, per-attempt backoff, max attempts, max wall-clock cap, or retry notification. Open PR #49800 also uses closing syntax for this issue and remains unmerged, so the issue should not be cleaned up before that implementation path is reviewed or closed.

## Best Possible Solution

Keep this issue open as the product tracker for configurable main LLM request retry. Maintainers should review whether to land a focused version of #49800, fold the broader work from #38640 into a first-class design, or add an `agents.defaults.llmRetry`/global policy that reuses existing transient classification while avoiding auth, billing, format, and quota/rate-limit retries unless explicitly chosen.
