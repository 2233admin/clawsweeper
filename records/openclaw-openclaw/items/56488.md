## Summary

Keep open. Current main still matches the reported scheduler gap: the completed one-shot guard exists, but only startup catch-up enables it. The normal timer path and `runDueJobs` still collect due jobs without `skipAtIfAlreadyRan`, and PR #56509 is an open implementation candidate that explicitly targets this issue.

## Best Possible Solution

Review and land a focused patch equivalent to PR #56509, or close that PR if maintainers choose a different implementation. The code change should enable `skipAtIfAlreadyRan` for normal timer and due-run collection while preserving transient retry semantics, with regression coverage for completed `kind: "at"` jobs that retain a past `nextRunAtMs`. Close this issue only after the implementation merges.
