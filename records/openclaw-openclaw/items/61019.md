## Summary

Keep this issue open. Current main still treats `openclaw models set` as a config-only default update; existing session rows and active runtime state are not migrated or refreshed automatically. Manual reset/patch paths can recompute or clear session model state, but that does not implement the reported active-session inheritance behavior.

## Best Possible Solution

Keep the issue open and implement a deliberate session refresh or migration path for global default changes. A conservative fix would update `openclaw models set` or a follow-up command to refresh sessions that only carry stale runtime/default-derived model identity, while preserving `modelOverrideSource: "user"` and legacy explicit overrides unless the user opts into resetting them.
