## Summary

Keep this PR open. Current main still does not implement the PR’s intended normal-close race handling: `executeGatewayRequestWithScopes` awaits `client.request()`, only sets `ignoreClose` after the await resumes, and the `onClose` handler still rejects immediately when `settled` and `ignoreClose` are false. The PR remains a focused gateway bug-fix candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open as a focused gateway bug-fix candidate. Rebase it onto current main, adapt the request-in-flight code-1000 deferral to the current teardown flow, keep regression coverage for the normal-close-after-response race, and verify abnormal disconnects still report formatted gateway close diagnostics.
