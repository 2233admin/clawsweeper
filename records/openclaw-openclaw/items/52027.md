## Summary

Keep this PR open. Current main has related gateway teardown improvements, but it does not implement the PR’s central close-race behavior: a close event that arrives before the awaited request continuation sets `ignoreClose` still rejects immediately. The PR remains a focused implementation candidate, with a low security/supply-chain footprint based on the provided diff.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. The best path is to rebase it onto current main, preserve the existing `stopGatewayClient()` teardown ordering, add or port the close-race regression test, and verify that abnormal disconnects still report `formatGatewayCloseError(...)` details.
