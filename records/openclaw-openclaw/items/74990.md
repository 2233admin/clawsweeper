## Summary

Keep this PR open. The Anthropic onboarding goal is valid and current main still has the older Claude CLI label/manual-auth flow, but the proposed branch needs fixes before merge: it invokes the interactive `/login` slash command as a shell argument instead of the documented auth subcommand, and the new changelog entry misses the repository-required contributor credit.

## What This Changes

The PR promotes the Anthropic Claude CLI auth choice as a Claude subscription path, adds a helper that detects or installs Claude Code and launches login, updates Anthropic docs and metadata, and adds/updates tests for the new onboarding flow.

## Best Possible Solution

Keep the PR open and amend it to use the supported Claude auth command for the wizard login subprocess, align all user-facing docs/prompts with that command, add the required changelog credit, then run the targeted Anthropic tests plus the changed gate before maintainer review.

## Reproduction Assessment

Yes. The problematic path is statically reproducible when the CLI is installed but `readClaudeCliCredentialsForSetup()` returns no credential: the PR prompts for sign-in and then spawns `claude /login` before checking credentials again.

## Solution Assessment

No, not as written. The overall direction is maintainable for the Anthropic plugin, but the login step should use the documented auth subcommand and the release note must follow the repo changelog contract.
