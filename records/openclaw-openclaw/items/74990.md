## Summary

Keep this PR open. Current main does not already implement the requested subscription-focused Anthropic onboarding flow, but the proposed patch is not merge-ready because the new install/login gate bypasses the existing configured Claude CLI command contract and can mis-handle hosts that use an absolute or custom Claude Code binary path.

## What This Changes

The PR relabels Anthropic Claude CLI onboarding as a Claude subscription path, adds prompted Claude Code detect/install/login handling, and updates Anthropic docs, changelog, metadata, and tests.

## Best Possible Solution

Keep the Anthropic-owned onboarding direction, but amend the implementation so the wizard resolves the effective claude-cli backend command from ctx.config, uses that command for detection and login, offers npm global install only for the default PATH command, and covers the custom absolute-command case with a regression test.

## Reproduction Assessment

Yes. Configure agents.defaults.cliBackends.claude-cli.command to an absolute Claude Code binary and run onboarding on a host where plain claude is not on PATH; the PR probes claude --version before using that config, so it prompts to install or fails incorrectly.

## Solution Assessment

No. The plugin-local subscription onboarding is the right product boundary, but the current patch is not the best implementation because it bypasses the established CLI backend command override instead of reusing it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Respect configured Claude CLI command:** `extensions/anthropic/cli-install.ts:40`
  - body: The new install gate probes `claude --version`, and the login helper later spawns the same hard-coded `claude` binary. OpenClaw already supports `agents.defaults.cliBackends.claude-cli.command` for hosts where Claude Code lives at an absolute or custom path; with that valid config and no `claude` on PATH, onboarding now prompts to install or fails instead of using the configured binary. Resolve the effective command from `ctx.config` for detect/login, and reserve npm install for the default PATH command.
  - confidence: 0.9
