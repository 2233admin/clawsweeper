## Summary

Keep open. Current main does not already implement the requested Anthropic onboarding change, and the PR head still has a blocking P2 regression: the new install/login gate hard-codes the PATH binary `claude` instead of honoring the existing configured Claude CLI backend command. The author addressed earlier `/login` and changelog feedback, but this branch still needs amendment and is reported dirty/not rebaseable.

## What This Changes

The PR relabels Anthropic Claude CLI auth as a Claude subscription path, adds prompted Claude Code detect/install/login handling to onboarding, updates provider metadata/docs/changelog, and adds tests.

## Best Possible Solution

Keep the subscription-first Anthropic onboarding direction, but amend the PR so detection and login use the effective configured `claude-cli` command, reserve `npm install -g @anthropic-ai/claude-code` for the default PATH command case, add a regression test for a configured absolute command with no `claude` on PATH, then rebase and re-run the changed-surface checks.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary and ensure `claude` is not on PATH; the PR still probes `claude --version` and runs `claude auth login` before the migration can rely on the configured backend command.

## Solution Assessment

No. The plugin-owned onboarding direction is appropriate, but the current implementation is not the best fix because it bypasses the existing CLI backend configuration contract instead of reusing the effective command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Respect configured Claude CLI command:** `extensions/anthropic/cli-install.ts:40`
  - body: The new install/login gate always probes and runs `claude`, but OpenClaw supports `agents.defaults.cliBackends.claude-cli.command` for hosts where Claude Code lives at an absolute or custom path. With that documented config and no `claude` on PATH, onboarding now prompts to install or fails instead of using the configured binary. Resolve the effective backend command from `ctx.config` for detect/login, and reserve npm install for the default PATH command.
