## Summary

Keep this PR open. The Claude subscription onboarding direction is useful and active, but latest head still has a blocking regression: the new install/login helper hard-codes `claude` from PATH and bypasses the documented configured-command contract for `claude-cli` users.

## What This Changes

This PR renames the Anthropic CLI onboarding auth choice as a Claude subscription path, adds an interactive Claude Code install/sign-in helper with tests, and updates Anthropic provider metadata, docs, and changelog text.

## Best Possible Solution

Amend this PR so the onboarding helper resolves the configured `claude-cli` backend command from `ctx.config` and uses it for detection, login prompts, and login execution; only offer npm global install for the default PATH command, and add a regression test for a configured absolute command with no `claude` on PATH.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary while `claude` is absent from PATH; the PR calls `ensureClaudeCliInstalled` before reading credentials and `detectClaudeCli` probes only `claude --version`.

## Solution Assessment

No. The plugin-owned subscription onboarding direction is appropriate, and the earlier login-command/changelog fixes are good, but the current patch is not the narrowest maintainable solution because it bypasses the existing custom CLI backend command contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Respect configured Claude CLI command:** `extensions/anthropic/cli-install.ts:40`
  - body: The new install/login gate always spawns `claude` from PATH before setup reads credentials, but OpenClaw supports `agents.defaults.cliBackends.claude-cli.command` for hosts where Claude Code is installed at an absolute/custom path. With that documented config and no `claude` on PATH, onboarding now prompts to install or fails instead of using the configured binary. Resolve the configured backend command from `ctx.config` for detect/login, and reserve npm install for the default PATH command.
