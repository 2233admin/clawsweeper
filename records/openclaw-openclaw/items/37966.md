## Summary

Keep this issue open. Current main still does not honor explicit `cacheRetention`/`cacheControlTtl` for the bundled LiteLLM OpenAI-compatible Claude route: the resolver excludes `openai-completions`, the payload-level `cache_control` injection remains OpenRouter-only, and the linked fix PR #38221 is closed unmerged and incomplete for this path.

## What This Changes

Honor explicit prompt-cache retention for LiteLLM-proxied Anthropic models, including payload-level Anthropic `cache_control` markers on the bundled LiteLLM OpenAI-compatible route.

## Best Possible Solution

Implement a narrow LiteLLM Anthropic prompt-cache pass-through: when the bundled LiteLLM `openai-completions` route is using a Claude/Anthropic-looking model and explicit `cacheRetention` or `cacheControlTtl` resolves to `short` or `long`, inject Anthropic `cache_control` markers before the LiteLLM proxy request. Do not inject for `none`, missing explicit config, non-Claude LiteLLM models, or arbitrary custom OpenAI-compatible providers; add targeted tests, update LiteLLM prompt-caching docs, and add a changelog entry with reporter/contributor credit.
