## Summary

Keep open. Current main and the latest release still use the Bookworm Docker base, so the requested Trixie migration has not landed. The PR remains a useful active candidate for the newer glibc baseline request, but it is not merge-ready because the head still has a failing digest test, user-facing Docker docs still describe Bookworm, and the branch needs maintainer review after recent Docker hardening landed on main.

## What This Changes

This PR migrates the main Dockerfile, smoke/e2e Dockerfiles, OCI base-image labels/cache IDs, and Dockerfile tests from pinned `node:24-bookworm`/`bookworm-slim` images to pinned `node:24-trixie`/`trixie-slim` images.

## Best Possible Solution

Land a rebased Trixie migration only after preserving current-main Docker hardening, fixing the digest ARG/test contract, updating `docs/install/docker.md` and the changelog for the base-image/build-ARG/cache-ID changes, validating the pinned image digests, and running the Docker/check lanes including an optional Docker CLI apt-install build smoke.
