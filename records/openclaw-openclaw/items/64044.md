## Summary

Keep open. Current main and the latest release still use Debian 12/bookworm, so this PR is not obsolete, but the PR head is not merge-ready: it is dirty against current main, has a concrete failing Dockerfile test from a missing digest ARG contract, and leaves public Docker docs on the old Bookworm baseline. Related #69107 was already closed in favor of this PR, so this remains the active migration candidate rather than a cleanup close.

## What This Changes

The PR migrates the main Dockerfile, smoke/e2e Dockerfiles, OCI labels/cache IDs, and Dockerfile tests from `node:24-bookworm`/`bookworm-slim` to pinned `node:24-trixie`/`trixie-slim` images.

## Best Possible Solution

Land a rebased Debian 13/Trixie migration only after preserving current-main Docker hardening, making the digest ARG contract consistent, updating public Docker docs and migration notes for the base image/build-ARG/cache-ID changes, verifying the pinned image digests plus optional Docker CLI apt behavior, and rerunning the Docker/check lanes; otherwise explicitly document that newer glibc requirements in derived images are outside the supported Docker baseline.
