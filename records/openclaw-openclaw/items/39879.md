## Summary

Keep this issue open. Current main still does not add WhatsApp outbound mention metadata: text sends remain `{ text }`, media sends only use captions, the typed send options cannot carry mentions, and the outbound adapter only wires quote metadata. Open PR #56863 is the related implementation vehicle, but it is still open and unmerged.

## Best Possible Solution

Keep this issue open until the bundled WhatsApp plugin ships native outbound mention support. A focused fix should add a typed outbound mention path, safely extract or accept `@+<digits>` / `@<digits>` mentions, resolve group or LID participant mappings where feasible, pass Baileys mention metadata for text and supported media captions including the separate visible text sent after voice notes, and add focused unit coverage. Review or narrow PR #56863 first because it already attempts this feature area.
