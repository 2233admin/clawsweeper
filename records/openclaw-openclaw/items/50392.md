## Summary

Keep open. Current main still does not implement the PR’s intended `/bash` policy inheritance: `handleBashChatCommand` forwards only timeout and notification fields into `createExecTool`, while `security`, `ask`, `pathPrepend`, safe-bin policy, and `approvalRunningNoticeMs` remain absent. The PR is a focused code/test change, not a duplicate or already-shipped fix.

## Best Possible Solution

Keep this PR open and either update it or apply an equivalent current-main patch that forwards the relevant `tools.exec` policy fields from `handleBashChatCommand` into `createExecTool`, including `approvalRunningNoticeMs` and likely `strictInlineEval`, while retaining existing `commands.bash` and elevated allowlist gates. Add a focused regression test for `/bash` policy inheritance, then run the targeted test and changed gate through the normal OpenClaw validation path.
