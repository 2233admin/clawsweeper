## Summary

Keep this PR open. Current main still does not implement the requested per-invocation `--skip-bootstrap` flag for `openclaw agents add`; it only supports the global `agents.defaults.skipBootstrap` setting and `openclaw onboard --skip-bootstrap`. The PR is a relevant implementation candidate, but it needs revision before merge because the reviewed diff includes `skipBootstrap` in the `hasExplicitOptions` watchlist, which would make `openclaw agents add <name> --skip-bootstrap` force non-interactive mode instead of allowing the wizard path.

## Best Possible Solution

Keep the PR open and ask for a focused revision: add the `--skip-bootstrap` option and `AgentsAddOptions.skipBootstrap`, OR it with `agents.defaults.skipBootstrap` at both `ensureWorkspaceAndSessions` call sites, do not count it as a non-interactive trigger, add a CLI-level test for `openclaw agents add <name> --skip-bootstrap` preserving the wizard path, keep the non-interactive forwarding tests, update the `agents add` docs, and remove unrelated test churn before maintainer review.
