## Summary

Keep this PR open. Current main still does not implement the requested per-invocation `--skip-bootstrap` flag for `openclaw agents add`, and the PR remains a relevant implementation candidate, but it needs revision for the documented interactive-mode regression and unrelated test churn.

## Best Possible Solution

Keep this PR open and request a focused revision: add `--skip-bootstrap` to `agents add`, add `AgentsAddOptions.skipBootstrap`, OR it with `agents.defaults.skipBootstrap` at both workspace setup call sites, keep it out of explicit non-interactive flag detection, add CLI-level coverage proving `openclaw agents add <name> --skip-bootstrap` can still use the wizard path, keep non-interactive forwarding coverage, update `docs/cli/agents.md` / the public CLI docs, and remove or split the unrelated image-generation test churn.
