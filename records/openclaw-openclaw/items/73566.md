## What This Changes

The PR adds a typed `PromptInputClosedError`, makes `promptYesNo()` reject on readline close before an answer, catches that case in `plugins uninstall` with exit-1 `--force` guidance, and adds prompt/uninstall regression tests.

## Best Possible Solution

Land this focused EOF-aware prompt handling after maintainer review, preserving piped answers and `--force`, and add the required Unreleased Fixes changelog line before merge if maintainers enforce the user-facing fix policy.

## Reproduction Assessment

Yes. Current main reaches a direct `readline.question()` await in `plugins uninstall`, and a Node 22 closed-stdin smoke reproduces exit 13; the PR-equivalent helper rejects with `PromptInputClosedError` and still accepts piped `y`.

## Solution Assessment

Yes. The shared close-aware prompt primitive plus uninstall-specific error handling is the narrow maintainable fix for the crash without banning non-TTY piped answers; broader prompt UX consistency can be handled separately.
