## Summary

Keep this PR open. Current main still has the reported closed-stdin gap: `promptYesNo()` directly awaits `readline.question()`, and `plugins uninstall` awaits that prompt without handling readline close/EOF. The PR is a focused implementation candidate for open bug #73562, with related context from #73551 explaining how users can reach the uninstall cleanup path.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to review and, if accepted, land the focused EOF-aware prompt handling plus regression tests, then let it close #73562. Keep #73551 separate unless a broader enable/disable validation PR explicitly resolves that stale-entry bug too.
