## Summary

Keep open. Current main still lacks EOF/close handling in the shared prompt helper and `plugins uninstall` still awaits the prompt without converting closed stdin into an actionable exit-1 error. The PR remains a focused implementation candidate for the same-author open bug #73562, with a narrow diff and no observed supply-chain or security-sensitive file changes.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, land the EOF-aware `promptYesNo()` handling, uninstall catch/exit-1 message, and focused regression tests so #73562 closes after merge; keep #73551 separate unless a broader plugin enable/disable validation fix explicitly resolves it.
