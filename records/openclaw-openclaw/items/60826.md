## Summary

Current main still persists raw CLI argv into config-audit records. The write audit builder, async/sync observe paths, and observe-recovery builder all copy `process.argv.slice(0, 8)` into records, and the append helpers serialize records directly without redaction. The related fix PRs #60871 and #70468 are closed unmerged, so they remain useful references but do not resolve this issue.

## Best Possible Solution

Keep this issue open. The best fix is to centralize config-audit process-info sanitization, or omit argv entirely, before any write, observe, or recovery audit record is appended. Reuse the existing redaction infrastructure where it fits, add OpenClaw-specific CLI flag coverage, and add focused tests for config write plus observe/recovery audit records.
