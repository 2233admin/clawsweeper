## Summary

Keep open. Current main still has the Nextcloud Talk length-mismatch early return and manual XOR comparison, while this PR's latest head implements the intended timing-safe comparison and is an active security-hardening PR with maintainer activity. It should not be closed or replaced by a separate automated repair job; it needs normal maintainer review, duplicate-test cleanup, and final CI/gate proof.

## What This Changes

The PR updates Nextcloud Talk webhook signature verification to use padded `crypto.timingSafeEqual`, adds signature/header regression coverage, and adds a changelog entry.

## Best Possible Solution

Land the Nextcloud Talk signature hardening after removing the duplicate test copies, preserving the mismatch-length invariant test, and completing the targeted Nextcloud Talk test plus the changed gate. Using `safeEqualSecret` from `plugin-sdk/security-runtime` would be the cleanest consistency pass, but the current padded `timingSafeEqual` production logic is a valid narrow fix.

## Reproduction Assessment

Yes. On current main, any non-empty supplied signature whose length differs from the SHA-256 hex digest returns at `extensions/nextcloud-talk/src/signature.ts:28` before any byte comparison; this is reproducible by direct source inspection and the existing short-signature test only checks the boolean result.

## Solution Assessment

Yes for the security behavior. The PR's padded `timingSafeEqual` call followed by an original-length gate is the narrow maintainable fix and matches the existing LINE/shared-helper pattern; the safer cleanup is to remove duplicate tests and optionally call `safeEqualSecret` instead of repeating the helper logic.
