## Summary

Current main has sub-agent concurrency configuration, queueing, and per-session child-limit rejection, but it does not expose global sub-agent lane capacity or remaining slots through `session_status`, a `sessions_capacity` tool, or a structured `capacity_exceeded` response. The related #57053 remains open but covers session time/token budget, not this specific sub-agent concurrency budget.

## Best Possible Solution

Keep this open and implement a small core orchestration surface, preferably a structured `sessions_capacity` or `session_status.details.subagents` object with `concurrency_limit`, active/queued sub-agent lane counts, per-requester `maxChildrenPerAgent` usage, and `remaining`. Back it with a read-only command-queue lane snapshot helper plus focused tests for `session_status`/new tool output and docs under the public sub-agents/session-tools pages.
