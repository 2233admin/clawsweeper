## Summary

Keep this PR open. Current main still has only the Kustomize/raw-manifest Kubernetes deployment path, while this PR proposes a new Helm chart surface that is not implemented elsewhere and has unresolved chart-correctness and deployment-security review comments. This is plausible contributor work, but it needs maintainer/product review and revisions rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to revise the chart so it preserves the current Kubernetes object/data contract, uses current image/version policy, adds render-time guards and rollout annotations, documents loopback exposure clearly, handles secrets safely, and adds Helm validation. If maintainers do not want Helm maintained in core, that should be an explicit product decision rather than an automated cleanup close.
