## Summary

Keep open. Current main does not implement the PR’s requested Chutes static catalog addition: the Chutes catalog, plugin manifest, and provider docs still list GLM-4.7/GLM-5 but no `zai-org/GLM-5.1-TEE`, while the discovery path still falls back to that static catalog when `/models` fails. The PR is a focused bundled Chutes provider update, and the earlier default-model bump noted by review was reverted, but it likely needs matching manifest/docs mirror updates before merge.

## Best Possible Solution

Keep this PR open for normal review. The best path is to add `zai-org/GLM-5.1-TEE` to the bundled Chutes static catalog while keeping `CHUTES_DEFAULT_MODEL_ID` unchanged, then update any mirrored plugin manifest/docs catalog surfaces and add a focused assertion that the fallback catalog includes the new id.
