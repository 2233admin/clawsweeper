## Summary

Current main still does not implement this PR’s intended behavior. The payload warning policy has no targeted suppression for `gateway` `config.schema.lookup` path misses, and the mutation classifier still treats `config.schema.lookup` as mutating because it checks the full compound action against a read-only allowlist that does not include `lookup`.

## Best Possible Solution

Keep this PR open as an implementation candidate, but coordinate it with #42898 or fold both fixes together. The best fix is to classify compound read-only gateway actions by their leaf verb, suppress only the exact `gateway` `config.schema.lookup` + `config schema path not found` benign miss, and add regression coverage for the full runtime path plus inverse cases proving other gateway failures still warn.
