## Summary

Keep this PR open. Current main still does not implement the intended suppression, and the PR as submitted is incomplete because the runtime still classifies `gateway` action `config.schema.lookup` as mutating before payload rendering.

## Best Possible Solution

Keep this PR open as an implementation candidate, but fold it together with #42898 or update it to include both halves: classify compound read-only gateway actions by leaf verb, suppress only the exact `gateway` `config.schema.lookup` + `config schema path not found` benign miss, and add regression coverage for the full runtime path plus inverse cases for mutating or unrelated gateway failures.
