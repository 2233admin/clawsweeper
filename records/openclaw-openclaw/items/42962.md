## Summary

Keep this PR open. Current main still lacks the targeted gateway schema lookup warning suppression, and the PR remains an incomplete implementation candidate because current runtime mutation classification still marks `gateway` action `config.schema.lookup` as mutating before payload rendering. The linked bug report #42856 is still open, and the related open PR #42898 covers the missing classification half rather than making this PR obsolete by itself.

## Best Possible Solution

Keep this PR open as a candidate, but update it or fold it together with #42898 so both required pieces land together: classify compound read-only gateway actions by leaf verb, suppress only the exact benign `gateway` `config.schema.lookup` + `config schema path not found` miss, and add regression coverage for the full runtime path plus inverse cases.
