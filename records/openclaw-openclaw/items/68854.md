## Summary

Keep this PR open. Current main still has the broad heredoc approval gate in gateway allowlist mode, so the requested regression fix is not implemented on main; however, this PR head is not merge-ready because it weakens the stated unquoted `<<-` heredoc approval behavior and adds tests that reference an undefined `warnings` binding.

## What This Changes

The PR changes gateway exec allowlist heredoc handling so quoted and backslash-quoted heredocs avoid the extra approval prompt while unquoted heredocs keep a warning and explicit approval path, with added gateway tests and an updated warning string in a subscribe-handler test.

## Best Possible Solution

Keep the PR open and request a narrow rework that derives heredoc approval from the parser’s heredoc classification or a shared helper, preserves auto-approval for quoted and backslash-quoted literal heredocs, still treats unquoted `<<EOF` and `<<-EOF` as approval-required when policy says so, and fixes the added tests to capture warnings directly.
