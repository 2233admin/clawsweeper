## Summary

Keep this issue open. Current main still routes `zai/glm-4.6v` image understanding through the generic OpenAI-compatible streaming image path using the configured ZAI provider base URL; I did not find a merged ZAI vision-specific endpoint/payload/abort fix, and the related #70915 route PR remains only an open candidate for a related 404 variant.

## Best Possible Solution

Keep this issue open. The best fix is to add a focused ZAI vision-model regression around `zai/glm-4.6v` through the image tool with the reported endpoint setup, then either patch the ZAI media-understanding provider locally or add a generic provider-facing image payload/endpoint seam if multiple OpenAI-compatible providers need it. Review #70915 as a related route candidate, but do not close this issue until the abort path is proven fixed on main.
