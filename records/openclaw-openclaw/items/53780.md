## What This Changes

## Best Possible Solution

Land a rebased, reviewed media-specific sanitizer that maps MediaFetchError failures to one stable channel-safe message at outbound action and hook boundaries, preserves existing diagnostics for non-media failures, uses cycle-safe cause traversal, and covers attachment hydration, broadcast results, and message_sent failures in tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

  - body: The new fallback converts every non-Error delivery failure into "unknown error" before emitting message_sent. Current main preserves thrown strings and plain objects through formatErrorMessage/String-style handling, so keep the media redaction but preserve the existing non-media formatting path.
  - confidence: 0.9
- **[P2] Preserve non-Error broadcast diagnostics:** `src/infra/outbound/message-action-runner.ts:348-349`
  - body: The broadcast failure path has the same regression: non-media thrown strings or objects now become "unknown error" in the per-target result payload. Only replace the text when the caught error is a media fetch failure.
  - confidence: 0.88
- **[P2] Guard cause-chain traversal against cycles:** `src/media/fetch.ts:40-41`
  - body: isMediaFetchError is called on arbitrary caught outbound errors, but it recursively follows Error.cause without tracking seen values. A cyclic cause chain would overflow before the original failure can be handled; use an iterative traversal with a seen set.
  - confidence: 0.79
