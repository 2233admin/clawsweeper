## Summary

Keep open. Current main still lacks the requested guarantees: startup catch-up can replay due or missed cron work from persisted state without a current contact-window policy, cron removal and stop do not abort active executions, and gateway shutdown does not expose an emergency outbound halt. The linked related work is adjacent or still open rather than a merged fix.

## What This Changes

The issue asks OpenClaw cron-scheduled WhatsApp deliveries to respect configured daytime windows after reconnect/restart and to provide stop/delete/emergency controls that immediately halt active outbound sends.

## Best Possible Solution

Keep this open and fix it in cron/gateway core with an explicit missed-run policy, active execution cancellation, and operator emergency controls. The desired end state is that reconnect/startup replay cannot violate contact windows, cron remove/disable and gateway stop abort matching active runs where possible, and operators have a clear emergency outbound halt with regression coverage for the America/Guatemala out-of-hours scenario.
