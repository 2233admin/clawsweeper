## Summary

Keep this PR open. Current main still drops the alternate single-media aliases on the loose outbound final-reply path, while the shared message-send action accepts those aliases, so the branch is addressing a real parity gap. It is not implemented on main, but it still needs maintainer review on fallback precedence and whether `fileUrl` should be included before merge.

## What This Changes

The PR changes plugin SDK outbound reply normalization and tests so loose final-reply payloads can resolve alternate single-media fields into `mediaUrl`.
