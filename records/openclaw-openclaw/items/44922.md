## Summary

Keep open. Current main still matches the reported cron-to-heartbeat prompt duplication path, and open PR #44973 remains the focused implementation candidate rather than a reason to close this issue before it lands.

## What This Changes

The issue asks OpenClaw to suppress the redundant scheduled-reminder heartbeat wrapper for main-session cron `systemEvent` jobs with `delivery.mode: "none"` while keeping the raw system event and delivery semantics intact.

## Best Possible Solution

Land a focused cron/heartbeat fix that marks main-session cron `systemEvent` entries as already transcript-injected and excludes only those tagged entries from cron reminder prompt construction, with regression coverage proving the raw system event remains visible and generic cron reminders still work.
