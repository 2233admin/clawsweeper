## Summary

Keep open: current main still appears to send the same main-session cron systemEvent text through both the queued `System:` prompt path and the cron heartbeat wrapper path. The existing open PR #44973 is the right implementation candidate to review instead of closing this issue or queueing a second repair.

## What This Changes

Respect `delivery.mode: "none"` for main-session cron `systemEvent` jobs by suppressing the redundant scheduled-reminder heartbeat wrapper when the event has already been injected into the session.

## Best Possible Solution

Keep this issue open while #44973, or an equivalent core fix, is reviewed and validated. The end state should distinguish main-session cron system events already injected into the session from cron events that still need a heartbeat wrapper, with focused regression coverage and unchanged outbound delivery semantics.
