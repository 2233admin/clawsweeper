## Summary

Keep open. The report is fresh, independently confirmed on v2026.4.27, and current main still has the same Active Memory watchdog signal wired through the embedded run and tool-abort path; current main adds partial timeout recovery but does not prove the embedded memory tool aborts, completed NONE classification, or prompt-error leakage are fixed.

## What This Changes

Fix Active Memory embedded recall so memory_search calls are not aborted prematurely, completed NONE recalls are not misclassified as timeouts, and stale prompt-error context does not leak into later recall runs.

## Best Possible Solution

Keep the Active Memory hard deadline, but separate deadline enforcement from ordinary memory tool execution until the deadline actually expires; treat a completed NONE recall as a clean empty result, and ensure prior prompt-error custom entries cannot appear in fresh Active Memory recall prompts.
