## Summary

Keep open. The report is fresh, has persisted-transcript evidence plus an independent confirmation, and current main still contains the matching Active Memory watchdog, embedded-runner abort propagation, timeout classification, and prompt-error custom-entry surfaces.

## What This Changes

Active Memory should not abort embedded `memory_search` calls or classify a completed embedded recall as timeout, and prior `prompt-error` entries should not leak into the next recall context.

## Best Possible Solution

Keep Active Memory bounded with an outer hard deadline, but avoid exposing the recall deadline as an already-aborted tool-dispatch signal before the deadline actually fires; classify a completed `NONE` recall as `empty`; filter prior `openclaw:prompt-error` custom entries from new LLM-boundary context; and add focused regression coverage for all three behaviors.

## Repair Work Prompt

Fix the Active Memory embedded recall bug reported in https://github.com/openclaw/openclaw/issues/74586. Observable behavior: inside the Active Memory embedded sub-agent, `memory_search` can return `AbortError: Aborted` while direct `memory_search` works; the embedded run can complete with `NONE` but Active Memory still reports `status=timeout`; prior `openclaw:prompt-error` custom entries can leak into the next recall context. Keep the fix scoped to Active Memory and the embedded-runner abort/prompt lifecycle. Preserve the hard Active Memory deadline, but do not let the recall deadline pre-abort ordinary memory tool dispatch before the deadline has actually won; classify a completed `NONE`/empty recall as `empty`, not `timeout`; and filter or otherwise prevent prior `openclaw:prompt-error` custom entries from reaching new LLM-boundary prompt context. Add focused regression tests in `extensions/active-memory/index.test.ts` and the nearest embedded-runner/abort tests. If `src/agents/pi-tools.abort.ts` or tool abort wrapping changes, include the before-tool-call integration coverage. Add a user-facing `CHANGELOG.md` fix entry crediting the reporter and independent confirmer, but do not credit bot or maintainer handles. Do not broaden provider timeout policy, do not change memory-core search semantics, and do not move plugin-owned Active Memory behavior into generic core unless a small generic seam is required.
