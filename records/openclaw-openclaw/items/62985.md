## Summary

Keep open. Current main still rejects Telegram account-level `agentId` in strict account config, and #63380 remains open, unmerged, and explicitly paired as the implementation candidate for this issue.

## What This Changes

The issue requests fixing a regression where Telegram multi-account configs with `channels.telegram.accounts.<id>.agentId` fail config validation after upgrading from 2026.4.5 to 2026.4.8.

## Best Possible Solution

A durable resolution should either restore backward-compatible `channels.telegram.accounts.<id>.agentId` with routing semantics and regression coverage, or deliberately keep `bindings[]` as the only supported account-routing path and improve upgrade docs or doctor output for this migration.
