## Summary

Keep open. Current main still lacks the durable post-reset startup window and queued-owner-message recovery steer that this PR is trying to add, and the related broader #53524 is still open rather than shipped. The PR should be reviewed or folded into a combined Discord thread/channel fix, not closed as implemented.

## Best Possible Solution

Keep this PR open for maintainer review or fold it into a combined successor with #53524: add a durable post-reset startup window for Discord thread and channel resets, wrap queued owner steers with startup-recovery instructions, clear the window only after the intended recovery lifecycle is complete, and keep focused regression coverage for the thread race plus channel coverage.
