## Summary

Keep this PR open. Current main still does not contain the durable post-reset startup window, queued owner-message recovery steer, or focused regression coverage that this PR adds. The broader Discord channel/thread follow-up remains an open implementation candidate rather than shipped proof, and the PR still has review feedback to resolve before landing.

## Best Possible Solution

Keep this PR open for maintainer review, or supersede it with a combined implementation that covers Discord thread and channel resets. The fix should add a durable post-reset startup marker, wrap queued owner steers with startup-recovery instructions, clear the marker only after the intended recovery lifecycle succeeds, and keep focused regression tests for both surfaces.
