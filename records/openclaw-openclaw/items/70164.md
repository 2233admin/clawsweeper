## Summary

Keep open. The issue is recent, has concrete WebSocket logs, and the current main code still has the same observable failure mode when no assistant event reaches the Gateway chat buffer: lifecycle end can emit `state:"final"` with no message. Current tests cover normal assistant streaming/non-streaming paths, but they do not cover the reported MiniMax/tool-result path where OpenClaw logs show tool result -> lifecycle end -> chat final with no assistant output. The provided related items do not supersede this report.

## Best Possible Solution

Keep the issue open and reproduce on current main with the MiniMax model plus the reported tool/search flow. Capture raw provider/embedded runner events, `agent` events, transcript writes, and Gateway WS frames from tool result through lifecycle end. If the model returns final text but no assistant event is emitted, fix the embedded provider/message-end emission path and add a regression test for tool-result followed by non-streaming final assistant text. If OpenClaw emits assistant events but only the Netty bridge drops them, move the fix or documentation to that relay/client boundary.
