## Summary

Keep open. Current main still does not prove the MiniMax/tool-result WebSocket path emits visible assistant text before `chat` final, and the reporter's follow-up points at OpenClaw gateway/embedded streaming rather than only an external Java/Netty relay.

## What This Changes

Fix WebSocket chat output so MiniMax/tool-call runs do not skip assistant text events and jump directly to `state:"final"` without a final message.

## Best Possible Solution

Keep this open until maintainers capture a current-main MiniMax run with a tool call followed by expected visible assistant text, including raw provider/pi-agent-core events and gateway WebSocket events. If provider events contain text, fix the embedded subscription or gateway projection path and add fixture-backed regression coverage. If provider events contain no visible assistant text, fix the MiniMax stream/provider integration or document the upstream limitation with a narrower follow-up.
