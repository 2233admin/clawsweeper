## Summary

Keep this PR open. Current main still lacks the two central behaviors described by the PR: command delivery does not synthesize a user-visible fallback when raw payloads normalize to zero deliverable outbound payloads, and Slack preview/progress turns that end with no delivered reply still clear state and return silently. The related open #45290 is broader delivery-recovery work, but the provided context does not show it superseding this PR's specific normalized-empty command fallback or Slack preview-only no-final fallback.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped rebase. The best path is to port or preserve the narrow fixes: synthesize a command-delivery fallback after outbound projection when all raw outputs normalize away, and add a Slack preview-only no-final fallback that clears stale drafts and posts in-thread status without regressing intentional silence or successful streamed/tool/block replies. Splitting out the unrelated UI formatting churn would make the bug fix much easier to review safely.
