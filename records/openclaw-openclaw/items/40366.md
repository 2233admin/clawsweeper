## Summary

Keep this PR open. Current main still lacks both central behaviors: command delivery can still silently no-op when raw payloads normalize to zero deliverable outbound payloads, and Slack preview/progress turns with no delivered final still clear state and return without the two-step in-thread fallback. The author addressed review feedback and recently updated the branch, so this remains an active implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped rebase. The best path is to port the narrow fixes onto current main: synthesize a command-delivery fallback after outbound projection when all raw outputs normalize away, preserve intentional silence/tool-sent success, and add a Slack preview-only no-final fallback with regression tests. Split unrelated UI formatting churn out of the branch before merge.
