## Summary

Keep this PR open. Current main still lacks the two central behaviors proposed here: command delivery does not turn normalized-empty payloads into a fallback reply, and Slack preview/progress turns with no delivered final still return without the proposed in-thread keepalive/status fallback. The PR is not obsolete, but it needs maintainer review or a narrower rebase because the current diff is XL and includes unrelated UI formatting churn.

Keep this PR open for maintainer review or ask for a narrower rebase. Related #45290 is open and appears complementary rather than a replacement for the Slack-specific no-final behavior.
