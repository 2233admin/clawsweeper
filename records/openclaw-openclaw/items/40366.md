## Summary

Keep this PR open. Current main still lacks both central behaviors: command delivery can no-op when raw payloads normalize to zero deliverable outbound payloads, and Slack preview/progress turns with no delivered final still clear state and return without the proposed in-thread keepalive/status fallback. The related open #45290 is not a merged replacement and does not clearly cover the Slack no-final behavior.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped rebase. The best path is to extract the narrow normalized-empty command fallback and Slack preview-only no-final fallback onto current main, preserve intentional silence/tool-success semantics, keep Slack fallback messages in-thread, add focused regressions, and split unrelated UI formatting churn before merge.
