## Summary

Keep this PR open. Current main still lacks the two central behaviors: `openclaw agent --deliver` can still silently no-op when non-empty raw payloads project to zero deliverable outbound payloads, and Slack preview/progress turns with no delivered final still clear state and return silently. The related open #45290 is broader outbound recovery work, but the provided context does not show it superseding this PR’s normalized-empty command fallback or Slack preview-only no-final fallback. The author also responded to review feedback and recently committed the missing Slack `sleep` helper, so this remains an active implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped rebase. The best path is to port the narrow fixes onto current main: synthesize a command-delivery fallback after outbound projection when all raw outputs normalize away, while preserving intentional silence/tool-sent empty success; and add a Slack preview-only no-final fallback that clears stale drafts and posts same-thread status without regressing healthy streamed/tool/block replies. Splitting unrelated UI formatting churn would make the fix much easier to review safely.
