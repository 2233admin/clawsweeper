## Summary

Keep open. Current main still hardcodes a 20s client timeout for browser file-chooser arming and the CLI upload request, while the existing browser action timeout model already defaults to 60s with slack for explicit waits. The PR is a narrow browser plugin bug fix with matching test coverage, and the provided diff does not touch security-sensitive CI, dependency, publishing, or secret-handling surfaces.

## What This Changes

The PR changes browser file-chooser arming and CLI upload request timeouts so upload hooks use a 60s/default action-style client timeout and adds client coverage for default and explicit chooser timeout propagation.

## Best Possible Solution

Keep this PR open for normal maintainer review and validation, then land it or an equivalent narrow change if the 60s/action-style client timeout is accepted. Treat it as fixing the upload hook client transport timeout only; leave the broader stale-trigger follow-up in #38844 separate.
