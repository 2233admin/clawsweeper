## Summary

Keep open: current main still has the reported browser upload client-timeout mismatch, while this PR is a narrow implementation that plausibly fixes it. The remaining action is maintainer review, especially the explicit CLI timeout slack edge case raised in review, not automatic closure or a replacement PR.

## What This Changes

The PR changes browser file-chooser arming to use the browser action timeout budget, raises the CLI upload wrapper default timeout to 60 seconds, and adds browser client assertions for default and explicit file-chooser request timeouts.

## Best Possible Solution

Land the narrow timeout alignment after addressing or explicitly accepting the CLI explicit-timeout slack edge case, and keep the broader stale-ref/upload-flow/error-mapping work tracked separately in #38844.
