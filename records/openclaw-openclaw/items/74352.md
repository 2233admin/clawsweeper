## Summary

Keep open for maintainer review. Current main still has the 20s upload hook client timeout, so the PR is not obsolete, but the proposed patch leaves the explicit CLI `--timeout-ms` path using the same value as both the server operation budget and the client transport deadline.

## What This Changes

The PR changes browser file-chooser arming to use the browser action timeout budget, raises the CLI upload default request timeout to 60s, and adds browser client timeout assertions.

## Best Possible Solution

Land the narrow timeout alignment after padding or explicitly accepting the CLI explicit-timeout edge case, while leaving upload path guardrails and Playwright semantics unchanged and keeping the broader upload-flow reliability work in #38844.
