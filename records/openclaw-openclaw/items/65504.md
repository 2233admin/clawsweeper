## Summary

Keep this open. Current main has mitigations for auth-profile timeout poisoning, deferred context-engine maintenance, QA readiness, and Telegram polling restart behavior, but the core requested contract is still missing: `agent.wait`, lifecycle snapshots, dedupe snapshots, caller types, protocol docs, and public docs still expose flat `ok|error|timeout` state rather than queue/preflight/provider/post-turn/gateway-draining attribution.

## What This Changes

The issue requests phase-aware live gateway timeout attribution, auth-profile cooldown protection, lower session-lane hold time after heavy turns, QA gateway cleanup, and reduced Telegram polling restart noise.

## Best Possible Solution

Keep this as the OpenClaw tracking item for an additive timeout-attribution contract: propagate phase-aware timing/timeout metadata through embedded-run telemetry, lifecycle snapshots, gateway dedupe snapshots, `agent.wait` responses, caller types, protocol schema/docs, and regression tests, while leaving lossless-claw scheduling to its separate track.
