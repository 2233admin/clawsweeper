## Summary

Keep this open. Current main has partial mitigations for two symptoms, but the core live-gateway attribution gap remains: `agent.wait`, gateway dedupe snapshots, lifecycle snapshots, caller types, and public docs still expose flat terminal states instead of queue/preflight/provider/post-turn/gateway-draining attribution.

## What This Changes

The issue requests phase-aware live gateway timeout attribution, auth-profile failure isolation, lower post-turn session-lane hold time, and QA/Telegram supervision cleanup.

## Best Possible Solution

Keep this as the OpenClaw tracking item for an additive timeout-attribution contract: add phase-aware timing/timeout metadata through embedded-run telemetry, lifecycle snapshots, gateway dedupe snapshots, `agent.wait` responses, caller types, protocol schema/docs, and regression tests, while leaving lossless-claw scheduling to the separate related track.
