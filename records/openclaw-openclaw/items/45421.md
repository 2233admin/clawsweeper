## Summary

Keep this PR open. Current main still does not implement Slack ack reaction run-start timing: the config contract has no `ackReactionTiming`, the Slack monitor context/provider do not carry it, and the Slack message path still starts ack/status reactions at receipt time. The PR is also paired with the same author's open Discord and Telegram companion PRs, so it should be reviewed or updated rather than closed as obsolete.

## Best Possible Solution

Keep the PR open and ask for an update against current main. The best path is to rebase the Slack implementation, preserve default receipt-time behavior, expose a clear Slack-scoped or intentionally shared `ackReactionTiming` config contract, defer the ack/status reaction from `onAgentRunStart`, preserve the existing scope gate and cleanup semantics, refresh Slack tests/docs, and coordinate shared type/schema decisions with #45449 and #46258.
