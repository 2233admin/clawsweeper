## Summary

Keep this PR open. Current main still does not implement Slack ack reaction run-start timing: the Slack config contract has no timing field, the strict schema rejects it, provider/context do not carry it, and the Slack runtime still starts ack/status reactions at receipt time. The PR is also paired with the same author's open Discord and Telegram companion PRs (#45449 and #46258), and the earlier scope-gate review was addressed in the PR branch, so this remains useful implementation work rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and ask for an update against current main. The best path is a rebased Slack implementation that keeps `received` as the default, adds a clearly scoped `ackReactionTiming` config contract, defers the initial ack/status reaction from `onAgentRunStart` when configured, preserves the existing scope gate and cleanup semantics, updates Slack docs/tests, and coordinates shared type/schema decisions with #45449 and #46258.
