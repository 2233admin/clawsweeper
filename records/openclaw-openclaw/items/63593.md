## Summary

Keep this PR open. Current main still has the false downgrade path for unresolved explicit `latest` because the package update gate treats any null registry target version as downgrade risk, and the PR remains a narrow fix with regression coverage after addressing the review concern about unresolved non-latest tags.

## What This Changes

The PR adds a normalized-tag exception in the CLI update downgrade-risk check for unresolved explicit `latest` lookups and adds regression tests for unresolved `latest` and unresolved non-latest dist-tags.

## Best Possible Solution

Rebase or adapt this PR onto current main, keep the narrow `latest`-only null-target exception, preserve the conservative unresolved non-latest downgrade guard, and land it with focused update CLI regression coverage plus the changed-gate validation appropriate for CLI behavior.
