## Summary

Keep this open. Current main still has the reported memory-core mismatch: daily ingestion stores heading-prefixed/list-normalized snippets, while promotion rehydration relocates candidates with whitespace-only live-window matching and silently returns zero when every selected candidate fails relocation.

## What This Changes

The issue requests fixing `memory promote --apply` so daily-ingestion short-term candidates rehydrate and promote correctly instead of being silently dropped by incompatible snippet normalization.

## Best Possible Solution

Fix this in `memory-core` by using a relocation normalization compatible with daily-ingestion snippets while still re-reading the live daily note and skipping deleted/stale sources. Add a regression test for a meaningful-heading list chunk flowing from daily ingestion through `applyShortTermPromotions`, plus one diagnostic when selected candidates all fail rehydration.

## Reproduction Assessment

Yes. A minimal daily note with a meaningful markdown heading followed by list bullets statically reproduces the mismatch: ingestion stores `Heading: item; item` with body-line ranges, while relocation compares against live markdown windows such as `## Heading - item - item` or `- item - item`.

## Solution Assessment

Yes, the best direction is compatible normalization during relocation plus a diagnostic. The workaround fallback to the stored candidate is not the best fix because it bypasses the live daily-note re-read promised in the docs.
