## Summary

Current main still contains the reported memory-core mismatch: daily ingestion stores heading-prefixed, list-joined snippets, while promotion relocation compares only whitespace-collapsed live markdown windows and silently drops candidates that cannot be rehydrated.

## What This Changes

Fix memory-core dreaming promotion so stored daily-ingestion snippets can be reliably rehydrated against live daily-note ranges before `memory promote --apply` writes to `MEMORY.md`.

## Best Possible Solution

Memory-core should use a shared or intentionally compatible normalization path between daily-ingestion snippet storage and live relocation windows, preserve live-source rehydration as the authority, and emit a clear aggregate diagnostic when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: a stored heading-prefixed/list-joined daily chunk does not match the whitespace-collapsed raw markdown window under the current equality/includes comparator.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; the issue's stored-candidate fallback is useful as an emergency workaround but would weaken the documented stale-source protection.
