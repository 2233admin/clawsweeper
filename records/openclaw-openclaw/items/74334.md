## Summary

Current main still has the reported memory-core mismatch: daily ingestion can store heading-prefixed, list-joined snippets while promotion rehydration compares only whitespace-collapsed live markdown ranges, so qualified apply candidates can be silently dropped. This should stay open and is a narrow fix-PR candidate.

## What This Changes

Fix the memory-core promotion apply path so daily-ingestion candidates with stored normalized snippets can be rehydrated from live daily notes instead of silently applying zero promotions.

## Best Possible Solution

Memory-core should use a shared or compatible normalization path for daily-ingestion snippets and live relocation windows, then keep live-source rehydration as the authority and add a clear aggregate diagnostic for all-failed rehydration runs.

## Reproduction Assessment

Yes. The source-level path is high confidence: a daily-ingestion heading/list chunk stores a normalized `Heading: item; item` snippet for body lines, while current relocation compares against raw markdown bullets collapsed only for whitespace; the focused string check reproduces the failed equality/includes conditions.

## Solution Assessment

Yes, the best fix is to align or share normalization for relocation while preserving live daily-note rehydration. The issue's one-line stored-candidate fallback is useful as a workaround but is not the maintainable fix because it can promote stale snapshots.

## Repair Work Prompt

Fix openclaw/openclaw#74334 in memory-core. Current `memory promote --apply` rehydrates selected candidates by comparing stored snippets against live daily-note windows using whitespace-only normalization, but daily ingestion stores snippets with heading prefixes, stripped list markers, `; `-joined list chunks, and 280-character truncation. Implement a narrow fix that makes relocation compare against a normalization compatible with daily-ingestion chunks, while preserving live source rehydration and deleted/edited-source skipping.
