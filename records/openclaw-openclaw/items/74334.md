## Summary

Current main still shows the reported memory-core mismatch, so this should stay open. Daily ingestion can persist heading-prefixed, list-joined snippets, while promotion apply still rehydrates by comparing only whitespace-collapsed live markdown windows and silently returns zero applied candidates when every selected candidate fails relocation.

## What This Changes

The issue requests fixing memory-core promotion apply so daily-ingestion snippets can be rehydrated from live daily notes instead of being silently dropped by incompatible normalization.

## Best Possible Solution

Memory-core should use a shared or compatible normalization path for daily-ingestion snippets and live relocation windows, keep live-source rehydration as the authority, add regression coverage for heading-prefixed list chunks, and emit a clear aggregate diagnostic when every selected candidate fails rehydration.

## Reproduction Assessment

Yes. The issue includes dry-run/apply output, and current-main source inspection plus a focused string check reproduce the failing equality/includes conditions for a heading-prefixed, list-joined stored snippet against raw markdown live windows.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; the stored-candidate fallback is useful as an operator workaround but is not the best product fix because it can promote stale ranges.

## Repair Work Prompt

Fix the memory-core promotion rehydration normalization mismatch reported in https://github.com/openclaw/openclaw/issues/74334. Current daily ingestion in extensions/memory-core/src/dreaming-phases.ts stores rich snippets by stripping list markers, prefixing meaningful headings as Heading: ..., joining list chunks with '; ', truncating, and recording body line ranges that exclude the heading. Current apply rehydration in extensions/memory-core/src/short-term-promotion.ts compares only whitespace-collapsed live markdown windows, so memory promote --apply can silently drop every selected daily-ingestion candidate. Implement a narrow fix that makes relocation compare against a normalization compatible with the ingestion snippet shape while preserving live-source rehydration as the authority; do not use the issue's stored-candidate fallback as the primary fix. Add a defensive aggregate warning or diagnostic when all selected candidates fail rehydration. Coordinate with open PR #74051 if it lands first because it also edits relocation span/tie-break behavior, but do not assume it fixes this normalization issue. Add regression coverage for a daily note with a meaningful heading plus bullet list whose stored candidate snippet is Heading: item; item and whose stored line range excludes the heading, then verify applyShortTermPromotions appends the live rehydrated promotion instead of returning applied: 0. Preserve deleted/stale-source skip behavior. Because this is user-facing memory behavior, add a CHANGELOG.md fix entry with Thanks @haumanto; do not add broad refactors or new config/API surface.
