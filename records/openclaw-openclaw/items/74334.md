## Summary

Current main still has the reported memory-core normalization mismatch: daily ingestion stores heading-prefixed/list-joined snippets, but promotion rehydration relocates candidates using only whitespace-collapsed live ranges and silently returns zero when all selected candidates fail relocation. This should stay open and is a good focused fix-PR candidate.

## What This Changes

Fix memory-core short-term promotion rehydration so stored daily-ingestion snippets can match live daily-note ranges, while preserving the live-file stale/deleted-snippet guard and adding observability when selected candidates all fail rehydration.

## Best Possible Solution

Keep this issue open and fix it in memory-core by making relocation compare against a normalization compatible with daily-ingestion snippets, preserving live source rehydration and deleted-source skipping. Add a regression test for a heading-prefixed list candidate produced by daily ingestion, plus a warning/diagnostic when selected candidates all fail rehydration.

## Repair Work Prompt

Fix openclaw/openclaw#74334 in the memory-core plugin. Current `memory promote --apply` rehydrates selected candidates with `relocateCandidateRange`, but daily ingestion stores snippets normalized by `extensions/memory-core/src/dreaming-phases.ts` with heading prefixes, list-marker stripping, `; ` list joins, and truncation. Make the relocation comparison in `extensions/memory-core/src/short-term-promotion.ts` compatible with those stored snippets while preserving the documented behavior that live daily files are re-read and deleted/stale snippets are skipped. Do not solve this only by blindly promoting the stored candidate when the live file no longer matches. Add a focused regression test in `extensions/memory-core/src/short-term-promotion.test.ts` for a daily-ingestion-shaped candidate whose stored snippet includes a heading prefix and joined bullet items while the recorded line range starts at the body lines. Also add a low-noise warning/diagnostic for the case where selected candidates are non-empty but all fail rehydration. Keep the change inside memory-core unless a small local shared helper between `dreaming-phases.ts` and `short-term-promotion.ts` is clearly cleaner. Add a user-facing changelog fix entry crediting `Thanks @haumanto` if the repo’s active changelog section requires it.
