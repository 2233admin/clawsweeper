## Repair Work Prompt

Fix the memory-core dreaming promotion rehydration bug reported in https://github.com/openclaw/openclaw/issues/74334. Current main stores daily-ingestion snippets in `extensions/memory-core/src/dreaming-phases.ts` with heading prefixes, list-marker stripping, `; ` list joins, and truncation, while `extensions/memory-core/src/short-term-promotion.ts` relocates candidates using only whitespace-collapsed exact/includes checks. Implement a shared or compatible normalization path so daily-ingested heading/list chunks can be rehydrated from live daily notes. Preserve live-source rehydration as the authority; do not replace failed relocation with a blind stored-candidate fallback. Add one aggregate warning or equivalent observable diagnostic when selected candidates all fail rehydration. Add regression coverage for a daily note with a meaningful heading and bullet list whose stored candidate applies successfully with the live line range, plus keep the stale/deleted snippet guard covered. Update `CHANGELOG.md` because this is user-facing promotion behavior. Be aware that #74051 touches relocation span and recall gates for #71976 but does not fix this normalizer mismatch; avoid expanding the fix into unrelated recall-gate policy.
