## Summary

Keep open: current main still has incompatible stored-snippet and live-window normalization in memory-core, so heading-prefixed daily-ingestion candidates can fail relocation and then be dropped from `--apply` without an aggregate diagnostic.

## What This Changes

The issue requests that memory-core promotion rehydrate daily-ingestion snippets whose stored form includes heading prefixes, joined list items, and truncation, instead of silently applying zero candidates.

## Best Possible Solution

Memory-core should use a shared or explicitly compatible normalization path for daily-ingestion snippets and relocation windows, keep live-source rehydration authoritative, and add one aggregate warning when selected candidates all fail rehydration.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current daily ingestion can store `Heading: item; item` snippets for body-only line ranges, while current relocation compares those against raw markdown windows after whitespace collapse only.

## Solution Assessment

Yes. Aligning or sharing normalization in memory-core is the narrow maintainable fix; the one-line stored-candidate fallback would restore nonzero counts but would bypass the live-source stale/deleted-note protection promised by the docs.
