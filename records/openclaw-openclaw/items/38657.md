## Summary

Keep this issue open. Current main still preserves existing generated provider `apiKey` and `baseUrl` values in the default merge path, so a deliberate same-provider Onboard/configure edit can still be masked by `agents/*/agent/models.json` or auth-profile state. Related PR #39103 addressed a narrower stale `baseUrl` precedence case, and #67893 later restored documented merge-mode `models.json` `baseUrl` preservation, leaving the broader provider-update contract unresolved.

## What This Changes

Onboard/configure should persist deliberate updates to an existing provider configuration across `openclaw.json`, regenerated `agents/*/agent/models.json`, and auth-profile state or clearly report that an existing provider was not updated.

## Best Possible Solution

Keep this issue open and define an explicit provider-update contract for deliberate Onboard/configure same-provider edits. The fix should distinguish user-initiated provider edits from ordinary merge-mode regeneration, update source config and generated state consistently, preserve SecretRef marker semantics, and cover Moonshot `.ai` to `.cn` baseUrl changes plus plaintext and SecretRef key replacement cases.
