## Summary

Keep open. Current main downloads Telegram document attachments and can add text-like/PDF content through the shared media path, but it still does not satisfy the common Office-document part of the request: Office MIME types are not in the default file allowlist, `application/vnd.*` Office files are skipped as binary before extraction, and the bundled document extractor advertises PDF only. The related DOCX implementation PR remains open, so this is not implemented on main yet.

## What This Changes

Request to extract text from Telegram document attachments such as PDF, TXT, DOCX, Markdown, and similar text-based files so agents can read their contents instead of only seeing a `<media:document>` placeholder.

## Best Possible Solution

Finish this through the shared document extraction pipeline rather than Telegram-specific parsing: review or adapt #57612 for DOCX, decide the allowed Office MIME/dependency/resource policy, keep binary-safety and untrusted-content wrapping intact, and add focused Telegram plus shared media/input-file coverage and docs for the supported formats.
