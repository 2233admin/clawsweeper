## Summary

Keep open. Current main already downloads Telegram document attachments and forwards the saved media path into the shared context, and it handles text/JSON/PDF extraction, but standard Office document extraction is still missing and explicitly skipped before the extractor path. The related DOCX implementation PR is still open and unmerged, so it is not a shipped fix.

## What This Changes

The issue requests text extraction for Telegram document attachments such as .docx, .pdf, .txt, and .md so the agent can read the contents instead of only receiving a media placeholder.

## Best Possible Solution

Finish this through the shared document extraction pipeline rather than Telegram-specific parsing: review or adapt the open DOCX work, add bounded Office MIME/extractor support only after parser and resource-limit policy is settled, preserve binary-safety checks and untrusted-content wrapping, and cover both Telegram media handoff and shared media/input-file behavior with focused tests and docs.
