## Risks / Open Questions

- Closing this PR now would leave current main without the proposed coercion for strict MCP servers that reject stringified object/array properties such as `params: "{}"`.
- Before merge, the proposed helper should avoid prototype-sensitive dynamic assignment, for example by using a null-prototype result object or explicit handling for `__proto__`, `constructor`, and `prototype`.
- Before merge, the proposed helper should bound or otherwise gate synchronous `JSON.parse` of model/tool-call strings to avoid avoidable CPU or memory spikes on oversized arguments.
- Before merge, the proposed helper should decide whether JSON Schema array-form `type` values such as `["object", "null"]` are in scope; the current PR patch handles only string `type` values.
