## Summary

Keep open. Current main still forwards bundle MCP tool arguments unchanged, so the PR addresses a real MCP validation failure; the proposed patch should not merge until the schema and security edge cases are fixed.

## What This Changes

The PR adds a bundle MCP materialization helper that parses top-level stringified object/array properties according to each tool input schema before calling the MCP runtime, plus regression tests.

## Best Possible Solution

Land a hardened version at the bundle MCP materialization boundary: coerce only schema-declared object/array properties, handle nullable `type` arrays, copy into a safe result object, apply a reasonable parse budget, and keep focused regression coverage.

## Reproduction Assessment

Yes. A materialized bundle MCP tool whose schema declares `params` as an object and receives `{ params: "{}" }` will currently forward the string unchanged through `materializeBundleMcpToolsForRun` and `SessionMcpRuntime.callTool`, matching the #70872 failure path.

## Solution Assessment

No, not yet. The materialization boundary is the right place for this fix, but this implementation needs safer key handling, parse limits, and JSON Schema type-array support before it is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Accept JSON Schema array type forms:** `src/agents/pi-bundle-mcp-materialize.ts:54-55`
  - body: Schemas may declare nullable object params as `type: ["object", "null"]`. This guard only accepts a string `type`, so those object/array params still pass through as JSON strings and strict MCP servers can continue rejecting them.
  - confidence: 0.86
- **[P2] Use a null-prototype result object:** `src/agents/pi-bundle-mcp-materialize.ts:47`
  - body: The helper copies model-controlled argument keys into a plain `{}` and returns it after any coercion. A key such as `__proto__` with an object value can mutate the returned object's prototype before it is sent to the MCP runtime; use a null-prototype object or define own data properties.
  - confidence: 0.84
- **[P2] Bound JSON.parse for coerced arguments:** `src/agents/pi-bundle-mcp-materialize.ts:58`
  - body: Every string value under an object/array schema is synchronously parsed without a size budget. An oversized tool argument can block the event loop or spike memory during tool execution, so the coercion path should reject or skip values above a conservative limit.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The PR introduces caller-controlled object copying and unbounded synchronous parsing in the tool-call path, so it needs security hardening before merge.

Concerns:

- **[medium] Prototype mutation through copied argument keys:** `src/agents/pi-bundle-mcp-materialize.ts:47`
  - body: `coerceStringifiedObjectProperties` copies tool-call keys into a plain object; special keys such as `__proto__` can alter the returned object's prototype when a coerced result is returned downstream.
  - confidence: 0.84
- **[medium] Unbounded synchronous JSON parsing:** `src/agents/pi-bundle-mcp-materialize.ts:58`
  - body: The helper parses schema-declared object/array strings without checking length or budget, allowing large tool-call strings to add avoidable CPU and memory pressure in the Node event loop.
