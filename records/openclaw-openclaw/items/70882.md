## Summary

Keep #70882 open. Current main still forwards bundle MCP tool inputs to the MCP SDK without schema-guided coercion of stringified object/array properties, and the existing materialize tests do not cover the linked #70872 Synology/Pydantic failure. The PR is a plausible core bug fix, though it should be revised for the bot-reviewed edge cases before landing.

## Best Possible Solution

Keep #70882 open and either revise this PR or land an equivalent fix in the bundle MCP materialize/call boundary. The implementation should coerce only schema-declared object/array string values before MCP calls, preserve already-correct inputs, include regression tests for #70872, and address the bot-reviewed safety and schema edge cases before merge.
