## Summary

Keep open. Current main still has the reported contradictory update output, and this PR is a small implementation PR that addresses the right surface; it should go through normal maintainer review, CI, and the listed manual dirty-checkout verification rather than being closed by cleanup.

## What This Changes

This PR forwards clean-check stdout through update step completion, uses it to render dirty worktree status in update progress and result output, prints offending dirty entries with `git stash -u` guidance, and adds tests plus a changelog entry.

## Best Possible Solution

Land a narrow presentation-layer fix that keeps the runner's existing dirty-tree detection, passes the clean-check stdout into the progress/result path, and verifies the spinner, skipped result, and dirty-file guidance with focused tests plus one manual untracked-file check.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to make `git status --porcelain` emit entries while exiting 0; current main's runner returns `status: skipped, reason: dirty`, while the progress renderer only sees the zero exit code and can print a clean working tree message.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix because it reuses the runner's existing stdout signal and changes reporting without changing git update semantics; final merge should still wait for CI/manual verification and any needed rebase.
