## Summary

Keep open. Current main still has the reported clean/dirty display mismatch, and the PR is a focused fix, but it is not merge-ready because the new dirty-entry rendering prints `git status --porcelain` output directly to the terminal without the repo's terminal-text sanitizer; the existing Copilot review comments flagged the same concrete issue.

## What This Changes

The PR forwards update-step stdout tails into progress completion, uses them to mark dirty clean-check output in progress/result rendering, surfaces dirty entries with `git stash -u` guidance, and adds focused tests plus a changelog entry.

## Best Possible Solution

Keep the stdoutTail plumbing and shared display helper approach, but sanitize dirty status lines in every terminal-rendering path before applying theme coloring, then land with the existing targeted tests plus CI and the manual dirty-checkout check.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: make `git status --porcelain` emit dirty entries while exiting 0; current main's runner returns `reason: dirty` while the progress renderer can still print the clean-working-tree success message.

## Solution Assessment

No, not as-is. The proposed architecture is the right narrow fix, but the new terminal output should route dirty lines through `sanitizeTerminalText()` before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Sanitize clean-check stdout before rendering it:** `src/cli/update-cli/progress.ts:169`
  - body: This renders `stdoutTail` from `git status --porcelain` directly in the live progress path, and the same helper feeds the non-progress step summary below. Dirty filenames are local user-controlled terminal text, so crafted control sequences can spoof or alter terminal output. Sanitize each line, for example with `sanitizeTerminalText()`, before applying `theme.warn()`.
  - confidence: 0.84
- **[P2] Sanitize dirty entries in the skipped result:** `src/cli/update-cli/update-command.ts:1626`
  - body: The dirty skipped-result branch also prints the same clean-check `stdoutTail` lines directly. When progress is hidden or the skipped result is the primary output, crafted filenames can still affect the terminal. Sanitize each line before passing it to `theme.muted()`.
  - confidence: 0.84
