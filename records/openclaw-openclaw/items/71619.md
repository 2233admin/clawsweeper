## What This Changes

This PR changes Linux daemon minimal service PATH assembly and tests to exclude nvm, fnm, Volta, asdf, and Bun roots while preserving stable user-bin, npm, pnpm, and Nix paths, with macOS version-manager behavior left in scope.

## Best Possible Solution

Rebase the PR onto current main and apply the same Linux policy in the current service-env helpers: remove Linux version-manager roots from the minimal PATH source, keep stable user-bin, npm, pnpm, and Nix behavior, preserve the security/path-normalization guards, and update focused service-env plus service-audit coverage for Linux env-configured version-manager roots.

## Reproduction Assessment

Yes. A source-level reproduction is to evaluate current main's Linux service path with HOME plus NVM_DIR, FNM_DIR, VOLTA_HOME, ASDF_DATA_DIR, or BUN_INSTALL and then audit that PATH; service-env includes those roots and service-audit treats expected entries as required.

## Solution Assessment

Yes, with current-main adaptation. Removing Linux version-manager roots at the minimal-PATH source is narrower than special-casing audit output, but the current branch cannot be merged as-is until it is rebased over newer PATH security and fallback-existence helpers.
