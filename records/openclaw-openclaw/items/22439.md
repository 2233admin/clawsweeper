## Summary

Keep open. Current main still has only binary bootstrap filtering and no `bootstrapTier` config, while this PR is paired with open proposal #22438 and maintainer guidance asking for product/API review. The PR head also has a concrete correctness issue: the proposed `standard` tier filters by basename, but real extra bootstrap files use the same recognized basenames as root files, so `standard` and `full` are not separated for the hook path.

## What This Changes

The PR adds `agents.defaults.bootstrapTier`, tier resolution and filtering for workspace bootstrap files, extra-bootstrap hook wiring, generated config metadata/help, and focused tier tests.

## Best Possible Solution

Keep the PR open with #22438 for maintainer product/API review. If maintainers accept this API, fix the extra-file provenance/path distinction, refresh against current main, and land the source, hook, schema/types, generated metadata/help/docs, and tests together; otherwise close it manually after the product decision.

Overall correctness: patch is incorrect

- **[P2] Distinguish real extra bootstrap files from root files:** `src/agents/workspace.ts:731-733`
