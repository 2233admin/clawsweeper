## Summary

Keep this PR open. Current main still does not implement the PR's intended ACP control-plane audit logging system, and the existing audit-adjacent surfaces are narrower config/command/gateway actor helpers rather than ACP session lifecycle audit logging. The PR remains potentially useful but needs a rebase and security hardening before merge.

## Best Possible Solution

Keep this PR open, but require a rebased and narrower ACP audit implementation before merge. The right product path is a best-effort AcpSessionManager audit seam with a real sink under the normal OpenClaw state/log directory, restrictive permissions, focused session init/close/cancel/run/runtime-control coverage, tests for sink failures and lifecycle semantics, and unrelated config/schema changes split out.
