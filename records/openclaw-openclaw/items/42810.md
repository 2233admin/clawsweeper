## Summary

Keep this PR open. Current main still does not implement the PR's intended ACP control-plane audit logging system: there is no ACP audit module, no audit logger dependency on AcpSessionManager, no production sink wiring, and the ACP lifecycle/control paths do not emit audit records. Existing audit-adjacent code covers narrower config writes, command hooks, or gateway actor formatting, not a durable ACP session audit trail. The PR remains potentially useful, but the proposed security feature needs a narrow rebase and hardening before merge.

## Best Possible Solution

Keep this PR open, but require a rebased and narrower ACP audit implementation before merge. The best path is a best-effort AcpSessionManager audit seam with a real production sink under the normal OpenClaw state/log directory, restrictive permissions, bounded buffering/failure behavior, correct retention/query semantics, focused coverage for session init/close/cancel/run/runtime-control events, and unrelated config/schema changes split out.

## Risks / Open Questions

- Closing this PR as implemented would lose a still-missing ACP control-plane audit trail on current main.
- Merging the PR as-is would be risky for a security/compliance feature because the proposed file sink has unresolved reliability, retention, buffering, and query-consistency concerns from review.
- The PR currently includes unrelated config/schema drift alongside the ACP audit work, so it needs a narrower rebase before maintainer review.
- A production implementation needs explicit sink wiring; a default NullAuditLogger-only path would create a false sense that ACP auditing is enabled.
