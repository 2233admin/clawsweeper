## Summary

Keep this issue open. Current main still lacks a hook/webhook-specific lifecycle policy for on-complete archive/delete, delayed cleanup, transcript retention, or hiding completed hook sessions. Hook agent work is still dispatched through isolated cron-style execution, but non-cron hook keys are persisted as ordinary agent-scoped session rows, and the cron reaper only covers canonical cron run-session aliases. The prior maintainer/Codex comment also kept this open, and live search found no open PR mentioning this issue.

## Best Possible Solution

Keep this issue open. The best maintainer path is a core `hooks.sessionCleanup` policy with config types, strict schema/help/docs, and runtime finalization for direct `/hooks/agent`, mapped hooks, Gmail preset handling, and isolated-run completion. It should define archive/delete/no-op modes, optional delay, transcript retention, completed-session visibility, generated `hook:*` keys, and focused tests for `agent:<id>:hook:gmail:*` persistence and cleanup behavior.
