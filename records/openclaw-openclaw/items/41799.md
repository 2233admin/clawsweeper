## Summary

Keep this PR open. Current main still does not implement the PR's intended compatibility behavior for legacy channel plugins whose `messaging.normalizeTarget` returns an object-shaped `{ ok, to }` value at runtime. The current code remains string-only, drops object results through `normalizeOptionalString`, falls back to the raw target, and has no regression coverage for the legacy object shape. The PR is a plausible focused bug fix, but it needs cleanup before maintainer review.

## Best Possible Solution

Keep the PR open and rework it against current main. Add a small runtime-only helper in `src/infra/outbound/target-normalization.ts` that treats the normalizer result as `unknown`, extracts a trimmed nonblank string `to` from legacy `{ ok, to }` values, and leaves malformed objects on the existing fallback/validation path. Cover it in `src/infra/outbound/target-normalization.test.ts` plus one current resolver or send-runner test, then remove the unrelated `.secrets.baseline` churn and broad OAuth mock before maintainer review.
