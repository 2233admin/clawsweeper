## Summary

Keep this PR open. Current main still does not implement the intended runtime compatibility behavior: object-shaped legacy normalizer results are not read for their `to` value, so the plugin-returned target is lost. The PR remains a focused bug fix, but it needs a rebase onto the current split outbound tests and cleanup of unrelated baseline/mock churn before maintainer review.

## Best Possible Solution

Keep the PR open and rework it against current main. Add a small runtime-only helper in `src/infra/outbound/target-normalization.ts` that extracts a trimmed nonblank string `to` from legacy `{ ok, to }` values, keep malformed objects on the existing fallback/validation path, and add focused coverage in `src/infra/outbound/target-normalization.test.ts` plus a current target-resolver or core-send runner test. Drop the unrelated `.secrets.baseline` churn and avoid the broad OAuth mock unless it is still necessary and narrowly scoped.
