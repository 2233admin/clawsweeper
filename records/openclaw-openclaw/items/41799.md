## Summary

Keep this PR open. Current main still does not implement the PR's intended runtime compatibility behavior: `messaging.normalizeTarget` is still typed and handled as string-only, object-shaped legacy results are not unwrapped for their `to` value, and there is no regression coverage for that legacy shape. The PR remains a plausible focused bug fix, but it needs rebase/cleanup against the current split outbound tests before maintainer review.

## Best Possible Solution

Keep the PR open and rework it against current main. Add a small runtime-only helper in `src/infra/outbound/target-normalization.ts` that treats the normalizer result as `unknown`, extracts a trimmed nonblank string `to` from legacy `{ ok, to }` values, and leaves malformed objects on the existing fallback/validation path. Add focused coverage in `src/infra/outbound/target-normalization.test.ts` plus a current resolver or core-send runner test, and remove the unrelated baseline/mock churn before maintainer review.
