## Summary

Keep this PR open. The reported OAuth beta-header gap is still present on current main, but the PR targets the older core-runner path while the active Anthropic behavior now lives in the bundled Anthropic provider plugin. The same-author linked #41444 is still open and this PR uses closing syntax for it, so the pair should remain open until the provider-path fix is reviewed, merged, or explicitly rejected.

## Best Possible Solution

Keep this PR open and port the useful fix into `extensions/anthropic/stream-wrappers.ts`. The provider-owned implementation should ensure OAuth-required betas are merged whenever an `sk-ant-oat` token is used, even when no extra-param betas resolve, while preserving the existing service-tier, fast-mode, and thinking-prefill composition and avoiding unintended loss of model/provider beta headers for API-key auth. Add plugin-level regression coverage for the #41444 scenario, then review and land this with the paired issue or close both only if maintainers reject the product behavior.
