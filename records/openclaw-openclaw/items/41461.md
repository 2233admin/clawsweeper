## Summary

Keep this PR open. The linked same-author issue is still open, current main still has the active Anthropic provider wrapper gated on resolved extra-param betas, and the PR’s useful fix needs to be ported from the older core-runner path into the bundled Anthropic provider plugin with focused regression coverage.

## Best Possible Solution

Keep this PR open as the implementation candidate for #41444, but port the useful logic into `extensions/anthropic/stream-wrappers.ts`. The provider-owned fix should apply the OAuth/default beta merge whenever direct Anthropic uses an `sk-ant-oat` token, even when no extra-param betas resolve, while preserving current API-key, service-tier, fast-mode, thinking-prefill, and custom-endpoint behavior. Add plugin-level regression coverage for OAuth token plus model/provider `anthropic-beta` headers with no `context1m` or `anthropicBeta` extra param.
