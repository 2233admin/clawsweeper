## Summary

Keep this PR open. Current main still has the Anthropic OAuth/default beta gap in the active provider-owned wrapper path, and this PR is paired with the same-author open bug #41444. The PR’s idea is useful, but its diff targets the older embedded-runner wrapper location rather than the current Anthropic plugin hook.

## Best Possible Solution

Keep this PR open as an implementation candidate for #41444, but port the useful logic to extensions/anthropic/stream-wrappers.ts: compose the Anthropic beta header wrapper for direct public Anthropic requests even when resolveAnthropicBetas returns no explicit betas, pass an empty beta list so OAuth/default beta merging still runs, preserve custom-endpoint suppression, and add a focused regression test for an OAuth token plus model/provider-level anthropic-beta with no context1m or anthropicBeta extra param.
