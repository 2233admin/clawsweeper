Keep this PR open. The reported OAuth beta-header problem still appears valid on current main, but the PR patch targets the older embedded-runner wrapper location while current main routes Anthropic stream wrapping through the bundled Anthropic plugin. The fix should be ported to the active plugin wrapper path and covered there before merge.

## Best Possible Solution

Keep this PR open and update it for current main: move the unconditional beta-wrapper composition into extensions/anthropic/stream-wrappers.ts, pass an empty beta list when no explicit extra-param betas are configured, preserve direct-Anthropic/custom-endpoint suppression for implicit beta markers, and add a focused regression test for an OAuth token plus model/provider-level anthropic-beta with no context1m or anthropicBeta extra param. The linked open issue #41444 should close only after that current-path fix lands.
