## Summary

Keep this PR open. Current main still has the reported Anthropic OAuth/model-header gap, but the PR’s patch targets the older core-runner path while current behavior is owned by the bundled Anthropic provider plugin. It remains a useful implementation candidate for the same-author open #41444, not a cleanup-close candidate.

## Best Possible Solution

Keep this PR open as an implementation candidate for #41444, but port the useful logic into extensions/anthropic/stream-wrappers.ts. The provider-owned fix should apply the OAuth/default beta merge for direct Anthropic requests even when no extra-param betas resolve, while preserving custom endpoint suppression, service-tier, fast-mode, thinking-prefill, and API-key behavior. Add focused plugin/transport regression coverage for sk-ant-oat plus model/provider anthropic-beta headers with no context1m or anthropicBeta extra param.
