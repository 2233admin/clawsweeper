## Summary

Keep open. Current main still has the Discord announce-routing gap: Discord announce targets return before session hydration, so the session account identity is not recovered, and the already-linked open PR #73975 is the active implementation path with closing syntax.

## What This Changes

The issue requests that Discord agent-to-agent announce delivery preserve the target session accountId from deliveryContext or lastAccountId instead of routing through the wrong/default Discord account.

## Best Possible Solution

Review and land PR #73975, or an equivalent narrow Discord-owned fix that opts Discord announce targets into session hydration and adds regression coverage proving deliveryContext.accountId or lastAccountId reaches the final gateway send.
