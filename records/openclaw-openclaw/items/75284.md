## Summary

Keep this PR open, but do not merge it as-is. The linked Anthropic prefill failure is credible, yet this patch deletes ordinary completed assistant turns from persistent session history; current main already has provider-scoped outbound cleanup for Anthropic/Gemini replay, so any on-disk repair needs a narrower context-aware design.

## What This Changes

The branch updates session-file repair to pop trailing assistant message entries from session JSONL files, report the count in repair warnings, and add tests for mid-transcript preservation and trailing-assistant drops.

## Best Possible Solution

Preserve completed stored transcripts and keep provider-specific prefill handling at the outbound replay boundary. If persistent repair is still needed, make it distinguish a provably incomplete/provider-specific artifact from a normal completed assistant reply and add regression coverage that ordinary user-to-assistant completed sessions survive repair.

## Reproduction Assessment

Yes for the PR-introduced regression. The PR's own header -> user -> assistant fixture exercises the new loop and rewrites the file to header -> user, and current main invokes repair before the next prompt is appended.

## Solution Assessment

No. Generic disk trimming is broader than the reported Anthropic failure because it treats every assistant tail as corrupt; provider-scoped outbound replay cleanup is the narrower maintainable path unless a safe incomplete-turn predicate is added.
