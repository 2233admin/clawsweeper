## Summary

Keep this PR open, but the current patch is not safe to merge. The related Anthropic prefill report is credible, yet this diff makes the generic on-disk repair delete every trailing assistant message, which also matches normal completed session transcripts and would erase successful replies on the next run.

## What This Changes

The branch updates `repairSessionFileIfNeeded` to count and pop trailing assistant message entries from session JSONL files, with session-file-repair tests adjusted for mid-transcript preservation and trailing-assistant drops.

## Best Possible Solution

Preserve stored completed transcripts and handle Anthropic/Gemini prefill at the provider replay boundary, or make any on-disk repair explicitly context-aware so it only removes provably incomplete/prefill artifacts and includes regression coverage for ordinary `user -> assistant` completed sessions.

## Reproduction Assessment

Yes. The PR’s own added test builds `header -> user -> assistant` JSONL and expects repair to remove the assistant; current main shows that repair runs before the next `activeSession.prompt(...)`, so that path would reproduce the data loss without needing live provider access.

## Solution Assessment

No. A generic disk rewrite is broader than the reported Anthropic failure; the existing provider-scoped outbound replay sanitizer is the safer pattern unless persistent repair can distinguish corrupt prefill state from a normal completed assistant reply.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve completed assistant turns during repair:** `src/agents/session-file-repair.ts:223-230`
  - body: This loop deletes any trailing assistant message from the JSONL file. `repairSessionFileIfNeeded` runs before each embedded attempt opens the session and before `activeSession.prompt(...)` appends the next user turn, while normal completed sessions commonly end `user -> assistant`; reopening one would therefore erase the last successful assistant reply from stored history. Please avoid trimming generic assistant tails from disk, or narrow this to a provably incomplete/provider-specific prefill state.
