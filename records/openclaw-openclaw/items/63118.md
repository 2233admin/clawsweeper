## Summary

Current main still lacks the public session lifecycle controls requested here. The CLI `sessions` surface lists stored sessions and registers cleanup/export maintenance commands, Gateway has lower-level `sessions.abort`/reset/delete/compact plumbing, and Chat/Control UI has a Stop path for the active chat run, but there are no public `openclaw sessions stop`, `kill`, `pause`, or `resume` commands and the Sessions table has no row-level Stop/Pause/Resume controls. The related open PR #35735 tracks delete-style `rm`/`clear` cleanup, not active-run lifecycle control.

## What This Changes

The issue requests CLI and Web Console session lifecycle management, including stop/kill/pause/resume commands, Sessions UI controls, cleanup/filtering, and resource-limit enforcement.

## Best Possible Solution

Keep this as the broad session lifecycle tracker. The best implementation path is to ship a narrow public `openclaw sessions stop <session-key>` first, backed by existing Gateway abort behavior and documented result states, then add Sessions-page Stop controls only where active-run state is reliable. Pause/resume, quotas, token thresholds, and alerting should wait until the runtime, provider cancellation, and accounting model can enforce them.
