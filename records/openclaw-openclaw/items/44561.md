## Summary

Keep this PR open. Current main still does not definitely implement the intended display-name stabilization: gateway session rows can still derive `displayName` from stale `origin.label`, and the UI prefers that row value over its `Main Session` fallback. The PR remains a plausible fix candidate, but it needs rebasing/adapting to the current shared row projection path.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Rebase it onto current main, put the main-session guard in `buildGatewaySessionRow` or a shared row-display helper, preserve explicit `entry.displayName`, use `resolveAgentMainSessionKey` and `resolveMainSessionKey`, and add regression coverage for default, custom, and global main sessions with stale `origin.label: "heartbeat"`.
