## Summary

Keep open. Current main still exposes poll creation fields on the shared `message` tool schema and still rejects `action: "send"` when non-empty or true poll-looking fields are present, before the request reaches Feishu or Telegram dispatch. The zero-default mitigation is present, but it only covers empty/false/zero defaults, not the reported model auto-fill case. Open PR #46455 remains an unmerged implementation candidate for this issue, so the right maintainer action is to review, land, or replace that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #46455. The durable fix should either sanitize accidental poll creation params before the `send` guard or narrow the LLM-facing schema so poll creation fields are only visible for poll-capable actions/channels, while preserving explicit `action="poll"` validation and adding regression coverage for Feishu and Telegram send-with-attachment calls that include non-empty auto-populated poll fields.
