## Summary

Keep open. Current main still exposes shared poll creation fields on the LLM-facing `message` tool and still rejects `action="send"` before dispatch whenever non-empty or true poll-looking fields are present. The zero-default mitigation is present, but it does not cover the Feishu/Telegram model auto-fill case described in the issue and comments. Open PR #46455 still explicitly targets this issue with closing syntax and remains unmerged, so the right maintainer action is to review, land, or replace that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #46455. The durable fix should either sanitize accidental poll creation params before the `send` guard or narrow the LLM-facing schema so poll creation fields are only visible for poll-capable actions/channels, while preserving explicit `action="poll"` validation and adding regression coverage for Feishu and Telegram send-with-attachment calls that include non-empty auto-populated poll fields.
