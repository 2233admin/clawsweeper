## Summary

Keep this issue open. Current main still exposes poll creation fields on the shared `message` tool schema and still rejects `action="send"` before Feishu's media send path when non-empty poll-looking fields are present. PR #46455 remains an open implementation candidate that explicitly links this issue for closure, so this issue should not be closed before that PR is reviewed, landed, or replaced.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #46455. The durable narrow fix should strip or ignore accidental poll creation params for `action="send"` before the poll guard, or narrow the LLM-facing schema so poll fields are only visible for poll-capable actions/channels, while preserving `action="poll"` validation. Add regression coverage for Feishu and Telegram send-with-attachment calls carrying non-empty auto-populated poll fields.
