## Summary

Keep this PR open. Current main still writes standalone REM reports as `# REM Sleep` followed by the raw REM preview body headings at `###`, while inline daily-note output uses the same raw body under `## REM Sleep`. The PR’s latest shape addresses the bot review by preserving inline `###` headings and lowering them only for standalone reports, so the work remains useful and is not implemented on main.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate for the remaining heading-depth bug. The best path is a rebased, mode-specific fix in `extensions/memory-core/src/dreaming-markdown.ts`, with regression coverage that inline daily-note REM output stays under `## REM Sleep` as `### ...` while standalone REM reports under `# REM Sleep` render those sections as `## ...`.
