## Summary

Keep open. Current main still uses synchronous filesystem APIs in `buildExportSessionReply`, so the PR's intended async migration is not implemented. The PR is focused on one command and remains potentially useful, but the proposed diff needs revision before merge because review already identified an unnecessary `access` guard before recursive `mkdir` and a redundant `return await`.

## Best Possible Solution

Keep the PR open and ask for a small revision: convert the export-session command to async fs APIs, load templates with `Promise.all`, call recursive `mkdir` directly without an `access` preflight, remove the redundant `return await`, and update `src/auto-reply/reply/commands-export-session.test.ts` to cover the async filesystem path before running the targeted test.
