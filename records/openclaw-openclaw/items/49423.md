## Summary

Keep this PR open. Current main still uses synchronous filesystem APIs in the user-facing `export-session` command, so the PR's intended cleanup is not implemented on main. The PR is focused, but existing review comments identify cleanup needed before merge: avoid `access` before recursive `mkdir`, remove redundant `return await`, and update tests that still mock sync fs calls.

## Best Possible Solution

Keep this PR open for a small revision: convert the command to promise-based fs APIs, load templates with `Promise.all`, call recursive `mkdir` directly, remove redundant `return await`, and update the colocated tests to exercise async fs behavior before normal maintainer review.
