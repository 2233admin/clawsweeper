## Summary

Keep open. Current main still lacks the requested env override, and this PR addresses a real plugin-loader operational gap, but the proposed fallback changes existing Jiti semantics by forcing filesystem caching on when the OpenClaw variable is unset.

## What This Changes

The PR adds an `OPENCLAW_JITI_CACHE_DIR` env override to Jiti loader options in the shared plugin loader, SDK root alias, and Matrix runtime wrapper, and disables `fsCache` in one regression-test fixture.

## Best Possible Solution

Land a narrow implementation that only supplies a trimmed custom cache directory when `OPENCLAW_JITI_CACHE_DIR` is non-empty, otherwise leaves `fsCache` undefined so Jiti keeps its own env/default handling; cover the shared loader, SDK root alias, and Matrix wrapper with focused regression tests and the required operator-facing documentation or changelog entry.
