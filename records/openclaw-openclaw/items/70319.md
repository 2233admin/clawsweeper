## Summary

Keep this issue open. Current main still has useful abort, queue cleanup, backend cancellation, and typing/progress primitives, but it does not implement the requested unified per-session run-generation invalidation or broad stale-output fencing. The same-author draft PR #70363 is open, unmerged, and explicitly partial in the provided context, so it is an implementation candidate rather than a close reason.

## Best Possible Solution

Keep this issue open as the canonical core interruptibility tracker. Review or split #70363, then land focused core changes for a session/run generation primitive, stale-output fences across reply/typing/followup/tool-progress/event emission paths, a pre-tool currentness gate in the embedded Pi tool path, new-message takeover that invalidates and cancels the active run, and regression tests proving stale runs cannot emit visible effects or launch further tools.
