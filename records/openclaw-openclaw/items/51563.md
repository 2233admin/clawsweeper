## Summary

Keep this PR open. Current main still computes `openclaw update status` package manager from the installed package's repo metadata, so the linked npm-global misdetection remains present. The PR targets the right user-visible bug, but should be refreshed before merge to reuse the stronger global-install detector already on main and cover the custom Bun install case raised in review.

## Best Possible Solution

Refresh this PR rather than close it: make `checkUpdateStatus` determine git vs package first, keep repo package-manager detection for git roots, and use or adapt `detectGlobalInstallManagerForRoot` for package roots. Add regression coverage for an npm-global OpenClaw package whose `package.json` declares pnpm, plus custom `BUN_INSTALL` coverage, then merge this PR or replace it with that smaller implementation.
