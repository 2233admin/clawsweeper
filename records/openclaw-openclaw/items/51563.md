## Summary

Keep this PR open. Current main still uses the package root's `package.json.packageManager` to populate `openclaw update status` for non-git package installs, so the PR's intended bug fix is not implemented on main even though linked #51401 is closed. The PR is focused and has no obvious security-sensitive side effects, but its implementation should likely be refreshed to reuse or adapt the newer global-install detection helper instead of adding separate path-only heuristics.

## Best Possible Solution

Keep this PR open, but refresh the implementation before merge: make `checkUpdateStatus` keep `detectPackageManager` for git checkouts while using a global-install detector for package roots, preferably by reusing or adapting `detectGlobalInstallManagerForRoot` from `src/infra/update-global.ts` so npm, pnpm, Homebrew/npm-prefix, Windows npm, and custom Bun roots are handled consistently. Add regression coverage for an npm global package root whose package.json declares pnpm and for Bun custom `BUN_INSTALL`, then review or supersede this PR with that implementation.
