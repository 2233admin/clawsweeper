## Summary

Keep open. Current main still lacks archive-aware `chat.history` and `sessions_history` support, so the branch is not obsolete, but the proposed implementation needs maintainer review and at least one metadata correctness fix before merge.

## What This Changes

Adds an optional `includeArchived` selector to `chat.history` and `sessions_history`, chains `.reset.<timestamp>` archives before the active transcript with synthetic reset markers, and updates Swift protocol models plus focused tests.

## Best Possible Solution

Land an additive archive-history read path only after the response metadata stays monotonic across archives, reset boundaries, and the primary transcript, and after maintainers confirm that explicit archive visibility is the intended product/security boundary for both RPC callers and agent tools.

## Reproduction Assessment

Yes. The current-main gap is directly reproducible from source: `includeArchived` is rejected by the schema, ignored by the handler, and unavailable in `sessions_history`; the PR finding is reproducible from the diff by chaining two archives, each of which starts `messageSeq` from zero.

## Solution Assessment

No, not as-is. The opt-in API shape is a plausible narrow solution, but the implementation should preserve continuous `__openclaw.seq` metadata and get reset-archive visibility semantics reviewed before it is the best maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Keep archive history sequence numbers continuous:** `src/gateway/session-utils.fs.ts:219`
  - body: `messageSeq` is initialized inside the per-archive helper, so `includeArchived=true` with multiple reset archives returns duplicate `__openclaw.seq` values, and the primary transcript starts at 1 again too. Current `readSessionMessages` gives callers monotonically increasing sequence metadata within one history response; duplicate sequence values can break consumers that use this metadata for cursors, deduplication, or history projections. Thread a shared sequence offset through archive reads, reset markers, and the primary segment.
  - confidence: 0.82
