## Summary

Keep open. Current main still does not warn during `cron edit --model`; it trims the model, writes it into the cron update payload, and sends `cron.update`. The disallowed-model check still happens later at isolated cron execution time, which is the user-facing DX gap this PR is trying to fix. The PR remains a plausible implementation candidate, but the review feedback about matching runtime model resolution needs to be addressed before merge.

## Best Possible Solution

Keep this PR open or supersede it with a narrower fix that performs the warning in the target gateway/runtime context, reuses `resolveCronModelSelection` or equivalent `resolveAllowedModelRef` semantics, and adds focused coverage for disallowed models plus valid aliases, shorthand refs, auth-profile suffixes, configured default providers, and remote-gateway config behavior.
