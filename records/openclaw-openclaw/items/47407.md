## Summary

Keep open. Current main still does not implement the PR's intended edit-time warning: `cron edit --model` trims the model, writes it into the cron update payload, and sends `cron.update`; the allowlist rejection still happens later during isolated cron execution. The PR remains a plausible implementation candidate, but its own review history flags correctness issues around matching runtime model resolution and target-gateway config semantics.

## Best Possible Solution

Keep this PR open, or replace it with a narrower fix that performs the warning against the target gateway/runtime context, reuses `resolveAllowedModelRef` or equivalent cron model-selection semantics, and adds focused CLI coverage for disallowed models plus valid aliases, bare refs, configured default providers, and auth-profile suffixes.
