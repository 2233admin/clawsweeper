## Summary

Keep this PR open. Current main still omits the model-derived max-token fallback in both OpenAI-compatible request builders, so the agents bug remains valid; however, the PR is not merge-ready because it bundles unrelated TUI changes that introduce typecheck and behavior regressions.

## What This Changes

This PR makes the OpenAI Responses and Completions request builders fall back from missing `options.maxTokens` to `model.maxTokens`, and also refactors TUI event handling/tests for orphaned streaming status.

## Best Possible Solution

Land a narrow agents-only fix that resolves a positive effective max-token value from explicit options first and model metadata second, applies it through the existing Responses/Completions compat fields, and adds omitted-option regression tests; handle TUI orphan-streaming work separately against current main.
