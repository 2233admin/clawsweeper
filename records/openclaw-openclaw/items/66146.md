## Summary

Keep open. Current main still omits OpenAI-compatible max-token fields when `options.maxTokens` is absent, so the central agents transport bug is not implemented; however this PR should not merge as-is because it also carries unrelated stale TUI event-handler changes that break the handler contract and regress local run/error status behavior.

## What This Changes

This PR makes the OpenAI Responses and Completions request builders fall back from missing `options.maxTokens` to `model.maxTokens`, and also changes TUI event handling/tests for orphaned streaming runs.

## Best Possible Solution

Land a narrow agents-only fix that resolves a positive effective max-token value from explicit options first and model metadata second, applies it through the existing Responses/Completions compat fields, and adds omitted-option regression tests; keep the TUI orphan-streaming work separate from this PR.
