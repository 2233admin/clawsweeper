## Summary

Keep open. Current main still only logs Mattermost REST health-check failures without terminating the WebSocket, so the PR targets a real recovery gap, and the Greptile counter-reset issue was addressed. The branch still introduces one concrete type-safety regression by making the required options object optional, which should be fixed before merge.

## What This Changes

The PR adds a Mattermost WebSocket health-check failure threshold that terminates the socket after consecutive `getBotUpdateAt` failures and resets the failure counter after successful checks.

## Best Possible Solution

Merge a revised Mattermost plugin fix that keeps `createMattermostConnectOnce` options required, adds focused unit coverage for consecutive health-check failure recovery, and coordinates the final monitor shape with the separate ping/pong keepalive work in #73979.
