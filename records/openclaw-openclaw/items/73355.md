## Summary

Keep this PR open. Current main still does not implement the PR's central runtime recovery changes: the cron timer catch handlers only log, there is no independent setInterval watchdog lifecycle, and the PR is paired with the same-author open bug report #73166 through closing syntax. The patch also has unresolved review concerns around double-arming timers and a test that does not exercise the catch path.

## Best Possible Solution

Keep the PR open and review it together with #73166. The best path is to address the double-arm and test-coverage concerns, or replace the patch with an equivalent narrowly tested runtime recovery design in `src/cron/service/timer.ts` plus lifecycle wiring in `ops.ts` and `state.ts`.
