## Summary

Keep open. Current main partially addresses the microphone part by allowing same-origin microphone access, but it still does not provide configurable Control UI Permissions-Policy or CSP headers, and the hardcoded CSP still blocks custom Control UI scripts from fetching arbitrary local inference endpoints.

## Best Possible Solution

Keep this open for maintainer product/security review. The best path is an explicit, validated `gateway.controlUi` header policy surface, likely narrow allowlists for safe directives such as `connect-src` additions and Permissions-Policy feature values, with config schema/help/docs/tests updated together. Avoid a raw arbitrary CSP string unless maintainers intentionally accept that security tradeoff.
