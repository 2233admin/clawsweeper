## Summary

Keep open. Current main still has a fixed Control UI CSP and fixed default Permissions-Policy behavior, and the config schema/docs do not expose a supported way to add custom local `connect-src` origins or policy directives. The same-origin microphone change only covers part of the example; the remaining configurable header-policy request needs maintainer security/API review.

## What This Changes

The issue requests configurable Control UI CSP and Permissions-Policy header settings under `gateway.controlUi` so custom Control UI roots can opt into needed browser permissions and connection origins.

## Best Possible Solution

Add a maintainer-approved, narrow, typed Control UI header-policy config in the gateway owner area, with strict directive/origin validation, generated config metadata, public docs, and focused tests for CSP serialization, Permissions-Policy serialization, config rejection, and HTTP header behavior.
