## Summary

Keep open. Current main still hardcodes the Control UI CSP and Permissions-Policy, and the strict `gateway.controlUi` config contract has no field for policy overrides or local `connect-src` additions. The same-origin microphone change only addresses part of the example; the remaining configurable-header request is valid but security-sensitive and needs maintainer API-shape review.

## What This Changes

Request to let `gateway.controlUi` configure Control UI Permissions-Policy and CSP directives for custom UI roots, including microphone and local `connect-src` needs.

## Best Possible Solution

Add a narrow, typed Control UI header policy config in the gateway owner area, with strict validation, generated config metadata, public docs, and focused tests for CSP serialization, Permissions-Policy serialization, schema rejection, and HTTP header behavior. Prefer explicit allowlisted directives and origins unless maintainers intentionally approve a broader escape hatch.
