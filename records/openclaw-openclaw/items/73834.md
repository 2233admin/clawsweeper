## Summary

Keep this PR open. Current main still lacks the PR's intended Google Meet realtime barge-in behavior: no transport-callable realtime `handleBargeIn()` seam, no Google Meet side-channel microphone command, no loopback input suppression during assistant playback, and no `chrome.bargeIn*` config/docs/manifest surface. The linked #73850 issue is open and treats this PR as the implementation candidate, so maintainer review or reshaping is the right cleanup path.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether the additive realtime `handleBargeIn()` contract is acceptable, keep the Google Meet side-channel microphone command optional and plugin-owned, preserve existing external bridge behavior, require focused Google Meet/OpenAI realtime tests plus SDK/config/docs checks, and do a small security review of the added local-command trust surface before merge.
