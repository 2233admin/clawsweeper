## Summary

Keep this PR open. Current main still lacks the requested Google Meet side-channel barge-in behavior and transport-callable realtime hook, but the proposed patch should not merge as-is because it suppresses the default command-pair input path even when no side-channel microphone command is configured.

## What This Changes

The PR adds an optional Google Meet Chrome barge-in microphone command, a realtime `handleBargeIn()` bridge hook, loopback-input suppression during assistant playback, config/schema/docs metadata, and focused Google Meet/OpenAI realtime tests.

## Best Possible Solution

Keep the PR open, preserve the additive realtime/provider direction, and reshape the Google Meet implementation so loopback suppression is tied to an active side-channel barge-in path or otherwise preserves existing command-pair human-audio forwarding. Review the contract and local-command trust model before merge.

## Reproduction Assessment

Yes. A static command-pair reproduction is clear from the PR diff: with default config and no `chrome.bargeInInputCommand`, provider output sets `suppressInputUntil`, and the next input stdout chunk returns before `bridge.sendAudio(...)`.

## Solution Assessment

No, not as-is. The side-channel microphone plus optional provider hook is a plausible narrow solution, but the patch must avoid disabling the existing input path for setups that have not opted into the side channel.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Gate input suppression on side-channel barge-in:** `extensions/google-meet/src/realtime.ts:384-388`
  - body: Because `chrome.bargeInInputCommand` is optional, default command-pair setups still have only the shared input command for human speech. This new suppression block drops that input during assistant playback, so those users lose provider-side VAD/barge-in instead of preserving existing behavior. Only suppress the shared input when the side-channel monitor is active, or keep forwarding input when no side channel is configured.
  - confidence: 0.86
