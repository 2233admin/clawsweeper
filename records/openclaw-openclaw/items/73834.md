## Summary

Keep this PR open. Current main has the earlier provider-side clear-output fix from #72524, but it does not implement this PR's distinct follow-up: an explicit realtime bridge `handleBargeIn()` seam, a Google Meet Chrome side-channel microphone monitor, input suppression during assistant playback, or the new `chrome.bargeIn*` config/docs surface. The PR is draft and security-relevant only in the normal local-command sense: it adds an operator-configured microphone capture command, with no CI/dependency/workflow/lockfile changes in the diff.

## Best Possible Solution

Keep the draft PR available for maintainer review. The best path is to validate the new `handleBargeIn` seam as an additive realtime-voice contract, add any missing Google Meet manifest/config metadata for the new `chrome.bargeIn*` keys, preserve the local-command security model in docs, and then run focused Google Meet/OpenAI realtime tests plus the appropriate plugin SDK/config checks before considering merge.
