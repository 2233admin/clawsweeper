## Summary

Keep this PR open for revision. Current main has partial crash-hardening overlap, but it does not fully implement the PR's raw upgrade-socket handling, and the PR still introduces a blocking uncaught-exception ordering bug that can suppress fatal wrappers with transient nested causes.

## What This Changes

The PR adds strict transient network/TLS suppression to the global uncaughtException handlers, attaches a raw upgrade-socket error listener, and adds classifier unit coverage.

## Best Possible Solution

Preserve the crash-hardening intent, but implement it through a narrow uncaught-exception policy that keeps fatal/config-style wrappers fatal, targets concrete TLS/undici/socket signatures or the raw socket source, and adds regression coverage for both CLI entrypoints plus the upgrade-socket path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fatal precedence before suppressing uncaught errors:** `src/cli/run-main.ts:164-169`
  - body: This handler now returns for isStrictTransientNetworkError(error) before any fatal/config-style classification. Because the strict classifier walks nested causes, a fatal wrapper such as ERR_WORKER_UNCAUGHT_EXCEPTION with a transient network cause can be logged as non-fatal and leave the CLI/gateway running after a fatal exception. Keep fatal checks ahead of transient suppression or make the strict check ignore fatal wrappers.
  - confidence: 0.86
