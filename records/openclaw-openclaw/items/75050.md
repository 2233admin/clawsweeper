## Summary

Keep this PR open. It addresses a real Feishu/Lark mixed-proxy docs gap related to #72595, but the current snippet can mislead installed-gateway users because transient PowerShell proxy variables are not persisted into the managed service environment by `openclaw gateway restart`.

## What This Changes

The PR adds Feishu/Lark mixed-proxy troubleshooting guidance to `docs/channels/feishu.md`, including `NO_PROXY` PowerShell examples, verification steps, and sensitive-log guidance.

## Best Possible Solution

Revise the troubleshooting section to document `NO_PROXY` as the narrow workaround while clearly distinguishing how the gateway process receives proxy variables for manual launches versus managed services; leave #72595 open for any future channel-level proxy policy.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable for the docs addition itself, but yes for the review finding: static inspection of the added snippet plus current service env filtering and restart code shows why `openclaw gateway restart` will not carry one-off PowerShell variables into an installed gateway service.

## Solution Assessment

Is this the best way to solve the issue? No as written: documenting `NO_PROXY` is a good narrow docs fix, but the examples need to explain the gateway launch environment instead of implying that a restart applies transient shell variables to the managed service.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Clarify how the proxy env reaches the gateway:** `docs/channels/feishu.md:235`
  - body: The examples set `$env:HTTPS_PROXY`, `$env:HTTP_PROXY`, and `$env:NO_PROXY` in the current PowerShell session and then run `openclaw gateway restart`. For an installed gateway service, OpenClaw filters these proxy keys out of managed service env sources and restart does not rebuild the service environment from transient shell variables, so users can follow the snippet and see no routing change. Please split manual process launch guidance from persistent managed-service or wrapper-environment guidance.
