## Summary

Keep open. The docs gap is still real on current main: runtime and schema constrain `commands.bashForegroundMs` to 0..30000 ms, but the user-facing slash-command docs and generated help text still omit the upper bound. This PR is not merge-ready as-is because its patch adds about 2693 lines to `docs/gateway/configuration-reference.md`, which current docs explicitly scope away from deep command catalog detail.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but request a narrow docs update instead of merging the current diff. The best fix is to update `docs/tools/slash-commands.md` and `src/config/schema.help.ts` so `commands.bashForegroundMs` is documented as integer milliseconds from 0 to 30000, default 2000, with `0` backgrounding immediately; regenerate or check generated config docs/schema artifacts as required. A concise note in `docs/gateway/config-channels.md` is reasonable if maintainers want it, but avoid duplicating the command catalog in `docs/gateway/configuration-reference.md`.
