## Summary

Keep this PR open. Current main still lacks effective iMessage inbound untrusted-content wrapping on the model-bound path, so the hardening topic is not obsolete; however, this patch is not merge-ready because it wraps only legacy envelope `Body` text while raw `BodyForAgent` continues to drive prompt assembly.

## What This Changes

The PR imports `wrapExternalContent` into iMessage inbound processing and wraps current-message and group-history body strings passed to `formatInboundEnvelope(...)`.

## Best Possible Solution

Revise this PR or replace it with a maintainer-reviewed iMessage hardening patch that protects the actual model-bound path through `UntrustedContext` or another prompt-bound field, keeps `RawBody` and `CommandBody` raw for command parsing, preserves reply/history/echo behavior intentionally, and adds focused iMessage tests.
