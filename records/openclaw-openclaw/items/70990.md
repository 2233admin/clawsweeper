## Summary

Keep this PR open. Current main still lacks the proposed `model_failover` and `model_failure_terminal` typed plugin hooks, and external plugins cannot observe those failover seams without a core hook-name contract, runner methods, and emission points. The PR remains a plausible narrow core plugin API change, but it needs maintainer review for payload shape, docs, and privacy gating before merge.

## What This Changes

The PR adds typed `model_failover` and `model_failure_terminal` plugin hooks, emits them from embedded failover and terminal before-reply failure paths, and adds hook-runner/failover-observation tests.

## Best Possible Solution

If maintainers accept this API, land it as a narrow core plugin hook addition: add the hook names, typed payloads, runner methods, failover and terminal emission points, tests, public docs, and an explicit privacy/access policy. Keep broader user-facing fallback notices, workspace-context re-injection, configurable exhausted-model messages, and circuit-breaker behavior tracked separately under #65824 or #39142.
