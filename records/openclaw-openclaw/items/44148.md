## Summary

Keep open. Current main still has WhatsApp self-chat/fromMe echo handling, but I found no implementation of the requested human-takeover behavior: a manual outgoing WhatsApp message from the bot number is not recorded as a per-conversation pause signal, and there is no typed `humanTakeover` or pause-duration config surface for WhatsApp.

## Best Possible Solution

Keep this issue open and implement the behavior in the bundled WhatsApp plugin. The likely path is to add a typed `channels.whatsapp.humanTakeover` config/schema/docs surface, detect non-gateway `fromMe` outgoing messages per WhatsApp account and conversation, persist a pause-until timestamp, and have the auto-reply path skip replies while the pause is active. Add focused WhatsApp tests for direct chats and any intended group behavior; introduce a shared core seam only if another bundled channel needs the same takeover contract.
