## Summary

Keep open. Current main still has WhatsApp self-chat and gateway-echo handling, but it does not implement the requested human-takeover behavior: there is no `humanTakeover` config surface, non-gateway outgoing `fromMe` DMs are dropped instead of recorded as a pause signal, and later customer replies still flow to the auto-reply path.

## Best Possible Solution

Keep this issue open and implement the behavior in the bundled WhatsApp plugin. Add a typed `channels.whatsapp.humanTakeover` config surface, detect non-gateway `fromMe` outgoing messages per account/conversation, record a pause-until timestamp, and make the WhatsApp auto-reply path skip replies while that pause is active. Cover direct chats first, define intended group behavior explicitly, and update docs/config metadata with the new user-facing option.
