## Summary

Keep this issue open. Current main has adjacent macOS status, HoverHUD, WebChat, Canvas, Talk, and session-menu surfaces, but it still does not implement configurable always-on-top floating bubbles per running agent with persisted positions, drag/anchor or mouse-follow behavior, and click-to-chat routing per agent/session.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-owned macOS product decision: either implement a native app surface under apps/macos/Sources/OpenClaw that derives bubble state from running agent/session activity, persists positions, supports drag/anchor or mouse-follow behavior, and opens chat routed to the selected agent/session, or first design a native macOS UI/window plugin API that would make an external ClawHub implementation feasible.
