## Summary

Keep open. Current main still has partial fallback/status/cron improvements, but it does not satisfy the central production execution-discipline requests: startup primary-model reachability is deliberately non-blocking, the exposed diagnostics surface does not provide structured runtime model/fallback/last-provider-error state, and cron creation still lacks a dry-run/preflight gate that resolves announce delivery before persisting the job.

## What This Changes

The issue requests production-operability improvements for model fallback visibility, startup provider health checks, agent-readable runtime diagnostics, cron creation validation/dry-run behavior, and clearer pre-context error reporting.

## Best Possible Solution

Keep this open as the production execution-discipline tracker or split it into canonical follow-ups: a configurable startup provider preflight/degraded-mode policy, a safe structured runtime diagnostics RPC or agent tool for selected model, active responding model, fallback state, and last provider error, and cron add dry-run/preflight delivery resolution that fails closed before persistence.
