## Summary

Keep open. Current main still routes the Control UI New Session button and typed `/new`/`/reset` through `chat.send` on the active session key, while the independent fresh-session path is `sessions.create`; open PR #73725 covers the New Chat button slice but is unmerged and intentionally leaves typed slash semantics explicit.

## What This Changes

The issue requests that Control UI `/new`, `/reset`, and New Session create a genuinely fresh independent session matching direct session creation.

## Best Possible Solution

Review and land or close #73725 for the New Chat button slice, then explicitly decide whether Control UI typed `/new` and `/reset` should also be intercepted locally to call `sessions.create` and `switchChatSession`. If maintainers want the report fully fixed, keep gateway `chat.send` reset semantics in-place and make only the Control UI-owned fresh-start UX create/switch to a dashboard session.
