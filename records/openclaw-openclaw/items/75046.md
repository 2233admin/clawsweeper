## Summary

Keep open. Current main does not implement a Cursor SDK core backend, and this active contributor PR is paired with an open feature issue, but the proposed patch is not ready to merge because it has concrete runner defects plus product, security, and supply-chain decisions that need maintainer review.

## What This Changes

The PR adds `@cursor/sdk` as a root dependency and wires a new `cursor-sdk` core runner, config block, auth env var, dispatch branches, and tests into chat reply, agent command, and cron execution paths.

## Best Possible Solution

Decide the Cursor SDK ownership boundary first. If maintainers want this in core, implement it as a documented runtime/harness integration that preserves OpenClaw cancellation, timeout, fallback, hooks, status semantics, auth UX, and security policy; otherwise keep Cursor on the existing ACP/acpx or plugin/ClawHub path.

## Reproduction Assessment

Yes for the review findings: static inspection of the PR head plus the `@cursor/sdk@1.0.7` public types is enough to reproduce the status-handling, configured-model, cloud-routing, and cancellation/timeout gaps. A live Cursor API key is not needed for those code paths.

## Solution Assessment

No. The proposed core backend is not the best merge path as-is because it bypasses the documented external-harness route and leaves several OpenClaw runtime contracts incomplete; the safer alternative is maintainer-approved runtime/plugin design followed by a narrower implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Check Cursor run terminal status:** `src/agents/cursor-sdk-runner.ts:163-165`
  - body: `run.wait()` returns a terminal status that can be `error` or `cancelled`, but the runner ignores it and returns a normal OpenClaw success from whatever text was collected. Convert non-`finished` statuses into cancellation or failover so failed Cursor runs do not look successful.
  - confidence: 0.91
- **[P2] Honor the configured Cursor SDK model:** `src/agents/cursor-sdk-runner.ts:117`
  - body: The schema exposes `agents.defaults.cursorSdk.model`, but `modelId` only reads `params.model` and otherwise falls back to `composer-2`. Users who configure the new Cursor SDK model field will silently get the default model unless a caller passes an override.
  - confidence: 0.88
- **[P2] Respect cloud runtime without a cloud block:** `src/agents/cursor-sdk-runner.ts:136-150`
  - body: `cursorSdk.runtime: "cloud"` is valid without a `cloud` object, but this condition falls through to local options when the block is omitted. A user explicitly selecting cloud would silently run locally; build cloud options whenever runtime is cloud, using defaults for optional cloud fields.
  - confidence: 0.86
- **[P2] Cancel timed-out or aborted Cursor runs:** `src/agents/cursor-sdk-runner.ts:156-163`
  - body: The runner accepts `timeoutMs` but never enforces it around `send`, `stream`, or `wait`, and its params do not accept the existing auto-reply abort signal. A hung Cursor run or user stop can leave execution stuck instead of cancelling the SDK run and failing over.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR adds a new third-party local/cloud agent harness, a root dependency, token env-var wiring, and a non-exact dependency override in core.

Concerns:

- **[medium] Map OpenClaw policy before running Cursor locally:** `src/agents/cursor-sdk-runner.ts:149`
  - body: The local path creates a Cursor agent for the workspace directory, but the patch does not map OpenClaw exec approvals, sandbox policy, plugin hooks, aborts, or timeout controls into Cursor's file/terminal/MCP-capable harness. That can make `cursor-sdk` an opt-in bypass around OpenClaw-owned execution controls.
  - confidence: 0.84
- **[medium] Review the new SDK supply-chain footprint:** `package.json:1614`
  - body: Adding `@cursor/sdk` at the root brings Cursor's SDK and its transitive runtime dependencies into every OpenClaw install. That may be acceptable, but it is a concrete supply-chain expansion and should be a maintainer-approved dependency decision.
  - confidence: 0.8
- **[low] Keep dependency overrides deterministic:** `package.json:1706`
  - body: The PR changes the root `undici` override from the exact `8.1.0` pin to `>=6.24.0`. Even with the current lockfile, broadening a security override should get explicit dependency-owner review because future resolution is less deterministic.
  - confidence: 0.78
