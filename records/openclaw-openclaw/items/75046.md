## Summary

Keep this PR open. The revised extension-only direction is closer to the project boundary than the earlier core backend version, but the patch still is not merge-ready: it registers only a harness without the provider/model/auth surface current main requires before a harness can run, and the Cursor runner still has timeout/abort gaps plus unresolved security and supply-chain review needs.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` plugin with an isolated `@cursor/sdk` dependency, manifest config, AgentHarness registration, Cursor runner/error-classification code, tests, lockfile changes, and a root `undici` pnpm override.

## Best Possible Solution

Maintainers should first decide whether Cursor SDK belongs as a bundled trusted plugin or as an external ClawHub/npm plugin. If bundled, the implementation should register a provider/auth/model catalog or explicitly document and test the required `models.providers` config, preserve OpenClaw abort/timeout/fallback semantics, document the runtime support contract, and land only after dependency/security review.

## Reproduction Assessment

Yes for the review findings. Static current-main tracing shows model resolution happens before the harness attempt, and static PR-head tracing shows the Cursor runner does not observe `params.abortSignal` and does not keep a single timeout around send/stream/wait; no live Cursor key is needed for those control-flow checks.

## Solution Assessment

No, this PR is not the best merge path as-is. The extension-only shape is a better boundary than the original core backend, but the patch needs provider/auth/catalog wiring, runtime policy handling, and security review before it can safely solve the linked feature request.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Register a provider before claiming cursor-sdk:** `extensions/cursor-sdk/index.ts:9`
  - body: The plugin only calls `api.registerAgentHarness(...)`, but current main resolves provider/model before invoking a plugin harness and throws when no model is found. With the advertised `CURSOR_API_KEY` plus `cursor-sdk` provider, there is no provider catalog, dynamic model resolver, or auth metadata to produce a `cursor-sdk/composer-2` model, so the first turn fails before this runner is reached.
  - confidence: 0.9
- **[P2] Keep the Cursor timeout active through send and wait:** `extensions/cursor-sdk/src/run-attempt.ts:97-104`
  - body: The timeout starts only after `agent.send()` returns and is cleared before `run.wait()`. Cursor SDK initialization is deferred until the first send, so a stalled send or terminal wait can exceed `params.timeoutMs` without cancellation or failover.
  - confidence: 0.88
- **[P2] Cancel Cursor runs when OpenClaw aborts:** `extensions/cursor-sdk/src/run-attempt.ts:97-104`
  - body: `runCursorSdkAttempt` never observes `params.abortSignal`. Existing OpenClaw stops and cron aborts therefore cannot cancel a Cursor run promptly; pass the signal through, race send/stream/wait, and call `run.cancel()` or dispose when it aborts.
  - confidence: 0.84
