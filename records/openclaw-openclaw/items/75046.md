## Summary

Keep this PR open. It is an active contributor feature PR paired with open #75044 and assigned to a maintainer, but the current patch still has blocking runtime-contract issues and security/dependency policy questions before it can be merged.

## What This Changes

The PR adds a bundled `cursor-sdk` plugin with an AgentHarness, Cursor SDK runner, manifest/config schema, tests, an isolated `@cursor/sdk` package, lockfile updates, and a workspace `undici` override.

## Best Possible Solution

Maintainers should first decide the product boundary. If this becomes a bundled plugin, it should add a real provider/auth/model catalog contract, preserve OpenClaw abort and timeout semantics across Cursor send/stream/wait, document the local/cloud runtime trust boundary, and land only after dependency and security review.

## Reproduction Assessment

Yes for the review findings. Static tracing on current `main` plus the PR head shows model resolution happens before the harness attempt, the timeout excludes `agent.send()` and `run.wait()`, and the runner never observes `params.abortSignal`.

## Solution Assessment

No, this PR is not the best merge path as-is. The extension-only direction is plausible, but the current implementation needs provider/catalog/auth wiring, runtime policy handling, and dependency/security approval before it safely solves the linked feature request.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Register a provider before claiming cursor-sdk:** `extensions/cursor-sdk/index.ts:9`
  - body: The plugin only registers an AgentHarness. Current main resolves provider/model/auth before invoking a harness, so `cursor-sdk/composer-2` has no provider catalog, dynamic model resolver, or auth metadata and can fail with `model_not_found` before this runner is reached.
  - confidence: 0.9
- **[P2] Keep the Cursor timeout active through send and wait:** `extensions/cursor-sdk/src/run-attempt.ts:99-124`
  - body: The timeout starts only after `agent.send()` returns and is cleared before `run.wait()`. Cursor SDK initialization is deferred until first send, so a stalled startup/send or terminal wait can exceed `params.timeoutMs` without cancellation or failover.
  - confidence: 0.88
- **[P2] Cancel Cursor runs when OpenClaw aborts:** `extensions/cursor-sdk/src/run-attempt.ts:99-106`
  - body: `runCursorSdkAttempt` never observes `params.abortSignal`. `/stop`, cron aborts, and gateway shutdown can leave Cursor work running until upstream completion; register an abort listener and cancel/dispose the SDK run promptly.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR adds a trusted local/cloud Cursor agent runtime, new native/runtime dependencies, and a workspace-wide dependency override without a completed policy mapping.

Concerns:

- **[medium] Map OpenClaw policy before local Cursor runs:** `extensions/cursor-sdk/src/run-attempt.ts:96`
  - body: The local path creates a Cursor SDK agent in the workspace and lets Cursor own file, shell, MCP, and subagent behavior, but the diff does not document or enforce how OpenClaw approvals, sandbox policy, aborts, and timeout controls apply to that execution surface.
  - confidence: 0.84
- **[medium] Review the Cursor SDK native dependency footprint:** `extensions/cursor-sdk/package.json:7`
  - body: `@cursor/sdk` brings sqlite3, ConnectRPC, and optional platform-specific Cursor SDK packages into a bundled staged-runtime plugin. That may be acceptable, but it is a concrete supply-chain expansion that should get maintainer dependency review before bundling.
  - confidence: 0.82
- **[low] Keep the undici override deterministic:** `package.json:1706`
  - body: The PR adds `undici: >=6.24.0` as a workspace-wide pnpm override. A broad range can affect future unrelated resolution and should be replaced with an exact reviewed pin or handled by the dependency owner.
  - confidence: 0.79
