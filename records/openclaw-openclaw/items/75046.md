## Summary

Keep open for maintainer review. Current main does not implement a core Cursor SDK backend, and this PR is paired with an open same-author feature issue, but the proposed patch has blocking runner correctness issues plus a product/security boundary question around adding a new local/cloud Cursor harness directly to core.

## What This Changes

This PR adds `@cursor/sdk` as a root dependency and wires a new `cursor-sdk` core runner, config block, auth env var, dispatch branches, and tests into chat reply, agent command, and cron execution paths.

## Best Possible Solution

Decide the Cursor SDK ownership boundary first. If maintainers want this in core, it should be implemented as a documented runtime/harness integration that preserves OpenClaw cancellation, timeout, fallback, hooks, security policy, and status semantics; otherwise the capability should be pursued through ACP/acpx or a plugin/ClawHub path.

## Reproduction Assessment

Yes for the review findings: static inspection of the PR head plus the `@cursor/sdk@1.0.7` public types is enough to reproduce the status-handling, create-error, cloud-routing, and cancellation gaps. A live Cursor API key is not needed to verify those code paths.

## Solution Assessment

No. The proposed core backend is not the best merge path as-is because it bypasses established external-harness guidance and has concrete runner defects; a safer alternative is maintainer-approved runtime/plugin design followed by a narrower implementation with docs and policy parity.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle failed Cursor run statuses:** `src/agents/cursor-sdk-runner.ts:146-148`
  - body: `run.wait()` returns a `RunResult` whose status can be `error` or `cancelled`, but the runner ignores `result.status` and returns a normal OpenClaw success even when Cursor reports a failed or cancelled run. Please convert non-`finished` statuses into the appropriate cancellation or `FailoverError` path so fallback and user-facing errors still work.
  - confidence: 0.9
- **[P2] Classify Agent.create failures:** `src/agents/cursor-sdk-runner.ts:136`
  - body: `Agent.create(...)` runs before the `try` block, so SDK `AuthenticationError`, `RateLimitError`, cloud configuration errors, or local runtime startup failures thrown during creation bypass the PR's failover classification entirely. Wrap creation in the same classified error path, disposing only when an agent handle was actually created.
  - confidence: 0.86
- **[P2] Respect cloud runtime without a cloud block:** `src/agents/cursor-sdk-runner.ts:120-134`
  - body: The schema allows `cursorSdk.runtime: "cloud"` while omitting `cursorSdk.cloud`, but this condition falls through to the local-agent options in that case. A user who explicitly selects cloud would silently run Cursor locally; build cloud options whenever runtime is `cloud`, using defaults for optional cloud fields.
  - confidence: 0.84
- **[P2] Wire cancellation and timeouts into Cursor runs:** `src/agents/cursor-sdk-runner.ts:139-146`
  - body: The new runner accepts `timeoutMs` but never enforces it around `agent.send`, `run.stream`, or `run.wait`, and the auto-reply branch does not pass the existing abort/reply-operation controls that CLI runs use. A hung Cursor stream or user `/stop` can leave the reply/cron execution stuck instead of cancelling the SDK run and failing over.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces a new third-party local/cloud agent harness in core and needs explicit maintainer security review before merge.

Concerns:

- **[medium] Map OpenClaw policy before running Cursor locally:** `src/agents/cursor-sdk-runner.ts:133`
  - body: The local Cursor SDK path creates an agent against the workspace directory, but the patch does not map OpenClaw exec approvals, sandbox policy, plugin hooks, or cancellation into Cursor's file/terminal-capable harness. That can make `cursor-sdk` an opt-in bypass around OpenClaw-owned execution controls.
  - confidence: 0.83
- **[medium] Review new SDK supply-chain footprint:** `package.json:1611`
  - body: Adding `@cursor/sdk` at the root brings Cursor's beta SDK plus optional platform packages into the core install. That may be acceptable, but it is a concrete supply-chain expansion and should be a maintainer-approved dependency decision, not an incidental feature detail.
  - confidence: 0.78
