## Summary

Current main still does not implement the requested Discord/session transcript claim-check. It has useful partial pieces: Discord saves inbound media to disk and emits MediaPath metadata, Gateway chat.send has a media:// offload path, and old replay/history views can prune inline image blocks, but the active agent path still hydrates image refs back into inline base64 ImageContent and the persistence guard does not replace user image blocks with durable media references before session append.

## What This Changes

Persist inbound Discord image turns as compact media references instead of embedding base64 image payloads in session transcripts, then hydrate bounded image data from disk when building model context.

## Best Possible Solution

Keep this open until OpenClaw has a backward-compatible session transcript claim-check for inbound image content: persist compact media reference/metadata instead of image.data, hydrate only bounded recent references when building model context, keep old inline-base64 transcripts readable through sanitizers, and align with the Gateway chat-history redaction work in #43982.

## Repair Work Prompt

Fix the remaining transcript bloat from inbound Discord/media images. Current main saves Discord media to disk and emits MediaPath notes, then the embedded runner rehydrates those notes into inline { type: "image", data, mimeType } blocks and the session persistence guard does not replace user image blocks before JSONL append. Implement a backward-compatible persistence claim-check/redaction so inbound image turns do not persist base64 image.data in session transcripts, while the active turn can still hydrate bounded image data for the model. Prefer a generic agent/session persistence seam over Discord-specific core IDs. Keep existing base64 transcripts readable and sanitized; do not break Gateway chat.send media:// offload behavior or the open Gateway history redaction direction in #43982. Add focused regression tests proving the original base64 payload is absent from persisted session entries and that prompt hydration/order still works for media notes/refs.
