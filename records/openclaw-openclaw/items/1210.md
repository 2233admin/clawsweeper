## Summary

Keep open: current main has replay-time pruning, media offload, and image-size mitigations, but it still does not implement the requested session transcript claim-check before JSONL persistence. The issue remains valid and narrow enough for a focused repair PR.

## What This Changes

Store inbound Discord image references in session transcripts instead of persisting base64 image data inline, then hydrate bounded recent references when building model context.

## Best Possible Solution

Add a backward-compatible session transcript claim-check for inbound image content: persist compact media reference and metadata instead of raw image.data, hydrate only bounded recent references during context construction, keep legacy inline-base64 transcripts readable, and align the visible history projection with #43982.
