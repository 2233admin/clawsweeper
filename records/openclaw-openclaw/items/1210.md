## Summary

Keep open: current main still has mitigations for replay/history size, but not the requested session transcript claim-check. Discord media is saved to disk and Gateway has adjacent offload paths, yet the agent persistence guard still permits user image content blocks with inline base64 data to be appended to JSONL transcripts.

## What This Changes

The issue requests storing compact media references for Discord/inbound images in session transcripts instead of inline base64 image payloads, then hydrating bounded image data only when building model context.

## Best Possible Solution

Add a backward-compatible session transcript claim-check for inbound image content: persist compact media reference and metadata instead of image.data, hydrate only bounded recent references during model-context construction, keep legacy inline-base64 transcripts readable, and coordinate the visible history redaction path with #43982.
