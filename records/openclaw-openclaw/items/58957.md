## Summary

Keep open. Current main has added explicit context-overflow handling in several paths, but the reported long-session/model-switch class is not fully solved: preflight compaction still skips heartbeat turns and also skips token-threshold evaluation when a persisted token count is fresh, which matches the later source-backed follow-up analysis in the issue.

## What This Changes

The issue requests clearer context-overflow reporting and safer compaction/recovery behavior when switching models or continuing a long session whose carried-over context is too large.

## Best Possible Solution

Keep this open and resolve it with a focused maintainer-approved plan that makes proactive compaction evaluate authoritative fresh token totals, decides the heartbeat policy explicitly, keeps context-overflow errors user-visible when recovery cannot happen, and reconciles the reserve-token threshold semantics with #66830 rather than relying on fallback routing or manual `/new` recovery.

## Reproduction Assessment

Yes for the remaining proactive-compaction dead zone: current main has deterministic source paths where heartbeat turns return before threshold checks and fresh `totalTokens` entries skip token-threshold preflight. The exact live provider model-switch failure still needs a live reproduction if maintainers want to validate the original UI symptom end to end.

## Solution Assessment

No close/implemented solution is justified. Current main has useful partial overflow handling, but the durable fix should address the preflight/fresh-token/heartbeat gates and reserve semantics, then add targeted regression tests around long sessions with configured fallback models.
