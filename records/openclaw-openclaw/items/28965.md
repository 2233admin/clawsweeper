## Summary

Keep this issue open. Current main has model status/list/set flows, model selection-source documentation, session listing/cleanup, and gateway single-session reset behavior, but it still lacks the requested first-class `openclaw models reconcile` command and scoped bulk `openclaw sessions model-reset` command. The remaining work spans model config normalization, generated registry semantics, persisted session mutation, and observability, so it needs maintainer design rather than automatic cleanup.

## Best Possible Solution

Keep this issue open until maintainers define the canonical model/config reconciliation contract or implement documented operator commands for `openclaw models reconcile` and scoped `openclaw sessions model-reset`. A good implementation would reuse existing models/sessions CLI surfaces, session-store target helpers, dry-run/JSON patterns, and add tests for clearing model/provider/auth-profile overrides without disturbing unrelated session metadata.
