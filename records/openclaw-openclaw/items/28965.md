## Summary

Keep open. Current main still has model status/list/set/scan and per-session patch/reset behavior, but it does not provide the requested first-class `openclaw models reconcile` command or scoped `openclaw sessions model-reset` operator workflow. The existing reset path intentionally preserves user-selected model/auth overrides, so the remaining request needs maintainer design rather than automatic closure.

## What This Changes

Add a model-configuration reconcile command and a scoped session model/provider/auth override reset command so default-model migrations can be made from one clear operator workflow.

## Best Possible Solution

Keep this issue open for maintainer design. Define the canonical model/config reconciliation contract first, then implement documented operator commands for `openclaw models reconcile` and scoped `openclaw sessions model-reset` with dry-run/JSON output, explicit target selection, and tests proving user/session metadata is preserved unless intentionally reset.
