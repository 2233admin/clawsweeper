## Summary

Keep open. Current main still contains the transcript rewrite/truncation suffix-replay mechanism described in the report, while `sessions_history` itself remains a bounded read path; overlapping open PRs already target the same transcript-rewrite area but are not merged.

## What This Changes

The report asks OpenClaw to prevent large `sessions_history`/tool-result truncation flows from duplicating prior assistant and toolResult entries in raw JSONL transcripts while preserving correct billing and model context.

## Best Possible Solution

Keep this open until #69486, #72511, or an equivalent maintainer-approved fix lands and is verified against the raw JSONL `sessions_history` case. The desired end state is that `sessions_history` remains a bounded read tool while transcript rewrite/truncation avoids low-value suffix replay or cleans abandoned entries without breaking append-only branch semantics.

Yes. The issue includes concrete raw JSONL evidence and a follow-up comment reproduced and localized the mechanism; current-main source still has the same truncation-to-rewrite suffix replay path. This review did not create live artifacts because it was read-only.
