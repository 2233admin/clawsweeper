## Summary

Keep open. Current main still has the reported raw transcript duplication mechanism in the tool-result truncation rewrite path: it branches at the first rewritten entry and re-appends the remaining branch suffix, including unchanged entries. `sessions_history` itself is read-only and active-branch history reduces visible duplicates, but it does not eliminate the transcript rewrite behavior this issue is tracking.

## What This Changes

The issue asks OpenClaw to stop raw JSONL transcript duplication triggered by large session-history/tool-result rewrite paths while preserving correct billing and model context behavior.

## Best Possible Solution

Keep this open and fix the maintenance rewrite path narrowly: preserve `sessions_history` as a pure read, keep active-branch history behavior, and adjust tool-result truncation/transcript rewrite so it avoids low-value deep suffix replays or rewrites only necessary tail ranges while preserving labels, compaction markers, and overflow recovery semantics.

## Reproduction Assessment

Yes. The issue includes concrete transcript evidence, a follow-up comment reproduced the mechanism on upstream main, and current main still contains the same truncation-to-rewrite branch-and-reappend path; I did not create live transcript artifacts because this review was read-only.

## Solution Assessment

Unclear. The current implementation is not the best complete solution because it still re-appends unchanged suffix entries, but the best repair requires a maintainer choice among tail-only rewrites, skip thresholds, or a lower-level transcript rewrite within the append-only session contract.
