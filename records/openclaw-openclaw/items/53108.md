Keep this PR open. Current main still has no `sandbox.docker.capAdd` config path: the public config type, Zod schema, resolved Docker config, generated schema/docs, and Docker create-arg builder only handle `capDrop` plus adjacent fields like `gpus`. The PR remains a coherent core sandbox feature, not a duplicate or already-implemented change.

## Best Possible Solution

Keep this PR open and review it as the focused core sandbox change.
