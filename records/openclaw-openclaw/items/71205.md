## Summary

Keep this PR open. Current main still has no `OPENCLAW_SKIP_STATE_DIR_HARDEN` opt-out, and config writes still call the state-dir hardening helper that chmods a matching config/state directory to `0o700`. The proposed branch is narrow and addresses the Greptile logging feedback, but it adds a security-sensitive permission-hardening escape hatch plus a user-visible env var, so it needs maintainer review, rebase/CI, and a docs decision before merge.
