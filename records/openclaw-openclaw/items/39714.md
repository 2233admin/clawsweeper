## Summary

Keep open. Current main still does not implement the PR's central Docker/DooD sandbox browser fix: there is no separate advertised/container-reachable browser bridge URL, sandbox CDP is still resolved through loopback-only assumptions, and the active plugin SDK/browser bridge contract still exposes only `baseUrl`. The draft PR is not mergeable as-is, but it remains useful bug context for a current-path rewrite.

## Best Possible Solution

Keep this PR open as useful context, or replace it with a current-path PR. The best fix is to add an explicit advertised/container-reachable browser bridge URL through `src/plugin-sdk/browser-bridge.ts` and `extensions/browser/src/browser/bridge-server.ts`, preserve the host-local `bridgeUrl` for gateway-side tools, inject the advertised URL into `buildEmbeddedSandboxInfo()`, resolve sandbox browser CDP host/port separately for Dockerized gateways, keep attach-only sandbox CDP tab creation off the persistent Playwright path when CDP becomes non-loopback, and add focused Dockerized gateway plus non-Docker regression coverage and docs.
