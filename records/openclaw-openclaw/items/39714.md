## Summary

Keep open. Current main still does not implement the PR's central Dockerized sandbox browser fix: sandbox CDP readiness and resolved CDP host remain loopback-only, the bridge exposes only a host-local baseUrl/bridgeUrl, embedded sandbox info injects that same URL, and the current plugin-sdk/browser plugin bridge contract has no advertised/container-reachable URL seam. The PR is useful context for #8273, but it is still a draft and needs a current-path rewrite plus security review before it can land.

## Best Possible Solution

Rewrite this PR against the current browser plugin path: add a backwards-compatible advertised/container-reachable bridge URL through src/plugin-sdk/browser-bridge.ts and extensions/browser runtime API, keep the host-local bridge URL for gateway-side browser tools, expose advertisedBridgeUrl on SandboxBrowserContext, inject that URL from buildEmbeddedSandboxInfo(), resolve sandbox browser CDP host/port separately for Dockerized gateways, handle attachOnly non-loopback sandbox profiles without the Playwright newPage path where appropriate, and add focused Dockerized gateway plus non-Docker regression coverage and docs.
