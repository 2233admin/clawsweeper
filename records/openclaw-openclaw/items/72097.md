## Summary

Keep open. Current main still resolves plugin tools into a concrete tool array before creating the embedded agent session, passes that fixed customTools/allowlist into the session, and documents plugin changes as requiring a Gateway restart. I found registry refresh after install, but no active-session toolset delta, per-turn plugin-tool re-resolution, or prompt-boundary update that would make newly installed plugin tools available inside an already-running agent turn. The related #44401 GatewayRestart broadcast work is adjacent but does not by itself solve tool injection.

## What This Changes

Request active agent sessions to refresh or receive newly registered plugin tools after a successful plugin install, without requiring the user to restart the Gateway or session first.

## Best Possible Solution

Add a core, turn-boundary refresh mechanism keyed off plugin registry/install changes: re-resolve plugin tools through the existing policy pipeline, update customTools, active-tool allowlists, prompt/tool inventory, and execution dispatch together, and cover install -> immediate use in an existing session with focused regression tests plus updated plugin runtime docs.
