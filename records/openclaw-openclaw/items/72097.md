## Summary

Keep open. Current main still resolves plugin-provided tools into a concrete run/session snapshot, while plugin install refreshes registry/config state and still tells users to restart the Gateway; I found no session-level tool delta, active-session re-query, or turn-boundary tool allowlist refresh path that would satisfy the requested behavior.

## What This Changes

Request that active agent sessions see newly installed plugin-provided tools after `openclaw plugins install` without requiring a Gateway or session restart.

## Best Possible Solution

Add a generic runtime lifecycle path that re-resolves plugin-provided tools after a successful install or registry refresh, applies the existing effective-tool policy pipeline, updates active sessions only at a safe turn boundary, and invalidates affected prompt/tool caches without duplicating plugin runtime activation.

## Reproduction Assessment

Yes. The issue gives a concrete install -> immediate use -> Gateway restart comparison, and current-main source inspection confirms active runs build a fixed tool snapshot during session setup while plugin install only refreshes registry/config state.

## Solution Assessment

Unclear because this issue has no patch. The requested direction is valid, but the safest implementation is likely turn-boundary re-resolution through existing policy filters rather than arbitrary mid-stream mutation of active session state.
