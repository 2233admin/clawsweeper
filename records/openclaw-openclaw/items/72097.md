## Summary

Keep open. Current main still refreshes plugin registry state after install, but active embedded agent sessions receive a fixed tool array and active tool-name allowlist during setup, while the docs still describe Gateway restart as the live plugin runtime boundary.

## What This Changes

The issue requests active agent sessions to pick up newly installed plugin-provided tools without requiring a gateway restart or new session.

## Best Possible Solution

Add a generic core lifecycle refresh that re-resolves plugin-provided tools after successful plugin install or registry refresh and applies only the allowed/effective tool definitions to active sessions at a safe turn boundary, with focused coverage for the install-to-immediate-use flow.

## Reproduction Assessment

Yes. The issue gives a concrete install -> immediate use -> restart comparison, and current-main source inspection confirms registry refresh exists while active session tool setup is fixed during run/session initialization.

## Solution Assessment

Unclear as a patch because this issue has no implementation diff. The requested direction is sound, but the safest implementation should refresh at a turn boundary or session lifecycle seam rather than mutate tools arbitrarily during an in-flight model call.
