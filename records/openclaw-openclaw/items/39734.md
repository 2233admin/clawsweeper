## Summary

Current main still does not provide a configurable Bedrock Converse `anthropic_beta` or `additionalModelRequestFields` passthrough. The original 1M-context preview motivation may be less urgent per the issue discussion, but the broader request remains concrete and is not implemented by the current Bedrock plugin or the related open PR.

## Best Possible Solution

Keep this open and implement a Bedrock-plugin-owned, validated config surface for user-supplied Bedrock Converse `anthropic_beta` values. The conservative path is to support model-level params such as `params.anthropicBeta` or a narrow `params.additionalModelRequestFields.anthropic_beta`, merge them into the existing Bedrock `onPayload` composition without overwriting existing fields, document the Bedrock config shape, and coordinate with #69491 if maintainers still want default beta injection.
