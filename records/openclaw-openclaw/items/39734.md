## Summary

Current main still does not implement configurable Amazon Bedrock Converse `anthropic_beta` or `additionalModelRequestFields` passthrough. The original 1M-context preview example may be obsolete per the issue discussion, but the broader request is concrete, unimplemented, and not covered by existing direct-Anthropic beta support or OpenAI-compatible `extra_body`.

## Best Possible Solution

Keep this open for a Bedrock-plugin-owned implementation or explicit maintainer product decision. A good fix would add a validated per-model or plugin config surface for Bedrock Converse `anthropic_beta` or selected `additionalModelRequestFields`, merge it in `extensions/amazon-bedrock/register.sync.runtime.ts` with focused payload-patch tests, update the strict plugin schema/UI hints, and document the supported Bedrock config in `docs/providers/bedrock.md`.

## Risks / Open Questions
