## Summary

Keep this issue open. Current main still has a global `agents.defaults.models` allowlist plus per-agent primary/fallback model overrides, but no typed per-agent model allowlist and no agent-scoped server-side filtering/rejection path for model catalogs or session model changes. Related #9010 is closed as a duplicate, while #65557 is broader account/provider policy and does not replace this narrower agent-scoped request.

## Best Possible Solution

Keep this open as the canonical per-agent model allowlist request. Maintainers should decide the agent-scoped config/API shape, document how it composes with `agents.defaults.models`, then enforce it consistently in `models.list`, slash/channel pickers, `sessions.patch`, OpenAI-compatible model override paths, Control UI/TUI model pickers, generated schema/docs, and regression tests.
