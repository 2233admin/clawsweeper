## Summary

Keep this issue open. Current main still has a global `agents.defaults.models` model catalog/allowlist and per-agent primary/fallback model overrides, but no per-agent model allowlist field and no agent-scoped model listing or complete server-side model-selection enforcement. Related #9010 is already closed as a duplicate, while #65557 is broader per-account/provider policy rather than a replacement for this narrower per-agent tracker.

## Best Possible Solution

Keep this issue open as the canonical per-agent model allowlist request. Add an explicit agent-scoped config field, define how it composes with `agents.defaults.models`, and enforce it in model listing plus every model-selection path: Gateway `models.list`, `/model`, `sessions.patch`, OpenAI-compatible override headers, Control UI/TUI/channel pickers, generated config schema/docs, and regression tests for both visibility and server-side rejection.
