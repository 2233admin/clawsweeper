## Summary

Keep open. Current main still has a global `agents.defaults.models` catalog/allowlist plus per-agent primary/fallback defaults, but no typed per-agent model allowlist and no agent-scoped Gateway model-list or session-patch contract. The existing discussion also keeps this as the narrower canonical request: #9010 is closed as a duplicate, while #65557 is broader provider/account policy work and does not supersede this issue.

## What This Changes

Add a per-agent model allowlist so specific agents can be prevented from selecting expensive or inappropriate models while sharing one gateway.

## Best Possible Solution

Add a documented, typed per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently in model listing, `/model` selection, session patching, UI/TUI selectors, and regression tests while leaving broader account/provider administration to #65557.
