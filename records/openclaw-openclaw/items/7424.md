## Summary

Keep open. Current main still has a global `agents.defaults.models` allowlist plus per-agent primary/fallback defaults, but no typed per-agent model allowlist and no agent-scoped enforcement in model listing or session model patching. The provided discussion also keeps this as the narrower request: #9010 is closed as the duplicate, while #65557 is broader account/provider policy work rather than a replacement.

## What This Changes

The issue requests a per-agent model allowlist so a configured agent can restrict which models family or other users can select.

## Best Possible Solution

Add a documented, typed per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently in `models.list`, `/models`, `/model` selection, session patching, UI/TUI selectors, and regression tests while leaving broader account/provider administration to #65557.
