## Summary

Keep this issue open. Current main still has a global `agents.defaults.models` catalog/allowlist and per-agent `agents.list[].model` primary/fallback overrides, but no agent-scoped model allowlist and no agent-scoped model listing or selection enforcement. Related #9010 is closed as a duplicate, while broader #65557 is still open but covers per-account/provider policy rather than replacing this narrower per-agent tracker.

## Best Possible Solution

Keep this issue open as the canonical per-agent model allowlist request. Add an explicit agent-scoped config field, define how it composes with `agents.defaults.models`, and enforce it in model listing and every model-selection path: Gateway `models.list`, `/model`, `sessions.patch`, OpenAI-compatible override paths, Control UI/TUI pickers, config schema/generated docs, and regression tests for both visibility and server-side rejection.
