## Summary

Keep open. Current main still provides only the global `agents.defaults.models` catalog/allowlist plus per-agent primary/fallback defaults; it does not expose a typed per-agent model allowlist or agent-scoped model-list/session-patch contract. The existing discussion keeps this as the narrower canonical request, with #9010 closed as duplicate and #65557 tracking broader account/provider policy.

## What This Changes

The issue requests a per-agent model allowlist so different agents can hide or reject expensive or inappropriate models while sharing one Gateway.

## Best Possible Solution

Add a documented, typed per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently in model listing, chat model selection, Gateway session patching, UI/TUI selectors, and regression tests while leaving broader account/provider administration to #65557.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is source and docs inspection showing only the global allowlist plus per-agent primary/fallback defaults, with no agent-scoped catalog field or `models.list` parameter.

## Solution Assessment

Unclear because there is no proposed patch yet. The maintainable direction is an additive config/protocol surface with explicit inheritance and enforcement semantics, rather than overloading the current strict primary/fallback model shape without maintainer API review.
