## Summary

Keep this open. Current main still implements only a global `agents.defaults.models` catalog/allowlist plus per-agent primary/fallback defaults and params; there is no typed per-agent model allowlist, `models.list` is not agent-scoped, and model selection/session patch validation still resolve against the global allowlist. The earlier duplicate #9010 is already closed against this narrower request, while #65557 is broader account/provider policy work and does not supersede this issue.

## What This Changes

The issue requests an agent-scoped model allowlist so a shared or family agent can be prevented from selecting expensive or unsuitable models.

## Best Possible Solution

Add a documented, typed per-agent model allowlist/catalog with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently in model listing, `/model` selection, session patching, UI/TUI selectors, and regression tests while leaving broader account/provider policy to #65557.
