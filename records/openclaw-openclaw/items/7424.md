## Summary

Current main still lacks a typed per-agent model allowlist. The code supports a global `agents.defaults.models` catalog/allowlist and per-agent `agents.list[].model` primary/fallback defaults, but no agent-scoped allowlist for model listing or selection. Related #9010 is already closed as the duplicate of this narrower request, while open #65557 is broader account/provider policy work and does not supersede it.

## What This Changes

Add an agent-scoped model allowlist so specific agents can only list and select permitted models, avoiding accidental expensive or unsuitable model choices.

## Best Possible Solution

Keep this open as the canonical per-agent model allowlist request. Maintainers should choose the additive public config/API shape and composition rules with `agents.defaults.models`, then enforce it consistently in `/models`, `/model`, `models.list`, `sessions.patch`, Control UI/TUI selectors, generated schema/help/docs, and regression tests. #9010 can remain the duplicate and #65557 can continue tracking broader per-account/provider policy.
