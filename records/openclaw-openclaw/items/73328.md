## Summary

Keep this issue open. Current main has a Bedrock known-context-window table and fixes several Claude 3.x/4.x discovery rows, but it is not a complete implementation of the requested per-model override/lookup behavior: at least one currently documented Bedrock Claude 4 model ID still falls through to the 32k default.

## Best Possible Solution

Keep the issue open and complete the Bedrock plugin-owned fix: extend `KNOWN_CONTEXT_WINDOWS` to cover every currently documented Bedrock Anthropic Claude ID, including region-prefixed inference profile variants through the existing normalization path, and add focused discovery tests for at least one previously missing Claude 4 ID. If maintainers prefer operator control over table churn, add a plugin-owned discovery override config such as `plugins.entries.amazon-bedrock.config.discovery.modelContextWindowOverrides` and document it.

## Evidence

The resolver only checks exact ID, regional-prefix-stripped ID, and a `:0`-stripped variant, so that model would not match this table.
