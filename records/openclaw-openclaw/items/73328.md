## Summary

Keep this issue open. Current main still falls back to the Bedrock discovery default for the reported unlisted Claude 3/3.5 IDs, and two open unmerged PRs already reference this issue with closing syntax, so the next step is maintainer selection of the canonical fix rather than cleanup closure.

## What This Changes

The issue requests Amazon Bedrock discovery to report correct 200K context windows for Claude 3.x/4.x models, either through built-in model/family mapping or a per-model override config surface, instead of using the 32K default.

## Best Possible Solution

Land one canonical Amazon Bedrock plugin-owned fix with regression coverage for unlisted Claude 3/3.5 base IDs and regional inference-profile variants. If the override-config direction is chosen, align the plugin manifest, zod schema, exported config type, and Bedrock docs in the same change.
