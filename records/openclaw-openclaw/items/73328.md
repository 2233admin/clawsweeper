## Summary

Keep this issue open. Current main still falls back to the Bedrock discovery default for unlisted Claude 3.x/3.5 model IDs and has no configurable per-model override surface; two open, unmerged PRs also explicitly target this issue, so maintainers should review and choose between those implementations rather than close it early.

## What This Changes

The issue asks Bedrock discovery to report correct per-model context windows for Claude 3.x/4.x models instead of using the 32000 default fallback.

## Best Possible Solution

Resolve this in the Bedrock plugin-owned discovery path by reviewing #73382 and #73959, choosing the desired product contract, adding regression coverage for missing Claude 3.x/3.5 IDs and inference profiles, and updating plugin schema/docs if a config knob is accepted.
