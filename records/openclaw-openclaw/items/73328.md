## Summary

Keep this issue open. Current main still leaves unmapped Amazon Bedrock Claude 3.x/4.x model IDs on the 32K discovery fallback, and open unmerged PRs #73359 and #73382 explicitly target this same issue with closing syntax.

## Best Possible Solution

Keep this issue open and resolve it through the Bedrock plugin-owned implementation path: review #73359 and #73382, choose either a documented per-model override knob or a complete Claude-family/table fallback, add discovery and inference-profile tests for the missing Claude 3.x/4.x IDs, and close only after the selected fix lands or maintainers explicitly reject the approach.
