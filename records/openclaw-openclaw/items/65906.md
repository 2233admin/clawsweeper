## Summary

Keep this PR open. Current main still lacks the requested response-prefix variables, so the PR is not obsolete or implemented; however, the PR head still needs revision because its runtime usage calculation can include output tokens and its callback wiring appears incomplete for direct channel dispatch paths.

## What This Changes

The PR adds `context` and `contextPercent` response-prefix variables, an `onContextUsage` reply-prefix callback wired through selected gateway/plugin dispatch paths, and unit tests for template resolution and prefix context formatting.

## Best Possible Solution

Land a revised version that derives `{context}` and `{contextPercent}` from prompt/context tokens only, threads the usage callback through every reply pipeline path that can render a response prefix, documents the variables in config/docs, and covers the resolver, prefix bundle, and at least one real channel or gateway dispatch path.
