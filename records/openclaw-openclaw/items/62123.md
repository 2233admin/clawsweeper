## Summary

Keep open. Current main has the reported gap for external plugin authors: the public hook catalog exposes observation-only `after_tool_call` and sync-only `tool_result_persist`, while the async pre-model tool-result middleware that does exist is explicitly bundled-only. The related open PR and issues are useful context, but they do not fully resolve this async external-plugin contract on main.

## What This Changes

Request an async plugin hook after tool execution and before transcript/model visibility that can replace or suppress a tool result.

## Best Possible Solution

Design one canonical public contract for post-execution, pre-model tool-result transformation, likely by either promoting bounded middleware to external plugins under an explicit trust policy or adding a typed hook with clear timeout, block, and ordering semantics, then align #29132, #52849, and #64907 around that contract.

## Reproduction Assessment

Yes, as an API-surface reproduction rather than a runtime bug. Current main shows no external async result-mutation hook, ignores Promise returns from `tool_result_persist`, and rejects non-bundled registrations of the existing async tool-result middleware.

## Solution Assessment

Unclear. The requested hook is a plausible narrow solution to the stated secondary-scan pattern, but the safest implementation needs maintainer API and security-policy judgment because current main deliberately keeps the async pre-model middleware bundled-only.
