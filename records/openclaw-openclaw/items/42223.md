## Summary

Keep this PR open. Current main still contains the active-tab exception that prevents a collapsed sidebar section from receiving `nav-section--collapsed` when the active tab is inside that section, so the PR's intended behavior is not implemented on main. The existing discussion and bot review identify the right root cause, but the patch needs to be rebased from the old `nav-group--collapsed` markup to the current `nav-section`/`showItems` render path and should add a focused browser regression test.

## Best Possible Solution

Keep this PR open as the implementation path. Update it against current main by changing the current `showItems`/`nav-section--collapsed` logic so a manually collapsed group hides its children even when it contains the active tab, while preserving icon-only sidebar behavior when `navCollapsed` is true. Add a focused Control UI browser regression test in `ui/src/ui/navigation.browser.test.ts` before merge.
