## Summary

Current main still has the active-tab exception that this PR is trying to remove. A manually collapsed sidebar section that contains the active tab still computes `showItems` as true, so the collapsed CSS class is not applied and child items remain visible. The PR remains a plausible focused bug fix, but it targets older `nav-group--collapsed` markup while current main uses `nav-section`/`showItems`, so it should stay open for rebase/adaptation and a focused regression test.

## Best Possible Solution

Keep this PR open as the implementation path. Rebase it onto current main, adapt the fix to the current `showItems`/`nav-section--collapsed` render path so manual group collapse hides children even when the active tab is inside that group, preserve icon-only sidebar behavior when `navCollapsed` is true, and add a focused Control UI browser regression test in `ui/src/ui/navigation.browser.test.ts`.
