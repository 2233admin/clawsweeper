## Summary

Keep this PR open. Current main still has the same active-tab exception the PR is trying to fix: a collapsed nav group containing the active tab still renders its children and does not receive the collapsed CSS class. The PR remains a plausible focused fix, but it needs to be rebased/adapted from the older `nav-group--collapsed` markup to the current `nav-section`/`showItems` render path and should add a focused browser regression test before merge.

## Best Possible Solution

Keep this PR open as the implementation path. Rebase it onto current main and adapt the fix to the current `showItems`/`nav-section--collapsed` logic so a manually collapsed group hides its children even when it contains the active tab, while preserving icon-only sidebar behavior when `navCollapsed` is true. Add a focused Control UI browser regression test in `ui/src/ui/navigation.browser.test.ts` before merge.
