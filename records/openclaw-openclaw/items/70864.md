## Summary

Keep open. Current main does not implement scoped mention-pattern policy, and the latest PR head is still a useful implementation attempt, but it remains incorrect because several wired channel runtimes read provider-scoped `mentionPatterns` that their strict config schemas do not accept.

## What This Changes

The PR adds shared scoped mention-pattern policy config/helpers, exports them through channel SDK/runtime surfaces, wires configured pattern gating through bundled channel handlers, and updates docs, generated config metadata, and tests.

## Best Possible Solution

Align the supported config boundary before merge: either add `MentionPatternsPolicySchema` to every channel/account schema whose runtime now reads provider-scoped policy, including generated metadata and docs, or narrow the runtime/docs to only the channels that intentionally support provider scopes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Add the policy schema for each supported provider/account or narrow the runtime/docs to the supported channels.
