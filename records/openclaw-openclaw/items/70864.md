## Summary

Keep open. Current main still lacks the scoped mention-pattern policy, and the latest PR head still has a source-visible correctness issue: Slack account-scoped `mentionPatterns` validates but the Slack runtime only resolves the top-level provider policy.

## What This Changes

The PR adds shared scoped mention-pattern policy config and runtime helpers, wires configured pattern allow/deny gating through bundled channel handlers, and updates generated config metadata, docs, and tests.

## Best Possible Solution

Before merge, make the config/runtime contract consistent: either pass each resolved account’s `mentionPatterns` policy into `resolveMentionPatternsEnabled` for providers that expose account-scoped policy, with focused tests, or remove account-scoped schema/docs exposure and keep the feature provider-level only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Honor Slack account mention pattern policy:** `extensions/slack/src/monitor/message-handler/prepare.ts:317-322`
  - body: `SlackAccountSchema` now accepts `channels.slack.accounts.<id>.mentionPatterns`, but this call only lets the resolver read top-level `channels.slack.mentionPatterns`. Named Slack account policies are silently ignored, so group mention gating can accept or drop messages incorrectly. Pass the merged account policy, for example `account.config.mentionPatterns`, as `providerPolicy`, or stop exposing the account-scoped key.
  - confidence: 0.88
