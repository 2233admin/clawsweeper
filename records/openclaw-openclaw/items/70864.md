Keep open. Current main still lacks scoped mention-pattern policy, and this PR is the active implementation, but the latest head still has an actionable iMessage account-scope bug that should be fixed on the source branch before maintainer review/merge.

## What This Changes

The PR adds shared mention-pattern policy config and SDK/runtime helpers, gates configured regex mention matching across bundled channel runtimes, and updates docs, generated config metadata, changelog, and tests.

## Best Possible Solution

Ship the shared scoped policy in this source PR after the iMessage account-level policy path is wired through and covered by a focused test, then let the pending CI and maintainer review finish on the same branch.

## Reproduction Assessment

Yes for the remaining review finding. Configure a multi-account iMessage setup with an account-level `channels.imessage.accounts.<id>.mentionPatterns` policy that differs from the global/top-level policy; the schema accepts it, but the inbound policy call has no account `providerPolicy`, so the account override cannot affect mention regex gating.

## Solution Assessment

No, not merge-ready yet. The shared resolver and channel-gating direction is the right maintainable shape, but iMessage account-scoped policy must use the same merged account config that the monitor already resolves before this is the best complete fix.
