## Summary

Keep open. Current main still has the Feishu quoted/replied-message fetch wired only to `parent_id`; the issue discussion identifies UI quote replies as using a separate quote message id, and current source/tests do not model, parse, prefer, or cover that quote-only path. The related #29501 work appears to have improved fetched-message content parsing, but not inbound quote-id selection.

## What This Changes

The issue requests that the Feishu plugin include quoted or parent message content in the agent input when users quote a message and mention the bot.

## Best Possible Solution

Implement the missing Feishu UI quote-id path in the bundled Feishu plugin: verify the incoming quote payload shape, add a typed optional quote message id to the event/context model, parse it, prefer it when fetching quoted content while retaining `parent_id` fallback, and add regression tests for quote-only UI replies plus existing parent-id and group visibility behavior.

## Repair Work Prompt

Fix Feishu UI quote replies so quoted message content reaches the agent when the event carries a quote-specific message id instead of `parent_id`.

Scope:
- Work inside the bundled Feishu plugin only; do not add core plugin APIs or unrelated channel changes.
- Verify the Feishu/Lark UI quote payload shape from current event examples/docs before finalizing the field name. The issue discussion points to `event.message.quote.message_id` / `quote.message_id`.
- Add a typed optional quote id to `FeishuMessageEvent` and `FeishuMessageContext`.
- Parse the quote id in `parseFeishuMessageEvent()`.
- In the quoted/replied fetch path, prefer the quote id when present and fall back to `parentId` for existing reply/thread behavior.
- Preserve current `parent_id`, `ReplyToId`, `ReplyToBody`, `contextVisibility`, and group sender allowlist behavior.
- Add regression coverage for a quote-only UI reply that fetches the quoted message, the existing parent-id path, and group visibility filtering for quoted context.
- Add a user-facing `CHANGELOG.md` entry if maintainers treat this as shipped behavior; credit the reporter/contributor appropriately and do not credit Codex/OpenClaw.

Do not change Feishu media parsing, outbound reply behavior, core channel contracts, or unrelated docs.
