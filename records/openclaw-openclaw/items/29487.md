## Summary

Keep this issue open. Current main still fetches Feishu quoted/replied content only through `parent_id`; it does not preserve or prefer the separate UI quote message id described in the issue discussion, so quote-only Feishu events can still reach the agent without the quoted content. The earlier #29501-related work did ship useful fetched-message parsing, but that is a different layer from selecting the quoted message id from inbound events.

## Best Possible Solution

Implement the missing Feishu UI quote-id path inside the bundled Feishu plugin: verify the incoming quote payload shape, add a typed optional quote message id to the event/context model, parse it, prefer it when fetching quoted content while retaining `parent_id` fallback, and add regression tests for quote-only UI replies plus existing parent-id and group visibility behavior.
