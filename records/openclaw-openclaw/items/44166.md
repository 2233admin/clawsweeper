## Summary

Keep this issue open. Current main still has the reported behavior: embedding batch retry classification is limited to rate-limit/service-style failures, unclassified transport strings fail immediately, and batch embedding has no retry-exhaustion split fallback. The supplied GitHub context also shows open same-author PR #44167 as the implementation candidate, so this issue should remain open until that PR is reviewed and merged or otherwise resolved.

## Best Possible Solution

Keep this issue open and use #44167 as the immediate implementation candidate. The fix should stay in `extensions/memory-core`: add narrow transient transport classification, preserve the existing backoff behavior, recursively split multi-item batches only after retry exhaustion, keep single-item and non-retryable failures terminal, and add focused tests for classification, ordering, and structured/text batch paths.
