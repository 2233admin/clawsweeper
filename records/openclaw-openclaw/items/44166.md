## Summary

Keep open. Current main at b85edb3f0cf6c2a8bec06537af9211a044266a6c still has the narrow memory embedding retry classifier and no recursive split fallback for exhausted multi-item embedding batches. The issue is also paired with open same-author PR #44167, which uses closing syntax and is the immediate implementation candidate.

## Best Possible Solution

Keep this issue open and review, update, or land PR #44167 as the focused implementation candidate. The fix should stay in memory-core: expand transient transport retry classification, retry with the existing backoff, recursively split exhausted multi-item text and structured embedding batches while preserving order, and add targeted regression tests for retry-before-split, single-item terminal failures, and non-transport service errors that should not split.
