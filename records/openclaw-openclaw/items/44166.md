## Summary

Keep open. Current main at 20e21173715ef488850023a90f72585a01cc379f still has the narrow memory embedding retry classifier, still throws from the retry loop without an exhausted-retry result that could trigger splitting, and still calls the text/structured batch retry wrappers directly. The issue is also paired with open PR #44167, so the right cleanup action is to review, update, land, or explicitly replace that PR rather than close this issue now.

## What This Changes

The issue requests that memory batch embedding retry transient transport failures and recursively split exhausted multi-item batches so remote memory reindex does not abort on recoverable network errors.

## Best Possible Solution

Keep this issue open and review, update, or land PR #44167 as the focused implementation candidate. The fix should stay in memory-core: classify narrow transient embedding transport failures as retryable, reuse the existing backoff loop, recursively split exhausted multi-item text and structured embedding batches while preserving order, and add targeted regression tests. Broader rollback/cache preservation work can remain separate in #59137.
