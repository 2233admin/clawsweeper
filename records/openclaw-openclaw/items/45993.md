## Summary

Keep this issue open. Current main has a provider-based `openclaw migrate` flow for importing from other agent systems and `openclaw backup create/verify`, but it does not implement same-product cross-platform `migrate export` / `migrate import`. The issue is also explicitly paired with open PR #53520, so the right cleanup action is to review/land or close that PR rather than close this issue now.
