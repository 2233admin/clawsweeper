## Summary

Keep open. Current main still supports provider-based imports from other agent systems plus backup create/verify, but it does not provide same-product cross-platform archive export/import or backup restore; an open fork PR already proposes this command surface and should be reviewed, replaced, landed, or closed before this tracker is resolved.

## What This Changes

Add `openclaw migrate export` and `openclaw migrate import` for selective cross-platform OpenClaw state migration with path remapping, env/secret handling, memory merge choices, and dependency checks.

## Best Possible Solution

Keep this issue as the canonical tracker for same-product OpenClaw migration. Maintainers should review the open fork PR against the existing migration-provider and backup architecture, then land or replace a focused implementation with archive validation, secret handling, path remapping, docs, and tests; if `migrate export/import` is not the preferred API, record the backup/restore direction here.
