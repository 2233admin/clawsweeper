## Summary

Keep this PR open. Current main still does not register or subscribe to Slack `file_shared` events, so the large-file and multi-file fallback remains unimplemented. The related #65299 report corroborates the same missing event path and describes a polling workaround; closed #40286 does not supersede this work because it was closed unmerged. This PR is still a plausible implementation candidate, but it needs a port from the removed `src/slack` tree into the current `extensions/slack` plugin plus setup, docs, and tests.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a rebase or replacement port into `extensions/slack`: add and register a `file_shared` event handler under `extensions/slack/src/monitor/events`, hydrate the matching Slack message with an `event_ts`-anchored bounded history lookup, dedupe Slack retries and normal `message`/`file_share` delivery, preserve current Slack authorization and media protections, update `extensions/slack/src/setup-shared.ts` and `docs/channels/slack.md` for the `file_shared` bot event, and add focused Slack plugin tests for large-file, multi-file, duplicate-delivery, busy-channel, and app-mention paths.

## Risks / Open Questions

- The PR cannot merge as-is because it targets the removed `src/slack` layout instead of the current `extensions/slack` plugin.
- The proposed handler increases the history window to 20 but still does not anchor the lookup to `event_ts`, so busy channels can still make the matching message unreliable.
- The PR does not update Slack setup manifest generation or docs, so users would not be instructed to subscribe to the `file_shared` bot event.
- No tests currently cover `file_shared` delivery, normal `message`/`file_shared` dedupe, Slack retry duplicate delivery, busy-channel history lookup, multi-file behavior, or app-mention interactions.
- A correct port must keep the existing Slack authorization and media protections intact, especially team/account mismatch drops, allowlists, mention gating, Slack-host URL validation, bearer-token handling, and channel/thread scope checks for file downloads.
