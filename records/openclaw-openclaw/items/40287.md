## Summary

Keep this PR open. Current main still does not implement the central Slack `file_shared` fallback, and the PR remains a plausible implementation candidate, but it targets the removed `src/slack` tree instead of the current `extensions/slack` plugin layout. The prior ClawSweeper review reached the same conclusion; I rechecked current main at d22a851253b0225e72847dc6b1a7b74d8035e214 and found no newer implementation.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a rebase or replacement port into `extensions/slack`: add and register a `file_shared` handler, hydrate the matching message with a timestamp-anchored bounded `conversations.history` lookup, dedupe Slack retries and normal `message`/`file_share` delivery, update Slack setup/docs for the required bot event, and add focused Slack plugin tests.
