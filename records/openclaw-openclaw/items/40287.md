## Summary

Keep this PR open. Current main still does not register, subscribe, hydrate, or test Slack `file_shared` events, so the large-file and multi-file fallback remains unimplemented. The PR is still a plausible implementation candidate, but it is not merge-ready because it targets the removed `src/slack` tree and needs a careful port into the current `extensions/slack` plugin with setup/docs/tests and anchored history lookup.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a rebase or replacement port into `extensions/slack`: add and register a `file_shared` event handler under `extensions/slack/src/monitor/events`, hydrate the matching Slack message through a bounded `event_ts`-anchored `conversations.history` lookup, dedupe Slack retries and normal `message`/`file_share` delivery, preserve current authorization and media protections, update `extensions/slack/src/setup-shared.ts` and `docs/channels/slack.md` for the `file_shared` bot event, and add focused Slack plugin tests for large-file, multi-file, duplicate-delivery, busy-channel, and app-mention paths.
