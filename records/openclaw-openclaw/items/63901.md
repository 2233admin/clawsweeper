## Summary

Keep open. Current main supports static `extra_body` pass-through and OpenRouter app-attribution headers, but it still does not provide session-aware `user` or `trace` request-body metadata for provider calls; the two related open PRs cover different slices of the remaining design.

## What This Changes

Request session, graph, task, or agent metadata be forwarded to OpenRouter/OpenAI-compatible provider requests so shared provider accounts can attribute costs per workload.

## Best Possible Solution

Keep this issue open until maintainers converge on a privacy-safe attribution design: likely an additive provider SDK/runtime seam for sanitized session attribution plus focused OpenRouter/LiteLLM-compatible `user` and `trace` handling, precedence tests, and docs that make third-party metadata forwarding explicit.
