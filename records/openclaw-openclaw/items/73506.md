## Summary

Keep this PR open. Current main still does not implement the intended FailoverError sessionId/lane/provider attribution, linked #42713 remains open, and the live PR is a focused additive change with no apparent supply-chain-sensitive edits. The remaining maintainer question is whether this helper-level patch is sufficient, or whether the fix must also thread session/lane through runWithModelFallback, FallbackSummaryError, and fallback decision logging.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land this focused additive FailoverError/helper change as a first step after confirming that gateway.err.log ingestion sees that path, or extend/supersede it with a complete fix that threads sessionId/lane through runWithModelFallback, FallbackSummaryError or fallback decision logging, and agent-command call sites, with regression coverage for exhausted fallback attribution.
