## Summary

Keep this PR open. Current main still lacks the requested FailoverError session/lane attribution, and the PR is a focused additive helper change, but it does not yet prove or complete the observable exhausted-fallback logging fix because the actual model-fallback summary path still has no sessionId/lane input and throws FallbackSummaryError after candidate exhaustion.

## What This Changes

The branch extends FailoverError and its describe/coerce helpers with optional sessionId/lane attribution, adds targeted failover-error tests, and adds a changelog entry claiming the linked failover logging bug is fixed.

## Best Possible Solution

Thread request-origin attribution from the agent command/run context into model fallback decisions and the exhausted FallbackSummaryError path, keep the FailoverError helper additions where useful, and add a regression test that proves exhausted fallback logs or emitted structured records include provider plus session/lane attribution.

## Solution Assessment

No. The helper changes are additive and likely useful, but they are not the narrow complete fix for the reported logging gap unless the caller and model-fallback summary/logging path also receive and emit the session/lane attribution.
