## Summary

Keep this PR open. Current main still does not implement the PR's intended FailoverError attribution change, and the linked open issue #42713 remains active. The PR is focused and low supply-chain risk, but it appears to be at most a partial candidate because the real exhausted model-fallback path still ends in FallbackSummaryError and model_fallback_decision logs without sessionId/lane plumbing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either extend this PR or supersede it with a narrow end-to-end observability fix that carries normalized sessionId/lane and provider/model/profile attribution through the actual exhausted model-fallback path, including FallbackSummaryError, logModelFallbackDecision, runWithModelFallback inputs, and runner call sites, with focused regression coverage for structured gateway error logs.
