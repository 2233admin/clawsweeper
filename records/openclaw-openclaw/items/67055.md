## Summary

Keep this PR open. Current main still threads Google Chat typing placeholders and still exposes the inbound thread as implicit reply context, so the reported bug is not fully fixed. The PR is a useful focused attempt with live DM verification, but it needs revision because its final delivery guards would also suppress explicit reply directives that the shared off-mode contract preserves.

## Best Possible Solution

Keep the PR open and request a focused revision: suppress only implicit Google Chat inbound thread propagation for ReplyToId, ReplyToIdFull, and the typing placeholder when account.config.replyToMode is "off"; keep final delivery honoring payload.replyToId so explicit reply tags still work; rebase onto the current monitor-reply-delivery.ts split; add a Google Chat monitor regression test for the off-mode DM path.
