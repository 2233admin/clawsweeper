## Summary

Keep this PR open. The reported Google Chat `replyToMode: "off"` bug is still present on current `main`, but the branch is not merge-ready because its final delivery guards would drop explicit reply directives that the shared reply-threading contract intentionally preserves, and current `main` has since split reply delivery into a separate file.

## What This Changes

The PR adds `replyToMode: "off"` checks around Google Chat monitor thread propagation and adds channel-level and per-account Google Chat config-resolution tests.

## Best Possible Solution

Revise or fold the branch into the current Google Chat threading work: filter only implicit inbound `ReplyToId`/`ReplyToIdFull` and typing-placeholder thread propagation when the resolved account has `replyToMode: "off"`, keep final delivery honoring already-filtered `payload.replyToId`, and add a focused monitor regression for off-mode DM replies.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Honor explicit reply targets in off mode:** `extensions/googlechat/src/monitor.ts:437`
  - body: `payload.replyToId` has already passed through the shared reply-threading flow, which preserves explicit `[[reply_to:*]]` and `[[reply_to_current]]` targets even when `replyToMode` is `off`. Filter the implicit inbound/typing thread source instead and let final delivery use `payload.replyToId`.
