## Summary

Keep this PR open. Current main still validates omitted plugin config as an empty object in both non-activating snapshot and CLI metadata paths, so the proposed fix is not obsolete or implemented on main. The branch is a focused implementation candidate with regression coverage, but it still needs maintainer review of the intentional deferred-validation boundary and the prior security concern before merge.

## What This Changes

The PR adds a non-activating plugin-loader guard that defers validation for undefined plugin config, preserves normalized/defaulted `api.pluginConfig`, adds snapshot and CLI metadata regressions, and includes two macOS SwiftFormat brace-only edits.

## Best Possible Solution

Land this PR or an equivalent loader change once maintainers explicitly accept the deferred-validation boundary: omitted config should not block non-activating metadata/snapshot discovery, activation and explicit-config paths should remain strict, `api.pluginConfig` should stay normalized/defaulted where safe, and the focused loader regressions should stay in place.
