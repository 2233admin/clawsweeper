## Summary

Keep this PR open. Current main still validates omitted plugin config as an empty object in the non-activating snapshot and CLI metadata paths, so the linked false-positive validation bug is not implemented on main. The PR is a plausible focused fix with regression coverage, but maintainer review should explicitly resolve the validation-bypass concern before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent loader change that distinguishes omitted config in non-activating snapshot/CLI metadata loads from real activation, preserves normalized/defaulted `api.pluginConfig`, keeps activation and explicit-config validation strict, retains the focused regressions in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`, and explicitly resolves the Aisle validation-bypass concern before merge. #64946 should remain separate because it tracks a different wrapped/unwrapped schema-validation noise path.
