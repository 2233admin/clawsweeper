## Summary

Keep open. Current main and the latest release still lack requester authorization for Discord guild-admin message actions, and the linked replacement PR that uses closing syntax is open rather than merged.

## What This Changes

Require Discord guild-admin message actions to use a trusted requester identity and verify the requester has the matching Discord guild permissions before bot-credentialed guild mutations run.

## Best Possible Solution

Land a scoped Discord plugin fix that marks guild-admin tool-driven actions as trusted-requester-only, forwards the trusted requester ID into guild-admin runtime calls, verifies the required Discord permission bits before privileged mutations, preserves disabled-action ordering and trusted local operator flows, and adds focused regression coverage.
