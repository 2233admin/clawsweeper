## Summary

Keep this issue open. Current main still has the Control UI `AUTH_TOKEN_MISMATCH` reconnect gap when no cached device-token retry can run, and `chat.history` still performs synchronous full-transcript reads before applying response limits. The Slack Socket Mode timeout tuning is present as a mitigation, but the latest issue discussion confirms v2026.4.26 still disconnects.

## Best Possible Solution

Fix this in core Control UI and Gateway history handling. Make `AUTH_TOKEN_MISMATCH` stop automatic reconnects when no trusted cached-device-token retry can be attempted, and after the single retry is exhausted. Separately, make `chat.history` avoid synchronous full-transcript reads/parses before recent-message and byte limits are applied, while preserving Pi branch semantics and existing display normalization. Add focused regression coverage for both paths, then use Slack Socket Mode testing as live validation rather than treating timeout tuning as the root fix.
