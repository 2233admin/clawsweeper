## Summary

Keep open. Current main still uses a Google preview model as the bundled/default Gemini choice and retains default-setting paths that can replace an existing primary unless the caller explicitly opts into preservation, so the reported cost-sensitive migration/default-policy problem is not resolved.

## What This Changes

The issue asks OpenClaw to prevent upgrades or migrations from silently changing configured primary and fallback models to higher-cost preview defaults without explicit user consent, with related billing warnings and cost-control follow-ups.

## Best Possible Solution

Define a default-model safety policy that preserves existing primary and fallback choices across upgrades unless the user explicitly accepts a preview or higher-cost replacement, keeps provider ID normalization separate from default promotion, adds a visible warning/revert path for migration-set preview defaults, and tracks cron/tool-grounding gaps separately.

## Reproduction Assessment

Partial yes. Current main source and tests give a high-confidence reproduction of the remaining policy risk: preview defaults remain and default helpers overwrite unless preservation is explicitly requested; the exact historical billing artifact still depends on the reporter's saved backups/session evidence.

## Solution Assessment

Unclear. A narrow helper patch alone would not cover pricing-tier consent, preview warnings, fallback preservation, release-note policy, and adjacent cron/tool-grounding concerns, so maintainer scope should come first.
