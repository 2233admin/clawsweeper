## Summary

Keep open. Current main still carries preview Google defaults and default-overwrite behavior, while the discussion includes a second paid-model surprise report. The remaining work is cost-sensitive migration/default policy, not conservative cleanup material.

## What This Changes

Prevent upgrades, migrations, and provider setup paths from silently changing configured default models or fallbacks to higher-cost preview models without explicit consent, pricing notice, and a revert path.

## Best Possible Solution

Define a model-default safety policy that preserves existing global and per-agent primaries/fallbacks across upgrades unless the user explicitly accepts a higher-cost or preview replacement, keeps provider ID normalization separate from default promotion, adds a one-time warning/revert path for migration-set preview defaults, and splits cron control plus tool-grounding visibility into narrower accepted follow-ups.

## Reproduction Assessment

Yes for the remaining code-policy risk: current main source and tests show Google/provider default paths can still promote a preview default and overwrite an existing primary unless a caller opts into preservation. The exact historical migration and billing window are not reproducible without the reporter's backup/session bundle.

## Solution Assessment

Unclear until maintainers choose the policy boundary. A narrow fix could extend preservation and consent around provider default promotion, but pricing-tier detection, warning UX, changelog policy, cron mutation tools, and tool-grounding indicators need maintainer scoping before automation should implement them.
