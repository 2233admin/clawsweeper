## Summary

Keep open. Current main still uses `google/gemini-3.1-pro-preview` as the Google default, the bundled Google API-key auth path still applies that default, and the Google default helper still has test coverage expecting an existing primary to be replaced. The newer configure preservation path helps one re-run flow, but it does not settle the reported upgrade/migration consent, price-tier warning, or default-promotion policy issue.

## What This Changes

The issue asks OpenClaw to prevent silent upgrade/migration changes from moving a user's configured Google primary model to a paid preview default without explicit consent, with related requests for pricing warnings, safer default promotion, billing-spike warnings, cron stop controls, and tool-grounding visibility.

## Best Possible Solution

Treat the model-migration and cost-consent concern as the primary product bug: keep existing user primaries across upgrades unless the user explicitly opts into a new preview or higher-cost default, separate alias normalization from default promotion, and add a clear warning/revert path for migration-set preview defaults. Split cron mutation tools and tool-grounding indicators into narrower follow-up issues if maintainers accept those directions.
