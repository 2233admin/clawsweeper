## Summary

Keep open. Current main still carries the Google Gemini preview default, the Google default helper still replaces an existing primary, and the preservation path added for configure re-runs is caller-scoped rather than a general migration consent policy. A second user reported a similar paid-model surprise, so this is not conservative cleanup material.

## What This Changes

The issue asks OpenClaw to prevent upgrades and migrations from silently promoting existing model defaults or fallbacks to higher-cost preview models without consent, pricing notice, or a revert path.

## Best Possible Solution

Define a model-default migration policy that preserves existing global and per-agent primaries/fallbacks unless the user explicitly chooses a higher-cost or preview replacement, keeps provider ID normalization separate from default promotion, and adds a one-time warning/revert path for migration-set preview defaults. Cron disable controls and tool-grounding indicators should become narrower follow-up issues if maintainers accept those directions.

## Reproduction Assessment

Yes for the current overwrite behavior: source and tests show `applyGoogleGeminiModelDefault` replaces an existing primary with the Google preview default. The exact historical upgrade and billing timeline remain unclear without the reporter's local backup/session bundle.

## Solution Assessment

Unclear until a maintainer scopes the policy. Preserving existing defaults plus a consent/warning path is the likely maintainable direction, but the issue bundles cost-sensitive migration policy with separate cron and tool-grounding surfaces.
