## Summary

Keep open. Current main still has cost-sensitive preview defaults and default-setting paths that overwrite an existing primary unless the caller opts into preservation, while the issue discussion includes a second paid-model surprise report; this needs maintainer policy rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop silent upgrade or migration paths from replacing a user's existing default model with a higher-cost preview model without explicit consent or warning.

## Best Possible Solution

Define and document a model-default safety policy that preserves existing global and per-agent primaries/fallbacks across upgrades unless the user explicitly accepts a preview or higher-cost replacement, separates provider ID normalization from default promotion, adds a visible warning/revert path for migration-set preview defaults, and splits cron control plus tool-grounding visibility into narrower follow-ups.

## Reproduction Assessment

Partial yes. Current main source and tests give a high-confidence reproduction of the remaining overwrite-by-default policy risk, but the exact historical billing artifact needs the reporter's backup config files and session logs.

## Solution Assessment

Unclear until maintainers choose the product policy. The narrowest maintainable direction is likely to make preservation/consent the default for migration and setup flows while keeping explicit `models set` or `--set-default` flows intentional, with docs and changelog guidance for pricing-sensitive defaults.
