## Summary

Keep open. Current main still carries the preview Google default and code/tests that can overwrite an existing primary unless the caller opts into preservation, while the discussion includes a second paid-model surprise report; this is a cost-sensitive default/migration policy issue rather than cleanup material.

## What This Changes

The issue requests safeguards so upgrades, onboarding, or auth re-application do not silently switch an existing default model to a higher-cost preview model, plus related warnings, changelog, cron-control, and tool-grounding follow-ups.

## Best Possible Solution

Define a model-default safety policy that preserves existing global and per-agent primaries/fallbacks across upgrades unless the user explicitly accepts a higher-cost or preview replacement, keeps provider ID normalization separate from default promotion, adds a one-time warning/revert path for migration-set preview defaults, and splits cron control plus tool-grounding visibility into narrower accepted follow-ups.

## Reproduction Assessment

Yes for the remaining source-level overwrite policy: the current Google default-model test already reproduces replacing an existing primary with `google/gemini-3.1-pro-preview`. The exact historical upgrade and billing window are not reproducible without the reporter's backup/session bundle.

## Solution Assessment

Unclear until maintainers choose the consent boundary. A narrow implementation could expand the existing preservation option and add preview/cost warnings, but pricing-tier detection, changelog policy, cron mutation tools, and tool-grounding indicators need product scoping first.
