## Summary

Keep open. Current main still has a cost-sensitive Google preview default and default-setting paths that overwrite an existing primary unless the caller explicitly opts into preservation; the issue also has a second user report of a paid-model surprise, so this is not conservative cleanup material.

## What This Changes

The issue asks OpenClaw to prevent upgrade/setup flows from silently replacing configured model defaults with paid preview or higher-cost defaults and to add related cost-control guardrails.

## Best Possible Solution

Define a model-default safety policy that preserves existing global and per-agent primaries/fallbacks across upgrades unless the user explicitly accepts a preview or higher-cost replacement, separates provider ID normalization from default promotion, adds warning/revert UX for migration-set preview defaults, and tracks cron control plus tool-grounding visibility as separate scoped follow-ups.

## Reproduction Assessment

Partial yes. Current main source and existing tests show the remaining policy risk around preview defaults and opt-in-only preservation, but the exact historical billing event requires the reporter's backup diffs and session logs.

## Solution Assessment

Unclear until maintainers choose the product policy. A narrow code patch could broaden preservation in setup/default application paths, but consent prompts, price-tier detection, changelog policy, and adjacent cron/tool-grounding requests need explicit maintainer scope.
