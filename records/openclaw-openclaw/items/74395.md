## Summary

Keep open. Current main still has preview Google defaults plus default-overwrite paths that are only selectively guarded, and the discussion now has an independent paid-model surprise report. This is a cost-sensitive model-default/migration policy item rather than safe conservative cleanup.

## What This Changes

Request to prevent upgrades or provider setup from silently replacing an existing default model and fallback chain with higher-cost preview models without explicit consent, warning, or a revert path.

## Best Possible Solution

Define a product-level model-default safety policy: preserve existing global and per-agent primaries/fallbacks across upgrades unless the user explicitly accepts a higher-cost or preview replacement, separate provider ID normalization from default promotion, add a one-time warning/revert path for migration-set preview defaults, and split cron control plus tool-grounding visibility into narrower follow-ups.

## Reproduction Assessment

Yes for the current code-policy risk: current main source and tests show the Google preview default and default-overwrite behavior remain unless a caller opts into preservation. The exact historical upgrade event is not reproducible without the reporter's private backup/session evidence.

## Solution Assessment

Unclear until maintainers choose the consent and cost-control boundary. A narrow helper change could extend preservation, but pricing-tier detection, preview-model warning UX, changelog policy, cron mutation tools, and tool-grounding indicators need maintainer scoping.
