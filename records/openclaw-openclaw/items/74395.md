## Summary

Keep open. Current main still has Google setup wired to a preview default, default-model helpers still overwrite existing primaries unless a caller explicitly opts into preservation, and the newer configure-only preservation path does not resolve the broader migration/default-promotion consent policy. The second user comment reports a similar paid-model surprise, so this needs maintainer scoping rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop silent default-model upgrades to higher-cost preview models and add consent, warning, revert, and documentation surfaces for billing-sensitive model changes.

## Best Possible Solution

Preserve existing global and per-agent primaries across upgrades unless the user explicitly selects a new higher-cost or preview default, keep provider ID normalization separate from default promotion, add a one-time warning/revert path for migration-set preview defaults, and split cron control plus tool-grounding indicators into narrower follow-up issues if maintainers accept those directions.
