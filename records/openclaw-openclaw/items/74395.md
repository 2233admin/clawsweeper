## Summary

Keep open. Current main still has the cost-sensitive behavior family the report is about: Google’s configured default is a preview model, setup/auth paths can still replace an existing primary unless a caller explicitly opts into preservation, and the issue now has a second user report of a paid-model surprise. This needs maintainer policy, not conservative cleanup.

## What This Changes

The issue requests safeguards against silent upgrade or migration paths replacing user-selected default models with higher-cost or preview defaults, plus related billing-control visibility improvements.

## Best Possible Solution

Define a default-model safety policy that preserves existing global and per-agent primaries/fallbacks across upgrades unless the user explicitly accepts a higher-cost or preview replacement, separates provider ID normalization from default promotion, adds a visible warning/revert path for migration-set preview defaults, and tracks cron control plus tool-grounding visibility as narrower follow-ups.

## Reproduction Assessment

Partial yes. Current main source and tests reproduce the remaining policy risk: Google still defaults to a preview model and default helpers overwrite existing primaries unless preservation is explicitly requested; the exact historical billing artifact needs the reporter's backups/session logs.

## Solution Assessment

Unclear until maintainers choose the product boundary. A narrow code fix could extend preservation around provider default promotion, but pricing-tier detection, interactive consent, changelog policy, cron mutation tools, and completion-claim grounding need explicit scope first.
