## Summary

Keep open. Current main still has the Google default wired to `google/gemini-3.1-pro-preview`, the Google default helper still replaces an existing primary model, and regression coverage explicitly expects that override. There is some newer preservation behavior for selected auth/configure paths, but it does not resolve the broader upgrade/migration consent and pricing-impact concern described here.

## What This Changes

The issue asks OpenClaw to stop silently migrating default models across price tiers, especially from Gemini 2.5 Pro to Gemini 3.1 Pro Preview, and to add consent, pricing notices, revert warnings, and related cost-control improvements.

## Best Possible Solution

Treat the model-migration/cost-consent concern as the primary actionable product bug: preserve existing user primaries unless the user explicitly opts into a new paid preview default, separate alias normalization from default promotion, add a one-time warning/revert path for migration-set preview defaults, and split cron/tool-grounding follow-ups into narrower issues if maintainers accept those directions.
