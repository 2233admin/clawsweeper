## Summary

Keep open. Current main still has the Google default wired to `google/gemini-3.1-pro-preview`, Google API-key setup still applies that default, and the default helper still replaces an existing primary model. A newer configure-auth preservation path reduces one adjacent clobber case, but it does not resolve the reported upgrade/migration consent and pricing-impact policy problem.

## What This Changes

The issue asks OpenClaw to prevent upgrade or migration flows from silently changing an existing default model to a paid Gemini preview model without explicit consent or pricing warning.

## Best Possible Solution

Treat the model-migration/cost-consent concern as the primary product bug: preserve existing user primaries unless the user explicitly opts into a new paid preview default, separate alias normalization from default promotion, add a one-time warning/revert path for migration-set preview defaults, and split cron/tool-grounding follow-ups into narrower issues if maintainers accept those directions.
