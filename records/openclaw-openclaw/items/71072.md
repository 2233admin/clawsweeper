## Summary

Keep this PR open. Current main still preserves the final empty-registry semantics for `onlyPluginIds: []`, but it does not implement the PR's intended O(1) fast-path before cache handling, loader scaffolding, discovery, and manifest-registry loading. The latest PR head now uses the current `PluginLoaderCacheState.isLoadInFlight` API, so the earlier stale-symbol concern appears addressed; it still needs normal maintainer review and targeted validation rather than cleanup closure.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate. The best path is to land a small fast-path on `loadOpenClawPlugins` before cache/discovery/manifest work, using the current `PluginLoaderCacheState` API, preserving activating empty-registry side effects, and keeping the focused test that proves discovery and manifest registry loading are skipped for `onlyPluginIds: []`. No docs or changelog entry appears necessary for this internal performance optimization unless maintainers want release-note visibility for startup performance.
