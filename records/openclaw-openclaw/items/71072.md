## Summary

Keep this PR open. Current main still returns an empty registry for `onlyPluginIds: []`, but it does not implement the PR's intended O(1) fast-path before cache handling, loader setup, discovery, and manifest-registry loading. The latest PR head is still the active implementation candidate, but its fast-path returns before the existing same-key load reentry guard, so it needs review/changes before merge rather than cleanup closure.

## Best Possible Solution

Keep the PR open and revise it in place: add the explicit empty-scope fast-path while preserving the existing load reentry contract, activation-clearing side effects, and the distinction between `undefined` and `[]`. The focused test should prove discovery and manifest registry loading are skipped, and a small reentry regression should cover the empty-scope activating path. No docs or changelog entry appears necessary unless maintainers want to mention startup-performance work.
