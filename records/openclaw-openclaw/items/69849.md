## Summary

Keep this issue open. Current main still pins and resolves @agentclientprotocol/sdk 0.20.0, has no SDK patch for the generated ACP Zod schemas, and the published latest SDK 0.21.0 still shows the affected optional/nullish array fields without default empty arrays, so the reported Claude schema-validation failure is not fixed or superseded.

## What This Changes

Add default empty arrays to ACP SDK optional/nullish array schema fields so Claude-style strict tool schema validation accepts OpenClaw ACP registration.

## Best Possible Solution

Keep this issue open for maintainer decision on dependency strategy: either wait for a fixed upstream @agentclientprotocol/sdk/Claude ACP release, or approve a temporary OpenClaw SDK patch that adds default empty arrays to the affected generated schemas. The change should stay scoped to ACP SDK schema compatibility and include focused ACP/Claude schema coverage.
