## Summary

Keep open. Current main has moved the root ACP SDK pin to 0.21.0, but the published 0.21.0 generated Zod schema still leaves the affected optional/nullish arrays without default empty arrays, and the bundled ACPX Claude path still resolves transitive ACP SDK 0.20.0 through `@agentclientprotocol/claude-agent-acp` and `acpx`.

## What This Changes

The issue requests default empty arrays for optional ACP SDK array fields so Claude validates OpenClaw ACP and ACPX tool schemas.

## Best Possible Solution

Resolve this through an explicit dependency strategy: prefer bumping to upstream ACP SDK, Claude adapter, and ACPX releases that add default empty arrays to the generated schemas; if upstream is not available soon, approve a temporary OpenClaw patch for the SDK generated schema and keep it tightly scoped with package, lockfile, patch metadata, and focused ACP schema coverage aligned.

## Reproduction Assessment

Yes for the schema-contract reproduction: the current and latest published ACP SDK schema can be streamed and inspected to show the affected optional/nullish arrays still lack `.default([])`, while current main consumes that SDK in active ACP paths. A live Claude registration failure was not rerun during this read-only review.

## Solution Assessment

Unclear because no patch is proposed here. The narrowest maintainable solution is an upstream fixed dependency bump when available; a temporary SDK patch is reasonable only after maintainer approval because this repository treats dependency patches as explicit-owner decisions.
