## Summary

Keep #69849 open. Current main still depends on @agentclientprotocol/sdk 0.20.0, uses it directly for the ACP bridge, and bundles acpx 0.6.1, whose lockfile snapshot also resolves to the same SDK version. Inspecting the pinned SDK tarball shows the reported optional array schemas are still present without defaults, so the requested Claude schema-compatibility fix is not implemented on main and is not clearly superseded by the related provider-schema PRs.

## Best Possible Solution

Keep this open until OpenClaw can consume an ACP SDK/acpx release that adds safe defaults for optional array schemas, or carry an explicit dependency patch with regression coverage proving Claude-compatible ACP registration. The related open provider-schema work (#66720) is adjacent but does not supersede this SDK-level ACP schema issue.
