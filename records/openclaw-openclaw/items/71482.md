## Summary

Keep this PR open. Current main still loads failed or invalid session stores as `{}` and the memory-core dreaming scrubber still proceeds to archive old dreaming transcripts with an empty reference set. The PR addresses a real data-loss path, but the proposed unconditional empty-store skip is over-broad and would also disable legitimate orphan cleanup for agents with truly empty stores. Security review found no workflow, dependency, lockfile, release, secret, or package-publishing surface in the PR diff; the risk is cleanup semantics around user transcript files.

## Best Possible Solution

Keep this PR open and request a tighter memory-core revision: distinguish failed or untrusted session-store loads from valid empty stores before destructive orphan archival, preserve legitimate orphan cleanup, and add focused regression tests for malformed/unreadable store data plus true empty-store cleanup.
