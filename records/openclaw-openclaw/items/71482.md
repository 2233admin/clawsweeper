## Summary

Keep open; this PR targets a plausible data-loss path, but the patch is over-broad and should not merge as-is. Current code still makes `loadSessionStore()` collapse failed reads into `{}`, but skipping cleanup for every empty store would also disable legitimate orphan transcript cleanup for agents with no active session-store entries.

## What This Changes

The PR adds an early guard in `scrubDreamingNarrativeArtifacts()` that logs and skips dreaming orphan archival for an agent whenever `loadSessionStore()` returns an empty session-store object.

## Best Possible Solution

Keep the data-loss protection, but make the cleanup decision depend on whether the session store actually failed to load, not on whether the loaded store has zero entries. A focused fix should preserve orphan cleanup for successfully loaded empty stores, skip archival only for unreadable/invalid transient store reads, and add tests for both cases.

## Reproduction Assessment

Yes. A focused unit test can arrange an invalid or unreadable `sessions.json` plus old dreaming transcript files to reproduce the false-archive path, and a second test with a valid empty store plus stale dreaming transcript files reproduces the PR regression.

## Solution Assessment

No. The proposed guard is not the best fix because it treats a real empty store as a failed load; a safer solution distinguishes load failure from successful empty state before deciding whether to skip archival.
