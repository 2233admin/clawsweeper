## Summary

Keep #71482 open. Current main still has the plausible data-loss path: `loadSessionStore()` can silently return `{}` on read/parse failure, and the dreaming scrubber then treats every old dreaming-marked transcript not present in the empty reference set as an orphan. The PR is not obsolete, but its current fix is too broad because a genuinely empty session store is a normal state and should not disable orphan cleanup indefinitely.

## Best Possible Solution

Keep the PR open, but revise it before merge. The safest maintainer path is an error-aware scrubber load: skip archival only when the store file exists but cannot be read or parsed, while still allowing cleanup when the store is genuinely empty or missing as a normal state. Add focused tests for both cases: invalid/unreadable `sessions.json` must not archive active transcripts, and a legitimate empty store must still archive old dreaming-marked orphan `.jsonl` files.

## Risks / Open Questions

- The PR as written trades the reported false-archival risk for a cleanup regression: agents with legitimately empty stores may accumulate old dreaming transcripts forever.
- Changing `loadSessionStore()` to throw globally would conflict with existing callers/tests that rely on graceful `{}` for missing or invalid stores, so the fix should be opt-in or local to the scrubber path.
