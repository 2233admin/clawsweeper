## Summary

Keep open. Current main still does not implement the PR's intended core MCP SecretRef support: `mcp.servers.*.env.*` and `mcp.servers.*.headers.*` remain primitive-only config/type surfaces, are absent from the command/runtime SecretRef target list, are not collected by the core runtime resolver, and are not documented as supported core SecretRef targets. The PR remains a live implementation candidate, but it expands a security-sensitive SecretRef/audit/configure surface and should go through normal maintainer plus secops review rather than being closed as obsolete.

## Best Possible Solution

Keep this PR open and route it through maintainer plus secops review. The best path is a focused implementation that adds `SecretInput` support to core `mcp.servers` env/header schema and types, registers those two SecretRef targets, resolves only active MCP transport surfaces at runtime, covers `secrets configure` and `secrets audit` heuristics with regression tests, updates generated config/docs artifacts, and coordinates active-transport assumptions with #72515.
