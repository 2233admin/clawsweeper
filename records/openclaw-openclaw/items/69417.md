## Summary

Keep open. Current main still does not implement first-class SecretRef/SecretInput support for core `mcp.servers.*.env.*` and `mcp.servers.*.headers.*`: the public type and Zod schema are primitive-only, the supported SecretRef target registry and command target sets do not include those MCP paths, and the core/agent runtime secret collectors do not resolve them. Current main does have generic `${VAR}` config substitution, which covers a narrower MCP header/env string-template symptom, but it does not supersede this PR's structured SecretRef registry, audit/configure, docs, and active-transport behavior. Related ACPX/plugin MCP SecretRef support, scalar MCP env propagation, static header forwarding, and dynamic per-request header work are distinct.

## Best Possible Solution

Keep the PR open, or replace it with a focused maintainer-owned patch that adds `SecretInput` to core MCP env/header schema and types, registers `mcp.servers.*.env.*` and `mcp.servers.*.headers.*` as supported SecretRef targets, includes them in agent command/runtime resolution, resolves only the active MCP transport surface, updates `secrets configure` and `secrets audit` heuristics, and documents/tests the new credential surface.
