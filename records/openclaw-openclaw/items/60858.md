## Summary

Keep open: current main still contains the empty-message compaction skip and fallback-boundary paths, and the discussion has fresh multi-channel repros showing the recurring safeguard behavior after the #71325 mitigation. The active #69270 PR is a plausible fix candidate, but it is open and not present on current main.

## What This Changes

The issue reports that empty or unpopulated compaction message arrays cause real-content guards to skip compaction or repeatedly write fallback boundaries instead of compacting long sessions.

## Best Possible Solution

Preserve the empty-preparation safety boundary, but reconcile trigger and accounting state so fallback or zero-message compactions reset stale usage and do not keep scheduling compaction. The final fix should cover manual `/compact`, auto-compaction, context-engine owned compaction, and preflight transcript-size paths with focused regression coverage.
