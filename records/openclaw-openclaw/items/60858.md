## Summary

Keep open. Current main still has the no-real-conversation guards that skip direct compaction or write only a fallback boundary, and the fresh issue discussion says the #71325 safeguard is only damping repeated triggers, not fixing the upstream empty-message condition. #69270 looks like the active implementation candidate, but it is still open and not merged into current main.

## Best Possible Solution

Keep this issue open and use it as live repro evidence while reviewing or replacing #69270. The fix should restore the compaction/session invariants upstream: compaction should either receive real persisted conversation messages or avoid scheduling repeated compaction when none exist, with regression coverage for manual and automatic compaction across channel-scoped sessions, context-engine windowing, and empty `session.messages` with real transcript content on disk.
