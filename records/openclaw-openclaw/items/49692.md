## Summary

Keep open. Current main has friendlier restart wording, but inbound work during the restart drain window is still rejected before it enters the preserved command queue, so the triggering user message can still be lost. The related Kafka plugin PR was closed unmerged and would not replace native in-process replay for this UX path.

## What This Changes

Request to soften gateway restart/drain UX and queue/replay inbound messages received during the restart drain window so users do not have to resend them.

## Best Possible Solution

Native in-process restart handling should accept inbound command entries during the SIGUSR1 drain window, preserve FIFO lane ordering without starting new work while draining, and let resetAllLanes replay the preserved queue after the gateway starts again; durable file-backed recovery can remain a separate follow-up.
