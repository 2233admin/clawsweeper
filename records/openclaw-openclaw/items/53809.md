## Summary

Keep this PR open: the linked data-loss bug is real and current main still has an unguarded gateway `agents.delete` path, but the PR is not merge-ready because its exact-path guard is narrower than the overlap-aware CLI behavior already shipped on main. PR URL: https://github.com/openclaw/openclaw/pull/53809

## What This Changes

The PR adds an exact realpath-normalized workspace lookup, uses it in CLI identity/delete and gateway agent deletion, and adds tests for preserving exact shared workspaces.

## Best Possible Solution

Use one shared, realpath-normalized, overlap-aware workspace ownership helper for both CLI and gateway deletion. Preserve the existing CLI JSON/docs behavior, add gateway coverage for exact, parent, child, symlink, and unique workspace cases, and keep protocol/client schemas consistent with any gateway result fields that are intentionally exposed.

## Reproduction Assessment

Yes. The PR body gives a direct shared-workspace reproduction, and current main shows the gateway path still calls `moveToTrashBestEffort(workspaceDir)` whenever `deleteFiles` is true; the existing CLI tests provide concrete exact/child/parent/symlink fixtures for the expected retention behavior.

## Solution Assessment

No. The proposed exact-match helper is not the best fix because it misses parent/child workspace overlap cases and conflicts with the broader CLI behavior already shipped; the safer path is to extract/reuse the current overlap-aware CLI logic for gateway deletion.
