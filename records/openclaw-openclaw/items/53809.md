## Summary

Keep this PR open, but do not merge it as-is. Current main still has the gateway/UI delete data-loss gap, while this PR's exact-match guard is narrower than the shipped CLI behavior and would miss parent/child workspace overlaps that current docs and tests protect.

## What This Changes

The PR adds an exact realpath-normalized shared-workspace lookup, uses it in CLI identity/delete and gateway agent deletion, and adds tests for exact shared-workspace preservation.

## Best Possible Solution

Use one realpath-aware bidirectional workspace-overlap helper for delete retention, apply it to the gateway path without weakening current CLI behavior, keep the CLI JSON contract intact, and add gateway regression tests for exact, parent, child, symlink, and unique workspace cases.
