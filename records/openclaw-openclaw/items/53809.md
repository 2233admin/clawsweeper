## Summary

Keep open. Current main has already shipped the CLI-side shared-workspace protection, but the gateway `agents.delete` path still trashes the workspace unconditionally. This PR is useful source work, but it is not merge-ready because its exact-path guard is narrower than the current CLI overlap semantics and would not protect parent/child workspace overlap cases.

## What This Changes

The PR adds an exact-workspace path helper and threads it through CLI and gateway agent deletion, set-identity lookup, and related tests so exact shared workspaces are preserved while agent-specific state is removed.

## Best Possible Solution

Land an update or narrow replacement that reuses/extracts the current CLI overlap and realpath workspace-retention logic for gateway `agents.delete`, keeps agent-specific state deletion unchanged, preserves the shipped CLI behavior, and adds gateway regression coverage for same, nested, parent, symlink, and unique workspace cases.
