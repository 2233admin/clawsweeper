## Summary

Keep this PR open. Current main has the shared-workspace guard for the CLI `agents delete` path, but the gateway/UI `agents.delete` handler still trashes `workspaceDir` unconditionally when `deleteFiles` is true. This PR's stated scope explicitly includes both CLI and gateway delete paths, so main does not yet implement the full intended change.

## Best Possible Solution

Keep this PR open or replace it with a fresh targeted fix that applies the current CLI overlap/symlink workspace-retention semantics to the gateway `agents.delete` handler, adds a gateway regression test for shared workspaces, and leaves the shipped CLI JSON/docs behavior intact.
