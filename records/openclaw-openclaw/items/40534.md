## Summary

Keep open: current main still has the core reasoning-marker propagation and cron filtering gaps this PR is trying to address, but the branch is stale against the current shared cron outcome flow and would miss multi-payload delivery plus interim/final callers unless it is rebased and moved into `resolveCronPayloadOutcome`.

## What This Changes

The PR propagates `isReasoning` through embedded run payload serialization and typing, filters cron announce helper selection plus caller fallback/error-recovery logic, and adds focused helper tests.

## Best Possible Solution

Land one canonical rebased fix in the shared cron outcome path: preserve `isReasoning` from `buildEmbeddedRunPayloads`, skip reasoning in summary/output/fallback, deliverable and multi-payload selection, and post-error recovery, with regression coverage for mixed reasoning plus final text, reasoning-only, and error-then-reasoning-only cron announce cases.
