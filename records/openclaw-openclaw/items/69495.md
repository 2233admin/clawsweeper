## Summary

Keep this PR open. Current main still does not implement the intended heartbeat `{ primary, fallbacks }` support: the schema and public type remain string-only, the heartbeat runner only forwards a primary model override, and the reply fallback path has no per-run heartbeat fallback override. The PR is the active implementation lane for #69434, but the open review comments around explicit empty fallback arrays still need to be addressed before merge.

## Best Possible Solution

Keep this PR open as the active implementation lane. Before merge, update it so `heartbeat.model` reuses the shared model schema/types, the heartbeat runner forwards primary plus fallback overrides including explicit empty arrays, `FollowupRun` and fallback resolution honor the per-run override, direct heartbeat reply fallback behavior is consistent, pricing cache/docs/generated schema/help/tests are aligned, and the changelog entry includes the required contributor attribution.
