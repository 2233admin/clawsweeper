## Summary

Keep this PR open. Current main still computes both preflight-compaction and memory-flush token gates from `reserveTokensFloor` only, while config help and the Pi settings path treat the floor as a minimum under `reserveTokens`. The PR remains the active implementation path for open #66830; the earlier similar PR #67297 is closed unmerged.

## Best Possible Solution

Keep this PR open and review/land this implementation or an equivalent `max(reserveTokens, reserveTokensFloor)` calculation through the preflight-compaction and memory-flush gates. Keep the regression coverage and changelog credit, then let open #66830 close after the PR merges; #67297 can remain the superseded closed attempt.
