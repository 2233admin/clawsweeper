## Summary

Keep this PR open. Current main still computes the preflight-compaction and memory-flush token thresholds from `reserveTokensFloor` only, even though config help and the Pi settings-manager path treat `reserveTokensFloor` as a floor under `reserveTokens`. The PR remains the active implementation candidate for #66830; the earlier related PR #67297 is closed unmerged. Security pass found no supply-chain or infrastructure changes in the PR diff surface.

## Best Possible Solution

Keep this PR open as the canonical implementation path for #66830. Land this helper or equivalent `max(reserveTokens, reserveTokensFloor)` logic through both preflight-compaction and memory-flush gates, keep the regression tests and changelog credit, then let the linked issue close after merge.
