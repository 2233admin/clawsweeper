## Summary

Current main still does not implement the PR's intended Windows fixes. PowerShell is advertised as a completion target and `.ps1` cache generation exists, but the installer still rejects PowerShell in `completion-runtime`, still emits POSIX `source` syntax, and Windows time-format detection still hard-codes `powershell` instead of using the existing PS7-preferring resolver. The PR remains useful, including its follow-up reload-hint fix, but it needs to be ported to the current runtime layout before it can land.

## Best Possible Solution

Keep this PR open and either rebase/port it or land a replacement that updates `src/cli/completion-runtime.ts` to install PowerShell completion with `. "..."`, emits a PowerShell-safe reload hint, updates `src/agents/date-time.ts` to use `resolvePowerShellPath()` with non-interactive flags, and adds focused regression tests for PowerShell profile install/detection plus Windows date-time executable selection.
