## Summary

Current main still does not implement this PR's intended Windows fixes. PowerShell is advertised as a completion target and `.ps1` cache generation exists, but `installCompletion()` still rejects the supported `powershell` shell, profile install/reload text still uses POSIX `source`, and Windows time-format detection still hard-codes `powershell` rather than the existing PS7-preferring resolver. The PR remains useful, but it needs to be ported from the old `completion-cli.ts` install layout to current `completion-runtime.ts`.

## Best Possible Solution

Keep this PR open and either rebase/port it or land a replacement that updates `src/cli/completion-runtime.ts` to install PowerShell completion with `. "..."`, emits a PowerShell-safe reload hint, updates `src/agents/date-time.ts` to use `resolvePowerShellPath()` with non-interactive flags, and adds focused regression tests for PowerShell profile install/detection plus Windows date-time executable selection. Coordinate reload/profile wording with related #44296.
