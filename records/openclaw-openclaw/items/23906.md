## Summary

Keep this issue open. Current main still has no per-call `sessions_spawn` tool selector or tool-less mode: the public tool schema, `SpawnSubagentParams`, and gateway `agent` RPC contract do not carry a `tools`, `toolsAllow`, or `disableTools` field. Static subagent tool policy and lower-level embedded-run primitives exist, but the requested API/plumbing and plugin/MCP semantics remain unresolved.

## Best Possible Solution

Keep this open and let maintainers choose the canonical per-spawn tool-policy API across this issue, #15032, and #62695. The implementation should add a backward-compatible `sessions_spawn` field, validate/document it, carry it through `SpawnSubagentParams` and the gateway `agent` RPC, map tool-less runs to embedded `disableTools:true`, map subsets to `toolsAllow`, and define plugin/MCP resolution plus precedence against existing tool policies.
