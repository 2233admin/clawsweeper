## Summary

Keep open. Current main still lacks a per-spawn tool selector or tool-less mode on `sessions_spawn`: the public tool schema, native spawn params, child `agent` RPC schema, and spawn handoff do not carry `tools`, `toolsAllow`, `disableTools`, or deny-list semantics. Lower-level embedded-run support exists, but it is not exposed through this API, and the issue comments add unresolved plugin/MCP tool-resolution behavior that should be decided with #15032 and #62695.

## What This Changes

Add a per-spawn tool override to `sessions_spawn` so spawned subagents can receive all tools, no tools, or a selected subset for local-model and least-privilege workflows.

## Best Possible Solution

Keep this open and have maintainers choose the canonical per-spawn tool-policy API across this issue, #15032, and #62695. A complete fix should add a backward-compatible `sessions_spawn` field, validate and document it, carry it through `SpawnSubagentParams` and the gateway `agent` RPC, map tool-less runs to embedded `disableTools: true`, map subsets to `toolsAllow`, and define plugin/MCP tool resolution plus precedence against existing policy layers.
