## Summary

Keep this PR open. Current main still does not implement the intended chat-channel session switching: `/sessions` is not registered or loaded, `/session` only supports `idle` and `max-age`, and the session config/store types do not include `session.historyLimit` or a per-SessionKey `sessionHistory` retention model. The PR remains a plausible implementation candidate, but the review discussion shows session-state safety questions that should be resolved through normal maintainer review before merge.

## Best Possible Solution

Keep this PR open for normal maintainer review or replacement by a successor implementation. The best path is a focused chat command implementation that registers `/sessions`, supports `/session <number|sessionId|back>`, persists bounded per-SessionKey history, deliberately snapshots/restores or clears all session-scoped runtime and security fields, documents `session.historyLimit`, and includes regression coverage for direct-chat switching, disabled history, transcript paths, LRU eviction, in-flight run handling, lifecycle hooks, and snapshot restore.
