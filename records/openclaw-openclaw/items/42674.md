## Summary

Keep this PR open. Current main still advertises inline attachment fields on shared `message.send`, but the active Telegram plugin path does not hydrate, validate, or forward `buffer`/`filename`/`contentType` to the Telegram transport. The PR is still a relevant implementation candidate for #41779, but it needs a current-main refresh and a stronger pre-decode base64 size guard before it is mergeable.

## Best Possible Solution

Keep this PR open as a useful implementation candidate for #41779. A mergeable version should refresh onto current main, coordinate the generic `send` media hydration work in #63578, implement the Telegram-specific forwarding and transport support under `extensions/telegram/src/action-runtime.ts`, `extensions/telegram/src/channel-actions.ts`, and `extensions/telegram/src/send.ts`, add focused regression coverage for buffer-only sends and data-URL normalization, and enforce a pre-decode base64 size bound before constructing a `Buffer`.
