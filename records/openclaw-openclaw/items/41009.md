## Summary

Keep open: current main still lacks the requested Serve/Funnel legacy bind migration, so the PR is not obsolete. The branch should not merge as-is because it weakens the current loopback-only validation for explicit `gateway.bind="auto"` and is stale against the current doctor/shared migration split.

## What This Changes

The PR adds legacy config detection, migration logic, shared bind-resolution helpers, validation reuse, and regression tests to normalize explicit legacy non-loopback `gateway.bind` values to `loopback` for Tailscale Serve/Funnel configs.

## Best Possible Solution

Refresh the fix on current main by adding a source-literal legacy rule plus a doctor/shared runtime gateway migration for `lan`, `tailnet`, and non-loopback `custom` under `serve` or `funnel`; keep explicit `auto` rejected by validation; and cover startup/doctor migration triggering, migration ordering before `allowedOrigins` seeding, and validation with focused tests.
