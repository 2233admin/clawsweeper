## Summary

Keep this open: current main still rejects the reported safe `diagnostics.cacheTrace` patch, and the linked implementation PR is open and unmerged, so this issue should remain open until PR #73098 or an equivalent fix lands.

## What This Changes

The issue requests a safe gateway `config.patch` path for enabling `diagnostics.cacheTrace` while keeping raw message, prompt, and system-prompt capture disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow change, so gateway `config.patch` and `config.apply` can enable cache tracing only when the resulting diagnostics config cannot capture raw content, with regression coverage for safe enablement and unsafe attempts.

## Reproduction Assessment

Yes. Static inspection of current main gives a high-confidence reproduction path: the reported patch changes `diagnostics.cacheTrace` leaves, those leaves are absent from the gateway allowlist, and the guard rejects disallowed paths before writing.

## Solution Assessment

Yes. A conditional allow based on the resulting `cacheTrace` config is the narrowest maintainable fix; unconditionally allowing the subtree would be less safe because the content-capture fields default to true.
