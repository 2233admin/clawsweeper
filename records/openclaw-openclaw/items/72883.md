## Summary

Keep open: current main still rejects the reported safe cache-trace patch through the agent-facing gateway tool, and PR #73098 is open, unmerged, and explicitly linked as the closing implementation candidate.

## What This Changes

The issue requests a supported gateway `config.patch` path for enabling `diagnostics.cacheTrace` when message, prompt, and system content capture are disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow gateway-tool guard, so safe cache-decision observability is allowed only when the resulting cache-trace config has message, prompt, and system content capture disabled.

## Reproduction Assessment

Yes. Static inspection of current main is enough: the reported patch changes `diagnostics.cacheTrace` leaf paths that are not allowlisted, and the guard throws before the gateway write call.

## Solution Assessment

Yes. A conditional allow path based on the post-merge cache-trace config is the narrowest maintainable fix; unconditionally allowing the whole subtree would be riskier because the content-capture flags default to `true`.
