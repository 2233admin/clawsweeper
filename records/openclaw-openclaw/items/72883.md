## Summary

Keep this open: current main still rejects the reported safe cache-trace patch through the agent-facing gateway config guard, and PR #73098 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests a supported gateway config.patch path for enabling diagnostics.cacheTrace observability when message, prompt, and system content capture are explicitly disabled.

## Best Possible Solution

Land PR #73098 or an equivalent narrow fix that permits cache-trace enablement only when the resulting config keeps includeMessages, includePrompt, and includeSystem false, with regression coverage for safe enablement and unsafe content-capture attempts.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the failure path: the reported patch changes diagnostics.cacheTrace leaves, those leaves are not allowlisted, and the gateway tool rejects them before forwarding config.patch.

## Solution Assessment

Yes. A conditional allow based on the resulting cacheTrace configuration is the narrowest maintainable direction; unconditionally allowing the subtree would weaken the existing fail-closed control-plane policy.
