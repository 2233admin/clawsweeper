## Summary

Current main still has the reported memory-wiki contract mismatch. `sources/*.md` files are compiled and linted as structured wiki source pages requiring `id`, `pageType`, and freshness metadata, while the Memory Wiki docs still describe `sources/` as imported raw material. The implementation consistently wraps raw imports in source-page frontmatter plus fenced content, so the narrowest safe fix is documentation clarification unless maintainers intentionally design a raw-file mode.

## Best Possible Solution

Keep this open and align the Memory Wiki source-page contract. The safest focused fix is docs-only: clarify that `sources/*.md` are structured wiki source pages with frontmatter, and that raw originals are imported through `openclaw wiki ingest`, bridge, or unsafe-local and preserved inside source-page content/provenance. If maintainers want direct raw files in the vault, track that separately as an explicit raw mode or ignored subdirectory rather than disabling source-page lint.

Fix the Memory Wiki `sources/` frontmatter ambiguity reported in https://github.com/openclaw/openclaw/issues/69700. Update docs only unless you find maintainer-approved raw-mode direction. Mention that direct `.md` files placed under `sources/` are part of the wiki contract and need frontmatter. No changelog entry is expected for docs-only clarification.
