## Summary

Keep open: current main still has the reported Memory Wiki contract mismatch. The implementation treats `sources/*.md` as structured source pages that compile and lint with required frontmatter, while the public Memory Wiki docs still describe `sources/` as imported raw material. This is actionable as a narrow docs-contract clarification rather than a cleanup close.

## What This Changes

The issue asks to resolve a Memory Wiki mismatch where `openclaw wiki lint` requires frontmatter for `sources/*.md` files even though docs describe `sources/` as raw imported material.

## Best Possible Solution

Clarify the Memory Wiki docs and CLI docs that `sources/*.md` are structured wiki source pages requiring page metadata, and that raw originals are preserved through ingest, bridge, or unsafe-local import content/provenance. A direct raw-drop folder should be tracked separately as a new mode if maintainers want it.

## Reproduction Assessment

Yes. Reading the current code gives a high-confidence reproduction path: place a Markdown file without frontmatter under top-level `sources/`, then lint will classify it as a source page and emit missing `id`, missing `pageType`, and missing freshness diagnostics.

## Solution Assessment

Yes. The narrowest maintainable fix is to align the docs with the existing structured source-page contract; exempting all `sources/` files from lint would conflict with compile, query, status, ingest, and test expectations for source-page metadata.
