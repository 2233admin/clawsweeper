## Summary

Keep open. Current main has an adjacent runtime dispatcher FormData normalizer, but the env proxy wrapper named in the report still calls undici.fetch directly with the caller's init body, and the audio runner still injects that wrapper when HTTP_PROXY/HTTPS_PROXY is configured. There is also an open contributor PR with closing syntax for this issue, so the issue should remain open until that PR or a replacement lands.

## What This Changes

The issue reports that proxy-backed audio transcription and batch uploads serialize global FormData as the literal string [object FormData] instead of multipart/form-data.

## Best Possible Solution

Land or replace #48556 with a narrow proxy-fetch fix that preserves native fetch-compatible multipart semantics under HTTP_PROXY/HTTPS_PROXY, with regression tests for both makeProxyFetch and resolveProxyFetchFromEnv using global FormData and undici FormData.
