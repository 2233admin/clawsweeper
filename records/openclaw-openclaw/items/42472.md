## Summary

Keep this open. Current main still strips a trailing NO_REPLY token and delivers the preceding text in the shared reply normalizer and outbound delivery projection, while only narrower exact-token, JSON-envelope, leading-token, and cron announce paths have been hardened. The issue is also paired with open implementation PRs, including one with closing syntax, so maintainer consolidation is the right next step rather than auto-closing or starting a separate repair lane.

## What This Changes

The issue requests suppressing the entire user-visible reply when model output includes or trails with NO_REPLY after internal preamble text.

## Best Possible Solution

Land one canonical fix, or a consolidated replacement, that defines a shared silent-intent helper and applies it consistently across reply normalization, outbound directives/delivery projection, gateway live/history projection, cron/subagent delivery, and runtime delivery classification, with tests for exact tokens, own-line/trailing tokens, inline false positives, media preservation, and streaming limits.
