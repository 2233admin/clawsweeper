## Summary

Keep this PR open. Current main still treats auth-profile and same-version auth-epoch mismatches as invalidations without returning the existing CLI session id, so the branch’s core behavior is not implemented. Newer main has compaction-based fresh-session reseeding, but that is narrower than preserving the active CLI session and does not cover ordinary uncompacted conversations.

## What This Changes

The PR changes CLI session reuse so auth-profile/auth-epoch mismatches keep the existing session id, prioritizes system-prompt and MCP invalidations over auth-only reuse, updates the CLI session reset log wording, and adjusts cli-session tests.

## Best Possible Solution

Land a current-main version of this fix that keeps config invalidations fresh, preserves `mcpResumeHash` and auth epoch version semantics, returns the existing CLI session id for auth-only profile/epoch changes when policy allows, and updates regression tests for auth-only plus mixed auth/config changes.
