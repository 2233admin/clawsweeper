## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reuse the selected routed provider config:** `extensions/speech-core/src/tts.ts:1830`
  - body: After preflight selects `initialRoute`, this line resolves config for `effectiveProvider` instead of the provider that actually produced routable text. If the preferred provider was skipped or threw while a fallback was routable, this can still throw from the skipped provider or gate the prepared text with the wrong model; carry the selected provider/config out of preflight and use that route consistently.
  - confidence: 0.87
- **[P2] Preserve line-start markdown reference labels:** `src/shared/text/emotion-tags.ts:221`
  - body: At line start, this guard returns false, so a valid markdown reference pair such as `[fast][1]` is treated as emotion syntax and the first label is stripped in `off`/`on` display paths. The branch documents this as #72821, but this PR introduces the sanitizer path, so it should preserve line-start reference links before merge unless a maintainer explicitly accepts the follow-up.
  - confidence: 0.79

  - body: This docs line exposes a new `/emotions` user command, but the PR patch does not add a matching `CHANGELOG.md` entry. Repo policy requires user-facing feature changes to appear in the active changelog with appropriate contributor credit, so the release notes would miss this command.
