## Summary

Keep this PR open. Current main still has 11 custom checkout blocks in `.github/workflows/ci.yml` that derive a Basic auth header from `github.token` and pass it to Git without registering the derived `auth_header` with `::add-mask::`. The PR is still useful and narrow: the provided diff only adds the mask directive after each header construction, but it touches CI token handling and should get maintainer security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open and review it as a focused CI security-hardening contribution. The best path is to land the mask additions, or an equivalent maintainer-owned variant, with no unrelated workflow graph, permission, action-ref, dependency, package, script, or release changes, then validate with the workflow security checks and normal CI for the PR head.
