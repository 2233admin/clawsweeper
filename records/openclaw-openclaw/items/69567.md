## Summary

Keep open. The PR is a valid CI token-log hardening change for the 11 custom checkout blocks in `.github/workflows/ci.yml`, but the same derived Basic-auth checkout header remains unmasked in both Testbox workflows, so merging this branch as-is would leave part of the reported exposure unfixed.

## What This Changes

The PR adds `echo "::add-mask::${auth_header}"` after each derived checkout `auth_header` construction in the 11 custom checkout blocks in `.github/workflows/ci.yml`.

## Best Possible Solution

Extend the focused log-redaction hardening so every current custom checkout block that constructs `auth_header` masks it immediately after construction and before any Git command uses it, covering `.github/workflows/ci.yml`, `.github/workflows/ci-check-testbox.yml`, and `.github/workflows/ci-build-artifacts-testbox.yml` without changing job graph, permissions, runners, action refs, or checkout semantics.
