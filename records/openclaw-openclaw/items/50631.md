## Summary

Keep this PR open. Current main still allows `gateway.tailscale.mode="serve"` with `gateway.auth.mode="none"`, the current tests preserve that successful path, and the PR is paired with the same-author open bug report #50630. The patch is a focused gateway-auth hardening candidate, but it should also cover install/repair validation before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate for #50630. Rebase it onto current main, replace the current Serve plus `auth.mode="none"` success test with a rejection test, add install/doctor/service-repair validation or normalization before service write, and preserve allowed coverage for local loopback `none`, Serve with token/password/trusted-proxy as intended, and Funnel's password-only rule.
