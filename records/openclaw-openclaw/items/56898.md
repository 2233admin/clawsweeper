## Summary

Keep this issue open. Current main still ships the bundled `wacli` skill with only minimal confirmation guidance; it does not include the requested recipient allowlist concept, redacted audit logging instructions, or bulk-send protections. This is not a ClawHub cleanup candidate because the affected surface is an already-bundled skill and the request is safety hardening for that shipped guidance.

## Best Possible Solution

Keep the issue open and implement a focused safety update to `skills/wacli/SKILL.md`: add recipient allowlist guidance, require explicit confirmation showing recipient identity/number plus full message/media before every send, add redacted audit-log instructions, and prohibit bulk/broadcast sends without full recipient-list and per-recipient confirmation. If maintainers want stronger guarantees than instructions can provide, track a separate wrapper/plugin enforcement design.
