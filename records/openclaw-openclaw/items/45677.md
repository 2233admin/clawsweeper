## Summary

Current main still has the reported RuntimeLogger metadata drop, so this PR is not obsolete and should stay open. The requested fix is valid, but the branch should be changed before merge because the underlying OpenClaw logger treats the first plain object argument as structured metadata and the last string as the message; forwarding `(message, meta)` likely will not preserve metadata as structured fields. The PR also carries unrelated test-only changes that should be split or rebased out.

## Best Possible Solution

Keep this PR open and request changes. Implement `createRuntimeLogging()` as a small adapter from the public `RuntimeLogger` API to the underlying structured logger convention: when `meta` is a non-empty record, call `logger.<level>(meta, message)`; otherwise call `logger.<level>(message)`. Add a regression test that proves structured diagnostic/file-log attributes survive through the runtime logger, then remove the unrelated test-only CI restore changes before merge.
