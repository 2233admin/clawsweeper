## Summary

Keep open. Current main still does not provide a channel slash command for manually executing Lobster or Task Flow workflows; the existing surfaces are status/inspection/cancel commands, the optional Lobster agent tool, and an external webhook plugin path. This is valid feature work, but it needs maintainer design because it would expose workflow execution from chat channels with authorization, cwd, JSON parsing, approval, and resume semantics.

## What This Changes

The issue requests a slash command to manually execute Task Flow or Lobster workflows from a channel, optionally with JSON arguments.

## Best Possible Solution

Keep the issue open for maintainer design. If accepted, implement the command in the Lobster/plugin-owned surface with `api.registerCommand()`, reuse the existing Lobster runner and managed Task Flow runtime, parse JSON arguments with clear user-safe errors, preserve approval/resume semantics, and document the channel security model. If maintainers decide this should not be bundled, move it to a ClawHub plugin only after confirming the current SDK can safely express the required session binding and execution authority.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the missing capability: `/tasks anything` returns usage only, Task Flow CLI exposes list/show/cancel, and Lobster registers an optional agent tool rather than a slash command.

## Solution Assessment

Unclear. A slash command is a plausible narrow solution for the reported channel workflow, but the exact command syntax and safety policy need maintainer judgment before this is a safe autonomous fix PR.
