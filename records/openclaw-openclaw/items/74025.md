## Summary

Keep open. Current main still exposes Lobster as an optional agent tool and Task Flow as inspect/cancel/status surfaces, but it does not provide an explicit channel slash command to manually run or resume Lobster/Task Flow workflows. The feature is valid, but channel-triggered workflow execution is side-effectful enough that the command contract and authorization/approval model need maintainer design before implementation.

## What This Changes

Request to add a slash command that manually runs or resumes Task Flow/Lobster workflows, with optional JSON arguments.

## Best Possible Solution

Keep the issue open for maintainer design, then implement a Lobster-owned plugin command rather than a core special case. The command should reuse the existing Lobster runner and managed TaskFlow runtime, bind to the active authorized session, parse JSON arguments deliberately, preserve approval/resume semantics, and add focused docs/tests for channel usage and failure cases.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence verification path. Current source and docs inspection shows no /lobster, /taskflow, or flow-run command, while /tasks rejects arguments and only reports status.

## Solution Assessment

Unclear until maintainers settle the command contract. A plugin-owned command wrapping existing Lobster/TaskFlow execution looks narrower than adding a core workflow command, but the security and UX details are not settled enough for an automated fix PR.

## Risks / Open Questions
