## Summary

Keep open. Current main still lacks the requested explicit slash command for manually executing Task Flow or Lobster workflows from a channel. Lobster is available as an optional agent tool, and Task Flow has status/inspection/cancel surfaces, but there is no `/lobster`, `/taskflow`, `/flow run`, or equivalent command path.

## Best Possible Solution

Keep this issue open for maintainer design. The likely implementation should live in the Lobster plugin as an owner-authorized `api.registerCommand()` command that reuses the existing Lobster runner and managed TaskFlow runtime, with structured JSON parsing, safe cwd/path handling, approval/resume semantics, user-safe errors, docs, and focused tests. If maintainers decide it should not be bundled, move it to a ClawHub plugin only after confirming the plugin API can safely provide the required execution and session-binding model.
