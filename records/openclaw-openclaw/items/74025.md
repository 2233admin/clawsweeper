## Summary

Keep open. Current main still exposes Lobster run/resume through the optional agent tool path and Task Flow through inspection/cancel surfaces, but it does not provide a channel slash command to manually run or resume workflows; the feature is plausible but needs maintainer design before an automated repair PR is safe.

## What This Changes

The issue requests a chat slash command for manually running or resuming Lobster and Task Flow workflows, with optional JSON arguments.

## Best Possible Solution

Keep this open for maintainer design, then implement a Lobster-owned plugin command if approved. It should reuse the existing Lobster runner and managed Task Flow runtime, bind only to the active authorized session, parse JSON arguments with an explicit schema and size limits, preserve approval/resume semantics, and update command docs plus focused tests without adding bundled workflow policy to core.

## Reproduction Assessment

Not applicable as a runtime bug, but yes for verifying the feature gap: source and docs inspection on current main show no /lobster, /taskflow, or flow-run command, while /tasks rejects arguments and only reports status.

## Solution Assessment

Unclear until maintainers settle the command contract. A plugin-owned command wrapping the existing Lobster/Task Flow execution paths appears narrower than a core workflow command, but the security and UX details are not defined enough for an automated fix PR.
