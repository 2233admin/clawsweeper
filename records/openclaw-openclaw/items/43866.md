## Summary

Keep open. Current main still drops `accountId` before `/hooks/agent` reaches cron delivery: the direct hook payload/normalizer, mapped hook action surface, replay cache scope, and hand-built delivery object all omit it, while the cron delivery pipeline already supports `delivery.accountId`. The linked closing PR #43951 remains open, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests preserving `/hooks/agent` `accountId` through hook normalization, replay scoping, and cron delivery dispatch so multi-account channel delivery can resolve the intended account.

## Best Possible Solution

Land or replace the focused hook bridge fix so direct and mapped hooks accept a trimmed `accountId`, include it in idempotency scope, construct `CronDelivery.accountId`, document the public hook field, and add direct hook, mapped hook, and isolated cron delivery regression tests.
