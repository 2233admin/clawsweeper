## Summary

Keep open. Current main still intentionally isolates Telegram forum topics and Telegram DM topics into topic/thread-specific session keys, while preserving topic/thread metadata for prompt and reply routing. There is no Telegram config schema, docs entry, or test coverage for an opt-in shared session mode, and the related items in the provided context cover narrower command targeting or topic-label display rather than this shared-context behavior.

## Best Possible Solution

Keep this open for a Telegram-plugin-owned design. If maintainers accept the behavior, add an explicit opt-in setting for forum groups and DM topics, preserve `MessageThreadId`/topic metadata and reply delivery targets, define how shared sessions interact with session tools and sandbox visibility, and add focused Telegram docs plus regression tests for forum topics and DM threads.
