## Summary

Keep open. Current main still intentionally isolates Telegram forum topics and Telegram DM threads into topic/thread-specific sessions while preserving topic metadata and reply routing. There is no accepted opt-in shared session mode in the Telegram config schema, docs, or tests, and the related reports cover narrower command targeting or topic-label display rather than shared cross-topic context.

## Best Possible Solution

Keep this open for Telegram-plugin-owned maintainer design. If accepted, add an explicit opt-in shared session mode for forum groups and DM topics, keep topic/thread metadata in prompt/session metadata, preserve reply delivery to the originating thread, define interactions with session tools/sandbox visibility, and add focused Telegram config docs plus regression tests for forum topics and DM threads.
