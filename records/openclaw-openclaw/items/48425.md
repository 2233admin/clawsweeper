## Summary

Keep this PR open. Current main still serializes full session entries and still sizes disk budget from unfiltered JSON, so the PR’s intended fix is not implemented. The PR remains a focused candidate for the sessions.json bloat reports, but it should be revised before merge because the current replacer drops any nested key named skillsSnapshot, not only the top-level SessionEntry field.

## Best Possible Solution

Keep this PR open and revise it into a session-entry-aware serialization helper that strips only the intended top-level SessionEntry.skillsSnapshot field, preserves systemPromptReport, makes disk-budget sizing use the exact same serialization path as saveSessionStoreUnlocked, and adds regression coverage proving nested/plugin-owned skillsSnapshot keys survive save/load. Then review and land it against the linked sessions.json bloat reports, especially open #45718.
