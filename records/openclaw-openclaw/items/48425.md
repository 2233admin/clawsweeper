## Summary

This PR should stay open. Current main still writes `skillsSnapshot` into `sessions.json` with the normal full-store `JSON.stringify` path, and the disk-budget estimator still measures the unfiltered store. The PR is a plausible fix for the linked session-store bloat reports, but it is not obsolete or implemented on main, and the remaining review concern is to scope the replacer so it only removes the intended `SessionEntry.skillsSnapshot` field.

## Best Possible Solution

Keep this PR open. The best path is to revise it so serialization strips only top-level `SessionEntry.skillsSnapshot`, preserves `systemPromptReport`, makes disk-budget sizing use the same serialization contract, and adds regression coverage that nested/plugin-owned `skillsSnapshot` keys survive save/load. After that, review and land it against the linked session-store bloat reports.
