## Summary

Keep open. Current main still presents SKILL.md as model-followed instructions through the skills catalog and read/follow prompt guidance, with load, eligibility, visibility, env, and sandbox controls but not the requested trusted/untrusted instruction boundary, role locking, per-skill memory/tool capability enforcement, or runtime monitoring. The recent issue discussion already kept this open, and the central prompt/skills files have not changed since that review.

## What This Changes

Add instruction isolation and capability enforcement for SKILL.md content so untrusted skills cannot override agent role, access memory/tools beyond declared permissions, or persist prompt-injection effects.

## Best Possible Solution

Define and document a trusted/untrusted skill precedence model, extend skill metadata and runtime policy with per-skill role, memory, and tool influence controls, enforce that boundary through prompt assembly plus runtime tool/memory policy, and add malicious SKILL.md regression coverage and docs.
