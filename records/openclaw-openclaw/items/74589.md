## Summary

Keep open. Current main still has an Anthropic-compatible payload path that can convert replay history to an empty `messages` array, and the linked closing PR #74731 is the focused implementation candidate but is not present on current main.

## What This Changes

The issue asks OpenClaw to prevent MiniMax M2.7 Anthropic-compatible requests from sending empty chat content after tool-call-heavy turns.

## Best Possible Solution

Review and land #74731 or an equivalent narrow final guard in the Anthropic-compatible transport payload builder, with regression coverage for empty or fully filtered converted messages. Keep the broader `/new` empty-prompt family tracked separately under #73926.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for a deterministic end-to-end MiniMax session, because the report is intermittent and lacks a minimal transcript. Yes for the core code path: current main can produce an empty converted Anthropic messages array before provider submission.

## Solution Assessment

Is this the best way to solve the issue? Yes, a final guard at the Anthropic-compatible transport boundary is the narrowest maintainable fix because it protects MiniMax and other Anthropic-compatible endpoints after all replay filtering has occurred.
