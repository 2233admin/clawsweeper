## Summary

Keep open. Current main still has plausible empty-payload paths for MiniMax/Anthropic-compatible runs: the embedded-runner prompt guard treats any non-empty replay array as content, runtime-only empty prompts bypass that guard, and the Anthropic transport filters blank/unsupported messages before sending the converted `messages` array without a final non-empty guard. The related #73926/#73929 work is useful, but it targets the `/new`/blank-replay class and is still open/unmerged, while this report describes post-tool-chain session pollution that should not be collapsed without a reproducing transcript or a merged central fix.

## What This Changes

The issue requests investigation and a defensive fix for intermittent MiniMax M2.7 400/2013 empty chat payloads after multi-tool-call agent turns.

## Best Possible Solution

Add a model-visible-content guard at the embedded-runner or Anthropic-compatible transport boundary, backed by a regression transcript that reproduces the post-tool-chain empty conversion. The fix should fail locally or skip/recover safely before provider submission, preserve valid tool-result continuations, and avoid fabricating user-visible content unless maintainers explicitly choose that behavior.
