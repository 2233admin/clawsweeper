## Summary

Keep open. Current main still has credible empty-payload paths for MiniMax/Anthropic-compatible submissions: the embedded-runner skip guard treats any replay entry as content, runtime-only prompts bypass that guard, and the Anthropic transport can filter all converted messages away before sending `messages` without a final non-empty check. The related #73926/#73929 work is relevant but is open/unmerged and scoped to `/new` blank-replay behavior, while this report describes post-tool-chain session corruption that still needs a reproducing transcript or a coordinated central guard.

## What This Changes

The issue reports intermittent MiniMax M2.7 400/2013 empty-chat-content failures after multi-tool-call agent turns, with a requested provider/core guard against empty message payloads.

## Best Possible Solution

Add a model-visible-content guard at the embedded-runner or Anthropic-compatible transport boundary, backed by a regression transcript for the post-tool-chain MiniMax failure. The fix should prevent `messages: []` from reaching providers, preserve valid tool-result continuations, and coordinate with #73929 rather than adding a separate placeholder policy unless maintainers explicitly choose that behavior.
