## Summary

Keep open. Current main still has the Anthropic-compatible transport path that can forward an empty converted `messages` array, and open closing PR #74731 is already the focused implementation candidate for this issue.

## What This Changes

The issue asks OpenClaw to prevent MiniMax M2.7 Anthropic-compatible requests from sending empty chat content after tool-call-heavy turns.

## Best Possible Solution

Review and land #74731, or an equivalent narrow final guard in `src/agents/anthropic-transport-stream.ts`, with regression coverage for empty or fully filtered converted Anthropic messages. Keep the broader `/new` empty-prompt family tracked separately under #73926.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for a deterministic end-to-end MiniMax session because the report is intermittent and lacks a minimal transcript; yes for the core code path, since current main can build an empty converted Anthropic `messages` array before provider submission.

## Solution Assessment

Is this the best way to solve the issue? Yes, the narrowest maintainable fix is a final non-empty guard at the Anthropic-compatible transport boundary after replay filtering has already run.
