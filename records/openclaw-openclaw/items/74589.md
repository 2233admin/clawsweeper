## Summary

Keep open. Current main still has the Anthropic-compatible transport path that can produce and forward an empty converted `messages` array, and this issue is paired with open closing PR #74731, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Prevent MiniMax M2.7 Anthropic-compatible tool-call chains from sending empty chat payloads and leaving sessions in a polluted state after provider rejection.

## Best Possible Solution

Review and land #74731, or an equivalent narrow fix, that enforces a non-empty Anthropic Messages payload after conversion and adds regression coverage for empty input and fully filtered replay history. Keep the broader `/new` blank-prompt work tracked separately under #73926.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for a deterministic end-to-end MiniMax session, because the report is intermittent and no minimal transcript is provided. Yes for a code-level path: empty or fully filtered replay history can still produce `messages: []` in the current Anthropic-compatible payload builder.

## Solution Assessment

Is this the best way to solve the issue? Yes, the best fix direction is a final guard in the Anthropic-compatible transport payload builder, because it is narrower than mutating session history and applies after all provider conversion/filtering has happened.
