## Summary

Keep this PR open. Current main still uses `schtasks` plus Startup-folder fallback for Windows gateway daemon management, so the proposed PowerShell ScheduledTasks work is not implemented or obsolete; however, the PR changes a Windows command-execution and auto-start boundary and still needs maintainer review for fallback policy, quoting, uninstall cleanup, and Windows validation.

## What This Changes

The PR changes `src/daemon/schtasks.ts` to use PowerShell ScheduledTasks APIs before `schtasks`, disable the scheduled-task execution time limit, add battery-power settings, and replace some Startup-folder fallback behavior with explicit administrator errors.

## Best Possible Solution

Keep this PR open for focused maintainer review of the Windows daemon design. The desired end state is a tested Windows implementation that uses the best available ScheduledTasks path, preserves a reliable fallback and uninstall cleanup story, updates public Windows daemon docs, and validates install/run/stop/restart/status behavior on real Windows or CI parity.
