## Summary

Keep this PR open. Current main still does not recognize the standalone `Transport endpoint is not connected` systemd/DBus error, so the PR’s intended daemon fallback and hint behavior is not implemented. The related maintainer PRs improved adjacent user-systemd failures but explicitly excluded this PR for central security handling, and the remaining change needs a refreshed review rather than cleanup closure.

## Best Possible Solution

Refresh and review this PR against current main by adding `transport endpoint is not connected` to the central `src/daemon/systemd-unavailable.ts` classifier, adding focused tests for availability, install activation hints, nonfatal install probe behavior, and machine-scope fallback, and explicitly resolving the security review either by tightening fallback classification to trusted error text or documenting why the existing local systemctl trust model is acceptable.
