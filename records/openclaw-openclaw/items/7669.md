## Summary

Keep this issue open. Current main still lacks a first-class npm `dev` dist-tag publish path: the public npm release workflow only exposes `beta` and `latest`, real publishes require release tags, the publish helper and release-plan contract only model stable/beta, tests do not cover a `dev` publish path, and the live npm registry currently has only `beta` and `latest` dist-tags for `openclaw`. The existing April 26 Codex review comment reached the same conclusion, and current main still matches that evidence.

## Best Possible Solution

Keep this issue open until maintainers either implement a first-class npm `dev` snapshot channel for moving `main` builds, including workflow trigger/input support, versioning rules, release-plan and publish-helper support, tests, post-publish verification, and docs; or explicitly retire npm `dev` snapshots and update release-channel, updater, installer, FAQ, CI, and release docs so `dev` is documented as git-only or otherwise unsupported for npm.
