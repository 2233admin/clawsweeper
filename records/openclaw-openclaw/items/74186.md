## Summary

Keep this PR open. Current main still bridge-resolves sandboxed media references before raw `media://` handling, so the sandbox-specific malformed `media:/inbound` path remains. The PR targets the right helper, but its media URI rewrite is sandbox/security-sensitive and should get maintainer review rather than automated closure.

## What This Changes

This PR adds early `media://` handling in `resolveSandboxedBridgeMediaPath` plus sandbox media path tests for fallback resolution, raw URI pass-through, non-media paths, and workspace-boundary enforcement.

## Best Possible Solution

Keep this PR open for maintainer/security review. The best fix is a narrow early branch in `resolveSandboxedBridgeMediaPath` that recognizes only validated `media://inbound/<id>` references, maps to `media/inbound/<id>` only when the staged sandbox file exists, preserves raw URI pass-through when appropriate, handles malformed URLs defensively, keeps workspace-boundary checks intact, and adds targeted helper plus sandboxed image/media tool coverage while coordinating with #74278.
