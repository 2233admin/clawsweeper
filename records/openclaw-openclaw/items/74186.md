## Summary

Keep this PR open. Current main still has the sandbox-specific `media://` bridge-resolution gap, so the branch targets a real bug, but the proposed fallback rewrite is not merge-ready because it maps unvalidated `media://` authorities and nested IDs to inbound files by basename before the existing media URI validator can reject them.

## What This Changes

The PR adds early `media://` handling to `resolveSandboxedBridgeMediaPath`, tries staged inbound fallback resolution through the sandbox bridge, enforces workspace-boundary checks for resolved host paths, and adds sandbox helper tests.

## Best Possible Solution

Land a narrow sandbox helper fix that validates `media://inbound/<id>` with the same authority and single-segment ID rules as `resolveInboundMediaReference`, uses the staged fallback only for that validated ID, returns the raw URI for downstream handling when fallback staging is unavailable, preserves workspace-boundary enforcement, and adds regression coverage for malformed, unsupported-host, and nested-ID inputs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate the media URI before rewriting to fallback:** `src/agents/sandbox-media-paths.ts:46-47`
  - body: The new branch rewrites any `media://` URL to `media/inbound/<basename>` when a matching staged file exists. That bypasses the existing media URI contract: `media://outbound/foo.png` and `media://inbound/a/b.png` should be rejected by `resolveInboundMediaReference`, but this code can silently load `media/inbound/foo.png` or `media/inbound/b.png` instead. Parse and validate the authority and decoded single-segment ID before probing the fallback path; otherwise return the raw URI for downstream validation.
  - confidence: 0.91
- **[P3] Handle malformed media URLs before the fallback probe:** `src/agents/sandbox-media-paths.ts:46`
  - body: `new URL(filePath)` is outside the `try` block, so malformed `media://` inputs such as an invalid host throw a raw `TypeError` from the sandbox helper instead of falling through to the media resolver's `LocalMediaAccessError` path. Guard the parse and return the raw URI, or convert the error through the existing media-reference validation path.
  - confidence: 0.78
