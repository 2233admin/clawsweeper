## Summary

Keep this PR open. It targets a real sandboxed `media://` resolution bug, but the current diff is not mergeable because the new fallback branch can rewrite unsupported or malformed media-store URIs into sandbox-validated staged files before the existing media URI validator sees them.

## What This Changes

The branch adds early `media://` handling to `resolveSandboxedBridgeMediaPath`, tries an inbound fallback path through the sandbox bridge, enforces host-path workspace checks on that fallback, and adds sandbox helper tests.

## Best Possible Solution

Land a narrow sandbox helper fix that detects `media://` before bridge resolution, validates only `media://inbound/<id>` with the same decoded single-segment ID rules as `resolveInboundMediaReference`, uses the staged fallback only for that validated ID, returns the raw URI when fallback staging is unavailable, preserves workspace-boundary enforcement, and adds regression coverage for unsupported host, nested ID, malformed URI, and happy-path fallback cases.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: `resolveSandboxedBridgeMediaPath` sends `media://inbound/...` into bridge path resolution, and the shared sandbox resolver uses `path.resolve`, which mangles that URI into `/workspace/media:/inbound/...`.

## Solution Assessment

No. Early detection is the right direction, but the proposed fallback rewrite is not the best fix until it preserves the existing media URI validation contract instead of collapsing arbitrary URI pathnames to a basename.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Validate media URIs before fallback rewrite:** `src/agents/sandbox-media-paths.ts:46-47`
  - body: The new branch probes `media/inbound/<basename>` for any `media://` URL. Inputs the existing resolver rejects, such as `media://outbound/foo.png` or `media://inbound/a/b.png`, can be accepted when a staged file with that basename exists because `loadWebMedia` then sees a sandbox-validated local path. Validate the host and decoded single-segment ID before probing; otherwise return the raw URI for downstream validation.
  - confidence: 0.91
- **[P3] Handle malformed media URLs before probing:** `src/agents/sandbox-media-paths.ts:46`
  - body: `new URL(filePath)` runs outside the fallback `try`, so a malformed `media://` string throws a raw `TypeError` from this helper instead of flowing through the existing `LocalMediaAccessError` media-reference path. Guard the parse and return the raw URI, or translate it through the same validation path.
  - confidence: 0.78
