## Summary

Keep this issue open. Current main still does not implement the central Discord mention-gating hardening described here: Discord preflight still evaluates configured mention patterns against the full resolved message text, routing does not inspect message text for competing configured agents, and the existing tests only cover generic explicit mention / ignoreOtherMentions behavior rather than the reported attention-window, quoted-later-mention, role-label, or other-agent delegation cases.

## Best Possible Solution

Implement a focused Discord plugin fix in `extensions/discord/src/monitor/message-handler.preflight.ts`: derive an opening attention window for mention matching, resolve configured Discord-bound agents that are explicitly or plainly addressed, safely map relevant role labels, drop competing-agent and configured-bot loop cases, and add colocated regression tests in `extensions/discord/src/monitor/message-handler.preflight.test.ts`. Update `docs/channels/discord.md` and public docs if the user-visible mention semantics change.
