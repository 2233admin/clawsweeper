## Summary

Keep open. Current main still lacks the central Discord preflight hardening requested here: no opening attention window for mention matching, no competing configured-agent detection, no role-label-to-agent handling, and no regression tests for quoted/later mentions or plain-text delegation.

## Best Possible Solution

Implement a focused Discord plugin fix in `extensions/discord/src/monitor/message-handler.preflight.ts`: derive a short opening attention text for mention matching, resolve configured Discord-bound agents that are explicitly or plainly addressed as competing targets, handle role attention only when it safely maps to the current agent, drop competing-agent bot-loop cases, and add colocated regression tests for every report case. Update `docs/channels/discord.md` only if the user-visible mention semantics change.

## Clownfish Work Prompt

Fix the Discord mention-gating/routing regression from openclaw/openclaw#44502 in the Discord plugin. In `extensions/discord/src/monitor/message-handler.preflight.ts`, scope mention-pattern matching to a short opening attention text instead of the full message body, so quoted/later examples do not satisfy requireMention. Detect configured Discord-bound agents that are explicitly mentioned or plainly delegated to as competing targets and drop for the current agent when the message is addressing another configured agent. Tighten role handling so unrelated role mentions do not count for the current agent, while safe role labels/metadata that map to the current agent remain valid. Drop messages from another configured agent bot in mention-only paths to avoid bot loops. Preserve existing behavior for explicit native bot mentions, reply-to-bot activation, bound-thread bypasses, `ignoreOtherMentions`, `allowBots="mentions"`, and control-command gating. Add focused tests in `extensions/discord/src/monitor/message-handler.preflight.test.ts` for: other configured agent explicitly mentioned in a home channel, plain-text delegation like "get Codey to help", plain agent names not counting as valid mentions in mention-gated channels, quoted/later mentions ignored, unrelated role mentions dropped, role labels that map to the current agent allowed, non-agent human mentions still handled under existing policy, and another configured agent bot dropped in mention-only channels. Update `docs/channels/discord.md` only if the user-visible mention semantics need documenting. Do not move Discord-owned behavior into core unless a small generic SDK seam is clearly required by multiple plugins.
