## Summary

Keep open. Current main still rejects workspace bootstrap symlinks whose canonical targets are sibling directories outside the workspace root, so the PR is not obsolete; however, the submitted patch relaxes a shared security boundary too broadly and needs maintainer/security review before it can merge.

## What This Changes

The PR changes `src/infra/boundary-path.ts` to allow boundary checks to accept paths under the user's `~/.openclaw` tree as a fallback when canonical targets fall outside the configured root.

## Best Possible Solution

Keep this PR and the linked issue open, but require a narrower design before merge: add an explicit workspace-bootstrap symlink policy or boundary mode, resolve the effective state root canonically, preserve direct-path and non-workspace boundary rejection, and add positive symlink plus negative credentials/session/sandbox regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict the fallback to symlink traversal:** `src/infra/boundary-path.ts:645-649`
  - body: `assertLexicalBoundaryOrCanonicalAlias` runs for any absolute path that is lexically outside the root, not only for followed symlinks. With this fallback, a direct path under `~/.openclaw/credentials` can pass the workspace boundary even though the PR intends to allow only symlinked bootstrap files.
  - confidence: 0.9
- **[P1] Scope the exception to OpenClaw workspace roots:** `src/infra/boundary-path.ts:316-317`
  - body: `applyResolvedSymlinkHop` is shared by every boundary root. As written, a sandbox or other non-workspace root can contain a symlink to `~/.openclaw/...` and pass confinement, weakening callers that expect the root boundary to remain local to that sandbox.
  - confidence: 0.86
- **[P2] Use the effective canonical state root:** `src/infra/boundary-path.ts:316-317`
  - body: The fallback compares canonicalized targets with `path.join(os.homedir(), ".openclaw")`, which ignores OpenClaw's configured home/state-dir handling and fails when the state root itself is a symlink. Resolve the effective state root through existing helpers and compare canonical paths.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch relaxes shared path-boundary enforcement and introduces concrete containment risks that should block merge until narrowed and tested.

Concerns:

- **[high] Direct state-tree access bypass:** `src/infra/boundary-path.ts:645`
  - body: The fallback in the outside-lexical and final containment checks can permit direct absolute paths under sensitive state directories such as credentials or sessions, even when no symlink traversal occurred.
  - confidence: 0.9
- **[high] Global boundary-root widening:** `src/infra/boundary-path.ts:316`
  - body: The symlink-hop fallback is not limited to workspace bootstrap roots, so sandbox/config/plugin callers can inherit a broader `~/.openclaw` trust boundary than their own root.
  - confidence: 0.86
- **[medium] Wrong root source for trust decision:** `src/infra/boundary-path.ts:316`
  - body: Using `os.homedir()` instead of the effective canonical OpenClaw state root can make the exception incorrect under configured or symlinked state locations.
  - confidence: 0.82
