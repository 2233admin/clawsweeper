## Summary

Keep this PR open. Current main still has the older memory-core fallback that relaxes to `hybrid.textWeight`, while this PR proposes deriving the fallback threshold from already-merged keyword-backed scores after temporal decay/MMR and adds regression coverage that is not present on main.

## Best Possible Solution

Keep this PR open. Port the intended fallback into `extensions/memory-core/src/memory/manager.ts`, derive retention from already-merged keyword-backed scores after temporal decay/MMR, add focused tests in the current memory-core test layout for BM25 `textScore < 1` and temporal decay, then resolve the remaining max-score recall concern before landing.
