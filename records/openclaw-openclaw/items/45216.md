## What This Changes

The PR adds top-level string `detail` recognition and message extraction in the shared assistant error formatter, plus exact-output regression tests for assistant-message and raw-error formatting paths.

## Best Possible Solution

Land a rebased or ported version of the focused string-`detail` parser support with the exact-output regression tests, and keep provider credential-gating or unconfigured-provider suppression tracked under the broader linked bug.

Yes. Source inspection and the linked bug provide a high-confidence path: on current main, `{ "detail": "..." }` is rejected by `parseApiErrorPayload`, cannot be extracted by `parseApiErrorInfo`, and therefore misses the user-facing formatter gates.

## Solution Assessment
