## Summary

Keep open. Current main still has the SDK compatibility gap described by the report: the focused channel reply SDK exports the reply-prefix helpers, but the deprecated root/compat SDK path used by migrating external plugins still does not expose them, so a legacy Feishu-style dispatch can resolve the helper to undefined.

## What This Changes

Restore compatibility for external plugins that call `createReplyPrefixContext` through the legacy `openclaw/plugin-sdk` or `openclaw/plugin-sdk/compat` surface after updating OpenClaw.

## Best Possible Solution

Restore the missing reply-prefix and reply-pipeline helpers on the deprecated compat/root SDK surface for the active migration window, keep the focused channel reply pipeline subpath as the canonical import, and lock both paths with contract tests so external plugins do not break before the announced removal date.

## Reproduction Assessment

Yes. The issue gives a deterministic Feishu dispatch path, and current main reproduces the underlying failure condition by source inspection: legacy root imports resolve through compat, while compat lacks the helper exported by the focused subpath.

## Solution Assessment

Yes. The narrow maintainable fix is to restore the missing public compatibility exports and tests; changing only Feishu code would leave other external plugins using the still-supported root/compat surface broken.
