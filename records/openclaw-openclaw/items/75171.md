## Summary

Keep open. Current main still has the same SDK compatibility gap reported by the issue: `createReplyPrefixContext` exists on the focused `channel-reply-pipeline` subpath used by bundled Feishu, but it is not available through the legacy root/compat SDK surface that the reported non-bundled Feishu plugin appears to load through during the migration window.

## What This Changes

The issue reports a Windows Feishu plugin regression in OpenClaw v2026.4.27 where incoming message dispatch fails because `createReplyPrefixContext` is missing from the `openclaw/plugin-sdk` compatibility surface.

## Best Possible Solution

Restore the missing channel reply helper exports on the legacy root/compat SDK surface for the documented migration window, keep the focused `channel-reply-pipeline` subpath as canonical, and add contract tests so external plugins fail with migration guidance rather than a runtime `TypeError`.

## Reproduction Assessment

Yes for the SDK failure. The issue provides deterministic Feishu dispatch steps, and current main can reproduce the underlying failure condition by reading the root SDK alias where `createReplyPrefixContext` is `undefined`.

## Solution Assessment

Yes, a narrow SDK compatibility repair is the best direction. Fixing only bundled Feishu would not help external plugins still using the supported deprecated root/compat import path during the migration window.
