## Summary

Keep open. The current main branch still has the SDK compatibility gap described in the report: the focused channel reply SDK exports `createReplyPrefixContext`, but the deprecated compat/root SDK path used by legacy external plugins still omits the reply-prefix and reply-pipeline helpers, so the reported Feishu dispatch failure remains plausible on the active migration surface.

## What This Changes

The issue reports a Windows Feishu/Lark plugin dispatch regression where a legacy `openclaw/plugin-sdk` import resolves `createReplyPrefixContext` to undefined after updating OpenClaw.

## Best Possible Solution

Restore the missing reply-prefix and reply-pipeline helpers on the deprecated compat/root SDK surface for the active migration window, keep `openclaw/plugin-sdk/channel-reply-pipeline` as the canonical import path, and add contract tests that lock both the legacy and focused surfaces.

## Reproduction Assessment

Yes. The issue gives a deterministic Feishu dispatch path, and current main reproduces the underlying failure condition by source inspection: legacy root imports resolve through the compat-backed root alias, while compat lacks the helper exported by the focused reply-pipeline subpath.

## Solution Assessment

Yes. The narrow maintainable fix is to restore the missing public compatibility exports and tests; changing only Feishu code would leave other external plugins using the still-supported legacy SDK surface broken.
