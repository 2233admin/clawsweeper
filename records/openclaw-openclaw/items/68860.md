## Summary

Keep open. Current main still drops upstream embedded Pi turn-boundary events, so the PR addresses a real observability gap; however, the latest PR head is not mergeable because the new turn handler contains malformed TypeScript.

## What This Changes

The PR adds embedded Pi `turn_start`/`turn_end` handling, forwards them as `stream: "turn"` events through the global and per-run agent event paths, and adds subscription regression tests.

## Best Possible Solution

Land a small current-main-compatible patch that forwards embedded Pi `turn_start` and `turn_end` through the same scheduled handler path as the other embedded events, emits stable `turn` payloads to both observer surfaces, and covers both delivery paths with focused tests.
