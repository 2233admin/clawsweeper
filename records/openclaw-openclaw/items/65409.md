Keep this PR open. Current main still builds the codex-cli system prompt report with an empty tool list, while the linked open issue describes the same missing bundle MCP report metadata and this PR is the active implementation candidate. The PR branch adds report MCP config derivation, materializes bundle MCP tools during CLI run preparation, feeds those tools into `buildSystemPromptReport`, and adds CLI bundle MCP regression coverage for reported tool entries and schema character counts.

## Best Possible Solution
