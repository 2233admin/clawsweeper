## Summary

Keep open. Current main still has DM-only pairing and group access remains explicit open/disabled/allowlist behavior, with no pending group approval state, owner notification on group add, group approve/reject command shape, or pending read-only group routing. The related open items cover adjacent group suppression or WhatsApp group-leave/webhook ideas rather than a canonical cross-channel group-pairing contract.

## What This Changes

The issue requests group pairing with owner notification, approve/reject controls, and optional read-only pending-group handling for Telegram/WhatsApp-style groups.

## Best Possible Solution

Define an additive cross-channel group-pairing contract: extend config/schema/docs, persist pending approvals by channel/account/group, deliver owner-visible group-add notifications, add authorized approve/reject operations, and wire Telegram plus WhatsApp through shared policy state. Treat reader-agent observation as a separately scoped mode unless maintainers explicitly include it in the approval contract.
