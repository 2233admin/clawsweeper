## Summary

Current main still does not implement the requested group pairing flow. DM pairing exists, but group policy remains open/disabled/allowlist with group authorization kept separate from DM pairing; there is no group pending approval state, owner notification for new group additions, group approve/reject CLI surface, or pending read-only reader-agent routing. The issue should stay open for maintainer/product design, coordinated with the adjacent open group-join and silent-observation work.

## Best Possible Solution

Keep this open for maintainer/product design. The best path is an additive group-pairing contract: add groupPolicy="pairing" to config/schema/docs, persist pending group approvals keyed by channel/account/group, expose approve/reject CLI operations, emit owner-visible group-add or first-seen-unconfigured-group notifications, and make Telegram/WhatsApp honor that state. The pending read-only reader-agent mode should be explicitly scoped into that design or split to a coordinated follow-up with #42400.
