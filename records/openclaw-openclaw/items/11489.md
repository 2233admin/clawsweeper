## Summary

Keep open. Current main still implements DM pairing only; group access is open, disabled, or allowlist-based, with no pending group approval state, owner notification on group add, group approve/reject command, or pending read-only group routing. The related open work covers adjacent silent observation or WhatsApp group hygiene, not the requested cross-channel group-pairing flow.

## What This Changes

Request a cross-channel group pairing flow that notifies the owner when a bot joins a new group, keeps unapproved groups from normal processing, supports owner approve/reject actions, and optionally routes pending groups through a read-only agent.

## Best Possible Solution

Define an additive cross-channel group-pairing contract: extend config/schema/docs, persist pending group approvals keyed by channel/account/group, send owner-visible group-add notifications, add authorized CLI or inline approve/reject operations, and wire Telegram/WhatsApp through shared policy state. Treat reader-agent or silent observation as a separately scoped mode unless maintainers explicitly choose to combine it with approval.
