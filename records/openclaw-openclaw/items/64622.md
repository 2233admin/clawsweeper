## What This Changes

## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Apply the timeout floor only to cross-agent sends:** `src/agents/tools/sessions-send-tool.ts:66-68`
  - body: `resolveAgentIdFromSessionKey()` always returns a non-empty id, falling back to `main`, so this condition floors every non-zero `sessions_send` when both session keys are present. Same-agent or local sends that ask for a short timeout would now wait up to 45 seconds instead of failing fast.
  - confidence: 0.94
- **[P1] Preserve the resolver-selected outbound session key:** `src/infra/outbound/message-action-threading.ts:89-96`
  - body: This rewrites every resolved outbound route to `currentSessionKey` whenever one exists. For cross-target sends from a main/webchat run to a QQ, Discord, or other external room, `ensureOutboundSessionEntry` will persist the external route metadata onto the current session instead of the route-specific session key.
  - confidence: 0.92
- **[P2] Keep source attribution in inter-session provenance:** `src/agents/tools/sessions-send-tool.ts:311-313`
  - body: The PR removes `sourceSessionKey` and `sourceChannel` from `sessions_send` provenance even though current main's inter-session prompt marker and transcript-hygiene contract use those fields when available. That weakens the receiver's ability to distinguish which session/channel produced routed content.
  - confidence: 0.87
