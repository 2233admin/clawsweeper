## Summary

Current main still does not implement the requested opt-in Discord visible reply-author mention injection. Discord native reply references are present via `replyToMode`, and existing mention rewriting only rewrites model-authored `@name` text when an `@` is already present; there is no `channels.discord.autoMentionReply` config/schema/docs field or send-path logic that prepends `<@originalAuthorId>` to the first outbound guild reply.

## Best Possible Solution

Keep this issue open and implement a focused Discord plugin feature: add a typed/schema/docs opt-in setting such as `channels.discord.autoMentionReply`, resolve the referenced guild message author safely when a native reply reference is present, prepend `<@authorId>` only to the first visible outbound guild reply when absent, and cover normal text, chunking, media/component/webhook/voice-adjacent paths, streaming preview/final behavior, DMs, deleted references, existing mentions, and bot/self-message cases.
