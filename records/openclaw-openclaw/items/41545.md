## Summary

Keep open. Current main still clears the Overview Gateway Token from the WebSocket URL input handler, current browser coverage still asserts that clearing behavior, and open PR #73923 references this issue with closing syntax as the likely canonical fix path.

## What This Changes

Editing the Overview Gateway Access WebSocket URL should preserve the existing Gateway Token unless the user explicitly edits it or the normalized gateway endpoint changes.

## Best Possible Solution

Keep this issue open and review or land one canonical fix PR, preferably #73923 if it passes maintainer review. The implementation should preserve the in-memory Gateway Token during same-normalized-endpoint WebSocket URL edits, clear or reload the token when the normalized gateway endpoint changes, and update browser/node regression coverage so the token-scoping boundary is explicit.
