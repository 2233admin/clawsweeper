## Summary

Keep this PR open. Current main still routes FreeBSD write/copy operations through the Python pinned-write helper, and `sameFileIdentity` remains strict for non-Windows `dev` mismatches. The PR's FreeBSD fallback is not implemented on main and still addresses the reported FreeBSD/ZFS failure, with a maintainer tradeoff to review around legacy write semantics.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to land an amended narrow FreeBSD fallback with explicit tests and an accepted legacy-write tradeoff, or supersede it with a pinned-write identity normalization that fixes FreeBSD while preserving the pinned helper's all-or-nothing target creation behavior.
