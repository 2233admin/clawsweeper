## Summary

Keep open. Current main still lacks the requested Signal audio-preflight hook: Signal forwards fetched audio metadata into the shared media-understanding path, but group mention gating happens before attachment download and there is no Signal call to `transcribeFirstAudio`, no `Transcript`, and no `MediaTranscribedIndexes` on the Signal inbound context.

## Best Possible Solution

Keep this open and implement a provider-backed Signal voice-note preflight. The fix should fetch eligible audio attachments after Signal access checks but before requireMention skips, call the shared `transcribeFirstAudio` helper, use the transcript for mention/body/command handling, set `Transcript` and `MediaTranscribedIndexes`, preserve media metadata for downstream inbound-audio behavior, and document Signal voice-note transcription. Do not add a shell `preflight_cmd` without a separate security-reviewed design.
