## Summary

Keep open. Current main does not implement this PR's intended fix: Active Memory still starts its outer abort watchdog at hook entry using the configured `timeoutMs`, and it still registers `before_prompt_build` without a hook-specific timeout. Current main also has a newer core hook-runner default that times out `before_prompt_build` after 15 seconds, with no per-registration override, so the linked Active Memory setup-time bug remains plausibly current. This PR is an active implementation candidate for open issue #72606, not cleanup-closeable.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review/land or revise the PR so Active Memory preserves configured `timeoutMs` for the embedded model run, uses a bounded outer setup/hook budget, updates the public plugin hook contract documentation if `{ timeoutMs }` is intended for plugin authors, and resolves the abort-message wording before merge.
