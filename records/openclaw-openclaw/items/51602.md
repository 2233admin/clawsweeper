## Summary

Keep open. The report is recent, specific, and not covered by a protected-label or maintainer-authored exception. Current main still routes TUI terminal input and rendered chat bodies through pi-tui, with only narrow OpenClaw wrapper behavior around shortcuts, backspace dedupe, OSC 8 links, and generic render sanitization. Main now pins pi-tui 0.70.5 and upstream pi-tui documents IME cursor support, but I did not find OpenClaw-local code, tests, changelog notes, release proof, or live macOS Korean/English IME evidence showing the reported stray glyph/separator artifacts are fixed. Related #39933 and #40063 do not supersede this exact chat-output artifact symptom.

## Best Possible Solution

Keep this open as a TUI/macOS IME rendering regression. Reproduce on current main or latest release with Korean/English input switching, capture screenshots plus raw TUI write/input evidence where possible, then fix the OpenClaw wrapper/render sanitization path or route a focused upstream pi-tui fix. Close only after targeted regression coverage, live reproduction proof, or a documented upstream fix/pin demonstrates that stray glyph and separator artifacts no longer appear in chat output.
