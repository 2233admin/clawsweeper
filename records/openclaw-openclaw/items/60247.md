## Summary

Keep this open. Current main still parses each Control UI WebSocket message as a complete JSON frame and drops malformed frames on parse failure; I did not find a source-backed fix or targeted regression proving long CJK streaming content survives the reported boundary/malformed-frame failure. Recent main has adjacent CJK stream-merge coverage, but that does not address the Control UI frame parsing path described here.

## Best Possible Solution

Keep this open and add focused coverage around the Control UI Gateway path: send long CJK assistant deltas through src/gateway/server-chat.ts, src/gateway/server-broadcast.ts, ui/src/ui/gateway.ts, and ui/src/ui/controllers/chat.ts, and assert the UI receives complete text without JSON.parse errors, stream loss, or truncation. If malformed partial-frame input is possible, fix it at ui/src/ui/gateway.ts with explicit buffering/reassembly or reconnect/history-recovery diagnostics; if not, reclassify the root cause to the provider/runtime parser with evidence.
