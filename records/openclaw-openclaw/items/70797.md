## Summary

Current main still appears vulnerable. The final visible-text sanitizer only strips bracketed downgraded tool-call markers, not a bare OpenAI-style `{"tool_calls":[...]}` payload, and the assistant preview path can still pass cleaned partial text to channel preview delivery without a streaming-aware raw tool-call JSON guard. The two related fix attempts were closed unmerged, so the issue should stay open for a focused fix PR.

## Best Possible Solution

Keep this open and land a narrow core fix that sanitizes or withholds bare OpenAI-style `tool_calls` JSON before assistant-visible final text and partial preview delivery reach channels. The fix should preserve ordinary JSON and fenced/inline code examples, handle incomplete streamed fragments safely, and add focused regression tests for the shared sanitizer plus the Telegram/direct preview path.
