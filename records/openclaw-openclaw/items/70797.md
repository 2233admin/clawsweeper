## Summary

Keep open. Current main still allows the reported bare OpenAI-style `tool_calls` JSON leak class through the assistant-visible sanitizer and Telegram/direct partial-preview path, while the two related fix attempts are closed unmerged.

## What This Changes

Prevent raw internal `tool_calls` JSON payloads from appearing in user-visible replies during Telegram direct background `exec` and follow-up `process poll` flows.

## Best Possible Solution

Add a shared, streaming-aware guard for bare OpenAI-style `tool_calls` JSON at the assistant-visible boundary, reuse it for final text and partial preview delivery, and cover ordinary JSON plus fenced/inline examples in regression tests.

## Reproduction Assessment

Yes for the leak class: current source shows a bare `tool_calls` JSON object passes through the sanitizer unchanged and can be sent through `onPartialReply`. The exact live Telegram/tinyrouter race remains sketched, not fully minimized.

## Solution Assessment

Yes: the narrow maintainable fix is to sanitize or withhold internal tool-call payload text before user-visible delivery, without adding provider-specific execution recovery or changing tool orchestration semantics.
