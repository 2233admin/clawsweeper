## Summary

Keep open. Current main still supports node-advertised capabilities and commands, plus plugin-registered node-host commands, but it does not let connected nodes advertise first-class LLM tool definitions or synthesize node-backed agent tools. The recent maintainer discussion also explicitly kept this open, and the linked openclaw-tescmd workaround remains useful context rather than a reason to close.

## Best Possible Solution

Keep this open for maintainer-approved Gateway protocol and runtime work: add an additive node tool-advertisement schema, validate and store advertised tools on connected NodeRegistry sessions, synthesize node-backed agent tools and catalog/effective inventory entries with node provenance, route execution through existing node.invoke authorization and node command policy, and document naming, reconnect, pairing, and tools.allow/tools.deny behavior.
