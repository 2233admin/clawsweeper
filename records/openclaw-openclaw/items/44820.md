## Summary

This issue should stay open. Current main has partial session artifact cleanup, but it still lacks the proactive startup/timer sweep for `.tmp` and transcript `.bak.*` artifacts described by the report, and open PR #47635 is still an unmerged implementation candidate linked from the issue timeline.

## Best Possible Solution

Keep this issue open and review PR #47635, or land an equivalent maintainer patch, after rebasing it onto current main. The fix should add a focused startup and periodic session artifact sweep that reuses existing retention settings for `.deleted.*` and `.reset.*`, safely removes old UUID-suffixed session `.tmp` files, trims transcript `.bak.*` archives, preserves active files, and adds regression tests for false positives and active-write safety.
