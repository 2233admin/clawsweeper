## Summary

This issue should stay open. Current main has useful reactive session maintenance and manual cleanup, but it still does not implement the reported automatic cleanup for all session artifact classes, especially orphaned `writeTextAtomic` `.tmp` files and bounded `.bak.*` transcript backups. The provided GitHub context also shows an open PR with closing syntax for this issue, so policy says the issue stays open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #47635 only if it is updated to current main and covers all relevant session-store roots, or land an equivalent maintainer patch. The best fix is a gateway/session maintenance sweep at startup and on a timer that uses existing session-store target discovery, respects retention for `.deleted.*` and `.reset.*`, safely removes old orphaned `writeTextAtomic` `.tmp` files, trims `.bak.*` transcript backups per base file, and includes focused tests for active-file preservation, false-positive archive names, custom `session.store` paths, and timer/startup wiring.
