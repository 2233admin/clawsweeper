## Summary

Keep open. Current main still has the Anthropic 1M beta-header behavior this PR is trying to remove, and the PR is paired with same-author open #45550 via closing syntax. Another open candidate, #45613, appears closer to the current plugin-owned path but is not merged or maintainer-selected as canonical. This PR should be kept for maintainer consolidation rather than closed as obsolete.

## Best Possible Solution

Keep this PR open as an implementation candidate for #45550, or consolidate it with #45613 after maintainer review. The useful fix should be rebased onto `extensions/anthropic/stream-wrappers.ts`, update `extensions/anthropic/stream-wrappers.test.ts`, `src/agents/pi-embedded-runner-extraparams.test.ts` if that integration coverage remains applicable, and `docs/providers/anthropic.md` plus `docs/reference/token-use.md`. It should preserve `context1m` for context-window sizing in `src/agents/context.ts` and explicitly decide how to strip or tolerate legacy manually configured `context-1m-2025-08-07` beta headers for OAuth auth.
