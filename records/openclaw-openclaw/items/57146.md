## Summary

Keep this PR open. The linked bug is still present on current main: the bundled `gh-issues` skill still contains legacy `/data/.clawdbot` paths, while current docs and config helpers point standard installs at `~/.openclaw`. The PR is a plausible implementation candidate, but it needs a rebase/update because GitHub reports it dirty against current main and the existing review correctly calls out a duplicate no-op token fallback introduced by the patch.

## Best Possible Solution

Refresh this PR or land an equivalent focused patch that replaces the remaining `/data/.clawdbot` references in `skills/gh-issues/SKILL.md` with the current config/state-dir pattern, uses `$HOME/.openclaw` for claims and cursor files, removes the duplicate fallback block flagged by review, and then closes the linked bug after merge.
