## Summary

Keep #57447 open. Current main still blocks cross-agent `sessions_send` under `tools.sessions.visibility=tree` before the A2A allow policy is evaluated, and the sandboxed send resolution path still applies spawned-session visibility restrictions before the send occurs. The linked fix PR #57483 is still open/unmerged in the provided GitHub context, and the later related PR #71959 is closed/unmerged, so there is no shipped or canonical completed replacement to close against.

## Best Possible Solution

Keep the issue open and fix it in core: make `sessions_send` authorization A2A-first for cross-agent targets while preserving visibility-first behavior for list/history/status, then adjust the sandboxed send resolution path so A2A-allowed sends can resolve the intended target without granting read/list visibility. Add regression tests for `tree`/sandboxed cross-agent send allowed by A2A, A2A-denied send blocked, and read/status/list still blocked under non-`all` visibility.
