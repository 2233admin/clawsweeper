## Summary

Keep open. Current main still returns raw parsed config for `/config show`, while the gateway config API already uses schema-aware redaction; open closing PR #65637 has not merged, so this needs maintainer/security review rather than cleanup closure.

## What This Changes

The issue requests that `/config show` chat replies redact sensitive config values before rendering full or path-scoped config JSON.

## Best Possible Solution

Ship a maintainer-approved patch that redacts the cloned parsed config before any `/config show` output, covers both full and path-scoped show replies with regression tests, and close this only after the fix merges and release provenance is known.

## Reproduction Assessment

Yes. The source path is direct: a valid config snapshot is cloned from `snapshot.parsed`, then full or path-scoped values are stringified into the chat reply without redaction; no live run was needed for high-confidence reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to apply the existing schema-aware config redaction to the cloned parsed config before path lookup and full JSON rendering, matching the `config.get` API instead of adding a separate chat-only masking path.
