## Summary

Current main has durable background task records, task-state notifications, ACP no-output notices, and disabled-by-default tool-loop detection, but it does not implement the issue's requested artifact-gated status policy, no-artifact/no-phase-change escalation timers, no-idle-after-green rule, guard state fields, `ENFORCEMENT_GUARDS_V1` flag, or structured guard telemetry. This is a broad runtime/product reliability change, so it should stay open for maintainer design rather than be auto-closed or queued as a narrow fix PR.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is an explicit execution-guard design that defines what counts as an artifact, where status updates are intercepted, how guard state persists in the task/session store, what telemetry schema is acceptable, and which runtime surfaces are in scope before a focused implementation is attempted.
