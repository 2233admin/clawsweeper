## Summary

Keep open. The requested filename-preservation work is still not present in current main for the normal send path, but this PR is not safe to merge as-is because it mixes the useful Feishu/outbound plumbing with confirmed Feishu media regressions, CODEOWNERS removal, credential-like local docs, and CI/cache changes outside the stated scope.

## What This Changes

The PR threads optional attachment filename values through outbound send contracts into Feishu media upload while also adding Feishu local-file handling, local docs/scripts/skills, OpenAI WebSocket header support, workflow/CODEOWNERS/cache changes, and a uv lockfile.

## Best Possible Solution

Carry only the filename plumbing needed for the message tool/outbound payload contracts and Feishu media send path, preserve loadWebMedia, mediaLocalRoots, and sanitizeFileNameForUpload, drop unrelated repo metadata/local tooling/provider changes, and add focused regression tests for explicit filename forwarding, basename fallback, remote non-image URLs, local-root rejection, and upload filename hardening.
