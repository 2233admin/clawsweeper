## Summary

Keep open. Current main has adjacent status, usage, sessions, model-auth, and high-context warning surfaces, but it still does not provide the requested persistent in-interface status area showing the active agent/model plus token, quota, and subscription context during operations. The related open requests #46398 and #71630 cover narrower slices and do not supersede this broader product/UX request.

## What This Changes

Add a persistent in-interface status area/status bar showing the active agent/model plus token, quota, and subscription context during operations.

## Best Possible Solution

Keep this open for maintainer product/UX review. The best path is to decide whether this remains the umbrella issue or is split across narrower follow-ups, then implement a persistent Control UI status surface that reuses sessions.changed/session rows, models.authStatus, and usage APIs with explicit privacy, freshness, and provider-unavailable states.
