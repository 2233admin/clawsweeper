## Summary

Keep open. The linked OpenRouter cron model-ID corruption is still not fixed on current main, but this PR’s blanket whitespace removal would regress valid model and fallback aliases that intentionally contain spaces, including persisted cron jobs normalized during load.

## What This Changes

The PR changes cron normalization so payload and top-level `model` and `fallbacks` values have all whitespace removed, and adds a unit test for whitespace-corrupted provider/model IDs.

## Best Possible Solution

Revise or supersede this PR with an alias-preserving repair: keep values exact after trim, resolve aliases first, and only repair accidental whitespace for clear provider/model references when the raw value fails resolution. Cover payload and top-level create/patch paths for both primary model and fallbacks, plus persisted-load behavior.

## Review Findings

Full review comments:

- **[P1] Preserve spaced model aliases before repair:** `src/cron/normalize.ts:181`
  - body: `payload.model` can be an alias, and alias lookup lowercases while preserving internal spaces. Rewriting `DeepSeek R1` or `My Fast Model` to `DeepSeekR1` makes valid cron jobs fail, and persisted jobs hit this path during store hydration.
  - confidence: 0.94
