## Summary

Keep open. The cron model-ID corruption report is valid and current main still lacks an alias-preserving repair, but the latest PR branch is not mergeable because corrupted slash refs can still bypass repair in allow-any configs and payload fallback candidates still use the old resolver path.

## What This Changes

The PR branch moves cron model-ID whitespace repair into shared model resolution, keeps cron normalization trim-only, and adds focused model-selection and cron tests for whitespace-corrupted refs and spaced aliases.

## Best Possible Solution

Use one alias-safe slash-ref repair path after exact alias lookup and before accepting corrupted provider/model refs, then share that path across explicit cron primary model resolution and fallback candidate resolution. Keep cron normalization trim-only, preserve spaced aliases, and cover allowlist, allow-any, multi-segment OpenRouter/custom refs, nested payloads, legacy top-level fields, and persisted hydration behavior.
