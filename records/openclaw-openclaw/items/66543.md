## Summary

Keep open. The linked cron model-ID bug is valid and the PR is moving toward the right alias-preserving shape, but the current patch still leaves corrupted primary refs untouched in allow-any model configs and leaves fallback candidates outside the new repair path.

## What This Changes

The PR adds a resolution-time compacted slash-ref retry in shared model selection, keeps cron normalization trim-only, and adds focused model-selection and cron tests for whitespace-corrupted model IDs.

## Best Possible Solution

Use one alias-safe repair path: exact alias lookup first, then compact accidental whitespace only for clear slash-form provider/model refs before accepting uncataloged allow-any refs, and share that repair for cron primary model overrides and fallback candidate resolution while keeping cron normalization trim-only.
