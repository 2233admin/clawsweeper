## Summary

Keep this PR open. Current main has a broad Dreaming archive-artifact skip that helps with fresh `.jsonl.deleted.<ts>` corpus leakage, but it does not implement the PR's session-id-aware transcript classification helpers, reverse session-key lookup, SDK exports, or operator-facing `dreaming.sessionFilter` controls. The PR is also paired with the same-author open issue #72611 and still needs maintainer review for the security concern raised against the PR head.

## Best Possible Solution

Keep this PR open for maintainer review. A revised version should either land the session-id-aware classification and `dreaming.sessionFilter` controls after resolving the source-label sanitization concern and generated docs/API expectations, or maintainers should explicitly split the remaining #72611 operator-filter work into a narrower follow-up if the existing archive skip is the desired fix for rotated artifacts.
