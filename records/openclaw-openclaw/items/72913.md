## Summary

Keep this PR open. The branch targets a real remaining operator-filter gap for Dreaming session ingestion, but it is not merge-ready against current main because current main has moved the memory host SDK implementation into `packages/memory-host-sdk`, while the PR still patches the old `src/memory-host-sdk` implementation shape and does not update the public Dreaming docs for the new config surface.

## What This Changes

The PR adds session-id-aware Dreaming transcript classification helpers, new `dreaming.sessionFilter` exclusion settings, focused tests, a changelog entry, and internal-system provenance filtering for heartbeat/cron transcripts.

## Best Possible Solution

Rebase the useful pieces onto current main, keep the SDK implementation in `packages/memory-host-sdk`, preserve the thin `src/memory-host-sdk` wrappers, document `dreaming.sessionFilter` in the public Dreaming and memory config references, and decide whether the session-id fallback remains necessary now that archive artifacts are skipped broadly.

## Reproduction Assessment

Partly yes. The missing operator-facing exclusion path is reproducible by source inspection on current main: the schema/docs and ingestion path have no `dreaming.sessionFilter`; the older rotated `.deleted` corpus leak is less clear on current main because archive artifacts are now skipped before ingestion.

## Solution Assessment

No, not as submitted against current main. The direction is plausible, but the branch needs a rebase onto the packaged SDK layout and docs/API alignment; a safer maintainer path may be to split the operator filters from the now-overlapping rotated archive mitigation.
