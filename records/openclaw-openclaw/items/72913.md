## Summary

Keep this PR open. Current `main` has a broad Dreaming archive skip that may reduce the reported rotated-transcript leak, but it does not implement the PR's session-id-aware transcript classification helpers, reverse session-key lookup, SDK exports, or operator-facing `dreaming.sessionFilter` controls. The PR is also paired with the still-open contributor issue it says it closes, and the proposed diff has a remaining security-review concern around transcript basename sanitization.

## Best Possible Solution

Keep the PR open for maintainer review. Either land a revised version that resolves the security finding and aligns schema/docs/API coverage, or explicitly split the remaining #72611 work into a narrower follow-up if maintainers decide the current broad archive skip is the desired fix for rotated artifacts.
