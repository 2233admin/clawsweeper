## Summary

Keep open. Current main has improved the request-scoped and unavailable-model fallback paths, but REM report writing and Dream Diary writing are still separate paths: a REM report can be written before any diary attempt, diary generation remains conditional and best-effort, and the Control UI still only displays DREAMS.md/dreams.md content.

## What This Changes

The issue requests that a successful memory-core REM dreaming phase create or update DREAMS.md so the Control UI Dreams diary is not empty.

## Best Possible Solution

Maintainers should define the Dream Diary contract explicitly. If REM completion with usable material must be visible in the Control UI, memory-core should append a deterministic fallback diary entry when the scheduled REM path writes a report but the narrative path does not produce one, with regression coverage for separate storage. If the diary is intentionally best-effort, the docs and UI empty state should say that a phase report can exist without DREAMS.md.
