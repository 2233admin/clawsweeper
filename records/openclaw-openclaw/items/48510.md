## Summary

Keep open. Current main still lacks optional behavioral capability descriptors for tools on the core tool type, plugin tool metadata, before_tool_call/trusted-policy payloads, tools.catalog/tools.effective schemas, and effective inventory projection. The related #48503 proposal is complementary design work, not a merged superseding implementation.

## What This Changes

Add optional behavioral capability metadata to built-in and plugin-registered tool definitions and surface it to policy hooks, catalogs, docs, and audit-related tooling.

## Best Possible Solution

Design an additive, versioned Plugin SDK and Gateway protocol contract for behavioral tool capability descriptors, allow built-in and plugin tools to declare it optionally, project it through tools.catalog and tools.effective, surface it to trusted policies and before_tool_call payloads, and update API baselines, contract tests, docs, and changelog together.
