## Summary

Keep open. Current main still lacks stable behavioral capability descriptors for tools, and the related #48503 hook-enrichment proposal remains open rather than merged or superseding this request.

## What This Changes

The issue requests optional behavioral capability descriptors on built-in and plugin-registered tool definitions for policy, onboarding, documentation, and audit consumers.

## Best Possible Solution

Design an additive, versioned Plugin SDK and Gateway protocol contract for behavioral tool capability descriptors, let built-in and plugin tools declare it optionally, project it through tools.catalog and tools.effective, and surface it to trusted policies plus before_tool_call payloads with docs, API baseline updates, and focused contract tests.
