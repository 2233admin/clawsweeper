## Summary

Keep open. Current main has adjacent tool metadata (`displayName`, `description`, `risk`, `tags`) and `before_tool_call` enforcement, but it does not provide the requested stable capability descriptors on tool definitions, does not project such descriptors through the tool catalog/effective inventory, and does not surface them to `before_tool_call` hooks. The linked #48503 remains open and complementary rather than a merged superseding fix.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is an additive plugin SDK/tool contract that defines a small capability descriptor type, lets core and plugin tools declare it, projects it through `tools.catalog`/`tools.effective`, and makes it available to `before_tool_call`/trusted tool policy evaluation, coordinated with #48503 rather than closing either item early.
