## Summary

Keep open. Current main still gates browser device identity on Secure Context/WebCrypto and documents remote plain HTTP as blocked by default; the bypass-scope and implied operator-scope parts are already implemented, so the remaining change is a product/security decision rather than cleanup closure.

## What This Changes

The issue requests Control UI browser device identity generation over non-secure LAN or HTTP reverse-proxy contexts by removing the Secure Context/WebCrypto dependency, plus related bypass and scope-normalization fixes.

## Best Possible Solution

Maintainers should make an explicit product/security call on non-secure LAN HTTP. If supported, add a browser-safe hashing path with explicit UI dependency metadata, narrow or remove the Secure Context identity-generation guard, update focused UI/gateway tests, and revise public Control UI docs. If not supported, keep the shipped bypass and scope-normalization fixes and document HTTPS, Tailscale Serve, trusted-proxy, and break-glass flows as the intended paths while using #32473 for the broader UX follow-up.
