## Summary

Keep open. Current main still lacks a supported visible coordinator-to-ACP-thread delivery path that bypasses main Discord channel-session metadata creation; the existing A2A work only makes follow-up/announce output visible after an internal `sessions_send`, not the initiating visible instruction path described here.

## What This Changes

Request a supported way for a coordinator agent to post a visible instruction into an ACP-bound Discord thread without creating `agent:main:discord:channel:<thread_id>` session metadata.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API and security review. It should post the coordinator-authored instruction through the selected Discord account into the ACP-bound thread, bypass outbound main-session entry creation only for that authorized call, preserve normal metadata writes elsewhere, and document the supported ACP coordination workflow.
