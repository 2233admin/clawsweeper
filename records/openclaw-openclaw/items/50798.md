## Summary

Keep open. Current main still lacks a supported coordinator-originated visible send into an ACP-bound Discord thread that bypasses derived main-agent Discord channel session metadata creation. The shipped A2A/ACP work covers internal injection, follow-up routing, visible announce/output delivery, and bound-thread command routing, but not the initiating visible proxy message requested here.

## What This Changes

Request a supported visible coordinator-to-ACP-thread delivery path that posts into a bound Discord worker thread without creating an `agent:main:discord:channel:<thread_id>` session.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API/security review. It should post the coordinator-authored instruction through an authorized Discord account into the ACP-bound thread, bypass only the derived main outbound session entry for that explicit call, preserve normal metadata writes elsewhere, and document the supported ACP coordination workflow.
