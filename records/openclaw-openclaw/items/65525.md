## Summary

Keep open. Current main still launches MCP stdio children with only the MCP SDK default environment plus explicit server env, so ambient gateway proxy variables and NODE_OPTIONS are not propagated. The issue is also paired with open PR #65574 using closing syntax, so it should remain open until that PR or an equivalent current-main fix lands.

## What This Changes

The issue requests that MCP stdio server child processes inherit gateway proxy-related environment variables and NODE_OPTIONS, or that the required explicit env workaround be documented clearly.

## Best Possible Solution

Review and rebase PR #65574, or land an equivalent current-main patch in the MCP stdio launch path that propagates safe ambient proxy-related env with precedence tests, preserves configured-env safety filtering, and makes a deliberate documented decision for NODE_OPTIONS.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: set HTTP_PROXY, HTTPS_PROXY, or NODE_OPTIONS on the gateway process and use a stdio MCP config with only command/args; current main resolves env from raw server config only, then spawns with SDK defaults that exclude those ambient keys.

## Solution Assessment

Unclear for the full requested NODE_OPTIONS behavior. Propagating safe ambient proxy env from the MCP stdio launch path is the narrow maintainable fix, but NODE_OPTIONS crosses the current startup-safety boundary and should be explicitly approved and tested rather than copied blindly.
