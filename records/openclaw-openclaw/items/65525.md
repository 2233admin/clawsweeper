## Summary

Keep this issue open. Current main still does not auto-propagate gateway proxy variables or NODE_OPTIONS into MCP stdio child processes: the launch config only forwards configured server env, and the stdio transport spawns with MCP SDK safe defaults plus explicit server env. The issue is also paired with open unmerged PR #65574 using closing syntax, so the maintainer path is to review, land, or replace that PR rather than close the issue early.

## Best Possible Solution

Keep this issue open and review #65574 as the implementation candidate, or replace it with a narrower current-main PR that auto-propagates safe proxy env vars to MCP stdio children while preserving the NODE_OPTIONS startup-safety policy unless maintainers explicitly decide otherwise. The fix should include MCP stdio unit coverage and docs clarifying inherited versus configured env.
