## Summary

Close as implemented on main. The newer merged #74107 replaced this PR's process-global registry approach with a broader local bridge and stable Codex relay-id implementation that covers duplicate package copies/processes and has focused tests; the fix is in the v2026.4.29-beta.1 release line.

## What This Changes

This PR moves native hook relay registry, invocation, and permission state from module-local Maps/arrays to a process-global `Symbol.for("openclaw.nativeHookRelay.state")` store and adds a test that registrations land in that shared state.

## Best Possible Solution

Keep the shipped #74107 bridge design as the maintained implementation: it solves the relay-copy failure without making the relay Maps themselves process-global, and it preserves a gateway fallback for older registrations.

## Reproduction Assessment

Yes. The linked report's failure path was `openclaw hooks relay` surfacing `Native hook relay unavailable` after gateway `nativeHook.invoke` returned `native hook relay not found`; current main has targeted tests for direct bridge invocation and stable relay replacement across Codex app-server runs.

## Solution Assessment

Yes. The merged local bridge plus stable relay id is a better maintained fix than this PR's global-state patch because it handles separate process/package-copy boundaries and uses a tokenized loopback/private-file bridge rather than exposing all relay state through `globalThis`.
