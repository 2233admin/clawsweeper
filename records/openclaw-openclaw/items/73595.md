## Summary

Keep open. Current main still has the reported source-level gap: outbound text sanitization is adapter-scoped and runs before message_sending hooks/presentation rendering, Discord still has no sanitizeText adapter wiring, and the existing plain-text sanitizer would not remove <previous_response> tags even on channels that use it. The reporter and follow-up commenter provided useful confirmation that the observed tags come from the Claude Code/claude-max boundary and that Discord is the affected delivery surface; no current code, tests, docs, commit history, or provided related item supersedes that remaining work.

## Best Possible Solution

Add a shared final outbound sanitizer after message_sending hooks and presentation rendering, immediately before sendPayload/sendFormattedText/sendTextChunks delivery. Seed it with explicit internal-only wrappers for <system-reminder> and <previous_response> including empty/self-closing variants, wire Discord into the same plain-text sanitization expectations as comparable channels, and add regression tests for the helper plus the Discord/shared delivery path after hook rewrites.
