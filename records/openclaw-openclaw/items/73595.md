## Summary

Keep open. Current main has useful runtime-context leak mitigations, but they cover OpenClaw’s canonical internal markers and legacy runtime-context prefaces, not the reported `<system-reminder>` / `<previous_response>` XML tags. The outbound delivery path also lacks a channel-agnostic final sanitizer after payload normalization and message-sending hooks, so the requested delivery-layer guard is not clearly implemented or superseded.

## Best Possible Solution

Add a shared final outbound text sanitizer that strips a documented allow-list of internal-only runtime tags, including the reported XML wrappers if they are confirmed prompt scaffolding, and apply it after message_sending/presentation normalization but before every channel send path. Add focused unit coverage in outbound payload/delivery tests plus a regression case for degraded assistant text containing the reported tags.
