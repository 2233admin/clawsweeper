## Summary

Keep open. Current main still does not implement the requested delivery-boundary sanitizer for leaked `<system-reminder>` / `<previous_response>` scaffolding. Existing sanitizers cover OpenClaw runtime-context markers and `<final>` tags, but Discord delivery has no `sanitizeText` hook and the shared plain-text helper would not strip underscore tag names or remove tag bodies anyway.

## Best Possible Solution

Add a shared final outbound sanitizer in the core delivery path after `message_sending`, presentation rendering, and payload normalization but before every channel send. Seed it with documented internal-only wrapper removal for `<system-reminder>...</system-reminder>` and `<previous_response>...</previous_response>` including empty/self-closing variants, wire Discord to the same sanitization expectations as other text channels, and add regression tests for helper behavior plus Discord/shared delivery after hook rewrites.
