## Summary

Keep open. Current main still leaves a read-only paired CLI unable to clear or reshape a pending scope-upgrade request: approve/list have a local loopback fallback, but reject still uses the gateway RPC path that requires `operator.pairing`, and the docs/tests describe only list/approve fallback coverage. The reporter also confirmed this is a real recovery block, not only command ergonomics.

## What This Changes

Request a supported recovery path for a CLI paired only with `operator.read` when auto-reissued repair requests ask for broader operator scopes that the CLI cannot approve or reject.

## Best Possible Solution

Add or document a supported recovery path that does not depend on the already-limited CLI having `operator.pairing`, while preserving explicit approval for broader operator access and keeping local-only recovery boundaries clear.
