## Summary

Keep open. Current main has adjacent fixes for WebSocket handshakes, subagent announce delivery, and stale-channel health restarts, but it still does not provide the requested gateway-level announce/connection/plugin-init timeout surface, and the channel startup/health paths still leave pre-handoff and unknown-connection states without a configurable fast-fail policy.

## What This Changes

Request to add configurable gateway timeouts for announce delivery, provider/channel connection, and plugin initialization stalls, with structured logging and optional restart behavior.

## Best Possible Solution

Keep this open and split the fix design by boundary: preserve existing handshake and health monitor knobs, add a bounded and documented channel startup/connection timeout only where core can enforce it generically, keep sessions_send announce timeout aligned with #59339, and handle plugin initialization through a plugin-loader/SDK contract if maintainers decide that core should enforce it.
