## Summary

Keep PR #71463 open. Current `main` still emits the per-call voice-call restore logs this PR targets, so the cleanup is not implemented or superseded. The PR is a plausible logging-only improvement, but maintainer review and bot review both identify remaining gaps: no log-summary regression test, no count for verified-active kept calls, and one per-call max-duration skip log still left in `initialize()`.
