## Summary

Keep this PR open. Current main still does not implement Telegram ack reaction timing control: there is no shared `messages.ackReactionTiming` config, no Telegram account/channel override, no Telegram docs entry for timing resolution, and the Telegram runtime still creates the ack/status reaction during message-context construction instead of deferring it to `onAgentRunStart`. The PR is also paired with still-open same-author Slack and Discord PRs for the same shared `AckReactionTiming` surface, so this should be coordinated rather than closed independently.

## Best Possible Solution

Keep this PR open as the Telegram implementation candidate. Rebase it onto current main, coordinate the shared `AckReactionTiming` type/schema/help/docs/generated metadata with the still-open Slack and Discord PRs, and land focused Telegram tests for default `received`, deferred `run-start`, mention/scope gating, remove-after-reply, and `run-start` with status reactions preserving queued-to-thinking progression.
