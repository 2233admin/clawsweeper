## Summary

Keep open. Current main still does not implement Telegram ack reaction timing control: there is no shared `AckReactionTiming` config, no Telegram account override, no Telegram docs entry, and the Telegram runtime still starts ack/status reactions during message-context construction rather than from an `onAgentRunStart` hook. This PR remains a plausible implementation candidate and should be coordinated with the open sibling channel PRs #45421 and #45449.

## Best Possible Solution

Keep this PR open as the Telegram implementation candidate. Rebase it onto current main, coordinate shared `messages.ackReactionTiming` config/schema/docs/generated metadata with #45421 and #45449, and verify focused Telegram tests for default `received`, deferred `run-start`, mention/scope gating, remove-after-reply, and `run-start` with status reactions.
