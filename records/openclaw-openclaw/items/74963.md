## Summary

Keep open. Current main still derives the packaged bundled runtime-deps stage key from the lexical `packageRoot` path, and the provided discussion shows #75048 is an open, unmerged fix PR with closing syntax for this exact hash-divergence bug. The extra WhatsApp and macOS/profile comments look like useful related runtime-deps data points, not evidence that the issue is already resolved.

## What This Changes

The issue asks OpenClaw to fix 2026.4.27 packaged multi-instance runtime-deps hash divergence that makes bundled channels and plugins import missing staged `dist` files.

## Best Possible Solution

Review and land #75048, or an equivalent narrow replacement, so packaged bundled runtime-deps hash a canonical filesystem path for the package root and include regression coverage for equivalent real/symlink/junction package-root views. After that lands, separately evaluate whether disabled or unconfigured bundled channel entrypoints still create noisy status/health imports.

## Reproduction Assessment

Yes. A focused reproduction is to expose the same packaged OpenClaw root through real and symlink/junction-style lexical paths and compare the resolved external runtime-deps install roots; current main still has the lexical hash seam that can split those roots.

## Solution Assessment

Yes. Canonicalizing `packageRoot` before hashing the stage key appears to be the narrowest maintainable fix because it converges the runtime-deps directory without broadening bundled channel loading or changing plugin activation policy.
