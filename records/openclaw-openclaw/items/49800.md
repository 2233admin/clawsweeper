## Summary

Keep this PR open. Current main still does not implement its bounded same-auth-profile retry for transient overloaded LLM errors; the prompt and assistant paths still route overloaded failures through auth-profile rotation, optional overload backoff before failover, model fallback, or surfaced error. The PR remains a focused implementation candidate, with #49376 still tracking the broader configurable LLM retry policy.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, add bounded same-auth-profile retry for transient overloaded errors in both prompt and assistant failover paths, preserve or add behavioral coverage for two overloaded attempts followed by success, normalize retry sleep cancellation as AbortError, and decide how this narrow policy relates to #49376.
