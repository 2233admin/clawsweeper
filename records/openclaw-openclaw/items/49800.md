## Summary

Keep open. Current main still handles overloaded LLM errors through auth-profile rotation, optional rotation backoff, and model fallback/surface-error; it does not implement this PR's bounded same-auth-profile retry loop or `retry_same_profile` observation. The discussion usefully split the cooldown concern into #49807, but this PR remains a distinct implementation candidate and still needs maintainer review plus the abort-handling review concern addressed.

## Best Possible Solution

Keep the PR open as a focused implementation candidate. Rebase it onto current main, preserve the intended bounded overloaded same-profile retry in both prompt and assistant paths, add or keep behavioral coverage, normalize retry-sleep aborts as `AbortError`, and then decide whether it lands as the narrow fix or is folded into the broader configurable retry work tracked by #49376.
