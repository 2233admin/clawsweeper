## Summary

Keep open. Current main still does not implement this PR's bounded same-auth-profile retry for transient overloaded LLM errors before auth-profile rotation, model fallback, or surfacing the error. The PR remains a focused implementation candidate, alongside the broader configurable retry request in #49376.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, preserve bounded same-auth-profile retry for transient overloaded errors in both prompt and assistant paths, keep behavioral coverage, normalize retry-sleep cancellation as AbortError, and decide whether the hardcoded narrow policy should land separately or be folded into #49376.
