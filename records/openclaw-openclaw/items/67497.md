## Summary

Keep this issue open. Current main still routes Feishu setup/status through `probeFeishu`, that probe still constructs the normal Feishu SDK HTTP client without a quiet/custom logger path, and the pinned SDK defaults the client logger to `info` and emits `client ready`; same-author PR #67734 remains open and unmerged as the implementation candidate.

## What This Changes

Request that Feishu setup/status probes stop showing `@larksuiteoapi/node-sdk` info logs in the interactive setup wizard while preserving the normal Feishu connection status line.

## Best Possible Solution

Land a Feishu-owned fix that gives setup/status probing a quiet or custom SDK logger path, preserves `Feishu: connected as ...`, avoids changing normal runtime logging more broadly than needed, and adds focused regression coverage in the Feishu client/probe/setup-surface tests.
