## Summary

Keep this issue open. Current main still routes Feishu setup status through `probeFeishu`, and that probe still constructs the normal `@larksuiteoapi/node-sdk` HTTP client without a quiet/custom logger. The supplied GitHub context also shows same-author PR #67734 is open, unmerged, and uses closing syntax for this issue, so the issue should remain open until that PR is reviewed and either landed or closed.

## Best Possible Solution

Keep this issue open and review PR #67734 as the implementation candidate. If correct, land a Feishu-owned change that gives setup/status probes a quiet or custom SDK logger while preserving `Feishu: connected as ...`, with focused regression coverage in Feishu client/probe/setup-surface tests. After that merges, close this issue as implemented.
