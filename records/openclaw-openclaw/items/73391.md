## Summary

Keep open. Current main still has the two CLI lazy-runtime behaviors this PR addresses: nested lazy reparse starts from the immediate parent and can drop parent options, and eager command-group registration still fire-and-forgets async dynamic imports. The PR is open, unmerged, active at head 14ac136ae9720790adf8760bc629f163d874f539, and its latest diff also addresses the earlier Greptile fallback-path concern.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Review the public `plugin-sdk/cli-runtime` compatibility impact, confirm the latest fallback-path fix is acceptable, then land after the targeted CLI/browser tests and changed gate are green. Current main does not yet implement the intended fix.
