## Summary

Keep open. Current main only partially supersedes the parent-option part of this PR via a newer lazy-command fallback fix, but the eager registration path still fires async command-group registration without awaiting it, so the reported eager-mode regression remains credible. Because the branch overlaps newer main changes and changes the exported CLI-runtime helper contract, this should get maintainer rebase/review rather than cleanup closure.

## What This Changes

The PR changes lazy CLI reparsing to use root/full command-path fallbacks, makes eager command-group registration awaited through core and browser CLI wiring, updates affected tests, and adds changelog entries.
