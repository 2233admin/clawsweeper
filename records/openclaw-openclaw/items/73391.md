## Summary

Keep open. Current main still has both defects described by this PR: lazy reparse starts from the immediate parent command and can drop parent options, and eager registration still fires async command-group imports without awaiting them. The PR is a focused implementation candidate with relevant regression tests; it should be reviewed rather than closed as obsolete.

## Best Possible Solution

Keep this PR open, review the focused lazy CLI fix, resolve or explicitly waive the fallback-argv review concern, then land it after the targeted CLI/browser tests and changed gate are green. Current main does not yet implement the intended behavior.

## Evidence

- **Current main lazy reparse still uses immediate parent:** `reparseProgramFromActionArgs` sets `root` to `actionCommand?.parent ?? program`, reads raw args from that command, builds fallback argv from only the leaf action command, and then reparses `program`. For browser lazy subcommands, `program` is the `browser` command, so parent options on the root invocation are not preserved by current main.
  - file: [src/cli/program/action-reparse.ts:10](https://github.com/openclaw/openclaw/blob/db40ec404a9178294dba3121758ff59459983a11/src/cli/program/action-reparse.ts#L10)
  - command: `nl -ba src/cli/program/action-reparse.ts | sed -n '1,180p'`
  - sha: [db40ec404a91](https://github.com/openclaw/openclaw/commit/db40ec404a9178294dba3121758ff59459983a11)
- **Commander dependency behavior supports the reported root cause:** Commander v14.0.3 initializes `rawArgs` on parse/parseAsync through `_prepareUserArgs`, but `_dispatchSubcommand` prepares and parses the subcommand without running `_prepareUserArgs` on that subcommand; `restoreStateBeforeParse` also clears `rawArgs`. This matches the PR’s explanation that reparsing from the immediate parent can see empty raw args. 
  - command: `Opened upstream commander v14.0.3 command.js source`
- **Current eager registration still races dynamic imports:** `registerCommandGroups` is synchronous and, in eager mode, loops over entries with `void entry.register(program)` before returning. Browser command groups perform dynamic imports inside `entry.register`, so callers can proceed to parse before subcommands are attached.
  - file: [src/cli/program/register-command-groups.ts:70](https://github.com/openclaw/openclaw/blob/db40ec404a9178294dba3121758ff59459983a11/src/cli/program/register-command-groups.ts#L70)

The browser plugin entrypoints also call `registerBrowserCli(program)` without awaiting it.
  - file: [extensions/browser/src/cli/browser-cli.ts:146](https://github.com/openclaw/openclaw/blob/db40ec404a9178294dba3121758ff59459983a11/extensions/browser/src/cli/browser-cli.ts#L146)
