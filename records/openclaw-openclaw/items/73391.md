## Summary

## What This Changes

The PR changes lazy CLI reparsing to re-dispatch from the root command with full fallback argv, makes eager command-group registration await dynamic imports through core/subCLI/browser CLI wiring, and updates related CLI tests and changelog entries.

## Best Possible Solution

Land a corrected version that keeps the root/full-path lazy reparse, awaits eager dynamic-import registration without lint violations, updates all newly async test callsites, fixes the eager status-action assertion, and folds the changelog bullets into the existing active fixes section.

- **[P2] Normalize registrations before passing them to Promise.all:** `src/cli/program/register-command-groups.ts:84`
  - body: `CommandGroupEntry.register` is typed as `Promise<void> | void`, so `Promise.all(entries.map((entry) => entry.register(program)))` trips the repo's `await-thenable` lint rule on this PR head. Wrap each registration in an async callback or `Promise.resolve(...)` so the eager path still awaits every dynamic import without failing lint.
  - confidence: 0.98
- **[P2] Await the now-async browser CLI registrar in lazy tests:** `extensions/browser/src/cli/browser-cli.lazy.test.ts:140`
  - body: After `registerBrowserCli` becomes async, the remaining un-awaited test calls are reported by CI as `no-floating-promises`. Please await the lazy-mode setup calls too, including the existing placeholder tests and the new parent-option regression test, so lint and any registration failures are handled deterministically.
  - confidence: 0.97

  - body: The mocked `status` command has no positional arguments, so Commander passes the `Command` object as the first and only action argument.
