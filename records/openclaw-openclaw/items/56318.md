It should not be auto-closed as implemented or superseded unless maintainers first split/land the remaining verified fixes or open narrower canonical follow-ups.

## Risks / Open Questions

- The PR is broad and touches security/code-execution surfaces: local shell spawning, secret resolver child environments, SQL schema construction, local media filesystem access, and Signal release download handling. Those should be reviewed as narrow, independently tested fixes rather than merged as one bundle.
- The PR's proposed `stream.write()` backpressure change has a concrete P1 review concern: callers can interpret normal backpressure as a terminal write failure and truncate logs.
- The PR's proposed session directory listing cache has a concrete P2 review concern: stale cached entries can misresolve recently rotated or reset session files.
