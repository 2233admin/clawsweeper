## Summary

Keep open. Current main and the latest tagged release still contain the reported prompt mismatch, and the runtime path can still turn unwrapped final-tag-enforced text into an empty-response retry/incomplete-turn path. The fix is narrow enough for a focused repair PR.

## What This Changes

The issue requests changing non-relayable async exec completion prompts so they use the runtime silent-reply token with required final-tag wrapping instead of asking for `HEARTBEAT_OK`.

## Best Possible Solution

Update the exec-event no-output and disabled-user-delivery prompt branches to request the exact final-wrapped silent reply, keep the generic final-tag and empty-response safeguards intact, and add focused regression coverage for both prompt cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildExecEventPrompt` with `deliverToUser:false` or a metadata-only successful exec event and observe the current `HEARTBEAT_OK only` prompt; with final-tag enforcement, bare `NO_REPLY` is dropped before payload assembly and reaches the empty-response retry path.

## Solution Assessment

Yes. The narrowest maintainable fix is prompt-side alignment with the existing silent-token/final-tag contract, with a runtime regression only if needed; weakening generic final-tag enforcement would be broader and riskier.
