## Summary

Keep this PR open for maintainer review. Current main still lacks the retry-disable behavior and the upstream SDK retry default supports the root-cause theory, but the proposed setter can persist `retry.enabled=false` into the file-backed/global Pi settings path, so the branch is plausible but not yet the best fix.

## What This Changes

The branch disables pi-coding-agent auto-retry in `createPreparedEmbeddedPiSettingsManager` and adds a Vitest assertion that the prepared settings manager reports retry disabled.

## Best Possible Solution

Land a narrow fix that disables pi-coding-agent agent-level auto-retry only for OpenClaw embedded sessions, without mutating user/global Pi settings, and coordinate that fix with the active #73781 and #74503 loop work.

- **[P2] Keep retry disablement session-scoped:** `src/agents/pi-project-settings.ts:57`
  - body: Calling `setRetryEnabled(false)` here is not a pure runtime override. `createEmbeddedPiSettingsManager` can return a file-backed manager when project settings policy is `trusted` and no plugin settings are loaded, and upstream `setRetryEnabled()` saves `retry.enabled` into global settings.
