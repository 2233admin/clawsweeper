## Summary

Keep this PR open. Current main still accepts the narrower Feishu Schema 2.0 `context.open_chat_id` callback shape, but it does not normalize nested Schema 2.0 operator identity from `operator.user_id.{open_id,user_id,union_id}`. The PR remains a focused fix for the remaining Feishu card-action gap, and maintainer discussion already identified it as the better base than the closed narrower PR #71692.

## Best Possible Solution

Keep this PR open and review or land the nested Schema 2.0 identity normalization on top of current main. The best implementation path is to extract `operator.user_id.{open_id,user_id,union_id}` and equivalent event-level nested identity fields, preserve existing `context.open_chat_id` handling and malformed-payload guards, keep the lifecycle regression test for the nested callback shape, and update the changelog attribution without changing outbound card schema, routing, permissions, or command semantics.
