## Summary

Keep this PR open. Current main still swallows `QueuedFileWriter` write failures with `.catch(() => undefined)`, so the warning behavior requested by #57031 is not implemented. Main has since grown `flush()` and hardened append handling, so the PR is still a useful implementation candidate but needs rebasing onto the current writer shape.

## Best Possible Solution

Keep the PR open as an implementation candidate. Rebase it onto current main, implement consecutive failure tracking around the existing `safeAppendFile(...)` queue while preserving `flush()` and the security hardening, then add focused tests for threshold warning, periodic re-warning, reset after a successful write, and existing safe-append behavior.
