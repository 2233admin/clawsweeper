## Summary

Keep open. Current main still has Google Vertex setup evidence accepting APPDATA-only ADC credentials while the runtime ADC loader only checks GOOGLE_APPLICATION_CREDENTIALS and the HOME gcloud fallback; this PR narrows that drift in the Google plugin with matching transport coverage and no concrete security concern.

## What This Changes

This PR updates the Google Vertex ADC resolver to check the Windows APPDATA gcloud ADC fallback after the HOME fallback and adds transport-stream coverage for that APPDATA path.

## Best Possible Solution

Land a narrow Google-plugin fix that keeps setup evidence and request-time ADC resolution aligned: explicit credential path first, HOME gcloud ADC second, APPDATA gcloud ADC third, with focused transport coverage and the changed gate before merge.
