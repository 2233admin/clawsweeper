## Summary

Keep open. The report is a concrete latest-release WebChat regression, and current main still has an immediate same-session `session.message` history-reload path after a renderable final clears `chatRunId`; existing tests cover the earlier deferred-before-final ordering but not the reported final/session.message/sessions.changed race.

## What This Changes

The issue reports that WebChat Control UI still renders duplicate assistant reply bubbles on OpenClaw 2026.4.27 after the earlier duplicate-message fixes for #66875, #66274, #72975, #72963, #72974, and #72753.

## Best Possible Solution

Add a regression-backed Control UI reconciliation fix that makes normal renderable assistant finals idempotent across `chat.final`, later same-session `session.message`, and `sessions.changed` events, while still allowing required history refreshes for silent finals, tool output, abort/error recovery, and genuinely external transcript updates.

## Reproduction Assessment

Unclear. The issue gives a concrete browser reproduction path and the source shows the remaining post-final reload path, but this read-only review did not live-run WebChat and the current tests do not reproduce the duplicate bubble after final followed by `session.message` or `sessions.changed`.

## Solution Assessment

Yes for the next step. The narrowest maintainable path is a focused Control UI event-ordering regression test followed by a small idempotent reload/reconciliation change, not a broad suppression of all history refreshes.
