## Summary

Keep this issue open. Current main still starts channel accounts concurrently in the gateway, Feishu `startAccount` still calls the per-account monitor path, and that per-account path still fetches bot identity directly without a shared startup probe queue. Open PR #63481 and fork PR BingqingLyu/openclaw#2190 also use closing syntax for this issue, so the issue should remain open until a linked implementation is merged or closed.

## Best Possible Solution

Keep this issue open and review PR #63481 or fork PR BingqingLyu/openclaw#2190. The fix should stay Feishu-owned: serialize or otherwise throttle bot identity probing across concurrent gateway `startAccount` calls, add regression coverage for the actual per-account gateway startup path, and avoid changing generic gateway concurrency unless maintainers decide that broader channel behavior should change. Reconcile that with #67466's startup-latency proposal before landing.
