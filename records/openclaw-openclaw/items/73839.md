## Summary

Keep this PR open for normal review. The direction is right and the earlier CLI type issue appears addressed, but the patch is still not merge-ready because it passes only the raw compaction mode into the Pi guard while current behavior treats a configured compaction provider as safeguard-owned too.

## What This Changes

The PR updates the Pi auto-compaction guard to accept `AgentCompactionMode`, disables Pi auto-compaction for safeguard mode, threads the mode through embedded and CLI compaction paths, reapplies the guard after resource reload, and adds focused `pi-settings` tests.

## Best Possible Solution

Land a narrow fix that derives one shared “OpenClaw owns compaction” predicate from the same resolved safeguard rules used by the embedded extension builder, then use that predicate for both initial guard application and post-reload reapplication while preserving the existing `ownsCompaction` behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists with `agents.defaults.compaction.provider` set and raw `mode: "default"`: current extension resolution enables safeguard, while the PR guard would see only `"default"` and leave Pi auto-compaction enabled.

## Solution Assessment

No. The PR is close, but the best fix should share or mirror the provider-aware safeguard resolver instead of passing only the raw mode string.

## Review Findings
