## Summary

Keep this PR open. Current main still uses the 30-second completion-cache timeout in both affected paths, and the PR is a focused follow-up to the shipped message-only fix; it needs maintainer review, current-main rebase/changelog cleanup, and validation rather than closure.

## What This Changes

This PR raises update and doctor completion-cache refresh timeouts from 30 seconds to 120 seconds, adds non-JSON update progress and success-duration output, adds regression coverage, and adds a changelog entry.

## Best Possible Solution

Land this PR or an equivalent maintainer patch after rebasing: raise both completion-cache refresh caps to 120 seconds, preserve JSON-mode silence and manual-refresh guidance, keep focused update and doctor regression tests, move the changelog entry to the current Unreleased Fixes section, and leave broader onboarding performance work to #74879.

## Reproduction Assessment

Yes. Current main reproduces the timeout path whenever completion cache generation takes more than 30 seconds, because both update follow-up and doctor repair pass `timeout: 30_000` to spawnSync; #72842 and the PR body provide matching ETIMEDOUT observations.

## Solution Assessment

Yes for the targeted timeout behavior. The proposed 120-second cap plus visible non-JSON progress is a narrow fix that preserves the existing non-fatal fallback, though the PR should be rebased and validated against current main before landing.
