## Summary

Keep this PR open. Current main still enforces the 30-second completion-cache refresh timeout in both the package-update follow-up path and the doctor completion repair path, and the existing regression test still asserts 30 seconds. The related merged PR #72850 shipped the friendlier timeout message only; this PR remains a focused follow-up for the still-current slow-cache behavior described in #72842.

## Best Possible Solution

Review and land this PR, or an equivalent maintainer patch, after rebasing onto current main: raise both completion-cache refresh timeouts to 120 seconds, preserve the non-fatal/manual-refresh behavior from #72850, add the non-JSON start/finish progress output if maintainers want visible slow-step feedback, and keep regression coverage for both update and doctor paths.
