## Summary

Keep this PR open. Current main still uses a 30-second completion-cache refresh timeout in both the update follow-up path and doctor repair path, and the existing regression test still asserts 30 seconds. The PR targets a real remaining gap after the merged friendly-message fix in #72850: slower successful cache rebuilds can still be killed before completion.

## Best Possible Solution

Review and land this PR, or an equivalent maintainer patch, after rebasing onto current main: raise both completion-cache refresh timeouts to 120 seconds, keep the non-fatal/manual-refresh behavior from #72850, add the proposed non-JSON progress output if maintainers want visible slow-step feedback, and keep regression coverage for both update and doctor paths.
