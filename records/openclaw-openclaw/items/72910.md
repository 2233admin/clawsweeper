## Summary

Keep this PR open. Current main still caps completion-cache refresh at 30 seconds in both the update follow-up path and doctor completion repair path; the related shipped work in #72850 improved the warning text for #72842 but did not raise the timeout or add the PR's progress output. The PR remains a focused implementation candidate with no obvious security-sensitive drift in the provided diff.

## Best Possible Solution

Review and land this PR, or an equivalent maintainer patch, because the remaining behavior is still current on main. The implementation should raise both completion-cache refresh timeouts to 120 seconds, preserve JSON-mode silence and the non-fatal/manual-refresh behavior from #72850, keep the non-JSON progress output if maintainers accept that UX, and retain regression coverage for both update and doctor completion-cache generation paths.
