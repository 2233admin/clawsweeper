## Summary

Keep this PR open. Current main still has the 30-second completion-cache refresh timeout in both the update follow-up path and the doctor completion repair path, while this PR is a focused follow-up that raises those caps to 120 seconds and adds regression coverage/progress output. The related shipped work in #72850 only improved the warning text for #72842; it did not solve the slow-but-successful cache refresh behavior this PR targets.

## Best Possible Solution

Review and land this PR, or an equivalent maintainer patch, because the remaining behavior is still current on main: raise both completion-cache refresh timeouts to 120 seconds, preserve the non-fatal/manual-refresh behavior from #72850, keep JSON-mode silence, include the non-JSON progress output if maintainers accept that UX, and retain regression coverage for both update and doctor completion-cache generation paths.
