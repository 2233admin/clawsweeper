## Summary

Keep this PR open. Current main still does not implement local-offset session transcript timestamps: OpenClaw-owned transcript headers still use `new Date().toISOString()`, multiple transcript append/branch/rewrite paths still open `SessionManager` directly, and the pinned `@mariozechner/pi-coding-agent` version still writes UTC `...Z` timestamps internally. The PR is still a focused implementation candidate, with maintainer review needed for the private-API wrapper and malformed-timestamp availability concern.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow transcript timestamp fix that centralizes explicit local-offset ISO formatting for OpenClaw-owned transcript headers and SessionManager-managed entries, covers branch/compaction/gateway-injection paths, adds regression tests for preserving the instant with an explicit offset, and handles malformed existing timestamps without persistent session open/write failures.
