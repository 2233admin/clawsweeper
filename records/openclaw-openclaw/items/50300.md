## Summary

Keep this PR open. Current main still does not implement local-offset session transcript timestamps: transcript header creation paths still use `new Date().toISOString()`, transcript append/inject/run paths still open `SessionManager` directly without a local timestamp wrapper, and the proposed `src/sessions/local-session-timestamps.ts` helper is absent from main. The PR remains a focused implementation candidate, but it needs maintainer review for the private `SessionManager` hook strategy and the invalid-timestamp availability concern raised in review.

## Best Possible Solution

Keep the PR open and update/review it against current main. The best path is a narrow transcript-timestamp fix that covers all header creation, append, rewrite, branch/fork, compaction, gateway injection, and recovery paths; uses an existing centralized local-offset formatter where possible; adds focused regression tests for local-offset strings preserving the instant; and handles malformed persisted timestamps without making a session permanently unopenable.
