## Summary

Keep this issue open. Current main still treats heartbeat as a scheduled agent turn that is backpressured by the shared main/global lane and resolved session lane; it does not implement a separate heartbeat worker or non-blocking execution lane. The discussion is still useful: GMTekAI correctly framed this as an architecture feature request, and Ryce's KeepMyClaw snapshot note is an interim guardrail rather than a core implementation.

## Best Possible Solution

Keep this open as an architecture/product feature request. The strongest path is a dedicated heartbeat execution lane or worker-style isolated heartbeat turn model with explicit contracts for session state, delivery ordering, tool side effects, wake retry cadence, concurrent inbound user messages, and how existing isolatedSession/lightContext/model overrides remain cost/context mitigations rather than completion of the feature.
