## Summary

Keep this issue open. Current main still implements heartbeat as a scheduled agent turn that is backpressured by the shared main/global lane and the resolved session lane; it does not add a dedicated heartbeat worker or non-blocking heartbeat execution lane. The comments from GMTekAI and Ryce are useful: GMTekAI correctly framed this as an architecture feature request, while Ryce's KeepMyClaw snapshot note is an interim guardrail rather than a core implementation.

## Best Possible Solution

Keep this open as an architecture/product feature request. The strongest path is a dedicated heartbeat execution lane or worker-style isolated heartbeat turn model with explicit contracts for shared session state, per-session ordering, tool side effects, wake retries, delivery behavior, and how existing isolatedSession/lightContext/model overrides remain cost/context mitigations rather than completion of this feature. Review #64963 and #72814 as narrower adjacent fixes, but do not treat them as closing this broader request unless maintainers intentionally narrow the scope.
