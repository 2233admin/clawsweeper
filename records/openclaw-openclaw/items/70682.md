## Summary

Keep open. Current main does not contain this PR's host read-tool wrapper, the linked security report remains open, and the PR is an active but oversized implementation candidate with blocking functional/security review concerns rather than conservative cleanup material.

## What This Changes

The PR adds workspaceOnly-aware host read-tool operations and tests, while also bundling unrelated web chat, embedded runner, config write, gateway locality, Telegram locking, Discord group fallback, media URI, and Anthropic schema changes.

## Best Possible Solution

Split this into focused PRs. The read-tool fix should define one effective filesystem policy for read/write/edit/sandbox/tool-host paths, make the intended default and opt-out behavior explicit, and cover cross-agent memory reads, absolute paths, traversal, symlink/hardlink escapes, and explicit workspaceOnly=false behavior with targeted tests.

## Security Review

Status: needs_attention

Summary: The PR is security-sensitive and still leaves a default host-read privacy path open while adding an unrelated symlink-following config write behavior.
