## Summary

Current main still hardcodes reasoning display as italic `Reasoning:\n_..._` text and does not expose a per-channel, per-agent, or global reasoning display-format setting. The related implementation PR #25336 was closed unmerged, and its review context called out incomplete parameter threading, so it does not implement or supersede the requested feature.

## Best Possible Solution

Keep this issue open. A complete fix should define a small reasoning display-format contract, preserve italic as the compatibility default, choose and document precedence for channel versus agent/global settings, thread the selected format through runner payloads, subscribe streaming, message-end block replies, and Telegram reasoning lanes, then add focused tests and docs for default italic plus configured code/spoiler/blockquote behavior.
