## Summary

Keep this PR open. Current main still does not route the final agent reply to `agents.defaults.imageModel` when an image arrives on a text-only primary model; the existing implementation only uses `imageModel` in media-understanding/tool paths and keeps the final reply on the normal resolved provider/model. The PR addresses a real gap, but the branch still needs maintainer review because review comments identify unresolved model-routing semantics around alias/fallback resolution, media-understanding ordering, actual image detection, unknown catalog entries, and override precedence. The security pass did not find supply-chain or CI-risk changes in the provided file list.

## What This Changes

The PR adds a `resolveImageModelAutoSwitch` helper and colocated tests so inbound image media can switch the reply provider/model to `agents.defaults.imageModel.primary` when the active model lacks vision support.

## Best Possible Solution

Implement the behavior in the current reply/model-selection path by positively detecting real image attachments, proving the active model cannot accept images, resolving `agents.defaults.imageModel` primary and fallbacks through the existing alias/catalog helpers, respecting explicit override sources, and coordinating with media-understanding so a final vision-model run does not also do redundant image-description work. Keep focused regression tests around WebChat attachments, media-understanding routing, model fallbacks, and directive/session overrides.
