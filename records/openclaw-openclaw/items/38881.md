## Summary

Keep this PR open. Current main still does not implement the PR's intended final-agent-run auto-switch from a text-only selected/default model to `agents.defaults.imageModel` when image input is present. Main has image-model routing for media-understanding/tool paths and recent WebChat offload preservation, but the final reply run still receives the active provider/model and uses normal model fallback. The PR addresses a real gap linked from #38796, but the proposed diff still needs maintainer-quality follow-up for the remaining functional review concerns before merge.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The best fix belongs in the reply/model-selection path: detect normalized current-turn image attachments, resolve `agents.defaults.imageModel` primary and fallbacks through the real alias/catalog helpers, switch the final `runPreparedReply` model only when the active model is positively known not to accept images, respect heartbeat/session/channel overrides, avoid duplicate media-understanding work, and add focused regression tests proving the final agent run receives the configured vision model.
