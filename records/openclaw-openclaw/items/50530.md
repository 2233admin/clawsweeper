## Summary

Keep open. Current main still preserves single newlines in Telegram outbound text: Telegram delivery passes reply text through markdown chunk/render helpers, the shared Markdown IR renders both softbreak and hardbreak tokens as `\n`, and tests explicitly expect newline mode to keep `Line one\nLine two` as one paragraph rather than collapse it. There is no documented or schema-backed `telegram.collapseNewlines`-style option yet.

## Best Possible Solution

Keep this issue open and implement a Telegram-owned, Markdown-aware soft-line-break normalization path. The safest route is an explicit `channels.telegram.collapseNewlines`/account-level option, plus tests showing softbreaks inside prose/list items collapse to spaces while blank lines, list item boundaries, blockquotes, hardbreaks, and code fences remain correct. If maintainers want this as the new default, make that decision in the PR and document the opt-out or migration behavior.
