## Summary

Keep #40125 open. Current main implements the related reliability pieces: pre-prompt compaction, overflow recovery with reset/new guidance, no context-overflow model fallback, /context reporting, and a Control UI context notice. I did not find the requested auto-reply/Telegram proactive warning path, one-time warning/urgent per-session state, or persistent-chat message that tells users to use /reset or /new before failure.

## Best Possible Solution

Keep the issue open as a focused persistent-channel UX enhancement. Add auto-reply/session-level context-pressure evaluation using existing contextTokens, totalTokensFresh, /context, and compaction metadata; persist a warning/urgent marker per session; send concise one-time guidance to routable persistent channels such as Telegram DMs; and include explicit /reset and /new recovery instructions while leaving the existing compaction and fallback behavior intact.
