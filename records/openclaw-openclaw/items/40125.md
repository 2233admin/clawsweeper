## Clownfish Work Prompt

Implement proactive context-pressure warnings for persistent chat sessions. Current main already has preflight compaction, /context, /status, post-overflow recovery, and a Control UI context notice, but Telegram and similar auto-reply sessions do not receive one-time warning/urgent guidance before provider invocation.

Expected boundary:
- Add a small helper in the auto-reply/session pipeline that evaluates fresh context usage before the model run, using existing totalTokens/contextTokens or the same fresh transcript fallback paths already used by preflight compaction where practical.
- Persist per-session warning state so warning and urgent thresholds are emitted once per threshold and reset naturally after /reset, /new, successful compaction that drops usage below the threshold, or session replacement.
- Send concise chat-visible guidance for persistent channel sessions: the session is nearing the model context limit, continuing in the same chat keeps history/tool output in context, use /reset to clear this session or /new to start fresh; mention /compact only if it is an appropriate supported recovery action.
- Avoid warning from stale token snapshots, heartbeats, CLI-only paths, and non-chat/internal runs.
- Do not change provider catalogs, fallback model selection policy, dependency versions, or channel-specific transport internals unless strictly needed.
- Add focused regression tests for warning threshold, urgent threshold, dedupe, stale-token suppression, and reset/new clearing behavior. Update docs if user-visible behavior changes.

Related refs: openclaw/openclaw#40125, openclaw/openclaw#50065, openclaw/openclaw#68494.
