## Summary

Keep this PR open. Current main still reuses the same Telegram draft-stream reply parameters after `forceNewMessage()`, so rotated preview messages can still carry `reply_to_message_id`. The open PR remains the canonical implementation candidate and its latest branch appears to address the earlier review concerns around races, first-send failure recovery, thread preservation, and `replyToMode: "all"`.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best path is to preserve the contributor PR as the canonical fix, run the Telegram draft-stream tests and changed gate on the latest branch, and handle the existing thread-fallback security question either in this review or as a narrow follow-up before merge.
