## Summary

Keep this PR open. Current main still builds one reusable Telegram draft-stream reply parameter object and passes only `replyToMessageId` into `createTelegramDraftStream`; it does not pass `replyToMode`, track a single-use reply target, or include the PR's regression tests. The live PR branch is still open and contains the focused repair plus coverage for first-only replies, thread preservation, retry-safe failures, `replyToMode: "all"`, and materialized draft previews. A security review comment on the touched thread-fallback path still needs maintainer handling before landing or explicit split-off.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or port the current repaired branch after resolving, explicitly accepting, or splitting the thread-fallback security concern: pass `replyToMode` into `createTelegramDraftStream`, apply `reply_to_message_id` only to the first successful rotated preview for single-use modes, preserve `message_thread_id` on later previews, keep `replyToMode: "all"` behavior, and retain the regression tests and changelog credit.
