## Summary

Keep this PR open. Current main still does not implement the PR’s Telegram draft-stream reply gating: rotated streaming preview messages continue to reuse the same `reply_to_message_id`, while the open PR now appears to address the earlier review concerns around `replyToMode: "all"` and first-send failure recovery. This is an implementation candidate for maintainer review, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or landing after the latest branch is checked.
