## Summary

Keep this PR open. Current main still does not implement the Telegram draft-stream reply gating: `createTelegramDraftStream` reuses one `replyParams` object for every preview send, `forceNewMessage()` does not consume single-use reply state, and dispatch still passes `replyToMessageId` without `replyToMode`. The live PR branch is still open and contains the intended fix plus regression coverage, but maintainers should resolve or explicitly split the thread-fallback security concern before landing.

## Best Possible Solution

Keep this PR open for maintainer review and either land or port the repaired branch: pass `replyToMode` into `createTelegramDraftStream`, apply `reply_to_message_id` only to the first successful rotated preview for single-use modes, preserve `message_thread_id` on later previews, keep `replyToMode: "all"` behavior, and retain the regression tests. Resolve the thread fallback security concern in this PR or split it into a maintainer-accepted follow-up before merge.
