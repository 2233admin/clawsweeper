## Summary

Current main still does not run an agentic memory flush against the previous session before `/new`, `/reset`, daily reset, or idle reset. It has related mitigations: manual reset hooks feed the bundled `session-memory` hook, stale daily/idle resets archive old transcripts, and startup context can preload recent canonical or slugged memory files. Those do not satisfy the requested pre-reset curated flush, and there are several open implementation candidates that should be reviewed rather than auto-closing this issue.

## Best Possible Solution

Keep this issue open as the canonical product/design thread for a unified pre-reset memory persistence flow. The best fix is a maintainer-reviewed implementation that runs a bounded, silent agentic flush against the previous session for manual `/new` and `/reset`, and decides the daily/idle rollover semantics alongside existing `session-memory` and plugin hooks. Review or consolidate the open related PRs (#45643, #50584, #61675, #71792, and #49409) before starting another implementation.
