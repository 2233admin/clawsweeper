## Summary

Keep open. Current main still does not implement the PR's intended resilient session-store recovery behavior, and the PR remains an unfinished implementation candidate with unresolved correctness, contract, and data-retention questions rather than something conservative cleanup should close.

## What This Changes

The PR branch changes session-store loading to throw on corrupt or invalid JSON, writes a `.prev` backup before atomic session-store saves, adds a gateway recovery helper that can restore from `.prev` or transcripts, routes `sessions.list` and `sessions.resolve` through that helper, and updates related tests.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. The desired end state is a deliberately chosen corrupt-store contract with focused implementation and coverage for `sessions.list`, `sessions.resolve`, `.prev` restoration, transcript fallback, and backup retention/cleanup; if maintainers decide this recovery model is not wanted, close it manually after that product judgment.
