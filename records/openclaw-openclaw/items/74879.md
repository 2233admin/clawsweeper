## Summary

Keep open. Current main still contains an automatic onboarding shell-completion path that can synchronously spend up to 30 seconds generating a completion cache with full plugin CLI discovery, while the update path already uses a faster skip-plugin mode; that matches the reported slowdown and I did not find a current-main fix or canonical duplicate in the local context.

## What This Changes

The issue reports that Docker onboarding in OpenClaw 2026.4.26+ is much slower than 2026.4.14 and that shell completion often times out around 30 seconds.

## Best Possible Solution

Automatic setup health work should stay fast and bounded: onboarding and doctor should refresh only core completion cache state, while full plugin-command completion rebuilds remain available through explicit manual completion commands.

## Reproduction Assessment

Yes for the code path: a fresh QuickStart onboarding run with no completion profile/cache reaches setupWizardShellCompletion, which calls a synchronous 30-second-bounded completion cache build. The exact multi-minute Docker timing still needs live validation.

## Solution Assessment

Yes, the narrow maintainable fix is to reuse or centralize the update path's skip-plugin, timeout-aware completion cache refresh for automatic onboarding/doctor flows while preserving explicit full completion generation for manual use.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74879. Current main puts shell-completion setup on the onboarding finalization path: QuickStart auto-installs completion, then src/commands/doctor-completion.ts spawns `openclaw completion --write-state` synchronously with a 30s timeout. That command loads plugin CLI commands unless OPENCLAW_COMPLETION_SKIP_PLUGIN_COMMANDS=1, while the update path already sets that skip flag for automatic completion refreshes. Make onboarding/doctor automatic completion-cache refreshes fast and bounded, preferably by centralizing/reusing the update helper or adding a shared helper that sets OPENCLAW_COMPLETION_SKIP_PLUGIN_COMMANDS=1 and preserves timeout/friendly failure behavior. Preserve explicit manual `openclaw completion --write-state` behavior for full plugin-command completion generation unless the user opts into the fast path. Add focused tests covering QuickStart/onboarding and doctor cache generation env/timeout behavior, keep existing update completion tests green, and add a changelog fix entry with reporter credit. Do not change broad plugin CLI registration semantics or plugin runtime loading outside the automatic completion-cache refresh path.
