## Summary

Keep this PR open. Current main still has no top-level `auth` reload rule, so `auth.profiles.*` changes still fall through to the conservative unknown-path gateway restart behavior; the PR is a focused implementation candidate paired with open bug report #68258. The diff has no unexplained CI, dependency, package, lockfile, or script changes, but the broad `auth.*` no-op rule should get maintainer review because `auth.cooldowns` is also a current top-level auth config surface.

## What This Changes

The PR adds a top-level `auth` no-op reload rule to the gateway reload planner and adds regression tests for OAuth auth-profile changes, nested `auth.*` paths, and mixed `gateway.auth.*` restart batches.

## Best Possible Solution

Land this PR or an equivalent patch that prevents `auth.profiles.*` OAuth metadata writes from restarting the gateway, preserves `gateway.auth.*` restart behavior, and includes regression coverage for both the no-op auth-profile case and mixed gateway-auth/auth-profile batches. If maintainers approve the broader `auth.*` policy, add or keep coverage that makes the intended treatment of `auth.cooldowns.*` explicit.
