## Summary

Current `main` still does not implement external workspace agent auto-discovery. The agent config contract, strict schema, CLI/docs, gateway enumeration, and regression tests remain centered on explicit `agents.list` entries plus the default agent; related hot reload and disk-session discovery work does not provide trusted workspace scanning or a plugin/API seam that contributes agent definitions before enumeration.

## What This Changes

Add automatic discovery and loading of agent configurations from configured external workspace directories so new workspace-local agents do not need manual `agents.list` registration.

## Best Possible Solution

Keep this open for maintainer design. The best path is either a first-class `agents.autoDiscover` config contract or a documented plugin-backed agent-definition contribution API that runs before normal agent enumeration. It should define trusted roots, accepted filenames, schema validation, deterministic scan order, explicit precedence versus `agents.list`, watch/hot-reload behavior, docs, and regression tests; #27706 and disk-session discovery work such as #32831 are related context, not replacements.
