## Summary

Current main still does not implement automatic discovery of agent configurations from external workspaces. The related zero-restart work has moved forward for configured `agents.list` changes, but the requested external workspace scan, `agents.autoDiscover` schema, config file pattern support, and plugin/API contribution path for agent definitions are still absent.

## Best Possible Solution

Keep this open for maintainer design. The best path is either a first-class `agents.autoDiscover` config contract or a documented plugin-backed agent-definition contribution API that runs before normal agent enumeration. It should define trusted roots, filenames, schema validation, deterministic scan order, explicit precedence versus `agents.list`, watch/hot-reload behavior, docs, and regression tests. Treat #27706 and current `agents.list` hot reload as related zero-restart context, not as a substitute for external workspace discovery.
