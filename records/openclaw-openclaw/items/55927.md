## Summary

Keep this PR open. Current main has backend session-row metadata support, but the Control UI still does not request or render the room/channel/title/last-message/Guardian/cost/lineage context this PR targets, and the PR review thread still has concrete correctness feedback to address before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate, but require a rebase onto current main and a focused cleanup pass: sync the UI row type with the current gateway row contract, request `includeDerivedTitles` and `includeLastMessage` where needed, render the session/chat context using the existing backend fields, fix the token/context-notice and duration issues from review, and keep the change limited to the Control UI plus targeted tests.
