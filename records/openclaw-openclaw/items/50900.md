## Summary

Keep this issue open. Current main still exposes `session.maintenance` as a global-only retention policy: the type, strict Zod schema, resolver, save-time pruning, load-time pruning, and docs all collapse age retention to one `pruneAfterMs` value. Adjacent cleanup exists for cron run sessions via `cron.sessionRetention` and for subagents via best-effort auto-archive, but neither implements ordered per-pattern `session.maintenance.rules` with first-match retention. The KeepMyClaw comment is useful external backup context, not a replacement for the requested core retention policy.

## Best Possible Solution

Keep the issue open and implement an additive core `session.maintenance.rules` contract if maintainers accept the matcher semantics. The likely path is to add ordered `{ match, pruneAfter }` rules to the session maintenance type/schema/resolver, apply first-match pruning by session key in save/load/cleanup paths while preserving current global fallback behavior, and document how this interacts with `cron.sessionRetention`, subagent auto-archive, active-key preservation, max-entry capping, and disk-budget cleanup.
