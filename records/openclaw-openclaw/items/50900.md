## Summary

Keep this issue open. Current main still has only global session maintenance retention plus a separate cron-run reaper; it does not implement ordered per-pattern or per-type `session.maintenance.rules`. The KeepMyClaw comment is useful backup context and #50755 is adjacent per-agent work, but neither provides the requested core session-store retention policy.

## What This Changes

The issue requests ordered per-pattern or per-type session maintenance retention rules so ephemeral cron/subagent sessions can be pruned faster than interactive sessions while falling back to the global `pruneAfter`.

## Best Possible Solution

Add an additive core `session.maintenance.rules` contract with ordered first-match retention by a documented matcher, global `pruneAfter` fallback, strict schema/generated-docs coverage, and pruning call sites that preserve active/running-session protections plus existing cron-specific behavior.
