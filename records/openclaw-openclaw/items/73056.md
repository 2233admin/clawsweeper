## Summary

Keep open. Current main still has the outer raw-session compaction pre-check that can skip before Pi SDK preparation exists, and an open implementation PR already references this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Prevent manual `/compact` from classifying over-threshold, tool-heavy sessions as idle when the limited raw message slice lacks a recent meaningful user message.

## Best Possible Solution

Land a narrow compaction fix that preserves the idle-session skip for genuinely empty or below-threshold sessions while allowing manual compaction to proceed when observed token usage is already over the configured threshold.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to build a limited session slice whose recent entries are tool results or boilerplate with the meaningful user turn more than 20 entries back, then invoke manual compaction at high token usage; current main can hit the outer skip before SDK preparation can add prefix context.

## Solution Assessment

Yes for the proposed direction, but it is not on current main. Passing reliable current token usage into manual `/compact` and bypassing only the over-threshold outer idle skip is narrower than changing the shared message classifier.
