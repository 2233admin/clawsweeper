## Summary

Keep open. Current main still has the outer raw-session compaction pre-check that can return `no real conversation messages` before Pi SDK preparation and turn-prefix context exist, and open PR #73092 already references this issue with closing syntax as the narrow implementation candidate.

## What This Changes

Manual `/compact` on long, tool-heavy Discord-thread sessions should not be skipped as idle merely because the limited raw session slice lacks a recent meaningful user message.

## Best Possible Solution

Land PR #73092, or an equivalent narrow change, so genuinely idle below-threshold sessions still skip but over-threshold manual compaction can proceed even when the limited raw tail is tool-heavy.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is clear: use a limited session slice with heartbeat/tool-result entries and the meaningful user turn outside the 20-entry lookback, then trigger manual compaction while fresh context usage is over threshold.

## Solution Assessment

Yes. Passing reliable observed token usage into manual compaction and bypassing only the outer idle skip at the compaction threshold is narrower than changing the shared classifier or removing the idle pre-check.
