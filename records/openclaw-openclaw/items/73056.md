## Summary

Keep open. Current main still has the outer raw-session compaction pre-check that can return `no real conversation messages` before Pi SDK preparation/turn-prefix context exists, and open PR #73092 already tracks the narrow fix with closing syntax, so this issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Prevent manual `/compact` from misclassifying over-threshold, tool-heavy Discord-thread sessions as idle when the limited raw message slice lacks a recent meaningful user turn.

## Best Possible Solution

Land a narrow fix that preserves the empty/idle-session skip for below-threshold sessions but allows over-threshold manual compaction to proceed even when the limited raw message slice is tool-heavy; #73092 is the active review target for that end state.

## Reproduction Assessment

Yes. A source-level reproduction is clear: feed manual compaction a limited message slice where recent entries are tool results or boilerplate and the meaningful user turn is more than 20 entries back, while context usage is over threshold; current main can hit the outer skip before SDK preparation supplies prefix context.

## Solution Assessment

Yes for the proposed direction. Passing reliable observed token usage into manual compaction and bypassing only the outer idle skip at threshold is narrower than changing the shared message classifier and keeps the original idle-cron protection intact.
