## Summary

Keep open. Current main still has the outer raw-message compaction pre-check that can return the reported idle skip before Pi SDK preparation exists, and an open linked fix PR is already tracking the narrow implementation path.

## What This Changes

Manual `/compact` should not classify an over-threshold, tool-heavy session as idle when the limited recent message slice lacks a nearby meaningful user prompt.

## Best Possible Solution

Land or close the linked implementation PR after review. The desired end state is a narrow compaction change that keeps the existing idle skip for empty sessions but lets over-threshold manual compaction proceed when the raw limited slice is tool-heavy.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: build a limited session slice where recent entries are tool results or boilerplate and the meaningful user turn is outside the 20-message lookback, then run manual compaction with high context usage; current main can hit the outer skip before SDK preparation can add prefix context.

## Solution Assessment

Yes for the linked direction. Passing observed token usage into manual `/compact` and bypassing only the outer idle skip when the session is already over threshold is narrower than changing the shared message classifier and preserves the original idle-session guard.
