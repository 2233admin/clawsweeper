## Summary

Keep open. Current main still has the outer raw-session compaction pre-check that can return `no real conversation messages` before Pi SDK preparation supplies turn-prefix context, and the provided GitHub context shows open PR #73092 is the active closing implementation candidate rather than a merged fix.

## What This Changes

Manual `/compact` should not classify a long, tool-heavy, over-threshold Discord-thread session as idle before compaction preparation runs.

## Best Possible Solution

Keep the idle-session skip for genuinely empty below-threshold sessions, but pass fresh token usage into manual `/compact` and allow over-threshold sessions to proceed past the outer pre-check; review and land PR #73092 or an equivalent narrow fix.

## Reproduction Assessment

Yes. A source-level reproduction is a limited raw message slice containing only heartbeat/tool-result entries with the meaningful user turn outside the 20-message lookback, plus an over-threshold session; current main returns the outer `no real conversation messages` skip before Pi SDK preparation can add prefix context.

## Solution Assessment

Yes. The threshold-bypass direction is the narrowest maintainable fix visible from current code because it preserves the idle-session optimization and avoids changing the shared real-conversation classifier.
