## Summary

Keep this PR open for maintainer review: current main has not implemented the Liquid Glass bottom-bar redesign, and the branch still has actionable iOS layout concerns, an unrelated provider-registry change, and a dirty merge state.

## What This Changes

The PR rebuilds the iOS home bottom toolbar as floating Liquid Glass-style status/action capsules, moves it from a bottom safe-area inset to an overlay, switches app startup to RootView, expands ScreenTab into the bottom safe area, and adds a DeepSeek provider contract registry entry.

## Best Possible Solution

Rebase and narrow the branch to the iOS bottom-bar redesign, preserve bottom content reachability and compact-layout behavior, remove the DeepSeek registry change from this PR, and coordinate with #40874 only if maintainers want one broader Liquid Glass direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Restore content avoidance for the floating toolbar:** `apps/ios/Sources/RootCanvas.swift:479`
  - body: Changing the toolbar from a bottom safe-area inset to a plain overlay removes the reserved vertical space for the control. Since ScreenWebView disables UIKit inset adjustment and the scaffold only pads for the OS safe area, bottom canvas content can render underneath the new toolbar and become hidden or hard to tap.
  - confidence: 0.88
- **[P2] Let the status label compress in compact widths:** `apps/ios/Sources/HomeToolbar.swift:90`
  - body: The added fixedSize(horizontal: true) makes the status title non-compressible while the trailing action cluster still consumes fixed width. With a longer state like Connecting plus an activity icon, compact widths or larger Dynamic Type can overflow and clip the action buttons that need to remain reachable.
  - confidence: 0.82
