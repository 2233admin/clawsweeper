## Summary

Keep open. Current main still uses the old full-width iOS home toolbar strip and bottom `safeAreaInset`, so the PR's intended Liquid Glass bottom bar is not implemented on main. The PR also needs review follow-up because the discussion raised functional layout concerns and the diff includes an unrelated provider registry change outside the iOS UI scope.

## Best Possible Solution

Keep the PR open for maintainer review. The practical path is to rebase onto current main, drop the unrelated provider registry edit, resolve the bottom-inset and compact-layout review concerns, then land the iOS Liquid Glass toolbar only if maintainers want that visual direction.
