## Summary

Keep open. Current main has an internal last-assistant-preserving thinking-block sanitizer and several narrower Anthropic/Bedrock replay fixes, but it does not provide the requested user-facing history policy or broad token-saving behavior. Newer Claude replay policy intentionally preserves prior thinking blocks for prompt-cache compatibility, so closing this would skip a real product/config decision.

## What This Changes

Request a configurable policy for handling assistant `thinking` and `redacted_thinking` blocks in replay history, with modes to keep them, strip all of them, or preserve only the latest assistant message.

## Best Possible Solution

Keep this as a focused feature request. If maintainers want the knob, add an explicit gateway or per-agent transcript-history policy, thread it through `resolveTranscriptPolicy()` and provider replay-policy merging, document the Anthropic/Bedrock prompt-cache and tool-continuation implications, and cover provider-safe defaults plus explicit overrides in tests.
