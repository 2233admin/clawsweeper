## Summary

Keep open. Current main has provider-policy replay sanitizers and a last-assistant-preserving thinking-strip helper, but it still does not expose the requested operator-facing `last-only` / `strip` / `keep` history policy. The remaining question is product/configuration scope because current Claude-family replay policy intentionally preserves thinking blocks for newer models to protect prompt-cache behavior while older/problematic replay paths are handled by provider policy.

## What This Changes

The issue requests a configurable thinking-history retention policy that can keep only the latest assistant thinking blocks, strip all thinking blocks, or preserve current behavior.

## Best Possible Solution

If maintainers accept the feature, add a documented provider-aware transcript-history retention setting that preserves current safe defaults, permits explicit operator overrides, and covers `last-only`, `strip`, and `keep` with replay-policy tests across Anthropic, Bedrock, and custom-provider paths.
