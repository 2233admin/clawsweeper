## Summary

Keep this PR open. Current main still has the exact documentation claim this PR removes, while the current iMessage runtime/schema check supports the PR’s premise: `dmHistoryLimit` is accepted by config for iMessage, but the imsg monitor path only uses group `historyLimit` and live `watch.subscribe` events, not a direct-message history replay adapter. The PR is docs/changelog-only and does not introduce a security-sensitive change.

## Best Possible Solution

Review and merge this docs-only PR if maintainers accept the documented option-2 resolution for the linked iMessage bug: keep the schema backward-compatible, stop claiming iMessage runtime support for `dmHistoryLimit`, and leave a future code fix to a separate testable iMessage history adapter if that product behavior is desired.
