## Summary

Keep this PR open. Current main still contains the exact `dmHistoryLimit` docs claim the PR removes, and the current iMessage schema/runtime evidence supports the PR premise: the field is accepted in config, but the iMessage monitor path uses group `historyLimit` and live `watch.subscribe` messages rather than a direct-message history replay adapter. The PR is docs/changelog-only and the security review found no code execution, workflow, dependency, lockfile, install, or release-script changes.

## Best Possible Solution

Review and merge this docs-only PR if maintainers accept option 2 for #73172: keep the schema backward-compatible, remove iMessage from the documented `dmHistoryLimit` supported list, and leave any future runtime support to a separate macOS-testable iMessage history adapter change.
