## Summary

Keep open. Current main still has the WebSocket reasoning-stream gap, but this PR is not merge-ready because it bypasses explicit reasoning visibility and hidden-output suppression paths that current code and docs preserve.

## What This Changes

The PR makes `emitReasoningStream` broadcast `stream: "thinking"` WebSocket agent events without requiring `streamReasoning` or `onReasoningStream`, while keeping the channel callback gated and relaxing three message-handler call-site guards.

## Best Possible Solution

Revise the fix so gateway clients can receive live `thinking` events when reasoning visibility is explicitly `stream`: forward the stored `reasoningLevel` into embedded runs, make `streamReasoning` independent of the channel callback, keep `onReasoningStream` optional, and preserve silent plus deterministic-approval suppression with focused negative tests.

## Reproduction Assessment

Yes. A static reproduction path is clear: on current main, a gateway run without `onReasoningStream` cannot set `streamReasoning`, and under the PR diff the relaxed call sites can emit `stream: "thinking"` while normal assistant output is suppressed.

## Solution Assessment

No. The PR moves the gate too far; the safer maintainable fix is to decouple WS broadcast from the channel callback while still honoring explicit reasoning visibility and existing hidden-output suppression.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate WS thinking on reasoning visibility:** `src/agents/pi-embedded-subscribe.ts:654-668`
  - body: Removing the `state.streamReasoning` return makes every non-silent call site able to publish reasoning to WebSocket clients, including sessions where `/reasoning` is off. Current docs and the runtime prompt describe reasoning as hidden unless `on` or `stream`; the narrower fix is to make `streamReasoning` depend on `reasoningMode === "stream"` without requiring the callback, then make only `onReasoningStream` optional.
  - confidence: 0.86
- **[P1] Respect approval suppression for tagged thinking:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:309`
  - body: The new tagged-thinking path runs for any non-silent update, but the same handler suppresses assistant output when deterministic approval prompts are pending or sent. If a model emits `<think>` text during that hidden approval turn, this branch can broadcast internal reasoning even though visible output is intentionally suppressed.
  - confidence: 0.82
- **[P1] Keep final reasoning behind output suppression:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:640`
  - body: Changing the message-end guard to `if (rawThinking)` lets final reasoning call `emitReasoningStream` without checking deterministic approval suppression. Reasoning emitted at message end should use the same hidden-output predicates as assistant and block-reply delivery before reaching WebSocket clients.
  - confidence: 0.8
