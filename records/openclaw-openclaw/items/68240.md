## Summary

Keep open. Current main still does not provide the requested onboard daemon-env flag, and the PR remains relevant, but the proposed patch needs maintainer/security review because it accepts arbitrary daemon environment keys after the existing daemon install filtering boundary has already run.

## What This Changes

The PR adds a repeatable `--daemon-env KEY=VALUE` onboard option, threads `daemonEnv` through onboarding options, and merges parsed entries into non-interactive and interactive gateway daemon install environments.

## Best Possible Solution

Add a reviewed daemon-env mechanism that normalizes portable keys, rejects dangerous host-env and override names through the existing policy helpers, emits clear diagnostics for malformed input, covers both onboarding install paths and CLI parsing with tests, and documents the supported flag and security limits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Validate daemon env keys before merging:** `src/commands/daemon-install-helpers.ts:337`
  - body: `parseDaemonEnvEntries()` writes any non-empty key into `extraEnv`, and both callers merge that map after `buildGatewayInstallPlan()` has already filtered dangerous service env keys. A user can pass `--daemon-env NODE_OPTIONS=...` or similar and persist values the daemon install path intentionally strips, so the new flag bypasses the host-env security boundary. Normalize keys and reject dangerous host-env/override names before assigning them.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete daemon service environment injection concern by accepting raw user-provided keys after existing host-env filtering has completed.

Concerns:

  - body: The parser accepts arbitrary keys and the install paths merge them over the filtered daemon environment, so blocked startup/injection names such as `NODE_OPTIONS` can be written into persistent service metadata despite current policy and tests excluding them.
