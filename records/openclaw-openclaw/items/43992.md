## Summary

Keep this PR open. Current main still does not implement the PR's core behavior: the gateway agent path stores a structured command result, but `agent.wait` terminal snapshots, cached/live wait responses, and subagent wait normalization still drop that result. Subagent announce/completion paths still rely on chat-history capture or stored frozen output, so they cannot use an `agent.wait` result when history has not caught up. The PR discussion is useful, including the author's follow-up on the fallback-overwrite concern, but the intended behavior is not shipped on main.

## Best Possible Solution

Keep this PR open or replace it with an updated patch against current main. The best implementation is to thread structured result through `AgentWaitTerminalSnapshot` and both cached/live `agent.wait` responses, preserve the result when lifecycle and dedupe resolution race, update `AgentWaitResult`/normalization for subagent callers, extract visible text from standard agent result shapes such as `{ payloads, meta }`, use that text only as a fallback when frozen chat-history output is absent, and add focused gateway plus subagent regression coverage. Before merge, align the gateway/plugin runtime contract, docs, and authorization review for the new result field.
