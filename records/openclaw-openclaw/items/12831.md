## Summary

Keep this issue open. Current main has useful backend observer plumbing (`tool-events` and `session.tool`) and Control UI can receive run-scoped tool events, but it still renders those events as chat-thread tool cards instead of a first-class Activity panel / observer mode. The TUI also still gates tool cards on the session verbose setting, and open draft PR #60814 is an implementation candidate rather than merged current-main behavior.

## What This Changes

Request a separate activity/tool-call observer mode so users can monitor tool calls, arguments, timing, status, and output without putting verbose details in the main chat stream.

## Best Possible Solution

Keep this issue open as the broad observer-mode tracker. The best path is maintainer-scoped product work that builds on the existing `tool-events` and `session.tool` plumbing, adds a dedicated Control UI Activity panel with independent state, scrolling, filtering, collapse/expand, and output truncation controls, then deliberately decides whether per-connection verbose settings, TUI split-pane behavior, and an external observer stream belong in the same milestone or narrower follow-ups. Treat #60814 as an open draft implementation candidate and #50762 as the narrower exec-log split.
