## Summary

Keep open. Current main has useful backend plumbing for tool-event delivery, but it still does not provide the requested first-class Activity panel / observer mode. Control UI still converts tool events into chat tool messages and renders them inside the chat thread, `session.tool` is not consumed as a dedicated observer stream, and the TUI still suppresses tool cards when verbose is off. The open draft PR #60814 remains an implementation candidate, not a merged resolution.

## Best Possible Solution

Keep this open as the broad observer-mode tracker. The best path is maintainer-scoped product work that builds on existing `tool-events` and `session.tool` plumbing, adds a dedicated Control UI Activity panel with independent state and scrolling, decides whether observer tabs need connection-scoped verbose/activity settings, and deliberately handles TUI and external stream semantics. Treat #60814 as an open draft implementation candidate and #46604/#74018 as closed related context.
