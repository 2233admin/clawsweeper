## Summary

Keep open. Current main has backend observer plumbing for tool events, but the user-facing observer mode is still incomplete: Control UI still renders live tool activity in the chat stream, there is no dedicated Activity panel state or session.tool handler, and TUI tool cards remain tied to the session verbose setting. The open draft dashboard PR should be reviewed or split rather than closing this tracker early.

## What This Changes

Request a separate activity/tool-call observer mode so users can watch agent tool activity without cluttering the main chat stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of the existing tool-events and session.tool plumbing with independent state, scrolling, filtering, collapse/expand, and truncation behavior, then split or accept follow-ups for per-connection verbose settings, a TUI split pane, and any external observer stream.
