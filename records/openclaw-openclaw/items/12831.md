## Summary

Keep this open. Current main has useful backend observer plumbing for run-scoped `tool-events` and session-scoped `session.tool`, but the user-facing request is still not implemented: Control UI still folds live tool events back into the chat thread, there is no first-class Activity panel/observer model, and the TUI still suppresses tool cards when session verbose is off. The related draft implementation PR remains unmerged, so this issue is still the broad tracker.

## What This Changes

Request to separate agent/tool activity from the main conversation through an Activity panel or observer mode, with related per-connection verbose, external stream, and TUI split-pane options.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build on the existing `tool-events` and `session.tool` plumbing with a dedicated Control UI Activity panel that has independent state, scrolling, filtering, collapse/expand, and truncation behavior, then deliberately split or accept follow-ups for per-connection verbose settings, a TUI split pane, and an external observer stream.
