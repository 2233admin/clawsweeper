## Summary

Keep open. Current main has useful backend tool-event plumbing, but the requested user-facing observer mode is still missing: Control UI live tool events are still folded into chat state/rendering, `session.tool` is exposed by the gateway but not handled by the Control UI dispatcher as a separate activity model, and TUI tool visibility remains tied to the session verbose setting. The related draft dashboard PR remains open, so this should stay as the broad tracker.

## What This Changes

The issue requests a separate activity/tool-call observer surface so users can monitor agent work without mixing verbose tool output into the main chat.

## Best Possible Solution

Keep this as the broad observer-mode tracker. The best implementation is a dedicated Control UI Activity panel built on the existing `tool-events` and `session.tool` plumbing, with independent state, scrolling, filtering, collapse/expand, truncation, and explicit follow-up boundaries for per-connection verbose settings, TUI split-pane support, and any external observer stream.
