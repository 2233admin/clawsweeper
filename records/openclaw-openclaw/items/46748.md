## Summary

Current main still defaults `openclaw logs` display to UTC unless `--local-time` is passed or the CLI process has a valid `TZ` environment variable. The issue also has open fix PRs (#46782 and #47365) with closing syntax, so it should stay open until one of those is reviewed/landed or explicitly closed.

## Best Possible Solution

Keep this issue open as the canonical tracker, review and consolidate #46782 and #47365, then land one small fix that makes human `openclaw logs` output default to local time while preserving explicit UTC output, with tests for default local display, UTC opt-out, `--follow`, and docs updated to match.
