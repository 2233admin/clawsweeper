## Summary

Keep open. Current `main` still persists the post-increment compaction count as the memory-flush marker, the gate still skips when the two counters are equal, the latest shipped tag still contains that path, and open PR #51421 already targets this exact issue with closing syntax.

## What This Changes

The issue reports that configured `memoryFlush` skips eligible auto-compaction cycles because the runner records the post-increment compaction count as the last flush marker.

## Best Possible Solution

Land #51421 or an equivalent focused patch that preserves the pre-increment flush marker after flush-triggered compaction, updates the synchronized-counter test expectation, adds consecutive eligible auto-compaction coverage, and preserves the documented once-per-cycle behavior.
