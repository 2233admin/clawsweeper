## Summary

Keep open. Current main still has the reported counter synchronization path, the latest shipped release still contains the same assignment, and open PR #51421 already targets this issue with a focused fix path.

## What This Changes

The issue reports that `memoryFlush` can skip alternating eligible auto-compaction cycles because `memoryFlushCompactionCount` is persisted with the post-increment `compactionCount`.

## Best Possible Solution

Land #51421 or an equivalent focused patch that preserves the pre-increment flush marker after flush-triggered compaction, updates the synchronized-counter test expectation, and adds consecutive eligible auto-compaction coverage while keeping the documented once-per-cycle behavior.
