## Summary

Keep open. Current main does not implement the requested broad filter for mixed assistant/tool-call messages; it only suppresses the `sessions_yield` wait-turn case. Non-`sessions_yield` assistant messages that also contain a tool call are still promoted as completion output, and the current tests explicitly assert that behavior. The directly linked fix PR was closed unmerged, so it is not proof of implementation on main.

## Best Possible Solution

Keep this issue open and decide the intended completion contract. If the report is accepted, implement the filter in `src/agents/subagent-announce-output.ts`: when an assistant message contains tool-call blocks, do not promote its plain text to `latestAssistantText` or frozen completion output, except for explicit final-answer / `<final>...</final>` content. Keep timeout/error partial-progress handling separate, then update `src/agents/subagent-announce-output.test.ts` to cover ignored mixed turns, preserved explicit final content, and unchanged assistant-only completions.
