## Summary

Keep this PR open. Current main still lacks the imsg RPC tapback metadata parsing, tapback filtering, and debounce-safe mixed-batch handling that this PR targets; the paired bug #60274 remains open, and the PR context shows an active maintainer repair path rather than a superseding landed fix.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or explicitly supersede it with an equivalent narrow main-branch patch. The fix should parse imsg v0.5 tapback metadata, filter tapbacks before pairing/access-control side effects and agent dispatch, preserve normal text when tapbacks arrive in a debounced burst, and keep full reaction-event support tracked separately under #39031.
