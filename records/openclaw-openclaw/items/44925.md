## Summary

Keep open. Current main has several subagent announce hardening paths, but it still contains an explicit retry-limit/expiry give-up path, tests that accept failed delivery after retries, and no merged last-resort user notification, auto-restart, or file-claim verification path covering the full report. The latest issue comment also says the retry-limit failure still reproduces on v2026.4.26, and the related open PR only targets Pattern 1.

## What This Changes

The issue asks OpenClaw to stop silently losing subagent completions by retrying or surfacing announce failures, notifying or restarting on timeout, and verifying claimed file creation before completion is reported.

## Best Possible Solution

Keep this open for maintainer design/review, then land a coherent async-completion delivery model that distinguishes internal completion, parent-session wake, external channel delivery, and confirmed user-visible notification. A reworked #45200 may be useful for Pattern 1, but the broader issue should either be split into focused follow-ups or resolved by a shared subagent/task delivery contract with regression coverage.
