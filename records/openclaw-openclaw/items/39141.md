## Summary

Keep #39141 open. Current main still lacks the prerequisite per-session activity/attention gateway API from #39127 and does not implement the requested opt-in gateway-side session activity watchdog, alert dedupe/cooldowns, operator delivery, or optional tokenized summaries. The existing channel health monitor, cron failure alerts, CLI idle watchdogs, and subagent orphan recovery are adjacent safeguards, not substitutes for a session-scoped activity watchdog.

## Best Possible Solution

Keep #39141 open until #39127 or an equivalent session-scoped activity/attention API with WebSocket state-change events lands. Then implement an opt-in gateway watchdog or plugin-accessible watchdog surface that consumes that state, supports no-token structured alerts, configurable thresholds, dedupe/cooldowns, scoped operator/channel/webhook delivery, optional rate-limited tokenized summaries, docs, and focused tests. Keep #39305 as the narrower sub-agent stall recovery track rather than treating it as a replacement for this broader operator watchdog.
