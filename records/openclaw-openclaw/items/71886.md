## Summary

Keep open for normal maintainer review. Current main still has the observable CSP gap for the browser-side tweakcn importer, and the PR is a narrow fix with no blocking code-review finding; the only remaining merge concern is that the PR head is currently marked unstable by CI failures outside the touched files.

## What This Changes

This PR adds `https://tweakcn.com` to the Control UI `connect-src` CSP directive and asserts that Gateway HTTP responses include the updated directive.

## Best Possible Solution

Land the narrow CSP allowlist and regression assertion after maintainer review and a clean or explained CI state; keep the broader configurable CSP/Permissions-Policy design in #47565 as separate product work.

## Reproduction Assessment

Yes. Current main can be checked statically: the importer fetches `https://tweakcn.com/r/themes/<id>` from the browser while the served Control UI CSP omits that origin from `connect-src`, so a valid tweakcn import is blocked by browser CSP.

## Solution Assessment

Yes. Adding a single HTTPS origin to `connect-src` is the narrowest maintainable fix for the bundled tweakcn importer and does not grant script execution, image wildcards, or subdomain access.
