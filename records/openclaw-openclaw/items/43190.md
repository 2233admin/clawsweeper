## Summary

Keep this PR open. Current main still lacks the archive plugin, the `channel_deleted` hook contract, Teams `channelDeleted` dispatch, and the proposed typed inbound metadata propagation; the PR is also broad and security-sensitive enough that it needs maintainer review rather than automated close or repair routing.

## What This Changes

This PR adds a bundled `msteams-channel-archive` plugin, persistent Teams message and attachment archiving/query tools, deleted-channel cleanup via a new `channel_deleted` hook, Teams inbound metadata propagation, tests, changelog/labeler updates, SDK baseline changes, and package/lockfile metadata.

## Best Possible Solution

Land only a reviewed, additive hook/API and Teams lifecycle dispatch if maintainers accept the seam; then either keep the archive implementation as a deliberately bundled Teams plugin with security-reviewed persistence and cleanup semantics, or move it to an external ClawHub/npm plugin using the stable SDK surface.
