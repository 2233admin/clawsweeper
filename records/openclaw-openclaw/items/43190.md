## Summary

Keep this PR open. Current main does not include the proposed Teams channel archive plugin, the `channel_deleted` hook, or Teams deleted-channel dispatch/cleanup behavior. The PR is a real implementation candidate, but it still needs maintainer review because it adds a new bundled plugin, expands plugin hook contracts, and has security-sensitive Graph cleanup behavior discussed in review comments.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a rebase and split. The smallest product path is a focused core/Teams API change for a typed deleted-channel lifecycle event and rich, documented message metadata, with tests and docs; the durable archive implementation should then either live as a reviewed bundled plugin with clear ownership or be published externally once the API seam exists.
