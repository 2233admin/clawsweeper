## Summary

Keep this PR open. Current main still has both Control UI behaviors the PR is trying to repair: a deferred same-session `session.message` reload is cleared but not replayed after a locally appendable final assistant event, and a role-less final payload is still treated as renderable assistant content for reload suppression. The branch also has active maintainer repair context and a narrow UI-only file list, so no conservative close reason applies.

## Best Possible Solution

Keep this PR in review and land it, or land an equivalent patch, so Control UI replays deferred same-session `session.message` reloads after locally appendable final assistant events and only suppresses final-event history reloads for explicit renderable assistant messages. Preserve the regression coverage and changelog entry, then close the relevant chat reload race reports after merge.
