## Summary

Keep this PR open. Current main still has the two behaviors this PR branch repairs: deferred same-session `session.message` reloads are cleared but not replayed after a final assistant payload, and role-less final payloads with visible text still suppress the final history reload.

## Best Possible Solution

Keep this PR in review and land it, or land an equivalent patch, so Control UI replays deferred same-session `session.message` reloads after locally appendable final events and only suppresses final-event history reloads for explicit renderable assistant messages. Preserve the regression coverage and contributor changelog credit, then close related chat reload race reports after the fix reaches main.
