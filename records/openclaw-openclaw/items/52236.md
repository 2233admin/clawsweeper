## Summary

Current main already solves the useful part of this Mattermost thread-routing PR through the shipped generic session delivery path: inbound thread sessions persist route/thread metadata during session initialization, system-event replies recover the thread id from the thread-scoped session key, and Mattermost outbound delivery maps that thread id back to `replyToId`. The submitted monitor-only patch is also known incomplete from the PR discussion, and the author later marked it abandoned.

## Best Possible Solution

Close this abandoned monitor-only PR as already implemented by the shipped generic session/thread routing path, and keep any separate heartbeat-only Mattermost concern tracked in a focused issue or PR if it still reproduces on current releases.
