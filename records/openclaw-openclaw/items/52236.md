## Summary

Keep open. Current main still has the Mattermost `updateLastRoute` write gated to direct messages and does not store `deliveryContext.threadId` there, so this PR is not implemented on main. The PR is also explicitly incomplete for heartbeat routing per the review discussion and author follow-up, so it should not be auto-closed as shipped or merged as-is.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up rather than closing it as implemented. The best path is a complete Mattermost thread-routing fix that persists the right delivery context for channel/group thread sessions, addresses the heartbeat and cron session-target gaps called out in the PR discussion, adds focused Mattermost monitor/outbound tests, and coordinates with the still-open #45082 and #52120 work. If maintainers decide Mattermost is no longer in core scope, close it manually with that product decision instead of treating it as already fixed.
