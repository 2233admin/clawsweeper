## Summary

Keep open. Current main still blocks skill-declared secret-like env names in Docker sandbox creation, so the PR is not obsolete. The PR also is not ready to close/merge because the latest supplied review context still has an unresolved P1 security concern around syncing env overrides from raw, not eligibility-filtered, skill entries into sandbox exec env.

## Best Possible Solution

Keep this PR open and request a revision before merge: constrain sandbox create allowlists and sandbox exec env sync to the same eligible/active skill set, or deliberately define a narrower configured-override set; update the sandboxed skills docs if `skills.entries.*.env/apiKey` will now enter sandbox execs; then rerun targeted sanitizer, sandbox create args, skills runtime, and changed-lane validation.
