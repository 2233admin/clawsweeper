## Summary

Keep open. Current main does not implement this PR's intended sandbox skill env allowlist/sync: secret-like keys in sandbox Docker env still get blocked, skill-configured env/apiKey remains host-only for sandboxed sessions, and the PR still has an unresolved security-sensitive review concern around disabled/ineligible skill env sync.

## Best Possible Solution

Keep this PR open or supersede it with a maintained equivalent. The best fix is to compute the active/eligible skill set once for the run, use that same set for prompt building, host skill env overrides, sandbox-create env rescue, and sandbox exec env sync, then add regression coverage for declared API keys, disabled/ineligible skills, always-blocked keys, null-byte values, snapshot and non-snapshot paths, plus a Docker sandbox proof when feasible.
