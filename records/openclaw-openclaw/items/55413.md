## Summary

Keep this PR open. Current main still does not canonicalize Discord PluralKit webhook proxy copies to the PluralKit original ID, so the branch is not obsolete; however the proposed patch remains incorrect because its lookup block runs after an early bot-drop path, which can discard PK traffic before `sender.isPluralKit` is known.

## What This Changes

The PR changes Discord inbound handling to look up PluralKit data for webhook-authored messages, carry a canonical PK original ID through preflight, use it as `MessageSid`, preserve the provider Discord ID as `MessageSidFull`, and add regression tests.

## Best Possible Solution

Land a rebased Discord fix that suppresses known OpenClaw bound-thread webhook echoes before external PK calls, performs PK lookup for every other PK-enabled inbound message before bot and mention gates, threads a trimmed `pluralkitInfo.original` through preflight into the current context builder, and verifies both canonical `MessageSid` and provider `MessageSidFull` behavior with regression coverage.
