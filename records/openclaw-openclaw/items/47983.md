## Summary

Keep open. Current main has the hidden file input, selected-file handling, and downstream attachment serialization, but the visible Attach file control still uses an unscoped document-wide lookup and there is no regression test for the reported button-to-picker activation path. The merged non-image attachment work in #72299 covers payload handling after selection, while the adjacent open PRs #72612 and #74082 do not explicitly close this picker activation report.

## What This Changes

The issue reports that the WebChat/Control UI upload button in OpenClaw 2026.3.13 does nothing when clicked instead of opening a file picker and attaching the selected file.

## Best Possible Solution

Keep the existing selected-file and Gateway attachment pipeline, but make the chat composer Attach file control activate its own file input through a component-scoped ref or label-backed input, add an explicit type="button", and cover the visible button-to-input click route in the chat view tests.
