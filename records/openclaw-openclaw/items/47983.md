## Summary

Keep open. Current main still has the hidden attachment input and post-selection attachment pipeline, but the visible Control UI/WebChat Attach file button is still wired through an unscoped document-wide lookup and the tests do not cover the reported visible-button-to-file-picker activation path. The merged non-image attachment work in #72299 helps after selection; the adjacent open #72612 and #74082 do not explicitly resolve this picker activation bug.

## What This Changes

The issue asks for the WebChat/Control UI upload button to open the browser file picker and allow selected files to be sent as chat attachments.

## Best Possible Solution

Keep the current attachment serialization and Gateway media path intact, but make the chat composer Attach file control activate its own file input through a Lit ref/createRef or label-backed input, add an explicit button type, and add a regression test that clicks the visible control and verifies the hidden input activation path.
