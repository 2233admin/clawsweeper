## Summary

Current main has a WebChat attachment picker, non-video file acceptance, preview handling, and tests for selecting a PDF attachment. However, the exact reported symptom is that the Chrome/Windows paperclip click did not open the native file picker, and the current button still uses the same hidden-input `document.querySelector(...).click()` trigger pattern that existed in the reported 2026.3.13-era code. This should stay open for a current-version retest or targeted browser proof before closing.

## Best Possible Solution

Keep this open and ask for a retest on current OpenClaw, ideally on Chrome/Windows. If the picker still does not open, make a scoped Control UI fix in the chat composer: wire the paperclip button to a component-scoped file input ref or accessible label-backed input instead of a global `document.querySelector`, ensure the button is `type="button"`, preserve current non-video attachment support, and add focused unit/browser coverage for the button-to-input trigger plus existing selection behavior.
