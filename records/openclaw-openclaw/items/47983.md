## Summary

Keep open because current main still leaves the reported visible upload-button path fragile: the attachment pipeline supports selected files, but the paperclip button still depends on a document-wide hidden-input click and lacks direct coverage for the dead-click route.

## What This Changes

The issue reports that the WebChat upload button appears in OpenClaw 2026.3.13 on Chrome/Windows but clicking it does not open the file picker.

## Best Possible Solution

Keep the shipped attachment pipeline, but make the chat composer attach control target its own file input through a component-scoped ref or label-backed input, add an explicit button type, and cover the visible button-to-input click path in the chat view tests.
