## Summary

Keep open. Current main has already picked up the gateway-bindable/default registry compatibility behavior, but it does not implement the PR's other two central plugin fixes: `registerTool` still appends registrations without a duplicate guard, and plugin tool factories still do not receive `plugins.entries.<id>.config` through their tool context.

## Best Possible Solution

Keep the PR open, but ask for a refreshed, focused version: preserve the already-shipped gateway-bindable compatibility behavior from main, add the exact-match tool registration dedupe with regression coverage, thread validated per-plugin config into `PluginToolRegistration` and `OpenClawPluginToolContext`, and split unrelated live-test HOME/profile changes into a separate review if they are still needed.
