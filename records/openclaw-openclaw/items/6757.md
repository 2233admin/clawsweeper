## Summary

Current main still lacks the requested in-session agent-triggered self-compaction surface. `session_status` remains status/model-only, the agent-facing `gateway` tool does not expose `sessions.compact`, and the existing RPC compaction path is admin/control-plane scoped rather than an in-run self-compaction tool. PR #43401 remains useful prior art, but it was closed unmerged.

## Best Possible Solution

Keep this issue open until OpenClaw adds a first-class in-session self-compaction action, preferably `session_status(action="compact")` or an equivalent agent tool. The implementation should use direct compaction for the current active run, preserve session visibility and permission semantics, mark the call as mutating, update compaction counters/token snapshots, and document the new agent-facing behavior. PR #43401 is useful prior art but is not shipped.
