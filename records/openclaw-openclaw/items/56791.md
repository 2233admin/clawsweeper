## Summary

Keep this PR open. Current main still does not implement the PR’s intended Signal quote metadata forwarding: Signal inbound context lacks ReplyToId, the Signal quote type still does not model signal-cli quote.id or quote.authorNumber, and current quote sender resolution still treats quote.author as a phone-number source. The provided PR context shows a focused Signal plugin fix with regression tests and no obvious security-sensitive supply-chain or CI changes, and a maintainer comment says the repaired branch is the canonical contributor path.

## Best Possible Solution

Review and land this focused Signal plugin PR, or make an equivalent main-branch change that forwards signal-cli quote id/body/sender metadata into ReplyTo* context, models quote.id and quote.authorNumber, resolves authorNumber versus UUID correctly, preserves visibility gating for hidden quote metadata, clears stale reply metadata across debounced merges, and keeps the added regression tests and changelog entry.
