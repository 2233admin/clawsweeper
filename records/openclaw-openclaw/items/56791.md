## Summary

Keep open. Current main at db40ec404a9178294dba3121758ff59459983a11 still does not implement the PR’s intended Signal quote metadata forwarding: Signal inbound context has no ReplyToId path, the Signal quote type omits id/authorNumber, and quote author resolution still treats quote.author as sourceNumber. The PR remains a focused implementation candidate, and the latest diff addresses the earlier review concerns around authorNumber precedence, visibility gating, debounced merges, and tests.

## Best Possible Solution

Review and land this focused Signal plugin PR, or land an equivalent main-branch change that forwards visible Signal quote id/body/sender metadata into ReplyTo* context, prefers authorNumber over UUID-like author values, gates hidden quote metadata by context visibility, clears stale reply metadata across debounced merges, and keeps the added regression tests plus changelog entry. Close only after that implementation is on main.
