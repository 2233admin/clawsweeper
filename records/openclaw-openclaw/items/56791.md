## Summary

Keep this PR open. Current main still does not implement the intended Signal quote metadata forwarding: `ReplyToId` is not forwarded, the Signal quote type still omits `id` and `authorNumber`, and current quote sender resolution still treats `quote.author` as a phone-number source. The live PR diff is focused on Signal monitor code/tests plus a changelog entry, and the earlier review concerns appear addressed on the PR branch rather than on main.

## Best Possible Solution

Review and land this focused Signal plugin PR, or make an equivalent main-branch change that forwards visible signal-cli quote ID/body/sender metadata into `ReplyTo*` context, preserves the context-visibility boundary, clears stale metadata across debounced merges, and keeps the added regression tests and changelog entry.
