## Summary

Keep this PR open. Current main still does not implement the intended Signal quoted-reply metadata forwarding: Signal does not pass ReplyToId into inbound context, its quote type omits signal-cli id/authorNumber, and sender resolution still treats quote.author as the phone-number source instead of using authorNumber/UUID-aware routing. The provided PR discussion remains relevant and the later branch repair appears aimed at the real review concerns, but equivalent code is not on main.

## Best Possible Solution

Keep this PR open and either review/land it or apply an equivalent Signal plugin change on main: type signal-cli quote id/authorNumber, route authorNumber before UUID fallback, forward ReplyToId/ReplyToBody/ReplyToSender/ReplyToIsQuote through finalizeInboundContext, clear stale reply metadata on debounced merges, and keep focused regression tests plus contributor changelog credit.
