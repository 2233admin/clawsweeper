## Summary

Keep open. Current main still lacks a first-class ephemeral or tmpfs-backed sandbox workspace lifecycle, and PR #73738 is open, unmerged, and explicitly proposes the missing lifecycle behavior with closing syntax, so this issue should remain tracked until that implementation is reviewed and landed or replaced.

## What This Changes

The issue requests an opt-in ephemeral or tmpfs-backed workspace mode for isolated sessions so file changes are discarded between runs.

## Best Possible Solution

Land a secops-reviewed opt-in sandbox workspace lifecycle with persistent behavior as the default, an ephemeral mode using run-owned workspace/runtime paths, guarded cleanup on success and setup failure, aligned config schema/generated metadata/docs/tests, and explicit docs on whether tmpfs-backed workspaces are supported or deferred.
