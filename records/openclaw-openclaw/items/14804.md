## Summary

Keep open. Current main still does not provide a first-class ephemeral or tmpfs-backed sandbox workspace lifecycle, and open PR #73738 is the linked implementation candidate with closing syntax, so this issue should remain open for maintainer/secops review rather than be closed or routed to a separate repair lane.

## What This Changes

The issue requests an opt-in ephemeral or tmpfs-backed workspace mode for isolated sandbox sessions so each run starts from a clean disposable filesystem state.

## Best Possible Solution

Land a reviewed, opt-in sandbox workspace lifecycle with persistent behavior as the default and an ephemeral mode that uses run-owned workspace/runtime paths, guarded cleanup, config/schema metadata, docs, and targeted coverage for sandbox context resolution, config validation, and failure cleanup.
