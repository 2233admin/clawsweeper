## Summary

Keep this open. Current main has the intended canonical Anthropic-plus-claude-cli runtime path and a regression test for it, but that does not close this fresh 2026.4.26 cold-start report because the failing packaged/hot-reload/session/auth path is not isolated, several adjacent CLI-backend skip/auth reports are still open, and no shipped/current fix for the exact observed OpenRouter direct route can be identified.

## What This Changes

The issue reports that a fresh 2026.4.26 install with `agents.defaults.agentRuntime.id` set to `claude-cli` ignores the CLI runtime and routes Telegram turns directly to OpenRouter without fallback diagnostics.

## Best Possible Solution

Add a focused triage/reproduction that distinguishes invalid flat config, last-known-good config recovery, stale session model overrides, bundled plugin activation, and auth-profile hydration; if the regression reproduces, make explicit `claude-cli` runtime selection fail closed or emit trajectory-visible runtime-skip diagnostics before any fallback/provider substitution is allowed.
