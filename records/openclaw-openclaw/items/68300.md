## Summary

Keep open. Current main still has message-only internal hook error logging, so this PR is not obsolete or implemented elsewhere. The related root-cause PR was closed unmerged, so maintainers should review this as a narrow diagnostics improvement while deciding whether the linked bootstrap TypeError still needs a separate fix.

## What This Changes

This PR updates `triggerInternalHook` so hook handler failures log the stack trace and structured `{ type, action, sessionKey }` event context alongside the existing error message.

## Best Possible Solution

Internal hook failures should emit enough non-sensitive structured detail to identify the failing handler, with a focused regression test for stack and event metadata. The bootstrap TypeError should remain separately trackable until either the new logs identify it or a defensive root-cause fix lands.
