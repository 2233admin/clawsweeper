## Summary

Keep open. Current main still matches the reported failure class: the managed web_search path does not propagate the agent tool AbortSignal into the web-search provider contract, Gemini receives only the configured timeout, and provider-raised AbortErrors still bypass the normal tool failure logging path.

## What This Changes

Fix Gemini-backed web_search so intermittent provider aborts are handled and logged correctly instead of returning a silent instantaneous AbortError.

## Best Possible Solution

Add a backwards-compatible web-search execution context that carries AbortSignal and abort diagnostics through the managed tool wrapper, runtime types, provider SDK helper, guarded fetch path, and Gemini implementation; then log/classify unexpected provider aborts while preserving true user/session cancellation as cancellation.

## Reproduction Assessment

Unclear for the live intermittent abort because no live Gemini credential was available and the report is timing-dependent. The source-level failure path is reproducible by having a selected provider throw an AbortError, which current main rethrows before normal tool failure logging.

## Solution Assessment

Not applicable as a proposed patch. The narrow maintainable direction is an additive execution-context seam plus cancellation/logging tests, rather than a Gemini-only timeout tweak.
