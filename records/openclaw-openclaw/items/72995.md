## Summary

Keep open. Current main still matches the report: the managed web_search path drops the tool AbortSignal before it reaches the web-search runtime or Gemini provider, and AbortError-shaped failures are rethrown before normal tool error logging. The issue is recent, concrete, and the older #72986 context appears to be duplicate background rather than a shipped fix.

## What This Changes

Investigate and fix intermittent near-instant AbortError failures from Gemini-backed web_search, including the missing gateway.err.log diagnostics.

## Best Possible Solution

Add a backwards-compatible web-search execution context that carries AbortSignal and abort diagnostics through the managed tool, runtime, provider helper layer, and Gemini implementation; classify/log unexpected AbortError failures safely, and cover the path with focused unit tests plus optional Gemini live proof.
