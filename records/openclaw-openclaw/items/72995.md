## Summary

Keep this issue open. The report is recent and actionable, #72986 is closed rather than a canonical merged fix, and current main still lacks a source-level fix or regression test for instantaneous Gemini web_search aborts and missing diagnostics.

## Best Possible Solution

Keep this issue open and investigate the Gemini provider-web-search path. The likely maintainer fix is to add a backwards-compatible AbortSignal/execution-context seam for web_search providers, forward the tool/run signal from `createWebSearchTool` through `runWebSearch`, pass it into trusted web-search fetches and Gemini citation resolution where applicable, explicitly classify/log AbortError failures, and add focused tests for instantaneous Gemini abort behavior plus timeoutSeconds handling.
