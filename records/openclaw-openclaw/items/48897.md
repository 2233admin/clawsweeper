## Summary

Keep open. Current main still pins `@mariozechner/pi-tui` to `0.70.5`, OpenClaw only adds a local duplicate-backspace input listener before delegating editor input to pi-tui, and the exact locked pi-tui printable CSI-u decoder still rejects Alt/Ctrl-modified printable sequences. That leaves the reported German AltGr `@`/`€` path unresolved, with no open PR found that references this issue with closing syntax.

## Best Possible Solution

Keep this open and fix the TUI input layer with a narrow regression-tested change. Best path is either an upstream pi-tui fix plus verified dependency bump, or a small OpenClaw input-normalization adapter before `CustomEditor` that decodes AltGr-produced printable CSI-u sequences as text while preserving existing Alt/Ctrl shortcut behavior. Add focused tests for German-layout `@`/`€` and duplicated printable Kitty events, and avoid dependency patches or overrides without maintainer approval.
