## Summary

Keep this PR open. Current main still throws `Failed to optimize image` when all JPEG optimization attempts fail, and this PR is a recent same-author implementation candidate linked with closing syntax to the still-open bug report #73424. The earlier ClawSweeper comment remains accurate against current main SHA 1fcf0a422f9b53c42dffc8defe46852b799391f3.

## Best Possible Solution

Keep this PR open as the implementation candidate for #73424. The best path is to refine the fallback so it keeps the max-byte clamp behavior, handles the returned MIME type correctly, and adds focused regression coverage for an optimizer failure that should still let a valid image reach the VLM.
