## Summary

Keep this PR open. Current main still throws `Failed to optimize image` when every JPEG resize/quality attempt fails, and this PR remains a same-author implementation candidate for the still-open bug report #73424. The PR needs refinement for MIME preservation and focused regression coverage before it can safely land.

## Best Possible Solution

Keep this PR open as the implementation candidate for #73424. The best path is to refine the fallback so it keeps the existing max-byte clamp, returns the correct MIME/content type when passing through original bytes, and adds a focused regression test that simulates optimizer failure while allowing a valid under-cap image to reach the image model.

That is the behavior this PR is trying to change.
