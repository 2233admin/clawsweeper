## Summary

Keep open. Current main still has the reported Slack directory gap: the directory CLI list path calls only config-backed directory methods, Slack exposes token-backed live directory readers separately, and Slack still falls back to the default null self resolver. The related Slack user-token sourcing work in #28103 is adjacent, but it did not connect these CLI commands to the live readers or add Slack current-account identity resolution.

## What This Changes

The issue asks for Slack `openclaw directory self`, `peers list`, and `groups list` to return token-backed Slack workspace identity, peers, and groups instead of `null` or empty arrays when Slack auth is valid.

## Best Possible Solution

Add a generic live-aware directory CLI list path that prefers live list methods when present and preserves config-backed fallback, then add Slack-owned current-account identity support using the existing directory entry contract and resolved Slack token model.

## Repair Work Prompt

Fix the Slack directory bug reported in https://github.com/openclaw/openclaw/issues/50776. Current main routes `openclaw directory peers list` and `openclaw directory groups list` through config-backed `plugin.directory.listPeers/listGroups`, while Slack exposes token-backed live readers as `listPeersLive/listGroupsLive`; `openclaw directory self` calls `plugin.directory.self`, but Slack inherits the default null resolver.
