## Summary

Keep open. Current main still matches the reported Slack directory failure path: the public directory CLI uses only config-backed Slack peer/group methods, Slack exposes token-backed live directory readers separately, and Slack does not override the default null self resolver. The issue is concrete, recent, non-maintainer-authored, and narrow enough for a focused repair PR.

## What This Changes

Slack directory commands should return the connected Slack identity, peers, and groups when Slack is connected with a valid read-capable user token instead of returning null or empty lists.

## Best Possible Solution

Implement a live-aware directory CLI path that uses provider live directory methods when available while preserving config-backed fallback, then add Slack-owned self identity support that returns the existing directory entry shape for the resolved current account.
