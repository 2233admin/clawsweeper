## Summary

Keep open. Current main still has the reported Slack directory gap: the public directory CLI reads only the config-backed peer/group methods, Slack exposes token-backed live readers separately, and Slack still inherits the default null self resolver. The earlier ClawSweeper comments remain accurate against current main; merged PR #28103 fixed adjacent Slack user-token sourcing, but it did not connect these CLI commands to Slack live directory reads or implement Slack current-account identity.

## What This Changes

The issue requests that Slack `openclaw directory` CLI reads return workspace peers, groups, and the current user when Slack is connected with valid bot/app/user tokens.

## Best Possible Solution

Make `openclaw directory peers/groups list` use live directory methods when a channel adapter provides them, with config-backed fallback preserved, and add Slack-owned token-backed `self` resolution that returns the existing `ChannelDirectoryEntry` shape.
