## Summary

Keep open. Current main and the latest release still leave the Control UI refresh button enabled during active send/run/stream states, and open maintainer PR #74690 is already the focused implementation candidate for this issue.

## What This Changes

The issue asks for `Refresh chat data` in the Control UI to be disabled, ignored, or made safe while an agent turn is active so it cannot stall the run or wedge the UI/gateway.

## Best Possible Solution

Ship the narrow Control UI disabled-state guard and browser render coverage from #74690, then let this issue close through that merge; keep the broader active-run responsiveness and media-mirroring work under #73836.
