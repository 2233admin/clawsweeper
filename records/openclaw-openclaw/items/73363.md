Current main still does not implement this PR’s intended fix: embedded reasoning payloads are marked internally but the returned payload object drops `isReasoning`, and the cron summary/output helpers still skip only `isError`. The PR is a focused candidate for the open cron announce reasoning leak linked from #73186, not obsolete maintainer-cleanup material.

## Best Possible Solution

Keep this PR open and review it as a focused implementation candidate for the #73186 cron announce reasoning leak. The best path is to land or request small adjustments after normal validation, while tracking any remaining duplicate-delivery behavior separately if current main still reproduces it.
