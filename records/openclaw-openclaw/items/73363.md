## Summary

Keep open. Current main still has the reported cron announce reasoning leak: the embedded runner creates a reasoning payload marker but drops it from returned payloads, and cron isolated-agent summary/output/delivery pickers still skip only error payloads. The latest PR head is an active focused implementation candidate after the Feishu deliverable-picker fix-up, but it needs maintainer review because related #40534 proposes a stricter policy for reasoning-only fallbacks and additional paths.

## What This Changes

The PR propagates embedded-runner `isReasoning` payloads, updates cron isolated-agent summary/output/deliverable selection helpers to skip reasoning blocks with fallback behavior, adds focused tests, and adds an Unreleased changelog entry.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to reconcile it with #40534, then either land this focused patch if the reasoning-only fallback is acceptable or ask for a small adjustment/replacement that applies the stricter policy consistently across summary, output text, deliverable payloads, first-text fallback, and error-recovery detection.
