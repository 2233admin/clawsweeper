## Summary

Keep open. Current main still has the reported cron announce reasoning-leak path, and this PR is an active focused fix, but the latest diff still allows reasoning-only payloads to be selected by the fallback path, so it needs maintainer review or a small adjustment before merge.

## What This Changes

The PR adds `isReasoning` propagation to embedded-runner payloads, teaches cron payload pickers to skip reasoning in their primary selection pass, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a version that preserves the `isReasoning` marker and suppresses reasoning payloads across cron summary, output, deliverable, reasoning-only, and firstText fallback paths, while preserving genuine error fallback behavior. If maintainers prefer the stricter end-to-end policy from #40534, this PR should either adopt that scope or be superseded by the broader fix.

## Reproduction Assessment

Yes. A helper-level reproduction is high confidence: pass a single `{ text: "thinking", isReasoning: true }` payload through the proposed picker behavior and it is selected for summary/output/delivery fallback, which then reaches direct cron delivery for channels without a downstream filter.

## Solution Assessment

No. Propagating `isReasoning` and filtering it in the primary pass is the right direction, but returning reasoning text in fallback is not the narrowest safe fix for a user-visible thinking leak; the safer alternative is to suppress reasoning from all cron announce fallback delivery and rely on final visible text, structured error text, or no delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

- **[P2] Suppress reasoning-only cron fallback payloads:** `src/cron/isolated-agent/helpers.ts:144`

For Feishu/direct cron announce there is no downstream `isReasoning` filter, and `dispatchCronDelivery()` sends the selected payloads directly, so a reasoning-only model turn still exposes internal thinking text in the same delivery path this PR is meant to fix. Keep reasoning payloads out of fallback delivery and use final visible text, error text, or no delivery instead.
