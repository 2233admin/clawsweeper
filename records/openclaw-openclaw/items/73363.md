## Summary

Keep this PR open. Current main still has the reported cron reasoning-leak path: the embedded runner creates an internal reasoning payload but drops `isReasoning` before returning payloads, and the cron summary helpers still select text using only `isError`. The PR is therefore not obsolete; it remains an active implementation candidate for the linked bug, with a maintainer decision needed on whether to keep it focused or fold in broader coverage from #40534.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this focused fix or a maintainer-equivalent patch that propagates `isReasoning` from `buildEmbeddedRunPayloads()` and makes cron summary selection prefer non-reasoning text, while deciding whether the broader #40534 end-to-end filtering belongs in the same change or a follow-up for #73186.
