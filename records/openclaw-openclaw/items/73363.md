## Summary

Keep this PR open. Current main still drops the `isReasoning` marker after creating reasoning payloads, and the cron summary helpers still only skip `isError`, so the PR is not obsolete. It remains a plausible implementation candidate for #73186, with a maintainer review point around whether to fold in the broader delivery/fallback coverage from #40534.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this focused fix or a maintainer-equivalent patch that propagates `isReasoning` from `buildEmbeddedRunPayloads()` and teaches cron summary selection to prefer non-reasoning text, while explicitly deciding whether the broader #40534 coverage belongs in this PR or a follow-up for #73186.
