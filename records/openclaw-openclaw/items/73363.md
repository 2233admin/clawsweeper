## Summary

Keep this PR open. Current main still has the bug this PR targets: `buildEmbeddedRunPayloads()` creates reasoning payloads with `isReasoning: true`, but the final returned payload object still drops that marker, and the cron isolated-agent summary/delivery helpers still filter only `isError`. The PR remains a focused implementation candidate for #73186, though maintainers should compare its narrower scope with the broader open #40534 approach.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this focused fix or a maintainer-equivalent patch that propagates `isReasoning` from `buildEmbeddedRunPayloads()` and makes cron announce selection prefer non-reasoning text, while deciding whether the broader #40534 behavior should be folded in or handled as a separate follow-up for #73186.
