## Summary

Keep this PR open. Current main still does not implement the PR's central fix: `buildEmbeddedRunPayloads()` creates reasoning payloads internally, but the returned payload object drops `isReasoning`, and the cron isolated-agent helper pickers still only skip `isError`. The PR remains an active implementation candidate for the linked Matrix/Feishu cron announce reasoning leak, though maintainers should review whether the deliverable-payload helpers also need to filter `isReasoning` before landing.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for the Matrix/Feishu cron announce reasoning leak. The likely maintainer path is to review and land this PR or a maintainer-equivalent patch, while deciding whether to expand it to cover deliverable-payload filtering or track that broader work separately against #73186/#40534.
