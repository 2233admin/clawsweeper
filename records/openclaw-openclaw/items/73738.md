## Summary

Keep this PR open. Current main still lacks the requested sandbox workspace lifecycle, and this PR is the active implementation path for the linked enhancement, but the latest head still has a concrete security-sensitive redirect validation gap in the package URL hardening that should block merge until addressed.

## What This Changes

The PR adds `workspaceLifecycle: "persistent" | "ephemeral"` sandbox config, run-scoped workspace/runtime cleanup, schema/docs/tests, Discord system-event trust annotations, a Docker E2E fixture tweak, and package-candidate URL download hardening.

## Best Possible Solution

Keep this PR as the implementation path for the linked ephemeral workspace enhancement, but require the package URL redirect path to reject redirects or validate and repin every redirected HTTPS target, add focused coverage for blocked/private redirect targets, and then get secops review plus the relevant sandbox/config/scripts gates before merge.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows the PR uses undici fetch without overriding the default redirect-following behavior, while the custom lookup validates only the original host and falls back to ordinary DNS for redirected hosts. I did not run a live exploit because this review is read-only.

## Solution Assessment

No, not as currently proposed. The sandbox lifecycle design is a plausible narrow solution, but the package URL hardening must handle redirects safely before this patch is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Reject unvalidated redirect hosts:** `scripts/resolve-openclaw-package-candidate.mjs:440`
  - body: The new dispatcher validates the original `package_url` host, but `undiciFetch(parsed, { dispatcher })` still follows redirects by default. `createPinnedLookup` falls back to ordinary DNS for any different redirected host, so an allowed URL can redirect to a private/internal HTTPS host and bypass the blocked-host/address checks. Set redirects to `error`/`manual`, or validate and repin every redirect target before downloading.
  - confidence: 0.88
