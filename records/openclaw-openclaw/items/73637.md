## Summary

Keep this PR open. Current main still has the three override-or-builder prompt assembly paths that drop `extraSystemPrompt` whenever `systemPromptOverride` resolves, while subagent spawn still relies on `extraSystemPrompt` to carry the `## Your Role` task block. The PR is still a plausible implementation candidate for the open bug #73624, but it needs maintainer review on the prompt-authority/security tradeoff and the documented override semantics before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The likely product/code path is to preserve subagent `extraSystemPrompt` across all three prompt assembly surfaces, add regression coverage for no override, override-only, and override-plus-extra behavior, and decide whether the combined-system-prompt approach is acceptable or whether a separate lower-priority message contract should be introduced across providers. If the PR lands, update `docs/gateway/config-agents.md` so the `systemPromptOverride` contract matches shipped behavior.
