## Summary

Keep this PR open. Current main still drops subagent `extraSystemPrompt` whenever `systemPromptOverride` resolves, so the PR is not obsolete or implemented on main. The PR is a focused implementation candidate for the linked bug, with a follow-up commit that responds to the prompt-priority security concern by placing `extraSystemPrompt` before the override, but it still needs normal maintainer review and completion of pending CI/security checks.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best fix is the shared `combineSystemPromptOverrideWithExtra` helper applied to embedded runs, compaction retries, and CLI harness runs, with regression tests for no override, override-only, and override-plus-extra. Before merge, maintainers should confirm the prompt ordering/contract, let CI and the refreshed security review finish, and consider a small docs adjustment for `systemPromptOverride` semantics.
