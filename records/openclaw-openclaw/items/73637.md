## Summary

Keep this PR open. Current main still has the reported short-circuit: when `systemPromptOverride` resolves, the normal prompt builder is skipped, so the `extraSystemPrompt` carrying the subagent `## Your Role` task block is not incorporated. The PR is a plausible focused fix, but it still needs normal maintainer review, especially on the prompt-priority/security concern raised in discussion.

## Best Possible Solution

Keep the PR open for review. The best path is a small shared composition helper applied to the embedded run, compaction retry, and CLI runner call sites, with regression tests for override-only, extra-only, and override-plus-extra behavior. Before merge, decide whether the extra task block should be appended, prepended, placeholder-inserted, or rendered as task data so `systemPromptOverride` remains authoritative by design.
