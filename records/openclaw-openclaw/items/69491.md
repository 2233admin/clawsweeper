## Summary

Keep this PR open. Current main still does not forward default Anthropic beta flags through Amazon Bedrock Converse, so the contribution targets a real provider gap, but the proposed patch is not ready because it overwrites existing Bedrock `anthropic_beta` values, uses a broad app-profile name heuristic for Anthropic-only fields, and is dirty against newer Bedrock request-shaping work on main.

## What This Changes

The PR adds an Amazon Bedrock stream wrapper and tests to inject default Anthropic beta flags into `additionalModelRequestFields.anthropic_beta` for Bedrock-hosted Claude models.

## Best Possible Solution

Keep this PR open and request a focused revision: implement a Bedrock-owned payload patch that merges and dedupes default Anthropic beta flags with existing `additionalModelRequestFields.anthropic_beta`, preserves other request fields, injects only for confirmed Anthropic Claude Bedrock targets, composes with guardrail/cache/temperature wrappers on current main, and adds regression tests for guardrail/no-guardrail plus existing beta-field composition. Keep the shape compatible with the open custom-beta discussion in #39734.
