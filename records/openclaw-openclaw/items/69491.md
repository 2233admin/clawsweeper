## Summary

Keep this PR open. Current main still does not add default Anthropic beta flags to Bedrock Converse `additionalModelRequestFields`, while the PR targets a real gap between the direct Anthropic wrapper and the Bedrock route. The proposed patch is not ready as-is because it overwrites existing `anthropic_beta` values, can collide with pi-ai's own `additionalModelRequestFields`, and uses the broader cache-semantics model heuristic for an Anthropic-only request field.

## Best Possible Solution

Keep the PR open and request a focused revision: add a Bedrock-owned payload patch that merges and dedupes the default beta flags with any existing `additionalModelRequestFields.anthropic_beta`, preserves other `additionalModelRequestFields` such as pi-ai reasoning fields, limits default-beta injection to confirmed Anthropic Claude Bedrock targets, and adds tests for guardrail/no-guardrail plus existing beta-field composition. Coordinate the data shape with #39734 so future custom Bedrock betas compose with these defaults.
