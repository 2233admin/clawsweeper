## Summary

Keep open. Current main still does not provide a public configurable local-media-root allowlist for the explicit image tool; the remaining request is valid but security-sensitive and overlaps an open maintainer-labeled PR, so it needs maintainer review rather than cleanup closure or an autonomous repair branch.

## What This Changes

Add a configurable `tools.image.mediaLocalRoots`-style allowlist so image-tool local reads can include iMessage attachment directories outside the default workspace/media roots.

## Best Possible Solution

Define and document a narrow opt-in local media root allowlist for image/media understanding, validate roots as explicit non-filesystem-root directories, route the resolved roots through the shared media-root resolver and `loadWebMedia`, preserve managed inbound-media behavior, and add schema/type/docs/tests for both configured acceptance and existing rejection cases.
