## Summary

Keep open. Current main still does not expose a public image-tool local media root allowlist: the explicit image tool resolves only default/workspace roots, while the media config schema and type contract have no `mediaLocalRoots`-style field. Managed inbound media and channel attachment-root handling cover adjacent cases, but not the requested opt-in roots for explicit image-tool reads.

## What This Changes

Request to add a configurable local media root allowlist for the image/media understanding tool so channel attachment paths can be read without manual copying.

## Best Possible Solution

Define and document a narrow opt-in local media root allowlist for image/media understanding, validate entries as explicit non-root local directories, route them through the shared root resolver and `loadWebMedia`, preserve managed inbound-media behavior, and cover both configured acceptance and existing rejection cases in schema, type, docs, and tests.
