## Summary

Keep this PR open. Current main still does not implement the requested autonomous/minimal-prompt context-pruning cap: `keepLastAssistants` remains 3 when cache-ttl pruning is enabled, the embedded extension factory has no prompt-mode input, and the run/compaction paths compute prompt mode but do not pass it into pruning setup. The PR discussion is relevant because the initial `skillsSnapshot` signal problem was addressed in the second PR commit by switching to `promptMode === "minimal"`.

## Best Possible Solution

Review this PR as an active implementation candidate. The best path is to rebase or merge the small prompt-mode plumbing on current main: add `promptMode` to embedded extension factory inputs, cap `settings.keepLastAssistants` to 1 only for effective minimal prompt mode, pass the effective run and compaction prompt modes into the factory, preserve explicit `keepLastAssistants: 0`, keep focused minimal/full tests, and address the stale model-id test nit if maintainers agree.
