## Summary

Keep open. Current main has useful LLM/docs navigation aids, including docs hubs, `docs:list`, live docs search, and system-prompt guidance to consult local docs/config schema, but it does not provide the requested generated `docs_map.md` sibling with per-page H2/H3/H4 heading hierarchy. The related ClawSweeper items are only title-search overlaps and do not supersede this docs-map request.

## Best Possible Solution

Keep #69442 open as a focused docs/tooling enhancement. The best implementation is a repo-owned docs-map generator that parses Markdown/MDX headings, emits a stable `docs_map.md` or equivalent public docs artifact beside existing LLM-facing docs entrypoints, and wires it into the docs change workflow/checks without replacing the current docs hubs, `docs:list`, or live docs search.
