## Summary

Keep open. Current main still has `pnpm docs:list`, a human docs hub, CLI docs search, and public `llms.txt`, but it does not provide the requested generated `docs_map.md` or `docs-map.md` with per-page H2/H3/H4 heading hierarchy for LLM navigation. The prior ClawSweeper review remains accurate against current main `f6d23ab5c2d8e7cbf50522b6c8137241524b6646`.

## Best Possible Solution

Add a generated public `docs_map.md` sibling to `llms.txt`, built from source docs in deterministic order with H2/H3/H4 extraction. Prefer a small Node stdlib generator with a check mode, wire it into the existing docs check/publish path, and avoid new third-party actions, broadened permissions, or unrelated docs rewrites.

## Clownfish Work Prompt

Implement the requested documentation map for this issue. Add a deterministic generated public `docs_map.md` (or equivalent public route if Mintlify requires a specific source placement) that is a sibling to public `llms.txt` and contains a nested outline of source docs pages with H2/H3/H4 headings so LLMs can inspect docs structure in one fetch.

Expected boundary:
- Use a small Node stdlib generator; do not add third-party runtime dependencies or new third-party GitHub Actions.
- Walk English source docs deterministically. Reuse or deliberately align exclusions with `scripts/docs-list.js` (`archive`, `research`) unless there is a documented reason to differ.
- Strip front matter before heading extraction. Include page path/title/summary when available, then H2/H3/H4 hierarchy. Avoid H1-only maps.
- Add a check mode so CI can fail on stale generated output.
- Wire the generator into the existing docs check and publish path, probably via `package.json` docs scripts and `scripts/docs-sync-publish.mjs`; avoid broadening `.github/workflows/docs-sync-publish.yml` permissions, action refs, or secret access.
- Do not rewrite unrelated docs, generated locale output, package locks, or workflow policy.
- Changelog is likely not required for a docs-only generated artifact; if a runtime or CLI surface changes, add a single active-version changelog entry crediting @GHesericsu.

Relevant context:
- Current `scripts/docs-list.js` lists paths/summary/read_when only and does not extract headings.
- `docs/start/hubs.md` is a human link hub, not a generated heading map.
- `src/commands/docs.ts` performs live docs search, not a static map.
- Public `llms.txt` exists but `/docs_map.md` and `/docs-map.md` currently 404.

Validation should include targeted generator tests for heading extraction, front matter stripping, deterministic ordering, and excluded directories; docs check/MDX/link checks as appropriate; and a final `git diff --check`.
