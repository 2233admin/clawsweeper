## Summary

Keep open. Current main and the latest release still have the reported installer behavior: the Unix installer can warn that the npm global bin directory is missing from the user's original PATH, skip verification by default, and then print the installed-successfully banner even though the reporter's next shell may still not find `openclaw`.

## What This Changes

The issue asks for the curl/bash install path on Ubuntu to leave `openclaw` installed and discoverable instead of reporting success while the command is unavailable.

## Best Possible Solution

Make the Unix installer's normal success path match what the user can run next: either ensure the CLI is discoverable in the user's next shell, or treat the missing original-PATH case as an action-required/non-success outcome with focused installer smoke coverage and matching troubleshooting docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run the npm-method installer with the npm global bin directory absent from the parent shell PATH and `OPENCLAW_VERIFY_INSTALL` unset; current main can warn about that PATH gap and still continue to a success banner.

## Solution Assessment

Yes. A narrow installer fix is the maintainable path; docs-only guidance or optional verification would not address the misleading default success path for first-time installs.
