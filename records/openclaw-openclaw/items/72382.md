## Summary

Keep open. Current main still allows the Unix curl installer to warn that the npm global bin directory is missing from the user's original PATH, skip verification by default, and then print an installed-successfully banner; the existing non-root smoke repairs PATH after the installer returns, so it does not prove the reporter's next command works.

## What This Changes

The issue asks the Ubuntu/Linux curl installer to leave the `openclaw` command available after a reported successful install.

## Best Possible Solution

Make the Unix installer's normal success path line up with what the user can run next: either ensure the CLI is discoverable without a manual PATH repair, or make the missing original-PATH case an action-required/non-success outcome with focused installer smoke coverage and matching troubleshooting docs.

## Reproduction Assessment

Yes, at source level. Run the npm-method installer with the npm global bin directory absent from the parent shell PATH and `OPENCLAW_VERIFY_INSTALL` unset; current main can warn about that PATH gap and still continue to a success banner.

## Solution Assessment

Yes, a narrow installer fix is the best direction. Docs-only guidance or optional verification would not address the misleading default success path for new installs.
