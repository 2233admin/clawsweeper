## Summary

Keep open. Current main still allows the reported Unix installer outcome: npm/global-bin PATH can be missing from the user's original shell, post-install verification is opt-in, and the installer can still print an installed-successfully banner afterward.

## What This Changes

The issue reports that the Unix curl installer can say OpenClaw installed successfully on Ubuntu 26.04 while `openclaw` is not available on PATH afterward.

## Best Possible Solution

Make the Unix installer's default success path match the user's next command: either ensure the installed CLI is discoverable in the relevant shell without a manual PATH export, or make the missing original-PATH case an explicit action-required/non-success outcome, with smoke coverage and docs aligned.

## Reproduction Assessment

Yes, at source level. Run the npm-method Unix installer with the npm global bin directory absent from the parent shell PATH and `OPENCLAW_VERIFY_INSTALL` unset; current main can warn about that PATH gap and still print the success banner.

## Solution Assessment

Yes, a narrow installer fix is the best direction. Docs-only guidance or optional `--verify` does not address the misleading default success path for new users.
