## Summary

Keep open. Current main still has gateway startup and doctor repair paths that return as soon as the runtime-deps scan reports no missing packages, while two open PRs already reference this issue with closing syntax. This issue should close only after one of those implementation candidates, or an equivalent fix, lands.

## What This Changes

The issue requests that gateway startup and `openclaw doctor --repair` detect and recover from partially written bundled runtime dependency installs left by interrupted staging.

## Best Possible Solution

Land one chosen runtime-deps fix that makes startup and doctor distinguish a fully materialized install root from a partially written package tree, with focused coverage for both early-return paths, then close this issue through the merged PR.

## Reproduction Assessment

Yes. A high-confidence unit-level reproduction is to create a packaged install root whose `node_modules/<dep>/package.json` has a satisfying version while the generated install manifest or package contents are incomplete, then observe the scan report no missing deps and the gateway/doctor early-return paths skip repair.

## Solution Assessment

Unclear until the open PRs are reviewed. The narrow direction of checking the materialized install plan before returning is appropriate, but maintainers need to choose between #75310 and the broader #75183 refactor.
