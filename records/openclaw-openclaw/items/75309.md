## Summary

Keep open. Current main still has the reported gateway startup and doctor repair early-return shape, and the same-author open PR #75310 is already linked as the implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that interrupted bundled runtime dependency staging can leave partial packages that gateway startup and doctor repair currently treat as satisfied.

## Best Possible Solution

Keep this issue open while reviewing the existing fix candidate, and land a narrow change that makes gateway startup and doctor repair treat partially materialized package-level runtime dependency roots as repair-worthy, with focused tests for both call sites.

## Reproduction Assessment

Yes. A high-confidence unit-level reproduction is clear from current main: create a staged install root where a dependency has a satisfying node_modules/<dep>/package.json but the overall package-level install is not materialized, then observe scanBundledPluginRuntimeDeps allow the gateway and doctor early returns to skip repair.

## Solution Assessment

Unclear until #75310 is reviewed. The requested direction is appropriate, but the safest implementation should use the current generated package.json materialization contract rather than the legacy .openclaw-runtime-deps.json file alone.
