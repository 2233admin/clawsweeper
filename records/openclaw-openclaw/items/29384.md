## Summary

Keep this open. Current main implements the GPU part through `sandbox.docker.gpus`, but there is still no `sandbox.docker.ipc`, Docker `--ipc` emission, or reviewed raw Docker argument passthrough; the remaining host-IPC/raw-args request is a sandbox isolation policy decision for maintainers/secops.

## What This Changes

Allow sandbox Docker configuration to pass custom Docker run parameters such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep this open for maintainer/secops review. Treat `sandbox.docker.gpus` as implemented on current main, then decide whether the remaining need should become a narrow typed and guarded `sandbox.docker.ipc` setting or be rejected as out of scope; avoid broad raw Docker args unless maintainers explicitly accept the validation, audit, docs, and support tradeoffs.
