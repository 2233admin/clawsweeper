## Summary

Keep this issue open. Current main implements the GPU half through `sandbox.docker.gpus`, but it still does not expose `--ipc=host`, a typed `sandbox.docker.ipc`, or reviewed raw Docker argument passthrough. The remaining host IPC/raw flag request is security-sensitive sandbox policy work, not conservative cleanup closure material.

## Best Possible Solution

Keep this issue open for maintainer/security review. Treat `sandbox.docker.gpus` as already implemented, then decide whether the remaining request should become a narrow typed and guarded `sandbox.docker.ipc` setting or be rejected as out of scope. Avoid broad raw Docker args unless maintainers explicitly accept the isolation and auditability tradeoffs.
