## Summary

Keep open. Current main still only preserves the caller-supplied active session key during session maintenance, not all active subagent child sessions. The PR addresses a real bug class, but the open Codex review is material: the proposed helper reads only the in-memory subagent run map, while current cross-process query paths use the persisted/in-memory snapshot helper.

## Best Possible Solution

Keep the PR open but require changes before merge: compute active subagent child session keys from the same persisted/in-memory snapshot used by other subagent query paths, thread that preserve set through pruneStaleEntries, capEntryCount, and enforceSessionDiskBudget, and add regression coverage for stale/capped/disk-budget active subagent sessions plus the persisted-snapshot case. Coordinate with the broader open issue #56034 if maintainers prefer a single canonical tracking item.
