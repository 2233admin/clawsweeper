## Summary

Do not auto-close #58660. Current main documents the canonical WeChat path as the external `@tencent-weixin/openclaw-weixin` plugin and provides explicit update/uninstall plus generic legacy-id/prefer-over metadata, but I did not find an implemented OpenClaw-core migration that maps `openclaw-weixin` to `openclawwechat` or automatically removes the old managed plugin directory. The issue is recent and describes a concrete installer/migration failure, so it needs maintainer or plugin-owner handling rather than cleanup closure.

## Best Possible Solution

Keep the issue open for maintainer/plugin-owner judgment. The likely best path is either plugin-owned migration docs/CLI for moving users from `openclawwechat` or old `openclaw-weixin` installs to the supported `@tencent-weixin/openclaw-weixin` path, or a narrowly designed core migration contract such as manifest-declared legacy/replaces ids with explicit uninstall confirmation. Core should not guess and delete similarly named plugins automatically.
