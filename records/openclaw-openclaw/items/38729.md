## Summary

Keep open for maintainer review. The PR targets a real Anthropic thinking capability gap, but the proposed hook is now on the legacy path and current `main` resolves Anthropic thinking levels through provider profiles first, so this patch would not expose Opus 4.6 `xhigh` in the active profile-driven UI/validation path.

## What This Changes

The PR adds an Anthropic `supportsXHighThinking` callback for Claude Opus 4.6 model IDs in `extensions/anthropic/register.runtime.ts`.

## Best Possible Solution

Keep Claude thinking support defined through `resolveClaudeThinkingProfile`. If maintainers approve Opus 4.6 max effort as user-facing, update that helper plus Anthropic tests and thinking docs; otherwise close this PR with the rationale that Opus 4.6 remains adaptive-only in the advertised profile.
