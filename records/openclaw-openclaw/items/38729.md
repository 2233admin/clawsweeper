## Summary

Keep this PR open, but do not merge the current patch as-is. Current main still does not expose Anthropic Claude Opus 4.6 `xhigh`/`max` in the provider profile, but the proposed legacy `supportsXHighThinking` hook is bypassed because Anthropic already supplies a non-empty `resolveThinkingProfile`; the remaining question is a maintainer/provider capability decision plus a profile/test/docs update if accepted.

## Best Possible Solution

If maintainers confirm Opus 4.6 should expose max effort, update that helper along with Anthropic regression tests and thinking docs; otherwise close this PR with the rationale that Opus 4.6 remains adaptive-only in the advertised profile.
