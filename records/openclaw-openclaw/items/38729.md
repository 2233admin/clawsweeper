## Summary

Keep this PR open. Current main still keeps Anthropic Claude 4.6 adaptive-only and advertises `xhigh`/`max` only through the provider thinking profile for Claude Opus 4.7. The PR's `supportsXHighThinking` hook is a deprecated compatibility hook and would be bypassed by the current non-empty Anthropic `resolveThinkingProfile`, so the PR needs maintainer policy/rework rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer decision and rework. If OpenClaw should advertise Claude Opus 4.6 max effort, implement it through `resolveClaudeThinkingProfile` or the Anthropic provider-profile path, add focused Opus 4.6/Sonnet 4.6 boundary tests, and update the public thinking docs. If the current Opus 4.7-only `xhigh`/`max` boundary is intentional, close this PR with that rationale. Do not merge the current hook-only diff as-is.
