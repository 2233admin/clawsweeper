## Summary

Current main still routes Telegram inbound text through `expandTextLinks(...)` only, and that helper only converts `text_link` entities. Bold/italic/code/pre/strikethrough/underline/spoiler entities are still not converted before `BodyForAgent`/`RawBody`, and the latest release has the same behavior. This remains actionable in the Telegram plugin.

## Best Possible Solution

Keep this issue open and fix it in the Telegram plugin inbound body path. Add a helper that converts Telegram formatting entities to agent-visible Markdown while preserving current `text_link` expansion, then cover plain text, captions, links plus formatting, adjacent/nested entities, and offset interactions in focused Telegram tests.
