## Summary

Current main still drops Telegram inbound formatting entities before the agent prompt. The reporter and the prior ClawSweeper comment both pointed at the same path, and local source confirms `resolveTelegramInboundBody` still builds `rawText` through `expandTextLinks(...)` only; that helper only rewrites `text_link` entities and tests still assert a `bold` entity leaves text unchanged. This is a narrow Telegram plugin bug, not a close candidate.

## Best Possible Solution

Keep this issue open and fix it in the Telegram plugin inbound body path. Add a helper that converts Telegram formatting entities to agent-visible Markdown while preserving current `text_link` expansion, then wire it into `resolveTelegramInboundBody` before `rawBody`/`bodyText` construction and cover text, captions, links plus formatting, adjacent/nested entities, and UTF-16 offset interactions.

## Clownfish Work Prompt

Fix the Telegram inbound formatting regression reported in openclaw/openclaw#52859. Current main passes Telegram inbound text through `expandTextLinks(...)` only, so non-link Telegram `MessageEntity` formatting is dropped before `BodyForAgent` and `RawBody`.

Expected boundary:
- Keep the change inside the Telegram plugin. Do not move Telegram-specific behavior into core and do not touch outbound formatting.
- Add or extend a helper in `extensions/telegram/src/bot/body-helpers.ts` that converts Telegram inbound formatting entities to agent-visible Markdown while preserving current `text_link` expansion.
- Support at least the requested entity types: `bold`, `italic`, `code`, `pre`, `strikethrough`, `underline`, and useful Telegram formatting such as `spoiler`; leave unsupported entity types unchanged.
- Handle `text_link` plus formatting against Telegram's original UTF-16 offsets. Avoid applying one transformation in a way that shifts offsets for the next transformation.
- Wire the helper through `resolveTelegramInboundBody` so `BodyForAgent` and `RawBody` preserve intended formatting for plain text and captions. Be careful not to unintentionally change command authorization or media placeholder behavior.
- Add focused tests for plain formatted text, captions, links plus formatting, adjacent entities, nested/overlapping allowed entities, code/pre formatting, and an emoji/UTF-16 offset case.
- Update the active changelog only if user-facing Telegram bug fixes are being recorded for this release; if so, add a single-line fix entry with `Thanks @egida307-star`.

Do not change dependency versions, generated baselines, core channel contracts, or unrelated Telegram delivery code.
