## Summary

Keep this PR open. Current main does not already include the requested plugin logging best-practice guidance; it only lists `api.logger` as an SDK field. The PR is docs-only with no observed security-sensitive diff, but its wording should be corrected before merge because current source makes `api.logger` required and only `debug` optional.

## Best Possible Solution

Keep the PR open for revision or maintainer edit: add concise plugin logging guidance to `docs/plugins/building-plugins.md` or the SDK docs, explain why plugin registration should avoid `console.log` when CLI commands emit JSON on stdout, and align the example with the current `PluginLogger` contract, such as `api.logger.info(...)` and `api.logger.debug?.(...)`.
