## Summary

Keep this PR open. Current main still lacks equivalent plugin logging best-practice guidance, but the patch should be revised before merge because it documents `api.logger` and required logger methods as optional, while the SDK contract only makes `debug` optional. The exact `openclaw config schema` reproduction should also be kept accurate against the current manifest-only schema path.

## Best Possible Solution

Revise this docs patch and then review/merge it: add concise plugin logging guidance to `docs/plugins/building-plugins.md` or the SDK docs, warn against stdout-writing `console.log` in plugin registration/runtime code, and align the example with the current contract: `api.logger.info(...)`, `api.logger.warn(...)`, `api.logger.error(...)`, and `api.logger.debug?.(...)`. Keep any `openclaw config schema` mention accurate against the current manifest-based schema path.
