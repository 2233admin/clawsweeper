## Summary

Keep this PR open. Current main still does not strip echoed inbound metadata in the shared outbound payload plan, and the linked bug remains open. The PR is relevant, but its current implementation uses an inbound UI sanitizer that also strips legitimate leading timestamp text, so maintainers should revise or consolidate it rather than close it as obsolete.

## Best Possible Solution

Keep this PR open for maintainer follow-up or consolidation with #50520 and #39847. The best fix is a narrow outbound-safe sanitizer in the shared createOutboundPayloadPlanEntry() path that removes copied inbound metadata sentinel blocks before all delivery projections while preserving legitimate timestamp-prefixed assistant output, with regression tests for both stripping and timestamp preservation.
