## Summary

Keep this PR open. Current main still does not implement the PR's intended shared outbound metadata-stripping safety net in `normalizeReplyPayloadsForDelivery` / `createOutboundPayloadPlan`, and the submitted patch also has a real functional review concern because `stripInboundMetadata()` strips leading timestamp prefixes from otherwise valid outbound text.

## Best Possible Solution

Revise and review this PR rather than close it. The best fix is a narrow outbound-safe sanitizer in the shared `createOutboundPayloadPlanEntry()` path that strips copied inbound metadata sentinel blocks before all delivery projections, while preserving legitimate assistant output such as timestamp-prefixed logs or transcripts. Add focused tests for both the metadata leak and the timestamp-preservation case, then land it against #39847 if the behavior matches maintainer intent.
