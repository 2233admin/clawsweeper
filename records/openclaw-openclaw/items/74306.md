## Summary

Keep open. Current main already implements the cron.add/cron.update and empty-prompt parts of this PR, but the #74019 allowlist bug still appears present. The PR should be narrowed because its cron.list/status/remove catch-all changes could mask real cron storage or service failures as INVALID_REQUEST.

## What This Changes

This PR wraps cron gateway handler calls in INVALID_REQUEST responses, changes the agent tool allowlist guard for disableTools runs without runtime toolsAllow, and adds an empty-prompt capability CLI guard/test commit.

## Best Possible Solution

Keep this PR open for maintainer review, but narrow it before merge: drop the obsolete cron and empty-prompt commits, keep the #74019 allowlist behavior with focused tests, and only add more cron handler wrapping if a specific user-input failure is proven and non-user service failures continue to rethrow.
