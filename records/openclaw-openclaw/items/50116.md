## Summary

Keep this PR open. Current main still does not handle Windows drive-letter `sessionFile` paths in the session resolver, but the PR patch should not be closed as implemented because it is not present on main and should not be landed unchanged without preserving agent/session context.

## Best Possible Solution

Keep the PR open but request a revision: handle Windows drive-letter session paths before POSIX containment using the repo's existing Windows-path helpers or `path.win32`, preserve the `agents/<agentId>/sessions/<file>` structure instead of collapsing blindly to the basename, reject degenerate paths safely, and add focused tests for both slash styles plus cross-agent and fallback cases.

## Evidence

- **Current resolver still uses POSIX path semantics:** On current main, `resolvePathWithinSessionsDir` only calls `path.isAbsolute()` for absolute detection. On POSIX, Windows drive-letter strings are treated as relative and then resolved under `realBase`, which is the broken behavior described by the PR.
  - file: [src/config/sessions/paths.ts:190](https://github.com/openclaw/openclaw/blob/2ccdbc7dd93bfcb08e100a479af79d8d8a211d97/src/config/sessions/paths.ts#L190)

  - sha: [2ccdbc7dd93b](https://github.com/openclaw/openclaw/commit/2ccdbc7dd93bfcb08e100a479af79d8d8a211d97)
- **Live Node path behavior matches the reported bug:** A read-only Node check on this POSIX runner showed `path.isAbsolute('C:\\Users\\foo\\sessions\\abc.jsonl') === false` and resolves it to `/home/node/.openclaw/agents/main/sessions/C:\\Users\\foo\\sessions\\abc.jsonl`; the `C:/Users/foo/...` variant is also treated as relative.
  - command: `node -e 'const path=require("node:path"); const sessionsDir="/home/node/.openclaw/agents/main/sessions"; for (const p of ["C:\\\\Users\\\\foo\\\\sessions\\\\abc.jsonl","C:/Users/foo/sessions/abc.jsonl"]) console.log(JSON.stringify({p,isAbsolute:path.isAbsolute(p),resolved:path.resolve(sessionsDir,p)}));'`

## Risks / Open Questions

- The PR patch as submitted may introduce transcript aliasing by reducing `C:\...\agents\bot2\sessions\sess-1.jsonl` to only `sess-1.jsonl` under the current agent's `sessionsDir`, which weakens the cross-agent session isolation that current main explicitly handles for POSIX absolute paths.
- The submitted patch lacks regression tests for Windows backslash paths, forward-slash drive paths, trailing separators or degenerate drive roots, and cross-agent Windows-authored session paths.
