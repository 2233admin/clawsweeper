## Summary

Keep this PR open. Current main still does not handle Windows drive-letter sessionFile paths on POSIX: the resolver uses host-native path.isAbsolute, so Windows-authored paths fall through as relative paths and reproduce the broken concatenation described by the PR. The PR is relevant, but the submitted basename-only fix still needs revision before merge.

## Best Possible Solution

Keep the PR open and request a revision: detect Windows absolute paths on POSIX using path.win32 or a shared helper, preserve agents/<agentId>/sessions/<file> context where present, route through the existing realpath/containment and cross-agent fallback logic, reject degenerate Windows paths safely, and add targeted regression tests for both slash styles plus cross-agent and symlink cases.
