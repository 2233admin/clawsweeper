## Summary

Keep open: checked-out main still lacks the requested public operator controls. The loader has an internal `installBundledRuntimeDeps` option, but `plugins.installBundledRuntimeDeps` is absent from the public config contract, Gateway startup still pre-stages bundled runtime deps without consulting an opt-out, and package-level repair still defaults execution to the shared install root. The adjacent open #75183 is relevant runtime-deps mitigation work, but it references #75069 and does not close this issueâ€™s config/default request.

## What This Changes

Expose a public `plugins.installBundledRuntimeDeps: false` opt-out, make isolated execution the default for bundled runtime-dependency installs, and document the resulting runtime-deps behavior matrix.

## Best Possible Solution

Add a public `plugins.installBundledRuntimeDeps` boolean to the config type, zod schema, help/labels, generated docs, and plugin docs; thread it through Gateway prestage, startup plugin loading, runtime registry callers, and loader verification so `false` skips all bundled plugin/channel runtime-deps installs and fails with a clear operator-managed dependency message. Separately make package-level bundled runtime-deps repair run in an isolated execution root with atomic replacement by default, preserving existing security/package-manager guardrails.

## Reproduction Assessment

Yes for the source-level gap: current main can be inspected to show the strict schema rejects the requested field, the loader opt-out is internal only, Gateway prestage ignores any user opt-out, and installer execution defaults to `installRoot`. The production disk bloat and event-loop stalls remain field evidence from the linked reports rather than live-reproduced in this read-only review.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: expose the existing loader seam through the public config contract and change the runtime-deps owner path, rather than relying on undocumented environment/systemd workarounds. #75183 may reduce some churn, but it is not a substitute for these operator controls.
