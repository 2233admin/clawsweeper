## Summary

Keep open: current main still has only an internal loader opt-out and continues to run package-level bundled runtime dependency repair without an isolated execution root by default, while the linked field reports make the operator impact concrete enough for a focused fix.

## What This Changes

The issue requests a public `plugins.installBundledRuntimeDeps` config opt-out, isolated bundled plugin runtime dependency installation by default, and docs for the supported operator matrix.

## Best Possible Solution

Add the config field to the public plugin config contract, thread it through gateway prestage and plugin loading so `false` skips bundled runtime dependency installs with clear missing-dependency failures, make package-level repair use an isolated execution root with atomic replacement by default, and update docs/tests/changelog for the supported operator matrix.

## Reproduction Assessment

Yes for the code-level gap: source inspection shows the strict config schema and typed config omit the requested field, while the loader already has only an internal opt-out. The production disk growth and stalls were not live-reproduced in this read-only review, but the linked field reports provide concrete deployment evidence.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable solution because it exposes an existing loader seam and reuses the existing isolated installer seam in the runtime-dependency owner path instead of adding another environment-only workaround.

## Repair Work Prompt

Fix the bundled plugin runtime dependency config gap reported in https://github.com/openclaw/openclaw/issues/75032. Add a public `plugins.installBundledRuntimeDeps?: boolean` config field with default true in the typed config, strict zod schema, schema help/labels, and generated docs metadata. Thread this value through gateway startup prestage and plugin loading so `false` skips bundled runtime dependency installs/prestage and surfaces clear missing-dependency failures instead of silently repairing. Make package-level bundled runtime dependency repair use an isolated execution root plus atomic `node_modules` replacement by default, reusing the existing installer seam; preserve existing plugin-root staging, script-disabled installs, no package-lock behavior, runtime-deps locks, and `OPENCLAW_PLUGIN_STAGE_DIR` path-list lookup semantics. Update docs for the matrix (`installBundledRuntimeDeps` true/false, isolated default, legacy/shared behavior if retained) and add a single-line changelog entry crediting @Charpup. Do not change dependency versions, package manager selection policy, or external plugin install/update behavior.
