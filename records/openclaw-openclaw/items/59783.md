## Summary

Keep open. Current main still passes Discord attachment and sticker names only as fetch path hints, then calls saveMediaBuffer without the originalFilename argument. The media store already supports originalFilename, and Telegram/WhatsApp already forward inbound filenames, so this PR's functional change is still relevant for #59744 rather than obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land the functional Discord fix, or reimplement it on main, by passing attachment.filename and candidate.fileName as the fifth saveMediaBuffer argument and updating the Discord tests to assert that call. Resolve the unrelated state.json ignore decision separately before merging.
