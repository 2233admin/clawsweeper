## Summary

Keep this issue open. Current main has Discord native-command authorization and channel policy checks, and it routes slash commands through the effective Discord route or bound session, but `/new` and `/reset` still do not provide explicit `@agent` targeting. A mentioned agent would be parsed as generic command input/reset body, not as a target-agent override. The related #58110 is broader multi-agent chat UI work and does not supersede this focused Discord native-command behavior.

## Best Possible Solution

Keep open and implement this in the bundled Discord native-command reset path. Add a supported optional target-agent option or mention/name parser for `/new` and `/reset`, resolve it through configured agents, Discord routing, bindings/thread bindings, authorization, and channel policy, set `CommandTargetSessionKey` only for the selected session, and add regression tests for shared guild channels, bound sessions, blocked/disabled channels, unauthorized targets, and the chosen no-target default behavior.
