## Summary

Keep open. This PR is a relevant first slice for #74481 and current main does not already implement OpenAI-compatible `/models` catalog discovery, but the branch intentionally leaves catalog resolution, merge/replace behavior, and Control UI refresh unwired. The patch is mostly self-contained; I found one minor cache-normalization issue that should be fixed before or during normal review.

## What This Changes

The branch adds an OpenAI provider discovery helper and tests that fetch `${baseUrl}/models`, map OpenAI-compatible model envelopes into `ModelDefinitionConfig[]`, cache results, and export the helper from the OpenAI plugin barrel.

## Best Possible Solution

Review this as part of #74481, settle the provider dynamic-catalog contract first, then either merge this primitive after small cleanup or fold it into the actual OpenAI provider hook with docs, focused tests, and follow-through for the Control UI/model-list path.
