## Summary

Keep open. Current main still lacks OpenAI `/models` discovery and the linked provider dynamic-catalog design remains unsettled, so this PR is neither obsolete nor implemented. The patch is a relevant implementation candidate with one small non-blocking cache-key cleanup before maintainer review.

## What This Changes

The PR adds an OpenAI discovery helper and tests that fetch an OpenAI-compatible `${baseUrl}/models` endpoint, map results to `ModelDefinitionConfig[]`, cache them, and export the helper from the OpenAI plugin barrel.

## Best Possible Solution

Settle the provider dynamic-catalog contract in #74481 first, then either merge this primitive after small cleanup or fold it into the actual OpenAI provider/catalog path with docs, focused tests, and Control UI/model-list follow-through.
