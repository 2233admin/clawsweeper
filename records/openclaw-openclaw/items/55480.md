## Summary

Keep open. Current main still loses Discord display channel labels that normalize to an empty ASCII slug, so the PR addresses a real display bug, but the branch is not mergeable as-is because it changes Discord GroupSpace routing from stable guild IDs to mutable slugs and does not preserve mixed ASCII/non-ASCII names described in the PR.

## What This Changes

The PR adds a Unicode-preserving Discord displayChannelSlug fallback when ASCII slug normalization returns empty and changes message/component GroupSpace construction to prefer guild slugs over guild IDs.

## Best Possible Solution

Implement a display-only Unicode-preserving Discord channel label path on current main that covers pure and mixed non-ASCII channel/thread names, while leaving configChannelSlug, allowlist matching, normalizeDiscordSlug, and GroupSpace ID-first policy routing unchanged.
