## Summary

Keep this PR open. Current main still has the reported behavior: the shared buffer-sniff path returns `file-type`'s MIME unchanged, `file-type` 22.0.1 can return `image/apng` for APNG/PNG buffers, and gateway attachment parsing prefers that sniffed MIME over the caller-provided MIME. The PR remains a plausible narrow fix for open #51881, but it needs rebase/adaptation to current `src/media/mime.ts` and changelog cleanup before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate for #51881. Rebase it onto current main, normalize sniffed `image/apng` to `image/png` in the shared buffer-sniff MIME path before gateway attachment MIME selection, keep a focused APNG regression test, preserve the current lazy import/bounded sniff/CAF fallback behavior, and clean up the changelog entry with the required attribution before maintainer review/merge.
