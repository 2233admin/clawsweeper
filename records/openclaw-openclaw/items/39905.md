## Summary

Keep PR #39905 open. Current main still lacks the PR's central Telegram per-conversation reply and preview-streaming override surface: scoped direct/group/topic config types and strict schemas do not expose `replyToMode` or scoped `streaming`, normal message dispatch still passes account-level startup values, and native command delivery still uses the captured account-level `replyToMode`. The related quoted-DM streaming subfix is overlapping context, but it does not supersede the broader conversation-scoped configuration feature.

## Best Possible Solution

Keep this PR, or a maintainer-owned follow-up, open until the feature is ported or explicitly rejected. The best implementation is a focused Telegram plugin change that adds typed and schema-validated scoped overrides under `direct`, `groups`, and `topics`; resolves effective `replyToMode` and preview streaming with topic > direct/group > account precedence before normal dispatch; applies the same reply-threading resolution to native command replies; uses the current nested `ChannelPreviewStreamingConfig` shape; and updates Telegram docs plus focused tests in the current `extensions/telegram` layout.
