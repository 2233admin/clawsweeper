## Summary

Keep this PR open. Current main still does not implement the requested `Global tools profile` overview row for `openclaw status` or `openclaw status --all`; the status commands now route through a shared overview surface/row builder that has no tool-profile slot. Related docs work in #39954 helps explain `tools.profile`, but it does not supersede this status-output change. The PR also needs rework against the current shared status pipeline and a maintainer decision on whether to show only the global baseline or an effective profile summary.

## Best Possible Solution

Keep this PR open and refresh it against the current shared status overview pipeline. The best implementation would add a shared formatter/row hook in the overview surface or row builder, decide whether the value is explicitly the global config baseline or an effective resolved profile summary, and add focused tests for both `openclaw status` and `openclaw status --all`.
