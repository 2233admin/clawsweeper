## Summary

Keep this PR open. Current main still does not implement the requested `Global tools profile` overview row or formatter, and both `openclaw status` and `openclaw status --all` now flow through a shared status overview surface that does not carry `cfg.tools`. Related docs/config-help work clarifies tool profiles, but it does not supersede this status-output behavior.

## Best Possible Solution

Keep this PR open. Refresh it against the current shared status overview pipeline, decide whether the overview should show only the global baseline or an effective profile summary, then add focused regression coverage for both `openclaw status` and `openclaw status --all`.
