## Summary

Keep open. Current main implements the TTS half of the request, but the STT half remains missing: agent entries accept `tts` only, STT runtime requests do not carry an agent id, and audio transcription still resolves from global `tools.media.audio`. The issue discussion already captured the same split, and the related STT/TTS PRs were closed unmerged.

## What This Changes

Request per-agent TTS and STT overrides so different agents can use different speech providers, voices, languages, prompts, and related speech settings in one OpenClaw instance.

## Best Possible Solution

Keep the issue open until a focused STT follow-up lands: add a documented `agents.list[].stt` contract, deep-merge it over `tools.media.audio` for the active agent, thread agent context through transcription entrypoints, and cover the main runtime/channel paths while preserving the shipped TTS behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: add `agents.list[].stt` to config and current strict agent schema has no such field, while STT runtime paths have no agent id or merge hook to apply it.

## Solution Assessment

Yes, the remaining direction should be a separate STT-specific config/runtime change rather than declaring the issue done from TTS. The safer implementation is to mirror the existing TTS merge pattern while keeping provider APIs backward-compatible.

## Repair Work Prompt

Implement the remaining per-agent STT override support for https://github.com/openclaw/openclaw/issues/66252. Scope this to STT only: keep the existing per-agent TTS behavior intact. Add an optional `agents.list[].stt` config contract, preferably reusing the existing `tools.media.audio` shape or a narrow compatible subset for STT fields such as enabled, language, prompt, models/provider selection, echoTranscript/echoFormat, providerOptions, request/baseUrl/headers where appropriate. Deep-merge the active agent STT override over global `tools.media.audio` before transcription runs, and thread active `agentId` through the media-understanding runtime surfaces that can transcribe audio, including plugin runtime aliases, preflight audio transcription, and Discord voice transcription paths. Do not add provider-specific hacks or change speech provider APIs unless a generic contract requires it. Add focused tests for schema acceptance/rejection, merge precedence, no-override compatibility, and at least one agent-aware transcription call path. Update user-facing docs for `agents.list[].stt`; add a changelog entry crediting `Thanks @acwilan` if behavior changes are user-visible.
