## Summary

Keep open. Current main implements the per-agent TTS half, but the requested per-agent STT/language/provider override is still missing: agent config and strict schema expose `agents.list[].tts` only, STT runtime paths still read global `tools.media.audio`, and the issue discussion explicitly kept STT open after the TTS commits landed.

## Best Possible Solution

Keep this issue open and scope the remaining work to per-agent STT: define the public config shape, likely `agents.list[].stt` or an explicitly chosen equivalent, decide merge precedence against global `tools.media.audio`, wire the effective config through media-understanding, audio preflight, plugin runtime file transcription, and Discord voice transcription, then add schema/type/docs/tests while preserving the shipped `agents.list[].tts` behavior.
