## Summary

Current main still does not implement the PR’s intended behavior: the default agent workspace is resolved under the effective home directory, while only non-default fallback workspaces use the normalized state dir. The PR targets a real gap, but the submitted patch is stale against the current `workspace-default.ts` split and uses `OPENCLAW_STATE_DIR` raw, so it should be revised rather than closed or merged as-is.

## Best Possible Solution

Keep this PR open and request a focused revision against current main: update `src/agents/workspace-default.ts` to derive the default workspace base from the normalized state-dir contract, preserve existing unset-env and `OPENCLAW_HOME` fallback behavior, keep `OPENCLAW_PROFILE` suffix behavior, and add targeted tests for unset env, absolute state dir, tilde/relative state dir, default profile, and non-default profile. If maintainers decide workspaces should remain explicitly separate from state dirs, close only after documenting that product decision.
