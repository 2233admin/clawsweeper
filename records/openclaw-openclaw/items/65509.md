## Summary

Keep open: current main still does not implement the requested default-agent workspace behavior, and the PR branch needs revision before it is mergeable. The branch also uses OPENCLAW_STATE_DIR raw, which conflicts with the normalized state-dir path contract and the PR's own tilde example.

## What This Changes

The PR branch changes the default workspace resolver so OPENCLAW_STATE_DIR becomes the base path before appending the default workspace or OPENCLAW_PROFILE workspace suffix.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept OPENCLAW_STATE_DIR as the default workspace base, revise the patch against src/agents/workspace-default.ts, normalize the env value with the existing path-resolution helper or state-dir contract, preserve unset-env and OPENCLAW_HOME fallback behavior, keep OPENCLAW_PROFILE suffix behavior, and add targeted tests plus docs updates. If maintainers want workspaces to remain a separate knob, document that decision and close the PR after that product call.
