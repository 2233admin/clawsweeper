## Summary

Keep open. Current main still exposes the full Voice Call realtime consult tool but does not provide the requested low-latency memory/session context path, and maintainer discussion already confirmed this as a real current-main gap rather than a duplicate of the merged realtime voice work.

## What This Changes

Add a default-off fast memory/session context lookup path for Voice Call realtime calls before falling back to full `openclaw_agent_consult`.

## Best Possible Solution

Implement the fast path inside the Voice Call plugin as an opt-in `realtime.fastContext` feature: config/schema/manifest/docs, partial-transcript accumulation, bounded memory/session lookup through the public memory-host search seam, explicit snippet-or-miss injection, reset for multiple questions per call, and preserved full-consult fallback.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: on current main, enabling Voice Call realtime with `realtime.toolPolicy` not `none` leaves remembered-context questions routed only through the full consult handler, while non-final transcript chunks are discarded before call-state processing.

## Solution Assessment

Yes. A default-off plugin-owned fast-context path is the narrowest maintainable direction because it keeps voice-call-specific latency policy out of core, uses the existing memory search seam, and preserves `openclaw_agent_consult` for deeper fallback work.

Add `realtime.fastContext` config/schema/manifest support in the Voice Call plugin with fields like `enabled`, `timeoutMs`, `sources`, `maxResults`, and `fallbackToConsult`. Keep defaults compatible and do not change existing `openclaw_agent_consult` behavior when the block is absent or disabled. In `extensions/voice-call/src/webhook/realtime-handler.ts` and related runtime code, accumulate/normalize partial user transcript chunks enough to detect common remembered-context utterances, perform a bounded lookup through the public `openclaw/plugin-sdk/memory-host-search` seam using the current agent/session key, inject top snippets or an explicit miss into the realtime session under the configured deadline, allow multiple context questions per call, and preserve full consult fallback for deeper work. Update docs at `docs/plugins/voice-call.md` and any generated config/help docs required by existing checks. Do not solve the broader memory search transport reliability tracked by #71784, do not fold in the consult thinking/fast-mode override work from #72237, and keep owner-specific behavior inside the Voice Call plugin unless a generic additive SDK seam is required.
