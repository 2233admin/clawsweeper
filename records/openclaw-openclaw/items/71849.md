## Summary

Keep open. Current main still exposes the full Voice Call realtime consult tool, but it does not implement the requested low-latency memory/session context path, and the maintainer discussion explicitly confirmed this remains a real Voice Call gap. The related open PR #72237 is adjacent consult tuning, not the fast-context retrieval path requested here.

## What This Changes

Add a default-off Voice Call realtime fast-context mode that accumulates partial transcripts, retrieves memory/session context under a short deadline, injects snippets or a miss signal into the realtime session, and falls back to full consult for deeper work.

## Best Possible Solution

Voice Call should own a default-off `realtime.fastContext` mode that uses the existing memory-host-search SDK seam for bounded memory/session lookup, injects either snippets or an explicit miss into the realtime session, resets for follow-up context questions, and keeps full `openclaw_agent_consult` as the slower fallback.

## Reproduction Assessment

Yes. A static reproduction path is high-confidence on current main: enable Voice Call realtime with `realtime.toolPolicy` not `none`, then context questions can only reach the full consult handler while non-final transcript chunks are ignored; live provider testing would validate latency but is not required to confirm the requested capability is absent.

## Solution Assessment

Yes. The default-off plugin-owned fast-context path is the narrowest maintainable solution because it keeps phone-call UX policy in Voice Call, reuses the public memory search seam, and avoids changing core realtime semantics or replacing full consult.

## Repair Work Prompt

Implement the Voice Call realtime fast-context path requested in https://github.com/openclaw/openclaw/issues/71849.

Scope:
- Add a default-off `realtime.fastContext` config block to the Voice Call plugin schema, manifest, defaults, config help, docs, and tests. Suggested fields: `enabled`, `timeoutMs`, `sources`, `maxResults`, and `fallbackToConsult`; keep names conservative if local config patterns suggest better ones.
- In the Voice Call realtime hot path, accumulate and normalize partial user transcript chunks enough to detect common remembered-context questions, instead of depending only on final transcript events.
- Use the public `openclaw/plugin-sdk/memory-host-search` seam for a bounded memory/session lookup. Prefer `sessionKey`, `sources`, and `maxResults`; do not import core internals from the plugin.
- Inject retrieved snippets, or an explicit empty/miss result on timeout, into the realtime session so the model can answer or gracefully miss within the configured deadline.
- Reset the gate after each lookup so multiple context questions in one call can each trigger retrieval.
- Keep `openclaw_agent_consult` available as fallback for deeper work; do not remove or replace the full consult tool.
- Do not solve #71784 in this PR except for handling lookup errors gracefully, and do not make broad core realtime or memory architecture changes.
- Do not commit real phone numbers, live call logs, credentials, generated artifacts, or provider-specific secrets.
- Add a changelog entry if user-visible config/behavior changes are made, crediting `@amzzzzzzz` and any real contributor being incorporated, never a bot.
