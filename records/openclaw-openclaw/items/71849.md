## Summary

Keep open. Current main and the latest release still route Voice Call realtime remembered-context turns through the full realtime consult tool, ignore non-final transcript chunks before call-state processing, and expose no `realtime.fastContext` config, manifest, or docs surface. The issue discussion also contains a maintainer confirmation that this is a real current-main gap rather than a duplicate of the existing realtime consult work.

## What This Changes

Add an opt-in Voice Call realtime fast-context path that detects remembered-context turns, performs bounded memory/session retrieval, injects snippets or an explicit miss into the realtime session, and leaves full `openclaw_agent_consult` as fallback.

## Best Possible Solution

Implement a default-off, plugin-owned `realtime.fastContext` feature in Voice Call that uses existing memory-host search surfaces, accumulates and normalizes partial transcript chunks, injects bounded snippets or an explicit miss into the realtime session, resets for follow-up context questions, and preserves full `openclaw_agent_consult` for deeper fallback work.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: on current main, remembered-context realtime turns can only use the full consult handler when `realtime.toolPolicy` is enabled, and non-final transcript chunks are discarded before call-state processing.

## Solution Assessment

Yes. The narrow maintainable path is an opt-in Voice Call plugin feature using existing memory-host search APIs and realtime provider tool-result plumbing; moving this into core or replacing the consult tool would be broader than required.
