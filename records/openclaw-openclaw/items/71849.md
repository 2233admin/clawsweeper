## Summary

Keep open. Current main still has the full Voice Call realtime consult path, but it does not implement the requested low-latency memory/session context path, and the maintainer discussion explicitly confirmed this as a current-main gap rather than a duplicate of adjacent realtime voice work.

## What This Changes

Add an opt-in fast memory/session context resolver for realtime Voice Call turns before falling back to the full `openclaw_agent_consult` run.

## Best Possible Solution

Implement a default-off `realtime.fastContext` feature inside the Voice Call plugin: config/schema/manifest/docs, partial transcript accumulation, bounded memory/session lookup via the public memory search seam, explicit snippet-or-miss injection into the realtime session, reset behavior for multiple context questions, and preserved full-consult fallback.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: on current main, enabling realtime Voice Call with `realtime.toolPolicy` not `none` exposes only the full consult tool, while non-final transcript chunks are discarded before call-state processing.

## Solution Assessment

Yes. A default-off, plugin-owned fast-context path is the narrowest maintainable direction because it keeps live-call latency policy in Voice Call, reuses the existing memory search seam, and leaves `openclaw_agent_consult` available for deeper fallback work.
