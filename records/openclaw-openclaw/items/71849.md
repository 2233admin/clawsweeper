## Summary

Keep open. Current main still exposes remembered-context questions in Voice Call realtime through the full `openclaw_agent_consult` path and does not implement the requested bounded fast memory/session context resolver; maintainer discussion also explicitly confirmed this as a current-main gap.

## What This Changes

The issue requests an opt-in fast context path for Voice Call realtime that accumulates partial transcripts, performs low-latency memory/session lookup, injects snippets or a miss into the realtime session, and preserves full consult as fallback.

## Best Possible Solution

Implement a default-off, plugin-owned `realtime.fastContext` feature in Voice Call that uses the public memory search seam, accumulates and normalizes partial user transcripts, injects snippets or an explicit miss under a short deadline, resets for follow-up questions, and leaves `openclaw_agent_consult` available for deeper fallback work.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: on current main, Voice Call realtime with `realtime.toolPolicy` enabled exposes the full consult tool and records only final transcripts, while config, manifest, and docs contain no fast-context path.

## Solution Assessment

Yes. A narrow opt-in Voice Call plugin path is the best fit because the latency contract is phone-call-specific and current memory-host search APIs appear sufficient; broader memory transport reliability remains separately tracked by #71784.
