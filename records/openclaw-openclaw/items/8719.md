## Summary

Keep open. Current main has useful hardening primitives, but it still does not implement the requested native data-centric Security Profile v1.1. The issue discussion includes an explicit maintainer keep-open review, #60981/#7722 only cover filesystem sandboxing, and the linked ClawMoat project is useful external prior art rather than a native OpenClaw implementation.

## Best Possible Solution

Keep this issue open as the native OpenClaw Security Profile umbrella until maintainers either define the v1.1 product/security spec or split it into canonical narrower issues. Review filesystem sandboxing through #60981/#7722, use ClawMoat as external prior art from the discussion, and track separate native follow-ups for data levels, protected OS presets, cross-tool exec/fs/export/network policy, append-only sensitive-action audit logs, approval and TTL elevation, session isolation, config integrity, and plugin/skill verification.
