## Summary

Keep open. Current main has useful adjacent hardening, but it does not implement the requested native data-centric OpenClaw Security Profile v1.1. The issue also has an explicit maintainer keep-open comment, and related #60981/#7722 cover only the filesystem/PathGuard subset rather than the cross-tool data-level model, network/export policy, audit log, TTL elevation, isolation, and plugin verification scope.

## Best Possible Solution

Keep this open as the native OpenClaw Security Profile umbrella until maintainers either define the v1.1 product/security spec or split it into canonical narrower issues. Use #60981/#7722 for the filesystem sandboxing subset, use ClawMoat as external prior art from the discussion, and track separate native work for data levels, protected OS presets, cross-tool exec/fs/export/network policy, append-only sensitive-action audit logs, approval and TTL elevation, session isolation, config integrity, and plugin/skill verification.
