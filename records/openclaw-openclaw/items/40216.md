## Summary

Keep open. The PR contains useful CJK token-estimation work, but it edits the obsolete `pi-extensions` context-pruning path while current main runs the `pi-hooks` pruner, and the active pruner still has raw-length gaps for soft-trim sizing and assistant tool-call arguments.

## What This Changes

The PR adds weighted CJK/non-Latin character estimation to the old context-pruning pruner and adds tests for ASCII, CJK, mixed text, empty strings, and surrogate-pair CJK characters.

## Best Possible Solution

Port the behavior into the active `src/agents/pi-hooks/context-pruning` pruner, use the shared estimator consistently for all text-bearing size estimates, keep soft-trim eligibility and extraction in one consistent budget unit, and add regressions for CJK soft trimming and serialized tool-call arguments.
