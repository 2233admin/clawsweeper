## Summary

Keep this PR open for maintainer review. Current main still caps Slack option values at 75 characters in both shared interactive rendering and native slash argument menus, while Slack's current option object contract lists `value` max length as 150 characters. ([docs.slack.dev](https://docs.slack.dev/reference/block-kit/composition-objects/option-object/?utm_source=openai)) The proposed diff is narrow, matches the documented boundary, and I did not find a blocking correctness issue.

## What This Changes

The PR raises Slack select option value caps from 75 to 150 characters in the shared interactive block renderer and native slash command argument menus, with updated 150/151 boundary coverage and an encoded slash-menu value test.

## Best Possible Solution

Land this narrow correction, or an equivalent maintainer-authored patch, so Slack select and slash-menu option values follow the documented 150-character option-value limit while preserving the existing 75-character text limit, 100-option cap, and fallback behavior for truly oversized encoded values.
