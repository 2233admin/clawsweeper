## Summary

Keep open. Current main still contains the macOS Canvas/A2UI auto-navigation and push-success behavior that can plausibly produce a blank Canvas after a successful `canvas a2ui_push`, and the fresh report is not superseded by the older stale-closed A2UI issues.

## Best Possible Solution

Keep this issue open and fix it in the macOS Canvas/A2UI owner path. First make the auto-navigation guard skip reloads when the current target already equals the last A2UI target, then add a macOS regression smoke that pushes minimal v0.8 A2UI JSONL and verifies visible WKWebView content rather than only bridge success. If it still renders blank, instrument `ensureA2UIHost`, host mounting, `applyMessages`, and WKWebView console errors to separate bundle load, message application, and surface rendering failures.
