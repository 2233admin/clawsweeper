## Summary

Keep open. Current main has the bare chat command acknowledgement fix, but the reported empty-history lifecycle path is still present: reset/session lifecycle code can invoke `before_reset` and `session_end` hooks with no transcript messages, and a gateway agent reset surface still intentionally turns bare `/new` or `/reset` into a model prompt. The issue’s Telegram/lossless-claw logs remain plausible against current source, so this is not conservative cleanup.

## What This Changes

The issue requests that `/reset` and `/new` skip empty-history session-end LLM work when the session is already fresh, avoiding a provider 400 and user-visible error.

## Best Possible Solution

Preserve the existing bare chat acknowledgement behavior, then add a narrow empty-history lifecycle guard or contract so model-backed reset/session-end work is skipped only when the previous transcript has no usable messages. The fix should keep non-empty `before_reset` and `session_end` hooks working and add regression coverage for Telegram/native reset plus gateway session reset with header-only or missing transcripts.

## Reproduction Assessment

Yes for triage: the issue provides concrete Telegram steps and provider logs, and current source shows reset lifecycle hooks can still run with empty message arrays. A live Telegram plus lossless-claw reproduction was not run in this read-only review.

## Solution Assessment

Unclear. Checking for empty history before model-backed lifecycle work is the right direction, but simply suppressing all session hooks would likely break plugin contracts; the safer solution is to guard the empty LLM-producing path while preserving useful reset lifecycle events.
