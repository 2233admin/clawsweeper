## Summary

Keep open. Current main has trusted bundled tool-result middleware that can rewrite tool output before model consumption, but the requested public plugin hook is still absent and external plugins are explicitly blocked from using the existing pre-model middleware seam. The related #62123 is relevant overlap, but it is also open and not an established canonical replacement.

## Best Possible Solution

Keep this open and consolidate it with #62123 under one maintainer-owned design thread, or implement a documented, versioned public hook/trust policy for pre-model tool-result mutation. The design should cover allow/replace/block semantics, async failure behavior, provenance metadata, size bounds, Pi and Codex ordering, SDK docs, and contract tests.
