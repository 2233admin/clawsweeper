## Summary

Keep open. Current main still has the reported Signal attachment bug: the HTTP RPC response reader enforces a fixed 1 MiB cap before `getAttachment` can return its base64 payload, while `channels.signal.mediaMaxMb` is only enforced after the RPC result exists. The same author has opened #73572 with closing syntax and a focused Signal-plugin fix, so this issue should remain open until that PR, or an equivalent replacement, is reviewed and merged.

## Best Possible Solution

Keep this issue open while #73572 remains open. Review and land that PR, or a narrower equivalent, in the Signal plugin: add a bounded per-call RPC response cap, derive the `getAttachment` cap from the effective `mediaMaxMb` with base64/JSON headroom, preserve a conservative default for non-attachment RPCs, and keep regression tests for both raised and oversized caps. Once that ships on main, close this issue as implemented.
