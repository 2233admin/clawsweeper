## Summary

Keep open. Current main still lacks the Feishu reply-path `mediaLocalRoots` plumbing this PR adds: `sendMediaFeishu` already requires scoped roots for local path reads, but `createFeishuReplyDispatcher` neither accepts nor forwards roots, and both Feishu bot dispatcher call sites omit `getAgentScopedMediaLocalRoots`. The PR is a focused implementation candidate, not implemented on main and not clearly superseded by a canonical merged fix. Security pass: the provided diff only touches Feishu bot/dispatcher plumbing, with no CI, dependency, secret, release, or package-resolution changes; the security-sensitive effect is scoped local file reads through the existing agent-root helper.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land one focused Feishu reply-path fix that threads `mediaLocalRoots` into `CreateFeishuReplyDispatcher`, computes roots with `getAgentScopedMediaLocalRoots` at both Feishu bot dispatcher call sites, and adds regression coverage for the dispatcher plumbing. After that merges, close or supersede the overlapping Feishu mediaLocalRoots PRs against the chosen canonical fix.
