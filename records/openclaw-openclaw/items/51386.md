## Summary

Current main still implements session transcript indexing as an opt-in experimental memory-search source, not as a stable/default-on multi-session continuity feature. The request also needs maintainer decisions on privacy defaults, known gateway-session gaps, and indexing cost before it is a safe code-change task.

## Best Possible Solution

Keep this open for memory/core maintainer review. The best path is to first decide the direct/channel/group privacy default, resolve or explicitly scope #53550, review the incremental sync and embedding-cost story from #59577, then add any approved default-on behavior through the config materialization/default pipeline with focused tests and public docs updates for opt-out, scope, stale results, and cost guidance.
