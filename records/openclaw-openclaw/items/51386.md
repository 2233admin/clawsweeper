## Summary

Keep open. Current main still has the session transcript indexing machinery, but it has not been graduated: `sessionMemory` remains experimental and off by default, default sources are still memory-only, docs still call it opt-in/experimental, and the config materialization pipeline still has no `applyMemorySearchDefaults` equivalent. The KeepMyClaw comment is useful external recovery context, but it does not replace the OpenClaw product/default/privacy/docs decision requested here.

## Best Possible Solution

Keep this issue open for memory/core maintainers. A good path is to first resolve or explicitly scope #53550, decide the default privacy policy for direct/channel/group sessions, review the incremental sync/cost story from #59577, add any chosen default-on behavior through the config materialization/default pipeline with focused tests, and update the public memory docs with opt-out, scope, stale-result, and embedding-cost guidance.
