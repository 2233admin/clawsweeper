## Summary

Keep this PR open. Current main still does not implement the PR's central Discord gateway WebSocket handshake timeout: `SafeGatewayPlugin.createWebSocket()` constructs `ws` with only the optional proxy agent, and the Discord tests currently assert `undefined` options in the no-proxy path rather than a timeout. The upstream Carbon issue was useful context, and Carbon 0.16.0's `GatewayPlugin.createWebSocket()` also constructs `new WebSocket(url)` without timeout options, while OpenClaw remains pinned to Carbon 0.16.0. Security review did not find an unrelated supply-chain or secrets-handling change in the PR; the remaining work is functional integration with current main's newer gateway instrumentation and regression tests.

## Best Possible Solution

Keep the PR open and update it against current main by adding a plugin-owned Discord gateway handshake timeout to the existing `WebSocketCtor` construction, preserving the current proxy agent, transport capture, injected testing constructor, and binaryType handling. Add focused tests in the Discord monitor test suite that assert the timeout option is passed in both direct and proxy paths; no core API change appears necessary.
