## Summary

Keep this PR open. Current main still drops Feishu card-action `form_value`, so the intended behavior is not implemented yet. The branch is relevant, but it appears incomplete because the live `card.action.trigger` parser still reconstructs `action` with only `value` and `tag`; real callback fields such as `form_value`, `option`, and `options` would not reach the decoder/handler. It also overlaps active Feishu callback-field work in #42754, #42908, #43202, and #43953.

## Best Possible Solution

Keep this PR open for maintainer review, but do not close it as implemented. The best path is to choose one canonical Feishu callback-field fix across #42754, #42908, #43202, #43953, and this PR: preserve the missing fields in `parseFeishuCardActionEventPayload`, pass them through the decoder/handler consistently, decide whether command text or structured JSON is the supported payload format, and add parser pass-through plus handler tests for real `card.action.trigger` payloads.
