## Summary

Keep this PR open. Current main still models and dispatches Feishu card actions from `action.value` only, while Feishu form-submit callbacks include `action.form_value`; the PR is still a live implementation candidate and overlaps several open Feishu callback-field threads that need a single maintainer choice on payload encoding.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to pick one canonical Feishu card callback-field fix across #42754, #42908, #43202, #43953, and this PR: preserve missing raw callback fields in `parseFeishuCardActionEventPayload`, expose them through the decoder and handler consistently, decide whether command text or structured JSON is the supported payload format, and add parser plus handler tests for real `card.action.trigger` form/selector payloads.
