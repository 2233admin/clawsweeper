## Summary

Keep this PR open. Current main still drops Feishu card-action `form_value`, and the PR is a relevant Feishu-scoped bug-fix candidate, but it appears incomplete because the live event parser still needs to pass sibling callback fields through to the handler.

## Best Possible Solution

Keep this PR open for maintainer review, require parser-to-handler pass-through coverage for live Feishu `card.action.trigger` events, and either land this narrow `form_value` fix or consolidate it with the broader Feishu callback-field work in #42754, #42908, and #43202.

## Risks / Open Questions

- The provided PR patch does not update `extensions/feishu/src/monitor.account.ts`, so the live `card.action.trigger` path may still discard `form_value` before the new decoder/handler logic runs.
- The PR appends raw `key=value` pairs into the command string. Maintainers should decide the intended escaping, ordering, and handling for whitespace, arrays/objects, and command-parser-sensitive characters before landing.
- Related PR #42908 appears broader (`option`, `options`, and `form_value`). Maintainers should avoid landing overlapping partial fixes without deciding whether this narrower PR supersedes, complements, or should be folded into the broader callback-field fix.
