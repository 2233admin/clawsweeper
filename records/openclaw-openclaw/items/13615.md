## Summary

Keep this issue open. Current main still has adjacent reactive provider 429/cooldown/fallback handling, provider usage readouts, and channel/control-plane retry helpers, but it does not implement the requested configurable outbound API rate limiting, queueing, token/cost budget enforcement, warning/status controls, or `openclaw limits` CLI. The existing maintainer comment also keeps this open as the canonical tracker, with #13686 closed unmerged as prior art and narrower related issues still open.

## What This Changes

The issue requests configurable outbound API rate limiting, throttling, token/cost budgets, usage visibility, alerts, and `openclaw limits` CLI controls.

## Best Possible Solution

Keep this open as the canonical core design tracker. The best end state is a maintainer-reviewed, opt-in outbound policy layer covering model-provider dispatch first, with explicit follow-ups or inclusion criteria for web-search/tool and channel/plugin calls, plus aligned schema, docs, status/CLI UX, persistence, warning thresholds, 429 backoff behavior, and token/cost budgets.
