## Summary

Keep this issue open. Current main has adjacent reactive provider 429 handling, auth-profile cooldown/fallback behavior, provider quota readouts, and inbound/control-plane/webhook rate limiters, but it still does not implement the requested configurable outbound API rate limiting/throttling layer, request/token budget enforcement, queueing/status controls, or `openclaw limits` CLI. A maintainer comment also explicitly kept this open as the canonical tracker, and the linked implementation PR was closed unmerged.

## Best Possible Solution

Keep this open as the canonical core product/design tracker. The best path is an opt-in outbound policy layer at model-provider, tool/web-search, and channel/plugin request boundaries, with config/schema/docs for provider/model/global RPM, TPM, daily/monthly windows, queue/backpressure controls, 429 retry/backoff behavior, token/cost budget enforcement, warning/status reporting, and an `openclaw limits` CLI. Treat #13686 as prior art and #14206, #42475, and #60450 as narrower related follow-ups, not replacements.
