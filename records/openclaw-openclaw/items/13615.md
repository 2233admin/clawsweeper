## Summary

Keep this open. Current main has adjacent failed-auth/control-plane/webhook rate limiting, provider 429 cooldown/fallback handling, and usage/cost reporting, but it still lacks the requested configurable outbound API limiter, queueing, token/cost budget enforcement, warning thresholds, alerts, and `openclaw limits` CLI. The issue discussion also includes a maintainer comment keeping this as the canonical broad design tracker, with the linked implementation PR closed unmerged rather than superseding the request.

## What This Changes

Add configurable rate limiting, throttling, token/cost budgets, observability, alerts, and CLI controls for external API calls across providers, tools, search, and channels.

## Best Possible Solution

Keep this as the canonical core design tracker. The desired end state is a maintainer-reviewed, opt-in outbound policy layer that starts with model-provider dispatch, explicitly defines whether tools/search/channels/plugins are in scope, and aligns config schema, docs, persistence, status/CLI UX, warning thresholds, 429 backoff, and token/cost budget enforcement.
