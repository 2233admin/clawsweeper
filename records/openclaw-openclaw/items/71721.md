## Summary

Keep open. Current main has partial hardening for Bonjour and display-only network discovery, but the gateway LAN and tailnet helpers still have throwing `os.networkInterfaces()` paths, and the linked open PR #71950 remains the closing-syntax implementation candidate for the remaining work.

## What This Changes

The issue asks OpenClaw Gateway to tolerate sandbox failures from Node `os.networkInterfaces()` instead of crashing and disconnecting the TUI.

## Best Possible Solution

Keep the issue open until PR #71950 or an equivalent narrow fix lands, preserving the raw reader contract for callers that intentionally need propagation while making startup-critical best-effort discovery paths degrade to missing addresses.

## Reproduction Assessment

Yes. A focused reproduction exists by mocking `os.networkInterfaces()` to throw `uv_interface_addresses failed`; current unit tests still assert the LAN and tailnet helpers throw on that path.

## Solution Assessment

Yes. The narrowest maintainable fix is to reuse the existing safe network-interface wrapper at best-effort gateway/tailnet discovery call sites rather than changing `readNetworkInterfaces()` globally.
