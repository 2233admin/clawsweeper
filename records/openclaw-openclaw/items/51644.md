## Summary

Keep open. Current main implements timing-based inbound debouncing for rapid same-sender messages, and related coalescing work is already shipped/tracked, but the requested semantic long-message topic detection/splitting and adaptive aggregation behavior are still not present in source, docs, config schema, or plugin hook contracts.

## Best Possible Solution

Keep this issue open and narrow the remaining work around semantic preprocessing beyond shipped `messages.inbound` debounce. The best path is a maintainer decision on whether this belongs in gateway-owned input preprocessing or a typed plugin SDK hook that can rewrite or split inbound content before agent dispatch, with explicit semantics for long messages, media/control commands, transcript history, and adaptive timing.

## Risks / Open Questions

- Closing now would drop the still-missing long-message multi-topic detection/splitting and adaptive-window portions of the request.
- A future implementation needs product/API design because splitting a user message into multiple turns affects transcript semantics, command handling, reply threading, latency, cost, and privacy.
- The current plugin and internal hook surfaces are mostly observational or claim/handle oriented, so a ClawHub skill/plugin cannot cleanly implement this without a new typed pre-agent rewrite/split contract.
