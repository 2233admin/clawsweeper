## Summary

Keep open. Current main already covers timing-based inbound debouncing for rapid same-sender bursts, including docs/config/tests and channel integrations, but this issue also asks for semantic long-message topic detection/splitting and adaptive aggregation windows. Those remaining pieces are not implemented in the current source, docs, config schema, or plugin hook contracts.

## Best Possible Solution

Keep this issue open and narrow the remaining work around semantic preprocessing beyond shipped `messages.inbound` debounce. The best path is a maintainer design decision on whether this belongs in gateway-owned input preprocessing or a typed plugin SDK hook that can rewrite or split inbound content before agent dispatch, with explicit semantics for long messages, media/control commands, transcript history, and adaptive timing.
