## Summary

Keep this issue open. Current main has durable workspace files, memory promotion/dreaming, standing orders, Feishu mention-fidelity fixes, and configured broadcast-style multi-agent routing, but it still does not provide the requested governed natural-language rule layer with precedence/supersession or a channel-neutral explicit multi-mention target-set contract where every mentioned bot replies once. The linked openclaw-listen-only workaround and open PR #42400 are useful partial context-isolation/listen-only work, not replacements for the requested product/API design.

## Best Possible Solution

Keep this issue open and scope it into design work: define a governed durable rule layer or typed memory/rule records with review, provenance, precedence, and supersession; specify how promoted rules interact with AGENTS.md, SOUL.md, MEMORY.md, dreaming, and session bootstrap refresh; and add a channel/plugin contract for explicit mention target sets so each explicitly mentioned bot/agent can dispatch one reply while unmentioned bots stay silent. Treat openclaw-listen-only and #42400 as useful partial context-isolation/listen-only work, not the canonical fix.

## Risks / Open Questions

- The issue combines two separate product surfaces: durable natural-language rule promotion and explicit multi-mention routing. Maintainers may want to split it into narrower design issues before implementation.
- OpenClaw already has overlapping partial mechanisms: workspace standing orders, MEMORY.md promotion, Feishu mention fidelity, visibleReplies=message_tool, and broadcast groups. A future fix should integrate with these instead of adding another prompt-only workaround.
- Durable rule learning has prompt-injection and governance risk. Any implementation should require explicit user review/approval, provenance, precedence rules, and rollback/supersession semantics rather than silently promoting arbitrary group-chat instructions.
