## Summary

Keep this issue open. Current main has useful related pieces: workspace bootstrap refresh, memory flush/dreaming, standing orders, boolean mention gating, Feishu mention fidelity, and configured broadcast routing. It still lacks the requested governed natural-language rule layer with precedence/supersession, and it still lacks a channel-neutral explicit multi-mention target-set contract where every explicitly mentioned bot replies once while unmentioned bots stay silent.

## Best Possible Solution

Keep this issue open for maintainer design. Define a governed durable rule or typed memory/rule layer with provenance, opt-in promotion, precedence, supersession, rollback, and clear interaction with AGENTS.md, SOUL.md, MEMORY.md, dreaming, and bootstrap refresh. Separately define a channel/plugin contract for explicit mention target sets so each explicitly mentioned bot can produce one visible reply while unmentioned bots stay silent. Treat flowolforg/openclaw-listen-only and #42400 as useful partial context-isolation/listen-only work, not replacements for this product/API design.

## Risks / Open Questions

- The issue combines two product surfaces that may need separate design tracks: durable natural-language rule promotion and explicit multi-mention routing semantics.
- Durably promoting conversational instructions has prompt-injection and governance risk, especially in group chats where observed messages may not be trusted instructions.
- Open PR #42400 should still be reviewed on its own listen-only merits, but it is not the canonical fix for this broader request.
