## Summary

Keep this issue open. Current main has related pieces: workspace bootstrap files, memory flush/dreaming, standing orders, mention gating, Feishu mention-fidelity fixes, and configured broadcast-style multi-agent routing. It still does not provide the requested governed natural-language rule layer with precedence/supersession, nor a channel-neutral explicit multi-mention target-set contract where every explicitly mentioned bot replies once. The linked openclaw-listen-only workaround and open PR #42400 are useful partial context-isolation/listen-only work, not replacements for the requested product/API design.

## Best Possible Solution

Keep this issue open and scope it into design work: define a governed durable rule layer or typed memory/rule records with review, provenance, precedence, rollback, and supersession; specify how promoted rules interact with AGENTS.md, SOUL.md, MEMORY.md, dreaming, and session bootstrap refresh; and add a channel/plugin contract for explicit mention target sets so each explicitly mentioned bot/agent can dispatch one reply while unmentioned bots stay silent. Treat openclaw-listen-only and #42400 as useful partial context-isolation/listen-only work, not the canonical fix.

## Risks / Open Questions

- The issue combines two product surfaces: durable natural-language rule promotion and explicit multi-mention routing. Maintainers may want to split it before implementation.
- Durable natural-language rule learning has prompt-injection and governance risk if arbitrary group-chat instructions are promoted silently.
- Existing partial mechanisms overlap with the request, so a fix should integrate with AGENTS.md, SOUL.md, MEMORY.md, dreaming, group mention gating, Feishu mention fidelity, visibleReplies, and broadcast groups instead of adding another prompt-only workaround.
