## Summary

Current main has the shared sender-scoped group-policy helper, but Feishu still does not wire it into group tool policy resolution. The Feishu schema only accepts group `tools` with `allow`/`deny`, not `toolsBySender` or `alsoAllow`, and `resolveFeishuGroupToolPolicy` still returns only `groupConfig?.tools`. The PR therefore remains a plausible focused implementation slice rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is to rebase it onto current main, replace the stale generated JSON baseline changes with the current `config-baseline.sha256` workflow, keep the Feishu schema/policy/tests focused on `groups.*.toolsBySender`, and then decide whether the Feishu-specific docs should mention the supported sender-key format.
