## Summary

Keep open. Current main still scans only JS/TS-family files, and the scanner tests still assert Markdown is not scannable. This PR remains a focused security-hardening candidate that supersedes stale-closed #10705, but the latest PR regex still misses practical download-and-execute variants, so the best path is to revise it rather than close it.

## Best Possible Solution

Keep this PR open as the active Markdown skill-scanning hardening candidate. Revise `markdown-download-exec` to cover sudo/env/wrapper prefixes, absolute interpreter paths, versioned interpreters, and line-continuation pipelines; add regression tests for those cases; then run the targeted scanner tests and the appropriate changed gate before merge. Separately decide whether direct ClawHub skill folder installs should call the built-in scanner before writing into the workspace.
