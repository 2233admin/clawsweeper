## Summary

Keep open. Current main still excludes Markdown from the built-in skill scanner, and the scanner tests still assert `.md` is not scannable. This PR remains a focused security-hardening candidate for `SKILL.md`, but the current PR regex still has practical false negatives in its critical download-and-execute rule, so it needs revision rather than closure or merge-as-is.

## Best Possible Solution

Keep this PR open as the active Markdown skill-scanning hardening candidate. Revise `markdown-download-exec` to cover sudo/env/wrapper prefixes, absolute interpreter paths, versioned interpreters, and line-continuation pipelines; add regression tests for those cases; then run the targeted scanner tests and appropriate changed gate before merge. Separately decide whether direct ClawHub skill folder installs should call the built-in scanner before writing into the workspace.

## Risks / Open Questions

- Closing this PR would leave current main with code-only built-in scanner coverage for skill install and audit paths; `SKILL.md` content would remain outside this scanner.
- Merging the PR as-is would improve Markdown coverage but leave easy false negatives in the critical download-and-execute rule for wrapped interpreters and line-continuation pipelines.
- Direct ClawHub skill folder installs remain a separate path from gateway-backed installer metadata scanning; maintainers should decide whether this PR stays narrowly about scanner file coverage or also wires scanning into that install path.
