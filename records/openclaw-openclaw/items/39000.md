## Summary

Current main still does not expose a plugin-facing `api.callGatewayMethod()` or `api.runtime.gateway.dispatchMethod`. Plugins can register gateway RPC handlers, and core has a private internal dispatcher used by specific runtime helpers, but the requested inter-plugin request/response seam remains unimplemented; related PR #46485 is still open and not merged.

## Best Possible Solution

Keep this issue open and route it through Plugin SDK/runtime design. The best path is to review, revise, or replace #46485 with a documented and tested plugin request/response seam, either a constrained `api.runtime.gateway.dispatchMethod` with explicit caller identity/scope semantics or a narrower typed capability/service API that satisfies inter-plugin reads without exposing the whole gateway method bus.
