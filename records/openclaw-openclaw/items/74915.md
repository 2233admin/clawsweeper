## Summary

Keep open. Current main still filters `web_fetch` provider resolution to bundled plugin candidates, while the docs expose `webFetchProviders` as a plugin contract; an open PR already references this issue with closing syntax, so this should stay open until that PR is reviewed and merged or closed.

## What This Changes

The issue reports that external plugin `webFetchProviders` are not considered by `web_fetch` provider resolution when Readability is disabled.

## Best Possible Solution

Review and land PR #74962, or an equivalent narrow runtime fix, so explicit/runtime web-fetch provider selection can resolve active external providers while preserving the sandboxed bundled-only constraint.

## Reproduction Assessment

Yes. The issue body's reproduction maps directly to current source: external providers are filtered out by bundled-origin resolution, and the Readability-disabled branch throws when provider resolution returns null.

## Solution Assessment

Yes in direction. The maintainable solution is a focused provider-resolution change using the existing runtime-provider seam, not a docs-only rollback of the public plugin contract.
