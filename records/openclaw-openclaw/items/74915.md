## Summary

Keep open. Current `main` still resolves `web_fetch` fallback providers from bundled plugin candidates, which matches the reported external-provider failure, and an open implementation PR already references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that `web_fetch` honor external plugin `contracts.webFetchProviders` and runtime-registered providers instead of limiting provider lookup to bundled plugins.

## Best Possible Solution

Review and land PR #74962, or an equivalent narrow patch, so non-sandboxed `web_fetch` resolves explicitly configured and runtime-selected providers across active plugin origins while preserving the existing SSRF, redirect, and sandboxed-provider restrictions.

## Reproduction Assessment

Yes. The issue body's reproduction maps directly to current source: external providers are filtered out by bundled-origin lookup before matching, and the Readability-disabled branch throws the reported error when provider resolution returns null.

## Solution Assessment

Yes, with review. The maintainable path is a focused resolver change using the existing runtime web-provider seams, plus regression coverage for external non-bundled providers and unchanged sandbox behavior.
