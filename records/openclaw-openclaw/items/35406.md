## Summary

Keep open. Current main still does not preflight PDF/Office document formats in the generic `read` wrapper before delegating to the upstream reader, while the open draft PR #57600 has explicit closing intent and is the active implementation candidate. The older #35494 attempt is closed unmerged, and current PDF/document-extraction support is adjacent rather than a fix for `read(path="*.docx")` or `read(path="*.pdf")`.

## Best Possible Solution

Keep this issue open and review/land #57600 or a maintainer-owned equivalent. The fix should add a small preflight in `src/agents/pi-tools.read.ts` before `executeReadWithAdaptivePaging`, reuse shared MIME detection for PDF and Office formats, cover both host and sandbox read paths, return a concise fallback that points PDFs to the `pdf` tool and Office files to parser/conversion tooling, and add regression tests for `.pdf`, `.docx`, `.xlsx`, and `.pptx`.
