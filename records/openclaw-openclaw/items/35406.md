## Summary

Keep open. Current main still routes generic `read` calls through the upstream text/image reader before any PDF or Office-document preflight, and the active implementation path is PR #57600, which is open, draft, and explicitly intended to close this issue. The older PR #35494 was useful prior context but is closed unmerged; the separate `pdf` tool helps PDF analysis but does not change `read(path="*.pdf")` or Office file behavior.

## What This Changes

Request that the `read` tool stop returning raw binary output for PDF and Office document files, either by returning a concise unsupported-file fallback or by extracting readable text.

## Best Possible Solution

Keep this issue open and review/land PR #57600 or a maintainer-owned equivalent. The fix should add a small preflight in `src/agents/pi-tools.read.ts` before adaptive paging, reuse shared MIME detection for PDF and Office formats, cover host and sandbox read paths, return a concise fallback that points PDFs to the `pdf` tool and Office files to parser/conversion tooling, and add regression tests for `.pdf`, `.docx`, `.xlsx`, and `.pptx`.
