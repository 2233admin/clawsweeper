## Summary

Keep this PR open. Current main still renders the Sessions Limit field as a string-backed plain input, while the PR is an unmerged implementation candidate that still needs rework around type consistency, blank/invalid input handling, and unrelated formatting churn.

## Best Possible Solution

Keep this PR open and request a focused revision. The final patch should either consistently migrate the Sessions limit filter to `number | undefined` across the view props, app state, controller state, initializer, render path, and tests, or keep editable string UI state and parse only at the controller/request boundary. It should use `type="number"`, `min="1"`, and `step="1"`, preserve a blank value while editing, never send invalid/non-integer limits to `sessions.list`, add targeted view/controller coverage, and drop unrelated formatting churn unless the repo formatter requires it.
