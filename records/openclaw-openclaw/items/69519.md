## Summary

Keep #69519 open. Current main still renders the Sessions Limit field as a plain text input and stores the filter as a string, so the PR's central user-facing fix is not implemented or otherwise superseded. The PR is recent and focused, but the provided review discussion identifies changes still needed before it is safe to merge.

## Best Possible Solution

Keep the PR open and ask for a narrow follow-up: remove unrelated CSS formatter churn, complete the limit type migration consistently across app state, view props, controller state, and tests, and decide whether empty/invalid input should be represented as undefined rather than a 0 sentinel before request params are built.
