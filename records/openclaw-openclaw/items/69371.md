## Summary

Keep open. Current main has adjacent ARM/low-power improvements, including configurable Gateway handshake budgets, startup-stage diagnostics, Pi setup guidance, global cron retries, outbound write-ahead delivery queueing, and preemptive compaction/byte guards, but it still does not implement the central lightweight sessions/status path, per-job cron retry contract, explicit background/retry message CLI, sessions_send processed receipt, or edge-device doctor profile requested here.

## What This Changes

Request improved OpenClaw stability on ARM64 edge devices through lightweight sessions/status commands, context-driven compaction controls, cron and outbound delivery retry behavior, sessions_send acknowledgements, and an edge-device doctor mode.

## Best Possible Solution

Keep this open and split it into focused maintainer follow-ups. Prioritize a measurable lightweight sessions/status path for large ARM-hosted stores, then decide whether existing reserve-token and transcript-byte compaction guards need a user-configurable context high-water policy. Handle per-job cron retry fields, explicit message send background/retry semantics, sessions_send receipt semantics, and an edge-device doctor profile as separate scoped changes. Treat #46097 as the already-addressed Gateway handshake slice.
