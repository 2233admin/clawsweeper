## Summary

Keep open. Current main has adjacent low-power/ARM improvements, including the configurable 15s gateway handshake timeout and startup optimization hints, but it does not implement the central request: lightweight large-session CLI paths, context high-water compaction policy, per-job cron retry overrides, durable/background message send retries, sessions_send delivery receipts, or an edge-device doctor mode.

## Best Possible Solution

Keep this issue open and split it into focused maintainer-owned follow-ups. Treat #46097 as the already-addressed handshake slice, then prioritize a measurable lightweight sessions/status path for large ARM-hosted stores, followed by separate decisions for context-pressure compaction policy, cron retry semantics, durable/background message delivery, sessions_send receipt semantics, and a Pi/edge-device doctor profile.
