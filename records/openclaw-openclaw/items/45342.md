## Summary

Keep this PR open. Current main still does not implement a TTL cache or `usage.cacheTtlMs` config for `loadProviderUsageSummary`, and the active status/usage paths still call the loader directly. The PR remains a plausible implementation candidate for the still-missing behavior, though it is stale against the newer plugin-owned provider-usage loader shape and would need rework before merge.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require a rebase or replacement patch against the current plugin-owned provider-usage path. The best fix should add a generic provider-usage cache seam in or near `src/infra/provider-usage.load.ts`, align config/schema/help/types if `usage.cacheTtlMs` is accepted, preserve injected config/env contracts, and add regression tests for cache hits, provider-list and agent/workspace/config boundaries, TTL expiry, disabled cache, explicit auth bypass, and non-caching of transient failures such as 429 or Timeout.
