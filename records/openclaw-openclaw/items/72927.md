## Summary

Keep open. Current main still exposes only Xiaomi MiMo preset-voice TTS plus optional style guidance; it does not read a `voiceDesign` config, register the VoiceDesign model id, document the schema, or build a VoiceDesign request body. The prior ClawSweeper comment is still accurate against the newer main snapshot.

## What This Changes

The issue requests provider-owned Xiaomi MiMo v2.5 TTS VoiceDesign support in `messages.tts.providers.xiaomi`, while preserving the existing `voice` behavior.

## Best Possible Solution

Extend only the bundled `xiaomi` speech provider with a documented VoiceDesign config shape, map it to Xiaomi's supported VoiceDesign request contract, leave preset `voice` and `style` behavior unchanged, and cover the new shape with focused provider tests plus TTS/provider docs.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: `voiceDesign` is not read, the VoiceDesign model id is absent, docs/tests omit it, and the request builder always sends `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi change is the narrowest maintainable path; a generic core TTS passthrough or SDK change would be broader than this Xiaomi-specific upstream contract requires.
