## Summary

Keep open. Current main supports Xiaomi MiMo TTS with preset voices and natural-language `style`, but it still has no `voiceDesign` config field, test coverage, docs, or request-body mapping; the upstream VoiceDesign API shape needs maintainer/API confirmation before a safe automated fix.

## What This Changes

Request to add Xiaomi MiMo v2.5 TTS VoiceDesign configuration support so the bundled Xiaomi speech provider can pass custom voice design parameters instead of only preset voices.

## Best Possible Solution

Confirm Xiaomi's official VoiceDesign API contract, then either add a provider-local `extensions/xiaomi` config/request mapping with focused tests and docs, or explicitly document that advanced VoiceDesign workflows should be handled by a ClawHub plugin while the bundled provider remains preset-voice plus style guidance.

## Reproduction Assessment

Yes for the in-repo config gap: static inspection of current main shows `voiceDesign` is not normalized, documented, tested, or included in the Xiaomi TTS request body. Live reproduction of actual VoiceDesign audio behavior remains unclear until Xiaomi credentials and the official request contract are available.

## Solution Assessment

Unclear. A bundled Xiaomi-provider change is the narrowest in-repo implementation if maintainers want first-party support, but the safer path is to verify the upstream schema before adding a public config field.
