## Summary

Keep this PR open. Current main still has the gateway inflight dedupe race the PR targets: `send` and `message.action` can await channel resolution after the inflight lookup and before registration, and `poll` still only checks completed dedupe state with no inflight path. The latest PR remains a focused implementation candidate, but the author also said it will be reworked, so maintainer review is the right next step rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The fix should synchronously claim/register the inflight promise before the first await for `send`, `message.action`, and `poll`; duplicate callers should await the same result with cached metadata; existing authorization, scope, channel selection, and dedupe cache semantics should be preserved; and focused concurrent regression tests should land in `src/gateway/server-methods/send.test.ts`.
