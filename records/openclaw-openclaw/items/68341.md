## Summary

Keep this PR open. Current main still has the gateway inflight dedupe race this PR targets: send and message.action register inflight work only after async channel resolution, and poll still has only completed-cache dedupe with no inflight path. The latest PR context remains a focused implementation candidate touching only gateway send source/tests, but the author also noted it will be reworked, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best implementation is to claim/register the inflight promise synchronously before the first await for send, message.action, and poll, have duplicate callers await the same result with cached metadata, preserve existing authorization/scope/channel semantics, and land focused concurrent regression tests in `src/gateway/server-methods/send.test.ts`.
