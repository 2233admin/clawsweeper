## Summary

Keep this PR open. Current main at f6d23ab5c2d8e7cbf50522b6c8137241524b6646 still has the gateway inflight dedupe race this PR targets: `send` and `message.action` can await channel resolution before the inflight promise is registered, and `poll` still has completed-result dedupe only. The latest provided PR context is focused on `src/gateway/server-methods/send.ts` and `src/gateway/server-methods/send.test.ts`, and the author has said it will be reworked, so maintainer review or replacement is still the right outcome.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The fix should synchronously claim/register the inflight promise before the first await for `send`, `message.action`, and `poll`; duplicate callers should await the same result with cached metadata; all three handlers should consistently handle cached, inflight, and ready states; existing authorization, owner propagation, scope forwarding, channel selection, and dedupe cache semantics should be preserved; and focused concurrent regression tests should land in `src/gateway/server-methods/send.test.ts`.
