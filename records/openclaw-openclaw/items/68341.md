## Summary

Keep open. Current main still does not implement the PR's central gateway inflight dedupe for poll, and send/message.action still have a race window before inflight work is registered. The latest PR diff is not obsolete, but it remains too mixed for cleanup closure: it bundles gateway dedupe, cron trust, failover classification, and maintenance-cache changes, while the MCP claim in the body is already handled on main and no longer appears in the final diff.

## Best Possible Solution

Keep the PR open, but ask for a narrower maintainer-reviewable version focused on the gateway idempotency race. The best path is to keep the poll/send/message.action inflight registration changes with focused concurrent-duplicate tests, and drop or split the cron, failover, MCP-body, and maintenance-cache pieces unless each has a concrete reproduction and aligned tests.
