## Summary

Keep this PR open. Current main still has the gateway inflight dedupe race the PR targets: `message.action` and `send` register the inflight promise only after async channel/setup work, and `poll` still has completed-cache dedupe without an inflight path. The PR remains a focused implementation candidate, but the author also said it will be reworked, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best implementation is to claim/register the inflight promise synchronously before the first await for `send`, `message.action`, and `poll`, have duplicate callers await the same result with cached metadata, preserve existing authorization/scope/channel semantics, and land focused concurrent regression tests in `src/gateway/server-methods/send.test.ts`.
