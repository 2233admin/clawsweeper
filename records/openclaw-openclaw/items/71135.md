## Summary

Keep #71135 open. Current main does not implement the proposed per-session working-context capsule or an equivalent persisted workingContext surface. The PR is a recent draft that adds a focused persistence primitive for the still-open #67511 pain point, and the related issue discussion includes a concrete user report of compaction losing environment details. It is not obsolete, duplicate, or ClawHub-only; it needs maintainer design review on storage shape and integration order.

## Best Possible Solution

Keep the PR open and route it through normal maintainer review for #67511. The best path is to decide the storage boundary first (sidecar capsule versus SessionEntry/transcript-backed workingContext), then either merge the primitive with clear follow-up tracking or ask the author to add the first integration point: update working context from command/session metadata, expose it in session_status and /status, and document the compaction/new-session behavior.
