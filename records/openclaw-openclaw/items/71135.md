## Summary

Keep this draft PR open. Current main still lacks the requested working-context capsule and the linked umbrella remains open; the patch is a useful implementation candidate, but it needs maintainer schema/privacy review and a concrete writer race fixed before merge.

## What This Changes

The PR adds `src/agents/session-working-context.ts` plus colocated Vitest coverage for resolving, normalizing, reading, writing, and clearing a JSON working-context capsule beside a session transcript.

## Best Possible Solution

Decide the durable schema and privacy model, fix the writer temp-file collision, then either land this primitive with focused tests as the storage contract or fold it into the first integration PR that updates and exposes working context across runtime, status, compaction, and new-session flows.

## Reproduction Assessment

Yes. The current-main gap is reproducible by targeted repository search plus inspection of session metadata, status rendering, post-compaction, successor transcript, and subagent spawn paths; the writer race is statically reproducible from the PR diff because concurrent same-process writes can share one temp path.

## Solution Assessment

Unclear as proposed. A small storage primitive is a reasonable first step, but the PR should not merge until the schema/privacy decision is made and the temp filename is unique per write; follow-up runtime wiring is required for the user-visible issue.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use a unique temp path for each context write:** `src/agents/session-working-context.ts:176`
  - body: `writeWorkingContext` derives the temp filename from only `process.pid` and `Date.now()`. Two writes to the same session in the same process and millisecond can use the same temp file, causing one rename/unlink to race the other and either fail persistence or write the wrong final capsule. Use `randomUUID()` or an existing atomic JSON helper so each call gets a unique temp path.
  - confidence: 0.87
