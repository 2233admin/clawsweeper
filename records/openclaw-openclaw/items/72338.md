Keep open. Current main has several relevant mitigations for the reported Gateway CPU and event-loop starvation cluster, including runtime-deps staging changes, subagent registry read caching, client readiness waits, and stuck-session diagnostics. Keep this issue as the broad Gateway CPU/liveness tracker until maintainers retest a packaged current build on affected hosts and can tie idle CPU, loopback probes, and one Telegram/WebChat turn to a specific shipped fix.
