## Summary

Keep this PR open. Current `origin/main` still builds Telegram model picker fallback labels from the provider-local `model` string, so an OpenRouter ref such as `openrouter/openai/gpt-5.4-mini` still displays as `openai/gpt-5.4-mini` when `modelNames` has no display-name entry. The PR is a narrow implementation candidate for #67792, with only Telegram label/test changes and no security-sensitive files in the provided diff; maintainer review should decide whether the long nested-label truncation concern belongs here or in a follow-up.

## Best Possible Solution

Keep this PR open as the implementation candidate for #67792. The best path is to review and land the narrow fallback-label/test fix if maintainers accept the current truncation boundary, or request one small follow-up in this PR that preserves the provider segment for long nested labels before merging.
