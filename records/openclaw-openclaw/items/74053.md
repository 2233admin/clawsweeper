## Summary

Keep this PR open. Current main still builds the heartbeat runtime context with `To: sender`, so the PR's intended `ctx.To = delivery.to` change is not implemented. Main does have broader session-state guards that preserve existing `lastTo` for heartbeat/cron/exec system-event turns, which may mitigate the described persisted-route corruption, but that is not equivalent to the runtime-context fix proposed here.

## Best Possible Solution

Review this PR as a narrow heartbeat routing hardening change. If maintainers agree that `ctx.To` should represent the actual delivery target rather than the synthetic sender identity, land the one-line runtime change with the regression test tightened to assert `undefined` for the no-target case. If maintainers decide the existing system-event session guard is sufficient, close this PR with that explicit rationale rather than as already implemented.
