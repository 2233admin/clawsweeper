## Summary

Current main already fixes the PR's observable data-corruption bug through the shipped system-event route-preservation path. `runHeartbeatOnce` still sets `ctx.To` from `sender`, but heartbeat/cron/exec system-event turns no longer persist `ctx.To` through `resolveLastToRaw`; they preserve the existing session `lastTo` and delivery context instead. The broader fix shipped in v2026.4.14 via #66073, so this PR's narrower `ctx.To` change is obsolete.

## Best Possible Solution

Close this PR as already implemented by the shipped #66073 route-preservation fix. Keep the current main behavior that preserves session delivery metadata for heartbeat/cron/exec system events; handle any future desire to make `ctx.To` less surprising as a separate small cleanup, not as a bug fix for the already-fixed persisted `lastTo` corruption path.
