## Summary

Keep open. Current main still lacks the requested operator-facing Dreaming session/cron exclusion config, and the explicitly paired implementation PR is open and unmerged, so this issue should remain available for maintainer review rather than be closed or given a competing repair job.

## What This Changes

The issue requests reliable exclusion of isolated cron transcripts from Dreaming session-corpus ingestion plus configurable Dreaming filters for cron jobs, session-key prefixes, agents, and source paths.

## Best Possible Solution

Review and land #72913, or replace it with a narrower equivalent that adds a documented Dreaming session filter contract, keeps cron transcript exclusion reliable for live and archived artifacts, and covers strict cron-run keys, base cron keys, and archive handling with regression tests.
