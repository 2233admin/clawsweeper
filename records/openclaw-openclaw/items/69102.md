## Summary

Current main still does not expose Gemini Flex service-tier configuration for the bundled Google provider. The direct Gemini transport builds REST JSON itself, and the current extra-param path only forwards shared options plus Google cachedContent, not serviceTier/service_tier.

## Best Possible Solution

Implement this in the bundled Google provider: accept agents.defaults.models["google/<model>"].params.serviceTier and service_tier for direct google-generative-ai requests, validate at least flex, emit top-level REST service_tier: "flex", preserve caller/onPayload payload values, add focused transport and extra-param tests, update Google provider docs, and add a user-facing changelog entry crediting @adriandubiel.
