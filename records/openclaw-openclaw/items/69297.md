## Summary

Keep open. Current main does not implement WhatsApp per-group admin privacy controls: the public WhatsApp group config type and strict schema lack `admin`, inbound policy has no configured-admin hook, and the real group ingress plus activation paths are not sender-admin-aware. The open PR is still the active implementation candidate for the linked WhatsApp privacy feature, not obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a focused WhatsApp implementation that adds exact-group and wildcard `admin` config across type/schema/generated metadata/docs/UI hints plus ingress, mention activation, ack behavior, command authorization, and regression tests, while splitting or justifying the unrelated QA/test/channel stabilization changes before merge.
