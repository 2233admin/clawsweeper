## Summary

Keep open. Current main still lacks configurable provider-level retry, and this PR is a live implementation path, but the branch is not merge-ready because it misses a zero-provider cleanup path, needs to preserve current OpenAI SDK request options, and does not update generated config schema artifacts for the new public config field. The overlapping open #38640 should be reconciled by maintainers rather than auto-closing either implementation.

## What This Changes

The branch adds opt-in `models.providers.<name>.retry` config, a provider retry runner registry and policy, abort-aware retry support, OpenAI-compatible transport wrapping, config help/labels/types/schema updates, and focused retry tests.

## Best Possible Solution

Update this implementation, or a single canonical replacement, so provider retry is opt-in, scoped correctly through config reloads, preserves existing timeout/request-option behavior, updates generated config and docs surfaces, and has targeted regression coverage before #38640 is resolved.
