## Summary

Current main still has the reported gap. `sessions_send` forwards a timeout-side `error` only if `agent.wait` supplies one, but the gateway wait path can still return a bare `{ status: "timeout" }` when lifecycle/dedupe waiting expires before a pending lifecycle/model error is promoted out of the retry-grace window.

## Best Possible Solution

Keep this issue open and implement a narrow gateway-side change: retain recent lifecycle/model failure context while `waitForAgentJob` is in its retry-grace state, then include a sanitized `error` or `reason` on the `agent.wait` timeout response. The existing `sessions_send` pass-through can then expose it. Add targeted tests for a target run that emits a model/runtime error before the caller wait budget expires, a transient error followed by retry/start that should not surface prematurely, and a true no-error timeout.
