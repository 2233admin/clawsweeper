## Summary

Keep open. Current main still has the pre-tool hook and trusted policy primitives, but it does not implement the requested framework-owned session PIN gate: there is no `session.auth.pin` config/schema field, no core authenticated bit on `SessionEntry`, no trusted TypeScript verifier/unlock path, and no session-auth decision in the pre-tool execution path. Because this is a security-sensitive core behavior and external plugins cannot register trusted tool policies, this needs maintainer design rather than conservative cleanup closure.

## Best Possible Solution

Keep this open for maintainer design. The best path is to add a documented session-auth config using existing secret/redaction conventions, persist a framework-owned authenticated flag on `SessionEntry`, clear it on `/new` and `/reset`, enforce it in the trusted pre-tool path before ordinary plugin hooks, and add a native command, gateway method, or internal verifier that compares PIN attempts in TypeScript outside model context. Cover config/schema validation, before-tool blocking, reset lifecycle, transcript/log redaction, and security docs.
