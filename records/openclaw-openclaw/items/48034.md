## Summary

Keep this PR open for maintainer handling rather than auto-closing. Current main still does not implement the intended rw-mode skill sync: `workspaceAccess: "rw"` selects the real agent workspace and bypasses the sandbox skill-sync branch. The submitted PR is also not mergeable as-is because the reviewed diff keeps the sync under that same branch and imports `resolveBundledSkillsDir` from a module that does not export it. The unrelated locale dropdown hunk is already implemented on main.

## Best Possible Solution

Keep the underlying bug work active, but replace or revise this PR rather than landing it as-is. The best fix likely belongs in `src/agents/sandbox/context.ts`: ensure the sandbox workspace skill mirror is prepared for rw sessions too, make prompt locations point at readable sandbox-workspace copies, preserve existing eligibility/filtering behavior, and add a targeted rw-mode regression test. The already-shipped locale dropdown change should stay separate.
