## Summary

Keep this PR open. Current main fixed the default Docker image root cause from the linked sandbox reports, but it did not implement this PR's intended mutation-error translation layer. The local Docker helper now emits a clearer missing-Python helper error and default-image setup fails earlier, but the remote shell bridge still invokes `python3` directly and there is no current `toFriendlySandboxMutationError`-style wrapper or regression coverage for the actionable guidance proposed here. Security review found no supply-chain or permission-expansion concern in the PR diff itself.

## Best Possible Solution

Refresh the PR against current main: reuse or adapt main's newer pinned-helper contract, preserve original errors with `cause`, add local and remote regression tests for missing-Python mutation failures, and decide whether the guidance should be Docker-specific repair text or generic custom-image/runtime guidance. If maintainers decide the merged #73362 default-image fix is enough, close this PR manually as superseded with that explicit product decision.
