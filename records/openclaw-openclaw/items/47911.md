## Summary

Keep open. Current main has several separate observability commands, but it still does not provide the requested unified `openclaw fleet status` command or equivalent all-in-one JSON surface across agents, provider/auth health, channels, active/stuck sessions, services, and recent errors. The related observability PR #50066 and signal-layer proposal #50072 are closed and did not land a replacement for this request.

## Best Possible Solution

Keep this open for maintainer product/design review. The best path is to define a single supported observability surface, likely either `openclaw fleet status --json` or an expanded `openclaw status --all --json`, that composes existing read-only status, sessions, tasks, models/auth, channel, service, and log summaries with clear redaction rules and tests. If maintainers prefer this outside core, first confirm the plugin/CLI APIs expose enough stable, non-secret state for a ClawHub plugin to implement it.
