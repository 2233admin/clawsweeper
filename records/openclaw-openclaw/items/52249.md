## Summary

Keep open. The old direct-resume root cause is stale, but current main still routes ACP parent-stream completion through session-scoped system events plus heartbeat wake delivery, and disabled or non-target heartbeat paths can still skip before a yielded parent gets another turn. The related #60402 is still the open implementation candidate, so closing now would be premature.

## What This Changes

The issue asks for ACP child completion to wake a yielded parent session without requiring a manual UI refresh or depending on optional heartbeat delivery.

## Best Possible Solution

Ship or replace the open implementation with a shared yielded-parent completion delivery path that does not depend on optional heartbeat scheduling, and add regression coverage for ACP completion waking a yielded parent when the parent agent is not heartbeat-enabled or heartbeat scheduling is disabled.
