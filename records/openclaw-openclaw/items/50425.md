## Summary

Keep open. Current main has related provider replay sanitization for thinking blocks, but it does not implement this PR’s intended `sessions_history` output fix: the tool still returns `thinking` content blocks after truncating/redacting them, and there is no current `redacted_thinking` stripping path for the session-history tool output. The PR also has unresolved review risks that need author/maintainer handling before it can safely land.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require an update against current main. The best fix should strip `thinking` and `redacted_thinking` from `sessions_history` tool output, preserve assistant turn shape when all blocks are omitted, avoid a public bypass that returns provider-owned reasoning, resolve the reported syntax/review issues, and add focused regression tests for both the direct tool result and any sessions_list preview behavior the PR changes.
