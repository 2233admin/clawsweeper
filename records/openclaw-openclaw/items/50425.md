## Summary

Keep open. Current main still does not implement this PR’s central behavior: `sessions_history` keeps `type: "thinking"` content blocks in the returned payload after truncating/redacting them, has no `redacted_thinking` omission path, and tests still assert that a returned `thinking` block exists. The linked bug context in #50415 and the review comments remain relevant; this PR is still a plausible implementation candidate, but it needs updates before it can land safely.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to update it against current main so `sessions_history` strips `thinking` and `redacted_thinking` content blocks from returned assistant messages, preserves valid assistant turn shape when all blocks are omitted, keeps any replay-preservation behavior out of the public tool schema, decides whether `sessions_list` previews need sanitized/backfilled behavior, and adds focused regression coverage for direct tool output plus any preview behavior changed.
