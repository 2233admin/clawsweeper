## Summary

Keep this issue open. Current main still has the active secrets RPC surface, adjacent SecretRef documentation, and SDK lock/atomic JSON primitives, but it does not yet contain the requested focused `src/gateway/server-methods` secrets guidance or a central plugin credential-persistence contract; the same-author paired PR #71156 is still open and unmerged as the active docs implementation candidate.

## What This Changes

The issue requests focused contributor documentation for credential governance, including secrets RPC invariants and plugin credential-persistence guidance around locks, atomic writes, TTL/lifecycle behavior, and no-silent-success failure semantics.
