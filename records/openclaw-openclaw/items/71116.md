## Summary

Keep open. Current main still has the active secrets RPC surface, SecretRef runtime docs, SDK file-lock/json-store primitives, and distributed plugin credential stores, but it does not yet include the requested secrets-specific server-methods guidance or a public plugin credential-persistence contract; the open paired PR #71156 is the right active path for maintainer scope review rather than a close or duplicate repair.

## What This Changes

The issue requests documentation of OpenClaw's credential-persistence contract plus scoped guidance for gateway secrets and extension credential hotspots, with optional CI token-scope audit and shared credential-helper follow-ups.

## Best Possible Solution

Land or revise the active docs work for the secrets server-methods invariants, add a public plugin credential-persistence contract that ties SecretRef runtime behavior to plugin-owned credential stores and SDK storage helpers, and split the CI audit or optional helper into narrower maintainer-approved follow-ups if those parts are still desired.
