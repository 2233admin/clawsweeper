## Summary

Keep open. Current main does not implement spoken Discord voice model escalation, so this PR is not obsolete, but the branch is not merge-ready: it targets the old core Discord path, mixes in a gateway registry fix already shipped by #47902, and the remaining voice feature has correctness and security-policy problems that need maintainer direction.

## What This Changes

The PR adds hardcoded spoken Discord voice commands for persistent session model escalation/de-escalation and also adds a gateway plugin HTTP registry bridge with regression tests.

## Best Possible Solution

If maintainers want spoken voice mode switching, it should come as a fresh, narrow Discord plugin change under `extensions/discord/src/voice` with explicit low-false-positive commands, command consumption before agent dispatch, correct first-turn session persistence, configured or allowlisted model targets instead of hardcoded cloud defaults, focused tests, and Discord docs. The gateway registry changes should be dropped because #47902 already covers that path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Require explicit voice mode commands:** `src/discord/voice/manager.ts:84-85`
  - body: The medium-tier pattern matches common conversational speech, so phrases like "think about this approach" can silently downgrade the session model. Spoken model switches need explicit command phrases with low false-positive risk.
  - confidence: 0.91
- **[P2] Consume matched voice commands before dispatch:** `src/discord/voice/manager.ts:692-693`
  - body: After applying an escalation trigger, the code falls through to `agentCommandFromIngress`, so a pure command such as "go deep" both changes the model and gets sent to the agent as a normal query.
  - confidence: 0.9
- **[P2] Persist overrides for missing session entries:** `src/discord/voice/manager.ts:783-785`
  - body: The session-store update returns when the session entry is absent, so the first spoken escalation in a fresh voice session is dropped before the agent command path creates the entry later in the turn.
  - confidence: 0.84
- **[P3] Allow clearing the gateway registry without casts:** `src/plugins/runtime.ts:58-59`
  - body: The new getter returns `PluginRegistry | null`, but the setter only accepts `PluginRegistry`, forcing test cleanup to use `null as never`. Widening the setter would keep the exported contract consistent.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff has no dependency, CI, or external code execution changes, but it introduces persistent hardcoded cloud model switching from transcribed speech.

Concerns:

- **[medium] Avoid hardcoded cloud routing from incidental speech:** `src/discord/voice/manager.ts:78`
  - body: A matched transcript can persistently switch future turns to hardcoded Anthropic models, and one trigger is broad enough to fire during ordinary conversation. That can change privacy, cost, and provider-policy behavior without an explicit configured allowed-model decision.
  - confidence: 0.88
