## Summary

Close as ClawHub/plugin scope: this is a vendor-specific beta SDK runtime request, while current main intentionally keeps core lean and already exposes plugin-owned agent harness and ACP runtime seams for this class of integration.

## What This Changes

Add `@cursor/sdk` as a first-class OpenClaw agent execution backend with config, auth, dispatch, and failover support for Cursor local and cloud runtimes.

## Best Possible Solution

Keep OpenClaw core generic and let a Cursor SDK integration ship as a trusted native plugin or ClawHub package that registers a provider plus agent harness, or an ACP runtime backend if that abstraction fits. Open a narrower core issue only if that implementation proves a missing generic plugin API.

## Reproduction Assessment

Not applicable; this is a feature request, not a reported bug. The verification path is a source/docs scope check: current main lacks a core `@cursor/sdk` implementation but already has plugin harness and ACP extension points for this class of integration.

## Solution Assessment

Yes. Treating this as plugin/ClawHub work is the narrowest maintainable direction because current main already exposes generic harness registration and runtime selection, while adding `@cursor/sdk` directly to core would hardcode a vendor-specific beta/commercial integration.
