## Summary

Keep this PR open. Current main still does not implement the requested DM pairing owner notification flow: fresh DM pairing writes pending state and replies to the requester, but there is no `pairing:request` event, no owner notification dispatch, and no top-level `pairing.notify` config surface. The PR is a real implementation candidate, but it needs maintainer review/rebase because it touches security-sensitive pairing and outbound messaging behavior and its startup hook target has drifted from current main.

## Best Possible Solution

Keep this PR open as the active implementation candidate, or replace it with a maintainer-owned version. The best path is a rebased, documented, versioned pairing request hook emitted only for fresh DM pairing requests, plus owner notification with sanitized fixed-format content, channel target normalization, PII-safe logs, schema/docs alignment, and focused tests around the shared pairing path.
