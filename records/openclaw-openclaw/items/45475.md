## Summary

Keep open. Current main still has no `channels.slack.ignoreSubtypes` config surface, and `message_changed` / `message_deleted` are still intentionally resolved as Slack subtype system events that can route to an agent session. The related `thread_broadcast` cross-reference is already fixed on main, but that does not supersede this request because edit/delete subtypes remain mapped to system events.

## Best Possible Solution

Keep this issue open and implement an opt-in Slack account config key such as `channels.slack.ignoreSubtypes`. The fix should default to current behavior, validate/document the key, and drop configured raw Slack message subtypes before system-event session routing/enqueue so configured edit/delete events do not wake an agent session.
