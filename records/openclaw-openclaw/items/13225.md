## Summary

Current main still does not implement the requested pre-tool-call model/session routing hook. `before_tool_call` can rewrite params, block, or require approval, and it exposes useful tool context, but it has no `precedingModelOutput`/`precedingText` field and no redirect/model/session result. `before_model_resolve` is an earlier prompt-only model-selection seam, not a hook over a pending assistant tool call. Open PR #66147 is a related implementation candidate for split tool-turn model routing, but it is open and not equivalent to this issue's plugin hook redirect contract.

## Best Possible Solution

Keep this issue open as a plugin/runtime API design request. The best path is an additive typed pre-tool routing contract, either extending `before_tool_call` or adding a separate hook, that defines what assistant context can be inspected and how a pending tool call may be redirected to another provider/model/session with clear audit, permission, auth-profile, fallback, and failure behavior. Preserve the existing params rewrite/block/approval behavior for narrower cases like per-agent TTS argument injection.
