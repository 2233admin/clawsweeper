## Summary

Current main still does not implement the requested pre-tool-call model/session routing contract. `before_tool_call` can rewrite params, block, or require approval, and it now has useful context such as `ctx.agentId` for cases like per-agent TTS argument injection, but it cannot inspect a pending assistant tool call and redirect that call to another model/session. `before_model_resolve` is earlier, prompt/attachment-only model selection. Related PR #66147 is an open implementation candidate for split tool-turn routing; #58079 and #62701 are context-field work and do not supersede the redirect semantics.

## Best Possible Solution

Keep this issue open as a plugin/runtime API design request. The best path is an additive typed pre-tool routing contract, either extending `before_tool_call` or adding a separate hook, that defines what assistant context can be inspected and how a pending tool call may be redirected to another provider/model/session with clear audit, permission, and failure behavior. Preserve the existing params rewrite/block/approval behavior, since that already covers narrower cases like per-agent TTS argument injection.
