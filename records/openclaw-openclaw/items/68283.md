## Summary

Keep this PR open. Current main partially addresses the reported pairing-required storm by pausing terminal gateway reconnect failures, but it does not implement this PR's bounded exponential retry behavior for transient native approval handler bootstrap failures. The PR is also stale against current main's terminal-startup handling, so it needs maintainer review/adaptation rather than being closed as implemented or queued as a straight automated replacement.

## What This Changes

The PR changes native approval handler bootstrap startup failures from a fixed 1-second retry loop to exponential backoff with a 300-second PAIRING_REQUIRED delay, pairing-action diagnostics, retry-state resets, and focused tests.

## Best Possible Solution

Adapt the transient retry backoff to current main while preserving the deliberate terminal handling for PAIRING_REQUIRED unless maintainers choose a different operator-recovery policy. Add tests that cover repeated transient failures, retry-state reset, existing-context startup, and the current terminal pairing-required path.
