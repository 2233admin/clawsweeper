## Summary

Keep open. Current main has adjacent session-write-lock hardening, including a typed timeout error and failover suppression, but it still does not implement this PR's central timeout propagation from embedded agent attempts into the session write lock, and the plugin SDK export/API shape still needs maintainer judgment.

## What This Changes

The PR passes `params.timeoutMs` into embedded session write-lock acquisition, replaces the generic lock-timeout throw with a `SessionWriteLockError` carrying `reason: "session_busy"`, and adds a session-lock unit test for that error shape.

## Best Possible Solution

Keep this PR open for maintainer review or contributor update. Rebase it onto current main, decide whether embedded attempts should use `params.timeoutMs` as the session-lock wait or a separate/configurable wait, reuse or deliberately replace the existing `SessionWriteLockTimeoutError`/code/helper, and add focused tests for timeout propagation plus the supported SDK export if that error is meant to be public.
