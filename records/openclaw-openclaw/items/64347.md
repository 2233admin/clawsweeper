## Summary

Keep open. Current main still lacks a native warm-reuse path for `sessions_spawn`: native subagent spawning creates a fresh UUID-backed child session key, the tool/config surfaces expose no native reuse key or pool selector, and docs/tests explicitly keep `resumeSessionId` ACP-only. The related merged PRs #64468 and #72331 addressed adjacent cache-marker and ACP-field behavior, not this native subagent reuse contract.

## What This Changes

The issue requests a native `sessions_spawn` warm-reuse mechanism so repeated delegations can reuse a subagent session instead of always creating a fresh UUID-backed session.

## Best Possible Solution

Define an explicit native subagent reuse contract before implementation: choose opt-in reuse versus automatic pooling, model the isolation/history tradeoff, add schema/types/docs for the chosen API, and cover native `runtime="subagent"` behavior with focused unit tests around key selection, liveness, cleanup, and context preparation.
