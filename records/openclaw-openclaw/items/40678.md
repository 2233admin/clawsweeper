## Summary

Keep open. Current main still preserves the `/new` TUI session isolation that caused the requested regression in prior single-user monitoring workflows, but it has not added an explicit TUI observe or mirror mode. The existing Gateway session subscription primitives could support part of the design, but the TUI does not consume them for a same-agent cross-channel activity view, and the related broader cross-channel sync issue remains open rather than merged or shipped.

## What This Changes

Request an opt-in TUI mode that mirrors or observes same-agent activity from iMessage, Telegram, cron jobs, and other channel sessions without undoing isolated `/new` TUI sessions.

## Best Possible Solution

Add an explicit authorized TUI/Gateway observe path that preserves default `/new` isolation, labels channel/session/run origin clearly, separates read-only observation from interactive sending, and covers aggregate live events plus recent-history previews with focused TUI and Gateway tests.
