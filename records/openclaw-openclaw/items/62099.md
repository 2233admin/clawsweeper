## Summary

Keep open. Current main still has the request-visible failure path: successful embedded runs await auth-profile bookkeeping, lock-path persistence failures fall into uncaught direct saves, and the provided context shows #67077 is an open, unmerged PR with closing syntax for this issue.

## What This Changes

The issue reports that Windows EPERM failures writing auth-profiles.json make post-request auth-profile bookkeeping fatal and can cascade through all configured model fallbacks.

## Best Possible Solution

Review and land #67077, or an equivalent focused patch, so markAuthProfileGood, markAuthProfileUsed, markAuthProfileFailure, and cooldown bookkeeping catch and warn-log auth-profiles.json persistence failures while credential and OAuth token writes continue to fail loudly. Add EPERM regression coverage for both lock-guarded and direct fallback save paths.
