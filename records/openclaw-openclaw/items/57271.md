## Summary

Keep open. Current main still rejects external `sandbox.docker.binds` sources by default when runtime bind source roots are set to the sandbox workspace and agent workspace. There is a break-glass `dangerouslyAllowExternalBindSources` path, but the sandboxing docs still show arbitrary host-path examples without explaining that override in the custom-bind section, and related least-privilege bind work remains open in #59718.

## Best Possible Solution

Keep #57271 open. Maintainers should either update `docs/gateway/sandboxing.md` to make `dangerouslyAllowExternalBindSources` explicit next to external bind examples, or intentionally change the product behavior if external non-dangerous bind sources should be accepted by default. The broader writable-bind-with-`workspaceAccess: "none"` design should continue in #59718 rather than being treated as a fix for this docs/runtime mismatch.
