## Summary

Keep this PR open. Current main still has no `safetyPosture` config surface, so the PR is not obsolete or implemented-on-main. The latest PR head is a plausible implementation candidate, but it still needs rebase plus product/security review because the preset only partly expands into current runtime policy and some advertised safety fields are not enforced.

## Best Possible Solution

Keep this PR open as the active implementation candidate, but require a revision before merge: rebase onto current `main`, move preset expansion into the current `materializeRuntimeConfig` flow, translate `full`/`limited`/`public` into actual existing tool policy or add a real typed tool-profile contract, honor `secureDmMode`, map memory posture to concrete memory config or remove that promise, update docs/schema/help, and add tests against config load plus effective sandbox/tool/memory behavior.
