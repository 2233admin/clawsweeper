## Summary

Keep open. Current main does not implement the requested `safetyPosture` config surface or the development/balanced/strict preset expansion. Main has the underlying sandbox, DM-scope, and tool-profile knobs, plus security audit guidance and local-onboarding DM isolation, but that is not the same as this PR’s intended additive preset feature. The latest PR diff is a plausible implementation candidate, but it is incomplete because it adds schema/types without generated schema output or runtime application of the preset.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a revision before merge: add additive runtime handling that expands the selected posture into existing `agents.defaults.sandbox`, `session.dmScope`, tool policy, and memory behavior; regenerate the base config schema/help metadata; document the public-facing config surface; and add focused config/runtime tests. If maintainers decide not to add a top-level preset abstraction, close only after recording that product decision and pointing contributors to the existing explicit sandbox/tool/session knobs.
