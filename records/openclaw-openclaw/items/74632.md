## Summary

Keep open. Current main still lacks a first-class session-owned runtime envelope in the gateway session RPC contract, and the same-author open PR #74799 is already the active implementation candidate, so this should stay available for maintainer review rather than being closed or duplicated.

## What This Changes

The issue requests an additive gateway protocol envelope on `sessions.create` and `sessions.patch` so external clients can declare per-session tool, path, network, and optional authorization policy that OpenClaw enforces before tool execution.

## Best Possible Solution

Keep this issue open while maintainers review #74799 or replace it with an accepted narrower design. The durable end state should be an additive documented protocol field, persisted session state, pre-execution enforcement that cannot be bypassed by hook or approval rewrites, focused create/patch/tool-call tests, and a separate explicit design for network enforcement or external callbacks if those cannot be enforced reliably in the same layer.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: include an `envelope` property in a `sessions.create` or `sessions.patch` gateway RPC payload and current main rejects it because both schemas omit the field and disallow additional properties.

## Solution Assessment

Unclear. A session-owned envelope is a plausible maintainable boundary for the reported gap, but the exact enforcement layer plus network/callback scope need maintainer security and product review; #74799 is the active candidate to evaluate.
