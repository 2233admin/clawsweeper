## Summary

Keep open. Current main still lacks a session-owned runtime envelope in the Gateway session RPC contract, and same-author PR #74799 is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Add an optional session-owned runtime policy envelope to `sessions.create` and `sessions.patch` so Gateway clients can declare and update per-session tool, path, network, and authorization constraints.

## Best Possible Solution

Keep this issue open while maintainers review #74799 or replace it with an accepted narrower design. The durable end state should be an additive protocol field with typed schema/docs, persisted `SessionEntry` state, pre-execution enforcement that survives policy/hook/approval parameter rewrites, focused create/patch and tool-blocking tests, and explicit follow-up decisions for network enforcement and external callbacks if those cannot be enforced reliably in the same layer.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: send `envelope` in a `sessions.create` or `sessions.patch` payload and the current TypeBox schemas reject it because the field is absent and additional properties are disabled; no live gateway test was run because this review was read-only.

## Solution Assessment

Unclear. A session-owned envelope is a plausible product boundary for the reported gap, but the exact enforcement layer and network/callback scope need maintainer security review; #74799 is the active candidate to review rather than supersede with a separate automated repair.
