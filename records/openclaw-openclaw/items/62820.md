The originally linked PR was closed unmerged after review noted its shared-helper approach missed MiniMax XML, but later WhatsApp outbound work now routes every WhatsApp-visible outbound path through assistant-visible/tool-XML sanitizers and covers auto-reply, sendPayload, and channel outbound normalization with regression tests. ## What This Changes

The issue requests stripping text/XML-form tool-call scaffolding from user-visible messaging replies, especially WhatsApp replies from models that emit MiniMax-style or XML tool calls as plain text.

## Best Possible Solution

Keep the current channel-boundary implementation: WhatsApp should continue sharing one outbound normalization stack for auto-reply, sendPayload, and direct channel delivery, while the existing assistant-visible sanitizer remains the shared place for MiniMax and generic tool-XML text cleanup.

## Reproduction Assessment

Yes. The issue provides a concrete raw XML reply shape, and current tests exercise the WhatsApp delivery/sendPayload paths plus the MiniMax assistant-visible sanitizer path; I did not run tests because this review was required to stay read-only.

## Solution Assessment

Yes. Normalizing at the WhatsApp outbound boundary is the narrow maintainable fix for the reported user-visible leak because it covers every WhatsApp-visible path, including payloads that bypass the older sanitizeUserFacingText helper.
