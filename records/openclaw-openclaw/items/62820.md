## Summary

Close as implemented on main. The observable WhatsApp/messaging leak is now covered by the canonical assistant-visible sanitizer and WhatsApp outbound normalization, and the earlier unmerged `sanitizeUserFacingText`-specific PR was superseded by later merged work on the actual delivery surfaces.

## What This Changes

The issue requests that XML-style text tool calls emitted by models be stripped from user-facing messaging output instead of being delivered as raw markup.

## Best Possible Solution

Keep the current main implementation: sanitize assistant-visible model text at extraction time and keep WhatsApp-specific final outbound normalization in the WhatsApp plugin. Any desire to make `sanitizeUserFacingText` a broader canonical XML scrubber should be tracked as a separate refactor, not as this user-visible bug.

## Reproduction Assessment

Yes. The original static path is reproducible from the report, and current main has focused tests proving raw tool-call XML is stripped in assistant extraction and WhatsApp outbound delivery; I did not run tests because this review was read-only.

## Solution Assessment

Yes. The current fix is broader and safer than the one-line helper patch because it uses the canonical assistant-visible sanitizer for model text and a channel-owned WhatsApp final sanitizer for plural XML wrappers and delivery edge cases.
