## Summary

Keep this issue open. Current main still renders the Control UI Sessions `Updated` column with the relative timestamp formatter, and the existing linked implementation PR #53445 is open and unmerged with closing syntax for this work.

## What This Changes

The issue requests that the Control UI session list show full local date/time such as `2026-03-20 14:30` instead of only a relative or time-only Updated value.

## Best Possible Solution

Land or replace the narrow Control UI change that formats the Sessions `Updated` column as local `YYYY-MM-DD HH:mm`, preferably with focused formatter tests, while preserving the existing `sessions.list` contract, sorting by `updatedAt`, and unrelated session-list behavior.
