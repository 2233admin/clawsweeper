Keep this issue open. Current main still renders the Control UI Sessions `Updated` column with the relative timestamp formatter rather than the requested full local date/time, and open PR #53445 already tracks the narrow UI change with closing syntax for this issue.

The issue requests that the Control UI Sessions list show `Updated` timestamps as full local date/time values like `YYYY-MM-DD HH:mm` instead of a relative or time-only label.

## Best Possible Solution

Review and land #53445 if acceptable, or replace it with an equivalent narrow Control UI change that formats the Sessions `Updated` column as local `YYYY-MM-DD HH:mm` while preserving the existing sessions API, row sorting, and row data contract.
