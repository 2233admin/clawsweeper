## Summary

Keep open. Current main still does not ship the Bitwarden/Vaultwarden integration from this PR, and the latest discussion points to porting it after the secrets-provider plugin seam rather than closing it as implemented, superseded, or ClawHub-only.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best maintainer path is to land or reject the manifest-first secret-provider plugin seam, then either port this into `extensions/secrets-bitwarden` with manifest-declared ownership, SDK factory, docs, tests, and security review, or rebase and harden the generic exec-wrapper approach if the project chooses to keep vendor integrations documented as exec recipes.
