## Summary

Keep this PR open. Current main still ships only the generic SecretRef transport baseline (`env`, `file`, `exec`) and does not include a Bitwarden/Vaultwarden source, a bundled `bw` resolver, Bitwarden docs, or a merged plugin-owned secret-provider seam. The PR remains relevant, but it needs maintainer review because the architecture is now split between the current exec-wrapper approach and the open #72548 plugin seam, with overlap from #29110.

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate.
