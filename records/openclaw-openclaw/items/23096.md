## Summary

Keep this PR open. Current main still only ships the generic SecretRef transports (`env`, `file`, `exec`) and has no Bitwarden/Vaultwarden docs, bundled `bw` resolver, native `bw` source, or plugin secret-provider contract. The PR is still a plausible implementation candidate, but it needs maintainer direction because #72548 is the active plugin-seam path and #29110 overlaps with a Bitwarden exec-resolver docs approach.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best path is for maintainers to decide between two concrete routes: land #72548 first and port this into a `secrets-bitwarden` plugin with manifest-declared `contracts.secretProviders`, SDK factory, validation, docs, and tests; or intentionally keep this as a hardened exec-wrapper/docs integration, coordinate it with #29110, refresh the PR body/closing references, and complete focused command-execution and error-redaction review before merge.
