## Summary

Keep open. The Feishu-owned direction is right and the linked bug is valid, but the patch currently adds an unconditional message-tool instruction that conflicts with supported automatic group delivery. It should be revised so the topic hint applies only to message-tool-visible replies or is worded conditionally when the model chooses the message tool.

## What This Changes

The PR adds Feishu topic-scope prompt helpers and tests, then appends a thread-reply message-tool hint to `GroupSystemPrompt` for `group_topic` and `group_topic_sender` sessions.

## Best Possible Solution

Keep the fix extension-local, but make the Feishu topic prompt compatible with delivery mode: either emit the override only for message-tool-only visible replies or phrase it as conditional guidance for when the model uses the message tool. Add focused coverage for topic-scoped message-tool-only and automatic-delivery sessions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a Feishu `group_topic` or `group_topic_sender` group turn in message-tool-only mode receives generic `action=send` guidance, while Feishu only routes topic tool replies through `action=thread-reply` plus `messageId`. The PR's introduced regression is reproducible by enabling automatic group visible replies and observing the conflicting prompt instructions.

## Solution Assessment

No, not as written. The extension-local prompt nudge is the narrow maintainable direction, but it should not unconditionally tell automatic-delivery group sessions to call the message tool.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Gate the topic hint to tool-only delivery:** `extensions/feishu/src/bot-content.ts:150-151`
  - body: This hint is appended for every topic-scoped Feishu group session, but group turns can still resolve to automatic delivery via `messages.groupChat.visibleReplies: "automatic"` or when the message tool is unavailable. In that mode the shared group prompt tells the model not to use the message tool because the Feishu dispatcher already sends the final reply with topic reply metadata, so the new unconditional instruction can produce an extra `message(action="thread-reply")` alongside the normal reply. Gate the hint to message-tool-only delivery or make the wording conditional on choosing the message tool.
  - confidence: 0.86
