## Summary

Keep this PR open. Current main still appends the HEARTBEAT.md workspace-path hint based only on prompt text, including supported user-configured heartbeat prompts, so the regression tracked by #40255 is not implemented on main. The PR remains a focused implementation candidate, but maintainers should explicitly resolve the prompt-safety tradeoff raised in review before merge.

## What This Changes

The PR updates heartbeat prompt resolution so non-empty user-configured heartbeat prompts are not modified by the workspace-path hint, while default/internal heartbeat prompts keep the hint, and adds a focused helper unit test.

## Best Possible Solution

Review and either land this PR or a narrow replacement that preserves user-configured heartbeat prompts while retaining default/internal prompt hinting and idempotency. The maintainer decision should explicitly cover whether custom prompts need a non-overridable or tool-layer HEARTBEAT.md path guard before closing #40255.
