## Summary

Keep PR #59312 open. Current main still lacks the requested `OpenClawPluginApi.executeTool(...)` surface or an equivalent plugin self-invocation helper, and the PR discussion contains substantive unresolved correctness/security concerns rather than a conservative auto-close condition.

## Best Possible Solution

Keep the PR open for maintainer/API review. The best path is to decide whether plugins should have a self-call helper at all, then either implement it through the existing guarded tool-resolution/execution policy path or clearly document a deliberately narrower trusted-plugin-only contract, with tests for own-tool lookup, factory-produced names, cross-plugin denial, optional-tool allowlist behavior, and `before_tool_call` semantics.
