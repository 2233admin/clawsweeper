## Summary

Keep this issue open. Current main still returns exec stdout/stderr-derived text and details from foreground exec, background process polling/logging, node-host exec formatting, runtime outcomes, and process-session storage without an exec-owned redaction pass, while the existing redaction helpers mainly protect later logging, transcript, or UI presentation layers.

## What This Changes

The issue asks OpenClaw to redact or block secret-bearing stdout/stderr from the exec tool before command output enters agent-facing tool results, process details, events, logs, or internal context.

## Best Possible Solution

Patch exec-owned output boundaries so stdout/stderr-derived data is redacted before it leaves or is retained by exec/process runtime surfaces. The likely end state is a shared exec output redaction helper applied to foreground results, runtime outcomes, background process poll/log/list details, node-host exec formatting, notify summaries, and any event or transcript handoff, with targeted tests proving common secret-looking values are masked while output caps still work.
