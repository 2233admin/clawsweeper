## Summary

Keep this PR open. Current main still normalizes whitespace, aliases, provider-style dotted/slashed names, and malformed function/tool counters, but it does not strip a leading single-underscore `mcp_` from `mcp_exec`, `mcp_read`, or `mcp_write`. The PR's proposed dependency patch is also obsolete for current main, which uses `@mariozechner/pi-agent-core` 0.70.5 and has no pi-agent-core patchedDependency.

## Best Possible Solution

Rework this PR against current main by extending `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`: resolve exact registered names first, then strip one leading `mcp_` only when the suffix is non-empty and resolves unambiguously to an allowed tool such as `exec`, `read`, or `write`. Preserve exact `mcp__bundle__*` names, avoid a pnpm dependency patch, and add focused regression tests for prefixed, bare, missing, bundled, and normal tool names.
