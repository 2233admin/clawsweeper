## Summary

Keep this PR open because the reported `mcp_`-prefixed tool-call failure still appears valid on current `main`, but the submitted branch is stale and should not merge as-is. Current `main` has an OpenClaw-owned stream normalization seam for exact pi-agent-core dispatch, yet it only handles exact names, existing aliases, id inference, and structured `functions.*`/`tools/*` suffixes; it does not map `mcp_exec` to `exec`. The PR's dependency patch targets an old `@mariozechner/pi-agent-core` version and changes a compiled dependency/install surface, so the better fix is a narrow source-level normalizer update with focused tests.

## What This Changes

The PR adds a root `pnpm.patchedDependencies` entry and a patch file for `@mariozechner/pi-agent-core@0.55.3` that strips a leading `mcp_` prefix before dependency-level tool lookup.

## Best Possible Solution

Implement the alias in OpenClaw's existing embedded-runner normalization path: exact registered names should resolve first, then one leading `mcp_` should be stripped only for a non-empty, non-`mcp__...` suffix that resolves unambiguously to an allowed canonical tool. Keep dependency patches out of the fix and add focused coverage for prefixed, bare, missing, bundled/plugin, and unchanged canonical tool names.
