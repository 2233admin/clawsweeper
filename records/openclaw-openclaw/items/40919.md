## Summary

Keep open. Current main still mitigates session churn with dirty-file tracking, delta thresholds, whole-file hashes, and embedding cache, but changed session transcripts still rebuild a full session entry and flow through the file-level clear-and-reinsert chunk writer. The related incremental-session-sync PR remains open rather than merged, so the issue should stay open for maintainer review of that implementation path.

## What This Changes

The issue requests incremental session JSONL memory sync that tracks appended transcript content and avoids full delete/reinsert of all session chunks after each changed-session sync.

## Best Possible Solution

Land or replace #59577 with a current-path memory-core and memory-host-sdk change that preserves valid existing session chunks, stores durable append or chunk metadata, falls back to full reindexing when the transcript is compacted or otherwise unsafe, and adds regression coverage for append growth, unchanged transcripts, overlap boundaries, truncation/compaction, and failed-write recovery.
