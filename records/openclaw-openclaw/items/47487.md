## Summary

Keep open. Current main still computes implicit `exec`/`process` access from any global `tools.exec` section and merges it with explicit `alsoAllow`, so an agent-level restricted profile can still receive runtime tools. The docs say `messaging` does not include `group:runtime`, and current tests still assert the implicit re-exposure behavior. A targeted fix PR (#47991) existed but is closed and unmerged; adjacent open work (#43061/#42165) addresses minimal-profile exposure, not this exact agent `alsoAllow` bypass.

## Best Possible Solution

Keep this issue open and route it to the agent tool-policy/security owners. The best fix is to clarify and enforce the contract that configuring global `tools.exec` safety settings must not grant runtime-tool access to agents with explicit restricted profiles/`alsoAllow`, then add regression coverage for agent-level explicit `alsoAllow` with global `tools.exec`, plus the adjacent global `tools.fs`/media root-expansion path. PR #47991 is useful prior art but should be re-reviewed or adapted rather than treated as already landed.
