## Summary

Keep open. Current main still widens restricted tool profiles when `tools.exec` is configured, and the focused fix PR was closed unmerged while the merged bundled-tool filtering work only covers a later final-filter path, not this profile authorization bug.

## What This Changes

The issue requests that restricted agent tool profiles honor an explicit `alsoAllow` list that excludes `exec`, even when global `tools.exec` is configured for safety settings.

## Best Possible Solution

Revise the tool-policy contract so global `tools.exec` configuration cannot widen restricted agent profiles, preserve any intended per-agent explicit opt-in semantics, add regression coverage for `messaging` plus explicit `alsoAllow`, document the global-versus-agent behavior, and review the analogous `tools.fs` path in the same policy pass.
