## Summary

Keep open. Current main still has global plugin config only under `plugins.entries.<id>.config`; agent entries accept several per-agent overrides, but not plugin config overrides. The issue discussion also adds a concrete production multi-agent need, and the related closed plugin-scoping thread does not cover deep-merged per-agent plugin config payloads.

## Best Possible Solution

Keep this issue open for core config and plugin-runtime design. The likely path is to add an agent-scoped plugin override shape, validate it against each plugin's config schema, deep-merge it with `plugins.entries.<id>.config`, preserve secret handling and redaction, and expose the effective config consistently to plugin hooks, tools, providers, and runtime helpers.
