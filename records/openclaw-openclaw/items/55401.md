## Summary

Keep open. Current main still has only global plugin configuration under `plugins.entries.<id>.config`; agent entries are strict and do not accept `plugins`, so the reported per-agent plugin config override remains missing. The related #40208 and #62700 items were about per-agent plugin allow/deny lifecycle scoping and are now closed, so they do not provide an active canonical tracker for this deeper config-merge request.

## Best Possible Solution

Keep the issue open for maintainer product/design judgment. A proper fix would define a core config contract for agent-scoped plugin entry overrides, including schema shape, merge precedence with `plugins.entries.<id>.config`, SecretRef/redaction handling, runtime cache keys, and how the effective plugin config is exposed to hooks, tools, providers, and docs/tests.
