## Summary

Keep open. Current main has config-driven preview streaming for Telegram, Slack, Discord, and Mattermost, plus session-scoped toggles such as `/fast`, `/verbose`, and `/reasoning`, but there is no built-in `/stream` or `/streaming` command, no persisted session/conversation preview-streaming override, and the channel preview resolvers still read channel/account config rather than a runtime override. The related items named in the issue are adjacent command/streaming work, not duplicates that cover this per-chat/session control.

## Best Possible Solution

Keep this open for maintainer product/design review. The best path is to define a shared preview-streaming override contract first, likely a session/conversation-scoped field resolved before channel config, then add a built-in `/stream` command with `status|off|partial|progress|reset` semantics, `/status` visibility, native command arg menus, and opt-in support in Telegram, Slack, and Discord. Unsupported channels should reply clearly instead of silently accepting the override.
