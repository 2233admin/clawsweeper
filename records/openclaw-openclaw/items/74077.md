## Summary

Keep open. Current main still lacks a `/stream` or `/streaming` command, a command handler, a persisted session/conversation streaming override, and a shared resolver input that Telegram, Slack, or Discord could consume. The related reports are relevant streaming and command-adjacent work, but none implements or supersedes the requested per-chat/session preview-streaming control.

## What This Changes

Request a shared slash/native command to show, set, and reset preview streaming mode for the current chat/session without editing channel config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override contract that resolves before channel/account config, expose it through a built-in `/stream` command with status/set/reset behavior, show active overrides in `/status`, and make preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from source and docs inspection: current main has no `/stream` registration, handler, session override field, or channel resolver input for a runtime override.

## Solution Assessment

Unclear until maintainers choose the persistence and scope model. The maintainable direction is a shared resolver/override seam plus channel opt-in, rather than a Telegram-only command or a command that cannot affect existing preview dispatch.
