## Summary

Keep open. Current main still lacks a built-in `/stream` or `/streaming` command, has no persisted session/conversation preview-streaming override, and the inspected Telegram, Slack, and Discord preview paths still resolve from channel/account config. The related reports are adjacent streaming, default, observability, and command-routing work, but none supersedes this shared per-chat/session control.

## What This Changes

Add a slash/native command to inspect, set, and reset preview streaming mode for the current chat/session without editing gateway config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override contract that resolves before channel config, expose it through a built-in `/stream` command with status/set/reset behavior, show active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: no `/stream` command is registered or handled, no session override field exists, and the current channel resolvers read config only.

## Solution Assessment

Unclear until maintainers choose the storage and scope model. The maintainable direction appears to be a shared resolver/override seam plus channel opt-in; a Telegram-only command or plugin-only command would not reliably influence the existing preview dispatch paths.
