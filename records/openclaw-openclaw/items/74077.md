## Summary

Keep this issue open. Current main still has config-driven preview streaming only: there is no `/stream` or `/streaming` built-in command, no loaded command handler, no session/conversation override field, and the inspected Telegram, Slack, and Discord preview paths resolve from channel/account config rather than runtime session state. The linked items are adjacent command-alias, command-routing, and channel streaming bugs, but none is a canonical implementation for this shared per-chat/session control.

## What This Changes

The issue requests a shared slash/native command such as `/stream` to show, set, or reset the effective preview streaming mode for the current chat/session without editing config or restarting the gateway.

## Best Possible Solution

Add a small shared preview-streaming override contract that resolves before channel config, then expose it through a built-in `/stream` command with show/set/reset behavior and `/status` visibility. Preview-capable channel plugins should opt into that contract, while unsupported channels should return an explicit unsupported response.
