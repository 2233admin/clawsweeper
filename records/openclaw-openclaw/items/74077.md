## Summary

Keep open. Current main still has preview streaming controlled by channel config and channel-specific dispatch paths, but no built-in `/stream` or `/streaming` command, no session/conversation override field, and no shared runtime override resolver consumed by Telegram, Slack, or Discord.

## What This Changes

Add a `/stream` or `/streaming` chat command to show, set, and reset preview-streaming mode for the current chat/session without editing gateway config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override resolver that runs before channel/account config defaults, expose it through a built-in status/set/reset command, surface active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: current main has no `/stream` registration or handler, no session override field, and the inspected preview paths resolve from config-only inputs.

## Solution Assessment

Unclear until maintainers choose the scope and persistence model. The maintainable direction is a shared override contract plus channel opt-in, rather than a Telegram-only command or a command that cannot affect the existing dispatch paths.
