## Summary

Keep open. Current main still has config-driven preview streaming and adjacent per-session commands, but no `/stream` or `/streaming` command, no session/conversation streaming override, and no shared override resolver consumed by Telegram, Slack, or Discord. The existing ClawSweeper comment remains directionally correct against current main at `a102f4dede6af52523473ac29575c7e442493d15`.

## What This Changes

Request to add a slash/native `/stream` or `/streaming` command that shows, sets, and resets preview streaming mode for the current chat/session without editing gateway config or restarting, ideally shared across Telegram, Slack, and Discord.

## Best Possible Solution

Add a shared preview-streaming override contract that resolves before channel/account config, expose it through a built-in `/stream` command with status/set/reset behavior, show active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: current main has no `/stream` registration or handler, no session override field, and the inspected channel preview paths resolve from config only.

## Solution Assessment

Unclear until maintainers choose the storage and scope model. The maintainable direction is a shared resolver/override seam plus channel opt-in, rather than a Telegram-only command or a command that cannot affect existing preview dispatch.
