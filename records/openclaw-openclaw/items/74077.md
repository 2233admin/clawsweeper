## Summary

Keep open. Current main still has config-driven preview streaming only: there is no `/stream` or `/streaming` built-in command, no loaded command handler, no session/conversation override field, and the inspected Telegram, Slack, and Discord preview paths still resolve their mode from channel/account config rather than runtime session state. The linked reports are adjacent streaming or command-routing work, but none is a canonical implementation of this shared per-chat/session control.

## What This Changes

Add a slash/native command to inspect, set, and reset preview streaming mode for the current chat or session without editing gateway config or restarting.

## Best Possible Solution

Add a shared preview-streaming override contract that resolves before channel config, expose it through `/stream` with status/set/reset behavior, surface active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main check: `/stream` is absent from the registry and handlers, and preview mode resolution still has no session override input.

## Solution Assessment

Unclear for immediate implementation. The requested direction is reasonable, but the best fix should start with a maintainer-approved shared override scope and plugin/channel contract rather than a Telegram-only or ad hoc command patch.
