## Summary

Keep open. Current main still exposes preview streaming as channel configuration plus a global owner-only `/debug` override path, but it does not provide the requested per-chat/session `/stream` or `/streaming` command, persisted session/conversation override, or shared channel override seam consumed by Telegram, Slack, and Discord.

## What This Changes

The issue requests a slash/native command to inspect, set, and reset preview streaming mode for the current chat/session without editing gateway config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override contract that resolves before channel/account config, expose it through a built-in `/stream` command with status/set/reset behavior, surface active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from source and docs inspection: current main has no `/stream` registration, handler, session override field, or channel resolver input for per-chat/session streaming overrides.

## Solution Assessment

Unclear until maintainers choose the storage and scope model. The maintainable direction is a shared resolver/override seam plus channel opt-in, rather than a Telegram-only command or a global `/debug` config override that affects more than the current conversation.
