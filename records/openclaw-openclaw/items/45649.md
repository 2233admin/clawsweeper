## Summary

Keep this issue open. The current Control UI still renders tool-message summaries as collapsed buttons with small muted text, 11px mono tool names, ellipsis truncation, a 760px tool-shell cap, and auto-expansion disabled at the chat render call. That does not provide strong evidence that the reported readability, early cut-off, or repeated-open annoyance has been fixed on current main.

Keep this issue open and fix it in the Control UI tool-card surface: make collapsed tool bubbles readable in light and dark themes, avoid unnecessary early ellipsis or the 760px cap when there is room, clean up the detail wording that produces awkward connector text, and either make tool-call expansion ergonomic by default or expose a clear persisted preference. Add focused rendering/style tests plus a screenshot/manual browser check.
