## Summary

Current main still does not implement the requested replay-context pruning for consecutive assistant NO_REPLY/silent-reply turns. The embedded runner sanitizes, validates, heartbeat-filters, limits history, and repairs tool-use/result pairing, but there is no pass that collapses consecutive silent replies. The linked implementation PR #13388 is closed and unmerged, so it remains prior art rather than shipped behavior.

## Best Possible Solution

Keep this issue open and implement a focused in-memory replay-context prune in the embedded runner history pipeline. Collapse only consecutive assistant silent replies, keep at most one by default, reuse shared silent-token detection, handle legacy string/text-block/JSON-envelope and empty-content silent stop shapes, and add focused replay-context tests. Treat closed PR #13388 as useful prior art, not as merged work.
