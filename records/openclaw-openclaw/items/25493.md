## Summary

Keep open. Current main still hardcodes the requested OpenClaw-owned system prompt sections and only offers whole-prompt replacement, hook mutation, GPT/provider overlay knobs, or provider-owned overrides for three named sections. The directly related implementation PR #25938 is closed and unmerged, while #53942 remains a related but narrower follow-up rather than a shipped fix.

## What This Changes

Allow users to edit or override OpenClaw-owned system prompt template sections, such as Safety and CLI/system tips, potentially through workspace files like SAFETY.md or a template language.

## Best Possible Solution

Keep this open for maintainer product/security judgment. The best path is to decide whether OpenClaw-owned sections such as Safety, Tool Call Style, CLI Quick Reference, and Self-Update should become schema-backed or workspace-file-backed overrides; if yes, implement that in the prompt/workspace surfaces with trust-boundary docs and regression tests, and if no, document systemPromptOverride, before_prompt_build, agent:bootstrap, and provider overlays as the supported customization paths while coordinating the answer with #53942.
