## Summary

Keep this issue open. Current main still hardcodes the requested OpenClaw-owned Safety, CLI Quick Reference, and Self-Update prompt sections, while the supported customization paths remain whole-prompt replacement, hook/plugin mutation, and provider-owned overlays for only three named sections. The closed #25938 PR is useful prior art but was not merged, and open #53942 is related but narrower, so this still needs maintainer product/security judgment rather than cleanup closure.

## Best Possible Solution

Keep this open for maintainer product/security judgment. The best path is to decide whether OpenClaw-owned prompt sections such as Safety, Tool Call Style, CLI Quick Reference, and Self-Update should become documented schema-backed or workspace-file-backed overrides. If yes, implement the override surface in src/agents/system-prompt.ts and src/agents/workspace.ts with explicit trust-boundary docs and regression tests. If maintainers decide these sections should stay fixed, document systemPromptOverride, before_prompt_build hooks, and provider overlays as the supported escape hatches and coordinate that answer with #53942.
