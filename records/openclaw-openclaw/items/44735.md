## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior for an enabled alternative Feishu/Lark channel owner without `preferOver` metadata, and the existing functional review concern around restrictive `plugins.allow` setups remains valid.

## Best Possible Solution

Keep this PR open and adapt it to the current split auto-enable files. The fix should live in `src/config/plugin-auto-enable.shared.ts` or a small helper it owns, and should skip bundled channel auto-enable only when another channel owner is actually effective under the same activation rules, including `plugins.enabled`, allow, deny, and explicit disable. Add regression coverage for the `openclaw-lark` Feishu scenario and a restrictive `plugins.allow` case; alternatively make the replacement explicit through manifest `channelConfigs.feishu.preferOver: ["feishu"]` and test that path.
