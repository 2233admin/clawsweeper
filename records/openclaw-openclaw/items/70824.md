## Summary

Keep open. Current main does not contain the PR's requested behavior, and the PR is not safely obsolete: the explicit `input` merge bug still exists, while the cross-provider image fallback part has a concrete provider-scoping concern and conflicts with an existing fail-closed test.

## Best Possible Solution

Keep the item open, but ask for a narrower maintainer-ready patch: preserve explicit `models.providers.<id>.models[].input` in `src/agents/models-config.merge.ts` with updated regression tests, and avoid broad cross-provider capability fallback unless it is paired with image-aware provider routing or limited to truly unresolved-provider cases. The current provider-scoped fail-closed behavior in `resolveGatewayModelSupportsImages` should remain until routing semantics are made explicit.
