## Summary

Keep open. Current main supports isolated agents and an agent-grouped chat session selector, but it still does not implement team/project parent nodes, configurable agent team/group metadata, independent team management, or workspace-derived hierarchical grouping. Related #49828, #30861, and #52683 overlap the multi-agent UX space, but none is the canonical tracker for this specific hierarchy request.

## Best Possible Solution

Keep this open for maintainer product design. Decide whether the request should be a lightweight presentation feature, such as optional agents.list[].group or project metadata plus grouped Control UI session/agent views, or part of a broader project-room/team model. If accepted, update the agent config contract, agents.list protocol/schema, Control UI chat/sessions/agents surfaces, docs, and focused UI/backend tests together.
