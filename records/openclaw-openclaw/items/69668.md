## Summary

Keep #69668 open. Current main has multi-agent isolation and an agent-grouped chat session selector, but it does not implement the requested team/project parent concept, nested team hierarchy, agent group metadata, or hierarchical Sessions/Agents UI. The adjacent WebChat agent/session switcher work from #45086 is implemented, but it solves only agent-level switching and does not supersede this team/project grouping request.

## Best Possible Solution

Keep this issue open for a focused Control UI/core config design. The likely path is an additive team/project grouping contract, such as `agents.list[].group` or a workspace-derived grouping mode, exposed through the agent/session list APIs and rendered as collapsible groups in the Chat selector, Sessions table, and Agents view. Update multi-agent/config docs and add UI/controller tests for duplicate role names across teams, default-agent handling, and preserved per-agent session isolation.
