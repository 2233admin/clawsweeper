## Summary

Keep open. Current main still lacks the Slack RTS/action-token path: there is no `assistant.search.context` or `action_token` implementation, no Slack `search` message action, and the existing Slack read action only wraps conversation history/replies. The adjacent open Slack `search.messages` work is useful but does not cover bot-token RTS with event-scoped action tokens.

## What This Changes

Add Slack Real-Time Search support by extracting Slack event `action_token` values and exposing a safe `assistant.search.context` search action or tool.

## Best Possible Solution

Implement the remaining work in the bundled Slack plugin with an explicit RTS contract: capture the ephemeral action token from authorized inbound Slack events, keep it scoped to the originating team/account/channel/thread/request, expose a typed read-only `assistant.search.context` capability, document required AI/search scopes and privacy limits, and deliberately separate or merge it with user-token search only after maintainer review.
