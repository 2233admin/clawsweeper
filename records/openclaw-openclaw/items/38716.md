## Summary

Keep open. Current main still lacks the requested Slack RTS path: no `assistant.search.context` or `action_token` handling exists in the Slack plugin, Slack message actions do not advertise or dispatch `search`, and the implemented read path is conversation history/thread reads rather than Slack Real-Time Search. The related `search.messages` work in #65691 is adjacent but does not cover assistant action-token RTS semantics.

## What This Changes

Request support for Slack Real-Time Search by capturing assistant/event `action_token` values and exposing a Slack message/tool action backed by `assistant.search.context`.

## Best Possible Solution

Implement this in the bundled Slack plugin with an explicit RTS contract: capture the inbound Slack action token only for the authorized user event, pass it through the relevant assistant/message context, expose a typed read-only `assistant.search.context` action, document the required AI/search scopes and privacy limits, and keep it distinct from user-token `search.messages` unless maintainers intentionally consolidate the APIs.
