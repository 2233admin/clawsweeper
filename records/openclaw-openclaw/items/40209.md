## Summary

Keep this PR open. Current main still stores system events under exact caller-provided session-key casing, while targeted heartbeat resolution canonicalizes agent-scoped session keys to lowercase before peeking and consuming events. The PR remains a focused implementation candidate for the linked channel-session heartbeat bug.

## Best Possible Solution

Review and land a focused fix that makes system-event queue session-key normalization consistent with heartbeat session resolution, with regression tests for mixed-case channel enqueue/peek/drain and cross-case dedupe. If non-agent session keys are intentionally case-sensitive, scope the normalization to parseable agent-scoped keys before merge.
