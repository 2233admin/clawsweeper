## Summary

Keep this PR open. Current main still stores system events under exact session-key casing, while heartbeat session resolution canonicalizes agent-scoped keys to lowercase before peeking/consuming events. The linked #34338 report and the PR diff describe a real plugin-facing mismatch that is not implemented on main yet.

## Best Possible Solution

Review and land a focused normalization fix for system-event session keys, with regression coverage for mixed-case channel enqueue/peek/drain and cross-case dedupe. If non-agent system-event keys must remain case-sensitive, normalize only recognized agent-scoped session keys before merging.
