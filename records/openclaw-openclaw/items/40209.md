## Summary

Keep this PR open. Current main still stores system events under the caller's exact session-key casing, while the heartbeat forced-session path canonicalizes agent session keys to lowercase before peeking or draining queued events. The PR remains a focused candidate for the linked channel-session heartbeat bug, and its diff is limited to the in-memory queue plus regression tests.

## Best Possible Solution

Review and land this PR or an equivalent patch that normalizes system-event queue keys consistently for enqueue, peek, drain, consume, and has-events. Keep the mixed-case channel lookup and cross-case dedupe regression coverage; if non-agent custom system-event keys must remain case-sensitive, normalize only canonical agent-scoped session keys and document that contract.
