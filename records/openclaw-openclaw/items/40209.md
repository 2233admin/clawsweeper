## Summary

Keep this PR open. Current main still stores system events under the caller's exact session-key casing, while the heartbeat forced-session path canonicalizes agent session keys to lowercase before peeking and consuming queued events. The PR remains a focused implementation candidate for the linked channel-session heartbeat bug, with no security-sensitive changes in the supplied two-file diff.

## Best Possible Solution

Review and land this focused system-event session-key normalization fix, or an equivalent patch that normalizes the queue key used by enqueue, peek, drain, consume, and has-events. Include regression coverage for mixed-case channel enqueue/peek/drain and cross-case dedupe; narrow the normalization if non-agent system-event keys must remain case-sensitive.
