## Summary

Keep this PR open. Current main still stores system-event queues under the caller's exact session-key casing, while forced heartbeat session routing canonicalizes agent session keys to lowercase before peeking or draining queued events. The PR remains a focused implementation candidate for the linked channel-session heartbeat bug, with a small security footprint limited to the queue helper and regression tests.

## Best Possible Solution

Review and land this PR or an equivalent patch that normalizes system-event queue keys consistently for enqueue, peek, drain, consume, and has-events. Keep the mixed-case channel lookup and cross-case dedupe coverage, and decide whether non-agent custom system-event keys should also become case-insensitive. Because the linked bug is user-visible for plugin-driven heartbeat delivery, a short changelog fix entry credited to @haynzz would be appropriate if this lands.
