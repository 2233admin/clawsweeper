## Summary

Keep this PR open. Current main still keys the in-memory system-event queue by exact caller-provided session-key casing, while targeted heartbeat resolution canonicalizes agent-scoped session keys to lowercase before peeking and consuming events. The linked report is closed stale, but its concrete reproduction and this PR’s fix still match current code behavior.

## Best Possible Solution

Review and either land this focused fix with mixed-case enqueue/peek/drain and cross-case dedupe regression coverage, or adjust it to canonicalize only recognized agent-scoped session keys before merge. Do not close this PR until current main can enqueue a mixed-case channel-session event and heartbeat can find and consume it through the lowercase canonical session key.
