## Summary

Keep open. Current main still strips `sessionKey` from hidden runs before lifecycle events reach the gateway, while the gateway only persists terminal session state when it can resolve a session key. The PR is focused and plausibly fixes the stuck-running session path, but it should get normal maintainer review because it intentionally changes hidden-run session identity visibility on the shared agent-event bus.

## What This Changes

The PR changes `emitAgentEvent()` to retain `sessionKey` for hidden `lifecycle` events, adds focused agent-event regression coverage, and adds a changelog entry.

## Best Possible Solution

Land or adapt a narrow fix that preserves lifecycle session identity for persistence while keeping hidden assistant/tool traffic redacted; if the privacy boundary requires it, route the preserved lifecycle key only to persistence/session-change paths rather than general client broadcasts.
