## Summary

Keep open. Current main serializes Slack sends per account/token/recipient/thread, but it still has no configurable minimum-delay throttle for Slack timestamp ordering; the broad related PR #26732 is closed unmerged and does not supersede the remaining work.

## What This Changes

The issue requests a configurable Slack-specific outbound send throttle so rapid sequential Slack messages get distinct timestamps and preserve display order.

## Best Possible Solution

Keep this issue open and implement a focused Slack plugin throttle: add a typed strict-schema setting with docs/UI hints, default it conservatively for Slack timestamp ordering, and apply it consistently to normal sends, DMs, thread replies, block messages, multipart text chunks, media-caption follow-ups, and slash response_url follow-ups without broadening behavior to other channels unless a shared API gap is proven.
