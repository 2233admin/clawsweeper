## Summary

Keep open. Current main serializes Slack sends per account/token/recipient/thread, but it still lacks the requested configurable minimum-delay throttle for Slack timestamp ordering, and the related broad outbound throttle PR was closed unmerged.

The issue requests a Slack-specific configurable outbound send throttle so rapid channel messages, thread replies, and DMs get distinct Slack timestamps and display in order.

## Best Possible Solution

Implement a focused Slack-owned throttle setting with typed config, strict schema/UI hints, docs, and tests, then apply it after the existing per-target serialization across normal posts, DMs, thread replies, block/text chunks, media-caption follow-ups, and slash response_url follow-ups.
