## Summary

Keep open. Current main serializes Slack sends per account/token/recipient/thread, but it still lacks the requested configurable minimum-delay throttle for Slack timestamp ordering, and the related broad throttle PR was closed unmerged.

## What This Changes

The issue requests a Slack-specific configurable outbound send throttle so rapid channel messages, thread replies, and DMs get distinct Slack timestamps and display in order.

## Best Possible Solution

Add a focused Slack plugin throttle setting with strict schema/types/UI hints/docs, then apply it consistently to all Slack outbound post surfaces: normal sends, DMs, thread replies, block replies, text chunks, media-caption follow-ups, and slash response_url follow-ups.
