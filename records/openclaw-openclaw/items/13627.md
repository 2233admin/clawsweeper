## Summary

Keep open. Current main serializes Slack sends per account/token/recipient/thread, but it still does not implement the requested configurable minimum-delay throttle for Slack timestamp ordering, and multipart plus slash-command follow-up paths can still send consecutive Slack messages without spacing.

## What This Changes

Request a configurable Slack-specific outbound send throttle so rapid agent replies get distinct Slack timestamps and preserve display order.

## Best Possible Solution

Implement this as a Slack plugin/account-level delivery setting with typed config, schema/UI hints, docs, and tests, then apply the spacing consistently to normal posts, DMs, thread replies, block replies, text chunks, media-caption follow-ups, and slash response_url follow-ups while preserving current per-target serialization.
