## Summary

Keep open. Current main still lacks the DeepSeek provider policy artifact, so the linked DeepSeek context/cost bug is not implemented on main. The PR is useful source work, but the current patch still needs changes: it hardcodes catalog metadata, disagrees with the current manifest for `deepseek-reasoner`, overwrites explicit values instead of hydrating only missing fields, and adds no regression test.

## Best Possible Solution

Keep this PR open but require changes, or replace it with a narrow ClawSweeper repair if the contributor branch is not practical to update. The fix should live in the DeepSeek plugin: add a typed lightweight `provider-policy-api.ts` that reuses the existing manifest-derived catalog, hydrates missing metadata for sparse DeepSeek model rows, preserves explicit user overrides, adds focused regression tests through the policy/defaults path, and avoids DeepSeek-specific logic in core.
