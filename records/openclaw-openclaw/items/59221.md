## Summary

Keep PR #59221 open. Current main does not implement the requested nine built-in text aliases, and the PR is a focused core command ergonomics change rather than a duplicate, ClawHub item, or external-admin task. It is not merge-ready as described because the review discussion surfaced a real `/m` behavior gap: model switching is parsed as an inline directive, while the PR only adds a command-registry alias.

## Best Possible Solution

Keep the PR open but request changes before merge: either drop `/m` from this batch or wire built-in model command aliases into directive parsing with targeted tests proving `/m <model>` applies the same model override as `/model <model>`. Keep the `/ms` docs split from the follow-up commit, and add/adjust registry, normalization, directive, and docs-parity tests for any aliases that remain.

## Risks / Open Questions

- Adding one-letter built-in aliases reserves common tokens and can shadow user-configured model aliases, so `/m` in particular needs explicit maintainer/product judgment or a compatibility-preserving implementation.
- A registry-only `/m` alias is likely insufficient because `/model` behavior is directive-driven rather than handled by a normal command handler.
