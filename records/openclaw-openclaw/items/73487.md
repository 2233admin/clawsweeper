## Summary

Keep open. Current main still has the Telegram quote-reply gate that prevents the answer preview stream from being created, while the Telegram docs, streaming docs, and doctor warning path still do not explain that `replyToMode` excludes tool-progress preview. The issue is also paired with open closing PRs, so it should stay open until a preferred fix is reviewed and merged.

## What This Changes

The issue requests that OpenClaw document and/or warn when Telegram native quote replies disable the tool-progress preview path despite `streaming.preview.toolProgress` being enabled.

## Best Possible Solution

Land the preferred Telegram-scoped clarification, ideally documenting the quote-reply/tool-progress trade-off and adding a doctor warning for affected configs, while leaving any persistent-message fallback as a separate product decision.

## Reproduction Assessment

Yes. Configure Telegram with preview streaming active, `streaming.preview.toolProgress` enabled, and `replyToMode` set to a quote-reply mode, then trigger tool use; current source shows no answer-lane stream is created, and the issue discussion confirms the behavior on v2026.4.27.

## Solution Assessment

Yes. A docs plus doctor-warning fix is the narrowest maintainable solution for the reported silent doc/runtime mismatch; changing runtime delivery to separate persistent messages would be broader behavior work.
