## Summary

Keep open. Current main still has the Telegram quote-reply gate that prevents the answer preview stream from being created, while the docs and doctor warning path still do not explain the `replyToMode` exclusion; the issue is also paired with open closing PRs, so it should stay open until maintainers choose and merge the preferred fix.

## What This Changes

The issue requests a Telegram docs, doctor-warning, or fallback-delivery fix for `replyToMode` quote replies silently suppressing tool-progress preview updates.

## Best Possible Solution

Land the preferred Telegram-scoped clarification, ideally documenting the quote-reply/tool-progress trade-off and adding a doctor warning for affected configs, while leaving any separate persistent-message fallback as a later product decision.

## Reproduction Assessment

Yes. Configure Telegram with preview streaming active, `streaming.preview.toolProgress` enabled, and `replyToMode` set to a quote-reply mode, then trigger tool use; current source and tests show no answer-lane stream is created, and the issue discussion confirms this on v2026.4.27.

## Solution Assessment

Yes. A docs plus doctor-warning fix is the narrowest maintainable solution for the reported silent doc/runtime mismatch; changing runtime delivery to persistent separate messages would be broader behavior work.
