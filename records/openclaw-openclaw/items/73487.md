## Summary

Keep open. Current main still has the Telegram quote-reply gate that prevents the answer preview stream from being created, while the Telegram and streaming docs plus doctor warning path still do not surface the `replyToMode` exclusion; the issue is also paired with open closing PRs, so it should not be closed before maintainers choose and merge a fix.

## What This Changes

The issue asks OpenClaw to surface or fix the incompatibility where Telegram native reply quoting disables the tool-progress preview path despite docs describing it as default-on when preview streaming is active.

## Best Possible Solution

Land the preferred Telegram-scoped clarification, ideally documenting the quote-reply/tool-progress trade-off and adding a doctor warning for affected configs; keep any separate persistent-message fallback as a later product decision.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger tool use on a quotable inbound message; current source shows no answer-lane stream is created, and the issue discussion confirms this on released builds.

## Solution Assessment

Yes. Keeping the issue open for the paired PRs is the right path; a docs plus doctor-warning fix is the narrowest maintainable answer to the silent doc/runtime mismatch, while a runtime fallback would be broader behavior work.
