## Summary

Keep open. Current main still has the Telegram native quote-reply gate that prevents the answer preview stream from being created, while the Telegram and streaming docs still describe tool-progress preview as default-on without the `replyToMode` caveat and the doctor adapter still has no warning for this config conflict. The issue also has open closing PRs #73495 and #73508, so it should remain open until maintainers choose and land the preferred fix.

## What This Changes

The issue asks OpenClaw to document, warn about, or otherwise address Telegram tool-progress preview being disabled when `replyToMode` uses native quote replies.

## Best Possible Solution

Land the preferred Telegram-scoped clarification, ideally documenting the quote-reply/tool-progress trade-off and adding a doctor warning for affected configs, while leaving any separate persistent-message fallback as a later product decision.

## Reproduction Assessment

Yes. Configure Telegram with preview streaming active, `streaming.preview.toolProgress` enabled, and `replyToMode` set to a quote-reply mode, then trigger tool use; current source/tests show no answer-lane stream is created, and the issue discussion confirms this on v2026.4.27.

## Solution Assessment

Yes. A docs plus doctor-warning fix is the narrowest maintainable solution for the reported silent doc/runtime mismatch; changing runtime delivery to persistent separate messages would be broader behavior work.
