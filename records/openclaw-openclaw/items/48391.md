## Summary

Keep this PR open. Current main still lacks the PR’s intended cron read-model fields and richer `cron.run` acknowledgment, while the PR is a focused implementation candidate with a real protocol/UI behavior gap still present.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to land a reviewed variant that adds a separate cron read-state/read-job type, projects `list()` and `listPage()` through it, updates the gateway schema without making derived fields writable via `cron.update`, and either documents or renames the immediate `cron.run` in-progress acknowledgment so clients do not confuse queued work with persisted running state.

## Risks / Open Questions
