## Summary

Keep this PR open. Current main still resolves only one provider `wrapStreamFn`, while the provider contract supports `hookAliases` and provider-owned stream wrappers. The PR addresses a real composition gap, but its implementation needs to be ported from the older `src/plugins/provider-runtime.ts` target into the current `src/plugins/provider-hook-runtime.ts` path before it can land.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. Port the chaining behavior to `src/plugins/provider-hook-runtime.ts`, compose the primary matched wrapper first and then other matching hook providers, filter candidates by provider id/aliases, dedupe by stable identity, add the primary-plus-hookAlias and unrelated-provider regression test, then run the normal changed plugin/core verification before landing.
