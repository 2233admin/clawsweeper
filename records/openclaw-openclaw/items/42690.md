## Summary

This issue should stay open. Current main has the data needed to compare selected and active models, and #55425 shipped an adjacent fix for persisted runtime models, but `session_status` still does not render the requested selected/default `Model:` line plus a separate non-fallback `Active:` line.

## Best Possible Solution

Keep this issue open. Implement the split in the shared status rendering path: keep the configured/session-selected model on `🧠 Model:`, add `↪️ Active: provider/model` when the resolved active runtime model differs and fallback state is not active, and add focused coverage in `src/auto-reply/status.test.ts` and `src/agents/openclaw-tools.session-status.test.ts`. If the active model is only available from transcript metadata, add a bounded last-`model-snapshot` lookup so `session_status` can use it without mutating the session default.
