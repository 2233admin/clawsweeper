## Summary

Keep open. Current main still lacks the requested split between the selected/session-default model and a routed active runtime model in `session_status`; the adjacent #55425 work is present, but it promotes runtime identity onto the main model line instead of rendering a separate `↪️ Active:` line for non-fallback runtime drift.

## Best Possible Solution

Keep this issue open and implement the split in the shared status path: keep `🧠 Model:` tied to the selected model source, render `↪️ Active: provider/model` when the active runtime model differs outside fallback state, preserve `↪️ Fallback:` for actual fallback notices, and add focused tests covering runtime-routed sessions and transcript model-snapshot recovery.

## Clownfish Work Prompt

Fix the `session_status` status-card model display bug for runtime-routed sessions. Current main promotes the runtime model into `🧠 Model:` when no explicit override exists, and the shared renderer only emits `↪️ Fallback:` for persisted fallback notice matches. Change the status flow so the selected model line remains tied to the selected source (explicit session override, effective channel override, otherwise agent/global default), while the active runtime model comes from `sessionEntry.model`/`modelProvider` or the latest transcript model identity when available. Render `↪️ Active: provider/model` when active differs from selected and `resolveActiveFallbackState` is inactive. Preserve existing `↪️ Fallback:` output for real fallback notices and do not change session model inheritance, `/model` mutation, fallback pinning, or provider routing behavior. Add regression coverage in `src/auto-reply/status.test.ts` for non-fallback runtime drift, in `src/agents/openclaw-tools.session-status.test.ts` for no-explicit-override routed sessions, and in `src/gateway/session-utils.fs.test.ts` or an equivalent helper test if the fix reads transcript `customType: "model-snapshot"`. Add a user-facing changelog fix entry thanking @sk8man121.
