## Summary

Keep this issue open. Current main still does not implement the requested `session_status` split between the configured/session-default model and a non-fallback routed active model. The renderer computes selected and active model refs, but the output path only renders `🧠 Model:` plus `↪️ Fallback:` for active fallback state; non-fallback runtime routing still has no `↪️ Active:` line. The adjacent #55425 behavior is narrower: it makes runtime session models appear as the selected card model when no explicit override exists, which can still collapse the distinction this issue asks to preserve.

## Best Possible Solution

Implement the split in the shared status rendering path: keep `🧠 Model:` tied to the configured or explicit session selection, add `↪️ Active: provider/model` when the resolved active runtime model differs and fallback state is not active, preserve existing `↪️ Fallback:` behavior for failover, and add focused coverage in `src/auto-reply/status.test.ts` plus `src/agents/openclaw-tools.session-status.test.ts`. If live session state lacks active identity, reuse the existing bounded transcript usage/model fallback without mutating the session default.
