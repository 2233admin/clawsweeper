## Summary

Keep this issue open. Current main still has the selected-vs-active model data, but the shared status renderer only emits the selected `🧠 Model:` line plus `↪️ Fallback:` for active fallback state; it does not render a non-fallback `↪️ Active:` line. The adjacent #55425 fix shipped in `v2026.3.28`, but it collapses the persisted runtime model into the primary status-card model when no explicit override exists, which is not the split requested here.

## Best Possible Solution

Keep this issue open. Implement the split in the shared status rendering path: keep the configured/session-selected model on `🧠 Model:`, add `↪️ Active: provider/model` when the resolved active runtime model differs and fallback state is not active, preserve the existing `↪️ Fallback:` line for failover, and add focused coverage in `src/auto-reply/status.test.ts` plus `src/agents/openclaw-tools.session-status.test.ts`. If active identity is absent from the session entry, add a bounded last-`model-snapshot` lookup rather than mutating the session default.
