## Summary

Keep this PR open. Current main still has the reported Feishu typing-reaction target bug, and this PR is a narrow implementation candidate; the only concrete issue I found is a low-priority edge case where the new inbound target should not use synthetic card-action message ids.

## What This Changes

The PR adds an optional `inboundMessageId` to the Feishu reply dispatcher, passes `ctx.messageId` from both Feishu inbound dispatcher call sites, uses that id for typing-reaction targeting with a `replyToMessageId` fallback, and updates Feishu tests.

## Best Possible Solution

Land or reimplement the narrow Feishu fix so replies still route to the topic root while typing reactions target the actual inbound message, with a guard for synthetic card-action events and regression tests for both the inbound target and fallback paths.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.86

Full review comments:

- **[P3] Skip synthetic card-action IDs for typing reactions:** `extensions/feishu/src/bot.ts:1214`
  - body: When `ctx.suppressReplyTarget` is true, card-action events without `open_message_id` intentionally leave `replyTargetMessageId` undefined because `ctx.messageId` is a synthetic `card-action-*` id. Passing `ctx.messageId` as `inboundMessageId` makes `onReplyStart` call Feishu `messageReaction.create` for that synthetic id, adding a failing API call in a path this PR says is unchanged. Guard this assignment with `!ctx.suppressReplyTarget` or an equivalent real-message check.
  - confidence: 0.84
