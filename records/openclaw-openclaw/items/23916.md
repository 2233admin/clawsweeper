## Summary

Keep open. Current main still renders the compact `openclaw devices list` paired-device table and docs, while the storage/Gateway layers carry richer metadata that is not surfaced as client purpose, OS/device family, last-active/detail output, or editable notes. Related work in #57449, #55973, and #13870 remains open or unmerged.

## What This Changes

Request to make `openclaw devices list` show more user-understandable paired-device information, including client purpose, OS/browser-style details, activity/detail output, and editable notes or names.

## Best Possible Solution

Keep this issue open until a PR lands. The best path is to review/rebase #57449 or a successor for Device ID/Name, Client, OS/device family, and Last active display; reconcile timestamp semantics with #55973; preserve `--json`; sanitize every device-controlled field; update `src/cli/devices-cli.test.ts` and `docs/cli/devices.md`; and keep editable names/notes coordinated with #13870.
