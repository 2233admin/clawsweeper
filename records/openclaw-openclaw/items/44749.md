## Summary

Keep this issue open. Current main still persists exec allow-always and allowlist-use updates by mutating a previously loaded ExecApprovalsFile snapshot and saving the whole file, so concurrent handlers can still overwrite each other. The keepmyclaw comment is a useful backup workaround, but it does not fix the internal OpenClaw race, and related PR #52403 was closed unmerged.

## Best Possible Solution

Keep this issue open and implement a locked exec-approvals mutation helper that reloads the latest exec-approvals.json inside the critical section, applies additive allowlist and metadata updates, and saves. Update addAllowlistEntry, addDurableCommandApproval, persistAllowAlwaysPatterns, recordAllowlistUse, and both gateway/node-host call paths to use it, with regression coverage for two stale snapshots adding different allow-always entries for the same agent.
