## Summary

Current main still appears vulnerable to the reported last-write-wins race. The exec approval store still mutates and saves the caller-provided `ExecApprovalsFile` snapshot, while both gateway and node-host allow-always paths pass already-loaded approval snapshots into that mutation path. The related fix PR #52403 is closed unmerged, and I did not find an equivalent locked or reload-before-write implementation on main.

## Best Possible Solution

Keep this issue open and fix exec approval persistence in the owner module. The best implementation is a small write-side mutation helper in `src/infra/exec-approvals.ts` that serializes approval-file mutations, reloads the latest `exec-approvals.json` inside the critical section, applies additive allowlist/durable/metadata changes, and saves once. Update the gateway and node-host allow-always paths to use that helper, and add regression coverage for two stale snapshots adding different allow-always entries for the same agent so both entries survive.
