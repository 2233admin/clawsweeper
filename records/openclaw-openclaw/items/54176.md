## Summary

Keep open. Current main still does not classify `application/msword` or `application/x-cfb` as binary in the media-understanding file extraction path, and #73799 is an open, unmerged implementation PR that uses closing syntax for this issue.

## What This Changes

The issue requests that legacy Word/OLE `.doc` attachments be treated as binary attachments, or converted before embedding, so raw binary bytes are not inserted into prompt history as text.

## Best Possible Solution

Add a narrow media-understanding guard that treats `application/msword` and `application/x-cfb` as binary before text extraction, preserve the existing vendor `+json`/`+xml` behavior, cover both MIME types with focused regression tests, and close this only after that lands on main.
