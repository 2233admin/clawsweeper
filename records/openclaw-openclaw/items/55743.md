## Summary

Current main has two adjacent pieces: markdown code blocks provide Copy, and Control UI can approve already-pending exec/plugin approval requests. It does not implement the requested command-card workflow that detects assistant-suggested terminal commands in chat and lets the operator approve and run that exact suggestion from the Control UI.

## Best Possible Solution

Keep this issue open and implement a narrow Control UI command-card layer for assistant markdown shell snippets. The card should keep the current Copy affordance, add Approve and Run only when a safe execution target is available, route execution through the existing gateway/node exec approval pipeline, bind exact command/cwd/host/session, stream output back into chat, and add focused UI/gateway tests plus Control UI and exec-approval docs.

## Risks / Open Questions

- This feature touches a high-risk execution path: implementation should bind the exact command, cwd, host, and session before approval and should reuse or extend existing exec approval safeguards rather than adding a parallel bypass.
- Risk classification for suggested commands is not present in the current chat renderer; a future implementation needs conservative labels and tests for destructive, network, config, service-restart, and read-only cases.
