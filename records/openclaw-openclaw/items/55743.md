## Summary

Keep this issue open. Current main still renders assistant markdown command snippets as generic Copy-only code blocks, and the existing approval UI only resolves already-pending exec/plugin approval requests. There is no Control UI path that turns assistant-suggested shell snippets into command cards with target metadata, risk labels, an Approve and Run action, or streamed output back into the chat.

## Best Possible Solution

Implement a narrow Control UI command-card path for assistant shell snippets: preserve Copy, add Approve and Run only when the exact command, cwd, host/node/session, and safe execution target are available, route execution through the existing exec approval/node invoke pipeline, stream output back into chat, and add focused UI, gateway, security, and docs coverage.

## Risks / Open Questions

- This feature would touch a high-risk local execution path. A safe implementation should route through the existing exec approval and `systemRunPlan` safeguards rather than adding a direct markdown-to-shell execution path.
- Current markdown rendering has no reliable command target metadata or risk classifier, so cwd/host binding and read-only/destructive/external labels need explicit design and tests.
