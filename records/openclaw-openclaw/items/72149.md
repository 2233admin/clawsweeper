## Summary

Keep this PR open. Current main still forwards `temperature` on the native Anthropic Messages path for Claude Opus 4.7 when thinking is disabled, and the PR is a narrow fix aligned with Anthropic's current migration guidance; the separate Bedrock Opus 4.7 temperature fix does not cover this native transport path.

## What This Changes

The PR updates the native Anthropic transport to omit `temperature` for Claude Opus 4.7 requests, normalizes the Opus 4.7 model-id helper, and adds a regression test for the omitted payload field.

## Best Possible Solution

Land this PR or an equivalent native Anthropic transport patch that strips `temperature` for Claude Opus 4.7 while preserving current behavior for other Anthropic models, then run the targeted Anthropic transport test and the normal changed gate before merge.

## Reproduction Assessment

Yes. A focused current-main check shows `claude-opus-4-7` with `options.temperature = 0.2` and no reasoning still enters the native `params.temperature` assignment path; related #72123 supplies live logs showing the corresponding Anthropic 400 response.

## Solution Assessment

Yes. The proposed fix is the narrowest maintainable boundary for the reported native Anthropic request-shape bug: remove the unsupported field at payload construction for Opus 4.7 only. Current native code does not appear to forward `top_p` or `top_k`, so a broader sampling filter is not required for this PR.
