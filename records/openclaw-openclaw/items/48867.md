## Summary

Current main does not contain a separate fix for the Android/Termux Chrome MCP stdin parent-death path. The issue has an explicit maintainer comment distinguishing it from the merged Chrome MCP command/args work in #71560, and the browser plugin still launches Chrome DevTools MCP through the SDK stdio transport without Termux-specific lifecycle/stdin handling.

## Best Possible Solution

Keep this issue open. The best fix is a targeted browser Chrome MCP lifecycle change for Android/Termux, likely by moving the Chrome MCP launch onto OpenClaw's managed stdio transport or adding explicit stdin/lifecycle handling, plus a focused regression test that simulates the Chrome MCP subprocess closing after parent-death/stdin-end stderr and verifies `profile="user"` reports or recovers correctly. The #71560 command/args work should remain related context, not the canonical closure.
