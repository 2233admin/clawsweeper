## Summary

Keep open. Current main still has token/setup-token persistence paths that write only to the selected agent auth store, while sibling-agent fan-out is present only in the OAuth helper. Live GitHub state also shows PR #52912 is open, unmerged, and uses closing syntax for this issue, so the issue should remain open until that implementation candidate lands or is closed.

## Best Possible Solution

Keep this issue open and review PR #52912. The best fix should centralize token auth-profile writes so interactive provider setup-token, non-interactive Anthropic setup-token, and `models auth paste-token` can all sync sibling agent stores consistently, with regression coverage for multi-agent installs. Follow up with clearer Anthropic setup-token prompt/docs text and post-write validation so successful onboarding cannot leave missing auth profiles silently.
