## Summary

Keep open. Current main still persists Anthropic setup-token and pasted token auth profiles only to the selected agent store, while sibling-agent fan-out exists only in the OAuth helper. The supplied GitHub context also shows open PR #52912 is unmerged and uses closing syntax for this issue, so this issue should stay open until that implementation candidate lands or is closed.

## Best Possible Solution

Keep this issue open and review PR #52912. The best fix is to centralize token auth-profile writes so interactive provider setup-token, non-interactive Anthropic setup-token, and `models auth paste-token` can all sync sibling agent stores consistently, with regression coverage for multi-agent installs. Then add clearer Anthropic setup-token prompt/docs text and post-write validation so onboarding cannot report success while leaving missing auth profiles.
