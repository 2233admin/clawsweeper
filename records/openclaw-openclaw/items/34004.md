## Summary

Current main has Discord presence support, including `set-presence` and `autoPresence`, but it does not implement the requested Discord `self-profile` action, `channels.discord.actions.selfProfile` gate, bot nickname/avatar mutation, strict self-target checks, multi-account `accountId` requirement, avatar validation, or `bestEffort` partial result behavior. The related PR #34010 was closed unmerged, and the broader related issue #33270 is closed, so this issue remains the best open tracker for the remaining Discord self-profile work.

## Best Possible Solution

Keep this open and implement the feature in the bundled Discord plugin, not generic core: add a `self-profile` message action in `extensions/discord`, add `channels.discord.actions.selfProfile` to the Discord config schema/types/docs with default `false`, support bot self nickname/avatar/status/activity updates, reject non-self selectors, require explicit `accountId` when multiple enabled Discord accounts exist, validate avatar MIME/base64/size before outbound mutation, and cover the behavior with focused Discord action/runtime tests plus docs.

## Risks / Open Questions

- Profile mutation is security-sensitive: a future implementation should stay default-off, self-only, and avoid adding arbitrary member edit paths beyond the existing moderation surfaces.
- The existing `set-presence` action already covers part of the requested status/activity behavior under `actions.presence`; adding `self-profile` should avoid silently changing existing `set-presence` semantics or confusing the two gates.
- Multi-account behavior needs explicit account routing before mutation; current presence code can use the default gateway when `accountId` is omitted.
