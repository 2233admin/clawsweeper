## Summary

Keep this issue open. Current main still does not implement write-time Markdown code-fence linting or auto-fix for workspace Markdown files: the requested utility/config/skip API is absent, the existing Markdown fence helper only parses existing fences, and the central workspace write paths still persist caller-provided Markdown unchanged. The related implementation PRs #37628 and #41207 were both closed unmerged, so they remain prior art rather than a shipped fix.

## Best Possible Solution

Keep this issue open until maintainers either implement a central Markdown code-fence lint/fix utility with sh/ts/python/json inference, change records, warn/auto-fix configuration, per-write skip support, and focused coverage across current workspace Markdown write paths, or explicitly decide that OpenClaw should not mutate workspace Markdown at write time.
