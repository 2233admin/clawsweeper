## Summary

Keep this PR open. Current main does not implement the requested exec child-process session/agent env vars, and this branch is useful source work for #66705, but it is not mergeable as-is because the node-host path puts trusted values into the normal env override payload that both node runtimes filter for shell-wrapper execution; it also lacks the required changelog entry and needs maintainer trust-model approval for exposing session routing metadata to every exec child process.

## What This Changes

The PR adds `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` injection for exec child processes across gateway/sandbox/PTY/node paths, plus docs and regression tests.

## Best Possible Solution

Revise the PR so local gateway/sandbox/PTY execution injects trusted values after user env, while node-host execution carries session/agent metadata separately from user env overrides and merges it into the final child env after shell-wrapper filtering in both TypeScript and macOS runtimes. Keep override prevention, add final-child node coverage and a changelog entry, then run the changed gate in Testbox before merge.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: `host=node` exec builds a shell-wrapper command and sends `requestedEnv` as `params.env`, while both node-host implementations filter shell-wrapper env overrides and drop non-allowlisted `OPENCLAW_*` keys before spawning the child.

## Solution Assessment

No. Adding these trusted values to `requestedEnv` is not the best fix for node execution because that payload is intentionally untrusted override data; the safer solution is a separate trusted runtime metadata path merged after sanitizer policy is applied.
