## Summary

Current main still does not implement this PR's intended behavior. Exec child processes on main receive OPENCLAW_SHELL, but not OPENCLAW_AGENT_ID or OPENCLAW_SESSION_KEY, and the docs/tests for the requested behavior are not present. The PR remains a focused implementation candidate for the linked open request #66705 rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review as the focused implementation candidate for #66705. The best path is to land this PR or an equivalent maintainer patch that injects trusted OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY across gateway, sandbox, PTY, and node exec paths, documents always-set empty-string semantics, and keeps regression coverage for override and clearing behavior.
