## Summary

Keep open. Current main does not implement this PR’s intended behavior: exec child processes still only receive OPENCLAW_SHELL, while OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY are absent from the local/sandbox/PTY runtime env, node-host requested env forwarding, docs, and tests. The PR is a focused implementation candidate for the linked open request #66705, not something already shipped on main.

## Best Possible Solution

Review and land this PR, or an equivalent maintainer patch, as the implementation candidate for #66705. The desired path is to inject trusted OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY across gateway, sandbox, PTY, and node exec paths, document empty-string and override semantics, and keep regression tests for each execution path.
