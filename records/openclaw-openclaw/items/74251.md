## Summary

Keep open. Current main still exposes the reported WSL2 global npm install path and continues to support Node 22.14+, so the Node 22.22.x WSL2 V8 fatal install crash is not obsolete. The Node 24 default helps missing-runtime installs, but it does not cover systems where Node 22.22.x is already active or identify the crashing npm lifecycle step.

## What This Changes

The issue reports that the official macOS/Linux/WSL installer crashes during `npm install -g openclaw@latest` on WSL2 Ubuntu 24.04 with Node 22.22.2, blocking installation and onboarding.
