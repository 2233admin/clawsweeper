## Summary

Keep open. Current main improves the clean missing-runtime path by defaulting the installer to Node 24, but it still officially supports and accepts existing Node 22.14+ runtimes, still installs through global npm, and has no V8/Turboshaft/RepresentationChanger-specific recovery or documented WSL2 workaround. The closed duplicate #74277 points back here, so this remains the canonical fresh install-crash report.

## What This Changes

The issue reports that the official WSL2/Linux installer reaches `npm install -g openclaw@latest` under Node v22.22.2 and crashes with native V8/Turboshaft fatal errors, blocking installation.

## Best Possible Solution

Reproduce the official installer on WSL2 Ubuntu 24.04.4 with Node 22.22.x present and with a clean no-Node path, capture the npm debug log and V8 crash dump, then land the narrow owner-path fix: installer runtime selection/diagnostics/workaround in `scripts/install.sh` if Node selection triggers it, or package lifecycle hardening around `scripts/postinstall-bundled-plugins.mjs` if that script is proven to crash npm/Node.

## Reproduction Assessment

Unclear. The report gives concrete WSL2 steps and native V8 crash traces, and current main still exposes the accepted Node 22 plus global npm install path, but this review did not execute WSL2 to reproduce it end to end.

## Solution Assessment
