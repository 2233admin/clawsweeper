## Summary

Keep this PR open for changes before merge. The core metadata shape is useful and the latest head addresses the Greptile null-vs-undefined edge cases, but the Telegram skipped-history path still cannot classify the linked sibling-bot repro, and one new exact-string test is stale against current main.

## What This Changes

The PR adds optional bot-targeting metadata to auto-reply history/template context, renders it in prompt metadata, and wires Telegram mention/reply parsing plus tests to populate it.

## Best Possible Solution

Keep the additive metadata contract, but only merge after the Telegram path can classify sibling-bot mentions in skipped history through a trusted per-group sibling-bot list or an explicit non-current-bot mention signal. The final patch should rebase onto current main and cover current-bot, sibling-bot, reply-to-bot, and ambient-history cases with focused tests.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current main stores skipped Telegram group messages without targeting metadata, and the PR head still passes only the current bot username to the new detector for that skipped-history branch.

## Solution Assessment

No. The core fields and prompt rendering are a reasonable direction, but the proposed Telegram wiring is not yet the narrowest complete fix for the linked multi-bot repro, and the stale exact-string test needs a current-main update.
