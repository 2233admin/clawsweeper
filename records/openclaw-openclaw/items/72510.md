## Summary

Keep this PR open. Current main still lacks the bot-targeting metadata path the linked report asks for, but the proposed patch still misses the central skipped-history case for messages addressed to a sibling bot and needs a rebase/test update against current group prompt behavior.

## What This Changes

The PR adds optional bot-targeting metadata to auto-reply history/template context, renders it in prompt metadata, and updates Telegram mention/reply parsing plus tests to populate those fields.

## Best Possible Solution

Keep the additive metadata contract, rebase onto current main, and wire Telegram skipped-history detection to a trustworthy known-bot list or an explicitly scoped alternative that covers the `@otherbot` repro. The final patch should preserve current group prompt guidance and test current-bot, sibling-bot, reply-to-bot, and ambient history annotations.
