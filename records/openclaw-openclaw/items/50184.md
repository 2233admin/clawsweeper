## Summary

Keep this issue open. Current main still matches the reported regression: Telegram DM streaming lanes are created with `previewTransport: "message"`, which bypasses the lower draft stream's `auto` behavior that would prefer `sendMessageDraft` for DM threads. The issue comments correctly identify the transport-precedence problem, and PR #47631 is an open, unmerged implementation candidate rather than a reason to close this issue now.

## Best Possible Solution

Keep this issue open and resolve it through PR #47631 or an equivalent Telegram plugin patch: let the answer lane pass `auto` so `createTelegramDraftStream` can choose native draft transport for DM threads, keep or explicitly decide the reasoning-lane transport, preserve materialization and duplicate-final guards, update the tests that currently assert forced `message`, and align Telegram docs/config if a DM transport preference is added.
