## Summary

Keep open. Current main still matches the reported regression: the Telegram dispatcher forces DM answer streaming through message-preview transport, even though the lower draft-stream helper would choose native draft transport for DM threads when left on auto. The issue is already paired with open PR #47631, so the right cleanup action is maintainer review of that implementation rather than closing this issue or queuing a separate repair.

## What This Changes

The issue requests restoring Telegram direct-message answer preview streaming so the reply layer does not force message transport over the lower draft-capable transport selection.

## Best Possible Solution

Land #47631 or an equivalent Telegram plugin change that lets the DM answer lane pass auto so createTelegramDraftStream can choose native draft transport, keeps reasoning on message transport, preserves duplicate-final safeguards, and aligns the Telegram docs/config surface with the selected behavior.
