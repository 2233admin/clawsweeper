## Summary

Current main still has the live reply-run deadlock path described by this issue. Replay/session-history repair can synthesize missing tool results after history rebuild, but `runReplyAgent` still drains `pendingToolTasks` with an unbounded `Promise.allSettled`, and that await occurs before the reply operation cleanup that releases the active session run. The related open PR #64733 remains the right implementation candidate or reference for an equivalent fix.

## Best Possible Solution

Keep this issue open. Review and land PR #64733, or an equivalent main-branch fix, that bounds or cancels the `pendingToolTasks` drain in `runReplyAgent`, preserves ordered tool-result delivery, ensures `replyOperation.complete()` or failure cleanup can run, logs/surfaces timeout cleanup, and adds focused regression coverage for a never-settling tool-result delivery task.
