## Summary

Keep open. Current main still has the cron-side observability gap described in the report: the embedded runner now records `sessions_yield` as paused/yielded, but the isolated cron outcome type, final classifier, and timer bookkeeping still treat the first yielded parent turn as a terminal successful cron run.

## What This Changes

The issue asks for isolated cron jobs using `sessions_yield` after spawning a subagent to keep cron status/duration aligned with the full parent-plus-subagent lifecycle.

## Best Possible Solution

Cron run bookkeeping should treat yielded isolated agent turns as nonterminal until the resumed parent and descendant subagents reach a real terminal outcome, so `finished` and `durationMs` represent the full orchestration. If maintainers want a staged path, add explicit yielded/pending metadata first while keeping deferred finalization tracked.

## Reproduction Assessment

Yes. The report gives a concrete isolated cron `agentTurn` using `sessions_spawn`, `sessions_yield`, and a long-running child; current source inspection shows that path still returns a normal `ok` cron outcome before any yielded lifecycle is consumed.

## Solution Assessment

Unclear until maintainers choose the public cron contract. Deferred finalization best matches the expected behavior, while a yielded marker is safer and narrower but only partially addresses monitoring accuracy.
