## Summary

Keep open. Current main still has the embedded-runner half of the yield signal, but the cron bookkeeping path does not consume it before recording a terminal successful run, so the reported monitoring gap remains.

## What This Changes

The issue asks cron run status and duration to account for an isolated parent session yielding to a subagent instead of reporting `ok` immediately after `sessions_yield`.

## Best Possible Solution

Cron run bookkeeping should treat yielded isolated agent turns as nonterminal until the resumed parent and its descendants reach a true terminal outcome, so run logs, task state, and duration describe the full orchestration. If a staged rollout is preferred, yielded/pending metadata should be explicit and documented as an interim state.

## Reproduction Assessment

Yes. The issue provides a concrete isolated cron `agentTurn` with `sessions_spawn`, `sessions_yield`, and a long-running child, and source inspection shows the yielded metadata is produced but not consumed by cron finalization.

## Solution Assessment

Unclear until maintainers choose the lifecycle contract. Deferred finalization best satisfies the reported expected behavior; a yielded marker is a narrower interim improvement but not a complete fix for duration accounting.
