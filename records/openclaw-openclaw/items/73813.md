## Summary

Keep open. Current main still has the reported observability gap: the embedded runner exposes yielded/paused metadata, but the isolated cron path still maps the first embedded result to terminal `status: "ok"` and the cron timer records duration at that point.

## What This Changes

The issue asks cron run bookkeeping for isolated `agentTurn` jobs that call `sessions_yield` after spawning subagents to stay nonterminal, or at least report duration through the resumed parent/subagent completion.

## Best Possible Solution

Cron should model yielded isolated agent turns as nonterminal for run bookkeeping, keeping the cron task/run active until the resumed parent and descendants reach a real terminal result before emitting `finished` and computing duration. If maintainers choose an incremental path first, it should explicitly expose yielded/paused metadata and keep the full deferred-duration fix tracked.

## Reproduction Assessment

Yes. The issue provides a concrete isolated cron `agentTurn` path using `sessions_spawn`, `sessions_yield`, and a long-running child, and current source inspection confirms cron finalization still follows the first yielded embedded result rather than the later resumed lifecycle.

## Solution Assessment

Unclear until maintainers choose the cron contract. Deferring the `finished` event is the most complete solution for status and duration, while adding a yielded marker is a smaller compatible interim but only partially solves the reported behavior.
