## Summary

Keep open. Current main still has the observability gap: yielded embedded runs are now represented as paused/yielded metadata, but isolated cron does not consume that metadata and still finalizes the cron run as `ok` with a duration measured only through the first parent turn. The related closed issues covered survival/reactivation failures, while this issue remains the cron bookkeeping/duration variant.

## Best Possible Solution

Keep this issue open and have a maintainer choose the cron contract direction. The strongest product fix is to keep yielded isolated cron runs in a running or explicit pending-yield state, then emit the final `finished` event and duration only after the resumed parent and descendants reach a real terminal outcome. If maintainers choose the smaller additive path first, add `yielded?: true` to the cron outcome/event/run-log contract and document that duration is partial for yielded runs.
