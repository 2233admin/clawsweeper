## Summary

Keep open. Current main still has an unguarded malformed `message.content` path in the OpenAI Responses/WebSocket conversion boundary, and the related fixing PRs discussed on the issue are closed unmerged. This is not implemented on main and should not be closed as stale or duplicate.

## Best Possible Solution

Keep this issue open and land a focused core fix at the OpenAI Responses/WebSocket ingress and replay boundaries: guard `response.output[].content` with `Array.isArray` before `.some` or iteration, normalize or deliberately drop string/object/null assistant content with logging, review the OpenResponses HTTP guard in light of its schema, and add regression tests for malformed provider payloads. Reuse the analysis from #43836 and #43864, but do not close until a patch is merged and verified.
