## Summary

Keep open: current main still emits the ambiguous TUI pairing-required hint without the concrete approve command, and open PR #73208 already targets the exact fix with closing syntax, so closure should wait for that PR to merge or be closed.

## What This Changes

The issue asks for the TUI pairing-required disconnect message to tell users to run `openclaw devices approve <request-id>` after listing pending requests.

## Best Possible Solution

Review and land or close PR #73208. The desired end state is for the TUI disconnect hint to match the existing CLI/docs flow by naming `openclaw devices list` and `openclaw devices approve <requestId>`, preferably using the existing pairing close-reason parser when an exact request ID is available.
