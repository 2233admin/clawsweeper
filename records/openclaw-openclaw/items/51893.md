## Summary

Current main already implements the intended custom-provider vision input behavior: non-Azure custom onboarding infers common vision model IDs as text+image, prompts or allows explicit override for unknown/forced cases, and has regression coverage including `claude-sonnet-4-6`. The PR's core fix is obsolete on main.

## Best Possible Solution

Keep the implementation already on main and close this older PR as superseded by shipped/current-main code. Any remaining refinements to the model-name heuristic should be handled as fresh, narrow follow-up issues or PRs against `src/commands/onboard-custom-config.ts`.
