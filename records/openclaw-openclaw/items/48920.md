## Summary

Keep this issue open. The specific `isolatedSession` mismatch from OpenClaw 2026.3.13 has since caught up in current main and in the latest release checked here, but the broader live-docs-versus-release drift remains: the public docs publish workflow mirrors `docs/**` from every `main` push, while current main is already versioned ahead of the latest release. Related #71301 overlaps on version-matched bundled docs for agents, but it does not fully cover the human-facing live docs version/gating request in this issue.

## Best Possible Solution

Keep the issue open for maintainer product/release-policy follow-up. The best path is to decide whether docs.openclaw.ai should remain latest-main with a visible source/version warning, publish from release tags, add per-feature availability callouts, or add CI that catches docs examples/config keys not present in the latest release; coordinate with #71301 for the complementary bundled/local docs work.
