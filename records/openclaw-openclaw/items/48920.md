## Summary

Keep open. The original `isolatedSession` mismatch is resolved in the latest release, but the broader live-docs-versus-release drift is still present: current `main` documents and accepts `heartbeat.skipWhenBusy`, while release `v2026.4.26` does not, and the docs publish workflow still syncs docs from `main`.

## What This Changes

The issue asks OpenClaw to prevent live documentation from advertising unreleased heartbeat configuration keys that fail strict config validation on the latest shipped release.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then encode it in the publishing workflow, site metadata, version markers, and any generated config-docs or validation checks needed to keep documented config keys aligned with the intended user version. Treat #71301 as complementary bundled-docs work, not a complete replacement for this live-docs policy gap.

## Risks / Open Questions

- The remaining fix is a maintainer policy choice: live docs can track latest main, latest release, explicit version badges, or a versioned docs selector, and each option has different release and support tradeoffs.
- If maintainers keep `docs.openclaw.ai` as latest-main documentation, the product gap moves to clearer version availability metadata and version-aware config validation errors rather than a publish workflow change.
- Changing the publish workflow or docs bundle strategy may affect the separate publish repo and i18n sync path, so this should be handled deliberately rather than as a narrow automated fix.
