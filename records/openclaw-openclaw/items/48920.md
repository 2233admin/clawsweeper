## Summary

Keep open. The original `isolatedSession` release mismatch is resolved in the latest release, but the broader live-docs-versus-release bug remains: current main documents and accepts `heartbeat.skipWhenBusy`, while the latest release tag does not. The related bundled-docs proposal is complementary, not a complete replacement for deciding how public live docs should track releases.

## What This Changes

The issue asks for `docs.openclaw.ai` to stay aligned with released OpenClaw versions, or clearly show feature availability, so documented heartbeat config keys do not break released installs.

## Best Possible Solution

Define an explicit compatibility policy for public docs, then implement the matching publish workflow, version metadata, generated config-doc compatibility checks, and/or version-aware validation errors so users can tell whether a documented config key is available in their installed OpenClaw version.
