## Summary

Keep open. Current main already handles string-backed assistant content at runtime, but this PR still adds focused subscription-level regression coverage that current main does not contain. The PR is test-only and does not introduce security-sensitive production, CI, dependency, or release-surface changes.

## Best Possible Solution

Keep this PR open for maintainer review. The runtime behavior is already present on current main, but the exact plain-string `message_end` subscription regression is still useful and not duplicated; the best path is to land this XS test after normal rebase/CI validation, or explicitly decline the extra coverage on maintainer grounds.
