## Summary

Current main already handles string-backed assistant message content at runtime, but it still does not contain the subscription-level regression test this PR adds for plain-string `message_end` content populating `assistantTexts`. The PR remains a focused, test-only implementation candidate rather than something to close as already implemented.

## Best Possible Solution
