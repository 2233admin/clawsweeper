## Summary

Keep this open. Current main still discovers and watches the personal `~/.agents/skills` source unconditionally, and the public config surface has no source-level disable setting; existing allowlists, per-skill disables, and prompt limits happen after discovery and do not solve the requested scan-skip behavior.

## What This Changes

The issue requests a config setting to disable built-in skill source directories, especially the personal `~/.agents/skills` source, before discovery and prompt injection.

## Best Possible Solution

Add a backwards-compatible skills load setting that skips disabled built-in roots before both discovery and watcher registration, with defaults preserving today’s precedence and behavior. Keep the broader on-demand skill-selection/router architecture as a separate design item.
