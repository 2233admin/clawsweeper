## Summary

Keep this PR open. Current main still does not implement the PR’s intended native outbound WhatsApp location messages through the message tool; it only has inbound location parsing. The supplied PR discussion also identifies real functional blockers in the proposed wiring, so the useful path is revision/review rather than cleanup close.

## Best Possible Solution

Keep the PR in normal review and ask for a focused revision: add a supported generic location-send contract or plugin action seam, expose location in WhatsApp message-tool discovery, wire production WhatsApp outbound through createWhatsAppOutboundBase and Baileys, add a gateway location RPC with validation/dedupe, cover message-tool/core/gateway/WhatsApp send API tests, and avoid logging raw coordinates.
