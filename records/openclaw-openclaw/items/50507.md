## Summary

Keep this PR open. Current main still does not implement native outbound WhatsApp location pins through the message tool, and the unmerged PR remains a plausible implementation candidate for the closed #2214 request and the earlier closed #2298 attempt rather than conservative cleanup material.

## Best Possible Solution

Keep the PR in normal maintainer review. The best path is a focused revision that adds a supported location-send contract or plugin action seam, advertises location in WhatsApp message-tool discovery, wires the production WhatsApp outbound-base/channel-outbound path to Baileys, adds a gateway location RPC with validation/idempotency, covers message-tool/core/gateway/WhatsApp tests, and avoids logging raw coordinates.
