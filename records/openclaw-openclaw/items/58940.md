## Summary

Keep PR #58940 open. Current `main` still contains the exact outer `Promise.race` around `runExec(path, ["--version"], { timeoutMs: 3000 })` in `findTailscaleBinary`, so the PR is not implemented or obsolete on main. The root-cause claim is also supported by current `runExec`: it maps `timeoutMs` to Node `execFile`'s `timeout`, and the local Node implementation kills the child on that timeout. This remains a focused, plausible bug fix rather than conservative cleanup fodder.

## Best Possible Solution

Keep the PR open for maintainer handling. The best path is to land or recreate the scoped change that removes the outer `Promise.race` in `src/infra/tailscale.ts`, preferably with a targeted regression test or explicit CI/manual proof for the hanging `tailscale --version` lifecycle. If adding a process-level test is too brittle, the maintainer can still accept the minimal patch after green changed-gate validation because current main still has the race and `runExec` already owns the timeout.
