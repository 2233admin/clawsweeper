## Summary

Keep this PR open for normal maintainer review. Current main still has the redundant outer timeout in the Tailscale binary probe, and the proposed diff is a narrow lifecycle fix that preserves the 3-second probe behavior while letting `runExec`/Node `execFile` own process termination.

## What This Changes

This PR removes the outer `Promise.race` timeout around `runExec(path, ["--version"], { timeoutMs: 3000 })` in `findTailscaleBinary` so the Tailscale binary probe relies on `runExec`'s `execFile` timeout.

## Best Possible Solution

Land this narrow cleanup after normal maintainer review and CI, optionally adding a small regression test for the Tailscale binary probe if maintainers want executable coverage for the timeout ownership.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: `findTailscaleBinary` can reject from the outer timer while the `runExec`/`execFile` child promise remains outstanding; a live process-table repro would require a hanging `tailscale --version` binary and tight timing.

## Solution Assessment

Yes. Removing the outer `Promise.race` is the narrowest maintainable fix because it preserves the timeout value and catch behavior while letting the existing `execFile` timeout own child termination.
