## Summary

Keep open. Current main still lacks the agent namespace isolation this PR is trying to add, and the PR is an active stacked memory-isolation implementation, but the branch is not mergeable as-is because it contains blocking schema migration and current-main regression issues.

## What This Changes

The PR adds agent_id namespace scoping across the memory SQLite index schema, FTS/vector queries, write/delete/status paths, QMD mcporter daemon state, tool provenance/scope handling, and related tests/CLI provider initialization.

## Best Possible Solution

Keep this PR open, review or land the earlier stack pieces first, then require a rebased/narrowed branch that preserves current-main memory performance/concurrency fixes while applying agent_id isolation consistently across schema migration, FTS/vector/search/status/write/delete, and QMD mcporter state.

## Reproduction Assessment

Yes. A focused static reproduction is high-confidence: on the PR head, an existing FTS table without agent_id makes readColumns return a Set, existingFtsColumns.length is undefined, the rebuild is skipped, and later FTS writes that include agent_id fail against the old table.

## Solution Assessment

No, not as currently proposed. The product direction is the right owner-local fix for shared memory indexes, but this branch needs the package migration bug and current-main regressions fixed before it is the maintainable solution.

  - body: `readColumns` returns a `Set`, so `existingFtsColumns.length` is always `undefined` and this rebuild check never runs. Existing databases keep a `chunks_fts` table without `agent_id`, then later inserts that include `agent_id` fail instead of migrating the table.
  - confidence: 0.98

  - body: This signature drops `configuredNonBatchConcurrency` and `providerId`, but the unchanged timeout/concurrency tests still pass those fields and current main uses them to honor user non-batch limits and cap Ollama at 1. The branch will either fail the test typecheck or silently regress those settings.
  - confidence: 0.9
- **[P2] Keep fallback vector search streaming:** `extensions/memory-core/src/memory/manager-search.ts:210-223`
  - body: The fallback path now calls `listChunks()` and sorts every matching chunk in memory. Current main streams rows with a bounded top-K accumulator, so this reintroduces the large-index memory spike that was recently fixed.
  - confidence: 0.9
- **[P2] Keep embedding-cache seeding cooperative:** `extensions/memory-core/src/memory/manager-sync-ops.ts:310-351`
  - body: `seedEmbeddingCache` was changed from an async iterator loop with periodic `setImmediate` yields to a synchronous `.all()` load and insert loop. Large cache rebuilds can again block the event loop and delay gateway health probes or other work.
  - confidence: 0.86
