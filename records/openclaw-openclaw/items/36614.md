## Summary

Keep open. Current main still has the write-side contamination path: per-channel-peer DMs resolve to scoped session keys, but the shared recorder and several direct-message channel paths still persist channel delivery metadata to the main session key. The related closing PRs are closed unmerged, and merged #38418 only addressed an adjacent read-side WebChat inheritance symptom.

## Best Possible Solution

Keep this issue open and finish the write-side routing fix. The conservative path is to centralize inbound last-route target selection around route.lastRoutePolicy and resolveInboundLastRouteSessionKey, then update every direct-message writer and direct updateLastRoute caller so lastRoutePolicy="session" writes delivery metadata to the scoped inbound session while lastRoutePolicy="main" preserves the existing main-session behavior. Add regression coverage for the central recorder plus representative iMessage, Slack, Signal, LINE, Matrix, Mattermost, and Discord component paths, while keeping the existing Telegram and WhatsApp guarded behavior intact.
