## Summary

Keep open. Current main still discovers Slack scopes only through `auth.scopes` and `apps.permissions.info`, so the reported modern `xoxb-*` token failure remains plausible. The related `auth.test` fallback PR was closed unmerged, and current tests still do not cover the missing fallback path.

## Best Possible Solution

Keep this issue open. The Slack plugin should add a modern Slack app token scope discovery path in `extensions/slack/src/scopes.ts`, likely using `auth.test` plus `x-oauth-scopes` as the primary path or as a fallback after the legacy workspace-app methods, preserve legacy behavior, and add focused regression tests for `unknown_method` fallback and capabilities diagnostics.
