## Summary

Keep open. Current main still has the Slack capabilities path that can return the reported joined `unknown_method` error: scope discovery only calls the two legacy methods, does not read Slack SDK response metadata, and the capabilities command displays that helper failure. The prior fallback PR is relevant but closed unmerged, and bounded GitHub search found no open replacement for this issue.

## What This Changes

Slack channel capabilities should report bot scopes normally for modern Slack bot tokens instead of returning `unknown_method` from legacy Slack scope APIs.

## Best Possible Solution

Implement the fix inside the Slack plugin: support granular bot-token scope discovery through `auth.test` response metadata while preserving legacy method compatibility, and cover the helper plus capability diagnostics with focused tests.
