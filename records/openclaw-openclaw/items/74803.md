## Summary

Keep open. Current main still lacks the requested warning or docs guardrail for short isolated cron agent turns, and the provided GitHub context shows open unmerged PR #74893 with closing syntax, so this issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests a warning, safer default, or prominent documentation for isolated `agentTurn` cron jobs whose short explicit timeout can be consumed by cold-start bootstrap when `lightContext` and `toolsAllow` are not set.

## Best Possible Solution

Review and land PR #74893 or an equivalent narrow change that warns on risky isolated cron `agentTurn` create/update shapes and updates cron docs, while preserving explicit user timeout semantics.

## Reproduction Assessment

Yes. The source-level path is high-confidence: create an isolated `agentTurn` cron job with `timeoutSeconds: 60`, no `lightContext`, and no restricted `toolsAllow`; current code honors the exact timeout and starts it before embedded-runner workspace/runtime bootstrap completes.

## Solution Assessment

Yes for the requested direction: a targeted warning plus docs is the narrowest maintainable fix because it explains the cold-start budget without silently overriding an explicit timeout. The open PR should be reviewed as the current implementation candidate.
