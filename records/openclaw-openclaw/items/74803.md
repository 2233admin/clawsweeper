## Summary

Keep open. Current main still has the reported cold-start budget gap, and the issue is now paired with open PR #74893 using closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a warning, safer default, or prominent documentation for isolated cron `agentTurn` jobs whose short explicit timeout can be spent on full bootstrap when `lightContext` and `toolsAllow` are not set.

## Best Possible Solution

Review and land PR #74893 if it implements a narrow create/update guardrail plus docs, or replace it with an equivalent conservative fix that preserves explicit timeout semantics while making the cold-start budget risk visible.

## Reproduction Assessment

Yes. A focused reproduction is available by creating an isolated `agentTurn` cron job with a short explicit timeout, no `lightContext`, and no restricted `toolsAllow`; current code shows the timer starts before embedded bootstrap work, matching the reported timeout-before-tool-call behavior.

## Solution Assessment

Yes. A warning/docs guardrail is the best narrow direction because silently inflating or ignoring a user-specified timeout would be more surprising than making the risky configuration discoverable.
