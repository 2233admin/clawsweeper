## Summary

Keep open. Current main still preserves the short explicit isolated cron `agentTurn` timeout behavior and does not add a cold-start warning; open PR #74893 already references this issue with closing syntax and should be reviewed before this issue closes.

## What This Changes

Request a guardrail for isolated cron `agentTurn` jobs with short explicit timeouts and no `lightContext` or `toolsAllow` so cold-start bootstrap does not consume the whole budget without useful guidance.

## Best Possible Solution

Review PR #74893 and land a narrow warning/docs guardrail for short explicit isolated cron `agentTurn` jobs without lightweight context or a tool allowlist, while preserving explicit timeout semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: create an isolated `agentTurn` cron job with a short explicit `timeoutSeconds`, no `lightContext`, and no `toolsAllow`; current main starts the outer timer before later workspace/runtime bootstrap work finishes.

## Solution Assessment

Unclear until PR #74893 is reviewed. A targeted warning and documentation fix looks safer than silently changing explicit timeout behavior, but maintainers still need to choose the threshold and user-facing surface.
