## Summary

Keep open. Current main still routes announce handoffs through a child-completion internal event plus rewrite instruction, and I found no requester conversation summary or bounded parent-session context in the prompt shape, active wake path, dormant direct-agent path, docs, or tests. The added Telegram stale-session report is useful adjacent evidence, but it is not resolved by current main and may need to stay linked to the broader completion-routing work.

## What This Changes

Issue requests that sub-agent or cron `delivery.mode: "announce"` handoffs include enough requester-session context to avoid out-of-context parent replies.

## Best Possible Solution

Add a bounded requester-context handoff for subagent and cron announce replies, using the requester session state where available and covering both active and dormant delivery paths with regression tests; keep any stale Telegram session-key routing fix either narrowly covered in the same seam or split to the existing routing cluster.

## Repair Work Prompt

Fix this issue by adding a bounded requester-context handoff for subagent and cron announce completion replies. Current main builds the announce trigger from only the child completion event plus rewrite instruction, so the requester session can answer without enough ongoing conversation context. Implement the fix in the core announce path, likely around `src/agents/subagent-announce.ts`, `src/agents/internal-events.ts`, and `src/agents/subagent-announce-delivery.ts`, so both active requester wakes and dormant direct-agent announce calls receive the same bounded requester context. Avoid copying raw or unbounded transcripts into internal events; prefer an existing sanitized/redacted session-history path or a compact requester-summary field with limits. Add regression coverage showing the announce prompt includes the bounded requester context and still redacts/limits sensitive raw transcript content. Consider the later Telegram stale-session report as related evidence, but do not broaden into a channel-specific routing rewrite unless the same generic seam is necessary; otherwise leave that for the existing routing cluster. Update `docs/tools/subagents.md` if the public announce-context contract changes, and add a user-facing changelog fix entry crediting the reporters. Do not change bundled channel behavior or provider defaults unrelated to announce handoff.
