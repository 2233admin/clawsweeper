## Summary

Keep open. Current main still drives subagent and cron announce replies from a child `task_completion` event plus a rewrite instruction, and the active and dormant delivery paths pass that trigger text without an explicit requester-context summary or regression coverage for the reported prompt-context failure. The related items cover routing/account/delivery resilience rather than this out-of-context completion-reply behavior.

## What This Changes

The issue requests that subagent and isolated cron announce completion handoffs include enough parent/requester conversation context for generated user-facing replies to stay on topic.

## Best Possible Solution

Add a bounded requester-context handoff for subagent and cron announce replies, using the parent/requester session state where possible and covering active and dormant delivery paths with regression tests, without leaking unrelated raw transcript content into child completion events.
