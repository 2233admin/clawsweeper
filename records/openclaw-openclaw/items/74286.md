## Repair Work Prompt

Fix the subagent/cron completion announce context bug described in this issue. Current main builds the announce `triggerMessage` from `src/agents/subagent-announce.ts` internal task-completion metadata only, formats it in `src/agents/internal-events.ts`, and sends it through `src/agents/subagent-announce-delivery.ts` as an `inter_session` gateway `agent` call. Implement a narrow fix so non-subagent requester announce turns have enough bounded requester conversation context to produce a contextually appropriate user-facing reply, especially for isolated cron runs that orchestrate subagents. Prefer existing session/transcript helpers and a small sanitized/limited context summary; do not dump full raw transcripts, expose internal metadata, change child subagent isolation defaults, or alter `context: "fork"` semantics. Add regression coverage for the direct announce path and the isolated cron/subagent follow-up path where feasible. Update `docs/tools/subagents.md` or cron docs only if the user-visible completion handoff behavior changes. Changelog only if the final PR changes user-facing behavior.
