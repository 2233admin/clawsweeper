## Summary

Keep this issue open. Current main still sends normal Feishu inbound messages through a same-chat promise queue before they reach the gateway reply pipeline, so `messages.queue.mode = "collect"` can still miss the active-run window and process later messages as separate turns. The related reports are useful context, but #70133 covers hung/stuck Feishu queue behavior and #67793 is broader, so neither is a better canonical replacement for this focused Feishu collect regression.

## Best Possible Solution

Fix the Feishu inbound dispatch path so normal same-chat messages can reach `dispatchReplyFromConfig` while the first run is still active, allowing gateway `collect` to enqueue and batch them. Preserve Feishu dedupe/debounce behavior plus the existing abort and `/btw` out-of-band lanes, and add a Feishu regression test that simulates one active run followed by multiple same-chat messages under `messages.queue.mode = "collect"`.
