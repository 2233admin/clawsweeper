## Summary

Keep this PR open. Current main still drops `accountId` at the `/hooks/agent` boundary, so the underlying bug is real, but the branch is stale and currently incorrect: it reintroduces a raw hook-name prompt-safety regression, conflicts with current main's newer cron message-tool policy, and misses required docs/changelog follow-through.

## What This Changes

The PR adds `accountId` parsing and forwarding for `/hooks/agent`, routes hook dispatch through cron normalization, adjusts cron message-tool policy, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land a rebased gateway-owned fix that trims and forwards `accountId`, includes it in hook replay scope, preserves sanitized hook metadata in cron prompt context, keeps current main's announce message-tool semantics, documents the new request field and trust model, and credits the contributor in the changelog.
