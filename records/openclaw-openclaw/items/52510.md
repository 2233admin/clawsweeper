## Summary

Keep open. Current main has partially tightened the macOS LaunchAgent scanner, but the doctor cleanup renderer still emits canonical gateway removal commands after any remaining extra gateway-like service is detected. The issue also has open same-author implementation candidates with closing syntax, so it should stay open until a PR is reviewed/merged or explicitly closed.

## Best Possible Solution

Keep this issue open and review PR #52515, or a rebased smaller equivalent, against current main. The best fix is to pass detected service context into cleanup hints or suppress canonical active-gateway removal hints, preserve the current scanner tightening for non-gateway companions, and add focused regressions for custom macOS LaunchAgents plus the reported `exec:keychain:` doctor SecretRef path.
