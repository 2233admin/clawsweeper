## Summary

Keep this issue open. Current main has useful partial cron diagnostics, including state counters, failure alerts, durable run-history JSONL, and generic liveness/readiness probes, but the central requested surfaces are still incomplete: scheduled/manual outcomes are not logged for every success, skipped, or non-throwing failure path; probes are not cron-aware; and `openclaw cron list` does not annotate overdue or repeatedly failing/skipped jobs. The overlapping diagnostics PR #44363 remains open, while #46501 was closed unmerged.

## What This Changes

The issue requests cron execution observability: per-attempt logs, skipped/failure warnings, cron health reporting, and `openclaw cron list` warnings for unhealthy jobs.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual run outcomes; warns on skipped and non-throwing error results; exposes cron health through the chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures/skips while preserving the existing run history, counters, and failure-alert behavior.
