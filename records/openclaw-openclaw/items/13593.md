## Summary

Keep this issue open. Current main still has cron state tracking, JSONL run history, failure alerts, background task records, and generic `/health`/`/readyz` probes, but it does not implement the central requested observability: structured logger output for every cron attempt, warning logs for skipped/non-throwing failures, cron-aware health/readiness, or `openclaw cron list` annotations for overdue/repeatedly failing jobs. The directly relevant PR #46501 is closed unmerged, while open PR #44363 overlaps cron diagnostics enough that maintainers should coordinate rather than start duplicate automated work.

## Best Possible Solution

Keep this issue open. The best path is a focused cron-observability change: add structured logging for scheduled timer runs, startup catch-up/missed runs, and manual runs; log warnings for skipped and non-throwing error outcomes; expose cron health either as `/health/cron` or as an explicit component in the existing readiness/health model; and add `openclaw cron list` annotations for overdue and repeatedly failing jobs. Review #44363 and the closed unmerged #46501 for prior art, but do not treat either as shipped.
