## Summary

Keep this issue open. Current main has cron counters, failure alerts, durable run-history JSONL, and generic health/readiness probes, but it still lacks the central requested cron observability: per-attempt structured logs for success/skipped/non-throwing failures, cron-aware health/readiness, and `openclaw cron list` warnings. The overlapping diagnostics PR #44363 is still open, while #46501 was closed unmerged.

## What This Changes

The issue requests structured cron execution logging, cron health signaling, and CLI warnings for failed, skipped, overdue, or repeatedly unhealthy scheduled jobs.

## Best Possible Solution

Land a coordinated cron-observability change that logs scheduled, startup catch-up, and manual run outcomes with bounded structured metadata; warns for skipped and non-throwing error results; exposes cron health through the chosen probe surface; and annotates cron list output for overdue jobs and repeated failures/skips. #46501 is useful prior art but should not be treated as shipped.
