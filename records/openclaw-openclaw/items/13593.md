## Summary

Keep this issue open. Current main has cron state counters, failure notifications, durable run-history JSONL, and generic `/health`/`/ready` probes, but it still lacks the central requested cron observability: per-attempt structured success/skipped/non-throwing failure logs, cron-aware health/readiness, and `openclaw cron list` warnings for overdue or repeatedly failing/skipped jobs. The related implementation PR #46501 is closed unmerged, and open PR #44363 overlaps cron diagnostics enough that maintainers should coordinate scope before starting duplicate automated work.

## What This Changes

The issue requests cron execution observability: per-attempt success/failure/skipped logging, cron health signals, and CLI warnings for overdue or failing jobs.

## Best Possible Solution

Keep this issue open and coordinate with #44363. The best fix is a focused cron-observability change: log scheduled, startup catch-up, and manual run attempts with status, timing, job id/name, and bounded errors; warn for skipped and non-throwing error outcomes; add cron health either as `/health/cron` or as an explicit component in the existing health/readiness model; and annotate `openclaw cron list` for overdue jobs and repeated failures/skips. #46501 is useful prior art but should not be treated as shipped.
