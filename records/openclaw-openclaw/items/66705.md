## Summary

Keep this issue open. Current main still has the trusted session and agent context available in exec setup, but it does not inject `OPENCLAW_SESSION_KEY` or `OPENCLAW_AGENT_ID` into child-process environments, and the open implementation PR #73349 uses closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Expose the active exec session key and agent id to child processes as `OPENCLAW_SESSION_KEY` and `OPENCLAW_AGENT_ID` environment variables.

## Best Possible Solution

Review and land or close #73349. The final implementation should inject trusted `OPENCLAW_SESSION_KEY` and `OPENCLAW_AGENT_ID` consistently across gateway, sandbox, PTY, Docker/backend, and node-host execution, with regression tests and docs for precedence over user-provided env values.
