## Summary

Keep this PR open. Current main still returns non-agent-scoped colon session keys unchanged in `sandbox explain`, so the PR’s intended normalization is not implemented on main. The PR also now addresses the earlier review gap by asserting the downstream `elevated.channel` and `allowedByConfig` behavior, but it likely still needs coordination with #51584 because current main’s channel inference depends on the active plugin channel registry.

## Best Possible Solution

Keep the PR open as a focused implementation candidate. Rebase it onto current main, retain the non-agent-scoped colon-key normalization, and either coordinate with #51584 or include the static channel-registry fallback so the regression test proves `sessionKey`, `elevated.channel`, and `allowedByConfig` end to end. If maintainers decide unscoped Telegram slash keys are no longer supported, update the slash-command docs and resolve #51245 explicitly instead.
