## Summary

Keep open. Current main still does not implement the documented/defaulted subagent thinking inheritance path: the resolver only considers explicit `sessions_spawn.thinking`, `subagents.thinking` on the target agent config, and `agents.defaults.subagents.thinking`. It does not read the requester session's current thinking level, the target agent's `thinkingDefault`, or `agents.defaults.thinkingDefault`, so the reported fallback-to-model-default behavior can still occur.

## Best Possible Solution

Keep this issue open. The likely fix belongs in the native subagent spawn planning path: make `resolveSubagentThinkingOverride` or its caller receive the requester session entry/requester agent config plus target config, resolve the documented precedence, persist the resolved child `thinkingLevel` before the child `agent` call, and add focused tests for explicit override, requester/per-agent subagent default, global subagent default, caller inheritance, and top-level `thinkingDefault` behavior.
