## Summary

Keep open. Current main still only exposes a Control UI assistant-message copy action labeled “Copy as markdown”; it writes the markdown text directly to the clipboard and has no Markdown/plain-text selector, no persisted copy-format preference, and no focused tests proving plain-text clipboard output.

## Best Possible Solution

Keep this issue open and implement a focused Control UI enhancement: add a compact Markdown vs plain-text copy choice, or a small persisted copy-format preference, in the assistant-message copy path. The implementation should live around ui/src/ui/chat/copy-as-markdown.ts and ui/src/ui/chat/grouped-render.ts, should define and test markdown-to-plain-text conversion behavior, and should avoid changing outbound channel sanitization unless deliberately covered.
