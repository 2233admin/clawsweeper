## Summary

Keep open for now. Current main appears to address the reported hard-coded image-tool timeout through `tools.media.image.timeoutSeconds` and matching per-model `timeoutSeconds`, with code, docs, tests, and changelog evidence. However, open same-author PR #56454 is still an unmerged implementation candidate that explicitly closes this issue, and fork PR #1581 mirrors the same head, so conservative cleanup should reconcile those PRs before closing the issue.

After that PR state is resolved, close this issue as implemented on main if no newer repro shows a remaining timeout gap.
