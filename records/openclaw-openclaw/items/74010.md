## Summary

Current main still does not implement this PR's intended behavior: both preflight compaction and memory-flush gates use `reserveTokensFloor` as the whole reserve budget, while the config/Pi settings path treats `reserveTokens` as the effective user-facing headroom with `reserveTokensFloor` as a floor. The live PR head is a focused implementation candidate for open #66830 and supersedes the closed, unmerged #69379 and #67297 attempts, but it should remain open for normal review and green CI.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent narrow core fix. The fix should resolve one effective compaction reserve from configured `reserveTokens`, `reserveTokensFloor`, and the small-context cap, then use that value consistently in preflight compaction and memory-flush gates with focused regression tests and preserved contributor credit for #69379/#67297.
