## Summary

Keep this issue open. Current main at b85edb3f0cf6c2a8bec06537af9211a044266a6c still does not implement Anthropic server-side compaction: the config/schema surface has no Anthropic compaction option, the Anthropic plugin wrapper only handles existing beta/header/service-tier/thinking-prefill behavior, the native Anthropic transport does not inject `context_management.edits`, and the streamed content/stop-reason model still lacks compaction block handling. Related PR #10214 is closed unmerged, while #54041 and #65287 are broader related trackers rather than clear canonical replacements for this focused request.

## Best Possible Solution

Keep this issue open as the focused Anthropic server-side compaction tracker until maintainers choose the config/API shape and relationship to #54041/#65287. A complete fix should add an explicit opt-in direct-Anthropic setting, validate and document it, inject the required Anthropic compaction beta plus `context_management.edits`, preserve and replay `compaction` blocks through native and wrapper streaming paths, handle `pause_after_compaction`/`compaction` stop behavior, account for compaction usage iterations, and add focused tests. ([platform.claude.com](https://platform.claude.com/docs/en/build-with-claude/compaction))
