## Summary

Keep open. Current main still does not provide the requested Anthropic-native server-side compaction path: config/schema support is absent, Anthropic request construction does not inject `context_management.edits`, stream parsing only preserves text/thinking/tool-call blocks, and Anthropic docs only cover 1M context beta rather than native compaction.

## What This Changes

Request asks OpenClaw to add opt-in Anthropic server-side compaction API support with config, Anthropic-only request injection, compaction response handling, logging, tests, and docs.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker and implement a maintainer-approved opt-in path in the Anthropic provider/transport/config/docs surface, including current beta/header handling, `context_management.edits`, compaction block history round-trip, `compaction` stop reason handling, usage metadata, and targeted tests coordinated with #54041 and #65287.
