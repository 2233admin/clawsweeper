## Summary

Keep this issue open. Current main still lacks Anthropic server-side compaction config/schema support, Anthropic `context_management.edits` request injection, compaction block streaming/history round-trip handling, and `compaction` stop-reason support. The closed unmerged #10214 remains useful prior art, while #54041 and #65287 are broader related trackers rather than a maintainer-designated replacement for this focused request.

## What This Changes

Request to add opt-in Anthropic server-side compaction support via config, direct Anthropic request injection, streaming/history handling, logging, and docs.

## Best Possible Solution

Keep this issue open as the focused Anthropic server-side compaction tracker. The best path is a maintainer-shaped implementation that adds an explicit opt-in direct-Anthropic setting or model param, validates and documents it, injects the required beta and `context_management.edits`, preserves provider compaction blocks through streaming/history replay, handles `compaction` stop reason and usage metadata, and coordinates naming with #54041 and #65287.
