## Summary

Keep this open. Current main has ingress `InputProvenance`, external-content marker wrapping, plugin approvals, and transcript-scoped persistence hooks, but it still does not carry provenance through tool results into durable state writes or expose read-time provenance for stored state. The related design items #48503, #48509, and #54508 are still open, and Kelisi808's PitStop comment adds concrete schema/audit asks that map to the same gap.

## Best Possible Solution

Keep this open as a core security/plugin-API design item unless maintainers explicitly choose #54508 as the canonical tracker. The concrete path is to design a generic provenance/taint contract for tool results and durable writes, likely alongside #48503 and #48509, then add read-time metadata, policy hooks, docs, and regression coverage across memory, files, cron, config, and preferences.
