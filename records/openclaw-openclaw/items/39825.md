## Summary

Keep this issue open. Current main still uses the same Discord voice join path: /vc join waits for @discordjs/voice Ready with a fixed 15s timeout, destroys the connection before registering a session when that wait aborts, and /vc status still reports no active session. Related #65039 and #63098 remain open and corroborate the same early voice join failure family rather than proving this is implemented or cleanly superseded.

## Best Possible Solution

Keep this open as a Discord plugin voice join readiness-abort tracker unless maintainers explicitly consolidate it into #65039 or #63098. The fix likely belongs in extensions/discord/src/voice/manager.ts with focused regression coverage for entersState failures before session activation, clearer pre-Ready networking diagnostics, and a maintainer decision between bounded retry/backoff, a longer or configurable channels.discord.voice.connectTimeoutMs, DAVE-enabled fallback, or documented rejection/deprecation of daveEncryption=false if Discord now requires DAVE.
