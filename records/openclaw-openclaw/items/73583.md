## Summary

Keep this PR open. Current main still routes native Telegram `/status` through the buffered reply dispatcher instead of the direct status helper, and it lacks the Telegram fallback for suppressed group finals. The generic group/channel `message_tool_only` contract already exists on main, but the Telegram-specific work in this PR is still an active implementation candidate, consistent with the prior ClawSweeper and Greptile review context.

## Best Possible Solution

Review and land this PR or an equivalent Telegram plugin fix after resolving the reviewer note. The implementation should special-case native Telegram `/status` after existing auth/session resolution, use `resolveDirectStatusReplyForSession`, deliver directly while preserving topic threading/reply-to behavior, keep `/fast status` on the generic native-command route, and intentionally document or test the Telegram suppressed-final fallback semantics for group/topic turns.
