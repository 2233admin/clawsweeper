## Summary

Keep open. Current main contains the partial native-command visibility fix from the related merged PR, but it still lacks the Telegram direct status fast path, the Telegram suppressed-source fallback hook, and the queued follow-up failure reply that this PR is trying to add. Because the branch changes core source-reply privacy behavior plus Telegram delivery, the right next step is maintainer review and validation, not cleanup closure.

## What This Changes

The PR routes Telegram native `/status` through direct status delivery, adds Telegram/core fallback handling for suppressed group/topic source replies, and routes queued follow-up run failures back to the originating Telegram topic with regression tests.

## Best Possible Solution

Land this PR or an equivalent maintainer-approved patch that preserves #73672's native-command visibility behavior, adds Telegram direct `/status` delivery with topic reply preservation, introduces a bounded suppressed-source fallback for Telegram group/topic replies, and routes follow-up run failures without leaking model/auth details.
