## Summary

Keep open. Current main still lacks the Telegram direct `/status` path, Telegram suppressed-source fallback, and follow-up failure reply, and the PR discussion includes fresh live Telegram topic reproduction. The latest PR is not merge-ready because its fallback can make explicitly tool-only Telegram group replies visible, which conflicts with the current group visibility contract.

## What This Changes

The PR changes Telegram native status delivery, Telegram group/topic reply fallback handling, topic sequential keys, core suppressed-source reply normalization, follow-up failure replies, and related regression tests.

## Best Possible Solution

Keep the PR active, but require the Telegram fallback to preserve explicit tool-only visibility settings while still fixing the 2026.4.27 default-regression path, then land with focused Telegram/core tests and maintainer-approved privacy semantics.

## Reproduction Assessment

Yes. Current main statically reproduces the silent-drop path for group/channel `message_tool_only` replies, and the PR discussion includes independent 2026-04-30 Telegram topic reproduction on v2026.4.27 with downgrade proof.

## Solution Assessment

No. The PR fixes real missing behavior, but the fallback should be gated so explicit `messages.groupChat.visibleReplies: "message_tool"` remains private, or moved behind a separate compatibility/config migration path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve explicit tool-only group visibility:** `extensions/telegram/src/bot-message-dispatch.ts:1029-1034`
  - body: The Telegram fallback sends any suppressed source reply through `sendPayload`, including rooms where the user explicitly configured `messages.groupChat.visibleReplies: "message_tool"`. That mode is documented as requiring the message tool for visible output, so ordinary assistant finals/blocks/progress would become visible in groups that were configured to keep them private. Please gate the compatibility fallback to the default regression case or add a separate opt-in so explicit tool-only configs remain private.
  - confidence: 0.86
