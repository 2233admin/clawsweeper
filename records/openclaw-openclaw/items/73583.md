## Summary

Keep this PR open. Current main still lacks the PR’s two central Telegram fixes: native `/status` still goes through the buffered reply dispatcher instead of the direct status helper, and Telegram group dispatch still does not pass the effective `sourceReplyDeliveryMode` into the reply runtime. The supplied PR discussion is consistent with the source check: this is an implementation candidate, not obsolete.

## Best Possible Solution

Review and land this PR or an equivalent Telegram plugin fix. The implementation should special-case native `/status` after existing auth/session resolution, call `resolveDirectStatusReplyForSession`, deliver the status reply directly while preserving topic threading and reply-to behavior, keep `/fast status` on the generic native-command route, and pass Telegram’s effective group `sourceReplyDeliveryMode` into the reply runtime with regression coverage for both paths.
