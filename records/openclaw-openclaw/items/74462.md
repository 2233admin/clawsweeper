## Summary

Keep open. Current `main` still has a strict durable config schema and `SessionConfig` type with no `contextSafety` field, while the normal config validator routes through that schema. I also found no `contextSafety` runtime symbol or the named rollover/handoff commands in the current source/docs/tests, so this should not be closed as already implemented.

## What This Changes

Add durable config schema, TypeScript types, metadata/docs, and any necessary runtime alignment for context-safety rollover settings expected under `session.contextSafety` and possibly a legacy top-level alias.

## Best Possible Solution

Decide the intended product contract first. If context safety is supported, add a real `ContextSafetyConfigSchema`, `SessionConfig` type, schema help/labels/docs metadata, validation tests, and runtime tests that prove the durable config is consumed; then decide whether top-level `contextSafety` should migrate as a legacy alias. If it is not intended, remove or document any generated/runtime-only alias so users are pointed at the existing compaction and session reset controls.
