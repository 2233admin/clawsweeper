## Summary

Keep open. Current main still rejects the requested durable config paths, and the reported runtime surface is not present in the current source tree, docs, tests, or checkout artifacts. This needs maintainer product/runtime confirmation before accepting schema keys, because a schema-only change could advertise unsupported behavior.

## What This Changes

The issue requests a durable `contextSafety` config contract, preferably under `session.contextSafety`, with matching schema, TypeScript types, metadata, docs, and a decision on a top-level legacy alias.

## Best Possible Solution

Decide the intended product contract first. If context safety is supported, add an end-to-end implementation: `ContextSafetyConfigSchema`, `session.contextSafety?: ContextSafetyConfig`, public config types, schema help/labels/generated metadata, docs, validation tests, and runtime tests proving the durable config is consumed; then explicitly decide whether top-level `contextSafety` is a legacy alias. If it is not intended, document the supported compaction/session reset controls and clean up any generated-package artifact that suggests otherwise.
