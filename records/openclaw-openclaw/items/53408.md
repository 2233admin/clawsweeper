## Summary

Keep this issue open. Current main still shows targeted malformed tool-argument repair for Kimi Anthropic and OpenAI-compatible completions, but the reported custom/local `copilot-api` `anthropic-messages` path is not covered by that gate or by focused regression tests for long fragmented `write`/`exec` arguments.

## What This Changes

The issue reports long-session `write` and `exec` tool calls arriving with missing `content` or `command` arguments, especially through a custom Anthropic-compatible `copilot-api` proxy with large CJK payloads.

## Best Possible Solution

Keep this open until a maintainer can capture or replay a custom/local `anthropic-messages` stream with large fragmented `input_json_delta` tool arguments, then add a provider-safe regression fix or explicit parse-failure diagnostic before empty arguments reach tool validation.
