## Summary

Keep open. Current main still matches the report: the stable system prompt is timezone-only, but exact `Current time:` text is still generated and inserted into heartbeat, isolated cron, post-compaction refresh, and session-reset prompts, with tests asserting that behavior. This remains a valid prompt-cache optimization/design issue rather than an implemented, duplicate, or stale cleanup candidate.

## What This Changes

The issue asks OpenClaw to keep exact wall-clock `Current time:` text out of cache-sensitive prompt prefixes for heartbeat, cron, post-compaction, and reset flows while preserving stable timezone context.

## Best Possible Solution

Make a provider-aware prompt composition change that keeps stable timezone metadata in the system prompt, preserves exact wall-clock time only where a turn truly needs it, and prevents volatile timestamps from landing in provider cache checkpoints or repeated stable prefixes. Regression coverage should include heartbeat, isolated cron, post-compaction refresh, session reset, and Anthropic/OpenAI-family payload shaping.

## Reproduction Assessment

Yes. A read-only source check on current main follows the reported paths and finds `Current time:` injected into heartbeat, isolated cron, post-compaction refresh, and reset prompts, with existing tests asserting those lines.

## Solution Assessment

Unclear for a mechanical fix. The requested direction is likely right, but the best implementation needs prompt/cache owner judgment because exact current time currently supports reset/startup grounding and recurring automation semantics.
