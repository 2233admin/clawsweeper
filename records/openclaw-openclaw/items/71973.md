## Summary

Keep open. Current main has the broader stable-system-prompt cache-boundary work, but the narrower wall-clock optimization remains unresolved: `Current time:` is still generated and injected into heartbeat, isolated cron, post-compaction, and session-reset prompt content. The prior ClawSweeper comment is consistent with the current source check and correctly leaves this as prompt/cache-boundary design work.

## What This Changes

The issue requests moving exact per-turn wall-clock `Current time:` injection out of cache-sensitive prompt prefixes for heartbeat, cron, and post-compaction automation turns while keeping stable timezone metadata available.

## Best Possible Solution

Make a provider-aware prompt composition change that keeps stable timezone metadata in the system prompt, includes exact wall-clock time only for turns that require it, and ensures volatile timestamps are not placed inside cache-tagged stable prefixes or checkpointed blocks. Add focused regression coverage for heartbeat, isolated cron, post-compaction refresh, session reset, and Anthropic/OpenAI payload cache-boundary behavior.

## Reproduction Assessment

Yes. A read-only source check on current main follows the reported paths and finds `Current time:` injected into heartbeat, isolated cron, post-compaction, and reset prompt content, with tests that still assert those lines.

## Solution Assessment

Unclear for the exact implementation. The requested direction matches the prompt-cache docs, but the safest fix needs prompt/cache owner design so exact time remains available only where semantically required and provider cache-control boundaries stay correct.
