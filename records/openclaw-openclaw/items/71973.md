## Summary

Keep this issue open. Current main has the broader timezone-only system prompt and system-prompt cache-boundary work, but the narrower report still matches source: exact `Current time:` text is generated and injected into heartbeat, isolated cron, post-compaction context, and session-reset prompts, with tests asserting that behavior. Anthropic-family payload shaping can still attach `cache_control` to the trailing user message, so the volatile-line placement remains an actionable prompt-cache design question rather than resolved cleanup.

## Best Possible Solution

Keep this open for maintainer design review. The best path is a provider-aware prompt/payload composition change that preserves timezone-only stable system prompt metadata, keeps exact wall-clock time available when a turn truly needs it, and prevents volatile timestamp text from landing in cache-tagged stable blocks or cache checkpoints. The follow-up should add regression coverage for heartbeat, isolated cron, post-compaction, session reset, and Anthropic-family cache-control payload shaping.
