## Summary

Keep open. Current main still has the reported inbound-dispatch hot path, and #74118 is an open, unmerged PR explicitly paired as the fix, so the issue should remain open until that PR is reviewed and either merged or replaced.

## What This Changes

Make `ensureRuntimePluginsLoaded` skip plugin registry resolution when a valid active plugin registry is already present, avoiding per-message plugin reloads and re-registration.

## Best Possible Solution

Keep this issue open while #74118 is reviewed. The final fix should either land the narrow active-registry short-circuit with focused regression coverage or replace it with an equally covered loader-level compatibility change, then close this issue after the implementation reaches `main`.

## Reproduction Assessment

Yes. The current-main code path is enough to reproduce the cache-miss shape statically: inbound dispatch calls `ensureRuntimePluginsLoaded`, that helper always resolves with explicit load options, and the resolver reloads when those options do not match the gateway boot cache key.

## Solution Assessment

Unclear until #74118 is reviewed. The proposed fast path is narrow and matches the helper's ensure-style contract, but maintainers should confirm it does not mask legitimate plugin reconfiguration before choosing it over a broader loader cache-compatibility fix.
