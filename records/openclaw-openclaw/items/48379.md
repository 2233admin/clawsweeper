## Summary

Keep this issue open. Current main still sends static OpenRouter app-attribution headers, the OpenRouter LLM wrapper does not receive agent identity for title construction, and the per-agent config schema has no OpenRouter app-title/header override. Related #50206 is adjacent generic agent-header work, but it is open and does not satisfy the OpenRouter dashboard title request.

## Best Possible Solution

Keep this issue open and implement OpenRouter-specific per-agent app attribution in the LLM request-header path. The conservative direction is to preserve the current `OpenClaw` default, thread resolved `agentId` or sanitized agent display identity into the OpenRouter wrapper/provider context, add a documented opt-in app-title override or a clearly specified automatic `OpenClaw/<agentId>` format, and cover it with provider-attribution/wrapper tests plus OpenRouter provider docs. Coordinate with #50206 so generic `X-OpenClaw-Agent` support does not get mistaken for the OpenRouter dashboard title fix.
