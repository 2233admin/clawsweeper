## Summary

Keep open. Current main still canonicalizes custom-scheme browser origins through `URL.origin`, which becomes the opaque string `null`, so an explicit `gateway.controlUi.allowedOrigins` entry such as `tauri://localhost` cannot match. The provided GitHub context also shows open PR #73511 uses closing syntax for this issue, so the issue should stay open until that fix or an equivalent one lands.

## What This Changes

Allow exact `gateway.controlUi.allowedOrigins` matches for hosted custom URL scheme origins such as `tauri://localhost` and `electron://app` without falling back to wildcard origin allowlisting.

## Best Possible Solution

Land #73511 or an equivalent narrow gateway fix that reconstructs `scheme://host` only for hosted custom schemes with opaque `URL.origin`, keeps literal `Origin: null` invalid, and adds accepted and rejected `tauri://` / `electron://` regression tests in `src/gateway/origin-check.test.ts`.
