## Summary

Keep open. Current main still treats Nextcloud Talk webhook `object.content` as raw text and gates group-room mentions with regex matching against that raw string, so rich-object placeholders like `{mention-user1}` are not resolved into explicit bot mention evidence.

## What This Changes

The issue requests fixing Nextcloud Talk mention gating so rich-object mention placeholders are resolved against the bot identity instead of being treated as plain message text.

## Best Possible Solution

Implement the fix in `extensions/nextcloud-talk`: parse rich-object `object.content`, preserve mention parameters, resolve bot placeholder mentions against a documented bot identity setting or existing account identity when valid, and feed explicit mention evidence into the mention gate with regression tests for bot and non-bot placeholders.
