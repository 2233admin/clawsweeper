## Summary

Current main still has the classifier bug this PR targets: dotted gateway read actions are compared against a flat read-only verb set, so `config.schema.lookup`, `config.get`, and `agents.files.list` still classify as mutating. The linked user-visible warning path is still reachable, and the related payload-level PR does not supersede this root classifier fix.

## Best Possible Solution

Keep this PR open for maintainer review or landing, or replace it with an equivalent fix in `src/agents/tool-mutation.ts`. The best implementation is a gateway-specific classifier that treats known read-scope gateway methods such as `config.schema.lookup`, `config.get`, `agents.files.list`, and `agents.files.get` as non-mutating, keeps writes such as `config.patch`/`config.apply` mutating, and preserves fail-closed handling for unknown cron/canvas actions. After that lands, the linked issue #42856 can close as implemented; the related PR #42962 should only proceed if maintainers still want an additional payload-specific warning policy.
