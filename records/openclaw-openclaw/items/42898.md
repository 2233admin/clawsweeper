## Summary

Keep this PR open. Current main still classifies dotted read-only gateway actions as mutating, while the gateway scope table and gateway tool both treat `config.schema.lookup` as a live read operation. The related payload-specific PR #42962 does not supersede this classifier fix because its suppression path depends on `mutatingAction === false`. The proposed diff is narrow and does not show a security or supply-chain-sensitive change.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent root fix in `src/agents/tool-mutation.ts`. The best implementation should classify known read-only gateway actions such as `config.get`, `config.schema.lookup`, `agents.files.list`, and `agents.files.get` as non-mutating, keep write actions such as `config.patch`, `config.apply`, and `update.run` mutating, preserve fail-closed handling for unsupported cron/canvas actions, and add the regression tests from this PR. After that ships, #42856 can close as implemented; #42962 should be reviewed as an optional narrower payload-policy follow-up.
