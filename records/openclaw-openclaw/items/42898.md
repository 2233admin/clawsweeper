## Summary

Keep this PR open. Current main still compares `gateway` actions against a flat read-only verb set, so dotted read-only gateway actions like `config.schema.lookup` still classify as mutating. The related payload-specific PR #42962 does not supersede this root classifier fix because it depends on `mutatingAction === false`. The PR diff is small and does not introduce an apparent security or supply-chain-sensitive change.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent root fix in `src/agents/tool-mutation.ts`. The best implementation should classify known read-only gateway methods such as `config.get`, `config.schema.lookup`, `agents.files.list`, and `agents.files.get` as non-mutating, keep writes such as `config.patch` and `config.apply` mutating, preserve fail-closed handling for cron/canvas unsupported actions, and add the regression tests from this PR. After that lands, #42856 can close as implemented; #42962 should be considered only as an optional additional payload-policy change.
