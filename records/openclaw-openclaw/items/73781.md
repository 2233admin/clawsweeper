## Summary

Keep open. Current main still leaves pi-coding-agent auto-retry enabled in the prepared embedded Pi settings path, the default tool-loop detector remains opt-in, and two open PRs already use closing syntax for this issue; the issue should stay open until a canonical fix lands or maintainers close those implementation candidates.

## What This Changes

The issue reports that after failed tool calls, the embedded runtime replays the prior agent tool call and can enter an infinite loop, especially with Qwen/Feishu workflows.

## Best Possible Solution

Land one canonical, session-scoped fix that disables the inner pi-coding-agent retry for OpenClaw embedded runs without persisting user settings, with a regression test for failed tool output and follow-up behavior. Treat broader repeated-thinking guardrails as a separate reviewed layer unless maintainers decide they are necessary for this same root cause.

## Reproduction Assessment

Unclear. The issue gives a concrete Qwen/Feishu failed-exec replay pattern, and source plus dependency behavior supports a plausible root cause, but this pass did not run the live provider/channel scenario or find an existing regression test that reproduces it exactly.

Unclear. The narrow session-scoped retry override in #74434 looks like the most direct root-cause fix, while #74503 adds a broader loop-breaking layer that may still need product/runtime review; current main is not the best solution because it leaves the inner SDK retry enabled.
