## Summary

Keep PR #70598 open. Current main and v2026.4.24 already implement the linked exec prompt-body fix for #70552, but the latest PR head also contains unmerged queue-preservation work from review feedback. Current main still consumes the whole inspected event queue after an exec heartbeat, so mixed exec plus cron/non-exec queues can drop events that were not shown in the exec prompt.

## Best Possible Solution

Keep the PR open or ask the author to rebase it against current main and reduce it to the remaining deferred-event preservation fix. The shipped exec prompt-body work can be left as-is, but main still needs a deliberate decision and tests for restoring non-exec inspected events after exec heartbeat runs without duplicate suppression.
