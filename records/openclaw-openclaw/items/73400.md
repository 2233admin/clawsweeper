## Summary

Keep this PR open. Current main still has the direct-before-thread silent-reply classification bug, so the contribution is not obsolete, but the latest branch still uses the generic permissive thread-suffix parser as a trust decision and leaves group/channel thread keys on the group path. The existing Aisle and ClawSweeper comments correctly identify the low-severity session-key trust concern, and the Greptile comment identifies the remaining parent-surface ordering gap.

## What This Changes

The PR adds a thread-suffix check to silent-reply conversation classification, adds direct/cron thread regression tests, and adds a changelog fix entry.

## Best Possible Solution

Land a refined fix that classifies trusted generated internal thread sessions as non-rewriting/internal without letting arbitrary caller-supplied direct session keys opt out of direct silent-reply rewrites, with tests for direct-thread, cron-thread, group/channel-thread configuration, and malformed or user-shaped suffix cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Validate thread provenance before returning internal:** `src/shared/silent-reply-policy.ts:81-83`
  - body: The new branch returns `internal` whenever `parseThreadSessionSuffix()` yields a `threadId`, but that parser accepts any final `:thread:` tail and `sessions.create` can preserve caller-provided key structure. A caller-shaped direct key ending in `:thread:x` can therefore bypass the direct silent-reply rewrite path; gate this on a trusted generated thread shape or use stricter classifier-specific validation.
  - confidence: 0.84
- **[P3] Classify trusted threads before group policy:** `src/shared/silent-reply-policy.ts:72-73`
  - body: The PR says threaded sessions should be internal regardless of parent surface, but the group/channel branch still runs before the thread check. With configured group policies that differ from internal policies, a `...:group:...:thread:...` key can still take group behavior, so the trusted-thread predicate needs to run before group/channel or the narrower behavior needs to be documented and tested.
  - confidence: 0.7
