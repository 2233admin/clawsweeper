## Summary

Keep this PR open. Current main still has the direct/DM silent-reply classification before any `:thread:` handling, so the PR’s intended bug fix is not implemented on main. The PR is focused and plausible, but it still needs maintainer review for the thread-suffix trust/security question raised in review and for the stated “regardless of parent surface” invariant.

## Best Possible Solution

Review and refine this PR rather than closing it. The likely fix belongs in the shared silent-reply classifier: classify trusted internal thread session keys before the direct/DM rewrite branch, add regression tests for direct-thread and cron-thread keys, and explicitly settle parser/trust semantics plus group/channel thread ordering before merge.
