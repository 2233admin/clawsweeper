## Summary

Keep open. Current main still has no Discord thread-reply mention gate: bot-owned autoThread threads bypass mention requirements, and Discord config/schema/docs expose no `requireMentionInThreadReplies`, `threadRequireMention`, or Discord equivalent of Slack's `thread.requireExplicitMention`. Related #40895 overlaps but is broader and still open, so it is not a clean canonical replacement for this focused per-channel policy request.

## Best Possible Solution

Keep this focused issue open and implement an opt-in Discord thread-reply mention policy. A conservative fix would add a per-channel Discord config key for autoThread follow-up replies, validate and document it, then update Discord preflight/allow-list logic so top-level autoThread starter messages can still be handled without mention while unmentioned follow-up replies in bot-owned autoThreads are dropped before model invocation when the new policy is enabled.

## Clownfish Work Prompt

Implement the Discord autoThread follow-up mention gate requested in https://github.com/openclaw/openclaw/issues/52568. Add an opt-in Discord config key scoped to guild channel config for bot-owned autoThread follow-up replies, preferably matching the issue intent (`requireMentionInThreadReplies`) while keeping current defaults unchanged. When enabled with `autoThread: true` and `requireMention: false`, top-level starter messages should still be accepted and spawn threads, but unmentioned follow-up replies inside the bot-owned autoThread should be skipped in Discord preflight before model invocation. Explicit bot mentions should still pass. Update `DiscordGuildChannelConfig`, the strict Zod schema, config metadata/UI hints as required by the repo's config-generation flow, and `docs/channels/discord.md`; add a changelog entry because this is user-facing. Update `extensions/discord/src/monitor/allow-list.ts` and `message-handler.preflight.ts` only as needed, preserving existing behavior for default configs, user-owned threads, unknown-owner threads, bound-thread/ACP routes, Slack, and non-autoThread Discord channels. Add focused regression tests for the resolver and preflight skip/allow behavior, plus config-schema acceptance. Coordinate with related #40895 in the PR body and state whether the PR fully or partially covers that broader report.
