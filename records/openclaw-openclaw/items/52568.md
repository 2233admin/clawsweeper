## Summary

Keep open. Current main still lacks the requested Discord middle-ground policy: bot-owned autoThread replies bypass mention requirements, and Discord config/schema/docs have no `requireMentionInThreadReplies`, `threadRequireMention`, or Slack-style `thread.requireExplicitMention` equivalent. Related #40895 is relevant and still open, but it is broader rather than a clear canonical duplicate for this focused config request.

## Best Possible Solution

Keep this focused issue open. A conservative implementation would add a Discord thread-reply mention policy under the guild/channel config or `channels.discord.thread`, wire it through `src/config/types.discord.ts`, `src/config/zod-schema.providers-core.ts`, generated config metadata/UI hints, and `docs/channels/discord.md`, then update `extensions/discord/src/monitor/allow-list.ts` and preflight tests so bot-owned `autoThread` starter messages can be handled freely while unmentioned follow-up replies are skipped before model invocation when the new policy is enabled.
