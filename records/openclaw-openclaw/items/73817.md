## Summary

Keep open. The PR addresses a real gap in current main’s audio media request config, but the proposed local OpenAI-compatible audio auth path can still rotate from the synthetic local marker into real provider keys on retry, which is a merge-blocking credential-handling issue for this SSRF/auth-sensitive change. The test mock divergence is a smaller follow-up.

## What This Changes

The PR adds audio-only `tools.media.audio.request.allowPrivateNetwork` and per-model request support, forwards audio request overrides through media provider execution, adjusts OpenAI-compatible audio local auth, updates schema metadata/docs/changelog, and adds regression tests.

## Best Possible Solution

Keep the audio-only schema and request-threading approach, but make synthetic local OpenAI-compatible audio auth terminal: once the local marker is selected, retries must not include real provider/env keys, explicit `request.auth` must still win, and image/video media request rejection must remain unchanged.

## Reproduction Assessment

Yes. Current main can be reproduced with a schema parse using `tools.media.audio.request.allowPrivateNetwork` and by inspecting the audio runner merge path; the PR regression can be reproduced with a unit test where a local OpenAI audio attempt throws a retryable 429/rate-limit error while `OPENAI_API_KEY` is present.

## Solution Assessment

No, not as-is. The schema and request-threading direction is the narrow maintainable fix, but the synthetic local auth path must avoid normal API-key rotation before this is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep local audio retries off real provider keys:** `src/media-understanding/runner.entries.ts:477-480`
  - body: When `resolveSyntheticLocalAudioExecutionKey` returns the local marker, this still calls `collectProviderApiKeysForExecution`, which appends real OpenAI keys from the environment. Because audio execution uses normal API-key rotation, a local/private endpoint that returns a retryable 429 can receive the upstream OpenAI key on the next attempt; return only the marker or otherwise disable provider-key rotation for this path.
  - confidence: 0.91
- **[P3] Align the local URL mock with production:** `src/media-understanding/runner.test-mocks.ts:6-16`
  - body: The new mock only treats loopback literals as local, while production also accepts RFC1918 addresses and `.local` names. Future LAN or `.local` audio tests would exercise the credential path in tests but the synthetic local path in production, so delegate to the real helper or mirror the full classifier.
  - confidence: 0.84

## Security Review
