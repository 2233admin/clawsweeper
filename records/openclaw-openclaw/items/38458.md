## Summary

Keep open. Current main still has only partial/global suppression behavior: `messages.suppressToolErrors` is documented and wired, but mutating tool failures such as `message`, `write`, and related send/edit actions are still intentionally surfaced to chat as `isError` warning payloads unless a run-level `suppressToolErrorWarnings` override is used or the assistant explicitly acknowledges the failed action. The requested per-channel or broader tool-output visibility policy is still missing, and related PR #51091 plus related issues #39406 and #73268 remain open rather than shipped behavior.

## Best Possible Solution

Keep this issue open for a core runtime/config design. Add a documented tool-output visibility policy with a global default and per-channel overrides, apply it before chat-surface delivery while keeping tool failures available to the agent, and add focused tests for mutating `message` and `exec` failures across streaming chat surfaces. Coordinate with #39406, #51091, and #73268 rather than treating any open item as shipped behavior.
