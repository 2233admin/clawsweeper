## Summary

Keep open. Current main still implements ordinary Discord message sends as a scalar `mediaUrl`/`filename` upload path: the action parser accepts one media source, the runtime forwards one `mediaUrl`, and the REST upload helper builds a `files` array with exactly one file. Component `media-gallery` support is a separate Discord components surface and does not provide native multi-file uploads for ordinary message sends or shared outbound replies. The existing issue discussion also already kept this open for that remaining gap.

## Best Possible Solution

Implement native multi-file sending in the bundled Discord plugin: accept an ordered multi-attachment input while preserving existing single `media`/`path`/`filePath`/`filename` behavior, load each file through the existing outbound media helpers and access checks, batch eligible uploads into one Discord `files` array up to the verified Discord limit, split overflow into additional messages, and update focused Discord tests, docs, and changelog.
