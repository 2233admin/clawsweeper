## Summary

Keep open. Current main still exposes ordinary Discord uploads as a single `mediaUrl`/`filename` path and shared outbound `mediaUrls` are still delivered as repeated one-file sends, so the requested native multi-file Discord message behavior remains unimplemented.

## What This Changes

Request native Discord multi-file message uploads, up to Discord's per-message attachment limit, instead of limiting ordinary sends to one file per message.

## Best Possible Solution

Implement native multi-file sending in the bundled Discord plugin: accept ordered multi-attachment input while preserving existing single `media`/`path`/`filePath`/`filename` behavior, load each file through existing outbound media helpers and access checks, send up to the verified Discord per-message attachment limit in one `files` array, split overflow into follow-up messages, and update focused Discord tests, docs, and changelog.
