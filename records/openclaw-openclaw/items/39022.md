## Summary

Current main still lacks per-model turn budgets for `/model`; the existing implementation parses and persists session model overrides without any turn countdown, warning, auto-revert notification, or chat-model `models.limits`/`maxTurns` config contract.

## What This Changes

Request to add `/model <name> --turns N` with warning near expiry, automatic reversion to the default model after N turns, and configurable per-agent model limits.

## Best Possible Solution

Keep this open for maintainer design. A good implementation would live in core model directive parsing, session state, model override persistence, and post-turn/session update logic: parse `/model <provider/model> --turns N`, validate any documented per-agent/per-model limit config, store a temporary override budget, decrement after the chosen completed-turn semantics, warn near expiry, clear the override through the existing helper when it expires, notify the user, and cover docs/tests.
