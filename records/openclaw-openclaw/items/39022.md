## Summary

Current main still supports persistent user model overrides, but it does not implement per-model turn budgets, warning-before-expiry, automatic revert to the session default, or a validated `models.limits` / `maxTurns` config contract. The prior ClawSweeper comment already identified the same missing core surfaces, and the newer main snapshot still matches that assessment.

## Best Possible Solution

Keep this open for maintainer design. A complete solution likely belongs in core model-selection and session state: parse `/model <provider/model> --turns N`, persist a temporary override budget, decrement it after completed user turns, warn near expiry, clear the override through the existing helper when the budget expires, send a revert notification, and add a validated config contract once maintainers settle the shape.
