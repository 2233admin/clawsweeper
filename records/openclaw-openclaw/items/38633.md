## Summary

Current main still has the reported docs/runtime mismatch: the public sandbox config example shows `pidsLimit: 256`, but runtime resolution, schema, and Docker argument emission leave `pidsLimit` unset unless the user configures it. The same state is present in the latest release tag v2026.4.26, while the related runtime-default and docs-clarification PRs are closed unmerged and the broader security issue remains open.

## Best Possible Solution

Keep this issue open until maintainers either clarify the docs that `pidsLimit: 256` is an explicit example value and remains unset unless configured, or intentionally add a runtime default in `resolveSandboxDockerConfig` with matching schema/type wording, Docker argument tests, generated docs, and public docs. Coordinate any runtime-default decision with #38604; a docs-only fix can stay narrow and could reuse the closed unmerged #38914 direction.
