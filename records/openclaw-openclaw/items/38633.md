## Summary

Current main still has the reported mismatch: the public agent sandbox config example shows `pidsLimit: 256`, while runtime resolution leaves `pidsLimit` unset unless configured and Docker args only emit `--pids-limit` for a positive value. The latest release tag still has the same state. A related runtime-default PR, #44098, is open for #38604, but it is unmerged and currently does not update the docs example, so this issue should stay open for maintainer coordination.

## Best Possible Solution

Keep this issue open until maintainers choose and land one coordinated path: either clarify the docs that `pidsLimit: 256` is an explicit example and remains unset unless configured, or land a runtime default with matching docs, schema/type wording, Docker arg tests, and release notes as appropriate. If #44098 is the chosen runtime-default path, it should be reviewed alongside #38604 and adjusted so the public docs no longer imply the wrong default.
