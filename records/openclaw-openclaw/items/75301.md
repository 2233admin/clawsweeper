## Summary

Keep open. Current main has partial cleanup paths for bundled runtime deps and session artifacts, but it still lacks the requested cross-cache `openclaw caches` inventory/prune surface, and the existing runtime-deps prune path intentionally does not remove stale versioned roots.

## What This Changes

Request an `openclaw caches` or doctor-integrated command that lists and safely prunes cache-class state under `~/.openclaw`, including stale plugin-runtime-deps roots, orphan transcripts, browser/tools downloads, and other regenerable caches.

## Best Possible Solution

Design a core-owned cache inventory and dry-run-first prune surface with explicit cache-vs-data classes, a deletion allowlist, current-version and lock protection for runtime deps, fresh/live transcript safeguards, and opt-in handling for expensive or user-adjacent caches such as memory, browser, and tools.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection provide high-confidence verification that current main lacks the requested cross-cache command while only partial cleanup surfaces exist.

## Solution Assessment

Unclear whether the final UI should be a top-level command or doctor integration. The safest direction is the requested class-based dry-run inventory plus narrow pruning rules; broad automatic deletion should wait for explicit maintainer guardrails.
