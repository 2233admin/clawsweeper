## Summary

Keep open. Current main still does not implement the active-run Ctrl+C hint: TUI Ctrl+C still resolves only to clear/warn/exit messaging and does not consult `activeChatRunId`. The paired same-author bug remains open, and broader related PR #44223 is open/unmerged rather than a shipped replacement. The provided diff is a narrow text-only TUI change with no apparent security-sensitive surface.

## Best Possible Solution

Keep this PR open for maintainer review. Either land this narrow TUI text clarification if Ctrl+C should remain clear/exit, or explicitly supersede it after a broader Ctrl+C behavior PR such as #44223 lands or is selected. A clean landing path would compute the active-run hint inside `handleCtrlC`, append it only to clear/warn statuses, preserve exit and `/abort` behavior, and add focused regression coverage or manual proof for the active-run timing edge case.
