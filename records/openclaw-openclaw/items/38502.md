## Summary

Keep open. Current main still uses Ctrl+C only for clear/warn/exit status text with no active-run hint, and this PR is a focused implementation for the linked TUI confusion; however the diff likely misses the immediate post-submit window because it only checks `activeChatRunId`, and #44223 remains an overlapping product choice for making Ctrl+C abort active runs.

## What This Changes

This PR appends a `/abort` active-run hint to the TUI Ctrl+C clear and warning status messages in `src/tui/tui.ts` without changing keybindings or abort behavior.

## Best Possible Solution

Settle one TUI interaction policy and cover both pending-submit and active-run states: either update this narrow status-text PR so the hint appears during the full window where a submitted run can still complete, or supersede it with #44223's abort-first behavior, with focused TUI tests and docs aligned to the chosen policy.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to start a TUI run, press Ctrl+C while the run is pending or active, and compare the visible clear/exit status against whether the backend run still completes.

## Solution Assessment

Unclear. The text-only approach is the narrowest solution if Ctrl+C should remain clear/exit, but this specific condition is too narrow for the reported quick Enter/Ctrl+C path; #44223 may be better if maintainers want Ctrl+C to abort runs directly.
