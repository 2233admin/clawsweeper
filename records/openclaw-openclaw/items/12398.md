## Summary

Keep this issue open. Current main still does not have a first-class cron postcondition contract, deterministic post-run checker, durable postcondition state, retry prompt injection, or plugin hook/API that can turn a clean isolated cron run into scheduler-owned failure/retry/escalation. The related local-command cron request is complementary, not a replacement for post-run verification of agentTurn jobs.

## Best Possible Solution

Keep this open for maintainer/product design. The best path is an additive cron postcondition contract with protocol/schema/CLI/UI/docs support, deterministic bounded verifier execution after isolated agent completion and before applyJobResult(), durable verifier result fields, run-log projection, retry prompt injection, and integration with existing retry/failure-alert delivery. If maintainers want this plugin-owned, first add a supported cron post-run hook/API that can fail the run and request scheduler-owned retry/escalation.
