## Summary

Keep this issue open. Current main at 20e21173715ef488850023a90f72585a01cc379f still routes Discord messages before making a cross-bot mention/reply-target decision, only compares mention/reply metadata to the current handler bot id, and has no default-owner suppression result or acceptance tests for the requested precedence.

## What This Changes

Request deterministic Discord single-target routing where explicit bot mentions win over reply targets, reply targets win over the per-channel default owner, and the default owner is suppressed whenever a targeted bot applies.

## Best Possible Solution

Keep this issue open until the Discord plugin makes one deterministic target decision before default routing: map configured Discord bot user IDs to OpenClaw accounts/agents, select explicit bot mentions first, select the referenced-message author second, and use the per-channel default owner only for untagged non-reply messages. Review #47393 and #67460 as ingredients, then add focused Discord preflight/routing tests for precedence and suppression.
