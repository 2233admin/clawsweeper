## Summary

Keep this issue open. Current main still resolves the Discord conversation route before making any cross-bot target decision, carries only the running handler's bot user ID into preflight, treats reply metadata as an implicit mention only when it matches that same handler ID, and lacks a suppression result that would prevent the default owner from responding when another configured bot is targeted.

## What This Changes

Request deterministic Discord multi-bot routing where explicit bot mentions win, reply-to-bot metadata is second, and the per-channel default owner handles only untagged non-reply messages.

## Best Possible Solution

Implement the precedence in the Discord plugin before shared default routing: build a configured Discord bot user ID to OpenClaw account/agent map, select explicit mentioned bot accounts first, select the referenced-message bot author second, and use the per-channel default owner only when no targeted signal exists. Add focused preflight/routing tests proving the default owner is suppressed in targeted cases.
