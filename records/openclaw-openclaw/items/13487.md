## Summary

Keep open. Current main still lacks cross-bot Discord target selection and default-owner suppression: Discord preflight routes before deriving explicit mention or reply-target state, only knows the running handler's bot user ID, and shared mention gating has no suppression result for messages targeting another configured bot. Related open PRs are useful source material, but no merged implementation covers the requested precedence yet.

## What This Changes

Request deterministic Discord multi-bot routing where explicit bot mentions win over Discord reply targets, reply targets win over the per-channel default owner, and targeted messages suppress the default owner.

## Best Possible Solution

Implement the precedence inside the Discord plugin before shared default routing: build a configured Discord bot user ID to OpenClaw account/agent map, select explicit mentioned bot accounts first, select the referenced-message bot author second, and use the per-channel default owner only when no targeted signal exists. Add focused Discord preflight, shared mention-gating, and route resolver tests proving default-owner suppression in targeted cases.
