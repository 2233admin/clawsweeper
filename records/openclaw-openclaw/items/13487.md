## Summary

Keep this issue open. Current main still does not implement Discord-wide single-target precedence of explicit bot mention, then reply-target bot, then default owner with default-owner suppression. Discord preflight resolves the route before mention/reply metadata, only knows the current monitor's optional bot user id, and the shared mention-gating/default routing paths still lack an other-agent/default-owner suppression decision.

## Best Possible Solution

Keep this issue open until the Discord plugin makes one deterministic target decision before default routing: map configured Discord bot user IDs to OpenClaw accounts/agents, select explicit bot mentions first, select the referenced-message author second, and use the per-channel default owner only for untagged non-reply messages. Add focused Discord preflight tests for explicit mention, reply-target, untagged default fallback, and default-owner suppression. Review #47393 and #67460 as possible ingredients, but neither is sufficient alone on current main.
