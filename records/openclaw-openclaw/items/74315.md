## Summary

Keep open. The PR addresses a real provider-resolution bug, but the current branch is still incomplete: it would break the bundled Kimi plugin's legacy explicit-provider override path and leaves the shared provider-alias policy contract inconsistent with the new `kimi-coding` identity. Current `main` has not implemented the intended behavior.

## What This Changes

The PR removes `kimi-coding` from `normalizeProviderId`'s Kimi alias branch and updates three model-selection test expectations to preserve `kimi-coding` as the provider.

## Best Possible Solution

Land a revised patch that keeps `kimi-coding` distinct for model catalog and selected model refs, preserves the bundled Kimi plugin's legacy `models.providers.kimi-coding` override behavior through a plugin-local compatibility lookup, and updates the shared alias-policy fixture/tests so `tools.byProvider` semantics are explicit.
