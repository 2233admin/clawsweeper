## Summary

Keep open: current main still lacks a provider-level media trust policy for model fallback, and the remaining work is already represented by open PR #57959 with closing syntax. This should stay open until maintainers review, revise, land, or replace that PR.

## What This Changes

Add provider-level fallback trust controls so cross-provider retries can distinguish text fallback from forwarding user images or attachments.

## Best Possible Solution

Review PR #57959 against current main and either land a revised version or replace it with a centralized fallback-boundary implementation that has primary and target provider context, covers explicit and prompt-detected media consistently, exposes documented user-facing trust configuration, and includes same-provider and cross-provider tests.
