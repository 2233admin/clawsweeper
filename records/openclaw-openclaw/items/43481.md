## Summary

Keep this issue open. Current main still does not provide provider-level fallback trust configuration for images/attachments, and the issue is paired with open same-author PR #57959 using closing syntax, so the right maintainer path is to review/land or close that PR before closing this issue.

## Best Possible Solution

Keep this issue open while maintainers review, revise, land, or close PR #57959. The likely product/code path is a focused fallback-boundary change that threads primary and target provider context through runWithModelFallback and runAgentAttempt, centralizes image/attachment trust decisions, suppresses prompt-detected image loading when the fallback target is not trusted, and documents any user-facing trust configuration if maintainers choose configurable behavior.
