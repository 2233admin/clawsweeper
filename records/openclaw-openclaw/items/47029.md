## Summary

Keep this PR open. Current main already covers the gateway SecretRef probe path and Feishu config-facing unresolved SecretRef tolerance, but it still does not implement the PR's unauthenticated loopback identity behavior; current source and tests intentionally keep device identity disabled for loopback probes without shared auth.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a rebase and narrowing before merge. Drop or adapt the already-covered gateway SecretRef and Feishu pieces, then explicitly decide whether unauthenticated loopback status probes should reuse cached paired operator identity. If maintainers want that behavior, implement it with read-only cached identity plus cached operator-token checks and tests for paired, unpaired, read-only-state, shared-auth, and remote-probe cases; otherwise close the PR manually as not accepted rather than as implemented.
