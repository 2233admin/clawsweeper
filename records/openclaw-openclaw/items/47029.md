## Summary

Keep this PR open. Current main has already covered the gateway SecretRef probe path and Feishu config-facing SecretRef tolerance through different code, but it has not implemented the PR's loopback identity behavior. Main currently tests and documents the opposite policy: unauthenticated loopback probes keep device identity disabled so status remains non-mutating.

## Best Possible Solution

Keep this PR open for maintainer judgment, but require a rebase and narrowing before merge. Drop or adapt the already-covered gateway SecretRef and Feishu changes, then decide explicitly whether unauthenticated loopback status probes should reuse a cached paired operator identity. If maintainers want that behavior, implement it in src/gateway/probe.ts with a read-only cached-identity plus cached-operator-token check, no identity creation, and tests for paired, unpaired, read-only-state, shared-auth, and remote-probe cases. If maintainers prefer the current non-mutating no-auth-loopback policy, close the PR manually as not accepted rather than as implemented.
