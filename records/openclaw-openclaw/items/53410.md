## Summary

Keep this PR open. Current main still treats numeric `-N` suffixes as semver prereleases in the update-check comparison path, while `src/config/version.ts` already models them as OpenClaw correction revisions. The proposed fix is directionally valid, but it needs review/rebase onto the current shared `src/infra/semver-compare.ts` helper and should address the low-risk numeric precision concern before merge.

## Best Possible Solution

Keep this PR open, but request a rebase/adaptation onto current main: update the OpenClaw update-version comparison path to rank numeric correction revisions above stable releases, preserve generic semver behavior for ClawHub/plugin comparisons, add/update regression tests for `2026.3.23-1 > 2026.3.23` and revision-vs-beta ordering, and address numeric suffix comparison without precision loss.
