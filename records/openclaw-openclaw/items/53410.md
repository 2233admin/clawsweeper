## Summary

Keep this PR open. Current main still has the update-path bug the PR targets: `compareSemverStrings()` delegates to the generic semver comparator, which treats numeric `-N` suffixes as prereleases, and the existing update-check test still expects `1.0.0-1` to rank below `1.0.0-beta.1`. The newer OpenClaw-specific version parser has the desired correction-release semantics, but the status/startup/update paths still do not use it.

## Best Possible Solution

Keep this PR open and review it as a real implementation candidate. The best current path is to rebase/adapt the fix to current main by routing OpenClaw package update comparisons through `compareOpenClawVersions()` or a revision-aware update-specific comparator, add the PR's CalVer regression coverage to the update/status/startup surfaces, and avoid changing ClawHub's normal semver range comparator unless maintainers explicitly want that broader behavior.
