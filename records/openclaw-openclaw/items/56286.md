## Summary

This issue should stay open. Current main still resolves Telegram exec approval callbacks by clearing inline buttons, not deleting the approval message, and the issue is paired with an open unmerged contributor PR (#56319) that explicitly tracks this fix.

## Best Possible Solution

Keep this issue open. Review and refresh PR #56319, or implement the same behavior on current main in `extensions/telegram/src/approval-handler.runtime.ts` and `extensions/telegram/src/bot-handlers.runtime.ts`: after approve/deny resolves, delete the Telegram approval message after the intended delay, with tests covering both the direct callback path and native approval finalization.
