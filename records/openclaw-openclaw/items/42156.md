## Summary

Keep open. Current main still acknowledges Telegram callback queries inside the downstream `callback_query` handler, after `sequentialize` is registered. The merged approval-lane change only moves approval callbacks to a separate sequential key; ordinary inline-button callbacks still share the normal chat/topic lane and can still wait behind an active turn.

## Best Possible Solution

Keep this issue open and land a focused Telegram plugin fix: acknowledge callback queries in a middleware registered before `botRuntime.sequentialize(getTelegramSequentialKey)`, avoid duplicate handler-level acknowledgement, keep callback action routing ordered, and add a regression proving ordinary callbacks are acknowledged before same-chat queued work is released.

## Clownfish Work Prompt

Fix this issue on current main. Observable bug: Telegram inline-button callback queries can sit behind `@grammyjs/runner` `sequentialize` for the same chat/topic, so `answerCallbackQuery` runs too late and the Telegram button spinner can time out. Scope the fix to the bundled Telegram plugin. Add a middleware in `extensions/telegram/src/bot-core.ts` that acknowledges callback queries before `botRuntime.sequentialize(getTelegramSequentialKey)` runs, catches/logs acknowledgement failures, and never blocks `next()`. Remove or avoid redundant handler-level acknowledgement in `extensions/telegram/src/bot-handlers.runtime.ts` while preserving callback routing, authorization, approval resolution, model menu, command pagination, and interactive plugin handling. Do not change generic core queue semantics or callback data formats. Add/update focused tests in `extensions/telegram/src/bot.create-telegram-bot.test.ts` and any necessary auth tests so an ordinary non-approval callback is acknowledged before its same-chat action is released from behind a busy sequentialized turn, and so acknowledgement failures still allow downstream processing. Keep docs/changelog minimal; add a changelog fix entry only if this PR is user-facing per maintainer policy, crediting the reporter appropriately.
