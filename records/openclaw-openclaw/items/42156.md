## Summary

Keep this issue open. Current main still answers Telegram callback queries inside the downstream callback handler after `sequentialize` is installed, and the newer approval-lane mitigation only covers parsed exec/plugin approval callbacks. Ordinary inline-button callback data still resolves to the normal chat lane, so it can queue behind an active same-chat/topic turn.

## Best Possible Solution

Implement the Telegram plugin fix on current main: add a pre-sequentialize middleware in `extensions/telegram/src/bot-core.ts` that immediately answers callback queries with caught/logged API failures, remove or avoid redundant handler-level acknowledgement in `extensions/telegram/src/bot-handlers.runtime.ts`, and add a regression test covering an ordinary inline-button callback while the same chat/topic lane is busy. The closed contributor PRs #42125, #42181, and #42516 are useful references but need a fresh maintainer-ready patch.
