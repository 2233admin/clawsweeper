## Summary

Keep this issue open. Current main still answers Telegram callback queries inside the downstream `callback_query` handler after grammY `sequentialize` is installed, so normal inline-button callbacks can still wait behind an active same-chat/topic turn. Main only has a narrower approval-callback lane mitigation, not the general pre-sequentialize acknowledgement described here.

## Best Possible Solution

Implement the Telegram plugin fix on current main: add a pre-sequentialize middleware in `extensions/telegram/src/bot-core.ts` that immediately answers callback queries with caught/logged failures, remove or avoid redundant handler-level acknowledgement in `extensions/telegram/src/bot-handlers.runtime.ts`, and add a regression test proving acknowledgement occurs before a queued same-topic callback action is released while an agent turn is busy. The closed contributor PRs #42125, #42181, and #42516 are useful references, but the final patch should be rebased and validated against current main.

## Risks / Open Questions

- The approval-lane change likely reduces symptoms for exec/plugin approval buttons only; normal native-command or plugin inline buttons still use the chat/topic lane.
- A correct fix should acknowledge callback queries before `sequentialize` while leaving the actual callback action processing ordered through the normal handler path.
- The pre-sequentialize acknowledgement path should catch/log Telegram API failures and intentionally handle updates that later fail authorization or dedupe checks.
