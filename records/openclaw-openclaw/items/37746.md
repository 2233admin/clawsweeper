## Summary

Current main still does not implement the requested security/API contract. Skills cannot declare heartbeat-callable sandboxed functions with parameter schemas, the heartbeat config/schema has no remote structured invocation surface, and the heartbeat runner still resolves text prompts and sends them through the model path. The prior ClawSweeper comment reached the same conclusion; there were no relevant current-main changes to the central files since that review.

## Best Possible Solution

Keep this open for maintainer product/security design. A complete fix should define a versioned metadata/API contract for declared callable functions, document and parse it, add a structured remote heartbeat fetch/parse path, validate call names and params against declared schemas, execute only through existing tool/plugin capability boundaries, and add regression tests proving unknown calls, schema failures, and prompt-injection text are not delivered to the model as free-form heartbeat instructions. ClawHub trust indicators can follow after the core contract exists.
