## Summary

Keep open. Current main still drops early TUI agent events for run IDs that have not yet been registered by a chat event, and the latest PR head is a narrow, plausible fix that now includes the BTW-run and optimistic-message guards raised in earlier review comments. The PR updates the TUI agent-event handler to pre-register same-session agent runs before the known-run filter, mirrors chat-run bookkeeping for active/optimistic runs, and adds optional `sessionKey` to the TUI `AgentEvent` type.
