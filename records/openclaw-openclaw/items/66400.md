## Summary

Keep this PR open. Current main still drops same-session TUI agent events that arrive before any chat delta registers the run, while the current PR diff narrowly adopts same-session early agent events and mirrors the chat-event bookkeeping that earlier bot reviews requested. I found no blocking correctness or security issue in the latest diff, but focused regression coverage is still the main remaining merge concern.

## What This Changes

The PR updates the TUI agent-event handler to pre-register same-session agent runs before the known-run filter and adds an optional `sessionKey` field to the TUI `AgentEvent` type.

## Best Possible Solution

Land this PR or an equivalent TUI fix with focused handler tests that cover same-session tool-first agent events, cross-session rejection, BTW run protection, and optimistic local-run state so live `/verbose full` tool cards render without waiting for history reload.
