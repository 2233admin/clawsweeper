## Summary

Keep this PR open. Current main still lacks Discord `CHANNEL_DELETE`-driven cleanup for persisted Discord channel sessions, and the PR remains a useful implementation candidate, but it cannot land as-is because it targets the removed `src/discord` layout instead of the active bundled Discord plugin under `extensions/discord`. The provided PR discussion is useful: the original report included production evidence of orphaned sessions, Greptile found transcript/archive correctness issues, and the latest PR head appears to address those issues in the stale path.

## Best Possible Solution

Keep the PR open and refresh it against `extensions/discord/src/monitor`: add and register a Carbon channel-delete listener in the active Discord plugin, remove exact deleted-channel session entries through plugin-safe session-store/runtime APIs or a narrow additive SDK seam, preserve transcript archival/cleanup and lifecycle behavior, add focused Discord plugin tests, and run the appropriate changed gate before landing.
