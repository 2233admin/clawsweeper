## Summary

Keep this PR open. Current main still does not implement Discord channel-deletion-driven session cleanup, and the PR remains a focused implementation candidate for the orphaned Discord channel session problem. It needs a refresh because active Discord code now lives under the bundled plugin layout, while the provided PR diff targets the old `src/discord` tree.

## Best Possible Solution

Keep this PR open as the implementation candidate, but refresh it against current main under `extensions/discord/src/monitor`: add and register a Carbon channel-delete listener, delete only the intended Discord channel session entries through plugin-safe session-store APIs, archive and clean up transcripts with store-dir restrictions, add focused Discord plugin tests, and run the changed gate before merge.
