## Summary

Keep open. Current main still has the unguarded key-only cleanup in the followup drain finally block, so the race this PR targets is not implemented on main. The linked open issue #68838 and the Greptile review both support that this is a narrow, plausible auto-reply queue fix, and the supplied diff touches only source and test files with no unexplained security-sensitive changes.

## What This Changes

This PR adds an identity check before deleting a drained followup queue and adds a regression test for the /stop plus immediate followup replacement race.

## Best Possible Solution
