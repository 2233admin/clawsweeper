## Summary

Keep this open. Current main has partial safeguards for timeout and accidental empty direct turns, but the requested always-visible fallback is still not implemented: final dispatch can still complete with no queued final reply for empty/filtered payloads, exact silent replies, and group/channel tool-only behavior, and there is no `dispatch.requireReply` configuration.

## What This Changes

The issue requests that OpenClaw always deliver a user-visible fallback when an agent completes with no replyable payload, including timeout and empty-output cases, optionally guarded by a `dispatch.requireReply` policy.

## Best Possible Solution

Define an explicit opt-in require-reply policy, likely at dispatch or surface scope, that can force a visible fallback where operators want it while preserving documented `NO_REPLY`, group/channel `message_tool` visible-reply mode, pending subagent silence, and media/tool deliveries by default. If maintainers only want the narrower provider/tool-result repair, route that through #46080 instead of changing global dispatch behavior.
