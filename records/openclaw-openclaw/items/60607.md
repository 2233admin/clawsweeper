## Summary

Keep open. Current main has partial empty-turn mitigations for direct chats, but it does not implement the reported always-notify guarantee. The auto-reply runner still returns no final payload when the model result or normalized reply payload list is empty, exact NO_REPLY is explicitly treated as intentional silence, and group/channel docs and tests confirm that some empty or reasoning-only turns intentionally produce no user-visible reply. The related #46080 report is a narrower Anthropic tool-result case and does not supersede this broader Feishu/dispatch policy request.

## Best Possible Solution

Keep this open for a policy-aware runtime fix: add a documented reply-required mode or fallback at the agent-runner/dispatch boundary that can notify users on unintentional empty completions, while preserving documented intentional silence for NO_REPLY, group/channel lurk mode, cron, and already-delivered message-tool outputs. Feishu should get a regression test for the queuedFinal=false, replies=0 path.
