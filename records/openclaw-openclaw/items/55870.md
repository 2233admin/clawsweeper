## Summary

Current main has not implemented the PR’s stated VS Code JSON formatter change: `[json]` still uses `oxc.oxc-vscode`. The PR also remains unsafe to merge as-is because its latest head includes unrelated checked-in runtime config files, while repo docs say OpenClaw config belongs outside the repo under `~/.openclaw/openclaw.json`.

## Best Possible Solution

Keep the PR open only for a clean revision: remove `.openclaw/openclaw.json` and the root `openclaw.json`, then either limit the diff to the JSON formatter change or also update the VS Code extension recommendation and explain why JSON should use Prettier despite the repo’s OXFMT/OXC default. If the contributor does not revise it, maintainers can close it later as a dirty chore PR rather than as implemented-on-main.
