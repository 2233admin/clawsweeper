## Summary

Keep open. Current main still does not provide a proactive fallback-chain test command: `/models` only handles provider/list/add flows, `openclaw models status --probe` is an auth-profile probe, and the runtime fallback resolver still starts from the effective primary before configured fallbacks. The direct implementation PR is closed unmerged, and the broader per-model health-check request does not replace this narrower command.

## What This Changes

The issue requests a `/models test-fallback` command that skips the primary model and proactively probes configured fallback candidates, with an optional all-chain mode.

## Best Possible Solution

Ship a first-class fallback-chain probe exposed through `/models test-fallback` that reuses OpenClaw's real model/auth fallback resolution, intentionally begins at configured fallback candidates, reports the responding provider/model plus per-candidate failures, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.
