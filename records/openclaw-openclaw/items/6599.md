## Summary

Keep this issue open. Current main still has fallback runtime and live auth probing, but it does not implement `/models test-fallback` or an equivalent `openclaw models test-fallback` command that intentionally skips the primary model and exercises the configured fallback chain. The direct implementation attempt, #43342, is closed and unmerged, and the latest issue discussion already records a maintainer/Codex keep-open review.

## Best Possible Solution

Keep this open until maintainers implement or explicitly decline a first-class fallback-chain probe. A strong implementation would add `/models test-fallback` and likely `openclaw models test-fallback`, reuse the real fallback resolution/run path with an intentional primary skip, support first-fallback and all-fallback modes, respect active agent/session auth context, add focused slash/CLI tests, and document how it differs from `models status --probe`.

## Risks / Open Questions

- A real fallback-chain probe would make live model calls, so UX should make token and rate-limit effects explicit.
- The implementation should avoid mutating user-visible session model overrides, auth-profile pinning, transcripts, or cooldown state except for intentional isolated probe artifacts.
- `models status --probe` overlaps conceptually, so docs and command text should distinguish auth reachability from exercising the actual fallback chain.
