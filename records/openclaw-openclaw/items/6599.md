## Summary

Keep open. Current main still does not implement `/models test-fallback` or `openclaw models test-fallback`; the shipped `models status --probe` path is an auth/profile reachability probe, not an intentional primary-skip fallback-chain walk. The directly related implementation PR #43342 is closed and unmerged, while open #63145/#63259 cover a different per-model health-probe surface rather than the requested fallback-chain command.

## What This Changes

Add a first-class `/models test-fallback` command, preferably with an `openclaw models test-fallback` CLI equivalent, to intentionally skip the primary model and proactively verify configured fallback models.

## Best Possible Solution

Keep this open until a maintainer either explicitly declines it or lands a first-class fallback-chain probe. The best implementation is a `/models test-fallback` slash command plus `openclaw models test-fallback` CLI command that reuses the real fallback/auth resolution path, intentionally starts at the first configured fallback, supports first-fallback and all-fallback modes, respects active agent/session auth context, avoids config/session/auth-state mutation, reports which model answered, and documents how it differs from `models status --probe` and any `--probe --all` per-model health check.
