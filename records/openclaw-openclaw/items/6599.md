## Summary

Keep open. Current main still lacks the requested proactive fallback-chain test command: the CLI and slash surfaces expose adjacent model list/status/fallback management, `models status --probe` remains an auth-profile probe, and the runtime fallback chain begins with the requested primary rather than intentionally starting at the first fallback. The direct implementation PR #43342 is closed and unmerged, while #63145 is broader per-model health checking rather than a replacement for this targeted command.

## What This Changes

The issue requests a `/models test-fallback` command that skips the primary model, probes configured fallback candidates, reports which model responded, and optionally tests the whole chain.

## Best Possible Solution

Ship a first-class fallback-chain probe that reuses OpenClaw’s real model/auth fallback resolution, intentionally begins at configured fallback candidates, reports provider/model/status for the responding candidate, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.
