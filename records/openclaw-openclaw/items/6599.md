## Summary

Keep this issue open. Current main still has model fallback runtime and live auth probing, but there is no `/models test-fallback` slash command or `openclaw models test-fallback` CLI equivalent that intentionally skips the primary model and verifies the configured fallback chain. The directly related implementation PR, #43342, is closed and unmerged.

## Best Possible Solution

Keep this open until maintainers add or explicitly decline a first-class fallback-chain probe. A good fix would add `/models test-fallback` and likely `openclaw models test-fallback`, reuse the real fallback resolution/run path with an intentional primary skip, support first-fallback and all-fallback modes, respect active agent/session auth context, avoid session/config mutation, add focused slash/CLI/runtime tests, and document how it differs from `models status --probe`.

## Clownfish Work Prompt

Implement a first-class fallback-chain probe for the open issue requesting `/models test-fallback`.

Expected behavior: add `/models test-fallback` for chat and `openclaw models test-fallback` for CLI. The default mode should intentionally skip the primary model and send a tiny live request to the first configured fallback, then report success/failure and the provider/model that responded. Add an `--all` or `all` mode to test each configured fallback in order. Reuse the real configured fallback resolution/auth/run path as much as possible, but do not persist fallback selection, overwrite session model overrides, mutate config, or leak secrets. Make token/rate-limit effects explicit in user-facing copy and keep max tokens low with tools disabled. Respect active agent/session context and agent-specific fallback overrides.

Implementation boundary: keep core generic, with no provider-specific IDs or recovery policy. Add a small internal helper/option in `src/agents/model-fallback.ts` only if needed to compute or run fallback candidates starting after the primary. Add CLI wiring in `src/cli/models-cli.ts` and a command module under `src/commands/models/`, plus slash parsing/handling in `src/auto-reply/reply/commands-models.ts`. Update `src/commands/models.ts` exports if a new command module is exported. Do not change provider auth storage semantics or unrelated model status/probe behavior.

Docs/changelog: document the command in `docs/cli/models.md`, `docs/concepts/models.md`, and `docs/tools/slash-commands.md` as appropriate. Add a user-facing changelog entry under the active version with `Thanks @jonwiley`.

Related refs: this issue, #43342, #6381, #4217, #3253.
