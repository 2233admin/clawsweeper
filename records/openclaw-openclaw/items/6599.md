## Summary

Keep open. Current main still does not provide the requested fallback-chain test command: the CLI model surface has status/list/set/fallbacks/scan/auth commands but no `test-fallback`, the slash `/models` parser still only handles provider/list/add flows, and the existing `models status --probe` path probes auth/profile reachability rather than intentionally starting at the first configured fallback. The directly related implementation PR #43342 is closed and unmerged, while #63145 is a different per-model health-check request.

## What This Changes

Add a `/models test-fallback` command, and optionally a CLI equivalent, to proactively verify configured model fallback chains by skipping the primary model and probing fallback candidates.

## Best Possible Solution

Ship a first-class fallback-chain probe that reuses the real fallback/auth resolution path, intentionally begins at configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broad per-model health checks.
