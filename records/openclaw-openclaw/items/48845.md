## Summary

Keep this PR open. Current main still lacks the missing-`chat_type` fallback and still uses the old UUID reaction synthetic ID path, while the PR’s latest diff has unresolved code-backed Feishu reaction dedupe and topic-routing concerns that need maintainer review before it can be landed or declared obsolete.

## Best Possible Solution

Keep the PR open and ask for a refreshed patch: infer Feishu chat type from known `chat_id` shapes when `chat_type` is missing, keep synthetic reaction IDs dedupe-safe and event-unique, carry the original message ID separately for Feishu reply/thread anchoring, preserve topic metadata, and add regression tests for missing `chat_type`, p2p/private classification, created/deleted/repeated reactions, and topic-session routing.

## Risks / Open Questions

- Closing this PR would discard a focused Feishu bugfix candidate while current main still drops reaction lookups when Feishu omits `chat_type`.
- Merging the PR as-is may still lose repeated same-user/same-emoji lifecycle events within the 24-hour Feishu dedupe TTL because the latest synthetic ID format is deterministic and lacks an event-unique token such as `action_time`, `reaction_id`, or a UUID.
- The chat-id fallback needs targeted coverage for group, p2p/private, and `oc_dm` shapes so direct-message reactions are not misclassified as group traffic.
- The reaction reply-anchor change should preserve original topic metadata (`root_id`/`thread_id`) or an equivalent explicit field so Feishu topic sessions do not fork history.
