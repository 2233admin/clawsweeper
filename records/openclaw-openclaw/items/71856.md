## Summary

Keep open. Current main still does not implement the PR's intended dynamic TUI startup conversation summary. The underlying Gateway/session API already has derived-title and last-message preview support, but the TUI startup path only refreshes agents, loads history, updates connection state, sends any auto-message, and renders. The latest supplied PR diff remains useful implementation work, but it still needs maintainer review around reconnect duplication, tests, and whether prior-session snippets should be displayed automatically at startup.

## Best Possible Solution

Keep this PR open for maintainer review. If the feature direction is accepted, rebase it onto current main, use the existing `sessions.list` derived-title/preview API in the TUI, add a first-connect-only guard such as `!reconnected`, make an explicit product/privacy call on automatic prior-session snippets, and add focused tests for formatting, success, empty sessions, API failure, local-mode skip, and reconnect behavior. Update the public TUI docs if the startup summary ships.
