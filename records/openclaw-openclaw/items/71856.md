## Summary

Keep this PR open. Current main does not already implement the dynamic TUI startup summary, and the latest PR remains a plausible implementation path, but the diff still has blocking behavior/data-scope issues before merge: it appends the summary on reconnect and fetches recent previews without current-agent scoping.

## What This Changes

The PR adds a TUI startup-summary formatter and a non-local `client.listSessions` call that displays a previous session title or last-message preview after startup history loads.

## Best Possible Solution

Revise this PR before merge: make the startup summary a one-time initial-connect behavior, scope the session lookup to the current agent/session model, add focused TUI coverage for success, empty/error, local-mode skip, reconnect, and agent scoping, and update public TUI docs if the behavior ships.

## Reproduction Assessment

Yes for the PR regressions: trigger `client.onConnected` again after a disconnect to see duplicate summary lines, and mock `sessions.list` with a newer session from another agent to see it selected because the call omits `agentId`. I did not execute those tests in this read-only pass.

## Solution Assessment

No in its current form. The Gateway sessions preview API is a reasonable source, but the patch needs reconnect guarding and current-agent scoping to be the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard startup summaries on reconnect:** `src/tui/tui.ts:1105`
  - body: The new block runs inside `client.onConnected`, which current main already uses for both first connection and reconnects via `reconnected = wasDisconnected`. Because the condition is only `!isLocalMode`, every Gateway reconnect re-runs `listSessions` and appends another copy of the summary lines to the chat log. Add the reconnect guard or track one-time startup insertion.
  - confidence: 0.93
- **[P2] Scope the session lookup to the current agent:** `src/tui/tui.ts:1107-1111`
  - body: This `listSessions` call omits `agentId`, while `sessions.list` returns all non-global/non-unknown sessions when no agent filter is supplied. In a multi-agent setup, opening the main TUI can display a title or preview from a newer work/research agent even though the docs and session picker scope sessions to the current agent. Pass the current agent id and keep the normal exclusions.
  - confidence: 0.86
