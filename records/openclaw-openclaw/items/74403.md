## Summary

Keep open. Current main still has the wrapper-order gap described by the linked DeepSeek regression, and this PR is the active implementation candidate. I did not find a concrete blocking correctness or security issue in the proposed patch, but the generic wrapper scope still needs maintainer review before merge.

## What This Changes

This PR deletes `reasoning_content` from OpenAI-completions messages after `extra_body` makes the final payload use disabled thinking, and adds a DeepSeek V4 regression test.

## Best Possible Solution

Land a narrow, tested disabled-thinking payload normalization path that makes DeepSeek V4 requests consistent whether disabled thinking comes from OpenClaw's own thinking level or from `params.extra_body`, while preserving enabled-thinking replay behavior.
