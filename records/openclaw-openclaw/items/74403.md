## Summary

Keep this PR open. Current main has DeepSeek V4 disabled-thinking cleanup when OpenClaw's resolved thinking level is `off`/`none`, but the generic `extra_body` wrapper still applies after the provider wrapper and only assigns the override without stripping stale `reasoning_content`. The PR is paired with #74374 and targets a real remaining DeepSeek V4 replay gap.

## What This Changes

This PR adds post-`extra_body` cleanup for OpenAI-completions payloads so messages lose `reasoning_content` when the final payload has `thinking.type: "disabled"`, plus a DeepSeek V4 regression test.

## Best Possible Solution

Land a narrow, tested cleanup that makes the final disabled-thinking DeepSeek V4 payload consistent whether disabled thinking comes from OpenClaw's thinking level or from `params.extra_body`, while preserving enabled-thinking replay behavior.
