## Summary

Keep open. This is a broad Android product-direction discussion, not a stale bug that can be closed conservatively. Current main already has a substantial Android Chat tab, Tailscale-aware connection docs, Assistant entrypoints, voice/camera features, and image attachment handling, but it still defaults post-onboarding to the Connect tab, is documented as an unreleased alpha companion node app, and does not prove the requested broader chat-first/default-mode or audio/video/fullscreen media improvements are complete. The linked external projects are useful context for maintainer review, especially the issue author’s clawchat2-public fork and the AndroidForClaw comment.

## Best Possible Solution

Keep the discussion open for Android/mobile maintainer triage. The best path is for maintainers to decide whether the official app should add a chat-first default/mode, then ask for focused PR slices against existing Android surfaces such as PostOnboardingTabs/RootScreen, ChatSheetContent/ChatController, ConnectTabScreen/Gateway docs, and chat media rendering. Avoid importing a broad fork wholesale; accept small, testable UX/media improvements with Android unit tests and docs updates where behavior changes.
