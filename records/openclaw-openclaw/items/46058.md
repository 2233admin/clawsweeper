## Summary

Keep open. Current main has real Android chat support, assistant-launch routing into Chat, image attachments, and Tailscale-aware connection docs, but the official Android app still defaults and documents a Connect-first companion-node flow; the remaining request is Android product direction plus possible focused upstreaming, not completed cleanup.

## What This Changes

The issue asks whether OpenClaw should add a chat-first official Android surface or accept focused Android UX, media, and connectivity improvements from independent Android forks.

## Best Possible Solution

Android/mobile maintainers should define the official app shape: keep Connect-first, add an explicit chat-first mode, or make Chat the default after pairing. Once that is decided, focused PR slices for chat UX, media playback, and Tailscale/connectivity can be reviewed without importing a whole fork.

## Reproduction Assessment

Not applicable as a bug reproduction. The current product state is high-confidence from source and docs: post-onboarding defaults to Connect, docs describe companion-node pairing, and chat/media support is present but image-oriented.

## Solution Assessment
