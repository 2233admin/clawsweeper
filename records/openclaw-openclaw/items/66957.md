## Summary

Keep open. Current main still resolves implicit providers before reading `models.mode`, so `replace` mode can still invoke plugin discovery and merge implicit providers despite the documented configured-providers-only contract. Related #63552 remains an open performance PR, not a merged fix for this replace-mode contract, and #68965 was closed as a duplicate of this issue.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `src/agents/models-config.plan.ts`: branch on `cfg.models?.mode === "replace"` before invoking `resolveImplicitProviders`, carry only explicitly configured providers through the existing normalization and source-managed secret enforcement path, and add regression coverage proving an injected implicit resolver is not called in replace mode while merge/default mode still discovers and merges implicit providers. Review #63552 separately unless it is updated and merged to satisfy this exact replace-mode contract.
