## Summary

Keep open. Current main still runs implicit provider discovery before reading `models.mode`, so `replace` mode can still pay plugin discovery cost despite the documented configured-providers-only contract. #73557 is open and explicitly uses closing syntax for this issue, so the right maintainer path is to review and land or close that PR rather than close the issue early.

## Best Possible Solution

Keep this issue open. Review and either land or close #73557. The eventual fix should branch on `cfg.models?.mode === "replace"` before any implicit provider discovery, use only explicit providers on that path, preserve normalization and source-managed secret enforcement, and add regression coverage proving the implicit resolver is not called in replace mode while merge/default mode still discovers providers.
