## Summary

Keep this PR open. Current main still implements the numbered `.bak.1` through `.bak.4` config backup ring and the colocated tests still assert that behavior. This PR is still a live implementation candidate for open #39923, with overlapping open PR #39982, so maintainers should choose or consolidate the datetime-backup approach rather than close it as obsolete.

## Best Possible Solution

Keep this PR open for maintainer review alongside #39923 and #39982. The best path is to select or consolidate one datetime-backup implementation in `src/config/backup-rotation.ts`, preserve or explicitly migrate legacy `.bak.N` files until retention is safe, make timestamp destinations collision-safe, keep bounded retention and permission hardening, update `src/config/config.backup-rotation.test.ts`, and add user-facing docs or a changelog entry only if the behavior ships.
