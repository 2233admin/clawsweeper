Keep this open. Current main still leaves both verified Feishu document write/append failure paths actionable: manual/self-built setup guidance does not tell users to grant the `document.convert` scope, and the non-batch descendant payload mapper can still overwrite normalized child arrays with the raw converted block value.

Fix `feishu_doc` write/append 400s on valid markdown by correcting Feishu document-convert permission guidance and descendant create payload normalization.

## Best Possible Solution

Keep the fix inside the Feishu plugin: align public docs, manual setup help, and diagnostics with the document-writing scope requirements, then normalize descendant payloads after spreading raw converted blocks and cover that with a focused regression test.
