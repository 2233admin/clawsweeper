## Summary

Keep open. Current main still does not implement the requested framework-level plugin circuit breaker: hook execution has static error handling and one void-hook timeout, plugin state is still loaded/disabled/error only, diagnostics and session locks do not attribute overruns to plugin+hook+session for breaker evaluation, and the plugin CLI/docs have no breaker status, forced-open, or reset surface. Related #55253 overlaps, but it is open and narrower than this issue’s lock attribution, persisted breaker registry, half-open/forced-open states, and operator controls.

## Best Possible Solution

Keep this issue open as core plugin/gateway stability work. The best path is an additive plugin-runtime breaker layer around hook and plugin-owned runtime execution, with structured plugin id plus hook/session telemetry, lock-overrun attribution, rolling failure/timeout budgets, persisted breaker state, status/diagnostics output, operator reset/forced-open controls, and tests proving one failing plugin is degraded without blocking unrelated plugins. Coordinate with #55253 only if maintainers decide to make one canonical tracking issue.
