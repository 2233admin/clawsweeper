## Summary

Keep open. Current main has partial resilience around plugin hook timeouts, static fail-open/fail-closed policy, plugin config isolation, and session-lock cleanup, but it does not implement the requested plugin-scoped circuit breaker with rolling budgets, lock attribution, persisted open/half-open/forced-open state, operator reset/status surfaces, or per-plugin degraded fail modes. The later deployment comment and related open #55253 reinforce that this is still active gateway/plugin-runtime reliability work.

## Best Possible Solution

Keep this open as the broad plugin/gateway runtime circuit-breaker tracker unless maintainers explicitly make #55253 canonical. The best implementation is a phased core runtime design: host-side hook timing/error metrics, plugin/hook/session lock attribution, rolling budgets, persisted closed/open/half-open/forced-open state, read-only status and reset controls, per-plugin fail modes, and regression tests proving one failing plugin degrades without blocking unrelated plugins.
