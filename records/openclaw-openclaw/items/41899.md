## Summary

Keep this issue open. Current main still has hook error handling, one void-hook timeout, invalid-config isolation, subsystem-specific breakers, and normal plugin enable/disable surfaces, but it does not implement the requested plugin-scoped circuit breaker with rolling budgets, lock attribution, persisted breaker state, open/half-open/forced-open transitions, status visibility, or operator reset controls. Related #55253 overlaps, but it is open and narrower than this issue.

## Best Possible Solution

Keep this issue open as core plugin/gateway stability work. The best implementation is an additive plugin-runtime breaker layer around hook and plugin-owned runtime execution, with structured plugin id plus hook/session telemetry, lock-overrun attribution, rolling failure/timeout budgets, persisted breaker state, status/diagnostics output, operator reset/forced-open controls, and tests proving one failing plugin is degraded without blocking unrelated plugins. Coordinate with #55253 only if maintainers choose a single canonical tracker.
