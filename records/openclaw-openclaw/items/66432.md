## Summary

Keep open. Current main still has the Feishu thread-history media gap: live inbound media is resolved, but fetched thread-history messages are still rendered from parsed placeholder content and do not retain raw Feishu body content for media-key extraction. The PR remains a focused implementation candidate with small review follow-ups, not obsolete or superseded.

## What This Changes

The PR adds raw Feishu thread-message content, resolves media for media-typed thread-history entries, inlines runner-compatible media hints into ThreadHistoryBody, and adds a happy-path Feishu bot test.

## Best Possible Solution

Finish this Feishu plugin fix by preserving raw Feishu API body content for thread-history records, resolving historical media through the existing media helper before ThreadHistoryBody formatting, keeping size limits and failure isolation, and covering both successful and failed historical media downloads in focused tests.
