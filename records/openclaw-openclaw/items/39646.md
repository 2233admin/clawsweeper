## Summary

Keep PR #39646 open. Current main still has the stale default-model reporting path for `openclaw models status --json`, rich `models status` output, auth-probe candidates, and `openclaw status`; the PR targets a real remaining bug, but the provided PR diff is incomplete and bundles unrelated changes.

## Best Possible Solution

Keep this open or replace it with a narrower maintainer patch. Centralize effective default-model resolution for `openclaw status`, `models status --json`, rich model-status output, and auth-probe candidates; validate removed configured providers without rejecting CLI-backed or plugin/synthetic providers; add regression coverage for the removed-provider case; split unrelated PR changes into separate focused PRs.
