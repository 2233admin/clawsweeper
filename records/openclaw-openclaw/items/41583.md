## Summary

Current main still lacks the requested toggle or minimize mode. Inbound channel metadata is still injected into both the system prompt and user-role prompt text, tests still assert metadata-on behavior for direct external-channel chats, and shared/Telegram/Discord config schemas still have no inbound metadata injection option. The related open PR #41299 is a separator/stripping proposal and explicitly leaves API/config changes out of scope.

## Best Possible Solution

Keep this open for maintainer product/security design. The best implementation would be a documented shared reply-runtime metadata mode that defaults to current metadata-on behavior, supports narrow per-channel/account or trusted-DM scoping, applies consistently to both the system metadata prompt and user-role metadata prefix, and adds regression tests for metadata-on behavior plus the opt-in minimized/disabled path. Review #41299 separately as a separator/stripping improvement, not as this feature.
