## Summary

Keep this issue open. Current main still injects both trusted system metadata and user-role untrusted metadata blocks for inbound channel messages, including direct non-webchat channels, and I did not find a shared, Telegram, or Discord config option to disable or minimize those blocks. The related #41299 remains an open separator/stripping proposal and does not implement the requested toggle.

## Best Possible Solution

Keep this issue open for maintainer product/security judgment. A good implementation would add a documented, narrowly scoped config option in the shared inbound reply prompt path, default metadata injection to enabled, cover trusted Telegram and Discord DM behavior with tests, and clearly document the provenance tradeoff.
