## Summary

Keep this issue open. Current main still always builds inbound trusted system metadata plus user-role untrusted metadata for channel messages, and I did not find a shared, Telegram, or Discord config option to disable or minimize those blocks. Related PR #41299 remains open and is scoped to adding a separator/stripper, not a metadata toggle.

## Best Possible Solution

Keep this open for maintainer product/security judgment. A good implementation would add a documented, narrowly scoped shared reply-runtime config option, default metadata injection to enabled, cover trusted Telegram and Discord DM behavior with tests, and clearly document the provenance tradeoff.
