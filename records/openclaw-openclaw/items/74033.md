## Summary

Keep open. Current main still reads each built-in external CLI credential source before checking stored profiles, so the PR is not implemented on main. The proposed one-file guard is relevant, but it would likely break the existing first-run bootstrap contract for Codex and Claude CLI credentials and needs auth/security maintainer review, especially because #73952 and #61694 overlap the same external-CLI sync semantics.

## Best Possible Solution

Keep this PR open for maintainer review, but request changes before landing. The best fix is a narrowly tested gate that avoids unrelated external CLI credential reads during runtime auth overlays while preserving the documented/tested first-run bootstrap behavior, or deliberately changing that behavior with explicit tests and docs. Reconcile the approach with #73952 and the broader opt-out request in #61694 so #74006 gets one coherent external-CLI sync policy.
