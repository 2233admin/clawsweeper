## Summary

Keep open. Current main still reads each built-in external CLI credential source before checking whether the relevant profile exists, so this PR is not implemented on main. The proposed guard targets the reported startup/keychain cost, but it also bypasses the current first-run bootstrap path that existing Codex and Claude CLI tests assert. The safer path is to preserve bootstrap while adding a scoped runtime overlay gate, likely coordinated with #73952 and the broader opt-out request in #61694.

## Best Possible Solution

Keep this PR open for maintainer review, but request changes before landing or supersede it with a coordinated fix such as #73952. The best implementation is a narrow runtime overlay gate that avoids unrelated external CLI credential reads and keychain prompts for status/startup paths while preserving the documented/tested first-run bootstrap behavior, with regression tests for both the skip case and the bootstrap case.
