## Summary

Keep open. Current main still stores config backups beside the active config as `.bak` plus numbered `.bak.N` files; the requested dedicated `config-backup/` directory with datetime names has not landed. The direct implementation PR was closed unmerged, while the open related PRs only cover timestamp suffixes for adjacent files, so this broader request remains valid.

## What This Changes

Store config backup files in a dedicated backup directory with datetime-based names instead of adjacent numbered `.bak` rotation files.

## Best Possible Solution

Keep this open until maintainers either consolidate it into a broader config-backup design issue or land a complete fix that moves config backups into a dedicated directory, uses deterministic datetime-based names with collision handling, preserves retention and permissions, handles legacy adjacent backups safely, and updates recovery, audit, doctor docs, and focused tests.
