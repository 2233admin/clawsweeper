## Summary

Keep this PR open. Current main has moved Moonshot onto the newer manifest/static catalog path, but the Kimi half of the PR’s intended behavior is still not implemented: Kimi has model definitions, yet no unauthenticated catalog exposure through manifest `modelCatalog`, `buildStaticProvider`, or `augmentModelCatalog`. The PR is useful but needs a rebase/update to current catalog architecture rather than closure.

## Best Possible Solution

Keep the PR open and update it to current main. The best path is to add Kimi `kimi-code` and `k2p5` through the current provider-owned offline catalog architecture, preferably manifest `modelCatalog` plus a static provider builder if runtime provider construction still needs it, then add regression coverage for provider-filtered model picking/listing without an API key. Leave Moonshot on the existing manifest/static catalog path unless a focused regression still reproduces there.
