## Summary

Keep this PR open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still has daily/idle rollover behavior, but it does not implement the PR’s one-shot scheduled-reset continuity hint or an equivalent assistant-message carry-forward path.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best fix is a rebased implementation that captures a bounded assistant-only transcript tail during scheduled daily/idle rollover before archive, strongly delimits or sanitizes the carried text, injects it through every prompt path including queued/deferred runs, clears it only after actual delivery, preserves explicit /new and /reset as blank-slate resets, and adds focused regression tests.
