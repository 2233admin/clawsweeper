## Summary

Keep this PR open. Current main still does not implement the scheduled daily/idle reset continuity hint, and the PR remains a relevant but incomplete implementation candidate rather than something already shipped or clearly superseded.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is to rebase it onto current main, add a safe assistant-only tail reader or equivalent continuity mechanism, sanitize or strongly delimit the carried assistant text, clear the hint only after it is actually included in the prompt, support deferred queue bodies, and add focused regression tests for daily/idle resets while preserving explicit reset blank-slate behavior.

## Risks / Open Questions

- Security review: the provided PR diff touches only TypeScript session/reply/transcript code, with no CI workflow, dependency, lockfile, install script, publish metadata, secret handling, or downloaded-artifact changes. The notable security concern is prompt-surface risk from embedding prior assistant text without sanitizing or clearly delimiting newlines/quotes.
- The proposed `applySessionHints`-only injection is likely incomplete for `followup`, `collect`, `steer`, and `steer-backlog` queue modes because queued prompts are built from `effectiveBaseBody` rather than the prefixed body.
- The PR needs regression coverage for daily and idle scheduled resets, explicit `/new` and `/reset` exclusion, one-shot clearing after actual delivery, multiline assistant text, and queued/deferred delivery paths.
