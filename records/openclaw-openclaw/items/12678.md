## Summary

Current main has meaningful adjacent controls, but it still does not implement the requested capability-manifest, explicit grant/revoke, pre-model default-deny filtering, execution-boundary authorization, or durable high-risk capability audit layer. The issue discussion also explicitly distinguishes the @vainplex/openclaw-governance plugin and current tool-policy work as useful prior art, not a replacement for the missing core/plugin-API capability grant model.

## Best Possible Solution

Keep this open as a core/plugin-API security hardening item. A reviewable path is schema/parser diagnostics for required and optional sensitive capabilities first, then explicit persisted grants/revocations, CLI/UI/doctor visibility, pre-model filtering after existing tool-policy resolution, execution-boundary checks for HTTP/MCP/non-prompt invocation paths, and durable audit records. Coordinate with #12219 for skill manifest shape and #63881 for MCP deny-list parity, while treating @vainplex/openclaw-governance as complementary runtime policy rather than the canonical grant layer.
