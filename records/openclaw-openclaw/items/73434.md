## Risks / Open Questions

- The PR is now partly stale: main already owns the drain timeout implementation, while the PR still has hand-rolled Layer 3 concept tests instead of exercising the landed `drainPendingToolTasks` helper.
- Changing `allowSyntheticToolResults` to true for every provider could affect custom or provider-owned replay semantics; this needs explicit maintainer/provider compatibility review.
- A default 60s per-tool-call timeout is a broad runtime behavior change and could regress legitimate long-running tools unless the policy, opt-out behavior, docs, and config metadata are aligned.
- The PR adds a config surface but the current diff only shows type/Zod/generated schema changes; docs/help metadata and generated-schema validation still need maintainer-quality proof before merge.
