The current PR diff still contains meaningful unmerged runtime work, so it should stay in maintainer review rather than be closed as implemented.

## Best Possible Solution

Keep this PR open as an active implementation candidate. ## Risks / Open Questions

- Changing `allowSyntheticToolResults` to true for all providers could alter replay semantics for OpenAI-compatible, Mistral, and custom providers; this needs provider-owner compatibility review rather than an automatic close.

- The branch is partly stale because current main already ships the pending-task drain layer; the PR should be rebased/narrowed so review focuses on the remaining synthetic-result and tool-timeout changes.
