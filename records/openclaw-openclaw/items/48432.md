## Summary

Keep this issue open. Current main still inserts spaces into long non-copy-sensitive tokens via `sanitizeRenderableText`, protects underscore file-like tokens but not hyphen-separated package-style names, and the TUI assistant rendering path still routes visible assistant text through that sanitizer. There is also an open closing PR (#69340), so the issue should remain open until that PR is reviewed and either landed or closed.

## Best Possible Solution

Keep this issue open while #69340 is reviewed. The best fix is to preserve copy-sensitive code/table content in the TUI without injecting display spaces, with focused regression coverage for hyphenated package names and fenced code blocks; if #69340 lands and proves the central user problem is solved, close this issue as implemented by that PR.
