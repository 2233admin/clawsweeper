## Summary

Keep open. Current main still rejects the reported local/self-hosted Firecrawl configuration: `resolveEndpoint()` requires HTTPS and allows only `api.firecrawl.dev`, and both Firecrawl search and scrape still route through that resolver. The latest release tag also contains the same restriction, and open implementation PRs #63941 and #74013 remain unmerged, so this issue should stay open until maintainers land a canonical fix.

## Best Possible Solution

Keep this issue open and review/land one focused Firecrawl plugin fix, preferably the explicit opt-in shape from #74013 or an equivalent maintainer-approved variant. The fix should keep the default cloud path HTTPS-only, allow trusted self-hosted/private endpoints only with documented config, preserve URL credential/query/hash sanitization, update `extensions/firecrawl/openclaw.plugin.json`, Firecrawl docs, web_fetch docs, and Firecrawl tests, then close this issue after the PR merges and ships.
