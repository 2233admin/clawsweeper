## Summary

Keep open. Current main still rejects `type: "aws-sdk"` entries in `auth-profiles.json` as `invalid_type` and drops them. The linked PR #69753 identified this exact path and proposed a scoped fix, but it was closed unmerged; current main still lacks an equivalent marker-skip implementation or regression test.

## Best Possible Solution

Keep #69708 open and implement the narrow parser/runtime fix proposed by the closed PR #69753 or an equivalent: treat persisted `type: "aws-sdk"` Bedrock entries as SDK-managed non-secret markers instead of invalid credentials, keep them out of invalid-entry warnings, preserve provider-level `auth: "aws-sdk"` fallback/default-chain behavior for explicit `amazon-bedrock:default` selections, and add regression tests covering both silent marker handling and mixed real-invalid entries.
