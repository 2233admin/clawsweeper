## Summary

Keep open. Current main still rejects canonical `auth-profiles.json` entries whose normalized `type` is `aws-sdk`, while Bedrock config, runtime auth, docs, and tests still expect `auth: "aws-sdk"` to use the AWS SDK default credential chain for EC2/IMDS setups.

## Best Possible Solution

Implement a narrow auth-profile store fix that treats `aws-sdk` as an SDK-managed non-credential marker during load, skips it without adding it to `AuthProfileCredential`, suppresses invalid-entry warnings only for that marker, preserves warnings for genuine invalid entries, and leaves Bedrock runtime auth on provider config/AWS SDK default-chain resolution. Add focused regression tests and a user-facing changelog fix entry crediting the reporter and useful #69753 context.
