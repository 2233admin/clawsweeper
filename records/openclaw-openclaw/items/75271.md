## Summary

Keep open. Current main has an outbound Anthropic/Gemini guard that strips trailing assistant prefill turns before a provider call, but the session-file repair helper still does not persistently trim a JSONL file that ends on a normal assistant message. The linked open PR #75284 explicitly targets this issue and is unmerged, so this issue should remain open while that implementation is reviewed.

## What This Changes

Repair agent/embedded session JSONL recovery so files ending in assistant turns no longer replay as Anthropic assistant prefill.

## Best Possible Solution

Land a focused session-file repair that removes trailing non-tool-call assistant message entries during `repairSessionFileIfNeeded`, with regression tests for single and multiple assistant tails, mid-transcript assistant entries, and existing empty-error/silent-reply behavior.

## Reproduction Assessment

Yes. A valid session JSONL ending with a normal `message.role: "assistant"` entry will pass through current `repairSessionFileIfNeeded` unchanged because no repair counter is incremented; the outbound provider-call strip is a separate runtime mitigation rather than a persistent file repair.

## Solution Assessment

Yes. Dropping trailing assistant entries during file repair is the narrowest maintainable fix because it removes provider-rejected replay state without fabricating a synthetic user turn.
