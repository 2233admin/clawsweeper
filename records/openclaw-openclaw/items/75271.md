## Summary

Keep open. Current main still does not persistently repair session JSONL files that end on a repaired assistant artifact; it only strips trailing assistant prefill from the outbound provider call. The open, unmerged PR https://github.com/openclaw/openclaw/pull/75284 explicitly targets this issue with closing syntax, so this issue should remain open while that PR is reviewed and either landed or closed.

## What This Changes

Repair agent/embedded session JSONL recovery so repaired files do not remain ending on an assistant turn that can replay as Anthropic assistant prefill.

## Best Possible Solution

Land a focused session-file repair that removes only replay-unsafe trailing assistant repair artifacts during repairSessionFileIfNeeded, with regression coverage for one and multiple assistant tails plus preservation cases for mid-transcript assistant entries, normal completed assistant turns, and existing silent-reply behavior.

## Reproduction Assessment

Yes. A valid session JSONL with a header, a user turn, and a trailing assistant error entry with empty content will be rewritten by current repair into fallback assistant content while still ending on assistant; that is the stored shape this issue reports.

## Solution Assessment

Yes. Persistent session-file repair is the right layer for the reported on-disk corruption, and dropping the unsafe trailing assistant repair artifact is narrower than appending a synthetic user turn or relying only on provider-call stripping.
