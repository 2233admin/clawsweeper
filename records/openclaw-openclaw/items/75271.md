## Summary

Keep open. Current main still has a source-level on-disk repair gap for JSONL files ending in an assistant message, and open PR #75284 already claims the targeted fix, so this issue should stay open until that PR merges or is closed.

## What This Changes

The issue asks embedded session repair to ensure repaired session JSONL does not end on an assistant turn, preventing Anthropic assistant-prefill 400s on resume.

## Best Possible Solution

Land or revise #75284 so `repairSessionFileIfNeeded` drops non-tool trailing assistant message entries on disk while preserving mid-transcript assistant turns and covering single and multiple trailing assistant entries with focused tests.

## Reproduction Assessment

Yes. A source-level reproduction is a session JSONL containing a valid header followed by a final assistant message; current repair either no-ops for normal assistant content or rewrites empty-error assistant content while still leaving assistant as the final entry.

## Solution Assessment

Yes, trimming trailing assistant entries during session-file repair is the narrowest maintainable fix. Appending synthetic user text would mutate transcript history, and outbound-only stripping leaves corrupted persisted files behind.
