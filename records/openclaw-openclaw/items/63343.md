## Summary

Current main still matches the reported race: `ensureTabAvailable()` discards the cold-start `openTab("about:blank")` result, relists once, and filters local-managed candidates to tabs with `wsUrl`. The linked implementation PR #63355 is closed unmerged, and no open replacement PR was found, so this should stay open as a narrow browser plugin repair candidate.

## What This Changes

The issue requests a browser plugin fix so `ensureTabAvailable()` can recover when local-managed Chrome tab creation succeeds before CDP `/json/list` exposes a per-tab `webSocketDebuggerUrl`.

## Best Possible Solution

Keep this issue open and land a clean, isolated browser plugin fix. Capture the cold-start `openTab()` result, briefly poll/retry when local-managed `wsUrl` filtering removes otherwise visible tabs, tolerate transient `listTabs()` failures during that recovery window, and fall back to the latest usable target-id tab when `wsUrl` never appears. Add focused tests that avoid real one-second waits, keep the AbortSignal follow-up separate, and add a user-facing changelog fix credited to the reporter.

## Repair Work Prompt

Fix the browser plugin `ensureTabAvailable()` local-managed `wsUrl` race reported in openclaw/openclaw#63343. Keep the change narrow and isolated to browser tab selection.

Expected behavior: when `ensureTabAvailable()` cold-starts and `listTabs()` is initially empty, capture the returned `openTab("about:blank")` tab and use it as a fallback candidate if CDP `/json/list` has not caught up. When `supportsPerTabWs` filtering removes all candidates but tabs exist, poll/retry briefly for `wsUrl` discovery, tolerate transient `listTabs()` failures during that recovery window, and fall back to the latest usable non-empty tab snapshot or the opened tab target if `wsUrl` never appears. Preserve existing target-id resolution, last-target behavior, tab aliases, managed tab cleanup, and SSRF/navigation policy. Do not include the separate AbortSignal/Playwright orphan-operation hardening from the issue body.

Use #63355 only as implementation context; do not resurrect unrelated branch changes. Add focused tests in `extensions/browser/src/browser/server-context.tab-selection-state.test.ts` for: opened tab result used when relist stays empty, polling until `wsUrl` appears, fallback to latest non-empty snapshot without `wsUrl` after polling exhausts, and transient `listTabs()` failure during polling. Avoid real one-second sleeps by using controllable timers or short/injected constants if needed. Add a user-facing changelog fix entry in the active version, credited with `Thanks @OpenCodeEngineer`.
