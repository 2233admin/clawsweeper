## Summary

Keep open. Current main still has the reported browser tab-selection race: `ensureTabAvailable()` discards the cold-start `openTab("about:blank")` result, immediately relists tabs, and filters local-managed candidates to entries with `wsUrl`, so a newly created target without a populated `webSocketDebuggerUrl` can still throw `BrowserTabNotFoundError`. The linked fix PR is closed unmerged, and no open replacement PR was found.

## What This Changes

The issue asks for the browser plugin tab-selection path to tolerate local-managed CDP targets whose per-tab `webSocketDebuggerUrl` appears after the target is created.

## Best Possible Solution

Land a clean, isolated browser plugin fix that captures the cold-start `openTab()` result, briefly polls/retries when `supportsPerTabWs` filtering removes otherwise visible tabs, tolerates transient `listTabs()` failures during that recovery window, and falls back to the latest usable target-id tab when `wsUrl` never appears. Add focused regression tests and a user-facing changelog credit for the reporter.

## Reproduction Assessment

Yes. The deterministic reproduction path is to make the selection harness return no tabs, have `openTab("about:blank")` return a created target, then make the next `listTabs()` return that target without `webSocketDebuggerUrl`; current main filters that tab out and throws `BrowserTabNotFoundError`.

## Solution Assessment

Yes. The narrowest maintainable solution is in the browser plugin tab-selection code and its colocated tests; broader Playwright AbortSignal cleanup should remain a separate follow-up.

## Repair Work Prompt

Expected behavior: when `ensureTabAvailable()` cold-starts and `listTabs()` is initially empty, capture the returned `openTab("about:blank")` tab and use it as a fallback candidate if CDP `/json/list` has not caught up. When `supportsPerTabWs` filtering removes all candidates but tabs exist, poll/retry briefly for `wsUrl` discovery, tolerate transient `listTabs()` failures during that recovery window, and fall back to the latest usable non-empty tab snapshot or the opened tab target if `wsUrl` never appears. Preserve existing target-id resolution, last-target behavior, tab aliases, managed tab cleanup, and SSRF/navigation policy. Do not include the separate AbortSignal/Playwright orphan-operation hardening from the issue body.
