## Summary

Keep open. Current main still routes Windows gateway exec through a single PowerShell spawn, and the shared spawn fallback helper only retries EBADF, not EPERM. I did not find an implemented fallback, diagnostic, or duplicate that covers the reported Windows 11 `spawn EPERM` / all-commands-fail path.

## Best Possible Solution

Keep this issue open and ask for or implement a focused Windows exec investigation: capture the selected shell path, cwd, and spawn error code; add a narrow fallback or clearer diagnostic for Windows PowerShell spawn EPERM if validated; and cover it with mocked Windows supervisor/exec tests before changing host execution behavior.
