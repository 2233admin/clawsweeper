## Summary

Keep this open. Current main still matches a plausible Windows host-exec `spawn EPERM` failure path: Windows exec selects one PowerShell wrapper, non-sandbox exec passes that argv through the supervisor, and the shared spawn fallback helper still retries only `EBADF` by default rather than `EPERM`. The related Microsoft tracker PR is only an open inventory item, not a merged fix or canonical replacement.

## What This Changes

The issue asks OpenClaw to fix Windows 11 agent exec runs where every Feishu or web command fails immediately with `spawn EPERM` even though the CLI itself runs.
