## Summary

Keep open. Current main still does not implement this PR's specific omitted-coreGatewayHandlers reuse path: the loader cache key includes gateway method names, but active runtime state does not remember those names and the compatibility check cannot reconstruct them when a helper/runtime caller opts into gateway binding but omits the handler map. The PR remains a focused implementation candidate, with the main review concern being preservation of the default-mode capability boundary during rebase/review.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, preserve the activatePluginRegistry hook-runner guard, persist active core gateway method names with the active registry state, reuse only when method-name equality proves compatibility, retain the default-mode non-inheritance regression test, and land after the targeted plugin loader/runtime tests plus the changed gate pass.
