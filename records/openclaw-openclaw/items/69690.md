## Summary

Keep open. Current main still lacks this PR's omitted-coreGatewayHandlers reuse path: the loader cache key includes gateway method names, but active runtime state does not persist those names for later compatibility reconstruction. The PR remains a focused implementation candidate and appears to address the earlier default-mode capability-boundary concern, so the next step is maintainer review rather than cleanup closure.

## What This Changes

This PR stores active gateway core method names in plugin runtime state and updates loader compatibility checks plus regression tests so gateway-bindable helper/runtime loads that omit handler maps can reuse a compatible active registry without letting default-mode loads inherit synthetic gateway bindings.
