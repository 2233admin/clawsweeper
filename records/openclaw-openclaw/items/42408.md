## Summary

Keep this issue open. Current main still has the missing exclusion/docs work: memory search config is include-only, builtin indexing still recursively indexes eligible Markdown under `extraPaths`, and the docs mention `extraPaths` but do not cover benchmark/queryset contamination or a user-configurable exclusion mechanism.

## Best Possible Solution

Keep open. Implement a documented memory indexing exclusion mechanism for the bundled memory-core path and shared config: schema/types/help, `resolveMemorySearchConfig`, file discovery/read access, watcher setup, reindex scope hashing, and regression tests showing benchmark/queryset Markdown under `extraPaths` can be excluded. Update the memory docs with absolute vs workspace-relative `extraPaths`, `memory status --deep`, reindexing after path changes, broad-path noise, and benchmark/evaluation contamination guidance.
