## Summary

Keep this issue open. Current main still partially covers the report with `memory status --deep` diagnostics for missing `extraPaths`, but the core requested exclusion surface and docs guidance for benchmark/queryset contamination are still missing.

## Best Possible Solution

Keep open. Implement a documented memory indexing exclusion mechanism in the bundled memory-core path and shared config: schema/types/help, `resolveMemorySearchConfig`, file discovery/read access, watcher setup, reindex scope hashing, and regression tests proving benchmark/queryset Markdown under `extraPaths` is excluded. Docs should also add path hygiene guidance for absolute versus workspace-relative `extraPaths`, `memory status --deep`, reindexing after path changes, broad-path noise, and benchmark/evaluation contamination.
