## Summary

Keep open. Current main has moved thread-bound subagent work toward generic plugin lifecycle hooks, and Feishu now has hook handlers, so the old hardcoded-channel concern appears partly addressed. The requested automatic channel policy/injection is still not implemented: `sessions_spawn` only treats `thread` as enabled when the model explicitly passes `thread: true`, and the public channel `threading` contract is still reply-threading focused rather than a subagent spawn policy.

## Best Possible Solution

Add a backward-compatible, typed subagent-threading policy to the channel/plugin contract or an equivalent generic hook/descriptor, have `sessions_spawn` derive effective `thread` and `mode` from that policy for the caller channel, and cover Feishu plus at least one non-auto-injecting channel in contract tests and docs.

- The current hook-based design partially solves the hardcoded-channel problem, so maintainers should decide whether the remaining work should be a declarative plugin policy, a tool-context/prompt policy, or an explicit non-goal.
- Subagent thread-binding docs appear to lag current code paths for Feishu/Matrix hooks, which may confuse implementation and review unless updated with the chosen design.
