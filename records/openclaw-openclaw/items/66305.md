## Summary

Keep open: current main still immediately clears successful empty non-read-only catalog loads, and this PR adds plausible negative-cache behavior with focused tests, but the branch is stale against current `readOnly` catalog semantics and should not be merged as-is.

## What This Changes

The PR adds a 60-second negative cache for successful empty `loadModelCatalog()` results plus tests for immediate reuse, `useCache: false`, stale concurrent loads, and retry after the window.

## Best Possible Solution

Port the negative-cache behavior onto current main with the empty-result retry state applied only to normal cached loads, while preserving read-only non-mutating catalog reads, configured-provider model merging, provider supplemental merging, explicit cache reset behavior, and transient error retryability.
