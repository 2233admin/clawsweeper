## Summary

Keep this PR open. Current main still drops the shared model catalog cache immediately after a successful empty load, so the proposed negative-cache fix is not implemented, but the branch is now unmergeable and needs to be ported over newer read-only catalog semantics before it can land.

## What This Changes

The PR adds a 60-second negative cache for successful empty `loadModelCatalog()` results and regression tests for empty-cache reuse, `useCache: false`, stale concurrent loads, and retry after the window.

## Best Possible Solution

Port the negative-cache behavior onto current main with a bounded empty-result retry window for normal cached loads, while preserving current read-only non-mutating behavior, configured-model merging, cache resets, stale-load guards, and focused regression coverage.
