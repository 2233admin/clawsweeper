## Summary

Keep open. Current main still implements only `mention` and `always` group activation, has no `contextualActivation` config/protocol/logging surface, and both Telegram and WhatsApp still skip or store unmentioned group messages before any LLM decision path. The linked implementation PR #38350 is closed and unmerged, while #1721 is closed and superseded by this broader request.

## What This Changes

Add opt-in contextual group activation for Telegram and WhatsApp so a lightweight model can decide when the bot should peek, engage, continue, or disengage in group chats without explicit mentions.

## Best Possible Solution

Keep this open as the canonical product/API discussion. The best path is either a first-class, opt-in contextual activation feature with privacy-safe defaults across config, Gateway sessions/protocol, Telegram and WhatsApp pre-dispatch gating, observability, CLI/docs/tests, or a typed pre-gating plugin/channel activation seam that lets an external plugin make the decision before bundled runtimes drop unmentioned group messages.
