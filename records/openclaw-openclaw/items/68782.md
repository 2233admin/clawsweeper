## Summary

Keep open. Current main has shipped meaningful partial mitigation for the runtime-dependency pain, but it still does not implement selective installation or the proposed strict install/runtime model: packaged installs still include the broad bundled plugin and skill surface, setup has no channel/plugin selector, config has no `strictMode` or install-selection field, and discovery still scans bundled/global roots before allow/deny policy is applied.

## What This Changes

Request to add selective installation and runtime loading controls for bundled plugins, skills, and channels, including setup selection, strict allow mode, lazy registration, and plugin status reporting.

## Best Possible Solution

Keep this as a maintainer-owned umbrella, but split it into narrower design-backed work: define the desired slim-core versus bundled-package model, decide whether strict discovery is a supported config contract, then implement setup/channel selection and docs without undoing the current runtime-deps scoping.
