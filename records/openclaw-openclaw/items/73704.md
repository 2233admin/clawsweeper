## Summary

Keep open. Current main still has the safeguard auto-compaction behavior this PR targets: the hook prefers the active session model, and the embedded safeguard runtime is registered with the session model instead of the resolved compaction override. The linked issue is still open and this PR is the active implementation candidate, with a pending functional review note to resolve before landing.

## Best Possible Solution

Review and land this PR, or an equivalent focused patch, after addressing the modelRegistry-miss fallback/warning question. The fix should resolve the effective compaction provider/model before safeguard runtime registration, prefer the configured runtime compaction model when present, preserve no-override fallback to the session model, preserve provider-switch auth-profile dropping, and keep regression coverage for both runtime registration and hook precedence. The linked issue should close through the merge.
