## Summary

Keep this PR open. Current main still has the safeguard auto-compaction behavior the PR targets: the hook prefers the active session model, and embedded safeguard runtime registration stores the session model instead of resolving `agents.defaults.compaction.model`. The linked bug remains open and this PR is the active implementation candidate.

## Best Possible Solution

Review and land this PR, or an equivalent focused patch. The fix should resolve the effective compaction provider/model before safeguard runtime registration, prefer the configured runtime compaction model when present, preserve the no-override session-model fallback, preserve provider-switch auth-profile dropping, warn clearly when an explicit override is missing from the registry, and keep regression coverage for runtime registration plus hook precedence. The linked issue should close through the merge.
