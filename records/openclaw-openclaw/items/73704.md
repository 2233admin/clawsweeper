## Summary

Keep open. Current main still has the safeguard compaction bug this PR targets, but the proposed patch should not merge as-is because the new runtime target path uses a raw registry lookup and can still ignore valid configured compaction models that require OpenClaw's shared model resolver.

## What This Changes

The PR threads model registry access into embedded safeguard runtime registration, resolves a configured compaction target before setting the runtime model, flips the safeguard hook to prefer the runtime model over the context model, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep this PR open and revise it so safeguard runtime registration resolves the configured compaction model through the same model-resolution contract used by embedded runs, while preserving the no-override session-model fallback, provider-switch credential behavior, and regression coverage for both registered and configured/dynamic model overrides.

## Reproduction Assessment

Yes. Current main can be verified from source by configuring safeguard mode with a different `agents.defaults.compaction.model`: runtime registration stores the session model, and the hook then prefers `ctx.model` over `runtime.model`. The remaining PR defect is also source-reproducible by using a valid configured fallback/dynamic model that `resolveModelWithRegistry` can resolve but `modelRegistry.find` cannot.

## Solution Assessment

No. The direction is right, but the raw `modelRegistry.find` in the proposed runtime target is not the narrowest maintainable fix because it bypasses OpenClaw's shared model resolver and can keep the override broken for configured provider models.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Use the shared resolver for override models:** `src/agents/pi-embedded-runner/extensions.ts:167-172`
  - body: This new path resolves the configured compaction override with `modelRegistry.find(provider, modelId)` only. That bypasses `resolveModelWithRegistry`/`resolveModelAsync`, which the embedded model path uses to handle configured provider fallback models and plugin dynamic models. For a valid `agents.defaults.compaction.model` that is resolved by those shared paths but not present in the raw registry, this returns `runtime.model` as undefined and safeguard auto-compaction falls back to the session model, so the override is still ignored. Please resolve the target through the shared model-resolution contract or pass an already-resolved compaction model into this helper.
  - confidence: 0.86
