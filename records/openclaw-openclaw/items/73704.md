## Summary

Keep open. Current main still has the safeguard auto-compaction bug this PR targets: the hook prefers the active session `ctx.model`, and the embedded safeguard runtime is still registered with the session model rather than the resolved `agents.defaults.compaction.model`. The linked issue #57901 remains open and is paired with this contributor PR, so the right cleanup path is to review and land this PR or an equivalent fix rather than close it as obsolete.

## Best Possible Solution

Review and land this PR or an equivalent focused patch that resolves the effective compaction provider/model before registering the safeguard runtime, makes the safeguard hook prefer the configured runtime compaction model when present, preserves no-override fallback to the session model, and keeps regression coverage for both runtime registration and hook precedence. Let linked issue #57901 close through the merge.
