## Summary

Keep this PR open. Current main still has the safeguard compaction behavior the PR targets: the hook prefers the active session model, and embedded safeguard runtime registration stores the session model instead of resolving `agents.defaults.compaction.model`. The linked bug is still open and this PR is the active implementation candidate.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent focused patch. The final fix should resolve the effective compaction provider/model before safeguard runtime registration, prefer the configured runtime compaction model when present, preserve the no-override session-model fallback, preserve provider-switch auth-profile dropping, clearly surface missing registry entries for explicit overrides, and keep regression tests for runtime registration plus hook precedence.
