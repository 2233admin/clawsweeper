## Summary

Keep open. Current main still does not honor a different `agents.defaults.compaction.model` in the safeguard auto-compaction path, and this PR is a focused implementation candidate for the remaining gap. The branch is not cleanup-closeable because it is still open, unmerged, dirty against main, and has failing check runs that need rebase/conflict resolution and fresh validation before maintainer review can finish.

## What This Changes

The PR resolves the configured safeguard compaction provider/model before embedded runtime registration, prefers the runtime model in the safeguard hook, threads `modelRegistry` through the embedded run and direct compact call sites, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent focused patch after rebasing: resolve the effective compaction provider/model before safeguard runtime registration, prefer that runtime model in the hook, preserve no-override fallback and provider-switch auth behavior, and keep the regression coverage tied to the safeguard path.

## Reproduction Assessment

Yes. On current main, configure safeguard compaction with `agents.defaults.compaction.model` set to a different model than the active session model; source inspection shows runtime registration stores the session model and the hook prefers `ctx.model`, so the override cannot win in that path.

## Solution Assessment

Yes for the code direction. Reusing the existing compaction target resolver and passing `modelRegistry` into the safeguard runtime setup is narrower and more maintainable than duplicating model parsing, but the branch must be rebased and validated before merge.
