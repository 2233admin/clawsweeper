## Summary

Keep this issue open. Current main still uses a 30,000 ms TUI streaming watchdog default, the production TUI does not pass any user/config override into the event handler, and the documented/configured ACP stream controls cover coalescing/projection rather than this client-side watchdog threshold. Related #69978 and PR #69026 are relevant implementation/UX follow-ups, but they are not merged and do not make this older active configurability request obsolete.

## Best Possible Solution

Keep the issue open and implement a supported TUI streaming watchdog control. The likely path is to read a documented openclaw.json field or persisted TUI preference, plumb the resolved value into createEventHandlers as streamingWatchdogMs, support a disable mode if accepted, update schema/help/docs when config-backed, and add focused tests for default 30s behavior, a longer threshold, disabled mode, and any per-run warning dedupe chosen from #69978.
