## Summary

Current main still has the reported TUI behavior: a 30-second client-side streaming watchdog, a user-visible `streaming watchdog` warning, and no user-facing config, CLI option, TUI setting, or slash command to tune or disable it. The focused #69026 PR and broader #69978 follow-up are both still open in the supplied GitHub context, so this issue should remain open for maintainer follow-up.

## Best Possible Solution

Keep this issue open and implement a user-facing TUI streaming watchdog control. A conservative fix would add a validated config or local TUI setting for threshold and disablement, plumb it through `TuiOptions` into `createEventHandlers`, document it on the public TUI/config pages, and add regression coverage for default behavior, configured threshold, disabled watchdog, and any `/watchdog` command if maintainers choose that UX. Coordinate with #69026 for the focused threshold/message change and #69978 for duplicate-warning/runtime-command design.
