## Summary

Keep open. Current main still does not implement a schema-backed global `agents.defaults.reasoningDefault`; it only supports per-agent `agents.list[].reasoningDefault`, and the main runtime/status helpers still resolve reasoning from session to per-agent default to fallback without consulting global defaults. The related open PR #71817 is useful partial per-agent channel work for Telegram/Feishu, while #73182 is a broader cost/leak report; neither supersedes this narrower global-default request.

## Best Possible Solution

Keep this issue open until OpenClaw adds a schema-backed `agents.defaults.reasoningDefault` with clear precedence: inline `/reasoning` and session override > per-agent `agents.list[].reasoningDefault` > global `agents.defaults.reasoningDefault` > model-capability/off fallback. The implementation should update config types, Zod schema, generated schema/help metadata, docs, main reply resolution, status/current-level helpers, and regression tests. Treat #71817 as partial per-agent channel work, not the canonical global-default fix.
