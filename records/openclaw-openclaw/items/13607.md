## Summary

Keep open. Current main still does not implement a schema-backed global `agents.defaults.reasoningDefault`; it only supports `agents.list[].reasoningDefault`, and the runtime/current-level docs and helpers still fall back from directive/session to per-agent default to `off` or model capability without consulting global defaults. The related open PR #71817 is partial per-agent Telegram/Feishu work, and the broader #73182 cost/leak report does not supersede this narrower global-default request.

## Best Possible Solution

Keep this issue open and implement a focused config/runtime/docs change adding `agents.defaults.reasoningDefault` with values `off | on | stream`. The change should preserve per-session and per-agent override precedence, wire the global default into main reply resolution and current-level/status helpers, update Zod/types/generated schema/help/docs, and add regression tests for global default inheritance plus override precedence.
