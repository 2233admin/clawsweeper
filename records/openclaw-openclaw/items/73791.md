## Summary

Keep open. Current main still lacks the missing-python fallback for writeFileWithinRoot/copyFileWithinRoot, and the linked fs-safe bug remains open; this PR is a focused fix candidate but should not merge until the real stream-copy missing-spawn path is handled and validated.

## What This Changes

The PR updates `src/infra/fs-safe.ts` to fall back to the existing legacy write/copy paths when the pinned write helper cannot spawn, and adds fs-safe tests for write fallback, copy fallback, non-spawn passthrough, and mock cleanup.

## Best Possible Solution

Land a corrected fallback that mirrors remove/mkdir for write and copy, handles real child spawn errors inside the pinned write helper for both buffer and stream inputs, and includes a helper-level or python3-less regression before using this PR to resolve #72362. Keep the broader gateway-image Python availability question in #75041 separate.

## Reproduction Assessment

Yes. The linked bug has a concrete Docker slim reproduction, current main still spawns bare `python3` and normalizes write/copy helper failures, and a focused local probe reproduced the stream-spawn failure shape that the PR’s mocked copy test does not cover.

## Solution Assessment

No, not yet. Falling back to the existing legacy path is a plausible narrow fix for buffer writes, but the copy stream path needs real missing-spawn handling in `runPinnedWriteHelper` or equivalent coverage before this is the best maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Handle real stream spawn errors before falling back:** `src/infra/fs-safe.ts:798`
  - body: The new copy fallback only runs if `runPinnedWriteHelper` rejects. With a real missing interpreter on the stream path, `events.once(child, "close")` rejects from the child `error` event before `pipeline(sourceStream, child.stdin)` settles; on Node 24 that can become an unhandled rejection/process exit, so this catch may never execute. Please handle the child spawn error in the helper or add a real missing-spawn stream regression before relying on this fallback.
  - confidence: 0.86
