## Summary

Keep open. Current main still relies on in-process session transcript update events plus `sessionsDirty` state, and the non-force sync path skips session reconciliation after a clean restart or missed event unless a full reindex, targeted session sync, or `--force` is used.

## What This Changes

Request memory-core session transcript catch-up reconciliation so missed `onSessionTranscriptUpdate` events or gateway restarts cannot leave session-memory indexing silently stale.

## Best Possible Solution

Add bounded session-source reconciliation in memory-core that treats on-disk transcript files and indexed `files` rows as the durable source of truth, marks missing or changed session files dirty, and lets normal session sync process those files without requiring `--force`. Keep the change plugin-owned, add focused regression coverage for a nonempty stale index after a fresh manager, and handle the status-count mismatch either in the same narrow patch or as a clearly linked follow-up.

## Reproduction Assessment

Yes. The code path itself gives a high-confidence reproduction: with `sources` including `sessions`, an existing index/meta, no full-reindex drift, and empty `sessionsDirtyFiles` after a fresh manager, `session-start` and non-force CLI sync skip `syncSessionFiles`, so an updated or missing-index session file remains invisible.

## Solution Assessment

Yes. A lightweight reconciliation/dirty-marking pass is the narrowest maintainable fix; forcing every search or every startup into a full reindex would be more expensive and would work against the existing async/session-delta design.

## Repair Work Prompt

Fix the session-memory stale-index bug tracked by https://github.com/openclaw/openclaw/issues/62625. Current main loses in-process `sessionsDirty` state across restarts and only marks session files dirty from `onSessionTranscriptUpdate`, so clean restarts or missed events leave transcript files unindexed until `--force` or full reindex.

Expected fix boundary: stay inside the memory-core plugin/runtime surfaces. Add a bounded reconciliation path for the builtin session source that compares `listSessionFilesForAgent(agentId)` with existing SQLite `files` rows for `source = "sessions"` and marks missing or changed files dirty (`sessionsDirtyFiles` + `sessionsDirty`) before normal non-force sync decisions need that state. Cover startup/session-start and manual `openclaw memory index` without `--force`; consider whether interval/on-search should also pick up disk drift so missed events during a long-running gateway do not stay invisible. Reuse existing helpers such as `loadMemorySourceFileState`, `resolveMemorySourceExistingHash`, `buildSessionEntry`, `sessionPathForFile`, and the shared session filename classifier instead of ad hoc `.jsonl` filtering. Do not add core migrations, bundled-owner ids, broad plugin API changes, or unconditional full reindexes on every startup.

Regression coverage: add or extend memory-core tests to create an existing nonempty session index, mutate or add a session transcript after simulating a fresh manager with empty `sessionsDirtyFiles`, then verify session-start or non-force CLI-style sync indexes the changed/missing session file. Keep the existing #39732 full-reindex behavior covered. If touching `memory status` counts, make `scanSessionFiles` use the same `isUsageCountedSessionTranscriptFileName` semantics as the indexer and add a focused CLI/status test.

Changelog: add a user-facing Fixes entry in the active changelog section, crediting the reporters with `Thanks @unitypeaceproject and @dwbutler`. Docs are only needed if a new user-visible knob or status field is added.
