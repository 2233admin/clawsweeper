## Summary

Current main still does not implement on-demand HEARTBEAT.md loading. The loader/cache reads the fixed workspace bootstrap set, including HEARTBEAT.md, before run-kind filtering, and docs still describe normal-run injection when heartbeat guidance is enabled. The related closed heartbeat lightweight-context work is adjacent but does not solve this specific default non-heartbeat loading path.

## Best Possible Solution

Keep this issue open and implement a narrow run-kind-aware bootstrap loading/cache path that skips HEARTBEAT.md for default non-heartbeat runs before disk reads and snapshot caching, while preserving HEARTBEAT.md for actual heartbeat runs and the existing lightweight heartbeat behavior. Update the system-prompt/heartbeat/token-use docs and focused bootstrap/cache tests to define the new contract.
