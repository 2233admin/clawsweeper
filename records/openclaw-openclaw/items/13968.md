## Summary

Current main still has not implemented the requested on-demand HEARTBEAT.md loading. The loader and session bootstrap cache still read the fixed workspace bootstrap file set before run-kind filtering, and the documented/default behavior still allows HEARTBEAT.md in normal default-agent runs when heartbeat guidance is enabled. The related #55311 work is useful but narrower; it forwards lightweight heartbeat context rather than making normal runs skip HEARTBEAT.md before load/cache/prompt injection.

## Best Possible Solution

Keep this issue open. Implement a run-kind-aware bootstrap loader/cache path that can skip HEARTBEAT.md for default non-heartbeat runs before disk reads, session snapshot caching, hooks, and prompt injection, while preserving HEARTBEAT.md for real heartbeat runs and the existing lightweight heartbeat behavior. Update heartbeat/system-prompt docs and focused tests to define the new normal-run contract.
