## Summary

Keep this issue open. Current main still does not implement configurable per-job delayed execution retry for recurring cron failures; existing cron.retry is global and applied only in the one-shot branch, while recurring errors use natural next-run scheduling with a backoff floor. The related implementation PR #69252 was closed unmerged, so there is no merged/shipped fix to close against.

## What This Changes

The issue requests optional per-cron-job retry configuration that re-runs failed recurring cron executions after configured delays before the next normal schedule.

## Best Possible Solution

Implement an additive core scheduler feature for per-job recurring execution retry: extend cron job types and Gateway schemas, add CLI add/edit flags, persist retry attempt state, classify transient errors, schedule retries at endedAt plus configured delay while attempts remain and before the next natural slot, and document how this differs from one-shot retry and recurring error backoff.
