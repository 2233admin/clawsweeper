## Summary

Keep this PR open. Current main has adjacent work, especially Brave llm-context validation/docs and generic restart deferral, but it still does not implement the PR's central bundle: the macOS wizard timeout remains 0.75s, Control UI persisted nav width is still capped at 400px, skill-creator still omits the direct quick_validate.py audit workflow, there is no automatic pre-reset memory-flush hook, and gateway restart deferral is not wired to active browser profiles.

## Best Possible Solution

Keep this PR open for maintainer/contributor action, but split or rebase it into narrower pieces: one for the macOS wizard timeout, one for Control UI nav width/storage bounds if that UX is still desired, one for skill-creator quick_validate documentation, and a separately reviewed reset-memory hook that addresses the existing review comments. Browser restart deferral should be redesigned against the current extensions/browser plugin service rather than the removed src/browser/server.ts shape.
