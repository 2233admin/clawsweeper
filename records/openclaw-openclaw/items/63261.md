## Summary

Keep open. Current main at 213f92a9 still has the reported Discord runtime owner-auth gap: the Discord config adapter reads only legacy dm.allowFrom, while docs and normalization treat top-level channels.discord.allowFrom as canonical. The prior fix PRs were closed unmerged, and the open related security-audit PR does not fix this runtime command-auth path.

## Best Possible Solution

Land a focused Discord plugin fix that resolves owner allowFrom from account.config.allowFrom ?? account.config.dm?.allowFrom in extensions/discord/src/shared.ts, with regression coverage through the production Discord plugin adapter and resolveCommandAuthorization. Also align extensions/discord/src/security.ts to the same top-level-or-legacy contract if maintainers agree the DM policy resolver should follow the canonical config shape. Do not change core fallback semantics or the separate security-audit PR scope unless explicitly requested.
