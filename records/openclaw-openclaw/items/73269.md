## Summary

Keep open. Current main has useful mitigations for this leak class, including commentary-phase suppression and message-tool-only defaults for group/channel rooms, but it still does not provide the requested per-agent auto-reply disablement or a shared public/internal/final-only delivery contract; the broader related visible-delivery boundary report remains open.

## What This Changes

The issue asks OpenClaw to stop automatically sending assistant text to a channel when tool calls follow, either through a per-agent auto-reply disablement or by delivering only final public text after tools complete.

## Best Possible Solution

Define one shared visible-delivery boundary that can distinguish public final assistant text from internal/commentary text, apply it consistently across final, preview, block, and channel-specific streaming paths, and document/configure the intended behavior for direct chats, group/channel rooms, Feishu, and explicit message-tool sends.

## Reproduction Assessment

Partly. The current tests and source give a high-confidence reproduction of the remaining policy gap: direct/source turns still default to automatic visible delivery, while group/channel turns are suppressed; the exact reporter Feishu transcript sequence still needs a focused synthetic or live fixture.

## Solution Assessment

No. The current message-tool-only controls and commentary-phase filtering are useful mitigations, but they are not the complete solution because they do not add per-agent auto-reply suppression or an explicit public/internal assistant audience contract.
