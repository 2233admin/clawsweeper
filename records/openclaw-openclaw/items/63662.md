## Summary

Keep open. Current main still lacks the proposed `agents.defaults.memoryInjection` and `agents.defaults.memorySearch.autoRecall` contract, and the adjacent Active Memory/LanceDB features do not implement the same memory-core configuration surface. The PR is useful source work, but it currently has blocking correctness findings and unrelated dirty changes, so it needs maintainer review rather than conservative cleanup closure.

## What This Changes

The PR adds configurable `MEMORY.md` injection modes, `memorySearch.autoRecall`, a memory-core `before_prompt_build` auto-recall hook, schema/docs/tests, and several unrelated script, QA, delivery-context, protocol, and UI fixture changes.

## Best Possible Solution

Ship this, if desired, as a narrow memory-core/config change: add the schema and docs for the new memory settings, implement `recall-only` as a bootstrap filter, make auto-recall use supported live config with hot-reload-safe hook registration, and keep unrelated guard/QA/protocol/delivery changes out of the branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Register auto-recall unconditionally:** `extensions/memory-core/src/auto-recall.ts:80-82`
  - body: The hook is skipped when startup config has no `autoRecall`, even though the handler reads live config per turn. Enabling `agents.defaults.memorySearch.autoRecall` by config hot reload will not add a `before_prompt_build` handler until restart, so the new setting appears ignored.
  - confidence: 0.88
- **[P2] Use the supported live config API:** `extensions/memory-core/src/auto-recall.ts:90`
  - body: This bundled plugin calls `api.runtime.config.loadConfig()` inside the prompt hook. That API is deprecated and covered by the repo's config-boundary guard; use `api.runtime.config.current()` or a passed live config path instead.
  - confidence: 0.84
- **[P2] Keep channel plugin delivery resolution first:** `src/utils/delivery-context.ts:116-126`
  - body: The new thread-child fallback runs before `messaging.resolveDeliveryTarget` for Telegram. Telegram's resolver normalizes topic conversations into `to` plus a numeric topic `threadId`; bypassing it can route topic replies with raw conversation IDs.
  - confidence: 0.87
