## Summary

Current main still rejects provider-only `models.providers.openai` entries because the core schema, generated schema, and TypeScript contract still require `models`. The embedding runtime would use the provider `baseUrl` after validation accepts it, and open PR #39766 still has closing syntax for this issue, so the issue should stay open until that or an equivalent fix lands.

## Best Possible Solution

Keep this issue open until #39766 or an equivalent focused fix lands. The fix should make `models.providers.<id>.models` optional with a validation-time default of `[]`, align the generated schema and TypeScript config contract, and add regression coverage showing provider-only OpenAI-compatible config is accepted and memory embeddings use `models.providers.openai.baseUrl` when `memorySearch.remote.baseUrl` is absent.
