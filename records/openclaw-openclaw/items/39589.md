## Summary

Current main still has the reported bug: provider-only `models.providers.openai` config remains invalid because the core schema, generated schema, and TypeScript contract still require `models`. The memory embedding runtime would use `models.providers.<id>.baseUrl` once validation accepts the entry, and open PR #39766 is still an unmerged implementation candidate with closing syntax, so this issue should stay open until that or an equivalent fix lands.

## Best Possible Solution

Keep this issue open until #39766 or an equivalent focused fix lands. The fix should make `models.providers.<id>.models` optional with a validation-time default of `[]`, regenerate and align the generated schema and TypeScript config contract, and add regression coverage showing provider-only OpenAI-compatible config is accepted and that memory embeddings use `models.providers.openai.baseUrl` when `memorySearch.remote.baseUrl` is absent.
