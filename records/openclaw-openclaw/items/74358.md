## Summary

Keep open. Current main still has a credible Slack silence path for the reporter’s object-only streaming config: Slack defaults it to partial/native streaming, flat top-level DMs remain preview-ineligible without a thread target, and the partial-text draft callback is only wired for the non-native preview path. The issue also overlaps open PR #56544, so the right next step is maintainer reconciliation rather than cleanup closure.

## What This Changes

Slack should keep in-flight answer preview text active while suppressing only tool-progress lines when `streaming.preview.toolProgress` is `false`.

## Best Possible Solution

Resolve the Slack object-config path so `streaming.preview.toolProgress: false` means answer preview remains on and in-preview tool lines are off. The fix should include Slack regression coverage for object-only config and flat DMs, reconcile or update #56544, and align Slack docs/config help with the shared streaming contract.

## Reproduction Assessment

Yes, at source level. The reporter provided a stable object-only config, and current main resolves that shape to partial/native streaming while still gating flat-DM preview eligibility and omitting the draft partial-text callback on native streaming paths.

## Solution Assessment

Unclear because there is no proposed patch here. Fixing only `suppressDefaultToolProgressMessages` is not sufficient; the safer solution must address Slack preview eligibility/native fallback behavior and the disabled tool-progress contract together.
