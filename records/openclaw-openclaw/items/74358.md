## Summary

Keep open. Current main still has unresolved Slack behavior around `streaming.preview.toolProgress: false`: the shared docs promise answer previews continue when only tool-progress lines are hidden, but Slack has separate native-streaming and draft-preview paths, no regression test for the disabled-progress case, and the latest discussion correctly identifies that the missing `mode`/`nativeTransport` config determines which path needs the fix.

## What This Changes

The issue reports that Slack preview streaming becomes silent while tools run when `streaming.preview.toolProgress` is disabled, instead of continuing answer-preview text without tool-progress lines.

## Best Possible Solution

Slack should explicitly satisfy the shared streaming contract for both Slack preview paths: when preview streaming is active, `streaming.preview.toolProgress: false` should hide interim tool-progress lines while preserving answer preview updates where that transport supports them, suppress generic progress chatter consistently, and align Slack docs/config help plus regression tests with that behavior.
