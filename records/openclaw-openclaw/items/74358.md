## Summary

Keep open. Current main still leaves the reporter's Slack object-only `streaming.preview.toolProgress: false` path plausibly silent in flat DMs: the config defaults to partial/native streaming, draft preview eligibility still requires a DM thread target, and the partial-text draft callback is not wired when native streaming is active or preview streaming is disabled. The report also overlaps open PR #56544, so this should be reconciled rather than closed or duplicated blindly.

## What This Changes

Restore Slack in-flight answer preview streaming for verbose Slack turns when `streaming.preview.toolProgress` is `false`, while hiding tool-progress lines instead of making the preview silent.

## Best Possible Solution

Resolve the Slack object-config DM path so `streaming.preview.toolProgress: false` means answer preview remains active and tool-progress lines are hidden, then add Slack regression coverage for flat DM/object config and update Slack docs/config help to match the shared streaming contract.
