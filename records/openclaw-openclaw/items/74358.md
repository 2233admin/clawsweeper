## Summary

Keep open. The reported Slack `streaming.preview.toolProgress: false` path is not implemented as the issue expects on current `main`: Slack still gates preview tool-progress and default-progress suppression from the same `previewToolProgressEnabled` boolean, current Slack docs/config help are inconsistent about whether `false` hides or separates progress, and there is no Slack regression coverage for the disabled-tool-progress preview-text path.

## What This Changes

Restore Slack draft preview text streaming for verbose Slack sessions when `streaming.preview.toolProgress` is false, so disabling tool-progress lines does not make in-flight previews silent.

## Best Possible Solution

Keep this open and land a narrow Slack-owned fix that decouples answer draft preview text from tool-progress bullet rendering. `streaming.preview.toolProgress: false` should keep draft preview text active while suppressing in-preview tool lines and avoiding separate generic progress chatter, with Slack docs/config help updated to match the shared streaming docs.
