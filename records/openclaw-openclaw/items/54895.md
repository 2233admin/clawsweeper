## Summary

Current main still lacks the requested inter-session progress stream for long-running `sessions_send` calls. The tool still supports fire-and-forget or synchronous wait, wait expiry still returns `status: "timeout"`, A2A announce remains completion-only, and the gateway exposes no agent-callable run-event read/subscription method. Adjacent Task Flow and ACP parent streaming do not cover observing a peer run initiated through `sessions_send`.

## What This Changes

Request inter-session progress streaming or subscription for long-running `sessions_send` runs so callers can distinguish accepted delivery from wait timeout and observe tool, assistant, and lifecycle progress.

## Best Possible Solution

Add an additive, permission-checked run-event observation surface, likely backed by bounded per-run event buffering, then expose it through either a `sessions_subscribe` tool or an explicit `sessions_send` progress option. The implementation should also separate accepted delivery from wait timeout and add coverage for long-running sends, event isolation, ACP/Codex behavior, and topic-bound delivery.
