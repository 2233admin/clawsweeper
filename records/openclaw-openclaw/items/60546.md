## Summary

Keep open. Current main still leaves the built-in microsoft-foundry provider on OpenAI-compatible Foundry APIs only, still offers discovered deployments without an Anthropic/Claude guardrail, and the linked guardrail PR #72446 is open and unmerged with closing syntax for this issue.

## Best Possible Solution

Keep this issue open while maintainers review #72446 or an equivalent microsoft-foundry fix. The near-term path is a built-in guardrail that filters or warns on Anthropic deployments; the longer-term product path is native Anthropic Foundry routing through `anthropic-messages` with the correct `/anthropic/v1/messages` endpoint and headers.
