## Summary

Keep open. Current main still swallows Lobster workflow-file resolution failures and then runs the same string as an inline pipeline, which matches the reported misleading fallback. The latest release tag has the same behavior, and #68106 remains open with closing syntax for this issue, so cleanup should wait for that PR or an equivalent fix to land.

## What This Changes

The issue requests that Lobster treat relative workflow-file-like `pipeline` values as workflow files or clear path errors instead of falling back to inline pipeline parsing.

## Best Possible Solution

Review and land #68106, or an equivalent focused Lobster plugin change, that rethrows workflow resolution errors for workflow-file-like inputs and adds regression coverage for missing relative workflow files while preserving normal inline pipelines.
