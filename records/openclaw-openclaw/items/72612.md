## Summary

Keep open. Current main still has the attachment state race and lacks the browser-side pre-read size guard, while the PR head is a focused implementation with targeted tests and no blocking review finding from this read-only pass.

## What This Changes

The PR adds append-only chat attachment updates, attachment error reporting, a browser pre-read size guard based on the gateway-effective attachment limit, Control UI bootstrap propagation for that limit, and focused UI/gateway regression tests.

## Best Possible Solution

Land this PR or an equivalent focused patch after maintainer review: keep the fix at the Control UI compose/bootstrap boundary, keep the server limit authoritative, and retain the regression tests for concurrent appends, oversized rejection, configured limits, and error formatting.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: trigger two overlapping attachment reads so each completion rebuilds state from an old props.attachments snapshot, and attach a file over the gateway limit to see FileReader invoked before any browser size check.

## Solution Assessment

Yes. The proposed approach is the narrow maintainable fix: append completed reads through live UI state, expose the existing gateway-effective limit to bootstrap, reject oversized files before FileReader I/O, and avoid changing transport or model execution semantics.
