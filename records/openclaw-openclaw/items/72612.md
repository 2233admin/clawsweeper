## Summary

Keep open. Current main still has replacement-style attachment updates built from captured `props.attachments` snapshots and no browser-side pre-read size guard, while this PR is a focused implementation of the missing append path, bootstrap limit propagation, user-visible attachment errors, and regression coverage. I did not find a concrete blocking bug in the latest patch.

## What This Changes

The PR updates the Control UI chat attachment flow to append completed uploads through live state, reject files over the gateway-effective attachment limit before `FileReader.readAsDataURL()`, surface attachment errors, pass the effective limit through Control UI bootstrap, and add focused UI/gateway tests.

## Best Possible Solution

Land this focused implementation, or an equivalent one, after maintainer review and validation: keep the fix at the Control UI compose/bootstrap boundary, use the gateway-effective limit as the browser pre-read guard, preserve server-side validation as authoritative, and retain the new regression coverage.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: overlapping file selection/paste/drop reads write back from stale `props.attachments`, and oversized supported files reach `FileReader.readAsDataURL()` before any UI size check.

## Solution Assessment

Yes. The PR is the narrow maintainable fix: append completed reads through live UI state, expose the existing gateway limit through bootstrap, reject oversized files before reading, and cover the behavior without changing transport or model execution semantics.
