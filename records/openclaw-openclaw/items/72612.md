## Summary

Keep this PR open. Current main still uses replacement-style attachment updates from captured `props.attachments` snapshots, still calls `FileReader.readAsDataURL()` without a browser-side size guard, and still does not expose the gateway-effective attachment limit through Control UI bootstrap. The PR remains a focused implementation candidate; #33404 is closed unmerged prior art, and #67916 tracks a separate local assistant-media bootstrap bug.

## Best Possible Solution

Keep this PR open for maintainer review and landing if CI/review stay clean. The best fix is this focused change or an equivalent one: append completed file reads through live attachment state, reject files over the gateway-effective `agents.defaults.mediaMaxMb` limit before `FileReader.readAsDataURL()`, surface attachment errors in the chat callout, pass the effective limit through Control UI bootstrap, and keep the targeted chat/bootstrap/gateway regression tests.
