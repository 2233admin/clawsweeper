## Summary

Keep this PR open, but do not merge it as-is. The Control UI hang is still credible on current main, while the patch leaves timed-out browser RPCs registered and applies late history responses through a narrower path than current main's history reconciliation.

## What This Changes

This PR changes Control UI chat history loading to race the initial `chat.history` request against a 10-second timeout, clear the timeout timer, late-apply guarded successful responses, and bump a freshness token when sending chat messages.

## Best Possible Solution

Land a current-main-native Control UI history timeout that exits the empty loading state with a targeted error, explicitly cancels or removes the timed-out browser RPC, safely ignores or handles late responses, and preserves startup retry, per-state request-version, session, optimistic-tail, stream, and hidden-history-message behavior.

## Reproduction Assessment

Yes. A high-confidence focused reproduction is to mock `state.client.request("chat.history")` with a never-settling promise; current main leaves `chatLoading` true, and the empty chat renderer shows the loading skeleton for that state.

## Solution Assessment

No. The fixed `Promise.race` timeout is directionally useful, but it is not the best solution because it does not cancel the underlying `GatewayBrowserClient.pending` entry and does not reuse current history reconciliation contracts.

## Review Findings

- **[P2] Cancel timed-out history RPCs:** `ui/src/ui/controllers/chat.ts:121-123`
  - body: When the timeout branch wins, `Promise.race` stops `loadChatHistory`, but the original `requestPromise` remains in `GatewayBrowserClient.pending` until a server response or socket close. Retrying stalled loads can accumulate pending requests, so the timeout path needs per-request cancellation or explicit pending removal.
  - confidence: 0.86

On current main the normal history load also hides heartbeat/synthetic/empty history artifacts and preserves optimistic tail messages, so a delayed response can reintroduce hidden entries or drop local optimistic content unless it uses the same reconciliation path.
  - confidence: 0.78
