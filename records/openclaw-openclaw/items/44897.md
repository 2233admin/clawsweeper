## Summary

Keep this PR open because current main still has no bounded client-side timeout for a never-settling initial `chat.history` request, but the proposed branch should not merge as-is: it is stale against newer WebChat history reconciliation code and leaves the underlying Gateway RPC pending after the UI timeout path wins.

## What This Changes

The PR modifies `ui/src/ui/controllers/chat.ts` to wrap `loadChatHistory`'s `chat.history` request in a 10-second `Promise.race`, clear the timer, late-apply a successful delayed history response under freshness guards, and bump a freshness token when sending chat messages.

## Best Possible Solution

Land a current-main-native timeout path for initial WebChat history loads that dismisses the loading state, explicitly cancels or removes the timed-out Gateway pending RPC, preserves the newer per-state request-version/session/stream/optimistic-tail guards and startup retry behavior, and adds focused fake-timer coverage for both UI state and pending cleanup.
