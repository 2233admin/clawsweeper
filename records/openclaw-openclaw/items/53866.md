## Summary

Current main does not implement Firecrawl browser auth during web-search setup. The PR still targets a real onboarding improvement, but the current architecture has moved web-search setup behind the shared flow and provider-owned `runSetup` seam, so this branch should not be closed as implemented or merged as-is.

## Best Possible Solution

Keep this PR open, but request a rewrite: implement Firecrawl browser auth inside `extensions/firecrawl` as provider-owned `runSetup`, keep the PKCE/helper code plugin-local, persist credentials only under `plugins.entries.firecrawl.config.webSearch.*` and `webFetch.*` while respecting SecretRef mode, add Firecrawl plugin tests plus shared flow coverage, and update the Firecrawl docs when that implementation lands.
