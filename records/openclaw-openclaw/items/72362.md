## Summary

Current main still leaves the reported missing-`python3` write failure unresolved: non-Windows writes still spawn the pinned Python helper, helper failures are still normalized to the generic `path is not a regular file under root` message, and the affected session-memory and workspace-only write paths still use that API. Two open PRs explicitly target this issue, so it should stay open until a fix lands or maintainers reject those candidates.

## Best Possible Solution

Keep this issue open and review #72531, #72635, or an equivalent maintainer-authored fix. The minimum acceptable path is a clear missing-`python3` diagnostic with regression coverage for writeFileWithinRoot/copyFileWithinRoot; the stronger path is a carefully bounded fallback or removal of the runtime Python dependency, plus Docker docs/doctor guidance if python3 remains required.
