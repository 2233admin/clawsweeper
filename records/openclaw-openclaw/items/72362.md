Keep this open. ## What This Changes

The issue asks OpenClaw to stop fs-safe write/copy consumers from failing in slim Docker images without `python3`, or at least to document the runtime dependency and surface a clear missing-interpreter error.

## Best Possible Solution

Keep this issue open until main lands one canonical fix that handles missing pinned-write helper spawn failures for write/copy paths with regression tests, preserves the existing boundary guarantees, and updates Docker docs or doctor diagnostics if `python3` remains a supported runtime requirement.
