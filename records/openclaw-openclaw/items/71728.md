## Summary

Keep open. Current main has adjacent grounding guidance, memory citations, inbound provenance, trajectory export, runtime truthfulness wording, and task completion summaries, but it still does not implement the RFC’s per-factual-claim confidence tiers, output-wide source attribution, known-versus-inferred semantics, or autonomous reporting for unverified claims.

## What This Changes

The issue requests a verifiable agent truthfulness contract with confidence tiers, source attribution, known-versus-inferred labeling, and unverified-claim reporting for autonomous runs.

## Best Possible Solution

Define a versioned truthfulness contract across agent runtime, task completion records, Gateway protocol, trajectory/export data, and visible completion reports, then implement the smallest enforceable slice with schema tests, runtime tests, and docs after maintainers choose the claim semantics.

## Reproduction Assessment

Not applicable: this is an RFC rather than a concrete bug report. The high-confidence check is source and docs inspection on current main, which shows adjacent mechanisms but no implemented per-claim confidence or unverified-claim contract.

## Solution Assessment

Unclear: the requested direction is reasonable, but the best implementation boundary has not been designed. A prompt-only change would be weaker than the requested verifiable trust contract; this needs protocol/runtime/reporting design first.
