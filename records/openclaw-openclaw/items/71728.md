## Summary

Keep open. Current main has adjacent verification prompts, inbound provenance metadata, memory citations, trajectory capture, and task-completion handoff metadata, but it still does not implement the RFC’s per-factual-claim confidence tiers, output-wide source attribution, known-versus-inferred semantics, or autonomous unverified-claim reporting contract. This remains a maintainer product/protocol design item, not conservative cleanup.

## What This Changes

The issue requests a truthfulness contract that marks factual claims with confidence/source/known-vs-inferred metadata and reports unverified claims in autonomous completions.

## Best Possible Solution

Define a versioned truthfulness contract across agent runtime, task records, gateway protocol, and user-facing reports, then implement the smallest enforceable slice with tests and docs once maintainers choose the claim/source/confidence semantics.
