## Summary

Keep this PR open. Current main still returns cron cache hits without promotion and evicts the first Map key, so the observable FIFO-not-LRU behavior remains. The current PR head is a narrow implementation candidate, and the latest branch update addresses the earlier review concern by adding key-level cache assertions.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent maintainer patch that promotes cron cache hits with delete/re-set and includes key-level regression coverage proving the promoted entry survives eviction while the next-oldest entry is removed.
