## Summary

Keep this PR open. Current main still returns cron cache hits without refreshing Map insertion order, then evicts the insertion-order head when the cache is full, so the FIFO behavior reported by the linked bug remains present. The PR branch is a narrow implementation candidate and, per the supplied discussion, was repaired to include key-level regression coverage after the earlier bot review called out weak tests. The security pass found no supply-chain or secret-handling surface in the supplied PR file list.

## Best Possible Solution

Review and land this PR, or land an equivalent maintainer patch, preserving the narrow production change that promotes cron cache hits with delete/re-set and the key-level regression test proving the promoted expression survives eviction while the next-oldest expression is removed.
