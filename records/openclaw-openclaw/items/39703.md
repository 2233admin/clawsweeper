## Summary

Keep this PR open. Current main at 7a23b2d945cc2d2e5c452e4641213e71de8dcc66 still has FIFO cron schedule cache behavior: cache hits return without promotion, and eviction deletes the first Map key. The current PR head is a narrow implementation candidate that addresses the linked bug and the earlier review concern by adding key-level LRU assertions.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent maintainer patch that promotes cron cache hits with delete/re-set and includes key-level regression coverage proving the promoted entry survives eviction while the next-oldest entry is removed.
