## Summary

Keep this PR open. Current main still has FIFO behavior in the cron schedule cache: cache hits return without refreshing Map insertion order, and eviction removes the first inserted key. The PR remains a narrow implementation candidate, now with key-level regression assertions in the provided PR context.

## Best Possible Solution

Review and land this PR, or land an equivalent maintainer patch that promotes cron cache hits with delete/re-set and keeps the key-level regression coverage proving the promoted cron expression survives eviction while the next-oldest entry is removed.
