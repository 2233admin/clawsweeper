## Summary

Keep this PR open. Current main still has FIFO cron schedule cache behavior: cache hits return immediately without updating Map insertion order, while eviction deletes the first Map key. The PR is still a focused implementation candidate, and the latest PR context shows the earlier weak test was repaired with key-level cache assertions.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent maintainer patch that promotes cron cache hits with delete/re-set and includes key-level regression coverage proving the promoted entry survives eviction while the next-oldest entry is removed.
