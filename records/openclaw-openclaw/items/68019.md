## Summary

Keep this PR open. Current main still ships the fixed-size memory chunking path only, so the proposed multi-strategy work is not obsolete; however, the branch needs maintainer review and a current-main rebase before it is mergeable.

## What This Changes

This PR adds selectable memory-search chunking strategies, extends memory chunking config/schema/help, routes memory-core indexing through a strategy resolver, persists strategy-aware reindex metadata, and adds chunking plus reindex-state tests.

## Best Possible Solution

Keep this PR as an implementation candidate, but require a current-main rebase that moves the strategy implementation and exports onto the package-backed memory SDK surface, then have maintainers choose the public chunking API and validate default compatibility, strategy behavior, metadata migration, docs/schema alignment, and model-call handling.
