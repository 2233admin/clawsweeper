## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: local Gateway connection details still target loopback for configured local binds, the server still does not bind loopback alongside non-loopback bind hosts, and reasonless close errors still do not surface a mode-aware auth hint. The bind-host half overlaps with open #40247, but the silent-disconnect auth diagnostic is not fully covered there, and the PR as written has unresolved security/test review concerns.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to let #40247 or an equivalent server-side change handle tailnet/custom loopback reachability, then narrow or rebase this PR onto src/gateway/connection-details.ts for a tested, mode-aware silent-close diagnostic that preserves the plaintext WebSocket security policy. Add focused tests for tailnet/custom bind behavior and token/password/trusted-proxy close-hint behavior before merge.
