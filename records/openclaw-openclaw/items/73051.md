## Summary

Keep open. Current main has scoped model controls for defaults, per-session chat, Gateway session patches, cron jobs, and subagent spawns, but no single owner-safe operation that retargets all persisted and active model selections; the prior ClawSweeper comment reached the same conclusion and no implementation is linked in the issue context.

## What This Changes

The issue requests a single command to retarget all model selections across the main agent, subagents, cron jobs, other tasks, and the active Telegram context when a provider is rate-limited.

## Best Possible Solution

Design an owner-only bulk model migration flow under the model command surface that validates the target provider/model, provides a dry-run preview, offers explicit scopes for defaults, sessions, cron, subagents, fallbacks, media models, and auth profiles, and preserves deliberate overrides unless the operator opts into changing them.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current main exposes separate scoped model controls but no single bulk model migration command or API.

## Solution Assessment

Unclear as phrased. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky.
