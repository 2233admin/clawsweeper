## Summary

Keep this issue open. Current main still renders the misleading `no bootstrap files` wording for the normal/deep status overview, and the provided GitHub context shows open PR #43614 still links this issue as the implementation candidate.

## Best Possible Solution

Keep this issue open and have a maintainer review, refresh, land, or supersede PR #43614. The narrow code path is `buildStatusAgentsValue`: describe `bootstrapPendingCount` as active bootstrapping, not general bootstrap-file injection, and add a regression test for mature workspaces where `BOOTSTRAP.md` is absent while regular workspace bootstrap files are still injected.
