## Summary

Keep open. Current main and the latest release still contain the misleading `no bootstrap files` status wording, and an open fixing PR already links this issue for the narrow wording/test change, so this should not close before that PR lands or is replaced.

## What This Changes

The issue asks for `openclaw status --deep` to stop saying `no bootstrap files` when no agent has a pending one-shot `BOOTSTRAP.md` file.

## Best Possible Solution

Land or adapt #43614’s narrow status wording and regression test so `bootstrapPendingCount` is described as active bootstrapping or pending `BOOTSTRAP.md`, while leaving workspace bootstrap injection behavior unchanged.

## Reproduction Assessment

Yes. The source path is high-confidence: with `bootstrapPendingCount` equal to zero, `buildStatusAgentsValue` returns `no bootstrap files`, while `bootstrapPendingCount` is computed only from `BOOTSTRAP.md` existence.

## Solution Assessment

Yes. A wording-only status helper change plus focused test coverage is the narrowest maintainable fix because the underlying pending-bootstrap detection and broader bootstrap injection behavior are already separate.
