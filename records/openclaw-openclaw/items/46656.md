## Summary

Keep this issue open. Current main still lacks the requested WebChat/Control UI inline-button contract: no WebChat inline-buttons config, no `buttons` or `chat.callback` gateway protocol, no final WebChat event projection for presentation/button blocks, and no Control UI normalized content or rendering path for clickable buttons. The closed narrower #54720 was already routed back to this older, more detailed tracker, so this issue remains canonical.

## Best Possible Solution

Keep this issue open as the canonical tracker. The best path is to design WebChat/Control UI support around the existing semantic `presentation` model, then add an additive Gateway protocol/config contract, final assistant payload projection, Control UI normalization/rendering for buttons, and a carefully scoped click route. Webhook routing should be decided explicitly, either as a secure core feature or a separate plugin/ClawHub follow-up.
