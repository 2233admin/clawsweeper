## Summary

Keep open. Current main still lacks the requested WebChat/Control UI inline-button path: the chat protocol has no button or presentation payload for WebChat events, there is no registered or scoped `chat.callback` RPC, WebChat config exposes only history truncation, and the Control UI message model/rendering has no interactive button item.

## What This Changes

Add WebChat/Control UI support for inline buttons, including a capabilities config path, button payload delivery, callback RPC handling, and Control UI rendering.

## Best Possible Solution

Implement an additive WebChat interactive-control path using the shared `presentation` model: preserve sanitized final-only button blocks through chat events and history, render them in Control UI with clicked/disabled state, and add a scoped `chat.callback` RPC that initially routes selected values back into the same session. Treat webhook routing as a separate hardened follow-up.
