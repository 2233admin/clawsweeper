## Summary

Keep open. Current main still builds light-phase Dream Diary input from a fixed lookback window of recent short-term entries, passes the same staged snippets into the narrative path, and appends successful narrative text without cross-cycle body/input duplicate suppression. Existing adjacent work fixed cron replay, self-ingestion, exact duplicate diary blocks, idempotent subagent parameters, and detached narrative concurrency, but it does not implement the requested per-phase last-emitted cutoff or same-body suppression for distinct timestamps.

## Best Possible Solution

Keep this open and implement a focused memory-core fix for Dream Diary duplicate suppression. The safest direction is to record a per-workspace/per-phase narrative input or body fingerprint under memory/.dreams and skip launching or appending a Dream Diary entry when the light-phase narrative material is identical to the last successful emitted material, while preserving phase signal recording, deep promotion behavior, existing exact duplicate cleanup, and the #70737 cron decoupling shape.

## Risks / Open Questions
