## Summary

Keep open. Current main still builds light dreaming from a fixed recent lookback and appends Dream Diary narratives without body-level duplicate suppression, while the provided GitHub context shows open PR #70332 is already targeting this gap and should be reviewed instead of closing the issue early.

## What This Changes

The issue reports repeated light-phase Dream Diary work-summary blocks across dreaming cycles and asks for incremental or duplicate-suppressed light output.

## Best Possible Solution

Land or rework a targeted `memory-core` change that suppresses repeated automatic light Dream Diary material across repeated sweeps, preferably with a stored phase input/body fingerprint or append-time body dedupe, while preserving genuinely new snippets and existing phase signal recording. Add regression coverage for repeated light sweeps producing one diary narrative and for new input still appending.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to seed a daily note or recall entry, trigger light dreaming repeatedly with the same lookback window and deterministic narrative output, then inspect that the same candidate can be emitted again as a new timestamped diary entry; current tests already assert the repeated light candidate path.

## Solution Assessment

Unclear. The issue's `lastEmittedAt` sketch is plausible, but a body or input fingerprint in the Dream Diary append path may be safer; open PR #70332 should be reviewed to decide whether its body-dedupe approach is the right narrow fix.
