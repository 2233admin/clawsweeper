## Summary

Keep this PR open, but do not merge the current patch as-is. Current main still has the reported leak because `agentRunStarts` only drains on terminal lifecycle events, but the PR's age-only sweep can delete start timestamps for still-active runs longer than 10 minutes, while current terminal lifecycle producers and tests allow `end`/`error` events without `startedAt`.

## What This Changes

The PR adds a TTL sweep in `pruneAgentRunCache()` to delete `agentRunStarts` entries older than `AGENT_RUN_CACHE_TTL_MS`, mirroring the existing `agentRunCache` sweep.

## Best Possible Solution

Revise the gateway lifecycle cache cleanup so orphaned `agentRunStarts` entries are eventually bounded while preserving `startedAt` for active runs until their terminal snapshot is recorded, with a focused regression test for both an orphaned start and a long-running start/end sequence.

## Reproduction Assessment

Yes. A source-level reproduction is clear: emit a lifecycle `start`, let the new TTL sweep run after 10 minutes, then emit a lifecycle `end` without `startedAt`; the patched code loses the fallback timestamp even though current tests and producers rely on that fallback.

## Solution Assessment

No. The PR targets the right leak, but an age-only TTL inside `pruneAgentRunCache()` is not the best fix because it cannot distinguish orphaned starts from active long-running runs; the safer path is active-aware cleanup or a retention policy tied to run liveness/maximum runtime plus tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve starts for active long-running runs:** `src/gateway/server-methods/agent-job.ts:39-42`
  - body: This sweep deletes `agentRunStarts` entries once their start timestamp is older than the 10-minute cache TTL, even if the run is still active. `createSnapshotFromLifecycleEvent()` uses that map as the fallback when terminal lifecycle events omit `startedAt`, and the embedded lifecycle handler emits terminal `end`/`error` events without `startedAt`. Since agent runs can legitimately last far longer than 10 minutes, `agent.wait` can return terminal snapshots missing `startedAt` for active long-running runs. Please make this cleanup active-aware or otherwise preserve start timestamps until the run terminalizes.
  - confidence: 0.94
