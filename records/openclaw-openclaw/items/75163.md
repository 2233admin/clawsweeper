## Summary

Keep open. Current main still lacks the provider-qualified alias resolution described by the report, and open PR #75198 explicitly references and implements this narrow fix but has not merged.

## What This Changes

The issue requests that Control UI/TUI mid-session model switches resolve display aliases such as `lmstudio-moe/MoE` to their configured canonical provider/model IDs before persisting the session override.

## Best Possible Solution

Land PR #75198 or an equivalent narrow shared-resolver fix that canonicalizes `provider/Alias` only when the provider matches the configured alias target, with resolver and `sessions.patch` regression tests proving the persisted override is the canonical provider/model ID.

## Reproduction Assessment

Yes. A high-confidence code-path reproduction is to configure `agents.defaults.models["lmstudio-moe/qwen3.6-35b-a3b"].alias = "MoE"`, send `sessions.patch` with `model: "lmstudio-moe/MoE"`, and observe current main treat the suffix as raw model `MoE` rather than the canonical configured ID.

## Solution Assessment

Yes for the linked shared-resolver direction. A UI-only workaround would leave other `sessions.patch`, `/model`, and tool-driven session switch callers with the same alias gap.
