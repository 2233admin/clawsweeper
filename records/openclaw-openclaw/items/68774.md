## Summary

Keep open. Current main still has both reported gaps: the dreaming contamination detector only accepts `Candidate:` or `Reflections:` at the start after prefix stripping, and promotion rehydration returns relocated daily-memory ranges without checking whether they land inside managed dreaming fences. The related #66852 fix shipped a broader self-ingestion guard, but it does not cover this inline-metadata-before-marker or rehydration-fence case; the live PR remains open, clean, and narrowly scoped.

## What This Changes

The PR broadens memory-core's dreaming contamination detector, adds a dreaming-fence overlap guard during short-term promotion rehydration, and adds focused regression coverage in `short-term-promotion.test.ts`.

## Best Possible Solution

Review and land this focused memory-core fix, or an equivalent patch, so staged dreaming metadata and any rehydrated ranges inside `openclaw:dreaming:*` fences cannot reach `MEMORY.md`. Keep any per-file caching or broader provenance redesign as separate follow-up work.
