## Summary

Keep this draft PR open. The proposed backstop is useful in shape, but the current diff appears to change existing WebChat chat.send timeout semantics by hard-capping ordinary omitted-timeout sends to 30 seconds, which would regress long-running WebChat agent turns.

## What This Changes

The PR adds a WebChat-only chat.send terminal timeout backstop, creates missing transcript files before eager user transcript updates, and adds gateway regression coverage for the timeout path.

## Best Possible Solution

Keep the terminal backstop, but derive its deadline from the same resolved chat.send timeout contract used by the abort controller, including coverage for omitted timeoutMs and explicit timeoutMs: 0 before this lands.

## Reproduction Assessment

Yes. A focused regression can mock a WebChat chat.send whose dispatch runs longer than 30 seconds with omitted timeoutMs or timeoutMs: 0; the PR logic would emit a timeout before current main's resolved agent timeout would expire.

## Solution Assessment

No. The transcript-file guard and abort-backed terminal path are reasonable, but the fixed 30-second default is not the narrowest maintainable fix because it bypasses the existing gateway timeout semantics.
