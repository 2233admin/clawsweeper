## Summary

Keep this PR open. The underlying dashboard chat-history payload bloat is still present on current main, but the branch is no longer in the right layer: current history, SSE history, and live display all use the shared chat display projector, while this PR changes the older server-methods sanitizer path.

## What This Changes

The PR branch adds gateway chat.history redaction for inline image data payloads in content blocks, accumulates omitted byte counts, and adds regression tests for image source payloads and malformed multi-payload blocks.

## Best Possible Solution

Port the redaction into src/gateway/chat-display-projection.ts, preserve accumulated omitted-byte accounting, cover image source.data, image data-URL url, OpenAI-style image_url.url, and malformed multi-payload blocks with projection/history/live-display tests, and leave durable transcript storage changes to #1210.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project a transcript message containing content[].source.data on an image block; current main's shared projector leaves that payload intact, and chat.history plus session.message use that projector.

## Solution Assessment

No. The PR's intent and accounting fix are good, but the implementation should move to the shared display projector so all current display surfaces use one sanitizer.
