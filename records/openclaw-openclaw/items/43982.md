## Summary

Keep this PR open. Current main still does not implement the PR's intended targeted chat-history sanitizer for inline image payload shapes such as image `source.data`, image data-URL `url`, or OpenAI-style `image_url.url`. Some newer webchat media paths now avoid persisting base64 by using managed media, but that is not equivalent to sanitizing legacy or malformed transcript content blocks returned through `chat.history`.

## Best Possible Solution

Keep this PR open for maintainer review or contributor follow-up. The best path is to port the inline-image payload sanitizer into `src/gateway/chat-display-projection.ts`, preserve accumulated omitted-byte accounting, and add focused projection/history tests for image `source.data`, image data-URL `url`, OpenAI-style `image_url.url`, and malformed multi-payload image blocks.
