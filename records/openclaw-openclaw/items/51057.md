## Summary

Keep this issue open. Current main still has the same product gap described in the previous ClawSweeper review: timeout-shaped failures can rotate profiles and eventually reach model fallback, but pure transport timeouts are still deliberately excluded from auth-profile failure persistence, and the requested timeout-specific failover budget/retry cap/general per-agent run timeout is not present.

## Best Possible Solution

Keep this open for a focused failover policy change: add a model-scoped short-lived timeout circuit breaker or bounded primary-provider failover budget that counts consecutive profile timeouts without treating every isolated transport stall as permanent auth failure. Add regression coverage for multi-profile timeout sequences reaching fallback quickly, single timeout recovery, external aborts, and cooldown scope.

## Risks / Open Questions

- Persisting every transport timeout as profile-wide auth failure could incorrectly poison shared auth health for isolated network stalls, provider stream hiccups, or user/external aborts.
- A timeout fix should distinguish pure transport stalls from rate-limit-shaped timeout text and should probably be short-lived, model-scoped, and tested across multi-profile fallback, single-timeout recovery, and abort paths.
