## Summary

Keep this PR open for maintainer review. Current main still has the Mattermost draft-preview bug the PR targets: the plugin schema does not accept the canonical streaming object and the inbound monitor always creates/wires draft preview streaming. The proposed patch is scoped to the Mattermost plugin, uses the shared channel streaming helper, adds focused schema/runtime tests, and I did not find a blocking correctness or security issue in the diff.

## What This Changes

The PR adds Mattermost support for the canonical streaming config object and gates draft-preview callbacks/finalization so `streaming.mode: "off"` falls back to normal final delivery.

## Best Possible Solution

Land a narrow Mattermost-owned fix that accepts the canonical streaming object, uses the shared SDK streaming resolver, disables draft preview cleanly for `mode: "off"`, and keeps broader notification UX or scalar-doc compatibility questions tracked separately under the related Mattermost streaming issues.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the reported configuration/runtime gap: the strict Mattermost schema has no `streaming` key, and inbound delivery always creates and wires the draft preview path.

## Solution Assessment

Yes for this PR's scoped bug fix. The shared helper plus Mattermost-local runtime gate is the narrow maintainable path; the only safer follow-up is to reconcile docs or legacy/scalar compatibility separately rather than expanding this patch without maintainer direction.
