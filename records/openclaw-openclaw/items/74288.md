## What This Changes

The PR changes `src/cli/capability-cli.ts` and `src/cli/capability-cli.test.ts` to make empty image-description and audio-transcription results report a missing configured provider differently from a configured provider returning no text.

## Best Possible Solution

Ship a narrow provider-aware infer CLI error path that separates missing media-understanding provider configuration from configured-provider empty output across image describe, describe-many, and audio transcribe, with focused tests and wording aligned with the rest of `openclaw infer`.
