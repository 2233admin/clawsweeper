## Summary

This PR should stay open for normal maintainer review. Current main still has the misleading empty-result errors for `infer image describe` and `infer audio transcribe`, and this open contributor PR is the active implementation attempt for the linked bug rather than something current main already supersedes.

## What This Changes

This PR adds provider-configuration checks to the CLI image-description and audio-transcription empty-result paths and extends `src/cli/capability-cli.test.ts` with no-provider cases.

## Best Possible Solution

Keep this PR open and review it against #73569. The best path is a narrow CLI fix that distinguishes no configured media-understanding provider from a configured provider returning empty text, covers image describe, image describe-many, and audio transcribe through the shared helper where possible, keeps provider-specific failures surfaced from the runtime, and adds targeted tests plus a changelog entry if maintainers treat this as user-facing.
