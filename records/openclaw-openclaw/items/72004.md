## Summary

Keep this PR open. Current main still appends active-run `final` and `aborted` assistant messages to `chatMessages` before clearing `chatStream`, while the render path still passes both committed messages and a live stream into `buildChatItems`. The PR remains a focused, green implementation candidate for the linked Control UI duplicate-render bug; it should be reviewed and landed, not closed as obsolete.

## Best Possible Solution

Review and land this PR, or an equivalent narrow patch, as the focused fix for #71992. The implementation should snapshot streamed fallback text, clear `chatStream`/`chatRunId`/`chatStreamStartedAt` before mutating `chatMessages` in active-run `final` and `aborted` branches, keep the order-sensitive regression test, and retain the changelog attribution.
