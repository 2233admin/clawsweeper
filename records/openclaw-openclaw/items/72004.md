## Summary

Keep this PR open. Current main still has the stream/finalization ordering the PR targets: `handleChatEvent` appends final and aborted assistant messages to `state.chatMessages` before clearing `state.chatStream`, while the renderer can build chat items from both committed history and a non-null live stream. The linked bug #71992 is open, the prior equivalent PR #34164 is closed unmerged, and the current PR is a focused three-file patch with no security-sensitive surface changes.

## Best Possible Solution

Keep this PR open and review it as the focused candidate fix for #71992. The best path is to land this reorder, or an equivalent patch, that captures streamed fallback text, clears `chatStream`/`chatRunId`/`chatStreamStartedAt` before mutating `chatMessages` in the active-run `final` and `aborted` branches, and keeps the regression test plus changelog attribution.
