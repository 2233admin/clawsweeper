## Summary

Keep open for maintainer review and landing. Current main still has the Control UI terminal-event ordering race, the PR is a narrow fix with focused regression coverage, and I found no blocking functional or security findings.

## What This Changes

This PR snapshots streamed fallback text, clears Control UI chat stream/run state before appending final or aborted assistant messages, and adds a regression test plus changelog entry.

## Best Possible Solution

Review and land this PR, or an equivalent narrow patch, in the Control UI chat controller. The desired end state is to preserve foreign-run terminal behavior, snapshot streamed fallback text, clear active stream/run state before committing terminal assistant messages, and keep order-sensitive regression coverage plus user-facing changelog credit.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create an active ChatState with a non-null chatRunId and chatStream, dispatch a final or aborted payload, and observe current main mutates chatMessages while chatStream is still non-null; buildChatItems can then render both the committed message and live stream item.

## Solution Assessment

Yes. The PR’s local reorder is the narrowest maintainable fix for this Lit batching race because it changes only intermediate mutation order while preserving final state and streamed fallback semantics; broader transcript or delivery dedupe work would not address this specific client render window.
