## Summary

Current main still does not implement the PR's intended behavior. `isCacheTtlEligibleProvider` still returns true only through provider plugin overrides, Anthropic-family/KiloCode Anthropic refs, and Google prompt-cache eligibility; there is no generic `modelApi === "openai-completions"` branch. The PR remains a focused source/test change, but maintainers should decide whether broad API-type eligibility is correct or whether this should stay provider/compat scoped.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, land a rebased version that adds the generic `modelApi === "openai-completions"` eligibility branch, positive tests for direct OpenAI-compatible providers such as OpenAI/DeepSeek/local-compatible configs, and a small clarification to the existing rejection test so provider-plugin overrides like OpenRouter remain understandable. Avoid changing provider plugin contracts, prompt-cache-key compatibility behavior, or unrelated pruning defaults in this PR.
