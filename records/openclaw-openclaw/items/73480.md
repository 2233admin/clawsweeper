## Summary

Keep open. Current main still has the Matrix recovery-key, backup usability, and full identity-trust split, still lacks a merged `/keys/upload` one-time-key collision mitigation, and the relevant Matrix PRs remain active work rather than shipped fixes.

## What This Changes

The issue reports Matrix E2EE recovery failing when recovery-key verification reports `Recovery key accepted: no` with `Backup usable: yes`, and `matrix encryption setup` fails on a Matrix `/keys/upload` one-time-key collision.

## Best Possible Solution

Keep this issue open as the user-facing Matrix E2EE recovery regression until maintainers land or reject the focused MAS reset and OTK collision fixes, then re-run the reported recovery-key verification and encryption setup flow with targeted unit coverage plus a live Matrix smoke.
