## Summary

Current main still arms and uses a forced IPv4 fallback for Telegram direct transports even when `channels.telegram.network` explicitly sets IPv6-friendly values. The issue also has an open implementation candidate, #42059, so this should stay open until that fix or an equivalent narrow patch lands.

## Best Possible Solution

Keep this issue open. Review and land #42059 if it is still the right patch, or apply an equivalent narrow fix in `extensions/telegram/src/fetch.ts` so sticky IPv4 fallback is not armed when `channels.telegram.network.autoSelectFamily` or `dnsResultOrder` was explicitly configured for the account. Add regression coverage for both API fetch and media download paths using `autoSelectFamily: true` plus `dnsResultOrder: "verbatim"`.
