## Summary

Keep open. Current main still lacks the PR’s intended OpenAI WebSocket first-response watchdog: the WS path sends the request and then waits on abort, close, or message handlers, while the generic idle timeout does not reset/degrade the WS session or fall back to HTTP/SSE. The branch appears useful, but it needs to be ported to the current `resolveEmbeddedAgentStreamFn` construction path before it can merge.

## What This Changes

The PR branch adds an OpenAI Responses WebSocket first-response watchdog, auto-mode HTTP fallback on silent first-response timeout, forced-websocket timeout errors, and regression tests for timeout, fallback, signal, and session-state edge cases.

## Best Possible Solution

Land a narrow OpenAI WS transport fix on current main: add a first-server-event watchdog to the WS request path, wire its budget through the current embedded stream-resolution helper, keep auto-mode recovery scoped to WS reset/degrade plus HTTP/SSE fallback, preserve forced-websocket errors, and cover the timeout, disabled-watchdog, bookkeeping/error sentinel, late completion, single-start, and abort-signal cases in tests.
