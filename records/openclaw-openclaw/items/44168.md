## Summary

Keep open. Current main has generic model idle-timeout protection and WebSocket fallback for connect/send/runtime errors, but it does not implement this PR's central OpenAI WebSocket first-response watchdog, silent first-event timeout recovery, auto-mode HTTP fallback for that timeout, forced-websocket timeout error, or the focused regression coverage. The prior discussion is positive, so the right cleanup outcome is maintainer review and rebase rather than closure.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, wire a derived first-response timeout through `resolveEmbeddedAgentStreamFn`, preserve the existing generic idle timeout, and keep/add focused tests for auto HTTP fallback, forced-websocket timeout errors, late WebSocket completion isolation, duplicate `start` prevention, and resolved abort-signal forwarding.
