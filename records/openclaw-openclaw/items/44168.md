## Summary

Keep open. Current main at 20e21173715e still lacks this PR’s OpenAI WebSocket first-response watchdog: the WS stream options expose only manager options and signal, the post-send wait has no first-event timer, and auto fallback is still driven by send/connect/close/error failures rather than a connected socket that never emits an initial server event.

## What This Changes

The PR branch adds an OpenAI Responses WebSocket first-response timeout, auto-mode HTTP fallback, forced-websocket timeout errors, and regression tests for timeout, fallback, signal, and session-state edge cases.

## Best Possible Solution

Keep this PR open and adapt it to current main. The best implementation is a narrowly scoped OpenAI WS first-response watchdog wired through `resolveEmbeddedAgentStreamFn`, with auto mode resetting/degrading the WS session and falling back to HTTP/SSE, forced websocket mode surfacing a timeout, and regression tests for disabled watchdogs, first server/bookkeeping events, late WS completion isolation, single `start` delivery, and abort-signal forwarding.
