## Summary

Keep open. Current main still has the voice-call manifest and docs declaring SecretRef support for `plugins.entries.voice-call.config.twilio.authToken`, but the runtime config schema only accepts a string and plugin registration parses through that schema before SecretRef resolution can help. This draft PR is a narrow, relevant implementation candidate rather than obsolete work.

## Best Possible Solution

Review and land this focused PR after final validation, or apply the same fix on main: make `twilio.authToken` a `SecretInput`, use `hasConfiguredSecretInput` for setup validation, resolve it strictly before constructing `TwilioProvider`, and keep the added regression coverage for config parsing and plugin registration.
