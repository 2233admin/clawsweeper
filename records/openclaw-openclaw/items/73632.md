## Summary

Keep this PR open. Current main still has the voice-call SecretRef mismatch this PR targets: `twilio.authToken` is advertised as SecretRef-capable in the manifest/docs, but the runtime zod schema still accepts only strings during plugin registration, and the Google realtime API-key path is still absent from bundled voice-call SecretInput collection.

## Best Possible Solution

Keep this PR open and review or land it, or apply the same focused fix on main: widen the voice-call Twilio auth-token schema to the shared SecretInput contract, treat configured SecretRefs as present during setup validation, strictly resolve to a string before constructing `TwilioProvider`, add `realtime.providers.*.apiKey` to bundled voice-call SecretInput metadata/docs/tests, and keep the targeted regression coverage.
