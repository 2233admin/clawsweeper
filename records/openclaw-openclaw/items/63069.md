## Summary

Keep this issue open. Current main still resolves shell completion profiles through hardcoded zsh/fish paths and a bash check/install mismatch, and the same-author implementation candidate #63082 is open and unmerged with closing syntax for this issue.

## What This Changes

Shell completion installation and status checks should write to the profile file actually loaded by the user shell, including zsh `ZDOTDIR`, fish `XDG_CONFIG_HOME`, and a consistent bash `.bashrc`/`.bash_profile` fallback.

## Best Possible Solution

Keep the issue open and review, land, or replace #63082 with an equivalent narrow fix that centralizes profile path resolution, uses that resolver consistently for check/install/update/doctor/setup hints, and adds focused unit coverage for zsh `ZDOTDIR`, fish `XDG_CONFIG_HOME`, and bash fallback parity.
