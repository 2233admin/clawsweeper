## Summary

Keep this issue open. Current main still implements steer as streaming/model-boundary injection with followup fallback, not the requested pause-or-abort, inject, and resume soft-steer lifecycle. The issue is also paired with open PRs that use closing syntax, plus existing maintainer review context points to a protected replacement PR for the same active non-streaming steer gap, so the right cleanup path is maintainer review of those PRs rather than closing this tracker.

## What This Changes

Add soft-steer behavior that can pause or abort active generation, inject the user's new message into context, and resume with awareness of that input instead of waiting for the next tool or model boundary.

## Best Possible Solution

Keep this issue open as the product/runtime tracker. Review and either land or close #64758/#2307 and the protected maintainer replacement #73365. A complete fix should define soft-steer lifecycle semantics across streaming generation, active non-streaming/tool execution, backend failure, fallback, and abort/resume boundaries; replace streaming-only gates with a safe lifecycle predicate; keep gateway hard-interrupt behavior distinct; update queue/agent docs; and add regression coverage for streaming steer, active non-streaming steering, fallback behavior, and abort/resume boundaries.
