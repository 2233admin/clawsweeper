## Summary

Keep this issue open because current main still implements steer as a streaming/model-boundary path with follow-up fallback rather than the requested pause-or-abort, inject, and resume lifecycle. The issue is also paired with open implementation work, including closing-syntax PR #64758 and maintainer-labeled replacement PR #73365, so it should remain open until the canonical PR path lands or is explicitly redirected.

## What This Changes

Add a soft-steer queue behavior that can interrupt or pause an active generation, inject the user's new message into context immediately, and resume with awareness of that message.

## Best Possible Solution

Keep this as the product/runtime tracker until #73365 or a narrower successor lands, with documented and tested lifecycle semantics for streaming, active non-streaming/tool execution, abort or resume behavior, backend failure, and follow-up fallback.
