## Summary

Keep this PR open. Current main still has the sweep-backed reminder false positive: the reminder guard only recognizes same-turn cron.add success or an enabled cron job with the same sessionKey, while the linked bug describes reminders persisted to agent-owned storage and delivered later by a separate isolated sweep cron. The PR is not ready as-is because its head commit contains raw U+0008 bytes in the regex and lacks the isolated-session filter/negative test called out in review, but it remains the canonical implementation candidate for #52528.

## Best Possible Solution

Keep this PR open and update it rather than close it. The best path is to fix the regex byte issue, require enabled isolated agentTurn sweep jobs for heuristic suppression, add a negative non-reminder isolated-cron test, then run the targeted reminder-guard runReplyAgent test. If maintainers prefer a broader unified fix, compare with the related #43404 line of work before landing so the same guard behavior is not split across competing PRs.
