## Summary

Keep open. Current main still only suppresses the reminder guard for same-session enabled cron jobs, while this PR's proposed sweep-backed suppression is useful but currently incorrect because the regex contains U+0008 control bytes and the heuristic is broader than the isolated-cron contract. The PR also overlaps with open reminder-guard work in #43404 and #47663, so the next step is author or maintainer consolidation rather than cleanup closure.

## What This Changes

The PR adds a reminder-sweep-like cron-job heuristic to the reminder guard and a runReplyAgent regression test so an existing sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Preserve the existing same-session cron suppression and add a narrow path for enabled isolated agentTurn reminder-sweep jobs, with the regex bytes fixed, the isolated boundary enforced, positive and negative regression coverage, and coordination with #43404 and #47663.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use word-boundary escapes for sweep detection:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:36`
  - body: The added regex contains raw U+0008 bytes around the alternation instead of `\b` word-boundary escapes. Normal cron names/messages such as `family-reminder sweep` do not contain backspaces, so the new sweep-cron suppression never matches and the guard note can still be appended.
  - confidence: 0.97
- **[P2] Limit sweep suppression to isolated cron jobs:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:24-26`
  - body: The helper applies the reminder/sweep heuristic to every enabled `agentTurn` job. A main, custom, or other-session cron whose prompt mentions reminders or sweeps could suppress the warning for an unrelated reply, which is broader than the PR's stated isolated-cron contract.
  - confidence: 0.86
