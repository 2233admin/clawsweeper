## Summary

Keep open. Current main still only suppresses the unscheduled-reminder note for successful cron.add calls or enabled jobs sharing the current session key, so the sweep-backed reminder behavior is not implemented there. The PR is useful source work, but its current patch is incorrect: the new regex contains literal backspace bytes and the new heuristic is broader than the stated isolated-cron contract.

## What This Changes

The PR adds a reminder-sweep-like cron heuristic to `hasSessionRelatedCronJobs` plus a runReplyAgent regression test so an existing reminder sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Preserve the existing same-session cron suppression, add a narrow path for enabled isolated agentTurn reminder-sweep jobs using real `\b` word-boundary escapes, and cover both positive sweep-backed and negative non-reminder isolated-cron cases while coordinating with the adjacent reminder-guard PRs.

## Reproduction Assessment

Yes. The PR diff itself provides the reproducer: the added sweep job text should match, but a direct Node probe of the PR-equivalent backspace regex returns false; current main also still appends the note unless the cron job shares the current session key.

## Solution Assessment

No. The direction is plausible, but this patch is not the best current solution because the regex is ineffective and the heuristic is broader than the isolated-cron contract; the safer path is to fix the regex, require `sessionTarget === "isolated"`, and add negative coverage.
