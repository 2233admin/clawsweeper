## Summary

Keep open. Current main still has the same-session-only reminder guard, so the PR is not obsolete, but the proposed patch is not merge-ready because the sweep-detection regex contains literal backspace characters and the new suppression heuristic is broader than the stated isolated-cron contract.

## What This Changes

The PR adds a reminder-sweep-like cron heuristic to `hasSessionRelatedCronJobs` and a runReplyAgent regression test so an existing sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Preserve the existing same-session cron suppression and add a narrow, documented path for enabled isolated `agentTurn` reminder-sweep jobs, with the regex fixed, the `sessionTarget` filter enforced, and positive plus negative regression coverage coordinated with the overlapping reminder-guard PRs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use real word-boundary escapes for sweep detection:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:35`
  - body: The added regex contains literal U+0008 control characters around the alternation instead of `\b` word-boundary escapes, so normal cron names/messages such as `family-reminder sweep` will not match and the new suppression path remains inactive.
  - confidence: 0.97
- **[P2] Restrict sweep suppression to isolated cron jobs:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:24-25`
  - body: The helper applies the reminder/sweep heuristic to every enabled `agentTurn` job, not only isolated jobs. A main, custom, or other-session cron whose prompt mentions reminders or sweeps could suppress the warning for an unrelated reply.
  - confidence: 0.86
