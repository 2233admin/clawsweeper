## Summary

Keep this PR open. Current main still does not implement the PR's central backend contract: `sessions.list` has no `lastHash` parameter, no result `hash`, no `{ unchanged: true }` short-circuit, and no cache invalidation counters for transcript or subagent mutations. The PR is an active split from broader #55959 work and remains a plausible implementation candidate that needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to review or replace the backend cache contract on current main: add the `sessions.list` `hash`/`lastHash`/`{ unchanged: true }` protocol contract, implement conservative cache-key invalidation for config/session/transcript/subagent state, preserve correct config.patch source-diff/SecretRef behavior, and land focused regression tests. Control UI and Apple client adoption should stay separate follow-ups.
