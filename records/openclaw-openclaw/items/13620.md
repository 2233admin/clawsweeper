## Summary

Keep this issue open. Current main still lacks outbound Telegram media-group/album sending: the Telegram plugin exposes only singular `mediaUrl` direct sends, reply delivery loops through `mediaUrls` one item at a time, outbound payload delivery uses the shared sequential media helper, and docs/tool schema still describe single-media sends. There is also an open external PR with closing syntax for this issue, so maintainer review/port/merge or closure of that implementation should happen before this issue is closed.

## Best Possible Solution

Keep this issue open and have a maintainer review, port, land, or close the open external implementation PR. The best fix belongs in the bundled Telegram plugin and should add a guarded `sendMediaGroup` path for 2-10 groupable photos/videos across direct Telegram message actions, reply delivery, and outbound payload delivery, with focused tests/docs and preserved single-media/non-groupable fallbacks.
