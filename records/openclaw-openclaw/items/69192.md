## Summary

Keep open. Current main still returns multiple hook 4xx responses without a hook journal warning, including the invalid JSON path and mapped-hook validation rejects described by the reporter. The direct prior fix PR #69267 was closed unmerged, so it remains useful prior art but not an implemented fix.

## Best Possible Solution

Implement a small, centralized hook rejection logging helper in `src/gateway/server/hooks-request-handler.ts` and route every non-2xx hook response through it. The helper should emit one sanitized warn line with status, stable reason code, safe path/content/body metadata, and inherited diagnostic trace context. Add focused tests for invalid JSON, timeout/too-large bodies, auth/path/method rejects, direct wake/agent validation, mapping validation, policy/session rejects, and the ambiguous mapping-message branch. Keep response compatibility unless intentionally improving error specificity with tests and changelog credit.
