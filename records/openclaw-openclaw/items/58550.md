## Summary

Keep open. Current main still uses the JSON session store and does not include the SQLite-backed session store, `session.storeType`, or migration CLI proposed by PR #58550. The linked performance problem remains plausibly actionable, but the PR is not merge-ready because the provided review thread still has later unresolved correctness concerns around SQLite write fallback, migration failure handling, migration memoization, and legacy-store migrations.

## Best Possible Solution

Keep the PR open or convert it into a maintainer-guided session-storage design thread. The best path is to fix the unresolved SQLite correctness issues, rebase onto current main's existing cache/maintenance/session-list behavior, and add focused tests for migration failure, write failure, legacy migration import, and retry-after-failure semantics before considering merge.
