## Summary

Current main still does not implement the PR's intended direct tool-result reply behavior. The use case is plausible and the latest PR diff appears to address the earlier Greptile nits, but the change alters core agent-loop delivery and abort semantics, relies on a new tool-result trust contract, and still mixes unrelated gateway/script test updates, so it should remain open for maintainer product/security review rather than conservative cleanup closure.

## What This Changes

The PR adds a `directReply` tool-result flag and helper constructors, forwards an `abortRun` callback into embedded subscription handlers, emits successful direct-reply tool output as block replies while skipping further inference, adds handler tests and a changelog entry, and also adjusts unrelated gateway/script tests.

## Best Possible Solution

Keep this PR open for maintainer product/security review. If OpenClaw wants the capability, land it as a generic typed tool-result direct-delivery contract with explicit trust rules, preserved sanitized text/media filtering, a typed abort/lifecycle reason, integration proof that no follow-up model pass runs, docs/changelog coverage, and split or justified unrelated gateway/script test changes.
