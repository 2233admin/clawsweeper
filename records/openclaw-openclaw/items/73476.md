## Summary

Current main still does not implement this PR's direct tool-result reply behavior. There is no `directReply` / `direct_reply` contract, no helper constructors, no subscription abort seam, and the embedded tool-end handler still emits normal tool output and then runs `after_tool_call` hooks. The PR remains a live implementation candidate, but the author’s external-sandbox scenario needs maintainer product/security review because it lets successful tool output bypass final LLM synthesis.

## Best Possible Solution

Keep this PR open for maintainer review. If OpenClaw wants this capability, land it as a generic typed tool-result contract with explicit trust policy, safe media filtering through the existing block-reply path, a typed abort reason, focused embedded-run tests proving direct delivery without a follow-up model pass, and changelog/docs cleanup. If maintainers prefer a different product direction, close the PR only after that decision is explicit.
