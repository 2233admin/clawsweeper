## Summary

Current main still does not implement this PR's intended direct tool-result reply behavior. The PR remains a live implementation candidate, and the author clarified the target use case is an external high-value skill service whose returned content should be delivered without another model pass. Because that changes a product/security trust boundary and overlaps with the upstream tool-result termination contract, it should stay open for explicit maintainer review rather than automated cleanup or closure.

## Best Possible Solution

Keep this PR open for explicit maintainer product/security review. If OpenClaw wants this capability, land it as a generic typed tool-result contract with a clear trust policy, documented relationship to upstream AgentToolResult.terminate, existing media filtering, typed abort reason, focused tests proving direct delivery without a follow-up model pass, and docs/changelog coverage. Consider splitting the unrelated CI assertion-test updates unless they are required for this branch to validate.
