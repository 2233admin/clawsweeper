## Summary

Current main still does not implement this PR's intended direct tool-result reply behavior. There is no `directReply`/`direct_reply` contract or helper, the embedded tool-end handler still emits normal tool output and then runs `after_tool_call`, and the runner's local `abortRun` is not exposed through embedded subscription params. The PR remains a live implementation candidate, but it changes a product/security boundary by letting successful tool output bypass final LLM synthesis, so it needs explicit maintainer review rather than automated cleanup or closure.

## Best Possible Solution

Keep this PR open for explicit maintainer product/security review. If OpenClaw wants this capability, land it as a generic typed tool-result contract with a clear trust policy, typed abort reason, existing media filtering, focused tests proving direct delivery without a follow-up model pass, and docs/changelog coverage. If maintainers prefer the normal synthesis path or a different plugin API, close this PR only after that decision is explicit.
