## Summary

Keep this PR open. Current main does not implement a `directReply` tool-result contract or forward an abort seam into tool handlers, and the PR remains a live implementation candidate. Because the proposed behavior lets a tool result become the visible final reply and abort the normal model loop, it needs maintainer product/security review rather than closure or automated repair.

## What This Changes

This PR adds an optional `directReply` flag and helper constructors for tool results, forwards an abort callback into embedded subscription handlers, emits successful direct-reply tool output as a block reply, adds handler tests and a changelog entry, and updates several unrelated auth, gateway, and script tests.

## Best Possible Solution

Keep this PR open for maintainer product/security review. If OpenClaw wants the capability, land it as a typed generic tool-result direct-delivery contract with explicit trust rules, preserved sanitized text/media filtering, a typed abort/lifecycle reason, integration proof that no follow-up model pass runs, docs/changelog coverage, and split or justified unrelated test changes.
