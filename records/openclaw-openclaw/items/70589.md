## Summary

Keep PR #70589 open. Current main still has the bug this PR targets: local agent JSON delivery writes the outbound result envelope through `runtime.log(JSON.stringify(...))`, while the runtime already has a structured stdout JSON writer and the CLI docs promise stdout is reserved for JSON output. The related #62453 is closed and unmerged, so it does not supersede this open PR.

## Best Possible Solution

Keep #70589 open and review it as the active fix for #62440. The best code path is a small change in `src/agents/command/delivery.ts` replacing the local JSON `runtime.log(JSON.stringify(...))` call with `writeRuntimeJson(runtime, buildOutboundResultEnvelope(...))`, plus a colocated regression test asserting `runtime.writeJson` is used for `openclaw agent --local --json`. #62453 can remain closed/unmerged as a duplicate attempt, not as the canonical implementation.
