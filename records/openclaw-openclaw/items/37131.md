## Summary

Keep this issue open. Current main has partial no-LLM direct invocation through HTTP `POST /tools/invoke` and skill `command-dispatch: tool`, but it still does not implement the requested umbrella contract across coding tools, cron `toolCall` payloads, CLI `--tool`/`--args`, and a generic `/tool ...` chat command. Open PR #63919 and open issue #18160 are useful narrower slices, not complete replacements.

## Best Possible Solution

Keep this issue open as the umbrella product/API request. The best path is an explicit maintainer decision: either design one security-reviewed direct-invocation contract that consistently covers HTTP, cron, CLI, and slash-command surfaces, or split the remaining work into canonical narrower trackers while keeping #63919 and #18160 linked as active slices.
