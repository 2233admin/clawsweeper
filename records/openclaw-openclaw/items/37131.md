## Summary

Keep open. Current main at 20e21173715ef488850023a90f72585a01cc379f has partial no-LLM direct invocation through Gateway HTTP `POST /tools/invoke` and skill-owned `command-dispatch: tool`, but the requested umbrella capability remains missing for coding tools on the HTTP resolver, cron direct tool payloads, CLI `--tool`/`--args`, and a generic `/tool ...` chat command. Open PR #63919 and open issue #18160 are useful narrower slices, not replacements for this umbrella request.

## What This Changes

Add a no-LLM direct tool invocation path for explicit tools such as exec/read/write/message/MCP skills across chat slash commands, CLI, and cron payloads.

## Best Possible Solution

Keep this issue open as the umbrella product/API request. The best path is a maintainer-owned, security-reviewed direct-invocation contract that either consistently covers HTTP, cron, CLI, and slash-command surfaces, or deliberately splits the remaining work into canonical narrower trackers while keeping #63919 and #18160 linked as active slices.
