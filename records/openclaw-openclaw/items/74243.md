## Summary

Keep this PR open. Current main still propagates optimizer backend failures through the shared media loader and still throws a generic `Failed to optimize image` after swallowing resize-attempt errors. The PR is a plausible consolidation candidate for the active optimizer-failure cluster, but it is not implemented on main and changes shared media/resource-guard behavior, so it needs normal maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review as the consolidation candidate. If maintainers accept the policy, land a focused fix that falls back only for already-under-cap non-HEIC images, preserves original MIME/content type, keeps pixel/dimension/HEIC failures fail-closed, preserves optimizer cause diagnostics, then close #73424 and #73148 while superseding #73451, #74170, #74228, and #73217 as appropriate. Leave #38260 separate because a native SIGILL crash needs crash-level investigation rather than a JavaScript optimizer fallback.
