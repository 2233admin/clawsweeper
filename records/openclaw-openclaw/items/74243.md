## Summary

Keep open: current main does not already implement the intended behavior, and this is an active implementation PR rather than a cleanup-close candidate. The remaining decision is maintainer review of the shared media fallback policy and guard boundary before merge.

## What This Changes

This PR changes shared media loading to return original under-cap non-HEIC image bytes when optimization fails, preserve MIME and optimizer cause details, keep guard failures closed, and add focused regression tests.

## Best Possible Solution

Review this PR as the consolidation path for the optimizer-failure cluster, land it if the fallback policy and guard treatment are accepted, then close #73424 and #73148 and supersede the narrower fallback/diagnostic PRs. Keep the native libvips SIGILL crash tracked separately in #38260 because a JavaScript fallback cannot handle a hard process crash.
