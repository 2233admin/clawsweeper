## Summary

Keep this PR open. Current main still propagates optimizer backend failures through `loadWebMedia` and still throws a bare `Failed to optimize image` when all JPEG resize attempts fail. The PR is a plausible consolidation candidate for #73424, #73451, #74170, and #74228, but it changes shared media-loader behavior used by the image tool and outbound channels, so it needs normal maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review as the canonical consolidation candidate. If maintainers accept the shared policy, land one focused fix that falls back only for already-under-cap non-HEIC images, preserves original MIME/content type, keeps pixel/dimension/HEIC failures fail-closed, and preserves optimizer cause diagnostics; then close #73424 and supersede the duplicate fallback/diagnostics PRs (#73451, #74170, #74228).
