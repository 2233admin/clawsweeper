## Summary

Keep this PR open. Current main still propagates image optimization failures through the shared media loader, and this PR is a plausible focused fix. The latest PR head also addressed the earlier pixel-limit concern by rethrowing image input guard errors and adding regression coverage, but the change still alters shared `loadWebMedia` behavior used by multiple channel/plugin surfaces, so it needs normal maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide the shared media-loader policy for optimizer/backend failures, then either land this PR or consolidate it with the #73424 cluster: under-cap non-HEIC images may fall back to the original, while HEIC conversion, pixel/dimension guards, and over-cap media still fail closed. The final patch should keep regression coverage for optimizer-backend failure, over-cap rejection, and the byte-small pixel-oversized guard.
