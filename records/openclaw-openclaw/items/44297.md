## Summary

Keep open. The prior ClawSweeper review is still accurate on current main: Slack external arg-menu registration failure remains non-fatal, but the degraded state is still only recorded through verbose logging and is not surfaced through channel health/status, doctor, or docs as a visible operator signal.

## Best Possible Solution

Keep this issue open and implement the signal in the bundled Slack plugin. Preserve the current non-fatal fallback, but publish the degraded external-arg-menu state through a visible runtime warning and/or account status health field, include it in the relevant status/doctor surface if that is the chosen operator path, and add a focused regression test that fails if the fallback is only logged through `logVerbose`. Update `docs/channels/slack.md` if the signal becomes user-facing.
