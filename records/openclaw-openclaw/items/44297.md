## Summary

Keep open. Current main still keeps Slack external arg-menu registration failure non-fatal, but the degraded mode is only logged through `logVerbose`; I did not find a visible runtime warning, account health/status field, doctor diagnostic, or Slack docs note that would satisfy the issue's operator-visibility request.

## Best Possible Solution

Implement this in the bundled Slack plugin: preserve the non-fatal fallback, but surface the degraded external-arg-menu state through a visible runtime warning and preferably an account status/health detail that `openclaw channels status --probe --json` can expose. Add a focused Slack slash monitor test that fails if the fallback only calls `logVerbose`, and update Slack docs if the signal is a user-facing status/troubleshooting surface.
