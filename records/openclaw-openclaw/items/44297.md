## Summary

Current main still keeps Slack external arg-menu registration failure non-fatal, but the degraded mode is only surfaced through `logVerbose`; there is no visible runtime warning, account status/detail, doctor diagnostic, or Slack docs guidance. This remains valid and narrow enough for a focused ClawSweeper repair PR.

## What This Changes

The issue requests that Slack external arg-menu registration fallback remain non-fatal but become visible to operators through warning, health/status, doctor, or diagnostics output.

## Best Possible Solution

Implement a narrow bundled Slack plugin fix: when external arg-menu options registration fails, keep falling back to static menus, emit a visible runtime warning, and publish a scoped degraded status/detail that `openclaw channels status --probe --json` can expose. Add focused tests for the non-fatal fallback plus the visible signal, and update Slack docs/changelog if the signal is user-facing.

## Repair Work Prompt

Implement a narrow bundled Slack plugin fix for the Slack external arg-menu fallback visibility issue. In `extensions/slack/src/monitor/slash.ts`, when `registerSlackMonitorSlashCommands()` catches `app.options()` registration failure for external arg menus, keep the existing non-fatal fallback to static menus, but emit a visible runtime warning and publish a scoped degraded status/detail that `openclaw channels status --probe --json` can expose. Wire status through the Slack monitor/provider path without marking the whole account disconnected or breaking the successful `external_select` path. Avoid leaking sensitive raw error details in public status. Add focused tests proving: registration still succeeds when `app.options()` throws; large arg menus fall back to `static_select`; a visible warning/status detail is emitted; the normal external-select path remains unchanged. Update `docs/channels/slack.md` and `CHANGELOG.md` only if the new signal is user-facing. Do not move Slack-specific behavior into core unless an additive generic status seam is strictly necessary.
