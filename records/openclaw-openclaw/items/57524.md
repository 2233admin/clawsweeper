## Summary

Keep open. Current main still has the sudo/systemd mismatch this PR targets: the unit file path is derived from HOME while systemctl can be routed to SUDO_USER's user scope under sudo. The PR is a plausible fix candidate, but it is not merge-ready because the current head only guards one CLI entrypoint, has unresolved review feedback, and includes unrelated build/admin side effects.

## What This Changes

The PR adds a Linux sudo guard to gateway install, post-install systemd linger/status diagnostics, unit tests for the guard, and a tsdown sourcemap build-config change.

## Best Possible Solution

Land a narrow, reviewed fix that rejects sudo-to-root gateway systemd installs before any unit write across all first-party gateway install paths, gives a clear non-sudo install message plus the separate loginctl linger command, preserves the existing systemctl machine-scope fallback for lifecycle actions, removes unrelated build changes, and adds focused regression tests for the guard and install entrypoints.
