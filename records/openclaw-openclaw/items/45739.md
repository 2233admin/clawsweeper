## Summary

Keep open. Current main still does not implement this PR’s intended ACP parent relay gateway-state fallback: the relay remains local `onAgentEvent` driven, while newer task-completion plumbing is a separate terminal notification path and does not recover missing live assistant output from `chat.history` or terminal state from `agent.wait`.

## Best Possible Solution

Keep this PR open for maintainer review or a current-main rework. If live `streamTo: "parent"` relay backfill is still desired alongside task-completion delivery, implement it in `src/agents/acp-spawn-parent-stream.ts` with scoped `chat.history`/`agent.wait` calls, short timeouts, and regression tests for missing assistant events, missing terminal events, lifecycle-before-transcript persistence, history-first/local-second duplication, and the `primeHistoryBaseline` race. If maintainers decide task-completion delivery replaces live relay backfill, close manually with that product decision rather than as automated implemented-on-main cleanup.
