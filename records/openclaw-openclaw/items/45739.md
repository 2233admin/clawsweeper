## Summary

Keep open. Current main has newer ACP background-task completion handling, but it does not implement this PR’s intended gateway-state fallback inside the ACP parent stream relay. The current relay still depends on local in-process assistant/lifecycle events, and the PR discussion contains unresolved correctness concerns around duplicate/race behavior.

## Best Possible Solution

Keep this PR open for maintainer review or a current-main rework. If live `streamTo: "parent"` relay backfill is still desired alongside task-completion delivery, implement it in `src/agents/acp-spawn-parent-stream.ts` with scoped `chat.history`/`agent.wait` calls, short timeouts, and regression tests for missing assistant events, missing terminal events, lifecycle-before-transcript persistence, history-first/local-second duplication, and the `primeHistoryBaseline` race. If maintainers decide task-completion delivery replaces live relay backfill, close manually with that product decision rather than as an automated implemented-on-main cleanup.
