## Summary

Keep #59299 open. It is a recent non-maintainer bug report with no protected labels, and the core claim is understandable: native Windows Gateway daemon exits when the laptop switches from AC to battery. Current main still manages native Windows gateways through Scheduled Tasks/Startup-folder fallback, and I found no shipped power-transition or battery-setting fix that would justify closing this as implemented or obsolete.

## Best Possible Solution

Keep the issue open for a native Windows maintainer/debug pass. The likely path is to reproduce against current main/latest release with Task Scheduler history, Event Viewer, `openclaw gateway status --json`, Gateway logs, and a stability bundle, then either fix the Scheduled Task definition to allow running/continuing on battery or document the required workaround. If the active Windows scheduled-task rewrite work is revived, consolidate the battery settings there with tests for rendered task settings and uninstall/fallback behavior.
