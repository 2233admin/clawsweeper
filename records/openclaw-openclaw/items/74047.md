## Summary

Keep open. Current main has partial guardrails for gateway service install-path drift: update can reinstall/restart from the updated package, doctor can detect entrypoint mismatch, and Linux doctor intentionally avoids rewriting an active systemd unit. That does not fully resolve this report because the reported failure is a split user/system install plus `/v1/chat/completions` timeouts after repointing to the newer install. An adjacent open PR by the same author covers legacy `clawdbot` unit resolution, but it does not settle the newer-path reliability problem or the supported migration policy.

## Best Possible Solution

Keep this issue open. The best path is to treat the legacy-unit and service-path drift pieces as diagnostics/migration work, review the adjacent legacy-unit PR, and separately collect gateway logs or a minimal reproduction for the newer-install `/v1/chat/completions` timeout before deciding whether the fix belongs in update/doctor, service install metadata, or the OpenAI-compatible Gateway endpoint.
