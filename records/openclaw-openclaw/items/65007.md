## Summary

Keep open. Current main still has the reported Google Chat behavior in all three areas: add-on parsing only handles message payloads, wildcard routed spaces can keep sender policy in allowlist mode, and reply delivery still passes replyToId directly as a Google Chat thread resource. The related #65058 PR is open and partial, so it is useful context but not enough to resolve this issue.

## What This Changes

Fix Google Chat add-on lifecycle payload parsing, wildcard space sender authorization, and reply thread resource handling for space events.

## Best Possible Solution

Fix this in the bundled Google Chat plugin with focused regression coverage: accept add-on message and space lifecycle payload variants, apply the shared sender-scoped group policy behavior for wildcard routed spaces without groupAllowFrom, and normalize Google Chat reply threading so a message resource is not sent where the API expects a thread resource.
