## Summary

Keep this PR open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still does not implement the PR's central changes: a daemon health-only read probe, a longer deadline-bounded restart probe timeout, or strict explicit-auth handling for daemon status URL overrides. The merged #72405 path covers the narrower local-vs-remote probe credential separation, but it does not supersede the remaining behavior in this PR.

## Best Possible Solution

Keep this PR open for maintainer judgment or ask the contributor to rebase and narrow it. The best path is a focused follow-up that adds a health-only probe/detail mode, uses it for daemon status/read probes where appropriate, adds deadline-bounded restart probe timeouts with tests, and explicitly resolves the security review concerns around override URLs, insecure ws:// handling, and device identity/token fallback while preserving #72405's local-vs-remote auth separation.
