## Summary

Keep open. Current main already renders newly generated Linux systemd gateway units with `KillMode=control-group`, and that partial default is present in the v2026.4.26 release tag. The main observability request is still missing: doctor/status do not collect or surface `KillMode`, `TasksCurrent`, or `MemoryCurrent`, and the doctor systemd unit audit only checks network-online ordering and `RestartSec`. The stale label should not close this because the report has concrete reproduction data and the remaining work is actionable.

## Best Possible Solution

Keep this issue open and implement the remaining observability first: extend the Linux systemd runtime/audit path to read `KillMode`, `TasksCurrent`, `MemoryCurrent`, and `MainPID`; surface those values in `openclaw gateway status --deep` and `openclaw status --deep`; add a doctor warning/repair hint when `KillMode=process` combines with suspicious task or cgroup memory counts. Leave the generated-unit default at the existing `control-group` unless maintainers explicitly choose `mixed`.
