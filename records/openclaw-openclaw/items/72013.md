## Summary

Keep open. Current main still lets pending ACP one-shot records with stable ACPX or agent session IDs enter startup identity reconciliation, so the reported gateway warning can still occur for historical terminal one-shot metadata; the focused source PR for this issue is closed unmerged.

## What This Changes

ACP startup should skip or downgrade unresolved terminal one-shot and closed ACP session records during pending identity reconciliation.

## Best Possible Solution

Add a narrow guard inside AcpSessionManager.reconcilePendingSessionIdentities before checked increments and runtime work so terminal one-shot records are skipped, while preserving normal persistent pending-identity reconciliation and adding focused regression tests.

## Reproduction Assessment

Yes. A focused manager unit fixture can return a persisted ACP record with mode:"oneshot", identity.state:"pending", and identity.acpxSessionId or identity.agentSessionId; current main passes the pending-plus-stable predicate and counts it for startup reconciliation.

## Solution Assessment

Yes. The startup skip is the narrowest maintainable fix for historical terminal one-shot metadata; the existing post-turn one-shot identity resolution helps future successful runs but does not prevent already-persisted pending one-shot records from being retried on every restart.
