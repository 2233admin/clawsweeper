## Summary

Keep open. Current main still lets persisted ACP one-shot records with pending identities and stable ACPX ids enter startup identity reconciliation, so the reported gateway warning remains possible for stale terminal metadata. The focused related PR #72023 is closed unmerged, and the shipped/current partial fixes do not add the startup-time one-shot guard.

## What This Changes

Suppress ACP startup identity-reconcile warnings for terminal one-shot or closed ACP session metadata with unresolved pending identities.

## Best Possible Solution

Add a narrow guard in `AcpSessionManager.reconcilePendingSessionIdentities` that skips persisted one-shot ACP records before `checked` is incremented or the runtime is touched, while preserving normal persistent pending-identity reconciliation. Add focused regression tests for pending one-shot records with stable ACPX ids and handle any legacy closed-state metadata deliberately if it is still possible in stored data.

## Reproduction Assessment

Yes. A persisted ACP session with `acp.mode: "oneshot"`, `identity.state: "pending"`, and `identity.acpxSessionId` currently passes the startup reconcile predicate and increments `checked`, which is enough to drive the gateway warning when it cannot resolve.

## Solution Assessment

Yes. The narrow startup-reconcile skip is the best maintainable fix for the noisy warning path; post-turn one-shot identity reconciliation improves future successful runs but does not cover existing stale or failed one-shot records.
