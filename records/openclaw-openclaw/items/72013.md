## Summary

Keep open. Current main still reconciles persisted ACP records by pending identity plus stable session id without excluding `mode: "oneshot"`, so historical pending one-shot records can still produce the startup warning. The focused related PR #72023 is closed and unmerged, so it has not resolved the issue on main.

## Best Possible Solution

Keep this issue open and land a small ACP control-plane fix equivalent to the closed PR #72023: skip persisted `mode: "oneshot"` ACP records in `AcpSessionManager.reconcilePendingSessionIdentities` before `checked` is incremented or the runtime is touched, preserve persistent pending identity reconciliation, and add regression tests for the skipped one-shot startup path. If historical raw `state: "closed"` records are real, handle that with a narrow legacy/raw predicate and test rather than a dead typed comparison.
