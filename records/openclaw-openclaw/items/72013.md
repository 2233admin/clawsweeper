## Summary

Keep open. Current main still lets pending ACP one-shot records with stable ACPX or agent session IDs enter startup identity reconciliation, so the reported gateway warning can still recur; the focused source PR for this exact issue is closed unmerged.

## What This Changes

Startup identity reconciliation should skip terminal one-shot ACP session records with pending identities so gateway startup does not log misleading `failed=N` warnings for stale metadata.

## Best Possible Solution

Add a narrow guard in `AcpSessionManager.reconcilePendingSessionIdentities` before `checked` increments so `mode: "oneshot"` records are skipped, preserve normal persistent pending-identity reconciliation, and add a focused regression test for pending one-shot metadata with a stable ACPX or agent session id.

## Reproduction Assessment

Yes. A focused manager unit fixture can return a persisted ACP entry with `mode: "oneshot"`, `identity.state: "pending"`, and `identity.acpxSessionId`; current main passes the pending-plus-stable predicate and counts it for startup reconciliation.

## Solution Assessment

Yes. The startup skip is the narrowest maintainable fix for historical terminal one-shot metadata; resolving future one-shot identities after turns is useful but does not stop already-persisted pending one-shot records from being retried on every restart.
