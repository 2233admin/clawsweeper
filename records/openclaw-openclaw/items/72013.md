## Summary

Keep open. Current main still allows persisted one-shot ACP sessions with pending identities and stable ACPX or agent session IDs to enter startup reconciliation, so the reported warning can still occur for historical terminal metadata; the focused related PR is closed unmerged.

## What This Changes

The issue asks startup ACP identity reconciliation to skip or quietly handle terminal one-shot or closed ACP session records whose identity remains pending.

## Best Possible Solution

Add a narrow guard inside AcpSessionManager.reconcilePendingSessionIdentities before checked increments so terminal one-shot ACP records are skipped without touching normal persistent pending-identity reconciliation, with focused regression coverage for pending one-shot records that carry stable ACPX IDs.

## Reproduction Assessment

Yes. A persisted ACP record with acp.mode:"oneshot", identity.state:"pending", and identity.acpxSessionId satisfies the current pending-plus-stable predicate and reaches the warning-producing checked path; this is reproducible with a manager unit fixture without live ACPX.

## Solution Assessment

Yes. The issue's startup skip direction is the narrowest maintainable fix for historical terminal one-shot metadata; post-turn identity reconciliation helps future successful runs but does not prevent old pending one-shot records from being retried on every gateway restart.
