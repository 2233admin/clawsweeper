## Summary

Keep open. Current main still lets pending one-shot ACP session metadata with stable ACPX or agent session IDs enter startup identity reconciliation and then logs the warning whenever any records were checked; the exact focused PR for this issue is closed unmerged, and the related open identity PR does not cover historical one-shot records.

## What This Changes

Startup ACP identity reconciliation should skip or quietly ignore terminal one-shot ACP session records whose persisted identity remains pending.

## Best Possible Solution

Add a narrow guard inside `AcpSessionManager.reconcilePendingSessionIdentities` before `checked` increments so terminal one-shot records, and any supported legacy closed records, are skipped without changing normal persistent pending-identity reconciliation. Add focused regression coverage for a pending one-shot record with a stable ACPX or agent session ID.

## Reproduction Assessment

Yes. A stored ACP session fixture with `mode:"oneshot"`, `identity.state:"pending"`, and `identity.acpxSessionId` or `identity.agentSessionId` follows the current predicate into the checked path, so this can be reproduced in `manager.test.ts` without live ACPX.

## Solution Assessment

Yes. The startup skip is the narrowest maintainable fix for historical terminal one-shot metadata; resolving future identities earlier is useful adjacent work but does not stop old pending one-shot records from being retried on every gateway restart.
