## Summary

Current main has several cron reliability improvements, including execution timeouts, recurring error backoff, failure alerts, isolated cron run-session retention, JSONL run history, and a plugin `cron_changed` hook. It still does not implement the requested recurring-job circuit breaker/auto-pause, remove-time session cleanup option, or automatic `.learnings/changelog.md` writer.

## Best Possible Solution

Keep this open. The best path is to add a backward-compatible recurring cron circuit-breaker config/schema/protocol/docs/test surface and enforce it in `src/cron/service/timer.ts`; design an explicit cleanup option across Gateway protocol, CLI, UI, and session/transcript ownership; then decide whether JSONL `cron.runs` plus the `cron_changed` plugin hook is the supported audit trail or whether core should also write `.learnings/changelog.md`.

## Risks / Open Questions

- The issue bundles three separable changes; implementation should likely split circuit breaker, remove cleanup, and audit/changelog behavior into distinct PRs or follow-up issues.
- Remove-time cleanup needs an explicit ownership contract so OpenClaw does not delete shared, custom, base, or still-referenced sessions/transcripts unexpectedly.
- The changelog request needs product direction: document JSONL run history plus the `cron_changed` plugin hook as the supported audit path, or add a deliberate core writer for `.learnings/changelog.md` if that file is intended behavior.
