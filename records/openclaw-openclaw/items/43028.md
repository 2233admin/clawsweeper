## Summary

Keep this issue open. Current main has cron execution timeouts, recurring error backoff, failure alerts, retention pruning for isolated cron run sessions, and JSONL run history, but it still does not implement the requested recurring-job circuit breaker, remove-time session cleanup option, or automatic `.learnings/changelog.md` recording.

## Best Possible Solution

Keep this open. The best path is to split the work: add a backward-compatible recurring execution circuit breaker in cron config/schema/protocol/docs/tests and enforce it in `src/cron/service/timer.ts`; design an explicit cleanup option for `cron.remove` across Gateway protocol, CLI/tool/UI, and session/transcript ownership; then decide whether JSONL `cron.runs` is the supported audit trail or whether core should also write `.learnings/changelog.md`.

## Risks / Open Questions

- The issue bundles three separable cron changes; maintainers may want separate implementation issues or PRs for the circuit breaker, remove cleanup, and changelog/audit decision.
- Remove-time cleanup needs an explicit ownership contract so OpenClaw does not delete shared, custom, base, or still-referenced sessions/transcripts unexpectedly.
- The changelog request needs product resolution: either document JSONL `cron.runs` as the supported audit trail or add a deliberate core writer for `.learnings/changelog.md` if that file is intended behavior.
