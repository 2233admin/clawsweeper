## Summary

Keep this issue open. Current main has cron execution timeouts, recurring backoff, failure alerts, retention pruning, and per-job JSONL run history, but it still does not implement the requested recurring-job circuit breaker, remove-time session cleanup option/contract, or automatic `.learnings/changelog.md` recording.

## Best Possible Solution

Keep this open and split the implementation path: add a backward-compatible recurring execution circuit breaker in cron config/schema/protocol/docs/tests and enforce it in `src/cron/service/timer.ts`; design an explicit cleanup option for `cron.remove` across gateway protocol, CLI, UI, and session/transcript ownership; then decide whether JSONL `cron.runs` is the supported audit trail or whether core should also write `.learnings/changelog.md`.
