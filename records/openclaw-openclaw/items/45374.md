## Summary

Keep open. Current main has shipped much of the stale-plugin cleanup side, but it still does not implement the PR's runtime drift surface for ordinary `gateway status` or doctor output. The remaining useful work is to rebase the PR and focus it on service state-dir drift, `~/.openclaw-tests/*` detection, and service-vs-CLI version mismatch.

## Best Possible Solution

Keep this PR open as an implementation candidate for the still-missing runtime drift surface. Rebase it onto current main, drop or adapt the stale-plugin cleanup pieces now handled by shared doctor helpers, and focus the patch on tested `gateway status`/doctor reporting for CLI-vs-service state-dir drift, `~/.openclaw-tests/*` usage, and service-vs-runtime version mismatch.
