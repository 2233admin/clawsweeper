## Summary

Keep this PR open. Current main has already implemented the stale-plugin and redundant bundled-plugin load-path cleanup through newer doctor helpers, but it still does not implement the PR's runtime drift surface for ordinary `gateway status` or doctor: no CLI-vs-service state-dir fields, no `~/.openclaw-tests/*` classifier, and no service-vs-CLI version mismatch reporting in those paths.

## Best Possible Solution

Keep this PR open as an implementation candidate for the still-missing runtime drift diagnostics. Rebase it onto current main, drop or adapt the stale-plugin cleanup pieces already handled by the shared doctor helpers, and focus the patch on tested `gateway status`/doctor reporting for CLI-vs-service state-dir drift, `~/.openclaw-tests/*` usage, and service-vs-runtime version mismatch.
