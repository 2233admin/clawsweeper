## Summary

Keep open. Current main still has the stale replay path described in the report, and open PR #50147 already targets the same bug with closing syntax, so this issue should wait for that PR or an equivalent fix to land.

## What This Changes

Aborted `messages.queue.mode: "interrupt"` runs should exit silently instead of packaging and delivering an older assistant reply before the new run responds.

## Best Possible Solution

OpenClaw should suppress payload construction and reply delivery for aborted interrupt runs that produced no new assistant output, while preserving legitimate non-aborted fallback behavior and adding focused regression coverage around the payload-builder and reply-runner boundary.
