Keep open. Current main still has the reported stale-delivery path, and the paired open PR #50147 is still the right review path rather than closing this issue before a fix lands.

## What This Changes

This issue reports that `messages.queue.mode: "interrupt"` can replay the previous assistant reply from an aborted run before delivering the new reply.

## Best Possible Solution

Land PR #50147 or an equivalent narrow patch that prevents aborted interrupt runs with no new assistant output from producing or delivering payloads, with focused regression coverage at the payload-builder and reply-runner boundary.
