## Summary

Keep open. Current main still lacks the requested per-job recurring cron retry controls, and open PR #51001 is an unmerged implementation candidate that links this issue for closure. Existing recurring error handling applies an error-backoff floor, but it can still schedule the later natural run for daily jobs, and the CLI/protocol do not expose `--retry-count`, `--retry-delay`, or `--retry-backoff`.

## Best Possible Solution

Keep this issue open and review PR #51001 as the active implementation candidate. A complete fix should add a durable per-job recurring retry contract, CLI add/edit flags, protocol/schema support, scheduler state that retries before the next natural run without cadence drift, docs, and focused regression coverage. After an equivalent implementation lands, reconcile this with related issue #38069 if both remain open.
