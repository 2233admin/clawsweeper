## Summary

Keep this PR open. Current main still eagerly computes image/video/music provider lists before the explicit model-config short-circuit, and the proposed thunk change is a narrow fix for the latency path reported in #74096; I did not find a blocking correctness or security issue in the supplied diff.

## What This Changes

This PR changes `resolveCapabilityModelConfigForTool` to accept a lazy provider-list thunk, updates the image/video/music generation model-config resolvers to pass thunks, and adds regression tests for the explicit-config short-circuit and incomplete-config fall-through paths.

## Best Possible Solution

Land this PR or an equivalent lazy-provider patch after rebase and validation, keeping the provider-runtime cache work from #73793 separate and adding a changelog fix entry if maintainers want the hosted-gateway latency improvement called out.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is to call any current image/video/music model-config resolver with a complete `agents.defaults.*GenerationModel` and a mocked runtime provider lister; current main invokes the lister before the helper can return the explicit model config. #74096 also supplies production stack traces for the same runtime path.

## Solution Assessment

Yes. The thunk boundary is the narrowest maintainable fix because it defers provider enumeration only until the explicit config is incomplete, reuses one provider list for the fall-through path, and does not change provider registry behavior or #73793 caching semantics.
