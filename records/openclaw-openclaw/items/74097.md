## Summary

Current main still has the eager provider-listing behavior targeted by this PR. The shared helper accepts a `CapabilityProvider[]`, returns early only after callers have already built that array, and the image/video/music callers still call `listRuntime{Image,Video,Music}GenerationProviders` before invoking the helper. The PR remains a focused source/test-only implementation candidate for linked #74096; related #73793 addresses the broader provider-cache path but does not make this lazy-call-site cleanup implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent patch, because current main still computes image/video/music provider lists eagerly before the explicit model-config short-circuit. Before merge, rebase or otherwise validate against current main and run the targeted media tool tests plus any build/type gate required for lazy-loading/plugin-runtime changes.
