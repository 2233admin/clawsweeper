## Summary

Keep open. Current main still has no `session.maintenance` transcript-size or transcript-line cap: the config types/schema/docs expose only entry pruning, reset-archive retention, and directory-level disk budget controls, while `rotateBytes` is now deprecated and ignored. The related implementation PR #66546 was closed unmerged in the provided GitHub context, so there is no shipped fix to close against.

## Best Possible Solution

Keep this issue open and implement a focused session-maintenance change: add explicit transcript size and/or line controls to the config type, zod schema, generated docs/help, runtime maintenance path, and `openclaw sessions cleanup`, with active-session-safe rotation semantics and regression tests for long-lived topic-bound transcripts.
