## Summary

Keep open. Current main still has the channel-only cron announce inference gap, and the fetched patch is a narrow fix that matches the existing delivery inference contracts; it needs maintainer review and validation rather than cleanup closure.

## What This Changes

The PR changes cron-tool add-action delivery inference so channel-only announce delivery still gets a concrete `delivery.to`, preserves explicit `delivery.to`, and adds three cron-tool regression tests.

## Best Possible Solution

Keep the existing cron announce delivery model but infer the concrete target when only `delivery.channel` is provided; explicit `delivery.to` should remain authoritative, and `webhook`/`none` modes should stay outside announce inference.

## Reproduction Assessment

Yes. A focused source-level reproduction is an add call for an isolated `agentTurn` from `agent:main:slack:channel:general` with `delivery: { mode: "announce", channel: "slack" }`; current main skips inference and leaves `delivery.to` unset, and the PR adds that regression case.

## Solution Assessment

Yes. Gating only on an explicit non-empty `delivery.to` is the narrowest maintainable fix because the existing helpers already know how to derive the target, and the patch preserves explicit targets while avoiding scheduler or resolver changes.
