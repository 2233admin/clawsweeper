## Summary

Keep open. Current main has adjacent hardening for plan-only turns, empty/error turns, tool-shaped text diagnostics, and proxy tool_choice defaults, but it still does not implement the requested guarantee: verifying or blocking final research/source claims when no matching web_search/browser/tool telemetry exists. The current strict-agentic path is scoped to OpenAI/OpenAI Codex GPT-5-family runs and does not cover the reported OpenRouter Claude route.

## Best Possible Solution

Keep this open as a core trust/integrity bug. The likely path is a configurable source/claim verification layer that compares final assistant claims about searches, browser activity, cited sources, and research completion against actual web_search/browser/tool-call telemetry, then warns or blocks delivery. Coordinate with #45049 for real tool-call enforcement and #49876 for fail-closed delivery behavior; #68051 is useful design context but does not supersede the remaining work.
