## Summary

Keep open. Current main has adjacent diagnostics, tool-shaped-text logging, provider tool-choice defaults, trajectory export, and tool-error surfacing, but it still does not implement the requested guarantee: block or warn when an agent claims Google/property/source research without matching web_search/browser/tool-backed evidence.

## Best Possible Solution

Keep this issue open as a core trust/integrity bug. The likely path is a configurable runtime source/claim verification layer that compares final claims about searches, browser activity, and cited sources against actual web_search/browser/tool-call telemetry and then warns or blocks delivery. Coordinate with #45049 for real tool-call enforcement and #49876 for fail-closed delivery behavior.
