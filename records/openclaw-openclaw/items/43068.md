## Summary

Keep this PR open. Current main still only has the existing shell-syntax/script preflight, not sandbox-only Python/Node research-script import validation. The PR targets a real command-execution hardening gap, but it needs changes before merge because review comments identify concrete parser bypasses in the proposed validator.

## Best Possible Solution

Keep this PR open and request changes. The best path is to rebase onto current main, add sandbox-only import-template validation after the existing script preflight, harden parsing for multiline Node import/re-export syntax and whitespace-separated dynamic require/import calls, handle Python bare-relative denied names, and add regression tests for sandbox-vs-host behavior plus the identified bypasses before maintainer review.
