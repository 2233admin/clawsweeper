## Summary

Keep this PR open. Current main still does not implement sandbox-only Python/Node research-script import validation; it only has the existing script preflight for shell-variable leakage, shell-as-JS mistakes, and ambiguous interpreter invocations. The PR targets a real command-execution hardening gap, but the proposed scanner has unresolved security-relevant parser bypasses called out in review, so it should not be closed as obsolete or merged as-is.

## Best Possible Solution

Keep this PR open and request changes. The best path is to rebase onto current main, add sandbox-only import-template validation after the existing script preflight, harden parsing for multiline Node import/re-export syntax and whitespace-separated dynamic require/import calls, handle Python bare-relative denied names, and add regression tests for sandbox-vs-host behavior plus the identified bypasses before maintainer review.
