## Summary

Keep this PR open. Current main still has the stale-cache behavior the PR targets, and the linked bug remains open, but the PR patch also changes generic error handling by clearing the sessions cache on every exception. That makes it an active implementation candidate that needs a narrower fix and regression tests, not a conservative cleanup close.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the linked sessions-count bug, but revise it narrowly: assign `state.sessionsResult = res ?? null` after a successful `sessions.list`, run checkpoint invalidation/refetch only when `res` exists, preserve current generic-error cache behavior, keep the missing-scope clear path, and add focused controller tests for undefined-success clearing plus transient-error preservation.
