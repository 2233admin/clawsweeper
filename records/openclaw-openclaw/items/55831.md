## Summary

Keep this PR open. Current main still has the stale-cache behavior this PR is trying to fix: an undefined successful `sessions.list` response is ignored because `sessionsResult` is only updated inside an `if (res)` guard. The PR is a plausible implementation candidate for linked #55705, but it needs revision because its current diff also clears the cached sessions result on every exception, which would erase the last-known-good snapshot during transient gateway/RPC errors.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #55705, but revise it to narrow the behavior: set `sessionsResult = res ?? null` on successful `sessions.list` responses, preserve the current last-known-good cache behavior for generic errors, keep the existing missing-scope clear path, and add focused unit tests for both the undefined-success clear and transient-error preservation before review/landing.
