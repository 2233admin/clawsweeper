## Summary

Keep open. Current main still lacks the requested configurable subagent operating-principle list and max-failure-attempt setting. Compaction tuning exists through reserve-token and keep-recent-token controls, but not the proposed ratio/message-count API; the linked PR #61021 was closed unmerged, so there is no merged implementation to close against.

## Best Possible Solution

Keep the issue open and narrow the design. The compaction portion may be addressed by documenting or refining the existing reserveTokens/keepRecentTokens controls, but the subagent prompt-policy portion still needs maintainer product judgment and a focused implementation across config types, zod schema, labels/help/generated schema, buildSubagentSystemPrompt, spawn/recovery threading if needed, docs, and tests.

## Risks / Open Questions

- Configurable removal of subagent prompt rules could weaken orchestration and recovery behavior unless defaults remain conservative and validation is explicit.
- Adding ratio/message-count compaction controls could overlap with existing reserveTokens/keepRecentTokens semantics, so maintainers should avoid two competing threshold APIs.
