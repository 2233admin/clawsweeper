## Summary

Keep this PR open. Current main still lacks the reserved hook session-key guard the PR is trying to add: hook request, default, and static/templated mapping session keys are only checked against request policy and allowed prefixes, not against internal `subagent:`, `acp:`, or `cron:` namespaces. The PR discussion also shows two review findings were addressed in follow-up commits, including startup validation for static mappings and trimming unwrapped `agent:<id>:` segments.

## Best Possible Solution

Review and land this PR or an equivalent maintainer-owned patch that adds a shared reserved-session-key helper, rejects `hooks.defaultSessionKey`, request-sourced keys, static mapping keys, and runtime templated mapping keys targeting `subagent:`, `acp:`, or `cron:`, and keeps the regression tests for direct request, mapping, default config, nested `agent:<id>:` wrappers, and whitespace-normalization cases.
