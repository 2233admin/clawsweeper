## Summary

Keep PR #58709 open. Current main still has the exact sidebar event-delegation gap described by the PR: markdown rendering emits `.code-block-copy` buttons for sidebar content, but `handleCodeBlockCopy` is only bound on `.chat-thread`, while `.chat-sidebar` is a sibling without the handler. The related WebChat copy-button work only covers button rendering and the main chat thread path, not this sidebar click path.

## Best Possible Solution

Keep the PR open and land this narrow sidebar binding, or apply the equivalent change on main. A small regression test for clicking a sidebar-rendered `.code-block-copy` button and observing `navigator.clipboard.writeText` would make the fix easier to preserve.
