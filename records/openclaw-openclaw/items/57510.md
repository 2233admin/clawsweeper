## Summary

Keep this PR open. Current `main` does not implement the proposed `pre_route` typed hook or an equivalent route/session override contract before dispatch. The existing hook surface has `inbound_claim`, `message_received`, `before_dispatch`, and `reply_dispatch`, but those do not expose the PR’s central same-turn `routeOverride`/`agentIdOverride` behavior. The PR is not superseded by current main and has useful discussion plus reviewer feedback that should be handled through normal review rather than cleanup closure.

## Best Possible Solution

Keep this PR open. The best path is to review or rebase the PR against current `main`, verify the final implementation covers same-turn `pre_route` route/session/account/agent overrides across normal, native-command, and ACP dispatch paths, keep the public SDK/docs/tests aligned, and land it if maintainers want this as a core plugin hook.
