## Summary

Keep open. The report targets the latest shipped release, current main has not changed the Slack slash-command dispatch path enough to prove this is fixed, and the remaining behavior sits at the Slack multi-app/native-command routing boundary rather than an already-implemented core fix.

## What This Changes

Fix Slack native slash-command routing when multiple Slack Apps in one workspace register the same built-in commands so the command is handled by the intended channel or agent.

## Best Possible Solution

Keep this open for maintainer review of Slack native-command support in multi-app workspaces: either document duplicate Slack slash command names as unsupported and recommend unique command names/channel bindings, or add a targeted routing rule/test so duplicate native commands dispatch to the channel-bound agent even when Slack delivers the payload through the app that currently owns the command name.
