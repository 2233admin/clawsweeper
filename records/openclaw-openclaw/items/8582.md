## Summary

Keep this issue open. Current main implements the ElevenLabs realtime STT and provider-generic Voice Call streaming parts, but the remaining DX asks are still actionable: config-pinned duplicate plugin overrides still warn, there is no per-id suppression/override acknowledgement config, and the docs still lack focused guidance for extending or forking a bundled plugin with new config keys.

## Best Possible Solution

Keep this issue open unless a maintainer splits it. The best path is to add a tested per-id duplicate override or suppression acknowledgement in plugin config and manifest-registry diagnostics, document that behavior, and add focused plugin docs explaining that custom config keys require updating openclaw.plugin.json configSchema when extending or forking a bundled plugin. Alternatively, create canonical follow-up issues for those remaining DX asks before closing this umbrella item.
