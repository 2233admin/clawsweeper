## Summary

Keep open. Current main still lacks the requested `openclaw skills uninstall <slug>` CLI command and still lacks a skills backend removal helper that deletes both the workspace skill directory and the `.clawhub/lock.json` entry. Related context supports keeping this as the canonical CLI issue: #14264 was closed in favor of this, #17503 was closed/unmerged and targeted dashboard/bundled-skill behavior, and #69319 is an open gateway-only PR rather than the requested CLI surface.

## Best Possible Solution

Keep this issue open and implement `openclaw skills uninstall <slug>` as the CLI-facing inverse of ClawHub skill install/update: resolve the active workspace, safely resolve the skill directory, remove that directory when present, remove the `.clawhub/lock.json` entry, support confirmation/`--yes`, `--dry-run`, `--json`, and `--agent`, update `docs/cli/skills.md` plus `docs/tools/clawhub.md`, and add focused backend and CLI tests. If #69319 lands first, reuse its backend primitive where the semantics match this CLI flow.

## Clownfish Work Prompt

Implement the CLI uninstall flow requested by this issue for current `main`.

Observable gap: `openclaw skills install/update` write and consume ClawHub skill metadata, but there is no `openclaw skills uninstall <slug>` command to remove both `<workspace>/skills/<slug>` and the matching `.clawhub/lock.json` entry. Manual deletion leaves the lockfile entry behind, so `openclaw skills update --all` can recreate the skill.

Expected boundary:
- Add a backend helper in `src/agents/skills-clawhub.ts` that safely plans/executes uninstall for one slug in a workspace.
- Reuse `resolveSkillInstallDir` / `resolveSafeInstallDir` for slug-to-path containment. Do not hand-roll path joins for recursive deletion.
- Remove the skill directory when present and remove the matching `.clawhub/lock.json` entry when present. Preserve unrelated lockfile entries.
- Support directory-only, lockfile-only, and not-installed cases with clear idempotent results.
- Add `openclaw skills uninstall <slug>` in `src/cli/skills-cli.ts` with `--agent`, `--yes`, `--dry-run`, and `--json` behavior. Use `promptYesNo` for confirmation unless `--yes` or dry-run applies.
- Add formatting helpers in `src/cli/skills-cli.format.ts` if that matches the local CLI style.
- Update `docs/cli/skills.md` and `docs/tools/clawhub.md` for the new command. Changelog entry is likely user-facing; include a single-line fix/feature note with a real contributor attribution if this PR is promoted.
- If #69319 has landed by the time work starts, reuse any compatible `uninstallSkillFromClawHub` backend semantics instead of creating a divergent implementation. This CLI issue is not solved by a gateway-only method.
- Do not add dashboard UI, bundled-skill uninstall, `.gitignore` mutation, sandbox purging, wildcard/multi-slug behavior, or plugin config mutation.

Related refs: #14264, #17503, #69299, #69319.
