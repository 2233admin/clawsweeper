## Summary

Keep this PR open. Current main still has the narrow streaming and user-facing final-tag regexes, so the attributed Gemini `<final ...>` leak targeted by this PR is not implemented on main. The PR also remains distinct from the broader Gemini plain-text reasoning leak tracked in #41494.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch. The best fix is a shared exact reasoning-tag matcher/parser used by streaming cleanup, user-facing sanitization, and live-test helpers, supporting attributed and self-closing `<final>`/`<think>` tags without matching hyphenated custom tags, plus regression coverage for streaming, sanitizer, and final-tag enforcement paths.
