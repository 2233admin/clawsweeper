## Summary

Keep this PR open. Current main still has the narrow final/thinking tag scanners on delivered-message paths, so the intended fix is not implemented on main. The PR is a useful candidate, especially after the author addressed attributed thinking and self-closing tags in discussion, but it still needs maintainer review for the exact tag-name matching concern raised in review.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch. The best fix is a shared exact tag-name matcher/parser used by streaming cleanup, user-facing sanitization, and live-test helpers, supporting attributes and self-closing tags without matching hyphenated custom tags. Add regression coverage for enforced streaming, non-enforced streaming cleanup, `sanitizeUserFacingText`, attributed `<final>` and `<think>` tags, and self-closing tag behavior; keep #41494 as the umbrella for undelimited Gemini reasoning output.
