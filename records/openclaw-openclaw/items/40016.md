## Summary

Keep this issue open. Current main has backend session label plumbing through `sessions.patch`, but it still does not expose a first-class `openclaw sessions rename`, `set-label`, or `set-subject` command, and the local `openclaw sessions` table/JSON path still does not project session labels, subjects, or display names.

## Best Possible Solution

Keep this issue open and implement a focused `openclaw sessions rename <session-key> <label>` or `set-label` command backed by the existing session label semantics. The command should set the generic `label`, not mutate channel-owned `subject` or `displayName`; the sessions list text and JSON output should include an effective human label while preserving the session key. Add command registration, local command/table tests, and docs at https://docs.openclaw.ai/cli/sessions.
