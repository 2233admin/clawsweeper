## Summary

Keep open. Current main improves runtime unknown-model handling and has a narrow Gmail hook model catalog check, but it still does not provide the requested generic startup or doctor diagnostics for invalid configured model IDs after updates.

## What This Changes

The issue requests startup, doctor, and changelog diagnostics when configured model IDs become invalid after an OpenClaw update.

## Best Possible Solution

Add one shared configured-model diagnostics layer used by both Gateway startup and openclaw doctor: collect model refs from defaults, fallbacks, agents, hooks, channels, tools, and media settings; resolve aliases like runtime; warn only when catalog evidence is authoritative; and document/changelog retired or renamed shipped model slugs when OpenClaw changes them.
