## Summary

Keep this issue open. Current main has adjacent runtime/fallback handling for invalid OpenRouter responses, and the related cryptic-error issue is closed, but the requested conservative startup and `openclaw doctor` diagnostics for stale configured model IDs are still missing.

## Best Possible Solution

Implement non-fatal configured-model diagnostics in gateway startup and `openclaw doctor`. Collect configured refs across defaults, fallbacks, media/tool refs, hooks, channel overrides, aliases, and per-agent overrides; warn only when a provider catalog was successfully loaded and is authoritative enough to prove the ref is absent, while preserving OpenRouter/custom-provider dynamic pass-through when authority is unavailable or intentionally incomplete. Release notes should call out bundled default slug changes or known retired slugs when OpenClaw changes compatibility behavior.

## Risks / Open Questions

- OpenRouter intentionally supports dynamic pass-through for newly published or custom upstream routes, so a hard failure on every catalog miss would create false positives.
- Configured provider models are merged into the catalog, so diagnostics need to distinguish authoritative provider catalog misses from user-declared or incomplete custom catalog data.
- A narrow primary-only warning would miss configured fallbacks, media model refs, per-agent overrides, hook refs, and alias-resolved refs that can drift after provider catalog changes.
