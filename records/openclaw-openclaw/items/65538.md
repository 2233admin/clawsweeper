## Summary

Keep open. Current main still renders the Control UI chat transcript as a polite live log while streamed assistant text is appended inside that same transcript, and open PR #65575 is already the focused implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests that Control UI chat suppress screen-reader announcements for incremental streaming tokens and announce only after the assistant response is fully rendered.

## Best Possible Solution

Land #65575 or an equivalent narrow Control UI fix that sets the chat transcript live-region behavior to quiet while sending or streaming, restores the expected idle announcement behavior afterward, and adds focused tests around `.chat-thread` for idle, sending, streaming, and final-render states.
