## Summary

Keep this PR open. Current main still contains the exact Slack preview-streaming gate the PR removes: direct messages without a `threadTs` return `false`, and the current regression tests assert that top-level DM preview streaming stays off. The PR is a narrow Slack dispatcher/test change and is not implemented or clearly superseded on main.

## Best Possible Solution

Review and either land this PR or apply the equivalent focused change on main: enable Slack draft preview streaming for all non-`off` modes, add/keep the flat-DM regression tests, remove or explain now-unused helper parameters, and update Slack streaming docs/config hints if flat DM draft previews are the intended product behavior.
