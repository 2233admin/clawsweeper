## Summary

Keep #38806 open. Current main at 43a003b8a0629ad2c73947b2a863d99dd506bcb9 still appears to route Feishu P2P thread replies through the non-thread direct-message path: inbound `root_id`/`thread_id` are parsed, but thread-aware reply mode, target selection, and reply metadata preservation are still gated on group chat state. The provided related context identifies #38808 as the still-open targeted fix candidate; #38814 and #38996 were closed unmerged.

## Best Possible Solution

Keep #38806 open and resolve it through #38808 or an equivalent targeted Feishu patch. The fix should detect P2P thread messages with `isDirect && Boolean(ctx.threadId)`, reply to the P2P thread root, pass `replyInThread`/`threadReply`, preserve reply metadata for direct threads, keep root-only P2P quote replies on the triggering message, and add regression tests for both P2P thread replies and root-only P2P quote replies.
