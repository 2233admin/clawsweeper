## Summary

Keep #69269 open. Current main appears to have addressed part of the token-accounting problem for manual gateway compaction, including `tokensAfter === 0`, but the broader issue is not implemented: both reset hook paths still emit `before_reset` without `messageProvider`, and compaction hook paths still pass the raw `messageChannel ?? messageProvider` value rather than using the requested shared normalization/inference rules. The linked fix PR #69270 remains open and is the right active path, not evidence that the issue can be closed.

## Best Possible Solution

Keep #69269 open and land or refresh #69270, or an equivalent maintainer-reviewed patch. The best fix is a shared hook-message-provider resolver used by auto compaction, engine-owned compaction, command reset hooks, and gateway reset hooks: normalize explicit providers, infer only from deliverable channel-scoped session keys, leave non-channel keys unresolved, and preserve the current zero-token accounting behavior with focused regression tests.
