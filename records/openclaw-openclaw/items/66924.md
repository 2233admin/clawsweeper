## Summary

Keep this PR open. Current main still maps `gemma-*` forward-compat models to the Gemini flash template without a Gemma context-window override, so the observable 1M inherited-context bug is not implemented on main. The PR is a focused Google provider fix with a matching regression test, but maintainer review should confirm the fallback value and metadata precedence before merge.

## What This Changes

The PR adds a Gemma-specific 131,072-token context-window fallback in the Google forward-compat resolver and a regression test for a Gemma 4 model cloned from the Gemini flash template.

## Best Possible Solution

Keep the fix in the Google provider boundary and land it with regression coverage, while ensuring Gemma fallback clones never inherit Gemini flash's 1M context and explicit configured or available Gemma metadata wins when present.
