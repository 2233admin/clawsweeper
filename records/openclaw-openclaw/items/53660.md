## Summary

Keep this PR open. Current main still does not implement append-only reuse for parsed session entries: `buildSessionEntry` reads and parses the whole transcript every time, its options only cover transcript classification, and the memory sync path calls it without any previous-entry cache. The PR is also not safe to close as obsolete because no current-main replacement or canonical merged PR was found, while the supplied review discussion identifies concrete fixes still needed before merge.

## Best Possible Solution

Keep the PR open as a useful optimization candidate, but require a rebase/rework against the current memory-host-sdk and memory-core paths. The safest direction is byte-based append handling, explicit prefix validation before reusing prior content, a bounded/cleared cache, and regression tests for ASCII append, non-ASCII transcripts, rewritten prefixes, stale/deleted session cleanup, and hash-equal cache warming.
