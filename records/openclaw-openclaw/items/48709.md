## Summary

Keep open. Current main still preserves and replays Google visible-text signatures and still routes the Google Gemini provider family through tagged reasoning with strict final-tag suppression; the mixed text-before-tool path has newer block-flush protection, but the remaining signature-bloat and tagged-reasoning failure modes are still real.

## What This Changes

The issue asks OpenClaw to stop Gemini textSignature bloat/replay, handle embedded `<think>` text safely, and deliver text emitted before tool calls instead of dropping it.

## Best Possible Solution

Handle this in the Google provider and replay boundary: preserve only signatures required for valid Gemini continuation, bound or avoid replaying stale visible-text signatures, and align Gemini reasoning delivery with native thinking or a safer fallback so malformed tagged output cannot silently suppress replies.
