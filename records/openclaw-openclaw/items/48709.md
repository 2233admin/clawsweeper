## Summary

Keep open. Current main still has the central Google/Gemini failure paths: visible-text signatures can be stored and replayed, Google replay sanitization does not target OpenClaw's `textSignature` field, and `google-generative-ai` still routes through tagged reasoning with strict final-tag suppression. The mixed text-before-tool delivery path now has partial flush protection, but the remaining signature-bloat and tagged-reasoning issues are not implemented or stale.

## What This Changes

The issue asks OpenClaw to prevent Gemini 2.5 Pro session failures caused by `textSignature` history bloat, embedded `<think>`/tagged-reasoning suppression, and dropped text from mixed text-plus-tool responses.

## Best Possible Solution

Handle this at the Google provider and replay boundary: preserve only Gemini signatures that are required for valid continuation, bound or avoid replaying stale visible-text signatures, and move Gemini reasoning delivery away from conflicting tagged/final-tag enforcement or add a safe provider-specific fallback so malformed tagged output cannot silently suppress replies.
