## Summary

Keep this issue open. Current main still retains Gemini `thoughtSignature` data on text blocks as `textSignature` and replays it for the same Google model, while the provider replay policy sanitizes only `thoughtSignature` fields and turn ordering, not persisted `textSignature` bloat. The mixed text/tool delivery concern appears partly addressed by current payload and gateway-event logic, but the central session-bloat and think-tag suppression concerns remain unresolved enough to need maintainer review.

## Best Possible Solution

Keep this open and split the fix into a provider-owned Gemini replay/session design: preserve signatures required for current-turn function calls, bound or externalize old text-only signatures so JSONL/context cannot balloon, and add targeted regression tests for Gemini 2.5 Pro textSignature replay, tagged reasoning delivery, and mixed text+tool final delivery. Coordinate the think-tag behavior with the existing tag-sanitization follow-ups instead of closing this as stale.
