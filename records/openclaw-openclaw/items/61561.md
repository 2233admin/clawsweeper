## Summary

Keep open. Current main still has no BlueBubbles `iMessageLite` chat GUID normalization, so the PR is useful and not obsolete, but the patch remains incomplete because documented `chat_guid:` and `guid:` target forms are parsed before the new raw-GUID normalization path and can still send `iMessageLite` unchanged.

## What This Changes

The PR adds a BlueBubbles `normalizeChatGuidService` helper, applies it to inbound webhook `chatGuid` values and bare raw chat GUID target parsing, and adds monitor-normalize tests for satellite `iMessageLite` GUIDs.

## Best Possible Solution

Revise this PR or a narrow replacement so BlueBubbles has one chatGuid service canonicalization path applied anywhere a `chatGuid` is parsed into a target or inbound message, then land it with targeted coverage for webhook ingestion, bare raw GUID targets, prefixed `chat_guid:` and `guid:` targets, and unchanged SMS/iMessage behavior.
