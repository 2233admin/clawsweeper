## Summary

Keep this PR open. Current main still has `resolveTelegramCommandAuth` calling `evaluateTelegramGroupPolicyAccess` with `useTopicAndGroupOverrides: false`, so native Telegram commands still skip topic/group `groupPolicy` overrides even though regular message handling uses the override-aware path. The PR is a focused functional fix, not obsolete cleanup; the best maintainer path is to land or replace it with the same boolean change plus a focused regression test.

## Best Possible Solution

Keep this PR open and have maintainers review/land a small Telegram plugin fix that changes native command auth to `useTopicAndGroupOverrides: true`, ideally with a regression test that passes a topic `groupPolicy: "disabled"` through `resolveTelegramGroupConfig` and verifies `/status`, `/reset`, or `/new` is blocked in that forum topic.
