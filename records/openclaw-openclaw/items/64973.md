## Summary

Keep this issue open. Current main still has the macOS local-save fallback and AppState direct local config writer that can bypass the gateway baseHash conflict path after a stale config.set, so the reported auth/token clobber and restart sequence is not fully fixed. The related #43244 remains useful for gateway RPC serialization, but its own scope does not cover direct macOS config writers.

## What This Changes

The issue reports that macOS Talk Mode against a local token-auth gateway can hit a stale config.set, mutate or clobber gateway auth and Talk provider config, cause gateway token mismatches, and restart the local gateway while Talk is running.

## Best Possible Solution

Make the macOS config mutation path conflict-aware end to end: stale config.set must refresh from config.get and retry or surface a non-destructive error, never fall through to a direct stale disk write; AppState gateway sync should use the same fresh snapshot/baseHash path or be narrowed to app-local fields; regression coverage should prove gateway.auth and all talk.providers survive Talk/local-gateway config changes without triggering an auth restart.

## Repair Work Prompt

Fix the macOS Talk Mode/local gateway stale config write path described in this issue. Scope the change to macOS config mutation behavior and tests. In `ConfigStore.save`, do not fall back to `OpenClawConfigFile.saveDict(root)` after a gateway `config.set` stale baseHash/conflict error; refresh with `config.get` and retry safely or surface a non-destructive error. Preserve the existing local disk fallback only for genuine gateway-unavailable/offline cases. Route `AppState.syncGatewayConfigNow` through the same fresh snapshot/baseHash path or otherwise restrict/guard it so it cannot clobber unrelated `gateway.auth` or `talk.providers` data. Add Swift regression coverage proving stale config.set rejection does not direct-write a stale root, `gateway.auth` survives local token-auth gateway config saves, multi-provider `talk.providers` survives selecting/using `mlx`, and AppState gateway sync preserves unrelated gateway/auth/talk config. Do not change core gateway `config.set` baseHash semantics, do not weaken gateway auth/reload behavior, and do not edit generated protocol files unless the implementation truly requires protocol changes. Changelog is expected only if the fix changes user-visible macOS behavior; if added, use the active changelog format with reporter credit.
