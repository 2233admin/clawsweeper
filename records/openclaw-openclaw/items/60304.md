## Summary

Keep open. Current main still routes OpenAI-compatible `reasoning_content` into hidden thinking blocks and the Xiaomi plugin has no Xiaomi/MiMo normalization hook, so this PR is not obsolete; the branch needs a refresh before merge because the added tests use a removed helper path and the new stream code adds an undeclared runtime import.

## What This Changes

The PR adds a Xiaomi-only `wrapStreamFn` wrapper that converts thinking-only terminal messages for `mimo-v2-pro` and `mimo-v2-omni` into text blocks, plus provider-level regression tests for target and non-target Xiaomi models.

## Best Possible Solution

Land a current-main Xiaomi-local provider stream wrapper that uses the existing provider hook boundary, preserves non-Xiaomi and non-target Xiaomi behavior, covers both affected MiMo models and `mimo-v2-flash` no-op behavior in tests, and avoids changing the generic OpenAI-compatible parser.

## Reproduction Assessment

Yes. The linked bug report provides the provider, models, and response shape, and current main still maps `reasoning_content` to `thinking` while Xiaomi has no wrapper; a fake OpenAI-completions stream with a terminal thinking-only message reproduces the visible-blank path without live Xiaomi traffic.

## Solution Assessment

Partly. A Xiaomi-owned wrapper is the narrow maintainable direction, but this PR is not merge-ready as-is because its test helper import is stale and its new stream runtime import is not reflected in the Xiaomi package metadata or avoided via shared SDK helpers.
