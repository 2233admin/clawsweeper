## Summary

Keep open. Current main has adjacent retrieval citations, Markdown promotion source markers, memory event logs, Memory Wiki provenance, and an optional LanceDB memory plugin, but it still does not implement the requested durable-memory provenance model with required structured source references, multiple supporting sources, merge/edit lineage, or a source-backed durable record contract. Same-author #42646 and #42648 remain open for the related canonical record model and write pipeline work.

## Best Possible Solution

Keep this issue open and implement it with #42646 and #42648: add typed sourceRefs/provenance to canonical durable memory records, require at least one source before durable promotion, preserve provenance through create/merge/edit/conflict outcomes and revision history, and expose those refs consistently in retrieval and review flows. Memory Wiki can consume or mirror the model, but should not substitute for the active durable-memory provenance contract.
