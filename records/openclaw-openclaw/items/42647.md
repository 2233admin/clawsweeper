## Summary

Keep open. Current main has adjacent retrieval citations, short-term recall source markers, memory host event logs, and Memory Wiki provenance, but it still does not implement the requested durable-memory provenance model: required structured source references, multiple supporting sources per durable memory, provenance-preserving merges/edits, and a source-backed durable promotion contract remain missing.

## Best Possible Solution

Keep this issue open and implement it with #42646 and #42648: add typed sourceRefs/provenance to canonical durable memory records, require at least one source before durable promotion, preserve provenance through create/merge/edit/conflict outcomes and revision history, and expose those refs consistently in retrieval and review flows. Memory Wiki can consume or mirror that model, but should not substitute for the active durable-memory provenance contract.

## Risks / Open Questions

- Closing now would conflate retrieval citations, single Markdown source markers, operational event logs, or Memory Wiki page provenance with the stronger durable-memory provenance contract requested here.
- The requested provenance model likely needs to be implemented together with the canonical record schema and write pipeline work tracked by #42646 and #42648 so create, merge, edit, conflict, and review flows preserve source history.
- Current main does not appear to have a durable-memory promotion gate that rejects source-less records, because durable promotion still appends Markdown rather than writing typed canonical records.
