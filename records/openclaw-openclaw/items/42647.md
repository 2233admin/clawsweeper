## Summary

Keep open. Current main has adjacent retrieval citations, single Markdown promotion source markers, memory host event logs, and Memory Wiki provenance, but it still does not implement the requested durable-memory provenance model with required structured source references, multiple supporting sources, merge/edit lineage, or a source-backed durable record contract. The same-author canonical-record and write-pipeline issues remain open and should be handled with this one.

## Best Possible Solution

Keep this issue open and implement it with #42646 and #42648: add typed sourceRefs/provenance to canonical durable memory records, require at least one source before durable promotion, preserve provenance through create/merge/edit/conflict outcomes and revision history, and expose those refs consistently in retrieval and review flows. Memory Wiki can consume or mirror the model, but should not substitute for the active durable-memory provenance contract.
