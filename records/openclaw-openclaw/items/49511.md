## Summary

Keep this PR open. Current main does not implement Mermaid rendering in Control UI WebChat: Mermaid fences still go through the generic fenced-code renderer as copyable code blocks, there is no Mermaid render scheduler, and the UI package does not depend on Mermaid. The PR is a real implementation candidate, but it still needs maintainer review for the documented rendering lifecycle/accessibility concerns and the dependency/security surface.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rebase or recreate a narrow Control UI change, resolve the Mermaid lifecycle/accessibility fallback issues, decide whether Mermaid rewriting is WebChat-only or supported across all markdown consumers, complete dependency/security review, then land it or close it by explicit maintainer product decision.
