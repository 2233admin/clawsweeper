## Summary

Keep this issue open. Current main still has useful broad controls for boot context size and coarse lightweight modes, but it does not implement configurable per-file bootstrap loading by session type or file frontmatter selectors. The prior Codex review comment remains accurate on the current main SHA: the relevant source/docs/test files have no diff from that reviewed state.

## Best Possible Solution

Keep this open and design an additive core bootstrap-selection contract. A good path is agents.defaults.bootstrapFiles or named bootstrap profiles with normalized run/session kinds, optional file frontmatter support if maintainers want self-declared files, evaluation in src/agents/workspace.ts and src/agents/bootstrap-files.ts, config/schema/docs updates, and tests that preserve the blanket-load default while proving direct/group/heartbeat/subagent examples.

## Risks / Open Questions

- Existing partial mitigations such as contextInjection, heartbeat/subagent/cron lightContext, and bootstrap hooks should be preserved as additive behavior evolves.
- There is still a docs/code mismatch: docs say sub-agent context only injects AGENTS.md and TOOLS.md, while current code/tests allow SOUL.md, IDENTITY.md, and USER.md for subagent/cron sessions.
- A real implementation needs a normalized session-kind contract so direct, group/channel, heartbeat, cron, subagent, and ACP runs do not diverge across runtime paths or plugin hooks.
