## Summary

Keep this PR open. The rebased head fixes the prior route-key JSON-shape blocker and preserves raw provider message-id keys, but the new fallback still hashes an empty timestamp when `Timestamp` is absent, which can collapse distinct same-body idless messages on the same route for the dedupe TTL.

## What This Changes

The PR adds a SHA-256 content/timestamp fallback key for inbound auto-reply dedupe when `MessageSid` is absent, keeps provider-id keys unchanged, adds focused tests, and updates the changelog.

## Best Possible Solution

Keep the fallback, but require a real timestamp before producing a `content:<sha256-128>` key. Preserve the current route-key JSON shape, raw provider-id suffixes, focused tests, and the changelog entry.

## Reproduction Assessment

Yes. Current main has a source-level reproduction for the reported gap: an inbound context with provider/peer data but no `MessageSid` returns no dedupe key, and dispatch only skips `duplicate` or `inflight`. The PR regression is also reproducible with two idless same-body contexts on the same route and no `Timestamp`.

## Solution Assessment

No, not quite. The route-scoped content/timestamp fallback is the narrow maintainable direction, but the implementation should return no fallback key when the timestamp is absent so it does not become content-only dedupe.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Require timestamps for fallback fingerprints:** `src/auto-reply/reply/inbound-dedupe.ts:46-48`
  - body: `Timestamp` is optional on the inbound context, but this fallback turns a missing timestamp into the empty string before hashing. Two distinct idless messages with the same text on the same route would then share one `content:<hash>` key and the second would be skipped for the inbound-dedupe TTL. Return `null` when the timestamp is absent and add a missing-timestamp test so the fallback only dedupes when the timestamp can separate ordinary repeated text.
  - confidence: 0.82
