## Summary

Keep this PR open. Current main still does not dedupe inbound retries when `MessageSid` is absent, and the PR remains a focused implementation candidate rather than something already implemented on main. The earlier bot concern about changing provider-id keys was addressed in the PR discussion, but the patch still needs maintainer review and rebase against the current route-key implementation.

## Best Possible Solution

Keep the PR open for maintainer review. Best path: rebase it onto current main, adapt the content/timestamp fallback to the current route-key builder, preserve raw provider-id key compatibility, add or keep focused tests for idless retries and same-body/different-timestamp non-collapse, then run the targeted inbound-dedupe test plus the changed gate before merge.
