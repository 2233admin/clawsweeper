## Summary

Keep open. Current main has the session-load pending-refresh improvement, but it does not implement the central dropdown fix from this PR: the Control UI session dropdown is still populated from `sessions.list` rows plus the current session key, while `sessions.list` is still built from actual session-store entries. A configured agent with no store row can still be absent. The PR also bundles unrelated and security-sensitive changes, including Slack HTTP request buffering and a committed runtime log, so it should not be merged as-is.

## Best Possible Solution

Do not merge this branch as-is. Extract or ask for a narrow current-main patch that seeds `resolveSessionOptionGroups` from `state.agentsList` using each agent’s main session key, or otherwise makes `sessions.list` expose configured agent main sessions without creating misleading stored history. Keep the focused regression tests for the no-session-row case, and drop the unrelated Slack, json-file retry, and committed log changes.
