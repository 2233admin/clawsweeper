## Summary

Keep this issue open. Current main still does not expose a configurable session-end agent turn: manual reset and idle/daily rollover rotate/archive the old session and emit lifecycle hooks, while the existing silent embedded agent turn remains scoped to pre-compaction memory flush. The reporter's follow-up is useful prior art, but applying it to session end needs maintainer design around opt-in config, bounded execution, recursion prevention, delivery suppression, model/tool policy, and transcript preservation.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is a generic, opt-in core lifecycle feature for a bounded silent final turn before manual reset and idle/daily rollover archive the previous transcript, likely reusing pieces of the existing pre-compaction memory-flush runner while adding config schema/help/docs and focused tests for ordering, timeout, no recursion, delivery suppression, transcript preservation, and tool/sandbox behavior.

## Risks / Open Questions

- A session-end agent turn with tool access is security- and cost-sensitive; it needs explicit policy for opt-in config, timeout/cancellation, model fallback, sandbox/tool permissions, recursion prevention, and outbound delivery suppression.
- The current `session-memory` hook is useful prior art for `/new` and `/reset`, but it is not equivalent to a configurable full-history final agent turn and does not by itself cover idle/daily rollover semantics.
