## Summary

Current main does not implement the requested configurable session-end agent turn. It has lifecycle hooks and a compaction-only silent memory flush, but resets and stale rollovers still replace/archive the session and emit observation hooks without an awaited final model/tool pass.

## Best Possible Solution

Keep this issue open. The best path is a generic core lifecycle design for an opt-in, bounded final session turn that works across manual resets and idle/daily rollover, reuses the existing agent runner safely, documents config and delivery behavior, and adds tests for ordering, timeout, no-recursion, and transcript preservation.
