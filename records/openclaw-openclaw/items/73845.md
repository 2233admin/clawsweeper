## Summary

Keep this issue open. Current main still has observation-only session lifecycle hooks and a compaction-only memory flush, but it does not implement a configurable final agent turn before `/new`, `/reset`, idle/daily rollover, or session archival.

## Best Possible Solution

Keep this issue open and design a generic, opt-in core lifecycle feature for a bounded final session-end agent turn. It should cover manual reset and idle/daily rollover, define whether compaction participates separately from the existing memory flush, reuse the existing agent runner safely, document config and delivery behavior, and add tests for ordering, timeout, no-recursion, tool policy, and transcript preservation.

## Risks / Open Questions

- A tool-enabled final agent turn during reset is cost- and security-sensitive; it needs explicit opt-in, timeout/cancellation behavior, no-recursion guards, delivery suppression, and clear sandbox/tool policy.
- Transcript ordering matters: the final turn would need to run before archival/reset while preserving current transcript bytes and avoiding duplicate session hooks or stale session state.
