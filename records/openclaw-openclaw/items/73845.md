## Summary

Keep this open. Current main still lacks a configurable session-end agent turn: the session config/schema have no `onEnd` or `agentTurn` field, reset and rollover paths archive/replace sessions while emitting only void lifecycle hooks, and the existing silent agentic memory flush remains scoped to pre-compaction. The request is valid, but it needs maintainer design because a final tool-capable turn affects cost, permissions, recursion, timeout behavior, transcript ordering, and delivery suppression.

## What This Changes

The issue requests a configurable `sessions.onEnd.agentTurn`-style feature that lets the agent run one final cleanup turn with conversation history and tool access before manual reset, idle timeout, or daily rollover archives the old session.

## Best Possible Solution

Add a generic, opt-in core lifecycle feature for a bounded silent final turn before manual reset and idle/daily rollover archive the outgoing transcript, reusing the compaction memory-flush runner where possible while adding typed config/schema/docs and regression tests for ordering, timeout, no recursion, delivery suppression, transcript preservation, and tool/sandbox behavior.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence verification path for the missing behavior: current main's config/schema reject the proposed setting, reset/rollover paths only emit void hooks, and the only silent agentic memory flush is called from the compaction path.

## Solution Assessment

Unclear until maintainer design. The current request points at the right precedent, but the safest solution is a bounded opt-in final-turn contract rather than enabling an unconditional full-tool agent run at every session boundary.
