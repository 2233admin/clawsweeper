## Summary

Keep open. Current main still lacks selective shared sessions across chosen group/channel/thread chats: routing builds isolated non-direct session keys, thread suffixing remains default, and the strict session config/types/docs expose only existing DM-centric controls rather than channel group mappings.

## Best Possible Solution

Keep this open as the canonical tracker for selective shared sessions across chosen group/channel/thread chats. The best path is an additive config design that preserves isolated defaults, maps explicit channel/thread identifiers to a shared session key, keeps replies routed to the originating channel, serializes grouped messages through that shared key, defines optional thread inheritance, and updates schema/types/help/docs plus routing/thread/sandbox regression coverage. Treat #7524, #43286, and #51805 as related design inputs rather than replacements.
