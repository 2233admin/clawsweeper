## Summary

Keep open. Current main still lacks an opt-in session grouping surface for selected Discord/Slack channels or threads; non-DM peers are still keyed by channel plus peer id, Slack threads still fork by thread timestamp, and the strict session schema/docs expose no channelGroups/thread-inheritance config for this request. The closed implementation PR was not merged, and the related open issues are useful design inputs rather than replacements.

## What This Changes

The issue requests opt-in shared session groups that route selected Discord/Slack channel/thread chats for the same agent into one session while preserving isolated defaults and reply routing.

## Best Possible Solution

Keep this open as the canonical tracker for selective shared sessions across chosen group/channel/thread chats. The best path is an additive config design with isolated defaults, explicit channel/thread membership, optional thread inheritance, reply routing to the originating channel, grouped queue serialization, schema/types/help/docs updates, and regression coverage across routing, Discord, Slack, and sandbox-sensitive session behavior. Treat #7524, #43286, #51805, and closed PR #20078 as design inputs.
