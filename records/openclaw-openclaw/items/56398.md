## Summary

Keep open. The reported exec-approval bug is still present on current main in the shared pending-approval helper, while this PR is useful but stale against the refactored current path and overlaps the broader open #56581 consolidation.

## What This Changes

This PR adds a guard so two-phase exec approval requests are not fast-failed when no approval client or forwarding target exists, and adds gateway server-method tests for two-phase pending versus single-phase fast-fail behavior.

## Best Possible Solution

Keep the intended behavior on current main by adding the equivalent two-phase no-route guard in `handlePendingApprovalRequest`, preserving single-phase fast-fail, and adding regression coverage for a two-phase exec approval with no approval clients, no forwarding delivery, and no turn-source route. Maintainers should decide whether that lands as a narrow adapted patch or through #56581.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `exec.approval.request` with `twoPhase: true`, no approval clients, no accepted delivery, and no turn-source channel; current main reaches the shared no-route expiry branch before the two-phase `accepted` response.

## Solution Assessment

Yes for the original base, but not as-is for current main. The proposed guard is the right narrow behavior change, but after the refactor it belongs in `handlePendingApprovalRequest` with regression coverage rather than only in the older `exec-approval.ts` branch.
