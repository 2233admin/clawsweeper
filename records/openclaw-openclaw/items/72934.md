## Summary

Keep open. The report is recent, concrete, and current main still has a source-backed browser snapshot timeout gap: the browser tool advertises `timeoutMs`, and gateway/node proxy layers can honor it when supplied, but the snapshot action, host snapshot client, and `/snapshot` route do not consistently carry a caller timeout through the snapshot execution path.

## What This Changes

The issue asks OpenClaw to make browser snapshot return page content/A11y output instead of timing out after browser open or navigate succeeds.

## Best Possible Solution

Repair the bundled browser plugin snapshot path so caller timeouts and cancellation are carried consistently from the agent tool through host client, gateway `browser.request`, node proxy, and `/snapshot` execution, with focused regression tests for local and node-proxied snapshots.

## Reproduction Assessment

Yes for triage: the report gives a concrete open/navigate/snapshot sequence, affected version, platform, and repeated cross-URL failure. Current-main source inspection independently shows the matching timeout plumbing gap, although I did not run a live macOS browser session.

## Solution Assessment

Yes. The narrow maintainable fix is to patch snapshot timeout/cancellation plumbing inside the browser plugin; a broader gateway rewrite or profile default change would go beyond the evidence.

## Repair Work Prompt

Fix the browser snapshot timeout regression from https://github.com/openclaw/openclaw/issues/72934. Observable bug: browser open/navigate succeeds, but `browser({ action: "snapshot", targetId, timeoutMs })` can time out because the snapshot path drops caller timeout/cancellation while screenshot and gateway/node proxy paths already support it. Keep the fix inside the bundled browser plugin unless a tiny generic seam is absolutely required. Expected boundary: carry normalized `timeoutMs` from `extensions/browser/src/browser-tool.ts` / `executeSnapshotAction` into local `browserSnapshot` and node `proxyRequest`, allow `browserSnapshot` to pass the timeout to `fetchBrowserJson` and `/snapshot`, and make the `/snapshot` route apply the timeout to Playwright/Chrome MCP/CDP snapshot and labels/screenshot work where supported. Preserve existing defaults and existing-session unsupported-timeout rules; do not change browser profile defaults, SSRF policy, core gateway ownership, or unrelated navigation behavior. Add focused regression coverage for local and node-proxied snapshot timeout forwarding, plus host client and route-level behavior.
