## Summary

Keep open. The report is recent, concrete, and current main still has a source-backed snapshot timeout gap: `timeoutMs` is exposed to callers and already works in gateway/proxy plumbing when supplied, but the snapshot action and host client do not pass it through to `/snapshot`.

## What This Changes

The issue reports that browser `open`/navigation succeeds but `snapshot` consistently times out on macOS OpenClaw 2026.4.25, preventing browser automation from returning page content or the accessibility tree.

## Best Possible Solution

Keep this open and repair the browser plugin snapshot path so caller `timeoutMs` and cancellation are carried consistently from the agent tool through host client, gateway `browser.request`, node proxy, and `/snapshot` execution, with focused regression coverage.
