## Summary

Keep open. Current main still has a source-backed browser snapshot timeout propagation gap: snapshot is a documented browser surface and the tool schema accepts `timeoutMs`, but the snapshot action, host client, and `/snapshot` route do not carry that timeout into the actual snapshot work.

## What This Changes

The issue reports that `browser` open/navigate succeeds but `browser` snapshot consistently times out on macOS OpenClaw 2026.4.25.

## Best Possible Solution

Normalize a snapshot timeout once, then propagate it through the browser tool snapshot action, host `browserSnapshot` client, node proxy request, `/snapshot` route plan, Chrome MCP `take_snapshot`, and Playwright/CDP snapshot helpers, with focused regression coverage for host, route, existing-session, and node-proxy paths.

## Reproduction Assessment

Yes for triage: the issue gives a concrete open-then-snapshot sequence, affected version, platform, and repeated cross-URL failures. Current-main source inspection independently confirms that a caller-provided snapshot timeout cannot affect the `/snapshot` route or inner snapshot calls.

## Solution Assessment

Yes: a narrow browser plugin timeout propagation fix is the maintainable path. A core gateway rewrite or broad browser profile migration would be larger than the evidence supports.

## Repair Work Prompt

Fix the browser snapshot timeout propagation bug reported in https://github.com/openclaw/openclaw/issues/72934. The observable bug is that `browser` open/navigate can succeed, but `browser` snapshot times out because `timeoutMs` is accepted by the tool schema but is not consistently propagated through the snapshot stack. Keep the change inside the bundled browser plugin unless a small generic helper is already local to that plugin. Normalize the snapshot timeout once and propagate it from `executeSnapshotAction` through host `browserSnapshot`, node proxy `/snapshot`, the `/snapshot` route/plan, Chrome MCP `take_snapshot`, and Playwright/CDP snapshot helpers. Add regression tests that prove host snapshot, node proxy snapshot, existing-session snapshot, and inner route calls receive the timeout budget. Do not broaden the fix into navigate behavior; #47203 tracks navigate-specific timeout behavior separately. Changelog: add a user-facing fix entry if repo policy requires it for browser tool timeout behavior, credited to the reporter if appropriate and without forbidden bot/maintainer thanks handles.
