## Summary

Keep open: current main still rejects current Buildx Docker attestation manifests that omit `artifactType`, and this PR is a narrow implementation with matching tests and successful PR checks. The right cleanup action is maintainer review and landing, not closing or replacing it.

## What This Changes

The PR updates the Docker attestation verifier to accept no-`artifactType` manifests when all layers are in-toto JSON, and adds Vitest coverage for the current Buildx shape plus a malformed-manifest rejection case.

## Best Possible Solution

Land this PR or an equivalent focused verifier update that accepts the current Buildx OCI-manifest-with-in-toto-layers shape while preserving legacy `artifactType` support, descriptor prefiltering, required SBOM/provenance predicate checks, and malformed-manifest diagnostics.

## Solution Assessment

Yes. This is the narrowest maintainable fix I found: it updates attestation-shape recognition at the existing verifier boundary without changing Docker release workflow permissions, publication behavior, CLI parsing, or predicate requirements.
