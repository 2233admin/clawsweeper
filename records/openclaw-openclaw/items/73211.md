## Summary

Keep open. Current main still has the Mattermost preview-streaming mismatch: docs advertise `channels.mattermost.streaming` and `off`, but the Mattermost schema/types do not expose that config and inbound replies still create and update the draft preview stream without resolving a streaming mode. The linked open PR #73252 uses closing syntax and is the active implementation path, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a Mattermost configuration path to opt out of draft-preview streaming so users can disable placeholder post creation and in-place post updates.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts the canonical `streaming.mode` config at channel and account scope, defaults to today’s preview behavior, disables draft-preview callbacks and in-place finalization when mode is `off`, and updates tests/docs/generated config metadata so the public contract matches runtime behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main’s strict Mattermost schema has no `streaming` key, while the monitor unconditionally creates `createMattermostDraftStream` and wires partial/tool/reasoning updates into it.

## Solution Assessment

Yes. The best path is the linked PR direction because it reuses the existing channel streaming config contract instead of adding a Mattermost-only `draftPreview` alias; maintainers still need to review and land or replace that PR.
