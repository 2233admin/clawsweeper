## Summary

Keep open. Current main still has the Mattermost preview-streaming contract mismatch: the docs advertise `channels.mattermost.streaming` with an `off` mode, but the strict Mattermost schema/types/generated metadata do not expose that key, and inbound replies still create and update the draft preview stream without resolving a preview streaming mode. The open linked PR #73252 remains the active implementation path, so this issue should stay open until that or an equivalent fix lands.

## What This Changes

Request a Mattermost configuration path to opt out of draft-preview streaming and align the schema, runtime behavior, generated metadata, and docs with the documented streaming modes.

## Best Possible Solution

Land #73252 or an equivalent Mattermost-local fix that accepts canonical `streaming.mode` config at channel and account scope, defaults omitted config to today’s preview behavior, disables draft-preview callbacks and in-place finalization when mode is `off`, and updates tests/docs/generated config metadata so the public contract matches runtime behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main’s strict Mattermost schema and generated config metadata lack `streaming`, while the inbound monitor unconditionally creates `createMattermostDraftStream` and routes partial/tool/reasoning updates into it.

## Solution Assessment

Yes. Reusing the existing channel `streaming.mode` contract is the narrowest maintainable fix; adding a Mattermost-only `draftPreview` public shape would duplicate a config concept the SDK already models.
