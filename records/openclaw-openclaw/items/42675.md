## Summary

Keep this PR open. The reported Discord startup failure is still present on current `main`, and the linked security/regression tracker #42219 remains open. The PR is directionally useful, but it is stale against the current plugin layout and incomplete unless the port also rejects a resolved identity with no usable `id`.

## Best Possible Solution

Keep this PR open as a useful implementation candidate. The best maintainer path is to port the fail-fast behavior into `extensions/discord/src/monitor/provider.startup.ts`, validate that the returned bot identity has a usable `id`, add focused regression tests for rejected and invalid identity lookups, and then resolve #42219 through the merged fix or close this PR as superseded by a newer complete implementation.
