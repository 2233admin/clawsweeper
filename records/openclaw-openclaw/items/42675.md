## Summary

Keep this PR open. Current main still fails open when Discord bot identity fetch rejects or resolves without a usable id, and the PR references the still-open security/regression tracker #42219 with closing syntax. The submitted patch is stale and incomplete, but it remains a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and ask for a current-main rewrite in `extensions/discord/src/monitor/provider.startup.ts` so Discord startup fails when `fetchUser("@me")` rejects or returns no usable bot id. Add focused regression tests for rejected and invalid identity responses, then consider the defense-in-depth mention-gate cleanup tracked by #42219.
