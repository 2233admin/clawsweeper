## Summary

Keep this PR open. Current main still fails open when Discord bot identity fetch rejects or resolves without a usable id, and the PR is tied to the still-open security/regression tracker #42219. The submitted patch is stale and incomplete, but it remains a focused implementation candidate rather than work that current main has already shipped.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a current-main rewrite in `extensions/discord/src/monitor/provider.startup.ts`: fail Discord startup when `fetchUser("@me")` rejects or returns no normalized bot id, add focused regression tests for both cases, and then handle any defense-in-depth mention-gate cleanup under the linked security/regression tracker.
