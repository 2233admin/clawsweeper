## Summary

Keep open. Current main still does not implement WhatsApp `edit` or `unsend`, and the PR remains useful source work for a real WhatsApp-plugin-owned capability that overlaps open issue #14344. It should not be closed as cleanup; it needs maintainer/security review before merge because it adds agent-visible message mutation over the existing Baileys session.

## What This Changes

The PR adds WhatsApp message `edit` and `unsend` actions by extending WhatsApp action discovery/dispatch, the active listener send API, Baileys edit/delete payload handling, capability metadata, and focused tests.

## Best Possible Solution

Land or revise a WhatsApp-plugin-owned implementation that advertises `edit` and `unsend` only when sends are enabled, dispatches through the existing message-action/gateway surface, reuses outbound authorization, builds own-message Baileys keys safely, adds source and docs coverage, and resolves the outstanding security/review comments before merge.
