## Summary

Keep this PR open. Current main still has the telemetry emitter gap described in the PR: isolated cron runs persist cacheRead/cacheWrite on the session entry, but the usage object returned for cron run-log telemetry only contains input_tokens, output_tokens, and total_tokens. The PR remains a narrow functional fix candidate, and the supplied diff does not introduce CI, dependency, install, release, or secret-handling changes.

## Best Possible Solution

Keep this PR open and review or land a port of its narrow production change, preferably with a focused isolated cron regression test that exercises an agent run with positive cacheRead/cacheWrite and asserts the returned cron telemetry, finished event, or run-log entry includes cache_read_tokens and cache_write_tokens.
