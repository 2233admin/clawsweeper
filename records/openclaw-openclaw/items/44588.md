## Summary

Keep this PR open. The checked-out main at b85edb3 still has the telemetry gap: isolated cron copies cacheRead/cacheWrite into the session entry, but the `CronRunTelemetry.usage` object that flows into JSONL run logs only gets input/output/total tokens. The PR is a narrow one-file implementation candidate, and the supplied diff does not show security-sensitive or unrelated changes.

## Best Possible Solution

Keep this PR open and review/land the narrow production change, ideally with a focused regression test on the isolated cron run result or run-log write path proving positive `usage.cacheRead` and `usage.cacheWrite` become `cache_read_tokens` and `cache_write_tokens` in emitted cron telemetry. No docs or changelog entry looks necessary for this internal telemetry bug fix unless maintainers want a user-facing cost-accounting note.
