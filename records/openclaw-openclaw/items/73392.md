## Summary

Keep open. Current main still writes LINE inbound media to OpenClaw temp storage and its tests assert that temp-path behavior, while this PR is a focused candidate to move the active LINE media path to the existing `saveMediaBuffer(..., "inbound", ...)` contract that WhatsApp already uses. The linked #73370 remains open, and the provided PR diff does not appear obsolete or unrelated.

## Best Possible Solution

Keep this PR open as a current implementation candidate for #73370. The best maintainer path is to review the narrow LINE runtime/test diff, run targeted validation, and land this PR or an equivalent change that persists LINE inbound media through `saveMediaBuffer(..., "inbound", ...)` while preserving the existing streaming size guard.
