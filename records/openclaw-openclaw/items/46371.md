## Summary

Keep open for maintainer review. Current main still does not implement the PR's two central changes, and the PR branch is not merge-ready because the OAuth half targets the removed `extensions/google-gemini-cli-auth` path while the active provider lives under `extensions/google`; the new OAuth test also leaks `PATH`, and the APPDATA fallback changes the OAuth credential-source trust boundary.

## What This Changes

This PR adds a Windows APPDATA npm-global fallback for Gemini CLI OAuth credential discovery in the old google-gemini-cli-auth plugin and adds opt-in `WORKING.md` workspace bootstrap loading with tests.

## Best Possible Solution

Split this into two focused current-main changes: port any accepted Windows Gemini CLI OAuth fallback to `extensions/google/oauth.credentials.ts` with narrow Windows tests and docs, and handle `WORKING.md` as a separate agent-bootstrap proposal that aligns loader behavior, session filtering, templates, and memory documentation.

## Security Review

Status: needs_attention

Summary: The PR changes the OAuth credential-source selection path and should receive maintainer security review before any replacement lands.

Concerns:

- **[medium] Review APPDATA OAuth credential trust boundary:** `extensions/google-gemini-cli-auth/oauth.ts:84`
  - body: The proposed fallback reads Gemini CLI OAuth constants from `%APPDATA%\npm\node_modules\@google\gemini-cli` even when no `gemini` executable was found on `PATH`. That broadens credential-source selection to a user-writable package tree not anchored to the executable OpenClaw discovered, so maintainers should validate it against the Gemini CLI OAuth trust model before accepting it.
  - confidence: 0.78
