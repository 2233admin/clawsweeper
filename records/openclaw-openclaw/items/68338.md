## Summary

Keep open. Current main still keeps an empty `MEDIA:` line as text instead of consuming the following media path, so this PR’s central behavior is not implemented. The PR is narrow and the follow-up commit addresses the bot review’s greedy-continuation concern, but it still needs normal maintainer review against current main.

## What This Changes

The PR changes `src/media/parse.ts` and `src/media/parse.test.ts` to parse an empty `MEDIA:` directive as using the immediately following line as a single media path while preserving surrounding ordered text/media segments.

## Best Possible Solution

Review the PR against current main, keep the change in the shared media parser if the directive grammar is accepted, and require focused parser tests for the path-on-next-line case, one-line-only consumption, fenced-block avoidance, and existing validation rules.
