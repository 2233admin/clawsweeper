## Summary

Keep this issue open. The reporter’s follow-up identified block streaming as a workaround, but current main still leaves Discord preview streaming off by default and only enables block replies when the global agent default or a Discord channel override is set. The current delivery code also explicitly buffers text-only block replies when block streaming is disabled, so the reported default Discord ordering problem is still plausible and not superseded by the related closed issues.

## Best Possible Solution

Keep this open for a Discord-owned fix. The likely path is either to make Discord preserve production order by flushing text before tool execution, or to add a Discord-specific default/override that enables an ordering-safe delivery mode without reintroducing the paragraph fragmentation covered by #57875. The change belongs in the Discord channel delivery/streaming path plus focused tests around interleaved text and tool boundaries.
