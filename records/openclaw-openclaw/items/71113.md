The PR adds distinct CI guard work that is not on current main, but the proposed patch is not merge-ready because its new path normalizer does not normalize Windows-style backslashes on Linux, which makes the added exemption test fail. The check-additional integration point is otherwise plausible, and the security pass found no concrete supply-chain concern.

## What This Changes

The PR adds an AGENTS.md line-count guard, a package script, check-additional boundary-shard wiring, and tests for oversized files, exemptions, sorting, and the default cap.

## Best Possible Solution

Fix the cross-platform path normalization, then validate the targeted test and size guard before maintainer review decides whether the 150-line policy and exemptions should land in check-additional.

## Reproduction Assessment

Yes for the review finding. On Linux, `path.sep` is `/`, so the proposed `split(path.sep)` normalizer leaves `docs\reference\templates\AGENTS.md` unchanged and the new Windows-style exemption assertion fails.

## Solution Assessment

No as proposed. The design is a narrow maintainable guard, but the implementation should normalize both `\` and `/` separators before comparing exemption paths.
