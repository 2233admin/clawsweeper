## Summary

Keep open. Current main still lacks the AGENTS.md size guard, and this draft PR is the active implementation path, but the patch is not merge-ready because its path normalization only handles the host separator and breaks its own Windows-style exemption test on Linux.

## What This Changes

Adds a TypeScript AGENTS.md line-count guard, wires it into the check-additional boundary runner and package scripts, and adds Vitest coverage for oversized files, exemptions, sorting, and path normalization.

## Best Possible Solution

Keep the PR open, fix the cross-platform path normalization helper, then have maintainers decide whether this AGENTS.md cap and exemption policy should become part of the check-additional gate.

## Reproduction Assessment

Yes. On a Linux host, the PR's normalizePath expression leaves backslashes intact, so the added Windows-style exemption assertion would fail; I verified that with a read-only Node expression rather than running Vitest in this review.

## Solution Assessment

No. The direction is narrow and plausible, but this exact patch should normalize both slash styles independent of host OS before it is considered for merge.
