## Summary

Keep open. Current main still does not implement the XML `</arg_value>` suffix cleanup this PR is trying to add, and the linked bug #48780 remains open. The PR is still a plausible implementation candidate, but it needs a current-main rework because its proposed `normalizeToolParams()` integration point is gone and review comments identify real data-integrity gaps.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up, but require a current-main rework: add a narrowly scoped sanitizer at the active tool-argument boundary for command/path-like fields, cover read path and exec string params intentionally, and add regression tests for Qwen/DashScope suffixes plus negative tests proving `content`, `oldText`, and `newText` are preserved when they legitimately end with XML-like text.

## Risks / Open Questions

- Closing this PR now would leave the linked Qwen/DashScope `</arg_value>` tool-argument corruption unimplemented on current main.

- The implementation needs a rebase because `normalizeToolParams()` no longer exists on current main.
- Exec handling should deliberately cover `command`, `workdir`, `host`, `security`, `ask`, and `node`, or document why a field is excluded.
