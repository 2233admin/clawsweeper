## Summary

Keep open. Current main does not implement the `</arg_value>` suffix stripping requested by this PR, the linked bug remains open, and the PR is still a plausible implementation candidate. It should not be closed as cleanup, but the exact patch needs rework because current main no longer has the `normalizeToolParams` layer and the PR review identified real data-corruption and coverage gaps.

## Best Possible Solution

Keep the PR open, but require a rebase/redesign against current main: implement a targeted sanitizer at the current tool argument boundary for command/path-like fields, cover exec string parameters deliberately, and add regression tests for Qwen/DashScope suffixes plus negative tests proving write/edit content such as `content`, `oldText`, and `newText` is not silently truncated.
