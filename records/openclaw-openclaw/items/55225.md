## Summary

Keep this PR open. Current main still lacks Feishu recalled-message handling, so the contribution targets a real missing behavior, but the proposed patch does not yet satisfy the central acceptance criteria because recalled-before-response sends can still fall back to direct messages and media replies are not mapped for deletion.

## What This Changes

The PR adds a Feishu recalled-message event handler, an in-memory user-message to bot-reply map with TTL and retry retention, send-result reply-path tracking, and text/card reply registration for recall cleanup.

## Best Possible Solution

Revise the PR so recalled-before-response suppression runs before any fallback direct send, all reply-path outputs that should be cleaned up are mapped or explicitly documented out of scope, and focused Feishu tests cover the recall event, mapping, idempotency, retry retention, fallback-direct suppression, and media behavior.
