## Summary

Keep open. Current main still masks the OpenAI Codex device-pairing code in remote/SSH mode, while this PR is a narrow implementation that surfaces the code through the interactive prompter and preserves the persistent runtime-log boundary.

## What This Changes

The PR changes OpenAI Codex device-code auth to always show the user code in the prompter note and refactors the provider test to separately assert prompter visibility and runtime-log omission.

## Best Possible Solution

Review and land this PR, or an equivalent OpenAI plugin-owned fix, so remote/SSH `openclaw models auth login --provider openai-codex` users can read the device-pairing code in the terminal prompter while `runtime.log` continues to omit it. Add a user-facing changelog entry, then close #74212 and treat closed PR #74221 as superseded by the prompter-only/log-safe implementation.
