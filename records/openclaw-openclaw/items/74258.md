## Summary

Keep open. Current main still masks the OpenAI Codex device-pairing code in remote/SSH mode, while this PR is a narrow valid fix that exposes the code only through the interactive prompter and preserves the persistent runtime-log boundary.

## What This Changes

This PR changes the OpenAI Codex device-code auth flow to always show the user code in the interactive prompter note, and splits the remote-mode test into prompter-surface and runtime-log guard tests.

## Best Possible Solution

Land this OpenAI plugin-owned prompter-only fix, or an equivalent patch, with a single-line changelog entry and CI proof; keep device codes out of persistent runtime logs, close #74212 after merge, and leave #74221 superseded.

## Reproduction Assessment

Yes. Current main has a direct unit-level reproduction: invoke the device-code auth method with `isRemote: true` and the prompter note contains the placeholder while the real `userCode` is available in the callback. The linked bug report also gives concrete SSH steps on OpenClaw 2026.4.23.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix because it changes only the OpenAI plugin's interactive prompter surface, preserves the persistent log boundary, and adds focused tests for both invariants; it still needs changelog and CI before merge.
