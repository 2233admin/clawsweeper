## Summary

Keep this PR open. Current main still has the Control UI local `/model` executor treating any non-empty argument as a model selection, so `/model status`, `/model list`, and `/model info` still go to `sessions.patch` instead of the info/list/status paths. The linked bug #46894 remains a valid implementation target, and the PR is not obsolete on main.

## Best Possible Solution

Keep this PR open for maintainer review or contributor refresh. The best path is to update the Control UI local `/model` handler and focused tests so `/model list` and `/model status` follow the shared directive semantics without regressing legitimate model IDs, then land this PR or replace it with a narrow fresh PR that closes #46894.

## Risks / Open Questions

- The PR’s current reserved-word approach would make exact model IDs named `status`, `list`, or `info` unselectable in the Control UI. Current model catalog entries accept arbitrary string IDs, so maintainer review should preserve legitimate model selection while fixing the command ambiguity.
- The PR maps `status`, `list`, and `info` to the bare model-info display, but current shared directive behavior treats `/model list` and `/model status` differently. The Control UI fix should align with those semantics rather than adding a second divergent command model.
