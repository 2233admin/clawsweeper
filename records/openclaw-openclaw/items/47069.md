## Summary

Keep this PR open, but do not merge it as-is. The underlying Control UI bug still exists on current main, while the proposed fix collapses `/model list` and `/model status` into the bare summary path instead of preserving the documented shared semantics.

## What This Changes

The PR adds a Control UI `MODEL_INFO_SUBCOMMANDS` set for `status`, `list`, and `info`, routes those arguments to the existing model-info display path, and adds regression tests for those aliases plus a model-setting case.

## Best Possible Solution

Refresh the Control UI local `/model` implementation so it matches the shared slash-command contract: bare `/model` shows the summary, `/model list` follows the model-browser/list semantics, `/model status` shows detailed model/auth/endpoint status, normal model selection continues to patch the session, and `info` is either documented across surfaces or left available as a normal model id.
