## Summary

Keep this PR open. Current main still routes Control UI `/model status`, `/model list`, and `/model info` through the local Web UI executor as non-empty model arguments, so the PR's intended bug fix is not implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review or contributor refresh. The best path is to update the Control UI local `/model` handler and tests so `/model list` and `/model status` follow the current shared directive semantics without regressing legitimate model selection, then land this PR or replace it with a narrower fresh PR that closes #46894.
