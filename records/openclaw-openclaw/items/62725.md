## Summary

Keep open. Current main still does not accept or apply `typingMode` or `typingIntervalSeconds` on `agents.list[]`: the per-agent schema is strict and lacks both fields, the per-agent TypeScript config type lacks both fields, and the reply/heartbeat paths still resolve typing settings from `agents.defaults` plus `session` only.

## What This Changes

The issue requests per-agent `typingMode` and `typingIntervalSeconds` overrides in `agents.list[]` entries.

## Best Possible Solution

Add a narrow core config/runtime change that accepts `typingMode` and `typingIntervalSeconds` in `agents.list[]`, resolves them for the selected agent with explicit precedence, and updates generated schema, docs, and focused tests so the config contract and reply/heartbeat behavior stay aligned.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: an `agents.list[]` entry with `typingMode` or `typingIntervalSeconds` reaches strict `AgentEntrySchema`, which does not define those keys, matching the reported unrecognized-key failure.

## Solution Assessment

Yes. This is best solved in the core agent config and reply/heartbeat resolution paths because typing indicators are already a core behavior with global and session settings; no plugin or ClawHub boundary is needed.
