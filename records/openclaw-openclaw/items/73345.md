## Summary

Keep this PR open. Current main still does not implement the intended `/hooks/agent` thread propagation: the hook payload type/normalizer omit `thread`, the gateway dispatcher builds `CronJob.delivery` without `threadId`, and the replay scope has no thread dimension. The underlying cron delivery path already supports `delivery.threadId`, so the PR is a plausible narrow adapter change rather than obsolete. Security review found no workflow, dependency, lockfile, package script, permission, or other supply-chain-sensitive file changes in the provided PR diff.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow gateway adapter change if `/hooks/agent` should expose the existing cron `delivery.threadId` capability, with follow-up adjustments in the PR for replay-key scope, public webhook docs, and the exact accepted thread value type.
