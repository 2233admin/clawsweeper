## Summary

Keep this PR open. Current main does not implement the requested /hooks/agent thread propagation: the gateway payload type and normalizer do not carry a thread field, and dispatchAgentHook builds CronJob.delivery without threadId. The underlying cron delivery contract already supports delivery.threadId, so the PR is plausibly useful rather than obsolete. Security review found no CI, dependency, script, permission, or supply-chain-sensitive changes in the provided diff, but the PR should account for thread in the hook idempotency/replay scope before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the narrow gateway adapter change if /hooks/agent should expose the existing CronJob.delivery.threadId capability, with a small follow-up in the PR to include thread in the replay-key scope and document the new webhook field.
