## Summary

Keep this PR open. Current main still does not parse a `thread` field from `/hooks/agent` payloads or propagate it into `CronJob.delivery.threadId`, while the cron delivery layer already supports `threadId`. The PR remains a plausible focused gateway adapter change, not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow gateway adapter change if `/hooks/agent` should expose the existing cron `delivery.threadId` capability, with regression coverage, docs updates, replay-key scoping for `thread`, and an explicit accepted value type.
