## Summary

Keep open. This PR is maintainer-authored and carries the protected `maintainer` label, so it needs explicit maintainer judgment. Current `main` also does not already implement the PR's forced-retirement recovery path: the health monitor still calls `stopChannel` without options, the `ChannelManager.stopChannel` API still has no options parameter, and timeout handling still preserves the wedged task sentinel rather than retiring it.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review the latest head against the lifecycle-overlap concern, confirm the OpenClaw-managed cleanup and stale-callback gating are sufficient, run targeted gateway tests plus the normal changed gate if landing, and merge only after maintainers accept forced retirement as the right recovery behavior.
