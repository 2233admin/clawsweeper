## Summary

Keep open. Current main still emits the hardcoded personal-assistant identity before SOUL.md project context, and the supported config surface is still full system prompt replacement rather than a focused identity or preamble override. The related open PR #48442 remains the active implementation candidate, while #44497 was closed unmerged.

## What This Changes

The issue requests a configurable agent identity/preamble so custom SOUL.md personas are not overridden by the hardcoded personal-assistant line at the start of the system prompt.

## Best Possible Solution

Review and land #48442 or a narrow successor that adds a documented identity-line/preamble configuration path while preserving current default output and full systemPromptOverride behavior, with focused coverage for configured overrides, SOUL.md ordering, promptMode none, and docs/schema alignment.

## Reproduction Assessment

Yes for the prompt-construction issue: current source deterministically emits the personal-assistant sentence before Project Context, and tests assert SOUL.md guidance in that lower section. I did not run live model calls in this read-only review, but the issue discussion describes the same observable behavior.

## Solution Assessment

Unclear until maintainers choose the API shape. A narrow configurable identity line is more maintainable than forcing users to replace the entire system prompt, while #48442’s broader disabledSections addition needs normal product/security review.
