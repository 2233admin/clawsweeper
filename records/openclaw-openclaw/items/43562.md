## Summary

Keep open. Current main still starts the assembled agent system prompt with the hardcoded personal-assistant identity, still places SOUL.md guidance later under Project Context, and still exposes only full system prompt replacement rather than a focused identity-line or preamble setting. The related open PR #48442 is the right implementation candidate to review before this issue can close; prior PR #44497 was closed unmerged.

## What This Changes

This issue requests a configurable top-of-system-prompt agent identity preamble, with the existing personal-assistant text as fallback or neutral SOUL.md-aware behavior when workspace identity files exist.

## Best Possible Solution

Review #48442 or a narrow successor that adds a documented identity-line configuration path with clear precedence and focused tests for default behavior, default and per-agent overrides, SOUL.md interaction, promptMode none, and full prompt replacement compatibility.
