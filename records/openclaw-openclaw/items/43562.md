## Summary

Keep this issue open. Current main still hardcodes the leading system-prompt identity as `You are a personal assistant running inside OpenClaw.`, still places SOUL.md later under Project Context with wording that defers to higher-priority instructions, and still lacks a focused `preamble`, `identityLine`, or equivalent config field. The only shipped config workaround is `systemPromptOverride`, which replaces the entire assembled prompt rather than the identity line.

## Best Possible Solution

Keep the issue open. The best maintainer path is to review and either land or close the open identity-line PR, or implement a focused core config surface for the leading identity/preamble line with clear default and per-agent precedence. That change should update config types, Zod/generated schema, docs, config mutation allowlists if user-editable, embedded/compaction prompt construction paths, and focused tests for SOUL.md ordering plus per-agent/default precedence.
