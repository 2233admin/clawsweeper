## Summary

Keep this PR open. Current main still does not implement the central outbound metadata route, and the PR remains a relevant implementation attempt, but it is not ready to land unchanged because the diff has a channel-context forwarding gap, hardcoded Hermes-specific core values, and an unrelated auth-profile credential change.

## Best Possible Solution

Keep the PR open and ask for a focused rebase/split. The likely best path is a generic, documented opaque outbound metadata seam that flows through sendMessage, gateway send, channel outbound context, queue persistence/recovery, and targeted tests. Hermes-specific metadata construction and topic/bot selection should be configurable/plugin-owned or explicitly accepted by a maintainer, and the unrelated auth-profile change should be removed from this PR.
