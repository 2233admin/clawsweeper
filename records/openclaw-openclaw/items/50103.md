## Summary

Keep this open. Current main still lacks additive `agents.defaults.systemPrompt` / `agents.defaults.rules` and per-agent equivalents; the implemented `systemPromptOverride` surface is schema-backed, documented, tested, and wired as a full replacement for the assembled prompt. The direct implementation PR is closed unmerged, #51598 is closed as a duplicate, and the still-open adjacent prompt PRs do not cover the requested defaults-plus-per-agent additive rules contract.

## What This Changes

Request to add config-driven additive agent prompt/rules fields under `agents.defaults` and per-agent config, including `$include` support and session-time injection after bootstrap files.

## Best Possible Solution

Define one additive agent prompt contribution contract alongside the existing full-replacement `systemPromptOverride`, document defaults-plus-per-agent merge order, reuse the existing `$include` resolver where possible, and thread the resolved contribution through CLI, embedded, channel-triggered, cron, and subagent session creation with schema/generated docs and focused tests.
