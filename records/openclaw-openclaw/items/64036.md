## Summary

Current main still has the reported final-chunk whitespace bug: `chunkTextByBreakResolver` trims chunks emitted inside the loop but pushes the final `remaining` string unchanged. The issue should stay open because open draft PR #72491 explicitly targets this bug with closing syntax and is still unmerged.

## Best Possible Solution

Keep this issue open until an implementation merges. The best path is to review, land, or close #72491; a correct fix should trim and skip the final residual chunk in `src/shared/text-chunking.ts`, add regression tests for the reported counterexamples, preserve documented short/under-limit behavior, and align the Matrix lightweight runtime copy plus guardrail if that copy remains necessary.
