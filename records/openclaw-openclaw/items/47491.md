## Summary

Keep this PR open. Current main still does not implement the PR’s remaining embedded auto-compaction hook behavior: the embedded before/after compaction path forwards only `sessionKey`, while the subscription params already contain `agentId` and `sessionId` and the plugin hook context type supports those fields. The reset-hook and direct compaction pieces are already covered on main, so the useful work is the current two-file embedded context/test change.

## Best Possible Solution

Keep this PR open, rebase if needed, and land the narrow embedded auto-compaction hook-context fix: forward `ctx.params.agentId`, `ctx.params.sessionId`, and `ctx.params.sessionKey` in `src/agents/pi-embedded-subscribe.handlers.compaction.ts`, and update `src/plugins/wired-hooks-compaction.test.ts` to assert all three fields for both `before_compaction` and `after_compaction`. Do not reintroduce the already-covered reset/direct-compaction changes unless a fresh current-main gap appears.
