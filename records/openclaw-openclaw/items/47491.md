## Summary

Keep this PR open. Current main still does not fully implement the PR's intended behavior: the embedded auto-compaction hook path passes only `sessionKey` to `before_compaction` and `after_compaction`, even though the subscription params already carry `agentId` and `sessionId`. The reset-hook and direct compaction slices are already covered on main, so the useful remaining work is the embedded callback context plus tests.

## Best Possible Solution

Keep this PR open, rebase it against current main, and narrow it to the still-missing embedded auto-compaction callback path. The likely landing shape is to forward `ctx.params.agentId`, `ctx.params.sessionId`, and `ctx.params.sessionKey` in `src/agents/pi-embedded-subscribe.handlers.compaction.ts`, with `src/plugins/wired-hooks-compaction.test.ts` asserting all three fields for both `before_compaction` and `after_compaction`. The already-implemented reset/direct-compaction parts should not be reintroduced unless the rebase reveals a fresh current-main gap.
