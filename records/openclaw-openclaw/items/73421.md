## Summary

Keep open. This is a fresh regression report against v2026.4.26 with concrete log patterns for Gateway WebSocket, Control UI/WebChat, browser.request, node.list, and Slack delivery delays. Current main has nearby post-release fixes, but I did not find a source, changelog, history, or related-item signal that clearly fixes or supersedes this combined NAS Docker stall/delayed-delivery issue.

## Best Possible Solution

Keep the issue open and investigate it as a Gateway control-plane/browser-path regression, not as a Slack-only delivery bug. The best next step is to collect timestamped Gateway WebSocket/request traces around node.list, browser.request, WebChat reconnects, and Slack socket reconnects on both 2026.4.23 and 2026.4.26, then add a targeted regression test or latency guard in the Gateway WebSocket/browser/node-list path once the blocking source is identified.
