## Summary

Keep this PR open. Current main still does not implement a Bonjour plugin config key for the mDNS service instance name: the plugin entry always derives `instanceName` from `ctx.machineDisplayName`, the bonjour manifest exposes an empty config schema, and the public Bonjour docs only document disable/hostname environment knobs. The existing advertiser already accepts `opts.instanceName`, and the plugin SDK already supplies `api.pluginConfig`, so this remains a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Review and land this PR or an equivalent bonjour-owned patch that adds `plugins.entries.bonjour.config.instanceName` to the plugin manifest, reads it through `api.pluginConfig`, routes it to the existing advertiser `instanceName` option, and keeps the focused tests/docs/changelog coverage.
