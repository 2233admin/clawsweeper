## Summary

Keep open. The PR is a useful partial fix for the already-aborted path, but it is not merge-ready because cancellation can still arrive during `AgentSession.prompt()` preflight before `Agent.activeRun` exists, leaving the same class of escaped prompt-start race.

## What This Changes

The PR adds a pre-prompt abort guard, Pi Agent abort/queue cleanup in abort handling and cleanup, test-harness methods, and a new abort-before-prompt regression test file.

## Best Possible Solution

Revise this PR so the actual Pi prompt-start boundary observes the run abort signal and cannot create `activeRun` after the outer attempt has been aborted, then keep focused coverage for pre-aborted, pre-activeRun, and activeRun-existing abort cases.

## Reproduction Assessment

Yes. Bug #74859 provides a deterministic pre-aborted-signal reproduction, and the remaining PR gap can be reproduced by delaying `AgentSession.prompt()` preflight before `Agent.prompt()` and aborting after this PR's outer guard has passed.

## Solution Assessment

No. The current approach is a partial fix: the pre-prompt guard is good for already-aborted signals, but the `agent.abort()` backstop cannot cancel anything before `Agent.activeRun` exists.
