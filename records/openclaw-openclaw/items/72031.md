## Summary

Keep open. Current main still routes Bedrock media-understanding auth through `requireApiKey`, which rejects `aws-sdk` auth with no static key before the AWS SDK credential chain can run. The open fix PR #72092 explicitly closes this issue, so the right maintainer action is to review, land, adapt, or close that PR rather than closing this issue first.

## What This Changes

The issue requests that Bedrock-backed image and related media-understanding paths accept `auth.mode === "aws-sdk"` without a static API key, matching the chat/simple-completion path.

## Best Possible Solution

Review, land, or supersede #72092. The end state should let image, MiniMax fallback, and audio/video media-understanding paths pass an empty API key only for `aws-sdk` auth, avoid storing empty runtime secrets, preserve strict errors for other auth modes, and cover the behavior with focused regression tests.

## Reproduction Assessment

Yes. The issue body gives concrete steps, and current source gives a high-confidence non-live reproduction: an `aws-sdk` auth result with no `apiKey` reaches `requireApiKey`, which throws before any Bedrock call can use SDK credentials.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror the simple-completion missing-key allowance in the media auth resolvers, limited to `aws-sdk`, rather than weakening `requireApiKey` globally.
