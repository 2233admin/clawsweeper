## Summary

Keep open. Current main still matches the reported regression: media-understanding auth paths call `requireApiKey` before Bedrock can use `aws-sdk` credential-chain auth, while the chat/simple-completion path already has an explicit missing-key allowance. The targeted closing PR #72092 is still open and unmerged, so this issue should remain open until that PR lands, is adapted, or is superseded.

## What This Changes

Allow Bedrock media-understanding image and related media auth paths to work with `auth: "aws-sdk"` and no static API key, matching the chat path.

## Best Possible Solution

Review, land, or supersede #72092. The desired end state is for image, MiniMax fallback, and audio/video media auth paths to pass an empty API key only when resolved auth mode is `aws-sdk`, avoid storing empty runtime keys, preserve existing missing-key errors for other auth modes, and add regression coverage for Bedrock aws-sdk media calls.

## Reproduction Assessment

Yes. The issue has concrete reproduction steps, and current source gives a high-confidence local reproduction without live AWS access: an `aws-sdk` auth result with no `apiKey` reaches `requireApiKey`, which throws before any Bedrock request can run.

## Solution Assessment

Yes. Mirroring the simple-completion missing-key allowance in the media-understanding auth resolvers is the narrow maintainable fix; weakening `requireApiKey` globally would be broader and riskier.
