## Summary

Keep open. Current main still unconditionally requires a non-empty API key in the media-understanding image and audio/video auth paths, while Bedrock docs and the chat/simple-completion path support `aws-sdk` auth without a static key. PR #72092 is an open, unmerged closing PR for this issue, so the best maintainer path is to review, land, or supersede that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open and review, land, or supersede PR #72092. The narrow fix belongs in media-understanding auth resolution: allow an empty `apiKey` only when `ResolvedProviderAuth.mode` is `aws-sdk`, skip storing empty runtime keys, pass `apiKey: ""` through to Bedrock-backed calls, and keep regression coverage for the image plus audio/video media-understanding paths.
