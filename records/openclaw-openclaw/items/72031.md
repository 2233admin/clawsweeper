## Summary

Keep open. Current main still has the reported media auth gap, and #72092 is an open closing PR for the same bug, so the issue should remain open until that PR lands or is superseded.

## What This Changes

Allow Bedrock media-understanding image/audio/video paths using `aws-sdk` auth to proceed without a static API key, matching the existing chat/simple-completion behavior.

## Best Possible Solution

Land or supersede #72092 with the narrow media-understanding fix: allow empty API-key material only for `aws-sdk` auth in the media resolvers, avoid storing empty runtime keys, preserve missing-key failures for other auth modes, and keep focused regression coverage for Bedrock image plus provider execution auth.

## Reproduction Assessment

Yes. Configure an `amazon-bedrock` image model with `auth: "aws-sdk"` and no `apiKey`; current main resolves `mode: "aws-sdk"` with no key, then reaches `requireApiKey` in `resolveImageRuntime` and throws before the Bedrock provider can use the AWS SDK credential chain.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance inside the media-specific auth resolvers is the narrow maintainable fix; weakening `requireApiKey` globally would broaden behavior beyond the reported Bedrock `aws-sdk` case.
