## Summary

Keep open. Current main still has no final-promotion review queue: managed deep dreaming ranks candidates, immediately sends them through the MEMORY.md append helper, and only writes the deep report afterward. The related reports cover adjacent quality, contamination, and phase-artifact storage behavior, but none adds a configurable final-promotion destination or approval contract.

## What This Changes

Request support for staging memory-core deep dreaming promotions in a review queue before they are written into MEMORY.md.

## Best Possible Solution

Add an explicit, additive final-promotion destination contract in memory-core that preserves current direct MEMORY.md writes as a compatible mode while allowing promoted candidates to be staged in a structured review artifact before canonical long-term memory is mutated; update the manifest, resolver, docs, and focused tests around the chosen contract.

## Reproduction Assessment

Not applicable as a feature/design request. The current behavior has a high-confidence source verification path: managed deep dreaming calls applyShortTermPromotions, and that helper appends selected candidates to MEMORY.md.

## Solution Assessment

Unclear until maintainers choose the product contract. Directionally, a separate final-promotion destination or approval setting is cleaner than overloading dreaming.storage.mode, which currently describes phase/report artifact placement.
