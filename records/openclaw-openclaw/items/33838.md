## Summary

Keep open. Current main still models node commands as string allowlist entries only; there is no per-command input/output schema declaration in connect, pairing, discovery, or invoke validation. Related #8287 is adjacent node-tool registration work, but it does not supersede this narrower gateway command-schema protocol gap.

## What This Changes

Request an additive mechanism for nodes to declare command input/output schemas so the gateway can validate node.invoke payloads and expose better caller APIs.

## Best Possible Solution

Keep this issue open for maintainer-designed additive gateway protocol work: preserve commands: string[] compatibility; add optional per-command schema metadata; persist and expose it through connect, pairing, registry/catalog, node.list/node.describe, generated clients, node-host/plugin runtime APIs, docs, and tests; and define explicit node.invoke input/output validation semantics while treating #8287 as related but separate node-tool registration work.
