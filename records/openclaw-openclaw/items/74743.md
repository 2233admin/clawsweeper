## Summary

Keep open. Current main still has the custom-session cron transcript gap, and the issue is already paired with open PR #74913 using closing syntax, so it should remain open until that PR or an equivalent focused fix lands.

## What This Changes

The issue reports that cron jobs targeting `--session session:<id>` deliver messages to the destination channel but do not persist those assistant responses into that destination session transcript.

## Best Possible Solution

Review and land PR #74913, or an equivalent focused patch, so successful direct cron deliveries pass the resolved destination session key and delivered text through the existing outbound mirror contract with idempotency and regression coverage for `session:<id>` jobs.

## Reproduction Assessment

Yes. The reporter's steps map directly to current main: custom session targets resolve to a persistent session key, direct delivery sends to the channel, but the delivery call omits `mirror` while transcript append is gated on `params.mirror`.

## Solution Assessment

Yes. Reusing the existing outbound mirror contract is the narrow maintainable solution; the open PR should be reviewed and landed or replaced with the same focused behavior.
