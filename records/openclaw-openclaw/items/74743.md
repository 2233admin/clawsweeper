## Summary

Keep open. Current main still matches the reported cron custom-session transcript gap: direct cron delivery can send to the requested channel, but it does not pass outbound mirror metadata, while transcript persistence only runs when mirror metadata is present. The issue is also paired with open PR #74913 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that cron jobs using `--session session:<id>` deliver the cron response to the channel but do not persist that assistant message in the destination session transcript.

## Best Possible Solution

The durable end state is a focused cron direct-delivery fix that mirrors successful delivered text into the resolved destination session transcript through the existing outbound mirror contract, with idempotency and regression coverage for `session:<id>` jobs.

## Reproduction Assessment

Yes. The reporter's steps map directly to current main: a custom-session cron job reaches direct delivery, that call omits `mirror`, and transcript append only runs when `params.mirror` is present.

## Solution Assessment

Yes. Reusing the existing outbound mirror contract is the narrow maintainable solution; the linked PR should be reviewed and landed or replaced with an equivalent focused patch.
