## Summary

Keep open. The Discord elevated fallback bug is still valid on current main, and this PR addresses the right runtime and diagnostics surfaces, but the branch is not merge-ready because it regresses the existing human `sandbox explain` output and still needs maintainer review on the channel-plugin seam for a security-sensitive elevated auth path.

## What This Changes

The PR adds a Discord elevated `allowFrom` fallback resolver, uses it from runtime elevated authorization and `sandbox explain` diagnostics with account context, and adds focused regression tests for fallback and explicit-empty override behavior.

## Best Possible Solution

Land a focused fix that keeps Discord-specific fallback resolution in the Discord plugin or another extension-owned adapter, lets runtime auth and `sandbox explain` share the same generic fallback path with account context, preserves the current human and JSON diagnostics contract, and covers fallback, explicit empty override, and multi-account diagnostics in tests.
