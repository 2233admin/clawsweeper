## Summary

Keep this issue open. Current main has partial fixes across the field report, including config schema/export docs, invalid config reload hardening, Discord timeout controls, onboarding auth-health fixes, cron staggering/concurrency controls, and update restart helpers. But the umbrella is not fully implemented or superseded: a concrete cron docs mismatch remains, `cron add` is still create-only, and the Discord/web-search/small-VPS observations have not been live-reverified or split into canonical focused follow-ups.

## Best Possible Solution

Keep this issue open for maintainer triage. Fix the cron guide so `openclaw cron list` is documented as enabled-only by default and `openclaw cron list --all` is shown for disabled jobs, then split remaining current behavior into focused follow-ups for idempotent cron setup/upsert, resource-pressure warnings or docs, isolated web-search reproduction, and any Discord/onboarding/update gaps not already covered by narrower resolved items.
