## Summary

Current main has partial cron improvements, but the umbrella field report should stay open. The cron docs still misstate plain `openclaw cron list` as listing all jobs while the CLI/service default is enabled-only, the JSON wrapper shapes are still not documented in the user-facing cron guide, and `cron add` remains create-only rather than an idempotent setup/upsert path.

## Best Possible Solution

Keep this issue open for maintainer triage. Fix the cron docs so `openclaw cron list` is documented as enabled-only by default with `openclaw cron list --all` for disabled jobs, document the `jobs` and `entries` JSON wrapper shapes, and decide whether cron should gain an idempotent create/upsert command for setup scripts. Then split any still-current Discord receive, isolated web-search, onboarding token, update-process, and small-VPS resource observations into focused follow-ups with fresh reproduction data.
