## Summary

Keep this issue open. Current main still does not apply an `op`-specific `OP_CACHE=false` default and the public 1Password exec-provider example still documents `args: ["read", ...]` without `--cache=false` or `OP_CACHE=false`. The existing exec provider already has generic `args` and `env` fields, so users can apply the workaround manually, but the reported OpenClaw-owned docs/defaults problem remains actionable and is not fully superseded by the broader external-secret batching issue #72278.

## Best Possible Solution

Keep the issue open and address it in the secrets provider surface: update the 1Password exec-provider documentation to show `--cache=false` or `env: { OP_CACHE: "false" }` for LaunchAgent/headless setups, and consider a narrow, well-documented provider helper or doctor warning rather than silently changing all exec provider behavior. Coordinate separately with #72278 for the broader batching/memoization model.
