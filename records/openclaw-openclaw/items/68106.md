## Summary

Keep open. Current main still swallows Lobster workflow-file resolution errors and falls back to inline pipeline parsing, so the PR is not obsolete; however, the proposed branch still lets missing workflow paths with spaces fall through to pipeline parsing, leaving part of the reported behavior unfixed before merge.

## What This Changes

The PR changes the Lobster embedded runner to treat workflow-extension pipeline inputs as file candidates, adds regression coverage for missing workflow paths and file-like inline arguments, and adds a changelog entry.

## Best Possible Solution

Update the detection heuristic so recognized workflow-file candidates that are bare filenames or path-like inputs, including paths with spaces, surface resolution errors, while true inline pipelines with file-like arguments still go through the pipeline runtime. Add a regression test for a missing spaced workflow path.

## Reproduction Assessment

Yes. The high-confidence unit path is `createEmbeddedLobsterRunner.run` with a missing `.lobster` pipeline and a mocked runtime; current main catches the stat failure and falls through to `runToolRequest({ pipeline })`, while the PR adds tests for the bare and slash variants.

## Solution Assessment

No. The PR is the right narrow area, but the whitespace ENOENT fallback is too broad and preserves the misleading inline-pipeline error for missing path-like workflow files that contain spaces.
