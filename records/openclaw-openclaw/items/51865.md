## Summary

Current main still has the reported false positive. The audit builds its known node-command set from platform allowlists only, while the documented high-risk commands remain valid but intentionally absent from defaults unless explicitly allowed. The linked fix PRs were closed unmerged, and the current audit tests still do not cover deny-only dangerous commands as valid known IDs.

## Best Possible Solution

Keep this issue open. The best fix is to separate known command IDs from currently allowed command IDs: add `DEFAULT_DANGEROUS_NODE_COMMANDS` or a shared complete node-command registry to the audit known-command set, then add a regression test in `src/security/audit-node-command-findings.test.ts` covering deny-only `camera.snap`, `camera.clip`, `screen.record`, `calendar.add`, `contacts.add`, and `sms.send` so they do not emit `gateway.nodes.deny_commands_ineffective`.
