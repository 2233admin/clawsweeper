## Summary

Keep open. The underlying bug still exists on current main, but this PR is not mergeable as it stands: the head branch has stale/conflicted loader edits that do not parse and type-contract changes in the wrong file after the registry-types split.

## What This Changes

The PR adds `skills` to loaded plugin records, threads manifest-declared skills through plugin loader record creation, defaults non-manifest records to an empty skills array, and adds regression coverage.

## Best Possible Solution

Rework the fix against current main by adding `skills` to the canonical `PluginRecord` contract in `src/plugins/registry-types.ts`, passing `manifestRecord.skills` through the existing manifest-backed loader and CLI registry creation paths, defaulting non-manifest records to `[]`, and placing focused regression coverage in the current loader test layout.
