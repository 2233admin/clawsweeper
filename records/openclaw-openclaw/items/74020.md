## Summary

Keep open because current main still derives gateway pricing bootstrap only from `models.pricing.enabled`, so `models.mode: "replace"` does not skip OpenRouter/LiteLLM pricing fetches. The provided context also shows #74038 is an open, unmerged closing PR, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for gateway startup to skip remote OpenRouter/LiteLLM pricing fetches when `models.mode` is `replace`.

## Best Possible Solution

Land, reject, or replace #74038 with a focused gateway/config change that derives pricing bootstrap enablement from both `models.pricing.enabled` and `models.mode`, defaults remote pricing bootstrap off for replace mode unless maintainers choose an explicit opt-in override, and updates regression tests plus schema/help/docs/changelog together.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, current main's helper still returns true, so startup and refresh can proceed into OpenRouter/LiteLLM pricing fetches for configured external refs.

## Solution Assessment

Yes. The narrowest maintainable fix belongs at the shared pricing enablement/default boundary used by startup and refresh, not as a one-off guard in only one caller; #74038 appears to target that boundary and should be reviewed or replaced with an equivalent fix.
