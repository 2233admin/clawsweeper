## Summary

Keep open. Current main still enables the gateway pricing bootstrap based only on `models.pricing.enabled`, so `models.mode: "replace"` does not skip the OpenRouter/LiteLLM pricing path; open PR #74038 is the linked implementation candidate and should be reviewed or replaced before this issue closes.

## What This Changes

The issue asks for gateway startup to skip remote model-pricing catalog fetches when `models.mode` is `replace`, while still allowing explicit pricing opt-in.

## Best Possible Solution

Review and land PR #74038 if maintainers accept its boundary, or replace it with the same narrow fix: derive pricing enablement from both `models.pricing.enabled` and `models.mode`, preserve `models.pricing.enabled: true` as an explicit opt-in, and update gateway tests plus config/docs help.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no explicit `models.pricing.enabled: false`, the helper still returns true, startup loads the pricing cache module, and refresh can fetch remote OpenRouter/LiteLLM catalogs when external refs remain.

## Solution Assessment

Yes. The maintainable fix is a narrow gateway pricing default change with regression coverage and docs/config-help updates; the linked PR describes that boundary, while this issue should stay open until that code is merged or superseded.
