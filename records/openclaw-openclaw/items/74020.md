## Summary

Keep open. Current main still treats gateway model pricing as enabled unless `models.pricing.enabled` is explicitly false, so `models.mode: "replace"` can still enter the startup pricing bootstrap path; open PR #74038 is already the linked implementation candidate and should be reviewed instead of closing this issue early.

## What This Changes

The issue requests that Gateway startup skip OpenRouter and LiteLLM pricing fetches when `models.mode` is `replace` and models are manually configured.

## Best Possible Solution

Review and land #74038 or an equivalent focused fix that derives pricing bootstrap enablement from both the explicit `models.pricing.enabled` override and `models.mode`, keeps an explicit true opt-in available if maintainers want it, and updates the regression tests plus config/help/docs surfaces together.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, the current helper still returns true, allowing startup and refresh code to proceed toward OpenRouter/LiteLLM catalog fetches. I did not run tests because this review was required to stay read-only.

## Solution Assessment

Yes. The narrow maintainable fix is at the shared pricing enablement/default boundary so both startup import and refresh execution observe replace mode; the already-open PR #74038 appears to target that path and should be handled before opening another repair lane.
