## Summary

Keep open. Current main still starts the gateway pricing bootstrap when `models.mode` is `replace`; the only implemented skip is `models.pricing.enabled: false`. The related replace-mode discovery work (#66957/#73557) is adjacent, but this report is a narrower model-pricing fetch path and is a safe focused fix candidate.

## Best Possible Solution

Implement a narrow gateway pricing-bootstrap guard for `models.mode: "replace"`: remote OpenRouter/LiteLLM pricing fetches should be skipped, configured local cost metadata should still work, and the existing explicit `models.pricing.enabled` toggle should remain the override for non-replace setups. Add regression tests in the model-pricing startup path and document the replace-mode behavior if user-facing docs change.
