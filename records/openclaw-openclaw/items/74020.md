## Summary

Keep open. Current main still enables the gateway model-pricing bootstrap based only on `models.pricing.enabled`, so `models.mode: "replace"` does not skip OpenRouter/LiteLLM pricing fetches. PR #74038 is open and explicitly says it fixes this issue, so the best maintainer path is to review/land or close that PR rather than closing this issue early.

## Best Possible Solution

Keep this issue open and review PR #74038. If maintainers accept that PR's boundary, land it so replace mode disables remote pricing bootstrap by default while preserving `models.pricing.enabled: true` as an explicit opt-in; otherwise close that PR and implement the same narrow guard with tests in the gateway pricing startup/refresh path plus docs/config-help updates.
