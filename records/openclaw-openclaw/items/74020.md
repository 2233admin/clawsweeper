## Summary

Keep open. Current main still enables Gateway model-pricing bootstrap solely from `models.pricing.enabled`, so `models.mode: "replace"` does not by itself skip OpenRouter/LiteLLM pricing fetches. Open PR #74038 uses closing syntax for this issue and remains unmerged, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Gateway startup skip remote OpenRouter/LiteLLM pricing-catalog fetches by default when `models.mode` is `replace`.

## Best Possible Solution

Review and land PR #74038 or an equivalent narrow fix that derives pricing enablement from both `models.pricing.enabled` and `models.mode`, preserves explicit `models.pricing.enabled: true` as an opt-in for replace mode, and updates regression tests plus config/help docs.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: with `models.mode: "replace"` and no explicit `models.pricing.enabled: false`, the helper still returns true, startup loads pricing refresh, and the refresh path can fetch OpenRouter/LiteLLM catalogs for external refs.

## Solution Assessment

Yes. The best current direction is the narrow derived-default fix described in PR #74038, because it changes only the pricing bootstrap default for replace mode while keeping explicit opt-in and non-replace behavior intact.
