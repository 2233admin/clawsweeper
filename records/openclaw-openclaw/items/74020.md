## Summary

Keep open. Current main still enables gateway model pricing by default without considering `models.mode`, and open PR #74038 already targets this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Gateway startup skip OpenRouter and LiteLLM pricing fetches when `models.mode` is `replace`.

## Best Possible Solution

Implement the pricing enablement default at the shared gateway pricing boundary so replace mode disables background pricing bootstrap unless pricing is explicitly opted back in, with matching regression tests and config/help/docs updates.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure `models.mode: "replace"` without `models.pricing.enabled: false`, and the current helper still returns true, allowing startup and refresh to reach the remote catalog fetch path.

## Solution Assessment

Yes. The narrow maintainable fix is to derive gateway pricing enablement from both `models.pricing.enabled` and `models.mode`, rather than adding one-off guards only in a startup caller.
