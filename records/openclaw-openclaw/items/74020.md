## Summary

Keep open. Current main still enables the gateway pricing bootstrap solely from `models.pricing.enabled`, so `models.mode: "replace"` does not skip OpenRouter/LiteLLM catalog fetches. PR #74038 is already open and linked as the implementation candidate, so this issue should stay open until that PR lands or maintainers choose a different narrow fix.

## Best Possible Solution

Keep this issue open and review PR #74038. If maintainers accept that boundary, land it so `models.mode: "replace"` disables remote pricing bootstrap by default while preserving `models.pricing.enabled: true` as an explicit opt-in; otherwise close that PR and implement the same narrow guard with regression coverage in the gateway pricing startup/refresh paths plus config-help/docs updates.
