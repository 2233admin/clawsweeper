## Summary

Keep this issue open. Current main has partial startup-sidecar mitigation for one newly reported prewarm path, but the central `models.list` cold path is still blocking and can still run unbounded or serial provider/auth work. The recent Apr 26 and Apr 28 comments add concrete fresh timing evidence, so this is not stale and not safely superseded by #70186.

## Best Possible Solution

Keep this issue open. The best path is a focused performance fix around the Gateway model catalog: make configured `models.list` avoid loading the full catalog when configured rows are enough, parallelize and time-box provider catalog hooks for normal Gateway operation, preserve stale/static catalog rows while slow providers refresh in the background, pass the no-keychain-prompt intent through external CLI auth overlay or defer those reads, and add regression coverage with deliberately slow provider catalog/auth hooks plus the prewarm timeout case.

## Evidence

- **current_issue_context:** The discussion includes fresh reproductions after the original report: an Apr 26 comment reports two `models.list` round trips of 55,973ms and 37,984ms on v2026.4.24, and Apr 28 comments identify macOS auth-storage Keychain delay plus a v2026.4.26 startup prewarm workaround.
