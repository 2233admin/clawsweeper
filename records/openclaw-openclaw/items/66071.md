## Summary

Keep this PR open. Current main still emits OpenResponses HTTP assistant SSE deltas without checking `phase`, so the central commentary-leak bug is not implemented on main. The PR is a focused implementation candidate, but it still needs maintainer review because the latest diff can still leak commentary in finalized text when commentary and final-answer chunks share one run.

## Best Possible Solution

Keep the PR open as a focused implementation candidate. The best path is to update it against current main so OpenResponses HTTP keeps separate visible/final-answer text from commentary text, preserves normal unphased token streaming, keeps completed output item phase metadata, and adds regression tests for commentary-only tool-call turns plus mixed commentary/final-answer turns. After that, run the targeted gateway OpenResponses tests and the normal changed gate.
