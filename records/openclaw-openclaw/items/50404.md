## Summary

Keep open. Current main still has only partial Control UI session management: Gateway session primitives, a Chat header session selector, and a separate Sessions table. It does not implement the requested native conversation-history sidebar with search, active highlighting, inline rename/context actions, archive semantics, and ChatGPT-style switching; related workarounds and open PR #73725 are useful but do not close the broader request.

## What This Changes

The issue requests a ChatGPT-style Control UI session sidebar with saved conversation history, switching, search, rename/delete/archive actions, and persistence.

## Best Possible Solution

Keep a native Control UI conversation-management track open. Review and either land or close #73725 for the immediate create-and-switch repair, then designate one canonical sidebar issue and implement the remaining UX against the existing Gateway session APIs, including derived titles/previews, search, active highlighting, inline label editing, delete behavior, a defined archive model if desired, mobile layout, and updated Control UI docs.
