## Summary

Keep this issue open. Current main has adjacent session primitives, a chat-session dropdown, and a separate Sessions table with search/label/delete controls, but it does not implement the requested native Control UI conversation sidebar. The most important reported bug is still visible on main: the Chat plus button dispatches `/new` into the active session reset path instead of creating and switching to a persisted dashboard session. An open maintainer PR, #73725, addresses that narrow create/switch slice and explicitly references this issue, so the best cleanup path is to review that PR first and then scope the broader sidebar/history UX rather than closing this report now.

## Best Possible Solution

Keep this issue open. Review and land or close #73725 for the immediate New Chat `sessions.create`/switch behavior, then decide whether this issue or #27526 is the canonical product track for a native Control UI conversation sidebar. A good implementation should reuse existing Gateway session primitives (`sessions.list`, `sessions.create`, `sessions.patch`, `sessions.delete`, preview/title options), define archive semantics before exposing archive UI, and update Control UI docs once behavior ships.
