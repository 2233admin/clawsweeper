## Summary

Keep this PR open. Current main still does not contain an EasyRunner deployment guide, but the proposed docs are not safe to merge as-is: they still present `gateway.controlUi.dangerouslyDisableDeviceAuth=true` as normal reverse-proxy setup, omit current `gateway.controlUi.allowedOrigins` requirements for non-loopback Control UI access, and target an older Platforms navigation shape instead of the current Install > Hosting surface.

## Best Possible Solution

Keep the PR open and request a rebased docs revision. The best path is to move or redirect the guide into the current Install > Hosting surface, align the Podman/Caddy setup with current bind and `gateway.controlUi.allowedOrigins` behavior, remove normal-use `dangerouslyDisableDeviceAuth` guidance, add a rebuild step before redeploying updates, and have maintainers review EasyRunner-specific reverse-proxy/trusted-proxy assumptions before merge.
