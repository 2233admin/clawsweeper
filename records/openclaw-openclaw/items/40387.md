## Summary

Keep this PR open. Current main still does not include an EasyRunner deployment guide, and provider-specific hosting/deployment docs are an established OpenClaw docs surface. The PR is not implemented on main and is not clearly out of scope, but it still needs contributor revision and maintainer docs/security review before it should merge.

## Best Possible Solution

Keep the PR open and ask for a revised, rebased docs contribution. The best path is to place or link EasyRunner under the current Install > Hosting surface, remove normal-use dangerouslyDisableDeviceAuth guidance, document a safe auth model with gateway.controlUi.allowedOrigins for the EasyRunner HTTPS domain, make the gateway bind/container networking requirements explicit, keep secrets in env or secret storage, rebuild localhost/openclaw:latest before redeploy/update, and have maintainers review the EasyRunner/Caddy/Podman commands for security and real deployment accuracy.
