## Summary

Keep this PR open. Current main still does not include an EasyRunner deployment guide, and provider-specific hosting docs are an established OpenClaw docs surface. The PR is not implemented, not superseded, and not ClawHub-only, but it still needs contributor revision and maintainer docs/security review because the proposed guide normalizes a dangerous Control UI auth bypass and misses current non-loopback bind/origin requirements.

## Best Possible Solution

Keep this PR open and ask for a revised, rebased docs contribution. The best path is to place or link EasyRunner under the current Install > Hosting docs surface, remove normal-use dangerouslyDisableDeviceAuth guidance, document a safe auth model with explicit gateway.controlUi.allowedOrigins for the EasyRunner HTTPS domain, set the gateway bind mode needed for EasyRunner/Caddy container networking, keep secrets in env or secret storage, rebuild the local image before redeploy/update, and verify the EasyRunner/Caddy/Podman commands against a real deployment before maintainer docs/security review.

- Navigation drift remains: the PR adds the guide under the older Platforms/provider placement instead of the current Install > Hosting surface.
