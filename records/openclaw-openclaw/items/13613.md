## Summary

Keep this issue open. Current main still has Gateway-local cron scheduling, process-local serialization, local job/state files, local status/CLI surfaces, and no accepted coordination backend or leader/lease contract. The linked horizontal-scaling proposal is still open and broader, so it does not supersede this cron-specific HA gap.

## What This Changes

Add multi-gateway cron coordination, such as leader election or distributed locks, with configuration, health/status, and CLI visibility so HA deployments do not execute the same scheduled jobs multiple times.

## Best Possible Solution

Define a supported distributed cron ownership model for HA deployments, then implement it behind a versioned config or plugin/API boundary with health/status visibility, CLI inspection, docs, and tests that simulate separate Gateway processes sharing scheduled jobs.
