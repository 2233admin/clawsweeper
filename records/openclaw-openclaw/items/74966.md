## Summary

Keep open. Current main still emits the generated runtime-deps install manifest without a `dependencies` field when the install plan is empty, and this open contributor PR is the narrow implementation tied to the linked npm v11 crash report. Static review found no blocking correctness or security issue; the remaining step is maintainer validation and merge decision.

## What This Changes

The branch makes bundled runtime-deps install manifests always include a `dependencies` object and adds an empty-spec regression test.

## Best Possible Solution

Merge the narrow manifest-helper change after maintainer validation, keeping #74949 open until this PR lands or another equivalent fix is merged. A targeted bundled runtime-deps test is sufficient for the manifest contract, with an optional Node v25/npm v11 packaged Gateway startup smoke for the reported crash path.

## Reproduction Assessment

Unclear for the full end-to-end crash loop: this read-only review did not run npm installs or Gateway startup. Yes for the manifest-shape edge case: current main’s helper statically omits `dependencies` for `ensureNpmInstallExecutionManifest(root, [])`.

## Solution Assessment

Yes. Always emitting `dependencies: {}` is the smallest maintainable fix for the generated manifest shape and preserves existing sorted non-empty dependency output without changing package-manager selection, install flags, or plugin policy.
