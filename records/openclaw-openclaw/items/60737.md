## Summary

Keep open. Current main supports explicit Telegram ACP binding flows, but it does not implement the requested per-DM/per-chat `defaultAcp` behavior that auto-spawns an ACP session on the first message in a new topic.

## Best Possible Solution

Keep this issue open and implement a typed Telegram default ACP binding surface, likely under `TelegramDirectConfig` and `TelegramGroupConfig` with optional account-level fallback. The inbound Telegram route should, after access checks and before normal agent routing, detect a topic with no explicit per-topic route/configured binding/runtime binding, spawn or ensure an ACP session through the existing ACP/session-binding services, bind that topic, and cover both DM topics and group forum topics with schema, docs, and regression tests.
