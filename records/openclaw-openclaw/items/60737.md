## Summary

Keep open. Current main still supports exact Telegram topic ACP bindings, per-topic `agentId` routing, and manual `/acp spawn --thread`, but it does not expose a per-DM/per-chat `defaultAcp`-style config or an inbound routing branch that auto-spawns an ACP session for an unbound new topic.

## What This Changes

The issue requests an opt-in Telegram default that automatically spawns and binds an ACP harness session for first messages in new DM or group topics, while explicit per-topic config takes precedence.

## Best Possible Solution

Design a first-class, opt-in Telegram default ACP binding surface with clear precedence and per-topic session isolation, then implement it through the existing binding and ACP spawn seams with matching config schema, docs, and focused Telegram/ACP tests.

## Security Review

Status: needs_attention
