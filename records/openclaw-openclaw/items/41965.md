## Summary

Keep open. Current main still does not implement a `MEDIA:/path#doc` delivery hint, and the existing `forceDocument` plumbing remains effectively Telegram-oriented. WhatsApp direct sends and auto-replies still classify image MIME/kind as image payloads unless the media is already a document, while related PR #42044 was closed unmerged and is only prior art.

## Best Possible Solution

Keep this issue open and implement one narrow, shared outbound media delivery hint. The best fix is to parse a reserved `#doc` suffix or carry structured `forceDocument` metadata before media resolution, strip only the reserved hint before local reads, preserve safe local-root checks, and propagate the hint through WhatsApp direct sends, outbound adapters, and auto-reply delivery so image bytes can be sent with a Baileys document payload when requested. Add focused parser/loader/WhatsApp tests and update user-facing docs; use #42044 as prior art, not as a close reason.
