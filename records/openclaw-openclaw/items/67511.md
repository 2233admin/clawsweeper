## Summary

Keep open. Current main still has only partial cwd/workspace breadcrumbs, not the requested durable per-session workingContext capsule across compact/new flows, and /status/docs still do not expose or explain working-state continuity. The related #71135 PR is still an open draft storage-only candidate, not a shipped fix.

## Best Possible Solution

Keep this issue open as the product/API tracker. Review #71135 for the storage shape, then land follow-up work that updates working context from run/exec/subagent/ACP paths, exposes cwd/repo/branch/sandbox/temp-clone/push-target state in /status, and updates workspace plus compaction docs to distinguish semantic context continuity from engineering working-state continuity.
