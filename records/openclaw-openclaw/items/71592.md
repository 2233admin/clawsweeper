## Summary

Keep this issue open. Current `main` still advertises `/status` and `/compact` to the TUI while the local TUI command switch has no `status` or `compact` handlers, so those commands still fall through to the embedded chat/model path instead of being handled locally or rejected explicitly.

## What This Changes

The issue requests that local embedded TUI stop forwarding advertised `/status` and `/compact` slash commands as ordinary model prompts and instead handle them or show an explicit local-mode response.

## Best Possible Solution

Local embedded TUI should keep autocomplete and docs honest by ensuring every advertised shared slash command either executes through a reusable local/shared command path or returns an explicit local-mode unsupported response. `/status` and `/compact` specifically should never be added to the transcript or sent into `agentCommandFromIngress()` as ordinary prompts.

## Reproduction Assessment

Yes. The high-confidence static reproduction is current-main source inspection: slash input enters `handleCommand()`, `/status` and `/compact` are advertised by shared-command autocomplete, the switch lacks cases for them, and the default path forwards the raw text to `sendChat()` and then `agentCommandFromIngress()` in local mode.

## Solution Assessment

Yes. The requested fix direction is the narrow maintainable path: intercept advertised commands at the local TUI command boundary, either by implementing local handlers or returning explicit unsupported output, instead of globally removing shared commands or continuing to send known slash commands to the model.
