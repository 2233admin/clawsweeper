## Summary

Keep this PR open. Current main does not implement the PR’s central user-visible change: there is still only a single `workspace` field, no `multipleWorkspaces` config/schema support, no composite symlink workspace helper, and no docs describing a shipped multi-workspace agent mode. The PR remains a plausible implementation candidate, but it still needs maintainer review around the later review comments and the security implications of expanding an agent’s visible filesystem scope through symlinks.

## Best Possible Solution

Keep this PR open as the current implementation candidate for multi-workspace agent support. The best path is to finish the review comments, add focused tests for composite symlink planning and all runtime setup paths, update the config schema/generated docs if accepted, and have a maintainer decide whether this belongs in core given the filesystem-scope implications.
