## Summary

Keep this PR open. Current main still does not implement the PR's central user-visible change: agent config is single-workspace, runtime resolution/setup returns and prepares one workspace root, and public docs still describe one active workspace rather than a shipped composite multi-workspace mode. The PR remains a plausible implementation candidate, but the filesystem/symlink edge cases and all-execution-path coverage need maintainer review before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for multi-workspace agent support. If maintainers want this in core, finish the outstanding symlink/filesystem review items, route all agent execution paths through one tested composite workspace setup seam, update schema/generated config docs and public docs, and add focused tests for collisions, root/relative paths, stale links, platform symlink behavior, reverse lookup, and CLI/cron/reply setup parity.

- The provided review comments indicate the PR may not yet pass `multipleWorkspaces` through every execution path that calls `ensureAgentWorkspace`, so merging as-is could leave CLI or cron agent runs with an unpopulated composite workspace.
