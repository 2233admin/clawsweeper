## Summary

Keep this PR open. Current main still does not implement the proposed `gateway-probe` plugin, and the PR is not safely closeable as ClawHub-only because the public plugin API covers the session/tool/model hook side but not the PR’s selected structured app logs or trusted/internal diagnostics. The remaining path needs maintainer product and security judgment.

## Best Possible Solution

Do not auto-close this PR. Maintainers should decide whether the product wants a bundled observability plugin with Kafka egress, or should redirect it to ClawHub after first adding a narrow, reviewed public SDK seam for structured app logs and trusted/internal diagnostic subscriptions. If accepted, the PR should be updated to current main’s SDK contracts and receive focused security review for data redaction, Kafka configuration, dependency provenance, and failure behavior.
