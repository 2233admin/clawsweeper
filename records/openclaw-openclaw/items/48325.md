## Summary

This PR is a substantive external contribution, not spam, but its central change is a new optional gateway telemetry/Kafka plugin. Current main does not already include `gateway-probe`, and the requested capability can be built as an external OpenClaw plugin using existing hook, service, diagnostic, and plugin-install surfaces. OpenClaw’s current product direction keeps core lean and sends optional plugins through ClawHub/npm first, with bundled promotion requiring a higher maintainer-owned product and security review bar.

## Best Possible Solution

Close the OpenClaw core PR and encourage publishing `gateway-probe` as an external ClawHub/npm plugin using the existing hook, service, diagnostic, and plugin-install surfaces. If that implementation finds a concrete missing public SDK seam, open a narrower SDK proposal; bundled promotion should wait for explicit maintainer product and security ownership.
