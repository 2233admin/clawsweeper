## Summary

Keep open. Current main still returns no transcript usage snapshot when a contentful transcript lacks positive provider usage or model metadata, and both status and session-list surfaces depend on that snapshot before they can display a nonzero context total.

## What This Changes

The issue requests a conservative transcript-derived context-usage fallback for local/OpenAI-compatible sessions when provider usage telemetry is missing or zero.

## Best Possible Solution

Add a generic transcript-usage fallback that keeps valid provider-reported usage authoritative, then estimates prompt/context tokens from visible transcript text only when no positive usage-derived context total exists, with targeted regression coverage for local/OpenAI-compatible no-telemetry transcripts.

## Reproduction Assessment

Yes. The current test fixture already creates a transcript with user and assistant content but no usage telemetry and expects the reader to return `null`; the status/session code then drops that missing snapshot before formatting context usage.

## Solution Assessment

Yes. A narrow change in the transcript usage extraction and its status/session fallback consumers is the best maintainable path; provider transports and OpenAI-compatible usage normalization should remain authoritative when they report positive usage.
