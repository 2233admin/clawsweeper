## Summary

Keep open. Current main has transcript-backed usage fallback for provider usage snapshots, but it still does not estimate context from transcript text when local/OpenAI-compatible usage telemetry is missing or zero; the existing test suite explicitly expects the no-usage transcript case to return null.

## Best Possible Solution

Keep this open and implement a narrow transcript-estimate fallback in the Gateway session usage path: preserve provider-reported usage when present, but when local/OpenAI-compatible transcripts contain conversation text and no positive usage telemetry, estimate prompt/context tokens conservatively, mark the snapshot usable for status/session rows, and add regression coverage for the no-telemetry local provider case.

In `src/gateway/session-utils.fs.ts`, keep provider-reported normalized usage authoritative, but when transcript content exists and provider usage is absent or all zero, estimate a conservative prompt/context token total from conversation transcript text and return a usable `SessionTranscriptUsageSnapshot` with `totalTokens` and `totalTokensFresh` so `buildGatewaySessionRow` and `/status` do not collapse to 0/undefined.
