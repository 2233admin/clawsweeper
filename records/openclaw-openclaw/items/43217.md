## Summary

Keep open. Current main does not implement this PR's intended resolved-model reasoning gate: the shared Google thinking sanitizer still takes only `modelId` and `thinkingLevel`, and the wrapper passes only `model.id`. The PR is stale against the moved shared helper and should be rebased/adapted rather than closed as implemented or superseded.

## Best Possible Solution

Keep this PR open and ask for a rebase onto the current shared Google thinking helper. The maintainer path is to update `src/plugin-sdk/provider-stream-shared.ts` and its wrapper call path so sanitization can use resolved model capability where needed, preserve current Gemini 3/Gemma/adaptive behavior, decide the model-family-correct Gemini 2.5 fixed-level mapping (`thinkingBudget` versus `thinkingLevel`) against the native Google transport, and add a `gemini-2.5-flash` fixed-level regression test plus a non-reasoning negative case.
