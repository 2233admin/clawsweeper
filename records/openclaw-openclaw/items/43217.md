## Summary

Keep open. Current main still does not implement this PR's intended resolved-model reasoning gate for the shared Google thinking sanitizer. The earlier ClawSweeper review remains consistent with current main: the code has moved into the shared Plugin SDK helper, but the sanitizer still uses model-id family checks and the wrapper still passes only `model.id`, not `model.reasoning`.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rebase/adapt the fix onto `src/plugin-sdk/provider-stream-shared.ts`, use the resolved model capability already available through the wrapper context where appropriate, preserve current Gemini 3/Gemma/adaptive behavior, align Gemini 2.5 fixed-level handling with the native Google transport, and add focused tests for `gemini-2.5-flash` with `reasoning: true` plus a non-reasoning negative case.
