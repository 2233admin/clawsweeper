## Summary

Keep this PR open. Current main has moved the Google thinking sanitizer into the shared provider stream helper, but it still does not implement the PR’s intended resolved-model reasoning gate or the fixed-level Gemini 2.5 Flash regression path. The PR should be rebased and adapted to the current model-family behavior rather than closed as implemented.

## Best Possible Solution

Keep this PR open and ask for a rebase onto the shared Google thinking helper. The maintainer path is to update `src/plugin-sdk/provider-stream-shared.ts` and its wrapper call path so sanitization can use resolved model capability where needed, preserve current Gemini 3/Gemma/adaptive behavior, repair fixed-level Gemini 2.5 payloads with the model-family-correct budget mapping, and add a `gemini-2.5-flash` fixed-level regression test plus a non-reasoning negative case. Related open PR #73416 addresses the Google API-shape guard, not this reasoning-capability/fixed-level Gemini 2.5 issue.
