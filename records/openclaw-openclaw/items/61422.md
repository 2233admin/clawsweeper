## Summary

Keep open: current main has not implemented this PR's intended ordering change, but the branch is not merge-ready because it flips the documented prompt-cache contract without updating the existing prompt-order tests, docs, or nearby rationale comments.

## What This Changes

The PR moves stable workspace Project Context files in `buildAgentSystemPrompt` from before `SYSTEM_PROMPT_CACHE_BOUNDARY` to after it so edits to files such as `MEMORY.md` do not invalidate the cached prefix.

## Best Possible Solution

Decide the intended cache contract first: either keep stable workspace files above the boundary and add a narrower volatility-aware path for frequently rewritten files, or accept moving Project Context below the boundary and update the tests, docs, changelog, and rationale comments together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: applying the PR makes `# Project Context` appear after `SYSTEM_PROMPT_CACHE_BOUNDARY`, while the current prompt-ordering test asserts the boundary comes after Project Context.

## Solution Assessment

No as submitted. The cache-invalidation problem is real, but moving every stable context file below the boundary contradicts the current documented design and leaves deterministic test failures; a narrower volatility-aware change or fully updated contract would be safer.
