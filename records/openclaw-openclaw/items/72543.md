## Summary

Keep this open. Current main still normalizes some `main` aliases, but the shared Control UI dropdown builder does not synthesize or pin the canonical main session when `sessions.list` omits it and the user is viewing another session.

## What This Changes

The issue requests that the Control UI chat session selector always include and normalize the canonical main session key `agent:main:main`.

## Best Possible Solution

Add a shared Control UI helper that derives the canonical main key from `hello.snapshot.sessionDefaults.mainSessionKey`, prepends a stable main-session option before discovered sessions, and reuses the same alias normalization for selected and persisted session keys.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resolveSessionOptionGroups` with an active non-main `sessionKey` and a `sessionsResult` that lacks `agent:main:main`; current code returns only supplied rows plus the active session, so the canonical main option is absent.

## Solution Assessment

Yes. The narrow maintainable fix is to inject through the shared option-group builder using Gateway `sessionDefaults`, while avoiding a hard-coded persona label and keeping desktop/mobile dropdown behavior aligned.
