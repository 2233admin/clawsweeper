## Summary

Keep open. Current main still has the reported gap: the shared Control UI session option builder only adds rows returned by `sessions.list` plus the currently selected key, so a non-main active session can leave `agent:main:main` absent from both desktop and mobile dropdowns even though the Gateway already exposes the canonical main key.

## What This Changes

The issue requests that the Control UI chat session dropdown always include a stable canonical main-session option and normalize `main` aliases to that canonical key.

## Best Possible Solution

Add one shared Control UI path that derives the canonical main key from `hello.snapshot.sessionDefaults.mainSessionKey`, guarantees that option in the shared option-group builder, keeps desktop and mobile selects aligned, and labels it from generic agent/session metadata rather than hard-coding a persona name.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resolveSessionOptionGroups` with active session `agent:main:subagent:x` and a `sessionsResult` that omits `agent:main:main`; the current helper returns discovered rows plus the active key, so the canonical main option is absent.

## Solution Assessment

Yes. The narrow maintainable fix is in the shared Control UI option builder plus the existing alias-normalization path; adding a new Gateway API or hard-coding `Chelmsworth` would be broader than needed.
