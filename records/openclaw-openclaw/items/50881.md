## Summary

Keep this PR open. Current main still does not implement the PR's intended Slack reaction idempotency: the shared Slack helpers still propagate `already_reacted` and `no_reaction`, and remove-own cleanup still calls Slack removal directly. Related open PRs #71843 and #58242 overlap on duplicate reaction adds, but they do not fully cover this PR's remove/no_reaction scope or constitute shipped main behavior.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rebase. The best path is a current-main Slack patch that treats `already_reacted` as success in the shared add helper, treats `no_reaction` as success for explicit removal and remove-own cleanup if maintainers accept that scope, preserves current `cfg`/token/client/write-client contracts, and adds focused Slack helper/action-runtime regression tests while coordinating with #69005, #71843, and #58242.
