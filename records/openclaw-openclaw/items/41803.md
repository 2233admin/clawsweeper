## Summary

Keep open. Current main still returns raw unrecognized assistant error text from the final `formatAssistantErrorText` fallback, and that value is pushed into embedded reply payloads. The PR addresses a real remaining bug, but should be refreshed before merge because its warning-log preview still truncates before redaction and does not use the existing log-control-character sanitizer.

## Best Possible Solution

Keep this PR open and ask for a small refresh against current main: preserve the generic user-safe catch-all for truly unrecognized assistant errors, keep explicit actionable branches ahead of it, redact the full raw text before truncating any operator preview, run that preview through `sanitizeForLog`, and retain the orphaned `tool_result`/long-string regression tests for the linked raw-error leak reports #11038 and #16948.
