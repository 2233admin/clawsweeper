## Summary

Keep open. Current main still returns raw unrecognized assistant error text from the final `formatAssistantErrorText` fallback, and that formatted text is used by embedded reply and lifecycle error surfaces. This PR remains a relevant implementation candidate, but it should be refreshed to address the remaining log-preview security concerns before merge.

## Best Possible Solution

Keep this PR open and ask for a focused refresh against current main: preserve the generic user-safe catch-all for truly unrecognized assistant errors, keep explicit actionable branches ahead of it, redact raw error text before truncating any operator preview, run the preview through `sanitizeForLog`, and keep the orphaned `tool_result` and long-string regression coverage for the linked raw-error leak reports #11038 and #16948.
