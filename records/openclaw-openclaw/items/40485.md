## Summary

Keep open. The source work addresses a real Telegram direct-chat reply-threading bug, but the branch changes removed legacy `src/telegram` files while current `main` runs the Telegram plugin from `extensions/telegram`; the active path still has the deleted-message behavior unless the fix is ported.

## What This Changes

This PR adds a legacy Telegram native-command `replyToMode` override so direct-chat `/new` and `/reset` replies are sent without reply threading, plus a private `/new` regression test.

## Best Possible Solution

Port the behavior into the active Telegram plugin: direct-chat native `/new` and `/reset` should send the first assistant reply without implicit reply threading, while groups, forum topics, plugin commands, explicit reply targets, menus, and other native commands preserve current threading semantics.
