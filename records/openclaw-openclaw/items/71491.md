## Summary

Keep open. Current main and the latest v2026.4.29 release still lack the Moonshot/Kimi outgoing payload repair for replayed assistant tool-call messages missing top-level `reasoning_content`, while open PR #74846 is the focused implementation path.

## What This Changes

The issue reports that Kimi K2.6 long conversations or cross-model fallback after LCM compaction can replay assistant tool-call messages without top-level `reasoning_content`, causing Moonshot 400 errors.

## Best Possible Solution

Land a narrow Moonshot-only payload-wrapper fix that preserves existing `reasoning_content`, adds an empty placeholder only for thinking-enabled assistant tool-call replay messages, and keeps the LCM upstream preservation work linked separately.

## Reproduction Assessment

Yes. The issue body and follow-up comment provide concrete long-session and cross-model fallback paths, and current-main source inspection shows the reported missing repair remains absent at the final outgoing payload boundary.

## Solution Assessment

Yes. The narrow final Moonshot stream-wrapper repair represented by PR #74846 is the most maintainable OpenClaw-side path because it runs after context assembly and mirrors the established DeepSeek V4 compatibility pattern without broad generic replay-policy changes.
