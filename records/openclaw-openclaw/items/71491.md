## Summary

Keep open. Current main still contains the earlier Kimi native tool-call ID fix and Moonshot `thinking.keep` forwarding, but I did not find a Moonshot/Kimi replay-history sanitizer or final payload repair that restores top-level `reasoning_content` after context-engine assembly or cross-model fallback. The report and follow-up comments are concrete and distinct from #70030/#70392.

## What This Changes

The issue asks OpenClaw to preserve or safely restore Kimi K2.6 `reasoning_content` on replayed assistant tool-call messages after long-context compaction and model fallback.

## Best Possible Solution

Add a Moonshot-owned replay or final-payload repair that preserves existing `reasoning_content` and fills a provider-safe placeholder for replayed assistant tool-call turns only when Kimi thinking replay requires it, with tests covering same-provider long history, context-engine assembly, and cross-model fallback shapes.
