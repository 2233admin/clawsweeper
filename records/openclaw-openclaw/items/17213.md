## Summary

Keep open. Current main has adjacent planning-only, liveness, replay-invalidation, and agent.wait safeguards, but it still exposes a generic lifecycle start/end/error model and terminal snapshots rather than the requested atomic run.begin -> run.exec -> mandatory run.commit/run.abort contract with structured abort blockers. The existing maintainer discussion also explicitly kept this as core reliability/product-contract work.

## What This Changes

Request an atomic run lifecycle that records intent, execution, and a mandatory commit or abort with structured blocker details so pre-action notices cannot be orphaned.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway protocol boundary, document intent, execution evidence, mandatory commit-or-abort terminal states, and structured abort/blocker details, then wire it through AgentEvent types, gateway schemas, runtime emitters, agent.wait, clients or plugin-visible surfaces, and regression tests.
