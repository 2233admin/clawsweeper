## Summary

Keep open. Current main has narrower strict-agentic, liveness, incomplete-turn, and agent.wait safeguards, but it still does not define the requested atomic run.begin/run.exec/run.commit-or-run.abort contract with structured abort blockers. The existing issue discussion also kept the item open and treated related work as context, not closure evidence.

## What This Changes

Request to enforce a transaction-style run lifecycle where announced action intent must proceed through execution and end in a mandatory commit or abort with a machine-readable blocker.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway protocol boundary, document intent, execution evidence, mandatory commit-or-abort terminal states, and structured blocker details, then wire it through AgentEvent types, gateway schemas, runtime emitters, agent.wait, clients or plugin-visible surfaces, and regression tests.
