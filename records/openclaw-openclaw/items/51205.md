## Summary

Keep open. The gateway timeout symptom overlaps the closed #51357/probe-auth thread and current main has newer handshake-timeout support, but the reporter explicitly followed up that the CLI symptom became intermittent/normal while model switching still failed. Current main supports `models.mode: "merge"`, custom `models.providers`, provider-qualified `/model` updates, and session patch validation, so this is not closeable as an invalid config report; the missing exact `/model` input, response, UI state, and logs are still the blocker.

## Best Possible Solution

Keep this open and ask for a latest-release reproduction focused on model switching: exact UI action or `/model <provider/model>` input, expected versus actual selected model, `openclaw models list --provider <id>` and `openclaw models status --json` with secrets redacted, the `sessions.patch` response or gateway log error, and whether the same provider/model works through a direct infer run. If it reproduces, the likely fix belongs in the configured-model catalog, allowlist resolution, `sessions.patch`, or Control UI model override path; the gateway timeout side can remain routed through the #51357/handshake-timeout work.
