## Summary

Keep this PR open, but do not merge it as-is. Current main still has the MiniMax metadata/runtime mismatch, but the proposed patch would make a generic MiniMax model key auto-select MiniMax Search without any Token Plan entitlement check, and current auto-detection chooses the first credentialed provider by priority.

## What This Changes

The PR updates MiniMax web-search metadata, plugin UI hints, docs, and a registration test so `MINIMAX_API_KEY` is advertised for MiniMax Search and setup can list it.

## Best Possible Solution

Land a revised MiniMax web-search metadata/docs fix after maintainer credential-policy review: MiniMax Search should be visible in setup, contract/runtime/docs should stay aligned, and generic `MINIMAX_API_KEY` should either be gated as search-capable or kept out of first-pass auto-detection with tests covering provider selection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate generic MiniMax keys before auto-detecting search:** `extensions/minimax/src/minimax-web-search-provider.ts:282`
  - body: Adding `MINIMAX_API_KEY` to this provider's `envVars` makes web-search auto-detection treat a generic MiniMax model key as search-capable. Since auto-detect returns the first credentialed provider by `autoDetectOrder`, users without Token Plan search access but with another working search key can be routed to MiniMax and hit runtime failures. Please keep auto-detect on Token Plan-specific keys, add a capability check, or otherwise guard this path.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff has no dependency or workflow supply-chain change, but it broadens automatic search-query routing based on a generic MiniMax API key.

Concerns:

- **[low] Unexpected MiniMax search routing:** `extensions/minimax/src/minimax-web-search-provider.ts:282`
  - body: By adding `MINIMAX_API_KEY` to web-search `envVars`, the patch can send web-search queries to MiniMax when the user only configured a MiniMax model key and did not explicitly choose MiniMax Search. That data-routing change should be explicitly accepted, gated, or deprioritized.
