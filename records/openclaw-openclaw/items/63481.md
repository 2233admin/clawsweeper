## Summary

Keep this PR open. Current main still starts channel accounts concurrently, Feishu still routes gateway startup through the per-account monitor path, and that path still calls the startup bot-identity probe directly with no shared queue. The linked bug remains open, and the newer Feishu startup-latency PR is also open rather than merged, so this PR is still an active implementation candidate rather than obsolete.

## What This Changes

The PR adds a shared promise-chain queue around Feishu `fetchBotIdentityForMonitor`, a test-only queue reset helper, and regression tests for concurrent caller serialization, failed-probe isolation, and abort handling.

## Best Possible Solution

Land a Feishu-owned startup probe fix that prevents concurrent per-account startup from bursting the Feishu bot probe endpoint, preserves prompt abort behavior for queued accounts, keeps focused regression coverage for concurrent callers, and is reconciled with the separate startup-latency work before merge.
