## Summary

Keep this issue open. Current main still treats non-zero poll duration defaults such as `pollDurationHours: 24` as poll creation intent on `message.send`, so the reported GPT-5.4 default-filled input can still hit the poll-only guard. The workspace media-root part appears partially addressed, and shared schema tests hide `components`, but the Discord component parser still throws on an all-empty modal default object if it reaches that path.

## Best Possible Solution

Keep this issue open and fix the message-tool boundary to normalize semantic no-op defaults before validation/routing. Add regression coverage for `action: "send"` with empty `pollQuestion`, `pollDurationHours: 24`, `pollMulti: false`, and all-empty Discord components, then make poll intent depend on actual poll content or another explicit poll signal rather than duration defaults alone.
