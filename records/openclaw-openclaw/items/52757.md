## Summary

Keep this issue open. Current main still treats a non-zero default poll duration such as `pollDurationHours: 24` as poll creation intent on `message.send`, and the Discord component parser still throws when an all-empty default modal object reaches it. The media-root part appears largely addressed by current agent-scoped media roots, but the core default-filled parameter bug remains.

## Best Possible Solution

Keep this issue open and fix the message-tool boundary so default-filled optional parameters are normalized before validation and routing. Poll intent on `message.send` should require meaningful poll content or an explicit poll signal rather than duration defaults alone, and Discord components should treat all-empty default objects as absent while preserving validation for non-empty malformed component payloads. Add focused regression tests for the GPT-5.4 default-filled send shape and leave the existing agent-scoped media-root security model intact.

## Clownfish Work Prompt

Fix openclaw/openclaw#52757 on current main. The observable bug is that GPT-5.4/default-filling models can send ordinary `message.send` calls with default optional fields (`pollDurationHours: 24`, `pollMulti: false`, empty `pollQuestion`, empty `pollOption`, and all-empty Discord `components`) and the message tool routes them into poll/component validation instead of a normal send. Preserve `action: "poll"` validation and do not broaden local media root/host file-read policy. Add focused regression tests for the default-filled send shape in poll helper/send-runner tests and Discord component/runtime tests.
