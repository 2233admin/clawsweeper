## Summary

Keep this PR open. Current main still does not implement the PR's intended runtime change: both the update.run supervised restart branch and the generic supervised SIGUSR1 restart branch still call exitProcess(0). The linked issue was mitigated for OpenClaw-managed systemd units by using Restart=always, but that is not the same as this PR's proposed supervised exit-code behavior.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to update it for current main so both generic supervised SIGUSR1 restarts and update.run supervised restarts exit non-zero on systemd with focused non-launchd tests, or to close it after an explicit maintainer decision that the canonical Restart=always systemd unit fully supersedes the runtime exit-code change.
