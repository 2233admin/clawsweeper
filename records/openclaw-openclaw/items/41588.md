## Summary

Current main still does not implement the requested activity-aware agent run timeout. The embedded runner still uses a single wall-clock run abort timer, the existing idle watchdog only covers model stream inactivity, and the public config/schema/docs do not expose separate idle and absolute run timeout controls.

## Best Possible Solution

Keep this issue open. The best implementation path is to add an agent-run idle timeout that is refreshed by assistant stream activity and tool lifecycle completion while retaining a separate absolute max runtime cap, then update config schema/help/docs and add focused tests proving active multi-tool runs are not aborted solely because the original wall-clock window elapsed.

- A fix should keep an absolute cap as discussed in the issue, otherwise activity-based extension could allow runaway runs to continue indefinitely.
