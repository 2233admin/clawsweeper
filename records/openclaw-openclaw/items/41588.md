## Summary

Current main still lacks an activity-aware agent run timeout. The default runtime timeout is now 48 hours and there is a model-stream idle watchdog, but the embedded runner still uses a single wall-clock abort timer with no reset on tool completion or assistant activity, and the public agent defaults schema still has no separate idle and absolute runtime timeout controls.

## Best Possible Solution

Keep this issue open. The best implementation path is to add a run-level idle timeout refreshed by assistant stream activity and tool lifecycle completion, retain a separate absolute max runtime cap, expose the contract in config schema/help/docs, and add focused tests proving active multi-tool runs are not aborted solely because the original wall-clock window elapsed.

## Evidence
