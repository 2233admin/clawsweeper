## Summary

Keep #58387 open. Current main does not implement hidden webchat bootstrap prompts: the gateway schema has no `hideUserMessage` field, the UI send path still always locally echoes the user turn, URL settings parsing has no `autostart` handling, and gateway transcript writes are not gated for hidden sends. The PR is focused on a real missing feature and the provided discussion shows the author addressed multiple concrete review findings rather than abandoning a low-signal change.

## Best Possible Solution

Keep the PR open for maintainer review. If the product direction is accepted, the best path is a tightly scoped additive gateway protocol change for `hideUserMessage`, admin-scope gating, hidden-send transcript safeguards, one-shot webchat `autostart` parsing bound to the intended session/gateway, and focused UI/gateway regression tests. If maintainers do not want hidden bootstrap turns in core webchat, they should explicitly close on product grounds rather than treating it as already implemented or stale.
