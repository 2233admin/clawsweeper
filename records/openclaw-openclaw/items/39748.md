## Summary

Keep this PR open. Current main still accepts unvalidated Nostr relay strings in the config schema, and the setup paths only check ws/wss protocol without restricting plaintext ws:// to loopback hosts. The PR is directionally useful after the author addressed the initial review feedback, but it needs a current-main refresh because the direct schema-only patch leaves setup gaps, targets a removed test file, and carries unrelated secrets-baseline churn.

## Best Possible Solution

Refresh this PR against current main with one shared Nostr relay URL validator used by NostrConfigSchema, setup-surface.ts, channel.setup.ts, generated config metadata, and focused current-main tests. The validator should allow remote wss:// relays, allow ws:// only for localhost/loopback development hosts, and reject http(s), javascript:, data:, file:, and invalid strings. Drop the unrelated .secrets.baseline churn and update docs/help text only if the user-facing contract needs clarification.
