Current main partially overlaps this PR through later directive-tag/code-block indentation fixes, but it still does not implement the PR's full outbound whitespace-preservation behavior. This PR should stay open for a narrow refresh or selective cherry-pick, plus maintainer review of the PR branch's whitespace edge-case comments.

## Best Possible Solution
