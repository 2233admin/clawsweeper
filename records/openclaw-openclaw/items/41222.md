## Summary

Keep this PR open. Current main overlaps the directive-tag/code-block side of the PR, but it still does not implement the full outbound whitespace-preservation behavior: the shared media parser still collapses/trims multiline body text after media extraction, and the no-inline-tag directive path still normalizes text rather than returning it byte-for-byte unchanged. The PR is also the open implementation candidate for the paired bug #41215, so it should be refreshed or selectively cherry-picked rather than closed as obsolete.
