## Summary

Keep this PR open. Current main does not implement the intended ZAI `X-Session-Id` sticky-routing header, and the existing prompt-cache support is not an equivalent replacement. The PR is still useful, but it needs maintainer/code-review follow-through because current main has moved and review comments flagged header-value validation and duplicate-header handling.

## Best Possible Solution

Keep the PR open and finish it as a provider-owned ZAI stream-wrapper change: inject a header-safe `X-Session-Id` from the agent session id, preserve current `tool_stream` and thinking behavior, replace any existing `x-session-id` header case-insensitively, add focused tests, and rebase docs/changelog/dependency metadata onto current main.
