## Summary

Keep this PR open: current main still lacks the requested ZAI `X-Session-Id` sticky-routing behavior, but the proposed branch is not merge-ready because it is dirty against current main and would miss existing ZAI thinking-wrapper behavior unless repaired.

## What This Changes

The PR adds a ZAI provider stream wrapper that injects `X-Session-Id` from `options.sessionId`, composes tool-stream defaults, updates docs/changelog, adds ZAI stream tests, and declares `@mariozechner/pi-ai` as a ZAI plugin dependency.

## Best Possible Solution

Keep the feature ZAI-owned, but rebase and repair the branch so session-header injection composes with current `wrapZaiStreamFn`, rejects header-invalid values, replaces existing session headers case-insensitively, uses the repo-current Pi runtime dependency, and carries the required changelog credit with targeted tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Compose with the current ZAI thinking wrapper:** `extensions/zai/index.ts:279`
  - body: Current main now routes ZAI through `wrapZaiStreamFn`, which keeps `tool_stream` and patches `thinking` for disabled and preserved-thinking modes. Replacing it with a wrapper that only handles tool-stream plus the session header would regress the tested ZAI thinking behavior after rebase.
  - confidence: 0.88
- **[P2] Reject invalid header values before injection:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts any non-empty string up to 256 chars, so CR/LF/NUL values can reach `X-Session-Id`. Fetch rejects those header values at send time, turning an optional affinity header into a failed ZAI request instead of omitting it.
  - confidence: 0.9
- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:41-42`
  - body: The merge keeps an existing `x-session-id` and then adds `X-Session-Id`. Header normalization combines those into a single comma-joined value, so remove any existing session header case-insensitively before setting the canonical one.
  - confidence: 0.9
- **[P2] Use the repo-current pi-ai version:** `extensions/zai/package.json:7-9`
  - body: The new ZAI runtime dependency pins `@mariozechner/pi-ai` to `0.69.0`, while current main and existing provider plugins use `0.70.6`. That can install a second older provider runtime copy and diverge stream behavior.
  - confidence: 0.87
- **[P3] Add the required changelog credit:** `CHANGELOG.md:9`
  - body: The added changelog entry lacks a `Thanks @...` attribution, but repo policy requires every added changelog bullet to include at least one credited GitHub username.
  - confidence: 0.84
