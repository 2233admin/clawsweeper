## Summary

Keep open. Current main has not implemented the requested behavior: startup secret activation still turns unresolved startup SecretRef preparation into a fatal startup error, and current channel secret tests still assert unresolved active Discord/Telegram env refs reject. The PR is useful but not merge-ready because the branch is nonmergeable/stale against the current gateway startup/reload layout and the review stream still has recovery/restart concerns after the latest head commit.

## What This Changes

The PR adds gateway startup logic to disable active channels whose env-backed startup secrets are missing, carries a recovery config for later `secrets.reload`, adds merge-patch support for recovery forwarding, and adds gateway reload regression coverage.

## Best Possible Solution

Rebase or replace this branch with a current-main implementation that performs channel-local startup degradation inside the current startup secrets activator, limits degradation to missing env availability errors, preserves config write/recovery invariants, and proves immediate recovery through `secrets.reload` with focused regression tests.
