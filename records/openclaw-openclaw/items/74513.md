## Summary

Keep this PR open. Current main still reports Full Release Validation queue time from the child workflow run creation timestamp, so downstream jobs blocked by `needs` can be misreported as runner-queued; the PR is a narrow fix with matching test coverage and no blocking review finding.

## What This Changes

This PR changes the Full Release Validation queue summary to fetch Actions jobs via the paginated jobs API and compute queue minutes from each job's `created_at` and `started_at` timestamps, with workflow-test assertions updated accordingly.

## Best Possible Solution

Land this focused workflow fix after maintainer review and normal workflow/changed validation so Full Release Validation reports actual runner queue wait separately from upstream DAG wait.
