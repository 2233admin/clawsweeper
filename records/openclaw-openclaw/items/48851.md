## Summary

Keep this PR open. Current main still does not track, persist, or display API call counts in `/status` or the response usage footer. The linked feature issue #48779 is already closed and the PR discussion addressed useful review feedback around followup turns and tool-loop calls, but the implementation has not landed on main and the status/usage code has since moved.

## Best Possible Solution

Keep the PR open as an implementation candidate. The best path is to rebase it onto current main, move the display changes into `src/status/status-message.ts` and `src/auto-reply/reply/agent-runner-usage-line.ts`, preserve the runner/subscription/session-store plumbing for per-run call counts, pass the field through both main and followup persistence paths, and add tests for status output, response footer output, followup turns, tool-loop multi-call turns, and zero/no-usage semantics before review or landing.
