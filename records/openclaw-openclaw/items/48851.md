## Summary

Keep this PR open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still does not implement the intended API/model call-count display in /status or the response usage footer, and it has no session or runner metadata path to persist that count. The PR remains a plausible implementation candidate: the author responded to bot review feedback by adding followup persistence and tool-loop counting commits, but it needs a rebase/port onto the current status and usage-line split plus a maintainer decision on zero/missing-usage semantics.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, plumb a per-run model/API call count through the embedded subscription/attempt/run metadata, persist it on SessionEntry from both main and followup paths, render it in /status and the response usage footer, and add focused tests for status output, footer output, followup turns, tool-loop multi-call turns, and zero/no-usage behavior.
