## Summary

Keep open. Current main still has the Feishu string-path upload helper behavior this PR targets: both helpers create fs.ReadStream instances and await Feishu SDK upload calls without a try/finally cleanup path if the SDK rejects. The PR remains a narrow implementation candidate, not obsolete on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a rebased, cleaned-up variant that wraps uploadImageFeishu and uploadFileFeishu SDK calls in try/finally, destroys only ReadStreams created from string-path inputs, preserves Buffer behavior, and adds focused rejection-path regression tests. If maintainers no longer want string-path helper support, remove that contract explicitly instead of leaving a leak-prone branch.
