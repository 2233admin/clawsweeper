## Summary

Keep open. Current main still has the Feishu string-path upload helpers creating ReadStreams without try/finally cleanup on SDK failure, while this PR is a focused one-file fix for that still-current behavior. The normal outbound path currently passes Buffers, but the helper contract still explicitly accepts string paths, and existing tests do not cover cleanup on the failing upload path.

## Best Possible Solution

Keep this PR open and either land a cleaned-up variant or ask the contributor to add regression tests. The best code path is to make uploadImageFeishu and uploadFileFeishu clean up only streams they create from string paths on both success and failure, while preserving Buffer behavior. If string-path inputs are no longer intended, remove that support explicitly instead of leaving an untested partial branch.
