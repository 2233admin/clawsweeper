## Summary

Keep this open. Current main still renders the `memory status` `Dreaming` line from the deep/short-term promotion resolver, so a globally enabled light-only configuration can still be reported as `Dreaming: off`. The issue is also paired with open, unmerged PR #67910, which explicitly proposes the phase-aware status fix, so this issue should wait for that PR or a replacement fix to land.

## What This Changes

The issue asks for `openclaw memory status` to report active light/deep/REM dreaming phase state instead of treating deep dreaming as the only source of truth.

## Best Possible Solution

Review and either land or replace #67910 with a narrow `memory-core` fix: derive the status summary from the full dreaming config or phase resolvers, report `off` only when no enabled phase is active, and add a CLI regression for global dreaming enabled with light enabled and deep disabled.
