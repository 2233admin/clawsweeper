## Summary

Keep open. Current main has compaction mitigations and CLI reseed handling, but I do not see source, docs, or regression coverage proving the reported stale-summary/topic-switch behavior is fixed for normal embedded sessions. The prior discussion's `npx clawaid` suggestion is a useful gateway health workaround, but it does not address the prompt/context ordering contract described in the report.

## Best Possible Solution

Keep this issue open. Add a focused regression in the embedded/default prompt or context assembly path: create a transcript with an old-topic compaction summary, append a fresh unrelated user message after the boundary, and assert the assembled prompt/trajectory makes the latest user request the active task while treating the summary as historical background. If it fails, fix this generically in prompt/context assembly or the legacy context engine rather than with extension-specific topic keywords.
