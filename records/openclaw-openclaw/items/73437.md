## Summary

Keep open. Current main still lacks a public per-LLM-call full-message rewrite hook: attempt-level context-engine assembly still runs before prompt submission, the loop-time assemble wrapper is limited to engines with `ownsCompaction: true`, and the public plugin hook catalog has observation hooks but no `transform_context`-style rewrite contract. The related implementation PR was closed unmerged, so this remains a valid maintainer API decision rather than cleanup that can be closed.

## What This Changes

Request a per-LLM-call context rewrite or assembly surface so context engines and plugins can inspect and adjust the full message array during multi-call tool loops.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract for plugin/context-engine authors, preserve existing `ContextEngine.assemble` compatibility unless maintainers explicitly choose a versioned semantic change, and add regression coverage for multi-tool-call loops, owning engines, non-owning engines, hook ordering, and budget enforcement.

## Reproduction Assessment

Yes for the code-level behavior. Current main shows the one pre-prompt assembly path, an internal loop wrapper only for `ownsCompaction` engines, and no public `transform_context` hook; a live multi-call run would be useful but is not necessary to reproduce the API gap.

## Solution Assessment

Unclear. The additive `transform_context` direction from the related PR appears safer than changing `assemble` frequency, but the exact hook ordering, permissions, and budget-enforcement guarantees need maintainer API review before implementation.
