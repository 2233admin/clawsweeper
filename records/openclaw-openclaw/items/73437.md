## Summary

Keep open. Current main only partially addresses loop-time context-engine assembly: owning context engines get an internal per-loop wrapper, but non-owning engines still get the generic tool-result guard plus the single pre-prompt assembly path. The requested public per-LLM-call context rewrite hook is still absent, and the related implementation PR was closed unmerged.

## What This Changes

Request to make context-engine assembly or an equivalent plugin context-transform hook run before every LLM call inside multi-call agent tool loops.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract for the plugin/context-engine API, preserve existing context-engine compatibility, and add regression coverage for multi-tool-call loops across owning engines, non-owning engines, and the public hook surface.
