## Summary

Keep open. Current main has an internal per-call assembly path for context engines that set `ownsCompaction: true`, and the upstream agent loop does call `transformContext` at the LLM-call boundary, but the requested public/non-owning full-message rewrite surface is still absent: `assemble` still runs in pre-prompt setup, the loop wrapper is gated to owning engines, and the related `transform_context` PR is closed unmerged.

## What This Changes

The issue requests that `ContextEngine.assemble` or an equivalent hook run before each LLM call in an agentic tool loop rather than only once per user prompt.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract for plugin/context-engine authors. Preserve existing `ContextEngine.assemble` compatibility unless maintainers explicitly choose a semantics change, and add regression coverage for owning engines, non-owning engines, and multi-tool-call loops.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current main shows pre-prompt assembly, an internal owns-compaction-only loop wrapper, and no public `transform_context` hook; a live multi-call LLM run would add end-to-end proof but is not required to see the API gap.

## Solution Assessment

Unclear. The existing `ownsCompaction` context-engine path is a plausible answer for selected compression engines, while the additive hook direction from PR #57585 may be safer than changing `assemble` semantics for every engine; maintainers should choose the public contract first.
