## Summary

Keep this open. Current main still carries the native Linux clipboard dependency path, upstream pi-coding-agent latest still loads @mariozechner/clipboard on Linux display sessions unless TERMUX_VERSION is set, and OpenClaw does not yet provide its own supported clipboard-disable guard before gateway and embedded Pi imports. The reporter and follow-up commenter provided useful coredump/workaround evidence, including a cleaner TERMUX_VERSION workaround, but that is still not an OpenClaw-owned fix.

## What This Changes

Prevent OpenClaw Gateway from crash-looping when pi-coding-agent loads the native @mariozechner/clipboard Linux addon by making that clipboard path optional, lazy, or isolated for gateway/server deployments.

## Best Possible Solution

Add a supported OpenClaw-owned guard or lazy import boundary that prevents the upstream native clipboard addon from loading before the first Pi SDK import in gateway/background paths, while preserving default desktop clipboard behavior unless an explicit opt-out or clearly server/headless mode applies. Document the supported workaround and avoid patching node_modules.
