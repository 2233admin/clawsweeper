## Summary

Keep this issue open. Current main upgraded @mariozechner/pi-coding-agent from the latest shipped release's 0.70.2 to 0.70.5, but the dependency graph still includes @mariozechner/clipboard-linux-x64-gnu, upstream 0.70.5 and 0.70.6 still load @mariozechner/clipboard on Linux display sessions unless TERMUX_VERSION is set, and OpenClaw still has gateway/session/embedded-run imports that can evaluate the top-level Pi package before any OpenClaw-owned clipboard opt-out exists.

## What This Changes

The issue reports OpenClaw Gateway crash loops on Linux caused by the native @mariozechner clipboard addon being loaded through the embedded Pi coding agent runtime, and asks for a safer opt-out, lazy load, or isolation path.

## Best Possible Solution

Add a supported OpenClaw-owned guard that prevents the upstream native clipboard addon from loading before any top-level Pi import in gateway/background embedded paths, or upgrade to an upstream Pi release that provides an explicit non-native clipboard disable. Keep default desktop behavior intact unless the new opt-out or a clearly headless/server mode applies, and document the supported workaround.
