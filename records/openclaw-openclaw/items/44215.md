## Summary

Keep this PR open. Current main still lacks Windows-specific PATH bootstrap directories in `ensureOpenClawCliOnPath`, so the requested behavior is not implemented, but the submitted diff needs maintainer-guided rework because it changes PATH command-resolution precedence in a security-sensitive boundary.

## What This Changes

The PR adds win32 PATH bootstrap candidates and platform delimiter handling in `src/infra/path-env.ts`, adds a Windows regression test in `src/infra/path-env.test.ts`, and includes an unrelated `gateway.push.*` schema-label hunk.

## Best Possible Solution

Keep this PR open for maintainer-guided rework. A safe fix should preserve the hardened PATH model, add only carefully chosen Windows system/runtime bootstrap directories, avoid default precedence for user-writable package-manager or WindowsApps aliases, handle Windows semicolon delimiters deterministically, remove the unrelated schema-label hunk, and add focused regression tests. The related `env.vars.PATH` part of #19910 should remain governed by the existing rejected-PATH-override and `tools.exec.pathPrepend` policy.
