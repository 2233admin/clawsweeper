## Summary

Keep this PR open. Current main still does not implement the Windows-specific PATH bootstrap or win32 delimiter handling, but the submitted branch is not merge-ready because it changes a command-resolution security boundary by prepending user-profile Windows package-manager and WindowsApps alias directories ahead of the existing PATH, has focused test gaps, and carries an unrelated schema-label hunk already present on main.

## What This Changes

The PR adds a win32 PATH bootstrap branch and semicolon delimiter handling in `src/infra/path-env.ts`, adds a Windows PATH regression test, and adds `gateway.push.*` schema labels.

## Best Possible Solution

Port the useful Windows support into the current hardened PATH model: handle semicolon-delimited Windows PATH deterministically, add only defensible Windows system/runtime bootstrap directories by default, keep user-writable package-manager and WindowsApps paths behind append/opt-in policy, remove the unrelated schema-label changes, and cover the behavior with focused regression tests.
