## Summary

Keep this PR open. Current main still does not implement the requested Windows PATH bootstrap behavior, but the proposed diff is not ready to land because it conflicts with the current hardened PATH model and carries unrelated/redundant schema-label changes.

## Best Possible Solution

Keep this PR open as useful but not ready. The maintainer path is to rebase it onto current main and rewrite the Windows bootstrap around the hardened model: use validated Windows install roots for trusted system/runtime dirs, use the Windows delimiter in tests and implementation, avoid prepending WindowsApps or user-writable package-manager shims, append or explicitly opt into mutable dirs, add deterministic coverage including `ProgramFiles(x86)`, and drop the unrelated schema-label hunk.
