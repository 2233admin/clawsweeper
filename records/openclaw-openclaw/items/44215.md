## Summary

Keep this PR open. Current main still does not implement the Windows PATH bootstrap requested by the PR, but the submitted patch is not ready to merge because it conflicts with the current PATH hardening model, has test-ordering/delimiter issues, and carries an unrelated schema-label hunk that current main already covers elsewhere.

## Best Possible Solution

Keep this PR open as a useful Windows bootstrap fix that needs maintainer-guided rework. The best path is to rebase onto current main, remove the unrelated schema-label changes, preserve the hardened PATH model, use the correct Windows delimiter, add trusted Windows system/runtime dirs carefully, append or explicitly opt into user-writable package-manager dirs, avoid prepending WindowsApps aliases, and add deterministic Windows tests including `ProgramFiles(x86)`. The linked `env.vars.PATH` part of #19910 should remain handled by the existing rejected-override/pathPrepend policy rather than by raw PATH overrides.
