## Summary

Keep this issue open. Current main still has raw prompt-mutating hook result fields and several hardcoded sanitizer paths for known metadata or memory tags, but it does not have the requested declarative plugin manifest field, aggregated registry, or core getter for plugin-injected context strip/sanitize metadata. This is a public plugin-manifest/SDK contract with privacy and downstream ingestion implications, so it needs maintainer API review rather than auto-close or ClawHub routing.

## Best Possible Solution

Keep this issue open for maintainer API review. The likely path is an additive, documented manifest/SDK contract for plugin-owned injected context blocks, a central registry/getter built from plugin manifests, and careful adoption by display/history, assistant-visible text, compaction, session export, and memory/dreaming ingestion paths with tests for user-authored lookalikes and fenced-code preservation. Coordinate with #54373 without treating it as a duplicate.
