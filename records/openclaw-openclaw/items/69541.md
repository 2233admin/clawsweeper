## Summary

Keep #69541 open. Current main has prompt-injection hooks and several point sanitizers, but it does not implement the RFC’s declarative `contracts.injectedContextTags` manifest field, an aggregate core getter, or registry-driven stripping across display, dreaming, compaction, and memory-retain consumers. Existing producers and consumers still rely on opaque prompt strings plus hardcoded tag/sentinel lists.

## Best Possible Solution

Keep this RFC open as core/plugin-contract work. The best path is to add a backwards-compatible manifest contract for injected context blocks, validate and normalize it in `src/plugins/manifest.ts`, expose a deterministic aggregate getter from the plugin registry/loader, declare bundled plugin tags, and migrate the display/history, assistant-visible text, memory-host session extraction, dreaming ingestion, and compaction/retain consumers to use the registry while preserving current hardcoded defaults for compatibility. The related provenance discussions can inform the design, but current main does not supersede this specific strip/sanitize/ingest contract.
