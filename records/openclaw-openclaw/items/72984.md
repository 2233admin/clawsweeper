## Summary

Keep open. Current main does not fully implement the PR's intended subagent-default precedence/fallback behavior, but the proposed patch still has concrete alias-resolution bugs in both new fallback paths, and the remaining behavior overlaps with active related work and a product-visible precedence decision already flagged in review discussion.

## What This Changes

The PR changes subagent configured model precedence, adds gateway and reply-time subagent default fallback helpers with tests, and also touches qa-lab type exports plus config-validation cleanup.

## Best Possible Solution

Rebase onto current main, keep the existing selected override persistence, make any reply/gateway fallback reuse the same alias-aware subagent spawn resolver or a shared alias-aware helper, and align docs/tests with the maintainer-approved precedence rule.

## Reproduction Assessment

Yes. Configure `agents.defaults.models` with an alias such as `gpt` for `openai/gpt-5.4` and set a subagent default to `gpt`; the PR's new fallback paths parse that bare alias under the parent/default provider instead of resolving the alias index.

## Solution Assessment

No. The direction is plausible, but this patch is not the best implementation because it duplicates fallback resolution without alias support and leaves the precedence change as an implicit behavior change; a shared resolver with explicit precedence tests is safer.

## Review Findings

Overall correctness: patch is incorrect
