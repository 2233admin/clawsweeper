## Summary

Keep open. Current main does not contain the fallback/precedence changes this PR proposes, but the proposed diff still introduces alias-resolution regressions in both new fallback paths and overlaps with open #72877, so it needs author/maintainer revision rather than cleanup closure.

## What This Changes

The PR changes subagent configured-model precedence, adds subagent-aware gateway and reply default fallbacks with tests, adjusts config-validation style, and exports a qa-lab runtime type.

## Best Possible Solution

Revise this PR or the overlapping canonical work so all subagent runtime fallback paths share the alias-aware spawn resolver or a small shared alias-aware helper, then settle the global subagent default precedence in code, tests, and docs before merge.

## Reproduction Assessment

Yes. Configure `agents.defaults.models` with an alias such as `gpt` for `openai/gpt-5.4` and set a subagent default to `gpt`; the proposed fallback paths feed that raw alias to `parseModelRef`, which source inspection shows resolves bare strings under the default provider.

## Solution Assessment

No. The direction is plausible, but this patch is not the best version because it bypasses the existing alias-aware resolver and changes precedence while related #72877 remains open; a shared resolver plus an explicit precedence decision is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Resolve aliases in reply-time subagent defaults:** `src/auto-reply/reply/directive-handling.defaults.ts:56`
  - body: `resolveSubagentConfiguredModelSelection` can return a configured alias such as `gpt`, but this helper passes the raw value to `parseModelRef`. In that configuration a subagent can boot and persist `<parent-provider>/gpt` instead of the configured provider/model. Reuse the alias-aware spawn resolver or the same alias index here.
  - confidence: 0.91
- **[P2] Resolve aliases in gateway subagent fallback:** `src/gateway/session-utils.ts:1135`
  - body: The new `resolveSessionModelRef` fallback has the same alias gap when `entry.model` is empty: a configured subagent alias is parsed as a bare model under the default provider rather than resolved through the alias index. That makes the race fallback disagree with the spawn-time model selection already covered by tests.
  - confidence: 0.88
