## Summary

Current main still does not migrate legacy `plugins.entries.memory-core.config.dreaming.storage: "both"` into the supported object shape. The total CLI outage is partly mitigated for allowlisted commands and gateway degraded startup, but strict plugin config validation still rejects the legacy value and `doctor --fix` currently quarantines the plugin config instead of preserving the storage mode.

## Best Possible Solution

Keep this issue open and add a narrow memory-core compatibility migration that converts legacy `dreaming.storage` strings (`inline`, `separate`, `both`) to `{ mode, separateReports: false }` before generic invalid-plugin quarantine runs. Keep the manifest schema strict, preserve the configured mode, add focused regression coverage, and include a changelog fix entry crediting the reporter.
