## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Forward the abort signal through the guard options:** `extensions/browser/src/browser/client-fetch.ts:189-190`
  - body: `fetchWithSsrFGuard` composes cancellation from its top-level `signal` option and then overwrites `init.signal` when `timeoutMs` creates a guard signal. With this PR, caller aborts passed to `fetchBrowserJson` no longer cancel the browser-control request promptly and instead wait for the timeout; pass `ctrl.signal` as the guard `signal` option too.
  - confidence: 0.9
- **[P2] Keep Telegram promotion keys behind the lazy facade:** `src/plugin-sdk/telegram.ts:25-26`
  - body: Initializing `namedAccountPromotionKeys` with `loadFacadeModule().namedAccountPromotionKeys` loads the bundled Telegram contract at `openclaw/plugin-sdk/telegram` import time. That breaks the lazy facade pattern used by adjacent exports and can make unrelated consumers pay the Telegram contract load or fail early; wrap it with the lazy array helper.
  - confidence: 0.86
- **[P2] Use the narrow account-id facade in Telegram setup:** `extensions/telegram/src/setup-contract.ts:1`
  - body: `setup-contract.ts` is imported by the hot Telegram channel setup path, but this import pulls from the broad `openclaw/plugin-sdk/setup` barrel. The file only needs `DEFAULT_ACCOUNT_ID` and `normalizeAccountId`, so use the narrow routing/account-id facade as Matrix does to avoid pulling setup-wizard helpers into startup.
  - confidence: 0.78
