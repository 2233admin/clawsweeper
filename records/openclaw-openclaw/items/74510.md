## Summary

Keep open. Current main still has the reported visibility gap for phaseless persisted assistant commentary: explicit commentary phase is hidden, but a text-only assistant message with no phase/textSignature and stopReason=toolUse still flows through chat.history and the Control UI shared visible-text extractor as a normal assistant reply.

## What This Changes

Control UI should distinguish or suppress intermediate assistant commentary/progress text so a single tool-using turn renders the final answer as the primary assistant reply.

## Best Possible Solution

Preserve commentary/final_answer metadata end to end where providers and persistence can supply it, and add a narrow legacy projection/rendering fallback for phaseless text-only intermediate assistant entries so the final answer stays primary without suppressing legitimate tool-call or media content.

## Reproduction Assessment

Yes. A source-level fixture with a user message, an assistant text-only entry with no phase/textSignature and stopReason="toolUse", then a final assistant answer exercises the current path: phase resolution is undefined, chat.history keeps the intermediate entry, and Control UI extraction can render it.

## Solution Assessment

Yes. Metadata preservation plus a constrained legacy fallback is the maintainable direction; hiding every assistant stopReason="toolUse" entry would be too broad because some tool-use turns may carry legitimate visible content or non-text blocks.
