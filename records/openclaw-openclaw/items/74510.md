## Summary

Keep open. Current main handles explicit `commentary`/`final_answer` metadata and some producer paths, but the reported persisted shape with unphased assistant text plus `stopReason: "toolUse"` can still pass through `chat.history` and Control UI as a normal assistant bubble.

## What This Changes

Prevent Control UI history from showing intermediate assistant commentary or progress messages as normal assistant replies alongside the final answer for one tool-using turn.

## Best Possible Solution

Preserve `commentary` and `final_answer` metadata through provider conversion, persistence, `chat.history`, and Control UI extraction, then add a narrow legacy fallback for already-persisted unphased intermediate assistant rows so the final answer remains the primary visible reply.

## Reproduction Assessment

Yes. A source-level fixture with an assistant text message lacking phase/textSignature and carrying `stopReason: "toolUse"` followed by the final assistant answer follows the current projection and UI extraction path and remains visible.

## Solution Assessment

Yes, the issue's proposed direction is the right shape: metadata preservation plus a constrained legacy fallback. Hiding every `toolUse` assistant row would be too broad because some pre-tool text can be intentionally user-visible.
