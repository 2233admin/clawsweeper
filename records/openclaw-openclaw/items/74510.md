## Summary

Keep this open: current main still has the reported failure mode when persisted assistant commentary lacks phase metadata. Phased commentary is handled, but unphased `stopReason: "toolUse"` assistant text can still pass through `chat.history` and Control UI as a normal assistant bubble.

## What This Changes

The issue requests preserving or reconstructing commentary versus final-answer metadata so Control UI does not render intermediate assistant progress as same-level replies beside the final answer.

## Best Possible Solution

Preserve `commentary` and `final_answer` metadata end to end through provider conversion, transcript persistence, `chat.history`, and Control UI extraction, then add a narrow legacy fallback for the reported unphased intermediate-commentary shape so the final answer remains the primary visible reply.

## Reproduction Assessment

Yes. A source-level fixture with an unphased assistant message containing plain text plus `stopReason: "toolUse"` reproduces the path: phase resolves to undefined, history projection keeps it, and Control UI extraction renders the text.

## Solution Assessment

Yes, a narrow metadata-preservation fix plus a constrained legacy fallback is the best maintainable direction. Hiding all tool-use assistant turns by stop reason alone would be too broad because some pre-tool text may be intentionally visible.
