## Summary

Keep open. Current main handles explicitly phased commentary, but the exact phaseless persisted shape described in the report can still survive chat history projection and render in Control UI because unphased assistant text remains a visible legacy fallback.

## What This Changes

The issue requests preserving or inferring assistant commentary/final-answer metadata so intermediate commentary is not rendered as same-level Control UI replies beside the final answer.

## Best Possible Solution

Preserve commentary/final_answer metadata end to end for new transcript entries, and add a narrow legacy fallback in history projection or shared extraction for phaseless intermediate tool-use assistant text so final answers remain the primary visible reply.

## Reproduction Assessment

Yes. A source-level fixture matching the report, assistant text with no phase/textSignature and stopReason toolUse followed by a final assistant message, follows a clear path through chat.history projection and Control UI extraction where the commentary text remains visible.

## Solution Assessment

Yes. The suggested direction is the right maintainable fix: preserve explicit phase metadata where available and add a constrained legacy fallback, rather than using stopReason alone as a broad visibility rule.
