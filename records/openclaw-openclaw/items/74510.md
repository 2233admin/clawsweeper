## Summary

Keep open: current main still has the reported failure mode for persisted assistant commentary that lacks phase metadata. Explicit `commentary` / `final_answer` metadata is handled, but an unphased assistant text entry with `stopReason: "toolUse"` can still survive `chat.history` projection and be rendered by Control UI as a normal assistant bubble.

## What This Changes

The issue asks OpenClaw to preserve or infer assistant commentary/final-answer phase boundaries so Control UI does not show intermediate progress text as same-level replies beside the final answer.

## Best Possible Solution

Preserve `commentary` and `final_answer` metadata through provider conversion, transcript persistence, history projection, and Control UI extraction, then add a narrow legacy fallback for already-persisted unphased intermediate commentary so the final answer remains the primary visible reply.

## Reproduction Assessment

Yes. A source-level fixture with an unphased assistant text message using `stopReason: "toolUse"` followed by a final assistant answer reproduces the path: phase resolution returns undefined, `chat.history` keeps it, and Control UI extraction renders it as visible text.

## Solution Assessment

Yes. A metadata-preservation fix plus a constrained legacy fallback is the narrowest maintainable solution; hiding every `toolUse` assistant message would likely suppress legitimate visible pre-tool text.
