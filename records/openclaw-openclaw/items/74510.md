## Summary

Keep open; the existing ClawSweeper comment is still accurate on current main. The code handles explicit `phase`/`textSignature` metadata, but the reported unphased `stopReason=toolUse` assistant row still resolves to no phase and remains renderable as a normal assistant message.

## What This Changes

The issue requests that Control UI distinguish or suppress intermediate assistant commentary/progress messages so the final answer remains the primary assistant reply for one user turn.

## Best Possible Solution

Preserve `commentary` and `final_answer` metadata through provider output, transcript persistence, and `chat.history`, with a conservative legacy fallback that collapses or hides only unphased intermediate commentary rows while keeping legitimate assistant/tool-call content visible.
