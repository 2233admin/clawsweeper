## Summary

Current main still gives Feishu WebSocket an ambient proxy-derived agent, and the regression tests still expect that behavior. This issue should stay open because #65802 is an open, unmerged implementation PR with closing syntax for the same requested fix.

## Best Possible Solution

Keep this issue open and review #65802 as the focused implementation candidate. The fix should stay in the Feishu plugin boundary: bypass ambient proxy inheritance for Feishu WebSocket by default, add a deliberate opt-in only if maintainers want to support proxying, update `extensions/feishu/src/client.test.ts`, and document any supported opt-in env.
