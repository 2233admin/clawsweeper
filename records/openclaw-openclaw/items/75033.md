## Summary

Keep this PR open for maintainer review. Current main still has the quoted HTTP-client diagnostic redaction gap, while the proposed patch is narrow, security-positive, and I found no blocking review findings in the diff.

## What This Changes

The PR adds shared redaction patterns for quoted HTTP client config/header fields and regression tests for `redactSensitiveText` plus `formatErrorMessage` cause-chain output.

## Best Possible Solution

Review and land this shared redaction update if maintainers are satisfied with the security boundary, while keeping the broader exec-output and config-command exposure work on their separate tracks.

## Reproduction Assessment

Yes. A code-level reproduction is an Error cause message like `request config: { appSecret: '...', headers: { authorization: 'Bearer ...' } }` passed through `formatErrorMessage`; current main appends that cause text and applies defaults that do not match those quoted fields.

## Solution Assessment

Yes for the narrow HTTP-client diagnostics gap. Extending the shared default redaction patterns is the smallest maintainable fix because logging, tool-payload, and error-formatting surfaces already depend on `redactSensitiveText`.
