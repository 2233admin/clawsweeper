## Summary

Keep this issue open. Current main still treats fallback as provider/model candidate selection, with no way for a fallback attempt to switch agent identity, workspace, agentDir, session file, skills snapshot, or model-specific context profile. The recent maintainer/Codex discussion also explicitly kept the issue open for design rather than cleanup closure.

## What This Changes

Request agent-level fallback, or equivalent fallback-specific workspace/context controls, so smaller fallback models can use a minimal agent context instead of the primary agent workspace.

## Best Possible Solution

Keep the issue open for a first-class core design: choose fallbackAgent/fallbackWorkspace semantics or a narrower model-specific context profile/budget mechanism, then update config types, strict schemas, schema help/docs, fallback candidate metadata, reply-runner/session plumbing, and regression tests proving fallback retries use alternate context without corrupting the original session.
