## Summary

Keep open. Current main still implements model fallback as provider/model candidate selection while preserving the original agent/session/workspace context; there is no config, schema, candidate metadata, or reply-runner path for switching to another agent, workspace, skills snapshot, session file, or model-specific context profile. The recent issue discussion also treats this as maintainer design work rather than cleanup closure.

## What This Changes

Requests a fallback mode that can switch to another configured agent or otherwise use a smaller model-specific workspace/context profile when the primary model fails.

## Best Possible Solution

Design a first-class fallback-context contract, then update config types, strict schemas, schema help/docs, fallback candidate metadata, reply-runner/session plumbing, and regression tests so fallback retries can use alternate context without corrupting the original agent state.
