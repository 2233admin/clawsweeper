## Summary

Keep this issue open. Current main still implements fallback as provider/model candidate selection, not as a way to switch to another agent, workspace, session file, skills snapshot, or context profile for fallback attempts. The issue also has a recent maintainer/Codex review comment explicitly keeping it open for design.

## What This Changes

Request agent-level fallback, or equivalent fallback-specific workspace/context controls, so smaller fallback models can use a minimal agent context instead of the primary agent workspace.

## Best Possible Solution

Keep this open for maintainer design. The best path is to choose a first-class core contract, either fallbackAgent/fallbackWorkspace semantics or a narrower model-specific context profile/budget mechanism, then update config types, strict schemas, schema help/docs, fallback candidate metadata, reply-runner/session plumbing, and regression tests proving fallback retries use alternate context without corrupting the original session.
