## Summary

Current main still strips Fireworks Kimi reasoning payload fields for every Kimi model, while this PR is a focused provider-local fix that now routes its regression test through the real Fireworks wrapper detection. The PR changes the Fireworks provider stream wrapper so Kimi K2.6 keeps root `reasoning`, `reasoning_effort`, and `reasoningEffort` payload fields while still forcing `thinking: { type: "disabled" }`, and adds Fireworks stream unit coverage for that behavior.
