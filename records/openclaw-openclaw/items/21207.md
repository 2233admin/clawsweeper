## Summary

Keep open. Current main still lacks the requested native batched file/search tool inputs, AI-callable named context block management tools, and opt-in submitted-plan step enforcement. Existing `update_plan`, strict-agentic retries, `/context`, `/compact`, and context-engine/plugin lifecycle hooks are adjacent groundwork, not the requested product/API surface.

## What This Changes

The issue requests native batch tool calls, AI-callable context block save/load/unload/list tools, and opt-in submitted-plan enforcement.

## Best Possible Solution

Keep this open and split it into maintainer-reviewed tracks: native/gateway-level batching for file and search tools, a deliberate context-block API if AI-controlled save/load/unload/list semantics are desired, and runner-owned opt-in plan state if strict submitted-plan gating is a product goal. Treat current `update_plan`, strict-agentic retries, `/context`, `/compact`, and context-engine lifecycle hooks as groundwork, not closure.
