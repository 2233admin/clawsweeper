## Summary

Keep open. Current main still has adjacent groundwork, but it does not implement the three requested native capabilities: batched file/search tool APIs, AI-callable named context block save/load/unload/list tools, or submitted-plan step enforcement.

## Best Possible Solution

Keep this open and split it into maintainer-reviewed tracks: native or gateway-level batching for file/search tools, an explicit AI-callable named context block API if save/load/unload semantics are a product goal, and runner-owned opt-in plan state with strict step gating if enforceable sequential execution is desired. Existing `parallel_tool_calls`, context-engine compaction/pruning, `/context`, `/compact`, `update_plan`, and strict-agentic retries should be treated as groundwork, not closure.
