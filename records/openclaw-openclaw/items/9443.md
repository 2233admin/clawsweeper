## Summary

Keep this issue open. Current main still does not publish official prebuilt Android APK release downloads: the public Android docs say the app is not publicly released, the Android README still marks release hardening incomplete, the release helper builds signed AAB bundles rather than APK release assets, CI only builds a Play debug APK for validation, and the latest GitHub release assets are macOS/source archives only. Existing discussion also points to community APK/client workarounds, but those remain unofficial and do not satisfy the request for official OpenClaw APK releases.

## Best Possible Solution

Keep this issue open as the canonical official Android APK release request. A complete fix should define the supported signed APK/flavor and distribution policy, add maintainer-approved release automation or another official distribution channel, publish official Android install assets where users are told to find them, and update https://docs.openclaw.ai/platforms/android with the official install path while keeping community APK links clearly unofficial unless adopted.
