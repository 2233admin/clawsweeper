## Summary

Keep this issue open. Current main still does not publish official prebuilt Android APK downloads: the public Android docs say the app is not publicly released, the Android release helper builds signed AAB bundles, CI only validates a Play debug APK build, and the latest release assets are macOS/source archives only. Related #35515 was already closed as a duplicate of this tracker and #53972 was closed unmerged as duplicate debug-CI work, while community APK/client links remain unofficial workarounds.

## Best Possible Solution

Keep this issue open as the canonical official Android APK release request. A complete fix should define the supported signed APK/flavor and distribution policy, add maintainer-approved release automation or an explicitly chosen distribution channel, publish official Android install assets where users are told to find them, and update https://docs.openclaw.ai/platforms/android with the official install path. Community APK links should remain documented, if at all, as unofficial workarounds unless maintainers explicitly adopt them.
