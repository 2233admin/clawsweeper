## Summary

Keep open. Current main still does not implement an agent-level delivery default for targetless programmatic or cross-session message sends; the existing fallbacks cover explicit message arguments, captured run/session delivery context, channel default behavior, and cron announce routing, but not the requested typed per-agent fallback.

## What This Changes

Add an optional per-agent default delivery target and channel so programmatic, webchat-routed, cron-created, and cross-session agent runs can use the message tool without reconstructing long channel IDs.

## Best Possible Solution

Define a typed, documented agents.list[].delivery contract using DeliveryContext-style fields such as channel, to, accountId, and threadId; validate it in config schema/help/docs, carry it through resolved agent config, and apply it only as the lowest-precedence trusted fallback after explicit message args and per-run/session deliveryContext.
