## Summary

Keep this PR open. Current main only partially has the plumbing: `sessions_spawn` now carries `requesterAgentIdOverride` in the ACP call context, but `spawnAcpDirect` still does not declare or consume that field, and ACP heartbeat/parent-stream eligibility still falls back to the parsed parent session key. That means the central behavior described by the PR is not implemented on main yet.

## Best Possible Solution

Review and land this PR or an equivalent rebased patch: add `requesterAgentIdOverride` to `SpawnAcpContext`, resolve ACP requester identity with override-first fallback in both heartbeat eligibility and requester-state resolution, and keep the regression tests for ACP tool passthrough plus implicit parent-stream gating.
