## Summary

Keep this issue open. Current main still resolves unset memory sync watching to true, and the normal gateway memory_search path can still open long-lived builtin or QMD chokidar watchers; the related watcher-freshness and ENOSPC work does not implement the requested gateway-default policy change.

## What This Changes

The issue requests changing the default memory search file watcher policy so headless gateway mode resolves unset `memorySearch.sync.watch` to `false` while preserving explicit user overrides and CLI/desktop defaults.

## Best Possible Solution

Add a runtime-aware memory sync default so unset `sync.watch` resolves to false only for the intended headless gateway manager context, while explicit true/false config remains authoritative and CLI/desktop interactive freshness stays covered by tests and docs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to leave `memorySearch.sync.watch` unset, resolve memory sync config for an enabled agent, then follow a gateway memory_search call through the default manager path where builtin or QMD managers start chokidar watchers because the resolved value is true.

## Solution Assessment

Yes, with a scoped implementation. The maintainable fix is not a blanket raw `gateway.mode` check, but a threaded headless/default-policy context in sync resolution or manager creation so user overrides and interactive local behavior remain stable.
