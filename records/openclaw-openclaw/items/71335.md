## Summary

Keep this issue open. Current main still resolves unset memory sync watch to true without any gateway/headless runtime input, and the long-lived builtin/QMD memory managers still create chokidar watchers when that resolved value is true. The related #64549 covered auto-indexing behavior in the same subsystem, but it does not supersede this default-policy request.

## What This Changes

Request that memorySearch sync.watch default to false for headless gateway operation while preserving explicit user overrides and the existing CLI/desktop default.

## Best Possible Solution

Add an explicit runtime-aware memory sync default so only the intended headless gateway path defaults watch to false, while explicit sync.watch true or false remains authoritative. Cover the resolver and both builtin and QMD manager paths, and update schema help or memory docs if the visible default changes.
