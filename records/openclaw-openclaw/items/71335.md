## Summary

Keep this open. Current main still resolves an unset memory sync watcher to `true`, and the normal long-lived memory manager path can still arm builtin or QMD chokidar watchers; the closed #64549 thread is adjacent watcher-freshness work, not this gateway default-policy change.

## What This Changes

Request that unset `agents.defaults.memorySearch.sync.watch` default to `false` for headless gateway mode while preserving explicit user overrides and CLI/desktop watch behavior.

## Best Possible Solution

Add an explicit runtime-aware default policy for memory sync so unset `sync.watch` resolves to false only in the intended long-lived headless gateway manager context, while explicit true/false config remains authoritative and transient CLI/status plus desktop/local freshness behavior stay covered by tests and docs.

## Reproduction Assessment

Yes. Leave `sync.watch` unset, resolve memory sync config for an enabled agent, then follow a normal gateway `memory_search` call into the full manager path; current main resolves watch to true and can arm builtin or QMD chokidar watchers.

## Solution Assessment

Unclear as originally phrased. The policy direction is narrow and reasonable, but the maintainable implementation should pass an explicit headless/full-manager default-policy context instead of keying only on raw `gateway.mode`.
