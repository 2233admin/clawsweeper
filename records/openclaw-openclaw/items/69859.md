## Summary

Keep open. Current main still does not implement the intended /hooks/agent threadId ingress: the HTTP payload type and normalizer, direct dispatch job, idempotency replay scope, docs, and tests all omit the field. Downstream cron delivery now supports delivery.threadId, so the PR is still useful, but the submitted branch needs rework because it routes threadId through the wrong seam for current main, includes an unrelated model-selection change, and adds a backup source file.

## What This Changes

The PR branch adds threadId fields across the gateway hook and cron delivery path, changes cron model-selection provider-prefix handling, and adds a src/media/web-media.ts.backup file.

## Best Possible Solution

Keep this PR open, but require a focused rebase/rework: parse and normalize an optional string or numeric threadId in /hooks/agent, include it in the replay idempotency scope, pass it into job.delivery.threadId, add regression coverage for Slack-style threaded hook dispatch, update the public hook docs, remove the backup file, and split the model-selection fix into a separate PR if still needed.
