## Summary

Keep open. Current main still has lower-level cron delivery support for explicit `threadId`, but the direct `POST /hooks/agent` path does not accept, normalize, document, or dispatch a posted `threadId`. This PR remains the canonical implementation thread for the closed related request #69856, but it needs rebase/rework because the submitted branch had a no-op propagation gap, an unrelated model-selection change, and an accidental backup file.

## Best Possible Solution

Keep this PR open, but require a focused rebase/rework: add `threadId` to the direct hook payload type and normalizer, include it in replay idempotency scope, carry it through `dispatchAgentHook` into `job.delivery.threadId`, add regression coverage for `/hooks/agent` dispatching a Slack-style thread id, update the public hook docs, remove the backup file, and split the unrelated model-selection fix into its own PR if still needed.
