## Summary

Keep this issue open. Current main still has both reported paths: the reminder guard only suppresses by matching the current session key, while isolated cron jobs are documented and implemented as dedicated fresh-session runs; compaction retry cleanup still clears `successfulCronAdds` after a successful `cron.add`. The provided discussion also identifies #43404 as an open implementation candidate with closing syntax for this issue, so the right maintainer path is to review or land that PR rather than close the report now.

## Best Possible Solution

Keep this issue open and review or land #43404, or an equivalent focused fix. The fix should pass agent identity into the reminder-guard lookup, treat enabled same-agent isolated cron jobs as coverage when appropriate, preserve `successfulCronAdds` across compaction retry cleanup, and add regression tests for both isolated-cron suppression and counter preservation through compaction.
