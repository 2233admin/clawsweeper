## Summary

Keep open. Current main still does not implement the PR's central behavior: Discord sender identity is not resolved through session.identityLinks, Discord inbound/native/component contexts do not carry a trusted sender principal, inbound trusted metadata does not render sender_principal, and the routing helper remains private rather than exported through the plugin SDK. The related #57709 is closed and unmerged in the provided context, so it does not supersede this PR.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, the best implementation path is a scoped Discord/inbound-metadata change: resolve canonical principals from session.identityLinks using trusted Discord snowflakes, prefer PluralKit pkInfo.sender when available, thread the value through preflight/process/component/native-command contexts as TrustedSenderPrincipal, render sender_principal in trusted inbound metadata, and either intentionally promote a generic routing helper through the plugin SDK with contract updates or keep the helper local if no public seam is needed.
