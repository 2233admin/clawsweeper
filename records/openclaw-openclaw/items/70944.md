## Summary

Keep #70944 open. Current main still only uses `session.identityLinks` to collapse direct-message session keys; it does not resolve Discord senders to a trusted canonical principal, does not thread that value through Discord preflight/process/native command contexts, and does not render `sender_principal` in the trusted inbound metadata prompt. The related #57709 is closed unmerged in the provided context, so it does not supersede this PR’s remaining work.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer branch. The remaining implementation should add a Discord-owned sender-principal resolver using `session.identityLinks`, thread it through Discord preflight, regular message processing, component/native command contexts, and shared inbound templating, and add focused tests for regular Discord users, PluralKit `pkInfo.sender`, no-match behavior, and `sender_principal` rendering. If the shared identity-link helper becomes public SDK, update the SDK contract/baseline/docs in the same change.
