## Summary

Keep open because current main still provides only partial spawned-session lineage. The code has spawnedBy, parentSessionKey, spawnDepth, and childSessions in some surfaces, but it does not implement the requested rootSessionKey and session-chain traceId contract across persistence, gateway schema/projections, agent-facing sessions_list output, or native/ACP spawn writes; the related implementation PR is closed and unmerged.

## What This Changes

The issue requests first-class spawned-session chain tracking with parent, root, trace, and depth metadata exposed through session APIs.

## Best Possible Solution

Land a focused additive core contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP child sessions, expose them through gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving older session compatibility.
