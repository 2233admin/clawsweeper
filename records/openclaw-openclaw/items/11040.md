## Summary

Keep open: current main still has only partial spawned-session lineage. It persists and exposes spawnedBy, parentSessionKey, spawnDepth, and childSessions in some paths, but it does not implement the requested first-class rootSessionKey and session-chain traceId contract across the persistent session model, gateway schema/projections, agent-facing sessions_list rows, or native/ACP spawn writes. The related implementation PR is closed and unmerged, and the issue discussion already points to a focused replacement rather than closure.

## What This Changes

Add backward-compatible first-class session/task chain metadata so spawned sessions expose parent/root/trace/depth lineage through persisted session entries, spawn writes, gateway session listing, and agent session-list rows.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and session-chain traceId, derives them consistently for native and ACP child sessions, exposes them through gateway and agent session-list projections, documents the fields, and proves multi-hop reconstruction while preserving compatibility for existing entries.
