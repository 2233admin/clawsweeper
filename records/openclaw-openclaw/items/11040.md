## Summary

Keep open because current main still only has partial parent/depth lineage and does not implement the requested first-class root/trace session-chain contract. The linked implementation PR is closed and unmerged, and the existing maintainer review comment reached the same conclusion.

## What This Changes

Add optional first-class spawned-session chain metadata, including parent/root/trace/depth, and expose it through session listing APIs and agent tooling.

## Best Possible Solution

Add optional persisted chain metadata to the session model and gateway/tool projections, derive it consistently in native and ACP spawn paths, document the fields, and prove multi-hop reconstruction with regression tests while preserving backward compatibility for existing sessions.
