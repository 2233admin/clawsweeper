## Summary

Keep open. Current main still implements only partial lineage (`spawnedBy`, `parentSessionKey`, `spawnDepth`, `childSessions`) and does not provide the requested Phase 1 session-chain contract because `rootSessionKey` and a session-chain `traceId` are absent from the persistent session model, gateway schema/projections/events, native/ACP spawn write paths, and agent-facing `sessions_list` rows. The directly related PR #11043 is closed and unmerged, and the maintainer comment already identified the same remaining gap.

## What This Changes

Add first-class, backward-compatible session/task chain metadata so spawned sessions persist and expose parent, root, trace, and depth information for deterministic DAG reconstruction.

## Best Possible Solution

Keep this open and implement the minimal Phase 1 core contract: persist optional `parentSessionKey`, `rootSessionKey`, `traceId`, and `spawnDepth` on spawned native and ACP sessions; derive them consistently from the requester chain; expose them through gateway schema/list rows/events and `sessions_list`; document the fields; and add multi-hop regression tests. If maintainers decide root/trace lineage is not a core contract, close only with that explicit product decision.
