## Summary

Current main still has the reported Feishu reply-mention gap: structured mentions are treated as authoritative, missed bot placeholders are not recovered, and reply `root_id` still becomes forced thread delivery. The linked implementation PR #39569 is open and uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Review/rebase and land PR #39569 or an equivalent Feishu plugin fix. Implement orphaned reply placeholder mention detection in `extensions/feishu/src/bot-content.ts`, suppress reply `rootId` only when a normal group reply explicitly mentions the bot, and add regression tests while preserving Feishu topic groups, `thread_id`, configured `replyInThread`, and @all behavior.
