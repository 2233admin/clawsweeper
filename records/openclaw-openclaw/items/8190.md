## Summary

Current main still lacks a global all-session auth refresh/reset. The public protocol and Gateway method registry expose only single-session reset, the handler requires one session key, and the reset service performs a hard rollover with a new session id/transcript rather than an auth-only context-preserving refresh. The existing account/auth-order work remains related but does not supersede this cross-session lifecycle request.

## Best Possible Solution

Keep this issue open as the canonical tracker for global auth refresh across active sessions. The likely fix is an additive Gateway/session capability such as sessions.refreshAuth with an all-sessions scope: enumerate active session stores, invalidate auth-backed runtime/CLI bindings and auth epochs, preserve session ids/transcripts/context where possible, define admin-scope protocol semantics, and add Gateway, channel/session-store, and docs coverage. Treat #7533 and #61709 as related auth-switching work, not substitutes.
