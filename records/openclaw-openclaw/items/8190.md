## Summary

Current main still does not provide a global all-session auth refresh/reset. The implemented Gateway reset remains single-session scoped and performs a hard rollover with a new session id/transcript path, while `/reset soft` only clears CLI auth bindings for the current addressed session. The existing auth-switching/auth-migration related work does not supersede the requested cross-session, context-preserving lifecycle capability.

## Best Possible Solution

Keep this issue open as the canonical tracker for global auth refresh across active sessions. The best path is an additive Gateway/session capability, for example sessions.refreshAuth with an all-sessions scope, that enumerates active session stores, invalidates auth-backed runtime/CLI bindings and auth epochs, preserves session ids/transcripts/context where possible, defines admin-scope protocol semantics, and adds Gateway, session-store, channel/runtime, tests, and docs coverage. Treat #7533 and #61709 as related auth-switching work, not substitutes.
