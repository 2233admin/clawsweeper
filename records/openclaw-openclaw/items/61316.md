## Summary

Keep open. Current main and the latest release still build custom truthy/falsy Sets directly, so the reported mixed-case custom-list miss and malformed-option crash path remain; the PR is a narrow implementation patch that should receive maintainer review rather than be closed or replaced automatically.

## What This Changes

The PR updates `src/utils/boolean.ts` to guard custom truthy/falsy lists with `Array.isArray`, lowercase custom entries, and adds regression tests in `src/utils/utils-misc.test.ts`.

## Best Possible Solution

Land a focused hardening patch for the shared helper with the new regression tests, filter custom-list entries to strings for clarity, and make an explicit maintainer decision to either update the browser plugin copy in the same PR or track it as a separate follow-up.

## Reproduction Assessment

Yes. Static inspection is enough: current main lowercases the input before comparing, but custom lists are not lowercased, and malformed non-iterable `truthy` or `falsy` values reach `new Set(...)`.

## Solution Assessment

Yes, for the shared core helper. The safest landing version should keep the narrow utility fix, preserve the regression tests, and resolve the browser-copy scope question before merge.
