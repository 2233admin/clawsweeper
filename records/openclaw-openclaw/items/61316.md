## Summary

Keep this PR open. Current main still lowercases the input value but builds custom truthy/falsy sets directly from caller-provided values, so mixed-case custom entries still miss and malformed non-iterable option values can still throw. The regression tests proposed by the PR are also absent on main, and the related earlier PR #59535 is closed/unmerged rather than a shipped replacement.

## Best Possible Solution

Review and land a focused patch that normalizes custom truthy/falsy arrays case-insensitively, ignores malformed non-array option values without throwing, filters custom-list entries to strings before lowercasing, adds the proposed unit coverage, and decides whether to update the browser plugin’s copied helper in the same change.

## Risks / Open Questions

- The PR still has an unresolved review note: filtering custom-list entries to strings before lowercasing would be cleaner than keeping unreachable non-string values in the `Set`.

- The browser plugin copy may need the same guard/normalization decision to avoid drift.
