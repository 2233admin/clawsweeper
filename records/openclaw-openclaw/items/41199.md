## Summary

Keep this issue open. Current main at 5de284c2e30f915b769fac08dbeab45f57749702 still contains both reported conflicts: `sessions_send` exposes `sessionKey`, `label`, and `agentId` as independent optional fields and rejects mixed `sessionKey` plus `label`, while the shared `message` schema still exposes poll creation fields and the send runner still rejects `action: "send"` when those fields are present. The related open PRs are implementation candidates for the two halves of the report, so they should be reviewed, landed, or intentionally superseded before closing this issue.

## Best Possible Solution

Keep this issue open while maintainers review and dedupe the focused PRs. The practical path is to land or intentionally close one `sessions_send` fix from #41255 or #59324, then review #46904 or an equivalent message-tool change. Close this issue only after both conflicts are fixed on main, or after any remaining half is split into a narrower canonical issue.
