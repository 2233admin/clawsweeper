## Summary

Keep this issue open. Current main still has both reported parameter-conflict paths: `sessions_send` rejects calls that include both `sessionKey` and `label`, and the shared `message` tool still exposes poll-creation fields while `runMessageAction()` rejects `action: "send"` calls that include those fields. The issue is also paired with open implementation PRs for both halves, including closing syntax on the `sessions_send` side, so the right maintainer path is review/merge or explicitly split the remaining work rather than close now.

## Best Possible Solution

Keep this issue open while maintainers review the linked implementation PRs. The clean path is to choose one canonical `sessions_send` fix from #41255 or #59324, review #46904 or an equivalent message-tool fix for the poll-param conflict, and close this issue only after both halves land or after any remaining half is split into a narrower canonical issue.
