## Summary

Keep this issue open. Current main still has both reported rejection paths: `sessions_send` rejects calls that include both `sessionKey` and `label`, and the shared `message` tool still exposes poll fields while `runMessageAction()` rejects `action: "send"` calls that include poll-creation params. The provided timeline also links open implementation candidates for both halves, including closing syntax against this issue, so this should wait for maintainer review/merge or an explicit split of the remaining work.

## Best Possible Solution

Keep this issue open while maintainers review the linked implementation PRs. The clean path is to choose one canonical `sessions_send` fix from #41255 or #59324, review #46904 or an equivalent message-tool fix for the poll-param conflict, and close this issue only after both halves land or after any remaining half is split into a narrower canonical issue.
