## Summary

Keep this issue open. Current main still contains both reported parameter-conflict behaviors: `sessions_send` rejects mixed `sessionKey` plus `label`, and the shared `message` tool schema still exposes poll creation params while `action: "send"` rejects non-empty poll fields. The provided GitHub context also lists open implementation candidates #41255 and #59324 for the `sessions_send` portion, plus #46904 for the message poll-param portion, so closing before those are reviewed, landed, or superseded would be premature.

## Best Possible Solution

Keep the issue open while maintainers review and dedupe the focused PRs. The practical path is to land or intentionally close one `sessions_send` fix from #41255 or #59324, then review #46904 or an equivalent message-tool change. Close this issue only after both conflicts are fixed, or after the remaining half is split into a narrower canonical issue.
