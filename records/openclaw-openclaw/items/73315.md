## What This Changes

The PR adds a private `apps/desktop` Tauri v2 workspace with a React shell embedding the Gateway Control UI, tray status and restart behavior, root desktop scripts, and Cargo/pnpm workspace and lockfile entries.

## Best Possible Solution

Keep this as the canonical desktop-app candidate if maintainers want the Tauri direction, then require the branch to match OpenClaw's config parser/paths, derive CSP from the actual dashboard origins it constructs, update platform docs/labeler/changelog, align typecheck scripts with repo-approved lanes, and prove Linux plus Windows behavior before merge.

No. The app shape is plausible for a Linux/Windows MVP, but the proposed patch is not yet the narrowest maintainable landing path until it reuses or mirrors the existing config contract, permits the gateway origins it generates, and updates the required public/repo surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Parse OpenClaw config with JSON5:** `apps/desktop/src-tauri/src/main.rs:297`
  - body: `openclaw.json` is documented and loaded by core as JSON5, but this path uses `serde_json::from_str`. A valid config with comments or trailing commas is treated as invalid and the desktop app falls back to loopback defaults, ignoring configured port, auth, and bind settings.
  - confidence: 0.91
- **[P2] Permit configured gateway origins in CSP:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The backend honors `gateway.tls.enabled` and `gateway.bind: "custom"` when building dashboard URLs, but the Tauri CSP only allows loopback HTTP/WS origins. HTTPS loopback or a valid custom host will be generated by Rust and then blocked by the WebView CSP.
  - confidence: 0.88
- **[P2] Update platform docs before landing:** `apps/desktop/README.md:3-4`
  - body: This adds a Linux/Windows desktop companion, but the public platform docs still say those companion apps are planned or unavailable. If merged as-is, the docs site would give stale platform guidance for the new app surface.
  - confidence: 0.84
- **[P2] Add labeler coverage for the desktop app:** `pnpm-workspace.yaml:3`
  - body: The PR registers `apps/desktop` as a new workspace, but `.github/labeler.yml` has app labels only for Android, iOS, macOS, and web UI. Future desktop changes would miss the expected app label routing required for new app surfaces.
  - confidence: 0.8
- **[P3] Align the typecheck script with repo policy:** `apps/desktop/package.json:12`
  - body: The new package adds a `typecheck` script that runs `tsc --noEmit`, but repo policy requires tsgo lanes and explicitly says not to add `typecheck`/`tsc --noEmit` scripts. Please wire this through the repo-approved check path or leave it out until that lane exists.
  - confidence: 0.78
