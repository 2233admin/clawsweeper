## Summary

Keep this PR open for maintainer review. Current main still does not have a Linux/Windows desktop companion, so the branch is not obsolete, but the proposed XL first-party Tauri app is not merge-ready because it still has concrete functional/docs/routing findings and needs maintainer product, ownership, validation, and duplicate-PR consolidation decisions.

## What This Changes

The branch adds a new private `apps/desktop` Tauri v2 workspace with Rust tray, gateway health and restart integration, a React/Tailwind shell embedding the Gateway Control UI, root desktop scripts, pnpm workspace registration, and Cargo/pnpm lockfile updates.

## Best Possible Solution

If maintainers want the Tauri direction, keep one canonical desktop-app PR, fix the CSP/docs/labeler issues, add or document the expected desktop validation path, prove Linux and Windows behavior, and leave installer packaging as a clearly tracked follow-up rather than mixing it into the MVP.

## Review Findings

Full review comments:

- **[P2] Permit configured gateway origins in CSP:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The backend honors `gateway.bind: "custom"` and builds dashboard/health URLs from `gateway.customBindHost`, but the Tauri CSP only permits `127.0.0.1` and `localhost` for `connect-src` and `frame-src`. A configured host such as `192.168.1.8` or `[::1]` will produce a URL that the webview blocks, so the companion cannot embed or poll the configured gateway.

  - confidence: 0.8
