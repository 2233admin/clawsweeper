## Summary

Current main still does not implement the requested `openclaw doctor` warning for provider-augmented or forward-compatible catalog models that are available but missing from a non-empty `agents.defaults.models` allowlist. The only model/allowlist doctor check I found is the hooks-specific validation, while runtime catalog loading still merges provider plugin supplemental rows and allowlist filtering still hides non-allowlisted catalog rows. The related implementation PR #46235 is closed unmerged, so this remains valid work.

## Best Possible Solution

Keep this issue open and implement a read-only doctor health contribution. The best fix is to mark or otherwise expose provider-augmented/forward-compatible catalog rows through a stable internal contract, compare those rows against a non-empty `agents.defaults.models` allowlist, and warn with concrete model refs plus `openclaw models allowlist add <model-id>` guidance. It should skip when no allowlist is configured, never mutate config, avoid provider-id heuristics, and treat API-vs-subscription cost warnings as separate follow-up scope.
