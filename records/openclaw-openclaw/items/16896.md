## Summary

Keep this issue open. Current main still lacks a Dashboard/WebChat reply workflow: there is no right-click/context-menu handler, message-level Reply action, selected quote state in the composer, or `chat.send` reply/quote field. The existing reply pill only renders assistant-supplied directive metadata, and the provided context makes this the canonical tracker after related #29732 was closed back here.

## Best Possible Solution

Implement a focused Dashboard/WebChat reply UX: add an accessible message-level Reply action plus right-click context menu, store the selected quoted message in composer state, focus the input, render a dismissible quote preview, and send either plain quoted Markdown for a UI-only first pass or additive protocol metadata if maintainers explicitly want persisted reply references. Keep related #29732 closed as duplicate context and use this issue as the canonical tracker.
