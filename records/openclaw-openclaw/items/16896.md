## Summary

Keep open. Current main still lacks the requested Dashboard/WebChat right-click reply workflow: there is no context-menu handler, no message-level Reply action, no selected reply target carried into composer state, and the existing reply pill only renders inbound assistant directive metadata. The closed related #29732 points back to this request as the canonical UI tracker, while open PR #62262 is gateway/channel reply-threading work and does not implement the Control UI message-reply UX.

## What This Changes

Add a Dashboard/WebChat message reply affordance, especially a right-click context-menu Reply action that quotes or references a selected message and focuses the composer.

## Best Possible Solution

Implement the feature in the Control UI chat owner area: add accessible message-level Reply controls plus a right-click menu, maintain composer-local selected-message quote state, focus the textarea, show a dismissible quote preview, and prepend a safe Markdown blockquote on send. Leave gateway protocol/storage reply metadata and channel-native threading for a separate explicit design if maintainers want it.
