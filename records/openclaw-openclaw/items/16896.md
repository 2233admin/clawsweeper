## Summary

Keep open. Current main still does not provide a Dashboard/WebChat user reply workflow: the chat thread has no context-menu handler, message actions remain copy/expand only, composer state has no selected reply target, and existing reply UI is only assistant directive display metadata. The closed related #29732 points back here as the canonical UI tracker, while open #62262 is broader gateway/channel threading work and does not implement this Control UI feature.

## What This Changes

Add a right-click context menu in Dashboard webchat that lets users reply to a selected message by quoting or referencing it in the composer and focusing the input.

## Best Possible Solution

Implement the first pass in the Control UI chat owner area: accessible message-level Reply controls plus right-click menu, selected-message quote state in the composer, textarea focus, a dismissible quote preview, and safe Markdown quote prepending on send. Leave native channel threading, storage migrations, and gateway protocol reply metadata for a separate maintainer-approved design.
