## Summary

Keep open. I rechecked current main after the prior ClawSweeper comment: the available-skills include-style use case is already covered by existing skill allowlists, but the requested prompt-section gates and the systemPromptOverride cookbook are still absent from source, schema, tests, and docs.

## What This Changes

Request adds default-preserving configuration to omit selected full-prompt sections, filter available_skills, and document safe systemPromptOverride composition.

## Best Possible Solution

Add narrow, default-preserving prompt gates for the remaining full-prompt sections, reuse the existing skill allowlist for `<available_skills>` filtering, and document which prompt blocks are structural versus optional for operators who use full overrides.

## Reproduction Assessment

Not applicable as a runtime bug. Static verification is high confidence: current main has the prompt sections, schemas, and docs in place, and the requested gates/cookbook are not present.

## Solution Assessment

Partly yes. Per-section gates are a maintainable way to reduce prompt cost, but the skills portion should be solved by documenting and preserving `agents.defaults.skills` / `agents.list[].skills` rather than adding an overlapping include/exclude schema by default.
