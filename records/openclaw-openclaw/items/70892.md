## Summary

Current main only partially overlaps with the request. The existing `agents.defaults.skills` and `agents.list[].skills` allowlists can narrow `<available_skills>`, but current main still lacks the requested per-section prompt gates for the CLI quick reference, reply-tag/directive guidance, and Messaging/component guidance, and it lacks the proposed `agents.defaults.prompt.availableSkills` include/exclude surface. Docs describe `systemPromptOverride`, the existing heartbeat gate, and skill allowlists, but not a safe override cookbook for structural versus droppable prompt blocks.

## Best Possible Solution

Keep this open for maintainer API/product judgment. The best path is to decide whether the supported boundary should be existing `agents.defaults.skills`/`agents.list[].skills` plus a documented `systemPromptOverride` cookbook, or a new narrow `agents.defaults.prompt.*` config surface with default-true gates. If new gates are accepted, update source, schema/help/generated docs, config docs, system-prompt docs, and focused prompt-rendering tests together.
