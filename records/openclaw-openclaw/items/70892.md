## Summary

Keep open. Current main already covers the skill allowlist portion through `agents.defaults.skills` and per-agent `skills`, but the requested per-section prompt gates and the `systemPromptOverride` cookbook are still missing from the config schema, renderer, tests, and docs.

## What This Changes

The issue requests optional prompt-section gates for CLI quick reference, reply tags, messaging component guidance, an available-skills filter, and docs for safe `systemPromptOverride` composition.

## Best Possible Solution

Add a narrow, default-preserving prompt configuration surface for the remaining full-prompt sections, wire it through both embedded and CLI prompt builders, document the existing skills allowlist as the preferred `<available_skills>` include filter, and add a concise systemPromptOverride table that separates required structure from optional prompt blocks.

## Reproduction Assessment

Not applicable as a runtime bug. Static current-main inspection gives a high-confidence verification path: the requested prompt-section fields are absent from the schemas, and the renderer still emits the CLI, reply-tag/directive, and messaging guidance without operator gates.

## Solution Assessment

Partly yes. The per-section gates are the narrowest maintainable direction for the remaining prompt-cost problem, but the available-skills portion should probably reuse the existing `agents.defaults.skills` / `agents.list[].skills` allowlist instead of adding a second overlapping include/exclude schema unless maintainers explicitly want subtractive filters.
