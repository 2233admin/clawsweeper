## Summary

Keep this issue open. Current main still lacks the requested global, per-surface, per-channel, or Signal-specific policy to silently swallow genuine empty-turn/incomplete-turn fallback payloads, and the maintainer discussion explicitly says the feature remains valid after the related false-positive fix.

## Best Possible Solution

Keep this issue open and implement a first-class incomplete-turn delivery policy in the embedded-runner/reply-run configuration path. It should default to current visible error behavior, support global plus per-surface or per-channel silent overrides for direct/TTS surfaces such as Signal, preserve WARN/server logging, and add regression coverage for direct chats, groups, genuine empty turns, reasoning-only turns, message-tool side effects, TTS-enabled delivery, and existing NO_REPLY semantics.
