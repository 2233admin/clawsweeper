## Summary

Keep this issue open. Current main still has only targeted suppressions for known false-positive empty turns; it does not expose a global, surface, channel, or Signal-specific policy that can silently swallow genuine incomplete-turn fallback payloads before final delivery or TTS. The maintainer discussion also explicitly says the #70396/#70623 fixes did not add the requested config knob.

## What This Changes

Add a configurable policy, globally or per channel/surface, to silently log and suppress empty-turn fallback responses instead of delivering the visible `Agent couldn't generate a response` error text.

## Best Possible Solution

Implement a first-class incomplete-turn delivery policy that defaults to current visible error behavior, supports a global default plus surface/channel override for direct voice/TTS contexts such as Signal, logs suppressed failures at WARN level with the same diagnostic context, and adds docs plus regression coverage for direct chats, groups, genuine empty turns, reasoning-only turns, message-tool side effects, TTS delivery, and existing `NO_REPLY` behavior.
