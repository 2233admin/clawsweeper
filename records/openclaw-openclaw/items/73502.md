## Summary

Current main still matches the report: bundled Active Memory hard-codes the legacy `memory_search` / `memory_get` recall surface, while bundled Memory LanceDB registers `memory_recall` / `memory_store` / `memory_forget`. The memory slot docs make `memory-lancedb` a supported memory backend selection, so this remains an actionable bundled-plugin integration bug rather than a stale or third-party issue.

## Best Possible Solution

Keep open. The best fix is in the bundled memory plugin boundary: make Active Memory choose the selected memory backend's recall contract through an existing or new typed memory capability, or make `memory-lancedb` expose the common `memory_search`/`memory_get` contract if that is the intended memory-slot API. Add a regression test for Active Memory with `plugins.slots.memory = "memory-lancedb"`, and align the Active Memory and Memory LanceDB docs so they describe one supported integration path.
