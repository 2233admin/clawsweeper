## Summary

Keep open. Current main still does not strip general Markdown before legacy iMessage delivery; it only converts Markdown tables, removes internal directive tags/scaffolding, and applies HTML-focused plain-text sanitization. The broader shared fix is still represented by open PR #56283 rather than merged current-main code.

## What This Changes

Request to strip or convert Markdown formatting to plain text before sending replies through the legacy iMessage channel.

## Best Possible Solution

Keep this issue open until #56283 or an equivalent current-main implementation lands. The best fix is a shared outbound Markdown stripping stage for non-Markdown-capable/plain-text delivery, applied after existing HTML/plain-text sanitization and internal directive cleanup, with focused tests for direct iMessage sends, monitor replies, media captions, config override behavior, Markdown-capable channel preservation, and empty-after-strip edge cases.
