## Summary

Keep open. Current checked-out main b85edb3f still does not strip general Markdown before legacy iMessage delivery; it converts Markdown tables and strips inline directive tags, but ordinary bold, headings, blockquotes, and backticks can still reach imsg. The related open PR #56283 remains the active implementation path rather than a merged fix.

## Best Possible Solution

Keep this issue open until #56283 or an equivalent main-branch implementation lands. The best fix is a shared outbound Markdown-stripping decision for non-Markdown-capable/plain-text delivery, applied after existing HTML/table/internal-directive handling, with focused coverage for direct iMessage sends, monitor replies, media captions, optional config override behavior, and empty-after-strip edge cases.
