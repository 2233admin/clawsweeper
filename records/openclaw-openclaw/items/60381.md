## Summary

Current main implements the evaluate portion of this browser-tool request, but the requested click force parameter is still missing from the public schema, normalizer, CLI, and Playwright click execution path. This issue should stay open for the remaining force-click work.

## Best Possible Solution

Keep this issue open and narrow the remaining work to force-click support: add force?: boolean to the browser act schema and BrowserActRequest click type, preserve it in normalizeActRequest and legacy flattened params, thread it through executeSingleAction and clickViaPlaywright, pass it to locator.click and locator.dblclick where supported, decide/document existing-session behavior, then add focused contract tests plus CLI/docs updates. Treat the evaluate half as already implemented, with #55598 as related context.
