## Summary

Current main still has the reported formatter behavior: punctuation-wrapped Home Assistant-style entity IDs can miss copy-sensitive classification and then get split with literal spaces. This issue should stay open while PR #39565 or an equivalent focused fix is reviewed and landed.

## Best Possible Solution

Keep this issue open and review/land PR #39565 or an equivalent focused fix. The fix should classify punctuation-stripped dotted Home Assistant entity IDs as copy-sensitive, preserve bare and punctuation-wrapped forms, add regression coverage in `src/tui/tui-formatters.test.ts`, and keep existing generic long-token chunking intact.
