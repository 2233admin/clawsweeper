## Summary

Current main still reproduces the reported formatter behavior. The TUI copy-sensitive classifier strips edge punctuation into a candidate, but the file-like branch still tests the raw token, so backtick- or quote-wrapped dotted entity IDs miss preservation and are chunked with literal spaces. PR #39565 remains an open, unmerged implementation candidate for this issue, so the issue should stay open while that PR or an equivalent fix is reviewed.

## Best Possible Solution

Keep this issue open and review/land PR #39565 or an equivalent focused fix. The fix should classify punctuation-stripped dotted Home Assistant-style entity IDs as copy-sensitive, preserve bare and punctuation-wrapped forms, add regression coverage in `src/tui/tui-formatters.test.ts` and any adjacent note-wrapper tests if touched, and keep existing generic long-token chunking intact.
