## Summary

Keep open. This is an open contributor PR paired with open #48123 by the same author, and current main only partially overlaps it: non-image `chat.send` attachments are persisted as `media://inbound` refs and transcript media, but ordinary dispatch still does not set the `ctx.MediaPaths`/`MediaTypes` workspace-staging contract that this PR proposes.

## Best Possible Solution

Keep this PR open as the implementation candidate paired with #48123. The best maintainer path is to rebase or split it against current main, preserve the shipped WebChat file-upload behavior, then either implement and test ordinary `chat.send` `ctx.MediaPaths`/`MediaTypes` workspace staging or explicitly decide that the shipped media-ref/transcript/tool path supersedes that design and close the PR plus paired issue together.
