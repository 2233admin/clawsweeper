## Summary

Keep this issue open. Current main still does not expose a supported `tools.exec.shell` contract: shell selection is resolved internally by the exec runtime, the public exec config type and strict schema omit `shell`, and the exec docs document fixed platform defaults rather than a configurable override. The related implementation PR remains open, so this is not stale cleanup.

## What This Changes

Requests a supported `tools.exec.shell` config key, including global and per-agent use, to override exec shell selection for Windows/WSL and other preferred shell environments.

## Best Possible Solution

Define a supported exec shell override contract with global and per-agent precedence, executable-plus-argument semantics, validation behavior, Windows/WSL cwd and PATH expectations, approval and safe-bin interactions, schema/help/docs updates, and targeted exec tests, then land it through PR #39413 or a maintainer-directed replacement.
