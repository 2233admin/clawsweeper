## Summary

Current main still does not implement a configurable `tools.exec.shell` override. The exec runtime still chooses shells through `getShellConfig()` with a Windows PowerShell default and POSIX `$SHELL`/`sh` fallback, while the `tools.exec` type/schema/docs expose host/security/ask/node/PATH/safe-bin settings but no shell field. A related implementation PR, #39413, is open and should be reviewed or replaced before this issue closes.

## Best Possible Solution

Keep this issue open and review/rework #39413 or an equivalent focused PR. The fix should add a global and per-agent `tools.exec.shell` contract only after defining shell executable/args semantics, validation, Windows/WSL cwd/PATH behavior, and approval/safe-bin interactions, then update the config type, zod schema, schema help, docs, and targeted exec tests.
