## Summary

Keep this PR open. Current main still does not implement the intended `sessions.list.defaults` model behavior: gateway session defaults are resolved from the global configured fallback rather than the effective default or filtered agent. The PR remains a useful candidate, but it needs rebase/narrowing and should resolve model, contextTokens, and thinking defaults from the same selected agent source; its latest head also bundles an unrelated hook-loader path change.

## Best Possible Solution

Keep the PR open for maintainer follow-up. Rebase it on current main, narrow it to the gateway session-defaults bug, make `getSessionDefaults` accept the normalized selected/default agent id, derive model, contextTokens, and thinking defaults from the same effective agent source, pass the filtered `agentId` through from `listSessionsFromStore`, and keep focused regressions for both default-agent and filtered-agent defaults. Split the hook path canonicalization into a separate PR if maintainers want that behavior.
