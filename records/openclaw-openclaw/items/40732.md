## Summary

Keep open. Current main has not implemented the CodeMirror raw JSON5 editor, so the PR is not obsolete, but the branch is not merge-ready because it must be rebased onto current raw config redaction behavior and still has a JSON5 breadcrumb parser edge-case bug.

## What This Changes

This PR replaces the raw config textarea with a CodeMirror 6 Lit editor, adds line/column and JSON5 path breadcrumbs, updates styling and locale strings, adds CodeMirror dependencies, and adds browser tests.

## Best Possible Solution

Land one canonical raw config editor implementation that keeps current redaction and raw round-trip semantics, fixes the JSON5 path parser edge case, regenerates locale output through the current UI i18n pipeline, and resolves the overlap with #40009 and #56401.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve raw secret redaction around the editor:** `ui/src/ui/views/config.ts:800-803`
  - body: Current main hides raw config text when sensitive values exist until the reveal toggle is pressed, and its browser test asserts no editor/textarea is mounted before reveal. This hunk mounts `config-raw-editor` with `props.raw` directly in raw mode, so carrying the patch forward would put token values into the visible UI/DOM before reveal; keep the current reveal wrapper around the new editor.
  - confidence: 0.88
- **[P2] Handle escaped JSON5 identifier keys:** `ui/src/ui/components/config-raw-editor-path.ts:39-40`
  - body: JSON5 member names may be ECMAScript `IdentifierName`s, and those allow Unicode escape starts such as `\u0066oo`. Since `isValueTokenStart` rejects `\`, the resolver skips or mislabels that valid key, so breadcrumb text and jump targets are wrong for valid configs.
  - confidence: 0.84
