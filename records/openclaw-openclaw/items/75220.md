## Summary

Keep open. Current main only fixes fresh minimal PATH generation; the forced reinstall path still merges PATH entries from the existing service environment, and the preservation filter does not reject the same version-manager/package-manager paths that the service audit reports as non-minimal.

## What This Changes

The issue asks for `openclaw gateway install --force` and doctor/status service PATH audits to converge so the recommended repair path does not recreate non-minimal PATH warnings.

## Best Possible Solution

Make service reinstall planning converge on the same PATH contract as the audit: preserve safe existing PATH additions only when they will not be reported as non-minimal, and add a regression test where a forced reinstall starts from an old unit PATH containing stale version-manager/package-manager entries.

## Solution Assessment

No. Current main is only a partial solution: it filters missing hard-coded fallback directories during fresh PATH generation, but the forced reinstall path still preserves stale existing PATH segments that the audit rejects.
