## Summary

Keep open. Current main still has the reported Control UI route: heartbeat-isolated session rows can be listed/selected and WebChat sends to the active session key directly. The issue is also paired with open closing PR #75356, so it should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that Control UI WebChat avoid displaying or routing user input to heartbeat-isolated `:heartbeat` session rows when the base `main:main` session has been pruned.

## Best Possible Solution

Land or replace the narrow marker-backed fix: project `heartbeatIsolatedBaseSessionKey` or an equivalent synthetic-session flag through the gateway session-list contract, hide or normalize heartbeat-isolated rows in the Control UI picker, and guard `chat.send` so stale heartbeat selections route to the base session while preserving real user sessions that happen to end in `:heartbeat`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available by listing only `agent:main:main:heartbeat` after the base row is pruned: current main adds that row to the picker and sends WebChat input with `sessionKey: state.sessionKey`.

## Solution Assessment

Yes, the linked PR direction is the right narrow solution if review confirms the details: use the stored heartbeat-isolated marker rather than suffix-only matching, normalize stale WebChat sends back to the base session, and add focused UI/gateway regression tests.
