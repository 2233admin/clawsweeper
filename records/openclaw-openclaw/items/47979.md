## Summary

Keep open. The Chrome 146 Control UI freeze remains actionable against current main: the detailed report and maintainer discussion point to the initial Dashboard v2 render path, and the suspected ResizeObserver, sticky/backdrop topbar styling, and eager post-hello loading paths are still present. The known shipped #45541 fix covers a later chat-history reload storm, not this reported post-connect/pre-follow-up-RPC lockup.

## Best Possible Solution

Keep this issue open for a targeted Control UI investigation. Reproduce or bisect in affected Chromium builds by disabling observeTopbar, removing sticky/backdrop topbar compositor effects, and staging the post-hello shell/render work. If confirmed, add a small Chromium-safe fallback or narrow layout fix plus a browser smoke that proves the dashboard remains interactive after gateway hello and sends the expected sessions/chat RPCs.
