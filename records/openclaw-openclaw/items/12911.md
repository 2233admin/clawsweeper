## Summary

Keep this issue open. Current main has browser Talk mode in `/chat` and it is now provider-shaped for OpenAI, Google, and backend relay transports, but the xAI/Grok realtime voice provider requested here is still not registered or exposed. The related open xAI work targets `/voiceclaw/realtime` and explicitly does not close the `/chat` provider-plugin surface.

## Best Possible Solution

Keep this issue open as the tracker for Grok/xAI Live Voice Mode in `/chat`. The concrete path is to add an xAI realtime voice provider on the provider-plugin surface, likely `extensions/xai/realtime-voice-provider.ts` registered with `api.registerRealtimeVoiceProvider(...)`, extend the browser-session or Gateway relay contract if xAI needs a different transport, then update Control UI/provider docs and focused Talk/xAI tests. Treat #73019/#73032 as adjacent `/voiceclaw/realtime` work, not a replacement for this issue.
