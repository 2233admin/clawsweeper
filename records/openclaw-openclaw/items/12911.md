## Summary

Keep this issue open. Current main has browser Talk mode in the Control UI chat path, but the remaining requested capability is specifically Grok/xAI realtime voice in `/chat`; the xAI plugin still does not register a realtime voice provider, and the related open xAI work targets `/voiceclaw/realtime`, not the Talk provider-plugin surface.

## Best Possible Solution

Keep this issue open as the tracker for Grok/xAI Live Voice Mode in `/chat`. The likely path is an xAI `RealtimeVoiceProviderPlugin` under `extensions/xai/`, registered with `api.registerRealtimeVoiceProvider(...)`, with manifest/docs/config updates and focused Talk-provider tests. If xAI's realtime voice transport does not fit the existing browser-session or relay shapes, make the smallest additive provider contract extension first and keep #73019/#73032 scoped to the separate `/voiceclaw/realtime` surface.
