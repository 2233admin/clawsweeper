## Summary

Keep open. Current main still does not expose the incoming user message, prompt, or query on the internal `agent:bootstrap` context. The adjacent plugin `before_prompt_build` hook does receive `prompt` and `messages`, and Active Memory uses that for query-conditioned recall, but that runs later as prompt/system-context injection and does not let a bootstrap hook rewrite the existing `bootstrapFiles` array before it is injected.

## Best Possible Solution

Add a backward-compatible optional field such as `userMessage?: string` or `prompt?: string` to `AgentBootstrapHookContext`, thread it from embedded and CLI prompt assembly paths where a user-triggered prompt exists, document when it is undefined for startup/heartbeat/non-user runs, and add focused tests for `applyBootstrapHookOverrides` plus the run path that resolves bootstrap context.
