## Summary

Keep this issue open. Current main still sends MCP stdio JSON-RPC frames unsigned, exposes no supported `mcps`/message-signing config, and does not document a plugin transport-wrapper seam. The related PR #49182 was closed unmerged after maintainer security concerns, and the reporter’s follow-up specifically asked whether a new transport-level plugin API is needed.

## Best Possible Solution

Keep this issue open for maintainer security/API design. The best path is either a generic, opt-in MCP transport signing/decorator seam with documented config and tests for replay, forgery, stripping, downgrade, key handling, and strict-mode failure behavior, or an explicit maintainer decision that MCPS remains external plus a documented plugin API that can safely wrap OpenClaw-managed stdio frames.
