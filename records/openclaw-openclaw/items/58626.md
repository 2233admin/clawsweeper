## Summary

Keep open. Current main has a useful bundled-plugin alias cleanup path, but it only recognizes aliases under the current OpenClaw package root; the reported pnpm/npm upgrade failure uses an older versioned package root that has been removed, so `plugins.load.paths` can still surface as a fatal missing-path config error.

## What This Changes

The issue asks OpenClaw to avoid or repair stale absolute bundled plugin paths left in config after pnpm/npm global package upgrades.

## Best Possible Solution

Extend the bundled plugin alias repair path to recognize versioned packaged OpenClaw roots from prior installs, ignore those stale bundled aliases during discovery so Gateway startup is not blocked, and have `doctor --fix` remove the obsolete load-path and matching install-record metadata.

## Reproduction Assessment

Yes. Seed config with a `plugins.load.paths` entry pointing at a missing older `.../node_modules/openclaw/dist/extensions/acpx` package path; current source shows it will miss the current-root alias check and become a `plugin path not found` validation issue.

## Solution Assessment

No, current main is not the complete fix. The maintainable path is to extend the existing bundled alias/doctor repair seam, not to add a broad startup migration or suppress all missing plugin paths.

## Repair Work Prompt

Fix the upgrade failure where `plugins.load.paths` and matching bundled plugin install metadata can point at an older removed versioned OpenClaw package directory after pnpm/npm global upgrades. Extend the existing bundled plugin load-path alias/doctor repair seam, rather than adding a broad startup config migration. The fix should only classify paths that structurally point at an OpenClaw packaged bundled plugin root, such as `.../node_modules/openclaw/dist/extensions/<pluginId>` or `dist-runtime/extensions/<pluginId>` from a prior package version, and whose plugin leaf maps to a currently bundled plugin. Discovery should warn/ignore those stale bundled aliases so Gateway startup is not blocked. `openclaw doctor --fix` should remove the stale `plugins.load.paths` entry and clean matching stale bundled `plugins.installs` or installed-index install records when safe. Do not suppress arbitrary missing external plugin paths. Add focused tests for an old versioned pnpm-style package root distinct from the current package root, plus any npm-style global root variant that fits the same matcher. Update docs/changelog if behavior changes; user-facing changelog entry should credit `@solink7` and, if mentioning the npm-global confirmation, `@SonicBotMan`.
