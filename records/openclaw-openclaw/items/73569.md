## What This Changes

The issue requests clearer `infer audio transcribe` and `infer image describe` errors when no audio or image-understanding provider is configured.

## Best Possible Solution

Merge one canonical fix that carries missing-provider/no-attempt state through the media-understanding result seam and converts it to actionable CLI configuration errors for image describe, image describe-many, and audio transcribe, while preserving the existing empty-result errors when a provider actually ran.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Source inspection on current `main` shows the deterministic path from no resolved media provider to skipped/no outputs, then `text: undefined`, then the generic CLI path-blaming error; no live CLI run was performed because this review was read-only.

## Solution Assessment

Is this the best way to solve the issue? Yes. The narrow maintainable fix is to distinguish missing-provider/no-attempt results from provider-ran-but-empty-output at the runtime or CLI boundary, then add focused regression tests for both branches.
