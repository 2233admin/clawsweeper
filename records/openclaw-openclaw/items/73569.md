## Summary

Keep open. Current main still matches the report: `infer image describe` and `infer audio transcribe` convert an empty media-understanding result into the generic path-blaming "No description/transcript returned" errors, while the media runner treats the no-resolved-provider case as a skipped decision with no outputs. The closed related reports in the issue body addressed provider-result/parsing failures, but this no-provider-config error branch remains unhandled.

## Best Possible Solution

Keep this issue open and fix the CLI/media-understanding boundary. The implementation should distinguish `no configured/resolved media provider` from `provider ran but returned empty text`, then emit actionable errors for `infer image describe`/`describe-many` and `infer audio transcribe`. For image, point at `agents.defaults.imageModel.primary` and/or `tools.media.image.models`; for audio, point at `tools.media.audio.models` or shared `tools.media.models` plus provider auth. Add focused CLI tests that mock the no-config branch and preserve the existing empty-provider-result errors for real empty responses.
