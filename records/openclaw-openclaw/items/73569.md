## Summary

Keep open. Current main still matches the report: `infer image describe` and `infer audio transcribe` still throw the generic path-blaming errors whenever the media-understanding result has no text, while the no-provider path in the runtime returns a skipped decision with no outputs. Two open PRs, #73576 and #73593, explicitly use closing syntax for this issue, so this issue should remain open until a fix PR is reviewed and merged or the candidates are closed.

## Best Possible Solution

Keep this issue open and review/land one canonical fix PR, likely #73593 or #73576. The fix should teach the CLI/media-understanding boundary to surface a separate actionable missing-provider configuration error for image description and audio transcription, while preserving the existing generic empty-result errors for the case where a configured provider ran and returned no text. Add focused CLI regression tests for both branches.
