## Summary

Keep this PR open. Current `main` still has the noisy `Gateway last log line` fallback and only a narrower gateway log summarizer, so the PR is not obsolete; however, the open review comments identify real behavior risks that should be resolved before merge.

## What This Changes

The PR filters routine gateway log-tail noise from `status --all`, groups repeated diagnostics into summaries, adds focused tests, and changes gateway error lookup to return only curated errors instead of arbitrary fallback log lines.

## Best Possible Solution

Land a revised version that keeps `status --all` pasteable and quieter while preserving curated auth/gateway failures, uses accurate labels for grouped diagnostics, and covers the new summarizer and daemon-diagnostics behavior with focused tests.
