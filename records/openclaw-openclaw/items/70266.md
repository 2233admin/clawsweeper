## Summary

Keep this issue open. Current main still renders the macOS Talk Mode overlay as the generic `TalkOrbView` Circle/RadialGradient surface and the Talk config/state path only carries voice settings plus `ui.seamColor`, while the existing gateway identity API already exposes resolved assistant avatar metadata. The reporter's follow-up correctly distinguishes this from the closed Control UI identity work in #22199.

## Best Possible Solution

Keep this issue open and implement a narrow macOS app change: resolve the active assistant identity/avatar for Talk Mode using the existing `agent.identity.get` and avatar-route contracts, render a circular cropped avatar inside the Talk overlay when available, preserve the existing rings/pulse/thinking arcs and click/drag/pause behavior, and fall back to the current orb whenever identity or image loading fails. The closed #22199 Control UI work should remain separate because it does not update this macOS Talk Mode surface.
