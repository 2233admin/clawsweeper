## Summary

Keep this PR open. Current main still does not preserve an explicit user-configured GitHub Copilot baseUrl during runtime auth, but this branch is stale against the current provider/runtime-auth architecture and should not merge as-is.

## What This Changes

The PR branch modifies `src/agents/pi-embedded-runner/run.ts` to conditionally copy a Copilot token-derived baseUrl into the runtime and effective models only when no custom baseUrl, or the default baseUrl, is present.

## Best Possible Solution

Keep this PR open as useful contributor context, but port the behavior into the current GitHub Copilot provider/runtime-auth flow. The fix should preserve explicit custom Copilot baseUrl values, still apply token-derived Business/Enterprise endpoints when the endpoint is absent/default or known token-derived, keep runtimeModel and effectiveModel synchronized, preserve lazy runtime imports, and add regression coverage for initial auth, refresh, context-window-capped effectiveModel, and auth profile rotation.
