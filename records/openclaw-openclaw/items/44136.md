## Summary

Keep open. Current main still does not implement the requested Copilot baseUrl precedence: the GitHub Copilot runtime-auth hook returns the token-derived endpoint and the embedded auth controller applies any prepared baseUrl into both runtime and effective models without checking whether the user configured a custom endpoint. The PR remains useful context, but it is stale against the current provider-runtime/auth-controller flow and should not be closed as implemented.

## What This Changes

The PR branch adds conditional Copilot token-derived baseUrl assignment in the embedded runner's initial-auth and token-refresh paths so explicit custom baseUrl values are preserved while runtime and effective models stay synchronized.

## Best Possible Solution

Port the behavior into the current GitHub Copilot provider/runtime-auth path: preserve explicit custom Copilot baseUrl values, still apply token-derived Business/Enterprise endpoints when the endpoint is absent/default or known token-derived, keep runtimeModel and effectiveModel synchronized, preserve lazy imports, and add focused regression coverage for initial auth, refresh, context-window capping, and auth profile rotation.
