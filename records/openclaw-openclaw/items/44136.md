## Summary

Keep open. Current main still applies GitHub Copilot token-derived baseUrl values unconditionally during runtime auth, so it does not implement this PR's central priority rule: preserving an explicit user-configured Copilot baseUrl such as a corporate proxy or Business endpoint. The submitted patch also targets the older runner shape and should not be merged as-is without porting to the current bundled plugin/auth-controller path.

## Best Possible Solution

Keep this PR open as useful contributor context, but port the behavior rather than merging the patch as-is. The best fix belongs in the current GitHub Copilot provider/runtime-auth flow: preserve explicit custom Copilot baseUrl values, still apply token-derived Business/Enterprise endpoints when the configured value is absent/default/token-derived, keep runtimeModel and effectiveModel synchronized, avoid eager Copilot runtime imports, and add regression coverage for custom proxy, default-to-token endpoint, context-window-capped effectiveModel, refresh, and profile rotation.
