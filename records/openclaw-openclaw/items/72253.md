## Summary

Keep this PR open. Current main still has the reported heartbeat exec-completion leakage path: structured `code 0` and `signal SIGTERM` events are recognized as exec completions but are not classified as internal-only, and exec completions still bypass the normal OK-skip path. The provided PR remains a focused implementation candidate; the remaining maintainer question is whether this tactical classifier should land as-is or be folded into the broader #69492 event-audience work.

## What This Changes

The PR tags local exec completion events with provenance metadata, filters successful and SIGTERM structured completions out of user-relay heartbeat prompts, preserves real failure relay, and adds heartbeat/bash regression coverage.

## Best Possible Solution

Review the PR as a narrow heartbeat fix: either land the trusted local-exec internal-only classifier with the proposed regression tests, or fold the same behavior into #69492 while preserving visible relay for actionable failures such as `code 1`.
