## Summary

Keep this issue open. Current `main` and the latest release are healthy for `v2026.4.26`, but the reported release/appcast invariant is still not enforced: the public macOS release workflow remains validation-only, `scripts/make_appcast.sh` can publish a derived GitHub release URL without checking release asset reachability, and `scripts/release-check.ts` only validates Sparkle version/build shape. The directly related verifier PR #66213 is still open and unmerged.

## Best Possible Solution

Keep this issue open as the canonical macOS updater/release consistency bug until release-owner work lands. The best path is to review and either land PR #66213 or replace it with a stronger guard that verifies stable releases have the expected `.zip`, `.dmg`, and `.dSYM.zip` assets, verifies the appcast enclosure URL before `appcast.xml` advances, protects appcast ownership, and optionally improves the macOS updater failure/fallback behavior for missing artifacts.
