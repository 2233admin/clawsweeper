## Summary

Keep open. Current main still has MCP config editing/counting and session-time MCP tool materialization, but no Control UI connectivity panel, no Gateway MCP health/refresh method, and no persisted per-server error state for UI diagnostics. The earlier ClawSweeper comment remains accurate against the newer main SHA.

## What This Changes

Request to add a Control UI MCP connectivity panel showing per-server status, exposed tools, reconnect/refresh, and surfaced error details.

## Best Possible Solution

Add an additive Gateway operator diagnostics API, backed by the MCP config/runtime loader, that returns each configured server name, transport summary, connected/error state, tool names/count, and redacted last error, plus a scoped refresh action. Then render it under Control UI Infrastructure > MCP and document the operator behavior.
