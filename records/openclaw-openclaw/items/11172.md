## Summary

Keep open. Current main still does not inject `OPENCLAW_AGENT_ID`, `CLAWDBOT_AGENT_ID`, `OPENCLAW_SESSION_KEY`, or related session identity variables into exec child-process environments. Subagent spawning already computes the target agent id and child session key, but exec environment construction only merges inherited/request/sandbox env and runtime markers. The related implementation PR #11497 is closed unmerged, #11477 was closed as a duplicate of this issue, and open PR #53262 concerns required `sessions_spawn.agentId` schema behavior rather than exposing identity to spawned commands.

## Best Possible Solution

Keep this issue open until OpenClaw either ships or explicitly rejects a documented core exec identity contract. If accepted, implement it in the shared exec environment path after agent/session context resolution, define advisory versus protected semantics and override precedence, decide whether to include `CLAWDBOT_AGENT_ID`, and add focused tests/docs for gateway, sandbox, node, PTY, and subagent-spawned sessions.
