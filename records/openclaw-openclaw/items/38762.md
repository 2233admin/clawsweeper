## Summary

Keep this open. Current main still ignores top-level `ref`, `targetId`, and `timeoutMs` whenever `browser action="act"` receives an object `request`, so the reported mixed call shape remains unsupported. The service-reachability error wrapping has been improved for application-level browser errors, but the underlying parameter-position inconsistency remains, and the related compatibility PR #38790 is closed unmerged with no open replacement found.

## What This Changes

The issue asks for `browser action="act"` to either accept top-level `ref` with a nested `request` object or return a clearer request-shape error instead of misleading browser-service guidance.

## Best Possible Solution

Land a narrow browser-tool compatibility normalizer in `extensions/browser/src/browser-tool.ts`: for `action: "act"` with an object `request`, clone the request and fill only missing `request.ref`, `request.targetId`, and finite numeric `request.timeoutMs` from top-level params. Preserve nested request precedence, keep fully flattened legacy behavior unchanged, add focused tests, and include a user-facing changelog fix entry crediting the reporter and prior PR contributor.
