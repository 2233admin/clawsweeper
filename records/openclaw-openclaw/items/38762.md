## Summary

Keep open. Current main still returns an object `request` for `browser action="act"` before merging any top-level `ref`, `targetId`, or `timeoutMs`, so the mixed shape reported here remains unsupported. The related compatibility PR #38790 is closed unmerged, and the current tests cover fully flattened params and nested-request precedence but not filling missing nested fields from top-level params.

## Best Possible Solution

Land a narrow browser-tool compatibility normalizer in `extensions/browser/src/browser-tool.ts`: when `action: "act"` receives an object `request`, clone it and fill only missing `request.ref`, `request.targetId`, and finite numeric `request.timeoutMs` from top-level params. Preserve nested request precedence, keep fully flattened legacy behavior unchanged, add focused browser-tool tests for mixed-shape compatibility and precedence, and add a user-facing changelog fix entry crediting the reporter/contributor.
