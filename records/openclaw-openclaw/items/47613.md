## Summary

Keep open. Current main still has the callback-gated upstream reasoning path, so this PR is not obsolete; the remaining decision is maintainer selection among overlapping implementations and confirmation that Gateway/WebSocket clients should receive live reasoning content.

## What This Changes

The PR changes `src/agents/pi-embedded-subscribe.ts` so `reasoningMode: "stream"` can emit `stream: "thinking"` agent events without requiring `onReasoningStream`, while still calling that callback when present.

## Best Possible Solution

Land one canonical Gateway reasoning-stream implementation that decouples `emitAgentEvent` from channel-specific callbacks, preserves explicit stream-mode, thinking-enabled, and silent-turn gates, adds no-callback WebSocket regression coverage, and updates `docs/tools/thinking.md` if Gateway streaming becomes supported.
