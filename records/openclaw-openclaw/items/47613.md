## Summary

Keep this PR open. Current main still has the exact no-callback reasoning-stream gap: `streamReasoning` and `emitReasoningStream()` both require `onReasoningStream`, while Gateway `chat.send` does not provide that callback. The PR is relevant, but should be rebased onto current main and covered with a no-callback regression test before landing.

## Best Possible Solution

Keep this PR available for review or use it as the focused basis for the fix. The best implementation is to enable `streamReasoning` for `reasoningMode === "stream"` while preserving `canShowReasoning`, keep the `silentExpected` suppression, emit `stream: "thinking"` independently of channel callbacks, call `onReasoningStream` only when provided, and add a regression test proving gateway/WebSocket clients receive thinking events without a channel callback.
