## Summary

Keep open. Current main still does not route automatic daily/idle rollovers through the reset persistence hook path used by manual `/new`, `/reset`, and gateway `sessions.reset`; only the secondary hook-config schema mismatch is fixed. Open PR #71792 explicitly closes this issue and #61675 overlaps, so the issue should remain open until maintainers land, replace, or reject those implementation candidates.

## What This Changes

The issue requests a `session:reset` or `session:before-reset` hook event so session memory and custom hooks can persist context when daily or idle session auto-resets occur, plus schema support for hook-specific config such as `session-memory.messages`.

## Best Possible Solution

Land #71792, #61675, or a maintainer-approved replacement that makes automatic daily and idle stale rollovers invoke the same effective pre-reset persistence surface as manual resets, passes the previous session entry/transcript to session-memory, preserves typed lifecycle hooks, and adds daily plus idle regression coverage.
