## Summary

Keep open. Current main has fixed the secondary hook-config schema mismatch, but the primary automatic reset persistence gap remains: daily and idle rollovers emit typed lifecycle hooks only, while session-memory still listens to command:new/reset. Open PR #71792 explicitly closes this issue and #61675 overlaps, so this needs maintainer review of the existing candidates rather than another autonomous fix.

## What This Changes

Request a reset hook surface for automatic and manual session resets so session-memory and custom hooks can persist pre-reset context, plus hook-specific configuration schema support for session-memory messages.

## Best Possible Solution

Land #71792, #61675, or a maintainer-approved replacement that wires automatic daily and idle rollovers into the same effective pre-reset persistence surface as manual resets, supplies previous session transcript data, preserves typed lifecycle hooks, and adds daily plus idle regression coverage.
