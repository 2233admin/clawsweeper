## Summary

Keep open. Current main still does not route automatic daily or idle stale rollovers through the reset persistence path used by session-memory; the secondary hook-config schema mismatch is fixed, but the primary auto-reset persistence request remains unresolved and is already paired with open implementation candidates.

## What This Changes

The issue requests a session reset or before-reset hook for automatic daily/idle session resets so session-memory and custom hooks can persist context, plus schema support for per-hook config such as session-memory messages.

## Best Possible Solution

A maintainer-approved patch should make automatic daily and idle rollovers invoke the same effective pre-reset persistence surface as manual resets, pass previous session entry/transcript data and reset reason to session-memory and custom hooks, preserve typed lifecycle hooks, and add daily plus idle regression coverage.

## Reproduction Assessment

Yes. Configure a stale daily or idle session and send the next message: current main starts a new session with resetTriggered false, archives the previous transcript, emits lifecycle hooks, and leaves session-memory gated to command:new/command:reset.

## Solution Assessment

Unclear. The requested direction is valid, but the best implementation should be selected from PR #71792, PR #61675, or a maintainer-approved replacement so the repo does not gain competing reset hook semantics.
