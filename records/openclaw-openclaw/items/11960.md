## Summary

Keep open. Current main still fixes only the secondary hook-config schema mismatch; the primary auto daily/idle rollover path captures and archives the previous session and emits typed `session_end`/`session_start` plugin hooks, but it still does not emit the internal `command:reset` hook or `before_reset` path used by `session-memory`. Open PR #71792 carries closing syntax for this issue, with #61675 as a larger related implementation candidate, so this should remain open until one lands or maintainers choose a replacement.

## Best Possible Solution

Keep this issue open until #71792, #61675, or a maintainer-approved replacement lands a narrow fix that fires the existing internal reset hook behavior, or an equivalent documented reset surface, during automatic daily and idle stale rollover. The fix should pass `previousSessionEntry` to session-memory, preserve typed `session_end`/`session_start`, run `before_reset` if that remains the intended plugin contract, avoid duplicate manual-reset hook emission, and add daily plus idle regression coverage. The schema/config portion should remain as-is because current main already allows per-hook keys like `messages`.
