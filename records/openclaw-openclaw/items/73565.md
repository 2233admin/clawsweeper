## Summary

Keep open: current main still has direct transcript append sinks that bypass the guarded redaction path, and PR #73563 is the open, unmerged implementation candidate rather than a landed fix.

## What This Changes

The issue requests applying the existing transcript redaction guard to session transcript `appendMessage` persistence paths in core session management, CLI attempts, gateway injection, and the bundled Codex mirror.

## Best Possible Solution

Land PR #73563 or an equivalent maintainer-approved fix that wraps every bare transcript append sink in `guardSessionManager`, threads config where needed, preserves transcript update semantics, and adds regression coverage for redacted and opt-out transcript writes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by sending token-bearing text through the config transcript append, CLI text-turn transcript, gateway inject helper, or Codex mirror while transcript redaction is enabled; current main still shows direct append paths on those surfaces.

## Solution Assessment

Yes. Reusing `guardSessionManager` is the narrowest maintainable direction because it applies the existing before-write hook, transcript redaction, and tool-result guard infrastructure instead of introducing a parallel scrubber.
