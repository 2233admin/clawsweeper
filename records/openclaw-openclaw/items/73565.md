## Summary

Keep open: current main still has direct transcript append sinks that bypass the config-aware guard redaction path, and the linked open PR is the active implementation candidate rather than a merged fix.

## What This Changes

The issue requests that session transcript `appendMessage` persistence paths in core, CLI, gateway inject, and bundled plugin mirror surfaces pass through the existing redaction guard before writing JSONL transcripts to disk.

## Best Possible Solution

Review and land PR #73563 or an equivalent maintainer-approved fix that wraps each bare transcript append sink in guardSessionManager, threads config where redaction mode matters, preserves idempotency and transcript update semantics, and adds regression coverage for redaction plus redactSensitive="off" behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists by sending secret-like content through the config transcript append, CLI text-turn transcript, gateway inject helper, or Codex mirror while transcript redaction is enabled; current main shows direct appendMessage calls on those paths.

## Solution Assessment

Yes. Reusing guardSessionManager is the narrowest maintainable solution because it applies the existing before-write hook, transcript redaction, and tool-result guard infrastructure instead of creating a parallel scrubber.
