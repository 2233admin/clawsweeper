## Summary

Keep this issue open. Current main still has direct session transcript append paths that bypass the existing `guardSessionManager` redaction hook, and open PR #73563 is explicitly linked as the implementation candidate for this security-sensitive gap.

## What This Changes

The issue asks OpenClaw to route session transcript `appendMessage` persistence paths through the existing redaction guard before messages are written to disk.

## Best Possible Solution

Review PR #73563 against the transcript persistence contract, ensure every bare append sink uses the existing guard/redaction path with config where applicable, and close this issue only after that fix lands.

## Reproduction Assessment

Yes. A focused source-level reproduction path exists: send token-bearing content through the config transcript append, CLI attempt transcript, or gateway inject helper with redaction enabled; current main writes through bare `SessionManager` instances while guarded attempt/compact paths show the expected hook.

## Solution Assessment

Yes. Reusing `guardSessionManager` at the identified append entry points is the narrowest maintainable direction because it applies the existing hook and redaction infrastructure instead of introducing a parallel scrubber.
