## Summary

Keep open. Current main still computes Discord inbound DM `ctx.To` from the channel-shaped reply target before the user-shaped DM conversation target is available, while the existing tests only cover `lastRoute` and binding restoration. The focused related fix PR is closed unmerged, so this issue still tracks real remaining work.

## What This Changes

Discord inbound direct messages should populate semantic context with `ctx.To = user:<userId>` while leaving Discord API delivery targeted at the DM channel ID.

## Best Possible Solution

Fix the bundled Discord message-handler context so direct-message inbound `ctx.To` uses the resolved `user:<authorId>` conversation identity, while `deliverTarget` remains channel-based for Discord API sends. Add a regression assertion on dispatched `ctx.To`, `ctx.From`, and `ctx.ChatType` for inbound DMs, and leave any component-interaction semantic change as a separate maintainer decision.
