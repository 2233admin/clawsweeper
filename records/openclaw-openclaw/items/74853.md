## Summary

Keep open. The docs split is useful, but it is not merge-ready against current main because it publishes stable plugin docs for workflow APIs that are still absent while the dependent implementation PRs remain open.

## What This Changes

This PR adds a host-hook examples and recipes docs page, links it into the plugin docs navigation, and adds zh-CN glossary labels for the new page.

## Best Possible Solution

Ship the recipe page only once the public SDK and generated contracts include the documented workflow methods, or keep the material in proposed/RFC docs with explicit future-status language until then.

## Reproduction Assessment

Not applicable as a runtime reproduction. The high-confidence static check is to compare the documented methods in `docs/plugins/host-hooks-examples.md` against `OpenClawPluginApi` and `buildPluginApi` on current main.

## Solution Assessment

No, not as a standalone merge against current main. Splitting the large recipe page is maintainable, but stable docs should not publish examples for unmerged SDK methods.
