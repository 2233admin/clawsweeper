## Summary

Keep open. This is an active stacked docs PR, but it should not merge against current main because it adds stable plugin docs for workflow SDK methods that are not exposed by the current public plugin API; its own body says the docs are intended to land after #73384 and #74483.

## What This Changes

The PR adds a product-neutral host-hook examples and recipes page, links it from the plugin docs navigation, and adds zh-CN glossary entries for the new page.

## Best Possible Solution

Keep the docs split, but land it only after the dependent workflow SDK follow-ups have merged and the examples match the generated public SDK contracts; until then, keep future API material out of stable navigation or label it as proposed design material.

## Reproduction Assessment

Not applicable as a runtime reproduction. The high-confidence static check is to compare the methods documented in the added recipe page against `OpenClawPluginApi`, `buildPluginApi`, and the current SDK overview on main.

## Solution Assessment

No, not as a standalone merge against current main. Splitting the large recipe page is maintainable, but stable docs should not publish examples for unmerged SDK methods.
