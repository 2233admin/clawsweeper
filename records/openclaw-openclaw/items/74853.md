## Summary

Keep open. This is an active stacked docs PR, and current main does not already contain the new recipe page or navigation entry, but the patch is not safe to merge standalone because it publishes plugin SDK examples for methods that are not exposed by current main.

## What This Changes

The PR adds a host-hook examples and recipes documentation page, links it from the plugin docs navigation, and adds zh-CN glossary entries for the new page.

## Best Possible Solution

Keep the docs split, but merge it only after the implementation stack exposes the documented SDK methods and the examples match the shipped API. If it needs to land earlier, keep the page out of stable navigation or trim it to the host-hook foundation already on main.

## Reproduction Assessment

Not applicable as a runtime reproduction; this is a docs PR. The high-confidence static check is comparing the added page and nav entry to current main's `OpenClawPluginApi`, API builder, and host-hook types.

## Solution Assessment

No, not as a standalone merge against current main. Splitting the docs is reasonable, but publishing examples for unmerged SDK methods would give plugin authors guidance that does not compile yet.
