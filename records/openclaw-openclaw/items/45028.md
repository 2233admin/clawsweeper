## Summary

Keep open. Current main preserves external route metadata in the session store, but ordinary browser Control UI sends still resolve as internal WebChat turns and the reply dispatcher intentionally suppresses external delivery for non-explicit WebChat turns. The related open PR #46356 targets the broader webchat mirroring behavior, so this is not cleanup-closeable yet.

## What This Changes

Control UI replies typed while viewing a Telegram-backed session should route back to the underlying Telegram conversation instead of staying on internal WebChat.

## Best Possible Solution

Add a narrow, explicit Control UI operator-reply or webchat-mirroring path for externally backed sessions that can deliver to the stored channel route without weakening the existing WebChat route-leakage safeguards. The end state should cover browser composer behavior, Gateway route resolution, dispatcher explicit-delivery handling, and regression coverage for Telegram-style channel-scoped sessions plus WebChat-native sessions.
