## Summary

Keep open: current main still violates the documented macOS attach-only contract by writing the disable marker and then invoking the launchd uninstall path. The PR is a narrow app-side fix for the unresolved attach-only launch flag and Debug Settings paths, and the related launchd lifecycle work does not supersede it.

## What This Changes

The PR removes launchd disable/uninstall calls from the macOS attach-only launch flag and Debug Settings toggle paths, adds a marker-only helper, and adds a Swift regression test for attach-only behavior without daemon commands.

## Best Possible Solution

Land the marker-only attach-only behavior for both the launch flags and Debug Settings toggle, with the focused regression test, after normal macOS Swift test and launchd lifecycle validation.
