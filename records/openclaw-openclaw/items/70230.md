## Summary

Current main still does not expose the requested `connected`, `rttMs`, or `lastHeartbeatAt` connection telemetry on the `health` RPC, and there is no `gateway.ping` method. The current implementation returns a cached service/channel/agent/session health snapshot, while WebSocket liveness is tracked only privately via tick handling.

## Best Possible Solution

Add an additive current-connection telemetry surface for authenticated Gateway clients. Prefer a `connection` block over loose top-level fields if it lands on `health`, and overlay it per response rather than caching it globally. A separate lightweight `gateway.ping` is also acceptable if maintainers prefer UI polling to avoid the heavier health snapshot path. Keep channel-specific stale-socket work separate.
