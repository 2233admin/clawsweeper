## Summary

Keep this issue open. Current main still pre-fills the interactive `agents add` workspace prompt from `resolveAgentWorkspaceDir(cfg, agentId)`, and that resolver still nests non-default agents under `agents.defaults.workspace`. The docs still describe peer-level `workspace-<agentId>` paths, and open fix PRs with closing syntax already exist, so the right cleanup path is to review and land or close one canonical PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and review the existing implementation PRs, preferably landing one narrow fix that changes the interactive `agents add` default for brand-new agents to a peer-level `workspace-<id>` path while preserving explicit workspaces and existing runtime resolver compatibility. Add focused regression coverage for the wizard prefill and resolver edge cases, plus a user-facing changelog entry if the fix merges.
