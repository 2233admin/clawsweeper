## Summary

Keep open. Current `main` still has the reported wizard default mismatch, the latest stable tag still has the same source behavior, and multiple open implementation PRs already target the bug, so cleanup should not close this issue before one of those PRs lands.

## What This Changes

The issue requests that the interactive `agents add <id>` wizard pre-fill the workspace prompt with the documented peer-level `workspace-<id>` path instead of nesting under the main `workspace/<id>` directory.

## Best Possible Solution

Review and land one narrow implementation PR that changes the interactive default for brand-new agents to a peer-level workspace suggestion, preserves explicit and existing workspace paths, adds wizard/resolver regression coverage, and includes a user-facing changelog entry when merged.

## Reproduction Assessment

Yes. The source path is direct: setup stores `agents.defaults.workspace`, the interactive wizard uses `resolveAgentWorkspaceDir(cfg, agentId)` for the prompt, and the non-default resolver branch appends the id to that fallback path.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: change the wizard prefill for new agents while keeping runtime resolution compatible for existing configs; the exact PR to land is a maintainer selection among the open implementations.
