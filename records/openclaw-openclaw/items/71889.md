## Summary

Keep open. Current main still seeds the interactive `agents add` workspace prompt from the generic runtime resolver, and that resolver still nests non-default agents under `agents.defaults.workspace`; the docs continue to describe peer-level `workspace-<agentId>` workspaces, and open fix PRs already reference this issue for maintainer review.

## What This Changes

The issue requests changing the interactive `openclaw agents add <id>` workspace default from a nested `<main workspace>/<id>` path to the documented peer-level `<main workspace>-<id>` layout.

## Best Possible Solution

Land one narrow fix from the existing PR set that changes the interactive default for brand-new agents to a peer-level workspace path, preserves explicit and legacy workspace paths, adds focused wizard/resolver regression coverage, and includes a user-facing changelog entry.

## Reproduction Assessment

Yes. The source path is deterministic: setup persists `agents.defaults.workspace`, `agents add` uses `resolveAgentWorkspaceDir` as the prompt default, and the resolver joins that fallback with a non-default agent id.

## Solution Assessment

Yes. The requested wizard-default fix is the narrowest maintainable direction; changing runtime resolution broadly could relocate existing agents unless maintainers explicitly choose that compatibility tradeoff.
