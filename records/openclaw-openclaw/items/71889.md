## Summary

Keep open. Current main still seeds the interactive `agents add` workspace prompt from the resolver path that nests a non-default agent under `agents.defaults.workspace`, while the docs continue to describe peer-level `workspace-<agentId>` workspaces; three open PRs already target the issue, so this should stay open until one fix is reviewed and merged.

## What This Changes

The issue requests changing the interactive `openclaw agents add <id>` workspace default from `<main workspace>/<id>` to the documented peer-level `workspace-<id>` layout.

## Best Possible Solution

Review and land one narrow existing fix that changes only the new-agent wizard default to a documented peer-level workspace, preserves explicit and legacy workspace paths, adds regression coverage for the prompt/default helper, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The current-main source path is deterministic: setup writes `agents.defaults.workspace`, the add wizard uses `resolveAgentWorkspaceDir` for the prompt default, and the resolver joins that fallback with the non-default agent id.

## Solution Assessment

Yes. A targeted wizard/default-helper fix is the best direction because it prevents new nested workspaces without silently migrating existing agents that already have explicit or legacy paths.
