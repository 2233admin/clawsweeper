## Summary

The report should stay open. Current main still calls `clearStaleAssistantUsageOnSessionMessages()` after a non-retry compaction end, and that helper still zeroes every assistant `usage` snapshot. There is a correct timestamp/index-gated pattern elsewhere in the runner, and the prior fix PR #50845 was closed unmerged.

## Best Possible Solution

Keep the issue open and apply a narrow fix in the embedded subscribe compaction handler: find the latest compaction summary, zero only assistant usage snapshots before that boundary using timestamp plus index fallback, and add a focused regression test proving post-compaction assistant usage is preserved. Reuse the replay-history logic shape rather than changing provider usage normalization or UI display code.
