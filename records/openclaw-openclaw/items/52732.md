## Summary

Keep this issue open. Current main at a820a307 still accepts `compaction` and `contextPruning` only under `agents.defaults`; `agents.list[]` remains strict without those fields, the resolved agent config does not merge them, and Pi compaction/pruning runtime paths still read defaults-only config. The linked #52751 is useful prior art but is open/unmerged, so it is not a reason to close this yet.

## Best Possible Solution

Keep this issue open. The best path is to add typed strict-schema support for `agents.list[].compaction` and `agents.list[].contextPruning`, merge each agent's values over `agents.defaults`, thread the effective settings through Pi compaction, safeguard compaction, cache-ttl pruning, memory-flush/preflight paths, and document/test precedence. Reviewing, revising, or superseding #52751 is the likely implementation route.
