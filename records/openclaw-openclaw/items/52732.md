## Summary

Keep this issue open. Current main still accepts `compaction` and `contextPruning` only under `agents.defaults`; `agents.list[]` remains strict without those fields, the per-agent resolver does not merge them, and Pi compaction/pruning runtime paths still read defaults-only config. The related #52751 is useful implementation prior art, but it is open and unmerged.

## Best Possible Solution

Keep this issue open. The best path is to add typed strict-schema support for `agents.list[].compaction` and `agents.list[].contextPruning`, merge each agent's values over `agents.defaults`, thread the effective settings through Pi compaction/pruning runtime paths, and document/test precedence. Reviewing, revising, or superseding #52751 is the likely implementation route.
