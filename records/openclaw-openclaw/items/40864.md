## Summary

Keep open. Current main has a working `sessions_send` path and a durable session-delivery queue, but ordinary `sessions_send` still dispatches directly to the Gateway `agent` method, returns timeout/error without a persisted delivery/ack state, and treats A2A announce delivery as best-effort. The durable queue is wired to restart-continuation recovery, not a general persistent inbox for at-least-once agent-to-agent delivery.

## What This Changes

The issue requests persistent at-least-once agent-to-agent delivery for `sessions_send`, including a durable inbox/queue, retries, acknowledgement, and idempotency handling.

## Best Possible Solution

Define a core reliable A2A delivery contract: persist accepted `sessions_send` messages before dispatch, drain them on startup or heartbeat, record stable acknowledgement/idempotency state, and cover timeout, restart, and duplicate-delivery behavior in session-tool, Gateway, queue, and docs coverage.
