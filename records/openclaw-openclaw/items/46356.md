## Summary

Keep this PR open. Current main does not implement the central webchat bidirectional mirror behavior: WebChat chat.send still stays on the internal webchat route, current tests explicitly preserve that no-inherit behavior for channel-scoped sessions, and there is no webchat mirror target/runtime helper on main. The PR may still be useful, but it is not obsolete: it also carries unresolved post-head review findings and broad unrelated/security-sensitive changes, so the best maintainer path is review, rebase, split, and fix rather than cleanup-close.

## Best Possible Solution

Keep this PR open, but do not land it as-is. The best path is to rebase onto current main, split the work into focused PRs, keep the webchat mirror change scoped to gateway/session delivery with regression tests for ordering, commands, dedupe, deliver=true, and channel target parsing, and handle Discord/WhatsApp proxy support against the current proxy-request-client/env-proxy implementation. Unrelated Matrix/Tlon/SDK/lockfile changes should be removed or moved to separate owner-reviewed PRs.
