## Summary

Keep this issue open. Current main still lacks WhatsApp support for `silentErrorReplies`, `suppressAgentErrors`, `errorPolicy`, `errorCooldownMs`, or custom agent-failure reply text, while Telegram has a dedicated error-policy path. The same author also has an open paired implementation PR (#59801), so closing the issue now would orphan the active implementation path.

## Best Possible Solution

Keep the issue open and resolve it through #59801 or a revised maintainer-approved implementation: add typed WhatsApp channel/account error policy config, schema/generated metadata/docs, runtime suppression/cooldown handling in WhatsApp auto-reply delivery, and focused tests for DM and account-scoped behavior.
