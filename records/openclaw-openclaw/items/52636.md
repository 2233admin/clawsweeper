## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Enforce the loopback 10x threshold in the lockout path:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `recordFailure()` still uses the normal `maxAttempts` lockout for loopback clients, while `check()` returns from `isExempt()` before honoring `lockedUntil` as long as the attempt count is below `maxAttempts * 10`. Once that normal lockout is set, additional loopback failures stop being recorded, so the counter never reaches the intended 10x ceiling and localhost failures remain allowed indefinitely.
  - confidence: 0.9
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` checks hex string lengths before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature decodes to a shorter buffer and makes verification throw instead of returning `false`, turning a bad signature into an exception path for callers.
  - confidence: 0.86
- **[P2] Refresh escalation state when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` updates circuit state but does not recompute `this.escalation`. After an open circuit closes, `getStatus()` and `getSummary()` can keep reporting `PARTIAL_PAUSE` or `FULL_PAUSE` until another circuit opens.
  - confidence: 0.84
- **[P2] Drop stale health scores when metrics leave the window:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters metrics to the recent window but only updates functions present in the new aggregate. Functions with no recent metrics keep old `CRITICAL` or `CIRCUIT_OPEN` scores, so `shouldThrottle()` can continue throttling idle or recovered functions from stale state.
  - confidence: 0.82
- **[P2] Collapse health metrics by function before storing scores:** `src/security/shield/gateway-shield.ts:287`
  - body: `aggregateByFunctionAndMinute()` returns buckets per function and minute, but this loop writes them into `healthScores` keyed only by function name. If metrics are not inserted in chronological bucket order, an older minute can overwrite a newer score and make throttling decisions depend on arrival order.
  - confidence: 0.78
