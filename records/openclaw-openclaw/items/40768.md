## Summary

Keep open. Current main still recognizes Feishu bot mentions by exact `open_id` comparison against the startup-resolved `botOpenId`, and the group `requireMention` path still drops messages when that exact match fails. The issue is also paired with open PR #40782, which uses closing syntax for this bug, so the best next step is PR review or replacement rather than closing this issue.

## What This Changes

The issue requests a Feishu mention-detection fix for multi-bot groups where app-scoped `open_id` values in WebSocket mention payloads do not match the receiving bot's startup-resolved `open_id`.

## Best Possible Solution

Keep this issue open and review, refresh, or replace PR #40782. The eventual Feishu-owned fix should verify the stable identity source with Feishu/Lark, store the needed per-account identity state in the Feishu monitor, update `checkBotMentioned()` plus mention stripping, mention-forward, and debounce paths consistently, add regression tests for app-scoped `open_id` mismatch, and include a user-facing changelog note if behavior changes.
