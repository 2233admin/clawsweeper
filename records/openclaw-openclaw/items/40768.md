## Summary

Keep open. Current main still gates Feishu group messages by comparing the pushed mention `open_id` to the startup-resolved `botOpenId`, while the startup/probe state stores only `botOpenId` and `botName`. That leaves the reported app-scoped `open_id` mismatch unresolved. The issue is also paired with open PR #40782, which uses closing syntax for this issue, so the right maintainer path is to review, refresh, or replace that PR rather than close the report first.

## Best Possible Solution

Keep this issue open and review #40782 or request a refreshed Feishu-owned replacement. The fix should verify the stable identity source with Feishu/Lark, store whatever per-account identity state is needed in the Feishu monitor, update `checkBotMentioned()` and related mention-forward/debounce paths consistently, add regression tests for app-scoped `open_id` mismatch, and include a user-facing changelog note if behavior changes.
