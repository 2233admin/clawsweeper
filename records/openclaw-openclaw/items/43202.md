## Summary

Keep this issue open. Current main still drops the requested Feishu card-action fields before the handler can use them: the typed event only carries `action.value` and `action.tag`, the production parser reconstructs only those two action fields, and the fallback message path serializes only `action.value`. The provided GitHub context also shows an open same-author implementation PR with closing syntax for this issue, plus another related open Feishu PR, so the maintainer path is to review or supersede those PRs rather than close the issue now.

## Best Possible Solution

Keep this issue open and review the linked Feishu implementation PRs, especially same-author PR #43953. The durable fix should preserve `form_value`, `input_value`, `name`, `option`, and `options` from the raw `card.action.trigger` callback through `parseFeishuCardActionEventPayload`, expose them on the typed event contract, serialize them only when present so simple button behavior stays backward compatible, and add regression coverage from raw callback payload through handler dispatch.
