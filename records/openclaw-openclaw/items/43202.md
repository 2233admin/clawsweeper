## Summary

Keep this issue open. Current main still drops the Feishu card-action fields requested here: the typed event only exposes `action.value` and `action.tag`, the production parser reconstructs `action` with only those fields, and the fallback synthetic-message path serializes only `action.value`. The provided discussion also shows an open same-author PR with closing syntax for this issue, so the maintainer path is to review, land, or supersede that PR rather than close the issue now.

## Best Possible Solution

Keep this issue open and review or supersede the linked Feishu implementation PRs, especially same-author PR #43953. The durable fix should preserve `form_value`, `input_value`, `name`, `option`, and `options` from the raw `card.action.trigger` callback through `parseFeishuCardActionEventPayload`, expose them on the typed event contract, serialize them only when present so simple button behavior stays backward compatible, and add regression coverage from raw callback payload through handler dispatch.
