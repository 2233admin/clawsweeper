## Summary

Keep open. Current main has explicit manual update paths for ClawHub-installed skills and tracked plugins, and the package auto-updater can run plugin sync only as a post-core-update step; it does not implement the requested opt-in background updater for installed skills/plugins themselves. The feature is valid, but it needs maintainer product and supply-chain design before implementation.

## What This Changes

The issue requests an opt-in mechanism to automatically update ClawHub-installed skills and installed plugins on gateway start or a configurable interval.

## Best Possible Solution

Design an explicit opt-in updater that reuses the existing tracked skill and plugin update helpers while adding consent, cadence controls, pinning or allowlist behavior, integrity/signing expectations, failure reporting, audit metadata, rollback guidance, and user-facing docs for the security model before enabling background installs of third-party code.
