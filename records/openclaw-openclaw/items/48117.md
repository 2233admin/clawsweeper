## Summary

Keep open. Current main and `v2026.4.26` still route the Windows PowerShell installer through a plain `npm.cmd install -g ... --no-fund --no-audit` call and fail on any nonzero npm exit; there is no Windows TLS trust-store remediation for `UNABLE_TO_VERIFY_LEAF_SIGNATURE`. The related exact fix PR #48416 was closed unmerged, while #51146 addressed the adjacent Linux/nvm TLS startup problem only.

## Best Possible Solution

Keep this issue open and implement a maintainer-reviewed Windows installer TLS recovery path. The safest direction is to detect npm TLS trust failures, preserve strict verification by default, prefer a scoped CA trust repair such as npm `cafile` or `NODE_EXTRA_CA_CERTS` where feasible, document enterprise-proxy recovery, and only use a scoped `--strict-ssl=false` retry if maintainers explicitly accept that security tradeoff. Update `scripts/install.ps1`, installer docs, and the Windows smoke/release-check coverage or live proof before closing.
