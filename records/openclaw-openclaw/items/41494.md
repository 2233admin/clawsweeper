## Summary

Keep open. Current main still has structured Gemini reasoning protections, but it does not cover the broader plain-text and multilingual reasoning preambles reported here, and the discussion points to open PR #60551 as an active candidate fix rather than shipped behavior.

## Best Possible Solution

Keep this issue open as the Gemini plain-text reasoning leakage tracker. Review and land PR #60551 or a replacement that adds a conservative shared outbound/message-tool sanitizer with tests for untagged planning prose, appended reasoning blocks, multilingual reasoning headers, tagged fragments, code/prose pass-through, and normal /reasoning visibility.

## Risks / Open Questions

- A broad plain-text sanitizer can false-positive legitimate prose, quoted examples, code blocks, or multilingual answers unless it is tightly scoped and covered by regression tests.
- Existing structured paths should not regress: native part.thought handling, isReasoning payload suppression, tagged <think>/<final> stripping, and intentional /reasoning stream visibility.
- Strict final-tag enforcement may prevent some leaks by dropping untagged output, but that is not the same as safely extracting the final answer from mixed reasoning plus answer text.
