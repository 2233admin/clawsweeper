## Summary

Keep this open. Current main still handles structured Gemini reasoning, explicit `isReasoning` payloads, and tagged `<think>/<final>` cleanup, but it does not implement the broader plain-text or multilingual reasoning-preamble stripping reported here. The discussion also points to open PR #60551 as an implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open as the Gemini plain-text reasoning leakage tracker. Review and land PR #60551 or a replacement that adds a conservative shared outbound/message-tool sanitizer with tests for untagged planning prose, appended reasoning blocks, multilingual reasoning headers, tagged fragments, code/prose pass-through, and normal `/reasoning` visibility.
