## Summary

Current main has structured Gemini reasoning protections, but it still does not cover the broader plain-text and multilingual reasoning preambles reported here. The recent discussion reports reproduction through v2026.4.15 and points to open PR #60551 as a candidate fix, so this issue should stay open.

## Best Possible Solution

Keep this open as the Gemini plain-text reasoning leakage tracker. Review and land PR #60551 or a replacement that adds a conservative outbound/message-tool sanitizer with tests for untagged planning prose, appended reasoning blocks, multilingual reasoning headers, tagged fragments, and normal prose/code pass-through.

## Risks / Open Questions

- A broad plain-text sanitizer can false-positive legitimate prose, quoted examples, code blocks, or multilingual answers unless it is tightly scoped and covered by regression tests.
- Existing structured paths should not regress: native `part.thought`, `isReasoning` payload suppression, tagged `<think>/<final>` stripping, and intentional `/reasoning stream` visibility.
- Strict final-tag enforcement may prevent some leaks by dropping untagged output, but that is not the same as safely extracting the final answer from mixed plain reasoning plus answer text.
