## Summary

Keep open. Current main still matches the report: the flattened `nodes` tool schema advertises `node` as optional, but the `describe` branch still reads `node` as required before default-node resolution can run. Same-author PR #51926 remains open and is an implementation candidate that says it would close this issue, so this issue should stay open until that PR is reviewed/landed or replaced.

## Best Possible Solution

Keep this issue open while PR #51926 remains open. The best path is to review and land that PR or replace it with a maintained fix in `src/agents/tools/nodes-tool.ts` that makes `describe` intentionally accept omitted `node` through default-node resolution or discovery output, plus a focused regression test in `src/agents/tools/nodes-tool.test.ts`; targeted or mutating actions should keep explicit node requirements.
