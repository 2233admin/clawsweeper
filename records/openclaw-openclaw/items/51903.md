## Summary

Keep open. Current `origin/main` still has the reported schema/runtime mismatch: the `nodes` tool schema accepts omitted `node`, but the `describe` branch requires it before default-node resolution can run. Same-author open PR #51926 remains the active closing implementation path, so this issue should stay open until that PR is reviewed, merged, or replaced.

## What This Changes

The issue requests that the agents `nodes` tool handle `action: "describe"` without an explicit `node` consistently with its schema instead of failing with `node required`.

## Best Possible Solution

Review and land PR #51926, or replace it with a maintained patch in `src/agents/tools/nodes-tool.ts` that intentionally accepts omitted `node` for the read-only `describe` action while preserving explicit-node requirements for targeted actions, with focused regression coverage in `src/agents/tools/nodes-tool.test.ts`.
