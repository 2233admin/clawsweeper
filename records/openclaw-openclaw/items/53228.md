## Summary

Keep this open. Current main still has adjacent Web Push and transcript-injection pieces, but it does not provide the requested shell-facing `openclaw notify <message>` command, typed admin-scoped Gateway notification RPC/event, or Control UI notification handler for connected UI sessions.

## What This Changes

Request a CLI command that broadcasts visible notifications to connected UI sessions from shell scripts without routing through the agent turn queue.

## Best Possible Solution

Define a core notification contract: a typed admin-scoped Gateway RPC, an allowlisted broadcast event for connected Control UI clients, a CLI entry point such as `openclaw notify <message>`, UI rendering for the event, and docs/tests explaining how it differs from `chat.inject`, `system event`, and Web Push.
