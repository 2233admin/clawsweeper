## Summary

Keep this PR open for maintainer triage. Current main has already absorbed or replaced some cleanup pieces, but it does not implement the central CI/docs behavior: the fast security workflow still runs `detect-private-key` and `zizmor` only, while the security docs still describe a stale `detect-secrets` job and changed-file path. The PR should be rebased and narrowed to current main before any merge decision.

## Best Possible Solution

Keep the PR open, but do not merge it as-is. The useful remaining product outcome is to rebase onto current main, add a `detect-secrets` step to `security-scm-fast` using the same trusted pre-commit config pattern as the existing security steps, update `docs/gateway/security/index.md` and `docs/ci.md` to match, and drop already-landed/stale cleanup such as the cron helper removal and zh-CN symlinks unless a maintainer explicitly wants them restored.

## Risks / Open Questions

- Security review: the PR touches CI secret scanning. On current main, the existing `detect-private-key` and `zizmor` steps use `--config "${PRE_COMMIT_CONFIG_PATH:-.pre-commit-config.yaml}"` after preparing a trusted base pre-commit config for pull requests. The PR patch as written adds `pre-commit run --all-files detect-secrets` without that config guard, so a rebase should adapt the step to the current trusted-config pattern before merge.
- The PR includes docs/zh-CN symlinks, but current main has no `docs/zh-CN` tree and the scoped docs policy says localized docs are not maintained in this repo. That part should be dropped or explicitly justified by a maintainer.
