## Summary

Keep this PR open. Current main still lacks the baseline-backed `detect-secrets` CI run, and the security docs still describe an older `secrets` job / PR fast path. The central change remains useful, but the branch should be rebased and narrowed because side changes are stale and the CI step needs the current trusted pre-commit config pattern.

## Best Possible Solution

Keep the PR open for maintainer follow-up, but do not merge it as-is. Rebase onto current main, add `detect-secrets` to `security-scm-fast` using the trusted pre-commit config pattern, update `docs/gateway/security/index.md` and `docs/ci.md` to describe the actual CI job, and drop the stale `docs/zh-CN` symlinks plus already-absorbed cron cleanup.
