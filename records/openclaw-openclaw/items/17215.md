## Summary

Keep open. Current main has adjacent upgrade-safety pieces, including update dry-run planning, post-update doctor repair, plugin sync/integrity-drift reporting, plugin compatibility notices, and backup docs, but it still does not implement the requested native pre/post-update compatibility scanner, `doctor --upgrade-scan`, risk-scored JSON/human report, smoke recommendations, or rollback recommendation hints. The existing maintainer/Codex comment also kept this open, and the related backup/rollback issue plus plugin-compatibility PR are still adjacent open work rather than merged replacements.

## What This Changes

Add a native pre/post-update compatibility scanner, likely through `doctor --upgrade-scan`, with schema/deprecation checks, smoke recommendations, risk-scored JSON/human reports, and rollback hints.

## Best Possible Solution

Keep this open as a core update/doctor safety enhancement. A maintainer should define a stable `openclaw update` and/or `openclaw doctor --upgrade-scan --json` report contract that reuses config validation/migrations, doctor health contributions, plugin compatibility diagnostics, update dry-run planning, plugin sync/integrity-drift checks, health/status probes, backup support, and documented rollback guidance. Coordinate with #14526 for backup/rollback policy and #54354 for plugin compatibility, but do not close this until the scanner/report contract is implemented, tested, and documented.
