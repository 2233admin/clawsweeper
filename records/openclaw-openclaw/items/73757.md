## Summary

Keep this PR open. Current main still has the documented `host=node`/`host=auto` mismatch, while the PR is a narrow docs-only correction that matches the existing runtime guard and regression test.

## What This Changes

The PR updates three exec-related docs pages to state that per-call `host=node` and `host=gateway` overrides from `tools.exec.host=auto` are allowed only when no sandbox runtime is active.

## Best Possible Solution

Land this docs-only correction, or apply equivalent wording to `docs/tools/exec.md`, `docs/tools/exec-approvals.md`, and `docs/nodes/index.md`, so the published exec docs match the current runtime/test contract; after that, #61009 can close as implemented.

## Reproduction Assessment

Yes. A static current-main check reproduces the mismatch: the docs say per-call `host=node` is allowed from `auto`, while `resolveExecTarget` rejects that request when `sandboxAvailable` is true and the regression test locks the rejection.

## Solution Assessment

Yes. The runtime behavior is already deliberate and tested, so updating the three misleading docs paragraphs is the narrowest maintainable fix for the reported problem.
