## Summary

Keep open. Current main still emits a macOS LaunchAgent with `KeepAlive=true` and `ThrottleInterval=1`, and the missing-config startup guard still exits with config error 78 before the in-process gateway loop can apply any restart delay or lock-recovery behavior. The related #60885 is adjacent launchd lifecycle work, but it does not specifically cover repeated missing-config respawns and log storms.

## What This Changes

The issue asks OpenClaw to prevent macOS launchd crash-loop log storms when missing or invalid gateway config causes repeated exits under `KeepAlive` with a one-second throttle.

## Best Possible Solution

Add macOS-specific service/startup behavior that prevents repeated missing-config or config-error respawns while preserving normal crash recovery, explicit restarts, and update restarts. The end state should include regression coverage for the LaunchAgent plist/startup guard path, docs for the troubleshooting behavior, and coordination with #60885 only where the broader launchd lifecycle policy overlaps.
