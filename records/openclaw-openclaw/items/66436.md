## Summary

Keep open. Current main still emits a macOS LaunchAgent with `KeepAlive=true` and `ThrottleInterval=1`, and the missing-config startup guard still exits with config error 78 before the gateway loop can apply any in-process restart delay or backoff. The related #60885 is adjacent launchd lifecycle work, but it does not explicitly cover repeated missing-config respawns and log storms.

## What This Changes

Prevent macOS LaunchAgent-managed gateways from repeatedly respawning and filling logs when startup exits on missing or invalid config.

## Best Possible Solution

Add a macOS-specific service/startup behavior that prevents repeated missing-config or config-error respawns while preserving normal crash recovery and intentional restarts. The end state should include regression coverage for LaunchAgent plist/startup behavior, a documented troubleshooting path, and coordination with #60885 only where the broader launchd lifecycle policy overlaps.
