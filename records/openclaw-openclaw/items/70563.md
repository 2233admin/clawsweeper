## Summary

Keep this issue open. Current main still has the reported Kimi Code shape: native `sessions_spawn` can forward a thinking override into the child run, but the Kimi provider wrapper only patches top-level `thinking` and does not preserve or backfill `reasoning_content` on replayed assistant messages. Related #50178 and #71491 are useful context for the same provider-family error, but they do not supersede this Kimi Code subagent path.

## What This Changes

The issue requests that thinking-enabled native subagent spawns with Kimi Code succeed instead of failing when replayed assistant tool-call messages lack `reasoning_content`.

## Best Possible Solution

Add a provider-owned replay/payload repair for Kimi Code so thinking-enabled outgoing Kimi requests keep existing `reasoning_content` and add a Kimi-accepted placeholder for replayed assistant messages with tool calls when needed, with focused regression coverage for `sessions_spawn` thinking defaults and Kimi tool-call continuation.
