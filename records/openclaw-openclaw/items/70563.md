## Summary

Keep this issue open. Current main still has the reported Kimi Coding failure shape: native subagent spawn can carry a thinking override into the child run, while the bundled Kimi provider only patches the top-level `thinking` payload and does not add or preserve `reasoning_content` on replayed assistant tool-call messages. The prior Codex review comment remains consistent with current `main`; related #35023 fixed a different Anthropic replay path and #50178 is broader/provider-different rather than a canonical Kimi fix.

## Best Possible Solution

Keep this issue open and fix the provider-owned replay/payload path for `extensions/kimi-coding`. When Kimi thinking is enabled, preserve provider reasoning metadata where available or add a Kimi-accepted `reasoning_content` placeholder for replayed assistant tool-call messages, then add focused regression coverage for thinking-enabled native `sessions_spawn` plus Kimi tool-call continuation. Cross-check #50178 for a reusable provider seam, but do not close this Kimi-specific report as implemented or duplicated yet.
