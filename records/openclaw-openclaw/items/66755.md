## Summary

Keep open. Current main still has exact-token-only silent detection, while this PR is an active implementation candidate for the linked reasoning-wrapper leak. The latest PR head addresses the prior blank-line false-positive review, and I did not find a new blocking correctness or security issue in the diff; maintainers still need to choose this narrow policy or a broader trailing-token policy such as #69394.

## What This Changes

The PR adds reasoning-wrapper-aware silent-token detection in `src/auto-reply/tokens.ts` with tests for bare `think` and `<think>` wrapped `NO_REPLY` outputs plus false-positive regressions.

## Best Possible Solution

Land one canonical fix that suppresses reasoning-only silent replies without dropping substantive replies, then close the linked bug and close competing open alternatives as superseded once the chosen implementation is merged.
