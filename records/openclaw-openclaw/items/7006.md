## Summary

Keep open. Current main still stores and displays the selected OpenRouter auto placeholder and locally estimated token cost, not OpenRouter's resolved routed model and provider-reported cost for `openrouter/auto`. The related ClawRouter comment is useful prior art, but it routes through a different service and does not satisfy transparency for OpenRouter's own auto router.

## Best Possible Solution

Keep this issue open and implement a focused OpenRouter auto transparency path: capture the response/chunk resolved model and provider-reported usage cost from OpenRouter's OpenAI-compatible transport, preserve selected-versus-resolved model metadata through assistant/session records, and surface the resolved model plus clearly labeled provider cost in `/usage` response footers and related usage/status surfaces. Treat #9016 and #68066 as related cost-tracking work, and PR #43026 plus the ClawRouter comment as useful prior art rather than replacements.
