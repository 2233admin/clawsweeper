## Summary

Keep open. Current main covers the workspace-file part through shared Project Context/bootstrap injection, but the requested Slack account/global `systemPrompt` surface is still absent from the typed config, strict schema, and Slack runtime prompt resolution paths.

## What This Changes

Request to add Slack-wide `channels.slack.systemPrompt` inheritance and workspace-file context parity so Slack does not require repeating the same prompt on every channel entry.

## Best Possible Solution

Add a documented Slack account/global prompt field that inherits through base-to-account config, stacks predictably with per-room overrides, and applies consistently across Slack rooms, private channels, DMs, MPIM/group DMs, and native slash commands while keeping shared Project Context as the workspace-file mechanism.
