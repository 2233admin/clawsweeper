## Summary

Keep this issue open. Current main already covers the workspace-file part through shared Project Context/bootstrap files, but the requested Slack account/global `systemPrompt` surface is still missing from the typed config, strict schema, Slack monitor context, message path, and native slash-command path.

## What This Changes

The issue requests a Slack-wide `systemPrompt` fallback and Slack workspace/bootstrap file context parity with Telegram, so users do not need to repeat the same prompt file per Slack room.

## Best Possible Solution

Keep this issue open and implement only the remaining Slack account/global prompt surface: add `channels.slack.systemPrompt` with per-account override/inheritance, apply it consistently to Slack message and native slash-command paths for channels, private channels, DMs, and MPIM/group DMs, and keep shared Project Context as the workspace-file solution.
