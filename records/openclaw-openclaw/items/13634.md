## Summary

Current main still implements the shared workspace bootstrap/Project Context part, but not the remaining Slack account/global `channels.slack.systemPrompt` surface. Slack only accepts `systemPrompt` inside per-channel entries and only feeds that prompt through room-style message/slash-command paths.

## Best Possible Solution

Keep this issue open and implement the remaining account/global Slack prompt surface. Add `channels.slack.systemPrompt` and per-account override support to Slack config types/schema/docs/config metadata, then plumb it through Slack message and native slash-command paths with precedence: exact channel entry, wildcard channel entry, then account/global prompt. Apply it consistently for public/private channels, DMs, and MPIM/group DMs. Do not add Slack-specific workspace file injection; shared Project Context already handles AGENTS.md/SOUL.md/MEMORY.md-style bootstrap files.
