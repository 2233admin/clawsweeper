## Summary

Keep open. Current main has a useful default-off package auto-updater under `update.auto.*`, but it is still the narrower stable-delay/jitter and beta-cadence updater, not the requested scheduled confirm/notify/silent workflow with timezone, channel notifications, and post-restart reporting.

## Best Possible Solution

Keep this open as the canonical updater engine/workflow request. Extend the existing `update.auto` package updater with schema/types/help/docs/tests for `mode`, `schedule`, `timezone`, `notifyAfterUpdate`, and `notifyChannel`; implement timezone-aware HH:MM or cron scheduling; add notify-only, confirm, and silent flows through safe configured channel targets; persist update state across restart; and report old/new version, changelog link or summary, and health after restart. Leave #65011 as the separate Control UI/onboarding/discoverability follow-up.
