## Summary

Keep this issue open. Current main has a useful default-off package auto-updater under `update.auto.*`, but it still does not implement the requested first-class workflow: wall-clock or cron scheduling, timezone selection, confirm/notify-only modes, channel-targeted notifications, or post-restart old/new version plus changelog/health reporting. Related PR #12896 is closed unmerged, and #65011 is an open UI/discoverability follow-up rather than a replacement for this engine/workflow request.

## Best Possible Solution

Keep this issue open as the canonical updater engine/workflow request. Extend the existing `update.auto` package updater with schema/types/help/docs/tests for `mode`, `schedule`, `timezone`, `notifyAfterUpdate`, and `notifyChannel`; implement timezone-aware HH:MM or cron scheduling; add notify-only, confirm, and silent flows through configured channel targets; persist update state across restart; and report old/new version, changelog link or summary, and health after restart. Leave #65011 as the separate Control UI/onboarding/discoverability follow-up.
