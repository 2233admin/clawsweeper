## Summary

Keep open. Current main still lacks the OpenRouter thinking-profile hook this PR is trying to add, but the proposed patch has a concrete regression: its custom profile drops the existing `minimal` thinking level for reasoning-capable OpenRouter models.

## What This Changes

The PR adds an OpenRouter `resolveThinkingProfile` hook that returns `off`, `low`, `medium`, `high`, and `xhigh` for cached reasoning-capable OpenRouter models while excluding the Hunter Alpha proxy route.

## Best Possible Solution

Revise the provider-owned OpenRouter thinking profile so it preserves the existing base levels and adds `xhigh` for reasoning-capable OpenRouter models, then add targeted tests for DeepSeek V4, non-reasoning models, and the Hunter Alpha exclusion.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence reproduction path: current main rejects `/think xhigh` because OpenRouter has no xhigh profile hook, and the PR diff would reject `/think minimal` for reasoning-capable OpenRouter models because the returned levels omit it.

## Solution Assessment

No. `resolveThinkingProfile` is the right narrow seam, but this PR is not the best implementation until it preserves `minimal` and has focused provider tests.

## Review Findings

Add `{ id: "minimal" }` to the returned levels.
