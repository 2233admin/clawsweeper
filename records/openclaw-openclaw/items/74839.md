## Summary

Keep open. Current main does not implement OpenRouter `xhigh` thinking-profile support, and the PR targets the right provider-owned hook for the linked bug without introducing a discrete blocking correctness issue in this read-only review.

## What This Changes

The PR adds an OpenRouter `resolveThinkingProfile` hook that exposes `off`, `low`, `medium`, `high`, and `xhigh` for reasoning-capable OpenRouter models.

## Best Possible Solution

Land a provider-owned OpenRouter thinking profile, preferably with targeted provider tests proving DeepSeek V4 reasoning models expose `xhigh` while non-reasoning and explicitly unsupported proxy routes do not.

## Reproduction Assessment

Yes. Current-main code inspection gives a high-confidence reproduction path: `openrouter/deepseek/deepseek-v4-pro` can resolve as reasoning-capable, but OpenRouter has no thinking profile hook, so `/think xhigh` is rejected before the existing transport wrapper can send `reasoning.effort`.

## Solution Assessment

Yes. `resolveThinkingProfile` is the narrowest maintainable hook for this behavior because the shared docs and code make provider plugins responsible for model-specific thinking level sets; adding focused tests would make the solution safer.
