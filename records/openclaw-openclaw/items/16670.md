## Summary

Keep open. Current main still has memorySearch docs, doctor guidance, and `openclaw memory status --deep`, but setup, onboarding, and configure do not provide a dedicated memory embedding provider/model/API-key or SecretRef setup step. The requested mandatory behavior needs maintainer product and credential UX judgment before implementation.

## What This Changes

Request to make memory embedding provider, model, API-key, and readiness setup a mandatory onboarding step so semantic memory works across sessions.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that configures `agents.defaults.memorySearch`, supports API-key or SecretRef input, verifies readiness with a controlled embedding probe or `openclaw memory status --deep`, and updates setup/configure/onboarding/memory docs for API-key and daemon-environment behavior.
