## Summary

Keep this issue open. Current main still has memorySearch config, docs, doctor diagnostics, and `openclaw memory status --deep`, but the setup/onboarding and configure flows still do not offer a dedicated memory embedding setup step for provider, model, API key or SecretRef, or readiness verification. The prior April 26 keep-open review remains accurate against current main because the relevant files did not change between that review SHA and `b85edb3f0cf6c2a8bec06537af9211a044266a6c`.

## Best Possible Solution

Keep this open until a maintainer decides whether memory setup should be mandatory, prominent optional, or diagnostic-only. If approved, add a dedicated setup/configure memory path that prompts for `agents.defaults.memorySearch` provider, model, and API key or SecretRef, verifies readiness through an embedding probe or `openclaw memory status --deep`, and updates the public onboarding/configure/memory docs for API-key, Codex OAuth, and daemon environment behavior.
