## Summary

Keep open. Current main has memorySearch configuration docs, doctor guidance, and `openclaw memory status --deep`, but the setup/onboarding/configure flows still do not provide a dedicated memory embedding provider/model/API-key or SecretRef setup step. The request also asks whether that step should be mandatory, which needs maintainer product judgment before an automated fix PR is appropriate.

## What This Changes

Request to add a prominent mandatory onboarding step for configuring and verifying `agents.defaults.memorySearch` embedding provider, model, and API-key/SecretRef settings.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that configures `agents.defaults.memorySearch` provider/model and API-key or SecretRef input, verifies readiness with an embedding probe or `openclaw memory status --deep`, and updates the setup/configure/onboarding/memory docs for API-key, Codex OAuth, and daemon environment behavior.
