## Summary

Keep open. Current main has memorySearch configuration docs, doctor diagnostics, and `openclaw memory status --deep`, but the setup, onboarding, and configure flows still do not provide a dedicated memory embedding provider/model/API-key or SecretRef setup step. The remaining decision is product and credential UX policy: whether this should be mandatory, prominently optional, or diagnostic-only.

## What This Changes

Request to add a mandatory onboarding/setup step that configures `agents.defaults.memorySearch` embedding provider/model credentials and verifies embedding readiness.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that configures `agents.defaults.memorySearch` provider/model plus API-key or SecretRef input, verifies readiness through a controlled embedding probe or `openclaw memory status --deep`, and updates setup/configure/onboarding/memory docs for API-key, Codex OAuth, and daemon-environment behavior.
