## Summary

Keep this issue open. Current main has memorySearch docs, doctor diagnostics, and `openclaw memory status --deep`, but `openclaw setup`, setup wizard/onboarding, and `openclaw configure` still do not provide a dedicated memory embedding provider/model/API-key or SecretRef setup step. The request also asks whether this should be mandatory, which needs maintainer product judgment.

## What This Changes

Request to make CLI setup/onboarding prompt for `memorySearch` embedding provider, model, API-key or SecretRef configuration, and readiness verification as a mandatory setup step.

## Best Possible Solution

Keep this issue open for maintainer decision on whether memory embedding setup should be mandatory, prominently optional, or diagnostic-only. If approved, add a focused memory setup/configure path that writes `agents.defaults.memorySearch` provider/model and API-key or SecretRef config, verifies readiness with an embedding probe or `openclaw memory status --deep`, and updates the setup/onboarding/configure/memory docs for API-key, Codex OAuth, and daemon environment behavior.
