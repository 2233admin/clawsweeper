## Summary

Keep open. Current main still matches the reported failure path: raw model runs and llm-task disable tool creation, but inherited explicit allowlist sources are still collected and the guard still errors when no callable tools remain. The second reporter's curl/log reproduction corroborates the same root cause, and the related fix PRs have not landed.

## What This Changes

LLM-only gateway modelRun and llm-task embedded runs should ignore inherited tool allowlists and proceed as no-tool provider calls.

## Best Possible Solution

Land one narrow source fix that treats explicit no-tool runs as no-tool provider calls when inherited allowlist sources exist, while keeping the empty explicit-allowlist guard strict for ordinary tool-enabled runs. Add regression coverage for both gateway modelRun and llm-task with global tools.profile/tools.alsoAllow configuration.

## Reproduction Assessment

Yes. The source path reproduces at high confidence without a live provider: tools.alsoAllow can produce explicit sources, modelRun/disableTools empties callable tools, and the guard still returns the reported error when disableTools is true.

## Solution Assessment

Not applicable for a patch verdict because this is an issue, but the local dist patch described in the report is directionally correct. The maintainable fix should be made in source with focused tests, not as installed dist edits.
