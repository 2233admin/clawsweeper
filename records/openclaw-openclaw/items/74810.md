## Summary

Keep open. Current main still has the reported failure shape: no-tool model runs disable tool creation but still collect inherited explicit allowlist sources, and the guard currently turns that into the pre-provider “No callable tools remain” error. Related #74019 remains the plugin-owned Lobster/llm-task side of the same bug, while this issue tracks the generic gateway modelRun path.

## What This Changes

LLM-only gateway modelRun and disableTools runs should ignore inherited tool allowlists and proceed as no-tool model calls when global tool allowlist additions are configured.

## Best Possible Solution

Adjust the embedded runner/tool allowlist precheck so explicit no-tool runs (`modelRun`, `promptMode: "none"`, or `disableTools`) are treated as no-tool model calls even when inherited allowlist sources exist, while keeping the explicit allowlist guard strict for normal tool-enabled runs. Cover both gateway modelRun and llm-task with focused regression tests against global `tools.profile`/`tools.alsoAllow`.

Fix the no-tool model run allowlist bug reported in https://github.com/openclaw/openclaw/issues/74810 and related https://github.com/openclaw/openclaw/issues/74019.
