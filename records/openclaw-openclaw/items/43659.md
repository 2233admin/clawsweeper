## Summary

Keep this PR open. Current main still reuses any valid cached GitHub Copilot API token without tying it to the incoming GitHub credential, while this PR remains a focused implementation candidate for the same-author paired bug #43658.

## Best Possible Solution

Keep this PR open for maintainer review and land it, or an equivalent fix, after accepting the credential-fingerprint cache-key design. The fix should cover both the active GitHub Copilot plugin token helper and the legacy agent helper, and should keep regression tests for same-profile cache hits, cross-profile cache misses, and legacy unowned cache entries.

## Risks / Open Questions

- The proposed fix persists a stable truncated SHA-256 fingerprint derived from the GitHub PAT. It is not the raw token, but maintainers should explicitly accept that correlation/privacy tradeoff or choose a different cache-key design.
- The PR still uses a single shared cache file, so alternating healthy profiles will evict each other and refetch tokens. That is an efficiency follow-up, not a reason to close this correctness fix.
- Local tests were not run because this review was constrained to read-only inspection.
