## Summary

Keep open. Current `origin/main` at `213f92a9ef0eb54972ef6cbb0e8b900adc8699b0` still does not fully implement the requested post-compaction context display: shared status formatting avoids known-stale totals by showing `?`, but it does not fall back to the latest compaction checkpoint `tokensAfter`. The provided timeline also shows open PR #72285 already targets the narrow fix, so this issue should stay open until that PR, or an equivalent fix, lands.

## Best Possible Solution

Keep this issue open and review #72285. The narrow fix should make shared status formatting prefer a fresh `totalTokens` value, and when totals are stale fall back to the latest compaction checkpoint `tokensAfter`, with a regression test in `src/auto-reply/status.test.ts`.
