## Summary

Keep open. Current main has related `openclaw security audit` checks, but it does not implement this PR’s central behavior: startup conflict detection/logging for exec, sandbox, and elevated config. The PR also has unresolved functional and security-sensitive review findings that should be addressed before merge.

## Best Possible Solution

Keep this PR open if maintainers still want startup diagnostics, but require a refresh against current main: implement a non-throwing detector after effective runtime config is known, evaluate default and per-agent exec/elevated/sandbox combinations, reuse existing security-audit semantics where possible, correct the `exec.security` wording, and add regression tests for agent overrides, CLI/runtime bind overrides, SSH sandbox config, and the original #31036 user flow.
