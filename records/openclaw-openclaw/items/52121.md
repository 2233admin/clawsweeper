## Summary

Keep this PR open. Current main has related docs, security-audit checks, and a hard runtime refusal for unauthenticated non-loopback gateway binds, but it still does not implement the PR's intended startup diagnostics for conflicting exec/sandbox/elevated configuration. The PR also still needs a refresh/security review before it could land because the proposed detector is coupled to gateway startup and advisory checks must not create false positives, miss per-agent effective config, or turn warning-only inspection into a boot blocker.

## Best Possible Solution

Keep this PR open for maintainer review or a refresh. The best path is a non-throwing startup diagnostic that runs after effective runtime config is known, reuses the existing security-audit semantics where possible, evaluates default and per-agent exec/elevated/sandbox combinations, treats the already-enforced no-auth bind case consistently, fixes the `exec.security` guidance, and adds regression tests for per-agent overrides, CLI/runtime bind overrides, SSH sandbox config, and the original channel-session debugging flow.
