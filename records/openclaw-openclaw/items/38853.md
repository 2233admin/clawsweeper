## Summary

Keep open. Current main still lacks Discord-specific Cloudflare/Error 1015 HTML 429 classification, fallback cooldown, and non-JSON body sanitization in the REST startup/allowlist paths, and #73395 is an open unmerged PR with closing syntax for this issue.

## What This Changes

Request to detect Discord Cloudflare/Error 1015 HTML 429 REST failures, back off or cool down reconnect attempts, and avoid repeatedly logging full HTML error pages.

## Best Possible Solution

Keep this issue open and route maintainer attention to #73395. The fix should stay in the Discord plugin REST startup/allowlist surface: classify Cloudflare/Error 1015 and other non-JSON HTML HTTP 429 responses, summarize non-JSON bodies before logging or throwing, honor Retry-After when present, apply a conservative fallback cooldown when absent, and add focused tests for /users/@me/guilds and /oauth2/applications/@me. After that PR or an equivalent fix merges, close this as implemented with merge and release provenance.
