## Summary

Keep open. Current main still has Auto-TTS off by default and compaction does not directly flip `ttsAuto`, but the normal agent tool list still exposes the `tts` tool and the model-facing tool description still lacks an explicit-user-request guard, leaving a plausible path for unsolicited dashboard audio replies.

## What This Changes

The issue asks OpenClaw to keep dashboard replies as text after long-session compaction unless the user explicitly requests TTS/audio output.

## Best Possible Solution

Land a narrow hardening PR that updates the agent-facing `tts` tool guidance, with focused tests, so the assistant only calls `tts` when the user explicitly requests speech/audio/voice or an intentional TTS workflow requires conversion. Do not change compaction behavior, disable existing TTS commands, or alter configured Auto-TTS behavior.
