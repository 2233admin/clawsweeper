## Summary

Keep open. Current main still has auto-TTS off by default and compaction does not directly flip `ttsAuto`, but the agent-facing `tts` tool is always registered and its current description does not tell the model to reserve speech for explicit user requests. That leaves a narrow, plausible current-main path for unsolicited dashboard audio replies after a long/compacted prompt changes model behavior.

## Best Possible Solution

Keep this issue open and land a narrow hardening PR: update the default agent-facing TTS tool guidance, and any matching prompt/test surface if needed, so the assistant only calls `tts` when the user explicitly asks for speech/audio/voice or a deliberate TTS workflow requires conversion. Do not change compaction or disable existing auto-TTS and slash-command behavior.
