## Summary

Current main still does not implement a cron-scoped tool policy such as `tools.cron.tools.allow/deny`, and cron sessions still do not inherit `tools.subagents.tools`. The remaining behavior is per-job `payload.toolsAllow` plus delivery-mode handling for the message tool, so this feature request remains actionable.

## Best Possible Solution

Keep this issue open for a focused cron tool-policy design. A complete fix should add `tools.cron.tools.allow/deny` or explicitly implement cron inheritance from `tools.subagents.tools`, define precedence against existing policy layers and per-job `toolsAllow`, update schema/types/help/labels/generated config metadata and docs, and add regression tests for isolated cron message-tool denial across delivery modes.
