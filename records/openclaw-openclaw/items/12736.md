## Summary

Current main still lacks a cron-scoped tool policy such as `tools.cron.tools.allow/deny`, and cron isolated sessions still do not inherit `tools.subagents.tools`. The remaining enforcement is global/agent/group/sandbox/subagent policy plus per-job `payload.toolsAllow` and delivery-mode message-tool handling, so the feature request remains valid.

## Best Possible Solution

Keep this issue open for maintainer design. A complete fix should add a cron-scoped policy such as `tools.cron.tools.allow/deny`, or deliberately define cron inheritance from `tools.subagents.tools`, then update config schema/types/help/labels/generated metadata, docs, and regression tests proving isolated cron message-tool denial across announce/webhook/none delivery and per-job `toolsAllow` precedence.
