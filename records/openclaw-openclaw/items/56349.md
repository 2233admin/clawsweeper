## Summary

Keep this issue open. Current main has real partial progress: `deliverOutboundPayloads` runs `message_sending` before shared outbound delivery, Telegram has explicit delivery-hook coverage, and the generic buffered dispatcher can inject a `beforeDeliver` hook. But the requested invariant is stronger than those partial fixes, and current main still has channel paths that do not pass through a single unbypassable pre-send boundary. The issue discussion’s 2026.4.15 Telegram reproduction and the linked closed Slack PR are directionally useful; the Slack gap is still visible in current source.

## Best Possible Solution

Keep this issue open as the canonical outbound policy-boundary tracker. The best implementation path is to make one shared dispatcher or SDK-enforced reply delivery boundary mandatory for agent replies, message tool sends, streaming fallbacks, proactive/heartbeat sends, and channel adapters, then add contract tests that prove `message_sending` cancel/modify behavior fires for each supported outbound path. Focused channel PRs are useful, but this issue should close only after the invariant is enforced or superseded by a narrower open canonical design/implementation thread.
