## Summary

Keep open. Current main still rejects the reported partial `agents.defaults.cliBackends.<backendId>` override because `command` is still required in the Zod schema and TypeScript config type. The PR is relevant, but it should also update the generated config schema before it is merge-ready.

## Best Possible Solution

Keep the PR open and ask for a small follow-up: make `command` optional in `CliBackendSchema` and `CliBackendConfig`, keep a resolved-backend type or equivalent spawn-time guard for the non-empty command invariant, add the partial `modelAliases` override regression, and regenerate/check the base config schema with the repo's config schema generator so runtime validation and schema/UI consumers agree.
