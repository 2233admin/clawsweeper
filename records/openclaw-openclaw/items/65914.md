## Summary

Keep this PR open. Current main still has the QMD vector status timeout cap, while the PR branch removes it and keeps checkpoint handling centered on existing session artifact helpers; the only concrete issue I found is a non-blocking regression-test gap.

## What This Changes

The PR removes the 5000 ms cap from `probeVectorAvailability()`, adds QMD manager regression tests for status timeouts and checkpoint-named transcripts, and adds an unreleased changelog entry.

## Best Possible Solution

Land the focused QMD status-timeout fix after strengthening the regression test to use a timeout above 5000 ms, keep checkpoint filtering in the centralized session artifact helpers, and leave the separate CLI file-count inconsistency tracked by #69759.
