## Summary

Keep open. Current main still caps QMD vector status probes at 5000 ms, while this PR’s production change directly aligns that path with documented timeout tuning and the other QMD search call sites. The earlier broad checkpoint-filtering concern has been addressed, CI on the PR head is green, and the only remaining finding is a non-blocking regression-test weakness.

## What This Changes

The PR removes the 5000 ms cap from QMD vector status probes, adds QMD manager regression coverage for configured timeouts and checkpoint-named session exports, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the focused QMD status-timeout fix after the maintainer either accepts the non-blocking test gap or asks for the test to use a timeout above 5000 ms. Keep checkpoint snapshot filtering centralized in the session artifact helpers and leave the memory status file-count inconsistency tracked by #69759.
