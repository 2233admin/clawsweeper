## Summary

Keep open. Current main still has the reported sanitizer gap: `web_fetch` basic HTML cleanup strips exact hidden patterns such as `font-size:0` and transparent/zero-alpha colors, but it does not normalize colors, compare foreground/background colors, or remove tiny nonzero font sizes. Ryce's comment correctly points toward normalized same-fg-bg handling rather than a narrow named-color list.

## Best Possible Solution

Keep this open and land a focused sanitizer hardening change in `src/agents/tools/web-fetch-visibility.ts`: parse inline style declarations conservatively, normalize common CSS color forms, drop same foreground/background text when both values are explicit and opaque, and add a carefully chosen tiny-font threshold with tests in `src/agents/tools/web-fetch-visibility.test.ts`. Because this is user-facing security hardening, include targeted web_fetch visibility tests and a changelog entry if maintainers accept the change.
