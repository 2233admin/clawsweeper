## Summary

Keep open. Current main still has the reported web_fetch visibility sanitizer gap, and the existing discussion correctly frames this as security-sensitive prompt-injection hardening with false-positive policy choices rather than a stale cleanup candidate.

## What This Changes

The issue requests hardening the shared web_fetch HTML sanitizer against near-invisible CSS content, including near-white text, tiny nonzero font sizes, and matching foreground/background colors.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque colors, remove elements whose explicit foreground and background colors normalize to the same value, and apply a maintainer-approved tiny-font threshold. Cover both the basic and Readability paths with targeted tests, and add a changelog entry if maintainers classify the change as user-facing security hardening.
