## Summary

Keep open. Current main still has the reported `web_fetch` sanitizer gap: hidden-style handling is exact-pattern based and does not normalize opaque CSS colors, compare foreground/background colors, or reject tiny nonzero font sizes. Because this is prompt-injection hardening with false-positive tradeoffs, it needs maintainer/security policy review rather than stale cleanup.

## What This Changes

The issue requests hardening `web_fetch` HTML sanitization against near-invisible CSS such as white or near-white text, tiny nonzero font sizes, and matching foreground/background colors.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque color forms, remove elements whose explicit foreground and background colors normalize to the same value, and apply a maintainer-approved tiny-font policy. Cover `sanitizeHtml`, the basic extraction path, and the bundled Readability plugin path with targeted tests; add changelog or security notes if maintainers classify this as user-facing hardening.
