Current main and the latest release still use the confusing config restart sentinel summary. The config.patch path builds a config-patch restart sentinel, the formatter still renders `Gateway restart config-patch ok (config.patch)`, and the sentinel tests still assert the old wording shape, so this should not be closed as implemented.

Keep this open and make a focused UX fix in the restart sentinel/config-write notice path: render config.patch/config.apply restart-required notices with wording that does not imply the restart already completed, preserve the user-provided note on the following line, and add targeted formatter coverage.
