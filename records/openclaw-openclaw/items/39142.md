## Summary

Keep open. Current main still has no supported `messages.modelExhausted` or equivalent config key, the strict `messages` schema would reject that key, and the before-reply all-models-failed path still selects built-in billing, rate-limit, or generic sanitized text rather than an operator-configured exhaustion message. The issue is concrete and current; adjacent PR #70990 is open plugin-hook work and does not change default user-facing fallback messaging.

## What This Changes

Add a configurable user-facing message, proposed as `messages.modelExhausted`, for all-model fallback exhaustion caused only by billing and/or rate-limit failures.

## Best Possible Solution

Add a supported `messages.modelExhausted` string config key and use it only when a `FallbackSummaryError` has one or more attempts and every attempt reason is `billing` or `rate_limit`. Preserve existing built-in billing, rate-limit, overload, context, and generic failure text when unset or when other reasons appear; update config schema/help/generated docs, public docs, changelog, and focused tests.
