## Summary

Keep open. Current main still implements and documents the mixed heartbeat scoping rule that disables listed agents without explicit `heartbeat` blocks once any listed agent has one, while the doctor preview-warning path has no targeted diagnostic for that configuration. The broader inheritance semantics remain tracked in #41879, but this issue is a narrow, valid doctor warning request.

## What This Changes

Add an `openclaw doctor` diagnostic warning when `agents.defaults.heartbeat` is configured, at least one listed agent has an explicit heartbeat block, and other listed agents do not.

## Best Possible Solution

Add a narrow, non-mutating doctor config-health warning that detects mixed `agents.defaults.heartbeat` plus explicit per-agent heartbeat blocks and names listed agents without explicit heartbeat blocks as not scheduled under the current rule. Leave scheduler inheritance behavior and any product-policy change to #41879.

## Reproduction Assessment

Yes. The checked-in source and tests provide a high-confidence reproduction path: configure `agents.defaults.heartbeat`, list `main` without `heartbeat`, and list `ops` with `heartbeat`; current code reports `main` disabled and schedules only `ops`.

## Solution Assessment

Yes. A doctor warning is the narrowest maintainable fix for this issue because current docs and tests treat the mixed scoping rule as intentional; changing that rule belongs in #41879 instead.
