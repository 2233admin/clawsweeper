## Summary

Current main still has the reported heartbeat scoping behavior, and I did not find a doctor/startup warning for agents that lose inherited heartbeats when another agent has an explicit heartbeat block. The broader behavior discussion remains open in #41879, but that prior review explicitly leaves this narrower doctor-warning request separate.

## Best Possible Solution

Keep this issue open and add a narrow doctor/config-health warning. Detect `agents.defaults.heartbeat` plus at least one explicit `agents.list[].heartbeat`, then warn for listed agents without an explicit block and point users to either add an explicit heartbeat block or remove the per-agent scoping. Do not change heartbeat scheduling/inheritance from this issue; leave that broader policy to #41879.
