## Summary

Keep this issue open. Current main still treats valid Feishu native card JSON in the plain `message` parameter as text: the shared runner only parses structured `presentation`, `delivery`, and `interactive` fields, the Feishu action handler only builds cards from `presentation`, and the native payload path only reads structured `channelData.feishu.card`, `presentation`, or `interactive` data. The fresh 2026-04-29 reproduction and adjacent PR context support that #72667 fixed native payload sends but not serialized card JSON compatibility.

## What This Changes

The issue asks for Feishu `message(action=send)` to treat valid card JSON in the plain `message` parameter as an interactive card instead of sending it as text.

## Best Possible Solution

Add Feishu-local compatibility detection for valid native interactive-card JSON strings on `send` and `thread-reply`, sanitize through the existing native-card sanitizer, route valid cards to `sendCardFeishu`, and leave malformed card JSON or ordinary JSON as text. Keep the shared message schema provider-agnostic unless a generic plugin seam is proven necessary.
