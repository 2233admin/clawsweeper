## Summary

Keep open. Current main still treats cron failure alerts as opt-in unless `cron.failureAlert.enabled` is explicitly true or a per-job alert config exists, so this PR's intended default-enabled behavior is not implemented. The proposed diff is narrow and appears to address the review-thread concerns, but it still needs maintainer review for the user-visible default change and any docs/changelog follow-through.

## What This Changes

This PR changes cron failure-alert resolution so recurring jobs alert by default unless explicitly disabled, and adds a regression test for the unset `failureAlert` path.

## Best Possible Solution

If maintainers accept default-enabled failure alerts, land the narrow resolver change with the regression test while preserving `failureAlert: false`, per-job overrides, cooldown, skipped-run opt-in behavior, and primary delivery fallback; add docs or changelog coverage for the changed default if required by release policy.
