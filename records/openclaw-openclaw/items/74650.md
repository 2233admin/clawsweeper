## Repair Work Prompt

Fix the ACP/OpenClaw ACPX scope-approval loop described in https://github.com/openclaw/openclaw/issues/74650. Keep the change narrowly scoped to the `openclaw acp` GatewayClient path: for loopback Gateway URLs with resolved token or password shared auth, omit the persisted default device identity so a stale lower-scope pairing baseline cannot force an `operator.admin` scope-upgrade loop. Preserve current behavior for remote Gateway URLs, unauthenticated connections, explicit device-token-only flows, and ordinary clients. Consider an integration-style stale-baseline regression in `src/gateway/server.silent-scope-upgrade-reconnect.poc.test.ts` only if the startup unit test is not enough. Do not change the ACPX dependency pin, broad Gateway auth policy, or unrelated ACP runtime behavior.
