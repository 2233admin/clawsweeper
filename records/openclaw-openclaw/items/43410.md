## Summary

Keep this issue open. Current main still has representative bundled skills with capability-only descriptions, `skills/canvas/SKILL.md` still lacks YAML frontmatter/description metadata, and open PR #44329 is an unmerged implementation candidate for this issue. Skill descriptions are used in the model routing prompt, so this is a functional metadata cleanup rather than purely cosmetic wording.

## Best Possible Solution

Keep this issue open as the umbrella for bundled skill description quality. Maintainers should review and either land or close #44329 and #40453, then update the remaining affected bundled skill descriptions or split any leftovers into narrower follow-ups. The cleanup should also decide whether `skills/canvas/SKILL.md` should become a loadable skill with frontmatter and whether public skill-authoring docs should be aligned with the stronger bundled skill-creator guidance.
