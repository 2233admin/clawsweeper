Keep open. Current main and the latest release still preserve the reported Mattermost slash-command split-state failure mode, and #68089 remains the open implementation candidate while #69038 was closed unmerged.

## What This Changes

The issue asks to fix a Mattermost native slash-command regression where callbacks return HTTP 503 "not yet initialized" despite successful command activation in v2026.4.15 and later.

## Best Possible Solution

Keep this as the canonical Mattermost slash-command bug until one reviewed fix lands and ships. The end state should guarantee that route registration and monitor activation share slash-command account state across supported loader paths, with regression coverage for the initialized callback no longer returning the 503 placeholder.
