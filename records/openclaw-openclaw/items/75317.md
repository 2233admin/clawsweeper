## Summary

Keep this PR open. Current main still has the narrow memory embedding retry classifier, query embeddings still bypass the retry loop, and memory status does not flag scanned-vs-indexed coverage drift; the PR addresses a real memory-core reliability gap, but it needs the required changelog entry and maintainer coordination with the older related batch-splitting PR before merge.

## What This Changes

The PR broadens memory-core embedding retry classification, retries query embeddings through the existing retry loop, and adds memory status warnings when scanned builtin memory sources outnumber indexed files.

## Best Possible Solution

Land a coordinated memory-core fix that keeps this PR's broader retry classification, query retry path, and status coverage warning, adds the required changelog entry, and either incorporates or explicitly defers the batch split fallback tracked by #44167/#44166.
