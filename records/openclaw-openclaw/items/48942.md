Keep open. Current main still keeps `deriveSessionTitle` inside `src/gateway/session-utils.ts`, still derives titles as `displayName -> subject -> firstUserMessage -> sessionId`, and still lacks the dedicated `session-title.ts` module plus explicit `entry.label` precedence from this PR. The original heartbeat metadata overwrite appears handled elsewhere on main, but this PR's extraction/test consolidation and label/origin precedence change are not implemented and still need maintainer review because `origin.label` precedence overlaps the open TUI session-title work in #39722 and #39790.

## Best Possible Solution

Keep this PR open for maintainer review or revision. A conservative path is to either close it manually if the original heartbeat bug is considered fully handled by current system-event metadata preservation, or revise it into a narrow, rebased test/extraction PR that preserves explicit `entry.label` ahead of transcript-derived titles while leaving `origin.label` precedence to the open TUI session-title work in #39722/#39790.
