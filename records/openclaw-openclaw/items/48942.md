## Summary

Keep open. Current main still lacks the dedicated title module and explicit `entry.label` precedence, so this PR is not obsolete, but the latest patch should not merge as-is because the extracted helper regresses current defensive handling of malformed persisted session-store fields.

## What This Changes

This PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, re-exports it from `session-utils`, adds dedicated tests, and changes title precedence so explicit session labels outrank transcript-derived titles while origin labels remain a fallback after transcript text.

## Best Possible Solution

Keep the useful extraction and explicit `entry.label` precedence, but preserve the current defensive string normalization for all title candidates and keep `origin.label` behind transcript-derived titles unless #39722/#39790 deliberately change the broader TUI session-title policy.

## Reproduction Assessment

Yes. On current main, a `SessionEntry` with `label: "My App"` and `firstUserMessage: "heartbeat"` still returns the transcript-derived title because `deriveSessionTitle` ignores `entry.label`, and the TUI picker prefers `derivedTitle` over `displayName`.

## Solution Assessment

No as submitted. The latest precedence direction is narrow enough for the heartbeat/user-label bug, but the implementation should use `normalizeOptionalString` instead of direct `.trim()` calls so malformed persisted session-store values do not crash session listing.

## Review Findings

Overall correctness: patch is incorrect
