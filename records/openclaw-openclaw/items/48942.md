## Summary

Keep open. Current main still does not implement the PR's central behavior: `deriveSessionTitle` remains in `src/gateway/session-utils.ts`, checks `displayName`, then `subject`, then `firstUserMessage`, and never checks explicit `entry.label` or `entry.origin?.label`. The PR remains a plausible implementation candidate, but its submitted `origin.label` precedence still needs maintainer review because it can change TUI titles for unlabeled direct-message sessions.

## Best Possible Solution

Keep this PR open and request revision before merge: preserve explicit `entry.label` ahead of transcript-derived titles if maintainers want the heartbeat label fix, keep automatically populated `origin.label` behind `firstUserMessage` unless product behavior intentionally changes, retain the useful test/module consolidation if desired, and coordinate with #39722/#39790.
