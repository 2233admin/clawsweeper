## Summary

Keep this PR open. Current main has not implemented the extraction or dedicated session-title test module, and the requested label precedence is still absent from deriveSessionTitle. The PR also has a substantive review concern: putting auto-populated origin.label ahead of firstUserMessage could regress TUI titles for unlabeled inbound DMs, so it is not safely closeable as implemented or obsolete.

## Best Possible Solution

Keep the PR open but require revision before merge: preserve explicit entry.label ahead of transcript-derived titles if that is still desired, keep auto-populated origin.label behind firstUserMessage unless maintainers choose otherwise, decide whether the helper extraction is still worth the churn, and align the tests with the broader #39722/#39790 TUI title behavior.
