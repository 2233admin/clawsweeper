## Summary

Keep open: the PR addresses a real current-main session-title gap, and the latest update appears to keep auto-populated origin labels behind transcript-derived titles, but the patch still drops defensive normalization for persisted title fields and should not merge as-is.

## What This Changes

The PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, re-exports it from `session-utils`, moves the focused tests into a new module, and changes explicit session labels to outrank transcript-derived titles.

## Best Possible Solution

Keep the extraction, compatibility re-export, and explicit `entry.label` precedence; keep `origin.label` behind transcript-derived titles unless the broader #39722/#39790 policy changes; restore `normalizeOptionalString` handling for all persisted optional title candidates and add a malformed-store regression test.

## Reproduction Assessment

Yes. Current main can be reasoned from source to return the transcript title for an entry with `label: "My App"` and `firstUserMessage: "heartbeat"`; the PR regression is also direct from the diff because a malformed persisted value such as `displayName: 42` reaches `.trim()` and throws.

## Solution Assessment

No as submitted. The overall direction is narrow and maintainable, but the implementation needs to preserve current main’s defensive normalization before it is the best fix.
