## Summary

Keep open. Current main still has thinking effort controls, reasoning visibility controls, display/history sanitizers, provider replay cleanup, and a plugin hook workaround, but no schema-backed policy that lets reasoning run while discarding thinking blocks from durable session transcripts and transcript APIs.

## What This Changes

Request a silent reasoning mode that preserves model reasoning quality while excluding `thinking`/`redacted_thinking` content from persisted session transcripts and downstream transcript consumers.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and channel/user reasoning visibility. The end state should specify which user/history/memory transcript surfaces discard thinking blocks, where provider-required replay state is retained, and how `sessions.get`, ACP `loadSession`, compaction, context pruning, plugin hooks, UI controls, and docs observe that policy.
