## Summary

Keep this issue open. Current main still does not provide the requested first-class config/runtime surfaces, and the remaining work crosses config API design, sandbox/security policy, model/provider selection, gateway node enforcement, and multi-instance product boundaries.

## What This Changes

The issue requests first-class config schema and runtime support for root sandbox mode, tag-based routing rules, multi-instance declarations, and gateway path deny rules.

## Best Possible Solution

Keep the issue open and split it deliberately: decide whether root sandbox should exist or document agents.defaults.sandbox as the intended contract, design any tag-aware provider/model routing in the agent/model-selection path, decide whether multi-instance declarations belong in config or gateway/profile management, and handle path deny rules through a security-reviewed path policy with enforcement across the relevant tool and node chokepoints.
