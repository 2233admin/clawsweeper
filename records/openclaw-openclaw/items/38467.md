## Summary

Current main still does not implement the requested WhatsApp phone-contact directory. The bundled WhatsApp plugin exposes config-backed `allowFrom` peers, the Baileys socket setup has no `contacts.upsert` or `contacts.update` contact-list ingestion, and shared name-resolution plumbing cannot resolve saved-contact names because WhatsApp does not populate `name`/`handle` directory entries. Related PRs are closed unmerged and do not supersede the request.

## Best Possible Solution

Keep this issue open. Implement the feature in the bundled WhatsApp plugin: subscribe to Baileys `contacts.upsert` and likely `contacts.update` where the active socket is managed, persist account-scoped contacts locally behind an explicit privacy/config switch, expose contacts through `directory.listPeers` as `ChannelDirectoryEntry` rows with stable `id`, `name`, `handle`, and `raw` metadata, and route outbound name resolution through unique directory matches while preserving WhatsApp authorization checks. Add focused tests for ingestion, query filtering, ambiguity, account isolation, persistence/logout cleanup, and docs for `https://docs.openclaw.ai/cli/directory` and `https://docs.openclaw.ai/channels/whatsapp`.
