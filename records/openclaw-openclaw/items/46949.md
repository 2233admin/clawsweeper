## Summary

Keep this PR open. Current main still has the ACP behavior this PR is trying to change: startup identity reconciliation can leave a oneshot runtime handle cached, maxConcurrentSessions enforcement still rejects at capacity without pressure eviction, and the new regression tests from the PR are not present on main. The PR is a plausible core ACP lifecycle fix, but it needs maintainer review/rebase and the remaining eviction-safety comments addressed before landing.

## Best Possible Solution

Keep this PR open as a plausible core ACP lifecycle fix. Maintainers should rebase it on current main, preserve the focused oneshot startup-release and pressure-eviction regression tests, address the timeout and eviction revalidation review comments, then run targeted ACP control-plane tests plus the changed gate before merge.
