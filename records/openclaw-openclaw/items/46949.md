## Summary

Keep this PR open. Current main still lacks the ACP pressure-reclaim behavior, but the PR head is still unmerged and has blocking lifecycle correctness issues around stale status probes and unnecessary eviction under pressure.

## What This Changes

The PR changes the ACP session manager to release cached oneshot handles after startup identity reconciliation, reclaim non-running oneshot or stale idle runtimes at max-concurrency pressure, and add ACP control-plane regression tests.

## Best Possible Solution

Rebase or replace the PR on current main, keep the fix scoped to ACP runtime lifecycle accounting, add hard timeouts and capacity/idleness rechecks in guarded eviction paths, and keep focused coverage for startup oneshot release and pressure reclaim behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Add a hard timeout around status probes:** `src/acp/control-plane/manager.core.ts:1282`
  - body: This awaits cached.runtime.getStatus(...) directly and relies on the adapter honoring the AbortSignal. The runtime contract only makes signal optional, so an adapter that ignores it can leave session initialization stuck forever whenever capacity pressure triggers this probe. Race the status promise against a timer so max-concurrency enforcement always progresses.
  - confidence: 0.88
- **[P2] Skip stale probing after oneshot eviction frees capacity:** `src/acp/control-plane/manager.core.ts:1105`
  - body: After dormant oneshot eviction frees a slot, this still runs the stale-pressure path before checking the cache size. A single new session can therefore evict an additional idle or stale runtime even though pressure was already relieved, unnecessarily dropping session context.
  - confidence: 0.86
- **[P2] Recheck capacity before clearing stale candidates:** `src/acp/control-plane/manager.core.ts:1297`
  - body: The actor-queue callback can run after another session has already closed and brought the cache below the configured limit. Without a size check immediately before clear/close, this path can evict a usable idle session even though the new session no longer needs a slot.
  - confidence: 0.81
- **[P2] Recompute blind idleness inside the actor lock:** `src/acp/control-plane/manager.core.ts:1288-1289`
  - body: The no-getStatus fallback decides staleness from candidate.idleMs, which was captured before waiting for actorQueue.run. If another actor-scoped operation touched the runtime while this callback was queued, the PR can still evict it based on stale idle time.
  - confidence: 0.8
