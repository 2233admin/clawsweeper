## Summary

Current main restores chat history, but queued-message and unsent-draft persistence are still not implemented on main. Open PR #73154 remains unmerged and is the active implementation candidate for part of this issue, so policy says to keep this issue open until that PR is reviewed and either merged or closed. ([github.com](https://github.com/openclaw/openclaw/pull/73154))

## Best Possible Solution

Keep this issue open while maintainers review PR #73154. The best product path is a tested Control UI implementation that persists/restores queued messages across refresh, reconnect, and session switch; keeps history reload on the existing `chat.history` contract; and either implements draft persistence in the same fix or intentionally splits it into a clearly linked follow-up before closing this issue.
