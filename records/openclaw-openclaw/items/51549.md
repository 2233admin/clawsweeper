## Summary

Keep open. Current main restores visible chat history through the gateway `chat.history` contract, but the WebChat/Control UI queued-message state and draft text remain browser-memory state. The issue is also paired with open PR #73154 using closing syntax, so policy says the issue should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open while maintainers review PR #73154. The best path is a tested Control UI fix that persists and restores queued messages across refresh, reconnect, and session switch; keeps history reload on the existing `chat.history` contract; and either implements draft persistence in the same change or splits it into a clearly linked follow-up before closing this issue.
