## Summary

Current main still appears to have the reported duplicate wrapper-dispatch path, and the linked open PR directly targets it with a narrow guard and regression test. Per cleanup policy, the issue should stay open while that closing PR is still pending.

## What This Changes

Request to dedupe subagent browser-session lifecycle cleanup so embedded-mode parallel completion paths do not invoke cleanupBrowserSessionsForLifecycleEnd twice for one runId.

## Best Possible Solution

Review and either land or reject #68669. If landed, the fix should keep the browser extension's existing drain-idempotent IPC behavior unchanged while adding a per-run lifecycle dispatch marker, resetting it on rearm, and covering the parallel completeSubagentRun path with a focused unit test.
