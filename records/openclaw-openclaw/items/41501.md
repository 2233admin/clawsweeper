## Summary

Keep this PR open. The Langfuse-specific tracing half still belongs outside core as a plugin/ClawHub adapter, but the PR also contains a concrete cron/subagent completion delivery-confirmation fix that current main still does not implement. Current main can observe outbound delivery through diagnostics/OpenTelemetry, but the gateway agent result still does not return explicit external-delivery metadata to the subagent completion path.

## Best Possible Solution

Do not close this PR as implemented and do not merge it as-is. Split a narrow core PR that makes gateway agent delivery return explicit external-delivery confirmation and requires that confirmation for completion-mode cron/subagent direct announces, preserving silent duplicate/no-reply behavior. Move the Langfuse PoC to a ClawHub/plugin adapter using diagnostics/OpenTelemetry events and plugin hooks; propose a generic plugin API extension only if the adapter proves a missing seam.
