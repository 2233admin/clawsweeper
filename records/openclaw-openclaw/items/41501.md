## Summary

Keep this PR open, but do not merge it as-is. Current main has a plugin/OpenTelemetry-shaped answer for the Langfuse observability goal, while the core delivery-confirmation fix is still not implemented: gateway `agent` delivery drops outbound delivery results, and subagent completion direct delivery still marks payload-only gateway responses as delivered.

## Best Possible Solution

Do not merge this PR as-is. Split a small core PR that makes gateway `agent` delivery return explicit external-delivery confirmation from `deliverOutboundPayloads`, then require that confirmation for completion-mode subagent direct announces or fall back to direct send when it is absent. Add regression coverage for payload-only gateway responses and best-effort delivery failures. Move the Langfuse PoC to a ClawHub/plugin adapter using diagnostics events and plugin hooks; propose a generic plugin SDK seam only if that adapter proves one is missing.
