## Summary

Keep this PR open for maintainer follow-up. Current main still does not implement the cron/subagent completion delivery-confirmation fix: the Gateway agent delivery path awaits external delivery but discards the returned delivery results, and the subagent completion announce path still treats a visible gateway payload as delivered. The Langfuse portion is also not on main and remains an optional observability integration better routed through the existing plugin/OpenTelemetry surfaces rather than merged into core as-is.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as-is. Split a narrow core fix that exposes explicit external delivery confirmation from deliverOutboundPayloads through the Gateway agent command result and makes completion-mode cron/subagent direct announces require that confirmation before returning delivered=true, with regression coverage for payload-only gateway responses and fallback behavior. Route the Langfuse PoC to a ClawHub/plugin adapter on top of diagnostics-otel/plugin hook surfaces, proposing a small generic plugin API extension only if that adapter proves a missing seam.
