## Summary

Keep this PR open, but do not merge it as-is. Current main already has a plugin/OpenTelemetry-shaped answer for the Langfuse-style observability request and has a direct cron delivery path that only marks delivery after outbound results. The subagent completion delivery-confirmation bug is still real on main: the direct announce path can treat a gateway `agent` response with visible payloads but no external-delivery metadata as delivered.

## Best Possible Solution

Do not merge this PR as-is. Split a small core PR that makes gateway `agent` delivery return explicit external-delivery confirmation from `deliverOutboundPayloads`, then require that confirmation for subagent completion direct announces or fall back to direct send when it is absent. Keep the current cron direct-delivery path, but add regression coverage that distinguishes cron direct outbound delivery from subagent gateway-agent delivery. Move the Langfuse PoC to a ClawHub/plugin implementation using diagnostics events and plugin hooks; if that adapter exposes a real missing public API, propose a generic plugin SDK seam.
