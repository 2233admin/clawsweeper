## Summary

Keep this PR open. The PR body consolidates the provider-key leak from #56441, the two-phase approval state-machine fix from #56398, and the webchat approval UX/routing work from #56445; current main only partially overlaps the webchat/session-only follow-up behavior and still does not implement the central provider-env scrub, the no-route two-phase pending behavior, or the Control UI pending-copy change.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main and narrow it to the remaining behavior: scrub known provider auth env vars in the host exec sanitizer with regression coverage, decide and test the no-route two-phase approval semantics in the shared approval handler, add the Control UI/webchat pending-copy path if still desired, and split or fix the live-test HOME/profile staging changes raised in review.
