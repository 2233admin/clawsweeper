## Summary

Keep this PR open. Current main still does not support `OPENCLAW_LOG_DIR` or `OPENCLAW_TMP_DIR`: logging defaults resolve through the preferred temp dir and config `logging.file`, while the temp-dir resolver only prefers `/tmp/openclaw` or the OS fallback. The PR is a focused core infra change, not superseded by the related proxy-env reports in the provided context, and its final diff appears to address the earlier review comments about tmp-dir precedence and trust checks. It still needs maintainer review, regression tests, and docs alignment before landing.

## Best Possible Solution

Keep the PR open and review it as a plausible core infra fix. The best path is to add focused tests for `OPENCLAW_LOG_DIR` and `OPENCLAW_TMP_DIR`, confirm `.env` timing works with CLI startup, document the accepted env vars if maintainers want this surface, and then land or request changes based on that review.
