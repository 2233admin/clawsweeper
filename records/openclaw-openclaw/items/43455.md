## Summary

Current main still matches the reported Browserbase breakage. The docs still recommend a static Browserbase `cdpUrl`, while the browser plugin treats configured CDP URLs as direct endpoints and has no Browserbase session creation or returned `connectUrl` lifecycle support. Browserbase’s current docs describe creating a session first and connecting with the returned `connectUrl`. 

## Best Possible Solution

Keep this issue open. The best maintainer path is to update `docs/tools/browser.md` so it no longer recommends the obsolete static Browserbase root URL, then implement tested Browserbase session lifecycle support in `extensions/browser` or a vendor-neutral remote-CDP session-provider seam. Coverage should verify session creation, returned `connectUrl` use, failures/rate limits, secret redaction, and session release or termination behavior.
