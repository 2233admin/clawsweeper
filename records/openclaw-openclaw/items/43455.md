## Summary

Current main still has the reported Browserbase breakage: the public browser docs recommend the static Browserbase WebSocket URL and the browser runtime treats `cdpUrl` as a direct/discovered CDP endpoint, while Browserbase's current documented flow creates a session first and uses the returned `connectUrl`. This should stay open and is suitable for one focused browser plugin repair.

## What This Changes

The issue requests Browserbase session lifecycle support so OpenClaw creates a Browserbase session via the Sessions API, connects to the returned CDP URL, and updates the documented Browserbase configuration away from the static `connect.browserbase.com` WebSocket.

## Best Possible Solution

Add Browserbase session lifecycle support inside the browser plugin: accept a Browserbase API-key/project configuration, create a session through Browserbase's Sessions API, connect Playwright to the returned `connectUrl`, release or close the session on profile/browser stop, and update docs/tests so generic `cdpUrl` providers remain unchanged.
