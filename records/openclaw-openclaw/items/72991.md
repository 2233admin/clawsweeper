## Summary

Keep this issue open. Current main still has only the existing bundled, plugin, managed, and workspace hook tiers. It does not implement the requested admin-owned machine-wide policy tier, hardened POSIX/Windows discovery roots, disable immunity, policy-first execution, or policy-specific CLI inventory. The prior ClawSweeper keep-open comment remains accurate against current main.

## Best Possible Solution

Keep open for core maintainer design. A complete fix likely belongs across src/hooks/*, src/cli/hooks-cli.ts, config/schema docs, and tests: add a policy hook source with hardened POSIX/Windows discovery, load it before existing hook tiers, make it immune to hooks.internal.enabled=false and openclaw hooks disable, expose policy status/list/info clearly, and document the enterprise trust model.

## Risks / Open Questions

- The implementation is security-sensitive and needs maintainer design review for POSIX ownership checks, Windows HKLM policy handling, symlink/hardlink rejection, fail-open behavior, audit inventory, and CLI semantics.
- External hook packs or plugins can supply handler logic, but they cannot by themselves enforce an admin-owned, bypass-resistant discovery tier ahead of user-controlled configuration.
