## Summary

Keep open. Current main still lacks the requested machine-wide, admin-owned hook policy tier, and this is a core security/product design item because plugins or user-managed hooks cannot provide OS-owned roots, policy-first execution, user-disable immunity, or auditable policy inventory.

## What This Changes

Add an admin-managed machine-wide hook discovery and execution tier that always loads before user hooks, cannot be disabled by user config or `openclaw hooks disable`, and exposes policy-specific CLI inventory.

## Best Possible Solution

Define a core policy-hook tier with hardened POSIX and Windows discovery, deterministic policy-first dispatch, explicit disable immunity, fail-open error handling for misconfiguration, policy-aware CLI output, docs, and regression tests before accepting implementation.

## Reproduction Assessment

Not applicable as a feature request. The gap is high-confidence from current-main source and docs because the existing hook model, discovery, CLI, and docs only cover bundled, plugin, managed, and workspace hooks gated by user-controlled config.

## Solution Assessment

Unclear pending maintainer security design. If OpenClaw accepts this enterprise governance capability, core hook loading and dispatch are the maintainable layer because current plugin and hook-pack surfaces cannot enforce machine-owned roots, trusted ordering, or user-disable immunity.
