## Summary

Keep this issue open. Current main still lacks the requested shared atomic counter, `openclaw sessions kv incr` CLI/RPC surface, named `sessions_spawn` mutex/exclusive option, or agent-facing core lock primitive. The plugin SDK has `api.registerTool` and `openclaw/plugin-sdk/file-lock`, so a ClawHub/plugin route is feasible, but choosing that instead of a core coordination primitive remains a maintainer API/product decision.

## Best Possible Solution

Keep this open for maintainer product/API judgment. The clean core path would be a small generic workspace/session-scoped atomic state primitive such as `openclaw sessions kv incr <key> [by]`, with matching Gateway RPC/tool support, lock-safe storage, and concurrency tests for overlapping cron-spawned sub-agents sharing a daily counter. If maintainers prefer ClawHub, document the supported plugin route using `api.registerTool`, `openclaw/plugin-sdk/file-lock`, and JSON/state helpers before closing.
