## Summary

Current main still does not implement native trusted integrity or signature verification for SKILL.md skills. ClawHub native skill metadata and local lock/origin records carry version/time information, the downloaded archive digest is only locally observed, runtime skill loading reads SKILL.md into the prompt without verification state, and adjacent ClawHub plugin archive verification is explicitly separate from native skill installs. The prior maintainer review in the issue discussion also keeps this as canonical skill integrity work.

## Best Possible Solution

Keep this issue open as the canonical native skill integrity/signature work. A complete fix should add trusted digest/signature/publisher metadata to the ClawHub skill API, verify native skill archives before extraction, persist expected integrity/signature state in .clawhub origin and lock metadata, reverify SKILL.md or full skill contents during skill snapshot/load, expose verification state in status/UI/CLI surfaces, and define/test the policy for unsigned skills and capability restrictions.
