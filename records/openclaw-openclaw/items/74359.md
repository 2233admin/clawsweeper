## Summary

Keep open for maintainer review: current main still has the root Dockerfile remote Bun installer, and this active contributor PR is a plausible fix. Because it changes Docker supply-chain behavior, it should land only after maintainer/security review, CI, and Docker build proof.

## What This Changes

The PR changes the root Dockerfile to copy Bun 1.3.9 from a digest-pinned `oven/bun` source stage and updates the Docker image digest test to cover ARG-backed `FROM` stages.

## Best Possible Solution

Land a reviewed Docker hardening change that removes build-time execution of the remote Bun installer, keeps the Bun source image digest-pinned and aligned with CI, models Docker ARG scope correctly in the digest test helper, and closes the linked issue through the merge after CI and Docker build proof pass.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the reported issue: `Dockerfile:47` still executes the remote Bun installer with `curl ... | bash`.

## Solution Assessment

Yes. Copying Bun from a digest-pinned official image stage is the narrow maintainable direction for this Dockerfile; the remaining improvement is to tighten the new test helper so it reflects Docker ARG scope before merge.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.8

Full review comments:

- **[P3] Limit ARG default collection to global scope:** `src/docker-image-digests.test.ts:36-43`
  - body: Docker only makes ARG defaults declared before the first `FROM` available for later `FROM` substitution. This helper scans ARG instructions from every stage, so a future stage-local ARG could satisfy the digest assertion even though Docker would not resolve that `FROM` as written. Stop collecting defaults once the first `FROM` is reached, or model ARG scope explicitly.
  - confidence: 0.74
