## Summary

Keep this PR open. Current main still has the Dockerfile Bun `curl | bash` bootstrap, and the PR directly addresses the linked Docker supply-chain issue, but it touches security-sensitive build behavior and still leaves a mutable `oven/bun:1.3.9` tag plus a digest-test gap that should get maintainer/security review before merge.

## What This Changes

The PR changes the root Dockerfile to replace the Bun installer retry loop with an `OPENCLAW_BUN_IMAGE` stage from `oven/bun:1.3.9` and `COPY --from` of `/usr/local/bin/bun` into the build stage.

## Best Possible Solution

Land a Docker hardening change that removes `bun.sh` remote script execution from the root Dockerfile, uses a digest-pinned or otherwise verified Bun source consistent with OpenClaw's Docker image policy, updates digest coverage to check all resolved `FROM` stages, and validates the image build path before closing the linked issue.
