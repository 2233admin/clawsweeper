## Summary

Keep this PR open for maintainer review. Current main still installs Bun in the root Dockerfile by piping the remote installer into bash, while this PR removes that path, pins the Bun source image by digest, and expands the digest regression test; I did not find a blocking correctness or security finding in the proposed patch.

## What This Changes

The PR replaces the root Dockerfile's Bun curl-pipe installer with a digest-pinned `oven/bun` source stage and extends Docker image digest tests to cover all ARG-backed `FROM` stages.

## Best Possible Solution

Land this Docker hardening change, or an equivalent maintainer-owned patch, after CI completes and a root Docker build or install-smoke proves the pinned Bun source stage works; then let the merge close #74356 and handle the docs example separately if maintainers want the same policy there.
