## Summary

Keep this PR open. Current main still installs Bun in the root Docker build by piping the remote installer into bash, while the PR removes that path, pins the Bun source image by digest, and adds regression coverage for all ARG-backed Docker stages. I found no introduced blocking bug, but this is an active external-contributor Docker supply-chain hardening PR, so it needs normal maintainer review, settled CI, and Docker build proof rather than cleanup closure.

## What This Changes

This PR updates the root Dockerfile to copy Bun 1.3.9 from a digest-pinned oven/bun source stage and extends src/docker-image-digests.test.ts to validate every ARG-backed FROM stage.

## Best Possible Solution

Merge this Docker hardening change, or an equivalent maintainer-owned patch, after CI settles and a root Docker build proves the pinned Bun source stage works; then let the merge close #74356 and track any docs-example hardening separately if out of scope.
