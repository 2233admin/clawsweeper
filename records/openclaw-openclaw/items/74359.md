## Summary

Keep this PR open. Current main still installs Bun in the root Docker build by piping the remote bun.sh installer into bash, while this PR is a focused and relevant Docker supply-chain hardening change. The latest PR head addresses the review-noted digest gap, but it is still an external-contributor change to a security-sensitive Docker build path and should receive maintainer/security review plus real Docker image validation before merge.

## What This Changes

This PR changes the root Dockerfile to copy Bun from a digest-pinned `oven/bun:1.3.9` source stage instead of running the remote Bun installer, and extends `src/docker-image-digests.test.ts` to check all ARG-backed Dockerfile `FROM` stages.

## Best Possible Solution

Land a reviewed Docker hardening change that removes build-time execution of the remote Bun installer, keeps the Bun source digest-pinned and aligned with the CI Bun version, expands digest regression coverage for all relevant source stages, and validates the root Docker image path before closing #74356.
