## Summary

Keep this PR open. Current main still executes the Bun installer through `curl | bash` during the root Docker image build, so the PR is not obsolete and it targets a real supply-chain hardening gap. The remaining review point is security/product judgment: the proposed replacement removes remote script execution, but defaults to a mutable `oven/bun:1.3.9` image tag instead of the digest-pinned pattern already used for the Node base images.

## What This Changes

The PR modifies `Dockerfile` to add an `OPENCLAW_BUN_IMAGE` build argument, create a Bun source stage from `oven/bun:1.3.9`, and copy `/usr/local/bin/bun` into the build stage instead of running the `bun.sh` installer script.

## Best Possible Solution

Land a Docker build hardening change that removes `curl | bash` for Bun while preserving build compatibility and matching OpenClaw's Docker supply-chain policy, ideally by using a digest-pinned Bun image default or another verified Bun binary source and expanding Dockerfile tests so all resolved `FROM` stages are checked.
