## Summary

Keep this PR open. Current main still does not implement a configurable Control UI browser title: the config type/schema do not accept `gateway.controlUi.title`, the bootstrap payload does not expose a title, the UI loader never applies one, and the shipped HTML still has the fixed `OpenClaw Control` title. The supplied PR context shows this is a focused implementation candidate for #51052, but the branch needs refresh work, especially generated config schema/hints/docs coverage, before it is merge-ready.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refresh the branch against current main, add `gateway.controlUi.title` consistently to the config type, zod schema, generated base schema/UI hints/docs, bootstrap contract and gateway payload, and client bootstrap loader, then keep focused set/unset tests and resolve the generated-schema review concern before merge.
