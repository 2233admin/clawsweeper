## Summary

Keep this issue open. Current main still has the reported Telegram streaming guard, ordinary text messages can populate the quote-candidate map under non-off replyToMode, and an open unmerged PR (#73534) explicitly proposes the fix. Per policy, the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Review and land #73534 or an equivalent Telegram plugin fix that changes `hasNativeQuoteReply` to only block preview streaming for genuine selected quote replies, then update regression coverage so `replyToMode: "first"` plus `streaming.mode: "partial"` on an ordinary text message creates and updates the draft stream while selected quote replies remain covered.
