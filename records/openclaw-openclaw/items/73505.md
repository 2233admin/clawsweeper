## Summary

Keep open. Current main still matches the reported Telegram regression: non-off `replyToMode` fills the native quote candidate map for a plain incoming message, and the dispatcher still treats any populated map as a reason to disable answer draft preview streaming. Closed PR #73534 is useful source work for a narrow replacement fix; open PR #39905 is adjacent broader configuration work, not a landed fix for this bug.

## What This Changes

The issue requests that Telegram `streaming.mode: "partial"` continue to show live preview edits for plain replies when `channels.telegram.replyToMode` is non-`off`.

## Best Possible Solution

Land a narrow Telegram plugin fix that uses the real selected-quote signal, not the presence of any native quote candidate, to disable preview streaming. Preserve final-delivery quote candidate fallback and keep broader per-conversation streaming controls separate from this regression fix.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: `replyToMode: "first"`, `streamMode: "partial"`, and a plain text message with `MessageSid` or `message_id` populate `replyQuoteByMessageId`, make `hasNativeQuoteReply` true, and prevent the answer draft stream from being created.

## Solution Assessment

Yes. The narrowest maintainable solution is to disable answer preview streaming only for genuine selected Telegram quotes, while keeping `replyQuoteByMessageId` available for final delivery fallback and preserving the existing selected-quote non-streaming path.
