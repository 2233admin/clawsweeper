## Summary

Keep this issue open. Current main still retries each built-in compaction `generateSummary` call up to three times for any non-abort, non-timeout error, and staged compaction still advances across split chunks and then a merge pass by converting failed chunk summaries into fallback text. I did not find a compaction-specific circuit breaker or consecutive-failure budget on main; the linked implementation PRs remain closed and unmerged in the provided issue context.

## Best Possible Solution

Implement a bounded failure guard in the built-in compaction summarization path. Focused tests should cover multi-split `summarizeInStages()` under persistent non-timeout model failures, proving later split/merge LLM calls stop after a small consecutive-failure budget and return the previous summary or existing structured fallback while preserving aborts, timeouts, provider fallback, and quality-guard retries.
