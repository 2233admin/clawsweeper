## Summary

Keep open. Current main still lacks the requested low default temperature behavior for tool-enabled embedded-runner turns, so this is not implemented-on-main, but the branch is not merge-ready: the agent change suppresses the new default based on reasoning visibility, misses compaction tool runs, and carries unrelated stale secrets and lockfile changes.

## What This Changes

The PR adds dynamic default temperatures for embedded-runner tool turns, forwards normal-attempt tool names into extra params, expands runner tests, and also changes file-secret allowInsecurePath handling, lockfile importers, and changelog entries.

## Best Possible Solution

Land a focused agent-only change that preserves explicit user temperatures and plain-chat omission, bases the thinking opt-out on actual provider generation-thinking constraints, applies the tool-name heuristic consistently to normal and compaction tool runs, and drops unrelated secrets, lockfile, and stale changelog changes.
