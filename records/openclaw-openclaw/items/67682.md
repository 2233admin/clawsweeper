## Summary

Keep this open. Current main still has only server-scoped MCP env/header config plus internal OpenClaw loopback context headers; it does not expose a per-agent MCP credential overlay or runtime/cache path that would let the same external MCP server use different agent identities. Related MCP issues cover tool filtering, server scoping, or dynamic HTTP headers, but they do not fully supersede stdio env/process credential attribution.

## What This Changes

Add per-agent MCP credential overrides so different agents using the same MCP server can authenticate with separate env/header identities.

## Best Possible Solution

Define a typed per-agent MCP identity/credential contract that covers env-backed stdio servers and HTTP headers, merge it before embedded Pi and CLI backend MCP config creation, include the effective credentials in runtime fingerprints without logging secrets, and add tests proving two agents can use the same server id with different effective credentials.
