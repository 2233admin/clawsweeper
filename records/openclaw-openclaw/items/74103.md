## Summary

Keep open. Current main still has the reported ACP background-task summary behavior, and this PR is a useful narrow fix, but it should not merge as submitted because the new fallback can expose silent control tokens as user-visible terminal text.

## What This Changes

The PR changes ACP background-task handling to increase the progress summary cap from 240 to 1200 characters and return normalized progress text as the terminal summary for successful non-blocked completions.

## Best Possible Solution

Land a narrow ACP manager fix that keeps the useful fallback-and-cap direction, strips exact or trailing silent control tokens before terminal delivery, preserves blocked-outcome detection, and adds focused regression tests for ordinary successful progress and control-token cleanup.

## Reproduction Assessment

Yes. The source path is high-confidence: a parent-owned ACP prompt run with non-empty, non-blocked `text_delta` progress reaches `resolveBackgroundTaskTerminalResult()`, current main returns `{}`, and terminal delivery uses only `terminalSummary`, producing the bare completion message.

## Solution Assessment

No, not as submitted. Returning progress as terminalSummary is the right narrow direction, but the fallback should clean silent control tokens and be covered by ACP manager tests before merge.

## Review Findings
