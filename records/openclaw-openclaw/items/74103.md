## Summary

Keep this PR open as a focused implementation candidate for #74070, but do not merge it as-is. Current main and the latest release still have the reported ACP background-task behavior, while the PR’s fallback can surface existing silent-control tokens as visible terminal text.

## What This Changes

The PR raises the ACP background-task progress accumulation cap from 240 to 1200 characters and returns normalized progressSummary as terminalSummary for non-blocked successful ACP background completions.

## Best Possible Solution

Land a narrow ACP manager fix that copies cleaned non-blocked progress into terminalSummary, preserves blocked-outcome detection, strips exact or trailing silent-control tokens, raises the cap if maintainers accept the 1200-character payload, and adds focused regression tests before closing #74070.
