## Summary

This PR should stay open because it addresses a real current-main ACP background-task completion bug, but the submitted patch still misses the silent-control-token cleanup requested by the linked report and needed to avoid visible token leaks.

## What This Changes

The PR changes `src/acp/control-plane/manager.core.ts` to raise the ACP background-task progress cap from 240 to 1200 characters and return normalized non-blocked progress as `terminalSummary`.

## Best Possible Solution

Land a narrow ACP manager fix that copies cleaned non-blocked progress into `terminalSummary`, preserves blocked-outcome detection, strips exact or trailing silent-control tokens before terminal delivery, and adds focused ACP regression tests for the success fallback and cap behavior.

Yes. The source-level path is high confidence: non-empty, non-blocked ACP progress reaches `resolveBackgroundTaskTerminalResult()`, current main returns `{}`, the success path stores `terminalSummary` as null, and the task formatter only displays `terminalSummary`.

## Solution Assessment
