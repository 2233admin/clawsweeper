## Summary

Current main still lacks a user-configurable Control UI/WebChat font-size or font-scale setting. Browser-local UI settings, Appearance, Quick Settings, gateway `ui` config/schema/types, CSS variables, and public docs all cover theme/roundness/identity but not text sizing. The closed #3027 does not supersede this remaining accessibility request, and the prior Codex review in the discussion reached the same keep-open result.

## Best Possible Solution

Implement a bounded browser-local Control UI text-size/font-scale setting first: persist it in `UiSettings`, expose it in Appearance and Quick Settings, apply it via CSS custom properties across chat/composer/sidebar/tool-card surfaces, add focused storage/rendering/settings tests, and document it on the public Control UI docs page. Extend the gateway `ui` schema/types/docs only if maintainers explicitly want server-side font sizing as a public config contract.
