## Summary

Keep this PR open. Current main still uses raw `contextWindowTokens - reserveTokensFloor - softThresholdTokens` threshold math in both the shared memory-flush gate and the runtime preflight/flush paths, so the linked `reserveTokensFloor >= contextWindow` bug is not implemented on main. The latest discussion also says a maintainer pushed a narrow repair to this branch to keep the contributor path canonical.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Maintainers should review and land this branch, or an equivalent fix, that centralizes memory-flush threshold calculation and makes both preflight compaction and memory flush preserve a positive context-aware gate, with regression coverage for oversized reserve floors and small context windows.
