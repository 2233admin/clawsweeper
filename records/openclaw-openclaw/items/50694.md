## Summary

Keep this PR open. Current main still uses raw memory-flush threshold math in the shared gate and both runtime paths, so the linked reserveTokensFloor >= contextWindow bug is not implemented on main. The provided discussion also says a maintainer pushed a narrow repair to this branch and the related #51316 attempt is closed, making this the active candidate path.

## Best Possible Solution

Review and land this branch, or an equivalent fix, that centralizes memory-flush threshold resolution and uses it consistently in shouldRunMemoryFlush, shouldRunPreflightCompaction, preflight diagnostics, and runMemoryFlushIfNeeded. The fix should preserve a positive context-aware gate for oversized reserve floors and small context windows, with regression tests for reserveTokensFloor equal to and greater than contextWindowTokens.
