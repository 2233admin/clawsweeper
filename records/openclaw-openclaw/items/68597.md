## Summary

Keep this PR open. Current main still appears to have the reported `memory_get` boundary gap: builtin and QMD reads authorize by lexical path plus direct-file `lstat`, without canonical containment of the resolved target under the allowed memory root. The PR is relevant security hardening, but it needs maintainer/security review and likely rebase/adaptation because current main moved the active host SDK implementation into `packages/memory-host-sdk`.

## What This Changes

The PR adds canonical realpath containment for builtin and QMD `memory_get` reads, exports the helper, and adds regression tests for intermediate directory symlink escapes plus transient missing-file and symlink-permission edge cases.

## Best Possible Solution

Land a rebased version that adds one shared canonical contained-file resolver in the active memory host SDK package, uses it from both builtin and QMD `memory_get` read paths, keeps missing-file races non-fatal, and preserves focused regression coverage for workspace memory, extra paths, and QMD collection directory-symlink escapes.
