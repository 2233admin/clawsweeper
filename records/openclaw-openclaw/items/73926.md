## Summary

Keep open. Current main has meaningful partial fixes for bare reset commands and blank visible prompt submissions, but the embedded runner still has an intentional runtime-only path that can call the provider with an empty visible prompt, and two open PRs still claim to close this issue; conservative cleanup should wait for maintainer reconciliation.

## What This Changes

Prevent `/new` or `/reset` plus active-memory/runtime-context races from submitting empty prompt/messages payloads to Anthropic/OpenAI-compatible providers and leaving sessions stuck processing.

## Best Possible Solution

Keep the issue open until maintainers either land or explicitly supersede the linked fix PRs, then close it only after the embedded-runner boundary has one clear rule: submit only when there is a non-blank visible prompt, prompt images, or provider-replayable non-empty history, while preserving the no-model bare `/new` acknowledgement path.
