## Summary

Keep open. Current main still lacks the Codex ACP auth bridge, so this PR is a relevant implementation lane for #73910, but it should not merge while the fallback copy path can place Codex credentials inside OpenClaw state backups and the branch is dirty against current main.

## What This Changes

The PR bridges an existing Codex `auth.json` into the ACPX-managed isolated Codex home, updates ACP agent docs and the changelog, adds ACPX regression coverage, and adjusts one Codex plugin test type.

## Best Possible Solution

Keep the auth bridge direction, but revise it so Codex auth is either symlink-only or stored/excluded in a credential-safe location, with backup coverage proving `acpx/codex-home/auth.json` is not archived. After that, rebase onto current main and land with the focused ACPX auth bridge tests plus changed-gate proof.

## Reproduction Assessment

Yes. Current main can be inspected or exercised by preparing ACPX with `CODEX_HOME` containing `auth.json`: the wrapper forces an isolated `codex-home`, `prepareIsolatedCodexHome` writes only generated config, and existing tests still assert no source auth bridge.

## Solution Assessment

No. Bridging only `auth.json` while preserving generated config isolation is the right narrow direction, but the current copy fallback is not the best solution because it persists credential material under the recursively backed-up state directory.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid copying Codex auth into backed-up state:** `extensions/acpx/src/codex-auth-bridge.ts:281`
  - body: When symlinking fails, this fallback writes the host Codex `auth.json` into `<stateDir>/acpx/codex-home`. `openclaw backup create` archives the state directory recursively and only filters extension `node_modules`, so this can package Codex tokens into routine backup archives. Keep this symlink-only, move the file to a credential-safe location, or add backup exclusion/redaction coverage before merging.
