## Summary

Keep open: the PR addresses a real current-main ACPX/Codex auth gap, but the current head still has blocking credential-handling and test-stability issues. The auth bridge direction is useful, but this patch should not merge while copied Codex auth can enter normal OpenClaw backups.

## What This Changes

The PR bridges an existing Codex `auth.json` into ACPX's generated isolated Codex home, documents that behavior, adds ACPX regression coverage, updates the changelog, and adjusts a Codex plugin test type.

## Best Possible Solution

Keep the auth bridge direction, but revise it so Codex auth is never archived by default: prefer a symlink-only bridge, move copied credentials to a credential-safe location, or add explicit backup exclusion/redaction with regression coverage. Also isolate the no-auth test from both `CODEX_HOME` and the default home before re-running the focused ACPX/Codex tests and changed gate.

## Reproduction Assessment

Yes. The original bug is reproducible from current main by preparing ACPX with a host Codex `auth.json`: OpenClaw forces an isolated Codex home that only receives generated config. The PR's security regression is reproducible when symlink creation fails, because the fallback writes `auth.json` into state and the backup path archives state recursively.

## Solution Assessment

No. Bridging only Codex `auth.json` while keeping Codex config hooks isolated is the narrow maintainable direction, but copying credentials into backed-up state and leaving an environment-sensitive test make this implementation unsafe as-is.

## Review Findings

Full review comments:

- **[P1] Avoid copying Codex auth into backed-up state:** `extensions/acpx/src/codex-auth-bridge.ts:281`
  - body: When symlink creation fails, this fallback writes the host Codex `auth.json` into `<stateDir>/acpx/codex-home`. `openclaw backup create` archives the state directory recursively and only filters extension `node_modules`, so routine backups can include Codex tokens. Keep the bridge symlink-only, move copied auth to credential-managed storage, or add backup exclusion/redaction coverage before merging.
  - confidence: 0.93
- **[P2] Isolate no-auth test from the default Codex home:** `extensions/acpx/src/codex-auth-bridge.test.ts:95`
  - body: Deleting `CODEX_HOME` still makes `sourceCodexAuthPath()` fall back to `os.homedir()/.codex/auth.json`. On machines where Codex is logged in normally, `prepareAcpxCodexAuthConfig` will bridge that file and the later `ENOENT` assertion fails. Point HOME to the temp root or otherwise stub the default Codex home for this no-auth case.
  - confidence: 0.88
