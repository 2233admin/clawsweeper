## Summary

Keep this PR open. Current main still has the Codex ACP auth gap, and the PR is aimed at the right bug, but the current head still introduces a concrete credential-backup exposure through the copy fallback path, so it is not ready to merge.

## What This Changes

The PR bridges an existing Codex `auth.json` into ACPX's isolated Codex home, updates ACP agent docs and the changelog, adds ACPX regression coverage, and adjusts one Codex plugin test type.

## Best Possible Solution

Keep the auth bridge direction, but make the credential handling backup-safe before merge: either use a symlink-only bridge, move copied credentials to an existing credential-safe location, or add explicit backup exclusion/redaction with regression coverage proving `acpx/codex-home/auth.json` is not archived.

## Reproduction Assessment

Yes. Current main can be checked by preparing ACPX with `CODEX_HOME` containing `auth.json`: the wrapper forces an isolated Codex home and writes only generated config. The PR finding is reproducible by forcing `fs.symlink` to fail, then observing the copied auth path falls under the recursively archived state directory.

## Solution Assessment

No. Bridging only `auth.json` while keeping Codex config hooks isolated is the narrow maintainable direction, but the copy fallback into OpenClaw state is not the best credential-handling policy unless backups explicitly exclude or redact it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
