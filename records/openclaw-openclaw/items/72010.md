## Summary

Keep open. Current main still does not implement this PR's intended diagnostic progress semantics: stuck-session checks are still age-based, there is no progress marker or repeated-warning backoff state, ACP runtime events and generic reply callbacks do not refresh diagnostic session activity, and docs/config help still describe the old threshold behavior. The closed #71969 is superseded by this active PR, not by main.

## Best Possible Solution

Keep this PR open for normal maintainer review and landing, or land an equivalent patch that adds diagnostic session progress marking, repeated stuck-warning backoff, ACP/runtime progress wiring gated by `diagnostics.enabled`, excludes typing keepalives as progress, and updates docs/generated config surfaces. #71969 should remain treated as the closed predecessor that this active PR replaces.
