## Summary

Keep open. Current main still lacks the PR’s requested no-progress stuck-session tracking/backoff, and the PR remains useful, but the latest head is dirty against current main and still misses several real generic dispatch progress callbacks.

## What This Changes

The PR adds diagnostic session progress/backoff state, wires ACP and selected generic reply progress callbacks to refresh stuck-session diagnostics, adjusts read-only channel plugin loader typing, updates diagnostics config docs/generated text, and updates related tests.

## Best Possible Solution

Rebase or repair this PR so it preserves current-main stuck-session recovery, wraps every non-typing real progress callback consistently in generic dispatch, keeps ACP progress gated by diagnostics.enabled, and lands with aligned tests, generated config docs, and a changelog entry.

## Reproduction Assessment

Yes. A focused resolver that calls only an omitted callback such as onCommandOutput, onItemEvent, or onPartialReply while the session is processing can show that lastActivity is not refreshed; the existing fake-timer diagnostic test pattern can then observe session.stuck after the threshold.

## Solution Assessment

No. The PR is solving the right problem at the right layer, but the generic dispatch implementation is incomplete because only a subset of real progress callbacks refreshes diagnostics; the safer fix is one shared wrapper pattern for all non-typing progress callbacks while leaving onReplyStart/typing keepalives excluded.
