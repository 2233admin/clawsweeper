Keep this PR open. Current main still renders the Control UI Sessions Updated column as a relative timestamp, so the PR addresses a real, still-open feature request, but the branch is dirty against main and its rebased form needs to preserve the localized missing-timestamp fallback before merge.

## What This Changes

The PR adds a `formatFullDateTime` UI formatter with tests and changes the Control UI Sessions Updated cell to show `YYYY-MM-DD HH:mm` while keeping the relative timestamp in a tooltip.

## Best Possible Solution

Rebase the implementation onto current main, keep the change limited to a reusable UI date-time formatter plus the Sessions Updated cell, pass the localized missing-value fallback through the new formatter path, retain the relative timestamp as the tooltip, and verify with focused formatter/Sessions tests plus a quick Control UI browser check.
