## Summary

Keep this PR open. Current main still renders the Control UI Sessions `Updated` column as relative text and has no full `YYYY-MM-DD HH:mm` formatter, so the PR addresses a real open feature request, but the branch is not merge-ready because its missing-timestamp path would regress the existing localized fallback.

## What This Changes

The PR adds a `formatFullDateTime` UI helper with tests and changes the Control UI Sessions `Updated` cell to show a local `YYYY-MM-DD HH:mm` timestamp while preserving the relative timestamp as the cell tooltip.

## Best Possible Solution

Rebase the contributor branch onto current main, keep the change limited to a reusable UI date-time formatter plus the Sessions `Updated` cell, pass the localized `t("common.na")` fallback for missing timestamps, retain relative time in the tooltip, and verify with focused formatter/Sessions tests plus a quick browser spot-check.
