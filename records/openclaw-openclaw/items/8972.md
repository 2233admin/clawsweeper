## Summary

Keep this issue open. Current main still treats Discord guild channel allowlist entries as exact channel id/name/slug keys, thread-parent fallbacks, or the literal `*` fallback. A configured key like `ws-*` is stored as a normal record key and is not evaluated as a glob. The category alternative is still tracked separately by open #60582, and the related additive-overrides semantics are open in #73387.

## Best Possible Solution

Keep this issue open and implement a documented Discord guild channel pattern form such as `ws-*` in the Discord-owned allowlist path. Preserve exact id/name/slug and thread-parent precedence, keep literal `*` as the broad fallback, define pattern ordering for multiple matches, update the exported types/Zod schema/docs/status/audit behavior, and add regression tests proving dynamically created matching Discord channels are accepted without per-channel config edits. Track category inheritance through #60582 and additive override semantics through #73387 unless maintainers decide to merge those designs.
