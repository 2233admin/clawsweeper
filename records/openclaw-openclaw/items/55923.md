## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: text-form tool-call artifacts are stripped from visible assistant text rather than surfaced as explicit errors, placeholder-only replies still flow through as ordinary assistant replies, WebChat `error` events still update hidden state only, and legacy `/ws?agent=...&token=...` clients still reach the challenge/timeout path. The PR remains the active contributor path, with bot review findings already addressed and a maintainer repair noted in the discussion.

## Best Possible Solution

Review and land this PR, or deliberately split only the WebChat overlap with #40277 while keeping the agent payload downgrade/placeholder detection and legacy WebSocket query-auth rejection together with their targeted tests. The current main branch does not yet solve the central behavior this PR addresses.
