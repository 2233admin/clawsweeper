The PR remains the active broad implementation candidate; #40277 overlaps only the WebChat subset.

## Best Possible Solution

Review and land this PR after resolving any current merge state, or deliberately split only the WebChat overlap with #40277 while keeping the agent payload downgrade/placeholder detection and legacy WebSocket query-auth rejection tracked together unless maintainers choose a new canonical PR.
