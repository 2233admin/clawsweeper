## Summary

Keep this PR open. Current main still lacks the PR's intended behavior: downgraded text-form tool calls are stripped rather than surfaced as explicit errors, placeholder-only replies are still treated as normal assistant text, WebChat error events only update hidden `lastError`, and legacy `/ws?agent=...&token=...` clients still go through challenge/timeout handling. The PR discussion also shows this branch remains the active contributor path, with earlier review findings addressed and a maintainer repair pushed.

## Best Possible Solution

Review and land this PR if current CI/Testbox proof remains green, or split only the WebChat overlap if maintainers prefer #40277 for that subset. The remaining agent payload and gateway WebSocket fixes are not implemented on main, and closing this PR now would drop the current canonical contributor path.
