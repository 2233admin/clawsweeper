## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: downgraded text-form tool calls are stripped from visible text rather than surfaced as explicit errors, placeholder-only replies are treated as ordinary assistant replies, WebChat error events only update hidden state, and legacy `/ws?agent=...&token=...` clients still enter the normal challenge/timeout path. The PR discussion also shows active maintainer/contributor follow-up, including fixes for review findings and a maintainer repair commit preserving this PR as the canonical path.

## Best Possible Solution

Review and land this PR if maintainers accept the heuristic tradeoffs, or split only if they want to merge the WebChat-only overlap from #40277 separately. The remaining main-branch gap belongs in the touched agent payload, WebChat controller, and gateway WebSocket connection paths; closing this PR now would drop behavior that is still not implemented on main.
