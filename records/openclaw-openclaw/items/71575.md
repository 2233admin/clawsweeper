## Summary

Keep open. Current main still routes Telegram tool/progress events into a visible `Working…` preview by default during partial streaming, and it still leaves default tool-progress delivery unsuppressed when preview streaming is off. The recent reproduction comment and open PR #72482 are useful context, but the PR is not merged and only addresses the non-streaming path, while the issue’s requested final-only Telegram behavior remains unresolved.

## Best Possible Solution

Keep this issue open for a Telegram-owned behavior decision and code change. Review and land #72482 or an equivalent non-streaming fix if correct, then address the partial-streaming path in `extensions/telegram/src/bot-message-dispatch.ts` so routine tool/progress lines are hidden by default or the product explicitly decides to keep them. Update the Telegram dispatch tests plus `docs/channels/telegram.md` and `docs/concepts/streaming.md` to match the chosen behavior.
