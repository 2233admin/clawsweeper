## Summary

Keep open. Current main and the latest release snapshot still default Telegram preview streaming to visible tool-progress updates, and the focused tests/docs explicitly preserve `Working…` plus tool/output preview lines. The report is valid but changing the default is a privacy-sensitive product/security decision, not a conservative cleanup close.

## What This Changes

The issue asks Telegram-bound sessions to hide intermediate tool-call status/output and deliver only the final assistant reply to chat users.

## Best Possible Solution

Maintainers should choose the Telegram privacy default for partial streaming. If final-only or tool-hidden delivery should be the default, update the Telegram preview-progress resolver/dispatch behavior, keep explicit opt-in support, preserve approval/media/error routing, and align focused tests, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by dispatching a Telegram turn with `streamMode: "partial"` and invoking `onToolStart` plus `onItemEvent`; the current test asserts the visible `Working…` draft update, and the discussion includes a 2026.4.25 DM/group reproduction.

## Solution Assessment

Unclear. The current implementation is documented and has opt-outs, but it does not solve the requested final-only default; flipping the default is technically narrow but needs maintainer product/security approval because current tests and docs intentionally preserve visible progress previews.
