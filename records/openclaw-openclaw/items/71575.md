## Summary

Keep open. Current main still deliberately sends visible Telegram `Working…` tool-progress preview edits when partial preview streaming is active; only explicit opt-outs or final-only streaming suppress them. The related #72482/#73994 work covers narrower non-streaming or failed-tool-warning paths and does not supersede the reported default partial-streaming behavior.

## What This Changes

Telegram should hide intermediate tool-call progress and raw tool output so chat users receive only the final assistant reply.

## Best Possible Solution

Settle the Telegram partial-streaming policy. If tool progress should be hidden by default, change the Telegram dispatch/config default for preview tool progress, preserve approval/media/error delivery, and update the Telegram tests plus public streaming docs; otherwise close only after documenting the current behavior as intentional with an explicit security rationale.
