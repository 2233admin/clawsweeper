## Summary

Keep open. Current main still renders Control UI assistant media with browser-visible `token=` URLs, while this PR moves rendered media URLs to short-lived scoped tickets and appears directionally correct. The latest head is mergeable but still has an unstable check state with a failing CI check, and the linked maintainer-scoped auth follow-up needs normal maintainer review before merge.

## What This Changes

The branch adds short-lived HMAC-scoped assistant media tickets in the Gateway, updates Control UI local-media rendering to use Authorization metadata probes plus `mediaTicket=` URLs, adds gateway/UI tests, and records the user-facing fix in the changelog.

## Best Possible Solution

Land the scoped-ticket flow after the current CI failure is triaged and maintainers accept the temporary legacy query-token compatibility boundary, then let the linked follow-up close through the merge.

## Reproduction Assessment

Yes. Current main reproduces the reported leak by rendering a local assistant attachment with `assistantAttachmentAuthToken`, which produces an assistant-media URL containing `token=`; the current Gateway route also accepts query-token media auth.

## Solution Assessment

Yes, with merge-readiness caveats. Short-lived source-scoped HMAC tickets are the narrowest maintainable way to preserve native browser media rendering without embedding long-lived credentials in rendered URLs; cookie auth or blob URLs would be broader changes.
