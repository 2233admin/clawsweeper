## Summary

Keep open. Current main still runs the Google Gemini CLI backend with `--skip-trust` plus JSON `--prompt`, but no `--approval-mode` or `--yolo`, while upstream Gemini CLI behavior makes the reported missing write/edit tools plausible in noninteractive mode. The linked fix PR was closed unmerged, and the separate OAuth account-risk concern is already covered by current docs and setup warnings.

## What This Changes

Ensure the `google-gemini-cli` provider can edit workspace files such as `IDENTITY.md` and `SOUL.md` when OpenClaw runs Gemini CLI noninteractively via OAuth on Linux.

## Best Possible Solution

Define the intended Gemini CLI headless approval policy in the Google plugin, then update fresh and resume backend args, docs, and regression coverage together so workspace edits work without bypassing OpenClaw's intended tool-policy posture.
