## Summary

Keep open. Current main only partially covers the PR intent: fallback retry prompt preservation and machine-clean agent JSON output are already handled, but explicit session-id rollover cleanup and one-shot sandbox browser-bridge teardown are not fully implemented. The PR branch also has review-confirmed TypeScript blockers and an unsafe teardown approach, so it should not be merged unmodified or closed as implemented.

## What This Changes

The PR changes agent fallback retry prompt handling, session persistence and rollover cleanup, `openclaw agent --json` output routing, sandbox browser-bridge disposal, and related tests across command, delivery, session, sandbox, and embedded-attempt paths.

## Best Possible Solution

Keep the PR open for maintainer follow-up, but require a rebase or replacement that drops already-covered retry/JSON changes, implements explicit-session rollover cleanup across all session-scoped continuity fields, and handles sandbox browser-bridge teardown with ownership, refcounting, or one-shot/private-scope gating plus overlap regression coverage.

## Reproduction Assessment

Yes. The branch-level blockers are reproducible from static inspection of the PR diff and review comments: the type-only import resolves to a nonexistent module, required `SandboxContext.scopeKey` would break existing constructors, and shared sandbox scopes can be derived from `resolveSandboxScopeKey` on current main.

## Solution Assessment

No. The PR addresses real bugs, but the current patch is incomplete and unsafe; the narrower maintainable solution is to split or rebase it around current main and redesign sandbox teardown so it cannot dispose resources owned by another active run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Fix the sandbox context import path:** `src/agents/sandbox/browser.create.test.ts:8`
  - body: The added type annotation imports `./sandbox/context.js` from inside `src/agents/sandbox`, which resolves to a nonexistent nested path. This will fail TypeScript module resolution; it should match the runtime import, `./context.js`.
  - confidence: 0.95
- **[P1] Update every SandboxContext constructor:** `src/agents/sandbox/types.ts:90`
  - body: The patch makes `SandboxContext.scopeKey` required, but existing helpers and tests that construct `SandboxContext` objects are not all updated. That leaves the branch with typecheck failures outside the edited fixture.
  - confidence: 0.88
- **[P1] Do not tear down shared browser bridges per attempt:** `src/agents/pi-embedded-runner/run/attempt.ts:1973`
  - body: Calling `disposeSandboxContext(sandbox)` unconditionally at attempt exit can remove the bridge for shared or agent-scoped sandboxes while another overlapping run still needs it. Teardown needs ownership/refcounting or one-shot/private-scope gating before this can be safe.
  - confidence: 0.84
- **[P2] Clear all CLI continuity fields on rollover:** `src/agents/command/attempt-execution.ts:100-107`
  - body: The rollover clear list removes `cliSessionIds` but leaves `cliSessionBindings` and `claudeCliSessionId`, so a fresh explicit session can still inherit stale CLI resume state. Clear the full continuity set or use the existing CLI-session clearing helper.
  - confidence: 0.86
