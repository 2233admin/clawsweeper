## Summary

Keep open. Current main still does not expose `channels.telegram.streamThrottleMs` or `channels.telegram.minInitialChars` through Telegram config validation, exported config types, docs/metadata, or production dispatch wiring, so this PR is not obsolete. It remains the active implementation path for #38066, but it needs maintainer review before merge because the branch exposes a user-configured timer value without an upper bound and changes shared draft-stream finalization behavior.

## What This Changes

The PR adds optional Telegram `streamThrottleMs` and `minInitialChars` config fields, threads them from Telegram dispatch into draft streaming, adjusts draft/message throttle and final-drain behavior, regenerates config metadata/baselines, and adds focused config, Telegram, and shared draft-stream tests.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to rebase it onto current main, expose `channels.telegram.streamThrottleMs` and `channels.telegram.minInitialChars` consistently across schema, exported config types, generated metadata/docs, and Telegram dispatch, add a timer-safe maximum or runtime clamp for `streamThrottleMs`, preserve draft-vs-message fallback and final-flush semantics, then land with targeted config, Telegram draft-stream, and shared draft-loop validation. #38066 should close only after the implementation merges.
