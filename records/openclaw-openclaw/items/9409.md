## Summary

Keep open. Current main still emits generic user-visible context-overflow messages across the runner, precheck, shared formatter/sanitizer, and auto-reply fallback paths. The requested session/channel label, context size, model limit, model identity, and overage are partly available in logs or metadata, but they are not surfaced to users. The direct implementation PR #15749 is closed unmerged, and broader context-visibility issue #67690 does not supersede this narrower diagnostic request.

## Best Possible Solution

Keep this open and implement a shared, safe context-overflow diagnostic formatter. It should accept provider/model identity, effective context budget or model limit, observed or estimated prompt/session token count, computed overage, and a redacted session/channel display label, then wire that formatter through run.ts, precheck/attempt handling, formatAssistantErrorText, sanitizeUserFacingText, normalize/auto-reply fallbacks, and focused regression tests. Add a user-facing changelog entry crediting @liaosvcaf.
