## Summary

Keep open. Current main still surfaces generic context-overflow copy in the runner, precheck path, shared formatter/sanitizer, and auto-reply fallbacks. Some diagnostics are logged or stored in metadata, but the requested user-visible session/channel label, token count, model limit, model identity, and overage are not wired through. The direct implementation PR #15749 is closed unmerged, and the broader context-visibility issue #67690 does not supersede this narrower diagnostic request.

## Best Possible Solution

Keep this issue open. The best maintainer path is to add a shared, safe context-overflow diagnostic formatter that accepts provider/model identity, effective context budget or model limit, observed or estimated token count, computed overage, and a redacted session/channel display label, then wire it through run.ts, precheck/attempt handling, formatAssistantErrorText, sanitizeUserFacingText, and auto-reply fallbacks with focused regression tests.
