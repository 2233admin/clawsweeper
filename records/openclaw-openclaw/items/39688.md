## Summary

Keep #39688 open. Current main still emits `message:received` and `message:sent` internal hooks through `fireAndForgetHook` without retaining, awaiting, or draining `event.messages` back to a user-visible reply path. The reported Feishu surface uses that shared dispatch path, and open PR #41375 remains the active unmerged fix path rather than shipped behavior.

## Best Possible Solution

Keep #39688 open until PR #41375 or an equivalent maintainer-approved fix lands. The fix should drain internal hook replies only where a safe reply target exists, add regression coverage proving hook-produced `event.messages` are delivered on replyable `message:received` surfaces, and update the hooks docs so the documented `event.messages` behavior matches the supported surfaces. Decide `message:sent` auto-delivery separately rather than closing this bug as already fixed.
