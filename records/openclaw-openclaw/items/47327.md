## Summary

Keep this PR open. Current main still has no parse-error diagnostics contract for skills: malformed SKILL.md files can still be dropped by the local loader, SkillEntry/SkillStatusEntry have no parseError field, and list/info/check output has no parse-error status or JSON bucket. Main does include a tolerant frontmatter parser that fixes the common unquoted-colon description case from #22134, but that is not the same as surfacing parse failures. The submitted PR is still relevant, though it needs the unresolved formatter coverage from review feedback before it is merge-ready.

## Best Possible Solution

Rework and rebase this PR as a focused diagnostics change: preserve SKILL.md read/parse diagnostics from the current local-loader path, carry them through SkillEntry and SkillStatusEntry, mark parse-error skills ineligible, surface them consistently in skills list/info/check text and JSON output, and add targeted regression tests. Keep the existing tolerant parser behavior for common unquoted-colon descriptions unless maintainers explicitly choose stricter YAML reporting.
