## Summary

Keep this PR open. Rechecking current main at 7a23b2d9 shows the intended diagnostics contract is still not implemented: malformed or unreadable local skills can still be skipped without a surfaced parse/read error, SkillEntry and SkillStatusEntry still have no parseError field, and skills list/info/check output still has no parse-error status or JSON bucket. Main does include a tolerant frontmatter parser that fixes common unquoted-colon descriptions from the linked issue, but that is not the same as surfacing SKILL.md failures. The PR remains relevant, though it needs a rebase onto the current local-loader path and the unresolved formatter/test coverage from prior review feedback.

## Best Possible Solution

Rework and rebase this PR as a focused diagnostics change: have the current local skill loader preserve read/parse/malformed SKILL.md diagnostics for skipped local skills, carry that through SkillEntry and SkillStatusEntry, mark parse-error skills ineligible, surface them consistently in skills list/info/check text and JSON output, and add targeted regression tests. Preserve the existing tolerant parser behavior for common unquoted-colon descriptions unless maintainers explicitly choose stricter YAML reporting.
