## Summary

Current main still has the reported Volcengine/BytePlus Kimi catalog mismatch. The old shared TypeScript catalog has moved into provider plugin manifests, but the Volcengine and BytePlus rows for Kimi K2.5 and Kimi K2 Thinking still use `maxTokens: 4096` and omit `reasoning`, which the manifest loader turns into `false`. Together still advertises Kimi K2.5 with `reasoning: true` and `maxTokens: 32768`, so the issue remains actionable provider-catalog work rather than a close candidate.

## Best Possible Solution

Keep this issue open. The best fix is a provider-owned catalog update in `extensions/volcengine/openclaw.plugin.json` and `extensions/byteplus/openclaw.plugin.json`, verified against current Volcengine/BytePlus vendor limits and reasoning controls, plus focused regression tests proving Kimi K2.5 and Kimi K2 Thinking metadata propagates through the manifest catalog builder and provider catalog surfaces. If vendor behavior legitimately differs from Together, document that provider-specific difference instead of normalizing by assumption.
