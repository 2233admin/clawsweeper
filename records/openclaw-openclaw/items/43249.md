## Summary

Keep this issue open. Current main has targeted Telegram 400 handling for known descriptions, but the report still only says generic recent 400 errors and does not include the Telegram `description`, OpenClaw version, affected send/receive/startup path, or chat/topic target shape needed to prove an implemented fix or safely close as stale.

## Best Possible Solution

Keep the issue open and ask the reporter for the redacted exact Telegram error JSON or full GrammyError line, OpenClaw version, whether the failure is send, receive, polling, webhook, or startup, the affected chat/topic target shape, and relevant gateway logs. Once the description is known, route the fix to the Telegram plugin path: formatting or empty text in `extensions/telegram/src/send.ts` or `extensions/telegram/src/bot/delivery.replies.ts`, target/topic handling in `extensions/telegram/src/targets.ts` and `extensions/telegram/src/reply-parameters.ts`, command-menu overflow in `extensions/telegram/src/bot-native-command-menu.ts`, or network/proxy behavior in `extensions/telegram/src/fetch.ts`, `extensions/telegram/src/network-errors.ts`, and polling/webhook startup code.
