## Summary

Keep this issue open. Current main has targeted Telegram handling for several known 400 families, but the report still lacks the exact Telegram `description`, OpenClaw version, failing path, and target shape needed to prove that the observable problem is fixed or safely close it as stale.

## Best Possible Solution

Keep the issue open and ask the reporter for the redacted exact Telegram error JSON or full GrammyError line, OpenClaw version, whether the failure is send, receive, polling, webhook, or startup, the affected chat/topic target shape, and relevant gateway logs. Once known, route the fix to the Telegram plugin path: formatting or empty text in `extensions/telegram/src/send.ts` or `extensions/telegram/src/bot/delivery.replies.ts`, target/topic handling in `extensions/telegram/src/targets.ts` and `extensions/telegram/src/reply-parameters.ts`, command-menu overflow in `extensions/telegram/src/bot-native-command-menu.ts`, or network/proxy behavior in `extensions/telegram/src/fetch.ts`, `extensions/telegram/src/network-errors.ts`, polling, or webhook startup code.
