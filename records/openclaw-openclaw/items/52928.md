## Summary

Current main still does not implement the requested `agents.setDefault` Gateway RPC. The related Control UI default-agent bug was fixed by staging `agents.list[].default` through the config form, but that leaves external desktop/web clients with the same `config.patch` workaround this issue is asking to replace.

## What This Changes

Request an ADMIN-scoped Gateway RPC method `agents.setDefault` that sets one configured agent as the default, clears the default flag on other agents, rejects unknown agent ids, and returns the new default id.

## Best Possible Solution

Add a small additive core Gateway method for default-agent selection while keeping `config.patch` as the advanced bulk-edit escape hatch. The method should share existing agent/config mutation helpers, validate the requested id against configured agents, persist exactly one `default: true` flag, and document the new public protocol contract.
