## Summary

Keep this PR open. Current main covers the gateway config.patch restart problem through the newer config-write flow, but it still does not implement the PR's MiniMax XML tool-call parsing/promotion path; main only strips that XML from visible assistant text. The PR also has unresolved parser correctness review concerns, so it should not be closed as implemented or merged as-is.

## Best Possible Solution

Keep this PR open for maintainer handling, but rework it before merge: split or rebase away the gateway config.patch half that current main already covers, then implement a robust MiniMax XML-to-toolUse path with XML entity decoding, attribute-order-independent parsing, and execution-level regression coverage. If maintainers decide legacy MiniMax M2.5 XML output is no longer supported, close it through an explicit product decision or a verified canonical successor.
