## Summary

Keep this PR open. Current main already covers the gateway config.patch restart behavior, but it still does not implement the PR's MiniMax XML tool-call parsing/promotion path; main only strips that XML from visible text. The PR also has unresolved parser-correctness review concerns, so it is not safe to close as implemented or merge as-is.

## Best Possible Solution

Keep this PR open for maintainer handling, but rework it before merge: split or rebase away the gateway config.patch half that current main already handles, then either implement a robust MiniMax XML-to-toolUse path with entity decoding, attribute-order-independent parsing, and execution-level regression coverage, or have a maintainer close the MiniMax XML work explicitly if newer MiniMax support paths supersede it.
