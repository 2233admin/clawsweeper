## Summary

Current main still does not implement the requested deterministic, public sub-agent announce suppression control. The visible `sessions_spawn` schema and docs do not expose `announce`, `silent`, `suppressAnnounce`, `announcePolicy`, or `announceTarget`; `agents.defaults.subagents` has no policy key and the zod schema is strict; suppression remains driven by sub-agent output tokens (`ANNOUNCE_SKIP` / `NO_REPLY`) or internal lifecycle fields. Prior direct implementation attempts #8306 and #8393 are closed unmerged, while open #27445 is broader parent-session routing rather than a replacement for this narrow skip control.

## Best Possible Solution

Keep this issue open until OpenClaw adds a documented framework-level sub-agent completion announce suppression control. The narrow fix should keep today’s default behavior, add an explicit skip path through `sessions_spawn` and/or `agents.defaults.subagents`, thread that state through registry/lifecycle/delivery without relying on `ANNOUNCE_SKIP` or `NO_REPLY`, treat intentional suppression as successful cleanup, and document it at https://docs.openclaw.ai/tools/subagents. Broader parent-session routing should remain tracked separately by #27445.
