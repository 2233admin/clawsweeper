## Summary

Keep this issue open. Current main still has no documented config option or public `sessions_spawn` parameter that deterministically suppresses sub-agent completion announces. The only implemented suppression path is still model/output-token driven (`ANNOUNCE_SKIP` or `NO_REPLY`), while the config schema and docs omit `announcePolicy`, `suppressAnnounce`, `silent`, or `announceTarget`. Prior implementation attempts #8306 and #8393 were closed unmerged, and open #27445 is broader routing work rather than a replacement for this narrower suppress/skip control.

## Best Possible Solution

Keep this open until OpenClaw adds a documented framework-level sub-agent announce suppression control, either under `agents.defaults.subagents` and/or as a public `sessions_spawn` parameter. The implementation should thread the chosen policy through registry and lifecycle delivery paths, treat intentional suppression as successful cleanup rather than failed delivery, update config/schema/help/docs at `https://docs.openclaw.ai/tools/subagents`, and add regression tests proving deterministic skip behavior without relying on `ANNOUNCE_SKIP` or `NO_REPLY`.
