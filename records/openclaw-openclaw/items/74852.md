## What This Changes

The PR adds read-only Gateway `environments.list` and `environments.status` RPC support with SDK call-through/types, protocol schemas/generated Swift models, read-scope entries, docs/changelog updates, and targeted tests while leaving create/delete unsupported.

## Best Possible Solution

## Reproduction Assessment

Yes. On current main, calling `oc.environments.list()` or `oc.environments.status(...)` follows hard-coded unsupported SDK paths, the tests assert that behavior, and Gateway has no registered `environments.*` methods.

## Solution Assessment

Yes at the design level, pending normal review. Routing SDK list/status through typed read-only Gateway RPCs with protocol/docs/tests, while keeping create/delete and per-run environment selection unsupported, is the narrow maintainable first step.
