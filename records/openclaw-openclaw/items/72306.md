## Summary

Keep open. Current main still does not satisfy the documented `sessionKey="current"` behavior for sparse channel-plugin requesters, and the linked bug remains open. The PR is active, plausibly addresses the regression, and overlaps with #74142, so conservative cleanup should leave it for maintainer review rather than close it.

## What This Changes

This PR adds a final `session_status` current-session fallback for channel-plugin requester keys and adds regression tests, including valid session-id persistence when a model override materializes the synthesized entry.

## Best Possible Solution

Adopt a single implementation that treats literal/default `current` as the requester only after literal current key/sessionId lookups fail, preserves existing visibility checks and literal-current precedence, and generates a valid session id before persisting model changes; keep the regression coverage near the session-status tests.
