## Summary

Keep this PR open. Current main still has the lifecycle gap described by the PR: pricing fetches use timeout-only abort signals, the 24-hour refresh timer is a normal ref'd timeout, and gateway shutdown only clears the scheduled timer without aborting an in-flight refresh. The provided PR context also shows the current draft head was force-pushed to a one-line comment-only change, so it is not currently mergeable as the stated fix, but the underlying bug remains unresolved.

## Best Possible Solution

Keep this PR open as a draft implementation candidate. The useful path is to restore or replace the functional change in src/gateway/model-pricing-cache.ts: unref the 24-hour pricing refresh timer, thread a shutdown AbortController into both LiteLLM/OpenRouter pricing fetches, abort in-flight refreshes from the returned stop callback, and add focused regression tests in src/gateway/model-pricing-cache.test.ts. Coordinate with #73486 only for startup deferral; it does not cover this lifecycle fix.
