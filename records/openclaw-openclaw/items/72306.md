## Summary

Keep this PR open. Current main still does not satisfy the linked channel-plugin `sessionKey="current"` regression, and the PR targets the right surface, but the latest patch can persist a synthesized session entry with `sessionId: ""` when a `model` override is supplied. It needs one small code guard and a regression test before merge, then should be reviewed with #74141 and overlapping #74142 so one canonical fix lands.

## What This Changes

The PR adds a final literal `sessionKey="current"` fallback in `session_status` that synthesizes a requester-scoped session entry for unstored channel-plugin agents, plus regression tests for a channel requester and non-current unknown keys.

## Best Possible Solution

Keep the PR open and amend it: preserve the final literal-current requester fallback, prevent synthesized fallback entries from being written by the model-override path, add a regression test proving `sessionKey="current"` plus `model` does not create a store record with an empty `sessionId`, then review it alongside #74141 and #74142 and land one canonical fix.
