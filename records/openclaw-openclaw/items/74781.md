## Summary

Keep open for maintainer review. Current main does not yet include the stacked Gateway wait-snapshot behavior, and the PR addresses a real remaining gap in #74751 by writing terminal wait snapshots for every run returned from a session-wide abort. It should not be auto-closed or replaced; the remaining decision is whether to fold or merge it with the open maintainer-labeled base PR.

## What This Changes

This PR updates the stacked Gateway `sessions.abort` path to normalize all returned aborted run IDs, write RPC timeout wait snapshots for each aborted run, and add a multi-run regression test.

## Best Possible Solution

Land this together with #74751 so `sessions.abort` records terminal wait snapshots for every aborted chat-send or agent-kind run while preserving the existing first-`abortedRunId` response shape and run-kind-specific dedupe namespaces.
