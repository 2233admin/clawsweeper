## Summary

Keep open. Current main still lacks any `groupScope` route-binding support, so the PR is not obsolete, but the proposed patch is not merge-ready: it only threads the new scope through inbound routing, leaves outbound and Telegram topic-derived session routes inconsistent, bundles the separate open IRC proactive-send work from #45662, and needs explicit privacy documentation before merge.

## What This Changes

The PR adds a per-binding `groupScope` route option for sending matched group/channel conversations to an agent main session, and also adds IRC monitor active-client reuse for proactive sends.

## Best Possible Solution

Refresh this as a first-class binding session-scope change on current main, with the canonical config shape chosen deliberately, generated schema/help and public docs updated, and focused tests covering inbound routing, outbound session derivation, Telegram topic overrides, and the default per-channel behavior. The IRC active-client fix should be split out or rebased after #45662 is resolved.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main rejects/does not expose `groupScope`, group/channel peers still derive non-main session keys, and the PR diff can be checked against outbound and Telegram recomputation paths that do not carry the new scope.

## Solution Assessment

No. The opt-in direction is plausible, but this patch is incomplete across session derivation paths and documentation, and it mixes an unrelated IRC fix that already has its own open PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Propagate groupScope into outbound session derivation:** `src/routing/resolve-route.ts:673`
  - body: When a binding sets `groupScope: "main"`, inbound routing can collapse group/channel traffic onto the main session, but outbound helpers still derive group targets from only global `dmScope`. Sends that omit an explicit session key can therefore fork the same configured conversation back into a per-channel session instead of the main session.
  - confidence: 0.89
- **[P2] Preserve groupScope in Telegram topic overrides:** `src/routing/resolve-route.ts:673`
  - body: Telegram forum topics with `topicAgentId` recompute `route.sessionKey` using `buildAgentSessionKey()` and only global `dmScope`. A topic matching `groupScope: "main"` would be overwritten back to a per-topic group session, so that supported channel path would not honor the new option.
  - confidence: 0.84
- **[P3] Fall back when active IRC sends fail:** `extensions/irc/src/send.ts:72-74`
  - body: When an active monitor client is found, the new IRC path calls `active.sendPrivmsg()` and returns. If the monitor connection drops after `isReady()` but before the write, proactive sends fail instead of trying the transient fallback preserved for missing active clients.
  - confidence: 0.72
