## Summary

Current main partially fixes the original report: numeric HTTP 500 and JSON Codex `server_error` payloads are now classified as transient failover errors. The later reported raw OpenAI wording beginning `The server had an error processing your request...` is still absent from the classifier and tests, and the assistant fallback policy still needs a failover signal, classified reason, or timeout before rotating/falling back. Keep this open for a focused regression and narrow classifier update.

## Best Possible Solution

Keep this issue open. Add a focused regression for an OpenAI Codex assistant error with `stopReason: "error"`, raw text beginning `The server had an error processing your request...`, and configured model fallbacks. Then update the failover classifier with a narrow server-error pattern, such as the exact OpenAI phrase, so the assistant path throws a `FailoverError` and the outer model fallback loop reaches the next configured model.
