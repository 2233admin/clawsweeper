## Summary

Keep this issue open. Current main has fixed the original numeric HTTP 500 and Codex JSON `server_error` paths, and that partial fix shipped in `v2026.4.26`, but the later concrete OpenAI wording beginning `The server had an error processing your request...` is still absent from the failover classifier and tests. For an assistant `stopReason: "error"` with that unclassified text, the assistant failover decision still needs `failoverFailure`, a classified reason, or a timeout before it rotates or throws a model-fallback `FailoverError`.

## Best Possible Solution

Keep this issue open and land a focused regression/fix for the raw OpenAI Codex assistant error text beginning `The server had an error processing your request...`. Add the narrow phrase to the server-error classifier, cover direct classification and the embedded runner fallback handoff, and preserve the already-shipped numeric 500 / JSON `server_error` behavior.
