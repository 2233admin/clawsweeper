## Summary

Keep open. Current main still lacks a token-free static cron message payload: the cron data model, Gateway schema, runtime validation, runtime execution, CLI, Control UI, agent tool schema, and docs all expose scheduled user-visible work as either `systemEvent` or `agentTurn`. The related direct-exec tracker is broader, remains open, and is not a shipped replacement for fixed text-to-channel delivery.

## What This Changes

Request a new static cron payload that sends fixed text directly to a channel target without starting an AI or agent turn.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel messages, or explicitly fold fixed text into the broader non-agent cron design, then wire the chosen contract through cron types, normalization, Gateway validation, runtime dispatch/run logs, CLI add/edit, Control UI, agent tool guidance, focused tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The missing behavior is high-confidence by source and docs inspection: current main exposes only `systemEvent` and `agentTurn`, and static text cannot be scheduled for direct channel delivery without an agent run.

## Solution Assessment

Unclear. A `staticMessage` payload is a plausible narrow solution for fixed reminders, but maintainers should decide whether it stands alone or becomes part of the broader non-agent cron execution/delivery contract tracked by #18160.
