## Summary

Keep this issue open. Current main still has no token-free static cron message payload: the cron type/schema/runtime/CLI/UI/docs surfaces still accept only `systemEvent` and `agentTurn`, and related #18160 is broader direct-exec work that remains open rather than a shipped replacement for fixed text-to-channel delivery.

## Best Possible Solution

Keep this issue open until OpenClaw either implements an additive `staticMessage`/fixed-text cron payload with direct channel delivery, or explicitly folds fixed text-to-channel delivery into a maintained non-agent cron design such as #18160. A complete design should update cron payload types, Gateway schemas/validators, normalization/patch handling, runtime delivery and run logs, CLI, Control UI, agent cron tool guidance, docs, and focused tests.
