## Summary

Keep open. Current main still does not provide a token-free static cron payload: cron payload types, Gateway validation, runtime validation/execution, CLI, Control UI, agent-tool guidance, tests, and docs all continue to model `systemEvent` and `agentTurn` only. Related #18160 is a broader open direct-exec design, not a shipped or exact replacement for fixed text-to-channel delivery, and the remaining API/security semantics need maintainer direction before implementation.

## What This Changes

The issue requests a `staticMessage` cron payload that sends fixed text directly to a channel target without invoking an agent or model call.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, then wire it through cron types, normalization, Gateway validation, job validation, runtime dispatch/run logs, CLI add/edit, Control UI, agent cron tool guidance, tests, and public docs while deciding its relationship to #18160.
