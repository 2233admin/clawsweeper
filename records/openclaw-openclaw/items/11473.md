## Summary

Keep open. Current main and the latest shipped tag still only model cron work as `systemEvent` or `agentTurn`; there is no token-free `staticMessage` payload, and related #18160 is broader open direct-exec work rather than a shipped replacement for fixed text-to-channel delivery.

## What This Changes

Request a new cron payload that sends fixed channel text directly without invoking AI processing or spending model tokens.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, or deliberately fold that contract into the broader direct-exec design, then wire it through cron types, normalization, Gateway validation, runtime dispatch/run logs, delivery plumbing, CLI add/edit, Control UI, agent cron tool guidance, focused tests, and public docs.
