## Summary

Keep open. Current main moved xhigh/thinking support to provider-owned thinking profiles and already covers direct OpenAI, OpenAI Codex, and GitHub Copilot xhigh models, but the PR's stated Vercel AI Gateway proxy case is still not implemented. The PR should not be closed as implemented, and it should be reworked rather than merged as-is because its core fallback approach is stale and the current PR head includes unrelated runtime loader churn.

## Best Possible Solution

Keep this PR open for rework. Rebase onto current main, drop unrelated channel/web-search/test churn, and implement the remaining useful behavior through provider-owned thinking profiles: likely a Vercel AI Gateway `resolveThinkingProfile` that recognizes trusted upstream OpenAI/Codex refs, or a narrow shared helper for trusted proxy refs. Add focused regression tests showing accepted Vercel OpenAI xhigh refs and rejected unsupported or untrusted namespaced refs.

## Risks / Open Questions

- Merging the PR as-is would push xhigh through a stale core fallback instead of the provider-owned `resolveThinkingProfile` seam now used on main.
- The PR's fallback design can over-grant xhigh for explicit provider/model mismatches unless reworked around trusted upstream refs or provider-owned policy.
- The current PR head includes unrelated `src/plugins/bundled-web-search.ts` runtime loader/export changes with unresolved P1 review concerns; that should be dropped from a thinking-focused fix.
- Security review: the current PR file list does not show CI workflow, dependency source, lifecycle script, secret-handling, or package-publishing changes, but its unrelated runtime plugin loading changes are security-sensitive enough to require separation and normal loader validation.
