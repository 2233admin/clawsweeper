## Summary

Keep this PR open, but it should be reworked rather than merged as-is. Current main already moved thinking support to provider-owned profiles and implements direct OpenAI/OpenAI Codex/GitHub Copilot xhigh profiles, so the old core allowlist expansion is the wrong integration point. The PR's Vercel AI Gateway proxy case still appears unimplemented on main: Vercel catalogs upstream OpenAI refs but does not expose a thinking profile hook, and sessions.patch validates xhigh against the effective provider/model. This matches the April 27 Codex review already posted in the PR discussion.

## Best Possible Solution

Keep this PR open for rework. Rebase onto current main, drop unrelated web-search and channel test churn, and implement the useful proxy behavior through provider-owned thinking profiles, likely in extensions/vercel-ai-gateway or a narrow shared helper for trusted upstream refs. Add focused thinking and sessions.patch regression tests for accepted Vercel OpenAI refs and rejected unsupported refs.
