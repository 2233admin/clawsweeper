## Summary

Keep this PR open. Current main has superseded the old core allowlist approach with provider-owned thinking profiles and already covers direct OpenAI, OpenAI Codex, and GitHub Copilot xhigh models, but it still does not implement the PR’s stated Vercel AI Gateway proxy-ref case. The PR also should not be merged as-is because its fallback design is stale and the current PR file list includes unrelated runtime plugin-loader churn with unresolved P1 review concerns.

## Best Possible Solution

Keep the PR open for rework. Rebase onto current main, drop unrelated channel/web-search/test churn, and implement the remaining useful behavior through provider-owned thinking profiles, likely a Vercel AI Gateway `resolveThinkingProfile` that recognizes trusted upstream OpenAI/Codex refs or a narrow shared helper for trusted proxy refs. Add focused regression coverage for accepted Vercel OpenAI xhigh refs and rejected unsupported or untrusted namespaced refs.
