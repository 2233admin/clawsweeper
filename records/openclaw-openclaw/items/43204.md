## Summary

Keep this PR open. Current main still does not implement the PR’s intended TTL-expiry feedback path: the typing TTL timer only logs and cleans up, there is no reply option or dispatch hook for a visible still-working notice, and the docs describe typing cadence but not TTL-expiry feedback. The PR is still relevant, but it should not be merged as-is because the head is stale and mixes the auto-reply fix with unrelated macOS/onboarding and Tlon dependency changes.

## Best Possible Solution

Keep this PR open as reference, but do not merge it as-is. The maintainable path is a fresh or heavily rebased auto-reply-only change against current main that adds the chosen TTL-expiry feedback contract, contains both synchronous throws and async rejections from custom handlers, preserves routed-reply ordering, adds focused tests for TTL expiry, suppressed typing, and routed replies, and drops unrelated macOS/Tlon/package-resolution changes. Coordinate with #40760 before choosing the final product behavior.
