## Summary

Keep this PR open. Current main still has the silent typing-TTL expiry path: the timer logs and cleans up, but there is no TTL-expiry callback, no reply option for caller-supplied TTL feedback, and no dispatch-level still-working notice. The PR remains relevant, but it should be refreshed into a focused auto-reply change and coordinated with open related PR #40760 rather than closed as implemented.

## Best Possible Solution

Keep this PR open as a useful reference, but require a fresh or heavily rebased auto-reply-only change against current main. The best patch would add the chosen TTL-expiry feedback contract, contain both synchronous throws and async rejections from custom handlers, preserve routed-reply ordering, add focused tests for TTL expiry, suppressed typing, and routed replies, drop unrelated macOS/Tlon/package-resolution changes, and coordinate the product behavior with #40760.
