## Summary

Keep open. Current main still has the behavior this PR is trying to fix: memory-wiki lint only indexes exact extensionless relative paths and basenames, then compares those against raw extracted wikilink targets. That means title-style links, slug variants, Obsidian links with fragments, and imported source-path suffixes can still be false-positive `broken-wikilink` warnings. The PR is a narrow candidate fix for open #73574 and does not show an obvious security or supply-chain regression from the provided diff.

## Best Possible Solution

Continue review of this PR as the implementation candidate for #73574. Keep the fix in the memory-wiki lint resolver, add or keep the focused regression test for title, slug, fragment, and imported source-path suffix variants, and require targeted format/test proof before landing.
