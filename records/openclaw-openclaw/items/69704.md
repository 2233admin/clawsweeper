## Summary

Keep this PR open. Current main already sends Brave requests with the `X-Subscription-Token` header, but it still lacks this PR's focused regression tests that assert header auth and confirm the API key is not sent as `apikey` or `key` query parameters.

## What This Changes

This PR adds two Brave web search provider regression tests covering `X-Subscription-Token` header auth and absence of `apikey`/`key` query parameters for web and llm-context modes.

## Best Possible Solution

Review and land this existing test-only PR, or add equivalent Brave provider regression coverage on main, so both web and llm-context modes assert `X-Subscription-Token` header auth and confirm the API key is not sent as `apikey` or `key` query parameters. No production runtime or core API change appears needed.
