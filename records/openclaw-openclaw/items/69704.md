## Summary

Keep PR #69704 open. Current main already sends Brave API keys via `X-Subscription-Token`, but the PR is specifically adding missing regression coverage for that behavior, and equivalent header/query-param assertions are not present on main.

## Best Possible Solution

Keep the PR open for maintainer review and likely merge or adapt its two focused regression tests into `extensions/brave/src/brave-web-search-provider.test.ts`, since current main has the Brave header-auth behavior but lacks the guard coverage tied to closed bug #69673.
