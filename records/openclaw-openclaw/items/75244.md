## Summary

Keep open. Current main still enforces the documented `parameters`-object contract and deliberately rejects `inputSchema`-only plugin tools, while the TypeBox-specific theory is not confirmed by the inspected TypeBox source or by OpenClaw's bundled memory-core pattern. The remaining decision is whether maintainers want a diagnostic-only improvement, a plugin-side fix, or an intentional plugin API compatibility change.

## What This Changes

The issue asks OpenClaw to stop silently dropping plugin tools whose TypeBox `parameters` schema is reported as missing, with better diagnostics and possible `inputSchema` compatibility.

## Best Possible Solution

Capture the failing `openclaw-remnic` returned tool object first, then either fix the plugin to return the documented `parameters` object or make an intentional OpenClaw contract change with focused validator tests, docs, and clear diagnostics.

## Reproduction Assessment

## Solution Assessment
