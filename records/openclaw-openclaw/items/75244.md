## Summary

Keep open. Current main still rejects plugin tools whose `parameters` value is missing or non-object, and the documented/native plugin contract still treats `inputSchema`-only registrations as malformed; the reported external memory-plugin failure is not fixed or superseded on main. The TypeBox root-cause theory remains unproven because the cited @sinclair/typebox version builds an ordinary object schema, so maintainers need the actual failing runtime value or an explicit API/diagnostics decision.

## What This Changes

Report that external memory plugin tools using TypeBox-backed schemas are rejected as `missing parameters object`, causing `memory_search` and `memory_get` to be dropped on each agent turn.

## Best Possible Solution

Keep the issue open until maintainers can capture the failing plugin's actual tool object or choose a deliberate contract change. The end state should be either a plugin-side fix that returns the documented `parameters` schema object, or a targeted OpenClaw improvement that adds actionable diagnostics and, if approved, documents/tests any new schema alias behavior.

## Reproduction Assessment

Unclear. The issue gives concrete external steps, but the failing plugin/runtime value is not available in this repo, and source inspection shows a normal @sinclair/typebox `Type.Object()` result should pass the current validator.

## Solution Assessment

Unclear. Better diagnostics in the malformed-tool path look like the narrowest safe core improvement, but accepting `inputSchema` or adding TypeBox-specific exceptions should wait for maintainer API judgment and proof of the actual failing value.
