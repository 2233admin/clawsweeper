## Summary

Keep open. Current main still requires plugin tools to return a non-array object at `parameters`, intentionally treats `inputSchema`-only plugin tools as malformed, and has no relevant change in the checked files since the earlier ClawSweeper review. The external failure may be real, but the TypeBox-specific root cause remains unconfirmed because the same module-level schema pattern exists in bundled memory-core and should satisfy the current object guard.

## What This Changes

The issue reports that TypeBox-backed plugin memory tools are rejected as malformed and dropped from each agent turn, with a request for better diagnostics and possible schema-contract changes.

## Best Possible Solution

Keep the report open until maintainers can capture the failing plugin's actual returned tool object or choose a deliberate contract change. The desired end state is either a plugin-side correction to return the documented `parameters` object, or an OpenClaw change that improves diagnostics and documents any new schema alias behavior with focused contract tests.

## Reproduction Assessment

Unclear. The issue gives concrete external steps, but this checkout does not contain the external plugin value, and current main shows that a real non-array object at `parameters` should pass the validator.

## Solution Assessment

Unclear. Better diagnostic detail is a narrow safe first step, but accepting `inputSchema` as a native `registerTool` alias is a public plugin-contract change rather than an obvious bug fix.
