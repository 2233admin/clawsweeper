## Summary

Keep open. Current main implements first-party local backup archive creation and verification, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled lifecycle and component-aware cron/session restore semantics. The issue discussion also points to useful community workarounds while the latest maintainer-side review kept the native umbrella open.

## What This Changes

Request a built-in OpenClaw backup/restore utility covering config, cron jobs, skills/workspaces, session history, approvals, pairing state, optional cloud storage, encryption, scheduling, and selective import/export.

## Best Possible Solution

Keep this as the native umbrella for a reviewed first-party backup lifecycle built on the existing create/verify archive and migration seams, with safe extraction, dry-run validation, explicit component modeling for cron and sessions, secret/credential policy, overwrite and rollback rules, cloud/encryption choices, and docs that point to community tools as optional workarounds.
