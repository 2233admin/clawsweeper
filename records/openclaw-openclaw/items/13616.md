## Summary

Keep open. Current main implements native local backup archive creation and verification, but not the requested native restore/import/list/cloud/encryption/scheduled backup workflow or component-aware cron/session restore semantics. The issue discussion already identifies useful community workarounds, especially openclaw-workspace-sync and Keep My Claw, but the latest maintainer-side comment treats them as workarounds rather than a merged first-party replacement.

## Best Possible Solution

Keep this issue open as the native umbrella for first-party backup restore/import and disaster recovery. Maintainers should coordinate the narrower restore and migration/profile portability work, then settle the design for dry-run validation, overwrite/rollback rules, path safety, secret handling, cron/session component semantics, and whether cloud/encryption/scheduling live in core or are explicitly delegated to ClawHub/plugin APIs. Document community options such as openclaw-workspace-sync and Keep My Claw as workarounds unless maintainers decide the remaining native scope should move out of core.
