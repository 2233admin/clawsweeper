## Summary

Keep open. Current main has first-party local backup archive creation and verification, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled-backup lifecycle and component-aware cron/session restore semantics. The community plugin/skill options in the discussion are useful workarounds, but recent maintainer-side review and current source both leave the native umbrella unresolved.

## What This Changes

The issue requests built-in OpenClaw backup and restore commands covering config, cron jobs, session history, workspace files, selective import/export, cloud storage, encryption, retention, and scheduled backups.

## Best Possible Solution

Keep this as the native umbrella for a reviewed first-party backup lifecycle built on the existing create/verify archive and migration/plugin seams, with safe extraction, dry-run validation, explicit component modeling for cron and sessions, secret/credential policy, overwrite and rollback rules, cloud/encryption/scheduling choices, and docs that point to community tools as optional workarounds.
