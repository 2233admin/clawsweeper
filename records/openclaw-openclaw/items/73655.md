## Summary

Keep this issue open. Current main has partial adjacent mitigations, but it does not resolve the reported Manifest/clawrouter port retry loop, plugin/gateway signal-listener accumulation, or session-store lock/fsync path that can starve WebSocket handshakes.

## Best Possible Solution

Keep this open as a gateway runtime lifecycle bug. The best maintainer path is to split focused fixes for Manifest/clawrouter idempotent port binding or parent-socket reuse, centralized/deduplicated plugin resource cleanup for signal handlers and sockets across reloads, and a session persistence change that keeps durable JSON/fsync work out of latency-sensitive locks or batches it. Add regression coverage for listener counts across reloads, duplicate listen attempts, bounded lock holds, and WebSocket handshake latency while session writes are active.
