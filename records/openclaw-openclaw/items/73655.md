## Summary

Keep this open. Current main has useful mitigations for startup/readiness and lock recovery, but the reported root causes are not closed: session persistence still holds the session-store lock through durable writes, embedded JSONL session locks still span the full attempt cleanup path, plugin replacement cleanup is limited to registered host hooks, and the reported Manifest/clawrouter listener implementation is not present in the checked-in source.

## What This Changes

The issue requests fixing a gateway restart resource/performance triad: Manifest listener rebinding, accumulating signal handlers, and session persistence stalls that starve WebSocket handshakes.

## Best Possible Solution

Split the work into maintainer-owned fixes that preserve the current mitigations while removing the root causes: make the Manifest/clawrouter listener startup idempotent, add a generic lifecycle surface for plugin-owned process resources, and redesign session persistence so slow durable writes and long JSONL attempt locks cannot block latency-sensitive Gateway handshakes or session lanes.
