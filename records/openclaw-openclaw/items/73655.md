## Summary

Keep open. Current main has useful mitigation and diagnostic work, but it does not remove the reported root causes: session-store updates still persist under the session-store lock, plugin replacement cleanup remains limited to registered host hooks, and the Manifest/clawrouter 2099/2100 listener implementation is not present in the checked-in source. The follow-up comments add independent non-Manifest Gateway starvation reports, so this is still active maintainer work rather than stale cleanup.

## What This Changes

The issue requests root-cause fixes for Gateway restart starvation caused by Manifest/clawrouter port rebinding, plugin signal-handler accumulation, and slow session JSONL/session-store persistence blocking WebSocket handshakes.

## Best Possible Solution

Split this into maintainer-owned root-cause fixes: make the Manifest/clawrouter listener startup idempotent, add or require a generic plugin lifecycle cleanup/dedup surface for process-level resources, and redesign session persistence so slow durable writes cannot hold latency-sensitive Gateway/session locks while preserving the intended durability contract.
