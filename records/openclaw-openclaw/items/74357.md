## Summary

Keep open. Current main still applies a private-network-allowing SSRF policy through the shared trusted web-tools endpoint used by bundled web-search helpers and SDK exports, and the linked implementation PR remains open rather than merged.

## What This Changes

The issue asks OpenClaw to split web-tools SSRF policy so fixed public web-search providers stay strict by default while SearXNG-style self-hosted providers explicitly opt into private-network access.

## Best Possible Solution

Review, update, and land or supersede https://github.com/openclaw/openclaw/pull/74360 with strict-by-default trusted web-tools/web-search behavior, an explicit validated self-hosted private-network path for SearXNG-style providers, matching SDK baseline/docs updates, and focused SSRF regression tests.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes, the requested policy split is the narrow maintainable direction. The issue should remain open until the paired PR or a superseding fix is maintainer/security approved, validated, and merged.
