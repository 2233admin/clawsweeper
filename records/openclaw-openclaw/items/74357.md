## Summary

Keep open. Current main still applies the private-network-allowing trusted web-tools SSRF policy through the shared web-search helpers, and the linked fix PR #74360 is open rather than merged. Because this touches SSRF behavior and public plugin SDK helpers, it needs maintainer/security review before the issue is resolved.

## What This Changes

The issue requests splitting web-tools SSRF policy so fixed cloud web-search providers use strict network checks while SearXNG and other self-hosted providers explicitly opt into private-network access.

## Best Possible Solution

Review and either land PR #74360 or replace it with an equivalent security-reviewed change that makes shared web-search helpers strict by default, adds an explicit self-hosted/private-network opt-in for SearXNG and third-party self-hosted providers, and locks the behavior with SSRF and provider tests.
