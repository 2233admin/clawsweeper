## Summary

Keep open. Current main still routes shared web-search and web-tools requests through a trusted SSRF policy that enables private-network access, and the paired implementation PR #74360 is open rather than merged. Because this is SSRF-sensitive and touches plugin-facing SDK helpers, it needs maintainer/security review before closure.

## What This Changes

The issue requests splitting the web-tools SSRF policy so private-network access is explicit for self-hosted providers such as SearXNG while bundled public web-search providers use a strict default.

## Best Possible Solution

Review, land, or supersede PR #74360 with a strict-by-default trusted web-tools/web-search policy, an explicit self-hosted private-network opt-in for SearXNG-style providers, additive SDK exports, and tests proving public providers block private redirects while self-hosted SearXNG remains usable.
