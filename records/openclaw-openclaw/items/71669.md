## Summary

Keep open. Current main still rejects the reported remote plain-HTTP Control UI path and documents the secure-context requirement, but it does not emit the requested server-side startup warning. An open PR already targets this issue with closing syntax, so the issue should remain open until that PR lands, is narrowed, or is replaced.

## What This Changes

The issue requests a gateway startup warning when Control UI is enabled in a configuration that will reject remote plain-HTTP connections requiring browser device identity.

## Best Possible Solution

Land a scoped gateway startup warning with regression coverage in the startup-log tests while preserving the existing remote plain-HTTP security policy; the linked PR can serve as the implementation path if maintainers are comfortable with its scope after review.
