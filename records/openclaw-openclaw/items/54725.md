## Summary

Keep open. Current main still does not expose sessionKey/sessionId to provider extra-param hooks, so the requested capability remains relevant, but this PR should not merge as-is because it advertises the new fields through derived provider contexts without populating them and needs a current-main runtime-plan rebase.

## What This Changes

The PR adds optional sessionKey/sessionId fields to the provider extra-params context and threads them from embedded attempts into provider extra-param preparation.

## Best Possible Solution

Expose session identity through one coherent provider hook contract: add the typed fields where they are actually supported, thread them through runtime-plan, embedded attempt, compaction, transport, and stream-wrapper paths when advertised, and add focused contract tests; alternatively keep the type deliberately narrower so unsupported hooks do not promise those fields.
