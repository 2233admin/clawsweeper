## Summary

Keep open. Current main supports process-level proxy routing and configured provider-request proxy overrides, but auth profile credential types, strict auth profile config, and resolved model auth still have no profile-scoped transport metadata. Because the request would store proxy credentials and alter provider egress routing, it needs maintainer product/security design rather than cleanup closure.

## What This Changes

Add an optional profile-scoped proxy setting so different auth profiles for the same model provider can route outbound requests through distinct proxies.

## Best Possible Solution

Define a first-class profile-scoped transport contract with secret-aware proxy credentials, redaction, validation, and clear precedence, then thread the selected auth profile's transport policy into model request construction while preserving existing process-level and provider-request proxy behavior.
