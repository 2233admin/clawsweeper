## Summary

Keep open. Current main still installs Docker Playwright Chromium under `/home/node/.cache/ms-playwright`, the Docker/browser docs still document that hidden path, and there is no Dockerfile-created stable `chrome` or `chromium` command on PATH. The issue is also paired with open implementation PR #42771 using closing syntax, so it should remain open until that PR is reviewed, landed, refreshed, replaced, or closed.

## What This Changes

The issue requests moving Docker Playwright browser files out of the hidden `/home/node/.cache/ms-playwright` path and exposing a stable browser binary on PATH.

## Best Possible Solution

Review, refresh, and either land or close #42771 or a maintainer replacement. The desired end state is a Docker browser install path outside `/home/node/.cache`, stable `chrome`/`chromium` commands on PATH, aligned Docker and browser-control docs, regression coverage for the Dockerfile contract, and Docker image validation for browser-enabled and browser-disabled builds.
