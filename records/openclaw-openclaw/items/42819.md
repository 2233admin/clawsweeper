## Summary

Keep open. Current main still installs Playwright Chromium into `/home/node/.cache/ms-playwright`, still documents that hidden cache path, and does not create a stable `chrome` or `chromium` executable on PATH. The linked open PR #42771 uses closing syntax for this issue, so cleanup should wait for maintainer review of that implementation path.

## What This Changes

Request Docker browser-enabled builds to place Playwright browser files outside `/home/node/.cache` and expose a stable Chrome/Chromium binary on PATH.

## Best Possible Solution

Review, refresh, and either land or close #42771, or replace it with a maintainer-owned patch that moves the browser assets to a stable non-hidden path, exposes Chrome/Chromium on PATH, updates Docker/browser docs, and adds regression coverage for the Dockerfile contract.
