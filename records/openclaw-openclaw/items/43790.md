## Summary

Keep open. Current main still does not implement explicit active-agent propagation for message-tool sends, so this is not obsolete; however the PR should not merge as-is because it can make an inferred fallback agent authoritative in config-unavailable routing paths, and the changelog entry misses required attribution.

## What This Changes

The PR threads an explicit active agent id through OpenClaw coding-tool construction into the message tool, adjusts tool-policy session fallback behavior, adds focused tests, and adds a changelog bullet.

## Best Possible Solution

Land a rebased, guarded routing fix that only treats a trusted runtime-supplied active agent id as explicit, preserves config-aware fallback resolution for inferred or opaque session keys, covers both explicit precedence and fallback cases in focused tests, and credits the changelog entry correctly.
