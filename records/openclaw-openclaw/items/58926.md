## Summary

Keep this PR open: current main still emits only bare `config get`/`config unset` not-found errors, and the PR is an active, mergeable implementation candidate. The latest branch addresses the earlier P2 review feedback; the remaining concern is a non-blocking completeness question about whether suggestions should draw from the same plugin/channel metadata inventory as config schema lookup.

## What This Changes

The PR adds a config-path suggestion helper backed by `FIELD_HELP`, unit tests for the matcher, and `Did you mean` hint output for `openclaw config get` and `openclaw config unset` not-found paths.

## Best Possible Solution

Land a config-path suggestion feature after maintainers decide whether the suggestion inventory should stay core-only or reuse the schema/hint inventory that already covers bundled channel and plugin config paths.

## Reproduction Assessment

Not applicable as a standalone bug report. The PR behavior has a clear before/after check: current main's `runConfigGet` and `runConfigUnset` only emit the bare not-found error, while the PR adds hints for close static config paths such as `gateway.prot`.

## Solution Assessment

Unclear. Adding hints at the `get` and `unset` not-found branches is the narrow integration point, but using only `FIELD_HELP` is less complete than deriving candidates from the existing schema/hint metadata used by config schema lookup.
