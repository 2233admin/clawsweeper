## Summary

Keep open. Current main still does not implement a global `openclaw agent --dry-run` mode: the agent CLI, Gateway request schema, embedded agent options, docs, and both managed/client tool adapter paths lack request-scoped dry-run plumbing. Existing dry-run support remains scoped to specific commands/tools, not the universal agent tool execution pipeline requested here.

## Best Possible Solution

Keep this issue open as core agent execution work. Use the reporter's linked branch as context, but implement the feature with request-scoped plumbing through `openclaw agent`, Gateway `AgentParamsSchema`, `AgentCommandOpts`, embedded run params, and both `toToolDefinitions()` and `toClientToolDefinitions()`. Add redacted stderr previews, structured `status: "dry_run"` results, docs for `openclaw agent --dry-run`, and focused CLI/Gateway/adapter regression coverage. Keep #13362, #13617, and #31624 separate because they cover narrower command/job/update dry-run scopes.
