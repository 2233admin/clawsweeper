## Summary

Current main still does not implement a global `openclaw agent --dry-run` mode. The agent CLI has no flag, the gateway/embedded agent request types do not carry dry-run state, and the universal tool adapter still executes real tools after before-tool-call policy handling. Existing dry-run support remains scoped to other commands or the message tool, and the plugin hook surface can block, rewrite, or request approval but cannot provide the requested structured preview result for every tool.

## Best Possible Solution

Keep this issue open as actionable core agent execution work. A good implementation would add `openclaw agent --dry-run`, thread request-scoped dry-run state through CLI, gateway/embedded agent params, and the tool definition adapter, intercept both OpenClaw-managed and client tool definitions after policy hooks, return structured preview payloads to the model, log redacted previews to stderr, update `docs/cli/agent.md`, and add focused adapter plus CLI/protocol tests. Keep #13362, #13617, and #31624 separate because they cover narrower command/job/update scopes.

## Risks / Open Questions

- A process-global dry-run flag would be unsafe for concurrent Gateway runs; this should be request-scoped if it crosses the Gateway boundary.
- Preview logging must reuse existing redaction/sanitization conventions before writing tool params to stderr.
- Dry-run interception should run after existing `before_tool_call` policy blocks/approvals so preview mode does not bypass safety policy.
