## Summary

Keep this issue open. Current main still does not provide a global `openclaw agent --dry-run` mode that leaves tools visible to the model while intercepting every managed/client tool execution with structured preview results. Existing dry-run support remains scoped to other commands or the message tool.

## Best Possible Solution

Keep the issue open as actionable core agent execution work. A good implementation would add `openclaw agent --dry-run`, thread dry-run state through CLI, gateway/embedded request params, and the tool definition adapter, intercept both OpenClaw-managed and client tool definitions after policy hooks, return structured preview payloads to the model, log redacted previews to stderr, update `docs/cli/agent.md` and `docs/tools/agent-send.md`, and add focused adapter, CLI, and protocol tests. This does not belong in ClawHub because the missing behavior is a core execution-pipeline capability, not an optional plugin/provider integration.

## Risks / Open Questions

- A gateway-capable implementation should use request-scoped state rather than a process-global flag, otherwise concurrent runs could bleed dry-run behavior across requests.
- Tool preview logging needs the existing redaction/sanitization conventions before writing params to stderr.
- Dry-run interception should happen after before-tool-call policy blocks/approvals so preview mode does not bypass safety policy.
