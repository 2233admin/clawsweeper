## Summary

Current main still lacks a global `openclaw agent --dry-run` mode. The existing dry-run support is scoped to other commands or the message/tool-invoke surfaces, while the agent CLI, gateway protocol, embedded agent options, and universal tool adapter do not carry or enforce a request-wide dry-run state.

## Best Possible Solution

Keep this open as actionable core agent execution work. A strong implementation would add `openclaw agent --dry-run`, thread request-scoped dry-run state through CLI, gateway protocol, embedded agent params, and the tool definition adapter, intercept both managed and client tool definitions after policy hooks, return structured preview payloads to the model, log redacted previews to stderr, and update `docs.openclaw.ai/cli/agent`, `docs.openclaw.ai/tools/agent-send`, and focused adapter/CLI/protocol tests. This should stay in core rather than ClawHub because the missing behavior is the shared agent execution pipeline, not an optional plugin capability.

## Risks / Open Questions

- A gateway-capable implementation should use request-scoped state, not a process-global flag, to avoid cross-run leakage between concurrent agent requests.
- Tool preview logging needs to reuse existing redaction and console sanitization before writing params to stderr.
- Dry-run interception should happen after before-tool-call policy blocks so preview mode does not bypass safety policy.
