## Summary

Keep this issue open. Current main still lacks a global `openclaw agent --dry-run` path: the agent CLI does not expose the flag, Gateway agent params do not accept or forward `dryRun`, embedded agent options do not carry dry-run state, and the Pi tool adapters still execute managed tools or delegate client tools rather than returning structured dry-run previews. The related dry-run items are narrower command/workflow requests and do not supersede this core agent execution request.

## Best Possible Solution

Implement this as core agent execution work, not stale cleanup. Add an `openclaw agent --dry-run` user surface, thread request-scoped dry-run state through CLI, Gateway protocol, embedded `AgentCommandOpts`, and the Pi tool-definition adapters, intercept managed and client tool calls after policy hooks, return structured `dry_run` payloads to the model, log redacted previews to stderr, and add focused adapter, CLI/protocol, docs, and regression coverage. Keep the narrower dry-run issues separate unless a maintainer creates a canonical umbrella.

## Risks / Open Questions

- A naive process-global implementation would be unsafe for Gateway or concurrent embedded runs; the dry-run state should be request-scoped.
- Preview logging must use existing redaction and console sanitization before printing tool parameters to stderr.
- Dry-run interception should stay after trusted policy and `before_tool_call` hooks so preview mode does not bypass safety policy.
