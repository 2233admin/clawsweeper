## Summary

Current main still has the two reported routing gaps: `resolveGroupSessionKey()` reads the first `From` segment as the provider, and restart-sentinel delivery can still merge base-session routing into partial thread metadata. Same-author PR #49750 is still open as the active implementation candidate, so this issue should stay open for maintainer review rather than be auto-closed.

## Best Possible Solution

Keep this issue open and review/land #49750 or an equivalent maintainer patch. The durable fix should make `resolveGroupSessionKey()` reuse or mirror the agent-aware session conversation parser, ensure parsed Slack thread targets and explicit A2A delivery context win over base fallback, and add focused regression coverage for agent-prefixed Slack thread metadata plus restart/announce routing.
