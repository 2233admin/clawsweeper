## Summary

Keep this issue open. Current main still has the inline-only Control UI composer and does not implement the requested Expand button, large draft modal, Apply/Cancel/Send workflow, or modal-specific Enter-newline plus Ctrl/Cmd+Enter-send behavior. The provided related items are adjacent composer/session/modal work rather than a canonical replacement, and the prior review comment in the issue also kept it open.

## Best Possible Solution

Keep this open for first-class Control UI work. The likely implementation belongs in the chat composer surface: add an Expand icon near existing composer controls, introduce a large-editor modal with separate draft state, support Close/Cancel, Apply, and Send semantics, make Enter insert newlines while Ctrl/Cmd+Enter sends inside the modal, and cover draft ownership, queued sends, attachments, focus/dialog behavior, and keyboard behavior in focused UI tests.
