## Summary

Keep this issue open. Current main still has an inline-only Control UI chat composer with one bounded textarea, no Expand button, no large composer modal, no Apply/Cancel/Send modal workflow, and no modal-specific Enter-newline plus Ctrl/Cmd+Enter-send behavior. The existing issue discussion already contains a recent maintainer/Codex review that reached the same conclusion, and the adjacent related items are about other composer/session/modal surfaces rather than a canonical implementation of this request.

## Best Possible Solution

Keep this open and implement it as focused Control UI work: add an Expand icon in the chat composer toolbar, introduce a large-editor modal with separate draft state, support Close/Cancel, Apply, and Send semantics, make Enter insert newlines while Ctrl/Cmd+Enter sends inside the modal, and cover draft, attachments, queued sends, focus/dialog behavior, and keyboard behavior in focused UI tests.

## Clownfish Work Prompt

Implement the expanded chat composer requested in openclaw/openclaw#16711. Add an Expand icon/button to the Control UI chat composer near the existing toolbar controls. Opening it should show a large editor modal seeded from the current draft. The modal must keep separate draft state until the user chooses Apply or Send. Close/Cancel should dismiss without changing the inline draft. Apply should copy the modal text back to the inline composer without sending. Send should apply the modal text and submit it through the existing chat send path. Inside the modal, Enter inserts a newline and Ctrl+Enter/Cmd+Enter sends. Preserve existing inline composer behavior, including Enter-to-send, attachments, queued sends, abort/busy state, focus restoration, and disabled/connected handling. Keep changes scoped to the Control UI; do not change Gateway chat.send semantics or unrelated session behavior. Add focused UI tests for render controls, modal draft ownership, Apply/Cancel/Send, and keyboard behavior. Add a user-facing changelog entry under Unreleased Changes with Thanks @dragidude.
