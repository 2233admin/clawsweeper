## Summary

Keep this PR open. Current main still routes `openai-codex-responses` system prompts through top-level `instructions` while assigning the converted message array directly to `input`, and it has no local guard for the empty-message case. The PR is a narrow implementation candidate for #73820, but maintainers should still decide whether fail-fast is the desired behavior versus synthesizing a valid input item for bootstrap-style runs.

## Best Possible Solution

Review this PR as the current implementation candidate for #73820. If maintainers want fail-fast behavior, land the narrow guard, regression tests, and changelog entry after normal CI. If the desired behavior is to preserve bootstrap-style runs, revise the fix to create a valid Codex input item without moving the system prompt out of top-level `instructions`, and keep the linked issue open until that implementation merges.
