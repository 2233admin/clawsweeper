## Summary

Keep open. Current main still has the reported Telegram token replacement path wired through `Modify settings`, and the token prompt still calls `.trim()` directly on the text prompt result. The broader setup wizard text-input path has the same direct `rawValue.trim()` pattern, while the two related fix PRs called out in the issue context are closed unmerged.

## What This Changes

The issue asks for `openclaw onboard` and `openclaw channels add` to replace an existing Telegram bot token without crashing when the text prompt result is nullish.

## Best Possible Solution

Normalize text prompt results before any string method is called in the setup wizard paths, with focused regression coverage for replacing an existing Telegram token and for the shared generic text-input path. Keep token storage semantics unchanged and preserve explicit cancel handling.
