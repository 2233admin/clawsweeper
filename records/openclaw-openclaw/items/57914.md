## Summary

Keep #57914 open. Current main still does not implement `agents.defaults.replyMode` or an equivalent tool-only delivery gate, and the PR remains a focused implementation attempt for an active auto-delivery leak problem rather than a closeable duplicate or shipped change.

## Best Possible Solution

Keep the PR open and ask for a refreshed, complete version or extract a maintainer patch. The best path is to land or fold in the exec-approval-unavailable marker, add the generated config/docs/changelog updates, and validate one shared `agents.defaults.replyMode: "tool-only"` behavior across non-ACP and ACP paths with tests covering block text, final text, tool summaries, explicit `message` sends, media/error/interactive/channelData finals, and exec approval pending/unavailable payloads.
