## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before the Mattermost post handler can run mention detection or wake routing, and the linked closing PR is still open and unmerged, so this issue should remain open until that PR or an equivalent focused fix lands.

## What This Changes

The issue requests Mattermost edit events to be treated as updated post events so adding an @mention via message edit can trigger the normal agent wake flow.

## Best Possible Solution

Land PR #72513 or an equivalent Mattermost-only fix that treats `post_edited` as a post-bearing WebSocket event, runs the updated post through the existing mention gate, and makes replay suppression aware of edit revisions where needed.

## Reproduction Assessment

Yes. The issue gives concrete Mattermost steps, and current main has a deterministic source-level reproduction: a `post_edited` payload reaches `monitor-websocket.ts` and returns before `onPosted`, so mention detection never runs.

## Solution Assessment

Yes. The narrow maintainable solution belongs in the Mattermost plugin WebSocket/replay path; there is no evidence that a core channel API change is needed for this bug.
