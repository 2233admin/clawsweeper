## Summary

Keep open. Current main still rejects Mattermost edited-post WebSocket events before the existing post, mention, and wake flow can see the updated message body, and PR #72513 is an open, unmerged implementation candidate that directly targets the bug.

## What This Changes

The issue reports that the bundled Mattermost plugin ignores `post_edited` WebSocket events, so @mentions added by editing an existing Mattermost message do not wake the tagged agent.

## Best Possible Solution

Land PR #72513 or an equivalent Mattermost-only fix that treats `post_edited` as a post-bearing event, routes the updated post through the existing mention gate, and makes replay dedupe safe for edited-post revisions.

## Reproduction Assessment

Yes. The issue gives concrete Mattermost steps, and the checked-out main tree has a deterministic source-level reproduction: a `post_edited` payload returns before `onPosted`, so the edited body cannot reach mention detection.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the bundled Mattermost WebSocket and replay path; current source and docs do not show a need for a core channel API change.
