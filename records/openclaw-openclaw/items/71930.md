## Summary

Keep open. Current main still filters Mattermost `post_edited` WebSocket events before the existing post, mention, and wake path can run, and the linked implementation PR #72513 is open and unmerged, so this issue should remain open for maintainer review of that fix path.

## What This Changes

Handle Mattermost `post_edited` WebSocket events so @mentions added by editing an existing message can trigger the same agent wake flow as original `posted` messages.

## Best Possible Solution

Review and land PR #72513, or an equivalent Mattermost-only patch, so `post_edited` is treated as a post-bearing WebSocket event, the updated body reaches the existing mention gate, and replay handling allows edited revisions without duplicate-waking unchanged edits.

Yes. A synthetic `post_edited` WebSocket payload on current main follows a deterministic early return in `monitor-websocket.ts` before `onPosted`, and the issue provides concrete live Mattermost steps confirmed on April 25, 2026.
