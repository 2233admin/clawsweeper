## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before they can reach the existing post, mention, and wake path, and the linked fix PR #72513 remains open and unmerged.

## What This Changes

The issue asks for the Mattermost plugin to evaluate edited-message WebSocket events so @mentions added by editing a post can wake the tagged agent.

## Best Possible Solution

Land PR #72513 or an equivalent Mattermost-only fix that treats `post_edited` as a post-bearing event, feeds the updated post through the existing mention gate, and makes replay handling safe for edited revisions.
