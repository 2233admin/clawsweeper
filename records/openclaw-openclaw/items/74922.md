## Summary

Keep open: current main still suppresses Discord ack/status reactions for guild/channel source turns resolved as `message_tool_only`, including the reported `ackReactionScope: "all"` configuration. The report is valid, but changing the gate would alter the v2026.4.27 visible-output privacy contract for always-on rooms, so it needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue requests that configured Discord `ackReaction` fire for inbound guild/channel messages after upgrading to v2026.4.27, despite channel-resolution failures and `ackReactionScope: "all"`.

## Best Possible Solution

Define the Discord/group-channel contract explicitly. If configured ack reactions are allowed in tool-only rooms, narrow the Discord ack/status reaction gate and update focused tests and docs; if they are not allowed, document the interaction between `ackReaction`, `ackReactionScope`, and `messages.groupChat.visibleReplies`, and add clearer logging or doctor guidance when reactions are suppressed by source reply mode.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists on current main: a guild/channel route with `ackReactionScope: "all"` and `ackReaction: "👀"` currently resolves to `message_tool_only` and expects no Discord reaction call.

## Solution Assessment

Unclear. Letting `ackReactionScope: "all"` bypass tool-only suppression would address the report, but it may conflict with the shipped v2026.4.27 privacy behavior that intentionally suppresses automatic status-reaction output in always-on rooms.
