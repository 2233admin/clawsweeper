## Summary

Keep open: current main still suppresses Discord ack/status reactions for guild/channel source turns when the reply mode is `message_tool_only`, even when `ackReactionScope: "all"` would otherwise pass the ack gate. The behavior appears tied to the intentional privacy change from #73046, so this needs maintainer product judgment rather than conservative cleanup closure.

## What This Changes

The issue asks for Discord `ackReaction` to fire in guild channels when `ackReaction: "👀"` and `ackReactionScope: "all"` are configured on v2026.4.27/current main.

## Best Possible Solution

Decide the Discord product contract, then either let explicit ack/status reaction config override the tool-only visibility gate with narrow tests, or keep the current behavior and update Discord/group docs plus logging so operators know to use `messages.groupChat.visibleReplies: "automatic"` for visible ack/status reactions.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: the Discord process test sets a guild/channel route with `ackReaction: "👀"` and `ackReactionScope: "all"`, then verifies no Discord reaction call because the source reply mode is `message_tool_only`.

## Solution Assessment

Unclear. Treating `ackReactionScope: "all"` as an explicit visibility opt-in would solve this report, but it may conflict with the shipped privacy behavior from #73046; a maintainer should choose between a behavior change and docs/logging clarification.
