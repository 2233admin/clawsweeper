## Summary

Keep open: current main still suppresses Discord ack and status reactions for guild/channel source turns resolved as `message_tool_only`, including the reported `ackReactionScope: "all"` configuration. Source inspection also shows the reported channel-resolution failure falls back to configured guild entries, so the remaining question is the privacy/product contract for configured reactions in tool-only group rooms rather than a simple resolution-drop fix.

## What This Changes

The report requests restoring automatic Discord `ackReaction` delivery in guild channels after the v2026.4.27 group/channel visible-output privacy change.

## Best Possible Solution

Define the Discord group/channel contract explicitly. If configured ack reactions should be visible in tool-only rooms, narrow the Discord ack/status reaction gate and update the focused tests; if not, document the `ackReaction`/`ackReactionScope` interaction with `messages.groupChat.visibleReplies` and add clearer suppression logging.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists on current main: a guild/channel route with `ackReactionScope: "all"` and default group visible replies resolves to `message_tool_only` and asserts no Discord reaction call.

## Solution Assessment

Unclear. Letting `ackReactionScope: "all"` bypass tool-only suppression would address the report, but it may conflict with the shipped privacy behavior that suppresses automatic visible output in always-on group/channel rooms.
