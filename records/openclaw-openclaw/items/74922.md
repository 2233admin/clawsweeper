## Summary

Keep open: current main still suppresses Discord ack/status reactions for guild/channel source turns that resolve to `message_tool_only`, including the reported `ackReactionScope: "all"` shape. The channel-resolution failure is useful live context, but source inspection shows that fallback returns configured guild entries; the unresolved decision is whether configured ack/status reactions should bypass the v2026.4.27 group-channel visible-output privacy default.

## What This Changes

The issue requests that Discord guild-channel automatic `ackReaction` fire again for configured `ackReaction: "👀"` and `ackReactionScope: "all"` setups after upgrading to v2026.4.27, including when REST channel resolution fails.

## Best Possible Solution

Define the Discord/group-channel contract explicitly. If configured ack/status reactions are allowed in tool-only rooms, narrow the Discord reaction gate and update the matching tests; if they are not allowed, document the interaction between `ackReaction`, `ackReactionScope`, `statusReactions`, and `messages.groupChat.visibleReplies`, and add clearer logging when reactions are suppressed by source reply mode.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists on current main: a guild/channel route with `ackReactionScope: "all"` and `ackReaction: "👀"` currently resolves to `message_tool_only` and expects no Discord reaction call.

## Solution Assessment

Unclear. Letting `ackReactionScope: "all"` bypass tool-only suppression would address the report, but it may conflict with the shipped v2026.4.27 policy that keeps automatic visible output private in always-on group/channel rooms.
