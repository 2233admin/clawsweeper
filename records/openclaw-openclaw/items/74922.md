## Summary

Keep open: current main still suppresses Discord ack/status reactions for guild/channel source turns when the reply mode is `message_tool_only`, and the issue’s config shape matches an existing regression test that expects no reaction call. The linked #73133 fix is a separate `MESSAGE_REACTION_ADD` notification path, so this needs maintainer product judgment rather than conservative closure.

## What This Changes

The issue requests restoring automatic Discord `ackReaction` behavior in guild channels after v2026.4.27 when `ackReactionScope: "all"` and per-channel user allowlists are configured.

## Best Possible Solution

Decide whether explicit Discord `ackReactionScope: "all"` should opt into visible ack/status reactions even when normal group/channel replies are tool-only. If yes, adjust the Discord monitor gate and tests narrowly; if no, document the `messages.groupChat.visibleReplies: "automatic"` workaround and add clearer logging/docs around suppressed ack reactions.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction already exists: the Discord process test sets a guild route with `ackReactionScope: "all"` and verifies no reaction call because the source reply mode is `message_tool_only`.

## Solution Assessment

Unclear. Treating all configured ack reactions as visible output would solve this report, but it may conflict with the shipped private-by-default group/channel behavior from #73046; the safer next step is a maintainer decision on that contract.
