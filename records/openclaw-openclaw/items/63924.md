## Summary

Keep this PR open. The Feishu schema hunk is already present on current main, but the PR is not fully obsolete: the health fix still needs a current-main rebase/test, while the provider-normalization and skill-symlink changes conflict with current provider identity and realpath containment contracts.

## What This Changes

The PR replaces Feishu's hand-written channel config schema, changes health snapshots to read best-effort config, normalizes Volcengine/BytePlus plan provider IDs to their base providers, and allows symlinked skills to fall back to unresolved paths when realpath containment fails.

## Best Possible Solution

Ask for a rebase and split. Drop the Feishu schema hunk, make the health fix a focused current-main change with a pinned-runtime regression test, preserve plan provider IDs while relying on manifest auth aliases, and handle symlink canonicalization separately with security-focused escape and allowed-alias coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Bypass the pinned config snapshot in health:** `src/commands/health.ts:353-354`
  - body: On the submitted branch, `readBestEffortConfig()` returns `loadConfig()` for valid files, so this still reads the gateway's pinned runtime snapshot in the normal case and does not fix stale channel configuration. The fallback is also unreachable because the helper returns an object.
  - confidence: 0.86
- **[P2] Keep plan provider ids distinct:** `src/agents/model-selection.ts:94-100`
  - body: Collapsing `volcengine-plan` and `byteplus-plan` in provider normalization makes catalog/runtime lookups treat coding-plan providers as the base providers. Current manifests give those plan providers separate IDs, base URLs, and model catalogs, sharing only auth through `providerAuthAliases`.
  - confidence: 0.89
- **[P1] Preserve realpath containment for skill roots:** `src/agents/skills/workspace.ts:212-219`
  - body: When a symlink under the skills root resolves outside the root, this returns the unresolved path because the textual path is inside the root. Subsequent loading follows the symlink and can load an outside `SKILL.md`, breaking the current containment boundary and symlink-escape tests.
  - confidence: 0.9
