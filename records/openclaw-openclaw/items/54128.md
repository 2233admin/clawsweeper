## Summary

Current main still does not implement the requested `memorySearch.local.maxThreads` setting. The local config schema, resolved memory config, memory embedding provider option types, node-llama shim, runtime call, and public docs all omit it, while the pinned node-llama-cpp API supports `maxThreads` on `getLlama` options.

## What This Changes

The issue requests a configurable `memorySearch.local.maxThreads` option for local node-llama-cpp embeddings so users can tune CPU thread usage during memory indexing.

## Best Possible Solution

Add a narrow optional `memorySearch.local.maxThreads` config field, validate it consistently with the node-llama-cpp contract, carry it through default/per-agent memory config resolution and memory embedding provider create options, and pass it to `getLlama` only when configured. Update focused tests and generated/public config docs so the local embedding tuning surface is complete.
