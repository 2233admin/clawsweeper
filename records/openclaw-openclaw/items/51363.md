## Summary

Keep open. Current main still derives Docker sandbox and sandbox-browser container names from the sandbox scope key alone, so separate HOME/workspace-root instances using the default main agent can still collide on `openclaw-sbx-agent-main-f331f052`. The related fix PR #51380 is closed unmerged, and the current sandbox history since the prior review does not show a naming fix.

## Best Possible Solution

Keep this issue open and implement a core Docker sandbox fix in `src/agents/sandbox/docker.ts` and `src/agents/sandbox/browser.ts`. The container-name slug input should include a stable instance discriminator such as resolved OpenClaw state/workspace root or HOME-derived identity, while preserving per-scope reuse inside the same instance. Add regression tests proving two different HOME/workspace roots with the same default `main` agent produce different Docker and browser container names, and document the one-time container recreate/orphan behavior.
