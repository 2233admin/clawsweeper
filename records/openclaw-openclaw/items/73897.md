## Summary

Keep this PR open. Current main still sends Android node-host shell wrappers through the generic POSIX `/bin/sh -lc` path, while the PR is a narrow targeted change that appears correct for Android/Termux nodes once `system.run` is allowed for that node. The remaining maintainer question is scope/docs around Android command allowlisting, not a reason to close the PR.

## What This Changes

The PR adds an Android/Termux shell path for `buildNodeShellCommand`, extends the unit test for normalized Android platform labels, and documents the node-host shell behavior in the node CLI and nodes docs.

## Best Possible Solution

Review and land this narrow wrapper/test/docs change or an equivalent patch, with maintainer attention to whether the Android docs should mention the `gateway.nodes.allowCommands` boundary or whether headless Android node hosts should get a distinct default policy from Android mobile nodes.

## Reproduction Assessment

Yes. A helper-level reproduction is high confidence: on current main, `buildNodeShellCommand("echo hi", "android")` follows the non-Windows fallback and returns `/bin/sh -lc`; a live Termux end-to-end run was not performed in this read-only review.

## Solution Assessment

Yes for the wrapper failure after an Android node is allowed to expose `system.run`. The patch is the narrowest code change for shell selection, but the best maintainer outcome should also settle the Android command-allowlist/docs wording before merge.
