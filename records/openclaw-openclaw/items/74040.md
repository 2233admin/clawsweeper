## Summary

Keep open. Current main still lacks the PR’s secondary content-quality guard, the linked bug remains open, and the PR is an active implementation candidate that needs maintainer review because the current diff can affect runtime-only prompt semantics and adds no focused regression coverage.

## What This Changes

The PR changes `runEmbeddedAttempt` to make `promptSubmission` mutable, detect whether replay history contains non-empty user text, and inject a safe greeting when prompt, images, and substantive user history are otherwise empty.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best fix is a conservative embedded-runner dispatch-boundary guard, preferably factored into a helper with tests for blank replay content, image-only submissions, runtime-only context, and active-memory stale-history reset behavior. The final shape should prevent empty provider payloads without changing hidden runtime-context semantics or documented bare `/new`/`/reset` behavior unless maintainers explicitly choose that product behavior.
