## Summary

Keep open. Current main still rejects Gateway `agent` requests on `session.sendPolicy=deny` before checking whether the request actually asked for delivery, and the issue is paired with open closing PR #73500, so it should wait for that PR or an equivalent fix to merge.

## What This Changes

Allow non-delivery Gateway agent invocations to run when `session.sendPolicy=deny`, while preserving the deny check for explicit external delivery.

## Best Possible Solution

Land PR #73500 or an equivalent focused change in `src/gateway/server-methods/agent.ts` that only enforces `sendPolicy=deny` when `request.deliver === true`, with regression coverage for non-delivery allow and explicit-delivery denial.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a Gateway `agent` request with `deliver` omitted or false and an effective `sendPolicy` of `deny` reaches the unconditional rejection before `wantsDelivery` is computed.

## Solution Assessment

Yes. Gating the Gateway policy check on `request.deliver === true` is the narrowest maintainable fix because it matches the embedded agent path while preserving the outbound-delivery safety check.
