## Summary

Keep open. Current main still enforces `resolveSendPolicy(...)` in the Gateway `agent` RPC before delivery planning, so a session-level `sendPolicy: "deny"` can reject a non-delivery `openclaw agent` request before the code computes `wantsDelivery` or the final `deliver` value. That matches the reported failure and is inconsistent with the embedded agent path and CLI/docs, where `--deliver` is optional and send policy is only applied when delivery is requested.

## Best Possible Solution

Fix `src/gateway/server-methods/agent.ts` so session send policy is enforced only when the Gateway agent request will actually deliver externally. Add focused regression coverage for `deliver:false` with `sendPolicy:deny` reaching the agent dispatch path, plus `deliver:true` remaining blocked. The earlier ClawSweeper comment on this issue already identified the same active gap, and current main still matches that analysis.
