## Summary

Keep open. Current main still rejects Gateway `agent` requests on `sendPolicy=deny` before computing whether the request wants delivery, while the embedded agent path already limits that policy check to explicit delivery. The issue is also paired with open, unmerged PR #73500, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for non-delivery Gateway `agent` invocations to run even when `session.sendPolicy=deny`, while preserving denial for actual outbound delivery.

## Best Possible Solution

Land PR #73500 or an equivalent focused change in `src/gateway/server-methods/agent.ts` that applies `sendPolicy=deny` only when `request.deliver === true`, with regression tests for both non-delivery allow and explicit-delivery denial.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a Gateway `agent` request for a session resolving to `sendPolicy=deny` with `deliver` omitted or false reaches the rejection before `wantsDelivery` is computed.

## Solution Assessment

Yes. Gating the Gateway policy check on `request.deliver === true` is the narrowest maintainable fix because it mirrors the embedded agent path and preserves the outbound-delivery safety check.
