## Summary

Keep open. Current main still enforces Gateway agent `resolveSendPolicy(...)` before computing whether the request actually wants external delivery, so a non-delivery `openclaw agent` request can still be rejected with `send blocked by session policy`. The linked open PR #73500 has explicit closing syntax and targets this exact behavior, so this issue should stay open until that PR, or an equivalent fix, is reviewed and merged.

## Best Possible Solution

Review and land #73500, or an equivalent change in `src/gateway/server-methods/agent.ts`, so session send policy is enforced only when the Gateway agent request will actually deliver externally. Add focused regression coverage proving `deliver: false` with `sendPolicy: deny` reaches agent dispatch, while `deliver: true` remains blocked.
