## Summary

Keep open. Current main still rejects Gateway `agent` requests on `sendPolicy=deny` before it computes whether the request is actually delivering externally, and the issue is already paired with open, unmerged PR #73500 for the focused fix.

## What This Changes

The issue asks for Gateway `agent` RPC calls without external delivery to bypass outbound `session.sendPolicy=deny` blocking while preserving the block for explicit delivery.

## Best Possible Solution

Land PR #73500 or an equivalent focused change in `src/gateway/server-methods/agent.ts` that enforces `sendPolicy=deny` only when `request.deliver === true`, with regression coverage for both non-delivery allow and explicit-delivery denial.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a Gateway `agent` request whose session resolves to `sendPolicy=deny` with `deliver` omitted or false reaches the rejection at `src/gateway/server-methods/agent.ts:1049` before `wantsDelivery` is computed.

## Solution Assessment

Yes. Gating the Gateway policy check on `request.deliver === true` is the narrowest maintainable fix because it mirrors the embedded agent path and preserves the outbound-delivery safety check.
