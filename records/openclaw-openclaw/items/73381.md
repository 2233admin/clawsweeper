## Summary

Keep open. Current main still matches the reported bug: the Gateway `agent` RPC resolves `sendPolicy` and returns `send blocked by session policy` before computing whether the request actually wants external delivery, while the embedded agent path gates that policy only when `deliver` is true. The linked #73500 PR specifically targets this issue but remains open in the provided context, so this issue should not be closed before that PR or an equivalent fix merges.

## What This Changes

The issue requests that Gateway agent invocations only enforce `session.sendPolicy=deny` for explicit external delivery, not for non-delivery internal agent runs.

## Best Possible Solution

Review and land #73500, or replace it with an equivalent focused patch in `src/gateway/server-methods/agent.ts` that only enforces `sendPolicy=deny` when Gateway `agent` requests explicitly deliver externally, plus regression tests for `deliver: false` allowed and `deliver: true` blocked.
