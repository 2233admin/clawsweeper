## Summary

Keep open. Current `origin/main` still routes `cron.sessionRetention` through a cron-run-only reaper, while default keep-mode subagent sessions and ACP sessions do not have an equivalent durable retention sweep. The related fix PR is closed unmerged, and the duplicate/adjacent reports point back to this as the canonical tracker rather than proving the gap is resolved.

## What This Changes

The issue asks for `cron.sessionRetention` or an equivalent retention path to prune completed subagent and ACP session entries and associated transcripts instead of pruning only isolated cron run sessions.

## Best Possible Solution

Keep this as the canonical tracker and design a restart-safe session-store-level retention path for completed native subagent and ACP run sessions. The end state should preserve active and intentionally persistent sessions, use durable terminal markers, delete or archive through existing session deletion and transcript archive helpers, run on startup or another catch-up path, and document the owning retention config.
