## Summary

Keep open. Current main still carries the same ACPX and ACP SDK boundary that the prior review identified: the bundled ACPX plugin remains on `acpx` 0.6.1, OpenClaw still depends on `@agentclientprotocol/sdk` 0.20.0, and the published SDK schema still requires numeric `usage_update.used` before ACPX's lenient projection can treat nullable usage fields as optional.

## What This Changes

The issue requests that OpenClaw accept ACP `session/update` `usage_update` notifications when usage fields such as `used` are `null` or omitted.

## Best Possible Solution

Align the ACPX runtime boundary with the ACP SDK contract: normalize or drop nullable `usage_update` fields before SDK validation, or consume a compatible upstream ACPX/SDK release, then add a regression test that exercises the Discord-bound ACPX runtime path with `used: null`.
