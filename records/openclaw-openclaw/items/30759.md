## Summary

Keep open. Current main has implemented the WebChat/Control UI upload transport for non-video files and can stage PDF/TXT-like files into the media path, but the central request also asks for Word document text extraction. Current main still declares only PDF extraction in the bundled document extractor, the default input-file allowlist includes text/json/pdf but not Word MIME types, and the media-understanding file-block path skips `application/vnd.*` Office MIME before extraction. Related #57612 is a useful open DOCX extraction candidate, but it is not on main and its current patch targets `input_file`/`read`, not the WebChat media-understanding skip path.

## Best Possible Solution

Keep this open and narrow the remaining work to document extraction policy: review/land or adapt #57612 for DOCX, wire DOCX through the WebChat media-understanding file-block path if maintainers want automatic extraction, explicitly reject or defer legacy `.doc` unless a safe extractor is approved, and align docs/tests across Control UI uploads, Gateway staging, input-file extraction, media-understanding, and document-extract.
