## Summary

Keep this PR open. Current main still does not implement `--profile-id` for `openclaw models auth login`, #40402 remains the canonical open user problem, and this PR is still relevant. It needs maintainer revision because the proposed generic post-login profile remap can run after provider-owned auth flows have already persisted credentials, so the target profile needs to be carried through a typed provider auth path before persistence.

## Best Possible Solution

Keep this PR open for maintainer revision or replacement. The best fix is to add the login option, carry the requested profile target through a typed provider auth context before credentials are persisted, have OpenAI Codex OAuth and device-code login honor that target inside the provider, validate provider prefix and collision behavior, avoid generic post-login rewrites, and update public docs plus CLI/provider regression tests for #40402.
