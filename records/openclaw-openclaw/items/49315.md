## Summary

Keep this PR open. Current checked-out main does not implement `--profile-id` for `openclaw models auth login`, and a read-only spot-check of latest `origin/main` showed the same missing surface. The related tracker #40402 is still open, so this PR remains an implementation candidate, but its generic post-login profile rewrite needs maintainer revision because some provider auth flows persist credentials before returning their `ProviderAuthResult`.

## Best Possible Solution

Keep this PR open for maintainer revision or review. The best implementation is to add the login flag, pass the requested profile target through provider auth context/options, validate that the requested id belongs to the selected provider, let supported providers such as OpenAI Codex honor it before persistence, avoid generic post-login profile rewrites, define collision and multi-profile behavior, and update public docs plus CLI/provider regression tests. Link the final implementation to #40402.
