## Summary

Keep this issue open. Current main still has `_clawdock_compose()` pass explicit `-f` files for `docker-compose.yml` and optional `docker-compose.extra.yml` only, so a user-created `docker-compose.override.yml` remains ignored by helper-driven starts/recreates. The latest release tag shows the same behavior, and Docker Compose's documented merge behavior supports the report.

## Best Possible Solution

Keep this open and fix `scripts/clawdock/clawdock-helpers.sh` so ClawDock includes `docker-compose.override.yml` when present, with documented precedence. The likely best order is base `docker-compose.yml`, generated `docker-compose.extra.yml`, then user-local `docker-compose.override.yml`, unless the sandbox overlay work in #59523 requires a different final ordering for Docker socket safety. Update ClawDock/Docker docs and add focused test coverage for the constructed compose file list.
