## Summary

Keep open. Current main still has no native exec-approvals deny surface that runs before broad allowlist/full decisions and matches full command content; the implementation, protocol schema, CLI/UI, and docs remain allowlist-oriented. The broader #6615 tracker and the closed unmerged #47848 attempt are relevant, but they do not make this focused shell-wrapper bypass report implemented or safely obsolete.

## What This Changes

The issue requests pre-allowlist exec approval deny patterns that match full command strings so dangerous shell-wrapped commands cannot be allowed solely because the shell binary is allowlisted.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, consistently matches canonical command text plus argv and shell payload across gateway and node execution, exposes the policy in approvals CLI and Control UI, documents precedence and limitations, and coordinates this focused tracker with #6615.
