## Summary

Keep this PR open for maintainer review. The latest patch is not ready to merge: the heartbeat task-block regex now stops at the end of the `tasks:` line under the multiline flag, the default-heartbeat enablement change conflicts with checked-in test expectations, and the runtime logger migration hides fatal error details from normal console output. The security pass did not find workflow, lockfile, dependency, package-script, or release-metadata changes.

## What This Changes

The PR changes heartbeat directive stripping and default heartbeat enablement, and migrates warnings/errors in command auth, config defaults, runtime error handlers, dotenv, temp-download, and Windows ACL helpers to `createSubsystemLogger`.

## Best Possible Solution

Revise the PR so the task-block removal preserves non-task HEARTBEAT.md directives while consuming the complete `tasks:` block, align heartbeat docs and tests with the intended global-defaults contract, and preserve console-visible formatted error text when moving fatal runtime paths to subsystem logging.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Strip the full tasks block from heartbeat context:** `src/infra/heartbeat-runner.ts:746`
  - body: With the multiline flag, the `$` branch in `(?=^[^\s]|\n\n|$)` matches the end of the `tasks:` line, so this replacement can remove only `tasks:` and leave the YAML task entries in `directives`. Any HEARTBEAT.md with tasks then appends raw task definitions to Additional context instead of stripping them.
  - confidence: 0.97
- **[P2] Update the heartbeat enablement contract tests:** `src/infra/heartbeat-summary.ts:41-43`
  - body: The new defaults check makes `isHeartbeatEnabledForAgent` return true for every agent when `agents.defaults.heartbeat` exists and no per-agent heartbeat blocks are configured. The checked-in test for that exact config still expects `ops` to be disabled, so the PR will fail the heartbeat test suite until the intended contract and coverage are updated together.
  - confidence: 0.9
- **[P2] Keep fatal error details in console output:** `src/index.ts:103-112`
  - body: These handlers used to print `formatUncaughtError(err)` directly. `createSubsystemLogger` only renders `message` or `consoleMessage` in pretty and compact console modes, so passing the Error only as `{ error }` makes CLI failures and uncaught exceptions print generic messages without the exception text or stack.
  - confidence: 0.88
