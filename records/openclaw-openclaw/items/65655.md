## Summary

Keep open. Current main still lacks both security changes this PR targets: Mattermost slash registration can still use a derived plain-HTTP callback URL, and callback token validation is still token-set based rather than bound to the registered slash trigger. The PR is also paired with the same author's open bug #65624 through closing syntax, so it should remain open for maintainer review instead of being auto-closed.

## What This Changes

This PR changes Mattermost docs, slash registration, HTTP callback handling, slash state, and tests to refuse insecure derived callback URLs and bind command tokens to their registered slash triggers.

## Best Possible Solution

Keep this PR open and either land it after addressing the remaining auth-policy gaps or replace it with a maintainer-equivalent Mattermost patch that rejects insecure derived callbacks, requires token-to-trigger binding for registered commands, and keeps the docs/tests aligned with #65624.
