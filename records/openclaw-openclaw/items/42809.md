## Summary

Keep this PR open. Current main still renders Feishu card-mode Markdown tables as raw markdown content, so the intended native table conversion is not implemented. The PR remains a useful candidate, but it is not ready to close or merge as-is because it only wires `buildMarkdownCard`, while current primary card paths also use structured cards, reply dispatch, outbound routing, and streaming-card updates.

## Best Possible Solution

Keep this PR open as an implementation candidate, but request a current-main rework. The best path is a shared Feishu Card JSON 2.0 table conversion helper that preserves `width_mode: "fill"`, integrates with `buildMarkdownCard`, `buildStructuredCard`/`sendStructuredCardFeishu`, reply-dispatcher, outbound, and streaming-card behavior where appropriate, protects both backtick and tilde fenced code blocks, adds table-element quote extraction, and coordinates fallback/table-limit semantics with #43690, #50429, and #70651.

## Risks / Open Questions

- Merging the PR as-is would leave `buildStructuredCard`, `sendStructuredCardFeishu`, reply-dispatcher delivery, and streaming-card updates rendering table-shaped Markdown as markdown-only card content.
- The PR's tests still expect legacy `wide_screen_mode: true`, while current main intentionally asserts `width_mode: "fill"`; the rework must preserve the current Card JSON 2.0 width contract.
- Native `table` elements need matching quoted-card extraction support, otherwise replies to table-only cards can lose useful context.
- The remaining review comments around tilde-fenced code block protection and table-limit fallback semantics should be resolved before merge, especially alongside #43690, #50429, and #70651.
