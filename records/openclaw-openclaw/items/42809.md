## Summary

Keep this PR open. Current main still does not implement the PR's intended Feishu Card JSON 2.0 native table conversion: table-shaped Markdown is still routed into card-mode paths that wrap it in markdown elements. The PR remains a useful implementation candidate, but it is incomplete against current main and has unresolved review concerns around non-`buildMarkdownCard` card paths, quoted-card extraction, fenced-code protection, and table-limit fallback coordination.

## Best Possible Solution

Keep this PR open as an implementation candidate, but request a current-main rework. The best path is a shared Feishu Card JSON 2.0 markdown-table conversion helper that preserves `width_mode: "fill"`, integrates with `buildMarkdownCard`, `buildStructuredCard`/`sendStructuredCardFeishu`, outbound, reply-dispatcher, and streaming-card behavior where appropriate, protects both backtick and tilde fenced code blocks, adds quote extraction for `table` elements, and coordinates table-limit fallback semantics with #43690, #50429, and #70651.
