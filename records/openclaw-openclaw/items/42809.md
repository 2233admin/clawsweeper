## Summary

Keep this PR open. Current main still does not implement native Feishu Card JSON 2.0 table conversion: Markdown-table-shaped text is still detected as card-worthy, but the card builders wrap it in markdown elements rather than producing Feishu table components. The PR remains a relevant implementation candidate, though it needs rework for the additional card paths and unresolved review concerns.

## Best Possible Solution

Keep this PR open and request a current-main rework. The best path is a shared Feishu Card JSON 2.0 markdown-table conversion helper that preserves `width_mode: "fill"`, integrates with `buildMarkdownCard`, `buildStructuredCard`/`sendStructuredCardFeishu`, outbound, reply-dispatcher, and streaming-card behavior where appropriate, protects backtick and tilde fenced code blocks, adds quote extraction for native table elements, and coordinates table-limit fallback semantics with #43690, #50429, and #70651.
