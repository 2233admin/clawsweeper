## Summary

Keep this PR open. Current main still has the exclusive `plugins.allow` behavior, and the live advisory/error surfaces named by the PR still do not consistently warn users that adding one plugin to `plugins.allow` disables unlisted plugins. The PR remains useful, but it is stale against current main because the CLI policy and security audit implementation moved to different files, and the final patch should add focused assertions rather than relying on substring-preserving tests.

## What This Changes

This PR updates four plugin allowlist advisory strings so they warn that `plugins.allow` is an exclusive allowlist that disables unlisted active plugins, including bundled plugins.

## Best Possible Solution

Update the PR for current main by moving the text changes to the active policy/audit files, keeping the safer explanatory wording that tells users to include every active plugin without generating a concrete allowlist, and pinning the four user-facing messages with targeted tests.
