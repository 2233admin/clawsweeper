## Summary

Keep open. Current main still does not add `--profile-id` or an equivalent operator-chosen OAuth alias to `openclaw models auth login`; only `paste-token` has that flag. The Codex OAuth identity path now improves automatic profile naming, but it does not satisfy the requested explicit profile naming workflow. #49315 remains the relevant open implementation candidate to review or supersede.

## Best Possible Solution

Keep this issue open as the tracker for explicit OAuth profile aliases. Review and land #49315 or equivalent work that adds `--profile-id <id>` to `models auth login`, carries it through login options and provider auth persistence, validates the requested id belongs to the selected provider, handles collisions and ambiguous multi-profile results safely, and updates CLI docs plus parser/persistence tests.
