## Summary

Keep this issue open. Current main still does not expose `--profile-id` or an equivalent operator-chosen OAuth profile alias on `openclaw models auth login`; only `paste-token` has that flag. The related Codex email-derived profile fix helps the overwrite bug, but it does not satisfy this explicit alias request, and open PR #49315 remains the active implementation candidate.

## Best Possible Solution

Keep this issue open as the tracker for explicit OAuth profile aliases. Review and land PR #49315 or equivalent work that adds `--profile-id <id>` to `models auth login`, threads the requested target through a typed provider/auth context before persistence, validates provider ownership and collisions, handles multi-profile auth results deliberately, and updates CLI docs plus parser/persistence/provider tests.
