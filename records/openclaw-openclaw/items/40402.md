## Summary

Keep open. Current main at d22a851253b0225e72847dc6b1a7b74d8035e214 still does not expose `--profile-id` or an equivalent operator-chosen OAuth profile alias on `openclaw models auth login`; only `paste-token` accepts `--profile-id`. The related OpenAI Codex identity-derived naming fix handles the overwrite bug for email/stable-subject profiles, but it does not satisfy this explicit alias feature request. PR #49315 remains the active implementation candidate and should be reviewed rather than closing this issue.

## Best Possible Solution

Keep this issue open as the tracker for explicit OAuth profile aliases. Review and land PR #49315 or equivalent work that adds `--profile-id <id>` to `models auth login`, threads it through login options and auth persistence, validates that the requested id belongs to the selected provider, handles collisions and multi-profile auth results safely, and updates CLI docs plus parser/persistence tests.
