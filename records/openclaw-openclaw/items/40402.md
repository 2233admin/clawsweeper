## Summary

Keep this issue open. Current main still does not implement `--profile-id` or an equivalent operator-chosen OAuth profile alias for `openclaw models auth login`. The related Codex OAuth identity work now improves automatic profile naming from email or stable token claims, but that is distinct from the requested explicit alias feature. The related open PR #49315 remains the active implementation candidate to review or supersede.

## Best Possible Solution

Keep this issue open as the tracker for explicit OAuth profile aliases. Review and land #49315 or equivalent work that adds `--profile-id <id>` to `models auth login`, threads it through login options and provider auth persistence, validates that the requested id belongs to the selected provider, handles collisions and multi-profile auth results safely, and updates CLI docs plus parser/persistence tests.
