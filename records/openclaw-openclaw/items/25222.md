## Summary

Keep this issue open. Current main still serializes/enqueues same-session contention and returns no immediate channel-visible busy/status reply with cancel or new-session guidance; the recent maintainer/Codex discussion also explicitly kept the item open for that remaining UX gap.

## Best Possible Solution

Keep this open and implement a generic active-session busy-reply path around active-run queue policy, runReplyAgent, and channel dispatch. It should return configurable channel-safe status text and action guidance for ordinary same-session contention, optionally expose reason codes, and add focused tests for queue policy, runner behavior, dispatch, and at least one bridge path.
