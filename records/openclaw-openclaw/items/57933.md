## Summary

Keep open. Current main has coarse session lifecycle plumbing and session change subscriptions, but it does not implement the requested per-session heartbeat/health model, granular states such as awaiting_model/tool_exec, stall/unresponsive detection, or Control UI live status indicators in the Sessions list. The related context is about channel/gateway health/status issues, not a canonical implementation of this feature.

## Best Possible Solution

Keep this open and implement it in the gateway/session runtime rather than treating it as solved by existing channel preview streaming. A good path is to add a structured per-session runtime health snapshot derived from chat/agent lifecycle, model stream, and tool execution events; expose it through sessions.list/sessions.changed or a dedicated sessions heartbeat RPC; add stall/unresponsive thresholds; then render compact badges/tooltips in the Control UI Sessions surface with focused protocol, gateway, and UI tests.
