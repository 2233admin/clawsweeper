## Summary

Keep this PR open. Current main still has the observable memory-lancedb failure mode: auto-capture passes raw user text into shouldCapture and stores that same raw text, while auto-recall fetches and injects the top three raw LanceDB results without envelope-sludge filtering. The PR is a focused two-file reimplementation of closed, noisy #67475, and its follow-up commit addresses the bot review concerns without touching workflows, dependencies, lockfiles, package metadata, or release scripts.

## Best Possible Solution

Keep this PR open for normal maintainer review and CI. The best path is to land a narrow memory-lancedb fix that sanitizes auto-captured user text before embedding/storage, filters contaminated recalled entries before prompt injection, keeps the sentinel logic aligned with the inbound metadata contract, and preserves the PR’s regression tests for the bot-review edge cases.
