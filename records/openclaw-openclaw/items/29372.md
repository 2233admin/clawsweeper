## Summary

Keep this issue open. Current main still does not implement `agents.list[].subagents.workspace`: the per-agent subagent config type and strict runtime/generated schemas omit the field, public docs only describe model/thinking/allowlist/requireAgentId defaults, and native omitted-`agentId`/same-agent spawns still preserve the caller workspace. The related cross-agent workspace inheritance work is present, but it solves explicit target-agent workspace selection, not a configurable default-subagent workspace override.

## Best Possible Solution

Keep this open for maintainer product/security decision. If approved, add a backwards-compatible `agents.list[].subagents.workspace` config key, normalize it like other agent workspace paths, update TypeScript/zod/generated schemas and public docs, and change native omitted-`agentId` plus explicit same-agent spawn workspace resolution to prefer that override while preserving explicit cross-agent target-workspace behavior.
