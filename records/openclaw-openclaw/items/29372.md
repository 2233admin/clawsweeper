## Summary

Keep this issue open. Current main still lacks `agents.list[].subagents.workspace`: the per-agent config type and strict runtime/generated schemas omit the key, and default/same-agent native subagent spawns still preserve the requester workspace. The related cross-agent workspace fix addresses target-agent isolation, but not this requested parent-configured default-subagent workspace override.

## What This Changes

Add `agents.list[].subagents.workspace` so a parent agent can set the workspace directory used by its default child subagents without creating separate child agent profiles.

## Best Possible Solution

Keep open for maintainer product/security review. If approved, add a backwards-compatible `agents.list[].subagents.workspace` key, normalize it like other agent workspace paths, update runtime/generated schemas and public docs, and make omitted-`agentId` plus explicit same-agent native subagent workspace resolution prefer that override while preserving current cross-agent target-workspace behavior.
