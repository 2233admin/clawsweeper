## Summary

Keep #57288 open. Current main supports WhatsApp outbound reactions and ack reactions, but the WhatsApp inbound pipeline does not parse Baileys reaction payloads or expose a WhatsApp reaction-notification config/path comparable to Telegram or Matrix.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin: parse Baileys `reactionMessage` payloads, add a documented WhatsApp inbound reaction notification mode analogous to Telegram/Matrix (`off`/`own`/`all` or the chosen product subset), apply existing WhatsApp access controls and routing, enqueue a stable system/inbound event with target message id, emoji, sender, and group context, and add unit tests plus WhatsApp docs/config updates.
