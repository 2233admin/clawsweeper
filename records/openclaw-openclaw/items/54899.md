## Summary

Keep open. Current main still lacks the Feishu Card Kit markdown sanitizer needed for the reported truncation, but this PR is not merge-ready because the proposed sanitizer can duplicate final streamed text and still misses tag-shaped and multi-backtick cases from normal Markdown output.

## What This Changes

This PR adds Feishu Card Kit streaming markdown sanitization and sanitizer tests, strips reasoning tags from partial Feishu stream updates, cleans card summaries, and falls back to post-format replies for markdown cards.

## Best Possible Solution

Keep the PR open and revise it so Feishu streaming cards use one canonical sanitized representation across partial updates and final closeout, escape unsafe tag-shaped angle-bracket text while preserving allowed Feishu tags, and protect all valid Markdown code span forms before angle-bracket escaping.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Sanitize final text before merging stream state:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` now stores sanitized content in `state.currentText`, but `close(finalText)` still merges raw final text. For `x < 10`, the partial state becomes `x \\< 10`; the raw final `x < 10` no longer overlaps, so close appends it and sends duplicated, unsanitized content.
  - confidence: 0.92
- **[P2] Escape tag-shaped unsafe angle brackets:** `extensions/feishu/src/streaming-card.ts:74`
  - body: The angle-bracket regex only catches `<` runs with no `>` before the end of the line. The linked truncation case includes tag-shaped text such as `<thinking>`, which is not whitelisted but remains unescaped and can still be parsed as an unclosed tag by Card Kit.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69`
  - body: The code-span placeholder only handles single-backtick delimiters. Valid Markdown spans delimited by multiple backticks leave their contents exposed to the angle-bracket escape, so examples like double-backtick `x < y` are rewritten with an inserted backslash.
  - confidence: 0.82
