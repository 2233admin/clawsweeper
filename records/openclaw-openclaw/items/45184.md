## Summary

Keep open. Current main has Matrix `allowBots` mention-only support, but that only gates messages sent by other configured Matrix accounts; it does not implement an opt-in pre-dispatch skip when a human message explicitly mentions a different configured bot account while this account has `requireMention: false`. The related #45450 thread was closed around bot-to-bot visibility, not this dispatch-suppression behavior.

## What This Changes

The issue requests an opt-in Matrix dispatch guard so a bot with general-message handling can skip messages that explicitly mention another bot account in the same OpenClaw instance.

## Best Possible Solution

Add a Matrix-owned opt-in room/account config that drops room messages before typing and LLM dispatch when they explicitly mention another configured Matrix account but not the current account, while keeping `allowBots` sender-policy semantics separate and updating schema, docs, generated metadata, and focused tests.

## Repair Work Prompt

Fix the Matrix multi-account dispatch issue where a bot configured to answer general room messages still dispatches when a human message explicitly mentions a different configured Matrix bot account in the same OpenClaw instance. Implement an opt-in Matrix-owned config on the account and/or room surface that drops room messages before typing/reply dispatch when the explicit mention target is another configured Matrix account and the current account was not mentioned. Keep existing `allowBots` behavior as sender policy for bot-authored messages; do not change Matrix DMs, @room handling, same-account self-loop filtering, or command handling for messages addressed to the current bot. Prefer Matrix-native `m.mentions.user_ids` plus the existing visible formatted-body safeguards; avoid trusting hidden metadata-only mentions. Update Matrix types/schema/config metadata/docs and add focused tests for: human mentions other configured bot with `requireMention: false` is skipped when opted in; current bot mention still dispatches; unmentioned general message still dispatches when opted in and no other bot is mentioned; bot-authored `allowBots` behavior remains unchanged. Add a user-facing changelog entry with reporter credit if repo policy requires it.
