## Summary

Keep open. Current main still handles Matrix bot filtering by sender identity and mention gating by the current account only, so a human-authored room message that explicitly mentions a different configured Matrix bot can still enter the reply dispatch path when this account has `requireMention: false`. The related #45450 work added `allowBots` for bot-authored messages, but that solves a different sender-visibility problem and does not provide the requested skip-on-other-bot-mention policy.

## What This Changes

Add an opt-in Matrix policy that skips dispatch for non-mentioned accounts when a room message explicitly mentions a different configured Matrix bot account.

## Best Possible Solution

Matrix should gain a documented, backward-compatible opt-in room/account policy that runs before typing, media work, session recording, and LLM dispatch; it should skip only when a visible mention targets another configured Matrix bot account while preserving self mentions, room mentions, DMs, command handling, and existing sender-based `allowBots` behavior.

## Repair Work Prompt

Fix the Matrix multi-account dispatch issue where a human-authored room message that visibly mentions Bot A can still dispatch to Bot B when Bot B has `requireMention: false`. Add a backward-compatible opt-in Matrix account/room policy, with a maintainer-friendly name such as `skipIfOtherBotMentioned`, that runs before typing, session recording, media downloads, history trigger preparation, and `dispatchReplyFromConfig`. The policy should detect visible mentions of configured Matrix bot user IDs other than the current account, preserve self mentions, `@room`, DMs, authorized control commands, existing `requireMention`/`autoReply` semantics, and existing `allowBots` sender policy. Keep the fix inside the Matrix plugin unless a truly generic SDK seam is needed; do not move Matrix-specific behavior into core. Update schema/types, docs, and changelog if the public config surface changes. Cover the human-sender/other-configured-bot mention case plus self mention and non-visible metadata edge cases.
