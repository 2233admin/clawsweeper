## Summary

Keep this PR open. Current main still has the response-before-microtask disconnect window for device pair removal, token rotation, and token revocation; it has no synchronous client invalidation seam and no per-RPC device-token invalidation check. The PR remains a plausible targeted gateway auth hardening change that needs maintainer review, not cleanup closure.

## Best Possible Solution

Keep this PR open, rebase/update it against current main if needed, and review it as a targeted gateway auth hardening change. The desired endpoint is a small patch that synchronously marks affected device-token clients invalid before admin responses are emitted, rejects later queued RPCs at the top of WS dispatch, preserves deferred socket close so successful admin responses can flush, and carries focused tests for remove, rotate, revoke, context filtering, and dispatch rejection ordering.
