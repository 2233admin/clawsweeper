## Summary

Keep this PR open. Current main still sends regular Telegram videos without probing or forwarding `width`/`height`, so the PR's intended bug fix is not implemented. The submitted patch is also not merge-ready as-is because it adds bare `ffprobe` execution, imports from core internals across the bundled-plugin boundary, changes the shared video size constant, and lacks targeted regression coverage for both Telegram outbound paths.

## Best Possible Solution

Keep the PR open and request a focused revision: implement video-dimension probing through `openclaw/plugin-sdk/media-runtime` using the trusted `runFfprobe`/resolver path, pass `width` and `height` only for regular Telegram `sendVideo` in `extensions/telegram/src/send.ts` and `extensions/telegram/src/bot/delivery.replies.ts`, leave GIF animations and video notes untouched, remove the unrelated `MAX_VIDEO_BYTES` change, and add targeted regression tests for both outbound paths.
