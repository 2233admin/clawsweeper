## Summary

Keep this PR open. Current main still sends regular Telegram videos without probing or passing width/height on both direct-send and reply-delivery paths, and Telegram/grammY both support width/height for sendVideo. The PR is a useful implementation candidate, but it should not merge as-is because the patch introduces a bare ffprobe execution path, imports core src internals from a bundled plugin, changes the global video media limit unrelated to the stated Telegram dimension fix, and lacks targeted regression tests.

## Best Possible Solution

Keep this PR open and request a focused revision: implement video-dimension probing through the existing plugin-SDK media runtime and trusted runFfprobe/resolveSystemBin path, pass width and height only to regular Telegram sendVideo in extensions/telegram/src/send.ts and extensions/telegram/src/bot/delivery.replies.ts, leave GIF/animation and video-note behavior untouched, remove or split the global media-limit change, and add targeted regression tests for both outbound paths plus exclusions. Add a user-facing changelog entry if the bug fix lands.
