## Summary

Keep this PR open. Current main still does not pass video width/height to Telegram `sendVideo` in either outbound path, and the PR is still a plausible fix candidate, but its submitted form needs revision for the plugin boundary, trusted ffprobe execution, unrelated media-limit change, and regression coverage. The April 26 maintainer/Codex review comment in the PR discussion also explicitly recommends keeping it open for focused follow-up.

## Best Possible Solution

Keep this PR open and request a focused revision: expose or add a video-dimension helper through `openclaw/plugin-sdk/media-runtime` or the Telegram local runtime barrel using the trusted `runFfprobe`/`resolveSystemBin` path, pass `width` and `height` only for regular `sendVideo` calls in `extensions/telegram/src/send.ts` and `extensions/telegram/src/bot/delivery.replies.ts`, keep GIF animations and video notes excluded, add regression tests for both outbound paths, and split or remove the unrelated generic media-limit change.
