## Summary

Keep open. Current main still has unbounded awaits around the memory_search manager setup and search execution, and the issue is already paired with open PR #49794 using closing syntax. The existing unavailable payload handles rejected failures, but it does not guarantee a tool result for never-settling manager/init/search promises.

## Best Possible Solution

Review and either land PR #49794 or replace it with an equivalent memory-core fix that wraps the full memory_search tool boundary, including runtime load, manager context/creation, QMD pending creation, builtin manager creation, and manager.search, in a hard deadline that always returns the existing unavailable payload. Add fake-timer regression tests for never-settling manager-init and search promises before closing this issue.
