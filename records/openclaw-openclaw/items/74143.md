## Summary

Keep open. Current main still appends Feishu block payloads with delta semantics while partial replies use snapshot merging, and the regression-shaped test still expects duplicated text. The linked fix PR remains open and unmerged, so this issue should stay open until that PR or an equivalent narrow fix lands.

## What This Changes

Fix Feishu card streaming so overlapping block payloads and partial snapshots merge instead of repeating earlier reply content.

## Best Possible Solution

Review and land PR #74146 or an equivalent Feishu plugin patch that routes card-mode block fallback text through overlap-aware snapshot merging, updates the duplicated-output test, and adds regression coverage for overlapping partial snapshots plus block payloads.

## Reproduction Assessment

Yes. The current unit-test scenario is a high-confidence reproduction path: partial `hello` followed by block `lo world` closes as `hellolo world`, and the source still implements that by concatenating delta block text.

## Solution Assessment

Yes. Based on current code, using the existing snapshot/`mergeStreamingText` path for block fallback text is the narrowest maintainable fix because it preserves non-overlap concatenation while deduping partial/block overlaps.
