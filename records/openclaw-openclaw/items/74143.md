## Summary

Keep open. Current main still has the reported Feishu streaming-card overlap bug, and open PR #74146 is still the focused unmerged implementation candidate for this issue.

## What This Changes

The issue asks OpenClaw to prevent Feishu streaming cards from duplicating overlapped text when partial snapshot replies and block payload replies both carry content for the same response.

## Best Possible Solution

Land PR #74146, or an equivalent narrow Feishu plugin dispatcher/test change, so block fallback text uses the existing overlap-aware snapshot merge while preserving block-only fallback delivery; then close this issue as implemented.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: partial `hello` followed by block `lo world` closes as `hellolo world`, and the code path explains it through raw delta concatenation.

## Solution Assessment

Yes. Switching block fallback updates to snapshot merge semantics is the narrowest maintainable fix because it reuses the existing helper and keeps the behavior inside the Feishu plugin dispatcher/tests.
