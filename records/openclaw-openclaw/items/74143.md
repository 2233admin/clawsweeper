## Summary

Keep open. Current main still has the Feishu streaming-card overlap bug, the latest release tag contains the same block-delta behavior, and open PR #74146 explicitly closes this issue but has not merged.

## What This Changes

The issue requests that Feishu streaming-card block payloads that overlap partial snapshot text be merged overlap-aware instead of concatenated into duplicated card content.

## Best Possible Solution

Keep this issue open until PR #74146, or an equivalent narrow Feishu plugin dispatcher/test fix, lands. The desired end state is for block fallback text to use the existing overlap-aware snapshot merge while preserving block-only fallback delivery and adding regression coverage for partial snapshot plus block overlap.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: partial `hello` followed by block `lo world` is expected by the existing Feishu dispatcher test to close as `hellolo world`, matching the reported duplicate-overlap behavior.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge semantics is the narrowest maintainable fix because it reuses `mergeStreamingText` and keeps the change within the Feishu plugin dispatcher/tests.
