## Summary

Keep open because current main still has the reported Feishu streaming-card overlap bug, and the linked fix PR #74146 is open and unmerged.

## What This Changes

The issue asks OpenClaw to prevent Feishu streaming cards from duplicating overlapped text when partial snapshot replies and block payload replies both carry content for the same response.

## Best Possible Solution

Review and land PR #74146, or an equivalent Feishu plugin dispatcher/test change, so block streaming-card payloads use the existing overlap-aware snapshot merge while preserving block-only fallback delivery; after that merge, this issue can close as implemented.

## Reproduction Assessment

Yes. The current dispatcher test provides a high-confidence source-level reproduction: partial `hello` followed by block `lo world` closes as `hellolo world`, and the current code explains why by using raw delta append for block payloads.

## Solution Assessment

Yes. Routing block payload text through the existing snapshot/`mergeStreamingText` semantics is the narrow maintainable fix because the helper already handles overlap and the change stays inside the Feishu plugin dispatcher/tests.
