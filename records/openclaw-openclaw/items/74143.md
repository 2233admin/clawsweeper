## Summary

Keep open. Current main still routes active Feishu block payload text through delta concatenation while partial callbacks use the overlap-aware snapshot merge, the current test fixture still expects the duplicated `hellolo world` result, and the linked fix PR #74146 is open and unmerged.

## What This Changes

The issue requests Feishu streaming-card replies to deduplicate overlapping block payload text against partial snapshot text instead of appending duplicated content.

## Best Possible Solution

Keep this issue open until PR #74146, or an equivalent narrow Feishu plugin dispatcher/test fix, lands. The desired end state is block fallback text using the existing overlap-aware snapshot merge while preserving block-only fallback delivery, with regression coverage for overlapping partial snapshots plus block payloads.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: under the existing Feishu streaming-card test, partial `hello` followed by block `lo world` closes as `hellolo world`, matching the reported duplicated-overlap behavior.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge path is the narrowest maintainable fix because it reuses `mergeStreamingText` and keeps the behavior local to the Feishu plugin dispatcher/tests.
