## Summary

Keep this PR open. Current main has overlapping terminal chat-event reconciliation, but it still does not implement the PR's central chat-streaming changes: stable assistant parser segment IDs, debounced streaming scroll, Swift decoding of agent-event session keys, or the broader live agent stream routing model. The related #40985 flicker-only PR is closed and this PR is the broader implementation candidate, not obsolete work.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase onto current main, split or justify the `scripts/install.ps1` change, preserve the still-missing parser stable IDs and streaming scroll debounce where valid, and settle the `ChatViewModel` live-stream routing contract around decoded agent-event `sessionKey`, pending run IDs, session switching, and concurrent same-session runs with focused OpenClawKit tests plus iOS build proof.
