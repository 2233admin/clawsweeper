## Summary

Keep open. Current main still lacks the agent-event session-key decoding and stream-routing pieces that motivate this chat fix, but the PR should not merge as-is because it makes later chat `onAppear` loads permanently no-op after the first successful bootstrap and includes an unrelated PowerShell installer failure-path change.

## What This Changes

This PR updates shared OpenClawKit chat parsing, agent-event payload decoding, chat view scrolling, ChatViewModel session/stream handling, Swift regression tests, and Windows installer failure handling.

## Best Possible Solution

Revise this into a focused shared chat fix: keep the session-key decoding, parser stability, stream-scoping, and regression coverage; preserve a later refresh path for chat appearances; and remove or split the Windows installer change into a dedicated installer PR with its own validation.
