## Summary

Keep this issue open. Current main at 5de284c2e30f915b769fac08dbeab45f57749702 still lacks an official Docker/Linuxbrew path, the reported openai-whisper skill remains brew-only, and the installer still returns a brew-not-installed failure when no brew executable is present. The discussion also has active workaround comments and open linked fix PRs with closing syntax, so the right cleanup path is PR review, not issue closure.

## Best Possible Solution

Keep this issue open until maintainers choose and land a focused Docker/skills fix. The most likely paths are an official opt-in Linuxbrew setup in the Docker build/setup flow, Linux-native installer metadata for brew-only bundled skills such as openai-whisper, or onboarding behavior that avoids offering brew-only installs in Docker when brew is unavailable. Review or close the linked implementation PRs first; close this issue only after the chosen fix reaches main.
