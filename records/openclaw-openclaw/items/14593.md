## Summary

Keep this issue open. Current main still has no official Docker/Linuxbrew path, the reported openai-whisper skill is still brew-only, and the installer still fails brew install specs when no brew executable is present. The thread also has active open fix PRs with closing syntax, so the right cleanup path is maintainer review of those PRs rather than closing this issue.

## Best Possible Solution

Keep this issue open while maintainers choose and land a focused Docker/skills fix. The strongest paths are an opt-in Linuxbrew Docker build/setup flow, Linux-native installer metadata for brew-only bundled skills such as openai-whisper, or onboarding behavior that avoids offering brew-only installs in Docker when brew is unavailable. Review or close the linked implementation PRs first; close this issue only after the chosen fix reaches main.
