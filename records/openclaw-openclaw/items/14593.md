## Summary

Keep open. Current main still leaves the reported Docker skill-install failure path unresolved: the Docker image/setup script have apt-package and Docker-CLI build options but no official Linuxbrew/Homebrew build option, the skill installer still returns the brew-missing failure for brew-only specs when brew is absent, onboarding still shows the generic Homebrew prompt without container-aware filtering, and the reporter's openai-whisper example remains brew-only. The issue is also explicitly paired with open fix PRs, so it should stay open until maintainers review/land or close those candidates.

## Best Possible Solution

Keep this issue open while maintainers choose one focused path: review and land or close #42303, #55940, and #57553, or implement an official alternative such as opt-in Linuxbrew Docker support, native Linux installer metadata/fallbacks for affected bundled skills, or container-aware onboarding filtering. The chosen fix should include Docker skill-dependency regression coverage and update https://docs.openclaw.ai/install/docker plus the skills docs.
