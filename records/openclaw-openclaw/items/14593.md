## Summary

Keep open. Current main still has no official Docker/Linuxbrew path, no Linux-native fallback for arbitrary brew-only skill installers, and the reporter’s `openai-whisper` example remains brew-only. Live API checks also show open implementation candidates #42303 and #55940 with closing intent for this issue, so the best maintainer action is to review or reject those PRs before closing.

## Best Possible Solution

Keep this issue open while maintainers choose one focused path: land or close the active PRs, or implement an official opt-in Linuxbrew Docker setup, Linux-native installer metadata/fallbacks for affected bundled skills, or onboarding/install filtering that avoids offering brew-only installs in Linux containers without brew. The chosen path should include Docker skill-dependency regression coverage and Docker/skills docs updates.
