## Summary

Keep open. Current main still has the unguarded `WebSocketTaskBox.sendPing()` continuation path, and the keepalive loop still calls it. This PR is a narrow one-file fix for that live crash path, so it is not implemented on main and is not clearly superseded by #41653, which remains open and bundles an unrelated workflow change.

## Best Possible Solution

Keep this PR open as a plausible focused fix. Add the explicit `import os`, consider a narrow test double that calls the ping callback twice, run the Swift package checks, then land one canonical `GatewayChannel.swift` fix and close the duplicate PR path (#41653 or this PR) after that decision is clear.
