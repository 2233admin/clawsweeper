## Summary

Current main still has the feature gap described by the reporter and the prior ClawSweeper comment: sessions_send provenance can identify inter-session routing with machine-oriented source session/channel/tool fields, and the UI can render an existing senderLabel, but current main does not persist or resolve a human-readable sender/recipient identity for session history display or filtering.

## Best Possible Solution

Keep this issue open. Implement an additive core session/provenance path that either persists display-safe source/target identity for sessions_send or resolves sourceSessionKey/target session keys at history render time, then expose it through chat.history/sessions.get and Control UI senderLabel/filtering. Add tests for receiving-session sender display and sender-session target/tool-result visibility.
