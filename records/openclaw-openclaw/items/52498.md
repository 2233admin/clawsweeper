## Summary

Current main still has the feature gap: sessions_send records machine provenance for inter-session messages, and the Control UI can render senderLabel when it already exists, but current code does not persist or resolve a human-readable sender/recipient identity for sessions_send history display or filtering.

## Best Possible Solution

Keep this issue open. Implement an additive sessions_send attribution path that either persists display-safe source/target identity with inter-session provenance or resolves sourceSessionKey/target session keys at chat.history/sessions_history render time, then expose it through the Control UI senderLabel/filter surface and sender-side tool/history details. Cover receiving-session sender display and sender-session target/id visibility with focused tests.
