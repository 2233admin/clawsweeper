## Summary

Keep open. Current main has a `message_sending` hook that can rewrite or cancel outbound text, and docs/types now mention optional `sessionKey`/`runId`, but the central request is not implemented: shared outbound delivery still calls `runMessageSending` with only routing fields, `OutboundSessionContext` has no run identifier, and `PluginHookMessageSendingEvent` exposes no bounded transcript/tool-call/result evidence for deterministic verification. Open PRs #73706, #44011, and #63007 overlap the session identity subset but remain open and do not cover the full runId plus evidence contract.

## What This Changes

Expose a pre-delivery outbound message hook with sessionKey/runId and enough bounded tool-call/result context for plugins to annotate, rewrite, or block unverified replies.

## Best Possible Solution

Keep this open until maintainers resolve or supersede #73706, #44011, and #63007 with one canonical additive hook-context change, then make an explicit plugin-hook contract decision for `runId` and a bounded recent transcript/tool-call/result evidence view across shared outbound delivery and direct reply paths, with docs and regression tests.

- There are multiple overlapping open PRs for the identity subset; landing more than one without a canonical contract could create conflicting semantics.
