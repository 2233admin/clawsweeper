## Summary

Keep open. Current main has a usable `message_sending` hook and typed `sessionKey`/`runId` fields, but the shared outbound pre-send path still does not pass either correlation value into the hook context, `OutboundSessionContext` has no `runId`, and there is still no bounded tool-call/transcript evidence view for deterministic verification. Open PRs #73706, #44011, and #63007 overlap on the session identity subset, but none is merged on current main and they do not fully cover the runId plus tool-history request.

## Best Possible Solution

Keep this issue open. Review and either land or supersede #73706 for the `sessionKey` subset, then make an additive plugin-hook contract decision for `runId` plus a bounded recent transcript/tool-call/result evidence view. The implementation should cover shared outbound delivery and direct reply paths, update `https://docs.openclaw.ai/plugins/hooks`, and add regression tests for sessionKey/runId propagation and evidence bounds.
