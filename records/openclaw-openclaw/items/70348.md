## Summary

Keep open. Current main still returns synthetic compaction markers through the Gateway `chat.history` projection path, and the shared sanitizer does not drop the PR's compaction, pre-compaction memory-flush, or post-compaction refresh targets. The PR remains relevant, but its raw prefix-based filtering has a real false-positive risk for legitimate user-authored messages.

## Best Possible Solution

Keep this PR open, but revise it before merge. The safer implementation is provenance or metadata-based filtering for synthetic/internal records, plus regression coverage that hides synthetic compaction and known internal memory artifacts while preserving user-authored messages with matching text prefixes. Coordinate with #43567 if maintainers prefer distinct system-block rendering over full hiding.
