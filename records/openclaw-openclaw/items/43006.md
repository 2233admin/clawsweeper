## Summary

Keep open. Current main still implements a single pre-compaction memory-flush gate, not the requested progressive 80%/90%/95% staged reminders, stage-specific prompts, persisted stage state, or forced-summary handoff. The related docs-only PR #43102 was closed unmerged and does not supersede the feature request.

## Best Possible Solution

Keep this issue open for maintainer design. A complete implementation likely needs an additive staged memory-flush contract, staged threshold configuration, per-stage dedup/persistence, stage-specific prompts, a defined forced-summary or compaction handoff, plus config schema, docs, and focused tests updated together.
