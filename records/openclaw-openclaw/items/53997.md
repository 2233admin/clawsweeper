## Risks / Open Questions

- The PR is based on an older in-repo ACPX runtime shape; current main delegates to the packaged `acpx/runtime`, so the patch likely needs rework in the `openclaw/acpx` package or a new OpenClaw wrapper seam before it can merge.
- The PR persists raw route arguments. Because MCP server config can include environment values, route metadata needs a redaction/omission design before writing durable artifacts.
- The review discussion includes an unresolved cleanup issue: artifact creation happens after abort listener registration but before the listener-removal `finally`, so setup failures can leave stale abort listeners.
- The PR is size XL and has not been validated against current main after the ACPX architecture changes.
