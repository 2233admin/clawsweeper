## Summary

Keep this PR open for maintainer review, but the current patch is not merge-ready. The proposed hook API exposes and mutates full transcript context without the existing hook policy gates, and the bundled AICM plugin reads and rewrites transcripts through a hard-coded global session path.

## What This Changes

The PR adds before_assemble and after_assemble plugin hook contracts with Pi embedded-runner wiring, plus a bundled AICM plugin, package metadata, and lockfile entries.

## Best Possible Solution

Split this into a narrow, documented context-assembly hook contract with explicit policy classification, ordering semantics, contract tests, and SDK/docs updates. Review AICM separately as an external or maintainer-owned plugin that uses scoped session/transcript APIs instead of filesystem discovery and direct transcript mutation.

## Reproduction Assessment

Yes. Static review of the PR head is enough: a non-bundled plugin can register before_assemble or after_assemble because they are valid hook names, but the registry gates only consult the existing prompt-injection and conversation hook sets.

## Solution Assessment

No. A guarded context-assembly hook may be a reasonable core API direction, but this patch combines public API design, Pi-only wiring, and a security-sensitive transcript editor; the safer path is a smaller hook contract first and a separate AICM review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate assemble hooks through hook policies:** `src/plugins/hook-types.ts:106-107`
  - body: The new before_assemble and after_assemble hooks receive or mutate full conversation messages, but the PR only adds them to PluginHookName. Because they are not also classified in the prompt-injection and conversation-access hook sets used by registry.ts, a non-bundled plugin can register raw transcript hooks without the current allowConversationAccess and allowPromptInjection gates.
  - confidence: 0.93
- **[P1] Scope AICM transcript access to the active session:** `extensions/aicm/index.ts:75`
  - body: AICM discovers transcripts from a hard-coded agents/main/sessions directory, chooses recent JSONL files, stores deletion state under current, and rewrites transcript files from agent-callable tools. In multi-agent or non-default installs, these tools can read or mutate a session other than the one invoking the tool; use request-scoped session context and OpenClaw transcript APIs before bundling this plugin.
  - confidence: 0.94
- **[P2] Feed accumulated messages into before_assemble hooks:** `src/plugins/hooks.ts:1326-1328`
  - body: runBeforeAssemble promises sequential transformations, but it delegates to runModifyingHook, which calls every handler with the original event. A later plugin can therefore return messages based on the unmodified list and undo an earlier plugin's deletion or reorder; this hook needs a chain runner that passes the accumulated message list into the next handler.
  - confidence: 0.91
- **[P3] Declare the AICM package as ESM:** `extensions/aicm/package.json:13`
  - body: The package exposes dist/index.js but does not set type: module, unlike the bundled plugin package convention. Since the TypeScript output uses ESM syntax, direct package entry loading can parse the built .js as CommonJS.
  - confidence: 0.79
- **[P3] Remove the stale broken source file:** `extensions/aicm/index.ts.broken:1`
  - body: The PR adds index.ts.broken with an older implementation that imports missing local modules. Even if TypeScript excludes it, keeping a broken source artifact in the bundled plugin tree makes package and review behavior unreliable.
  - confidence: 0.82
