This PR adds config HMAC sidecar signing/verification, integrity warnings and audit finding, explicit admin checks for config-mutating gateway methods, security docs, and a CodexBar onboarding option.

## Security Review

Status: needs_attention

Summary: The diff is security-sensitive and introduces concrete integrity-enforcement design risks before the branch is mergeable.

Concerns:

- **[high] Missing-signature upgrade lockout:** `src/config/io.ts:1107`
  - body: The branch treats missing sidecar signatures on established configs as suspicious and rejects them, which can lock out existing valid installations after upgrade when a gateway token exists but no `.sig` file has been bootstrapped.
  - confidence: 0.9
- **[medium] Best-effort writes conflict with enforcement:** `src/config/io.hmac-integrity.ts:55`
  - body: Signature writes are best-effort while reads reject missing or empty signatures, so a transient sidecar write failure after a config write can become a later self-inflicted startup or reload failure.
  - confidence: 0.87
- **[medium] Token lookup bypasses injected dependencies:** `src/config/io.hmac-integrity.ts:23`
  - body: The HMAC token helper reads the real home directory and process environment directly, making dependency-injected config IO tests likely to skip enforcement and making alternate state-home behavior inconsistent.
  - confidence: 0.86
