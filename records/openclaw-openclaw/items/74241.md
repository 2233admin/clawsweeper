## Summary

Keep open. Current main has partial timezone support, but it still only resolves agent-visible timezone context from global `agents.defaults` or a Microsoft Teams-specific override; it does not provide the requested per-agent/profile timezone surface or generic user-local runtime context.

## What This Changes

The issue requests user-local timezone injection and per-profile timezone configuration so agent turns reason from the user’s local time instead of the container host clock.

## Best Possible Solution

Keep this issue open and add a narrow timezone-resolution seam: per-agent/profile `userTimezone` and `timeFormat` overrides that fall back to `agents.defaults`, feed gateway `agent`/`chat.send` timestamp injection and system prompt timezone context, and document how channel metadata such as Teams `clientInfo.timezone` interacts with configured defaults.
