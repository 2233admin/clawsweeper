## Summary

Keep open. Current main still routes google-antigravity through the pinned pi-ai 0.70.5 Cloud Code Assist provider, whose upstream Antigravity user-agent default is still 1.21.9, and OpenClaw’s active secrets runtime auth snapshot can keep serving in-memory auth state until the runtime snapshot is refreshed or the same process saves the store.

## What This Changes

The issue requests fixing google-antigravity provider failures caused by an outdated Antigravity user-agent version and stale in-memory credential snapshots after token refresh.

## Best Possible Solution

Keep this issue open and handle it as a maintainer-reviewed provider/auth bug: either update or shim the Antigravity version default for google-antigravity, then add a targeted auth-runtime refresh/invalidation path so successful OAuth refresh or explicit credential reload makes the active gateway snapshot see fresh credentials without a restart.
