## Summary

Keep this PR open. Current main still lacks the provider-model env-proxy auto-upgrade, and the proposed patch is narrow, covered by focused tests, and matches existing guarded-fetch proxy gates. The branch needs maintainer review, rebase/mergeability cleanup, and CI rather than cleanup closure.

## What This Changes

This PR changes provider model guarded fetch to use trusted env-proxy mode when HTTP(S) proxy environment rules apply and no explicit provider dispatcher policy exists, with regression tests for auto-upgrade and explicit-dispatcher preservation.

## Best Possible Solution

Land this PR or an equivalent patch after rebase and CI, keeping the auto-upgrade limited to provider model requests with no explicit dispatcher policy and with HTTP_PROXY/HTTPS_PROXY routing actually active; preserve strict behavior for direct, TLS, env-proxy, and explicit-proxy provider transport overrides.
