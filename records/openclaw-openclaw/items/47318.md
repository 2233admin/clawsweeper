## Summary

Keep this PR open. Current main still does not implement the intended `openclaw config set` invalid-model warning; `runConfigSet` writes parsed operations and only reports generic update messages. The PR remains useful for the closed stale bug #20522, but it needs to be rebased onto the current config-set flow and should drop or split the unrelated skills eligibility change.

## Best Possible Solution

Keep this PR open as a useful implementation candidate. Rebase it onto current main by integrating scoped model-reference detection into `runConfigSet` for both single and batch value operations, warning through the provided runtime when a non-empty catalog lacks the parsed model, using `splitTrailingAuthProfile`, `parseModelRef`, and `findModelInCatalog`, and adding tests that exercise the real `config set` command path. Drop or split the unrelated skills-status eligibility change.
