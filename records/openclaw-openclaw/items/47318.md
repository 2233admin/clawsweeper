## Summary

Keep this PR open. Current main still does not implement the intended non-blocking invalid-model warning for `openclaw config set`; the config-set path writes accepted operations and emits only generic update messages. The PR remains a useful implementation candidate for #20522, but it needs to be rebased onto the current config-set flow and should split out the unrelated skills-status change.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main by adding scoped model-reference detection inside `runConfigSet` for both single and batch value operations, warning through the provided runtime when a non-empty catalog lacks the parsed model, using the existing `splitTrailingAuthProfile`, `parseModelRef`, and `findModelInCatalog` helpers, and adding tests that exercise the real config-set command path. Drop or split the unrelated skills-status change.
