## Summary

Keep open. Current main still contains a concrete installer/setup fallthrough: the fresh-install finalization path detects existing config, runs doctor, and logs that onboarding is being skipped, but then continues into `Starting setup` and executes `openclaw onboard` when a TTY is available. The large model-list and OpenRouter onboarding pieces appear present, so the remaining actionable bug is the packaged setup path forcing onboarding after config detection.

## What This Changes

Fix an Umbrel packaged setup regression where the onboarding wizard cannot get past model selection/OpenRouter and keeps forcing setup despite an existing OpenClaw config.

## Best Possible Solution

Update the packaged installer/setup flow so an existing `OPENCLAW_CONFIG_PATH` or legacy config exits the onboarding path after doctor/dashboard handling, while keeping the model picker and OpenRouter support already present. Add regression coverage for the existing-config install path and verify the Docker/Umbrel-style setup entrypoint still supports explicit onboarding when no config exists.
