## Summary

Current main has the Telegram approval-routing behavior this report asks for: exec approval target resolution carries turn-source channel/account/thread metadata, Telegram native approvals resolve origin targets from Telegram turn-source data, `agentFilter` can fall back to the agent id encoded in the session key, and multi-account Telegram approval handling now fails closed when the request is not bound to a single eligible account. The most specific current-main proof is 6dec2e1852381dd5343ab8be6bf8b7b0b04152a5; I did not prove that post-release Telegram account-disambiguation hardening is in the latest shipped release.

## Best Possible Solution

Keep the current main implementation: preserve turn-source-first approval routing, Telegram account/request disambiguation, and agent-filter fallback from session keys. If this reappears after the next release, open a fresh issue with current version, Telegram account ids redacted, session key shape, gateway approval request payload, and the exact chat/account where the prompt was delivered.
