## Summary

Keep open. Current main still has the unbounded task-completion result path, so the PR is not obsolete; the implementation is useful but should be rebased/reconciled with current main’s provider catalog expectations before merge.

## What This Changes

The PR caps task-completion internal-event results in agent prompt formatting, adds runtime-context/plain-prompt truncation tests, and adjusts provider catalog test expectations.

## Best Possible Solution

Rebase the PR onto current main, keep the internal-event truncation helper and focused agent tests, and drop or reconcile the provider catalog helper/test changes so catalog assertions remain exact. If maintainers still want a configurable cap, track that separately through #57148 rather than blocking the bounded default.

## Reproduction Assessment

Yes. A deterministic formatter check is enough: on current main, a task-completion event with a very large `result` reaches both `formatAgentInternalEventsForPrompt` and `formatAgentInternalEventsForPlainPrompt` without truncation.

## Solution Assessment

Mostly yes. Bounding at the internal-event formatting boundary is the narrowest maintainable fix for both runtime-context and plain prompt delivery, but the unrelated provider catalog test weakening should not ship as-is.
