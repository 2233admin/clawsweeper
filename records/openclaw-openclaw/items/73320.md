## Summary

Keep open. Current main still inserts unbounded task-completion result text into both internal runtime-context and plain prompt materialization, so this PR addresses a real remaining gap. The branch needs maintainer review/rebase because current main already restored the Codex mini catalog fixture, while the PR weakens catalog cardinality coverage.

## What This Changes

The PR adds a fixed 12,000-character head/tail truncation path for task-completion internal-event results with new formatter tests, and also adjusts OpenAI Codex catalog test expectations.

## Best Possible Solution

Land the focused internal-event result cap with formatter coverage, keep catalog tests exact after rebasing onto current main, and leave a configurable `maxSubagentResultChars` setting to #57148 or a follow-up if maintainers want more than a bounded default.

## Reproduction Assessment

Yes. A deterministic source-level repro is to call `formatAgentInternalEventsForPrompt` or `formatAgentInternalEventsForPlainPrompt` with a task-completion event whose `result` is very large; current main inserts the sanitized result directly, so prompt size grows with the child output.

## Solution Assessment

Mostly yes. Formatting task-completion events is the narrow boundary that covers both OpenClaw internal runtime context and ACP/plain prompt delivery, but the stale provider catalog test changes should be dropped or reconciled during rebase.
