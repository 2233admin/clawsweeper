## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: task-completion internal event results are sanitized but embedded without a size bound in both runtime-context and plain prompt formats. The PR remains potentially useful, but it should be reviewed and narrowed because its current head also includes unrelated provider catalog test changes.

## Best Possible Solution

Keep the PR open and ask for a narrowed rebase on current main: implement the internal task-completion result bound in `src/agents/internal-events.ts`, add focused regression coverage for both `formatAgentInternalEventsForPrompt` and `formatAgentInternalEventsForPlainPrompt`, preserve explicit truncation markers and delimiter escaping, and remove or separately justify the provider catalog test changes. After that, run the targeted formatter, lint, and agent tests for this surface.
