## Summary

Keep this PR open. Current main still embeds task-completion results without a prompt-size bound in both internal runtime-context and plain prompt renderers, so the PR's central behavior is not implemented on main. The PR remains a useful implementation candidate, but its current head should be narrowed because it also changes unrelated provider catalog tests.

## Best Possible Solution

Keep this PR open and ask for a narrowed rebase on current main: implement the task-completion result bound in `src/agents/internal-events.ts`, preserve delimiter escaping, include explicit truncation markers with useful head/tail context, cover both `formatAgentInternalEventsForPrompt` and `formatAgentInternalEventsForPlainPrompt`, and remove or separately justify the provider catalog test changes before running the targeted formatter, lint, and agent tests.
