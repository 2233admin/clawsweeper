## Summary

Keep open. Current main still embeds task completion results into both internal runtime-context prompts and plain prompts without a size bound, while this PR adds a focused truncation helper plus tests for both paths. The diff is narrow and appears useful; it is not obsolete on main and should be reviewed rather than closed.

## Best Possible Solution

Keep this PR open as a focused hardening candidate. The best path is to review and land a scoped main implementation that bounds `AgentTaskCompletionInternalEvent.result` in both runtime-context and plain prompt renderers, keep the colocated tests, verify exact-head CI, and leave broader fail-closed completion-delivery cleanup to the related open trackers.
