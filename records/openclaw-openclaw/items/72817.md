## Summary

Keep open. Current main still lacks the guarded source-require-to-Jiti fallback for bundled source public artifacts, while the PR is open, cleanly mergeable, covered by focused regression tests, and has green reported checks. I did not find a blocking correctness or security issue in the proposed patch.

## What This Changes

This PR adds a guarded Jiti retry for bundled source public artifacts whose source `require` path fails on emitted-style `.js` imports, tightens related test mocks, and extracts bundled entry load-profile emission into a helper.

## Best Possible Solution

Have a maintainer review the source-checkout loader fallback as an accepted plugin-runtime boundary and land this PR if that tradeoff is intended, keeping the focused regression tests and existing changelog coverage.

## Reproduction Assessment

Yes. Current main’s source public-artifact fast path directly uses `sourceArtifactRequire`, and the PR’s test models the `ERR_MODULE_NOT_FOUND` path for `./config-defaults.js` before asserting the Jiti fallback.

## Solution Assessment

Yes. The PR keeps the existing source `require` fast path, retries only module-not-found failures whose message points at a `.js` specifier, and preserves syntax errors on the original path; I did not find a narrower maintainable fix.
