## Summary

Keep open. Current main still lacks the guarded source-artifact require-to-Jiti fallback, while this PR adds a focused fix with regression coverage and no blocking correctness or security finding; the remaining decision is maintainer review of the plugin loader fallback behavior before merge.

## What This Changes

The PR adds a guarded Jiti retry for bundled source public-artifact require failures caused by emitted-style `.js` imports, expands loader/runtime-deps tests, and extracts bundled entry load-profile emission into a helper.

## Best Possible Solution

Have a maintainer review the guarded fallback as the intended loader boundary, keep the focused regression tests, and land this PR if that execution-path behavior is accepted; a replacement repair branch is not needed while this PR remains focused and mergeable.

## Reproduction Assessment

Yes. Current main selects source public artifacts and directly calls `sourceArtifactRequire`, while bundled source artifacts such as `extensions/anthropic/provider-policy-api.ts` use emitted-style `.js` sibling imports; the PR's new test models that module-not-found path.

## Solution Assessment

Yes. The proposed fix preserves the existing source-require fast path and only retries through the existing Jiti loader for module-not-found errors whose message references a `.js` specifier, which is narrower than always using Jiti or retrying every failure.
