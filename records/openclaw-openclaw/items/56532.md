## Summary

Keep this PR open. Current main still lacks configurable memory-lancedb embedding timeout and retry settings, and the branch remains a focused implementation candidate, but it needs maintainer review because the new settings do not cover the auto-recall request path and the baseUrl/retry security concern remains unresolved.

## What This Changes

The PR adds memory-lancedb `embedding.timeoutMs` and `embedding.maxRetries` config/schema/manifest UI support, parses bounded values, and passes them into the OpenAI-compatible embedding client options with config tests.

## Best Possible Solution

Land a refreshed memory-lancedb change only after the config contract, manifest schema, runtime wiring, and docs describe one clear timeout/retry model. The implementation should either apply the configured values to all OpenAI-compatible embedding calls, including auto-recall, or intentionally document and test any exception for prompt-build recall.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Respect config on auto-recall requests:** `extensions/memory-lancedb/index.ts:291-292`
  - body: The PR sets `timeoutMs` and `maxRetries` only as OpenAI client defaults, but current auto-recall calls `embeddings.embed(..., { timeoutMs: DEFAULT_AUTO_RECALL_TIMEOUT_MS })` and the client then sends per-request `{ timeout: 15000, maxRetries: 0 }`. OpenAI request options override client defaults, so a user-configured timeout or retry count is ignored on the prompt-build recall path this feature is meant to protect.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff does not add a new dependency or workflow surface, but it does add configurable retries around an existing configurable OpenAI baseUrl sink that already expands environment variables.

Concerns:

- **[medium] Review retry amplification on configurable baseUrl:** `extensions/memory-lancedb/index.ts:292`
  - body: `embedding.baseUrl` is already user-configurable and expanded with `${ENV_VAR}` before reaching the OpenAI client; this PR adds a user-controlled `maxRetries` value for that same request sink. If untrusted config can reach this plugin, retries can amplify SSRF or secret-in-URL exfiltration attempts, so maintainers should either harden baseUrl/env expansion or explicitly accept the trust model before merge.
  - confidence: 0.78
