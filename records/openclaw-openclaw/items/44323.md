## Summary

Keep this PR open. Current main has runtime-aware memory-search doctor diagnostics, but it does not contain the PR's proposed config-load memorySearch.provider validator. The PR is also too broad to treat as maintainer cleanup as-is: its validation is narrower than the stated provider-existence/required-field goal and it adds unrelated cron and agent lifecycle/watchdog runtime modules that need separate review.

## Best Possible Solution

Keep this PR open, but ask for a refreshed, split-down version. The best path is a narrow memorySearch validation/diagnostics PR that reuses the current noteMemorySearchHealth flow where possible, aligns with the documented runtime contract, adds focused tests for unsupported providers or explicitly invalid fields, and moves the cron reliability plus agent lifecycle/watchdog work into separate proposals.
