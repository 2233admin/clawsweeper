## Summary

Keep open. Current main still routes ordinary top-level Slack room/channel messages with replyToMode="all" to the base channel session while reply delivery metadata points at the message timestamp, matching the reported split. The issue also has open PR #64080 with closing syntax, so it should stay open until that PR or an equivalent maintainer fix lands.

## What This Changes

The issue requests that Slack top-level Slack channel/group messages with replyToMode="all" use one thread-scoped session for inbound processing and reply delivery instead of also accumulating in the parent channel session.

## Best Possible Solution

Keep this issue open and review or land #64080, or an equivalent maintainer patch, so ordinary Slack room/channel top-level messages with replyToMode="all" use :thread:<message.ts> for inbound session routing and align with MessageThreadId. Preserve replyToMode="off" shared channel continuity, first/batched behavior, seeded app-mention roots, true thread replies, runtime bindings, and parent-session inheritance.
