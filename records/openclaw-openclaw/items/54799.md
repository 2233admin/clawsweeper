## Summary

Keep this open. Current main still has skill metadata, status, CLI, cron, and hook surfaces, but it does not implement declarative `SKILL.md` trigger metadata, scheduler reconciliation from skill metadata, event-triggered skill registration, last-triggered status, or a skills audit command.

## What This Changes

The issue requests optional schedule/event trigger metadata in `SKILL.md` frontmatter, with OpenClaw registering those triggers, showing last-triggered status, and auditing never-triggered skills.

## Best Possible Solution

Add a versioned `metadata.openclaw.triggers` contract for skills, validate schedule and event trigger forms, reconcile schedule triggers into managed cron jobs, define approved lifecycle events for event-triggered skills, persist trigger run history, and expose last/next trigger state through `openclaw skills list`, `openclaw skills info`, and a skills audit command.
