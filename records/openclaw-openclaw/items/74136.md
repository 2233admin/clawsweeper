## Summary

Keep open. Current main still has strict announce-skip parsing, and this PR is the canonical open fix for the linked bug after the parallel PRs were closed; the implementation direction is narrow and sound, but release/docs alignment and the failing exact-head parity check need handling before merge.

## What This Changes

The PR broadens `isAnnounceSkip()` to accept `ANNOUNCE_SKIP` as the trimmed standalone final line of multi-line output, adds focused token-helper unit tests, and adds an announce-flow regression test.

## Best Possible Solution

Land this PR or an equivalent narrow patch after adding the required release note, aligning the sub-agent docs if final-line tolerance is now intended public behavior, and clearing the exact-head CI failure while keeping `REPLY_SKIP` strict and rejecting embedded or non-final `ANNOUNCE_SKIP` text.

## Reproduction Assessment

Yes. The source-level reproduction is `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` on current main: the helper compares the whole trimmed string to the token, and the announce call sites treat the unmatched text as deliverable output.

## Solution Assessment

Yes. The proposed final-line-only check is the narrowest maintainable fix for the reported leak because it preserves exact-token behavior, rejects mid-text/trailing-content token uses, and leaves reply suppression unchanged; the safer follow-through is docs/changelog plus CI cleanup, not a broader parser.
