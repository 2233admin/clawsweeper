## Summary

Keep this PR open for maintainer review and likely merge. Current main still has the strict announce-skip equality that #74071 reports, while this PR is a narrow implementation with focused helper coverage and an announce-flow regression test; the only remaining maintainer questions are docs/changelog alignment and the unrelated-looking failing parity check on the PR head.

## What This Changes

The PR changes `isAnnounceSkip()` to accept `ANNOUNCE_SKIP` as the trimmed standalone final line of multi-line output, adds helper unit tests, and adds an announce-flow regression test.

## Best Possible Solution

Land this PR or an equivalent narrow patch that accepts `ANNOUNCE_SKIP` only as the trimmed standalone final line for announce suppression, preserves strict `REPLY_SKIP` behavior, and updates docs/changelog if maintainers want the tolerant form to be documented for users.

Yes. The high-confidence source-level reproduction is `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` on current main, which fails the strict equality helper and then reaches announce delivery call sites that depend on that predicate.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix for the observed leak because it changes only announce suppression semantics, keeps reply suppression strict, and adds both helper-level and announce-flow regression coverage.
