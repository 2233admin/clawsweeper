## Summary

Keep open. Current main still does not implement the PR's central derived/virtual model-ref behavior: `agents.defaults.models` entries cannot declare `routeTo`, runtime model resolution does not route a selected derived ref to an upstream model, and current docs/tests only cover existing aliases plus provider-level or `models.providers.*.models[]` context caps.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to update it against current main with a documented generic routed-model contract: add `routeTo` and per-entry `contextTokens` to `agents.defaults.models` types, Zod schema, generated schema/help as needed; route both sync and async model resolution through the effective provider/model; define whether params inherit from the upstream entry; and add focused tests for picker/list visibility, runtime execution, context budgeting, route cycles, and cross-provider dynamic-provider preparation.
