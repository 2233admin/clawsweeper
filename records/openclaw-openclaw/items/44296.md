## Summary

Keep open. Current `main` still emits the generic `$PROFILE` placeholder and vague PowerShell reload guidance in onboarding shell-completion setup, and the related fix PR #47333 is closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open. The best fix belongs in the core CLI/wizard completion path: resolve a concrete PowerShell profile path, emit a safe actionable reload hint with correct quoting, add Windows/PowerShell-focused tests for onboarding output, and align the shared completion runtime/doctor completion behavior so install, status, and onboarding messaging agree.
