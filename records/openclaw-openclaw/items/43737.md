## Summary

Keep this issue open. Current main still has the active-session path where `sessions.list` can omit a stale main session while returning a recent heartbeat session, and the chat dropdown only adds returned rows plus the current selected key. The provided GitHub context also shows open PR #43984 using closing syntax as the implementation candidate, so this issue should stay open until that PR lands, is closed, or is superseded.

## Best Possible Solution

Review and land PR #43984, or close/supersede it with an equivalent narrow fix. The best fix belongs in the gateway session-list behavior, with regression coverage for a stale resolved main session plus an active heartbeat session, while preserving the active filter for other stale sessions.
