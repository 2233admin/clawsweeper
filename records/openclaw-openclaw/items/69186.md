## Summary

Keep open. Current main has the chat terminal-event plumbing needed to trigger a completion cue and has Web Push subscription/test UI, but it does not implement an opt-in Control UI completion sound or success notification for finished agent turns. The prior related hook request is covered by `agent_end`, but this request is specifically for in-browser Control UI feedback.

## What This Changes

Add an optional completion/success sound or notification when an agent finishes a response in the OpenClaw Control UI, with possible settings for enablement and custom sound choice.

## Best Possible Solution

Implement a browser-local, opt-in completion chime in the Control UI that fires on a successful terminal `chat` event for the active user run. Keep the first pass to a built-in generated sound and a persisted setting under Notifications; treat custom external sound URLs or cross-channel notification behavior as separate product work.

## Reproduction Assessment

Yes. Static inspection of current main shows the Control UI receives terminal chat states and clears active run state, while the Notifications UI only manages Web Push subscription/test state and there is no non-talk completion audio path.

## Solution Assessment

Yes, for a narrow first implementation. The maintainable path is to reuse the existing Control UI terminal chat-event path and browser-local `UiSettings`; plugin `agent_end` is useful for external automation but does not satisfy an in-tab Control UI sound setting.
