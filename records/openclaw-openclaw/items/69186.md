## Summary

Current main still does not implement the requested Control UI/WebChat completion sound. The active chat final-event path appends/clears state only, terminal gateway handling reloads or resets stream/tool state only, and the existing Notifications surface is Web Push subscribe/unsubscribe/test rather than a browser-local audible cue. The related `agent_end` plugin hook from #59199 is useful context, but it does not satisfy this browser UI setting request.

## Best Possible Solution

Keep this issue open and implement a focused Control UI/WebChat feature: a browser-local, default-off completion sound setting with a test button, using a short generated Web Audio tone when the active chat run in the current session reaches a successful final event. Do not broaden gateway protocol, Web Push delivery, plugin hooks, or remote custom sound URLs in the first pass.
