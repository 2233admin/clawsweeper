## Summary

Keep this PR open. Current main still has the reported `/allowlist --store` policy gap: store-only edits skip the existing `resolveConfigWriteDeniedText` check and then call `updatePairingStoreAllowlist` directly. The PR is paired with open issue #72360 and is a focused two-file security hardening patch, so the right outcome is maintainer review/merge or an equivalent narrow fix rather than cleanup closure.

## Best Possible Solution

Keep this PR open and review/merge it, or supersede it with an equivalent narrow patch on main: apply `resolveConfigWriteDeniedText` to store-targeted `/allowlist add|remove --store` mutations before `updatePairingStoreAllowlist`, using the explicit channel/account config-write target, and keep regression tests for channel-level false, account-level false, and allowed store edits. Close #72360 only after this implementation or an equivalent maintainer-approved fix lands.
