## Summary

Keep open. Current main still derives the Mattermost interaction signing secret from the bot token with a public hardcoded HMAC key, the docs and tests still encode that deterministic contract, and the linked same-author fix PR is open and unmerged, so this issue should remain the security tracker until a reviewed fix lands.

## What This Changes

The issue requests replacing Mattermost's hardcoded bot-token-derived interaction HMAC secret with a separate per-account persisted signing secret for button callbacks.

## Best Possible Solution

Land PR #64546 or a reviewed successor that introduces a persistent, SecretRef-aware per-account Mattermost interaction signing secret distinct from the bot token, updates schema/types/docs/tests, and closes this issue only after the fix is merged and shipped.
