## Summary

Keep open. Current main still has the reported auth-epoch sensitivity for Claude credentials that parse as token credentials, and the linked implementation PR #74493 is open and unmerged with closing syntax, so this issue should wait for that PR to be reviewed, landed, or rejected. The reporter’s follow-up strengthens the path forward by confirming that #74493’s version bump and identity-only token hashing match the observed migration/reset behavior.

## What This Changes

The issue asks OpenClaw to stop resetting long-running Claude CLI sessions when OAuth token rotation or a transient access-token-only keychain read changes the auth-epoch hash.

## Best Possible Solution

Land a reviewed version of #74493, or an equivalent maintainer patch, that removes rotating token material from Claude/token auth-epoch hashes, bumps the auth-epoch version to avoid legacy migration resets, preserves real identity-change invalidation, and adds targeted regression tests plus a user-facing changelog credit.
