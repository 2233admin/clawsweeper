## Summary

Keep open. Current main still hashes Claude token-shaped credentials and token auth-profile material into the CLI auth epoch, so token rotation or a transient token-shaped Claude credential can still invalidate reusable sessions; the open closing PR is the right implementation path to review before this issue closes.

## What This Changes

Prevent Claude CLI OAuth token rotation or transient token-shaped reads from changing CLI auth epochs and resetting active reusable sessions mid-conversation.

## Best Possible Solution

Land #74493 or an equivalent narrow fix that makes Claude token-shaped auth epochs identity-based, removes raw token material from token auth-profile epoch hashing where appropriate, bumps the epoch version for migration, and adds regression coverage for token rotation and real identity changes.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction path: changing a Claude CLI `type: "token"` credential changes the resolved epoch, and same-version epoch mismatches reset reusable sessions with `reason=auth-epoch`.

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer, paired with a version bump and regression tests, is the narrow maintainable fix; parser hardening can remain a separate defense-in-depth follow-up.
