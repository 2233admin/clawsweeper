## Summary

Keep open. Current main still hashes Claude token-shaped credentials and token auth-profile token material into the CLI auth epoch, so token rotation can still produce same-version auth-epoch mismatches and reset active sessions. The linked open PR #74493 uses closing syntax for this issue and should be reviewed or replaced before the issue is closed.

## What This Changes

Claude CLI auth epochs should remain stable during OAuth token rotation or transient token-shaped credential reads so active conversations are not reset mid-session.

## Best Possible Solution

Review and land #74493 or an equivalent narrow patch that makes Claude token-shaped epochs identity-based, removes raw token material from token auth-profile epoch hashing where appropriate, bumps the auth-epoch version, and keeps regression coverage for both rotation stability and identity changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to return a Claude CLI credential with type "token", change only credential.token, and compare resolveCliAuthEpoch; current main changes the epoch, and same-version epoch mismatches reset sessions with reason "auth-epoch".

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer is the narrowest maintainable fix for the observed reset loop; parser hardening can remain defense-in-depth after the main token-epoch contract is fixed.
