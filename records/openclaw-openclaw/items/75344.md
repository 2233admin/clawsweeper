## Summary

Keep open. Current main still does not preserve normal structured heartbeat-task execution when due commitments are present, and this PR is a narrow code/test repair for that regression rather than something already implemented on main.

## What This Changes

The branch removes due commitments from heartbeat file-gate bypassing, computes structured due heartbeat tasks separately, suppresses commitment delivery while those tasks run, and adds regression coverage.

## Best Possible Solution

Land a narrow separation where structured due heartbeat tasks run first with normal tool policy and due commitments remain pending for a later no-tools heartbeat turn, with focused regression coverage around both the prompt contents and commitment store state.

## Reproduction Assessment

Yes. A session with `commitments.enabled`, `heartbeat.target: "last"`, a due commitment, and a due structured `HEARTBEAT.md` task exercises the bad path; the PR's new test encodes that setup and expected no-tools avoidance.

## Solution Assessment

Yes for the stated structured-task regression. The patch keeps the commitment no-tools boundary intact while avoiding a mixed prompt, which is narrower than changing global tool policy for commitment turns.
