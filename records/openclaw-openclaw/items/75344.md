## Best Possible Solution

Land a narrow fix that keeps untrusted commitment delivery in no-tools mode while ensuring unrelated due heartbeat tasks run with their normal tool policy; if commitment freshness is important for always-due task setups, extend the fix with a separate no-tools commitment turn after the task turn.

## Solution Assessment

Yes for the narrow regression. The PR preserves the existing no-tools commitment safety boundary and separates unrelated due task prompts; the main alternative is a second no-tools commitment turn instead of deferral.
