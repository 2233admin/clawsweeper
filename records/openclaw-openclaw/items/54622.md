## Summary

Current main still serializes same-session agent runs, but the lane-wait data remains diagnostic/logging-only and is not surfaced as a typed Gateway or CLI busy/timeout error. The prior ClawSweeper comment kept this open, and the current db40ec40 source still supports that decision.

## Best Possible Solution

Keep this open and fix the Gateway/CLI contract. Add a typed session-lane busy or lane-wait-timeout result from the queue/agent dispatch path, propagate lane, queueAhead, and waitedMs through the Gateway final/error response, make `openclaw agent` print an actionable message, and add focused regression coverage for blocked same-session CLI dispatch. If waiting remains the default, add and document an explicit queue policy such as `--force-queue` or `--wait`.
