## Summary

Keep open. Current main has partial coverage for fallback lifecycle events, plugin prompt/tool hooks, last-known-good config validation recovery, subagent depth/concurrency caps, usage-cost reporting, and on-demand model probes, but the umbrella still tracks unresolved reliability, security, and product-policy gaps. The related failover follow-up covers only a narrow hook/event slice and does not supersede the non-verbose notice, context re-injection, rollback, filesystem-policy, prompt-injection, budget, or scheduled-health-routing requests.

## What This Changes

The issue requests a consolidated set of reliability, safety, hook, recovery, budget, model-health, and policy improvements for intensive daily OpenClaw use.

## Best Possible Solution

Keep the umbrella open for maintainer triage or replace it with focused canonical issues before closing it. Split the remaining work by surface: fallback visibility/context re-injection, config-write health rollback, crash recovery, doctor preview/diff, per-path filesystem policy, repeated subagent failure protection, prompt-injection policy, spend gates, and scheduled model-health routing.
