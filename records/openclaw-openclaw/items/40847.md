## Summary

Keep open. The requested diagnostic improvement is still useful, but this PR’s patch does not reliably implement it: the no-provider warning is placed after an earlier return, and all-skipped image attempts still produce no warning.

## What This Changes

The PR adds warning logs in `src/media-understanding/runner.ts` when selected image attachments produce zero media-understanding outputs, including failed-attempt/provider context and a no-provider message.

## Best Possible Solution

Implement the diagnostic in the runner control flow: warn before the no-provider early return, warn when selected image attempts all skip or return empty output, keep failed-attempt context, use the repository logger, and cover each path with focused media-understanding tests.
