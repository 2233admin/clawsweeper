## Summary

Keep open: current main still has the mtime-based models-config cache, so the PR is not obsolete. The PR is useful, but the final patch still leaves a size-only oversize hash sentinel that can produce stale cache hits for same-length oversized `models.json` or `auth-profiles.json` edits, matching the unresolved Aisle/Greptile cache-integrity concern.

## What This Changes

The PR replaces mtime-based `ensureOpenClawModelsJson` cache inputs with content-derived auth-profile fingerprints, adds a stored `models.json` content hash for drift checks, hardens models file hashing/chmod paths, and adds fingerprint-cache tests.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and change `safeHashRegularFile` so oversized or unsafe reads cannot produce stable cache hits; fail closed to force re-plan or hash bounded content plus size while preserving the new content-hash cache behavior.

## Reproduction Assessment

Yes. A focused unit test can create an oversized `models.json` or `auth-profiles.json`, warm the cache, rewrite different content with the same byte length, and observe the same `oversize:<size>` value keeping the cache hot instead of forcing a re-plan.

## Solution Assessment

No. The content-hash direction is the right fix for mtime churn, but the size-only oversize sentinel is not the safest implementation for a cache-integrity check; failing closed or hashing bounded content would be narrower and more defensible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Force misses for oversized hash inputs:** `src/agents/models-config.ts:111-115`
  - body: When `lst.size > maxBytes`, `safeHashRegularFile` returns `oversize:${lst.size}`. The ready-cache check compares only that string, so a same-length edit to an oversized `models.json` or `auth-profiles.json` is treated as unchanged and bypasses the drift/auth-profile invalidation this PR is adding. Return `null` to force a re-plan, or hash bounded content plus the size/truncation marker.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff improves several file-handling paths, but the size-only oversize sentinel leaves a concrete cache-integrity bypass to fix before merge.

Concerns:

- **[medium] Size-only oversize sentinel permits stale cache hits:** `src/agents/models-config.ts:115`
  - body: For oversized files, the proposed hash depends only on byte length. Same-size content changes to `models.json` or `auth-profiles.json` can therefore keep the ready cache valid even though the PR promises content/drift detection.
  - confidence: 0.91

  - body: When `O_NOFOLLOW` is unavailable, the final chmod/hash paths fall back to plain `O_RDONLY`; that leaves the symlink race only partially mitigated on such platforms. This is lower impact than the sentinel issue but should be called out or guarded explicitly.
  - confidence: 0.68
