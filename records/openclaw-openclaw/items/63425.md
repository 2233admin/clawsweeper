## Summary

Keep open. Current main still lets a stale positive session `contextTokens` win in the normal `/status` path even when `channels.modelByChannel` has selected the active provider/model and no explicit session `/model` override is present.

## What This Changes

The issue asks for `/status` to show the active `modelByChannel` model's context window instead of an older persisted session `contextTokens` value.

## Best Possible Solution

Teach status/session reconciliation to treat a channel-selected model as authoritative for the displayed context window when there is no explicit session model override, while preserving `/model` precedence, configured context caps, and fallback runtime snapshot behavior.
