## Summary

Keep open. Current main still implements approval-required gateway and node exec as an immediate `approval-pending` tool result plus a detached approval/follow-up path, so the requested turn-pause behavior is not implemented.

## What This Changes

The issue requests that foreground exec approval requests pause or suspend the active agent turn until the user approves, denies, or the request times out.

## Best Possible Solution

Define one explicit core contract for foreground interactive exec approvals: either the tool call awaits the final approve/deny/timeout result, or the current turn hard-stops in a resumable pending state until the decision is known. Keep background/headless behavior separate and cover gateway, node, denial, timeout, heartbeat, and follow-up delivery paths.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: a foreground gateway or node exec that requires approval returns `details.status = "approval-pending"` after registration, while a detached continuation waits for the final decision and later emits `Exec finished` or `Exec denied`.

## Solution Assessment

Unclear. Awaiting the approval decision matches the issue request, but current docs and tests intentionally encode immediate pending return plus async follow-up, so maintainers should choose whether the safest contract is blocking the tool call or suspending/resuming the turn after a pending marker.
