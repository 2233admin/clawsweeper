## Summary

Keep #70953 open. Current main still treats foreground exec approval as an immediate `approval-pending` tool result with detached follow-up execution after approval resolution, not as a hard pause of the agent turn. The issue’s heartbeat comment also remains relevant because heartbeat is not classified as a headless exec trigger in the approval fallback path.

## Best Possible Solution

Keep #70953 open. The best path is a core agent/exec change that makes foreground exec approval a hard stop: either await `exec.approval.waitDecision` before returning to the model, or add runner-level suspension so no subsequent model/tool steps execute until approval is allowed, denied, or timed out. Preserve explicit background behavior separately, make heartbeat/automation deterministic, update docs, and add regression coverage that prevents multiple pending foreground approval requests and prevents model continuation based on an unresolved approval.
