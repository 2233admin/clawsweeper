## Summary

Keep open. Current main still documents and implements memory as separate controls: `memory.backend` selects builtin vs QMD recall, `memory.qmd.*` configures QMD, `agents.defaults.memorySearch.*` controls builtin SQLite indexing, and `agents.defaults.compaction.memoryFlush` independently writes daily Markdown. The issueâ€™s central request for a strict single memory-mode contract with built-in-only, qmd-only, mixed behavior, and workspace-scoped SQLite storage remains unresolved.

## Best Possible Solution

Keep this as a focused maintainer design task. Define either `memory.type` or a documented successor that covers builtin-only, qmd-only, and mixed semantics across write targets, search/index backend, QMD fallback strictness, active-memory recall, per-agent/per-workspace SQLite scoping, migration behavior, config schema/help/docs, and doctor guidance. Treat narrower memory CRUD work such as #70540 as related but not sufficient to close this issue.
