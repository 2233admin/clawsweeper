## Summary

Keep open. Current main still maps adaptive Claude Opus 4.7 `minimal`/`low` to `low` and emits that as `output_config.effort`; this PR remains a focused implementation candidate for the paired open bug #70322, but maintainers should review the provider scope because related #69928 reports a broader Copilot-only medium-capability mismatch.

## What This Changes

This PR changes the Anthropic Messages transport so Claude Opus 4.7 maps `minimal` and `low` adaptive effort to `medium`, and adds transport tests for Opus 4.7 low/minimal plus an Opus 4.6 low regression guard.

## Best Possible Solution

The desired end state is a regression-tested compatibility rule for the `github-copilot` plus Claude Opus 4.7 adaptive-effort surface that sends only supported effort values, preserves direct Anthropic behavior, and reconciles the low/minimal report with the broader medium-only report.
