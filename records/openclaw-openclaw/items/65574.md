## Summary

Keep this PR open. Current main still does not propagate ambient gateway proxy env or NODE_OPTIONS into MCP stdio child processes; it only forwards configured server env, then the transport merges that with the MCP SDK's restricted default environment. The PR addresses a real production failure linked from #65525, but it is stale against current main's stdio env safety filter and needs maintainer review around ambient NODE_OPTIONS policy before it can land.

## Best Possible Solution

Keep this PR open for a focused rebase/rework. The best fix is a current-main patch in the MCP stdio launch path that propagates safe ambient proxy-related env with explicit precedence tests, preserves the configured-env safety filter and warning callbacks, and makes a deliberate maintainer-approved decision for NODE_OPTIONS with docs and changelog aligned. After that lands, #65525 can close as implemented.
