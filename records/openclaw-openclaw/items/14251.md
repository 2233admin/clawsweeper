## Summary

Current main still leaves the reported LaunchDaemon lifecycle gap open: macOS gateway stop/uninstall are wired to the managed per-user LaunchAgent path, while system-level LaunchDaemon support remains a related product/security decision across the open LaunchDaemon requests.

## What This Changes

Request that macOS gateway stop/uninstall detect system-level `/Library/LaunchDaemons/ai.openclaw.gateway.plist` installs and either manage them or give accurate elevated-command guidance.

## Best Possible Solution

Define one macOS launchd scope contract, then update gateway lifecycle operations to detect matching `/Library/LaunchDaemons` installs, avoid false user-scope success when a system daemon remains active, provide safe elevated-command guidance or an explicit system lifecycle path, and align status, doctor/docs, and regression tests with that contract.
