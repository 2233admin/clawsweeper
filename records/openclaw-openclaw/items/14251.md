## Summary

Keep open. Current main still routes macOS gateway lifecycle through the user LaunchAgent path, and the existing deep-status scan is diagnostic only and filters out the canonical gateway label rather than managing a matching system LaunchDaemon. The related LaunchDaemon mode work is not a complete replacement for the stop/uninstall lifecycle request, and the privileged scope contract still needs maintainer review.

## What This Changes

Request gateway stop, uninstall, and status behavior that detects macOS system-level LaunchDaemons under `/Library/LaunchDaemons` and gives accurate lifecycle control or guidance.

## Best Possible Solution

Define one macOS launchd scope model, then update install/status/stop/uninstall/restart, docs, and regression tests so user LaunchAgents and system LaunchDaemons are detected accurately and privileged operations either use an explicit safe path or return precise elevated commands without false success.
