## Summary

Current main still has the macOS system LaunchDaemon lifecycle gap. The gateway stop/uninstall command surface remains scoped to the per-user LaunchAgent path and GUI launchd domain, while system LaunchDaemon discovery is diagnostic-only via deep status/doctor paths. The prior maintainer/Codex review in the issue also explicitly kept this open and tied it to related LaunchDaemon scope work in #51860 and #55208.

## Best Possible Solution

Keep this issue open and resolve it through a coordinated macOS launchd scope design: detect `/Library/LaunchDaemons/ai.openclaw.gateway.plist` during stop/uninstall, avoid user-scope success responses while a system KeepAlive daemon remains active, provide exact safe sudo bootout/removal guidance or an explicit system lifecycle path, and add focused tests/docs for user LaunchAgents versus system LaunchDaemons alongside #51860 and #55208.
