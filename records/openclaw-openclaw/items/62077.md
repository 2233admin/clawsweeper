## Summary

Keep open. Current main still prepends full `Conversation info` and `Sender` metadata blocks for direct Telegram messages, there is no shipped config knob for compact/minimal direct-chat metadata, and the related compact-metadata PR is still open and unmerged.

## What This Changes

Request to add an optional minimal or compact inbound metadata mode for direct chats so repeated Telegram DM metadata does not bloat long conversations or encourage envelope-shaped hallucinations.

## Best Possible Solution

Add a backwards-compatible `messages.inboundMetadataMode` or equivalent config path that is passed into `buildInboundUserContextPrefix`, compacts direct-chat metadata without weakening untrusted-context framing, preserves full metadata for groups and rich contexts, and updates config schema, docs, sanitizer coverage, and inbound prompt tests.

## Reproduction Assessment

Yes. The prompt-builder seam is enough to reproduce with high confidence: a Telegram DM context has `ChatType: "direct"` and `OriginatingChannel: "telegram"`, which current main turns into full conversation and sender metadata blocks.

## Solution Assessment

Unclear. The open compact-direct PR is a plausible narrow solution, but it is not merged and does not exactly implement the once-per-session sender-info variant, so maintainer review should decide whether that PR is the desired product shape.
