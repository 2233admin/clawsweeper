## Summary

Keep this PR open. Current main still always includes `BOOTSTRAP.md` in `loadWorkspaceBootstrapFiles` and reports it as missing on failed reads, even after workspace setup is complete. The separate Control UI fix from #17491 only affects the file-list RPC, not the runtime context/bootstrap loader targeted by this PR. The discussion also contains useful independent reproduction and live-patch verification, so this remains a focused implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent core change in `src/agents/workspace.ts`: check explicit setup completion using a read-only state lookup, omit only genuinely absent (`ENOENT`) `BOOTSTRAP.md` after setup completion, keep present files and non-ENOENT failures visible, and retain the added regression tests.
