## Summary

Keep this PR open. Current main still always includes `BOOTSTRAP.md` in the runtime workspace bootstrap loader and turns a failed read into a missing context entry, even when workspace setup has completed. The merged Control UI behavior from #17491 is a separate file-list path and does not fix the `/context`/runtime injection behavior targeted by this PR. The PR remains a focused implementation candidate, with discussion and independent live-patch verification supporting the same root cause.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent core change in `src/agents/workspace.ts`: use an explicit read-only setup-completion lookup, omit only genuinely absent (`ENOENT`) `BOOTSTRAP.md` after setup completion, keep present files and non-ENOENT failures visible, and retain the added regression tests in `src/agents/workspace.test.ts`.
