## Summary

Keep open. Current main still exposes and evaluates the Discord guild/channel access model as `requireMention` plus `users`/`roles` allowlists. It has partial role allowlist support, but it does not implement the requested `bypass`/`mention`/`deny` lists, permission shortcuts, deny precedence, per-channel additive inheritance with `inherit: false`, or a backward-compatible migration path from the old fields. The open #73387 overlap is a related design prerequisite, not a reason to close this broader access-control request.

## Best Possible Solution

Keep this open for a Discord plugin access-control design pass. The best path is a backward-compatible schema and evaluator in the bundled Discord plugin, aligned with #73387, with docs, config schema updates, doctor/migration behavior for `requireMention` and `users`, and regression tests for deny precedence, role/permission matching, mention-only users, bypass users, and channel inheritance.
