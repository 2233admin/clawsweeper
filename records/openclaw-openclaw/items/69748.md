## Summary

Keep open. Current main still implements Discord guild/channel access as `requireMention` plus flat `users`/`roles` allowlists, with one boolean mention policy per guild/channel. The requested tiered `deny`/`bypass`/`mention` evaluator, permission shortcuts, additive per-channel inheritance with `inherit: false`, and backward-compatible migration/deprecation path are not implemented. Related #73387 is still open and is a design dependency, not a complete superseding fix.

## What This Changes

The issue requests replacing Discord `requireMention` and `users` access control with `bypass`, `mention`, and `deny` lists that support roles, permission shortcuts, and per-channel inheritance/overrides.

## Best Possible Solution

Keep this open for a Discord plugin access-control design pass. The best path is a backward-compatible schema and evaluator in the bundled Discord plugin, aligned with #73387: support deny-first `bypass`/`mention`/`deny` lists, role and permission entries, groupPolicy-aware channel inheritance, explicit `enabled: false` and `inherit: false` behavior, old-field compatibility warnings or doctor migration, public docs, schema updates, and regression tests for mixed senders and channel overrides.
