## Summary

Current main still has the reported race: outbound delivery creates/mirrors session metadata, while generic Telegram callback data is immediately routed as a synthetic inbound message, and there is no current prewarm helper that hydrates the skills/bootstrap snapshot before that first callback turn. The issue should stay open because the linked open PR #68433 uses closing syntax and is the active implementation candidate, but it has not merged.

## What This Changes

Ensure the first Telegram inline-button callback for a freshly mirrored outbound session runs only after the session's workspace/skills bootstrap context is available.

## Best Possible Solution

Land a focused fix that prewarms the mirrored outbound session's skills/bootstrap snapshot immediately after session metadata is recorded, without setting `systemSent` or consuming first-turn lifecycle behavior, then let the linked PR or an equivalent replacement close this issue.
