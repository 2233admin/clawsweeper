## Summary

Keep this PR open. Current main still returns `channel` for nullish Mattermost channel types, current tests still lock in that fallback, and key dispatch paths still route with the resolved `kind` rather than failing closed. The PR is also paired with open dependent PR #57565 and still needs maintainer review for the blank-`channel_type` fallback/public `ChatType` contract question.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, address the blank-`channel_type` fallback so API lookup wins before dropping, decide whether `unknown` should be a public `ChatType`/SDK contract or a Mattermost-local sentinel, and then land this PR or an equivalent Mattermost fail-closed fix with mapper, dispatch, and regression-test coverage.
