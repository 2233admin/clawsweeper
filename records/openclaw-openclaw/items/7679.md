## Summary

Keep this issue open. Current main still defaults Telegram DM access to pairing, still issues pairing challenges to unknown DM senders, and setup/docs/tests still use manual numeric allowFrom entry rather than token-verified owner ID capture during onboarding.

## Best Possible Solution

Keep this issue open and implement it as Telegram-owned onboarding/security work: generate a short verification token, ask the intended owner to send that exact token to the bot, capture the matching numeric Telegram from.id, write dmPolicy="allowlist" plus allowFrom for that account, update schema/runtime/docs/tests, and preserve compatibility for existing explicit pairing-mode configs.
