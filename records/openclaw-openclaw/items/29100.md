## Summary

Current main still lacks a supported cost metadata overlay for Amazon Bedrock auto-discovered models. The plugin discovery schema, legacy config types/schema, cache key, docs, and discovery conversion path still omit costs, while discovered foundation and inference-profile models receive zero-cost metadata; the related implementation PR #31160 is closed unmerged and #53286 remains open for the aws-sdk display-layer part.

## What This Changes

Allow cost-per-token metadata to be attached to Amazon Bedrock models discovered through Bedrock discovery without duplicating explicit provider model entries.

## Best Possible Solution

Implement a backward-compatible Amazon Bedrock plugin pricing overlay, preferably under plugins.entries.amazon-bedrock.config.discovery.costs with legacy models.bedrockDiscovery.costs compatibility only where needed. Apply configured costs to both foundation model IDs and inference profile IDs, include normalized pricing in the discovery cache key, update manifest/schema/help/docs/tests, and coordinate with #53286 so configured Bedrock costs are displayed for aws-sdk-auth status and usage surfaces.

Implement cost metadata support for Amazon Bedrock auto-discovered models. Add a backward-compatible pricing overlay, preferably `plugins.entries.amazon-bedrock.config.discovery.costs` keyed by Bedrock model ID, with legacy `models.bedrockDiscovery.costs` migration/compatibility only where needed.
