## Summary

Keep this issue open. Current main still has no supported cost metadata overlay for Amazon Bedrock auto-discovered models: the plugin discovery schema, shared Bedrock discovery type, strict config schema, discovery implementation, cache key, tests, and Bedrock docs all omit a costs map. The direct implementation PR #31160 is closed and unmerged, while #53286 remains a related open display-layer follow-up for Bedrock aws-sdk cost visibility.

## Best Possible Solution

Keep this open and implement a backward-compatible pricing overlay in the Amazon Bedrock plugin, preferably under `plugins.entries.amazon-bedrock.config.discovery.costs` with legacy `models.bedrockDiscovery` migration only where needed. Apply the overlay to foundation model IDs and inference profile IDs, include deterministic cache-key handling, update plugin manifest/schema/help/docs/tests, and coordinate with #53286 so configured Bedrock costs are actually shown for aws-sdk-auth usage surfaces.

## Clownfish Work Prompt

Implement cost metadata support for Amazon Bedrock auto-discovered models. Current main assigns `DEFAULT_COST` to discovered Bedrock foundation models and inference profiles, while `plugins.entries.amazon-bedrock.config.discovery` has no costs map. Add a backward-compatible plugin-owned overlay, preferably `plugins.entries.amazon-bedrock.config.discovery.costs`, keyed by Bedrock model/profile ID with `input`, `output`, `cacheRead`, and `cacheWrite` USD-per-1M-token fields. Preserve existing discovery behavior and AWS credential behavior; this must be metadata only.

Expected boundary:
- Start in the bundled Amazon Bedrock plugin, not generic core, unless a small shared type/schema seam is required.
- Update `BedrockDiscoveryConfig` and the plugin manifest schema/UI hints/help for the new cost map.
- Apply costs to discovered foundation model IDs and inference profile IDs. Consider deterministic matching for profile IDs like `us.anthropic...` and base foundation IDs like `anthropic...`.
- Include the normalized costs map in the Bedrock discovery cache key so pricing config changes do not reuse stale discovery results.
- Preserve legacy config migration behavior for `models.bedrockDiscovery` only to the extent needed for compatibility; do not reintroduce broad core ownership of Bedrock-specific policy.
- Coordinate with #53286: if configured Bedrock aws-sdk costs still do not appear in `/status` or per-response usage output, update the cost display gating/tests so auth mode does not hide explicitly configured Bedrock costs.
- Update `docs/providers/bedrock.md` and `docs/reference/token-use.md` as needed. Add a user-facing changelog entry with `Thanks @mg-mavios`; also credit @Pan-Binghong and/or @NikhilGaddam if the PR directly covers their related reports or prior implementation context.
- Do not change AWS auth resolution, model routing, provider IDs, or unrelated pricing bootstrap behavior.

Related refs: this issue, #31160, #53286.
