## Summary

Keep open. Current main still lacks a configurable stopReason:"length" policy for native mode:"run" subagents: the runner preserves stopReason metadata, but lifecycle and subagent completion still map a normal terminal end to an ok outcome, and the config/tool/docs surfaces do not expose onLengthTruncation or maxContinuations.

## What This Changes

Request configurable runtime handling for mode:"run" subagent sessions that hit provider stopReason:"length", at minimum surfacing a hard failure and optionally continuing with a bounded resume prompt.

## Best Possible Solution

Keep this open and implement a narrow first PR for native mode:"run" subagents: when the child terminal assistant has stopReason:"length", the parent/orchestrator should receive an explicit error or partial signal instead of a plain ok outcome. Start with an opt-in fail policy plus config schema, docs, and focused tests; bounded continue-mode should wait until replay and side-effect semantics are designed.
