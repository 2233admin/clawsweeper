## Summary

Keep open. Current main has adjacent safeguards for empty/error/reasoning-only turns and it propagates some stopReason metadata, but it still does not implement the requested configurable stopReason="length" policy for mode:"run" subagent sessions. The discussion includes concrete production and local-model reproductions, and related #63210/#63689 context points to the same truncation family rather than an obsolete report.

## Best Possible Solution

Keep this open for agents/subagents maintainer follow-up. The safest first product/code direction is a narrow fail-mode signal for native mode:"run" subagent runs when the child assistant stops with stopReason="length", with tests proving the parent/orchestrator sees a failed run rather than an ok partial result. Continue-mode should follow only after replay safety and side-effect behavior are specified.
