## Summary

Keep this issue open. Current main still suppresses an explicit `pty:true` request whenever exec resolves to a sandbox runtime, without a targeted warning/error, and the same-repo fix PR #39349 remains open with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review/land #39349 or replace it with an equivalent narrow fix. The fix should either pass `pty:true` through to sandbox backends that support it, or keep suppression but emit a clear warning/rejection before `usePty` is set false.
