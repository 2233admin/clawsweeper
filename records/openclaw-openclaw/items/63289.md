## Summary

Keep this issue open. Current main has tool guidance, tool catalog plumbing, Anthropic `tool_choice` forwarding, OpenResponses client-tool forcing, and the bundled gog calendar skill, but I did not find a current-main guard that prevents an assistant from answering an explicit calendar or raw shell-output request without emitting any tool call first. The related reports cover malformed/missing tool results or other delivery paths, not this zero-tool-call calendar path.

## Best Possible Solution

Keep this issue open and have a maintainer define a narrow contract for explicit freshness or raw-output requests, such as calendar checks and “run this command and paste raw output.” A good implementation would likely live around the Pi embedded runner/prompt/tool policy boundary, add a deterministic guard or provider request policy for explicit tool-required turns, and include a mocked Anthropic/gog-style regression plus trajectory or transcript evidence that the assistant cannot finalize without the required tool event. Avoid turning every arbitrary factual question into mandatory tool use.
