## Summary

Keep open. Current main still lacks a supported cross-channel pre-processing text acknowledgement: message config has ack reactions/status reactions but no ackText field, the runtime schema is strict, docs are reaction/typing-only, and the closest before_dispatch hook is terminal rather than send-and-continue. An open Feishu-only PR (#47961) is relevant partial work but does not cover the requested Slack/WhatsApp/WebChat/core contract.

## Best Possible Solution

Keep this issue open for maintainer product/API follow-up. The best path is either a supported messages.ackText configuration with scope plus per-channel/account override behavior, or a documented non-terminal pre-processing hook that can send interim text and continue normal dispatch. #47961 can be reviewed as a Feishu-specific partial improvement, but it should not close this cross-channel request by itself.
