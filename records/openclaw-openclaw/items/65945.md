## Summary

Keep open. Current main still handles Node/RPC image attachments as agent input and supports delivery of agent-produced media, but it does not preserve an opt-in auto-forward flag or delivery-ready copies of the original Node attachments for outbound IM delivery. The request is valid, but the behavior needs maintainer product/security review before implementation.

## What This Changes

Request an opt-in Gateway/Node `agent.request` behavior that forwards original image attachments to the selected messaging recipient when `deliver: true`, even if the agent reply has no media.

## Best Possible Solution

Keep this open until maintainers approve the contract, then add an explicit opt-in field, carry sanitized delivery-ready attachment references through the gateway command path, and inject them only when `deliver: true`, the target is authorized/resolved, the channel supports the media, and the agent did not provide its own media.
