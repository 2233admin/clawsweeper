## Summary

Current main still blocks or skips unauthorized WhatsApp group traffic but does not implement auto-leave, a WhatsApp group-join event/webhook, or a WhatsApp leave-group message action. Related #11489 remains open and broader, so it does not supersede this WhatsApp-specific request.

## Best Possible Solution

Keep this open for deliberate bundled WhatsApp plugin work: add opt-in config/schema/docs for unauthorized-group auto-leave, implement an account- and target-authorized WhatsApp Web/Baileys group leave operation, add group-add/participant-update detection or equivalent owner notification, consider a WhatsApp leaveGroup message action, cover blocked-group and allowlist behavior with tests, and coordinate the event shape with #11489.
