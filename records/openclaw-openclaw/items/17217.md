## Summary

Keep open. Current main has documented session lifecycle controls plus `/new`, `/reset`, `sessions.reset`, and `sessions.compact`, but `contextPruning.ttl` is still documented and implemented as in-memory old tool-result trimming during prompt assembly. It does not expire normal conversation history on wall-clock TTL, and there is no explicit prune-context gateway/API method matching the request.

## Best Possible Solution

Keep this open for maintainer product/API follow-up. The best path is either to add an explicit gateway/protocol prune-context surface plus context-engine/Pi implementation and tests for TTL-based conversation-history expiry or summarization, or to clarify docs/config that `contextPruning.ttl` does not expire normal conversation history and point users to `/compact`, `/new`, `/reset`, and `sessions.reset` as the supported controls.
