## Summary

Keep open. Current main documents session lifecycle and has `/new`, `/reset`, `sessions.reset`, and `sessions.compact`, but cache-ttl pruning is still scoped to in-memory old tool-result trimming during prompt/context assembly. It does not expire normal conversation history on wall-clock TTL, and there is still no explicit prune-context gateway/API method matching the request.

## Best Possible Solution

Keep this open for maintainer product work. Either add an explicit gateway/protocol prune-context surface plus Pi/context-engine implementation and tests for TTL-based conversation-history expiry/summarization, or clarify docs that `contextPruning.ttl` does not expire normal conversation history and point users to `/compact`, `/new`, `/reset`, and `sessions.reset` as the supported controls.
