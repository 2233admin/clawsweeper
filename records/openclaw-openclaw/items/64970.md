## Summary

Keep open. Current main still hardcodes the per-text-block `sessions_history` truncation limit at 4000 characters, while the comparable bash output limits already use the existing env-int helper. The linked implementation PR #65029 proposed the same narrow fix but is closed unmerged, so the reported behavior remains actionable.

## What This Changes

Allow `SESSIONS_HISTORY_TEXT_MAX_CHARS` to override the current 4000-character `sessions_history` text truncation cap.

## Best Possible Solution

Add a narrow env-backed limit for `sessions_history` text blocks while preserving the 4000 default, redaction behavior, and the existing 80 KiB hard response cap. Cover the override with a focused regression test, and document/changelog the user-facing knob if maintainers want it supported publicly.
