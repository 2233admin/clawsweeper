## Summary

Close as ClawHub/third-party integration work. This PR adds a workspace-local, service-specific AlpaCore MCP bridge and VS Code MCP auto-start config. Current main already has generic MCP configuration and bundle MCP support, and the project direction routes optional capabilities, MCP bundles, and third-party integrations through plugins/ClawHub unless a missing generic core API is identified. The security pass also found the PR would add workspace auto-start code execution and local HTTP calls, which is a better fit for opt-in plugin/bundle installation than core workspace defaults.

## Best Possible Solution

Keep OpenClaw core unchanged. Package the AlpaCore bridge and VS Code MCP definition as an external ClawHub/plugin bundle, with opt-in install docs and any needed security notes. Reopen core work only if that packaging path exposes a missing generic MCP/plugin API that OpenClaw should add.
