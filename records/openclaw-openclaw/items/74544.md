## What This Changes

The issue asks for `openclaw memory status --deep` to complete or fail with observable progress and a bounded diagnostic error instead of silently waiting up to the embedding batch timeout.

## Best Possible Solution

Review and land PR #75106 if its budget and output contract are acceptable, or land an equivalent memory-core fix that bounds the status-only embedding probe, emits non-TTY progress, and reports a distinct timeout state while leaving real indexing batch budgets intact.
