## Summary

Keep open. Current main only partially disambiguates some rows by rendering the raw parsed channel id when an agent identity is available, but the Sessions UI still does not carry Gateway channel fields, resolve channel catalog labels/icons, or render a channel badge/name for the session list; the optional custom label request is already covered by the existing Label input.

## What This Changes

The issue requests showing a user-facing channel name or icon in the Control UI sessions list instead of relying on raw channel-bearing session keys.

## Best Possible Solution

The desired end state is a Sessions list that carries channel identity through the browser row contract and live update path, resolves it through the existing channel catalog metadata, and renders a stable channel label/badge while preserving label editing, search, sort, pagination, selection, and chat navigation.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/53378. In the Control UI Sessions list, show a user-facing channel identity for each channel-backed session instead of relying on raw session keys or only `Agent Name (channel-id)`. Current Gateway rows already expose channel metadata in `src/gateway/session-utils.types.ts` and `src/gateway/session-utils.ts`, and `sessions.changed` broadcasts channel fields, but the browser `GatewaySessionRow`, `SESSION_EVENT_ROW_FIELDS`, `SessionsProps`, and Sessions rendering do not carry or resolve channel labels/icons. Keep the fix narrow: extend the UI row type with the needed channel fields, preserve those fields in live `sessions.changed` merges, pass `channelsSnapshot.channelLabels`, `channelSystemImages`, and/or `channelMeta` from `ui/src/ui/app-render.ts` into `renderSessions`, resolve a display label from row channel metadata with parsed-key fallback, and render a compact channel badge or column in `ui/src/ui/views/sessions.ts`. Preserve custom Label editing, agent identity labels, search, sort, pagination, row selection, and chat navigation. Update focused tests in `ui/src/ui/views/sessions.test.ts` and `ui/src/ui/controllers/sessions.test.ts`; update `ui/src/i18n/locales/en.ts` for any new visible string. Because this is user-facing UI behavior, add a single active changelog fix entry crediting `Thanks @gelibing8-rgb` if the repo’s current changelog policy requires it. Do not move channel-owned behavior into core and do not change unrelated session storage semantics.
