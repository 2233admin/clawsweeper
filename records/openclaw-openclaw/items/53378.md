## Summary

Current main still does not implement the requested Control UI Sessions channel identity display. The Sessions table still has no Channel column or badge, the key cell only switches away from the raw key when agent identity metadata exists, and the UI row/event path still does not type or preserve the backend channel metadata needed for a robust Telegram/Feishu/plugin-channel label.

## Best Possible Solution

Keep this issue open. Implement the Sessions list feature in the Control UI by carrying channel, groupChannel, chatType, and origin through the UI GatewaySessionRow and SESSION_EVENT_ROW_FIELDS; pass channelLabels/channelMeta into renderSessions; render a compact Channel column or key-cell badge for Telegram, Feishu, and plugin channels; and add focused view/controller tests for Telegram/Feishu rows without agent identity plus live update preservation.
