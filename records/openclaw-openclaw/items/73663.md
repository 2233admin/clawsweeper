## Summary

Keep open. Current main does not contain the Bedrock Opus 4.7 temperature guard requested by the issue, and the Bedrock failover classifier still does not recognize the reported nested Anthropic invalid_request_error as a failover-worthy schema rejection. The provided related PR #72149 is about the native Anthropic path, not the amazon-bedrock Converse path.

## Best Possible Solution

Keep this issue open. The best fix is provider-owned: add an amazon-bedrock payload patch or capability flag that omits inferenceConfig.temperature for us.anthropic.claude-opus-4-7, global.anthropic.claude-opus-4-7, and anthropic.claude-opus-4-7, with regression coverage around the Bedrock wrapper/onPayload path. Separately, classify Bedrock ValidationException messages containing nested Anthropic invalid_request_error, including deprecated temperature, as a non-retryable schema/format failure that surfaces clearly and advances the configured model fallback chain.
