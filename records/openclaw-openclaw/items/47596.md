## Summary

Keep open. Current main has the ACP `streamTo="parent"` relay and docs for parent progress summaries, but the relay still ignores tool, item, command output, patch, approval, and thinking streams. The event bus already has structured tool/thinking event types, so this is a real remaining ACP parent-relay visibility gap rather than an already-implemented request. Related #67590 is broader channel preview/tool-progress work, not a replacement for this ACP parent-session relay request.

## Best Possible Solution

Keep this open and define the ACP parent-relay visibility policy first. The likely implementation belongs in `src/agents/acp-spawn-parent-stream.ts`, using existing `tool`/`item`/`command_output`/`patch`/`approval`/`thinking` events to emit bounded, redacted summaries to the parent session, with tests covering verbose-level behavior and no raw result leakage by default.
