## Summary

Keep open. Current main still does not expose or forward Bedrock `requestMetadata` from `openclaw.json`. The generic `agents.*.params` config surface would accept arbitrary keys, but the embedded runner only forwards a small allowlist into pi-ai stream options, and the bundled Amazon Bedrock plugin only handles discovery, guardrails, region, and cache behavior. The provided related context for #60602/#60698 is relevant but does not supersede this broader configurable metadata request.

## Best Possible Solution

Add Bedrock-owned support in the bundled `amazon-bedrock` plugin rather than core hardcoding: define a validated `requestMetadata` config shape, decide merge precedence between provider/plugin config and `agents.defaults.models["amazon-bedrock/<model>"].params` or `agents.list[].params`, forward the sanitized record into the pi-ai Bedrock stream options, and update Bedrock docs plus targeted plugin/extra-param tests.
