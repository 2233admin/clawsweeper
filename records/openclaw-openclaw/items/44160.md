## Summary

Keep this issue open. Current main still lacks the requested Mattermost WebSocket ping/pong keepalive, and the existing Mattermost health path only polls the bot account `update_at` value. Open PR #57621 is the active implementation candidate for the same Mattermost half-open socket problem, but it is still open/unmerged in the provided GitHub context, so this issue should not be closed as implemented.

## Best Possible Solution

Keep this issue open and treat PR #57621, or a maintainer-owned successor, as the current implementation path. The best fix belongs in `extensions/mattermost/src/mattermost/monitor-websocket.ts`: extend the Mattermost WebSocket seam for ping/pong support, send periodic protocol pings, terminate on missed pong so `runWithReconnect` can recover, clean up all timers on close/error/abort/settle paths, and add focused Vitest coverage for healthy pong, missed pong termination, and timer cleanup. Broader Mattermost deaf-WebSocket tracking can remain in #41837.
