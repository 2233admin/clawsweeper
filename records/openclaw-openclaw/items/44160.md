## Summary

Keep open. Current main still lacks Mattermost WebSocket protocol ping/pong keepalive: the socket abstraction only listens for open/message/close/error and the connect path only starts the existing bot update_at health poll. The latest release tag also lacks ping/pong, while open PR #73979 is the active unmerged implementation candidate for this exact gap.

## Best Possible Solution

Keep this issue open until PR #73979 or an equivalent focused Mattermost plugin patch lands. The best fix belongs in extensions/mattermost/src/mattermost/monitor-websocket.ts: extend the websocket abstraction with ping/pong, send periodic protocol pings, terminate on missed pong so the existing reconnect loop recovers, clean up keepalive and update_at timers on all settle paths, and keep the tests/changelog from PR #73979 or equivalent coverage.
