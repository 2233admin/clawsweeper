## Summary

Current main still has the SQLite task registry binding `task: record.task` directly, and a live in-memory `node:sqlite` check still rejects object values with the same binding error this PR is intended to prevent. The PR remains a narrow, relevant fix rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent narrow fix that serializes non-string task values before SQLite binding, preserves existing string values unchanged, and adds the structured-payload regression test in `src/tasks/task-registry.store.test.ts`.
