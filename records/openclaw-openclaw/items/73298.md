## Summary

Keep open. Current main has strong adjacent fixes for the reported v2026.4.26 failure class, but the issue is broader than any one fixed defect: it combines Debian npm-global startup delay, Codex model fallback, OAuth refresh-token failures, gateway handshakes/timeouts, and near-unusable responsiveness. ## What This Changes

The issue reports a Debian npm-global Codex setup becoming severely slow after upgrading from OpenClaw 2026.4.2 to 2026.4.26, with logs implicating startup model warmup, unknown Codex model resolution, OAuth refresh failures, and gateway timeouts.

## Best Possible Solution

Keep the issue open until maintainers can retest the affected Debian npm-global Codex setup on current main or the next release after 388019f5. If the slowdown remains, capture startup trace and embedded-run stage timings, then split the fix by measured phase: gateway startup/prewarm in `src/gateway/server-startup-post-attach.ts`, Codex catalog/auth in `extensions/openai` and `src/agents/auth-profiles`, and run-prep/runtime-deps latency in `src/agents/pi-embedded-runner` or `src/plugins` only if traces implicate those paths.
