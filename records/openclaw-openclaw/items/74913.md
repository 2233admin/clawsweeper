## Summary

Keep open. Current main still lacks direct cron transcript mirroring, and this PR is an active contributor fix with a narrow cron-owned implementation, focused regression coverage, and no blocking functional or security finding from this read-only pass.

## What This Changes

The PR adds direct cron delivery transcript mirroring using the stored agent-scoped cron session key and adds regression tests for custom-session and media-only mirror payloads.

## Best Possible Solution

Land the cron-owned mirror fix after maintainer review and exact-head CI, keeping the implementation scoped to direct cron delivery and preserving the stored session-key/idempotency behavior covered by the new tests.

## Reproduction Assessment

Yes. The linked bug provides concrete cron CLI steps, and current main source shows the reproduction path: direct cron delivery can send externally through deliverOutboundPayloads without supplying the mirror required to append a transcript entry.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix for this bug: add the missing mirror at the existing direct-delivery call site using the stored cron session key and filtered outbound payload projection, instead of adding a broader session migration or core routing change.
