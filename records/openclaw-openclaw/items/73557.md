## Summary

Keep this PR open for normal maintainer review. Current main still runs implicit provider discovery before mode-specific handling, while the documented `models.mode: "replace"` contract says only configured providers should be used; the PR is a narrow implementation with focused regression coverage and no definite correctness findings from this review.

## What This Changes

The PR changes agent model-provider planning to skip `resolveImplicitProviders` when `models.mode` is `replace` and adds unit tests for replace, merge, and default behavior.

## Best Possible Solution

Land a narrow guard before implicit provider discovery when `models.mode` is `replace`, keep merge/default behavior and explicit-provider normalization unchanged, and track pricing bootstrap delays separately in #74020.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure `models.mode` as `replace` and call the planning path with an injected `resolveImplicitProviders`; the resolver is invoked before mode handling.

## Solution Assessment

Yes. The PR's callsite guard is the narrowest maintainable fix for this bug because it prevents the expensive discovery work before it starts while leaving merge/default discovery semantics intact.
