## Summary

Keep this PR open. Current main still violates the documented `models.mode: "replace"` contract, and the PR targets the right resolver boundary, but the exact head is not mergeable as submitted because CI reports a type error on the new implicit-provider assignment.

## What This Changes

The PR changes model-provider planning to compute `models.mode` before provider discovery, skip implicit provider discovery in replace mode, and add unit coverage for replace, merge, and default behavior.

## Best Possible Solution

Land a corrected narrow guard in `resolveProvidersForModelsJsonWithDeps` that avoids implicit discovery before it starts when `models.mode` is `replace`, preserves merge/default behavior and provider normalization, adds the required changelog credit, and leaves pricing bootstrap work to #74020.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure `models.mode: "replace"`, inject or observe `resolveImplicitProvidersForModelsJsonWithDeps`, and it is still called before mode handling; #66957 also includes concrete startup-delay logs.

## Solution Assessment

Yes in direction, but not as submitted. The resolver-callsite guard is the narrowest maintainable fix, but the current implementation must handle the default implicit resolver's optional return type before CI can pass.
