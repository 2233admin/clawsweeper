## Summary

Keep this PR open. Current main still resolves implicit providers before any `models.mode` branch, while the documented replace-mode contract says only configured providers should be used; the PR is a focused implementation candidate for the still-open linked bug rather than obsolete cleanup.

## What This Changes

The PR changes models-config planning to skip implicit provider discovery when `models.mode` is `replace` and adds a colocated unit test covering replace, merge, and default behavior.

## Best Possible Solution

Land a narrow guard before implicit discovery when `models.mode` is `replace`, keep merge/default behavior unchanged, preserve provider normalization and source-managed secret enforcement after explicit providers are selected, and leave pricing-bootstrap behavior to the separate tracking issue.
