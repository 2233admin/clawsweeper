## Summary

Keep this PR open. Current main still drops unmentioned Feishu group thread replies before dispatch, and the proposed config surface is not implemented. The PR has useful focused work and live-verification context, but it also has an unresolved P1 review concern in the provided discussion and overlaps with open PR #40530, so maintainers should choose the canonical Feishu mention-gating design rather than auto-close it.

## Best Possible Solution

Keep this PR open until a maintainer chooses the Feishu thread mention policy. The best path is to decide between this PR's `threadFollowUp: off|topic|active` design and #40530's explicit `requireMentionInThread` design, then land one rebased implementation with docs/changelog updates, regression tests for `requireMention: true` thread follow-ups, and a fixed broadcast path.
