## Summary

Keep this PR open. Current main still does not implement a Feishu thread-follow-up mention policy, and unmentioned group thread replies still hit the existing requireMention gate before dispatch. The PR is also not cleanly superseded: open PR #40530 covers the same user problem with a different config API, so maintainers need to choose the canonical Feishu thread mention design rather than closing this as obsolete.

## Best Possible Solution

Keep this PR open until a maintainer chooses the Feishu thread mention policy. The best path is to pick one canonical API between this PR's threadFollowUp: off|topic|active active-topic design and #40530's requireMentionInThread override, fix the current PR's skipMentionForThread broadcast scoping issue if this design is chosen, then land with docs/changelog updates and regression coverage for requireMention: true topic-thread follow-ups, including broadcast-mode behavior.
