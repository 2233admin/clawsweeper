## Summary

Keep this issue open. Current main has the Telegram forum `threadId` announce-delivery fix, but the other reported behaviors still remain: `ANNOUNCE_SKIP` suppresses the final visible send instead of falling back to the target reply, and the A2A ping-pong path can still write the target reply back into the requester session as a fresh `inter_session` turn. The related full-fix PRs #43869 and #44064 are closed and unmerged, so they do not supersede the remaining work.

## Best Possible Solution

Keep this issue open. The maintainer fix should preserve the current `threadId` announce routing, add a visible-delivery fallback for Telegram topic handoffs when the announce step returns `ANNOUNCE_SKIP` after a real target reply, and suppress or redesign same-agent Telegram topic-to-topic ping-pong before it records target replies as new `inter_session` requester turns. Closed PRs #43869 and #44064 are useful prior-art references, but they are not merged and do not supersede the remaining work.
