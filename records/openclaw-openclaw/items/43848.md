## Summary

Keep open. Current main has fixed the Telegram forum-topic `threadId` loss and added an inter-session `isUser=false` mitigation, but the remaining visible-delivery and requester transcript pollution parts of the report still match current A2A behavior.

## What This Changes

The issue requests that `sessions_send` preserve Telegram forum topic routing, keep visible target-topic handoffs when announce returns `ANNOUNCE_SKIP`, and avoid writing topic-to-topic handoff turns back into the requester transcript.

## Best Possible Solution

Keep the shipped `threadId` preservation and inter-session marking, then land one coordinated A2A fix that defines the skip-token boundary, reuses a real target reply for visible Telegram topic delivery when appropriate, and prevents same-agent topic-to-topic handoffs from contaminating requester context.
