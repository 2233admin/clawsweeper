## Summary

Keep this PR open. Current main still does not implement the PR's hook announcement policy: gateway hook dispatch still falls back on `!result.delivered`, the cron result type has no `announceToMain` override, and current gateway tests still assert fallback for a `deliver:false` hook result. The PR remains a live implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to review the announcement policy against the existing cron `deliveryAttempted` contract, confirm target-agent routing still lands surfaced events in the resolved destination, and merge the PR if the targeted gateway/cron lanes and changed gate stay green.
