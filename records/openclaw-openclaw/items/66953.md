## Summary

Keep open. Current main still has the reported failure class: outbound delivery-mirror writes pair a route/session key with the current or effective request agent id, and the Feishu outbound route builds its session key from whatever agent id is supplied. The report is recent, concrete, and corroborated, and the related duplicate-delivery issues do not cover wrong-agent transcript ownership.

## What This Changes

This issue requests that delivery-mirror transcript entries for multi-account Feishu outbound sends be written under the writer agent session instead of the main/default agent session.

## Best Possible Solution

Define a single ownership contract for outbound delivery mirrors, then add a focused regression for Feishu or the generic outbound route with session.dmScope="per-account-channel-peer", separate writer/main account bindings, and a delivery-mirror append where request/default agent and writer route can differ. The implementation should make delivery, route construction, session entry creation, and transcript append agree on the writer-owned agent/session rather than creating a main-agent ghost session.
