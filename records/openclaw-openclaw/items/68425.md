## Summary

Keep open. Current main still redacts agent-visible `sessions_history` text but only exposes the existing `contentRedacted` boolean, with no stable machine-readable marker, `notice`, or redacted-field metadata. The linked PR #68483 is still an open, unmerged implementation candidate with closing syntax, so this issue should remain open until that PR is reviewed and landed or explicitly declined.

## What This Changes

The issue requests a stable machine-readable marker or structured notice on redacted tool output so agents can detect that placeholders are not real secret values.

## Best Possible Solution

Land PR #68483 or an equivalent narrow patch that adds a stable marker/notice to agent-visible redacted tool output while preserving existing redaction patterns and placeholder formats, with focused tests for the shared redaction helper and `sessions_history` response shape.
