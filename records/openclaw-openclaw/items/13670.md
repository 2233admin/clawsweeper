## Summary

Current main still does not implement the requested always-visible Control UI topbar display for the active model plus token/context usage. The shell topbar renders the dashboard breadcrumb, command search, chat mobile controls, and theme mode toggle; model and usage data exist in adjacent Chat, Usage, Debug, and slash-command surfaces but are not wired into the persistent topbar.

## Best Possible Solution

Keep this issue open until maintainers either add a focused Control UI topbar status component or explicitly decline that placement. A good implementation should derive the active model from the current session/default session state, derive fresh token/context usage from session state (`totalTokens`, `contextTokens`, `totalTokensFresh`) rather than Debug polling, handle stale/unavailable states, and include responsive UI coverage.
