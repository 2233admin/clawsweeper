## Summary

Keep this issue open. Current main still does not implement an always-visible Control UI topbar display for the active model plus token/context usage. The relevant data exists in Debug, Usage, Overview, chat, TUI, and CLI surfaces, but the Control UI topbar remains navigation/search plus chat mobile controls and theme mode only.

## Best Possible Solution

Keep this issue open until maintainers either add a focused Control UI topbar status component or explicitly decline that placement. A good implementation would live in the topbar render path, derive the active model from the current session/default model state, derive fresh token/context usage from session usage data rather than debug-only polling, handle stale/unavailable states, and include responsive UI coverage.
