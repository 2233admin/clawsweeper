## Summary

Keep this PR open. Current main still has a single preauth WebSocket handshake timer that is cleared only after async auth has completed, so main does not implement the PR’s intended valid-connect/auth-phase split. The PR remains useful as the repaired successor to #49751, but the branch has an unresolved security finding: repeated valid preauth `connect` frames can reset the new connect-auth watchdog while auth is pending.

## Best Possible Solution

Keep this PR open and repair the branch in place: make connect-auth watchdog arming one-shot or set an explicit `connecting` state before the first auth await, reject or close duplicate preauth `connect` frames deterministically, clear both timers on close and successful registration, and add regression coverage for stalled auth plus repeated valid connect frames. Preserve the #49751 contributor credit and update docs/changelog to match the final behavior.
