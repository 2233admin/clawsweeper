## Summary

Keep open. Current main still has a credible package-managed gateway auto-update failure path: the gateway schedules the updater, the auto-update child inherits the gateway service environment, and `openclaw update` refuses package updates from inside that service process. Open PR #74870 also references this issue with closing syntax, so the issue should remain open until that PR is reviewed, landed, or replaced.

## What This Changes

The issue requests that Homebrew-installed gateways with `update.auto.enabled` automatically install a detected stable update after the configured delay and restart.

## Best Possible Solution

Resolve the Homebrew auto-update contract explicitly: either implement a safe Homebrew-aware apply flow outside the live gateway process, or detect Homebrew-managed installs and skip with a clear `brew upgrade openclaw` hint plus aligned docs and focused tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a package-managed gateway process with auto-update due: the auto-update child inherits gateway service markers and then the package update guard exits before invoking the package manager.

## Solution Assessment

Unclear. The linked PR is a narrow plausible fix if maintainers choose skip-and-hint for Homebrew, but it does not implement the originally expected auto-install behavior.
