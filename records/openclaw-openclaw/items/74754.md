## Summary

Keep open. Current main still has a credible Homebrew/package auto-update failure path, and open PR #74870 already references this issue with closing syntax, so the safe outcome is to review, rebase, land, or close that PR rather than close the issue early.

## What This Changes

The issue requests that Homebrew-installed gateways with `update.auto.enabled` auto-install stable updates after `stableDelayHours` instead of only detecting them.

## Best Possible Solution

Resolve the Homebrew auto-update contract explicitly: either implement a safe Homebrew-aware apply flow outside the live gateway process, or intentionally skip Homebrew-managed installs with a clear `brew upgrade openclaw` hint plus aligned docs, changelog, and tests. If the skip-and-hint contract is accepted, rebase and review PR #74870 or replace it with an equivalent narrow patch.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a package-managed gateway service with auto-update due: the gateway schedules the updater, the child update process inherits service markers, and `openclaw update` refuses package updates from inside that service process. The reporter's exact two-hour timing remains partly ambiguous because default stable jitter can add up to 12 hours unless configured to zero.

## Solution Assessment

Unclear. PR #74870 is a plausible narrow solution if maintainers choose skip-and-hint for Homebrew installs, but it does not implement the originally expected automatic Homebrew upgrade behavior.
