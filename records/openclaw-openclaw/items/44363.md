## Summary

Keep this PR open for maintainer judgment. Current main has substantial cron reliability foundations, including persisted run status, run logs, failure alerts, skipped/error counters, delivery diagnostics, and startup handling for interrupted running jobs, but it still does not contain the PR's proposed doctor-facing cron reliability metrics/recommendations. The PR itself is not ready to land as-is: it is XL, mixes unrelated memory-search onboarding/config validation and agent watchdog/lifecycle work into a cron doctor change, and the current PR head still has correctness issues in the proposed diagnostics code.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not land it as-is. The best path is to split out the unrelated memory-search and agent-watchdog/lifecycle changes, rebase a focused cron diagnostics change onto current cron state/run-log APIs, and decide whether the remaining doctor-facing reliability summary belongs in openclaw doctor or should instead be handled through the existing cron list/runs/status surfaces and the open #13593 product tracker.
