## Summary

Keep open. Current main still does not expose WhatsApp message delete/revoke: WhatsApp discovery advertises only react/poll, dispatch only supports react, the runtime rejects non-react actions, and the listener API has no delete/unsend/revoke helper. Open PR #70813 is already linked as the implementation candidate and is unmerged in the provided context, so this issue should stay open until that PR or an equivalent maintainer-reviewed change lands.

## What This Changes

Request adding a WhatsApp message delete/revoke action so agents can retract their own sent WhatsApp messages through the message tool.

## Best Possible Solution

Land #70813 or an equivalent focused WhatsApp plugin change that advertises the agreed public action, routes through the gateway listener/runtime, builds Baileys own-message delete keys, enforces authorization and fromMe constraints, handles group participants, and updates focused tests, docs, and changelog.
