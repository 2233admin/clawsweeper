## Summary

Keep open. Current main still does not expose WhatsApp message delete/revoke, and the issue is already paired with open PR #70813, so the right cleanup action is to leave it open until that PR or an equivalent maintainer-reviewed change lands.

## What This Changes

The issue requests a WhatsApp message `delete`/revoke action so agents can retract their own previously sent WhatsApp messages.

## Best Possible Solution

Keep this as the canonical bundled WhatsApp plugin request until #70813 or an equivalent focused PR lands. The end state should be a WhatsApp-owned message action implementation that advertises the agreed delete/unsend action, routes through the gateway listener/runtime, builds the Baileys own-message delete key, enforces authorization and fromMe constraints, handles groups correctly, and updates focused tests, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: WhatsApp delete is not advertised, `supportsAction` rejects non-react actions, and the runtime throws `Unsupported WhatsApp action` if a non-react action reaches it.

## Solution Assessment

Yes, the narrow WhatsApp plugin implementation path is the best direction rather than a core change. The open paired PR appears aligned with that direction, but it remains unmerged and needs maintainer review for naming and mutation-safety details.
