## Summary

Keep open. Current main still does not expose WhatsApp delete/revoke: discovery advertises only WhatsApp react/poll, the channel action adapter routes only react, the runtime throws for non-react actions, and user-facing docs list delete for other channels but not WhatsApp. Related #70813 remains open and unmerged, so this issue should remain the canonical bundled WhatsApp plugin request.

## What This Changes

Add a WhatsApp message delete/revoke action to the message tool so agents can retract their own sent WhatsApp messages through the Baileys-backed channel.

## Best Possible Solution

Keep this issue open as the canonical bundled WhatsApp plugin request until #70813 or an equivalent focused PR lands. The end state should be a WhatsApp-owned action implementation that advertises the agreed public action, routes through the gateway listener/runtime, builds the Baileys own-message delete key, enforces target authorization and fromMe constraints, handles group participants correctly, and updates focused tests, docs, and changelog.
