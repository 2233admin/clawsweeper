## Summary

Current main at b85edb3f0cf6c2a8bec06537af9211a044266a6c still does not expose WhatsApp message delete/revoke. The shared message action name exists, and Baileys documents a delete payload, but the bundled WhatsApp plugin only advertises/accepts/routes reaction and poll-related behavior; delete is rejected before reaching any WhatsApp transport. Related #38723 was closed as a duplicate, so this issue remains the canonical request.

## Best Possible Solution

Keep this open as the canonical bundled WhatsApp plugin feature request. The best fix is a focused WhatsApp-plugin change: advertise `delete`, accept it in `supportsAction`, route it through the gateway-owned WhatsApp action runtime, add a listener/send helper that calls Baileys delete with the verified message key shape, enforce bot-owned/fromMe deletion with group participant handling, and update focused tests, message/WhatsApp docs, and the user-facing changelog.
