## Summary

Keep this PR open. Current main still has only inbound WhatsApp view-once unwrapping; outbound `viewOnce` is not exposed through the CLI/tool/gateway schemas, not serialized through outbound payloads or queue recovery, and not consumed by the WhatsApp or Signal senders. The PR is therefore not obsolete, but it needs maintainer review because the provided review thread still shows unresolved privacy/behavior concerns around validation, double-fetching media during preflight, and WhatsApp auto-reply text leakage.

## Best Possible Solution

Keep the PR open and ask for a focused update/review: rebase on current main, keep the protocol change additive, thread `viewOnce` through CLI/tool/gateway/outbound/queue surfaces, implement WhatsApp and Signal sender support with fail-closed validation before any send, avoid consuming media URLs during validation, cover monitor auto-reply paths, and update docs/changelog only once the behavior is accepted.
