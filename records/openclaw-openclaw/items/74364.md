## Summary

Keep open. Current main still has outbound channel paths that can deliver reflected `<relevant-memories>` blocks, and the open closing PR #74436 is unmerged, so this issue should remain open until maintainers review and land a complete fix.

## What This Changes

The issue requests stripping memory-plugin `<relevant-memories>...</relevant-memories>` scaffolding from assistant text before it is delivered through messaging channels.

## Best Possible Solution

Apply the existing code-fence-aware assistant-visible sanitizer at every assistant-origin outbound boundary, including final normalized replies, block chunks/finals, and partial preview updates, then add focused regressions for code-fence preservation and Telegram default streaming behavior.

Yes.
