## Summary

Keep open. Current main already has a shared assistant-visible sanitizer for memory scaffolding, but the outbound block-reply streaming path still does not apply it consistently, and the linked closing PR is open and unmerged.

## What This Changes

The issue requests stripping `<relevant-memories>` blocks from assistant text before outbound delivery through messaging channels such as Telegram and Discord.

## Best Possible Solution

Review and adjust #74436 so the shared code-fence-aware assistant-visible sanitizer is applied at every outbound assistant delivery boundary, including `sanitizeUserFacingText`, block replies, final flushes, and partial/preview updates, with focused regression tests before landing.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed assistant text containing a `<relevant-memories>...</relevant-memories>` block through the non-phase `text_delta`/`text_end` block-reply path and observe that current main only strips think/final/tool scaffolding before delivery.

## Solution Assessment

Yes, the narrow maintainable direction is to reuse the existing code-fence-aware sanitizer at outbound delivery boundaries. The current main implementation is incomplete, and the open PR should be reviewed or expanded rather than closing this issue now.
