## Summary

Keep open. Current main has a code-fence-aware sanitizer for assistant-visible `<relevant-memories>` scaffolding, but the outbound user-facing and streaming block-reply paths still do not consistently apply it, and the linked closing PR #74436 is open/unmerged in the provided GitHub context.

## What This Changes

The issue requests stripping reflected `<relevant-memories>` blocks from assistant text before delivery through messaging channels such as Telegram, Discord, Slack, and iMessage.

## Best Possible Solution

Review and adjust #74436 so the existing code-fence-aware assistant-visible sanitizer is applied at all assistant-origin outbound delivery boundaries, with focused tests for user-facing sanitizer output, text_end/message_end block replies, and code-fence preservation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed assistant text containing `<relevant-memories>...</relevant-memories>` through the non-phase `text_end` block-reply path and observe that current cleanup strips think/final/tool scaffolding but not memory scaffolding.

## Solution Assessment

Yes. Reusing the shared code-fence-aware sanitizer at outbound delivery boundaries is the narrow maintainable solution; current main does not yet apply it consistently, so this issue should stay open until the linked PR lands or is replaced.
