## Summary

Keep this PR open for maintainer review. Current main still has the reported memory behavior, and this PR is a focused implementation candidate for the linked dreaming promotion bug, but the widened live-note relocation window is privacy-sensitive and can promote more adjacent content when matching by containment. No supply-chain, workflow, dependency, lockfile, or secret-handling changes were introduced.

## What This Changes

This PR changes memory-core light dreaming ordering, promotion rank/apply gates, live-note relocation span, docs, and regression tests so daily-only signals can affect scoring but no longer satisfy `minRecallCount`.

## Best Possible Solution

Keep the recall-first ordering and recall-or-grounded gate direction, but tighten relocation so exact matches win and containment matches choose the smallest or tightly bounded window before promotion. Then keep the docs/tests aligned with the final contract and run the targeted memory-core tests plus the changed-lane gate in the maintainer validation path.
