## What This Changes

The PR updates memory-core light dreaming ordering, short-term promotion rank/apply gates, live daily-note rehydration, regression tests, and memory/dreaming docs so daily-only signals can score candidates without satisfying `minRecallCount`.

## Best Possible Solution

Review and land this PR if maintainers agree that `minRecallCount` should mean recall-or-grounded evidence rather than all short-term signals, preserving the regression tests and docs so the linked memory-dreaming issue closes through the existing closing reference after merge.

## Reproduction Assessment

Yes. Current main can be checked statically: light dreaming sorts by recency before `recallCount`, and promotion/apply compare aggregate signals including `dailyCount` to `minRecallCount`; the PR adds focused regression tests for both behaviors.

## Solution Assessment

Yes, pending maintainer agreement on the contract. The PR keeps daily signals in scoring but removes them from the recall gate in both rank and apply paths, which is the narrowest maintainable fix for the reported daily-only saturation without discarding daily-ingestion evidence.
