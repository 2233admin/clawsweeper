## Summary

Keep open. This PR is the active focused fix for a real memory-core dreaming/promotion bug, and current main still has the reported daily-signal gate and light-sort behavior. The latest head addresses the earlier nearest-repeat relocation concern, but the widened quality-2 containment scan can still bind to a broad original-start window and append unrelated daily-note lines when the moved snippet is embedded farther away, so it needs another author/maintainer pass before merge.

## What This Changes

The branch updates memory-core dreaming sorting, short-term promotion ranking/apply gates, live daily-note relocation, docs, and regression tests so daily-only signals can affect scoring without satisfying `minRecallCount`.

## Best Possible Solution

Keep the recall-first light ordering and recall-or-grounded promotion gate direction, but revise relocation so broad containment windows cannot append unrelated daily-note lines when a tighter same-occurrence match exists. Add a regression for an embedded snippet shifted beyond one line from the stored range, keep the docs aligned with the final contract, and run the targeted memory-core tests plus the changed-lane gate before merge.
