## Summary

Current main still does not provide the requested live Browser Preview Panel in the Control UI. The browser plugin has useful backend primitives, and there is now a plugin UI descriptor registry/RPC, but the Control UI still has no browser tab, route, render branch, or live preview consumer; the related #63926 remains open and narrower around visible browser management/login handoff.

## What This Changes

Add a live Control UI/Dashboard browser preview panel showing the managed browser in near real time, including URL/title, console errors, and optional manual interaction for remote/VPS users.

## Best Possible Solution

Keep this issue open as the broad remote browser-preview/dashboard tracker. The best path is maintainer-reviewed design for an admin-only Control UI browser surface that reuses browser.request/browser-control primitives where safe, coordinates with #63926 for visible browser management/login handoff, and decides whether the browser plugin needs a concrete Control UI contribution surface beyond today’s descriptor metadata.
