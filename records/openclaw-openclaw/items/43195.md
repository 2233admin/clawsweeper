## Summary

Keep this PR open. Current main already shares the unhandled-rejection handler registry through `globalThis`, but `installUnhandledRejectionHandler()` still has no installed guard and still attaches a new `process.on("unhandledRejection", ...)` listener on every call. The PR's core behavior remains relevant, though it should be rebased onto the newer main structure and backed by a focused regression test.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent maintainer patch, by adding a shared installed guard to `installUnhandledRejectionHandler()` in `src/infra/unhandled-rejections.ts`. Preserve the current global handler Sets and add focused regression coverage that repeated installer calls do not increase `process.listenerCount("unhandledRejection")`.
