## Summary

Keep this PR open. Current main already shares the unhandled-rejection handler registry through globalThis, but installUnhandledRejectionHandler() is still not idempotent and repeated calls can attach repeated process unhandledRejection listeners. The PR’s central installer-state fix remains useful, but it needs a rebase onto the current handler structure and focused regression coverage.

## Best Possible Solution

Keep the PR open, or replace it with an equivalent maintainer patch, by adding shared installer state to installUnhandledRejectionHandler() in src/infra/unhandled-rejections.ts. Preserve the current global handler Sets, make repeated installer calls a no-op, and add regression coverage for listener-count stability.
