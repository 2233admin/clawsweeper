## Summary

Keep open. Current main still matches the reported Telegram Windows regression surface: polling startup remains gated on successful deleteWebhook cleanup, recoverable cleanup failures still retry before polling, native command sync still calls deleteMyCommands then setMyCommands, and the ALL_PROXY/OPENCLAW_PROXY_URL transport fix proposed in #74151 is not present on main. Adjacent mitigations for Windows Jiti loading and apiRoot diagnosis exist, but they do not prove the 2026.4.24+ Windows/npm regression is fixed.

## Best Possible Solution

Keep this issue open as the broader Windows Telegram regression tracker. The best path is maintainer review of #47384 and #74151, or equivalent Telegram-owned changes, followed by a Windows global-npm package smoke that covers deleteWebhook, polling startup, deleteMyCommands/setMyCommands, proxy env handling, and bot-native-commands.runtime loading. If diagnostics show a full /bot<TOKEN> apiRoot, route that through openclaw doctor --fix, but do not treat it as the only root cause without reproducing the release regression.
