## Summary

Keep open. Current main has bounded progressive disclosure for a flat skills catalog, including compact/truncated prompt output, but it still lacks the requested skill dependency graph, depth-based expansion, DOT graph output, and `skills.maxDepth` config. Related token-overhead issues #52861 and #73248 overlap, but they do not fully cover this specific graph contract.

## What This Changes

The issue requests dependency-aware skill loading, depth-limited expansion, DOT graph output, and a `skills.maxDepth` config option to reduce prompt token use.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract first, then implement graph construction and prompt expansion that respect existing eligibility, allowlists, prompt budgets, exposure rules, and docs, with DOT output and config/schema coverage.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. The high-confidence check is current-main source and docs inspection, which shows the graph, DOT CLI, dependency metadata, and depth config surfaces are absent.

## Solution Assessment
