## Summary

Keep open. Current main has the adjacent token-saving behavior from #6691: a bounded `<available_skills>` metadata catalog and one selected `SKILL.md` read on demand. It does not implement this request's skill dependency graph, dependency parsing, depth-limited expansion, `openclaw skills graph --dot`, or a skill dependency-depth config surface.

## Best Possible Solution

Keep open for maintainer product/API design. The likely direction is an explicit skill dependency contract, a graph builder that respects current eligibility and allowlist rules, optional `openclaw skills graph --dot` output, and carefully named runtime depth controls that do not conflict with `agents.defaults.subagents.maxSpawnDepth`.
