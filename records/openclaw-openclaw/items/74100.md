## Summary

Keep open. Current main has a bounded metadata-only skills catalog and on-demand SKILL.md reads, but it does not implement dependency declarations, dependency-depth expansion, DOT graph output, or a skills.maxDepth-style config surface; related #52861 and #73248 overlap on token pressure but do not clearly supersede this specific skill-graph request.

## What This Changes

Request a skill dependency graph that loads skill dependency chains on demand, exposes graph output, and adds configurable expansion depth to reduce prompt token cost.

## Best Possible Solution

Keep the issue open for maintainer design around a backward-compatible skill dependency contract, graph builder that respects existing eligibility and allowlists, bounded runtime expansion semantics, DOT inspection output, config/schema/docs updates, and focused regression coverage.
