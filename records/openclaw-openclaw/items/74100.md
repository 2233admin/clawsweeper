## Summary

Keep open. Current main has bounded, progressive disclosure for a flat skills catalog, but it does not implement the requested skill dependency graph, depth-based expansion, DOT graph output, or `skills.maxDepth` configuration; related token-overhead issues overlap but do not replace this specific graph/API request.

## What This Changes

The issue requests dependency-aware on-demand skill loading, dependency graph visualization, and a configurable skill expansion depth to reduce skill prompt token cost.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract first, then implement graph construction that respects existing eligibility, allowlists, prompt budgets, and exposure rules, with DOT output, config/schema/docs, and focused tests for prompt resolution and CLI formatting. Coordinate the design with #52861 and #73248 so token-overhead work has one coherent direction.
