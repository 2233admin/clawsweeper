## Summary

Keep open. Current main has the related token-saving behavior from #6691, namely a bounded `<available_skills>` metadata catalog and selected `SKILL.md` reads on demand, but it still does not implement skill dependency declarations, dependency-depth expansion, graph/DOT output, or a `skills.maxDepth`-style config surface.

## What This Changes

Request a Skill Graph feature that parses skill-to-skill dependencies, expands only the needed dependency depth at runtime, exposes `openclaw skills graph --dot`, and adds configurable skill graph depth to reduce prompt token cost.

## Best Possible Solution

Keep this open for maintainer design. The durable path is a backward-compatible skill dependency contract, a graph builder that respects current skill eligibility and allowlists, optional DOT output for inspection, and runtime expansion semantics that integrate with the bounded `<available_skills>` catalog without conflating skill dependency depth with subagent spawn depth.
