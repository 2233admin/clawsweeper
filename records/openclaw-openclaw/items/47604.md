## Summary

Keep this PR open. Current main still has no Wear OS companion implementation, and the timeline shows maintainer routing signal through assignment plus the `no-stale` label. The proposed patch is not merge-ready as-is because the new Wear app enables backup for files that include the device-auth private key, so this needs maintainer product/security review rather than cleanup closure.

## What This Changes

The PR adds a Wear OS companion app, shared Android gateway helpers, phone-side Wear proxy, direct watch gateway access, tile and complication services, background reply notifications, tests, and Android/Wear docs.

## Best Possible Solution

Keep this PR open while maintainers decide whether Wear OS support belongs in the Android app. If it does, land it through staged review or a split replacement, with the Wear backup/device-identity issue fixed before merge and fresh Android/Wear security and device proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Disable Wear backup for device identity files:** `apps/android/wear/src/main/AndroidManifest.xml:10-12`
  - body: The new Wear app enables backup and its backup rules include the whole `file` domain, but `GatewayDeviceIdentityStore` writes `privateKeyPkcs8Base64` under `files/openclaw/identity/device.json`. That would allow cloud backup or device transfer to carry the device-auth signing key, unlike the current phone app which disables/excludes backup. Disable Wear backup or explicitly exclude identity/auth files before shipping this module.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The diff enables Wear app backup while storing a device-auth private key under files included by the backup rules.

Concerns:

- **[high] Wear backup can export device-auth private key:** `apps/android/wear/src/main/AndroidManifest.xml:10`
  - body: `android:allowBackup="true"` plus `backup_rules.xml` and `data_extraction_rules.xml` including `domain="file" path="."` covers `files/openclaw/identity/device.json`, where the shared identity store writes `privateKeyPkcs8Base64`. This weakens the device-auth trust model and differs from current main's phone app backup exclusion.
