## Summary

Keep this PR open. Current main still does not ship a Wear OS companion, so the feature is not obsolete, but the branch is not merge-ready because the proposed Wear app can back up the device-auth private key and the overall Android/Wear scope still needs maintainer product and security review.

## What This Changes

The PR adds Android `:shared` and `:wear` modules, a Wear OS companion app, a phone-side Wear Data Layer proxy, direct watch gateway access, chat/session/settings UI, tile/complication/notification support, tests, and Android/Wear documentation.

## Best Possible Solution

Keep the PR open while maintainers decide whether Wear OS belongs in-tree. If yes, first fix the Wear backup/private-key exposure, then review or split the branch into staged pieces around shared gateway helpers, phone proxying, direct watch auth, Wear UI, notifications, tile/complication services, docs, tests, and fresh device proof.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Disable backup for Wear identity files:** `apps/android/wear/src/main/AndroidManifest.xml:10-12`
  - body: The new Wear app enables Android backup while its backup/data-extraction rules include the whole file domain.
