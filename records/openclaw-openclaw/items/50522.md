Keep this PR open. Current main still sends Discord `ThreadStarterBody` unconditionally, and the reply runner can still prepend a supplied starter block on non-new sessions when thread history is absent. The linked bug #41355 is open, and this PR remains the active narrow implementation candidate with maintainer follow-up on its branch.
