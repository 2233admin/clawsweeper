## Summary

Keep this issue open. Current main and the latest tagged release still use plain apt-get in the active sandbox-common build path, and the related implementation PR is closed unmerged. The requested retry hardening is still valid, but the exact --fix-missing behavior should get maintainer review because it affects Docker sandbox image reproducibility.

## Best Possible Solution

Keep this issue open and land a focused, maintainer-reviewed Dockerfile hardening change. Add bounded apt retry handling to Dockerfile.sandbox-common and consider Dockerfile.sandbox for the cold-start path; decide explicitly whether --fix-missing belongs in these reproducible sandbox image builds, then add a narrow source-level test in src/docker-build-cache.test.ts or an adjacent Dockerfile test to lock the retry contract.
