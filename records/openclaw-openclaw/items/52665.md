## Summary

Keep this issue open. Current main still uses plain `apt-get update` and `apt-get install` in the active sandbox-common image build path, with no `Acquire::Retries` or `--fix-missing`; the related implementation PR #52724 is closed and unmerged.

## Best Possible Solution

Keep the issue open and land a focused Dockerfile hardening change. Add bounded apt retry handling to `Dockerfile.sandbox-common` and consider the cold-start `Dockerfile.sandbox` path; decide deliberately whether `--fix-missing` belongs in these reproducible image builds, then add a narrow source-level test in `src/docker-build-cache.test.ts` or an adjacent Dockerfile test to lock the retry contract.
