## Summary

Keep this issue open. Current main still uses configured agent workspaces as a replacement for the runtime primary workspace: when `agents.list` contains sub-agent workspaces, `ctx.workspaceDir` is only appended if the configured list is empty, so the reported primary workspace skip remains possible. The existing ClawSweeper discussion correctly identified this gap; current main SHA edbab0e2db7f450f4d63f9700289ae1ba838a419 has not changed that behavior.

## Best Possible Solution

Fix memory-core dreaming workspace enumeration so scheduled dreaming processes the runtime primary workspace plus configured agent workspaces, deduped by normalized path. The likely implementation belongs in the runtime dreaming path and shared dreaming phase workspace resolver, with matching `doctor.memory.status` aggregation and regression tests for a non-empty `agents.list` of sub-agents while the primary workspace is only available from runtime/status context.
