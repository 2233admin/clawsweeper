## Summary

Keep this open. Current main still builds the dreaming workspace list from configured agent workspaces and only falls back to the runtime primary workspace when that configured list is empty, so a setup with AGI sub-agent workspaces can still skip the primary workspace described in the report.

## What This Changes

Ensure memory-core scheduled dreaming processes the primary agent workspace even when configured AGI sub-agent workspaces are also present.

## Best Possible Solution

The runtime dreaming path should combine the runtime primary workspace with configured agent workspaces and dedupe by normalized path, while preserving configured agent fan-out and aligning doctor/status reporting so the scheduled sweep and user-visible status agree.
