## Summary

Keep open. Current main still routes async gateway and node exec approval completion output through the compact notification formatter, which keeps only the 400-character tail and collapses whitespace before the follow-up agent turn. The linked fix PR #41170 is closed and unmerged, so there is no merged implementation to close this issue against.

## Best Possible Solution

Reopen/rebase #41170 or reimplement the same narrow fix on current main: keep ordinary background notifyOnExit compact summaries unchanged, but make async approval completion for gateway and node deliver captured exec output with newlines preserved up to the existing capture cap, and explicitly mark capture-cap truncation. Add regression coverage for multiline output over 400 characters and a cap-hit case through the approval follow-up path.
