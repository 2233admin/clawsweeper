## Summary

Keep open. Current main still sends async gateway and node exec approval completion output through the compact notification formatter, which keeps a 400-character tail and collapses whitespace before the follow-up agent turn. The linked fix PR is closed and unmerged, so there is no merged implementation to close against.

## Best Possible Solution

Keep this open and rebase/reopen PR #41170 or reimplement the same narrow fix on current main: keep ordinary background notifyOnExit compact summaries unchanged, but make async approval completion for gateway and node deliver captured exec output with newlines preserved up to the existing capture cap, and explicitly mark capture-cap truncation. Add regression coverage for multiline output over 400 characters and a cap-hit case through the approval follow-up path.

## Clownfish Work Prompt

Fix the async exec approval completion output truncation bug on current main.

Observable bug: when an approved async exec completes through gateway or node approval follow-up delivery, the agent receives only a compact 400-character tail with newlines collapsed because the completion path reuses notifyOnExit formatting. This should not affect ordinary background notifyOnExit summaries.

Expected fix boundary:
- For gateway async approval completion, deliver the captured outcome output with newlines preserved up to the existing exec capture cap.
- For node async approval completion, do the same for stdout/stderr/error payloads.
- Add an explicit truncation marker/note when the existing capture cap was hit; do not silently drop the front of output.
- Keep normal background notifyOnExit behavior compact and unchanged.
- Do not raise exec capture limits or change approval/security policy.
- Do not add core channel-specific behavior for Discord; fix the generic approval follow-up path.
