## Summary

Keep open. Current `main` still lacks bundled Vapi support in the Voice Call plugin, and the recent maintainer discussion explicitly kept this as a product/ownership decision rather than a stale or already-implemented request. The existing Clawdia Vapi bridge is useful related context, but it does not provide `provider: "vapi"` in `@openclaw/voice-call`.

## What This Changes

Add a Vapi telephony provider to the bundled Voice Call plugin, including provider code, config schema/env support, runtime registration, tests, docs, and manifest metadata.

## Best Possible Solution

Maintainers should choose the ownership path first. If Vapi belongs in the bundled Voice Call plugin, implement it end-to-end across provider code, config/env/schema validation, runtime selection, manifest metadata/secret inputs, tests, docs, and changelog. If it should remain external, document the Clawdia bridge path or add a public telephony-provider seam before tracking Vapi as ClawHub/plugin work.
