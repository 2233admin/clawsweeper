## Summary

Keep open. Current main still does not implement Vapi as a bundled Voice Call provider, the recent maintainer discussion explicitly kept this issue open for product and ownership judgment, and the existing Clawdia Vapi bridge is related external context rather than `provider: "vapi"` support in `@openclaw/voice-call`.

## What This Changes

Add Vapi as a Voice Call plugin telephony provider, including provider code, config schema, runtime registration, tests, and docs.

## Best Possible Solution

Maintainers should decide the ownership path first. If Vapi belongs in the bundled Voice Call plugin, implement it end-to-end across provider code, config/env/schema validation, runtime selection, manifest metadata and secret inputs, tests, docs, and changelog. If it should remain external, document the Clawdia bridge path or add a public telephony-provider seam before tracking Vapi as ClawHub/plugin work.
