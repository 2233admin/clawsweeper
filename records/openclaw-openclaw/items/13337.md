## Summary

Keep open. Current main and the latest release do not implement Vapi as a bundled `@openclaw/voice-call` telephony provider: provider ids, config/env validation, runtime provider selection, provider exports/files, manifest schema, tests, and public docs remain limited to Telnyx, Twilio, Plivo, and Mock. The Clawdia Vapi bridge is useful adjacent work, but it is documented as an external HTTP bridge, and current plugin docs expose speech/realtime provider seams rather than a public Voice Call telephony-provider registration seam.

## Best Possible Solution

Keep this issue open for maintainer/product judgment. If Vapi should be first-party in the bundled Voice Call plugin, implement a `VapiProvider`, Vapi config/env/schema validation, runtime provider selection, manifest metadata, provider/runtime tests, and docs at https://docs.openclaw.ai/plugins/voice-call. If maintainers prefer external ownership, document the Clawdia bridge path or add a public Voice Call telephony-provider registration seam, then track Vapi as a ClawHub/plugin item.
