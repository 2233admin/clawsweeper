## Summary

Keep open. Current main still does not implement `maxCostPerRun` or any equivalent per-run cost cap for heartbeat or embedded cron runs. The same-author tracking issue #49823 was closed as superseded by this PR, so this PR remains the canonical thread for the requested capability. It is not merge-ready as-is because the discussion and source review show unresolved correctness concerns around transcript-aware estimation, effective model resolution, and coverage beyond heartbeat dispatch.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best implementation should add the config type, strict schema, schema help/labels, docs, and focused tests, then enforce the cap at a shared embedded-run dispatch point using the same effective agent model and session context that the actual run will use. It should either estimate transcript/system/tool overhead conservatively or document the limitation plainly, and it should resolve the existing review comments before merge.
