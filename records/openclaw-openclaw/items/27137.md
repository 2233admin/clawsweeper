## Summary

Keep open. Current main still lacks the requested unified `authorizeIngress()` / `IngressContext` gate; the existing shared helpers centralize lower-level DM/group policy decisions, but channel and event authorization remains distributed. The maintainer comment already kept this open for design follow-up, the provided #49782 title-overlap item is unrelated ACP work, and the merged #58245 Discord voice fix is only a narrow per-channel gate, not a superseding unified solution.

## Best Possible Solution

Keep this open for maintainer security design. The best path is an additive core/security or SDK-backed `authorizeIngress`-style contract that accepts normalized adapter inputs for channel, account, event kind, sender identity, chat context, configured/store allowlists, command surface, and standardized reason codes, then migrates representative message, reaction, pin, interaction, callback, slash, and voice paths with contract tests proving new ingress surfaces cannot bypass the shared gate.
