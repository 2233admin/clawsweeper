## Summary

Keep open. Current main still lacks the requested unified cross-channel `authorizeIngress()` contract; it has lower-level DM/group policy helpers and per-channel/per-surface authorization paths. The existing maintainer discussion also explicitly keeps this as a security/design RFC rather than a cleanup-close candidate.

## What This Changes

Request to add a unified cross-channel ingress authorization gate that centralizes DM/group allowlist and policy evaluation while channel adapters provide identity, account, event-kind, and context inputs.

## Best Possible Solution

Keep the RFC open until maintainers approve an additive ingress authorization contract, then centralize policy evaluation behind that contract while preserving channel-specific identity/context adapters and migrate representative message, reaction, pin, interaction, callback, slash, voice, and media paths with contract tests.
