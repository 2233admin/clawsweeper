## Summary

Keep open. Current main still exposes whole-install `openclaw backup create` and `openclaw backup verify` only; there is no restore subcommand, no `--agent` backup scope, and no `--as` clone flow. The reporter’s tested shell-script workaround and the config-only community tool show real demand, while related #41111 and #59280 are adjacent generic restore work rather than a superseding per-agent backup/restore design.

## What This Changes

Add per-agent backup, restore, and clone support to the backup CLI with `--agent <id>` and optional restore-as-new-agent `--as <id>`.

## Best Possible Solution

Keep this as a valid feature request. The maintainable end state is a generic safe restore foundation first, then per-agent archive creation and restore with optional `--as`, scoped workspace plus `agents/<agentId>` payloads, config-entry rewrite for `id`/`name`/`workspace`/`agentDir`, collision handling, docs, and targeted safety tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main absence is high-confidence from source, docs, and CLI tests showing create/verify only with no restore or per-agent backup scope.

## Solution Assessment

Unclear as an immediate implementation path. The requested CLI direction is product-appropriate, but the best maintainable solution should build on the generic restore safety model and explicitly define credential/session clone policy before code lands.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested restore/clone feature is security-sensitive because it would copy credentials-adjacent per-agent state and write archive contents back to disk.

Concerns:

- **[medium] Define restore archive safety rules:**
  - body: A per-agent restore command would consume archives and write config, workspaces, agent state, auth-profile files, and sessions. Core implementation needs path containment, symlink handling, manifest validation, dry-run, conflict, rollback, and partial-failure rules before merge.
  - confidence: 0.9

  - body: The requested archive includes `agents/<agent>/agent/` and `agents/<agent>/sessions/`, so clone/restore may duplicate credentials and private transcript history. Maintainers should decide inclusion, redaction, permissions, and documentation before implementation.
  - confidence: 0.86
