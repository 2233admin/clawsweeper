## Summary

Keep open. Current main still does not expose a browser `act` kind for CDP/IME-native text insertion, and the existing text paths still route through Playwright `fill`/`type` or Chrome MCP `fill`. The report is recent, concrete, unprotected, and the prior review comment already found the same missing browser action surface.

## Best Possible Solution

Implement an additive browser plugin `act` kind such as `insertTextIme` or `cdpInsertText` in `extensions/browser`. Update the tool schema, `BrowserActRequest`, `ACT_KINDS`, normalization, CLI/docs, `/act` dispatch, and regression tests. For managed/raw CDP profiles, focus the target element/tab and send page-scoped CDP `Input.insertText`; for existing-session Chrome MCP profiles, either provide an equivalent low-level insertion route or return a clear `ACT_EXISTING_SESSION_UNSUPPORTED` response.
