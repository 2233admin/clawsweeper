## Summary

Keep open. Current main still does not implement the Android-side behavior: Android history parsing preserves text content verbatim, Android rendering displays those text parts directly, and the current gateway chat-history projection does not provide an equivalent merged filter for these two artifacts. The PR is narrow and the security review did not find supply-chain or secret-handling changes, but it needs maintainer review on whether the cleanup belongs in Android only or in shared chat-history projection, plus regression coverage.

## What This Changes

The PR changes Android ChatController history parsing to drop short cron completion marker-like user messages and unwrap simple top-level `<final>...</final>` wrappers for assistant/system text before rendering.

## Best Possible Solution

Keep the PR open and have maintainers choose the intended boundary: land a narrowly tested Android display cleanup if Android is uniquely affected, or move the sanitization into shared chat-history projection if all clients should stop seeing these artifacts, while preserving user-authored `<final>` examples and legitimate cron-related messages.
