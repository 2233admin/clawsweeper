## Summary

Current main still does not implement this PR's intended change. The native `sessions_spawn` path starts the child `agent` run without `inputProvenance`, while the paired tracker #47070 remains open and this draft PR is still the focused implementation candidate.

## Best Possible Solution

Keep this PR open as the implementation candidate for #47070. The likely path is to rebase or refresh the small patch on current main, add `inputProvenance` to the native `spawnSubagentDirect()` child `agent` call with `kind: "inter_session"`, `sourceSessionKey`, `sourceChannel`, and `sourceTool: "sessions_spawn"`, keep the focused regression test, and land it after maintainer review.
