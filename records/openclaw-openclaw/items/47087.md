## Summary

Keep open: current main still lacks native `sessions_spawn` provenance, and this draft PR targets the right missing path, but the proposed channel lineage is incomplete for channel-only requester contexts that current spawn tooling supports.

## What This Changes

The branch adds inter-session `inputProvenance` to native `sessions_spawn` child agent calls and adds a workspace regression test for the child call parameters.

## Best Possible Solution

A refreshed implementation should pass provenance through native spawn using the requester session key and direct requester channel source, preserve existing delivery behavior, and cover both routable and channel-only spawn contexts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve channel provenance without requiring a delivery target:** `src/agents/subagent-spawn.ts:628`
  - body: `sourceChannel` is read from `requesterOrigin?.channel`, but `requesterOrigin` is produced by `normalizeDeliveryContext()`, which returns `undefined` unless the context has a routable target. Current `sessions_spawn` callers can pass `agentChannel` without `agentTo`, so those child runs would still lose the requester channel lineage this PR is adding. Use the requester channel directly and add coverage for a channel-only spawn.
  - confidence: 0.87
