## Summary

Current main still does not implement this PR's intended change. Native `sessions_spawn` starts the child `agent` run without `inputProvenance`, while the paired tracker #47070 remains open and this PR is the focused implementation candidate.

## Best Possible Solution

Keep this PR open as the implementation candidate for #47070. The likely maintainer path is to refresh the small patch on current main, add `inputProvenance` to the native `spawnSubagentDirect()` child `agent` call with `kind: "inter_session"`, `sourceSessionKey`, `sourceChannel`, and `sourceTool: "sessions_spawn"`, keep the focused regression test, then review and land it so the paired issue closes normally.
