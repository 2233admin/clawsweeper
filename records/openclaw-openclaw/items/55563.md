## Summary

Keep open. Current main still has the browser CLI/docs mismatch for the reported `openclaw browser tabs --browser-profile remote` form, and the related compatibility PR #55567 was closed unmerged. The gateway side has later service-audit mitigation for version-managed Node paths, but that does not make this issue closeable while the browser regression remains user-facing.

## Best Possible Solution

Keep this issue open and resolve the two regressions deliberately: either restore `openclaw browser tabs --browser-profile <name>` compatibility with a focused regression test, or update the WSL2 guide and migration/changelog notes if the new option order is intentional. Separately, add source/test or documented proof that `doctor --fix` repairs or clearly diagnoses WSL2 systemd units pinned to nvm-managed Node paths.
