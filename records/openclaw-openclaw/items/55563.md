## Summary

Keep open. Current main has some later gateway-service repair work, but this two-regression report is not fully and cleanly implemented: the legacy `openclaw browser tabs --browser-profile remote` syntax still lacks an explicit current regression test or leaf-command option, and the gateway doctor path still has environment-specific behavior that needs a targeted WSL2/systemd+nvm reproduction before closing. The related PR #55567 was closed unmerged, so it is not a shipped canonical fix.

## Best Possible Solution

Keep this issue open and split or finish the remaining work deliberately: add a regression test and implementation for `openclaw browser tabs --browser-profile <name>` or update the affected docs/changelog if the new syntax is intentional; separately verify `doctor --fix` repairs or clearly diagnoses a systemd unit with a version-managed Node ExecStart, including the stopped/failed restart-loop state described by the reporter.
