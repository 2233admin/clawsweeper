## Summary

Keep open. Current main still lacks the install-time warning this PR proposes: the Linux npm-prefix repair path still rewrites npm's prefix to `~/.npm-global` with the old info/success messages and no follow-up warning about future global npm updates without sudo. The linked bug #44365 is already closed as superseded by this open PR, and recent maintainer context says ProjectClownfish pushed a narrow repair so this contributor path can remain canonical.

## Best Possible Solution

Keep this PR open for maintainer review and land it, or replace it with an equivalent maintainer-owned warning in `fix_npm_permissions()` that says the installer is redirecting global npm installs to `~/.npm-global` and future updates for that install should use `npm i -g` without sudo. A focused installer test for the non-writable-prefix branch would make the fix stronger.
