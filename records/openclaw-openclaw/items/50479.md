## Summary

Keep open. Current main has newer duplicate global-install detection and documented `openclaw update` guidance, but it still does not implement this PR’s intended install-time warning when `fix_npm_permissions()` rewrites npm prefix to `~/.npm-global`, nor the explicit no-sudo follow-up warning. The linked issue #44365 was already closed as superseded by this PR, so this PR remains the canonical target-repo implementation candidate rather than something safely closeable.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to land a small, tested installer-message change in `fix_npm_permissions()` or replace it with an equivalent maintainer-authored warning that clearly says the installer is saving `prefix=~/.npm-global` to the user npm config and that future global npm updates should not be run with sudo for that install.
