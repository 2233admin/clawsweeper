## Summary

Keep open. Current main still models hot channel reloads as whole-channel restart targets and the reload handler still restarts Telegram without an account id, while the open same-author PR with closing syntax remains unmerged. The issue is valid and still paired with its implementation candidate, so closing before that PR lands would be premature.

## What This Changes

The issue requests account-scoped gateway hot reloads for `channels.<channel>.accounts.<accountId>.*` changes so account-only edits do not restart the entire channel.

## Best Possible Solution

Represent account-scoped channel restart targets in the reload plan, detect strict `channels.<channel>.accounts.<accountId>.*` changes, let mixed or channel-global changes win as full-channel restarts, route account-only reloads through the existing account-aware lifecycle, and cover the planner and handler behavior with focused regression tests.
