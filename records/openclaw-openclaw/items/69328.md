## Summary

Keep this PR open. Current main still does not implement the ACP zero-diff worktree verifier or the PR's direct active-run session.message append path; the overlapping Control UI final-reload reconciliation only defers and replays history loads after terminal events. The PR also has unresolved correctness concerns around untracked files and unborn HEAD repositories before any zero-diff gate should affect task terminal state.

## What This Changes

The PR adds an ACP worktree diff verifier and terminal-status gate for oneshot ACP task runs, plus Control UI handling that appends session.message payloads during active chat runs with transcript dedupe tests.

## Best Possible Solution

Keep this PR open for maintainer review and rebase. The best path is to decide whether OpenClaw wants an ACP oneshot zero-diff verification gate, fix the verifier to account for untracked files and unborn HEAD before it can affect terminal state, and reconcile the direct session.message append idea with the current deferred final-reload Control UI design and dedupe contract.

## Risks / Open Questions

- The proposed zero-diff helper checks only tracked changes relative to HEAD, so untracked new files and unborn-HEAD repositories can be treated as no-change runs because the helper fail-closes to hasChanges=false.
- Changing ACP terminal success into a blocked/failed zero-diff outcome is a product and reliability policy decision for oneshot ACP runs, especially because persistent sessions are deliberately different.
- The proposed active-run direct append path mutates chatMessages alongside the current deferred final-reload design; Greptile's review notes a dedupe gap if identity exists only at the event level rather than inside the stored message object.
- Security review found no dependency, workflow, lockfile, package-publishing, or secret-handling changes, but the new terminal path would execute git from PATH in a task cwd and should be reviewed as part of the verifier design.
