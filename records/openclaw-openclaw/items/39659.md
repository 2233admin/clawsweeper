## Summary

Keep #39659 open. Current main documents and implements lower-level isolation primitives for profiles, manual gateway ports, Docker Compose env overrides, ClawDock single-project helpers, and `--container` execution targeting, but it still lacks the requested first-class Docker multi-instance lifecycle with named create/list/delete/update, managed port/path allocation, compose project/container isolation, and cleanup/update metadata. Related PR #54126 is closed unmerged and targeted profile management rather than Docker instance lifecycle.

## Best Possible Solution

Keep this issue open until maintainers implement or explicitly decline a supported Docker multi-instance lifecycle. A good implementation would add a documented `openclaw instance ...` or Docker/ClawDock equivalent that records per-instance metadata, validates or allocates unique host ports and paths, isolates compose project/container names, and provides safe create/list/update/delete cleanup with focused tests and Docker install docs.

## Risks / Open Questions

- The requested feature overlaps existing `--profile`, `--container`, Docker Compose, Docker setup, and ClawDock helper surfaces, so maintainers should choose one coherent product surface before implementation.
