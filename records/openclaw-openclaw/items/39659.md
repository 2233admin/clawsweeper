## Summary

Keep this issue open. Current main still has Docker single-project setup, ClawDock helper commands, host-side `--profile` path isolation, and `--container` exec targeting, but it does not provide the requested named Docker instance lifecycle with create/list/delete/update, managed per-instance metadata, port/path allocation, or Compose project isolation. The related profile-management PR #54126 was closed unmerged and is not present on main.

## Best Possible Solution

Keep this issue open until maintainers either implement or explicitly decline a supported Docker multi-instance lifecycle. A good implementation would add a documented `openclaw instance` command or Docker/ClawDock equivalent that records per-instance metadata, validates or allocates unique host ports and paths, isolates Compose project/container names, and provides safe create/list/update/delete cleanup with Docker docs and targeted tests.
