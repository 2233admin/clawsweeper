## Summary

Keep open: the PR targets a real gap that current main still has, but it should not merge as-is because the new remediation text points users at an unsupported `openclaw config.patch` command in the exact failure path it is trying to improve.

## What This Changes

The PR adds `hooks.enabled`/`hooks.token` cross-field validation to config validation and makes the gateway startup error more actionable when hooks are enabled without a token.

## Best Possible Solution

Land a corrected version that rejects `hooks.enabled: true` with a missing or blank `hooks.token` during merged config write validation, keeps the runtime guard as a safety net, uses only supported CLI/RPC remediation syntax, and adds focused validation, gateway RPC, and runtime error tests.
