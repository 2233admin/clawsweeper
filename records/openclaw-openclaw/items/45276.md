## Summary

Keep open. Current main still compares the candidate address after normalization against raw lowercased tailnet strings, so equivalent expanded/compressed IPv6 forms can miss; this PR applies the same normalizer to both sides and adds focused regression coverage. The remaining decision is normal maintainer review, rebase/current CI, and whether the exported helper contract should be kept.

## What This Changes

The PR normalizes stored tailnet IPv4 and IPv6 addresses before `isLocalGatewayAddress` compares them, and adds gateway tests for equivalent IPv6 spellings, IPv4-mapped IPv4, loopback, and empty inputs.

## Best Possible Solution

Keep the narrow normalization approach if maintainers want `isLocalGatewayAddress` preserved: compare both candidate and tailnet addresses through the shared IP normalizer, retain the focused canonical-form regression tests, then merge only after current CI is green.
