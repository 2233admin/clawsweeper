## Summary

Keep open. Current main still compares tailnet IPv6 addresses by lowercased raw text, so expanded and compressed forms of the same Tailscale ULA can fail to match; the PR’s production change is a narrow normalization fix and the updated test approach addresses the earlier Greptile mock concern. The helper currently has no call sites on main, so this is best treated as a small exported-helper correctness fix that needs normal maintainer review, rebase, and CI rather than cleanup closure.

## What This Changes

The PR normalizes stored tailnet IPv4/IPv6 addresses before comparing them in `isLocalGatewayAddress` and adds gateway net tests for IPv6 canonical forms, uppercase IPv6, IPv4-mapped input, loopback, and empty inputs.

## Best Possible Solution

Land a rebased, CI-green version of the normalization change if maintainers want to keep `isLocalGatewayAddress` as a reliable exported helper; if the helper remains unused, maintainers may also choose to remove or defer it deliberately rather than leaving a partially tested contract.
