## Summary

Keep this PR open, but the proposed patch is not mergeable as-is. The requested Discord button context behavior is still missing on current main, and the branch edits obsolete Discord paths plus adds Discord-specific prompt guidance in core instead of using the active plugin hook.

## What This Changes

This PR stores Discord card text on component entries, appends that context to synthesized button/select fallback interaction text, and adds Discord Components v2 prompt guidance.

## Best Possible Solution

Move the useful behavior into the active Discord plugin: capture visible component-card text while building entries, store it with button/select entries, append it to synthesized fallback interaction text only when appropriate, keep Discord prompt wording in `messageToolHints`, and cover button plus select regressions.
