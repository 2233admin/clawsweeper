## Summary

Keep this PR open. Current main does not implement the intended OpenClawBrain install identity normalization: discovery still derives `@openclawbrain/openclaw` as the unscoped id `openclaw`, install still logs manifest/package mismatch chatter for non-matching scoped package names, and the repo has no OpenClawBrain-specific regression coverage or alias entry.

## Best Possible Solution

Review and land this PR or an equivalent current-main patch that gives plugin identity aliases a single source of truth, normalizes `@openclawbrain/openclaw` to `openclawbrain` in discovery, suppresses the known manifest/package mismatch info during install, and keeps the targeted discovery/install regression tests.
