## Summary

Keep this issue open. Current main has sub-agent session isolation, sandbox/tool-policy controls, trusted tool policies, and normal before_tool_call decisions, but it still does not provide a plugin-returned isolation directive, per-operation quarantined execution constraints, or structured trust/provenance metadata on isolated results. The linked prerequisite hook-enrichment issues remain open, so this is still a core/plugin API and security design item rather than an implemented feature or ClawHub-only plugin request.

## Best Possible Solution

Keep this open as a core/plugin API design item. The best path is an additive, documented, versioned hook/runtime capability that builds on before_tool_call, trusted tool policies, subagents, sandboxing, and tool policy, while explicitly defining enforcement guarantees, per-operation constraints, and structured trust/provenance metadata. It should be coordinated with #48503, #48506, and #48509 before being treated as a single implementation task.
