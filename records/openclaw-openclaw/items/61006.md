## Summary

Keep open. Current main partially improves the old synthetic replacement by prepending a retry marker to the original body, but the marker is still passed as the embedded runner's visible prompt and can be persisted as a user turn. The existing transcriptPrompt/runtime-context API supports the requested separation, but this fallback retry path does not use it, and the current unit test still asserts the prefixed prompt.

## Best Possible Solution

Preserve the original user body as the transcript-visible prompt for fallback retries, while carrying retry/prelude context through transcriptPrompt separation or hidden runtime context. Update resolveFallbackRetryPrompt coverage to expect the original body for history retries, and add an embedded-run regression proving persisted role:user content does not include the fallback marker or prelude.

## Risks / Open Questions

- A direct change that returns only the original body could remove useful fallback context for cross-provider/Claude CLI retries unless that context is moved into transcriptPrompt separation, hidden runtime context, a lifecycle event, or UI metadata.
- Until fixed, UI history and transcript consumers may still see non-user-authored fallback text as a role:user message.
