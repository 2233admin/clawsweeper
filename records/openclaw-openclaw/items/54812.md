## Summary

Keep open. Current main still authorizes config writes only at channel/account scope; the requested DM/group/topic/conversation-scoped policy is absent from the shared authorization model, command gates, bundled channel schemas, docs, and contract tests. Existing ClawSweeper comments on the issue remain accurate for the checked-out main state.

## What This Changes

Request per-context `configWrites` overrides for DMs, groups, topics, conversations, and future multi-context channel types instead of only channel/account-wide grants.

## Best Possible Solution

Add a backward-compatible generic config-write policy model: carry normalized conversation identity through channel/plugin boundaries, origin scopes, write targets, and denial messages; define inheritance from channel/account defaults into DM/group/topic/thread overrides; expose narrow SDK helpers; update bundled multi-context schemas/docs; and add contract tests for allow, deny, and inheritance behavior.
