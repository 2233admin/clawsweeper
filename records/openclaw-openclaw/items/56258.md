## Summary

Keep open. Current main has overlapping shutdown hardening for Telegram polling, lifecycle hooks, MCP/LSP runtime disposal, WebSocket close, and HTTP close, but it does not implement this PRâ€™s central change: a per-subsystem timeout around each async stop/close step in `createGatewayCloseHandler`. Several subsystem calls are still awaited sequentially without a local deadline, and the PR also has unresolved review concerns that need author/maintainer handling before merge.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The best path is either to update this PR with a safe shared shutdown-step/race helper that covers every async teardown step, handles late rejections, and preserves HTTP restart correctness, or to explicitly fold that scope into a canonical shutdown hardening PR such as #41296 before closing this one as superseded.
