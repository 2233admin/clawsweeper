## Summary

Keep open. Current main still intentionally emits the runtime empty-allowlist warning for auto-discovered workspace/global plugins, and tests assert that behavior; only metadata snapshots and allowlisted/tracked plugins are quiet. The maintainer discussion already calls this a real UX/policy issue, so there is no strong close reason.

## What This Changes

The issue asks for first-run plugin allowlist/provenance warnings to be less alarming and more actionable, including clearer plugin IDs, docs/CLI remediation, and safer handling of channel-id/config confusion.

## Best Possible Solution

Keep the native-plugin trust warning, but make the runtime UX actionable: distinguish first-run/no explicit plugin config from configured-but-mismatched allowlists, print the exact plugin IDs to add, point to `openclaw plugins list --enabled --verbose` and `openclaw plugins inspect`, and add focused regression coverage for workspace/global discovery plus channel/plugin ID examples.
