## Summary

Keep open. Current main has not materially changed the affected browser CLI registration/output path since the reported v2026.4.26 release, and the existing source only emits JSON for `open`/`tabs` if the parent `json` option reaches the manage handler. The report remains a plausible automation-breaking regression that needs a focused fix and regression coverage through the real lazy browser command path.

## What This Changes

The issue reports that `openclaw browser --json open` and `openclaw browser --json tabs` emit plain text on stderr instead of JSON on stdout in release 2026.4.26.

## Best Possible Solution

Add regression coverage that exercises `openclaw browser --json open` and `openclaw browser --json tabs` through the real lazy browser registration path, then make parent JSON mode reliably reach the manage handlers so stdout stays structured JSON and ordinary logs stay off stdout.
