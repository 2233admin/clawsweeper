## Summary

Keep this PR open. Current main still does not implement the requested `encoding_format` behavior in the active package-backed remote embedding helper, the OpenAI direct embedding path, or the OpenAI batch path. The submitted PR is narrow and not security-sensitive, but as-is it only changes the mirror `src/memory-host-sdk` file and would miss the package export used by the plugin SDK.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rework the fix against current main so the active package helper and OpenAI direct/batch embedding paths send `encoding_format: "float"` only where appropriate, preferably via a provider-scoped or configurable `encodingFormat` option with focused regression tests. If #43910 becomes the accepted implementation, land that and then close this PR as superseded.

## Risks / Open Questions

- Landing this PR exactly as submitted may miss the active package-backed SDK helper because `src/plugin-sdk/memory-core-host-engine-embeddings.ts` re-exports `packages/memory-host-sdk`.
- A blanket `encoding_format: "float"` on shared OpenAI-compatible helpers may regress providers that reject that parameter; current docs explicitly call out that some OpenAI-compatible embedding providers reject `encoding_format`.
- The OpenAI direct and batch embedding request builders would remain uncovered unless the final fix updates those paths or routes them through a shared configurable option.
