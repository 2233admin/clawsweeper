## Summary

Keep this PR open. Current main still omits `encoding_format` from the active remote embedding helper and the OpenAI direct/batch request builders, so the compatibility problem described by the PR is not implemented on main. The PR is plausible and low-risk, but it is incomplete against current main because the public SDK export now points at the package implementation and related OpenAI paths build their own request bodies. Related #43910 is still only an open candidate from the provided context, not shipped replacement evidence.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rework it against current main so the active package helper and OpenAI direct/batch embedding request builders send `encoding_format: "float"` only where appropriate, with focused regression tests for a strict OpenAI-compatible proxy. If maintainers prefer the configurable approach, review and land #43910 first, then close or supersede this PR after that implementation is merged and shipped.
