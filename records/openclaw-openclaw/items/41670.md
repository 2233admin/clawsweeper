## Summary

Keep this PR open. Current main still omits `encoding_format` from the active OpenAI-compatible embedding request bodies, so the user-visible compatibility problem is not implemented on main. The PR is a plausible focused bug fix, but the one-line change is incomplete against current main because it only changes the `src/memory-host-sdk` mirror path while the public plugin SDK export and several provider paths build request bodies elsewhere. Related #43910 is still open and unmerged, so it is a candidate replacement rather than shipped evidence.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rework it against current main so the active package helper and the OpenAI direct/batch embedding request builders send `encoding_format: "float"` where appropriate, with focused regression tests for a strict OpenAI-compatible proxy. Alternatively, let #43910 supersede this after that configurable implementation is reviewed, merged, and shipped.
