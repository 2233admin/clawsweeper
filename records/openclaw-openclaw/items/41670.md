## Summary

Keep this PR open. Current main still omits `encoding_format` from the active package-backed remote embedding helper, the mirrored helper, and the bundled OpenAI direct/batch embedding paths. The submitted patch is narrow and the compatibility request is real, but it only changes the mirrored `src/memory-host-sdk` helper, while the public SDK facade exports the package implementation. The broader related #43910 remains open, so it is an implementation candidate rather than a merged superseding fix.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as-is. Rework the fix against current main so the active `packages/memory-host-sdk` helper and the bundled OpenAI inline/batch embedding paths support an owner-approved `encodingFormat`/`encoding_format` policy. Given the documented provider split, prefer a configurable or provider-scoped policy over a global hardcoded field, and update schema/types/docs plus focused tests. If #43910 becomes the accepted implementation and merges, close this PR as superseded by that merged work.
