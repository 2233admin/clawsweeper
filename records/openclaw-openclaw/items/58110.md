## Summary

Keep this open. Current main has useful multi-agent primitives, but it still does not implement the central requested product shape: a chat-app UI with agents as contacts, visible agent availability, per-agent conversations as first-class chat windows, and user-created group chats that invite multiple agents into one shared collaborative room. Recent discussion also adds useful production context around budget/status, context handoff, and capability discovery, so this remains actionable product/UI work rather than cleanup.

## Best Possible Solution

Keep this open as a product/UI umbrella unless maintainers intentionally split it. The best path is to define acceptance criteria for a Control UI contact/sidebar experience, per-agent conversation switching/history, visible agent availability or economic state, and a first-class multi-agent room/invitation model, while reusing existing `agents.list`, `sessions.list`, session routing, Broadcast Groups, and sub-agent primitives where they fit.
