## Summary

Keep open. Current main has useful primitives for isolated agents, session switching, Broadcast Groups, sub-agents, and agent administration, but it still does not implement the central requested product shape: agents as chat contacts, visible per-agent availability/economic state, first-class per-agent chat windows, and user-created group chats that invite multiple agents into a shared collaborative room. The KinthAI comment adds relevant production context around budget/status, context handoff, and capability cards, so this is still actionable product/UI work rather than cleanup.

## Best Possible Solution

Keep this open as a product/UI umbrella unless maintainers intentionally split it. The best path is to define acceptance criteria for a Control UI contact/sidebar experience, per-agent conversation switching/history, visible agent availability or economic state, and a first-class multi-agent room/invitation model, while reusing existing `agents.list`, `sessions.list`, session routing, Broadcast Groups, and sub-agent primitives where they fit.
