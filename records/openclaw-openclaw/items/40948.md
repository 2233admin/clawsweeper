## Summary

Current main still does not strip leading `<session-recap>` / `<session_recap>` blocks from the shared inbound metadata sanitizer, while the Control UI and TUI still route user-visible text through that sanitizer. The provided GitHub context also shows #41013 remains open and unmerged with closing syntax for this issue, so this issue should stay open until that PR lands or maintainers choose a replacement fix.

Review #41013 or apply an equivalent narrow fix in `src/auto-reply/reply/strip-inbound-meta.ts`: add a session-recap quick check, strip only complete leading `<session-recap>` / `<session_recap>` blocks in both `stripInboundMetadata()` and `stripLeadingInboundMetadata()`, preserve mid-message and malformed literals, and add focused regression tests in `src/auto-reply/reply/strip-inbound-meta.test.ts` plus any UI/TUI consumer coverage maintainers want.
