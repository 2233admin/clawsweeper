## Summary

Keep this issue open. Current main still has no session-recap handling in the shared inbound metadata sanitizer, while the Control UI and TUI still render user-visible text through that sanitizer. The issue is also paired with open PR #41013 using closing syntax, so it should wait for that PR to land or for maintainers to choose an equivalent replacement fix.

## What This Changes

The issue requests stripping leading `<session-recap>` / `<session_recap>` scaffolding from user-visible Control UI and TUI user message rendering while preserving literal mid-message content.

## Best Possible Solution

Keep this issue open and review PR #41013, or apply an equivalent narrow fix in `src/auto-reply/reply/strip-inbound-meta.ts`: add a session-recap quick check, strip only complete leading `<session-recap>` / `<session_recap>` blocks in both exported strip functions, preserve mid-message and malformed literals, and add focused regression tests in `src/auto-reply/reply/strip-inbound-meta.test.ts` plus consumer coverage if maintainers want it.
