## Summary

Keep open. Current main still has a source-proven Anthropic Messages payload path where adjacent same-role turns can survive: replay validation only merges consecutive `user` turns, the focused test still expects adjacent `assistant` turns to remain separate, and the Anthropic transport returns converted payload entries without a final same-role coalescing pass. The prior MiniMax and trailing-prefill reports address related strict-turn-shape problems but do not supersede this custom `anthropic-messages` proxy case.

## What This Changes

The issue requests Anthropic-compatible message normalization so custom `anthropic-messages` providers do not send consecutive same-role messages that strict proxies reject with 400 errors.

## Best Possible Solution

Add focused Anthropic-compatible normalization at the replay or outbound transport boundary that coalesces adjacent user and assistant payload entries and strips invalid trailing assistant payloads without changing unrelated provider policies. Update the focused tests, transcript hygiene docs, and changelog if the shipped behavior changes.

## Reproduction Assessment

Yes. Feed `validateAnthropicTurns` a `user, assistant, assistant` history and current main preserves all three turns; separately, the Anthropic transport can append adjacent `user` payloads when tool results are converted after a user entry because it returns `params` without a final coalescing pass.

## Solution Assessment

Yes. A narrow Anthropic Messages replay/transport normalization is the maintainable fix; patching the bundled dependency directly or applying a global all-provider merge would be broader and risk changing providers with different replay contracts.
