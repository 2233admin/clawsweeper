## Summary

Keep open. Current main still preserves adjacent assistant turns and can return uncoalesced Anthropic Messages payload entries for custom `anthropic-messages` providers; I did not find a merged fix or a paired open PR that closes this issue. This is a narrow provider payload-shape bug suitable for a focused fix PR.

## What This Changes

The issue requests Anthropic-compatible message normalization so consecutive same-role payload entries are merged before API calls, avoiding strict custom proxy 400s.

## Best Possible Solution

Add a focused Anthropic Messages outbound normalization step after transport transforms, or an equivalent replay-boundary helper, that coalesces adjacent user and assistant payloads without changing unrelated provider policies. Cover assistant/assistant and transport-generated user/user adjacency with regression tests, and update transcript hygiene docs plus the changelog if behavior changes.

Yes. ## Solution Assessment

Yes. The best direction is a narrow Anthropic-compatible replay or outbound-payload normalization; a global all-provider merge or direct bundled dependency patch would be broader and riskier than fixing the strict Anthropic Messages contract at OpenClaw's boundary.
