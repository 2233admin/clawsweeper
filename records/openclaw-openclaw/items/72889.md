## Summary

Keep open. Current main still leaves the reported Anthropic-compatible assistant duplicate path unresolved: the Anthropic replay validator only coalesces consecutive user turns, the current test suite still asserts consecutive assistant turns are preserved, and the native Anthropic Messages transport returns converted payload messages without a final same-role coalescing pass.

## What This Changes

The issue asks OpenClaw to normalize Anthropic-compatible outbound messages so adjacent same-role turns do not trigger 400 errors from stricter custom proxies.

## Best Possible Solution

Add a focused Anthropic-compatible normalization step at the replay or native transport boundary that coalesces adjacent user and assistant payloads while preserving tool-result batching, signed/redacted thinking blocks, and the existing thinking-enabled prefill policy. Document the behavior and add a user-facing changelog fix entry if shipped behavior changes.

## Reproduction Assessment

Yes. A high-confidence focused reproduction is to pass an Anthropic history with adjacent assistant messages through `validateAnthropicTurns` and observe the current test-backed length of three, or to inspect `convertAnthropicMessages` returning adjacent converted roles without a final merge.

## Solution Assessment

Yes for the narrow same-role coalescing request, but not for the workaround's unconditional trailing assistant stripping. Trailing assistant prefill behavior should remain policy-gated or be handled as a separate provider capability decision.
