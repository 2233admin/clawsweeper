## Summary

Keep open. Current main still retries Feishu WebSocket startup rejection, but after `WSClient.start()` returns it waits only for abort and does not supervise SDK retry exhaustion or a later silent stall. The provided context also shows open maintainer PR #73739 with closing syntax for this issue, so the right cleanup path is to review and land or replace that implementation candidate before closing.

## Best Possible Solution

Keep this issue open and use PR #73739 as the current implementation candidate. The fix should stay in the Feishu plugin transport, observe SDK retry exhaustion or post-start stalls through the SDK callback/reconnect-info surface, recreate the WSClient with bounded abortable backoff, preserve shutdown cleanup, and add focused regression coverage. Close this issue only after that PR or an equivalent replacement lands on main.
