## Summary

Keep open. Current main still only retries Feishu WebSocket startup rejection; after `WSClient.start()` returns it waits for abort and does not supervise SDK retry exhaustion or a later silent stall. Live GitHub data also shows open maintainer PR #73739 links this issue for closure, so cleanup should wait for that PR or an equivalent replacement to land.

Keep this issue open and use PR #73739 as the current maintainer implementation candidate. Close this issue only after that PR or an equivalent replacement lands on main.
