## Summary

Keep this issue open. Current main still does not preserve or forward `extraBody`/`extra_body` for the bundled OpenAI TTS provider: TTS config can carry object-valued provider fields, but the OpenAI speech provider normalizes only known fields and `openaiTTS` serializes a fixed `/audio/speech` body. A newer generic OpenAI-compatible speech helper has an extra-body-style extension point, and DeepInfra uses it, but the bundled OpenAI provider has not been moved to that path or given equivalent passthrough handling.

## Best Possible Solution

Implement a narrow bundled OpenAI speech-provider change: accept `extraBody` and `extra_body` JSON records under `messages.tts.providers.openai` and legacy `messages.tts.openai`, preserve them through config normalization and provider reads, merge them into both normal and telephony `/audio/speech` request bodies with documented collision precedence, add focused request-body tests, and document the field on the public TTS/OpenAI provider docs.
