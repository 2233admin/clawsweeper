## Summary

Keep #39900 open. Current main still does not implement TTS `extraBody`/`extra_body` passthrough for the bundled OpenAI speech provider: generic TTS config can carry object fields, but the OpenAI provider normalizes only known TTS fields and `openaiTTS` serializes only the fixed `/audio/speech` payload.

## Best Possible Solution

Keep the issue open and implement a narrow bundled OpenAI speech-provider change: accept `extraBody` and `extra_body` JSON records in `messages.tts.providers.openai`, preserve them through `resolveConfig`, merge them into both normal and telephony `/audio/speech` request bodies with documented collision precedence, add focused request-body tests, and document the field on the OpenAI provider docs page.
