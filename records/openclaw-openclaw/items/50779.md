## Summary

Keep this issue open. Current main has broad Control UI media support for assistant/generated images, user upload MediaPath entries, and MEDIA: attachments, but the specific read-tool result shape still has gaps: direct tool-result image blocks with `{ type: "image", data, mimeType }` are not converted into renderable image URLs, live tool events flatten result payloads to text, and tool-message images are hidden behind the collapsed tool disclosure when they are detected. The related open PR #52025 is a direct implementation candidate, so this should stay open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open and review PR #52025 or a replacement focused PR. The fix should teach the Control UI to preserve/render read-tool image blocks shaped as `{ type: "image", data, mimeType }`, including nested tool-result content where applicable, and make the image visibly available outside an otherwise collapsed tool disclosure. It should also decide whether live tool events should carry structured image content instead of text-only output. Add focused UI regression coverage around read-tool image results; no core media policy or remote image fetching change is needed.
