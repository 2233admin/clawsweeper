## Summary

Current main still has the reported Control UI gap: the gateway returns an effective verbose level, but the UI history loader drops it and the top-level chat render still hardcodes tool auto-expansion off. This should stay open because #74398 is an open, unmerged PR paired to this exact issue with closing syntax.

## What This Changes

Control UI tool cards should initialize expanded when the effective chat verbose level is `full`.

## Best Possible Solution

Carry the effective `chat.history.verboseLevel` through Control UI state, derive `autoExpandToolCalls` from `verboseLevel === "full"`, preserve manual tool-card toggles, clear stale verbose state on the same paths as thinking state, and cover the controller/render path with focused regression tests plus a user-facing changelog fix entry.
