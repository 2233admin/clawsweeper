## Summary

Keep open. Current main still has the reported Control UI data-flow gap: the gateway returns the effective verbose level, but the Control UI history loader stores only thinking level and the top-level chat render still hardcodes tool auto-expansion off. The issue also has open PR #74398 with closing syntax for this bug, so it should remain open until that PR lands or is replaced.

## What This Changes

The issue asks for Control UI tool cards to default expanded when the effective verbose setting is `full`.

## Best Possible Solution

Carry the effective `chat.history.verboseLevel` through Control UI state, derive tool auto-expansion from `verboseLevel === "full"`, preserve manual disclosure toggles, clear stale verbose state on the same paths as thinking state, and cover the controller/render path with focused regression tests plus the appropriate user-facing changelog entry.
