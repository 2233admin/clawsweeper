## Summary

Current main still leaves Control UI tool-card auto-expansion disabled: the chat renderer has an `autoExpandToolCalls` path, and the gateway already returns an effective `verboseLevel` from `chat.history`, but `app-render` passes `autoExpandToolCalls: false` unconditionally and the chat controller drops `verboseLevel` from the history response. The related fix PR #50034 was closed unmerged, so this remains actionable.

## Best Possible Solution

Keep this issue open and land a narrow Control UI fix that derives `autoExpandToolCalls` from the effective chat verbose level, preferably from the existing `chat.history.verboseLevel` response, while preserving manual per-card toggles and collapsed defaults when verbose is not `full`. Add regression coverage for global `agents.defaults.verboseDefault = "full"` and per-session verbose behavior.
