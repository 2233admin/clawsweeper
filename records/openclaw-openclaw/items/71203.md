## Summary

Keep open. Current main still has the stale non-default-agent models.json gap, and this PR targets the right gateway startup surface, but the latest head still drops the explicit scoped, metadata-only provider discovery options for CLI-only model configurations and is also missing the required changelog entry for a user-facing fix.

## What This Changes

The PR adds a gateway startup helper that refreshes models.json for the active/default agent directory and configured agent directories, wires it before channel startup, and expands gateway startup tests for custom agents.

## Best Possible Solution

Land a revised startup refresh that regenerates unique active/default and configured agent models.json files independently of primary-model prewarm while always keeping provider discovery bounded to metadata-only scoped providers, including an explicit empty providerDiscoveryProviderIds array for CLI-only configs.

## Reproduction Assessment

Yes. Configure the default/configured agent model providers as CLI-backed so the PR computes providerDiscoveryProviderIds as an empty array; the current ternary turns that into {}, so ensureOpenClawModelsJson receives an undefined discovery scope instead of a scoped-empty metadata-only request.

## Solution Assessment

No for the latest PR head. The feature direction is narrow and maintainable, but the implementation should preserve providerDiscoveryProviderIds: [] together with providerDiscoveryEntriesOnly: true and the startup timeout, plus add the required changelog entry.
