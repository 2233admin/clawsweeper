## Summary

Keep this PR open. The branch addresses a real gap in current main, but the latest head still drops the explicit empty provider-discovery scope for CLI-only startup refresh, which can reintroduce unscoped provider runtime discovery before channels start.

## What This Changes

The PR adds a gateway startup helper that refreshes `models.json` for the active/default and configured agent directories, wires it before channel startup, and adds gateway startup regression tests for custom agents and the synthetic-main fallback.

## Best Possible Solution

Land a revised startup refresh that keeps configured-agent cache regeneration independent of primary-model prewarm while always preserving bounded metadata-only provider discovery, including explicit empty provider scopes for CLI-only configurations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep CLI-only startup refresh scoped:** `src/gateway/server-startup-post-attach.ts:163-170`
  - body: When every collected provider is a CLI backend, `providerDiscoveryProviderIds` filters down to `[]`, but this branch replaces the scoped options with `{}`. `ensureOpenClawModelsJson` then reaches provider discovery without `providerDiscoveryEntriesOnly` or an explicit `onlyPluginIds` scope, so a CLI-only gateway can load unrelated provider runtimes before channels start. Pass the empty scope with metadata-only options, or otherwise disable discovery for that case, and cover it with a regression test.
  - confidence: 0.86
