## Summary

Keep this PR open. Current main still does not refresh configured non-default agent models.json caches during gateway startup: startup only schedules the primary-model prewarm path, that path returns before calling ensureOpenClawModelsJson when no default primary model is configured, and the current tests assert that no models.json refresh happens in that case. The PR is a plausible implementation candidate for the stale-cache report, but it still needs normal maintainer review rather than ClawSweeper closure.

## What This Changes

The PR adds a gateway startup helper that refreshes models.json for the default and configured agent directories with scoped provider discovery, exports it for tests, and adds regression coverage for custom agents with and without a default primary model.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best fix is a startup refresh path that refreshes unique default and configured agent dirs independently of primary-model prewarm, preserves configured agentDir and active default-dir semantics, keeps provider discovery scoped and metadata-only, handles per-dir failures as best-effort, and avoids delaying channel startup beyond a clearly bounded cache refresh.
