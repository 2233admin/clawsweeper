## Summary

Keep this PR open. Current main still skips the linked stale-cache case because startup `models.json` refresh only happens inside primary-model prewarm, which returns early without `agents.defaults.model.primary` and only targets the active/default agent dir when it runs. The latest PR branch is a focused implementation candidate for that gap, with the prior synthetic-main/test-mock nits addressed, so the right next step is maintainer re-review and completion of latest-head CI rather than cleanup closure.

## What This Changes

The PR adds a gateway startup helper that refreshes the default and configured agent `models.json` files with scoped metadata-only provider discovery, wires it before channel startup, and adds regression tests for custom agents plus synthetic-main dedupe.

## Best Possible Solution

Land a reviewed startup refresh path that refreshes unique active/default and configured agent dirs independently of primary-model prewarm, keeps provider discovery metadata-only and provider-scoped, preserves configured `agentDir` and workspace semantics, handles per-dir failures as best-effort, and ships with targeted gateway startup regression coverage plus the changed gate green.
