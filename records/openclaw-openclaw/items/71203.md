## Summary

Keep this PR open. Current main still has the stale-cache gap for configured non-default agents, but the proposed patch needs one revision because CLI-only model configurations can drop the scoped metadata-only discovery options and fall back to broad provider runtime discovery during gateway startup.

## What This Changes

The PR adds a gateway startup helper that refreshes `models.json` for the active/default and configured agent directories, wires it before channel startup, exports it for tests, and adds regression coverage for custom agents with and without a default primary model.

## Best Possible Solution

Land a revised startup refresh that refreshes unique active/default and configured agent dirs independently of primary-model prewarm, while always preserving bounded metadata-only provider discovery, including explicit empty provider scopes for CLI-only configurations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep CLI-only startup refresh scoped:** `src/gateway/server-startup-post-attach.ts:164-170`
  - body: When every configured model provider is a CLI backend, `providerDiscoveryProviderIds` becomes `[]`, but this branch replaces the scoped startup options with `{}`. `ensureOpenClawModelsJson` then reaches provider discovery without `providerDiscoveryEntriesOnly` or `onlyPluginIds`, which current discovery treats as an unscoped runtime provider load; a CLI-only gateway can run unrelated provider discovery before channels start. Please pass the empty provider scope with `providerDiscoveryEntriesOnly: true` or otherwise disable discovery for this case, and cover it with a regression test.
  - confidence: 0.86
