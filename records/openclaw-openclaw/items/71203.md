## Summary

Keep this PR open for maintainer review. Current main still does not refresh configured non-default agent models.json files during startup; it only prepares the active/default agent directory through primary-model prewarm, and that path returns before cache work when no default primary model is configured. The PR remains a concrete candidate for the linked stale-cache bug, with normal review needed around startup latency and branch validation.

## What This Changes

This PR adds a gateway startup helper that refreshes models.json for the active default and configured agent directories with scoped metadata-only provider discovery, wires it before primary-model prewarm, and adds gateway regression tests for custom agents with and without a default primary model.

## Best Possible Solution

Land or replace a narrow gateway startup fix that refreshes unique active/default and configured agent directories independently of primary-model prewarm, using the existing listAgentIds/resolveAgentDir seams, bounded metadata-only provider discovery, best-effort per-dir error handling, and regression tests for custom agents without a default primary model.
