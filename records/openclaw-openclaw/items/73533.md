## Summary

Keep open. Current main still has the Windows-sensitive behavior: `resolvePreferredOpenClawTmpDir()` checks and returns `/tmp/openclaw` before falling back to `os.tmpdir()`. The PR addresses the linked #60713 bug in the right area, but it should not be closed as obsolete or merged as-is because the new Windows regression test appears to use `path.win32.join` while production uses native `path.join`, which makes the test expectation diverge on Linux/macOS CI.

## Best Possible Solution

Keep this PR open as the implementation candidate for #60713. The best path is to fix the test expectation to mirror production `path.join`, rebase if needed, run the targeted temp-dir test plus the changed gate, and then land the focused resolver change if CI stays green.
