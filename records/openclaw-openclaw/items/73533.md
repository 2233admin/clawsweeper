## Summary

Keep this PR open. Current main still has the Windows-sensitive resolver behavior the PR is trying to change: it probes and returns `/tmp/openclaw` before any Windows guard, while related log paths use that resolver and gateway locks use `os.tmpdir()`, matching the split-state bug described in #60713. The latest PR head is separate from main and remains a focused implementation candidate, with the earlier Greptile path-join concern apparently addressed in the latest diff.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for #60713. Review and land the Windows guard after targeted tests and the changed gate are green; once merged, the linked Windows temp split-state issue can close as implemented by the landed fix.
