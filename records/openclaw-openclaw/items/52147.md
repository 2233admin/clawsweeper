## Summary

Keep open. Current main mitigates the reporter's original default-600s scenario by raising the default agent runtime timeout to 48 hours, but the core classification bug remains: a non-compaction run timeout still has no tool-execution timeout cause and can still drive assistant profile rotation/model fallback as a timeout.

## Best Possible Solution

Add phase-aware timeout state in the embedded runner, preferably a small timeoutCause enum or timedOutDuringToolExecution flag derived from the existing active item/tool lifecycle. When the global run timer fires during active tool execution, return a clean run/tool timeout or partial-progress result without rotating auth profiles or falling back models. Keep model idle/provider request timeouts and high-context timeout compaction on their existing retry/fallback paths, with focused regression coverage.
