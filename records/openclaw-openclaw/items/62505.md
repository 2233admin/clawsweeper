## Summary

Keep open. Current main still has an actionable failure path for the narrowed report: background exec completion enqueues an exec-event wake, but heartbeat processing is disabled before pending exec events are inspected when `heartbeat.every` resolves to `0m`; the related fix PR was closed unmerged.

## What This Changes

The issue requests restoring OpenAI Codex/Coding Agent completion behavior after a regression where coding work and long-running background execs acknowledge work but never deliver final results.

## Best Possible Solution

Land a focused heartbeat/exec fix that allows targeted `exec-event` wakes to process queued completion events when only the scheduled heartbeat interval is disabled, while preserving the global heartbeat enable switch and documenting the intended `0m` semantics.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure `heartbeat.every: "0m"`, let a background exec enqueue an `exec-event`, and current main skips heartbeat processing as disabled before inspecting that event.

## Solution Assessment

Unclear but the narrowest maintainable direction appears to be letting only targeted exec-event wakes bypass the per-agent interval gate. If maintainers intend `0m` to suppress all event delivery, the safer solution is a docs/config clarification rather than code pretending completion wakes will still fire.

## Repair Work Prompt

Fix the background exec completion regression from https://github.com/openclaw/openclaw/issues/62505. Current main enqueues exec completion system events and requests an `exec-event` wake, but `heartbeat.every: "0m"` causes `runHeartbeatOnce` and `startHeartbeatRunner` to skip before inspecting targeted exec events. Implement the narrow behavior intended by the issue discussion and closed PR #67913: targeted `exec-event` wakes should process queued exec completion events even when normal scheduled heartbeat cadence is disabled with `every: "0m"`, while the global heartbeat enable switch remains respected. Do not turn `0m` back into a scheduled cadence and do not broaden unrelated heartbeat/cron/commitment behavior. Add focused regression coverage for a disabled-interval exec-event wake with captured delivery context and for the scheduler targeted wake path. Update docs/changelog only if the behavior contract changes user-visible semantics; any changelog entry must credit the reporter/commenters, not bots.
