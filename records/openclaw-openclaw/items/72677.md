## Summary

Keep this PR open. Current main still does not implement the proposed ghost-run warning, run-log warning annotation, or `cron.ghostRunWarningThresholdMs` config surface, and the PR is an active implementation candidate for #63106 rather than something already implemented or superseded. The diff is mostly additive observability, but it needs maintainer review because it is warning-only and does not resolve the broader main-session payload handoff bug reported in #73189.

## What This Changes

This PR adds a configurable ghost-run warning and cron run-log annotation for fast `ok` main-session `systemEvent` cron jobs using `wakeMode: "next-heartbeat"`, updates config schema/help metadata and run-log parsing/search, adds Gateway cron tests, and includes small lint/test-helper cleanups outside the cron warning path.

## Best Possible Solution

Keep this PR open for maintainer review. Maintainers should decide whether the additive `cron.ghostRunWarningThresholdMs` plus run-log warning annotation is the right narrow observability fix for #63106, or whether the fix should instead wait for a deeper main-session handoff/health correction that also accounts for #73189. If accepted, land it with targeted cron, config schema/help, generated schema, run-log, formatter, and lint validation.
