## Summary

Keep #39777 open. Current main still has the reported Control UI Debug Snapshots behavior: disconnected refresh returns silently, the Snapshots card has no inline loading/progress state, and the related PR #39806 remains open/unmerged rather than shipped.

## Best Possible Solution

Keep the issue open. The maintainable fix belongs in the Control UI Debug surface: pass connection state or a refresh-disabled reason into `renderDebug()`, show an inline loading/progress state in the Snapshots card while `debugLoading` is true, show a clear disconnected callout when refresh cannot run, and add focused tests for `loadDebug()` and `renderDebug()`. Review or refresh PR #39806 as the existing implementation attempt.
