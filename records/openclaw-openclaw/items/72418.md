## Summary

Keep this open. Current main still preserves the reported device-less backend self-pairing path for loopback/shared-secret backend clients, and the same reporter has an open, unmerged closing PR, so the issue should remain available for maintainer security review rather than being auto-closed.

## What This Changes

The issue asks OpenClaw to prevent loopback clients with a shared gateway token from self-declaring the `gateway-client` backend identity to skip device pairing and retain operator scopes without a registered device.

## Best Possible Solution

Keep the issue open until maintainers review the linked PR and choose the product/security direction: either document and close it as intended trust-model behavior, or harden the gateway handshake by binding backend self-pairing to a server-side backend credential or identity with focused regression coverage.
