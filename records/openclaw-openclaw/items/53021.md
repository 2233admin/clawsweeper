## Summary

Keep open. Current main still has the LAN display-link problem in several user-facing caller paths, and the PR contains useful normalization intent, but the branch should not merge as-is because it is stale against the current status renderer and mixes unrelated security/privacy-sensitive embedded-agent and skill changes into a Control UI URL fix.

## What This Changes

The PR maps some LAN Control UI display-link callers to loopback, broadens embedded-agent error messages with provider/profile/session context and raw transport reasons, and adds a bundled Drop Box document-intake skill with a PDF renderer.

## Best Possible Solution

Land a narrow current-main fix that centralizes display-only LAN-to-loopback normalization for browser Control UI URLs and applies it to dashboard, configure completion, status overview, status --all, daemon status, and setup finalization while leaving runtime bind/probe/custom/tailnet semantics unchanged; split the agent error-copy and Drop Box skill work into separate reviews.
