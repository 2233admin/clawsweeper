## Summary

Keep open. Current main still exposes `message_sending` as a delivery-level hook only: the event type and `applyMessageSendingHook()` do not carry `stopReason`, `hasToolCalls`, or a typed agent context, even though agent run metadata already has `stopReason` and pending tool-call data. Related PR #39761 is an open, unmerged implementation candidate rather than shipped behavior.

## What This Changes

Request to expose agent completion metadata such as `stopReason` and `hasToolCalls` in the `message_sending` plugin hook so plugins can suppress tool-call preamble messages structurally.

## Best Possible Solution

Thread `EmbeddedPiRunMeta` or a narrowed agent-completion context through agent command delivery and outbound delivery, expose it backward-compatibly on `PluginHookMessageSendingEvent`, add focused hook payload tests, and update the plugin hook docs. If #39761 is the chosen implementation, review/rebase it and close this issue only after an equivalent change lands on main.
