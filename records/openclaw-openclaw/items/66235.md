## Summary

Keep open. Current main still exposes `message_sending` as a delivery-level hook without `stopReason`, `hasToolCalls`, or a broader agent completion context, and the related implementation PR remains open rather than shipped.

## What This Changes

Expose agent completion/tool-call context such as `stopReason` or `hasToolCalls` to the `message_sending` plugin hook metadata so hooks can suppress pre-tool-call preambles structurally.

## Best Possible Solution

Thread a narrowed agent completion context through agent command delivery and inbound auto-reply dispatch, expose it backward-compatibly on `PluginHookMessageSendingEvent`, add focused hook payload tests, and update the plugin hook docs. If #39761 is the chosen implementation, review and rebase it; otherwise replace it with an equivalent narrower PR.
