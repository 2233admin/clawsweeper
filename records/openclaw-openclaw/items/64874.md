## Summary

Keep this draft PR open. Current main still has the same product gap the PR targets: bootstrap truncation is visible in `/context list` and stored prompt-report data, but `/status` does not show workspace-file health, edit mismatch errors do not mention bootstrap truncation, and the injected warning still lacks edit-recovery guidance. The patch is a plausible implementation candidate, but it needs rebase, current-limit handling, and tests before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, move the status rendering change to `src/status/status-message.ts`, derive thresholds from `SessionSystemPromptReport` or current config instead of hardcoded 20,000, add focused Vitest coverage for the warning text, edit mismatch hint, and `/status` output, and have maintainers decide whether `/status` should show a concise health summary or point users to `/context list` for full per-file details.
