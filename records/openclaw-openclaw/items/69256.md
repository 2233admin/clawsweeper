## Summary

Keep this PR open. Current main still deletes isolated direct-delivery cron sessions without checking active descendant subagent runs, while the PR adds the missing guard, deferred cleanup retry, docs, changelog, and regression coverage. This is not obsolete on main and should proceed through normal review rather than maintainer cleanup closure.

## Best Possible Solution

Keep the PR open and review/land this branch, or an equivalent patch, after CI and maintainer review. The best fix belongs in src/cron/isolated-agent/delivery-dispatch.ts: make direct deleteAfterRun cleanup subagent-aware, preserve the parent cron session while descendants are active, retry cleanup after the descendant tree drains, and keep the added regression tests/docs/changelog aligned.
