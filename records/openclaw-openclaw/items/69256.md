## Summary

Keep open. Current main still deletes isolated direct-delivery cron sessions without checking active descendant runs, while this PR adds the missing descendant guard and deferred cleanup coverage for the structured/threaded direct-delivery path. I did not find a blocking code-review bug in the patch, but the branch is currently dirty against main and the latest check set is not green, so it still needs maintainer rebase/validation before merge.

## What This Changes

This PR updates cron direct-delivery cleanup to wait for active subagents before `deleteAfterRun` session deletion, adds deferred cleanup retry tests, and documents the behavior in cron docs and the changelog.

## Best Possible Solution

Rebase this PR onto current main, keep the direct-cleanup descendant guard and regression tests, rerun the targeted cron dispatch test plus the changed gate, and then merge if CI is green.

## Reproduction Assessment

Yes. A high-confidence code-path reproduction exists by calling `dispatchCronDelivery` with `deleteAfterRun: true`, structured content or a threaded direct target, and active descendants under `runSessionKey`; current main reaches `sessions.delete` without the descendant guard.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix: add the same descendant awareness to direct cleanup that the text-finalization path already has, plus a deferred cleanup retry so `deleteAfterRun` is not permanently skipped. It still needs rebase and CI proof before landing.
