## Summary

Keep this PR open. Current main still deletes isolated direct-delivery cron sessions without checking active descendant subagent runs, while the PR is an active, maintainer-touched implementation candidate for that missing behavior.

## Best Possible Solution

Keep the PR open and review or land this branch, or an equivalent patch. The fix belongs in `src/cron/isolated-agent/delivery-dispatch.ts`: make direct `deleteAfterRun` cleanup check run-scoped active descendants, preserve the parent cron session while descendants are active, retry cleanup after the descendant tree drains, and keep the regression tests plus docs/changelog aligned.
