## Summary

Keep this issue open. Current main still has a credible Control UI blank-screen path matching the report and SnowSky1's related #54552 context: malformed persisted cron jobs can be returned to the UI without payload validation, while shared render, Cron list, and edit/clone paths still dereference `job.payload`. The related fix PR was closed unmerged, and no open PR currently references this issue.

## What This Changes

The issue reports that Control UI left-menu sections such as channels, instances, sessions, agents, and skills now render only a title/blank page instead of editable configuration options.

## Best Possible Solution

Make Control UI resilient to malformed persisted cron rows at the boundary and at UI consumption sites: normalize or filter invalid rows before `cron.list`/`cron.listPage` exposes them, add null-safe UI rendering/edit guards so malformed data cannot blank unrelated sections, and cover the regression with focused jsdom/controller tests plus a user-facing changelog fix entry.
