## Summary

Keep open. Current main still has the core denial-loop behavior: normal-session denied exec approval followups start an agent turn, node exec.denied events enqueue and wake the session, the reported Chinese stop phrases are not recognized, and the queue overflow behavior remains capped/summarized rather than stop-prioritized. Existing stop cleanup mitigates recognized /stop-style commands, but it does not resolve this report.

## What This Changes

The issue requests stronger stop/deny handling so exec approval denials do not cascade into repeated agent turns, stop commands take priority, delayed exec work is visible/cancellable, and message queues do not drop stop intent.

## Best Possible Solution

Keep this issue open as the canonical denial-loop tracker. The best fix should be maintainer-reviewed across exec approval followup delivery, node exec event handling, stop phrase/priority abort handling, and session queue cleanup: recognize the reported stop phrases, clear queued work on stop, prevent repeated denied exec completions from starting normal agent turns after repeated denials or after an abort, and coordinate with #69570 and #70953.
