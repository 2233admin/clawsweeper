## Summary

Keep open. Current main still lacks a configurable MCP serve operator-scope request path, and the focused implementation PR linked from the discussion is closed unmerged, so the requested least-privilege bridge behavior remains a live gap.

## What This Changes

The issue requests a supported way for `openclaw mcp serve` to request narrower operator scopes, such as repeatable `--scope` flags, config/env support, or named read/write/approval modes.

## Best Possible Solution

Add a narrow, backwards-compatible MCP serve scope selector that validates operator scope names, propagates the selected list through `OpenClawMcpServeOptions` and `OpenClawChannelBridge` into `GatewayClient`, preserves the current three-scope default when omitted, and documents cached-token and static tool-list behavior. Keep the broader per-session tool-policy work in #49183 separate.

## Reproduction Assessment

Yes. Static source inspection against current main reproduces the report: the CLI lacks `--scope`, the MCP serve option type has no scope field, and the bridge hardcodes read/write/approvals scopes; a live gateway is not required to verify this missing plumbing.

## Solution Assessment

Yes, the narrowest maintainable solution is repeatable validated `--scope <operator.scope>` plumbing into the existing `GatewayClient.scopes` contract, with docs and focused tests. Named modes or env/config aliases can be layered later if maintainers want more UX.

## Risks / Open Questions

- Scope selection changes an authorization-adjacent path; the implementation should validate against known operator scopes and fail closed for unknown values.
- Cached device-token behavior needs clear handling: `GatewayClient` reuses cached approved scopes when reusing a stored device token, so explicit MCP scope requests must not silently mislead users about an already-paired identity.
- If `tools/list` remains static, docs should state that invocation-time gateway authorization is canonical so clients do not treat registration as proof of authority.
