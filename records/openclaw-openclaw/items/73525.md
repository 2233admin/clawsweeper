## Summary

Keep open. Current main still has the split needed to reproduce the reported default-status failure: the visible Channels table is built from read-only channel discovery, while live `channels.status` data is not merged into table rows and default text status now only enables live channel checks for `--all` or `--deep`. The issue also has an open closing PR (#73662), so this should stay open until that PR or an equivalent fix is reviewed and merged.

## What This Changes

`openclaw status` should show configured/running channel rows in its Channels section instead of rendering a header-only table when `openclaw channels status` can see the live accounts.

## Best Possible Solution

Keep the fast default-status contract while ensuring configured channel rows are still visible, likely by making read-only discovery/setup fallback produce rows for configured channels or by explicitly deciding to enrich default status from the already reachable gateway. For `--all` and `--deep`, live `channels.status.channelAccounts` can fill missing rows as long as the unreachable-gateway fallback remains read-only.
