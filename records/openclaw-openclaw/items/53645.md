## Summary

Current main still appears to have the Feishu plugin action-layer bug described in the report: generic message sends can carry a resolved `replyTo`, and the lower Feishu send helpers know how to send replies, but ordinary Feishu `send` actions still pass no reply target unless the action is `thread-reply`.

## What This Changes

The issue asks for Feishu message-tool replies using `replyTo` to show quoted replies consistently after OpenClaw 2026.3.23-2.

## Best Possible Solution

Fix the Feishu plugin action handler so ordinary `send` maps explicit `replyTo` and supported reply-id aliases into `replyToMessageId` for text/card sends and `replyToId` for media sends, while leaving `thread-reply` as the explicit `reply_in_thread` path.
