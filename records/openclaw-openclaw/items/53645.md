## Summary

Current main still leaves a Feishu plugin action-path gap: ordinary `send` actions drop the resolved `replyTo` target before calling the Feishu send helpers, so the reported quoted-reply instability remains valid.

## What This Changes

The issue requests reliable Feishu quoted replies when the message tool uses the `replyTo` parameter after upgrading to OpenClaw 2026.3.23-2.

## Best Possible Solution

Preserve the generic core reply resolution and fix the Feishu plugin action layer so ordinary `send` forwards explicit `replyTo`/`reply_to` to text, presentation/card, and media sends without changing `thread-reply` semantics.
