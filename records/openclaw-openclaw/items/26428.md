## Summary

Keep open. Current main still installs, detects, and documents fish completion through `config.fish`, while fish's own completion contract supports autoloading from `~/.config/fish/completions/<command>.fish`; the directly related implementation PR is closed unmerged, so the requested behavior remains unresolved.

## What This Changes

Install fish shell completions into `~/.config/fish/completions` instead of sourcing the cached OpenClaw fish completion from `~/.config/fish/config.fish`.

## Best Possible Solution

Fish completion install and status checks should follow fish autoload semantics: write the generated fish script to the user completion directory as `<binName>.fish`, treat that file as the installed state, clean only OpenClaw-owned legacy `config.fish` completion entries, and align doctor, setup, update, docs, tests, and changelog wording.
