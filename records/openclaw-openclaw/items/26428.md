## Summary

Current main still installs fish completions by sourcing the cached script from `~/.config/fish/config.fish`, detects installation from that profile, and documents/profile-messages shell-profile installation. There is still no fish autoload install/detection path under `~/.config/fish/completions`, and the directly related implementation PR #27135 is closed unmerged.

## Best Possible Solution

Implement a fish-specific completion install and detection path in `src/cli/completion-runtime.ts`: copy the cached fish script to `${XDG_CONFIG_HOME:-$HOME/.config}/fish/completions/<binName>.fish`, detect that file as installed, safely remove legacy OpenClaw-owned `config.fish` completion blocks/source lines, and update doctor/setup messaging, docs, changelog, and focused regression tests. Use PR #27135 as prior art but rebase the behavior onto current main.
