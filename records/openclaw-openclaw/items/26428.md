## Summary

Current main still installs and detects fish completions through `~/.config/fish/config.fish` profile sourcing instead of fish's autoload completions directory. The directly related PR #27135 is closed unmerged, so the requested behavior remains valid and narrow enough for a focused fix PR.

## What This Changes

Install fish shell completions as `openclaw.fish` under fish's autoload completions directory instead of sourcing them from `~/.config/fish/config.fish`.

## Best Possible Solution

Keep this issue open and implement a focused fish-specific completion install/detection path in `src/cli/completion-runtime.ts`: copy the cached fish script to `${XDG_CONFIG_HOME:-$HOME/.config}/fish/completions/<binName>.fish`, detect that file as installed, clean up only legacy OpenClaw-owned `config.fish` entries, and update doctor/setup messaging, CLI docs/help text, changelog, and regression tests. PR #27135 is useful prior art but is closed unmerged, so a fresh or rebased fix is the right path.
