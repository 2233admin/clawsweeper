## Summary

Keep PR #59170 open. Current main still only supports global `tools.sessions.visibility` and `tools.agentToAgent`; the requested per-agent `agents.list[].tools.sessions` / `agentToAgent` schema and runtime threading are not present. The PR tracks a real open feature request (#59149) with additional user demand, but it needs rebase/rescope against current main because session access policy now lives in `src/plugin-sdk/session-visibility.ts`, and the PR appears to need cleanup for unrelated files.

## Best Possible Solution

Keep the PR open and ask for a clean rebase against current main. The best implementation is to add `agents.list[].tools.sessions` and `agents.list[].tools.agentToAgent` to the typed schema/help/generated config surface, update `src/plugin-sdk/session-visibility.ts` to resolve per-agent overrides with global fallback, thread the requester agent's `tools` through list/history/send/status, keep per-agent A2A `allow` as a target-only filter, and remove unrelated files from the PR.
