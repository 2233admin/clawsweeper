## Summary

Keep open. Current main still excludes `MINIMAX_OAUTH_TOKEN` from MiniMax managed `web_search` credential resolution and setup metadata, and the latest tagged release has the same behavior. There is already an open unmerged implementation candidate, #65785, with closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review #65785 or an equivalent maintainer patch. If MiniMax Search accepts portal OAuth tokens, update the MiniMax web search runtime resolver, web-search provider envVars/contract metadata, manifest UI hint, tests, and docs to include `MINIMAX_OAUTH_TOKEN`. If the search endpoint intentionally requires a separate Coding Plan credential, update the docs and missing-credential message to make that split explicit for OAuth chat users.
