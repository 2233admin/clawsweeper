## Summary

Keep open. Current main and the latest v2026.4.27 release still omit MINIMAX_OAUTH_TOKEN from MiniMax web_search credential resolution, metadata, UI hint, tests, and docs, and #65785 is an open unmerged implementation candidate with closing syntax for this issue.

## What This Changes

MiniMax web_search should work with the minimax-portal OAuth token instead of requiring a separate MiniMax Coding Plan API key.

## Best Possible Solution

Review and either land #65785 or an equivalent maintainer patch if MiniMax Search accepts OAuth bearer tokens; otherwise update the MiniMax Search docs, metadata, and missing-credential message to make the separate Coding Plan credential requirement explicit.

## Reproduction Assessment

Yes. With only MINIMAX_OAUTH_TOKEN available and no configured webSearch apiKey, MINIMAX_CODE_PLAN_KEY, MINIMAX_CODING_API_KEY, or MINIMAX_API_KEY, the current resolver returns undefined and web_search follows the missing-key path.

## Solution Assessment

Unclear until the MiniMax Search endpoint credential contract is confirmed. If OAuth bearer tokens are valid for that endpoint, #65785 is the narrow maintainable fix boundary because it updates the resolver, error text, provider metadata, and regression coverage.
