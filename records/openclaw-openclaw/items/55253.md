## Summary

Keep open. Current main has partial resilience work for plugin init failures, selected hook timeouts, typing cleanup, and tool-loop detection, but it still does not implement this issue's requested plugin-scoped circuit breaker, generic plugin tool-call timeout, or typing-TTL-driven session recovery. Open #41899 overlaps and may become the canonical design thread, but it has not been explicitly chosen as the replacement for this narrower incident report.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it into #41899. The best implementation is core plugin-runtime resilience: add plugin-scoped breaker state and rolling budgets around hook/tool/register failures, define generic plugin tool timeout/abort semantics, expose status/reset/fail-mode controls, and decide whether typing TTL should mark a run failed, restart/recover the session, or emit a clear outage reply.
