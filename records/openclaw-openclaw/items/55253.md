## Summary

Keep open. Current main has useful partial hardening around plugin load/register failures, invalid plugin config isolation, hook-level timeouts, Active Memory recall timeout, typing cleanup, and stuck-session diagnostics, but it still does not implement the requested plugin-scoped circuit breaker, generic hard timeout around plugin-owned tool execution, or typing-TTL-driven session recovery. Open #41899 overlaps the circuit-breaker portion only, so it is not a complete duplicate of this issue.

## Best Possible Solution

Keep this issue open or explicitly split/consolidate it so no scope is lost. The best fix is a core plugin-runtime resilience design: plugin-scoped rolling failure budgets across register, hook, and tool execution paths; timeout and AbortSignal semantics for plugin-owned tools; persisted open/half-open/forced-open breaker state; operator status/reset controls; and an explicit product decision for whether typing TTL or stuck-session diagnostics should restart, fail, or otherwise recover the active session with user-visible outage handling.
