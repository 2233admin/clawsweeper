## Summary

Keep open. Current main still wakes or continues only the restart sentinel session; it does not enumerate agent session stores or broadcast a GatewayRestart-style system event to other recently active sessions. This PR remains plausible unmerged work, but it needs maintainer review and a current-main rebase because restart recovery has since gained durable continuation queueing and narrower lock-based main-session recovery.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with an updated patch. The best implementation path is to adapt current restart sentinel handling to enumerate known agent session stores, select truly interrupted recent sessions using the agreed SessionEntry timestamp/status contract, skip cron/global/unknown and other non-resumable pseudo-sessions, enqueue the restart system event plus heartbeat wake for each selected session, preserve the existing durable continuation queue for the triggering session, and add focused regression tests for recent, stale, skipped, and multi-agent sessions.
