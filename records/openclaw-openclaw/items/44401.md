## Summary

Keep open. Current main still wakes only the restart-sentinel session after a Gateway restart; it does not broadcast a GatewayRestart system event to all recently active agent sessions. The PR remains substantive unmerged work, but it likely needs a current-main rebase and focused regression tests around cross-session wake selection.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with an updated patch against current main. The best implementation should add a current-main-aware restart broadcast helper that enumerates known agent session stores, selects recently interrupted sessions using an agreed timestamp/status contract, skips pseudo and automation sessions, enqueues the GatewayRestart system event plus heartbeat wake for each selected session, preserves the existing durable continuation queue for the triggering session, and adds focused regression tests.
