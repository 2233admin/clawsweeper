## Summary

Keep open. The feature is valid and useful, but this PR is now aimed at stale core web-search files while current main resolves `web_search` through provider-owned Brave plugin definitions. As written, a rebase would not expose or forward `goggles` through the active Brave provider, and the docs/changelog changes also need to move to the current published surfaces and policy shape.

## What This Changes

The PR adds an optional Brave-only `goggles` parameter to the old core `web_search` schema/runtime/cache path, updates Brave/web-search docs and tests, and adds a changelog entry.

## Best Possible Solution

Port the feature into `extensions/brave` as provider-owned behavior: expose a Brave-only per-query `goggles` string in the active tool schema, forward it to both Brave web and llm-context requests, include it in cache keys without lowercasing or missing-value collisions, cover the runtime behavior in Brave provider tests, and update the published docs plus a credited changelog entry.
