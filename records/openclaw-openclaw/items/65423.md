## Summary

Keep this PR open. Current main still builds embedded auth profile candidates in deterministic order, so the subagent thundering-herd problem is not implemented or superseded. The PR is a useful narrow fix candidate, but the branch needs revision because its tests duplicate the private shuffle helper and the implementation must preserve locked-profile and provider-preferred ordering semantics. Security review found no supply-chain-sensitive surface in the PR diff.

## What This Changes

The PR adds a Fisher-Yates shuffle for embedded-run auth profile candidates when `params.spawnedBy` indicates a subagent run, plus a new shuffle-focused test file.

## Best Possible Solution

Land a small production-owned candidate-selection helper used by `runEmbeddedPiAgent`: locked profiles stay single-candidate, top-level runs keep existing order, spawned automatic runs disperse candidates without displacing a provider-preferred first profile, and tests cover that production helper without probabilistic flakiness.
