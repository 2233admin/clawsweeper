## Summary

Keep this PR open. Current main still has the same observable gaps the PR targets: isolated cron rollover cleanup is skipped, isolated heartbeat and queued follow-up runs do not request run-end bundled MCP cleanup, the reply options contract cannot carry that flag, and the session MCP runtime manager still has stale in-flight cache deletion hazards. The diff is narrow and appears directionally correct, with no blocking review finding from this read-only pass.

## What This Changes

The PR adds one-shot bundled MCP cleanup propagation for isolated cron, isolated heartbeat, queued follow-up, and fallback runner paths, hardens session MCP runtime replacement/disposal ordering, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land a rebased narrow cleanup that disposes bundled MCP runtimes for isolated cron rollovers, isolated heartbeat runs, and queued follow-up one-shot runs, while preserving persistent-session behavior and covering the runtime replacement race with focused tests and the existing changelog credit for #68450.

## Reproduction Assessment

Yes. The source PR and this PR provide a concrete live A/B reproduction for accumulating bundled MCP connections under isolated cron and heartbeat load, and current-main code inspection shows the missing cleanup flag paths and rollover early return remain present.

## Solution Assessment

Yes, with normal maintainer review. The proposed flag propagation plus runtime-manager identity checks are a narrow fix for the observed one-shot lifecycle gaps and avoid the unrelated Matrix changes from #68450; a rebase and changed-gate run are the safer path than closing or starting a separate replacement.
