## Summary

Keep this issue open. Current main still exposes memory recall/read tools and prompt/search-result caps, but it does not implement the requested model-facing bounded memory CRUD tool with hard per-store character budgets, rejected over-limit writes, substring replace/remove, duplicate detection, capacity reporting, or credential-pattern screening. Related #70540 is an open draft and only covers a first `memory_add` slice, so it is not a shipped implementation or a full superseding fix.

## Best Possible Solution

Keep this open for a focused memory-core design. The best path is to add a structured model-facing memory write surface with config-backed per-store character budgets, atomic add/replace/remove semantics, duplicate detection, credential-pattern screening, and over-limit responses that include usage/current entries. If the limit is meant to be hard, the implementation also needs a shared enforcement guard so generic file writes, memory flush, and promotion/dreaming cannot bypass the same budget. #70540 can be reviewed as a first slice, but it should not close this issue unless the remaining hard-budget CRUD behavior is implemented or split into a canonical follow-up.
