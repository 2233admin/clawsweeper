## Summary

Current main still lacks the Feishu-specific `match.accountId` routing field and wildcard example, so this docs PR is not implemented or superseded. Keep it open, but revise the wording before merge to distinguish inbound binding account buckets from `channels.feishu.defaultAccount`, which the Feishu page documents as outbound API selection.

## What This Changes

This PR adds Feishu channel documentation for binding `match.accountId` semantics, including omitted-account behavior and an `accountId: "*"` wildcard routing example.

## Best Possible Solution

Keep this PR open and revise the Feishu docs patch. Add `match.accountId` to the Feishu routing fields, explain omitted `match.accountId` as the default routing bucket rather than all accounts, document explicit ids and `accountId: "*"`, and clearly distinguish that from `channels.feishu.defaultAccount` for outbound/default API account selection. Coordinate wording with #49769 while preserving a Feishu-specific example in `docs/channels/feishu.md`.
