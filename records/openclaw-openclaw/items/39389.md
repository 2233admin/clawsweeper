## Summary

Keep open. Current main still has the Feishu-specific docs gap this PR targets, but the proposed wording needs revision before merge because Feishu's `defaultAccount` is documented as outbound API selection while inbound routing uses the literal default account bucket unless `accountId: "*"` is set.

## What This Changes

The PR adds Feishu channel documentation for binding `match.accountId` semantics, including omitted-account behavior and an `accountId: "*"` routing example.

## Best Possible Solution

Revise the Feishu routing docs to list `match.accountId`, distinguish omitted default-bucket matching from explicit account ids and `accountId: "*"`, keep a Feishu-specific wildcard example, and explicitly separate inbound binding scope from `channels.feishu.defaultAccount` outbound account selection.
